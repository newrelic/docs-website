New Relic Infrastructure provides a collection of predefined alerts to help you monitor data from supported AWS integrations. Receive notifications for errors or fluctuations in performance, and refer to the runbook for each alert for recommended actions. To see the features of specific integrations and the data you can collect, see the AWS integrations list.

AWS API Gateway [#gateway-alerts]

Below are some recommended alerts, and actions for alert violations, for the AWS API Gateway service:

HTTP 5xx Errors

Condition name

HTTP 5xx Errors

Cloud entity type

AWS API Gateway API

Description

The API is generating server-side errors.

Recommended action

Review the API configuration, including the authorization service. Or you might need to rule out errors generated by the services and resources mapped to the API.

Reference

For more information, see Gateway response types.

AWS EBS [#ebs-alerts]

Below are some recommended alerts, and actions for alert violations, for the AWS EBS service:

Low Burst Balance

Condition name

Low Burst Balance

Cloud entity type

AWS EBS Volume

Description

The burst bucket has run out of I/O credits, for gp2 volumes, or throughput credits, for st1 and sc1 volumes.

This means your volume performance is limited to the baseline level.

Recommended action

Consider using a larger gp2 volume, with a higher baseline performance level, or switching to an io1 volume for workloads that require sustained IOPS performance greater than 10,000 IOPS.

Reference

For more information, see I/O Characteristics and Monitoring.

Queue Length

Condition name

Queue Length

Cloud entity type

AWS EBS Volume

Description

Some read and write operation requests are waiting to be completed.

Recommended action

Queue length must be correctly calibrated with I/O size and latency to avoid creating bottlenecks either on the guest operating system or on the network link to EBS.

Reference

For more information, see I/O Characteristics and Monitoring.

Cloud entity type

AWS File System

Description

The file system is close to reaching the I/O limit of the General Purpose performance mode.

Condition name

Cloud entity type

AWS S3 Bucket

Description

The bucket is generating server-side errors (returning HTTP 500 errors).

These errors might indicate that the AWS S3 service is unable to handle requests at that point in time, usually due to high number of requests.
