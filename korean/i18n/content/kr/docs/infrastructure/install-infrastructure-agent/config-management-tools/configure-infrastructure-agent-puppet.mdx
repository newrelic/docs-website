Puppet을 사용하여 `newrelic-infra` 모듈을 사용하여 New Relic의 인프라 에이전트를 설치 및 구성하는 방법을 알아보려면 계속 읽으십시오. Puppet 사용 방법에 대한 설명은 [Puppet 설명서](https://docs.puppet.com) 를 참조하십시오. 이것은 지역 사회가 지원하는 노력입니다.

여기에서는 이 구성 관리 도구를 사용하는 데 필요한 기본 정보를 제공합니다. 자세한 구성은 회사 표준을 따라야 합니다.

## 요구 사항

Infrastructure Puppet 모듈에는 다음과 같은 요구 사항이 있습니다.

* Puppet Enterprise 2021.5.x, 2021.4.x, 2021.3.x, 2021.2.x, 2021.1.x, 2021.0.x, 2019.8.x, 2019.7.x, 2019.5.x, 2019.4.x, 2019.3.x, 2019.2.x, 2019.1.x, 2019.0.x, 2018.1.x, 2017.3.x, 2017.2.x, 2017.1.x, 2016.5.x, 2016.4.x
* Puppet 4.0.0 이상 8.0.0 미만
* 데비안, CentOS, OpenSUSE, RedHat, SLES, Ubuntu, Windows, Amazon Linux

## 모듈 설명 [#description]

`newrelic-infra` 모듈을 사용하여 다음을 수행합니다.

* New Relic의 인프라 에이전트 패키지 저장소 소스를 추가합니다.
* New Relic 인프라 에이전트를 설치, 구성 및 관리합니다.

New Relic Puppet 모듈은 [Puppet Forge](https://forge.puppet.com/newrelic/newrelic_infra) 에서 사용할 수 있습니다.

## `newrelic-infra` 모듈 실행 [#beginning]

기본 `newrelic-infra` 모듈을 실행하려면 기본 `::agent` 클래스를 선언합니다.

## 모듈과 함께 인프라 에이전트 설치 [#usage]

`newrelic-infra` 과의 모든 상호작용은 기본 `agent` 클래스를 통해 수행됩니다.

Puppet을 사용하여 New Relic의 인프라 에이전트를 설치하려면 다음을 사용하십시오.

```
class { 'newrelic_infra::agent':
    ensure      => 'latest',
    license_key => 'YOUR_LICENSE_KEY',
}
```
