## 문제

[Microsoft Azure Cloud Services](/docs/agents/net-agent/azure-installation/install-app-azure-cloud-services) 를 통해 New Relic의 .NET 에이전트를 설치한 다음 NuGet 패키지를 설치했습니다. 일부 트래픽을 생성하고 앱에 대한 데이터 수집이 시작될 때까지 몇 분 정도 기다린 후에도 데이터는 여전히 APM 사용자 인터페이스에 표시되지 않습니다.

## 해결책

.NET 에이전트를 시작하고 앱에 연결하려면 하나 이상의 메트릭을 수집해야 합니다. New Relic이 계측하는 외부 호출 또는 데이터베이스 호출이 있는 경우 앱은 [APM **요약** 페이지](/docs/apm/applications-menu/monitoring/apm-overview-page) 에 데이터를 보고합니다. 또한 외부 호출에 대한 정보는 앱의 [서비스 맵](/docs/data-analysis/user-interface-functions/view-your-data/service-maps-visualize-monitor-apps-entire-architecture) 에 표시됩니다.

사용자 정의 계측이 없으면 `Worker` 역할은 데이터베이스 호출과 외부 호출만 보고합니다. ( `Worker` 역할 [계측은 사용자 지정 애플리케이션 계측](/docs/agents/net-agent/instrumentation/instrumenting-non-iis-apps) 과 유사합니다.) 다른 세부 정보를 보려면 New Relic의 [.NET API](/docs/agents/net-agent/features/net-agent-api) 를 사용하여 사용자 지정 측정항목을 수집하고 [사용자 지정 대시보드](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards) 에서 확인해야 합니다.
