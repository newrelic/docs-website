[속성](/docs/features/agent-attributes) 은 이벤트 또는 트랜잭션의 속성을 결정하는 정보를 포함하는 키-값 쌍입니다. 이러한 키-값 쌍을 사용하면 애플리케이션에 대한 통찰력을 높이고 New Relic의 데이터에 주석을 추가할 수 있습니다. 기본 및 [사용자 지정](/docs/agents/manage-apm-agents/agent-data/collect-custom-attributes) 속성은 모두 APM 트랜잭션 추적, 분산 추적 및 오류 분석에서 볼 수 있습니다. 및 [APM](/docs/insights/explore-data/attributes/apm-default-attributes-insights) 및 [브라우저 이벤트](/docs/insights/explore-data/attributes/browser-default-attributes-insights#browser-attributes-table) .

이러한 각 대상으로 보낼 속성을 정확히 사용자 지정할 수 있습니다. PHP 에이전트 속성, 활성화 또는 비활성화 방법, 에이전트가 [대상](/docs/features/agent-attributes#destinations) 에 포함하거나 제외할 속성을 결정하기 위해 따르는 규칙에 대해 알아보려면 계속 읽으십시오.

## 속성이 전송되는 위치 변경 [#change-attribute-destination]

<Callout variant="tip">
  `newrelic.browser_monitoring.attributes.enabled` 을 제외한 모든 대상은 기본적으로 PHP 에이전트에서 속성 수집에 열려 있습니다.
</Callout>

New Relic [대상](/docs/apm/other-features/attributes/agent-attributes#destinations) 으로 전송되는 속성을 변경하려면:

1. [`.enabled` 대상 속성](/docs/agents/php-agent/attributes/enabling-disabling-attributes#destination-on-off) 을 변경하여 속성 컬렉션에 대한 대상을 열거나 닫습니다.

2. 대상의 [`.include` 또는 `.exclude` 속성](/docs/agents/php-agent/attributes/enabling-disabling-attributes#destination-attribute-specific) 에 속성 이름을 추가하여 [기본 속성 설정](#attributes) 을 변경합니다.

   설정이 서로 대체되는 [속성 규칙](/docs/agents/php-agent/attributes/enabling-disabling-attributes#attruls) 을 참조하세요.

## PHP 에이전트 업그레이드 [#upgrading]

PHP 에이전트 **4.9** 이상으로 업그레이드할 때 구성 파일을 업그레이드하십시오. 더 이상 사용되지 않는 속성에 대한 자세한 내용은 속성 [활성화 및 비활성화](/docs/php/php-agent-enabling-and-disabling-attributes#deprecated) 를 참조하십시오.
