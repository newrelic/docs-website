title: 역방향 프록시 문제
metaDescription: How to configure apache2 reverse proxy to work with Gitlab

---

CodeStream에서 병합 요청을 열 때 `404 Not Found` 오류가 발생할 수 있습니다. GitLab 그룹/프로젝트 URL은 기본 apache2 역방향 프록시에서 처리되지 않는 인코딩된 슬래시로 생성됩니다. apache2가 이러한 URL을 성공적으로 확인할 수 있도록 하려면 apache2 구성에 다음을 추가하세요.

* AllowEncodedSlashes 추가
* ProxyPass 줄 끝에 `nocanon` 추가

예를 들어:

```
AllowEncodedSlashes NoDecode
ProxyPass / http://33.211.55.3:9000/ nocanon
```
