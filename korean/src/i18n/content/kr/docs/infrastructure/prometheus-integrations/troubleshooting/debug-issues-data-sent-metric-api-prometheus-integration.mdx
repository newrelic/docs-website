## 문제

New Relic의 [Metric API](/docs/data-ingest-apis/get-data-new-relic/metric-api/introduction-metric-api) 로 전송되는 데이터에 대해 보고된 문제의 원인을 식별하는 데 문제가 있습니다. Docker 또는 Kubernetes용 Prometheus OpenMetrics 통합에서 Metric API로 전송되는 정확한 데이터를 확인하려고 합니다.

## 해결책

표준 출력 스트림으로 전송되는 원시 데이터를 표시하도록 Prometheus OpenMetrics 통합을 구성할 수 있습니다. Metric API로 전송되는 정확한 데이터를 보려면 `EMITTERS` 환경 변수를 `"api,stdout"` 로 설정합니다.
