## 문제

PHP 에이전트의 [APM **요약**](/docs/apm/applications-menu/monitoring/apm-overview-dashboard) 페이지에서 **호스트** 테이블의 PHP 에이전트 인스턴스 수는 얼마입니까? 이 수치가 변하는 이유는 무엇이며 불일치가 나타나는 이유는 무엇입니까?

## 원인

**요약** 페이지에 나열된 각 서버에는 인스턴스 수가 포함됩니다. 예를 들어 **42개의 앱 인스턴스** . 인스턴스 수는 마지막으로 보고된 수집에서 PHP 에이전트가 사용한 열린 파일 설명자의 최대 수를 나타냅니다. 이러한 파일 설명자는 개방형 I/O 소켓이며 에이전트/데몬 통신 채널을 반영합니다.

명령줄, 백그라운드 또는 외부 프로세스가 실행 중인 경우 인스턴스 수가 증가합니다. 이 숫자는 처리량이 많은 경우에도 영향을 받을 수 있습니다. 예를 들어 Apache 서버는 증가된 로드를 수용하기 위해 연결 풀을 늘리도록 구성할 수 있습니다.

서버 간에 예상되는 값의 차이는 한 서버에서는 실행되고 다른 서버에서는 실행되지 않는 백그라운드 작업을 반영할 수 있습니다. 예기치 않은 수는 로드 밸런서가 잘못 구성되었음을 나타낼 수도 있습니다.
