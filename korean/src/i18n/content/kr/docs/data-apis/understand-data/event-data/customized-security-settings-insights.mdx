기본적으로 New Relic 제품은 UI 차트에 사용된 다양한 데이터를 보고하며 [쿼리](/docs/using-new-relic/data/understand-data/query-new-relic-data) 에 사용할 수 있습니다. 당사 제품은 민감한 정보를 전송하도록 명시적으로 조정되지 않는 한 민감한 정보를 전송하지 않습니다.

## 기본 이벤트 및 속성 [#default-events]

당사 제품은 일련의 [기본 이벤트 및 속성](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products) 을 보고합니다. 누군가가 구성을 통해 명시적으로 활성화하지 않는 한 기본 세트에 없는 요청 매개변수 또는 기타 속성을 보내지 않습니다.

## 보고된 데이터 조정 [#default-security]

New Relic 제품에 대한 보안 설정을 평가할 때 [기본 이벤트 및 속성](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products) 을 검토하십시오. 기본 속성에는 민감한 데이터가 포함되어 있지 않습니다. 일반적으로 효과적인 성능 모니터링에 필요한 데이터일 뿐입니다. 기본 보안 설정을 변경하지 않는 한 당사 제품은 다른 데이터를 전송하지 않습니다.

요구 사항에 따라 다음 상황 중 하나 또는 모두가 적용될 수 있습니다.

* 기본 목록에 우려되는 데이터가 포함된 경우 해당 속성이 수집되지 않도록 비활성화할 수 있습니다. 편집 방법은 사용 중인 제품의 설명서를 참조하세요.
* 기본적으로 보고되지 않은 속성을 보내야 하는 경우 [해당 속성이 보고되도록](/docs/insights/insights-data-sources/custom-data/send-custom-data-insights) 설정할 수 있습니다. 이 경우 [높은 보안 모드](/docs/accounts-partnerships/accounts/security/high-security) 를 사용하지 마십시오. 이렇게 하면 사용자 지정 속성을 수집하는 기능이 비활성화됩니다.
