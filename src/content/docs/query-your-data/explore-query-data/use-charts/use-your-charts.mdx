Once you've created a chart, you can customize the appearance of it to best present the data. You can share a chart in different formats and add a chart to a new or existing dashboard.

## Change the appearance of your chart [#change-appearance]

When you run your query in NRQL mode or view your chart while using metrics and events to specify data, New Relic analyzes your data and applies a [chart type](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-types) that fits your data.

There are several options of chart types to choose from. To change the chart type, use the <DNT>**Chart type**</DNT> menu to the right of the current chart.

## Customize your charts [#customize-charts]

While we try our best to optimize how we display your data, sometimes you may have other needs. Depending on the chart type, there are additional customization options available.

<Callout variant="important">
  The system used for representing byte units (such as KB, MB, GB, TB, etc.) in these charts follows the metric representation. This means that:

  * 1 KB (kilobyte) = 1,000 bytes
  * 1 MB (megabyte) = 1,000 KB
  * 1 GB (gigabyte) = 1,000 MB
  * 1 TB (terabyte) = 1,000 GB

  This metric system is in line with the International System of Units (SI), which uses powers of 10 for calculations and representation.
</Callout>

Read the sections below to learn more about the customization features. You can also see [this blog](https://newrelic.com/blog/how-to-relic/dashboard-chart-customizations) and [companion video](https://www.youtube.com/watch?v=LA2jM_asl94) to get a guided walkthrough for a few of the customization options.

**To access chart customization options**:

1. On the widget you want to customize, from the <Icon name="fe-more-horizontal"/> menu, select <DNT>**Edit**</DNT>.
2. In the <DNT>**Customize this visualization**</DNT> panel, configure the options described in the sections below.

<CollapserGroup>
  <Collapser
    id="widget-link-description"
    title="Widget links and descriptions"
  >
    You can add links and descriptions to your dashboard widgets to provide additional context and navigation options.

    **Widget links** appear as clickable links on widget titles, allowing users to navigate to related dashboards, documentation, or external resources. When users view the dashboard, clicking the widget title opens the configured link.

    **Widget descriptions** provide additional context about what a widget displays, shown as a tooltip when users point to the widget.

    **To add a link or description**:

    1. To add a link, in the <DNT>**URL**</DNT> field, enter a URL. Note the following requirements:

       * Maximum length: 2,048 characters
       * Supported protocols: `http://` and `https://` only
       * Invalid URLs will display a validation error.

    2. To add a description, in the <DNT>**Description**</DNT> field, enter a plain text description.
    3. Click <DNT>**Apply changes**</DNT>.

    **To remove a link or description**:

    1. Clear the <DNT>**URL**</DNT> or <DNT>**Description**</DNT> field.
    2. Click <DNT>**Apply changes**</DNT>.

    **To add, edit, or remove widget links and descriptions via JSON**:

    1. From the dashboard <Icon name="fe-more-horizontal"/> menu, select <DNT>**Manage JSON**</DNT>.
    2. In the JSON editor, locate the widget you want to modify and update the `link` and `description` fields. To add or edit, enter a value. To remove, set the field to `null`:

       ```json
       widgets: [{
         id: "WIDGET_ID"
         link: "https://example.com"  // Enter URL or null to remove
         description: "Your widget description"  // Enter text or null to remove
       }]
       ```

    3. Click <DNT>**Save changes**</DNT>.

    If you encounter issues with widget links not working as expected, see [Widget link issues](/docs/query-your-data/explore-query-data/dashboards/troubleshooting-chart-errors/#widget-link-issues) for troubleshooting help.
  </Collapser>

  <Collapser
    id="custom-date-time"
    title="Format date and time"
  >
    Customize the date and time format for tables and billboards from the <DNT>**Data format**</DNT> section. You have the flexibility to leave it as is or modify the format to suit your needs. Choose from the following options:

  
    * <DNT>**Numeric**</DNT>: Retains the timestamp as numeric values.
    * <DNT>**Date**</DNT>: Displays dates in a standard format.
    * <DNT>**Duration**</DNT>: Shows the time span between the timestamp and the present moment.
    * <DNT>**Time to Now**</DNT>: Indicates the time elapsed since the timestamp.

    Select the most suitable format to configure data presentation based on your needs.
  </Collapser>

  <Collapser
    id="customize-y-axis"
    title="Customize the Y axis"
  >
    To display data within certain values on line charts and area charts, in the <DNT>**Y-axis**</DNT> section, set a minimum and maximum value for the axis.

    If no customization option is selected, [dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) automatically displays the full Y axis from 0 to the top value plus a margin.

    To disable or enable the legend for line charts, area charts, and histograms, in the <DNT>**Legend**</DNT> section, use the <DNT>**Visibility**</DNT> toggle.
  </Collapser>

  <Collapser
    id="bar-charts-pie-charts-tables"
    title="Bar charts, pie charts, and tables"
  >
    If the number of [facets](/docs/query-your-data/explore-query-data/dashboards/filter-new-relic-one-dashboards-facets) on a query is larger than 5,000 for bar charts, pie charts, or tables, the <DNT>**Other groups**</DNT> facet aggregates the rest of the facets. To show or hide the <DNT>**Other groups**</DNT> option, in the <DNT>**Other groups**</DNT> section, use the <DNT>**Visible**</DNT> toggle.

  </Collapser>

  <Collapser
    id="dash-color"
    title="Color customization"
  >
    You can customize the color of your charts per series in area, bar, line, pie, and stacked bar charts.

    To customize the color per series, in the <DNT>**Colors**</DNT> section, select the <Icon name="fe-plus"/> icon, choose the series you want to change, and select the desired color.
  </Collapser>

  <Collapser
    id="dash-units"
    title="Units customization"
  >
    You can customize the unit on your Y axis and in each of your series. It is available in area, line, and stacked bar charts.

    To customize the unit, in the <DNT>**Units**</DNT> section, select the <Icon name="fe-plus"/> icon, choose the series that needs another unit, and select the unit.
  </Collapser>

  <Collapser
    id="dash-null"
    title="Null value display options"
  >
    Time series charts sometimes look broken when there is a zero in the event attribute being queried. However, a null value may not always represent the situation correctly.

    You can choose an option for displaying null values to make your time series charts easier to understand. They include:

    * Leave as null (default): Null values show as gaps in the chart, which can result in disconnected lines. You may want to use this option if it's important to find gaps in reporting, or when it doesn't make sense to connect a continuous line.
    * Remove the value: Null points won't exist in the chart. For example, a chart that displays a data point per hour in a day would show fewer than 24 points.
    * Preserve the last value: Null values will keep the value from the previous period. For example, if a chart reports cumulative sales each hour, but is missing an hour's data, the missing hour would reflect the same number of sales as the previous hour.
    * Transform the value to zero: The missing data shows up as a zero. This option could be useful when no data means there is no meaningful transaction, count, or usage.

      <img
        title="Null value chart options"
        alt="Null value chart options"
        src="/images/dashboards_screenshot-crop_null-values.webp"
      />

      <figcaption>
        You can choose from four ways to display null values in your charts.
      </figcaption>

    To customize how your null values appear, in the <DNT>**Null values**</DNT> section, select the option to apply. To use different options for each series, select the <Icon name="fe-plus"/> icon beside <DNT>**Null values per series**</DNT> to configure them.
  </Collapser>

  <Collapser
    id="thresholds"
    title="Thresholds"
  >
    Thresholds are visual cues that add context to your charts and tables, making data interpretation clearer. Here's how they can elevate your visualizations:

    * Boundary lines: Establish minimum and maximum acceptable values in time series charts.

    * Colored regions: Shade areas between thresholds to visually identify normal operating ranges or highlight critical zones.

    * Conditional cell coloring: Apply color codes to table cells based on their values, instantly revealing data outliers or trends.

      <Video
        type="wistia"
        id="ikpu24sqjr"
      />

    * In time series charts:

      1. Edit the chart and expand the <DNT>**Thresholds**</DNT> menu.
      2. Select <DNT>**+**</DNT> to add a threshold and enter a <DNT>**Name**</DNT>.
      3. Enter values for <DNT>**From**</DNT> and <DNT>**To**</DNT>. If you want to create a single line, enter the same number for both fields.
      4. Select a severity level. This determines the color of the line or region.

         <img
           title="Configure chart thresholds in time series charts"
           alt="Screenshot of how to configure thresholds in a time series chart"
           src="/images/charts_screenshot-full_chart-threshold-time-series.webp"
         />

    * In tables:

      1. Edit the chart and expand the <DNT>**Thresholds**</DNT> menu.
      2. Select <DNT>**+**</DNT> to add a threshold and select a <DNT>**Column**</DNT>.
      3. Enter values for <DNT>**From**</DNT> and <DNT>**To**</DNT>. To set an upper bound, leave the <DNT>**From**</DNT> field empty. To set a lower bound, leave the <DNT>**To**</DNT> field empty.

         <img
           title="Configure chart thresholds in tables"
           alt="Screenshot of how to configure thresholds in a table"
           src="/images/charts_screenshot-full_chart-threshold-table.webp"
         />
  </Collapser>
</CollapserGroup>

Most charts have various options, including a chart-embed option, getting a chart as an image, and adding a chart to a dashboard.
