## Widget links and descriptions issues [#widget-link-issues]

If you're experiencing issues with widget links or descriptions, check the following common problems and solutions.

<table>
  <thead>
    <tr>
      <th style={{ width: "400px" }}>
        <DNT>
          **Problem**
        </DNT>
      </th>

      <th style={{ width: "400px" }}>
        <DNT>
          **Solution**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Widget link is not clickable
      </td>

      <td>
        Verify the URL is valid and uses a supported protocol (`http://` or `https://`). Links with other protocols aren't supported.
      </td>
    </tr>

    <tr>
      <td>
        Description tooltip not appearing
      </td>

      <td>
        Ensure the description field contains text. Point to the widget title area to display the tooltip. If it still doesn't appear, try refreshing the dashboard.
      </td>
    </tr>

    <tr>
      <td>
        URL validation error
      </td>

      <td>
        Check that the URL doesn't exceed 2,048 characters and uses only `http://` or `https://` protocols.
      </td>
    </tr>

    <tr>
      <td>
        Link opens to 404 error
      </td>

      <td>
        Verify the target URL is correct and accessible. External links must be publicly available or accessible within your network.
      </td>
    </tr>

    <tr>
      <td>
        Cannot edit widget link or description
      </td>

      <td>
        Confirm you have edit permissions for the dashboard. Only users with appropriate permissions can modify widget settings.
      </td>
    </tr>

    <tr>
      <td>
        API mutation returns no errors but changes don't appear
      </td>

      <td>
        Refresh the dashboard page. If changes still don't appear, verify the widget ID is correct and the mutation completed successfully using the NerdGraph API explorer.
      </td>
    </tr>
  </tbody>
</table>

## Hidden variable issues [#hidden-variable-issues]

If you're experiencing issues with hidden template variables, check the following common problems and solutions.

<table>
  <thead>
    <tr>
      <th style={{ width: "400px" }}>
        <DNT>
          **Problem**
        </DNT>
      </th>

      <th style={{ width: "400px" }}>
        <DNT>
          **Solution**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Can't see a variable you need
      </td>

      <td>
        Enter Edit mode to see all variables, including hidden ones. If you need to interact with the variable regularly, consider unhiding it.
      </td>
    </tr>

    <tr>
      <td>
        Accidentally hid a variable
      </td>

      <td>
        Enter Edit mode, find the variable with the visibility-off icon, and toggle the <DNT>**Hide on variables bar**</DNT> setting off.
      </td>
    </tr>

    <tr>
      <td>
        Hidden variables not working in API calls
      </td>

      <td>
        Verify you're using the `hiddenOnVariablesBar` field in the `options` object. This field should be available in the `VariableOptions` type.
      </td>
    </tr>

    <tr>
      <td>
        Want to hide multiple variables at once
      </td>

      <td>
        Currently, you need to hide variables one at a time through the UI. For bulk operations, use the [NerdGraph dashboards API](/docs/apis/nerdgraph/examples/nerdgraph-dashboards/#variable-visibility) to update multiple variables programmatically.
      </td>
    </tr>
  </tbody>
</table>
