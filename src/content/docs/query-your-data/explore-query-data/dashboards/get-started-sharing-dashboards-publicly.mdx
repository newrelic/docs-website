As a New Relic user, you may need to share real-time monitoring data with stakeholders—customers, business partners, or executives—who don't have a New Relic account. Setting up public dashboards involves multiple configuration steps across account policies, access management, and role creation, which can be difficult to piece together from separate reference pages.

Public dashboards lets you share live, read-only views of your dashboards via secure URLs. This tutorial walks you through the complete end-to-end setup, from enabling the policy to generating your first shareable URL.

<Callout variant="important">
  The dashboard sharing feature is available only to **Pro** and **Enterprise** customers with [Advanced Compute](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing) enabled. Chart sharing is available to **Pro**, **Enterprise**, and **Core Compute** customers and does not require Advanced Compute.
</Callout>

## Pricing [#pricing]

<DNT>**Public Dashboards**</DNT> is an Advanced Compute Product feature offered as an [add-on](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing) or as a part of the Compute pricing model. Advanced [CCUs](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#compute-capacity-unit) are consumed when queries run from the dashboard visualizations, which happens in the following cases:

- On page load
- On time picker changes
- On refresh-rate triggers

You can monitor your usage from **[New Relic Administration](https://one.newrelic.com/admin-portal) > Consumption Management > Advanced capabilities**.

## Setup steps [#setup-steps]

<Steps>

<Step>

### Before you begin [#prerequisites]

Confirm the following before proceeding:

- Your organization has [Advanced Compute](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing) enabled. You can't share dashboards publicly without it.
- The person performing Steps 2 and 3 has the [Authentication domain manager](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings) organization-scoped role.
- The dashboard you want to share is not set to **Private**. To verify, go to **[one.newrelic.com > Dashboards](https://one.newrelic.com/dashboards)** > your dashboard > <Icon name="fe-more-horizontal"/> > **Settings > Permissions**.

</Step>

<Step>

### Enable the Live URL Creation policy [#enable-policy]

An Authentication Domain Manager must enable the **Live URL Creation** policy for each account that sources data to the dashboard. This is a one-time setup per account.

Use the [NerdGraph API explorer](https://one.newrelic.com/nerdgraph-graphiql) to run the following mutation. Replace `1, 2` with the actual account IDs:

```graphql
mutation {
  dashboardUpdateLiveUrlCreationPolicies(
    policies: {
      accountIds: [1, 2],
      liveUrlCreationAllowed: true
    }
  ) {
    liveUrlCreationPolicies {
      accountId
      liveUrlCreationAllowed
    }
  }
}
```

A successful response confirms `liveUrlCreationAllowed: true` for each account. To check the current policy status before enabling it, see [Policy enablement](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#policy-enablement).

</Step>

<Step>

### Configure user access [#configure-access]

An Authentication Domain Manager must create a custom role with live URL permissions and assign the user to a group with access to the relevant accounts.

1. **Create a custom role.** Go to **[New Relic Administration](https://one.newrelic.com/admin-portal) > Access Management > Roles** and [create a custom role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) with the following capabilities based on your user's needs:
   - `Live Url (individual) > Modify`: Lets the user create live URLs and update their properties.
   - `Live Url (individual) > Delete`: Lets the user revoke their own live URLs.
   - `Live Url (all) > View`: Lets the user view URLs created by all users in the account.
   - `Live Url (all) > Delete`: Lets the user revoke URLs created by any user in the account.

2. **Add the user, role, and account to a group.** Go to **[New Relic Administration](https://one.newrelic.com/admin-portal) > Access Management > Groups** and [create a group or manage an existing one](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#group-access):
   - Under **Members**, add the user.
   - Under **Account access**, associate the custom role with each account that sources data to the dashboard.

<Callout variant="tip">
  If the dashboard pulls data from multiple accounts, add the custom role for each of those accounts under **Account access** in the group.
</Callout>

For a detailed walkthrough with screenshots, see [Access management](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#access-management).

</Step>

<Step>

### Create and share a live URL [#create-url]

Once the policy is enabled and user access is configured, the dashboard owner can create a live URL.

<Callout variant="caution">
  If a shared dashboard isn't password-protected, anyone with the live URL can view all of its content. Enable password protection to limit access to your intended recipients.
</Callout>

1. Go to **[one.newrelic.com > Dashboards](https://one.newrelic.com/dashboards)**.
2. Open the dashboard you want to share.
3. In the top-right corner, go to **Share** > **Share dashboard**.
4. Click **Continue**. In the **Share dashboard** dialog, make adjustments as needed:
   - The **Protect with a password** toggle is on by default. Turn it off only if password protection isn't required.
   - Select a link expiration duration from the **Link expiration** drop-down. The default is 30 days.
5. Click **Save changes**.
6. Copy the live URL and password (if enabled) and share them with your external users.

<Callout variant="important">
  Passwords can't be retrieved after the initial display. Store the password securely before closing the dialog. For guidance, see [Password management best practices](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#password-management).
</Callout>

You can also create and manage live URLs programmatically via the [NerdGraph API](/docs/apis/nerdgraph/examples/manage-live-dashboard-urls-via-api).

</Step>

</Steps>

## After setup [#after-setup]

With the live URL created and shared, here are the common management tasks:

- **Update URL expiration**: Go to **Share** > **Manage public link** > update the **Link expiration** drop-down > **Save changes**.
- **Reset the password**: Go to **Share** > **Manage public link** > turn on **Reset password** > **Save changes**. Share the new password with your external users.
- **Revoke a URL**: Go to **Share** > **Manage public link** > **Revoke public link** and confirm, or revoke from **Administration > Shared Dashboards**.
- **View all shared URLs**: Go to **[one.newrelic.com](https://one.newrelic.com)** > your name > **Administration > Shared Dashboards**.
- **Monitor audit events**: Query the `NrAuditEvent` table to track URL creation, updates, and revocations. See [Audit trail and logging](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#audit-trail-dashboard).
