New Relic Monitoring for SAP Solutions captures SAP telemetry data (Metrics, Events, Logs, and Traces) and sends it to the New Relic platform for analysis and visualization.

## Central monitoring system vs. monitored systems

SAP systems that provide telemetry data are known as monitored systems. The central monitoring system is where <DNT>**NRM4SAP**</DNT> is installed—it collects, transforms, and transmits telemetry data. Typically, you install <DNT>**NRM4SAP**</DNT> on a single SAP system such as <DNT>**SAP Solution Manager (SolMan)**</DNT>. <DNT>**NRM4SAP**</DNT> has no functional dependencies on SolMan—it is used here only as a reference point. <DNT>**NRM4SAP**</DNT> uses standard SAP data sources like <DNT>**ST-PI**</DNT>, MTE Classes (components within <DNT>**CCMS**</DNT>), and performance data from <DNT>**STAD/ST03N**</DNT>, so you do not need to install additional agents on monitored systems.

Starting with version 3.0, <DNT>**NRM4SAP**</DNT> only requires ABAP components, which are deployed exclusively on the central monitoring system.

## Data flow architecture

<DNT>**RFC**</DNT> calls pull SAP telemetry data into the central monitoring system. The <DNT>**ABAP Enterprise Connector**</DNT> processes the data and transmits it securely to New Relic via <DNT>**HTTPS**</DNT>.

## What's included

After installation, you can use out-of-the-box dashboards and alerts:

- **Management visibility**: View SAP system health and performance alongside other IT systems in centralized dashboards.

- **Custom monitoring**: Tailor monitoring to specific business needs, such as tracking integration points in order processing workflows. Build custom dashboards and visualizations using collected telemetry data.

- **Proactive alerting**: Set alerts to receive immediate notifications of service disruptions. This helps you maintain continuity and promptly address issues.
