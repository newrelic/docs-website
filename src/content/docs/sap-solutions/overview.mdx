New Relic Monitoring for SAP Solutions captures SAP telemetry data (Metrics, Events, Logs, and Traces) and sends it to the New Relic platform for analysis and visualization.

## Central monitoring system vs. monitored systems

SAP systems that provide telemetry data are known as monitored systems. The central monitoring system is where NRM4SAP is installed—it collects, transforms, and transmits telemetry data. Typically, you install NRM4SAP on a single SAP system such as SAP Solution Manager (SolMon). NRM4SAP utilizes standard SAP data sources like <DNT>**ST_PI**</DNT>, <DNT>**CCMS**</DNT>, and the performance database, removing the need for additional agent installations on monitored systems.

## Data flow architecture

RFC calls pull SAP telemetry data into the central monitoring system. The ABAP Enterprise Connector processes the data and transmits it securely to New Relic via HTTPS.

This architecture provides centralized visibility into SAP system performance for monitoring and troubleshooting.

## Use cases

SAP systems are essential for enterprise business processes including financials, logistics, material management, warehousing, sales, and distribution. NRM4SAP provides visibility into these SAP operations to monitor their health and performance.

### Integration and issue resolution

Integrate NRM4SAP to monitor SAP systems that support key business operations like e-commerce order processing and warehouse management. NRM4SAP provides observability into SAP system operations, enabling you to detect and resolve issues before they impact business processes.

### Post-installation and next steps

After installation, customers can immediately leverage out-of-the-box dashboards and alerts provided by NRM4SAP, yielding several advantages:

- **Management visibility**: View SAP system health and performance alongside other IT systems in centralized dashboards.

- **Custom monitoring**: Tailor monitoring to specific business needs, such as tracking integration points in order processing workflows. Build custom dashboards and visualizations using collected telemetry data.

- **Proactive Alerting**: By setting crucial alerts, administrators can be immediately warned of service disruptions, enabling them to maintain continuity and promptly address any issues.

Use NRM4SAP to monitor and manage your SAP environment.
