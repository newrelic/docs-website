This technical reference lists all data providers available in New Relic Monitoring for SAP Solutions (NRM4SAP). Use this when configuring specific monitoring groups. Data providers are organized by monitoring groups, which you select when [configuring monitored systems](/docs/sap-solutions/install-and-configure/configure-new-relic-abap-add-on-package/#connect-systems).

<Callout variant="tip">
  **Note**: These tables contain detailed technical specifications. Most users only need to reference the **DP_NAME** and **OUTPUT_TYPE** columns.
</Callout>

## Understanding data provider attributes

Each data provider has the following technical attributes:

- **MANDT**: Client number (typically 001)
- **DP_NAME**: Unique identifier for the data provider
- **DP_OUTPUT_TYPE**: Type of telemetry data (METRIC, EVENT, LOG, TRACE, CONFIG, BTP-LOG)
- **COMMUNICATOR**: ABAP class responsible for data collection coordination
- **DATAPROVIDER**: ABAP class that retrieves the data
- **FINISHER**: ABAP class that processes and transmits the data
- **DP_STATUS**: Status indicator (S = Suspended, blank = Active)
- **DP_GROUP**: Monitoring group assignment
- **METRICTYPE**: For metrics, indicates the aggregation type (GAUGE, COUNT, SUMMARY)
- **PERIOD**: Collection frequency numeric value
- **PERIOD_UNIT**: Time unit for collection (MIN, HRS, DAY)
- **SKIP_RT**: Skip runtime flag (X = Yes)
- **RESERVED**: Reserved for future use

<Callout variant="tip">
  Tables in this reference can be scrolled horizontally to view all columns. Data providers marked with "S" in the STATUS column are suspended by default.
</Callout>
