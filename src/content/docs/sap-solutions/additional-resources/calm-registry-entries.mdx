Configure registry entries (Call IDs) for CloudALM data providers to enable monitoring of SAP Cloud ALM services.

## Overview

This utility program provides functionality to add, change, delete, and autogenerate configurations for CloudALM data providers.

- **Transaction Code**: `/NEWR/GEN_DP_CALLID`
- **Program Name**: `/NEWR/GENERATE_CONFIG_CALLID`
- **Configuration Table**: `/NEWR/CALLID_CNF`

## Prerequisites

Ensure the data provider `NR_CALM_LANDSCAPE` is active and collecting data before you execute this utility.

## Setup procedure

<Steps>
  <Step>
    ## Configure data provider and service type mapping

    1. Execute transaction `/NEWR/GEN_DP_CALLID`.
    2. Click the **DP/CALM Services Type Maintenance Link**.
    3. Select the service types you want to monitor.
    4. Link each data provider to the appropriate SAP system or service configured in SAP Cloud ALM.
    5. Save the configuration.

    <Callout variant="tip">
      Service type represents SAP systems or services enabled in your SAP Cloud ALM environment.
    </Callout>
  </Step>

  <Step>
    ## Add new Call ID records

    1. In the selection screen, enter the data provider and service type you need to configure.
    2. Select the radio button **Add new (Call ID) records to Registry**.
    3. Click **Execute**.
    4. The system displays all service type instances configured in Cloud ALM, each with a unique LMS ID.
    5. Select the instances you want to monitor.
    6. Delete any entries not needed.
    7. Click **Update and save configuration**.
  </Step>

  <Step>
    ## Verify the configuration

    1. Go to transaction `/NEWR/REGISTRY`.
    2. Select the data provider.
    3. Click **Configure** to view the saved Call ID entries.
    4. Click **Test** to verify the configuration.

    **Expected result**: The connection test shows successful status.
  </Step>
</Steps>

## Maintenance options

### Update existing entries

If you need to change or delete existing Call ID records:

1. Execute transaction `/NEWR/GEN_DP_CALLID`.
2. Select **Copy/Change/Delete existing (Call ID) records in Registry**.
3. Click **Execute**.
4. Make the required changes.
5. Save the configuration.

### Clear all entries

To remove all Call ID records (for initial setup or service pack upgrades):

1. Execute transaction `/NEWR/GEN_DP_CALLID`.
2. Select **Backup & delete all existing (Call ID) records in Registry**.
3. Click **Execute**.

<Callout variant="important">
  This creates a backup before deletion. Use this option only during initial setup or when instructed during a service pack upgrade.
</Callout>

### Refresh service definitions

If you cannot locate a service type or LMS ID:

1. Click **Retrieve new services (LMSID) setup in Cloud ALM**.
2. This fetches the latest services configured in your Cloud ALM account.
