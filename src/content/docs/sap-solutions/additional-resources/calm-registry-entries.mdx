Configure registry entries (Call IDs) for CloudALM data providers to enable monitoring of SAP Cloud ALM services.

## Overview

This utility program provides functionality to add, change, delete, and autogenerate configurations for CloudALM data providers.

- <DNT>**Transaction Code**</DNT>: <DNT>**/NEWR/GEN_DP_CALLID**</DNT>
- <DNT>**Program Name**</DNT>: <DNT>**/NEWR/GENERATE_CONFIG_CALLID**</DNT>
- <DNT>**Configuration Table**</DNT>: <DNT>**/NEWR/CALLID_CNF**</DNT>

## Prerequisites

Ensure the data provider <DNT>**NR_CALM_LANDSCAPE**</DNT> is active and collecting data before you execute this utility.

## Setup procedure

<Steps>
  <Step>
    ## Configure data provider and service type mapping

    1. Execute transaction `/NEWR/GEN_DP_CALLID`.
    2. Click the <DNT>**DP/CALM Services Type Maintenance Link**</DNT>.
    3. Select the service types you want to monitor.
    4. Link each data provider to the appropriate SAP system or service configured in SAP Cloud ALM.
    5. Save the configuration.

    <Callout variant="tip">
      <DNT>**Service type**</DNT> represents <DNT>**SAP systems**</DNT> or <DNT>**services**</DNT> enabled in your <DNT>**SAP Cloud ALM**</DNT> environment.
    </Callout>
  </Step>

  <Step>
    ## Add new Call ID records

    1. In the selection screen, enter the data provider and service type you need to configure.
    2. Select the radio button <DNT>**Add new (Call ID) records to Registry**</DNT>.
    3. Click <DNT>**Execute**</DNT>.
    4. The system displays all <DNT>**service type instances**</DNT> configured in <DNT>**Cloud ALM**</DNT>, each with a unique <DNT>**LMS ID**</DNT>.
    5. Select the instances you want to monitor.
    6. Delete any entries not needed.
    7. Click <DNT>**Update and save configuration**</DNT>.
  </Step>

  <Step>
    ## Verify the configuration

    1. Go to transaction `/NEWR/REGISTRY`.
    2. Select the <DNT>**data provider**</DNT>.
    3. Click <DNT>**Configure**</DNT> to view the saved <DNT>**Call ID entries**</DNT>.
    4. Click <DNT>**Test**</DNT> to verify the configuration.

    **Expected result**: The connection test shows successful status.
  </Step>
</Steps>

## Maintenance options

### Update existing entries

If you need to change or delete existing <DNT>**Call ID records**</DNT>:

1. Execute transaction `/NEWR/GEN_DP_CALLID`.
2. Select <DNT>**Copy/Change/Delete existing (Call ID) records in Registry**</DNT>.
3. Click <DNT>**Execute**</DNT>.
4. Make the required changes.
5. Save the configuration.

### Clear all entries

To remove all <DNT>**Call ID records**</DNT> (for initial setup or service pack upgrades):

1. Execute transaction `/NEWR/GEN_DP_CALLID`.
2. Select <DNT>**Backup & delete all existing (Call ID) records in Registry**</DNT>.
3. Click <DNT>**Execute**</DNT>.

<Callout variant="important">
  This creates a backup before deletion. Use this option only during initial setup or when instructed during a service pack upgrade.
</Callout>

### Refresh service definitions

If you cannot locate a <DNT>**service type**</DNT> or <DNT>**LMS ID**</DNT>:

1. Click <DNT>**Retrieve new services (LMSID) setup in Cloud ALM**</DNT>.
2. This fetches the latest services configured in your <DNT>**Cloud ALM account**</DNT>.
