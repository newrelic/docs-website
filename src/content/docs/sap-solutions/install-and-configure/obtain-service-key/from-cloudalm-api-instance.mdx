This guide shows you how to obtain a service key from the SAP Cloud Application Lifecycle Management (CALM) API instance. You'll need this service key to configure monitoring for SAP Cloud ALM.

## Prerequisites

- Access to <DNT>**CALM subaccount**</DNT> with appropriate permissions
- Confirmation from your <DNT>**BTP (Business Technology Platform) administrator**</DNT> regarding <DNT>**Space**</DNT>
- Required scopes identified from [Managing Scopes](https://help.sap.com/docs/cloud-alm/apis/managing-scopes)

1. Log in to the <DNT>**CALM (Cloud Application Lifecycle Management) subaccount**</DNT>, then click <DNT>**Instances and Subscriptions**</DNT>.
2. Click <DNT>**Create**</DNT> at the top right.
3. In the pop-up screen, select:
   - <DNT>**Service**</DNT>: <DNT>**SAP Cloud ALM API**</DNT>
   - <DNT>**Plan**</DNT>: <DNT>**Standard**</DNT>
   - <DNT>**Runtime Environment**</DNT>: <DNT>**Cloud Foundry**</DNT>
   - <DNT>**Space**</DNT>: Check with your <DNT>**BTP administrator**</DNT>
   - <DNT>**Instance Name**</DNT>: Follow your organization naming convention

   <img
      title="SAP Cloud ALM API"
      alt="The Create Instance dialog showing Service, Plan, Runtime Environment, Space, and Instance Name fields for SAP Cloud ALM API"
      src="/images/sap-cloud-alm-api.webp"
      width="600"
   />

4. Click <DNT>**Next**</DNT>.
5. Review and insert required scopes from [Managing Scopes](https://help.sap.com/docs/cloud-alm/apis/managing-scopes) along with the instance name under the parameters tab.

   <img
      title="Manage Scopes"
      alt="The parameters tab showing the required scopes configuration for the CALM instance"
      src="/images/manage-scope.webp"
      width="600"
   />
6. Click <DNT>**Next**</DNT> to review the details.

   <img
      title="Review Details"
      alt="The review screen showing the summary of the SAP Cloud ALM API instance configuration before creation"
      src="/images/cloud-alm-new-instance.webp"
      width="600"
   />
7. Click <DNT>**Create**</DNT>.
8. Once created, locate the service under the <DNT>**Instances**</DNT> tab.
9. Click the ellipsis (`...`) in the row of your newly created instance, then select <DNT>**Create Service Key**</DNT>.
10. Enter a <DNT>**Service Key Name**</DNT> and select <DNT>**Key Type**</DNT>: `ClientId/Secret`.
11. Click <DNT>**Create**</DNT>.
12. Click the ellipsis (`...`) in the row of your <DNT>**service key**</DNT>, then select <DNT>**Download**</DNT>.

The <DNT>**service key file**</DNT> downloads as a <DNT>**JSON file**</DNT>. Store it securely - you'll need the credentials from this file to configure your monitoring setup.
