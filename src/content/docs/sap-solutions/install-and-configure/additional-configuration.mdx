Complete these configuration tasks for each monitored system.

## Upload authorization role [#upload-authorization-role]

1. Use <DNT>**SAP transaction**</DNT> `PFCG`.
2. Go to <DNT>**Role**</DNT> > <DNT>**Upload**</DNT> and upload the file <DNT>**_NEWR_DATA_PROVIDER.SAP**</DNT> from the installation package.
3. Confirm that the role `/NEWR/DATA_PROVIDER` loads.

   <img
     title="Authorization Role"
     alt="A screenshot displaying the authorization role upload"
     src="/images/authorisation-role.webp"
     width="600"
   />

4. Select the <img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" /> icon after the role loads.
5. On the <DNT>**Authorizations**</DNT> tab, select <img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" /> <DNT>**Change Authorization Data**</DNT>, and then activate it using the <img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" /> icon.

   <img
     title="Authorizations Tab"
     alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload"
     src="/images/sap-authorization-1.webp"
     width="500"
   />

6. If a profile name is suggested, select the green check icon.
7. Verify that <DNT>**Profile(s) were updated**</DNT> appears on the status bar.

<Callout variant="important">
  If you need <DNT>**SLT module monitoring**</DNT> on the system, also upload the `/NEWR/SLT_COLLECTOR` role from the `_NEWR_SLT_COLLECTOR.SAP` file in the installation package.
</Callout>

## Create New Relic RFC user ID [#create-rfc-user-id]

After you activate the <DNT>**authorization profile**</DNT>, use transaction `SU01` to create a new <DNT>**RFC user ID**</DNT> (for example, `NR4SAPDP`). Set the <DNT>**user type**</DNT> to <DNT>**Communication**</DNT> on the monitored (data provider) systems and <DNT>**System**</DNT> on the central monitoring system (CMS). Then add the role `/NEWR/DATA_PROVIDER`.

## Create RFC destinations for monitored systems [#create-rfc-for-monitored-systems]

Use transaction `SM59` to create an <DNT>**RFC destination**</DNT> for each monitored system.

1. Select <DNT>**ABAP Connections**</DNT> and then select the <DNT>**Create**</DNT> icon to set up a new <DNT>**RFC destination**</DNT>.
2. Enter a descriptive <DNT>**RFC Destination**</DNT> name. We recommend: `NR_<SYSID>CLNT<CLNT#>`
3. Set <DNT>**Connection Type**</DNT> to `3 - ABAP Connection`.
4. On the <DNT>**Technical Settings**</DNT> tab, enter:
   - <DNT>**Target Host**</DNT> (fully qualified domain or IP) and <DNT>**Instance No.**</DNT>
   - <DNT>**Gateway Host**</DNT> and <DNT>**Service**</DNT> if necessary.

   <img
     title="Create RFC destinations for monitored systems"
     alt="A screenshot displaying the how to create RFC destinations for monitored systems"
     src="/images/rfc-destination.webp"
     width="600"
   />

   <Callout variant="tip">
   Use <DNT>**load balancing**</DNT> whenever possible. It allows data collection to continue even when a single connection goes down. For detailed steps, see [Setting Up an RFC Destination with Load Balancing](https://help.sap.com/doc/saphelp_nw75/7.5.5/en-US/cd/d9a38c1709414b8bc6a4380c398063/content.htm?no_cache=true) in the SAP documentation.
   </Callout>

5. On the <DNT>**Logon & Security**</DNT> tab, enter the <DNT>**RFC user**</DNT> (`NR4SAPDP`) and password.
6. Select <DNT>**Save**</DNT>.
7. Go to <DNT>**Utility > Test > Connection Test**</DNT> and <DNT>**Authorization Test**</DNT>.
   - <DNT>**Expected result**</DNT>: Both tests show successful status.

## Set HTTP destination to New Relic cloud [#set-http-destination-to-new-relic-cloud]

Use transaction `SM59` to set up <DNT>**HTTPS connections**</DNT> to the <DNT>**New Relic API endpoints**</DNT>.

<Callout variant="important">
Create individual <DNT>**RFC destinations**</DNT> for <DNT>**Events**</DNT>, <DNT>**Logs**</DNT>, <DNT>**Metrics**</DNT>, and <DNT>**Traces**</DNT> separately.
</Callout>

1. Enter a <DNT>**Destination Name**</DNT> (for example, `NR_API_EVENT`).
2. Set <DNT>**Connection Type**</DNT> to `G` (HTTP Connections to External Server).
3. Enter <DNT>**Target Host**</DNT>, <DNT>**Service No.**</DNT> as <DNT>**443**</DNT> (see [standard port](/docs/sap-solutions/reference/network-accessibility) for <DNT>**HTTPS**</DNT>), and <DNT>**Path Prefix**</DNT> (see [installation readiness checklist](/docs/sap-solutions/reference/pre-installation-checklist/#installation-readiness-checklist) for details).
4. If you need a proxy, configure it on the <DNT>**Technical Settings**</DNT> tab.
5. On the <DNT>**Security Options**</DNT> tab, select the <DNT>**Active**</DNT> option for <DNT>**SSL**</DNT>.

   <img
     title="HTTP destination to New Relic"
     alt="A screenshot displaying the connection setup for HTTP destination to New Relic"
     src="/images/http-destination-to-new-relic.webp"
     width="600"
   />

6. If the SSL certificate isn't configured yet, download the <DNT>**root and intermediate certificates**</DNT> from the New Relic endpoint in your browser and import them into <DNT>**SAP**</DNT> using transaction `STRUST`.
7. Select <DNT>**Connection Test**</DNT> to verify the connection.
   - <DNT>**Expected result**</DNT>: A successful connection prompts you to enter logon data.
   - <DNT>**If test times out**</DNT>: A firewall is likely blocking the connection. Consult your network team to ensure access is granted to the <DNT>**New Relic domains**</DNT>.
8. At the logon prompt, select <DNT>**Cancel**</DNT> to proceed. An <DNT>**HTTP 4xx**</DNT> response code screen confirms the connection is working.
9. Repeat this procedure for all <DNT>**New Relic API endpoints**</DNT>.

## Create BTP/CloudALM authentication endpoint [#create-authentication-endpoint]

Select the tab for your service:

<Tabs>
  <TabsBar>
    <TabsBarItem id="btp-process-integration-runtime">
      <DNT>**BTP process integration runtime**</DNT>
    </TabsBarItem>
    <TabsBarItem id="cloud-alm-api">
      <DNT>**CloudALM API**</DNT>
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="btp-process-integration-runtime">
      1. Open the downloaded <DNT>**Service Key**</DNT> to identify and record the `clientId`, `clientsecret`, and `tokenurl`.
      2. Navigate to the URL specified in your service key using a web browser.
      3. From that page, obtain the <DNT>**SAP on demand**</DNT>, <DNT>**Digicert global**</DNT>, and <DNT>**Digicert root**</DNT> certificates.

      <img
        title="BTP process integration runtime"
        alt="A screenshot displaying the connection setup for BTP process integration runtime"
        src="/images/btp-process-integration.webp"
        width="600"
      />

      4. Navigate to transaction code `STRUST` and select <DNT>**SSL Client (Standard)**</DNT> while in edit mode.
      5. Individually upload the three certificates, select <DNT>**Add to Certificate List**</DNT> for each, and then save your changes.
      6. Navigate to Tcode `SMICM`, then select <DNT>**Administration > ICM > Exit Soft > Global**</DNT>. Executing this action triggers a restart of all <DNT>**ICM processes**</DNT> within the system.

      <img
        title="Administration ICM"
        alt="A screenshot displaying the Administration ICM"
        src="/images/administration-icm.webp"
        width="600"
      />

      7. Access the SAP system and navigate to transaction code `SM59`, then establish a new <DNT>**RFC connection**</DNT> within the <DNT>**HTTP Connections to External Server**</DNT> category.
      8. Configure the destination by dividing the token URL into its respective host and path prefix components.
      9. Set the service number to `443` and modify the path prefix by appending `?grant_type=client_credentials`.

      <img
        title="Service No."
        alt="A screenshot displaying the Service No."
        src="/images/service-no.webp"
        width="600"
      />

      10. On the <DNT>**Login & Security**</DNT> tab, select <DNT>**Basic Authentication**</DNT> and enter `clientId` and `clientsecret` as the username and password.
      11. Set <DNT>**Secure Protocol**</DNT> status to <DNT>**Active**</DNT> and save the configuration.
      12. Select <DNT>**Connection Test**</DNT>.
    </TabsPageItem>

    <TabsPageItem id="cloud-alm-api">
      1. Note the `clientId`, `clientsecret`, and `url` from the downloaded <DNT>**Service Key**</DNT>.
      2. Go to the <DNT>**url**</DNT> in a web browser and download the <DNT>**Digicert root**</DNT>, <DNT>**global**</DNT>, and <DNT>**SAP on demand**</DNT> certificates.

      <img
        title="CloudALM API"
        alt="A screenshot displaying the connection setup for CloudALM API"
        src="/images/cloudalm-api.webp"
        width="600"
      />

      3. Navigate to transaction code `STRUST` and select <DNT>**SSL Client (Standard)**</DNT> while in edit mode.
      4. Individually upload each of the three certificates, then select <DNT>**Add to Certificate List**</DNT> and save your changes.
      5. Navigate to Tcode `SMICM`, then select <DNT>**Administration > ICM > Exit Soft > Global**</DNT>. Executing this action triggers a restart of all system <DNT>**ICM processes**</DNT>.

      <img
        title="Administration ICM"
        alt="A screenshot displaying the Administration ICM"
        src="/images/administration-icm.webp"
        width="600"
      />

      6. Access transaction code `SM59` after logging into the SAP system, then establish a new <DNT>**RFC connection**</DNT> under the <DNT>**HTTP Connections to External Server**</DNT> category.
      7. Configure the <DNT>**Target Host**</DNT> with the appropriate URL and set the <DNT>**Service No.**</DNT> to `443`.

      <img
        title="SM59 Configuration"
        alt="A screenshot displaying the SM59 configuration"
        src="/images/sm59-configuration.webp"
        width="600"
      />

      8. Navigate to the <DNT>**Login & Security**</DNT> tab and choose <DNT>**Basic Authentication**</DNT> in the <DNT>**Login with user**</DNT> section. Provide the `clientId` as the username and the `clientsecret` as the password.
      9. Locate the <DNT>**Secure Protocol**</DNT> status below and set it to <DNT>**Active**</DNT>.
      10. Apply the configuration changes by saving, then select <DNT>**Connection Test**</DNT>.
    </TabsPageItem>
  </TabsPages>
</Tabs>

## Create database connection [#create-database-connection]

Use transaction `DBACOCKPIT` to define the remote <DNT>**HANA Database connections**</DNT>. In the left pane, double-click <DNT>**Database Connections**</DNT>, then select <DNT>**Add**</DNT> to create a new connection.

<img
  title="Database Connection"
  alt="A screenshot displaying the Database Connection"
  src="/images/database-connection.webp"
  width="600"
/>

Under <DNT>**Database Details**</DNT>, perform the following steps:

1. Input the <DNT>**Connection Name**</DNT> (using the <DNT>**SAP system ID**</DNT> is recommended).
2. Pick <DNT>**HANA Database**</DNT> as the designated <DNT>**Database System**</DNT>.
3. Provide a database user ID (`NRAGENT` is the recommended ID), then enter and confirm your password.

   <img
     title="SAP Connection Name"
     alt="A screenshot displaying the SAP Connection Name"
     src="/images/connection-name.webp"
     width="600"
   />

4. After saving your entries, select <DNT>**Test**</DNT> to verify the connection to the database.
   - <DNT>**Expected result**</DNT>: Connection test shows successful status.
