## Use /NEWR/INSTALL menu [#menu-newr-install]

1. Log in to the production client of the central monitoring system.
2. Navigate to the <DNT>**SAP Easy Access root menu**</DNT> by typing `/n` in the command field and pressing <DNT>**Enter**</DNT>.
3. Access the <DNT>**`/NEWR/INSTALL`**</DNT> menu by typing `/n/NEWR/INSTALL` in the command field.

After installing the add-on on the central monitoring system:

  - Ensure the installer has proper authorization to perform the configuration.
  - Add the parameter <DNT>**`/NEWR/*`**</DNT> to the code authorization object(s) in the installer's `profile/role`.

  <Callout variant="tip">
    If upgrading from a previous version, refer to [Upgrade from Previous Version](/docs/sap-solutions/upgrade/install-upgrade-package) to determine which steps are necessary.
  </Callout>

## Connect systems [#connect-systems]

<Callout variant="important">
  Complete any pending configuration steps before proceeding.
</Callout>

The following options are available in the `/NEWR/INSTALL` menu:

### Set up HTTP and RFC Destinations

Use this option to invoke transaction <DNT>**`SM59`**</DNT> and verify your [RFC](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems) and [HTTPS](/docs/sap-solutions/install-and-configure/additional-configuration/#set-http-destination-to-new-relic-cloud) connections.

### Set up Connection to Remote Database
  - Use SAP transaction <DNT>**DBACOCKPIT**</DNT> to verify the setup of connections to remote HANA databases.
  - Ensure that a database connection is created for each remote HANA database you wish to monitor.
  - For detailed instructions, see [Create Database Connection](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection).

### Configure Connections to NR API Endpoints
  1. Click <DNT>**New Entries**</DNT>.
  2. Enter your New Relic account details:
      - <DNT>**New Relic Account ID**</DNT> (optionally prefixed with SAP-)
      - <DNT>**API Ingest Key**</DNT>
      - <DNT>**Metric Endpoint**</DNT> (see [Network endpoints](/docs/sap-solutions/before-you-begin/#network-endpoints))
      - <DNT>**Event Endpoint**</DNT>
      - <DNT>**Log Endpoint**</DNT>
      - <DNT>**Trace Endpoint**</DNT>
  3. Click <DNT>**Save**</DNT>.

### Define the Monitored Systems

  1. Click <DNT>**New Entries**</DNT> to enter the <DNT>**SAP System ID**</DNT> and <DNT>**Client**</DNT> number.
  2. In <DNT>**Data Provider Installation Information**</DNT>, specify the RFC <DNT>**Destination**</DNT>, select the SAP <DNT>**Software Component**</DNT>, enter the <DNT>**Database Vendor**</DNT>, and provide the <DNT>**DB Connection Name**</DNT> (HANA database only).
  3. Click <DNT>**Monitoring Group**</DNT>, select <DNT>**Yes**</DNT> in the dialog box to confirm.
  4. Choose the necessary monitoring groups, and click the generate icon to complete the process.
  5. Click <DNT>**BACK**</DNT> or the <DNT>**F3**</DNT> key to go to the next step.

## Generate configuration [#generate-configurations]

### Generate Configurations for New Data Providers

  - Use this menu option to generate data providers for a new release or patch. For fresh installations, all Data Providers (DP) are generated.
  - On the first screen, confirm the release and patch details. Empty fields indicate pre-existing configuration.
  - To regenerate, select <DNT>**Re-generate Data Provider Configurations**</DNT> in <DNT>**Maintain Monitored Systems**</DNT> or use <DNT>**Reset Installation Flag (For Re-install only)**</DNT> to reset the installation flag and regenerate configurations.

  <img
    title="Maintain Monitored Systems"

### Verify Installation Completeness

Click <DNT>**Verify Installation Completeness**</DNT> to ensure everything is configured correctly. If any item displays a red indicator, click the execution button in the <DNT>**Action Needed**</DNT> column.

## Verify configuration [#verify-configuration]

  1. To ensure data providers are correctly configured, open the <DNT>**Data Provider Registry Maintenance**</DNT> folder.
  2. Click <DNT>**Data Provider Registry**</DNT>, or use transaction <DNT>**`/n/NEWR/REGISTRY`**</DNT>.
  3. Select a <DNT>**Data Provider Name**</DNT> and verify configuration parameters by clicking <DNT>**Configure**</DNT>.
  4. Test the data provider by clicking <DNT>**TestDP**</DNT>. The test should complete successfully, confirming the data provider is configured correctly.
