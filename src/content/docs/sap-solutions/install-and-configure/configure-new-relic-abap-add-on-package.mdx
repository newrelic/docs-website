After installing the ABAP add-on, use the <DNT>**`/NEWR/INSTALL`**</DNT> menu to connect your SAP systems to New Relic, configure API endpoints, define monitoring groups, and verify the setup.

## Use /NEWR/INSTALL menu [#menu-newr-install]

1. Log in to the production client of the central monitoring system.
2. Type `/n` in the command field and press <DNT>**Enter**</DNT> to go to the <DNT>**SAP Easy Access root menu**</DNT>.
3. Type `/n/NEWR/INSTALL` in the command field to open the <DNT>**`/NEWR/INSTALL`**</DNT> menu.

Before configuring, ensure the following:

  - The installer has proper authorization to perform the configuration.
  - The `/NEWR/*` parameter is added to the `S_TCODE` authorization object in the installer's `profile/role`.

  <Callout variant="tip">
    If upgrading from a previous version, refer to [Upgrade from Previous Version](/docs/sap-solutions/upgrade/install-upgrade-package) to determine which steps are necessary.
  </Callout>

## Connect systems [#connect-systems]

<Callout variant="important">
  Complete any pending configuration steps before proceeding.
</Callout>

In the `/NEWR/INSTALL` menu, open the <DNT>**Install New Release and Patch**</DNT> folder and complete the following steps.

### Set up HTTP and RFC Destinations

Use transaction `SM59` to verify your <DNT>**RFC**</DNT> and <DNT>**HTTPS**</DNT> connections:

- For RFC and HTTPS setup, see [Create RFC for monitored systems](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems) and [Set HTTP destination to New Relic cloud](/docs/sap-solutions/install-and-configure/additional-configuration/#set-http-destination-to-new-relic-cloud).
- If you monitor BTP accounts, also create an HTTP destination for BTP/CALM authentication. See [Create authentication endpoint](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint).
- If you monitor a <DNT>**SAP Cloud Connector**</DNT>, configure an RFC destination of type <DNT>**G**</DNT> (External HTTP) using port <DNT>**8443**</DNT> (default Cloud Connector HTTPS admin port). The user account must have the required authorizations and password.

### Set up Connection to Remote Database

If you did not complete this step during pre-installation, complete it now.

  - Open SAP transaction `DBACOCKPIT` to verify connections to remote <DNT>**HANA databases**</DNT>.
  - Create a database connection for each remote HANA database you want to monitor.
  - For detailed instructions, see [Create Database Connection](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection).

### Configure Connections to New Relic API Endpoints

You can configure multiple New Relic accounts in this section.

  1. Click <DNT>**New Entries**</DNT>.
  2. Enter your New Relic account details:
      - <DNT>**New Relic Account ID**</DNT> (optionally prefixed with SAP-)
      - <DNT>**API Ingest Key**</DNT>
      - <DNT>**Metric Endpoint**</DNT> (see [Network endpoints](/docs/sap-solutions/before-you-begin/#network-endpoints))
      - <DNT>**Event Endpoint**</DNT>
      - <DNT>**Log Endpoint**</DNT>
      - <DNT>**Trace Endpoint**</DNT>
  3. Click <DNT>**Save**</DNT>.

### Define the Monitored Systems

  1. Click <DNT>**New Entries**</DNT> and enter the <DNT>**SAP System ID**</DNT> and <DNT>**Client**</DNT> number.
  2. In <DNT>**Data Provider Installation Information**</DNT>, provide the following:
      - <DNT>**RFC Destination**</DNT>
      - <DNT>**SAP Software Component**</DNT>
      - <DNT>**Database Vendor**</DNT>
      - <DNT>**DB Connection Name**</DNT> (HANA databases only)
      - <DNT>**New Relic Account ID**</DNT> (optional) — if provided, data from this system goes only to that account, overriding the account in the collection manager job variant. If left blank, the collection manager job variant account is used.
  3. Click <DNT>**Monitoring Group**</DNT> and select <DNT>**Yes**</DNT> to confirm.
  4. Choose the required monitoring groups and click the generate icon.
  5. Click <DNT>**BACK**</DNT> or press <DNT>**F3**</DNT> to continue.

### BTP/CALM accounts

When BTP monitoring is active, add BTP/CALM accounts as monitored systems. Use the following values:

- <DNT>**SAP System ID**</DNT>: Use a pseudonym or alias. Do not use reserved terms such as `CALM` or `BTP`.
- <DNT>**Client**</DNT>: Leave blank.
- <DNT>**Destination**</DNT>: Enter the authentication HTTP destination.
- <DNT>**Software Component**</DNT>: Leave blank.
- <DNT>**Database Vendor**</DNT>: Leave blank.
- <DNT>**DB Connection Name**</DNT>: Leave blank.
- <DNT>**New Relic Account ID**</DNT>: Enter the specific New Relic account designated to receive the data.
- <DNT>**API Host**</DNT>: Enter the API host name. This corresponds to the `url` value in the service key.

On the next screen, click <DNT>**Monitoring Group**</DNT>. Choose the appropriate monitoring groups for the system and click <DNT>**Finish**</DNT>.

### Check RFC Destinations

Use this option to verify the RFC destinations for the New Relic API endpoints and each monitored system. The RFC destination designated as <DNT>**NONE**</DNT> is excluded from this test.

## Generate configuration [#generate-configurations]

### Generate Configurations for new data providers

  - Run this option to generate all new data providers for a new release or patch. For fresh installations, all data providers are generated in this single step.
  - On the first screen, confirm the release and patch details. If the installation release and patch fields are blank, the configuration is already established.
  - To regenerate configurations, navigate to <DNT>**Maintain Monitored Systems**</DNT> and select <DNT>**Re-generate Data Provider Configurations**</DNT>. You can also select <DNT>**Reset Installation Flag (For Re-install only)**</DNT> to clear the current installation flag and regenerate.

  <Callout variant="tip">
    For data providers based on CALM, see [Generate Registry Entries for CALM Data Providers](/docs/sap-solutions/monitor/data-providers-reference).
  </Callout>

### Verify installation completeness

Click <DNT>**Verify Installation Completeness**</DNT>. If any item shows a red indicator, click the button in the <DNT>**Action Needed**</DNT> column.

## Verify configuration [#verify-configuration]

  1. Open the <DNT>**Data Provider Registry Maintenance**</DNT> folder.
  2. Click <DNT>**Data Provider Registry**</DNT>, or use transaction `/n/NEWR/REGISTRY`.
  3. Select a <DNT>**Data Provider Name**</DNT> and click <DNT>**Configure**</DNT> to verify the parameters.
  4. Click <DNT>**TestDP**</DNT> to test the data provider. A successful test confirms the data provider is configured correctly.
