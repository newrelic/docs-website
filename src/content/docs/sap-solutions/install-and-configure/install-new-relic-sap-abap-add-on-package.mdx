The ABAP component of <DNT>**NRM4SAP**</DNT> is packaged as a SAP add-on and can be installed using transaction <DNT>**SAINT**</DNT> on <DNT>**client 000**</DNT>.

## Add-on installation file [#add-on-installation-file]

- **Initial installation**: Download the add-on file `K-xxxAOIINNEWR.SAR` to a folder on your local PC.
- **Upgrade from previous version**: Use the file named `K-xxxAOUINNEWR.SAP`.

## Install SAP add-on file [#install-sap-add-on-file]

1. Log in to <DNT>**client 000**</DNT>, which is the default client for configuration and system administration tasks.
2. Type `SAINT` in the <DNT>**SAP GUI**</DNT> command field and click <DNT>**Enter**</DNT>.
3. Navigate to <DNT>**Installation Package**</DNT> > <DNT>**Load Packages**</DNT>.
4. Choose based on where your installation file is located:

  <Tabs>
    <TabsBar>
      <TabsBarItem id="sar-archive-from-frontend">SAR archive from frontend</TabsBarItem>
      <TabsBarItem id="sar-archive-from-application-server">SAR archive from application server</TabsBarItem>
      <TabsBarItem id="eps-files-from-application-server">EPS files from application server</TabsBarItem>
    </TabsBar>
    <TabsPages>
      <TabsPageItem id="sar-archive-from-frontend">
        Use this method if the SAR file is on your local PC.

        1. Navigate and open the unzipped installation file.
        2. In the <DNT>**Content of the compressed file `K-xxxAOIINNEWR.SAR`**</DNT> popup, click <DNT>**Decompress**</DNT>.

        <img
          title="Content of the compressed file"
          alt="A screenshot displaying the Content of the compressed file"
          src="/images/content-of-compressed-file-1.webp"
          width="80%"
        />

        3. The message <DNT>**File K-xxxAOIINNEWR.SAR successfully transferred, decompressed, and deleted**</DNT> appears.
      </TabsPageItem>
      <TabsPageItem id="sar-archive-from-application-server">
        Use this method if the SAR file is already on your application server.

        1. Ensure there is a download directory under the trans directory on the application server (`/usr/sap/trans/download`).
        2. Copy the add-on file to this directory.

      </TabsPageItem>
      <TabsPageItem id="eps-files-from-application-server">
        Use this method if you need to extract the SAR file into PAT files on the server.

        1. Untar the `SAR` file into the `PAT` file first. Execute the command below on the OS (Linux in this case):

          ```bash
          cd /usr/sap/trans
          cp /tmp/K-300AOIINNEWR.SAR /usr/sap/trans
          SAPCAR -xvf K-300AOIINNEWR.SAR
          ```

        2. In the popup, <DNT>**Do you want to upload OCS Packages from the EPS inbox?**</DNT>, select <DNT>**Yes**</DNT>. Ensure the OCS file is uploaded without errors.
      </TabsPageItem>
    </TabsPages>
  </Tabs>

5. Click <DNT>**Start**</DNT>, select the <DNT>**Installable Add-on packages for your system**</DNT>, and click <DNT>**Continue**</DNT>.
6. When asked <DNT>**Do you want to add Modification Adjustment Transports to the queue?**</DNT>, click <DNT>**No**</DNT>.

  <img
      title="Do you want to add Modification Adjustment Transports to the queue?, Click No"
      alt="A screenshot displaying question, Do you want to add Modification Adjustment Transports to the queue?"
      src="/images/modify-adjustement-transports.webp"
      width="80%"
  />

  <Callout variant="important">
    Click <DNT>**Yes**</DNT> if adjustment transports are provided in the packaging list with instructions.
  </Callout>

7. Click the green check button (checkmark icon) to proceed to the next screen.

  <Callout variant="important">
    If the <DNT>**Errors during Disassembling OCS Packages**</DNT> page appears, click <DNT>**Ignore**</DNT>.
  </Callout>

8. Once the package is installed, click <DNT>**Finish**</DNT>. The <DNT>**Sending the runtime analysis**</DNT> popup appears. Choose one of the following:
    - <DNT>**Send**</DNT>: Opens a feedback form with the runtime analysis data in a local browser window. You can use this form to send data to SAP.
    - <DNT>**Display**</DNT>: Displays the runtime analysis first and then send the feedback form in a local browser window.
    - <DNT>**Do not send**</DNT>: Leaves the dialog without sending the runtime analysis data.
9. Verify the installation in the <DNT>**Installed Components**</DNT> tab. You should see the New Relic add-on listed with a status indicating successful installation. Then logout from <DNT>**Client 000**</DNT>.

  <Callout variant="tip">
    You may type <DNT>**/nex**</DNT> in the <DNT>**SAP GUI**</DNT> command field to log out of the system.
  </Callout>
