## Set up system workload

1. Go to <DNT>**All entities > Workloads > + Create a workload**</DNT>. For more information, see [Create and define workloads](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads/).
2. Enter a name for your <DNT>**workload**</DNT> and select an account from the dropdown list.
3. Next to <DNT>**Entity Type = All**</DNT>, select the <DNT>**+**</DNT> icon and enter `sap.system = {SAP System SID}` in the filter bar.

    <img
      title="SAP System"
      alt="A screenshot displaying the SAP System filter"
      src="/images/sap-system.webp"
    />

4. Select <DNT>**+ Add this query**</DNT> to add the query to your <DNT>**workload definition**</DNT>.
5. Select <DNT>**Create a workload**</DNT>.

Your new <DNT>**workload**</DNT> appears in <DNT>**All entities**</DNT> and includes all entities tagged with your <DNT>**SAP system**</DNT>.

## Install service pack

<DNT>**Service packs**</DNT> are delivered as `.SAR` files with the naming convention `K-<Version Number>SP<SP Number>IN<Software Component>.SAR`.

1. Install the <DNT>**service pack**</DNT> using transaction `SPAM`.
2. Load the new <DNT>**configuration template**</DNT> from <DNT>**client 000**</DNT>.
3. Navigate to <DNT>**Install New Release and Patch**</DNT> > <DNT>**Copy Template Tables**</DNT> from <DNT>**Client 000**</DNT>.
4. Generate the new <DNT>**data provider configurations**</DNT> by following the steps in <DNT>**Install New Release and Patch**</DNT> > <DNT>**Generate Configuration for New Data Providers**</DNT>.
5. Select <DNT>**Install New Release and Patch**</DNT> > <DNT>**Verify Installation Completeness**</DNT> to verify the installation.
