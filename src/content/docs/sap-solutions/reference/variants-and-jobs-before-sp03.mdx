## Create Variants for /NEWR/ABAP_COLLECTION_MANAGERN

1. Use transaction `SE38` to set up `/NEWR/ABAP_COLLECTION_MANAGERN`.
2. On the <DNT>**ABAP: Variants - Initial Screen**</DNT>, enter `/NEWR/ABAP_COLLECTION_MANAGERN` in the <DNT>**Program**</DNT> field.
3. Select <DNT>**Variant**</DNT> and click <DNT>**Create**</DNT>.
4. Follow the below steps to update the values:

<Tabs>
  <TabsBar>
    <TabsBarItem id="trace-variant">
      TRACE variant
    </TabsBarItem>
    <TabsBarItem id="idoc-variant">
      IDOC variant
    </TabsBarItem>
    <TabsBarItem id="system-variant">
      SYSTEM variant
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="trace-variant">
      1. Enter the variant name, e.g., <DNT>**TRACE_ONLY**</DNT>, and click <DNT>**Create**</DNT>.
      2. Ensure the <DNT>**New Relic Account ID**</DNT> is auto-populated.
      3. Set <DNT>**Records per transmission**</DNT> to <DNT>**10000**</DNT>.
      4. Click <DNT>**Multiple Selection (Active)**</DNT> to add data provider names.
      5. Add the following in <DNT>**Select Single Values**</DNT>:
          - `NR.SAP.TRANSACTION`
          - `NR.SAP.TRANSACTION.COMPLETE`
          - `NR.SAP.SYSTEM.PROCESS`
      6. Select <DNT>**Sequential Processing**</DNT> as the <DNT>**Execution Mode**</DNT>.
      7. If <DNT>**DIA WP**</DNT> ≥ 30, set to <DNT>**6**</DNT>; otherwise, set to <DNT>**3**</DNT>.
      8. Click <DNT>**Attributes**</DNT>, add a description, and click <DNT>**Save**</DNT>.
    </TabsPageItem>

    <TabsPageItem id="idoc-variant">
      1. Enter the variant name, e.g., <DNT>**IDOC_ONLY**</DNT>, and click <DNT>**Create**</DNT>.
      2. Ensure the <DNT>**New Relic Account ID**</DNT> is auto-populated.
      3. Set <DNT>**Records per transmission**</DNT> to <DNT>**10000**</DNT>.
      4. Click <DNT>**Multiple Selection (Active)**</DNT> to add data provider names.
      5. Add the following in <DNT>**Select Single Values**</DNT>:
          - `NR_SAP_IDOC`
          - `NR_SAP_IDOC_COMPLETE`
      6. Select <DNT>**Sequential Processing**</DNT> as the <DNT>**Execution Mode**</DNT>.
      7. Click <DNT>**Attributes**</DNT>, add a description, and click <DNT>**Save**</DNT>.
    </TabsPageItem>

    <TabsPageItem id="system-variant">
      1. Enter the variant name, e.g., <DNT>**System**</DNT>, and click <DNT>**Create**</DNT>.
      2. Ensure the <DNT>**New Relic Account ID**</DNT> is auto-populated.
      3. Set <DNT>**Records per transmission**</DNT> to <DNT>**10000**</DNT>.
      4. Click <DNT>**Multiple Selection (Active)**</DNT> to add data provider names.
      5. Add variants from <DNT>**TRACE**</DNT> and <DNT>**IDOC**</DNT> providers in <DNT>**Exclude Single Values**</DNT>.
      6. Select <DNT>**Parallel Processing**</DNT> as the <DNT>**Execution Mode**</DNT>.
      7. If <DNT>**DIA WP**</DNT> ≥ 30, set to <DNT>**6**</DNT>; otherwise, set to <DNT>**3**</DNT>.
      8. Click <DNT>**Attributes**</DNT>, add a description, and click <DNT>**Save**</DNT>.
    </TabsPageItem>
  </TabsPages>
</Tabs>

## Schedule collection jobs

1. Use transaction `SM36` to schedule three jobs.
2. Enter the job name for each job and set the <DNT>**Job Class**</DNT> to <DNT>**C**</DNT>.
3. Click <DNT>**Step**</DNT>, enter program name `RSBTONEJOB2`, and specify the variant for each job.
4. Click <DNT>**Start Condition**</DNT>, set the start date and time, check <DNT>**Periodic Job**</DNT>, click <DNT>**Period Values**</DNT>, then <DNT>**Other Period**</DNT>, and enter <DNT>**5**</DNT> for minutes.
