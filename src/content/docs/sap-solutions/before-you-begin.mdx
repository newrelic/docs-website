Complete these prerequisites before installation:

## Prerequisites checklist

### New Relic account
- <DNT>**Account ID**</DNT>: [Obtain your account ID](/docs/accounts/accounts-billing/account-setup/account-id)
- <DNT>**License Key**</DNT>: Get a New Relic <InlinePopover type="licenseKey"/> for authentication and data access
- <DNT>**Data Center Region**</DNT>: [Choose your data center location (US or EU)](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) and configure your account

### SAP system requirements
- <DNT>**Software versions**</DNT>: Verify your <DNT>**SAP systems**</DNT> meet the required software component release levels. See [Data providers reference](/docs/sap-solutions/monitor/data-providers-reference) for system-specific requirements
- <DNT>**Monitored systems**</DNT>: <DNT>**SAP_BASIS 740+**</DNT>, <DNT>**Kernel 753 Patch 602+**</DNT>, <DNT>**ST-PI 740 SP16+**</DNT>
- <DNT>**Central monitoring system**</DNT>: <DNT>**SAP_BASIS 740+**</DNT>, <DNT>**ST-PI 740 SP14+**</DNT>

### Network access
- <DNT>**HTTPS access**</DNT> to <DNT>**New Relic endpoints**</DNT> (see [endpoint URLs](#network-endpoints))
- <DNT>**Firewall rules**</DNT> configured
- <DNT>**Proxy settings**</DNT> configured (if required)

### BTP monitoring (if applicable)
If monitoring <DNT>**BTP accounts**</DNT>, ensure one of the following:
- <DNT>**CloudALM account**</DNT> configured for <DNT>**BTP monitoring**</DNT>
- <DNT>**Process Integration Runtime**</DNT> service subscription (required for integration monitoring)

<Callout variant="important">
  You may need approval or configuration assistance from your security or network teams. Complete these steps before installation and review the [installation readiness checklist](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness).
</Callout>

## Prepare monitored systems

The agentless architecture uses RFC connections to retrieve data.

### Version requirements

Verify the required software versions:
- <DNT>**SAP_BASIS**</DNT>: Release <DNT>**740**</DNT> or above
- <DNT>**Kernel**</DNT>: Release <DNT>**753 Patch 602**</DNT> or above
- <DNT>**ST-PI**</DNT>: Release <DNT>**740 SP16**</DNT> or above

### User configuration

- [Upload the authorization role](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
- [Create a system user](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) with the uploaded role for <DNT>**RFC connections**</DNT>

### HANA database setup (if applicable)

If monitoring a <DNT>**HANA database**</DNT>, create a <DNT>**HANA database user**</DNT> `NRAGENT` with <DNT>**Catalog Read**</DNT> and <DNT>**Monitoring**</DNT> access:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

Collect the <DNT>**HANA server host**</DNT> and <DNT>**tenant DB port**</DNT> for <DNT>**SQL connection**</DNT>.

### Other databases

For other databases, activate the standard <DNT>**CCMS database KPIs**</DNT> by running the <DNT>**ABAP program**</DNT> `RSDBKPIMON`.

### System requirements

- At least <DNT>**20 dialog work processes**</DNT> and <DNT>**10 background work processes**</DNT>
- Outbound <DNT>**HTTPS access**</DNT> to the <DNT>**New Relic data center**</DNT> (see endpoint URLs above)

### Network configuration

- <DNT>**Proxy settings**</DNT> (if required):
  - Server domain name/IP address and port
  - Proxy access credentials (if authentication is necessary)

- <DNT>**Firewall policies**</DNT>: Allow <DNT>**HTTPS traffic**</DNT> to <DNT>**New Relic endpoints**</DNT>

### RFC destinations

For each monitored system, use transaction `SM59` and the remote system ID to create <DNT>**RFC destinations**</DNT> and test connections. See [Create RFC destinations for monitored systems](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems).

<Callout variant="important">
  The <DNT>**RFC user**</DNT> must have the <DNT>**/NEWR/DATA_PROVIDER**</DNT> role assigned.
</Callout>

### Remote HANA database monitoring (if applicable)

To monitor remote <DNT>**HANA databases**</DNT>:

- Ensure the central monitoring system has access to the <DNT>**HANA server**</DNT> and <DNT>**tenant DB port**</DNT>
- For non-S4HANA central monitoring systems, install one of these components:
  - <DNT>**HANA Client v2.0**</DNT> (compatible with the <DNT>**HANA DB version**</DNT>)
  - <DNT>**HANA Client Lib file**</DNT>:
    - `libSQLDBCHDB.so` (Linux)
    - `libSQLDBCHDB.dll` (Windows)
    - Locate the library in the <DNT>**SAP Kernel folder**</DNT> and assign ownership to the <DNT>**SID Admin ID**</DNT> (e.g., `sd1adm`)
- For each remote <DNT>**HANA database**</DNT>, use transaction `SAP DBACOCKPIT` to create a <DNT>**DB connection**</DNT>. See [Create DB connection](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)

## Prepare BTP monitoring (if applicable)

To monitor a <DNT>**BTP account**</DNT>, you need either a <DNT>**CloudALM instance**</DNT> set up to monitor the <DNT>**BTP account**</DNT>, or a subscription to the <DNT>**Process Integration Runtime**</DNT> service (applicable only to integration monitoring). <DNT>**Service keys**</DNT> are required for connection authentication. See [How to retrieve service keys](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).

Use <DNT>**service keys**</DNT> to create [external HTTP destinations](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) and as the <DNT>**API host**</DNT> for <DNT>**API data providers**</DNT>.
