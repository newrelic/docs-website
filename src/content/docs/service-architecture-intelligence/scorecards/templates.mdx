The <DNT>Scorecards</DNT> templates library provides a streamlined way to address engineering and business use cases. By using best practices from large enterprises, the library offers solutions for common engineering challenges.

Each template in the library is fully customizable to meet your organization's standards. You can modify queries, rules, and thresholds to tailor the <DNT>Scorecards</DNT> to your needs.

Some templates label their rules with maturity levels (`L0` through `L3`).

## Scorecard templates
<CollapserGroup>
  <Collapser
    id="product-readiness"
    title={<DNT>Production Readiness</DNT>}
  >
    **Purpose:** Validates if your services are ready for production deployment by checking defined objectives, alert configurations, and recent deployment activity.

    **Importance:** Helps you find readiness gaps before deploying to production.

    **Prerequisites:**
    - [Service level management](/docs/service-level-management/intro-slm)
    - [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    - [New Relic alerts](/docs/alerts/overview)
    - [Change tracking](/docs/change-tracking/overview)

**Key rules:**
    - <DNT>**Defined SLOs**</DNT>: Establishes clear targets for performance and reliability.
    - <DNT>**APM service alerts configuration**</DNT>: Ensures alerts are set for prompt issue detection.
    - <DNT>**Recent deployments**</DNT>: Verifies successful deployments within the last month.
    - <DNT>**Runbook availability**</DNT>: Confirms that runbooks exist for alert conditions triggered in the last month.
  </Collapser>

    <Collapser
    id="aws-reliability"
    title={<DNT>AWS reliability</DNT>}
  >
    **Purpose:** Assesses reliability against AWS standards for security and redundancy in cloud operations.

    **Importance:** Helps you find reliability and security gaps in your cloud infrastructure.

    **Prerequisites:** [New Relic AWS monitoring](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations)

**Key rules:**

    - <DNT>**EFS Encrypted check**</DNT>: Confirms encryption of Elastic File System (EFS) volumes.
    - <DNT>**RDS Instance check retention period**</DNT>: Checks backup retention periods for Relational Database Service (RDS).
    - <DNT>**RDS DeletionProtection check**</DNT>: Verifies deletion protection.
    - <DNT>**RDS MultiAZ presence check**</DNT>: Ensures deployment across multiple availability zones (MultiAZ).
    - <DNT>**RDS AutoMinorVersionUpgrade check**</DNT>: Confirms automatic minor version upgrades.

  </Collapser>

    <Collapser
    id="dora-metrics"
    title={<DNT>DevOps Research and Assessment (DORA) metrics</DNT>}
  >
    **Purpose:** Evaluates resource allocation and process improvements using key performance indicators.

    **Importance:** Quantifies software delivery performance, aiding strategic decision-making.

    **Prerequisites:**

    - [Change tracking](/docs/change-tracking/overview)
    - [New Relic alerts](/docs/alerts/overview)
    - [GitHub integration](/docs/service-architecture-intelligence/github-integrations/github-cloud-integration)

**Key rules:**

    - <DNT>**Deployment frequency**</DNT>: Tracks successful deployment rates.
    - <DNT>**Mean time to recovery**</DNT>: Measures recovery time from failures.
    - <DNT>**Change failure rate**</DNT>: Tracks the percentage of changes that result in failures.
    - <DNT>**Lead time for changes**</DNT>: Measures the time taken from code commit to deployment.

  </Collapser>

  <Collapser
    id="tag-compliance"
    title={<DNT>Tag Compliance</DNT>}
  >
    **Purpose:** Establishes and enforces consistent tagging practices for resources within your organization.

    **Importance:** Consistent tags help you filter and group services across views such as Maps and catalogs.

    **Prerequisites:** [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)

**Key rules:**
    - <DNT>**Team Tag Exists**</DNT>: Checks that services carry an owning-team tag for clear ownership and accountability.
    - <DNT>**Environment Tag Exists**</DNT>: Checks that services carry a consistent environment tag for filtering and grouping.
    - <DNT>**APM Criticality is set**</DNT>: Checks that services have a criticality tag to prioritize by business importance.
  </Collapser>

  <Collapser
    id="observability-maturity-intelligent-observability"
    title={<DNT>Observability Maturity: Intelligent Observability</DNT>}
  >
    **Purpose:** Establishes a comprehensive tagging strategy and complete visibility into service dependencies and relationships.

    **Importance:** Highlights gaps in tags, relationships, trace, and synthetic coverage so you can address them.

    **Prerequisites:**

    - [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    - [Synthetic monitoring](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring)

**Key rules:**
    - <DNT>**(L0) Team Tag Coverage**</DNT>: Identifies entities missing a `team` tag.
    - <DNT>**(L0) Environment Tag Coverage**</DNT>: Identifies entities missing an `environment` tag.
    - <DNT>**(L0) Uninstrumented Entities**</DNT>: Flags HTTP service and database entities with unknown relationships that need supplemental instrumentation.
    - <DNT>**(L0) Distributed Trace Coverage**</DNT>: Measures the percentage of APM entities emitting distributed trace data.
    - <DNT>**(L0) Synthetic Coverage**</DNT>: Measures the percentage of APM entities covered by synthetic monitors.
  </Collapser>

  <Collapser
    id="observability-maturity-business-uptime"
    title={<DNT>Observability Maturity: Business Uptime</DNT>}
  >
    **Purpose:** Develops a comprehensive alerting strategy and incident management approach for service reliability.

    **Importance:** Highlights alert gaps, noise, and SLI coverage so you can prioritize critical services.

    **Prerequisites:**

    - [New Relic alerts](/docs/alerts/overview)
    - [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    - [Infrastructure monitoring](/docs/infrastructure/introduction-infra-monitoring)
    - [Service level management](/docs/service-level-management/intro-slm)

**Key rules:**
    - <DNT>**(L1) Infrastructure Alert Coverage**</DNT>: Identifies hosts and Kubernetes pods that lack alert coverage.
    - <DNT>**(L1) Service Delivery Alert Coverage**</DNT>: Identifies services, browser, mobile, and synthetic monitors that lack alert coverage.
    - <DNT>**(L1) Critical Alert Coverage**</DNT>: Flags overreliance on critical alert conditions that can cause alert fatigue.
    - <DNT>**(L1) Alert Noise**</DNT>: Identifies policies that generate a high volume of incidents.
    - <DNT>**(L2) Service Level Coverage**</DNT>: Checks whether entities have a defined service level indicator (SLI).
    - <DNT>**(L2) Alerts Mean Time To Close**</DNT>: Evaluates the time taken to close incidents, targeting resolution within 30 minutes.
    - <DNT>**(L2) APM Criticality Tag Coverage**</DNT>: Identifies entities missing a `criticality` tag.
    - <DNT>**(L3) Service Level Attainment**</DNT>: Evaluates the compliance score for each defined SLI, using a default threshold of 95%.
  </Collapser>

  <Collapser
    id="observability-maturity-engineering-excellence"
    title={<DNT>Observability Maturity: Engineering Excellence</DNT>}
  >
    **Purpose:** Emphasizes efficient resource usage and robust security practices by monitoring CPU and memory usage, integrating change tracking, and detecting vulnerabilities.

    **Importance:** Highlights resource, change-tracking, and vulnerability gaps so you can improve efficiency and security.

    **Prerequisites:**

    - [Infrastructure monitoring](/docs/infrastructure/introduction-infra-monitoring)
    - [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    - [Change tracking](/docs/change-tracking/overview)
    - [Vulnerability management](/docs/vulnerability-management/getting-started/overview)

**Key rules:**
    - <DNT>**(L1) CPU Utilization**</DNT>: Checks the 95th percentile of CPU usage for infrastructure entities against efficiency best practices.
    - <DNT>**(L1) Memory Utilization**</DNT>: Checks the 95th percentile of memory usage for infrastructure entities against efficiency best practices.
    - <DNT>**(L2) Change Tracking**</DNT>: Checks for change-tracking events across your APM entities.
    - <DNT>**(L3) Service Vulnerabilities**</DNT>: Checks the percentage of APM entities with detected vulnerabilities.
  </Collapser>

  <Collapser
    id="observability-maturity-digital-experience"
    title={<DNT>Observability Maturity: Digital Experience</DNT>}
  >
    **Purpose:** Monitors frontend performance and user interaction through JavaScript errors, HTTP 5xx errors, and Core Web Vital metrics.

    **Importance:** Highlights frontend issues and SLI gaps so you can improve reliability and user experience.

    **Prerequisites:**

    - [Browser monitoring](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)
    - [Service level management](/docs/service-level-management/intro-slm)

**Key rules:**
    - <DNT>**(L1) JS Error Rate**</DNT>: Checks the percentage of browser entities with JavaScript errors.
    - <DNT>**(L1) Service Error Rate**</DNT>: Checks the percentage of APM services reporting 5xx errors in HTTP requests.
    - <DNT>**(L2) Core Web Vitals: Largest Contentful Pane (LCP)**</DNT>: Flags entities whose 75th-percentile LCP exceeds 2.5 seconds.
    - <DNT>**(L2) Core Web Vitals: Interaction to Next Paint (INP)**</DNT>: Flags entities whose 75th-percentile INP exceeds 200 milliseconds.
    - <DNT>**(L2) Core Web Vitals: Cumulative Layout Shift (CLS)**</DNT>: Flags entities whose 75th-percentile CLS exceeds 0.1.
    - <DNT>**(L3) Service Level Attainment**</DNT>: Evaluates the compliance score for each defined SLI, using a default threshold of 95%.
  </Collapser>

  <Collapser
    id="security"
    title={<DNT>Security</DNT>}
  >
    **Purpose:** Assesses vulnerability risk across APM services and infrastructure.

    **Importance:** Surfacing critical and high-severity vulnerabilities, active ransomware CVEs, and exploit likelihood helps prioritize remediation.

    **Prerequisites:**

    - [Vulnerability management](/docs/vulnerability-management/getting-started/overview)
    - [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    - [Infrastructure monitoring](/docs/infrastructure/introduction-infra-monitoring)

**Key rules:**
    - <DNT>**APM: Under 3 Critical Severity**</DNT>: Checks that an APM service has fewer than 3 critical-severity vulnerabilities.
    - <DNT>**APM: Under 5 High Severity**</DNT>: Checks that an APM service has fewer than 5 high-severity vulnerabilities.
    - <DNT>**APM: Active Ransomware**</DNT>: Flags reported CVEs that have an active ransomware campaign.
    - <DNT>**APM: Exploit Probability**</DNT>: Flags critical or high-severity vulnerabilities likely to be exploited (95th+ percentile EPSS).
    - <DNT>**INFRA: Under 3 Critical Severity**</DNT>: Checks that a host or container has fewer than 3 critical-severity vulnerabilities.
    - <DNT>**INFRA: Under 5 High Severity**</DNT>: Checks that a host or container has fewer than 5 high-severity vulnerabilities.
    - <DNT>**INFRA: Active Ransomware**</DNT>: Flags reported CVEs that have an active ransomware campaign.
    - <DNT>**INFRA: Exploit Probability**</DNT>: Flags critical or high-severity vulnerabilities likely to be exploited (95th+ percentile EPSS).
  </Collapser>

  <Collapser
    id="ai-quality-governance"
    title={<DNT>AI Quality & Governance</DNT>}
  >
    **Purpose:** Measures the operational quality and governance of AI responses across safety, cost efficiency, and model compliance.

    **Importance:** Highlights runtime errors, token cost, and model usage so you can manage AI responsibly and cost-effectively.

    **Prerequisites:** [AI monitoring](/docs/ai-monitoring/intro-to-ai-monitoring)

**Key rules:**
    - <DNT>**LLM Runtime Error Rate**</DNT>: Checks that LLM-enabled applications maintain a low operational error rate.
    - <DNT>**Cost Efficiency - Average tokens per assistant response**</DNT>: Checks that the average token count per assistant response stays within budget.
    - <DNT>**Model Governance - Approved model usage for assistant responses**</DNT>: Checks that only approved models are used for assistant responses in production.
  </Collapser>

</CollapserGroup>
