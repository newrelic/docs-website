If you use the infrastructure agent or [on-host integrations](/docs/integrations/host-integrations/get-started/introduction-host-integrations) that report data via the infrastructure agent, there are several ways to configure data reporting. Here are two common options for managing data reporting:

* [Enable/disable process metrics](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#enable-process-metrics)
* [Select specific attributes to report](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#include-matching-metrics)

For other agent configuration options, see [Configuration](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings).

For our infrastructure integrations, you can also change the frequency of data reporting:

* For [on-host integrations](/docs/infrastructure/host-integrations/host-integrations-list): use a specific integration's `interval` [configuration](/docs/integrations/integrations-sdk/file-specifications/host-integration-configuration-overview) setting.
* For cloud integrations (AWS, Azure, Google Cloud): edit the [polling frequency](/docs/integrations/new-relic-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations).

For more about New Relic data management in general, see [Manage data](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-your-data).
