[New Relic infrastructure integrations](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 
now include an integration for AWS Security Lake, allowing you to send your security log data to New Relic. 

Collect and send telemetry data to New Relic from [Security Lake](https://aws.amazon.com/security-lake/) using our integration. You can use this integration to monitor your services, query incoming data, and build dashboards to observe everything at a glance.

## Activate integration [#activate]

To enable this integration, set up S3 log forwarder. We suggest using our application for ease and convenience, but you can also set your own.

## Configuration Steps [#steps]

1. [Create a Security Lake Subscriber.](/#createSubscriber)
2. [Install our dedicated Amazon Security Lake Log Forwarder.](/#installLogForwarder)
3. [Find and use your Log data](/#logs)

## Create Security Lake Subscriber [#createSubscriber]

1. Navigate to the Security Lake feature in your [AWS Console](console.aws.amazon.com/securitylake).
2. Select the `Subscribers` option, and select `Create Subscriber`
3. Give the Subscriber a `name` and select a `region`
4. Select which log and event sources you want sent to New Relic 
4. fill out the remaining details as described below:

</Callout>

## Install our Amazon Security Lake Log Forwarder [#installLogForwarder]

For more details on this step, see our [Amazon Security Lake log forwarder docs](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3/).

1. Open the [AWS Serverless Application Repository](https://serverlessrepo.aws.amazon.com/applications) in your browser.
   <img title="AWS Lambda - Select region" alt="AWS Lambda - Select region" src={serverlessAWSLambdaSelectRegion}/>
2. Search for `newrelic` and check **Show apps that create custom IAM roles or resource policies** to find `newrelic-securitylake-s3-processor-LogForwarder`.
4. Click the `newrelic-securitylake-s3-processor-LogForwarder` details and click **Deploy**.
5. From the subscriber creation in the previous step, you will need to copy your `SecurityLakeSQSArn` and `SecurityLakeSubscriberRoleArn` into the fields with the same name here.
6. Input the `ExternalID` that you added in the previous step.
7. Input your <InlinePopover type="licenseKey" /> into the `NRLicenseKey` field
8. Acknowledge and select `Deploy`

## Find and use Log data [#logs]

To find your logs on New Relic, go to **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Logs** and set **Attributes** to `product.name`, choose one or few of your interest.

The followings source types of logs are currently supported:

* [ROUTE 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [VPC FLOW](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [CLOUDTRAIL](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [SECURITY HUB](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)


<Callout variant="info">
  Amazon Security Lake uses the [OCSF Schema](https://schema.ocsf.io/) for it's logs.
</Callout>
