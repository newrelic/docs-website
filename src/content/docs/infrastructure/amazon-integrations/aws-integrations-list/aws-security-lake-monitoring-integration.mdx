[New Relic infrastructure integrations](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 
now include an integration for AWS Security Lake, sending its metrics data to New Relic. 

Collect and send telemetry data to New Relic from your [Security Lake](https://aws.amazon.com/security-lake/) using our integration. Monitor your services, query incoming data, and build dashboards to observe everything at a glance.

## Activate integration [#activate]

To enable this integration, set up S3 log forwarder. We suggest to use our application for ease and convenience but you can also set your own.

## Configuration 

Install our dedicated [Amazon Security Lake log forwarder](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3/) but instead of Lambda trigger, use AWS SQS service. Set up variables asked by the app accordingly.

## Logs

To find your logs on New Relic, go to **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Logs** and set **Attributes** to `product.name`, choose one or few of your interest.

4 source types of logs are currently supported:

* [ROUTE 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [VPC FLOW](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [CLOUD TRAIL](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [SECURITY HUB](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)
