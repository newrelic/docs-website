[New Relic infrastructure integrations](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) include an Amazon Rekognition integration for reporting your Rekognition data to New Relic. Here, we explain how to activate our integration and what data can be reported.

Enable the [AWS CloudWatch Metric Streams integration](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-metric-stream/) to monitor all CloudWatch metrics from your AWS services, including custom namespaces. Individual integrations are no longer our recommended option.

## Features [#features] 

[Amazon Rekognition](https://aws.amazon.com/rekognition/) (Rekognition) is an AWS service to automate and lower the cost of your image recognition and video analysis with machine learning. With New Relic's Rekognition integration, you'll be able to monitor number of requests, response times, detected face count, and more. You'll be able to create custom charts of your data and [create alert conditions](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information) based on changes in Rekognition data or configuration.

## Activate integration [#activate]

To enable this integration, see how to [connect AWS services to New Relic](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream/#set-up-metric-stream).

## Find and use data [#find-data]

Data from this integration is attached to the Metric data type. Go to **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Metrics & Events** and filter by `aws.rekognition` to find this data in the UI.

You can [query and explore your data](/docs/using-new-relic/data/understand-data/query-new-relic-data) using the **Metric** table and `namespace='AWS/Rekognition'`.

You can see two entities for AWS Rekognition on New Relic as mentioned below, one for custom projects and another for Operations.
