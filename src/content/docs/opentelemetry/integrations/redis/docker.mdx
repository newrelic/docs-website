<Steps>

<Step>

### Before you begin [#prereq-nrdot]

Ensure you have:

* **Docker** and **Docker Compose** (v2+) installed
* Your New Relic <InlinePopover type="licenseKey"/>
* **Redis** running and accessible (version 6.0+ recommended, 4.0+ supported with limited metrics)
* Outbound HTTPS (port 443) to New Relic's [OTLP endpoint](/docs/opentelemetry/best-practices/opentelemetry-otlp)

</Step>

<Step>

### Create the collector configuration [#config-nrdot]

Create `otel-collector-config.yaml` in your project directory:

```yaml
extensions:
  health_check:
    endpoint: "0.0.0.0:13133"

receivers:
  redis:
    endpoint: "redis:6379"  # Update with your Redis service name:port
    collection_interval: 10s
    metrics:
      redis.maxmemory:
        enabled: true
      redis.role:
        enabled: false
      redis.cmd.calls:
        enabled: true
      redis.cmd.usec:
        enabled: true
      redis.clients.max_input_buffer:
        enabled: false
      redis.clients.max_output_buffer:
        enabled: false
      redis.replication.backlog_first_byte_offset:
        enabled: false
    resource_attributes:
      server.address:
        enabled: true
      server.port:
        enabled: true

processors:
  memory_limiter:
    check_interval: 5s
    limit_mib: 512
    spike_limit_mib: 128

  resourcedetection:
    detectors: [env, system]
    timeout: 5s
    override: false
    system:
      resource_attributes:
        host.name:
          enabled: true
        host.id:
          enabled: true

  # Uncomment the section below to use a custom human-readable name for your
  # Redis entity instead of the default server.address:server.port identifier.
  # resource/redis:
  #   attributes:
  #     - key: redis.instance.id
  #       value: "my-redis-instance"
  #       action: upsert

  attributes/entity_tags:
    actions:
      - key: instrumentation.provider
        value: opentelemetry
        action: upsert

  cumulativetodelta:
    include:
      match_type: regexp
      metrics:
        - redis\.commands\.processed
        - redis\.connections\.received
        - redis\.connections\.rejected
        - redis\.keys\.evicted
        - redis\.keys\.expired
        - redis\.keyspace\.hits
        - redis\.keyspace\.misses
        - redis\.net\.input
        - redis\.net\.output
        - redis\.cpu\.time
        - redis\.cmd\.calls
        - redis\.cmd\.usec
        - redis\.uptime

  filter/cardinality:
    metrics:
      datapoint:
        - 'metric.name == "redis.cpu.time" and attributes["state"] != "sys" and attributes["state"] != "user"'
        - 'metric.name == "redis.cmd.calls" and attributes["cmd"] != "get" and attributes["cmd"] != "set" and attributes["cmd"] != "del" and attributes["cmd"] != "hget" and attributes["cmd"] != "hset" and attributes["cmd"] != "hgetall" and attributes["cmd"] != "lpush" and attributes["cmd"] != "rpop" and attributes["cmd"] != "zadd" and attributes["cmd"] != "expire"'
        - 'metric.name == "redis.cmd.usec" and attributes["cmd"] != "get" and attributes["cmd"] != "set" and attributes["cmd"] != "del" and attributes["cmd"] != "hget" and attributes["cmd"] != "hset" and attributes["cmd"] != "hgetall" and attributes["cmd"] != "lpush" and attributes["cmd"] != "rpop" and attributes["cmd"] != "zadd" and attributes["cmd"] != "expire"'

  transform/metadata_nullify:
    metric_statements:
      - context: metric
        statements:
          - set(description, "")
          - set(unit, "")

  batch:
    send_batch_size: 2048
    send_batch_max_size: 4096
    timeout: 10s

exporters:
  otlphttp:
    endpoint: ${env:OTEL_EXPORTER_OTLP_ENDPOINT}
    headers:
      api-key: ${env:NEW_RELIC_LICENSE_KEY}
    compression: gzip

service:
  extensions: [health_check]
  pipelines:
    metrics:
      receivers: [redis]
      processors: [memory_limiter, resourcedetection, attributes/entity_tags, cumulativetodelta, filter/cardinality, transform/metadata_nullify, batch]
      exporters: [otlphttp]
```

<Callout variant="tip">
**Want a user-friendly entity name?** By default, your Redis entity is named using the `server.address:server.port` combination. To use a custom human-readable name instead, uncomment the `resource/redis` section in the config above, set your preferred name in the `redis.instance.id` value, and add `resource/redis` to the processors pipeline.
</Callout>

</Step>

### Optional: Configure authentication [#auth-nrdot]

If your Redis instance requires authentication, add the credentials to the `redis` receiver section in your config:

<CollapserGroup>

  <Collapser id="nrdot-docker-auth-password" title="Legacy password (requirepass)">

```yaml
receivers:
  redis:
    endpoint: "redis:6379"
    password: "${env:REDIS_PASSWORD}"
```

Add `REDIS_PASSWORD` to your `.env` file:

```bash
REDIS_PASSWORD=your-redis-password
```

  </Collapser>

  <Collapser id="nrdot-docker-auth-acl" title="ACL username + password (Redis 6+)">

```yaml
receivers:
  redis:
    endpoint: "redis:6379"
    username: "${env:REDIS_USERNAME}"
    password: "${env:REDIS_PASSWORD}"
```

Add to your `.env` file:

```bash
REDIS_USERNAME=otel-monitor
REDIS_PASSWORD=your-monitor-password
```

  </Collapser>

  <Collapser id="nrdot-docker-auth-tls" title="TLS/SSL encrypted connection">

```yaml
receivers:
  redis:
    endpoint: "redis:6380"
    password: "${env:REDIS_PASSWORD}"
    tls:
      insecure: false
      ca_file: /etc/ssl/redis/ca.crt
      cert_file: /etc/ssl/redis/client.crt
      key_file: /etc/ssl/redis/client.key
```

### Optional: Collect Redis logs [#logs-nrdot]

Add the filelog receiver to your `otel-collector-config.yaml`:

```yaml
receivers:
  # ... existing redis receiver ...
  filelog/redis:
    include:
      - /var/log/redis/redis.log
    start_at: end
    operators:
      - type: regex_parser
        regex: '^(?P<pid>\d+):(?P<role>[XCSM]) (?P<timestamp>\d+ \w+ \d+ \d+:\d+:\d+\.\d+) (?P<level>[.*#\-]) (?P<message>.*)$'
        timestamp:
          parse_from: attributes.timestamp
          layout: '%d %b %Y %H:%M:%S.%L'
    resource:
      db.system: redis
```

Add a logs pipeline to the service section:

```yaml
service:
  pipelines:
    metrics:
      # ... existing metrics pipeline ...
    logs:
      receivers: [filelog/redis]
      processors: [memory_limiter, resourcedetection, attributes/entity_tags, batch]
      exporters: [otlphttp]
```


</Step>

<Step>

### Create the Docker Compose file [#compose-nrdot]

Create `docker-compose.yml` to run the collector alongside your existing Redis. The `.env` file in the same directory is automatically loaded by Docker Compose:

```yaml
services:
  otel-collector:
    image: newrelic/nrdot-collector:latest
    env_file: .env
    volumes:
      - ./otel-collector-config.yaml:/etc/otel/config.yaml:ro
      - /etc/machine-id:/etc/machine-id:ro
      # Uncomment if collecting Redis logs:
      # - /var/log/redis:/var/log/redis:ro
      # Uncomment if collecting host metrics:
      # - /:/hostfs:ro
      # Uncomment if using TLS:
      # - ./certs:/etc/ssl/redis:ro
    environment:
      - NEW_RELIC_LICENSE_KEY=${NEW_RELIC_LICENSE_KEY}
      - OTEL_EXPORTER_OTLP_ENDPOINT=${OTEL_EXPORTER_OTLP_ENDPOINT}
      # Uncomment if collecting host metrics:
      # - HOST_PROC=/hostfs/proc
      # - HOST_SYS=/hostfs/sys
      # - HOST_ETC=/hostfs/etc
      # Uncomment if using Redis authentication:
      # - REDIS_PASSWORD=${REDIS_PASSWORD}
      # - REDIS_USERNAME=${REDIS_USERNAME}
    command: ["--config=/etc/otel/config.yaml"]
    ports:
      - "13133:13133"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:13133"]
      interval: 10s
      timeout: 5s
      retries: 3
```

<Callout variant="tip">
If you enabled any optional features (logs, host metrics, TLS, or authentication) in the previous steps, uncomment the corresponding lines in the Docker Compose file above.
</Callout>

</Step>

<Step>

### Create the environment file [#env-nrdot]

Create a `.env` file in the same directory as your `docker-compose.yml`:

```bash
NEW_RELIC_LICENSE_KEY=YOUR_LICENSE_KEY
OTEL_EXPORTER_OTLP_ENDPOINT=https://otlp.nr-data.net
```

</Step>

<Step>

### Start and verify [#verify-nrdot]

```bash
docker compose up -d
```

Verify data in New Relic:

```sql
SELECT count(*) FROM Metric WHERE metricName LIKE 'redis.%' AND instrumentation.provider = 'opentelemetry' SINCE 5 minutes ago
```

</Step>

          metric_relabel_configs:
            - source_labels: [__name__]
              regex: '(go_|process_|promhttp_|redis_exporter_).*'
              action: drop

processors:
  memory_limiter:

### Create the Docker Compose file [#compose-prometheus]

Create `docker-compose.yml` to run the redis_exporter and collector alongside your existing Redis. The `.env` file in the same directory is automatically loaded by Docker Compose:

```yaml
services:
  redis-exporter:
    image: oliver006/redis_exporter:latest
    environment:
      - REDIS_ADDR=redis://redis:6379  # Update with your Redis service name:port
      # Uncomment if using Redis authentication:
      # - REDIS_PASSWORD=${REDIS_PASSWORD}
      # - REDIS_USER=${REDIS_USERNAME}
    ports:
      - "9121:9121"
    healthcheck:
      test: ["CMD", "wget", "-q", "--spider", "http://localhost:9121/metrics"]
      interval: 10s
      timeout: 5s
      retries: 3

  otel-collector:
    image: otel/opentelemetry-collector-contrib:latest
    env_file: .env
    volumes:
      - ./otel-collector-config.yaml:/etc/otelcol-contrib/config.yaml:ro
      - /etc/machine-id:/etc/machine-id:ro
    environment:
      - NEW_RELIC_LICENSE_KEY=${NEW_RELIC_LICENSE_KEY}
      - OTEL_EXPORTER_OTLP_ENDPOINT=${OTEL_EXPORTER_OTLP_ENDPOINT}
    command: ["--config=/etc/otelcol-contrib/config.yaml"]
    ports:
      - "13133:13133"
    depends_on:
      redis-exporter:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:13133"]
      interval: 10s
      timeout: 5s
      retries: 3
```

<Callout variant="tip">
To use the NRDOT collector instead, replace the `otel-collector` image with `newrelic/nrdot-collector:latest` and update the config mount path to `/etc/otel/config.yaml`.
</Callout>
