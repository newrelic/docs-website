New Relic now empower your team to monitor your Oracle database performance using OpenTelemetry (OTEL) with New Relic's database monitoring capabilities. This integration provides comprehensive insights into database metrics, query performance, and system health using the OpenTelemetry Collector.

This OTEL-based approach complements our existing On-Host Integration (OHI) by leveraging OpenTelemetry standards for database monitoring, making it easier to integrate with your existing observability stack.

## Prerequisites [#prerequisites]

Before you begin, ensure you have the following:


* Valid New Relic [license key](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)
* Linux on `AMD64` architecture with following distributions supported:
  * RHEL 7 or later
  * CentOS 7 or later
  * OEL 7 or later
* Network connectivity to New Relic's OTLP endpoint
* Oracle Database 12c or later
* On the Oracle database, you've [Oracle Instant Client](http://www.oracle.com/technetwork/database/database-technologies/instant-client/downloads/index.html) on the machine running the OpenTelemetry Collector

### Configure OpenTelemetry Collector [#configure-collector]

1. Edit your `config.yaml` file available at `/etc/nrdot-collector-host/`, then add the following code snippet:

    <Callout variant="important">

      If you're an existing NRDOT Collector user, ensure to merge the above configuration with your existing `config.yaml` file. This includes adding the new receivers, processors, and exporters for Oracle Database monitoring while retaining your current configurations.
    </Callout>

```yaml
receivers:
  newrelicoracledb/cdb:
    endpoint: "<YOUR_DB_HOST>:<YOUR_DB_PORT>"
    username: "<YOUR_DB_USERNAME>"
    password: "<YOUR_DB_PASSWORD>"
    service: "<YOUR_CDB_SERVICE_NAME>"
    collection_interval: 30s
    enable_query_monitoring: false

  newrelicoracledb/pdb1:
    endpoint: "<YOUR_DB_HOST>:<YOUR_DB_PORT>"
    username: "<YOUR_DB_USERNAME>"
    password: "<YOUR_DB_PASSWORD>"
    service: "<YOUR_PDB_SERVICE_NAME>"
    collection_interval: 30s
    query_monitoring_response_time_threshold: 0 
    query_monitoring_count_threshold: 20
    enable_query_monitoring: true 
    enable_interval_based_averaging: true
    interval_calculator_cache_ttl_minutes: 10
    query_monitoring_interval_seconds: 30 

exporters:
  debug:
    verbosity: detailed
  otlp:
    endpoint: "<YOUR_NEWRELIC_OTLP_ENDPOINT>" # refer New Relic OTLP docs at https://docs.newrelic.com/docs/opentelemetry/best-practices/opentelemetry-otlp/
    headers:
      api-key: ""
    compression: gzip

processors:
  resource:
    attributes:
      - key: host.address
        value: "<YOUR_DB_HOST>"
        action: upsert
      - key: host.port
        value: "<YOUR_DB_PORT>"
        action: upsert
  resourcedetection/db_safe:
    detectors: ["system"]
    override: false  
    system:
      hostname_sources: ["os"]
      resource_attributes:
        host.id:
          enabled: true

service:
  pipelines:
    metrics:
      receivers: [newrelicoracledb/cdb, newrelicoracledb/pdb1]
      processors: [resourcedetection/db_safe, resource]
      exporters: [debug, otlp]
    logs/newrelicoracledb:
      receivers: [newrelicoracledb/pdb1]
      processors: [resourcedetection/db_safe, resource]
      exporters: [otlp, debug]
  telemetry:
    logs:
      level: info
    metrics:
      level: detailed
```
