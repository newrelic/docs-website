After setting up OpenTelemetry monitoring for your Amazon ECS environment, you can find and query your data across New Relic's Infrastructure and query interfaces. This guide helps you locate your ECS metrics and create meaningful queries for monitoring and troubleshooting.

## Where to find your ECS data [#find-data]

Your ECS monitoring data appears in multiple locations within New Relic, depending on the type of metrics and your ECS launch type.

### Infrastructure monitoring [#infrastructure-view]

Navigate to **[one.newrelic.com](https://one.newrelic.com) > All Capabilities > Infrastructure** to view your ECS infrastructure:

#### For ECS on EC2
1. **Hosts view**: Filter by host to see your EC2 instances running ECS containers
2. **Containers view**: See individual container performance within your ECS tasks
3. **Services view**: Monitor your ECS services and their health status

#### For ECS Fargate
1. **Containers view**: Primary location for Fargate container metrics
2. **Services view**: Monitor your ECS services running on Fargate

<Callout variant="tip">
Use the filter options to narrow down to specific clusters, services, or container names to quickly find the data you need.
</Callout>

### Metrics and events [#metrics-events]

Go to **[one.newrelic.com](https://one.newrelic.com) > All Capabilities > Query your data** to:
- Create custom charts and dashboards
- Set up alerts based on specific metric thresholds
- Perform detailed analysis with NRQL queries

## Essential attributes for filtering [#key-attributes]

Use these attributes to filter and group your ECS metrics:

- **`ClusterName`**: ECS cluster name
- **`ServiceName`**: ECS service name
- **`TaskId`**: Unique task identifier
- **`TaskDefinitionFamily`**: Task definition family name
- **`LaunchType`**: Either "EC2" or "FARGATE"
- **`container.name`**: Individual container name
- **`container.image.name`**: Container image information

## Query examples [#query-examples]

### Monitor container CPU utilization

```sql
SELECT average(container.cpu.utilization)
FROM Metric
WHERE ClusterName = 'your-cluster-name'
AND ServiceName = 'your-service-name'
TIMESERIES AUTO
FACET container.name
```

### Track memory usage trends

```sql
SELECT max(container.memory.usage.total) / 1024 / 1024 as 'Memory (MB)'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET ServiceName
```

### Compare performance across launch types

```sql
SELECT average(container.cpu.utilization)
FROM Metric
TIMESERIES AUTO
FACET LaunchType
WHERE ClusterName = 'your-cluster-name'
```

### Monitor host-level metrics (EC2 only)

```sql
SELECT average(system.cpu.utilization) as 'Host CPU %',
       average(system.memory.utilization) as 'Host Memory %'
FROM Metric
WHERE LaunchType = 'EC2'
AND ClusterName = 'your-cluster-name'
TIMESERIES AUTO
```

### Identify containers with high network traffic

```sql
SELECT rate(sum(container.network.io), 1 minute) as 'Network I/O per minute'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET container.name
ORDER BY 'Network I/O per minute' DESC
```

### Monitor storage I/O patterns

```sql
SELECT rate(sum(container.blockio.io_service_bytes_recursive), 1 minute) as 'Storage I/O Rate'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET operation, ServiceName
```

## Create custom dashboards [#custom-dashboards]

1. Navigate to **[one.newrelic.com](https://one.newrelic.com) > All Capabilities > Dashboards**
2. Click **Create a dashboard**
3. Add charts using the query examples above
4. Organize by ECS clusters, services, or container performance views

### Recommended dashboard widgets

- **Container health overview**: CPU and memory utilization per service
- **Network performance**: I/O trends across containers
- **Host resource usage** (EC2 only): System-level CPU, memory, and disk metrics
- **Service scaling metrics**: Task counts and resource allocation trends

## Set up alerts [#setup-alerts]

Create proactive monitoring with alert conditions:

### High container CPU usage
```sql
SELECT average(container.cpu.utilization)
FROM Metric
WHERE ClusterName = 'your-cluster-name'
```
**Threshold**: Alert when above 80% for 5 minutes

### Memory exhaustion warning
```sql
SELECT average(container.memory.usage.total)
FROM Metric
WHERE ServiceName = 'critical-service'
```
**Threshold**: Alert when approaching container memory limits

### Host resource alerts (EC2 only)
```sql
SELECT average(system.memory.utilization)
FROM Metric
WHERE LaunchType = 'EC2'
```
**Threshold**: Alert when system memory exceeds 85%

## Troubleshooting data gaps [#troubleshooting]

If you don't see expected data:

1. **Verify collector status**: Check container logs for the OpenTelemetry collector
2. **Confirm network connectivity**: Ensure outbound HTTPS access to New Relic
3. **Check license key**: Verify the license key is correctly configured in SSM Parameter Store
4. **Review task definition**: Confirm the collector container is running with proper resource allocation
