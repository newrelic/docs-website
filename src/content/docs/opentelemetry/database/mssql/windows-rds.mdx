<Step>

### Setup the NRDOT Collector [#setup-sql]

Install the NRDOT Collector on an EC2 Windows instance or a local Windows machine that can connect to your RDS SQL Server instance.

<Callout variant="important">
  For RDS monitoring, install the NRDOT Collector on an EC2 instance in the same VPC as your RDS instance for optimal network connectivity and lower latency.
</Callout>

1. Download and install the NRDOT Collector using PowerShell:

   ```powershell
   $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i nrdot-collector.msi /qn /norestart" -Wait
   ```

2. Ensure your security group allows inbound connections from the NRDOT Collector host to your RDS instance on port 1433 (or your custom port).

</Step>

<Step>

### Configure database user [#configure-user-sql]

Connect to your RDS SQL Server instance using SQL Server Management Studio (SSMS) or sqlcmd with your admin credentials.

Run the following script to create the `newrelic` monitoring user with appropriate permissions:

```sql
USE [master];
GO
CREATE LOGIN [newrelic] WITH PASSWORD = '<PASSWORD>';
GO

GRANT VIEW SERVER STATE TO [newrelic];
GRANT VIEW ANY DEFINITION TO [newrelic];
GRANT VIEW ANY DATABASE TO [newrelic];
GO

DECLARE @name SYSNAME;
DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
SELECT [name]
FROM [master].[sys].[databases]
WHERE [name] NOT IN ('master', 'msdb', 'model', 'rdsadmin', 'distribution')
AND [state] = 0;
OPEN db_cursor;
FETCH NEXT FROM db_cursor INTO @name;
WHILE @@FETCH_STATUS = 0
BEGIN
  BEGIN TRY
    EXEC('USE [' + @name + '];
    IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''newrelic'')
    BEGIN
      CREATE USER [newrelic] FOR LOGIN [newrelic];
    END;
    GRANT VIEW DATABASE STATE TO [newrelic];');
  END TRY
  BEGIN CATCH
    PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
  END CATCH
  FETCH NEXT FROM db_cursor INTO @name;
END
CLOSE db_cursor;
DEALLOCATE db_cursor;
GO
```

<Callout variant="important">
  **For RDS:** Use your RDS master username (or a user with `rds_superuser` role) to execute these commands. The script automatically excludes the `rdsadmin` database which is reserved by AWS.
</Callout>

</Step>

### Update Windows Service Registry [#update-registry-sql]

Update the Windows Service Registry to use the new mssql-config.yaml file.

**Method 1: PowerShell**

Run the following command in PowerShell as Administrator:

```powershell
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
```

**Method 2: Command Prompt (Alternative)**

If PowerShell is not available, you can use Command Prompt instead:

1. Open your Start Menu, type `cmd`
2. Right-click Command Prompt and choose **Run as administrator**
3. Copy and paste this exact command into the black CMD window and hit Enter:

```cmd
sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
```

Install the NRDOT Collector on a domain-joined EC2 Windows instance in the same VPC as your RDS SQL Server instance.

<Callout variant="important">
  For RDS monitoring with Windows Authentication, the EC2 instance running the NRDOT Collector must be joined to the same Active Directory domain as your RDS instance.
</Callout>

1. Download and install the NRDOT Collector using PowerShell:

   ```powershell
   $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i nrdot-collector.msi /qn /norestart" -Wait
   ```

2. Ensure your security groups allow:
   - Inbound connection from EC2 to RDS on port 1433 (SQL Server)
   - Outbound connections from EC2 to Active Directory (ports 53, 88, 135, 389, 445, 464, 636, 3268, 3269)

</Step>

### Configure SQL Server permissions [#configure-permissions-domain]

Connect to your RDS SQL Server instance and grant the necessary permissions to your domain user.

1. **Create the domain login** (if it doesn't exist):

```sql
USE [master];
GO
CREATE LOGIN [DomainName\DomainUser] FROM WINDOWS;
GO

GRANT VIEW SERVER STATE TO [DomainName\DomainUser];
GRANT VIEW ANY DEFINITION TO [DomainName\DomainUser];
GRANT VIEW ANY DATABASE TO [DomainName\DomainUser];
GO
```

2. **Grant database-level permissions** across all user databases.

Use the following statements to grant READ access privileges to the domain user across all databases:

```sql
DECLARE @principal SYSNAME = N'DomainName\DomainUser';   

DECLARE @name SYSNAME;
DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
      SELECT [name] FROM master.sys.databases                                                                                                              
      WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
        AND [state] = 0;                                                                                                           
  OPEN db_cursor;                                                                                                                                        
  FETCH NEXT FROM db_cursor INTO @name;                                                                                                                    
  WHILE @@FETCH_STATUS = 0                                                                                                                                 
  BEGIN                                                                                                                                                    
      BEGIN TRY                                                                                                          
          DECLARE @sql nvarchar(max) = N'                                                                                                                  
              USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''' + @principal + ''')                                                    
              BEGIN                                                                                                                                        
                  CREATE USER [' + @principal + '] FOR LOGIN [' + @principal + '];                                                                        
              END;                                                                                                                                         
              GRANT VIEW DATABASE STATE TO [' + @principal + '];                                                                                          
              ALTER ROLE db_datareader ADD MEMBER [' + @principal + '];';                                                                                 
          EXEC sp_executesql @sql;                                                                                                                        
          PRINT 'Success: ' + @name;                                                                                                                      
      END TRY                                                                                                                                              
      BEGIN CATCH                                                                                                                                          
          PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();                                                                                             
      END CATCH                                                                                                                                            
      FETCH NEXT FROM db_cursor INTO @name;                                                                                                               
  END                                                                                                                                                      
  CLOSE db_cursor;                                                                                                                                        
  DEALLOCATE db_cursor;
GO
```

<Callout variant="important">
  Replace `DomainName\DomainUser` with your actual domain and username. For RDS with AWS Managed Microsoft AD, the format is typically `corp\username` or `yourdomain\username`.
</Callout>

</Step>

```yaml
        enabled: true
      sqlserver.database.tempdb.version_store.size:
        enabled: true
      sqlserver.deadlock.rate:
        enabled: true
      sqlserver.os.wait.duration:
        enabled: true
      sqlserver.processes.blocked:
        enabled: true
      sqlserver.memory.grants.pending.count:
        enabled: true
      sqlserver.database.file.size:
        enabled: true
      sqlserver.memory.area:
        enabled: true

    # Enable top query and query sample log collection
    events:
      db.server.query_sample:
        enabled: true
      db.server.top_query:
        enabled: true

    # Top query collection configuration

      lookback_time: 60s

exporters:
  otlphttp:
    endpoint: <YOUR_NEWRELIC_OTLP_ENDPOINT>

  telemetry:

      level: none

    metrics:
      receivers: [nrsqlserver]
```

### Configure NRDOT service to run as domain user [#configure-service-domain]

```powershell
# Stop the NRDOT Collector service
Stop-Service nrdot-collector

# Configure service to run as domain user
$username = "DomainName\DomainUser"
$password = Read-Host -AsSecureString "Enter password for $username"
$credential = New-Object System.Management.Automation.PSCredential($username, $password)

# Set service to run as domain user
$service = Get-WmiObject -Class Win32_Service -Filter "Name='nrdot-collector'"
$service.Change($null,$null,$null,$null,$null,$null,$username,$credential.GetNetworkCredential().Password)

# Verify the configuration
Get-WmiObject Win32_Service -Filter "Name='nrdot-collector'" | Select Name, StartName
```

<Callout variant="important">
  Replace `DomainName\DomainUser` with your actual domain and username. The service must run under a domain account that has the permissions granted in the previous step.
</Callout>

Use the following statements to grant READ access privileges to the gMSA account across all databases:

```sql
DECLARE @principal SYSNAME = N'DomainName\DomainUser';   

DECLARE @name SYSNAME;
DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
      SELECT [name] FROM master.sys.databases                                                                                                              
      WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
        AND [state] = 0;                                                                                                           
  OPEN db_cursor;                                                                                                                                        
  FETCH NEXT FROM db_cursor INTO @name;                                                                                                                    
  WHILE @@FETCH_STATUS = 0                                                                                                                                 
  BEGIN                                                                                                                                                    
      BEGIN TRY                                                                                                          
          DECLARE @sql nvarchar(max) = N'                                                                                                                  
              USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''' + @principal + ''')                                                    
              BEGIN                                                                                                                                        
                  CREATE USER [' + @principal + '] FOR LOGIN [' + @principal + '];                                                                        
              END;                                                                                                                                         
              GRANT VIEW DATABASE STATE TO [' + @principal + '];                                                                                          
              ALTER ROLE db_datareader ADD MEMBER [' + @principal + '];';                                                                                 
          EXEC sp_executesql @sql;                                                                                                                        
          PRINT 'Success: ' + @name;                                                                                                                      
      END TRY                                                                                                                                              
      BEGIN CATCH                                                                                                                                          
          PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();                                                                                             
      END CATCH                                                                                                                                            
      FETCH NEXT FROM db_cursor INTO @name;                                                                                                               
  END                                                                                                                                                      
  CLOSE db_cursor;                                                                                                                                        
  DEALLOCATE db_cursor;
GO
```

<Callout variant="important">
  Replace `<DOMAIN>\<gMSA_NAME>$` with your actual gMSA account name. The `$` at the end is required for gMSA accounts. For example: `corp\sqlgmsa$`
</Callout>

</Step>

### Configure NRDOT Collector [#configure-gmsa]

Create the configuration for gMSA authentication:

1. **Create the configuration file.** The filename must be `mssql-config.yaml`. Run the following command in PowerShell as Administrator:

```yaml
# NRDOT Collector Configuration for RDS SQL Server - gMSA Authentication

receivers:
  nrsqlserver:
    collection_interval: 15s
    datasource: "server=<HOSTNAME-OR-RDS-ENDPOINT>;integrated security=true;encrypt=true;TrustServerCertificate=true;"

    # Enable comprehensive database metrics
    metrics:
      sqlserver.database.count:
        enabled: true
      sqlserver.database.io:
        enabled: true
      sqlserver.database.latency:
        enabled: true
      sqlserver.database.operations:
        enabled: true
      sqlserver.database.tempdb.space:
        enabled: true
      sqlserver.database.tempdb.version_store.size:
        enabled: true
      sqlserver.deadlock.rate:
        enabled: true
      sqlserver.os.wait.duration:
        enabled: true
      sqlserver.processes.blocked:
        enabled: true
      sqlserver.memory.grants.pending.count:
        enabled: true
      sqlserver.database.file.size:
        enabled: true
      sqlserver.memory.area:
        enabled: true

    # Enable top query and query sample log collection
    events:
      db.server.query_sample:
        enabled: true
      db.server.top_query:
        enabled: true

    # Top query collection configuration
    top_query_collection:
      lookback_time: 60s
      max_query_sample_count: 1000
      top_query_count: 250
      collection_interval: 60s

    collect_full_query_text: true
    allowed_comment_keys:
      - nr_service_guid

    # Query sample collection configuration
    query_sample_collection:
      max_rows_per_query: 100

processors:
  memory_limiter:
    check_interval: ${env:NR_MEM_LIMITER_CHECK_INTERVAL:-1s}
    limit_mib:       ${env:NR_MEM_LIMITER_LIMIT_MIB:-200}
    spike_limit_mib: ${env:NR_MEM_LIMITER_SPIKE_MIB:-50}

  batch:

exporters:
  otlphttp:
    endpoint: <YOUR_NEWRELIC_OTLP_ENDPOINT>
    headers:
      api-key: <YOUR_NEWRELIC_LICENSE_KEY>

      insecure: false
    compression: gzip

  debug:
    verbosity: detailed

service:
  telemetry:
    metrics:
      level: none

  pipelines:
    metrics:
      receivers: [nrsqlserver]
      processors: [memory_limiter, batch]
      exporters: [otlphttp, debug]

    logs:
      receivers: [nrsqlserver]
      processors: [memory_limiter, batch]
      exporters: [otlphttp, debug]
```

- `<HOSTNAME-OR-RDS-ENDPOINT>`: Your RDS SQL Server endpoint

</Callout>
