Troubleshoot common issues you may encounter when monitoring your Microsoft SQL Server with the NRDOT collector.

## Common issues

<CollapserGroup>

  <Collapser
    id="network-connectivity"
    title="Network connectivity issues"
  >

If you encounter any network connectivity issues with the NRDOT Collector, run the following commands to test connectivity to your SQL Server instance on port 1433:

* For Windows, run:

  ```shell
  Test-NetConnection -ComputerName <YOUR_IP_ADDRESS> -Port <YOUR_PORT_NUMBER>
  ```

* For Linux, run:

  ```bash
  nc -zv <YOUR_IP_ADDRESS> <YOUR_PORT_NUMBER>
  ```

</Collapser>

  <Collapser
    id="ssl"
    title="SSL connection issues"
  >

  To connect to Microsoft SQL Server using SSL without validation of the certificate, ensure that the `enable_ssl` parameter is set to `true` and the `trust_server_certificate` parameter is set to `true` in your NRDOT Collector configuration.

  ```yaml
  enable_ssl: true                                                                                                                            
  trust_server_certificate: true
  ```

  </Collapser>

</CollapserGroup>
