Monitor your Oracle Database performance using New Relic Distribution of OpenTelemetry (NRDOT) with comprehensive insights into database metrics, query performance, and system health.

This NRDOT-based approach complements our existing On-Host Integration (OHI) by leveraging OpenTelemetry standards for database monitoring, making it easier to integrate with your existing observability stack.

## How you can configure NRDOT for Oracle Database [#configure]

<Steps>

<Step>

### Prerequisites [#prerequisites]

Ensure your environment meets our compatibility and prerequisites defined for Oracle Database monitoring with NRDOT.

</Step>

<Step>

### NRDOT Collector installation [#installation]

Choose your installation path based on your infrastructure:

- **[Guided install](/docs/opentelemetry/database/oracledb/guided)**: You can install the NRDOT Collector for Oracle Database monitoring using the guided install from the New Relic platform. Go to <DNT>[one.newrelic.com](https://one.newrelic.com/) > + Integration & Agents > Oracle (OpenTelemetry)**</DNT>. Follow the on-screen instructions to set up the integration for your environment.
- **Self-hosted environments**: Manual installation on your own infrastructure:
  - **[On-host Container Database (CDB)](/docs/opentelemetry/database/oracledb/on-host-cdb)**
  - **[On-host Pluggable Database (PDB)](/docs/opentelemetry/database/oracledb/on-host-pdb)**
- **[AWS RDS environments](/docs/opentelemetry/database/oracledb/rds)**: Manual installation for your Amazon RDS environments.

</Step>

<Step>

### Find and use your data [#find]

Once your data is being collected, you can access comprehensive Oracle Database monitoring through New Relic's UI.

To find your Oracle Database entity in New Relic:

1. Go to <DNT>**[https://one.newrelic.com](https://one.newrelic.com) > All Capabilities > Databases**</DNT>.
2. Set the search criteria as <DNT>`instrumentation.provider = opentelemetry`</DNT>.
3. Select your Oracle Database from the list of entities.

</Step>

</Steps>
