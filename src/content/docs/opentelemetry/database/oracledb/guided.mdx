You can install the NRDOT Collector for Oracle Database monitoring using the guided install from the New Relic platform, which provides a platform-guided setup through New Relic's UI. This is the recommended approach for quick starts.

<Steps>

<Step>

### Prerequisites [#prerequisites]

Before you begin, ensure you have the following:

* Valid New Relic [license key](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)
* Supported architecture: Linux with AMD64 and ARM64
* Network connectivity to [New Relic OTLP endpoint](/docs/opentelemetry/best-practices/opentelemetry-otlp)
* Oracle Database 19c or later
* This integration is available as a part of New Relic public preview program. Contact your Organization Manager to opt in from the [Previews & Trials](https://one.newrelic.com/admin-portal/promotion-management/home) page.

</Step>

<Step>

### Guided install [#guided]

To install the NRDOT Collector using the guided install:

1. Log in to your [New Relic account](https://one.newrelic.com/).
2. Go to <DNT>**left navigation pane > + Integration & Agents > Oracle (OpenTelemetry)**</DNT>.
3. On the <DNT>Select an account</DNT> screen, select the account you want to install the Oracle Database integration on, and click <DNT>**Continue**</DNT>.
4. On the <DNT>Enter your credential</DNT> screen, select one of the following options, then click **Continue**:
   * **Use an existing key**: If you already have a license key, provide the license key. For more information, refer to [User keys](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
   * **Create a new key**: If you don't have a license key, click **Create a new key** to create one.
5. On the <DNT> Compatibility and requirements</DNT> screen, ensure that all the requirements are met, and click <DNT>**Continue**</DNT>.
6. On the <DNT> Database user configuration</DNT> screen, select your deployment type to install the integration.
7. Follow the on-screen instructions to set up the integration for the selected environment.
8. After installation, on the <DNT>Test your integration</DNT> screen, click <DNT>**Test connection**</DNT> to verify that the NRDOT Collector can connect to your Oracle Database instance and send data to New Relic. If the test's successful, you'll see a confirmation message.

</Step>

<Step>

### (Optional) Set up APM-database correlation [#correlation]

To correlate your application performance with database operations, you can set up database service identification. This feature allows you to see exactly which applications are generating specific database workloads. For more information, refer to [set up database service identification](/docs/opentelemetry/database/database-service-identification/) to get APM-database correlation in New Relic.

<Callout variant="important">
  To view database performance data in APM, both entities must be in the same New Relic account. If the entities are in different New Relic accounts, you must have access to both accounts to view the data.
</Callout>

</Step>

<Step>

### (Optional) Configure secret management [#secret-management]

The secret management feature allows you to securely manage sensitive information, such as database credentials. This helps to enhance the security of your monitoring setup by avoiding hardcoding sensitive data in configuration files. For more information, refer to [secret management](/docs/opentelemetry/database/database-service-identification/#secret-management).

</Step>

<Step>

### Find and use your data [#find]

Once your data is being collected, you can access comprehensive Oracle Database monitoring through New Relic's UI.

To find your Oracle Database entity in New Relic:

1. Go to <DNT>**[https://one.newrelic.com](https://one.newrelic.com) > All Capabilities > Databases**</DNT>.
2. Set the search criteria as <DNT>`instrumentation.provider = opentelemetry`</DNT>.
3. Select your Oracle Database from the list of entities.

</Step>

</Steps>
