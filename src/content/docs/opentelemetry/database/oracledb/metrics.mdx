Monitor your Oracle Database performance with comprehensive metrics collected by the NRDOT collector. The following metrics are available for Oracle Database monitoring with NRDOT. Enable the relevant scrapers in your configuration to collect them.

## Available metrics [#otel-metrics]

The following metrics are available for Oracle Database monitoring with NRDOT. Enable the relevant scrapers in your configuration to collect them.

<CollapserGroup>

  <Collapser

    <table>

          <th>Metric name</th>

      <tbody>
        <tr>
          <td>`oracledb.cpu_time`</td>
          <td>Cumulative CPU time, in seconds</td>
        </tr>
        <tr>
          <td>`oracledb.dml_locks.limit`</td>
          <td>Maximum limit of active DML (Data Manipulation Language) locks, -1 if unlimited. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.dml_locks.usage`</td>
          <td>Current count of active DML (Data Manipulation Language) locks. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.enqueue_deadlocks`</td>
          <td>Total number of deadlocks between table or row locks in different sessions</td>
        </tr>
        <tr>
          <td>`oracledb.enqueue_locks.limit`</td>
          <td>Maximum limit of active enqueue locks, -1 if unlimited. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.enqueue_locks.usage`</td>
          <td>Current count of active enqueue locks. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.enqueue_resources.limit`</td>
          <td>Maximum limit of active enqueue resources, -1 if unlimited. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.enqueue_resources.usage`</td>
          <td>Current count of active enqueue resources. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.exchange_deadlocks`</td>
          <td>Number of times that a process detected a potential deadlock when exchanging two buffers and raised an internal, restartable error</td>
        </tr>
        <tr>
          <td>`oracledb.executions`</td>
          <td>Total number of calls (user and recursive) that executed SQL statements</td>
        </tr>
        <tr>
          <td>`oracledb.hard_parses`</td>
          <td>Number of hard parses</td>
        </tr>
        <tr>
          <td>`oracledb.logical_reads`</td>
          <td>Number of logical reads</td>
        </tr>
        <tr>
          <td>`oracledb.parse_calls`</td>
          <td>Total number of parse calls</td>
        </tr>
        <tr>
          <td>`oracledb.pga_memory`</td>
          <td>Session PGA (Program Global Area) memory</td>
        </tr>
        <tr>
          <td>`oracledb.physical_reads`</td>
          <td>Number of physical reads</td>
        </tr>
        <tr>
          <td>`oracledb.processes.limit`</td>
          <td>Maximum limit of active processes, -1 if unlimited. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.processes.usage`</td>
          <td>Current count of active processes. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.sessions.limit`</td>
          <td>Maximum limit of active sessions, -1 if unlimited. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.sessions.usage`</td>
          <td>Count of active sessions</td>
        </tr>
        <tr>
          <td>`oracledb.tablespace_size.limit`</td>
          <td>Maximum size of tablespace in bytes, -1 if unlimited</td>
        </tr>
        <tr>
          <td>`oracledb.tablespace_size.usage`</td>
          <td>Used tablespace size in bytes</td>
        </tr>
        <tr>
          <td>`oracledb.transactions.limit`</td>
          <td>Maximum limit of active transactions, -1 if unlimited. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.transactions.usage`</td>
          <td>Current count of active transactions. **Not supported in RDS.**</td>
        </tr>
        <tr>
          <td>`oracledb.user_commits`</td>
          <td>Number of user commits. When a user commits a transaction, the redo generated that reflects the changes made to database blocks must be written to disk. Commits often represent the closest thing to a user transaction rate.</td>
        </tr>
        <tr>
          <td>`oracledb.user_rollbacks`</td>
          <td>Number of times users manually issue the ROLLBACK statement or an error occurs during a user's transactions</td>
        </tr>
      </tbody>
    </table>
