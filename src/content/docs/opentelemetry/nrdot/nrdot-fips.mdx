New Relic supports FIPS compliance by providing versions of the `nrdot-collector`, designed for organizations that require Federal Information Processing Standards (FIPS) [140-3](https://csrc.nist.gov/pubs/fips/140-3/final) compliance.

## FIPS-compliant NRDOT features [#features]

The `nrdot-collector` supports FIPS 140-3 by using BoringCrypto, which is validated under [NIST Certificate #4735](https://csrc.nist.gov/projects/cryptographic-module-validation-program/certificate/4735), and ensuring that all TLS communications use approved cipher suites.

For detailed technical information and our validate procedures, see the [NRDOT FIPS technical documentation](https://github.com/newrelic/nrdot-collector-releases/blob/main/fips/README.md).

## Installation and usage [#installation]

Versions of the `nrdot-collector` that support FIPS are only available as docker images. They are hosted in the same docker repository but their version string has a `-fips` suffix, e.g. [newrelic/nrdot-collector:1.12.0-fips](https://hub.docker.com/layers/newrelic/nrdot-collector/1.12.0-fips/images/sha256-dfd0ee3c35f25338178dadfc95227b01b324093f635084960a1e8fcb693d07d2).

Apart from that, the regular [use case-specific](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector#use-cases) installation instructions apply but keep in mind that you will have to install NRDOT via docker as mentioned above.
