New Relic offers FIPS-compliant versions of the `nrdot-collector`, designed for organizations that require Federal Information Processing Standards (FIPS) [140-3](https://csrc.nist.gov/pubs/fips/140-3/final) compliance. These versions are ideal for government agencies, contractors, and enterprises operating in regulated environments that demand certified cryptographic modules.

## FIPS-compliant NRDOT features [#features]

The `nrdot-collector` achieves FIPS 140-3 compliance by using BoringCrypto, which is FIPS 140-3 validated under [NIST Certificate #4735](https://csrc.nist.gov/projects/cryptographic-module-validation-program/certificate/4735), and ensuring that all TLS communications use approved cipher suites. This provides the security assurances required for regulated environments while maintaining full OpenTelemetry compatibility.

For detailed technical information about the complete chain from NRDOT to the official NIST FIPS certificate, including the validation procedures, see the [NRDOT FIPS compliance technical documentation](https://github.com/newrelic/nrdot-collector-releases/blob/main/fips/README.md).

## Installation and usage [#installation]

FIPS-compliant versions of the `nrdot-collector` are only available as docker images. They are hosted in the same docker repository but their version string has a `-fips` suffix, e.g. [newrelic/nrdot-collector:1.12.0-fips](https://hub.docker.com/layers/newrelic/nrdot-collector/1.12.0-fips/images/sha256-dfd0ee3c35f25338178dadfc95227b01b324093f635084960a1e8fcb693d07d2).

Apart from that, the regular [use case-specific](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector#use-cases) installation instructions apply but keep in mind that you will have to install NRDOT via docker as mentioned above.
