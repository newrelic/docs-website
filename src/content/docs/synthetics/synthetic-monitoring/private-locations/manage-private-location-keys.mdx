We've updated the New Relic synthetics [private locations](/docs/synthetics/synthetic-monitoring/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations) UI. As part of our private location keys migration to secret store, we've removed key visibility from the list view, filter, and metadata areas.

## What's changing [#whats-changing]

* **Key column removed from list view**: The private location key column has been removed from the private locations list
* **Key removed from filter**: You can no longer filter private locations by key
* **Key removed from metadata & tags**: Private location keys are no longer displayed in the metadata & tags section of the private location overview page
* **Access through Edit only**: Keys are now only accessible through the Edit screen where you can copy them for your [synthetics job manager](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager) configuration

## Access your private location key [#access-key]

To access your private location key:

1. Go to <DNT>**[one.newrelic.com > Synthetic monitoring > Private locations](https://one.newrelic.com/synthetics-nerdlets/private-location-list)**</DNT>.
2. Find the private location for which you need the key.
3. Click the ellipsis icon <Icon name="fe-more-horizontal"/> next to the location.
4. Select <DNT>**Edit**</DNT>.
5. In the Edit screen, you'll see the <DNT>**Private location key**</DNT> field with options to copy the key for your synthetics job manager configuration.

## Filter and search private locations [#filter-locations]

You will no longer be able to search or filter your private locations by key. This option has been removed from the filter.

<img
  title="Filter private locations"
  alt="Screenshot showing the updated filter options without the key filter"
  src="/images/synthetic_screenshot-crop_private-location-filter.webp"
/>

<figcaption>
  The private location key filter has been removed from the UI.
</figcaption>

Instead, you can filter and search your private locations by:
* **Location name**: Search by the name you assigned to the private location
* **Status**: Filter by active or inactive locations
* **[VSE (Verified Script Execution)](/docs/synthetics/synthetic-monitoring/private-locations/verified-script-execution-private-locations)**: Filter by locations with VSE enabled
