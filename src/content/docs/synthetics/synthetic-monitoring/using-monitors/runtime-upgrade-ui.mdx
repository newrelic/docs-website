New Relic is [ending support for older runtimes](/eol/2025/11/eol-chrome-140-node-22-synthetics/) in 2026. To simplify migrating synthetic monitors to the latest supported runtimes, we auto-upgraded every monitor that didn't require your action. You can view the remaining monitors that still need your attention in the runtime upgrade tool.

From the runtime upgrade tool, you can upgrade your scripted browser and scripted API monitors deployed in public locations without leaving New Relic.

## Prerequisites [#prerequisites]

Before you start upgrading monitors, ensure you meet the following requirements:

* **Bulk runtime upgrade permission:** Required to apply bulk updates through the UI. [Product admins](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) have it by default. It can also be added to [custom roles](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).
* **Entitlements for AI-assisted script generation:** Requires the `nrai_ccu` or `nrai_discount_usage` entitlement. FedRAMP accounts are blocked from this experience. Contact your New Relic account team to enable entitlements.

## Upgrade your remaining monitors [#upgrade-options]

For each monitor that requires attention, you can choose one of two paths:

<CollapserGroup>
  <Collapser
    id="fix-manually"
    title="Fix manually"
  >
    Use the <DNT>**Fix manually**</DNT> option to write and edit your monitor scripts rather than relying on AI generation. To update a monitor manually:

    1. Go to <DNT>**[one.newrelic.com](https://one.newrelic.com)** > **Synthetic Monitoring** > **Runtime Upgrades**</DNT>.
    2. Locate the monitor and click <DNT>**Fix manually**</DNT>.
    3. In the <DNT>**Fix script manually**</DNT> window, edit your script code to ensure it's compatible with the new runtime environment.
    4. Click <DNT>**Close and start validation**</DNT>. The monitor moves to the <DNT>**Monitors we're validating**</DNT> table while the system tests the updated script to ensure it executes without errors.
    5. Once validation is successful and the status shows as <DNT>**Fixed**</DNT>, click the <DNT>**Ready to update**</DNT> tab, select the checkbox next to the validated monitor, and click <DNT>**Upgrade selected**</DNT>.

    Once finished, a confirmation notification appears and the monitor moves to the <DNT>**Updated**</DNT> tab where you can verify the successful upgrade.

  </Collapser>

  <Collapser
    id="fix-with-ai"
    title="Fix with AI"
  >
    Use the <DNT>**Fix with AI**</DNT> option to let <DNT>New Relic AI (NRAI)</DNT> generate an updated script for you. The AI-assisted script generation is mainly geared toward fixing `timeout` errors incurred during runtime upgrades. For other error types, it's likely the script itself needs an update, and you should fix it [manually](#fix-manually).

    To update a monitor using AI:

    1. Go to <DNT>**[one.newrelic.com](https://one.newrelic.com)** > **Synthetic Monitoring** > **Runtime Upgrades**</DNT>.
    2. Locate the monitor and click <DNT>**Fix with AI**</DNT>.
    3. Review the AI-suggested script. To view the specific changes made to your original code, click <DNT>**Show comparison**</DNT> to open a side-by-side diff view.
    4. Click <DNT>**Continue to validation**</DNT>. The system temporarily places the monitor in a validation state to test the new script and ensure it executes without errors.
    5. Once the script is validated, click the <DNT>**Ready to update**</DNT> tab, select the checkbox next to the validated monitor, and click <DNT>**Upgrade selected**</DNT>.

    Once finished, a confirmation notification appears and the monitor moves to the <DNT>**Updated**</DNT> tab where you can verify the successful upgrade.

  </Collapser>
</CollapserGroup>

You can run multiple upgrades at once, which allows you to work on several monitors in parallel without waiting for the system to finish validating each script.
