Currently New Relic supports two versions of the Synthetics API: v1 and v3. Version 3 was released October 2016. **Version 1 is deprecated** and will eventually no longer be available. No termination date has been announced. However, no further development or modifications will be made to v1.

<Callout variant="caution">
  Recommendation: Create new monitors using the [v3 Synthetics API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api) and migrate v1 scripts to their v3 equivalent.
</Callout>

To use the REST API for New Relic's synthetic monitoring, you must [activate API access and generate your Admin User's API key](/docs/apis/rest-api-v2/requirements/api-keys#creating) from your account settings. Then you can make standard API calls via the command line.

## Managing simple and scripted monitors [#simple-monitor-calls]

You must use your Admin User's API key to make Synthetics REST API calls. The account's REST API key will not work.

<Callout variant="caution">
  The Synthetics REST API limits an account's rate of requests to three requests per second. Requests made in excess of this threshold will return a 429 response code.
</Callout>

These examples show curl commands:

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="get-all-monitors"
    title="Get all monitors"
  >
    To view a list of all monitors in New Relic for your account, send a GET request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors`. For example:

    ```
    curl -v \
         -H 'X-Api-Key:<var>{Admin_User_Key}</var>' https://synthetics.newrelic.com/synthetics/api/v1/monitors
    ```

    A successful request will return a **200 OK** response. The [data returned](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api#api-attributes) will be a JSON object in the following format:

    ```
    {
      "count": integer,
      "monitors": [
        {
          "id": <var>UUID</var>,
          "name": <var>string</var>,
          "type": <var>string</var>,
          "frequency": <var>integer</var>,
          "uri": <var>string</var>,
          "locations": <var>array of strings</var>,
          "status": <var>string</var>,
          "slaThreshold": <var>double</var>,
          "userId": <var>integer</var>,
          "apiVersion": <var>string</var>
        }
      ]
    }
    ```
  </Collapser>

  <Collapser
    className="freq-link"
    id="get-specific-monitor"
    title="Get a specific monitor"
  >
    To view a single existing monitor in New Relic, send a GET request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}`. Replace the <var>&#x7B;id}</var> in the following example with the specific monitor ID.

    ```
    curl -v \
         -H 'X-Api-Key:<var>{Admin_User_Key}</var>' https://synthetics.newrelic.com/synthetics/api/v1/monitors/<var>{id}</var>
    ```

    A successful request will return a **200 OK** response. The [data returned](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api#api-attributes) will be a JSON object in the following format:

    ```
    {
      "id": <var>UUID</var>,
      "name": <var>string</var>,
      "type": <var>string</var>,
      "frequency": <var>integer</var>,
      "uri": <var>string</var>,
      "locations": <var>array of strings</var>,
      "status": <var>string</var>,
      "slaThreshold": <var>double</var>,
      "userId": <var>integer</var>,
      "apiVersion": <var>string</var>
    }
    ```

    An invalid monitor ID will return the error **404 Not Found**: The specified monitor doesn't exist.
  </Collapser>

  <Collapser
    className="freq-link"
    id="add-simple-ping-monitor"
    title="Create a monitor"
  >
    To add a new monitor to your account in New Relic, send a POST request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors` with a JSON payload that describes the monitor:

    ```
    {
      "name": <var>string</var> [required],
      "type": <var>string</var> (SIMPLE, BROWSER, SCRIPT_API, SCRIPT_BROWSER) [required],
      "frequency": <var>integer</var> (minutes) [required, must be one of 1, 5, 10, 15, 30, 60, 360, 720, or 1440],
      "uri": <var>string</var> [required for SIMPLE and BROWSER type],
      "locations": <var>array of strings</var> (send a GET request to https://synthetics.newrelic.com/synthetics/api/v1/locations to get a list of valid locations) [at least one required],
      "status": <var>string</var> (ENABLED, MUTED, DISABLED) [required],  
      "slaThreshold": <var>double</var>, 
    }
    ```

    In addition, to **add a scripted monitor** via the REST API, call an [additional API endpoint](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitor-scripts-rest-api) to send the script for the monitor just created.

    Replace the [Synthetics REST API attributes](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api) in the following example with your specific values.

    ```
    curl -v \
         -X POST -H 'X-Api-Key:<var>{Admin_User_Key}</var>' \
         -H 'Content-Type: application/json' https://synthetics.newrelic.com/synthetics/api/v1/monitors \
         -d '{  "name" : "<var>monitor1</var>",  "frequency" : <var>15</var>, "uri" : "<var>http://my-uri.com</var>",   "locations" : [ "<var>AWS_US_WEST_1</var>" ],   "type" : "<var>browser</var>"}'
    ```

    A successful request will return a **201 Created** response, with the URI of the newly-created monitor specified in the `location` header. Possible error codes include:

    * **400 Bad Request**: One or more of the monitor values is invalid, or the format of the request is invalid. For example, the frequency is out of bounds or one or more of the specified locations is invalid (See the error message in the body of the response.)
    * **402 Payment Required**: Creating the monitor will increase your scheduled checks past your account's purchased check limit.
  </Collapser>

  <Collapser
    className="freq-link"
    id="update-monitor"
    title="Update an existing monitor"
  >
    To update an existing monitor in New Relic, send a PUT request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}`. In addition, for scripted monitors, follow the procedures to [update the BASE64 encoded script](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitor-scripts-rest-api).

    Replace the <var>&#x7B;id}</var> in the following example with the specific monitor ID, and replace the [Synthetics REST API attributes](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api) with your specific values.

    ```
    curl -v \
         -X PUT -H 'X-Api-Key:<var>{Admin_User_Key}</var>' \
         -H 'Content-Type: application/json' https://synthetics.newrelic.com/synthetics/api/v1/monitors/<var>{id}</var> \
         -d '{  "name" : "<var>updated monitor name</var>", "type" : "<var>simple</var>", "frequency" : <var>15</var>, "uri" : "<var>http://my-uri.com/</var>",   "locations" : [ "<var>AWS_US_WEST_1</var>" ],   "status" : "<var>enabled</var>", "slaThreshold": "7.0" }'
    ```

    PUT requests are intended to replace target entities, so all attributes required in the JSON payload when [creating a new monitor](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#add-simple-ping-monitor) are also required when updating an existing monitor.

    A successful request will return a **204 No Content** response, with an empty body. Possible error codes include:

    * **400 Bad Request**: One or more of the monitor values is invalid, or the format of the request is invalid. For example, the frequency is out of bounds or one or more of the specified locations is invalid (See the error message in the body of the response.)
    * **404 Not Found**: The specified monitor does not exist.
  </Collapser>

  <Collapser
    className="freq-link"
    id="delete-monitor"
    title="Delete an existing monitor"
  >
    To delete an existing monitor in New Relic, send a DELETE request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}`. Replace the [<var>&#x7B;id}</var>](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api) in the following example with the specific monitor ID.

    ```
    curl -v \
         -H 'X-Api-Key:<var>{Admin_User_Key}</var>' \
         -X DELETE https://synthetics.newrelic.com/synthetics/api/v1/monitors/<var>{id}</var>
    ```

    A successful request will return a **204 No Content** response, with an empty body. An unsuccessful request will return the response, **404 Not Found**: The specified monitor does not exist.
  </Collapser>

  <Collapser
    className="freq-link"
    id="list-locations"
    title="Get list of valid locations"
  >
    To retrieve the list of valid locations in New Relic, use the following command.

    ```
    curl -v \
         -X GET -H 'X-Api-Key:<var>{Admin_User_Key}</var>' https://synthetics.newrelic.com/synthetics/api/v1/locations
    ```
  </Collapser>
</CollapserGroup>

## Managing scripted monitors [#scripted-monitor-calls]

In addition to the general API, there are several API methods for the **scripted browser** (`SCRIPT_BROWSER`) and **api test** (`SCRIPT_API`) monitor types.

These examples show curl commands.

<CollapserGroup>
  <Collapser
    id="get-all-monitors"
    title="Get monitor script"
  >
    To view the script associated with a specific `SCRIPT_BROWSER` or `SCRIPT_API` monitor in New Relic for your account, send a GET request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}/script`. Replace the <var>&#x7B;id}</var> with the specific monitor ID. For example:

    ```
    curl -v
         -H 'X-Api-Key: <var>{Admin_User_Key}</var>' 
         https://synthetics.newrelic.com/synthetics/api/v1/monitors/<var>{id}</var>/script
    ```

    A successful request will return a **200 OK** response. The data returned will be a JSON object in the following format:

    ```
    {
      "scriptText": <var>BASE64 encoded string</var>
    }
    ```

    Possible error codes include:

    * **403 Forbidden:** The specified monitor is not of type SCRIPT_BROWSER or SCRIPT_API.
    * **404 Not Found:** The specified monitor doesn't exist or the script associated with the monitor doesn't exist.
  </Collapser>

  <Collapser
    id="procedure"
    title="Add scripted monitor"
  >
    To add a new scripted monitor to New Relic with the REST API:

    1. Follow [standard API procedures to add a new monitor](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#add-simple-ping-monitor), and identify the `type` as a `SCRIPT_BROWSER` or `SCRIPT_API`.
    2. Update the new monitor with a BASE64 encoded version of the script to the `${MONITOR_UUID}/script` endpoint.

    For more information, refer to the [example](#sample-script).
  </Collapser>

  <Collapser
    id="update-monitor-script"
    title="Update monitor script"
  >
    To update the script associated with a specific `SCRIPT_BROWSER` or `SCRIPT_API` monitor in New Relic for your account, send a PUT request to `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}/script` with a JSON payload that contains the `scriptText` (required). The `scriptLocations` data is required only for private locations with [**Verified Script Execution**](/docs/synthetics/new-relic-synthetics/private-locations/verified-script-execution-private-locations) turned on.

    The password used to generate the HMAC string must match the password set for the private location. When generating the HMAC string, use the SHA256 algorithm.

    ```
    {
      "scriptText": <var>BASE64 encoded String</var>,
      "scriptLocations": [
        {
          "name": <var>Location name</var>,
          "hmac" <var>BASE64 encoded String of SHA256 HMAC for location</var>
        }
      ]
    }
    ```

    Replace the <var>&#x7B;id}</var> with the specific monitor ID. Here is an example for the script:

    ```
    var assert = require('assert');
    assert.equal('1', '1');
    ```

    This example uses `password` as the password for the `scriptLocation`.

    ```
    curl -v
    	-H 'X-Api-Key: '<var>{Admin_User_Key}</var>'
    	-H 'content-type: application/json'
    	https://synthetics.newrelic.com/synthetics/api/v1/monitors/<var>{id}</var>/script
    	-d '{ "scriptText": "dmFyIGFzc2VydCA9IHJlcXVpcmUoJ2Fzc2VydCcpOw0KYXNzZXJ0LmVxdWFsKCcxJywgJzEnKTs=","scriptLocations": [ { "name": "my_vse_enabled_location", "hmac": "MjhiNGE4MjVlMDE1N2M4NDQ4MjNjNDFkZDEyYTRjMmUzZDE3NGJlNjU0MWFmOTJlMzNiODExOGU2ZjhkZTY4"} ]}'
    ```

    A successful request will return a **204 No Content** response with an empty body. Possible error codes include:

    * **400 Bad Request:** Invalid BASE64 encoded string for `scriptText` or `hmac`.
    * **403 Forbidden:** The specified monitor is not of the type SCRIPT_BROWSER or SCRIPT_API.
    * **404 Not Found:** The specified monitor does not exist.
  </Collapser>
</CollapserGroup>

## Scripted browser example

Here is an example of using New Relic's REST API and the bash script to create a scripted browser monitor.

<CollapserGroup>
  <Collapser
    id="sample-script"
    title="Scripted browser API example"
  >
    The following example shows curl commands to create a scripted browser monitor.

    * At the top of the script, replace the variables with your specific values.
    * For the `scriptfile` variable, identify the filename for the script to be created. Here is a sample script that can be saved as `sample_synth_script.js` to use in the example:

    ```
    var assert = require("assert");
    $browser.get(<var>"http://example.com"</var>).then(function(){ 
      // Check the H1 title matches "Example Domain" 
      return $browser.findElement($driver.By.css("h1")).then(function(element){ 
        return element.getText().then(function(text){ 
          assert.equal("Example Domain", text, "Page H1 title did not match"); 
        }); 
      }); 
    }).then(function(){ 
      // Check that the external link matches "http://www.iana.org/domains/example" 
      return $browser.findElement($driver.By.css("div > p > a")).then(function(element){ 
        return element.getAttribute("href").then(function(link){ 
          assert.equal("http://www.iana.org/domains/example", link, "More information link did not match"); 
        }); 
      }); 
    });
    ```
  </Collapser>

  <Collapser
    id="bash-script"
    title="Bash script example"
  >
    This example show the bash script that will create the `SCRIPTED_BROWSER` monitor.

    ```
    #!/bin/bash

    # Admin API key from your account settings
    adminAPIKey=''
    # Other attributes found at https://docs.newrelic.com/docs/apis/synthetics-rest-api/monitor-examples/attributes-synthetics-rest-api#api-attributes
    monitorName='<var>Test API Script</var>'
    monitorType='<var>SCRIPT_BROWSER</var>'
    frequency=<var>1440</var>
    locations='<var>"AWS_US_WEST_1"</var>, <var>"AWS_US_EAST_1"</var>'
    slaThreshold=<var>7.0</var>
    # Location of the file with your script
    scriptfile=<var>sample_synth_script.js</var>

    # Test that the script file exists (does not validate content)
    if [ -e "<var>$scriptfile</var>" ]
    then
      script=$(cat "<var>$scriptfile</var>")

      payload="{  \"name\" : \"<var>$monitorName\</var>", \"<var>frequency\</var>" : <var>$frequency</var>,    \"locations\" : [ <var>$locations</var> ],   \"status\" : \"ENABLED\",  \"type\" : \"<var>$monitorType\</var>", \"slaThreshold\" : <var>$slaThreshold,</var> \"uri\":\"\"}"
      echo "Creating monitor"

      # Make cURL call to API and parse response headers to get monitor UUID
      shopt -s extglob # Required to trim whitespace; see below
      while IFS=':' read key value; do
        # trim whitespace in "value"
        value=${value##+([[:space:]])}; value=${value%%+([[:space:]])}
        case "$key" in
            location) LOCATION="<var>$value</var>"
                    ;;
            HTTP*) read PROTO STATUS MSG <<< "$key{$value:+:$value}"
                    ;;
        esac
      done < <(curl -sS -i  -X POST -H "X-Api-Key:<var>$adminAPIKey</var>" -H 'Content-Type: application/json' https://synthetics.newrelic.com/synthetics/api/v1/monitors -d "$payload")

      # Validate monitor creation & add script unless it failed
      if [ $STATUS = 201 ]; then
        echo "Monitor created, $LOCATION "
        echo "Uploading script"
          # base64 encode script
          encoded=`echo "$script" | base64`
          scriptPayload='{"scriptText":"'$encoded'"}'
            curl -s -X PUT -H "X-Api-Key:<var>$adminAPIKey</var>" -H 'Content-Type: application/json' "$LOCATION/script" -d $scriptPayload
            echo "Script uploaded"
      else
        echo "Monitor creation failed"
      fi

    else
      echo "script file not found, not creating monitor"
    fi
    ```
  </Collapser>
</CollapserGroup>

## Specific monitor endpoint [#specific-monitor]

When making REST API calls for a specific monitor, include the `monitor_uuid` as part of the endpoint. The `monitor_uuid` is the GUID which is part of the URL. For example, a selected [Synthetics monitor](/docs/synthetics/new-relic-synthetics/pages/synthetics-overview-page-view-monitors-performance) has this URL:

```
https://synthetics.newrelic.com/accounts/<var>nnnn</var>/monitors/<var>ab123-c456d-e78-90123-f45g</var>
```

The `monitor_uuid` is the value that follows `/monitors/`.
