The Compute Optimizer tool provides actionable insights to help you manage compute spend and implement automated compute controls. This means you can identify inefficient areas, receive clear recommendations for optimization, and ultimately achieve potential compute savings, improved query efficiency, and a more streamlined New Relic experience.

<Callout variant="tip">

  Compute Optimizer is available to customers on Legacy Compute (CCU) or Core Compute plans only. It is not available to customers on Data + User + Advanced Compute plans.

</Callout>

## Access Compute Optimizer
You'll find the Compute Optimizer in the administration panel within New Relic.

1. Log in to New Relic.
2. Open your personal settings by clicking your user name in the bottom-left corner of the New Relic interface. Go to the administration panel.
3. Click **Compute Optimizer** to access the dashboard.

## The Compute Optimizer dashboard
The Compute Optimizer dashboard includes tabs for **Alert Conditions** and **Dashboards**, allowing you to focus on specific product capabilities. Both tabs share common functionality:

- **Time picker:** Filter data based on specific time periods, with options like "Since 1 day ago (UTC)"
- **Refresh recommendations:** Update the recommendations with the latest data
- **Export report:** Download the recommendations for offline analysis

Each tab displays metrics related to compute usage and potential savings, though the specific metrics vary by tab (see individual tab sections below).

## Alert Conditions tab

The Alert Conditions tab helps you identify and optimize inefficient alert conditions that are consuming excessive compute resources.

### Dashboard metrics
- **Inefficient usage:** Shows the total core CCU consumed by alert conditions that have been flagged as inefficient, with a percentage of total alerts CCU
- **Potential savings:** Shows the estimated core CCU savings you can achieve, with a percentage of total alerts CCU (excludes recommendations without savings estimates)

### Search and filter
Use the search bar to find specific alert conditions by name. You can also filter recommendations by:
- **Component name:** Filter by all available alert conditions or specific ones
- **Product capability:** Focus on Alert Conditions specifically

### Recommendations table
The recommendations table breaks down the identified optimization opportunities for alert conditions:

- **Component name:** Identifies the specific alert condition that has an optimization opportunity
- **Product capability:**  Displays "Alert Conditions" for all entries in this tab
- **CCU usage:** Shows the current compute consumption (measured in core CCU) for each alert condition
- **Recommendation:** Offers actionable insights for optimizing your compute. Common recommendations include:
  - "Add a `WHERE` clause outside the `SELECT` statement"
  - "Consider deleting this alert condition. Merge thresholds and/or keep the condition with smaller LOS"
  - "This alert condition has duplicates. Consider deleting the duplicated alert conditions"
- **CCU saving estimate:** Quantifies the potential compute reduction (in core CCU) for each individual recommendation
- **Reduction estimate:** Shows the percentage of compute that could be saved (for example, "100%", "0%")

### Taking action
The action menu (three dots) for each recommendation offers additional options, including access to optimized queries wherever applicable, making it easy to implement the suggested changes directly.

## Dashboards tab

The Dashboards tab helps you identify and optimize dashboard widgets and pages that are consuming excessive compute resources or need performance improvements.

### Dashboard metrics
- **Total Usage:** Shows the total core CCU consumed by all dashboards, with a percentage of total dashboard CCU
- **Potential savings:** Shows the estimated core CCU savings you can achieve, with a percentage of total dashboard CCU (excludes recommendations without savings estimates)

### Search and filter
Use the search bar to find specific dashboards, widgets, or pages by name. You can also filter recommendations by:
- **Dashboard:** Filter by all dashboards or specific ones
- **Item:** Filter by all items (widgets and pages) or specific ones
- **Recommendation:** Filter by all recommendations or specific recommendation types

### Grouped view
The Dashboards tab organizes recommendations by dashboard, showing the total number of groups and items (for example, "3 groups | 26 items"). Each dashboard can be expanded to view individual recommendations for:
- **Dashboard widgets:** Individual widgets within dashboards that need optimization
- **Dashboard pages:** Pages within dashboards that need optimization

### Recommendations table
The recommendations table shows optimization opportunities organized by item:

- **Item:** Identifies the specific dashboard widget or page that has an optimization opportunity. Items are grouped under their parent dashboard name.
- **Recommendation:** Offers actionable insights for optimizing your dashboards. Common recommendations include:
  - "Increase this widget's refresh rate to 5+ minutes for optimal performance. It's currently refreshing too often"
  - "Split this page into multiple pages or remove unused widgets. It's exceeding the recommended limit of 9 widgets per page"
  - Additional recommendations may be available (indicated by "+1 more" or similar notation)
- **Usage:** Shows the current compute consumption (measured in core CCU) for each item, with sorting capability
- **Estimated savings:** Quantifies the potential core CCU savings and percentage reduction for each recommendation

### Taking action
The action menu (three dots) for each recommendation offers additional options for implementing the suggested optimizations.

All insights presented in the Compute Optimizer are dynamic and based on your time picker settings, allowing you to analyze data relevant to specific periods.
