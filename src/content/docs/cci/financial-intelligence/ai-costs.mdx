The <DNT>**AI Costs**</DNT> page provides cost visibility for AI services across your cloud providers. It surfaces token-level costs broken down by service, model, and token type, giving you a unified view of AI spending across multi-cloud environments and multiple LLM models.

## Access AI Costs

1. Go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Cloud Cost Intelligence**</DNT>.
2. Click <DNT>**AI Costs**</DNT> in the left navigation.

## Cloud provider filters

To filter AI cost data by cloud provider, select an option from the provider tabs at the top of the page:

- <DNT>**All**</DNT>: All cloud providers
- <DNT>**AWS**</DNT>: Amazon Web Services only
- <DNT>**Azure**</DNT>: Microsoft Azure only
- <DNT>**GCP**</DNT>: Google Cloud Platform only

## AI top spends

The <DNT>**AI Top Spends**</DNT> section surfaces your highest AI cost areas across four dimensions. Each widget shows the total for the selected period, the comparison value from the previous period, the percentage change, and a breakdown of the top three items.

### Services

Displays total AI costs grouped by cloud AI service (for example, AmazonBedrock, Azure AI Services, AmazonSageMaker). Use this to identify which services are driving the most spend across your cloud providers.

### Models

Displays total AI costs grouped by LLM model (for example, Claude Opus 4.6, Claude Sonnet 4.6). Use this to compare cost across model versions and providers.

### Token Cost

Displays total AI spend broken down by token type:

- <DNT>**Cache Write Token**</DNT>: Cost for writing tokens to cache
- <DNT>**Input Token**</DNT>: Cost for input tokens processed by the model
- <DNT>**Cache Read Token**</DNT>: Cost for reading tokens from cache

Use this to understand which token operations contribute most to your AI costs.

### Token Usage

Displays total token consumption (in billions) broken down by token type. Use this alongside Token Cost to identify high-volume, low-cost patterns versus low-volume, high-cost patterns that may indicate optimization opportunities.

<Callout variant="tip">
Compare the <DNT>**Token Cost**</DNT> and <DNT>**Token Usage**</DNT> widgets together to understand the cost-per-token efficiency across different token types and models.
</Callout>

## Cost Trends

The <DNT>**Cost Trends**</DNT> section provides time-based analysis of your AI costs with interactive charts and controls.

### Chart controls

- <DNT>**View options**</DNT>: Switch between <DNT>**Hourly**</DNT>, <DNT>**Daily**</DNT>, <DNT>**Weekly**</DNT>, or <DNT>**Monthly**</DNT> views
- <DNT>**Group by**</DNT>: Select dimensions to group cost data (up to 3 selections)
- <DNT>**View by**</DNT>: Choose a visualization format (availability depends on your group-by selections)

### Understanding the data

- <DNT>**Total cost**</DNT>: Aggregate AI spending for the selected period with percentage change versus the previous period
- <DNT>**Average cost (Daily)**</DNT>: Daily average AI spending with percentage change versus the previous period
- <DNT>**Stacked bar chart**</DNT>: Visualizes AI cost distribution over time, color-coded by cloud provider

### Visualization options

**With a single group-by selection:**
- <DNT>**Bar chart**</DNT>: Displays AI cost data as vertical bars
- <DNT>**Line chart**</DNT>: Shows AI cost trends over time
- <DNT>**Tree chart**</DNT>: Hierarchical visualization of AI cost data
- <DNT>**Data table**</DNT>: Tabular view of AI cost data

**With multiple group-by selections:**
- <DNT>**Heat map**</DNT>: Color-coded visualization across dimensions
- <DNT>**Data table**</DNT>: Tabular view of AI cost data

## Filters and customization

### Date range selector

Use the date range selector at the top right to:
- Choose a predefined time period
- Set custom start and end dates
- View costs in your preferred timezone (UTC displayed by default)

### Engineering costs filter

If your organization uses cost allocation tags, filter by the <DNT>**Engineering costs**</DNT> tag to narrow AI cost analysis to specific teams, projects, or cost centers tagged in your cloud environment.
