Query your Cloud Cost Intelligence (CCI) data with the New Relic Query Language (NRQL) to analyze and optimize your cloud spending. To run any of the queries on this page, go to <DNT>**All Capabilities > Cloud Cost Intelligence**</DNT> and open the <DNT>**Query your data**</DNT>.

## CloudCost [#cloudcost]

Example query:

```sql
FROM CloudCost SELECT * SINCE 24 hours ago
```

## KubernetesCost [#kubernetescost]

Example query:

```sql
FROM KubernetesCost SELECT * SINCE 1 week ago
```

For accurate Kubernetes costs, multiply `cpu_weightage + memory_weightage` in your queries:

```sql
SELECT SUM(net_unblended_cost * (cpu_weightage + memory_weightage) + network_net_unblended_cost) FROM KubernetesCost SINCE '2024-09-01' UNTIL '2024-09-10'
```

This multiplies `net_unblended_cost` by the sum of CPU and memory weightage for accurate cost allocations based on resource usage.

## Example queries [#example-queries]

Use these queries as starting points for common cost analysis tasks. Adjust the attributes, time ranges, and filters to match your environment.

Find your total cloud spend broken down by service over the last month:

```sql
FROM CloudCost SELECT SUM(Line_item_net_unblended_cost) FACET Line_Item_Product_Code SINCE 1 month ago
```

See how daily cloud spend is trending over time:

```sql
FROM CloudCost SELECT SUM(Line_item_net_unblended_cost) TIMESERIES 1 day SINCE 30 days ago
```

Identify the most expensive Kubernetes namespaces:

```sql
FROM KubernetesCost SELECT SUM(Total_cost) FACET Namespace_name SINCE 1 week ago LIMIT 10
```

Compare CPU cost to memory cost per cluster:

```sql
FROM KubernetesCost SELECT SUM(Cpu_costs), SUM(Memory_costs) FACET Cluster_name SINCE 1 week ago
```

Find under-utilized containers where CPU usage is low:

```sql
FROM KubernetesCost SELECT average(Cpu_usage) FACET Container_name WHERE Cpu_usage < 0.2 SINCE 3 day ago
```

Review estimated costs by owning team:

```sql
FROM CloudCostEstimate SELECT SUM(Cost) FACET Owning_team SINCE 24 hours ago
```
