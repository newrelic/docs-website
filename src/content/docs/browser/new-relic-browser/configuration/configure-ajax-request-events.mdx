In [agent version 1211](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1211/) or higher, all network requests made by a page are recorded as `AjaxRequest` events. You can use the deny list configuration options in the <DNT>Application settings</DNT> page to filter which requests record events.

## Use the deny list [#deny-list]

By default, we record `AjaxRequest` events for all network requests made by your application.

* To stop recording all events, add a singular `*` (asterisk) wildcard character.
  * Example: `*`
* To stop recording events for a full site, add the domain name.
  * Example: `mywebsite.com`
* To stop recording events for a specific page, add the domain and path.
  * Example: `mywebsite.com/photos`
* Starting with [agent version 1.307.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.307.0/), anywhere we find an `*` (asterisk), we’ll match any length of string.
  * Example: `mywebsite.com/shop/*` (for a path), `*.mywebsite.com` (for a subdomain), `mywebsite.com/*/shop/*/checkout` (for mid-URL strings).

The protocol, port, search and hash of a URL are ignored by the deny list.

## Access the deny list [#access-deny-list]

To update the deny list of URLs your application will filter from creating events, go to the app settings page:

1. Go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Application settings**</DNT>.
2. Under <DNT>**Ajax Request Deny List**</DNT>, add the filters you would like to apply to your app.
3. Select <DNT>**Save application settings**</DNT> to update the agent configuration.
4. [Redeploy the browser agent](/docs/browser/new-relic-browser/installation/upgrade-browser-agent) (either restarting the associated <InlinePopover type="apm"/> agent or updating the copy/paste browser installation).

## Validate deny list filters [#validation]

To validate whether the filters you have added work as expected, run a NRQL query for `AjaxRequest` events matching your filter.

```sql
FROM AjaxRequest SELECT * WHERE requestUrl LIKE `%example.com%`
```

## Enable payload capture [#enable-payload-capture]

<Callout variant="important">
  Payload capture requires browser agent version **TBD `{{INFORMATION_NEEDED}}`** or higher.
</Callout>

Payload capture lets you record the full request and response headers and bodies for AJAX calls. This is useful for debugging failures that aren't visible from metadata alone — such as errors hidden in response bodies.

To enable payload capture:

1. Go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Application settings**</DNT>.
2. In the <DNT>**AJAX Request Payload Capture**</DNT> section, select the capture mode from the following options:
   * `failures`: Captures payloads only for requests with error status codes (4xx, 5xx).
   * `all`: Captures payloads for all requests, including successful ones. Use this to debug failures hidden in response bodies, such as GraphQL errors returned with a successful status.
3. Select <DNT>**Save application settings**</DNT>.

<Callout variant="caution">
  Enabling payload capture for `all` requests can significantly increase your data ingest. Use `failures` mode unless you specifically need to capture successful responses.
</Callout>

You can also configure payload capture using our [NerdGraph mutation](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-payload-capture).

### How payload capture interacts with the deny list [#payload-deny-list]

Payload capture respects your existing deny list. For any AJAX request:

* If the URL is on the deny list, the event is dropped entirely — no metadata or payload is captured.
* If the URL is not on the deny list, **capture mode** determines whether payloads are captured alongside the standard metadata.
