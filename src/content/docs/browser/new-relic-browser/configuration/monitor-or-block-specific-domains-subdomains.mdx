[Browser](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) lets you choose which domains or subdomains that a browser app monitors. This is useful for limiting monitoring only to the application activity you consider important. For example, if you use the [copy/paste method](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) to deploy your browser app, you can set domain conditions to protect the security of your [browser monitoring license key and app ID](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id).

<Callout variant="tip">
  Domain conditions only filter data based on the url of the reported page. Domain condition filters don't apply to AJAX data reported to New Relic. To filter out `AjaxRequest` events, read [our configuration docs](/docs/browser/new-relic-browser/configuration/filter-ajax-request-events/).
</Callout>

## Configure domain conditions [#configure]

To view or edit your domain condition settings:

1. Go to **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Application settings > Domain conditions**.
2. If there are no domain conditions in place, select **Enable domain conditions**. If conditions exist, select **Edit conditions**.
3. Select the option to build a list of allowed domains or to build a list of blocked/denied domains.
4. Select the domain conditions (maximum 10 conditions).
5. Select the domain conditions with the **Starts with** condition, then enter the URL with the subdomain (maximum 10 conditions).

   </figcaption>
6. Review and confirm your domain condition settings.

A preview text box displays in real time what domains match the conditions you set. You will also see an estimate of the impact the proposed settings will have on your data.

## Turn off domain conditions [#disable]

To disable domain conditions and return to monitoring all domains:

1. Go to **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Domain conditions**.
2. Set the **Enable domain conditions** option to **Off**.
