Manage session replay and modification capabilities using the user management access functionality. This feature enables customization of user roles and access permissions to define who can modify session replay settings.

## Revoke SR modify access for members/accounts from Admin Group [#modify-all-admin-role]

<Steps>

  <Step>
    Go to <DNT>**Administration > Access Management > Groups**</DNT>.
  </Step>

  <Step>
    Click the ellipsis **(...)** icon and select <DNT>**Manage Group Access**</DNT>.
  </Step>

  <Step>
    Expand the <DNT>**Members**</DNT> section, select the required user.
  </Step>

  <Step>
    Click <DNT>**Remove users from group**</DNT>.
  </Step>

  <Step>
    Expand the <DNT>**Account access**</DNT> section, click the ellipsis **(...)** icon.
  </Step> 

  <Step>
     Select <DNT>**Delete role**</DNT>.
  </Step>

</Steps>

## Create a custom role for session replay read access [#create-custom-role]

<Steps>

  <Step>
    Go to <DNT>**Administration > Access Management > Roles**</DNT>.
  </Step>

  <Step>
    Click <DNT>**Add New Custom Roles**</DNT> and enter a custom role name.
  </Step>

  <Step>
    In the <DNT>**Browser**</DNT> section, you may:
    
    - Select the <DNT>**View**</DNT> checkbox and ensure the <DNT>**Modify**</DNT> checkbox is not selected to grant read-only access to <DNT>**Session replays**</DNT>.
    - Select both <DNT>**View**</DNT> and <DNT>**Modify**</DNT> checkboxes to grant modify access to <DNT>**Session replays**</DNT>,.

   <img
     title="Browser Role ID"
     alt="A screenshot of Browser Role ID"
     src="/images/browser-role-id.webp"
   />
  </Step>

  <Step>
    Click <DNT>**Save**</DNT>.
  </Step>

  <Step>  
    Select the <DNT>**Groups**</DNT> tab and click <DNT>**Create new group**</DNT>.
  </Step>

  <Step>
    Enter a <DNT>**Group name**</DNT> and click <DNT>**Create group**</DNT>.
  </Step>

  <Step>
    Expand the <DNT>**Members**</DNT> section, select the user(s) from the **Select a user** drop-down list, and click **Add user to group**.
  </Step>

  <Step>
    Expand the <DNT>**Account acess**</DNT> section, and in the <DNT>**Add account roles to this group**</DNT> field, select the following:
    
    * <DNT>**Select a role**</DNT>
    * <DNT>**Select an account**</DNT>
  </Step>

  <Step>
    Click <DNT>**Add role**</DNT>.
  </Step>

</Steps>

## Grant session replay modify capability [#grant-sr-modify-capability]

<Steps>

  <Step>
    Go to <DNT>**Administration > Access Management > Roles**</DNT>.
  </Step>

  <Step>
    In the <DNT>**Custom role**</DNT> section, click the ellipsis **(...)** icon and select <DNT>**Manage role**</DNT>.
  </Step>

  <Step>
    Click the <DNT>**Edit**</DNT> icon and go to the <DNT>**Browser**</DNT> section.
  </Step>

  <Step>
    In the <DNT>**Session replays**</DNT> section select the <DNT>**Modify**</DNT> checkbox.
  </Step>

  <Step>
    Click <DNT>**Save**</DNT>.
  </Step>

</Steps>

<Callout variant="important">
  * **Session Replay** settings will only be accessible by [All Product Admins](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) role and the **SR Modify** users. Users under existing Read capability won't be able to view the settings*.
  * It may take 5-10 minutes for the upgraded settings to appear in the UI for the accounts and users.
</Callout>
