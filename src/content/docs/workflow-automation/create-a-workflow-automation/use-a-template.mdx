The Workflow Automation templates provide a quick way to create workflows that automate tasks and processes in New Relic. You can use these templates as starting points and each template in the library is fully customizable to meet your organization's standards.

## Workflow Automation templates

<CollapserGroup>

  <Collapser
    id="send-report-to-slack"
    title="Send report to Slack"
  >

    **Purpose:**

    Create a workflow that runs a NRQL query, converts the results to CSV, and sends the report to a Slacxk channel.

    **Prerequisites:**

    - A configured slack application with a token and a target channel.
    - New Relic credentials.

  </Collapser>

  <Collapser
    id="api-gateway-rollback"
    title="API gateway rollback"
  >
    **Purpose:**

    Detects a recent deployment related to an issue and initiates a rollback of an AWS API gateway configuration after approval.

    **Prerequisites:**

    - AWS credentials with permissions for API Gateway and Systems Manager.
    - A configured Slack app for receiving notifications and approvals.

  </Collapser>

  <Collapser
    id="ec2-instance-management"
    title="EC2 instance management"
  >
    **Purpose:**

    Automate provisioning, scaling, and termination of AWS EC2 instances for optimal performance and cost.

    **Prerequisites:**

    - AWS credentials with permissions for EC2 and Systems Manager.
    - Access to an AWS account with EC2 instances configured.
    - An active New Relic alert condition for EC2 metrics.
    - A configured Slack app for receiving notifications and approvals.
    - Enable the [AWS CloudWatch Metric Streams integration](https://docs.newrelic.com/install/aws-cloudwatch/) to monitor all **CloudWatch** metrics from your AWS services, including custom namespaces. Individual integrations are no longer a recommended option.
    - To enable this, follow standard procedures by following the instructions of [Amazon EC2 monitoring integration](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-ec2-monitoring-integration/#activate).
  </Collapser>
</CollapserGroup>
