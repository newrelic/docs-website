## Request parameters [#requestparameters]

  - Name
    - `String`, alphanumeric
    - The [WorkflowDefinition](/docs/workflow-automation/workflow-automation-apis/definition-schema) name
    - Case sensitive.
  - Version
    - `String`
      - *Optional*, otherwise uses latest version.

## Response elements [#responseparameters]

  If the action is successful, the service sends back::

    - Yaml
      - Type
        - [Workflow Definition Schema](/docs/workflow-automation/workflow-automation-apis/definition-schema)
    - Name
      - `String`, alphanumeric
      - The [WorkflowDefinition](/docs/workflow-automation/workflow-automation-apis/definition-schema) name.
    - Version
      - `Integer`
      - The version for the workflow definition. Values are managed internally in a sequential manner, starting from version 1.

## Example [#example]

```graphql
  {
    actor {
      account(id: 12345678) {
        workflowAutomation {
          workflow(name: "find_entity", version: 1) {
            definition {
              name
              version
              yaml
            }
          }
        }
      }
    }
  }
```
