## Request parameters [#requestparameters]

- AccountID
  - Integer
  - New Relic account that owns the workflow definition to be scheduled.

- Definition
  - ScopeType
    - Enum
    - Optional, defaults to `ACCOUNT`

  - Name
    - String, alphanumeric
    - The [WorkflowDefinition name](/docs/workflow-automation/workflow-automation-apis/definition-schema)
    - Case sensitive

  - Version
    - Integer
    - Optional, otherwise uses latest version.

- CronExpression
  - String, [Cron type format](https://en.wikipedia.org/wiki/Cron), default to UTC timezone.

- Inputs
  - List of objects with the format:
    - key: `string`
    - value: `string`

- Overlap Policy
  - Available values specified at [schema]: `SKIP (default)`, `CANCEL`
  - Optional

## Response elements [#responseparameters]

If the action is successful, the service sends back the following data:

  - ScheduleID
    - `String`

## Example [#example]

```graphql
    mutation {
      workflowAutomationCreateSchedule(
        scope: { type: ACCOUNT id: "12345678" }
        definition: {name: "my-daily-workflow", version: 1}
        cronExpression: "0 0 * * *"
      ) {
        scheduleId
      }
    }
```
