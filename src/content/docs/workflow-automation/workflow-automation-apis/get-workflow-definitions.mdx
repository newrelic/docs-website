## Request parameters [#requestparameters]

  - Cursor
    - `String`, base64 encoded
    - *Optional*

## Response elements [#responseparameters]

  If the action is successful, the service sends back the next cursor if it exists and a list of workflow definitions:

    - NextCursor
      - `String`, base64 encoded
      - Optional
    - Results.definition
      - Name
        - `String`, alphanumeric
        - The [WorkflowDefinition] name.
    - Scope:
      - Id
        - `String`
      - Type
        - Enum
    - Version
      - `Integer`
      - The version for the workflow definition. Values are managed internally in a sequential manner, starting from version 1.

## Example [#example]

```graphql
    {
      actor {
        account(id: 12345678) {
          workflowAutomation {
            workflows(cursor: "") {
              results {
                definition {
                  name
                  scope {
                    id
                    type
                  }
                  version
                }
              }
            }
          }
        }
      }
    }
```
