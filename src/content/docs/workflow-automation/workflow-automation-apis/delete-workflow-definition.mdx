## Request parameters [#requestparameters]

  - Scope
    - id
      - `String`
      - Scope under which the workflow definition should be created.
    - Type
      - `Enum`
      - *Optional*, defaults to `ACCOUNT`.
  - Definition
    - `String`, conforming to [name pattern](/docs/workflow-automation/workflow-automation-apis/definition-schema).
    - The [WorkflowDefinition](/docs/workflow-automation/workflow-automation-apis/definition-schema) name.

## Response elements [#responseparameters]

  If the action is successful, the service will return the following data:

    - Name
      - `String`, alphanumeric
      - The [WorkflowDefinition](/docs/workflow-automation/workflow-automation-apis/definition-schema) name.
    - Scope
      - Id
        - String
      - Type
        - Enum

  <Callout variant="important">
      - In order to delete a definition, there must be no schedules using the definition. If the definition is currently used by a schedule, deleting the definition will fail and return an error accordingly.
      - All running workflow runs will continue to completion even if the definition is deleted.
  </Callout>
