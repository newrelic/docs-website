Set up Workflow Automation programmatically using the [NerdGraph API](/docs/apis/nerdgraph/get-started/nerdgraph-explorer) as an alternative to the UI.

Start by using the [NerdGraph API explorer](https://one.newrelic.com/nerdgraph-graphiql?state=c8579e50-f154-fec6-22b8-f9fdcfc1990a) to create and edit workflow automation information. Definitions for each field are available in NerdGraph. For help with the GraphiQL IDE, see the [NerdGraph API explorer tutorial](/docs.newrelic.com/docs/apis/nerdgraph/get-started/nerdgraph-explorer). For authentication details related to secret management, refer to [Our GraphQL API](/docs.newrelic.com/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).

<Callout title="important">
  The APIs the following APIs are available throgh NerdGraph:
</Callout>

## UpdateWorkflowDefinition [#updateworkflowdefinition]
<CollapserGroup>
    <Collapser
        id="UpdateWorkflowDefinition"
        title="Used to update an existing workflow definition."
    >

  ### Request Parameters

    * Scope
      * Id: 
        * `String`: Scope under which the workflow definition should be created.
      * Type
        * Enum: *Optional*, defaults to `ACCOUNT`.

    * Definition
      * Yaml
        * String
          * Workflow definition YAML or encoded as a base64 string.
          * Type: Workflow Definition Schema
          * The name specified in the workflow definition YAML is case sensitive.

  ### Response Elements

    Once the action is successful, the service sends back:

      * Yaml: `String`
        * The YAML that was sent with the original request.
      * Name: `String`, alphanumeric
        * The WorkflowDefinition name.
      * Version: `Integer`
        * Increments the previous version by 1.

  ### Example

  ```graphql
    mutation {
      autoflowsUpdateWorkflowDefinition(
        definition: {yaml: "<workflow definition YAML>"}
        scope: {id: "11608789", type: ACCOUNT}
      ) {
        definition {
          name
          version
          yaml
        }
      }
    }
  ```

## DeleteWorkflowDefinition [#deleteworkflowdefinition]

<CollapserGroup>
    <Collapser
        id="DeleteWorkflowDefinition"
        title="Used to delete all the workflow definition versions with the specified name."
    >

  ### Request parameters

    * Scope
      * id
        * `String`: Scope under which the workflow definition should be created.
      * Type
        * `Enum`: *Optional*, defaults to `ACCOUNT`.
    * Definition
      * `String`:
        * Conforming to name pattern.
        * The WorkflowDefinition name.

  ### Response parameters

    Once the action is successful, the service will return the following response:

      * Name: `String`, alphanumeric
        * The WorkflowDefinition name.
      * Scope
        * Id
          * String
        * Type
          * Enum

    <Callout variant="tip">
        * In order to delete a definition, there must be no schedules using the definition. If the definition is currently used by a schedule, deleting the definition will fail and return an error accordingly.
        * All running workflow runs will continue to completion even if the definition is deleted.
    </Callout>

    </Collapser>
</CollapserGroup>

## StartWorkflowRun [#startworkflowrun]

<CollapserGroup>
    <Collapser
        id="startworkflowrun"
        title="Starts a workflow run for a specified workflow definition."
    >

  ### Request parameters

    * AccountID
      * Integer
      * NR account that the workflow definition was created under.

    * Definition
      * ScopeType
        * Enum
        * Optional, defaults to `ACCOUNT`

      * Name
        * String, alphanumeric
        * The WorkflowDefinition name
        * Case sensitive

      * Version
        * Integer
        * Optional, otherwise uses latest version.
    
    * Input
      * List of objects with the format:
      * key: string
      * value: string

  ### Response parameters

    Once the action is successful, the service sends back the following data:

      * RunID
        * `String`
        * The workflow instance ID for this execution

  ### Example

  ```graphql
      mutation {
        autoflowsStartWorkflowRun(
          accountId: 11933347
          definition: {name: "entity_tag"}
          workflowInputs: [
            {key: "entityGuid", value: "MTE5MzMzNDd8VklafERBU0hCT0FSRHxkYToxMDQ4NzI"}, 
            {key: "tagKey", value: "demo1"}, {key: "tagValue", value: "abc"}
            ]
        ) {
          runId
        }
      }
  ```
    </Collapser>
</CollapserGroup>

  ### Request parameters

        * String, alphanumeric

        * `String`

  ### Example
