Read on to learn how to uninstall the New Relic PHP agent. For instructions on how to temporarily disable the agent, see [Disable the agent](/docs/agents/manage-apm-agents/installation/disable-apm-agent#php).

This document explains how to uninstall the [New Relic PHP agent](/docs/agents/php-agent/getting-started/new-relic-php) for the following systems:

* RedHat and CentOS
* Ubuntu and Debian
* Tar archive (any [supported system](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems))

## Uninstalling the agent [#install-overview]

When uninstalling New Relic from a PHP application, you must remove the New Relic agent and also remove associated New Relic configuration files.

To uninstall the PHP agent from your system:

1. Remove the agent files from your system:

   <CollapserGroup>
     <Collapser
       id="remove-redhat-centos"
       title="RedHat or CentOS"
     >
       Run the `newrelic-install` script with no options and select the uninstall option from the main menu when invoking the script.
     </Collapser>

     <Collapser
       id="remove-ubuntu-debian"
       title="Ubuntu or Debian"
     >
       Use `apt-get` to remove the agent:

       ```
       apt-get remove newrelic-php5
       ```
     </Collapser>
   </CollapserGroup>
2. Remove any remaining New Relic configuration files. Use the `newrelic-install` or `apt-get remove` commands with the `purge` option.

   ```
   newrelic-install purge
   ```

   OR

   ```
   apt-get remove --purge newrelic
   ```
3. If you use a package manager, remove files specific to the manager:

   <CollapserGroup>
     <Collapser
       id="uninstall-redhat-centos"
       title="RedHat or CentOS"
     >
       Using `yum`:

       ```
       sudo yum remove newrelic-php5
       ```

       Using **`rpm` (32-bit)**:

       ```
       rpm -e newrelic-php5-common-<var>X.X.X.X (YOUR VERSION #)</var>-1.noarch.rpm newrelic-daemon-<var>X.X.X.X (YOUR VERSION NUMBER)</var>-1.i386.rpm newrelic-php5-<var>X.X.X.X</var>-1.i386.rpm
       ```

       Using **`rpm` (64-bit)**:

       ```
       rpm -e newrelic-php5-common-<var>X.X.X.X (VERSION NUMBER)</var>-1.noarch.rpm newrelic-daemon-<var>X.X.X.X (YOUR VERSION NUMBER)</var>-1.x86_64.rpm newrelic-php5-<var>X.X.X.X</var>-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="uninstall-ubuntu-debian"
       title="Ubuntu or Debian"
     >
       Using `apt-get`:

       ```
       sudo apt-get purge newrelic-php5
       ```

       Using **`dpkg` (32-bit)**:

       ```
       dpkg -P newrelic-php5-common_<var>X.X.X.X (YOUR VERSION NUMBER)</var>_all.deb newrelic-daemon_<var>X.X.X.X</var>_i386.deb newrelic-php5_<var>X.X.X.X</var>_i386.deb
       ```

       Using **`dpkg` (64-bit)**:

       ```
       dpkg -P newrelic-php5-common_<var>X.X.X.X (YOUR VERSION NUMBER)</var>_all.deb newrelic-daemon_<var>X.X.X.X</var>_amd64.deb newrelic-php5_<var>X.X.X.X</var>_amd64.deb
       ```
     </Collapser>
   </CollapserGroup>
4. Restart your web servers (Apache, Nginx, PHP-FPM, etc.).
