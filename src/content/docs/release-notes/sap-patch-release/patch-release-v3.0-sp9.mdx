## Patch files

V3.0 SP9 has two files:
  - K-300SP09NEWR.SAR
  - _NEWR_DATA_PROVIDERV3.9.SAP

## Procedures to implement the service pack

### Service pack installation and configuration

  - **Installation Location**: Install this service pack on the Central Monitoring System's client 000.
  - **Subsequent Configuration**: All further configuration should be performed in the Central Monitoring System's business client.
  - **Detailed Instructions**: Refer to the **Installation Guide V3.0** (Appendix VIII: Install Service Pack) or the **Admin Guide V3.0** (Section: Install Service Pack) for comprehensive installation instructions.

### Authorization role update

  - This service pack includes an authorization role update in the new role file `_NEWR_DATA_PROVIDERV3.9.SAP`.
  - **Details**: Refer to the [Update in the authorization role](#update-in-the-authorization-role) section for more information.
  - **Action Required**: Use this new role file to update the RFC users' authorization profiles in the monitored systems.
  - **Note**: The authorization role file `_NEWR_DATA_PROVIDER.SAP` is also updated, so new installations will automatically incorporate the new authorization objects.

## New features

This service pack introduces several new features, including:

  - Backup and Restore functions within the Registry transaction (`/NEWR/Registry`).
  - The ability to generate registry configurations for only selected monitored systems.

See [Instructions for the new features](#instructions-for-the-new-features) section for instruction on using these new features.

## Updates in this patch

### BTP monitoring related updates

  - Support pagination in **BTP** and **CALM** based API data providers function.
  - Special pagination support for `NR_BTP_IS_MPL`
    - New finisher function: `/NEWR/BTP_MPL_FINISHER_V2`
    - New configuration Parameter `API_PAGINATION_LIMIT`. This parameter is
    default to 500 (delivered). SAP API allows up to 1000 records per call.
  - Query change for CALM data provider `NR_CALM_LANDSCAPE`. The `API_QUERY` is
    updated with a new parameter `?limit=999`. This change is due to a change in **SAP CALM API**, now by default only returns 100 records. Adding this limit allows a more efficient data retrieval.

## Other updates

### New features

  - **Generate Data Provider Configuration**: Now, you can easily generate or remove data provider configurations for one or more specified monitored systems.
  - **Backup and Restore Data Provider Registry**: A new feature has been added to enable backing up and restoring the **Data Provider Registry**.
  - **API_TRACE Data Provider Type Support**: The resend program (`/NEWR/RESEND_FAILED_API_TRANS`) now supports the new `API_TRACE` data provider type.

### Improvements

  - **Improved NR_SAP_ABAP_WS payload retrieval**: The `NR_SAP_ABAP_WS` data provider now allows the retrieval of attributes under a **Node Only** tag, which streamlines data access and processing.
  - **Enhanced job log details**: The collection manager job (`/NEWR/ONE_COLLECTION`) now includes additional messages in the job log, offering better visualization of job execution details.

## Instructions for the new features [#instructions-for-the-new-features]

### Recommended service patch application procedure

Follow these steps to apply a New Relic Service Pack:

  - Apply the Service Pack file:
    1. Log in to the **Central Monitoring System** client 000,
    2. Use transaction **SPAM** to apply the Service Pack file.
  - Copy configuration templates:
    1. Log in to the **Central Monitoring System** business client.
    2. Start the menu `/n/NEWR/INSTALL`.
    3. Use the **Copy Template Tables from Client 000** menu option.
  - Generate configurations for new data providers:
    1. Use the **Generate Configurations for New Data Providers** menu option to generate delta changes for the installed Service Pack.

<Callout type="important">
  - Avoid adding or changing monitored systems during the Service Pack application.
  - If needed, modify monitored systems after the Service Pack is applied using the **Maintain Monitored Systems** menu.
  - When adding new monitored systems, only generate newly added systems, do not re-generate the entire configuration. Refer to the [Generate data provider configuration for specific monitored systems](#data-provider-specific-monitor) section for more details.
</Callout>

## Update data provider configuration template
It is now recommended to update only the configuration templates for new custom data
providers, instead of updating all templates using the "Update Configuration Templates"
option from the installation menu. Use the backup function to save any configuration
changes. Template changes should be applied specifically to the data providers
required for newly onboarded systems.
To update the template for a specific data provider:

1. Use transaction SE38 to run the program /NEWR/REP_INSTALL_REPLACING.
2. Set the "Transfer Stage" to 0 (do not change).
3. Specify only the data provider template from the registry entries that needs to be
updated.

This focused approach ensures that only necessary templates are modified,
streamlining the update process and minimizing potential disruptions to other
configurations.

## Backup and restore data provider configuration
These new backup and restore features in the New Relic Service Pack's Registry
transaction (/NEWR/REGISTRY) significantly enhance configuration management
capabilities. They allow for both granular and comprehensive backups and restoration
of data provider configurations.
Here's a breakdown of the new functions:

### Backup and restore functions in New Relic Registry (/NEWR/REGISTRY)

Available in change mode:

● Backup One: Creates a backup of the configuration for a single, selected data
provider.
● Backup All: Creates a backup of the configurations for all currently active data
providers.
● Restore One: Restores the configuration for a single data provider using a
previously created backup, whether it was a "Backup One" or "Backup All" version.
● Restore All: Restores the configurations for all data providers from one of the
"Backup All" versions.

These functions are also accessible via the "Table View" menu when you are in "Change
Mode."

## Enhancements for better management:
● Backup comments: When creating a backup, you have the option to add a
descriptive comment or description to help you easily identify the backup version
in the future.
● Version selection for restore: When restoring a data provider's configuration, you
can choose from a list of previously backed-up versions.
● Restore options: After selecting a specific backup version for restoration, you
have the following options:
○ Restore: This button directly initiates the restoration of the configuration
from the chosen backup version.
○ DP header from backup: Allows you to view the technical settings of the
Data Provider from the selected backup version. This includes details like
the data provider function name and frequencies.
○ DP configuration from backup: Enables you to view the specific
configuration parameters within the selected backup version.

These new functionalities provide greater control and flexibility over managing New
Relic configurations, allowing for easier recovery from accidental changes and
streamlined updates.
