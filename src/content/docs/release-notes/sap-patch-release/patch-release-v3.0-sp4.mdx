## Patch files

V3.0 SP4 has one file:
  - K-300SP04NEWR.SAR

## Updates in this patch

- V3.0 SP4 includes the following features:

  - **New utilities**

    - **Self monitoring/protection utility**

      Issues in the monitored systems, such as missing authorization, RFC interface changes between SAP versions, and resource constraints (CPU, memory, etc.), may cause data provider functions to dump in the remote monitored systems. The new self monitoring/protection utility records and alerts on the dump caused by data collection and stops data collection on the specific remote system until the issues are resolved.

      The new utility includes the following components:

      - Monitoring RFC calls to the remote systems. Captures any System Failure exceptions and records Data Provider Name, RFC Destination, and Remote Function Module on the `/NEWR/PROTECTION` table.
      - Error messages are collected and sent to the New Relic platform as log entries (`LOGTYPE = 'NR.RFC.CALL.STATUS.LOG'`), which can be used to create alerts.
      - The erred Data Provider + RFC destination combinations are skipped in the future data collection runs, until the issues are resolved and removed from the transaction `/NEWR/PROTECTION`.

    - **New Relic account ID field**

      A column (New Relic Account ID) is added to the Monitored System Configuration screen. The Monitored System/Database can now send data to a default NR account. If left blank, the collection job will decide which NR account to send data to.

## Bug fixes

  - The HANA database connection instance is now released after each SQL query and recreated for the new query. This is to fix the issue that too many database connections are created for the HANA data providers.
