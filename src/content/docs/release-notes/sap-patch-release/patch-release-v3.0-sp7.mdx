## Patch files

V3.0 SP7 has one file:
  - K-300SP07NEWR.SAR

## Procedures to implement the service pack

### Service pack installation and configuration

  - **Installation Location**: Install this service pack on the Central Monitoring System's client 000.
  - **Subsequent Configuration**: All further configuration should be performed in the Central Monitoring System's business client.
  - **Detailed Instructions**: Refer to the **Installation Guide V3.0** (Appendix VIII: Install Service Pack) or the **Admin Guide V3.0** (Section: Install Service Pack) for comprehensive installation instructions.
  - This service pack includes enhancement on data provider `NR_SAP_ABAP_WS`. If customizing this data provider is desired, see the [Manual configuration steps](#manual-config) for instructions.

## Updates in this patch

- BTP Monitoring Related Updates:

  - Allow that **BTP/CALM** API Host to be accessed via the same proxy server configured for the Authentication HTTP destination. This enhancement is useful when the Central Monitoring requires a proxy server to access **BTP** and/or **CALM** API endpoints.

  - Export the **BTP/CALM** service identifier attribute (`SOURCE_ACCT`) when telemetry data is sent to New Relic cloud. This enhancement is useful when multiple **BTP/CALM** accounts are monitored.

## Other updates

  - Update `NR_SAP_ABAP_WP` to allow retrieval of all data and attributes under a **Node Only** tag.
  - Fixing bug in `NR_SAP_JOB_ALL` where the timestamp of the unfinished jobs is using the local time as the UTC time.
  - Bugs in **Test** function of the registry have been fixed.
  - New configuration parameter `BYPASS_PARA_ID` for skipping the configuration parameter for pattern recognition when updating the configuration template. This prevents word strings in certain parameters, such as `API_PATH` and `API_QUERY` to be recognized as `SYS_ID`.

## Manual configuration steps [#manual-config]

### Configure NR_SAP_ABAP_WS to retrieve XML payload tags

Data Provider `NR_SAP_ABAP_WS` is designed with capability to retrieve the values of the payload web service XML tag values. In the past only **data** type or **attribute** type of tags can be retrieved, This service pack allows the **Node** type tag can also be specified for retrieval.

The payload tag to be extracted is configured using configuration parameter `TAG_LIST`. Here is an example:

<img
  title="Image of SAP NR_SAP_ABAP_WS data provider configuration for XML payload tag retrieval"
  alt="Image of SAP NR_SAP_ABAP_WS data provider configuration for XML payload tag retrieval"
  width="800"
  src="/images/NR_SAP_ABAP_WS.webp"
/>

In previous versions, only data or attribute tag names can be configured for retrieval. Here is a XML fragment:

```xml
<NodeTag Attribute = Attribute_value>
<DataTag> Data_value </DataTag>
</NodeTag>
```

To retrieve the data from the XML, the `TAG_LIST` is configured with the value:
`Attribute;DataTag`.

Then the NR_SAP_ABAP_WS event will have these attributes:

`Attribute`: Attribute_value
`DataTag`: Data_value

Now the `TAG_LIST `can be configured as `NodeTag`, in this case, the event attributes is as following:

`NodeTag_Attribute`: Attribute_value
`NodeTag_DataTag`: Data_value
