## Overview

In this service pack, we are introducing a series of API-based data providers to monitor the **SAP BTP/CALM** accounts.

New data collection authentication method for BTP Integration Suite **API Management**

- In addition to the two data collection authentication methods introduced in SP05, a new data collection authentication method is introduced in SP06 using the **API Management**, **API Portal**.
- This option requires subscription to the service **API Management**, **API Portal** along with plan **apiportal-apiaccess**.
- Please ask your **BTP Global Account Administrator** to subscribe to this service under **Entitlements > Service Assignments**.

## Updates in this patch

  - BTP Monitoring Related Updates:

    - **New data providers**: In SP06, New data providers for BTP monitoring has been added, while existing BTP data providers are enhanced:
      a) `NR_BTP_IS_APIMGMT`- Data provider to retrieve BTP integration suite **API Management** logs. This data provider is assigned to the Monitoring Group `BTP_APIM`. The NR event generated from this data provider is `NR_IS_APIM_LOG`.
      b) `NR.CALM.HM`- Data provider to retrieve all the **Health Monitoring** (HM)data via **CloudALM HM API**. This data provider is assigned to Monitoring Group `CALM`. The metric generated from this data provider can be identified using `dp_name` = `NR.CALM.HM`.
      c) `NR.CALM.JAM`- Data provider to retrieve all the **Job and Automation** (JAM) data via **CloudALM JAM API**. This data provider is assigned to Monitoring Group `CALM`. The metric generated from this data provider can be identified using `dp_name` = `NR.CALM.JAM`.
      d) `NR.CALM.RUM`- Data provider to retrieve all the **Real User Monitoring** (RUM) data via **CloudALM RUM API**. This data provider is assigned to Monitoring Group `CALM`. The metric generated from this data provider can be identified using `dp_name` = `NR.CALM.RUM`.

## Bug fixes/enhancements

    - Fixed the duration calculation and missing RETRY data issue in DP `NR_BTP_IS_MPL`.
    - Added test functionality for `API_METRIC DP's` to display output as JSON from the Registry screen.

## Other updates

    - Testing Features on New Relic Registry are updated. OTel API based metric can display the result in JSON format when tested from Registry now.
    - The protection feature, which will stop data provider with remote errors, can now be switched on or off by a global configuration parameter
    - Enhanced all the trace DP FM `/NEWR/SAP_WORKLOAD_TRACE*` to truncate the EID namespace based on the expressions (parameter id - `EID_EXCLUDE_EXPR`) stored in registry configuration.
