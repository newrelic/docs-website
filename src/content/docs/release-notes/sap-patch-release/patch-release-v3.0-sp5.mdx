## Overview

Starting from this service pack, we are introducing a series of API-based data providers to monitor BTP accounts.

BTP data are collected in two methods:
  - Through BTP Process Integration Runtime Service. This option requires subscription to the "Process Integration Runtime Service" in the monitored BTP accounts.
  - Through SAP CloudALM. This option requires setting up a CloudALM account to monitor the BTP accounts.

## Patch files

V3.0 SP5 has one file:
  - K-300SP05NEWR.SAR

## Updates in this patch

  - V3.0 SP5 includes the following features:

    - New data provider types

      a) API Event - Event data from API calls

      b) API Metric - Metric data from API calls

    - New data providers

      In SP05, the following new data providers are included:

      a) `NR_BTP_IS_MPL`- Data provider to retrieve BTP integration suite iFlow message logs. This data provider is assigned to Monitoring Group BTP_IS. The NR event generated from this data provider is `NR_IS_IFLOW_LOG`.

      b) `NR_CALM_IS_MPL`- Data provider to retrieve BTP integration suite iFlow message logs via CloudALM Integration Monitoring. This data provider is assigned to Monitoring Group CALM. The NR event generated from this data provider is `NR_IS_IFLOW_LOG`.

      c) `NR_CALM_LANDSCAPE`- Data provider to retrieve CloudALM landscape monitoring data. This data provider is assigned to Monitoring Group CALM.

    - Updated Monitored System configuration screen to accommodate API based monitoring.

      - **API Host** field is added to the configuration screen.

## Bug fixes

    - Fixed the issue when ST-PI version in the monitoring system and monitored systems do not match. It may cause the data provider function for `NR_SAP_SYSTEM_USER_SESSION` to dump.

    - Fixed the issue when RFC Destination filter is applied. It does not pick up monitored systems that have no NR account specified in the configuration.

    - Automatic switching between HTTP and HTTPS protocols when calling SAPControl web services.

    - Add sec to the schedule starting time field on the Run Time control table. This requires the runtime table to be cleared after installing the SP05.

    - Fixed issue in numeric attributes handling in finisher functions.
