## New Relic PHP Agent v10.2.0.314

### New Features
* newrelic-daemon: Informational messages will be logged if the number of connected applications falls below 25 or exceeds 200. [#534](https://github.com/newrelic/newrelic-php-agent/pull/534)

### Bug Fixes
* agent: Fixed SIGSEGV in Drupal 7 `drupal_http_request` instrumentation for PHP 7.4, 8.0 and 8.1. [#538](https://github.com/newrelic/newrelic-php-agent/pull/538)
* agent: Fixed displaying `newrelic.application_logging*.enabled` value in PHP info output. [#535](https://github.com/newrelic/newrelic-php-agent/pull/535)
* agent: When forwarding application logs, send the log message as a well-formed JSON string that is correctly escaped. [#540](https://github.com/newrelic/newrelic-php-agent/pull/540)
* agent: Reduce the severity of a log message when an agent does not detect environmental variables. [#539](https://github.com/newrelic/newrelic-php-agent/pull/539)


### Support statement
* New Relic recommends that you upgrade the agent regularly and at a minimum every 3 months. For more information on supported agent versions and EOL timelines, check out our [PHP EOL policy](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).
