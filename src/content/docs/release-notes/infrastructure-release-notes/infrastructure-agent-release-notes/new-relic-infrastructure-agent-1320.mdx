A new version of the agent has been released. Follow standard procedures to [update the infrastructure agent](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). We recommend you upgrade the agent regularly, at least every 3 months.

As of this release, the oldest supported version is [infrastructure agent 1.12.5](https://docs.newrelic.com/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1125).

## New 
* Added `logs_queue_size` configuration option for integrations config files. This will allow specifying how many log lines will be reported for each integration if collection fails #1439
* Added swap metrics #1437
* Add network profiler to help troubleshooting network issues. To enable it add log include filter component `HttpTracer` #1433
* Added host NTP offset as an attribute to SystemSample to detect real collection time of the samples #1432

## Changed
* Bump newrelic/nri-prometheus to v2.16.6 #1436

## Fixed
* Issue when checks were still performed even if the status server failed to initialize #1438
* Hide license key when logging FB config file #1442
