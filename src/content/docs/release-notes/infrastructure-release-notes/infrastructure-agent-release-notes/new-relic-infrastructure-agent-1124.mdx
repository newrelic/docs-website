## Notes

A new version of the agent has been released. Follow standard procedures to [update your infrastructure agent](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent).

### Added

* [#77](https://github.com/newrelic/infrastructure-agent/pull/77), [#84](https://github.com/newrelic/infrastructure-agent/pull/84) The Log Forwarder now supports Winlog events and allows EventID filtering

For more information on the Log Forwarder, [see Forward your logs using New Relic Infrastructure (Winlog)](https://docs.newrelic.com/docs/logs/enable-log-management-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent#winlog).

### Fixed

* [#75](https://github.com/newrelic/infrastructure-agent/pull/75) Fixed a bug that produced this error message can't save submission time when incorrectly trying to save submission time for the host entity that was not required.

### Changed

* [#82](https://github.com/newrelic/infrastructure-agent/pull/82) The built-in [Flex integration](https://github.com/newrelic/nri-flex) has been updated to version 1.3.4. For more information, [see the Flex changelog](https://github.com/newrelic/nri-flex/releases/tag/v1.3.4).
* [#78](https://github.com/newrelic/infrastructure-agent/pull/78) Upgraded fsnotify to v1.4.9 to improve the detection of file system changes.
