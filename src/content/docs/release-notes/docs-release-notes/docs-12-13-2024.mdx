### New docs

* [Logging with the iOS agent](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/ios-remote-logging/) introduces enhanced logging APIs in the iOS agent, including log level configuration and automatic log collection features.
* The [Kubernetes APM auto-attach](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator), formerly known as the Kubernetes agent operator, now streamlines full-stack observability for Kubernetes environments by automating APM instrumentation alongside Kubernetes agent deployment.
* [Forward AWS services logs to New Relic](/install/aws-logs/?service=aws_services) include a new log integration method using a CloudFormation template to forward logs, customize triggers, and configure endpoints from New Relic One, in addition to existing methods.

### Major changes

* Updated the version support information (under prerequisites section) about the [service broker for VMware Tanzu Application Service (TAS)](/docs/infrastructure/other-infrastructure-integrations/cloudfoundry-integrations/vmware-tanzu-service-broker-integration/).
* Introduced Podman environment support in the [synthetic job manager](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager/).
* Updated [Node.js agent compatibility](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/).   
* Deprecated the Snowflake integration with New Relic. Refer the [topic](/docs/infrastructure/host-integrations/host-integrations-list/snowflake-integration/) to understand on usage of Snowflake.
* Updated [JMX monitoring integration](/install/jmx/?dropdown1=linux&deployment=apt) to include instructions for setting `JAVA_HOME` when Java isn't found on the PATH.
* Updated [Remove applications from New Relic](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/#remove-otel-apps) to include guidance on using NerdGraph to delete entities for OpenTelemetry apps.
* Made compliance-related updates in [FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp/#applicable-services), [HIPAA](/docs/security/security-privacy/compliance/certificates-standards-regulations/hipaa/#applicable-services), [ISO 27001 standard](/docs/security/security-privacy/compliance/certificates-standards-regulations/iso-27001/#applicable-services), [PCI DSS](/docs/security/security-privacy/compliance/certificates-standards-regulations/pci-dss/#applicable-services), [HITRUST](/docs/security/security-privacy/compliance/hitrust/#not-scope) and regulations based on New Relic's 2024 annual assessments.
* Introduced [Automatically observe page resource assets](/docs/browser/new-relic-browser/browser-pro-features/page-resources/) and updated [Experimental features in browser monitoring](/docs/browser/new-relic-browser/configuration/experimental-features/) pages to detail new experimental features for tracking browser performance events with New Relic Browser.

### Minor changes

* New Relic extended end of life support for [Azure SQL database integration polling API](/eol/2024/04/eol-04-24-24-polling-api/) to July 31, 2025.
* Updated [Ruby agent requirements and supported frameworks](/docs/apm/agents/ruby-agent/getting-started/ruby-agent-requirements-supported-frameworks/#http_clients) to include support for additional message broker gem.
* In [EntitiesByDomainTypeQuery]((/docs/new-relic-solutions/build-nr-ui/sdk-component/query-and-storage/EntitiesByDomainTypeQuery/)), `sortBy` attribute is replaced by `sortType` for fetching sorting criteria.
* Updated the latest supported version for [Compatibility and requirements for the Node.js agent](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#instrumented-modules).
* Added an example in the Node.js agent configuration details to demonstrate the use of prefixes to [exclude](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#tx-attributes-exclude) and [include](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#tx-attributes-include) attributes from transaction events.
* Updated the [Confirming bulk import activity](/install/vm/?deployment=new#confirming-bulk-import-activity) to indicate a `success` status of `true` in the HTTP response from the `v1/security/import/dependabot` endpoint.
* Updated the [Not seeing replays in the UI](/docs/browser/browser-monitoring/browser-pro-features/session-replay/#Troubleshoot-missing-data) in Session replay to note that dropping PageView event data may prevent replays from displaying.
* Updated [Monitor Jenkins with OpenTelemetry and New Relic](/docs/infrastructure/other-infrastructure-integrations/monitoring-jenkins-ot/#quickstart) to ensure the prebuilt dashboard reflects custom OTEL_SERVICE_NAME configurations by adjusting entity.name conditions.
* Updated [Ruby agent requirements and supported frameworks](/docs/apm/agents/ruby-agent/getting-started/ruby-agent-requirements-supported-frameworks/#ruby_versions) to include support for Rails / Active Record 7.1, 7.2 and 8.0 and OpenSearch reference
