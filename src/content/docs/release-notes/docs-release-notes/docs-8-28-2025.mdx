### New docs

* Added [Access management for managing cardinality](/docs/data-apis/ingest-apis/metric-api/cardinality-access-management/) to control user access to cardinality-related metrics and manage permissions for metric ingestion limits.
* Added [Monitor cardinality](/docs/data-apis/ingest-apis/metric-api/monitor-cardinality/) to track and monitor your metric cardinality usage to optimize data ingestion and avoid unexpected costs.


### Minor changes

* Added [new configuration parameter](/docs/ebpf/linux-installation/#config-params) `DROP_DATA_FOR_ENTITY` for ebpf.

### Release notes

* [Browser agent v1.296.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.296.0)
  * Improved session trace data collection by implementing 16KB payload size threshold for data harvests, consistent with other browser agent features.
  * Fixed a bug that removes the event buffer inspection event to avoid a memory leak on using `console.log` with the `newrelic` window event listener.

* [Kubernetes integration v3.45.0](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-3-45-0/)
   * This integration is included in the following chart versions:
     * [newrelic-infrastructure-3.50.0](https://github.com/newrelic/nri-kubernetes/releases/tag/newrelic-infrastructure-3.50.0)
     * [nri-bundle-6.0.11](https://github.com/newrelic/helm-charts/releases/tag/nri-bundle-6.0.11)

* [React native agent v1.5.9 ](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-159)
   * Resolved the issue to update import statements for error stack parsing compatible with React Native version 0.78 and lower.
* [React native agent v1.5.8 ](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-158)
   * Updated the Native iOS agent to version 7.5.9.
* [React native agent v1.5.7 ](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-157)
   * Updated the Native Android agent to version 7.6.8.
   * Updated the Native iOS agent to version 7.5.7.

* [Lambda extension v2.3.21](/docs/release-notes/lambda-extension-release-notes/lambda-extension-2.3.21)
   * APM fusion mode is in limited preview.
* [Lambda extension v2.3.22](/docs/release-notes/lambda-extension-release-notes/lambda-extension-2.3.22)
   * Added `APM` connect retry logic.
   * Added `godebug` to resolve AWS Firewall issue.
   * Added `requestId` extraction from logs for `Node.js` runtime.
   * Updated Readme for `--enable-license-key-secret`.
* [Lambda extension v2.3.23](/docs/release-notes/lambda-extension-release-notes/lambda-extension-2.3.23)
   * Added option to build `nodejs` test Layers for local testing.
   * Updated AWS SDK and Go Version to address Security Vulnerabilities `[CVE-2020-8911, CVE-2020-8912, CVE-2025-0913, CVE-2025-4673]`.

* [Ruby Agent v9.21.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-9-21-0)
   * In Serverless APM mode, transaction names now include the event source name as a prefix. This enhancement improves transaction identification and traceability in the New Relic UI by clearly indicating the triggering event source.
   * Reverted changes to thread tracking logic for span initialization. This change restores the previous behavior of tracking the thread where spans start, addressing issues that arose from the updated logic.
