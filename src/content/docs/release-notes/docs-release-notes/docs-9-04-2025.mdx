### Release notes

* [Infrastructure agent v1.67.3](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1673/)
  * Upgraded embedded integration versions.

* [Kubernetes integration v3.45.0](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-3-45-0/)
  * This integration is included in the following chart versions:
    * [newrelic-infrastructure-3.50.0](https://github.com/newrelic/nri-kubernetes/releases/tag/newrelic-infrastructure-3.50.0)
    * [nri-bundle-6.0.11](https://github.com/newrelic/helm-charts/releases/tag/nri-bundle-6.0.11)

* [Ruby agent v9.21.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-9-21-0/)
  * In Serverless APM mode, transaction names now include the event source name as a prefix. This enhancement improves transaction identification and traceability in the New Relic UI by clearly indicating the triggering event source.
  * Reverted changes to thread tracking logic for span initialization. This change restores the previous behavior of tracking the thread where spans start, addressing issues that arose from the updated logic.

* [React Native agent v1.5.9](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-159/)
  * Resolved the issue to update import statements for error stack parsing compatible with React Native version 0.78 and lower.
