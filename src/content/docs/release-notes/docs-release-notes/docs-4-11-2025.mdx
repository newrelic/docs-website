### New doc

* [AWS SQS message queue instrumentation](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) supports distributed trace headers for SQS messages starting from version 2.1.0, with guidance on custom instrumentation for message receiver operations.
* [Alert Intelligence Tool](/docs/agentic-ai/tools/alert-intelligence-tool) enhances the ability to assess application health and performance through agentic integration with our partners.
* [Confluent Cloud managed streaming for Apache Kafka](/docs/infrastructure/other-infrastructure-integrations/confluent-cloud-integration) provides guidance on monitoring Kafka clusters and topics, and setting up alerts for Confluent Cloud.
* [New Relic AI summary for Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/nrai-summary) summarizes and maps session replay event trail data, enabling a better understanding of errors and successes in users' browser journeys.
* [Response intelligence with New Relic AI](/docs/alerts/alert-event-management/response-intelligence-ai) reduces the time to resolve issues and alert events by leveraging New Relic AI with response intelligence.
* [Browser logs](/docs/browser/browser-monitoring/browser-pro-features/browser-logs/overview) capture real-time browser console logs to enhance observability, simplify debugging, and improve user experience.
* [Introduction to Transaction 360](/docs/apm/transactions/workload-performance-monitoring/introdution) offers a revolutionary approach to application monitoring and troubleshooting by automatically capturing and correlating all entities and data related to specific problems or questions.
* [AWS logs in Context](/docs/logs/logs-context/aws-logs-in-context) provides a comprehensive, integrated log management solution that enhances troubleshooting efficiency and system observability within AWS environments.
* Manage and reinstall your current instrumentation with [Agent Control](/docs/new-relic-control/agent-control/reinstrumentation).
* [Agentic Control](/docs/new-relic-control/agent-control/overview) is a lightweight agent supervisor that manages other New Relic and New Relic distributions of OpenTelemetry agents by communicating with Fleet Control to remotely configure, update, and monitor the health of infrastructure-level instrumentation.
* [Pipeline control](/docs/new-relic-control/pipeline-control/overview) allows you to manage the flow of data from your monitored entities to New Relic.
* [Docker container](/docs/infrastructure/host-integrations/host-integrations-list/docker-container-monitoring-integration) integration monitors and reports metrics from Docker containers to New Relic.
* [Lambda response monitoring](/docs/serverless-function-monitoring/aws-lambda-monitoring/response-stream) functions in Node.js applications to gain real-time insights into the performance and behavior of each invocation.


### Major changes

* Added [Amazon CloudWatch Metric Streams](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams), [ElastiCache (Memcached)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached), and [Amazon MQ (ActiveMQ)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) as [Supported entity relationships](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships) for OpenTelemetry.
* Added Amazon CloudWatch Metric Streams with [Amazon Managed Kafka (MSK)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-managed-kafka-msk) to generate MSK entities in New Relic. These entities are tagged with `aws.clusterName` or `aws.kafka.ClusterName`, `aws.awsRegion` or `aws.region`, and `aws.topic` or `aws.kafka.Topic`.
* Added additional [Solr troubleshooting](/docs/apm/agents/java-agent/troubleshooting/java-solr-troubleshooting) instructions for the metric changes in version 8.10.
* New Relic has discontinued the use of the following IP ranges:
    * 3.145.244.128/25
    * 3.77.79.0/25
    * 3.27.118.128/25
    * 20.51.136.0/25
    * 4.197.217.128/25
    * 18.246.82.0/25
    * 158.177.65.64/29
    * 159.122.103.184/29
    * 161.156.125.32/28
    
    As of May 1, 2025, these ranges may be reallocated by the cloud provider to other customers for purposes beyond our control. Please update your [network](/docs/new-relic-solutions/get-started/networks) configurations accordingly.
* Added a troubleshooting document to [retrieve the New Relic license key](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/retrieve-license) for issues using AWS Secrets Manager with New Relic.
* Added a troubleshooting document for [Lambda functions not showing as instrumented](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-not-instrumented-lambda-function).
* Added a troubleshooting document for issues [adding multiple AWS accounts with respective regions](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-unable-to-add-multiple-aws-accounts/).
* Added a troubleshooting document for issues [using AWS Secrets Manager with New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-using-aws-secrets-manager/).
* Added New Relic Agentic integrations to [FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp).
* Added a new document on [New Relic's EC2 Image Builder integration](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-ec2-image-builder-integration).
* Added [Dashboards](/docs/codestream/observability/dashboards) to CodeStream.
* Added [OpenTelemetry Collector for Kubernetes monitoring](/docs/opentelemetry/get-started/collector-k8s-monitoring/opentelemetry-collector-k8s-intro) and [New Relic Distributions of the OpenTelemetry Collector](/docs/opentelemetry/nrdot/nrdot-collector).
* Extended the deadline to May 14, 2025, for updating the Synthetic IP ranges of your [Synthetic monitors](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips/).
* Added new IP ranges for Synthetic monitoring in [Synthetic monitoring](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips/).
* Updated the [Query account document](/docs/accounts/accounts/account-maintenance/query-account-audit-logs-nrauditevent) to include queries for logged users.
* Updated the feature flags for [Android monitoring](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/configure-settings).
* Updated the [Anomaly detection document](/docs/alerts/create-alert/set-thresholds/anomaly-detection) to include information on a new field for anomaly NRQL alert conditions, allowing users to configure seasonality.

* Updated the [supported hours](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings) section.
