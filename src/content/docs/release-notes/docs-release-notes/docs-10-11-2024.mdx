### New docs

* [No crash data appears in release build (Android)](/docs/mobile-monitoring/new-relic-mobile-android/troubleshoot/no-crash-data-appears) tells you what to do if you don't see data related to an application crash on the New Relic UI.
* [How can I use WSL and CodeStream?](/docs/codestream/troubleshooting/using-wsl) provides step-by-step instructions on using Windows Subsystem for Linux (WSL) with CodeStream.
* [`UserAction`](/docs/browser/browser-monitoring/browser-pro-features/user-actions) is a new event type in browser monitoring that helps you understand user behavior with your web application.
* [WithLlmCustomAttributes (Python agent API)](/docs/apm/agents/python-agent/python-agent-api/withllmcustomattributes-python-agent-api) describes the new context manager API, including its requirements, parameters, return values, and an example usage.

### Minor changes

* Added US and EU data center endpoints for the APM agent in [New Relic telemetry endpoints](/docs/new-relic-solutions/get-started/networks/#new-relic-dualstack-endpoints).
* Updated instructions on how to find CodeStream client-side logs when using [Visual Studio](/docs/codestream/troubleshooting/client-logs/#visual-studio).
* Reworded [Notification options for muting rules](/docs/alerts/get-notified/muting-rules-suppress-notifications/#notify) for better clarity.
* Updated the compatibility information for the following packages in [instrumented modules](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#instrumented-modules):
    * `@aws-sdk/client-bedrock-runtime`
    * `@aws-sdk/client-dynamodb`
    * `@aws-sdk/client-sns`
    * `@aws-sdk/client-sqs`
    * `@aws-sdk/lib-dynamodb`
    * `@grpc/grpc-js`
    * `@langchain/core`
    * `@smithy/smithy-client`
    * `next`
    * `openai`
* Corrected the second EU data center endpoint to `212.32.0.0/20` in [data ingest IP blocks](https://docs.newrelic.com/docs/new-relic-solutions/get-started/networks/#ingest-blocks).
* Specified that New Relic's alerts service is executed in the United States whether your data is stored in [New Relic's EU or US data center](/docs/alerts/overview/#eu-datacenter).
* Modified information about how to set a [custom attribute](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#custom-attributes). Also, updated the examples.
* Updated the steps to [unacknowledge and close an issue](/docs/alerts/get-notified/acknowledge-alert-incidents/#unacknowledge-issue).
* Added information on [acknowleding and closing multiple issues](/docs/alerts/incident-management/Issues-and-Incident-management-and-response/#bulk-actions).
* Expanded the [chart refresh rates](/docs/query-your-data/explore-query-data/use-charts/chart-refresh-rates) introduction to call out that the refresh rate is configurable only if you're on the New Relic compute consumption pricing plan.
* Corrected the required Transport Layer Security (TLS) version information in [TLS encryption](/docs/new-relic-solutions/get-started/networks/#tls).
* Updated Step 6 in the [Kubernetes integration installation](/install/kubernetes) procedure to match the current New Relic UI.
* Noted the importance of the non-billable event, `InternalK8sCompositeSample`, in [Query Kubernetes data](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/find-use-your-kubernetes-data/#view-data), after the table.
* Rewrote the instructions to [view and manage API keys](/docs/apis/intro-apis/new-relic-api-keys/#keys-ui) for accuracy and clarity.
* Updated instructions on how to use the API explorer in [Use the API Explorer](/docs/apis/rest-api-v2/api-explorer-v2/use-api-explorer) and [List application ID, host ID, instance ID](/docs/apis/rest-api-v2/get-started/list-application-id-host-id-instance-id) to match the current API explorer.
* Added back the example for [correlating infrastructure with OpenTelementry APM](/docs/opentelemetry/get-started/collector-infra-monitoring/opentelemetry-collector-infra-intro/#infrastructure-correlation).
* Noted how to avoid duplicate metric names when [migrating to the Azure monitor integration](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#migrate).
