### New docs

* Added Synthetics dual stack support installation guides for [Docker](/docs/synthetics/synthetic-monitoring/private-locations/install-synthetics-dual-stack-docker/), [Kubernetes](/docs/synthetics/synthetic-monitoring/private-locations/install-synthetics-dual-stack-kubernetes/), and [Podman](/docs/synthetics/synthetic-monitoring/private-locations/install-synthetics-dual-stack-podman/) environments to support both IPv4 and IPv6 testing.
* Added [Synthetics private locations runtime transition guide](/docs/synthetics/synthetic-monitoring/private-locations/runtime-transition-guide/) for migrating monitors to Node.js 22 and Chrome 146, with in-place and parallel migration strategies, NRQL queries for monitoring the transition, and AI-assisted monitor fixes.
* Added [Azure Monitor vs. Azure Polling](/docs/infrastructure/amazon-integrations/get-started/azure-monitor-vs-azure-polling/) comparison guide to help users choose the right Azure integration method for collecting Azure metrics in New Relic.

### Major changes

* Comprehensively updated [Cloud Cost Intelligence (CCI)](/docs/cci/overview/) documentation with revised UI workflows, updated budget management, new cost overview page, Kubernetes cost insights, NRQL queries for CCI, and setup guides for AWS, Azure, and GCP.

### Minor changes

* Updated [.NET agent compatibility requirements](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/) to include Hangfire framework support.
* Updated [eBPF overview](/docs/ebpf/overview/) with FedRAMP compliance information.
* Updated [NGINX OpenTelemetry Kubernetes integration](/docs/infrastructure/other-infrastructure-integrations/nginx/nginx-otel-kubernetes/) with Helm installation instructions for NGINX OTel instrumentation.
* Updated [Python agent configuration](/docs/apm/agents/python-agent/configuration/python-agent-configuration/) with Hybrid Agent configuration settings for OpenTelemetry API support.
* Updated [Metric API limits and restricted attributes](/docs/data-apis/ingest-apis/metric-api/metric-api-limits-restricted-attributes/) by removing outdated data points per minute (DPM) limit documentation across metric API, Prometheus, and data optimization pages.
