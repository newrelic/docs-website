### New docs

* [IAST configuration](/docs/iast/iast-configuration) explains how to configure and schedule IAST scans.
* [OCSF integration](/docs/infrastructure/host-integrations/host-integrations-list/ocsf-integration) describes how to use this new integration to monitor your Open Cybersecurity Schema Framework data.

### Minor changes

* Updated the [requirements and compatibility](/install/vsphere/#requirements-and-compatibility) information for [vSphere monitoring integration](/install/vsphere). If your environment consists of more than 800 virtual machines, we've highlighted a workaround so you can report all of your data.
* Updated the code snippet required to [forward Envoy logs to New Relic](/docs/infrastructure/host-integrations/host-integrations-list/envoy-integration/#forward-envoy-logs).
* Updated the code snippets required to [configure NRI-Flex for Nextcloud](/docs/infrastructure/host-integrations/host-integrations-list/nextcloud-integration/#integrate) and to [forward Nextcloud logs to New Relic](/docs/infrastructure/host-integrations/host-integrations-list/nextcloud-integration/#forward-nextcloud-logs-to-new-relic).
* Updated [.NET agent compatibility and requirements](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) for the most up to date versioning (as of these release notes):
    * `StackExchange.Redis` (both minimum supported version and latest verified compatible version)
    * `AWS Bedrock` (latest verified compatible version)
    * `Confluent.Kafka` (latest verified compatible version)
    * `AWSSDK.SQS` (both minimum supported version and latest verified compatible version)
* Updated the [Node.js agent compatibility information](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) for `@aws-sdk/client-bedrock-runtime`, `@aws-sdk/client-dynamodb`, `@aws-sdk/client-sns`, `@aws-sdk/client-sqs`, `@aws-sdk/lib-dynamodb`, `@aws-sdk/smithy-client`, `@langchain/core`, `@smithy/smithy-client`, `express`, `fastify`, `mysql2`, `next`, and `openai`.
* Updated the instructions on how to [see your control plane data](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/configure-control-plane-monitoring/#check-integration-works) in [configure control plane monitoring](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/configure-control-plane-monitoring).
