### New docs

* Added [IBM MQ OpenTelemetry integration](/docs/opentelemetry/integrations/ibm-mq/overview) for monitoring IBM MQ with OpenTelemetry, covering host, Kubernetes, metrics, and troubleshooting setups.
* Added [Redis OpenTelemetry integration](/docs/opentelemetry/integrations/redis/overview) for monitoring Redis with OpenTelemetry, covering Docker, host, and Kubernetes setups.
* Added [HAProxy OpenTelemetry integration](/docs/opentelemetry/integrations/haproxy/overview) for monitoring HAProxy with OpenTelemetry, covering self-hosted and Kubernetes setups.
* Added [Azure VNet flow log monitoring](/docs/network-performance-monitoring/setup-performance-monitoring/cloud-flow-logs/azure-vnet-flow-log-monitoring) for forwarding and querying Azure virtual network flow log data.

### Major changes

* [Cross account destinations](/docs/apis/nerdgraph/examples/cross-account-destinations/overview) is now generally available.

### Minor changes

* Added `blocks` parameter documentation to the `sendSlack` [workflow automation action](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) for sending Slack Block Kit messages, along with a table of supported block types.
* Updated the [Node.js agent compatibility requirements](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) with the latest automated compatibility report.
* Updated the [Docker Linux install](/install/dotnet/?deployment=linux&docker=yesDocker) and [Docker Windows install](/install/dotnet/?deployment=windowsInstall&docker=yesDocker) examples for the .NET agent with instructions for installing a specific version instead of the latest.
* Corrected the [Intelligent workload](/docs/new-relic-solutions/new-relic-one/workloads/create-intelligent-workload) membership update interval from 5 minutes to 10 minutes, noting it's subject to change during public preview.
* Clarified in [NRQL query time range](/docs/nrql/using-nrql/query-time-range) that raw metric datapoint inclusion depends on a datapoint's begin time, not its end time.
* Updated [Workload status configuration](/docs/new-relic-solutions/new-relic-one/workloads/workload-status-configuration) to clarify which status methods are available for Standard versus Intelligent workloads.
* Added guidance for removing a user from an authentication domain or organization to [multiple user records](/docs/accounts/accounts-billing/account-setup/multiple-user-records).
* Updated the [.NET agent compatibility requirements](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) to clarify IBM DB2 support and limitations for both .NET Core and .NET Framework.
* Added support for Ubuntu 26.04 (Resolute) to [log forwarding via the infrastructure agent](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent), via Fluent Bit 5.0.6.

### EOLs

* [End of life for Drupal 7 and 9 monitoring in the PHP agent](/eol/2026/07/eol-07-09-26-drupal-7-9-monitoring): Support ends October 15, 2026.
* [End of life for Node.js 20 runtime support](/eol/2026/07/eol-07-09-26-nodejs-20-runtime): New Relic agents no longer support the Node.js 20 runtime, which reached end-of-life on April 30, 2026.

### Release notes

* Stay up-to-date with our latest releases:

  * [NRDOT Host Monitoring v1.19.0](/docs/release-notes/nrdot-release-notes/nrdot-host-2026-07-06):
    * Verified the host use case to work with `nrdot-collector:1.19.0`.

  * [Fluent Bit 5.0.8 / New Relic Fluent Bit Output Plugin 3.7.0](/docs/release-notes/fluentbit-release-notes/fluentbit-26-06-30):
    * Upgraded to Go 1.26.4 for CVE fixes.
    * The infrastructure agent now recommends Fluent Bit 5.0.8 (Linux) and 5.0.7 (Windows) packages.

  * [iOS agent v7.7.4](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-774):
    * Simulator devices now report as `simulator-(deviceType)`.
    * Removed the KMP detection that caused instability in 7.7.3.
    * Fixed crashes in async web request recording and a harvest config race condition.

  * [Python agent v13.2.0](/docs/release-notes/agent-release-notes/python-release-notes/python-agent-130200):
    * Added instrumentation for Google ADK agent workflows.
    * Added token counting for Anthropic, Google GenAI, OpenAI, and AWS Bedrock.
    * Fixed an instrumentation crash with `graphql-core` v3.2.10 and reduced the outbound `tracestate` header size.

  * [Node.js agent v14.2.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-2-0) and [v14.2.1](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-2-1):
    * Added Node.js 26 support and a `profiling.source_mapping.enabled` config option.
    * Fixed a memory leak caused by duplicate `uncaughtException` handlers registered by the `winston` instrumentation.

  * [Kubernetes integration v4.3.7](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-4-3-7):
    * Released as part of the newrelic-infrastructure-4.3.7 and nri-bundle-8.0.2 chart versions.

  * [Browser agent v1.318.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.318.0):
    * Added a `web_sockets` init config block to enable WebSocket monitoring (disabled by default).
    * Added internal detection for FCP, INP, CLS, and LCP when the agent is spawned by a micro frontend.

  * [Mobile app for Android v5.43.3](/docs/release-notes/mobile-apps-release-notes/new-relic-android-release-notes/new-relic-android-5433):
    * Bug fixes and UI improvements.

  * [React Native agent v1.8.7](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-187) and [Flutter agent v1.2.9](/docs/release-notes/mobile-release-notes/flutter-release-notes/flutter-agent-129):
    * Updated the native iOS agent to v7.7.4.

  * Synthetics runtimes — [Job Manager v584](/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-584), [Node API runtime rc1.28](/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.28), [Node browser runtime rc1.28](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.28) and [rc1.29](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.29), and [Ping runtime v1.75.0](/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.75.0):
    * Patched underlying OS and dependency CVEs across all synthetics images.
    * Streamed large HAR payloads (over 100 MB) directly to disk instead of buffering in memory, for the Node browser runtime.
