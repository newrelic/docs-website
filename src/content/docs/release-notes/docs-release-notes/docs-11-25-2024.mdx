### New docs

* [Amazon MemoryDB integration](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-memorydb-monitoring-integration) describes how to activate the integration with New Relic. The integration enables you to collect telemetry data about your Amazon MemoryDB clusters, query incoming data, and build dashboards.
* [Add custom tags (labels) to application logs](/docs/logs/logs-context/custom-tags-agent-forwarder-logs) explains that you can add custom labels to agent-forwarded logs. 

### Major changes

* Updated the [AWS integrations documentation](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations) for clarity and organization. Key changes include emphasizing CloudWatch Metric streams as the primary AWS integration method, enhancing navigation and reducing redundancy by consolidating and streamlining content, and making copy edits for better readability.
* Updated the steps to [create a postmortem](/docs/alerts/alert-event-management/postmortems-applied-intelligence/#create-postmortem) for accuracy.
* Made compliance-related updates in [Certifications, standards, and regulations](/docs/security/security-privacy/compliance/regulatory-audits-new-relic-services) based on New Relic's 2024 annual assessments.
* Added .NET 9 and ASP.NET Core 9 compatibility for the .NET agent, clarified support stance on out-of-support .NET versions, and removed outdated .NET Framework references in [.NET agent: compatibility and requirements](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements).
* Rewrote [Fluent Bit plugin for log forwarding](/docs/logs/forward-logs/fluent-bit-plugin-log-forwarding) for clarity.
* Updated [CodeStream notifications](/docs/codestream/codestream-settings/account-settings) for accuracy.
* Added the [Node.js agent variable](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#cloud_config) details to create a relationship between cloud providers and APM applications.
* Updated the steps to [create an Azure Event Hub](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export/#create-event-hub).

### Minor changes

* Updated the .NET and Ruby agent versions in [Add custom tags (labels) to application logs](/docs/logs/logs-context/custom-tags-agent-forwarder-logs).
* Provided an example to show the use of logging APIs with Unity in [Mobile logs](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs).
* Updated [.NET agent compatibility and requirements](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements).
* Updated [Node.js agent compatibility and requirements](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent).
* Updated default behavior and configuration guidance for network device inclusion and NetworkSample generation in [infrastructure events](/docs/infrastructure/infrastructure-data/default-infra-data).
* Corrected the name of the `localStorage` API and updated information about when session tracking may not work in [Session tracking](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking).
* Updated Node.js agent configuration details to [enable](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels) and [exclude](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels-exclude) labels to getting added to application logs.
