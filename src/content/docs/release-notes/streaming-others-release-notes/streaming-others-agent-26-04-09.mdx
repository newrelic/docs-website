## New features

### QoE Analytics

Added comprehensive video quality tracking with advanced performance metrics.

## What's added

### QoE metrics

  - `averageBitrate`:Time-weighted average bitrate across playback
  - `peakBitrate`: Highest bitrate achieved during playback
  - `startupTime`: Time from content request to first frame (excludes pre-roll ads)
  - `rebufferingRatio`: Percentage of playback time spent rebuffering
  - `totalRebufferingTime`: Total milliseconds in buffer states
  - `totalPlaytime`: Total content playback time in milliseconds
  - `hadStartupError`: Boolean indicating if startup failed before content start
  - `hadPlaybackError`: Boolean indicating if playback errors occurred during session

## New API functions

  - `nrActivateQoeTracking(nr)`: Enable QoE tracking (disabled by default)
  - `nrSetQoeAggregateIntervalMultiplier(nr, multiplier)`: Configure event frequency

## Implementation details

  - QoE events synchronized with harvest cycles
  - Intelligent deduplication prevents redundant events
  - Per-session metric isolation and cleanup
  - Time-weighted bitrate calculations for accuracy
  - Smart startup time calculation excluding ad time

## Default configuration

  - QoE tracking disabled by default
  - Harvest multiplier: 1 (every harvest cycle)
