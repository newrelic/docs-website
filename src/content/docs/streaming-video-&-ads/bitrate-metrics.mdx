The New Relic Streaming Video & Ads agent reports bitrate telemetry attributes that let you monitor streaming quality at the playback level. This page defines each attribute, lists the event types that carry it, and provides example NRQL queries for use in dashboards and alerts.

<Callout variant="important">
  Bitrate attributes are available in Streaming Video & Ads agent version X.X.X and later. Confirm the minimum version with your engineering team before querying these attributes.
</Callout>

## Event types [#event-types]

Bitrate attributes are reported on the following event types:

| Event type | Description |
|------------|-------------|
| `VideoAction` | Fired during normal video playback lifecycle events (play, pause, seek, buffer, bitrate change). |
| `VideoErrorAction` | Fired when a playback error occurs. Includes bitrate context at the time of the error. |

## Bitrate attributes [#attributes]

<CollapserGroup>
  <Collapser
    id="attr-bitrate"
    title="bitrate"
  >
    **Type:** numeric (kbps)

    **Event types:** `VideoAction`, `VideoErrorAction`

    The bitrate of the video stream at the moment the event was fired, expressed in kilobits per second (kbps). This reflects the actual rendition selected by the player's adaptive bitrate (ABR) algorithm at the time of the event.

    A sudden drop in `bitrate` values across sessions can indicate CDN or network degradation.
  </Collapser>

  <Collapser
    id="attr-bitrateAvg"
    title="bitrateAvg"
  >
    **Type:** numeric (kbps)

    **Event types:** `VideoAction`, `VideoErrorAction`

    The average bitrate delivered over the current session up to the point the event fired. Computed as a rolling average of all rendition segments played.

    Use `bitrateAvg` to understand session-level streaming quality over time, as opposed to instantaneous `bitrate` which can fluctuate rapidly.
  </Collapser>

  <Collapser
    id="attr-bitrateMax"
    title="bitrateMax"
  >
    **Type:** numeric (kbps)

    **Event types:** `VideoAction`, `VideoErrorAction`

    The highest bitrate rendition played during the current session. Reflects the best quality the player was able to deliver to the viewer.
  </Collapser>

  <Collapser
    id="attr-bitrateMin"
    title="bitrateMin"
  >
    **Type:** numeric (kbps)

    **Event types:** `VideoAction`, `VideoErrorAction`

    The lowest bitrate rendition played during the current session. A consistently low `bitrateMin` may indicate that viewers are experiencing degraded quality throughout their session.
  </Collapser>

  <Collapser
    id="attr-bitrateShift"
    title="bitrateShift"
  >
    **Type:** numeric (count)

    **Event types:** `VideoAction`

    The cumulative number of bitrate shifts (rendition changes) that have occurred in the current session up to the point the event fired. Frequent bitrate shifts may indicate an unstable network connection or overly aggressive ABR behaviour.
  </Collapser>

  <Collapser
    id="attr-targetBitrate"
    title="targetBitrate"
  >
    **Type:** numeric (kbps)

    **Event types:** `VideoAction`

    The bitrate the player requested or targeted for the next segment before ABR adjustment. Comparing `targetBitrate` to `bitrate` helps identify cases where the player intended to play a higher quality rendition but was unable to sustain it.
  </Collapser>
</CollapserGroup>

<Callout variant="caution">
  Attribute names and availability may vary by platform (Browser, iOS, Android, Roku). Verify exact attribute names against the agent-specific API reference before building production queries. Confirm the complete attribute list with the Streaming Video & Ads engineering team.
</Callout>

## Example NRQL queries [#queries]

Use these queries as starting points in the [Raw NRQL Editor](/docs/query-your-data/explore-query-data/dashboards/custom-widget-support/#nrql-editor) or [query builder](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/).

### Average bitrate over time [#avg-bitrate-timeseries]

Visualizes average bitrate across all sessions as a time series. Use to detect degradation trends.

```sql
SELECT average(bitrateAvg) AS 'Avg Bitrate (kbps)'
FROM VideoAction
TIMESERIES AUTO
SINCE 24 hours ago
```

### Average bitrate by platform [#avg-bitrate-by-platform]

Compares average bitrate across platforms (Browser, iOS, Android, Roku) to identify platform-specific quality issues.

```sql
SELECT average(bitrateAvg) AS 'Avg Bitrate (kbps)'
FROM VideoAction
FACET platform
SINCE 24 hours ago
```

### Distribution of bitrate renditions played [#bitrate-distribution]

Shows the spread of bitrate values delivered to viewers. A heavily left-skewed distribution indicates most viewers received low-quality streams.

```sql
SELECT histogram(bitrate, width: 500, buckets: 20)
FROM VideoAction
SINCE 24 hours ago
```

### Sessions with frequent bitrate shifts [#bitrate-shift-sessions]

Identifies sessions with more than 5 bitrate shifts, which may signal an unstable connection.

```sql
SELECT count(*) AS 'Unstable sessions'
FROM VideoAction
WHERE bitrateShift > 5
FACET sessionId
SINCE 1 hour ago
LIMIT 100
```

### Bitrate at time of error [#bitrate-at-error]

Shows the bitrate value recorded when a playback error occurred. Use to correlate low bitrate with error rates.

```sql
SELECT average(bitrate) AS 'Bitrate at Error (kbps)', count(*) AS 'Error count'
FROM VideoErrorAction
FACET errorMessage
SINCE 24 hours ago
```

### P50 / P95 bitrate by content type [#bitrate-percentiles]

Returns percentile bitrate values segmented by content type (VOD vs. live). Useful for SLO tracking.

```sql
SELECT percentile(bitrateAvg, 50, 95) AS 'Bitrate percentile (kbps)'
FROM VideoAction
FACET contentType
SINCE 7 days ago
```

### Bitrate using the Raw NRQL Editor with dashboard filters [#bitrate-with-filter]

To make a custom bitrate widget that responds to dashboard-level filters, include the `/* {andWhereClause} */` placeholder:

```sql
SELECT average(bitrateAvg) AS 'Avg Bitrate (kbps)'
FROM VideoAction
WHERE contentType = 'vod' /* {andWhereClause} */
TIMESERIES AUTO
SINCE 24 hours ago
```

See [Filter placeholders](/docs/streaming-video-&-ads/custom-widget-support/#placeholders) for full details.

## Setting alerts on bitrate metrics [#alerts]

To create an alert when average bitrate drops below an acceptable threshold:

1. Run the following NRQL in the alert condition query field:

   ```sql
   SELECT average(bitrateAvg) FROM VideoAction SINCE 5 minutes ago
   ```

2. Set the threshold to trigger when the value falls below your target (for example, 1500 kbps).
3. Assign the condition to an alert policy.

For full instructions, see [Create NRQL alert conditions](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/).
