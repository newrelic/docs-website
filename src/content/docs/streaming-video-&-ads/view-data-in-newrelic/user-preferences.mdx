When you configure the **Video Overview** or **Ad Overview** dashboards, New Relic automatically saves your settings to your user profile. The next time you return to either dashboard, your configuration is restored so you don't have to start from scratch.

Preferences are saved per user and per dashboard type. Your Video Overview settings never affect your Ad Overview settings, and vice versa.

## How preferences are saved [#how-saving-works]

Preferences are saved **automatically** each time you change a setting — there is no manual save button. Changes are written asynchronously in the background so the dashboard stays responsive while saving.

To reduce unnecessary writes when you make rapid adjustments, saves are briefly batched together (debounced). This means a series of quick changes is treated as a single save operation.

Saved preferences persist across:

- Browser refreshes
- Logout and login
- Different browsers or devices, as long as you are signed in with the same account

## Default behavior [#defaults]

If you have no saved preferences for a dashboard (for example, on your first visit), the dashboard opens with the product default configuration.

The moment you change any setting, that change is saved and becomes your new default for future visits.

## Reset to defaults [#reset-to-defaults]

To clear your saved preferences and return the dashboard to the original product defaults:

1. Open the <DNT>**Video Overview**</DNT> or <DNT>**Ad Overview**</DNT> dashboard.
2. Locate the <DNT>**Reset to default view**</DNT> control in the filters bar or the overflow menu.
3. Click <DNT>**Reset to default view**</DNT>.

The dashboard reloads with the product default configuration. Your saved preferences for that dashboard are cleared. The next change you make starts a new set of saved preferences.

<Callout variant="tip">
  Resetting only affects the dashboard you are currently viewing. Resetting Video Overview preferences does not affect your Ad Overview preferences.
</Callout>

## Loading and saving states [#loading-saving-states]

The dashboard indicates when preferences are being loaded or saved:

- **On load**: Saved preferences are applied before the dashboard fully renders, minimizing visible changes between the default and your personalized view.
- **On save**: Saves happen in the background. The dashboard remains fully usable while a save is in progress.

## Error handling [#error-handling]

If preference retrieval fails when the dashboard loads, the dashboard falls back to the product default configuration and displays an error notification. Your workflow is not blocked.

If a preference save fails, the dashboard continues to function normally. The failure is logged internally. Your in-session configuration is preserved even if it could not be written to your profile.

If a previously saved metric, dimension, or facet is no longer available (for example, because it was deprecated), the dashboard substitutes a reasonable default for that setting, such as the first available option, so no widgets are left empty or broken.
