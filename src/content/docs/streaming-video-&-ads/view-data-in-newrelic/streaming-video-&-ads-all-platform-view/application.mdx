## Overview

Application settings provides centralized configuration for your streaming video & ads monitoring across all platforms. Configure QoE (Quality of Experience) scoring, select which KPIs to display, set quality thresholds, and choose which metric charts appear on your video overview and ad overview pages.

<Callout variant="tip">
  This document covers settings for all platforms. Settings at this level apply across all your streaming entities.

  For entity-specific settings, see [Application settings (Single application)](/docs/streaming-video-&-ads/view-data-in-newrelic/streaming-video-&-ads-single-application-view/application).
</Callout>

### About application settings

Application settings is located in **Settings > Application** and contains four main configuration sections:

1. **QoE health index scores status** - Set status level thresholds (Good/Warning/Critical) that determine color indicators on the QoE health index card
2. **Key performance indicators (KPI) display** - Select which KPI cards appear on video overview and ad overview pages
3. **Key performance indicators (KPIs) threshold** - Define quality thresholds that determine a "Good" vs "Impacted" viewing session
4. **Quality metric charts** - Choose which trend charts appear in the video quality metrics and ad quality metrics sections

### What is QoE score

QoE (Quality of Experience) score provides a single percentage representing overall streaming health across all platforms. A session is "Good" only if it meets ALL quality thresholds; failing any single threshold makes it "Impacted."

Formula: `QoE health index = (Number of Good view sessions / Total number of attempts) Ã— 100`

### Who should read this

- **Administrators**: Configure quality standards, thresholds, and KPI displays that apply across all streaming applications at the account level

## Configure quality thresholds

Define what constitutes a "Good" viewing session. A session must meet ALL thresholds to be classified as "Good." Failing even one threshold makes it "Impacted."

These thresholds apply to the QoE health index calculation for all platforms.

### Video failure threshold

Choose whether a "Good" session can contain failure events:
- Must not contain any failure events (recommended)
- Can contain failure events

### Startup time threshold (seconds)

Maximum acceptable time for the first frame to display.
- Standard range: 2 - 5 seconds
- Recommended: less than 5 seconds

### Rebuffering ratio threshold (%)

Maximum acceptable rebuffering as a percentage of total playback time.
- Standard range: 0.25 - 1.0%
- Recommended: less than 0.4%
- Enter value in decimals (e.g., 0.4 for 0.4%)

### Rebuffering time threshold (seconds)

Maximum acceptable total rebuffering time during a session.
- Standard range: 15 - 30 seconds
- Recommended: less than 2 seconds

## View configured settings

Your selections appear on the video overview page:

- **QoE health index card**: Displays with the color (green/yellow/red) based on your status level thresholds
- **Selected KPI cards**: Only enabled KPIs appear
- **Quality metric charts**: Only selected charts appear in the video quality metrics section

To view your configuration, go to <DNT>**[one.newrelic.com](https://one.newrelic.com) > All Capabilities > Streaming Video & Ads > Video Overview**</DNT>.

## Use cases by role

### As an administrator

Configure quality standards that apply across all streaming applications. Set thresholds based on your content type, audience expectations, and network conditions. Select which KPIs and charts to display based on team needs.

### As a business analyst or executive

Use the QoE health index percentage as a single metric to report on video business health to leadership. Monitor whether the score meets business targets across all platforms.

### As a product manager

Monitor QoE score before and after product launches or app updates across all platforms. Filter by platform or appVersion to see if changes improved or degraded QoE. Use dimensions (like region or ISP) to identify if specific areas are affected.

### As an operations engineer

Monitor QoE health index in real-time during live broadcasts. If score drops suddenly, use dimensions to identify which platforms or regions are impacted. View impacted sessions to see specific failure patterns.

## Data access via NerdGraph API

Access QoE health index and related metrics programmatically via the [NerdGraph API](/docs/apis/nerdgraph). Available metrics include QoE health index percentage, count of "Good" and "Impacted" views, and QoE score filtered by attributes like device type, region, or ISP.
