The New Relic Kubernetes integration supports monitoring Windows nodes in hybrid Kubernetes clusters. This allows you to get comprehensive observability across both Linux and Windows workloads in a single cluster.

## Prerequisites [#prerequisites]

Before you can monitor Windows nodes, ensure your environment meets these requirements:

### Cluster requirements
* **Mixed node cluster**: Your Kubernetes cluster must have at least one Linux node (for the core integration components) and one or more Windows nodes
* **Kubernetes version**: 1.20 or higher
* **Windows Server version**: LTSC 2019 (build 10.0.17763) or LTSC 2022 (build 10.0.20348)
* **Container runtime**: containerd (recommended) or Docker

### Windows node requirements
* **Node access**: Administrative access to configure Windows nodes
* **Network connectivity**: Windows nodes must be able to communicate with New Relic endpoints
* **Resource allocation**: Sufficient CPU and memory for the monitoring components

## Check your Windows version [#windows-version]

<Callout variant="important">
  When using containers in Windows, the container host version and the container image version must be the same. Our Kubernetes integration can run on Windows versions LTSC 2019 and LTSC 2022.
</Callout>

To check your Windows version, open a PowerShell or command prompt window on your Windows node and run:

```shell
systeminfo
```

Look for the lines beginning with `OS Name` and `OS Version`. These lines provide the OS information and build number:
* **Windows Server LTSC 2019**: Corresponds to build number `10.0.17763`
* **Windows Server LTSC 2022**: Corresponds to build number `10.0.20348`

This check is only to confirm that your environment is compatible; you don't need to specify this version information during installation.

## Enable Windows monitoring [#enable-windows]

To enable Windows node monitoring in your New Relic Kubernetes integration:

### Using Helm (recommended)

1. **Update your values file**: Add Windows configuration to your `values-newrelic.yaml` file:

   ```yaml
   global:
     licenseKey: _YOUR_NEW_RELIC_LICENSE_KEY_
     cluster: _K8S_CLUSTER_NAME_

   newrelic-infrastructure:
     enabled: true
     enableWindows: true

   newrelic-logging:
     enabled: true
     enableWindows: true
   ```

2. **Install or upgrade**: Run the Helm upgrade command with Windows support:

   ```shell
   helm upgrade --install newrelic-bundle newrelic/nri-bundle \
   --namespace newrelic --create-namespace \
   -f values-newrelic.yaml \
   --set newrelic-infrastructure.enableWindows=true \
   --set newrelic-logging.enableWindows=true
   ```

### Using the guided installer

1. Go to **[one.newrelic.com](https://one.newrelic.com/) > Integrations & Agents**
2. Click the **Guided install** tile, then select **Kubernetes**
3. During the configuration step, enable **Windows monitoring** in the advanced options
4. Follow the generated installation commands

## What gets deployed [#deployment-components]

When Windows monitoring is enabled, the integration creates separate DaemonSets for Windows nodes:

### Windows-specific DaemonSets

The integration automatically creates Windows DaemonSets based on the Windows versions detected in your cluster:

```shell
kubectl -n newrelic get daemonsets
NAME                                                DESIRED   CURRENT   READY   UP-TO-DATE   AVAILABLE   NODE SELECTOR
newrelic-bundle-nrk8s-kubelet                       2         2         2       2            2           kubernetes.io/os=linux
newrelic-bundle-nrk8s-kubelet-windows-ltsc2019      0         0         0       0            0           kubernetes.io/os=windows,node.kubernetes.io/windows-build=10.0.17763
newrelic-bundle-nrk8s-kubelet-windows-ltsc2022      1         1         1       1            1           kubernetes.io/os=windows,node.kubernetes.io/windows-build=10.0.20348
```

<Callout variant="tip">
  Multiple Windows-specific DaemonSets may appear, but pods are only created for Windows versions that exist in your cluster.
</Callout>

### Components on Windows nodes

Each Windows node runs:
* **Infrastructure Agent**: Collects system metrics, processes, and inventory data
* **Kubelet integration**: Gathers container and pod metrics from the Windows kubelet
* **Fluent Bit** (if logging is enabled): Forwards container logs to New Relic

## Advanced configuration [#advanced-config]

### Optimize DaemonSet creation

If you want to prevent unnecessary DaemonSets for Windows versions not in your cluster, specify only the versions you need:

```yaml
windowsOsList:
  - version: ltsc2022                      # Human-readable version identifier
    imageTagSuffix: ltsc2022               # Tag to be used for nodes running this version
    buildNumber: 10.0.20348                # Build number used as a node selector
```

### Configure resource limits

Set appropriate resource limits for Windows monitoring components:

```yaml
newrelic-infrastructure:
  enableWindows: true
  windows:
    resources:
      limits:
        memory: 512Mi
        cpu: 500m
      requests:
        memory: 256Mi
        cpu: 100m
```

### Custom Windows configuration

You can customize Windows-specific settings:

```yaml
newrelic-infrastructure:
  enableWindows: true
  windows:
    # Enable debug logging for troubleshooting
    debug: false
    
    # Custom configuration file path
    config:
      custom_attributes:
        environment: "production"
        team: "platform"
```

## Verify Windows monitoring [#verify]

After installation, verify that Windows monitoring is working:

### Check pod status

1. **View Windows pods**:
   ```shell
   kubectl -n newrelic get pods -l app.kubernetes.io/name=newrelic-infrastructure -o wide
   ```

2. **Check logs for Windows pods**:
   ```shell
   kubectl -n newrelic logs -l app.kubernetes.io/name=newrelic-infrastructure --tail=100
   ```

### Verify data in New Relic

1. **Go to the Kubernetes cluster explorer**: Navigate to **[one.newrelic.com](https://one.newrelic.com/) > Kubernetes**
2. **Select your cluster**: Find your cluster in the list
3. **Check Windows nodes**: Verify that Windows nodes appear in the cluster view with metrics
4. **View node details**: Click on a Windows node to see detailed metrics and processes

## Troubleshooting [#troubleshooting]

### Common issues

**Windows pods not starting**
* Check that Windows nodes have sufficient resources
* Verify the Windows version matches supported versions
* Ensure container runtime is properly configured

**Missing Windows metrics**
* Confirm that the Windows DaemonSet is running on Windows nodes
* Check the Infrastructure Agent logs for errors
* Verify network connectivity to New Relic endpoints

**HostProcess security errors**
* Ensure your cluster allows HostProcess containers
* Check that appropriate security policies are configured
* Review Windows node security settings

For more detailed troubleshooting information, see [Troubleshooting Windows integration](/docs/kubernetes-pixie/kubernetes-integration/troubleshooting/troubleshooting-windows).
