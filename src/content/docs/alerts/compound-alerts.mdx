As a New Relic user, you may receive individual alert events from multiple related conditions, creating alert noise and making it difficult to identify truly critical issues. While New Relic's existing correlation feature helps reduce this noise, you may need more precise control over when alerts trigger based on specific condition combinations.

Compound alerts let you define custom alert logic by combining multiple alert conditions using logical operators (AND, OR, NOT). You can create targeted alerts that trigger only when specific combinations of conditions are met, or when one condition triggers but another does not, giving you fewer, more relevant notifications with the full context you need.


## What you can do with compound alerts [#features]

Compound alerts let you define sophisticated alert logic by combining existing alert conditions. You can use compound alerts to:

* **Reduce alert noise**: Receive a single compound alert event instead of separate alerts from multiple related conditions. This reduces the overall volume of notifications.
* **Combine up to five alert conditions**: Select up to five existing alert conditions from your account. Combine them using AND, OR, and NOT operators with parentheses for complex logic.
* **Combine cross-account conditions**: Include alert conditions that target data from different accounts. All component conditions must have the same owning account, but they can query data from other accounts.
* **Control facet matching**: Choose how to handle shared facets across conditions. The compound alert can trigger when the same facet value meets the threshold across all conditions, or when any facet value meets the criteria.

## Use cases [#use-cases]

Here are some use cases in which you might use compound alerts:

* **Detect memory leaks**: Alert when memory > 50% (infrastructure) AND throughput &lt; 1k (APM), which helps identify situations where memory increases without corresponding traffic.
* **Database performance issues**: Alert when database response time > 2s AND (CPU usage > 80% OR connection pool > 90%) AND NOT during maintenance window, helping identify performance degradation while avoiding noise during planned maintenance.

## Permissions and access control [#permissions]

Compound alerts use the same permission model as standard alert conditions:

* Any user with permission to create alert conditions in an account can create compound alert conditions in that account.
* All users with access to the account can view and manage compound alert conditions in that account.

## Create a compound alert condition [#create-compound-alert]

To create a compound alert condition:

<Steps>
  <Step>
    To access the compound alerts page, from the New Relic platform, go to **Alerts** and click **Compound Alerts** from the left navigation menu.
  </Step>

  <Step>
    All component conditions must be from the same account. From the dropdown, select the account.

    <Callout variant="tip">
    If you would like to include an alert condition that targets data from another account, consider setting up a cross-account alert condition in the same owning account you're configuring the compound alert in.
    </Callout>
  </Step>

  <Step>
    Click **Create a compound alert**.
  </Step>

  <Step>
    Enter a name for the compound alert.
  </Step>

  <Step>
    To select the alert conditions you want to combine, click **Add conditions**, choose up to five conditions, and add them. Selected conditions are scoped to the account you choose and are automatically assigned variables (a, b, c, d, e) for use in the trigger expression. You can only select standard alert conditions, not other compound alert conditions.

    <Callout variant="tip">
    Each condition may have multiple thresholds like warning, critical, or loss of signal. When any threshold of a condition is met, that condition becomes eligible to trigger the compound alert notification.
    </Callout>

  </Step>

    <Step>
    If your conditions have shared facets, to control facet matching behavior, select the checkbox so the alert triggers only when the same facet value meets thresholds across all conditions, or leave it unchecked so the alert triggers regardless of matching facet values.

    <Callout variant="tip">
    When a condition is faceted by multiple attributes, a match is considered when at least one attribute is the same across conditions.
    </Callout>
  </Step>

  <Step>
    To define how conditions should work together, in the **Tell us how you want these conditions to work together** field, write a trigger expression using the assigned variables and logical operators (AND, OR, NOT). Use parentheses to group expressions for complex logic.

    Examples: `a AND b`, `a OR b OR c`, `a AND (b OR c)`, `a AND NOT b`, `a AND (b OR c) AND NOT d`
  </Step>



  <Step>
    To specify how long the trigger expression must evaluate to true before opening an alert event, set the **Threshold duration**.
  </Step>

  <Step>
    To organize your alert events, from the **Policy** dropdown, select a policy. Learn more about [alert policies](https://docs.newrelic.com/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/).
  </Step>

  <Step>
    (Optional) To link to your incident-handling procedure, add a **Runbook URL**. This URL will be included in notifications.
  </Step>

  <Step>
    To enable the compound alert condition immediately, toggle **Enable on save**.
  </Step>

  <Step>
    Review your configuration and save it. The condition appears on the **Compound Alerts** page.
  </Step>

</Steps>

## Manage compound alert conditions [#manage-compound-alerts]

To view all compound alert conditions, go to **Alerts** > **Compound Alerts**. Click any compound alert name to edit its configuration. Changes to component conditions automatically reflect in the compound alert. To delete, click <Icon name="fe-more-horizontal"/> > **Delete**. Deleting a compound alert does not delete the individual conditions that were combined in it—those conditions remain active individually.

## Understand compound alert events [#alert-events]

When a compound alert condition's trigger expression evaluates to true, New Relic opens a compound alert event. The event includes a link to your runbook URL. To view specific threshold breaches for each component condition, click through to the individual alert events.

### Alert events and issue correlation

When working with compound alerts, it's important to understand how alert events are created and correlated:

* **Multiple alert events**: A compound condition creates its own compound alert event. However, each individual component condition also creates its own separate alert events when their thresholds are breached.
* **Correlating to the same issue**: To ensure all related alert events (both compound and component) are grouped into the same issue, configure your component conditions to use the same policy as your compound alert condition. This allows New Relic's issue correlation to group them together.
* **Viewing related events**: From a compound alert event, you can navigate to the individual component alert events to see specific threshold breaches and detailed metrics for each condition.


## Limitations [#limitations]

When you use compound alerts, keep these limitations in mind:

* **Maximum conditions**: You can combine up to five conditions in a single compound alert.
* **No nesting**: You can't use a compound alert condition as a component condition in another compound alert.
* **Account scope**: All component conditions must be from the same owning account (though they can target data from other accounts via cross-account conditions).
* **No custom tags**: Compound alert conditions don't support adding custom tags.
* **Facet matching**: When conditions are faceted by multiple attributes, the matching logic considers a match when at least one attribute is the same (granular control is not currently supported).
