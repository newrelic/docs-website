To create a compound alert condition:

<Steps>
  <Step>
    Go to <DNT>**Alerts**</DNT> and click <DNT>**Compound Alerts**</DNT> in the left navigation menu.
  </Step>

  <Step>
    All component conditions must be from the same owning account. From the dropdown, select the account.

    <Callout variant="tip">
    To include a condition that targets data from another account, set up a cross-account alert condition in the same owning account as the compound alert.
    </Callout>
  </Step>

  <Step>
    Click <DNT>**Create a compound alert**</DNT> and enter a name for the compound alert.
  </Step>

  <Step>
    Click <DNT>**Add conditions**</DNT> and choose up to five conditions. Each condition is automatically assigned a variable <DNT>(a, b, c, d, e)</DNT> for use in the trigger expression. You can only select standard alert conditions, not other compound alert conditions. See [compound alert limitations](/docs/alerts/create-alert/create-alert-condition/compound-alerts/compound-alerts#limitations) for all restrictions.


    <img
      title="Condition selection in compound alert builder"
      alt="A screenshot dispaying the condition selection process in the compound alert builder"
      src="/images/select-2-5-alerts.webp"
    />

    <Callout variant="tip">
    Each condition may have multiple thresholds like warning, critical, or loss of signal. When any threshold of a condition is met, that condition is treated as triggered in the compound expression.
    </Callout>
  </Step>

  <Step>
    If your conditions share facets, check the checkbox to trigger only when the same facet value meets thresholds across all conditions. Leave it unchecked to trigger regardless of matching facet values.

    <Callout variant="tip">
    When a condition is faceted by multiple attributes, a match occurs when at least one attribute is the same across conditions.
    </Callout>
  </Step>

  <Step>
    To define how conditions should work together, in the **Tell us how you want these conditions to work together** field, write a trigger expression using the assigned variables and logical operators <DNT>(AND, OR, NOT)</DNT>. Use parentheses to group expressions for complex logic.

    Examples: <DNT>`a AND b`, `a OR b OR c`, `a AND (b OR c)`, `a AND NOT b`, `a AND (b OR c) AND NOT d`</DNT>
  </Step>

  <Step>
    To specify how long the trigger expression must evaluate to true before opening an alert event, set the **Threshold duration**.
  </Step>

  <Step>
    To organize your alert events, from the <DNT>**Policy**</DNT> dropdown, select a policy. Learn more about [alert policies](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/).

    <Callout variant="tip">
    To group compound and component alert events into the same issue, configure your component conditions to use the same policy you select here.
    </Callout>
  </Step>

  <Step>
    (Optional) To link to your incident-handling procedure, add a <DNT>**Runbook URL**</DNT>. This URL will be included in notifications.
  </Step>

  <Step>
    To enable the compound alert condition immediately, toggle <DNT>**Enable on save**</DNT>.
  </Step>

  <Step>
    Review your configuration and save it. The condition appears on the <DNT>**Compound Alerts**</DNT> page.
  </Step>
</Steps>

## Manage compound alert conditions [#manage-compound-alerts]

To view all compound alert conditions, go to <DNT>**Alerts** > **Compound Alerts**</DNT>.

* <DNT>**Edit**</DNT>: Click any compound alert name to edit its configuration. Changes to component conditions automatically reflect in the compound alert.
* <DNT>**Delete**</DNT>: Click <Icon name="fe-more-horizontal"/> > <DNT>**Delete**</DNT>. Deleting a compound alert does not delete the individual component conditions — those remain active individually.
