Here us a quick look at the contributing changes that went into effect after our image refactor.

## Adding images to the repo:
<SideBySide>
    <Side>
        **Old image convention:**

        images used to be in `./src/images`

        ```
        ├──src/
        │  └──content/
        │  │  └──docs/
        │  │    └── example-page.mdx
        │  └──images/
        │     └── example-image.webp
        ```
    </Side>
    <Side>
        **New image convention:**

        images are now in `./static/images`

        ```
        ├──src/
        │  └──content/
        │     └──docs/
        │        └── example-page.mdx
        ├──static/
        │  └──images/
        │     └── example-image.webp
        ```

    </Side>
</SideBySide>

## Using the image in an MDX file

<SideBySide>

    <Side>
        **Old image convention:**

        Previously, we had to import our image paths as a variable then use that variable as the `<img>` source. 

        ```js

            import exampleImage from 'images/example-image.webp'

            <img
                title="Image title."
                alt="Alternate text."
                src={exampleImage}
            />

        ```
    </Side>
    <Side>
        **New image convention:**

        Now the image source is simply the path to the image relative to the static folder:

        ```html
            <img
                title="Image title."
                alt="Alternate text."
                src="/images/example-image.webp"
            />
        ```

    </Side>
</SideBySide>

## Using the image in a MD file

Using images in markdown is almost the same as before. We just changed the path used slightly:

<SideBySide>

    <Side>
        **Old image convention:**

        `./images`
        

        ```md
            ![Image title.](./images/kinesis_data_firehose.webp "Alternate text.")
        ```
    </Side>
    <Side>
        **New image convention:**

        `/images`

        ```md
            ![Image title.](/images/kinesis_data_firehose.webp "Alternate text.")
        ```

    </Side>
</SideBySide>
