<Callout title="Feature Availability" variant="important">
  Fleet Control for Kubernetes clusters is generally available (GA). Support for managing agents on Linux and Windows hosts is currently in public preview.

  For a complete list of supported agents and their environments, see our [agent type compatibility documentation](/docs/new-relic-control/agent-control/agent-types).

  The public preview feature is provided pursuant to our [pre-release policies](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

You can use [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/) to programmatically manage Fleet Control fleets, members, and deployments. These mutations provide complete control over fleet operations, enabling you to automate fleet management, integrate with CI/CD pipelines, and manage instrumentation at scale.

This tutorial provides examples of the available Fleet Control mutations and queries. You can use these to create fleets, manage fleet members, and deploy agent configurations.

## Prerequisites

* A [New Relic account](https://newrelic.com/signup) with access to your API user key
* The **Organization Manager** role or a custom role with fleet management permissions
* Familiarity with [GraphQL](https://graphql.org/learn/) and [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/)

<Callout variant="tip">
You can also use the [Fleet Control CLI](https://github.com/newrelic/newrelic-cli/tree/main/internal/fleetcontrol) for command-line fleet management. The CLI is built on the same NerdGraph API.
</Callout>

<Callout variant="important">
Agent configurations are managed through the [Blob Storage API](/docs/new-relic-control/fleet-control/blob-storage-api-reference), not NerdGraph. All configuration operations use a separate REST API.
</Callout>

## Fleet management mutations

<CollapserGroup>

<Collapser
  id="create-fleet"
  title="Create a fleet">

Use the `fleetControlCreateFleet` mutation to create a new fleet for managing groups of Kubernetes clusters or hosts.

### Input parameters

<table>
  <thead>
    <tr>
      <th>Parameter</th>
      <th>Data Type</th>
      <th>Is it Required?</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`fleetEntity`</td>
      <td>Object</td>
      <td>Yes</td>
      <td>The fleet entity creation input containing fleet configuration.</td>
    </tr>
    <tr>
      <td>`fleetEntity.name`</td>
      <td>String</td>
      <td>Yes</td>
      <td>The fleet name. Must be unique within your organization.</td>
    </tr>
    <tr>
      <td>`fleetEntity.managedEntityType`</td>
      <td>Enum</td>
      <td>Yes</td>
      <td>The type of entities this fleet will manage. Options: `HOST` (for Linux/Windows hosts) or `KUBERNETESCLUSTER`.</td>
    </tr>
    <tr>
      <td>`fleetEntity.scope`</td>
      <td>Object</td>
      <td>Yes</td>
      <td>The entity scope defining whether the fleet is account-scoped or organization-scoped.</td>
    </tr>
    <tr>
      <td>`fleetEntity.scope.type`</td>
      <td>Enum</td>
      <td>Yes</td>
      <td>Scope type. Options: `ACCOUNT` or `ORGANIZATION`.</td>
    </tr>
    <tr>
      <td>`fleetEntity.scope.id`</td>
      <td>ID</td>
      <td>Yes</td>
      <td>The account ID or organization ID, depending on scope type.</td>
    </tr>
    <tr>
      <td>`fleetEntity.description`</td>
      <td>String</td>
      <td>No</td>
      <td>A description of the fleet's purpose.</td>
    </tr>
    <tr>
      <td>`fleetEntity.operatingSystem`</td>
      <td>Object</td>
      <td>No</td>
      <td>Operating system type for HOST fleets. Required if `managedEntityType` is `HOST`.</td>
    </tr>
    <tr>
      <td>`fleetEntity.operatingSystem.type`</td>
      <td>Enum</td>
      <td>Yes (if operatingSystem provided)</td>
      <td>OS type. Options: `LINUX` or `WINDOWS`.</td>
    </tr>
    <tr>
      <td>`fleetEntity.tags`</td>
      <td>Array</td>
      <td>No</td>
      <td>List of tag objects for categorizing the fleet.</td>
    </tr>
  </tbody>
</table>

### Sample request

```graphql
mutation CreateFleet {
  fleetControlCreateFleet(
    fleetEntity: {
      name: "Production Web Servers"
      description: "Fleet for all production Linux web server hosts"
      managedEntityType: HOST
      operatingSystem: {
        type: LINUX
      }
      scope: {
        type: ACCOUNT
        id: "YOUR_ACCOUNT_ID"
      }
      tags: [
        {
          key: "environment"
          values: ["production"]
        }
        {
          key: "team"
          values: ["platform"]
        }
      ]
    }
  ) {
    entity {
      id
      name
      description
      managedEntityType
    }
  }
}
```

### Sample response

```json
{
  "data": {
    "fleetControlCreateFleet": {
      "entity": {
        "id": "MTIzNDU2fEZMRUVUfEZMRUVUfDEyMzQ1Njc4OQ",
        "name": "Production Web Servers",
        "description": "Fleet for all production Linux web server hosts",
        "managedEntityType": "HOST"
      }
    }
  }
}
```

## Fleet member management mutations

<CollapserGroup>

<Collapser
  id="add-members"
  title="Add members to a fleet">

Use the `fleetControlAddFleetMembers` mutation to add managed entities to fleet rings. Rings allow you to organize entities within a fleet for staged rollouts (for example, canary, staging, production).

### Input parameters

<table>
  <thead>
    <tr>
      <th>Parameter</th>
      <th>Data Type</th>
      <th>Is it Required?</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`fleetId`</td>
      <td>ID</td>
      <td>Yes</td>
      <td>The fleet entity ID.</td>
    </tr>
    <tr>
      <td>`members`</td>
      <td>Array</td>
      <td>Yes</td>
      <td>List of ring inputs specifying which entities to add to which rings.</td>
    </tr>
    <tr>
      <td>`members[].ring`</td>
      <td>String</td>
      <td>Yes</td>
      <td>The ring name (for example, "canary", "production").</td>
    </tr>
    <tr>
      <td>`members[].entityIds`</td>
      <td>Array</td>
      <td>Yes</td>
      <td>List of entity GUIDs to add to the ring.</td>
    </tr>
  </tbody>
</table>

### Sample request

```graphql
mutation AddFleetMembers {
  fleetControlAddFleetMembers(
    fleetId: "MTIzNDU2fEZMRUVUfEZMRUVUfDEyMzQ1Njc4OQ"
    members: [
      {
        ring: "canary"
        entityIds: [
          "MTIzNDU2fElORlJBfE5BfDEyMzQ1Njc4OQ",
          "MTIzNDU2fElORlJBfE5BfDk4NzY1NDMyMQ"
        ]
      }
      {
        ring: "production"
        entityIds: [
          "MTIzNDU2fElORlJBfE5BfDExMTExMTExMQ"
        ]
      }
    ]
  ) {
    items {
      id
      name
      ring
    }
  }
}
```

<Collapser
  id="create-deployment"
  title="Create a deployment">

Use the `fleetControlCreateFleetDeployment` mutation to create a deployment that defines which agent versions and configurations to deploy to a fleet.

### Input parameters

<table>
  <thead>
    <tr>
      <th>Parameter</th>
      <th>Data Type</th>
      <th>Is it Required?</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`fleetDeployment`</td>
      <td>Object</td>
      <td>Yes</td>
      <td>The deployment configuration.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.fleetId`</td>
      <td>ID</td>
      <td>Yes</td>
      <td>The fleet entity ID.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.scope`</td>
      <td>Object</td>
      <td>Yes</td>
      <td>The entity scope (same as fleet scope).</td>
    </tr>
    <tr>
      <td>`fleetDeployment.agents`</td>
      <td>Array</td>
      <td>Yes</td>
      <td>List of agent configurations to deploy.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.agents[].agentType`</td>
      <td>String</td>
      <td>Yes</td>
      <td>Agent type. Options: `NRInfra`, `NRDOT`, `FluentBit`, `NRPrometheusAgent`, `PipelineControlGateway`, `NRApmOperator`, `NReBPFAgent`.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.agents[].version`</td>
      <td>String</td>
      <td>Yes</td>
      <td>Agent version to deploy.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.agents[].configurationVersionList`</td>
      <td>Array</td>
      <td>Yes</td>
      <td>List of configuration version IDs for this agent.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.name`</td>
      <td>String</td>
      <td>No</td>
      <td>Deployment name.</td>
    </tr>
    <tr>
      <td>`fleetDeployment.description`</td>
      <td>String</td>
      <td>No</td>
      <td>Deployment description.</td>
    </tr>
  </tbody>
</table>

### Sample request

```graphql
mutation CreateDeployment {
  fleetControlCreateFleetDeployment(
    fleetDeployment: {
      fleetId: "MTIzNDU2fEZMRUVUfEZMRUVUfDEyMzQ1Njc4OQ"
      name: "Infrastructure Agent v1.50.0"
      description: "Deploy infra agent 1.50.0 with updated config"
      scope: {
        type: ACCOUNT
        id: "YOUR_ACCOUNT_ID"
      }
      agents: [
        {
          agentType: "NRInfra"
          version: "1.50.0"
          configurationVersionList: [
            { id: "MTIzNDU2fEFHRU5UQ09ORklHfDEyMzQ1" }
          ]
        }
      ]
    }
  ) {
    entity {
      id
      name
      description
    }
  }
}
```

<Collapser
  id="trigger-deployment"
  title="Trigger a deployment">

Use the `fleetControlDeploy` mutation to trigger a deployment to roll out to specific fleet rings.

### Input parameters

<table>
  <thead>
    <tr>
      <th>Parameter</th>
      <th>Data Type</th>
      <th>Is it Required?</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`id`</td>
      <td>ID</td>
      <td>Yes</td>
      <td>The deployment ID to trigger.</td>
    </tr>
    <tr>
      <td>`policy`</td>
      <td>Object</td>
      <td>Yes</td>
      <td>Deployment policy specifying which rings to deploy to.</td>
    </tr>
    <tr>
      <td>`policy.ringDeploymentPolicy`</td>
      <td>Object</td>
      <td>Yes</td>
      <td>Ring deployment policy.</td>
    </tr>
    <tr>
      <td>`policy.ringDeploymentPolicy.ringsToDeploy`</td>
      <td>Array</td>
      <td>Yes</td>
      <td>List of ring names to deploy to.</td>
    </tr>
  </tbody>
</table>

### Sample request

```graphql
mutation TriggerDeployment {
  fleetControlDeploy(
    id: "MTIzNDU2fERFUExPWU1FTlR8MTIzNDU"
    policy: {
      ringDeploymentPolicy: {
        ringsToDeploy: ["canary"]
      }
    }
  ) {
    fleetId
  }
}
```
