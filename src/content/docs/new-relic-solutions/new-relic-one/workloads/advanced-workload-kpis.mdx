Custom KPIs let you define and monitor business-specific metrics at the workload level, connecting technical performance to business outcomes.

## What are workload KPIs? [#definition]

Workload KPIs are custom metrics you define using NRQL queries that evaluate data across all entities in your workload. They help you:

* Track business metrics (revenue, conversions, user actions)
* Monitor aggregated performance (average response time across services)
* Measure end-to-end metrics (total transaction duration)
* Alert on workload-level thresholds

## Add a KPI [#add-kpi]

1. Open your advanced workload
2. Go to <DNT>**KPIs**</DNT> or <DNT>**Monitor**</DNT> section
3. Click <DNT>**Add KPI**</DNT>
4. Configure:
   * **Name**: Descriptive KPI name
   * **NRQL query**: Query that calculates the metric
   * **Target value**: Optional performance target
   * **Unit**: Display unit (%, ms, count, etc.)
5. Click <DNT>**Save**</DNT>

### Example KPIs

**Average checkout duration:**
```sql
SELECT average(duration)
FROM Transaction
WHERE appName IN ({{workloadEntityNames}})
AND name LIKE '%checkout%'
```

**Total error rate:**
```sql
SELECT percentage(count(*), WHERE error IS true)
FROM Transaction
WHERE appName IN ({{workloadEntityNames}})
```

**Business transactions per minute:**
```sql
SELECT rate(count(*), 1 minute)
FROM Transaction
WHERE appName IN ({{workloadEntityNames}})
AND transactionType = 'Web'
```

<Callout variant="tip">
  Use `{{workloadEntityNames}}` to automatically scope queries to current workload entities.
</Callout>

## View and monitor KPIs [#monitor]

Your KPIs appear on the workload summary page with:

* Current value
* Trend indicator
* Comparison to target (if set)
* Time series chart

KPIs update in real-time as workload membership changes.

## Create KPI alerts [#alerts]

To alert on a KPI:

1. Click the KPI
2. Select <DNT>**Create alert**</DNT>
3. Configure:
   * Threshold (above/below target)
   * Duration before triggering
   * Notification channels
4. Save the alert condition

## Edit or delete KPIs [#manage]

1. Go to your workload's KPI section
2. Click <Icon name="fe-more-horizontal"/> next to the KPI
3. Select <DNT>**Edit**</DNT> or <DNT>**Delete**</DNT>

## Best practices [#best-practices]

* **Start with 3-5 KPIs**: Focus on most critical metrics
* **Use meaningful names**: Make KPIs self-explanatory
* **Set realistic targets**: Base targets on historical data
* **Review regularly**: Update KPIs as business goals evolve
* **Test queries**: Verify NRQL returns expected results before saving

## Limitations [#limitations]

* Maximum KPIs per workload: 10 (preview limit)
* Query timeout: 60 seconds
* KPIs scope to current workload entities only
* Historical KPI data retained for 30 days
