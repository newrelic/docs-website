To use the following command line options with the Diagnostics CLI:

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Option
      </th>

      <th>
        Usage
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `-a`

        `-attach`
      </td>

      <td>
        Attach for automatic upload to a New Relic account. Max upload size is 4 GB. This uses a validated <InlinePopover type="licenseKey"/> from your environment.
      </td>
    </tr>

    <tr>
      <td>
        `-api-key`
      </td>

      <td>
        User API key to use when uploading results to your New relic account. Max upload size is 4 GB. This bypasses using a license key and uses your <InlinePopover type="userKey"/> to attach results instead.
      </td>
    </tr>

    <tr>
      <td>
        `-r`

        `-region`
      </td>

      <td>
        The [region](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) your New Relic account is hosted in. Accepted values: EU or US. Case insensitive. Default: US
      </td>
    </tr>

    <tr>
      <td id="browser-check">
        `-browser-url STRING`
      </td>

      <td>
        Diagnostics CLI version 1.1.9 or higher

        When invoked, this will only run diagnostics checks related to <InlinePopover type="browser"/>. This command checks that New Relic's browser monitoring agent is present and returns the agent version, the injection method ([via <InlinePopover type="apm"/>](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app) or [via copy/paste](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app)), and the loader type (Pro, Lite, SPA). To be used to provide detail to New Relic Support when troubleshooting intranet sites.
      </td>
    </tr>

    <tr>
      <td>
        `-c STRING`

        `-config-file STRING`
      </td>

      <td>
        Override default agent configuration file location. Can be used to specify either a folder to search in addition to the default folders, or a path to a specific configuration file.
      </td>
    </tr>

    <tr>
      <td>
        `-filter STRING`
      </td>

      <td>
        Filter task results by result status. Accepts a comma separated list. Accepts `Success`, `Warning`, `Failure`, `Error`, `None`, or `Info`.
        Example syntax:

        ```
        "Success,Warning,Failure"
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-h`

        `-help`
      </td>

      <td>
        Display complete list of command line options. To list all tasks to be run, use `-h tasks`.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-include STRING`
      </td>

      <td>
        Include a file or directory (including subdirectories) in the `nrdiag-output.zip`. Limit 4GB. Use the `-a` or `-api-key` flag to upload the results to New Relic.

        Example syntax:

        ```
        -include ./path/to/file
        ```

        ```
        -include \some\path\
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-interactive`
      </td>

      <td>
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-output-path STRING`
      </td>

      <td>
        Specifies a different output directory to write the results for `nrdiag-output.zip`, `nrdiag-output.json`, and `nrdiag-filelist.txt`. Default location is `./`.
      </td>
    </tr>

    <tr>
      <td>
        `-o`

        `-override STRING`
      </td>

      <td>
        Pass in arguments to override when requested by New Relic Support. Format: `identifier.property=value`.

        Example syntax:

        ```
        -override Java/Config/Agent.Status=Success
        ```

        ```
        -o Base/Config/Validate.agentlanguage=PHP
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-p STRING`

        `-proxy STRING`
      </td>

      <td>
        Provide proxy to be used in HTTP connection tasks. Can be HTTP or HTTPS. Proxy should be in the format `http(s)://proxyIp:proxy`. If the Diagnostics CLI finds a proxy in the agent config file, it will use that proxy by default.
        In most cases `port` is not needed. If the Diagnostics CLI finds a proxy in the agent config file, it will use that proxy by default.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-pw STRING`
      </td>

      <td>
        Proxy password, if necessary. If the Diagnostics CLI finds a proxy in the agent config file, it will use that proxy by default.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-user STRING`
      </td>

      <td>
        Proxy username, if necessary. If the Diagnostics CLI finds a proxy in the agent config file, it will use that proxy by default.
      </td>
    </tr>

    <tr>
      <td>
        `-q`
      </td>

      <td>
        Quiet output only prints the high-level results and not the explanatory output. Suppresses file addition warnings if `-y` is also used.
        Does not contradict `-v`. Inclusion filters are ignored.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-qq`
      </td>

      <td>
        Very quiet output only prints a single summary line for output (implies `q`). Suppresses file addition warnings if `-y` is also used.
        Does not contradict `-v`. Inclusion filters are ignored.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-skip-version-check`
      </td>

      <td>
        Skip the automatic check for a newer version of the application.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-s STRING`

        `-suites STRING`
      </td>

      <td>
        Run a [suite](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag/#suites), a collection of tasks that target specific products or issues. To specify multiple suites, separate them with commas. To get a list of all suites, run:

        ```
        ./nrdiag -h suites
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-t STRING`

        `-tasks STRING`
      </td>

      <td>
        Run only a subset of tasks, either by agent or by task type. To specify multiple tasks, separate them with commas and/or with a wildcard `*`. For a list of all tasks, run:

        ```
        ./nrdiag -h tasks
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-usage-opt-out`
      </td>

      <td>
        Decline to send anonymous Diagnostics CLI tool usage data to New Relic for this run.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-list-scripts`
      </td>

      <td>
        List available scripts.
      </td>
    </tr>

    <tr>
      <td>
        `-script STRING`
      </td>

      <td>
        View the specified script. Use with `-run` to run the script.
      </td>
    </tr>

    <tr>
      <td>
        `-run`
      </td>

      <td>
        Use with `-script` to run the script.
      </td>
    </tr>

    <tr>
      <td>
        `-script-flags`
      </td>

      <td>
        Use with `-run -script` to pass command line flags to the script.
      </td>
    </tr>

    <tr>
      <td>
        `-v`

        `-verbose`
      </td>

      <td>
        Display verbose logging during check execution. Disabled by default.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-version`
      </td>

      <td>
        Displays current Diagnostics CLI version. This also prompts to check for a newer version and prompts to download if a newer version is available.
        Takes precedence over `-skip-version-check`.
        Type: bool
      </td>
    </tr>

    <tr>
      <td>
        `-y`

        `-yes`
      </td>

      <td>
        Respond `yes` to any prompt that comes up while running. Disabled by default.
        Type: bool
      </td>
    </tr>
  </tbody>
</table>
