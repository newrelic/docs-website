### Usage

```js
import { Icon } from 'nr1'
```

### Examples

```js
<Icon type={Icon.TYPE.HARDWARE_AND_SOFTWARE__SOFTWARE__PLUGIN__S_OK} />;
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `className` <h5>string</h5>
      </td>

      <td>
        Appends class names to the component. Should be used only for positioning and spacing purposes.
      </td>
    </tr>

    <tr>
      <td>
        `color` <h5>string</h5>
      </td>

      <td>
        Color of the icon. By default it gets the current value of the css `color` property of the element.
      </td>
    </tr>

    <tr>
      <td>
        `spacingType` <h5>enum</h5>
      </td>

      <td>
        Spacing property. Spacing is defined as a tuple of zero to four values, which follow the same conventions as CSS properties lie `margin` or `padding`. To omit a value, use `SPACING_TYPE.OMIT`.

        <OptionReference array>
          Icon.SPACING_TYPE.EXTRA_LARGE,
          Icon.SPACING_TYPE.LARGE,
          Icon.SPACING_TYPE.MEDIUM,
          Icon.SPACING_TYPE.NONE,
          Icon.SPACING_TYPE.OMIT,
          Icon.SPACING_TYPE.SMALL,
        </OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>object</h5>
      </td>

      <td>
        Inline style for custom styling. Should be used only for position and spacing purposes.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>string</h5>
      </td>

      <td>
        Adds a `data-test-id` attribute. Use it to target the component in unit and E2E tests. For a test id to be valid, prefix it with your nerdpack id, followed by a dot. For example, `my-nerdpack.some-element`. **Note:** You might not see `data-test-id` attributes as they are removed from the DOM, to debug them pass a `e2e-test` query parameter to the URL.
      </td>
    </tr>
