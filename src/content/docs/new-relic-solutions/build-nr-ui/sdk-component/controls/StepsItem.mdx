Child element of the `<Steps>` component.

### Usage

```js
import { StepsItem } from 'nr1'
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `ariaLabel` <h5>string</h5>
      </td>

      <td>
        Provide a label for screen readers to describe the step. This is only required if the screen reader label should be different to the `label` prop.
      </td>
    </tr>

    <tr>
      <td>
        `checked` <h5>boolean</h5>
      </td>

      <td>
        If `true`, marks the step as completed.
      </td>
    </tr>

    <tr>
      <td>
        `children` <h5>REQUIRED</h5> <h5>node</h5>
      </td>

      <td>
        Content to render inside the item.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>string</h5>
      </td>

      <td>
        Appends class names to the component.Should be used only for positioning and spacing purposes.
      </td>
    </tr>

    <tr>
      <td>
        `disabled` <h5>boolean</h5>
      </td>

      <td>
        Use disabled to prevent users from opening the step by clicking on its label.
      </td>
    </tr>

    <tr>
      <td>
        `expanded` <h5>boolean</h5>
      </td>

      <td>
        When true, it will show the content of the step.
      </td>
    </tr>

    <tr>
      <td>
        `label` <h5>REQUIRED</h5><h5>string</h5>
      </td>

      <td>
        Label of the step.
      </td>
    </tr>

    <tr>
      <td>
        `onClick` <h5>function</h5>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>object</h5>
      </td>

      <td>
        Inline style for custom styling.Should be used only for positioning and spacing purposes.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>string</h5>
      </td>

      <td>
        Adds a `data-test-id` attribute. Use it to target the component in unit and E2E tests.For a test id to be valid, prefix it with your nerdpack id, followed up by a dot.For example, `my-nerdpack.some-element`.
        **Note:** You might not see `data-test-id` attributes as they are removed from the DOM, to debug them pass a `e2e-test` query parameter to the URL.
      </td>
    </tr>

    <tr>
      <td>
        `value` <h5>any</h5>
      </td>

      <td>
        Used by the `<Steps>` to identify the active step.
      </td>
    </tr>
