## nr1 autocomplete [#nr1-autocomplete]

By default, the command displays the autocomplete instructions for `zsh`. If you want instructions for bash, run `nr1 autocomplete bash`.

### Usage

`$ nr1 autocomplete`

### Arguments

<table>
  <tbody>
    <tr>
      <td>
        `SHELL`
      </td>

      <td>
        The shell type you want instructions for.
      </td>
    </tr>
  </tbody>
</table>

### Options

<table>
  <tbody>
    <tr>
      <td>
        `-r, --refresh-cache`
      </td>

      <td>
        Refreshes cache and ignores displaying instructions.
      </td>
    </tr>
  </tbody>
</table>

### Examples

* `$ nr1 autocomplete`
* `$ nr1 autocomplete zsh`
* `$ nr1 autocomplete bash`
* `$ nr1 autocomplete --refresh-cache`

## nr1 create [#nr1-create]

Creates a new component from our template, either a Nerdpack, Nerdlet, launcher, or catalog. The CLI will walk you through this process.

To learn more about Nerdpacks and their file structure, see [Nerdpack file structure](/docs/new-relic-solutions/build-nr-ui/nerdpack-file-structure). For more on how to set up your Nerdpacks, see our [Nerdpack CLI commands](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-nerdpack).

### Usage

`$ nr1 create`

### Options

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        If present, overrides existing files without asking.
      </td>
    </tr>

    <tr>
      <td>
        `-n, --name=NAME`
      </td>

      <td>
        Names the component.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --type=TYPE`
      </td>

      <td>
        Specifies the component type.
      </td>
    </tr>

    <tr>
      <td>
        `--path=PATH`
      </td>

      <td>
        The route to the component.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        The authentication profile you want to use.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Adds extra information to the output.
      </td>
    </tr>
  </tbody>
</table>

## nr1 help [#nr1-help]

Shows all nr1 commands by default. To get details about a specific command, run `nr1 help COMMAND_NAME`.

### Usage

`$ nr1 help`

### Arguments

<table>
  <tbody>
    <tr>
      <td>
        `COMMAND_NAME`
      </td>

      <td>
        The name of a particular command.
      </td>
    </tr>
  </tbody>
</table>

### Examples

* `$ nr1 help`
* `$ nr1 help nerdpack`
* `$ nr1 help nerdpack:deploy`

## nr1 nrql [#nr1-nrql]

Fetches data from databases using a NRQL query.

To learn more about NRQL and how to use it, see our [NRQL docs](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/).

### Usage

`$ nr1 nrql OPTION ...`

### Options

<table>
  <tbody>
    <tr>
      <td>
        `-a, --account=ACCOUNT`
      </td>

      <td>
        This is **required**. The user account ID.
      </td>
    </tr>

    <tr>
      <td>
        `-q, --query=QUERY`
      </td>

      <td>
        This is **required**. The NRQL query to run.
      </td>
    </tr>

    <tr>
      <td>
        `-u, --ugly`
      </td>

      <td>
        Displays the content without tabs or spaces.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        The authentication profile you want to use.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Adds extra information to the output.
      </td>
    </tr>
  </tbody>
</table>

## nr1 profiles [#nr1-profiles]

Displays a list of commands you can use to manage your profiles. Run `nr1 help profiles:COMMAND` for more on their specific usages. You can have more than one profile, which is helpful for executing commands on multiple New Relic accounts.

To learn more about setting up profiles, see [our Github workshop](https://github.com/newrelic/nr1-workshop/blob/master/lab-cli/INSTRUCTIONS.md).

### Usage

`$ nr1 profiles:COMMAND`

### Commands

<table>
  <tbody>
    <tr>
      <td>
        `profiles:add`
      </td>

      <td>
        Adds a new profile to your profiles keychain.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:default`
      </td>

      <td>
        Chooses which profile should be default.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:list`
      </td>

      <td>
        Lists the profiles on your keychain.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:remove`
      </td>

      <td>
        Removes a profile from your keychain.
      </td>
    </tr>
  </tbody>
</table>

Updates to latest version of the CLI. You can specify which channel to update if you'd like.

### Usage

`$ nr1 update`

### Arguments

<table>
  <tbody>
    <tr>
      <td>
        `CHANNEL`
      </td>

      <td>
        The name of a particular channel.
      </td>
    </tr>
  </tbody>
</table>

### Examples

* `$ nr1 update`
* `$ nr1 update somechannel`
