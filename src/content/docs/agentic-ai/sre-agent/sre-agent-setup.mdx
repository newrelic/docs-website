Learn how to set up and configure the <DNT>SRE Agent</DNT> for your organization, including prerequisites, permissions, and alert integration.

## Prerequisites [#prerequisites]

Before setting up the <DNT>SRE Agent</DNT>, ensure you meet the following requirements:

### Account access
Access to the <DNT>SRE Agent</DNT> is available for customers on the <DNT>**Advanced Compute**</DNT> or <DNT>**Advanced Compute Add-on**</DNT> SKU.

### User permissions

The <DNT>SRE Agent</DNT> operates at the organization level. To complete setup, you must be both an **Organization Administrator** and **Authentication Domain Manager**.

Your user role must include these organization-level capabilities:

* **<DNT>SRE Agent</DNT>** - Modify, delete, other (Ask, Add to Automation)
* **System identity management** - Create, read, update, and delete permissions

<Callout variant="tip">
  If you don't have these permissions, you'll see the message: "This looks like a job for an admin." Contact your organization administrator to enable the agent for you.
</Callout>

## Set up the SRE Agent [#setup]

The onboarding workflow automatically handles the complete setup process. This includes:

* Provisioning a system identity to represent the agent
* Assigning the required permissions
* Creating an alert destination for notification workflows

<Callout variant="important">
  **What is a system identity?** A system identity is a non-human identity that allows the agent to authenticate directly to other services without requiring a user account. System identities are scoped to a single New Relic organization.
</Callout>

To onboard the <DNT>SRE Agent</DNT>:

1. Go to <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>
1. In the upper right, select <DNT>**Ask AI**</DNT>, then select the <DNT>**Configure**</DNT> tab (gear icon).
2. Under <DNT>**Agents**</DNT>, select <DNT>**SRE Agent**</DNT>.
3. In the <DNT>**Agent name**</DNT> field, enter a name for your agent. The system automatically adds <DNT>`SRE Agent -`</DNT> as a prefix and applies this name to the system identity and access management components.
4. From the dropdown, select the accounts you want the agent to analyze.

   <Callout variant="important">
     Adding more accounts allows the agent to better identify connections and surface insights across your stack. However, the agent's analysis will be visible to any users with SRE Agent interaction access, regardless of their direct access to those accounts.
   </Callout>

5. Review the **Permissions** summary. The system grants the <DNT>SRE Agent</DNT> standard read-only access to your selected accounts. Admins can modify these permissions later, but reducing access below read-only limits the agent's telemetry access and may result in incomplete analysis, missed anomalies, or degraded root-cause identification.
6. Select <DNT>**Add Agent**</DNT>. The system provisions the system identity, assigns read-only permissions, and creates a cross-account <DNT>"SRE Agent"</DNT> alert destination.

<Callout variant="tip">
  Creating the alert destination does not automatically assign any alert conditions or policies to the <DNT>SRE Agent</DNT>. See [Set up the SRE Agent with alerts](#alerts) to configure that.
</Callout>

## Set up the SRE Agent with alerts [#alerts]

Connect the <DNT>SRE Agent</DNT> to your alert workflows so that <DNT>SRE Agent</DNT> insights are automatically attached to every fired alert or issue.

### Phased approach recommendation

To validate performance, use a phased approach: start with a small subset of alert conditions or policies, confirm the agent is working well for your environment, then expand more broadly.

The <DNT>SRE Agent</DNT> uses the cross-account destination feature for alert integration. You must enable the cross-account destination preview before adding the <DNT>SRE Agent</DNT> to your alerts.

### Configure alert workflows

To set up the <DNT>SRE Agent</DNT> with alerts:

1. Go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Alerts > Issues & Activity > Notification Workflows > Add a workflow**</DNT>.
2. In your workflow configuration, under the <DNT>**Notify**</DNT> section, select <DNT>**AI agent**</DNT>.
3. Choose the <DNT>SRE Agent</DNT> destination created during onboarding and click <DNT>**Save message**</DNT>.

Once configured, when you open a recently fired alert or issue, you'll automatically see the <DNT>SRE Agent</DNT> summary for that item.
