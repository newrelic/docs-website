This guide walks you through setting up the New Relic SRE Agent, from verifying prerequisites to configuring alert integration.

## Prerequisites [#prerequisites]

Before setting up the SRE Agent, ensure you meet the following requirements:

### Account access
Access to the SRE Agent is available for customers on the **Advanced Compute** or **Advanced Compute Add-on** SKU.

### User permissions
Because the SRE Agent operates at the organization level, you must be an Organization Administrator and Authentication Domain Manager to complete setup. Your user role must include the following organization-level capabilities:

* **SRE Agent** - Modify, Delete, Other (Ask, Add to Automation)
* Permissions to manage system identities (create, read, update, and delete)

<Callout variant="tip">
  If you don't have these permissions, you'll see the message: "This looks like a job for an admin." Contact your organization administrator to enable the agent for you.
</Callout>

## Set up the SRE Agent [#setup]

The onboarding workflow handles the full setup automatically, including provisioning a system identity to represent the agent, assigning required permissions, and creating an alert destination you can add to notification flows.

<Callout variant="important">
  **What is a system identity?** A system identity is a non-human identity that allows the agent to authenticate directly to other services without requiring a user account. System identities are scoped to a single New Relic organization.
</Callout>

To onboard the SRE Agent:

1. Open the **Ask AI** panel and navigate to the **Configure** tab (the gear icon).
2. Under **Agents**, select **SRE Agent**.
3. In the **Agent name** text field, provide a recognizable name. The name you choose will be automatically prepended with `SRE Agent -` and applied to the system identity and associated access management components.
4. Use the dropdown to select the specific **account(s)** you want the agent to analyze.

   <Callout variant="important">
     Adding more accounts allows the agent to better identify connections and surface insights across your stack. However, the agent's analysis will be visible to any users with SRE Agent interaction access, regardless of their direct access to those accounts.
   </Callout>

5. Review the **Permissions** summary. Upon initial setup, the SRE Agent is granted the standard read-only role on the selected accounts. Admin users can modify these grants later, but reducing access below the standard read-only role will limit the telemetry the agent can access, resulting in incomplete analysis, missed anomalies, or degraded root-cause identification.
6. Click **Add Agent**. The system will provision the system identity, assign the necessary read-only grants, and create a cross-account "SRE Agent" alert destination.

<Callout variant="tip">
  Creating the alert destination does not automatically assign any alert conditions or policies to the SRE Agent. See [Set up the SRE Agent with alerts](#alerts) to configure that.
</Callout>

## Set up the SRE Agent with alerts [#alerts]

Connect the SRE Agent to your alert workflows so that SRE Agent insights are automatically attached to every fired alert or issue.

### Phased approach recommendation

To validate performance, we recommend a phased approach: start with a small subset of alert conditions or policies, confirm the agent is working well for your environment, then expand more broadly.

Access to SRE Agent as an alert destination is available via the cross-account destination feature. To add the SRE Agent to alerts, you must opt-in to the cross-account destination preview.

### Configure alert workflows

To set up the SRE Agent with alerts:

1. Go to **Issues and Activity &rarr; Notification Workflows &rarr; Add Workflow**.
2. In your workflow configuration, under the **Notify** section, select **AI Agent**.
3. Choose the SRE Agent destination created during onboarding and click **Save message**.

Once configured, when you open a recently fired alert or issue, you'll automatically see the SRE Agent summary for that item.
