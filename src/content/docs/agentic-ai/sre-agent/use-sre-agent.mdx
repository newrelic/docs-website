Once you've [set up the SRE Agent](/docs/agentic-ai/sre-agent/sre-agent-setup), you can start using it to analyze your environment and investigate incidents. This guide provides usage tips and best practices for getting the most out of your <DNT>SRE Agent</DNT>.

## Usage tips [#usage-tips]

Follow these best practices to get optimal results from your <DNT>SRE Agent</DNT>:

### Time range considerations

The <DNT>SRE Agent</DNT> is designed to prioritize live analysis and typically defaults to searching for events, issues, and entities within the **last 24 hours**. If you want to analyze something older, specify a targeted time window for analysis.

### Query optimization

The agent uses tools designed to convert natural language into NRQL queries, which you can then review and run in the query builder. The <DNT>SRE Agent</DNT> is designed to analyze telemetry data and is not currently intended or designed to perform source code review or source code analysis. When referencing long conversations, limit ambiguous pronoun usage to help improve the accuracy of the results.

### Alert integration best practices

Once you've integrated the <DNT>SRE Agent</DNT> with your alert workflows:

* **Review automated summaries**: When you open a recently fired alert or issue, you'll automatically see the <DNT>SRE Agent</DNT> summary for that item.
* **Ask follow-up questions**: Use the agent to dive deeper into specific aspects of an incident.
* **Leverage remediation suggestions**: Pay attention to specific technical interventions the agent suggests, such as rolling back deployments or adjusting resource allocations.

## Getting the most from SRE Agent analysis

### For alert triage
* Let the agent filter and prioritize incoming notifications by evaluating context against historical baselines.
* Use the agent's assessment of whether an event is a high-priority anomaly or an expected fluctuation.

### For health monitoring
* Request instant pulse checks on any part of your stack.
* Leverage the agent's correlation of golden signal metrics with underlying infrastructure performance.

### For change management
* Use the agent to review deployment markers and compare pre and post change telemetry.
* Monitor for performance regressions, new error patterns, or latency spikes after updates.

### For incident investigation
* Allow the agent to traverse distributed traces and analyze log patterns automatically.
* Use the agent to identify potential technical failures or downstream dependencies contributing to outages.
