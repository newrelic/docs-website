Once you've [set up the SRE Agent](/docs/agentic-ai/sre-agent/sre-agent-setup), you can start using it to analyze your environment and investigate incidents. This guide provides usage tips and best practices for getting the most out of your SRE Agent.

## How the SRE Agent analyzes your environment [#how-it-works]

The SRE Agent acts as an intelligent layer between your complex system data and actionable insights. When you ask the agent a question or when it responds to alerts, it follows three primary analysis pillars:

### 1. Comprehensive context gathering [#context-gathering]

The agent begins by mapping your architecture. It identifies relevant entities (services, hosts, or containers), explores their relationships and dependencies, and checks for recent change events or deployments that might have triggered a shift in performance.

### 2. Deep telemetry investigation [#telemetry-investigation]

Once context is established, the agent dives into the data:

* **Metrics and trends**: Analyzes throughput, latency, and error rates while forecasting future system behavior.
* **Logs and errors**: Sifts through distributed traces, error groups, and log patterns to isolate the location and cause of an anomaly.
* **Health and alerts**: Reviews active incidents, alert policies, and security vulnerabilities to understand the current blast radius.

### 3. Automated root cause and impact analysis [#root-cause-analysis]

The agent performs causal analysis—correlating performance dips with specific transaction patterns, infrastructure bottlenecks (such as Kafka lag), and end-user impact. It can also convert natural language into complex NRQL queries to surface data points that manual investigation might miss.

## Usage tips [#usage-tips]

Follow these best practices to get optimal results from your SRE Agent:

### Time range considerations

* The SRE Agent is optimized for live analysis and searches events, issues, and entities from the **last 24 hours** by default.
* To analyze older data, specify a time range. You can use `SINCE <time> UNTIL <time>` phrasing (similar to NRQL) to target a specific window.

### Query optimization

* Use the agent to convert natural language into NRQL queries, then run them directly in the query builder.
* In longer conversations, avoid ambiguous pronoun references for more accurate results.

### Alert integration best practices

Once you've integrated the SRE Agent with your alert workflows:

* **Review automated summaries**: When you open a recently fired alert or issue, you'll automatically see the SRE Agent summary for that item.
* **Ask follow-up questions**: Use the agent to dive deeper into specific aspects of an incident.
* **Leverage remediation suggestions**: Pay attention to specific technical interventions the agent suggests, such as rolling back deployments or adjusting resource allocations.

## Getting the most from SRE Agent analysis

### For alert triage
* Let the agent filter and prioritize incoming notifications by evaluating context against historical baselines.
* Use the agent's assessment of whether an event is a high-priority anomaly or an expected fluctuation.

### For health monitoring
* Request instant pulse checks on any part of your stack.
* Leverage the agent's correlation of golden signal metrics with underlying infrastructure performance.

### For change management
* Use the agent to review deployment markers and compare pre- and post-change telemetry.
* Monitor for performance regressions, new error patterns, or latency spikes after updates.

### For incident investigation
* Allow the agent to traverse distributed traces and analyze log patterns automatically.
* Use the agent to identify potential technical failures or downstream dependencies contributing to outages.
