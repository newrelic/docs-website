Learn how to set up and configure <DNT>Autopilot</DNT> for your organization, including prerequisites, permissions, and alert integration.

## Prerequisites [#prerequisites]

Before setting up <DNT>Autopilot</DNT>, ensure you meet the following requirements:

### Advanced Compute pricing

You must opt your account into Advanced Compute pricing to use <DNT>Autopilot</DNT>. If you want to add this SKU, contact your New Relic account team.

### Feature control manager

You can turn <DNT>Autopilot</DNT> on or off for your entire organization using [Feature control manager](/docs/accounts/accounts-billing/new-relic-one-user-management/feature-control-manager). Only an **Organization Manager** can edit these settings. After you turn on <DNT>Autopilot</DNT> in <DNT>Feature control manager</DNT>, you still need to configure it in the <DNT>AI</DNT> panel using the steps below.

### Admin permissions (for setup)

To give users permission to use <DNT>Autopilot</DNT>, a user must be an **Organization Manager** and **Authentication Domain Manager**.

<Callout variant="tip">
  If you don't have these permissions, you'll see the message: "This looks like a job for an admin." Contact your organization manager to enable the agent for you.
</Callout>

### User permissions (for access)

To use <DNT>Autopilot</DNT>, users must be [basic, full, or core users](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/).

To give users permission to use <DNT>Autopilot</DNT>, an admin or authentication domain manager must assign users a role that includes these organization-level capabilities:
* **<DNT>Autopilot</DNT>** - Read, other (Ask, Add to Automation)
* **New Relic MCP Server** - Read

### Preview opt ins

To access <DNT>Autopilot</DNT>, you must opt into the following public preview trials:
* Autopilot
* [MCP Server](/docs/agentic-ai/mcp/overview/)
* [Cross account destinations](/docs/apis/nerdgraph/examples/cross-account-destinations/overview/)

<Callout variant="tip">
Cross account destinations is only required if you want to set up alerts with <DNT>Autopilot</DNT>. Without this preview, <DNT>Autopilot</DNT> will remain partially configured.
</Callout>

## Set up Autopilot [#setup]

Use the onboarding flow to automatically set up:

* A system identity to represent the agent
* Required permissions for basic use
* An alert destination

<Callout variant="tip">
  You can configure only one <DNT>Autopilot</DNT> agent per organization.
</Callout>

<Callout variant="important">
  **What is a system identity?** A system identity is a non-human identity that allows the agent to authenticate directly to other services without requiring a user account. System identities are scoped to a single New Relic organization.
</Callout>

To onboard <DNT>Autopilot</DNT>:

1. Go to <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>
1. In the upper right, select <DNT>**Ask AI**</DNT> to open the <DNT>AI</DNT> panel. Select the **three-dot menu**, then select <DNT>**Configure**</DNT>.
2. Under <DNT>**Agents**</DNT>, select <DNT>**Autopilot**</DNT>.
3. In the <DNT>**Agent name**</DNT> field, enter a name for your agent. The system automatically adds <DNT>`Autopilot -`</DNT> as a prefix and applies this name to the system identity and access management components.
4. From the dropdown, select the accounts you want the agent to analyze.

   <Callout variant="important">
     You can add an unlimited number of accounts for <DNT>Autopilot</DNT> to analyze. Adding more accounts allows the agent to better identify connections and surface insights across your stack. However, the agent's analysis will be visible to any users with <DNT>Autopilot</DNT> interaction access, regardless of their direct access to those accounts.
   </Callout>

5. Review the **Permissions** summary. The system grants <DNT>Autopilot</DNT> standard read-only access to your selected accounts.

   <Callout variant="important">
     Admin users will have the ability to modify the accounts and grants provided to the agent. A reduction in grants from the standard read-only role will limit the telemetry data <DNT>Autopilot</DNT> can access, which can result in incomplete analysis, missed anomalies, or degraded root-cause identification capabilities.
   </Callout>

6. Click <DNT>**Add Agent**</DNT>. The system takes a few moments to complete the setup.

<Callout variant="tip">
  Creating the alert destination does not automatically assign any alert conditions or policies to <DNT>Autopilot</DNT>. See [Integrate with alerts](#alerts) to configure that.
</Callout>

<Callout variant="tip">
  If the <DNT>**Configure**</DNT> tab shows "Partially configured" next to <DNT>Autopilot</DNT>, retry the configuration steps above to complete setup.
</Callout>

## Integrate with alerts [#alerts]

Connect <DNT>Autopilot</DNT> to your alert workflows so that <DNT>Autopilot</DNT> insights are automatically attached to every fired alert or issue.

### Configure alert workflows

<Callout variant="tip">
**Phased approach recommendation:** Start with a small subset of alert conditions or policies to validate performance, confirm the agent works well for your environment, then expand more broadly.
</Callout>

To set up <DNT>Autopilot</DNT> with alerts:

1. Go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Alerts > Issues & Activity > Notification Workflows > Add a workflow**</DNT>.
2. In your workflow configuration, under the <DNT>**Notify**</DNT> section, select <DNT>**AI agent**</DNT>.
3. Choose the <DNT>Autopilot</DNT> destination created during onboarding and click <DNT>**Save message**</DNT>.

<Callout variant="tip">
  **The alert prompt is editable.** The <DNT>Autopilot</DNT> destination uses a default prompt, "Show me what's wrong, check for root cause, show me health charts." You can edit this prompt freely to focus the analysis on what matters for that workflow.
</Callout>

Once configured, when you open a recently fired alert or issue, you'll automatically see the <DNT>Autopilot</DNT> summary for that item.
