## Syntax

```py
newrelic.agent.generator_trace(name=None, group=None, label=None, params=None, terminal=False)
```

Used to instrument generators that aren't instrumented by default.

## Description

`generator_trace` is a decorator for adding to generator functions. Adding this decorator lets you collect additional [transaction](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) information (including transaction trace information).

The `generator_trace` decorator can be used on generators and coroutines with agent version 2.102.0.85 or higher. Timing of these objects begins when consumption starts, and ends when the object is exhausted or goes out of scope.

You can use the decorator in conjunction with existing decorators, including those for static and class methods. When nesting multiple decorators, have the `generator_trace` decorator as the outermost decorator.

## Alternate call forms [#alternate]

For setups where you cannot use the decorator, these alternate call forms are available:

### The wrapper

If you know in advance where the specific generators you want to trace are, you could use the `generator_trace` decorator. However, if you <DNT>**don't**</DNT> know all the generators that need to be traced (for example, if they're being looked up dynamically as part of a routing system), then you must use the `GeneratorTraceWrapper` to wrap the generator at the time of registration or at the time of calling.

### Path-based wrapping [#path-based]

`wrap_generator_trace` is used for wrapping generators outside of the code they're declared in. For example: you might use this to instrument library code that you don't want to modify.

For more about the differences between these call formats, see [Different call formats](/docs/python-agent-api-different-call-forms).

## Examples

### `generator_trace` example [#trace_example]

An example of using the `generator_trace` decorator:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
def gen():
    for i in range(5):
        yield i
```

An example of using the `generator_trace` decorator with async generator:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
async def gen():
    for i in range(5):
        yield i
```

### Wrapper example [#trace-wrap-example]

An example of using the `GeneratorTraceWrapper`:

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen)
```

If you want to name the metric after the endpoint name (rather than naming the metric based on the identifier for the generator being called), you can supply the name to use plus an alternate metric path prefix when the `GeneratorTraceWrapper` object is created.

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name="gen_5", group="Python/Generator")
```

If you want to name the metric dynamically (rather than naming the metric based on the identifier for the generator being called), you can supply the name as a generator when the `GeneratorTraceWrapper` object is created.

```py
import newrelic.agent

def gen(x):
    for i in range(x):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name = lambda x: f"gen_{x}")
```
