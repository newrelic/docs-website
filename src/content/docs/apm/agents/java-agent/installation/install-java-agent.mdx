This guide provides step-by-step instructions for installing the New Relic Java agent to monitor your Java applications. Before you begin, make sure you meet the [compatibility requirements](/docs/apm/agents/java-agent/get-started/compatibility-requirements-java-agent).

## Download the Java agent [#download]

1. If you don't have one already, [create a New Relic account](https://newrelic.com/signup). It's free, forever.
2. Download the Java agent:
   * From the New Relic UI: Go to **[one.newrelic.com](https://one.newrelic.com) > Add data > Application monitoring > Java**
   * Or download directly from [Maven Central](https://search.maven.org/search?q=g:com.newrelic.agent.java)

3. Unzip the `newrelic-java.zip` file to a location on your system. The archive contains:
   * `newrelic.jar` - The Java agent
   * `newrelic.yml` - The configuration file
   * `README.txt` - Documentation

## Configure the agent [#configure]

1. Open `newrelic.yml` in a text editor.

2. Find the `license_key` configuration option and set it to your [New Relic license key](/docs/apis/intro-apis/new-relic-api-keys/#license-key):

   ```yaml
   license_key: YOUR_LICENSE_KEY_HERE
   ```

3. Set your [application name](/docs/apm/agents/java-agent/configuration/name-your-java-application):

   ```yaml
   app_name: My Application
   ```

4. (Optional) Configure other [settings](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file) as needed.

## Add the agent to your JVM [#add-agent]

To enable the Java agent, add the `-javaagent` flag to your JVM startup command:

```bash
java -javaagent:/path/to/newrelic.jar -jar your-application.jar
```

<Callout variant="important">
  The `-javaagent` flag must come **before** the `-jar` flag in your startup command.
</Callout>

### Examples by application server

<CollapserGroup>
  <Collapser
    id="tomcat"
    title="Tomcat"
  >
    Add the `-javaagent` flag to your `CATALINA_OPTS` environment variable:

    **Linux/Mac:**
    ```bash
    export CATALINA_OPTS="$CATALINA_OPTS -javaagent:/path/to/newrelic.jar"
    ```

    **Windows:**
    ```batch
    set CATALINA_OPTS=%CATALINA_OPTS% -javaagent:C:\path\to\newrelic.jar
    ```
  </Collapser>

  <Collapser
    id="jetty"
    title="Jetty"
  >
    Add the `-javaagent` flag to your Jetty startup script or configuration file:

    ```bash
    java -javaagent:/path/to/newrelic.jar -jar start.jar
    ```
  </Collapser>

  <Collapser
    id="jboss"
    title="JBoss/WildFly"
  >
    Edit your `standalone.conf` (Linux/Mac) or `standalone.conf.bat` (Windows):

    **Linux/Mac:**
    ```bash
    JAVA_OPTS="$JAVA_OPTS -javaagent:/path/to/newrelic.jar"
    ```

    **Windows:**
    ```batch
    set "JAVA_OPTS=%JAVA_OPTS% -javaagent:C:\path\to\newrelic.jar"
    ```
  </Collapser>

  <Collapser
    id="weblogic"
    title="WebLogic"
  >
    Edit your domain's `startWebLogic.sh` (Linux/Mac) or `startWebLogic.cmd` (Windows):

    **Linux/Mac:**
    ```bash
    export JAVA_OPTIONS="$JAVA_OPTIONS -javaagent:/path/to/newrelic.jar"
    ```

    **Windows:**
    ```batch
    set JAVA_OPTIONS=%JAVA_OPTIONS% -javaagent:C:\path\to\newrelic.jar
    ```
  </Collapser>

  <Collapser
    id="websphere"
    title="WebSphere"
  >
    In the WebSphere Administrative Console:

    1. Go to **Servers > Server Types > WebSphere application servers**
    2. Select your server
    3. Go to **Java and Process Management > Process Definition**
    4. Click **Java Virtual Machine**
    5. Add to the **Generic JVM arguments** field:
       ```
       -javaagent:/path/to/newrelic.jar
       ```
  </Collapser>

  <Collapser
    id="spring-boot"
    title="Spring Boot"
  >
    Add the `-javaagent` flag when running your Spring Boot application:

    ```bash
    java -javaagent:/path/to/newrelic.jar -jar your-spring-boot-app.jar
    ```

    Or configure in your `application.properties` or startup script.
  </Collapser>
</CollapserGroup>

## Restart your application [#restart]

Restart your Java application for the changes to take effect.

## Verify the installation [#verify]

Within a few minutes, your application should begin sending data to New Relic. To verify:

1. Go to **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services**
2. Look for your application name
3. Click on it to view performance data

If you don't see data after 5-10 minutes, see [troubleshooting tips](/docs/apm/agents/java-agent/troubleshooting/no-data-appears-java).
