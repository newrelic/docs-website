  Back to the full settings index: [Java agent configuration](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file)
</Callout>

## JFR (Real-time profiling)

The Java agent uses Java Flight Recorder (JFR) to collect high fidelity JVM data for [Real-time profiling](/docs/agents/java-agent/features/real-time-profiling-java-using-jfr-metrics/).

Configure real-time profiling in the `jfr` section in the agent YAML. These configuration options can be overridden with [system properties](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file#System_Properties) prefixed by `newrelic.config.jfr.` or with [environment variables](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file#Environment_Variables) prefixed with `NEW_RELIC_JFR_`.

    <Callout variant="important">

    The maximum number of JFR events stored in the agent's memory queue. Increasing this value can help prevent gaps in JFR data, but it will also increase the agent's resource consumption. If the queue is too large, data may be dropped if backend pipeline limits are exceeded. For more information, refer to [event API ingest limits](/docs/data-apis/ingest-apis/event-api/introduction-event-api/#limits).

    <Callout variant="important">
      This applies to Java agent [version 8.17.0 or higher](/docs/release-notes/agent-release-notes/java-release-notes).
    </Callout>
  </Collapser>
