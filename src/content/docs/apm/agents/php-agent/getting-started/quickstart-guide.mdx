Get your PHP application monitored in **15 minutes**. This guide walks you through the essential steps to start monitoring with New Relic.

## Your onboarding journey

Follow these milestones to successfully onboard your PHP application:

### ✓ Milestone 1: Check compatibility
**Time: 2 minutes**

Before you begin, verify your environment meets our requirements:

- Review [compatibility and requirements](/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements)

- Have your [license key](/docs/apis/intro-apis/new-relic-api-keys/#license-key) ready

</Callout>

### ✅ Milestone 3: Verify data is flowing
**Time: 5-10 minutes**

This is the most important step - let's confirm your agent is working:

1. **Restart your web server:**
   ```bash
   # Apache
   sudo systemctl restart apache2

   # Nginx + PHP-FPM
   sudo systemctl restart php-fpm && sudo systemctl restart nginx
   ```

2. **Generate traffic** to your application (browse for 30 seconds)

3. **Wait 3-5 minutes** for data to appear

4. **Check your dashboard:**
   - Go to <DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > APM & services**</DNT>
   - Look for your application name

<Callout variant="tip">
**✓ Seeing data?** Excellent! Your agent is working. Continue to Milestone 4.

**✗ No data after 10 minutes?** Don't worry - see our [verify installation troubleshooting guide](/docs/apm/agents/php-agent/installation/verify-installation) for help.
</Callout>

### ⚙️ Milestone 4: Configure your application
**Time: 3-5 minutes (optional)**

Now that data is flowing, optimize your setup with these essential configurations:

**Must-do configurations:**
- [Name your application](/docs/apm/agents/php-agent/configuration/name-your-php-application) - Give it a meaningful name instead of the default
- [Configure your license key](/docs/apm/agents/php-agent/configuration/php-agent-configuration#inivar-license) - If not set during install

**Recommended configurations:**
- [Enable distributed tracing](/docs/apm/agents/php-agent/configuration/distributed-tracing-php-agent) - See the full path of requests across services
- [Configure error collection](/docs/apm/agents/php-agent/configuration/php-agent-configuration#inivar-err-enabled) - Capture and analyze errors

See all options in our [configuration overview](/docs/apm/agents/php-agent/getting-started/configuration-overview).

## 🎉 You're all set!

Congratulations! You're now monitoring your PHP application with New Relic. Here's what you can do next:

### Explore your data

- **APM Summary** - View response times, throughput, and errors
- **Transactions** - Analyze slow transactions and database calls
- **Errors** - Debug exceptions and error traces
- **Service maps** - Visualize your application architecture

### Advanced features

- [Code-level metrics](/docs/apm/agents/php-agent/features/php-code-level-metrics) - See performance at the function level
- [Logs in context](/docs/logs/logs-context/configure-logs-context-php/) - Connect logs with traces and errors
- [Custom attributes](/docs/apm/agents/php-agent/attributes/php-agent-attributes) - Add business context to your data
- [Multiple accounts](/docs/apm/agents/php-agent/features/multiple-accounts) - Report to multiple New Relic accounts
