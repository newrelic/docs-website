Configure the New Relic PHP agent to customize how it monitors your application. This page helps you understand what to configure and when.

## Configuration methods

You can configure the PHP agent using multiple methods:

### 1. INI file (recommended)

The primary configuration file:
- **Location:** `/etc/php/8.1/mods-available/newrelic.ini` (Ubuntu/Debian) or `/etc/php.d/newrelic.ini` (RedHat/CentOS)
- **Scope:** Affects all PHP applications on the server
- **Requires:** Web server restart after changes

```ini
; Example configuration
newrelic.appname = "My PHP Application"
newrelic.license = "YOUR_LICENSE_KEY"
newrelic.enabled = true
newrelic.loglevel = "info"
```

See [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration) for all INI options.

### 2. Per-directory INI settings

Configure different settings for different applications or directories:
- Uses `.htaccess` or `nginx.conf`
- Override global settings per application
- No restart required for Apache `.htaccess`

See [per-directory INI settings](/docs/apm/agents/php-agent/configuration/php-directory-ini-settings).

### 3. API calls (runtime)

Change settings programmatically in your code:
```php
// Change app name at runtime
newrelic_set_appname("My App");

// Add custom parameters
newrelic_add_custom_parameter("user_tier", "premium");
```

See [PHP agent API](/docs/apm/agents/php-agent/php-agent-api/guide-using-php-agent-api).

### 4. Environment variables

Set configuration via environment variables:
```bash
export NEWRELIC_LICENSE_KEY="your_license_key"
export NEWRELIC_APPNAME="My Application"
```

Useful for containerized environments. See [environment variables](/docs/apm/agents/php-agent/other-installation/understanding-net-agent-environment-variables).

## Essential configurations

Start with these essential settings to ensure proper monitoring:

### 1. Application name (required)

Give your application a meaningful name:

```ini
newrelic.appname = "Production - Web Store"
```

**Why it matters:**
- Identifies your app in the New Relic UI
- Default is generic `PHP Application`
- Can report to multiple apps (separated by semicolons)

See [name your PHP application](/docs/apm/agents/php-agent/configuration/name-your-php-application).

### 2. License key (required)

Your New Relic license key:

```ini
newrelic.license = "YOUR_40_CHAR_LICENSE_KEY"
```

**How to get it:**
- Find in [API keys UI](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher)
- 40-character hexadecimal string
- Keep it secret!

### 3. Enable/disable the agent

Turn monitoring on or off:

```ini
newrelic.enabled = true  ; or false to disable
```

**Use cases:**
- Disable in development environments
- Enable only in production
- Temporarily disable for maintenance

## Recommended configurations

After essential setup, configure these for better insights:

### Distributed tracing

See requests across multiple services:

```ini
newrelic.distributed_tracing_enabled = true
```

**Benefits:**
- Track requests across microservices
- See complete transaction paths
- Identify bottlenecks across services

See [distributed tracing](/docs/apm/agents/php-agent/configuration/distributed-tracing-php-agent).

### Transaction naming

Control how transactions are named:

```ini
; Use framework-detected names
newrelic.framework = "laravel"  ; or symfony, drupal, etc.
```

Or use the API:
```php
newrelic_name_transaction("checkout-process");
```

See [transaction naming](/docs/apm/agents/php-agent/features/php-custom-instrumentation#name-transactions).

### Error collection

Configure how errors are captured:

```ini
; Capture all errors
newrelic.error_collector.enabled = true

; Record exceptions
newrelic.error_collector.record_database_errors = true

; Ignore specific exceptions
newrelic.error_collector.ignore_exceptions = "MyCustomException"
```

### Browser monitoring (Real User Monitoring)

Enable frontend monitoring:

```ini
newrelic.browser_monitoring.auto_instrument = true
```

See [browser monitoring](/docs/apm/agents/php-agent/features/browser-monitoring-php-agent).

## Configuration by use case

### Multi-tenant applications

Report different parts to different New Relic applications:

```ini
; Default application
newrelic.appname = "Multi-Tenant App"
```

Then use the API per tenant:
```php
// Change app name based on tenant
newrelic_set_appname("Tenant: " . $tenant_id);
```

See [multiple accounts](/docs/apm/agents/php-agent/features/multiple-accounts).

### High-traffic applications

Optimize for performance:

```ini
; Increase transaction trace threshold
newrelic.transaction_tracer.threshold = "500ms"  ; Only trace slow transactions

; Reduce detailed traces
newrelic.transaction_tracer.detail = 0  ; 0 = minimal overhead

; Limit traced SQL statements
newrelic.transaction_tracer.explain_threshold = "1000ms"
```

### Development vs. production

Use different configs per environment:

**Development:**
```ini
newrelic.enabled = false  ; Or separate app name
newrelic.loglevel = "verbosedebug"
```

**Production:**
```ini
newrelic.enabled = true
newrelic.appname = "Production - My App"
newrelic.loglevel = "info"
```

### Docker/containers

Configure via environment variables:

```dockerfile
ENV NEWRELIC_LICENSE_KEY="your_key"
ENV NEWRELIC_APPNAME="Containerized App"
ENV NEWRELIC_DISTRIBUTED_TRACING_ENABLED=true
```

## Advanced configurations

### Daemon configuration

Configure the proxy daemon separately:

**File:** `/etc/newrelic/newrelic.cfg`

```ini
[daemon]
logfile = /var/log/newrelic/newrelic-daemon.log
loglevel = info
proxy = "http://proxy.example.com:8080"  ; If behind proxy
```

See [proxy daemon settings](/docs/apm/agents/php-agent/configuration/proxy-daemon-newreliccfg-settings).

### Custom attributes

Add business context to transactions:

```php
// Add user information
newrelic_add_custom_parameter("user_id", $user->id);
newrelic_add_custom_parameter("plan", "premium");
newrelic_add_custom_parameter("region", "us-east");
```

See [PHP agent attributes](/docs/apm/agents/php-agent/attributes/php-agent-attributes).

### Transaction traces

Fine-tune when and how transactions are captured:

```ini
; Capture traces for slow transactions
newrelic.transaction_tracer.enabled = true
newrelic.transaction_tracer.threshold = "apdex_f"  ; 4x Apdex T

; Capture all slow SQL queries
newrelic.transaction_tracer.explain_enabled = true
newrelic.transaction_tracer.explain_threshold = "500ms"

; Record SQL with parameters (be careful with sensitive data!)
newrelic.transaction_tracer.record_sql = "obfuscated"  ; or "raw" or "off"
```

### Security and privacy

Control what data is sent:

```ini
; High security mode (requires account-level setting)
newrelic.high_security = true

; Don't capture parameters (for sensitive data)
newrelic.attributes.include = "request.headers.*"
newrelic.attributes.exclude = "request.parameters.*"

; Strip credit card numbers from URLs
newrelic.transaction_tracer.record_sql = "obfuscated"
```

See [PHP agent security](/docs/apm/agents/php-agent/getting-started/apm-agent-security-php).

## Testing configuration changes

After making changes:

1. **Restart web server:**
   ```bash
   sudo systemctl restart apache2  # or nginx, php-fpm
   ```

2. **Verify changes loaded:**
   ```bash
   php -i | grep newrelic.appname
   ```

3. **Check for errors:**
   ```bash
   sudo tail -50 /var/log/newrelic/php_agent.log
   ```

4. **Generate traffic and verify in UI** (wait 3-5 minutes)

## Common configuration patterns

### Pattern 1: Basic production setup
```ini
newrelic.appname = "Production - My App"
newrelic.license = "YOUR_LICENSE_KEY"
newrelic.enabled = true
newrelic.loglevel = "info"
newrelic.distributed_tracing_enabled = true
newrelic.browser_monitoring.auto_instrument = true
```

### Pattern 2: Development/staging
```ini
newrelic.appname = "Staging - My App"
newrelic.license = "YOUR_LICENSE_KEY"
newrelic.enabled = true
newrelic.loglevel = "verbosedebug"  ; More detailed logging
```

### Pattern 3: Multi-app reporting
```ini
; Report to both "My App" and "All Apps" rollup
newrelic.appname = "My App;All Apps"
```

### Pattern 4: Performance-optimized
```ini
newrelic.enabled = true
newrelic.appname = "High Traffic App"
newrelic.transaction_tracer.threshold = "1000ms"  ; Only slow transactions
newrelic.transaction_tracer.detail = 0  ; Minimal overhead
```

## Troubleshooting configuration

### Changes not taking effect

1. Verify you edited the correct INI file
2. Restart web server (not just reload)
3. Check PHP is loading the file: `php --ini | grep newrelic`

### Configuration syntax errors

Check logs for parse errors:
```bash
sudo tail -50 /var/log/newrelic/php_agent.log
```

### Multiple configuration files

PHP may load configs from multiple locations:
```bash
# See all INI files being loaded
php --ini

# Check actual configuration values
php -i | grep newrelic
```
