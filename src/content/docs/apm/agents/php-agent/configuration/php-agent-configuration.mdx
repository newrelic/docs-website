The PHP agent is configured through two files and an optional per-directory override mechanism. Most installations only ever touch `newrelic.ini`. The other mechanisms exist for specific deployment patterns — containers, multi-app servers, external daemon management — and you can ignore them until you need them.

## Order of precedence [#precedence]

When the same setting is defined in multiple places, the PHP agent uses this order — higher entries win:

<table>
  <thead>
    <tr>
      <th style={{ width: "60px" }}>Priority</th>
      <th style={{ width: "220px" }}>Source</th>
      <th>Notes</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1 (highest)</td>
      <td>Per-directory INI settings</td>
      <td>`php_value` / `php_flag` in `.htaccess`, `<VirtualHost>`, or PHP-FPM pool config. Applied at request time — overrides everything below.</td>
    </tr>
    <tr>
      <td>2</td>
      <td>Environment variables (`NEW_RELIC_*`)</td>
      <td>Set in the server or container environment. Override values in `newrelic.ini` for the same setting.</td>
    </tr>
    <tr>
      <td>3</td>
      <td>`newrelic.ini` (global)</td>
      <td>The primary config file, applied server-wide. Most teams only configure here.</td>
    </tr>
    <tr>
      <td>4 (lowest)</td>
      <td>Built-in defaults</td>
      <td>Applied when no other source sets the value. Defaults are listed on each settings reference page.</td>
    </tr>
  </tbody>
</table>

<Callout variant="important">
  **Server-side configuration does not apply to the PHP agent.** Unlike some other New Relic agents, PHP configuration cannot be managed from the New Relic UI. The only two values you can change from the UI are the app's **display alias** and its **Apdex-T threshold** — all other settings must be changed in the files or environment variables listed above.
</Callout>

## newrelic.ini — the primary config file [#newrelic-ini]

`newrelic.ini` is the PHP agent's main configuration file, created automatically by the `newrelic-install` script. Every installation uses it.

### Where to find it

```bash
php --ini | grep newrelic
```

Typical locations:

| Platform | Path |
|---|---|
| Amazon Linux / RHEL / CentOS | `/etc/php.d/newrelic.ini` |
| Ubuntu / Debian | `/etc/php/{version}/mods-available/newrelic.ini` |
| SLES | `/etc/php{version}/conf.d/newrelic.ini` |
| Custom PHP build | directory shown by `php-config --ini-dir` |

### Minimum required settings

```ini
newrelic.license = "YOUR_LICENSE_KEY"
newrelic.appname = "My Application"
```

Everything else has a working default. Restart your web server after any change.

### Settings reference

<Callout variant="important">
  Always restart your web server after changing `newrelic.ini` settings. Otherwise they may not [take effect immediately](/docs/agents/php-agent/troubleshooting/ini-settings-not-taking-effect-immediately).
</Callout>

## newrelic.cfg — daemon config file [#newrelic-cfg]

`newrelic.cfg` configures the `newrelic-daemon` process directly. **Most installations never need this file** — the daemon starts automatically in agent mode. You only need `newrelic.cfg` when managing the daemon yourself.

| Scenario | Why you need `newrelic.cfg` |
|---|---|
| Agent and daemon in separate containers | Agent can't start the daemon itself — daemon must be pre-started with its socket address configured. |
| Init-system-managed daemon (systemd unit) | You control daemon lifecycle outside PHP. |
| Custom proxy between daemon and New Relic | Route daemon-to-collector traffic through your proxy. |
| Custom SSL certificates | `ssl_ca_bundle` / `ssl_ca_path` only exist in `newrelic.cfg`. |

For setup instructions and the full settings list, see [Proxy daemon (newrelic.cfg) settings](/docs/apm/agents/php-agent/configuration/proxy-daemon-newreliccfg-settings).

## Per-directory INI settings [#per-directory]

Any `newrelic.ini` setting with **PERDIR scope** can be overridden per virtual host, directory, or `.htaccess` file — without touching the global config. Use this when a single server hosts multiple PHP apps that need different app names, license keys, or tracing settings.

For setup instructions and examples for Apache, Nginx, and PHP-FPM, see [PHP per-directory INI settings](/docs/apm/agents/php-agent/configuration/php-directory-ini-settings).

## Environment variables [#env-vars]

Every `newrelic.ini` setting can be set via an environment variable using the pattern `NEW_RELIC_` + the setting name uppercased with dots replaced by underscores:

| `newrelic.ini` setting | Environment variable |
|---|---|
| `newrelic.license` | `NEW_RELIC_LICENSE_KEY` |
| `newrelic.appname` | `NEW_RELIC_APP_NAME` |
| `newrelic.loglevel` | `NEW_RELIC_LOG_LEVEL` |
| `newrelic.distributed_tracing_enabled` | `NEW_RELIC_DISTRIBUTED_TRACING_ENABLED` |

Use this for Docker/Kubernetes deployments (inject as container env vars — no config file needed in the image), CI/CD pipelines, or any scenario where you need different settings per environment without file changes.

<Callout variant="tip">
  Environment variables take precedence over `newrelic.ini` values. If a setting isn't behaving as expected, check for an env var override with `printenv | grep NEW_RELIC`.
</Callout>
