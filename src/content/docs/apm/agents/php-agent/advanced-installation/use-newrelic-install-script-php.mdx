The interactive `newrelic-install` script simplifies installing the PHP agent. It handles multiple PHP installations and non-standard locations, and will:

- Copy the correct PHP modules into place
- Install sample `ini` files
- Configure the New Relic proxy daemon

## Find the installer script [#location]

The script location depends on how the agent was installed:

| Installation method | Script location |
|---|---|
| System package manager (rpm/apt) | `/usr/bin/newrelic-install` |
| Tar archive | `./newrelic-install` inside the extracted directory |

## Run the install mode [#install-install]

<Steps>
  <Step>
    ## Invoke install mode [#invoke]

    Run the script using **one** of these methods:

    ```bash
    # Option A: pass install directly
    sudo newrelic-install install

    # Option B: run with no arguments and select install from the menu
    sudo newrelic-install
    ```
  </Step>

  <Step>
    ## Provide your license key [#license-key]

    At the prompt, enter your New Relic <InlinePopover type="licenseKey"/>. This key is inserted into all `newrelic.ini` files created during installation.
  </Step>

  <Step>
    ## Select PHP version (if multiple detected) [#select-version]

    If `newrelic-install` finds more than one PHP installation, it presents a selection menu. If only one version is found, this step is skipped.

    Example menu:

    ```
    1)   /usr/bin
    2)   /usr/local/php/8.1/bin
    3)   /usr/local/php/8.2/bin

    0)   Exit

    Select (1-3, 0 to exit, or all):
    ```

    - Enter a single number to target one version
    - Enter a comma- or space-separated list for multiple versions
    - Enter `all` to target every version listed
    - Enter `0` to exit

    <Callout variant="important">
      For PHP installations that allow multiple `.ini` files, the script copies a template file into place if a modified one is not already installed. Note which `.ini` files you need to modify manually after install.
    </Callout>
  </Step>

  <Step>
    ## Install the daemon (if applicable) [#daemon]

    If the daemon was not already installed by the package manager, the script installs it now.

    For upgrades and reinstalls, the script prompts for confirmation before overwriting the existing daemon binary.

    A successful install ends with:

    ```
    New Relic is now installed on your system. Congratulations!
    ```

    The script saves an install archive at `/tmp/nrinstall-nnnn.tar` containing the install log and system information. Keep this file — it is useful for troubleshooting with New Relic support.
  </Step>

  <Step>
    ## Restart your web server [#restart-web-server]

    Restart your web server to activate the PHP agent:

    ```bash
    # Apache
    sudo systemctl restart httpd        # RHEL/CentOS
    sudo systemctl restart apache2      # Ubuntu/Debian

    # Nginx + PHP-FPM
    sudo systemctl restart php-fpm
    sudo systemctl restart nginx
    ```

    If you use PHP-FPM, restart it separately before restarting Nginx.
  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    1. Send at least **two requests** to your application — the PHP daemon requires two transactions on first start before it begins reporting.
    2. Wait 2–3 minutes, then open [one.newrelic.com > APM](https://one.newrelic.com/apm) and look for your app by name.

    <Callout variant="tip">
      App not appearing? Run `php -r "phpinfo();" | grep -i newrelic` to confirm the extension loaded. For other causes, see [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php).
    </Callout>
  </Step>
</Steps>

## Install daemon only [#install-daemon-mode]

To install the daemon without the PHP extension — for example when the daemon runs in a separate container — use PHP agent 9.2 or higher:

```bash
sudo newrelic-install install_daemon
```

See [Docker and other container environments](/docs/agents/php-agent/advanced-installation/install-php-agent-docker) for container-specific setup.

## Uninstall [#install-uninstall]

<CollapserGroup>
  <Collapser
    id="uninstall-keep"
    title="Uninstall and keep config files"
  >
    Removes the agent binaries but preserves `newrelic.ini` and the daemon config file. Use this when upgrading to a new agent version.

    ```bash
    sudo newrelic-install uninstall
    ```

    Or run with no arguments and select the uninstall option from the menu.
  </Collapser>

  <Collapser
    id="uninstall-permanent"
    title="Permanently uninstall New Relic"
  >
    Removes all configuration files and links created at installation time. Does **not** remove software packages installed by your package manager.

    ```bash
    sudo newrelic-install purge
    ```

    To also remove the packages themselves, see [Uninstalling RedHat and CentOS](/docs/php/php-agent-installation-redhat-and-centos#uninstalling) or [Uninstalling Ubuntu and Debian](/docs/php/php-agent-installation-ubuntu-and-debian#uninstalling).
  </Collapser>
</CollapserGroup>

## Troubleshooting [#troubleshoot]

If installation fails, review [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php). When contacting support, attach the install archive (`/tmp/nrinstall-nnnn.tar`) and the output of [`phpinfo()`](/docs/agents/php-agent/troubleshooting/submitting-troubleshooting-results-php) from your web server.
