Use this page if the standard installer didn't find your PHP installation — for example, self-compiled PHP or a self-contained PHP stack.

## Prerequisites [#prerequisites]

Check these requirements before you start.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>Requirement</th>
      <th>Details</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>**New Relic account**</td>
      <td>You need a [New Relic account](https://newrelic.com/signup) and your <InlinePopover type="licenseKey"/>. Have it ready — you'll need it during configuration.</td>
    </tr>
    <tr>
      <td>**PHP installation info**</td>
      <td>Run `php -i` or check `phpinfo()` output to find: PHP extension version (`20090626`, `20100525`, `20121212`, `20131226`, `20151012`, or `20160303`), the extension/module install directory, and whether PHP was compiled with ZTS (Zend Thread Safety) support. ZTS applies only to [PHP agent versions 9.17 and earlier](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-9170300/).</td>
    </tr>
    <tr>
      <td>**PHP agent package**</td>
      <td>The `newrelic-install` script or agent tarball must be present on the host. See [tar file installation](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file) if you need to download it first.</td>
    </tr>
    <tr>
      <td>**Permissions**</td>
      <td>`sudo` or root access required to write to system directories and restart services.</td>
    </tr>
  </tbody>
</table>

<Steps>
  <Step>
    ## Install the agent [#install]

    Choose the method that fits your environment:

    <CollapserGroup>
      <Collapser
        id="script-install"
        title="Use the newrelic-install script (recommended)"
      >
        Point the install script at your non-standard PHP location using one of these options, then run it:

        <CollapserGroup>
          <Collapser
            id="add-php-to-path"
            title="Add your PHP location to PATH"
          >
            The install script needs to find your `php` or `php-config` binary. Add the directory containing those programs to your current `PATH`, then run the script:

            ```bash
            sudo newrelic-install install
            ```
          </Collapser>

          <Collapser
            id="nr-install-path"
            title="Set NR_INSTALL_PATH"
          >
            A colon-separated list of directories for PHP installations in addition to those in `PATH`. Make sure these directories contain `php` or `php-config`:

            ```bash
            NR_INSTALL_PATH=/u/php/5.3/bin; export NR_INSTALL_PATH
            newrelic-install install
            ```
          </Collapser>

          <Collapser
            id="nr-install-phplist"
            title="Set NR_INSTALL_PHPLIST"
          >
            A colon-separated list of exact directories to search. This option ignores `PATH` and `NR_INSTALL_PATH`:

            ```bash
            NR_INSTALL_PHPLIST=/usr/local/bin:/u/php/5.3/bin; export NR_INSTALL_PHPLIST
            newrelic-install install
            ```
          </Collapser>
        </CollapserGroup>

        When the script finds multiple PHP versions, it presents a selection menu — choose your specific version from the list.
      </Collapser>

      <Collapser
        id="manual-install"
        title="Install manually"
      >
        Use this method only if the install script cannot work in your environment. Gather the following values from `phpinfo()` first:

        <table>
          <thead>
            <tr>
              <th width={125}>Variable</th>
              <th>Where to find it</th>
            </tr>
          </thead>
          <tbody>
            <tr>
              <td>`PHPAPI`</td>
              <td>Labeled `PHP Extension` in the `phpinfo()` header.</td>
            </tr>
            <tr>
              <td>`ARCH`</td>
              <td>Run `file /path/to/php`. `ELF-32` → `x86`; `ELF-64` → `x64`.</td>
            </tr>
            <tr>
              <td>`MODULEDIR`</td>
              <td>Labeled `extension_dir` in the PHP Core section of `phpinfo()`.</td>
            </tr>
            <tr>
              <td>`PHPZTS`</td>
              <td>Look for `Thread Safety` at the top of `phpinfo()`. Disabled → empty string; Enabled → `-zts` (with leading hyphen). ZTS applies only to [PHP agent 9.19.0.309 and earlier](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-9200310/).</td>
            </tr>
            <tr>
              <td>`NRBASEDIR`</td>
              <td>Base directory of the New Relic agent install, for example `/usr/lib/newrelic-php5` or the directory where you extracted the tar archive.</td>
            </tr>
          </tbody>
        </table>

        Then run these commands, replacing the example values with your actual settings:

        ```bash
        NRBASEDIR=/usr/lib/newrelic-php5; export NRBASEDIR
        MODULEDIR=/usr/lib/php/modules; export MODULEDIR
        ARCH=x64; export ARCH
        PHPAPI=20090626; export PHPAPI
        PHPZTS="-zts"; export PHPZTS
        rm -f $MODULEDIR/newrelic.so
        ln -s $NRBASEDIR/agent/$ARCH/newrelic-${PHPAPI}${PHPZTS}.so \
          $MODULEDIR/newrelic.so
        ```

        Then copy the daemon binary:

        ```bash
        cp $NRBASEDIR/daemon/newrelic-daemon.$ARCH /usr/bin/newrelic-daemon
        ```
      </Collapser>
    </CollapserGroup>
  </Step>

  <Step>
    ## Configure your app [#configure]

    **If you used the install script:** The script creates `newrelic.ini` with your license key pre-filled. Open it and set your app name:

    ```ini
    ; Required — all apps with the same name report together in New Relic
    newrelic.appname = "My Application"
    ```

    **If you installed manually:** Copy the ini template and configure it:

    1. Copy `$NRBASEDIR/scripts/newrelic.ini.template` to the directory PHP scans for additional ini files (find it in `phpinfo()` under `Scan this dir for additional .ini files`). If that shows `(none)`, add the settings to your main `php.ini`.
    2. Rename the copied file to `newrelic.ini`.
    3. Set your <InlinePopover type="licenseKey"/> and app name in the file:

       ```ini
       newrelic.license = "YOUR_LICENSE_KEY"
       newrelic.appname = "My Application"
       ```

    Then restart your web server:

    ```bash
    # Apache
    sudo systemctl restart httpd        # RHEL/CentOS
    sudo systemctl restart apache2      # Ubuntu/Debian

    # Nginx + PHP-FPM
    sudo systemctl restart php-fpm
    sudo systemctl restart nginx
    ```

  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    1. Check `phpinfo()` for a `newrelic` section — if it appears, the agent loaded successfully.
    2. Confirm two `newrelic-daemon` processes are running:

       ```bash
       # RedHat-based systems
       ps -aef | grep newrelic-daemon

       # Debian-based systems
       ps aux | grep newrelic-daemon
       ```

    3. Send some traffic to your app, then wait 3–5 minutes for results to appear in your [APM Summary page](/docs/apm/apm-ui-pages/monitoring/apm-summary-page-view-transaction-apdex-usage-data/).

    <Callout variant="tip">
      If the manual install breaks your site, remove `newrelic.ini` and the `newrelic.so` symlink, then restart your web server to restore the previous state. For troubleshooting help, see [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php) or [get support](https://support.newrelic.com).
    </Callout>
  </Step>
</Steps>

**Your app is reporting data.** Need to tune beyond the defaults? The [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration) reference covers every available setting.
