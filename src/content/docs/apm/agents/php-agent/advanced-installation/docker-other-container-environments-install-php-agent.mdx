The PHP agent works inside Docker containers, but it requires both its components — the PHP extension and the daemon — to be present and communicating. How you arrange those two components is the key decision.

## Prerequisites [#prerequisites]

Check these requirements before you start. If your environment doesn't match, you'll hit a blocker mid-install.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>Requirement</th>
      <th>Details</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>**New Relic account**</td>
      <td>You need a [New Relic account](https://newrelic.com/signup) and your <InlinePopover type="licenseKey"/>. Have it ready before you start — the Dockerfile injects it at runtime via an environment variable.</td>
    </tr>
    <tr>
      <td>**Container runtime**</td>
      <td>Docker Engine 20.10+ (or a compatible runtime such as Podman). The examples on this page use `docker` CLI syntax.</td>
    </tr>
    <tr>
      <td>**PHP version**</td>
      <td>PHP 7.2 or higher. See [PHP agent compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) for the full version list. Alpine Linux (musl libc) is supported.</td>
    </tr>
    <tr>
      <td>**Internet access**</td>
      <td>Required during image build to download the agent tarball from `download.newrelic.com`. If your build host is air-gapped, download the tarball separately and `COPY` it into the Dockerfile.</td>
    </tr>
    <tr>
      <td>**Base image**</td>
      <td>Any official PHP image (`php:fpm`, `php:apache`, etc.) or a custom image with PHP already installed. The install script detects the PHP version automatically.</td>
    </tr>
  </tbody>
</table>

## Option 1: Same container (recommended) [#install-same-container]

Install the agent and daemon together in a single image. This is the simplest setup and works for most deployments.

<Steps>
  <Step>
    ## Build the image [#same-container-build]

    The Dockerfile below installs the latest PHP agent and configures it to read your license key and app name from environment variables at runtime. Copy this into your project and extend it as needed.

    ```dockerfile
    FROM php:fpm

    # Install the latest New Relic PHP agent
    RUN \
        cd /tmp \
        # Discover the latest released version
        && export NEW_RELIC_AGENT_VERSION=$(curl -s https://download.newrelic.com/php_agent/release/ | grep -o '[1-9][0-9]\?\(\.[0-9]\+\)\{3\}' | head -n1) \
        # Detect libc type (glibc vs musl/Alpine)
        && export NR_INSTALL_PLATFORM=$(ldd --version 2>&1 | grep -q musl && echo "linux-musl" || echo "linux") \
        # Download the agent
        && curl -o newrelic-php-agent.tar.gz https://download.newrelic.com/php_agent/release/newrelic-php5-${NEW_RELIC_AGENT_VERSION}-${NR_INSTALL_PLATFORM}.tar.gz \
        # Install
        && tar xzf newrelic-php-agent.tar.gz \
        && NR_INSTALL_USE_CP_NOT_LN=1 NR_INSTALL_SILENT=0 ./*/newrelic-install install \
        # Wire up env var substitution for license key and app name
        && sed -ie 's/[ ;]*newrelic.license[[:space:]]=.*/newrelic.license=${NEW_RELIC_LICENSE_KEY}/' $(php-config --ini-dir)/newrelic.ini \
        && sed -ie 's/[ ;]*newrelic.appname[[:space:]]=.*/newrelic.appname=${NEW_RELIC_APP_NAME}/' $(php-config --ini-dir)/newrelic.ini \
        # Clean up
        && rm newrelic-php-agent.tar.gz && rm -rf newrelic-php5-*-linux
    ```

    <Callout variant="important">
      **Alpine Linux?** Use `newrelic-php5-${NEW_RELIC_AGENT_VERSION}-linux-musl.tar.gz` instead. The `ldd --version` check above handles this automatically.
    </Callout>
  </Step>

  <Step>
    ## Set required environment variables [#same-container-env]

    When you run a container from this image, pass exactly two required variables:

    ```bash
    docker run \
      -e NEW_RELIC_LICENSE_KEY="YOUR_LICENSE_KEY" \
      -e NEW_RELIC_APP_NAME="My PHP App" \
      your-image-name
    ```

    | Variable | Required? | What it does |
    |---|---|---|
    | `NEW_RELIC_LICENSE_KEY` | **Required** | Authenticates your agent with New Relic. Find it in **Account settings → API keys**. |
    | `NEW_RELIC_APP_NAME` | **Required** | Names your app in the APM UI. All containers sharing the same name report as one application. |
  </Step>

  <Step>
    ## Configure your app (optional) [#same-container-configure]

    Pass additional environment variables to tune agent behavior. These are the settings most teams adjust:

    | Environment variable | Default | When to change |
    |---|---|---|
    | `NEW_RELIC_LOG_LEVEL` | `info` | Set to `debug` temporarily if data isn't appearing |
    | `NEW_RELIC_TRANSACTION_TRACER_RECORD_SQL` | `obfuscated` | Set to `off` if you have strict data policies |
    | `NEW_RELIC_ERROR_COLLECTOR_ENABLED` | `true` | Set to `false` only if you handle all error reporting yourself |
    | `NEW_RELIC_DISTRIBUTED_TRACING_ENABLED` | `true` | Keep enabled — required for end-to-end tracing across services |

    For the full list of supported environment variables, see [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration).
  </Step>

  <Step>
    ## Prevent first-transaction data loss [#same-container-timeout]

    By default, the agent triggers daemon startup on the first request and does not wait for the connection to complete. This causes the first transaction to be dropped. To prevent this, add these two settings to `newrelic.ini` (or inject them as additional `sed` commands in your Dockerfile):

    ```ini
    newrelic.daemon.start_timeout = 5s
    newrelic.daemon.app_connect_timeout = 15s
    ```

    These tell the agent to wait up to 5 seconds for the daemon to start and 15 seconds for the app connection — so no transactions are lost on container startup.
  </Step>

  <Step>
    ## Verify data is flowing [#same-container-verify]

    1. Start the container and send at least two requests to your application.
    2. The daemon requires two transactions on first start before it begins reporting data — this is expected.
    3. Wait a few minutes, then open [New Relic APM](https://one.newrelic.com/apm) and confirm your app appears.

    <Callout variant="tip">
      No data after a few minutes? See [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php).
    </Callout>
  </Step>
</Steps>

## Option 2: Separate containers [#install-diff-containers]

Run the PHP agent and daemon in separate containers. This is useful when you have many short-lived app containers that would waste resources spinning up a daemon each, or when you want centralized daemon management.

<Callout variant="caution">
  **Security note:** Data between the agent and daemon is **not encrypted**. If agent and daemon containers are on different hosts or an untrusted network, use a private network connection between them.
</Callout>

<Steps>
  <Step>
    ## Set up the daemon container [#daemon-container]

    **Option A — Use the official New Relic daemon image from Docker Hub:**

    ```bash
    docker pull newrelic/php-daemon
    docker run --name nr-daemon \
      -p 31339:31339 \
      newrelic/php-daemon
    ```

    **Option B — Build your own daemon image:**

    ```bash
    # Install the daemon only (no PHP extension)
    ./newrelic-install install_daemon

    # Start the daemon listening on a TCP port
    newrelic-daemon --address=0.0.0.0:31339 --watchdog-foreground
    ```

    The `--address` flag sets the port the daemon listens on. The `--watchdog-foreground` flag keeps the daemon in the foreground so Docker can manage the process.
  </Step>

  <Step>
    ## Configure the agent container [#agent-container]

    Install the PHP agent in your app container exactly as you would for a same-container setup (steps 1–2 above), then add one additional configuration in `newrelic.ini`:

    ```ini
    ; Required: Point the agent at your daemon container
    ; Replace "nr-daemon" with the hostname or IP of your daemon container
    ; Replace 31339 with the port you configured in Step 1
    newrelic.daemon.address = "nr-daemon:31339"

    ; Required as always
    newrelic.license = "YOUR_LICENSE_KEY"
    newrelic.appname = "My PHP App"
    ```

    | Setting | Required? | Notes |
    |---|---|---|
    | `newrelic.daemon.address` | **Required for separate containers** | Must match the `--address` on the daemon. Use `HOST:PORT` format. |
    | `newrelic.license` | **Required** | Can be set via env var (see Option 1 Dockerfile). |
    | `newrelic.appname` | **Required** | Can be set via env var (see Option 1 Dockerfile). |
  </Step>

  <Step>
    ## Verify the connection [#diff-container-verify]

    1. Start the daemon container first. The daemon must be running before app containers try to connect.
    2. Start the app container and send at least two requests.
    3. Check daemon logs for connection messages: `docker logs nr-daemon`
    4. Wait a few minutes, then confirm your app appears in [New Relic APM](https://one.newrelic.com/apm).

    <Callout variant="important">
      **Container startup order matters.** The daemon must be ready before your PHP application starts processing requests. In Docker Compose, use `depends_on` with a health check on the daemon container. In Kubernetes, use an init container or readiness probe.
    </Callout>
  </Step>
</Steps>
