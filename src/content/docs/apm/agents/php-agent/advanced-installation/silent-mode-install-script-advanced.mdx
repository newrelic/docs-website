When deploying New Relic via Puppet, Chef, Ansible, or any other configuration management system, the `newrelic-install` script can run in **silent mode** — no interactive prompts, fully controlled by environment variables.

## Prerequisites [#prerequisites]

Check these requirements before you start.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>Requirement</th>
      <th>Details</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>**New Relic account**</td>
      <td>You need a [New Relic account](https://newrelic.com/signup) and your <InlinePopover type="licenseKey"/>. Set it as `NR_INSTALL_KEY` before running the script.</td>
    </tr>
    <tr>
      <td>**Configuration management tool**</td>
      <td>Puppet, Chef, Ansible, or any tool that can set environment variables and run shell commands on target hosts.</td>
    </tr>
    <tr>
      <td>**PHP agent package**</td>
      <td>The `newrelic-install` script must be available on target hosts — either via package manager or tar archive. See the [PHP installation overview](/docs/agents/php-agent/installation/php-agent-installation-overview) for options.</td>
    </tr>
    <tr>
      <td>**PHP version**</td>
      <td>PHP 7.2 or higher on target hosts. See [PHP agent compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).</td>
    </tr>
    <tr>
      <td>**Permissions**</td>
      <td>`sudo` or root access on target hosts — required to write to system directories and restart services.</td>
    </tr>
  </tbody>
</table>

<Steps>
  <Step>
    ## Choose a distribution method [#install-choose]

    Before running the script, decide how you're distributing the agent package to target hosts:

    | Method | When to use it |
    |---|---|
    | **Native package manager** (`rpm` or `dpkg`) | You're already managing packages with Puppet/Chef and want standard package lifecycle (install, update, remove). Note: Ubuntu post-install scripts create extension symlinks automatically but do **not** call `newrelic-install` — your automation must do that explicitly. |
    | **Tar archive** | Non-standard PHP locations, custom PHP stacks, or when you need to place files in a specific directory regardless of package manager. |
  </Step>

  <Step>
    ## Run the silent install [#minimum-install]

    Two environment variables are required. Set them before calling `newrelic-install`:

    ```bash
    export NR_INSTALL_SILENT=1
    export NR_INSTALL_KEY="YOUR_LICENSE_KEY"

    newrelic-install install
    ```

    <Callout variant="important">
      `NR_INSTALL_SILENT` must be set to suppress interactive prompts. Without it, the script blocks waiting for user input and your automation run will hang.
    </Callout>

    For additional environment variables to control install behavior (PHP path, architecture, daemon location), see the [environment variable reference](#install-environment) below.
  </Step>

  <Step>
    ## Configure your app [#post-install-config]

    The script creates `newrelic.ini` with your license key populated. You still need to set at minimum:

    ```ini
    ; Required — all apps with the same name report together in New Relic
    newrelic.appname = "My Application"
    ```

    Common optional settings to manage via config management:

    ```ini
    ; Disable the agent without uninstalling (useful for staging)
    newrelic.enabled = true

    ; Log verbosity — leave at "info" in production
    newrelic.loglevel = "info"

    ; Labels for filtering in the APM UI
    newrelic.labels = "env:production;team:backend"
    ```

    For the full list of settings, see [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration).
  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    After your configuration management run completes:

    1. Restart the web server on target hosts.
    2. Send at least **two requests** to the application — the daemon requires two transactions on first start.
    3. Run on a target host to confirm the agent loaded:

       ```bash
       php -r "phpinfo(INFO_GENERAL);" | grep -i newrelic
       ```

    4. Wait a few minutes, then confirm the app appears in [New Relic APM](https://one.newrelic.com/apm).

    <Callout variant="tip">
      No data after a few minutes? See [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php).
    </Callout>
  </Step>
</Steps>

## Environment variable reference [#install-environment]

Set these before calling `newrelic-install`. Variables marked **Required** must be set for silent mode to work.

<Callout variant="important">
  `NR_INSTALL_SILENT` must be set to run in silent mode. Without it, the script prompts for user input and blocks your automation.
</Callout>

<CollapserGroup>
  <Collapser
    id="NR_INSTALL_SILENT"
    title="NR_INSTALL_SILENT — Required"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>flag</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none — silent mode is off by default)</td>
        </tr>
      </tbody>
    </table>

    Set to any value to enable silent mode. Suppresses all status messages and skips all user prompts. When set, you must also invoke the script with either `install` or `uninstall` as the command line argument.
  </Collapser>

  <Collapser
    id="NR_INSTALL_KEY"
    title="NR_INSTALL_KEY — Required for new installs"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>string</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none)</td>
        </tr>
      </tbody>
    </table>

    Set to your New Relic <InlinePopover type="licenseKey"/>. Required when creating a new `newrelic.ini` file. If `newrelic.ini` already exists on the target host (for example, during an upgrade), this variable is ignored and the existing license key is preserved.
  </Collapser>

  <Collapser
    id="NR_INSTALL_PHPLIST"
    title="NR_INSTALL_PHPLIST — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>colon-separated list of directories</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none — auto-detected from PATH)</td>
        </tr>
      </tbody>
    </table>

    Explicit list of directories in which to install the agent. When set, this overrides PATH-based detection — only directories in this list are targeted. Use when you have multiple PHP installs and want to control exactly which ones get the agent.
  </Collapser>

  <Collapser
    id="NR_INSTALL_PATH"
    title="NR_INSTALL_PATH — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>colon-separated list of directories</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none)</td>
        </tr>
      </tbody>
    </table>

    Adds directories to `$PATH` before the script searches for PHP installations. Use when PHP is installed in a non-standard location that isn't already on the system PATH during your automation run.
  </Collapser>

  <Collapser
    id="NR_INSTALL_NOKSH"
    title="NR_INSTALL_NOKSH — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>flag</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none)</td>
        </tr>
      </tbody>
    </table>

    Prevents the script from re-executing itself under bash or ksh. By default the script re-execs under a known-compatible shell. Set this if your `/bin/sh` is already bash or ksh and you want to skip the re-exec step.
  </Collapser>

  <Collapser
    id="NR_INSTALL_SHELL"
    title="NR_INSTALL_SHELL — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>path</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none)</td>
        </tr>
      </tbody>
    </table>

    Path to a replacement shell for the script to re-exec under. Use on systems where `/bin/sh` is a very old Bourne shell or a pre-3.x Zsh that lacks required features. Ignored if `NR_INSTALL_NOKSH` is set.
  </Collapser>

  <Collapser
    id="NR_INSTALL_ARCH"
    title="NR_INSTALL_ARCH — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>string (`x86` or `x64`)</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(auto-detected)</td>
        </tr>
      </tbody>
    </table>

    Overrides automatic architecture detection. Rarely needed — only set this if auto-detection produces the wrong result on your target system.
  </Collapser>

  <Collapser
    id="NR_INSTALL_INITSCRIPT"
    title="NR_INSTALL_INITSCRIPT — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>string</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none — uses default daemon startup script name)</td>
        </tr>
      </tbody>
    </table>

    Name of the daemon startup script to install. Only needed if you're replacing the default `newrelic-daemon` init script name with a custom one.
  </Collapser>

  <Collapser
    id="NR_INSTALL_DAEMONPATH"
    title="NR_INSTALL_DAEMONPATH — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>path</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>`/usr/bin/newrelic-daemon` (Linux); `/opt/newrelic/bin/newrelic-daemon` (Solaris)</td>
        </tr>
      </tbody>
    </table>

    Full install path for the daemon binary. Change this only if you need the daemon in a non-default location. If changed, you may also need to update init scripts or their configuration files.
  </Collapser>

  <Collapser
    id="NR_INSTALL_USE_CP_NOT_LN"
    title="NR_INSTALL_USE_CP_NOT_LN — Optional"
  >
    <table>
      <tbody>
        <tr>
          <th>Type:</th>
          <td>flag</td>
        </tr>
        <tr>
          <th>Default:</th>
          <td>(none — uses symlinks by default)</td>
        </tr>
      </tbody>
    </table>

    Copies the agent into place instead of creating symbolic links. Use when the agent archive is extracted into a directory (such as root's home directory) that the PHP process cannot access via symlink traversal.
  </Collapser>
</CollapserGroup>
