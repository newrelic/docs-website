## Prerequisites [#prerequisites]

Check these requirements before you start. If your environment doesn't match, you'll hit a blocker mid-install.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>Requirement</th>
      <th>Details</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>**New Relic account**</td>
      <td>You need a [New Relic account](https://newrelic.com/signup) and your <InlinePopover type="licenseKey"/>. The install script prompts for it — have it ready before you begin.</td>
    </tr>
    <tr>
      <td>**Operating system**</td>
      <td>Ubuntu or Debian on **x86_64** only. Running aarch64? Use the [tar file method](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file) instead — apt/dpkg repos only carry x86_64 builds.</td>
    </tr>
    <tr>
      <td>**PHP version**</td>
      <td>PHP 7.2 or higher. See [PHP agent compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) for the full supported version list.</td>
    </tr>
    <tr>
      <td>**Web server**</td>
      <td>Apache (mod_php), Nginx + PHP-FPM, or a standalone PHP-FPM pool. CLI scripts are also instrumented automatically.</td>
    </tr>
    <tr>
      <td>**Permissions**</td>
      <td>`sudo` or root access required — needed to install packages and run the `newrelic-install` script.</td>
    </tr>
    <tr>
      <td>**Package manager**</td>
      <td>`apt` or `dpkg` must be available. If neither works for your setup, use the [tar file method](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file) instead.</td>
    </tr>
  </tbody>
</table>

<Steps>
  <Step>
    ## Install the agent [#install]

    The package is named `newrelic-php5` for historical reasons but works for all [supported PHP versions](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).

    1. Have your <InlinePopover type="licenseKey"/> ready.

    2. Add the New Relic repository and install:

       <CollapserGroup>
         <Collapser
           id="manager-apt"
           title="apt (recommended)"
         >
           **Step 1 — Trust the New Relic GPG key**

           For all Ubuntu releases and Debian releases _before_ Debian Trixie:

           ```bash
           wget -O- https://download.newrelic.com/NEWRELIC_APT_2DAD550E.public | sudo gpg --import --batch --no-default-keyring --keyring /usr/share/keyrings/download.newrelic.com-newrelic.gpg
           ```

           For Debian Trixie and later:

           ```bash
           wget -O- https://download.newrelic.com/NEWRELIC_APT_2DAD550E.public | sudo gpg --dearmor -o /usr/share/keyrings/download.newrelic.com-newrelic.gpg
           ```

           **Step 2 — Add the New Relic apt repository**

           ```bash
           echo 'deb [signed-by=/usr/share/keyrings/download.newrelic.com-newrelic.gpg] http://apt.newrelic.com/debian/ newrelic non-free' | sudo tee /etc/apt/sources.list.d/newrelic.list
           ```

           **Step 3 — Update and install**

           ```bash
           sudo apt-get update
           sudo apt-get install newrelic-php5
           ```

           <Callout variant="tip">
             If `apt-get update` returns a signature error mentioning `548C16BF` or `32FA4C172DAD550E`, re-run the GPG key import step above — the key is missing or outdated.
           </Callout>
         </Collapser>

         <Collapser
           id="manager-dpkg"
           title="dpkg (manual package download)"
         >
           Download the three `.deb` packages from [New Relic's apt repository](https://apt.newrelic.com/debian/dists/newrelic/non-free/binary-amd64/), replacing `X.X.X.X` with the [latest version](/docs/release-notes/agent-release-notes/php-release-notes):

           * `newrelic-php5-common_X.X.X.X_all.deb`
           * `newrelic-daemon_X.X.X.X_amd64.deb`
           * `newrelic-php5_X.X.X.X_amd64.deb`

           Then install all three:

           ```bash
           sudo dpkg -i newrelic-php5-common_X.X.X.X_all.deb newrelic-daemon_X.X.X.X_amd64.deb newrelic-php5_X.X.X.X_amd64.deb
           ```

           <Callout variant="caution">
             With `dpkg` you must manually download and install future updates. Use `apt` if you want automatic updates.
           </Callout>
         </Collapser>

         <Collapser
           id="manager-tarball"
           title="tarball (if apt/dpkg are unavailable)"
         >
           If neither apt nor dpkg work for your setup, [install from the binary tarball](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file).
         </Collapser>
       </CollapserGroup>

    3. Run the install script. It prompts for your license key and sets up `newrelic.ini` automatically:

       ```bash
       sudo newrelic-install install
       ```

       <Callout variant="tip">
         If the apt install completed without errors, `newrelic-install` may have already run automatically. Check whether `newrelic.ini` exists before running it again:
         ```bash
         php --ini | grep newrelic
         ```
       </Callout>

    4. Restart your web server:

       ```bash
       # Apache
       sudo systemctl restart apache2

       # Nginx + PHP-FPM
       sudo systemctl restart php-fpm
       sudo systemctl restart nginx
       ```

  </Step>

  <Step>
    ## Configure your app [#configure]

    The install script creates `newrelic.ini` (location varies by PHP setup) with your license key pre-filled. The only setting you **must** change before going live is the app name.

    **1. Set a unique app name**

    Open `newrelic.ini` and replace the default:

    ```ini
    newrelic.appname = "Your App Name"
    ```

    Every app that shares the same name reports its data together in New Relic. Use a name that clearly identifies this service (for example, `"Payments API - Production"`).

    Can't find `newrelic.ini`? Run:

    ```bash
    php --ini | grep newrelic
    ```

    **2. Confirm your license key**

    The install script should have set this, but verify it is correct:

    ```ini
    newrelic.license = "YOUR_LICENSE_KEY"
    ```

    **3. Common optional settings**

    These are the settings most teams adjust after install — no need to leave this page:

    | Setting | Default | When to change |
    |---|---|---|
    | `newrelic.loglevel` | `"info"` | Set to `"debug"` temporarily if data isn't appearing |
    | `newrelic.transaction_tracer.record_sql` | `"obfuscated"` | Set to `"off"` if you have strict data policies; never use `"raw"` in production |
    | `newrelic.error_collector.enabled` | `true` | Set to `false` only if you handle all error reporting yourself |
    | `newrelic.distributed_tracing_enabled` | `true` | Keep enabled — required for end-to-end tracing across services |

    For the full settings reference, see [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration).
  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    1. Send at least **two requests** to your application — the PHP daemon requires two transactions on first start before it begins reporting.

    2. Wait 2–3 minutes, then open [one.newrelic.com > APM](https://one.newrelic.com/apm) and look for your app by name.

    <img
      title="APM Summary page showing a PHP application reporting data"
      alt="The New Relic APM Summary page with transaction throughput, response time, and error rate charts visible for a PHP application."
      src="/images/apm_screenshot-full_php-guided-install-restart-webserver.png"
    />

    <figcaption>
      Once data flows, the APM Summary page shows transactions, throughput, and errors in real time.
    </figcaption>

    <Callout variant="tip">
      App not appearing after a few minutes? Run `php -r "phpinfo();" | grep -i newrelic` to confirm the extension is loaded. For other common causes, see [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php).
    </Callout>
  </Step>
</Steps>

## Optional: Headless installation [#preseeding]

For automated or unattended deployments, pre-seed your license key and app name into `debconf` before installing — this skips interactive prompts entirely:

```bash
echo newrelic-php5 newrelic-php5/license-key string "YOUR_LICENSE_KEY" | sudo debconf-set-selections
echo newrelic-php5 newrelic-php5/application-name string "Your App Name" | sudo debconf-set-selections
DEBIAN_FRONTEND=noninteractive sudo apt-get install -y newrelic-php5
```

<Callout variant="important">
  `debconf` preseeding only works when using the standard PHP packages provided by your distribution. If you're on a custom PHP build, the `newrelic-install` script prompt cannot be bypassed this way.
</Callout>
