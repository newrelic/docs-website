## Install the agent [#install]

Even though the package name for New Relic's PHP agent refers to PHP 5, the package works for all [supported PHP versions](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).

1. Make sure you have your <InlinePopover type="licenseKey"/> accessible.
2. Add the new RPM key
   ```bash
   rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public
   ```
3. Use either of the following ways to obtain the installation package:

   <CollapserGroup>
     <Collapser
       id="tell-rpm"
       title={<>Tell the package manager (<InlineCode>rpm</InlineCode>) about the New Relic repository</>}
     >
       ```bash
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-4.noarch.rpm
       ```
     </Collapser>

     <Collapser
       id="download-rpm"
       title="Download the rpm file from New Relic"
     >
       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```

       * ```
         newrelic-daemon-X.X.X.X-1.x86_64.rpm
         ```

       * ```
         newrelic-php5-X.X.X.X-1.x86_64.rpm
         ```
     </Collapser>
   </CollapserGroup>
4. Install the agent and daemon using your preferred package manager:

   <CollapserGroup>
     <Collapser
       id="manager-yum"
       title="yum"
     >
       ```bash
       sudo yum install newrelic-php5
       ```

       The first time you install New Relic for PHP, yum prompts you to accept the New Relic public key. New Relic's key ID is `548C16BF`.
     </Collapser>

     <Collapser
       id="manager-rpm"
       title="rpm"
     >
       Replace `X.X.X.X` with the most recent [PHP agent version number](/docs/release-notes/agent-release-notes/php-release-notes) when you run this command:

       ```bash
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="manager-tarball"
       title="tarball"
     >
       If <DNT>**yum**</DNT> and <DNT>**rpm**</DNT> do not work with your host config, [install from the binary tarball](/docs/php/php-agent-installation-tar-files).
     </Collapser>
   </CollapserGroup>
5. Run the `newrelic-install` script and follow the instructions.

   ```bash
   sudo newrelic-install install
   ```
6. Restart your web server (Apache, NGINX, PHP-FPM, etc.).
7. Generate traffic to your application, and wait a few minutes for it to send data to New Relic. Then, [check your app's performance in the UI](/docs/apm/applications-menu/monitoring/apm-overview-page).

## Name and configure your app [#configure]

**Required: Name your application**

[Give your app a meaningful name](/docs/apm/agents/php-agent/configuration/name-your-php-application). This is how New Relic groups your metrics — if you skip this, all apps report under the default name and become impossible to tell apart.

You can set it directly in `newrelic.ini`:

```ini
newrelic.appname = "Your App Name"
```

Or use this command to set it automatically (replace `YOUR_APP_NAME`):

```bash
sudo find /etc/php.d /etc/php /etc/php.ini -name 'newrelic.ini' -type f 2>/dev/null | \
  xargs sudo sed -i 's/newrelic.appname = "PHP Application"/newrelic.appname = "YOUR_APP_NAME"/g'
```

**Before production: Review key settings**

Check your [transaction tracing settings](/docs/apm/agents/php-agent/configuration/php-agent-configuration#inivar-tt-settings) and [error collection](/docs/apm/agents/php-agent/configuration/php-agent-configuration#inivar-error-collector) before going live. See the full [PHP agent configuration reference](/docs/apm/agents/php-agent/configuration/php-agent-configuration) for all options.

## Verify data is flowing [#verify]

1. Restart your web server (Apache, Nginx, PHP-FPM, etc.).
2. Send at least two requests to your application — the PHP daemon requires two transactions on first start before it begins reporting data.
3. Run the following command to manually trigger the agent and confirm it initializes:

   ```bash
   php -r "phpinfo(INFO_GENERAL);"
   ```

4. Wait a few minutes, then [check your app's performance in New Relic](/docs/apm/applications-menu/monitoring/apm-overview-page).
