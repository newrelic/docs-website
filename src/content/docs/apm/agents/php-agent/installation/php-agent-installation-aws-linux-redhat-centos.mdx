<Callout variant="important">
    The New Relic PHP Agent RPM signing key was rotated in March 2026. To continue using the agent uninterrupted, the new key must be imported on all systems installing via rpm/yum with the following command:

```rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public```

For more information, refer to the [PHP agent installation: RPM key rotation 2026](/docs/apm/agents/php-agent/installation/php-agent-installation-rpm-key-rotation) documentation.
</Callout>

Our PHP agent auto-instruments your code so you can start monitoring applications. This page covers installation on Amazon Linux 2, RedHat, or CentOS (x86_64).


## Prerequisites [#prerequisites]

Check these requirements before you start. If your environment doesn't match, you'll hit a blocker mid-install.

  <thead>
    <tr>

    </tr>
  </thead>

    <tr>
      <td>**New Relic account**</td>

    </tr>

<Steps>
  <Step>
    ## Install the agent [#install]

    The package is named `newrelic-php5` for historical reasons but works for all [supported PHP versions](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).

    1. Import the New Relic RPM signing key:

       ```bash
       sudo rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public
       ```

    2. Install the agent package using your preferred method:

    <CollapserGroup>
      <Collapser
        id="manager-yum"
        title="yum (recommended)"
      >
        ```bash
        sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-4.noarch.rpm
        sudo yum install newrelic-php5
        ```

        The first time you install, yum prompts you to accept the New Relic public key (key ID `548C16BF`).
      </Collapser>

      <Collapser
        id="manager-rpm"
        title="rpm (manual package download)"
      >
        Download these three packages from New Relic, replacing `X.X.X.X` with the [latest version](/docs/release-notes/agent-release-notes/php-release-notes):

        * `newrelic-php5-common-X.X.X.X-1.noarch.rpm`
        * `newrelic-daemon-X.X.X.X-1.x86_64.rpm`
        * `newrelic-php5-X.X.X.X-1.x86_64.rpm`

        Then install all three:

        ```bash
        sudo rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
        ```
      </Collapser>

      <Collapser
        id="manager-tarball"
        title="tarball (if yum/rpm are unavailable)"
      >
        If neither yum nor rpm work for your setup, [install from the binary tarball](/docs/php/php-agent-installation-tar-files).
      </Collapser>
    </CollapserGroup>

    3. Run the install script. When prompted, enter your <InlinePopover type="licenseKey"/>. The script sets up `newrelic.ini` automatically:

       ```bash
       sudo newrelic-install install
       ```

    4. Restart your web server:

       ```bash
       # Apache
       sudo systemctl restart httpd

       # Nginx + PHP-FPM
       sudo systemctl restart php-fpm
       sudo systemctl restart nginx
       ```
  </Step>

  <Step>
    ## Configure your app [#configure]

    The install script creates `/etc/php.d/newrelic.ini` (or the equivalent for your PHP setup) with your license key pre-filled. The only setting you **must** change before going live is the app name.

    1. Open `newrelic.ini` and set a unique app name:

       ```ini
       newrelic.appname = "Your App Name"
       ```

       Every app that shares the same name reports its data together in New Relic. Use a name that clearly identifies this service (for example, `"Payments API - Production"`).

       Can't find `newrelic.ini`? Run:

       ```bash
       php --ini | grep newrelic
       ```

  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    1. Send at least **two requests** to your application — the PHP daemon requires two transactions on first start before it begins reporting.

    2. Wait 2–3 minutes, then open [one.newrelic.com > APM](https://one.newrelic.com/apm) and look for your app by name.

    <Callout variant="tip">
      App not appearing after a few minutes? Run `php -r "phpinfo();" | grep -i newrelic` to confirm the extension is loaded. For other common causes, see [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php).
    </Callout>
  </Step>
</Steps>

## Common optional settings [#optional-settings]

Most teams adjust these after verifying data is flowing — no need to leave this page:

| Setting | Default | When to change |
|---|---|---|
| `newrelic.loglevel` | `"info"` | Set to `"debug"` temporarily if data isn't appearing |
| `newrelic.transaction_tracer.record_sql` | `"obfuscated"` | Set to `"off"` if you have strict data policies; never use `"raw"` in production |
| `newrelic.error_collector.enabled` | `true` | Set to `false` only if you handle all error reporting yourself |
| `newrelic.distributed_tracing_enabled` | `true` | Keep enabled — required for end-to-end tracing across services |

For the full settings reference, see [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration).
