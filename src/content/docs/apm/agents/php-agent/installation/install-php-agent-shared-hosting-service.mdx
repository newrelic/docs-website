Basic shared hosting plans typically do not allow installation of the New Relic PHP agent. Installing the agent requires administrator (root) permissions that most shared hosting plans do not grant.

## Hosting providers that do not support the PHP agent [#hosts]

The following providers are known to block PHP agent installation at certain subscription levels:

| Provider | Affected plans |
|---|---|
| DreamHost | Shared Web Hosting |
| WordPress.com | All hosted blog plans |
| GoDaddy | Economy, Deluxe, Ultimate, WordPress Blog Hosting |
| HostGator | Shared Web Hosting |

This list is not exhaustive. If your provider is not listed, contact them directly to ask whether root command-line access is available.

<Callout variant="important">
  Windows operating systems are not supported by the PHP agent on any hosting type.
</Callout>

## Check if your plan supports installation [#check]

Before contacting support, verify the following:

1. Confirm your system meets New Relic's [PHP compatibility requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).
2. Check that your hosting plan grants root (or `sudo`) command-line access.
3. Attempt the standard [PHP installation procedure](/docs/agents/php-agent/installation/php-agent-installation-overview).
4. If installation fails, gather the following before contacting your host or New Relic support:
   - Exact steps attempted
   - Success or failure messages
   - OS and version (for example, Ubuntu Linux 22.04 64-bit)
   - Hosting provider name and subscription level
