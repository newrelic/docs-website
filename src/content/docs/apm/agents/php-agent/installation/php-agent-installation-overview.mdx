## PHP components [#components]

Our [PHP agent](/docs/agents/php-agent/getting-started/new-relic-php) consists of two basic components:

* A PHP extension, which collects data from your application
* A local proxy daemon, which transmits the data to New Relic

Your system's configuration components, including the web server, operating system, system permissions, PHP installation itself, package manager, etc., are all important factors during installation and configuration, because the agent is a part of that.

For more information about supported PHP releases, frameworks, and other system requirements, see [PHP compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).

  <Step>
    ## Check prerequisites [#prerequisites]

    Before you install, make sure you have:

    * A New Relic account — [sign up free](https://newrelic.com/signup) if you don't have one
    * PHP 7.1 or higher — see [full compatibility requirements](/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements) for supported versions, frameworks, and operating systems
    * Root or sudo access on the host
    * Your <InlinePopover type="licenseKey"/> ready — find it in **Account settings**
  </Step>

  <Step>
    ## Install the agent [#install-agent]

      The fastest path is the guided install — it detects your environment and runs the setup automatically.

  <Step>
    ## Name and configure your app [#configure]

    **Required: Name your application**

    [Give your app a meaningful name](/docs/apm/agents/php-agent/configuration/name-your-php-application) in `newrelic.ini`. This is how New Relic groups your metrics — if you skip this, all apps on the account report under the default name and become impossible to tell apart.

    **Before production: Review key settings**

    Check your [transaction tracing settings](/docs/apm/agents/php-agent/configuration/php-agent-configuration#inivar-tt-settings) and [error collection](/docs/apm/agents/php-agent/configuration/php-agent-configuration#inivar-error-collector) before going live. See the full [PHP agent configuration reference](/docs/apm/agents/php-agent/configuration/php-agent-configuration) for all options.
  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    1. Restart your web server (Apache, Nginx, PHP-FPM, etc.).
    2. If your application has low traffic, send a few requests to it.
    3. Wait a few minutes, then [check your app's performance in New Relic](/docs/apm/applications-menu/monitoring/apm-overview-page).

  </Step>
