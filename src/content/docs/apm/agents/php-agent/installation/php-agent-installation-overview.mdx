## Before you install [#prerequisites]

Make sure you have:

* A New Relic account — [sign up free](https://newrelic.com/signup) if you don't have one
* PHP 7.2 or higher — see [full compatibility requirements](/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements) for supported versions, frameworks, and operating systems
* Root or sudo access on the host
* Your <InlinePopover type="licenseKey"/> ready — find it in **Account settings**

## How the PHP agent works [#components]

The PHP agent has two components that work together:

* **PHP extension** — collects data from your application
* **Local proxy daemon** — transmits the data to New Relic

Your web server, OS, system permissions, PHP installation, and package manager all factor into how these components are installed and configured. If something goes wrong during installation, knowing which component is affected — the extension (configured via `php.ini`) or the daemon (configured via `newrelic.cfg`) — helps you isolate the issue quickly. For full compatibility details, see [PHP compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).

### Standard installation [#standard-install]

<CollapserGroup>
  <Collapser
    id="install-cli"
    title="CLI / Guided install (recommended for supported Linux hosts)"
  >
    **Use when:** You're on a standard Linux host and want the fastest path to getting data into New Relic. The guided install detects your environment, injects your license key, and completes the full installation automatically — no manual steps required.

    [Start guided install →](https://one.newrelic.com/marketplace/install-data-source?state=d1b7b1cb-4089-8af0-16b2-86c7072e2cae)

    Or run it directly from your host:

    ```bash
    curl -Ls https://download.newrelic.com/install/newrelic-cli/scripts/install.sh | bash && \
    sudo NEW_RELIC_API_KEY=YOUR_API_KEY \
    NEW_RELIC_ACCOUNT_ID=YOUR_ACCOUNT_ID \
    /usr/local/bin/newrelic install -n php-agent-installer
    ```
  </Collapser>

  <Collapser
    id="install-redhat-centos"
    title="RedHat, CentOS, or Amazon Linux 2 (x86_64)"
  >
    **Use when:** Your host runs RedHat, CentOS, or Amazon Linux 2 (x86_64) and you prefer managing packages via `yum` or `rpm`. You can [tell the package manager](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#tell-rpm) to pull from the New Relic repository, or [download the RPM directly](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#download-rpm).

    <Callout variant="important">
      This guide covers **Amazon Linux 2 (x86_64) only**. Amazon Linux 2023 (both x86_64 and aarch64) is not supported via RPM — use the [tar archive method](#install-tar) instead.
    </Callout>

    [RedHat / CentOS / Amazon Linux 2 install guide →](/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos)
  </Collapser>

  <Collapser
    id="install-ubuntu-debian"
    title="Ubuntu or Debian (x86_64)"
  >
    **Use when:** Your host runs Ubuntu or Debian and you manage packages with `apt-get` or `dpkg`. This is the standard path for Debian-based systems — the `newrelic-install` script is typically not required.

    [Ubuntu / Debian install guide →](/docs/apm/agents/php-agent/installation/php-agent-installation-ubuntu-debian)
  </Collapser>

  <Collapser
    id="install-tar"
    title="Tar archive (all platforms, including aarch64)"
  >
    **Use when:** Your OS isn't covered by the package manager methods above, you're on ARM64/aarch64, you're not using a package manager, or other methods have failed. The `.tar` archive works on all [supported operating systems](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems) and is the only supported method for aarch64.

    [Tar archive install guide →](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file)
  </Collapser>

  <Collapser
    id="install-docker"
    title="Docker"
  >
    **Use when:** Your PHP application runs inside a Docker container. This method uses a Docker-specific setup to install the agent and configure the daemon within your container image.

    [Docker install guide →](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent)
  </Collapser>

  <Collapser
    id="install-shared-hosting"
    title="Shared hosting"
  >
    **Use when:** You're on a shared or managed hosting plan where root access is not available. Support depends on whether your hosting provider allows New Relic installation.

    [Shared hosting install guide →](/docs/agents/php-agent/installation/install-php-agent-shared-hosting-service)
  </Collapser>
</CollapserGroup>

### Advanced & automation [#advanced-install]

<CollapserGroup>
  <Collapser
    id="install-newrelic-script"
    title="newrelic-install script"
  >
    **Use when:** You have multiple PHP installations on the same host, or PHP is installed in a non-standard location that the guided install doesn't detect. The interactive `newrelic-install` script copies the correct PHP modules into place, installs sample `ini` files, and configures the proxy daemon across all detected PHP versions.

    The script is bundled with the package manager and tar installs:
    - **Package manager install:** script is at `/usr/bin/newrelic-install`
    - **Tar install:** script is in the directory where you extracted the archive

    [newrelic-install script guide →](/docs/apm/agents/php-agent/advanced-installation/use-newrelic-install-script-php)
  </Collapser>

  <Collapser
    id="install-non-standard"
    title="Non-standard PHP (advanced)"
  >
    **Use when:** You're running self-compiled PHP, a self-contained PHP stack, or a setup where the default installer can't find your PHP installation. Before installing, use `php -i` or `phpinfo()` to identify your PHP extension version, the extension install directory, and whether ZTS (Zend Thread Safety) is enabled.

    [Non-standard PHP install guide →](/docs/apm/agents/php-agent/advanced-installation/php-agent-installation-non-standard-php-advanced)
  </Collapser>

  <Collapser
    id="install-gae"
    title="Google App Engine (GAE) Flexible"
  >
    **Use when:** Your PHP application is deployed on Google App Engine's Flexible Environment. This setup requires adding the agent to your Docker-based GAE runtime configuration.

    [GAE Flexible install guide →](/docs/apm/agents/php-agent/advanced-installation/install-new-relic-php-agent-gae-flexible-environment)
  </Collapser>

  <Collapser
    id="install-config-mgmt"
    title="Ansible, Chef, or Puppet"
  >
    **Use when:** You manage a fleet of hosts with a configuration management tool and need a repeatable, unattended install. Each tool has its own integration:

    - **Ansible** — Install the `newrelic.newrelic_install` role from Ansible Galaxy and add it to your playbook.
    - **Chef** — Use the `newrelic-install` Chef Cookbook; replace default attributes with your account details and upload to your Chef server.
    - **Puppet** — Install the `newrelic_installer` module from Puppet Forge and instantiate the `newrelic_installer::install` class with your license key and account details.

    For unattended (silent mode) installs, the `newrelic-install` script can be driven entirely by environment variables — useful when you can't run an interactive session. See [silent mode guide →](/docs/apm/agents/php-agent/advanced-installation/silent-mode-install-script-advanced)

    [Config management install guide →](https://docs.newrelic.com/docs/infrastructure/host-integrations/installation/new-relic-guided-install-overview/)
  </Collapser>
</CollapserGroup>
