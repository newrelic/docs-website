## Before you install [#prerequisites]

Make sure you have:

* A New Relic account — [sign up free](https://newrelic.com/signup) if you don't have one
* PHP 7.2 or higher — see [full compatibility requirements](/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements) for supported versions, frameworks, and operating systems
* Root or sudo access on the host
* Your <InlinePopover type="licenseKey"/> ready — find it in **Account settings**

<CollapserGroup>
  <Collapser
    id="install-cli"
    title="CLI / Guided install (recommended for supported Linux hosts)"
  >
    The guided install runs a single command that detects your environment, injects your license key, and completes the full installation automatically.

    [Start guided install →](https://one.newrelic.com/marketplace/install-data-source?state=d1b7b1cb-4089-8af0-16b2-86c7072e2cae)

    Or run it directly from your host:

    ```bash
    curl -Ls https://download.newrelic.com/install/newrelic-cli/scripts/install.sh | bash && \
    sudo NEW_RELIC_API_KEY=YOUR_API_KEY \
    NEW_RELIC_ACCOUNT_ID=YOUR_ACCOUNT_ID \
    /usr/local/bin/newrelic install -n php-agent-installer
    ```
  </Collapser>

  <Collapser
    id="install-redhat-centos"
    title="RedHat, CentOS, or Amazon Linux 2 (x86_64)"
  >
    Use the RPM package with `yum` or `rpm`. You can [tell the package manager](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#tell-rpm) to pull from the New Relic repository, or [download the RPM directly](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#download-rpm).

    <Callout variant="important">
      This guide covers **Amazon Linux 2 (x86_64) only**. Amazon Linux 2023 (both x86_64 and aarch64) is not supported via RPM — use the [tar archive method](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file) instead.
    </Callout>

    [RedHat / CentOS / Amazon Linux 2 install guide →](/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos)
  </Collapser>

  <Collapser
    id="install-ubuntu-debian"
    title="Ubuntu or Debian (x86_64)"
  >
    Use `apt-get` or `dpkg`. Typically the `newrelic-install` script is not required for Debian-based systems.

    [Ubuntu / Debian install guide →](/docs/apm/agents/php-agent/installation/php-agent-installation-ubuntu-debian)
  </Collapser>

  <Collapser
    id="install-tar"
    title="Tar archive (all platforms, including aarch64)"
  >
    The `.tar` archive works on all [supported operating systems](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems) and is the only method for aarch64 (ARM64). Use this if your OS isn't listed above, you're not using a package manager, or other methods fail.

    [Tar archive install guide →](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file)
  </Collapser>

  <Collapser
    id="install-docker"
    title="Docker"
  >
    Run the PHP agent inside a container using the Docker-specific setup.

    [Docker install guide →](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent)
  </Collapser>

  <Collapser
    id="install-shared-hosting"
    title="Shared hosting"
  >
    If your host does not allow root access, check whether your hosting provider supports New Relic.

    [Shared hosting install guide →](/docs/agents/php-agent/installation/install-php-agent-shared-hosting-service)
  </Collapser>

  <Collapser
    id="install-ansible"
    title="Ansible"
  >
    Install the `newrelic.newrelic_install` role from Ansible Galaxy, then add it to your playbook. The role handles agent installation, app naming, and daemon configuration across your fleet.

    [Ansible install guide →](https://docs.newrelic.com/docs/infrastructure/host-integrations/installation/new-relic-guided-install-overview/)
  </Collapser>

  <Collapser
    id="install-chef"

    [Chef install guide →](https://docs.newrelic.com/docs/infrastructure/host-integrations/installation/new-relic-guided-install-overview/)

## PHP components [#components]

Our [PHP agent](/docs/agents/php-agent/getting-started/new-relic-php) consists of two basic components:

* A PHP extension, which collects data from your application
* A local proxy daemon, which transmits the data to New Relic

Your system's configuration components, including the web server, operating system, system permissions, PHP installation itself, package manager, etc., are all important factors during installation and configuration, because the agent is a part of that.

For more information about supported PHP releases, frameworks, and other system requirements, see [PHP compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).
