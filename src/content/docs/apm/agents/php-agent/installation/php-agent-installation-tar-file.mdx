You can install the PHP agent from compressed tar archives on any supported operating system and processor architecture. However, for RedHat or CentOS on x86_64 it is more common to use the [RPM package](/docs/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos/). For Ubuntu or Debian on x86_64 it is more common to use the [Debian package](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian).

## Prerequisites [#prerequisites]

Check these requirements before you start. If your environment doesn't match, you'll hit a blocker mid-install.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>Requirement</th>
      <th>Details</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>**New Relic account**</td>
      <td>You need a [New Relic account](https://newrelic.com/signup) and your <InlinePopover type="licenseKey"/>. The install script prompts for it — have it ready before you begin.</td>
    </tr>
    <tr>
      <td>**Operating system**</td>
      <td>Any supported Linux distribution on x86_64 or aarch64 (ARM64). For Alpine Linux, use the `musl` variant of the archive.</td>
    </tr>
    <tr>
      <td>**PHP version**</td>
      <td>PHP 7.2 or higher. See [PHP agent compatibility and requirements](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) for the full supported version list.</td>
    </tr>
    <tr>
      <td>**Web server**</td>
      <td>Apache (mod_php), Nginx + PHP-FPM, or a standalone PHP-FPM pool. CLI scripts are also instrumented automatically.</td>
    </tr>
    <tr>
      <td>**Permissions**</td>
      <td>`sudo` or root access required — needed to run the `newrelic-install` script.</td>
    </tr>
    <tr>
      <td>**Download tool**</td>
      <td>`curl` or `wget` must be available to download the archive from `download.newrelic.com`.</td>
    </tr>
  </tbody>
</table>

<Steps>
  <Step>
    ## Install the agent [#install]

    The package is named `newrelic-php5` for historical reasons but works for all [supported PHP versions](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release). The steps for first-time installation and updates are identical.

    1. Have your <InlinePopover type="licenseKey"/> ready.

    2. Download the appropriate archive from [download.newrelic.com/php_agent/release/](https://download.newrelic.com/php_agent/release/), replacing `X.X.X.X` with the [latest version](/docs/release-notes/agent-release-notes/php-release-notes):

       | Architecture | Archive filename |
       |---|---|
       | x86_64 (glibc) | `newrelic-php5-X.X.X.X-linux.tar.gz` |
       | x86_64 (musl / Alpine) | `newrelic-php5-X.X.X.X-linux-musl.tar.gz` |
       | aarch64 (glibc) | `newrelic-php5-X.X.X.X-linux-aarch64.tar.gz` |
       | aarch64 (musl / Alpine) | `newrelic-php5-X.X.X.X-linux-musl-aarch64.tar.gz` |

       <Callout variant="tip">
         The package name references PHP 5 for historical reasons — it supports all [currently supported PHP versions](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).
       </Callout>

    3. Decompress and extract the archive (replace `OS` with your platform variant):

       ```bash
       gzip -dc newrelic-php5-X.X.X.X-OS.tar.gz | tar xf -
       ```

    4. Change into the extracted directory:

       ```bash
       cd newrelic-php5-X.X.X.X-OS
       ```

    5. Run the install script. It prompts for your license key and sets up `newrelic.ini` automatically:

       ```bash
       sudo ./newrelic-install install
       ```

    6. Restart your web server:

       ```bash
       # Apache
       sudo systemctl restart httpd        # RHEL/CentOS
       sudo systemctl restart apache2      # Ubuntu/Debian

       # Nginx + PHP-FPM
       sudo systemctl restart php-fpm
       sudo systemctl restart nginx
       ```

    <InstallFeedback/>
  </Step>

  <Step>
    ## Configure your app [#configure]

    The install script creates `newrelic.ini` with your license key pre-filled. The only setting you **must** change before going live is the app name.

    **1. Set a unique app name**

    Open `newrelic.ini` and replace the default:

    ```ini
    newrelic.appname = "Your App Name"
    ```

    Every app that shares the same name reports its data together in New Relic. Use a name that clearly identifies this service (for example, `"Payments API - Production"`).

    Can't find `newrelic.ini`? Run:

    ```bash
    php --ini | grep newrelic
    ```

    **2. Confirm your license key**

    The install script should have set this, but verify it is correct:

    ```ini
    newrelic.license = "YOUR_LICENSE_KEY"
    ```

    **3. Common optional settings**

    These are the settings most teams adjust after install — no need to leave this page:

    | Setting | Default | When to change |
    |---|---|---|
    | `newrelic.loglevel` | `"info"` | Set to `"debug"` temporarily if data isn't appearing |
    | `newrelic.transaction_tracer.record_sql` | `"obfuscated"` | Set to `"off"` if you have strict data policies; never use `"raw"` in production |
    | `newrelic.error_collector.enabled` | `true` | Set to `false` only if you handle all error reporting yourself |
    | `newrelic.distributed_tracing_enabled` | `true` | Keep enabled — required for end-to-end tracing across services |

    For the full settings reference, see [PHP agent configuration](/docs/apm/agents/php-agent/configuration/php-agent-configuration).
  </Step>

  <Step>
    ## Verify data is flowing [#verify]

    1. Send at least **two requests** to your application — the PHP daemon requires two transactions on first start before it begins reporting.

    2. Wait 2–3 minutes, then open [one.newrelic.com > APM](https://one.newrelic.com/apm) and look for your app by name.

    <img
      title="Restart your web server — PHP guided install"
      alt="The New Relic guided install wizard showing restart commands for PHP-FPM, Apache, and Nginx with copy-to-clipboard buttons."
      src="/images/apm_screenshot-full_php-guided-install-restart-webserver.png"
    />

    <figcaption>
      Once data flows, the APM Summary page shows transactions, throughput, and errors in real time.
    </figcaption>

    <Callout variant="tip">
      App not appearing after a few minutes? Run `php -r "phpinfo();" | grep -i newrelic` to confirm the extension is loaded. For other common causes, see [No data appears (PHP)](/docs/agents/php-agent/troubleshooting/no-data-appears-php).
    </Callout>
  </Step>
</Steps>
