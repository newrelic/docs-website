After installing the PHP agent, follow these steps to verify it's sending data to New Relic. If data doesn't appear, use the troubleshooting decision tree below.

## Verification steps

### Step 1: Restart your web server

The PHP agent won't start until you restart your web server:

```bash
# Apache
sudo systemctl restart apache2
# or
sudo service apache2 restart

# Nginx + PHP-FPM
sudo systemctl restart php-fpm
sudo systemctl restart nginx

# PHP-FPM only
sudo systemctl restart php8.1-fpm  # Adjust version as needed
```

<Callout variant="important">
Restarting is required because PHP loads extensions only at startup. A simple config reload won't work.
</Callout>

### Step 2: Generate application traffic

The agent only sends data when your application handles requests:

1. Open your application in a web browser
2. Click through several pages or endpoints
3. Spend at least **30 seconds** generating traffic
4. Try different features to trigger various code paths

### Step 3: Wait for data (3-5 minutes)

The PHP agent:
- Collects data continuously
- Buffers data locally
- Sends to New Relic every **60 seconds**
- May take **3-5 minutes** for first data to appear in UI

<Callout variant="tip">
**Pro tip:** While waiting, verify the agent is loaded (see Check #1 below).
</Callout>

### Step 4: Check your APM dashboard

1. Log in to [one.newrelic.com](https://one.newrelic.com)
2. Go to <DNT>**All capabilities > APM & services**</DNT>
3. Look for your application name

## ❌ Troubleshooting: No data after 10 minutes

If data doesn't appear after 10 minutes, follow this decision tree:

### Check #1: Is the PHP extension loaded?

Verify the New Relic extension is installed and loaded by PHP:

```bash
php -m | grep newrelic
```

**Expected output:**
```
newrelic
```

<CollapserGroup>
  <Collapser title="✅ Extension is loaded → Continue to Check #2">
    Good! The extension is installed. Move on to Check #2.
  </Collapser>

  <Collapser title="❌ No output or 'newrelic' not found">
    The extension isn't loaded. This means:

    **Possible causes:**
    1. Installation didn't complete successfully
    2. Wrong PHP version (you have multiple PHP installations)
    3. Extension file not in the correct location

    **Solutions:**

    1. **Verify installation files exist:**
       ```bash
       # Find PHP extension directory
       php -i | grep extension_dir

       # Check if newrelic.so exists there
       ls -la /path/to/extensions/ | grep newrelic
       ```

    2. **Check which PHP version you're running:**
       ```bash
       php -v  # CLI version
       ```

       Then check your web server's PHP version:
       ```php
       # Create a test file: /var/www/html/info.php
       <?php phpinfo(); ?>
       ```
       Visit it in your browser and check the version.

    3. **Verify newrelic.ini is loaded:**
       ```bash
       php --ini | grep newrelic
       ```

       If missing, check:
       - `/etc/php/8.1/mods-available/newrelic.ini` (Ubuntu/Debian)
       - `/etc/php.d/newrelic.ini` (RedHat/CentOS)

    4. **Reinstall the agent:**
       - Return to [installation overview](/docs/apm/agents/php-agent/getting-started/installation-overview)
       - Make sure to run the `newrelic-install` script
       - Select the correct PHP version when prompted
  </Collapser>
</CollapserGroup>

---

### Check #2: Is the daemon running?

The New Relic daemon must be running to send data:

```bash
ps aux | grep newrelic-daemon
```

**Expected output:**
```
root  12345  /usr/bin/newrelic-daemon ...
```

<CollapserGroup>
  <Collapser title="✅ Daemon is running → Continue to Check #3">
    The daemon is up. Proceed to Check #3.
  </Collapser>

  <Collapser title="❌ Daemon is not running">
    **Solution - Start the daemon:**

    The daemon usually starts automatically, but you can start it manually:

    ```bash
    # Start daemon
    sudo /etc/init.d/newrelic-daemon start
    # or
    sudo systemctl start newrelic-daemon
    ```

    If it won't start, check daemon logs:
    ```bash
    sudo tail -100 /var/log/newrelic/newrelic-daemon.log
    ```

    **Common daemon issues:**

    - **License key error:** Check your license key in `/etc/newrelic/newrelic.cfg`
    - **Permission denied:** Daemon needs write access to `/var/log/newrelic/`
    - **Port already in use:** Another process is using port 3142 (default daemon port)

    See [starting the daemon](/docs/apm/agents/php-agent/advanced-installation/starting-php-daemon-advanced) for advanced configuration.
  </Collapser>
</CollapserGroup>

### Check #3: Is the agent connecting?

Check the PHP agent log for connection messages:

```bash
sudo tail -50 /var/log/newrelic/php_agent.log
```

**Look for these messages:**

<CollapserGroup>
  <Collapser title='✅ "app connected" - Good!'>
    If you see:
    ```
    "app":"MyApp" "connect":"ok"
    ```

    The agent is successfully connecting. If data still doesn't appear, try:

    1. **Check application name:**
       - The app name in logs should match what you expect
       - Check `newrelic.appname` in `newrelic.ini`
       - [Name your application](/docs/apm/agents/php-agent/configuration/name-your-php-application)

    2. **Verify account/region:**
       - Confirm you're checking the correct New Relic account
       - If you're in EU, use `one.eu.newrelic.com`

    3. **Generate more traffic:**
       - Agent may be sampling or throttling
       - Create sustained traffic for 2-3 minutes
  </Collapser>

  <Collapser title='❌ "license key invalid"'>
    **Solution:**

    1. **Verify your license key:**
       - Get the correct key from [one.newrelic.com/launcher/api-keys-ui.api-keys-launcher](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher)
       - It should be a 40-character hexadecimal string

    2. **Update the license key:**

       Edit `/etc/php/8.1/mods-available/newrelic.ini` (path may vary):
       ```ini
       newrelic.license = "YOUR_LICENSE_KEY_HERE"
       ```

       Or set via environment variable:
       ```bash
       export NEWRELIC_LICENSE_KEY="YOUR_LICENSE_KEY_HERE"
       ```

    3. **Restart everything:**
       ```bash
       sudo systemctl restart newrelic-daemon
       sudo systemctl restart apache2  # or your web server
       ```
  </Collapser>

  <Collapser title='❌ "daemon connection failed"'>
    The agent can't reach the daemon.

    **Solutions:**

    1. **Verify daemon is running:** See Check #2

    2. **Check daemon socket/port:**

       In `newrelic.ini`:
       ```ini
       ; Default: /tmp/.newrelic.sock
       newrelic.daemon.port = "/tmp/.newrelic.sock"
       ```

       Make sure:
       - Socket file exists
       - PHP process has permission to access it
       - If using TCP, port 3142 is accessible

    3. **Check file permissions:**
       ```bash
       ls -la /tmp/.newrelic.sock
       # Should be readable by web server user (www-data, apache, nginx)
       ```

    See [daemon connectivity](/docs/apm/agents/php-agent/advanced-installation/starting-php-daemon-advanced#daemon-external) for advanced troubleshooting.
  </Collapser>

  <Collapser title='❌ No log file exists'>
    If `/var/log/newrelic/php_agent.log` doesn't exist:

    **Solutions:**

    1. **Check log directory permissions:**
       ```bash
       ls -la /var/log/newrelic/
       ```

       The web server user needs write access:
       ```bash
       sudo chown -R www-data:www-data /var/log/newrelic/
       # Or for RedHat/CentOS:
       sudo chown -R apache:apache /var/log/newrelic/
       ```

    2. **Verify logging is enabled:**

       In `newrelic.ini`:
       ```ini
       newrelic.logfile = "/var/log/newrelic/php_agent.log"
       newrelic.loglevel = "info"
       ```

    3. **Check SELinux (RedHat/CentOS):**
       ```bash
       sudo setenforce 0  # Temporarily disable to test
       ```

       If this fixes it, configure SELinux properly:
       ```bash
       sudo semanage fcontext -a -t httpd_log_t "/var/log/newrelic(/.*)?"
       sudo restorecon -R /var/log/newrelic
       ```

    See [no log file](/docs/apm/agents/php-agent/troubleshooting/no-log-file-java) for more solutions.
  </Collapser>
</CollapserGroup>

### Check #4: Is your app running PHP?

This might seem obvious, but verify:

```bash
# Check a page returns PHP output
curl -I http://localhost/your-app | grep "X-Powered-By"
```

**Expected:**
```
X-Powered-By: PHP/8.1.0
```

<Callout variant="important">
The PHP agent **only instruments PHP code**. If your app:
- Is mostly static HTML → Limited or no data
- Uses PHP only for a few pages → Data only from PHP pages
- Routes everything through a frontend proxy → Agent won't see external traffic
</Callout>

### Check #5: Application environment issues

<CollapserGroup>
  <Collapser title="Docker containers">
    **Common Docker issues:**

    1. **Daemon not accessible:**
       - If agent and daemon in separate containers, ensure networking is configured
       - Use `newrelic.daemon.address` to specify daemon location
       - Example: `newrelic.daemon.address = "daemon-container:31339"`

    2. **Container restarts:**
       - Data clears if container is ephemeral
       - Use volume mounts for persistent logs

    See [Docker installation](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent) for details.
  </Collapser>

  <Collapser title="Multiple PHP versions">
    If you have multiple PHP versions:

    1. **Identify active version:**
       ```bash
       # CLI version
       php -v

       # Web server version (create info.php)
       <?php echo PHP_VERSION; ?>
       ```

    2. **Install for correct version:**
       - Run `newrelic-install` for each PHP version
       - Check `/etc/php/8.1/mods-available/newrelic.ini` exists for your version

    3. **Verify each installation:**
       ```bash
       /usr/bin/php8.1 -m | grep newrelic
       /usr/bin/php8.2 -m | grep newrelic
       ```
  </Collapser>

  <Collapser title="Shared hosting">
    If you're on shared hosting:

    1. **Contact your host:** Many shared hosts don't allow custom PHP extensions
    2. **Check if New Relic is pre-installed:** Some hosts offer New Relic as an add-on
    3. **Alternative:** Use [New Relic's browser monitoring](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent) for frontend monitoring

    See [shared hosting installation](/docs/apm/agents/php-agent/installation/install-php-agent-shared-hosting-service).
  </Collapser>

  <Collapser title="PHP-FPM specific">
    For PHP-FPM:

    1. **Restart both PHP-FPM and web server:**
       ```bash
       sudo systemctl restart php8.1-fpm
       sudo systemctl restart nginx
       ```

    2. **Check FPM pool configuration:**
       - Verify PHP-FPM user matches web server user
       - Check socket/port permissions

    3. **Verify process:**
       ```bash
       ps aux | grep php-fpm
       ```
  </Collapser>
</CollapserGroup>

## Still not working?

If you've checked everything and still see no data:

### Enable debug logging

1. **Edit `newrelic.ini`:**
   ```ini
   newrelic.loglevel = "verbosedebug"
   ```

2. **Restart web server:**
   ```bash
   sudo systemctl restart apache2
   ```

3. **Generate traffic and collect logs:**
   ```bash
   # Agent log
   sudo tail -100 /var/log/newrelic/php_agent.log > ~/php_agent_debug.log

   # Daemon log
   sudo tail -100 /var/log/newrelic/newrelic-daemon.log > ~/daemon_debug.log
   ```

4. **Review for errors:**
   Look for ERROR or WARN messages with details on the failure

### Get help

With your debug logs ready:

1. **Search the [community forum](https://discuss.newrelic.com/)** - Someone may have solved your issue
2. **Post in the forum** - Include:
   - PHP version (`php -v`)
   - Web server (Apache/Nginx/PHP-FPM)
   - OS and version
   - Installation method used
   - Relevant log excerpts (remove license key!)
3. **Contact [New Relic support](https://support.newrelic.com/)** with your logs
