## Problem

With the .NET agent enabled for an Asp.Net Core "Minimal APIs" application, endpoints with the same route path but different HTTP request methods appear as 
a single web transaction.

## Solution

Use [`SetTransactionName()`](/docs/agents/net-agent/net-agent-api/settransactionname-net-agent-api) to give each endpoint a unique transaction name.

```cs
// map a minimal API with GET and POST endpoints on the same route
app.MapGet("/minimalApi", () =>
{
    NewRelic.Api.Agent.NewRelic.SetTransactionName(null, name: "minimalApi/Get");
    return Results.Ok("Get: minimalApi");
});
app.MapPost("/minimalApi", () =>
{
    NewRelic.Api.Agent.NewRelic.SetTransactionName(null, name: "minimalApi/Post");
    return Results.Ok("Post: minimalApi");
});

```
