Custom instrumentation allows you to enrich your AI monitoring data with additional context and metadata that's specific to your application's needs. This guide covers the most common custom instrumentation scenarios to help you get deeper insights into your AI application's performance and behavior.

## Add attributes to LLM calls [#add-attributes-llm-calls]

You can add custom attributes to individual LLM calls to capture additional context about the request. This is useful when you want to track specific parameters, user segments, or business logic associated with each LLM invocation. Custom attributes help you filter and analyze your LLM performance data based on dimensions that matter to your application.

<CollapserGroup>
  <Collapser
    id="python-llm-attributes"
    title="Python"
  >
    Use `WithLlmCustomAttributes` as a context manager to attach custom key-value pairs to the LLM event:

    ```python
    import newrelic.agent
    from openai import OpenAI

    client = OpenAI()

    @newrelic.agent.background_task()
    def call_llm():
        with newrelic.agent.WithLlmCustomAttributes({"user_id": "1234", "custom": "attr"}):
            response = client.chat.completions.create(
                messages=[{
                    "role": "user",
                    "content": "Say this is a test",
                }],
                model="gpt-4o-mini",
            )
    ```

    Refer to our API docs for [`WithLlmCustomAttributes`](/docs/apm/agents/python-agent/python-agent-api/withllmcustomattributes-python-agent-api/).
  </Collapser>

  </Collapser>

## Add attributes to AI traces [#add-attributes-traces]

Adding custom attributes to AI traces allows you to enrich entire trace spans with contextual information. This enables you to segment and analyze your AI application's behavior across different scenarios, environments, or user cohorts. Trace-level attributes are particularly useful for tracking high-level business metrics or operational context that applies to the entire request flow.

<CollapserGroup>
  <Collapser
    id="python-trace-attributes"
    title="Python"
  >
    Use `add_custom_attribute` to attach attributes to the active trace span:

    ```python
    import newrelic.agent
    from openai import OpenAI

    client = OpenAI()

    @newrelic.agent.background_task()
    def call_llm():
        newrelic.agent.add_custom_attribute("llm.conversation_id", "my-convo-id")
        response = client.chat.completions.create(
            messages=[{
                "role": "user",
                "content": "Say this is a test",
            }],
            model="gpt-4o-mini",
        )
    ```

    Refer to our API docs for [`add_custom_attribute`](/docs/apm/agents/python-agent/python-agent-api/addcustomattribute-python-agent-api/).
  </Collapser>
</CollapserGroup>

## Add conversation ID to AI traces [#add-conversation-id]

To track multi-turn conversations and understand user interactions over time, you can add a `conversation_id` attribute to your AI traces. This allows you to group multiple user requests that belong to the same conversation session, making it easier to analyze conversation flows, measure engagement duration, and identify issues that span across multiple interactions within a single conversation.

<CollapserGroup>
  <Collapser
    id="python-conversation-id"
    title="Python"
  >
    Use `add_custom_attribute` with the `llm.conversation_id` key to associate a trace with a specific conversation:

    ```python
    import newrelic.agent
    from openai import OpenAI

    client = OpenAI()

    @newrelic.agent.background_task()
    def call_llm():
        newrelic.agent.add_custom_attribute("llm.conversation_id", "my-convo-id")
        response = client.chat.completions.create(
            messages=[{
                "role": "user",
                "content": "Say this is a test",
            }],
            model="gpt-4o-mini",
        )
    ```

    Refer to our API docs for [`add_custom_attribute`](/docs/apm/agents/python-agent/python-agent-api/addcustomattribute-python-agent-api/).
  </Collapser>
</CollapserGroup>
