New Relic has several types of [APIs](/docs/apis/getting-started/intro-apis/introduction-new-relic-apis) that require their own type of API keys, based on the data set they can retrieve. This document describes our APIs, the keys required to execute certain functions, and how to manage access to these API keys.

## Personal API key

Requirements and availability:

* For users on our [New Relic One user model](/docs/accounts/accounts-billing/new-relic-one-pricing-users/transition-guide-new-pricing-plan#user-models), including users on accounts on our [newer pricing plan](/docs/accounts/accounts-billing/new-relic-one-pricing-users/pricing-billing): You cannot manage this API key from the UI, but must use NerdGraph. [Learn more.](/docs/accounts/accounts-billing/new-relic-one-pricing-users/transition-guide-new-pricing-plan#limitations)

This API key allows you to use [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) (our GraphQL API). You can use NerdGraph to [query entities](/docs/apis/graphql-api/tutorials/use-new-relic-graphql-api-query-entities), see [relationships between them](/docs/apis/graphql-api/tutorials/graphql-relationships-api-tutorial), [add tags to entities](/docs/apis/graphql-api/tutorials/graphql-tagging-api-tutorial), and more.

Personal API keys are generated on a per-user basis. Each person on your account can have their own personal API key, and **these keys should not be shared**. This lets you keep track of changes that are made across your organization. You can create and manage personal API keys [via the NerdGraph API](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-personal-api-keys).

[Some users](/docs/accounts/accounts-billing/new-relic-one-pricing-users/transition-guide-new-pricing-plan#limitations) can use these UI options:

<CollapserGroup>
  <Collapser
    id="user-api-key"
    title="Create personal API keys via UI"
  >
    To generate an API key for a user from the UI:

    1. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and go to **Account settings > Users and roles**.
    2. Select the user and click the **API keys** tab.
    3. Select **+ New API key**.
    4. When prompted, confirm key creation.
  </Collapser>

  <Collapser
    id="view-personal-key"
    title="View personal API keys via UI"
  >
    You cannot view another user's entire API key, only the first few characters. Restricted users also cannot view their personal API key.

    To view your personal API key:

    1. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and go to **Account settings > Users and roles**.
    2. Select your name from the list of users.
    3. Select the **API keys** tab to view your personal keys.
  </Collapser>

  <Collapser
    id="delete-personal-key"
    title="Delete personal API keys via UI"
  >
    To delete a personal API key:

    1. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and go to **Account settings > Users and roles**.
    2. Select the user.
    3. Select the **API keys** tab.
    4. Select the **delete** <Icon name="trash-2"/>
       icon.
  </Collapser>
</CollapserGroup>

## REST API key

Before you can use New Relic's [REST API](/docs/apm/apis/requirements/getting-started-new-relic-rest-api-v2) or the [API Explorer](/docs/apis/getting-started-with-new-relics-api-explorer), you must have a REST API key for your New Relic account. For master accounts with [sub-accounts](/docs/subscriptions/creating-sub-accounts), each sub-account must use its own REST API key.

Only the account owner or admin can generate a REST API key for the account and each sub-account. Then, users in the account or sub-account can use the REST API key to view (but not add, change, or delete) data from the New Relic REST API and API Explorer.

**Availability**: Each New Relic account can have **one** REST API key.

<CollapserGroup>
  <Collapser
    id="activate-rest-api"
    title="Activate REST API key"
  >
    To activate API access:

    1. Click your username in **[one.newrelic.com](http://one.newrelic.com)** and go to **Account settings > API keys**.
    2. Select <Icon name="rotate-cw"/>
       **Regenerate REST API key**.
    3. Select **(Show key)** to view your account's new REST API key.
    4. Copy your account's REST API key to use with New Relic's REST API.
  </Collapser>

  <Collapser
    id="regenerate-rest-api-key"
    title="Regenerate REST API key"
  >
    To change your account's REST API key:

    1. Select <Icon name="rotate-cw"/>
       **Regenerate REST API key**.
    2. Confirm the deletion of the previous REST API key by typing `DELETION IS FOREVER` in the text field.
    3. Select **Regenerate**.
    4. Copy the new REST API key to use with New Relic's REST API.
  </Collapser>

  <Collapser
    id="view-rest-api"
    title="View REST API keys"
  >
    To view REST API keys:

    1. Click your username in **[one.newrelic.com](http://one.newrelic.com)** and go to **Account settings > API keys**.
    2. Select **(Show key)**.
  </Collapser>

  <Collapser
    id="delete-admin-key"
    title="Delete REST API key"
  >
    To delete your account's REST API key:

    1. Before you delete an API key, make sure you have the correct account or sub-account.
    2. Click your username in **[one.newrelic.com](http://one.newrelic.com)** and go to **Account settings > API keys**.
    3. Select the **delete** <Icon name="trash-2"/>
       icon for the REST API key.
    4. Confirm the deletion by typing `DELETION IS FOREVER` in the text field.
    5. Select **Delete**.
  </Collapser>
</CollapserGroup>

## Admin's API key

Requirements and availability:

* Not available for users on the [New Relic One user model](/docs/accounts/accounts-billing/new-relic-one-pricing-users/transition-guide-new-pricing-plan#user-models), including accounts on our [newer pricing plan](/docs/accounts/accounts-billing/new-relic-one-pricing-users/pricing-billing).
* [Requires an](/docs/accounts/accounts-billing/new-relic-one-pricing-users/pricing-billing) [Owner or Admin](/docs/accounts-partnerships/accounts/account-setup/users-roles#roles) user role.
* Each user can have **one** personal **Admin user's API key** per account.

Use the New Relic Admin user's API key to manage access to the REST API and API Explorer. Users can use the account REST API key to view data with the API Explorer while Owners and Admins can use an Admin user's API key to view, add, change, or delete data.

The Admin user's API key grants full permissions to use the New Relic REST API and API Explorer, including **GET > List**, **POST > Create**, **PUT > Add** or **Update**, and **DELETE** functions.

In addition, you must use your Admin API key to [manage dashboards](/docs/insights/insights-api/manage-dashboards/insights-dashboard-api) using the API Explorer functions, including **CREATE > Post**, **UPDATE > Put**, **SHOW > Get**, **SHOW > List**, and **DELETE** functions. Other account users cannot view dashboard data from the API Explorer or use the Dashboard API functions.

<CollapserGroup>
  <Collapser
    id="admin-api"
    title="Activate Admin user's API key"
  >
    To activate Admin API access:

    1. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and go to **Account settings > API keys**.
    2. Next to your name, select the generate <Icon name="rotate-cw"/>
       icon to generate your Admin user's API key.
    3. Select **(Show key)** to view your new Admin user's API key.
    4. Copy your Admin user's API key to use with New Relic's REST API.
  </Collapser>

  <Collapser
    id="view-admin-api"
    title="View Admin's API key"
  >
    An index of Admin user's API keys appears below the account's REST API key. The list includes the Admin's full name and the date their key was last used.

    * You can view your own Admin user's API key: From the **Admin** index, select **(Show key)** for your name.
    * You can only view other Admin's names. The keys for all other Admins are partially obscured.
  </Collapser>

  <Collapser
    id="regenerate-admin-key"
    title="Regenerate Admin user's API key"
  >
    To change an Admin user's API key:

    1. Select the Admin user's <Icon name="rotate-cw"/>
       **Regenerate REST API key** icon.
    2. Confirm the deletion of the previous Admin user's API key by typing `DELETION IS FOREVER` in the text field.
    3. Select **Regenerate**.
    4. Copy the new Admin user's API key to use with New Relic's REST API.
  </Collapser>

  <Collapser
    id="delete-admin-key"
    title="Delete an Admin's API key"
  >
    <Callout variant="caution">
      If you delete an Admin from the New Relic account, or change their role from **Admin** to **User** or **Restricted user**, this will also automatically remove their Admin user's API key.
    </Callout>

    To delete an Admin user's API key:

    1. Before you delete an API key, make sure you have the correct account or sub-account.
    2. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and select **Account settings > API keys**.
    3. From the list of Admins, locate the person whose key must be deleted.
    4. Select the **delete** <Icon name="trash-2"/>
       icon for the REST API key or the Admin user's API key.
    5. Confirm the deletion by typing `DELETION IS FOREVER` in the text field.
    6. Select **Delete**.
  </Collapser>
</CollapserGroup>

## Insert API key

Tips on availability and access:

* Because Insert API keys are associated with an account and not a specific user, anyone in the account with access to an Insert API key can use it.
* As a best practice for security purposes, we recommend you use different Insert API keys for different applications or different data sources.

This key is required for sending data via the [Event API](/docs/insights/insights-data-sources/custom-data/insert-custom-events-insights-api), [Log API](/docs/enable-new-relic-logs-http-input), [Metric API](/docs/introduction-new-relic-metric-api), and [Trace API](/docs/apm/distributed-tracing/trace-api/introduction-new-relic-trace-api), or via our integrations that use these APIs. This key reports custom data to New Relic, including event, log, metric, or trace data. Once your custom data is sent to us, you can query and visualize that data.

<CollapserGroup>
  <Collapser
    id="insert-key-create"
    title="Generate an Insert API key"
  >
    To register an Insert API key:

    1. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and select **Account settings**.
    2. Select **API keys**, and then select **Insights API keys**.
    3. Next to the **Insert keys** heading, select the <Icon name="plus"/>
       symbol and follow the instructions.
  </Collapser>

  <Collapser
    id="insert-key-edit-delete"
    title="Edit or delete an Insert API key"
  >
    To edit or delete an Insert API key:

    1. Click your username in [**one.newrelic.com**](http://one.newrelic.com) and select **Account settings**.
    2. Select **API keys**, and then select **Insights API keys**.
    3. Under **Insert Keys**, find the key you want to modify. Select **Edit** or **Delete**.
  </Collapser>
</CollapserGroup>

## Query key

Use this key to query data via New Relic's query APIs. Structure queries including your key based on the [specifications](https://docs.newrelic.com/docs/insights/insights-api/get-data/query-insights-event-data-api#create-request) for the API you're working with.

<CollapserGroup>
  <Collapser
    id="x-query-key-create"
    title="Generate a Query key"
  >
    You must have access to the UIs you plan to query and the correct permissions in order to register or modify a Query key.

    <Callout variant="important">
      Query keys are generated for an account. They are not associated with a specific user. Anyone in the account with access to the Query key can use it.
    </Callout>

    To register an Query key:

    1. Go to **[insights.newrelic.com](https://insights.newrelic.com/ "Link opens in a new window.") > Manage data > API keys**.
    2. Select the plus icon next to the **Query keys** heading.
    3. Enter a short, clear description of the key.
    4. Select **Save your notes**.
  </Collapser>

  <Collapser
    id="x-query-key-edit-delete"
    title="Edit or delete a Query key"
  >
    To edit or delete an Query key:

    1. Go to **[insights.newrelic.com](https://insights.newrelic.com/) > Manage data > API keys**.
    2. Under **Query Keys**, find the key you want to modify. Select **Edit** or **Delete**.
  </Collapser>
</CollapserGroup>
