New Relic's REST API Explorer (v2) makes it easy to test and send requests for any available API endpoint. After you select your choice of functions for the type of API call (applications, browsers, etc.), the user interface provides an interactive form to view requirements and test your parameter values. It also provides a live source of documentation about the API values.

## API key requirements [#api-key]

Before you can use the [API Explorer](https://rpm.newrelic.com/api/explore), API access must be activated and an [User API key](/docs/apis/rest-api-v2/requirements/rest-api-key) must be generated for your account.

Tips:

* When you use the API Explorer you can insert your [User API key](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) at the top of the UI and that key will appear automatically in the <DNT>**Request**</DNT> and <DNT>**Parameters**</DNT> sections of the Explorer.

## Use the API Explorer [#use-api-explorer]

1. Go to the <DNT>**[API Explorer](https://api.newrelic.com/docs/)**</DNT>.
2. From the <DNT>**Servers**</DNT> dropdown, choose a US or EU-based API URL.
3. Click <DNT>**Authorize**</DNT>, enter your user API key, and click <DNT>**Authorize**</DNT> again.
4. Expand one of the available API functions: <DNT>**GET**</DNT>, <DNT>**PUT**</DNT>, <DNT>**DELETE**</DNT>.
5. (Optional) Add <DNT>**Parameters**</DNT> values for your API call to filter the response (refer to the UI for descriptions and requirements for v2.)
6. From the <DNT>**Media type**</DNT> dropdown, choose a format for your request: JSON or XML.
7. Click <DNT>**Try it out**</DNT>, then <DNT>**Execute**</DNT>.
