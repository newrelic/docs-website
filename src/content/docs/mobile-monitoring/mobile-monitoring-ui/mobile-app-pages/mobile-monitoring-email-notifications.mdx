New Relic's <InlinePopover type="mobile"/> provides several emailed reports, including:

* [Release notifications](#version-trends) (aka version trend emails)
* [Crash report emails](#crash-reports)

## Get emails on new version performance [#version-trends]

New Relic's mobile monitoring provides you with a [<DNT>**Release versions**</DNT>](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/release-versions-page) page. You can also enable version trend emails (aka release notifications). These reports provide a convenient communication tool to share, track, and review your app's progress, adoption, and other trends. Each email includes a link to the <DNT>**Version trends**</DNT> UI to see more details.

### Edit version trend email settings [#version-trend-ui]


    To configure the email reports you receive for a specific mobile app, go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT> and edit the release notification email settings.
 

## Crash report emails [#crash-reports]

When your mobile app has a new kind of crash, you want to know what went wrong so you can prevent that type of error in the future. With crash report emails, you are notified when a unique crash is detected.

To configure if you receive crash email reports for a specific mobile app, go to <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Settings > Application**</DNT> and edit the crash report email settings.
