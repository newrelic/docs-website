Curious about how your latest app launch is performing? Use our Release versions page to view crash rates, user engagement, and overall performance metrics between different release versions, which you can use to analyze the success of your latest release.

## View the Release versions page [#view-release-versions-page]

1. Go to [one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile.
2. Select your mobile app.
3. In the left-hand menu, click <DoNotTranslate>**Release versions**</DoNotTranslate>.

<Callout variant="tip">
  Not seeing data? Try the following:

  * Adjust the time filter to include data before a new version released.
  
  * Make sure your mobile agent is using iOS agent version 7.0.0 and higher or Android agent version 6.5.0 and higher.

  </Callout>


## Drill down into specific metrics [#drill-down]

The Release versions page displays high-level metrics on each app version:

* <DoNotTranslate>**Overall performance score**</DoNotTranslate>: Calculated by overall customer crash rate and HTTP error-free rates.
* <DoNotTranslate>**Crash free user %**</DoNotTranslate>: Percentage of users not experiencing crashes on a specific version.
* <DoNotTranslate>**User adoption**</DoNotTranslate>: Percentage of users utilizing a specific version.

Click <DoNotTranslate>**View summary**</DoNotTranslate> or <DoNotTranslate>**View details**</DoNotTranslate> to expand into a specific metrics for an app version. With easy-to-understand visualizations and detailed charts, the version summary page empowers you to analyze key performance indicators and make informed decisions to optimize your app's user experience.

<img
    title="Details of release versions"
    alt="Screenshot of release versions detailed view"
    src={mobileReleaseVersionsDetails}
/>

For each metric listed above, the Release versions page includes a dynamic time series chart, allowing you to track the metric's performance over time. By observing trends, spikes, or drops in the chart, you can quickly identify patterns and correlations that can inform your decision-making.


## Create alerts [#create-alerts]

Now that you've got visibility into your app's performance, it's time to set up alerts for key performance changes. 

Simply click into a metric, then from the chart menu, click <DoNotTranslate>**Create alert condition**</DoNotTranslate> and follow the steps in the UI.
