Use this page to troubleshoot React Native source map uploads and `MobileJSError` symbolication. For how to upload source maps, see [React Native JavaScript error reporting](/docs/mobile-monitoring/new-relic-monitoring-react-native/react-native-agent-js-error-reporting).

## Troubleshooting [#troubleshooting]

### Source map errors in the UI [#sourcescontent]

If you see any of the following errors in the UI, your source map file is missing the `sourcesContent` attribute:

* `Source map not found.`
* `Source map missing.`
* `Incorrect map detected.`
* `Incorrect and missing maps detected.`

Your source map must include the `sourcesContent` attribute for New Relic to un-minify your stack traces. Regenerate the source map so this attribute is included, then upload it again. Most bundlers include the original source content when you enable an option such as `includeSources` (or the equivalent for your build tool).

### You have a React Native app but don't see the menu item [#no-menu-item]

If you have a React Native app but don't see the React Native menu item in the UI, check that you're on an updated agent version and that your app is sending data. The menu item doesn't appear until the app sends data with the React Native platform tag attached to it.

### You don't see both MobileJSError and handled exception events [#mobilejserror-events]

With the new agent version, you won't see both `MobileJSError` events and handled exception events for the same JavaScript errors. JavaScript errors are recorded as `MobileJSError` events instead.

`MobileJSError` is currently supported for React Native only. Capacitor and Cordova aren't supported yet.

## Frequently asked questions [#faq]

### Does New Relic accept zipped files? [#faq-zip]

Yes. You can upload a gzipped or zipped (`.zip`) source map. The build scripts automatically zip large `.map` files before upload.

### What is the maximum upload payload size? [#faq-size]

200 MB. Source maps larger than 200 MB after zipping aren't stored for symbolication; build telemetry is sent instead. For details, see [File size limitations](/docs/mobile-monitoring/new-relic-monitoring-react-native/react-native-agent-js-error-reporting#file-size-limitations).

### Why don't my line numbers match? [#faq-line-numbers]

Line numbers don't match when the uploaded source map doesn't correspond exactly to the JavaScript bundle that produced the error. Make sure the `jsBundleId` and `appVersion` used for the upload match the values the app reports for that build — this is especially important after a CodePush or other over-the-air (OTA) update, when the JavaScript bundle diverges from the native binary version. Also confirm your source map is version 3.

### Why aren't my files symbolicated? [#faq-not-symbolicated]

Common causes include:

* The source map is missing the `sourcesContent` attribute (see [Source map errors in the UI](#sourcescontent)).
* The `jsBundleId` or `appVersion` don't match between the upload and the running app (see [Why don't my line numbers match?](#faq-line-numbers)).
* The source map exceeded the [200 MB limit](#faq-size), so only build telemetry was sent.
* The data hasn't finished syncing yet. After an upload it can take a few minutes for the data to become available.
* The source map's API key and the application token belong to different New Relic accounts.

### How can I see what I uploaded to New Relic? [#faq-list]

Use the list command in NerdGraph or NRQL. See [List and delete React Native source maps](/docs/mobile-monitoring/new-relic-monitoring-react-native/list-delete-react-native-source-maps).
