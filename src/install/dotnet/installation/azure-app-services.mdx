To add your app settings:

1. Make sure you have your <InlinePopover type="licenseKey"/> before adding the New Relic .NET agent's app settings.
2. In the Azure portal, search for and open <DNT>**App Services**</DNT> to go to the web app you want to monitor. Next, select <DNT>**Configuration > Application settings**</DNT>.
3. From <DNT>**Application settings**</DNT>, select <DNT>**+ New application setting**</DNT> and add these key/value pairs:

   <table>
     <thead>
       <tr>
         <th width={300}>
           Key
         </th>

         <th>
           Value
         </th>
       </tr>
     </thead>

     <tbody>
       <tr>
         <td>
           `NEW_RELIC_LICENSE_KEY`
         </td>

         <td>
           YOUR_LICENSE_KEY
         </td>
       </tr>

       <tr>
         <td>
           `NEW_RELIC_APP_NAME`
         </td>

         <td>
           YOUR_APP_NAME
         </td>
       </tr>
     </tbody>
   </table>
4. Save and restart your web app.

Setting the <InlinePopover type="licenseKey"/> and app name with <DNT>**Application settings**</DNT> will override the `newrelic.config`.

To verify your app's environment variables in your app's Kudu diagnostic console, go to <DNT>**Environment > Environment variables**</DNT>.

For example:

```
https://myappname.scm.azurewebsites.net/Env.cshtml#envVariables
```
