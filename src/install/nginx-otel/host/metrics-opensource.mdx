The NGINX OpenTelemetry integration collects the following metrics from the HTTP stub status module.

### Metric data [#metrics]

<table>
  <thead>
    <tr>
      <th style={{ width: "350px" }}>
        Metric
      </th>
      <th>
        Description
      </th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        `nginx.connections_accepted`
      </td>
      <td>
        Total number of accepted client connections (Sum)
      </td>
    </tr>
    <tr>
      <td>
        `nginx.connections_handled`
      </td>
      <td>
        Total number of handled client connections (Sum)
      </td>
    </tr>
    <tr>
      <td>
        `nginx.connections_current`
      </td>
      <td>
        Current number of active client connections. Includes `waiting`, `reading`, and `writing` states (Gauge)
      </td>
    </tr>
    <tr>
      <td>
        `nginx.requests`
      </td>
      <td>
        Total number of client requests (Sum)
      </td>
    </tr>
  </tbody>
</table>

### Connection state metrics

These metrics break down `nginx.connections_current` by connection state:

<table>
  <thead>
    <tr>
      <th style={{ width: "350px" }}>
        Metric
      </th>
      <th>
        Description
      </th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        `nginx_connections_reading`
      </td>
      <td>
        Current connections where NGINX is reading the request header (Gauge)
      </td>
    </tr>
    <tr>
      <td>
        `nginx_connections_writing`
      </td>
      <td>
        Current connections where NGINX is writing the response back to the client (Gauge)
      </td>
    </tr>
    <tr>
      <td>
        `nginx_connections_waiting`
      </td>
      <td>
        Current idle client connections waiting for a request (Keep-Alive) (Gauge)
      </td>
    </tr>
  </tbody>
</table>

### Resource attributes

All metrics include the following resource attributes:

- `nginx.server.endpoint` - The stub status endpoint URL
- `nginx.deployment.name` - Your configured deployment name
- `nginx.display.name` - Formatted display name combining server and deployment
- `host.name` - The hostname where NGINX is running
- `host.id` - The host identifier
