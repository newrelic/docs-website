The MySQL Query Performance Monitoring collects the following metrics:

<CollapserGroup>
  <Collapser
    id="slow-query-metrics"
    title="Slow Query Metrics"
  >
    Metrics related to slow queries.

    <table>
      <thead>
        <tr>
          <th style={{ width: "350px" }}>
            Event Attribute Name
          </th>
          <th>
            Description
          </th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>
            `query_id`
          </td>
          <td>
            Identifier for the slow query. The `query_id` event attribute stores the ID of the slow query.
          </td>
        </tr>
        <tr>
          <td>
            `query_text`
          </td>
          <td>
            The text of the slow query. The `query_text` event attribute contains the SQL query text.
          </td>
        </tr>
        <tr>
          <td>
            `database_name`
          </td>
          <td>
            The database the slow query was executed against. The `database_name` event attribute indicates the database in which the query ran.
          </td>
        </tr>
        <tr>
          <td>
            `schema_name`
          </td>
          <td>
            The schema the slow query was executed against. The `schema_name` event attribute specifies the schema in which the query ran.
          </td>
        </tr>
        <tr>
          <td>
            `execution_count`
          </td>
          <td>
            Number of times this slow query has been executed.  The `execution_count` event attribute tracks the number of executions.
          </td>
        </tr>
        <tr>
          <td>
            `avg_cpu_time_ms`
          </td>
          <td>
            Average CPU time in milliseconds spent executing this slow query. The `avg_cpu_time_ms` event attribute stores the average CPU time.
          </td>
        </tr>
        <tr>
          <td>
            `avg_elapsed_time_ms`
          </td>
          <td>
            Average elapsed time in milliseconds to execute this slow query. The `avg_elapsed_time_ms` event attribute stores the average elapsed time.
          </td>
        </tr>
        <tr>
          <td>
            `avg_disk_reads`
          </td>
          <td>
            Average number of disk reads performed by this slow query. The `avg_disk_reads` event attribute tracks the average number of disk reads.
          </td>
        </tr>
        <tr>
          <td>
            `avg_disk_writes`
          </td>
          <td>
            Average number of disk writes performed by this slow query. The `avg_disk_writes` event attribute tracks the average number of disk writes.
          </td>
        </tr>
        <tr>
          <td>
            `has_full_table_scan`
          </td>
          <td>
            Indicates whether the query performed a full table scan ("true" or "false"). The `has_full_table_scan` event attribute indicates whether a full table scan occurred.
          </td>
        </tr>
        <tr>
          <td>
            `statement_type`
          </td>
          <td>
            The type of SQL statement (e.g., "SELECT", "INSERT", "UPDATE"). The `statement_type` event attribute identifies the type of SQL statement.
          </td>
        </tr>
        <tr>
          <td>
            `last_execution_timestamp`
          </td>
          <td>
            Timestamp of the last time this slow query was executed. The `last_execution_timestamp` event attribute stores the timestamp of the last execution.
          </td>
        </tr>
         <tr>
          <td>
            `collection_timestamp`
          </td>
          <td>
            Timestamp of the collection of this metrics data. The `collection_timestamp` event attribute represents the time the metrics data was collected.
          </td>
        </tr>
      </tbody>
    </table>
  </Collapser>
