.NET向け [APM](/docs/apm)があれば、ご利用の.NETアプリケーションとサービスをモニターできます。この.NET向けAPMの入門では、このプロダクトを使用する理由、また使用を開始するにあたっての方法について学習します。

## 機能 [#features]

[New Relic .NETエージェントをインストール](/docs/agents/net-agent/installation/install-enable-new-relic-net-agent)後は、次のことを実行できます。

* [APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm)を使用して[ご利用のアプリの高次元な概要を入手し](/docs/apm/applications-menu/monitoring/apm-overview-page)、[トランザクショントレース](/docs/apm/transactions/transaction-traces/transaction-traces)、[データベースクエリ](/docs/apm/transactions/transaction-traces/sql-statements)、そして[エラー](/docs/apm/applications-menu/events/view-apm-error-analytics)などのコードレベルの詳細を確認して、[大規模な分散システム上のアクティビティを追跡しましょう](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)。
* アプリをスムーズに実行するため、[Alerts](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic)からプロアクティブに通知を受信できます。
* また[クエリビルダー](/docs/query-your-data/explore-query-data/query-builder/use-advanced-nrql-mode-specify-data)を使用して、データクエリを実行し、そのデータでカスタム[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-new-relic-one-dashboards)を作成することもできます。
* [Infrastructureモニタリング](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server) をインストールすれば、アプリのホスト環境のパフォーマンスを表示できます。

## .NET Frameworkと.NET Coreの双方へのサポート [#core-features]

New Relicの.NETエージェントは、[.NET Framework](https://docs.newrelic.co.jp/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent)および[.NET Core](https://docs.newrelic.co.jp/docs/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core)をどちらもサポートしており、VB.NET、C#、CLIなどの.NETと互換性のあるすべての言語で動作します。

.NET Coreのエージェントのサポートは、Microsoftの.NET Coreの互換性、速度、拡張API機能、プラットフォーム間機能を利用しています。エージェントは、バージョン 2.0以前のMicrosoft .NET Coreバージョンをサポート**していません**。

New Relicの.NETに対するサポートを使用すると、以下のように動的または分散環境でアプリをモニターできます。

* クラウドで管理されているサーバーのVMイメージ
* ホスト上のVMサーバー
* Microsoft Azure App Services
* 自社運用のWindowsシステムおよびLinuxシステム
* Amazon AWS EC2のVM

## エージェントのインストール [#installation]

.NETエージェントをインストールする前に、[New Relicアカウントを作成してください](/docs/accounts-partnerships/accounts/account-setup/creating-your-new-relic-account)。次に、[.NET Frameworkの要件](/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent)または [.NET Coreの要件](/docs/agents/net-agent/getting-started/compatibility-requirements-net-core-20-agent)を確認します。

インストールの準備ができたら、ランチャーを使用するか、以下のオペレーティングシステムに適切なインストール手順を確認します。[Windows](/docs/agents/net-agent/installation/install-net-agent-windows)または[Linux](/docs/agents/net-agent/installation/install-net-agent-linux)。

## エージェントの設定 [#configuration]

[エージェントの設定では、](/docs/agents/net-agent/installation-configuration/name-your-net-application)アプリケーションに分かりやすい名前を付けることが最も重要です。複数のアプリケーションやサーバーが存在する場合、New Relicは[メトリックスを集計する目的でこのアプリケーション名を使用します](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name) 。また、エージェントには、多数の[設定オプション](/docs/agents/net-agent/installation-configuration/net-agent-configuration)があり、インストレーション後にさらにカスタマイズを行うこともできます。

## ソースコードをチェック [#source-code]

.NETエージェントは、オープンソースソフトウェアです。つまり、[ソースコードを参照](https://github.com/newrelic/newrelic-dotnet-agent)して改善を送信したり、独自のフォークを作成して構築したりできます。詳細については、[README](https://github.com/newrelic/newrelic-dotnet-agent/blob/main/README.md) を参照してください。
