Cuando el rendimiento del sistema se ve afectado, necesita saber si se debe a su código, su infraestructura o la red subyacente. Y necesita saberlo rápidamente para poder concentrar sus esfuerzos. Con Monitoreo de red puede correlacionar y analizar la aplicación, la infraestructura, la experiencia digital y los datos de la red en todo el lugar, y comprender cómo el rendimiento de la red y el rendimiento general del sistema se impactan entre sí.

<Callout variant="tip">
  ¡Monitoreo de red ahora está incluido en la New Relic [autorización moderada FedRAMP!](/docs/security/security-privacy/compliance/data-encryption/)

  Utilice la instalación guiada para [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#setup-snmp-monitoring), [flujos de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#setup-network-flow-monitoring) y [syslog de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#setup-network-syslog-monitoring) para detectar y configurar automáticamente los [extremos de API autorizados por FedRAMP](/docs/security/security-privacy/compliance/fedramp-compliant-endpoints/#data-ingest-apis), o revise el proceso de configuración manual en nuestros documentos para [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#manual-setup), [flujos de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#manual-setup) y [syslog de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#manual-setup).
</Callout>

Monitoreo de red agrega el contexto de datos de red a los datos de aplicaciones e infraestructura que ya recopila en New Relic. Al monitorear los datos de su red, puede:

* Analice y comprenda el rendimiento de toda su stack (aplicación e infraestructura) para obtener una comprensión holística del rendimiento de su sistema.
* Ten todos los datos en una única plataforma para eliminar puntos ciegos.
* Vea a primera vista si una red está implicada en un problema.

## Tipos de datos de red [#network-data-types]

Puede monitor los siguientes tipos de datos de red:

* <DNT>
    **Device performance via SNMP**
  </DNT>

  : El Protocolo simple de administración de red (SNMP) es un protocolo de capa de aplicación para intercambiar información de administración entre dispositivos de red. Para enviar datos SNMP a New Relic, [configure el monitoreo de datos SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring).

* <DNT>
    **Network syslogs**
  </DNT>

  : Proporciona información valiosa sobre los cambios, errores y otros eventos del dispositivo para comprender rápidamente el '¿Por qué?' detrás de los cambios en el rendimiento de la telemetría. Para enviar datos de syslog de red, [configure el monitoreo de syslog de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring).

* <DNT>
    **Cloud flow logs**
  </DNT>

  : Detalla las comunicaciones entre hosts y otros servicios mediante la captura de la dirección IP y el puerto de origen y destino, metadatos que describen ambos extremos, telemetría clave del rendimiento de la red y el estado de un intento de comunicación específico. [Configurar el monitoreo log de flujo de la nube](/docs/network-performance-monitoring/setup-performance-monitoring/aws-vpc-flow-monitoring).

* <DNT>
    **Network flow logs**
  </DNT>

  : captura información sobre el tráfico IP que va hacia y desde las interfaces de red en su red local. Para enviar datos de flujo de red a New Relic, [configure el monitoreo de datos de flujo de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring).

<Callout variant="important">
  Recomendamos configurar tanto SNMP como datos de flujo de red para una mejor visibilidad de su red. Esto proporcionará tanto rendimiento métrico como patrones de tráfico para solucionar problemas y optimizar su red.
</Callout>
