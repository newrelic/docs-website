Configurar <InlinePopover type="alerts"/>para que le notifiquen los problemas con su stack es una de las cosas más importantes que puede hacer para monitor sus datos. Pero crear y administrar Alerta a menudo resulta abrumador. En los siguientes tutoriales, lo guiaremos a través de las tres formas principales de crear alertas con New Relic. También le mostraremos cómo usar esas alertas para recopilar información y usar esos datos para mejorar su stack. Si es completamente nuevo en alerta, tenemos algunos conceptos y terminología básicos de alertas que debe aprender antes de comenzar.

## ¿Qué son alerta? [#alerts-what]

Alerta son condiciones que, cuando se cumplen, le envían una notificación sobre el evento desencadenante. Puede enviar notificaciones a cualquier número de personas en cualquier número de equipos. Cuando se configura correctamente, alerta le permite resolver un problema de manera más eficiente al enviar notificaciones directas a los grupos que pueden solucionar un problema. Hay cinco partes principales para alertar:

* <DNT>
    **Incidents**
  </DNT>

  son eventos donde se infringe un umbral de condición. Ejemplos comunes de incidentes son cuando la latencia supera un cierto límite, una página no se carga o se produce un error de JavaScript.

* <DNT>
    **Issues**
  </DNT>

  son una colección de uno o más incidentes que provocan el envío de una notificación

* <DNT>
    **Conditions**
  </DNT>

  son objetos de configuración que usted configura para abrir un incidente

* <DNT>
    **Policies**
  </DNT>

  son agrupaciones de condiciones donde usted define cómo se le notifica cuando ocurre un incidente

* <DNT>
    **Notifications**
  </DNT>

  son los mensajes que recibe cuando se abre, reconoce o cierra un problema

<Callout variant="tip">
  ¿Cuál es la diferencia entre <DNT>**Incidents**</DNT> y <DNT>**Issues**</DNT>?

  Un <DNT>**incident**</DNT> es un evento individual que detalla un síntoma de un problema, mientras que un <DNT>**issue**</DNT> es una agrupación de incidentes que describe un problema mayor en su sistema.
</Callout>

## ¿Dónde están alerta en New Relic? [#alerts-where]

En New Relic, la mayoría de las veces administras tu alerta desde [<DNT>**one.newrelic.com > Alerts**</DNT>](https://one.newrelic.com/alerts-ai). Esta sección de la plataforma tiene una página <DNT>**Overview**</DNT> que le brinda desgloses visuales de todas sus alertas, así como una página <DNT>**Alert conditions (policies)**</DNT> que le brinda una lista de todas sus políticas de alertas, así como un lugar para crear otras nuevas.
