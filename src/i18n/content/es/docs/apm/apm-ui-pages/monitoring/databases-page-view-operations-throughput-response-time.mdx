En APM, la página <DNT>**Databases**</DNT> muestra la base de datos y los datos de caché de una aplicación. La página muestra la base de datos de transacciones individuales como una tabla ordenable y las operaciones, el rendimiento y el tiempo de respuesta como gráficos. Si se han reportado [consultas lentas](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) , se muestran cuando selecciona una transacción.

Dividimos la información de la base de datos en tres partes:

* Descripción general de All-base de datos (página principal)

  * Las cinco principales operaciones de base de datos por tiempo de reloj

  * Base de datos tiempo de respuesta

  * Rendimiento de la base de datos (solicitudes de operación por minuto o

    <DNT>
      **rpm**
    </DNT>

    )

* Principal base de datos de operaciones (subpágina)

  * Tiempo de consulta
  * Rendimiento
  * Consumo de tiempo por persona que llama

* Tabla con todas las operaciones de la base de datos (subpágina)

## Navegación [#database-page]

Elija una de las siguientes opciones para abrir la página:

* Vaya a

  <DNT>
    **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > All entities > (select an app) > Monitor > Databases**
  </DNT>

  .

* Vaya a

  <DNT>
    **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select an app) > Monitor > Databases**
  </DNT>

  .

<Callout variant="tip">
  Para obtener una descripción general de alto nivel de todas sus aplicaciones y servicios, utilice nuestro [explorador de entidades](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer).
</Callout>

## Filtrar operaciones por proveedor de base de datos [#db_filtering]

Si está disponible: encima de la lista de operaciones de la base de datos, los botones le permiten [filtrar](#availablity) hasta un proveedor de base de datos específico.

* Para filtrar el contenido por proveedor de base de datos, seleccione su nombre.

* Para eliminar el filtro, seleccione

  <DNT>
    **All**
  </DNT>

  .

## Revisar las operaciones principales de la base de datos [#top_db_operations]

La página principal <DNT>**Databases**</DNT> muestra una lista de las principales operaciones de base de datos que puede ordenar de diversas formas.

Para obtener detalles sobre una operación, puede pasar el cursor sobre el registro o hacer clic en él para ver gráficos adicionales.

Cuando hace clic en un registro, puede ver el gráfico <DNT>**Time consumption by caller**</DNT> . Esto muestra una lista de la transacción que llama a la operación seleccionada, con barras de colores que indican el porcentaje de tiempo consumido por cada persona que llama. Para ver la página <DNT>**Transactions**</DNT> de una transacción de la persona que llama, seleccione la barra de color para esa transacción en el gráfico.

## Mostrar todas las operaciones de la base de datos [#db_displaying]

Para mostrar todas las operaciones de la base de datos [si están disponibles](#availability):

* En la parte inferior de la lista de operaciones, seleccione el enlace

  <DNT>
    **Show all database operations table...**
  </DNT>

  .

* Para ordenar la tabla por cualquier columna, seleccione el encabezado de la columna.

* Para filtrar la lista de operaciones, utilice el cuadro de búsqueda en la parte superior de la tabla.

* Para ver detalles sobre una operación específica, seleccione el nombre de la operación.

## Eliminar traza [#db_deleting]

Para eliminar toda la traza de tu aplicación de forma permanente:

1. Seleccione el botón

   <DNT>
     **Delete all traces**
   </DNT>

   debajo de la lista de operaciones de la base de datos.

2. Seleccione el símbolo de confirmación.

<Callout variant="tip">
  Si está realizando una limpieza general de registros, es posible que también desee eliminar [la traza de la transacción](/docs/traces/deleting-transaction-traces) o [la traza de error](/docs/apm/applications-menu/events/viewing-apm-errors-error-traces). Esto eliminará datos potencialmente confidenciales y al mismo tiempo conservará otros datos de su aplicación (como Apdex, información desplegable, etc.).
</Callout>

## Característica adicional basada en la versión del agente [#availablity]

Las características adicionales en la página <DNT>**Databases**</DNT> están disponibles con las siguientes versiones del agente. Para obtener más información, consulte [Actualizar el agente New Relic](/docs/agents/manage-apm-agents/installation/update-new-relic-agent).

* Vaya a 1.4 o superior
* Java 3.14.0 o superior
* .NET 4.1.134 o mas alto
* Node.js 1.17.0 o superior
* PHP 4.19.0 o superior
* Python 2.44.0 o superior
* Ruby 3.11.0 o superior

Estas características incluyen:

* Múltiples bases de datos y tipos de caché
* Etiquetas para diferentes tipos en la lista de operaciones, gráficos y consulta lenta
* Operaciones de filtrado y gráficos para un proveedor de base de datos o tipo de caché específico
* Ver una tabla de todas las operaciones de la base de datos
* Posibilidad de ver la base de datos en [mapas de servicios.](/docs/apm/applications-menu/monitoring/service-maps)

Además, si su agente APM admite [detalles a nivel de instancia y host](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues), puede ver tanto los detalles de la consulta como la instancia específica donde se ejecutó la consulta.
