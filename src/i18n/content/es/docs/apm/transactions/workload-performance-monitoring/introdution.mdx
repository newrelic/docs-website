En el panorama tecnológico actual en rápida evolución, las aplicaciones modernas se volvieron cada vez más complejas, lo que lleva a un aumento significativo en el tiempo medio de resolución (MTTR). Esta complejidad surge de los diversos componentes que componen la aplicación moderna, como la base de datos, la función serverless, los browsers, la aplicación móvil, la cola de mensajes, la infraestructura, el modelo de IA y los servicios. El seguimiento de estos servicios individuales ya no es suficiente. Los propietarios de aplicaciones requieren una visibilidad integral de todo su ecosistema de aplicaciones.

Transaction 360 de New Relic ofrece un enfoque detallado para la supervisión y resolución de problemas mediante la captura y correlación automática de todas las entidades y datos relacionados con un problema o pregunta específicos.

Las características clave de Transaction 360 incluyen:

* **Descubrimiento automático**: mapeo dinámico de relaciones de servicio y visualización del flujo de transacciones en tiempo real.
* **Visibilidad integral**: monitoreo de todo el stack, seguimiento de transacciones entre servicios e infraestructura para visibilidad del browser.
* **Análisis histórico**: acceso a semanas de datos históricos, lo que permite el análisis de tendencias y el reconocimiento de patrones.

<Callout variant="important">
  Debe habilitar el rastreo distribuido para emplear Transaction 360. La mayor parte de la experiencia está disponible para todos; sin embargo, los componentes Mapas avanzados y Mapa de flujo de traza en la vista Carga de trabajo de transacciones requieren Calcular avanzado.
</Callout>

## Empezando

Para comenzar a emplear Transaction 360, debe habilitar el rastreo distribuido en su cuenta New Relic . Una vez habilitada, podrás acceder a la función Transacción 360 desde la página de inicio New Relic One.
