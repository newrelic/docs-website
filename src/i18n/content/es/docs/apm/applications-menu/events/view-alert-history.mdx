La página <DNT>**Alerts**</DNT> de APM muestra una tabla de alertas de las últimas dos semanas que puede desglosar en información detallada, incluido el rendimiento, el porcentaje de tasa de errores y el historial. Puede ver el historial de alertas de una aplicación específica o de todas las aplicaciones.

## Alertas para aplicaciones específicas [#alert_procedures]

Para ver una lista de alertas para una aplicación específica: vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & Services > (select an app) > Events > Issues & activity**</DNT>.

La página <DNT>**Issues & Activity**</DNT> muestra una lista de problemas, incidentes, anomalías y autopsias, si las hubiera. Utilice cualquiera de [las característicasUI ](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)estándar de New Relic para profundizar y obtener más detalles.

Cada notificación de alerta incluye el ícono de alerta e información resumida. Para ver detalles sobre una notificación de alerta en la página de alertas, seleccione su enlace. Por ejemplo:

* Para ver detalles sobre una alerta específica, seleccione su nombre.

* Para limitar la lista de alertas, utilice

  <DNT>
    **Filter**
  </DNT>

  .

* Para ver todas las alertas de todas sus aplicaciones y hosts, seleccione el enlace

  <DNT>
    **All alerts**
  </DNT>

  .

* Para seleccionar una máquina virtual Java (JVM) específica, si corresponde, seleccione su elección en el menú

  <DNT>
    **JVMs**
  </DNT>

  .
