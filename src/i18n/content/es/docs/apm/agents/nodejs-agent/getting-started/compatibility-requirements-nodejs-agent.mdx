Nuestro agente Node.js incluye instrumentación integrada del marco, servidores de aplicaciones, base de datos y sistemas de colas de mensajes de Node.js más populares. Para el marco y la biblioteca que no están instrumentados de fábrica, puede ampliar el agente con nuestra [API del agente Node.js.](/docs/apm/agents/nodejs-agent/api-guides/nodejs-agent-api/)

Nuestro agente Node.js está disponible públicamente en el [repositorio de administrador de paquetes (npm) de Node](https://npmjs.org/package/newrelic) , así como en [GitHub](https://github.com/newrelic/node-newrelic). ¿Listo para probar el agente Node.js de New Relic? [¡Crea una cuenta New Relic ](https://newrelic.com/signup)!

## Requisitos para instalar el agente

Antes de [instalar el agente](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent), verifique que su sistema cumpla con sus requisitos mínimos. Para obtener el mejor rendimiento, utilice la última versión activa de soporte a largo plazo (LTS) de Node.js.

<CollapserGroup>
  <Collapser
    id="operating systems"
    title="Sistema operativo compatible"
  >
    El agente Node.js es compatible con el siguiente sistema operativo:

    * Linux
    * Sistema operativo inteligente
    * macOS 10.7 y superior
    * Windows Server 2008 y superior
  </Collapser>

  <Collapser
    id="host"
    title="Servicios de alojamiento"
  >
    También es compatible con estos servicios de hosting:

    * [Entorno flexible de Google App Engine (GAE)](/docs/agents/nodejs-agent/hosting-services/install-new-relic-nodejs-agent-gae-flexible-environment)
    * AWS EC2
    * [Microsoft Azure](/docs/nodejs/nodejs-agent-on-microsoft-azure)
    * [Heroku](/docs/nodejs/nodejs-agent-on-heroku)
  </Collapser>

  <Collapser
    id="security"
    title="Requerimientos de seguridad"
  >
    Como [medida de seguridad estándar para la recopilación de datos](/docs/accounts-partnerships/accounts/security/data-security), su servidor de aplicaciones debe admitir SHA-2 (256 bits). SHA-1 no es compatible.
  </Collapser>

## instrumentar con Node.js [#instrument]

Después de la instalación, el agente se instrumenta automáticamente con nuestro catálogo de biblioteca y marco Node.js compatibles. Esto le brinda acceso inmediato a información granular específica de sus aplicaciones y servidores web.

Para marcos o bibliotecas no compatibles, deberá instrumentar el agente usted mismo utilizando la [API del agente Node.js.](/docs/apm/agents/nodejs-agent/api-guides/nodejs-agent-api)

    id="langtech"
    title="Idiomas y tecnologías compatibles"
  >
    [TypeScript](https://www.npmjs.com/package/typescript) es un lenguaje de programación que puede compilarse en JavaScript. Después de la instalación, el agente de Node.js instrumentó automáticamente sus aplicaciones TypeScript, brindándole acceso inmediato a los datos de rendimiento desde el primer momento.

    [Los módulos ES](https://nodejs.org/api/esm.html#introduction) son el estándar oficial para empaquetar código JavaScript para compartirlo/reutilizarlo. El agente de Node.js tiene <DNT>**experimental**</DNT> soporte para la aplicación del módulo ES que ejecuta Node.js versión 16.12.0 o superior y versión del agente 9.1.0 o mas alto. Consulte nuestra [documentación del módulo ES](/docs/apm/agents/nodejs-agent/installation-configuration/es-modules) para obtener información e instrucciones adicionales.
