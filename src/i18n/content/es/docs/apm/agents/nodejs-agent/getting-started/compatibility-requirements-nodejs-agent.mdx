Nuestro agente Node.js incluye instrumentación incorporada del marco Node.js más popular, servidores de aplicaciones, bases de datos y sistemas de colas de mensajes. Para los frameworks y bibliotecas que no vienen instrumentados de fábrica, puedes ampliar el agente con nuestra [Node.js API del agente](https://newrelic.github.io/node-newrelic/API.html)

Nuestro agente Node.js está disponible públicamente en el [repositorio de administrador de paquetes (npm) de Node](https://npmjs.org/package/newrelic) , así como en [GitHub](https://github.com/newrelic/node-newrelic). ¿Listo para probar el agente Node.js de New Relic? [¡Crea una cuenta New Relic ](https://newrelic.com/signup)!

## Requisitos para instalar el agente

Antes de [instalar el agente](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent), verifique que su sistema cumpla con sus requisitos mínimos. Para obtener el mejor rendimiento, utilice la última versión activa de soporte a largo plazo (LTS) de Node.js.

<CollapserGroup>
  <Collapser id="operating systems" title="Sistema operativo compatible">
    El agente Node.js es compatible con los siguientes sistemas operativos:

    * Linux
    * Sistema operativo inteligente
    * macOS 10.7 y superior
    * Windows Server 2008 y superior
  </Collapser>

  <Collapser id="host" title="Servicios de alojamiento">
    También es compatible con estos servicios de hosting:

    * [Entorno flexible de Google App Engine (GAE)](/docs/agents/nodejs-agent/hosting-services/install-new-relic-nodejs-agent-gae-flexible-environment)
    * AWS EC2
    * [Microsoft Azure](/docs/nodejs/nodejs-agent-on-microsoft-azure)
    * [Heroku](/docs/nodejs/nodejs-agent-on-heroku)
  </Collapser>

  <Collapser id="security" title="Requerimientos de seguridad">
    Como [medida de seguridad estándar para la recopilación de datos](/docs/accounts-partnerships/accounts/security/data-security), su servidor de aplicaciones debe admitir SHA-2 (256 bits). SHA-1 no es compatible.
  </Collapser>

Para un marco o biblioteca no compatible, deberá instrumentar el agente usted mismo empleando la [Node.js API del agente](https://newrelic.github.io/node-newrelic/API.html)

## Soporte de monitoreo de IA

El agente Node.js admite la siguiente plataforma e integración de IA.

### Amazon Bedrock

A través del módulo `@aws-sdk/client-bedrock-runtime` , admitimos todos los modelos (solo texto) a través de la API `Converse` ; para la API `InvokeModel` , admitimos los siguientes modelos:

| Modelo           | Imagen | Texto | Visión |
| ---------------- | ------ | ----- | ------ |
| Amazon Titan     | ❌      | ✅     | -      |
| Claude antrópico | ❌      | ✅     | ❌      |
| Adherir          | ❌      | ✅     | -      |
| Meta Llama3      | ❌      | ✅     | -      |

Nota: si un modelo admite transmisión, también instrumentamos la variante de transmisión.

### LangChain

Se admiten las siguientes características generales de LangChain:

| agente | cadena | Herramientas | Tiendas de vectores |
| ------ | ------ | ------------ | ------------------- |
| ✅      | ✅      | ✅            | ✅                   |

Los modelos/proveedores generalmente son soportados transitivamente por nuestra instrumentación del módulo del proveedor.

| Proveedor      | Soportado | Transitivamente |
| -------------- | --------- | --------------- |
| Azure OpenAI   | ❌         | ❌               |
| Amazon Bedrock | ❌         | ❌               |
| OpenAI         | ✅         | ✅               |

### OpenAI

| ❌     | ✅       | ✅              | ✅              | ❌        | ❌        |
