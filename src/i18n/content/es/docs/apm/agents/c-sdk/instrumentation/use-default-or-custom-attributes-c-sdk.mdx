En New Relic, <DNT>**attributes**</DNT> son pares de valores principales que contienen información que determina las propiedades de un [evento](/docs/accounts-partnerships/getting-started-new-relic/glossary#event) o [transacción](/docs/accounts-partnerships/getting-started-new-relic/glossary#transaction). Estos pares de valores principales pueden ayudarle a obtener mayor información valiosa en su aplicación y [consultar sus datos](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder).

## Ver y usar atributo [#locations]

Tanto [el atributo APM predeterminado](/docs/insights/new-relic-insights/decorating-events/apm-default-attributes-insights#transaction-event) como [el atributo personalizado](/docs/agents/manage-apm-agents/agent-data/collect-custom-attributes) para su aplicación C aparecen en:

* APM [traza de la transacción](/docs/apm/transactions/transaction-traces/introduction-transaction-traces) y [análisis de errores](/docs/apm/applications-menu/error-analytics/introduction-error-analytics)
* [Evento APM](/docs/insights/explore-data/attributes/apm-default-attributes-insights)

## Atributo específico de C [#attributes]

<Callout variant="important">
  Antes de crear un atributo personalizado, revise la lista de [términos reservados utilizados por NRQL](/docs/insights/insights-data-sources/custom-data/insights-custom-data-requirements-limits#reserved-words) de New Relic. De lo contrario, pueden producirse resultados inesperados.
</Callout>

Para agregar un atributo personalizado a su aplicación C, llame a una de [las funciones de atributo en la API del SDK de C](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#abfe6f64a8eec7d60d8588f8969781d34); por ejemplo, `newrelic_add_attribute_double()`. El nombre de clave para su atributo personalizado depende de lo que especifique cuando llame a la función.
