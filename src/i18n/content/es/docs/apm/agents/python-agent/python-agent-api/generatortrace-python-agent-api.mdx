## Sintaxis

```py
newrelic.agent.generator_trace(name=None, group=None, label=None, params=None, terminal=False)
```

Se utiliza para instrumentar generadores que no están instrumentados por defecto.

## Descripción

`generator_trace` es un decorador para agregar a funciones generadoras. Agregar este decorador le permite recopilar información adicional de la [transacción](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) (incluida la información de la traza de la transacción).

El decorador `generator_trace` se puede usar en generadores y corrutinas con la versión del agente 2.102.0.85 o superior. El registro de tiempo de estos objetos comienza cuando se inicia el consumo y termina cuando el objeto se agota o sale del ámbito.

Puede utilizar el decorador junto con decoradores existentes, incluidos los de métodos estáticos y de clase. Al anidar varios decoradores, tenga el decorador `generator_trace` como el decorador más externo.

## Formularios de llamada alternativos [#alternate]

Para configuraciones en las que no puede utilizar el decorador, estos formularios de llamada alternativos están disponibles:

### The wrapper

Si sabe de antemano dónde están los generadores específicos que desea trazar, podría usar el decorador `generator_trace`. Sin embargo, si <DNT>**don&apos;t**</DNT> conoce todos los generadores que necesitan traza (por ejemplo, si se buscan dinámicamente como parte de un sistema de enrutamiento), entonces debe usar el `GeneratorTraceWrapper` para envolver el generador en el momento del registro o en el momento de la llamada.

### Envoltura basada en rutas [#path-based]

`wrap_generator_trace` se utiliza para envolver generadores fuera del código en el que se declaran. Por ejemplo: podría usar esto para instrumentar el código de la biblioteca que no desea modificar.

Para obtener más información sobre las diferencias entre estos formatos de llamadas, consulte [Diferentes formatos de llamadas](/docs/python-agent-api-different-call-forms).

## Ejemplos

### `generator_trace` [#trace\_example][#trace_example]

Un ejemplo de uso del decorador `generator_trace` :

```py
import newrelic.agent

@newrelic.agent.generator_trace()
def gen():
    for i in range(5):
        yield i
```

Un ejemplo de uso del decorador `generator_trace` con un generador asíncrono:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
async def gen():
    for i in range(5):
        yield i
```

### Ejemplo de contenedor [#trace-wrap-example]

Un ejemplo de uso de `GeneratorTraceWrapper`:

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen)
```

Si desea nombrar la métrica con el nombre del extremo (en lugar de nombrar la métrica en función del identificador del generador que se está llamando), puede proporcionar el nombre a usar más un prefijo de ruta de métrica alternativo cuando se crea el objeto `GeneratorTraceWrapper`.

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name="gen_5", group="Python/Generator")
```

Si desea nombrar la métrica dinámicamente (en lugar de nombrar la métrica en función del identificador del generador que se está llamando), puede proporcionar el nombre como generador cuando se crea el objeto `GeneratorTraceWrapper`.

```py
import newrelic.agent

def gen(x):
    for i in range(x):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name = lambda x: f"gen_{x}")
```
