## Sintaxis

```py
newrelic.agent.database_trace(sql, dbapi2_module=None, async_wrapper=None)
```

Se utiliza para instrumentar llamadas a bases de datos.

## Descripción

`database_trace` se utiliza para agregar más detalles a sus [trazas de la transacción](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction-trace) en forma de segmentos adicionales. Todas las llamadas reportadas con `database_trace` aparecerán en la [página Bases de datos de APM](/docs/apm/applications-menu/monitoring/databases-slow-queries-page). `database_trace` devuelve un [partial](https://docs.python.org/3.12/library/functools.html#functools.partial) de `DatabaseTraceWrapper` que se puede usar como decorador de una función para medir el tiempo de las llamadas a su base de datos.

El decorador `database_trace` se puede usar en generadores y corrutinas con la versión del agente 2.102.0.85 o superior.

Si no puede utilizar el decorador en su aplicación, puede utilizar uno de estos otros formatos de llamada:

* <DNT>**The context manager**</DNT>: El formulario del administrador de contexto es `DatabaseTrace`.
* <DNT>**The wrapper**</DNT>: El formulario contenedor es `DatabaseTraceWrapper`. Se puede utilizar para devolver una función envuelta sin el uso de un decorador.
* <DNT>**The path-based wrapper**</DNT>: El formulario contenedor basado en ruta es `wrap_database_trace`. Esto aplica el `DatabaseTraceWrapper` a un objeto determinado mediante parches de mono. Esto toma los mismos parámetros que el decorador más un parámetro `module` y `object_path` adicional.

Para obtener una explicación de los usos de estos diferentes formatos de llamadas, consulte [Diferentes formatos de llamadas](/docs/python-agent-api-different-call-forms). Consulte [Ejemplos](#examples) para ver ejemplos de llamadas.

## Valores de retorno

`database_trace` devuelve un `DatabaseTraceWrapper()` parcial.

## Ejemplos

### `database_trace` [#trace\_example][#trace_example]

Un ejemplo de uso del decorador `database_trace` :

```py
import newrelic.agent

@newrelic.agent.database_trace(sql="SELECT * FROM users WHERE id = 1")
def get_user(id):
    ...
```

Un ejemplo del uso del decorador `database_trace` con rutinas nativas:

```py
import newrelic.agent

@newrelic.agent.database_trace(sql="SELECT * FROM users WHERE id = 1")
async def get_user(id):
    ...
```

### Ejemplo de administrador de contexto [#context-mgr-example]

Un ejemplo de uso del administrador de contexto `DatabaseTrace` : esto proporcionará tiempos de cuánto tiempo lleva realizar &quot;custom\_action&quot;.

```py
import newrelic.agent


def complex_query(a, b, c):
    with Connection(host, port, db) as conn:
        with newrelic.agent.DatabaseTrace(
            sql="SELECT * FROM users WHERE id = 1",
        ):

            conn.custom_action()
```

### Ejemplo de contenedor [#trace-wrap-example]

Un ejemplo de uso de `DatabaseTraceWrapper`:

```py
import newrelic.agent

def get(id):
    ...


get = newrelic.agent.DatabaseTraceWrapper(
    get, sql="SELECT * FROM users WHERE id = 1"
)
```
