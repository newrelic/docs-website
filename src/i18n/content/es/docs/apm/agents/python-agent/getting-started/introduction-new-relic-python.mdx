Nuestro agente Python monitorea su aplicación Python para ayudarlo a identificar y resolver [problemas de rendimiento](#monitor-performance). También puede ampliar su monitoreo de rendimiento para recopilar y analizar [datos comerciales](#business-data) para ayudarlo a mejorar la experiencia de los clientes y tomar decisiones comerciales impulsadas por datos. Con opciones flexibles para [instrumentación personalizada y API](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation), el agente Python ofrece múltiples componentes básicos para personalizar los datos que necesita de su aplicación.

Nuestro Python funciona con una amplia variedad de [marcos web y mecanismos de alojamiento](/docs/agents/python-agent/getting-started/instrumented-python-packages), incluidos Django, Gunicorn, WSGI, CherryPy, uWSGI y más. También puede instalar el agente Python en un [entorno flexible de Google App Engine](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment).

## monitor el rendimiento de la aplicación [#monitor-performance]

Después de [instalar](#installation) el agente Python, comienza a recopilar datos sobre su aplicación. Puede ver los datos como gráficos y tablas en la [UIde New Relic](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one).

<DNT>
  **View the big picture of your app:**
</DNT>

* monitor [el Apdex (satisfacción del usuario)](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) de su aplicación.
* Obtenga un resumen de alto nivel de su aplicación con [la página Resumen](/docs/apm/applications-menu/monitoring/apm-overview-page).
* Habilite [rastreo distribuido](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) para ver la actividad en una arquitectura que tiene muchos servicios.
* Instale [el monitoreo de infraestructura](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure) y vea datos detallados del servidor/host para su aplicación.

<DNT>
  **Find errors and problems quickly:**
</DNT>

* Seguimiento [de claves de transacción](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events) específicas para su negocio.
* Crea [un tablero personalizado](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards) para métricas importantes.
* [Alerta](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) a tu equipo cuando ocurre un error o problema antes de que afecte a tu usuario.
* Ver el rendimiento [después de un despliegue](/docs/apm/applications-menu/events/deployments-page).

<DNT>
  **Drill down into performance details:**
</DNT>

* Examinar [la traza de la transacción](/docs/apm/transactions/transaction-traces/transaction-traces) a nivel de código.
* Examine [consulta de la base de datos traza](/apm/transactions/transaction-traces/troubleshoot-sql-statements).
* Examinar [la traza del error](/docs/apm/applications-menu/events/view-apm-error-analytics).
* Utilice [sesiones de generador de perfiles de subprocesos](/docs/apm/applications-menu/events/thread-profiler-tool) para ver el rastreo detallado de la pila de subprocesos muestreados

<DNT>
  **View logs for your APM and infrastructure data:**

Reúna los datos de su registro y de la aplicación para que la resolución de problemas sea más fácil y rápida. No es necesario cambiar a otra página de UI.

* Con [el contexto de inicio de sesión](/docs/logs/logs-context/configure-logs-context-python/), puede ver el mensaje de registro relacionado con sus errores y la traza directamente en UI de su aplicación.
* También puede ver el inicio de sesión en el contexto de [los datos de su infraestructura](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), como el clúster de Kubernetes.

## Instalar el agente Python [#installation]

Antes de instalar el agente Python, asegúrese de que su sistema cumpla con el [requisito del sistema](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent). También debes crear una [cuenta New Relic](https://newrelic.com/signup).

Admitimos una serie de marcos web y bibliotecas desde el primer momento, incluidos Django, WSGI y Gunicorn. Si utiliza uno de los [marcos web compatibles](/docs/agents/python-agent/getting-started/instrumented-python-packages), la instalación es sencilla. Si utiliza un framework no compatible, el proceso implicará algunas adiciones al código de su aplicación y/o archivos del servidor web.

Para un proceso de instalación rápido y sencillo que funcionará para la mayoría de las configuraciones, siga estos sencillos pasos:

1. Descargue e instale el paquete Python.
2. Crear archivo de configuración.
3. Integre el agente Python con su aplicación.

## monitor secuencias de comandos no web, tareas en segundo plano y funciones [#non-web]

El agente Python también le permite monitor secuencias de comandos no web, procesos de trabajo, tareas y funciones. El proceso de instalación de estas transacciones no web es similar al que se utiliza para una aplicación web, con una diferencia importante: en lugar de pasar por el proceso de integración estándar descrito en las [instrucciones de instalación](/docs/agents/python-agent/getting-started/python-agent-quick-start), deberá "ajustar" manualmente cualquier función que desee monitorear. . Para obtener más información, consulte [Tareas y procesos no web](/docs/agents/python-agent/supported-features/python-background-tasks). Para obtener instrucciones sobre cómo monitorear las tareas de Celery, consulte [Tareas en segundo plano de Celery](/docs/agents/python-agent/back-end-services/python-agent-celery).

## ¿Qué sigue después de la instalación? [#instrumentation]

Una vez que tenga el agente en funcionamiento, algunos de los siguientes pasos sugeridos son:

* Explore sus datos y siéntase cómodo con la interfaz de usuario.
* Lea nuestros documentos sobre [nuestras otras soluciones de observabilidad](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability) y la [página de APM](/docs/apm).
* [Cambie el nombre de su aplicación](/docs/agents/manage-apm-agents/app-naming/rename-your-application) u otras [opciones de configuración](/docs/agents/python-agent/installation-configuration/python-agent-configuration).
* Obtenga información sobre cómo configurar [instrumentación personalizada](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation) para la actividad de la aplicación que no se monitorea de forma predeterminada.
* Considere el [SDK de telemetría de Python](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic).

## Resolución de problemas

Después de completar el proceso de instalación, sus datos deberían aparecer en la [UIde APM](/docs/apm/applications-menu/monitoring/apm-overview-page) en cinco minutos. Si no es así, utilice estos recursos de resolución de problemas:

* Si no aparecen datos, sigue estos [pasos de resolución de problemas](/docs/agents/python-agent/troubleshooting/no-data-appears-python).
* Si tiene problemas al instalar o ejecutar el agente Python en un nuevo host, [pruebe que el paquete esté instalado correctamente](/docs/agents/python-agent/installation-configuration/testing-python-agent) y que pueda comunicarse con el servicio de recolección de datos de New Relic.
* Para otros problemas, consulte la [lista completa de documentación de resolución de problemas](/docs/agents/python-agent/troubleshooting/).
