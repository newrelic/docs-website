## Sintaxis [#syntax]

```py
get_browser_timing_header(nonce=None)
```

Genera un encabezado de agente <InlinePopover type="browser"/>para inyectar el agente de JavaScript.

## Descripción [#description]

<Callout variant="important">
  El parámetro nonce para compatibilidad con CSP no autorizado se agregó en la versión 9.4.0 del agente Python.

  En las versiones 9.3.0 y anteriores del agente Python, la API [`get_browser_timing_footer`](/docs/agents/python-agent/python-agent-api/get_browser_timing_footer) también era necesaria para habilitar la sincronización browser . A partir de la versión 9.4.0 y posteriores del agente, esto ya no es necesario.
</Callout>

Esta llamada devuelve el fragmento de encabezado JavaScript del agente del navegador que puede utilizar para habilitar [el monitoreo del navegador](/docs/agents/python-agent/supported-features/page-load-timing-python). Puede utilizar esto si la instrumentación automática no es posible para su aplicación, si utiliza una política de seguridad de contenido no especificada o si desea recopilar datos del navegador solo para páginas específicas.

Esta es una alternativa manual a [la inserción automática de este JavaScript](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser#select-apm-app) por parte del agente. Para aprender cómo habilitar el monitoreo del navegador usando esta llamada junto con `get_browser_timing_footer`, consulte [Instrumento manual de monitoreo del navegador](/docs/agents/python-agent/supported-features/page-load-timing-python#manual_instrumentation).

Esta llamada debe realizarse para cada solicitud web en la que desee inyectar el fragmento de JavaScript. <DNT>**cannot**</DNT> se llamará en el ámbito global y se almacenará en caché.
