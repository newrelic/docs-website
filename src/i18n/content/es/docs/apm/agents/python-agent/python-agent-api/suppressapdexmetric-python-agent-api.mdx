## Sintaxis

```py
newrelic.agent.suppress_apdex_metric(flag=True)
```

Evita que la transacción actual contribuya a Apdex.

## Descripción

Esta llamada suprime la generación de la [métrica Apdex](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) para una transacción web. Es posible que desee hacer esto cuando una transacción de larga duración afecte con frecuencia la puntuación Apdex promedio de su aplicación.

Para cancelar la supresión de una transacción previamente suprimida, establezca el indicador en `False`.

<Callout variant="tip">
  También puede suprimir la métrica Apdex en el diccionario ambiental de WSGI. Para hacerlo, establezca la clave `newrelic.suppress_apdex_metric` para la solicitud específica en el diccionario de entorno WSGI pasado por el servidor WSGI a la aplicación WSGI que se está monitoreando.
</Callout>

## Valores de retorno

Ninguno.

## Ejemplos

### Desactive Apdex para transacciones específicas [#apdex-example]

Si tiene funciones frecuentes y de larga duración en su aplicación que están causando que su Apdex promedio baje, puede llamar a `suppress_apdex_metric` donde se genera la transacción:

```py
import newrelic.agent
newrelic.agent.suppress_apdex_metric()
```
