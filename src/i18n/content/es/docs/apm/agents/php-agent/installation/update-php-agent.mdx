Para aprovechar al máximo las últimas características, mejoras y parches de seguridad importantes de New Relic, le recomendamos que actualice su agente PHP a la última versión. Para obtener información adicional sobre actualizaciones de agentes específicos, consulte las [notas de la versión del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes).

Este documento explica cómo actualizar el agente para:

* RedHat y CentOS
* Ubuntu y Debian

Para actualizar a través de <DNT>**tar archive**</DNT>, siga los [procedimientos de instalación del archivo tar](/docs/agents/php-agent/installation/php-agent-installation-tar-file). (Los procedimientos para instalar y actualizar con el archivo tar son los mismos).

## Utilice el paquete `newrelic-php5` [#newrelic-php5]

El nombre del paquete para el agente PHP de New Relic es `newrelic-php5`.

<Callout variant="tip">
  Aunque el nombre del paquete hace referencia a PHP 5, este paquete funciona para todas [las versiones de PHP compatibles](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).
</Callout>

2. RedHat o CentOS: ejecute el script `newrelic-install` y siga el [símbolo de confirmación del script de instalación](/docs/php/the-newrelic-install-script) antes de sobrescribir una instalación anterior.

   ```bash
   sudo newrelic-install install
   ```

   Ubuntu o Debian: siga el símbolo del administrador de paquetes para ejecutar el script `newrelic-install` como parte de su proceso de instalación.

3. Detenga cualquier proceso residual `newrelic-daemon` ejecutando

   ```
   kill $(pgrep newrelic-daemon)
   ```

   Si `newrelic-daemon` está configurado para [iniciarse en modo externo](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#selecting-external), utilice el administrador de servicios de su sistema operativo para reiniciarlo.

4. [Reinicie su servidor web](/docs/agents/php-agent/troubleshooting/why-when-you-must-restart-your-web-server) (Apache, Nginx, PHP-FPM, etc.).

5. Espere unos minutos hasta que su aplicación envíe datos a New Relic.

        </DNT>

          **PHP 5.2 is no longer supported.**
        </DNT>

        New Relic le recomienda encarecidamente que utilice una versión compatible de PHP (5.6 o superior). Si desea continuar ejecutando el agente PHP New Relic con PHP 5.2, le recomendamos utilizar la versión 6.9 del agente. Sin embargo, en este caso, sólo podemos ofrecer soporte limitado.
      </td>
    </tr>

        </DNT>
