## Problema

Actualizaste PHP a una [versión compatible](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release), pero el agente PHP dejó de funcionar o informar.

## Solución

Si actualiza su instalación de PHP a otra versión principal (por ejemplo, 5.3 a 5.4), debe reinstalar el agente PHP. Actualizar PHP a versiones menores no requiere reinstalar el agente.

Para reinstalar el agente, siga las [instrucciones de instalación](/docs/agents/php-agent/installation/php-agent-installation-overview) de su plataforma o utilice el comando `newrelic-install install` para iniciar <DNT>**[newrelic-install script](/docs/agents/php-agent/advanced-installation/using-newrelic-install-script)**</DNT>.

Si está utilizando un administrador de paquetes (por ejemplo, `apt-get upgrade`) para actualizar PHP sin especificar paquetes, es posible que también esté [actualizando el agente PHP de New Relic](/docs/agents/php-agent/installation/upgrading-php-agent) en el proceso. Para garantizar que la actualización se aplique correctamente:

1. Reinicie su servidor web (Apache, Nginx, PHP-FPM, etc.).
2. Espere cinco minutos para ver si el agente comienza a informar.
3. Si el agente aún no informa, [verifique la configuración del daemon](/docs/agents/php-agent/troubleshooting/verifying-php-daemon). Si está configurado para iniciarse en modo externo, debe [iniciar manualmente el daemon](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#daemon-external) para que el agente comience a generar informes.
