[New Relic para PHP](/docs/agents/php-agent/getting-started/introduction-new-relic-php) le ofrece varias formas de cambiar el nombre de la aplicación que se informa y se muestra en New Relic. Este documento le brinda una descripción general de esas opciones de nombres.

Para obtener una descripción general de los nombres de aplicaciones APM (no específicos de PHP), consulte [Asigne un nombre a su aplicación](/docs/agents/manage-apm-agents/app-naming/name-your-application).

## Establecer o cambiar el nombre de la aplicación [#naming-options]

New Relic diferencia la aplicación por su nombre. Cuando cambia el nombre con el que informa una aplicación, New Relic crea una nueva entidad de aplicación y rompe la continuidad de los datos entre la entidad nueva y la antigua. Para cambiar el nombre de una aplicación sin interrumpir la continuidad de los datos, utilice la opción de cambio de alias UI .

Estas son las formas de cambiar el nombre de su aplicación PHP en New Relic:

* [Configure el archivo `newrelic.ini` usando `newrelic.appname`](/docs/agents/php-agent/configuration/php-agent-configuration#inivar-appname): la opción recomendada y más utilizada.
* [Establecer configuraciones INI por directorio](/docs/agents/php-agent/configuration/php-directory-ini-settings): útil cuando tiene un servidor web que sirve varias aplicaciones y desea tener diferentes configuraciones o nombres para ellas. (Los procedimientos para [Apache](/docs/agents/php-agent/configuration/php-directory-ini-settings#perdir-apache), [PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#php-fpm_per-dir) y [NGINX en PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#PHP-FPM_nginix) están documentados).
* [Cambiar alias a través de UI](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-alias): esto cambia solo el alias en la UI, no el nombre de la aplicación subyacente que se informa. (Esta opción no interrumpe la continuidad de los datos en APM).
* [Cambiar a través de API](/docs/agents/php-agent/php-agent-api/newrelic_set_appname): una opción si tiene acceso para modificar el código de su aplicación, está de acuerdo con instrumentarlo directamente y considera aceptables las implicaciones de continuidad de la traza de esta llamada.

## Utilice varios nombres de aplicaciones [#multiple]

<Callout variant="important">
  La capacidad de configurar varios nombres de aplicaciones solo está disponible en las versiones 2.4 o superiores del agente PHP.
</Callout>

Puede utilizar varios nombres de aplicaciones para agregar datos de varias aplicaciones con el mismo nombre en APM (a veces denominado "acumular" sus datos). Para obtener más información sobre cómo funcionan varios nombres de aplicaciones, consulte [Usar varios nombres de aplicaciones](/docs/agents/manage-apm-agents/app-naming/use-multiple-names-app).

Para PHP, puede configurar hasta tres nombres de aplicaciones. El nombre de la aplicación principal es el primero, y el segundo y tercer nombre se utilizan para las categorías de agregación de datos más generales.
