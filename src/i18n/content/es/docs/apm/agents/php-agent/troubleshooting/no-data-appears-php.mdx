## Problema

Después de [instalar el agente PHP de New Relic](/docs/agents/php-agent/installation/php-agent-installation-overview), generar algo de tráfico y esperar al menos cinco minutos, no aparece ningún dato en su UI de New Relic.

## Solución

Si no aparecen datos después de generar tráfico a su aplicación y esperar al menos cinco minutos, use [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) para detectar automáticamente problemas comunes y sugerir resolución de problemas. Si eso no resuelve el problema, intente lo siguiente:

      </td>

    </tr>

    <tr>
      <td>
        Versión PHP no estándar
      </td>

      <td>
        Si está utilizando una versión no estándar de PHP, siga los [procedimientos de instalación avanzados](/docs/php/php-agent-installation-non-standard-php) para asegurarse de que el instalador predeterminado pueda encontrar su versión de PHP.
      </td>
    </tr>

    <tr>
      <td>
        Nombre de la aplicación
      </td>

      <td>
        Asegúrese de que sus aplicaciones tengan un [nombre único y descriptivo.](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name) Por ejemplo, si tiene varias aplicaciones con el mismo nombre, como el nombre de aplicación predeterminado `PHP Application`, los datos de cada una de estas aplicaciones se acumulan en el nombre de la aplicación predeterminada y puede parecer como si una aplicación individual no estuviera generando informes.
      </td>
    </tr>

    <tr>
      <td>
        Servidor web
      </td>

      <td>
        Reinicie su servidor web (Apache, Nginx, PHP-FPM, etc.) y espere unos minutos hasta que aparezcan los datos.
      </td>
    </tr>

    <tr>
      <td>
        `phpinfo()`
      </td>

      <td>
        Marque `phpinfo()` para [verificar que instaló el agente PHP](/docs/agents/php-agent/troubleshooting/using-phpinfo-verify-php) y que el <InlinePopover type="licenseKey"/>que contiene es correcto.
      </td>
    </tr>

    <tr>
      <td>
        SELinux
      </td>

      <td>
        Si su sistema usa SELinux, [configure SELinux](/docs/agents/php-agent/troubleshooting/data-stops-reporting-while-using-selinux#solution) para que funcione con el agente PHP.
      </td>
    </tr>

    <tr>
      <td>
        Archivo de registro
      </td>

      <td>
        1. Verifique que tanto el agente como el daemon estén [escribiendo en su archivo de registro](/docs/agents/php-agent/troubleshooting/generating-logs-troubleshooting-php).
        2. De lo contrario, [reinstale el agente](/docs/agents/php-agent/installation/php-agent-installation-overview).
        3. Verifique los permisos del archivo de registro para asegurarse de que el propietario log sea el mismo que el usuario de New Relic.
      </td>
    </tr>

    <tr>
      <td>
        Permisos del agente PHP
      </td>

      <td>
        Asegúrese de tener los [permisos correctos](/docs/agents/php-agent/troubleshooting/determining-permissions-requirements) para utilizar el agente PHP.

    </tr>
