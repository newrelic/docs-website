## Problema

Tiene problemas con la instalación de New Relic para PHP pero no está seguro de cuál es la causa.

## Solución

Monitorear sus New Relic Logs puede ayudar a diagnosticar el problema. El agente PHP y daemon de New Relic tienen su propio registro. Para monitor el registro:

1. Verifique ambos registros para ver el contenido:

   * <DNT>
       **[Default agent log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-logfile)**
     </DNT>

     : `/var/log/newrelic/php_agent.log`

   * <DNT>
       **[Default daemon log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)**
     </DNT>

     : `/var/log/newrelic/newrelic-daemon.log`

2. Establezca el nivel de registros para el agente en `verbosedebug` y establezca el nivel de registros para el daemon en `debug`. Para más información:

   * [Configurar el agente](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)

     <DNT>
       [**loglevel**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
     </DNT>

     [como](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)

     <DNT>
       [**verbosedebug**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
     </DNT>

   * [Establecer el daemon](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

     <DNT>
       [**loglevel**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)
     </DNT>

     [en](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

     <DNT>
       [**debug**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)
     </DNT>

3. Reinicia tu aplicación.

4. Reúna de 5 a 10 minutos de tiempo log con su nivel de registros configurado en los valores apropiados.

5. Examine su registro y utilice la información para diagnosticar el problema.

6. Cuando haya terminado de resolver problemas, restablezca el nivel de registros. No los mantenga en `verbosedebug` o `debug`.

<Callout variant="caution">
  Observe el tamaño de su archivo de registro. Dependiendo de su aplicación, pueden crecer muy rápidamente.
</Callout>
