Consulte [las notas de la versión del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes/) para conocer la última versión del agente PHP.

Para obtener una lista actualizada de todas las versiones de agente PHP compatibles actualmente, consulte la [Política de fin de vida del agente PHP](/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).

Si está utilizando una versión o plataforma PHP no compatible (como un sistema operativo de 32 bits, FreeBSD o compilaciones ZTS), le recomendamos deshabilitar los mecanismos de actualización automática para los paquetes del agente PHP. Si no evita las actualizaciones automáticas, esto podría provocar la instalación accidental de una versión más nueva del agente. Es posible que la versión más nueva ya no admita características críticas esenciales para la recopilación de datos APM, lo que provocará interrupciones en las capacidades de monitoreo.

Para desactivar la actualización automática, tiene dos opciones:

* Agregue los paquetes del agente PHP a una lista de exclusión para actualizaciones de paquetes, o

* Versión fije el paquete del agente PHP a una versión del agente que admita las características antiguas no compatibles. Los paquetes del agente PHP afectados son:

  * `newrelic-php5`
  * `newrelic-php5-common`
  * `newrelic-daemon`

## Seguridad de subprocesos de Zend [#zts]

No se admiten las compilaciones de PHP compiladas con Zend Thread Safety (ZTS).

## Compilación justo a tiempo (JIT) [#jit]

[La compilación Just-In-Time](https://php.watch/articles/jit-in-depth) es compatible a partir de la versión [10.18.0.8](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-10-18-0-8/) del agente. PHP deshabilita automáticamente JIT para cualquier versión anterior del agente PHP.

## Fibras [#fibers]

No se admiten [fibras](https://www.php.net/manual/en/language.fibers.php) .

## Permisos [#permissions]

Instalando el acceso root del agente PHP <DNT>**requires**</DNT> a:

* Acceso de lectura/escritura al directorio de extensiones PHP.
* Acceso de lectura/escritura a los datos de configuración de PHP, ya sea en el directorio de configuración de PHP (preferido) o en `php.ini`.
* Reinicie el servidor web o el grupo de procesos PHP.

Ejecutando el acceso raíz del agente <DNT>**does not require**</DNT> . El agente necesita acceso de lectura/escritura a los siguientes archivos (la ubicación de estos archivos se puede [personalizar en la configuración del agente PHP](/docs/php/php-agent-phpini-settings)):

* [Un archivo que almacena el agente archivo de log](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)
* [Un archivo que almacena el extremo del socket del agente.](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-port)
* [Un archivo que almacena el ID del proceso del agente.](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-pidfile)

## Clave de licencia [#license\_key][#license_key]

Su New Relic <InlinePopover type="licenseKey" />es una cadena hexadecimal de 40 caracteres que New Relic proporciona cuando se [registra](https://newrelic.com/signup) para obtener su cuenta. Para obtener más información, consulte el [script de instalación](/docs/php/the-newrelic-install-script) de PHP o [la configuración de la clave de licencia en el archivo PHP `ini` ](/docs/php/php-agent-phpini-settings#inivar-license).

## Procesadores [#processor]

Se admiten las siguientes arquitecturas de procesador:

* x86\_64 (también conocido como amd64)
* aarch64 (también conocido como ARM64)

<Callout variant="important">
  La compatibilidad con aarch64 (ARM64) solo está disponible en PHP 8.0+.
</Callout>

## Sistema operativo [#operating-systems]

Cuando los proveedores anuncien el fin de la vida útil (como en [la página Fin del soporte estándar de Ubuntu](https://wiki.ubuntu.com/Releases)), continuaremos brindando soporte a esas últimas versiones durante un año luego de la fecha de fin de la vida útil del proveedor. Sin embargo, si la versión de PHP que estás usando ya no tiene soporte oficial, el soporte podría finalizar antes de un año.

Es por eso que recomendamos utilizar siempre la última versión del sistema operativo compatible oficialmente con el proveedor. Las últimas versiones de nuestro agente pueden funcionar en versiones del sistema operativo que ya no están disponibles, pero ya no probamos ni brindamos soporte oficial al agente PHP con versiones anteriores.

Para deshabilitar la recopilación de información del host, configure una de las siguientes opciones en el archivo `newrelic.ini` :

* Deshabilitar informes de instancia con `newrelic.datastore_tracer.instance_reporting.enabled = false`
