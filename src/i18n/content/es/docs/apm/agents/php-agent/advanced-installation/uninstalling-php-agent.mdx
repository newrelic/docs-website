Continúe leyendo para aprender cómo desinstalar el agente PHP de New Relic. Para obtener instrucciones sobre cómo deshabilitar temporalmente el agente, consulte [Deshabilitar el agente](/docs/agents/manage-apm-agents/installation/disable-apm-agent#php).

Este documento explica cómo desinstalar el [agente PHP New Relic](/docs/agents/php-agent/getting-started/new-relic-php) para los siguientes sistemas:

* RedHat y CentOS
* Ubuntu y Debian
* Archivo tar (cualquier [sistema compatible](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems))

## Desinstalar el agente [#install-overview]

Al desinstalar New Relic de una aplicación PHP, debe eliminar el agente New Relic y también eliminar los archivos de configuración asociados de New Relic.

Para desinstalar el agente PHP de su sistema:

1. Elimine los archivos del agente de su sistema:

   <CollapserGroup>
     <Collapser
       id="remove-redhat-centos"
       title="RedHat o CentOS"
     >
       Ejecute el script `newrelic-install` sin opciones y seleccione la opción de desinstalación en el menú principal al invocar el script.
     </Collapser>

     <Collapser
       id="remove-ubuntu-debian"
       title="Ubuntu o Debian"
     >
       Utilice `apt-get` para eliminar el agente:

       ```bash
       apt-get remove newrelic-php5
       ```
     </Collapser>
   </CollapserGroup>

2. Elimine los archivos de configuración restantes de New Relic. Utilice los comandos `newrelic-install` o `apt-get remove` con la opción `purge` .

   ```bash
   newrelic-install purge
   ```

   O

   ```bash
   apt-get remove --purge newrelic
   ```

3. Si utiliza un administrador de paquetes, elimine los archivos específicos del administrador:

   <CollapserGroup>
     <Collapser
       id="uninstall-redhat-centos"
       title="RedHat o CentOS"
     >
       Usando `yum`:

       ```bash
       sudo yum remove newrelic-php5
       ```

       Usando <DNT>**`rpm` (32-bit)**</DNT>:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (YOUR VERSION #)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```

       Usando <DNT>**`rpm` (64-bit)**</DNT>:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (VERSION NUMBER)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="uninstall-ubuntu-debian"
       title="Ubuntu o Debian"
     >
       Usando `apt-get`:

       ```bash
       sudo apt-get purge newrelic-php5
       ```

       Usando <DNT>**`dpkg` (32-bit)**</DNT>:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_i386.deb newrelic-php5_X.X.X.X_i386.deb
       ```

       Usando <DNT>**`dpkg` (64-bit)**</DNT>:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_amd64.deb newrelic-php5_X.X.X.X_amd64.deb
       ```
     </Collapser>
   </CollapserGroup>

4. Reinicie sus servidores web (Apache, Nginx, PHP-FPM, etc.).
