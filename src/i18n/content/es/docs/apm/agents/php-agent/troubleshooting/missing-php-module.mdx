## Problema

<DNT>**newrelic**</DNT> La sección del módulo no aparece en `phpinfo()`.

## Solución

Una sección de descripción del módulo `newrelic` que falta en la página `phpinfo()` indica que es posible que el archivo <DNT>**`newrelic.so`**</DNT> no se haya instalado en el directorio de extensiones. Para determinar si el módulo PHP New Relic existe:

1. En su [página`phpinfo()` ](/docs/php/php-kb-103), busque la sección

   <DNT>
     **PHP Core**
   </DNT>

   .

2. Para identificar el directorio desde donde se cargan sus extensiones PHP, busque el valor local de la configuración `extension_dir` .

3. Verifique que `newrelic.so` esté presente en el directorio especificado por `extension_dir`.

Si el archivo `newrelic.so` no está presente en su directorio de extensiones, haga lo siguiente:

* Siga [los procedimientos de instalación de PHP no estándar](/docs/php/php-agent-installation-non-standard-php) de New Relic.

  O

* Vincula manualmente la versión correcta desde `/usr/lib/newrelic-php5/agent/x32` para 32 bits o `/usr/lib/newrelic-php5/agent/x64` para 64 bits. O, si realizó la instalación desde un archivo tar, copie o vincule desde el directorio <DNT>**agent**</DNT> donde extrajo el archivo tar.
