En general, los planes de hosting básicos no permiten la instalación estándar del agente PHP de New Relic, porque se requieren permisos de administrador (root) para instalarlo. Su proveedor de alojamiento puede permitir o proporcionar la instalación de marcos/aplicaciones web comunes (como WordPress, Drupal, Joomla, etc.), pero la instalación del agente PHP aún requiere permisos no permitidos en muchos planes básicos.

Si es necesario, comuníquese con los desarrolladores/proveedor de alojamiento de su sitio web para obtener asistencia técnica.

## Servicios de alojamiento compartido [#hosts]

En general, los proveedores de hosting compartido (que no brindan acceso a usuario raíz de línea de comando) y los proveedores de hosting que no tienen un sistema operativo tradicional (como PaaS) no permiten instalar New Relic agente a través de los canales típicos. Por ejemplo, estos proveedores de hosting **no** admiten la integración de New Relic para ciertos niveles de suscripción:

* DreamHost: alojamiento web compartido
* Blog alojado en WordPress.com
* GoDaddy: alojamiento de blogs de WordPress económico, de lujo, definitivo
* Alojamiento web HostGator

Antes de instalar el agente PHP de New Relic, asegúrese de que su nivel de servicio de alojamiento sea compatible con New Relic. También asegúrese de tener permisos de administrador para los procedimientos de instalación desde la línea de comando en el nivel raíz.

## Consejos de instalación y resolución de problemas. [#installation]

Para ayudar a garantizar que su aplicación pueda integrarse exitosamente con New Relic:

1. Asegúrese de que su sistema cumpla con [los requisitos de compatibilidad PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) de New Relic.

   <Callout variant="important">
     El sistema operativo Windows no es compatible.
   </Callout>

2. Siga [los procedimientos de instalación de PHP](/docs/agents/php-agent/installation/php-agent-installation-overview) de New Relic.

3. Si su sistema cumple con los requisitos de compatibilidad de New Relic pero su instalación no tiene éxito, recopile la siguiente información:

   * Pasos exactos que probaste
   * Resultados, incluidos mensajes de éxito o fracaso.
   * Su sistema operativo y versión (por ejemplo, Ubuntu Linux versión 12.04 de 64 bits)
   * Su proveedor de servicios de hosting y nivel de suscripción

4. Si corresponde, comuníquese con los desarrolladores de su sitio web para obtener ayuda.
