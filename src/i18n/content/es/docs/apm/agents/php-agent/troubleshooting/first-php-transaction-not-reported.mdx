## Problema

Instaló el agente PHP New Relic y generó algo de tráfico, pero el agente no envía la primera transacción.

## Solución

Este es el comportamiento esperado. Cuando el [daemon PHP](/docs/agents/php-agent/getting-started/new-relic-daemon-processes) está en su estado inicial, debe ocurrir <DNT>**two**</DNT> transacción antes de que comience a informar:

* Con la primera transacción, el daemon PHP agrega su aplicación recién informada a su propia tabla de información de la aplicación.
* Una vez que se agrega la aplicación, el daemon PHP informa la segunda transacción a New Relic.

<DNT>**Recommendation:**</DNT> Ejecute un proceso php trivial, como `phpinfo()`, antes de ejecutar los procesos que desea informar.

## Causa

Este es el comportamiento esperado del daemon PHP, que inicialmente requiere que se informen dos transacciones.
