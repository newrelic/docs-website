<Callout variant="important">
  La clave de firma RPM del agente PHP de New Relic se rotará en marzo de 2026. Para continuar usando el agente sin interrupciones, la nueva clave debe importarse en todos los sistemas que realizan la instalación mediante rpm/yum con el siguiente comando. Para más información, consulte [Instalación del agente PHP: rotación de claves RPM 2026](/docs/apm/agents/php-agent/installation/php-agent-installation-rpm-key-rotation).

  `rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public`
</Callout>

Nuestro agente PHP instrumenta automáticamente su código para que pueda comenzar a monitorear aplicaciones. Siga este procedimiento para instalar el agente PHP de New Relic para APM en Amazon Linux 2, RedHat o CentOS en arquitectura x86\_64. Para la instalación en arquitectura aarch64, utilice el método de [archivo tar](/docs/php/php-agent-installation-tar-files). <DNT>**Exception:**</DNT> Si tiene una versión anterior instalada, [actualice el agente](/docs/agents/php-agent/installation/upgrading-php-agent) en su lugar.

## Instalar el agente [#install]

Aunque el nombre del paquete del agente PHP de New Relic se refiere a PHP 5, el paquete funciona para todas [las versiones de PHP compatibles](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).

1. Asegúrate de tener tu <InlinePopover type="licenseKey" />accesible.

2. Agregue la nueva clave RPM (Solo aplicable a partir de marzo de 2026).

   ```bash
   rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public
   ```

3. Utilice cualquiera de las siguientes formas para obtener el paquete de instalación:

   <CollapserGroup>
     <Collapser
       id="tell-rpm"
       title={<>Informe al administrador de paquetes (<InlineCode>
         rpm
       </InlineCode>) sobre el repositorio New Relic</>
       }
     >
       ```bash
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-3.noarch.rpm
       ```
     </Collapser>

     <Collapser id="download-rpm" title="Descargue el archivo rpm de New Relic">
       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```
       * ```
         newrelic-daemon-X.X.X.X-1.x86_64.rpm
         ```
       * ```
         newrelic-php5-X.X.X.X-1.x86_64.rpm
         ```
     </Collapser>
   </CollapserGroup>

4. Instale el agente y daemon usando su administrador de paquetes preferido:

   <CollapserGroup>
     <Collapser id="manager-yum" title="yum">
       ```bash
       sudo yum install newrelic-php5
       ```

       La primera vez que instale New Relic para PHP, use el símbolo yum para aceptar la clave pública de New Relic. El ID de clave de New Relic es `548C16BF`.
     </Collapser>

     <Collapser id="manager-rpm" title="rpm">
       Reemplace `X.X.X.X` con el [número de versión más reciente del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes) cuando ejecute este comando:

       ```bash
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser id="manager-tarball" title="tarball">
       Si <DNT>**yum**</DNT> y <DNT>**rpm**</DNT> no funcionan con la configuración de su host, [instálelos desde el archivo tarball binario](/docs/php/php-agent-installation-tar-files).
     </Collapser>
   </CollapserGroup>

5. Ejecute el script `newrelic-install` y siga las instrucciones.

   ```bash
   sudo newrelic-install install
   ```

6. Reinicie su servidor web (Apache, NGINX, PHP-FPM, etc.).

7. Genere tráfico a su aplicación y espere unos minutos hasta que envíe datos a New Relic. Luego, [verifique el rendimiento de su aplicación en la UI](/docs/apm/applications-menu/monitoring/apm-overview-page).
