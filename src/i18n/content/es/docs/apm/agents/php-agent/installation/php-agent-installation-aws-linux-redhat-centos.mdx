Nuestro agente PHP auto-instrumentado su código para que pueda iniciar la aplicación de monitoreo. Siga este procedimiento para instalar el agente PHP de New Relic para APM utilizando Amazon Linux 2, RedHat o CentOS. <DNT>**Exception:**</DNT> Si tiene instalada una versión anterior, [actualice el agente](/docs/agents/php-agent/installation/upgrading-php-agent) .

## Instalar el agente [#install]

Aunque el nombre del paquete del agente PHP de New Relic se refiere a PHP 5, el paquete funciona para todas [las versiones de PHP compatibles](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).

1. Asegúrate de tener tu <InlinePopover type="licenseKey"/>accesible.

2. Utilice cualquiera de las siguientes formas para obtener el paquete de instalación:

   <CollapserGroup>
     <Collapser
       id="tell-rpm"
       title={<>Informe al administrador de paquetes (<InlineCode>rpm</InlineCode>) sobre el repositorio New Relic</>}
     >
       Para <DNT>**32-bit**</DNT> sistemas, ejecute:

       ```bash
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/i386/newrelic-repo-5-3.noarch.rpm
       ```

       Para <DNT>**64-bit**</DNT> sistemas, ejecute:

       ```bash
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-3.noarch.rpm
       ```
     </Collapser>

     <Collapser
       id="download-rpm"
       title="Descargue el archivo rpm de New Relic"
     >
       Para sistemas <DNT>**32-bit**</DNT> , descargue estos tres archivos de los [paquetes de 32 bits](https://download.newrelic.com/pub/newrelic/el5/i386/) (reemplazando `X.X.X.X` con el [número de versión más reciente del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes)):

       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```

       * ```
         newrelic-daemon-X.X.X.X-1.i386.rpm
         ```

       * ```
           newrelic-php5-X.X.X.X-1.i386.rpm
         ```

         Para sistemas <DNT>**64-bit**</DNT> , descargue estos tres archivos de los [paquetes de 64 bits](https://download.newrelic.com/pub/newrelic/el5/x86_64/) (reemplazando `X.X.X.X` con el [número de versión más reciente del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes)):

       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```

       * ```
         newrelic-daemon-X.X.X.X-1.x86_64.rpm
         ```

       * ```
         newrelic-php5-X.X.X.X-1.x86_64.rpm
         ```
     </Collapser>
   </CollapserGroup>

3. Instale el agente y daemon usando su administrador de paquetes preferido:

   <CollapserGroup>
     <Collapser
       id="manager-yum"
       title="yum"
     >
       ```bash
       sudo yum install newrelic-php5
       ```

       La primera vez que instale New Relic para PHP, use el símbolo yum para aceptar la clave pública de New Relic. El ID de clave de New Relic es `548C16BF`.
     </Collapser>

     <Collapser
       id="manager-32-bit-rpm"
       title="rpm de 32 bits"
     >
       Reemplace `X.X.X.X` con el [número de versión más reciente del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes) cuando ejecute este comando:

       ```bash
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```
     </Collapser>

     <Collapser
       id="manager-64-bit-rpm"
       title="rpm de 64 bits"
     >
       Reemplace `X.X.X.X` con el [número de versión más reciente del agente PHP](/docs/release-notes/agent-release-notes/php-release-notes) cuando ejecute este comando:

       ```bash
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="manager-tarball"
       title="tarball"
     >
       Si <DNT>**yum**</DNT> y <DNT>**rpm**</DNT> no funcionan con la configuración de su host, [instálelos desde el archivo tarball binario](/docs/php/php-agent-installation-tar-files).
     </Collapser>
   </CollapserGroup>

4. Ejecute el script `newrelic-install` y siga las instrucciones.

   ```bash
   sudo newrelic-install install
   ```

5. Reinicie su servidor web (Apache, NGINX, PHP-FPM, etc.).

6. Genere tráfico a su aplicación y espere unos minutos hasta que envíe datos a New Relic. Luego, [verifique el rendimiento de su aplicación en la UI](/docs/apm/applications-menu/monitoring/apm-overview-page).
