## Problema

El agente PHP de New Relic no admite ningún modelo de multiprocesamiento (MPM) de Apache que implique subprocesos.

## Solución

El equipo de desarrollo del lenguaje PHP [desaconseja el uso de un MPM con subprocesos con Apache](http://www.php.net/manual/en/install.unix.apache2.php "El enlace se abre en una ventana nueva.") y hemos optado por no implementar soporte para MPM con subprocesos en el agente PHP por razones de estabilidad similares. Esto incluye los MPM <DNT>**worker**</DNT> y <DNT>**event**</DNT> .

Si el MPM <DNT>**prefork**</DNT> predeterminado y sin subprocesos no es adecuado para sus necesidades por motivos de rendimiento, le sugerimos utilizar <DNT>**mod_fcgid**</DNT> y <DNT>**php-fpm**</DNT> para utilizar PHP a través de FastCGI.
