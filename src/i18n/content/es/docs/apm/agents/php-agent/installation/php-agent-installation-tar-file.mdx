Puede instalar el agente PHP de New Relic desde archivos tar comprimidos en cualquier plataforma compatible. Sin embargo, para RedHat o CentOS es más común utilizar el [paquete RPM](/docs/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos/). Para Ubuntu o Debian es más común utilizar el [paquete Debian](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian).

## Descargar la distribución tar [#download]

A diferencia de otras opciones de instalación, los archivos tar no requieren ninguna configuración de repositorio especial. Todo lo que necesitas hacer es descargar el archivo y seguir estas instrucciones.

Descargue el archivo de distribución tar apropiado desde <DNT>**[download.newrelic.com/php_agent/release/](https://download.newrelic.com/php_agent/release/)**</DNT>. Por ejemplo:

* Para FreeBSD, descargue `newrelic-php5-X.X.X.X-freebsd.tar.gz`.
* Para Alpine Linux, descargue `newrelic-php5-X.X.X.X-linux-musl.tar.gz`.

<Callout variant="tip">
  El nombre del paquete para el agente PHP es `newrelic-php5`. Aunque el nombre del paquete hace referencia a PHP 5, este paquete funciona para todas [las versiones de PHP compatibles](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release).
</Callout>

## Instalar o actualizar [#install]

Con la instalación del archivo tar, los pasos para la instalación por primera vez y para la actualización son los mismos. Reemplace XXXX con la versión correcta de New Relic. Para instalar o actualizar el agente:

1. Descargue el archivo tar apropiado del [sitio web de New Relic](https://download.newrelic.com/php_agent/release/) y guárdelo en un disco local en una ubicación conveniente. No utilice la carpeta `/tmp` .

2. Descomprime y extrae el archivo:

   ```bash
   gzip -dc newrelic-php5-X.X.X.X-OS.tar.gz | tar xf -
   ```

3. Cambie al directorio recién creado:

   ```bash
   cd newrelic-php5-X.X.X.X-OS
   ```

4. Ejecute el script de instalación:

   ```bash
   ./newrelic-install
   ```

5. Cambie el [nombre de la aplicación](/docs/site/naming-your-application) predeterminada por un nombre significativo.

6. Reinicie su servidor web (Apache, Nginx, PHP-FPM, etc.).

7. Espere unos minutos hasta que su aplicación envíe datos y luego [verifique el rendimiento de su aplicación en la UI](/docs/apm/applications-menu/monitoring/apm-overview-page).
