## Problema

New Relic no informa datos a los pocos minutos de reiniciar su servidor web después de instalar el agente PHP.

## Solución

Utilice `phpinfo()` para verificar que el agente se instaló correctamente:

1. Cree una página web `.php` simple dentro de su sitio web que contenga lo siguiente:

   ```php
   <?php phpinfo(); ?>
   ```

2. Navegue a esta página en un browser web y desplácese por la lista alfabética de módulos para encontrar `newrelic`. Si no ve `newrelic` en esta lista, está utilizando una versión no estándar de PHP o no ha reiniciado su servidor web (u otro host PHP) desde que instaló New Relic.

3. Reinicie su servidor web para ver si `newrelic` aparece en su lista de módulos.

4. Si `newrelic` no aparece en su lista de módulos, [verifique su directorio de extensiones](/docs/php/php-kb-104) para ver si tiene una [versión no estándar de PHP](/docs/php/php-agent-installation-non-standard-php).

5. Si `newrelic` aparece en su lista de módulos, verifique que la entrada `license.key` muestre los primeros y últimos caracteres de su<InlinePopover type="licenseKey"/>

6. Si `license.key` no aparece, [verifique el directorio de archivos de configuración cargados](/docs/agents/php-agent/troubleshooting/checking-loaded-configuration-files-directory)
