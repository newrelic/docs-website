## Sintaxis

```php
newrelic_set_user_attributes(string $user_value, string $account_value, string $product_value)
```

Cree un atributo personalizado relacionado con el usuario. newrelic_add_custom_parameter es más flexible.

## Requisitos

Versión del agente [3.1.5.111](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-315111) o superior.

## Descripción

<Callout variant="tip">
  Esta llamada solo le permite asignar valores a claves preexistentes. Para obtener un método más flexible para crear pares de valores principales, utilice [`newrelic_add_custom_parameter`](/docs/agents/php-agent/php-agent-api/newrelic_add_custom_parameter).
</Callout>

A partir de la versión 4.4, llamando a:

```php
newrelic_set_user_attributes("a", "b", "c");
// is equivalent to calling these three methods:

newrelic_add_custom_parameter("user", "a");  
newrelic_add_custom_parameter("account", "b"); 
newrelic_add_custom_parameter("product", "c");
```

Los tres parámetros son obligatorios, pero pueden ser cadenas vacías.

## Valores de retorno

Esta función devolverá `true` si el atributo se agregó correctamente.

## Ejemplos

### Grabar tres atributos de usuario. [#record-three]

```php
function example() {
    if (extension_loaded('newrelic')) { // Ensure PHP agent is available
        newrelic_set_user_attributes("MyUserName", "MyAccountName", "MyProductName");
    }
}
```

### Registre dos atributos de usuario y un atributo vacío [#record-two]

```php
function example() {
    if (extension_loaded('newrelic')) { // Ensure PHP agent is available
        newrelic_set_user_attributes("MyUserName", "", "MyProductName");
    }
}
```
