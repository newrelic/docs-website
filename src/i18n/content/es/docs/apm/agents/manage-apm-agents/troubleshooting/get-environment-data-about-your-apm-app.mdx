## Problema

Necesita una forma sencilla de obtener información sobre el entorno de su aplicación, como por ejemplo:

* Qué servicios utiliza una versión de agente en particular
* ¿Qué aplicaciones tienen versiones de agente desactualizadas?
* Qué hosts usan qué agente
* ¿Qué valores se aplican a la configuración específica del agente?

## Solución

Para ver qué servicios y configuraciones se aplican a un agente, o para ver qué agente usa un servicio o configuración específicos, vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>. Haga clic en su aplicación en la lista y luego haga clic en <DNT>**Settings > Environment > Environment snapshot**</DNT>.

La página <DNT>**Environment snapshot**</DNT> le ayuda a conectar la carga del agente que incluye datos ambientales, para que tenga una idea de qué datos de monitoreo se registran. Esto también le ayuda a identificar más fácilmente las discrepancias entre sus servicios y determinar qué es posible que deba actualizarse.

En el siguiente ejemplo, el usuario pasa el mouse sobre el segundo agente de la lista. La mayoría de las configuraciones para este agente son consistentes con otros, pero es fácil ver configuraciones de host específicas para procesadores lógicos, núcleos físicos y niveles de parche Ruby. Tenga en cuenta también que este agente utiliza una versión anterior de Ruby.

La información de la página <DNT>**Environment snapshot**</DNT> depende del agente APM que utilice. Por ejemplo, las aplicaciones que usan el agente Ruby muestran configuraciones para gemas, mientras que las aplicaciones que usan el agente de Java muestran configuraciones para jars.
