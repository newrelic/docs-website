La configuración del lado del servidor transfiere algunas configuraciones principales del archivo de configuración de su agente de idioma al [recolector de New Relic](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#collector). Dependiendo del agente de idioma, las configuraciones disponibles pueden incluir:

* Rastreo de transacciones
* Grabación y registro SQL
* Planes de consulta SQL
* Colección de errores
* Perfilado de hilos
* rastreo distribuido
* Rastreo multiaplicación
* Creación de perfiles en tiempo real (solo agente de Java)

## Requisitos [#requirements]

La configuración del lado del servidor no está disponible para nuestro agente PHP.

## Centralización y seguridad [#features]

La configuración del lado del servidor proporciona una manera fácil de administrar las configuraciones disponibles desde el lado del servidor de New Relic. Los ajustes de configuración centralizados también pueden ayudarle a garantizar el cumplimiento de los estándares y la gestión de seguridad de su configuración.

Cualquier cambio en una aplicación de monitoreo se aplica automáticamente a todos los agentes APM que monitorean esa aplicación, incluso si se ejecutan en varios hosts. El agente aún sigue las [configuraciones locales](/docs/agents/manage-apm-agents/configuration/configure-agent) para cualquier configuración que no se pueda establecer.

## Prioridad de configuración del lado del servidor [#precedence]

Esta característica brinda la conveniencia de administrar los ajustes de configuración disponibles directamente desde el lado de New Relic, sin la necesidad de desplegarlos o reiniciarlos. Las opciones disponibles para cambiar la configuración y el orden de prioridad que toman cuando usa variables de entorno u otras opciones de configuración dependerán del agente de lenguaje utilizado.

Para obtener más información sobre la jerarquía de configuraciones, consulte la ilustración del agente específico:

* [Go jerarquía](/docs/agents/go-agent/instrumentation/go-agent-configuration#options)
* [Jerarquía de Java](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#config-options-precedence)
* [Jerarquía .NET](/docs/agents/net-agent/installation-configuration/net-agent-configuration#config-options-precedence)
* [Jerarquía de Node.js](/docs/agents/nodejs-agent/installation-configuration/configuring-nodejs#hierarchy)
* [Jerarquía PHP](/docs/agents/php-agent/configuration/php-agent-configuration#config-options-precedence) (no se admite la configuración del lado del servidor)
* [Jerarquía de Python](/docs/agents/python-agent/installation-configuration/python-agent-configuration#options)
* [Jerarquía Ruby](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#Options)

## Configurar desde la UI [#enable-ssc]

El agente PHP no admite la configuración del lado del servidor. Para habilitar los ajustes de configuración del lado del servidor para aplicaciones de monitor desde la UI:

1. Vaya a

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM**
   </DNT>

   .

2. Haga clic en su aplicación. Luego haga clic en

   <DNT>
     **Settings > Application > Server-side agent configuration**
   </DNT>

   .

Después de habilitar la configuración del lado del servidor, puede ver y cambiar las configuraciones disponibles a través de la UI.

## Configurar desde la API NerdGraph [#enable-nerdgraph]

Para saber cómo habilitar esto con NerdGraph, consulte el [tutorial de NerdGraph](/docs/apis/nerdgraph/examples/apm-config-nerdgraph).

## Ver o cambiar los ajustes de configuración del lado del servidor [#customize-ssc]

Si utiliza la configuración del lado del servidor, aún debe incluir su `license_key` y `app_name` en el archivo de configuración local. Estas configuraciones son necesarias para que el agente se comunique con el recolector de New Relic.

El agente PHP no admite la configuración del lado del servidor. Para ver o cambiar los ajustes de configuración del lado del servidor disponibles a través de la UI para aplicaciones que utilizan otro agente New Relic:

1. Vaya a

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM**
   </DNT>

   .

2. Haga clic en su aplicación. Luego haga clic en

   <DNT>
     **Settings > Application > Server-side agent configuration**
   </DNT>

   .

Una vez que configure cualquiera de estas opciones desde la UI, anularán cualquier opción conflictiva en el [archivo de configuración del agente](#precedence).
