Agente se puede configurar para recopilar variables de entorno relacionadas con Kubernetes y agregarlas al evento Transaction para proporcionar contexto entre su clúster de Kubernetes y sus servicios. Para obtener detalles sobre los beneficios, consulte [esta publicación de blog](https://blog.newrelic.com/engineering/monitoring-application-performance-in-kubernetes/).

Para obtener más información sobre la compatibilidad del agente y las instrucciones de instalación, consulte nuestra [documentación de Kubernetes](/docs/integrations/kubernetes-integration/metadata-injection/kubernetes-apm-metadata-injection).
