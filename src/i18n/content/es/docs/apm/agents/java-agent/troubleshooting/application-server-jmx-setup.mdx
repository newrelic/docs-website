## Problema

En APM, la pestaña <DNT>**Threads**</DNT> en la [página<DNT>**JVMs**</DNT> ](/docs/agents/java-agent/features/jvm-metrics-page)muestra varias métricas JMX admitidas por su servidor de aplicaciones, como subprocesos activos, inactivos y totales. Para ver métrica en la pestaña <DNT>**Threads**</DNT> , debe habilitar los informes JMX en el servidor de su aplicación.

Si los informes JMX no están habilitados, verá una pestaña <DNT>**Threads**</DNT> vacía con un enlace a la documentación relevante para que su servidor de aplicaciones habilite JMX.

## Solución

Para resolver el problema, lea las siguientes secciones según su servidor de aplicaciones:

<CollapserGroup>
  <Collapser
    id="JBoss"
    title="JBoss"
  >
    El soporte local para JMX está habilitado en los servidores de aplicaciones JBoss de forma predeterminada. Si ha cambiado la configuración predeterminada, consulte:

    * [JBoss EAP 6.0](https://access.redhat.com/site/documentation/en-US/JBoss_Enterprise_Application_Platform/6/html/Administration_and_Configuration_Guide/index.html)
    * [JBoss AS 7.1](https://docs.jboss.org/author/display/AS71/Admin%20Guide.html#8094211_AdminGuide-JMX)
    * [JBoss como 6](https://docs.jboss.org/jbossas/6/Admin_Console_Guide/en-US/html/)
  </Collapser>

  <Collapser
    id="Jetty"
    title="Jetty"
  >
    Para habilitar el soporte local para JMX en los servidores de aplicaciones Jetty, configure el MBeanContainer con el archivo <DNT>**etc/jetty-jmx.xml**</DNT> . Para más información, ver:

    * [Jetty 7 y 8](http://wiki.eclipse.org/Jetty/Tutorial/JMX)
    * [Jetty 9](http://www.eclipse.org/jetty/documentation/current/jmx-chapter.html)
  </Collapser>

  <Collapser
    id="Glassfish"
    title="Glassfish"
  >
    Si bien el soporte local para JMX está habilitado en los servidores de aplicaciones Glassfish de forma predeterminada, los MBeans que realmente proporcionan información métrica tienen el valor predeterminado `OFF`. Para que el servidor de aplicaciones recopile la métrica adecuada, se deben configurar los siguientes componentes en <DNT>**Monitoring Service**</DNT>:

    * [Glassfish 4.x](https://glassfish.java.net/docs/4.0/administration-guide.pdf): Los componentes del contenedor web, el servicio HTTP y el servicio de transacciones deben configurarse en el nivel de monitoreo ALTO.
    * [Glassfish 3.1](http://docs.oracle.com/cd/E18930_01/html/821-2416/ablur.html): Los componentes del contenedor web, el servicio HTTP y el servicio de transacciones deben configurarse en el nivel de monitoreo ALTO.
    * [Glassfish 3.0](http://docs.oracle.com/cd/E19798-01/821-1751/ablvr/index.html): Los componentes del contenedor web, el grupo de subprocesos y el servicio de transacciones deben configurarse en el nivel de monitoreo BAJO.
    * [Glassfish 2.1](http://docs.oracle.com/cd/E19879-01/820-4335/ablur/index.html): Los componentes del contenedor web, el grupo de subprocesos y el servicio de transacciones deben configurarse en el nivel de monitoreo BAJO.
  </Collapser>

  <Collapser
    id="Resin"
