## Problema

Está utilizando la [plataforma Solr](http://lucene.apache.org/solr/) en su aplicación web, pero los datos de Solr no aparecen en laUI <InlinePopover type="apm"/>de su aplicación desde <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select a Java app) > Monitor**</DNT>.

## Solución

Para asegurarse de que Solr esté [configurado](/docs/agents/java-agent/getting-started/compatibility-requirements-java-agent#other) correctamente y que los datos se informen a New Relic, haga lo siguiente según corresponda:

* <DNT>
    **JMX:**
  </DNT>

  Asegúrese de que JMX esté habilitado en su configuración de Solr en `solrconfig.xml` usando la etiqueta `<jmx/>` . El archivo `solrconfig.xml` es el archivo de configuración con la mayor cantidad de parámetros que afectan al propio Solr.

* <DNT>
    **Jetty:**
  </DNT>

  Si es usuario de Jetty, [habilite manualmente JMX para Jetty](https://wiki.eclipse.org/Jetty/Tutorial/JMX#Enabling_JMXConnectorServer_for_Remote_Access).

* <DNT>
    **Solr metrics:**
  </DNT>

  Revisa la consola JMX para Solr métrica. La caché y las actualizaciones de Solr provienen de MBeans.
