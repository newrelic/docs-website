## Problema

[Instalaste el <InlinePopover type="browser"/>agente](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) para tu aplicación Java, pero la aplicación no informa datos en la [UIdel navegador](/docs/browser/new-relic-browser/getting-started/browser-overview-page-website-performance-summary).

## Solución

Si su servidor de aplicaciones no utiliza el compilador Jasper JSP (por ejemplo, utiliza WebSphere o WebLogic), la instrumentación automática no está disponible. En su lugar, deberás [manualmente browserde instrumentos](/docs/java/page-load-timing-in-java#manual_instrumentation).

Si su servidor de aplicaciones utiliza el compilador JSP y aún no recibe datos del browser, es posible que tenga un conflicto de configuración. El agente de Java de New Relic te permite establecer el nombre de una aplicación en el archivo de configuración `newrelic.yml` o automáticamente con tu `web.xml` `display-name`.

Para verificar que su aplicación informa con el nombre correcto:

1. Generar temporalmente [el registro de nivel`finest` ](/docs/agents/java-agent/troubleshooting/generating-logs-troubleshooting).
2. Busque en el archivo de registro `"enabled for"`. Verifique que el nombre de su aplicación aparezca en la lista de aplicaciones que informan [datos de tiempobrowser ](/docs/agents/java-agent/instrumentation/page-load-timing-java).
3. Si el nombre de la aplicación que aparece no coincide con el nombre esperado, verifique que esté configurando el mismo nombre de la aplicación en `newrelic.yml` y en la configuración `web.xml` `display-name` .
