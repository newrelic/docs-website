## Problema

Tu aplicación utiliza el agente de Java de New Relic y te encuentras con un [problema de agrupación métrica](/docs/agents/manage-apm-agents/troubleshooting/metric-grouping-issues).

## Solución

Pruebe estos procedimientos de resolución de problemas antes de que los problemas de agrupación métrica se conviertan en un problema demasiado grande.

<CollapserGroup>
  <Collapser
    id="reconfigure-java"
    title="Asegúrese de que la denominación basada en componentes esté habilitada"
  >
    De forma predeterminada, el agente de Java [nombra la transacción](/docs/agents/java-agent/instrumentation/naming-web-transactions) según su framework y los diversos componentes de su aplicación. Sin embargo, si el agente de Java está configurado para deshabilitar la denominación basada en componentes, no podrá nombrar correctamente esas transacciones. Esto a menudo conducirá a un problema de agrupación métrica.

    Para solucionar problemas de configuración de su agente de Java: Si se ha deshabilitado [la denominación basada en componentes](/docs/agents/java-agent/instrumentation/naming-web-transactions#reverting) , vuelva a habilitarla.
  </Collapser>

  <Collapser
    id="upgrade"
    title="Actualizar el agente de Java"
  >
    Algunos problemas de agrupación métrica se pueden resolver simplemente [actualizando su agente New Relic](/docs/agents/java-agent/installation/upgrade-java-agent). A menos que tenga una buena razón para no hacerlo, New Relic recomienda actualizar siempre a la versión más reciente.

    Para solucionar problemas de su versión de agente de Java, consulte las [notas de la versión de agente de Java](/docs/release-notes/agent-release-notes/java-release-notes). Las notas de la versión resumen las correcciones y los cambios, incluidas situaciones en las que se han corregido problemas de agrupación métrica relacionados con agentes.
  </Collapser>

  <Collapser
    id="manually-name-transactions"
    title="Nombrar manualmente la transacción"
  >
    Si su versión de agente de Java está actualizada y configurada correctamente, pero aún tiene un problema de agrupación métrica, es posible que deba [nombrar manualmente su transacción](/docs/agents/java-agent/instrumentation/naming-web-transactions). Para ayudarle a comenzar, consulte [el programa de ejemplo de New Relic utilizando la API de Java](/docs/agents/java-agent/custom-instrumentation/java-agent-api-example-program).

    La [API del agente de Java](/docs/agents/java-agent/custom-instrumentation/java-agent-api) le permite [establecer un nombre de transacción](http://newrelic.github.io/java-agent-api/javadoc/com/newrelic/api/agent/NewRelic.html#setTransactionName%28java.lang.String,%20java.lang.String%29) dentro de su aplicación. Esto ayuda a eliminar nombres e identificaciones únicos en nombres métricos.
  </Collapser>

  <Collapser
    id="resolve-limit"
    title="Resolver límite métrico puesto por New Relic"
  >
    Después de detectar un problema de agrupación métrica, New Relic aplica automáticamente un límite métrico a su aplicación. Esto evita que entren nuevos nombres métricos en su aplicación.

    El límite métrico evita que el problema empeore. Sin embargo, antes de que New Relic pueda eliminar el límite, se debe resolver la causa del problema de la agrupación métrica.

    Una vez que se haya solucionado el problema subyacente, [el soporte de New Relic](https://support.newrelic.com) comprobará su aplicación para asegurarse de que:

    * Su agente de Java actualizado ya no envía las métricas anteriores que causaban problemas de agrupación.

    * Sus nuevos [nombres de transacciones](/docs/agents/java-agent/instrumentation/naming-web-transactions) excluyen correctamente las partes únicas del nombre.

      Después de confirmar que el problema de agrupación métrica está resuelto, New Relic Support actualizará manualmente su aplicación para que nuevamente se permitan nuevos nombres métricos.
  </Collapser>
</CollapserGroup>
