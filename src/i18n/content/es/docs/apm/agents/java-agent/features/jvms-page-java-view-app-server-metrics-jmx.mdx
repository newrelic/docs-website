Puede utilizar el agente de Java de New Relic para recopilar y ver métricas adicionales proporcionadas por los servidores de aplicaciones más comunes a través de JMX. Las métricas recopiladas varían según el servidor de la aplicación e incluyen mediciones de grupos de subprocesos, sesiones HTTP y transacciones.

## Ver JVM métrica [#dashboard]

Los servidores de aplicaciones recopilan e informan diferentes métricas, que aparecen en la página <DNT>**JVMs**</DNT> de APM. (Para aplicaciones que usan New Relic a través de Heroku, la página <DNT>**JVMs**</DNT> se llama <DNT>**[Instances](/docs/accounts-partnerships/partnerships/heroku-new-relic/heroku-instances-dashboard)**</DNT>).

Como mínimo, cada servidor de aplicaciones recopila y reporta datos métricos en la memoria. Para obtener instrucciones sobre cómo habilitar JMX métrica, consulte la documentación del servidor de su aplicación.

Para ver JVM métrica:

1. Vaya a

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select an app) > Monitoring > JVMs > (select a JVM)**
   </DNT>

   .

2. Desde la página

   <DNT>
     **Memory**
   </DNT>

   de la JVM seleccionada, utilice cualquiera de [las funciones de interfaz de usuario](/docs/accounts-partnerships/education/getting-started-new-relic/standard-dashboard-features) estándar de New Relic para profundizar en información detallada.

3. Para ver otras métricas, seleccione cualquier [pestaña disponible](#dashboard-tabs).

4. Opcional: para utilizar el [generador de perfiles de subprocesos](/docs/applications-menu/thread-profiler) para buscar cuello de botella en los datos sobre los subprocesos en ejecución, seleccione

   <DNT>
     **Profile this JVM**
   </DNT>

   .

## Habilitar para Spring Boot 2.2 y superior con Tomcat

Para permitir que New Relic acceda a Tomcat JMX métrica cuando se ejecuta con Spring Boot 2.2 y superior, la propiedad del sistema `server.tomcat.mbeanregistry.enabled` debe establecerse en `true`. Consulte las [notas de la versión Spring Boot 2.2](https://github.com/spring-projects/spring-boot/wiki/Spring-Boot-2.2-Release-Notes#tomcats-mbean-registry).

## Habilitar WebSphere PMI métrica [#default-pmi]

Para permitir que New Relic acceda a WebSphere PMI métrica, deberá configurar WebSphere para monitor los conjuntos de estadísticas necesarios.

<CollapserGroup>
  <Collapser
    id="enable-all-stat-sets"
    title="Habilitar todos los conjuntos de estadísticas (recomendado)"
  >
    El enfoque más sencillo es permitir la recopilación de todos los conjuntos de estadísticas:

    1. Si está utilizando Java 2 Security, asegúrese de haber [otorgado a New Relic los permisos adecuados](/docs/agents/java-agent/installation/install-java-agent-java-2-security).

    2. Desde la consola de administración de WebSphere, seleccione

       <DNT>
         **Monitoring and Tuning > Performance Monitoring Infrastructure (PMI) > (your server name)**
       </DNT>

       .

    3. Seleccione la casilla de verificación

       <DNT>
         **Enable Performance Monitoring Infrastructure (PMI)**
       </DNT>

       .

    4. En la sección

       <DNT>
         **Currently monitored statistic set**
       </DNT>

       , seleccione

       <DNT>
         **All**
       </DNT>

       y luego guarde los cambios.
  </Collapser>

  <Collapser
    id="enable-custom-stat-sets"
    title="Habilitar conjuntos de estadísticas personalizados (avanzado)"
  >
    También puedes recopilar solo el PMI métrico que New Relic monitorea por defecto. Para recopilar sólo un poco de PMI métrica:

    1. Si está utilizando Java 2 Security, asegúrese de haber [otorgado a New Relic los permisos adecuados](/docs/agents/java-agent/installation/install-java-agent-java-2-security).

    2. Desde la consola de administración de WebSphere, seleccione

       <DNT>
         **Monitoring and Tuning > Performance Monitoring Infrastructure (PMI) > (selected app server)**
       </DNT>

       .

    3. Seleccione la casilla de verificación

       <DNT>
         **Enable Performance Monitoring Infrastructure (PMI)**
       </DNT>

       .

    4. Seleccione

       <DNT>
         **Custom**
       </DNT>

       . Desde la lista de conjunto de estadísticas personalizadas, habilite las siguientes métricas:

       * En la categoría

         <DNT>
           **ThreadPool**
         </DNT>

         , seleccione las casillas de verificación

         <DNT>
           **ActiveCount**
         </DNT>

         y

         <DNT>
           **PoolSize**
         </DNT>

         .

       * En la categoría

         <DNT>
           **Servlet Session Manager**
         </DNT>

         , seleccione las casillas de verificación

         <DNT>
           **ActiveCount**
         </DNT>

         y

         <DNT>
           **LiveCount**
         </DNT>

         .

       * En la categoría

         <DNT>
           **Transaction Manager**
         </DNT>

         , seleccione las casillas de verificación

         <DNT>
           **CommittedCount**
         </DNT>

         ,

         <DNT>
           **RolledbackCount**
         </DNT>

         y

         <DNT>
           **GlobalTimeoutCount**
         </DNT>

         .

    5. En el panel

       <DNT>
         **Messages**
       </DNT>

       , seleccione

       <DNT>
         **Save directly to the master configuration**
       </DNT>

       .
  </Collapser>
</CollapserGroup>
