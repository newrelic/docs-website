## Problema

Después de [instalar el agente Ruby de New Relic](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation), generar algo de tráfico y esperar cinco minutos, no aparece ningún dato en la UI de New Relic.

## Solución

Para solucionar problemas de datos faltantes:

1. Verifique su archivo de configuración:

   * Asegúrese de que su archivo de configuración esté instalado en `config/newrelic.yml` dentro del directorio de su aplicación.

   * Asegúrese de que su archivo de configuración sea YML válido.

   * Para descargar una copia limpia de este archivo con su

     <InlinePopover type="licenseKey"/>

     completado previamente, vaya a

     <DNT>
       **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (user menu) > Account settings > Download a clean configuration file**
     </DNT>

     .

2. Reinicie su servidor web.

3. Utilice [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) para intentar identificar automáticamente el problema.

4. Examine el archivo de registro del agente en `log/newrelic_agent.log` para detectar errores o advertencias que aparezcan al reiniciar los procesos del servidor de aplicaciones. Si este archivo de registro no está presente o no se está escribiendo en él, consulte [Sin archivo de registro](/docs/agents/ruby-agent/troubleshooting/no-log-file-ruby).

5. Consulte el log del agente de New Relic para ver si hay una entrada que tenga `INFO : Reporting to:` seguido de una URL. Asegúrese de que la URL coincida con la cuenta y la aplicación que está viendo en la UI de New Relic.

6. Asegúrese de que el agente New Relic esté configurado para iniciarse en el entorno en el que se va a desplegar. Para obtener más información, consulte [Controlar cuándo se inicia el agente Ruby](/docs/agents/ruby-agent/troubleshooting/controlling-when-ruby-agent-starts).

7. Si no hay errores obvios en el nivel `info` , [genere un registro de depuración detallado](/docs/agents/ruby-agent/troubleshooting/generating-logs-troubleshooting-ruby).

8. Intente [actualizar a la última versión](/docs/agents/ruby-agent/installation-configuration/updating-deprecated-ruby-agents#update) de la gema <DNT>**newrelic_rpm**</DNT> .

9. Si está utilizando algo que no sea Rails, establezca `RACK_ENV` en un entorno que aparezca en el archivo `newrelic.yml` , como desarrollo o producción.
