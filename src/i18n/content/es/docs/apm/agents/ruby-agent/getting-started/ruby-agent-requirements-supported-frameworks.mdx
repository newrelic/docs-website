Antes de [instalar el agente Ruby de New Relic](/docs/agents/ruby-agent/installation-and-configuration/ruby-agent-installation), asegúrese de cumplir con estos requisitos de sistema operativo compatible, requisitos de seguridad y marco compatible.

Si aún no tiene una, comience [creando una cuenta New Relic](https://newrelic.com/signup). Es gratis, para siempre.

## Sistema operativo [#os]

El agente Ruby admite sistemas operativos similares a UNIX, como Linux, Solaris, FreeBSD y macOS.

## Requerimientos de seguridad [#security]

Como [medida de seguridad estándar para la recopilación de datos](/docs/accounts-partnerships/accounts/security/data-security), su servidor de aplicaciones debe admitir SHA-2 (256 bits). SHA-1 no es compatible. New Relic requiere TLS 1.2 o superior para todas las conexiones entrantes.

## Otro software APM [#other-monitoring-software]

Si su aplicación utiliza otro software de monitoreo del rendimiento de aplicaciones (APM) (APM) además de nuestro agente, no podemos garantizar que nuestro agente funcionará correctamente y no podemos ofrecer soporte técnico. Para obtener más información, consulte [Errores al utilizar otro software de monitoreo](/docs/apm/new-relic-apm/troubleshooting/errors-while-using-new-relic-apm-alongside-other-apm-software).

* RabbitMQ

* [Bunny](/docs/agents/ruby-agent/features/message-queues): 2.0 o superior (las versiones 2.0.x - 2.6.x son compatibles por última vez con la versión del agente 8.16.0)
