Antes de [instalar el agente New Relic Ruby ](/docs/agents/ruby-agent/installation-and-configuration/ruby-agent-installation)de, cerciorar de que su sistema cumpla con estos requisitos de sistema operativo compatible, protocolos de seguridad, Ruby versiones y marco/biblioteca compatibles.

Si aún no tiene una, comience [creando una cuenta New Relic](https://newrelic.com/signup). Es gratis, para siempre.

## Sistema operativo [#os]

El agente Ruby es compatible con sistemas operativos similares a UNIX, como Linux, Solaris, FreeBSD y macOS. Windows no recibe soporte oficial para entornos de producción, pero puede funcionar para desarrollo.

## Requerimientos de seguridad [#security]

* **Recopilación de datos:** como [medida de seguridad estándar para la recopilación de datos](/docs/accounts-partnerships/accounts/security/data-security), su servidor de aplicaciones debe admitir SHA-2 (256 bits) para la comunicación SSL con New Relic. SHA-1 no es compatible.
* **TLS:** New Relic requiere TLS 1.2 o superior para todas las conexiones entrantes del agente.

## Notas sobre las versiones compatibles [#notes\_on\_supported\_versions][#notes_on_supported_versions]

* El agente puede ser compatible con una gama más amplia de versiones (más antiguas o más nuevas) que las que se enumeran explícitamente. Sin embargo, las versiones enumeradas son objeto de las pruebas más rigurosas.
* Para obtener la información más actualizada sobre las nuevas versiones admitidas o las obsoletas, consulte siempre las [notas de la versión del agente Ruby](/docs/release-notes/agent-release-notes/ruby-release-notes).
* Si una biblioteca no está listada, aún puede estar instrumentada si se basa en una tecnología subyacente que *está* instrumentada (por ejemplo, un framework creado en Rack o un ORM que usa un adaptador de base de datos estándar). Para bibliotecas críticas no incluidas en la lista, puede explorar [la instrumentación personalizada](/docs/apm/agents/ruby-agent/api-guides/ruby-custom-instrumentation) o [enviar una solicitud de característica](https://github.com/newrelic/newrelic-ruby-agent/issues/new?template=feature_request.md).

Si tiene preguntas sobre versiones o bibliotecas específicas que no se tratan aquí, visite nuestro [foro comunitario Explorers Hub](https://forum.newrelic.com/).

## Otro software APM [#other\_monitoring\_software][#other_monitoring_software]

Si su aplicación utiliza otro software de monitoreo del rendimiento de aplicaciones (APM) (APM) además de nuestro agente, no podemos garantizar que nuestro agente funcionará correctamente y no podemos ofrecer soporte técnico. Para obtener más información, consulte [Errores al utilizar otro software de monitoreo](/docs/apm/new-relic-apm/troubleshooting/errors-while-using-new-relic-apm-alongside-other-apm-software).
