## Problema

Instaló el agente .NET de New Relic a través de [los servicios en la nube de Microsoft Azure](/docs/agents/net-agent/azure-installation/install-app-azure-cloud-services) y luego instaló el paquete NuGet. Después de generar algo de tráfico y esperar unos minutos para que se comiencen a recopilar datos para su aplicación, los datos aún no aparecen en la interfaz de usuario de APM.

## Solución

Para que el agente .NET se inicie y se conecte a su aplicación, se debe recopilar al menos una métrica. Si tiene una llamada externa o una llamada de base de datos instrumentada por New Relic, su aplicación reportará datos a su [página APM <DNT>**Summary**</DNT> ](/docs/apm/applications-menu/monitoring/apm-overview-page). Además, la información sobre llamadas externas aparecerá en [los mapas de servicios](/docs/data-analysis/user-interface-functions/view-your-data/service-maps-visualize-monitor-apps-entire-architecture) de tu aplicación.

Sin instrumentación personalizada, `Worker` roles solo informan llamadas de base de datos y llamadas externas. instrumentar una función `Worker` es similar a [instrumentar una aplicación personalizada](/docs/agents/net-agent/instrumentation/instrumenting-non-iis-apps)). Para ver otros detalles, debe recopilar métricas personalizadas utilizando [la API .NET](/docs/agents/net-agent/features/net-agent-api) de New Relic y verlas en un [panel personalizado](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards).
