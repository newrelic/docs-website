Con nuestro agente .NET para monitoreo del rendimiento de aplicaciones (APM), podrás:

* Utilice [APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm) para obtener una [descripción general de alto nivel de su aplicación](/docs/apm/applications-menu/monitoring/apm-overview-page), para ver detalles a nivel de código como [traza de la transacción](/docs/apm/transactions/transaction-traces/transaction-traces), [consulta de la base de datos](/docs/apm/transactions/transaction-traces/sql-statements) y [errores](/docs/apm/applications-menu/events/view-apm-error-analytics), y para [realizar un seguimiento de la actividad en grandes sistemas distribuidos](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing).
* Reciba notificaciones proactivas de [alertas](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) para asegurarse de que su aplicación esté funcionando sin problemas.
* Utilice el [generador de consultas](/docs/query-your-data/explore-query-data/query-builder/use-advanced-nrql-mode-specify-data) para consultar sus datos y crear [un panel](/docs/query-your-data/explore-query-data/dashboards/introduction-new-relic-one-dashboards) personalizado con esos datos.
* Instale [monitoreo de infraestructura](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server) para ver el rendimiento del entorno host de su aplicación.

## Instalar el agente [#installation]

Para utilizar el agente .NET:

1. Asegúrese de que su sistema cumpla con la compatibilidad del agente .NET: [.NET Framework](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-framework) \| [.NET Core](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core)
2. [Regístrese](https://newrelic.com/signup) para obtener una cuenta New Relic.
3. Instale el agente .NET a través del iniciador del producto a continuación o siguiendo nuestros procedimientos de instalación estándar en los documentos.

## Soporte tanto para .NET framework como para .NET Core [#core-features]

El agente .NET de New Relic es compatible con [.NET Framework](/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent) y .[.NET Core](/docs/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core) y funciona con todos los lenguajes compatibles con .NET, como VB.NET, C# y CLI.

El soporte del agente para .NET Core aprovecha la compatibilidad, la velocidad, la característica API ampliada y las capacidades multiplataforma de .NET Core de Microsoft. El agente <DNT>**does not**</DNT> admite versiones de Microsoft .NET Core anteriores a la 2.0.

Con el soporte de New Relic para .NET, puedes monitor tus aplicaciones en entornos dinámicos o distribuidos, tales como:

* Imágenes de máquina virtual (VM) de servidor administrado en la nube
* Servidores de máquina virtual (VM) en el host
* Servicios de aplicaciones de Microsoft Azure
* Sistemas Windows y Linux autohospedados
* Máquina virtual (VM) AWS EC2

## Ver el registro de sus datos de infraestructura y APM [#logs-context]

También puede reunir los datos de su registro y de la aplicación para que la resolución de problemas sea más fácil y rápida. Con [el contexto de inicio de sesión](/docs/logs/logs-context/net-configure-logs-context-all), puede ver el mensaje de registro relacionado con sus errores y la traza directamente en UI de su aplicación. También puede ver el inicio de sesión en el contexto de [los datos de su infraestructura](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), como el clúster de Kubernetes. No es necesario cambiar a otra página de UI.

## Comprueba el código fuente [#source-code]

El agente .NET es un software de código abierto. Eso significa que puedes [explorar su código fuente](https://github.com/newrelic/newrelic-dotnet-agent) y enviar mejoras, o crear tu propia bifurcación y compilarla. Para obtener más información, consulte el [archivo README](https://github.com/newrelic/newrelic-dotnet-agent/blob/main/README.md).
