  A partir de la versión 6.20 del agente .NET, el monitor de estado .NET de New Relic es <DNT>**deprecated**</DNT> y ya no está disponible. Para [las versiones 6.20 o superiores del agente .NET](/docs/release-notes/agent-release-notes/net-release-notes), utilice [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics).
</Callout>

El monitor de estado es una herramienta de diagnóstico y estado obsoleta para el agente .NET de New Relic. Está diseñado para:

* Brindarle visibilidad del estado actual del agente .NET que se ejecuta en su servidor.
* Ayudarle a solucionar problemas comunes de instalación.
* Proporcione un enlace rápido a cada una de sus aplicaciones instrumentadas en newrelic.com.
* Proporcione acceso rápido al archivo New Relic Logs para cada aplicación instrumentada.

Para obtener ayuda adicional, consulte la documentación de resolución de problemas del agente New Relic .NET.

## Ver el estado de las aplicaciones de monitor [#viewing]

<Callout variant="important">
  A partir de la versión 6.20 del agente .NET, el monitor de estado .NET de New Relic ya no está disponible. Para [las versiones 6.20 o superiores del agente .NET](/docs/release-notes/agent-release-notes/net-release-notes), utilice [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics).
</Callout>

El monitor de estado proporciona visibilidad sobre qué aplicaciones están siendo instrumentadas actualmente por el agente .NET y el estado de cada aplicación. No verá ningún dato sobre una aplicación a menos que haya habido tráfico en ella recientemente. Para conservar los recursos de su sistema, el monitor de estado no se actualiza automáticamente.

Para iniciar el monitor de estado:

1. Desde el menú <DNT>**Start**</DNT> de Windows o la pantalla <DNT>**Start**</DNT> , busque `Status monitor`.

   O

2. Haga clic derecho en el ícono New Relic en el área de estado de la barra de tareas de Windows (bandeja del sistema) y luego seleccione <DNT>**Monitored applications**</DNT>.

3. Para ver si una aplicación se está instrumentando, asegúrese de que todas sus casillas de verificación estén en verde.

4. Después de seleccionar una aplicación específica, seleccione cualquiera de las opciones disponibles para ver la aplicación en New Relic, ver el archivo de registro, [reparar la instalación](#repairing) si se encuentran errores o actualizar y luego ver el estado más reciente de los procesos y aplicaciones de su monitor.

## Reparación instalación [#repairing]

La instalación del agente .NET de New Relic es sólida y fácil de usar. Sin embargo, existen ciertos escenarios en los que la instalación anterior puede entrar en conflicto con la última versión. Para solucionar problemas comunes, repare su instalación:

1. En la lista de aplicaciones del monitor de estado, seleccione una aplicación y luego seleccione <DNT>**Repair installation**</DNT>.

   O

   En la bandeja del sistema de Windows, seleccione el icono New Relic y luego seleccione <DNT>**Tools > Repair installation**</DNT>.

2. Una vez finalizada la reparación de la instalación: desde la línea de comando, realice un `IISRESET`.
