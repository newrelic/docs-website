Para equilibrar la necesidad de registro y al mismo tiempo considerar el uso del disco, cada agente .NET limitará el uso del disco a 250 MB mediante el proceso log continuo. El agente primero iniciará log en el archivo `newrelic_agent_UNIQUENAME.log` y creará el archivo si no existe. Una vez que el archivo alcance los 50 MB de tamaño, el agente:

1. Cree un nuevo archivo de registro.
2. Traslade cada archivo de registro existente a un nuevo nombre numerado secuencialmente (hasta cuatro archivos archivados).
3. Elimina el cuarto archivo.

Para rodar el archivo de registro, el antiguo `newrelic_agent_UNIQUENAME.log` se convierte en el nuevo `newrelic_agent_UNIQUENAME.log(1)`. Luego, el antiguo `newrelic_agent_UNIQUENAME.log(1)` se convierte en el nuevo `newrelic_agent_UNIQUENAME.log(2)` y así sucesivamente. Se elimina el antiguo `newrelic_agent_UNIQUENAME.log(4)` .
