    El informe de análisis de capacidad de APM muestra cuántas instancias de su aplicación se están ejecutando y qué tan ocupadas están.     El informe de análisis de capacidad solo está disponible para el agente de lenguaje [Python](#agent-capacity-analysis) y Ruby.

       ## Aplicaciones Python de uno o varios subprocesos [#agent-capacity-analysis]

       Para el agente Python, los informes de análisis de capacidad solo funcionan para aplicaciones tradicionales de uno o varios subprocesos. Esto se debe a que la forma en que New Relic calcula la carga del sistema no se traduce en sistemas basados en rutinas.

       Cuando se utiliza una aplicación tradicional de subproceso único o multiproceso, la información métrica generada por el agente Python se captura y se informa solo si el componente de extensión C opcional se puede compilar e instalar. Cuando utiliza sistemas basados en rutinas conjuntas como gevent o eventlet junto con un servidor WSGI, el informe de análisis de capacidad no mostrará ningún dato excepto los reinicios del agente.

    El informe de análisis de escalabilidad de APM muestra el tiempo de respuesta, el tiempo de la base de datos y el tiempo de CPU en relación con la carga de la aplicación. Esto le ayuda a evaluar el rendimiento e identificar posibles requisitos de hardware a medida que crece su aplicación.

    El informe de análisis de escalabilidad incluye tres pestañas: tiempo de respuesta, base de datos y CPU APM. Cada pestaña contiene un cartel y un gráfico y representa un aspecto diferente del rendimiento.

    ## Seleccionar opciones para períodos de tiempo [#time]

    El informe de escalabilidad tiene un [selector de tiempo](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) que le permite ver datos en diferentes períodos de tiempo. El período de tiempo mínimo recomendado es de 30 minutos porque al analizar la escalabilidad de una aplicación, los rangos de tiempo más bajos no brindan una imagen completa. El selector de tiempo utiliza el valor predeterminado de los últimos 30 minutos.

    ## Ver datos de instancias individuales [#instances]

    Cada pestaña incluye un selector de instancias que le permite seleccionar una instancia específica y el gráfico y el cartel filtrarán los datos de esa instancia.

       ## Aplicación seleccionada [#featured_apps]

       New Relic selecciona las 20 mejores aplicaciones por recuento de llamadas que cumplen con estos criterios:

    * La aplicación debe tener al menos un rendimiento de 25 solicitudes por minuto (rpm).
    * La aplicación debe incluir transacción web.

      Las aplicaciones con monitoreo de navegador vinculado tendrán prioridad sobre aquellas que no lo tengan, incluso si el recuento de llamadas es menor.

      ## Aceptar o no recibir correos electrónicos de informes [#opting_out]

      Para administrar los correos electrónicos de su informe de rendimiento semanal, vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, haga clic en el [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) y seleccione <DNT>**User preferences**</DNT>. Tenga en cuenta que puede optar por participar o no por separado para cada cuenta de New Relic a la que tenga acceso.
