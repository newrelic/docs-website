Después de [habilitar el monitoreo serverlesspara AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-aws-lambda-monitoring), puede agregar una configuración adicional para ajustar sus datos.

## Habilitar seguimiento infinito [#infinite-tracing]

Rastreo distribuido se incluye de forma predeterminada si configura sus funciones utilizando capas Lambda.

Si su función Lambda se comunica con servicios que utilizan nuestro rastreo final distribuido llamado [Infinite Tracing](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing), debe habilitar esta función para su función Lambda. Esto garantiza que sus intervalos Lambda estén incluidos con el resto de los intervalos rastreados por Infinite Tracing.

Para configurar esto para su función Lambda, consulte los [pasos de Infinite Tracing](/docs/distributed-tracing/infinite-tracing/set-trace-observer) para crear un observador de traza y seleccionar qué función Lambda (fuentes de datos) debe informar intervalos.

## Configurar alerta [#alerts]

Nuestro <InlinePopover type="alerts"/>le permite recibir notificaciones sobre cualquier cosa que pueda monitor con New Relic, incluida APM métrica, transacción clave, consulta NRQL y más. Consulte [nuestra documentación de alerta](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/).

## Añadir evento personalizado [#custom-events]

Además de los datos que proporcionamos por defecto, también puedes configurar tu propio [evento](/docs/using-new-relic/welcome-new-relic/get-started/glossary#event) o [atributo](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute).

Para obtener detalles sobre estas configuraciones específicas del idioma, consulte [Configuración de atributos personalizados y eventos en AWS Lambda](https://discuss.newrelic.com/t/configuring-custom-attributes-and-events-in-aws-lambda/96043).
