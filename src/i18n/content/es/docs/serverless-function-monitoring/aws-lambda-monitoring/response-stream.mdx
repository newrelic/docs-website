Puede monitorear la función Lambda AWS que emplea [el streaming de respuesta](https://aws.amazon.com/blogs/compute/introducing-aws-lambda-response-streaming/) en la aplicación Node.js para obtener tiempo real información valiosa sobre el rendimiento y el comportamiento de cada invocación. Esta solución se integra con [el monitoreo de IA](/docs/ai-monitoring/intro-to-ai-monitoring/) para permitirle rastrear las respuestas del modelo y recopilar datos de observabilidad relacionados con su carga de trabajo de IA. Al emplear estas capacidades, puede optimizar el rendimiento de la aplicación sin servidor y mantener una supervisión integral de los procesos impulsados por IA.

Este documento lo ayudará a:

* Configurar la función de transmisión de respuesta de monitoreo Lambda
* Comprender la métrica y los datos disponibles para la función de transmisión de respuesta Lambda
* Aprenda a usar la New Relic UI para monitorear y observar el rendimiento de su función de transmisión de respuesta Lambda

## Configurar la transmisión de respuesta Lambda [#setup]

### Requisitos previos [#prerequisites]

* Compruebe que su función Lambda cumple con nuestros [requisitos y compatibilidad](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/compatibility-requirement-lambda-monitoring).
* Emplee la [última capa lambda](https://layers.newrelic-external.com) disponible para Node.js.

### Configurar la transmisión de respuesta Lambda [#configure-response-streaming]

Para habilitar el monitoreo de transmisión de respuestas para su Node.js función Lambda:

1. Instalar y configurar [el monitoreoLambda en New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/configure-serverless-aws-monitoring).

2. Agregue las [variables de entorno necesarias](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda) para la Node.js función de transmisión de respuesta Lambda

3. Almacene los cambios e implemente la función.

   Luego de la implementación, puedes ver la transmisión de datos de la respuesta Lambda en la plataforma New Relic . Para obtener más información, consulte [Acceder a la transmisión de datos de respuesta Lambda ](#access-ui).

## Acceda a la transmisión de datos de la respuesta Lambda [#access-ui]

Para ver la transmisión de datos de la respuesta Lambda en la plataforma New Relic :

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; left navigation pane &gt; All Capabilities &gt; Serverless Functions**</DNT>.

2. Seleccione la función Lambda requerida que emplea la transmisión de respuesta.

3. Ver resumen, invocaciones, métricas de CloudWatch y rastreo distribuido para su respuesta en streaming función Lambda.

4. En la sección **More views** , haga clic en **AI Responses**.

   Para obtener más información sobre las respuestas de IA, consulte [Ver respuestas de IA](/docs/ai-monitoring/explore-ai-data/view-ai-responses).

## Métrica clave para el streaming de respuesta [#key-metrics]

En la página **Cloudwatch metrics** , monitoree las siguientes métricas relacionadas con la transmisión de respuestas:

* **Bytes salientes transmitidos:** muestra la cantidad promedio de bytes transmitidos por la función Lambda.
* **Rendimiento de transmisión saliente:** muestra la velocidad promedio a la que la función Lambda transmite bytes.
