ATP recopila métricas de procesos a través del `hostmetricsreceiver` de OpenTelemetry y las enriquece con metadatos de filtrado inteligente. Puede consultar estos datos para:

* Monitoree los estados de los procesos e identifique `stuck` o procesos problemáticos
* Monitoree cómo los umbrales dinámicos de ATP se adaptan a sus patrones de carga de trabajo
* Mida la eficiencia del filtrado para entender la efectividad de la reducción de datos
* Detecte anomalías y comportamiento inusual de los procesos
* Configurar alertas para el monitoreo de procesos críticos

Todas las métricas de procesos están disponibles en el espacio de nombres `process.*` en New Relic, con los metadatos específicos de ATP almacenados en el campo `process.atp`.

## Monitoreo del estado del proceso [#process-state]

<CollapserGroup>
  <Collapser id="process-count-by-name" title="Distribución de procesos por estado">
    Esta consulta muestra un desglose de todos los procesos que se ejecutan en su host, categorizados por su estado de ejecución. Es esencial para identificar procesos `stuck` o el agotamiento de recursos.

    ```sql
    FROM Metric
    SELECT uniqueCount(process.pid) as 'Process Count'
    FACET process.state, CASES(
      WHERE process.state = 'R' AS 'Running',
      WHERE process.state = 'S' AS 'Sleeping (Interruptible)',
      WHERE process.state = 'D' AS 'Uninterruptible Sleep (Disk I/O)',
      WHERE process.state = 'Z' AS 'Zombie',
      WHERE process.state = 'T' AS 'Stopped',
      WHERE process.state = 'I' AS 'Idle Kernel Thread',
      WHERE process.state = 'X' AS 'Dead',
      WHERE process.state = 'W' AS 'Paging'
    )
    ```
  </Collapser>
</CollapserGroup>

## Monitoreo de inteligencia de ATP [#atp-intelligence]

<CollapserGroup>
  <Collapser id="dynamic-thresholds" title="Seguimiento de umbrales dinámicos">
    Esta consulta muestra los valores actuales de los umbrales dinámicos comparados con las métricas reales del proceso, lo que le ayuda a entender qué tan cerca están los procesos de activar acciones de ATP.

    ```sql
    FROM Metric
    SELECT
      latest(numeric(jparse(`process.atp`)[threshold_details][`process.cpu.utilization`][threshold])) * 100 AS 'Process CPU Utilization',
      latest(numeric(jparse(`process.atp`)[threshold_details][`process.memory.usage`][threshold])) AS 'Memory Usage'
    WHERE host.name = 'YOUR_HOST_NAME'
    TIMESERIES MAX
    ```

    **Cómo usar esta consulta:**

    * Reemplace `YOUR_HOST_NAME` con su nombre de host real
    * Puede modificar los nombres de las métricas en la función `jparse` para rastrear los umbrales de otras métricas
    * Busque patrones donde los valores reales se acerquen consistentemente a los valores de umbral
  </Collapser>

  <Collapser id="filtering-efficiency" title="Eficiencia de filtrado de ATP">
    Esta consulta mide la eficacia con la que ATP reduce el volumen de telemetría mientras retiene datos críticos.

    ```sql
    FROM Metric
    SELECT latest(numeric(jparse(`process.atp`)['filtering_summary']['efficiency_ratio'])) * 100 as 'Filtering Efficiency'
    WHERE metricName like 'process.%'
    AND `process.atp` IS NOT NULL
    ```
  </Collapser>
</CollapserGroup>

## Consultas de detección de anomalías [#anomaly-detection]

<CollapserGroup>
  <Collapser id="anomaly-overview" title="Conteo de anomalías a lo largo del tiempo">
    Esta consulta visualiza con qué frecuencia ATP detecta anomalías en diferentes ejecutables, ayudándole a identificar qué procesos son más propensos a un comportamiento inusual.

    ```sql
    FROM Metric
    SELECT latest(numeric(jparse(`process.atp`, 'filtering_summary.stage_hits.anomaly_detection'))) AS 'Anomaly Count'
    WHERE metricName like 'process.%'
    FACET process.executable.name
    TIMESERIES MAX
    ```
  </Collapser>

  <Collapser id="memory-usuage" title="Anomalías en el uso de memoria">
    Esta consulta resalta procesos específicos que han experimentado anomalías de uso de memoria detectadas por ATP.

    ```sql
    FROM Metric
    SELECT max(process.memory.usage) as 'Memory Usage'
    WHERE metricName = 'process.memory.usage'
    AND numeric(jparse(`process.atp`)['filtering_summary']['stage_hits']['anomaly_detection']) > 0
    AND process.atp IS NOT NULL
    FACET process.pid, process.executable.name, host.name
    TIMESERIES MAX
    ```
  </Collapser>

  <Collapser id="cpu-usage" title="Anomalías de uso de CPU">
    Esta consulta muestra los procesos que han experimentado anomalías en la utilización de la CPU.

    ```sql
    FROM Metric
    SELECT max(process.cpu.utilization) * 100 as 'CPU Utilization'
    WHERE metricName like 'process.cpu.utilization'
    AND numeric(jparse(`process.atp`)['filtering_summary']['stage_hits']['anomaly_detection']) > 0
    AND process.atp IS NOT NULL
    FACET process.pid, process.executable.name, host.name
    TIMESERIES MAX
    ```

    **Nota**: Puede modificar estas consultas para monitorear anomalías en cualquier otra métrica de proceso cambiando el filtro `metricName`.
  </Collapser>
</CollapserGroup>

## Monitoreo de desempeño [#performance-monitoring]

<CollapserGroup>
  <Collapser id="context-switches" title="Cambios de contexto por minuto">
    Esta consulta rastrea la tasa de cambio de contexto por proceso. Las altas tasas de cambio de contexto a menudo indican contención de CPU o multihilo ineficiente.

    ```sql
    SELECT rate(sum(process.context_switches), 1 minute) AS 'Context Switches/min'
    FROM Metric
    WHERE metricName = 'process.context_switches'
    FACET process.executable.name, process.pid
    TIMESERIES MAX
    ```
  </Collapser>
</CollapserGroup>

## Configuración de alertas

<CollapserGroup>
  <Collapser id="data-alert" title="Alerta de brecha de datos">
    Utilice esta consulta para crear una alerta que se active cuando la telemetría de procesos deje de fluir por completo, lo que indica una posible falla del agente o problemas de red. Para obtener más información sobre la configuración de alertas, consulte [Creación de condiciones de alerta](/docs/alerts/create-alert/create-alert-condition/alert-conditions/).

    ```sql
    FROM Metric
    SELECT count(*)
    WHERE metricName LIKE 'process.%'
    ```
  </Collapser>
</CollapserGroup>
