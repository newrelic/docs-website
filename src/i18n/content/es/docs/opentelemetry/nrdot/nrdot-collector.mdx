New Relic Distribution de OpenTelemetry Collector (NRDOT) es una distribución prediseñada de OpenTelemetry [Collector](https://opentelemetry.io/docs/collector/) compatible con New Relic. NRDOT está diseñado para funcionar sin problemas con los productos y servicios de New Relic, y es la forma recomendada de recopilar, procesar y exportar telemetry data abiertos para su uso con New Relic. New Relic proporciona una única distribución llamada `nrdot-collector` que está diseñada para funcionar con numerosas configuraciones, cada una de las cuales se integra con la ingesta de New Relic, e impulsar casos de uso específicos del dominio.

Además, los usuarios avanzados pueden crear distribuciones personalizadas con componentes adicionales utilizando [OCB](https://opentelemetry.io/docs/collector/custom-collector/) si los componentes [incluidos en NRDOT](https://github.com/newrelic/nrdot-collector-releases/blob/main/distributions/nrdot-collector/manifest.yaml) no cubren sus necesidades.

<Callout variant="tip">
  Hay escenarios en los que quizás desee implementar el recolector de una manera menos dogmática; le recomendamos leer nuestra documentación sobre [el procesamiento del recolector](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) para comprender cómo configurar el recolector para su caso de uso específico.
</Callout>

Antes de profundizar en uno de nuestros casos de uso, recomendamos familiarizarse con los conceptos básicos del OpenTelemetry Collector para comprender cómo está estructurada nuestra configuración y cómo personalizarla según sus necesidades.

<Callout variant="tip">
  El Collector NRDOT admite el cumplimiento de [FIPS 140-3](https://csrc.nist.gov/pubs/fips/140-3/final). Sin embargo, tenga en cuenta que esto solo se aplica a las imágenes Docker con el sufijo `-fips` en la etiqueta de versión (por ejemplo, `newrelic/nrdot-collector:1.12.0-fips`). Esta versión utiliza BoringCrypto ([certificado NIST](https://csrc.nist.gov/projects/cryptographic-module-validation-program/certificate/4735)) y garantiza que todas las comunicaciones TLS utilicen conjuntos de cifrado aprobados. Para obtener más información, consulte la [documentación de NRDOT FIPS](https://github.com/newrelic/nrdot-collector-releases/blob/main/fips/README.md).
</Callout>

## Despliegue [#deployments]

El recolector en sí es un binario único que puede implementar de distintas maneras. Las estrategias de despliegue típicas son:

* Como agente: desplegar en el mismo host que la aplicación.
* Como puerta de entrada: desplegar como punto central de recogida de telemetría.

Cada caso de uso de NRDOT viene con un despliegue específico en mente, y aunque la configuración se puede personalizar para adaptarse a sus necesidades, recomendamos seguir las recomendaciones de la documentación respectiva, a menos que tenga una razón de peso para desviarse de ellas.

Puede encontrar información adicional sobre los patrones de despliegue en la [documentación de OpenTelemetry Collector](https://opentelemetry.io/docs/collector/deployment/).
