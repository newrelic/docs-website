Después de configurar el monitoreo de HAProxy con OpenTelemetry, sus datos aparecen en New Relic en unos pocos minutos. Esta página le muestra cómo encontrar sus entidades de HAProxy, explorar métricas y escribir consultas NRQL.

## Encuentre sus datos de HAProxy [#find-data]

<CollapserGroup>
  <Collapser id="method1-entities" title="Usa el explorador de entidades">
    1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities**</DNT>.
    2. Busque sus entidades de HAProxy por nombre, o filtre por tipo de entidad `HAPROXYINSTANCE`.
    3. Seleccione una entidad para ver la página de resumen con las métricas doradas (sesiones por segundo, requests por segundo, errores de conexión).
  </Collapser>

  <Collapser id="method2-dashboard" title="Utilice integraciones y agentes">
    1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Integrations &amp; Agents**</DNT>.
    2. Haga clic en <DNT>**Dashboards**</DNT>.
    3. Busque <DNT>**HAProxy (OpenTelemetry Collector)**</DNT> y selecciónelo.
    4. Haga clic en <DNT>**View dashboard**</DNT> para ver visualizaciones prediseñadas para sus métricas de HAProxy.
  </Collapser>

  <Collapser id="method3-query" title="Use el generador de consultas">
    1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Query your data**</DNT>.
    2. Ejecute una consulta para encontrar sus datos de HAProxy:
       ```sql
       FROM Metric SELECT uniques(metricName)
       WHERE metricName LIKE 'haproxy.%'
       SINCE 1 hour ago
       ```
  </Collapser>
</CollapserGroup>

## Consulta tus datos con NRQL [#query-data]

Use estas consultas NRQL de ejemplo para analizar sus datos de HAProxy:

```sql
-- Sessions per second across frontends
FROM Metric SELECT average(haproxy.sessions.rate)
WHERE haproxy.service_name = 'FRONTEND'
FACET haproxy.proxy_name
TIMESERIES

-- Error rates by backend
FROM Metric SELECT rate(sum(haproxy.responses.errors), 1 minute) AS 'Errors/min'
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name
TIMESERIES

-- Bytes throughput (in + out)
FROM Metric SELECT rate(sum(haproxy.bytes.input), 1 minute) AS 'Bytes In/min',
  rate(sum(haproxy.bytes.output), 1 minute) AS 'Bytes Out/min'
FACET haproxy.proxy_name
TIMESERIES

-- Backend server health
FROM Metric SELECT latest(haproxy.active) AS 'Active Servers',
  latest(haproxy.backup) AS 'Backup Servers'
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name

-- Request queue depth (indicates backend saturation)
FROM Metric SELECT max(haproxy.requests.queued)
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name
TIMESERIES
```

## Solucionar problemas de visibilidad de datos [#troubleshoot]

<CollapserGroup>
  <Collapser id="troubleshoot-no-data" title="No aparecen datos en New Relic">
    * Espere unos minutos para la ingesta inicial de datos.
    * Verifique el rango de tiempo: asegúrese de estar viendo datos recientes (últimos 30 minutos).
    * Consulte cualquier métrica de HAProxy:
      ```sql
      FROM Metric SELECT count(*) WHERE metricName LIKE 'haproxy.%' SINCE 30 minutes ago
      ```
    * Verifique que el recolector se esté ejecutando:
      ```bash
      # Self-hosted
      sudo systemctl status otelcol-contrib
      # or
      sudo systemctl status nrdot-collector

      # Kubernetes
      kubectl get pods -n newrelic
      kubectl logs -n newrelic -l app.kubernetes.io/name=nr-k8s-otel-collector
      ```
    * Verifique que el extremo de estadísticas de HAProxy sea accesible desde el recolector.
  </Collapser>

  <Collapser id="troubleshoot-intermittent-data" title="Los datos aparecen de forma intermitente">
    * Verifique los logs del recolector en busca de errores:
      ```bash
      # Self-hosted
      sudo journalctl -u otelcol-contrib -n 50 --no-pager

      # Kubernetes
      kubectl logs -n newrelic deploy/nr-k8s-otel-collector-deployment --tail=50
      ```
    * Verifique que el extremo de estadísticas de HAProxy sea accesible de manera constante.
    * Compruebe si hay problemas de conectividad de red entre el recolector y el extremo OTLP de New Relic.
    * Asegúrese de que HAProxy no esté limitando la tasa de requests del extremo de estadísticas.
  </Collapser>
</CollapserGroup>
