          | `attributes/entity_tags`     | Marca `instrumentation.provider: opentelemetry` en cada mĂŠtrica para que pueda limitar las consultas a la ruta de OpenTelemetry.                                                      |

          #### QuĂŠ hace esta configuraciĂłn [#config-explanation-prometheus]

          Cada componente en la canalizaciĂłn tiene un trabajo especĂ­fico:

          | Componente                   | DescripciĂłn                                                                                                                                                                                          |
          | ---------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
          | `health_check`               | Expone un extremo de estado en `0.0.0.0:13133` para que pueda confirmar que el recolector se estĂĄ ejecutando.                                                                                        |
          | `prometheus` receptor        | Extrae del extremo `redis_exporter` (por defecto `localhost:9121`) cada 10 segundos y descarta las mĂŠtricas `go_*`, `process_*`, `promhttp_*` y `redis_exporter_*` propias del exportador.           |
          | `memory_limiter`             | Limita el uso de memoria del recolector (lĂ­mite flexible de 512 MiB, pico de 128 MiB) para proteger el host.                                                                                         |
          | `resource_detection`         | Detecta el host y agrega `host.name` y `host.id`, vinculando las mĂŠtricas de Redis a la entidad de host subyacente.                                                                                  |
          | `resource/redis_identity`    | Establece `redis.instance.id`, que identifica la entidad de Redis en New Relic. Es necesario aquĂ­ porque el receptor de Prometheus no proporciona `server.address` ni `server.port`.                 |
          | `attributes/entity_tags`     | Marca `instrumentation.provider: opentelemetry` en cada mĂŠtrica para que pueda limitar las consultas a la ruta de OpenTelemetry.                                                                     |
          | `metricstransform`           | Renombra las mĂŠtricas de Prometheus del exportador (por ejemplo, `redis_uptime_in_seconds`) a los nombres de Redis de New Relic (`redis.uptime`) y agrega la etiqueta `state` a las mĂŠtricas de CPU. |
          | `cumulativetodelta`          | Convierte los contadores acumulativos â comandos, aciertos de keyspace, desalojos, etc. â en valores delta para que New Relic grafique las tasas correctamente.                                      |
          | `filter/cardinality`         | Descarta los puntos de datos de alta cardinalidad (estados de CPU distintos de `user` y `sys`, y mĂŠtricas por comando para comandos poco comunes) para controlar el costo de ingesta.                |
          | `transform/metadata_nullify` | Borra las descripciones y unidades de las mĂŠtricas para reducir el tamaĂąo de la carga.                                                                                                               |
          | `batch`                      | Agrupa los puntos de datos antes de la exportaciĂłn (2048 por lote, hasta 4096) y los vacĂ­a al menos cada 10 segundos para reducir la sobrecarga de la red.                                           |
          | `otlp_http`                  | Exporta las mĂŠtricas procesadas a New Relic a travĂŠs de OTLP con compresiĂłn gzip, autenticadas con su clave de licencia.                                                                             |

          <Callout variant="important">
            El procesador `resource/redis_identity` con `redis.instance.id` es **obligatorio** para el enfoque del receptor de Prometheus. A diferencia del receptor de Redis nativo, el receptor de Prometheus no proporciona `server.address` ni `server.port` â por lo que `redis.instance.id` es la Ăşnica forma de identificar su entidad de Redis en New Relic. EstablĂŠzcalo en un nombre Ăşnico y descriptivo para cada instancia (por ejemplo, `prod-redis-cache:6379`).
          </Callout>
        </Step>
