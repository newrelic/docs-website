La mayoría de las aplicaciones dependen de Redis para las cosas que deben ser rápidas, como el almacenamiento en caché, el almacenamiento de sesiones, las tablas de clasificación en tiempo real y las colas de mensajes. Pero cuando Redis se ralentiza, por lo general se nota primero en otro lugar, como una página lenta o una solicitud que agota el tiempo de espera. Sin una buena visibilidad, es difícil saber si el verdadero problema es que la memoria se está agotando, que las claves se están expulsando de manera demasiado agresiva, que la tasa de aciertos de caché está disminuyendo o que una réplica se ha quedado atrás.

New Relic proporciona una ruta de monitoreo basada en OpenTelemetry para Redis. El [receptor de Redis](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/redisreceiver) del Collector de OpenTelemetry recopila las métricas que Redis expone a través de su comando INFO y las envía a New Relic, sin requerir un agente independiente. New Relic sintetiza cada instancia de Redis como su propia entidad, agrupa los despliegues en clúster bajo una entidad de Redis Cluster y proporciona dashboards y alertas prediseñados.

<Callout variant="tip">
  Si buscas la integración en el host tradicional de Redis de New Relic (nri-redis), consulta la [integración de monitoreo de Redis](/docs/infrastructure/host-integrations/host-integrations-list/redis/redis-integration/).
</Callout>

## Empezar [#get-started]

Para monitorear Redis con OpenTelemetry, complete tres pasos: revise los requisitos previos, configure el recolector y vea sus datos.

### Requisitos previos [#prereq]

Asegúrese de que su entorno cumpla con todos los requisitos para la integración definida para su despliegue:

* Para [autoalojado](/docs/opentelemetry/integrations/redis/host/#prereq)
* Para [Docker](/docs/opentelemetry/integrations/redis/docker/#prereq)
* Para [Kubernetes](/docs/opentelemetry/integrations/redis/kubernetes/#prereq)

### Configuración del recolector [#collector-setup]

Elija su ruta de instalación según su infraestructura:

* Para [autoalojado](/docs/opentelemetry/integrations/redis/host)
* Para [Docker](/docs/opentelemetry/integrations/redis/docker)
* Para [Kubernetes](/docs/opentelemetry/integrations/redis/kubernetes)

### Ver tus datos [#view-data]

Una vez que haya completado la configuración del recolector, puede ver sus métricas de Redis en New Relic, consultarlas con NRQL, crear dashboards y configurar alertas. Para obtener más información, consulte la [documentación para buscar y consultar sus datos](/docs/opentelemetry/integrations/redis/view-data).

La integración de Redis de New Relic rastrea una amplia variedad de métricas. Para obtener más información sobre los tipos de datos y atributos compatibles, consulte la guía de [referencia de métricas de Redis](/docs/opentelemetry/integrations/redis/metrics).

## Próximos pasos [#next-steps]

Después de instalar la integración:

* **[Vea sus datos](/docs/opentelemetry/integrations/redis/view-data)**: explore los dashboard, ejecute consultas NRQL y configure alertas para sus métricas de Redis.
* **[Revise la referencia de métricas](/docs/opentelemetry/integrations/redis/metrics)**: comprenda todas las métricas disponibles y sus atributos.
* **[Solución de problemas](/docs/opentelemetry/integrations/redis/troubleshooting)**: resuelva problemas comunes con la recopilación de datos y la síntesis de entidades.
