Después de configurar el monitoreo de OpenTelemetry para tu entorno de Amazon ECS, puedes encontrar y consultar tus datos en la infraestructura y las interfaces de consulta de New Relic. Esta guía le ayuda a localizar sus métricas de ECS y crear consultas significativas para el monitoreo y la resolución de problemas.

## Dónde encontrar tus datos de ECS [#find-data]

Sus datos de monitoreo de ECS aparecen en múltiples ubicaciones en New Relic, dependiendo del tipo de métrica y su tipo de lanzamiento de ECS.

### Monitoreo de infraestructura [#infrastructure-view]

Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Infrastructure**</DNT> para ver su infraestructura de ECS:

#### Para ECS en EC2

1. Vista de hosts: filtre por host para ver sus instancias EC2 que ejecutan contenedores de ECS
2. Vista de contenedores: vea el rendimiento del contenedor individual dentro de sus tareas de ECS

#### Para ECS Fargate

1. Vista de contenedores: ubicación principal para las métricas de contenedor de Fargate

<Callout variant="tip">
  Utilice las opciones de filtro para acotar a clústeres, servicios o nombres de contenedores específicos para encontrar rápidamente los datos que necesita.
</Callout>

### Métrica y evento [#metrics-events]

Vaya a **<DNT>[one.newrelic.com](https://one.newrelic.com)</DNT> &gt; Query your data** para:

* Crear gráficos y dashboards personalizados
* Configura alertas basadas en umbrales de métrica específicos

## Ejemplos de consulta [#query-examples]

### Monitorear la utilización de la CPU del contenedor

```sql
SELECT average(container.cpu.utilization)
FROM Metric
WHERE ClusterName = 'your-cluster-name'
AND ServiceName = 'your-service-name'
TIMESERIES AUTO
FACET container.name
```

### Rastree las tendencias de uso de memoria

```sql
SELECT max(container.memory.usage.total) / 1024 / 1024 as 'Memory (MB)'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET ServiceName
```

### Monitorear métricas a nivel de host (solo EC2)

```sql
SELECT average(1 - system.cpu.utilization) * 100 AS 'CPU used %',
       average(system.memory.utilization) * 100 AS 'Memory used %'
FROM Metric
WHERE aws.ecs.launchtype = 'ec2'
AND aws.ecs.cluster.arn = 'your-cluster-arn'
TIMESERIES AUTO
```

## Resolución de problemas de vacíos de datos [#troubleshooting]

Si no ve los datos esperados:

1. **Verifique el estado del recolector**: revise los logs del contenedor para el recolector OpenTelemetry
2. **Confirme la conectividad de red**: asegúrese de tener acceso HTTPS saliente a New Relic
3. **Comprobar clave de licencia**: verifique que la clave de licencia esté configurada correctamente en SSM Parameter Store
4. **Revisar la definición de la tarea**: confirme que el contenedor del recolector se esté ejecutando con la asignación de recursos adecuada
