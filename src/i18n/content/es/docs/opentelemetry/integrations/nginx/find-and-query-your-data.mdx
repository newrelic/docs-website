## Vea sus datos de NGINX en New Relic [#find-data]

Una vez que su configuración esté completa y los datos fluyan, puede ver sus métricas y logs de NGINX en New Relic y crear alertas personalizadas.

<CollapserGroup>
  <Collapser id="method1-integrations" title="A través de integraciones y agentes">
    1. Ir a <DNT>[one.newrelic.com](https://one.newrelic.com) &gt; **Integrations &amp; Agents**</DNT>
    2. Hacer clic <DNT>**Dashboards**</DNT>
    3. Busque y haga clic en el dashboard <DNT>**NGINX OTel overview**</DNT>
    4. Se abre una página <DNT>**Dashboard setup**</DNT>. Si ya configuró la fuente de datos, haga clic en <DNT>**Skip this step**</DNT>. De lo contrario, haga clic en <DNT>**See installation docs**</DNT> y siga las instrucciones para configurar su fuente de datos. Después de completar la configuración, regrese a la página de configuración del dashboard y haga clic en <DNT>**Skip this step**</DNT>.
    5. Haga clic en <DNT>**View dashboard**</DNT>.
  </Collapser>

  <Collapser id="method2-infrastructure" title="A través del monitoreo de infraestructura">
    1. Ir a <DNT>[one.newrelic.com](https://one.newrelic.com) &gt; **On-host integrations**</DNT>
    2. Busque su servidor NGINX por nombre de implementación (por ejemplo, `production-web-01`)
    3. Haga clic en su servidor para ver métricas detalladas
  </Collapser>
</CollapserGroup>

## Consulta tus datos con NRQL

Puede crear consultas personalizadas para analizar sus datos de NGINX:

Ejemplo de consulta:

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## Solucionar problemas de visibilidad de datos

<CollapserGroup>
  <Collapser id="troubleshoot-no-data" title="Si no ve datos">
    * Espere un momento: la ingesta inicial de datos puede tardar unos minutos
    * Verifique el rango de tiempo: asegúrese de estar viendo datos recientes
    * Verifique el nombre de la implementación: use el nombre exacto de la configuración de su Collector
    * Compruebe los datos con NRQL:
      ```sql
      FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
      ```
  </Collapser>

  <Collapser id="troubleshoot-intermittent-data" title="Si los datos aparecen de forma intermitente">
    * Verifique los logs del recolector en busca de errores:
      ```bash
      sudo journalctl -u otelcol-contrib -n 50
      ```
    * Verifique que NGINX esté generando tráfico - Pruebe con `curl http://localhost`
    * Verifique la autenticación: asegúrese de que la clave de licencia y el punto final sean correctos
  </Collapser>
</CollapserGroup>
