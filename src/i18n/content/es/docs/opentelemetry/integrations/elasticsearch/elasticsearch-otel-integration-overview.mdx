La integración de New Relic Elasticsearch OpenTelemetry (OTel) proporciona observabilidad integral para sus clústeres Elasticsearch utilizando protocolos OpenTelemetry estándar de la industria. Al aprovechar el OpenTelemetry Collector, obtienes una canalización unificada que recopila de manera eficiente tanto métricas como logs de tu infraestructura de Elasticsearch, lo que garantiza una monitorización rápida y confiable que se integra directamente con New Relic.

<Callout variant="important">
  **Elija su enfoque de monitoreo:** Esta integración basada en OpenTelemetry ofrece un monitoreo moderno y neutral para el proveedor. Si prefiere nuestro enfoque tradicional o necesita compatibilidad específica, consulte nuestra [integración estándar de Elasticsearch](/docs/infrastructure/host-integrations/host-integrations-list/elasticsearch/elasticsearch-integration).
</Callout>

**¿Por qué elegir este enfoque de OpenTelemetry?**

* **Recopilación unificada de datos**: Un único recopilador gestiona métricas, logs y rastreos de Elasticsearch y sistemas host.
* **Neutralidad de proveedor**: Utiliza estándares de código abierto que funcionan con cualquier plataforma de observabilidad, sin bloqueo de proveedor.
* **Alto rendimiento**: El procesamiento por lotes, la compresión y la reducción de la cardinalidad integrados minimizan el impacto en los recursos.
* **Arquitectura preparada para el futuro**: Construida sobre estándares abiertos en evolución que se adaptan a medida que crece tu infraestructura.
* **Integración nativa de New Relic**: procesamiento de datos optimizado y experiencia de dashboard sin problemas.

**Caso de uso:** Una empresa de comercio electrónico utiliza Elasticsearch para su función de búsqueda de productos. Al usar esta integración, su equipo de operaciones puede ver si su servicio de búsqueda está en buen estado, qué tan rápido responden las búsquedas y recibir notificaciones antes de que se agote la memoria, todo desde un solo dashboard. Cuando los clientes comienzan a quejarse de resultados de búsqueda lentos, el equipo puede ver rápidamente si es un problema de Elasticsearch, un problema del servidor o algo más al ver todos sus datos de monitoreo juntos.

<img title="Elasticsearch OpenTelemetry monitoring dashboard" alt="Screenshot showing Elasticsearch cluster health, performance metrics, and resource usage in New Relic dashboards" src="/images/elasticsearch-otel-solution.webp" />

**Lo que obtendrás**

Supervise su infraestructura de Elasticsearch con:

* **Monitoreo del estado del clúster**: rastree el estado del nodo, la distribución de fragmentos y el estado del clúster en tiempo real.
* **Información sobre el rendimiento**: supervise la latencia de búsqueda, las tasas de indexación y las métricas de rendimiento de la JVM.
* **Utilización de recursos**: Vea el uso de CPU, memoria, disco y red en todos los nodos.
* **Alertas proactivas**: Recibe notificaciones antes de que los problemas afecten a tus usuarios con umbrales inteligentes.
* **Correlación de pila completa**: Conecte las métricas de Elasticsearch con los datos de la aplicación y la infraestructura.

## Metrics and data reference [#metrics-reference]

The Elasticsearch OpenTelemetry integration collects over 50 metrics covering cluster health, performance, JVM statistics, and infrastructure monitoring. These metrics help you monitor critical aspects like shard distribution, search latency, memory usage, and resource utilization.

For detailed information about all available metrics, their descriptions, and how your data is organized in New Relic, see the [Elasticsearch metrics reference](/docs/opentelemetry/integrations/elasticsearch/metrics-reference).
