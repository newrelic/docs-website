El monitoreo de contenedores Docker proporciona visibilidad en tiempo real de sus aplicaciones en contenedores para garantizar un rendimiento confiable y prevenir cuellos de botella de recursos en sistemas distribuidos. Al usar OpenTelemetry Collector, obtiene un monitoreo integral mediante un enfoque flexible e independiente del proveedor que funciona en diversas plataformas de orquestación de contenedores.

## ¿Por qué monitorear contenedores Docker? [#why-monitor]

El monitoreo de contenedores es esencial para mantener el rendimiento y la confiabilidad de las aplicaciones en entornos contenerizados. OpenTelemetry proporciona un enfoque estandarizado para recopilar y analizar estos datos críticos.

### Beneficios clave

* **Optimización de recursos**: Rastree el uso de CPU, memoria, red y E/S para identificar cuellos de botella de recursos y optimizar la asignación de recursos de contenedores
* **Monitoreo proactivo**: Obtén alertas antes de que los contenedores agoten los recursos, detecta fallas en los contenedores anticipadamente y monitorea la salud de los contenedores
* **Información operativa**: Comprenda los patrones del ciclo de vida de los contenedores, realice un seguimiento de las tendencias de rendimiento, monitoree los patrones de reinicio de los contenedores y las tasas de fallas
* **Gestión de costos**: Monitoree la utilización de recursos para dimensionar correctamente los contenedores y optimizar los costos de infraestructura
* **A prueba de futuro**: El estándar de código abierto OpenTelemetry ofrece una integración independiente del proveedor que se adapta a medida que su infraestructura evoluciona

### Caso de uso común

Ya sea que ejecute microservicios, aplicaciones web o cargas de trabajo de procesamiento por lotes, el monitoreo de Docker lo ayuda a mantener servicios en contenedores confiables. Reciba notificaciones cuando los contenedores consuman memoria excesiva, detecte cuándo sus contenedores de API están saturados de solicitudes o identifique qué contenedores experimentan cuellos de botella en la red. Este monitoreo es esencial para aplicaciones en contenedores que manejan tráfico de usuarios, procesamiento de trabajos en segundo plano, pipelines de datos y cualquier carga de trabajo donde el rendimiento del contenedor impacta directamente en la confiabilidad de la aplicación y la experiencia del usuario.

## Cómo funciona [#how-it-works]

El OpenTelemetry Collector se conecta a su daemon de Docker mediante la API de Docker Stats para recopilar métricas integrales de los contenedores:

**Recopilación de datos:** El [receptor Docker Stats](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/dockerstatsreceiver) se conecta al socket de Docker (generalmente `/var/run/docker.sock`) para recopilar métricas de contenedores en tiempo real, incluidas las estadísticas de CPU, memoria, red y E/S de bloques.

**Flujo de datos:**

1. El colector consulta la API de estadísticas del daemon de Docker a intervalos regulares (predeterminado: 15 segundos)
2. Las métricas de los contenedores se recopilan, procesan y enriquecen con metadatos del contenedor (nombre, imagen, ID)
3. Los datos se agrupan en lotes para una transmisión eficiente a fin de reducir la sobrecarga de la red
4. Las métricas se exportan a New Relic a través del exportador OTLP
5. New Relic crea automáticamente entidades de contenedor y rellena los dashboards de monitoreo

**Lo que obtiene:** Las métricas clave incluyen uso y utilización de CPU, uso y porcentaje de memoria, E/S de red (bytes transmitidos/recibidos, paquetes descartados/errores), operaciones de E/S de bloques y conteo de procesos. Los metadatos del contenedor, como el nombre, la imagen y el ID, se adjuntan automáticamente a todas las métricas.

Para ver los nombres completos de las métricas, las descripciones y las recomendaciones de alertas, consulte la [referencia de métricas de Docker](/docs/opentelemetry/integrations/docker-monitoring/metrics-reference).
