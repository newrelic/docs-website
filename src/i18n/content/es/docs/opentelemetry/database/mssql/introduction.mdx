Monitoree el rendimiento de su base de datos Microsoft SQL Server utilizando New Relic Distribution of OpenTelemetry (NRDOT) con información valiosa y completa sobre las métricas de la base de datos, el rendimiento de las consultas y el estado del sistema.

Este enfoque basado en NRDOT complementa nuestra actual On-Host Integration (OHI) al aprovechar los estándares de OpenTelemetry para el monitoreo de bases de datos, lo que facilita la integración con su stack de observabilidad existente.

## Cómo puede configurar NRDOT para SQL Server [#configure]

<Steps>
  <Step>
    ### Requisitos previos [#prerequisites]

    Asegúrese de que su entorno cumpla con nuestra [compatibilidad y requisitos previos](/docs/opentelemetry/database/mssql/compatibility).
  </Step>

  <Step>
    ### Instalación del Collector NRDOT [#installation]

    Elija su ruta de instalación según su infraestructura:

    * **Instalación guiada**: configuración guiada por la plataforma a través de la UI de New Relic (recomendado para inicios rápidos)

    * **Entornos autohospedados**: instalación manual en su propia infraestructura

      * **[Windows autoalojado](/docs/opentelemetry/database/mssql/windows-hosted)**
      * **[Linux autoalojado](/docs/opentelemetry/database/mssql/linux-hosted)**

    * **Entornos de AWS RDS**: monitoreo de SQL Server administrado en la nube

      * **[Monitoreo de Windows RDS](/docs/opentelemetry/database/mssql/windows-rds)**
      * **[Monitoreo de Linux RDS](/docs/opentelemetry/database/mssql/linux-rds)**
  </Step>

  <Step>
    ### Encuentra y utiliza tus datos [#find]

    Una vez que sus datos se estén recopilando, puede acceder al monitoreo integral de bases de datos SQL Server a través de la UI de New Relic.

    Para encontrar su entidad de base de datos SQL Server en New Relic:

    1. Vaya a <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>.
    2. En el menú desplegable **<DNT>Entity type</DNT>**, seleccione <DNT>**MSSQL instance**</DNT> y luego haga clic en <DNT>**Apply**</DNT>.
    3. Seleccione su base de datos SQL Server de la lista de entidades.
  </Step>
</Steps>
