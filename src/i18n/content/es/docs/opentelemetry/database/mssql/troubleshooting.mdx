Solucione problemas comunes que puede encontrar durante el monitoreo de su Microsoft SQL Server con el recolector NRDOT.

## Problemas comunes

<CollapserGroup>
  <Collapser id="network-connectivity" title="Problemas de conectividad de red">
    Si encuentra algún problema de conectividad de red con el NRDOT Collector, ejecute los siguientes comandos para probar la conectividad con su instancia de SQL Server en el puerto 1433:

    * Para Windows, ejecute:

      ```shell
      Test-NetConnection -ComputerName <YOUR_IP_ADDRESS> -Port <YOUR_PORT_NUMBER>
      ```

    * Para Linux, ejecute:

      ```bash
      nc -zv <YOUR_IP_ADDRESS> <YOUR_PORT_NUMBER>
      ```
  </Collapser>

  <Collapser id="ssl" title="Problemas de conexión SSL">
    Para conectarse a Microsoft SQL Server mediante SSL sin validación del certificado, asegúrese de que el parámetro `enable_ssl` esté establecido en `true` y el parámetro `trust_server_certificate` esté establecido en `true` en la configuración de su NRDOT Collector.

    ```yaml
    enable_ssl: true                                                                                                                            
    trust_server_certificate: true
    ```
  </Collapser>
</CollapserGroup>
