<Callout title="Avance">
  Todavía estamos trabajando en esta característica, ¡pero nos encantaría que la probaras!

  Esta función se proporciona actualmente como parte de una vista previa de conformidad con nuestras [políticas de prelanzamiento](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy/).
</Callout>

Puede instalar el NRDOT Collector para el monitoreo de Microsoft SQL Server utilizando la instalación guiada desde la plataforma de New Relic, que proporciona una configuración guiada por la plataforma a través de la UI de New Relic. Este es el enfoque recomendado para inicios rápidos.

<Steps>
  <Step>
    ### Requisitos previos [#prerequisites]

    Asegúrese de que su entorno cumpla con nuestra [compatibilidad y requisitos previos](/docs/opentelemetry/database/mssql/compatibility).
  </Step>

  <Step>
    ### Instalación [#guided]

    Para instalar el NRDOT Collector utilizando la instalación guiada:

    1. Inicie sesión en su [cuenta de New Relic](https://one.newrelic.com/).

    2. Vaya a <DNT>**left navigation pane &gt; + Integration &amp; Agents &gt; MSSQL (OpenTelemetry)**</DNT>.

    3. En la pantalla <DNT>Select an account</DNT>, seleccione la cuenta en la que desea instalar la integración de MSSQL y haga clic en <DNT>**Continue**</DNT>.

    4. En la pantalla <DNT>Enter your credential</DNT>, seleccione una de las siguientes opciones y luego haga clic en **Continue**:

       * **Use una clave existente**: si ya tiene una clave de licencia, proporcione la clave de licencia. Para obtener más información, consulta [Claves de usuario](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
       * **Crear una nueva clave**: si no tiene una clave de licencia, haga clic en **Create a new key** para crear una.

    5. En la pantalla <DNT>Select platform</DNT>, seleccione la plataforma para instalar la integración de MSSQL y haga clic en <DNT>**Continue**</DNT>.

    6. Siga las instrucciones en pantalla para configurar la integración para la plataforma seleccionada.

    7. Después de la instalación, en la pantalla <DNT>Test your integration</DNT>, haga clic en **Test connection** para verificar que el Collector NRDOT pueda conectarse a su instancia de SQL Server y enviar datos a New Relic.
  </Step>

  <Step>
    ### (Opcional) Configurar la correlación de APM y base de datos [#correlation]

    Para correlacionar el rendimiento de su aplicación con las operaciones de la base de datos, puede configurar la identificación del servicio de base de datos. Esta función le permite ver exactamente qué aplicaciones están generando workloads de base de datos específicas. Para obtener más información, consulte [configurar la identificación de servicios de base de datos](/docs/opentelemetry/database/database-service-identification/) para obtener la correlación entre APM y la base de datos en New Relic.

    <Callout variant="important">
      Para ver los datos de rendimiento de la base de datos en APM, ambas entidades deben estar en la misma cuenta de New Relic. Si las entidades están en diferentes cuentas de New Relic, debe tener acceso a ambas cuentas para ver los datos.
    </Callout>
  </Step>

  <Step>
    ### (Opcional) Configurar la gestión de secretos [#rds-secret-management]

    La función de gestión de secretos le permite administrar de forma segura información confidencial, como las credenciales de bases de datos. Esto ayuda a mejorar la seguridad de su configuración de monitoreo al evitar la codificación rígida de datos sensibles en los archivos de configuración. Para obtener más información, consulta [gestión de secretos](/docs/opentelemetry/database/database-service-identification/#secret-management).
  </Step>

  <Step>
    ### Encuentra y utiliza tus datos [#find]

    Una vez que sus datos se estén recopilando, puede acceder al monitoreo integral de bases de datos SQL Server a través de la UI de New Relic.

    Para encontrar su entidad de base de datos SQL Server en New Relic:

    1. Vaya a <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>.
    2. En el menú desplegable **<DNT>Entity type</DNT>**, seleccione <DNT>**MSSQL instance**</DNT> y luego haga clic en <DNT>**Apply**</DNT>.
    3. Seleccione su base de datos SQL Server de la lista de entidades.
  </Step>
</Steps>
