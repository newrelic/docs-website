          ### Requisitos previos [#prerequisites-sql]

        <Step>
          ### Configurar el colector NRDOT [#setup-sql]

          Instale el Collector NRDOT en una instancia EC2 de Windows o en una máquina local de Windows que pueda conectarse a su instancia de RDS SQL Server.

          <Callout variant="important">
            Para el monitoreo de RDS, instale el Collector NRDOT en una instancia EC2 en la misma VPC que su instancia de RDS para obtener una conectividad de red óptima y una menor latencia.
          </Callout>

          1. Descarga e instala el NRDOT Collector usando PowerShell:

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

          2. Asegúrese de que su grupo de seguridad permita conexiones entrantes desde el host del Collector de NRDOT a su instancia de RDS en el puerto 1433 (o su puerto personalizado).
        </Step>

        <Step>
          ### Configurar el usuario de la base de datos [#configure-user-sql]

          Conéctese a su instancia de RDS SQL Server usando SQL Server Management Studio (SSMS) o sqlcmd con sus credenciales de administrador.

          Ejecute el siguiente script para crear el usuario de monitoreo de `newrelic` con los permisos adecuados:

          ```sql
          USE [master];
          GO
          CREATE LOGIN [newrelic] WITH PASSWORD = '<PASSWORD>';
          GO

          GRANT VIEW SERVER STATE TO [newrelic];
          GRANT VIEW ANY DEFINITION TO [newrelic];
          GRANT VIEW ANY DATABASE TO [newrelic];
          GO

          DECLARE @name SYSNAME;
          DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
          SELECT [name]
          FROM [master].[sys].[databases]
          WHERE [name] NOT IN ('master', 'msdb', 'model', 'rdsadmin', 'distribution')
          AND [state] = 0;
          OPEN db_cursor;
          FETCH NEXT FROM db_cursor INTO @name;
          WHILE @@FETCH_STATUS = 0
          BEGIN
            BEGIN TRY
              EXEC('USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''newrelic'')
              BEGIN
                CREATE USER [newrelic] FOR LOGIN [newrelic];
              END;
              GRANT VIEW DATABASE STATE TO [newrelic];');
            END TRY
            BEGIN CATCH
              PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
            END CATCH
            FETCH NEXT FROM db_cursor INTO @name;
          END
          CLOSE db_cursor;
          DEALLOCATE db_cursor;
          GO
          ```

          <Callout variant="tip">
            **Para RDS:** use su nombre de usuario maestro de RDS (o un usuario con el rol `rds_superuser`) para ejecutar estos comandos. El script excluye automáticamente la base de datos `rdsadmin` que está reservada por AWS.
          </Callout>
        </Step>

          ### Parámetro de configuración

          La siguiente tabla describe los parámetros de configuración clave para el receptor nrsqlserver:

          | Parámetro              | Descripción                                                                                                                                                                                  |
          | ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
          | `<YOUR_USERNAME>`      | Ingrese su nombre de usuario de la base de datos para la autenticación                                                                                                                       |
          | `<YOUR_PASSWORD>`      | Ingrese su contraseña de la base de datos para la autenticación                                                                                                                              |
          | `<YOUR_SERVER>`        | Ingrese el nombre de host o la dirección IP de SQL Server                                                                                                                                    |
          | `<YOUR_PORT>`          | Ingrese el número de puerto de SQL Server. El valor predeterminado es 1433.                                                                                                                  |
          | `<YOUR_OTLP_ENDPOINT>` | Ingrese el endpoint OTLP de New Relic. Para obtener más información, consulta la [documentación de los endpoints OTLP de New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/). |
          | `<YOUR_LICENSE_KEY>`   | Ingresa tu clave de licencia de New Relic.                                                                                                                                                   |
          | `collection_interval`  | Intervalo en segundos para recopilar métricas de base de datos. Predeterminado: 15s                                                                                                          |

          <Callout variant="tip">
            **Nota:** La opción `max_concurrent_queries` controla cuántas consultas de métricas ejecuta el receptor en paralelo contra SQL Server durante cada ciclo de recopilación.

            ```yaml
            receivers:
              nrsqlserver:
                max_concurrent_queries: 4  # default is 4 if omitted or set to 0
            ```
          </Callout>

          <Callout variant="tip">
            **Nota:** si habilita todas las métricas disponibles, aumente el valor de `limit_mib` (por ejemplo, 1000) para evitar que el limitador de memoria descarte datos.
          </Callout>
        </Step>

        <Step>
          ### Actualizar el registro del servicio de Windows [#update-registry-sql]

          Actualice el registro de servicios de Windows para usar el nuevo archivo mssql-config.yaml.

          **Método 1: PowerShell**

          Ejecute el siguiente comando en PowerShell como administrador:

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **Método 2: símbolo del sistema (alternativo)**

          Si PowerShell no está disponible, puede usar el símbolo del sistema en su lugar:

          1. Abra el menú Inicio, escriba `cmd`
          2. Haz clic derecho en el símbolo del sistema y elige **Run as administrator**
          3. Copie y pegue este comando exacto en la ventana negra de CMD y presione Enter:

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
        </Step>

          **Requisitos adicionales para la autenticación de dominio de Windows con RDS:**

        <Step>
          ### Habilite su servidor Microsoft SQL [#enable-sql-server-domain]

          Conéctese a su instancia de RDS SQL Server usando SQL Server Management Studio (SSMS) o sqlcmd con sus credenciales de administrador.

          Ejecute el siguiente script para crear un inicio de sesión de dominio de Windows y otorgar los permisos necesarios:

          ```sql
          USE master;
          CREATE LOGIN [DomainName\DomainUser] FROM WINDOWS;
          GRANT VIEW ANY DATABASE   TO [DomainName\DomainUser];
          GRANT VIEW SERVER STATE TO [DomainName\DomainUser];
          GRANT VIEW ANY DEFINITION TO [DomainName\DomainUser];
          GO
          ```

          <Callout variant="important">
            Reemplace `DomainName\DomainUser` con su dominio y nombre de usuario reales (por ejemplo, `CORP\SQLMonitor`).
          </Callout>
        </Step>

        <Step>
          ### Otorgar privilegios de acceso de LECTURA [#grant-permissions-domain]

          Utilice las siguientes sentencias para otorgar privilegios de acceso de LECTURA al usuario de dominio en todas las bases de datos:

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
                AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''DomainName\DomainUser'')
                BEGIN
                  CREATE USER [DomainName\DomainUser] FOR LOGIN [DomainName\DomainUser];
                END;
                GRANT VIEW DATABASE STATE TO [DomainName\DomainUser];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="tip">
            **Para RDS:** El script excluye automáticamente la base de datos `rdsadmin` que está reservada por AWS. Asegúrese de reemplazar `DomainName\DomainUser` con sus credenciales de dominio reales.
          </Callout>
        </Step>

        <Step>
          ### Configurar el NRDOT Collector [#setup-domain]

          Instale el Collector de NRDOT en una instancia EC2 de Windows unida al dominio o en una máquina local de Windows que sea parte del mismo dominio y pueda conectarse a su instancia de RDS SQL Server.

          <Callout variant="important">
            Para el monitoreo de RDS, instale el Collector NRDOT en una instancia EC2 en la misma VPC que su instancia de RDS para obtener una conectividad de red óptima y una menor latencia.
          </Callout>

          1. Descarga e instala el NRDOT Collector usando PowerShell:

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

          2. **Cambiar el inicio de sesión del servicio a la cuenta de dominio (usando PowerShell):**

             Configure el servicio NRDOT Collector para que se ejecute como su usuario de dominio usando PowerShell:

             ```powershell
             # Configure service to use domain account
             # The space after obj= and password= is required syntax for sc.exe
             sc.exe config "nrdot-collector" obj= "DomainName\DomainUser" password= "<password>"

             # Restart the service
             Restart-Service nrdot-collector
             ```

             Verifique la identidad del servicio:

             ```powershell
             Get-WmiObject Win32_Service -Filter "Name='nrdot-collector'" | Select Name, State, StartName
             # Expected: StartName = DomainName\DomainUser
             ```

             <Callout variant="important">
               Reemplace `DomainName\DomainUser` con su dominio y nombre de usuario reales (por ejemplo, `CORP\SQLMonitor`), y `<password>` con la contraseña del usuario del dominio.
             </Callout>

          3. Asegúrese de que su grupo de seguridad permita conexiones entrantes desde el host del Collector de NRDOT a su instancia de RDS en el puerto 1433 (o su puerto personalizado).
        </Step>

        <Step>
          ### Habilite su servidor Microsoft SQL [#enable-sql-server-gmsa]

          Conéctese a su instancia de RDS SQL Server y ejecute los siguientes comandos SQL para crear un inicio de sesión para la cuenta gMSA y otorgar los permisos necesarios:

          ```sql
            USE master;
            CREATE LOGIN [DomainName\gMSA] FROM WINDOWS;
            GRANT VIEW ANY DATABASE TO [DomainName\gMSA];
            GRANT VIEW SERVER STATE TO [DomainName\gMSA];
            GRANT VIEW ANY DEFINITION TO [DomainName\gMSA];
          ```

          <Callout variant="important">
            Reemplace `DomainName\gMSA` con su nombre de dominio real y el nombre de la cuenta gMSA (por ejemplo, `CORP\SQL-gMSA$`). Tenga en cuenta que las cuentas gMSA generalmente terminan con un símbolo `$`.
          </Callout>
        </Step>

        <Step>
          ### Otorgar privilegios de acceso de LECTURA [#grant-read-access-rds-gmsa]

          Use las siguientes sentencias para otorgar privilegios de acceso de LECTURA a la cuenta gMSA en todas las bases de datos:

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
              AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''<DOMAIN>\<gMSA_NAME>$'')
                BEGIN
                  CREATE USER [<DOMAIN>\<gMSA_NAME>$] FOR LOGIN [<DOMAIN>\<gMSA_NAME>$];
                END;
                GRANT VIEW DATABASE STATE TO [<DOMAIN>\<gMSA_NAME>$];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="important">
            Reemplace `DomainName\gMSA` con su nombre de dominio real y el nombre de la cuenta gMSA (por ejemplo, `CORP\SQL-gMSA$`). Este script crea un usuario de base de datos para la cuenta gMSA en todas las bases de datos de usuario, excluyendo las bases de datos del sistema y las bases de datos específicas de RDS.
          </Callout>
        </Step>

        <Step>
          ### Configurar el NRDOT Collector [#setup-gmsa]

          Instale el Collector de NRDOT en una instancia EC2 de Windows que esté unida al dominio y autorizada para usar la cuenta gMSA.

          <Callout variant="important">

          ### Parámetro de configuración

          La siguiente tabla describe los parámetros de configuración clave para el receptor nrsqlserver:

          | Parámetro              | Descripción                                                                                                                                                                                  |
          | ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
          | `<YOUR_RDS_ENDPOINT>`  | Ingrese su extremo de RDS SQL Server                                                                                                                                                         |
          | `<YOUR_PORT>`          | Ingrese el número de puerto de SQL Server. El valor predeterminado es 1433.                                                                                                                  |
          | `<YOUR_OTLP_ENDPOINT>` | Ingrese el endpoint OTLP de New Relic. Para obtener más información, consulta la [documentación de los endpoints OTLP de New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/). |
          | `<YOUR_LICENSE_KEY>`   | Ingresa tu clave de licencia de New Relic.                                                                                                                                                   |
          | `collection_interval`  | Intervalo en segundos para recopilar métricas de base de datos. Predeterminado: 15s                                                                                                          |

          <Callout variant="tip">
            La opción `max_concurrent_queries` controla cuántas consultas de métricas ejecuta el receptor en paralelo contra SQL Server durante cada ciclo de extracción.

            ```yaml
            receivers:
              nrsqlserver:
                max_concurrent_queries: 4  # default is 4 if omitted or set to 0
            ```
          </Callout>

          ### Actualizar el registro del servicio de Windows [#update-registry-gmsa]

          Actualice el registro de servicios de Windows para usar el nuevo archivo mssql-config.yaml.

          **Método 1: PowerShell**

          Ejecute el siguiente comando en PowerShell como administrador:

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **Método 2: símbolo del sistema (alternativo)**

          Si PowerShell no está disponible, puede usar el símbolo del sistema en su lugar:

          1. Abra el menú Inicio, escriba `cmd`
          2. Haz clic derecho en el símbolo del sistema y elige **Run as administrator**
          3. Copie y pegue este comando exacto en la ventana negra de CMD y presione Enter:

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
