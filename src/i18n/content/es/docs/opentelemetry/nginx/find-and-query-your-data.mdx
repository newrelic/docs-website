## Vea sus datos de NGINX en New Relic [#find-data]

Una vez que su configuración esté completa y los datos fluyan, puede ver sus métricas y logs de NGINX en New Relic y crear alertas personalizadas.

**Método 1: A través de integraciones y agentes**

1. Ve a [one.newrelic.com](https://one.newrelic.com) &gt; **Integrations &amp; Agents**
2. Haga clic en **Dashboards**
3. Busque y haga clic en el dashboard **NGINX OTel overview**
4. Seleccione su cuenta y haga clic en **View dashboard**

**Método 2: A través de la supervisión de la infraestructura**

1. Vaya a [one.newrelic.com](https://one.newrelic.com) &gt; **On-host integrations**
2. Busque su servidor NGINX por nombre de implementación (por ejemplo, `production-web-01`)
3. Haga clic en su servidor para ver métricas detalladas

## Consulta tus datos con NRQL

Puede crear consultas personalizadas para analizar sus datos de NGINX:

**Ejemplo de consulta:**

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## Solucionar problemas de visibilidad de datos

Si no ve datos:

1. Espere un momento: la ingesta inicial de datos puede tardar unos minutos
2. Verifique el rango de tiempo: asegúrese de estar viendo datos recientes
3. Verifique el nombre de la implementación: use el nombre exacto de la configuración de su Collector
4. Compruebe los datos con NRQL:
   ```sql
   FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
   ```

Si los datos aparecen de forma intermitente:

1. Verifique los logs del recolector en busca de errores:
   ```bash
   sudo journalctl -u otelcol-contrib -n 50
   ```
2. Verifique que NGINX esté generando tráfico - Pruebe con `curl http://localhost`
3. Verifique la autenticación: asegúrese de que la clave de licencia y el punto final sean correctos
