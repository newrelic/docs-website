## ¿Por qué monitorear NGINX con OpenTelemetry? [#why-monitor]

La monitorización de NGINX es esencial para mantener el rendimiento y la fiabilidad de las aplicaciones web. OpenTelemetry proporciona un enfoque estandarizado para recopilar y analizar estos datos críticos.

### Beneficios clave

* **Optimización del rendimiento**: Identifique los cuellos de botella, realice un seguimiento de los tiempos de respuesta, tome decisiones de escalado basadas en datos
* **Monitorización proactiva**: Recibe alertas antes del impacto del usuario, detecta problemas de forma temprana
* **Información operativa**: comprenda los patrones de tráfico, realice un seguimiento del cumplimiento de los SLA, mantenga los registros de auditoría
* **Preparado para el futuro**: estándar OpenTelemetry de código abierto, integración neutral al proveedor

### Caso de uso

Ya sea que estés ejecutando un sitio web de comercio electrónico, un gateway de API o una plataforma de entrega de contenido, la monitorización de NGINX te ayuda a mantener servicios web rápidos y confiables. Recibe notificaciones cuando tu proceso de pago se ralentice, detecta cuándo tus endpoints de API están sobrecargados o identifica qué regiones geográficas están experimentando problemas. Esta monitorización es esencial para sitios web que gestionan transacciones de clientes, backends de aplicaciones móviles, arquitecturas de microservicios y cualquier aplicación web donde el tiempo de inactividad o el rendimiento lento impacten directamente a tus usuarios y a tu negocio.

## Métricas disponibles [#metrics]

El OpenTelemetry Collector recopila automáticamente métricas clave de rendimiento de NGINX, incluidas las estadísticas de conexión, los recuentos de solicitudes y los indicadores de estado del servidor.

**Métricas clave recopiladas:**

* **Métricas de conexión**: conexiones activas, aceptadas, manejadas y sus estados
* **Métricas de solicitud** - Solicitudes totales y tasas de solicitud a lo largo del tiempo
* **Estado del servidor**: indicadores de eficiencia y rendimiento de la conexión

**Para una referencia completa de métricas y atributos:** Consulte [Referencia de métricas y atributos de NGINX OpenTelemetry](/docs/opentelemetry/nginx/nginx-otel-metrics-reference/) para obtener descripciones detalladas, tipos de métricas y atributos de recursos para implementaciones tanto en el host como en Kubernetes.
