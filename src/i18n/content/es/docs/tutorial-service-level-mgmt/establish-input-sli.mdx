Utilizamos monitoreo sintético para simular conexiones de todo el mundo. Si hay una interrupción entre el punto de origen y la salida, las comprobaciones sintéticas fallarán, afectarán las puntuaciones de sus objetivos de nivel de servicio y activarán una alerta crítica muy alta. Nos referimos a esto como rendimiento de entrada.

Básicamente, el rendimiento de la entrada se mide como la capacidad de conectarse a su aplicación, también llamado "tiempo de actividad". Puede encontrar los pasos para configurar SLI de rendimiento de entrada a continuación, así como ejemplos de consulta NRQL para que los pruebe.

<Steps>
  <Step>
    ## Crea tus cheques sintéticos

    El nivel de servicio de rendimiento de entrada más común suele denominarse "conectividad" o "tiempo de actividad". Es una simple verificación de los extremos de salud de la API o la carga de una URL. Puede hacer ambas cosas utilizando nuestro servicio de monitoreo sintético. Antes de crear su indicador de nivel de servicio, consulte [Agregar un monitor browser simple](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#simple) y [Agregar una prueba de API con script](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#complex) para saber cómo comenzar a generar informes de datos si aún no lo ha hecho.
  </Step>

  <Step>
    ## Crea tu indicador de nivel de servicio

    Una vez que tenga el informe de datos, puede utilizar el servicio de administración a nivel de servicio para crear un indicador de entrada y un objetivo mediante el siguiente procedimiento:

    1. Desde el [explorador de entidades](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts#find), seleccione **Service levels**.

    2. Haga clic en **+ Add a service level indicator**. Si no ve el botón **Add a service level** , consulte con su administrador de New Relic acerca de sus permisos.

    3. Filtre sus tipos de entidad a `Synthetic monitors`, como en la captura de pantalla siguiente:

       <img
         alt="Filter entity types in service levels"
         src="/images/apm_screenshot-crop_add-input-service-level.webp"
       />

    4. Busque su monitor de monitoreo sintético en la lista y haga clic en él. Esto habilita el botón **Continue** en el panel izquierdo. Haga clic en **Continue**.

    5. Verá un botón para las configuraciones recomendadas para un nivel de servicio **Success** (que se muestra a continuación). Pinchalo.

       <img
         alt="Synthetics input service level guided flow"
         src="/images/apm_screenshot-crop_synthetics-setup.webp"
       />

    6. Realice los cambios apropiados en la etiqueta, el título y la descripción según sea necesario.

    7. Clic en **Save**.
  </Step>
</Steps>
