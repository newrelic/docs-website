Sintético monitor son comprobaciones de API o instancias browser virtual que monitor su sitio web, registrando cada verificación en detalle. El monitor puede:

* Capture números agregados, incluida una descripción general o un resumen.
* Proporciona estadísticas detalladas sobre los eventos de alerta de tiempo de inactividad y los recursos de la página.
* Recopile códigos de respuesta personalizados que detallen el rendimiento de su monitor.

Puedes crear un monitor Sintético sin instalación. Para empezar:

1. Crea una [cuentaNew Relic ](https://newrelic.com/signup).
2. Vaya a <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT> y haga clic en **Crear un monitor**.
3. Seleccione el monitor que desea crear. Si es tu primera vez, te recomendamos crear un ping o un monitor de pasos para comenzar.

## Crear un monitor [#adding-monitors]

¿Listo para monitor sus aplicaciones web con uno (¡o varios!) de nuestro [monitor Sintético](/docs/synthetics/synthetic-monitoring/using-monitors/intro-synthetic-monitoring/#types-of-synthetic-monitors)? Los pasos a continuación lo guiarán a través del proceso para todos nuestros monitores.

  <Collapser id="simple" title="Agregue un ping o un simple monitor browser">
    1. Ir a <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT>
    2. Haga clic en el mosaico <DNT>**Availability**</DNT> para monitor de ping o en <DNT>**Page load performance**</DNT> para monitor browser simple.

    Espere unos minutos, luego verifique su monitor desde el [índice de<DNT>**Monitors**</DNT> ](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-summary).

    Tienes algunas opciones de configuración adicionales para tu ping o monitor browser simple. Para ambos tipos de monitor, puede configurar:

    * Un <DNT>**validation string**</DNT> que habilita el monitoreo de subcadenas para [la validación de respuestas](#response-validation).

    * <DNT>**Custom Headers**</DNT> que acompañará las solicitudes enviadas por su monitor.

    * <DNT>**Verify SSL**</DNT> para validar la cadena del certificado SSL. Se puede duplicar ejecutando la siguiente sintaxis:

      ```sh
      openssl s_client -servername YOUR_HOSTNAME -connect YOUR_HOSTNAME:443 -CApath /etc/ssl/certs -verify_hostname YOUR_HOSTNAME > /dev/null
      ```

      Si se devuelve un código de salida distinto de cero, el monitor fallará.

    Con solo el monitor de ping, puede optar por configurar:

    * La opción <DNT>**Bypass HEAD request**</DNT> para omitir la solicitud `HEAD` predeterminada y utilizar el verbo `GET` en su lugar.
    * <DNT>**Redirect is Failure**</DNT>, que categorizará el redireccionamiento como fallido, en lugar de seguir el redireccionamiento a la nueva URL.
    * <DNT>**Custom Headers**</DNT> se puede agregar al ping y al monitor browser simple. Estos encabezados se agregarán a las solicitudes enviadas por el monitor.

    Con solo un simple monitor browser , puede seleccionar entre Chrome o Firefox y habilitar la configuración [de emulación del dispositivo](/docs/synthetics/synthetic-monitoring/using-monitors/device-emulation) , incluido el tipo de dispositivo y la orientación de la pantalla.

¿Quiere implementar más de un monitor a la vez? Recomendamos consultar el [tutorial de monitoreo sintético de la API NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-synthetics-tutorial). Puede automatizar la creación monitor con llamada API.

También puede crear monitores con nuestra [API REST de monitoreo sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api). Para las API REST, realice una solicitud `GET` para recuperar los detalles de configuración del monitor Sintético elegido. Una vez que haya recuperado esos datos, use `POST` para crear una &quot;copia&quot; del monitor elegido.

## Editar un monitor [#editing-monitors]

<img src="/images/synthetic_screenshot-full_edit-monitor-from-general.webp" title="Edit a monitor page" alt="A screenshot that shows the New Relic UI when you're editing your monitor." />

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets) &gt; General**</DNT>: Mantenga su monitor actualizado para que New Relic siempre reciba los datos más pertinentes.
</figcaption>

No puede cambiar monitor el [tipo de](#setting-type) un después de monitor , pero puede editar otras monitor configuraciones .

1. Vaya a <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> y luego seleccione el monitor que desea editar.

2. En el menú lateral, seleccione un enlace para cambiar las siguientes [configuraciones](#settings):

   * Seleccione <DNT>**Settings &gt; General**</DNT> para editar [el nombre](#setting-name), la URL, [las ubicaciones](#setting-location), [la frecuencia](#setting-frequency) y las opciones avanzadas.
   * Para <DNT>**Scripted browser**</DNT> los <DNT>**API test**</DNT> monitores y , seleccione <DNT>**Settings &gt; Script**</DNT> para editar la del monitor script.
   * Para alertas de monitoreo sintético, haga clic en <DNT>**Manage alerts**</DNT>.

3. Seleccione <DNT>**Save changes**</DNT> para confirmar.

  Convierta al [nuevo tiempo de ejecución sintético](/docs/synthetics/synthetic-monitoring/getting-started/new-runtime) cuando edite su monitor. Asegúrate de actualizar tu lista de &apos;permitidos&apos; para aceptar solicitudes de red de [nuevos rangos de IP públicas](/docs/synthetics/synthetic-monitoring/administration/new-synthetic-public-minion-ips).

## Eliminar un monitor [#deleting-monitors]

<img src="/images/synthetic_screenshot-crop_delete-monitor.webp" title="Edit a monitor page to delete monitor" alt="A screenshot of the Edit a monitor page with a red arrow that shows where someone should look to delete a monitor." />

<figcaption>

Debe tener [privilegios de administrador](/docs/synthetics/synthetic-monitoring/administration/user-roles-synthetic-monitoring/) para poder eliminar un monitor.

Para eliminar un monitor:

1. Vaya a <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> y luego seleccione el monitor que desea editar.
2. Haga clic en <DNT>**General**</DNT> y luego haga clic en el botón Eliminar monitor.

También puede eliminar un monitor con nuestra [API NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-synthetics-tutorial) o la [API REST de monitoreo sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#delete-monitor).

## Configuraciones del monitor [#settings]

Al configurar el monitor, están disponibles las siguientes configuraciones:

<CollapserGroup>
  <Collapser id="setting-type" title="Tipo">
    Seleccione el tipo de monitor que desea crear. El [tipo](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) de monitor no se puede cambiar una vez creado el monitor.

    * <DNT>**Ping**</DNT>: especifique una única URL para monitor la disponibilidad. New Relic verificará esta URL a través de HEAD o solicitud GET. El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) no configurable para este monitor es de 60 segundos.
    * <DNT>**Simple browser**</DNT>:Especifique una única URL para monitor a través del browser real. Una vez en cada [intervalo de frecuencia](#setting-frequency), New Relic verificará esta URL a través de un SeleniumChrome o Firefox browser con . El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers/#waiting-elements) no configurable para este monitor es de 60 segundos.
    * <DNT>**Scripted browser**</DNT>: [Cree un script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) para ejecutar Seleniumun Chrome o Firefox browser con . El browser sigue cada paso del script para verificar que el comportamiento complejo funcione como se espera (por ejemplo, buscar en un sitio web y luego hacer clic en uno de los resultados de búsqueda). El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) no configurable para este monitor es de 180 segundos.
    * <DNT>**API test**</DNT>: Cree un script API para garantizar que sus extremos de API funcionen correctamente. Para obtener más información, consulte [Escribir pruebas de API](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests). El tiempo de espera no configurable para este monitor es de 180 segundos.
    * <DNT>**Step monitor**</DNT>:Una opción sin código para configurar uno o más pasos para ejecutar un SeleniumChrome o Firefox con browser. El browser sigue cada paso para verificar que el comportamiento complejo funcione como se espera (por ejemplo, buscar en un sitio web y luego hacer clic en uno de los resultados de búsqueda). El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) no configurable para este monitor es de 180 segundos.
    * <DNT>**Certificate check**</DNT>:Comprueba si un certificado SSL caducará en un número configurable de días o menos.
    * <DNT>**Broken links**</DNT>:Pruebe todos los enlaces que se encuentren en una URL para cerciorar de que respondan con códigos de respuesta HTTP exitosos.
  </Collapser>

  <Collapser id="setting-name" title="Nombre del monitor">
    Define un nombre para el monitor. Los nombres monitor no pueden contener corchetes angulares sin codificar (`<` y `>`). Para incluir corchetes angulares en el nombre de un monitor, codifíquelos como entidad de corchetes HTML (`&lt;` para `<` y `&gt;` para `>`) en la UI o API.
  </Collapser>

  <Collapser id="setting-location" title="Ubicación">
    Seleccione las ubicaciones donde desea que se ejecute su monitor. Seleccione más ubicaciones para asegurarse de que su aplicación esté disponible para usuarios de todo el mundo. Si tiene alguna [ubicación privada](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations), también aparecerá aquí. Puede utilizar el [extremo `location` de Sintético API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#list-locations) para recuperar una lista de ubicaciones válidas para su cuenta.

    Su monitor ejecutará una verificación desde cada ubicación seleccionada durante cada [intervalo de frecuencia](#setting-frequency). Por ejemplo, si selecciona tres ubicaciones y define una frecuencia de 15 minutos, su monitor ejecutará tres comprobaciones en cada período de 15 minutos (o 8640 comprobaciones por mes).
  </Collapser>

  <Collapser id="setting-frequency" title="Frecuencia">
    Seleccione la frecuencia con la que se ejecuta el monitor, en incrementos de minutos, horas o 1 día. Esta frecuencia se aplica a cada [ubicación](#setting-location). Por ejemplo, si selecciona tres ubicaciones y una Frecuencia de <DNT>**15 minutes**</DNT>, su monitor ejecutará tres comprobaciones, en promedio cada 5 minutos, en cada período de 15 minutos (o 8640 comprobaciones por mes).
  </Collapser>

  <Collapser id="response-validation" title="Validación de respuesta (opcional)">
    Especifique el texto a buscar en la página DOM. Cuando se utilizan browser simples o [tiposmonitor ](/docs/synthetics/synthetic-monitoring/getting-started/types-synthetic-monitors/#types-monitors)de ping, hay un límite de 1 MB (10^6 bytes) en la carga de la página.
  </Collapser>

  <Collapser id="apdex-t" title="Apdex T (opcional)">
    Especifique el umbral para el <DNT>**tolerable**</DNT> tiempo de respuesta de su monitor. El valor predeterminado es 7 segundos (7000 ms). Para obtener más información, consulte [Comprender el informe SLA métrico](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-aggregate-monitor-metrics/#understanding).
  </Collapser>
</CollapserGroup>
