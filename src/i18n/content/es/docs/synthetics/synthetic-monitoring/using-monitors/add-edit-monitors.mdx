Sintético monitor son comprobaciones de API o instancias browser virtual que monitor su sitio web, registrando cada verificación en detalle. El monitor puede:

* Capture números agregados, incluida una descripción general o un resumen.
* Proporcione estadísticas detalladas sobre incidentes de tiempo de inactividad y recursos de la página.
* Recopile códigos de respuesta personalizados que detallen el rendimiento de su monitor.

Puedes crear un monitor Sintético sin instalación. Para empezar:

1. Crea una [cuentaNew Relic ](https://newrelic.com/signup).

2. Vaya a

   <DoNotTranslate>**[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DoNotTranslate>

   y haga clic en **Crear un monitor**.

3. Seleccione el monitor que desea crear. Si es tu primera vez, te recomendamos crear un ping o un monitor de pasos para comenzar.

    Tienes algunas opciones de configuración adicionales para tu ping o monitor browser simple. Para ambos tipos de monitor, puede configurar:

    * Un <DoNotTranslate>**validation string**</DoNotTranslate> que habilita el monitoreo de subcadenas para [la validación de respuestas](#response-validation).

    * <DoNotTranslate>**Custom Headers**</DoNotTranslate> que acompañará las solicitudes enviadas por su monitor.

    * <DoNotTranslate>**Verify SSL**</DoNotTranslate> para validar la cadena del certificado SSL. Se puede duplicar ejecutando la siguiente sintaxis:

      ```
      openssl s_client -servername {YOUR_HOSTNAME} -connect {YOUR_HOSTNAME}:443 -CApath /etc/ssl/certs -verify_hostname {YOUR_HOSTNAME} > /dev/null
      ```

      Si se devuelve un código de salida distinto de cero, el monitor fallará.

    * La opción

      <DoNotTranslate>**Bypass HEAD request**</DoNotTranslate>

    * <DoNotTranslate>**Redirect is Failure**</DoNotTranslate>

      , que categorizará el redireccionamiento como fallido, en lugar de seguir el redireccionamiento a la nueva URL.

    * <DoNotTranslate>**Custom Headers**</DoNotTranslate>

      se puede agregar al ping y al monitor browser simple. Estos encabezados se agregarán a las solicitudes enviadas por el monitor.

También puede crear monitores con nuestra [API REST de monitoreo sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api). Para las API REST, realice una solicitud `GET` para recuperar los detalles de configuración del monitor Sintético elegido. Una vez que haya recuperado esos datos, use `POST` para crear una "copia" del monitor elegido.

## Configuraciones del monitor [#settings]

Al configurar el monitor, están disponibles las siguientes configuraciones:

<CollapserGroup>
  <Collapser
    id="setting-type"
    title="Tipo"
  >
    Seleccione el tipo de monitor que desea crear. El [tipo](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) de monitor no se puede cambiar una vez creado el monitor.

    * <DoNotTranslate>**Ping**</DoNotTranslate>

      : especifique una única URL para monitor la disponibilidad. New Relic verificará esta URL a través de HEAD o solicitud GET. El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) no configurable para este monitor es de 60 segundos.

    * <DoNotTranslate>**Simple browser**</DoNotTranslate>

      : Especifique una única URL para monitor a través de browser real . Una vez en cada [intervalo de frecuencia](#setting-frequency), New Relic verificará esta URL a través de un browser Google Chrome con tecnología Selenium. El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers/#waiting-elements) no configurable para este monitor es de 60 segundos.

    * <DoNotTranslate>**Scripted browser**</DoNotTranslate>

      : [cree una script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) para controlar un browser Google Chrome con tecnología Selenium. El browser sigue cada paso del script para verificar que el comportamiento complejo funcione según lo esperado (por ejemplo, buscar en un sitio web y luego hacer clic en uno de los resultados de la búsqueda). El [tiempo de espera](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) no configurable para este monitor es de 180 segundos.

    * <DoNotTranslate>**API test**</DoNotTranslate>

      : Cree un script API para garantizar que sus extremos de API funcionen correctamente. Para obtener más información, consulte [Escribir pruebas de API](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests). El tiempo de espera no configurable para este monitor es de 180 segundos.
  </Collapser>

  <Collapser
    id="setting-name"
    title="Nombre del monitor"
  >
    Define un nombre para el monitor. Los nombres monitor no pueden contener corchetes angulares sin codificar (`<` y `>`). Para incluir corchetes angulares en el nombre de un monitor, codifíquelos como entidad de corchetes HTML (`&lt;` para `<` y `&gt;` para `>`) en la UI o API.
  </Collapser>

  <Collapser
    id="setting-location"
    title="Ubicación"
  >
    Seleccione las ubicaciones donde desea que se ejecute su monitor. Seleccione más ubicaciones para asegurarse de que su aplicación esté disponible para usuarios de todo el mundo. Si tiene alguna [ubicación privada](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations), también aparecerá aquí. Puede utilizar el [extremo `location` de Sintético API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#list-locations) para recuperar una lista de ubicaciones válidas para su cuenta.

    Su monitor ejecutará una verificación desde cada ubicación seleccionada durante cada [intervalo de frecuencia](#setting-frequency). Por ejemplo, si selecciona tres ubicaciones y define una frecuencia de 15 minutos, su monitor ejecutará tres comprobaciones en cada período de 15 minutos (o 8640 comprobaciones por mes).
  </Collapser>

  <Collapser
    id="setting-frequency"
    title="Frecuencia"
  >
    Seleccione la frecuencia con la que se ejecuta el monitor, en incrementos de minutos, horas o 1 día. Esta frecuencia se aplica a cada [ubicación](#setting-location). Por ejemplo, si selecciona tres ubicaciones y una Frecuencia de <DoNotTranslate>**15 minutes**</DoNotTranslate>, su monitor ejecutará tres comprobaciones, en promedio cada 5 minutos, en cada período de 15 minutos (o 8640 comprobaciones por mes).
  </Collapser>

  <Collapser
    id="setting-alerts"
    title="Alerta"
  >
    Especifique una dirección de correo electrónico para recibir <InlinePopover type="alerts"/>cuando falla un monitor. O conecte un monitor a una política de alertas existente para obtener más opciones de notificación. Para obtener más información, consulte [Alertas para monitoreo sintético](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics).
  </Collapser>

  <Collapser
    id="response-validation"
    title="Validación de respuesta (opcional)"
  >
    Especifique el texto a buscar en la página DOM. Cuando se utilizan browser simples o [tiposmonitor ](/docs/synthetics/synthetic-monitoring/getting-started/types-synthetic-monitors/#types-monitors)de ping, hay un límite de 1 MB (10^6 bytes) en la carga de la página.
  </Collapser>

  <Collapser
    id="apdex-t"
    title="Apdex T (opcional)"
  >
    Especifique el umbral para el <DoNotTranslate>**tolerable**</DoNotTranslate> tiempo de respuesta de su monitor. El valor predeterminado es 7 segundos (7000 ms). Para obtener más información, consulte [Comprender el informe SLA métrico](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-aggregate-monitor-metrics/#understanding).
  </Collapser>
</CollapserGroup>
