New Relic [dejará de ofrecer soporte para los entornos de ejecución más antiguos](/eol/2026/05/eol-05-18-26-node-16-chrome-134-runtimes/) en 2026. Para simplificar la migración de los monitores sintéticos a los runtimes compatibles más recientes, actualizamos automáticamente todos los monitores que no requerían su intervención. Utilice la herramienta de actualización de tiempo de ejecución para ver los monitores restantes que aún necesitan su atención.

## Actualiza tus monitores restantes [#upgrade-options]

Para cada monitor que requiera atención, puede elegir uno de dos caminos.

<Tabs>
  <TabsBar>
    <TabsBarItem id="fix-manually">
      Corregir manualmente
    </TabsBarItem>

    <TabsBarItem id="fix-with-ai">
      Corregir con IA
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="fix-manually">
      Para actualizar un monitor escribiendo o editando manualmente los scripts de su monitor, siga los pasos.

      1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.

      2. Ubique el monitor y haga clic en <DNT>**Fix manually**</DNT>.

      3. En la ventana <DNT>**Fix script manually**</DNT>, actualice su código de script para el nuevo entorno de ejecución.

      4. Haga clic en <DNT>**Close and start validation**</DNT>. El monitor se mueve a la tabla <DNT>**Monitors we&apos;re validating**</DNT> mientras el sistema valida el script actualizado.

         * Si la validación falla, haga clic en **⋯** junto al monitor, seleccione <DNT>**Fix manually**</DNT> del menú, actualice el script y haga clic en <DNT>**Close and start validation**</DNT> de nuevo.

      5. Una vez que la validación sea exitosa y el estado se muestre como <DNT>**Fixed**</DNT>, haga clic en la pestaña <DNT>**Ready to update**</DNT>.

      6. Seleccione la casilla de verificación junto al monitor validado y haga clic en <DNT>**Upgrade selected**</DNT>.

      Una vez finalizado, aparece una notificación de confirmación y el monitor se mueve a la pestaña <DNT>**Updated**</DNT> donde puede verificar la actualización exitosa.
    </TabsPageItem>

    <TabsPageItem id="fix-with-ai">
      <Callout title="Nota">
        * Esta característica requiere Calcular avanzado y no está disponible para cuentas <DNT>**FedRAMP**</DNT>. Comuníquese con el equipo de su cuenta de New Relic para obtener soporte.
        * Siempre pruebe los scripts generados por IA antes de desplegar los monitores en producción.
      </Callout>

      Use esta opción para permitir que <DNT>**New Relic AI (NRAI)**</DNT> genere un script actualizado para usted. La generación de script asistida por IA está orientada principalmente a corregir `timeout errors` que se producen durante las actualizaciones de tiempo de ejecución. Para los monitores con `script errors`, el script podría necesitar otros cambios. Puede hacerlos manualmente o usar <DNT>[SRE Agent](/docs/agentic-ai/sre-agent/overview/)</DNT> para depurar y obtener información valiosa sobre posibles actualizaciones relacionadas con el script.
    </TabsPageItem>
  </TabsPages>
</Tabs>

Puede ejecutar varias actualizaciones a la vez, lo que le permite trabajar en varios monitores en paralelo sin esperar a que el sistema termine de validar cada script.
