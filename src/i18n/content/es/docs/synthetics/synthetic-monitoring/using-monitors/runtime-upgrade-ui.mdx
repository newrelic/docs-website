New Relic [finalizará el soporte para runtimes más antiguos](/eol/2025/11/eol-chrome-140-node-22-synthetics/) en 2026. Para simplificar la migración de los monitores sintéticos a los runtimes compatibles más recientes, actualizamos automáticamente todos los monitores que no requerían su intervención. Utilice la herramienta de actualización de tiempo de ejecución para ver los monitores restantes que aún necesitan su atención.

## Requisitos previos [#prerequisites]

Antes de comenzar a actualizar los monitores, asegúrese de cumplir con los siguientes requisitos:

* **Permiso de actualización masiva del tiempo de ejecución:** requerido para aplicar actualizaciones masivas a través de la UI. [Los administradores de productos](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) tienen este permiso por defecto. También puede agregarlo a [roles personalizados](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).
* **Derechos de IA (solo <DNT>Fix with AI</DNT>):** requiere el derecho `nrai_ccu` o `nrai_discount_usage`. Las cuentas de FedRAMP están bloqueadas para esta experiencia. Comuníquese con su equipo de cuenta de New Relic para habilitar los derechos.

## Actualiza tus monitores restantes [#upgrade-options]

Para cada monitor que requiera atención, puede elegir una de dos rutas:

<CollapserGroup>
  <Collapser id="fix-manually" title="Corregir manualmente">
    Use la opción <DNT>**Fix manually**</DNT> para escribir y editar sus scripts de monitor en lugar de depender de la generación de IA. Para actualizar un monitor manualmente:

    1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.

    2. Ubique el monitor y haga clic en <DNT>**Fix manually**</DNT>.

    3. En la ventana <DNT>**Fix script manually**</DNT>, actualice su código de script para el nuevo entorno de ejecución.

    4. Haga clic en <DNT>**Close and start validation**</DNT>. El monitor se mueve a la tabla <DNT>**Monitors we&apos;re validating**</DNT> mientras el sistema valida el script actualizado.

       <Callout title="Nota">
         Si la validación falla, haga clic en **⋯** junto al monitor, seleccione <DNT>**Fix manually**</DNT> del menú, actualice el script y haga clic en <DNT>**Close and start validation**</DNT> de nuevo.
       </Callout>

    5. Una vez que la validación sea exitosa y el estado se muestre como <DNT>**Fixed**</DNT>, haga clic en la pestaña <DNT>**Ready to update**</DNT>.

    6. Seleccione la casilla de verificación junto al monitor validado y haga clic en <DNT>**Upgrade selected**</DNT>.

    Una vez finalizado, aparece una notificación de confirmación y el monitor se mueve a la pestaña <DNT>**Updated**</DNT> donde puede verificar la actualización exitosa.
  </Collapser>

  <Collapser id="fix-with-ai" title="Corregir con IA">
    Utilice la opción <DNT>**Fix with AI**</DNT> para que <DNT>New Relic AI (NRAI)</DNT> genere un script actualizado para usted. La generación de scripts asistida por IA está orientada principalmente a corregir `timeout` errores producidos durante las actualizaciones de runtime. Para otros tipos de errores, es probable que el propio script necesite cambios que debería realizar [manualmente](#fix-manually).

    Para actualizar un monitor usando IA:

    1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.
    2. Ubique el monitor y haga clic en <DNT>**Fix with AI**</DNT>.
    3. Revise el script sugerido por IA. Para ver los cambios específicos realizados en su código original, haga clic en <DNT>**Show comparison**</DNT> para abrir una vista de diferencias en paralelo.
    4. Haga clic en <DNT>**Continue to validation**</DNT>. El sistema coloca temporalmente el monitor en un estado de validación para probar el nuevo script y asegurarse de que se ejecute sin errores.
    5. Una vez validado el script, haga clic en la pestaña <DNT>**Ready to update**</DNT>.
    6. Seleccione la casilla de verificación junto al monitor validado y haga clic en <DNT>**Upgrade selected**</DNT>.

    Una vez finalizado, aparece una notificación de confirmación y el monitor se mueve a la pestaña <DNT>**Updated**</DNT> donde puede verificar la actualización exitosa.
  </Collapser>
</CollapserGroup>

Puede ejecutar varias actualizaciones a la vez, lo que le permite trabajar en varios monitores en paralelo sin esperar a que el sistema termine de validar cada script.
