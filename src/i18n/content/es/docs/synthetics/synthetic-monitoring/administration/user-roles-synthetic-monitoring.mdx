Si es un [usuario de plataforma completa](/docs/accounts/accounts-billing/new-relic-one-pricing-users/users-roles#user-type), en teoría tiene acceso completo a todas las características de New Relic, pero si puede usar todas esas características depende del rol que se le asigne dentro de su organización de New Relic.

Los detalles relacionados con los permisos dependen de si un usuario de New Relic está en nuestro modelo de usuario más nuevo o en nuestro [modelo de usuario](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) original.

## Usuarios en nuestro modelo de usuario más nuevo [#new-pricing]

Para obtener detalles sobre cómo funcionan los permisos de New Relic, consulte [conceptos de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#groups). Para ver las capacidades granulares relacionadas con Sintético, consulte [Capacidades](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).

## Usuarios en el modelo de usuario original. [#original-pricing]

Para los usuarios de nuestro modelo de usuario original, aquí hay algunos detalles:

<CollapserGroup>
  <Collapser
    id="default-access"
    title="Modelo de usuario original: acceso predeterminado"
  >
    * [Los administradores](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#roles) tienen acceso completo a todas las características de monitoreo sintético de una cuenta.

    * [El usuario](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) tiene acceso a todas las características de monitoreo sintético, excepto [las credenciales seguras](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers) y la capacidad de eliminar el monitor, monitor el tiempo de inactividad y la ubicación privada.

    * [El usuario restringido](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) en cuentas con precios originales basados en productos tiene acceso de solo visualización a todas las características de monitoreo sintético excepto:

      * [Credenciales seguras](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
      * Ubicación privada (aunque pueden consultar una cantidad limitada de datos vía NRQL)

      Esto significa que el usuario con este rol no tiene acceso a todas las capacidades de creación, edición y eliminación de la característica de monitoreo sintético (así como a la capacidad de ver credenciales seguras) hasta que se le asigne un rol con las capacidades apropiadas.
  </Collapser>

  <Collapser
    id="custom-access"
    title="Modelo de usuario original: roles personalizados"
  >
    Para comenzar, siga [los procedimientos estándar para crear una función personalizada](/docs/accounts/accounts/roles-permissions/add-roles-permissions#create/docs/accounts/accounts/roles-permissions/add-roles-permissions#create). Puede agregar capacidades <DNT>**View**</DNT> y <DNT>**Edit**</DNT> a cualquier rol para lo siguiente:

    * [Monitores](/docs/synthetics/new-relic-synthetics/using-monitors)
    * [Monitor de secuencia de comandos](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-synthetics-api-tests)
    * [Ubicación privada](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations)
    * [Monitor el tiempo de inactividad](/docs/synthetics/new-relic-synthetics/using-monitors/monitor-downtimes-disable-monitoring-during-scheduled-maintenance-times)
    * [Credenciales seguras](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
  </Collapser>
</CollapserGroup>
