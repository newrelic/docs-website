Utilice la página <DoNotTranslate>**Resources**</DoNotTranslate> para comprender el impacto en el tiempo de carga de cada recurso en su sitio web. Utilice los gráficos [de tiempo de carga de recursos](#resources-load-time), [tiempo invertido por terceros](#third-parties) y [códigos de respuesta irregulares](#response-codes) para obtener una descripción general de alto nivel, y utilice la [lista de recursos](#resources-list) para identificar recursos de alto impacto, como imágenes o archivos CSS de gran tamaño.

Profundice aún más y [vea una carga de ese recurso](#resource-results), utilizando los encabezados de solicitud y respuesta exactos para identificar un problema de rendimiento.

## Comprender la métrica de recursos [#understanding]

Utilice la página <DoNotTranslate>**Resources**</DoNotTranslate> para comprender el impacto en el rendimiento de cada recurso en su sitio web:

* <DoNotTranslate>**Resources list**</DoNotTranslate>

  : enumera los recursos en el sitio web de su monitor por

  <DoNotTranslate>**average download time**</DoNotTranslate>

  . Utilice esta lista para optimizar el rendimiento de su sitio web reduciendo los tiempos de descarga. También puede ver el tiempo de carga de cualquier recurso en particular buscándolo en el campo

  <DoNotTranslate>**Search all resources**</DoNotTranslate>

  . Seleccione un recurso para [ver la métrica de recursos individuales](#individual).

* <DoNotTranslate>**Load time**</DoNotTranslate>

  : representa gráficamente el tiempo de carga promedio de todos los recursos, desglosado por tipo de recurso (como `jpeg`, `json` o `javascript`). Utilice este gráfico para comprender cómo cambia el tiempo de carga de su sitio web con el tiempo.

* <DoNotTranslate>**Time spent by third parties**</DoNotTranslate>

  : representa gráficamente el tiempo de carga promedio de todos los recursos de terceros, como fuentes web, una línea de tiempo de Twitter integrada o una imagen alojada externamente. Utilice este gráfico para ver si los sitios web de terceros están afectando el rendimiento de su sitio web.

* <DoNotTranslate>**Irregular response codes**</DoNotTranslate>

  : Grafica el número de códigos de respuesta HTTP negativos (también llamados [códigos de estado HTTP](http://en.wikipedia.org/wiki/List_of_HTTP_status_codes)) recibidos por el monitor. Este gráfico incluye todas las respuestas en los rangos 300 (redirección), 400 (error del cliente) y 500 (error del servidor). No cuenta las respuestas en los rangos de 100 o 200. Sintético también recopila una serie de [códigos de respuesta personalizados](/docs/synthetics/new-relic-synthetics/using-monitors/synthetics-response-codes).

Al analizar estas métricas, puede localizar recursos problemáticos y mejorar su rendimiento. Por ejemplo, una imagen de logotipo no optimizada puede provocar ralentizaciones para todos los visitantes de su sitio, mientras que un rastreador que se proporciona únicamente desde un servidor con sede en EE. UU. podría provocar un rendimiento lento para los visitantes que no son de EE. UU.

## Ver recurso detallado métrica [#detailed-metrics]

También puede ver métricas detalladas para cada recurso en su sitio. Para ver métricas detalladas, [busque un recurso específico](#locate-resource) o simplemente seleccione un recurso interesante de la página <DoNotTranslate>**Resources**</DoNotTranslate> . Después de seleccionar un recurso, puede ver la descripción general métrica de ese recurso o [ver una carga de recursos individual](#resource-results):

* <DoNotTranslate>**Max duration**</DoNotTranslate>

  ,

  <DoNotTranslate>**Min duration**</DoNotTranslate>

  ,

  <DoNotTranslate>**Avg duration**</DoNotTranslate>

  : Métrica de tiempo de carga para este recurso durante el [período de tiempo seleccionado](#time-frame).

* <DoNotTranslate>**For 50% of requests**</DoNotTranslate>

  : El tiempo máximo de descarga para el

  <DoNotTranslate>**fastest**</DoNotTranslate>

  50 % de las solicitudes. Esto significa que el 50% de las solicitudes de este recurso se completaron en el tiempo especificado o más rápido.

* <DoNotTranslate>**For 95% of requests**</DoNotTranslate>

  : El tiempo máximo de descarga para el

  <DoNotTranslate>**fastest**</DoNotTranslate>

  95% de las solicitudes. Esto significa que el 95% de las solicitudes de este recurso se completaron en el tiempo especificado o más rápido.

* <DoNotTranslate>**Average load time**</DoNotTranslate>

  : Grafica el tiempo de carga promedio para este recurso durante el [período de tiempo seleccionado](#time-frame). Utilice este gráfico para verificar que las optimizaciones de su sitio web (como disminuir el tamaño de una imagen o minimizar su CSS) estén funcionando.

También puede [ocultar y mostrar elementos del gráfico](#hide-unhide), filtrar por ubicación y ordenar la lista [de resultados de recursos](#resource-results) por <DoNotTranslate>**Download time**</DoNotTranslate> o <DoNotTranslate>**DNS lookup**</DoNotTranslate>. Pase el mouse sobre el nombre del recurso para ver la ruta completa (útil con nombres de recursos muy largos generados dinámicamente).

## Ver cargas de recursos individuales [#resource-results]

Después de ver [las métricas detalladas](#detailed-metrics) de un recurso en particular, puede seleccionar una carga de recursos individual para comprender en profundidad un incidente de bajo rendimiento. Luego puede ver un desglose del tiempo total de conexión para comprender dónde ocurrió el problema y examinar los encabezados de solicitud y respuesta.
