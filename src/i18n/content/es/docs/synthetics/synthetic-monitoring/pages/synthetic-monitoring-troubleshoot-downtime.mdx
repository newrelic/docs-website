Use la página <DNT>**Failures**</DNT> para localizar y solucionar problemas de eventos de alerta de tiempo de inactividad u otros errores. Un evento de alerta de tiempo de inactividad ocurre siempre que un monitor falla al ejecutarse completamente. Por ejemplo, un monitor de ping está &quot;caído&quot; cuando la solicitud GET falla, mientras que un monitor de navegador con scripts está &quot;caído&quot; si alguna parte del script falla al ejecutarse. Después de localizar un tiempo de inactividad de interés, selecciónelo para [ver resultados detallados de ese evento de alerta de tiempo de inactividad](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) y solucionar problemas.

## Ver la página de fallas [#viewing]

Para acceder a las fallas de su monitor: <DNT>**[one.newrelic.com &gt;Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; (seleccione un monitor) &gt; <DNT>**Failures**</DNT>.

## Ver tiempo de inactividad individual [#individual]

Puede seleccionar eventos individuales de alerta de tiempo de inactividad para verlos con más detalle. Dependiendo de la falla específica, un resultado de tiempo de inactividad podría incluir solo el mensaje de error del servidor (como <DNT>**Server replied with &quot;HTTP 500&quot; error**</DNT>) o una [vista de cascada](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) completa o parcial.

Los resultados de tiempo de inactividad incluyen cascadas cuando solo una parte de un monitor se ejecutó correctamente. Por ejemplo, una [redirección 301](http://en.wikipedia.org/wiki/HTTP_301) podría enlazar a una página web con fallas. El navegador ejecuta correctamente la redirección, pero la página web de destino devuelve un error. Utilice el mensaje de error o el gráfico de cascada para solucionar problemas del evento de alerta de tiempo de inactividad.
