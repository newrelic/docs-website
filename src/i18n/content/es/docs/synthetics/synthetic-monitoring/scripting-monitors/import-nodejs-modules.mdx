Además del entorno de script basado en WebDriverJS del monitoreo sintético estándar, New Relic admite muchos módulos populares de Node.js para mejorar su [browser con script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) y las pruebas de API.

## Importar módulos opcionales [#import-procedure]

Para importar un módulo compatible, siga el procedimiento de importación estándar correspondiente. Por ejemplo:

* Para importar <DNT>**crypto-js**</DNT>, llame a:

  ```
  var crypto = require('crypto-js');
  ```

* Para importar <DNT>**chai**</DNT>, llame a:

  ```
  var assert = require('chai').assert;
  ```

## Módulos de terceros compatibles

Las diferentes versiones monitor con script de Sintético admiten diferentes módulos de terceros. Para obtener información sobre los módulos admitidos por diferentes versiones de monitor, consulte la [tabla de comparación de versiones de monitor](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment).

## Módulos principales de Node.js compatibles [#supported-core-modules]

<CollapserGroup>
  <Collapser
    id="NewRuntime"
    title={<>Chrome 100+, Node.js 16.10 o posterior</>}
  >
    Todos [los módulos principales de Node.js](https://nodejs.org/docs/latest-v16.x/api/all.html) están disponibles en Chrome 100, Node.js 16.10 o tiempos de ejecución más recientes.
  </Collapser>

  <Collapser
    id="LegacyRuntime"
    title={<>Tiempos de ejecución legacy</>}
  >
    El tiempo de ejecución de monitoreo sintético legacy de New Relic también admite los siguientes módulos principales opcionales de Node.js:

    * [afirmar](http://nodejs.org/api/assert.html)
    * [cripto](http://nodejs.org/api/crypto.html)
    * [DNS](https://nodejs.org/api/dns.html)
    * [fs](https://nodejs.org/api/fs.html)
    * [neto](https://nodejs.org/api/net.html)
    * [querystring](http://nodejs.org/api/querystring.html)
    * [timers](http://nodejs.org/api/timers.html)
    * [tls](https://nodejs.org/api/tls.html)
    * [url](http://nodejs.org/api/url.html)
    * [útil](http://nodejs.org/api/util.html)
    * [zlib](https://nodejs.org/api/zlib.html)
  </Collapser>
</CollapserGroup>

## Módulos no compatibles

Puede utilizar cualquier módulo de nodo en un [minion privado en contenedores](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms). Para obtener instrucciones sobre cómo configurar módulos personalizados para minion privado en contenedores, consulte [configuración de llamadas por minuto](/docs/synthetics/new-relic-synthetics/private-locations/containerized-private-minion-cpm-configuration#custom-modules).

No se garantiza que los módulos no compatibles funcionen y no están cubiertos por el soporte.
