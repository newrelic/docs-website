Esta guía cubre la configuración y el despliegue del administrador de trabajos de New Relic Synthetics para admitir pruebas de doble stack (IPv4 e IPv6) en su entorno de ubicación privada usando Podman.

## Requisitos previos [#prerequisites]

Antes de comenzar, cerciorar de tener lo siguiente:

* <DNT>**Podman**</DNT>: Podman debe estar instalado y en ejecución en su instancia de host

* <DNT>**slirp4netns**</DNT>: requerido para el soporte de IPv6 en contenedores rootless (versión 1.0.0 o superior, generalmente instalado con Podman)

* <DNT>**Permissions**</DNT>: debe tener acceso adecuado para ejecutar contenedores rootless

* <DNT>**Public images**</DNT>: el soporte de stack dual requiere las siguientes versiones de imagen o posteriores, disponibles en Docker Hub:

  * [Administrador de trabajos `519` o posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-519/): `newrelic/synthetics-job-manager`
  * [Tiempo de ejecución de Ping `1.65.0` o posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.65.0/): `newrelic/synthetics-ping-runtime`
  * [Entorno de ejecución de API de Node `rc1.15` o posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.15/): `newrelic/synthetics-node-api-runtime`
  * [Runtime de navegador Node `rc1.15` o posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.15/): `newrelic/synthetics-node-browser-runtime`

<Steps>
  <Step>
    ## Valida tu infraestructura [#validate-infrastructure]

    Antes de continuar, verifique que su máquina host y su red estén configuradas para redes dual stack.

    Pruebe la conectividad IPv6 ejecutando `ping6 ipv6.google.com` o `ping -6 ipv6.google.com`, o el comando equivalente para su máquina.

    <Callout variant="important">
      Si la prueba falla, asegúrese de que su VPC, subred y grupos de seguridad estén configurados para permitir el tráfico IPv6 antes de continuar con esta instalación. Su entorno debe admitir tanto IPv4 como IPv6 para la operación de doble stack.
    </Callout>
  </Step>

  <Step>
    ## Configurar el entorno de Podman [#setup-podman]

    Configure el entorno de Podman siguiendo la [Guía de dependencias de Podman](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager/#podman-dependencies).
  </Step>

  <Step>
    ## Crear un pod de Podman [#create-pod]

    Cree un pod con la dirección IP de su máquina host:

    ```bash
    podman pod create --network slirp4netns --name SYNTHETICS --add-host=podman.service:IP_ADDRESS
    ```

    Reemplace `IP_ADDRESS` con la dirección IP de su máquina host.
  </Step>

  <Step>
    ## Lanzamiento del gestor de trabajos de Sintéticos [#launch-job-manager]

    <Callout variant="tip">
      Reemplace `<YOUR_PRIVATE_LOCATION_KEY>` con su clave de ubicación privada real de la UI de New Relic.
    </Callout>

    ```bash
    podman run \
      -e PRIVATE_LOCATION_KEY=<YOUR_PRIVATE_LOCATION_KEY> \
      --pod SYNTHETICS \
      -d \
      --restart unless-stopped \
      -e "CONTAINER_ENGINE=PODMAN" \
      -e "PODMAN_API_SERVICE_PORT=8000" \
      -e "PODMAN_POD_NAME=SYNTHETICS" \
      newrelic/synthetics-job-manager
    ```

    El gestor de trabajos de Sintéticos ahora se ejecutará con el soporte de dual stack habilitado, permitiendo a sus monitores probar extremos tanto IPv4 como IPv6.
  </Step>

  <Step>
    ## Verificar la instalación [#verify-installation]

    Para confirmar que su gestor de trabajos de Sintéticos se está ejecutando correctamente:

    1. Vea los logs del administrador de trabajos:

       ```bash
       podman logs -f YOUR_CONTAINER_ID
       ```

    2. En la UI de New Relic, vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Synthetic monitoring &gt; Private locations**</DNT> y verifique que su ubicación privada se muestre como saludable.
  </Step>
</Steps>
