<DNT>**Versions Affected:**</DNT> 3.0.59 y antes

<DNT>**Fixed In:**</DNT> 3.0.60

<DNT>**Vulnerability Identifier:**</DNT> NR22-01

<DNT>**Priority:**</DNT> Alto

## Resumen [#summary]

New Relic lanzó Minion privado en contenedor (llamadas por minuto) versión 3.0.60 para eliminar específicamente las subdependencias en log4j versión 1.2.17.

New Relic ha determinado que la versión 1.2.17 de log4j se incluyó en las subdependencias de nuestro paquete de compilación para Minion privado en contenedores antes de la versión 3.0.60. La versión 1.x de Log4j tiene CVE altos y críticos sobresalientes de [CVE-2021-4104](https://nvd.nist.gov/vuln/detail/CVE-2021-4104) y [CVE-2019-17571](https://nvd.nist.gov/vuln/detail/CVE-2019-17571) y ya no recibe soporte de la Fundación Apache para abordar estos problemas.

## Elementos de acción [#action]

Recomendamos encarecidamente a los clientes que actualicen todos sus minion privados en contenedores a [la versión 3.0.60](/docs/release-notes/synthetics-release-notes/containerized-private-minions-release-notes/containerized-private-minion-cpm-3060) o posterior lo antes posible. Esta versión ha excluido por completo todo uso de log4j versión 1.x de dependencia. Puedes actualizar tus llamadas por minuto a través de la versión 1.0.48 de Helm Charts.

Este paso ayudará a remediar las vulnerabilidades de log4j en su minion privado New Relic Containerized (llamadas por minuto). Para obtener orientación de seguridad adicional sobre log4j en otros productos New Relic, revise los boletines de seguridad de New Relic en nuestra [página de documentación](/docs/security/security-privacy/information-security/security-bulletins/).

<DNT>
  **Customers using log4j directly in their applications should carefully review the [Apache Log4j Security Vulnerabilities page](https://logging.apache.org/log4j/2.x/security.html) for remediation details that should be considered.**
</DNT>

## Preguntas frecuentes [#faq]

<DNT>
  **I've updated to Containerized Private Minion (CPM) version 3.0.58 already, do I need to update to version 3.0.60?**
</DNT>

Sí, New Relic recomienda encarecidamente actualizar en este momento para abordar las vulnerabilidades críticas en las subdependencias de log4j identificadas en el paquete de compilación de minion privado en contenedores. La Fundación Apache ha emitido una recomendación para desaprobar todo uso de log4j versión 1.x debido a que el proyecto no tiene soporte y tiene vulnerabilidades pendientes. llamadas por minuto 3.0.60 y versiones posteriores son las únicas versiones de llamadas por minuto disponibles sin ningún uso de log4j versión 1.x.
