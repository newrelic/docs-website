## Resumen

Una actualización de seguridad para el agente de Java corrige un problema por el cual el agente puede no ofuscar adecuadamente toda la información de consulta SQL cuando `record_sql` está configurado en `off`.

<DNT>**Release date:**</DNT> Mayo 2, 2018

<DNT>**Vulnerability identifier:**</DNT> NR18-09

<DNT>**Priority:**</DNT> Bajo

## Información de vulnerabilidades [#vuln-info]

Las ejecuciones del agente de Java de New Relic explican planes para [una traza de la transacción lenta](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) y una consulta SQL lenta. Cuando un cliente ha configurado `record_sql` en `off`, ejecuta PostgreSQL y ejecuta un plan de explicación, es posible que el agente no oculte adecuadamente toda la información en la consulta.

## Soluciones alternativas

Deshabilite `explain` planes con `transaction_tracer` en la [configuración del agente de Java](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#h2-Transaction-tracer).
