## Información de vulnerabilidades [#vuln-info]

El agente New Relic Node.js utiliza `https-proxy-agent` como una opción para enviar datos al [recolector New Relic ](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#collector)a través de un servidor proxy HTTP o HTTPS. El agente de Node.js utilizó una versión de este módulo que era vulnerable a la exposición de memoria no inicializada y a la denegación de servicio. Esta solución actualiza el módulo a una versión que no es vulnerable.

## Circunstancias mitigantes [#workarounds]

La activación de estas vulnerabilidades de seguridad requiere el control de la configuración de autenticación del proxy del agente.

* [Snyk: `https-proxy-agent`](https://snyk.io/vuln/npm:https-proxy-agent:20180402)
* [Informe HackerOne](https://hackerone.com/reports/319532)

## Soluciones alternativas

New Relic no ha identificado ninguna solución para estas vulnerabilidades.
