## Notas

Esta versión del agente de Python corrige un problema con MetricNormalizationRules, corrige una advertencia de obsolescencia en la detección de la versión del paquete y agrega soporte para los nuevos comandos introducidos en [Redis v5](https://pypi.org/project/redis/).

Instale el agente usando `easy_install/pip/distribute` a través del [Índice de paquetes de Python](https://pypi.python.org/pypi/newrelic) o descárguelo directamente del [sitio de descarga de New Relic](https://download.newrelic.com/python_agent/release).

## Mejoras

* **Agrega soporte para los nuevos comandos introducidos en Redis v5** Agrega soporte completo de instrumentación para [Redis v5](https://pypi.org/project/redis/).

## Corrección de errores

* **Se corrigió un fallo en MetricNormalizationRules** Se corrigió un fallo que se producía al usar MetricNormalizationRules del lado del cliente para ignorar la métrica (establecida a través de NerdGraph y disponible bajo una bandera de característica). Ahora se debe ignorar correctamente la métrica.

* **Corregir DeprecationWarning en la detección de versiones de paquetes para Python 3.10+** Se corrigió una DeprecationWarning emitida por la detección de versiones de paquetes utilizada internamente para Python 3.10+.Cuando esté disponible importlib.metadata, es preferible usarlo para evitar comprobar el atributo **de versión** obsoleta en los módulos. (¡Gracias @renanivo por la contribución!)

## Declaración de apoyo
