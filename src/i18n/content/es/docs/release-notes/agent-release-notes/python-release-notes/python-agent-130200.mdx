## Notas

Esta versión del agente de Python agrega soporte de instrumentación para [Google ADK](https://pypi.org/project/google-adk), agrega el conteo de tokens para múltiples proveedores de IA ([Anthropic](https://pypi.org/project/anthropic), [Google GenAI](https://pypi.org/project/google-genai), [OpenAI](https://pypi.org/project/openai) y [AWS Bedrock](https://pypi.org/project/botocore)), reduce el tamaño del encabezado `tracestate` saliente y soluciona un bloqueo en la instrumentación de [graphql-core](https://pypi.org/project/graphql-core).

Instale el agente usando `easy_install/pip/distribute` a través del [Índice de paquetes de Python](https://pypi.python.org/pypi/newrelic), usando `conda` a través del [Índice de paquetes de Conda-Forge](https://anaconda.org/conda-forge/newrelic), o descárguelo directamente desde el [sitio de descargas de New Relic](https://download.newrelic.com/python_agent/release).

## Características mejoradas

* Se agregó instrumentación para Google ADK

  * El agente de Python ahora instrumenta los flujos de trabajo de agente de IA de [Google ADK](https://pypi.org/project/google-adk) (Agent Development Kit).

  * Los componentes instrumentados incluyen:

    * `LlmAgent` vinculado a los backends de Gemini o VertexAI a través de [`google.genai`](https://pypi.org/project/google-genai)
    * Instancias de `FunctionTool` definidas por el usuario
    * Flujos de trabajo de subagentes, incluidos los agentes secundarios y las instancias de `AgentTool`
    * Flujos de trabajo de múltiples agentes, incluida la clase `Workflow` y las clases obsoletas `LoopAgent`, `ParallelAgent` y `SequentialAgent`

* Se agregó el conteo de tokens para Anthropic, Gemini, OpenAI y Bedrock

  * El agente de Python ahora informa los conteos de tokens para todas las invocaciones de LLM en múltiples proveedores de IA:

    * [SDK de Anthropic](https://pypi.org/project/anthropic)
    * [SDK de Google GenAI](https://pypi.org/project/google-genai)
    * [SDK de OpenAI](https://pypi.org/project/openai)
    * AWS Bedrock a través de [botocore](https://pypi.org/project/botocore) y [aiobotocore](https://pypi.org/project/aiobotocore)

## Errores

* Se solucionó el bloqueo de instrumentación para graphql-core v3.2.10

  * Anteriormente, [graphql-core](https://pypi.org/project/graphql-core) v3.2.10 cambió el nombre del atributo `errors` a `collected_errors` en `ExecutionContext`, lo que provocaba que el agente fallara. Esto se ha solucionado y el agente ahora funciona correctamente con graphql-core v3.2.10 y versiones posteriores.

* Se eliminó `spanId` del encabezado `tracestate` saliente

  * Anteriormente, el agente incluía `spanId` en el encabezado `tracestate` saliente. Debido a que `spanId` ya está presente en el encabezado `traceparent`, esta era una duplicación innecesaria. Se ha eliminado `spanId` de `tracestate` para reducir el tamaño del encabezado saliente.
