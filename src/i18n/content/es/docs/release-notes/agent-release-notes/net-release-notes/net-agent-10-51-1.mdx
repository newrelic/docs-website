### Aviso

* Los clientes que usan `RecordDatastoreSegment()` con un nombre de proveedor personalizado verán que los nombres de las métricas cambian de `Datastore/*/Other/*` a `Datastore/*/<VendorName>/*`. Actualice cualquier dashboard, alerta o consulta que haga referencia a los antiguos nombres de métricas basados en Other. ([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))

### Correcciones

* `RecordDatastoreSegment()` La API usará correctamente el nombre del proveedor proporcionado por quien realiza la llamada en lugar de ignorarlo. ([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))
* Asegurar que todos los segmentos de StackExchange.Redis se creen y se agreguen a la transacción antes de que se recopile ([#3574](https://github.com/newrelic/newrelic-dotnet-agent/issues/3574)) ([4152ea2](https://github.com/newrelic/newrelic-dotnet-agent/commit/4152ea25b23318c68d372447c905c622ac5ac868))

### Actualizando su agente

* Siga los procedimientos estándar para [actualizar el agente .NET](/docs/agents/net-agent/installation-configuration/update-net-agent).
* Si está empleando un agente particularmente antiguo, revise la lista de cambios principales y procedimientos para [actualizar el agente .NET legacy](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net).

Recomendamos actualizar a la última versión del agente tan pronto como esté disponible. Si no puede actualizar a la última versión, actualice su agente a una versión que no tenga más de 90 días de antigüedad. Obtenga más información sobre [cómo mantener a su agente actualizado](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent). Consulte el [documento de política de fin de vida](/docs/apm/agents/net-agent/getting-started/net-agent-eol-policy) del agente .NET de New Relic para obtener información sobre las versiones del agente y las fechas de soporte.
