## Agente PHP de New Relic v12.8.0.37

### Nueva característica

* Se agregó instrumentación de Redis Cluster ([#1212](https://github.com/newrelic/newrelic-php-agent/pull/1212) / [#1206](https://github.com/newrelic/newrelic-php-agent/pull/1206)). Resuelve [#130](https://github.com/newrelic/newrelic-php-agent/issues/130).

### Actualizaciones de seguridad

* Se actualizó golang a 1.26.4 ([#1232](https://github.com/newrelic/newrelic-php-agent/pull/1232))
* Se actualizaron las dependencias desactualizadas ([1241](https://github.com/newrelic/newrelic-php-agent/pull/1241))

### Corrección de errores

* Se corrigió el desbordamiento de búfer de realpath en install-util ([#1221](https://github.com/newrelic/newrelic-php-agent/pull/1221))
* Se corrigió un desbordamiento de stack ([#1222](https://github.com/newrelic/newrelic-php-agent/pull/1222))
* Se corrigió el recuento de instancia/reportes ([#1231](https://github.com/newrelic/newrelic-php-agent/pull/1231))
* Se mitigó un posible desbordamiento de snprintf ([#1223](https://github.com/newrelic/newrelic-php-agent/pull/1223))
* Se eliminó la doble asignación de la cadena request\_uri ([#1236](https://github.com/newrelic/newrelic-php-agent/pull/1236))

### Declaración de apoyo

* El soporte del agente de PHP para las siguientes versiones de bibliotecas y frameworks se descontinuará y New Relic recomienda actualizar a la versión más reciente con soporte del proveedor:

  * El soporte para Slim 2 se descontinuará en todas las versiones del agente PHP después del 31 de julio de 2026.
  * El soporte para Drupal 7 se descontinuará en todas las versiones del agente PHP después del 30 de septiembre de 2026.
  * El soporte para Drupal 9 se descontinuará en todas las versiones del agente PHP después del 30 de septiembre de 2026.
  * El soporte para Laravel 11 se descontinuará en todas las versiones del agente PHP después del 30 de septiembre de 2026
  * El soporte para Joomla 3 se descontinuará en todas las versiones del agente PHP después del 30 de septiembre de 2026

* New Relic recomienda que actualices el agente periódicamente y como mínimo cada 3 meses. Para obtener más información sobre las versiones de agente compatibles y los plazos de EOL, consulte nuestra [política de EOL del agente PHP de New Relic](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).

* Se debe consultar la página [de requisitos y compatibilidad del agente PHP](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/) para obtener la información más reciente sobre las versiones y plataformas de PHP compatibles.

<Callout variant="important">
  **Para la instalación empleando una versión o plataforma de PHP no compatible, se recomienda encarecidamente que deshabilite los mecanismos de actualización automática para los paquetes PHP del agente.** Esto se puede hacer agregando los paquetes del agente PHP a una lista de exclusión para las actualizaciones de paquetes. O puede fijar la versión del paquete PHP del agente a una versión del agente que admita las características antiguas no admitidas. Si no se evitan las actualizaciones, es posible que se instale una versión más nueva del agente y que se elimine el soporte para la función requerida que no tiene soporte. Esto interrumpiría la recopilación de datos de APM.

  Los paquetes del agente PHP que se ven afectados son:

  * newrelic-php5
  * newrelic-php5-common
  * newrelic-daemon
</Callout>
