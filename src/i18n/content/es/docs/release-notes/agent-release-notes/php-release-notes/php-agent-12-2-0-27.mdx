## Agente New Relic PHP v12.2.0.27

### Nueva característica

* Laravel 12 verificado y compatible.
* Laravel Horizon ahora es compatible con Laravel 10.x+ y PHP 8.1+.

### Seguridad

* [#1129](https://github.com/newrelic/newrelic-php-agent/issues/1129) - Actualizar la versión de golang a 1.25.3 - [#1130](https://github.com/newrelic/newrelic-php-agent/pull/1130)

### Corrección de errores

* fix(agent): Se corrigió el manejo de excepciones de trabajos en cola Laravel - [#1122](https://github.com/newrelic/newrelic-php-agent/pull/1122)

### Declaración de apoyo

* En una próxima versión, para garantizar las mejores prácticas y la seguridad, se rotará la clave de firma rpm. Más cerca de esa fecha de lanzamiento, se proporcionarán fechas específicas y acciones necesarias para rpm usuario.

* Como se anunció previamente, el soporte de agente PHP para las siguientes versiones de biblioteca/framework finalizó oficialmente a partir del 30 de septiembre de 2025:

  * WordPress 5.9
  * Drupal 8.x
  * Laravel 6.x, 8.x
  * PHPUnit (todas las versiones)

* El soporte de agente PHP para las siguientes versiones de biblioteca y framework se descontinuará en todas las versiones de agente PHP luego del 28 de febrero de 2026:

  * Para las siguientes versiones de biblioteca y framework, New Relic recomienda actualizar a la última versión compatible:

    * Symfony 4.x
    * Laravel 7.x, 9.x, 10.x
    * Slim 3.x

  * Se interrumpirá el soporte para la siguiente biblioteca y marco de trabajo en todas las versiones:

    * MediaWiki
    * Marco Zend
    * Lumen

* New Relic recomienda que actualices el agente periódicamente y como mínimo cada 3 meses. Para obtener más información sobre las versiones de agente compatibles y los plazos de EOL, consulte nuestra [política de EOL del agente PHP de New Relic](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).

* Se debe consultar la página [de requisitos y compatibilidad del agente PHP](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/) para obtener la información más reciente sobre las versiones y plataformas de PHP compatibles.

<Callout variant="important">
  **Para la instalación empleando una versión o plataforma de PHP no compatible, se recomienda encarecidamente que deshabilite los mecanismos de actualización automática para los paquetes PHP del agente.** Esto se puede hacer agregando los paquetes del agente PHP a una lista de exclusión para las actualizaciones de paquetes. O puede fijar la versión del paquete PHP del agente a una versión del agente que admita las características antiguas no admitidas. Si no se evitan las actualizaciones, es posible que se instale una versión más nueva del agente y que se elimine el soporte para la función requerida que no tiene soporte. Esto interrumpiría la recopilación de datos de APM.

  Los paquetes del agente PHP que se ven afectados son:

  * newrelic-php5
  * newrelic-php5-common
  * newrelic-daemon
</Callout>
