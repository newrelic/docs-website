## Corrección de errores

* Se resolvió un problema de pérdida de memoria en el que los oyentes de traza no se eliminaban correctamente después de que la aplicación cambiaba entre primer plano y segundo plano varias veces.
* Se solucionó el problema con la sincronización de hilos que ocasionalmente provocaba que los datos no se informaran a New Relic.
* Se abordaron problemas relacionados con los reportes de logs durante los ANR para mejorar la estabilidad y el rendimiento.
* Se corrigieron problemas de clases duplicadas relacionados con la biblioteca Curtains para garantizar la compatibilidad y evitar conflictos.
* Se resolvió un problema de excepción de puntero nulo en el método setUserId para mejorar la solidez.
* Se solucionaron problemas con Retrofit Interceptor donde las URL no se informaban correctamente si la solicitud fallaba debido a errores de red en el dispositivo.

## Declaración de apoyo

Recomendamos actualizar el agente al menos cada tres meses. Encuentre políticas y fechas específicas para el soporte de agentes Android en la [Política de EOL de agentes de monitoreo de celulares](/docs/mobile-monitoring/new-relic-mobile/get-started/mobile-agents-eol-policy/#android-eol).
