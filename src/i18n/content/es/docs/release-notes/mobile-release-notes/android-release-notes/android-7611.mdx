## Correcciones

* Se corrigió un problema `NullPointerException` en el proceso de ejemplificación de memoria.
* Se agregó conversión segura durante la inicialización de los tiempos de lanzamiento de la aplicación para evitar la compatibilidad con `ClassCastException` en el SDK de Microsoft Intune MAM.
* Se mejoró la protección de devolución de llamada de okhttp para evitar fallos del agente debidos a excepciones en el código del usuario.
* Se corrigieron los fallos duplicados de los reportes de fallos nativos.

## Declaración de apoyo

Recomendamos actualizar el agente al menos cada tres meses. Encuentre políticas y fechas específicas para el soporte de agentes Android en la [Política de EOL de agentes de monitoreo de celulares](/docs/mobile-monitoring/new-relic-mobile/get-started/mobile-agents-eol-policy/#android-eol).
