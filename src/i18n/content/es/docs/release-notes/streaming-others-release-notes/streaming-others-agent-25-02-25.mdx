## Nuevas funciones y mejoras

* Nuevos tipos de eventos introducidos

  * **VideoAction**: este tipo de evento representa acciones estándar relacionadas con la reproducción y la interacción de video.
  * **VideoErrorAction**: este tipo de evento captura errores que ocurren durante la reproducción de video, proporcionando información detallada y valiosa para la resolución de problemas.
  * **VideoAdAction**: tipo de evento dedicado para rastrear acciones relacionadas con anuncios de video, lo que garantiza un mejor detalle y generación de reportes.
  * **VideoCustomAction**: un tipo de evento flexible para acciones personalizadas e interacción dentro de la aplicación de video.
  * **ConnectedDeviceSystem**: este tipo de evento monitorea eventos a nivel general del sistema.

## Desusos

* Video de Roku
  * El tipo de evento `RokuVideo` quedó obsoleto. El usuario debería pasar a emplear los tipos de eventos más nuevos que ofrecen capacidades de seguimiento y especificidad mejoradas.
* Sistema Roku
  * El tipo de evento `RokuSystem` quedó obsoleto. El usuario debería pasar a emplear los tipos de eventos más nuevos que ofrecen capacidades de seguimiento y especificidad mejoradas.

## Se introdujeron nuevos atributos

* Introdujimos un conjunto de nuevos atributos para mejorar el seguimiento y el análisis de datos. Estos atributos proporcionan contexto y detalles adicionales para los nuevos tipos de eventos. Consulte la [documentación del modelo de datos](https://github.com/newrelic/video-agent-roku/blob/4.0.0/DATAMODEL.md) para obtener más detalles.

## Impacto y acciones requeridas

* **Transición desde RokuVideo**: los usuarios que actualmente rastrean el evento `RokuVideo` deben adaptar sus implementaciones para emplear los tipos de eventos recientemente introducidos. Este cambio proporciona mayor especificidad y alcance en el seguimiento de acciones en las interacciones de video.
* **Implementaciones de actualización**: los integradores deben actualizar sus sistemas para aprovechar los nuevos atributos introducidos con el lanzamiento, que brindan un contexto y detalles más completos para el seguimiento de eventos.
* **Pruebas y validación**: recomendamos probar la implementación de nuevos tipos de atributos y eventos para garantizar la compatibilidad y la transmisión correcta de datos.
