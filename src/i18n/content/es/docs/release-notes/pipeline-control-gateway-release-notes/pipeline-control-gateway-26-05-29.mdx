## Modo de solo configuración (sin Flux)

* Ahora puede instalar y operar el gateway de Control de pipelines sin requerir controladores de Flux en su clúster. En este modo, New Relic continúa administrando la configuración de su pipeline (muestreo, filtrado, transformaciones) automáticamente a través de la UI, mientras que usted administra la infraestructura del gateway (escalamiento, versiones) manualmente mediante Helm.
* Esto reduce la huella de permisos de cluster-admin a solo acceso a ConfigMap a nivel de namespace, lo que lo hace adecuado para entornos con restricciones de seguridad y orientados al cumplimiento.
* Un sidecar config-watcher integrado detecta automáticamente los cambios de configuración enviados desde la UI y reinicia los pods de gateway — no se requieren operadores externos (como Stakater Reloader).
* Para obtener instrucciones de instalación, consulte [Instalar gateway sin Flux](/docs/new-relic-control/pipeline-control/gateway/install-gateway-without-flux/).

## Modo de despliegue de DaemonSet

* Se agregó soporte para desplegar el gateway como un DaemonSet de Kubernetes (un pod por nodo) en lugar de un despliegue con HPA. Esto es útil para entornos que requieren procesamiento de telemetría a nivel de nodo o una asignación de recursos consistente por nodo.
* Habilite el modo DaemonSet configurando `daemonset.enabled: true` en sus valores de Helm. El modo DaemonSet también admite el sidecar config-watcher y `customConfigMap` para instalaciones de solo configuración.
* Los modos DaemonSet y despliegue son mutuamente excluyentes — el chart usa guardas condicionales para asegurar que solo un modo esté activo a la vez.
