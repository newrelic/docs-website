## Soporte para múltiples flotas

* Ahora puede crear y administrar múltiples flotas de gateways dentro de una sola organización. Esto permite a las unidades de negocio o entornos separados (por ejemplo, prod vs. de prueba) mantener reglas independientes de muestreo, filtrado y transformación.
* Navegue fácilmente entre diferentes configuraciones usando el nuevo desplegable del selector de flota en la UI de Control de pipelines.

### Esquema YAML actualizado

* Se migró a una configuración simplificada que elimina el anidamiento específico de señales (por ejemplo, `logs:`, `spans:`) en favor de una matriz de reglas directa, lo que hace que la configuración sea más plana y fácil de leer.
* Para los procesadores de filtros, el nuevo esquema admite un campo `context` explícito. Esto permite una selección más precisa de subtipos de datos, como **puntos de datos de métricas** y **eventos de span**.

### UX de despliegue mejorada

* La nueva página de despliegue ahora completa automáticamente los valores predeterminados para el nombre y la descripción del despliegue, lo que reduce significativamente los pasos manuales necesarios para implementar cambios.
* Se agregó una vista previa de diferencias de configuración en paralelo para los procesadores de transformación. Esto le permite comparar su nueva lógica YAML con la versión actual antes de finalizar un despliegue.
