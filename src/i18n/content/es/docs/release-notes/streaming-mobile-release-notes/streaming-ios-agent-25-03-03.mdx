## Nuevas características y mejoras

* **Nuevos tipos de eventos introducidos**

  * **VideoAction**: este tipo de evento representa acciones estándar relacionadas con la reproducción y la interacción de video.
  * **VideoErrorAction**: este tipo de evento captura errores que ocurren durante la reproducción de video, proporcionando información detallada y valiosa para la resolución de problemas.
  * **VideoAdAction**: tipo de evento dedicado para rastrear acciones relacionadas con anuncios de video, lo que garantiza un mejor detalle y generación de reportes.
  * **VideoCustomAction**: un tipo de evento flexible para acciones personalizadas e interacción dentro de la aplicación de video.

## Desusos

* **MobileVideo**
  * El tipo de evento `MobileVideo` quedó obsoleto. El usuario debería pasar a emplear los tipos de eventos más nuevos que ofrecen capacidades de seguimiento y especificidad mejoradas.

## Se introdujeron nuevos atributos

* Introdujimos un conjunto de nuevos atributos para mejorar el seguimiento y el análisis de datos. Estos atributos proporcionan contexto y detalles adicionales para los nuevos tipos de eventos [. Consulte la documentación del modelo de datos](https://github.com/newrelic/video-agent-ios/blob/v3.0.0/DATAMODEL.md) para obtener más detalles. Tipos de evento • VideoAction: evento activado por una interacción de video general, como iniciar, pausar o buscar. • VideoAdAction: evento relacionado con la reproducción de anuncios, como iniciar, completar u omitir un anuncio. • VideoErrorAction: evento que se activa cuando se detectan errores durante la reproducción de un video o un anuncio. • VideoCustomAction: evento personalizado definido para capturar acciones o interacciones específicas más allá de los tipos de eventos predeterminados.

## Impacto y acciones requeridas

* **Transición desde MobileVideo**: los usuarios que actualmente rastrean el evento `MobileVideo` deben adaptar sus implementaciones para emplear los tipos de eventos recientemente introducidos. Este cambio proporciona mayor especificidad y alcance en el seguimiento de acciones en las interacciones de video.
* **Implementaciones de actualización**: los integradores deben actualizar sus sistemas para aprovechar los nuevos atributos introducidos con el lanzamiento, que brindan un contexto y detalles más completos para el seguimiento de eventos.
* **Pruebas y validación**: recomendamos probar la implementación de nuevos tipos de atributos y eventos para garantizar la compatibilidad y la transmisión correcta de datos.
