Se ha lanzado una nueva versión del agente. Siga los procedimientos estándar para [actualizar el agente de infraestructura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recomienda que actualice el agente regularmente y, como mínimo, cada 3 meses. A partir de este lanzamiento, la versión compatible más antigua es el [agente de infraestructura 1.65.3](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1653/).

## Cambió

* Se corrigió una condición de pánico causada por una operación de doble cierre. [2270](https://github.com/newrelic/infrastructure-agent/pull/2270)

* Se corrigió el comportamiento de vinculación del servidor local para que el valor predeterminado sea estrictamente localhost. [2271](https://github.com/newrelic/infrastructure-agent/pull/2271)

* Actualizaciones de dependencias:

  * Se actualizó `newrelic/nri-flex` a `v1.18.8`. [#2278](https://github.com/newrelic/infrastructure-agent/pull/2278)
  * Se actualizó `newrelic/nri-winservices` a `v1.5.3`. [#2279](https://github.com/newrelic/infrastructure-agent/pull/2279)
  * Se actualizó `newrelic/nri-prometheus` a `v2.29.4`. [#2276](https://github.com/newrelic/infrastructure-agent/pull/2276)
  * Se actualizó `newrelic/nri-docker` a `v2.8.1`. [#2277](https://github.com/newrelic/infrastructure-agent/pull/2277)
  * Se actualizó la dependencia principal del agente a `v1.26.5`. [#2275](https://github.com/newrelic/infrastructure-agent/pull/2275)
  * Se actualizó `newrelic-fluent-bit-output` a `v3.7.0` (Linux) para resolver las vulnerabilidades de seguridad CVE-2026-39820, CVE-2026-42499, CVE-2026-33814, CVE-2026-42501, CVE-2026-33811 y CVE-2026-42504. [#2280](https://github.com/newrelic/infrastructure-agent/pull/2278)
