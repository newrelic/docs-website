Se ha lanzado una nueva versión del agente. Siga los procedimientos estándar para [actualizar el agente de infraestructura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recomienda que actualice el agente regularmente y, como mínimo, cada 3 meses. A partir de este lanzamiento, la versión compatible más antigua es el [agente de Infrastructure 1.63.0](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1630/).

### Agregado

* feat:Soporte para cuenta de servicio con privilegios mínimos de NewRelic Infrastructure Agent en Windows ([#2152](https://github.com/newrelic/infrastructure-agent/pull/2152))

  Esta versión introduce soporte integral para ejecutar el agente de infraestructura bajo cuentas de servicio con privilegios mínimos en Windows:

  * La actualización conserva la cuenta de servicio: Durante la actualización, el instalador ahora detecta y conserva la cuenta de inicio de sesión de servicio existente (por ejemplo, un dominio personalizado o una cuenta gMSA) en lugar de restablecerla a LocalSystem. Los permisos sobre el directorio de datos de la aplicación se vuelven a otorgar automáticamente a la cuenta conservada.

  * La instalación nueva utiliza `LocalSystem` por defecto: Las nuevas instalaciones continúan registrando el servicio bajo `LocalSystem`. Use `set-service-account.ps1` (nuevo script complementario) después de la instalación para cambiar a una cuenta con privilegios mínimos. Admite cuentas locales, cuentas de dominio estándar y cuentas de servicio administradas de grupo (gMSA). También admite las credenciales almacenadas en el Administrador de credenciales de Windows.

  * Seguridad de rollback: Se escribe un archivo marcador antes de la creación del servicio para que la acción de rollback de MSI solo elimine los servicios creados por la ejecución actual — no eliminará un servicio preexistente durante una actualización fallida.

  * Logging de depuración: La instalación ahora escribe un log de depuración con marca de tiempo (`newrelic_installer_debug.log`), lo que facilita la resolución de problemas.

  * Impacto: No hay cambios disruptivos para las instalaciones existentes que utilizan la cuenta SYSTEM predeterminada. Los nuevos parámetros son totalmente opcionales.
