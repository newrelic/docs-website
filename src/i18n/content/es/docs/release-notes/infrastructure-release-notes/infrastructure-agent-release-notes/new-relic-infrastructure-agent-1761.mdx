Se lanzó una nueva versión del agente. Siga los procedimientos estándar para [actualizar el agente de infraestructura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recomienda que actualices el agente periódicamente y como mínimo cada 3 meses. A partir de esta versión, la versión compatible más antigua es [agente de infraestructura 1.58.1](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1581/).

## Qué ha cambiado

#### Errores

* Se solucionó un problema en el que `set-service-account` fallaba para los usuarios locales en [#2247](https://github.com/newrelic/infrastructure-agent/pull/2247).

#### Seguridad

* Se actualizó containerd a la versión 1.7.32 para solucionar la vulnerabilidad CVE-2026-46680 en [#2250](https://github.com/newrelic/infrastructure-agent/pull/2250).
* Se actualizó nri-flex a la versión 1.18.4 para solucionar la vulnerabilidad CVE-2026-45570 en [#2252](https://github.com/newrelic/infrastructure-agent/pull/2252).

**Registro de cambios completo**: [https://github.com/newrelic/infrastructure-agent/compare/1.76.0...1.76.1](https://github.com/newrelic/infrastructure-agent/compare/1.76.0...1.76.1)
