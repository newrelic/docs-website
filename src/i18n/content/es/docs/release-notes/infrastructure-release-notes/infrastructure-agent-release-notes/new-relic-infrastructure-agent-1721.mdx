Se lanzó una nueva versión del agente. Siga los procedimientos estándar para [actualizar el agente de infraestructura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recomienda que actualices el agente periódicamente y como mínimo cada 3 meses. A partir de esta versión, la versión compatible más antigua es [agente de infraestructura 1.58.1](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1581/).

## Agregado

* Se agregó la opción de configuración `logging_http_client_timeout` para controlar el tiempo de espera del cliente HTTP (en segundos) para el plugin de salida fluent-bit de New Relic en [#2167](https://github.com/newrelic/infrastructure-agent/pull/2167)
* Se agregó el esquema de carga para RHEL10 en [#2171](https://github.com/newrelic/infrastructure-agent/pull/2171)

## Cambió

* Establecer la bandera `win_network_interface_v2` como verdadera (true) de forma predeterminada en [#2173](https://github.com/newrelic/infrastructure-agent/pull/2173)
* Se actualizó la versión mínima para las pruebas de paquetes (pkg-tests) en [#2172](https://github.com/newrelic/infrastructure-agent/pull/2172)

## Seguridad

* Se abordó la vulnerabilidad crítica - CVE-2025-15467 y la vulnerabilidad alta - CVE-2025-69419
