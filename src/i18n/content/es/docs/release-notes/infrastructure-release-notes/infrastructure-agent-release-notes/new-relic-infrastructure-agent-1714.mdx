Se lanzó una nueva versión del agente. Siga los procedimientos estándar para [actualizar el agente de infraestructura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recomienda que actualices el agente periódicamente y como mínimo cada 3 meses. A partir de esta versión, la versión compatible más antigua es [agente de infraestructura 1.58.1](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1581/).

## Fijado

* Solucionado `transmitBytesPerSecond` que mostraba cero durante el tráfico alto en Windows (NR-502872) en [#2166](https://github.com/newrelic/infrastructure-agent/pull/2166)

  <Callout variant="important">
    Para habilitar esta solución, debe establecer la opción de configuración [`win_network_interface_v2`](/docs/infrastructure/infrastructure-agent/configuration/infrastructure-agent-configuration-settings#win-network-interface-v2) en `true`. Esta opción utiliza contadores de 64 bits en lugar de contadores de 32 bits para las métricas de red en Windows.
  </Callout>
