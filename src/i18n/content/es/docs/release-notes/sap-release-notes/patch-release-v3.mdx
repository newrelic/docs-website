## Actualizaciones en este parche

SP12 es una colección de Notas de New Relic que abordan correcciones de errores y nuevas funciones:

### NR 1344

El nuevo parámetro `AIF_TABLES` para `NR_SAP_AIF_ERROR` permite recuperar mensajes de error de tablas distintas a `/AIF/STD_IDX_TBL`.

### NR 12-02

Soluciona problemas y agrega nuevas funcionalidades para proveedores de datos basados en API:

* Se corrigió el volcado de código en `/NEWR/API_DATA_COLLECTOR` cuando el destino de la API no es accesible
* Se corrigió una falla de recursos al extraer datos de CALM Landscape
* Se corrigió una falla de recursos al extraer el BTP Message Processing Log
* Se agregó validación para el ID de servicio (LMSID) configurado como CALL ID para todos los proveedores de datos bajo el grupo de monitoreo CALM
* Se agregó una verificación de validez del certificado antes de llamar a la API de recopilación de datos.

### NR 12-03

Implementa el retraso de recolección y una ventana fija de recolección de datos para el gestor de recolección y pruebas por ID de llamada en el registro.

### NR 12-05

Implementa las siguientes mejoras:

* Soporte de paginación para proveedores de datos en el grupo de monitoreo de CALM
* Nuevo parámetro de programa para permitir que el programa de verificación de RFC verifique un único destino RFC
* Configuración para forzar la prueba del proveedor de datos solo para un ID de llamada

Soluciona los siguientes problemas:

* ID de llamada en blanco en la salida del gestor de recolección
* Problema de validación de credenciales para `NR_CF_AUDIT_EVENTS`

### NR 12-06

El nuevo proveedor de datos `NR_SAP_ABAP_WS_COMPLETE` recopila eventos de servicios web ABAP.

### Correcciones de errores adicionales

Además de las notas anteriores, SP12 también soluciona los siguientes problemas:

* Se corrigió el problema de marca de tiempo con los módulos de funciones del proveedor de datos de recuperación de tablas `/NEWR/RFC_EVENTS_COLLECTOR` y `/NEWR/RFC_EVENTS_COLLECTOR2`
* Se agregó el ID de transacción a los eventos de IDoc para facilitar la vinculación con los registros de rastreo

## Actualizaciones manuales de configuración

### Agregar nuevo parámetro a NR\_SAP\_AIF\_ERROR

Para configurar el nuevo parámetro `AIF_TABLES`:

1. Aplique el transporte incluido al Central Monitoring System.
2. Vaya a la transacción `/n/NEWR/REGISTRY`.
3. Seleccione el proveedor de datos `NR_SAP_AIF_ERROR` y haga clic en **Configure**.
4. Agregue el parámetro `AIF_TABLES` e ingrese el nombre de la tabla especificado en `/AIF/T_IDX_TBL` para la combinación configurada de espacio de nombres/interfaz/versión.

<Callout variant="tip">
  Si `AIF_TABLES` se deja en blanco o no se especifica, la tabla de índices usa por defecto la tabla de índices única estándar `/AIF/STD_IDX_TBL`.
</Callout>

### Configurar el retraso de recopilación y la ventana de recopilación fija

Se han agregado dos nuevos campos de configuración al Registro de proveedores de datos para abordar las brechas de datos causadas por los límites de recolección flotantes y la latencia de la base de datos.

**Nuevos campos en el registro de proveedores de datos:**

* **DATA\_COLLECTION\_DELAY:** Especifica un retraso intencional en minutos para restar de las marcas de tiempo de recopilación. Este búfer garantiza que los datos tardíos de las actualizaciones de la base de datos se capturen antes de que se cierre la ventana de recolección.
* **FIXED\_SCHEDULES:** Indicador para habilitar períodos de recolección fijos. Cuando se establece en `X`, se eliminan los segundos de las marcas de tiempo de recolección para crear ventanas de recolección consistentes y predecibles.

**Cómo funciona:**

**Horarios fijos** (`FIXED_SCHEDULES = 'X'`):

* Las marcas de tiempo de recolección se normalizan eliminando los segundos.
* Ejemplo: 12:12:23 se convierte en 12:12:00.
* Crea límites de recolección consistentes y alineados en todas las colecciones.

**Retraso en la recolección de datos** (`DATA_COLLECTION_DELAY > 0`):

* Resta el retraso especificado en minutos de las marcas de tiempo de inicio y fin de la recopilación de datos.
* Desplaza la ventana de recolección hacia atrás para capturar actualizaciones de la base de datos perdidas.
* Ejemplo: un retraso de 2 minutos cambia 10:00:00 - 10:05:00 a 09:58:00 - 10:03:00.

## Nuevos proveedores de datos para sistemas de negocio ABAP

### NR\_SAP\_ABAP\_WS\_COMPLETE

Procesa encabezados de mensajes de servicios web y registros de datos (etiquetas) en paralelo a través de sistemas monitoreados sin límites de registros.

**Configuración:**

* **Tipo:** Evento
* **Comunicador:** N/A
* **Función de proveedor de datos:** `/NEWR/WS_MESSAGES_ARFC`
* **Proveedor de datos RFC habilitado:** Sí
* **Función de finalizador:** `/NEWR/WS_EVENTS_FINISHER`
* **Grupo de monitoreo:** `SAP_APPL`

### Descripción general funcional

Este proveedor de datos viene preactivado. Parámetros obligatorios:

* **`ADAPTER_TYPE`**

  * **Propósito:** Especificar el tipo de adaptador que se seleccionará del encabezado del mensaje de servicio web ABAP. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Campos de tabla separados por `;`, pueden ser varias líneas

* **`DESTINATION`**

  * **Propósito:** Especificar destino RFC
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Destino RFC

* **`DPC_COLL_TIME`**

  * **Propósito:** Determinar el intervalo de tiempo
  * **Signo:** Solo inclusivo
  * **Opción:** BT
  * **Valores:** Marca de tiempo de inicio y marca de tiempo de finalización generadas por el sistema

* **`IB_IF_NAME`**

  * **Propósito:** Nombre de la interfaz receptora que se seleccionará del encabezado del mensaje del servicio web ABAP. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Nombres de interfaces de receptor separados por `;`, pueden ser varias líneas

* **`IB_PARTY`**

  * **Propósito:** Parte receptora a seleccionar del encabezado del mensaje de servicio web ABAP. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Parte receptora separada por `;`, puede ser de varias líneas

* **`JOIN_CONDITION`**

  * **Propósito:** Especificar los registros de unión para la tabla `/NEWR/JOIN_COND` para su posterior procesamiento
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Muestra títulos separados por `;`, puede ser de varias líneas

* **`NR_ACCID`**

  * **Propósito:** Especificar el ID de cuenta de New Relic
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** ID de cuenta de New Relic

* **`OB_IF_NAME`**

  * **Propósito:** Nombre de la interfaz del remitente que se seleccionará del encabezado del mensaje de servicio web ABAP. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Nombres de interfaces de remitente separados por `;`, pueden ser varias líneas

* **`OB_PARTY`**

  * **Propósito:** Parte emisora a seleccionar del encabezado del mensaje de servicio web ABAP. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Partes remitentes separadas por `;`, pueden ser varias líneas

* **`SETTING_SWITCH`**

  * **Propósito:** Especificar las opciones utilizadas por el procesamiento distribuido

  * **Signo:** Solo inclusivo

  * **Opción:** EQ

  * **Valores:** Múltiples opciones separadas por `;` para controlar el proceso distribuido:

    * **HEADONLY=**

      * `SPACE` (predeterminado): Proporciona tanto datos de encabezado como de TAG (si se especifican nombres de TAG en el parámetro `TAG_LIST`).
      * `X`: Solo proporciona datos de encabezado incluso si se especifican nombres de TAG en el parámetro `TAG_LIST`.

    * **FIRSTONLY=**

      * `X` (predeterminado): El proveedor de datos extrae datos exclusivamente de la primera aparición de cada etiqueta listada en el parámetro `TAG_LIST`.
      * `SPACE`: El proveedor de datos extrae datos de todas las ocurrencias de cada TAG listado en el parámetro `TAG_LIST`.

    * **FREE%=nn:** Especifique el porcentaje de los procesos de trabajo disponibles en el sistema de monitoreo para determinar el límite deseado de procesos de trabajo libres:

      * Si el número de Procesos de trabajo libres en el sistema de monitoreo actual es igual o mayor que este Límite deseado de procesos de trabajo libres, continúe el procesamiento de distribución.

      * De lo contrario, espere 1 segundo y repita la verificación anterior hasta que:

        * El procesamiento de distribución puede continuar, o
        * El tiempo de espera total supera el Tiempo máximo de espera especificado (consulte `MWAIT#=nn`).

      * El valor predeterminado para esta opción es 30(%).

    * **MINFT#=nn:** Especifique el límite mínimo de procesos de trabajo libres:

      * Si los Procesos de trabajo libres deseados contabilizados por el sistema de monitoreo caen por debajo de este umbral mínimo, el sistema se pausará por 1 segundo antes de intentar un procesamiento adicional.
      * El valor predeterminado para esta opción es 3.

    * **MWAIT#=nn:** Especifique el tiempo máximo de espera (en segundos).

      * Si el tiempo de espera total (consulte `FREE%=nn`) supera el máximo especificado, el proceso de distribución se cancelará para conservar los recursos del sistema.
      * El valor predeterminado para esta opción es 60.

    * **MREC#=nnnnn:** Especifique el límite de número de los registros de **encabezado de mensaje de servicio web** recuperados cada vez que se llama al **módulo de funciones** `/SDF/IMA_DC_WS_ABAP`.
      * El valor predeterminado para esta opción es 10000.

    * **JSEND#=nnnn:** Especifique el límite de registros de atributos de resultados enviados al servidor de New Relic cada vez.
      * El valor predeterminado para esta opción es 5000.

    * **USER=user:** Especifique el nombre de usuario RFC para cada sistema monitoreado para obtener el número total de procesos de trabajo utilizados actualmente por ese sistema. El `user` es el ID de usuario de SAP.

    * **DPWP#=nn:** Especifique el **Límite máximo de procesos de trabajo**.

      * Durante el procesamiento distribuido: Si el número de Procesos de trabajo libres en el sistema de monitoreo actual es menor o igual a este límite máximo, continúe el procesamiento distribuido.
      * De lo contrario, espere 1 segundo y luego regrese para verificar nuevamente.
      * Continúe este ciclo hasta que el procesamiento distribuido pueda continuar o el tiempo de espera total exceda el Tiempo de espera máximo especificado (`MWAIT#=nn`).
      * Nota: Esta verificación solo se ejecuta cuando el parámetro USER se especifica con valores no iniciales.
      * El valor predeterminado para esta opción es 8.

* **`STATE_GROUP`**

  * **Propósito:** Grupo de estado de procesamiento a seleccionar del encabezado del mensaje de servicio web ABAP. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Grupos de estados de procesamiento separados por `;`, pueden ser de varias líneas

* **`SYS_ID`**

  * **Propósito:** Determinar el ID del sistema
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** ID del sistema

* **`TAG_LIST`**

  * **Propósito:** Especifique los nombres de las etiquetas que se seleccionarán de los datos del mensaje de servicio web ABAP y se mostrarán como salida en los datos del mensaje. Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Campos de tabla separados por `;`, pueden ser varias líneas

* **`USERNAME`**

  * **Propósito:** Nombres de usuario que crearon el mensaje a seleccionar del encabezado del mensaje de servicio web ABAP (`CREATED_BY`). Opción de filtro del módulo de funciones `/SDF/IMA_DC_WS_ABAP`.
  * **Signo:** Solo inclusivo
  * **Opción:** EQ
  * **Valores:** Nombres de usuario separados por `;`, pueden ser múltiples líneas
