### Nuevos documentos

* Se agregó exhaustiva [documentación de la integración de OpenTelemetry de Elasticsearch](/docs/opentelemetry/integrations/elasticsearch/elasticsearch-otel-integration-overview/) que cubre la instalación para entornos independientes y de Kubernetes mediante el OTel Collector de New Relic y Prometheus.
* Se agregó nueva [documentación de integración de OTel Kafka para entornos autoadministrados de Kubernetes](/docs/opentelemetry/integrations/kafka/kubernetes-self-managed/) junto con documentación ampliada para el monitoreo de Kafka autohospedado con Prometheus.

### Cambios importantes

* Se actualizó la documentación de permisos de Inteligencia para arquitectura de servicios (SAI) para reflejar que Catálogos y Mapas ya no requieren roles adicionales:

  * [Catálogo de aplicaciones](/docs/service-architecture-intelligence/catalogs/applications-catalog/) — se aclaró que no se necesitan roles adicionales para acceder al catálogo de aplicaciones.
  * [Catálogo de respositorio](/docs/service-architecture-intelligence/catalogs/repository-catalog/) — se aclaró que no se necesitan roles adicionales para acceder al catálogo de respositorio.
  * [Mapas avanzados](/docs/service-architecture-intelligence/maps/advanced-maps/) — guía de permisos actualizada y detalles de configuración ampliados.

* Se actualizaron los [requisitos de compatibilidad y permisos de AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/compatibility-requirement-lambda-monitoring/) con detalles ampliados sobre los permisos de IAM requeridos para el monitoreo de Lambda, incluidos los permisos adicionales necesarios para la instrumentación y la recopilación de datos.

* Se actualizaron [los documentos de instrumentación de Lambda y SQS del agente de .NET](/docs/apm/agents/net-agent/getting-started/net-agent-approaches-lambda/) para corregir las limitaciones del rastreo distribuido y el comportamiento de la instrumentación de Lambda en .NET.

* Se agregó un [video tutorial a la documentación de seguimiento de cambios de NerdGraph](/docs/change-tracking/config/nerdgraph/) que explica cómo configurar eventos de seguimiento de cambios.

* Se agregó Ruby 3.4 como entorno de ejecución compatible a la [documentación de instrumentación en capas de AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/introduction-lambda/).

* Se actualizó la [documentación del conector de AWS de New Relic Lens](/docs/new-relic-lens/set-up-connectors/) para documentar el prefijo `nrlens-` requerido para la nomenclatura de conectores.

* Se actualizó la [UI y las tareas de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/) para reemplazar los nombres de dominio de ejemplo por valores de marcador de posición conformes.

### Cambios menores

* Se actualizaron los [requisitos de compatibilidad del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/) con las versiones compatibles verificadas más recientes para varios paquetes instrumentados, incluidos Microsoft.Azure.Cosmos, EnyimMemcachedCore y AWSSDK.DynamoDBv2.
* Se actualizó la página de [requisitos de compatibilidad del monitoreo de IA](/docs/ai-monitoring/compatibility-requirements-ai-monitoring/) para dirigir a los lectores a la [página de compatibilidad del agente de Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/) para ver la lista completa de bibliotecas de monitoreo de IA compatibles.
* Se actualizó la [documentación de Inteligencia para costos en la nube de GCP](/docs/cci/get-started/gcp-cci/) para agregar un enlace de navegación a las páginas de Configuración general e Integración para facilitar el acceso después de la configuración.
* Se actualizó la [página de cumplimiento de FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp/) para eliminar New Relic eAPM de la lista de servicios no autorizados por FedRAMP.
* Mejora menor en el documento [Migrar del sondeo de API a CWMS](/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/migrate-from-api-polling-to-cwms/).

### Notas de la versión

Mantener actualizado con nuestros últimos lanzamientos:

* [Agente Node.js v14.1.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-1-0/):

  * Se agregó soporte para la instrumentación nativa de OTEL de Next.js a través del agente híbrido.

  * API `instrumentLoadedModule` obsoleta.
