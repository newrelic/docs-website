### Nuevos documentos

* [Consejos para reducir la sobrecarga del agente PHP](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips/) enumera los ajustes de configuración que puede cambiar para reducir la sobrecarga del agente en su sistema.

### Cambios importantes

* Se reorganizó y reescribió [Cree su propia integración: New Relic Flex](/docs/infrastructure/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) para mayor claridad y exhaustividad.
* Se agregó la sección [Futuros de Scala](/docs/apm/agents/java-agent/frameworks/scala-installation-java/#futures) en [Instrumentación de Scala](/docs/apm/agents/java-agent/frameworks/scala-installation-java).
* Se actualizaron tres temas de Gestión de vulnerabilidades, incluidos [Introducción a la gestión de vulnerabilidades ](/docs/vulnerability-management/overview), [Comprender la priorización de vulnerabilidades ](/docs/vulnerability-management/understanding-prioritization)y [Gestionar vulnerabilidades como equipo de seguridad ](/docs/vulnerability-management/security-workflow).
* Se agregó información sobre cómo usar la mutación [`cancelAsyncQuery`](/docs/apis/nerdgraph/examples/async-queries-nrql-tutorial/#example-cancel-query).

### Cambios menores

* Enlaces actualizados a las imágenes de Python y Ruby de New Relic compatibles con su función Lambda.
* Se señaló la naturaleza costosa de las consultas de pérdida de señal independientes en [Crear condiciones de alerta NRQL](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions).
* Se actualizaron las [opciones de la CLI de New Relic](/docs/new-relic-solutions/build-nr-ui/newrelic-cli/#nr-cli-commands-options).
* Se refinó el planteamiento del problema en [consejos para reducir la sobrecarga del agente PHP](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips) para aclarar la sobrecarga de rendimiento.
* Se modificó la descripción de los [límites de trazas](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works/#trace-limits) para indicar que ahora puede ver estos límites en la interfaz de límites.
* Se actualizaron las descripciones de los atributos `description` y `incidentId` en [atributos de evento`NrAiIncident` ](/docs/alerts/create-alert/condition-details/alert-event-attributes/#NrAiIncident-attributes).
* Se actualizaron las [Métricas a nivel de código](/docs/codestream/observability/code-level-metrics) para reflejar la nueva extensión de Ruby requerida.
* Se actualizó la documentación de monitoreo de Kubernetes para indicar que New Relic ahora admite Kubernetes 1.31 en lugar de 1.26.
* Se agregó una entrada para la nueva instrumentación de `AWSSDK.DynamoDBv2` en la tabla de [mensajería](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#messaging-core).
* Se actualizaron los [requisitos de compatibilidad](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) para el agente de Node.js.
* Se aclaró cuándo no usar la función `numeric()` en [Conversión de tipos](/docs/nrql/nrql-syntax-clauses-functions/#type-conversion)
* Se agregó información sobre un nuevo límite en el registro de consultas de alertas en
