### Nuevos documentos

* La [guía de migración de .NET OpenTracing a capas de AWS Lambda](/docs/apm/agents/net-agent/getting-started/opentracing-to-layers-migration-guide) le ayuda a migrar de .NET OpenTracing a la capa Lambda del agente APM de .NET.
* Use la [instalación manual de XCFramework](/docs/mobile-monitoring/new-relic-mobile-ios/installation/xcframework-installation) cuando desee instalar el agente iOS de New Relic manualmente.
* El boletín de seguridad [NR24-02 - OpenSSH in New Relic Salesforce Exporter](/docs/security/new-relic-security/security-bulletins/security-bulletin-nr24-02) describe las recomendaciones y acciones necesarias si está utilizando New Relic Salesforce Exporter.

### Cambios importantes

* Se finalizaron las actualizaciones y mejoras a nuestra ramificación [instalar el agente .NET](/install/dotnet/) para facilitar la búsqueda de la información que buscas y configurar el monitoreo de tu aplicación.
* En la [referencia de NRQL](/docs/nrql/nrql-syntax-clauses-functions/#func-histogram), se agregaron argumentos adicionales y valores predeterminados, así como actualizaciones a los [tiempos de respuesta de histogramas](histogram-response-times-defaults) para facilitar la escritura de consultas relacionadas con estas funciones de agregación.
* En nuestra interfaz de usuario, hemos reemplazado `Add data` con `Integrations & Agents`. Hemos revisado nuestro sitio de documentación para reflejar esto.
* Se aclararon las diferencias entre cancelar una cuenta y eliminar una organización en [bajar de categoría o cancelar New Relic](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-account) y [Tutorial de NerdGraph: Ver, crear, renombrar o cancelar cuentas](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph).

### Cambios menores

* En [gráficos y dashboards: visualización de sus datos](/docs/nrql/get-started/charts-and-dashboards-with-nrql/#performance-improvement-for-dashboard-and-charts), se agregaron consejos sobre cómo mejorar el rendimiento de sus dashboards y gráficos.
* En la [configuración del agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#process-container-decoration), se agregó una sección que describe cómo el agente decora las muestras de procesos con información del contenedor cuando `process_container_decoration` está habilitado.
* Información simplificada de versiones compatibles en [Agente .NET: compatibilidad y requisitos para .NET Core](/docs/apm/agents/net-agent/configuration/net-agent-configuration/).
* Se agregó una nota en varios documentos de configuración de APM para Python cuando una opción de configuración se deshabilita si el modo de alta seguridad está habilitado.
* En [gestión y respuesta a problemas y eventos de alerta](/docs/alerts/alert-event-management/Issues-and-alert-event-management-and-response/#search-filters), se agregó una nota aclarando cuándo se crean los eventos de alerta de advertencia y críticos.
* Se actualizó el diagrama en la parte superior de [instrumentar su función Lambda no contenerizada](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/instrument-your-own/).
* Se eliminó &quot;Usuarios de RightScale y New Relic&quot;. RightScale fue una asociación que dejó de existir hace mucho tiempo.
* Tenga en cuenta que ServiceNow ITSM es una integración de notificaciones de alertas heredada. Use la [integración certificada de ServiceNow](/docs/alerts/get-notified/notification-integrations/#servicenow-certified) en su lugar.
* Combinamos y reubicamos toda nuestra documentación de la [integración de Kubernetes versión 2](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/k8s-version2/overview/) en una subcategoría de nuestro sitio. La versión 3 de Kubernetes es la más reciente y le recomendamos usarla en su lugar.
* No puede [rotar nuestras claves de API REST](/docs/apis/intro-apis/new-relic-api-keys/#rotate-rest-api-key), porque esta clave antigua está obsoleta. Si está usando una clave de API REST, elimínela y use [una de nuestras otras claves](/docs/apis/intro-apis/new-relic-api-keys/#rotate-keys) en su lugar.
