### Nuevos documentos

* Se agregaron guías de instalación de Sintéticos con soporte dual stack para entornos de [Docker](/docs/synthetics/synthetic-monitoring/private-locations/install-synthetics-dual-stack-docker/), [Kubernetes](/docs/synthetics/synthetic-monitoring/private-locations/install-synthetics-dual-stack-kubernetes/) y [Podman](/docs/synthetics/synthetic-monitoring/private-locations/install-synthetics-dual-stack-podman/) para admitir pruebas tanto de IPv4 como de IPv6.
* Se agregó la [Guía de transición del entorno de ejecución de ubicaciones privadas de Sintéticos](/docs/synthetics/synthetic-monitoring/private-locations/runtime-transition-guide/) para migrar monitores a Node.js 22 y Chrome 146, con estrategias de migración in situ y en paralelo, consultas NRQL para el monitoreo de la transición y correcciones de monitores asistidas por IA.
* Se agregó la guía de comparación [Monitoreo de Azure vs. sondeo de Azure](/docs/infrastructure/amazon-integrations/get-started/azure-monitor-vs-azure-polling/) para ayudar a los usuarios a elegir el método de integración de Azure adecuado para recopilar métricas de Azure en New Relic.

### Cambios importantes

* Se actualizó de manera integral la documentación de [Inteligencia para costos en la nube (CCI)](/docs/cci/overview/) con flujos de trabajo de la UI revisados, gestión de presupuestos actualizada, nueva página de descripción general de costos, información valiosa de costos de Kubernetes, consultas de NRQL para CCI y guías de configuración para AWS, Azure y GCP.

### Cambios menores

* Se actualizaron los [requisitos de compatibilidad del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/) para incluir el soporte del framework Hangfire.
* Se actualizó la [descripción general de eBPF](/docs/ebpf/overview/) con información de cumplimiento de FedRAMP.
* Se actualizó la [integración de NGINX OpenTelemetry Kubernetes](/docs/infrastructure/other-infrastructure-integrations/nginx/nginx-otel-kubernetes/) con instrucciones de instalación de Helm para la instrumentación de NGINX OTel.
* Se actualizó la [configuración del agente Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration/) con ajustes de configuración del Agente híbrido para el soporte de la API de OpenTelemetry.
* Se actualizó [Límites de API de métrica y atributos restringidos](/docs/data-apis/ingest-apis/metric-api/metric-api-limits-restricted-attributes/) al eliminar la documentación desactualizada del límite de puntos de datos por minuto (DPM) en las páginas de API de métrica, Prometheus y optimización de datos.
