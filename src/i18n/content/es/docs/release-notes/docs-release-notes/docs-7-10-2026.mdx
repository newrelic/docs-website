### Nuevos documentos

* Se agregó la [integración de IBM MQ OpenTelemetry](/docs/opentelemetry/integrations/ibm-mq/overview) para el monitoreo de IBM MQ con OpenTelemetry, que cubre las configuraciones de host, Kubernetes, métricas y resolución de problemas.
* Se agregó la [integración de Redis OpenTelemetry](/docs/opentelemetry/integrations/redis/overview) para el monitoreo de Redis con OpenTelemetry, que cubre las configuraciones de Docker, host y Kubernetes.
* Se agregó la [integración de OpenTelemetry de HAProxy](/docs/opentelemetry/integrations/haproxy/overview) para el monitoreo de HAProxy con OpenTelemetry, que cubre las configuraciones autohospedadas y de Kubernetes.
* Se agregó el [monitoreo de logs de flujo de Azure VNet](/docs/network-performance-monitoring/setup-performance-monitoring/cloud-flow-logs/azure-vnet-flow-log-monitoring) para reenviar y consultar datos de logs de flujo de la red virtual de Azure.

### Cambios importantes

* [Destinos entre cuentas](/docs/apis/nerdgraph/examples/cross-account-destinations/overview) ya está disponible de forma general.

### Cambios menores

* Se agregó la documentación del parámetro `blocks` a la [acción de automatización del flujo de trabajo](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) de `sendSlack` para enviar mensajes de Slack Block Kit, junto con una tabla de tipos de bloques compatibles.
* Se actualizaron los [requisitos de compatibilidad del agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) con el último informe de compatibilidad automatizado.
* Se actualizaron los ejemplos de [instalación de Docker en Linux](/install/dotnet/?deployment=linux&docker=yesDocker) y de [instalación de Docker en Windows](/install/dotnet/?deployment=windowsInstall&docker=yesDocker) para el agente de .NET con instrucciones para instalar una versión específica en lugar de la más reciente.
* Se corrigió el intervalo de actualización de membresía de la [carga de trabajo inteligente](/docs/new-relic-solutions/new-relic-one/workloads/create-intelligent-workload) de 5 minutos a 10 minutos, y se indicó que está sujeto a cambios durante la vista previa pública.
* Se aclaró en el [rango de tiempo de la consulta NRQL](/docs/nrql/using-nrql/query-time-range) que la inclusión de puntos de datos de métrica sin procesar depende de la hora de inicio de un punto de datos, no de su hora de finalización.
* Se actualizó la [configuración de estado de la carga de trabajo](/docs/new-relic-solutions/new-relic-one/workloads/workload-status-configuration) para aclarar qué métodos de estado están disponibles para las cargas de trabajo estándar frente a las inteligentes.
* Se agregó una guía para eliminar un usuario de un dominio de autenticación o una organización a [múltiples registros de usuario](/docs/accounts/accounts-billing/account-setup/multiple-user-records).
* Se actualizaron los [requisitos de compatibilidad del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) para aclarar el soporte y las limitaciones de IBM DB2 tanto para .NET Core como para .NET framework.
* Se agregó compatibilidad con Ubuntu 26.04 (Resolute) al [reenvío de logs a través del agente de infraestructura](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent), a través de Fluent Bit 5.0.6.

### EOL

* [Fin de vida útil para el monitoreo de Drupal 7 y 9 en el agente PHP](/eol/2026/07/eol-07-09-26-drupal-7-9-monitoring): el soporte finaliza el 15 de octubre de 2026.
* [Fin de vida útil para el soporte del tiempo de ejecución de Node.js 20](/eol/2026/07/eol-07-09-26-nodejs-20-runtime): los agentes de New Relic ya no admiten el tiempo de ejecución de Node.js 20, que llegó al final de su vida útil el 30 de abril de 2026.

### Notas de la versión

* Mantener actualizado con nuestros últimos lanzamientos:

  * [Monitoreo de host de NRDOT v1.19.0](/docs/release-notes/nrdot-release-notes/nrdot-host-2026-07-06):

    * Se verificó que el caso de uso del host funcione con `nrdot-collector:1.19.0`.

  * [Fluent Bit 5.0.8 / plug-in de salida New Relic Fluent Bit 3.7.0](/docs/release-notes/fluentbit-release-notes/fluentbit-26-06-30):

    * Se actualizó a Go 1.26.4 para correcciones de CVE.
    * El agente de infraestructura ahora recomienda los paquetes de Fluent Bit 5.0.8 (Linux) y 5.0.7 (Windows).

  * [Agente iOS v7.7.4](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-774):

    * Los dispositivos simuladores ahora se reportan como `simulator-(deviceType)`.
    * Se eliminó la detección de KMP que causaba inestabilidad en 7.7.3.
    * Se solucionaron los bloqueos en el registro de solicitudes web asíncronas y una condición de carrera de configuración de recolección.

  * [Agente de Python v13.2.0](/docs/release-notes/agent-release-notes/python-release-notes/python-agent-130200):

    * Se agregó instrumentación para los flujos de trabajo del agente de Google ADK.
    * Se agregó el recuento de tokens para Anthropic, Google GenAI, OpenAI y AWS Bedrock.
    * Se solucionó un bloqueo de instrumentación con `graphql-core` v3.2.10 y se redujo el tamaño del encabezado `tracestate` saliente.

  * [Agente de Node.js v14.2.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-2-0) y [v14.2.1](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-2-1):

    * Se agregó soporte para Node.js 26 y una opción de configuración `profiling.source_mapping.enabled`.
    * Se solucionó una fuga de memoria causada por controladores `uncaughtException` duplicados registrados por la instrumentación de `winston`.

  * [Integración de Kubernetes v4.3.7](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-4-3-7):

    * Lanzado como parte de las versiones de chart newrelic-infrastructure-4.3.7 y nri-bundle-8.0.2.

  * [Agente de Browser v1.318.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.318.0):

    * Se agregó un bloque de configuración de inicio `web_sockets` para habilitar el monitoreo de WebSocket (deshabilitado de forma predeterminada).
    * Se agregó detección interna para FCP, INP, CLS y LCP cuando el agente es generado por un micro frontend.

  * [Aplicación móvil para Android v5.43.3](/docs/release-notes/mobile-apps-release-notes/new-relic-android-release-notes/new-relic-android-5433):

    * Correcciones de errores y mejoras de la UI.

  * [Agente de React Native v1.8.7](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-187) y [agente de Flutter v1.2.9](/docs/release-notes/mobile-release-notes/flutter-release-notes/flutter-agent-129):

    * Se actualizó el agente nativo de iOS a la v7.7.4.

  * Tiempo de ejecución sintético — [Job Manager v584](/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-584), [tiempo de ejecución de la API de Node rc1.28](/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.28), [tiempo de ejecución de Browser de Node rc1.28](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.28) y [rc1.29](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.29), y [tiempo de ejecución de Ping v1.75.0](/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.75.0):

    * Se aplicaron parches a los CVE del sistema operativo subyacente y de dependencia en todas las imágenes de Sintéticos.
    * Se transmitieron cargas HAR grandes (más de 100 MB) directamente al disco en lugar de almacenarlas en la memoria, para el tiempo de ejecución del navegador Node.
