### Cambios importantes

* Se reestructuró la [documentación de Cloud Cost Intelligence (CCI)](/docs/cci/overview) para mejorar la organización y el descubrimiento de las funciones, incluidos los presupuestos, las recomendaciones y las integraciones.
* Se actualizó el [catálogo de acciones de automatización de flujos de trabajo](/docs/workflow-automation/setup-and-configuration/actions-catalog/actions-catalog) con revisiones exhaustivas de la documentación de las acciones y se agregaron dos nuevas acciones para la ingesta de datos de New Relic y consultas de NRDB.
* Se actualizaron las [ofertas de soporte técnico global](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings) con una estructura de documentación de soporte de OpenTelemetry mejorada y se agregó una sección de soporte de Cuenta y Facturación.
* Se actualizó [Trae tu propio caché](/docs/distributed-tracing/infinite-tracing-on-premise/bring-your-own-cache) con ejemplos de configuración simplificados y una estructura de documentación mejorada.
* Se actualizó la [integración de Databricks](/docs/infrastructure/other-infrastructure-integrations/databricks-integration) para reflejar los cambios de la última versión.

### Novedades

* [Cambios de nomenclatura - &quot;Incidentes&quot; ahora son &quot;Eventos de alerta&quot;](/whats-new/2026/01/whats-new-01-23-incidents-to-alert-events) - A partir del 16 de febrero de 2026, New Relic cambiará el nombre de &quot;Incidentes&quot; a &quot;Eventos de alerta&quot; en toda la interfaz de usuario de la plataforma para mejorar la claridad y alinearse con la nomenclatura estándar de la industria.
* [Mejora la seguridad de la cuenta con la autenticación multifactor](/whats-new/2026/01/whats-new-01-28-2026-mfa) - Protege tus dominios de autenticación al requerir autenticación multifactor (MFA) para todos los usuarios.
