### Nuevos documentos

* Se agregaron documentos de [administración a nivel de servicio](/docs/service-level-management/intro-slm) para dos nuevas características de GA: Windows de mantenimiento para programar el tiempo de inactividad planificado que excluye los eventos erróneos de los cálculos de SLI, y cumplimiento de facetas para desglosar el cumplimiento de SLO y los presupuestos de errores por atributo.

### Cambios importantes

* Se agregó la guía de migración de la v14 a [Actualizar el agente de Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent), que incluye cambios importantes, instrumentación eliminada y versiones mínimas compatibles actualizadas.
* Se reestructuró el [plug-in de Apollo Server para Node.js](/docs/apm/agents/nodejs-agent/extend-your-instrumentation/apollo-server-plugin-nodejs) para reflejar la migración de la instrumentación de `@apollo/server` de un plug-in independiente a la instrumentación tradicional en la v14.
* Se actualizaron [Instalar el agente eBPF de New Relic para Kubernetes](/docs/ebpf/k8s-installation) y [Instalar el agente eBPF de New Relic para hosts de Linux](/docs/ebpf/linux-installation) con el contenido de la versión v1.4.1, que incluye nuevas opciones de configuración y pasos de instalación.

### Cambios menores

* Se reestructuró la documentación sobre el uso compartido de dashboard y gráficos para tener páginas dedicadas al uso compartido público de [dashboards](/docs/query-your-data/explore-query-data/dashboards/share-dashboards-publicly/) y [gráficos](/docs/query-your-data/explore-query-data/dashboards/share-charts-publicly/).
* Se agregó la herramienta `list_entity_performance_risk_groups` a la [referencia de herramientas de MCP](/docs/agentic-ai/mcp/tool-reference) para obtener grupos de riesgo de rendimiento de la Bandeja de entrada de riesgos de rendimiento.
* Se actualizó la referencia de Mobile SDK [setMaxEventPoolSize](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-pool-size) con un aviso importante para brindar contexto adicional.
* Se actualizaron los [requisitos de compatibilidad del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) con las últimas versiones compatibles del framework.
* Se corrigieron los nombres de los dashboard de nginx en la documentación de la [integración de NGINX](/docs/opentelemetry/integrations/nginx/find-and-query-your-data) y la [integración de NGINX Plus](/docs/opentelemetry/integrations/nginx-plus/nginx-plus-otel) para reflejar los nombres actuales de los dashboard.
* Se actualizaron [Precios y facturación de New Relic One](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) e [Introducción a la Inteligencia para arquitectura de servicios](/docs/service-architecture-intelligence/getting-started) para aclarar que las capacidades de SAI están incluidas sin costo adicional para todos los usuarios de la plataforma completa.
* Se actualizaron [Nuevo runtime](/docs/synthetics/synthetic-monitoring/using-monitors/new-runtime), [Administrar runtimes de monitores](/docs/synthetics/synthetic-monitoring/using-monitors/manage-monitor-runtimes) y [Resolución de problemas de actualización de runtime](/docs/synthetics/synthetic-monitoring/troubleshooting/runtime-upgrade-troubleshooting) para centralizar las referencias de la versión de Chrome a &quot;Más reciente&quot;.
* Se actualizó la [Guía de transición del entorno de ejecución del monitoreo sintético](/docs/synthetics/synthetic-monitoring/private-locations/runtime-transition-guide) con fechas de EOL específicas para las transiciones del entorno de ejecución.
* Se corrigió un problema de formato en [Sintaxis, cláusulas y funciones de NRQL](/docs/nrql/nrql-syntax-clauses-functions) al eliminar una referencia interna al nombre de la aplicación.
* Se corrigió la sangría del bloque de código en [Actualizar el agente de Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent).

### Publicaciones de novedades

* Se agregó [Actualización de SLM: Maintenance Windows y el cumplimiento de facetas ahora están disponibles de forma general](/whats-new/2026/05/whats-new-05-18-slm-maintenance-windows-facet-compliance) para anunciar la disponibilidad general (GA) de las ventanas de mantenimiento programadas y el seguimiento del cumplimiento de SLO basado en facetas en la administración a nivel de servicio.

### Notas de la versión

* Mantener actualizado con nuestros últimos lanzamientos:

  * [Agente Node.js v14.0.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-0-0):

    * Se eliminó el soporte para Node.js 20 y `fastify` 3.x (cambio radical).
    * Se eliminó el soporte para el seguimiento multiaplicación (CAT) y las políticas de licencia, aplicación y seguridad (LASP).
    * Se migró la instrumentación de `@apollo/server` del paquete `@newrelic/apollo-server-plugin` a la instrumentación tradicional.
    * Se eliminó la instrumentación para `koa-route` y `koa-router`.
    * Versiones mínimas compatibles actualizadas para `@nestjs/cli`, `@nestjs/core`, `next`, `bluebird`, `mysql2` y `cassandra-driver`.
    * Se actualizó el valor predeterminado de `config.distributed_tracing.exclude_newrelic_header` a `true`.

  * [Runtime de la API de Node rc1.21](/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.21):

    * Se corrigieron múltiples vulnerabilidades de seguridad, incluidas CVE-2024-50004, CVE-2024-58096, CVE-2025-37926, CVE-2025-38201 y otras.

  * [Runtime de Node browser rc1.21](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.21):

    * Se actualizó Chrome de 148.0.7778.167-1 a 148.0.7778.178-1.
    * Se corrigieron múltiples vulnerabilidades de seguridad, incluidas CVE-2026-3832, CVE-2026-3833, CVE-2026-5260 y otras.

  * [Integración de Kubernetes v4.1.1](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-4-1-1):

    * Lanzado como parte del chart `newrelic-infrastructure-4.1.1` y `nri-bundle-7.0.12`.

  * [Agente de infraestructura v1.75.0](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1750):

    * Se agregó soporte para el extremo de OCI IMDS v2.
    * Se actualizaron las dependencias `nri-docker`, `nri-winservices`, `nri-prometheus` y `nri-flex`.
    * Se actualizó la versión de Go a 1.26.3.

  * [Plug-in de salida de Logstash v2.0.0](/docs/release-notes/logstash-release-notes/logstash-26-03-05):

    * Se cambió al cliente HTTP Manticore con reutilización de conexiones para mejorar el rendimiento.
    * Se corrigió la compresión gzip para resolver los problemas de deserialización de la carga.
    * Se agregaron parámetros de tiempo de espera de conexión y de socket configurables por el usuario.
    * Se actualizaron las pruebas de integración para validar la entrega de logs a New Relic.
