### Nuevos documentos

* Se agregó la documentación de la [Bandeja de entrada de riesgos de rendimiento](/docs/errors-inbox/performance-risks/introduction) para la nueva vista previa pública que detecta consultas N+1, SQL lento y otros antipatrones de codificación.
* Se agregaron [acciones de incidente de ServiceNow](/docs/workflow-automation/setup-and-configure/actions-catalog/servicenow/servicenow-incident) para la automatización del flujo de trabajo, que abarcan las operaciones de crear, obtener, listar, resolver y actualizar.

### Cambios importantes

* Se actualizó la [documentación del procesador de gateway de Control de pipelines](/docs/new-relic-control/pipeline-control/gateway/yaml-overview) para reflejar el esquema YAML v2.2.0 simplificado con una matriz `rules` plana y un campo `context` explícito.
* Se actualizaron la [configuración del agente de Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration) y las referencias de la API de traza para la versión v13.0.0, incluidas las nuevas configuraciones `traces.include` y `traces.exclude` de OpenTelemetry.
* Se actualizó la [UI de actualización del tiempo de ejecución sintético](/docs/synthetics/synthetic-monitoring/using-monitors/runtime-upgrade-ui) con un diseño de pestañas renovado para los flujos de trabajo <DNT>**Fix manually**</DNT> y <DNT>**Fix with AI**</DNT>.

### Cambios menores

* Se agregaron notas sobre la inyección de dependencias a la [API del agente de .NET](/docs/apm/agents/net-agent/net-agent-api/net-agent-api) que explican cómo usar `IAgent` de forma segura con las API de transacción y span del agente.
* Se agregó la opción de autenticación `Managed Identity` al [reenvío de logs de Azure Event Hub](/docs/logs/forward-logs/azure-log-forwarding) para plantillas de ARM v3.2.0+.
* Se agregó la configuración del atributo `aws.cluster.arn` a las guías de monitoreo de OpenTelemetry de [ECS EC2](/docs/opentelemetry/integrations/ecs-monitoring/ecs-ec2) y [ECS Fargate](/docs/opentelemetry/integrations/ecs-monitoring/ecs-fargate).
* Se agregó compatibilidad con la biblioteca Logging, el logger de evento de Rails y Semantic logger a los [framework compatibles con el agente de Ruby](/docs/apm/agents/ruby-agent/getting-started/ruby-agent-requirements-supported-frameworks).
* Se agregaron las entradas `active_record_use_table_name` y Dalli 5.0 a la [configuración del agente Ruby](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration).
* Se actualizaron los [ejemplos de la API de NerdGraph de Sintéticos](/docs/apis/nerdgraph/examples/synthetics-api/scripted-api-monitor) para recomendar `runtimeTypeVersion: "22.20.0"` y señalar la actualización forzada del 18 de noviembre de 2026 para los tiempos de ejecución de Node.js más antiguos.
* Se agregó una sección plegable de namespace de APM a [Administrar tus datos](/docs/data-apis/manage-data/manage-data-retention) para aclarar qué tipos de eventos de APM quedan fuera del namespace de rastreo distribuido.
* Se agregó un bucle `for` con un ejemplo de expresión `jq` al [Esquema de definición de automatización de flujo de trabajo](/docs/workflow-automation/workflow-automation-apis/definition-schema).
* Se actualizó la [acción de notificación de New Relic](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) con nuevos formatos de resultados de consulta `JSON`, `CSV`, `PSV` y `TSV`, y un ejemplo de selector `success`.
* Se actualizó el [Resumen de boletines de seguridad](/docs/security/security-privacy/information-security/security-bulletins) con la nueva entrada [NR26-05](/docs/security/new-relic-security/security-bulletins/security-bulletin-nr26-05) para `CVE-2026-31431` en el monitoreo sintético.
* Se actualizaron los [requisitos del agente de infraestructura](/docs/infrastructure/infrastructure-agent/requirements-infrastructure-agent) con opciones de inicio de sesión del servicio de Windows, que incluyen cuentas de Local System, Active Directory, Microsoft Entra ID y gMSA.
* Se actualizó el [reporte de compatibilidad del agente de Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) con las últimas versiones compatibles de los paquetes.

### Publicaciones de novedades

* Se agregó [Bandeja de entrada de riesgos de rendimiento ahora está en versión preliminar pública](/whats-new/2026/05/whats-new-05-14-performance-risks-inbox) para anunciar el acceso gratuito a la versión preliminar pública para detectar consultas N+1, SQL lento, consultas de la base de datos excesivas y secuenciales, requests HTTP lentas y cargas HTTP grandes.

### Notas de la versión

* Mantener actualizado con nuestros últimos lanzamientos:

  * [Agente de Python v13.0.0](/docs/release-notes/agent-release-notes/python-release-notes/python-agent-130000):

    * Se eliminó el soporte para las funciones Lambda de Python en el archivo de configuración (seguridad).
    * Se excluyó el encabezado de rastreo distribuido `newrelic` por defecto; solo se envían `traceparent` y `tracestate` de W3C.
    * Se agregó instrumentación para el streaming de Anthropic y el streaming de respuestas de Gemini.
    * Se agregó el atributo `http.statusCode` a los eventos de transacción; se discontinuó `response.status`.
    * Se agregaron las configuraciones `opentelemetry.traces.include` y `opentelemetry.traces.exclude` para las trazas del agente híbrido.

  * [Agente Node.js v13.20.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-13-20-0):

    * Se agregó instrumentación para Google ADK `BaseAgent.runAsync` y `FunctionTool.runAsync`.
    * Se actualizó la instrumentación de `@azure/functions` al tipo de suscriptor.
    * Se actualizó la forma en que se ocultan las claves de licencia en los logs.
    * Se corrigieron los problemas de instrumentación de `connect` y AWS SDK v3 con versiones de dependencias más recientes.

  * [Agente Ruby v10.5.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-10-5-0):

    * Se agregó soporte para Dalli 5.0 y se corrigió la instrumentación del metaprotocolo para `pipelined_get` y `read_multi_req`.
    * Se agregó la opción de configuración `active_record_use_table_name` para usar nombres de tablas en lugar de nombres de clases en las métricas.
    * Claves de licencia parcialmente ocultas en los logs del agente (primeros 10 caracteres visibles).
    * Se solucionó la incompatibilidad de la instrumentación de Semantic logger con `rails_semantic_logger`.

  * [Agente PHP v12.7.0.36](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-12-7-0-36):

    * Se actualizó la versión de Go a 1.26.3 (seguridad).

    * Se garantizó que los spans sin nombre no se registren para el rastreo infinito.
    * Se permitió la vinculación del parámetro `mysqli` en execute.
