### Nuevo documento

* [AWS SQS cola de mensajes instrumentación](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) admite encabezados rastreo distribuidos para mensajes SQS a partir de la versión 2.1.0, con orientación sobre instrumentación personalizada para operaciones de recepción de mensajes.
* [alerta Intelligence Tool](/docs/agentic-ai/tools/alert-intelligence-tool) mejora la capacidad de evaluar el estado y el rendimiento de las aplicaciones a través de la integración agente con nuestros socios.
* [La transmisión gestionada de Confluent Cloud para Apache Kafka](/docs/infrastructure/other-infrastructure-integrations/confluent-cloud-integration) proporciona orientación sobre cómo monitorear el clúster y los temas de Kafka, y configurar alertas para Confluent Cloud.
* [El resumenNew Relic AI para Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/nrai-summary) resume y mapea los datos del trazado de eventos de Session Replay, lo que permite una mejor comprensión de los errores y éxitos en los recorridos del usuario del browser.
* [La inteligencia de respuesta con New Relic AI](/docs/alerts/alert-event-management/response-intelligence-ai) reduce el tiempo de resolución de problemas y eventos de alerta al aprovechar New Relic AI con inteligencia de respuesta.
* [Los logs de Browser](/docs/browser/browser-monitoring/browser-pro-features/browser-logs/overview) capturan logs de la consola de browser en tiempo real para mejorar la observabilidad, simplificar la depuración y mejorar la experiencia del usuario.
* [Introducción a Transacción 360](/docs/apm/transactions/workload-performance-monitoring/introdution) ofrece un enfoque revolucionario para la aplicación de monitoreo y resolución de problemas al capturar y correlacionar automáticamente todas las entidades y datos relacionados con problemas o preguntas específicas.
* [AWS Logs in Context](/docs/logs/logs-context/aws-logs-in-context) proporciona una solución de administración de logs completa e integrada que mejora la eficiencia de la resolución de problemas y la observabilidad del sistema dentro de los entornos AWS .
* Gestiona y reinstala tu instrumentación actual con [agente Control](/docs/new-relic-control/agent-control/reinstrumentation).
* [Agentic Control](/docs/new-relic-control/agent-control/overview) es un supervisor de agentes liviano que gestiona otras distribuciones New Relic y New Relic de agentes OpenTelemetry comunicar con el control de flota para configurar, actualizar y monitorear de forma remota el estado de la instrumentación a nivel de infraestructura.
* [El controlPipeline ](/docs/new-relic-control/pipeline-control/overview)le permite gestionar el flujo de datos desde la entidad monitoreada a New Relic.
* La integración [deDocker contenedor](/docs/infrastructure/host-integrations/host-integrations-list/docker-container-monitoring-integration) monitorea y reporta métrica desde Docker contenedor a New Relic.
* Funciones [de monitoreo de respuestaLambda ](/docs/serverless-function-monitoring/aws-lambda-monitoring/response-stream)en la aplicación Node.js para ganar tiempo real información valiosa sobre el rendimiento y comportamiento de cada invocación.

### Cambios importantes

* Se agregaron [Amazon CloudWatch Metric Streams](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams), [ElastiCache (Memcached)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached) y [Amazon MQ (ActiveMQ)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) como [relaciones de entidades compatibles](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships) con OpenTelemetry.

* Se agregaron Amazon CloudWatch Metric Streams con [Amazon Managed Kafka (MSK)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-managed-kafka-msk) para generar entidad MSK en New Relic. Estas entidades están etiquetadas con `aws.clusterName` o `aws.kafka.ClusterName`, `aws.awsRegion` o `aws.region`, y `aws.topic` o `aws.kafka.Topic`.

* Se agregaron instrucciones adicionales [de resolución de problemasSolr ](/docs/apm/agents/java-agent/troubleshooting/java-solr-troubleshooting)para los cambios métricos en la versión 8.10.

* New Relic ha descontinuado el uso de los siguientes rangos de IP:

  * 3.145.244.128/25
  * 3.77.79.0/25
  * 3.27.118.128/25
  * 20.51.136.0/25
  * 4.197.217.128/25
  * 18.246.82.0/25
  * 158.177.65.64/29
  * 159.122.103.184/29
  * 161.156.125.32/28

  A partir del 1 de mayo de 2025, estos rangos podrán ser reasignados por el proveedor cloud a otros clientes para fines fuera de nuestro control. Actualice su configuración [de red](/docs/new-relic-solutions/get-started/networks) según corresponda.

* Se agregó un documento de resolución de problemas para [recuperar la clave de licencia de New Relic ](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/retrieve-license)para problemas al usar AWS Secrets Manager con New Relic.

* Se agregó un documento de resolución de problemas para [la función Lambda que no se muestra como instrumentada](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-not-instrumented-lambda-function).

* Se agregó un documento de resolución de problemas para problemas [al agregar múltiples cuentas AWS con sus respectivas regiones](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-unable-to-add-multiple-aws-accounts/).

* Se agregó un documento de resolución de problemas para problemas [al usar AWS Secrets Manager con New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-using-aws-secrets-manager/).

* Se agregó la integración de New Relic Agentic a [FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp).

* Se agregó un nuevo documento sobre [la integración de EC2 Image Builder de New Relic](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-ec2-image-builder-integration).

* Se agregaron [Dashboards](/docs/codestream/observability/dashboards) a CodeStream.

* Se agregó [OpenTelemetry Collector para la monitorización Kubernetes ](/docs/opentelemetry/get-started/collector-k8s-monitoring/opentelemetry-collector-k8s-intro)y [distribucionesNew Relic del OpenTelemetry Collector](/docs/opentelemetry/nrdot/nrdot-collector).
