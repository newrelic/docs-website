### Nuevos documentos

* Se anunció el 16 de junio de 2025 como la fecha de fin de vida útil del [métrica y evento Explorer](/eol/2025/05/metrics-and-event-explorer/).
* Se agregaron [instrucciones para crear interacción con clientes](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-custom-Interaction/) a través del agente Android .

### Cambios importantes

* Se agregó la variable de entorno `NEW_RELIC_CONFIG_FILENAME` a la [configuración del agente Node.js.](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#filename)
* Se agregó una configuración a la Ruby [configuración de rastreo distribuido del agente](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/#distributed_tracing-sampler-remote_parent_sampled) que gestiona el comportamiento de la ejemplificación de transacciones con un padre remoto cuando el indicador traza se configura en traceparent.
* Se agregó Google Gen AI SDK como biblioteca compatible con el agente Python en la página [Compatibilidad y requisitos para el monitoreo de IA](/docs/ai-monitoring/compatibility-requirements-ai-monitoring/) .
* Se agregó una característica en la [página de condición de alerta](/docs/alerts/create-alert/create-alert-condition/alert-conditions/#disable-health-status-reporting) que le permite evitar cambios automáticos en la entidad estado de salud cuando ocurre una notificación de alerta.
* Se agregaron mejoras en la personalización de la vista de la tabla de registro en la página [Usar la UI de log](/docs/logs/ui-data/use-logs-ui/#customizable-table-views) .
* Se agregó información sobre [el uso de NRDiag](/docs/new-relic-control/agent-control/troubleshooting/#nrdiag) para solucionar problemas comunes al instalar u operar Agente Control.
* Se extendió la fecha de fin de vida útil al 21 de mayo de 2025 para eliminar los rangos [de IP más antiguos del monitor Sintético minion público](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips/#dynamic-ip-addresses) de la lista de rangos de IP JSON.
* Se actualizó el documento [Soporte del agente para ARM64/Graviton y x86\_64](/docs/apm/agents/manage-apm-agents/configuration/support-arm-graviton-x86-64/) para reflejar que el agente y la integración en el host admiten ARM64 exclusivamente en entornos Linux.
* Se eliminaron Dashboards de la lista de activos que se eliminan cuando [el usuario elimina un elemento](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/#delete-users).
* Se agregó información sobre nuevas capacidades de usuario para la [UIde la aplicación No responde (ANR)](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding/#view-anr-new-relic) en la plataforma New Relic .
* [Pautas de elegibilidad](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/eligibility-guidelines-new-relic-nonprofit-program/#exceptions) actualizadas para el programa Observability for Good de New Relic.

### Cambios menores

* Se agregaron referencias de funciones alternativas NerdGraph o NRQL para los ejemplos de API REST v2.
* Se actualizaron la [compatibilidad y los requisitos para el agente Node.js.](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/)
* Se actualizaron la [compatibilidad y los requisitos para el agente Python](/docs/apm/agents/python-agent/getting-started/compatibility-requirements-python-agent/).
* Actualizar la [compatibilidad y los requisitos para el agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/).
* Se agregaron enlaces a las páginas de introducción para [los servicios que no están autorizados por FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp/#not-scope).
* Se agregó una condición con respecto a la [visibilidad del botón Cargar dSYM](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/upload-dsyms-bitcode-apps/#mobile-ui) en la New Relic UI.
* Se actualizó [la documentación](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-cosmos-db-document-db-monitoring-integration/#metric-data) para reflejar que `Microsoft.DocumentDB/mongoClusters` no es compatible con la integración de monitoreo de Azure Cosmos DB (Document DB).
