### Nuevos documentos

* [La gestión de cardinalidad](/docs/data-apis/ingest-apis/metric-api/cardinality-management) le proporciona herramientas administrativas para administrar y revisar el uso de la cardinalidad de su organización.

### Cambios importantes

* Instrucciones mejoradas sobre el [seguimiento de usuarios afectados con errors inbox](/docs/errors-inbox/error-users-impacted/), con una nueva captura de pantalla y una tabla que describe los diversos métodos de implementación.
* Se actualizaron [Vincular aplicaciones instrumentadas con OpenTelemetry a Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) y [Explorar su clúster de Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/kubernetes-cluster-explorer) con información actual sobre la instalación y configuración de OpenTelemetry.
* Se reescribió [Reglas de silencio: Suprimir notificaciones](/docs/alerts/get-notified/muting-rules-suppress-notifications) para describir cómo crear, administrar, suprimir y revisar sus eventos de alerta y problemas.
* Reorganizamos nuestra documentación de [solución de problemas de Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/troubleshooting/overview) para mejorar la navegación y la facilidad de búsqueda.
* Revisamos nuestra documentación de [OpenTelemetry](/docs/opentelemetry/opentelemetry-introduction) para mayor claridad y exhaustividad.
* Se agregaron instrucciones para configurar la API de tiempo de ejecución de Composer para la detección de paquetes de gestión de vulnerabilidades en la [configuración del agente PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-vulnerability-management).

### Cambios menores

* Se han actualizado algunos documentos para informar que los gráficos de cartelera no se renderizarán correctamente al compartirse a través de enlaces, imágenes exportadas y archivos PDF.
* Se corrigieron varios documentos en los que se mezclaban comandos de Linux con nuestros comandos de shell de Windows.
* Se eliminaron algunos documentos heredados, luego se agregaron redirecciones a sus versiones actuales.
* Se actualizaron las [condiciones de alerta](/docs/alerts/create-alert/create-alert-condition/alert-conditions) con la IU actual.
* Se agregaron nuevos atributos de [`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming) a nuestro diccionario de datos.
* Se actualizó la documentación de monitoreo de rendimiento de red relacionada con la API de sondeo de Meraki y la configuración predeterminada para usar ICMP con KTranslate.
* Se agregó `NewRelic.Labels` a los [ajustes de configuración de .NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration).
* En [Rastrear usuarios afectados con la bandeja de entrada de errores](/docs/errors-inbox/error-users-impacted/#set-users), se agregó la llamada de API `newrelic_set_user_id` de PHP.
* Se agregaron nuevos valores de tipo de evento prohibidos a [Requisitos y límites de datos para datos de eventos personalizados](/docs/data-apis/custom-data/custom-events/data-requirements-limits-custom-event-data): los prefijos de cadena `EntityRelationship`, `RelationshipUpdate` y `Relationship[0-9]`.
* En [Primeros pasos con New Relic y Terraform](/docs/infrastructure-as-code/terraform/terraform-intro), se describió cómo usar CodeStream con Terraform.
