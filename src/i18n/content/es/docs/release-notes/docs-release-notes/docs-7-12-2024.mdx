### Nuevos documentos

* Un nuevo documento de instalación ramificado para la [integración de monitoreo de Kubernetes](/install/kubernetes/) le ayuda a encontrar la información de instalación específica que necesita.
* Instale, configure y monitoree sus datos de [dbt Cloud](/docs/more-integrations/dbt-cloud/dbt-cloud-integration) que se ejecutan en Apache Airflow y utilizan Snowflake.
* Nuestro nuevo [operador del agente de Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator) automatiza la instrumentación del agente APM para que ya no tenga que administrarlos manualmente. Ahora en vista previa pública.
* El agente de iOS agregó soporte para enviar [logs móviles](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs) e [instrumentación de watchOS](/docs/mobile-monitoring/new-relic-mobile-ios/watchOS/instrument-watchos) a New Relic.
* [Solucionar problemas si no aparecen datos para AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-no-data-appears) describe soluciones para cuando no ve sus datos de AWS Lambda en la interfaz de usuario.
* [Reporte de fallos nativos del agente de Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-native-crash-reporting) describe cómo habilitar, configurar y depurar el reporte de fallos.
* [Plantilla de título para condiciones de alerta NRQL](/docs/alerts/create-alert/condition-details/title-template) describe cómo usar las plantillas de título para crear títulos significativos para sus eventos de alerta.
* [Observabilidad de LLM de Traceloop con OpenLLMetry](/docs/opentelemetry/get-started/traceloop-llm-observability/traceloop-llm-observability-intro) describe cómo utilizar esta herramienta basada en OpenTelemetry para monitorear el rendimiento de su modelo de IA.

### Cambios importantes

* Las actualizaciones de contenido y estilo para vincular aplicaciones [instrumentadas con APM](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes) y [instrumentadas con OpenTelemetry](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) mejoran la legibilidad y facilitan la configuración. También hemos actualizado nuestra introducción a la [integración de Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration) para mejorar el estilo y la claridad.
* Actualizamos nuestra documentación para reflejar el cambio de precio por GB, por ejemplo en [facturación y reglas de ingesta de datos](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices).
* Se agregaron nuevos atributos de [`MobileApplicationExit`](/attribute-dictionary/?dataSource=Mobile&event=MobileApplicationExit) a nuestro diccionario de datos.
* Se agregó información de instalación específica de Azure para [instalar el agente .NET](/install/dotnet/?deployment=azure&azure=azuresiteextension).
* El 3 de julio de 2024, nuestra [integración de Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements/#compatibility) dejó de admitir Kubernetes v1.25 y versiones anteriores.
* Una gran reescritura de [Conozca New Relic AI, su asistente de observabilidad](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-ai) describe cómo aprovechar al máximo nuestras herramientas de IA.
* Se actualizaron los [ajustes de configuración del agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#is_integrations_only) con información de configuración sobre el nuevo ajuste `is_integrations_only`. Habilite esta configuración si está instrumentando un host desde una fuente distinta al agente de infraestructura, como un colector de OpenTelemetry o un exportador de nodos de Prometheus.

### Cambios menores

* Varias actualizaciones relacionadas con el nuevo operador del agente de Kubernetes.
* Las actualizaciones de la documentación de OpenTelemetry corrigieron algunos enlaces de anclaje rotos, agregaron información de configuración de tiempo de espera al [endpoint OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp/#payload-limits) y otras mejoras de consistencia.
* Se actualizó [AWS PrivateLink](/docs/data-apis/custom-data/aws-privatelink/#regions-zones) para aclarar que admitimos múltiples regiones
* Actualizamos nuestra [tabla de compatibilidad del agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#system) para incluir la compatibilidad con la versión 22 de Node.js.
* Movimos nuestra documentación de resolución de problemas de Azure del agente .NET a la [categoría de resolución de problemas](/docs/apm/agents/net-agent/azure-installation/azure-cloud-services-no-data-appears/).
* Se actualizó `issueId` y se agregó `entityGuid` en [/docs/data-apis/ingest-apis/security-data-api](/docs/data-apis/ingest-apis/security-data-api/#req-fields).
* Se corrigió un error en el fragmento de código en la [integración de Cacti](/docs/infrastructure/host-integrations/host-integrations-list/cacti-integration/).
* Se actualizó la [instrumentación sin capas para el monitoreo de Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-layerless/#dotnet) para aclarar que Lambda de .NET no requiere capas.
* Se actualizó [controlar cuándo inicia el agente Ruby](/docs/apm/agents/ruby-agent/troubleshooting/control-when-ruby-agent-starts) para describir cómo desactivar el agente en contextos específicos.
* Eliminamos los formularios de registro integrados en todo nuestro sitio de documentación.
* Se agregaron instrucciones sobre cómo configurar y reenviar los logs de Temporal en la [integración de monitoreo autohospedado de Temporal](/docs/infrastructure/host-integrations/host-integrations-list/temporal-monitoring-integration/#temporal-logs-configuration).
* El soporte para el monitoreo de [Infraestructura](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/#os), [rendimiento SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#linux), [syslog de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#linux) y [datos de flujo de red](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#linux) en CentOS 7 finalizó el 30 de junio de 2024.
* Se actualizaron los [límites de tasa con NRQL](/docs/nrql/using-nrql/rate-limits-nrql-queries/#query-duration) con el tiempo de espera de consulta predeterminado de 5 segundos de NerdGraph.
* Se actualizó [aplicación no responde](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding) con nuevas capturas de pantalla y detalles de configuración del lado del servidor.
* Se corrigió un error de sintaxis para `disable_all_plugins` en la guía de resolución de problemas [reducir la huella de CPU del agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/troubleshooting/reduce-infrastructure-agents-cpu-footprint).
* Las [métricas a nivel de código](/docs/codestream/observability/code-level-metrics/#requirements) de CodeStream para PHP requieren la extensión PHP Intelephense.
* En [eliminar aplicaciones de New Relic](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/#wait-longer), se aclaró que puede eliminar una aplicación cuando New Relic no ha recibido datos de esa aplicación durante 12 horas o más.
* Fusionamos nuestros documentos de reducción de ingesta de datos y gobernanza de datos de Kubernetes en una sola página de [reducción de datos ingestados](/docs/kubernetes-pixie/kubernetes-integration/installation/reduce-ingest).
* Se actualizó el [plugin de AWS FireLens para el envío de logs](/docs/logs/forward-logs/aws-firelens-plugin-log-forwarding/#firelens-plugin) con varias regiones nuevas de AWS que ahora admitimos y sus repositorios de imágenes.
* Se actualizaron los [flujos de trabajo de eventos de alerta](/docs/alerts/get-notified/alert-event-workflows/#navigate-notification-log) con las columnas actuales del log de notificaciones.
* Se actualizó la [compatibilidad y requisitos para el agente Java](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#JDBC) para informarle que nuestro agente Java versión 8.13.0 o superior soporta HikariCP 2.4.0 hasta la más reciente. (Esto reemplaza el módulo incubador de HikariCP.)
* El agente .NET ahora admite [`Elastic.Clients.Elasticsearch`](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#app-frameworks-core) versión 8.14.5.
