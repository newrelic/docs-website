### Nuevos documentos

* [La integración de Amazon MemoryDB](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-memorydb-monitoring-integration) describe cómo activar la integración con New Relic. La integración le permite recopilar datos de telemetría sobre sus clústeres de Amazon MemoryDB, consultar los datos entrantes y crear dashboards.
* [Agregar etiquetas personalizadas (labels) a los logs de la aplicación](/docs/logs/logs-context/custom-tags-agent-forwarder-logs) explica que puede agregar etiquetas personalizadas a los logs reenviados por el agente.

### Cambios importantes

* Se actualizó la [documentación de integraciones de AWS](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations) para mayor claridad y organización. Los cambios clave incluyen destacar CloudWatch Metric streams como el método principal de integración de AWS, mejorar la navegación y reducir la redundancia al consolidar y simplificar el contenido, y realizar ediciones de texto para mejorar la legibilidad.
* Se actualizaron los pasos para [crear un post mortem](/docs/alerts/alert-event-management/postmortems-applied-intelligence/#create-postmortem) para mayor precisión.
* Se realizaron actualizaciones relacionadas con el cumplimiento en [Certificaciones, estándares y regulaciones](/docs/security/security-privacy/compliance/regulatory-audits-new-relic-services) basadas en las evaluaciones anuales de 2024 de New Relic.
* Se agregó compatibilidad con .NET 9 y ASP.NET Core 9 para el agente .NET, se aclaró la postura de soporte sobre las versiones de .NET sin soporte y se eliminaron las referencias obsoletas de .NET Framework en [Agente .NET: compatibilidad y requisitos](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements).
* Se reescribió el [plugin de Fluent Bit para el envío de logs](/docs/logs/forward-logs/fluent-bit-plugin-log-forwarding) para mayor claridad.
* Se actualizaron las [notificaciones de CodeStream](/docs/codestream/codestream-settings/account-settings) para mayor precisión.
* Se agregaron los detalles de la [variable del agente Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#cloud_config) para crear una relación entre los proveedores de la nube y las aplicaciones APM.
* Se actualizaron los pasos para [crear un Azure Event Hub](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export/#create-event-hub).

### Cambios menores

* Se actualizaron las versiones de los agentes .NET y Ruby en [Agregar etiquetas personalizadas (labels) a los logs de la aplicación](/docs/logs/logs-context/custom-tags-agent-forwarder-logs).
* Se proporcionó un ejemplo para mostrar el uso de las API de logging con Unity en [Logs móviles](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs).
* Se actualizaron la [compatibilidad y los requisitos del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements).
* Se actualizó la [compatibilidad y requisitos del agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent).
* Se actualizó el comportamiento predeterminado y la guía de configuración para la inclusión de dispositivos de red y la generación de NetworkSample en [eventos de infraestructura](/docs/infrastructure/infrastructure-data/default-infra-data).
* Se corrigió el nombre de la API `localStorage` y se actualizó la información sobre cuándo el seguimiento de sesiones puede no funcionar en [Seguimiento de sesiones](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking).
* Se actualizaron los detalles de configuración del agente de Node.js para [habilitar](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels) y [excluir](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels-exclude) etiquetas que se agregan a los logs de la aplicación.
