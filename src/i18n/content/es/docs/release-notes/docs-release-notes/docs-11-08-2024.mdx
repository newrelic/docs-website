### Nuevos documentos

* La [integración de FOSSA](/docs/vulnerability-management/integrations/fossa/) describe cómo esta nueva integración respalda sus iniciativas de seguridad de aplicaciones al mostrar vulnerabilidades explotables en sus bibliotecas de código abierto.

### Cambios importantes

* Se reorganizó la [categoría de documentación de infraestructura](/docs/infrastructure/introduction-infra-monitoring/) y se alinearon las URL con su ubicación en el repositorio. Trabajo tras bastidores que facilita encontrar y trabajar en esta documentación.
* Una reescritura significativa de la [API del agente .NET](/docs/apm/agents/net-agent/net-agent-api/net-agent-api/) eliminó información obsoleta, reorganizó cierto contenido, actualizó el código de ejemplo, agregó más detalles sobre lo que hacen las API, añadió un enlace a nuestro repositorio de código de muestra y agregó etiquetas de identificación a las API de rastreo distribuido.

### Cambios menores

* Se agregó DynamoDB a [compatibilidad y requisitos del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#database-core).
* La etiqueta `hostStatus: shutdown` evita la apertura de un evento de alerta de pérdida de señal. Actualizamos [crear condiciones de alerta NRQL](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/) y el [tutorial de NerdGraph sobre pérdida de señal y relleno de vacíos](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/).
* Se actualizaron los [datos predeterminados de monitoreo de infraestructura](/docs/infrastructure/infrastructure-data/default-infra-data/) con nuestra agregación de datos mejorada.
* Se agregó un [aviso de desuso para Kubernetes](/eol/2024/10/deprecation-notice-v1.26-and-lower/).
* Se mejoró la legibilidad en [crear condiciones de infraestructura de &quot;host not reporting&quot;](/docs/infrastructure/infrastructure-alerts/create-infrastructure-host-not-reporting-condition/).
* En la [instrumentación sin capas](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/enable-serverless-monitoring-aws-lambda-layerless/), se agregaron ejemplos para el módulo ES y nuevos pasos para instalar la capa de solo extensión.
* Se reescribió la introducción de [introducción al monitoreo de IA](/docs/ai-monitoring/intro-to-ai-monitoring/) y se actualizaron las capturas de pantalla.
