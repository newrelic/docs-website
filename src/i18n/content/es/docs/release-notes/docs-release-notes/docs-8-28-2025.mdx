### Nuevos documentos

* Se agregó [Administración de acceso para gestionar la cardinalidad](/docs/data-apis/ingest-apis/metric-api/cardinality-access-management/) para controlar el acceso de los usuarios a las métricas relacionadas con la cardinalidad y gestionar licencias para los límites de ingestión de métricas.
* Se agregó [la cardinalidad de Monitor](/docs/data-apis/ingest-apis/metric-api/monitor-cardinality/) para rastrear y monitorear el uso de la cardinalidad métrica para optimizar la ingesta de datos y evitar costos inesperados.

### Cambios menores

* Se agregó [el nuevo parámetro de configuración](/docs/ebpf/linux-installation/#config-params) `DROP_DATA_FOR_ENTITY` para ebpf.

* [Agente Ruby v9.21.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-9-21-0)

  * En el modo APM Serverless, los nombres de transacciones ahora incluyen el nombre de origen del evento como prefijo. Esta mejora mejora la identificación y trazabilidad de las transacciones en la UI New Relic al indicar claramente la fuente del evento desencadenante.
  * Se revirtieron los cambios en la lógica de seguimiento de subprocesos para la inicialización del intervalo. Este cambio restaura el comportamiento anterior de seguimiento del hilo donde comienzan los tramos, solucionando problemas que surgieron de la lógica actualizada.
