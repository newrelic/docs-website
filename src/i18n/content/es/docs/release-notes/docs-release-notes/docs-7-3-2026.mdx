### Nuevos documentos

* Se agregó una página de resolución de problemas dedicada para [El instalador MSI de Windows falla con el error 1603](/docs/infrastructure/infrastructure-troubleshooting/troubleshoot-infrastructure/windows-msi-install-fails-error-1603), para que pueda resolver las fallas de instalación del agente de infraestructura en Windows.

### Cambios importantes

* Se reestructuró la documentación de [monitoreo de Microsoft SQL Server](/docs/opentelemetry/database/mssql/introduction) de OpenTelemetry en una sección dedicada de varias páginas, desde la [configuración guiada](/docs/opentelemetry/database/mssql/guided) hasta las instrucciones de host y Amazon RDS, métricas, compatibilidad y resolución de problemas.
* Se actualizó el monitoreo de [ECS en EC2](/docs/opentelemetry/integrations/ecs-monitoring/ecs-ec2) y [ECS en Fargate](/docs/opentelemetry/integrations/ecs-monitoring/ecs-fargate) con una opción de NRDOT Collector junto con el OpenTelemetry Collector, para que pueda ejecutar el sidecar con la distribución compatible de New Relic.

### Cambios menores

* Se actualizó la documentación de la [base de datos de Oracle](/docs/opentelemetry/database/otel-oracledb) de OpenTelemetry y de [identificación del servicio de base de datos](/docs/opentelemetry/database/database-service-identification) con los parámetros corregidos.
* Se documentó cómo [exportar dashboards y gráficos a través de la API de NerdGraph](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/#export-api) como PDF o PNG, con dimensiones personalizadas, anulaciones de variables de plantilla y filtros de entidad, y se actualizaron los pasos de exportación de la UI para incluir PNG.
* Se documentó cómo [consultar conectores de conocimiento específicos](/docs/agentic-ai/knowledge-integration/overview/#query-specific-connectors) en el chat de New Relic AI, usando `/{connector-name}` para apuntar a una sola fuente o una búsqueda amplia en todos los conectores.
* Se agregó el tipo de gráfico de medidor a la documentación de [tipos de gráficos](/docs/query-your-data/explore-query-data/use-charts/chart-types).
* Se actualizaron los [requisitos de compatibilidad del agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) con el último informe de compatibilidad.
* Se actualizaron los [requisitos de compatibilidad del agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) con las últimas versiones de bibliotecas compatibles.
* Se actualizaron los detalles del [soporte híbrido del agente Ruby](/docs/apm/agents/manage-apm-agents/opentelemetry-api-support), lo que incluye aumentar la versión mínima compatible del agente Ruby a 10.6.0, y las instrucciones de configuración de las [integraciones de AWS](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations).
* Se aclaró en la documentación de la [UI de ingesta de datos](/docs/data-apis/manage-data/manage-data-coming-new-relic) que la vista automática no admite el selector de tiempo y muestra datos en intervalos de 5 minutos por hasta un día, por lo que debe usar las vistas diaria, semanal o mensual para rangos más largos.
* Se eliminaron las API de superposición de la documentación de [navegación del SDK](/docs/new-relic-solutions/build-nr-ui/sdk-component/platform-apis/navigation).

### Publicaciones de novedades

* Se agregó [Exportaciones de dashboard mejoradas y una nueva mutación de instantánea de imagen de gráfico en NerdGraph](/whats-new/2026/06/whats-new-0626-dashboards) para anunciar dos nuevas mutaciones de NerdGraph para generar instantáneas en PDF de páginas completas de dashboard e imágenes de gráficos independientes.

### Notas de la versión

* Mantener actualizado con nuestros últimos lanzamientos:

  * [NRDOT v1.19.0](/docs/release-notes/nrdot-release-notes/nrdot-2026-06-29):

    * Se agregaron los componentes `nroracledbreceiver`, `nrsqlserverreceiver`, `secretsmanagerprovider` y `aesprovider` para el monitoreo de base de datos.

  * [Agente de infraestructura v1.77.1](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1771):

    * Se actualizaron las dependencias empaquetadas, incluyendo `golang.org/x/net`, la imagen base de Alpine (v3.24) y containerd (v1.7.33).

  * [Agente PHP v12.8.0.37](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-12-8-0-37):

    * Se agregó la instrumentación de Redis Cluster.
    * Se actualizó golang a 1.26.4 y se actualizaron las dependencias desactualizadas.
    * Se corrigió un desbordamiento de búfer de realpath en install-util y un desbordamiento de stack.

  * [Agente Android v7.7.7](/docs/release-notes/mobile-release-notes/android-release-notes/android-777):

    * Se movió la compresión de imágenes de Session Replay fuera del hilo principal para reducir la sobrecarga de captura.
    * Se corrigieron las cargas sin conexión obsoletas al aplicar un tiempo de vida que descarta los datos en cola durante demasiado tiempo.
    * Se garantizó que los eventos almacenados en el búfer se envíen en la recolección final durante el cierre de la aplicación, y se corrigieron los atributos de sesión que podían perderse durante la recolección de cierre.

  * [Agente iOS v7.7.3](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-773):

    * Se agregó la funcionalidad de almacenamiento sin conexión para Session Replay móvil.
    * Se cambió el modo de enmascaramiento predeterminado de Session Replay para no permitir el desenmascaramiento programático.
    * Se solucionó un bucle infinito en EventDeserializer que bloqueaba el agente en el lanzamiento.

  * [Integración de Kubernetes v4.3.6](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-4-3-6):

    * Lanzado como parte de las versiones de chart newrelic-infrastructure-4.3.6 y nri-bundle-8.0.0.

  * [Aplicación móvil para Android v5.43.2](/docs/release-notes/mobile-apps-release-notes/new-relic-android-release-notes/new-relic-android-5432):

    * Correcciones de errores y mejoras de la UI.

  * [Aplicación móvil para iOS v6.9.30](/docs/release-notes/mobile-apps-release-notes/new-relic-ios-release-notes/new-relic-ios-6110):

    * Se agregó una vista de detalles de logs en la aplicación para inspeccionar los atributos de log completos, filtrar por atributo y copiar valores.
    * Se agregaron encuestas configurables en la aplicación para recopilar comentarios de los usuarios.
    * Confiabilidad de inicio de sesión mejorada junto con el rendimiento y la estabilidad.

  * [Agente React Native v1.8.6](/docs/release-notes/mobile-release-notes/react-native-release-notes/react-native-agent-186), [Agente Flutter v1.2.8](/docs/release-notes/mobile-release-notes/flutter-release-notes/flutter-agent-128), [Agente Capacitor v1.6.8](/docs/release-notes/mobile-release-notes/capacitor-release-notes/capacitor-agent-168), [Agente Unity v1.6.2](/docs/release-notes/mobile-release-notes/unity-release-notes/unity-agent-162), [Agente Cordova v7.1.7](/docs/release-notes/mobile-release-notes/cordova-release-notes/cordova-agent-717), y [Agente .NET MAUI v1.2.5](/docs/release-notes/mobile-release-notes/net-maui-release-notes/net-maui-agent-125):

    * Agente nativo de Android actualizado a la versión 7.7.7 y agente nativo de iOS a la versión 7.7.3.

  * Tiempos de ejecución sintéticos — [Job Manager v574](/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-574), [tiempo de ejecución de la API de Node rc1.27](/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.27), [Runtime de Node browser rc1.27](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.27) y [tiempo de ejecución de Ping v1.73.0](/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.73.0):

    * Se aplicaron parches a los CVE del sistema operativo subyacente y de dependencia en todas las imágenes de Sintéticos.
    * Para el runtime de Node browser, se eliminaron los registros de rendimiento, los archivos HAR y las capturas de pantalla de los registros del contenedor.
