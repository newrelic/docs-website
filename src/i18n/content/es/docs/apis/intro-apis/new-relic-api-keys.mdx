Nuestras soluciones de monitoreo y API utilizan clave de API para autenticar y verificar su identidad. Estas claves permiten que solo las personas aprobadas en su organización informen datos a New Relic, accedan a esos datos y configuren características. Las claves principales son la clave de licencia (para informar datos) y la <InlinePopover type="userKey" />(para trabajar con NerdGraph, nuestra API GraphQL).

<img title="Screenshot of the API keys UI" alt="Screenshot of the API keys UI" src="/images/accounts_screenshot-crop_api-keys-ui.webp" />

Para comenzar con la clave de API:

1. [Regístrese para obtener su cuenta gratuita](https://newrelic.com/signup) si aún no lo ha hecho
2. Utilice nuestro [agente e integración](https://newrelic.com/instant-observability/?category=infrastructure-and-os) para comenzar a incorporar datos en New Relic
3. Cree y administre su clave de API desde la [página de UI de clave de API](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) para que pueda comenzar a observar sus datos de inmediato

## Ver y administrar clave de API [#keys-ui]

Puedes administrar la mayoría de las claves de API desde la [página de la interfaz de usuario de claves de API](https://one.newrelic.com/api-keys) ubicada en el [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings/#user-menu) cuando haces clic en <DNT>**API keys**</DNT>. También puedes [usar NerdGraph para administrar claves](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys)

**Claves de ingestión (para la cuenta):** Cuando creas una clave de ingestión, te proporcionamos temporalmente la clave completa. Sin embargo, una vez que se crea la clave, solo se muestran los primeros 8 caracteres en la interfaz de usuario de claves de API. Si necesitas recuperar la clave completa después de la creación, haz clic en los puntos suspensivos en la interfaz de usuario de claves de API, copia el ID de la clave y [usa NerdGraph para consultar la clave completa.](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys/#query-keys)

**Claves de usuario (para tu propio uso):** Cuando creas una clave de usuario para ti, te proporcionamos temporalmente la clave completa. Sin embargo, una vez que se crea la clave, solo se muestran los primeros 8 caracteres en la interfaz de usuario de claves de API. Si necesitas recuperar la clave completa después de la creación, haz clic en los puntos suspensivos en la interfaz de usuario de claves de API, copia el ID de la clave y [usa NerdGraph para consultar la clave completa.](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys/#query-keys)

**Claves de usuario (para otros usuarios):** Cuando creas una clave de usuario para otro usuario, solo verás la versión truncada de la clave en la interfaz de usuario de claves de API y a través de NerdGraph.

## Recomendaciones para gestionar clave de API [#security-practices]

Si su clave de API cae en las manos equivocadas, puede presentar un riesgo de seguridad. Por ejemplo:

* Alguien con tu <InlinePopover type="licenseKey" />podría enviar datos arbitrarios a tu cuenta.
* Alguien con una de las claves de usuario de un miembro de su equipo podría ver sus datos de New Relic y realizar cambios en su cuenta de New Relic.

Debes tratar tu clave de API de forma segura, como lo harías con las contraseñas y otra información confidencial. Algunas recomendaciones:

* Para la clave de licencia y la clave del browser , considere implementar una estrategia de rotación de claves: crear claves nuevas y eliminar las antiguas según un cronograma establecido. Consideraciones:

  * No puedes eliminar las claves de ingestión originales asociadas con una cuenta, por lo que recomendamos crear claves de ingestión adicionales que luego puedas eliminar. Esto garantiza que adoptes una postura de seguridad sólida.
  * Tenga en cuenta que esto no se aplica al token de la aplicación móvil; no puede eliminar un token ni crear tokens adicionales.

* Para el <InlinePopover type="userKey" />:

  * Indique a los miembros de su equipo que mantengan seguras sus claves de usuario.
  * Cuando los miembros abandonan tu organización, elimina sus ID de usuario de New Relic. Esto deshabilita todas las claves de usuario asociadas con su ID de usuario.

Cada clave está vinculada a una cuenta específica y el usuario puede crear tantas como desee. La clave de licencia no se puede actualizar y debe eliminarse y reemplazarse creando una nueva clave.

    Puedes encontrar las claves de licencia para tu cuenta en la interfaz de usuario de claves de API en [<DNT>**one.newrelic.com/api-keys**</DNT>](https://one.newrelic.com/api-keys) (o [<DNT>**one.eu.newrelic.com/api-keys**</DNT>](https://one.eu.newrelic.com/api-keys) para nuestro centro de datos de la UE).     De forma predeterminada, todas las cuentas de New Relic se crean con una clave de licencia de cuenta original, que es compartida por toda la organización. No puede eliminar esta clave de la clave de API UI, pero puede comunicarse con el soporte de New Relic para rotar su clave.

    1. log sesión como el usuario de New Relic cuya clave desea rotar.

    3. Haga clic en <DNT>**Create a key**</DNT> en la esquina superior derecha de la página <DNT>**API keys**</DNT> .

       <img title="Screenshot of the copy key options" alt="Screenshot of the copy key options" src="/images/apis-and-data_screenshot-crop_copy-key.webp" />

    6. Una vez que se haya actualizado cualquier dependencia externa, puede eliminar de forma segura la clave anterior haciendo clic en el ícono <DNT>**...**</DNT> en la misma fila que la clave anterior y luego haciendo clic en <DNT>**Delete**</DNT>.

       <img title="Screenshot of delete key option" alt="Screenshot of delete key option" src="/images/apis-and-data_screenshot-crop_delete-key.webp" />

## Lea más sobre nuestra clave de API [#key-details]

Para crear o administrar clave de API, use la UI en [one.newrelic.com/launcher/api-keys-ui.api-keys-launcher](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) o nuestra [API NerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys). Para más detalles, seleccione una clave:

<CollapserGroup>
  <Collapser id="license-key" title="Clave de licencia">
    Nuestra clave principal utilizada para la ingesta de datos se llama clave de licencia. En la clave de API UI y en [NerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys), a veces se hace referencia a esta clave como `ingest - license`.

    La clave de licencia es necesaria para casi toda la ingesta de datos de New Relic. Las excepciones son el monitoreo de datos del navegador (que utiliza una clave browser ) y el monitoreo de datos móviles (que utiliza un token de aplicación móvil).

    La clave de licencia es una cadena hexadecimal de 40 caracteres asociada con una cuenta de New Relic. Cuando se [registra](/docs/subscriptions/creating-your-new-relic-account) por primera vez en New Relic, se crea una organización con una sola cuenta y su propia clave de licencia. Si se agregan más cuentas, cada cuenta comienza con su propia clave de licencia. La clave de licencia creada originalmente para una cuenta no se puede eliminar, pero puede crear claves de licencia adicionales que se pueden administrar y eliminar, y esto es útil para implementar prácticas de seguridad como la rotación de claves. Si necesita rotar la clave de licencia de cuenta original de una cuenta, [comuníquese con el soporte](https://support.newrelic.com).

    Para impedir que un usuario pueda ver o administrar la clave de licencia, asígnele un rol sin esos permisos: [modelo de usuario original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#add-on) | [modelo de usuario más nuevo](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions).
  </Collapser>

  <Collapser id="browser-key" title="Clave browser">
    Monitoreo de browser utiliza una clave browser para reportar datos, en lugar de la clave de licencia. La clave del navegador se utiliza para asociar datos del [agente de monitoreo del navegador](/docs/browser/new-relic-browser/getting-started/introduction-browser-monitoring) a su cuenta.

    No puede administrar ni eliminar la clave del browser original creada cuando se creó su cuenta, pero puede crear nuevas claves browser y eliminarlas. Para obtener ayuda para rotar la clave original del browser de una cuenta, [comuníquese con el soporte técnico](https://support.newrelic.com).
  </Collapser>

  <Collapser id="mobile-app-token" title="Token de aplicación móvil">
    Monitoreo de móviles utiliza un token de aplicación móvil para reportar datos, en lugar de la clave de licencia. Consulte [Token de aplicación móvil](/docs/mobile-monitoring/new-relic-mobile/maintenance/viewing-your-application-token) para obtener más información.
  </Collapser>

  <Collapser id="user-key" title="Clave de usuario">
    Se requieren claves de usuario New Relic , a veces denominadas &quot;clave de API personal&quot;, para usar [NerdGraph](/docs/apis/intro-apis/introduction-new-relic-apis/#graphql) y nuestra [API REST](/docs/apis/intro-apis/introduction-new-relic-apis/#rest-api).

    Un <InlinePopover type="userKey" />está vinculado a un usuario de New Relic específico y no se puede transferir. La clave de usuario le permite consultar cualquier cuenta a la que se le haya otorgado acceso, no solo la cuenta específica a la que estaba asociada la clave. Si un usuario de New Relic se elimina en New Relic, sus claves de usuario también se desactivan y no funcionarán.

    Aunque proporcionan acceso de usuario a varias cuentas, las claves de usuario están vinculadas a una única cuenta específica: la cuenta desde la que fueron creadas. La importancia de esto es que si se elimina una cuenta, las claves de usuario asociadas con esa cuenta ya no funcionarán. (Además, para nuestra API de llamadas REST se limitan a la cuenta asociada con esa clave de usuario).

    To prevent a user from viewing or managing user keys, [assign them a role without those permissions](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions).
  </Collapser>
</CollapserGroup>
