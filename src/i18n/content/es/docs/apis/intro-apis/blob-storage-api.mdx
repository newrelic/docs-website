<Callout title="característica Disponibilidad" variant="important">
  <DNT>Fleet Control for Kubernetes</DNT> clusters está disponible de forma general (GA). El soporte para administrar agentes en <DNT>Linux and Windows hosts</DNT> se encuentra actualmente en versión preliminar pública.

  Para obtener una lista completa de agentes compatibles y sus entornos, consulte nuestra [documentación de compatibilidad de tipos de agentes](/docs/new-relic-control/agent-control/agent-types).

  La función de vista previa pública se proporciona de conformidad con nuestras [políticas de prelanzamiento](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

El <DNT>Blob Storage API</DNT> es un servicio <DNT>New Relic</DNT> diseñado para cargar y gestionar archivos en tu cuenta. <DNT>NerdGraph</DNT> está optimizado para consultas y mutaciones de datos estructurados, por lo que la API de Blob Storage se utiliza para operaciones que involucran la transferencia de contenido de archivos y el control de versiones.

Dentro de <DNT>Fleet Control</DNT>, el <DNT>Blob Storage API</DNT> gestiona las configuraciones de los agentes—encargándose de la creación, el control de versiones, la recuperación de contenido y la eliminación de los archivos de configuración.

<Callout variant="important">
  Las configuraciones del agente en <DNT>Fleet Control</DNT> utilizan el <DNT>**Blob Storage API**</DNT>, no <DNT>NerdGraph</DNT>. Para operaciones de flota, miembro y despliegue, consulte el **tutorial de<DNT>NerdGraph</DNT> **.
</Callout>

## Requisitos previos

* Una [cuenta de<DNT>New Relic</DNT> ](https://newrelic.com/signup)con una clave de API de usuario
* Su <DNT>New Relic Organization ID</DNT>
* Los [permisos](/docs/new-relic-control/fleet-control/overview#required-permissions) adecuados para administrar configuraciones

## Autenticación

Todas las solicitudes a la API de <DNT>Blob Storage</DNT> requieren autenticación mediante una clave de API de usuario de <DNT>New Relic</DNT>.

**Generar una clave de API:**

1. Vaya a [one.newrelic.com](https://one.newrelic.com)
2. Haga clic en su nombre en la esquina inferior izquierda
3. Selecciona **API Keys**
4. Crea una clave de **User** (no una clave de Browser o Licence)

**Incluir en los encabezados de la solicitud:**

```
Api-Key: NRAK-YOUR-USER-API-KEY
```

## Endpoint base

```
https://blob-api.service.newrelic.com/v1/e
```

Para las cuentas de la región de la UE, use:

```
https://blob-api.service.eu.newrelic.com/v1/e
```

    ### Tipos de agentes admitidos

    **Agentes de infraestructura** (para HOST y KUBERNETESCLUSTER):

    * `NRInfra` - Agente de New Relic Infrastructure
    * `NRDOT` - Distribución de New Relic del Colector de OpenTelemetry
    * `FluentBit` - Agente de logging Fluent Bit
    * `NRPrometheusAgent` - Agente de Prometheus
    * `PipelineControlGateway` - Gateway de Pipeline Control
    * `NRApmOperator` - Operador de APM para Kubernetes
    * `NReBPFAgent` - agente eBPF

    ### Solicitud de muestra

    ```bash
    curl -X POST \
      https://blob-api.service.newrelic.com/v1/e/organizations/YOUR_ORG_ID/AgentConfigurations \
      -H 'Api-Key: NRAK-YOUR-API-KEY' \
      -H 'Content-Type: application/x-yaml' \
      -H 'NewRelic-Entity: {"name": "Production Infra Config", "agentType": "NRInfra", "managedEntityType": "HOST"}' \
      -d 'license_key: YOUR_LICENSE_KEY
    log:
      level: info
      forward: true
    integrations:
      - name: nri-docker
        enabled: true'
    ```

    ### Ejemplo de respuesta

    ```json
    {
      "entityGuid": "<YOUR_ENTITY_GUID>",
      "blobId": "<YOUR_BLOB_ID>",
      "blobVersionEntity": {
        "entityGuid": "<YOUR_ENTITY_GUID>",
        "version": 1
      }
    }
    ```

    **Importante:** Guarde el `entityGuid` de la respuesta. Lo necesitará para versionar, recuperar y eliminar la configuración.

## Mejores practicas

* **Almacena los GUID de la entidad:** Guarda el `entityGuid` devuelto por las operaciones de creación. Los necesitarás para las operaciones de versionado, recuperación y eliminación.
* **Versione de forma incremental:** Cree nuevas versiones para los cambios de configuración en lugar de eliminar y volver a crear las configuraciones.
* **Use nombres descriptivos:** Los nombres de configuración deben indicar claramente su propósito y entorno de destino.
* **Validar YAML:** Asegúrese de que el contenido de su configuración sea YAML válido antes de cargarlo.
* **Verifique la compatibilidad del tipo de agente:** Asegúrese de que su tipo de agente sea compatible con el tipo de entidad administrada (HOST o KUBERNETESCLUSTER).
* **Proteja su clave de API:** Nunca exponga su clave de API de usuario en el código del lado del cliente ni en repositorios públicos.
* **Verifique los códigos de estado HTTP:** La API devuelve 2xx para operaciones exitosas, 404 para no encontrado y otros códigos de estado para errores.
