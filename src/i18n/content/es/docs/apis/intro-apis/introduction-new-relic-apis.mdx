New Relic ofrece una variedad de API y SDK que puede utilizar para:

* Envía datos a New Relic.
* Recuperar datos de New Relic.
* Ver y configurar ajustes.

¿Buscas clave de API? Ver [clave de API](/docs/apis/intro-apis/new-relic-api-keys).

## API para la ingesta de datos [#data-type-apis]

Nuestras API de ingesta de datos son algunas de nuestras [muchas soluciones para generar informes de datos](https://newrelic.com/instant-observability). Nuestras API se pueden utilizar directamente, pero también son la ruta de ingesta subyacente para muchas de nuestras herramientas de generación de informes de datos. Si recién está comenzando a reportar datos a New Relic, le recomendamos comenzar en [Instalar New Relic](/docs/new-relic-solutions/new-relic-one/install-configure/install-new-relic).

## NerdGraph (GraphQL) [#graphql]

[NerdGraph](/docs/apis/graphql-api/getting-started/introduction-new-relic-graphql-api) es la API que recomendamos para consultar datos de New Relic, consultar información de cuentas y realizar una variedad de configuraciones características. Para saber qué puedes hacer, consulta [los tutoriales de NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#tutorials).

NerdGraph es nuestra API más nueva y es nuestro intento de reunir en un solo lugar algunas de nuestras API más antiguas. Tenga en cuenta que todavía hay algunas funciones que puede realizar con [las API REST](#rest-api) que aún no se pueden realizar con NerdGraph, y es por eso que algunas organizaciones de New Relic todavía usan la API REST.

## API REST [#rest-api]

Nuestra [API REST](/docs/apis/rest-api-v2) es nuestra API anterior para consultas y configuración, que [NerdGraph](#graphql) está en proceso de reemplazar. Estamos realizando un trabajo mínimo y continuo en la API REST. La API REST todavía tiene algunas capacidades de configuración que NerdGraph aún no tiene, pero su primera opción debería ser usar NerdGraph porque ahí será donde centraremos nuestra atención y agregaremos características.

## API por característica [#product-apis]

Las herramientas y características de New Relic, como APM, monitoreo de infraestructura, <InlinePopover type="browser"/>y alerta, a menudo se usan juntas y, a veces, pueden superponerse en funcionalidad. Es por eso que varias API pueden ser relevantes para cada área. Algunas funciones de la API dependerán de su [acceso a características y datos](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data).
