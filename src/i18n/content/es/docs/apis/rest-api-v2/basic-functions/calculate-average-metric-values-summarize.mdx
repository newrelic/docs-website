De forma predeterminada, la API de llamada REST de New Relic devuelve una serie de valores de datos métricos basados en [restricciones de rango de tiempo](/docs/apm/apis/requirements/extracting-metric-data#time). Para obtener el promedio de estos valores, incluya `&summarize=true` en su llamada API. Por ejemplo:

```
curl -X GET "https://api.newrelic.com/v2/applications/${APP_ID}/metrics/data.json" \
     -H "Api-Key:<a href="/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key">$API_KEY</a>" -i \
     -d 'names[]=Agent/MetricsReported/count&from=2014-08-11T14:42:00%2B08:00&to=2014-08-11T15:12:00%2B08:00&summarize=true'
```

Si tiene una [cuenta de la región de la UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center), la URL anterior comienza con `https://api.eu.newrelic.com`.

<Callout variant="tip">
  Para evitar resumir datos, omita `summarize` en su llamada API. No es necesario especificar `&summarize=false`.
</Callout>
