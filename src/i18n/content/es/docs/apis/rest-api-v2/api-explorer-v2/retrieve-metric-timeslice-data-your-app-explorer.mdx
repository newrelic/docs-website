Cuando utilice REST API Explorer (v2) de New Relic para obtener [datos de intervalo de tiempo de métrica](/docs/telemetry-data-platform/ingest-manage-data/understand-data/new-relic-data-types#timeslice-data) para su aplicación, necesita:

* Una [clave de API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)
* El [ID de la aplicación](#app_id)
* El tipo de datos de intervalo de tiempo de métrica disponibles para su aplicación, incluidos los [nombres y valores de las métricas](#view_metrics)

## Ver el ID de tu aplicación [#app_id]

Para ver [el ID de tu aplicación](/docs/apm/apis/requirements/identification-code):

1. Desde el [Explorador de API](https://rpm.newrelic.com/api/explore "El enlace se abre en una ventana nueva.") REST de New Relic, seleccione

   <DNT>
     **Applications > GET List**
   </DNT>

   .

2. Si no ha iniciado sesión en New Relic, proporcione una [clave de API](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing) para su aplicación.

3. Opcional: desde

   <DNT>
     **Applications > List**
   </DNT>

   , complete los valores para los filtros `name`, `ids` o `language` .

4. Seleccione

   <DNT>
     **Send Request**
   </DNT>

   .

5. Desde la sección

   <DNT>
     **Response**
   </DNT>

   , copie el `id` de la aplicación.

Continúe con el procedimiento para ver [nombres métricos](#view_metrics).

## Ver nombres métricos para tu aplicación [#view_metrics]

Para ver los datos de intervalo de tiempo de métrica disponibles para tu aplicación:

1. En el Explorador de API REST de New Relic, seleccione

   <DNT>
     **Applications > GET Metric Names**
   </DNT>

   .

2. Desde

   <DNT>
     **Applications > Metric Names**
   </DNT>

   , escribe o pega tu [ID de la aplicación](/docs/apm/apis/requirements/identification-code).

3. Complete una métrica específica o parcial `name`, o déjela en blanco para ver la lista de nombres y valores de métricas disponibles.

4. Seleccione

   <DNT>
     **Send Request**
   </DNT>

   .

5. En la sección

   <DNT>
     **Response**
   </DNT>

   , revisa los `metrics` disponibles de la aplicación.

Continúe con el procedimiento para obtener [datos](#app_metric_data) [métricos](#app_metric_data) de intervalos de tiempo.

## Obtén datos de intervalo de tiempo de métrica para tu aplicación [#app_metric_data]

Para obtener los valores métricos de su aplicación, copie los valores de los resultados <DNT>**Metric Names**</DNT> y péguelos en los campos <DNT>**Metric Timeslice Data**</DNT> :

1. En el Explorador de API REST de New Relic, seleccione

   <DNT>
     **Applications > GET Metric Data**
   </DNT>

   .

2. Desde

   <DNT>
     **Applications > Metric Data**
   </DNT>

   , escribe o pega tu [ID de la aplicación](/docs/apm/apis/requirements/identification-code).

3. Escriba o pegue uno o más `names` (de

   <DNT>
     **GET Metric Names**
   </DNT>

   ) para su aplicación. Comience cada nombre en una nueva línea.

4. Seleccione

   <DNT>
     **Send Request**
   </DNT>

   .

5. En la sección

   <DNT>
     **Response**
   </DNT>

   , revisa los `metric_data` disponibles de la aplicación.

Después de revisar la sección <DNT>**Response**</DNT> , puede continuar haciendo ajustes y seleccionar <DNT>**Send Request**</DNT> nuevamente. Por ejemplo:

* Para especificar un [rango de tiempo](/docs/apm/apis/api-v2-examples/specifying-time-range-api-v2) para el intervalo de tiempo de datos métricos, complete los valores `from` y `to` .
* Para crear un [promedio](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) de los datos durante el intervalo de tiempo, establezca `summarize` en `true`. O bien, para conservar los datos de cada período de tiempo para el rango que haya especificado, deje `summarize` en blanco.
