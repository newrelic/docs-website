El rendimiento promedio del navegador aparece en la parte superior derecha del gráfico <DNT>**Throughput by browser**</DNT> en su [página<DNT>**Summary**</DNT> ](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance#viewing). Se trata del promedio de páginas por minuto (PPM) por tipo de browser en el rango de tiempo que se muestra en el gráfico.

## Rendimiento medio de página [#h2-average]

<Callout variant="important">
  Si bien los ejemplos emplean New Relic API REST v2 de , recomendamos emplear [NRQL funciones](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/) para ejecutar el intervalo de tiempo de métrica consulta. Cada valor de API se puede asignar a una función NRQL equivalente. Para aprender cómo crear una consulta NRQL basada en estos ejemplos API , consulte nuestra [documentación](/docs/apis/rest-api-v2/migrate-to-nrql/).
</Callout>

Para obtener el [promedio](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) del [rango de tiempo](/docs/apm/apis/requirements/specifying-time-range-api-v2) (el valor predeterminado son los últimos 30 minutos), use el siguiente comando. Asegúrese de reemplazar las variables `$APP_ID` y `$API_KEY` en este ejemplo con su [ID de aplicación](/docs/apm/apis/requirements/identification-code) específico y [su clave API REST](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing).

```sh
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json" \
     -H "X-Api-Key:$API_KEY" -i \
     -d 'names[]=EndUser&values[]=requests_per_minute&summarize=true'
```
