<Callout title="avance">
  Todavía estamos trabajando en esta característica, ¡pero nos encantaría que la probaras!

  Esta característica se proporciona actualmente como parte de un programa de vista previa de conformidad con nuestras [políticas de prelanzamiento](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Utilice el <DNT>NerdGraph API</DNT> para crear, probar, listar, actualizar y eliminar <DNT>cross account destinations</DNT>. <DNT>Cross account destinations</DNT> le permiten centralizar la gestión de notificaciones a nivel de la organización.

## Crear un destino [#create-destination]

Para crear un destino, se deben proporcionar diferentes entradas para cada tipo de destino.

<Callout variant="important">
  Debe proporcionar un objeto de alcance que identifique dónde se creará el destino. El objeto de alcance requiere:

  * id: El ID de la cuenta u organización.
  * type: El tipo de alcance (Cuenta u Organización).

  Una propiedad opcional two\_way\_integration está disponible para integraciones que permiten la integración bidireccional.
</Callout>

    Crear un destino de Slack entre cuentas en New Relic requiere autenticación OAuth entre <DNT>NerdGraph API</DNT> y Slack. Los siguientes pasos muestran cómo generar la URL de autenticación y completar la configuración.

    <Steps>
      <Step>
        **Generar la URL de OAuth**

        1. Abra el <DNT>[NerdGraph Explorer](https://api.newrelic.com/graphiql)</DNT> y pegue la siguiente consulta en el editor de consultas. Esta consulta genera una URL única de OAuth de Slack para su organización.

           ```graphql
           {
             actor {
               organization {
                 aiNotifications {
                   oAuthUrl(type: SLACK) {
                     url
                     transactionId
                     error {
                       description
                       details
                       type
                     }
                   }
                 }
               }
             }
           }
           ```

        2. Haga clic en <DNT>**Play** (Execute Query)</DNT> en la parte superior del explorador.
      </Step>

      <Step>
        **Acceda al enlace de autenticación**

        1. Una vez que se ejecute la consulta, verá una respuesta JSON en el lado derecho de la pantalla que contiene una URL.
        2. Localice el campo `url` en la respuesta.
        3. Copie la cadena de URL completa (todo lo que está dentro de las comillas).
        4. Abra una nueva pestaña en su navegador y pegue la URL.
      </Step>

      <Step>
        **Completar la autorización de Slack**

        1. El enlace lo lleva a una página de autorización de Slack.
        2. Si se le solicita, inicie sesión en el espacio de trabajo de Slack que desea conectar.
        3. Revise los permisos solicitados por New Relic.
        4. Para autorizar la conexión, haga clic en <DNT>**Allow**</DNT>.
        5. Debería ser redirigido a una página de &quot;Éxito&quot; de New Relic.
      </Step>

      <Step>
        **Verifique el destino**

        Después de una autenticación OAuth exitosa, el destino se crea automáticamente a nivel de organización, lo que lo hace disponible en todas las cuentas de su organización.

        1. Navegue hasta <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Alerts &gt; Destinations**</DNT>.
        2. En la lista <DNT>**Destinations**</DNT>, ahora debería ver una nueva entrada para el espacio de trabajo de Slack que acaba de autenticar.
      </Step>
    </Steps>
  </Collapser>

              nextCursor
              entities {
                id
                name
                scope {
                  id
                  type
                }
              }
              totalCount
            }
          }
        }
      }
    }
