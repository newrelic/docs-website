New Relic te permite usar NerdGraph para crear [monitores de navegador simples](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors). Los monitores de navegador simples realizan comprobaciones automatizadas basadas en el navegador de su aplicación sitio web, simulando la interacción real del usuario al cargar páginas en un entorno de navegador completo. Este tutorial proporciona ejemplos de cómo emplear la API NerdGraph para automatizar la creación de monitores de navegador simples.

### Solicitud de muestra

```graphql
mutation {
  syntheticsCreateSimpleBrowserMonitor(
    accountId: ACCOUNT_ID
    monitor: {
      browsers: [BROWSERS]
      devices: [DEVICES]
      locations: { public: ["LOCATION_1", "LOCATION_2"] }
      name: "YOUR_MONITOR_NAME"
      period: PERIOD
      runtime: {
        runtimeType: "RUNTIME_TYPE"
        runtimeTypeVersion: "RUNTIME_TYPE_VERSION"
        scriptLanguage: "SCRIPT_LANGUAGE"
      }
      status: STATUS
      uri: "MONITORED_URI"
      advancedOptions: {
        customHeaders: { name: "HEADER_NAME", value: "HEADER_VALUE" }
        enableScreenshotOnFailureAndScript: ENABLE_SCREENSHOT
        responseValidationText: "VALIDATION_TEXT"
        useTlsValidation: TLS_VALIDATION
      }
      apdexTarget: APDEX_TARGET
    }
  ) {
    errors {
      description
      type
    }
  }
}
```

### Ejemplo de respuesta

Una respuesta exitosa devuelve `null` para los errores:

```json
{
  "data": {
    "syntheticsCreateSimpleBrowserMonitor": {
      "errors": null
    }
  }
}
```

Si hay algún problema al crear el monitor, la matriz `errors` contendrá objetos con campos `description` y `type` que explican qué salió mal.

## Actualizar el tiempo de ejecución de un monitor de navegador simple [#upgrade-monitor-runtime]

Puedes actualizar un monitor de navegador simple para emplear el entorno de ejecución de Chrome más reciente. Recomendamos usar la versión `LATEST` para asegurarse de que su monitor se mantenga actualizado automáticamente con las versiones más recientes de Chrome.

<Callout variant="important">
  **Gestión de versiones de Chrome**: New Relic está pasando a un modelo de versiones `LATEST` para los entornos de ejecución de Chrome. Versiones específicas de Chrome (como 134 y 140) llegarán al final de su vida útil y solo `LATEST` recibirá mantenimiento en el futuro. El **18 de agosto de 2026**, los monitores que aún utilicen versiones anteriores de Chrome se actualizarán de manera forzada a la última versión estable (Chrome 147 o superior). Cuando Chrome lance una nueva versión, estará disponible en los monitores de navegador simples como `LATEST` dentro de una semana. Para garantizar que sus monitores reciban automáticamente las últimas actualizaciones y características de Chrome, actualice a `runtimeTypeVersion: "LATEST"` antes de esta fecha.
</Callout>

## Eliminar un monitor de navegador simple [#delete-monitor]

Cuando ya no se necesita un monitor de navegador simple, puedes eliminarlo de forma permanente empleando la mutación `syntheticsDeleteMonitor`.

Para eliminar un monitor, consulte la sección [Eliminar monitor Sintético](/docs/apis/nerdgraph/examples/synthetics-api/ping-monitor/#delete-monitor).
