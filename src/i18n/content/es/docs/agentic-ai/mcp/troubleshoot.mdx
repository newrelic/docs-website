## El flujo OAuth no se inicia.

* Cerciorar de que su navegador pueda acceder a `localhost`.
* Comprueba la configuración del firewall.
* Verifique que sus herramientas puedan acceder a las URL `OAuth`.

## clave de API no funciona

* Verifique el formato de la clave (comienza con `NRAK-`).
* Verifica las licencias clave en New Relic.
* Cerciorar de que la clave no caducó.

## No se detectó el servidor MCP

* Resetear la aplicación por completo.
* Comprobar la validez de la sintaxis `JSON`.
* Verifique las rutas de los archivos y las licencias.

## Problemas con el proxy remoto de mcp

* Cerciorar de que `Node.js` esté instalado.
* Comprueba la conexión a Internet.
* Verifique que npx pueda acceder al paquete `mcp-remote`.

## Acceso denegado para grupos personalizados

### Problema

No puede conectarse o autenticar correctamente con el servidor New Relic AI MCP y recibe el siguiente mensaje de error: `Missing required capabilities to access MCP server. Please ensure you have the necessary permissions.` Esto suele ocurrir si es afiliado a un grupo de usuarios personalizado sin ninguna concesión a nivel de organización.

### Causa

Su grupo personalizado carece de los controles de acceso o roles a nivel de organización necesarios para emplear el servidor MCP extremo. El acceso a esta característica de toda la organización debe otorgar explícitamente a su grupo o rol de usuario.

### Resolución

El administrador de su organización que tiene el rol `Authentication Domain Manager` debe implementar una de las siguientes dos soluciones para otorgarle acceso:

1. Otorgue al grupo un rol a nivel de organización:
   * El administrador debe agregar uno de los siguientes roles al grupo personalizado al que pertenece: `Organization Read Only`, `Organization Manager` o `Organization Product admin`.
2. Trasladarte a un grupo autorizado:
   * El administrador debe agregarlo a un grupo de usuarios existente que ya tenga uno de los roles específicos requeridos a nivel de organización enumerados anteriormente.

Para obtener información detallada sobre los conceptos de gestión de usuarios, roles y concesiones de organizaciones, consulte la [documentación de gestión de usuariosNew Relic ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts). Si su organización emplea el Sistema para la Gestión de Identidades entre Dominios (SCIM) para la gestión de usuarios, estos ajustes de grupos y roles deben realizar dentro de su proveedor de identidad externo (IdP).

## Mensaje de error

* **Conexión rechazada**: Comprueba la URL y la conectividad de red de tus herramientas.
* **Se requiere autenticación**: verifique que la clave de API siga activa y se ingresó correctamente o complete el flujo `OAuth`.
* **Cliente no válido**: problema de configuración `OAuth`, intente con el proxy `mcp-remote`.
* **`Time.UnmarshalJSON`**: Emplear `mcp-remote` proxy para `OAuth`.
