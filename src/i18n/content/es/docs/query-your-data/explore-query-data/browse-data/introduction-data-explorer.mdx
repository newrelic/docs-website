Navega por todos tus datos visualmente usando <DNT>**Metrics and events**</DNT> y sin necesidad de saber escribir [consultas en formato NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language).

¿Necesita realizar una búsqueda más detallada? Lea cómo [consultar sus datos](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/). Si aún no lo ha hecho, cree su cuenta New Relic gratuita a continuación para comenzar a monitorear sus datos hoy.

## Por qué es importante [#why-matters]

¿Investigas el estado de tus sistemas? ¿Necesita planificar recursos, identificar y responder a incidentes o solucionar problemas de conductas erróneas? métrica y evento facilita la identificación, obtención y visualización de los datos que busca a través de menús visuales, sin necesidad de utilizar NRQL ni crear consultas.

Con <DNT>**Metrics and events**</DNT> puedes acceder a todos los datos almacenados en la base de datos de New Relic (NRDB) de una manera rápida e intuitiva. Luego puede seleccionar <DNT>**Dimensions**</DNT> para facetar por dimensiones específicas de esos datos.

Otras cosas que métrica y evento te ayudan a hacer:

* Vea datos desde diferentes puntos de vista: desde datos sin procesar hasta diferentes visualizaciones que brindan información valiosa sobre evolución, distribución y más.
* Profundice en los datos mediante filtros.
* Agregue sus búsquedas a un [dashboard](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards).
* Entienda cómo funciona [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) : métrica y evento muestra cómo se construyen las consultas.

## Tipos de datos que puedes explorar [#data-types]

En el contexto de <DNT>**Metrics and events**</DNT>, `event` se refiere a todos los objetos almacenados en la base de datos de New Relic (NRDB). métrica y evento te permiten explorar:

* Métrica:

  * [Métrica dimensional](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) (el tipo de datos `Metric` ).

  * [intervalo de tiempo de datos métricos](/docs/data-apis/understand-data/new-relic-data-types/#timeslice-data):

    <InlinePopover type="apm"/>

    agente, agente móvil y el

    <InlinePopover type="browser"/>

    agente reportan este tipo de datos. Para explorar este tipo de datos, debes elegir un monitor de entidad por parte de uno de esos agentes, y luego verás esas opciones.

* Eventos: el [tipo de datos del evento](/docs/data-apis/understand-data/new-relic-data-types/#events-new-relic).

* log: el tipo de datos `Log` (que también puede explorar a través de nuestra [UIde registro](/docs/logs/ui-data/use-logs-ui)).

* traza: el tipo de datos `Span` (que también puedes explorar a través de nuestra [UIde traza](/docs/distributed-tracing/ui-data/query-distributed-trace-data)).

## Explora tus datos [#explore-data]

Para acceder al explorador de datos, vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > Metrics and events**</DNT>.

Métrica y evento constan de estos:

* Sección de alcance en la parte superior. Puede seleccionar entre `metrics` o `events` y `account`. Si seleccionas métrica, puedes filtrar por entidad.
* Área de navegación de datos a la izquierda. Si elige `metrics`, las opciones son `metric` y `dimensions`. Si elige `events`, las opciones son `event type`, `plot` y `dimensions`.
* Espacio de trabajo. Esta área le muestra el resultado de sus elecciones.

Para usar métrica y evento:

1. Definir el alcance. Seleccione el tipo de datos (métrica o evento), la cuenta y la entidad.

2. Seleccione el rango de tiempo usando el [selector de tiempo](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker).

3. Utilice los bloques de la izquierda para explorar los datos disponibles y realizar su búsqueda. Sólo puedes seleccionar un elemento por bloque. Los bloques se pueden buscar.

## Visualice su métrica personalizada después de crearla [#visualize-metric]

Sigue estos pasos para visualizar tu métrica personalizada a través del explorador de métricas:

1. Accede al explorador de datos, ve a <DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > Metrics and events**</DNT>.

2. Elige tu entidad.

3. Haga clic en el enlace <DNT>**See timeslice metrics for `your entity`**</DNT> .

## Casos de uso [#uses-cases]

Vea los siguientes ejemplos para aprender cómo y cuándo usar métrica y evento.

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="use-case-validate-data"
    title="Descubre tus nuevos datos"
  >
    <DNT>
      **I've just connected new instrumentation and want to see if new data is available.**
    </DNT>

    1. Seleccione la cuenta y el evento o métrica que genera los nuevos datos.
    2. Utilice las diferentes herramientas de métricas y eventos para jugar con los nuevos datos que están disponibles: eche un vistazo a los datos sin procesar de ese evento o métrica como una tabla, déles forma de lista o haga clic para verlos trazados como un gráfico. .
    3. Después de seleccionar un evento o métrica, descubra la forma de los datos en sus dimensiones. Guiado por la cardinalidad, puedes ver los diferentes puntos de vista de cualquier dato.
    4. ¿Encontraste algo relevante? Guárdelo en un dashboard o compártalo con sus colegas.
  </Collapser>

  <Collapser
    className="freq-link"
    id="use-case-new-data"
    title="Valida tus datos"
  >
    <DNT>
      **I changed a custom event/metric and need to check if this change has been successful.**
    </DNT>

    1. En métricas y eventos, seleccione la cuenta, tipo de datos y evento/métrica al que realizó cambios.
    2. Verifique que la entidad esté reportando datos y que se estén trazando todos los atributos.
    3. Busque el atributo al que realizó cambios y verifique que la actualización se haya realizado correctamente.
  </Collapser>

  <Collapser
    className="freq-link"
    id="use-case-troubleshooting"
    title="Resolución de problemas problemas de datos"
  >
    <DNT>
      **I know there's something off with an event/metric from an alert or dashboard. I need to know the root cause about the event/metric/attribute behavior.**
    </DNT>

    1. En métrica y evento, use los menús para seleccionar el evento o métrica que no se comporta como se esperaba y deje que métrica y evento tracen ese gráfico.
    2. Desde allí, puedes profundizar en las dimensiones de esos datos y filtrar por aquellos atributos que sean relevantes.
    3. También podrás ver esos datos desde diferentes perspectivas: su distribución, ranking de valores o evolución en el tiempo.
    4. ¿Encontraste algo relevante? Guárdelo en un dashboard o compártalo con sus colegas.
  </Collapser>
</CollapserGroup>
