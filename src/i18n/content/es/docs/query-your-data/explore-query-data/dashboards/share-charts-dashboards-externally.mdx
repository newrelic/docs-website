En New Relic, puedes compartir dashboards y gráficos con espectadores fuera de tu organización, incluso si no tienen una cuenta de New Relic . Esta característica amplía la audiencia de su información valiosa al tiempo que mantiene los controles de seguridad y las capacidades de auditoría. Se puede acceder al dashboard y a los gráficos compartidos a través de URL en tiempo real de acceso público que usted crea desde la plataforma New Relic .

Cuando un usuario externo accede al dashboard o gráficos compartidos, la página se muestra con los mismos datos y visualizaciones que la entidad original, con las siguientes capacidades interactivas:

* **Intervalo de tiempo**: los espectadores pueden elegir el intervalo de tiempo de un menú desplegable para ver los datos de un periodo específico.
* **Modo de visualización**: los espectadores pueden seleccionar el modo claro u oscuro según sus preferencias.
* **Modo TV**: los espectadores pueden habilitar un modo de TV de pantalla completa que optimiza el dashboard para su visualización en una pantalla de televisión.

<Callout variant="important">Si bien la función para compartir gráficos está disponible para los clientes de **Pro**, **Enterprise** y **Core Compute**, la función para compartir paneles solo está disponible para los clientes de **Pro** y **Enterprise** con **[Advanced Compute](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing)** habilitado.Para obtener más información sobre los costos asociados y las prácticas de gestión para la función de compartir dashboard , consulte la sección [Precios](#public-dashboard-pricing) .</Callout>

<Callout variant="caution">Cualquier persona que tenga las URL dashboard o del gráfico en tiempo real puede ver toda la información del dashboard o del gráfico. Comparta la información con cuidado y de acuerdo con las políticas y procedimientos internos de su compañía.</Callout>

## Requisitos previos [#prerequisites-public-dashboard]

Esta característica incluye múltiples capas y condiciones de seguridad para garantizar un intercambio de datos controlado y seguro. Para compartir el dashboard o los gráficos públicamente, cerciorar de que la siguiente configuración esté establecido. Consulte [Gestión de seguridad y acceso](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts) para obtener detalles de configuración:

* La licencia `Live URL control` está habilitado para todas las cuentas que obtienen acceso al dashboard.
* El rol y las licencias personalizadas están configurados para que usted pueda compartir dashboards y gráficos.
* Las cuentas que generan el dashboard o el gráfico tienen licencias para que su función comparta datos externamente.
* Tienes acceso a todas las cuentas que generan el dashboard o el gráfico.
* Usted es el propietario del dashboard o gráfico que desea compartir.
* El [licenciadashboard ](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-permissions)no está establecido en <DNT>**Private**</DNT>.

      ## Comparte tu dashboard [#share-dashboard]

      Puede compartir su dashboard generando una URL dashboard en tiempo real y gestionando su duración de vencimiento. Además, puedes ver la lista de URL generadas con detalles y revocarlas en cualquier momento desde la plataforma New Relic. Para garantizar la seguridad y el cumplimiento, monitorear el registro de auditoría para crear, actualizar o revocar las URL dashboard compartido.

      <Callout variant="tip">Puede crear y gestionar URL dashboard en tiempo real a través de la **UI** y la [API de NerdGraph](/docs/apis/nerdgraph/examples/manage-live-dashboard-urls-via-api).</Callout>

      <CollapserGroup>
        <Collapser id="create-dashboard-url" title="Crear una URL dashboard en tiempo real desde la UI">
          Para crear una URL activa y de acceso público para su dashboard:

          1. Vaya a <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT>.

          2. Abra el dashboard que desea compartir.

          3. En la esquina superior derecha, haga clic en el ícono <DNT>**Share**</DNT> .

          4. Seleccione <DNT>**Share dashboard**</DNT>. Aparece un mensaje de confirmación para reconocer lo siguiente:

             * El dashboard no contiene información confidencial.
             * Acceder a este dashboard compartido desde la URL en tiempo real cada vez puede consumir su CCU.

          5. Haga clic en <DNT>**Continue**</DNT>. Aparece el cuadro de diálogo <DNT>**Share dashboard**</DNT> .

          6. Asigna una duración para que la URL esté activa. La URL caduca luego de la duración especificada.

          7. Haga clic en <DNT>**Save changes**</DNT>. Aparece una notificación sobre la creación exitosa de la URL y se copia al portapapeles.

          8. Para visualizar la página dashboard compartido, haga clic en la notificación o navegue manualmente hasta ella en una nueva pestaña con la URL copiada.

          <Callout variant="important">
            Cualquier actualización de contenido realizada en el dashboard real de la plataforma New Relic se refleja automáticamente en la página dashboard compartido. Para evitar la exposición accidental de datos, solo el propietario dashboard puede actualizar un dashboard compartido públicamente.
          </Callout>
        </Collapser>

        <Collapser id="update-live-url" title="Actualizar la URL dashboard en tiempo real">
          Cerciorar de tener los [licencias](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#permissions-to-custom-role) necesarios y de ser el propietario del dashboard para actualizar las propiedades (duración en tiempo real) del dashboard compartido.

          1. Vaya a <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT>.
          2. Abra el dashboard que está compartiendo.
          3. Haga clic en el icono <DNT>**Share**</DNT> .
          4. Seleccione <DNT>**Manage public link**</DNT>. Aparece el cuadro de diálogo <DNT>**Share dashboard**</DNT> .
          5. Actualice la duración desde el menú desplegable <DNT>**Link expiration**</DNT> . La opción <DNT>**Never expires**</DNT> mantiene la URL activa durante 30 años.
          6. Haga clic en <DNT>**Save changes**</DNT>.
        </Collapser>

        <Collapser id="view-dashboard-links" title="Ver el dashboard compartido">
          Puede acceder a los nombres de dashboard compartidos hipervinculados con sus URL activas, ubicados en New Relic One. Dependiendo de sus [licencias](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#permissions-to-custom-role), puede ver las URL que creó o las creadas por todos los usuarios de la cuenta.

          Para ver los enlaces dashboard compartido:

          1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
          2. En la navegación de la izquierda, haga clic en su nombre en la parte inferior.
          3. Navegar a <DNT>**Administration &gt; Shared Dashboards**</DNT>. La página Dashboards compartidos proporciona una lista de dashboards compartidos, incluidas sus fechas de creación y vencimiento, y el propietario de cada dashboard.

          Puede hacer clic en los nombres dashboard para abrir las páginas para compartir en una nueva pestaña.
        </Collapser>

        <Collapser id="revoke-dashboard-url" title="Revocar la URL de un dashboard en tiempo real">
          Dependiendo de sus [licencias](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#permissions-to-custom-role), puede revocar los enlaces que creó o los creados por todos los usuarios de la cuenta. Puedes hacerlo de las siguientes maneras:

          * En el dashboard que estás compartiendo, haz clic en el ícono <DNT>**Share**</DNT> y selecciona <DNT>**Manage public link**</DNT>. En el cuadro de diálogo <DNT>**Share dashboard**</DNT> , seleccione <DNT>**Revoke public link**</DNT>.
          * Desde la página <DNT>**[Shared Dashboards](#view-dashboard-links)**</DNT>, seleccione el <Icon name="fe-more-horizontal" /> icono para el enlace dashboard y haga clic en <DNT>**Revoke**</DNT>.

          <Callout variant="tip">
            Sólo el propietario puede eliminar un dashboard con un enlace compartido públicamente. Al eliminar el dashboard se revoca automáticamente el enlace para compartir público. Si restaura el dashboard, deberá crear una nueva URL activa.
          </Callout>
        </Collapser>

        <Collapser id="audit-trail-dashboard" title="Auditoría de pista y logging">
          La creación, actualización o revocación de las URL dashboard compartido genera un registro de auditoría en [NrAuditEvent](https://docs.newrelic.com/docs/accounts/accounts/account-maintenance/query-account-audit-logs-nrauditevent/). Para ver los registros en el generador de consultas:

          1. Vaya a <DNT>**[one.newrelic.com &gt; Query Your Data](https://one.newrelic.com/nr1-core)**</DNT>.
          2. Seleccione la cuenta desde la cual se crean los enlaces dashboard público.
          3. Ejecute cualquiera de las siguientes consultas para ver los registros de auditoría:

          <CollapserGroup>
            <Collapser id="url-last-hour" title="Obtenga detalles de las URL dashboard en tiempo real creadas o revocadas en la última hora.">
              Para obtener los detalles de las URL dashboard en tiempo real creadas o revocadas en la última hora, ejecute la siguiente consulta:

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard.%'
              ```
            </Collapser>

            <Collapser id="url-created-last-hour" title="Obtenga detalles de las URL dashboard en tiempo real creadas en la última hora.">
              Para obtener los detalles de las URL dashboard en tiempo real creadas en la última hora, ejecute la siguiente consulta:

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard.create%'
              ```
            </Collapser>

            <Collapser id="url-revoked-last-hour" title="Obtenga detalles de las URL dashboard en tiempo real revocadas en la última hora.">
              Para obtener los detalles de las URL dashboard en tiempo real revocadas en la última hora, ejecute la siguiente consulta:

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard.revoke%'
              ```
            </Collapser>

            <Collapser id="url-from-specific-user" title="Obtenga detalles de las URL dashboard en tiempo real creadas por un usuario específico en la última hora.">
              Para obtener los detalles de las URL dashboard en tiempo real creadas por un usuario específico en la última hora, reemplace `<user_email>` con el ID de email del usuario real y ejecute la siguiente consulta:

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard%'
              AND actorType='user'
              AND actorEmail=<user_email>
              ```
            </Collapser>

            <Collapser id="url-in-account" title="Obtenga todas las URL dashboard en tiempo real creadas en una cuenta en la última hora.">
              Para obtener los detalles de las URL dashboard en tiempo real creadas en una cuenta en la última hora, reemplace `<account_id>` con el ID de la cuenta real y ejecute la siguiente consulta:

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard%'
              AND scopeType='account'
              AND scopeId=<account_id>
              ```
            </Collapser>

            <Collapser id="url-count" title="Obtenga la lista de dashboards desde los cuales se generan las URL en tiempo real en un periodo de tiempo específico.">
              Para obtener la lista de dashboards desde los cuales se crean las URL en tiempo real en un periodo de tiempo específico, ejecute la siguiente consulta. Muestra la cantidad de URL creadas para cada dashboard.

              <Callout variant="tip">
                La siguiente consulta de ejemplo recupera registros de la última semana. Puede ajustar el periodo de tiempo según sea necesario.
              </Callout>

              ```sql
              WITH aparse(description, '%for dashboard *') AS dashboardGUID
              FROM NrAuditEvent
              SELECT count(*) AS 'dashboards'
              WHERE actionIdentifier LIKE '%public_dashboard.create%'
              FACET dashboardGUID
              SINCE 1 week ago
              ```
            </Collapser>

            <Collapser id="operations-count" title="Obtenga la lista de operaciones realizadas con las URL dashboard en tiempo real en un periodo de tiempo específico.">
              Para obtener la lista de operaciones realizadas con las URL dashboard en tiempo real en un periodo de tiempo específico, ejecute la siguiente consulta.

              <Callout variant="tip">
                La siguiente consulta de ejemplo recupera registros de la última semana. Puede ajustar el periodo de tiempo y la faceta según sea necesario.
              </Callout>

              ```sql
              WITH aparse(description, '%for dashboard *') AS dashboardGUID,
              aparse(actionIdentifier, '%public_dashboard.*') AS publicDashOperation
              FROM NrAuditEvent
              SELECT uniqueCount(actionIdentifier) AS 'number of operations'
              WHERE publicDashOperation IS NOT NULL
              FACET actorEmail, dashboardGUID, publicDashOperation
              SINCE 1 week ago
              ```
            </Collapser>
          </CollapserGroup>
        </Collapser>

        <Collapser id="public-dashboard-pricing" title="Precios">
          Al emplear URL dashboard en tiempo real, es importante comprender los costos asociados y cómo gestionarlos de manera efectiva. <DNT>**Public Dashboards**</DNT> es una característica avanzada del producto Calcular que se ofrece como [complemento](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing) o como parte del modelo de precios de Calcular. [Los CCU](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#compute-capacity-unit) avanzados se consumen cuando se ejecutan consultas desde las visualizaciones dashboard , lo que sucede en los siguientes casos:

          * Al cargar la página
          * En el selector de tiempo cambia
          * Acerca de los activadores de frecuencia de actualización

          Puede monitorear sus costos de uso desde **[New Relic Administration](https://one.newrelic.com/admin-portal) &gt; Compute Management &gt; Advanced capabilities**.

          ### Prácticas de gestión de costos

          Para mantener sus precios óptimos, considere las siguientes prácticas estándar:

          * [**Control de acceso**](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts): gestione quién puede crear, editar o eliminar enlaces tanto a nivel de cuenta como de rol de usuario. Esto ayuda a controlar y evitar que el usuario tenga acceso excesivo a estas capacidades.

          * [**Gestión de caducidad de enlaces**](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#update-live-url): establezca tiempos de caducidad para los enlaces para garantizar que solo sean visibles durante un periodo determinado, lo que reduce el posible uso excesivo.

          * **Gestión de widgets**:

            * Agrega o elimina widgets de tu dashboard interno para optimizar la consulta de datos.
            * Actualizar el widget para ignorar los cambios en el selector de tiempo, evitando consultas adicionales cuando el usuario ajusta el rango de tiempo.
            * Gestione la frecuencia de actualización de cada widget para evitar actualizaciones automáticas, alinear con las preferencias de su dashboard interno. Para obtener más detalles, consulte [Personalizar las frecuencias de actualización de los gráficos](/docs/query-your-data/explore-query-data/use-charts/chart-refresh-rates/#customize-chart-refresh).
        </Collapser>
      </CollapserGroup>

      ### Limitaciones

      Las siguientes son las limitaciones para compartir el dashboard públicamente:

      * Al visualizar un dashboard compartido en <DNT>**TV mode**</DNT>, este permanece activo durante hasta 72 horas. El espectador debe actualizar el browser para continuar viendo.
      * Si comparte un dashboard con un widget que usa variables de plantilla, vínculos de facetas, visualizaciones personalizadas o tablas de log, estos widget no mostrarán ninguna información en la vista compartida.
      * Si la configuración de licencias del dashboard es <DNT>**Private**</DNT>, no podrás compartir el dashboard públicamente. Para verificar esto, navegue a <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT> &gt; Your dashboard &gt; <Icon name="fe-more-horizontal" /> icono &gt; <DNT>**Settings &gt; Permissions**</DNT>.

      ## Comparte tu gráfico [#share-chart-externally]

      Para compartir un gráfico con un usuario externo, cerciorar de completar todos los pasos de configuración descritos en [Requisitos previos](#prerequisites-public-dashboard).

      <Callout variant="tip">
        Puede crear y gestionar URL de gráficos en tiempo real a través de la UI y la [APIde NerdGraph](/docs/apis/nerdgraph/examples/manage-live-chart-urls-via-api/).
      </Callout>

      Para generar un enlace para compartir para su gráfico:

      1. Vaya a <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT>.
      2. Abra el dashboard en el que está disponible el gráfico.
      3. En la esquina superior derecha del gráfico, haga clic en el <Icon name="fe-more-horizontal" /> icono.
      4. Seleccione <DNT>**Get chart link**</DNT>. Aparece una notificación confirmando la creación exitosa del enlace público, que también se copia en el portapapeles.
      5. Para abrir la página del gráfico en tiempo real, haga clic en la notificación o búsquela manualmente en una nueva pestaña con la URL copiada.
