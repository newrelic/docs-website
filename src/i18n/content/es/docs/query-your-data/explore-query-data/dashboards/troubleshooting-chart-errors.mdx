## Problemas de enlaces y descripciones de widgets [#widget-link-issues]

Si tiene problemas con los enlaces o las descripciones de los widgets, consulte los siguientes problemas comunes y soluciones.

<table>
  <thead>
    <tr>
      <th style={{ width: "400px" }}>
        <DNT>
          **Problem**
        </DNT>
      </th>

      <th style={{ width: "400px" }}>
        <DNT>
          **Solution**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        No se puede hacer clic en el enlace del widget
      </td>

      <td>
        Verifique que la URL sea válida y utilice un protocolo admitido (`http://` o `https://`). No se admiten enlaces con otros protocolos.
      </td>
    </tr>

    <tr>
      <td>
        La información sobre herramientas de la descripción no aparece
      </td>

      <td>
        Asegúrese de que el campo de descripción contenga texto. Señale el área del título del widget para mostrar la descripción emergente. Si sigue sin aparecer, intente actualizar el dashboard.
      </td>
    </tr>

    <tr>
      <td>
        Error de validación de URL
      </td>

      <td>
        Verifique que la URL no exceda los 2048 caracteres y utilice solo los protocolos `http://` o `https://`.
      </td>
    </tr>

    <tr>
      <td>
        El enlace lleva a un error 404
      </td>

      <td>
        Verifique que la URL de destino sea correcta y accesible. Los enlaces externos deben estar disponibles públicamente o ser accesibles dentro de su red.
      </td>
    </tr>

    <tr>
      <td>
        No se puede editar el enlace o la descripción del widget
      </td>

      <td>
        Confirme que tiene permisos de edición para el dashboard. Solo los usuarios con los permisos adecuados pueden modificar la configuración del widget.
      </td>
    </tr>

    <tr>
      <td>
        La mutación de la API no devuelve errores, pero los cambios no aparecen
      </td>

      <td>
        Actualice la página del dashboard. Si los cambios aún no aparecen, verifique que el ID del widget sea correcto y que la mutación se haya completado con éxito utilizando el explorador de la API NerdGraph.
      </td>
    </tr>
  </tbody>
</table>

## Problemas de variables ocultas [#hidden-variable-issues]

Si tiene problemas con las variables de plantilla ocultas, consulte los siguientes problemas y soluciones comunes.

<table>
  <thead>
    <tr>
      <th style={{ width: "400px" }}>
        <DNT>
          **Problem**
        </DNT>
      </th>

      <th style={{ width: "400px" }}>
        <DNT>
          **Solution**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        ¿No ve una variable que necesita?
      </td>

      <td>
        Ingrese al modo de edición para ver todas las variables, incluidas las ocultas. Si necesita interactuar con la variable regularmente, considere mostrarla.
      </td>
    </tr>

    <tr>
      <td>
        Se ocultó una variable accidentalmente
      </td>

      <td>
        Ingrese al modo de edición, busque la variable con el icono de visibilidad desactivada y desactive la configuración <DNT>**Hide on variables bar**</DNT>.
      </td>
    </tr>

    <tr>
      <td>
        Las variables ocultas no funcionan en las llamadas de API
      </td>

      <td>
        Verifique que está usando el campo `hiddenOnVariablesBar` en el objeto `options`. Este campo debería estar disponible en el tipo `VariableOptions`.
      </td>
    </tr>

    <tr>
      <td>
        ¿Desea ocultar varias variables a la vez?
      </td>

      <td>
        Actualmente, debe ocultar las variables una por una a través de la interfaz de usuario. Para operaciones masivas, utiliza la [API de dashboards de NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-dashboards/#variable-visibility) para actualizar múltiples variables programáticamente.
      </td>
    </tr>
  </tbody>
</table>
