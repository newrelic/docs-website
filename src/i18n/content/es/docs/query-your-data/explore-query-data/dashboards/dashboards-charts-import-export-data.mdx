Puede emplear [el dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) para visualizar y realizar un seguimiento del estado de su sistema, abordar problemas y planear pasos adicionales. Aprenda a compartir toda esa información con su equipo, clientes u otras partes interesadas con varias opciones de importación y exportación.

## Importar un dashboard como JSON [#import-json]

Puede importar un dashboard como JSON cuando desee modificar un dashboard listo para usar a nivel de código. Por ejemplo, si desea crear un dashboard personalizado fusionando dos dashboards existentes, puede modificar el JSON de un dashboard base y luego importarlo nuevamente a New Relic.

Para importar un dashboard como JSON:

1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT>.

2. Abra el JSON de un dashboard existente haciendo clic en el icono <DNT>**...**</DNT> y luego seleccione <DNT>**Copy JSON**</DNT>.

3. Vuelva a la página <DNT>**Dashboards**</DNT> .

4. En la esquina superior derecha, haga clic en <DNT>**Import dashboard**</DNT>.

5. Pega tu código JSON.

6. Elija la configuración de cuenta y licencias para el dashboard. No puede cambiar la cuenta una vez que la configuró, pero puede [cambiar las licencias](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-settings) en cualquier momento.

7. Haga clic en <DNT>**Save**</DNT>.

## Exportar un dashboard como PDF [#export-dashboards-pdf]

Para exportar su dashboard como un archivo PDF, en la esquina superior derecha del dashboard, haga clic en el icono <DNT>**...**</DNT> y seleccione <DNT>**Export dashboard as PDF**</DNT>.

<img title="Export dashboard as PDF" alt="Screenshot of how to export a dashboard as a PDF file" src="/images/dashboards_screenshot-full_export-pdf.webp" />

<figcaption>
  Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT> y abra un dashboard existente.

Considere lo siguiente al exportar un dashboard como PDF a través de la UI:

* No se admiten [visualizaciones personalizadas](/docs/query-your-data/explore-query-data/dashboards/add-custom-visualizations-your-dashboards/).

Para exportar como PDF o PNG con opciones adicionales, como dimensiones personalizadas y anulaciones de variables de plantilla, use la [API de NerdGraph](#export-api).

## Exportar dashboard y gráficos mediante la API de NerdGraph [#export-api]

Use la API de NerdGraph para exportar mediante programación dashboards y widgets de gráficos individuales como imágenes de instantáneas. La API admite opciones que no están disponibles en la UI, incluidas dimensiones personalizadas, anulaciones de variables de plantilla y filtros de entidad.

### Exportar un dashboard como PDF o PNG [#api-dashboard-snapshot]

La mutación `dashboardCreateSnapshotUrl` genera una URL de instantánea para una página completa del dashboard.

<Callout variant="tip">
  El parámetro `guid` requiere el GUID de la **página**, no el GUID del dashboard. Un dashboard puede tener varias páginas. Para encontrar los GUID de la página, use la consulta `entitySearch` o abra la página del dashboard y seleccione <DNT>**Manage JSON**</DNT> en el menú <DNT>**...**</DNT>.

  ```graphql
  {
    actor {
      entitySearch(query: "parentId ='YOUR_DASHBOARD_GUID' AND tags.isDashboardPage = 'true'") {
        results {
          entities {
            guid
            name
          }
        }
      }
    }
  }
  ```
</Callout>

**Esquema de mutación:**

```graphql
mutation {
  dashboardCreateSnapshotUrl(
    guid: EntityGuid!
    params: SnapshotUrlInput
  ): String
}

input SnapshotUrlInput {
  timeWindow: SnapshotUrlTimeWindowInput
  display: SnapshotUrlDisplayInput
  variables: [SnapshotUrlVariableInput!]
  filter: String
  format: SnapshotUrlFormat
}

input SnapshotUrlTimeWindowInput {
  beginTime: EpochMilliseconds
  endTime: EpochMilliseconds
  duration: Milliseconds
}

input SnapshotUrlDisplayInput {
  width: Int
  height: Int
}

input SnapshotUrlVariableInput {
  name: String!
  values: [String!]!
}

enum SnapshotUrlFormat {
  PDF
  PNG
}
```

### Obtener una imagen de gráfico [#api-widget-snapshot]

La mutación `dashboardWidgetCreateSnapshotUrl` genera una URL de instantánea PNG para un solo widget de gráfico. El widget no necesita existir en un dashboard — puede definirlo en línea, lo que resulta útil para generar imágenes de gráficos de forma programática.

**Esquema de mutación:**

```graphql
mutation {
  dashboardWidgetCreateSnapshotUrl(
    widget: DeclarativeUiWidget!
  ): WidgetSnapshotResult
}

type WidgetSnapshotResult {
  url: String!
}
```

El parámetro `widget` acepta un `DeclarativeUiWidget` — un objeto JSON con tres capas anidadas:

```
declarative/widget envelope
└── widget node        ← title and widget-level props
    └── visualization  ← visualization type and configuration
```

```json
{
  "type": "declarative/widget",
  "version": 1,
  "content": {
    "type": "widget",
    "props": {
      "title": "My Chart Title"
    },
    "content": {
      "type": "visualization",
      "id": "viz.line",
      "props": {
        // visualization-specific config (nrqlQueries, colors, thresholds, etc.)
      }
    }
  }
}
```

Si está construyendo la definición del widget desde un dashboard que ha exportado como JSON, utilice este mapeo de campos:

</Callout>

## Exportar un gráfico como CSV [#export-charts-csv]

Para exportar el gráfico de tabla de dashboard como un archivo CSV, en la esquina superior derecha del gráfico, haga clic en <DNT>**...**</DNT> y seleccione <DNT>**Export as CSV**</DNT>.

<img title="Export chart as CSV" alt="Screenshot of how to export a table chart as a CSV file" src="/images/dashboards_screenshot-crop_export-csv.webp" />

Tenga en cuenta lo siguiente al exportar un gráfico como CSV:

* Los datos siempre se exportan en hora UTC
* Exportar no admite funciones de agregador como `average()`
* La exportación no admite la cláusula `COMPARE_WITH`
* En algunos escenarios, la UI puede realizar un formateo adicional de los datos que no se replican en el CSV exportado, lo que puede generar pequeñas discrepancias (como -1.1E-17 se representa como 0 en la UI pero no en el CSV).
* No es posible exportar los resultados de una consulta entre cuentas.
