La característica de configuración de sesión de New Relic le permite establecer límites de tiempo de inactividad antes de que las sesiones del browser de su usuario caduquen automáticamente.

## Requisitos [#requirements]

Si utiliza nuestro [modelo de usuario más nuevo](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), consulte [Configuración de sesión](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#session-mgmt).

## Descripción general [#overview]

La configuración de sesión le permite establecer límites de tiempo de inactividad antes de que las sesiones browser de su usuario caduquen automáticamente. Aparece un mensaje tres minutos antes de que el sistema cierre la sesión. El usuario deberá volver a iniciar sesión para continuar. Para las cuentas configuradas con [SAML Single Sign On (SSO)](/docs/subscriptions/saml-service-providers), hay una opción adicional disponible para establecer la frecuencia con la que se vuelven a autenticar las sesiones del browser del usuario.

Usuario y usuario restringido pueden ver el período de tiempo de espera automático, pero no pueden cambiarlo. Para ver el valor del tiempo de espera: vaya a <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Authentication > Session configuration**</DNT>.

## Seleccione el valor del tiempo de espera de la sesión [#procedures]

El proceso para seleccionar el valor del tiempo de espera de la sesión es el mismo para la configuración SAML y no SAML. Para obtener opciones de configuración de SAML adicionales, consulte [Reautenticación browser SAML SSO](/docs/accounts-partnerships/accounts/account-maintenance/set-session-timeouts#saml_reauthentication).

Para seleccionar un período predefinido para los tiempos de espera de sesión con cuentas SAML SSO, el propietario de la cuenta debe haber identificado previamente la URL de cierre de sesión en los ajustes de configuración de SAML SSO. Si esto no se ha configurado, el administrador de la cuenta puede ver la barra deslizante de tiempo de espera de la sesión, pero no cambiarla.

Si el administrador también es el propietario de la cuenta, <DNT>**Session configuration**</DNT> incluye un enlace para ir directamente a la configuración SAML SSO de New Relic e identificar la URL de cierre de sesión. Para obtener más información, consulte [Configuración de SSO](/docs/subscriptions/setting-up-sso#timeout_url).

Para seleccionar un período predefinido para los tiempos de espera de sesión para los usuarios en nuestro [modelo de usuario original](#requirements):

1. Vaya a:

   <DNT>
     **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Authentication > Session configuration**
   </DNT>

   .

2. Utilice la barra deslizante para seleccionar un período de tiempo para que las sesiones inactivas caduquen y log sesión automáticamente.

3. Opcional: seleccione la opción de casilla de verificación si no desea que caduquen las sesiones browser del usuario restringido.

4. Seleccione

   <DNT>
     **Save my changes**
   </DNT>

   .

Los cambios entran en vigor inmediatamente.

## Seleccione la reautenticación browser SAML SSO [#saml_reauthentication]

Para seleccionar un período predefinido para que [las sesiones browser autenticadas mediante SAML SSO](/docs/subscriptions/setting-up-sso) se vuelvan a autenticar:

1. Vaya a:

   <DNT>
     **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Authentication > Session configuration**
   </DNT>

   .

2. Utilice la barra deslizante

   <DNT>
     **SAML re-authentication time**
   </DNT>

   para seleccionar un período de tiempo para que New Relic verifique la sesión browser .

3. Seleccione

   <DNT>
     **Save my changes**
   </DNT>

   .

## Redirigir después del tiempo de espera de SAML [#saml_timeout]

Si cierra la sesión debido a un tiempo de espera de inactividad en una cuenta configurada para SAML, se le enviará a la página de inicio de sesión de New Relic. Debido a que su cuenta está configurada para SAML, no tiene un inicio de sesión directo en New Relic. Para ser redirigido a su proveedor SAML para autenticación:

1. Introduzca su dirección de correo electrónico en el campo

   <DNT>
     **Email**
   </DNT>

   .

2. Deje el campo

   <DNT>
     **Password**
   </DNT>

   en blanco.

3. Haga clic en el botón

   <DNT>
     **Sign In**
   </DNT>

   .

Luego será redirigido a su proveedor SAML. Una vez reautorizado, regresará al sitio web de New Relic.
