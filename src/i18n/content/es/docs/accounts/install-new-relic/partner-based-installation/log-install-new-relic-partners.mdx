Si se registró en New Relic a través de uno de los socios de New Relic, es posible que deba seguir procedimientos adicionales de inicio de sesión único (SSO) para log sesión y comenzar a usar New Relic. También es posible que se necesiten pasos especiales para completar el proceso de implementación. Este documento proporciona procedimientos básicos de inicio de sesión para cuentas creadas a través de socios de New Relic, con enlaces a recursos adicionales según corresponda.

## Cree una cuenta New Relic (todos los socios) [#partner-account]

Si recibió una oferta especial de un socio de New Relic, siga el enlace del socio directamente para crear una cuenta de New Relic.

## Iniciar sesión y desplegar New Relic (la mayoría de los socios) [#partner-users]

Recibirá un mensaje de confirmación por correo electrónico con un enlace para confirmar su cuenta, iniciar sesión en New Relic y completar el proceso de implementación.

## Inicie sesión y despliegue New Relic (socios especiales) [#partner-users]

Estos socios requieren procedimientos de inicio de sesión y despliegue específicos para que usted pueda utilizar su cuenta New Relic.

<Callout variant="tip">
  Si se registró para obtener una cuenta de New Relic a través de un socio que no figura aquí, [siga los procedimientos estándar](#partner-users) para iniciar sesión y desplegar New Relic.
</Callout>

### Socios N - Z [#n-z-partners]

<CollapserGroup>
  <Collapser
    id="network-redux"
    title="Pantheon"
  >
    Para crear una cuenta de New Relic a través de [Pantheon](https://pantheon.io/features/new-relic) e iniciar sesión en New Relic:

    1. Desde [la página de socio de Pantheon con New Relic](https://pantheon.io/features/new-relic), seleccione el enlace <DNT>**sign up**</DNT> .

    2. Siga las instrucciones en línea para completar el proceso de instalación.

    3. Espere a que New Relic comience a recopilar datos para su aplicación.

       Después de unos minutos, podrá comprobar el rendimiento de su aplicación en la [página APM <DNT>**Overview**</DNT> ](/docs/apm/applications-menu/monitoring/applications-overview-dashboard). Para obtener más información, consulte [la documentación de Pantheon](https://pantheon.io/docs/new-relic/).
  </Collapser>

  <Collapser
    id="rackspace"
    title="Herramientas en la nube de Rackspace"
  >
    Para crear una cuenta a través de [Rackspace Cloud Tools](https://www.rackspace.com) e iniciar sesión en New Relic, siga estos procedimientos básicos. Para obtener más información, consulte [el complemento Rackspace Cloud Load Balancer](/docs/accounts-partnerships/accounts/install-new-relic/partner-based-installation/rackspace-cloud-load-balancer-plugin).

    1. Inicia sesión en su cuenta de CloudTools en [cloudtools.rackspace.com/myapps](https://cloudtools.rackspace.com/myapps).

    2. Desde <DNT>**Use Your Applications**</DNT>, seleccione <DNT>**New Relic**</DNT>.

    3. Seleccione el idioma de su agente (Java, .NET, PHP, Python, Ruby) y luego siga las instrucciones en línea para completar el proceso de instalación.

    4. Espere a que New Relic comience a recopilar datos para su aplicación.

       Espere unos minutos y luego verifique el rendimiento de su aplicación en la [página APM <DNT>**Overview**</DNT> ](/docs/apm/applications-menu/monitoring/applications-overview-dashboard).
  </Collapser>

  <Collapser
    id="wordpress-w3-edge"
    title="W3 Edge y WordPress"
  >
    <Callout variant="tip">
      Para crear una cuenta a través de [W3 Total Cache](https://wordpress.org/plugins/w3-total-cache/) e iniciar sesión en New Relic, debe instalar y ejecutar el agente PHP de W3, no solo obtener una clave de API.
    </Callout>

    Desplegar el agente requiere <DNT>**administrative access**</DNT>, que incluye:

    * Acceso SSH/línea de comando

    * Acceso root o sudo para ejecutar el instalador de New Relic y reiniciar su servidor web

      Si tiene un paquete económico, de alojamiento compartido o de blogger, o si no está seguro de cuáles son los permisos de su cuenta, comuníquese con su proveedor de alojamiento. Por ejemplo, si su sitio de WordPress está en servidores compartidos, es posible que no pueda instalar New Relic. Su proveedor debería poder brindarle información sobre los permisos de su cuenta o actualizar su cuenta si es necesario.

      Para instalar el complemento New Relic para usuarios de W3 Edge/Wordpress:

    1. log sesión en su cuenta de WordPress en [wordpress.org/support/bb-login.php](https://wordpress.org/support/bb-login.php).

    2. Siga [las instrucciones de New Relic](/docs/agents/php-agent/installation/wordpress-users-new-relic) para completar el proceso de instalación o comuníquese con [el soporte de W3TC](http://wordpress.org/support/plugin/w3-total-cache) para obtener ayuda.

    3. Espere a que New Relic comience a recopilar datos.

       Unos minutos después de desplegar el agente, los datos de rendimiento de su aplicación aparecerán en la [página Descripción general de aplicaciones](/docs/apm/applications-menu/monitoring/applications-overview-dashboard)de APM.

       <Callout variant="tip">
         [WordPress](https://wordpress.org/plugins/rt-newrelic-browser/) también ofrece un complemento para nuestro [monitoreo de navegador](/docs/agents/php-agent/installation/wordpress-users-new-relic#browser).
       </Callout>
  </Collapser>
