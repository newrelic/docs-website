New Relic tiene dos modelos de usuario: un modelo de usuario más antiguo denominado <DNT>**original user model**</DNT> y nuestro modelo de usuario más nuevo, lanzado en julio de 2020.

## Determina tu modelo de usuario [#determine-user-model]

Cuando log sesión en New Relic, el registro de usuario asociado con ese inicio de sesión está en un modelo de usuario u otro. Para determinar en qué modelo de usuario estás:

* Vaya a la parte inferior izquierda y haga clic en el [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings).

* Si ve una pestaña

  <DNT>
    **Administration**
  </DNT>

  , está en nuestro modelo de usuario más nuevo. Si ve una pestaña

  <DNT>
    **Account settings**
  </DNT>

  , está en nuestro modelo de usuario original.

## Encuentre los documentos adecuados [#docs]

El modelo de usuario en el que se encuentra afecta la forma en que se administran sus usuarios ([lea sobre las diferencias del modelo de usuario](#differences)). Tenemos diferentes conjuntos de documentos para los dos modelos de usuario:

* [Documentos para nuestro modelo de usuario original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model)
* [Documentos para nuestro modelo de usuario más nuevo](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-managing-users)

¿No estás seguro de en qué modelo de usuario estás? Consulte [Determinar el modelo de usuario](#determine-user-model).

## UIde usuario de gestión de usuarios [#manage-users]

Para saber cómo utilizar la UI de usuario de administración de usuarios:

* Para usuarios de nuestro modelo de usuario más nuevo: consulte [Administrar usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#where).

* Para usuarios de nuestro modelo de usuario original: desde

  <DNT>
    **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**
  </DNT>

  , haga clic en el [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), haga clic en

  <DNT>
    **Account settings**
  </DNT>

  y luego haga clic en

  <DNT>
    **Users and roles**
  </DNT>

  . Para obtener información, consulte [Usuario original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/).

## Comparación de modelos de usuario [#differences]

Cuando inicia log en New Relic, el registro de usuario asociado con su inicio de sesión está en un modelo de usuario u otro. Aquí hay una explicación de las diferencias entre los modelos de usuario:

* <DNT>
    **Newer user model**
  </DNT>

  ([ver documentos](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-managing-users)): este modelo de usuario más nuevo y [mejorado](https://www.newrelic.com/blog/how-to-relic/user-management-and-access-security) se lanzó en julio de 2020. Es el modelo de usuario predeterminado para nuevos registros de New Relic. Estas son las principales diferencias con el modelo de usuario original:

  * Todas sus cuentas y usuarios están contenidos en una [organización](/docs/new-relic-account-structure) de alto nivel. Esto le da a la organización un mayor control sobre la gestión de los usuarios y a qué pueden acceder. Un impacto de esto es que los usuarios que trabajan con múltiples organizaciones New Relic pueden tener una dirección de correo electrónico asociada con [múltiples inicios de sesión](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data/#account-access).
  * El usuario de este modelo tiene un sistema diferente para gestionar permisos y acceso a la cuenta. [Conozca cómo funciona el acceso de usuarios.](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts)

* <DNT>
    **Original user model**
  </DNT>

  ([ver documentos](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/)): algunos aspectos de este modelo anterior que son diferentes de nuestro modelo más nuevo:

  * No había tanto control a nivel de organización sobre el usuario. Por ejemplo, un usuario de New Relic tenía la capacidad de acceder a varias cuentas a las que se les había otorgado acceso mediante un único inicio de sesión.
  * Los usuarios del modelo de usuario original tienen un [sistema de gestión de usuarios](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) diferente.

Para obtener más información sobre los beneficios del nuevo modelo, consulte nuestra [publicación de blog sobre los cambios en el modelo de usuario](https://www.newrelic.com/blog/how-to-relic/user-management-and-access-security) y una [explicación de la nueva estructura de cuenta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure). Para impactos y limitaciones, ver [impactos característicos](#limitations).

## Requisitos para nuestro modelo de usuario más nuevo [#requirements]

Estos son los factores que rigen qué organización tiene usuario en el modelo más nuevo:

* Todas las organizaciones de New Relic que se registraron después del 30 de julio de 2020 tienen usuario en este modelo (y también tienen el [modelo de precios más nuevo](#pricing-plans)).
* Algunas organizaciones antiguas de New Relic han migrado sus usuarios al nuevo modelo mediante New Relic o mediante el [procedimiento de migración de usuarios](/docs/accounts/original-accounts-billing/original-users-roles/user-migration).
* Algunos socios de New Relic (por ejemplo, revendedores, proveedores de servicios administrados) han migrado sus usuarios al nuevo modelo. (Si es una organización de New Relic con usuario en el modelo original y requiere arrendamiento múltiple, comuníquese con su representante de cuenta para obtener más información).

## Impactos característicos del modelo de usuario. [#limitations]

El nuevo modelo de usuario ofrece [muchos beneficios](https://newrelic.com/blog/how-to-relic/user-management-and-access-security). Aquí hay algunos impactos característicos que quizás le interese conocer antes de decidirse a cambiar:

* Para notificación de alerta clásica (no [notificación](/docs/alerts-applied-intelligence/notifications/intro-notifications) nueva): el usuario en el nuevo modelo no aparecerá como una opción al [elegir usuario para recibir notificación de alerta](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/#user). El resultado de esto es que estos usuarios no recibirán [la notificación push de la aplicaciónNew Relic Mobile ](/docs/mobile-apps/new-relic-mobile-apps/authentication-alerts/alerting-new-relic-mobile-apps). Como solución alternativa, puede utilizar su [dirección de correo electrónico](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/#email) como canal de notificación.
* No puede utilizar la API REST para [generar una lista de usuarios](/docs/apis/rest-api-v2/account-examples-v2/listing-users-your-account/) (solo usuario del modelo original). En su lugar, usaría la [UIde usuario de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#where).

## Relación entre modelo de usuario y modelo de precios [#relation-to-pricing]

El modelo de usuario no está directamente relacionado con nuestros dos modelos de precios. Para obtener información sobre cómo se relacionan, consulte [Modelo de precios y modelo de usuario](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models#pricing-user-table).
