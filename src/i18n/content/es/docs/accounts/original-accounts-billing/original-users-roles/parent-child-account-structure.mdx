La organización New Relic puede tener una estructura de cuenta padre/hijo. Esto es principalmente relevante para organizaciones cuyos usuarios están en nuestro [modelo de usuario original](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), pero puede aplicarse a organizaciones con usuarios en ambos modelos de usuario. Estas son las principales diferencias en cómo se les aplica la estructura padre/hijo:

* Modelo de usuario original: nuestro modelo de usuario original utiliza una estructura de cuenta principal/infantil, donde las cuentas secundarias heredan la configuración de las cuentas principales, incluidas las reglas relacionadas con el acceso de los usuarios a las cuentas. Nuestras [cuentas asociadas](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions) también utilizan este modelo. Si su organización tiene usuarios en este modelo, esos usuarios pueden [crear cuentas para niños](#creating). Si tiene un usuario en este modelo, [continúe leyendo a continuación](#hierarchy) para saber cómo funciona esta estructura de cuenta.
* Modelo de usuario más nuevo: nuestro modelo de usuario nuevo y mejorado. Las organizaciones con usuarios en este modelo pueden tener una estructura de cuenta principal/secundaria, pero ya no es relevante para administrar el acceso de los usuarios a las cuentas y se usa principalmente para determinar las relaciones de las cuentas para algunas características y para agregar la facturación a nivel de organización. Para obtener más información sobre cómo funciona el acceso a la cuenta para el modelo de usuario más nuevo, consulte [Estructura de la cuenta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure).

## Jerarquía de cuentas principales y secundarias [#hierarchy]

Para organizaciones con usuarios en nuestro [modelo de usuario original](#requirements), sus cuentas pueden tener una estructura de cuenta principal/infantil, con cada cuenta secundaria adjunta a una cuenta principal única. Cada cuenta puede tener varias aplicaciones que le reportan. Es posible que la cuenta principal también tenga una aplicación que le informe, pero no todos los datos enviados desde las aplicaciones de la cuenta infantil se acumulan en la cuenta principal.

Para saber cómo cambiar entre cuentas en la UI, consulte [Acceso a la cuenta](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data#account-access).

Para comprender cómo las cuentas principales y secundarias escalan con las cuentas de la región de la UE, consulte [Jerarquía de cuentas de la región de la UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#global-account-hierarchy).

<Callout variant="tip">
  Si tiene una estructura de cuenta de asociación, consulte [Cuentas de asociación](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions).
</Callout>

## Clave de licencia para cuentas infantiles [#keys]

En una organización de New Relic, su cuenta principal de New Relic sirve como cuenta principal con su propia [clave de licencia](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key). Cada cuenta infantil tiene su propia clave de licencia individual. Cada aplicación puede utilizar solo una clave de licencia, por lo que cada aplicación puede informar a una sola cuenta.

## Crear cuentas infantiles [#creating]

Para agregar cuentas a su organización New Relic, debe tener [la edición](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) Pro o Enterprise.

La forma de agregar una cuenta depende del [modelo de usuario](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) en el que se encuentren sus usuarios:

* Usuarios en el modelo de usuario más nuevo: consulte [Estructura de la cuenta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure/#new-model).

* Usuarios en modelo de usuario original: Siga estas instrucciones:

  1. Vaya a:

     <DNT>
       **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Summary**
     </DNT>

     .

  2. Haga clic en

     <DNT>
       **Add account**
     </DNT>

     .

  3. Escriba el nombre de la cuenta del niño y luego seleccione

     <DNT>
       **Add this account**
     </DNT>

     .

  4. Continúe con los procedimientos para agregar usuario a la cuenta infantil.

## Agregar usuario a cuentas [#users]

Para tener una organización New Relic con más de una cuenta, debes tener [la edición](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) Pro o Enterprise.

La forma de agregar un usuario a una cuenta infantil en su organización depende del [modelo de usuario](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) en el que se encuentre su usuario:

* Modelo de usuario más nuevo: se debe asignar un usuario a un grupo con acceso a la cuenta específica. Para obtener más información, consulte [conceptos de gestión de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts).

* Modelo de usuario original: en general, los permisos de un usuario en una cuenta principal se heredan automáticamente para ese usuario en cuentas secundarias. Para agregar usuario:

  1. Vaya a:

     <DNT>
       **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Summary**
     </DNT>

     .

  2. De la lista de cuentas infantiles, seleccione la cuenta infantil.

  3. En la sección

     <DNT>
       **Active users**
     </DNT>

     de la cuenta infantil, seleccione

     <DNT>
       **Add user**
     </DNT>

     , complete el correo electrónico, el nombre, la función y el título del usuario según corresponda y luego seleccione

     <DNT>
       **Add this user**
     </DNT>

     .

  4. Opcional: [cambie las funciones de propietario o administrador](/docs/accounts-partnerships/accounts/account-setup/adding-updating-users) de la cuenta infantil según corresponda.

## Configurar cuentas con SAML SSO [#saml]

La forma de configurar SAML SSO dependerá del [modelo de usuario](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) en el que se encuentren sus usuarios:

* Modelo de usuario original: si [configura SAML SSO](/docs/accounts-partnerships/accounts/saml-single-sign/setting-sso) en el nivel de cuenta principal, los usuarios de su cuenta secundaria también podrán iniciar sesión mediante SSO. Si

  <DNT>
    **don't**
  </DNT>

  configura SAML SSO en el nivel principal, puede configurarlo en el nivel de cuenta secundaria. Sin embargo,

  <DNT>
    **do not**
  </DNT>

  configura SAML SSO en ambos niveles. Para obtener más información, consulte [Configuración de SAML con varias cuentas](/docs/accounts-partnerships/accounts/saml-single-sign/configuring-saml-multiple-accounts).

* Modelo de usuario más nuevo: consulte [SAML SSO](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more).

## Actualizar aplicación de cuenta infantil [#subaccount_apps]

<DNT>
  **Requirements:**
</DNT>

* [Edición](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) Pro o Enterprise.
* Debes estar en nuestro [modelo de usuario original](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) (si estás en [nuestro modelo de usuario más nuevo](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) y deseas agregar cuentas, habla con tu representante de cuenta).
* Debe ser propietario o administrador.

Cuando cambia el archivo de configuración de la aplicación para moverla de una cuenta a otra, New Relic la trata como una nueva aplicación en la nueva cuenta infantil. New Relic no mueve ningún dato histórico de la aplicación.

Para instalar una nueva aplicación para una cuenta infantil:

1. Asegúrese de haber seleccionado la cuenta principal: desde el [selector de cuentas](/docs/new-relic-solutions/get-started/glossary/#account-switcher), seleccione la cuenta que desee.

2. En el [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), seleccione

   <DNT>
     **Account settings**
   </DNT>

   .

3. Siga los procedimientos estándar para instalar una nueva aplicación.

O

Cambie el <InlinePopover type="licenseKey"/>en el archivo de configuración de la aplicación existente para moverla a otra cuenta infantil.
