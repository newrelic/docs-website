<Callout variant="important">
  Este documento se aplica a organizaciones con usuarios en nuestro [modelo de usuario original](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).
</Callout>

Desde el 30 de julio de 2020, las nuevas suscripciones a New Relic tienen usuarios en nuestro nuevo modelo de usuario. Los clientes más antiguos que aún no han [migrado sus usuarios al nuevo modelo](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) todavía están en nuestro modelo de usuario original. El modelo de usuario original quedará cada vez más obsoleto a medida que más clientes de New Relic migren al nuevo modelo.

A continuación se muestran algunos aspectos importantes del modelo de usuario original y enlaces a documentos importantes:

* Nuestro modelo de usuario original no tenía límites firmes entre organización/clientes y estaba mucho más centrado en el usuario. Esto significaba que a un usuario se le podía otorgar acceso a datos de diferentes organizaciones y podía acceder a esos datos desde un único inicio de sesión. [Lea más sobre las diferencias entre el modelo antiguo y el nuevo](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).
* En el modelo de usuario original, el acceso de un usuario a las cuentas se basa en: a) ser agregado a una cuenta específica, ob) tener acceso a una cuenta principal y heredar el acceso a las cuentas secundarias de esa cuenta. Consulte nuestros [documentos originales de administración de usuarios](/docs/accounts/original-accounts-billing/original-users-roles). Vea una [explicación de cómo funcionan las cuentas padre/hijo](/docs/accounts/original-accounts-billing/original-users-roles/mastersub-account-structure).
