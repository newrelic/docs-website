Comprender la estructura de la organización y las cuentas de New Relic le ayudará a encontrar datos en New Relic y configurar el acceso de los usuarios.

## Organización y cuentas [#organization-accounts]

En New Relic, un <DNT>**organization**</DNT> representa clientes de New Relic. Su organización New Relic contiene todo lo relevante para usted y los miembros de su equipo: las cuentas, los usuarios y los datos de su organización.

Cuando te registras en New Relic, esto crea una organización que contiene una sola cuenta. Nuestras [ediciones Gratuita y Estándar](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing#editions) solo pueden tener una única cuenta. Pro y edición Enterprise la organización puede agregar cuentas.

Una cuenta puede tener muchas fuentes de informes de datos. Por ejemplo, una sola cuenta podría tener informes de datos de nuestro agente de infraestructura, un agente <InlinePopover type="apm" />y otra integración. Todos los datos reportados a New Relic requieren una [identificación de cuenta](/docs/accounts/accounts-billing/account-structure/account-id), que le indica a New Relic a qué cuenta pertenecen esos datos. Esa identificación también se usa para algunas tareas específicas de la cuenta, como realizar llamadas API.

Es posible que observe una cuenta especial en su lista de cuentas bajo el encabezado de <strong>la organización</strong> para algunas funciones relacionadas con la organización y en la experiencia de consulta. Esta cuenta almacena datos para la función de ámbito de la organización y usted empleará esta cuenta para cualquier consulta que emplee datos de la función de ámbito de la organización. No puedes cancelar ni ingerir datos en esta cuenta. Esto también estará en tu lista de cuentas cuando consultes NerdGraph. Se devolverá como `<organization ID> Storage Account`.

Cuando [configuras el acceso de usuario](#account-access), otorgas acceso de usuario a cuentas específicas.

## Acceso entre cuentas [#cross-account-access]

New Relic proporciona una característica multiplataforma que le permite compartir datos entre cuentas. Por ejemplo, puede asignar un usuario a varias cuentas, lo que comparte datos de esas cuentas con el usuario incluso si no es el propietario de la cuenta. Otras características como rastreo distribuido, custom <InlinePopover type="dashboards" />y carga de trabajo comparten datos de entidad monitor en otras cuentas, siempre y cuando tengas asignado acceso a esas cuentas.

Para obtener instrucciones sobre cómo realizar consultas entre cuentas empleando el generador de consultas o API de NerdGraph, o para configurar alertas entre cuentas, consulte la página [de Alertas entre cuentas](/docs/alerts/create-alert/create-alert-condition/cross-account-alert).

## Razones para agregar cuentas [#add-accounts]

El motivo por el que agregarías más cuentas depende de tus objetivos y tu estructura. Algunas razones para agregar cuentas incluyen:

* Separar los entornos de producción y de prueba. Por ejemplo, puede nombrar una cuenta `Invoice processing dev` y otra `Invoice processing prod`.
* Para facilitar la separación de la facturación para diferentes departamentos.
* Separar proyectos de cualquier tipo que tengan conjuntos de datos bastante distintos.
* Controlar el acceso de los usuarios a diferentes proyectos o conjunto de datos por motivos de seguridad. Por ejemplo, es posible que desee configurar una cuenta para contratistas externos, que solo contenga datos de determinada entidad, para limitar a qué tiene acceso ese grupo.

Para saber cómo agregar y cambiar el nombre de cuentas, consulte [Agregar cuentas](/docs/accounts/accounts-billing/account-structure/add-accounts).

## Cómo acceden los usuarios a las cuentas [#account-access]

En su organización, su usuario de New Relic tiene acceso a cuentas específicas que son relevantes para sus deberes y responsabilidades. Para administrar el acceso de los usuarios a las cuentas, usted [otorga a los grupos acceso a roles específicos en cuentas específicas](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts). Nuestro sistema de administración de usuarios le permite crear el acceso de usuario que necesita, ya sea una configuración relativamente simple con solo unos pocos roles en unas pocas cuentas o una configuración compleja con muchos roles en muchas cuentas. [Obtenga más información sobre la gestión de usuarios.](/docs/accounts/accounts-billing/new-relic-one-user-management)

Tenga en cuenta que algunas características, como <InlinePopover type="dashboards" />y carga de trabajo, pueden mostrar datos de diferentes cuentas de una organización. Esto significa que si a un usuario no se le otorga acceso a todas las cuentas relevantes, es posible que le falten datos.

Para obtener más información sobre los problemas de acceso, consulte [Factores que afectan el acceso](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data/).
