Además de los factores de facturación principales de ingesta y usuario facturable, también puede utilizar complementos facturables opcionales para mejorar su experiencia con New Relic:

* <DNT>
    **Compute Add On**
  </DNT>

  : Este complemento se aplica a la facturación basada en el uso de las consultas que realiza con las pruebas de seguridad de aplicaciones interactivas, Live Archives y otros complementos que estén disponibles. Mide su uso en función de [la unidad de capacidad de computación (CCU)](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions#compute-capacity-unit).

* <DNT>
    **EU Data Center for New Relic - Data, and New Relic - Data Plus**
  </DNT>

  : Este complemento se aplica cuando seleccionas la Unión Europea como tu región de datos, según esté disponible.

* <DNT>
    **Extended Retention for New Relic - Data, and New Relic - Data Plus**
  </DNT>

  : Este complemento se aplica si excede el período de tiempo predeterminado que se conservan sus datos. Esto se aplica a todos sus datos, no solo a los registros, y es una buena opción si necesita realizar muchas consultas pequeñas o consultas sobre grandes volúmenes de datos.

* <DNT>
    **Live Archives**
  </DNT>

  : amplíe la duración del almacenamiento log al menos 90 días más que la retención estándar. Este complemento es mejor si necesita mantener un registro que consultará con poca frecuencia. Live Archives también requiere calcular Add On.

* <DNT>
    **New Relic Synthetic Checks**
  </DNT>

  : Este complemento se aplica si sus checks exceden el número predeterminado de checks de monitoreo sintético.

* <DNT>
    **Vulnerability Management**
  </DNT>

  : Esta característica está incluida con Data Plus, pero el complemento aplica si usas la característica Gestión de vulnerabilidades sin Data Plus.

Para comenzar con cualquiera de estos complementos opcionales, comuníquese con el equipo de su cuenta de New Relic.
