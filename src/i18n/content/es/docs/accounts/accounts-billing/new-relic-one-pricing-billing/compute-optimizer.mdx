El Optimizador de computación proporciona información útil y valiosa para ayudarlo a gestionar el gasto de computación e implementar controles de computación automatizados. Esto le permite identificar áreas ineficientes, recibir recomendaciones claras para la optimización y, en última instancia, lograr posibles ahorros de cálculo, una mayor eficiencia de consulta y una experiencia de New Relic más optimizada.

Esta herramienta está diseñada para dos tipos de usuarios:

* **Especialistas y creadores de dashboard:** usuarios que crean dashboard y condiciones de alerta y desean asegurarse de que estén optimizados y no desperdicien recursos
* **Administradores:** los responsables de mantener los costos de New Relic dentro de su empresa y garantizar que el uso de cálculo no exceda los límites de presupuesto

<Callout variant="important">
  Calcular Optimizer está disponible únicamente para clientes con planes Calcular heredados (CCU) o Calcular Core. No está disponible para clientes con planes Datos + Usuario + Calcular Avanzado.
</Callout>

## Acceda al Optimizador de Calcular

Encontrarás el Compute Optimizer en el panel de administración de New Relic.

1. Inicie sesión en New Relic.
2. Haga clic en su nombre de usuario y luego seleccione <DNT>**Administration**</DNT>.
3. Haga clic en <DNT>**Compute Optimizer**</DNT> para acceder al dashboard.

## Compute Optimizer

El optimizador de computación incluye las vistas <DNT>**Alert Conditions**</DNT> y <DNT>**Dashboards**</DNT>, lo que le permite enfocarse en capacidades específicas del producto. Ambas vistas comparten funcionalidad común:

* <DNT>**Time picker:**</DNT> Filtrar datos en función de períodos de tiempo específicos
* <DNT>**Refresh recommendations:**</DNT> Actualiza las recomendaciones con los últimos datos
* <DNT>**Export report:**</DNT> Descargue las recomendaciones para el análisis offline

Cada vista muestra diferentes métricas relacionadas con el uso de calcular y los ahorros potenciales (consulte las secciones individuales a continuación).

Los Insights son dinámicos y se filtran según la configuración de su selector de tiempo, lo que le permite analizar períodos de tiempo específicos.

      Las condiciones de alerta le ayudan a identificar y optimizar las condiciones de alerta ineficientes que consumen recursos de cómputo excesivos.

      <img title="Compute Optimizer" alt="Image of Compute Optimizer" src="/images/compute_optimize.webp" />

      ### Métricas del dashboard

      * <DNT>**Inefficient usage:**</DNT> Muestra el total de CCU central consumido por las condiciones de alerta que se han marcado como ineficientes, expresado como un porcentaje del total de CCU de alertas.
      * <DNT>**Potential savings:**</DNT> Muestra el ahorro estimado de CCU central que puede lograr, expresado como un porcentaje del total de CCU de alertas (excluye las recomendaciones sin estimaciones de ahorro)

      ### Buscar y filtrar

      Utilice la barra de búsqueda para encontrar condiciones de alerta específicas por nombre. También puede filtrar las recomendaciones por:

      * <DNT>**Component name:**</DNT> Filtrar por todas las condiciones de alerta disponibles o por algunas específicas
      * <DNT>**Product capability:**</DNT> Enfóquese específicamente en las condiciones de alerta

      ### Tabla de recomendaciones

      La tabla de recomendaciones desglosa las oportunidades de optimización identificadas para las condiciones de alerta:

      * <DNT>**Component name:**</DNT> Identifica la condición de alerta específica que tiene una oportunidad de optimización

      * <DNT>**Product capability:**</DNT> Muestra &quot;Condiciones de alerta&quot; para todas las entradas

      * <DNT>**CCU usage:**</DNT> Muestra el consumo de computación actual (medido en core CCU) para cada condición de alerta

      * <DNT>**Recommendation:**</DNT> Ofrece información procesable valiosa para optimizar su calcular. Las recomendaciones comunes incluyen:

        * &quot;Agregar una cláusula `WHERE` fuera de la declaración `SELECT` &quot;
        * &quot;Considere eliminar esta condición de alerta. Combinar umbrales y/o mantener la condición con un LOS menor&quot;
        * &quot;Esta condición de alerta tiene duplicados. Considere eliminar las condiciones de alerta duplicadas&quot;

      * <DNT>**CCU saving estimate:**</DNT> Cuantifica la reducción potencial de computación (en CCU central) para cada recomendación individual

      * <DNT>**Reduction estimate:**</DNT> Muestra el porcentaje de calcular que se podría ahorrar (por ejemplo, &quot;100 %&quot;, &quot;0 %&quot;)

      ### Tomar medidas

      El menú de acción (tres puntos) para cada recomendación ofrece opciones adicionales, incluido el acceso a consultas optimizadas cuando corresponda, lo que facilita implementar directamente los cambios sugeridos.
    </TabsPageItem>

    <TabsPageItem id="dashboards">
      Los Dashboards le ayudan a identificar y optimizar los widgets y las páginas del dashboard que consumen recursos de cálculo excesivos o necesitan mejoras de rendimiento.

      <img title="Compute Optimizer Dashboards view" alt="Image of the Compute Optimizer Dashboards view" src="/images/compute_optimizer_dashboard.webp" />

      ### Métricas del dashboard

      * <DNT>**Total Usage:**</DNT> Muestra el total de CCU central consumida por todos los dashboard, expresado como un porcentaje del total de CCU de dashboard
      * <DNT>**Potential savings:**</DNT> Muestra los ahorros estimados de CCU principal que puede lograr, expresados como un porcentaje del total de CCU del dashboard (excluye las recomendaciones sin estimaciones de ahorro)

      ### Buscar y filtrar

      Utilice la barra de búsqueda para encontrar dashboard, widget o páginas específicos por nombre. También puede filtrar las recomendaciones por:

      * <DNT>**Dashboard:**</DNT> Filtrar por todos los dashboards o por algunos específicos
      * <DNT>**Item:**</DNT> Filtrar por todos los elementos (widget y páginas) o específicos
      * <DNT>**Recommendation:**</DNT> Filtrar por todas las recomendaciones o tipos de recomendación específicos

      ### Vista agrupada

      Los Dashboards organizan las recomendaciones por dashboard, mostrando la cantidad total de grupos y elementos (por ejemplo, &quot;3 grupos | 26 elementos&quot;). Dentro de un dashboard, puede haber varias recomendaciones. Cada dashboard se puede expandir para ver recomendaciones individuales en tres niveles:

      * <DNT>**Dashboard level:**</DNT> Recomendaciones que se aplican a todo el dashboard (por ejemplo, reducir el rango de tiempo para todo el dashboard)
      * <DNT>**Page level:**</DNT> Recomendaciones para páginas específicas dentro del dashboard
      * <DNT>**Widget level:**</DNT> Recomendaciones para widgets específicos dentro del dashboard (por ejemplo, usar filtros WHERE de manera más efectiva en un widget específico)

      ### Tabla de recomendaciones

      La tabla de recomendaciones muestra las oportunidades de optimización organizadas por elemento:

      * <DNT>**Item:**</DNT> Identifica el dashboard, la página o el widget específico que tiene una oportunidad de optimización. Los elementos se agrupan bajo el nombre de su dashboard principal.
      * <DNT>**Recommendation:**</DNT> Ofrece información procesable valiosa para optimizar su dashboard. Las recomendaciones varían según las ineficiencias específicas detectadas y pueden incluir varias sugerencias por dashboard.
      * <DNT>**Usage:**</DNT> Muestra el consumo de computación actual (medido en core CCU) para cada elemento, con capacidad de ordenamiento
      * <DNT>**Estimated savings:**</DNT> Cuantifica el ahorro potencial de CCU central y la reducción porcentual para cada recomendación

      <Callout variant="important">
        Un guion (--) en las columnas <DNT>**Usage**</DNT> y <DNT>**Estimated savings**</DNT> indica que la cantidad exacta de ahorro no se puede calcular. Sin embargo, implementar estas recomendaciones seguirá reduciendo el consumo de cómputo.
      </Callout>

      ### Visualización de los detalles de la recomendación

      Al hacer clic en cualquier recomendación, se abre un panel lateral con:

      * El texto detallado de la recomendación
      * Un enlace al dashboard afectado
      * La consulta original y una consulta optimizada (cuando corresponda), lo que le permite copiar e implementar la versión optimizada

      Utilice la consulta optimizada para actualizar su dashboard y mejorar el rendimiento.
