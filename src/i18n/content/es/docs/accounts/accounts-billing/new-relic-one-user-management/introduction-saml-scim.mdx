Para configurar controles automáticos sobre cómo se agrega su usuario de New Relic a New Relic, cómo se administra y cómo log sesión, ofrecemos estas características:

* <DNT>**SAML SSO**</DNT>: esto permite que su usuario emplee un servicio de proveedor de identidad de inicio de sesión único (SSO) (SSO) para iniciar sesión en New Relic, en lugar de emplear el email y la contraseña predeterminados. Requiere una [edición paga](https://newrelic.com/pricing).
* <DNT>**SCIM provisioning**</DNT>, también conocido como gestión automatizada de usuarios: el aprovisionamiento de SCIM permite a las organizaciones emplear su servicio de proveedor de identidad para automatizar cómo se agregan y actualizan sus usuarios en New Relic. Las organizaciones que emplean SCIM casi siempre emplearán SAML SSO. Requiere una [edición paga](https://newrelic.com/pricing).

Para obtener información sobre cómo habilitar SAML y SCIM, empleará la [UIdel dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) para configurar SAML y SCIM. Tenga en cuenta que Okta, OneLogin y Azure tienen aplicaciones New Relic dedicadas: si las usa, debe elegir la que se llama **New Relic by organization** y no **New Relic by account**.
