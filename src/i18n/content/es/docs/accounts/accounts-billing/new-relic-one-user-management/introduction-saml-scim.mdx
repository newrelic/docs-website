Para configurar controles automáticos sobre cómo se agrega su usuario de New Relic a New Relic, cómo se administra y cómo log sesión, ofrecemos estas características:

* <DNT>
    **SAML SSO**
  </DNT>

  : esto le permite a su usuario usar un servicio de proveedor de identidad de inicio de sesión único (SSO) para log sesión en New Relic, en lugar de usar el correo electrónico + contraseña predeterminados. Requiere una [edición paga](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).

* <DNT>
    **SCIM provisioning**
  </DNT>

  , también conocida como gestión automatizada de usuarios: el aprovisionamiento SCIM permite a la organización utilizar su servicio de proveedor de identidad para automatizar cómo se agregan y actualizan sus usuarios en New Relic. Una organización que utiliza SCIM casi siempre utilizará SAML SSO. Requiere [edición Pro o Enterprise](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).

Para obtener información sobre cómo habilitar SAML y SCIM, elija su modelo de usuario a continuación. Si no está seguro de en cuál se encuentra, consulte [Descripción general de los modelos de usuario](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).

<CollapserGroup>
  <Collapser
    id="new-account-model"
    title="Nuestro nuevo modelo de usuario"
  >
    Para los usuarios de [nuestro modelo de usuario más nuevo](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), utilizarán la [UIde usuario del dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) para configurar SAML y SCIM. Tenga en cuenta que Okta, OneLogin y Azure tienen aplicaciones New Relic dedicadas: si las usa, debe elegir la que se llama "New Relic por organización" y no "New Relic por cuenta".
  </Collapser>

  <Collapser
    id="original-account-model"
    title="Modelo de usuario original"
  >
    Si su organización utiliza el modelo de usuario original, le recomendamos migrar sus usuarios mediante el [procedimiento de migración de autoservicio](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) si puede hacerlo. Si no cumple con los requisitos pero desea habilitar SAML y/o SCIM, hable con su representante de cuenta de New Relic.

    Aquí hay algunos consejos y enlaces a documentos:

    * SAML: si desea o debe permanecer en el modelo de usuario original y no puede migrar al nuevo modelo, consulte nuestros [documentos SAML originales](/docs/accounts/accounts/saml-single-sign/saml-service-providers).
    * SCIM + SAML: el aprovisionamiento SCIM crea registros de usuario en nuestro modelo de usuario más nuevo. Es por eso que le recomendamos que primero migre sus usuarios al nuevo modelo de usuario antes de usar SCIM. Para obtener más información sobre cómo migrar al nuevo modelo, consulte [Migrar usuario](/docs/accounts/original-accounts-billing/original-users-roles/user-migration).
  </Collapser>
</CollapserGroup>
