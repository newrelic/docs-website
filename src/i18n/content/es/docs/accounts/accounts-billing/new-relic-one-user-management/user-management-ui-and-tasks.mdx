Este documento explica cómo [encontrar y utilizar la UIde usuario de administración de usuarios](#where) y cómo realizar algunas [tareas comunes de administración de usuarios](#workflow).

## UIde usuario de gestión de usuarios [#where]

Para encontrar las opciones de administración de usuarios de su organización New Relic: En el [menú de usuarios](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), seleccione <DNT>**Administration**</DNT>. Estas son las páginas de UI relacionadas con la administración de usuarios que puede encontrar allí:

* <DNT>
    **User management**
  </DNT>

  ([enlace a UI](https://one.newrelic.com/admin-portal/organizations/users-list)): utilícelo para agregar usuarios, actualizar el tipo de usuario, administrar grupos de usuarios, aprobar solicitudes de actualización de usuarios y controlar la configuración de la sesión.

* <DNT>
    **Invitation approval**
  </DNT>

  : Aquí es donde puedes aprobar que el usuario sea agregado a New Relic. Estos usuarios aún no se consideran aprovisionados y no son facturables.

* <DNT>
    **Access management**
  </DNT>

  ([enlace a UI](https://one.newrelic.com/admin-portal/organizations/organization-detail)): use esta UI para ver, crear y administrar [grupos y roles](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

* <DNT>
    **API keys**
  </DNT>

  : utilice la [UIde clave de API](/docs/apis/intro-apis/new-relic-api-keys) para ver y administrar la clave de API de sus usuarios.

* <DNT>
    **Domain capture**
  </DNT>

  : utilice la captura de dominio para dirigir a su usuario al lugar correcto cuando intente registrarse en New Relic. [Obtenga más información sobre la captura de dominio.](/docs/accounts/accounts-billing/account-setup/domain-capture)

* <DNT>
    **Authentication domains**
  </DNT>

  ([enlace a UI](https://one.newrelic.com/admin-portal/centralized-admin-user-auto-provisioning/home)): use esto para controlar cómo se agregan sus usuarios (desde New Relic o desde un proveedor de identidad), cómo log sesión (manualmente o mediante SAML SSO) y más. Para obtener más información, consulte [Dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more).

Para obtener consejos sobre cómo administrar usuarios, consulte [Tareas comunes](#workflow) y nuestros [videos de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos).

## Requisitos [#requirements]

La forma más sencilla de ver cuáles son sus permisos de administración de usuarios es ir a la [UIde usuario de administración de usuarios](#where) y ver a qué tiene acceso.

Algunos requisitos y restricciones de gestión de usuarios:

* Para agregar y editar grupos y roles, su organización debe tener la edición [Pro o Enterprise](https://newrelic.com/pricing) .

* Requisitos relacionados con los permisos:

  * Requiere un [tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) de usuario principal o usuario de plataforma completa.
  * La mayoría de los permisos de administración requieren la [configuración de administración](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) relacionada con el dominio de autenticación y organización del usuario.

* Para evitar conflictos de configuración, intente asegurarse de que solo una persona en su organización administre el usuario a la vez.

## Detalles UI [#user-mgmt]

Aquí hay una captura de pantalla de la UI <DNT>**User management**</DNT>. Explicaremos los encabezados de las columnas a continuación.

<img
  title="User management UI"

* <DNT>
    **Name**
  </DNT>

  : El nombre del usuario.

* <DNT>
    **Email**
  </DNT>

  : La dirección de correo electrónico del usuario. Para usuarios con `Pending`: este usuario ha sido agregado pero aún no ha verificado su correo electrónico. Incluso en el estado `Pending` , un usuario se considera aprovisionado y, por lo tanto, se puede facturar si es un usuario principal o de plataforma completa.

* <DNT>
    **Type**
  </DNT>

  : El [tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) del usuario.

* <DNT>
    **Last active**
  </DNT>

  : La última fecha en que un usuario inició sesión en la plataforma. (Esto **no** realiza un seguimiento del uso de nuestras API que requieren un

  <InlinePopover type="userKey"/>

  ). Esto puede resultar útil para decidir degradar a un usuario a un tipo de usuario inferior.

* <DNT>
    **ID**
  </DNT>

  : ID asignada a la New Relicde un usuario.

Para obtener información sobre los grupos en los que se encuentran los usuarios, puede descargar un archivo TSV desde la UI o [usar NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#query-user-groups).

## Consejos sobre tareas comunes de gestión de usuarios [#workflow]

A continuación se muestran algunas tareas de administración de usuarios que quizás desee realizar:

<CollapserGroup>
  <Collapser
    id="learn-about-users"
    title="Más información sobre el usuario"
  >
    Para obtener más información sobre su usuario, vaya a <DNT>**User management UI**</DNT> y examine el usuario específico para obtener más detalles. También puede ir a la UI <DNT>**Access management**</DNT> para examinar los grupos: las funciones que tienen esos grupos y los usuarios de cada grupo.

    Para ver todos sus usuarios y sus membresías grupales, puede [usar NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups#query-user-groups).
  </Collapser>

  <Collapser
    id="add-users"
    title="Añadir usuario"
  >
    Tenga en cuenta que agregar un usuario directamente a New Relic significa que los usuarios con un tipo de usuario facturable son facturables inmediatamente, independientemente de si esos usuarios son `Pending` y aún no han iniciado sesión en New Relic. Las opciones para agregar usuarios a través de la UI de usuario de administración de usuarios incluyen:

    * En la [UI](#where)

      <DNT>
        [**User management**](#where)
      </DNT>

      [](#where), seleccione

      <DNT>
        **Add user**
      </DNT>

      .

    * En

      <DNT>
        [**Access management UI**](#where)
      </DNT>

      , al crear o editar un grupo, agregue un usuario a ese grupo.

      También puedes utilizar la característica <DNT>**Invite users**</DNT> . Para usarlo, vaya al [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) y haga clic en <DNT>**Add user**</DNT>. Detalles sobre cómo invitar a un usuario:

    * Si tiene [permisos](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings)

      <DNT>
        [**Authentication domain**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings)
      </DNT>

      [](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings), esto lo llevará a la UI

      <DNT>
        **Add users**
      </DNT>

      , donde puede agregar usuarios directamente.

    * Si no tienes ese permiso, puedes invitar a un compañero de equipo. Esto solicita la revisión por parte de un administrador de ese usuario que se agrega a New Relic. Las invitaciones pendientes que no se han revisado se marcan como [`Pending invite approval` en la UIde usuario de administración de usuarios](#user-mgmt). Cuando un administrador revisa a un usuario invitado, completa el flujo de trabajo

      <DNT>
        **Add user**
      </DNT>

      y ese usuario se considera agregado en New Relic.

      Algunos puntos importantes sobre cómo agregar usuarios:

    * También puede utilizar [nuestra API NerdGraph para administrar el usuario](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).

    * Si utiliza [la gestión automatizada de usuarios](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign), solo puede agregar usuarios desde su proveedor de identidad.
  </Collapser>

  <Collapser
    id="edit-user-type"
    title="Editar el tipo de usuario"
  >
    Antes de cambiar el tipo de usuario de un usuario, le recomendamos comprender:

    * [Cómo decidir un tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#choose-user-type).
    * [Cómo se calculan los usuarios](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing), incluidos consejos sobre cómo cronometrar la adición de nuevos usuarios o cómo ajustar su tipo de usuario.
    * [Reglas de degradación del usuario](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#user-downgrade-rules).
    * ¿Utiliza [la gestión automatizada de usuarios](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign)? Tienes [otras opciones para gestionar el tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).

      Para cambiar el tipo de usuario de los usuarios en la UI:

    1. Desde la [UI](#where)

       <DNT>
         [**User management**](#where)
       </DNT>

       [](#where), haga clic en las casillas de verificación de los usuarios cuyo tipo de usuario desea editar.

    2. Una vez que comiences a seleccionar usuario, aparecerá una opción para

       <DNT>
         **Edit type**
       </DNT>

       .

       También puede editar el tipo de usuario y el grupo de un usuario específico haciendo clic en ese usuario.

       Para saber cómo utilizar nuestra API para administrar la información del usuario, consulte [Administrar usuarios con NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).

       Para administrar cómo los usuarios actualizan su tipo de usuario, consulte la [configuración del dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).
  </Collapser>

  <Collapser
    id="bulk-edit-users"
    title="Editar usuario en masa"
  >
    Para editar la configuración para varios usuarios a la vez:

    1. Vaya a la [UI](#where)

       <DNT>
         [**User management**](#where)
       </DNT>

       [](#where)y seleccione las casillas de verificación de los usuarios cuyo grupo o tipo de usuario desea editar.

    2. Las opciones de edición aparecerán en la parte superior de la UI.

       Nota: si utiliza [la gestión de usuarios automatizada](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign), los grupos deben editarse desde su proveedor de identidad.
  </Collapser>

  <Collapser
    id="add-user-managers"
    title="Asignar usuario para administrar otro usuario"
  >
    Hay dos opciones para darle a su usuario la posibilidad de administrar otros usuarios:

    * La [configuración del dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) otorga al usuario capacidades de toda la organización relacionadas con la administración de usuarios, grupos y dominios de autenticación.

    * La función [de administrador de grupo](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#group-admin) es mucho más restringida: le da al usuario la posibilidad de agregar y eliminar usuarios de un grupo específico.

      Para brindar capacidades de administración de usuarios en toda la organización, puede agregar un usuario al grupo <DNT>**Admin**</DNT> predeterminado, que tiene esa configuración habilitada. También puedes crear un grupo personalizado y darle esa configuración.

      Para obtener un tutorial sobre tareas comunes de administración de usuarios, consulte el [tutorial de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial).
  </Collapser>

  <Collapser
    id="delete-users"
    title="Eliminar usuario"
  >
    Para eliminar un usuario: vaya a la [UI<DNT>**User management**</DNT> ](#where)y seleccione las casillas de verificación de uno o más usuarios. Luego haga clic en <DNT>**Delete users**</DNT>.
  </Collapser>

  <Collapser
    id="export-users"
    title="Exportar una hoja de cálculo de usuario"
  >
    Para exportar una lista de todos los usuarios y sus detalles (nombres, direcciones de correo electrónico, tipo de usuario actual, grupos, ID de usuario): desde la [UI<DNT>**User management**</DNT> ](#where), haga clic en el icono de descarga junto al botón <DNT>**Add user**</DNT>. Hay una opción para descargar una lista de todos los usuarios en ese dominio de autenticación.

    Alternativamente, para exportar un subconjunto de usuarios, primero seleccione el usuario que desea exportar y luego presione el botón de descarga.
  </Collapser>

  </Collapser>
