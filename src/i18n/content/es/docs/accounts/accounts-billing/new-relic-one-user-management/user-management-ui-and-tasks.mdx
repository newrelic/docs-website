Este documento explica cómo [encontrar y utilizar la UI de administración de usuarios](#where) y cómo realizar algunas [tareas comunes de administración de usuarios](#workflow).

## UI de gestión de usuarios [#where]

Para encontrar las opciones de administración de usuarios de su organización New Relic: En el [menú de usuarios](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), seleccione <DNT>**Administration**</DNT>. Estas son las páginas de UI relacionadas con la administración de usuarios que puede encontrar allí:

* <DNT>**[User management](https://one.newrelic.com/admin-portal/organizations/users-list)**</DNT>: Use this to add users, update user type, manage users&apos; groups, approve user upgrade requests, and control session settings.

* <DNT>**[Invitation approval](https://one.newrelic.com/admin-portal/organizations/invitation-approval)**</DNT>: Aquí es donde puedes aprobar que el usuario sea agregado a New Relic. Estos usuarios aún no se consideran aprovisionados y no son facturables.

* <DNT>**[Feature control manager](https://one.newrelic.com/admin-portal/feature-control-management/home)**</DNT>: Use this to [activate and manage Intelligent Observability features](/docs/accounts/accounts-billing/new-relic-one-user-management/feature-control-manager) across multiple accounts in your organization.

* <DNT>**[Access management](https://one.newrelic.com/admin-portal/organizations/organization-detail)**</DNT>: The primary interface for managing RBAC, organized into four dedicated tabs:

  * <DNT>**[Groups](https://one.newrelic.com/admin-portal/organizations/groups)**</DNT>:Crear y gestionar grupos de usuarios y asignar usuarios a grupos
  * <DNT>**[Accounts](https://one.newrelic.com/admin-portal/organizations/accounts)**</DNT>:Crear y gestionar cuentas dentro de su organización
  * <DNT>**Roles**</DNT>:Crear y gestionar roles personalizados con selección de alcance explícito (organización, cuenta o nivel de entidad)
  * <DNT>**Access Grants**</DNT>:Crear concesiones de acceso que vinculen grupos a roles sobre un objetivo específico

* <DNT>**API keys**</DNT>: utilice la [UI de clave de API](/docs/apis/intro-apis/new-relic-api-keys) para ver y administrar la clave de API de sus usuarios.

* <DNT>**Domain capture**</DNT>: utilice la captura de dominio para dirigir a su usuario al lugar correcto cuando intente registrarse en New Relic. [Obtenga más información sobre la captura de dominio.](/docs/accounts/accounts-billing/account-setup/domain-capture)

* <DNT>**Authentication domains**</DNT> ([enlace a UI](https://one.newrelic.com/admin-portal/centralized-admin-user-auto-provisioning/home)): use esto para controlar cómo se agregan sus usuarios (desde New Relic o desde un proveedor de identidad), cómo log sesión (manualmente o mediante SAML SSO) y más. Para obtener más información, consulte [Dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more).

Para obtener consejos sobre cómo administrar usuarios, consulte [Tareas comunes](#workflow) y nuestros [videos de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos).

## Requisitos [#requirements]

La forma más sencilla de ver cuáles son sus permisos de administración de usuarios es ir a la [UI de administración de usuarios](#where) y ver a qué tiene acceso.

Algunos requisitos y restricciones de gestión de usuarios:

* Para agregar y editar grupos y roles, su organización debe tener la edición [Pro o Enterprise](https://newrelic.com/pricing) .

* Requisitos relacionados con los permisos:

  * Requiere un [tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) de usuario principal o usuario de plataforma completa.
  * La mayoría de las licencias de administración de usuarios requieren roles de ámbito organizacional con licencias de administración de dominio de autenticación.

* Para evitar conflictos de configuración, intente asegurarse de que solo una persona en su organización administre el usuario a la vez.

## Detalles UI [#user-mgmt]

Aquí hay una captura de pantalla de la UI <DNT>**User management**</DNT>. Explicaremos los encabezados de las columnas a continuación.

<img title="User management UI" alt="User management UI" src="/images/accounts_screenshot-crop_user-mgmt-ui.webp" />

* <DNT>**Name**</DNT>: El nombre del usuario.
* <DNT>**Email**</DNT>: La dirección de correo electrónico del usuario. Para usuarios con `Pending`: este usuario ha sido agregado pero aún no ha verificado su correo electrónico. Incluso en el estado `Pending` , un usuario se considera aprovisionado y, por lo tanto, se puede facturar si es un usuario principal o de plataforma completa.
* <DNT>**Type**</DNT>: El [tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) del usuario.
* <DNT>**Last active**</DNT>: La última fecha en que un usuario inició sesión en la plataforma. (Esto **no** realiza un seguimiento del uso de nuestras API que requieren un <InlinePopover type="userKey" />). Esto puede resultar útil para decidir degradar a un usuario a un tipo de usuario inferior.
* <DNT>**ID**</DNT>: ID asignada a la New Relic de un usuario.

Para obtener información sobre los grupos en los que se encuentran los usuarios, puede descargar un archivo TSV desde la UI o [usar NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#query-user-groups).

## Consejos sobre tareas comunes de gestión de usuarios [#workflow]

A continuación se muestran algunas tareas de administración de usuarios que quizás desee realizar:

<CollapserGroup>
  <Collapser id="learn-about-users" title="Más información sobre el usuario">
    Para obtener más información sobre su usuario, vaya a <DNT>**User management UI**</DNT> y examine el usuario específico para obtener más detalles. También puede ir a la UI <DNT>**Access management**</DNT> para examinar los grupos: las funciones que tienen esos grupos y los usuarios de cada grupo.

    Para ver todos sus usuarios y sus membresías grupales, puede [usar NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups#query-user-groups).
  </Collapser>

  <Collapser id="add-users" title="Añadir usuario">
    Tenga en cuenta que agregar un usuario directamente a New Relic significa que los usuarios con un tipo de usuario facturable son facturables inmediatamente, independientemente de si esos usuarios son `Pending` y aún no han iniciado sesión en New Relic. Las opciones para agregar usuarios a través de la UI de administración de usuarios incluyen:

    * En la [UI<DNT>**User management**</DNT> ](#where), seleccione <DNT>**Add user**</DNT>.
    * En [<DNT>**Access management UI**</DNT>](#where), al crear o editar un grupo, agregue un usuario a ese grupo.

    También puedes utilizar la característica <DNT>**Invite users**</DNT> . Para usarlo, vaya al [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) y haga clic en <DNT>**Add user**</DNT>. Detalles sobre cómo invitar a un usuario:

    * Si tiene roles de ámbito de organización con licencias de administración de dominio de autenticación, esto lo llevará a la UI <DNT>**Add users**</DNT>, donde puede agregar usuarios directamente.
    * Si no tienes ese permiso, puedes invitar a un compañero de equipo. Esto solicita la revisión por parte de un administrador de ese usuario que se agrega a New Relic. Las invitaciones pendientes que no se han revisado se marcan como [`Pending invite approval` en la UI de administración de usuarios](#user-mgmt). Cuando un administrador revisa a un usuario invitado, completa el flujo de trabajo <DNT>**Add user**</DNT> y ese usuario se considera agregado en New Relic.

    Algunos puntos importantes sobre cómo agregar usuarios:

    * También puede utilizar [nuestra API NerdGraph para administrar el usuario](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).
    * Si utiliza [la gestión automatizada de usuarios](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign), solo puede agregar usuarios desde su proveedor de identidad.
  </Collapser>

  <Collapser id="edit-user-type" title="Editar el tipo de usuario">
    Antes de cambiar el tipo de usuario de un usuario, le recomendamos comprender:

    * [Cómo decidir un tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#choose-user-type).
    * [Cómo se calculan los usuarios](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing), incluidos consejos sobre cómo cronometrar la adición de nuevos usuarios o cómo ajustar su tipo de usuario.
    * [Reglas de degradación del usuario](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#user-downgrade-rules).
    * ¿Utiliza [la gestión automatizada de usuarios](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign)? Tienes [otras opciones para gestionar el tipo de usuario](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).

    Para cambiar el tipo de usuario de los usuarios en la UI:

    1. Desde la [UI<DNT>**User management**</DNT> ](#where), haga clic en las casillas de verificación de los usuarios cuyo tipo de usuario desea editar.
    2. Una vez que comiences a seleccionar usuario, aparecerá una opción para <DNT>**Edit type**</DNT>.

    También puede editar el tipo de usuario y el grupo de un usuario específico haciendo clic en ese usuario.

    Para saber cómo utilizar nuestra API para administrar la información del usuario, consulte [Administrar usuarios con NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).

    Para administrar cómo los usuarios actualizan su tipo de usuario, consulte la [configuración del dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).
  </Collapser>

  <Collapser id="bulk-edit-users" title="Editar usuario en masa">
    Para editar la configuración para varios usuarios a la vez:

    1. Vaya a la [UI<DNT>**User management**</DNT> ](#where)y seleccione las casillas de verificación de los usuarios cuyo grupo o tipo de usuario desea editar.
    2. Las opciones de edición aparecerán en la parte superior de la UI.

    Nota: si utiliza [la gestión de usuarios automatizada](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign), los grupos deben editarse desde su proveedor de identidad.
  </Collapser>

  <Collapser id="bulk-import-users" title="Importación masiva de usuario">
    Para importar varios usuarios a New Relic a la vez, tienes estas opciones:

    * **Aprovisionamiento de SCIM (recomendado para la administración continua)**: configure [la administración automatizada de usuarios](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign/#how-to) para sincronizar automáticamente usuarios y grupos de su proveedor de identidad (como Azure AD, Okta o OneLogin) con New Relic. Esta es la mejor opción para las organizaciones que desean gestionar usuarios de forma masiva y mantenerlos sincronizados. Requiere [edición Pro o Enterprise](https://newrelic.com/pricing).

    * **API SCIM (para importaciones masivas únicas o integración personalizada)**: emplee nuestra [APISCIM](/docs/accounts/accounts/automated-user-management/scim-support-automated-user-management/#create-user) para crear usuarios mediante programación.

    * **API de NerdGraph (para importaciones con scripts)**: emplee nuestra [APIde NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users/#create-users) para crear usuarios individualmente, que pueden programar para operaciones masivas.

    <Callout variant="tip">
      New Relic actualmente no admite la carga directa de archivos CSV u hojas de cálculo para importaciones masivas de usuarios. Para realizar importaciones en masa, emplee uno de los métodos automatizados anteriores.
    </Callout>
  </Collapser>

  <Collapser id="add-user-managers" title="Asignar usuario para administrar otro usuario">
    Hay dos opciones para darle a su usuario la posibilidad de administrar otros usuarios:

    * Los roles dentro del ámbito de la organización con licencias de administración de dominio de autenticación otorgan al usuario capacidades en toda la organización relacionadas con la administración de usuarios, grupos y dominio de autenticación.

    * La función [de administrador de grupo](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#group-admin) es mucho más restringida: le da al usuario la posibilidad de agregar y eliminar usuarios de un grupo específico.

      Para otorgar capacidades de administración de usuarios a toda la organización, puede agregar un usuario al grupo predeterminado <DNT>**Admin**</DNT>, que tiene roles de alcance de la organización con estas licencias. También puede crear un grupo personalizado y asignarle roles de ámbito organizacional con licencias de administración de dominio de autenticación.

      Para obtener un tutorial sobre tareas comunes de administración de usuarios, consulte el [tutorial de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial).
  </Collapser>

  <Collapser id="delete-users" title="Eliminar usuario">
    Para eliminar un usuario: vaya a la [UI<DNT>**User management**</DNT> ](#where)y seleccione las casillas de verificación de uno o más usuarios. Luego haga clic en <DNT>**Delete users**</DNT>.

    Cuando se elimina un usuario, se eliminan los siguientes activos del usuario:

    * Favoritos
    * Configuración de correo electrónico semanal
    * Preferencias de inclusión/rechazo de correo electrónico
    * [Claves de usuario específicas del usuario](/docs/apis/intro-apis/new-relic-api-keys/#user-api-key)
    * [Datos de NerdStorage](/docs/new-relic-solutions/build-nr-ui/nerdstorage)de las aplicaciónes New Relic
  </Collapser>

  <Collapser id="export-users" title="Exportar una hoja de cálculo de usuario">
    Para exportar una lista de todos los usuarios y sus detalles (nombres, direcciones de correo electrónico, tipo de usuario actual, grupos, ID de usuario): desde la [UI<DNT>**User management**</DNT> ](#where), haga clic en el icono de descarga junto al botón <DNT>**Add user**</DNT>. Hay una opción para descargar una lista de todos los usuarios en ese dominio de autenticación.

    Alternativamente, para exportar un subconjunto de usuarios, primero seleccione el usuario que desea exportar y luego presione el botón de descarga.
  </Collapser>

  <Collapser id="access-grants" title="Dar acceso de usuario a cuentas y roles">
    Consulta nuestro [tutorial de gestión de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model).
  </Collapser>

  <Collapser id="new-group" title="Crear nuevos grupos y roles personalizados">
    Ver el [tutorial de gestión de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model).
  </Collapser>

  <Collapser id="new-group" title="Configurar SAML SSO y/o aprovisionamiento SCIM">
    Puede emplear un proveedor de identidad de inicio de sesión único (SSO) para iniciar sesión en New Relic mediante SAML SSO. Para la gestión masiva de usuarios, también querrás emplear el aprovisionamiento SCIM (también llamado gestión automatizada de usuarios). Le permite sincronizar automáticamente usuarios y grupos de su proveedor de identidad con New Relic.

    El aprovisionamiento de SCIM ofrece estos beneficios para la gestión masiva de usuarios:

    * Importe y sincronice automáticamente un gran número de usuarios desde su proveedor de identidad
    * Mantenga la información del usuario actualizada automáticamente
    * Gestione usuarios y grupos de forma centralizada desde su proveedor de identidad

    Para configurar el aprovisionamiento de SAML SSO o SCIM, consulte [Introducción a SAML SSO o SCIM](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-saml-scim). Tenga en cuenta que estas características requieren la [edición Pro o Enterprise](https://newrelic.com/pricing).
  </Collapser>

  <Collapser id="basic-full-upgrade" title="Controle cómo los usuarios actualizan su tipo de usuario">
    Consulte la [configuración del dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).
  </Collapser>
</CollapserGroup>
