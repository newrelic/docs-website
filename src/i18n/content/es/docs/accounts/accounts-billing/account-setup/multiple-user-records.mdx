Como usuario de New Relic, puede tener más de un registro de usuario utilizando la misma dirección de correo electrónico. Este sería el caso si tuvieras:

* Acceso a más de una [organización New Relic](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure)
* Acceso a más de un [dominio de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) en una organización New Relic

## Cómo contamos y rastreamos al usuario [#user-records]

Cuando se crea un usuario en New Relic, ese usuario recibe una identificación de usuario. Para encontrar el ID de un usuario, puede ir a la [UI<DNT>**User management**</DNT> ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks)y buscar la información de un usuario específico.

<Callout variant="tip">
  Una forma de encontrar su propio ID de usuario es inspeccionar el código fuente de una página UI de New Relic cargada y buscar `userId` en esa página.
</Callout>

Para determinar la cantidad de usuarios en una organización de New Relic con fines de facturación, contamos los ID de usuario y luego eliminamos los duplicados de los usuarios que tienen las mismas direcciones de correo electrónico.

Cuando actualiza su dirección de correo electrónico en New Relic, esto no causará facturación doble. Si está actualizando su correo electrónico y tiene varios registros de usuario en una organización, asegúrese de actualizarlo para todos los registros de usuario.

## Cambiar entre registros de usuario [#switch]

Cuando inicia sesión en New Relic, puede ir al [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) para cambiar a otros registros de usuario que tengan la misma dirección de correo electrónico. Cuando elija un registro de usuario del menú, se le indicará que iniciara sesión para ese registro de usuario (ya sea directamente en New Relic o a través de su proveedor de identidad).

Para acceder a registros de usuario que tienen una dirección de correo electrónico diferente, debe log y log a iniciar sesión.

## Mensaje "Multiple logins found" ("Se encontraron varios inicios de sesión") [#login-message]

Si su correo electrónico está asociado con varios registros de usuario, verá un mensaje `Multiple logins found` al iniciar sesión y se le pedirá que verifique su correo electrónico. Esto puede suceder si tiene acceso a múltiples organizaciones de New Relic o a múltiples [dominios de autenticación](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) en una organización.

<img
  title="Login UI for email addresses with multiple user records"
  alt="A screenshot of what is shown when you have an email address associated with multiple New Relic logins"
  src="/images/accounts_screenshot-crop_login-with-multiple-accounts-found.webp"
/>

<figcaption>
  Cuando tenga dos o más registros de usuario para la misma dirección de correo electrónico, recibirá un mensaje al respecto durante el inicio de sesión.
</figcaption>

Cuando un registro de usuario está marcado como `Original` en la página de inicio de sesión, como se muestra arriba, significa que ese registro de usuario está en nuestro [modelo de usuario original](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models).

Verificar su dirección de correo electrónico le permitirá elegir a qué registro de usuario acceder. Si no verifica su dirección de correo electrónico, New Relic intentará iniciar sesión con el registro de usuario creado más recientemente asociado con su dirección de correo electrónico.

Para evitar tener que verificar su correo electrónico para cada inicio de sesión, haga clic en la casilla de verificación <DNT>**Remember me**</DNT> en la pantalla de inicio de sesión.

Para obtener más información sobre problemas de inicio de sesión, consulte [Solucionar problemas de inicio de sesión](/docs/accounts/accounts-billing/account-setup/troubleshoot-new-relics-password-email-address-login-problems).

## Eliminar registros de usuario [#delete]

Si se registró accidentalmente en New Relic, consulte [Eliminar una organización](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-unused-org).

Si desea eliminar un registro de usuario, utilice la [UIde usuario de administración de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks).
