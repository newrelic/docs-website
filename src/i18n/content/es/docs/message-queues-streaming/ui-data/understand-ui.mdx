New Relic Queues and Streams ofrece una solución integral para monitorear su clúster de Kafka, brindando información detallada valiosa sobre problemas potenciales como fallas, cuello de botella y limitación.

Este documento lo ayudará a:

* Encuentre sus de Kafka telemetry data en la New Relic UI.
* Comprenda las métricas clave y los datos disponibles en la New Relic UI.
* Aprenda a emplear la New Relic UI para monitorear y observar el rendimiento de su cola de mensajes y transmisión de datos de Kafka.

## Cómo acceder a datos de colas y transmisiones en la New Relic UI

Para ver los datos del clúster en la New Relic UI, vaya <DNT>**[one.newrelic.com](https://one.newrelic.com/) &gt; All capabilities &gt; Queues &amp; Streams**</DNT> a.

<img title="Message-queues-and-streaming-monitoring" alt="Message-queues-and-streaming-monitoring" src="/images/message-queues-streaming.webp" />

Para ver los datos del clúster de Kafka, en la página de **Queues &amp; Streams summary** , en la columna **Name** , seleccione el proveedor del clúster de Kafka. También puede emplear la barra de búsqueda o el filtro para buscar un clúster de Kafka específico según el proveedor, el nombre de la cuenta o su ID de cuenta.

## Métricas y datos clave en la New Relic UI

Con New Relic Queues and Streams para Kafka, obtienes:

* **Instant tiempo real Kafka información valiosa:** Extraiga el cliente clave Kafka métrica sin esfuerzo con el agente de Java para ganar rápidamente tiempo real información valiosa conectando los puntos entre sus hosts y servicios a través de una UI unificada.
* **Detecta valores atípicos y soluciona problemas más rápido:** detecta instantáneamente anomalías en las claves métricas en todos los componentes del cliente de Kafka (productores y consumidores) para garantizar un flujo de datos fluido.
* **Identifique el cuello de botella:** el uso de latencia métrica, las tasas de limitación, la profundidad de la cola y el recuento de reintentos identifican el cuello de botella que afecta el flujo de datos y aumenta la latencia.
* **Descubra problemas ocultos del corredor:** analice las métricas del lado del cliente, como el tiempo de aceleramiento del productor/consumidor y el crecimiento de la tasa de reintentos para detectar posibles problemas del lado del corredor y limitaciones de recursos.
* **Aumente el flujo de datos con la optimización del lado del cliente:** identifique flujos de datos sin comprimir a través de métricas como la relación de compresión promedio del productor e implemente estrategias de compresión eficientes para reducir el tamaño de los datos y mejorar el rendimiento.

### Resumen

<img title="Message-queues-and-streaming-monitoring" alt="Message-queues-and-streaming-monitoring" src="/images/kafka-monitoring.webp" />

* La sección **Summary** proporciona una descripción general del clúster de Kafka, incluida la cantidad de agentes, temas y particiones.

* **Kafka Navigator** proporciona una representación gráfica de los clústeres, intermediarios o temas de Kafka saludables y no saludables. El panal verde representa el estado saludable, mientras que el panal amarillo representa el estado no saludable. Coloque el cursor sobre el mosaico en forma de panal para ver los detalles del clúster, el agente o el tema. También puede hacer clic en el mosaico de panal para ver detalles adicionales del clúster, el agente o el tema.

* **Incoming throughput by clusters (bytes/sec)** proporciona los datos de seriales temporales del rendimiento entrante del clúster Kafka en bytes por segundo.

* **By Cluster bar chart visualisations** proporcionan un recuento de temas que se escriben o leen desde todos los agentes en el clúster de Kafka, junto con la cantidad de particiones y agentes en estos clústeres.

### Temas

<img title="Message-queues-and-streaming-monitoring" alt="Message-queues-and-streaming-monitoring" src="/images/kafka-topics.webp" />

La sección de temas proporciona una vista detallada de las 20 principales entidades de temas en el clúster de Kafka, incluido el rendimiento entrante y saliente y la tasa de mensajes. Puede hacer clic en la entidad del tema para ver detalles adicionales del tema, como bytes de entrada, bytes de salida, mensajes de entrada, mensajes recuperados, etc.
