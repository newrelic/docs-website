Nuestro lenguaje de consulta New Relic, NRQL, tiene límites de tasa establecidos para cada organización de New Relic para garantizar que la tasa de consultas de nuestros clientes no afecte la experiencia de otros clientes.

Rara vez debería encontrar límites en su consulta NRQL. Aquí hay algunas pautas a seguir para garantizar que la limitación de consultas sea poco común:

* Limite la cantidad de consultas complejas (por ejemplo, consultas con cláusulas `FACET` o `TIMESERIES` , o consultas de más de un millón de eventos) que se ejecutan al mismo tiempo.
* Limite el número de consultas que se ejecutan simultáneamente durante períodos prolongados a un máximo de cinco, especialmente si incluyen consultas complejas.

Para ver si está alcanzando los límites relacionados con la consulta, utilice la [UIde límites](/docs/data-apis/manage-data/view-system-limits/).

## Límites de consulta NRQL [#query-limits]

Tenemos límites en:

* Cuánto tiempo puede ejecutarse una consulta antes de generar errores y detenerse.
* El número de consultas que puedes realizar en un rango de tiempo determinado.

### Límite de duración de la consulta [#query-duration]

El límite de duración de la consulta es cuánto tiempo puede ejecutarse una consulta NRQL antes de que deje de ejecutarse y se considere un error. Este límite depende de su [opción de datos](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing#data-prices):

* Opción de datos original: 1 minuto

* Datos más:

  * 10 minutos (usando el [generador de consultas](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder) o [NerdGraph](/docs/apis/nerdgraph/examples/async-queries-nrql-tutorial))
  * 2 minutos en otras ubicaciones UI además del generador de consultas

### Límites en el número de consultas [#query-count-limits]

Este límite solo se aplica a las consultas NRQL ejecutadas a través de nuestras API y no se aplica a las consultas ejecutadas desde la UI.

Este límite es de 3.000 consultas por cuenta por minuto. Cuando supere este límite, la consulta podrá ser rechazada hasta que el número de consultas ya no supere el límite.
