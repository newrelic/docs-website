Un sistema Kubernetes es intrínsecamente complejo y tiene fama de ser difícil de comprender. Comprender a Kubernetes como concepto es una tarea compleja en sí misma y querer comprender un sistema de Kubernetes específico solo agrega más complejidad. ¿Cómo se solucionan los problemas de contenedores efímeros que giran hacia arriba y hacia abajo antes de poder acceder a ellos? ¿Cómo entiende la salud de su sistema en su conjunto si tiene cientos de contenedores orquestados a la vez? ¿Cómo se analizan los intrincados sistemas en capas funcionales?

En esta serie de tutoriales, aprenderá cómo monitor su sistema Kubernetes con New Relic y cómo puede optimizar su comprensión de Kubernetes en su conjunto.

## Configurar New Relic:

Para comenzar, debe configurar New Relic con su sistema Kubernetes. Los pasos a continuación lo guiarán a través del proceso:

  <Step>
    ## Instale el agente New Relic:

    <img
      title="New Relic guided install"
      alt="A screenshot showing guided install flow for Kubernetes in the New Relic UI."
      src="/images/kubernetes_screenshot-full_guidedInstall.webp"
    />

    La integración de New Relic Kubernetes le brinda total observabilidad del estado y el rendimiento de su entorno. Con los datos que proporciona, puede monitor el estado de todo su clúster de Kubernetes, verificar pods individuales o profundizar en servicios y aplicaciones específicos. Este agente recopila telemetry data de su clúster utilizando varias integraciones de New Relic, como la integración de eventos de Kubernetes, el agente Prometheus y el complemento New Relic Logs Kubernetes.

    Hay varias formas de integrar su sistema Kubernetes. Para este serial de tutoriales, recomendamos encarecidamente emplear los pasos de instalación guiada que se indican a continuación. Para conocer otras rutas de instalación, consulte [nuestros documentos de instalación de Kubernetes](/install/kubernetes).

  <Step>
    ## Envía datos desde tu aplicación

    Ejecutar docenas o cientos de contenedores causa trabajo y dificultad de mantenimiento. Kubernetes abstrae el contenedor en conceptos de nivel superior, con el clúster como el nivel más alto. Esta abstracción le ayuda a comprender su clúster como un todo, pero dificulta comprender lo que sucede a nivel de aplicación.

    Para reducir esta complejidad, puede informar datos de su aplicación en contenedores a New Relic. Esto no solo le permite ver el estado de su aplicación, sino que también le permite correlacionar los datos de la aplicación con la infraestructura subyacente de Kubernetes.

    <img
      title="apm summary page"
      alt="A screenshot depicting the APM summary page."
      src="/images/apm_screenshot-full_apm-complete-summary-page-.webp"
    />

    [Siga las instrucciones de este documento](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes/) para correlacionar los datos de la aplicación en su pod con el clúster. Utilizará nuestras capacidades de monitoreo del rendimiento de la aplicación (APM) para extraer datos de rendimiento de su aplicación y enviarlos a New Relic.
  </Step>

  <Step>
    ## Envía datos de tus servicios

    La correlación de servicios como Cassandra y MySQL con sus datos de Kubernetes requiere una configuración similar a la forma en que informó los datos de la aplicación.

    <img
      title="New Relic infra page"
      alt="A screenshot showing the New Relic infrastructure monitoring UI."
      src="/images/infrastructure_screenshot-full_infra-main.webp"
    />

    Para correlacionar sus datos, consulte nuestro documento sobre [servicios de monitoreo en Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/monitor-services/monitor-services-running-kubernetes/). Este proceso utilizará nuestra integración en el host para varios servicios compatibles mediante un Helm Chart.
  </Step>

  <Step>
    ## Explora tus datos

    <img
      width="100%;"
      title="New Relic - Kubernetes Navigator entity details"
      alt="New Relic - Kubernetes Navigator entity details"
      src="/images/kubernetes_screenshot-full_navigator-animated.gif"

    Para confirmar que está enviando todos sus datos relacionados con Kubernetes a New Relic, explore los gráficos de Kubernetes. Vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Kubernetes**</DNT>. Explore sus datos y vea si puede tener una idea general del estado de su sistema Kubernetes. Confirme que está viendo datos de todo su clúster, módulo individual y cualquier servicio y aplicación que espere. Una vez que esté listo, continúe con los siguientes pasos.
  </Step>
</Steps>
