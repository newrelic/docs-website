<Callout title="avance">
  Todavía estamos trabajando en esta característica, ¡pero nos encantaría que la probaras!

  Esta característica se proporciona actualmente como parte de un programa de vista previa de conformidad con nuestras [políticas de prelanzamiento](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Cloud Cost Intelligence admite el etiquetado personalizado junto con las etiquetas predeterminadas en los informes de costos y uso (CUR) para la asignación de costos de recursos. Filtre y agrupe etiquetas para un análisis detallado de gastos. En entornos de Kubernetes, aplique etiquetas a los clústeres y recursos. Los menús desplegables de selección de etiquetas se obtienen de los datos de CUR y el evento `K8SContainerSample`.

## Configurar ajustes

<Steps>
  <Step>
    ### Navegue a la configuración general

    Vaya a pestaña <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT> &gt; **General**.
  </Step>

  <Step>
    ### Configurar etiqueta

    Seleccione etiquetas de recursos y etiquetas de Kubernetes de los archivos de costo y uso:

    1. De la lista **Choose Resource tags for cost allocation**, seleccione las etiquetas de recursos para categorizar los costos de la nube en todos los servicios.
    2. De la lista **Choose Kubernetes tags for cost mapping**, seleccione las etiquetas de Kubernetes para categorizar los costos.
  </Step>

  <Step>
    ### Gestionar el acceso a telemetry data

    1. Seleccione la pestaña **Telemetry Data**.
    2. De la lista **Select accounts for telemetry data**, elija las cuentas para los datos de telemetría.

    <Callout variant="important">
      La estimación de costos en tiempo real y la asignación de K8s son compatibles a través de datos de telemetría. Reutiliza la canalización de telemetría existente para las métricas de utilización. No se ingieren datos nuevos, excepto la ingestión de archivos CUR.
    </Callout>
  </Step>
</Steps>
