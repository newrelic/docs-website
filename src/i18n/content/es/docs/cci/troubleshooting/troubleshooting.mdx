Si encuentra problemas durante la configuración, encuentre soluciones a continuación:

## Datos faltantes en los informes

Si no ve el Informe de uso de costos (CUR), la estimación de costos en tiempo real o los datos de asignación de costos de Kubernetes:

* **Motivo:** Requisitos previos incompletos

* **Solución:**

  1. Verifique que se hayan completado todos los requisitos previos de configuración
  2. Verifique la configuración de la configuración
  3. Comuníquese con [el soporte de Cloud Cost Intelligence](mailto:cci-support@newrelic.com) si el problema persiste

## La conexión falla durante la configuración

* **Motivo:** Conexión de red inestable
* **Solución:** Asegúrese de tener una conexión a Internet estable y vuelva a intentar la configuración. Consulte con el administrador de la red las restricciones del firewall si el problema persiste.

## Permisos insuficientes

* **Motivo:** El usuario carece de los permisos necesarios
* **Solución:** Verifique que tiene permisos de administrador o las políticas de IAM requeridas en AWS y New Relic. Póngase en contacto con el administrador si es necesario.

## Error de configuración de ARN

* **Mensaje:** &quot;El ARN que ingresó no permite el acceso correcto a su cuenta de AWS. Verifique el ARN y vuelva a intentarlo&quot;.
* **Motivo:** Configuración de ARN incorrecta
* **Solución:** Verifique que el ARN coincida con la configuración requerida. Corrija los errores y vuelva a intentar la configuración.

## Riesgo de seguridad con identificación externa

* **Mensaje:** &quot;Riesgo de seguridad: el ARN que ingresó tiene licencias amplias. Se puede emplear cualquier &apos;id externo&apos; para asumir el rol. Agregue el ID externo proporcionado &apos;%s&apos; como externalId al crear el rol de AWS.
* **Motivo:** Política de ID externo demasiado permisiva
* **Solución:** Actualice la configuración del rol de AWS para usar el ID externo proporcionado como externalId.

## Bucket de S3 no encontrado

* **Motivo:** El bucket de S3 no existe o el nombre del bucket es incorrecto
* **Solución:** Verifique que el nombre del bucket coincida con su cuenta de AWS. Asegúrese de que el prefijo de la ruta de S3 sea `cost-and-usage-reports`. Verifique la ortografía, cree un bucket si es necesario o corrija el nombre.

## Nombre de conexión duplicado

* **Mensaje:** &quot;%s Inteligencia para costos en la nube El nombre de la conexión AWS ya existe. Ingrese un nuevo nombre de conexión.&quot;
* **Motivo:** El nombre de la conexión ya está en uso
* **Solución:** Elija un nombre de conexión único y vuelva a intentarlo.

## Configuración de cuenta de pagador duplicado

* **Mensaje:** &quot;La cuenta de AWS Payer%s ya está vinculada a esta cuenta de New Relic&quot;.
* **Motivo:** Cuenta de pagador de AWS ya configurada
* **Solución:** Verifique que la cuenta de pagador no esté ya vinculada. Utilice una cuenta de pagador diferente si es necesario.

## Soporte adicional

Si los problemas persisten, comuníquese con [el soporte de Cloud Cost Intelligence](mailto:cci-support@newrelic.com).

## Configuración de CloudFormation

Si **Create CFT Stack** falla:

1. Busque la fila Status en la pestaña/columna Events de la pila seleccionada.
2. Busque **CREATE\_FAILED** y comparta el mensaje de motivo del estado con New Relic.
3. Vaya a **CloudWatch** y, en **CloudWatch logs**, seleccione el nombre de su pila de CloudFormation y haga clic en **View in Logs console**.
4. En **Log Streams**, abra el log y comparta el log con el soporte de New Relic.
