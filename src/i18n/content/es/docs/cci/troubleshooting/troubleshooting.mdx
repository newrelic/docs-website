Al configurar Inteligencia para costos en la nube, es posible que encuentres algunos problemas. A continuación se presentan algunos problemas comunes, posibles razones y pasos sugeridos para la resolución de problemas que lo ayudarán a resolverlos de manera efectiva:

## Los datos no aparecen en New Relic

Si no ve el Reporte de uso de costos (CUR), la estimación de costos en tiempo real o los datos de asignación de costos de Kubernetes en su cuenta de New Relic, aquí hay algunos pasos para solucionar el problema:

* **Motivo:** Puede que existan lagunas en el cumplimiento de los prerrequisitos.

* **Solución:**

  1. Verifique nuevamente que completó todos los requisitos previos requeridos.
  2. Verifique su configuración de instalación.
  3. Si el problema persiste, comunicar con [el soporte de Inteligencia para costos en la nube](mailto:cci-support@newrelic.com).

## Caída de conexión

* **Motivo:** Una conexión de red inestable podría provocar que se caiga la configuración.
* **Solución:** cerciorar de tener una conexión a Internet estable e intente realizar la configuración nuevamente. Si el problema persiste, consulte con su administrador de red sobre posibles restricciones del firewall.

## Privilegios de usuario inadecuados

* **Motivo:** El usuario no tiene las licencias necesarias para configurar Inteligencia para costos en la nube.
* **Solución:** revise las licencias de su cuenta de AWS y New Relic. Cerciorar de tener los privilegios adecuados para realizar la configuración. Comunicar con su administrador si es necesario.

## Error de configuración de ARN

* **Mensaje:** &quot;El ARN que ingresó no permite el acceso correcto a su cuenta de AWS. Verifique el ARN y vuelva a intentarlo&quot;.
* **Motivo:** Configuración de ARN incorrecta que no permite el acceso.
* **Solución:** Verifique nuevamente el ARN que ingresó y cerciorar de que coincida con la configuración requerida para el acceso. Corrija cualquier error y vuelva a intentar la configuración.

## Riesgo de seguridad con identificación externa

* **Mensaje:** &quot;Riesgo de seguridad: el ARN que ingresó tiene licencias amplias. Se puede emplear cualquier &apos;id externo&apos; para asumir el rol. Agregue el ID externo proporcionado &apos;%s&apos; como externalId al crear el rol de AWS.
* **Motivo:** Una política de identificación externa demasiado permisiva supone un riesgo de seguridad.
* **Solución:** actualice la configuración de su rol de AWS para cerciorar de que el ID externo proporcionado se use como externalId.

## El cubo no existe

* **Motivo:** el bucket de AWS S3 especificado no existe.
* **Solución:** Verifique que el nombre del depósito coincida con el de su cuenta de AWS. El nombre también debe tener el prefijo `cost-and-usage-reports`. Cree el depósito si no existe o corrija el nombre en la configuración.

## Error en el nombre del depósito

* **Motivo:** el depósito AWS S3 especificado no existe o hay un error tipográfico en el nombre del depósito.
* **Solución:** Verifique que el nombre del depósito coincida con el de su cuenta de AWS e incluya el prefijo `cost-and-usage-reports`. Cerciorar de que el nombre esté escrito correctamente. Si el depósito no existe, créelo o corrija el nombre en la configuración.

## Nombre de conexión duplicado

* **Mensaje:** &quot;%s Inteligencia para costos en la nube El nombre de la conexión AWS ya existe. Ingrese un nuevo nombre de conexión.&quot;
* **Motivo:** El nombre de conexión que ingresó ya está en uso.
* **Solución:** elija un nombre de conexión único y vuelva a intentarlo.

## Configuración de cuenta de pagador duplicado

* **Mensaje:** &quot;La cuenta de AWS Payer%s ya está vinculada a esta cuenta de New Relic&quot;.
* **Motivo:** Intentar configurar la misma cuenta de pagador de AWS más de una vez.
* **Solución:** Verifique que la cuenta de pagador de AWS no esté ya vinculada. Emplee una cuenta de pagador diferente si es necesario.

## Soporte adicional

Si su problema persiste luego de seguir estos pasos de resolución de problemas, comunicar con [el soporte de Inteligencia para costos en la nube](mailto:cci-support@newrelic.com).
