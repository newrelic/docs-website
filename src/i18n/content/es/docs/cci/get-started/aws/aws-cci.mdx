Configure su entorno de AWS para Cloud Cost Intelligence mediante la configuración de la facturación consolidada, los Informes de costos y uso (CUR), los roles y políticas de Identity and Access Management (IAM) y las regiones de AWS necesarias.

## Requisitos previos

Asegúrese de tener lo siguiente antes de iniciar cualquiera de los métodos de configuración:

### Requerido para todas las configuraciones

* **Cuenta de AWS con facturación consolidada**: [Configurar la facturación consolidada para AWS Organizations](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html). Esto es necesario para agregar datos de costos de todas las cuentas de su organización, lo que permite un análisis de costos integral en toda su infraestructura de AWS.
* **Privilegios de IAM**: Se requiere acceso administrativo a la cuenta de AWS de destino para implementar recursos, crear buckets de S3, políticas de IAM y roles de IAM.

### Requerido solo para la configuración automatizada

* **Clave de API de New Relic**: Genera una clave de API válida para la creación del stack. Recomendamos crear una clave dedicada para la integración de CCI. Genere esto en la UI de New Relic: [Menú de usuario &gt; API Keys](https://onenr.io/07wklp68lRL).

## Elija su método de configuración

Cloud Cost Intelligence ofrece dos formas de configurar la integración con AWS:

* [**Automatice AWS con CloudFormation (Recomendado)**](/docs/cci/get-started/aws/automated-setup-cloudformation): Despliegue rápidamente una pila de CloudFormation preconfigurada que aprovisiona automáticamente todos los recursos de AWS necesarios. Ideal para una incorporación más rápida con una configuración manual mínima.
* [**Integre manualmente su cuenta de AWS**](/docs/cci/get-started/aws/manual-integrate-aws-account): Configure manualmente cada recurso de AWS para tener un control total del proceso de configuración. Ideal para organizaciones con requisitos específicos de cumplimiento o personalización.
