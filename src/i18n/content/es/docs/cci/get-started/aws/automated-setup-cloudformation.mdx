Utilice la plantilla automatizada de AWS CloudFormation para acelerar su configuración de <DNT>**Cloud Cost Intelligence**</DNT>. El stack preconfigurado aprovisiona automáticamente el entorno de AWS requerido, incluida la configuración del bucket de S3, la configuración de exportación de CUR, la [verificación de la configuración de Trusted Advisor](/docs/cci/financial-intelligence/cci-recommendations/#aws-trusted-advisor) y los roles y políticas de IAM necesarios. Revise todos los permisos de los recursos en la consola de AWS CloudFormation antes de desplegar.

## Consideraciones de costos

La plantilla crea automáticamente un bucket de S3 para la ingesta de datos de CUR. La región predeterminada es `us-east-2`, pero puede cambiarla utilizando los parámetros de CloudFormation para adaptarse a las necesidades de su infraestructura.

## Instrucciones de configuración

<Steps>
  <Step>
    ### Acceda a la inteligencia de costos de la nube en New Relic

    Navegar a <DNT>[**one.newrelic.com**](https://one.newrelic.com/)</DNT> <DNT>**Integrations &amp; Agents**</DNT> <DNT>**Cloud Cost Intelligence - AWS**</DNT>.
  </Step>

  <Step>
    ### Elegir la automatización de CloudFormation

    Seleccione <DNT>**Automate AWS with CloudFormation (Recommended)**</DNT> y haga clic en <DNT>**Continue**</DNT>.
  </Step>

  <Step>
    ### Lanzar CloudFormation en AWS

    1. Confirma el nombre de tu conexión y selecciona tu región de datos de New Relic.
    2. Haga clic en <DNT>**Launch your CloudFormation in AWS**</DNT>.
    3. Inicie sesión en su cuenta de AWS con <DNT>**Admin role**</DNT> cuando se le solicite. La pantalla <DNT>**CloudFormation Stacks**</DNT> se abre en la consola de AWS.
  </Step>

  <Step>
    ### Configurar parámetros de la pila

    En el formulario <DNT>**Create Stack**</DNT>, configure los siguientes parámetros:

    <DNT>**Pre-populated parameters (do not modify):**</DNT>

    * <DNT>**New Relic Account ID**</DNT>: Prellenado. No cambiar.
    * <DNT>**New Relic Data Region**</DNT>: Prellenado. No cambiar.
    * <DNT>**External ID**</DNT>: Prellenado. No cambiar.

    <DNT>**Required parameters:**</DNT>

    * <DNT>**NewRelicAPIKey**</DNT>: Ingrese su clave de API de New Relic. Genere esto en la UI de New Relic: menú de usuario &gt; API Keys.
    * <DNT>**Stack Name**</DNT>: Valor predeterminado proporcionado. Edite si es necesario, pero asegúrese de que el nombre sea único a nivel global en todos los stacks de AWS CloudFormation. Recomendamos mantener el valor predeterminado.
    * <DNT>**Bucket Name**</DNT>: Valor predeterminado proporcionado. Edite si es necesario, pero asegúrese de que el nombre sea globalmente único en todos los buckets de S3.
    * <DNT>**Connection Name**</DNT>: Valor predeterminado proporcionado. Edite si es necesario para sus convenciones de nomenclatura.

    <DNT>**Optional parameters:**</DNT>

    * <DNT>**Grant Access to Trusted Advisor**</DNT>: El valor predeterminado es <DNT>**Yes**</DNT>. Cuando está habilitado, permite verificar si la información valiosa de costos de <DNT>**Trusted Advisor**</DNT> está habilitada para su cuenta y recupera recomendaciones. Si elige <DNT>**No**</DNT> o <DNT>**Trusted Advisor**</DNT> no está habilitado, las recomendaciones de <DNT>**Trusted Advisor**</DNT> no estarán disponibles.

    <Callout variant="important">
      <DNT>**CCI**</DNT> Las recomendaciones nativas son independientes de <DNT>**Trusted Advisor**</DNT> y funcionarán sin importar esta configuración.
    </Callout>

    Todos los demás parámetros son opcionales y se pueden configurar según sea necesario.
  </Step>

  <Step>
    ### Crear la pila

    Seleccione <DNT>**I acknowledge that AWS CloudFormation might create IAM resources**</DNT> y haga clic en <DNT>**Create stack**</DNT>.
  </Step>

  <Step>
    ### Verificar la creación de la pila

    Cuando la creación del stack se completa correctamente, el estado muestra <DNT>**CREATE\_COMPLETE**</DNT>.
  </Step>

  <Step>
    ### Verificar la generación del reporte CUR

    Espere 24 horas, luego verifique que aparezca un informe CUR en formato parquet en su bucket de AWS.

    <Callout variant="important">
      La ruta a continuación es un ejemplo. Su ruta real tendrá diferentes marcas de tiempo, UUIDs y periodos de facturación.

      <DNT>**Amazon S3**</DNT> &gt; <DNT>**Buckets**</DNT> &gt; <DNT>**bucket\_name**</DNT> &gt; <DNT>**cost-and-usage-reports/**</DNT> &gt; <DNT>**bucket\_name-cci/data**</DNT> &gt; <DNT>**BILLING\_PERIOD=2025-12/**</DNT> &gt; <DNT>**2025-12-24T03:05:19.886Z-a2e23b2a-588a-4305-97fd-be712f555b75/**</DNT> &gt; <DNT>**bucket\_name-cci-00001.snappy.parquet**</DNT>
    </Callout>

    La configuración de su integración está completa.
  </Step>
</Steps>

## Limitaciones

La plantilla automatizada de <DNT>**CloudFormation**</DNT> no:

* Cree o configure cualquier recurso de cómputo o de red.
* Habilite cualquier servicio adicional de AWS o actualice el nivel de soporte.
* Automatice el proceso de ingesta de logs, trazas y cualquier otra métrica.
* Cree roles o políticas de acceso de escritura.

## Políticas y permisos configurados

La pila automatizada configura los siguientes niveles de acceso:

* <DNT>**Billing Data Access**</DNT>: Permiso de solo lectura al bucket de S3 específico creado para almacenar los Informes de costos y uso.
* <DNT>**Service Pricing Access**</DNT>: Acceso de solo lectura a la API de AWS Pricing para correlacionar su uso con las tarifas públicas.
* <DNT>**Configuration Automation**</DNT>: Permisos limitados necesarios para generar las exportaciones de CUR y verificar el estado de la integración durante la fase de configuración.
* <DNT>**Optional Optimization Access**</DNT>: (Si se selecciona) Acceso de solo lectura a AWS Trusted Advisor para mostrar recomendaciones de ahorro de costos.
