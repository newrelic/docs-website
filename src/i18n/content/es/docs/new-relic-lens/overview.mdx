Como usuario de New Relic, es posible que necesite analizar sus datos de telemetría junto con datos almacenados en múltiples bases de datos, almacenes de datos y hojas de cálculo. Exportar datos manualmente de cada fuente y combinarlos con sus datos de observabilidad fuera de línea consume tiempo y reduce la productividad. Además, a menudo no puede consolidar todos los datos en una sola base de datos debido a requisitos de cumplimiento, seguridad u operativos.

<DNT>New Relic Lens</DNT> aborda estos desafíos. <DNT>Lens</DNT> es una plataforma unificada de consultas dentro de New Relic que le permite conectar y consultar múltiples fuentes de datos directamente desde New Relic. Escriba una consulta en <DNT>Lens</DNT> utilizando ANSI SQL y realice uniones entre bases de datos para analizar y correlacionar datos de varias fuentes. <DNT>Lens</DNT> envía el trabajo computacional a cada fuente de datos, que procesa su parte de la consulta y devuelve solo los resultados. <DNT>Lens</DNT> luego combina estos resultados, minimizando la transferencia de datos y manteniendo la seguridad al no ingerir nunca datos sin procesar de sus fuentes externas.

## Cómo funciona [#how-it-works]

El uso de <DNT>Lens</DNT> implica los siguientes pasos:

<Steps>
  <Step>
    ### Habilitar Lens

    <DNT>Lens</DNT> está disponible como vista previa pública. Consulte con su <DNT>Organization Manager</DNT> para optar por participar desde la página <DNT>**[Previews &amp; Trials](https://one.newrelic.com/admin-portal/promotion-management/home)**</DNT>.
  </Step>

  <Step>
    ### Comprobar permisos

    Si es un <DNT>Organization Manager</DNT>, tiene los permisos necesarios para acceder a todas las funciones de <DNT>Lens</DNT> de forma predeterminada. Si no es un <DNT>Organization Manager</DNT>, comuníquese con su <DNT>Authentication Domain Manager</DNT> para [crear un rol personalizado con los permisos necesarios](/docs/new-relic-lens/overview/#permissions).
  </Step>

  <Step>
    ### Configuración <DNT>connectors</DNT>

    <DNT>Lens</DNT> incluye <DNT>system connectors</DNT> integrado para datos de NRDB. Con los permisos necesarios, puede configurar [<DNT>data connectors</DNT>](/docs/new-relic-lens/set-up-connectors) para conectarse a fuentes de datos externas:

    * **Almacenes de datos en la nube**: <DNT>Snowflake</DNT>, <DNT>Redshift</DNT>, <DNT>ClickHouse</DNT>
    * **Bases de datos relacionales**: <DNT>PostgreSQL</DNT>, <DNT>MySQL</DNT>, <DNT>SQL Server</DNT>
    * **Bases de datos documentales**: <DNT>MongoDB</DNT>, <DNT>Elasticsearch</DNT>
    * **Hojas de cálculo**: <DNT>Google Sheets</DNT>
    * **Lagos de datos**: <DNT>Iceberg</DNT>
    * **Métricas y monitoreo**: <DNT>Prometheus</DNT>, <DNT>AWS CloudWatch</DNT>
  </Step>

  <Step>
    ### Consultar datos

    Escribe consultas SQL en el [Query Builder, Notebooks o dashboards personalizados](/docs/new-relic-lens/query-data) usando el formato `connection_name.schema.table` para referenciar tablas y unir datos de varias fuentes.
  </Step>

  <Step>
    ### Crear <DNT>materialized views</DNT> (opcional)

    Para consultas complejas que se ejecutan con frecuencia, puede guardar los resultados como [<DNT>materialized views</DNT>](/docs/new-relic-lens/materialized-views) para reducir costos y mejorar el rendimiento.
  </Step>
</Steps>

## Casos de uso [#use-cases]

Lens permite varios escenarios de análisis de datos:

* **Análisis de presupuesto**: Compare los datos de utilización de New Relic de <DNT>NRDB</DNT> con los datos de presupuesto almacenados en <DNT>Snowflake</DNT> para hacer un seguimiento de los gastos frente a los presupuestos asignados.
* **Optimización del flujo de trabajo**: Reemplace múltiples pasos manuales de exportación/importación (de una base de datos a otra) con una única consulta SQL que combine todos los datos necesarios, reduciendo costos y mejorando la eficiencia.
* **Agregar datos de varias regiones**: Rastree el tráfico de usuarios en diversas regiones geográficas al unir datos de diferentes fuentes de datos.
* **Optimización del rendimiento con <DNT>materialized views</DNT>**: Un dashboard ejecutivo combina datos de más de 5 bases de datos y se actualiza semanalmente. Al almacenar los resultados de la consulta `JOIN` de múltiples bases de datos en un <DNT>materialized view</DNT>, evitas ejecutar la consulta computacionalmente costosa cada vez que alguien carga el dashboard. Esto reduce los costos de consulta y mejora significativamente la velocidad de carga de la página.

## Dónde puede usar Lens [#where-to-use]

Puede usar Lens en las siguientes áreas de la plataforma de New Relic:

* **Constructor de consultas**: Escriba y ejecute consultas SQL directamente en el constructor de consultas.
* <DNT>**Notebooks**</DNT>: Cree bloques de SQL en notebooks para combinar consultas con visualizaciones y documentación en Markdown.
* **Dashboards de clientes**: Agregue gráficos basados en SQL a sus paneles privados.

<Callout variant="important">
  Los gráficos basados en SQL no se admiten en los paneles públicos.
</Callout>

## Roles y permisos de usuario [#permissions]

<DNT>Connector</DNT> y el acceso <DNT>materialized view</DNT> están controlados por permisos RBAC específicos. Los usuarios con el rol <DNT>**Organization Manager**</DNT> tienen todos los permisos de RBAC relacionados con <DNT>Lens</DNT> de forma predeterminada.

Para otros usuarios, el <DNT>**Authentication Domain Manager**</DNT> debe [crear un rol personalizado con alcance de organización a través de NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#organization-scoped-role) con los permisos de <DNT>Lens</DNT> requeridos. Al crear el rol, agregue los ID de permisos de las tablas siguientes al arreglo `permissionIds`.

  /* UI-based permission assignment (for future use when UI support is available): For other users, the &lt;DNT&gt;**Authentication Domain Manager**&lt;/DNT&gt; needs to create an [organization-scoped custom role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) with the required &lt;DNT&gt;Lens&lt;/DNT&gt; permissions from the &lt;DNT&gt;**Query Your Data**&lt;/DNT&gt; section. check... */
</DNT>
