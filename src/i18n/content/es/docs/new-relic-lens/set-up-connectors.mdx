Antes de consultar datos con [<DNT>Lens</DNT>](/docs/new-relic-lens/overview), necesita configurar conexiones a sus fuentes de datos. <DNT>Lens</DNT> proporciona tanto <DNT>system connectors</DNT> para acceder a los datos de NRDB como <DNT>data connectors</DNT> para conectarse a bases de datos externas y otras fuentes de datos.

## <DNT>Data connectors</DNT> [#data-connectors]

<DNT>Data connectors</DNT> conectar <DNT>Lens</DNT> a fuentes de datos externas como bases de datos, almacenes de datos y hojas de cálculo. Para crear, editar o eliminar <DNT>connectors</DNT>, necesita [permisos RBAC específicos](/docs/new-relic-lens/overview#permissions).

### Control de acceso [#access-control]

<DNT>Lens</DNT> proporciona dos formas de controlar quién puede consultar <DNT>connectors</DNT>:

* **Permisos RBAC**: Los usuarios con permisos RBAC <DNT>**Lens connectors**</DNT> pueden consultar todos los <DNT>connectors</DNT> de la organización.
* **Control de acceso detallado**: Los usuarios sin permisos RBAC amplios aún pueden consultar <DNT>connectors</DNT> específicos si se les otorga acceso detallado a esos <DNT>connectors</DNT>.

### Crear un <DNT>connector</DNT> [#create-connector]

Para configurar un <DNT>data connector</DNT>:

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Connectors**</DNT>.

2. Haga clic en <DNT>**Create a connector**</DNT>.

3. Seleccione su tipo de <DNT>connector</DNT> del menú desplegable.

4. Ingrese la información de conexión requerida para el tipo de <DNT>connector</DNT> seleccionado.

5. Para otorgar acceso a este conector a usuarios o grupos que no tienen permisos RBAC, configure el control de acceso detallado desde la sección <DNT>**Access control**</DNT>:

   1. Seleccione el <DNT>**Auth domain**</DNT> que contiene los usuarios o grupos.
   2. Seleccione el <DNT>**User or group**</DNT>.
   3. Seleccione el rol<DNT>**Lens Viewer**</DNT>.
   4. Para agregar más usuarios o grupos, haga clic en <DNT>**Add**</DNT> y repita los pasos anteriores.

6. Haga clic en <DNT>**Create**</DNT>.

Después de crear un <DNT>connector</DNT>:

* Para ver el esquema o eliminar el <DNT>connector</DNT>, seleccione el menú <Icon name="fe-more-horizontal" /> de la fila <DNT>connector</DNT>.
* Para actualizar la configuración de control de acceso, seleccione el <DNT>connector</DNT>, agregue o elimine concesiones de acceso y guarde los cambios.
* Para actualizar la configuración de <DNT>connector</DNT>, elimine y vuelva a crear el <DNT>connector</DNT> con los nuevos detalles.

    ### Generar la clave de credenciales de la cuenta de servicio [#generate-credentials-key]

    Para autenticar <DNT>Lens</DNT> con <DNT>Google Sheets</DNT>, debe crear una cuenta de servicio de <DNT>Google Cloud</DNT> y generar una clave JSON.

    <Steps>
      <Step>
        ### Crear una cuenta de servicio [#create-service-account]

        Para crear una cuenta de servicio de <DNT>Google Cloud</DNT>, siga las instrucciones en [Crear cuentas de servicio](https://cloud.google.com/iam/docs/service-accounts-create).
      </Step>

      <Step>
        ### Configurar permisos para la creación de claves [#set-up-permissions]

        Para crear claves para su cuenta de servicio, necesita los permisos adecuados. Para obtener información sobre los permisos necesarios, consulte [Permisos necesarios](https://cloud.google.com/iam/docs/keys-create-delete#required-permissions).
      </Step>

      <Step>
        ### Crear una clave JSON [#create-json-key]

        Para generar una clave JSON para su cuenta de servicio, siga los pasos en [Crear una clave de cuenta de servicio](https://cloud.google.com/iam/docs/keys-create-delete#creating).
      </Step>

      <Step>
        ### Convertir la clave JSON a base64 [#convert-json-key-to-base64]

        Para usar la clave JSON en <DNT>Lens</DNT>, convierta todo el contenido del archivo JSON a codificación base64 y guárdelo en un archivo de texto.
      </Step>
    </Steps>

    ### Configurar el acceso a Google Sheets [#setup-gsheets-access]

    Después de generar la clave de credenciales, configure el acceso a sus Google Sheets.

    <Steps>
      <Step>
        ### Comparta sus hojas con la cuenta de servicio [#share-sheets-with-service-account]

        Para permitir que <DNT>Lens</DNT> acceda a sus datos, otorgue permisos de lectura a la dirección de correo electrónico de la cuenta de servicio para cada hoja de cálculo de Google que desee conectar.
      </Step>

      <Step>
        ### Crear una hoja de metadatos [#create-metadata-sheet]

        Para administrar las hojas disponibles para <DNT>Lens</DNT>, cree una nueva hoja de cálculo de Google con las siguientes cuatro columnas:

        * **Sheetname**: El nombre de cada hoja a la que desea acceder con <DNT>Lens</DNT>. Todas las hojas listadas deben ser accesibles para la cuenta de servicio.
        * **Sheetid**: El identificador único de cada hoja. Puede encontrar esto en la URL de la hoja: `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`.
        * **Propietario**: El propietario de la hoja. Usar `Trino` como valor predeterminado.
        * **Notas**: Cualquier nota o descripción adicional para la hoja.
      </Step>

      <Step>
        ### Compartir la hoja de metadatos [#share-metadata-sheet]

        Para permitir que <DNT>Lens</DNT> lea los metadatos, otorgue permiso de lectura a su cuenta de servicio para la hoja de metadatos.
      </Step>

      <Step>
        ### Copiar el ID de la hoja de metadatos [#copy-metadata-sheet-id]

        Para configurar el <DNT>connector</DNT>, copie el ID de la hoja de la URL de la hoja de metadatos. El ID de la hoja aparece en la URL como: `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`.
      </Step>
    </Steps>

    ### Conceder <DNT>Lens</DNT> [#grant-iceberg-access]

    Para permitir que <DNT>Lens</DNT> consulte sus tablas Iceberg, cree una política y un rol de IAM en su cuenta de AWS que otorguen acceso de lectura a su Glue Data Catalog y a su bucket de S3.

    <Callout variant="important">
      El Glue Data Catalog y el bucket de S3 que contiene sus datos de Iceberg deben estar en la misma región de AWS.
    </Callout>

    <Steps>
      <Step>
        ### Crear una política de IAM [#create-iceberg-policy]

        [Cree una política de IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) con los siguientes permisos para permitir que <DNT>Lens</DNT> lea los metadatos de Glue y los archivos de datos en S3. Reemplace `{REGION}`, `{ACCOUNT_ID}`, `{DATABASE_NAME}` y `{S3_BUCKET_NAME}` con sus valores. Tome nota del nombre de la política que ingrese, ya que lo necesitará para adjuntar esta política al rol de IAM.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Action": [
                "glue:GetDatabase",
                "glue:GetTable",
                "glue:GetTables",
                "glue:SearchTables",
                "glue:GetPartition",
                "glue:GetPartitions",
                "glue:BatchGetPartition"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:catalog",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:database/{DATABASE_NAME}",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:table/*"
              ]
            },
            {
              "Action": [
                "s3:GetObject",
                "s3:ListBucket",
                "s3:GetBucketLocation"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:s3:::{S3_BUCKET_NAME}",
                "arn:aws:s3:::{S3_BUCKET_NAME}/*"
              ]
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### Cree un rol de IAM con una relación de confianza [#create-iceberg-role]

        [Cree un rol de IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) y configure la relación de confianza para permitir que la cuenta de servicio de <DNT>Lens</DNT> lo asuma mediante un ID externo único. Al configurar el rol, utilice estos valores:

        * **Tipo de entidad de confianza**: Cuenta de AWS
        * **ID de cuenta**: `017663287629` (cuenta de AWS de New Relic)
        * **Requerir ID externo**: Habilitado. Ingrese una cadena única de su elección y guárdela para la configuración de <DNT>Lens</DNT> <DNT>connector</DNT>.
        * **Política de permisos**: Busque el nombre de la política que creó en el paso anterior y selecciónelo.

        La política de confianza debe coincidir con el siguiente formato. Reemplace `<YOUR_EXTERNAL_ID>` por el ID externo que ingresó.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        Después de crear el rol, abra la página de resumen del rol y copie el **Role ARN**. Necesitará este ARN y el ID externo al configurar <DNT>Lens</DNT> <DNT>connector</DNT>.
      </Step>
    </Steps>
  </Collapser>

    ### Conceder <DNT>Lens</DNT> [#grant-cloudwatch-access]

    Para permitir que <DNT>Lens</DNT> consulte sus métricas de CloudWatch, cree una política y un rol de IAM en su cuenta de AWS que otorguen acceso de lectura a CloudWatch.

    <Steps>
      <Step>
        ### Crear una política de IAM [#create-cloudwatch-policy]

        [Cree una política de IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) con los siguientes permisos para permitir que <DNT>Lens</DNT> lea las métricas de CloudWatch de su cuenta de AWS. Anote el nombre de la política que ingrese, ya que lo necesitará para adjuntar esta política al rol de IAM.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Action": [
                "cloudwatch:ListMetrics",
                "cloudwatch:GetMetricData",
                "cloudwatch:GetMetricStatistics",
                "tag:GetResources"
              ],
              "Resource": "*"
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### Cree un rol de IAM con una relación de confianza [#create-cloudwatch-role]

        [Cree un rol de IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) y configure la relación de confianza para permitir que la cuenta de servicio de <DNT>Lens</DNT> lo asuma mediante un ID externo único. Al configurar el rol, utilice estos valores:

        * **Tipo de entidad de confianza**: Cuenta de AWS
        * **ID de cuenta**: `017663287629` (cuenta de AWS de New Relic)
        * **Requerir ID externo**: Habilitado. Ingrese una cadena única de su elección y guárdela para la configuración de <DNT>Lens</DNT> <DNT>connector</DNT>.
        * **Política de permisos**: Busque el nombre de la política que creó en el paso anterior y selecciónelo.

        La política de confianza debe coincidir con el siguiente formato. Reemplace `<YOUR_EXTERNAL_ID>` por el ID externo que ingresó.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        Después de crear el rol, abra la página de resumen del rol y copie el **Role ARN**. Necesitará este ARN y el ID externo al configurar <DNT>Lens</DNT> <DNT>connector</DNT>.
      </Step>
    </Steps>
  </Collapser>
