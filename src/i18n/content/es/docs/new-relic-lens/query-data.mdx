Puede escribir consultas <DNT>Lens</DNT> en **Query Builder**, <DNT>**Notebooks**</DNT> y **dashboards personalizados**. Esta página explica cómo consultar datos en cada plataforma.

<Callout variant="tip">
  Al consultar fuentes de datos, <DNT>Lens</DNT> no restringe el número de filas que puede recuperar de fuentes externas. Sin embargo, las consultas a <DNT>NRDB</DNT> a través de <DNT>Lens</DNT> mantienen el mismo límite de 5,000 filas que <DNT>NRQL</DNT>.
</Callout>

## Requisitos previos [#prerequisites]

Antes de comenzar:

* Verifique que tenga los [permisos](/docs/new-relic-lens/overview#permissions) necesarios para consultar fuentes de datos.
* Si necesita consultar fuentes de datos externas, asegúrese de que los [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors) estén configurados.

## Consulta en el constructor de consultas [#query-builder]

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
2. Desde el cajón en la parte inferior de la página, abra la consola <DNT>**Query your data**</DNT>.
3. En el editor de consultas, seleccione <DNT>**Lens data (SQL)**</DNT> como el lenguaje de consulta.
4. Si desea consultar datos de NRDB, seleccione la cuenta en el selector de cuentas. Otros <DNT>data connectors</DNT> están disponibles a nivel de organización y no requieren selección de cuenta.
5. Escribe tu consulta SQL utilizando el [formato de referencia de tabla](#table-format): `connection_name.schema.table`
6. Haga clic en <DNT>**Run**</DNT>. El gráfico se renderiza según los resultados de su consulta.

## Guardar consulta en <DNT>Notebooks</DNT> [#notebooks]

Los [Notebooks](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/) son documentos compartibles que combinan consultas, visualizaciones y documentación en Markdown.

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
2. Desde el cajón en la parte inferior de la página, abra la consola <DNT>**Query your data**</DNT>.
3. Habilite el interruptor junto a <DNT>**Notebooks**</DNT>. En la parte inferior de la página, aparece el botón <DNT>**+ SQL**</DNT>.
4. Haga clic en <DNT>**+ SQL**</DNT> para agregar un bloque SQL.
5. Si desea consultar datos de NRDB, seleccione la cuenta en el selector de cuentas. Otros <DNT>data connectors</DNT> están disponibles a nivel de organización y no requieren selección de cuenta.
6. Escribe tu consulta SQL utilizando el [formato de referencia de tabla](#table-format): `connection_name.schema.table`
7. Haga clic en <DNT>**Run**</DNT>. El gráfico se renderiza según los resultados de su consulta.
8. Haga clic en <DNT>**Save**</DNT>.

Para obtener más orientación, consulte la [documentación de Notebooks](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/).

## Consulta en paneles personalizados [#dashboards]

Puede agregar gráficos basados en SQL a sus [paneles personalizados](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards):

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**</DNT>.
2. [Crea un nuevo dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create-new) o abre uno existente.
3. Haga clic en <DNT>**+ Add widget**</DNT> y seleccione un tipo de gráfico.
4. En el editor de consultas, seleccione <DNT>**Lens data (SQL)**</DNT> como el lenguaje de consulta.
5. Si desea consultar datos de NRDB, seleccione la cuenta en el selector de cuentas. Otros <DNT>data connectors</DNT> están disponibles a nivel de organización y no requieren selección de cuenta.
6. Escribe tu consulta SQL utilizando el [formato de referencia de tabla](#table-format): `connection_name.schema.table`
7. Haga clic en <DNT>**Run**</DNT>. El gráfico se renderiza según los resultados de su consulta.
8. Haga clic en <DNT>**Add to dashboard**</DNT>. Para obtener más orientación, consulte la [documentación de dashboards personalizados](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/).

<Callout variant="important">
  Los gráficos basados en SQL solo se admiten en dashboards privados. Los paneles públicos no admiten consultas SQL.
</Callout>

## Sintaxis de SQL [#syntax]

<DNT>Lens</DNT> utiliza ANSI SQL para consultar fuentes de datos, por lo que no necesita aprender lenguajes de consulta específicos de bases de datos.

### Formato de referencia de tabla [#table-format]

Al escribir consultas en <DNT>Lens</DNT>, haga referencia a las tablas usando el formato:

```
connection_name.schema.table
```

Por ejemplo:

* `telemetry.Transaction` - Consultar la tabla Transaction de la telemetría (NRDB) <DNT>connector</DNT>
* `snowflake.sales.orders` - Consultar la tabla orders de un <DNT>Snowflake</DNT> <DNT>connector</DNT> llamado &quot;snowflake&quot; en el esquema &quot;sales&quot;
* `gsheets.default.employees` - Consultar un <DNT>Google Sheets</DNT> <DNT>connector</DNT> llamado &quot;gsheets&quot;

### Sentencias SQL admitidas [#supported-sql]

<DNT>Lens</DNT> admite sentencias SQL ANSI estándar, incluidas:

* `SELECT` - Consultar datos de una o más tablas
* `JOIN` - Combine datos de múltiples tablas (incluyendo uniones entre bases de datos)
* `WHERE` - Filtrar resultados
* `GROUP BY` - Datos agregados
* `ORDER BY` - Ordenar resultados
* `LIMIT` - Restringir el número de filas devueltas
* `WITH` (CTEs) - Defina expresiones de tabla comunes para consultas complejas

### Consejos de rendimiento [#performance-tips]

* **Utilice filtros anticipadamente**: Aplique cláusulas `WHERE` para reducir la cantidad de datos procesados.
* **Limitar resultados**: Use `LIMIT` para restringir las filas devueltas al explorar datos.
* **Considere <DNT>materialized views</DNT>**: Para consultas complejas que se ejecutan con frecuencia, cree un <DNT>materialized view</DNT>.
* **Optimice las uniones**: Al unir tablas grandes, filtre los datos antes de la unión cuando sea posible.

## Diferencias de sintaxis de consulta con NRQL y ejemplos [#syntax-differences]

<DNT>Lens</DNT> usa ANSI SQL en lugar de NRQL. Los siguientes ejemplos muestran diferencias clave y patrones de consulta comunes.

<CollapserGroup>
  <Collapser id="groupby" title="Usa `GROUP BY` en lugar de `FACET`">
    Al agrupar resultados, utilice `GROUP BY` e incluya la columna agrupada en su sentencia `SELECT`.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    FACET enduser.id
    ```

    **SQL:**

    ```sql
    SELECT "enduser.id", count(*)
    FROM telemetry.PageViewTiming
    GROUP BY "enduser.id"
    ```
  </Collapser>

  <Collapser id="where-timestamp" title="Usa `WHERE` con `timestamp` en lugar de `SINCE`">
    Filtre los rangos de tiempo usando `WHERE` con la columna `timestamp` e intervalos de fechas.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    SINCE 1 day ago
    ```

    **SQL:**

    ```sql
    SELECT count(*)
    FROM telemetry.PageViewTiming
    WHERE timestamp > current_date - INTERVAL '1' DAY
    ```
  </Collapser>

  <Collapser id="timeseries" title="Series de tiempo con formato epoch">
    Para crear visualizaciones de series de tiempo, estructure su consulta de modo que la primera columna contenga fechas en formato epoch. Utilice `to_unixtime()` para convertir fechas.

    ```sql
    WITH orders AS (
      SELECT
        to_unixtime(date(date_parse(order_date, '%Y-%m-%d'))) as dt,
        count(*) as orders
      FROM gsheets.default.orders
      GROUP BY 1
    )
    SELECT *
    FROM orders
    WHERE dt >= to_unixtime(date('2025-10-19'))
      AND dt <= to_unixtime(date('2025-10-21'))
    ORDER BY dt ASC
    ```
  </Collapser>

  <Collapser id="cross-db-join" title="`JOIN` entre bases de datos">
    Utilice la sintaxis estándar de SQL `JOIN` para combinar datos de múltiples fuentes en una sola consulta.

    ```sql
    WITH sf AS (
      SELECT DISTINCT user_id, assigned_success_manager
      FROM demogorgon.default.relibank_opportunity_matrix
    )
    SELECT
      sf.assigned_success_manager,
      MAX(nrdb.largestContentfulPaint)
    FROM telemetry.PageViewTiming nrdb
    LEFT JOIN sf ON nrdb."enduser.id" = sf.user_id
    WHERE timingName = 'largestContentfulPaint'
      AND nrdb.timestamp > current_date - INTERVAL '1' DAY
    GROUP BY sf.assigned_success_manager
    ```
  </Collapser>

  <Collapser id="query-mv" title="Consultar vistas materializadas">
    Haga referencia a <DNT>materialized views</DNT> usando el prefijo `mv.`:

    ```sql
    SELECT *
    FROM mv.weekly_performance_summary
    WHERE week_number = 42
    ```

    Puede unir <DNT>materialized views</DNT> con otras fuentes de datos:

    ```sql
    SELECT
      mv.store_id,
      mv.total_sales,
      s.store_name,
      s.region
    FROM mv.store_sales_mv mv
    JOIN snowflake.retail.stores s ON mv.store_id = s.store_id
    WHERE mv.total_sales > 10000
    ```
  </Collapser>
</CollapserGroup>
