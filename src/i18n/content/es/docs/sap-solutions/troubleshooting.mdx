## Los proveedores de datos de disponibilidad del servidor no funcionan

**Síntoma**: los proveedores de datos `NR_SAP_INSTANCE_AVAILABILITY` y `NR_SAP_SERVER_LIST` no logran recopilar datos.

**Causa**: el perfil `DEFAULT.PFL` en los servidores monitoreados no está configurado para los servicios web <DNT>**SAPCONTROL**</DNT>.

**Solución**:

1. Abra el archivo `DEFAULT.PFL` y verifique si esta línea existe:

`service/protectedwebmethods = SDEFAULT`

2. Si falta la línea, agregue el siguiente parámetro:

`service/protectedwebmethods = SDEFAULT -GetQueueStatistic -ABAPGetWPTable -EnqGetStatistic -GetProcessList -GetEnvironment -GetVersionInfo -GetSystemInstanceList`

3. Reinicie el <DNT>**SAP service**</DNT>:

```yaml
stopsap sapstartsrv
startsap sapstartsrv
```

4. Verifique la configuración:

```yaml
sapcontrol -prot GSOAP_HTTP -nr 00 -function GetSystemInstanceList
```

**Resultado esperado**: el comando devuelve la lista de instancias del sistema sin errores.
