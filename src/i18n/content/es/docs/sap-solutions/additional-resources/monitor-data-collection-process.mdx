## Estado de ejecución y log de transmisión

Desde el **Data Provider Registry** (transacción `/NEWR/REGISTRY`), puede consultar la<img src="/images/runtimestatus.webp" class="inline" style={{height: "40px", width: "120px"}} alt="RuntimeStatus icon" title="RuntimeStatus" />

y<img src="/images/transmissionlog.webp" class="inline" style={{height: "40px", width: "120px"}} alt="transmissionlog" title="Transmissionlog.webp" />para cada proveedor de datos. El log de transmisión muestra el estado y la hora de cada transmisión.

## Comprobar el estado del trabajo y el spool

1. Emplee la transacción `SM37` para comprobar el estado de los trabajos programados. Ingrese `/NEWR/*` en el campo **ABAP Program Name** y haga clic en **Execute**.
2. Preste atención a la duración de la ejecución; un tiempo anormalmente largo indica un problema potencial.
3. Verifique si hay trabajos superpuestos y ajuste el cronograma según sea necesario. El spool de trabajo proporciona el resultado para cada proveedor de datos.

## Consultar log de aplicación

Los procesos de recogida de datos quedan registrados en el log de aplicación (transacción SLG1). consulta con el log de aplicación Objeto `/NEWR/DP`.

## Dashboard de monitoreo de recopilación de datos de SAP

Los datos métricos de recolección de datos también son enviados a New Relic para auto-monitoreo. Emplee el dashboard [**SAP Data Collection Monitoring**]() listo para usar para monitorear los procesos de recopilación de datos para cada proveedor de datos.
