Configure SAP BTP Integration Suite (integración en la nube) como middleware para transmitir datos de monitoreo a New Relic cuando la conectividad directa no esté disponible.

## Descripción general

El programa collection manager proporciona una opción para enviar datos a New Relic mediante SAP Cloud Integration (SAP CPI) como middleware en lugar de transmisión directa.

| **Método**            | **Descripción**                                             |
| --------------------- | ----------------------------------------------------------- |
| Directo               | Transmisión directa desde SAP a New Relic (método estándar) |
| SAP Cloud Integration | Uso de SAP CPI como middleware                              |

## Pasos de configuración

<Steps>
  <Step>
    ## Determine el método de autorización

    Seleccione el tipo de autorización que utilizará para el iFlow:

    * <DNT>**ClientId/Secret**</DNT> (OAuth2)
    * <DNT>**Certificate**</DNT>
    * <DNT>**External Certificate**</DNT>
  </Step>

  <Step>
    ## Crear instancia de servicio y clave

    1. Cree una instancia de servicio para <DNT>**SAP Process Integration Runtime**</DNT>.
    2. Asigne un <DNT>**service key**</DNT> o <DNT>**certificate**</DNT> utilizando el método de autorización seleccionado.
    3. Descargue el <DNT>**service key**</DNT> para su uso posterior.

    Para obtener instrucciones detalladas, consulte [Obtener clave de servicio para <DNT>**BTP Monitoring**</DNT>](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).
  </Step>

  <Step>
    ## Configure y despliegue iFlow

    Use el <DNT>**iFlow**</DNT> proporcionado por New Relic como plantilla:

    1. Copie la plantilla <DNT>**iFlow**</DNT>.
    2. Actualice el método de autorización para que coincida con su selección del Paso 1.
    3. Asegúrese de que los siguientes mapeos estén configurados:

    ### Encabezados permitidos

    Configure <DNT>**allowed headers**</DNT> para que contenga:

    ```
    API-Key|targetURL
    ```

    ### Método de autorización

    * Para <DNT>**ClientId/Secret**</DNT> o <DNT>**Certificate**</DNT>: seleccione <DNT>**User Role**</DNT>
    * Para <DNT>**External Certificate**</DNT>: seleccione <DNT>**Client Certificate**</DNT> y agregue <DNT>**root certificates**</DNT>

    ### Modificador de contenido de la solicitud

    Configure <DNT>**exchange properties**</DNT> para que contenga:

    * <DNT>**targeturl**</DNT>
    * <DNT>**apikey**</DNT>

    ### Script de Groovy

    Agregue un <DNT>**script**</DNT> para establecer encabezados dinámicamente:

    ```groovy
    import com.sap.gateway.ip.core.customdev.util.Message;
    def Message processData(Message message) {

      message.setHeader("API-Key", message.getProperty("apikey"));
      message.setHeader("Content-Type", "application/json");
      message.setHeader("Accept", "application/json");

      message.setHeader("CamelHttpUri", message.getProperty("targeturl"));

      def turl = message.getHeader("CamelHttpUri", String);

      def logText = "---Headers---\n";

      message.getHeaders().each { k, v -> logText += "\${k} = \${v}\n" };
      messageLogFactory.getMessageLog(message)?.addAttachmentAsString("AllHeaders", logText, "text/plain");

      messageLogFactory.getMessageLog(message)?.addAttachmentAsString("URL", turl, "text/plain");

      return message;
    }
    ```

    ### Modificador de contenido de la respuesta

    Configure el <DNT>**message body**</DNT> para devolver:

    ```json
    {
      "statusCode": "\${header.CamelHttpResponseCode}",
      "response": \${body}
    }
    ```

    ### Configuración de conexión HTTP

    * <DNT>**Address**</DNT>: `\${header.CamelHttpUri}`
    * <DNT>**Method**</DNT>: <DNT>**POST**</DNT>
    * <DNT>**Timeout**</DNT>: <DNT>**60000 ms**</DNT>
    * <DNT>**Request Headers**</DNT>: `*`
    * <DNT>**Response Headers**</DNT>: `*`

    4. Despliegue el <DNT>**iFlow**</DNT>.
    5. Tenga en cuenta <DNT>**iFlow endpoint URL**</DNT>.
  </Step>

  <Step>
    ## Crear destino RFC

    1. Emplee la transacción <DNT>**SM59**</DNT> para crear un nuevo <DNT>**RFC destination**</DNT>.

    2. Ingrese la siguiente informacion:

       * <DNT>**RFC Destination**</DNT>: un nombre descriptivo (por ejemplo, <DNT>**NR\_CPI\_ENDPOINT**</DNT>)
       * <DNT>**Connection Type**</DNT>: <DNT>**G**</DNT> (Conexión HTTP a servidor externo)
       * <DNT>**Description**</DNT>: conexión a New Relic a través de la integración en la nube

    3. En la pestaña <DNT>**Technical Settings**</DNT>:

       * <DNT>**Target Host**</DNT>: El host de su <DNT>**iFlow endpoint**</DNT>
       * <DNT>**Service No.**</DNT>: <DNT>**443**</DNT>
       * <DNT>**Path Prefix**</DNT>: la ruta desde su <DNT>**iFlow endpoint**</DNT>

    4. En la pestaña <DNT>**Logon &amp; Security**</DNT>:

       * Configure la autenticación según el método seleccionado:

         * Para <DNT>**ClientId/Secret**</DNT>: ingrese <DNT>**ClientId**</DNT> como nombre de usuario y <DNT>**ClientSecret**</DNT> como contraseña
         * Para <DNT>**Certificate**</DNT>: configure el <DNT>**certificate details**</DNT>

    5. En la pestaña <DNT>**Security Options**</DNT>:
       * Seleccione <DNT>**Active**</DNT> para <DNT>**SSL**</DNT>

    6. Guarde la configuración.

    7. Haga clic en <DNT>**Connection Test**</DNT> para verificar la conexión.
  </Step>

  <Step>
    ## Configurar el administrador de recopilación

    1. Ejecute la transacción <DNT>**/NEWR/ONE\_COLLECTION**</DNT> o cree un <DNT>**variant**</DNT> para <DNT>**/NEWR/ABAP\_COLLECTION\_MANAGERN**</DNT>.
    2. En la sección <DNT>**Data Transmission to NR**</DNT> :
       * Seleccione <DNT>**SAP Cloud Integration**</DNT> en lugar de <DNT>**Direct**</DNT>
    3. En el campo <DNT>**iFlow Endpoint**</DNT>:
       * Ingrese el <DNT>**RFC destination name**</DNT> creado en el Paso 4 (por ejemplo, <DNT>**NR\_CPI\_ENDPOINT**</DNT>)
    4. Guarde el <DNT>**variant**</DNT>.
    5. Programar o ejecutar el <DNT>**collection job**</DNT>.
  </Step>
</Steps>

## Verificar la transmisión de datos

1. Ejecute el <DNT>**collection manager job**</DNT>.

2. Verifique el <DNT>**iFlow monitoring**</DNT> en <DNT>**SAP Cloud Integration**</DNT>:

   * Iniciar sesión en <DNT>**SAP BTP Integration Suite**</DNT>
   * Vaya a <DNT>**Monitor**</DNT> &gt; <DNT>**Integrations**</DNT>
   * Verificar el procesamiento exitoso de mensajes

3. Compruebe la llegada de datos en <DNT>**New Relic**</DNT>:

   * Iniciar sesión en <DNT>**New Relic**</DNT>
   * Navegue a su <DNT>**SAP dashboards**</DNT>
   * Verifica que se estén recibiendo datos de monitoreo

<Callout variant="important">
  Si modifica el <DNT>**iFlow template**</DNT> estándar, asegúrese de que se conserven todos los mapeos y encabezados requeridos para mantener la compatibilidad con el <DNT>**collection manager program**</DNT>.
</Callout>
