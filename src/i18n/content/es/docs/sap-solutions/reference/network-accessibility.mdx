## Quién debería usar esta referencia

* Administradores de SAP Basis que configuran conexiones RFC
* Administradores de red configurando reglas de firewall
* Equipos de instalación preparando los requisitos de red

## Configuración de destino RFC

Configure estos destinos RFC (Remote Function Call) en su sistema SAP para habilitar la comunicación con las API de New Relic. Use los extremos para la región donde está alojada su cuenta de New Relic.

**Nota**: el tipo de conexión &quot;G&quot; indica una conexión HTTP a un servidor externo.

Selecciona la configuración del extremo según la región de tu cuenta de New Relic:

* Use los **extremos de la región de EE. UU.** si su cuenta está en el centro de datos de EE. UU.
* Use **extremos de la región de la UE** si su cuenta está en el centro de datos de la UE

## Verificar conectividad

Después de configurar los destinos RFC, pruebe la conectividad usando el código de transacción de SAP `SM59`:

1. Seleccione el destino RFC
2. Hacer clic <DNT>**Connection Test**</DNT>
3. Verificar la conexión exitosa a los extremos de New Relic
