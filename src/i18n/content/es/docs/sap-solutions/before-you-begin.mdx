Complete estos requisitos previos antes de la instalación:

## Lista de verificación de requisitos previos

### Cuenta de New Relic

* <DNT>**Account ID**</DNT>: [obtener su ID de cuenta](/docs/accounts/accounts-billing/account-setup/account-id)
* <DNT>**License Key**</DNT>: Obtenga una <InlinePopover type="licenseKey" />de New Relic para la autenticación y el acceso a los datos
* <DNT>**Data Center Region**</DNT>: [elige la ubicación de tu centro de datos (EE. UU. o UE)](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) y configura tu cuenta

### Requisitos del sistema SAP

* <DNT>**Software versions**</DNT>: Verifique que sus <DNT>**SAP systems**</DNT> cumplan con los niveles de lanzamiento de componentes de software requeridos. Consulte [Referencia de proveedores de datos](/docs/sap-solutions/monitor/data-providers-reference) para conocer los requisitos específicos del sistema
* <DNT>**Monitored systems**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**Kernel 753 Patch 602+**</DNT>, <DNT>**ST-PI 740 SP16+**</DNT>
* <DNT>**Central monitoring system**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**ST-PI 740 SP14+**</DNT>

### Acceso a la red

* <DNT>**HTTPS access**</DNT> a <DNT>**New Relic endpoints**</DNT> (ver las [URLs de los extremos](#network-endpoints))
* <DNT>**Firewall rules**</DNT> configurado
* <DNT>**Proxy settings**</DNT> configurado (si se requiere)

### Monitoreo de BTP (si corresponde)

Si monitorea <DNT>**BTP accounts**</DNT>, asegúrese de uno de los siguientes:

* <DNT>**CloudALM account**</DNT> configurado para <DNT>**BTP monitoring**</DNT>
* <DNT>**Process Integration Runtime**</DNT> suscripción de servicio (requerida para el monitoreo de integración)

<Callout variant="important">
  Es posible que necesite aprobación o asistencia de configuración de sus equipos de seguridad o de red. Complete estos pasos antes de la instalación y revise la [lista de verificación de preparación para la instalación](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness).
</Callout>

## Preparar los sistemas monitoreados

La arquitectura sin agente utiliza conexiones RFC para recuperar datos.

### Requisitos de versión

Verificar las versiones de software requeridas:

* <DNT>**SAP\_BASIS**</DNT>: Versión <DNT>**740**</DNT> o superior
* <DNT>**Kernel**</DNT>: Versión <DNT>**753 Patch 602**</DNT> o superior
* <DNT>**ST-PI**</DNT>: Versión <DNT>**740 SP16**</DNT> o superior

### Configuración de usuario

* [Cargue el rol de autorización](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* [Cree un usuario del sistema](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) con el rol cargado para <DNT>**RFC connections**</DNT>

### Configuración de la base de datos HANA (si corresponde)

Si realiza el monitoreo de un <DNT>**HANA database**</DNT>, cree un <DNT>**HANA database user**</DNT> `NRAGENT` con acceso de <DNT>**Catalog Read**</DNT> y <DNT>**Monitoring**</DNT>:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

Recopile el <DNT>**HANA server host**</DNT> y el <DNT>**tenant DB port**</DNT> para <DNT>**SQL connection**</DNT>.

### Otras bases de datos

Para otras bases de datos, active el <DNT>**CCMS database KPIs**</DNT> estándar ejecutando el <DNT>**ABAP program**</DNT> `RSDBKPIMON`.

Si su versión no está en la lista, comuníquese con su ejecutivo de cuenta de New Relic para obtener un paquete complementario.

### Requisito del sistema

* Al menos <DNT>**20 dialog work processes**</DNT> y <DNT>**10 background work processes**</DNT>
* <DNT>**HTTPS access**</DNT> saliente al <DNT>**New Relic data center**</DNT> (ver las URLs de los extremos arriba)

### Configuración de red

* <DNT>**Proxy settings**</DNT> (si es requerido):

  * Nombre de dominio/dirección IP y puerto del servidor
  * Credenciales de acceso al proxy (si la autenticación es necesaria)

* <DNT>**Firewall policies**</DNT>: Permitir a <DNT>**HTTPS traffic**</DNT> <DNT>**New Relic endpoints**</DNT>

### Destinos RFC

Para cada sistema monitoreado, use la transacción `SM59` y el ID del sistema remoto para crear <DNT>**RFC destinations**</DNT> y probar las conexiones. Consulte [Crear destinos RFC para sistemas monitoreados](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems).

<Callout variant="important">
  El <DNT>**RFC user**</DNT> debe tener asignado el rol <DNT>**/NEWR/DATA\_PROVIDER**</DNT>.
</Callout>

### Monitoreo de base de datos HANA remota (si corresponde)

Para monitorear <DNT>**HANA databases**</DNT> remoto:

* Asegúrese de que el sistema de monitoreo central tenga acceso al <DNT>**HANA server**</DNT> y <DNT>**tenant DB port**</DNT>

* Para sistemas de monitoreo central que no son S4HANA, instale uno de estos componentes:

  * <DNT>**HANA Client v2.0**</DNT> (compatible con el <DNT>**HANA DB version**</DNT>)

  * <DNT>**HANA Client Lib file**</DNT>:

    * `libSQLDBCHDB.so` (Linux)
    * `libSQLDBCHDB.dll` (Windows)
    * Ubique la biblioteca en el <DNT>**SAP Kernel folder**</DNT> y asigne la propiedad al <DNT>**SID Admin ID**</DNT> (p. ej., `sd1adm`)

* Para cada <DNT>**HANA database**</DNT> remoto, use la transacción `SAP DBACOCKPIT` para crear un <DNT>**DB connection**</DNT>. Consulte [Crear conexión de base de datos](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)

## Preparar el monitoreo de BTP (si corresponde)

Para monitorear un <DNT>**BTP account**</DNT>, necesitas un <DNT>**CloudALM instance**</DNT> configurado para monitorear el <DNT>**BTP account**</DNT>, o una suscripción al servicio <DNT>**Process Integration Runtime**</DNT> (aplicable solo al monitoreo de integración). Se requieren <DNT>**Service keys**</DNT> para la autenticación de la conexión. Consulte [Cómo recuperar claves de servicio](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).

Use <DNT>**service keys**</DNT> para crear [destinos HTTP externos](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) y como el <DNT>**API host**</DNT> para <DNT>**API data providers**</DNT>.
