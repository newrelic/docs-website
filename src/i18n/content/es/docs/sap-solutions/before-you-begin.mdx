Complete estos requisitos previos antes de la instalación:

## Lista de verificación de requisitos previos

### Cuenta de New Relic

* **ID de cuenta**: [obtenga su ID de cuenta](/docs/accounts/accounts-billing/account-setup/account-id)
* **Clave de licencia**: obtenga una <InlinePopover type="licenseKey" />de New Relic para la autenticación y el acceso a los datos
* **Región del centro de datos**: [elige la ubicación de tu centro de datos (EE. UU. o UE)](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) y configura tu cuenta

### Requisitos del sistema SAP

* **Versiones de software**: verifique que sus sistemas SAP cumplan con los niveles de lanzamiento de componentes de software requeridos. Consulte [Referencia de proveedores de datos](/docs/sap-solutions/monitor/data-providers-reference) para conocer los requisitos específicos del sistema
* **Sistemas monitoreados**: SAP\_BASIS 740+, Kernel 753 Patch 602+, ST-PI 740 SP16+
* **Sistema de monitoreo central**: SAP\_BASIS 740+, ST-PI 740 SP14+

### Acceso a la red

* Acceso HTTPS a los extremos de New Relic (consulte las [URL de los extremos](#network-endpoints))
* Reglas de firewall configuradas
* Configuración de proxy establecida (si se requiere)

### Monitoreo de BTP (si corresponde)

Si monitorea cuentas de BTP, asegúrese de una de las siguientes opciones:

* Cuenta de CloudALM configurada para el monitoreo de BTP
* Suscripción al servicio Process Integration Runtime (requerida para el monitoreo de integración)

<Callout variant="important">
  Es posible que necesite aprobación o asistencia de configuración de sus equipos de seguridad o de red. Complete estos pasos antes de la instalación y revise la [lista de verificación de preparación para la instalación](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness).
</Callout>

## Preparar los sistemas monitoreados

La arquitectura sin agente utiliza conexiones RFC para recuperar datos.

### Requisitos de versión

Verificar las versiones de software requeridas:

* **SAP\_BASIS**: versión 740 o superior
* **Kernel**: versión 753 parche 602 o superior
* **ST-PI**: release 740 SP16 o superior

### Configuración de usuario

* [Cargue el rol de autorización](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* [Cree un usuario del sistema](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) con el rol cargado para las conexiones RFC

### Configuración de la base de datos HANA (si corresponde)

Si monitorea una base de datos HANA, cree un usuario `NRAGENT` de base de datos HANA con acceso de lectura de catálogo y monitoreo:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

Recopile el host del servidor HANA y el puerto de la base de datos del inquilino para la conexión SQL.

### Otras bases de datos

Para otras bases de datos, active los KPI estándar de base de datos de CCMS ejecutando el programa ABAP `RSDBKPIMON`.

Si su versión no está en la lista, comuníquese con su ejecutivo de cuenta de New Relic para obtener un paquete complementario.

### Requisito del sistema

* Al menos 20 procesos de trabajo de diálogo y 10 procesos de trabajo en segundo plano
* Acceso HTTPS saliente al centro de datos de New Relic (consulte las URL de los extremos arriba)

### Configuración de red

* **Configuración de proxy** (si se requiere):

  * Nombre de dominio/dirección IP y puerto del servidor
  * Credenciales de acceso al proxy (si la autenticación es necesaria)

* **Políticas de firewall**: permitir el tráfico HTTPS a los extremos de New Relic

### Destinos RFC

Para cada sistema monitoreado, use la transacción `SM59` y el ID del sistema remoto para crear destinos RFC y probar las conexiones. Consulte [Crear destinos RFC para sistemas monitoreados](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems).

<Callout variant="important">
  El usuario RFC debe tener asignado el rol **/NEWR/DATA\_PROVIDER**.
</Callout>

### Monitoreo de base de datos HANA remota (si corresponde)

Para monitorear bases de datos HANA remotas:

* Asegúrese de que el sistema de monitoreo central tenga acceso al servidor HANA y al puerto de la base de datos del inquilino.

* Para sistemas de monitoreo central que no son S4HANA, instale uno de estos componentes:

  * HANA Client v2.0 (compatible con la versión de la base de datos HANA)

  * Archivo de biblioteca de cliente de HANA:

    * `libSQLDBCHDB.so` (Linux)
    * `libSQLDBCHDB.dll` (Windows)
    * Localice la biblioteca en la carpeta SAP Kernel y asigne la propiedad al ID de administrador del SID (p. ej., `sd1adm`)

* Para cada base de datos HANA remota, use la transacción `SAP DBACOCKPIT` para crear una conexión de base de datos. Consulte [Crear conexión de base de datos](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)

## Preparar el monitoreo de BTP (si corresponde)

Para monitorear una cuenta de BTP, necesita una instancia de CloudALM configurada para monitorear la cuenta de BTP o una suscripción al servicio Process Integration Runtime (aplicable solo al monitoreo de integración). Las claves de servicio son necesarias para la autenticación de la conexión. Consulte [Cómo recuperar claves de servicio](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).

Utilice claves de servicio para crear [destinos HTTP externos](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) y como host de API para los proveedores de datos de API.
