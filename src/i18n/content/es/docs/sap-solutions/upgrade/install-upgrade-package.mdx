Para actualizar desde una versión anterior de New Relic Monitoring for SAP Solutions, use el paquete de actualización que coincida con su versión instalada actual.

## Requisitos previos

* <DNT>**Administrator access**</DNT> a <DNT>**SAP system**</DNT>
* Versión actual identificada, consulte la [sección Identificar versión actual](#identify-current-version)
* Respaldo de la configuración existente
* Correct <DNT>**upgrade package**</DNT> for your installed version

## Identificar la versión actual [#identify-current-version]

1. Navegue hasta <DNT>**System &gt; Status**</DNT>.
2. Search for <DNT>**NEWR**</DNT> to identify the <DNT>**product version**</DNT>.
3. In <DNT>**client 000**</DNT>, execute transaction `SAINT` to update the <DNT>**ABAP add-on**</DNT>.

## Respaldar y actualizar la configuración [#backup-and-update-configuration]

<Callout variant="tip">
  Save any modified <DNT>**data provider configurations**</DNT> or <DNT>**custom data providers**</DNT> to a <DNT>**configuration template**</DNT> to prevent overwriting during the upgrade. Ensure that the templates are saved before copying the new template from <DNT>**client 000**</DNT>, even if the <DNT>**V3.0 ABAP Add-on**</DNT> is not installed.
</Callout>

1. Access the menu `/n/NEWR/INSTALL` and navigate to <DNT>**Update or Save Configuration Template**</DNT>.

2. Execute `/NEWR/REP_DELETE_INS_TEMPLATE` to remove the <DNT>**HDB monitoring template**</DNT> related to <DNT>**Java monitoring**</DNT>.

3. Set up the <DNT>**ABAP data connection**</DNT> to the <DNT>**HANA database**</DNT> using `DBACockpit`.

4. Copy the <DNT>**configuration template**</DNT> from <DNT>**client 000**</DNT>.

5. Modify the <DNT>**monitoring system configuration**</DNT>:

   * Add the new <DNT>**DB connection**</DNT>.
   * Remove references to the <DNT>**Java Enterprise Connector**</DNT> (e.g., <DNT>**NR\_EI\_AGENT**</DNT>).
   * Add <DNT>**HANA\_DB**</DNT> to the <DNT>**monitoring groups**</DNT>.
   * Re-generate <DNT>**data provider configurations**</DNT> to update settings.
