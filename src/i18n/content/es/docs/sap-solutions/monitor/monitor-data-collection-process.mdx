Esta guía le muestra cómo monitorear el proceso de recopilación de datos para New Relic Monitoring for SAP Solutions. Puede usar múltiples métodos para verificar que la recopilación de datos esté funcionando correctamente.

## Requisitos previos

* Access to <DNT>**SAP system**</DNT> with appropriate transaction permissions
* <DNT>**New Relic Monitoring for SAP Solutions**</DNT> installed and configured

## Estado de ejecución y log de transmisión

From the <DNT>**Data Provider Registry**</DNT> (transaction `/NEWR/REGISTRY`), you can check the <DNT>**Runtime Status**</DNT> and <DNT>**Transmission Log**</DNT> for each data provider. The transmission log shows the status and time of each transmission.

## Comprobar el estado del trabajo y el spool

1. Use transaction `SM37` to check the status of scheduled jobs. Enter `/NEWR/*` in the <DNT>**ABAP Program Name**</DNT> field and click <DNT>**Execute**</DNT>.
2. Revise la duración de la ejecución. Un tiempo de ejecución anormalmente largo indica un posible problema.
3. Verifique si hay trabajos superpuestos y ajuste la programación según sea necesario. Acceda al spool del trabajo desde los detalles del trabajo para ver el resultado de cada proveedor de datos.

## Consultar log de aplicación

Data collection processes are logged in the application log (transaction `SLG1`). Query with the <DNT>**Application Log Object**</DNT> `/NEWR/DP`.

## Dashboard de monitoreo de recopilación de datos de SAP

Data collection metrics are also sent to New Relic for self-monitoring. Use the out-of-the-box <DNT>**SAP Data Collection Monitoring**</DNT> dashboard to monitor the data collection processes for each data provider.
