Esta guía le muestra cómo monitorear el proceso de recopilación de datos para New Relic Monitoring for SAP Solutions. Puede usar múltiples métodos para verificar que la recopilación de datos esté funcionando correctamente.

## Requisitos previos

* Acceso a <DNT>**SAP system**</DNT> con los permisos de transacción apropiados
* <DNT>**New Relic Monitoring for SAP Solutions**</DNT> instalado y configurado

## Estado de ejecución y log de transmisión

Desde el <DNT>**Data Provider Registry**</DNT> (transacción `/NEWR/REGISTRY`), puede consultar el <DNT>**Runtime Status**</DNT> y el <DNT>**Transmission Log**</DNT> para cada proveedor de datos. El log de transmisión muestra el estado y la hora de cada transmisión.

## Comprobar el estado del trabajo y el spool

1. Utilice la transacción `SM37` para comprobar el estado de los trabajos programados. Ingrese `/NEWR/*` en el campo <DNT>**ABAP Program Name**</DNT> y haga clic en <DNT>**Execute**</DNT>.
2. Revise la duración de la ejecución. Un tiempo de ejecución anormalmente largo indica un posible problema.
3. Verifique si hay trabajos superpuestos y ajuste la programación según sea necesario. Acceda al spool del trabajo desde los detalles del trabajo para ver el resultado de cada proveedor de datos.

## Consultar log de aplicación

Los procesos de recopilación de datos se registran en el log de aplicación (transacción `SLG1`). Consulta con el <DNT>**Application Log Object**</DNT> `/NEWR/DP`.

## Dashboard de monitoreo de recopilación de datos de SAP

Las métricas de recolección de datos también se envían a New Relic para automonitoreo. Emplee el dashboard <DNT>**SAP Data Collection Monitoring**</DNT> listo para usar para monitorear los procesos de recopilación de datos para cada proveedor de datos.
