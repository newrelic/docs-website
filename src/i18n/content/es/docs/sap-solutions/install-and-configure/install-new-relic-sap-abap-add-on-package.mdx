El componente ABAP de <DNT>**NRM4SAP**</DNT> está empaquetado como un complemento de SAP y se puede instalar mediante la transacción `SAINT` en <DNT>**client 000**</DNT>.

## Archivo de instalación del complemento

* <DNT>**Initial installation**</DNT>: Download the add-on file `K-xxxAOIINNEWR.SAR` to a folder on your local PC
* <DNT>**Upgrade from previous version**</DNT>: Use the file named `K-xxxAOUINNEWR.SAP`

## Instalar el archivo complementario de SAP

1. Iniciar sesión en <DNT>**client 000**</DNT>
2. Escriba `SAINT` en el campo de comando <DNT>**SAP GUI**</DNT>
3. Vaya a <DNT>**Installation Package**</DNT> &gt; <DNT>**Load Packages**</DNT>
4. Elija el método adecuado según dónde se encuentre su archivo de instalación:

<DNT>**Which method should you use?**</DNT>

* <DNT>**SAR archive from frontend**</DNT> (Most common): Installation file is on your local computer
* <DNT>**SAR archive from application server**</DNT>: SAR file has been pre-uploaded to `/usr/sap/trans/download/`
* <DNT>**EPS files from application server**</DNT>: You need to extract the SAR file into PAT files on the server first

    <TabsPageItem id="sar-archive-from-application-server">
      1. Ensure there is a <DNT>**download directory**</DNT> under the <DNT>**trans directory**</DNT> on the application server (`/usr/sap/trans/download`)
      2. Copy the <DNT>**add-on file**</DNT> to this directory
    </TabsPageItem>

    <TabsPageItem id="eps-files-from-application-server">
      1. Extract the <DNT>**SAR file**</DNT> into <DNT>**PAT files**</DNT>. Execute the following commands:

      ```bash
      cd /usr/sap/trans
      cp /tmp/K-xxxAOIINNEWR.SAR /usr/sap/trans
      SAPCAR -xvf K-xxxAOIINNEWR.SAR
      ```

      2. En la ventana emergente <DNT>**Do you want to upload OCS Packages from the EPS inbox?**</DNT>, seleccione <DNT>**Yes**</DNT>
      3. Ensure the <DNT>**OCS file**</DNT> is uploaded without errors
    </TabsPageItem>
  </TabsPages>
</Tabs>

## Complete la instalación

1. Haga clic en <DNT>**Start**</DNT>, seleccione <DNT>**Installable Add-on packages for your system**</DNT> y haga clic <DNT>**Continue**</DNT>

2. Cuando se le pregunte <DNT>**Do you want to add Modification Adjustment Transports to the queue?**</DNT>:

   * Haga clic en <DNT>**No**</DNT> para la instalación estándar (por defecto)
   * Haga clic en <DNT>**Yes**</DNT> solo si su paquete incluye transportes de ajuste con instrucciones específicas

   <img title="Do you want to add Modification Adjustment Transports to the queue?" alt="A screenshot displaying question about Modification Adjustment Transports" src="/images/modify-adjustement-transports.webp" width="80%" />

3. Haga clic en el botón de marca de verificación verde para continuar.

   <Callout variant="important">
     Si aparece la página <DNT>**Errors during Disassembling OCS Packages**</DNT>, haga clic en <DNT>**Ignore**</DNT>. Este error es esperado para ciertos tipos de paquetes y no afectará la instalación.
   </Callout>

4. Una vez instalado el paquete, haga clic <DNT>**Finish**</DNT>

5. Aparece la ventana emergente <DNT>**Sending the runtime analysis**</DNT>. Seleccione uno de los siguientes:

   * <DNT>**Send**</DNT>: Opens a feedback form with the <DNT>**runtime analysis data**</DNT> in a local browser window
   * <DNT>**Display**</DNT>: Displays the <DNT>**runtime analysis**</DNT> first, then opens the feedback form
   * <DNT>**Do not send**</DNT>: Closes the dialog without sending <DNT>**runtime analysis data**</DNT>

6. Verify the installation in the <DNT>**Installed Components**</DNT> tab. The <DNT>**New Relic add-on**</DNT> should be listed with a status indicating successful installation

7. Cerrar sesión en <DNT>**Client 000**</DNT>
