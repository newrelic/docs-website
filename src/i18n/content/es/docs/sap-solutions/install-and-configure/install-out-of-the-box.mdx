El monitoreo New Relic para SAP incluye una lista de componentes listos para usar. Siga las instrucciones a continuación para localizarlos e instalarlos. Para realizar estas tareas, debe tener privilegios de administrador de New Relic en la cuenta.

## Instalar el tablero

1. Vaya a **[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**.
2. Para ver todos los paneles de SAP, en la sección **Recommended Dashboards**, haga clic en **view all**. Consulte [Dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) para obtener una introducción a nuestra característica personalizada del panel.

<Callout variant="important">
  También puede navegar a **Integrations &amp; Agents** y buscar **New Relic Monitoring for SAP Solutions** para descargar automáticamente todo el tablero.
</Callout>

3. Busque SAP e instale el siguiente panel:

   * **Dashboards SAP**
   * **Cabina SAP**
   * **Monitoreo de recopilación de datos de SAP**
   * **Estimación de ingesta de datos de SAP**

<Callout variant="important">
  Para el dashboard de **SAP Cockpit**, debe [crear una tabla de búsquedas](#create-lookup-table) denominada `NR_LKUP_SYSTEM`.
</Callout>

## Agregar filtro dashboard

1. Vaya a **[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**.

2. Habilitar la vinculación de facetas para el filtrado:

   * Haga clic en los puntos suspensivos (…) en la esquina superior derecha del widget.
   * Seleccione **Edit** para abrir la pantalla de edición.
   * Active y desactive el interruptor **Filter the current dashboard**.

3. Se puede habilitar la vinculación de facetas para tablas, gráficos circulares y widgets de gráficos de barras para filtrar por sistema SAP, instancia, estado IDOC, trabajos, etc.

## Crear tabla de búsquedas (solo SAP cockpit) [#create-lookup-table]

El dashboard de **SAP Cockpit** emplea una tabla de búsqueda para los sistemas SAP y sus entornos.

1. Crea la tabla de búsquedas con dos columnas: `SYS_ID` y `Env`, y almacénala como un archivo CSV. A continuación se muestra un ejemplo del archivo de búsqueda:

   * SYS\_ID Env
   * GD1 DEV
   * SD2 DEV
   * Prueba AQ1
   * Pruebas AQ2
   * SP1 Prod
   * SP2 Prod

2. Vaya a **Log &gt; Lookup Table** en el menú.

3. Haga clic en **Add a table**.

4. En la barra lateral [Agregar una tabla de búsquedas](/docs/logs/ui-data/lookup-tables-ui/), nombre la tabla como **NR\_LKUP\_SYSTEM** y cargue el archivo CSV.

5. Para verificar el contenido de la tabla, emplee el generador de consultas con `FROM lookup(NR_LKUP_SYSTEM) SELECT *` y luego haga clic en **Run**.
