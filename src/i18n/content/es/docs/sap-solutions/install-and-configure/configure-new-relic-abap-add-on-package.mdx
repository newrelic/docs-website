## Uso del menú /NEWR/INSTALL [#menu-newr-install]

1. Inicie sesión en el cliente de producción del sistema de monitoreo central.
2. Inicie la configuración del proveedor de datos accediendo al menú del área `/NEWR/INSTALL`.
3. Navegue hasta el **menú raíz de SAP Easy Access** antes de iniciar el menú de área escribiendo `/n/NEWR/INSTALL` en el campo de comando.

<Callout variant="tip">
  Si no está seguro de si se encuentra en el **menú raíz de SAP Easy Access**, escriba `/n` y presione **Enter**.
</Callout>

Luego de instalar exitosamente el complemento en el sistema de monitoreo central:

* Cerciorar de que el instalador tenga la autorización adecuada para realizar la configuración.
* Tenga en cuenta el nuevo código de transacción introducido por el paquete complementario, que figura en **Transaction Code for NRM4SAP Installation and Configuration**.
* Agregue el parámetro `/NEWR/*` a los objetos de autorización de código en el `profile/role` del instalador.

<Callout variant="tip">
  Si está actualizando desde una versión anterior, consulte [Actualizar desde una versión anterior](/docs/sap-solutions/upgrade-from-a-previous-version/install-upgrade-package) para determinar qué pasos son necesarios.
</Callout>

## Conectar sistemas y generar configuración. [#connect-systems]

<Callout variant="important">
  Complete los pasos de configuración antes de la instalación si aún no se completaron.
</Callout>

* **Configurar destinos HTTP y RFC**: emplee esta opción para invocar la transacción `SM59` y verificar sus conexiones [RFC](/docs/sap-solutions/additional-resources/detailed-configuration/#create-rfc-for-monitored-systems) y [HTTPS](/docs/sap-solutions/additional-resources/detailed-configuration/#set-http-destination-to-new-relic-cloud).

* **Configurar la conexión a la base de datos remota**:

  * Emplee la transacción SAP **DBACOCKPIT** para verificar la configuración de las conexiones a las bases de datos remotas de HANA.
  * Cerciorar de que se cree una conexión de base de datos para cada base de datos HANA remota que desee monitorear.
  * Para obtener instrucciones detalladas, consulte [Crear conexión de base de datos](/docs/sap-solutions/additional-resources/detailed-configuration/#create-database-connection).

  <img title="Configure New Relic" alt="A screenshot displaying the SAP install new release and patch" src="/images/configure-newrelic.webp" width="500" />

* **Configurar conexiones a NR extremos de API**:

  1. Haga clic en **New Entries**.
  2. Ingrese <DNT>**New Relic Account ID**</DNT> (opcionalmente con el prefijo SAP-), **API Ingest Key**, **Metric Endpoint**, **Event Endpoint**, **Log Endpoint** y **Trace Endpoint** para la cuenta New Relic.
  3. Clic en **Save**.

  <img title="New Entry" alt="A screenshot displaying the SAP HANA database new entry" src="/images/new-entry-1.webp" width="1000" />

* **Copiar tablas de plantilla del cliente 000**: ejecute esta opción para copiar la tabla del cliente 000 haciendo clic en el botón<img src="/images/new-relic-sap-clock.webp" class="inline" style={{height: "20px", width: "20px"}} alt="clock indicator in New Relic" title="Multiple apps name indicator in New Relic" />

  icono y confirme el éxito con el mensaje de éxito.

* **Definir los sistemas monitoreados**: emplee esta opción para definir los sistemas monitoreados.

  1. Haga clic en **New Entries** para ingresar el **SAP System ID** y el número de **Client**.
  2. En **Data Provider Installation Information**, especifique el **Destination** RFC, seleccione el **Software Component** SAP, ingrese el **Database Vendor** y proporcione el **DB Connection Name** (solo base de datos HANA).
  3. Haga clic en **Monitoring Group** y seleccione **Yes** en el cuadro de diálogo para confirmar.
  4. Seleccione los grupos de monitoreo necesarios y haga clic en el botón<img src="/images/generate.webp" class="inline" style={{height: "25px", width: "50px"}} alt="generate icon in New Relic" title="Multiple apps name indicator in New Relic" />para completar el proceso.
  5. Haga clic en **BACK** o en la tecla **F3** para ir al siguiente paso.

  <img title="New Entry" alt="A screenshot displaying the SAP HANA database new entry" src="/images/details-of-added-entry.webp" width="600" />

<Callout variant="tip">
  Mantenga la lista de **Monitoring Groups** navegando a **Expert Mode &gt; Installation Setups &gt; Monitoring Group Maintenance**.
</Callout>

* **Verificar destinos RFC**: emplee esta opción de menú para probar los destinos RFC para todos los sistemas monitoreados y NR extremos de API, excepto &apos;NINGUNO&apos;.

* **Generar configuración para nuevos proveedores de datos**:

  * Emplee esta opción de menú para generar proveedores de datos para una nueva versión o parche. Para una nueva instalación, se generan todos los proveedores de datos (DP).
  * En la primera pantalla, confirme los detalles del lanzamiento y del parche; los campos vacíos indican una configuración preexistente.
  * Para regenerar, seleccione **Re-generate Data Provider Configurations** en **Maintain Monitored Systems** o use **Reset Installation Flag (For Re-install only)** para restablecer el indicador de instalación y regenerar la configuración.

  <img title="Maintain Monitored Systems" alt="A screenshot displaying the SAP maintain monitored systems" src="/images/maintain-monitored-systems.webp" width="500" />

* **Verificar la finalización de la instalación**: haga clic en **Verify Installation Completeness** para cerciorar de que todo esté configurado correctamente. Si algún elemento muestra un indicador rojo, haga clic en el botón de ejecución en la columna **Action Needed**.

## Verificación de la configuración

1. Para garantizar que los proveedores de datos estén configurados correctamente, abra la carpeta **Data Provider Registry Maintenance**.
2. Haga clic en **Data Provider Registry** o emplee la transacción `/n/NEWR/REGISTRY`.
3. Seleccione un **Data Provider Name** y verifique la configuración de parámetros haciendo clic en **Configure**.
4. Pruebe el proveedor de datos haciendo clic en **TestDP**.
