## Programa de verificación de RFC [#rfc-check]

Programe `/NEWR/REGISTRY_RFC_CHECK` cada 1 a 3 minutos, asegurándose de que se ejecute con mayor frecuencia que el <DNT>**data collection job**</DNT>. Utilice el nombre del trabajo `NR_RFC_CHECK`.

## Programar el gestor de recopilación de datos ABAP [#abap-collection-manager]

Programe tres trabajos de recopilación de datos con variantes centradas en los siguientes grupos de proveedores de datos. Para obtener instrucciones sobre cómo programar trabajos del administrador de recopilación antes de `SP03`, consulte [Variantes y trabajos anteriores a SP03](/docs/sap-solutions/reference/variants-and-jobs-before-sp03). Separe los proveedores de datos en tres grupos para la programación. Para obtener una lista completa de todos los proveedores de datos, consulte la [Referencia de proveedores de datos](/docs/sap-solutions/monitor/data-providers-reference).

* <DNT>**TRACE group**</DNT> incluye los proveedores de datos relacionados con la base de datos de rendimiento (STAD):

  * [`NR.SAP.TRANSACTION`](/docs/sap-solutions/monitor/data-providers-reference/#sap-appl-metrics): Proveedor de datos de trazas
  * [`NR.SAP.TRANSACTION.COMPLETE`](/docs/sap-solutions/monitor/data-providers-reference/#sap-appl-metrics): recopilación completa de la base de datos de rendimiento con proceso RFC en segundo plano paralelo
  * `NR.SAP.SYSTEM.PROCESS` y `NR_SAP_PROGRAM`: suspendido después de SP02
  * `NR_SAP_WORKLOAD_COMPLETE` y `NR.SAP.WORKLOAD.COMPLETE`

* <DNT>**IDOC group**</DNT> incluye proveedores relacionados con IDOC (puede tomar un tiempo significativo en producción):

  * [`NR_SAP_IDOC`](/docs/sap-solutions/monitor/data-providers-reference/#idoc-all): proveedor de datos de eventos de estado principal de IDOC
  * [`NR_SAP_IDOC_COMPLETE`](/docs/sap-solutions/monitor/data-providers-reference/#idoc-all): recuperación completa de IDOC con recopilación RFC en segundo plano
  * `NR_SAP_IDOC_Configuration`: descargar datos maestros de IDOC para mejorar el rendimiento

* <DNT>**SYSTEM group**</DNT> incluye todos los proveedores de datos restantes. Consulte la [lista completa por grupo de monitoreo](/docs/sap-solutions/monitor/data-providers-reference).

#### Programar cada trabajo

1. Iniciar transacción `SM36`.
2. Introduzca el <DNT>**Job Name**</DNT> para cada trabajo y establezca el <DNT>**Job Class**</DNT> en <DNT>**C**</DNT>.
3. Haga clic en <DNT>**Step**</DNT>, ingrese el <DNT>**Program Name**</DNT> `/NEWR/ONE_COLLECTION` y seleccione el <DNT>**Variant**</DNT>.
4. Haga clic en <DNT>**Start Condition**</DNT>, configure <DNT>**Start Date**</DNT> y <DNT>**Time**</DNT> y marque la casilla <DNT>**Periodic Job**</DNT>.
5. Vaya a <DNT>**Period Values**</DNT>, haga clic en <DNT>**Other Period**</DNT> e ingrese <DNT>**5**</DNT> para los minutos.

## Programar reenvío del programa [#resubmit-program]

1. Cree un <DNT>**variant**</DNT> llamado <DNT>**DEFAULT**</DNT> para el programa `/NEWR/RESEND_FAILED_API_TRANS` para volver a enviar transmisiones fallidas para el <DNT>**New Relic account**</DNT> en caso de un error de transmisión.

2. Especifique los siguientes valores:

   * [<DNT>**New Relic Account ID**</DNT>](/docs/accounts/accounts-billing/account-setup/account-id)
   * <DNT>**HTTP Response Codes**</DNT>: Excluir `000` y `413`, y el rango de `200` a `299`.

   <img title="Resend failed API transactions" alt="A screenshot displaying the resend failed API transactions" src="/images/resend-failed-api-transaction.webp" width="700" />

3. Programe este trabajo para que se ejecute cada 15 minutos para volver a enviar transmisiones fallidas, como aquellas con <DNT>**HTTP response code**</DNT> `500`, y purgar periódicamente <DNT>**transmission log entries**</DNT>.
