Complete estas tareas de configuración para cada sistema monitoreado.

## Rol de autorización de carga [#upload-authorization-role]

1. Use <DNT>**SAP transaction**</DNT> `PFCG`.

2. Go to <DNT>**Role**</DNT> &gt; <DNT>**Upload**</DNT> and upload the file <DNT>**\_NEWR\_DATA\_PROVIDER.SAP**</DNT> from the installation package.

3. Confirme que el rol `/NEWR/DATA_PROVIDER` se carga.

   <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

4. Select the<img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   ícono después de que se cargue el rol.

5. On the <DNT>**Authorizations**</DNT> tab, select<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   <DNT>**Change Authorization Data**</DNT>, and then activate it using the<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />icono.

   <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

6. Si se sugiere un nombre de perfil, seleccione el icono de marca de verificación verde.

7. Verify that <DNT>**Profile(s) were updated**</DNT> appears on the status bar.

## Crear New Relic RFC [#create-rfc-user-id]

After you activate the <DNT>**authorization profile**</DNT>, use transaction `SU01` to create a new <DNT>**RFC user ID**</DNT> (for example, `NR4SAPDP`). Set the <DNT>**user type**</DNT> to <DNT>**System**</DNT> or <DNT>**Communication**</DNT> and add the role `/NEWR/DATA_PROVIDER`.

<img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />

## Crear destinos RFC para sistemas monitoreados [#create-rfc-for-monitored-systems]

Use transaction `SM59` to create an <DNT>**RFC destination**</DNT> for each monitored system.

1. Select <DNT>**ABAP Connections**</DNT> and then select the <DNT>**Create**</DNT> icon to set up a new <DNT>**RFC destination**</DNT>.

2. Enter a descriptive <DNT>**RFC Destination**</DNT> name. We recommend: `NR_<SYSID>CLNT<CLNT#>`

3. Establezca <DNT>**Connection Type**</DNT> en `3 - ABAP Connection`.

4. On the <DNT>**Technical Settings**</DNT> tab, enter:

   * <DNT>**Target Host**</DNT> (fully qualified domain or IP) and <DNT>**Instance No.**</DNT>
   * <DNT>**Gateway Host**</DNT> and <DNT>**Service**</DNT> if necessary.

   <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

5. On the <DNT>**Logon &amp; Security**</DNT> tab, enter the <DNT>**RFC user**</DNT> (`NR4SAPDP`) and password.

6. Seleccione <DNT>**Save**</DNT>.

7. Go to <DNT>**Utility &gt; Test &gt; Connection Test**</DNT> and <DNT>**Authorization Test**</DNT>.

   * <DNT>**Expected result**</DNT>: Both tests show successful status.

## Establecer el destino HTTP en cloud New Relic

Use transaction `SM59` to set up <DNT>**HTTPS connections**</DNT> to the <DNT>**New Relic API endpoints**</DNT>.

<Callout variant="important">
  Create individual <DNT>**RFC destinations**</DNT> for <DNT>**Events**</DNT>, <DNT>**Logs**</DNT>, <DNT>**Metrics**</DNT>, and <DNT>**Traces**</DNT> separately.
</Callout>

1. Enter a <DNT>**Destination Name**</DNT> (for example, `NR_API_EVENT`).

2. Set <DNT>**Connection Type**</DNT> to `G` (HTTP Connections to External Server).

3. Enter <DNT>**Target Host**</DNT>, <DNT>**Service No.**</DNT> as <DNT>**443**</DNT> (see [standard port](/docs/sap-solutions/reference/network-accessibility) for <DNT>**HTTPS**</DNT>), and <DNT>**Path Prefix**</DNT> (see [installation readiness checklist](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness) for details).

4. If you need a proxy, configure it on the <DNT>**Technical Settings**</DNT> tab.

5. On the <DNT>**Security Options**</DNT> tab, select the <DNT>**Active**</DNT> option for <DNT>**SSL**</DNT>.

   <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />

6. If needed, download the <DNT>**root certificate**</DNT> from [New Relic](https://one.newrelic.com) and add it to <DNT>**SAP**</DNT> using transaction `STRUST`.

7. Select <DNT>**Connection Test**</DNT> to verify the connection.

   * <DNT>**Expected result**</DNT>: A successful connection prompts you to enter logon data.
   * <DNT>**If test times out**</DNT>: Check your firewall settings.

8. Repeat this procedure for all <DNT>**New Relic API endpoints**</DNT>.

## Crear autenticación BTP/CloudALM extremo [#create-authentication-endpoint]

Siga las instrucciones a continuación según el servicio:

      9. Set <DNT>**Secure Protocol**</DNT> status to <DNT>**Active**</DNT> and save the configuration.
      10. Seleccione <DNT>**Connection Test**</DNT>.

## Crear una conexión a la base de datos [#create-database-connection]

Use transaction `DBACOCKPIT` to define the remote <DNT>**HANA Database connections**</DNT>.

1. Double-click <DNT>**Database Connections**</DNT> and select <DNT>**Add**</DNT>.

   <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

2. Enter a <DNT>**Connection Name**</DNT> (preferably the same as the <DNT>**SAP system ID**</DNT>).

3. Seleccione <DNT>**HANA Database**</DNT> como <DNT>**Database System**</DNT>.

4. Enter <DNT>**Database user ID**</DNT> (for example, `NRAGENT`) and password.

5. Enter <DNT>**Database Host**</DNT> and <DNT>**SQL Port**</DNT>.

   <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

6. Select <DNT>**Save**</DNT>, and then select <DNT>**Test**</DNT> to verify the connection.

   * <DNT>**Expected result**</DNT>: Connection test shows successful status.
