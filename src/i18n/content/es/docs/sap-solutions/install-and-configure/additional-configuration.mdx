Complete estas tareas de configuración para cada sistema monitoreado.

## Rol de autorización de carga [#upload-authorization-role]

1. Use <DNT>**SAP transaction**</DNT> `PFCG`.

2. Vaya a <DNT>**Role**</DNT> &gt; <DNT>**Upload**</DNT> y cargue el archivo <DNT>**\_NEWR\_DATA\_PROVIDER.SAP**</DNT> desde el paquete de instalación.

3. Confirme que el rol `/NEWR/DATA_PROVIDER` se carga.

   <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

4. Seleccione el<img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   ícono después de que se cargue el rol.

5. En la pestaña <DNT>**Authorizations**</DNT>, seleccione<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   <DNT>**Change Authorization Data**</DNT> y luego actívelo mediante el<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />icono.

   <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

6. Si se sugiere un nombre de perfil, seleccione el icono de marca de verificación verde.

7. Verifique que <DNT>**Profile(s) were updated**</DNT> aparezca en la barra de estado.

## Crear New Relic RFC [#create-rfc-user-id]

Luego de activar el <DNT>**authorization profile**</DNT>, use la transacción `SU01` para crear un nuevo <DNT>**RFC user ID**</DNT> (por ejemplo, `NR4SAPDP`). Establezca el <DNT>**user type**</DNT> en <DNT>**System**</DNT> o <DNT>**Communication**</DNT> y agregue el rol `/NEWR/DATA_PROVIDER`.

<img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />

## Crear destinos RFC para sistemas monitoreados [#create-rfc-for-monitored-systems]

Use la transacción `SM59` para crear un <DNT>**RFC destination**</DNT> para cada sistema monitoreado.

1. Seleccione <DNT>**ABAP Connections**</DNT> y luego seleccione el ícono <DNT>**Create**</DNT> para configurar un nuevo <DNT>**RFC destination**</DNT>.

2. Ingrese un nombre de <DNT>**RFC Destination**</DNT> descriptivo. Recomendamos: `NR_<SYSID>CLNT<CLNT#>`

3. Establezca <DNT>**Connection Type**</DNT> en `3 - ABAP Connection`.

4. En la pestaña <DNT>**Technical Settings**</DNT>, ingrese:

   * <DNT>**Target Host**</DNT> (dominio completo o IP) y <DNT>**Instance No.**</DNT>
   * <DNT>**Gateway Host**</DNT> y <DNT>**Service**</DNT> si es necesario.

   <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

5. En la pestaña <DNT>**Logon &amp; Security**</DNT>, ingrese el <DNT>**RFC user**</DNT> (`NR4SAPDP`) y la contraseña.

6. Seleccione <DNT>**Save**</DNT>.

7. Vaya a <DNT>**Utility &gt; Test &gt; Connection Test**</DNT> y <DNT>**Authorization Test**</DNT>.

   * <DNT>**Expected result**</DNT>: Ambas pruebas muestran un estado exitoso.

## Establecer el destino HTTP en cloud New Relic

Emplee la transacción `SM59` para configurar <DNT>**HTTPS connections**</DNT> al <DNT>**New Relic API endpoints**</DNT>.

<Callout variant="important">
  Cree <DNT>**RFC destinations**</DNT> individuales para <DNT>**Events**</DNT>, <DNT>**Logs**</DNT>, <DNT>**Metrics**</DNT> y <DNT>**Traces**</DNT> por separado.
</Callout>

1. Ingrese un <DNT>**Destination Name**</DNT> (por ejemplo, `NR_API_EVENT`).

2. Establezca <DNT>**Connection Type**</DNT> en `G` (Conexiones HTTP a servidor externo).

3. Ingrese <DNT>**Target Host**</DNT>, <DNT>**Service No.**</DNT> como <DNT>**443**</DNT> (consulte el [puerto estándar](/docs/sap-solutions/reference/network-accessibility) para <DNT>**HTTPS**</DNT>) y <DNT>**Path Prefix**</DNT> (consulte la [lista de verificación de preparación para la instalación](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness) para obtener más detalles).

4. Si necesita un proxy, configúrelo en la pestaña <DNT>**Technical Settings**</DNT>.

5. En la pestaña <DNT>**Security Options**</DNT>, seleccione la opción <DNT>**Active**</DNT> para <DNT>**SSL**</DNT>.

   <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />

6. Si es necesario, descargue el <DNT>**root certificate**</DNT> de [New Relic](https://one.newrelic.com) y agréguelo a <DNT>**SAP**</DNT> usando la transacción `STRUST`.

7. Seleccione <DNT>**Connection Test**</DNT> para verificar la conexión.

   * <DNT>**Expected result**</DNT>: una conexión exitosa le muestra un símbolo para que ingrese los datos de inicio de sesión.
   * <DNT>**If test times out**</DNT>: verifique la configuración de su firewall.

8. Repita este procedimiento para todos los <DNT>**New Relic API endpoints**</DNT>.

Siga las instrucciones a continuación según el servicio:

      <img title="Service No." alt="A screenshot displaying the Service No." src="/images/service-no.webp" width="600" />

## Crear una conexión a la base de datos [#create-database-connection]

Use la transacción `DBACOCKPIT` para definir el <DNT>**HANA Database connections**</DNT> remoto.

1. Haga doble clic en <DNT>**Database Connections**</DNT> y seleccione <DNT>**Add**</DNT>.

   <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

2. Ingrese un <DNT>**Connection Name**</DNT> (preferiblemente el mismo que el <DNT>**SAP system ID**</DNT>).

3. Seleccione <DNT>**HANA Database**</DNT> como <DNT>**Database System**</DNT>.

4. Ingrese <DNT>**Database user ID**</DNT> (por ejemplo, `NRAGENT`) y la contraseña.

5. Ingrese <DNT>**Database Host**</DNT> y <DNT>**SQL Port**</DNT>.

   <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

6. Seleccione <DNT>**Save**</DNT> y luego seleccione <DNT>**Test**</DNT> para verificar la conexión.

   * <DNT>**Expected result**</DNT>: la prueba de conexión muestra un estado exitoso.
