Puede instalar el agente eBPF de New Relic en su clúster de Kubernetes para monitorear todo el estado del sistema. El agente eBPF proporciona una visibilidad profunda del rendimiento de la aplicación sin requerir cambios de código ni desplegar un agente específico del idioma.

## Instalar el agente eBPF de New Relic

<Steps>
  <Step>
    ### Antes de que empieces [#requirements]

    Debe cumplir con los requisitos previos descritos en la documentación [de compatibilidad y requisitos de eBPF](/docs/ebpf/requirements#k8s) para el clúster de Kubernetes.
  </Step>

  <Step>
    ### Instalar el agente eBPF [#install]

    Para instalar el agente eBPF:

    1. Log sesión en su cuenta New Relic .

    2. Vaya a <DNT>**[left navigation pane &gt; + Integration &amp; Agents &gt; eBPF Agent](https://onenr.io/0oR8XWW36RG)**</DNT>.

    3. En la pantalla <DNT>Select an account</DNT>, seleccione la cuenta en la que desea instalar el agente eBPF y haga clic en <DNT>**Continue**</DNT>.

    4. En la página <DNT>Select an installation method</DNT>, seleccione <DNT>**Kubernetes**</DNT> y haga clic en <DNT>**Continue**</DNT>.

    5. En la pantalla <DNT>Enter your user key</DNT>, seleccione una de las siguientes opciones y luego haga clic en <DNT>**Continue**</DNT>:

       * <DNT>**Use an existing key**</DNT>: Si ya tiene una clave de usuario, proporcione la clave de usuario. Para obtener más información, consulta [Claves de usuario](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
       * <DNT>**Create a new key**</DNT>: Si no tienes una clave de usuario, haz clic en <DNT>**Create a new key**</DNT> para crear una.

    6. En la pantalla <DNT>Configure the Kubernetes integration</DNT>:

       1. Ingrese el nombre de la implementación de Kubernetes.
       2. (Opcional) Ingrese el namespace para la integración. El namespace predeterminado es <DNT>`newrelic`</DNT>.
       3. Haga clic en <DNT>**Continue**</DNT>.

       <Callout variant="tip">
         Si elige un namespace personalizado para su instrumentación New Relic (en lugar del <DNT>`newrelic`</DNT> predeterminado), le recomendamos excluir ese namespace del monitoreo agregándolo al parámetro de configuración <DNT>`allDataFilters.dropNamespaces`</DNT>. Esto evita que el agente eBPF monitoree el módulo de instrumentación por sí mismo. Por ejemplo, si emplea <DNT>`newrelic-mon`</DNT> como su namespace, configure: <DNT>`allDataFilters.dropNamespaces: ["kube-system", "newrelic-mon"]`</DNT>.
       </Callout>

    7. En la pantalla <DNT>Install the Kubernetes integration</DNT>:

       1. Copie y pegue el comando que se muestra para instalar el agente eBPF en su clúster de Kubernetes usando Helm.

       2. (Opcional) Para descargar el archivo de configuración de <DNT>`values.yaml`</DNT>, haz clic en <DNT>**Download**</DNT>. Para obtener más información sobre los parámetros de configuración, consulte [Parámetros de configuración de K8s](#config-params).

       3. (Opcional) Actualice el archivo <DNT>`values.yaml`</DNT> según sea necesario y guárdelo.

       4. (Opcional) Para aplicar los cambios de configuración, ejecute el siguiente comando:

          ```bash
              helm repo update ; helm upgrade --install nr-ebpf-agent newrelic/nr-ebpf-agent -n newrelic --values values.yaml
          ```

       5. Para verificar la instalación, ejecute el siguiente comando:

          ```bash
              kubectl get pods -n newrelic
          ```
  </Step>

  <Step>

    2. En el banner de búsqueda, establezca el criterio de búsqueda como <DNT>`instrumentation.name = nr_ebpf`</DNT>:

  </Step>

## Actualizar el agente eBPF [#upgrade]

Para actualizar el agente eBPF en un clúster de Kubernetes:

* **Para una actualización estándar**: use el siguiente comando Helm para actualizar a la última versión:

```bash
KSM_IMAGE_VERSION="v2.13.0" && helm repo add newrelic https://helm-charts.newrelic.com && helm repo update && kubectl create namespace "newrelic" ; helm upgrade --install nr-ebpf-agent newrelic/nr-ebpf-agent --set licenseKey=<key> --set cluster="<cluster-name>" --namespace=newrelic
```

* **Para actualizar una versión específica:** Para actualizar a una versión específica, emplee el indicador `--version` :

```bash
KSM_IMAGE_VERSION="v2.13.0" && helm repo add newrelic https://helm-charts.newrelic.com && helm repo update && kubectl create namespace "newrelic" ; helm upgrade --install nr-ebpf-agent newrelic/nr-ebpf-agent --set licenseKey=<key> --set cluster="<cluster-name>" --namespace=newrelic --version=0.2.5
```

Reemplace `<key>` con su clave de licencia de New Relic y `<cluster-name>` con el nombre de su clúster.

## Parámetro de configuración [#config-params]

El archivo [`values.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-ebpf-agent/values.yaml) contiene las siguientes secciones de configuración:

<Callout variant="tip">
  El agente eBPF genera automáticamente nombres de entidades de forma diferente según el entorno:

  * En los hosts o Docker, estos nombres son una combinación del nombre del proceso, su directorio o ID de contenedor y el puerto de escucha. Por ejemplo, `ruby:/home/ubuntu/app:[5678]` o `java:f4aead533895:[8080]`.

  * En Kubernetes, estos nombres se derivan del nombre del servicio, por ejemplo, `mysql-database-service`.

  Asignar un nombre personalizado a la aplicación:

  * Puede asignar un nombre personalizado a su aplicación configurando la variable de entorno `NEW_RELIC_APP_NAME` tanto para Kubernetes como para las aplicaciones en el host.
</Callout>

## Desinstalar el agente eBPF [#uninstall]

Para desinstalar el agente eBPF de su clúster de Kubernetes:

```bash
helm uninstall nr-ebpf-agent -n newrelic
```

<Callout variant="tip">
  Este comando eliminará todos los componentes del agente eBPF de su clúster. El namespace permanecerá a menos que lo elimine explícitamente.
</Callout>
