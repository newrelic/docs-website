Puede instalar el agente eBPF de New Relic en su host Linux para monitorear el estado de todo su sistema. El agente eBPF proporciona una visibilidad profunda del rendimiento de la aplicación sin requerir cambios de código ni desplegar un agente específico del idioma.

## Instalar el agente eBPF de New Relic [#install-linux]

<Steps>
  <Step>
    ### Antes de que empieces [#requirements]

    Debe cumplir con los requisitos previos descritos en la documentación [de requisitos y compatibilidad de eBPF](/docs/ebpf/requirements#linux) para hosts Linux.
  </Step>

  <Step>
    ### Instalar el agente eBPF [#install]

    Para instalar el agente eBPF:

    1. Log sesión en su cuenta New Relic .

    2. Vaya a <DNT>**[left navigation pane &gt; + Integration &amp; Agents &gt; eBPF Agent](https://onenr.io/0oR8XWW36RG)**</DNT>.

    3. En la pantalla <DNT>Select an account</DNT>, seleccione la cuenta en la que desea instalar el agente eBPF y haga clic en <DNT>**Continue**</DNT>.

    4. En la página <DNT>Select an installation method</DNT>, seleccione <DNT>**On a host**</DNT> y haga clic en <DNT>**Continue**</DNT>.

    5. En la pantalla <DNT>Enter your user key</DNT>, seleccione una de las siguientes opciones y luego haga clic en <DNT>**Continue**</DNT>:

       * <DNT>**Use an existing key**</DNT>: Si ya tiene una clave de usuario, proporcione la clave de usuario. Para obtener más información, consulta [Claves de usuario](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
       * <DNT>**Create a new key**</DNT>: Si no tienes una clave de usuario, haz clic en <DNT>**Create a new key**</DNT> para crear una.

    6. En la pantalla <DNT>Configure the host integration</DNT>, ingrese el nombre de despliegue para el host, luego haga clic en <DNT>**Continue**</DNT>.

    7. En la pantalla <DNT>Install the agent</DNT>, ejecute el comando proporcionado en su terminal para instalar el agente eBPF a través del paquete de Debian.

       1. Copie y pegue el comando mostrado para instalar el agente eBPF en su host Debian.

       2. Para verificar la instalación, ejecute el siguiente comando:

          ```bash
          sudo systemctl status newrelic-ebpf-agent
          ```

       Tras una instalación exitosa, el estado de estos servicios se mostrará como activo (en ejecución).

       <Callout variant="tip">
         Puede personalizar la configuración del agente eBPF editando el archivo <DNT>`newrelic-ebpf-agent.conf`</DNT> disponible en <DNT>`/etc/newrelic-ebpf-agent/newrelic-ebpf-agent.conf`</DNT>. Para obtener más información sobre el parámetro de configuración, consulte [parámetro de configuración](#config-params).
       </Callout>
  </Step>

  <Step>
    ### Acceda a los datos eBPF en New Relic [#access-data]

    Una vez instalado el agente eBPF, comienza a recopilar datos automáticamente de su host Linux. Puede acceder a estos datos en UI OpenTelemetry de New Relic. Para obtener más información sobre UI OpenTelemetry New Relic, consulte [UI OpenTelemetry APM ](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-ui).

    **Para ver los datos eBPF en New Relic:**

    1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; APM &amp; Services**</DNT>.
    2. En el banner de búsqueda, establezca el criterio de búsqueda como <DNT>`instrumentation.name = nr_ebpf`</DNT>:
       <img style={{ align: 'left' }} title="eBPF filters" alt="eBPF filter for eBPF data in New Relic OpenTelemetry UI" src="/images/ebpf_filters.webp" />
  </Step>
</Steps>

## Actualizar el agente eBPF [#upgrade]

Para actualizar el agente eBPF en un host Linux, vuelva a ejecutar el comando de instalación desde la UI New Relic. Este proceso instalará la nueva versión del agente y eliminará automáticamente la anterior.

<Callout variant="important">
  No se admite la actualización a través de un administrador de paquetes. Debes emplear el comando de instalación desde la UI New Relic para una actualización exitosa.
</Callout>

Para obtener el último comando de instalación:

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; + Integration &amp; Agents &gt; eBPF Agent**</DNT>.

2. Sigue el flujo de instalación para obtener el comando actualizado.

3. Ejecute el comando en su host para actualizar el agente.

4. Reinicie el servicio del agente ebpf usando el siguiente comando:

   ```bash
   sudo systemctl restart newrelic-ebpf-agent
   ```

## Parámetro de configuración [#config-params]

El archivo `newrelic-ebpf-agent.conf` contiene el siguiente parámetro de configuración:

<Callout variant="tip">
  El agente eBPF genera automáticamente nombres de entidades de forma diferente según el entorno:

  * En los hosts o Docker, estos nombres son una combinación del nombre del proceso, su directorio o ID de contenedor y el puerto de escucha. Por ejemplo, `ruby:/home/ubuntu/app:[5678]` o `java:f4aead533895:[8080]`.

  * En Kubernetes, estos nombres se derivan del nombre del servicio, por ejemplo, `mysql-database-service`.

  Asignar un nombre personalizado a la aplicación:

  * Puede asignar un nombre personalizado a su aplicación configurando la variable de entorno `NEW_RELIC_APP_NAME` tanto para Kubernetes como para las aplicaciones en el host.
</Callout>

## Desinstalar el agente eBPF [#uninstall]

Para desinstalar el agente eBPF de un host Linux, emplee las herramientas de administración de paquetes de su sistema:

```bash
sudo apt-get purge newrelic-ebpf-agent
```

<Callout variant="tip">
  No es necesario detener el servicio antes de ejecutar el comando de desinstalación.
</Callout>
