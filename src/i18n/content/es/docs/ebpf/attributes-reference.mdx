Esta referencia cubre los atributos que son exclusivos del agente eBPF de New Relic para eBPF APM y las métricas de red de eBPF. Los datos de eBPF fluyen hacia los tipos de datos estándar de NRDB (`Metric`, `Span` y `Log`), y estos atributos están disponibles para consultar, facetar y alertar en NRQL.

El agente eBPF también emite atributos de convención semántica estándar de OpenTelemetry (por ejemplo, `http.request.method`, `db.system`, `jvm.memory.usage`) y campos de span estándar de OpenTelemetry (`trace_id`, `span_id`, `kind`, etcétera). Consulte las [convenciones semánticas de OpenTelemetry](https://opentelemetry.io/docs/specs/semconv/) y la [especificación de la API de traza](https://opentelemetry.io/docs/specs/otel/trace/api/#span) para ver sus definiciones.

Toda la telemetría de eBPF contiene los siguientes atributos de ámbito:
