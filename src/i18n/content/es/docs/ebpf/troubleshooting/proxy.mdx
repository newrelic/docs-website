Si el <DNT>eBPF agent</DNT> de New Relic no enruta la telemetría a través de su proxy como se esperaba, utilice esta página para diagnosticar errores comunes. Para obtener instrucciones de instalación y referencia de configuración, consulte [Configurar un proxy para el <DNT>eBPF agent</DNT>de New Relic](/docs/ebpf/proxy-configuration).

## Errores comunes [#common-errors]

<CollapserGroup>
  <Collapser id="error-400" title="El proxy HTTP devolvió el código de respuesta 400">
    **Problema:** el proxy no admite el método HTTP <DNT>`CONNECT`</DNT>.

    **Solución:** use un proxy compatible con <DNT>`CONNECT`</DNT>. Consulte [Servidores proxy compatibles](/docs/ebpf/proxy-configuration#supported-proxies).
  </Collapser>

  <Collapser id="error-https-scheme" title="Esquema 'https' no admitido en el URI del proxy">
    **Problema:** gRPC no admite <DNT>`https://`</DNT> como esquema de proxy. Esta es una limitación de gRPC, no una restricción del agente.

    **Solución:** establezca el esquema del proxy en <DNT>`http`</DNT> (por ejemplo, <DNT>`otlpProxy.scheme: "http"`</DNT>). Su telemetry data aún viaja con cifrado TLS de extremo a extremo a través del túnel proxy. Solo la conexión al proxy en sí es en texto plano.
  </Collapser>

  <Collapser id="error-socket-closed" title="Socket cerrado o conexión rechazada">
    **Problema:** el servidor proxy no se está ejecutando, o el host o el puerto es incorrecto.

    **Solución:** verifique que el proxy sea accesible usando el [comando de prueba<DNT>`curl`</DNT> ](/docs/ebpf/proxy-configuration#test-connectivity).
  </Collapser>

  <Collapser id="error-connect-aborted" title="Proxy CONNECT abortado">
    **Problema:** el proxy rechazó la solicitud <DNT>`CONNECT`</DNT> debido a una regla de firewall, una ACL o un destino no admitido.

    **Solución:** revise los registros del servidor proxy y confirme que las requests <DNT>`CONNECT`</DNT> al puerto `443` estén permitidas desde la IP de origen del agente.
  </Collapser>

  <Collapser id="error-tls-required" title="Configuración de proxy ignorada: el proxy gRPC HTTP CONNECT requiere TLS">
    **Problema:** un proxy está configurado, pero TLS está deshabilitado (por ejemplo, <DNT>`TLS_ENABLED=false`</DNT>). La implementación del proxy HTTP CONNECT de gRPC solo funciona cuando la conexión al extremo OTLP utiliza TLS.

    **Solución:** asegúrese de que TLS esté habilitado (está habilitado de forma predeterminada). El agente usará entonces el proxy configurado.
  </Collapser>

  <Collapser id="error-credential-chars" title="El nombre de usuario del proxy contiene caracteres especiales o en mayúsculas">
    **Problema:** su nombre de usuario o contraseña del proxy contiene caracteres que el analizador de URI de gRPC no acepta. Consulta [Limitaciones de caracteres de las credenciales](/docs/ebpf/proxy-configuration#credential-limitations).

    **Solución:** use una cuenta proxy con un nombre de usuario alfanumérico en minúsculas y una contraseña que use solo letras, dígitos y los caracteres especiales permitidos.
  </Collapser>
</CollapserGroup>
