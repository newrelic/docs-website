Esta página proporciona información completa sobre compatibilidad para el agente eBPF New Relic, incluida la base de datos compatible, los requisitos de implementación y las capacidades de consulta de datos que se aplican a todos los métodos de instalación.

<Tabs>
  <TabsBar>
    <TabsBarItem id="k8s">Para Kubernetes</TabsBarItem> <TabsBarItem id="linux">Para Linux</TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="k8s">
      * Asegúrese de que se cumplan todos los [requisitos de la distribución de Linux](/docs/ebpf/requirements/#kernel-requirements) para el nodo de Kubernetes.

      ### Entornos de Kubernetes [#kubernetes-environments]

      Se admiten los siguientes entornos de Kubernetes:

      * Servicio Amazon Elastic Kubernetes (EKS)
      * Motor Google Kubernetes (GKE)
      * Servicio Azure Kubernetes (AKS)
      * Red Hat OpenShift
      * Microk8s

      ### Soporte para el entorno de ejecución de contenedores [#container-runtime]

      Se admiten los siguientes tiempos de ejecución de contenedores:

      * Entorno de ejecución predeterminado de Docker (runc)
      * containerd
      * CRI-O
      * Podman
      * Youki
      * crun
    </TabsPageItem>

    <TabsPageItem id="linux">
      ### Requisitos del kernel [#kernel-requirements]

      * Versiones del kernel de Linux:

        * `x86-64` (amd64): `v5.8` y superior
        * `arm64` (aarch64): `v5.8` y superior

      <Callout variant="important">
        Algunas distribuciones de Linux permiten ejecutar versiones anteriores del kernel en versiones más recientes del sistema operativo. Aunque su distribución esté [listada a continuación](#supported-linux-distributions), el agente eBPF no funcionará si la versión del kernel es inferior a `5.8`.
      </Callout>

      ### Distribuciones de Linux compatibles [#supported-linux-distributions]

      Las siguientes distribuciones de Linux son compatibles:

      * CentOS `9` y superior
      * Debian `11` y superior
      * Red Hat Enterprise Linux `8.1` y superior
      * Ubuntu `20.04` y superior
      * Amazon Linux `2` and `2023`
      * Bottle Rocket `1.51.0` y superior
      * AlmaLinux `9.7`

      ### Requisito del sistema [#system-requirements]

      * Privilegios de root (Requeridos para la instalación y operación)
      * Un mínimo de `2 GB` de RAM
    </TabsPageItem>
  </TabsPages>
</Tabs>

### Soporte para protocolos adicionales [#protocol-support]

El agente eBPF también admite el monitoreo de estos protocolos:

* HTTP

* Apache Thrift

* gRPC

* Casandra

* kafka

* AMQP

  <Callout variant="tip">
    Para Cassandra, Kafka y AMQP, el agente eBPF no admite el enlace de segmentos.
  </Callout>

<Callout variant="tip">
  El monitoreo de eBPF es independiente del lenguaje y proporciona métricas de latencia y rendimiento para cualquier aplicación. Sin embargo, [la vinculación de segmentos de transacciones](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm/#segments) depende del lenguaje y actualmente no es compatible con PHP y Node.js.
</Callout>

### Entity naming

The eBPF agent automatically generates entity names differently depending on the environment:

* **In hosts or Docker**: Names are a combination of the process name, its directory or container ID, and the listening port. For example, `ruby:/home/ubuntu/app:[5678]` or `java:f4aead533895:[8080]`.

* **In Kubernetes**: Names are derived from the service name, for example, `mysql-database-service`.

eBPF data is accessible through multiple New Relic interfaces:

1. **APM & Services UI**: Go to **[one.newrelic.com](https://one.newrelic.com) > APM & Services** and filter by `instrumentation.name = nr_ebpf`

2. **NRQL queries**: Use the query examples above to explore your data

3. **OpenTelemetry UI**: eBPF data integrates with New Relic's OpenTelemetry experience
