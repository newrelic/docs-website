Utilice Pixie [entidad](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/) para ver métricas doradas y una muestra de solicitudes individuales de cuerpo completo enviadas a un servicio específico utilizando un protocolo específico. Pixie entidad funciona con datos que Pixie ha trazado automáticamente. Seleccione un protocolo a continuación para obtener más información:

* [DNS](#dns)
* [MySQL](#mysql)
* [PostgreSQL](#postgres)
* [Redis](#redis)
* [kafka](#kafka)
* [AMQP](#amqp)

Puedes encontrar la entidad Pixie en la sección <DoNotTranslate>**Other entities**</DoNotTranslate> de la página <DoNotTranslate>**All entities**</DoNotTranslate> :

## DNS [#dns]

Seleccione un servicio de la lista de entidades <DoNotTranslate>**DNS - Pixie**</DoNotTranslate> para ver gráficos de series temporales de tiempo de respuesta, tasa de errores y rendimiento para todas las solicitudes de DNS enviadas a ese servicio. Inspeccione la tabla Solicitudes de DNS de muestra para ver las solicitudes de DNS de cuerpo completo y las respuestas recibidas por este servicio.

Las solicitudes DNS trazadas automáticamente por Pixie se pueden utilizar para:

* Determine rápidamente qué servicios están realizando solicitudes de DNS.
* Obtenga información de rendimiento y tiempo de respuesta de DNS de alto nivel.
* Observe los desequilibrios de rendimiento entre los servidores DNS.
