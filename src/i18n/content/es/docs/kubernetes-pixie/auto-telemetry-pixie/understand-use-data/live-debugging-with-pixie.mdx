Utilice la pestaña <DNT>**Live debugging with Pixie**</DNT> para consultar el conjunto completo de telemetry data que Pixie recopila en el almacenamiento a corto plazo en su clúster.

Obtenga más información sobre el modelo de almacenamiento híbrido para la integración de New Relic Pixie [aquí](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/overview).

La pestaña <DNT>**Live debugging with Pixie**</DNT> contiene una vista integrada de la UI de Pixie. Las vistas en Pixie funcionan con el script PxL (script escrito en el lenguaje PxL de Pixie) para consultar y visualizar los telemetry data capturados por eBPF.

Pixie proporciona muchos [scripts de código abierto PxL](https://github.com/pixie-io/pixie/tree/main/src/pxl_scripts) que le permiten depurar:

* Tráfico en múltiples formatos: HTTP y HTTPs (incluido el cifrado), DNS, Postgres, MySQL, Cassandra, Redis, Kafka. Más información: [Solicite un tutorial de seguimiento](https://docs.px.dev/tutorials/pixie-101/request-tracing/).
* Base de datos solicitud rendimiento. Más información: [consulta de la base de datos tutorial de creación de perfiles](https://docs.px.dev/tutorials/pixie-101/database-query-profiling/).
* Mapas de servicios para saber qué servicios se comunican entre sí. Más información: [Tutorial de rendimiento del servicio](https://docs.px.dev/tutorials/pixie-101/service-performance/).
* Mapas de tráfico de red para saber qué nodos se comunican entre sí. Más información: [Monitoreo de red tutorial](https://docs.px.dev/tutorials/pixie-101/network-monitoring/).
* Uso de recursos por nodo y pod. Obtenga más información: [Tutorial de salud de infraestructura](https://docs.px.dev/tutorials/pixie-101/infra-health/).
* Picos en la utilización de la CPU. Más información: [Tutorial de creación de perfiles de CPU continuos](https://docs.px.dev/tutorials/pixie-101/profiler/).

## Ejecutar un script PxL [#run-script]

<Callout variant="important">
  Para obtener mejores resultados, seleccione `5 minutes` en el selector de tiempo cuando utilice la pestaña <DNT>**Live Debugging with Pixie**</DNT> . Para examinar datos de Pixie menos recientes, consulte la sección [Explorar datos de Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/explore-pixie-data) .
</Callout>

Para ejecutar una script en la pestaña <DNT>**Live debugging with Pixie**</DNT>:

1. Seleccione una script en el menú desplegable `script`.
2. Complete los argumentos del script requeridos. Los argumentos obligatorios se indican con un `*` junto al nombre del argumento.
3. Ejecute el script usando el botón `RUN` en la parte superior derecha o usando el método abreviado de teclado: `ctrl+enter` (Windows, Linux) o `cmd+enter` (Mac).

## Siga los enlaces profundos del script [#deep-links]

Las vistas en Pixie contienen enlaces profundos que le permiten navegar fácilmente entre entidades Kubernetes .

Por ejemplo, la tabla <DNT>**Nodes**</DNT> en el script `px/cluster` (en la imagen de arriba) genera una lista de todos los nodos de su clúster. Haga clic en un nodo para acceder al script `px/node` para ese nodo en particular. El script `px/node` muestra una descripción general del pod, el uso de CPU y memoria y el tráfico de red de un nodo en particular.
