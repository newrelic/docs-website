## Problema

Tienes un problema con el [dockerhub de New Relic](https://hub.docker.com/u/newrelic) y el [`registry.k8s.io`](https://github.com/kubernetes/registry.k8s.io) de Google durante la instalación.

## Solución

Comprueba que agregaste su dominio a tu lista de &apos;permitidos&apos;. La instalación extrae las imágenes del contenedor desde esta ubicación. Puedes [probar la conectividad a `registry.k8s.io`](https://kubernetes.io/blog/2023/03/10/image-registry-redirect/#how-can-i-check-if-i-am-impacted) para encontrar el dominio de registro de Google adicional para agregar a tu lista blanca. `registry.k8s.io` generalmente redirecciona a su dominio de registro local. Por ejemplo, `asia-northeast1-docker.pkg.dev` según su región.
