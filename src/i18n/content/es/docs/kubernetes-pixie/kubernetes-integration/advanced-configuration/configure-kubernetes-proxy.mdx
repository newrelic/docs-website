Si está ejecutando nuestra integración Kubernetes con un proxy, debe configurar cada componente (el [agente de infraestructura](/docs/infrastructure/install-infrastructure-agent), el registro y minion sintético) para que funcione correctamente con su proxy.

## Instalar el agente de infraestructura con un proxy [#infrastructure-agent]

El proxy se puede configurar configurando la opción de configuración `global.proxy` y configurar toda la integración de Kubernetes al mismo tiempo. La configuración global se [comparte con la dependencia](https://github.com/newrelic/helm-charts/tree/master/charts/nri-bundle#configure-dependencies) modificando el archivo `values.yaml` del `nri-bundle`:

```yaml
global.proxy: https://user:password@hostname:port
```

Esto configurará todo el agente de infraestructura, la integración Kubernetes , el Adaptador métrico y el agente Prometheus al mismo tiempo.

<Callout variant="important">
  Todavía no admitimos Proxy en el operador de infraestructura de Kubernetes.
</Callout>
