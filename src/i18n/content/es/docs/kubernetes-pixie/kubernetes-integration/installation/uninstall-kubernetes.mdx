La integración de New Relic Kubernetes se puede desinstalar fácilmente de su plataforma.

Asegúrese de estar operando en el contexto apropiado en la máquina donde ejecuta Helm y `kubectl`. Puede comprobar los contextos disponibles con este comando:

```
kubectl config get-contexts
```

Cambie al contexto deseado usando:

```
kubectl config use-context CONTEXT_NAME
```

## Desinstalar Kubernetes usando Helm [#uninstall-helm]

Si utilizó Helm para instalar la integración de Kubernetes, ejecute:

```
helm uninstall newrelic-bundle -n newrelic
```

<Callout variant="important">
  La instalación guiada utiliza `newrelic-bundle` como nombre de versión predeterminado y `newrelic` como namespace predeterminado. Si instala nuestro paquete con un nombre de versión o namespace diferente, deberá sustituir los argumentos en el comando anterior: `helm uninstall <RELEASE-NAME> -n <RELEASE-NAMESPACE>`. Puede usar `helm list --all-namespaces` y buscar el paquete `nri-bundle` para encontrar el nombre de la versión y el namespace de la versión.
</Callout>

## Desinstalar Kubernetes usando manifiesto [#uninstall-manifest]

Si instaló la integración de Kubernetes usando un manifiesto, use el mismo manifiesto para desinstalarlo:

```
kubectl delete -f newrelic.yaml
```
