<Callout variant="tip">
  Esta guía es para la migración única desde una versión anterior a la 3.xx hacia la versión 3.xx. Si ya está en la versión 3.xx y necesita actualizar a una versión más reciente, siga los procedimientos de actualización estándar para su plataforma.
</Callout>

Para facilitar al máximo la migración desde versiones anteriores, hemos desarrollado una capa de compatibilidad que traduce la mayoría de las opciones de configuración a sus nuevos equivalentes. Esta capa de compatibilidad es temporal y se eliminará en el futuro. Le recomendamos leer esta guía detenidamente y migrar la configuración bajo supervisión humana.

## Migración a la versión 3.xx

Para experimentar la nueva vista seleccionada, debe migrar a la versión 3.xx del agente de medios.

### Complicaciones de tipo de evento dual

Existe la posibilidad de que luego de la migración, algunos de sus usuarios finales no optaron por la actualización de la aplicación, o muy pocos lo hicieron. Durante este periodo intermedio, es posible que experimentes dos tipos de eventos, lo que puede resultar complicado. Para gestionar esto:

* Reúne toda la consulta y acción mapeo.
* Reporte a su usuario final que durante este periodo interino, podrá ver datos en múltiples eventos y cómo se relacionan.

<Callout variant="important">
  Esto se aplica al agente móvil. Para el browser, se actualizará automáticamente.
</Callout>

### Visualización de datos luego de la migración

Luego de la migración, si desea ver los datos del último mes, le recomendamos que emplee cualquier dashboard personalizado o consulte:

```sql
SELECT * FROM pageAction
```

<Callout variant="important">
  Con definiciones de eventos con nombre, pierden la capacidad dashboard para consultar los datos antiguos.
</Callout>

### evento Definición mapeo

El mapeo de tipos de eventos entre las versiones anteriores y las nuevas es específico del cliente según su implementación. Para garantizar una transición fluida y mantener la integridad de los datos, contacta al soporte al cliente de New Relic para obtener un plan de migración personalizado.

Le recomendamos que siga atentamente esta guía y se cerciore de que todas las configuraciones se migren bajo supervisión. La capa de compatibilidad es una solución temporal y es importante realizar la transición a la nueva configuración lo antes posible.
