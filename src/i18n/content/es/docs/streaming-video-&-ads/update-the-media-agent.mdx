To make migrating from earlier versions as easy as possible, we have developed a compatibility layer that translates most of the configurable options chart to their new counterparts. This compatibility layer is temporary and will be removed in the future. We encourage you to read this guide carefully and migrate the configuration under human supervision.

## Migración a la versión 3.xx

Para experimentar la nueva vista seleccionada, debe migrar a la versión 3.xx del agente de medios.

### Complicaciones de tipo de evento dual

Existe la posibilidad de que luego de la migración, algunos de sus usuarios finales no optaron por la actualización de la aplicación, o muy pocos lo hicieron. Durante este periodo intermedio, es posible que experimentes dos tipos de eventos, lo que puede resultar complicado. Para gestionar esto:

* Reúne toda la consulta y acción mapeo.
* Reporte a su usuario final que durante este periodo interino, podrá ver datos en múltiples eventos y cómo se relacionan.

<Callout variant="important">
  Esto se aplica al agente móvil. Para el browser, se actualizará automáticamente.
</Callout>

### Visualización de datos luego de la migración

Luego de la migración, si desea ver los datos del último mes, le recomendamos que emplee cualquier dashboard personalizado o consulte:

```sql
SELECT * FROM pageAction
```

<Callout variant="important">
  With named event definitions, they loose the dashboard capability for querying the old data.
</Callout>

### evento Definición mapeo

Se proporciona un mapeo detallado de cada tipo de evento antiguo con el nuevo para ayudarlo a migrar cualquier script o flujo de trabajo. Este mapeo es crucial para garantizar una transición sin problemas y mantener la integridad de los datos.

We recommend you carefully follow this guide and ensure that all configurations are migrated under supervision. The compatibility layer is a temporary solution, and it is important to transition to the new configuration as soon as possible.

Para obtener información más detallada, comunicar con el soporte al cliente de New Relic .
