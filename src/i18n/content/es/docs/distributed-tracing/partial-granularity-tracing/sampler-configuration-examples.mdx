Si bien la configuración de muestreo predeterminada funciona para todos los servicios, puede que no sea óptima. Los agentes de APM proporcionan opciones de configuración de muestreo para que pueda personalizar el muestreo y ajustarlo a las necesidades de su servicio. Esta página presenta algunos ejemplos reales de diferentes tipos de servicios y cómo puede modificar la configuración de muestreo para adaptarse a esas necesidades.

## Ejemplo 1: Distribución deficiente de endpoints capturados debido al muestreo ascendente

La configuración predeterminada utiliza el muestreador adaptativo. Esto significa que cualquier traza entrante que haya sido muestreada por un servicio ascendente también será muestreada por el servicio actual. A veces, esto puede ocasionar que un endpoint en particular se muestree mucho, mientras que otros endpoints se descuiden y apenas se muestreen.

### Solución 1: Utilice el muestreador de proporción para muestrear trazas muestreadas por un servicio upstream

La siguiente configuración solucionará este problema al muestrear cualquier traza entrante que haya sido muestreada por un servicio ascendente solo el 10% del tiempo. Esto deja espacio para que otros endpoints sean muestreados al no llenar el depósito de spans con trazas que fueron muestreadas por un servicio upstream.

```json
{
    distributed_tracing: {
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.1
                }
            }
        }
    }
}
```

### Solución 2: Use el muestreador de proporción para capturar una porción de las trazas muestreadas por un servicio ascendente con granularidad completa y capturar el resto con granularidad parcial.

Si aún necesita capturar información mínima sobre una traza cuando es muestreada por un servicio upstream, puede utilizar la siguiente configuración en su lugar. Esta configuración captura el 5% de todas las trazas muestreadas por un servicio upstream con granularidad completa y el resto con granularidad parcial. Al capturar la mayoría de las trazas muestreadas por un servicio upstream con granularidad parcial, el depósito de spans no se llena tan rápido y deja espacio para que se muestreen otros endpoints.

```json
{
    distributed_tracing:{
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.5
                }
            }
            partial_granularity: {
                enabled: true,
                type: "compact",  # If capturing AI data set this to "essential"
                remote_parent_sampled: always_on
            }
        }
    }
}
```

## Ejemplo 2: Distribución deficiente de endpoints capturados debido a que algunos endpoints se llaman raramente

La configuración predeterminada utiliza el muestreador adaptativo. Esto significa que captura aproximadamente 10 trazas por minuto. Debido a la probabilidad aleatoria, es más probable que el muestreador adaptativo capture las llamadas a endpoints que ocurren con mayor frecuencia. Esto significa que los endpoints que rara vez se invocan tienen más probabilidades de pasarse por alto y no ser muestreados.

La siguiente configuración solucionará este problema al muestrear cualquier traza entrante que haya sido muestreada por un servicio ascendente solo el 10% del tiempo. ```json
{
    distributed_tracing: {
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.1
                }
            }
        }
    }
}
