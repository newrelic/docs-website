New Relic tiene las siguientes restricciones con la integración AWS para el reenvío de logs al ingerir logs a través de S3 o Firehose:

### Para ingerir logs de AWS desde S3 [#s3-restrictions]

* El tamaño del archivo de log debe ser inferior a 80 MB.
* El tamaño de una sola línea de log debe ser inferior a 8 MB.
* Las líneas de log entre 1 MB y 8 MB se dividirán en varios mensajes y se reenviarán a New Relic.
* New Relic actualmente no admite el análisis de líneas de registro ni de JSON del cliente.
* New Relic actualmente solo admite los formatos de compresión `Gzip` y `Bzip2` para archivos de log. Procesa archivos en otros formatos comprimidos como archivos sin comprimir.

### Para ingerir logs de CloudWatch [#cloudwatch-restrictions]

* New Relic actualmente no admite el análisis de líneas de registro ni de JSON del cliente.
* Actualmente, New Relic solo admite [capas Lambda](https://github.com/newrelic/newrelic-lambda-layers) para analizar la lógica.
