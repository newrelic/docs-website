Cuando se implementa una stack de CloudFormation, se crean activadores en un depósito S3 y filtros en un grupo de registros de CloudWatch. Al eliminar la stack se eliminarán estos activadores y filtros.

Si crea manualmente activadores en un depósito S3 o filtros en grupos de registros de CloudWatch y establece el destino como una función Lambda creada por New Relic, estos también serán compatibles. Sin embargo, no recomendamos este enfoque. Eliminar la stack no eliminará estos activadores o filtros creados manualmente, ya que la stack no tiene licencia para eliminarlos.

Esta guía es importante para garantizar una gestión eficaz de los recursos y evitar que queden activadores o filtros no deseados luego de eliminar una stack .
