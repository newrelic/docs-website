Puede enviar su [registro de Amazon Security Lake](https://docs.aws.amazon.com/security-lake/latest/userguide/internal-sources.html) a New Relic mediante la aplicación Serverless `newrelic-securitylake-s3-processor-LogForwarder`, que se puede implementar fácilmente desde [el repositorio de aplicaciones AWS Serverless](https://serverlessrepo.aws.amazon.com/applications).

Reenviar registros desde Security Lake a New Relic le brindará <InlinePopover type="logs"/>capacidades mejoradas para recopilar, procesar, explorar, consultar y alertar sobre sus datos log .

## Instalar la aplicación sin Serverless [#install-app]

Para instalar la aplicación Serverless para reenviar su registro de Security Lake a New Relic:

1. Asegúrate de tener un

   <InlinePopover type="licenseKey"/>

   .

2. Abra el [repositorio de aplicaciones AWS Serverless](https://serverlessrepo.aws.amazon.com/applications) en su browser.

   <img title="AWS Lambda - Select region" alt="AWS Lambda - Select region" src={serverlessAWSLambdaSelectRegion}/>

3. Busque `newrelic` y marque

   <DoNotTranslate>**Show apps that create custom IAM roles or resource policies**</DoNotTranslate>

   para encontrar `newrelic-securitylake-s3-processor-LogForwarder`.

4. Haga clic en los detalles `newrelic-securitylake-s3-processor-LogForwarder` y haga clic en

   <DoNotTranslate>**Deploy**</DoNotTranslate>

   .

5. Siga las instrucciones de la aplicación para configurar el parámetro necesario:

   * `SecurityLakeSQSArn`
   * `SecurityLakeSubscriberRoleArn`
   * `ExternalID`
   * `NRLicenseKey`

Obtendrá `SecurityLakeSQSArn` y `SecurityLakeSubscriberRoleArn` de su proveedor de Security Lake.

Tenga en cuenta que la región de origen del evento debe coincidir con la región Lambda.
