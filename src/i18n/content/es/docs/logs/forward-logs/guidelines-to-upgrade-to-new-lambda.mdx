Si desea actualizar del antiguo Lambda al nuevo, le recomendamos eliminar el despliegue anterior de New Relic S3 Lambda y CloudWatch Lambda. Alternativamente, puede eliminar los activadores creados previamente antes de implementar el nuevo.

Si elige emplear ambas soluciones, cerciorar de que no haya activadores duplicados para evitar enviar registros duplicados desde el mismo recurso. No recomendamos esto, ya que puede provocar que se alcancen los límites de lectura de registros de S3 y CloudWatch, lo que puede generar inconsistencias en los datos que llegan a New Relic.

<Callout variant="tip">
  Para regresar a la integración AWS para el reenvío de registros, [haga clic aquí](/install/aws-logs/?service=aws_services).
</Callout>
