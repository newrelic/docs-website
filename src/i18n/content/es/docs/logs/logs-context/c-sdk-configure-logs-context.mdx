>
  A partir de abril de 2022, no admitimos la capacidad C SDK. Para obtener más detalles, consulte nuestra [publicación en el foro de soporte](https://discuss.newrelic.com/t/q1-bulk-eol-announcement-fy23/181744).
</Callout>

El logs en el contexto no está disponible para el monitoreo de aplicaciones mediante el SDK de C. En su lugar, puede crear su propia biblioteca de extensiones de registro utilizando la [log API](/docs/logs/log-management/log-api/introduction-log-api/). Asegúrese de que su aplicación tenga habilitado rastreo distribuido, para que pueda comprender rápidamente qué sucede con las solicitudes a medida que viajan a través de los servicios ascendentes y descendentes.

Para habilitar rastreo distribuido para aplicaciones monitoreado por C SDK, [instale](/docs/distributed-tracing/enable-configure/quick-start/) o [actualice](/docs/agents/c-sdk/install-configure/update-your-c-sdk-library/) a la última versión de C SDK. rastreo distribuido requiere [C SDK versión 1.1.0 o superior](/docs/release-notes/agent-release-notes/c-sdk-release-notes/).
