## Problema

No ve datos en la UI después de [instalar un agente New Relic](/docs/agents/manage-apm-agents/installation/install-agent).

## Solución

Debería comenzar a ver datos unos minutos después de instalar un agente de New Relic y generar tráfico para su aplicación. Si no ve datos, puede utilizar la utilidad [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) para identificar automáticamente problemas comunes. Para obtener consejos adicionales sobre resolución de problemas, consulte los documentos específicos del agente.

Además, puedes probar estos pasos de resolución de problemas que se aplican a todos los agentes APM:

<CollapserGroup>
  <Collapser
    id="deleted_apps"
    title="Aplicación eliminada o renombrada en APM"
  >
    Una aplicación debe dejar de informar datos durante al menos una hora antes de que puedas reutilizar ese nombre. También debe volver a conectarse con el recolector de New Relic (reiniciarse) antes de que se acepten nuevos datos.

    La aplicación permanece en la memoria caché del recolector durante una hora antes de vaciarse. Durante ese tiempo se marca como "eliminado", por lo que no se aceptan nuevos datos. Además, los datos están asociados con una aplicación en ejecución que se ha eliminado hasta que se reinicia el agente.

    Para más información, ver:

    * [Nombra tu aplicación](/docs/site/naming-your-application)
    * [Usar varios nombres para una aplicación](/docs/site/using-multiple-names-for-an-app)
  </Collapser>

  <Collapser
    id="data_collection"
    title="No hay conexión con el recolector"
  >
    Su aplicación no se verá afectada si el agente de New Relic no puede conectarse con el [recolector](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#collector). Se siguen recopilando datos y se cargan tan pronto como se restablece la conexión.

    Mientras la red está inactiva o el recolector no está disponible, es posible que vea espacios en los que faltan datos en los gráficos de memoria y CPU de APM. El agente seguirá intentando volver a conectarse y, cuando lo consiga, volverá a ver los datos que aparecen en la UI.

    Durante el tiempo que el agente no puede comunicarse con el recolector, todavía está recopilando datos. Una vez que pueda conectarse nuevamente, cargará los datos y completará el segmento que falta para que no haya confusión sobre si su aplicación estuvo inactiva o simplemente no informó datos. Para ahorrar memoria, los datos se agregarán y promediarán durante el período, por lo que verá barras planas y gráficos durante el período en el que no pudo comunicarse con el recopilador.
  </Collapser>
</CollapserGroup>
