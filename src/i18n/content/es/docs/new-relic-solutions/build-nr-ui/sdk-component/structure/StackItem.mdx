Elemento secundario del componente `<Stack>` .

El `<Stack>` puede alterar la altura, el ancho y el orden del elemento stack según el diseño seleccionado.

### Uso

```js
import { StackItem } from 'nr1'
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>Nodo</h5> <h5>REQUERIDO</h5>
      </td>

      <td>
        Contenido que se mostrará dentro del elemento stack .
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>cadena</h5>
      </td>

      <td>
        Agrega nombres de clase al componente. Debe usar solo con fines de posicionamiento y espaciado.
      </td>
    </tr>

    <tr>
      <td>
        `grow` <h5>booleano</h5>
      </td>

      <td>
        Si `true`, el elemento llena el espacio restante. Para que funcione, necesitarás extender tu stack para ocupar todo el espacio disponible en la dirección de tus artículos. Esto se puede lograr usando los accesorios `fullWidth` o `fullHeight` de la stack, dependiendo de su dirección.
      </td>
    </tr>

    <tr>
      <td>
        `shrink` <h5>booleano</h5>
      </td>

      <td>
        Si `true`, el elemento se reduce si es necesario.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objeto</h5>
      </td>

      <td>
        Estilo en línea para un estilo personalizado.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>cadena</h5>
      </td>

      <td>
        Agrega un atributo `data-test-id` . Úselo para apuntar al componente en pruebas unitarias y E2E. Para que una identificación de prueba sea válida, antepóngale su identificación de nerdpack, seguido de un punto. Por ejemplo, `my-nerdpack.some-element`.

        **Nota:** Es posible que no veas el atributo `data-test-id` ya que se eliminan del DOM. Para depurarlos, pasa un parámetro de consulta `e2e-test` a la URL.
      </td>
    </tr>
