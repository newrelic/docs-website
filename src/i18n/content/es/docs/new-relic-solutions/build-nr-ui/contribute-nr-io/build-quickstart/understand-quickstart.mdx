Antes de continuar con la creación de un inicio rápido, debe comprender su estructura de directorios.

Para comenzar, navegue hasta el repositorio GitHub de inicios rápidos de New Relic y bifurquelo. Luego, clona tu propio repositorio en tu máquina local usando `git clone` y ábrelo en el IDE de tu elección.

En la raíz de su proyecto clonado, los directorios dentro de la carpeta `\_template` reflejan los directorios en el nivel superior del repositorio y contienen las plantillas para los respectivos bloques de construcción de observabilidad en el inicio rápido.

Con la reciente actualización de la estructura de los inicios rápidos, el directorio `\_template` ahora tiene la siguiente estructura:

```
alert-policies/
    example-alert-policy/
        baseline-alert.yml
        static-alert.yml

dashboards/
    example-dashboard/
        example-dashboard.json
        example-dashboard.png

data-sources/
    example-data-source/
        config.yml
        logo.svg

quickstarts/
    example-quickstart/
        config.yml
        logo.svg
```

Cada uno de estos directorios contiene plantillas que puede emplear para crear entidades para su inicio rápido. La estructura de las nuevas contribuciones debe coincidir con la estructura que se ve aquí. Consulte los dashboards y alertas asociados en los inicios rápidos `config.yml`.

Para crear un inicio rápido, primero debe agregar su entidad como dashboard y alertas al inicio rápido.
