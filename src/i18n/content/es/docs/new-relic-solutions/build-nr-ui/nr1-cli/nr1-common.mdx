## nr1 autocompletar [#nr1-autocomplete]

De forma predeterminada, el comando muestra las instrucciones de autocompletar para `zsh`. Si desea instrucciones para bash, ejecute `nr1 autocomplete bash`.

### Uso

`$ nr1 autocomplete`

### Argumentos

<table>
  <tbody>
    <tr>
      <td>
        `SHELL`
      </td>

      <td>
        El tipo de shell para el que desea recibir instrucciones.
      </td>
    </tr>
  </tbody>
</table>

### Opciones

<table>
  <tbody>
    <tr>
      <td>
        `-r, --refresh-cache`
      </td>

      <td>
        Actualiza el caché e ignora las instrucciones que se muestran.
      </td>
    </tr>
  </tbody>
</table>

### Ejemplos

* `$ nr1 autocomplete`
* `$ nr1 autocomplete zsh`
* `$ nr1 autocomplete bash`
* `$ nr1 autocomplete --refresh-cache`

## nr1 crear [#nr1-create]

Crea un nuevo componente a partir de nuestra plantilla, ya sea un Nerdpack, Nerdlet, pícher o catálogo. La CLI lo guiará a través de este proceso.

Para obtener más información sobre Nerdpacks y su estructura de archivos, consulte [Estructura de archivos de Nerdpack](/docs/new-relic-solutions/build-nr-ui/nerdpack-file-structure). Para obtener más información sobre cómo configurar sus Nerdpacks, consulte nuestros [comandos CLI de Nerdpack](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-nerdpack).

### Uso

`$ nr1 create`

### Opciones

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        Si está presente, anula los archivos existentes sin preguntar.
      </td>
    </tr>

    <tr>
      <td>
        `-n, --name=NAME`
      </td>

      <td>
        Nombra el componente.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --type=TYPE`
      </td>

      <td>
        Especifica el tipo de componente.
      </td>
    </tr>

    <tr>
      <td>
        `--path=PATH`
      </td>

      <td>
        La ruta al componente.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        El perfil de autenticación que desea emplear.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Agrega información adicional a la salida.
      </td>
    </tr>
  </tbody>
</table>

## ayuda nr1 [#nr1-help]

Muestra todos los comandos nr1 de forma predeterminada. Para obtener detalles sobre un comando específico, ejecute `nr1 help COMMAND_NAME`.

### Uso

`$ nr1 help`

### Argumentos

<table>
  <tbody>
    <tr>
      <td>
        `COMMAND_NAME`
      </td>

      <td>
        El nombre de un comando en individuo.
      </td>
    </tr>
  </tbody>
</table>

### Ejemplos

* `$ nr1 help`
* `$ nr1 help nerdpack`
* `$ nr1 help nerdpack:deploy`

## nr1 NRQL [#nr1-nrql]

Obtiene datos de la base de datos mediante una consulta NRQL .

Para obtener más información sobre NRQL y cómo usarlo, consulte nuestros [documentos NRQL](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/).

### Uso

`$ nr1 nrql OPTION ...`

### Opciones

<table>
  <tbody>
    <tr>
      <td>
        `-a, --account=ACCOUNT`
      </td>

      <td>
        Esto es **requerido**. El ID de la cuenta de usuario.
      </td>
    </tr>

    <tr>
      <td>
        `-q, --query=QUERY`
      </td>

      <td>
        Esto es **requerido**. La consulta NRQL que se ejecutará.
      </td>
    </tr>

    <tr>
      <td>
        `-u, --ugly`
      </td>

      <td>
        Muestra el contenido sin pestañas ni espacios.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        El perfil de autenticación que desea emplear.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Agrega información adicional a la salida.
      </td>
    </tr>
  </tbody>
</table>

## perfiles nr1 [#nr1-profiles]

Muestra una lista de comandos que puede emplear para gestionar sus perfiles. Ejecute `nr1 help profiles:COMMAND` para obtener más información sobre sus usos específicos. Puede tener más de un perfil, lo cual resulta útil para ejecutar comandos en varias cuentas de New Relic.

Para obtener más información sobre cómo configurar perfiles, consulte [nuestro taller de Github](https://github.com/newrelic/nr1-workshop/blob/master/lab-cli/INSTRUCTIONS.md).

### Uso

`$ nr1 profiles:COMMAND`

### Comandos

<table>
  <tbody>
    <tr>
      <td>
        `profiles:add`
      </td>

      <td>
        Agrega un nuevo perfil al llavero de su perfil.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:default`
      </td>

      <td>
        Elige qué perfil debe ser predeterminado.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:list`
      </td>

      <td>
        Enumera los perfiles en su llavero.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:remove`
      </td>

      <td>
        Elimina un perfil de su llavero.
      </td>
    </tr>
  </tbody>
</table>

### Uso

`$ nr1 update`

### Argumentos

<table>
  <tbody>
    <tr>
      <td>
        `CHANNEL`
      </td>

      <td>

      </td>
    </tr>
  </tbody>
</table>

### Ejemplos

* `$ nr1 update`
