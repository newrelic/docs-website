## `nr1 plugins:install` [#nr1-pluginsinstall]

### Instalar un complemento [#nr1-pluginsinstall-install]

Instala un complemento en la CLI. Puede instalar el complemento desde npm o una URL de Git.

Tenga en cuenta que la instalación de un complemento anulará el complemento principal.

Por ejemplo, si tiene un complemento principal que tiene un comando `hello` , la instalación de un complemento con un comando `hello` anulará la implementación del complemento principal. Esto es útil si desea actualizar la funcionalidad principal del complemento sin parchear ni actualizar toda la CLI.

### Uso [#nr1-pluginsinstall-usage]

`$ nr1 plugins:install PLUGIN`

### Argumentos [#nr1-pluginsinstall-arguments]

* `PLUGIN`: el nombre, ruta o URL del complemento que desea instalar. Esto es **requerido** .

### Opciones [#nr1-pluginsinstall-options]

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        Ejecuta `yarn install --force`. Esto obliga a volver a descargar todos los paquetes del complemento.
      </td>
    </tr>

    <tr>
      <td>
        `-h, --help`
      </td>

      <td>
        Muestra ayuda CLI.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Agrega información adicional a la salida.
      </td>
    </tr>
  </tbody>
</table>

### Ejemplos [#nr1-pluginsinstall-examples]

* `$ nr1 plugins:install myplugin`
* `$ nr1 plugins:install https://github.com/someuser/someplugin`
* `$ nr1 plugins:install someuser/someplugin`

### Alias [#nr1-pluginsinstall-aliases]

* `$ nr1 plugins:add`

## `nr1 plugins:link` [#nr1-pluginslink]

### Vincula tu complemento [#nr1-pluginslink-link]

Vincula un complemento local a la CLI para su desarrollo.

Tenga en cuenta que vincular un complemento anulará el complemento instalado por el usuario o el complemento principal.

Por ejemplo, si tiene un complemento principal o instalado por el usuario que tiene un comando `hello` , vincular un complemento con un comando `hello` anulará la implementación del complemento principal o instalado por el usuario. Esto es útil para el trabajo de desarrollo.

### Uso [#nr1-pluginslink-usage]

`$ nr1 plugins:link PLUGIN`

### Argumentos [#nr1-pluginslink-arguments]

* `PLUGIN`: el nombre, ruta o URL del complemento que desea vincular. Esto es **requerido** .

### Opciones [#nr1-pluginslink-options]

<table>
  <tbody>
    <tr>
      <td>
        `-h, --help`
      </td>

      <td>
        Muestra ayuda CLI.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Agrega información adicional a la salida.
      </td>
    </tr>
  </tbody>
</table>

### Ejemplos [#nr1-pluginslink-examples]

* `$ nr1 plugins:link myplugin`
* `$ nr1 plugins:link someuser/someplugin`
