En la última lección, terminaste la aplicación Prueba A/B que estuviste creando a lo largo de este curso. Ahora es el momento de prepararlo para su publicación. Cuando publica su aplicación en el catálogo de aplicaciones New Relic , el usuario puede verla y suscribir. Puedes ayudar a tu usuario mostrándole y diciéndole qué hace la aplicación, cómo usarla y más.

## Crear información del catálogo. [#create]

Para proporcionar información al catálogo sobre su aplicación, debe crear el directorio `catalog` en su Nerdpack.

<Steps>
  <Step>
    Cambie al directorio _describe-app/ab-test_ del [repositorio de trabajos del curso](https://github.com/newrelic-experimental/nru-programmability-course):

    ```sh
    cd nru-programmability-course/describe-app/ab-test
    ```
  </Step>

  <Step>
    Cree los directorios _del catálogo_ :

    ```sh
    nr1 create --type catalog
    ```

    Esto crea un directorio raíz `catalog` con archivos de plantilla para ingresar información personalizada sobre su aplicación y un directorio `catalog` para cada elemento de Nerdpack donde se puede almacenar la captura de pantalla.
  </Step>

  <Step>
    ## Documentación [#documentation]

    Esta aplicación combina y presenta datos de New Relic y Nerdsletter API para que podamos tomar una decisión informada sobre qué versión de diseño da como resultado una subscripción al boletín informativo de mayor calidad.

    También presenta un botón para finalizar la prueba, que almacena la fecha y las descripciones de la versión en el almacenamiento local.

    ¡También podemos usar esta aplicación para futuras Pruebas A/B!

    Para este curso, mantendrá la documentación concisa simplemente describiendo lo que hace la aplicación. En tus propios proyectos, también puedes incluir instrucciones para contribuir al proyecto en un contexto de código abierto, instalar la dependencia de la aplicación o cualquier otra cosa que pueda ayudar a un usuario a navegar por las complejidades de tu aplicación o código.
  </Step>

  <Step>
    Actualización `catalog/config.json`:

    ```json
    {
      "tagline": "Win @ newsletter subscriptions",
      "details": "Display test data for our newsletter subscription A/B test",
      "categoryTerms": ["browser agent"],
      "keywords": ["a/b test"],
      "repository": "https://github.com/newrelic-experimental/nru-programmability-course",
      "whatsNew": "Initial release! Includes:\n- A variety of charts for understanding the test results\n- An end test button for storing some test data in a table",
      "support": {
        "email": {
          "address": ""
        },
        "issues": {
          "url": ""
        },
        "community": {
          "url": ""
        }
      }
    }
    ```

    Aquí, especificó un lema, detalles del proyecto, la categoría de observabilidad instantánea en la que colocar la aplicación, palabras clave de búsqueda, el repositorio de código fuente y una breve lista de todo lo incluido en el lanzamiento inicial. En otros proyectos, también puedes agregar un email de soporte o enlaces de soporte.
  </Step>

  <Step>
    Almacene [esta captura de pantalla](https://github.com/newrelic-experimental/nru-programmability-course/raw/main/publish/ab-test/catalog/screenshots/screenshot.webp) en los dos directorios siguientes:

    * `nru-programmability-course/describe-app/ab-test/catalog/screenshots`
    * `nru-programmability-course/describe-app/ab-test/nerdlets/ab-test-nerdlet/catalog/screenshots`

      Podrás ver un carrusel de captura de pantalla de usuario en la página **Overview** de tu aplicación en el catálogo.
  </Step>

  <Step>
    Almacene el siguiente icono como `icon.png` en `nru-programmability-course/describe-app/ab-test`.

    Esto actuará como el ícono del catálogo de la aplicación en la página **Overview** .
  </Step>
</Steps>

Ahora agregó documentación, captura de pantalla, metadatos y un ícono a su aplicación para que el usuario pueda entender qué hace y por qué podría necesitarla. Por ahora, todos estos archivos son locales y deben enviar al catálogo. Pero antes de que puedas hacer eso, tu aplicación debe estar publicada en el catálogo.
