La observabilidad empresarial es el arte de capturar, medir y gestionar la interacción en vivo entre la telemetría de TI y el análisis empresarial.

A menudo, la estrategia de observabilidad se implementa en silos, lo que significa que conectar los puntos con el negocio puede ser reactivo, una ocurrencia tardía y extremadamente difícil. La observabilidad empresarial consiste en alinear activamente las iniciativas clave con el rendimiento de los sistemas, las aplicaciones y los procesos.

Al implementar prácticas de observabilidad empresarial, la organización puede tomar decisiones informadas que impacten todas sus operaciones y logren ganancias financieras significativas.

## Próximos pasos [#next-steps]

Ofrecemos muchas soluciones para ayudarle con su práctica de observabilidad empresarial. Estas son algunas de las principales estrategias utilizadas por nuestros clientes más importantes.

### Informar datos personalizados [#custom-data]

Reportamos muchos datos útiles de forma predeterminada, pero para aprovechar al máximo el uso de New Relic, querrás reportar datos personalizados que contengan información que sea única y significativa para tu negocio.

Antes de agregar datos personalizados, debe elaborar un plan de cómo funcionará, para asegurarse de que será una implementación eficiente y organizada (y que no creará demasiados datos "ruidosos").

Los métodos para agregar datos personalizados incluyen:

* Configurando su agente e integración existentes
* Usando API del agente
* Usando nuestras API métrica, evento, log y traza para ingresar cualquier dato

Para obtener sugerencias para agregar datos personalizados, consulte [Datos personalizados](/docs/data-apis/custom-data/get-custom-data-from-any-source).

### Visualizaciones UI personalizadas [#visualizations]

Así como puede personalizar los datos que ingresa, puede personalizar las visualizaciones de esos datos para que coincidan con su estructura comercial. Ejemplos de esto:

* Crear [tableros personalizados](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) que sean monitoreo de IA para resolver problemas comerciales específicos.
* Cree [carga de trabajo](/docs/new-relic-solutions/new-relic-one/workloads/use-workloads), que son agrupaciones importantes para el negocio de su entidad de monitor
