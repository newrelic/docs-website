Nuestra característica de carga de trabajo te brinda la posibilidad de agrupar y monitor entidades de la forma que desees. Esto le brinda el poder de organizar sus telemetry data de la manera que tenga más sentido para su negocio.

## ¿Qué es una carga de trabajo? [#definition]

New Relic monitorea una amplia gama de entidades y datos, desde aplicaciones del lado del cliente y API de backend, hasta la infraestructura subyacente. Para darle sentido a este gran conjunto de datos, le brindamos la capacidad de crear y monitor carga de trabajo.

Nuestra característica <DNT>**workloads**</DNT> te permite agrupar y monitor entidades de la forma que tenga sentido para ti. Por ejemplo, podría crear una carga de trabajo para una unidad de negocio o equipo específico. Nuestra UI de cargas de trabajo proporciona una vista agregada del estado y la actividad de la entidad en esa carga de trabajo. De este modo, podrá comprender mejor cómo funciona su lógica empresarial, desde los servicios frontend hasta los servicios backend, en toda su stack.

Una carga de trabajo puede incluir:

* Cualquier entidad de monitoreo de New Relic, incluidas aplicaciones, aplicaciones browser , aplicaciones móviles, base de datos y hosts.
* [dashboard](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards).
* Otra carga de trabajo: esto es útil para equipos complejos que necesitan dividir y superponer cargas de trabajo.

A continuación se muestran algunos ejemplos de cosas que una organización podría tener en una carga de trabajo:

* Una aplicación sin servidor que incluye una puerta de enlace API, algunas funciones sin servidor y una base de datos y almacenamiento administrados.
* Una aplicación browser y las API de backend que la admiten.
* Una colección de microservicios Java y la infraestructura en la que se ejecutan.
