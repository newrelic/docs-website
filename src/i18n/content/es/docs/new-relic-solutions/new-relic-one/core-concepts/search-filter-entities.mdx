La barra de filtro de entidad le permite filtrar hasta la [entidad](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic) del monitor que desea examinar y le permite guardar filtros para su uso posterior.

## Encuentra la barra de filtro [#find]

El filtro de entidad se encuentra en la parte superior de muchas páginas UI de New Relic. Es visible en la vista <DNT>**All entities**</DNT>, que es lo primero que ve cuando [carga New Relic](https://one.newrelic.com), y es visible en nuestra UI de APM, nuestraUI <InlinePopover type="browser"/>, nuestra UI de infraestructura y más.

## Consejos para usar la barra de filtros [#tips]

Algunos consejos para usar la barra de filtros:

* Escriba

  <DNT>
    **string**
  </DNT>

  de caracteres y/o números (por ejemplo, un entorno) para buscar cualquier entidad que tenga esta cadena en su nombre o ID. Al escribir, la UI sugiere elementos que coinciden con la cadena que estás ingresando, para que puedas seleccionar uno de ellos en el menú desplegable. También puede filtrar por el nombre de

  <DNT>
    **entity**
  </DNT>

  ,

  <DNT>
    **entity type**
  </DNT>

  ,

  <DNT>
    **account ID**
  </DNT>

  ,

  <DNT>
    **environment**
  </DNT>

  o

  <DNT>
    [**tag**](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor)
  </DNT>

  . Los parámetros de selección, una vez creados, tienen un fondo azul.

* Si filtra utilizando una cadena, el filtro tendrá el operador `=` .

* Si filtra usando el menú desplegable, una vez que se ingresa el primer elemento del elemento buscado (clave de etiqueta o atributo), debe seleccionar un operador (`=`, `!=`, `LIKE`, `NOT_LIKE`, `IN`, `NOT IN`) para su filtro.

* Una vez seleccionado el operador, complete el filtro seleccionando el valor para completar el elemento de búsqueda.

Para agregar más filtros, primero debe seleccionar un operador, `AND` o `OR.` Consejos sobre cómo usarlos:

* Utilice `AND` para indicar que desea restringir la selección eliminando entidad de la lista de resultados. También puede utilizar el operador AND para agregar condiciones que deben cumplirse en la lista de resultados, por ejemplo,

  <DNT>
    **entityType = Services `AND` location = APAC**
  </DNT>

  .

  * Tenga en cuenta que

    <DNT>
      **entityType = Services `AND` entityType = Hosts**
    </DNT>

    no devuelve ningún resultado, ya que las entidades solo pueden tener un tipo y ninguna entidad coincidiría con esta condición.

* Utilice `OR` para agregar más entidad a la selección. Por ejemplo, el filtro

  <DNT>
    **entityType = Services `OR` entityType = Hosts**
  </DNT>

  devuelve cada entidad a la que tiene acceso que sea del tipo `Services` o `Hosts`.

Si su filtro es muy complejo, intente utilizar [el generador de consultas NRQL](https://one.newrelic.com/data-exploration/query-builder). Para obtener más información sobre la consulta NRQL, consulte [Cómo utilizar NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-how-nrql-works).
