Suscríbete a un Nerdpack para usarlo en la plataforma New Relic. Dar de baja de Nerdpacks que ya no empleas.

## Actualice su CLI

Antes de ejecutar cualquier comando, cerciorar de tener la última versión de la CLI:

```shell
nr1 update
```

## Comprueba tus licencias

Para suscribir a un Nerdpack, necesita:

* Acceso a la cuenta que lo publicó
* Los [licencias necesarias para gestionar Nerdpacks](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps)

## Publica tu Nerdpack

Debes publicar los Nerdpacks que crees antes de poder suscribirte a ellos:

1. [Publica tu Nerdpack](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish)
2. [Etiqueta tu Nerdpack](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) si está publicado y aún no tiene etiqueta

## Suscríbete a un Nerdpack

Suscribir a un Nerdpack publicado, ya sea con la UI sitio web o la CLI `nr1`, para usarlo en su cuenta. La subscripción le permite seleccionar la versión de etiqueta que desea emplear, así como la cuenta con la que desea suscribir. Esto te permite controlar qué Nerdpacks usas en tu cuenta.

<Callout variant="tip">
  Cuando la etiqueta a la que te suscribiste recibe una actualización, New Relic actualiza tu Nerdpack automáticamente.
</Callout>

<Steps>
  <Step>
    Desde la página de inicio de [New Relic](https://one.newrelic.com) , vaya a <DNT>**+ Integrations & Agents**</DNT> en la barra de navegación superior.
  </Step>

  <Step>
    Encuentra tu Nerdpack publicado usando el filtro de aplicaciones o la barra de búsqueda. Todos los Nerdpacks publicados se muestran aquí. Sin embargo, es posible que el que estás buscando esté filtrado. Ajusta el filtro para encontrar el Nerdpack que estás buscando.
  </Step>

  <Step>
    Selecciona el Nerdpack que deseas agregar a tu cuenta de New Relic.
  </Step>

  <Step>
    Haz clic en **Add this app**. Este botón muestra **Manage access** si ya estás suscrito al Nerdpack.
  </Step>

  <Step>
    Seleccione la cuenta a la que suscribir y la versión de la etiqueta a la que suscribir. Luego, actualice su(s) cuenta(s).

    <Callout variant="tip">
      En la UI sitio web y en algunos lugares de la CLI, nos referimos a **tags** como **channels**. Esta es una reliquia de una implementación anterior del control de versiones de Nerdpack. Haciendo caso omiso de los detalles históricos y de implementación, considérelos sinónimos.
    </Callout>

    Ahora usted y los afiliados a las cuentas a las que se suscribió pueden lanzar Nerdpack desde New Relic.
  </Step>

  <Step>
    Si te suscribiste a una aplicación, encuéntrala en **Your apps**. Si se suscribió a una visualización, búsquela en **Custom visualizations**.
  </Step>
</Steps>

## Suscríbete desde la línea de comando

Si publicó su propio Nerdpack, puede que sea más rápido suscribir a su Nerdpack con la CLI `nr1` .

<Steps>
  <Step>
    Navega al directorio raíz de tu Nerdpack:

    ```shell
    cd my-awesome-nerdpack
    ```

    Estas instrucciones se refieren a tu Nerdpack como `my-awesome-nerdpack`.
  </Step>

  <Step>
    Cerciórate de que tu Nerdpack fue publicado y tenga la etiqueta:

    ```shell
    nr1 nerdpack:info
    [output] Id: {purple}ab123c45-678d-9012-efg3-45hi6jkl7890
    [output] Region: {muted}us
    [output] Account ID: 1234567
    [output] Local version: 1.0.0
    [output] Subscription Model: OWNER_AND_ALLOWED
    [output] Version Count: 1
    [output]
    [output] Version  Date               Tags
    [output] -------  -----------------  ------
    [output] 1.0.0    {muted}a few seconds ago  {green}STABLE
    ```

    Deberías ver al menos una versión [publicada](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish) con una [etiqueta](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag). No tiene que ser `STABLE`. Puede suscribir a cualquier versión de etiqueta.
  </Step>

  <Step>
    Suscríbete a tu Nerdpack:

    ```shell
    nr1 nerdpack:subscribe --channel=STABLE
    [output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}STABLE{normal} channel
    ```

    Aquí, te suscribiste a la etiqueta `STABLE` . También puede suscribir a las etiquetas `DEV` y `BETA` desde la CLI, pero el comando de subscripción no permite etiquetas arbitrarias. Para suscribir a cualquier otra versión de la etiqueta, debe emplear la UI sitio web.

    De manera similar a la UI sitio web, el término **channel** `nr1 nerdpack:subscribe` es una referencia a una implementación anterior de las versiones de Nerdpack.

    <Callout
      variant="tip"
      title="Detalle técnico"
    >
      `nr1 nerdpack:subscribe` es un alias para `nr1 subscription:set`.

      Lea la documentación [`nr1 subscription:set`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionset) para obtener más información. O vea la página de ayuda `nr1` :

      ```shell
      nr1 subscription:set --help
      ```
    </Callout>
  </Step>
</Steps>

## Dar de baja de un Nerdpack

Cancelar la subscripción a un Nerdpack para eliminarlo de tu cuenta.

<Steps>
  <Step>
    Desde [New Relic](https://one.newrelic.com), vaya a <DNT>**Apps**</DNT> en la barra de navegación superior.
  </Step>

  <Step>
    Haz clic en el ícono "i" al lado del Nerdpack al que estás suscrito. Esto abre la página de detalles.
  </Step>

  <Step>
    Haga clic en **Manage access**.
  </Step>

  <Step>
    Deselecciona las cuentas de las que deseas cancelar la subscripción. Luego, actualice esas cuentas.
  </Step>
</Steps>

## Dar de baja usando la línea de comando

Cancele rápidamente la subscripción de su perfil CLI de Nerdpacks.

<Steps>
  <Step>
    Navega al directorio raíz de tu Nerdpack:

    ```shell
    cd my-awesome-nerdpack
    ```
  </Step>

  <Step>
    Ver la subscripción de tu perfil:

    ```shell
    nr1 subscription:list
    [output] {muted}Listing subscribed deployed nerdpacks on your account:
    [output] Version  UUID                                  Name              Date         User
    [output] -------  ------------------------------------  ----------------  -----------  -------
    [output] 0.6.4    {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal}  Account Maturity  {muted}a month ago{normal}  2774472
    ```
  </Step>

  <Step>
    Dar de baja del Nerdpack con el UUID:

    ```shell
    nr1 nerdpack:unsubscribe --nerdpack-id=384de6e5-8e57-4d56-9d13-299ee7ae8641
    [output] Unsubscribed account {green}1234567{normal} from the nerdpack {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal} on the {green}STABLE{normal} channel.
    ```

    <Callout variant="tip">
      `nr1 nerdpack:unsubscribe` es un alias para `nr1 subscription:unset`.

      Lea la documentación [`nr1 subscription:unset`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionunset) para obtener más información. O vea la página de ayuda `nr1` :

      ```sh
      nr1 subscription:unset --help
      ```
    </Callout>
  </Step>
</Steps>

## Resuelve problemas al acceder a tu Nerdpack publicado

A veces, cuando te suscribes a un Nerdpack, puedes tener problemas. Ya sea que no pueda suscribir o vea una subscripción duplicada en su cuenta, considere algunas soluciones comunes para resolver estos problemas.

### Etiqueta tu versión Nerdpack

Cuando te suscribes a un Nerdpack, eliges una etiqueta específica. Entonces, si el Nerdpack al que estás intentando suscribirte no tiene una etiqueta, o no tiene la etiqueta a la que te diriges, [etiqueta](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) tu versión de lanzamiento:

```shell
nr1 nerdpack:subscribe --channel=DEV
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}DEV{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
[output]
nr1 nerdpack:tag --tag=DEV
[output] {success}✔  Tagged {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} version {green}1.0.0{normal} as {green}DEV{normal}.
[output]
nr1 nerdpack:subscribe --channel=DEV
[output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}DEV{normal} channel.
```

Si te suscribes sin un argumento `--channel` , el canal predeterminado es `STABLE`, así que cerciórate de tener una versión `STABLE` en este caso:

```shell
nr1 nerdpack:subscribe
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}STABLE{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
```

### Suscríbete con la UIsitio web

Cuando se suscribe con la CLI, pasa un argumento `--channel` para seleccionar una etiqueta. Un **channel** es, a todos los efectos, una **tag**. Este término es una reliquia de una implementación pasada del control de versiones de Nerdpack.

Sin embargo, aunque los términos son sinónimos, el uso de `--channel` está limitado en la CLI. Cuando te suscribes con un `--channel`, solo puedes elegir entre `DEV`, `BETA` y `STABLE`. Entonces, para suscribir a una etiqueta diferente, use la UI sitio web:

```shell
nr1 nerdpack:subscribe --channel=QA
[output] {error}›{normal}   Error: Expected --channel=QA to be one of: {green}DEV{normal}, {green}BETA{normal}, {green}STABLE
[output] {error}›{normal}   See more help with --help
[output] {error}›{normal}   Code: UNKNOWN
```
