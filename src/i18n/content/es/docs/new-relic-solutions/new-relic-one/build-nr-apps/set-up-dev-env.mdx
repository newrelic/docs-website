Si decidió crear una aplicación personalizada o modificar una de nuestras aplicaciones de código abierto, necesita algunas herramientas esenciales:

* La [interfaz de línea de comando (CLI) New Relic ](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==).
* Una clave de API, que obtienes cuando descargas la CLI.

Dependiendo de lo que quieras hacer con tu aplicación, es posible que tengas algunas configuraciones adicionales. Esta guía cubre:

* Cómo descargar la CLI de New Relic para crear o modificar aplicaciones
* Contribuir con contenido a este sitio web.

## Antes de que empieces

Antes de comenzar, le recomendamos leer primero sobre [permisos](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps).

Para comenzar a construir, debes tener:

* Una cuenta [de cuenta de github](https://github.com/) . Si bien no es estrictamente necesaria para crear aplicaciones, una cuenta de GitHub le permite [descargar y personalizar nuestras aplicaciones de código abierto](https://opensource.newrelic.com/explore-projects) y contribuir con un [proyecto de código abierto](https://opensource.newrelic.com/contributing/).
* Una [cuenta de desarrolladores New Relic ](https://newrelic.com/signup/?partner=Developer+Edition). Si aún no tiene una, puede obtener una cuenta de prueba gratis para desarrollar la aplicación New Relic .
* [npm](https://www.npmjs.com/get-npm). Si instaló [Node.js](https://nodejs.org/en/download/), entonces ya tiene `npm`, que se emplea para compartir, reutilizar y actualizar código JavaScript. Esto es necesario para trabajar con componentes de React que son el framework para las aplicaciones New Relic y este sitio web.

### Una nota sobre el apoyo

Desarrollar una aplicación New Relic es lo mismo que desarrollar cualquier aplicación JavaScript/React. Ofrecemos soporte para ayudar con nuestras herramientas de construcción[ (nuestra](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==) [biblioteca](/docs/new-relic-solutions/build-nr-ui/sdk-component/intro-to-sdk) CLI y SDK). Sin embargo, no ofrecemos soporte para preguntas o problemas básicos de codificación de JavaScript o React.

Para preguntas frecuentes comunes sobre la construcción, consulte la [página de Explorers Hub sobre cómo construir en New Relic](https://discuss.newrelic.com/c/build-on-new-relic).

  Emplee la [extensión New Relic VSCode](https://marketplace.visualstudio.com/items?itemName=new-relic.nr1) o el [paquete de extensiones New Relic VSCode](https://marketplace.visualstudio.com/items?itemName=new-relic.new-relic-extension-pack) para crear sus aplicaciones.

## Preparar para crear o modificar aplicaciones

<Steps>
  <Step>
    ### Descargue la CLI y la clave de API

    En la [página de la aplicación Build New Relic ](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==), complete los pasos de **Quick start** para descargar.

    Estos seis pasos de inicio rápido le brindarán una clave de API para usar en el desarrollo de aplicaciones y la CLI New Relic para crear e implementar aplicaciones. Al finalizar el inicio rápido, tendrá un proyecto que consta de lo siguiente:

    * Un **Nerdpack**. El paquete que contiene todos los archivos requeridos por su aplicación. Contiene dos tipos de archivos que puedes personalizar para crear tu aplicación: Nerdlets y el iniciador.
    * Uno o más archivos **Nerdlet** . Una vista o ventana de UI específica. Un Nerdlet es un paquete React JavaScript que incluye un archivo `index.js` , un `stylesheet` y un archivo `config` en formato JSON. Puede contener cualquier funcionalidad JS (gráficos, campos interactivos, información sobre herramientas, etc.).
    * Un archivo `launcher` : esta es la base del iniciador, que se emplea para abrir su aplicación desde New Relic luego de publicarla.
  </Step>

  <Step>
    ### Empezar a construir

    Si estás listo para codificar, `cd` a tu Nerdpack y comienza.

    Si desea obtener más información sobre cómo crear aplicaciones, pruebe estas guías paso a paso:

    * [Construye una aplicación &quot;¡Hola, mundo!&quot;](/docs/new-relic-solutions/tutorials/build-hello-world-app) muestra cómo crear una pequeña aplicación, publicarla en New Relic y compartirla con otros suscribiéndole cuentas.
    * [Las páginas vistas del mapa por región](/docs/new-relic-solutions/tutorials/map-pageviews-region/) lo guían a través de los pasos para crear una de nuestras populares aplicaciones de código abierto. Aprenderá a agregar una consulta personalizada a una aplicación y verla en una tabla, luego agregará esos datos a un mapa.
  </Step>
</Steps>

## Contribuya al sitio `developer.newrelic.com`

Este sitio es de código abierto y nos gustaría recibir su opinión. Crea una solicitud de extracción \[la] si ves un error que sabes cómo solucionar. Envíenos un problema de GitHub si ve algunas lagunas en el contenido en las que desea que trabajemos. O escribe una guía completamente nueva si tienes una que te gustaría compartir. Sigue leyendo para aprender.

<Steps>
  <Step>
    ### Bifurca el repositorio de GitHub `developer-website`

    Bifurcar el repositorio le permite trabajar en su propia copia de los archivos `developer.newrelic.com` y [crear el sitio localmente](https://github.com/newrelic/developer-website#developernewreliccom). También nos permite gestionar más fácilmente las solicitudes de extracción entrantes.

    1. Vaya a la página [del sitio web de desarrolladores](https://github.com/newrelic/developer-website) en GitHub.
    2. Seleccione el botón **Fork** en la parte superior derecha de la página.
    3. Elija la cuenta a la que desea bifurcar y espere unos segundos mientras se crea la bifurcación.

    [Sincronice de manera regular](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/syncing-a-fork) para mantener su bifurcación actualizada con cambios y adiciones a la rama principal.
  </Step>

  <Step>
    ### Realizar una solicitud de característica o documentación

    1. En cualquier página, seleccione el botón **GitHub** en la parte superior de la página.
    2. Seleccione el tipo de cambio que desea.
    3. Complete el formulario de GitHub.
  </Step>

  <Step>
    ### Contribuir con una nueva guía.

    Consulte nuestras [pautas para contribuyentes](https://github.com/newrelic/developer-website/blob/master/CONTRIBUTING.md#creating-new-pages), que lo guiarán a través del proceso.
  </Step>
</Steps>
