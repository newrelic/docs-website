La mayoría de las veces, los comandos CLI `nr1` simplemente funcionarán y no requerirán configuración adicional. Podrás crear Nerdpacks, Nerdlets, pícher y visualizaciones sin problemas. Pero a veces necesitas algo especial para tu configuración.

En esta guía, aprenderá cómo habilitar la configuración avanzada para su Nerdpack.

## Antes de que empieces

Cerciorar de tener la última versión de la CLI `nr1` :

```sh
nr1 update
```

## Amplíe la configuración de su paquete sitio web

Puede emplear nuestro archivo `.extended-webpackrc.js` para ampliar su paquete sitio web para la configuración. Esto le permite agregar sus propios alias, secuencias de comandos y configuración cuando entrega o implementa localmente su aplicación.

## Agregar certificados adicionales

Cuando trabaje con `nr1` detrás de un proxy corporativo, es posible que vea un `RequestError`:

```sh
nr1 create --type nerdpack --name new-nerdpack
[output] {error}RequestError: Error:{normal} self signed certificate in certificate chain
```

Node emplea su propia cadena de certificados. Entonces, si recibe este error, debe agregar su certificado corporativo a las CA "raíces" de Node:

```sh
export NODE_EXTRA_CA_CERTS=[cert]
```
