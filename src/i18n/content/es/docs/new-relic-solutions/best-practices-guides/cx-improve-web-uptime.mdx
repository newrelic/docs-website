Esta guía incluye algunas ideas sobre cómo mejorar el tiempo de actividad web.

## 1. Investigar checks sintéticos

Investigar y resolver controles sintéticos fallidos cuando se produzcan. Consejos sobre cómo hacer esto:

* Consulta varias ubicaciones.
* Configure una alerta para notificarle cuando fallen varias ubicaciones.

## 2. Crear carga de trabajo

Cree cargas de trabajo que correlacionen el monitor Sintético con la aplicación, los servicios y la infraestructura browser . Incluya un vínculo a la carga de trabajo en el runbook de alerta. También puede encontrar la carga de trabajo correlacionada para un Sintético mediante la búsqueda global.

Asegúrese de que <InlinePopover type="alerts"/>esté configurado para cada nivel de su carga de trabajo. De esta manera puedes ver la salud de cada uno de los niveles en una sola vista. Esto le ahorrará tiempo en la resolución de problemas. No es necesario crear una notificación de alerta para cada nivel para beneficiarse de esta vista.

## 3. Investigar primero las interrupciones en el nivel más bajo

Cuando se produce una interrupción, comience a investigar en el nivel más bajo que esté alertando. Por ejemplo, si ve que tiene un problema de infraestructura y un problema de JavaScript, investigue la infraestructura antes de JavaScript a menos que tenga una segunda persona o equipo en quien pueda delegar eso.

Utilice las herramientas que están disponibles para usted para la resolución de problemas:

* Asegúrate de que rastreo distribuido esté habilitado tanto para

  <InlinePopover type="browser"/>

  como para

  <InlinePopover type="apm"/>

  .

* Utilice la browser UI para comprender qué sucede en el nivel de usuario final.

* Utilice Lookout para comprender qué está causando oscilaciones o problemas recurrentes.
