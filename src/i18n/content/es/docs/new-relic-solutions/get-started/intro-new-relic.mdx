¡Bienvenido a la plataforma de observabilidad de New Relic! Nuestra plataforma le brinda una visibilidad profunda de sus sistemas. Le ayuda a resolver problemas que quizás no conocía, independientemente de la tecnología que utilice. Puedes ver cómo funciona New Relic [aquí](#how-it-works) o [comenzar](#get-started) ahora si quieres comenzar de inmediato.

New Relic ofrece muchas formas de aumentar la visibilidad. Esto le permite monitor, solucionar problemas y mejorar el rendimiento digital. A continuación se ofrece una breve descripción general de nuestras capacidades y características más populares.

# Cómo funciona [#how-it-works]

El primer paso hacia la observabilidad requiere recopilar muchos datos de rendimiento, conocidos como telemetry data. Nuestra plataforma de observabilidad recopila datos de sus servicios utilizando agente. Un agente es un archivo o unas pocas líneas de código que recopila datos sobre su servicio y los envía a nuestra plataforma para su análisis. La instrumentación es el proceso de hacer algo medible. agente se integra en un servicio como parte de este proceso. Puedes instrumentar un tanque de gasolina instalando un medidor que mida qué tan lleno o vacío está tu tanque. Usted monitor su tanque; cuando se acerca el vacío, sabes que es hora de volver a llenarlo.

New Relic hace algo similar. Un agente es un archivo o fragmento de código que se adjunta a un servicio en particular durante un procedimiento de instalación. Al igual que un medidor de gas, nuestro agente realiza un seguimiento del rendimiento del servicio mediante monitoreo métrico como tiempo de respuesta y tasa de errores. Al analizar estos datos, puede obtener información valiosa sobre el funcionamiento de su servicio e identificar áreas de mejora. Primero se instala un agente en su entorno con una configuración predeterminada. El proceso de instalación, configuración e instrumentación puede variar. Depende del servicio del que quieras recopilar datos.

Estos son los agentes de New Relic:

* **Agente APM**: Para aplicación del lado del servidor

* **Agente de Browser**: Para la aplicación browser

* **Agente de infraestructura**: Para hosts e integración en el host

* **Agente Mobile**: Para aplicación móvil

Además de estos agentes, también apoyamos la integración de código abierto. Reportan datos de estándares abiertos para instrumentación. Estos estándares incluyen OpenTelemetry y herramientas como DropWizard, Prometheus y más.

# Empezar [#get-started]

<Steps>
  <Step>
    ## Regístrate en New Relic si aún no lo has hecho

    ¿Listo para comenzar? Si aún no lo ha hecho, [regístrese para obtener una cuenta New Relic](https://newrelic.com/signup). Es gratis, para siempre.

    Registrarse toma solo unos minutos. Una vez que tenga una cuenta, puede comenzar a monitorear su aplicación.
  </Step>

  <Step>
    ## Añade tus datos

    Agregue sus datos a New Relic para comenzar a analizarlos y comprenderlos. Nuestra instalación guiada detectará su entorno para que usted sólo tenga que seleccionar los sistemas que desea monitor.

    Haga clic en el botón <DNT>**Guided install**</DNT> . O, si su organización está ubicada en un [centro de datos de la UE](/docs/using-new-relic/welcome-new-relic/get-started/our-eu-us-region-data-centers), haga clic en <DNT>**EU Guided install**</DNT>.

    <ButtonGroup>
      <ButtonLink role="button" to="https://one.newrelic.com/marketplace?state=15321ec0-7cd8-0c04-52bf-7b65778f2e8c" variant="primary">
        Instalación
      </ButtonLink>

      <ButtonLink role="button" to="https://one.eu.newrelic.com/marketplace?state=15321ec0-7cd8-0c04-52bf-7b65778f2e8c" variant="primary">
        Instalación de la UE
      </ButtonLink>
    </ButtonGroup>

    Si desea agregar más datos de otras fuentes, consulte [nuestra página de inicio rápido](https://newrelic.com/instant-observability) para agregar fuentes de datos a su sistema de monitoreo.

    Si prefieres una instalación manual, consulta la [instalación manual de New Relic](/docs/new-relic-solutions/get-started/manual-installation).
  </Step>

  <Step>
    ## Explora tus datos

    [Analice los datos](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/) de todas las aplicaciones conectadas en New Relic para identificar problemas y ver el rendimiento métrico. Examinar estos datos le ayuda a identificar problemas dentro de su aplicación y a tomar decisiones informadas para optimizar el rendimiento. Explore sus datos para obtener visibilidad instantánea de todos sus telemetry data.
  </Step>

  <Step>
    ## Consulta tus datos

    Escriba su primera consulta utilizando nuestro [lenguaje de consultaNew Relic (NRQL)](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/). Puede consultar datos desde su aplicación y comprender información valiosa a partir de sus datos. Con NRQL puedes analizar tus datos para una mejor toma de decisiones. consulte sus datos para obtener información valiosa sobre su aplicación, hosts y actividad crítica para el negocio. Correlacionar el rendimiento métrico con los KPI del negocio para acelerar el tiempo medio de detección (MTTD) y MTTR.
  </Step>

  <Step>
    ## Configurar un dashboard

    Aunque [nuestros inicios rápidos](https://newrelic.com/instant-observability) incluyen un dashboard predefinido para monitor su sistema, usted puede crear y configurar su [dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/). Nuestro dashboard le permite crear y compartir visualizaciones interactivas de cualquier dato que necesite ver desde cualquier lugar de la plataforma New Relic . Utilice el dashboard para proporcionar contexto y visibilidad de lo que está haciendo su sistema en tiempo real.
  </Step>

  <Step>
    ## Configurar alertas

    Aunque [nuestros quickstarts](https://newrelic.com/instant-observability) incluyen alertas predefinidas para monitorear tu sistema, puedes [crear y configurar](/docs/tutorial-create-alerts/create-an-alert/) tus alertas. Utilice alertas para establecer umbrales y recibir notificaciones de cambios en el rendimiento en todo su sistema. Esto le ayudará a anticiparse a los problemas de rendimiento para evitar impactos negativos en sus usuarios. Mantenga a su equipo enfocado y productivo con la gestión inteligente de eventos de alertas.
  </Step>
</Steps>
