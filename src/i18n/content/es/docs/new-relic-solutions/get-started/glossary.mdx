Ya sea que esté considerando New Relic o ya esté utilizando nuestras capacidades, este glosario de terminología común puede resultarle útil. Y si desea comenzar, consulte [Introducción a New Relic](/docs/new-relic-solutions/get-started/intro-new-relic).

<CollapserGroup>
  <Collapser id="account" title="cuenta">
    Una organización New Relic puede tener una o más cuentas. Una cuenta puede considerarse un espacio de trabajo: un espacio para monitor o analizar datos relacionados con un proyecto específico o un equipo específico. Cada cuenta tiene su propio ID de cuenta, y ese ID se utiliza para algunas tareas específicas de la cuenta, como realizar llamadas API. Para obtener más información sobre por qué crearía una cuenta, consulte [Estructura de la cuenta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure).

    La palabra &quot;cuenta&quot; a veces se utiliza en un sentido industrial más general. Por ejemplo, a veces se utiliza para referirse a una organización de New Relic, a un cliente de New Relic o al registro de usuario de un usuario de New Relic. Sin embargo, en términos técnicos, en New Relic una cuenta se refiere a una cuenta con un ID de cuenta.
  </Collapser>

  <Collapser id="account-family" title="familia de cuentas">
    Una organización New Relic puede tener una o más [cuentas](#account).

    Una &quot;familia de cuentas&quot; se refiere a una cuenta padres y las cuentas hijos bajo ella: en otras palabras, un grupo de cuentas relacionadas unidas por facturación.
  </Collapser>

  <Collapser id="account-switcher" title="cambio de cuenta">
    <img title="The account switcher" alt="The account switcher lets you switch between accounts." src="/images/accounts_screenshot-crop_account-switcher.webp" />

    Si tiene acceso a más de una cuenta en una [organización](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) de múltiples cuentas, puede usar el selector de cuentas para cambiar entre cuentas. Está ubicado en la parte superior derecha de la [página de inicio de New Relic](https://one.newrelic.com) y en algunas otras vistas de lista de entidades.

    Para obtener más información sobre los factores que afectan el acceso a las cuentas, consulte [Factores que afectan el acceso](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data).

    Para encontrar la configuración de la cuenta, use el [menú de usuario](#user-menu).
  </Collapser>

  <Collapser id="agent" title="agente">
    Un agente de New Relic es una pieza de software instalable que se integra con múltiples tipos de tecnologías (por ejemplo, marco web, sistema operativo y tipos de base de datos) y reporta datos a New Relic, generalmente en una cadencia específica. Tenemos <InlinePopover type="apm" />agente, un agente de infraestructura, un <InlinePopover type="browser" />agente y un agente de monitoreo de móviles. Las diversas tecnologías con las que se integra nuestro agente son solo algunas de [nuestras muchas integraciones](https://newrelic.com/instant-observability).

    Para más información, ver:

    * [New Relic Observabilidad instantánea](https://newrelic.com/instant-observability)
    * [Instalar agente](/docs/agents/manage-apm-agents/installation/install-agent)
  </Collapser>

  <Collapser id="aggregation-delay" title="retraso de agregación">
    Para nuestra [característica de alerta](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/choose-your-aggregation-method), esto se refiere al tiempo en segundos que se debe esperar a que la ventana de agregación se llene con datos. Requerido cuando se utilizan `aggregation_method` tipos `CADENCE` o `EVENT_FLOW`.
  </Collapser>

  <Collapser id="aggregation-function" title="función de agregación">
    En el contexto de alerta NRQL, puede utilizar [funciones NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions), como `sum()`, `average()` o `latest()` para elegir cómo se deben procesar los puntos de datos en una [ventana de agregación](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/choose-your-aggregation-method) en un único punto de datos. El único punto de datos agregado es lo que pasa por el proceso de evaluación de alerta.
  </Collapser>

  <Collapser id="aggregation-method" title="método de agregación">
    En el contexto de New Relic alerta, agregamos datos en ventanas y necesitamos determinar cuándo termina la ventana actual y comienza la siguiente. [Aggregation\_method](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/choose-your-aggregation-method) es la lógica que nos dice cuándo tenemos todos los datos para una ventana de agregación determinada. Una vez que se cierra la ventana, los datos se agregan en un solo punto y se evalúan según el umbral. Este campo es opcional. Se puede especificar uno de los tres valores siguientes:

    * `EVENT_FLOW`: (Predeterminado) Cada ventana de agregación esperará hasta que comience a ver llegar la marca de tiempo que superó su propia configuración de retraso. Una vez que esto ocurre, los datos se publican. Depende de la marca de tiempo de los datos que llegan, por lo que el tiempo de reloj ya no es relevante. Funciona mejor para fuentes que llegan con frecuencia y con baja dispersión de eventos (alto rendimiento métrico)

    * `CADENCE`: Lógica clásica de New Relic donde cada ventana de evaluación espera exactamente tanto tiempo como la configuración `aggregation_delay` , utilizando el tiempo de reloj como temporizador. Se requiere `aggregation_delay` al usar esta opción. Los datos que lleguen demasiado tarde se descartarán, lo que puede provocar alertas falsas.

    * `EVENT_TIMER`: Cada ventana de agregación tiene un temporizador, configurado en la configuración `aggregation_timer` . El temporizador comienza a funcionar tan pronto como aparece el primer punto de datos para esa ventana de agregación (según la timestamp del punto de datos). El `aggregation_timer` se restablece para cada nuevo punto de datos que llega para esa ventana. Una vez que `aggregation_timer` llega a 0, se publica la ventana de agregación. Ideal para datos dispersos y por lotes, como integración en la nube y log de errores poco frecuentes.
  </Collapser>

  <Collapser id="aggregation_timer" title="temporizador de agregación">
    En el contexto de [alerta New Relic](/docs/alerts-applied-intelligence/overview), este es el tiempo en segundos que se debe esperar después de recibir cada punto de datos, para garantizar que se procese todo el lote. Requerido cuando se utiliza el tipo `EVENT_TIMER` `aggregation_method` .
  </Collapser>

  <Collapser id="aggregation-window" title="ventana de agregación">
    En el contexto de [New Relic <InlinePopover type="alerts" />](/docs/alerts-applied-intelligence/overview), la alerta de transmisión recopila datos en períodos de tiempo específicos. Estas ventanas de tiempo son personalizables.

    Los puntos de datos se recopilan juntos según sus marcas de tiempo y se reportan como un lote. La ventana de agregación personalizable ofrece mayor flexibilidad y menos eventos de alerta falsos al alertar sobre puntos de datos irregulares o menos frecuentes.
  </Collapser>

  <Collapser id="AI" title="AI">
    La IA, o inteligencia artificial, se refiere a sistemas informáticos que pueden comprender, aprender y realizar tareas que normalmente requieren inteligencia humana. En New Relic, cuando decimos IA, nos referimos a la capa de su entorno que emplea un modelo de lenguaje extenso (LLM) para generar una respuesta (o tomar una decisión) basada en el símbolo final del usuario.
  </Collapser>

  <Collapser id="alert" title="alerta">
    Un <DNT>**alert**</DNT> comunica un evento o evento de alerta que el personal designado puede rastrear a través de [alertas](/docs/alerts/new-relic-alerts-beta/getting-started/alerting-new-relic).

    Para obtener una explicación de cómo se relacionan los conceptos básicos de alerta, consulte [Conceptos y flujo de trabajo](/docs/alerts-applied-intelligence/overview/#concepts-terms).
  </Collapser>

  <Collapser id="alert-condition" title="condición de alerta">
    Un <DNT>**alert condition**</DNT> (o <DNT>**condition**</DNT>), identificado por su `condition_id` numérico único, contiene los criterios para crear un evento de alerta. La condición incluye el [umbral](#alert-threshold) establecido para un [intervalo de tiempo de métrica](/docs/data-analysis/metrics/analyze-your-metrics/data-collection-metric-timeslice-event-data#timeslice-data) o una [métrica personalizada](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-custom-metrics-alert-condition) a lo largo del tiempo en un [objetivo](#alert-target) elegido.

    Para obtener una explicación de cómo se relaciona una condición con otros conceptos básicos de alerta, consulte [Conceptos y flujo de trabajo](/docs/alerts-applied-intelligence/overview/#concepts-terms).
  </Collapser>

  <Collapser id="alert-evaluation" title="evaluación de alerta">
    En el contexto de alerta, la transmisión de datos se evalúa en un conjunto de ventanas de agregación para determinar si una condición de alerta se está incumpliendo o se está recuperando.

    El tiempo de la ventana de agregación es el tiempo que recopilaremos datos antes de ejecutar la condición de consulta NRQL. El tiempo de evaluación de compensación es el tiempo que desea que esperemos por los datos tardíos antes de evaluarlos.

    Si una ventana no tiene ningún punto de datos, se trata como un espacio por pérdida de señal.
  </Collapser>

  <Collapser id="alert-policy" title="politica de alertas">
    En el contexto de las alertas, una política de alertas es una colección de una o más [condiciones de alerta](/docs/alerts/new-relic-alerts/defining-conditions/define-alert-conditions), uno o más [canales de notificaciones](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-channels/) y una configuración de [<DNT>**Alert event preference**</DNT>](/docs/alerts/organize-alerts/specify-when-alerts-create-events).

    Una condición contenida en una política puede abrir un evento de alerta dependiendo de la [configuración de preferencias de eventos de alerta](/docs/alerts/organize-alerts/specify-when-alerts-create-events). En este caso, todos los canales asociados a la política generarán notificaciones.

    Para obtener una explicación de cómo se relaciona una política con otros conceptos básicos de alerta, consulte [Conceptos y flujo de trabajo](/docs/alerts-applied-intelligence/overview/#concepts-terms).
  </Collapser>

  <Collapser id="anomaly" title="anomalía">
    Cuando un servicio o aplicación se comporta de manera anormal, la característica de alerta de New Relic lo marca como una anomalía. Puedes encontrar anomalías en los feeds de actividades de New Relic. También puede crear una configuración personalizada para recibir notificaciones o rastrear anomalías en otros tipos de entidades, como hosts y aplicaciones móviles.

    Para más información, ver \[detección de anomalías] (/docs/alerta-inteligencia aplicada/inteligencia aplicada/proactive-detection/proactive-detection-inteligencia aplicada/)
  </Collapser>

  <Collapser id="apdex" title="apdex">
    <DNT>**Apdex**</DNT> es una forma estándar de la industria de medir la satisfacción del usuario con el tiempo de respuesta de una aplicación o servicio. New Relic califica cada respuesta como <DNT>**Satisfied**</DNT>, <DNT>**Tolerated**</DNT> o <DNT>**Frustrated**</DNT> y utiliza estas calificaciones para calcular una puntuación general de satisfacción del usuario.

    Para más información, consulte [Apdex: Medir la satisfacción del usuario](/docs/site/apdex-measuring-user-satisfaction).
  </Collapser>

  <Collapser id="apdex_f" title="apdex_f">
    El tiempo de respuesta por encima del cual una transacción se califica como <DNT>**frustrating**</DNT>. El valor predeterminado es cuatro veces `apdex_t`.

    * Las solicitudes que se completan en menos de `apdex_t` se clasifican como <DNT>**satisfied**</DNT>.

    * Las solicitudes que tardan más de `apdex_t`, pero menos de cuatro veces `apdex_t` (`apdex_f`), son <DNT>**tolerated**</DNT>.

    * Cualquier solicitud que demore más de `apdex_f` se califica como <DNT>**frustrating**</DNT>.

      Para más información, consulte [Apdex: Medir la satisfacción del usuario](/docs/site/apdex-measuring-user-satisfaction).
  </Collapser>

  <Collapser id="apdex_t" title="apdex_t">
    El tiempo de respuesta por encima del cual una transacción se considera <DNT>**tolerable**</DNT>. El valor predeterminado es 0,5 segundos, pero puede [cambiarlo en la configuración de Apdex](/docs/site/changing-your-apdex-settings).

    * Las solicitudes que se completan en menos de `apdex_t` se clasifican como <DNT>**satisfied**</DNT>.

    * Las solicitudes que toman más de `apdex_t` pero menos de `apdex_f` son <DNT>**tolerated**</DNT>.

    * Cualquier solicitud que demore más de `apdex_f` se califica como <DNT>**frustrating**</DNT>.

      Para más información, consulte [Apdex: Medir la satisfacción del usuario](/docs/site/apdex-measuring-user-satisfaction).
  </Collapser>

  <Collapser id="api" title="API (interfaz de programación de aplicaciones)">

  <Collapser id="nr-apm" title="APM">
    New Relic [APM](/docs/apm/new-relic-apm/getting-started/introduction-new-relic-apm) (monitoreo del rendimiento de aplicaciones (APM)) proporciona monitoreo del rendimiento de su aplicación web o no web. <InlinePopover type="apm" />admite aplicaciones que utilizan varios [lenguajes de programación](/docs/agents/manage-apm-agents/installation/install-agent#apm-install).
  </Collapser>

  <Collapser id="application" title="aplicación">
    Para propósitos de New Relic, cualquier programa instrumentado por New Relic.
  </Collapser>

  <Collapser id="application_id" title="ID de la aplicación">
    Algunas soluciones de New Relic asignan a una aplicación de monitor un <DNT>**application ID**</DNT> único, a menudo abreviado como <DNT>**app ID**</DNT>. Cuando está presente, esta identificación está disponible en la UI. También se reporta como atributo y se puede consultar.

    Para saber cómo determinar esto, consulte [Buscar ID de aplicación](/docs/accounts/install-new-relic/account-setup/app-id-other-product-ids).

  <Collapser id="application-name" title="Nombre de la aplicación">
    El nombre que New Relic combina con su <InlinePopover type="licenseKey" />para identificar de forma única una aplicación en particular. Para obtener más información, consulte [Asigne un nombre a su aplicación](/docs/agents/manage-apm-agents/app-naming/name-your-application).
  </Collapser>

  <Collapser id="attribute" title="atributo">
    <DNT>**Attributes**</DNT> son pares de valor principal adjuntos a objetos de datos reportados a New Relic. El atributo agrega detalles y es similar a la etiqueta o etiquetas en otro software SaaS. Puede explorar estos datos [consultando o buscando a través de la UI](/docs/using-new-relic/data/understand-data/query-new-relic-data) o utilizando el [diccionario de datos](/attribute-dictionary).

    Ejemplos:

    * APM informa un evento `Transaction` . Esto incluye datos de tiempo para la transacción en un atributo `duration` , que podría tener un valor de `.002`.

    * Nuestro monitoreo de infraestructura informa un evento `ProcessSample` . Esto incluye una variedad de atributos de uso de CPU, incluido un atributo `cpuSystemPercent` , que podría tener un valor de `.01`.

    * Nuestro SDK de telemetría informa un tipo de datos `Metric` para almacenar métricas, con atributos adjuntos como `metricName` y `newrelic.source`.

      Algunas herramientas de New Relic le permiten informar [atributos personalizados](/docs/apm/other-features/attributes/collecting-custom-attributes) para mejorar su monitoreo.

      Para obtener más información sobre el atributo en APM, consulte [Atributo del agente](/docs/apm/other-features/attributes/agent-attributes).
  </Collapser>

  <Collapser id="collector" title="recolector">
    El componente que recopila datos del [agente](#agent) New Relic que se ejecuta en un servidor de aplicaciones, un dispositivo móvil o browser de usuario final. Mientras el agente está instalado en el servidor de aplicaciones de un usuario, el recopilador está ubicado centralmente en nuestro centro de datos.

    Para comunicarse con el recopilador, el agente debe poder comunicarse con [el dominio y las direcciones IP de New Relic](/docs/apm/new-relic-apm/getting-started/networks). (El dominio o IP exacto depende de la herramienta de monitoreo de New Relic). El recolector recibe e interpreta estos datos y los almacena en una base de datos. Luego, los datos se recuperan y presentan en la [UI de New Relic](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one) y mediante nuestras diversas API REST.
  </Collapser>

  <Collapser id="conditions" title="condiciones">
    Utilice condición de alerta para definir cuándo y por qué se notificará a su equipo si su entidad no está funcionando de la manera deseada. Con las condiciones, usted establece los criterios para las notificaciones sobre tiempos de retraso, errores de código, puntuaciones de Apdex, uso elevado de CPU y más.

    Para obtener más información, consulta [eventos de alerta](/docs/new-relic-solutions/get-started/glossary#alert-event).

  <Collapser id="container" title="contenedor">
    Un contenedor es una unidad estándar de software que contiene todas las piezas de código necesarias y todas sus dependencias para que su aplicación se ejecute sin problemas en una variedad de entornos informáticos.

  <Collapser id="cpm" title="CPM (llamadas por minuto)">
    La cantidad de llamadas que recibe su aplicación cada minuto. Esto suele corresponder al número de páginas vistas o conexiones externas y suele ser lo mismo que [RPM (solicitudes por minuto)](#rpm).

  <Collapser id="cpu_burn" title="Quema de CPU">
    El tiempo consumido por el código menos el tiempo de espera de una transacción. Este es el tiempo realmente dedicado a procesar la transacción. Aparece en la UI de New Relic en la parte superior de la vista de transacciones para el agente que lo proporciona (solo Ruby y PHP).
  </Collapser>

  <Collapser id="custom-attribute" title="atributo personalizado">
    Un par de valor principal agregado a una transacción o evento para obtener información adicional sobre él. Para obtener más información, consulte [atributo personalizado](/docs/apm/other-features/attributes/collecting-custom-attributes).
  </Collapser>

  <Collapser id="custom-dashboard" title="tablero personalizado">
    Un dashboard personalizable con gráficos y tablas que incluye datos de múltiples fuentes de datos de New Relic. Para obtener más información, consulte [el dashboard](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards).
  </Collapser>

  <Collapser id="custom-event" title="evento personalizado">
    Un [evento](#event), en términos de New Relic, es un objeto de datos con [un atributo](#attribute) adjunto. New Relic informa tipos de eventos predeterminados, como `Transaction` y `TransactionError`. También puedes crear tu propio evento. evento puede ser [consulta](/docs/using-new-relic/data/understand-data/query-new-relic-data) y se utilizan en alguna otra característica.

    Puedes generar eventos personalizados con [el agente APM](/docs/insights/new-relic-insights/adding-querying-data/inserting-custom-events-new-relic-apm-agents), el [monitoreo de navegador agente](/docs/insights/insights-data-sources/custom-data/insert-via-new-relic-browser), el [monitoreo de móviles agentes](/docs/insights/insights-data-sources/custom-events/insert-custom-events-attributes-mobile-data), y vía el [evento API](/docs/insights/insights-data-sources/custom-data/insert-custom-events-insights-api). Alternativamente, puede agregar [un atributo personalizado](#custom-attribute) a algún evento New Relic predeterminado existente.
  </Collapser>

  <Collapser id="custom-instrumentation" title="instrumentación personalizada">
    <DNT>**Custom instrumentation**</DNT> le permite extender el monitoreo de New Relic a elementos de código instrumentados que New Relic no instrumenta automáticamente. La instrumentación personalizada es útil cuando su [marco](#framework) no es compatible con New Relic o cuando New Relic no logra captar algún elemento de su programa. También puede utilizar instrumentación personalizada para impedir que una transacción se informe por completo. Para más información, consulte [instrumentación personalizada](/docs/features/custom-instrumentation).
  </Collapser>

  <Collapser id="custom-metric" title="métrica personalizada">
    [datos de intervalo de tiempo de métrica](#metric) que se registran manualmente mediante una llamada API. <DNT>**Custom metrics**</DNT> te permite registrar métricas arbitrarias; por ejemplo, datos de tiempo o recursos informáticos. Todos los nombres métricos personalizados deben tener el prefijo `Custom/`. Para más información, consulte [métrica personalizada](/docs/features/custom-metrics).

    No confundir con [datos de instrumentación personalizada](#custom-instrumentation-data).
  </Collapser>

  <Collapser id="degradation-period" title="periodo de degradación">
    En el contexto de las alertas, cuando una fuente de datos entra en un estado de infracción, comienza un <DNT>**degradation period**</DNT> de tiempo. El período de degradación se establece en el umbral de la condición. Se abrirá un evento de alerta si la fuente permanece en un estado de infracción durante todo el período de degradación. Además:

    * Si la fuente de datos entra en un estado de no incumplimiento antes de que transcurra todo el tiempo, la cuenta regresiva del período de degradación se restablece y no se abre un evento de alerta.
    * Si su umbral de condición de alerta está configurado como `at least once in`, el período de degradación siempre dura un minuto.
  </Collapser>

  <Collapser id="host" title="anfitrión">
    En New Relic, un <DNT>**host**</DNT> significa uno de los siguientes:

    * Un <DNT>**physical machine**</DNT> es un dispositivo basado en hardware con recursos físicos dedicados, incluidos memoria, procesamiento y almacenamiento. Cada máquina tiene su propio sistema operativo en el que se ejecuta la aplicación.

    * Un <DNT>**cloud instance**</DNT> es un tipo de máquina virtual que se ejecuta en la nube pública. En este contexto, la máquina virtual y la instancia de la nube son diferentes de la máquina virtual Java (JVM) y el contenedor.

  <Collapser id="alert-threshold" title="umbral">
    <DNT>**Thresholds**</DNT> son configuraciones de condiciones de alerta que definen un evento de alerta. Los valores de umbral incluyen el valor que una fuente de datos debe superar para activar un evento de alerta y la configuración relacionada con el tiempo que define un evento de alerta; por ejemplo:

    * Pasar un cierto valor durante al menos x minutos

    * Pasar un cierto valor solo una vez en x minutos

      Mientras la fuente de datos supera un determinado valor, comienza un [período de degradación](#degradation-period) . Asimismo, cuando esa fuente de datos deja de pasar un determinado valor, comienza un [período de recuperación](#recovery-period) . La duración de estos dos períodos de tiempo se define en la configuración del umbral de condición de alerta.

      Los umbrales tienen un umbral crítico (rojo) obligatorio y un umbral de advertencia (amarillo) opcional. En la interfaz de usuario, el indicador de [estado de salud](#health-status) de la entidad cambiará a amarillo o rojo cuando se supere un umbral y se abrirá un evento de alerta.

      Para obtener más información, consulte [Definir umbral](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-thresholds-trigger-alert). Para obtener una explicación de cómo umbral se relaciona con otros conceptos básicos de alerta, consulte [Conceptos y flujo de trabajo](/docs/alerts-applied-intelligence/overview/#concepts-terms).
  </Collapser>
