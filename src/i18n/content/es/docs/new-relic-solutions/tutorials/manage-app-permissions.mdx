## Permisos de Nerdpack: [#nerdpack-permissions]

Conozca las [diferencias de tipos de usuario](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) entre cómo los usuarios básicos, los usuarios full platform y los usuarios core pueden usar y administrar Nerdpacks.

### Usuario básico [#basic-users-new-users]

Si es un usuario básico, está limitado en sus capacidades de Nerdpack. Solo puede crear y servir Nerdpacks localmente. Para publicar, etiquetar, suscribirte a y usar tus Nerdpacks, un administrador debe actualizarte a usuario de plataforma completa o usuario Core.

<Callout variant="tip">
  Si eres un usuario básico, generalmente no puedes usar ningún Nerdpacks. Sin embargo, hay algunos Nerdlets que New Relic mantiene que puedes usar. Estos son raros y actualmente no hay forma de distinguirlos en nuestra interfaz sitio web. E incluso si puedes usar un Nerdlet en un Nerdpack, nunca podrás usar visualizaciones personalizadas. Para ello, debe actualizar a usuario de plataforma completa o usuario principal.
</Callout>

### Usuario principal y usuario de plataforma completa [#core-full-platform-users-new]

Si es un usuario core o un usuario de plataforma completa, o tiene el privilegio **&quot;modificar&quot; Nerdpacks**, o no lo tiene.

Se requiere el privilegio de **Nerdpacks &quot;modify&quot;** para publicar, suscribir y etiquetar Nerdpacks. Entonces, si no tiene el privilegio de **Nerdpacks &quot;modify&quot;** , solo puede crear y servir Nerdpacks localmente y usar Nerdpacks a los que sus cuentas ya estén suscritas.

Para publicar, etiquetar o suscribir a su Nerdpack, un administrador debe otorgarle la capacidad de **Nerdpacks &quot;modify&quot;** .

<Callout variant="tip">
  La capacidad de **Nerdpacks &quot;modify&quot;** está incluida tanto en el grupo de **User** como en el de **Admin**, los únicos grupos disponibles de forma predeterminada. Entonces, en la mayoría de los casos, tendrás la capacidad de gestionar Nerdpacks como usuario de plataforma completa o usuario principal. Sin embargo, si eres un usuario de plataforma completa o un usuario principal asignado a un grupo personalizado que no incluye el privilegio de **Nerdpacks &quot;modify&quot;** , no podrás gestionar Nerdpacks.
</Callout>

## Capacidades de cuenta padre/hijo [#parent-child]

Algunas organizaciones New Relic tienen [estructuras de cuentas padre/hijo](/docs/accounts/original-accounts-billing/original-users-roles/parent-child-account-structure). Para estas organizaciones, las cuentas secundarias reportan datos de respaldo a sus cuentas principales. Esta jerarquía organizacional afecta las capacidades de Nerdpack.
