Comprenda los requisitos para gestionar y usar Nerdpacks en New Relic.

## Capacidades de un vistazo [#capabilities]

Su capacidad para gestionar y usar Nerdpacks se ve afectada por su usuario de New Relic:

* Modelo de usuario (modelo original o más nuevo)
* Tipo de usuario
* Roles asignados

Ya sea que esté en el [modelo de usuario original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) o [en nuestro modelo de usuario más nuevo](/docs/accounts/accounts-billing/new-relic-one-user-management/new-relic-one-user-model-understand-user-structure/), revise las tablas a continuación para obtener un resumen de sus capacidades de Nerdpack.

## Licencias de Nerdpack: modelo de usuario más nuevo [#nerdpack-permissions]

Conozca las diferencias entre cómo el usuario básico, el usuario de plataforma completa y el usuario principal en [nuestro modelo de usuario más nuevo](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) pueden usar y gestionar Nerdpacks.

### Usuario básico [#basic-users-new-users]

Si eres un usuario básico de nuestro modelo de usuario más nuevo, tus capacidades de Nerdpack están limitadas. Solo puedes crear y servir Nerdpacks localmente. Para publicar, etiquetar, suscribir y usar sus Nerdpacks, un administrador debe actualizarlo a un usuario de plataforma completo o usuario principal.

<Callout variant="tip">
  Si eres un usuario básico, generalmente no puedes usar ningún Nerdpacks. Sin embargo, hay algunos Nerdlets que New Relic mantiene que puedes usar. Estos son raros y actualmente no hay forma de distinguirlos en nuestra interfaz sitio web. E incluso si puedes usar un Nerdlet en un Nerdpack, nunca podrás usar visualizaciones personalizadas. Para ello, debe actualizar a usuario de plataforma completa o usuario principal.
</Callout>

### Usuario principal y usuario de plataforma completa [#core-full-platform-users-new]

Si es un usuario principal o un usuario de plataforma completa en nuestro modelo de usuario más nuevo, tiene el privilegio de **Nerdpacks "modify"** o no.

Se requiere el privilegio de **Nerdpacks "modify"** para publicar, suscribir y etiquetar Nerdpacks. Entonces, si no tiene el privilegio de **Nerdpacks "modify"** , solo puede crear y servir Nerdpacks localmente y usar Nerdpacks a los que sus cuentas ya estén suscritas.

Para publicar, etiquetar o suscribir a su Nerdpack, un administrador debe otorgarle la capacidad de **Nerdpacks "modify"** .

<Callout variant="tip">
  La capacidad de **Nerdpacks "modify"** está incluida tanto en el grupo de **User** como en el de **Admin**, los únicos grupos disponibles de forma predeterminada. Entonces, en la mayoría de los casos, tendrás la capacidad de gestionar Nerdpacks como usuario de plataforma completa o usuario principal. Sin embargo, si eres un usuario de plataforma completa o un usuario principal asignado a un grupo personalizado que no incluye el privilegio de **Nerdpacks "modify"** , no podrás gestionar Nerdpacks.
</Callout>

## Licencias de Nerdpack: modelo de usuario original [#nerdpack-permissions-original]

Conozca las diferencias entre cómo el usuario básico, el usuario completo, el usuario principal y los administradores en nuestro [modelo de usuario original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) pueden usar y gestionar Nerdpacks.

### Usuario básico [#basic-users-original]

Si eres un usuario básico en nuestro modelo de usuario original, tus capacidades de Nerdpack están limitadas. Solo puedes crear y servir Nerdpacks localmente. Para publicar, etiquetar, suscribir y emplear sus Nerdpacks, un administrador debe actualizarlo a usuario completo.

<Callout variant="tip">
  Si eres un usuario básico, generalmente no puedes usar ningún Nerdpacks. Sin embargo, hay algunos Nerdlets que New Relic mantiene que puedes usar. Estos son raros y actualmente no hay forma de distinguirlos en nuestra interfaz sitio web. E incluso si puedes usar un Nerdlet en un Nerdpack, nunca podrás usar visualizaciones personalizadas. Para ello, debe actualizar a usuario de plataforma principal o completa.
</Callout>

### Usuario principal y usuario de plataforma completa [#core-full-platform-users-original]

Si es un usuario de plataforma principal o completa en nuestro modelo de usuario original, tiene la función adicional **Nerdpack Manager** o no.

Se requiere el rol de **Nerdpack Manager** para publicar, suscribir y etiquetar Nerdpacks. Entonces, si no tiene la función **Nerdpack Manager** , solo puede crear y servir Nerdpacks localmente y usar Nerdpacks a los que sus cuentas ya estén suscritas.

Para publicar, etiquetar o suscribir a un Nerdpack, un administrador debe actualizarlo a usuario principal o usuario de plataforma completo u otorgarle la función complementaria **Nerdpack Manager** .

### Propietario/usuario administrador [#owner-admin-users]

Si es propietario o usuario administrador, puede realizar cualquiera de las capacidades de Nerdpack. Puede crear, servir, publicar, etiquetar, suscribir y emplear todos y cada uno de los elementos de Nerdpack, tanto Nerdlets como visualizaciones. Esto incluye Nerdpacks creados por New Relic o por una de sus cuentas.

## Capacidades de cuenta padre/hijo [#parent-child]

Algunas organizaciones New Relic tienen [estructuras de cuentas padre/hijo](/docs/accounts/original-accounts-billing/original-users-roles/parent-child-account-structure). Para estas organizaciones, las cuentas secundarias reportan datos de respaldo a sus cuentas principales. Esta jerarquía organizacional afecta las capacidades de Nerdpack.
