Después de enviarnos sus datos de OpenTelemetry y abrir su servicio (entidad) en la UI, puede hacer clic en <DoNotTranslate>**Go runtime**</DoNotTranslate> para identificar qué instancia de servicio tiene patrones de rendimiento inusuales o incorrectos relacionados con el tiempo de ejecución del lenguaje.

<Callout variant="important">
  Debe instalar un paquete de instrumentación independiente para recopilar la métrica de tiempo de ejecución utilizada en esta vista. Consulte [los detalles de Runtime métrica](#metric-details) a continuación.
</Callout>

Puede elegir varias instancias de servicio para comparar en función de resúmenes de métricas clave: tiempo de respuesta, rendimiento, tasa de errores, tiempo de recolección de basura y uso de memoria. Luego, puede comparar todas las métricas de esas instancias usando gráficos de series temporales para detectar problemas y determinar si están involucradas recolección de basura, rutinas u otros recursos de tiempo de ejecución.

A continuación se muestra un flujo de trabajo típico:

1. Haga clic en

   <DoNotTranslate>**Go runtime**</DoNotTranslate>

   .

2. Encuentre una instancia interesante de Go runtime usando la tabla de salud métrica resumida:

   * Utilice la barra de filtro para limitar su búsqueda.
   * Ordene para encontrar valor atípico, por ejemplo, instancia con mayor tiempo de recolección de basura.

3. Seleccione aquellas instancias interesantes de Go runtime.

4. Haga clic en

   <DoNotTranslate>**Compare**</DoNotTranslate>

   para ver una visualización de la métrica de estado y tiempo de ejecución facetada por instancia.

    Cuando profundiza en una instancia específica o compara varias instancias, la UI mostrará gráficos impulsados por datos métricos de tiempo de ejecución de Go. Puedes encontrar la instrumentación que genera Go runtime métrica [aquí](https://pkg.go.dev/go.opentelemetry.io/contrib/instrumentation/runtime).
  </Collapser>

  <Collapser
    className="freq-link"
    id="metrics-or-spans"
    title="Métrica o tramos"
  >
    En la tabla y los gráficos, puede [optar por utilizar métrica o intervalos](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-find-entities/#metric-span-toggle) para alimentar las señales doradas (también conocidas como métrica de salud): tiempo de respuesta, rendimiento, tasa de errores.

    Las métricas de tiempo de ejecución, como las de recolección de basura, uso de memoria y gorutinas, solo están disponibles a partir de datos métricos y no de intervalos. Por lo tanto, no se ven afectados por este interruptor de palanca.
  </Collapser>

  <Collapser
    className="freq-link"
    id="config-steps"
    title="Cómo garantizar que tus datos aparezcan"
  >
    Para que sus datos aparezcan en esta sección, asegúrese de que tengan el atributo de recurso `service.instance.id` para representar la lista de instancias de tiempo de ejecución de Go.
  </Collapser>
