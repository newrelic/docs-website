Para maximizar la cantidad de datos que puede enviar por solicitud, recomendamos habilitar la compresión en todos los exportadores OTLP. New Relic admite la compresión `gzip` y `zstd` para cargas útiles OTLP transportadas a través de gRPC o HTTP con un tamaño de carga máximo permitido de 1 MB (10^6 bytes).

## Configuración del recolector de OTel

La compresión predeterminada para el exportador OTLP en el recolector OTel es actualmente `gzip`, mientras que la compresión `zstd` a menudo resulta más eficiente. Consulte [comparación de compresión](https://github.com/open-telemetry/opentelemetry-collector/blob/main/config/configgrpc/README.md#compression-comparison) para obtener más detalles sobre la información del punto de referencia.

Para habilitar `zstd` en el recolector, adapte la configuración del exportador del recolector:

```yml
exporters:
  otlphttp:
    ...
    compression: zstd
```

Si desea que admitamos otros formatos de compresión, háganoslo saber en el [canal CNCF Slack](https://cloud-native.slack.com/archives/C024DRQ63UP). Puede [registrarse para obtener una cuenta Slack de CNCF aquí](https://slack.cncf.io/).
