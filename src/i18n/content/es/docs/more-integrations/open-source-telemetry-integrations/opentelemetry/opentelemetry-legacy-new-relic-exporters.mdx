New Relic publicó una serie de exportadores que se utilizan para enviar datos de OpenTelemetry a través del protocolo propietario de New Relic. Estos exportadores están en desuso y prefieren enviar datos a New Relic utilizando el protocolo OpenTelemetry.

<Callout variant="important">
  Los exportadores se enumeran aquí como referencia. Para migrar desde estos exportadores, siga nuestra [guía de inicio rápido](/docs/integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-quick-start).
</Callout>

## Exportadores de lenguajes New Relic [#newrelic-language-exporters]

Como se mencionó anteriormente, nuestros exportadores específicos de idiomas están obsoletos. Estos exportadores permitieron que la aplicación exportara datos de OpenTelemetry directamente a New Relic. El [extremo OTLP de New Relic](#native-otlp) hace que esta estrategia quede obsoleta ya que permite que la aplicación exporte datos al extremo patentado de New Relic.

Los exportadores y su estado de mantenimiento son los siguientes:

* [Go](https://github.com/newrelic/opentelemetry-exporter-go): Archivado
* [Java](https://github.com/newrelic/opentelemetry-exporter-java): Archivado
* [Python](https://github.com/newrelic/opentelemetry-exporter-python): Archivado
* [.NET](https://github.com/newrelic/newrelic-telemetry-sdk-dotnet/tree/main/src/NewRelic.OpenTelemetry): Archivado
* [Collector](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/v0.50.0/exporter/newrelicexporter): Eliminado a partir del 10 de mayo de 2022.

## Configuración extrema para exportadores de New Relic [#h2-change-endpoints]

Puedes cambiar el extremo de New Relic donde envías tus datos.

### Extremo US [#default-endpoints]

De forma predeterminada, los exportadores de New Relic OpenTelemetry envían datos a estos centros de datos de EE. UU.:

* Abarca: [https://trace-api.newrelic.com/trace/v1](https://trace-api.newrelic.com/trace/v1)
* métrica: [https://metric-api.newrelic.com/metric/v1](https://metric-api.newrelic.com/metric/v1)
* Registros: [https://log-api.newrelic.com/log/v1](https://log-api.newrelic.com/log/v1)

Es posible que deba anular estos extremos predeterminados para enviar datos a la región de la UE o utilizar Infinite Tracing.

### Región extrema de la UE [#eu-region]

Para enviar telemetry data al extremo de New Relic en la región de la UE, utilice lo siguiente:

<Callout variant="tip">
  Estas URL no se aplican a Infinite Tracing.
</Callout>

* Abarca: [https://trace-api.eu.newrelic.com/trace/v1](https://trace-api.eu.newrelic.com/trace/v1)
* métrica: [https://metric-api.eu.newrelic.com/metric/v1](https://metric-api.eu.newrelic.com/metric/v1)
* Registros: [https://log-api.eu.newrelic.com/log/v1](https://log-api.eu.newrelic.com/log/v1)
