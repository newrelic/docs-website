Los límites de New Relic sobre atributos se aplican a datos de cualquier fuente, incluidos los datos de OTLP. Consulte [límites de atributo métrico](/docs/data-apis/ingest-apis/metric-api/metric-api-limits-restricted-attributes/) y [límites de atributo de evento](/docs/data-apis/ingest-apis/event-api/introduction-event-api/#limits) para conocer otros límites:

* Longitud máxima del nombre del atributo: 255 caracteres
* Longitud máxima del valor del atributo: 4095 caracteres
* Tamaño máximo del valor de la matriz de atributos: 64 entradas

Si un atributo de intervalo excede el límite, puede usar [variables de entorno de límites de intervalo](https://github.com/open-telemetry/opentelemetry-specification/blob/d6bcc0cb072d8d6f6ced856f1f23c451648a3caa/specification/sdk-environment-variables.md#span-limits-) para configurar las longitudes máximas. Si un [atributo de recurso](https://github.com/open-telemetry/opentelemetry-specification/blob/d6bcc0cb072d8d6f6ced856f1f23c451648a3caa/specification/sdk-environment-variables.md#general-sdk-configuration) es el infractor, puede configurar `OTEL_RESOURCE_ATTRIBUTES=<offending-attribute>=unset` para anularlo.

No todos los SDK de idiomas admiten las variables de entorno de límites de amplitud, por lo que depende del idioma que esté utilizando. Si su idioma no los admite, le recomendamos que abra un problema de GitHub en el repositorio del SDK del idioma respectivo para que la comunidad OpenTelemetry pueda resolverlo.
