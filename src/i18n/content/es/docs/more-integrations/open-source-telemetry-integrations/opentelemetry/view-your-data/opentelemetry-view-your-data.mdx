Después de enviar datos de OpenTelemetry a New Relic, puede utilizar una variedad de páginas de UI para analizarlos. Echa un vistazo a estas opciones:

* [Encuentra entidad y datos:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-find-entities) Encuentra tu entidad y profundiza en los datos más relevantes
* [Resumen:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-summary-page) Ver el estado general del servicio y comenzar la resolución de problemas
* [rastreo distribuido:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-distributed-tracing-page) Seguir solicitudes a través de sus sistemas distribuidos que involucren este servicio
* [transacción:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-transactions-page) ajuste el rendimiento de las solicitudes al extremo de un servicio y diagnostique problemas de latencia
* [Bases de datos:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-database-page) monitor tus llamadas de base de datos
* [Servicios externos:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-external-services-page) vea cómo se están desempeñando las solicitudes entre su servicio y otros, hasta el extremo involucrado.
* [JVM:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-jvms-page) monitor el rendimiento de su JVM
* [log:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-logs-page) diagnosticar problemas con el registro de aplicación
* [Página de errores:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-errors-page) vea qué errores está experimentando el usuario de su servicio
* [Errors Inbox:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-errors-inbox-page) clasificación y resolución de errores
* [Explorador métrica:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-metrics-explorer-page) Mira métricas específicas.
* [Cree consultas personalizadas:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-data-explorer-query-builder) pruebe estas herramientas de bricolaje para profundizar en sus datos

## Convenciones para datos de OpenTelemetry [#conventions]

El proyecto OpenTelemetry especifica convenciones semánticas para [métrica](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/metrics/semantic_conventions), [traza](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/trace/semantic_conventions), [log](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/logs/semantic_conventions) y [evento](https://github.com/open-telemetry/opentelemetry-specification/blob/main/specification/logs/semantic_conventions/events.md) para que la telemetría pueda ser interpretada consistentemente por herramientas de observabilidad. New Relic se basa en estas convenciones para presentar telemetry data en la UI.

El agente de instrumentación automática y los SDK de OpenTelemetry generalmente siguen las convenciones semánticas, pero como el proyecto está en rápido desarrollo, es posible que tenga que ajustar su instrumentación para que coincida con las convenciones.

Si los datos no se muestran como esperaba en la UI, consulte la documentación de la página UI correspondiente, donde puede encontrar una referencia de las convenciones semánticas utilizadas en esa página.
