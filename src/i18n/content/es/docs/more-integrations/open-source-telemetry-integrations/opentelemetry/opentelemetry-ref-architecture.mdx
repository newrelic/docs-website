Si necesita orientación para implementar OpenTelemetry con New Relic, esta arquitectura de referencia debería ayudar a sus desarrolladores de software, DevOps, arquitectos empresariales y gerentes de tecnología a colaborar en el proyecto. Es una plantilla que muestra una vista panorámica de formas comunes de implementar OpenTelemetry con New Relic.

El diagrama muestra una implementación que ingiere telemetría de OpenTelemetry, Prometheus, Jaeger y un agente New Relic <InlinePopover type="apm"/>. Los tres servicios (A, B y C) están instrumentados para enviar métrica, traza y log.

Para los datos recopilados por OpenTelemetry, puede enviarlos directamente a New Relic a través del extremo OTLP o puede enviarlos primero a un recolector OpenTelemetry opcional. Si utiliza el recopilador, agrega y mejora las canalizaciones de datos antes de que la telemetría se envíe al backend (Kafka para análisis de datos y New Relic para observabilidad de un extremo a otro).
