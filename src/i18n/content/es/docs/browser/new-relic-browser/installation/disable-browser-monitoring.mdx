Si está utilizando browser para monitor su aplicación, puede desactivar manualmente el monitoreo de su aplicación o de páginas específicas. Los motivos para desactivar <InlinePopover type="browser"/>en determinadas páginas incluyen:

* Algunas páginas pueden no ser relevantes para el monitoreo.
* Desea monitor solo páginas específicas.
* Desea eliminar el script browser para fines de resolución de problemas.

## Deshabilitar el monitoreo de páginas/URL específicas [#disable]

Para deshabilitar browser en una página o URL específica, siga los procedimientos según su método de implementación:

<CollapserGroup>
  <Collapser
    id="agent"
    title="Desplegar con el agente APM seleccionado"
  >
    Si utilizó el agente <InlinePopover type="apm"/>para insertar automáticamente su fragmento de JavaScript, puede deshabilitar la inyección solo para esas páginas en particular utilizando la API o el archivo de configuración del agente de lenguaje APM. Para obtener más información, consulte las instrucciones para su agente:

    * Go (actualmente no aplicable)
    * [Java](/docs/agents/java-agent/instrumentation/page-load-timing-java#disable-auto-specific)
    * [.NET](/docs/agents/net-agent/features/page-load-timing-net#disable_instrumentation_pageview)
    * [Nodo.js](/docs/agents/nodejs-agent/supported-features/page-load-timing-nodejs#disabling): Este agente no inserta JavaScript automáticamente. Para desactivar el monitoreo en ciertas páginas, simplemente elimine la API de llamada de esas páginas.
    * [PHP](/docs/agents/php-agent/features/new-relic-browser-php-agent#disable)
    * [Python](/docs/agents/python-agent/supported-features/page-load-timing-python#disabling_instrumentation)
    * [Ruby](/docs/agents/ruby-agent/installation-configuration/ignoring-specific-transactions#page-load-timing-rum)
  </Collapser>

  <Collapser

  </Collapser>

## Deshabilitar el monitoreo de dominio [#domains]

Para desactivar el monitoreo de un dominio o subdominio específico, actualice [las condiciones de dominio](/docs/browser/new-relic-browser/configuration/monitor-or-block-specific-domains-subdomains) de su navegador.

## Desactivar agente del navegador [#disable-agent]

Si utilizó el [método desplegable copiar/pegar](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app) y desea dejar de recopilar datos para una aplicación de navegador, simplemente elimine el fragmento de JavaScript del navegador de las páginas que no desea monitor. Si utilizó el [agente APM](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app) para instalar el agente del navegador, puede desactivar algunas de las características de monitoreo del navegador individualmente o puede desactivar el navegador por completo.

1. Vaya a

   <DNT>
     **[one.newrelic.com](https://one.newrelic.com/all-capabilities), click on Browser > (select a browser app) > Settings > Application settings**
   </DNT>

   .

2. Para deshabilitar solo la [característica de nivel de cuenta Pro](/docs/browser/new-relic-browser/getting-started/new-relic-browser), seleccione

   <DNT>
     **Lite**
   </DNT>

   .

3. Para desactivar completamente el monitoreo del navegador, seleccione

   <DNT>
     **Off**
   </DNT>

   .

4. Seleccione

   <DNT>
     **Save application settings**
   </DNT>

   .

5. Si corresponde, reinicie su agente

   <InlinePopover type="apm"/>

   .

Cuando el monitoreo del navegador está desactivado, el agente no instrumentará páginas con script de monitoreo.
