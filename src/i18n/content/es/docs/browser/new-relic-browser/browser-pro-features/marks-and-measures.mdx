[Las marcas](https://developer.mozilla.org/en-US/docs/Web/API/Performance/mark) y [medidas](https://developer.mozilla.org/en-US/docs/Web/API/Performance/measure) son API estándar del navegador que lo ayudan a observar e informar sobre el rendimiento de sus sitios web. Estos eventos nativos genéricos le permiten medir la duración de cualquier tarea o proceso en su aplicación. Con el monitoreo de navegador de New Relic, ahora puede rastrear automáticamente marcas y medidas y almacenarlas como `BrowserPerformance` evento, lo que le brinda una visibilidad más profunda del rendimiento real de su sitio.

## ¿Por qué emplear marcas y medidas de monitoreo?

El seguimiento automático de marcas y medidas con New Relic proporciona información procesable y valiosa sobre el rendimiento de su aplicación sitio web. Al capturar estos eventos, podrás:

* Identifique áreas lentas o problemáticas en su código.
* Medir el impacto de una nueva característica o implementación.
* Compare el rendimiento en diferentes recorridos de usuario.
* Cree dashboards personalizados y alertas basadas en datos reales de los usuarios.

## Requisitos previos [#prerequisites]

* Un agente del navegador **Pro** o **Pro + SPA**, versión 1.272.0 o superior.

<Callout variant="important">
  La detección automática de marcas y medidas no está disponible para el agente Lite del navegador.
</Callout>

## Habilitar la recopilación automática [#enable-collection]

Mejore la observabilidad y las capacidades de depuración de su aplicación siguiendo estas sencillas instrucciones de configuración.

<Steps>
  <Step>
    Vaya a **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Seleccione la aplicación de su browser .
  </Step>

  <Step>
    En el menú de la izquierda, haga clic en **Application settings**.
  </Step>

  <Step>
    Cerciorar de que esté seleccionado el agente del browser **Pro** o **Pro + SPA** .
  </Step>

  <Step>
    Para recopilar marcas automáticamente, active **Capture marks**. Para recopilar medidas automáticamente, active **Capture measures**. Para recopilar detalles de las marcas o medidas observadas como atributo personalizado, active **Add details**.
  </Step>
</Steps>

## Desactivar la recopilación de marcas y medidas [#disable-marks-and-measures]

Para deshabilitar marcas y/o medidas para una aplicación de navegador existente:

<Steps>
  <Step>
    Vaya a **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Seleccione la aplicación de su browser .
  </Step>

  <Step>
    En el menú de la izquierda, haga clic en **Application settings**.
  </Step>

  <Step>
    Para deshabilitar las marcas, desactive **Capture marks**. Para deshabilitar las medidas, desactive **Capture measures**.
  </Step>
</Steps>

## Consumo de datos

`BrowserPerformance` eventos siguen el mismo precio de consumo que los otros bytes de tu navegador. La cantidad de bytes producidos depende del recuento y del atributo de los datos.

Una vez habilitado, el agente capturará automáticamente las marcas y medidas a medida que ocurran en su aplicación.

## Encuentra tus datos en New Relic [#find-data]

Los datos de marcas y medidas se almacenan como `BrowserPerformance` incluso en New Relic, y se indican como `mark` o `measure` en el campo `entryType`. Consulte el [diccionario de datos](https://docs.newrelic.com/attribute-dictionary/?event=BrowserPerformance) para obtener más información sobre los campos disponibles para usar con `BrowserPerformance` evento. Para obtener un dashboard rápido y útil, consulte este [inicio rápido](https://newrelic.com/instant-observability/browser-marks-and-measures); de lo contrario, puede consultar y visualizar estos datos para obtener información valiosa sobre el rendimiento de su aplicación.

**Encuentra las medidas más lentas por duración media**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET entryName 
LIMIT 10 
ORDER BY average(entryDuration) DESC
```

**Cuente el número de marcas y medidas por página**

```nrql
FROM BrowserPerformance 
SELECT count(*) 
WHERE entryType IN ('mark', 'measure') 
FACET pageUrl
```

**Ver la duración media de la medida por navegador**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET userAgentName
```

**Duración de las medidas de seguimiento por país**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET countryCode
```

**Encuentre todos los nombres de medidas únicos registrados en las últimas 24 horas**

```nrql
FROM BrowserPerformance 
SELECT uniqueCount(entryName) 
WHERE entryType = 'measure' 
SINCE 24 hours ago
```

**Comparar las duraciones de las medidas para computadoras de escritorio y dispositivos móviles**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET deviceType
```

**Enumere todas las medidas para una sesión específica**

```nrql
FROM BrowserPerformance 
SELECT * 
WHERE entryType = 'measure' 
AND session = 'abcdefghijklmnop'
```

Emplee estas consultas o cree las suyas propias para crear dashboards personalizados, configurar alertas y monitorear el rendimiento real de su aplicación sitio web con precisión.
