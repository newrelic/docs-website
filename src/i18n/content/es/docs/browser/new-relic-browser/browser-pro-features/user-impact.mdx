La página <DNT>**User impact**</DNT> en <InlinePopover type="browser" />le ayuda a comprender cómo los problemas de rendimiento técnico afectan la experiencia real de sus usuarios. Esta página une las métricas de rendimiento cuantitativas con el análisis cualitativo del comportamiento del usuario, proporcionando información sobre los patrones de frustración del usuario y su correlación con los problemas técnicos.

La página User impact consta de dos áreas principales de análisis:

* <DNT>Frustration metrics</DNT>
* <DNT>Performance risk</DNT>

Cada una proporciona diferentes perspectivas para comprender los problemas de experiencia del usuario y sus causas técnicas fundamentales.

**Requisito previo:**

* Se requiere la versión 1.269 o posterior del agente Browser para capturar las señales de frustración y los datos de riesgo de rendimiento.
* Debe habilitarse [Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/get-started/) para acceder a las repeticiones de sesión vinculadas desde esta página.

## Acceder al análisis de Impacto del usuario [#access]

Para ver la página User impact:

1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app)** &gt; **Users** &gt; **User impact**</DNT>.
2. Seleccione <DNT>**Frustration metrics**</DNT> o <DNT>**Performance risk**</DNT> de la pestaña.

<Callout variant="tip">
  También puede navegar aquí directamente desde la página [Page views](/docs/browser/new-relic-browser/browser-pro-features/page-views-examine-page-performance) haciendo clic en el enlace <DNT>**Analyze frustration signals**</DNT> o <DNT>**Analyze performance risk**</DNT>.
</Callout>

## Analizar las métricas de frustración [#frustration-metrics]

La pestaña Frustration metrics se centra en los indicadores de comportamiento del usuario que señalan confusión, molestia o dificultad para completar tareas. Este análisis le ayuda a identificar dónde los usuarios tienen más dificultades y a priorizar las correcciones en función de los puntos débiles reales de los usuarios.

La pestaña Frustration metrics incluye las siguientes secciones:

* Clic de rabia: Identifica las páginas y los elementos donde los usuarios exhiben un comportamiento de clic rápido y repetido, lo que indica frustración con componentes de la interfaz de usuario que no responden o están rotos.
* Páginas más afectadas: Destaca las páginas específicas que generan la mayor cantidad de señales de frustración, lo que le permite centrarse en áreas de alto impacto.
* Elementos de página más afectados: Proporciona información detallada sobre qué elementos específicos de la interfaz de usuario están causando la mayor frustración del usuario.

## Analizar la métrica de rendimiento [#performance-impact]

La pestaña Performance risk examina cómo los problemas de rendimiento técnico afectan las sesiones de los usuarios, centrándose en las métricas de rendimiento medibles y su correlación con los problemas de experiencia del usuario.

La pestaña Performance risk incluye las siguientes secciones:

* Vitales web: Analiza cómo los vitales web principales deficientes (LCP, INP, CLS) impactan las sesiones y la experiencia del usuario.
* Análisis de errores: Investiga los patrones de errores de JavaScript y su efecto en las sesiones de los usuarios.
* Páginas más afectadas: Identifica qué páginas experimentan el mayor impacto relacionado con el rendimiento del usuario.
* Análisis geográfico: Examina el riesgo de rendimiento en diferentes ubicaciones geográficas para identificar problemas regionales.
