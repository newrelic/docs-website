## Problema

No estás viendo [el rastreo de sesión](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle) de tu aplicación de navegador.

## Solución

Si su aplicación está [instrumentada con APM](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation) y otras [características asociadas con el Pro agente del navegador](/docs/browser/new-relic-browser/browser-pro-features) están funcionando, siga estos pasos:

1. Compruebe si está [utilizando varios nombres para su aplicación](/docs/apm/new-relic-apm/installation-configuration/using-multiple-names-app). rastreo de sesión se escriben únicamente en la aplicación principal.

2. Si está utilizando varios nombres, verifique los nombres de sus

   <InlinePopover type="apm"/>

   aplicaciones para verificar que la aplicación aparezca en primer lugar.

3. Si la aplicación no aparece primero, busque rastreo de sesión en el nombre de la aplicación que aparece primero.

Otros motivos por los que es posible que no veas los datos del rastreo de sesión incluyen:

* El usuario final no utiliza un navegador que admita la [API de sincronización de recursos](http://www.w3.org/TR/resource-timing/).
* El usuario final no puede publicar datos en el extremo `/resources` de `bam.nr-data.net`.
* Los clientes del usuario final no permiten las [cookies de sesión](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking) necesarias.
