## Problema

Nuestro <InlinePopover type="browser"/>está insertando su instrumentación JavaScript en una ubicación que causa problemas con el marcado o la carga de la página.

## Solución

El agente New Relic intenta inyectar JavaScript en la ubicación óptima, pero si encuentra problemas, pruebe estos pasos de resolución de problemas.

1. Verifique si el monitoreo del navegador es la causa [deshabilitando el monitoreo del navegador](/docs/browser/new-relic-browser/installation/disable-browser-monitoring/).

2. Si deshabilitar el monitoreo del navegador resuelve el problema, investigue el código fuente de la página. Vuelva a habilitar el monitoreo del navegador y verifique los siguientes posibles problemas:

   * Más de una instancia de la etiqueta `<html>` en la página (no en un iFrame).
   * Más de una instancia de la etiqueta `<head>` o `<body>` , o falta una etiqueta `<head>` o `<body>` .
   * Aparición de cualquier etiqueta `<head>` o `<body>` dentro del JavaScript de la página.
   * Utilice [http://validator.w3.org](http://validator.w3.org) para verificar la fuente de su página y verifique si hay problemas graves de validez de marcado o problemas de codificación de caracteres.

3. Elimine cualquier conflicto que encuentre revisando la estructura de la página, [deshabilitando el monitoreo del navegador para esa página](/docs/browser/new-relic-browser/installation-configuration/disable-browser-monitoring-specific-pages) o cambiando a [instrumentado manualmente a través de la API del agente](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring#agent-instrumentation) para tener un control total sobre la inyección script .

4. Si encuentra otras situaciones, comuníquese con [el soporte de New Relic](https://support.newrelic.com/) y proporcione un enlace accesible a su página o el código fuente de la misma. Asegúrese de que el monitoreo del navegador esté habilitado en su enlace o código fuente.
