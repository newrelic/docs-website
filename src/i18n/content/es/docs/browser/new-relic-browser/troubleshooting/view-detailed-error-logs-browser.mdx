## Problema

Desea utilizar la consola de su browser para ver el mensaje de error detallado del entorno, pero cuando <InlinePopover type="browser"/>está activo, la consola de su browser no muestra la traza de error detallada.

## Solución

El monitoreo del navegador debe capturar y volver a generar errores para poder registrarlos, pero algunas versiones browser no muestran el rastreo del stack para los errores reiniciados. Para resolver este problema, habilite el modo de desarrollo del browser en su browser local.

[Las versiones 593 o superiores del agente de monitoreo de navegador](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) incluyen una solución alternativa para este comportamiento, que se puede habilitar en la consola. El siguiente procedimiento es para Google Chrome, pero puedes adaptarlo a otros navegadores.

1. En Chrome, seleccione el ícono de menú y luego seleccione <DoNotTranslate>**More tools > JavaScript Console**</DoNotTranslate>.

2. Desde la consola de JavaScript, establezca el siguiente valor para `nr_flags` en el almacenamiento local:

   ```js
   localStorage.setItem('__nr_flags', 'dev')
   ```

3. Actualiza la página donde deseas ver tus errores de JavaScript.

4. Para confirmar que el modo de desarrollo está activo, consulte el registro del agente del navegador de la consola para ver el mensaje `NR AGENT IN DEVELOPMENT MODE`.

5. Navegue por el sitio y luego active los errores que desea rastrear.

Para deshabilitar el modo de desarrollo, ejecute el siguiente comando en la consola:

```js
localStorage.removeItem('__nr_flags')
```
