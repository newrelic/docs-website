### Almacenamiento de sesiones Browser [#browser-session-storage]

Consolidamos el control para recopilar datos de rastreo de sesión y reproducción de sesión dentro del agente del navegador. Ahora puedes gestionar todos los datos a nivel de sesión directamente desde el interruptor <DNT>**Access browser local storage**</DNT> en esta página de configuración.

/\* \* \*\*Opción para activar/desactivar el almacenamiento local del navegador\*\*: Este control unificado determina cómo se recopilan los datos de nivel de sesión (incluidos los ID de sesión y otros atributos de nivel de sesión). \* \*\*Cuando está activado (por defecto)\*\*: El agente de navegador de New Relic emplea los mecanismos de almacenamiento persistente del navegador para recopilar datos de rastreo de sesión y Session Replay. \* \*\*Cuando está DESACTIVADO\*\*: El agente de navegador de New Relic detiene inmediatamente el uso del almacenamiento persistente del navegador. Esta acción impide la recopilación de todos los datos de rastreo de sesión y Session Replay, lo que le brinda un control directo sobre el manejo de la información relacionada con la sesión por parte del navegador y mejora su capacidad para cumplir con los requisitos de privacidad y cumplimiento de datos. \*/

<Callout variant="important">
  A partir de la versión 1.303.0 del agente de Browser, se eliminará el interruptor de **Access local storage** de la UI para simplificar nuestra función de privacidad y consentimiento.

  * La forma recomendada de gestionar la recopilación de datos del agente del navegador y el consentimiento del usuario es con la configuración del modo de consentimiento. Para obtener más información, consulte [los documentos de consentimiento](/docs/browser/new-relic-browser/configuration/consent-mode/).

  * Si actualmente tiene el interruptor de **Access local storage** configurado en `OFF`, encontrará que los interruptores de modo de consentimiento, reproducción de sesión y rastreo de sesión también están inactivos. Para habilitar estas características, debe establecer el atributo `cookie_enabled` en `true` a través de [APIde NerdGraph](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-browser-application). Esta acción reactivará los interruptores en la UI, lo que le permitirá configurar correctamente el modo de consentimiento, la reproducción de sesión y el rastreo de sesión.
</Callout>

## API browser [#apis]

También puede personalizar la instrumentación del agente de su browser para capturar trazas y eventos personalizados, informar errores específicos adaptados a sus necesidades y ajustar el tiempo de recolección de datos para un rendimiento óptimo. Consulte [Uso de las API del navegador](/docs/browser/new-relic-browser/browser-apis/using-browser-apis/) para comenzar.
