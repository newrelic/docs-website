### Almacenamiento de sesiones Browser [#browser-session-storage]

Consolidamos el control para recopilar datos de rastreo de sesión y reproducción de sesión dentro del agente del navegador. Ahora puedes gestionar todos los datos a nivel de sesión directamente desde el interruptor <DNT>**Access browser local storage**</DNT> en esta página de configuración.

* **El interruptor para acceder al almacenamiento local del navegador**: este control unificado determina cómo se recopilan los datos a nivel de sesión (incluidos los identificadores de sesión y otros atributos a nivel de sesión).

  * **Cuando está activado (predeterminado)**: el agente del navegador New Relic emplea los mecanismos de almacenamiento persistente del navegador para recopilar datos de seguimiento de sesión y reproducción de sesión.
  * **Cuando está desactivado**: el agente del navegador New Relic detiene inmediatamente el uso del almacenamiento persistente del navegador. Esta acción evita la recopilación de todos los datos de rastreo de sesión y de reproducción de sesión, lo que le brinda control directo sobre el manejo de la información relacionada con la sesión por parte del navegador y mejora su capacidad para cumplir con los requisitos de privacidad y cumplimiento de datos.

Para llegar a la página <DNT>**Application settings**</DNT> :

1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT> y seleccione una aplicación.
2. En la navegación izquierda, desplácese hacia abajo hasta <DNT>**Settings**</DNT> y haga clic en <DNT>**Application settings**</DNT>.

También puede personalizar la instrumentación del agente de su browser para capturar trazas y eventos personalizados, informar errores específicos adaptados a sus necesidades y ajustar el tiempo de recolección de datos para un rendimiento óptimo.
