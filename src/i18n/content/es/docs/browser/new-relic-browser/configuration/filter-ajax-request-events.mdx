En la [versión 1211 del agente](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1211/) o superior, todas las solicitudes de red realizadas por una página se registran como eventos `AjaxRequest`. Puede usar las opciones de configuración de la lista de denegación en la página <DNT>**Application settings**</DNT> para filtrar qué solicitudes registran eventos.

## Usando la lista de denegación

De forma predeterminada, registramos eventos `AjaxRequest` para todas las solicitudes de red realizadas por su aplicación.

* Para dejar de registrar todos los eventos, agregue un único carácter comodín `*` (asterisco).
  * Ejemplo: `*`
* Para dejar de registrar eventos para un sitio completo, agregue el nombre de dominio.
  * Ejemplo: `mywebsite.com`
* Para dejar de registrar eventos para una página específica, agregue el dominio y la ruta.
  * Ejemplo: `mywebsite.com/photos`
* A partir de la [versión 1.307.0 del agente](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.307.0/), en cualquier lugar donde encontremos un `*` (asterisco), coincidirá con cualquier longitud de cadena.
  * Ejemplo: `mywebsite.com/shop/*` (para una ruta), `*.mywebsite.com` (para un subdominio), `mywebsite.com/*/shop/*/checkout` (para cadenas de URL intermedias).

La lista de denegación ignora el protocolo, el puerto, la búsqueda y el hash de una URL.

## Accede a la lista de denegados

Para actualizar la lista de URL denegadas que su aplicación filtrará para la creación de eventos, vaya a la página de configuración de la aplicación:

1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Settings &gt; Application settings**</DNT>.
2. En <DNT>**Ajax Request Deny List**</DNT>, agregue los filtros que desea aplicar a su aplicación.
3. Seleccione <DNT>**Save application settings**</DNT> para actualizar la configuración del agente.
4. [Vuelva a implementar el agente del navegador](/docs/browser/new-relic-browser/installation/upgrade-browser-agent) (ya sea reiniciando el agente <InlinePopover type="apm" />asociado o actualizando la instalación de copiar y pegar del navegador).

## Validación

Para validar si los filtros que ha agregado funcionan como se esperaba, ejecute una consulta NRQL para `AjaxRequest` evento que coincida con su filtro.

```sql
FROM AjaxRequest SELECT * WHERE requestUrl LIKE `%example.com%`
```
