## ¿Qué función cumple esta característica?

Esto permite la integración de opciones de aceptación de la política de privacidad para que el usuario pueda elegir cómo la aplicación sitio web recopila, transmite y procesa los datos monitorear por el agente del browser. Al habilitar esta característica, puede permitir que el usuario decida si consiente que el agente del browser envíe datos a través de la red.

Hay disponible una nueva propiedad en el objeto de inicialización del agente del browser llamada `browser_consent_mode`. Esta configuración controla si se debe emplear el modo de consentimiento en el agente. Disponible en el agente Browser Lite, Browser Pro o Pro + SPA, [versión 1.303.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.303.0) o superior.

## ¿Como lo uso?

Para usar esta característica, deberá visitar la página de configuración de su aplicación y activar el interruptor `"Turn on consent"`. Esta opción está desactivada por defecto.

NerdGraph puede emplear como alternativa para activar y desactivar la configuración. Para ello, ejecute una consulta de mutación en el objeto graphQL `browserSettings` para la propiedad `browser_consent_mode`, estableciendo `enabled` en `true` o `false`.

Una vez que integraste una política de privacidad o un formulario de aceptación en tu aplicación sitio web, si un usuario hace clic en uno de los siguientes enlaces:

1. “Sí, doy mi consentimiento.”

   Deberá invocar el uso de una nueva API llamada `consent(accept?: boolean)`. Al llamar a la [API consent()](/docs/browser/new-relic-browser/browser-apis/consent), el parámetro accept es verdadero de forma predeterminada.

   La característica Browser con **autoStart: true** enviará datos a New Relic después de que se proporcione el consentimiento.

   La característica Browser con **autoStart: false** deberá ser invocada empleando la función [APIstart()](/docs/browser/new-relic-browser/browser-apis/start) para iniciar la característica del agente diferido para enviar datos a través de la red.

2. “No, no doy mi consentimiento.”

   Dado que el valor predeterminado es verdadero, deberá llamar a `consent(false)`, teniendo en cuenta el argumento falso. El agente no enviará datos a través de la red para ese usuario.

## Consideraciones adicionales

* Cuando `browser_consent_mode` está habilitado, se almacenarán en búfer hasta \~30 segundos y un máximo de 1 MB de datos en el navegador del usuario final y no se enviarán a New Relic hasta que el usuario seleccione el consentimiento y se llame a la API de consentimiento. Debido a estas limitaciones, es posible que algunos datos se descarten y no se envíen a New Relic cuando se invoque la API de consentimiento.

* Si [el almacenamiento de sesión](/docs/browser/new-relic-browser/configuration/browser-app-settings-page/#browser-session-storage) está habilitado, las decisiones transmitidas a la API de consentimiento persistirán entre páginas.

* Si [el almacenamiento de sesión](/docs/browser/new-relic-browser/configuration/browser-app-settings-page/#browser-session-storage) está deshabilitado, será necesario llamar a la API de consentimiento en cada carga de página.
