[browser](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) le permite elegir qué dominio o subdominios monitorea una aplicación del navegador. Esto es útil para limitar el monitoreo solo a la actividad de la aplicación que considere importante. Por ejemplo, si utiliza el [método de copiar y pegar](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) para desplegar la aplicación de su navegador, puede establecer condiciones de dominio para proteger la seguridad de su [clave de licencia<InlinePopover type="browser"/>y su ID de aplicación](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id).

<Callout variant="tip">
  Las condiciones del dominio solo filtran datos según la URL de la página informada. Los filtros de condición de dominio no se aplican a los datos AJAX reportados a New Relic. Para filtrar el evento `AjaxRequest` , lea [nuestros documentos de configuración](/docs/browser/new-relic-browser/configuration/filter-ajax-request-events/).
</Callout>

## Configurar las condiciones del dominio [#configure]

Para ver o editar la configuración de las condiciones de tu dominio:

1. Vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Application settings > Domain conditions**</DNT>.

2. Si no existen condiciones de dominio vigentes, seleccione <DNT>**Enable domain conditions**</DNT>. Si existen condiciones, seleccione <DNT>**Edit conditions**</DNT>.

3. Seleccione la opción para crear una lista de dominios permitidos o para crear una lista de dominios bloqueados/denegados.

4. Seleccione las condiciones del dominio (máximo 10 condiciones).

5. Seleccione las condiciones del dominio con la condición <DNT>**Starts with**</DNT> , luego ingrese la URL con el subdominio (máximo 10 condiciones).

     title="browser_domain_conditions.png"
     alt="browser_domain_conditions.png"
     src="/images/browser_screenshot-full_domain-settings.webp"
   />

   <figcaption>
     <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select a browser app) > Settings > Domain conditions:**</DNT> Utilice esta característica para limitar el monitoreo solo a la actividad importante de su aplicación o para mejorar la seguridad de la clave de licencia de su browser y la ID de su aplicación.
   </figcaption>

6. Revise y confirme la configuración de condiciones de su dominio.

Un cuadro de texto de vista previa muestra en tiempo real qué dominio coincide con las condiciones que usted estableció. También verá una estimación del impacto que tendrán las configuraciones propuestas en sus datos.

## Desactivar las condiciones del dominio [#disable]

Para deshabilitar las condiciones de dominio y volver a monitorear todos los dominios:

1. Vaya a

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Domain conditions**
   </DNT>

   .

2. Establezca la opción

   <DNT>
     **Enable domain conditions**
   </DNT>

   en

   <DNT>
     **Off**
   </DNT>

   .
