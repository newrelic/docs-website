Nuestro <InlinePopover type="browser"/>coloca un pequeño script de "cargador" en la sección principal de cada página en un sitio de monitoreo. Esta instrumentación de JavaScript ha sido diseñada para:

* Proporcione la máxima visibilidad del rendimiento del frontend, cronometre con precisión el tiempo que tardan las páginas en cargarse en todos los navegadores e informe detalles sobre errores de JavaScript y llamadas AJAX.
* Tener un impacto mínimo en el tiempo general de carga de la página y la optimización de motores de búsqueda (SEO), incluida la clasificación en los motores de búsqueda, la indexación, la eficiencia de rastreo u otras preocupaciones relacionadas con SEO.

## Eficiencia de JavaScript en línea [#why-javascript]

El método más eficaz para monitorear el navegador es incluir una cantidad mínima de código de instrumentación JavaScript en línea en el encabezado de la página del monitor. Este código luego recupera el resto del código necesario una vez que la página termina de cargarse.

Otros métodos de monitoreo del navegador incluyen código JavaScript al final del cuerpo de la página o utilizando exclusivamente un script externo. Sin embargo, estos métodos pueden limitar la visibilidad de la experiencia del usuario final.

## Impacto en SEO [#seo-impact]

Los resultados de las pruebas de rendimiento indican que el uso del monitoreo de JavaScript del navegador tiene un efecto insignificante en el tiempo de carga de la página. Además, no tiene ningún impacto negativo en la forma en que los usuarios o los motores de búsqueda interactúan con su sitio.

La recomendación constante de Google a los propietarios de sitios web es crear un sitio que sea valioso para el usuario y accesible para los rastreadores de los motores de búsqueda. Los rankings de Google favorecen los sitios que brindan la información más relevante y la mejor experiencia del usuario.

El monitoreo del navegador puede ayudarlo a mejorar la experiencia del usuario al identificar el rendimiento del cuello de botella, incluyendo:

* [Cargas de página lentas](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity)
* [Errores problemáticos de JavaScript](/docs/errors-inbox/browser-tab)
* [Llamadas AJAX largas](/docs/browser/new-relic-browser/browser-pro-features/ajax-dashboard-identifying-time-consuming-calls)

Identificar errores de Javascript es especialmente útil porque los robots de Google ejecutan cada vez más el código JavaScript en los sitios web que rastrean para acceder al contenido proporcionado por la aplicación web con alto contenido de AJAX. Un error de JavaScript que anteriormente solo era visible para un usuario humano (por ejemplo, un botón roto) podría afectar la capacidad de los robots de Google para interactuar correctamente con su sitio.

Entendemos que el SEO y el tráfico remitido por los motores de búsqueda son de vital importancia para muchas empresas. Cuando se usa de manera efectiva, el monitoreo de navegador puede incluso aumentar la clasificación de un sitio en Google y otros motores de búsqueda al mejorar el rendimiento y la experiencia del usuario.
