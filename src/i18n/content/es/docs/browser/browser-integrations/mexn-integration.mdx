MEAN, MERN y MEVN stack son kits de desarrollo de stack completos de código abierto. Cada kit utiliza una única base de código para desarrollar una aplicación web. New Relic ha consolidado las tres pilas en un solo inicio rápido. Nuestra integración MExN utiliza el agente <InlinePopover type="browser"/>, el agente Node.js y la integración de MongoDB para brindarle un dashboard prediseñado con sus métricas más importantes (como tiempo de respuesta, utilización de CPU, tráfico y frecuencias de inicio de sesión).

## Instalar [#install]

<Steps>
  <Step>
    ### Opcional: instalar el agente de infraestructura [#infra-install]

    Instalar el agente de infraestructura es opcional, pero al hacerlo obtendrá métricas importantes sobre el host donde se ejecuta su aplicación.

    Para hacer esto, siga [los pasos de instalación del agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent).
  </Step>

  <Step>
    ### Instalar MongoDB inicio rápido

    1. Consulte los requisitos de nuestro agente MongoDB en nuestra [documentación](/docs/infrastructure/host-integrations/host-integrations-list/mongodb-monitoring-integration/) antes de instalar el agente.

    2. Abra la [instalación de inicio rápido de MongoDB](https://newrelic.com/instant-observability/mongodb).

    3. Haga clic en

       <DNT>
         **Install now**
       </DNT>

       para iniciar la instalación del agente MongoDB.
  </Step>

  <Step>
    ### Instalar el agente Node.js

    1. Consulte nuestros requisitos del agente Node.js en nuestra [documentación](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/).

    2. Abra la [instalación de inicio rápido de Node.js.](https://newrelic.com/instant-observability/node-js)

    3. Haga clic en

       <DNT>
         **Install now**
       </DNT>

       para iniciar la instalación del agente Node.js.
  </Step>

  <Step>
    ### Instalar el agente del navegador [#browser-install]

    1. Desde [one.newrelic.com](https://one.newrelic.com), vaya a la [página](https://one.newrelic.com/marketplace)

       <DNT>
         [**+ Add data**](https://one.newrelic.com/marketplace)
       </DNT>

       [](https://one.newrelic.com/marketplace).

    2. Haga clic en

       <DNT>
         **Browser and mobile**
       </DNT>

       y luego haga clic en

       <DNT>
         **Browser monitoring**
       </DNT>

       .

    3. Seleccione la opción

       <DNT>
         **Select copy/paste JavaScript code**
       </DNT>

       (para obtener más información sobre este código, consulte [nuestros documentos de instalación del navegador](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent)).

    4. Complete el resto de los pasos y haga clic

       <DNT>
         **Enable.**
       </DNT>

    5. Vaya al archivo `index.html `de su aplicación React o aplicación Angular o aplicación Vue.

    6. Pegue la etiqueta del fragmento de código de nuestro justo antes de que finalice la etiqueta del cuerpo (browser script `</body>`).

    7. Reinicie su aplicación.

       En un par de minutos, tu aplicación se enviará métrica a [one.newrelic.com](https://one.newrelic.com).
  </Step>
</Steps>

## Encuentra tus datos [#find-data]

Para obtener su dashboard MExN:

1. Desde [one.newrelic.com](https://one.newrelic.com), vaya a la página

   <DNT>
     **+Add data**
   </DNT>

   .

2. Haga clic en

   <DNT>
     **Dashboards**
   </DNT>

   .

3. En la barra de búsqueda, escriba `MExN`.

4. Debería aparecer el dashboard de MExN. Haga clic en él para instalarlo.

Su dashboard MExN se considera un panel personalizado y se puede encontrar en la UI <DNT>**Dashboards**</DNT>. Para obtener documentos sobre el uso y edición del panel, consulte [nuestros documentos dashboard ](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Para obtener información sobre los datos reportados, consulte los documentos de cada una de las herramientas que instaló:

* [monitoreo de eventos del navegador](/docs/data-apis/understand-data/event-data/events-reported-browser-monitoring).
* [Datos de infraestructura](/docs/infrastructure/manage-your-data/data-instrumentation/default-infrastructure-monitoring-data/#infrastructure-events).
* [Datos de MongoDB](/docs/infrastructure/host-integrations/host-integrations-list/mongodb/mongodb-monitoring-integration-new/#find-and-use/).
* [Datos de Node.js.](/docs/apm/agents/nodejs-agent/getting-started/introduction-new-relic-nodejs/)

Aquí hay una consulta NRQL que verifica el tiempo promedio de carga de la ventana para los distintos navegadores utilizados por el usuario en las computadoras de escritorio:

```sql
SELECT average(windowLoad) 
FROM PageViewTiming 
WHERE (deviceType = 'Desktop') 
FACET `userAgentName` 
SINCE 604800 seconds 
AGO TIMESERIES
```
