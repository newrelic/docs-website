La función de [tiempo de carga de página](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) de la supervisión del navegador puede rastrear sesiones utilizando la API localStorage que puede almacenar un identificador de sesión simple.

## Cómo funciona [#process]

Este es el proceso básico para el seguimiento de sesiones:

1. Una sesión comienza cuando un usuario visita un sitio web donde está instalado el monitoreo del navegador de New Relic y la configuración de privacidad para el seguimiento está habilitada.
2. A medida que se carga la página del usuario, nuestro <InlinePopover type="browser" />script establece un identificador de sesión simple en el browser.
3. Luego, el script lee el ID en cargas, recargas y restauraciones de páginas posteriores, dentro de la misma pestaña o en pestañas nuevas del mismo dominio.
4. Si ocurre uno de los siguientes eventos, la sesión finalizará y comenzará una nueva sesión:

* Una sesión ha alcanzado un tiempo máximo de 4 horas.

* Se carga una página con una sesión preexistente que comenzó más de 4 horas antes

* No se &quot;interactúa&quot; con ninguna página de la sesión activa durante 30 minutos consecutivos. Nuestra definición de interacción con una página web consiste en

  * Haciendo clic
  * Desplazamiento
  * Mecanografía

* La sesión se restablece mediante programación al emitir una llamada para actualizar el ID de usuario a través de [setUserId()](/docs/browser/new-relic-browser/browser-apis/setuserid)

* El seguimiento de sesiones no funcionará correctamente en estas situaciones:

  * Si el usuario tiene el almacenamiento DOM deshabilitado en su browser.
  * Si el browser o la página están configurados para **no** permitir la carga y ejecución de scripts de origen de terceros a través de políticas de seguridad u otros medios (lo que naturalmente significa que el agente en su conjunto no funcionará).
  * Las API de almacenamiento del navegador, como localStorage, se borran durante una sesión activa.
