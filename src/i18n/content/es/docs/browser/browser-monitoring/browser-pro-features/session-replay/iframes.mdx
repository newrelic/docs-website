Cuando está habilitada, la función de reproducción de sesión podría no mostrar datos de reproducción para el contenido de iframe. Esta limitación resulta del comportamiento de Session Replay en escenarios de iframe, que varía según el origen del iframe y la ubicación del agente del browser.

Supongamos que un sitio web contiene una ventana de nivel superior y una ventana secundaria `iframe`. Los comportamientos de Session Replay, en función de la ubicación del agente del browser, son los siguientes:

* **Iframes del mismo origen**:

  * Con el agente del browser en la ventana de nivel superior, Session Replay graba tanto la ventana como el contenido `iframe` .
  * Con el agente del browser únicamente en `iframe`, Session Replay graba únicamente el contenido dentro de `iframe`.
  * Con el agente del browser tanto en la ventana de nivel superior como en `iframe`, Session Replay registra cada contexto de forma independiente, lo que da como resultado dos sesiones distintas.

* **Iframes de origen cruzado**:

  * Con el agente del browser en la ventana de nivel superior, Session Replay captura solo el contenido de la ventana; el contenido `iframe` no será visible en la repetición.
  * Con el agente del browser en el iframe, Session Replay captura solo el contenido `iframe` .

**Limitaciones:**

* La repetición de sesión no admite `<frame>` elementos.
