Las acciones del usuario en el monitoreo del browser lo ayudan a comprender el comportamiento del usuario, como dónde hace clic, se desplaza e interactúa con su aplicación sitio web. Puede emplear acciones de usuario para vincular acciones entre sí, lo que lo ayudará a obtener información valiosa sobre dónde su aplicación sitio web crea fricción y aumenta la frustración del usuario.

Las acciones del usuario se rastrean de forma predeterminada para el agente Pro y Pro+SPA, pero no están disponibles para el agente Lite del browser. Le recomendamos que primero confirme que está empleando el agente Pro o Pro+SPA:

Vaya a <DNT>**[one.newrelic.com &gt; All Capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT> (seleccione la aplicación de su browser ) &gt; <DNT>**Application settings**</DNT>. Desde esta página, verifique que el agente del browser <DNT>**Pro**</DNT> o <DNT>**Pro + SPA**</DNT> esté seleccionado. Cerciorar de tener la versión v1.268.0 o superior.

## Ver acciones del usuario [#view-events]

Cuando un usuario carga un sitio web, nuestro agente del browser configura escuchas de eventos para eventos de interacción clave. Cuando el agente del browser observa estas acciones, puede recolectar los eventos generados y reenviarlos a New Relic.

Las acciones del usuario pueden recolectar estos eventos:

* auxclick
* desenfocar (solo ventana)
* hacer clic
* Copiar
* foco (solo ventana)
* tecla abajo
* pasta
* Desplazar hacia el final

Para ver las acciones del usuario capturadas por el agente del browser, cree una consulta NRQL para el tipo de evento `UserAction` en [el generador de consultas](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/). A continuación se muestra un ejemplo de una consulta básica:

```sql
FROM UserAction 
SELECT *
```

De forma predeterminada, los datos de eventos browser se almacenan durante ocho días, pero la retención real de los datos depende de su cuenta. Para obtener más detalles sobre las acciones del usuario, consulte el [Diccionario de datos](/attribute-dictionary/).

## Consumo de datos [#data-consumption]

Las acciones del usuario se agregan secuencialmente, lo que significa que si se realizan múltiples acciones en rápida sucesión en el mismo objetivo DOM , se agrupan como una sola acción del usuario. Esto mantiene bajos los costos de consumo y al mismo tiempo lo ayuda a comprender cómo interactúa el usuario con su aplicación sitio web.

Las acciones del usuario siguen el mismo precio de consumo que el resto de bytes de su browser . La cantidad de bytes producidos depende del recuento, la longitud y los niveles de actividad del usuario en las sesiones.
