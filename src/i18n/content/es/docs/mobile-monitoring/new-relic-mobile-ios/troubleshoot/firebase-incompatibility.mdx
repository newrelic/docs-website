## Problema

Se pueden mostrar conflictos, fallas o errores cuando tanto el agente de iOS de New Relic como otros agentes de seguimiento de telemetry data de terceros, como Firebase, se incluyen dentro de una aplicación.

Errores conocidos:

* Fallo en las funciones instrumentadas.
* No se pudo capturar el registro de fallos.

## Solución

Nuestro agente de iOS y otros agentes de seguimiento telemetry data de terceros, como Firebase, no deben habilitarse al mismo tiempo. Recomendamos habilitar solo uno u otro para evitar este problema.

Si no es una opción eliminar el otro SDK, existen algunos pasos que puede seguir para evitar estos conflictos.

* Inicie nuestro agente antes de iniciar cualquier otro SDK de terceros.
* Deshabilite algunas de las funciones de nuestro agente. Si esta es una opción que funciona para usted, consulte nuestra [guía de API](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/configure-settings/) para obtener más información.

Si necesita ayuda adicional, obtenga soporte en [support.newrelic.com](https://support.newrelic.com).
