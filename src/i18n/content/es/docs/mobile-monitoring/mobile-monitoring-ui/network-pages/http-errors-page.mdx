## Descubra la página de errores HTTP [#ui-interactions]

Desde la página <DNT>**HTTP errors**</DNT> , investigue las solicitudes HTTP y los errores de red:

* Utilice cualquier función de página estándar para buscar tendencias en

  <DNT>
    **Errors and failures**
  </DNT>

  gráficos.

* Objetivo tipos específicos de errores y fallas agrupando, ordenando y filtrando los datos.

* Encuentre anomalías en los errores de su solicitud con perfiles de error HTTP.

* Seleccione un error o falla para ver sus detalles.

También puede definir [alertas NRQL](/docs/alerts/new-relic-alerts/defining-conditions/create-alert-conditions-nrql-queries) que se centren en tipos de errores para sus servicios críticos o [consultar los datos de su aplicación](#insights).

## Perfiles de error HTTP [#error-profiles]

Los perfiles de error proporcionan detalles visuales sobre diferencias significativas en la frecuencia de diferentes valores para eventos de error HTTP. Para cada [atributo](/docs/new-relic-solutions/get-started/glossary/#attribute), el perfil de error incluye:

* Un mapa de calor que muestra cómo se distribuye el atributo del error para los valores que más se desvían
* Una etiqueta que compara la distribución del atributo de error con la de otros errores.

<img
  title="HTTP error profiles"
  alt="Screenshot of http error profiles"
  src="/images/mobile_screenshot-full_http-error-profiles.webp"
/>

Esto le ayuda a eliminar más conjeturas al resolver los errores HTTP de su aplicación móvil. Puede determinar más fácilmente si ignora el error de forma segura o si debe intentar resolver el error con una nueva implementación, cambio de código, comunicación con los clientes u otras acciones.

## Profundice en un solo error [#error-summary]

Para ver detalles sobre un error o falla, seleccione la URL de solicitud de la lista de los 20 errores principales. Desde la página de resumen de errores, puede ver la información de la versión, el atributo de solicitud y el cuerpo de la respuesta, y obtener un desglose de los tipos de error para la URL de solicitud.

## Obtener detalles de la traza [#trace-details]

Monitoreo de móviles capturará los detalles de la respuesta de las solicitudes HTTP que devuelven un código de estado de nivel 400 o 500. Además, el mensaje de error generado desde las aplicaciones de Android incluirá un rastreo del stack.

Para ver detalles sobre una traza de error en la página <DNT>**Errors**</DNT> , seleccione su enlace URL de solicitud. Desde aquí puedes:

* Ver el cuerpo de la respuesta.
* Comparta los detalles del error con otras personas por correo electrónico.

El cuadro de errores también aparece en la página [Resumen](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/mobile-apps-summary-page/) de la aplicación móvil seleccionada.

## Solucionar errores desconocidos o errores de URL [#troubleshoot-unknown-url]

El agente móvil mantiene una lista de tipos de excepciones. En algunos casos, las excepciones personalizadas generadas por la aplicación quedan fuera de esta lista. Cuando esto sucede, `Unknown` puede aparecer en la página móvil <DNT>**Errors**</DNT> .

Si encuentra `Unknown` en su lista de errores y necesita ayuda para investigar qué tipos de excepciones se omiten, obtenga asistencia en [support.newrelic.com](https://support.newrelic.com "El enlace se abre en una ventana nueva.").
