La página UI <DNT>**HTTP requests**</DNT> le ayuda a comprender mejor las solicitudes HTTP asociadas con su aplicación móvil y cómo esas llamadas de red afectan el rendimiento.

## Ver la página de solicitudes HTTP [#find-page]

Para ver la página <DNT>**HTTP requests**</DNT> :

1. Vaya a

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Network > HTTP requests**
   </DNT>

   .

2. Utilice nuestras [funciones UI estándar](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) para buscar tendencias en los gráficos de análisis HTTP.

3. Objetivo específico de solicitud y atributo de respuesta mediante [la agrupación, clasificación y filtrado](#groups-sort-filter) de datos.

## Comprender los datos de la solicitud HTTP [#understand-data]

A continuación se muestran algunos lugares para encontrar los datos de solicitudes HTTP más importantes:

<CollapserGroup>
  <Collapser
    id="response-time-chart"
    title="Gráfico de tiempo de respuesta"
  >
    El gráfico de tiempo de respuesta muestra el rendimiento de las llamadas de red de su aplicación en percentiles. Úselo para comparar el tiempo de respuesta promedio con los percentiles 1, 50 y 99. percentil le permite filtrar el valor atípico que puede estar haciendo que su tiempo de respuesta promedio sea mayor de lo esperado.
  </Collapser>

  <Collapser
    id="errors-network-failures-chart"
    title="Cuadro de errores HTTP y fallas de red"
  >
    Este gráfico muestra las llamadas de red fallidas que está experimentando su aplicación. Seleccione el título del gráfico para ir a la [página<DNT>**HTTP errors**</DNT> ](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/network-errors-http-error-network-failure-analysis)y obtener más detalles sobre los errores y fallas.
  </Collapser>

  <Collapser
    id="total-requests-chart"
    title="Solicitudes totales"
  >
    Ordene por <DNT>**Total requests**</DNT> para identificar qué solicitudes de red se utilizan con más frecuencia. La razón por la que esto puede ser útil es porque las llamadas de red más lentas pueden usarse con poca frecuencia, mientras que las solicitudes utilizadas con más frecuencia pueden ser más dignas de optimización incluso si no son las más lentas.
  </Collapser>
</CollapserGroup>

Para obtener una descripción de la página UI <DNT>**HTTP requests**</DNT> no empresarial, consulte [Solicitudes HTTPlegacy ](#non-enterprise-http-requests).

## Ver y compartir datos de solicitudes HTTP [#insights]

Para ver cualquier gráfico <DNT>**HTTP requests**</DNT> en Insights:

1. Seleccionar

   <Icon name="fe-more-horizontal"/>

   para cualquier gráfico.

2. Seleccione

   <DNT>
     **View query > View in Insights**
   </DNT>

   .

3. Opcional: agregue los datos a un dashboard o [compártalos mediante un vínculo permanente](/docs/data-analysis/user-interface-functions/share-your-data/permalink).

4. Para profundizar en los datos de tu solicitud, consulta [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) evento y [atributo](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes).
