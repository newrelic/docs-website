Cuando la [condición de alerta](/docs/mobile-apps/alert-settings) que estableció para la tasa de errores y el tiempo de respuesta de su aplicación móvil cruza un umbral durante un período prolongado, el agente <InlinePopover type="mobile"/>creará un evento de problema de Advertencia (amarillo) o Crítico (rojo). Si un evento crítico (rojo) ha estado abierto durante varios minutos, el dispositivo móvil enviará una notificación de alerta al usuario que haya sido configurado para recibir notificaciones de alerta. También enviará una notificación cuando se cierren todos los eventos críticos.

Para ver la página de condición de alerta, vaya a <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Alert conditions**</DoNotTranslate>.

## Detalles de la página de condición de alerta

Cada notificación de alerta incluye el ícono de alerta e información resumida. Utilice cualquiera de nuestras [característicasUI ](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)aplicables para profundizar en información detallada.
