La página Resumen proporciona una instantánea operativa de su aplicación móvil. Todos los gráficos, tablas y estadísticas muestran datos para un [período de tiempo](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) y una versión de la aplicación específicos.

La página Resumen captura cinco métricas clave de desempeño:

<table>
  <thead>
    <tr>
      <th width={200}>
        Métrica
      </th>

      <th>
        Descripción
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Aplicación tasa de fallas
      </td>

      <td>
        Número de sesiones fallidas a lo largo del tiempo como porcentaje de todas las sesiones, desglosadas por versión de la aplicación. En la parte superior derecha se muestra el porcentaje promedio de sesiones fallidas, el número total de fallas y el recuento de usuarios únicos afectados por esas fallas durante el período de tiempo.
      </td>
    </tr>

    <tr>
      <td>
        Lanzamiento de la aplicación
      </td>

      <td>
        Monitoreo del número de sesiones de lanzamiento de la aplicación durante la ventana de tiempo.

        El lanzamiento de una aplicación se define por el período de tiempo entre la carga de la aplicación en una pantalla y el momento en que la aplicación se envía a segundo plano.
      </td>
    </tr>

    <tr>
      <td>
        Errores HTTP (fallos de red)
      </td>

      <td>
        Número de solicitudes de red que dan como resultado un `http status code error` (`400` o superior) como porcentaje de todas las solicitudes completadas y el número de `failed network calls` (por ejemplo, un error de conexión) como porcentaje de las solicitudes completadas.
      </td>
    </tr>

    <tr>
      <td>
        Tiempo de respuesta HTTP
      </td>

      <td>
        Tiempo de respuesta promedio de todas las solicitudes http completadas de cada uno de los cinco principales hosts con los que se comunica su aplicación. Los cinco hosts principales se calculan en función del recuento de solicitudes completadas a cada host.
      </td>
    </tr>

    <tr>
      <td>
        Interacción frecuente
      </td>

      <td>
        Las cinco interacciones de usuario ejecutadas con mayor frecuencia en su aplicación.
      </td>
    </tr>
  </tbody>
</table>

## Ver la página Resumen [#summary-page]

1. Vaya a [one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile.
2. Selecciona tu aplicación móvil. La página Resumen se carga de forma predeterminada.

## (Recomendado) Configurar monitoreo de móviles alerta [#alerts]

Recomendamos crear alertas para problemas móviles clave, como solicitudes de red lentas y aumentos en la tasa de fallas. Para configurar fácilmente estas alertas desde la página Resumen:

1. Haga clic en <DNT>**Add recommended conditions**</DNT>.

   <img
     width="80%;"
     style={{align: "left"}}
     title="Mobile Summary page"
     alt="Screenshot of the New Relic mobile Summary page"
     src="/images/mobile_screenshot-full_mobile-alert-conditions.webp"
   />


2. Elija una serie de condiciones de alerta recomendadas.

   <img
     width="70%;"
     style={{align: "left"}}
     title="Mobile recommended alert conditions"
     alt="Mobile recommended alert conditions"
     src="/images/mobile_screenshot-crop_mobile-add-alert.webp"
   />

3. Haga clic en <DNT>**Next**</DNT>.

4. Seleccione una política para recibir notificaciones. Puede seleccionar una existente o [crear una nueva política](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name).

5. Haga clic en <DNT>**Activate conditions**</DNT>.
