      ## Sintaxis [#syntax]

      ### Java [#java]

      ```java
      NewRelic.recordBreadcrumb(
                  string $Name,
                  map<string, object> $eventAttributes
              )
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.recordBreadcrumb(
                  breadcrumbName: String?,
                  attributes: MutableMap<String?, Any?>?
              )
      ```

      ## Descripción [#description]

      Esta llamada crea y registra un evento `MobileBreadcrumb` , que se puede consultar con NRQL y en el [crash trazar el evento](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail). Las rutas de navegación móviles son útiles para el análisis de fallos; Deben crearse para la actividad de la aplicación que pueda ser útil para la resolución de problemas.

      Además de cualquier atributo personalizado que elijas, el evento también tendrá asociado [un atributo de sesión](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list). A diferencia del uso de [`setAttribute`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/create-attribute/), agregar atributos a un evento de ruta de navegación los agrega solo a ese evento; No son atributos de sesión.

      ## Descripción [#description]

      Esta llamada crea y registra un evento `MobileBreadcrumb` , que se puede consultar con NRQL y en el [crash trazar el evento](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail). Las rutas de navegación móviles son útiles para el análisis de fallos; créelos para la actividad de la aplicación que crea que le ayudará a solucionar fallos.

      Además de cualquier atributo personalizado que elijas, el evento también tendrá asociado [un atributo de sesión](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list). A diferencia de usar [`setAttribute`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/create-attribute/), agregar atributos a un evento de ruta de navegación los agrega solo a ese evento; No son atributos de sesión.

      Usar esta llamada tiene el mismo resultado que usar la [llamada`recordCustomEvent` ](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-custom-events/)con `MobileBreadcrumb` como tipo de evento. Para conocer el contexto sobre cómo utilizar esta API, lea sobre el envío de atributos personalizados y eventos:

      * [Objective-C](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/mobile-sdk-api-guide#objc-custom-att-events)
      * [Swift](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/mobile-sdk-api-guide#swift-custom-att-events)

      ## Sintaxis [#syntax]

      ```csharp
      RecordBreadcrumb(string name, Dictionary<string, object> attributes): bool;
      ```

      ## Descripción [#description]

      Esta llamada crea y registra un evento `MobileBreadcrumb` , que se puede consultar con NRQL y en el [crash trazar el evento](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail). Las rutas de navegación móviles son útiles para el análisis de fallos; Deben crearse para la actividad de la aplicación que pueda ser útil para la resolución de problemas.

      Además del atributo personalizado que elijas, el evento también tendrá asociado [un atributo de sesión](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list). A diferencia del uso de [`setAttribute`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/create-attribute/), agregar atributos a un evento de ruta de navegación los agrega solo a ese evento; No son atributos de sesión.
