Las capacidades <InlinePopover type="mobile"/>de New Relic brindan una visibilidad más profunda del rendimiento y la resolución de problemas de su aplicación móvil Android, iOS o híbrida. Puede utilizar monitoreo de móviles para mejorar la experiencia del usuario de su aplicación o examinar HTTP y el rendimiento de la red para colaborar de manera más efectiva con sus equipos de backend.

Una vez que siga nuestro sencillo flujo de instalación, podrá configurar aún más el monitoreo de móviles para adaptarlo a su entorno móvil.

## Identificar oportunidades para acelerar el rendimiento de las aplicaciones [#speed-performance]

* Cree [un panel](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) para consultar, visualizar y compartir datos de rendimiento.
* Identificar [excepciones manejadas](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/introduction-mobile-handled-exceptions) para agilizar el flujo de trabajo.
* Vea tendencias con sesiones, dispositivos, ubicaciones geográficas, sistema operativo, operadores, solicitudes y respuestas, etc. con [perfiles de fallas](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes/#profiles) y [perfiles HTTP](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/http-errors-page/#error-profiles).
* Configure [notificaciones de alerta](/docs/mobile-monitoring/new-relic-mobile/getting-started/alert-information-new-relic-mobile) para tiempo de ejecución, errores, etc.
* Defina [atributo personalizado y evento](/docs/insights/insights-data-sources/custom-events/insert-custom-events-attributes-mobile-data) para obtener detalles adicionales y específicos sobre los datos móviles que más importan a su organización.

## Monitor problemas de backend [#backend]

Puede utilizar las [páginas de red](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/analyze-network-requests-using-mobilerequest-event-data/) en nuestra UI de monitoreo de móviles para identificar problemas que surgen a través de solicitudes HTTP, errores y otros problemas de red. También puede [consultar sus datos](/docs/nrql/nrql-examples/nrql-query-examples-mobile-monitoring/) y crear <InlinePopover type="dashboards"/>personalizado para compartir con equipos relevantes, como los desarrolladores de API backend.
