Un <InlinePopover type="alerts" />bien definido ayuda a notificar a personas o equipos sobre cambios en sus sistemas. Puede emplear cualquiera de nuestras capacidades [de alerta](/docs/alerts-applied-intelligence) en todas las entidades que monitor con New Relic.

Por ejemplo, puede usar [alertas](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts) para administrar la política de alertas y condiciones que se centran en la métrica de las aplicaciones móviles que monitor con New Relic. Después de [definir la condición de alerta](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) para sus aplicaciones móviles, puede ver esa información en la UI de alertas o en otras experiencias UI , como laUI <InlinePopover type="mobile" />.

## Condición de alerta móvil [#conditions]

Utilice cualquiera de las siguientes condiciones y umbrales cuando configure alerta para sus aplicaciones móviles:

<CollapserGroup>
  <Collapser id="execution" title="Tiempo de ejecución">
    Las métricas de tiempo de ejecución incluyen <DNT>**Database**</DNT>, <DNT>**Images**</DNT>, <DNT>**JSON**</DNT>, <DNT>**Network**</DNT> y <DNT>**View loading**</DNT>. Defina el umbral para estas métricas seleccionando un tiempo y una duración.
  </Collapser>

  <Collapser id="errors" title="Errores">
    El error métrico incluye <DNT>**Network error percentage**</DNT> y <DNT>**Status error percentage**</DNT>. Defina el umbral para estas métricas seleccionando un porcentaje de error y una duración.
  </Collapser>

  <Collapser id="crash" title="Informe de fallos">
    Es posible que desee recibir una notificación cuando <DNT>**crash rate**</DNT> de su aplicación supere un determinado umbral. Defina el umbral para estas métricas seleccionando un porcentaje de error y una duración.

    <Callout variant="tip">
      Reciba notificaciones sobre el exceso de tasa de fallas de su aplicación tan pronto como suceda configurando [un canal de notificación](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts), incluidos PagerDuty y Slack.
    </Callout>
  </Collapser>

  <Collapser id="custom" title="Costumbre">
    Cree su propio <DNT>**custom metric**</DNT> para satisfacer necesidades de alerta específicas. Agregue un nombre para la métrica personalizada y defina su propio umbral.
  </Collapser>
</CollapserGroup>

## Ver políticas y condiciones [#policy]

Para ver la política de alertas y la información sobre las condiciones de una aplicación móvil específica:

1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt; Alert conditions**</DNT>.
2. Desde la página <DNT>**Alert conditions**</DNT> , utilice las herramientas disponibles para buscar, ordenar, ver o actualizar la condición de alerta y sus políticas asociadas.

## Ver eventos y actividades [#activity]

Para ver información resumida sobre [eventos](/docs/alerts/overview#event) y otra actividad directamente desde la interfaz de monitoreo móvil:

1. Vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile**</DNT>.
2. Desde el índice, pasa el cursor sobre el [estado de salud codificado por colores](/docs/alerts/overview#health-status) de la entidad, selecciona un enlace de la lista <DNT>**Mobile activity**</DNT> o selecciona una aplicación móvil para ver detalles adicionales.

Por ejemplo, si ocurre un evento de alerta <DNT>**Critical**</DNT>:

* El indicador de estado de salud se vuelve rojo en el índice del móvil y en la aplicación seleccionada.
* El color de fondo de varios gráficos cambia a rosa.
* En su lista de aplicaciones móviles monitoreadas, la sección <DNT>**Application activity**</DNT> muestra eventos de alerta <DNT>**Warning**</DNT> (amarillo) y <DNT>**Critical**</DNT> (rojo) según corresponda.

Para obtener más información sobre un evento de alerta, pase el mouse por encima o seleccione cualquier área rosa en un gráfico.

## Ver eventos de alerta [#alert-events]

<Callout variant="important">
  En julio de 2020, hicimos la transición de los eventos de alerta para aplicaciones de navegador, aplicaciones móviles y monitores sintéticos a un nuevo formato en <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>. Para obtener más información, consulta la documentación de [alertas](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai).
</Callout>

Si una condición de alerta tiene [umbrales](/docs/alerts/overview#alert-threshold) configurados para eventos de alerta <DNT>**Warning**</DNT> (amarillo) o <DNT>**Critical**</DNT> (rojo), el [estado de salud](/docs/alerts/overview#health-status) codificado por colores para una entidad de producto cambiará para indicar un evento de alerta. Puedes ver los eventos de alerta directamente desde la página de la aplicación móvil en New Relic:

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Mobile**</DNT>.
2. Seleccione una aplicación móvil y luego revise su <DNT>**Open alert events**</DNT>.
