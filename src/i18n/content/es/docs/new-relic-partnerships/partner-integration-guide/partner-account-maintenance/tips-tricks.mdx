A continuación se ofrecen algunos consejos y trucos para crear y eliminar cuentas de socios de New Relic.

## Creación y eliminación de cuentas [#account]

Cuando crea una cuenta a través de la API de New Relic Partner, el sistema verifica la cadena enviada como nombre de cuenta para asegurarse de que sea única. Si no es único, el sistema agregará un número a la cadena enviada para garantizar la unicidad; por ejemplo, `account-name_1`.

Cuando se cancela una cuenta, no se elimina de la base de datos de New Relic. Más bien, se desactiva eliminando todas las suscripciones de la cuenta. Si intenta restablecer una cuenta llamando a la API de creación de cuenta usando el mismo nombre de cuenta, el sistema verificará la unicidad, determinará que el nombre no es único y creará una nueva cuenta con un número adjunto. En la mayoría de los casos esto no es lo que se desea.

Para evitar este problema, como parte de la creación de la cuenta, almacene el número `account_id` con el nombre de la cuenta. Al volver a crear una cuenta, verifique la existencia de esta identificación. Si se detecta, en lugar de usar la API de creación de cuenta, use la API de cambio de suscripción para agregar una nueva suscripción a la cuenta. Agregar una suscripción activa a la cuenta reactivará la cuenta.

## Cuenta de propietario de asociación [#partner-owner]

La cuenta de propietario de la asociación no forma parte de la asociación. Es dueño de la sociedad. Los intentos de tratarlo como si estuviera en la sociedad fracasarán. Esto incluye pero no se limita a:

* Intentar realizar SSO en la cuenta del propietario de la asociación utilizando el secreto compartido de la asociación
* Esperar que el precio de la asociación se aplique a la cuenta del propietario de la asociación
