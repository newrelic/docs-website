Nuestra API de asociación permite a los socios New Relic y a las cuentas New Relic configuradas como [asociaciones de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) administrar cuentas, usuarios y configuraciones relacionadas con la suscripción.

## Requisitos [#requirements]

<DNT>**There are several major limitations to using the Partnership API.**</DNT> Su organización de New Relic no debe utilizar esta API a menos que su representante de New Relic le haya indicado específicamente que la utilice. Lea toda la sección de requisitos para comprender las limitaciones.

La API de asociación puede ser utilizada por dos tipos de cuentas de New Relic : [socios](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) (proveedor de servicios administrado y revendedores New Relic ) y [asociaciones de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) (organizaciones más grandes a las que se les ha dado acceso a nuestra estructura de cuentas de asociación).

La API de llamada de asociación requiere autenticación tanto con la clave de API REST de su cuenta de propietario de asociación como con su ID de socio.

Las cuentas que son socios genuinos New Relic (proveedor de servicios administrado o revendedores) no tienen restricciones en el uso de la API de asociación. Las organizaciones New Relic constituidas como [asociaciones de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) tienen las siguientes restricciones.

### Restricciones de asociación con clientes [#customer-restrictions]

Si su organización New Relic está configurada como una [asociación de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships), existen algunas restricciones si su organización está en nuestro [modelo de precios](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) más nuevo o en nuestro [modelo de usuario](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) más nuevo. Detalles:

* <DNT>
    **User model.**
  </DNT>

  Si su organización New Relic tiene un usuario en [nuestro modelo de usuario más nuevo](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), ya no puede usar la API de llamada de Partnership que administra cuentas o usuarios. Aquí hay detalles sobre lo que usarías en su lugar:

  * En lugar de utilizar el objeto API [usuario de](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-user-object) Partnership ,[administraría el usuario a través de NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).
  * En lugar de utilizar el [objeto de cuenta](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-account-object) de Partnership API o [el objeto de cuenta infantil](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-sub-account-object), [administraría las cuentas a través de NerdGraph](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph).

* <DNT>
    **Pricing model.**
  </DNT>

  Si utiliza nuestro [modelo de precios basado en el uso](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) más nuevo, no puede utilizar la API de llamada que rige nuestro [modelo de precios original](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans). Los impactos de la API son:

  * No puedes usar el [objeto de suscripción.](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-subscription-object/)
  * No puedes usar la [API de aprovisionamiento de NerdGraph](/docs/apis/nerdgraph/tutorials/provisions-your-subscriptions-nerdgraph/).

Para determinar su modelo de precios o modelo de usuario, consulte [Descripción general de los cambios en el modelo de precios y de usuario](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model).

## Cosas que puedes hacer [#functions]

A continuación se ofrece una descripción general de lo que puede hacer con esta API:

<DNT>
  **Account:**
</DNT>

Algunas organizaciones que cumplan con los [requisitos](#requirements) pueden utilizar el [objeto de cuenta](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-account-object/) para crear y actualizar cuentas. Puedes hacer lo siguiente con él:

* Crear nuevo
* Espectáculo
* Actualizar
* Cancelar
* Mostrar uso
* Establecer administrador principal
* Establecer suscripción

También hay un [objeto de cuenta infantil](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-sub-account-object) para crear cuentas infantiles.

<DNT>
  **Users:**
</DNT>

Alguna organización que [cumpla con los requisitos](#customer-restrictions) puede utilizar el [objeto usuario](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-user-object) para:

* Agregar acceso de usuario a la cuenta
* Eliminar el acceso de usuario de la cuenta

<DNT>
  **Subscription:**
</DNT>

Las organizaciones que [cumplen con los requisitos](#customer-restrictions) y se encuentran en nuestro modelo de precios de producto original pueden usar el [objeto de suscripción](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-subscription-object) para configurar varios rasgos relacionados con la suscripción.

### Requisitos de contraseña [#password]

Las contraseñas pasadas para la creación de cuentas tienen estos requisitos:

* De 8 a 50 caracteres de longitud
* Solo contiene letras, números o caracteres especiales.
* No puede contener espacios
* Debe contener al menos 1 letra
* Debe contener al menos 1 número o carácter especial
