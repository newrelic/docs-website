Los socios que incluyan New Relic en su consola pueden "diseñar" New Relic con su propia apariencia, incluidos diseños, menús y subdominios personalizados.

## Encabezados y pies de pagina [#headers-footers]

Para implementar encabezados y pies de página:

1. Cree una página de encabezado y/o pie de página, que consta de contenido incluido en la etiqueta &lt;div>. Asegúrese de que las páginas no importen ningún CSS ni contengan ningún CSS en línea.

2. Tome cualquier CSS necesario para el encabezado y el pie de página y póngalo a disposición en un archivo CSS separado. Por ejemplo, anteponga una cadena personalizada como `yourcompany_` a los distintos nombres de estilo, para evitar anular las definiciones de estilo estándar de New Relic.

3. Publique los tres archivos en ubicaciones de acceso público alojadas por el socio. Utilice https para que sus clientes no vean advertencias de contenido mixto. Por ejemplo:

   ```
   https://yourcompany.com/newrelic_header.htm
   https://yourcompany.com/newrelic_footer.htm
   https://yourcompany.com/newrelic_styles.css
   ```

4. Ingrese las URL para el encabezado, pie de página y ubicaciones de la página CSS en los campos apropiados en la consola New Relic <DNT>**Partnership**</DNT> .

<Callout variant="tip">
  Para ver los encabezados, log sesión en una de las cuentas de la asociación.
</Callout>

## Diseños y estilos [#styles]

Los diseños y estilos de encabezado y pie de página se cargan y almacenan en caché en los servidores recolectores de New Relic para su inserción en cada salida de página. Las actualizaciones realizadas en el diseño del sitio del socio se actualizan automáticamente cada hora.

Se pueden agregar al diseño enlaces personalizados al sitio web del socio. Si el inicio de sesión único está habilitado, el socio puede inyectar información específica de los clientes en la sesión del usuario New Relic para que la utilice el diseño. Para obtener más información, consulte [Inicio de sesión único y control de acceso](/docs/accounts-partnerships/partnerships/partner-integration-guide/single-sign-access-control).

## Subdominio

Opcionalmente, los socios pueden mostrar un subdominio personalizado; por ejemplo, <DNT>**yourcompany.newrelic.com**</DNT>, para clientes que acceden a cuentas de New Relic. Para organizar esto, comuníquese con el equipo de habilitación empresarial de New Relic.
