Las claves [API de asociación](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) se utilizan para autenticar llamadas con cuentas de asociación en New Relic. Una distinción importante al trabajar con clave de API es el propietario de la cuenta versus el propietario de la asociación. El propietario de la sociedad es una forma especial de cuenta que se utiliza para administrar datos sobre la cuenta.

## Requisitos [#requirements]

<DoNotTranslate>
  **The Partnership API is not available to your organization unless you're specifically instructed by your New Relic representative that you should use it.**
</DoNotTranslate>

Para conocer los requisitos, lea [Introducción a la API de asociación](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/).

## Clave de licencia

Esta es la clave devuelta en el XML al crear una cuenta. Lo utiliza el agente para identificar la cuenta.

## Clave de API de socio [#partner-key]

Al utilizar la API de asociación, debe utilizar la clave de API REST de la cuenta de propietario de asociación. Esta es la clave utilizada en la API de llamada donde ves `x-api-key: <api key>`.

## Localiza tu clave de API [#locating-keys]

Para ver o regenerar sus claves, log en el sitio New Relic con las credenciales de propietario de su asociación y luego vaya a <DoNotTranslate>**Partnerships > Edit settings**</DoNotTranslate>. Las claves se enumeran en la parte inferior de la página.

<Callout variant="caution">
  Las claves regeneradas no se pueden revertir.
</Callout>

## Mejores prácticas de seguridad [#security]

<DoNotTranslate>**Recommendation:**</DoNotTranslate> Las claves utilizadas para acceder al sitio New Relic deben regenerarse periódicamente como mejores prácticas de seguridad. El cambio de regeneración de una clave se produce inmediatamente.

Si codifica las claves en su código y desea implementar una nueva versión de integración con la nueva clave en producción, asegúrese de coordinar entre la regeneración de claves y la actualización del sitio. Esto ayudará a minimizar el tiempo de inactividad. En esta situación, considere recuperar dinámicamente estas claves utilizando la API Mostrar cuenta.
