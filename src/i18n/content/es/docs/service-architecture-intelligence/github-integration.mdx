  Esta característica se proporciona actualmente como parte de un programa de vista previa de conformidad con nuestras [políticas de prelanzamiento](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).

La integración de GitHub con New Relic importa usuarios y equipos de su cuenta de GitHub para mejorar la visibilidad de su [capacidad de inteligencia de arquitectura de servicios](/docs/service-architecture-intelligence/getting-started) en la plataforma New Relic .

Esta integración agiliza el proceso de incorporación de usuarios y equipos, simplifica la gestión de roles y aclara la propiedad de las entidades, lo que reduce la configuración, el tiempo y el esfuerzo. Esta integración de monitoreo de IA tiene como objetivo mejorar la gestión y visibilidad de [catálogos](/docs/service-architecture-intelligence/catalogs/catalogs) y [cuadros de mando](/docs/service-architecture-intelligence/scorecards/getting-started) dentro de New Relic. Para obtener más información, consulte [Capacidad de inteligencia de arquitectura de servicio](/docs/service-architecture-intelligence/getting-started).

**Prerrequisitos:**

* Inicie sesión en su cuenta de GitHub que planea integrar con New Relic.
* Debe ser el administrador de la organización o el administrador del dominio de autenticación.

**Para configurar la integración de GitHub:**

1. Vaya a **[one.newrelic.com &gt; + Integration &amp; Agents &gt; GitHub integration](https://one.newrelic.com/marketplace/install-data-source?state=9306060d-b674-b245-083e-ff8d42765e0d)**.

2. En la pantalla **Select an account** , mantenga la selección de cuenta predeterminada y haga clic en **Continue**.

3. En la pantalla de **Integration setup** :

   1. Para conectarse con su cuenta de GitHub, haga clic en **Install**. La observabilidad de New Relic se abre en GitHub Marketplace.

   2. Complete la instalación de la aplicación dentro de su organización GitHub para recuperar datos relacionados con el usuario, los equipos y el repositorio. Una vez completada la instalación, serás redirigido a la plataforma New Relic.

   3. Haga clic en **First time sync**.

   4. *(Opcional)* Haga clic en **On-demand sync** para sincronizar los datos manualmente.

      <Callout variant="tip" title="consejo">
        Puede sincronizar manualmente los datos una vez cada 4 horas. El botón de **On-demand sync** permanece deshabilitado si la sincronización se realizó dentro de las 4 horas anteriores.
      </Callout>

   5. Luego de ver el mensaje de sincronización iniciada, haga clic en **Continue**. La pantalla de **GitHub Integration** mostrará el recuento de equipos y repositorio, actualizar cada 5 segundos. Espere unos minutos para que se complete la importación de todos los datos.

      <img title="GitHub integration" alt="GitHub integration" src="/images/github-integration-progress.webp" />

4. En la pantalla de **GitHub integration** :

   * Haga clic en **Go to Teams** para ver la información de los equipos importados en [Teams](/docs/service-architecture-intelligence/teams/teams).

   <Callout variant="tip" title="consejo">
     You can auto-assign GitHub repositories to their teams by creating a `teamOwningRepo` custom property in GitHub. Create the custom property at the organization level and assign a value for the custom property at the repository level. Additionally, you can set up a custom property for multiple repositories at the organization level simultaneously. The integration will automatically assign the repositories to the respective teams based on the custom property. For more information on creating custom properties, refer to the [GitHub documentation](https://docs.github.com/en/organizations/managing-organization-settings/managing-custom-properties-for-repositories-in-your-organization).
   </Callout>
