La biblioteca de plantillas de <DNT>Scorecards</DNT> proporciona una forma optimizada de abordar los casos de uso de ingeniería y negocios. Al utilizar las mejores prácticas de grandes empresas, la biblioteca ofrece soluciones para desafíos de ingeniería comunes.

Cada plantilla de la biblioteca es totalmente personalizable para cumplir con los estándares de su organización. Puede modificar las consultas, las reglas y los umbrales para adaptar el <DNT>Scorecards</DNT> a sus necesidades.

Algunas plantillas etiquetan sus reglas con niveles de madurez (de `L0` a `L3`).

## Plantillas de cuadro de mando

<CollapserGroup>
  <Collapser
    id="product-readiness"
    title={<DNT>Preparación para producción</DNT>
    }
  >
    **Propósito:** Valida si sus servicios están listos para la implementación de producción verificando los objetivos definidos, la configuración de alertas y la actividad de implementación reciente.

    **Importancia:** ayuda a encontrar brechas de preparación antes de desplegar en producción.

    **Prerrequisitos:**

    * [Administración a nivel de servicio](/docs/service-level-management/intro-slm)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [Alertas New Relic](/docs/alerts/overview)
    * [Seguimiento de cambios](/docs/change-tracking/overview)

    **Reglas clave:**

    * <DNT>**Defined SLOs**</DNT>: establece objetivos claros para el rendimiento y la confiabilidad.
    * <DNT>**APM service alerts configuration**</DNT>: comprueba que las alertas estén configuradas para la detección rápida de problemas.
    * <DNT>**Recent deployments**</DNT>: verifica los despliegues exitosos durante el último mes.
    * <DNT>**Runbook availability**</DNT>: confirma que existen runbooks para las condiciones de alerta activadas en el último mes.
  </Collapser>

  <Collapser
    id="aws-reliability"
    title={<DNT>Confiabilidad de AWS</DNT>
    }
  >
    **Propósito:** evalúa la confiabilidad frente a los estándares de AWS para seguridad y redundancia en operaciones cloud.

    **Importancia:** ayuda a encontrar brechas de confiabilidad y seguridad en su infraestructura de la nube.

    **Prerrequisitos:** [Monitoreo de AWS de New Relic](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations)

    **Reglas clave:**

    * <DNT>**EFS Encrypted check**</DNT>: confirma el cifrado de los volúmenes de Elastic File System (EFS).
    * <DNT>**RDS Instance check retention period**</DNT>: comprueba los períodos de retención de copias de seguridad para el servicio de base de datos relacional (RDS).
    * <DNT>**RDS DeletionProtection check**</DNT>: verifica la protección contra eliminación.
    * <DNT>**RDS MultiAZ presence check**</DNT>: comprueba el despliegue en múltiples zonas de disponibilidad (MultiAZ).
    * <DNT>**RDS AutoMinorVersionUpgrade check**</DNT>: confirma las actualizaciones automáticas de versiones menores.
  </Collapser>

  <Collapser
    id="dora-metrics"
    title={<DNT>Métrica de investigación y evaluación DevOps (DORA)</DNT>
    }
  >
    **Propósito:** Evalúa la asignación de recursos y las mejoras de procesos empleando indicadores de rendimiento clave.

    **Importancia:** Cuantifica el rendimiento de la entrega de software, lo que ayuda a la toma de decisiones estratégicas.

    **Prerrequisitos:**

    * [Seguimiento de cambios](/docs/change-tracking/overview)
    * [Alertas New Relic](/docs/alerts/overview)
    * [Integración con GitHub](/docs/service-architecture-intelligence/github-integrations/github-cloud-integration)

    **Reglas clave:**

    * <DNT>**Deployment frequency**</DNT>: rastrea las tasas de despliegue exitosas.
    * <DNT>**Mean time to recovery**</DNT>: mide el tiempo de recuperación ante fallas.
    * <DNT>**Change failure rate**</DNT>: rastrea el porcentaje de cambios que resultan en fallas.
    * <DNT>**Lead time for changes**</DNT>: mide el tiempo transcurrido desde la confirmación del código hasta el despliegue.
  </Collapser>

  <Collapser
    id="tag-compliance"
    title={<DNT>Cumplimiento de etiqueta</DNT>
    }
  >
    **Propósito:** establece y hace cumplir prácticas de etiquetado consistentes para los recursos dentro de su organización.

    **Importancia:** las etiquetas consistentes le ayudan a filtrar y agrupar servicios en vistas como mapas y catálogos.

    **Requisitos previos:** [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)

    **Reglas clave:**

    * <DNT>**Team Tag Exists**</DNT>: comprueba que los servicios lleven una etiqueta del equipo propietario para una clara propiedad y responsabilidad.
    * <DNT>**Environment Tag Exists**</DNT>: comprueba que los servicios lleven una etiqueta de entorno coherente para filtrar y agrupar.
    * <DNT>**APM Criticality is set**</DNT>: comprueba que los servicios tengan una etiqueta de criticidad para priorizar por importancia comercial.
  </Collapser>

  <Collapser
    id="observability-maturity-intelligent-observability"
    title={<DNT>Madurez de observabilidad: observabilidad inteligente</DNT>
    }
  >
    **Propósito:** establece una estrategia de etiqueta integral y visibilidad completa de las dependencias y relaciones del servicio.

    **Importancia:** resalta las brechas en la cobertura de etiquetas, relaciones, traza y sintético para que pueda abordarlas.

    **Prerrequisitos:**

    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [Monitoreo sintetico](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring)

    **Reglas clave:**

    * <DNT>**(L0) Team Tag Coverage**</DNT>: identifica las entidades a las que les falta una etiqueta `team`.
    * <DNT>**(L0) Environment Tag Coverage**</DNT>: identifica las entidades a las que les falta una etiqueta `environment`.
    * <DNT>**(L0) Uninstrumented Entities**</DNT>: marca las entidades de servicio HTTP y base de datos con relaciones desconocidas que necesitan instrumentación adicional.
    * <DNT>**(L0) Distributed Trace Coverage**</DNT>: mide el porcentaje de entidades de APM que emiten datos de rastreo distribuido.
    * <DNT>**(L0) Synthetic Coverage**</DNT>: mide el porcentaje de entidades de APM cubiertas por monitores sintéticos.
  </Collapser>

  <Collapser
    id="observability-maturity-business-uptime"
    title={<DNT>Madurez de la observabilidad: tiempo de actividad del negocio</DNT>
    }
  >
    **Propósito:** desarrolla una estrategia integral de alertas y un enfoque de gestión de incidentes para la confiabilidad del servicio.

    **Importancia:** resalta las brechas de alerta, el ruido y la cobertura de SLI para que pueda priorizar los servicios críticos.

    **Prerrequisitos:**

    * [Alertas New Relic](/docs/alerts/overview)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [Monitoreo de infraestructura](/docs/infrastructure/introduction-infra-monitoring)
    * [Administración a nivel de servicio](/docs/service-level-management/intro-slm)

    **Reglas clave:**

    * <DNT>**(L1) Infrastructure Alert Coverage**</DNT>: identifica los hosts y los pods de Kubernetes que carecen de cobertura de alertas.
    * <DNT>**(L1) Service Delivery Alert Coverage**</DNT>: identifica los servicios, el navegador, los dispositivos móviles y los monitores sintéticos que carecen de cobertura de alerta.
    * <DNT>**(L1) Critical Alert Coverage**</DNT>: marca la dependencia excesiva de las condiciones de alerta crítica que pueden causar exceso de alertas.
    * <DNT>**(L1) Alert Noise**</DNT>: identifica las políticas que generan un alto volumen de incidentes.
    * <DNT>**(L2) Service Level Coverage**</DNT>: comprueba si las entidades tienen un indicador de nivel de servicio (SLI) definido.
    * <DNT>**(L2) Alerts Mean Time To Close**</DNT>: evalúa el tiempo necesario para cerrar incidentes, con el objetivo de resolverlos en 30 minutos.
    * <DNT>**(L2) APM Criticality Tag Coverage**</DNT>: identifica las entidades a las que les falta una etiqueta `criticality`.
    * <DNT>**(L3) Service Level Attainment**</DNT>: evalúa la puntuación de cumplimiento para cada SLI definido, utilizando un umbral predeterminado del 95 %.
  </Collapser>

  <Collapser
    id="observability-maturity-engineering-excellence"
    title={<DNT>Madurez de la observabilidad: excelencia en ingeniería</DNT>
    }
  >
    **Propósito:** enfatiza el uso eficiente de los recursos y las prácticas de seguridad sólidas mediante el monitoreo del uso de la CPU y la memoria, la integración del seguimiento de cambios y la detección de vulnerabilidades.

    **Importancia:** cumplir con los estándares de recursos, seguimiento de cambios y seguridad le ayuda a ejecutar una infraestructura eficiente y segura.

    **Prerrequisitos:**

    * [Monitoreo de infraestructura](/docs/infrastructure/introduction-infra-monitoring)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [Seguimiento de cambios](/docs/change-tracking/overview)
    * [Security RX](/docs/vulnerability-management/getting-started/overview)

    **Reglas clave:**

    * <DNT>**(L1) CPU Utilization**</DNT>: comprueba el percentil 95 del uso de CPU para las entidades de infraestructura frente a las mejores prácticas de eficiencia.
    * <DNT>**(L1) Memory Utilization**</DNT>: comprueba el percentil 95 del uso de memoria para la entidad de infraestructura frente a las mejores prácticas de eficiencia.
    * <DNT>**(L2) Change Tracking**</DNT>: comprueba si hay eventos de seguimiento de cambios en sus entidades de APM.
    * <DNT>**(L3) Service Vulnerabilities**</DNT>: comprueba el porcentaje de entidades de APM con vulnerabilidades detectadas.
  </Collapser>

  <Collapser
    id="observability-maturity-digital-experience"
    title={<DNT>Madurez de la observabilidad: experiencia digital</DNT>
    }
  >
    **Propósito:** monitorea el rendimiento del frontend y la interacción del usuario a través de errores de JavaScript, errores HTTP 5xx y métricas de Métricas web principales.

    **Importancia:** cumplir con los estándares de errores de frontend y Core Web Vitals le ayuda a mantener una sólida experiencia del usuario.

    **Prerrequisitos:**

    * [Monitoreo del navegador](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)
    * [Administración a nivel de servicio](/docs/service-level-management/intro-slm)

    **Reglas clave:**

    * <DNT>**(L1) JS Error Rate**</DNT>: comprueba el porcentaje de entidades de navegador con errores de JavaScript.
    * <DNT>**(L1) Service Error Rate**</DNT>: comprueba el porcentaje de servicios de APM que reportan errores 5xx en las requests HTTP.
    * <DNT>**(L2) Core Web Vitals: Largest Contentful Pane (LCP)**</DNT>: marca las entidades cuyo LCP del percentil 75 supera los 2.5 segundos.
    * <DNT>**(L2) Core Web Vitals: Interaction to Next Paint (INP)**</DNT>: marca las entidades cuyo INP del percentil 75 supera los 200 milisegundos.
    * <DNT>**(L2) Core Web Vitals: Cumulative Layout Shift (CLS)**</DNT>: marca las entidades cuyo CLS del percentil 75 supera 0.1.
    * <DNT>**(L3) Service Level Attainment**</DNT>: evalúa la puntuación de cumplimiento para cada SLI definido, utilizando un umbral predeterminado del 95 %.
  </Collapser>

  <Collapser
    id="security"
    title={<DNT>Seguridad</DNT>
    }
  >
    **Propósito:** evalúa el riesgo de vulnerabilidades en los servicios de APM y la infraestructura.

    **Importancia:** sacar a la luz las vulnerabilidades críticas y de alta gravedad, los CVE de ransomware activos y la probabilidad de explotación ayuda a priorizar la corrección.

    **Prerrequisitos:**

    * [Security RX](/docs/vulnerability-management/getting-started/overview)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [Monitoreo de infraestructura](/docs/infrastructure/introduction-infra-monitoring)

    **Reglas clave:**

    * <DNT>**APM: Under 3 Critical Severity**</DNT>: comprueba que un servicio de APM tenga menos de 3 vulnerabilidades de gravedad crítica.
    * <DNT>**APM: Under 5 High Severity**</DNT>: comprueba que un servicio de APM tenga menos de 5 vulnerabilidades de alta gravedad.
    * <DNT>**APM: Active Ransomware**</DNT>: marca los CVE informados que tienen una campaña de ransomware activa.
    * <DNT>**APM: Exploit Probability**</DNT>: marca las vulnerabilidades críticas o de alta gravedad que probablemente sean explotadas (percentil 95+ de EPSS).
    * <DNT>**INFRA: Under 3 Critical Severity**</DNT>: comprueba que un host o contenedor tenga menos de 3 vulnerabilidades de gravedad crítica.
    * <DNT>**INFRA: Under 5 High Severity**</DNT>: comprueba que un host o contenedor tenga menos de 5 vulnerabilidades de alta gravedad.
    * <DNT>**INFRA: Active Ransomware**</DNT>: marca los CVE informados que tienen una campaña de ransomware activa.
    * <DNT>**INFRA: Exploit Probability**</DNT>: marca las vulnerabilidades críticas o de alta gravedad que probablemente sean explotadas (percentil 95+ de EPSS).
  </Collapser>

  <Collapser
    id="ai-quality-governance"
    title={<DNT>Calidad y gobernanza de IA</DNT>
    }
  >
    **Propósito:** mide la calidad operativa y la gobernanza de las respuestas de IA en términos de seguridad, rentabilidad y cumplimiento del modelo.

    **Importancia:** cumplir con los estándares de calidad y gobernanza de la IA ayuda a mantener el uso de la IA de manera responsable y rentable.

    **Prerrequisitos:** [monitoreo de IA](/docs/ai-monitoring/intro-to-ai-monitoring)

    **Reglas clave:**

    * <DNT>**LLM Runtime Error Rate**</DNT>: comprueba que las aplicaciones habilitadas para LLM mantengan una baja tasa de errores operativa.
    * <DNT>**Cost Efficiency - Average tokens per assistant response**</DNT>: comprueba que el recuento promedio de tokens por respuesta del asistente se mantenga dentro del presupuesto.
    * <DNT>**Model Governance - Approved model usage for assistant responses**</DNT>: comprueba que solo se utilicen modelos aprobados para las respuestas del asistente en producción.
  </Collapser>
</CollapserGroup>
