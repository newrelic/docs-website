Su infraestructura requiere inspección y mantenimiento periódicos. Al igual que un urbanista que diseña una nueva carretera o un puente, las decisiones sobre lo que necesita su sistema deben surgir de una observación cuidadosa de las limitaciones existentes. Para las ciudades, esto equivale a agregar nuevos carriles a las vías principales cuando aumenta la congestión del tráfico por la mañana o por la noche. Para una pila de tecnología, esto parece una serie de decisiones de recursos que afectan el rendimiento de sus API, servicios y aplicaciones.

Con nuestro agente de infraestructura, puede recopilar periódicamente datos métricos y de eventos para saber si su infraestructura está escalando adecuadamente. ¿Qué procesos afectan el uso de la CPU y el disco? ¿Cómo se comporta una aplicación justo antes de un pico de CPU? ¿Sus hosts están reaccionando ante una falla en la base de datos? La rapidez con la que pueda responder estas preguntas es la diferencia entre una interrupción total, un incidente menor o un escalado automático sin problemas.

## Objetivos [#objectives]

Esta serie de tutoriales le guiará en el uso de datos de infraestructura para solucionar diferentes escenarios de incidentes. Si bien New Relic ofrece una integración de suite completa para monitor la nube, entornos en contenedores y varias máquinas virtuales, esta serie es específica para los datos reportados con un agente de infraestructura que no tiene integración adicional.

Dicho esto, este documento explica cómo puede instrumentar completamente su sistema. Vas a:

* Instale el agente de infraestructura con nuestra instalación guiada
* Comprenda las recomendaciones de la instalación guiada para que pueda tomar una decisión informada sobre la instrumentación.
* Aprenda sobre la integración de infraestructura

## Instalar el agente de infraestructura [#install]

Cuando instala el agente de infraestructura, está instalando un software que puede integrarse con cualquier cantidad de tecnologías. El agente recopila los datos a los que tiene acceso y luego los informa a New Relic, donde puede explorarlos a través de nuestra UI. Monitorear su infraestructura le permite responder a incidentes con más claridad y dirección, para que pueda pasar de la reacción a la proacción en su respuesta a incidentes.

<Steps>
  <Step>
    ## Comience con nuestra instalación guiada

    Para instalar el agente de infraestructura, recomendamos utilizar nuestra [instalación guiada](https://one.newrelic.com/nr1-core?state=a36f15a1-c5ce-528c-2ec2-8bcaacc40c03). Nuestra instalación guiada es un instalador CLI que permite a New Relic inspeccionar y luego hacer recomendaciones sobre instrumentación adicional para su sistema. Estas recomendaciones son para nuestro agente y cualquier integración relevante para su sistema.
  </Step>

  <Step>
    ## Instrumenta tu sistema con agente

    La fidelidad de sus datos depende en gran medida de lo que haya elegido informar a New Relic. Es la diferencia entre comprar una cámara de apuntar y disparar que puede capturar la esencia de una escena sin detalles, o una cámara DSLR profesional que devuelve la vida a una escena.

    Cuando recomendamos que instrumente completamente su sistema, nos referimos a que instale un agente o una integración que pueda capturar datos en cualquier servidor, host en la nube, máquina virtual, servicio o aplicación. Cuando la instalación guiada le hace recomendaciones, le informa que hay un agente (o integración) disponible para capturar datos para esa fuente potencial.

## ¿Cómo se relacionan agente e integración entre sí? [#agent-integration]

Una vez que haya instalado el agente de infraestructura, puede integrarlo con su proveedor de nube, tipo de máquina virtual u orquestación de contenedores mediante integración. La integración le permite personalizar cómo se habilita el agente de infraestructura para que solo instale lo que necesita. Por ejemplo, si utiliza Microsoft Azure pero no AWS, puede ignorar la integración de AWS y, en su lugar, instalar solo la integración de Microsoft Azure.

Si optó por no participar en una instalación con nuestra instalación guiada pero luego cambió de opinión, puede consultar nuestras ofertas de integración en nuestro documento [Introducción a las integraciones de infraestructura](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations) .
