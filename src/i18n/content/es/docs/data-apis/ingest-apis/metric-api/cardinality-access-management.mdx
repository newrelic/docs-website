Como administrador de dominio de autenticación, realice los siguientes pasos para otorgar acceso a un usuario para gestionar la cuenta y la cardinalidad métrica. Consulte con el usuario sobre sus necesidades específicas y realice la configuración en consecuencia. Para obtener más información sobre la gestión de usuarios, consulte [el tutorial de gestión de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/).

<Steps>
  <Step>
    ### Agregar licencias a un rol personalizado [#permissions-to-custom-role]

    [Cree un nuevo rol personalizado](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) con las siguientes licencias o agréguelos a un rol existente. Consulte con el usuario para comprender el nivel de acceso requerido.

    * `Manage account cardinality`:Ajusta la cotización de cardinalidad de la cuenta.

      * `View`:Puede ver la cotización de cardinalidad de la cuenta.
      * `Modify`:Puede actualizar la cotización de cardinalidad de la cuenta.

    * `Manage metric cardinality`:Ajusta la cotización de cardinalidad de la métrica.

      * `View`:Puede ver la cotización de cardinalidad de la métrica.
      * `Modify`:Puede actualizar la cotización de cardinalidad de la métrica.

    * `Manage pruning rules`:Gestionar reglas de poda para métrica.

      * `View`:Puede ver las reglas de poda de la métrica.
      * `Modify`:Puede crear y eliminar reglas de poda para la métrica.

    * `Customer limits management`:Ajuste los límites de las diferentes capacidades en la cuenta, incluida la cardinalidad.

      * `View`:Puede ver los límites disponibles para las capacidades.
      * `Modify`:Puede ajustar los límites de las capacidades.
  </Step>

  <Step>
    ### Agregar rol de usuario y cuenta a un grupo [#group-access]

    Para la gestión de cardinalidad, [cree un nuevo grupo](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#group-access) o gestione un grupo existente de la siguiente manera:

    1. Vaya a **[New Relic Administration](https://one.newrelic.com/admin-portal) &gt; Access Management &gt; Groups**.
    2. Haga clic en <DNT>**Create a new group**</DNT> o seleccione <DNT>**Manage group access**</DNT> de la lista. <Icon name="fe-more-horizontal" /> Menú de un grupo existente.
    3. Para crear un grupo nuevo, proporcione un nombre y haga clic en <DNT>**Create group**</DNT>. Se abre el panel <DNT>**Admin**</DNT> del grupo.
    4. En <DNT>**Members**</DNT>, agregue el usuario.
    5. En <DNT>**Account access**</DNT>, asocie el rol personalizado con la cuenta deseada para la gestión de cardinalidad.
    6. Si la gestión de cardinalidad involucra varias cuentas, en <DNT>**Account access**</DNT>, incluya el rol personalizado para cada cuenta relevante dentro del grupo.
  </Step>
</Steps>
