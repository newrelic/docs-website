Para aprovechar al máximo New Relic, es posible que necesite o desee informar datos personalizados. Le brindamos las herramientas que necesita para introducir cualquier tipo de datos en New Relic.

## Echa un vistazo a nuestras soluciones seleccionadas [#options]

New Relic informan una gran cantidad de datos automáticamente, ¡no se requiere configuración adicional! Cuando utiliza soluciones de monitoreo como <InlinePopover type="apm"/>, <InlinePopover type="browser"/>, <InlinePopover type="mobile"/>, monitoreo de infraestructura o cualquiera de nuestra variedad [de integración e inicios rápidos](https://newrelic.com/instant-observability), recibe de forma predeterminada varios tipos de datos de rendimiento.

Pero es posible que desees incorporar datos a New Relic que no se recopilan de forma predeterminada. Quizás desee una solución basada en API que no requiera la instalación de un agente. Quizás desee incorporar telemetry data de otro servicio de análisis a New Relic. A continuación, describiremos varias formas de obtener más conjuntos de datos personalizados en New Relic.

## Reportar datos personalizados con API del agente [#agent-apis]

Si está utilizando el agente New Relic (APM, browser o dispositivo móvil), puede configurarlo para enviar datos personalizados. Para obtener más información, consulte los documentos del agente que está utilizando.

## Informar datos de infraestructura personalizados [#flex-integration]

Si nuestras integraciones de infraestructura no satisfacen sus necesidades, puede utilizar nuestra [integración Flex](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) para informar cualquier tipo de métrica o configuración relacionada con la infraestructura a New Relic.

Si es una organización empresarial mediana o grande y desea utilizar la tecnología [SAP ®](https://www.sap.com/) para gestionar sus operaciones comerciales y relaciones con los clientes, puede utilizar nuestra [integración de SAP](/docs/data-apis/custom-data/sap-integration).

## Informar datos de especificaciones y servicios de telemetría de código abierto [#third-party-telemetry]

Puede revisar toda nuestra integración en [nuestra página de inicio rápido](https://newrelic.com/instant-observability). Estas son algunas de nuestras integraciones más populares con servicios y especificaciones de telemetría de código abierto de terceros:

* [OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/introduction-opentelemetry-new-relic)
* [Prometeo](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* [Zipkin](/docs/distributed-tracing/trace-api/report-zipkin-format-traces-trace-api)
* [EstadísticasD](/docs/more-integrations/open-source-telemetry-integrations/statsd/statsd-monitoring-integration)
* [Grafana](/docs/more-integrations/grafana-integrations/get-started/grafana-support-prometheus-promql)
* [Asistente para soltar](/docs/more-integrations/open-source-telemetry-integrations/dropwizard/dropwizard-reporter)
* [Micrómetro](/docs/more-integrations/open-source-telemetry-integrations/micrometer/micrometer-metrics-registry)

## Informe atributo personalizado y evento [#events]

Puedes decorar algún evento reportado por New Relic con metadatos personalizados. Para obtener más información al respecto, consulte [atributo personalizado](/docs/data-apis/custom-data/custom-events/collect-custom-attributes).

También puedes reportar un evento completamente personalizado a New Relic con nuestra [API de eventos](/docs/data-apis/ingest-apis/event-api/introduction-event-api).

Los atributos personalizados y eventos se pueden consultar y trazar.

## Informar datos de traza personalizados [#tracing]

Nuestra [traza API](/docs/distributed-tracing/trace-api/introduction-trace-api) le permite enviar datos de rastreo distribuido a New Relic y consolidar datos de traza de múltiples fuentes en un solo lugar. Aceptamos datos de traza en dos formatos:

* Formato Zipkin
* New Relic (si no tienes datos en formato Zipkin, usarías este)

## Informar datos log personalizados [#logs]

Si nuestra [integración de registro](/docs/logs/get-started/get-started-log-management) existente no satisface sus necesidades, puede utilizar nuestra [APIlog ](/docs/logs/log-api/introduction-log-api)para enviar cualquier dato de registro arbitrario a New Relic.

## Cree visualizaciones de datos personalizadas [#visualizations]

¿Está interesado en crear consultas, gráficos y paneles personalizados? Para comenzar a realizar consultas, consulte [Introducción a NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language). Cuando realiza una consulta, puede tomar el gráfico resultante y agregarlo a [un dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Si necesita más potencia y personalización de la que permiten nuestros gráficos y <InlinePopover type="dashboards"/>, también puede crear su propia [aplicación New Relic](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application) o personalizar una existente desde [el catálogo de aplicaciones](https://opensource.newrelic.com/nerdpacks). New Relic se encuentran en nuestra plataforma y se pueden compartir con su equipo o con el público.
