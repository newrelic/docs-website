[APM](/docs/apm/new-relic-apm/getting-started) informa [datos de eventos](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) que aparecen en algunas pantallas UI y también están disponibles para [consultas y gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data). Seleccione un nombre de evento en la siguiente tabla para ver su atributo.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        evento
      </th>

      <th>
        Descripción
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="transaction-event">
        [`Transaction`](/attribute-dictionary/?event=Transaction)
      </td>

      <td>
        Una transacción es una unidad de trabajo lógica en una aplicación de software. El evento `Transaction` incluye información sobre la aplicación, llamadas a la base de datos, la duración de la transacción y cualquier error que pueda ocurrir.
      </td>
    </tr>

    <tr>
      <td id="transactionerror-event">
        [`TransactionError`](/attribute-dictionary/?event=TransactionError)
      </td>

      <td>
        Los errores de transacción ocurren cuando una solicitud genera una excepción en la ruta del código que se tomó para completar esa solicitud. La cantidad de errores de transacción no es igual a la cantidad de transacciones, porque puede especificar si desea [recopilar, ignorar o marcar los errores como se esperaba](/docs/agents/manage-apm-agents/agent-data/manage-errors-apm-collect-ignore-or-mark-expected).
      </td>
    </tr>

    <tr>
      <td id="span-event">
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        Los eventos span se crean cuando un agente New Relic, u otras herramientas de instrumentación de rastreo, instrumentan operaciones que forman parte de un rastreo distribuido. Consulte la [definición de tramo](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#span).
      </td>
    </tr>
  </tbody>
</table>
