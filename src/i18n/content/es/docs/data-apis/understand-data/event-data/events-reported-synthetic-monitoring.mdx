[El monitoreo sintético en New Relic](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) informa [datos de eventos](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) que se muestran en algunas pantallas UI y también están disponibles para [consultas y gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data). Seleccione un nombre de evento en la siguiente tabla para ver su atributo.

    <tr>
      <td id="syntheticcheck-error">
        [`SyntheticCheck`](/attribute-dictionary/?event=SyntheticCheck)
      </td>

      <td>
        `SyntheticCheck` devuelve métrica de una ejecución de un monitor específico. Estas métricas incluyen información de duración del monitor, ubicación de la verificación del monitor, tamaño de los encabezados de solicitud y respuesta, el tipo de monitor y una timestamp.

        Cada vez que un monitor Sintético ejecuta una verificación, los detalles sobre la verificación se capturan en el tipo de evento`SyntheticCheck` . `SyntheticCheck` evento contiene detalles específicos de la verificación para proporcionar visibilidad, como el estado, el tipo de monitor y el tamaño de los encabezados de solicitud y respuesta.
      </td>
    </tr>

    <tr>
      <td id="syntheticrequest">
        [`SyntheticRequest`](/attribute-dictionary/?event=SyntheticRequest)
      </td>

      <td>
        `SyntheticRequest` devuelve resultados de solicitudes HTTP individuales realizadas durante una verificación. Los datos recopilados incluyen información del trabajo, ubicación, tipo de contenido para la solicitud, información de duración, tamaño de la solicitud e información de carga de la página.

        Con cada verificación de monitor simple o programada, capturamos cada solicitud HTTP individual realizada durante la verificación. Los detalles HTTP se capturan a un nivel más granular que el tipo de evento `SyntheticCheck` .
      </td>
    </tr>

    <tr>
      <td>
        [`SyntheticPrivateLocationStatus`](/attribute-dictionary/?event=SyntheticsPrivateLocationStatus)
      </td>

      <td>
        Cada verificación de monitor que se ejecuta en una ubicación privada genera detalles de capacidad para esa ubicación privada. Estos detalles se capturan en un evento `SyntheticPrivateLocationStatus` . Esto proporciona visibilidad de la capacidad de una ubicación privada y si se necesitan subordinados adicionales para soportar la carga de trabajo.
      </td>
    </tr>

    <tr>
      <td id="synthetic-private-minion">
        [`SyntheticPrivateMinion`](/attribute-dictionary/?event=SyntheticsPrivateMinion)
      </td>

      <td>
        Si tiene una ubicación privada, como las que se encuentran dentro de su firewall, puede ver información sobre esas ubicaciones con el evento `SyntheticPrivateMinion` .

        Cada minion privado en ejecución envía detalles de salud a `SyntheticPrivateMinion` cada 30 segundos. Esto le permite comprender el estado del minion privado que se ejecuta en la ubicación.
      </td>
    </tr>
  </tbody>
</table>
