De forma predeterminada, los productos New Relic informan una variedad de datos utilizados en nuestros gráficos UI y que están disponibles para [consultas](/docs/using-new-relic/data/understand-data/query-new-relic-data). Nuestros productos no transmitirán información sensible sin estar explícitamente instrumentados para hacerlo.

## Evento y atributo predeterminados [#default-events]

Nuestros productos reportan un conjunto de [eventos y atributos predeterminados](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products). Nunca enviaremos parámetros de solicitud ni ningún otro atributo que no esté en el conjunto predeterminado, a menos que alguien lo haya habilitado explícitamente a través de la configuración.

## Ajustar los datos reportados [#default-security]

Al evaluar la configuración de seguridad para un producto New Relic, revise el [evento y atributo predeterminados](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products). El atributo predeterminado no contiene datos confidenciales. En general, son simplemente los datos necesarios para un monitoreo efectivo del desempeño. Nuestros productos no envían otros datos a menos que cambie la configuración de seguridad predeterminada.

Dependiendo de sus requisitos, se pueden aplicar una o ambas situaciones:

* Si la lista predeterminada contiene datos que le preocupan, puede desactivar la recopilación de esos atributos. Para saber cómo editarlo, consulte la documentación del producto que está utilizando.
* Si necesita enviar atributos que no se informan de forma predeterminada, puede [habilitar que se informen esos atributos](/docs/insights/insights-data-sources/custom-data/send-custom-data-insights). En ese caso, no utilice [el modo de alta seguridad](/docs/accounts-partnerships/accounts/security/high-security): esto desactivará la capacidad de recopilar atributos personalizados.
