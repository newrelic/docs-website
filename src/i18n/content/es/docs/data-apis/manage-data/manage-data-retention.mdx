Para encontrar la UI de retención de datos: desde el [menú de usuario](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), haga clic en <DNT>**Manage your data**</DNT> y haga clic en <DNT>**Data retention**</DNT>.

En la [página de](https://one.newrelic.com/admin-portal/limits-data-retention/data-retention) UI <DNT>**Data retention**</DNT>, puede revisar y gestionar durante cuánto tiempo se almacenan los diferentes tipos de datos. Es posible que desee una retención más corta para algunos tipos de datos con el fin de proteger información confidencial, o puede que desee una retención más larga (hasta siete años) para poder ver y analizar datos y tendencias durante un periodo más largo.

Para conocer los planes y costos para cambiar la retención, consulte [Opciones de datos: Data Plus y original](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#upgrade-data-plus).

## Amplíe su retención de datos para análisis y cumplimiento a largo plazo

Ampliar la retención de datos le permite realizar análisis, visualización y alertas a largo plazo para todas sus métricas, eventos, logs y trazas en todas sus fuentes. Sin embargo, es importante administrar esos datos por motivos de costo, rendimiento y, en algunos casos, de cumplimiento. Nuestro centro de gestión de datos proporciona las herramientas que necesita para comprender y controlar de dónde provienen sus datos y ajustar qué se almacena y durante cuánto tiempo.

<Callout variant="tip">
  Con [nuestra opción Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus), obtiene hasta 90 días adicionales de retención (para la mayoría de los tipos de datos) y otras capacidades de nivel empresarial, como consultas de mayor duración, cumplimiento de FedRAMP e HIPAA, características de seguridad adicionales y más.
</Callout>

## Requisitos [#requirements]

La forma en que los usuarios gestionan la retención de datos puede depender de cuántas cuentas tenga en su organización y de lo que estén intentando hacer.

### Organización de cuenta única [#single-account-organizations]

Si está en una organización que solo tiene una cuenta, su [**Billing User (usuario de facturación)**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/) puede gestionar lo siguiente:

* **Máximo por contrato:** este es el valor más alto que puede establecer para la **Effective retention** de un tipo de datos. Si desea aumentar esto, consulte nuestro plan [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices) .
* **Retención efectiva:** es la cantidad de tiempo que se retienen actualmente los datos para un tipo de datos. Para que el usuario pueda editar esto, la organización (cuenta) debe tener [la edición Standard o superior](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#how-pricing-works).

Si su usuario de facturación no puede realizar ninguna de estas dos tareas, comunicar con el soporte o con su representante de New Relic.

### Organización de cuentas múltiples [#multiple-account-organizations]

Si estás en una organización que tiene más de una cuenta, el usuario puede gestionar lo siguiente:

* **Máximo por contrato:** este es el valor más alto que puede establecer para la **Effective retention** de un tipo de datos. Si desea aumentar esto, consulte nuestro plan [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices) .

* **Retención efectiva:** es la cantidad de tiempo que se retienen actualmente los datos para un tipo de datos. Para que el usuario pueda editar esto:

  * La organización (cuenta) debe estar en [Pro o una edición superior](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#how-pricing-works).
  * El tipo de usuario debe ser **Full platform** (Plataforma completa).
  * El usuario necesitará licencias de **View** y **Modify** para la función [**de retención de eventos de Insights**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#data-retention) que [apunta a las cuentas](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/) para las que necesita editar **la retención efectiva**.

Si su usuario no puede realizar ninguna de estas dos tareas luego de consultar los detalles anteriores, comunicar con el soporte o con su representante de New Relic.

## Detalles de retención para diversos tipos de datos [#data-details]

En esta sección se encuentran detalles sobre algunos tipos diferentes de datos, incluidos algunos tipos de datos que tienen configuraciones de retención que no se pueden cambiar.

<CollapserGroup>
  <Collapser id="dimensional-metrics" title="Dimensional métrica (métrica API)">
    Los datos [métricos dimensionales](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) sin procesar (el tipo de datos `Metric` ) se agregan (acumulan) para algunos rangos de tiempo diferentes y esas agregaciones se conservan durante diferentes períodos de tiempo. Los puntos de datos métricos dimensionales sin procesar se almacenan durante 30 días. Todos los puntos de datos agregados (la agregación comienza en un minuto) se conservan durante 13 meses.

    Solo la ingesta de datos sin procesar cuenta para el uso; los datos agregados no.

    Los períodos de retención métrica no son editables.
  </Collapser>

  <Collapser id="metric-timeslice" title="Intervalo de tiempo de datos métricos (APM, browser, móvil)">
    [Los datos de intervalo de tiempo de métrica](/docs/using-new-relic/data/understand-data/new-relic-data-types#timeslice-data) son generados por <InlinePopover type="apm" />(tanto métrica personalizada como métrica predeterminada), <InlinePopover type="mobile" />y el agente del navegador. Un intervalo de tiempo de métrica consta de tres partes: un nombre de métrica, el segmento de tiempo que representa la métrica (el &quot;intervalo de tiempo&quot;) y un valor numérico (la medición). APM, <InlinePopover type="browser" />y el agente móvil pueden recopilar miles de intervalos de tiempo métricos por minuto para una variedad de métricas de rendimiento, como tasa de errores, uso de ancho de banda y tiempo de recolección de basura.

    * resúmenes de 1 minuto que se almacenan durante 8 días

    * Los resúmenes de 1 minuto también se agregan a los resúmenes de 1 hora que se retienen durante 90 días.

      Solo la ingesta inicial de datos sin procesar cuenta para el uso; los datos agregados no.

      Algunas claves métricas se almacenan indefinidamente. Continúe leyendo para conocer la clave métrica por producto.

## Editar retención de datos [#adjust-retention]

Para ver y editar su retención, vaya a [la vista de retención de datos](https://one.newrelic.com/admin-portal/limits-data-retention/data-retention) y haga clic en <DNT>**Edit data retention**</DNT>. Allí verá opciones para editar la retención de datos o actualizar a Data Plus. Si no ve eso, revise [los requisitos](#requirements) y hable con su representante de cuenta de New Relic.

Opciones para aumentar la retención:

* Si aún no lo ha hecho, cambie a [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus), que ofrece una retención de datos significativamente más prolongada y otros beneficios.
* Para Data Plus u opción de datos original: agregar retención de datos. 30 días de retención de datos adicionales para su organización cuestan $0,05 por GB ingerido por mes.

Puntos importantes sobre cómo aumentar la retención:

* Cuando habilita un período de retención más alto, la retención aumenta en todos los ámbitos para todos los tipos de datos enumerados en la [UI<DNT>**Data retention**</DNT> ](#find-ui). No se puede aumentar solo para tipos de datos específicos.
* Una vez que habilita un período de retención más alto, reducir la retención para fuentes de datos específicas no tiene ningún impacto en su facturación. Para conocer formas de reducir la ingesta y la facturación de datos, consulte [Administrar la ingesta de datos](/docs/data-apis/manage-data/manage-data-coming-new-relic).
* Cuando ajustas la retención, puede tardar hasta 24 horas en surtir efecto.

### Reducir la retención [#reduce-retention]

Cuando reduce el período de retención de una fuente de datos, los datos existentes de ese tipo no desaparecerán inmediatamente. Esos datos pueden ser visibles y devueltos en consulta durante hasta varias semanas. Contamos con medidas de seguridad para garantizar que no elimine datos accidentalmente, pero esto también significa que reducir los valores de retención no es una buena estrategia para [eliminar datos](#deleting-data) rápidamente.

Para obtener información sobre las opciones para reducir la ingesta de datos, consulte [Administrar la ingesta de datos](/docs/data-apis/manage-data/manage-data-coming-new-relic).

## Eliminar datos existentes [#deleting-data]

Una vez que [telemetry data](/docs/using-new-relic/data/understand-data/new-relic-data-types) se informan a la base de datos de New Relic y están disponibles para su consulta, esos datos no se pueden editar ni eliminar. Esta es una decisión de diseño decidida que optimiza la velocidad y el rendimiento de New Relic. Los datos caducarán una vez finalizado su [plazo de conservación de datos](#adjust-retention) . Si desea eliminar datos que ya han sido reportados, comuníquese con su representante de New Relic, si tiene uno.

## Exportar y transmitir datos [#export-data]

Una organización con Data Plus puede exportar sus datos para almacenarlos a largo plazo utilizando sus herramientas preferidas. Las características relacionadas con esto incluyen:

* [Exportación de streaming de datos](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export)
* [Exportación de datos históricos](/docs/apis/nerdgraph/examples/nerdgraph-historical-data-export)

## Gestionar la retención a través de API [#api-examples]

Además de cambiar la configuración de retención en la UI, puede utilizar nuestra [API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph). A continuación se incluyen algunos ejemplos de solicitudes de NerdGraph. Tenga en cuenta que los cambios en la retención de datos pueden tardar hasta 24 horas en surtir efecto.

Antes de usar la API para ajustar la retención, revise [las reglas sobre cómo ajustar la retención](#adjust-retention).

<CollapserGroup>
  <Collapser id="api-list-namespaces" title="Lista de espacios de nombres de eventos de retención personalizables">
    Para enumerar los espacios de nombres de eventos de retención personalizables para su cuenta:

    ```graphql
    query {
     actor {
       account(id: YOUR_ACCOUNT_ID) {
         dataManagement {
           customizableRetention {
             eventNamespaces {
               namespace
             }
           }
         }
       }
     }
    }
    ```
  </Collapser>

  <Collapser id="api-list-rules" title="Listar reglas activas en una cuenta">
    Para enumerar reglas activas en una cuenta:

    ```graphql
    query {
     actor {
       account(id: YOUR_ACCOUNT_ID) {
         dataManagement {
           eventRetentionRules {
             id
             deletedAt
             deletedById
             createdAt
             createdById
             retentionInDays
             namespace
           }
         }
       }
     }
    }
    ```
  </Collapser>

    ```graphql
     actor {
       account(id: YOUR_ACCOUNT_ID) {

             id

             deletedAt
             createdById
             createdAt
             retentionInDays
             namespace
           }
         }
       }
     }
    }
    ```

    Para crear una nueva regla:

    ```graphql
    mutation {
     dataManagementCreateEventRetentionRule(accountId: YOUR_ACCOUNT_ID, namespace: "APM", retentionInDays: 8) {

       createdAt
       retentionInDays
    ```

  <Collapser id="api-delete-rule" title="Eliminar una regla existente">
    Si elimina una regla aplicada a una cuenta infantil, esa cuenta infantil adoptará el valor de retención actual de su cuenta principal para el namespace determinado. Si elimina una regla aplicada a una cuenta principal, el valor de retención predeterminado se volverá a aplicar para el namespace determinado. Para eliminar una regla existente:

    ```graphql
    mutation {
     dataManagementDeleteEventRetentionRule(accountId: YOUR_ACCOUNT_ID, namespace: "APM") {
       id
       deletedById
       deletedAt
       createdById
       createdAt
       retentionInDays
       namespace
     }
    }
    ```
  </Collapser>

  <Collapser id="api-delete-rule" title="Copiar reglas de cuentas infantiles existentes a otros">
    Puede especificar una cuenta secundaria como plantilla o cuenta de origen y hacer que otras cuentas secundarias de destino adopten los mismos valores de retención de namespace de la cuenta secundaria de origen. Nota: todas las cuentas infantiles deben estar dentro de la misma cuenta principal. Para copiar un conjunto de retenciones existente:

    ```graphql
    mutation {
      dataManagementCopyRetentions(
        sourceAccountId: SOURCE_ACCOUNT_ID,
        destinationAccountIds: [DESTINATION_ACCOUNT_ID_1, DESTINATION_ACCOUNT_ID_2, DESTINATION_ACCOUNT_ID_3])
      {
        success
        failure
      }
    }
    ```

    El éxito y el fracaso devolverán listas de ID de cuentas de destino que tuvieron éxito (o fallaron) al copiar reglas de la cuenta de origen.

## Herencia de cuenta [#account-structure]

La configuración de retención de datos de una [cuenta principal](/docs/accounts/original-accounts-billing/original-users-roles/mastersub-account-structure) la heredan sus cuentas secundarias. En este contexto, la primera cuenta de una organización New Relic se considera la cuenta principal y otras cuentas agregadas a la organización heredan la configuración de esa cuenta.
