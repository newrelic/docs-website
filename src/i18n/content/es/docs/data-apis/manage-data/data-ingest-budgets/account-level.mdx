Para las organizaciones con múltiples cuentas de New Relic, cada una con distintos requisitos de ingesta de datos, los presupuestos de ingesta de datos proporcionan monitoreo, visibilidad y alertas por cuenta para asegurar que las cuentas se mantengan dentro de sus límites de uso definidos.

## Tipos de presupuesto por estructura de cliente

El tipo de presupuestos que puede crear depende de la estructura de su cliente:

### Para clientes de un solo inquilino

Puede crear <DNT>**account level budgets**</DNT> para cuentas individuales o grupos de hasta 5 cuentas dentro de su organización.

### Para clientes de múltiples inquilinos

<DNT>**Managing organizations**</DNT> puede crear <DNT>**account level budgets**</DNT> para cuentas dentro de la organización administradora (la misma funcionalidad que los clientes de un solo inquilino).

<DNT>**Managed organizations**</DNT> no pueden crear presupuestos.

## Configurar presupuestos a nivel de cuenta

<Steps>
  <Step>
    ### Abrir Presupuestos de datos

    1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com/)**</DNT>.
    2. En el <DNT>[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)</DNT>, seleccione <DNT>**Administration** &gt; **Data Management** &gt; **Data Budgets**</DNT>.
  </Step>

  <Step>
    ### Crear un presupuesto

    En la sección <DNT>**Functional budgets**</DNT> , seleccione <DNT>**+ Add functional budget**</DNT>.

    <Callout variant="important">
      <DNT>**For multi-tenant customers**</DNT>, esta sección muestra:

      * <DNT>**Organization level budgets**</DNT> para organizaciones administradas
      * <DNT>**Account level budgets**</DNT> para la organización de gestión

      <DNT>**Managed organizations**</DNT> no pueden crear presupuestos a nivel de cuenta.
    </Callout>
  </Step>

  <Step>
    ### Asigna tu cotización

    1. Ingrese un nombre para su presupuesto.
    2. De la lista desplegable, seleccione y asigne hasta 5 cuentas a este presupuesto.

    <Callout variant="tip">
      Las cuentas que ya tienen presupuestos asignados aparecerán deshabilitadas en la lista de selección.
    </Callout>

    3. Seleccione <DNT>**Next**</DNT>.
  </Step>

  <Step>
    ### Establecer límites de cotización mensual

    1. En la sección <DNT>**Set monthly budget limits**</DNT>, verá cuántos GB quedan en la cotización de su organización.
    2. Ingrese la cantidad de GB que desea asignar a este presupuesto a nivel de cuenta.

    <Callout variant="tip">
      Puede asignar más del presupuesto de su organización. Esto permite una planificación flexible, pero puede resultar en excedentes.
    </Callout>

    3. Seleccione <DNT>**Next**</DNT>.
  </Step>

  <Step>
    ### Configurar notificación

    1. Para recibir una advertencia cuando haya utilizado un cierto porcentaje de los GB asignados a este presupuesto para un mes específico, en la sección <DNT>**Thresholds**</DNT>, seleccione <DNT>**+ Add a threshold**</DNT> e ingrese el porcentaje en el que desea recibir la notificación (por ejemplo, 75 %, 90 % o 95 %).

    2. Para asegurar que se envíen notificaciones cuando el presupuesto exceda el umbral, en la sección <DNT>**Destinations**</DNT>, agregue al menos un destino para recibir notificaciones. Puede elegir entre las siguientes opciones:

       * <DNT>**Email**</DNT>: Ingrese las direcciones de correo electrónico de los usuarios de New Relic para notificar sobre el presupuesto.
       * <DNT>**Webhook**</DNT>: Ingrese la URL del webhook para enviar notificaciones a un punto final específico.

    3. Seleccione <DNT>**Finish**</DNT>.
  </Step>

  <Step>
    ### Revisar los detalles del presupuesto

    Después de crear el presupuesto, la página <DNT>**Data Budgets**</DNT> muestra la siguiente información para cada presupuesto:

    <table>
      <thead>
        <tr>
          <th>
            Nombre del campo
          </th>

          <th>
            Descripción
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            <DNT>**Budget name**</DNT>
          </td>

          <td>
            El nombre que le asignó a la cotización.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**Status**</DNT>
          </td>

          <td>
            Estado actual del presupuesto.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**GB allocation**</DNT>
          </td>

          <td>
            Cuántos datos (GB) se asignaron a este presupuesto.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**GB utilized**</DNT>
          </td>

          <td>
            Cuántos de los datos asignados se emplearon.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**Utilization percentage**</DNT>
          </td>

          <td>
            Porcentaje del presupuesto que se ha utilizado.
          </td>
        </tr>
      </tbody>
    </table>
  </Step>
</Steps>

## Gestionar cotizaciones existentes

Para administrar un presupuesto existente:

1. Ubique el presupuesto en la lista y seleccione el menú de tres puntos.

2. Elija entre las siguientes opciones:

   * <DNT>**View details**</DNT>: vea información detallada sobre la cotización, incluidos gráficos de utilización, umbral, configuraciones de notificación, cuentas asociadas y fuentes de datos.
   * <DNT>**Edit**</DNT>: modificar la configuración de asignación de cotización, umbral o notificación.
   * <DNT>**Delete**</DNT>: eliminar la cotización.

### Ver detalles de la cotización

Seleccione un nombre de presupuesto o elija <DNT>**View details**</DNT> del menú de tres puntos para ver:

* <DNT>**Total allocation**</DNT>: el total de GB asignados a este presupuesto
* <DNT>**Utilization**</DNT>: Gráfico visual que muestra el uso a lo largo del tiempo con desglose día a día
* <DNT>**Thresholds**</DNT>: el umbral de notificación que configuraste
* <DNT>**Notification destinations**</DNT>: dónde se envían las alertas cuando se superan los umbrales
* <DNT>**Associated accounts**</DNT>: Las cuentas incluidas en este presupuesto
* <DNT>**Data sources**</DNT>: Información sobre las fuentes de datos que contribuyen al uso de la cotización, categorizadas por tipo de fuente
* <DNT>**Budget time range**</DNT>: El período de tiempo cubierto por este presupuesto
