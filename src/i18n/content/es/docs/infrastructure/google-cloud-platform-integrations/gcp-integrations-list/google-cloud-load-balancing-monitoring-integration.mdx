[La integración de New Relic](/docs/infrastructure/introduction-infra-monitoring) con [Google Cloud Platform (GCP)](https://cloud.google.com) incluye una integración para informar datos de [Google Cloud Load Balancing](https://cloud.google.com/load-balancing/) a New Relic. Este documento explica cómo activar la integración de GCP Cloud Load Balancing y describe los datos que se pueden informar.

## Característica

Google Cloud Load Balancing es un servicio administrado basado en software para distribuir el tráfico en una o varias regiones. Proporciona un escalado automático inmediato y sin interrupciones y admite los últimos protocolos de entrega de aplicaciones.

## Activar la integración [#activate]

Para habilitar la integración, siga los procedimientos estándar para [conectar su servicio GCP a New Relic](/docs/connect-google-cloud-platform-services-infrastructure).

## Frecuencia de sondeo [#polling]

New Relic integración consulta tus servicios de GCP según un intervalo de sondeo, que varía según la integración. La frecuencia de sondeo para Google Cloud Load Balancer es de cinco minutos. La resolución es de 1 punto de datos cada minuto.

## Buscar y utilizar datos [#find-data]

Después de activar la integración y esperar unos minutos (según la [frecuencia de](#polling) sondeo ), los datos aparecerán en la New Relic UI. Para [buscar y utilizar sus datos](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), incluidos enlaces a su <InlinePopover type="dashboards" />y configuración de alertas, vaya a <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; GCP &gt; (select an integration)**</DNT>.

## Datos métricos [#metrics]

Para ver [datos métricos](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics) para su integración de equilibrio de carga de GCP en New Relic, [cree una consulta NRQL ](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights#nrql)para [los eventos](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `GcpHttpLoadBalancerSample`, `GcpHttpExternalRegionalLoadBalancerSample`, `GcpInternalHttpLoadBalancerSample`, `GcpTcpSslProxyLoadBalancerSample` y `GcpInternalLoadBalancerSample` y sus atributos relacionados.

      <td>

    <tr>
      <td>
        `https.BackendResponseBytes`
      </td>

        La cantidad de bytes enviados como respuestas desde el backend (o caché) al balanceador de carga HTTP/S externo.
      </td>
    </tr>

      <td>
        `https.FrontendTcpRtt`
      </td>

      <td>

      </td>
    </tr>

    <tr>
      <td>
        `https.RequestBytes`
      </td>

      <td>

      </td>
    </tr>

    <tr>
      <td>
        `https.Requests`
      </td>

      <td>
        La cantidad de requests atendidas por el balanceador de carga HTTP/S externo.
      </td>
    </tr>

    <tr>
      <td>
        `https.ResponseBytes`
      </td>

      <td>
        La cantidad de bytes enviados como respuestas desde el balanceador de carga HTTP/S externo a los clientes.
      </td>
    </tr>

    <tr>
      <td>
        `https.TotalLatencies`
      </td>

      <td>
        Media de la latencia (en milisegundos) calculada desde que el proxy balanceador de carga HTTP/S externo recibió la solicitud hasta que el proxy recibió el ACK del cliente en el último byte de respuesta.
      </td>
    </tr>
