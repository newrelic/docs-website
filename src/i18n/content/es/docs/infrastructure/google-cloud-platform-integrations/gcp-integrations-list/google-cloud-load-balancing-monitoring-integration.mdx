[La integración de New Relic](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) con [Google Cloud Platform (GCP)](https://cloud.google.com) incluye una integración para informar los datos [de equilibrio de carga Google Cloud ](https://cloud.google.com/load-balancing/)a New Relic. Este documento explica cómo activar la integración de GCP Cloud Load Balancing y describe los datos que se pueden informar.

## Característica

Google Cloud Load Balancing es un servicio administrado basado en software para distribuir el tráfico en una o varias regiones. Proporciona un escalado automático inmediato y sin interrupciones y admite los últimos protocolos de entrega de aplicaciones.

## Activar la integración [#activate]

Para habilitar la integración, siga los procedimientos estándar para [conectar su servicio GCP a New Relic](/docs/connect-google-cloud-platform-services-infrastructure).

## Frecuencia de sondeo [#polling]

New Relic integración consulta tus servicios de GCP según un intervalo de sondeo, que varía según la integración. La frecuencia de sondeo para Google Cloud Load Balancer es de cinco minutos. La resolución es de 1 punto de datos cada minuto.

## Buscar y utilizar datos [#find-data]

Después de activar la integración y esperar unos minutos (según la [frecuencia de](#polling) sondeo ), los datos aparecerán en la New Relic UI. Para [buscar y utilizar sus datos](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), incluidos enlaces a su <InlinePopover type="dashboards"/>y configuración de alertas, vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > GCP > (select an integration)**</DNT>.

## Datos métricos [#metrics]

Para ver [datos métricos](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics) para su integración de equilibrio de carga de GCP en New Relic, [cree una consulta NRQL ](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights#nrql)para [los eventos](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `GcpHttpLoadBalancerSample`, `GcpTcpSslProxyLoadBalancerSample` y `GcpInternalLoadBalancerSample` y sus atributos relacionados.

    </tr>

      <td>
        `https.ResponseBytes`

      </td>

      </td>

### Muestra de GcpTcpSslProxyLoadBalancer [#gcp-tcp-ssl-proxy-load-balancer-sample]

Consulta `GcpTcpSslProxyLoadBalancerSample` evento en New Relic para ver datos del siguiente atributo:

<table>
  <thead>
    <tr>
      <th style={{ width: "300px" }}>
        Atributo
      </th>

      <th>
        Descripción
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `tcpSslProxy.ClosedConnections`
      </td>

      <td>
        Número de conexiones que terminaron a través del proxy TCP/SSL.
      </td>
    </tr>

    <tr>
      <td>
        `tcpSslProxy.EgressBytes`
      </td>

      <td>
        Número de bytes enviados desde la máquina virtual (VM) al cliente mediante proxy.
      </td>
    </tr>

    <tr>
      <td>
        `tcpSslProxy.FrontendTcpRtt`
      </td>

      <td>
        Media del RTT suavizado (en milisegundos) medido por la stack TCP del proxy, cada minuto, los bytes de la capa de aplicación pasan del proxy al cliente.
      </td>
    </tr>

    <tr>
      <td>
        `tcpSslProxy.IngressBytes`
      </td>

      <td>
        Número de bytes enviados desde el cliente a la máquina virtual (VM) mediante proxy.
      </td>
    </tr>

    <tr>
      <td>
        `tcpSslProxy.NewConnections`
      </td>

      <td>
        Número de conexiones que se crearon a través del proxy TCP/SSL.
      </td>
    </tr>

    <tr>
      <td>
        `tcpSslProxy.OpenConnections`
      </td>

      <td>
        Número actual de conexiones pendientes a través del proxy TCP/SSL.
      </td>
    </tr>
  </tbody>
</table>
