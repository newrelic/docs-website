En nuestra UI usuario de monitoreo de infraestructura, la página UI <DNT>**Events**</DNT> es una transmisión en vivo de actividad importante del sistema y del host, incluidos eventos de cambio de inventario, cambios de configuración y eventos de análisis log . El feed de eventos le ayuda a comprender las correlaciones entre estos eventos y el rendimiento del sistema. Busque y filtre su evento para reducir el tiempo medio para detectar y reparar problemas de infraestructura.

<Callout variant="tip">
  Tenga en cuenta que `event` aquí tiene un significado específico de la infraestructura que es diferente de nuestro [tipo de datos de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data).
</Callout>

Encuentre la página <DNT>**Events**</DNT> yendo a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**</DNT>.

## Cronograma del evento [#event-timeline]

El gráfico de eventos proporciona una instantánea del [evento](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page) de infraestructura que ocurre dentro del mismo rango de tiempo que la métrica mostrada. Al pasar el mouse, puede ver la cantidad de eventos por categoría codificada por colores para investigar cualquier pico anómalo.

<img
  title="infra-events-timeline.png"
  alt="infra-events-timeline.png"
  src="/images/infrastructure_screenshot-crop_events-timeline.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**</DNT>: La línea de tiempo del evento muestra patrones en los que el evento ocurre en el mismo período de tiempo para la métrica mostrada.
</figcaption>

Al comparar la línea de tiempo con los gráficos de la página de infraestructura, puede identificar rápidamente problemas en su ecosistema. Por ejemplo, si se produce un pico masivo de CPU, puede hacer clic en la línea de tiempo del evento para ese rango de tiempo para encontrar el evento que lo causó. Desde allí podrás profundizar más para descubrir el problema real.

## Tipos de eventos [#types]

New Relic recopila una variedad de eventos de cambio para que puedas comprender cada cambio en tu entorno:

  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        <DNT>
          **Events**
        </DNT>
      </th>

      <th>
        <DNT>
          **Comments**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Incidente de alerta
      </td>

      <td>
        Cuando se abre o cierra un [incidente](/docs/new-relic-solutions/get-started/glossary/#alert-incident), New Relic genera un evento que indica el host y [la condición de alerta](/docs/infrastructure/new-relic-infrastructure/configuration/infrastructure-alerts-add-edit-or-view-host-alert-information) asociada.
      </td>
    </tr>

    <tr>
      <td>
        Conexión de agente
      </td>

      <td>
        Cuando un [agente](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#agent) de infraestructura se conecta a New Relic, nuestra plataforma genera un evento <DNT>**Agent connected**</DNT> . Si New Relic no recibe datos de un agente durante tres minutos, la plataforma genera un evento <DNT>**Agent disconnected**</DNT> .
      </td>
    </tr>

    <tr>
      <td>
        Cambios de inventario
      </td>

      <td>
        Estos eventos se generan cuando se agregan, eliminan o modifican datos [de inventario](/docs/infrastructure-inventory-page) . Selecciona la fuente <Icon name="fe-filter"/> icono para comprender qué categoría corresponde a la ruta de inventario modificada. Para obtener detalles adicionales, seleccione un evento de inventario para ver una comparación en paralelo del estado antiguo y nuevo.

        El evento de inventario puede incluir:

        * <DNT>
            **Kernel**
          </DNT>

          (incluye módulos y configuración): Se puede agregar, modificar o eliminar.

        * <DNT>
            **Metadata**
          </DNT>

          (incluye diversa información adicional sobre los hosts): se puede agregar, modificar o eliminar.

        * <DNT>
            **Packages**
          </DNT>

          : Puede instalarse, eliminarse o modificarse.

        * <DNT>
            **Services**
          </DNT>

          : Puede iniciarse, detenerse o reiniciarse.

        * <DNT>
            **Sessions**
          </DNT>

          (incluye usuario): Se puede conectar o desconectar.
      </td>
    </tr>
  </tbody>
</table>
