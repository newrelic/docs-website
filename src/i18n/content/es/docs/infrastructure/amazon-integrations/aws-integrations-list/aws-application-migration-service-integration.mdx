Ahora puede emplear el AWS [Application Migration Service](https://docs.aws.amazon.com/mgn/latest/ug/what-is-application-migration-service.html) para instalar el New Relic agente en servidores que migran a AWS.

## Configuración [#setup]

Para que AWS instale el [agente deNew Relic Infrastructure ](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/), deberá crear una plantilla posterior al lanzamiento a través de la consola AWS . Siga estos pasos:

1. Desde la [consola de AWS](https://console.aws.amazon.com/mgn/home), abra <DNT>**Application Migration Services** </DNT>.
2. Abra el servidor de origen deseado de la lista disponible.
3. Haga clic en la pestaña **Post-launch settings** \[Configuración posterior al lanzamiento].
4. Seleccione **Edit &gt; Install New Relic agent** y agregue su ID de cuenta y clave de licencia.

## Monitorizando sus datos [#monitor-data]

Una vez completada la configuración, su aplicación AWS será monitoreada por el agente New Relic Infrastructure. Para encontrar sus datos en New Relic:

1. Vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com/) &gt; All entities**</DNT>.
2. En la sección de entidad **Hosts** , ubique su instancia.
