[La integración New Relic Infrastructure ](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)incluye una integración AWS [Elastic Beanstalk](https://aws.amazon.com/elasticbeanstalk) para informar sus datos de Beanstalk a New Relic. Este documento explica la característica de la integración, cómo activarla y qué datos se pueden reportar.

AWS Elastic Beanstalk es un servicio dinámico que permite un fácil despliegue y escalabilidad para su aplicación. La "aplicación" en AWS Beanstalk es el entorno. Una "aplicación" puede tener varios entornos asociados.

## Requisitos [#requirements]

AWS Elastic Beanstalk enviará algunas métricas básicas a Amazon CloudWatch de forma predeterminada. Además, puede habilitar informes de salud mejorados como métrica personalizada; habilitar estas métricas puede agregar cargos adicionales a los precios de su cuenta de Amazon CloudWatch. Para habilitar métricas de CloudWatch mejoradas, consulte [Publicación de métricas personalizadas de Amazon CloudWatch para un entorno](http://docs.aws.amazon.com/elasticbeanstalk/latest/dg/health-enhanced-cloudwatch.html).

## Activar la integración [#activate]

De forma predeterminada, no se selecciona ninguna métrica en la configuración de Elastic Beanstalk. Debes seleccionarlos para <DNT>**each**</DNT> el entorno para generar métrica en la New Relic UI. Si no se selecciona ninguna métrica, no se informará ningún dato a New Relic.

Para habilitar esta integración:

1. Siga las instrucciones en [Conectar servicios de AWS a New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

2. <DNT>
     **each**
   </DNT>

   Actualice su configuración de AWS Elastic Beanstalk para seleccionar qué métrica (la opción predeterminada no es ninguna métrica)UI observar en el entorno : en la Elastic Beanstalk,

   <DNT>
     **Configuration > Health**
   </DNT>

   vaya a y luego configure la métrica personalizada de CloudWatch para los informes de estado.

Después de seleccionar la métrica que desea que se informe para cada entorno, Elastic Beanstalk generará datos desde el momento en que se crean los objetos de datos `instance` y `environment` .

## Configuración y sondeo [#polling]

Puede cambiar la [frecuencia de sondeo](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) y filtrar datos usando [las opciones de configuración](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Información de sondeo predeterminada para la integración de AWS Elastic Beanstalk:

* New Relic intervalo de sondeo: 5 minutos
* Intervalo de datos de Amazon CloudWatch: 1 minuto

## Ver y usar datos [#find-data]

Para [ver y utilizar sus datos de integración](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS**</DNT> y seleccione uno de los enlaces de integración de AWS Elastic Beanstalk.
