Nuestra AWS monitor integración de Internet CloudWatch reporta los monitor datos de New Relic su de Internet CloudWatch a la plataforma .

## Característica [#features]

Con esta integración, puede recopilar y enviar telemetry data a New Relic desde su [monitorde Internet CloudWatch](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-InternetMonitor.html). Una vez integrado, puede monitor sus servicios, consultar datos entrantes, monitor registros y usar un panel prediseñado para observar todo de un vistazo.

## Obtener datos métricos [#get-metrics-data]

Tenemos dos opciones para reportar estos datos. La ruta recomendada es configurar nuestra [integración de Amazon CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream). Alternativamente, puede utilizar nuestra [integración anterior de Amazon que se basa en encuestas](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations).

## Obtener datos de registro [#get-logs-data]

Los datos de registro se pueden informar a New Relic utilizando nuestro [Lambda de ingestalog ](/docs/logs/forward-logs/aws-lambda-sending-cloudwatch-logs). Asegúrese de [agregar activadores](/docs/logs/forward-logs/aws-lambda-sending-cloudwatch-logs/#create-trigger) a Lambda y seleccionar el grupo log correspondiente a `internet-monitor` en los niveles `country`, `city`, `subdivision` y `metro`.

## Buscar y utilizar datos [#find-use-data]

Los datos métricos de esta integración se adjuntan al tipo de datos `Metric` . Para ver tus datos: ve a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Metrics and events**</DNT> y filtra por `aws.internetmonitor`.
