New Relic proporciona una integración para [Amazon VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/what-is-vpc-lattice.html).

## Característica [#about]

[Amazon VPC Lattice](https://aws.amazon.com/vpc/lattice/) es un servicio de red de aplicaciones completamente administrado que se utiliza para conectar, proteger y monitor los servicios de su aplicación en múltiples cuentas y nubes privadas virtuales (VPC).

New Relic admite dos integración de monitoreo para Amazon VPC Lattice: [métricas de CloudWatch](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) y [registro de acceso](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html).

Para las métricas de CloudWatch, solo necesita configurar la integración una vez y se incluirán todos los espacios de nombres AWS disponibles, incluido `AWS/VPCLattice`. Las métricas de CloudWatch para VPC Lattice se mantienen al día con el tráfico entre sus servicios de Lattice y sus grupos objetivo.

<Callout variant="important">
  Habilite la [integraciónAWS CloudWatch Metric Streams ](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-metric-stream/)para monitor todas las métricas de CloudWatch desde sus servicios de AWS, incluido el espacio de nombres personalizado. La integración individual ya no es nuestra opción recomendada.
</Callout>

## Activar la integración del registro de acceso [#activate-logs-integration]

<Callout variant="important">
  Para utilizar Kinesis Firehose, debe tener creada una secuencia de entrega de Kinesis Data Firehose. Si no lo hace, puede crear uno accediendo a la característica Amazon Kinesis Firehose desde su [consolaAWS ](https://console.aws.amazon.com/firehose)y creando un flujo de entrega con `Destination: New Relic`.

  [Obtenga más información sobre cómo crear un flujo de entrega de Amazon Kinesis Data Firehose.](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html?icmpid=docs_console_unmapped)
</Callout>

La integración del registro de acceso para Amazon VPC Lattice utiliza [Amazon Kinesis Firehose](https://aws.amazon.com/kinesis/data-firehose/). Puede habilitar el monitoreo del registro de acceso para sus servicios Lattice agregando monitoreo ya sea a la red de servicios o individualmente en los servicios que desea rastrear.

Para configurar esta integración:

1. Desde su consola de AWS, navegue hasta el [servicio AWS VPC](https://console.aws.amazon.com/vpc).

2. Desde el panel izquierdo, debajo del

   <DNT>
     **VPC Lattice**
   </DNT>

   desplegable, seleccione `Service Networks` O `Services`, según la opción que desee utilizar para habilitar el monitoreo.

3. Seleccione el servicio o red de servicios que desea monitor.

4. Seleccione la pestaña `Monitoring` .

5. Seleccione `Edit access logs`.

6. Active el monitoreo `Access logs` .

7. Seleccione la casilla de verificación etiquetada `Kinesis Data Firehose delivery stream`.

8. Seleccione una secuencia de entrega del menú desplegable que esté configurada para apuntar a New Relic.

9. Guardar cambios.

## Buscar y utilizar datos de registro de acceso [#find-log-data]

Puede consultar y explorar sus datos en nuestra UI de registro utilizando este filtro: `ServiceNetworkArn LIKE '%vpc-lattice%'`.

Explore [el registro de acceso para VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html) para obtener más información sobre los datos log que se envían.

## Encuentre y utilice datos de métricas de CloudWatch [#find-metric-data]

Puede consultar y explorar los datos de CloudWatch a través de [nuestro explorador de consulta métrica](/docs/data-apis/understand-data/metric-data/query-metric-data-type/).

Consulte los detalles de [métricas de CloudWatch para VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) para obtener más información sobre los datos métricos que se envían.
