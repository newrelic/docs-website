[Desinstalar el agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations) no desinstala automáticamente ninguna parte de su [infraestructura integración](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

Para desinstalar [AWS Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream/) o cualquier otra integración AWS , siga los pasos a continuación.

    <tr>
      <td id="disable-aws-integrations">
        Deshabilite la integración AWS CloudWatch Metric Streams
      </td>

      <td>
        Para deshabilitar la integración AWS CloudWatch Metric Streams :

        1. Desde la consola AWS , elimine todos los recursos necesarios que se crearon durante la configuración de la integración AWS , incluidos los flujos de AWS CloudWatch, el extremo AWS Kinesis Data Firehouse y el depósito de respaldo AWS S3. Si utilizó la plantilla de AWS CloudFormation, siga los pasos para [eliminar una pila en la consola de AWS CloudFormation](https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/cfn-console-delete-stack.html).

        2. Complete el paso 1 para cada región activa AWS donde conectó AWS CloudWatch Metric Streams.

        3. En

           <DNT>
             **Infrastructure > AWS > AWS Status Dashboard**
           </DNT>

           , asegúrese de que no se estén incorporando nuevas actualizaciones de métricas en New Relic.

        4. Complete los pasos para [desconectar la cuenta de AWS](#unlink-aws).
      </td>
    </tr>

    <tr>
      <td id="disable-aws-integrations">
        Deshabilite una o más integraciones de sondeo AWS
      </td>

      <td>
        Para deshabilitar servicios mientras mantiene su cuenta de AWS vinculada a New Relic:

        1. Desde

           <DNT>
             **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure**
           </DNT>

           , seleccione

           <DNT>
             **AWS > Manage services**
           </DNT>

           .

        2. Desde su página

           <DNT>
             **Edit AWS account**
           </DNT>

           , desmarque la casilla de verificación de cada servicio activo que desee deshabilitar.

        3. Guarde sus cambios.
      </td>
    </tr>

    <tr>
      <td id="unlink-aws">
        Desconectar la cuenta de AWS
      </td>

      <td>
        Para desconectar completamente su cuenta de AWS de New Relic, debe desvincular su cuenta de AWS. Esto deshabilita toda la integración New Relic asociada con esa cuenta AWS .

        1. Vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS > Manage services**</DNT>.

        2. Desde su página <DNT>**Edit AWS account**</DNT> , seleccione <DNT>**Unlink this account**</DNT>.

        3. Guarde sus cambios.

        4. Inicie sesión en AWS y seleccione [<DNT>**Services > IAM > Roles**</DNT>](https://console.aws.amazon.com/iam/home#roles).

        5. Seleccione la casilla de verificación del rol que desea eliminar y luego seleccione <DNT>**Role Actions > Delete Role**</DNT>.

           Desvincular su cuenta de AWS deshabilitará la relación de confianza configurada a través de su ARN.
      </td>
    </tr>
