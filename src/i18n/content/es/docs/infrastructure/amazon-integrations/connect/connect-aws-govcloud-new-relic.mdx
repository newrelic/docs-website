Las regiones [de AWS GovCloud (EE. UU.)](https://aws.amazon.com/govcloud-us/) están diseñadas para abordar las necesidades regulatorias específicas de Estados Unidos (agencias federales, estatales y locales), las instituciones educativas y el ecosistema de soporte. Es una región aislada AWS diseñada para alojar datos confidenciales y carga de trabajo regulada en la nube, lo que ayuda a los clientes a cumplir con los requisitos de cumplimiento del gobierno de EE. UU. El [conjunto disponible de servicios de AWS](https://aws.amazon.com/govcloud-us/details/) es un subconjunto del ecosistema de AWS.

New Relic te brinda la confianza para desplegar tus servicios más críticos en GovCloud, permitiéndote monitor y observar todo tu ecosistema desde [nuestra plataforma](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one).

La capacidad AWS CloudWatch Metric Streams no está disponible en las regiones de GovCloud.

<Callout variant="tip">
  Puede utilizar Terraform para automatizar el proceso de habilitar la integración en la nube. Lea cómo hacerlo en el [sitio de documentación oficial de Terraform](https://registry.terraform.io/providers/newrelic/newrelic/latest/docs/guides/cloud_integrations_guide).
</Callout>

## Conecte AWS GovCloud a New Relic

Para comenzar a recibir datos de Amazon con la integración de New Relic AWS , conecte su cuenta de Amazon a New Relic.

1. Obtenga [sus credenciales](#govcloud-credentials).

2. Vaya a

   <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > GovCloud**</DoNotTranslate>

   .

3. Haga clic en

   <DoNotTranslate>**Add AWS GovCloud account**</DoNotTranslate>

   .

4. Asigne un nombre a su cuenta de AWS, proporcione las credenciales para conectar su cuenta y haga clic en

   <DoNotTranslate>**Submit**</DoNotTranslate>

   .

5. Seleccione los servicios web de Amazon que desea monitorear con la integración New Relic Infrastructure y luego haga clic en

   <DoNotTranslate>**Save**</DoNotTranslate>

   .

## Cómo obtener credenciales de GovCloud para New Relic [#govcloud-credentials]

1. Desde la [consola de IAM](https://console.amazonaws-us-gov.com/iam/home#/users), haga clic en <DoNotTranslate>**Add user**</DoNotTranslate>.

2. Para <DoNotTranslate>**User name**</DoNotTranslate>, escriba <DoNotTranslate>**NewRelicInfrastructure-Integrations**</DoNotTranslate>. Para <DoNotTranslate>**Select AWS access type**</DoNotTranslate>, seleccione como <DoNotTranslate>**Programmatic access**</DoNotTranslate>.

   <img
     title="AWS-add-user.png"
     alt="Add a user in the AWS IAM console."
     src={infrastructureAwsAddUser}
   />

   <figcaption>
     <DoNotTranslate>**AWS IAM console > Add user**</DoNotTranslate>: agregue <DoNotTranslate>**NewRelicInfrastructure-Integrations**</DoNotTranslate> como usuario.
   </figcaption>

3. Adjunte la política: busque <DoNotTranslate>**ReadOnlyAccess**</DoNotTranslate>, seleccione la casilla de verificación de la política denominada <DoNotTranslate>**ReadOnlyAccess**</DoNotTranslate> y luego haga clic en <DoNotTranslate>**Next: Tags**</DoNotTranslate> (agregar una etiqueta es opcional). Alternativamente, puede [crear su propia política administrada](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) y limitar los permisos que otorga New Relic según los servicios de AWS que desee monitor.

   <img
     title="AWS-set-permissions.png"
     alt="AWS-set-permissions.png"
     src={infrastructureAwsSetPermissions}
   />

   <figcaption>
     <DoNotTranslate>**AWS IAM console > Add user > Set permissions**</DoNotTranslate>: seleccione <DoNotTranslate>**ReadOnlyAccess**</DoNotTranslate>.
   </figcaption>

4. En la página <DoNotTranslate>**Tags**</DoNotTranslate> , haga clic en <DoNotTranslate>**Next: Review**</DoNotTranslate>. Revise el resumen de detalles del usuario y haga clic en <DoNotTranslate>**Create user**</DoNotTranslate>.

   <img
     title="AWS-create-user.png"
     alt="In IAM console, create the user."
     src={infrastructureAwsCreateUser}
   />

   <figcaption>
     <DoNotTranslate>**AWS IAM console > Add user > Set permissions > Tags > Review**</DoNotTranslate>: verifique que la información del nuevo usuario sea precisa.
   </figcaption>

5. Su usuario debería haberse creado correctamente. Descargue las credenciales de seguridad del usuario haciendo clic en el botón <DoNotTranslate>**Download .csv**</DoNotTranslate> y luego haga clic en <DoNotTranslate>**Close**</DoNotTranslate>.
