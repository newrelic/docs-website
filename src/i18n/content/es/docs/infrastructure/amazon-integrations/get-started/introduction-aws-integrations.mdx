La integración de Amazon le permite monitor sus datos AWS en varias características New Relic . Habilitar la integración [AWS CloudWatch Metric Streams](/docs/integrations/amazon-integrations/aws-integrations-list/aws-metric-stream) es la solución recomendada para monitor todas las métricas de CloudWatch de todos los servicios de AWS (incluido el espacio de nombres personalizado). Además de esto, hay disponibles integraciones adicionales para obtener una visibilidad ampliada de los servicios clave de AWS más allá de las métricas de CloudWatch disponibles. Para obtener una referencia completa de las métricas admitidas, consulte las métricas de CloudWatch disponibles para cada servicio en las páginas de documentación AWS .

## Conecte AWS y New Relic [#connect]

Para obtener datos de AWS, siga el procedimiento para [conectar AWS a New Relic](/docs/infrastructure/amazon-integrations/getting-started/connect-aws-services-infrastructure).

Se puede habilitar una integración <DNT>**API Polling**</DNT> adicional además de AWS CloudWatch Metric Streams para extraer datos que no están disponibles como métricas de CloudWatch. Las siguientes integraciones no son reemplazadas por las corrientes métricas:

* AWS Billing
* AWS CloudTrail
* AWS Health
* AWS Trusted Advisor
* AWS VPC
* AWS X-Ray

Finalmente, otras integraciones pueden requerir una configuración adicional en su cuenta AWS :

* [AWS VPC Flow Logs](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-vpc-flow-logs-monitoring-integration/)
* [Formación en la nube de AWS](https://github.com/newrelic/cloudformation-partner-integration)

## Integración y costos AWS [#cloudwatch-billing]

Tenga en cuenta los siguientes elementos:

* El precio AWS CloudWatch Metric Streams se define en función del número de actualizaciones métricas. Para obtener información actualizada sobre precios, consulte [Precios de AWS CloudWatch](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Metric-Streams.html).

* AWS Kinesis Data Firehose se utiliza como método de entrega. Para obtener más información, consulte la [página de precios de AWS Firehose](https://aws.amazon.com/kinesis/data-firehose/pricing/).

* AWS Config se puede habilitar opcionalmente en su cuenta de AWS y usarse para enriquecer las métricas de CloudWatch con metadatos de recursos. Con AWS Config, se le cobra según la cantidad de elementos de configuración registrados. Consulte la [página de precios de AWS Config](https://aws.amazon.com/config/pricing/) para obtener más detalles.

* [AWS de etiquetas Resource Groups API](https://docs.aws.amazon.com/resourcegroupstagging/latest/APIReference/overview.html) se utiliza para enriquecer la métrica con una etiqueta personalizada definida en los AWS recursos asociados (aplicable tanto a flujos de sondeo como de métrica).

* Si la integración de sondeo está habilitada (en lugar de flujos métricos), New Relic utiliza la API de Amazon CloudWatch para obtener métricas de los servicios de AWS que monitor. La cantidad de llamadas a la API de CloudWatch aumenta a medida que habilita una mayor integración. Agregue recursos AWS a esas integraciones o escale esas integraciones en más regiones. Esto puede hacer que las solicitudes a la API de CloudWatch excedan los límites de

  <DNT>
    **1 million free**
  </DNT>

  otorgados por AWS y [aumenten su factura de CloudWatch](/docs/infrastructure/amazon-integrations/troubleshooting/cloudwatch-billing-increase).

* AWS ofrece un monitoreo mejorado para algunos de sus servicios, lo que permite una medición más métrica y más frecuente. Por ejemplo, consulte [Costos de monitoreo mejorados de RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_Monitoring.OS.overview.html#USER_Monitoring.OS.cost).

## Ver sus datos de AWS [#insights]

Una vez que finalice la configuración, los datos de sus servicios de AWS se reportan directamente a New Relic. La entidad AWS para muchos servicios se enumera en nuestro [explorador de entidades](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer). métrica y evento aparecen en [métrica y evento](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/). Los datos AWS también son visibles en la UI de infraestructura.

Para ver sus datos de AWS: vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS**</DNT>. Para cualquiera de las integraciones AWS enumeradas:

* Para transmisiones activas, seleccione el enlace <DNT>**Explore your data**</DNT> .

  O

* Para otras integraciones, explore el dashboard disponible o haga clic en el enlace <DNT>**Explore Data**</DNT>.

Puede ver y reutilizar la consulta NRQL tanto en el panel preconfigurado <InlinePopover type="dashboards"/>como en el <DNT>**Events explorer**</DNT>. Esto le permite adaptar la consulta a sus necesidades específicas.

## Disponibilidad de región [#availability]

La mayoría de los servicios de AWS ofrecen un extremo regional para reducir la latencia de datos entre los recursos de la nube y la aplicación. New Relic puede obtener datos de monitoreo de servicios y extremos que están ubicados en todas [las regionesAWS ](http://docs.aws.amazon.com/general/latest/gr/rande.html), excepto China.
