La integración Prometheus de New Relic lo ayuda a obtener experiencias seleccionadas para servicios que exponen métricas en formato Prometheus o Open métrica, agrupando todo lo que necesita para comenzar a monitorear como un profesional desde el primer momento.

Estas experiencias incluyen el servicio entidad en entidad explorer con métricas doradas y tablero de entidad, un dashboard descripción general del servicio y alertas recomendadas.

## Empezar [#get-started]

Para comenzar necesitas enviar [datos métricos de Prometheus](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic) a New Relic. Cada servicio puede requerir requisitos previos adicionales para habilitar y configurar el extremo métrico que se va a desechar. Puede revisar configuraciones y requisitos específicos para cada servicio en particular en la [lista de integración de Prometheus](/docs/infrastructure/prometheus-integrations/integrations-list/integrations-list-intro#list).

## Encuentra y usa los datos. [#find-use-data]

### Métrica

Prometheus métrica se almacenan como [dimensionales métricas](/docs/data-apis/understand-data/metric-data/query-metric-data-type/). Puede consultar utilizando [NRQL](/docs/telemetry-data-platform/get-data/apis/query-metric-data-type/) o utilizar el Explorador de datos para explorar las métricas y facetas disponibles y filtrar por dimensiones.

### Entidad

La mayor parte de la integración da como resultado que esos monitores de entidades estén disponibles en nuestro [explorador de entidades](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer). Esta integración permite capacidades generales de la plataforma, incluidas métricas doradas, panel de entidad, carga de trabajo y nivel de servicio.

### Panel

Puede configurar sus servicios <InlinePopover type="dashboards"/>a través de [nuestra página de observabilidad instantánea](https://onenr.io/06vjAmnD3jP). El panel proporciona una descripción general de alto nivel del rendimiento del servicio y sigue las mejores prácticas de observabilidad recomendadas para cada servicio. Una vez que los haya importado, puede adaptarlos y personalizarlos según sus requisitos específicos.

### Alerta

Puede importar la política de alertas recomendada desde [nuestra página de observabilidad instantánea](https://onenr.io/06vjAmnD3jP). Una vez importadas, [las condiciones de alertaNRQL ](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions)están disponibles, pero no se habilitarán automáticamente. Revise la configuración de notificaciones y ajuste la condición según sea necesario antes de activarlas.
