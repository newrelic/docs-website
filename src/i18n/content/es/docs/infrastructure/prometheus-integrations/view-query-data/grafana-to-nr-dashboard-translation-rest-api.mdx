Puede usar la API REST de traducción de dashboard de Grafana a New Relic para convertir su dashboard de Grafana existente en un dashboard nativo de New Relic. Esta API puede ayudar en la migración de Grafana a New Relic para los clientes que actualmente utilizan el plug-in de Grafana para visualizar sus datos de New Relic. Ya sea que busque consolidar sus herramientas de observabilidad, migrar toda su configuración de monitoreo a New Relic o simplemente probar los dashboards de New Relic junto con su configuración existente de Grafana, esta API hace que sea más fácil comenzar.

Esta API utiliza la lógica de traducción de PromQL de New Relic para traducir sus consultas. Para ver una lista de las características y funciones de PromQL admitidas, consulte [Compatibilidad de características de PromQL](/docs/infrastructure/prometheus-integrations/view-query-data/supported-promql-features).

## Requisitos previos [#prerequisites]

Antes de usar esta API, necesitará:

* Una [clave de API de usuario de New Relic](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)
* Su dashboard de Grafana exportado como JSON (en Grafana: Configuración → Modelo JSON → Copiar al portapapeles)
* Tu ID de cuenta de New Relic (si deseas anular la cuenta predeterminada)

## Realice una solicitud a los extremos de API [#api-endpoints]

Para traducir un dashboard de Grafana a un dashboard de New Relic, envíe una solicitud `POST` al extremo adecuado para su región con el dashboard que desea traducir en el cuerpo de la solicitud.

* Para los EE.UU., `https://prometheus-api.newrelic.com/api/v1/translate/dashboard/grafana`
* Para la UE, `https://prometheus-api.eu.newrelic.com/api/v1/translate/dashboard/grafana`

### Autenticación [#authentication]

Todas las llamadas API requieren una [clave de API de usuario](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key). Incluya esta clave en el encabezado `x-api-key` de su solicitud.

### Cuerpo de la solicitud [#request-body]

El cuerpo de la solicitud debe ser un objeto JSON con el siguiente formato:

```
{
  "accountId": int [optional],
  "dashboard": object [required]
}
```

Donde `dashboard` es el objeto JSON del dashboard de Grafana que desea traducir, y `accountId` es la cuenta desde la que desea que su dashboard traducido consulte datos. Si no se incluye `accountId`, el dashboard traducido consultará datos de la cuenta del usuario que realiza la solicitud.

### Formato de respuesta [#response-format]

La API devuelve un objeto JSON que contiene el dashboard de New Relic traducido:

```json
{
  "dashboard": {
    "name": "Translated Dashboard",
    "permissions": "PUBLIC_READ_WRITE",
    "pages": [...]
    }
}
```

## Resolución de errores de traducción [#troubleshooting]

Cuando el traductor encuentra problemas que impiden la traducción completa de un widget o consulta, crea &quot;Action Items&quot; — widgets especiales de Markdown que explican qué salió mal y sugieren correcciones.

## Ejemplos [#examples]

<CollapserGroup>
  <Collapser id="dashboard-translation-basic" title="Ejemplo: llamada API básica para traducir un dashboard de Grafana">
    ```bash
    export NR_USER_KEY="your_user_key"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --slurpfile dashboard "${JSON_FILE}" '{dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-account-override" title="Ejemplo: traducir dashboard con sobrescritura del ID de la cuenta">
    Si desea sobrescribir la cuenta desde la cual su dashboard consultará datos, puede incluir `accountId: int` en la carga de la solicitud. Esto hará que todas las consultas en el dashboard consulten datos de la cuenta especificada.

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-and-upload" title="Ejemplo: subir el dashboard traducido a New Relic usando la API de NerdGraph">
    <Callout variant="tip">
      Para obtener más información sobre cómo usar NerdGraph y encontrar tu extremo regional, consulta [Introducción a New Relic NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).
    </Callout>

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)
    export NERDGRAPH_ENDPOINT="US/EU Regional Nerdgraph endpoint"

    NR_DASHBOARD=$(curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard')


    curl -X POST ${NERDGRAPH_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "API-Key: ${NR_USER_KEY}" \
    -d '{
    "query": "mutation($accountId: Int!, $dashboard: DashboardInput!) { dashboardCreate(accountId: $accountId, dashboard: $dashboard) { entityResult { guid name } errors { description } } }",
    "variables": {
    "accountId": '${ACCOUNT_ID}',
    "dashboard": '${NR_DASHBOARD}'
    }
    }'
    ```
  </Collapser>
</CollapserGroup>
