## Problema

Al ejecutar la integración de Prometheus OpenMetrics para Kubernetes, observa reinicios y lagunas en los datos enviados a New Relic.

## Solución

Al ejecutar la integración de Prometheus OpenMetrics para Kubernetes con 500 000 puntos de datos por minuto, asegúrese de establecer estos límites:

* Límite de CPU: 1 núcleo
* Límite de memoria: 1Gb

Recomendación: ejecute siempre el raspador de Kubernetes con una réplica.

Agregar más réplicas dará como resultado datos duplicados. Si los límites de CPU y memoria no son suficientes, esto puede provocar reinicios y lagunas en los datos.

Para verificar el estado y reiniciar el evento del raspador:

```
kubectl describe pod -l "app=nri-prometheus"
```
