Nuestra instalación guiada crea un comando CLI personalizado para su entorno que descarga e instala la CLI de New Relic y el agente de infraestructura. Le recomendamos que consulte nuestra [guía de mejores prácticas de monitoreo de infraestructura](/docs/new-relic-solutions/best-practices-guides/full-stack-observability/infrastructure-monitoring-best-practices-guide/) antes de comenzar.

Nuestra instalación guiada descubre la aplicación, la infraestructura y las fuentes log que se ejecutan en su entorno y recomienda cuáles deben instrumentarse. La instalación automatiza la configuración y despliegue de cada sistema que elijas para el instrumento.

## Por qué es importante [#why-it-matters]

Con nuestra instalación guiada, puede instrumentar su aplicación e infraestructura y comenzar a ver sus datos en New Relic en minutos.

La instalación guiada utiliza nuestra interfaz de línea de comando (CLI), el [agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent) para su entorno anfitrión y una [biblioteca de recetas de instalación](https://github.com/newrelic/open-install-library) para instrumentar su aplicación e infraestructura para usted. Eso significa menos trabajo para usted.

Debido a que nuestras recetas de instrumentación son de código abierto, puede modificar recetas existentes o crear otras nuevas para satisfacer sus necesidades.

## Banderas CLI [#cli-flags]

Personalice y depure su instalación con indicadores CLI. Utilice el indicador &quot;asumir sí&quot; `-y` para la instalación automatizada con valores predeterminados razonables, cambie la detalle de la salida con `--debug` o `--trace`, agregue una etiqueta o use recetas de instalación personalizadas. Puede encontrar una lista completa de opciones disponibles en nuestro repositorio de código abierto en [GitHub](https://github.com/newrelic/newrelic-cli/blob/main/docs/cli/newrelic_install.md).

## Algunos detalles técnicos [#technical-details]

La instalación guiada New Relic utiliza recetas de instalación de código abierto para la integración del instrumento en el host. Estas recetas incluyen comandos de instalación y configuración, información sobre el registro y metadatos relacionados con lo que se está instalando. Están escritos en formato YAML para cada tipo de sistema a instrumentar y tienen todos los detalles de instalación necesarios para instalar el agente de infraestructura para una integración específica. Hay información adicional disponible en nuestro repositorio de código abierto en [GitHub](https://github.com/newrelic/open-install-library/blob/main/docs/README.md).

## Recetas [#recipes]

La instalación guiada automatiza el descubrimiento, configuración e instalación de New Relic agente e integración. Sin embargo, puede haber ocasiones en las que desee implementarlos uno por uno utilizando el comando de instalación CLI.

Para instalar cualquier agente e integración individual, usaría un comando similar al siguiente, que especifica el tipo de integración que desea instalar. Esta es la sintaxis para Linux:

```bash
curl -Ls https://download.newrelic.com/install/newrelic-cli/scripts/install.sh | bash && sudo NEW_RELIC_API_KEY=INSERT_YOUR_API_KEY NEW_RELIC_ACCOUNT_ID=INSERT_YOUR_ACCOUNT_ID NEW_RELIC_REGION=INSERT_YOUR_REGION /usr/local/bin/newrelic install -n INSERT_THE_RECIPE_NAME
```

Por ejemplo, para instalar la integración de Apache:

```bash
curl -Ls https://download.newrelic.com/install/newrelic-cli/scripts/install.sh | bash && sudo NEW_RELIC_API_KEY=INSERT_YOUR_API_KEY NEW_RELIC_ACCOUNT_ID=INSERT_YOUR_ACCOUNT_ID /usr/local/bin/newrelic install -n apache-open-source-integration
```

Esta es la sintaxis para Windows:

```powershell
[Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; (New-Object System.Net.WebClient).DownloadFile("https://download.newrelic.com/install/newrelic-cli/scripts/install.ps1", "$env:TEMP\install.ps1"); & PowerShell.exe -ExecutionPolicy Bypass -File $env:TEMP\install.ps1; $env:NEW_RELIC_API_KEY='INSERT_YOUR_API_KEY'; $env:NEW_RELIC_ACCOUNT_ID='INSERT_YOUR_ACCOUNT_ID'; $env:NEW_RELIC_REGION='INSERT_YOUR_REGION'; & 'C:\Program Files\New Relic\New Relic CLI\newrelic.exe' install
```

<Callout variant="important">
  En Windows, nuestra instalación guiada solo admite Microsoft SQL Server, log y el agente de infraestructura.

  Todas las demás integraciones solo son compatibles con Linux.
</Callout>

Nuestro código abierto agente e integración envían rendimiento métrico y datos de inventario desde sus servidores y aplicación a la plataforma New Relic . Puede ver <InlinePopover type="dashboards" />prediseñado de sus datos métricos, crear políticas de alertas y crear sus propias consultas y gráficos personalizados.

## Instalar versiones específicas del agente de infraestructura [#install-specific-versions]

La CLI ahora admite la instalación de versiones específicas del agente de infraestructura. Esta característica es particularmente útil para entornos que requieren coherencia en todas las implementaciones o necesitan mantener la compatibilidad con sistemas específicos.

De forma predeterminada, si no especifica una versión, la CLI instala automáticamente la última versión disponible del agente de infraestructura. Para instalar una versión específica, agregue el número de versión al nombre de la receta usando el símbolo `@` :

```shell
newrelic install -n infrastructure-agent-installer@X.XX.X
```

Donde `X.XX.X` representa el número de versión (por ejemplo, `1.64.0`). Para encontrar las versiones disponibles del agente de infraestructura y sus detalles de lanzamiento, consulte las [notas de lanzamiento del agente de infraestructura](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/).

Para Linux:

```bash
curl -Ls https://download.newrelic.com/install/newrelic-cli/scripts/install.sh | bash && sudo NEW_RELIC_API_KEY=INSERT_YOUR_API_KEY NEW_RELIC_ACCOUNT_ID=INSERT_YOUR_ACCOUNT_ID NEW_RELIC_REGION=INSERT_YOUR_REGION /usr/local/bin/newrelic install -n infrastructure-agent-installer@1.64.0
```

Para Windows:

```powershell
[Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; (New-Object System.Net.WebClient).DownloadFile("https://download.newrelic.com/install/newrelic-cli/scripts/install.ps1", "$env:TEMP\install.ps1"); & PowerShell.exe -ExecutionPolicy Bypass -File $env:TEMP\install.ps1; $env:NEW_RELIC_API_KEY='INSERT_YOUR_API_KEY'; $env:NEW_RELIC_ACCOUNT_ID='INSERT_YOUR_ACCOUNT_ID'; $env:NEW_RELIC_REGION='INSERT_YOUR_REGION'; & 'C:\Program Files\New Relic\New Relic CLI\newrelic.exe' install -n infrastructure-agent-installer@1.62.0
```

<Callout variant="important">
  Esta característica funciona en todos los hosts Linux y Windows , pero no es compatible con macOS. Al instalar en macOS, siempre se instalará la última versión, independientemente de la versión especificada.
</Callout>

## Solucionar problemas comunes [#troubleshoot]

A medida que identifiquemos áreas donde falla la instalación guiada, las documentaremos aquí y brindaremos alguna guía de resolución de problemas.

<CollapserGroup>
  <Collapser className="freq-link" id="mysql" title="MySQL: permisos de usuario incorrectos">
    Para monitor los datos de salud MySQL, necesita un nombre de usuario y una contraseña válidos con permisos específicos.

    Estos comandos crearán un usuario y otorgarán los permisos necesarios:

    Cree un usuario `newrelic@localhost` con una contraseña específica.

    ```bash
    sudo mysql -e "CREATE USER 'newrelic'@'localhost' IDENTIFIED BY 'YOUR_SELECTED_PASSWORD';"
    ```

    Otorgue privilegios de replicación a `newrelic@localhost` con un máximo de 5 conexiones.

    ```bash
    sudo mysql -e "GRANT REPLICATION CLIENT ON *.* TO 'newrelic'@'localhost' WITH MAX_USER_CONNECTIONS 5;"
    ```

    Otorgue privilegios de selección a `newrelic@localhost` con un máximo de 5 conexiones.

    ```bash
    sudo mysql -e "GRANT SELECT ON *.* TO 'newrelic'@'localhost' WITH MAX_USER_CONNECTIONS 5;"
    ```

    Una vez hecho esto, su próximo intento de instalación guiada debería funcionar.
  </Collapser>

  <Collapser className="freq-link" id="nginx" title="NGINX: Sin URL de estado">
    Para monitor su servidor NGINX, deberá configurar una URL de estado válida.

    `status_url`: La URL configurada para proporcionar la métrica usando el módulo de estado. Si el valor predeterminado de `127.0.0.1` es incorrecto, sustituya la dirección/FQDN/URL de su sistema. Ejemplo:

    `status_url: http://127.0.0.1/status`

    Puede leer más sobre `status_url` en estos documentos de NGINX:

    * Para NGINX código abierto: [módulo de estado de código auxiliar HTTP](https://nginx.org/en/docs/http/ngx_http_stub_status_module.html)

    * Para NGINX Plus: [módulo de estado HTTP](https://nginx.org/en/docs/http/ngx_http_status_module.html) y [módulo API HTTP](http://nginx.org/en/docs/http/ngx_http_api_module.html)

      Hay diferentes formas de configurar `status_url`, dependiendo de cómo se instaló NGINX:

    * Si está habilitado a través de Kubernetes: consulte [Supervisar servicios que se ejecutan en Kubernetes](/docs/monitor-service-running-kubernetes).

    * Si está habilitado a través de Amazon ECS: consulte [Monitorear servicios que se ejecutan en ECS](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs).

    * Si está instalado en el host: edite la configuración en el archivo de configuración YAML de la integración, `nginx-config.yml`.
  </Collapser>
</CollapserGroup>
