Antes de [instalar nuestro agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic), asegúrese de que su sistema y cualquier integración en el host que configure cumplan con los requisitos. También necesitas una cuenta New Relic. [Registrate gratis](https://newrelic.com/signup). No se requiere tarjeta de crédito.

## Arquitectura del procesador [#processors]

El agente de infraestructura admite esta arquitectura de procesador:

* <DNT>**Linux**</DNT>: Arquitectura de procesador de 64 bits para x86 (también requiere administrador de paquetes de 64 bits y dependencia)
* <DNT>**Windows**</DNT>: Arquitectura de procesador de 64 bits para x86
* <DNT>**ARM**</DNT>: La arquitectura arm64, incluido [el procesador AWS Graviton 2,](https://aws.amazon.com/ec2/graviton/) es compatible con sistemas operativos Linux compatibles. integración en el host también son compatibles (con la excepción de la integración de Oracle).
* <DNT>**macOS**</DNT>: tanto x86 de 64 bits como Apple Silicon.

## Sistema operativo [#os]

El agente de infraestructura apoya a estos sistemas operativos hasta el final de la vida útil de su fabricante. Es posible que el agente aún sea compatible con versiones anteriores, pero consulte las instrucciones paso a paso para cada sistema operativo si la instalación guiada ya no está disponible para una versión específica del sistema operativo.

* Para ECS clúster, implementar la [integración ECS](/docs/integrations/elastic-container-service-integration/installation/install-ecs-integration).

## Nombre único de host [#hostname]

El agente de infraestructura utiliza el nombre de host para identificar de forma única cada servidor. Para evitar métricas inexactas al combinar varios servidores bajo un solo nombre de host, asegúrese de que cada servidor de monitor tenga un nombre de host único.

* Puede utilizar la configuración opcional [`display_name`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#display_name) para anular el nombre de host predeterminado.
* Los servidores llamados `localhost` no se informan porque es un nombre predeterminado e inherentemente no único.

## Permisos

El agente de infraestructura requiere estos permisos:

* <DNT>**Linux**</DNT>: De forma predeterminada, [el agente se ejecuta e instala como root](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#agent-mode-intro). También puede seleccionar modos de ejecución [privilegiados](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-privileged) o [no privilegiados](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-unprivileged) .
* <DNT>**Windows**</DNT>: El agente debe instalarse desde una cuenta de administrador y requiere privilegios de administrador para ejecutarse.
* <DNT>**macOS**</DNT>: El agente se puede instalar desde cualquier cuenta de usuario.

## Bibliotecas

Para [las versiones del agente 1.1.19 o superiores](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes), necesita la biblioteca `libcap` para poder realizar la instalación. Está disponible en el repositorio oficial de su distribución.

## Acceso a la red [#network]

Para reportar datos a New Relic, nuestro agente de infraestructura debe tener acceso de salida a [ciertos dominios y puertos](/docs/apm/new-relic-apm/getting-started/networks#infrastructure). Si su sistema necesita un proxy para conectarse a estos dominios, utilice la configuración [`proxy`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#proxy) .

## Software de contenedor [#container]

El agente de infraestructura [instrumenta el contenedor Docker](/docs/infrastructure-docker-instrumentation) cuando se instala en el servidor host. Admitimos las versiones 1.12 o superiores Docker .

## Uso de CPU, memoria y disco [#resources]

El agente de infraestructura es bastante liviano. Para conocer el uso típico de CPU, memoria y disco, consulte nuestra página sobre [sobrecarga de rendimiento del agente](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead). Para obtener más información sobre los sistemas de archivos compatibles, consulte [atributo de muestra de almacenamiento](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#storagesample-attributes).
