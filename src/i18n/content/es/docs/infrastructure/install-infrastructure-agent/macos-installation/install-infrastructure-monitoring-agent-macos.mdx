Con el agente de monitoreo de infraestructura de New Relic para macOS, puede monitor el rendimiento métrico clave en hosts macOS. El agente puede ejecutarse en su propio hardware o en sistemas en la nube como Amazon EC2.

Nuestro agente de monitoreo de infraestructura [es compatible](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent) con todas las versiones de macOS generalmente disponibles y compatibles con Apple.

## Instalar para MacOS [#install-macos]

Antes de la instalación, debes revisar los [requisitos](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure). Luego, para instalar el agente de monitoreo de infraestructura para macOS, puede utilizar nuestra [instalación guiada](/docs/full-stack-observability/observe-everything/get-started/new-relic-guided-install-overview/) o seguir las instrucciones de este documento para completar una instalación básica.

## Limitaciones [#limitations]

Las siguientes capacidades del agente de infraestructura aún no están disponibles en macOS:

* [Reenviador de Logs](/docs/logs/enable-log-management-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent/)
* [integración en el host](/docs/integrations/host-integrations/get-started/introduction-host-integrations/) (incluida la integración integrada como docker y Flex)
* Despliegue automatizado a través de herramientas de gestión de configuración (Chef, Ansible, Puppet)

## Instrucciones paso a paso [#instructions]

Para instalar el agente de monitoreo de infraestructura, siga las instrucciones paso a paso:

1. Revisar los [requisitos del agente y el sistema operativo soportado](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure).

2. Asegúrese de que [Homebrew](https://brew.sh/) esté instalado en el sistema. Puedes comprobar tu instalación de Homebrew con:

   ```bash
     which brew
   ```

   Si no está instalado, puede instalarlo con este comando (o consulte las instrucciones actualizadas [de Homebrew](https://brew.sh/) ):

   ```bash
     /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

3. Luego, abra la terminal y ejecute el siguiente comand::

   ```bash
     brew install newrelic-infra-agent -q
   ```

4. Inicie el servicio de agente de infraestructura:

   ```bash
     brew services start newrelic-infra-agent
   ```

5. Crea el archivo de configuración y agrega tu <InlinePopover type="licenseKey"/>:

   Intel x86:

   ```bash
     sudo mkdir -p /usr/local/etc/newrelic-infra/
     echo "license_key: YOUR_LICENSE_KEY" | sudo tee -a /usr/local/etc/newrelic-infra/newrelic-infra.yml
   ```

   Silicio de manzana:

   ```bash
     sudo mkdir -p /opt/homebrew/etc/newrelic-infra/
     echo "license_key: YOUR_LICENSE_KEY" | sudo tee -a /opt/homebrew/etc/newrelic-infra/newrelic-infra.yml
   ```

Espere unos minutos y luego [vea su servidor en la UI de infraestructura](/docs/infrastructure/infrastructure-ui-pages/infra-ui-overview). Si no aparecen datos después de esperar unos minutos, sigue los [pasos de resolución de problemas](/docs/infrastructure/new-relic-infrastructure/troubleshooting/no-data-appears-infrastructure).

## ¿Que sigue? [#configure]

La única [opción de configuración](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent) requerida es la configuración [`license_key`](/docs/Infrastructure-configure-your-agent#conf-license_key) , que se crea como parte de los procedimientos de instalación. Quizás también quieras:

* [Agregue un atributo personalizado](/docs/Infrastructure-configure-your-agent#conf-custom_attributes) para anotar los datos de su infraestructura.
* [Conecte su cuenta de AWS](/docs/infrastructure-amazon-aws-ec2-integration#connect) si sus servidores están alojados en Amazon EC2.
* Agregue otra [integraciónNew Relic Infrastructure ](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)para recopilar datos de servicios externos.

Para actualizar a la última versión, siga los procedimientos estándar para [actualizar el agente de monitoreo de infraestructura](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent).
