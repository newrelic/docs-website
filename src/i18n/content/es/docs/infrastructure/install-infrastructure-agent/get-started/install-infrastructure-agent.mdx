El agente de monitoreo de infraestructura de New Relic es un archivo ejecutable [liviano](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead) que recopila datos sobre sus hosts. También informa datos de [algunos servicios de terceros](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations), si están habilitados, y también [registra datos](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent).

Actualmente, el agente de monitoreo de infraestructura puede ejecutarse en muchas distribuciones de Linux, Windows y macOS. Hay varias formas de instalar y desplegar el agente, según su configuración y necesidades. Este documento describe cómo funciona el agente de monitoreo de infraestructura y cómo instalarlo. Necesitará una cuenta New Relic, así que si aún no tiene una, [créela](https://newrelic.com/signup) gratis para siempre.

## Inicio rápido: utilice nuestra instalación guiada [#quick]

La forma más rápida de comenzar con nuestro agente de monitoreo de infraestructura es mediante nuestra instalación guiada. Nuestra instalación guiada no solo instala el agente de infraestructura, sino que también intenta descubrir servicios y fuentes log en su entorno y recomienda cuáles instrumentar.

¿Listo para comenzar? Haga clic en un botón según [la región de su centro de datos de New Relic](/docs/accounts/accounts-billing/account-setup/choose-your-data-center).

## Instalar el agente de monitoreo de infraestructura [#install]

### Linux [#Linux]

Si aún no tienes una cuenta de New Relic, la instalación guiada no funcionará. Si quieres seguir el procedimiento manualmente, [consulta nuestro tutorial](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/).

### Windows [#Windows]

Si aún no tienes una cuenta de New Relic, la instalación guiada no funcionará. Si desea seguir el procedimiento manualmente utilizando nuestro instalador MSI, [consulte nuestro tutorial](/docs/infrastructure/install-configure-manage-infrastructure/windows-installation/install-infrastructure-windows-server-using-msi-installer).

## Un agente, muchas capacidades [#overview]

Nuestro agente de monitoreo de infraestructura recopila datos de rendimiento y estado sobre los recursos y procesos del sistema del host donde está habilitado (local o virtualizado). Al mismo tiempo, actúa como reenviador de dos tipos de datos: servicios centrales métricos, que son recopilados por [integración en el host](/docs/integrations/host-integrations/getting-started/introduction-host-integrations), y [registros](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent).

Si desea recopilar datos sobre los servicios principales que se ejecutan en su host, primero debe [instalar el agente de monitoreo de infraestructura](#install) y luego [instalar o habilitar la integración en el host](/docs/integrations/host-integrations/installation/install-infrastructure-host-integrations).

## Comprueba el código fuente [#source-code]

El agente de monitoreo de infraestructura es un software de código abierto. Eso significa que puedes [explorar su código fuente](https://github.com/newrelic/infrastructure-agent) y [enviar mejoras](https://github.com/newrelic/infrastructure-agent/blob/master/CONTRIBUTING.md), o crear tu propia bifurcación y compilarla. Para obtener más información, consulte el [archivo README](https://github.com/newrelic/infrastructure-agent/blob/master/README.md).

## Que sigue

Después de haber instalado el agente de monitoreo de infraestructura:

* Aprenda a [configurar el agente](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent) o editar la [plantilla de configuración](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example).
* Instale [la integración en el host](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) (por ejemplo, Apache o MySQL).
* [Habilite el reenvío de registros utilizando el agente de infraestructura](/docs/logs/enable-new-relic-logs/1-enable-logs/forward-your-logs-using-new-relic-infrastructure).
* Aprenda a [gestionar el agente](/docs/infrastructure/install-infrastructure-agent/manage-your-agent).
