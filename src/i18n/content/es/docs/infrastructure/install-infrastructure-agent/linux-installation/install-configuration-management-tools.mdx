Tiene varias formas de comenzar con nuestro [agente de monitoreo de infraestructura](/docs/infrastructure/infrastructure-monitoring/get-started/get-started-infrastructure-monitoring/):

* A través de nuestra [instalación guiada](/docs/new-relic-one/use-new-relic-one/cross-product-functions/install-configure/new-relic-guided-install-overview/).
* Siguiendo las instrucciones paso a paso para [instalar el agente manualmente](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux#manual-install).
* Utilizar herramientas de gestión de configuración.

Para instalar el agente de monitoreo de infraestructura con una herramienta de administración de configuración, haga clic en el siguiente mosaico:

## Instalar usando archivos tarball [#custom-setups-tarball]

Para <DNT>**custom**</DNT> escenarios de configuración, puede instalar el agente de monitoreo de infraestructura utilizando [nuestros archivos tarball](https://download.newrelic.com/infrastructure_agent/binaries/linux/) en modo [asistido](/docs/infrastructure/install-configure-manage-infrastructure/linux-installation/tarball-assisted-install-infrastructure-linux) o [manual](/docs/infrastructure/install-configure-manage-infrastructure/linux-installation/tarball-manual-install-infrastructure-linux) .

Esto es especialmente útil cuando necesitas adaptar la configuración de instalación predeterminada a tu entorno, o instalar el agente de monitoreo de infraestructura en distribuciones que carecen del paquete `newrelic-infra` en su repositorio.

<Callout variant="important">
  Tenga en cuenta que la instalación personalizada del agente de infraestructura mediante archivos tarball no es compatible oficialmente.
</Callout>
