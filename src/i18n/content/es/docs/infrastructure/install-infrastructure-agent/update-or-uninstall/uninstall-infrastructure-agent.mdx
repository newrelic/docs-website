Sus hosts se agregan automáticamente cuando instala el agente de infraestructura para su sistema operativo [Linux](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-linux) o [Windows](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server), o [actualiza](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent) el agente. De manera similar, sus hosts desaparecen automáticamente cuando desinstala el agente. No es necesario agregar ni eliminar manualmente sus hosts.

La desinstalación del agente no afecta directamente ninguna de sus [integraciones](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure). Para desinstalar una integración, consulte [Desinstalar integración](/docs/infrastructure/install-configure-infrastructure/update-or-uninstall/uninstall-integrations).

<Callout variant="important">
  Dado que no hay ningún script automatizado en el proceso de desinstalación, es posible que queden archivos antiguos al desinstalar. Asegúrese de eliminar manualmente los archivos obsoletos.
</Callout>

## Desinstalar el agente de infraestructura de Linux [#uninstall-linux]

Si utilizó el procedimiento de instalación predeterminado para su agente de infraestructura para entornos Linux, use sus herramientas de administración de paquetes para desinstalarlo. No es necesario detener el servicio antes de ejecutar el comando de desinstalación.

<CollapserGroup>
  <Collapser
    id="apt-uninstall"
    title={<>Desinstalar con apt (<img src="/images/os_icon_debian.webp" title="Debian icon" alt="Debian.webp" style={{ height: '32px', width: '32px', verticalAlign: 'middle' }}>debian,<img src="/images/os_icon_ubuntu.webp" title="ubuntu icon" alt="ubuntu icon" style={{ height: '32px', width: '32px', verticalAlign: 'middle' }}/>ubuntu)</img></>}
  >
    Ejecute el siguiente comando como root:

    ```
    sudo apt-get remove newrelic-infra
    ```
  </Collapser>

  <Collapser
    id="yum-uninstall"
    title={<>Desinstalar con yum (<img src="/images/os_icon_amazon-linux.webp" title="amazon linux.webp" alt="amazon linux.webp" style={{ height: '32px', width: '32px', verticalAlign: 'middle' }}>amazon linux,<img src="/images/os_icon_centos.webp" title="centos icon" alt="centos icon" style={{ height: '32px', width: '32px', verticalAlign: 'middle' }}/>CentOS,<img src="/images/os_icon_redhat.webp" title="redhat icon" alt="redhat icon" style={{ height: '32px', width: '32px', verticalAlign: 'middle' }}/>RHEL)</img></>}
  >
    Ejecute el siguiente comando como root:

    ```
    sudo yum remove newrelic-infra
    ```
  </Collapser>

  <Collapser
    id="zipper-uninstall"
    title={<>Desinstalar con Zypper (<img src="/images/os_icon_suse.webp" title="suse icon" alt="suse icon" style={{ height: '32px', width: '32px', verticalAlign: 'middle' }}>SLES)</img></>}

    Ejecute el siguiente comando como root:

    ```
    sudo zypper -n remove newrelic-infra
    ```
  </Collapser>
</CollapserGroup>

Si siguió un proceso [de](/docs/infrastructure/install-configure-infrastructure/linux-installation/assisted-install-infrastructure-linux) [instalación manual](/docs/infrastructure/install-configure-infrastructure/linux-installation/manual-install-infrastructure-linux) o asistida, deberá eliminar manualmente todos los archivos creados al descomprimir el archivo del agente.

<Callout variant="important">
  También debe desinstalar cualquier integración de monitoreo de la nube como nuestra integración de Azure o Google Cloud Platform . De lo contrario, seguirás viendo el host en tu UI sin datos.
</Callout>

## Desinstalar el<img style={{ width: '32px', height: '32px', verticalAlign: 'middle'}} class="inline" title="Windows Server" alt="Windows Server icon" src="/images/os_icon_windows.webp">agente de infraestructura windows</img> [#uninstall-windows]

Si utilizó el procedimiento de instalación predeterminado para el agente de infraestructura para entornos Windows, para desinstalar:

1. Asegúrese de tener [derechos de administrador](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure#permissions) en su grupo de administración de Windows.

2. [Detenga al agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status#windows).

3. Desde Windows

   <DNT>
     **Control Panel**
   </DNT>

   , utilice la herramienta

   <DNT>
     **Add/Remove Programs and Features**
   </DNT>

   para desinstalar el agente de infraestructura.

4. Desde Windows

   <DNT>
     **Program Files**
   </DNT>

   , elimine manualmente la carpeta New Relic para eliminar todos los archivos asociados con el agente de infraestructura para Windows.

Si siguió un proceso [de](/docs/infrastructure/install-configure-infrastructure/windows-installation/assisted-install-infrastructure-windows) [instalación manual](/docs/infrastructure/install-configure-infrastructure/windows-installation/manual-install-infrastructure-windows) o asistida, deberá eliminar manualmente todos los archivos creados al descomprimir el archivo del agente.

## Desinstalar el<img style={{ width: '32px', height: '32px', verticalAlign: 'middle'}} class="inline" title="macOS" alt="macOS icon" src="/images/os_icon_mac.webp">macOS agente de infraestructura</img> [#uninstall-macos]

Si utilizó el procedimiento de instalación predeterminado para el agente de infraestructura para entornos macOS, para desinstalar:

1. [Detener al agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status#macOS):

   ```
     brew services stop newrelic-infra-agent
   ```

2. Desde la terminal, ejecute el comando de desinstalación:

   ```
     brew uninstall newrelic-infra-agent
   ```

## Opcional: purgar los archivos restantes [#leftover-files]

Si utiliza herramientas de administración de paquetes estándar para la plataforma seleccionada, el proceso de desinstalación generalmente deja la configuración y otros archivos diversos. Si necesita eliminar por completo alguno de los archivos restantes después de desinstalar el agente New Relic Infrastructure, siga los procedimientos estándar para su sistema operativo.
