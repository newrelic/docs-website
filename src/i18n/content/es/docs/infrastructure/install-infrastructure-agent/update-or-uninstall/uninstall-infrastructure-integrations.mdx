[Desinstalar el agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations) no afecta directamente ninguna [integración de su infraestructura](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure): si desinstala el agente, su integración permanecerá. Del mismo modo, si [desactivas o desinstalas tu integración](#uninstall-integrations), el agente de infraestructura permanecerá.

Para desinstalar cualquiera de tus integraciones, sigue el procedimiento correspondiente al tipo de integración.

## Integracion on-host [#uninstall-on-host]

Si utilizó el paquete de integración, [consulte las instrucciones del paquete de integración](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations#uninstall-package).

  <Collapser
    id="uninstall-kubernetes"
    title="Kubernetes"
  >
    Asegúrese de utilizar el contexto adecuado en la máquina donde ejecutará Helm y `kubectl`:

    Puedes consultar los contextos disponibles con:

    ```
    kubectl config get-contexts
    ```

    Y cambie al contexto deseado usando:

    ```
    kubectl config use-context CONTEXT_NAME
    ```

    Si usó Helm para instalar la integración de Kubernetes, simplemente desinstálelo usando Helm:

    ```
    helm uninstall --namespace NAMESPACE_USED_DURING_INSTALLATION RELEASE_NAME
    ```

    Si instaló la integración de Kubernetes usando un manifiesto, use el mismo manifiesto para desinstalarlo:

    ```
    kubectl delete -f newrelic-infrastructure.yaml
    ```
  </Collapser>

## Alejándose del paquete de integración [#uninstall-package]

Si bien todavía es posible utilizar el paquete de integración, recomendamos eliminarlo por completo y trabajar con la integración de forma individual.

El último paquete de integración contiene las siguientes versiones de la integración:

* Apache 1.1.2
* Cassandra 2.0.3
* MySQL 1.1.5
* Nginx 1.0.2
* Redis 1.0.1

Si elimina el paquete de integración y desea continuar usando la integración relacionada en el host, deberá instalarlos uno por uno. Para desinstalar el paquete y reinstalar su integración:

1. Elimine el paquete de integración siguiendo estas instrucciones. Los archivos de configuración de la integración anterior no se eliminarán, por lo que no tendrás que configurarlos nuevamente.
