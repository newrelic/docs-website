El análisis a nivel de consulta en New Relic está diseñado para permitir que los administradores de bases de datos (DBA) y otros profesionales de TI obtengan información detallada y valiosa sobre cómo funcionan las consultas SQL individuales. Esta capacidad ayuda a identificar y solucionar problemas como tiempo de respuesta lento, carga alta del sistema y latencia, directamente desde la instrumentación de la base de datos, sin depender únicamente del monitoreo del rendimiento de aplicaciones (APM) (agente APM). Aborda los puntos críticos clave al ofrecer datos completos, herramientas de visualización y recomendaciones prácticas para optimizar la consulta. Además, puede profundizar en consultas específicas para identificar las causas fundamentales de las degradaciones del rendimiento.

Beneficios de emplear el análisis a nivel de consulta de New Relic:

* **Optimización del rendimiento:** identifica consultas lentas o ineficientes que se pueden optimizar para lograr un mejor rendimiento.
* **Gestión de recursos:** proporciona información valiosa sobre el uso de recursos, lo que ayuda en la asignación eficiente de recursos de CPU, memoria y I/O .
* **Diagnóstico de problemas:** facilita la identificación y resolución de problemas que causan ralentizaciones o fallos en la aplicación.

## Acceda al dashboarda nivel de consulta [#access]

Puede acceder al dashboard de nivel de consulta desde el **panel izquierdo &gt; All Capabilities &gt; Dashboards** en su cuenta New Relic .

* Integre y luego configure su base de datos en New Relic para el monitoreo a nivel de consulta. Para obtener más información, consulte:

  * [Integración PostgreSQL](/install/postgresql)
  * [Integración MySQL](/install/mysql)
  * [Integración con Microsoft SQL](/install/microsoft-sql)

**Para acceder al dashboard de nivel de consulta:**

1. Inicie sesión en su cuenta de New Relic.
2. Vaya a **All capabilities &gt; Dashboards**.
3. Seleccione su base de datos dashboard.

## Uso del dashboarda nivel de consulta [#using-query-dashboard]

El análisis a nivel de consulta implica examinar consultas SQL individuales dentro de una base de datos para evaluar su rendimiento e impacto. Proporciona información valiosa sobre los tiempos de ejecución, el consumo de recursos y el potencial cuello de botella, lo que permite al usuario optimizar las operaciones de la base de datos.

El de New Relic nivel de consulta dashboard tiene tres pestañas:

* [consulta análisis](#query-analysis)
* [Detalles de consulta](#query-details)
* [Análisis del tiempo de espera](#wait-time)

### consulta análisis [#query-analysis]

El análisis de consulta proporciona una descripción general de alto nivel de la base de datos de rendimiento, lo que permite al usuario identificar rápidamente las áreas que requieren atención. Destaca indicadores de rendimiento clave como tiempos de ejecución y evento de espera.

#### Uso del análisis de consulta [#use-query-analysis]

* Identifique la base de datos con tiempos de ejecución promedio altos o eventos de espera significativos empleando el widget.
* Analice las tendencias en los recuentos de ejecución y I/O de disco para correlacionarlas con problemas de rendimiento.
* Emplee detalles de bloqueo para resolver consultas que causan o experimentan bloqueos.

### Detalles de consulta [#query-details]

La pestaña Detalles de la consulta permite al usuario explorar en profundidad las ejecuciones de consulta individuales, proporcionando métricas detalladas y planes de ejecución para optimizar el rendimiento.

#### Uso de los detalles de la consulta [#use-query-details]

* Identifique las consultas con alto uso de recursos empleando el widget Detalles de consulta individual.
* Analizar los planes de ejecución para detectar ineficiencias e implementar optimizaciones recomendadas.

### Análisis del tiempo de espera [#wait-time]

La pestaña Análisis del tiempo de espera ayuda al usuario a comprender y mitigar los retrasos que afectan el rendimiento de la consulta.

#### Uso del análisis del tiempo de espera [#use-wait-time]

* Concentrar en abordar el evento de espera de mayor impacto empleando el widget Top Wait evento.
* Emplee la tendencia de tiempos de espera de instancias de base de datos para identificar periodos de mayores tiempos de espera.
* Acceda a los detalles de la consulta Wait para diagnosticar fuentes de retrasos y priorizar la consulta para su optimización.
