Cada cuenta de nube que vincula al monitoreo de infraestructura de New Relic (por ejemplo, [AWS](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) o [Azure](/docs/integrations/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations)) tiene un <DNT>**account status dashboard**</DNT> que muestra cómo nuestra plataforma maneja los datos de ese proveedor de servicios de nube.

## Por qué es importante [#reasons-to-use]

Las razones para monitor este dashboard incluyen:

* <DNT>
    **Understand API call usage.**
  </DNT>

  Utilizamos API en la nube para recopilar métricas y datos de inventario. El panel muestra recuentos y frecuencias para esas llamadas API. Es posible que desee monitor por dos razones: la llamada API puede costar dinero y su uso API puede verse limitado en ciertos niveles de uso. Para disminuir el uso, puede [configurar los ajustes de integración](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) para reducir la cantidad de componentes que monitor o la frecuencia con la que lo hacen.

* <DNT>
    **Troubleshoot missing data or other data issues.**
  </DNT>

  El dashboard muestra información que afecta la forma en que los datos del informe de integración, incluyendo:

  * Errores que afectan la recopilación de datos de New Relic, como errores de permisos o errores de agotamiento de cuotas.
  * Cambios en la configuración de integración, como cambios en el intervalo de sonido o la activación o desactivación de la colección de etiquetas.

## Comprender los datos dashboard [#understand-data]

Los datos específicos que se muestran en el dashboard de estado de la cuenta diferirán según el proveedor de servicios de la nube. Los gráficos comunes incluyen:

* <DNT>
    **Data updates**
  </DNT>

  : Muestra actualizaciones de datos métricos o datos de inventario (las actualizaciones se muestran como un valor `1` ).

* <DNT>
    **Account changes**
  </DNT>

  : Acciones que afectan el funcionamiento de la integración. Por ejemplo: cambiar el nombre o desvincular una cuenta en la nube, cambiar el intervalo de sonido y otras opciones de configuración.

* <DNT>
    **Data freshness**
  </DNT>

  : timestamp del último punto de datos recopilado para cada integración.

* <DNT>
    **Fetching errors**
  </DNT>

  : Estos indican problemas con la recopilación de datos. (Esto puede ser un problema por parte del proveedor de la nube).

Las opciones para comprender mejor los datos del gráfico incluyen:

* Pase el mouse sobre un gráfico

  <Icon name="fe-help-circle"/>

  icono para ver una descripción del gráfico (si está disponible).

* [Vea la consulta NRQL subyacente del gráfico](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform).

## Encuentra dashboard de estado de la cuenta [#find-dashboard]

Para encontrar el dashboard de estado de la cuenta de un proveedor de servicios en la nube:

1. Desde

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure**
   </DNT>

   y seleccione un proveedor de servicios en la nube (por ejemplo,

   <DNT>
     **AWS**
   </DNT>

   ).

2. Seleccione

   <DNT>
     **Account status dashboard**
   </DNT>

   para la cuenta de la nube.
