Nuestro [monitoreo de infraestructura](/docs/infrastructure) proporciona una integración para [Microsoft Azure Functions](https://azure.microsoft.com/en-us/services/functions/) que informa datos de su servicio Azure Functions a New Relic. Este documento explica cómo activar esta integración y describe los datos que se pueden capturar.

## Característica

Nuestra integración Azure Functions reporta datos métricos sobre su servicio de Functions, como el número de ejecuciones de funciones, bytes enviados y recibidos y recuentos de errores HTTP. También recopila datos sobre el estado y configuración del servicio.

Puede monitor y alertar sobre sus datos Azure Functions con nuestro [monitoreo de infraestructura](/docs/infrastructure), y puede [crear consultas personalizadas y paneles de gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Activar la integración [#requirements]

Para habilitar esta integración, siga los procedimientos estándar para [activar su servicio Azure para el monitoreo de infraestructura](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## Configuración y sondeo [#polling]

Puede cambiar la frecuencia de sondeo y filtrar datos usando [las opciones de configuración](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Información [de sondeo](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) predeterminada para la integración de Azure Functions:

* Intervalo de sonda: 5 minutos
* Resolución: 1 punto de datos por minuto

## Buscar y utilizar datos [#find-and-use]

Para encontrar sus datos de integración, vaya a <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, en el navegador superior, haga clic en <DNT>**Infrastructure**</DNT>, luego haga clic en <DNT>**Azure**</DNT> y busque la integración.

Puedes [consultar y explorar tus datos](/docs/using-new-relic/data/understand-data/query-new-relic-data) utilizando el [tipo de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureFunctionsAppSample` .

Para obtener más información sobre cómo buscar y utilizar datos de integración, consulte [Comprender y utilizar datos](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Datos métricos [#metrics]

Para ver las métricas reportadas por la integración Azure Functions , consulte el tipo de evento `AzureFunctionsAppSample`. Utilice los metadatos asociados con cada métrica para [filtrar y facetar](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) los datos que se informan. Para obtener información métrica detallada, consulte la documentación [métrica admitida por Azure](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-functions) .

## Otros datos del sistema [#other-attributes]

La integración Azure Functions también recopila estos [atributos](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) sobre el servicio y su configuración:

* `availabilityState`
* `defaultHostName`
* `enabled`
* `isPremiumApp`
* `javaVersion`
* `netFrameworkVersion`
* `nodeVersion`
* `operatingSystem`
* `phpVersion`
* `pythonVersion`
* `platformArchitecture`
* `state`
* `usageState`
