[New Relic monitoreo de infraestructura](/docs/infrastructure) proporciona una integración para [Microsoft Azure Service Bus](https://docs.microsoft.com/en-us/azure/service-bus-messaging/service-bus-messaging-overview) que reporta datos de su servicio Azure Service Bus a New Relic. Este documento explica cómo activar esta integración y describe los datos que se pueden capturar.

## Característica

La integración de New Relic para Azure Service Bus informa datos métricos sobre su servicio Service Bus, como la cantidad de mensajes en la cola, la cantidad de mensajes no entregados y el tamaño de bytes de la cola. También recopila datos sobre el estado y configuración del servicio.

Puede monitor y alertar sobre los datos de Azure Service Bus desde [New Relic](/docs/infrastructure) y puede [crear consultas personalizadas y paneles de gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Activar la integración [#requirements]

Para habilitar esta integración, siga los procedimientos estándar para [activar su servicio Azure en New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## Configuración y sondeo [#polling]

Puede cambiar la frecuencia de sondeo y filtrar datos usando [las opciones de configuración](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Información [de sondeo](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) predeterminada para la integración de Service Bus:

* Intervalo de sonda: 5 minutos
* Resolución: 1 punto de datos por minuto

## Buscar y utilizar datos [#find-and-use]

Para encontrar sus datos de integración, vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**</DNT> y busque la integración.

Puedes [consultar y explorar tus datos](/docs/using-new-relic/data/understand-data/query-new-relic-data) utilizando estos [tipos de eventos](/docs/data-apis/understand-data/new-relic-data-types/#event-data):

* Datos del espacio de nombres: [`AzureServiceBusNamespaceSample`](#namespace)
* Datos de cola: [`AzureServiceBusQueueSample`](#queue)
* Datos de suscripción: [`AzureServiceBusSubscriptionSample`](#subscription)
* Datos del tema: [`AzureServiceBusTopicSample`](#topic)

Para obtener más información sobre cómo buscar y utilizar datos de integración, consulte [Comprender y utilizar datos](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Datos métricos [#metrics]

Hay varios conjuntos de métricas reportadas por la integración de Azure Service Bus. Para obtener más información sobre cómo se estructuran los datos y se informan a New Relic, consulte [Comprender y utilizar los datos de integración](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric).
