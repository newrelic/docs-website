[New Relic monitoreo de infraestructura](/docs/infrastructure) proporciona una integración para [Microsoft Azure App Service](https://azure.microsoft.com/en-us/services/app-service/) que reporta datos de Azure Apps Service a la plataforma New Relic . Este documento explica cómo activar esta integración y describe los datos que se pueden capturar.

## Característica

Un Azure App Service es un servicio para alojar y ejecutar aplicaciones web, API REST y backend móvil en una plataforma completamente administrada. La integración de New Relic para Azure App Service reporta datos métricos sobre sus aplicaciones web, que incluyen:

* Número de solicitudes recibidas
* Número de respuestas y errores HTTP

También recopila datos sobre el estado y la configuración del servicio. Usando New Relic, puedes:

* Vea los datos de Azure Storage en un panel de infraestructura prediseñado.
* Ejecute consultas personalizadas y visualice los datos en [la New Relic UI](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).
* Cree condición de alerta para notificarle sobre cambios en los datos de Almacenamiento.

## Activar la integración [#activate]

Para habilitar la integración, siga los procedimientos estándar para [activar su servicio Azure en New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## Configuración y sondeo [#polling]

Puede cambiar la frecuencia de sondeo y filtrar datos usando [las opciones de configuración](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

New Relic consulta sus servicios de aplicaciones de Azure de acuerdo con un intervalo [de sondeo](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) predeterminado, que varía según la integración. Frecuencia de sondeo para la integración de Azure App Services:

* Intervalo de sonda: 5 minutos
* Resolución: 1 punto de datos por minuto

## Buscar y utilizar datos [#find-and-use]

Para [explorar sus datos de integración](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**</DNT>.

Los [tipos de eventos](/docs/data-apis/understand-data/new-relic-data-types/#event-data) son:

* `AzureAppServiceWebAppSample`: representa cada una de las aplicaciones web de Azure App Service en la cuenta.
* `AzureAppServiceHostNameSample`: Representa cada uno de los nombres de dominio vinculados a una aplicación web que se puede utilizar para acceder a ella.
* `AzureAppServicePlan:` Representa un conjunto de recursos de cálculo para que se ejecute una aplicación web.

## Datos métricos [#metrics]

Para ver las métricas reportadas por la integración de Azure App Service, consulte los tipos de eventos `AzureAppServiceWebAppSample` o `AzureAppServicePlanSample` . Utilice los metadatos asociados con cada métrica para [filtrar y facetar](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) los datos que se informan. Para obtener información métrica detallada, consulte la documentación [métrica admitida por Azure](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-excluding-functions) .
