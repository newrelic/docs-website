[New Relic monitoreo de infraestructura](/docs/infrastructure) proporciona una integración para [Microsoft máquina virtual de Azure (máquina virtual (VM))](https://docs.microsoft.com/en-us/azure/virtual-machines/) que reporta datos de su servicio Azure máquina virtual (VM) a New Relic. Este documento explica cómo activar esta integración y describe los datos que se pueden capturar.

## Característica

La integración de New Relic para máquina virtual de Azure informa datos sobre su servicio de máquina virtual (VM), como el ID de la máquina virtual (VM), el tamaño de la máquina virtual (VM), el conjunto de disponibilidad y el nombre de la región. Puede monitor y alertar sobre los datos de su máquina virtual (VM) de Azure desde [New Relic](/docs/infrastructure), y puede [crear consultas personalizadas y paneles de gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Requisitos

Los requisitos incluyen:

* [New Relic Infrastructure instalado](/docs/infrastructure/new-relic-infrastructure/installation) con [agente de infraestructura versión 1.0.775](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) o superior. ([Actualice el agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent)).
* [New Relic Azure activada](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations)

## Activar la integración [#requirements]

Para habilitar esta integración, siga los procedimientos estándar para [activar su servicio Azure en New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

<Callout variant="important">
  Debes instalar el agente de infraestructura en cada máquina virtual (VM) para ver la métrica de ese host. Conectar su suscripción de Azure permite New Relic acceder a los metadatos de la máquina virtual (VM).
</Callout>

## Configuración y sondeo [#polling]

Puede cambiar la frecuencia de sondeo y filtrar datos usando [las opciones de configuración](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Información [de sondeo](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) predeterminada para la integración virtual de la máquina:

* Intervalo de sonda: 5 minutos
* Resolución: 1 punto de datos por minuto

## Buscar y utilizar datos [#find-and-use]

Para encontrar sus datos de integración, vaya a <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**</DNT> y busque la integración.

Puedes [consultar y explorar tus datos](/docs/using-new-relic/data/understand-data/query-new-relic-data) utilizando el [tipo de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureVirtualMachineSample` . El valor `provider` es `AzureVirtualMachine`.

Para obtener más información sobre cómo buscar y utilizar datos de integración, consulte [Comprender y utilizar datos](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Otros datos del sistema [#other-attributes]

La máquina virtual de integración de Azure también recoge el siguiente [atributo](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) sobre el servicio y su configuración:

* Región
* Zona de disponibilidad
* Tipo de instancia
* ID de la instancia

## Resolución de problemas

Si usa <DNT>**Host not reporting**</DNT> condición de alerta, la importación de metadatos de Azure para su máquina virtual (VM) cambiará el registro de hosts y generará una notificación de alerta falsa. Para evitar falsos positivos:

1. Para cada instancia de Azure, [deshabilite](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#maintain-alerts) su condición de alerta

   <DNT>
     **Host not reporting**
   </DNT>

   .

2. [Actualice el agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent) para todas las instancias que utilizaron esta condición de alerta.

3. [Vuelva a habilitar](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#maintain-alerts) cada

   <DNT>
     **Host not reporting**
   </DNT>

   condición de alerta.
