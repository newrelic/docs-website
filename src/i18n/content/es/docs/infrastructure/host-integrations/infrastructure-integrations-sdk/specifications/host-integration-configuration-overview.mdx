Nuestra [integración en el host](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations) envía datos al agente de infraestructura, que a su vez envía esos datos a New Relic. La forma en que la integración interactúa con el agente está controlada por la configuración de cada integración.

Para obtener más información sobre a qué integración se aplica esto, consulte [integración en el host instalación](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations).

## Descripción general de cómo funciona la configuración [#overview]

Algunas de nuestra integración en el host son programas externos ejecutados por el [agente de infraestructura](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent). Cada integración monitorea un [servicio específico](/docs/integrations/host-integrations/host-integrations-list). Una integración tiene, como mínimo, estos archivos:

* Un [ejecutable](/docs/infrastructure/infrastructure-integrations/custom-integrations/create-integration-executable-file) que exporta varios [tipos de datos](/docs/integrations/new-relic-integrations/get-started/introduction-infrastructure-integrations#data-types) en un formato JSON esperado por el agente.
* Uno o más archivos de configuración en formato YAML (por ejemplo, la [configuración de integración de Apache)](/docs/integrations/host-integrations/host-integrations-list/kafka-monitoring-integration#config). (Recomendamos [limpiar](http://www.yamllint.com/) los archivos de configuración YAML antes de usarlos para evitar problemas de formato).

Tenga en cuenta que, además de la configuración de la integración en el host específica, también puede editar la [configuración del agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/configuration/configure-infrastructure-agent).

## Ubicación del archivo de configuración [#file-location]

Con integración estándar en el host instalación, la configuración se ubica en el directorio del agente de infraestructura. El agente determina esta ubicación de configuración mediante una [configuración en su propio archivo de configuración](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#plugin-dir).

Para algunas implementaciones, la configuración de la integración se ubicará en otro lugar. Por ejemplo:

* Servicios que se ejecutan en [Kubernetes](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes): la configuración se encuentra en el archivo de configuración de integración de Kubernetes.
* Servicios que se ejecutan en [Amazon ECS](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs): la configuración se coloca en la consola de AWS.

## Formatos de configuración [#config-formats]

Integración en el host utiliza dos formatos de configuración:

* <DNT>
    **Standard**
  </DNT>

  : A partir de diciembre de 2019, [la versión 1.8.0 del agente de infraestructura](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) comenzó a admitir un nuevo formato utilizado por algunas integraciones. Este formato utiliza un único archivo de configuración y proporciona otras mejoras. Para más detalles, consulte [Configuración estándar](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-standard-configuration-format).

* <DNT>
    **Legacy**
  </DNT>

  : Este es el formato utilizado por la mayoría de las integraciones en el host. Esta configuración utiliza dos archivos: un archivo de definición y un archivo de configuración. Para obtener más detalles, consulte [la configuraciónlegacy ](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-legacy-configuration-format/).
