Nuestra [integración](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) IBM MQ recopila y envía métricas dimensionales desde [IBM MQ](https://www.ibm.com/products/mq).

Puede ver estos [datos métricos](#metrics) en un panel prediseñado, crear políticas de alertas y crear consultas y gráficos personalizados.

<img
  title="IBM MQ quickstart dashboard"
  alt="Image of the dashboard available through the IBM MQ quickstart"
  src="/images/infrastructure_screenshot-crop_ibmmq-dashboard.webp"
/>

<figcaption>
  dashboard instalado a través del [New Relic IBM MQ monitor inicio rápido](https://one.newrelic.com/nr1-core?state=c95ed954-7277-e44d-7fd7-351a77b3ff37).
</figcaption>

Para obtener datos de la infraestructura IBM MQ, nuestra integración IBM MQ utiliza un [exportador Prometheus](https://github.com/ibm-messaging/mq-metric-samples/tree/master/cmd/mq_prometheus) de terceros, que expone Prometheus métrica en el puerto especificado en la configuración. La integración recopila estas métricas, las transforma en entidad, las filtra y luego las envía a New Relic.

<Callout variant="important">
  La integración de IBM MQ no admite la arquitectura ARM64.
</Callout>

Para instalar la integración de monitoreo de IBM MQ, ejecute los siguientes pasos:

1. [Instalar y activar la integración](#install).
2. [Configurar la integración](#config).
3. [Buscar y utilizar datos](#find-and-use).

## Compatibilidad y requisitos [#req]

### Versiones de IBM MQ [#IBMMQ-versions]

Nuestra integración es compatible con IBM MQ 9.0 o superior.

### Sistema operativo soportado [#supported-os]

* linux<img style={{ width: '32px', height: '32px'}} class="inline" title="Linux" alt="Linux" src="/images/os_icon_linux.webp"/>
* Windows<img style={{ width: '32px', height: '32px'}} class="inline" title="Windows" alt="Windows" src="/images/os_icon_windows.webp"/>

Para obtener una lista completa de versiones específicas de Linux y Windows, consulte la tabla de [sistemas operativos compatibles](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/#operating-systems).

<Callout variant="important">
  La arquitectura ARM64 no es compatible.
</Callout>

### Requisito del sistema [#system-reqs]

* [Instale el agente de infraestructura](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic), versión mínima 1.24.0.
* Distribución Linux o versión Windows [compatible con el agente de infraestructura](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure#operating-systems).
* [Biblioteca de IBM MQ Redistributable Client](https://public.dhe.ibm.com/ibmdl/export/pub/software/websphere/messaging/mqdev/redist/) en el cuadro de agente.
* Verifique o agregue la ruta de la biblioteca en la variable de entorno `LD_LIBRARY_PATH` en Linux o siga [estos pasos](https://www.ibm.com/docs/en/ibm-mq/9.1?topic=windows-testing-communication-between-client-server) en Windows.
* Asegúrese de que [el monitoreo de estadísticas de cola](https://www.ibm.com/docs/en/capmp/8.1.4?topic=monitoring-enabling-queue-statistics-queue-manager-mq) esté habilitado en MQ.
* [Cree un usuario](https://www.ibm.com/docs/en/mq-appliance/9.1?topic=commands-usercreate-create-user) en MQ con permisos de solo lectura.

## Instalar y activar [#install]

Para instalar la integración, siga las instrucciones para su entorno:

<CollapserGroup>
  <Collapser
    id="linux-install"
    title="Instalación de linux"
  >
    1. Instale [el agente de infraestructura](/docs/integrations/host-integrations/installation/install-infrastructure-host-integrations/#install) y reemplace la variable `INTEGRATION_FILE_NAME` con `nri-ibmmq`.

    2. Cambie el directorio a la carpeta de configuración de integración ejecutando:

       ```shell
       cd /etc/newrelic-infra/integrations.d
       ```

    3. Copie el archivo de configuración de muestra ejecutando:

       ```shell
       sudo cp ibmmq-config.yml.sample ibmmq-config.yml
       ```

    4. Edite el archivo de configuración `ibmmq-config.yml` con su editor favorito. Consulte algunos [ejemplos de archivos de configuración](#examples).
  </Collapser>

  <Collapser
    id="windows-install"
    title="Instalación de Windows"
  >
    1. Descargue la imagen del instalador MSI `nri-ibmmq` de:

       [https://download.newrelic.com/infrastructure_agent/windows/integrations/nri-ibmmq/nri-ibmmq-amd64.msi](https://download.newrelic.com/infrastructure_agent/windows/integrations/nri-ibmmq/nri-ibmmq-amd64.msi)

    2. Para instalar desde el símbolo del sistema de Windows, ejecute:

       ```
       msiexec.exe /qn /i <var>PATH\TO\</var>nri-ibmmq-amd64.msi
       ```

    3. En el directorio de integración, `C:\Program Files\New Relic\newrelic-infra\integrations.d\`, cree una copia del archivo de configuración de muestra ejecutando:

       ```
       copy ibmmq-config.yml.sample ibmmq-config.yml
       ```

    4. Edite el archivo `ibmmq-config.yml` como se describe en los [ajustes de configuración](#config).
  </Collapser>
</CollapserGroup>

## Configurar la integración [#config]

Para configurar la integración, edite la configuración en el archivo de configuración YAML de la integración, `ibmmq-config.yml`. La configuración de formato YAML de una integración es donde puede colocar las credenciales de inicio de sesión requeridas y configurar cómo se recopilan los datos. Las opciones que cambie dependen de su configuración y preferencia. El archivo de configuración tiene configuraciones comunes aplicables a todas las integraciones.

Para leer todo sobre estas configuraciones comunes, consulte nuestro documento [de formato de configuración](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-newer-configuration-format/#configuration-basics) .

## Archivos de muestra ibmmq-config.yml [#examples]

<CollapserGroup>
  <Collapser
    id="basic-config"
    title="Configuración básica"
  >
    Esta es la configuración básica utilizada para recopilar métricas:

    ```yml
    integrations:
      - name: nri-ibmmq
        config:
          hostname: localhost
          port: 1414
          queue_manager: QM1
          channel: SYSTEM.DEF.SVRCONN
          username: your_mq_user
          password: your_mq_password
          monitored_queues: '!SYSTEM.*,*'
          monitored_channels: '*'
          monitored_topics: '*'
          monitored_subscriptions: '*'
          exporter_port: 9157
    ```
  </Collapser>

  <Collapser
    id="monitor-two-instances"
    title="Monitor diferentes instancias"
  >
    Esta es la misma configuración básica utilizada para recopilar métricas que muestra cómo ejecutar la integración dos veces conectándose a dos hosts. Tenga en cuenta que cada instancia de la integración generará un exportador adicional escuchando en un puerto diferente.

    ```yml
    integrations:
      - name: nri-ibmmq
        config:
          hostname: host1
          port: 1414
          queue_manager: QM1
          channel: SYSTEM.DEF.SVRCONN
          username: your_mq_user
          password: your_mq_password
          monitored_queues: '!SYSTEM.*,*'
          monitored_channels: '*'
          monitored_topics: '*'
          monitored_subscriptions: '*'
      - name: nri-ibmmq
        config:
          hostname: host2
          port: 1414
          queue_manager: QM1
          channel: SYSTEM.DEF.SVRCONN
          username: your_mq_user
          password: your_mq_password
          monitored_queues: '!SYSTEM.*,*'
          monitored_channels: '*'
          monitored_topics: '*'
          monitored_subscriptions: '*'
    ```
  </Collapser>

  <Collapser
    id="PasswordLess-config"
    title="Configuración sin contraseña"
  >
    Si no se especifica la contraseña, la autenticación de usuario/contraseña está deshabilitada y el nombre de usuario no debe especificarse en la configuración.

    El exportador intentará conectarse enviando como clientID al usuario que ejecuta la integración. Dependiendo del modo de instalación del agente podrá ser "root" o "nri-agente".

    Dicho usuario debe poder acceder al administrador de colas sin proporcionar una contraseña.

    ```yml
    integrations:
      - name: nri-ibmmq
        config:
          hostname: localhost
          port: 1414
          queue_manager: QM1
          channel: SYSTEM.DEF.SVRCONN
          monitored_queues: '!SYSTEM.*,*'
          monitored_channels: '*'
          monitored_topics: '*'
          monitored_subscriptions: '*'
          exporter_port: 9157
    ```
  </Collapser>

  <Collapser
    id="filters-config"
    title="Configuración con filtros"
  >
    Esta es la configuración básica que utiliza filtros para restringir las colas de monitorización.<br/>En este ejemplo monitor todas las colas denominadas `DEV.QUEUE` e ignoramos específicamente la cola `DEV.QUEUE.1`. Se pueden agregar otros patrones separados por comas:

    ```yml
    integrations:
      - name: nri-ibmmq
        config:
          hostname: localhost
          port: 1414
          queue_manager: QM1
          channel: SYSTEM.DEF.SVRCONN
          username: your_mq_user
          password: your_mq_password
          monitored_queues: 'DEV.QUEUE*,!DEV.QUEUE.1'
          monitored_channels: '*'
          monitored_topics: '*'
          monitored_subscriptions: '*'
          exporter_port: 9157
    ```
  </Collapser>

  <Collapser
    id="mtls-config"
    title="Configuración con mTLS y autenticación básica"
  >
    Esta es una configuración avanzada que muestra cómo configurar mTLS. El archivo `ccdt` debe indicar qué certificado presentar al servidor a través del campo `certificateLabel` de la sección `transmissionSecurity` . El certificado debe estar en el almacén de claves con dicha etiqueta y el servidor debe confiar en él. Además, el almacén de claves debe contener un certificado capaz de verificar el presentado por el servidor.

    ```yml
    integrations:
      - name: nri-ibmmq
        config:
          queue_manager: QM1
          channel: SYSTEM.DEF.SVRCONN
          username: your_mq_user
          password: your_mq_password
          monitored_queues: 'DEV.QUEUE*,!DEV.QUEUE.1'
          monitored_channels: '*'
          monitored_topics: '*'
          monitored_subscriptions: '*'
          exporter_port: 9157

          # Configuring mTLS
          mqsslkeyr: "/key"
          ccdt_url: "file:///ccdt.json"
          home: /tmp
    ```
  </Collapser>
</CollapserGroup>
