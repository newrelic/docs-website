Este documento explica cómo nuestra integración en el host recopila y reporta datos.

## Proceso de recopilación de datos y presentación de informes. [#data-collection]

Así es como la integración en el host envía datos a New Relic:

1. Al inicio, el agente de infraestructura escanea el directorio que contiene los archivos de definición de la integración.
2. El agente de infraestructura registra cada ejecutable de integración definido en el archivo de definición.
3. El agente escanea un directorio dedicado en busca de archivos de configuración de integración.
4. Si esos archivos de configuración especifican una integración que se ha registrado con el agente de infraestructura, el agente configura y programa la integración.
5. En el intervalo programado (el valor predeterminado es 15 segundos), el agente recopila los datos de la integración y los prepara para la transmisión. Cada 60 segundos, envía esos datos a New Relic, junto con cualquier otro dato de infraestructura.
6. Después de una pasada de recopilación exitosa, el ejecutable de integración sale.

## Integración arquitectura [#example-diagram]

A continuación se muestra un ejemplo de cómo funciona la integración de SQL Server:

* El host 1 con el agente de infraestructura instalado necesita una regla de salida para el puerto `1433`.
* El host 2 con SQL Server necesita una regla de entrada para el puerto `1433`.

En este ejemplo, el host 1 extrae datos del host 2.

## Estructura de archivos y especificaciones. [#file-structure]

Comprender la estructura de archivos de estas integración en el host puede ayudarlo a personalizar su integración, comprender y utilizar sus datos y solucionar problemas.

Estas integraciones se adhieren a un conjunto de especificaciones de código abierto. Para obtener una explicación de estas especificaciones de archivos, consulte [Especificaciones de archivos](/docs/integrations/integrations-sdk/file-specifications).
