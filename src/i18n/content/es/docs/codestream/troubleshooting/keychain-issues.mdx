Si tiene que iniciar sesión en New Relic CodeStream cada vez que reinicia su IDE de JetBrains o abre un nuevo proyecto, es probable que el IDE no guarde ni recupere el token de autenticación de su contraseña local segura.

[Este artículo de JetBrains](https://www.jetbrains.com/help/idea/reference-ide-settings-password-safe.html) proporciona detalles sobre cómo puede verificar que esté configurado correctamente.

Por ejemplo, si estás en una Mac, la opción <DNT>**in native Keychain**</DNT> es la que quieres usar, como se muestra en la captura de pantalla siguiente.

<img
  alt="Passwords"
  src="/images/codestream_screenshot-crop_jetbrains-password-keychain.webp"
/>

Si el llavero nativo ya está seleccionado, también es posible que la primera vez que usó el símbolo para permitir el acceso al llavero IDE lo haya negado. Si ese es el caso, puedes ir a la aplicación <DNT>**Keychain Access**</DNT>k y corregirlo.
