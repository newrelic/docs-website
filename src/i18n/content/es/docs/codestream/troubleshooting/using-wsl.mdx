title: ¿Cómo puedo emplear WSL y CodeStream?
metaDescription: You can run CodeStream in Windows using WSL

Para emplear CodeStream y WSL (Subsistema de Windows para Linux), siga estos pasos:

1. Cerciorar de que Git esté instalado en la máquina Windows, así como en su distribución WSL. Si bien las versiones instaladas entre ambos pueden diferir, es mejor cerciorar de que la versión de Windows coincida con la que esté instalada en su distribución WSL.

2. Clona tu repositorio git en tu distribución WSL (por ejemplo, `~/code/YOUR_PROJECT_FOLDER`).

3. En Windows, permita que git se comunique a través de esta ruta UNC marcando el directorio como `safe`. Haga esto en su terminal de Windows con este comando, reemplazando `YOUR_DISTRO`, `YOUR_USERNAME` y `YOUR_PROJECT_FOLDER` para que coincidan con su configuración:

   ```shell
   git config --global --add safe.directory '//wsl.localhost/YOUR_DISTRO/home/YOUR_USERNAME/code/YOUR_PROJECT_FOLDER' 
   ```

4. En su IDE, abra el proyecto usando la ruta UNC de WSL. Por ejemplo, si tuviera Ubuntu, su nombre de usuario fuera `codestream` y la carpeta del proyecto fuera `Amazing_App`, su ruta UNC sería `//wsl.localhost/Ubuntu/home/codestream/code/Amazing_App`.

<Callout variant="tip">
  Git en Windows solo se usa para realizar solicitudes a su repositorio clonado localmente en WSL y no necesita comunicar con el remoto (por ejemplo, GitHub, GitLab, etc.).
</Callout>

Si tiene problemas luego de completar estos pasos, [recopile su archivo de log](/docs/codestream/troubleshooting/client-logs) y comunicar con el soporte al cliente New Relic para una investigación más exhaustiva.
