Esta página proporciona una referencia completa para la eliminación http disponible en el catálogo de acciones de automatización del flujo de trabajo. Estas acciones le permiten realizar operaciones de eliminación de solicitudes http.

## Requisitos previos

Antes de emplear acciones HTTP en la automatización de flujos de trabajo, cerciorar de tener:

* Objetivo extremos de URL de API.
* Cualquier credencial de autenticación requerida (clave de API, token, etc.).
* Comprensión de los formatos de solicitud/respuesta de la API.

<Callout variant="tip">
  Las acciones HTTP admiten la sintaxis secreta para cualquier valor de encabezado, lo que le permite pasar de forma segura datos confidenciales como la clave de API. Consulte [Gestión de secretos](/docs/workflow-automation/limitations-and-faq/workflow-best-practices/#secure-credentials) para obtener más información.
</Callout>

## Eliminar datos sitio web [#http-delete]

Realiza una solicitud HTTP DELETE para eliminar datos en un extremo de API.

<Callout variant="important">
  Si necesita pasar datos confidenciales a una entrada, por ejemplo un encabezado `Api-Key`, puede usar valores almacenados a través de la mutación [secretsManagementCreateSecret](https://one.newrelic.com/nerdgraph-graphiql) de NerdGraph.

  Ejemplo:

  ```json
    {
      "inputs": [
        {
          "key": "headers",
          "value": "{\"Api-Key\": \"${{ :secrets:NR_API_KEY }}\"}"
        }
      ]
    }
  ```
</Callout>
