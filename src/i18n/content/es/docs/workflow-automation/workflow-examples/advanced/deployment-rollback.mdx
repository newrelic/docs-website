Supervise el estado de la entidad de New Relic y active las notificaciones de reversión a través de AWS SQS o webhooks HTTP cuando se detecten problemas.

**Requisitos:**

* Monitoreo de entidades de New Relic
* (Opcional) Cola de AWS SQS o punto final de webhook HTTP

**Lo que hace este flujo de trabajo:**

* Monitorear la gravedad de la alerta de la entidad durante la duración especificada (predeterminada 30 minutos)
* Verifique el estado de la entidad cada minuto
* Registrar el estado no saludable si la entidad se vuelve CRÍTICA o ADVERTENCIA
* Envía notificaciones de reversión de AWS SQS (si están configuradas) con detalles de la entidad
* Enviar notificaciones de retroceso de webhook HTTP (si está configurado)
* Registrar el éxito si la entidad permanece en buen estado durante toda la duración
* Deje de monitorear si no se encuentra la entidad

**Acciones clave**: `newrelic.nerdgraph.execute`, `newrelic.ingest.sendLogs`, `aws.execute.api`, `(sqs.send_message)`, `http.post`
