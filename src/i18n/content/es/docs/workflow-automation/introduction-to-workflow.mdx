Workflow Automation automatiza la respuesta a incidentes y la gestión de infraestructura sin código. Cree flujos de trabajo que respondan a alertas, se integren con herramientas como AWS y Slack, y ejecuten tareas operativas automáticamente.

## Requisitos

Para usar Workflows, necesita:

* Una cuenta de New Relic
* Plan de precios de Cómputo avanzado (aCCU)

<Callout variant="important">
  Workflow Automation y la gestión de secretos se incluyen con Advanced Compute (aCCU). Si no está seguro de si tiene aCCU, verifique la [configuración de su cuenta](https://one.newrelic.com/admin-portal/plan-info/home) o comuníquese con su representante de cuenta.
</Callout>

## Beneficios clave

* **Respuesta a incidentes más rápida**: cambie automáticamente el tamaño de las instancias EC2, revierta el despliegue o resetear los servicios cuando se activen alertas, sin necesidad de intervención manual.
* **Conecta tus herramientas**: Integra New Relic con AWS, Slack, Jira, ServiceNow y cualquier API HTTP para enviar datos de alerta enriquecidos donde trabaja tu equipo.
* **Agregar puntos de aprobación**: Incluir supervisión humana en el workflow crítico; aplicar confirmación del equipo antes de ampliar la infraestructura de producción o ejecutar reversiones.
* **Gestiona la complejidad**: Emplea lógica condicional, bucles y transformaciones de datos para automatizar tus procesos operativos únicos.

## Desencadenar flujos de trabajo

Puede iniciar flujos de trabajo de tres maneras:

* **Desde alertas**: Iniciar flujos de trabajo automáticamente cuando se infrinjan las condiciones de alerta
* **Bajo demanda**: active manualmente el flujo de trabajo a través API cuando sea necesario
* **Según lo programado**: ejecuta flujo de trabajo automáticamente en momentos específicos mediante expresiones cron

Para flujos de trabajo activados por alertas, consulte [Enviar notificaciones desde flujos de trabajo](/docs/workflow-automation/setup-and-configure/create-destinations#trigger-from-alerts).
