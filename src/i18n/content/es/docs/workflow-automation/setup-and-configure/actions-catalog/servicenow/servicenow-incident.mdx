Esta página proporciona una referencia de las acciones de incidente de ServiceNow disponibles en el catálogo de acciones de automatización de flujo de trabajo. Estas acciones le permiten crear y gestionar incidentes en ServiceNow.

## Requisitos previos

Antes de usar las acciones de ServiceNow en la automatización del flujo de trabajo, asegúrese de tener:

* Una instancia de ServiceNow con los permisos apropiados.
* Una clave de API REST de ServiceNow para la autenticación (pasada como el encabezado `x-sn-apikey`).
* Los permisos necesarios para crear y gestionar incidentes en su instancia de ServiceNow.

En la entrada de la acción, se debe proporcionar `apiKey`. Se utiliza para autenticar al usuario —la clave se pasa como parte del encabezado de la solicitud `x-sn-apikey`.

## Configure la autenticación con clave de API

Para obtener más información, consulte [Crear una política de acceso a la API REST con perfiles de autenticación de clave de API](https://www.servicenow.com/docs/r/platform-security/authentication/create-api-access-policy.html) en la documentación de ServiceNow.

Para generar y configurar una clave de API REST en ServiceNow:

<Steps>
  <Step>
    ### Crear un perfil de autenticación entrante

    1. Navegue hasta <DNT>**System Web Services &gt; API Access Policies &gt; Inbound Authentication Profile**</DNT>.
    2. Cree un nuevo **perfil de autenticación de clave de API**.
    3. Define el **Auth Parameter** como un encabezado HTTP (recomendado) usando el nombre de parámetro `x-sn-apikey`.
  </Step>

  <Step>
    ### Generar una clave de API REST

    1. Navegue hasta <DNT>**System Web Services &gt; API Access Policies &gt; REST API Key**</DNT>.
    2. Crea una nueva clave asociada a una cuenta de servicio dedicada que tenga los roles y permisos adecuados.
    3. Guarde y copie el valor del token generado — es una clave secreta.
  </Step>

  <Step>
    ### Configurar una política de acceso a la API

    1. Navegue hasta <DNT>**System Web Services &gt; API Access Policies &gt; REST API Access Policies**</DNT>.
    2. Cree una nueva política, seleccione la API REST objetivo (por ejemplo, **Table API** o una **Scripted API REST**).
    3. Agregue el **Inbound Authentication Profile** a la lista relacionada de esta política.
  </Step>
</Steps>
