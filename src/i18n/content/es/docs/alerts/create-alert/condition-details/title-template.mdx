Se utiliza una plantilla de título cuando los eventos de alerta son abiertos por la condición. Sobrescribe el título predeterminado.

## ¿Por qué emplear una plantilla de título? [#why-titles]

Una condición de alerta es la base para la creación de eventos de alerta. Ya sea que use nuestro modo guiado o cree una condición a partir de un gráfico o una política, usar una plantilla de título única es el paso final para crear una condición de alerta.

Usar la plantilla de título es opcional, pero lo recomendamos. Una condición de alerta define un conjunto de umbrales que desea monitorear. Si se supera cualquiera de esos umbrales, se crea un evento de alerta. Las plantillas de títulos significativas lo ayudan a identificar problemas y resolver interrupciones más rápido.

## Antes de que empieces [#begin]

Aquí hay algunos aspectos clave que debe tener en cuenta antes de emplear nuestra plantilla de título:

* **Compatibilidad**: la característica de plantilla de título está disponible solo para condiciones NRQL .
* **Límite de caracteres**: la longitud máxima de una plantilla de título es de 256 caracteres.
* **Formato**: la plantilla de título debe seguir un formato [de manillar](https://handlebarsjs.com/guide/) válido.
* **Eventos de alerta de señal perdida**: Para los eventos de alerta activados por una señal perdida, siempre se usará el título predeterminado. Para obtener más información sobre la pérdida de señal, consulta [Pérdida de señal y relleno de vacíos](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/).

## Comprender la plantilla de título [#example]

Supongamos que administra un sitio de comercio electrónico y ha creado una condición de alerta para activar un evento de alerta cuando los errores de red son altos. Puede usar una plantilla de título para que los eventos de alerta de esta condición sean fácilmente identificables.

Ha nombrado esta condición de alerta `High Network Errors`. Pero desea que el título del evento de alerta refleje más información sobre la entidad y la prioridad.

**Ejemplo**:

```handlebars
{{conditionName}} Affecting {{entity.name}} - {{entity.type}} - {{priority}}
```

Esta plantilla generaría el siguiente título de evento de alerta:

```
High Network Errors Affecting Entity123 - Kubernetes Pod - Critical
```

## Emplee la plantilla de título [#use-titles]

Puede interactuar con la plantilla de título de dos maneras: a través de la UI o usando GraphQL. Siga estos pasos para aprender cómo hacerlo.

<CollapserGroup>
  <Collapser id="template-ui" title="Crear una plantilla de título en la UI">
    1. Vaya al generador de condiciones NRQL. Para obtener instrucciones, consulte [condición de alerta](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/alert-conditions/).
    2. Cree su condición NRQL.
    3. En el paso final, **Add details**, ingrese su plantilla de título en el campo **Title template** .
    4. Guarde la condición.
  </Collapser>

  <Collapser id="template-ui" title="Crea una plantilla de título en GraphQL">
    En GraphQL, el campo de la plantilla de título se llama `titleTemplate`. A continuación se muestra un ejemplo de mutación para actualizar una condición estática de NRQL y agregar una plantilla de título:

    ```graphql
    mutation {
      alertsNrqlConditionStaticUpdate(
        accountId: 123456
        id: 456789
        condition: {titleTemplate: "{{conditionName}} has triggered an incident"}
      ) {
        titleTemplate
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## Emplee atributo para definir mejor su condición [#attributes]

Puede incluir un subconjunto de atributos de eventos de alerta en sus plantillas de título para agregar más contexto.

Así es cómo:

* **Emplee el formato `{{AN_ATTRIBUTE}}`**. Por ejemplo:

  ```handlebars
  {{conditionName}} has triggered an alert event
  ```

* **Atributos de etiqueta**: Para etiqueta, use el formato `{{tags.TAG_NAME}}`.

* **Etiqueta de cláusula de faceta**: si una condición NRQL usa una cláusula `FACET`, puede usar el formato `tags` para usar esos valores en su plantilla de título. Por ejemplo, si la consulta NRQL incluyera `FACET hostName, cluster`, podría usar esto:

  ```handlebars
  Host {{tags.hostName}}, cluster {{tags.cluster}} alert event
  ```

Los nombres de las etiquetas no pueden contener espacios, pero los valores expandidos sí. Para obtener una lista completa de los atributos disponibles, consulte [Atributos de eventos de alertas](/docs/alerts/create-alert/condition-details/alert-event-attributes).
