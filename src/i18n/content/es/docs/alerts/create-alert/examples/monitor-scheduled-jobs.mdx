Puede utilizar la configuración [de umbral crítico](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/advanced-techniques/set-thresholds-alert-condition) y [pérdida de señal](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions#signal-loss) para monitor los trabajos programados y recibir notificaciones cuando no sucedan.

Por ejemplo, si su sistema tiene un mantenimiento programado cada 24 horas, le gustaría saber cuándo no se realiza ese mantenimiento.

Esta técnica solo funciona para trabajos programados regularmente que ocurren con más frecuencia que el umbral máximo de pérdida de señal de 48 horas.

## Lo que necesitas saber

Para aprovechar al máximo este procedimiento, necesitará saber cómo [crear una condición](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions).

## Cómo funciona

Por lo general, las condiciones utilizan umbrales críticos para activar eventos de alerta. Sin embargo, también puede activar eventos de alerta con una pérdida de señal. Para monitorear un trabajo programado y recibir una notificación si no se ejecuta, configure el temporizador de pérdida de señal para que coincida con la programación del trabajo.

## Monitor un trabajo programado

1. [Cree una condición](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) relacionada con el trabajo que desea monitor.
2. Haga clic en <DNT>**+ Add lost signal threshold**</DNT> y luego establezca el tiempo de duración de la señal de vencimiento más largo que el ciclo programado del trabajo de monitoreo.
3. Marque <DNT>**Add lost signal threshold**</DNT>.
4. Asegúrese de nombrar su condición antes de guardarla.
