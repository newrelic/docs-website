Cuando la expresión de activación de una condición de alerta compuesta se evalúa como verdadera, New Relic abre un evento de alerta compuesta. El evento incluye un enlace a la URL de su runbook. Para ver los incumplimientos de umbrales específicos para cada condición de componente, haga clic en los eventos de alerta individuales.

## Eventos de alerta y correlación de problemas [#alert-events-correlation]

Las condiciones de alerta compuesta crean múltiples eventos de alerta relacionados:

* <DNT>**Multiple alert events**</DNT>: una condición de alerta compuesta crea su propio evento de alerta compuesta. De forma predeterminada, cada condición de componente también crea sus propios eventos de alerta cuando incumple sus umbrales. Para reducir el ruido, puede silenciar estos eventos desde la configuración de la condición de alerta. Si silencia los eventos de alerta de una condición de componente, las etiquetas de esa condición no aparecerán en el evento de alerta compuesta.
* <DNT>**Correlating to the same issue**</DNT>: para garantizar que todos los eventos de alerta relacionados (tanto compuestos como de componentes) se agrupen en el mismo problema, configure las condiciones de sus componentes para que utilicen la misma política que su condición de alerta compuesta. Esto permite que la correlación de problemas de New Relic los agrupe.
* <DNT>**Viewing related events**</DNT>: Desde un evento de alerta compuesta, puede navegar a los eventos de alerta de los componentes individuales para ver los incumplimientos de umbral y las métricas de cada condición.
