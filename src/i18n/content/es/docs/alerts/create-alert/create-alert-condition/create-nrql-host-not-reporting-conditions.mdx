El modo guiado de condiciones NRQL simplifica la creación de alertas de &quot;host not reporting&quot; (HNR). En lugar de construir manualmente una consulta NRQL, este enfoque guiado le ayuda a definir rápidamente las condiciones para detectar cuándo los agentes de infraestructura dejan de enviar datos. Este método ofrece mayor flexibilidad para seleccionar grupos de hosts específicos, personalizar los umbrales de alerta y refinar los criterios de eventos de alerta en comparación con las alertas HNR tradicionales.

## Característica [#features]

Se genera un evento HNR cuando el agente de infraestructura no puede transmitir datos a nuestro recolector dentro de un periodo de tiempo específico.

## Crear una condición de alerta NRQL HNR [#create-nrql-hnr-alert-condition]

1. Navegación a una página de política de alertas.

2. Haga clic en **New alert condition**.

3. Seleccione **Use guided mode**.

4. Seleccione **Hosts** en la categoría **Tell us where to look** . (Nota: Si no ve la opción &quot;hosts&quot;, es posible que necesite seguir [instrucciones para instrumentar su infraestructura](/docs/infrastructure/host-integrations/installation/new-relic-guided-install-overview/)).

   <img title="creating a condition step 2" alt="screenshot of alert condition step 2" src="/images/alerts_screenshot-crop_tell-us-where-to-look.webp" />

5. Si **Hosts** no se selecciona automáticamente para usted en **Select an entity type**, elija **Hosts** nuevamente.

6. Haga clic en **Next**.

7. En **Select signal catgory**, seleccione **Host**.

8. En **Select a metric to monitor**, seleccione **Host not reporting**.

   <img title="host not reporting selection" alt="screenshot of selecting host not reporting option" src="/images/alerts_screenshot-crop_host-not-reporting-selection.webp" />

9. Ahora, puedes usar el filtro de entidad para limitar el alcance de los hosts que deseas monitor.

10. Haga clic en **Next**.

    <DNT>
      /* &lt;img title=&quot;set condition thresholds&quot; alt=&quot;screenshot of setting remaining thresholds&quot; src=&quot;/images/alerts_screenshot-crop_set-condition-thresholds.webp&quot;/&gt; */
    </DNT>

11. En **Consider the signal lost after**, tiene la opción de ajustar la ventana de tiempo de 30 segundos a 48 horas. También tienes la opción de ajustar la [configuración de pérdida de señal](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/#signal-loss), lo que incluye seleccionar la opción de omitir la apertura de un evento de alerta cuando se espera que la señal termine (por ejemplo, si se espera que tu host se apague).

12. Siga los pasos restantes para finalizar la configuración y almacenar su condición de alerta.

## Opción: No abrir evento de alerta de &quot;señal perdida&quot; en terminación esperada [#do-not-open-lost-signal]

Cuando seleccionas la opción **Do not open &quot;lost signal&quot; alert event on expected termination**, le indicas a New Relic que omita abrir un evento de alerta cuando se espera que la señal finalice. Esto es útil cuando sabe que un host se va a apagar.

Para habilitar esta opción, también debe tener marcado **Open new &quot;lost signal&quot; alert event**. De esta manera, New Relic sabe que debe abrir un nuevo evento de alerta cuando se pierde la señal, pero no cuando se espera que la señal se pierda.

<Callout variant="important">
  Para evitar que se abra un evento de alerta de pérdida de señal cuando &quot;No abrir evento de alerta de &quot;señal perdida&quot; en terminación esperada&quot;, se debe agregar la etiqueta `termination: expected` a la entidad del host. Esta etiqueta nos indica que se esperaba que la señal terminara. Vea cómo agregar la etiqueta [directamente a la entidad de host](/docs/new-relic-solutions/new-relic-one/core-concepts/use-tags-help-organize-find-your-data/#add-tags).
