## La consulta de alerta NRDB coincidió con puntos de datos por minuto [#query-limit]

El límite de condición de alerta `Matched data points per minute` se aplica a la tasa total de [puntos de datos](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/understand-technical-concepts/streaming-alerts-key-terms-concepts) coincidentes para la consulta de alerta en una [cuenta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) New Relic.

Si se excede este límite, no podrá crear ni actualizar condiciones para la cuenta afectada hasta que la tasa descienda por debajo del límite. Las condiciones de alerta existentes **no se ven** afectadas.

Puede ver los puntos de datos coincidentes y cualquier incidente de límite en la [UI de límites](/docs/data-apis/manage-data/view-system-limits).

Para comprender qué condiciones conducen al mayor rendimiento, puede realizar una consulta como:

```sql
FROM NrAiSignal
SELECT sum(aggregatedDataPointsCount) AS 'alert matched data points'
FACET conditionId
```

Algunos consejos para optimizar sus puntos de datos coincidentes:

* Si está utilizando [ventanas deslizantes](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions/#sliding-window-aggregation), tenga en cuenta que esto puede aumentar significativamente la cantidad de puntos de datos. Para reducir la cantidad de puntos de datos, puede utilizar una duración de agregación más larga.
* Utilice cláusulas `WHERE` para reducir la cantidad de datos sobre los que se está alertando. Usar `WHERE` en lugar de `FACET` puede generar alertas más eficientes en algunos casos.
* Combina alerta similar. Si tiene varias condiciones de alerta que son similares, considere agruparlas con filtros combinados.

Para solicitar un aumento de límite, hable con su representante de cuenta de New Relic.

Tenga en cuenta que el uso [de ventanas deslizantes](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/create-smoother-charts-sliding-windows) puede aumentar significativamente la cantidad de puntos de datos. Considere utilizar una duración más larga de agregación de ventana deslizante para reducir la cantidad de puntos de datos producidos.

## Alerta consulta operaciones de escaneo por minuto [#query-scan-limit]

El límite de condición de alerta `Alert query scan operations per minute` se aplica a la tasa total de operaciones de escaneo de consulta en eventos ingeridos. Una operación de escaneo de consulta es el trabajo que realiza el New Relic pipeline para hacer coincidir el evento ingerido con la alerta de consulta registrada en una New Relic [cuenta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure).

Si se excede este límite, no podrá crear ni actualizar condiciones para la cuenta afectada hasta que la tasa descienda por debajo del límite. Las condiciones de alerta existentes **no se ven** afectadas.

Puede ver sus operaciones de escaneo de consulta y cualquier incidente de límite en la [UIde límites](/docs/data-apis/manage-data/view-system-limits).

Al hacer coincidir evento con alerta consulta, se deben examinar todos los eventos del [tipo de datos](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/#what-you-can-query) al que hace referencia la consulta. A continuación se muestran algunas formas comunes de tener menos eventos en un tipo de datos determinado (lo que disminuirá las operaciones de escaneo de alerta consulta):

* Al alertar sobre datos de log, emplee [particiones de log ](/docs/tutorial-manage-large-log-volume/organize-large-logs/)para limitar qué logs se escanean para alerta consulta.

* Al alertar sobre evento personalizado, divida los tipos de evento personalizado más grandes.

* Emplee evento personalizado en lugar de alertar sobre evento de transacción.

* [Crear métrica](/docs/data-apis/convert-to-metrics/analyze-monitor-data-trends-metrics/) para agregar datos.

* Emplee [el intervalo de tiempo de consulta métrica](/docs/data-apis/understand-data/metric-data/query-apm-metric-timeslice-data-nrql/) cuando sea posible en lugar de alertar sobre el evento de transacción.

Además de los consejos anteriores, limpiar cualquier alerta consulta no empleada o innecesaria (condición de alerta) disminuirá la cantidad de operaciones de escaneo de consulta.

Para solicitar un aumento de límite, hable con su representante de cuenta de New Relic.
