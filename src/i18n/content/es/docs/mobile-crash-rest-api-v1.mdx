La API de fallos móviles es una forma RESTful de recuperar información sobre [los fallos de su aplicación móvil](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes) que se han informado a New Relic. Reúne los datos de atributos y subprocesos disponibles para un conjunto determinado de tipos y ocurrencias de fallas.

Puede utilizar la API para:

1. GET tus tipos de fallos más recientes
2. GET una lista de ID de ocurrencias de un solo tipo de falla
3. GET Thread, biblioteca y atributo para un ID de ocurrencia particular

## Contenido [#in-page-toc]

## Requisitos previos

Para utilizar la API Crash en estos ejemplos, necesita:

* Su clave de API de Crash de New Relic. Pasa esta clave como el valor del encabezado `X-API-KEY`.
* Su [ID de cuenta](/docs/accounts-partnerships/accounts/account-setup/account-id)de New Relic
* Su monitoreo de móviles [ID de la aplicación](/docs/apis/rest-api-v2/requirements/finding-product-id#mobile)

Para encontrar el ID de tu cuenta y el ID de la aplicación móvil, abre tu aplicación móvil en [one.newrelic.com](https://one.newrelic.com), haga clic en el ícono <DNT>**(i)**</DNT> y lea los valores de la pestaña <DNT>**Metadata**</DNT>.

<Callout variant="tip">
  <DNT>**Note**</DNT>: `X-API-KEY`s tienen un límite de 600 requests por minuto.
</Callout>
