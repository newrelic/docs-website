Agent Control es un supervisor de agentes liviano que gestiona otros agentes New Relic y distribuciones New Relic de OpenTelemetry . Se comunica con control de flota para configurar, actualizar y monitorear de forma remota el estado de la instrumentación a nivel de infraestructura en Kubernetes.

## Requisitos previos

Antes de comenzar a implementar Agent Control en su clúster de Kubernetes, cerciorar de cumplir con los siguientes requisitos previos:

* **Helm 3:** La versión 3 de Helm debe estar instalada en su máquina. Para obtener instrucciones de instalación, consulte [Instalación de Helm](https://helm.sh/docs/intro/install).
* **Clave de licencia New Relic :** Necesitará una clave de licencia para reportar telemetría a su cuenta New Relic .
* **Nombre del clúster de Kubernetes:** tenga listo el nombre de su clúster de Kubernetes. Harás referencia a él durante el proceso de instalación.
* **Administrador de dominio de autenticación:** para configurar el Agent Control y conectarlo a control de flota, debe tener el rol de Administrador de dominio de autenticación. Para obtener más información, consulte [Conceptos de gestión de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings).
* **Instrumentación existente:** Si su clúster de Kubernetes ya está instrumentado con New Relic, debe desinstalar la instrumentación existente antes de instalar Agent Control. Luego de instalar Agent Control, puede usar control de flota para instalar y gestionar agente de infraestructura e instrumentación en el clúster. Para obtener orientación, consulte [Gestionar la instrumentación existente con Agent Control](/docs/new-relic-control/agent-control/reinstrumentation).

<Callout variant="caution" title="advertencia">
  No se admiten múltiples instalaciones de agente Control en el mismo clúster.
</Callout>

## Compatibilidad

El Agent Control es compatible con:

* Últimas tres versiones menores de Kubernetes
* Minikube, Kind, Amazon EKS, Azure AKS y Google GKE

Para conocer los requisitos del sistema relacionados con el agente de infraestructura New Relic Kubernetes y el recopilador New Relic OpenTelemetry (NRDOT), consulte:

* [Integración Kubernetes : compatibilidad y requisitos](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements)
* [OpenTelemetry para Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel)
