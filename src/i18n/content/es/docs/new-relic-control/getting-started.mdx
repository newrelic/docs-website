New Relic Control es una solución de gestión de observabilidad que proporciona control centralizado y escalable sobre las tuberías de agentes y telemetría en todo su entorno. Esta documentación cubre los componentes de la solución, la funcionalidad principal y las opciones de implementación.

## Componentes

* **[Agent Control](/docs/new-relic-control/agent-control/overview):** un componente supervisor liviano que gestiona otros agentes New Relic y la distribución New Relic de OpenTelemetry . Se comunica con control de flota para configurar, actualizar y monitorear de forma remota el estado de la instrumentación a nivel de infraestructura.
* **[Control de flota](/docs/new-relic-control/fleet-control/overview):** una interfaz centralizada que proporciona visibilidad y gestión remota de todos los agentes monitorear en su infraestructura. Permite a los equipos instalar, configurar, monitorear y actualizar el agente a escala.
* **[Control de pipelines](/docs/new-relic-control/pipeline-control/overview):** el componente de gestión de datos de New Relic Control que ayuda a filtrar y descartar datos de bajo valor antes de la ingesta de la base de datos.

## Alcance

Agente Control y Fleet están en disponibilidad general (GA) para Kubernetes. En esta etapa, apoyan la gestión de la configuración de instrumentación a nivel de infraestructura para clúster de Kubernetes, incluyendo:

* **Agente New Relic :** incluye agente basado en el agente de infraestructura New Relic y la distribución New Relic del recolector OpenTelemetry (NRDOT).
* **Instrumentación basada en estándares:** Incluye Prometheus y Fluent Bit.

La compatibilidad con el entorno de host Linux se encuentra en versión preliminar. Consulte la documentación de los hosts Linux para obtener información sobre las funciones disponibles.

El control de pipelines está disponible de forma general (GA) y se puede usar para gestionar datos del agente APM descartando datos de bajo valor. Para obtener detalles sobre los agentes compatibles, consulte [Versiones de agentes compatibles](/docs/new-relic-control/pipeline-control/modify-agent-configuration/#supported-agent-versions)

## Permisos requeridos

* **Agent Control:** Requiere el rol de Administrador de dominio de autenticación para configurar Agent Control y conectarlo a control de flota. Para obtener más información, consulte [Conceptos de gestión de usuarios](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings).
* **Control de flota:** Requiere el rol de Gerente de organización (denominado Configuración administrativa en el producto) para crear y editar flotas dentro de control de flota.
* **Control de pipelines:** Requiere el rol de Gerente de la organización para configurar y gestionar la puerta de enlace de Control de pipelines y las reglas de Control de pipelines.
