<DNT>Fleet Control</DNT> proporciona una gestión centralizada y remota para sus agentes de New Relic, ayudándole a administrar todo el ciclo de vida de la instrumentación a escala. Esta función está disponible de forma general (GA) para gestionar agentes en clústeres de Kubernetes. El soporte para administrar agentes en hosts Linux y Windows está actualmente en vista previa pública.

Con <DNT>Fleet Control</DNT>, puedes:

* **Centralice las operaciones del agente y reduzca el trabajo manual:** instale, monitorear, configure y actualice todos los agentes e integraciones en un solo lugar.
* **Personalice flotas para la gestión masiva:** agrupe agentes según sus necesidades de instrumentación y actualice fácilmente versiones de agentes para flotas completas con unos pocos clics. No se necesita guión
* **Elimine las brechas de telemetría:** gestione agentes en el clúster de Kubernetes con un supervisor liviano (Agent Control) que se escala en todo su entorno.

Para navegar a <DNT>Fleet Control</DNT>, vaya a <DNT>**All capabilities → New Relic Control → Fleets**</DNT>. Puede usar el pin para agregar New Relic Control a la barra de navegación izquierda para facilitar el acceso.

## ¿Por qué usar <DNT>Fleet Control</DNT>?

Gestionar la instrumentación en un entorno grande puede generar desafíos significativos. <DNT>Fleet Control</DNT> está diseñado para resolver estos problemas al proporcionarle un plano de control centralizado para:

* **Evite que el agente esté desactualizado:** mantenga el agente actualizado para evitar riesgos de seguridad e incompatibilidades y para cerciorar de poder aprovechar las últimas innovaciones New Relic.
* **Evite la deriva de configuración:** elimine las brechas ocultas de observabilidad y los problemas de cumplimiento garantizando una configuración consistente en toda su entidad.
* **Reparar instrumentación rota:** identifique y resuelva rápidamente problemas con su instrumentación para mantener la confianza en sus datos de observabilidad.
* **Reduzca el esfuerzo manual:** deje de depender de scripts únicos y actualizaciones manuales para gestionar su agente e integración.

## ¿Qué es una flota?

En <DNT>Fleet Control</DNT>, una flota es una colección homogénea de entidades gestionadas (como clústeres de Kubernetes o hosts) que comparten configuraciones de instrumentación idénticas. Las flotas sirven como agrupaciones lógicas de entidades administradas, lo que permite una gestión simplificada y una configuración de observabilidad consistente a escala.

Una flota se compone de tres componentes principales:

* **Despliegue activo:** un resumen del último conjunto de acciones aplicadas a la flota. Una implementación puede incluir una o más acciones, como instalar, eliminar o actualizar la configuración de un agente, y refleja el resultado final de esos cambios.
* **Agentes y configuraciones:** Los agentes y las configuraciones que se instrumentan en las entidades gestionadas de la flota.
* **Entidad gestionada:** todas las entidades gestionadas asignadas que están unidas a la flota.

### Entendiendo la entidad gestionada

En el contexto de <DNT>Fleet Control</DNT>, una entidad gestionada es un clúster de Kubernetes, un host Linux o un host Windows que ejecuta instrumentación de New Relic gestionada por <DNT>Agent Control</DNT>. Cada flota está asociada con un tipo específico de entidad gestionada (clúster de Kubernetes, host de Linux o host de Windows), y todas las entidades dentro de una flota deben coincidir con ese tipo. La configuración del agente se especifica a nivel de flota, y cada entidad gestionada en la flota comparte la misma configuración.

* **Entidad administrada:** Un clúster de Kubernetes o host que tiene Agent Control instalado. Esto significa que la entidad es capaz de ser administrada por <DNT>Fleet Control</DNT>, pero no necesariamente está unida a una flota todavía.
* **Entidad gestionada asignada:** Una entidad gestionada que se ha unido a una flota. Esta entidad ahora es administrada activamente por <DNT>Fleet Control</DNT>, y su configuración está determinada por la flota a la que pertenece.
* **Entidad gestionada no asignada:** Una entidad gestionada que no está unida actualmente a ninguna flota. Aunque tiene Agent Control instalado y se puede administrar, actualmente está operando con su propia configuración local.

<Callout title="Importante" variant="important">
  No puede agregar un clúster de Kubernetes o un host a una flota antes de instalar el agente Control para convertirlo en una entidad gestionada.
</Callout>

## Característica clave

* **Gestión centralizada:** ofrece una plataforma unificada para gestionar agentes que se ejecutan en flotas de clúster de Kubernetes y hosts, lo que le permite gestionar la instrumentación en toda su infraestructura a escala.
* **Visibilidad del tiempo real:** gestione el estado de su instrumentación con una vista clara del estado de la flota, el estado de los agentes, el progreso del despliegue y los errores de configuración.
* **Organización de flotas flexible:** cree flotas distintas para representar diferentes estándares de instrumentación, entornos de aplicación o necesidades específicas del equipo. Esta estructura admite tanto equipos de observabilidad centralizados que gestionan toda la configuración como modelos descentralizados donde equipos individuales poseen y gestionan sus flotas específicas empleando licencias granulares.
* **Configuración remota:** Admite la configuración remota del agente e integración New Relic responsable de monitorear varios aspectos de su entorno y reportar puntos de datos clave, incluidos métricas, eventos, logs y traza (MELT).

## Requisitos previos

Antes de configurar <DNT>Fleet Control</DNT>, asegúrese de que su sistema cumpla con los siguientes requisitos previos:

* **Inscripción en la Vista previa pública para hosts:** Para utilizar <DNT>Fleet Control</DNT> para administrar agentes en hosts Linux o Windows, debe inscribirse en el programa de Vista previa pública para esta funcionalidad específica. La administración de agentes en clústeres de Kubernetes está disponible de forma general y no requiere inclusión voluntaria.
* **Agent Control:** Debes tener [Agent Control](/docs/new-relic-control/agent-control/setup) instalado en cualquier clúster de Kubernetes o host que planees administrar. Este es un componente requerido para toda la funcionalidad de <DNT>Fleet Control</DNT>.

## Permisos requeridos

**Permisos a nivel de organización:** Se requiere el rol de **Organization Manager** (denominado configuración administrativa en el producto) para crear y editar flotas dentro de <DNT>Fleet Control</DNT>. Alternativamente, puede [crear un rol personalizado](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#create-role) con permisos de gestión de flotas.
