Control de flota proporciona administración remota y centralizada para su agente New Relic, ayudándole a gestionar todo el ciclo de vida de la instrumentación a escala. Esta característica está generalmente disponible (GA) para gestionar agentes en el clúster de Kubernetes. El soporte para la administración de agentes en hosts se encuentra actualmente en versión preliminar pública.

Con el control de flota, puedes:

* **Centralice las operaciones del agente y reduzca el trabajo manual:** instale, monitorear, configure y actualice todos los agentes e integraciones en un solo lugar.
* **Personalice flotas para la gestión masiva:** agrupe agentes según sus necesidades de instrumentación y actualice fácilmente versiones de agentes para flotas completas con unos pocos clics. No se necesita guión
* **Elimine las brechas de telemetría:** gestione agentes en el clúster de Kubernetes con un supervisor liviano (Agent Control) que se escala en todo su entorno.

Para navegar a control de flota, vaya a **Capabilities → New Relic Control → Fleets**. Puedes usar el pin para agregar New Relic Control a la barra de navegación del lado izquierdo para un fácil acceso.

## ¿Por qué emplear control de flota?

La gestión de la instrumentación en un entorno grande puede generar desafíos importantes. El control de flota está diseñado para resolver estos problemas al brindarle un plano de control centralizado para:

* **Evite que el agente esté desactualizado:** mantenga el agente actualizado para evitar riesgos de seguridad e incompatibilidades y para cerciorar de poder aprovechar las últimas innovaciones New Relic.
* **Evite la deriva de configuración:** elimine las brechas ocultas de observabilidad y los problemas de cumplimiento garantizando una configuración consistente en toda su entidad.
* **Reparar instrumentación rota:** identifique y resuelva rápidamente problemas con su instrumentación para mantener la confianza en sus datos de observabilidad.
* **Reduzca el esfuerzo manual:** deje de depender de scripts únicos y actualizaciones manuales para gestionar su agente e integración.

## ¿Qué es una flota?

En control de flota, una flota es una colección homogénea de entidades gestionadas (como un clúster de Kubernetes) que comparten una configuración de instrumentación idéntica. Las flotas sirven como agrupaciones lógicas de entidades gestionadas, lo que permite una administración optimizada y una configuración de observación consistente a escala.

Una flota se compone de tres componentes principales:

* **Despliegue activo:** un resumen del último conjunto de acciones aplicadas a la flota. Una implementación puede incluir una o más acciones, como instalar, eliminar o actualizar la configuración de un agente, y refleja el resultado final de esos cambios.
* **agente &amp; configuración:** El agente y configuración que se están instrumentando en el clúster gestionado de Kubernetes
* **Entidad gestionada:** todas las entidades gestionadas asignadas que están unidas a la flota.

### Entendiendo la entidad gestionada

En el contexto de control de flota, una entidad gestionada se refiere a un clúster de Kubernetes que ejecuta New Relic Infrastructure e instrumentación de log gestionada por el Agent Control. La configuración del agente se especifica a nivel de flota y cada clúster de Kubernetes en la flota comparte la misma configuración que otros clústeres en esa flota.

* **Entidad gestionada:** Un clúster de Kubernetes u host que tiene instalado el agente Control (AC). Esto significa que la entidad es capaz de ser gestionada por control de flota, pero no está necesariamente unida a una flota todavía.
* **Entidad gestionada asignada:** una entidad gestionada que se unió a una flota. Esta entidad ahora está gestionada activamente por el control de flota, y su configuración está dictada por la flota a la que pertenece.
* **Entidad gestionada no asignada:** una entidad gestionada que actualmente no está unida a ninguna flota. Si bien tiene instalado el Agente Control y se puede gestionar, actualmente está funcionando empleando su propia configuración local.

<Callout title="importante">
  No puede agregar un clúster de Kubernetes o un host a una flota antes de instalar el agente Control para convertirlo en una entidad gestionada.
</Callout>

## Característica clave

* **Gestión centralizada:** ofrece una plataforma unificada para gestionar agentes que se ejecutan en flotas de clúster de Kubernetes y hosts, lo que le permite gestionar la instrumentación en toda su infraestructura a escala.
* **Visibilidad del tiempo real:** gestione el estado de su instrumentación con una vista clara del estado de la flota, el estado de los agentes, el progreso del despliegue y los errores de configuración.
* **Organización de flotas flexible:** cree flotas distintas para representar diferentes estándares de instrumentación, entornos de aplicación o necesidades específicas del equipo. Esta estructura admite tanto equipos de observabilidad centralizados que gestionan toda la configuración como modelos descentralizados donde equipos individuales poseen y gestionan sus flotas específicas empleando licencias granulares.
* **Configuración remota:** Admite la configuración remota del agente e integración New Relic responsable de monitorear varios aspectos de su entorno y reportar puntos de datos clave, incluidos métricas, eventos, logs y traza (MELT).

## Requisitos previos

Antes de configurar control de flota, cerciorar de que su sistema cumpla con los siguientes requisitos previos:

* **Opt-in de vista previa pública para hosts:** para usar el control de flota para gestionar agentes en los hosts, debe optar por participar en el programa de vista previa pública para esta función específica. La administración de agentes en el clúster de Kubernetes está disponible de manera general y no requiere subscripción.
* **Agente Control:** debe tener [Agente Control](/docs/new-relic-control/agent-control/setup) instalado en cualquier clúster de Kubernetes u host que planee gestionar. Este es un componente necesario para todas las funciones de control de flotación.

## Permisos requeridos

**Licencias de ámbito de organización:** el rol de **Organization Manager** (conocido como una configuración administrativa en el producto) es necesario para crear y editar flotas dentro del control de flota. Alternativamente, puede [crear un rol personalizado](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#create-role) con licencias de administración de flotas.
