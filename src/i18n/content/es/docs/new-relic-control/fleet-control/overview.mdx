control de flota proporciona comando centralizado y remoto sobre flotas de clústeres gestionados para la gestión del ciclo de vida de la instrumentación a escala. Emplee control de flota para:

* **Centralice las operaciones del agente y reduzca el trabajo manual:** instale, monitorear, configure y actualice todos los agentes e integraciones en un solo lugar.
* **Personalice flotas para la gestión masiva:** agrupe agentes según sus necesidades de instrumentación y actualice fácilmente versiones de agentes para flotas completas con unos pocos clics. No se necesita guión
* **Elimine las brechas de telemetría:** gestione agentes en el clúster de Kubernetes con un supervisor liviano (Agent Control) que se escala en todo su entorno.

Para navegar a control de flota, vaya a **Capabilities → New Relic Control → Fleets**. Puedes usar el pin para agregar New Relic Control a la barra de navegación del lado izquierdo para un fácil acceso.

## ¿Qué es una flota?

En control de flota, una flota es una colección homogénea de entidades gestionadas (como un clúster de Kubernetes) que comparten una configuración de instrumentación idéntica. Las flotas sirven como agrupaciones lógicas de entidades gestionadas, lo que permite una administración optimizada y una configuración de observación consistente a escala.

Una flota se compone de tres componentes principales:

* **Implementación activa:** el resumen de implementación más reciente

* **configuración:** La configuración que se está aplicando a tu clúster de Kubernetes

* **entidad:** Todo el clúster de Kubernetes dentro de la flota

### Entendiendo la entidad gestionada

En el contexto de control de flota, una entidad gestionada se refiere a un clúster de Kubernetes que ejecuta New Relic Infrastructure e instrumentación de log gestionada por el Agent Control. La configuración del agente se especifica a nivel de flota y cada clúster de Kubernetes en la flota comparte la misma configuración que otros clústeres en esa flota.

<Callout title="importante">
  No se puede agregar un clúster de Kubernetes a una flota antes de instalar el Agent Control en ese clúster.
</Callout>

## Característica clave

* **Gestión centralizada:** Ofrece una plataforma unificada para gestionar agentes que se ejecutan en flotas de clúster de Kubernetes, lo que le permite gestionar la instrumentación en toda su infraestructura a escala.
* **Agrupación y propiedad:** funciona como un mecanismo de agrupación y como un límite de propiedad, lo que permite que diferentes equipos gestionen sus respectivas flotas de forma independiente y al mismo tiempo mantengan prácticas de observabilidad consistentes.
* **Configuración remota:** Admite configuración remota de varios agentes. Para más detalles, ver [el alcance](/docs/new-relic-control/getting-started/#scope).

Estos agentes son responsables de monitorear varios aspectos de su entorno y reportar puntos de datos clave, incluyendo métricas, eventos, logs y trazas (MELT).

## Requisitos previos

Antes de configurar control de flota, cerciorar de que su sistema cumpla con los siguientes requisitos previos:

* **Suscribir:** como Agent Control y control de flota están en una vista previa pública, debes suscribirte para usar el producto.
* **Agent Control:** cerciorar de instalar [Agent Control](/docs/new-relic-control/agent-control/setup) en el clúster de Kubernetes que planea agregar a sus flotas.

## Permisos requeridos

**Licencias de alcance de la organización:** el rol de Gerente de la organización (denominado configuración administrativa en el producto) es necesario para crear y editar flotas dentro del control de flota. Alternativamente, puede [crear un rol personalizado](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#create-role) con licencias de administración de flotas.
