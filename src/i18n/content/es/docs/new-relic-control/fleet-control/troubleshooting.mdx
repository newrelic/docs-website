Este documento describe los pasos para solucionar problemas comunes durante la instalación u operación de <DNT>Fleet Control</DNT>.

## Solución de problemas de despliegues con eventos de despliegue

<DNT>Fleet Control</DNT> proporciona una pestaña <DNT>deployment events</DNT> que le brinda visibilidad granular de cada evento que ocurre durante un despliegue. Esta es la herramienta principal para solucionar problemas de despliegue y comprender las causas raíz.

### Acceso a eventos de despliegue

Hay dos formas de acceder a la pestaña Eventos de despliegue:

**Desde el historial de despliegue:**

1. Navegue a la pestaña <DNT>deployments</DNT> dentro de una flota específica.
2. Haga clic en la fila del despliegue que desea investigar.
3. En la vista de resumen de despliegue, haga clic en la pestaña <DNT>Deployment Events</DNT>.

**Después de crear una nueva implementación:**

1. Después de iniciar un despliegue, la vista de progreso se completará y pasará a la vista de resumen del despliegue.
2. En esta vista de resumen, haga clic en la pestaña <DNT>Deployment Events</DNT> para ver inmediatamente los eventos para la resolución de problemas o verificación.

### Comprensión de la tabla de eventos de implementación

La tabla <DNT>deployment events</DNT> muestra datos atómicos a nivel de evento para cada acción realizada durante un despliegue. La tabla incluye las siguientes columnas:

* <DNT>**Timestamp:**</DNT> Cuándo ocurrió el evento.
* <DNT>**Status:**</DNT> El estado del evento (por ejemplo, <DNT>Completed, Config Applied, Applying</DNT>).
* <DNT>**Managed entity name:**</DNT> El nombre del clúster o host donde ocurrió el evento.
* <DNT>**Agent name:**</DNT> El agente específico involucrado en el evento.
* <DNT>**Agent type:**</DNT> El tipo de agente (por ejemplo, <DNT>New Relic Infrastructure, Fluent Bit</DNT>).
* <DNT>**Event:**</DNT> El tipo de evento que ocurrió.
* <DNT>**Details:**</DNT> Información adicional sobre el evento, que a menudo incluye mensajes de error para eventos fallidos.
* <DNT>**Duration:**</DNT> Cuánto tiempo tardó el evento en completarse.

### Filtrado y búsqueda de eventos

La pestaña <DNT>deployment events</DNT> ofrece varias formas de acotar los eventos:

* <DNT>**Search:**</DNT> Utilice la barra de búsqueda para encontrar eventos específicos por palabra clave.
* <DNT>**Filter by entities:**</DNT> Filtre para mostrar eventos de una entidad administrada específica. Esto es especialmente útil cuando se tiene una gran flota (por ejemplo, 100 clústeres) y solo una o unas pocas entidades fallaron.
* <DNT>**Filter by status:**</DNT> Filtre los eventos por su estado (por ejemplo, <DNT>Completed, Config Applied, Applying</DNT>) para enfocarse en eventos exitosos o problemáticos.

### Resolución de problemas avanzada con NRQL

Para una resolución de problemas más avanzada, puede acceder a la consulta NRQL subyacente que alimenta la tabla Eventos de despliegue:

1. En la pestaña <DNT>Deployment Events</DNT>, localice el botón a la derecha del encabezado de la tabla de eventos.
2. Haga clic en este botón para abrir una pestaña de consulta NRQL en el <DNT>Data Explorer</DNT>.
3. La consulta se completará previamente con el ID de despliegue y mostrará columnas adicionales con información más detallada, que incluye varios ID y metadatos.
4. Puede modificar esta consulta si es necesario para investigar más a fondo el problema.
5. Utilice las funciones estándar de <DNT>Data Explorer&apos;s</DNT>, como alternar entre las vistas de formato de tabla y JSON.
