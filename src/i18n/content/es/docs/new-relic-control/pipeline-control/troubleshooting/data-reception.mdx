\* Si no recibe telemetry data de los productores configurados para enviar a la puerta de enlace, pero la puerta de enlace está produciendo sus propios datos de monitoreo, esta guía lo ayudará a diagnosticar y resolver problemas comunes. Siga los pasos a continuación según el problema específico que encuentre: \*/

## Problema: Mala configuración del productor de telemetría

El productor de telemetría está mal configurado, lo que provoca que no se envíen telemetry data a la puerta de enlace.

## Solución

* Acceda a los logs del productor para identificar errores de configuración.
* Consulte la documentación del API, del agente o del productor de telemetría correspondiente para conocer los pasos de configuración correctos.

## Problema: Tengo una tarifa limitada

Estás alcanzando los límites de velocidad en tus telemetry data.

## Solución

* Verifique los códigos de respuesta del cliente HTTP de la puerta de enlace para los códigos de estado 429 recibidos desde la API de New Relic.
* Revisar evento creado en tu cuenta indicando limitación de velocidad.
* Consulte la [documentación de limitación de velocidad](https://docs.newrelic.com/search-results/?query=rate%20limiting&page=2) para obtener orientación sobre cómo gestionar y ajustar las velocidades telemetry data .
