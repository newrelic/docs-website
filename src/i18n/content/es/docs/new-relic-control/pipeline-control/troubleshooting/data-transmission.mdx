\* Si tiene problemas con la recepción de datos en New Relic, esta guía lo ayudará a diagnosticar y resolver problemas comunes. Siga los pasos a continuación según el problema específico que encuentre: \*/

## Problema: Problemas de resolución de DNS entre el productor de telemetría y la puerta de enlace

Los agentes no pueden conectarse a la puerta de enlace debido a errores de resolución de DNS.

## Solución

* Acceda a los logs del agente (a través de UI o directamente) y busque errores de conexión.
* Consulte a su administrador de red para ajustar la configuración de DNS en función de su infraestructura y topología de red. Consulte [la documentación del agente](/docs/apm/new-relic-apm/getting-started/introduction-apm/) para obtener más detalles.

## Problema: Problemas con el certificado SSL del agente-puerta de enlace

Hay un problema con la configuración del certificado SSL entre el agente y la puerta de enlace.

## Solución

* Verifique los logs del agente para detectar errores de conexión SSL.
* Cerciorar de que los certificados SSL estén correctamente configurados y sean válidos, teniendo en cuenta su infraestructura y topología de red. Consulte [la documentación del agente](/docs/apm/new-relic-apm/getting-started/introduction-apm/) para obtener más detalles.

## Problema: Mala configuración de la puerta de enlace

Los datos llegan a la puerta de enlace, pero no se pueden publicar en New Relic.

## Solución

* Verificar solicitud saliente y error métrico.
* Revisar regla métrica fallida.
* Inspeccione los logs de los pods que no se inician.
* Corrija la configuración de la puerta de enlace y cerciorar de que todos los pods estén operativos.

## Problema: productor/protocolo de telemetría no compatible

Los datos se envían desde una API o protocolo no compatible.

## Solución

* Verificar la compatibilidad con New Relic. Un cliente API incorrecto devolverá un código de estado `501` .
* Presentar una solicitud de soporte para características.
* Configurar el productor de telemetría para enviar datos directamente a New Relic.

## Problema: Regla destructiva

Una regla está descartando todos los datos, impidiendo que lleguen a New Relic.

## Solución

* Verificar datos de caída métrica.
* Modifique o elimine la regla destructiva para permitir el flujo de datos.

## Problema: faltan datos

Faltan datos en el backend de New Relic luego de la ingestión.

## Solución

* Revise el error métrico y verifique los tiempos de espera del lado del cliente.
* Evaluar los signos de agotamiento de recursos.
* Comprobar [el estado de New Relic](https://status.newrelic.com/).
* Examinar los logs de la puerta de enlace durante el tiempo afectado
