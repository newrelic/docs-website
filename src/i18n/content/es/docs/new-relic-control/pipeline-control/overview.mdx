El control de tuberías es el componente de gestión de datos de New Relic Control. Puede ayudar a filtrar y descartar datos de bajo valor antes de incorporarlos a la plataforma New Relic.

Control de pipelines consta de dos subcomponentes: Control de pipelines gateway (simplemente denominado gateway) y Control de pipelines cloud rules.

La puerta de enlace está diseñada para filtrar datos o atributos antes de que salgan de su red. Las cloud reglas realizan ese procesamiento dentro de la New Relic cloud. En ambos escenarios, Control de pipelines emplea una sintaxis de consulta NRQL idéntica para procesar los datos.

Para emplear la puerta de enlace, primero debes configurarla en tu entorno. Las reglas cloud no requieren configuración.

Para comprender los costos asociados con el uso de Control de pipelines, consulte [Costos de Control de pipelines](/docs/new-relic-control/pipeline-control/costs).

## Requisitos previos

Antes de comenzar, debes tener:

* Una cuenta de New Relic con licencias de administrador de productos de la organización y de administrador de la organización

<Callout variant="important" title="importante">
  El administrador de producto de la organización y el rol de usuario del gerente de la organización están obligados a:

  * Instalar puerta de enlace
  * Agregar clúster a una puerta de enlace
  * Actualizar la puerta de enlace (por ejemplo, nuevas reglas y cambios de configuración)
  * desplegar una configuración de puerta de enlace
  * Ver reglas de puerta de enlace
</Callout>

* Familiaridad con las operaciones y la gestión de Kubernetes
* Acceso de administrador al clúster de Kubernetes para implementar y gestionar recursos dentro de su clúster
* Licencias para crear y gestionar nombres de host DNS privados dentro de su red
* Licencias para crear y gestionar registros DNS
* Helm instalado localmente
* Comprensión de NRQL para crear y gestionar reglas de canalización

<Callout variant="important">
  * Las reglas de control de pipelines solo se aplican a los datos que llegan desde el momento en que se crea la regla, no eliminan los datos que [ya se ingirieron](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-data-retention#data-deletion).
  * Al crear reglas, usted es responsable de garantizar que las reglas identifiquen y descarten con precisión los datos que cumplan las condiciones que estableció. Usted también es responsable de monitorear la regla, así como los datos que divulga a New Relic.
</Callout>
