Configure e implemente la puerta de enlace de Control de pipelines para gestionar el flujo de datos desde su entidad monitoreada a New Relic.

## Consideraciones para el uso de la puerta de enlace

A medida que integra y emplea el componente de puerta de enlace, tenga en cuenta estas consideraciones importantes:

### atributo y gestión de datos

* **Soporte de atributos:** El gateway utiliza las convenciones semánticas de OpenTelemetry para los atributos. Consulte la [documentación del gateway](/docs/new-relic-control/pipeline-control/gateway/overview) para obtener detalles sobre los atributos admitidos.
* **Manejo de datos/atributos:** la puerta de enlace reenvía sus requests para eliminar datos o atributos para datos de intervalos de tiempo métricos, trazas y trazas directamente a New Relic sin modificaciones.

### Análisis e integración de datos

* **Análisis de logs:** La puerta de enlace no incluye el análisis de Grok para logs. El análisis de Grok es un método empleado para extraer datos estructurados de mensajes de log no estructurados empleando patrones predefinidos y expresiones regulares. Es posible que necesite métodos alternativos para estructurar los datos de log para realizar consultas y visualizarlos.
* **Integración con Prometheus:** La puerta de enlace no admite la escritura remota de Prometheus para métrica, lo que puede afectar su integración con sistemas basados en Prometheus.

### Licencias y cumplimiento

* **Gestión de claves de licencia:** la puerta de enlace no admite varias claves de licencia de fuentes nativas del Protocolo OpenTelemetry (OTLP), como agente recolector o SDK APM . Considere esto al gestionar sus licencias.
* **Estándares de cumplimiento:** la puerta de enlace no cumple con los estándares de FedRAMP, lo que puede afectar su uso si necesita cumplir con los estándares de seguridad federales.

## Configurar la puerta de enlace de Control de tuberías [#set-up-gateway]

<img title="Install gateway" alt="Screenshot of the button to start Pipeline Control gateway installation" src="/images/install-pipeline-control.webp" />

Para configurar la puerta de enlace de Control de tuberías:

1. En su cuenta New Relic , en Integración y agente, haga clic en **Set up Pipeline Control** o busque **gateway** y seleccione **Pipeline Control gateway**.
2. Cerciorar de que la cuenta seleccionada sea la que desea emplear para el Control de tuberías. En caso contrario, haga clic en **Select an account**, seleccione la cuenta de su elección y luego haga clic en **Continue**.
3. Para ingresar sus credenciales, proporcione una clave de licencia existente o cree una nueva clave. Si crea una nueva clave, copie la clave. Luego, haga clic en **Continue**.
4. Seleccione o cree una flota de gateways:

* Seleccione una flota existente si desea que su nuevo gateway comparta una configuración existente (reglas de muestreo, filtrado y transformación) con otros gateways en su organización.
* Ingrese un nombre único para la flota de gateways si necesita una configuración nueva y aislada para un entorno o unidad de negocio específico.

5. Seleccione un clúster de Kubernetes existente mediante el menú desplegable o cree uno nuevo.

6. Si crea un nuevo clúster, ingrese el nombre y namespace del clúster. El namespace predeterminado es `newrelic`.

7. Descargue el archivo de configuración, `values-newrelic-gateway.yaml` a un directorio.

8. Ejecute el comando de instalación que se muestra en pantalla:

   ```bash
   helm repo add newrelic https://helm-charts.newrelic.com && helm repo update && \
   helm upgrade --install agent-control-bootstrap -n newrelic-agent-control \
   newrelic/agent-control-bootstrap --create-namespace --values </path/to/values-newrelic-gateway.yaml>
   ```

   <Callout variant="tip">
     Reemplace `</path/to/values-newrelic-gateway.yaml>` con la ruta donde se encuentra el archivo de configuración.
   </Callout>

9. Haga clic en **Continue**.

10. Haga clic en **Test connection** y espere hasta que la conexión sea exitosa.

Para ver los detalles de la configuración de su puerta de enlace, vaya a New Relic Control y haga clic en **Gateway**.

## ¿Por qué usar múltiples flotas?

Imagine que una corporación (**The Corp**) tiene dos divisiones: **OrgA** (necesita el 100 % de los datos por seguridad) y **OrgB** (necesita un muestreo del 1 % para controlar los costos por alto volumen).

Con solo una flota, **The Corp** está atascada. Si **OrgB** aplica un muestreo del 1% para ahorrar dinero, **OrgA** pierde el 99% de sus datos de seguridad de vuelo. El control de costos de un equipo rompe el monitoreo de misión crítica del otro equipo.

Al usar múltiples flotas, **The Corp** puede separar sus &quot;cerebros&quot;:

* **Flota de misión crítica**: **OrgA** se une a esta flota para obtener el 100 % de retención de datos.

* **Flota de alto volumen**: **OrgB** se une a esta flota para un muestreo del 1 % y una redacción conforme al GDPR para sus sensores europeos.

Como resultado, los equipos independientes ahora pueden administrar reglas únicas de muestreo, seguridad y cumplimiento sin afectar al resto de la organización.

Instalar un gateway ahora es como contratar a un trabajador. Solo tienes que decirles a qué equipo (flota) unirse.

## Agregar más clúster

Para agregar un nuevo clúster a la configuración de su puerta de enlace,

1. Abra New Relic Control y luego seleccione **Gateway**.
2. En la tabla de clúster, haga clic en **Add a cluster**.
3. Siga las instrucciones de [configuración de la puerta de enlace](/docs/new-relic-control/pipeline-control/set-up-and-deploy-pipeline-control#set-up-gateway).
4. Regrese a la página de resumen de la puerta de enlace. Deberías ver el clúster recién agregado.

Después de crear sus clústeres de gateway, configure el procesamiento del pipeline mediante la configuración YAML. Consulte [Configuración de Gateway](/docs/new-relic-control/pipeline-control/gateway/overview).
