New Relic Pathpoint는 비즈니스의 고유한 요구 사항을 충족하도록 사용자 지정할 수 있는 유연한 구성 시스템을 제공합니다. Pathpoint는 대화식 GUI에서 구성을 지원합니다. JSON 구성을 다운로드 및 업로드할 수도 있습니다. 이는 구성의 버전을 제어하려는 경우에 특히 유용합니다.

이 수준의 사용자 지정을 통해 조직의 특정 요구 사항에 맞는 맞춤형 관찰 가능성 솔루션을 만들 수 있습니다.

이 페이지에서는 다음을 배우게 됩니다.

* GUI로 Pathpoint를 구성하는 방법
* JSON 교환 형식을 사용하는 방법

## JSON 교환 형식 사용

이전 섹션에 표시된 단계, 단계 및 접점 편집기를 사용하여 Pathpoint 구현을 구성하는 것은 매우 쉽고 직관적입니다. 경우에 따라 파일에 구성을 저장하고 나중에 가능하면 다른 계정이나 Pathpoint 인스턴스에서 해당 구성을 가져오는 것이 유용하거나 필요할 수 있습니다. 프로덕션 Pathpoint 구현의 경우 구성 아카이브를 git과 같은 버전 제어 시스템에 JSON으로 저장하는 것이 좋습니다.

### 현재 구성을 JSON으로 다운로드

Pathpoint는 현재 JSON 구성 파일을 컴퓨터에 직접 다운로드할 수 있는 유연성을 제공합니다. 이 기능은 파일을 변경해야 하는 경우에 특히 유용합니다.

현재 활성 JSON 구성 파일을 다운로드하려면:

<Steps>
  <Step>
    New Relic 계정에서 **Apps > Pathpoint**로 이동합니다. 를 클릭하십시오<img src={threeLines} class="inline" style={{height: "30px", width: "30px"}} alt="Three lines" title="Three lines"/>

    왼쪽 상단 모서리에.
  </Step>

  <Step>
    **JSON configuration**클릭합니다.
  </Step>

  <Step>
    그런 다음 아래쪽 화살표를 클릭하십시오.

    <img
      title="download active JSON config file"
      alt="download active JSON config file"
      src={downloadFile}
    />
  </Step>

  이제 활성 JSON 구성 파일이 다운로드되었습니다.
</Steps>

### JSON 파일을 업로드하여 현재 구성 바꾸기

새 JSON 구성 파일을 로드하려면:

<Steps>
  <Step>
    New Relic 계정에서 **Apps > Pathpoint**로 이동하고 클릭합니다.<img src={threeLines} class="inline" style={{height: "30px", width: "30px"}} alt="Three lines" title="Three lines"/>

    .
  </Step>

  <Step>
    결과 창에서 **JSON configuration**클릭합니다.
  </Step>

  <Step>
    그러면 대화 상자 창이 열립니다. **Description**아래에서 새 JSON 구성 파일을 설명합니다. 그런 다음 메모를 제공하고 **From file** 클릭하여 JSON 구성 파일을 찾습니다.

    <img
      title="Upload new JSON config file"
      alt="Upload new JSON config file"
      src={uploadJson}
    />
  </Step>

  <Step>
    Pathpoint를 업데이트하는 데 사용할 JSON 구성 파일을 찾고 **Open**클릭합니다.

    <img
      title="open new JSON config file"
      alt="open new JSON config file"
      src={openFile}
    />
  </Step>

  마지막으로 선택한 특정 JSON 파일의 업데이트와 함께 Pathpoint가 표시됩니다.
</Steps>
