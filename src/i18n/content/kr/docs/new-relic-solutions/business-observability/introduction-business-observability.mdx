비즈니스 관찰 가능성은 IT 원격 측정과 비즈니스 분석 간의 실시간 상호 작용을 캡처, 측정 및 관리하는 기술입니다.

종종 관찰 가능성 전략은 사일로에서 구현됩니다. 즉, 점을 비즈니스에 다시 연결하는 것은 반응적이고 나중에 생각되며 매우 어려울 수 있습니다. 비즈니스 관찰 가능성은 시스템, 애플리케이션 및 프로세스의 성능에 대한 주요 이니셔티브를 적극적으로 조정하는 것입니다.

비즈니스 관찰 가능성 관행을 구현함으로써 조직은 전체 운영에 영향을 미치고 상당한 재정적 이익을 달성하는 정보에 입각한 결정을 내릴 수 있습니다.

## 시작하시겠습니까? [#get-started]

아직 하지 않았다면 지금 [무료 New Relic 계정에 가입하여](https://newrelic.com/signup) 비즈니스 관찰 가능성 여정을 시작하십시오.

비즈니스 관찰 가능성 실행에 도움이 되는 많은 솔루션을 제공합니다. 다음은 대규모 고객이 사용하는 주요 전략 중 일부입니다.

### 맞춤 데이터 보고 [#custom-data]

기본적으로 많은 유용한 데이터를 보고하지만 New Relic 사용을 최대한 활용하려면 비즈니스에 고유하고 의미 있는 정보가 포함된 사용자 지정 데이터를 보고해야 합니다.

사용자 지정 데이터를 추가하기 전에 어떻게 작동할지 계획을 세워 효율적이고 조직적으로 구현하고 "노이즈" 데이터를 너무 많이 생성하지 않도록 해야 합니다.

사용자 지정 데이터를 추가하는 방법은 다음과 같습니다.

* 기존 에이전트 및 통합 구성
* 에이전트 API 사용
* Metric, Event, Log 및 Trace API를 사용하여 모든 데이터 가져오기

사용자 정의 데이터 추가에 대한 팁은 [사용자 정의 데이터를](/docs/data-apis/custom-data/get-custom-data-from-any-source)참조하십시오.

### 사용자 정의 UI 시각화 [#visualizations]

가져오는 데이터를 사용자 정의할 수 있는 것처럼 비즈니스 구조에 맞게 해당 데이터의 시각화를 사용자 정의할 수 있습니다. 이에 대한 예:

* 특정 비즈니스 문제 해결을 목표로 하는 [맞춤형 대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) 생성
* 모니터링 대상의 업무상 중요한 그룹인 [워크로드](/docs/new-relic-solutions/new-relic-one/workloads/use-workloads)생성
