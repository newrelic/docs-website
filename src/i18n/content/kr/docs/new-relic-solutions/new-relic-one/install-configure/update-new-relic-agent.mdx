New Relic 관찰 가능성 플랫폼은 애플리케이션 또는 스택의 모든 부분을 모니터링할 수 있는 유연성을 제공합니다. 이것은 원격 측정 데이터를 수집하여 New Relic으로 보내는 New Relic 에이전트의 힘을 통해 가능합니다. 다른 소프트웨어와 마찬가지로 에이전트를 관리해야 하므로 에이전트를 최신 상태로 유지해야 합니다.

에이전트 버전에 대한 정보는 [릴리스 정보](/docs/release-notes) 를 참조하십시오. 최신 버전을 다운로드하고 설치하려면 해당 에이전트에 대한 문서를 참조하십시오.

## 에이전트를 업데이트하는 이유

에이전트를 최신 상태로 유지해야 하는 세 가지 주요 이유는 다음과 같습니다.

* **보안:** 우리 코드나 우리 코드가 의존하는 라이브러리에서 취약점을 발견하면, 우리는 그것을 패치하고 그것을 버전 업데이트로 당신에게 배송합니다. 에이전트를 최신 상태로 유지하면 누군가가 이러한 취약점 중 하나를 악용하여 작업에 영향을 줄 수 있는 위험이 줄어듭니다.
* **버그 수정:** 최신 버그 수정으로 최신 에이전트 버전을 사용하고 있는지 확인하여 버그를 식별하고 보고하는 시간과 노력을 절약하세요.
* **새로운 기능:** 더 많은 환경을 보고, 앱에서 문제를 찾고, 더 효율적으로 종료할 수 있도록 에이전트를 지속적으로 개선하고 있습니다. New Relic이 제공하는 모든 것을 활용하려면 최신 에이전트 버전을 사용 중인지 확인하십시오.

## 에이전트를 정기적으로 업데이트

가능한 한 빨리 최신 에이전트 버전으로 업데이트하는 것이 좋습니다. 조직에서 최신 버전으로 업그레이드하지 못하도록 하는 관행을 확립한 경우 에이전트가 최대 90일 이전 버전으로 정기적으로 업데이트되었는지 확인하십시오.

## 에이전트 버전 추적

New Relic 에이전트 groundskeeper 앱을 사용하여 에이전트 버전을 추적하십시오. 액세스하려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Apps > Agent groundskeeper**로 이동하십시오.

  title="Groundskeeper"
  alt="An image showing the application Groundskeeper"

/>

다음을 보려면 관리인 앱을 사용하십시오.

* 상담원 나이
* 에이전트를 사용하는 애플리케이션 이름
* 버전 번호
* 버전 출시일

## 에이전트 업데이트 시 고려 사항

때때로 새 에이전트 버전에는 주요 변경 사항이 포함됩니다. 에이전트 업데이트를 준비할 때 [특정 언어 에이전트에 대한 문서를](/docs/new-relic-solutions/new-relic-one/install-configure/compatibility-requirements-new-relic-agents-products/) 검토하십시오.

때때로 릴리스되는 새로운 기능에는 분산 추적 및 컨텍스트의 로그와 같은 추가 데이터 수집이 필요합니다. 수집된 모든 바이트가 가치를 제공하도록 열심히 노력하고 있지만 수집 속도를 조정할 수 있는 방법이 있습니다. 수집 최적화에 대한 자세한 내용은 계정 관리자에게 문의하거나 [지원 포럼](https://discuss.newrelic.com) 을 확인하십시오.

## 자동으로 에이전트 배포

에이전트가 최신 상태를 유지할 수 있도록 배포 자동화를 권장합니다. 자동화 솔루션에는 Ansible, Chef 및 Puppet이 포함됩니다. 자세한 내용은 [APM 모범 사례 가이드](/docs/new-relic-solutions/best-practices-guides/full-stack-observability/apm-best-practices-guide/#agents-current) 를 참조하세요.
