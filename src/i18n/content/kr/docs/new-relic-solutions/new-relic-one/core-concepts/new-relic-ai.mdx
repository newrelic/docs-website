NRAI(New Relic AI)는 엔지니어뿐만 아니라 팀의 모든 구성원이 관찰 가능성을 향상할 수 있도록 설계되었습니다. 우리는 OpenAI의 LLM(대형 언어 모델)을 통합 원격 측정 데이터 플랫폼과 결합하여 시스템과 New Relic의 작동 방식을 깊이 이해하는 AI 도우미를 만들었습니다. New Relic AI는 간단한 언어로 질문함으로써 스택의 복잡성을 이해하고, 서비스 중단을 해결하고, 효율성을 향상시키는 데 도움이 됩니다.

## New Relic AI와 상호작용하는 방법 [#work-with-NRAI]

AI 채팅, NRQL 콘솔 등 빠르고 쉬운 질문 방법을 제공하여 시스템의 핵심에 접근할 수 있도록 도와드립니다.

### AI에게 질문하기 [#ask-AI]

대화형 채팅 경험을 통해 언제든지 AI와 작업할 수 있습니다. 여러분이 해야 할 일은 대부분의 New Relic 플랫폼 페이지 오른쪽 상단에 있는 **Ask AI** 버튼을 클릭하는 것뿐입니다.

New Relic AI는 다음과 같은 다양한 기능을 제공합니다.

* 자연어를 쿼리로 변환(NRQL/Nerdgraph)
* 이상 감지
* 오류 로그 분석
* 스택 추적 이해
* 문서를 활용하여 New Relic 플랫폼 개념을 이해하세요.

우리의 AI는 일반적인 질문과 쉬운 언어를 사용하여 엔지니어와 소통하도록 훈련되었습니다. 온라인 상점을 운영하고 있는데 `Checkout-service` 의 응답 시간이 지연되고 있다는 알림을 받기 시작한다고 가정해 보겠습니다. 문제를 파헤칠 시간이 없다면 New Relic AI에게 "무슨 일이 일어나고 있나요?"라고 물어보면 됩니다. 혹은 "오늘은 뭘 먼저 처리해야 하지?" New Relic AI는 시스템에서 무엇이 손상되었는지 알려주고 차트와 표를 통해 데이터를 탐색하는 데 도움을 줄 수 있습니다.

### NRQL 쿼리 생성에 대한 도움 받기 [#NRQL-console]

[NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/) 은 시스템에 대한 통찰력을 얻는 데 사용할 수 있는 강력한 도구이지만, 바쁜 경우 필요한 데이터를 얻기 위해 올바른 쿼리를 만들 시간이 없을 수도 있습니다. 쿼리를 실행하는 데 어려움을 겪고 있다면 AI에게 도움을 요청하세요. "이 서비스는 어떻게 작동하나요?"와 같은 질문을 할 수 있습니다. 또는 "지난 24시간 동안 이 서비스의 성능은 어떻습니까?"

NRQL 콘솔에서 NRAI를 활성화하려면:

1. New Relic UI 좌측 하단의 사용자 메뉴로 이동합니다.
2. **Features** \[기능] 탭을 클릭합니다.
3. **New Relic AI** 섹션까지 아래로 스크롤하고 **Enable** \[활성화 를] 클릭합니다.

### 오류 로그 분석에 대한 도움 받기 [#errors]

오류 로그 줄은 구문 분석하기 어려울 수 있습니다. New Relic AI는 오류에 대해 자세히 알아보고, 관련 정보를 쿼리하고, 다음 단계에 대해 질문하는 데 도움이 됩니다.

<img
  title="explain this error"
  alt="A screenshot depicting the explain this error feature in New Relic Logs"
  src={aiNewRelicAiExplainThisError}
/>

<figcaption>
  AI에게 로그 오류에 대해 문의하려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities)** > **Logs** > **All logs** >로 이동하여 `errors` 을(를) 검색하세요.
</figcaption>

로그 오류에 대해 New Relic AI와 채팅하려면:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities)** > **Logs** 로 이동합니다.
2. **All logs** \[모든 로그 를] 클릭합니다.
3. `errors` 을(를) 검색하세요.
4. 결과 로그 줄을 클릭하고 로그 세부 정보 패널 상단에서 **Explain this error** \[이 오류 설명] 버튼을 찾으세요. AI가 오류 요약을 제공합니다.

AI에게 스택에 대해 알리고 후속 질문을 하여 추가 조사를 진행할 수 있습니다. 예를 들어 다음과 같이 질문할 수 있습니다.

* 이 오류는 얼마나 자주 발생합니까? 즉, 오류율은 얼마입니까? 1%인가요, 50%인가요?
* 어떤 호스트에서 이러한 오류가 발생합니까? 내 호스트 전체에 영향을 미치는지, 아니면 호스트 중 하나 또는 두 개에만 영향을 미치는지 알고 싶습니다.

AI를 사용하여 스택 추적을 분석하면 컨텍스트 전환이 줄어들고 분류 및 분석 속도가 빨라집니다.

New Relic AI를 사용하여 오류 받은 편지함의 스택 추적을 조사하려면:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities)** > **Errors inbox** 으로 이동합니다.
2. 워크로드를 선택하세요.
3. 로그 세부정보 패널 상단에서 **Ask AI** \[AI에게 질문하기] 버튼을 찾아보세요. AI가 오류 요약을 제공합니다.

AI에게 스택에 대해 알리고 후속 질문을 하여 추가 조사를 진행할 수 있습니다. 예를 들어 다음과 같이 말할 수 있습니다.

```
Hi, I have an Azure environment and I am having issues with Redis being periodically unavailable. Can you tell me more about this error? How can I diagnose this issue? 

```

## 데이터 보안

데이터 보안과 개인 정보 보호는 소프트웨어를 구축하고 유지 관리하는 모든 사람의 주요 관심사입니다. 우리는 귀하가 AI에게 요청하는 모든 것이 안전하다는 것을 알려드리고 싶습니다. NRAI는 New Relic 고객 데이터를 보호하는 것과 동일한 보안 제어 및 표준으로 구축되었습니다. 우리는 귀하의 데이터를 다른 계정이나 고객과 절대 공유하지 않습니다. 우리는 개인 정보 보호 및 보안 표준을 충족하는 공급업체와 협력하고 서비스를 제공하기 전에 이러한 중요한 영역의 잠재 공급업체를 신중하게 평가합니다.

우리의 AI는 다양한 입력을 사용하여 응답을 개선하고 항상 최상의 답변을 제공합니다. 예를 들어 제품 설명서, 대규모 예제 쿼리 라이브러리, 원격 측정 데이터와 같은 입력을 사용합니다. 그러나 AI가 필요에 따라 데이터를 스캔하여 컨텍스트를 제공하고 차트를 작성하고 응답 시간을 이해하는 데 도움을 주더라도 해당 데이터는 New Relic에 _저장되지 않습니다_ .

## 항상 배우는 중

New Relic AI는 사용할수록 더욱 정확해지고 반응성이 향상됩니다. 우리는 당신이 원하는 AI 경험을 구축하고 싶습니다. AI와 채팅할 때 좋아요 또는 싫어요를 눌러주세요. 올바르지 않거나 개선할 수 있는 사항을 발견하면 알려주시기 바랍니다. 사용자 메뉴로 이동하여 **Help** \[도움말 을] 클릭한 다음 **Give us feedback** \[피드백 제공 을] 클릭하세요.

또한 New Relic AI는 여러 언어를 인식하고 응답할 수 있습니다. 다국어 지원 사용 사례가 있는 경우 AI로 테스트하고 피드백을 제공해 주세요.

New Relic AI 응답은 AI가 생성한 제안입니다. 정보와 응답을 최신 상태로 유지하기 위해 노력하고 있지만 제안이 항상 정확하거나 완전하거나 오류가 없는 것은 아닙니다. 어떤 조치를 취하기 전에 다른 소스를 사용하여 AI가 제공한 정보를 사람이 확인하도록 권장하고 싶습니다. AI의 제안은 정확성에 대한 어떠한 보증이나 보장 없이 제공됩니다.
