서비스 맵은 아키텍처를 시각적으로 사용자 정의할 수 있는 표현입니다. 지도는 애플리케이션, 데이터베이스, 호스트, 서버 및 out-of-process 서비스를 포함하여 앱의 연결 및 종속성을 자동으로 표시합니다.

<Callout variant="tip">
  New Relic UI에서는 out-of-process 서비스를 [웹 외부](/docs/using-new-relic/welcome-new-relic/get-started/glossary#web-external) 또는 백그라운드 외부 데이터라고 합니다.
</Callout>

상태 표시기 및 성능 메트릭은 아키텍처의 모든 부분에 대한 현재 운영 상태를 보여줍니다.

서비스 맵은 다음과 같은 이점을 제공합니다.

* 아키텍처의 앱과 서비스가 연결되고 통신하는 방식을 이해합니다.
* 전체 환경의 현재 상태 및 운영 상태를 빠르게 확인합니다.
* 운영 문제를 해결하고 호스트 및 인스턴스 수준까지 문제의 영향을 이해합니다.
* 아키텍처에 대한 공동의 이해를 공유하고 추진합니다.

## 운영 문제 식별 [#identify-issues]

서비스 맵은 사용자 환경의 현재 상태를 빠르게 볼 수 있도록 [색상으로 구분](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one#alerts) 됩니다. 맵에서 노드를 선택하여 추가 성능 메트릭과 각 노드 연결의 전체 목록 및 상태 확인을 봅니다.

맵은 노드를 맵에 자동으로 연결하므로 맵의 어떤 앱이 다른 앱에 연결되는지 확인할 수 있습니다. 이렇게 하면 호출 응용 프로그램과 특정 데이터베이스 간의 성능 문제의 영향을 해결하고 평가하는 데 도움이 됩니다.

서비스 맵을 사용하면 업스트림 및 다운스트림 종속성의 컨텍스트에서 서비스와 해당 상태를 볼 수 있습니다. 사고 문제를 해결할 때 지도를 사용하여 근본 원인을 식별합니다.

다음 주제에 대한 자세한 내용은 [서비스 맵 사용 방법을](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one) 확인하세요.

* 설정 없이 모든 엔터티 보기: 대부분의 엔터티는 서비스 맵의 종속 항목에 자동으로 연결됩니다.
* 조직의 계정 전체에서 엔터티를 봅니다.
* 대시보드에 맵을 추가합니다.
