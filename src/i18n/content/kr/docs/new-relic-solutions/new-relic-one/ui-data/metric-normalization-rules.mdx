애플리케이션이 그룹에서 더 잘 관리할 수 있는 많은 개별 메트릭을 보내는 경우가 있을 수 있습니다. 이들 중 대부분은 URL에서 명명된 웹 트랜잭션 메트릭에서 발생합니다. 이 문제에 대한 설명은 [메트릭 그룹화 문제](/docs/using-new-relic/cross-product-functions/troubleshooting/metric-grouping-issues) (MGI)를 참조하세요.

높은 카디널리티를 줄이고 메트릭 그룹화 문제를 방지하기 위해 New Relic은 메트릭 타임슬라이스 데이터(정규화)를 그룹화하거나 필터링하는 규칙을 지원합니다. 경우에 따라 메트릭 수가 매우 많은 메트릭 그룹화 문제가 감지되면 성능 저하로부터 플랫폼을 보호하기 위해 규칙이 자동으로 생성됩니다.

## 요구 사항 [#requirements]

측정항목 그룹화 문제 및 측정항목 정규화는 Google에서 보고하는 [측정항목 타임슬라이스 데이터](/docs/data-apis/understand-data/new-relic-data-types/#timeslice-data)에만 적용됩니다. <InlinePopover type="apm"/>에이전트, 모바일 에이전트 및 브라우저 에이전트.

## 메트릭 정규화 규칙 관리 [#create-rules]

[엔터티 탐색기](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts#find) 의 각 서비스 대시보드에서 액세스할 수 있는 메트릭 정규화 도구를 사용하여 높은 카디널리티 메트릭에서 생성된 노이즈를 처리하는 새 규칙을 만들고 관리할 수 있습니다. 그런 다음 왼쪽 사이드바에서 **Metric normalization**을 선택합니다.

여기에서 기존 규칙을 보거나 새 규칙을 만들 수 있습니다. 규칙을 클릭하여 수정하거나 **새 규칙 만들기를 클릭하여 새 규칙** 을 만듭니다. 규칙을 구성하기 위한 새 창이 표시됩니다.

사용 가능한 필드는 다음과 같습니다.

* **순서** : 규칙의 우선 순위를 결정합니다. 숫자가 낮을수록 우선 순위가 높습니다.

* **표현식 일치** : 규칙에 포함할 모든 메트릭을 그룹화하는 정규식을 입력합니다.

* **일치** : 여기에서 위의 정규식과 일치하는 측정항목의 미리보기를 볼 수 있습니다.

* **Action** : 메트릭에 대해 수행하려는 작업입니다.

  * **바꾸기** : 일치하는 메트릭을 정규 표현식으로 **대체** 필드에 설명된 값으로 바꿉니다.
  * **Ignore** : 정규식과 일치하는 메트릭을 무시합니다.
  * **새로운 메트릭 거부** : 이미 보고된 메트릭만 작성하고 정규식과 일치하는 메트릭은 무시합니다.

* **교체** : **교체** 가 활성화된 경우에만 활성화됩니다. 일치하는 메트릭은 필드 값으로 대체됩니다. 정규식이 그룹을 캡처하는 경우 그룹 `1` 및 `2` 에 대해 각각 `\1` 또는 `\2` 와 함께 자리 표시자를 사용할 수 있습니다.

* **활성** : 규칙을 삭제할 수 없지만 비활성화할 수 있습니다. 규칙을 활성화하거나 비활성화하려면 토글을 클릭합니다. 규칙을 제거하려면 New Relic의 지원팀에 문의하세요.

* **종료** : 활성화된 경우 연결된 패턴이 일치할 때 규칙 폭포수가 종료됩니다.

* **참고** : 규칙에 대한 내부 참고사항입니다. 규칙에 영향을 미치지 않습니다.

필드를 설정한 후 **만들기** (또는 기존 규칙을 **편집** 하는 경우 편집)를 클릭하면 **활성** 상태인 한 규칙이 즉시 적용됩니다.
