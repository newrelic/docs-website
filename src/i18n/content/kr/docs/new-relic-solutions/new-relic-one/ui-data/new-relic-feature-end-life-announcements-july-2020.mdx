New Relic 경험을 개선하기 위해 플랫폼을 크게 개선했습니다. 이에 따라 앞으로 몇 주 동안 일부 이전 기능에 대한 지원을 종료할 예정입니다. 많은 제품이 이미 "수명 종료"로 공개적으로 발표되었습니다. 이러한 변경은 당사 플랫폼과의 상호 작용을 개선하기 위한 당사의 약속을 나타냅니다.

우리의 임무는 원격 측정 데이터를 통합하고 전체 스택을 연결하며 보다 완벽한 소프트웨어를 구축하는 데 도움이 되는 하나의 관측 가능성 플랫폼입니다. 이를 위해 우리는 문제 해결, 경고 및 데이터 시각화 경험을 단순화하는 데 중점을 두고 있습니다.

다음은 동일한 목표를 달성하기 위해 새로운 기능을 활용하는 방법에 대한 몇 가지 유용한 링크 및 가이드와 함께 변경된 사항에 대한 자세한 내용입니다. 항상 그렇듯이 저희가 도와드리겠습니다. [Explorers Hub](https://discuss.newrelic.com/t/new-relic-feature-end-of-life-announcements/105772/) 에서 연락하거나 질문이 있는 경우 계정 팀에 문의하세요.

## 이전 Kubernetes 통합 에이전트 버전 [#kubernetes]

통합 환경을 개선하기 위해 2020년 8월 12일 수요일부터 v1.7 이상을 사용하는 Kubernetes 통합이 더 이상 사용되지 않습니다. `infrastructure.newrelic.com` 도메인의 UI 환경은 더 이상 사용할 수 없습니다.

* 이미 [최신 Kubernetes 에이전트 버전](/docs/release-notes/platform-release-notes/kubernetes-integration-release-notes) 을 사용 중인 경우 조치가 필요하지 않습니다.
* v1.7 또는 이전 버전을 사용하는 경우 Kubernetes 성능 데이터를 계속 보려면 통합을 업데이트해야 합니다.

설명서의 지침에 따라 최신 버전의 Kubernetes 통합으로 업그레이드하십시오.

**<Icon style={{color: 'green'}} name="fe-info"/>성공적인 전환을 위한 조치 항목:**

1. [Kubernetes 에이전트 지원 중단 알림](/docs/release-notes/platform-release-notes/kubernetes-integration-release-notes/deprecation-notice-kubernetes-v17-or-lower) 을 검토하세요.
2. 표준 절차에 [따라 Kubernetes 에이전트를 최신 버전으로 업그레이드하십시오](/docs/integrations/kubernetes-integration/installation/kubernetes-installation-configuration#update) .
3. [Kubernetes 클러스터 탐색기 UI](/docs/integrations/kubernetes-integration/understand-use-data/kubernetes-cluster-explorer) 에 대해 알아보세요.

자세한 내용은 [Explorers Hub 게시물](https://discuss.newrelic.com/t/important-end-of-life-kubernetes-integrations-v1-7-or-older/102167) 을 참조하세요.

## Synthetics의 "롤업 기준" [#rollup]

Synthetics는 복잡한 소프트웨어 시스템을 관찰할 수 있는 단일 플랫폼을 제공하기 위해 개선된 New Relic 플랫폼으로 이동하고 있습니다. 이를 위해 서비스, 애플리케이션 및 모니터에 태그를 지정(또는 레이블 지정)하는 데 사용하는 도구를 통합하고 있습니다. 단종된 특정 기능은 **검색 모니터** 드롭다운 메뉴 아래의 **롤업** 기준 기능입니다.

별도의 **Rollup by** 드롭다운 없이 New Relic에서 태그별로 모니터를 그룹화할 수 있습니다. 새로운 합성 모니터 목록 페이지를 사용하기 위해 변경할 필요는 없습니다. 새로운 경험은 2020년 7월 20일 이후에 사용할 수 있습니다.

선적 서류 비치:

* [New Relic의 태그 지정은](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) [Synthetics 모니터의 레이블을](/docs/using-new-relic/user-interface-functions/organize-your-data/labels-categories-organize-apps-monitors) 대체합니다.
* [New Relic의 Explorer](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer-view-performance-across-apps-services-hosts) 는 [Synthetics 모니터 인덱스](/docs/synthetics/new-relic-synthetics/pages/synthetics-monitors-index-access-your-monitors) 를 대체합니다.

**<Icon style={{color: 'green'}} name="fe-info"/>성공적인 전환을 위한 조치 항목:**

현재 합성 롤업을 사용하는 경우:

만료되는 롤업을 다시 만들려면 Explorer Hub 게시물의 절차를 따르세요.

1. 롤업 EOL에 대한 [Explorers Hub 게시물](https://discuss.newrelic.com/t/end-of-life-notice-synthetics-labels-and-synthetics-apm-group-by-tag/103781) 로 이동합니다. 특히 **New Relic Workloads가 애플리케이션 및 모니터 그룹을 시각화하는 데 어떻게 도움이 됩니까?를 찾으** 십시오. 게시물의 **FAQ** 섹션에서.
2. Explorers Hub 게시물의 절차에 따라 사용 중지되는 롤업을 다시 만드세요.

## 포함된 차트

중복성을 줄이고 보다 통합된 경험을 제공하기 위해 [포함된 차트](/docs/apm/apm-ui-pages/features/embed-charts-external-webpages) 기능이 New Relic **Get 차트** 로 대체됩니다. 현재 사용 중이고 New Relic 도메인 외부에서 호스팅되는 포함된 차트는 계속 작동합니다. 변경 사항에는 다음이 포함됩니다.

* 차트를 생성하는 차트 메뉴의 이름은 **포함** 에서 **차트** 가져오기 링크로 변경됩니다.
* 더 이상 지원되지 않는 차트의 경우 **포함** **이 이 차트에 대해 차트 가져오기 링크가 지원되지 않음** 으로 대체됩니다.
* 포함된 모든 차트 링크를 나열하는 APM UI 페이지는 더 이상 사용할 수 없습니다.

선적 서류 비치:

* [탐험가 허브 게시물](https://discuss.newrelic.com/t/change-to-shareable-charts-in-apm-embedded-charts/102240)
* New Relic의 [향상된 쿼리 빌더 기능](/docs/chart-builder/use-chart-builder/use-charts/use-your-charts#get-chart-link) 은 [EOL 내장 차트 기능](/docs/apm/apm-ui-pages/features/embed-charts-external-webpages) 을 대체합니다.

**<Icon style={{color: 'green'}} name="fe-info"/>성공적인 전환을 위한 조치 항목:**

공개적으로 액세스 가능한 링크를 생성하여 내부 및 외부 웹 사이트에 추가할 수 있습니다. 차트 링크를 보기 위해 사용자는 New Relic에 로그인할 필요가 없습니다. [차트를 포함하는](/docs/chart-builder/use-chart-builder/use-charts/use-your-charts#get-chart-link) 기능은 유지되지만 **포함** (UI에서 제거됨)을 클릭하는 대신 차트 가져오기 **링크** (같은 위치에 있음)를 클릭합니다. 이를 통해 우리가 중단하는 것보다 훨씬 더 광범위한 차트 비주얼 및 동작으로 링크를 얻을 수 있습니다.

* 포함된 차트가 있고 **이 차트에 대해 차트 가져오기 링크가 지원되지 않는다는** 메시지가 표시되면 다른 쿼리를 실행하고 사용 가능한 차트 유형을 선택한 다음 **포함** 을 선택합니다.

## 연결된 에이전트 페이지 [#connected-agents]

연결된 에이전트를 찾는 것이 그 어느 때보다 쉬워졌습니다! **데이터 쿼리** 링크를 통해 이미 이 작업을 수행할 수 있습니다. [이 `NrDailyUsage` 쿼리](https://discuss.newrelic.com/t/replicate-the-connected-agents-report-in-insight/77575) 를 실행하여 연결된 에이전트 및 호스트 목록을 가져오기만 하면 됩니다. 이전 **연결된 상담원** 페이지는 단종되어 제거됩니다.

선적 서류 비치:

* [탐험가 허브 게시물](https://discuss.newrelic.com/t/replicate-the-connected-agents-report-in-insight/77575)
* [NRQL 쿼리 도구](/docs/query-your-data/nrql-new-relic-query-language/query-tools)
* [NrDailyUsage 데이터 정의](/attribute-dictionary/?event=NrDailyUsage)

**<Icon style={{color: 'green'}} name="fe-info"/>성공적인 전환을 위한 조치 항목:**

연결된 앱 및 호스트를 쿼리하려면 쿼리 빌더 또는 GraphQL API와 같은 기존 [NRQL 쿼리 도구](/docs/query-your-data/nrql-new-relic-query-language/query-tools) 를 사용할 수 있습니다.

**권장 사항:** 앱 및 호스트에 대한 포괄적인 데이터를 탐색하는 최상의 결과를 얻으려면[쿼리 빌더](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder#specify-data) 또는 [NerdGraph API](/docs/apis/nerdgraph/examples/nerdgraph-nrql-tutorial) 를 사용하십시오.

## 비활성 앱 [#inactive-apps]

2020년 6월 8일부터 93일 동안 데이터를 보고하지 않은 APM 애플리케이션은 계속 표시되지 않습니다. 게시된 APM [데이터 보존 지침](/docs/accounts/accounts/data-management/overview-data-retention-components#apm) 에 따라 데이터를 보고하지 않은 애플리케이션은 New Relic UI에서 90일 동안 사용할 수 있습니다.

90일이 지나면 해당 애플리케이션이 UI에서 제거됩니다. 그러나 주요 지표는 구독 수준에 따라 [New Relic REST API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2) 를 통해 계속 사용할 수 있습니다. 이것이 애플리케이션이 [REST API Explorer](/docs/apis/rest-api-v2/api-explorer-v2/introduction-new-relics-rest-api-explorer) 를 통해 영구적으로 삭제될 때까지 애플리케이션 이름이 예약된 상태로 유지되는 이유입니다.

**<Icon style={{color: 'green'}} name="fe-info"/>성공적인 전환을 위한 조치 항목:**

더 이상 데이터를 보고하지 않는 이전 앱에 대해 앱 이름을 재사용하려는 경우:

* [REST API v2](/docs/apis/rest-api-v2/application-examples-v2/summary-data-examples-v2) 또는 [API 탐색기](https://api.newrelic.com/) 를 사용하여 앱 ID, 이름, 상태(회색 찾기), 마지막으로 보고된 등을 포함한 요약 데이터를 가져옵니다.
* 앱 [에이전트를 제거합니다](/docs/using-new-relic/cross-product-functions/install-configure/uninstall-agent) .
* REST API 또는 API 탐색기를 사용하여 앱을 삭제합니다.

## 종합 모니터 경고 알림 및 조건 [#synthetics-alerts]

통합된 경험을 제공하기 위해 Synthetics 모니터 [경고 알림](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics) 및 [조건 위반](/docs/synthetics/new-relic-synthetics/pages/view-monitors-alert-information) 을 더 이상 사용하지 않습니다. 이 페이지를 새로운 Synthetics 모니터 개요 환경으로 교체합니다. 이 새로운 경험은 단일 보기에서 모니터 결과와 함께 모니터의 열린 위반 및 경고 조건에 대한 가시성을 제공하므로 위반 또는 경고 조건을 보기 위해 여러 탭을 열 필요가 없습니다.

**<Icon style={{color: 'green'}} name="fe-info"/>성공적인 전환을 위한 조치 항목:**

필요한 조치는 없지만 다음을 포함하는 [New Relic 응용 인텔리전스](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai) 에 대한 설명서를 검토하여 새로운 UI 환경을 준비할 수 있습니다.

* 경고 등에서 이미 익숙한 경고 기능
* 소음을 줄이고 사고 해결 프로세스를 가속화하는 사고 인텔리전스
* 관련 문제를 표시하고 원인에 대한 자동 설명을 제공하는 사전 감지
