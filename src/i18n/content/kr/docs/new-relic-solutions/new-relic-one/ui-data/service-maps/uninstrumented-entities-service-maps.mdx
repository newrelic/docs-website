New Relic은 원격 측정을 분석하여 계측의 공백을 감지할 수 있습니다. 이러한 격차는 데이터베이스, 앱 또는 기본적으로 다른 유형의 엔터티일 수 있습니다.

예를 들어, 한 애플리케이션은 계측되지 않은 Amazon RDS 데이터베이스에 정보를 저장할 수 있습니다. 그런 일이 발생하면 계측의 데이터 공백을 반영하기 위해 "계측되지 않은" 엔터티 형태로 표시됩니다. [서비스 맵](/docs/new-relic-solutions/new-relic-one/ui-data/service-maps/introduction-service-maps) 에서 계측되지 않은 엔터티와 계측된 엔터티(이 예에서는 `mysql.demo.ep.rds.com` 간의 관계를 볼 수 있습니다.

계측되지 않은 엔터티와 계측된 엔터티는 모양이 다릅니다. 계측되지 않은 엔터티에는 점선 테두리가 있고 계측된 엔터티에는 단색 테두리가 있습니다.

## 엔터티를 수동으로 계측 [#instrument]

서비스 맵에서 계측되지 않은 엔터티를 계측하려면 다음 안내를 따르세요.

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 으로 이동, 그런 다음 **APM 및 서비스** , **모바일** 또는 **브라우저** 를 선택합니다.
2. 엔터티를 선택합니다.
3. 왼쪽에서 **Service map** \[서비스 맵 을] 클릭합니다.
4. **Instrument what's missing** \[누락된 항목 계측을] 클릭하고 각 엔터티의 안내 UI 지침을 따릅니다.
