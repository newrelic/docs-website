대부분의 경우 `nr1` CLI 명령은 추가 설정이 필요 없이 바로 작동합니다. 문제 없이 Nerdpack, Nerdlet, 실행 프로그램 및 시각화를 생성할 수 있습니다. 하지만 때로는 설정에 특별한 것이 필요할 수도 있습니다.

이 가이드에서는 Nerdpack에 대한 고급 설정을 활성화하는 방법을 배우게 됩니다.

## 시작하기 전에

`nr1` CLI가 최신 버전인지 확인하세요.

```sh
nr1 update
```

## 웹팩 설정 확장

설정을 위해 웹팩을 확장하려면 `.extended-webpackrc.js` 파일을 사용할 수 있습니다. 이를 통해 앱을 로컬에서 제공하거나 구현하거나 배포할 때 자신만의 별칭, 펼치기 및 설정을 추가할 수 있습니다.

## 추가 인증서 추가

회사 프록시 뒤에서 `nr1` 으로 작업할 때 `RequestError` 이 표시될 수 있습니다.

```sh
nr1 create --type nerdpack --name new-nerdpack
[output] {error}RequestError: Error:{normal} self signed certificate in certificate chain
```

노드는 자체 인증서 체인을 사용합니다. 따라서 이 오류가 발생하는 경우 Node의 "루트" CA에 회사 인증서를 추가해야 합니다.

```sh
export NODE_EXTRA_CA_CERTS=[cert]
```
