뉴렐릭 플랫폼에서 사용하려면 Nerdpack을 구독하세요. 더 이상 사용하지 않는 Nerdpack 구독을 취소하세요.

## CLI 업데이트

명령을 실행하기 전에 최신 버전의 CLI가 있는지 확인하십시오.

```shell
nr1 update
```

## 권한을 확인하세요

Nerdpack을 구독하려면 다음이 필요합니다.

* 게시한 계정에 액세스
* [Nerdpack 관리에 필요한 권한](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps)

## Nerdpack 게시

Nerdpack을 구독하려면 먼저 생성한 Nerdpack을 게시해야 합니다.

1. [Nerdpack 게시](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish)
2. [Nerdpack이 게시되었지만 아직 태그가 없는 경우 태그를 지정하세요.](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag)

## Nerdpack을 구독하세요

웹 UI 또는 `nr1` CLI를 통해 게시된 Nerdpack을 구독하여 계정에서 사용하세요. 구독하면 사용하려는 태그 버전과 구독하려는 계정을 선택할 수 있습니다. 이를 통해 귀하의 계정에서 사용하는 Nerdpack을 제어할 수 있습니다.

<Callout variant="tip">
  구독하신 태그가 업데이트되면 뉴렐릭이 Nerdpack을 자동으로 업데이트합니다.
</Callout>

<Steps>
  <Step>
    [뉴렐릭](https://one.newrelic.com) 홈페이지에서 상단 탐색 메뉴의 <DNT>**+ Integrations & Agents**</DNT> 으로 이동하세요.
  </Step>

  <Step>
    앱 필터나 검색창을 사용하여 게시된 Nerdpack을 찾으세요. 게시된 모든 Nerdpack이 여기에 표시됩니다. 그러나 찾고 있는 항목이 필터링되었을 수 있습니다. 필터를 조정하여 원하는 Nerdpack을 찾으세요.
  </Step>

  <Step>
    뉴렐릭 계정에 추가하려는 Nerdpack을 선택하세요.
  </Step>

  <Step>
    **Add this app** \[이 앱 추가 를] 클릭합니다. 이미 Nerdpack을 구독하고 있는 경우 이 버튼에는 **Manage access** \[액세스 관리가] 표시됩니다.
  </Step>

  <Step>
    구독할 계정과 구독할 태그 버전을 선택하세요. 그런 다음 계정을 업데이트하세요.

    <Callout variant="tip">
      웹 UI와 CLI의 일부 위치에서는 **tags** \[태그를] **channels** \[채널] 이라고 합니다. 이는 Nerdpack 버전 관리의 과거 구현의 유물입니다. 기록 및 구현 세부정보를 무시하고 동의어로 간주합니다.
    </Callout>

    이제 귀하와 귀하가 구독한 계정의 구성원은 뉴렐릭에서 Nerdpack을 설치할 수 있습니다.
  </Step>

  <Step>
    앱을 구독했다면 **Your apps** \[내 앱] 아래에서 찾으세요. 시각화를 구독한 경우 **Custom visualizations** \[사용자 정의 시각화] 아래에서 찾으세요.
  </Step>
</Steps>

## 명령줄에서 구독

자신만의 Nerdpack을 게시한 경우 `nr1` CLI를 사용하여 Nerdpack을 구독하는 것이 더 빠를 수 있습니다.

<Steps>
  <Step>
    Nerdpack의 루트 디렉터리로 이동합니다:

    ```shell
    cd my-awesome-nerdpack
    ```

    이 지침에서는 Nerdpack을 `my-awesome-nerdpack` 이라고 합니다.
  </Step>

  <Step>
    Nerdpack이 게시되고 태그가 지정되었는지 확인하세요.

    ```shell
    nr1 nerdpack:info
    [output] Id: {purple}ab123c45-678d-9012-efg3-45hi6jkl7890
    [output] Region: {muted}us
    [output] Account ID: 1234567
    [output] Local version: 1.0.0
    [output] Subscription Model: OWNER_AND_ALLOWED
    [output] Version Count: 1
    [output]
    [output] Version  Date               Tags
    [output] -------  -----------------  ------
    [output] 1.0.0    {muted}a few seconds ago  {green}STABLE
    ```

    태그가 있는 [게시된](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish) 버전이 하나 [이상 표시되어야](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) 합니다. `STABLE` 일 필요는 없습니다. 태그가 지정된 모든 버전을 구독할 수 있습니다.
  </Step>

  <Step>
    Nerdpack을 구독하세요:

    ```shell
    nr1 nerdpack:subscribe --channel=STABLE
    [output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}STABLE{normal} channel
    ```

    여기서는 `STABLE` 태그를 구독했습니다. CLI에서 `DEV` 및 `BETA` 태그를 구독할 수도 있지만 구독 명령은 임의 태그를 허용하지 않습니다. 다른 태그된 버전을 구독하려면 웹 UI를 사용해야 합니다.

    웹 UI와 유사하게 **channel** \[채널] `nr1 nerdpack:subscribe` 이라는 용어는 Nerdpack 버전의 이전 구현을 참조합니다.

    <Callout
      variant="tip"
      title="기술적인 세부사항"
    >
      `nr1 nerdpack:subscribe` 은(는 `nr1 subscription:set` 의 별칭입니다.

      자세히 알아보려면 [`nr1 subscription:set`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionset) 문서를 읽어보세요. 또는 `nr1` 도움말 페이지를 확인하세요.

      ```shell
      nr1 subscription:set --help
      ```
    </Callout>
  </Step>
</Steps>

## Nerdpack 구독 취소

Nerdpack 구독을 취소하여 계정에서 제거하세요.

<Steps>
  <Step>
    [뉴렐릭](https://one.newrelic.com) 에서 상단 탐색 메뉴의 <DNT>**Apps**</DNT> 으로 이동하세요.
  </Step>

  <Step>
    구독 중인 Nerdpack 옆에 있는 "i" 아이콘을 클릭하세요. 그러면 세부정보 페이지가 열립니다.
  </Step>

  <Step>
    **Manage access** \[액세스 관리 를] 클릭합니다.
  </Step>

  <Step>
    구독을 취소하려는 계정을 선택 취소하세요. 그런 다음 해당 계정을 업데이트하세요.
  </Step>
</Steps>

## 명령줄을 사용하여 구독 취소

Nerdpacks에서 CLI 프로필 구독을 빠르게 취소하세요.

<Steps>
  <Step>
    Nerdpack의 루트 디렉터리로 이동합니다:

    ```shell
    cd my-awesome-nerdpack
    ```
  </Step>

  <Step>
    프로필 구독 보기:

    ```shell
    nr1 subscription:list
    [output] {muted}Listing subscribed deployed nerdpacks on your account:
    [output] Version  UUID                                  Name              Date         User
    [output] -------  ------------------------------------  ----------------  -----------  -------
    [output] 0.6.4    {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal}  Account Maturity  {muted}a month ago{normal}  2774472
    ```
  </Step>

  <Step>
    UUID를 사용하여 Nerdpack 구독을 취소하세요.

    ```shell
    nr1 nerdpack:unsubscribe --nerdpack-id=384de6e5-8e57-4d56-9d13-299ee7ae8641
    [output] Unsubscribed account {green}1234567{normal} from the nerdpack {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal} on the {green}STABLE{normal} channel.
    ```

    <Callout variant="tip">
      `nr1 nerdpack:unsubscribe` 은(는 `nr1 subscription:unset` 의 별칭입니다.

      자세히 알아보려면 [`nr1 subscription:unset`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionunset) 문서를 읽어보세요. 또는 `nr1` 도움말 페이지를 확인하세요.

      ```sh
      nr1 subscription:unset --help
      ```
    </Callout>
  </Step>
</Steps>

## 게시된 Nerdpack 액세스 관련 문제 해결

때로는 Nerdpack을 구독할 때 문제가 발생할 수 있습니다. 구독할 수 없거나 계정에 중복 구독이 표시되는 경우 이러한 문제를 해결하기 위한 몇 가지 일반적인 솔루션을 고려해 보세요.

### Nerdpack 버전에 태그를 지정하세요.

Nerdpack을 구독할 때 특정 태그를 선택합니다. 따라서 구독하려는 Nerdpack이 태그가 아니거나 타겟팅하는 태그가 없는 경우 릴리스 버전을 [태그하세요](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) .

```shell
nr1 nerdpack:subscribe --channel=DEV
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}DEV{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
[output]
nr1 nerdpack:tag --tag=DEV
[output] {success}✔  Tagged {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} version {green}1.0.0{normal} as {green}DEV{normal}.
[output]
nr1 nerdpack:subscribe --channel=DEV
[output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}DEV{normal} channel.
```

`--channel` 인수 없이 구독하는 경우 기본 채널은 `STABLE` 이므로 이 경우 `STABLE` 버전이 있는지 확인하세요.

```shell
nr1 nerdpack:subscribe
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}STABLE{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
```

### 웹 UI로 구독

CLI로 구독할 때 `--channel` 인수를 전달하여 태그를 선택합니다. **channel** \[채널은] 모든 의도와 목적에 있어서 **tag** \[태그] 입니다. 이 용어는 Nerdpack 버전 관리의 과거 구현의 유물입니다.

그러나 용어는 동의어이지만 CLI에서는 `--channel` 사용이 제한됩니다. `--channel` 로 구독하는 경우 `DEV`, `BETA` 및 `STABLE` 중에서만 선택할 수 있습니다. 따라서 다른 태그를 구독하려면 웹 UI를 사용하세요.

```shell
nr1 nerdpack:subscribe --channel=QA
[output] {error}›{normal}   Error: Expected --channel=QA to be one of: {green}DEV{normal}, {green}BETA{normal}, {green}STABLE
[output] {error}›{normal}   See more help with --help
[output] {error}›{normal}   Code: UNKNOWN
```
