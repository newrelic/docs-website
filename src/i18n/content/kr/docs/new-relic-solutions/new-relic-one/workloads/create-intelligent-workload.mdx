지능형 워크로드는 분산 추적을 사용하여 트랜잭션 흐름의 모든 엔티티를 자동으로 추적합니다. [표준 워크로드](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads) 와 달리 지능형 워크로드는 분산 추적을 사용하여 엔티티를 검색하고 시스템이 발전함에 따라 5분마다 멤버십을 업데이트합니다.

## 시작하기 전에 [#prereqs]

[지능형 워크로드 요구 사항을](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview#intelligent-workloads) 충족하는지 확인하십시오.

<Callout variant="tip">
  를 처음 사용하는 경우, [개요를](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview) 검토하여 지능형과 표준을 언제 사용해야 하는지 이해하십시오.
</Callout>

## 지능형 워크로드를 생성합니다. [#create]

지능형 워크로드를 생성하는 방법은 세 가지가 있습니다. 본인에게 가장 편리한 방법을 선택하세요.

<CollapserGroup>
  <Collapser id="create-from-recommendations" title="권장 워크로드에서 생성">
    뉴렐릭은 귀하의 거래 거래를 자동으로 분석하고, 가치가 높은 지능형 워크로드를 추천해 드립니다. 이것이 지능형 워크로드를 시작하는 가장 빠른 방법입니다.

    ### 추천 상품 보기

    해당 페이지로 이동하면 뉴렐릭이 추천하는 해당 페이지를 찾았을 경우 상단에 배너가 표시됩니다.

    ```
    We found [X] recommended Intelligent Workloads. We made it easy to review our recommendations and set up Intelligent Workloads quickly.
    ```

    <DNT>**Review recommendations**</DNT> 클릭하여 추천 모달을 여세요.

    ### 검토 후 추천 사항을 기반으로 생성

    **Recommended Intelligent Workloads** \[권장 지능형 워크로드] 모달은 모니터링을 통해 이점을 얻을 수 있는 트랜잭션을 보여줍니다.

    1. **카테고리별 필터링**: 탭을 사용하여 카테고리를 전환하세요.

       * **Key Transactions** \[주요 프로세서]: 트래픽이 많거나 비즈니스에 중요한 트랜잭션
       * **Impacted Transactions** \[영향받는 거래]: 성능 문제 또는 오류가 발생한 거래

    2. **정렬 기준**: 드롭다운 메뉴를 사용하여 다음 기준으로 정렬하세요.

       * 가장 높은 오류율
       * 가장 느린 평균 응답 시간
       * 가장 시간이 많이 걸리는 부분

    3. **프로세서 세부 정보 검토**: 각 권장 사항에는 다음이 표시됩니다.

       * **Transaction name** \[프로세서 이]름: 워크로드의 초점 프로세서
       * **Focal entity** \[초점(Focal)]: 프로세서를 처리하는 기본 서비스입니다.
       * **Response Time** \[응답 시간]: 평균 지연 시간 (예: &quot;0.57s&quot;)
       * **Error Rate** \[오류율]: 실패한 requests 비율 (예: &quot;0.6%&quot;)
       * **Total Duration** \[총 소요 시간]: 처리 시간 합계 (예: &quot;77.60초&quot;)

    4. **생성하기**: 추천 항목 옆에 있는 <DNT>**Create**</DNT> 클릭하세요.

    추천을 기반으로 생성할 때 뉴렐릭은 자동으로 참조 및 참조 선택 항목을 미리 채워주고, **Add details** \[세부 정보 추가] 단계로 바로 이동하여 다음을 수행할 수 있습니다.

    * 워크로드 이름을 지정하세요
    * 설명을 추가하세요 (선택 사항)
    * 초점 서비스 및 프로세서 확인
    * 설정을 완료하세요

    <Callout variant="tip">
      지능형 워크로드를 처음 사용하거나 전용 모니터링을 통해 가장 큰 이점을 얻을 수 있는 트랜잭션을 신속하게 파악하려는 경우, 권장 사항을 기반으로 생성하는 것이 이상적입니다.
    </Callout>
  </Collapser>

  <Collapser id="create-workloads-page" title="워크로드 페이지에서 생성">
    이 스냅샷 페이지는 지능형 스냅샷을 생성하는 데 도움이 되는 안내 설정 마법사를 제공합니다.

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Workloads**</DNT> 으)로 이동합니다.
    2. <DNT>**Create a workload**</DNT> 을(를) 클릭합니다.
    3. <DNT>**Intelligent Workload**</DNT> 유형을 선택하세요.
    4. 마법사를 시작하려면 <DNT>**Setup**</DNT> 클릭하세요.

    <img title="Select Intelligent Workload type" alt="Screenshot showing workload type selection with Standard and Intelligent options" src="/images/create_intelligent_workload.webp" />

    ### 1단계: 엔티티를 선택합니다 [#select-entity]

    귀하의 중요한 프로세서가 포함된 서비스 또는 미팅을 선택하십시오:

    1. 이름으로 해당 항목을 검색하거나 찾아보세요.
    2. 필요한 경우 엔티티 유형별로 필터링하세요(APM 서비스, 브라우저 앱 등).
    3. 귀하의 주요 프로세서를 처리하는 프로세서를 선택하세요.
    4. <DNT>**Next**</DNT> 을(를) 클릭합니다.

    <Callout variant="tip">
      귀하의 프로세서 흐름의 중심이 되는 파티션을 선택하세요. 예를 들어, 결제 과정을 모니터링하는 경우 주변 로깅 서비스 대신 결제 서비스를 선택하십시오.
    </Callout>

    ### 2단계: 프로세서 선택 [#pick-transaction]

    모니터하려는 특정 프로세서를 선택하십시오:

    1. 선택한 대상에 대한 거래 목록을 살펴보세요.
    2. 검색을 사용하여 특정 소니 이름 찾기
    3. 귀하의 비즈니스를 대표하는 데스크탑을 선택하세요
    4. 딸깍 하는 소리 <DNT>**Next**</DNT>

    워크로드는 이 프로세서에 참여하는 모든 행위를 추적트레이스를 기반으로 추적합니다.

    ### 3단계: 세부 정보 추가 [#add-details]

    귀하의 상황에 대한 정보를 제공하십시오:

    1. **Name** \[이]름: 기술 이름을 입력합니다(예: &quot;Checkout 프로세서&quot;, &quot;사용자 인증 흐름&quot;)
    2. **Description** \[설명] (선택 사항): 이 워크로드가 모니터링하는 내용과 그 중요성에 대한 맥락을 추가하십시오.
    3. **Account** \[계정]: 워크로드가 생성될 계정을 선택하세요.
    4. 딸깍 하는 소리 <DNT>**Next**</DNT>

    ### 4단계: 미리 보기 및 최종 확정 [#preview-finalize]

    워크로드 설정을 검토하세요.

    1. **Focal 서비스가** 올바른지 확인하십시오.
    2. **Focal 프로세서가** 의도한 워크플로우와 일치하는지 확인하세요.
    3. **Name** \[이]름 및 **Description** \[설명을]검토하십시오.
    4. 딸깍 하는 소리 <DNT>**Create Workload**</DNT>

    귀하의 Intelligent 워크로드가 프로세서 참여 추적을 즉시 시작합니다.
  </Collapser>

  <Collapser id="create-from-t360" title="트랜잭션 360에서 생성">
    이미 프로세서 360에서 프로세서를 보고 있는 경우 인텔리전트 워크로드로 빠르게 저장할 수 있습니다.

    1. 이동 <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services**</DNT>
    2. 핵심 워크플로의 일부인 서비스를 선택하세요.

    <img title="Select entity for Intelligent Workload" alt="Screenshot showing entity selection step in Intelligent Workload creation" src="/images/intelligent-workload-select-entity.webp" />

    3. 모니터하려는 특정 프로세서로 이동하여 <DNT>**Transaction 360**</DNT> 클릭합니다.
    4. <DNT>**Save as intelligent workload**</DNT> 을(를) 클릭합니다.
    5. 워크로드의 기술 이름을 입력하세요.
    6. <DNT>**Save**</DNT> 을(를) 클릭합니다.

    <Callout variant="tip">
      개별 API 호출보다는 완전한 비즈니스 흐름흐름(예: &quot;checkout&quot;, &quot;사용자-인증&quot;)을 나타내는 데스크탑 이름을 선택하세요.
    </Callout>
  </Collapser>
</CollapserGroup>
