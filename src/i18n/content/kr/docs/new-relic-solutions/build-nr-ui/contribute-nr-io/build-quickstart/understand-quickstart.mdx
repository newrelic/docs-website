<Callout variant="tip">
  이 절차는 퀵스타트를 만드는 방법을 가르치는 과정의 일부입니다. 아직 확인하지 않으셨다면 [과정 소개를](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/contribute-quickstart) 확인해 보세요.

  이 절차는 실습의 마지막 절차 위에 구축되므로 이 절차를 진행하기 전에[대시보드를 생성](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/create-a-dashboard) 하고 [알림을 생성해야](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/create-alerts/) 합니다.
</Callout>

퀵스타트 생성을 진행하기 전에 디렉토리 구조를 이해해야 합니다.

시작하려면 New Relic 빠른 시작의 GitHub로 이동하여 포크하세요. 그런 다음 `git clone` 사용하여 자신의 저장소를 로컬 머신에 복제하고 원하는 IDE에서 엽니다.

복제된 프로젝트의 루트에서 `\_template` 폴더 내의 디렉터리는 리소스의 최상위 수준에 있는 디렉터리를 미러링하며 퀵스타트의 각 옵저버빌리티 빌딩 블록에 대한 템플릿을 포함합니다.

빠른 시작 구조에 대한 최근 업데이트로 인해 이제 `\_template` 디렉터리의 구조는 다음과 같습니다.

```
alert-policies/
    example-alert-policy/
        baseline-alert.yml
        static-alert.yml

dashboards/
    example-dashboard/
        example-dashboard.json
        example-dashboard.png

data-sources/
    example-data-source/
        config.yml
        logo.svg

quickstarts/
    example-quickstart/
        config.yml
        logo.svg
```

각 디렉토리에는 퀵스타트용 엔터티를 생성하는 데 사용할 수 있는 템플릿이 포함되어 있습니다. 새로운 기여의 구조는 여기에 표시된 구조와 일치해야 합니다. 빠른 시작 `config.yml` 에서 관련 대시보드 및 알림을 참조하세요.

퀵스타트를 구축하려면 먼저 대시보드와 같은 엔터티를 추가하고 퀵스타트에 알림을 추가해야 합니다.

<Callout variant="tip">
  이 절차는 퀵스타트를 만드는 방법을 가르치는 과정의 일부입니다. 다음 단원으로 계속 진행하세요. [대시보드를 퀵스타트에 추가하세요](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/build-quickstart/add-dashboard).
</Callout>
