[퀵스타트를](https://opensource.newrelic.com/projects/newrelic/newrelic-quickstarts) 구축하는 과정의 첫 번째 단계는 라이브러리를 익스텐션하거나 델메트리 데이터를 뉴렐릭으로 보내도록 플랫폼을 구성하는 것입니다. 이를 설정하면 제품 사용자가 자신의 계정에서 해당 데이터를 볼 수 있게 됩니다.

사용자가 제품에서 뉴렐릭으로 데이터를 쉽게 보낼 수 있도록 제품을 설정하는 것은 중요합니다. 왜냐하면 제공할 퀵스타트는 대시보드, 알림 또는 제공하는 기타 리소스에서 해당 데이터를 사용하기 때문입니다. 사용자가 데이터를 볼 수 있게 되면 퀵스타트를 사용하여 이를 시각화하고 몇 분 안에 실행 가능한 인사이트를 얻을 수 있습니다.

## 델메트리 데이터 유형 [#telemetry-data-types]

효과적인 퀵스타트를 구축하려면 라이브러리나 플랫폼에서 최소한 한 가지 유형의 텔레메트리 데이터를 뉴렐릭으로 보내야 합니다. 뉴렐릭은 [다음과 같은 4개의 텔레메트리 데이터 유형을](/docs/data-apis/understand-data/new-relic-data-types) 지원합니다.

* [지표](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-metrics): 지표는 시스템의 성능과 동작을 집계한 측정값입니다. 제품이 데이터베이스인 경우 CPU 사용률, 메모리 사용률 및 쿼리 처리량과 같은 지표를 보낼 수 있습니다. 지표는 일반적으로 뉴렐릭으로 전송되는 데이터의 양을 제한하려는 경우에만 사용됩니다. 오류율 및 처리량과 같은 많은 지표는 이벤트를 집계하여 컴퓨트할 수 있습니다.
* [이벤트](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-events): 이벤트는 제품에서 발생하는 일을 포착합니다. 플랫폼이 애플리케이션 배포를 자동화하는 경우 작업이 실행될 때마다 이벤트를 생성할 수 있습니다. 보안을 위한 특수 스캔을 수행하는 경우 이벤트를 감지할 때마다 이벤트가 생성될 수 있습니다.
* [로그](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-logs): 로그는 사용자가 시스템에서 일어나는 일을 확인하는 데 도움이 되는 시간 기반 텍스트 기록입니다.
* [트레이스](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-traces):트레이스는 시스템을 통해 이동하는 단일 요청의 세부 정보를 캡처합니다.
