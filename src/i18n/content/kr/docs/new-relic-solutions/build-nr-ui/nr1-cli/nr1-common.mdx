## nr1 자동 완성 [#nr1-autocomplete]

기본적으로 이 명령은 `zsh` 에 대한 자동 완성 지침을 표시합니다. Bash에 대한 지침을 보려면 `nr1 autocomplete bash` 실행하세요.

### 용법

`$ nr1 autocomplete`

### 인수

<table>
  <tbody>
    <tr>
      <td>
        `SHELL`
      </td>

      <td>
        지침을 원하는 쉘 유형입니다.
      </td>
    </tr>
  </tbody>
</table>

### 옵션

<table>
  <tbody>
    <tr>
      <td>
        `-r, --refresh-cache`
      </td>

      <td>
        캐시를 새로 고치고 표시 지침을 무시합니다.
      </td>
    </tr>
  </tbody>
</table>

### 예

* `$ nr1 autocomplete`
* `$ nr1 autocomplete zsh`
* `$ nr1 autocomplete bash`
* `$ nr1 autocomplete --refresh-cache`

## nr1 생성 [#nr1-create]

Nerdpack, Nerdlet, 실행기 또는 카탈로그와 같은 템플릿에서 새 구성 요소를 만듭니다. CLI가 이 프로세스를 안내합니다.

Nerdpack과 파일 구조에 대해 자세히 알아보려면 [Nerdpack 파일 구조를](/docs/new-relic-solutions/build-nr-ui/nerdpack-file-structure) 참조하세요. Nerdpack을 설정하는 방법에 대한 자세한 내용은 [Nerdpack CLI 명령을](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-nerdpack) 참조하세요.

### 용법

`$ nr1 create`

### 옵션

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        있는 경우 묻지 않고 기존 파일을 재정의합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-n, --name=NAME`
      </td>

      <td>
        구성요소의 이름을 지정합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --type=TYPE`
      </td>

      <td>
        구성요소 유형을 지정합니다.
      </td>
    </tr>

    <tr>
      <td>
        `--path=PATH`
      </td>

      <td>
        구성 요소에 대한 경로입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

## nr1 도움말 [#nr1-help]

기본적으로 모든 nr1 명령을 표시합니다. 특정 명령에 대한 세부정보를 보려면 `nr1 help COMMAND_NAME` 실행하세요.

### 용법

`$ nr1 help`

### 인수

<table>
  <tbody>
    <tr>
      <td>
        `COMMAND_NAME`
      </td>

      <td>
        특정 명령의 이름입니다.
      </td>
    </tr>
  </tbody>
</table>

### 예

* `$ nr1 help`
* `$ nr1 help nerdpack`
* `$ nr1 help nerdpack:deploy`

## nr1 NRQL [#nr1-nrql]

NRQL 쿼리를 사용하여 데이터베이스에서 데이터를 가져옵니다.

NRQL 및 사용 방법에 대해 자세히 알아보려면 [NRQL 문서를](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/) 참조하세요.

### 용법

`$ nr1 nrql OPTION ...`

### 옵션

<table>
  <tbody>
    <tr>
      <td>
        `-a, --account=ACCOUNT`
      </td>

      <td>
        이는 **필수** 입니다. 사용자 계정 ID입니다.
      </td>
    </tr>

    <tr>
      <td>
        `-q, --query=QUERY`
      </td>

      <td>
        이는 **필수** 입니다. 실행할 NRQL 쿼리입니다.
      </td>
    </tr>

    <tr>
      <td>
        `-u, --ugly`
      </td>

      <td>
        탭이나 공백 없이 내용을 표시합니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

## nr1 프로필 [#nr1-profiles]

프로필을 관리하는 데 사용할 수 있는 명령 목록을 표시합니다. 구체적인 사용법에 대한 자세한 내용을 보려면 `nr1 help profiles:COMMAND` 실행하세요. 프로필은 두 개 이상 있을 수 있으며, 이는 여러 뉴렐릭 계정에서 명령을 실행하는 데 유용합니다.

프로필 설정에 대해 자세히 알아보려면 [Github 워크숍을](https://github.com/newrelic/nr1-workshop/blob/master/lab-cli/INSTRUCTIONS.md) 참조하세요.

### 용법

`$ nr1 profiles:COMMAND`

### 명령

<table>
  <tbody>
    <tr>
      <td>
        `profiles:add`
      </td>

      <td>
        프로필 키체인에 새 프로필을 추가합니다.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:default`
      </td>

      <td>
        어떤 프로필을 기본값으로 할지 선택합니다.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:list`
      </td>

      <td>
        키체인의 프로필을 나열합니다.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:remove`
      </td>

      <td>
        키체인에서 프로필을 제거합니다.
      </td>
    </tr>
  </tbody>
</table>

## nr1 업데이트 [#nr1-update]

최신 버전의 CLI로 업데이트합니다. 원하는 경우 업데이트할 채널을 지정할 수 있습니다.

### 용법

`$ nr1 update`

### 인수

<table>
  <tbody>
    <tr>
      <td>
        `CHANNEL`
      </td>

      <td>
        특정 채널의 이름입니다.
      </td>
    </tr>
  </tbody>
</table>

### 예

* `$ nr1 update`
* `$ nr1 update somechannel`
