## `nr1 config:delete` [#nr1-configdelete]

### 설정 삭제 [#nr1-configdelete-remove]

특정 설정의 값을 제거합니다.

### 용법 [#nr1-configdelete-usage]

`$ nr1 config:delete OPTION`

### 옵션 [#nr1-configdelete-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        이는 **필수** 입니다. 구성의 핵심입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        있는 경우 이 설정은 지정된 프로필로 실행하는 동안에만 적용됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

### 별칭 [#nr1-configdelete-aliases]

* `nr1 config:remove`
* `nr1 config:rm`

### 예 [#nr1-configdelete-exmples]

* `$ nr1 config:delete --key=proxyHttp`

## `nr1 config:set` [#nr1-configset]

### 구성 설정 [#nr1-configset-set]

설정 키가 주어지면 특정 설정 값을 설정합니다. 기본적으로 명령은 키를 제공한 후 새 값을 사용하지만 `--k, --key=KEY` 옵션을 사용하여 이 단계를 건너뛸 수도 있습니다.

### 용법 [#nr1-configset-usage]

`$ nr1 config:set OPTION`

### 옵션 [#nr1-configset-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        이는 **필수** 입니다. 구성의 핵심입니다.
      </td>
    </tr>

    <tr>
      <td>
        `-V, --value=VALUE`
      </td>

      <td>
        구성의 값입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        사용하려는 인증 프로필
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        있는 경우 이 설정은 지정된 프로필로 실행하는 동안에만 적용됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

### 예 [#nr1-configset-examples]

* `$ nr1 config:set --key=proxyEnabled`
* `$ nr1 config:set --key=proxyEnabled --value=ENABLED`
