## `nr1 catalog:info` [#nr1-cataloginfo]

### 카탈로그 세부정보 가져오기 [#nr1-cataloginfo-details]

카탈로그에 표시되는 애플리케이션에 대한 정보를 표시합니다.

특정 Nerdpack 폴더 내에서 실행하면 해당 Nerdpack의 정보가 표시됩니다. 로컬 Nerdpack에서 정보를 가져오지 않으려면 `--nerdpack-id=NERDPACK_ID` 옵션을 사용하여 특정 Nerdpack에서 쿼리하세요.

### 용법 [#nr1-cataloginfo-usage]

`$ nr1 catalog:info`

### 옵션 [#nr1-cataloginfo-options]

<table>
  <tbody>
    <tr>
      <td>
        `-f, --field=FIELD`
      </td>

      <td>
        정보를 원하는 필드를 지정합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-i, --nerdpack-id=NERDPACK_ID`
      </td>

      <td>
        정보를 얻을 Nerdpack을 지정합니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

## `nr1 catalog:submit` [#nr1-catalogsubmit]

### 카탈로그에 정보 보내기 [#nr1-catalogsubmit-send]

애플리케이션의 카탈로그 디렉터리에 추가하는 정보를 수집하고 이를 카탈로그에 저장합니다.

이 명령은 Nerdpack 폴더에서 실행되어야 합니다. 이 명령은 규칙 이름을 사용하여 특정 파일을 검색합니다.

### 용법 [#nr1-catalogsubmit-usage]

`$ nr1 catalog:submit`

### 옵션 [#nr1-catalogsubmit-options]

<table>
  <tbody>
    <tr>
      <td>
        `-P, --skip-screenshots`
      </td>

      <td>
        스크린샷 자산 업로드를 건너뜁니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>
