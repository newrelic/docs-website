## `nr1 nerdpack:build` [#nr1-nerdpackbuild]

너드팩을 구축합니다.

### 용법 [#build-usage]

```sh
nr1 nerdpack:build
```

### 옵션 [#build-options]

<table>
  <tbody>
    <tr>
      <td>
        `--extra-metadata-path=extra-metadata-path`
      </td>

      <td>
        `default: extra-metadata.json` 추가 메타데이터로 json 파일 경로를 지정합니다.
      </td>
    </tr>

    <tr>
      <td>
        `--prerelease=prerelease`
      </td>

      <td>
        지정된 경우 생성된 파일의 현재 버전에 값이 추가됩니다. 예를 들어 `--prerelease=abc`. 그러면 버전은 `1.2.3-abc` 이 됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

### 설명 [#build-description]

Nerdpack을 자바스크립트 및 CSS 번들로 조합하기 위해 웹팩 프로세스를 실행합니다. 다른 많은 CLI 명령과 마찬가지로 Nerdpack의 `package.json` 수준에서 실행해야 합니다. 하위 디렉터리에서 `nr1.json` 파일을 검색하고 유효성을 검사한 후 빌드를 시도합니다. 또한 빌드 프로세스를 시작하기 전에 `nr1 nerdpack:validate` 실행합니다.

## `nr1 nerdpack:clone` [#nr1-nerdpackclone]

git 저장소에서 nerdpack을 복제합니다.

### 용법 [#clone-usage]

```sh
nr1 nerdpack:clone
```

### 옵션 [#clone-options]

<table>
  <tbody>
    <tr>
      <td>
        `-f`, `--force`
      </td>

      <td>
        대상 폴더가 있으면 교체하십시오.
      </td>
    </tr>

    <tr>
      <td>
        `-p`, `--path=path`
      </td>

      <td>
        복제할 디렉터리(기본값은 저장소 이름)입니다.
      </td>
    </tr>

    <tr>
      <td>
        `-r`, `--repo=repo`
      </td>

      <td>
        이는 **필수** 입니다. 위치는 HTTPS 또는 SSH 경로입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

### 설명 [#clone-description]

git 저장소가 주어지면 사용을 시작할 수 있도록 모든 작업을 수행합니다. 여기에는 다음이 포함됩니다.

1. 저장소를 복제합니다.
2. 저장소를 원격 업스트림으로 설정합니다.
3. `npm` 사용하여 의존성/종속성을 모두 설치합니다.
4. 프로필을 사용하여 새 UUID를 생성하고 커밋합니다.

    </tr>

    <tr>
      <td>

      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`

  </tbody>

### 설명 [#publish-description]

이 명령은 Nerdpack을 빌드하고 레지스트리에 업로드합니다.

추가 매개변수가 전달되지 않으면 이 명령은 자동으로 nerdpack 버전을 `STABLE` 으로 태그합니다. 이 동작을 비활성화하려면 `--skip-tag` 플래그를 명령에 전달하세요.

태그에 대한 자세한 내용은 `nr1 nerdpack:tag --help` 참조하세요.

## `nr1 nerdpack:tag` [#nr1-nerdpacktag]

특정 nerdpack 버전에 태그를 지정합니다.

### 용법 [#tag-usage]

```sh
nr1 nerdpack:tag
```

### 옵션 [#tag-options]

<table>
  <tbody>
    <tr>
      <td>
        `-V`, `--from-version=from-version`
      </td>

      <td>
        태그할 버전입니다. 제공되지 않은 경우 `package.json` 에 있는 항목을 사용합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-i`, `--nerdpack-id=nerdpack-id`
      </td>

      <td>
        Nerdpack uuid를 구현하다, 배포하다. 제공되지 않은 경우 `nr1.json` 에 있는 항목을 사용합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-t`, `--tag=tag`
      </td>

      <td>
        `default: STABLE` 태그 이름
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        사용하려는 인증 프로필입니다.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        출력에 추가 정보를 추가합니다.
      </td>
    </tr>
  </tbody>
</table>

### 설명 [#tag-description]

하나의 버전에는 여러 개의 태그가 있을 수 있지만, 각 태그는 하나의 버전에만 적용할 수 있습니다. 기존 태그가 있는 다른 버전의 태그는 이전 태그의 태그를 해제합니다.

구독자에게 표시하려는 버전에 태그를 지정하려면 `STABLE` 사용하는 것이 좋습니다. 자세한 내용은 `nr1 nerdpack:subscribe` 명령을 참조하세요.

### 별칭 [#tag-aliases]

```sh
$ nr1 nerdpack:deploy
```
