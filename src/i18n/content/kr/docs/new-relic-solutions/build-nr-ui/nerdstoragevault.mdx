여러분의 뉴킹릭 제작을 돕기 위해 뉴킹릭 One SDK를 제공합니다. 이 페이지에서는 NerdStorageVault를 사용하여 암호화된 스토리지 솔루션에 데이터를 저장하는 방법을 알아봅니다.

## 앱에서 NerdStorageVault 사용 [#using]

NerdStorageVault를 사용하여 다음과 같은 민감한 비밀 데이터를 저장하고 검색하세요.

* 개인 액세스 토큰
* 의 키
* API 키
* 기타 제3자 비밀

<Callout variant="tip">
  NerdStorageVault는 AES-256 암호화로 암호화됩니다.
</Callout>

## 사용 고려 사항 [#usage]

* 이 저장 공간은 Nerdpack마다 고유하며 다른 Nerdpack과 공유할 수 없습니다.
* NerdStorageVault는 현재 Nerdpack의 컨텍스트 내에서만 사용할 수 있습니다.
* NerdStorageVault에 직접 쿼리하거나 변형할 수 없습니다.
* [NerdGraph API 탐색기를](https://api.newrelic.com/graphiql) 통해 필드와 응답 유형을 탐색할 수 있습니다.
* 지원되는 유일한 범위는 ACTOR 또는 특정 사용자와 관련된 데이터입니다.

## 구성 요소 라이브러리 [#component]

현재 NerdStorageVault는 [Nr1 커뮤니티 구성 요소 라이브러리](https://www.npmjs.com/package/@newrelic/nr1-community/v/1.3.0-alpha.5) 의 알파 릴리스에서 사용할 수 있으며 Nr1 핵심 구성 요소 라이브러리에서는 사용할 수 없습니다. 오픈 소스 구성 요소를 사용하면 추가 기능을 위해 Nr1 핵심 구성 요소를 강화할 수 있습니다.

## NerdGraph 쿼리 [#nerdgraph]

다음은 NerdStorageVault와 상호 작용하기 위해 구성 요소에서 사용하는 쿼리의 예입니다.

<Callout variant="tip">
  NerdStorageVault는 현재 Nerdpack의 컨텍스트 내에서만 사용할 수 있습니다.
</Callout>

### 쓰다 [#write]

```graphql
mutation {
  nerdStorageVaultWriteSecret(
    scope: { actor: CURRENT_USER }
    secret: { key: "test", value: "secret_test" }
  ) {
    status
    errors {
      message
      type
    }
  }
}
```

### 삭제 [#delete]

```graphql
mutation {
  nerdStorageVaultDeleteSecret(
    scope: { actor: CURRENT_USER }
    key: "test_key"
  ) {
    status
    errors {
      message
      type
    }
  }
}
```

### 단일 비밀 가져오기 [#fetch-single]

```graphql
query {
  actor {
    nerdStorageVault {
      secret(key: "test_key") {
        key
        value
      }
    }
  }
}
```

### 비밀 목록 가져오기 [#fetch-list]

```graphql
query {
  actor {
    nerdStorageVault {
      secrets {
        key
        value
      }
    }
  }
}
```

### 제한 [#limits]

* ACTOR당 최대 10개의 비밀을 저장할 수 있습니다.
* 비밀 값은 5000자로 제한됩니다.
* 핵심 가치는 64자로 제한됩니다.

### NerdStorageVault 작업을 위한 권한 [#permissions]

데이터 쓰기, 삭제, 가져오기 등 NerdStorageVault에 대한 변경 사항을 유지하려면 [변경 사항 유지 권한이 있는 사용자 역할이](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 있어야 합니다.
