  코스의 각 레슨은 마지막 레슨을 기반으로 하므로 이 레슨을 시작하기 전에 마지막 레슨인 nerdlet에 탐색 추가를 완료했는지 확인하세요.

마지막 강의에서는 이 과정 전체에서 구축해 온 A/B 테스트 애플리케이션을 완료했습니다. 이제 출판을 준비할 차례입니다. New Relic 앱 카탈로그에 앱을 게시하면 사용자가 해당 앱을 보고 구독할 수 있습니다. 앱의 기능, 사용 방법 등을 보여주고 알려줌으로써 사용자를 도울 수 있습니다.

## 카탈로그 정보 생성 [#create]

앱에 대한 정보를 카탈로그에 제공하려면 Nerdpack에 `catalog` 디렉터리를 만들어야 합니다.

<Steps>
  <Step>
    코스워크 [저장소](https://github.com/newrelic-experimental/nru-programmability-course) 의 _explain-app/ab-test_ 디렉터리로 변경합니다.

    ```sh
    cd nru-programmability-course/describe-app/ab-test
    ```
  </Step>

  <Step>
    _카탈로그_ 디렉터리를 만듭니다.

    ```sh
    nr1 create --type catalog
    ```

    그러면 앱에 대한 맞춤 정보를 입력하기 위한 템플릿 파일이 포함된 루트 `catalog` 디렉터리와 스크린샷을 저장할 수 있는 각 Nerdpack 항목에 대한 `catalog` 디렉터리가 생성됩니다.
  </Step>

  <Step>
    ## 선적 서류 비치 [#documentation]

    이 기능은 New Relic과 Nerdsletter API 의 데이터를 결합하고 제시하므로 어떤 디자인 버전이 더 높은 품질의 뉴스레터 구독을 생성하는지에 대한 정보에 입각한 결정을 내릴 수 있습니다.

    또한 로컬 저장소에 날짜 및 버전 설명을 저장하는 테스트 종료 버튼도 제공됩니다.

    향후 A/B 테스트에도 이 앱을 사용할 수 있습니다!

    이 과정에서는 앱이 수행하는 작업만 설명하여 문서를 간결하게 유지합니다. 자신의 프로젝트에는 오픈 소스 컨텍스트에서 프로젝트에 기여하기 위한 지침, 앱의 의존성/종속성 설치 또는 사용자가 앱이나 코드의 복잡성을 탐색하는 데 도움이 될 수 있는 기타 사항을 포함할 수도 있습니다.
  </Step>

  <Step>
    업데이트 `catalog/config.json` :

    ```json
    {
      "tagline": "Win @ newsletter subscriptions",
      "details": "Display test data for our newsletter subscription A/B test",
      "categoryTerms": ["browser agent"],
      "keywords": ["a/b test"],
      "repository": "https://github.com/newrelic-experimental/nru-programmability-course",
      "whatsNew": "Initial release! Includes:\n- A variety of charts for understanding the test results\n- An end test button for storing some test data in a table",
      "support": {
        "email": {
          "address": ""
        },
        "issues": {
          "url": ""
        },
        "community": {
          "url": ""
        }
      }
    }
    ```

    여기에서 태그라인, 프로젝트 세부 정보, 앱을 배치할 Instant 옵저버빌리티 카테고리, 검색 키워드, 소스 코드 대상 및 초기 릴리스에 포함된 모든 항목의 짧은 목록을 지정했습니다. 다른 프로젝트에서는 지원 이메일이나 지원 링크를 추가할 수도 있습니다.
  </Step>

  <Step>
    [이 스크린샷을](https://github.com/newrelic-experimental/nru-programmability-course/raw/main/publish/ab-test/catalog/screenshots/screenshot.webp) 다음 두 디렉터리에 저장합니다.

    * `nru-programmability-course/describe-app/ab-test/catalog/screenshots`
    * `nru-programmability-course/describe-app/ab-test/nerdlets/ab-test-nerdlet/catalog/screenshots`

      사용자는 카탈로그의 앱 **Overview** \[개요] 페이지에서 스크린샷 캐러셀을 볼 수 있습니다.
  </Step>

  <Step>
    다음 아이콘을 `nru-programmability-course/describe-app/ab-test` 에 `icon.png` 로 저장합니다.

    이는 **Overview** \[개요] 페이지에서 앱의 카탈로그 아이콘 역할을 합니다.
  </Step>
</Steps>

이제 사용자가 앱의 기능과 필요한 이유를 이해할 수 있도록 문서, 스크린샷, 메타데이터, 아이콘을 앱에 추가했습니다. 현재 이러한 파일은 모두 로컬이므로 카탈로그에 제출해야 합니다. 하지만 그렇게 하기 전에 앱 자체를 카탈로그에 게시해야 합니다.

다음 강의에서는 앱을 게시하고, 카탈로그 정보를 제출하고, New Relic에서 결과를 확인하겠습니다.
