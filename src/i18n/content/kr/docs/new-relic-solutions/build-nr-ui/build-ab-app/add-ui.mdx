애플리케이션이 체계적으로 구성되어 있으면 더 읽기 쉽고 이해하기 쉽습니다. A/B 테스트 팀에는 차트는 많지만 조직은 없습니다. 다음 몇 단원에서는 사용자 인터페이스 구성 요소를 사용하여 일부 조직을 애플리케이션에 가져오고 새로운 기능을 제공하게 됩니다. 먼저, `Grid` 구성요소를 사용하여 디자인 가이드의 레이아웃과 일치하도록 차트를 정렬합니다. 둘째, 각 차트가 무엇을 나타내는지 명확히 하기 위해 제목을 추가합니다. 셋째, 앱 상단에 A 및 B 디자인 버전에 대한 설명을 추가합니다. 마지막으로 A/B 테스트를 종료하기 위한 섹션을 만듭니다.

다음 단원에서는 디자인 가이드에 있는 것처럼 보이도록 차트를 정렬합니다.
