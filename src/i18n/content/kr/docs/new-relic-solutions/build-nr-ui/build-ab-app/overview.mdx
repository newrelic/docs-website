웹사이트를 개발하고 New Relic의 브라우저 모델링을 사용하여 웹사이트를 렌더링했다고 상상해 보세요. New Relic의 핵심 기능 세트는 많은 정보를 제공하지만 사용자 정의된 정보를 원합니다. 귀하는 사이트의 새로운 디자인이 사용자의 관심을 더 잘 끌 수 있는지 결정하려고 합니다. 즉, 디자인을 A/B 테스트하고 데이터를 기반으로 결정을 내리려는 것입니다.

이 작업을 위해 React, New Relic One 소프트웨어 개발 키트, 그리고 최신 웹 기술의 무한한 힘을 사용하여 New Relic 피규어를 만들 수 있습니다.

New Relic 클러스터는 웹 개발에서 가장 인기 있는 JavaScript 라이브러리 중 하나인 React를 기반으로 제작되었습니다. React 코드를 자유롭게 작성할 수 있으므로 앱 로직을 사용자 정의하거나 구성 요소를 직접 디자인하거나 풍부한 오픈 소스 구성 요소 라이브러리를 활용할 수 있습니다. 따라서 A/B 테스트 앱의 경우 결과에 따라 테스트를 종료하는 사용자 지정 논리를 작성하려는 경우 그렇게 할 수 있습니다.

New Relic One 소프트웨어 개발 키트(SDK)를 사용하면 New Relic 플랫폼에 배포하고 구현하고 제공할 수 있습니다. 또한 데이터 수집, 정보 표시, 사용자 삽입 복합 처리 등을 위한 다양한 React 구성 요소를 제공합니다. `Button` 및 `Dropdown` 같은 구성요소를 사용하여 New Relic 고유의 모양과 느낌을 주는 대화형 환경을 만듭니다. `Table` 및 `Chart` 구성요소를 사용하여 New Relic 계정이나 다른 곳의 데이터를 표시합니다. A/B 테스트 애플리케이션을 구축할 때 SDK의 `NrqlQuery` 구성요소를 사용하여 계정에서 브라우저 데이터를 가져옵니다.

사용자 정의 React 코드, SDK 구성 요소 및 광범위한 오픈 소스 라이브러리를 사용하여 New Relic에서 A/B 테스트를 생성할 수 있습니다. 하지만 직접 앱을 만들기 전에 앱 카탈로그를 확인하여 다른 사람이 당신보다 나은지 확인하는 것이 좋습니다! 카탈로그에 이미 해당 앱이 있는 경우 몇 번의 클릭만으로 계정에 추가할 수 있으며, 이는 New Relic에서 앱을 만드는 또 다른 이점입니다.

이 과정을 통해 여러분은 A/B 테스트를 실행하고 관리하기 위한 실제 New Relic 디자인을 구축하게 됩니다. 경쟁 디자인에 대한 브라우저 데이터를 시각화하고, 과거 테스트의 이력 데이터를 확인하고, 심지어 성공적인 디자인을 선택하고 테스트를 종료하는 것까지 모두 New Relic 구성에서 할 수 있습니다! 하지만 차트를 작성하고 http 요청을 작성하는 잡초에 빠지기 전에 New Relic 제작이 무엇으로 구성되어 있는지 알아야 합니다.
