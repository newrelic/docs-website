<Callout variant="tip">
  이 강의는 처음부터 New Relic 제작 방법을 가르치는 과정의 일부입니다. 아직 확인하지 않았다면 개요를 확인하세요.

  과정의 각 단원은 마지막 단원을 바탕으로 구성되므로 이 단원을 시작하기 전에 마지막 단원인 Nerdpack 만들기를 완료했는지 확인하세요.
</Callout>

New Relic 피규어를 제작할 때 플랫폼에서 보는 것은 매우 중요합니다. 이는 최종적으로 게시될 환경에서 애플리케이션을 설계, 구현 및 디버그하는 데 도움이 됩니다. `nr1` 사용하면 New Relic이 귀하에게 제공할 수 있도록 귀하의 제작을 호스팅하는 개발 서버를 설치할 수 있습니다.

<Steps>
  <Step>
    Nerdpack의 규정 작성 내부에서 Nerdpack을 제공하십시오.

    ```bash
    nr1 nerdpack:serve
    ```

    Nerdpack이 성공적으로 구축되고 특수를 볼 준비가 되면 New Relic 링크가 포함된 메시지가 표시됩니다.

    ```bash
    [output] {green}✔{plain}  Server ready! Test it at: {purple}https://one.newrelic.com/?nerdpacks=local
    [output] {blue}↩{plain}  Server will reload automatically if you modify any file!
    ```

    추가된 쿼리에 주목하세요. `nerdpacks=local`. 이 쿼리 모델은 로컬에서 제공되는 Nerdpack을 표시하라고 지시합니다.

    <Callout variant="tip">
      New Relic 플랫폼에 귀하의 구성이 표시되지 않으면 쿼리 문자열에 `nerdpacks=local` 포함했는지 확인하세요!
    </Callout>
  </Step>

  <Step>
    제공된 URL로 이동합니다. 홈페이지에서 **Apps** \[앱] 를 선택하면 New Relic 피규어를 볼 수 있습니다.
  </Step>

  <Step>
    **Your apps** \[내 앱] 섹션에서 'A/B 테스트 실행기'라는 실행기를 찾습니다.
  </Step>

  <Step>
    루트 Nerdlet과 기본 환영 메시지를 보려면 실행 프로그램을 선택하세요.
  </Step>
</Steps>

축하합니다. 첫 New Relic 작품을 선보이셨습니다!

명령 출력에서 Nerdpack의 파일을 변경하면 서버가 다시 로드된다는 점에 유의하세요. `nerdlets/ab-test-nerdlet/index.js`) 업데이트하여 시도해 보세요.

```js fileName=nerdlets/ab-test-nerdlet/index.js lineHighlight=5
import React from 'react';

export default class AbTestNerdletNerdlet extends React.Component {
    render() {
        return <h1>A/B test results</h1>;
    }
}
```

앱이 자동으로 새로 고쳐져 새 제목이 표시됩니다.

<Callout variant="course">
  이 강의는 처음부터 New Relic 제작 방법을 가르치는 과정의 일부입니다. 다음 단원인 A/B 테스트 애플리케이션에 차트 구성 요소를 추가하세요.
</Callout>
