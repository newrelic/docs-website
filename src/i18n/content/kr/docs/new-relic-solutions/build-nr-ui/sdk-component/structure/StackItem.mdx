`<Stack>` 구성요소의 하위 요소입니다.

`<Stack>` 는 선택한 레이아웃에 따라 스택 항목의 높이, 너비, 순서를 변경할 수 있습니다.

### 용법

```js
import { StackItem } from 'nr1'
```

### 소품

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>필수</h5> <h5>노드</h5>
      </td>

      <td>
        스택 항목 내부에 표시할 콘텐츠입니다.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>끈</h5>
      </td>

      <td>
        구성 요소에 클래스 이름을 추가합니다. 위치 지정 및 간격 지정 목적으로만 사용해야 합니다.
      </td>
    </tr>

    <tr>
      <td>
        `grow` <h5>부울</h5>
      </td>

      <td>
        `true` 인 경우 항목이 남은 공간을 채웁니다. 이를 작동시키려면 항목 방향으로 사용 가능한 모든 공간을 차지하도록 스택을 확장해야 합니다. 이는 방향에 따라 스택의 `fullWidth` 또는 `fullHeight` 소품을 사용하여 달성할 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        `shrink` <h5>부울</h5>
      </td>

      <td>
        `true` 인 경우 필요한 경우 항목이 축소됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>물체</h5>
      </td>

      <td>
        사용자 정의 스타일을 위한 인라인 스타일.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>끈</h5>
      </td>

      <td>
        `data-test-id` 속성을 추가합니다. 이를 사용하여 단위 및 E2E 테스트의 구성 요소를 목표로 삼습니다. 테스트 ID가 유효하려면 앞에 nerdpack ID를 붙이고 그 뒤에 점이 옵니다. 예를 들어 `my-nerdpack.some-element`.

        **참고:** `data-test-id` 속성이 DOM 에서 제거될 때 표시되지 않을 수 있습니다. 이를 디버그하려면 `e2e-test` 쿼리 변수를 URL에 전달해야 합니다.
      </td>
    </tr>
  </tbody>
</table>
