**이 목록은 최신입니다. 네트워크, IP, 도메인, 포트 및 엔드포인트는 2022년 3월 22일에 마지막으로 업데이트되었습니다.**

API 클라이언트 또는 에이전트가 New Relic과 통신하는 데 사용하는 네트워크, IP 주소, 도메인, 포트 및 끝점의 목록입니다. [TLS는 모든 도메인에 필요합니다.](#tls)

FedRAMP 엔드포인트에 대한 정보는 [FedRAMP 엔드포인트](/docs/security/security-privacy/compliance/fedramp-compliant-endpoints) 설명서를 참조하십시오.

<Callout variant="tip">
  이 문서는 에이전트 및 통합이 New Relic의 도메인에 액세스할 수 있도록 하는 방법을 설명합니다. 네트워크 성능을 모니터링하려면 [네트워크 성능 모니터링 시작하기 를 참조하십시오](https://docs.newrelic.com/docs/network-performance-monitoring/get-started/npm-introduction/) .
</Callout>

## TLS 암호화 [#tls]

고객의 데이터 보안을 보장하고 [데이터 암호화](/docs/security/new-relic-security/compliance/data-encryption) 에 대한 [FedRAMP](https://marketplace.fedramp.gov/#/product/new-relic?sort=productName&productNameSearch=new%20relic) 및 기타 표준을 준수하려면 모든 도메인에 TLS(전송 계층 보안)가 필요합니다. 모든 도메인에 대해 선호하는 프로토콜은 TLS 1.2입니다. 자세한 내용 [은 TLS 1.2에 대한 New Relic의 Explorers Hub 게시물](https://discuss.newrelic.com/t/january-tls-updates-apis-and-browser-apm/88869) 을 참조하세요.

또한 다음을 제외한 대부분의 도메인에는 TLS 1.2가 **필요** 합니다.

* APM 에이전트 연결
* 브라우저 에이전트 연결
* 이벤트 API

필수 및 지원되는 프로토콜 버전에 대한 향후 업데이트 [는 New Relic의 Explorers Hub에서 `Security Notifications` 태그를](https://discuss.newrelic.com/t/security-notifications/45862) 따르십시오.

## 사용자 대면 도메인 [#user-facing-domains]

New Relic이 제대로 작동하려면 브라우저가 여러 도메인과 통신할 수 있어야 합니다. New Relic이 이 섹션에 나열된 여러 통합 도메인과 통신할 수 있도록 허용 목록을 업데이트하세요. 도메인을 차단하면 개별 제품 기능에 문제가 발생하거나 페이지가 완전히 로드되지 않을 수 있습니다.

## APM 에이전트 [#agents]

네트워크 성능과 [데이터 보안](/docs/security/new-relic-security/data-privacy/new-relic-security-protecting-your-data-content) 을 강화하기 위해 New Relic은 IP 범위가 넓은 CDN 및 DDoS 방지 서비스를 사용합니다. New Relic 에이전트를 사용하려면 방화벽이 다음 네트워크 및 포트로 나가는 연결을 허용해야 합니다. 다음 IP 연결을 허용 목록에 추가하려면 사용하는 운영 체제 및 방화벽에 대한 특정 절차를 따르십시오.

### 포트 443 권장 [#port-443]

**권장 사항:** 암호화된 HTTPS 트래픽에 대한 보안 채널인 포트 443을 사용합니다. 일부 New Relic 에이전트는 모든 HTTP 트래픽에 열려 있는 보안되지 않은 채널인 포트 80도 제공합니다.

일부 에이전트는 포트 80과 포트 443을 모두 사용하도록 구성할 수 있지만 포트 443(기본값)을 선택하는 것이 좋습니다. 포트 80을 사용하는 기존 구성이 있는 경우 기본 New Relic 연결인 포트 443을 사용하도록 업데이트할 수 있습니다.

## 픽시 통합 [#pixie-integration]

[Pixie 통합](https://github.com/newrelic/newrelic-pixie-integration) 은 Kubernetes 클러스터에서 실행되고 Pixie에서 선별된 관찰 가능성 데이터 세트를 가져와 OTLP(OpenTelemetry Protocol)를 사용하여 New Relic으로 보냅니다.

Pixie 통합을 위해서는 다음에 대한 아웃바운드 네트워크 액세스가 필요합니다.

* work.withpixie.ai:443
* otlp.nr-data.net:4317(미국 지역 계정)
* otlp.eu01.nr-data.net:4317(EU 지역 계정)

<Callout variant="tip">
  `4317` 포트가 작동하지 않으면 `443` 포트를 사용할 수 있습니다.
</Callout>

## 코드스트림 [#codestream]

New Relic [CodeStream](/docs/codestream/start-here/what-is-codestream) 은 개발 팀이 자연스럽고 상황에 맞는 방식으로 코드를 논의하고 검토할 수 있도록 하는 개발자 협업 플랫폼입니다.

다음 도메인을 사용합니다.

* `api.codestream.com`
* `*.pubnub.com`
* `*.pubnub.net`
* `*.pndsn.com`
* `*.pubnub.io`

## 오픈 텔레메트리 [#opentelemetry]

New Relic은 원격 측정 데이터 내보내기를 위한 OTLP(OpenTelemetry Protocol)를 지원합니다. 이를 통해 OpenTelemetry 커뮤니티에서 개발한 공급업체 중립 구성 요소를 사용하여 데이터를 New Relic으로 내보낼 수 있습니다.

OTLP 데이터를 New Relic으로 내보내려면:

1. 계정 라이선스 키로 헤더(api-key)를 추가하도록 OTLP 내보내기를 구성합니다.
2. 지역에 따라 내보내기가 New Relic으로 데이터를 보내는 엔드포인트를 구성합니다. 자세한 내용은 [OpenTelemetry 빠른 시작](/docs/integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-quick-start/) 을 참조하세요.

* otlp.nr-data.net:(443/4317/4318) (미국 지역 계정)
* otlp.eu01.nr-data.net:(443/4317/4318) (EU 지역 계정)

네트워크 블록:

* 162.247.240.0/22(미국 지역 계정)
* 185.221.84.0/22(EU 지역 계정)
