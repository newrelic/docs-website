## 로그 설치 [#install-logs]

뉴렐릭는 빠르고 확장 가능한 솔루션을 제공합니다. <InlinePopover type="logs"/>플랫폼을 사용하면 단일 위치에서 로그를 다른 텔레메트리 및 인프라 데이터와 연결할 수 있습니다.

* [컨텍스트에서APM 로그](/docs/logs/logs-context/get-started-logs-context/), [피규어 에이전트](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/) 또는 기타 [로그 포워딩 솔루션을](/docs/logs/forward-logs/enable-log-management-new-relic/) 사용하여 로그 관리를 활성화합니다.

* 오류, 반사 트레이스, 호스트 등을 포함하여 [플랫폼 전체에서 관련 로그 데이터를](/docs/logs/logs-context/get-started-logs-context/#response-time-example) 탐색하세요.

* [로그 UI를](/docs/logs/ui-data/use-logs-ui/) 사용하여 로그, 속성, 패턴, 라이브 테일 로깅 및 쿼리에서 의미 있는 패턴을 찾으세요. 그뿐만 아니라 공지 조건을 추가하고, 로그 데이터를 쿼리하고,

  <InlinePopover type="dashboards"/>

  생성하여 팀과 공유할 수 있습니다.

## 네트워크 모니터링 설치 [#install-network]

"네트워크인가?" 이해하기 네트워크 원격 측정을 수집하고 New Relic 플랫폼의 기능을 사용하여 각각의 고유한 원격 측정 유형을 탐색하고 탐색하는 것으로 시작합니다.

어디서부터 시작해야 할지 모르십니까? [안내식 배포 마법사](https://one.newrelic.com/network-performance-monitoring) 를 확인하십시오.

추가하려는 원격 분석을 이미 알고 계십니까? Instant Observability로 [여기](https://onenr.io/0VjYDqkZ3R0) 에서 시작하십시오.

## 합성 모니터 설치 [#synthetic-monitors]

[신세틱 모니터링을](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring/) 사용하면 웹사이트나 API 엔드포인트를 사전에 모니터링하여 콘텐츠가 완벽하게 작동하는지 확인할 수 있습니다. 스크립트로 작성된 브라우저 테스트를 전 세계 위치에서 귀하의 사이트로 보내 귀하의 콘텐츠가 언제 어디서나 항상 작동하는지 확인하십시오.

신세틱 모니터링은 [컨테이너화된 보조 미니언](/docs/synthetics/synthetic-monitoring/private-locations/install-containerized-private-minions-cpms/) 기능을 제외하고 설치가 필요하지 않습니다.

## 모든 소스의 맞춤 데이터 보고 [#custom-data]

당사의 보다 선별된 에이전트 및 통합이 귀하에게 필요한 것을 제공하지 못하는 경우, 모든 소스의 데이터를 보고할 수 있는 API 및 SDK에 대한 액세스를 제공합니다. 자세한 내용은 [모든 소스 페이지에서 뉴렐릭으로 사용자 정의 데이터 가져오기를](/docs/data-apis/custom-data/get-custom-data-from-any-source/) 참조하세요.
