이 규칙은 시스템 자산 내에 환경 태그가 있는지 여부에 따라 점수를 제공합니다. 이 문서에서는 점수에 대한 해석을 설명하고 태그 지정 전략을 개선하기 위해 취할 수 있는 조치에 대한 지침을 제공합니다.

## 설명

점수는 가장 최근의 스캔을 기반으로 하며, 목표에 &quot;환경&quot; 태그 값이 없으면 실패합니다.

## 점수를 해석하는 방법

환경 태그 적용 범위 점수는 시스템에서 적절한 환경 태그를 갖춘 비율을 반영합니다. 결과를 이해하는 방법은 다음과 같습니다.

* **높은 점수(80-100%)**: 대부분의 분해 태그가 적절하게 처리되어 성숙한 태그 관행을 나타냅니다.
* **중간 점수(50-79%)**: 일관성을 개선할 기회가 있는 중간 범위
* **낮은 점수(0-49%)**: 시스템 조직 및 문제 해결, 해결에 지장을 줄 수 있는 제한된 환경 태그

낮은 점수는 조직이 인프라 전반에 걸쳐 메타데이터 표준이나 의미 규칙을 완전히 구현하지 못했음을 나타내는 경우가 많습니다. 이로 인해 다음 작업이 어려울 수 있습니다.

* 문제가 영향을 미치는 환경을 빠르게 식별합니다.
* 환경별 모니터링 및 경고 규칙 적용
* 옵저버빌리티 데이터를 효과적으로 정리하고 필터링하세요

<Callout variant="important">
  환경 태그는 운영상의 명확성을 위해 중요합니다. 이들 없이는 생산 중인 인시던트가 안정성 문제와 혼동하여 응답 시간이 지연되고 비즈니스에 잠재적인 영향을 미칠 수 있습니다.
</Callout>

## 권장 조치

다음 전략을 사용하여 환경 태그 적용 범위를 개선하세요.

### 1. 태그 지정 방식을 검토하고 문서화하세요.

먼저 명확한 태그 기준을 확립하세요.

* 환경 범주(예: 프로덕션, 스테이징, 개발, 테스트)를 정의합니다.
* 지리적 세그먼트(예: 미국 동부, 유럽 서부, 아시아 태평양)를 고려하세요.
* 기능적 부문(예: 고객 중심, 내부, 실험)을 설명합니다.
* 태그 지정 규칙을 문서화하고 팀과 공유하세요.

### 2. 조직에 맞게 규칙을 사용자 지정하세요

귀하의 특정 요구 사항에 맞게 점수표 규칙을 조정하세요.

* 조직의 특정 태그 값을 포함하도록 규칙을 수정하세요.
* 필요한 경우 여러 환경 관련 태그에 대한 유효성 검사를 추가합니다.
* 컨텍스트를 제공하는 인접 태그를 포함합니다(예: 지역, 클러스터, 팀).
* 특정 태그 형식이나 명명 규칙을 예상하는 규칙을 설정합니다.

### 3. 일관된 에이전트 구현, 배포 방식 구현

태그가 인프라 전반에 체계적으로 적용되었는지 확인하세요.

* 환경 변수를 사용하여 에이전트 구현, 배포 중에 태그를 설정하세요.
* 에이전트 확장을 구성하여 환경 태그를 자동으로 적용합니다.
* 태그 지정을 포함하는 인프라 코드 관행 구현
* 일관된 태그를 적용하는 구현 템플릿, 배치 템플릿 생성

### 4. 외부 데이터 소스 활용

기존 시스템에서 자동 태그 지정을 살펴보세요.

* cloud 제공업체 태그(AWS, Azure, GCP)와 통합
* ServiceNow와 같은 설정 관리 데이터베이스의 동기화 태그
* 컨테이너화된 환경에 Kubernetes 레이블 및 주석 사용
* CI/CD 파이프라인과 연결하여 자동으로 태그 표시, 배포하다 서비스

### 5. 태그 지정 전략을 구성하고 검증하세요

시간이 지나도 태그 품질을 유지하기 위한 프로세스를 만듭니다.

* 환경별로 필터링되는 워크로드를 사용하여 유사한 시스템을 그룹화합니다 태그
* 다양한 환경 세그먼트에서 태그 완전성을 정기적으로 감사합니다.
* 태그 준수 여부를 모니터링하기 위한 대시보드 만들기
* 다양한 환경에서 태그를 유지 관리하기 위한 팀 책임 설정

### 6. 포괄적인 태그 전략 개발

태그를 중심으로 조직적 정렬을 구축하세요.

* 명명 규칙 및 필수 태그를 다루는 태그 가이드라인을 만듭니다.
* 일관된 태그 지정 관행의 중요성에 대한 팀 교육
* 태그 거버넌스 및 품질 관리를 위한 프로세스 구축
* 태그가 인시던트 대응 및 시스템 관리를 지원하는 방법을 문서화하세요.

## 중요한 고려 사항

환경 태그 적용 점수를 사용할 때 다음 요소를 염두에 두십시오.

**귀하의 조직에 맞게 사용자 정의**: 점수표 규칙은 일반적인 지침을 제공하지만, 귀하의 특정 환경 구조는 다를 수 있습니다. 운영상의 필요성, 조직 구조, 구현 및 배포 패턴을 기반으로 결과를 평가합니다.

**지속적인 개선에 집중하세요**. 태그 지정 전략은 인프라와 조직 변화에 따라 발전해야 합니다. 시스템이 성장하고 변화하더라도 태그가 관련성과 유용성을 유지하는지 확인하기 위해 접근 방식을 정기적으로 검토하세요.

**표준화와 유연성의 균형**: 일관성이 중요하지만, 환경에 따른 특정 요구 사항도 고려하세요. 일부 서비스에는 다른 서비스에 필요하지 않은 추가 태그가 필요할 수 있지만, 핵심 환경 태그가 있는 한 이는 허용됩니다.

**자동화를 고려하세요**. 수동 태그 지정은 오류가 발생하기 쉽고 확장성이 없습니다. 구현, 배치 파이프라인 및 배치 관리 도구와 통합되는 자동화된 태그 솔루션에 투자하세요.

## 다음 단계

환경 태그 적용 범위를 개선하려면 다음을 수행하세요.

1. **작게 시작하세요**: 가장 중요한 프로덕션 서비스부터 시작하여 적절한 환경 태그가 있는지 확인하세요.
2. **가능한 경우 자동화**: 태그를 구현, 배치 프로세스에 통합하여 일관성을 보장합니다.
3. **진행 상황 모니터링**: 점수표를 사용하여 시간 경과에 따른 개선 사항을 추적합니다.
4. **점진적으로 확장**: 핵심 서비스에 태그가 지정되면 개발 및 스테이징 환경으로 확장합니다.

일관된 환경 태그는 효과적인 옵저버빌리티, 더 빠른 인시던트 처리 및 전체 보기에 대한 더 나은 시스템 구성을 위한 기반을 만듭니다.
