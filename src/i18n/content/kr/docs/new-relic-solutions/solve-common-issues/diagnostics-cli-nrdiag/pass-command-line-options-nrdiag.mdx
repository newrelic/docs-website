진단 CLI에서 다음 명령줄 옵션을 사용하려면:

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        옵션
      </th>

      <th>
        용법
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `-a`

        `-attach`
      </td>

      <td>
        뉴렐릭 계정에 자동 업로드를 위해 첨부합니다. 최대 업로드 크기는 4GB입니다. 이는 환경에서 검증된 <InlinePopover type="licenseKey" />사용합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-api-key`
      </td>

      <td>
        사용자 뉴렐릭 계정에 결과를 업로드할 때 사용할 API 키입니다. 최대 업로드 크기는 4GB입니다. 이렇게 하면 라이선스 키를 사용할 필요가 없고 대신 <InlinePopover type="userKey" />을 사용하여 결과를 첨부할 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        `-r`

        `-region`
      </td>

      <td>
        New Relic 계정이 호스팅되는 [지역](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) . 허용 값: EU 또는 US. 대소문자를 구분하지 않습니다. 기본값: 미국
      </td>
    </tr>

    <tr>
      <td id="browser-check">
        `-browser-url STRING`
      </td>

      <td>
        진단 CLI 버전 1.1.9 이상

        이것이 호출되면 <InlinePopover type="browser" />과 관련된 진단 검사만 실행됩니다. 이 명령은 뉴렐릭의 배포 에이전트가 존재하는지 확인하고 에이전트 버전, 패치 메서드([ <InlinePopover type="apm" />](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app)또는 [복사/붙여넣기를 통해](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app)) 및 로더 유형(Pro, Lite, SPA)을 반환합니다. 문제 해결, 인트라넷 사이트 문제 해결 시 뉴렐릭 지원팀에 세부 정보를 제공하는 데 사용됩니다.
      </td>
    </tr>

    <tr>
      <td>
        `-c STRING`

        `-config-file STRING`
      </td>

      <td>
        기본 에이전트 구성 파일 위치를 재정의합니다. 기본 폴더 외에 검색할 폴더를 지정하거나 특정 구성 파일의 경로를 지정하는 데 사용할 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        `-filter STRING`
      </td>

      <td>
        결과 상태별로 작업 결과를 필터링합니다. 쉼표로 구분된 목록을 허용합니다. `Success` , `Warning` , `Failure` , `Error` , `None` 또는 `Info` 를 허용합니다. 구문 예:

        ```
        "Success,Warning,Failure"
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-h`

        `-help`
      </td>

      <td>
        명령줄 옵션의 전체 목록을 표시합니다. 실행할 모든 작업을 나열하려면 `-h tasks` 을 사용합니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-include STRING`
      </td>

      <td>
        `nrdiag-output.zip`에 파일 또는 디렉터리(하위 디렉터리 포함)를 포함합니다. 4GB로 제한합니다. `-a` 또는 `-api-key` 플래그를 사용하여 결과를 New Relic에 업로드합니다.

        구문 예:

        ```
        -include ./path/to/file
        ```

        ```
        -include \some\path\
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-interactive`
      </td>

      <td>
        유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-k8s-namespace STRING`
      </td>

      <td>
        뉴렐릭 리소스를 스크래핑할 네임스페이스를 지정합니다. Agent-control을 사용하는 경우, &apos;-ac-에이전트-네임스페이스&apos; 플래그를 설정하여 Agent-control 에이전트가 실행 중인 세탁스페이스를 지정할 수도 있습니다.

        구문 예:

        ```
        -k8s-namespace newrelic
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-ac-agents-namespace STRING`
      </td>

      <td>
        에이전트 제어 실행 에이전트를 긁어낼 위치를 지정하십시오.

        구문 예:

        ```
        -ac-agents-namespace newrelic
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-output-path STRING`
      </td>

      <td>
        `nrdiag-output.zip` , `nrdiag-output.json` 및 `nrdiag-filelist.txt` 에 대한 결과를 기록할 다른 출력 디렉토리를 지정합니다. 기본 위치는 `./` 입니다.
      </td>
    </tr>

    <tr>
      <td>
        `-o`

        `-override STRING`
      </td>

      <td>
        New Relic Support에서 요청할 때 재정의할 인수를 전달합니다. 형식: `identifier.property=value` .

        구문 예:

        ```
        -override Java/Config/Agent.Status=Success
        ```

        ```
        -o Base/Config/Validate.agentlanguage=PHP
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-p STRING`

        `-proxy STRING`
      </td>

      <td>
        HTTP 연결 작업에 사용할 프록시를 제공합니다. HTTP 또는 HTTPS일 수 있습니다. 프록시는 `http(s)://proxyIp:proxy` 형식이어야 합니다. 진단 CLI가 에이전트 구성 파일에서 프록시를 찾으면 기본적으로 해당 프록시를 사용합니다. 대부분의 경우 `port` 은 필요하지 않습니다. 진단 CLI가 에이전트 구성 파일에서 프록시를 찾으면 기본적으로 해당 프록시를 사용합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-pw STRING`
      </td>

      <td>
        필요한 경우 프록시 암호. 진단 CLI가 에이전트 구성 파일에서 프록시를 찾으면 기본적으로 해당 프록시를 사용합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-user STRING`
      </td>

      <td>
        필요한 경우 프록시 사용자 이름. 진단 CLI가 에이전트 구성 파일에서 프록시를 찾으면 기본적으로 해당 프록시를 사용합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-q`
      </td>

      <td>
        Quiet 출력은 설명 출력이 아닌 상위 수준 결과만 인쇄합니다. `-y` 도 사용되는 경우 파일 추가 경고를 억제합니다. `-v` 과 모순되지 않습니다. 포함 필터는 무시됩니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-qq`
      </td>

      <td>
        매우 조용한 출력은 출력에 대해 단일 요약 줄만 인쇄합니다( `q` 을 의미함). `-y` 도 사용되는 경우 파일 추가 경고를 억제합니다. `-v` 와 모순되지 않습니다. 포함 필터는 무시됩니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-skip-version-check`
      </td>

      <td>
        최신 버전의 애플리케이션에 대한 자동 확인을 건너뜁니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-s STRING`

        `-suites STRING`
      </td>

      <td>
        특정 제품 또는 문제를 대상으로 하는 작업 모음인 [제품군](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag/#suites) 을 실행합니다. 여러 제품군을 지정하려면 쉼표로 구분하십시오. 모든 제품군 목록을 얻으려면 다음을 실행하십시오.

        ```
        ./nrdiag -h suites
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-t STRING`

        `-tasks STRING`
      </td>

      <td>
        에이전트 또는 작업 유형별로 작업의 하위 집합만 실행합니다. 여러 작업을 지정하려면 쉼표 및/또는 와일드카드 `*` 로 구분합니다. 모든 작업 목록을 보려면 다음을 실행하십시오.

        ```
        ./nrdiag -h tasks
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-usage-opt-out`
      </td>

      <td>
        이 실행을 위해 익명의 진단 CLI 도구 사용 데이터를 New Relic에 보내는 것을 거부합니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-list-scripts`
      </td>

      <td>
        사용 가능한 스크립트를 나열합니다.
      </td>
    </tr>

    <tr>
      <td>
        `-script STRING`
      </td>

      <td>
        지정된 스크립트를 봅니다. 스크립트를 실행하려면 `-run` 과 함께 사용하세요.
      </td>
    </tr>

    <tr>
      <td>
        `-run`
      </td>

      <td>
        스크립트를 실행하려면 `-script` 과 함께 사용하세요.
      </td>
    </tr>

    <tr>
      <td>
        `-script-flags`
      </td>

      <td>
        명령줄 플래그를 스크립트에 전달하려면 `-run -script` 과 함께 사용하세요.
      </td>
    </tr>

    <tr>
      <td>
        `-v`

        `-verbose`
      </td>

      <td>
        검사 실행 중 자세한 로깅을 표시합니다. 기본적으로 비활성화되어 있습니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-version`
      </td>

      <td>
        현재 진단 CLI 버전을 표시합니다. 또한 새 버전을 확인하라는 메시지가 표시되고 사용 가능한 새 버전이 있으면 다운로드하라는 메시지가 표시됩니다. `-skip-version-check` 보다 우선합니다. 유형: 부울
      </td>
    </tr>

    <tr>
      <td>
        `-y`

        `-yes`
      </td>

      <td>
        실행하는 동안 나타나는 모든 프롬프트에 `yes` 에 응답합니다. 기본적으로 비활성화되어 있습니다. 유형: 부울
      </td>
    </tr>
  </tbody>
</table>
