진단 CLI를 사용하려면:

1. [릴리스 정보](/docs/release-notes/diagnostics-release-notes/diagnostics-cli-release-notes/) 를 검토하여 최신 버전인지 확인하십시오.
2. Linux, macOS 및 Windows용 실행 파일이 포함 [된 최신 버전을 다운로드합니다](https://download.newrelic.com/nrdiag/nrdiag_latest.zip) .
3. 플랫폼의 실행 파일을 [애플리케이션의 루트 디렉토리 위치](/docs/agents/manage-apm-agents/troubleshooting/find-agent-root-directory) 로 이동합니다.
4. 보다 정확한 문제 해결을 위해 New Relic 에이전트의 [로깅 수준](/docs/agents/manage-apm-agents/troubleshooting/new-relic-agent-logs-troubleshooting) 을 일시적으로 높입니다. 로깅 수준을 변경한 후 애플리케이션을 다시 시작합니다.
5. 실행 파일을 실행합니다. **권장 사항:** 문제 해결 범위를 더 쉽게 지정하려면 [작업 모음](#suites) (CLI 옵션)과 함께 `nrdiag` 을 실행합니다.

진단 CLI는 에이전트 구성 파일 및 기타 관련 데이터에 대해 루트 디렉터리와 하위 디렉터리를 자동으로 검색합니다.

       ```
       docker exec -it -w APPLICATION_ROOT IMAGE_NAME nrdiag
       ```


## 스위트 플래그(권장되는 CLI 옵션) [#suites]

제품군은 특정 제품 또는 문제를 대상으로 하는 상태 확인 모음입니다. 제품군을 사용하면 문제 해결 범위를 좁히고 오탐지 발생을 줄일 수 있습니다.

사용 가능한 제품군 목록을 검토하려면 `-help suites` 옵션과 함께 진단 CLI를 실행합니다.

```
./nrdiag -help suites
```

`nrdiag` 으로 제품군을 실행하려면 `-suites` 플래그와 하나 이상의 제품군 이름(예: `java` )을 제공하여 인수로 실행할 수 있습니다.

**Linux 및 macOS:**

* 32비트 시스템의 경우:

  ```
  ./nrdiag -suites SUITE NAMES
  ```

* 64비트 시스템의 경우:

  ```
  ./nrdiag_x64 -suites SUITE NAMES
  ```

**Windows:**

PowerShell에서 실행하려면 `cmd` 시작 부분에 `./` 를 추가합니다.

* 32비트 시스템의 경우:

  ```
  nrdiag.exe -suites SUITE NAMES
  ```

* 64비트 시스템의 경우:

  ```
  nrdiag_x64.exe -suites SUITE NAMES
  ```

## zip에 추가 파일 포함 [#include-additional-files]

지원팀과 공유하고 싶은 추가 파일이 있는 경우 `-include` 명령줄 플래그를 사용하여 `nrdiag-output.zip` 파일에 포함할 수 있습니다. 이것은 단일 파일 또는 디렉토리와 함께 사용할 수 있습니다. 디렉토리가 제공되면 모든 하위 디렉토리가 포함됩니다. 포함된 파일의 총 크기 제한은 4GB입니다.

이것을 `-attach` 과 함께 사용 [하여 파일을 New Relic 계정에 업로드하십시오](#attach-account-results) .

**Linux 및 macOS:**

* 32비트 시스템의 경우:

  ```
  ./nrdiag -include ./path/to/file -attach
  ```

* 64비트 시스템의 경우:

  ```
  ./nrdiag_x64 -include /entire/directory/ -attach
  ```

**Windows:**

PowerShell에서 실행하려면 `cmd` 시작 부분에 `./` 를 추가합니다.

* 32비트 시스템의 경우:

  ```
  nrdiag -include Path\To\File -attach
  ```

* 64비트 시스템의 경우:

  ```
  nrdiag_x64.exe -include \Entire\Directory\ -attach
  ```

## New Relic 계정에 결과 업로드 [#attach-account-results]

<Callout variant="important">
  시스템이 외부 IP 주소에 연결 **하지 않도록** 구성된 경우 이 방법은 작동하지 않습니다. 대신 New Relic Support에 이메일로 [출력 파일](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/interpret-nrdiag-output) 을 첨부하십시오. 라이센스 키에 문제가 있는 경우 지원에 이메일로 출력 파일도 보내야 합니다.
</Callout>

### 자동 계정 업로드 [#automatic]

진단 CLI가 실행될 때 결과를 New Relic 계정에 자동으로 업로드하려면 `-attach` 또는 `-a` 명령줄 플래그를 사용합니다. 이렇게 하면 업로드를 위해 환경에서 찾은 모든 새 유물 라이선스 키의 유효성을 검사합니다.

결과를 계정에 업로드하면 `nrdiag-output.json` 및 `nrdiag-output.zip` 의 콘텐츠가 자동으로 업로드됩니다.

**Linux 및 macOS:**

* 32비트 시스템의 경우:

  ```
  ./nrdiag -attach
  ```

* 64비트 시스템의 경우:

  ```
  ./nrdiag_x64 -attach
  ```

**Windows:**

PowerShell에서 실행하려면 `cmd` 시작 부분에 `./` 를 추가합니다.

* 32비트 시스템의 경우:

  ```
  nrdiag.exe -attach
  ```

* 64비트 시스템의 경우:

  ```
  nrdiag_x64.exe -attach
  ```
