진단 CLI( `nrdiag` )는 New Relic 제품의 일반적인 문제를 자동으로 감지하는 유틸리티입니다. 진단 CLI가 문제를 감지하면 문제 해결 단계를 제안합니다. 진단 CLI는 문제 해결 데이터를 New Relic 계정에 자동으로 업로드할 수 있습니다.

진단 CLI는 오픈 소스이며 [GitHub](https://github.com/newrelic/newrelic-diagnostics-cli) 에 있습니다.

에이전트에 대한 추가 문제 해결 단계는 [데이터가 표시되지 않음을](/docs/agents/manage-apm-agents/troubleshooting/not-seeing-data) 확인하세요.

## 호환성

진단 CLI는 Linux, macOS 및 Windows에서 사용할 수 있습니다. 다음과 같은 일반적인 구성 문제를 감지할 수 있습니다.

* **APM**: 모두 사용 가능

  <InlinePopover type="apm"/>

  자치령 대표. Go 에이전트의 경우 기본 연결 확인만 사용할 수 있습니다.

* **Browser monitoring** \[브라우저 모니터링]: 탐지 대상

  <InlinePopover type="browser"/>

  대리인.

* **인프라 모니터링** : Linux 및 Windows 에이전트

* **모바일 에이전트** : iOS 및 Android

* **종합 모니터링** : Containerized private minions(CPM)

진단 CLI를 실행하는 데는 `superuser` 또는 `admin` 권한이 필요하지 않지만 일부 확인에는 이러한 권한이 권장됩니다. 스캔한 파일을 읽을 수 있는 권한이 없으면 오류를 반환합니다.

## 시작하다

진단 CLI를 사용하려면:

1. 필요에 따라 [작업 제품군](/docs/new-relic-solutions/solve-common-issues/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag) 및 [명령줄 옵션](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/pass-command-line-options-nrdiag) 을 포함하여 진단 CLI를 실행합니다.
2. New Relic 계정에 결과를 업로드하려면 [-attach 또는 -api-key](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/pass-command-line-options-nrdiag) 플래그를 제공하세요.
3. 선택 사항: [구성 파일 설정](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/validate-config-file-settings-nrdiag) 을 확인합니다.
4. [출력](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/interpret-nrdiag-output) 을 해석합니다.

다른 New Relic 도구와 마찬가지로 진단 CLI 서비스는 귀하와 고객의 데이터 개인 정보를 보호하도록 설계되었습니다. 자세한 내용은 [진단 CLI 라이선스 및 보안](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/diagnostics-cli-licensing-security) 설명서를 참조하십시오.
