* [당사 플랫폼의 지원 리소스에 액세스하십시오.](#nr-support)

## 플랫폼의 지원 리소스에 액세스 [#nr-support]

플랫폼을 탐색하는 동안 모든 [제품 내 지원](https://one.newrelic.com/redirect/support) 리소스에 접근할 수 있습니다. 오른쪽 상단 탐색 메뉴에서 <DNT>**?**</DNT> 아이콘을 선택하기만 하면 됩니다.

제품 내 지원 보기는 New Relic, 문서 및 탐색기 허브의 새로운 기능에 대한 가시성을 제공합니다. 지원 사이드바는 모든 지원 도구에 대한 빠른 액세스를 제공합니다.

<img title="New Relic's in-product support" alt="In-product support" src="/images/solutions_screenshot-crop_product-support.webp" />

## 무료 지원 포럼에서 질문하세요 [#nr-forum]

[뉴렐릭의 지원 포럼은](https://discuss.newrelic.com/) 모든 사용자에게 무료로 제공되는 포럼입니다. 뉴렐릭 사용자와 직원은 매일 대화에 참여하여 문제를 해결하고, 해결 방법을 찾고, 새로운 기능에 대해 논의합니다.

## 새 유물 진단 도구 실행 [#diagnostics]

[New Relic Diagnostics](/docs/using-new-relic/cross-product-functions/troubleshooting/new-relic-diagnostics) 는 Linux, Windows 및 Mac용 자동 진단 도구입니다. 에이전트에서 문제가 감지되면 솔루션을 제안하고 티켓에 첨부할 수 있는 문제 해결 로그를 저장합니다.

## New Relic Docs 및 New Relic University에서 답 찾기 [#find-answers]

새 Relic의 [docs 사이트](https://docs.newrelic.com) 에는 유용한 설치, 구성 및 문제점 해결 팁이 포함되어 있습니다. [기본 페이지](http://docs.newrelic.com)에서 자주 사용되는 범주 및 주제 (예: [릴리스 정보](/docs/release-notes)) 를 선택하십시오. 또는 [페이지에서 검색](http://docs.newrelic.com/search)하십시오.

추가 비디오, 웨비나 및 새 릴릭 기능 사용에 대한 기타 정보는 [New Relic University](https://learn.newrelic.com/) 및 [newrelic.com/resources](https://newrelic.com/resources/videos)을 방문하십시오.

## 문서에 기여 [#docs-contribution]

각 [공개 문서](https://docs.newrelic.com) 의 오른쪽에는 피드백 위젯이 있습니다. 페이지가 도움이 되는지 여부를 알려주거나, 문제를 생성하거나, 페이지를 편집하는 데 사용합니다.

<img src="/images/solutions_screenshot-crop_docs-feedback.webp" style={{ width: '300px' }} alt="New Relic Docs - Feedback" title="New Relic Docs - Feedback" />

우리 문서는 오픈 소스이며 [GitHub](https://github.com/newrelic/docs-website) 에서 사용할 수 있습니다. 여러분의 기여를 권장합니다! 우리는 문서가 유용하고 완전하며 정확한지 확인하는 데 정말로 관심을 두고 있습니다. 페이지를 편집하려면 문서에서 <DNT>**Edit this doc**</DNT> 버튼을 클릭하여 필요하다고 생각되는 편집 내용이 포함된 풀을 생성하세요. 우리는 당신이 스타일에 대해 걱정하지 않기를 바랍니다. 파일을 편집하면 우리 팀의 기술 작가가 파일의 스타일, 문법, 형식을 검토합니다.

### 필요한 걸 찾지 말아요? 문서 문제 파일 [#file-doc-issue]

설명서에서 답변을 찾을 수 없는 경우, 문제를 제출하여 도움을 요청하실 수 있습니다. 문서를 개선할 수 있는 곳을 발견하면 저희에게도 알려주세요! 이를 수행하려면 <DNT>**Edit this doc**</DNT> 버튼을 클릭하세요. 이렇게 하면 이슈를 생성할 수 있는 github 페이지로 이동합니다. 우리는 귀하의 문제를 조사하여 해결책을 찾겠습니다.

## 플랫폼에 사례를 제출하세요 [#file-ticket]

원하는 것을 찾을 수 없고 구독 단계에 기술 지원이 포함되어 있는 경우 플랫폼에서 바로 지원 요청을 제출하세요.

1. 오른쪽 상단에서 <DNT>**Account** dropdown</DNT> 에서 계정을 선택하세요.

2. 오른쪽 상단 탐색 메뉴에서 **Help(?)** 아이콘을 선택하세요.

3. **도움말** 사이드바가 열립니다. 리소스 섹션에서 <DNT>**Your support cases**</DNT> 선택하세요.

4. <DNT>**+New case**</DNT> 을(를) 클릭합니다.

5. 관련 세부 정보를 입력하세요. 해당되는 경우 URL을 포함하거나 파일 첨부를 선택하여 기록 파일, 뉴렐릭 진단 파일, 스크린샷 또는 기타 유용한 첨부 파일을 포함합니다.

   <Callout variant="tip">
     <DNT>**Review**</DNT> 클릭하면 요청을 제출하기 전에 입력한 모든 세부 정보를 검토할 수 있습니다.
   </Callout>

6. <DNT>**Create Case**</DNT> 을(를) 클릭합니다.

   <Callout variant="important">
     지원 사례를 만드는 기능은 사용자 계정의 [지원 플랜](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings/) 에 따라 달라집니다.
   </Callout>

## 시스템 상태 확인 [#check-status]

항상 [status.newrelic.com을](https://status.newrelic.com) 방문하여 시스템 상태를 확인하는 것이 좋습니다. 진행 중인 사건이 있는 경우 자세한 정보를 확인할 수 있습니다.
