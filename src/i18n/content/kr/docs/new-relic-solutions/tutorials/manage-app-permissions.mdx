## Nerdpack 권한: [#nerdpack-permissions]

기본 사용자, 전체 플랫폼 사용자, 코어 사용자가 Nerdpack을 사용하고 관리하는 방법 간의 [사용자 유형 차이점을](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 알아보세요.

### 기본 사용자 [#basic-users-new-users]

일반 사용자라면 Nerdpack 사용 기능에 제한이 있습니다. Nerdpack은 로컬에서만 생성하고 실행할 수 있습니다. Nerdpacks를 게시, 태그 지정, 구독 및 사용하려면 관리자가 사용자를 전체 플랫폼 사용자 또는 핵심 사용자로 업그레이드해야 합니다.

<Callout variant="tip">
  기본 사용자라면 일반적으로 Nerdpack을 사용할 수 없습니다. 그러나 뉴렐릭이 유지관리하는 Nerdlet 중 일부는 사용할 수 있습니다. 이는 매우 드물며 현재 웹 인터페이스에서 이를 구별할 수 있는 방법이 없습니다. 그리고 Nerdpack에서 Nerdlet을 사용할 수 있더라도 사용자 정의 시각화를 사용할 수 없습니다. 이를 위해서는 전체 플랫폼 사용자 또는 핵심 사용자로 업그레이드해야 합니다.
</Callout>

### 코어 사용자 및 전체 플랫폼 사용자 [#core-full-platform-users-new]

핵심 사용자 또는 전체 플랫폼 사용자라면 **Nerdpacks &quot;수정&quot;** 권한이 있거나 없습니다.

Nerdpack을 게시하고 구독하고 태그를 지정하려면 **Nerdpacks &quot;modify&quot;** \[Nerdpacks &quot;수정&quot;] 권한이 필요합니다. 따라서 **Nerdpacks &quot;modify&quot;** \[Nerdpacks &quot;수정&quot;] 권한이 없으면 로컬에서 Nerdpack을 생성 및 제공하고 계정이 이미 구독한 Nerdpack을 사용할 수만 있습니다.

Nerdpack을 게시, 태그 지정 또는 구독하려면 관리자가 **Nerdpacks &quot;modify&quot;** \[Nerdpacks &quot;수정&quot;] 기능을 부여해야 합니다.

<Callout variant="tip">
  **Nerdpacks &quot;modify&quot;** \[Nerdpacks &quot;수정&quot;] 기능은 기본적으로 사용 가능한 유일한 그룹인 **User** 및 **Admin** 그룹 모두에 포함되어 있습니다. 따라서 대부분의 경우 Nerdpack을 전체 플랫폼 사용자 또는 핵심 사용자로 관리할 수 있습니다. 그러나 전체 플랫폼 사용자이거나 **Nerdpacks &quot;modify&quot;** \[Nerdpacks &quot;수정&quot;] 권한이 포함되지 않은 사용자 정의 그룹에 할당된 핵심 사용자인 경우 Nerdpacks를 관리할 수 없습니다.
</Callout>

## 부모/자녀 계정 기능 [#parent-child]

일부 뉴렐릭 조직에는 [상위/하위 계정 구조가](/docs/accounts/original-accounts-billing/original-users-roles/parent-child-account-structure) 있습니다. 이러한 조직의 경우 하위 계정은 상위 계정에 데이터 백업을 보고합니다. 이 조직 계층은 Nerdpack 기능에 영향을 미칩니다.
