뉴렐릭에서 Nerdpack을 관리하고 사용하기 위한 요구 사항을 이해합니다.

## 기능 개요 [#capabilities]

Nerdpack을 관리하고 사용하는 능력은 뉴렐릭 사용자의 영향을 받습니다.

* 사용자 모델(원래 또는 최신 모델)
* 사용자 유형
* 할당된 역할

[원래 사용자 모델을](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) 사용하든 [최신 사용자 모델을](/docs/accounts/accounts-billing/new-relic-one-user-management/new-relic-one-user-model-understand-user-structure/) 사용하든 관계없이 Nerdpack 기능 요약을 보려면 아래 표를 검토하세요.

## Nerdpack 권한: 최신 사용자 모델 [#nerdpack-permissions]

[최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 기본 사용자, 전체 플랫폼 사용자 및 핵심 사용자가 Nerdpack을 사용하고 관리하는 방법의 차이점을 알아보세요.

### 기본 사용자 [#basic-users-new-users]

최신 사용자 모델의 기본 사용자라면 Nerdpack 기능이 제한됩니다. Nerdpack은 로컬에서만 생성하고 제공할 수 있습니다. Nerdpack을 게시하고, 태그하고, 구독하고, 사용하려면 관리자가 귀하를 전체 플랫폼 사용자 또는 핵심 사용자로 업그레이드해야 합니다.

<Callout variant="tip">
  기본 사용자라면 일반적으로 Nerdpack을 사용할 수 없습니다. 그러나 뉴렐릭이 유지관리하는 Nerdlet 중 일부는 사용할 수 있습니다. 이는 매우 드물며 현재 웹 인터페이스에서 이를 구별할 수 있는 방법이 없습니다. 그리고 Nerdpack에서 Nerdlet을 사용할 수 있더라도 사용자 정의 시각화를 사용할 수 없습니다. 이를 위해서는 전체 플랫폼 사용자 또는 핵심 사용자로 업그레이드해야 합니다.
</Callout>

### 코어 사용자 및 전체 플랫폼 사용자 [#core-full-platform-users-new]

귀하가 최신 사용자 모델의 핵심 사용자이거나 전체 플랫폼 사용자인 경우 **Nerdpacks "modify"** \[Nerdpacks "수정"] 권한이 있거나 그렇지 않습니다.

Nerdpack을 게시하고 구독하고 태그를 지정하려면 **Nerdpacks "modify"** \[Nerdpacks "수정"] 권한이 필요합니다. 따라서 **Nerdpacks "modify"** \[Nerdpacks "수정"] 권한이 없으면 로컬에서 Nerdpack을 생성 및 제공하고 계정이 이미 구독한 Nerdpack을 사용할 수만 있습니다.

Nerdpack을 게시, 태그 지정 또는 구독하려면 관리자가 **Nerdpacks "modify"** \[Nerdpacks "수정"] 기능을 부여해야 합니다.

<Callout variant="tip">
  **Nerdpacks "modify"** \[Nerdpacks "수정"] 기능은 기본적으로 사용 가능한 유일한 그룹인 **User** 및 **Admin** 그룹 모두에 포함되어 있습니다. 따라서 대부분의 경우 Nerdpack을 전체 플랫폼 사용자 또는 핵심 사용자로 관리할 수 있습니다. 그러나 전체 플랫폼 사용자이거나 **Nerdpacks "modify"** \[Nerdpacks "수정"] 권한이 포함되지 않은 사용자 정의 그룹에 할당된 핵심 사용자인 경우 Nerdpacks를 관리할 수 없습니다.
</Callout>

## Nerdpack 권한: 원래 사용자 모델 [#nerdpack-permissions-original]

기본 사용자, 전체 사용자, 핵심 사용자 및 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) 의 관리자가 Nerdpack을 사용하고 관리하는 방법의 차이점을 알아보세요.

### 기본 사용자 [#basic-users-original]

원래 사용자 모델의 기본 사용자라면 Nerdpack 기능이 제한됩니다. Nerdpack은 로컬에서만 생성하고 제공할 수 있습니다. Nerdpack을 게시하고, 태그하고, 구독하고, 사용하려면 관리자가 귀하를 정식 사용자로 업그레이드해야 합니다.

<Callout variant="tip">
  기본 사용자라면 일반적으로 Nerdpack을 사용할 수 없습니다. 그러나 뉴렐릭이 유지관리하는 Nerdlet 중 일부는 사용할 수 있습니다. 이는 매우 드물며 현재 웹 인터페이스에서 이를 구별할 수 있는 방법이 없습니다. 그리고 Nerdpack에서 Nerdlet을 사용할 수 있더라도 사용자 정의 시각화를 사용할 수 없습니다. 이를 위해서는 코어 또는 전체 플랫폼 사용자로 업그레이드해야 합니다.
</Callout>

### 코어 사용자 및 전체 플랫폼 사용자 [#core-full-platform-users-original]

귀하가 원래 사용자 모델의 핵심 또는 전체 플랫폼 사용자인 경우 **Nerdpack Manager** 추가 기능 역할이 있거나 없습니다.

Nerdpack을 게시하고, 구독하고, 태그를 지정하려면 **Nerdpack Manager** 역할이 필요합니다. 따라서 **Nerdpack Manager** 역할이 없으면 로컬에서 Nerdpack을 생성 및 제공하고 계정이 이미 구독한 Nerdpack을 사용할 수만 있습니다.

Nerdpack을 게시, 태그 지정 또는 구독하려면 관리자가 귀하를 핵심 사용자 또는 전체 플랫폼 사용자로 업그레이드하거나 귀하에게 **Nerdpack Manager** 추가 기능 역할을 부여해야 합니다.

### 소유자/관리자 [#owner-admin-users]

소유자이거나 관리 사용자인 경우 Nerdpack 기능을 모두 수행할 수 있습니다. Nerdlet과 시각화 모두를 포함한 모든 Nerdpack 항목을 생성, 제공, 게시, 태그 지정, 구독 및 사용할 수 있습니다. 여기에는 뉴렐릭 또는 귀하의 계정 중 하나가 구축한 Nerdpack이 포함됩니다.

## 부모/자녀 계정 기능 [#parent-child]

일부 뉴렐릭 조직에는 [상위/하위 계정 구조가](/docs/accounts/original-accounts-billing/original-users-roles/parent-child-account-structure) 있습니다. 이러한 조직의 경우 하위 계정은 상위 계정에 데이터 백업을 보고합니다. 이 조직 계층은 Nerdpack 기능에 영향을 미칩니다.
