<Callout variant="tip">
  이 강의는 뉴렐릭 플랫폼에서 사용자 정의 시각화를 구축하는 방법을 가르치는 과정의 일부입니다.

  과정의 각 단원은 마지막 단원을 기반으로 하므로 이 단원을 시작하기 전에 마지막 단원 [인 설정으로 시각화 사용자 정의를](/docs/new-relic-solutions/tutorials/custom-visualization-config) 완료했는지 확인하세요.
</Callout>

이전 단원에서는 데이터를 표현하는 방법을 선택할 수 있는 설정 옵션을 시각화에 추가했습니다. 이는 차트 유형을 선택할 수 있지만 이제는 시각화 인스턴스를 만들기 전에만 선택할 수 있다는 점에서 이전 버전의 코드와 다릅니다. 그러면 시각화 인스턴스를 생성한다는 것은 무엇을 의미할까요? 시각화를 구성한 후에는 어떻게 사용합니까?

이 단원에서는 시각화 인스턴스를 생성하고 해당 인스턴스를 대시보드에 추가하는 방법을 알아봅니다.

<Callout variant="tip">
  이전 단계에서 길을 잃었고 이 수업을 시작하기 위해 파일이 어떤 모습이어야 하는지 알고 싶다면 Github에서 [과정 프로젝트를](https://github.com/newrelic/nru-custom-visualization-course) 확인하세요.
</Callout>

## 대시보드에 사용자 정의 시각화 추가 [#add-dashboard]

<Steps>
  <Step>
    Nerdpack의 루트 디렉터리에서 시각화를 게시하고 구독합니다.

    ```sh
    nr1 nerdpack:publish
    nr1 nerdpack:subscribe
    ```

    <Callout variant="important">
      Nerdpack을 게시하려면 Nerdpacks "수정" 권한(기본적으로 부여됨)이 있는 전체 사용자여야 합니다.
    </Callout>

    다음으로 뉴렐릭 앱 오버레이에서 게시된 시각화를 찾으세요.
  </Step>

  <Step>
    [뉴렐릭 홈페이지](https://one.newrelic.com) 에서 오른쪽 상단 탐색 표시줄에 있는 **Apps** 버튼을 선택하세요.
  </Step>

  <Step>
    **Apps** 페이지에서 **Custom Visualizations** \[사용자 정의 시각화] 실행 프로그램을 클릭합니다.
  </Step>

  <Step>
    **RadarOrTreemap 을** 선택합니다.
  </Step>

  <Step>
    이전 단원에서는 차트 유형을 선택하기 위해 플랫폼 설정 패널을 사용하도록 시각화를 변경했습니다.
  </Step>

  <Step>
    **Add to dashboard** \[대시보드에 추가 를] 클릭합니다. 그런 다음 사용 가능한 대시보드 목록에서 대시보드를 선택하거나 **New dashboard** \[새 대시보드 를] 선택합니다.

    새 대시보드를 만들기로 결정한 경우 계정과 이름을 선택하세요.

    **대시보드에 추가를** 클릭합니다.
  </Step>

  <Step>
    대시보드로 이동하여 사용자 정의 시각화를 확인하세요.

    다른 차트 유형을 사용하여 다른 버전의 시각화를 만들어 동일한 대시보드에 추가할 수 있습니다. 이것이 바로 이러한 인스턴스를 시각화라고 부르는 이유입니다.
  </Step>
</Steps>

## 시각화 삭제 [#delete]

대시보드에 시각화를 추가한 후에는 편집할 수 없습니다. 기본 쿼리, 계정 ID, 차트 유형 또는 이전에 구성 가능한 기타 옵션은 변경할 수 없습니다. 따라서 시각화가 마음에 들지 않으면 대시보드에서 시각화를 삭제하고 이 가이드의 단계를 다시 진행하여 새 인스턴스를 추가하세요.

<Steps>
  <Step>
    시각화의 오른쪽 모서리에 있는 타원 버튼을 클릭하고 **Delete** \[삭제 를] 클릭합니다.
  </Step>

  <Step>
    시각화를 삭제할 것인지 확인하세요.
  </Step>
</Steps>
