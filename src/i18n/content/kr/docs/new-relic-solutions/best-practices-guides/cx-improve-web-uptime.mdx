이 가이드에는 웹 가동 시간을 개선하는 방법에 대한 몇 가지 아이디어가 포함되어 있습니다.

## 1. 합성수지 조사

실패한 합성 검사가 발생할 때 이를 조사하고 해결합니다. 이 작업을 수행하는 방법에 대한 팁:

* 여러 위치를 확인하십시오.
* 여러 위치에서 오류가 발생하면 알리도록 경고를 설정합니다.

## 2. 워크로드 생성

합성 모니터를 브라우저 애플리케이션, 서비스 및 인프라와 상관시키는 워크로드를 생성합니다. 경고 Runbook에 워크로드에 대한 링크를 포함합니다. 글로벌 검색을 사용하여 합성에 대한 상관된 워크로드를 찾을 수도 있습니다.

다음을 확인하세요. <InlinePopover type="alerts"/>워크로드의 각 계층에 대해 구성됩니다. 이렇게 하면 하나의 보기에서 각 계층의 상태를 볼 수 있습니다. 이렇게 하면 문제 해결에 소요되는 시간을 절약할 수 있습니다. 이 보기의 이점을 활용하기 위해 각 계층에 대한 경고 알림을 만들 필요는 없습니다.

## 3. 가장 낮은 계층에서 먼저 정전 조사

중단이 발생하면 경고하는 가장 낮은 계층에서 조사를 시작합니다. 예를 들어 인프라 문제와 JavaScript 문제가 있는 경우 이를 위임할 수 있는 두 번째 사람이나 팀이 없는 한 JavaScript보다 먼저 인프라를 조사하십시오.

문제 해결에 사용할 수 있는 도구를 사용하십시오.

* 분산 추적이 활성화되어 있는지 확인하십시오.

* Lookout을 사용하여 펄럭이거나 재발하는 문제를 일으키는 원인을 이해하는 데 도움이 됩니다.
