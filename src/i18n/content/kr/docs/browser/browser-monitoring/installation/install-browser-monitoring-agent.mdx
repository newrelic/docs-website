브라우저는 "에이전트"라고도 하는 [JavaScript 스니펫](/docs/browser/new-relic-browser/page-load-timing/instrumentation-page-load-timing) 을 사용하여 앱의 웹페이지를 계측합니다. JavaScript는 브라우저 모니터링을 위해 데이터를 수집합니다. 브라우저 에이전트를 설치하기 위해 여러 배포 옵션 중에서 선택할 수 있습니다.

## 브라우저 모니터링 활성화 [#enable]

브라우저 모니터링을 활성화하면 Browser Pro+SPA가 기본 에이전트입니다. 그러면 자동으로 모든 브라우저 모니터링 기능에 액세스할 수 있습니다. 브라우저 모니터링 옵션에 대한 자세한 내용은 이 문서의 [브라우저 에이전트 유형](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent/#agent-types) 을 참조하십시오.

브라우저 모니터링을 활성화하려면:

1. **[one.newrelic.com](https://one.newrelic.com)** 으로 이동하여 **데이터 추가** 를 클릭한 다음 **브라우저 및 모바일** 을 클릭합니다.
2. UI의 지침에 따라 **New Relic Browser** 와 같이 앱에 사용할 브라우저 모니터링용 데이터 소스를 선택하십시오.
3. 앱에 대한 트래픽을 생성한 다음 데이터가 New Relic에 나타날 때까지 몇 분 정도 기다리십시오.
4. 선택 사항: 설치가 완료되고 데이터가 표시되면 추가 에이전트 구성을 위해 [**앱 설정** 페이지](/docs/browser/new-relic-browser/configuration/browser-settings-ui-options-apdex-geography/) 로 이동하거나 [브라우저 에이전트 유형](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent/#agent-types) 을 변경합니다.

브라우저 모니터링 에이전트를 활성화한 후 웹페이지 데이터가 New Relic에 표시되기까지 몇 분이 소요될 수 있습니다. 문제가 있는 경우 문제 [해결 팁](/docs/browser/browser-monitoring/troubleshooting/troubleshoot-your-browser-monitoring-installation/) 을 따르세요.

## 배포 옵션 [#options]

브라우저 모니터링을 배포하는 데 사용하는 옵션에 관계없이 최종 결과는 동일합니다. 브라우저 모니터링 JavaScript 스니펫("에이전트"라고도 함)이 앱 페이지에 삽입됩니다. 선택하는 방법은 선호도와 비즈니스 요구 사항에 따라 다릅니다.

### APM 모니터링 앱 활성화 [#select-apm-app]

[브라우저 모니터링을 활성화](#enable) 하면 APM 에이전트를 사용하여 브라우저 모니터링 JavaScript 스니펫을 자동으로 삽입할 수 있습니다. APM에서 이미 모니터링 중인 앱용 에이전트를 설치하는 가장 쉬운 방법입니다.

APM 모니터링 앱은 [APM **애플리케이션** 색인](/docs/apm/apm-ui-pages/monitoring/view-your-applications-index/) 에 나열됩니다.

### 복사/붙여넣기로 활성화 [#copy-paste-app]

[브라우저 모니터링을 활성화](#enable) 하면 JavaScript 스니펫을 앱의 웹페이지에 수동으로 삽입할 수 있습니다. 복사/붙여넣기 옵션을 사용하면 웹페이지 성능을 모니터링하는 데 필요한 JavaScript 스니펫의 정확한 위치를 제어할 수 있습니다. 다음과 같은 경우에 유용합니다.

* CDN에서 제공하는 독립 실행형 앱, 정적 사이트 및 캐시된 페이지
* 표준 서버 측 앱처럼 브라우저 앱에 밀접하게 연결되지 않은 APM 앱(예: 클라이언트 측 앱이 REST API 백엔드와 통신할 때)

복사/붙여넣기 방법을 배우려면 이 짧은 YouTube 비디오(약 1분 50분)를 시청하십시오.

JavaScript 스니펫 사용에 대한 몇 가지 팁:

* **웹페이지에 배치:** 코드 스니펫을 복사한 다음 페이지에 인라인으로 가능한 한 `<head>` 요소 상단에 붙여넣되 위치를 구분하는 `<meta>` 태그(예: `X-UA-Compatible` 또는 `charset` 정보). 인라인 `head` 배치에 대한 자세한 내용은 [JavaScript 배치 요구 사항](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-browser-monitoring#javascript-placement) 을 참조하십시오.
* **라이선스 키 및 앱 ID:** 생성된 자바스크립트 하단 부근에 브라우저 라이선스 키와 애플리케이션 ID가 있습니다. 이것은 [REST API](/docs/apis/rest-api-v2/browser-examples-v2) 및 [API Explorer](/docs/apis/rest-api-v2/api-explorer-v2/getting-started-new-relics-api-explorer) 에서 유용합니다.

### APM 에이전트를 사용하여 웹 페이지 계측 [#agent-instrumentation]

이 정보는 APM에서도 모니터링하는 앱에 적용됩니다.

APM 에이전트는 페이지 로드 타이밍에 필요한 JavaScript로 웹페이지를 계측할 수 있습니다. APM 에이전트의 API를 사용하여 JavaScript 스니펫을 웹페이지에 수동으로 추가하는 경우 계측 스니펫을 가능한 한 상단에 가깝게 삽입하십시오. 이를 통해 브라우저의 [AJAX 호출](/docs/browser/new-relic-browser/dashboard-details/ajax-dashboard) 및 [JavaScript 오류](/docs/browser/new-relic-browser/dashboard-details/javascript-errors-dashboard) 에 대한 자세한 정보를 활용할 수 있습니다.

자세한 내용은 APM 에이전트에 대한 지침을 참조하세요.

* [C SDK](/docs/agents/c-sdk/instrumentation/guide-using-c-sdk-api/#browser)
* [가다](/docs/agents/go-agent/features/install-new-relic-browser-go-apps)
* [자바](/docs/agents/java-agent/instrumentation/page-load-timing-java)
* [.그물](/docs/agents/net-agent/features/page-load-timing-net)
* [노드.js](/docs/agents/nodejs-agent/supported-features/page-load-timing-nodejs)
* [PHP](/docs/agents/php-agent/features/page-load-timing-php)
* [파이썬](/docs/agents/python-agent/supported-features/page-load-timing-python)
* [루비](/docs/agents/ruby-agent/features/page-load-timing-ruby)

### REST API 사용 [#rest-api]

이 정보는 APM에서도 모니터링하는 앱에 적용됩니다.

[REST API를](/docs/browser/new-relic-browser/browser-agent-apis/adding-or-listing-browser-apps-api-v2) 사용하면 브라우저 모니터링 UI 외부에서 배포를 관리할 수 있습니다. 이는 여러 앱을 배포하는 대규모 조직에 유용합니다.

## 브라우저 에이전트 유형: Lite, Pro, Pro+SPA [#agent-types]

브라우저 에이전트에는 Lite, Pro 및 Pro+SPA의 세 가지 유형이 있습니다. 에이전트 유형은 청구에 영향을 주지 않습니다.

상담원 유형이 가격과 어떤 관련이 있는지에 대한 세부정보:

* [New Relic One 가격 책정](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) : 이 가격 책정 모델에는 청구 요소로 데이터 수집이 있습니다. 데이터 수집을 줄이려면 설치 후 더 작은 에이전트 유형으로 다운그레이드하는 것이 좋습니다.
* [원래 가격](/docs/accounts/original-accounts-billing/product-pricing/product-based-pricing) : 브라우저 모니터링 기능에 대한 액세스는 에이전트 유형이 아닌 구독 계획에 따라 결정됩니다. 즉, 기본 Pro+SPA 에이전트를 사용하지 않을 이유가 없습니다.

초기 에이전트 설치가 완료된 후 [**앱 설정** 페이지](/docs/browser/new-relic-browser/configuration/browser-settings-ui-options-apdex-geography/) 로 이동하여 구성을 편집하거나 구독을 변경할 수 있습니다.
