활성화되면 세션 리플레이 기능이 iframe 콘텐츠에 대한 재생 데이터를 표시하지 않을 수 있습니다. 이러한 제한은 iframe의 원본과 브라우저 에이전트의 배치에 따라 달라지는 iframe 시나리오의 세션 리플레이 동작으로 인해 발생합니다.

웹 페이지에 최상위 창과 자식 `iframe` 이 포함되어 있다고 가정해 보겠습니다. 브라우저 에이전트의 배치에 따른 세션 리플레이 동작은 다음과 같습니다.

* **동일 출처 iframe**:

  * 최상위 창에 있는 배포 에이전트를 사용하면 세션 리플레이가 창과 `iframe` 콘텐츠를 모두 기록합니다.
  * `iframe` 에만 있는 배포 에이전트를 사용하면 세션 리플레이는 `iframe` 내의 콘텐츠만 기록합니다.
  * 최상위 창과 `iframe` 모두에 배포 에이전트를 사용하면 세션 리플레이가 각 컨텍스트를 독립적으로 기록하여 두 개의 별개 세션이 생성됩니다.

* **교차 출처 iframe**:

  * 최상위 창에 있는 배포 에이전트를 사용하면 세션 리플레이가 창 내용만 캡처합니다. `iframe` 콘텐츠는 재생에 표시되지 않습니다.
  * iframe의 배포 에이전트를 사용하면 세션 리플레이가 `iframe` 콘텐츠만 캡처합니다.

**제한 사항:**

* 세션 리플레이는 `<frame>` 요소를 지원하지 않습니다.
