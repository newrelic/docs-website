뉴렐릭의 세션 리플레이를 활용하려면 먼저 브라우저 설정에서 이 기능을 활성화하세요. 이 단계는 웹 앱에서 사용자 복합 작용을 캡처하고, 인사이트를 사용자 여정에 제공하고, 문제 해결, 문제를 효과적으로 해결하는 데 필수적입니다. 세션 리플레이를 활성화하고 에이전트 에이전트가 올바르게 구성되었는지 확인하려면 아래 단계를 따르세요.

<Callout variant="important">
  세션 재생을 활성화하거나 비활성화하고, 전체 세션이나 오류가 있는 세션의 샘플 속도를 조정하는 등의 특정 설정이 서버 측 설정으로 이동되었습니다. 예를 들어, 세션 리플레이 기능을 포함하도록 페이지에서 스니펫을 업데이트한 후에는 샘플 속도를 변경하거나 해당 기능을 비활성화하거나 다시 활성화할 때 스크립트을 다시 배포할 필요가 없습니다. 기능이 구현, 배포하다 펼쳐에 포함되지 않은 경우 서버 측 설정은 아무런 영향을 미치지 않습니다. 이는 브라우저 에이전트 버전 `1.259.0` 이상에 적용됩니다.
</Callout>

## 세션 리플레이 활성화 [#enable-session-replay]

1. <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; All Capabilities &gt; Browser**</DNT> 으)로 이동합니다.

2. 브라우저 앱을 선택하세요.

3. 왼쪽 메뉴에서 <DNT>**Application settings**</DNT> 클릭합니다.

4. <DNT>**Application settings**</DNT> 페이지에서 <DNT>**Pro**</DNT> 또는 <DNT>**Pro + SPA**</DNT> 브라우저 에이전트가 선택되었는지 확인하세요. 세션 리플레이는 Lite 브라우저 에이전트에 사용할 수 없습니다.

5. <DNT>**Session traces**</DNT> 및 <DNT>**Session replay**</DNT> 켜짐으로 전환합니다.

   <Callout variant="important">
     <DNT>**Session replay**</DNT> 설정을 볼 수 없는 경우. 추가 도움이 필요하면 관리자에게 문의하세요.
   </Callout>

## 샘플링 속도 설정 [#configure-sampling-rates]

다음 샘플에 대해 샘플링 속도(0%-100%)를 설정합니다.

* <DNT>**User sessions**</DNT> 모든 사용자 세션의 무작위 샘플을 기록합니다.

* <DNT>**Errored session**</DNT> 오류가 있는 모든 세션의 무작위 샘플을 기록합니다. 에이전트는 클라이언트 측에서 최대 30초 동안 데이터의 슬라이딩 윈도우를 버퍼링하고, 오류가 발생하면 버퍼링된 데이터가 수집되고 나머지 세션이 기록됩니다. 오류가 발생하지 않으면 리플레이는 수집되지 않습니다.

예를 들어 세션 샘플링 비율을 50%로 설정하고 오류 샘플링 비율을 100%로 설정하면 다음을 의미합니다.

* 전체 사용자 세션의 절반이 녹화됩니다.

* 오류가 발생한 모든 세션은 50% 샘플에 속하지 않더라도 기록됩니다.
