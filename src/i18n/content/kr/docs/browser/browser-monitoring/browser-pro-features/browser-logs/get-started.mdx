## 브라우저 로그를 자동으로 감지합니다 [#automatic-log]

자동 로깅은 브라우저 `console` 에서 발행된 메시지를 수집하여 귀하의 프론트엔드 군대의 옵저버빌리티를 극대화하는 데 도움을 줍니다.

Browser logs are tracked by default at the `WARN` level for <DNT>Pro</DNT> and <DNT>Pro+SPA</DNT> agents, but unavailable for the Lite browser agent. We recommend that you first confirm you&apos;re using the <DNT>**Pro**</DNT> or <DNT>**Pro + SPA**</DNT> browser agent.

<Callout variant="important">
  Certain settings, such as enabling or disabling browser log collection and adjusting sample rates or log verbosity, are server-side configurations. Once you add the agent snippet to your pages, you don&apos;t need to redeploy the script to change these settings. However, server-side configurations have no effect if the feature aren&apos;t included in the deployed script. This applies to browser agent version `1.259.0` or later.
</Callout>

## 자동 로그 수집 활성화 (기존 브라우저 앱의 경우) [#enable-automatic-log]

다음의 간단한 설정 지침에 따라 귀하의 옵저버빌리티 및 디버깅 기능을 향상시키십시오.

<Steps>
  <Step>
    **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser** 로 이동합니다.
  </Step>

  <Step>
    브라우저 앱을 선택하세요.
  </Step>

  <Step>
    왼쪽 메뉴에서 **Application settings** \[애플리케이션 설정 을] 클릭합니다.
  </Step>

  <Step>
    Ensure the <DNT>**Pro**</DNT> or <DNT>**Pro + SPA**</DNT> browser agent is selected.
  </Step>

  <Step>
    Toggle **Automatically collect console logs** to **ON** to collect messages sent to the browser&apos;s console. <br />Toggle **Collect logs manually** to **ON** to collect logs sent programatically via `newrelic.log()` or `newrelic.wrapLogger()`.
  </Step>

  <Step>
    샘플링 속도를 편집하여 로그를 수집할 사용자 세션의 비율을 지정합니다. 0〜100 사이의 값을 입력할 수 있습니다.

    <Callout variant="important">
      브라우저 로깅의 기본 샘플링 비율은 사용자 세션의 100%로 설정되어 있습니다. 이 세션 샘플링 속도는 `0%` 에서 `100%` 사이의 값으로 설정할 수 있습니다. 예를 들어 세션 샘플링 속도를 `50%` 로 설정하면 무작위로 선택된 사용자 세션의 약 절반에서 로그 이벤트가 수집됩니다.
    </Callout>
  </Step>

  <Step>
    Select the desired browser log verbosity level. The default verbosity level is set to **WARN**. The levels you can choose from:

    * `ERROR` - collect logs with `ERROR` level
    * `WARN` - collect logs with `WARN` or `ERROR` level
    * `INFO` - collect logs with `INFO`, `WARN`, or `ERROR` level
    * `DEBUG` - collect logs with `DEBUG`, `INFO`, `WARN`, or `ERROR` level
    * `TRACE` - collect logs with `TRACE`, `DEBUG`, `INFO`, `WARN`, or `ERROR` level

    <br />

    <Callout variant="important">
      콘솔 메서드를 통해 전달된 데이터는 직렬화 및 [난독화를](/docs/browser/new-relic-browser/configuration/obfuscate-browser-agent-data) 거칠 수 있습니다. 크기와 빈도에 따라 이는 애플리케이션 성능과 데이터 비용에 부정적인 영향을 미칠 수 있습니다. 일반적으로 콘솔 메서드에 큰 객체나 많은 양의 데이터를 전달하는 것은 권장되지 않습니다.
    </Callout>
  </Step>
</Steps>

## 브라우저 로그 비활성화 [#disable-browser-logs]

기존 브라우저 앱의 브라우저 로그를 비활성화하려면:

<Steps>
  <Step>
    **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser** 로 이동합니다.
  </Step>

  <Step>
    브라우저 앱을 선택하세요.
  </Step>

  <Step>
    왼쪽 메뉴에서 **Application settings** \[애플리케이션 설정 을] 클릭합니다.
  </Step>

  <Step>
    Toggle **Automatically collect console logs** and **Collect logs manually** to **OFF**.
  </Step>
</Steps>

## 데이터 소비

로그는 다른 브라우저 바이트와 동일한 소비 가격을 따릅니다. 생성되는 바이트 양은 메시지 수와 길이에 따라 달라집니다.

자동 로깅 기능을 사용하면 로깅 이벤트에 맞춤 속성을 추가하는 경우를 제외하고는 `newrelic.log` 또는 `newrelic.wrapLogger` 브라우저 API 호출할 필요가 없습니다.
