일부 New Relic 기능에는 브라우저 모니터링 라이센스 키 또는 New Relic 애플리케이션 ID가 필요합니다. New Relic 수집기는 이 키를 읽고 들어오는 데이터가 속한 계정을 결정합니다.

[복사/붙여넣기 방법을](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser#copy-paste-app) 사용하여 앱을 <InlinePopover type="browser"/>에 추가하면 앱의 [브라우저 <DNT>**Settings**</DNT> 페이지에](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring) 생성된 자바스크립트 스니펫이 표시됩니다. 생성된 JavaScript의 하단에는 브라우저 `licenseKey` ( `browser_monitoring_key` 라고도 함) 및 `applicationID` 가 포함되어 있습니다.

* 브라우저 모니터링 `licenseKey` (`browser_monitoring_key`)은 기본

  <InlinePopover type="licenseKey"/>

  에 연결되어 있지만 동일하지는 않은 고유 키입니다. 브라우저 키는 계정의 라이선스 키를 확인하는 데 사용할 수 없습니다.

* 브라우저 `applicationID` 는 브라우저 모니터링에서 각 앱의 고유 식별 코드입니다.

`browser_monitoring_key` 및 `applicationID` 의 보안을 보호하기 위해 [도메인을 모니터링하거나 차단할 수 있습니다](/docs/browser/new-relic-browser/installation-configuration/monitor-or-block-specific-domains) .

## UI에서 브라우저 라이선스 키 및 앱 ID 가져오기 [#procedures]

이 절차는 복사/붙여넣기 배포 방법을 사용할 때 <DNT>**only**</DNT> 적용합니다. [APM 앱 배포 방법을](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser) 사용하는 경우 <DNT>**Settings**</DNT> 페이지에 JavaScript 스니펫이 표시되지 않습니다.

<img
  title="lkey_appId.png"
  alt="lkey_appId.png"
  src="/images/browser_screenshot-crop_license-app-key.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com](https://one.newrelic.com) > Integrations & Agents > Browser monitoring > (choose the copy/paste install method) > (go through each installation step)**</DNT>: 개발자 설정이 끝나면 JavaScript 스니펫에 귀하의 국방 키와 구축 ID가 표시됩니다.
</figcaption>

브라우저 모니터링 라이센스 키 및 애플리케이션 ID 정보만 얻으려면:

1. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > Settings**
   </DNT>

   으)로 이동합니다.

2. 생성된 JavaScript를 선택하여 복사하여 텍스트 파일에 붙여넣습니다.

3. 브라우저 모니터링

   <InlinePopover type="licenseKey"/>

   및 앱 ID 정보(종료 `</script>` 태그 바로 위)를 제외하고 텍스트 파일에서 모든 내용을 삭제하고 저장합니다.
