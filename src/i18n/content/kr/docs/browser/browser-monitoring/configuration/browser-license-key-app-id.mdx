일부 New Relic 기능에는 브라우저 모니터링 라이센스 키 또는 New Relic 애플리케이션 ID가 필요합니다. New Relic 수집기는 이 키를 읽고 들어오는 데이터가 속한 계정을 결정합니다.

[복사/붙여넣기 방법](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser#copy-paste-app) 을 사용하여 브라우저 모니터링에 앱을 추가하면 앱의 [브라우저 **설정** 페이지](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring) 에 생성된 JavaScript 스니펫이 표시됩니다. 생성된 자바스크립트의 하단에는 브라우저 `licenseKey` ( `browser_monitoring_key` 라고도 함) 및 `applicationID` 가 포함됩니다.

* 브라우저 모니터링 `licenseKey` ( `browser_monitoring_key` )은 기본 [계정 라이센스 키](/docs/accounts/accounts-billing/account-setup/new-relic-license-key/) 에 연결되지만 동일하지는 않은 고유 키입니다. 브라우저 키는 계정의 라이센스 키를 결정하는 데 사용할 수 없습니다.
* 브라우저 `applicationID` 는 브라우저 모니터링에서 각 앱의 고유 식별 코드입니다.

`browser_monitoring_key` 및 `applicationID` 의 보안을 보호하기 위해 [도메인을 모니터링하거나 차단할 수 있습니다](/docs/browser/new-relic-browser/installation-configuration/monitor-or-block-specific-domains) .

## UI에서 브라우저 라이선스 키 및 앱 ID 가져오기 [#procedures]

이 절차는 복사/붙여넣기 배포 방법을 사용하는 경우 **에만** 적용됩니다. [APM 앱 배포 방법](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser) 을 사용하는 경우 **설정** 페이지에 JavaScript 스니펫이 표시되지 않습니다.

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Browser > Settings:** 복사/붙여넣기 배포를 위해 생성된 JavaScript의 끝에는 브라우저 모니터링 라이선스 키 및 애플리케이션 ID가 포함됩니다.
</figcaption>

브라우저 모니터링 라이센스 키 및 애플리케이션 ID 정보만 얻으려면:

1. **[one.newrelic.com](https://one.newrelic.com) > Browser > Settings** 으로 이동합니다.
2. 생성된 JavaScript를 선택하여 복사하여 텍스트 파일에 붙여넣습니다.
3. 브라우저 모니터링 라이선스 키와 앱 ID 정보(끝 `</script>` 태그 바로 위)를 제외한 텍스트 파일의 모든 항목을 삭제하고 저장합니다.
