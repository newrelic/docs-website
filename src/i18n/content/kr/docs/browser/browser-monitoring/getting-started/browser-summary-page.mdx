브라우저 모니터링의 **요약** 페이지에는 앱의 실제 사용자 브라우저 성능이 요약되어 있습니다. **요약** 페이지에서는 다음을 수행할 수 있습니다.

* 가장 큰 콘텐츠가 포함된 페인트(LCP), 첫 번째 입력 지연(FID) 및 누적 레이아웃 이동(CLS)을 포함한 핵심 웹 핵심 정보를 한 눈에 확인합니다.
* 앱의 브라우저 측 성능 추세 보기
* 페이지 로드 타이밍 문제를 빠르게 해결
* 다른 브라우저 UI 페이지로 이동

## 요약 페이지 보기 [#viewing]

앱의 브라우저 성능 요약을 보려면 다음 경로 중 하나를 사용하세요.

* **[one.newrelic.com](https://one.newrelic.com) >Explorer > Browser applications** 으로 이동합니다. [Explorer](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/) 를 사용하여 모든 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/) , 즉 애플리케이션 및 호스트에서 모든 요소의 사용자 지정 그룹화에 이르기까지 데이터를 보고하는 식별 가능한 모든 항목에 액세스합니다.
* [one.newrelic.com](https://one.newrelic.com) 으로 이동하여 **브라우저** 를 클릭하고 앱을 선택합니다.

선택한 앱의 **요약** 페이지에서 [표준 New Relic 페이지 기능](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) 을 사용하여 자세한 정보를 드릴다운합니다.

## 핵심 웹 바이탈 [#core_vitals]

**Core** Web Vitals는 [Google의 Core Web Vitals](/docs/browser/new-relic-browser/page-load-timing-resources/pageviewtiming-async-or-dynamic-page-details) 에 따라 브라우저가 어떻게 작동하는지 보여줍니다. 여기에는 다음이 포함됩니다.

* LCP: 가장 큰 함량의 페인트
* FID: 첫 번째 입력 지연
* CLS: 누적 레이아웃 변경

## 사이트에서 사용자 시간 [#user_time]

**사이트 위젯의 사용자 시간** 은 사용자가 사이트에 머문 시간을 보여줍니다.

## 초기 페이지 로드 및 경로 변경 차트 [#page_load]

**초기 페이지 로드 및 경로 변경** 차트는 URL 로드 또는 다시 로드로 인한 기존 URL 변경의 로드 시간을 보여줍니다. 이 차트는 [페이지 로드 시간 페이지](/docs/browser/new-relic-browser/dashboard-details/page-views-dashboard) 에서 [페이지 로드 타이밍 프로세스](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 에 대한 자세한 정보와 함께 나타납니다.

## 사용자 중심 페이지 로드 시간 차트 [#user_load]

**사용자 중심 페이지 로드 시간** 차트는 사용자의 페이지 로드 시간을 보여줍니다. 이 차트는 [페이지 로드 시간 페이지](/docs/browser/new-relic-browser/dashboard-details/page-views-dashboard) 에서 [페이지 로드 타이밍 프로세스](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 에 대한 자세한 내용과 함께 나타납니다.

## 처리량 차트 [#throughput-chart]

**처리량** 차트는 브라우저 처리량을 분당 페이지 수(ppm)로 표시합니다. 차트의 오른쪽 상단에 있는 값은 선택한 시간 범위의 평균 값입니다.

[SPA 모니터링](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring) 을 활성화하고 **요약** 페이지에 **SPA 로드 시간** 차트가 표시되면 **처리량** 차트도 SPA 데이터를 사용합니다.

<Callout variant="tip">
  분당 앱 서버 요청 수(rpm)는 브라우저 [페이지 로드 타이밍](/docs/browser/new-relic-browser/page-load-timing/page-load-timing-process) 의 분당 페이지 수(ppm)와 다른 측정값을 표시할 수 있습니다.
</Callout>

## 프런트 엔드 대 백 엔드 차트 [#front_vs_back]

**프런트 엔드 대 백 엔드** 차트는 APM 서비스 응용 프로그램 연결에 연결됩니다. APM 및 추적에 대한 자세한 내용은 [언어 에이전트 및 분산 추적](docs/distributed-tracing/enable-configure/language-agents-enable-distributed-tracing) 을 참조하세요.

## 추가 차트 [#other-charts]

[페이지 로드 시간 페이지](/docs/browser/new-relic-browser/dashboard-details/page-views-dashboard) 및 [JavaScript 오류 페이지](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-examining-errors-over-time) 에서도 이러한 측정항목의 요약 버전을 볼 수 있습니다.

* 기기 유형별 첫 상호작용
* 사용자 에이전트의 첫 번째 상호 작용
* 자바스크립트 오류
* URL별 가장 긴 첫 번째 입력 지연

## 관련 기관 [#related_entities]

**관련 항목** 위젯은 브라우저와 관련된 모든 항목을 표시합니다. 엔터티 간의 연결에 대해 자세히 알아보려면 [엔터티 관계](docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/#related-entities) 에 대해 읽어보세요.

## 합성 모니터 보기 [#synthetic_monitors]

**합성 모니터 보기** 버튼을 누르면 **합성** 페이지로 이동합니다. [종합 모니터링](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring/) 은 사용자 트래픽을 시뮬레이션할 수 있습니다. 이를 통해 고객이 알아차리기 전에 중요한 엔드포인트의 중단 및 성능 저하를 사전에 감지하고 해결할 수 있습니다.
