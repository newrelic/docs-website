브라우저 모니터링 **요약** 페이지는 앱의 실제 사용자 브라우저 성능 문제를 해결하는 데 도움이 됩니다. **요약** 페이지를 사용하여 다음을 수행합니다.

* 가장 큰 콘텐츠가 포함된 페인트(LCP), 첫 번째 입력 지연(FID) 및 누적 레이아웃 전환(CLS)을 포함한 핵심 웹 필수 정보를 한 눈에 확인합니다.
* 앱의 브라우저 측 성능 추세를 봅니다.
* 페이지 로드 타이밍 문제를 빠르게 해결합니다.
* 다른 브라우저 UI 페이지로 직접 이동하여 문제 및 문제를 더 자세히 조사하십시오.
* 권장 알림 조건을 추가합니다.

## 요약 페이지 보기 [#viewing]

브라우저 앱의 성능 요약을 보려면 다음 단계를 따르세요.

1. **[one.newrelic.com](https://one.newrelic.com) > All Capabilities**으로 이동합니다.
2. **브라우저** 클릭
3. 브라우저 앱을 선택합니다.

## 핵심 웹 바이탈 데이터를 사용하여 SEO 개선 [#core_vitals]

**Core Web Vitals** 차트를 사용하여 [Google의 Core Web Vitals](https://web.dev/vitals/#core-web-vitals) 에 따라 브라우저가 어떻게 작동하는지 이해하세요. 여기에는 다음이 포함됩니다.

* LCP: 적재 성능을 측정하는 가장 큰 함량의 페인트
* FID: 사용자 입력과 브라우저 응답 간의 상호 작용을 측정하는 첫 번째 입력 지연
* CLS: 브라우저 페이지의 시각적 안정성을 측정하는 누적 레이아웃 이동

점수가 높을수록 조직이 비용을 절감하고 SEO에 영향을 줄 수 있습니다.점수가 낮으면 조직 웹 사이트에 대한 사용자의 인식에 영향을 미쳐 고객이 불만족하거나 비즈니스 기회를 잃을 수 있습니다.

핵심 웹 바이탈 모니터링에 대한 자세한 내용은 핵심 웹 바이탈 [가이드를](/docs/browser/new-relic-browser/guides/guide-to-core-web-vitals) 참조하십시오.

## 사용자 경험 평가 [#user_time]

핵심 web vitals 데이터가 포함된 차트 외에도 다음 데이터를 사용하여 앱에 대한 사용자 경험을 평가하세요.

* **사이트에서 사용자 시간**
* **사용자 중심 페이지 로드 시간** 및 **URL별 가장 긴 첫 번째 입력 지연** : [페이지 로드 타이밍 프로세스](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 에 대한 자세한 내용을 보려면 차트 제목을 클릭하여 [**페이지 보기** UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance) 로 직접 이동합니다.

## 브라우저 성능 문제 찾기 [#performance]

다음 차트를 사용하여 브라우저와 관련된 성능 문제를 식별하십시오.

* **초기 페이지 로드 및 경로 변경** : 이 차트는 URL 로드 또는 다시 로드로 인한 기존 URL 변경의 로드 시간을 보여줍니다.자세한 내용을 보려면 차트 제목을 클릭하여 [**페이지 보기** UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance) 로 직접 이동하세요.
* **처리량** : 이 차트는 분당 페이지 수(ppm)로 브라우저 처리량을 보여줍니다.해당하는 경우 이 차트에는 단일 페이지 앱(SPA) 데이터도 표시됩니다.자세한 내용을 보려면 차트 제목을 클릭하여 [**페이지 보기** UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance) 로 직접 이동하세요.
* **자바스크립트 오류가 있는 페이지 조회수** : 자세한 내용을 보려면 차트 제목을 클릭하여 [**JS 오류** UI](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors) 로 바로 이동하세요.

## 다른 문제로 인한 문제 조사 [#other-issues]

성능 문제는 브라우저와 관련이 있거나 다른 문제와 관련이 있을 수 있습니다.브라우저 성능에 영향을 줄 수 있는 기타 문제에 대한 자세한 내용을 보려면 다음을 확인하십시오.

* **프런트엔드 대 백엔드**: 이 차트는 연결된

  <InlinePopover type="apm"/>

  서비스 신청. [분산 추적](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing/)에 대한 자세한 내용을 보려면 차트 제목을 클릭하십시오.

* **장치 유형** (데스크톱, 태블릿 또는 모바일) **별 첫 번째 상호 작용 및 사용자 에이전트에 의한 첫 번째 상호 작용** : 자세한 내용을 보려면 차트 제목을 클릭하여 [**페이지 보기** UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance) 로 바로 이동합니다.

* **관련 엔터티** : 이 위젯을 사용하면 브라우저에 연결된 [엔터티](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/) 문제로 인해 성능 문제가 발생할 수 있는지 여부를 확인할 수 있습니다.

* **종합 모니터 보기** : [종합 모니터링](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring/) 은 사용자 트래픽을 시뮬레이션할 수 있습니다.이 링크는 고객이 알아차리기 전에 중요한 엔드포인트와 관련된 중단 및 성능 저하를 사전에 감지하고 해결하는 데 도움이 될 수 있습니다.

## 권장 알림 조건 추가 [#recommended-alert-conditions]

브라우저 모니터링 **Summary** \[요약] 페이지를 통해 엔터티에 권장되는 경고 조건을 추가할 수 있습니다.

발견되지 않은 엔터티를 더 보려면 [경고 적용 범위 격차 및 조건 권장 사항](/docs/alerts-applied-intelligence/applied-intelligence/alert-coverage-gaps/) 페이지를 참조하십시오.

권장 알림 조건을 추가하려면 다음 단계를 따르세요.

1. 브라우저 모니터링 **Summary**\[요약] 페이지를 엽니다.

2. **Add recommended conditions**\[권장 조건 추가 를]클릭합니다.

3. 여러 권장 경보 조건을 선택하십시오.

   <img

     style={{align: "left"}}
     title="Browser monitoring recommended alert conditions"
     alt="Browser monitoring recommended alert conditions"

4. **Next**클릭합니다.

5. 알림을 받을 정책을 선택하세요. 기존 항목을 선택하거나 [새로 만들](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name)수 있습니다.

6. **Activate conditions**\[조건 활성화를]클릭합니다.
