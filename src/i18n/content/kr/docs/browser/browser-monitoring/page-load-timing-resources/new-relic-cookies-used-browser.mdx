<Callout variant="important">
  브라우저 에이전트 버전 1220 이상에서는 [세션 쿠키 사용이 더 이상 사용되지 않으므로](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1220/#removed-3rd-party-cookies)이 정보는 브라우저 에이전트 버전 1216 이하를 사용하는 경우에만 관련이 있습니다.

  이러한 쿠키는 브라우저 앱 모니터링과 관련이 있습니다. 데이터 프라이버시 및 데이터 삭제 요청에 대한 자세한 내용은 [New Relic 개인 데이터 요청을](/docs/using-new-relic/new-relic-security/security/new-relic-personal-data-requests)참조하십시오.
</Callout>

이 문서는 브라우저 모니터링 및 쿠키와 관련된 개인 정보 보호 및 쿠키 자체에 대한 세부 정보를 설명합니다. 브라우저 에이전트 버전 1220 이전에는 최종 사용자 브라우저에서 [세션 추적을](/docs/browser/new-relic-browser/page-load-timing-resources/session-tracking) 위해 다음 [쿠키가](/docs/browser/new-relic-browser/performance-quality/security-new-relic-browser#cookies) 생성되었습니다.

## 데이터 개인 정보 보호 및 법률 준수 [#gdpr]

기본 브라우저 에이전트 구성은 EU 개인정보 보호법(GDPR) 및 캘리포니아 소비자 개인정보 보호법(CCPA)과 같은 일반 개인정보 보호법에 정의된 개인 데이터를 유지하지 않습니다. New Relic은 당사 제품에서 개인 데이터를 처리하기로 선택한 고객이 해당 개인 정보 보호법에 따라 처리할 수 있도록 조치를 취했습니다. 아래의 추가 정보를 사용하여 이러한 법률을 준수할 수 있도록 하십시오.

        IP 주소
      </td>

      <td>
        New Relic은 데이터 수집 프로세스의 일부로 최종 사용자로부터 IP 주소를 간단히 수집하고 활용합니다. 최종 사용자의 브라우저와 [New Relic 데이터 수집기](/docs/using-new-relic/welcome-new-relic/getting-started/glossary) 간에 연결이 설정되면 IP가 캡처됩니다. IP 주소는 호출자 ID와 마찬가지로 웹 요청에 포함됩니다. New Relic은 누군가가 당신에게 전화를 걸 때 전화번호를 보는 것처럼 IP를 사용합니다.

        IP 주소는 추가 세부 정보에 매핑되는 조회 값으로 사용되므로 New Relic 고객은 성능 문제를 진단할 수 있습니다.

        IP 주소 조회 값은 다음과 같습니다.

        * [`countryCode`](/attribute-dictionary/pageview/countrycode)

        * [`regionCode`](/attribute-dictionary/pageview/regioncode)

        * [`city`](/attribute-dictionary/pageview/city)

        * `asn (Autonomous System Number)`

        * `asnOrganization (ISP/Business/Institution)`

        * [`asnLatitude`](/attribute-dictionary/pageview/asnlatitude)

        * [`asnLongitude`](/attribute-dictionary/pageview/asnlongitude)

          IP 주소 조회 값에 대한 매핑 프로세스는 일반적으로 몇 분 안에 완료되지만 일부 극단적인 경우에는 처리하는 데 최대 24시간이 소요될 수 있습니다. 매핑 프로세스가 완료되면 New Relic은 더 이상 IP 주소를 유지하지 않습니다.
      </td>
    </tr>

    <tr>
      <td>
        기본 쿠키(더 이상 사용되지 않음)
      </td>

      <td>
        브라우저 에이전트 버전 v1217 이하에서 에이전트는 기본적으로 쿠키를 설정합니다. 이러한 에이전트의 경우 브라우저 에이전트와 New Relic 수집기가 쿠키를 배치합니다. New Relic 수집기는 브라우저 에이전트가 연결하고 데이터를 전송할 때 쿠키를 배치합니다.

        방문자가 특정 유형의 쿠키에 동의할 때만 태그/스니펫을 로드하는 조건부 스크립트 로더를 사용하는 쿠키 동의 관리자를 사용할 수 있습니다. 이 옵션을 선택하면 방문자가 동의하고 브라우저 에이전트가 실행될 때까지 페이지 성능이 캡처되지 않을 위험이 있습니다.

        [브라우저 에이전트 v1169가](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1169)출시되면서 New Relic UI의**애플리케이션 설정** 페이지를 사용하여 브라우저 앱에 대한 쿠키 수집( [`JSESSIONID`](#jsessionid)및 [`NREUM`](#nreum) )을 끌 수 있습니다. 이 구성 값이 false(OFF)로 설정되면 더 이상 이벤트 데이터에 세션 정보가 표시되지 않습니다.

        기본적으로 이 값은 새로 만든 브라우저 앱뿐만 아니라 기존의 모든 브라우저 앱에 대해 true(ON)로 설정됩니다. 자세한 내용은 [쿠키 수집 및 세션 추적에 대한 문서를](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking)참조하십시오.
      </td>
    </tr>

    <tr>
      <td>
        사용자 정의 브라우저 구성을 통한 개인 데이터
      </td>

      <td>
        New Relic은 [브라우저 모니터링 API](/docs/browser/new-relic-browser/browser-agent-spa-api) 를 통해 에이전트를 수동으로 구성하는 옵션을 제공합니다. 이것은 New Relic의 서비스 약관에 명시된 제한 사항에 따라 개인 데이터를 캡처하는 데 사용할 수 있습니다. (예를 들어, 신용 카드 번호나 개인 데이터의 민감한 범주는 계약상 금지되어 있습니다.) New Relic은 맞춤형 구성이 가능한 개인 데이터 수집을 포함하여 에이전트가 지정한 지침에 응답합니다.

        귀하(데이터 컨트롤러)가 귀하의 고객 중 한 명으로부터 요청을 받고 처리를 위해 New Relic으로 보낸 개인 데이터를 삭제해야 한다고 결정하는 경우 [New Relic 개인 데이터 요청](/docs/using-new-relic/new-relic-security/security/new-relic-personal-data-requests) 을 참조하십시오.
      </td>
    </tr>
  </tbody>
</table>

## 더 이상 사용되지 않는 쿠키 수집에 대한 세부 정보 [#cookies]

  <Collapser
    id="jsessionid"
    title="JSESSIONID(사용되지 않음)"
  >
    `JSESSIONID` 쿠키는 에이전트 버전 1220 이상에서 더 이상 사용되지 않습니다. 세션 추적은 더 이상 쿠키를 통해 수행되지 않습니다.

    New Relic이 애플리케이션의 세션 수를 모니터링할 수 있도록 **세션 식별자** 를 저장하는 데 사용됩니다. 쿠키 값은 Jetty에서 생성됩니다.

    <Callout variant="important">
      탐색 타이밍 쿠키는 에이전트 버전 v1223 이상에서 더 이상 사용되지 않습니다. 현재 브라우저 에이전트 버전에서 지원하는 모든 브라우저는 [Navigation Timing API를](/docs/browser/new-relic-browser/browser-agent-apis/navigation-timing-api)지원합니다.
    </Callout>

    이 쿠키는 [Navigation Timing API](/docs/browser/new-relic-browser/browser-agent-apis/navigation-timing-api) 를 지원하지 않는 브라우저에서만 생성됩니다. 브라우저가 Navigation Timing API를 지원하는 경우 기본 인터페이스를 사용하여 탐색 시작 시간을 결정할 수 있습니다.

    Mozilla Firefox 버전 7 이상은 Navigation Timing API를 지원합니다. 그러나 브라우저 모니터링이 이 쿠키를 사용하여 작동하는 탐색 타이밍 API의 Firefox 초기 구현에 버그가 있습니다.

  <Collapser
    id="nragent"
    title="NRAGENT(더 이상 사용되지 않음)"
  >
    NRAGENT 쿠키는 브라우저 에이전트 [버전](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) v443 이하를 사용하는 경우에만 설정됩니다. New Relic 수집가가 최종 사용자에게 토큰을 전달할 때 생성됩니다.

    최종 사용자 메트릭은 브라우저의 JSONP 요청(스크립트 검색)을 통해 New Relic 수집기에 전달됩니다. 반환된 JavaScript에는 다음 탐색(페이지가 언로드될 때)에서 쿠키에 기록되는 토큰 식별자를 설정하기 위한 호출이 포함됩니다. 이것은 New Relic 수집기와 New Relic 에이전트 간의 도메인 간 통신을 달성합니다.

    이 쿠키는 최종 사용자 메트릭을 집계하는 New Relic 수집기와 연결된 웹 응용 프로그램에서 실행 중인 에이전트 간에 통신하는 데 사용됩니다. 토큰은 애플리케이션 [계층](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#tier) 트랜잭션 추적을 식별하고 해당 브라우저 추적과 연관시킵니다.
