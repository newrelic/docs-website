브라우저 모니터링의 [페이지 로드 타이밍](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 기능은 간단한 세션 식별자가 포함된 [쿠키](/docs/browser/new-relic-browser/page-load-timing-resources/new-relic-cookies-used-browser#jsessionid) 를 사용하여 세션을 추적할 수 있습니다.

## 작동 원리 [#process]

다음은 세션 추적의 기본 프로세스입니다.

1. 사용자가 New Relic의 브라우저 모니터링이 설치되고 쿠키 수집이 활성화된 웹사이트를 방문하면 세션이 시작됩니다.
2. 사용자의 브라우저가 열리면 브라우저 모니터링 스크립트가 세션을 식별하는 간단한 쿠키를 설정합니다.
3. 그런 다음 스크립트는 후속 페이지 로드 시 쿠키를 읽습니다.
4. 사용자의 브라우저가 닫힐 때까지 동일한 세션이 계속됩니다.

다음 상황에서는 세션 추적이 제대로 작동하지 않습니다.

* 사용자가 브라우저에서 쿠키를 비활성화한 경우
* 브라우저가 New Relic 쿠키와 관련된 정보가 모니터링되는 사이트의 요청과 함께 전송되는 것을 허용 **하지 않도록** 구성된 경우

## 쿠키 수집 활성화 또는 비활성화 [#settings]

New Relic은 데이터 프라이버시를 진지하게 생각합니다. 기본적으로 당사는 브라우저 에이전트가 수집한 개인 데이터를 보유하지 않으며 귀하는 세션 추적을 제어할 수 있습니다. 웹사이트 방문자의 웹 브라우저에서 쿠키 저장을 활성화 또는 비활성화하려면:

1. **[one.newrelic.com](https://one.newrelic.com) > Browser > (앱 선택) > Settings > Application settings** 으로 이동합니다.
2. 앱의 **개인 정보** 설정에서 **쿠키 저장소** 를 전환합니다.
