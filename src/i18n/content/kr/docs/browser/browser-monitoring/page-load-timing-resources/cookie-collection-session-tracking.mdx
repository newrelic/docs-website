브라우저 모니터링의 [페이지 로드 타이밍](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 기능은 간단한 세션 식별자를 저장할 수 있는 localStorage API를 사용하여 세션을 추적할 수 있습니다.

## 작동 원리 [#process]

다음은 세션 추적의 기본 프로세스입니다.

1. 사용자가 New Relic의 브라우저 모니터링이 설치되고 추적을 위한 개인 정보 설정이 활성화된 웹사이트를 방문하면 세션이 시작됩니다.
2. 사용자 페이지가 로드되면 <InlinePopover type="browser" />스크립트가 브라우저에 간단한 세션 식별자를 설정합니다.
3. 그런 다음 스크립트는 동일한 탭 내에서 또는 동일한 도메인의 새 탭에서 후속 페이지 로드, 다시 로드 및 복원에서 ID를 읽습니다.
4. 다음 이벤트 중 하나가 발생하면 세션이 종료되고 새 세션이 시작됩니다.

* 세션이 최대 4시간에 도달했습니다.

* 4시간 이상 전에 시작된 기존 세션으로 페이지가 로드되었습니다.

* 활성 세션의 어떤 페이지도 연속 30분 동안 &quot;상호 작용&quot;하지 않습니다. 웹 페이지와의 상호 작용에 대한 우리의 정의는 다음으로 구성됩니다.

  * 클릭
  * 스크롤링
  * 타자

* [setUserId()](/docs/browser/new-relic-browser/browser-apis/setuserid)를 통해 사용자 ID를 업데이트하는 호출을 실행하면 세션이 프로그램적으로 재설정됩니다.

* 다음 상황에서는 세션 추적이 제대로 작동하지 않습니다.

  * 사용자가 브라우저에서 DOM 저장소를 비활성화한 경우.
  * 브라우저 또는 페이지가 보안 정책 또는 기타 수단을 통한 타사 원본 스크립트의 로드 및 실행을 허용 **하지 않도록** 구성된 경우(자연스럽게 전체 에이전트가 작동하지 않음을 의미함).
  * localStorage와 같은 브라우저 저장 API는 활성 세션 중에 지워집니다.
