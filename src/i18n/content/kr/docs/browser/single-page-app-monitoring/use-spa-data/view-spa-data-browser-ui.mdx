[SPA(단일 페이지 앱) 모니터링을](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring) 선택한 경우 브라우저 <DNT>**Page views**</DNT> 페이지에 SPA 경로 변경 및 초기 페이지 로드에 대한 데이터가 포함됩니다.

<img title="spa_overview.png" alt="spa_overview.png" src="/images/browser_screenshot-full_spa-pageviews.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT>: [SPA 모니터링을 선택](/docs/browser/single-page-app-monitoring/get-started/add-apps-single-page-app-monitoring) 하면 브라우저 <DNT>**Page views**</DNT> 페이지에 경로 변경 및 관련 비동기 브라우저 활동과 같은 SPA 데이터가 표시됩니다.
</figcaption>

## 단일 페이지 앱(SPA) 데이터 [#app-views]

SPA 데이터를 보려면: <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT> 으)로 이동하세요.

초기 페이지 로드 및 경로 변경 사항은 브라우저 상호 작용 이름 [별로 자동으로 그룹화](/docs/browser/new-relic-browser/configuration/group-browser-metrics-urls#hierarchy) 됩니다. [세그먼트에 대한 허용 목록 설정으로](/docs/browser/new-relic-browser/installation-configuration/url-whitelists-grouping-browser-metrics) 이를 조정할 수 있습니다.

[SPA API를](/docs/browser/new-relic-browser/browser-agent-spa-api) 사용하여 사용자 지정 경로 이름을 설정하면 사용자 지정 경로 이름이 표시됩니다. [<DNT>**Group page by**</DNT>](/docs/browser/single-page-app-monitoring/use-spa-data/view-spa-data-new-relic-browser#group-facet) 드롭다운 메뉴를 사용하여 페이지 로드 및 경로 변경을 그룹화하는 방법을 변경할 수 있습니다.

기본적으로 페이지 로드 및 경로 변경 목록은 가장 많은 시간이 소요되는 보기를 목록 상단에 표시합니다. <DNT>**Sort by**</DNT> 드롭다운 메뉴를 사용하여 평균 응답 시간, 평균 응답 시간, 분당 처리량별로 정렬할 수도 있습니다.

[그룹화된 URL](/docs/browser/new-relic-browser/configuration/group-browser-metrics-urls#hierarchy) 로 특정 보기를 검색하려면 <DNT>**Sort by**</DNT> 드롭다운 메뉴 아래 검색창에 입력하세요. 예를 들어 결제 페이지를 나타내는 URL을 찾으려면 `checkout` 검색하세요.

초기 <DNT>**Page view**</DNT> 페이지의 차트는 다음과 같이 표시됩니다.

* 평균 응답 시간이 가장 느린 5개의 보기
* 처리량이 가장 높은 5가지 보기

조사 중인 시간 범위를 변경하려면 페이지 상단 근처의 [시간 선택기를](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) 사용하십시오. (과거 8일이 지난 시간 범위를 선택하면 일부 필터링 및 그룹화 기능을 사용할 수 없습니다.)

## SPA 보기 필터링

<img title="spa_filter.png" alt="spa_filter.png" src="/images/browser_screenshot-full_spa-filters.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views &gt; Filter**</DNT>: 경로 변경, 초기 페이지 로드, 위치 및 브라우저 유형과 같은 기타 속성을 필터링하려면 <DNT>**Filter**</DNT> 를 사용합니다.
</figcaption>

초기 페이지 로드 또는 경로 변경만 보려면 <DNT>**Filter**</DNT> 드롭다운 메뉴를 사용하세요. 예를 들어 경로 변경 사항만 보려면 <DNT>**Filter &gt; Route change**</DNT> 선택합니다.

또한 필터를 사용하면 앱 이름, 브라우저의 지리적 위치, 브라우저 유형 등 페이지 로드 및 경로 변경의 다른 속성을 기준으로 필터링할 수 있습니다. 예를 들어 오리건주 포틀랜드 시의 브라우저에서 발생한 페이지 로드 및 경로 변경 사항만 보려면 <DNT>**Filter &gt; City &gt; Portland**</DNT> 선택합니다.

## 그룹 스파 보기 [#group-facet]

<DNT>**Group page by**</DNT> 드롭다운 메뉴를 사용하여 속성별로 페이지 보기 목록을 그룹화할 수 있습니다. 예를 들어 브라우저 유형별로 평균 응답 시간을 비교하려면 <DNT>**Group page by &gt; userAgent**</DNT> 선택하세요.

필터링과 그룹화를 함께 사용하면 매우 구체적인 데이터를 빠르게 찾을 수 있습니다. 예를 들어 특정 URL이 다른 브라우저에서 로드되는 방식을 비교하려면 다음을 수행합니다.

1. <DNT>**Filter**</DNT> 드롭다운 메뉴에서 <DNT>**targetURL**</DNT> 선택한 후 학습하려는 URL을 선택하세요.
2. <DNT>**Group page by**</DNT> 드롭다운 메뉴에서 <DNT>**userAgent**</DNT> 선택하세요.

## SPA 자세히 보기 [#details]

<img title="spa_details.png" alt="spa_details.png" src="/images/browser_screenshot-full_pageview-details.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views &gt; (select a view)**</DNT>: 목록에서 보기를 선택하면 다양한 세부정보와 분류가 표시됩니다.
</figcaption>

세부 정보를 보려면 개별 페이지 로드 또는 경로 변경을 선택하십시오. 둘 중 하나를 선택하면 [브라우저 상호 작용](/docs/browser/single-page-app-monitoring/understand-spa-data-structure#browser-interactions)에 소요된 시간에 대한 분석이 제공되고 [시간 선택기](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)에서 선택한 창과 일치하는 시계열에 해당 데이터가 표시됩니다.

모든 경로 변경 보기는 이론적으로 초기 페이지 로드일 수도 있습니다. (예를 들어, 경로 변경 URL이 다른 사람에게 전송되고 다른 사람이 이를 로드하면 이제 New Relic에 대한 초기 페이지 로드로 간주됩니다.) 이것이 SPA 보기 세부 정보 페이지에 초기 페이지 로드 및 경로 모두에 대한 차트가 있는 이유입니다. 변경. 이를 통해 뷰가 초기 페이지 로드로 수행하는 방식과 경로가 변경될 때의 성능을 비교할 수 있습니다.

세 가지 차트 표시 옵션이 있으며 <DNT>**Avg initial page load time**</DNT> 차트 제목 오른쪽에 있는 아이콘을 사용하여 선택할 수 있습니다. 기본 표시는 [색상으로 구분된 누적 영역 차트](/docs/apm/new-relic-apm/ui-functions/standard-dashboard-features) 입니다. [히스토그램 표시](/docs/apm/applications-menu/features/histograms-viewing-data-distribution) 나 [백분위수 선 그래프](/docs/apm/applications-menu/features/percentiles-comparing-ranked-data) 로 전환할 수도 있습니다.

또한 세부정보 페이지에는 초기 페이지 로드와 경로 변경을 결합한 <DNT>**Throughput**</DNT> 차트가 있습니다. 차트는 처리량이 가장 높은 5개 페이지를 차트 아래에 나열하고 다른 모든 페이지를 <DNT>**Other**</DNT> 에 통합합니다.

다음은 페이지 로드 및 경로 변경에 대해 표시되는 특정 성능 데이터에 대한 세부정보입니다.

<CollapserGroup>
  <Collapser id="page-load" title="초기 페이지 로드 성능 세부정보">
    초기 페이지 로드의 경우 성능 세부 정보에는 평균 백엔드 시간, 프런트엔드 시간 및 창 onload 이벤트가 포함됩니다.

    * <DNT>**Backend time**</DNT> 네트워크, 웹 앱 시간 및 요청 대기열이 포함됩니다.
    * <DNT>**Frontend time**</DNT> DOM 처리, 페이지 렌더링 및 모든 XHR 완료 시간이 포함됩니다.
    * 창 로드 이벤트가 실행되면 빨간색 가로선이 표시됩니다. 이는 SPA 모니터링이 활성화되지 않은 상태에서 브라우저 에이전트가 측정한 기존 페이지 로드 타이밍에 해당합니다. SPA 모니터링을 사용하면 <DNT>**frontend time**</DNT> 이 완료되기 전에 창 로드 이벤트가 발생하는 것이 일반적입니다. (SPA 페이지 로드 타이밍이 기존 페이지 로드 타이밍과 어떻게 다른지에 대한 자세한 내용은 [SPA 데이터 수집 이해를](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection#page-load-timing-diff) 참조하세요.)
  </Collapser>

  <Collapser id="route-change" title="경로 변경 실적 세부정보">
    경로 변경의 경우 성능 차트에 JS 기간 및 대기 시간이 표시됩니다.

    * <DNT>**JS Duration**</DNT> 는 정의상 동기적이라고 할 수 있는 블록결합 동안의 모든 자바스크립트 실행 시간의 합입니다.
    * 남은 시간은 <DNT>**Waiting time**</DNT> 이라고 하며 전체 기간에서 JS 기간을 뺀 값입니다.
    * SPA v2.1+의 경우 JS 지속 시간만 해당
  </Collapser>
</CollapserGroup>
