Nuxt 대시보드를 사용하면 애플리케이션 오류율, 초기 페이지 로드 평균, 경로 변경, 처리량 및 JavaScript 오류가 포함된 페이지 조회수를 쉽게 추적할 수 있습니다. 우리의 에이전트 에이전트와 Vue <InlinePopover type="browser" />로 구축된 Nuxt 대시보드를 사용하면 올인원 위치에서 중요한 정보를 볼 수 있습니다.

## 설치 [#install]

<Steps>
  <Step>
    ### 선택사항: 인프라 에이전트 설치 [#infra-install]

    인프라 에이전트 설치는 선택 사항이지만 이렇게 하면 앱이 실행 중인 호스트에 대한 중요한 메트릭을 얻을 수 있습니다.

    이를 수행하려면 [인프라 에이전트 설치 단계를](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent)따르십시오.
  </Step>

  <Step>
    ### Nuxt 브라우저 모니터링을 설치하세요 [#browser-install]

    1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> 에서 [<DNT>**+ Integrations &amp; Agents**</DNT> 페이지](https://one.newrelic.com/marketplace) 로 이동합니다.
    2. <DNT>**Nuxt**</DNT> 을 검색하세요. (참고: Nuxt 타일은 브라우저 또는 모바일 컬렉션에 없으므로 검색 기능을 사용해야 합니다.)
    3. 검색 결과에 <DNT>**Nuxt**</DNT> 타일이 나타나면 클릭하고 지침에 따라 JavaScript 코드 스니펫을 생성하세요.
  </Step>

  <Step>
    ### 통합 구성 [#config]

    Nuxt 기능에 JavaScript 스니펫을 추가하려면:

    1. Nuxt 애플리케이션의 <DNT>**`public`**</DNT> 폴더에 JavaScript 파일을 생성하세요.

       ```bash
       touch public/<FILENAME>.js
       ```

    2. 용매 에이전트 스니펫을 `<script>` 태그 없이 새 JavaScript 파일에 복사하여 붙여넣습니다(스니펫 자체만 정의할 기능이 없습니다).

       ```js
       // PASTE THE GENERATED BROWSER AGENT SNIPPET HERE WITHOUT <script> TAGS
       ```

    3. `nuxt.config.js` (또는 `nuxt.config.ts`) 파일을 열고 `app.head` 섹션에 스크립트를 추가하세요.

       ```js
       export default defineNuxtConfig({
         compatibilityDate: '2025-07-15',
         devtools: { enabled: true },
         app: {
           head: {
             script: [
               {
                 src: "/<FILENAME>.js"
               },
             ],
           },
         },
       })
       ```

    4. 응용 프로그램을 다시 시작하십시오.

    몇 분 안에 애플리케이션이 메트릭을 [one.newrelic.com](https://one.newrelic.com)으로 보냅니다.
  </Step>
</Steps>

## 애플리케이션 모니터링 [#monitor]

이제 탐색 모음에서 <DNT>**Browser**</DNT> 선택하면 <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> 에서 애플리케이션을 볼 수 있습니다. <DNT>**one.newrelic.com**</DNT> 플랫폼에서 캡처된 모든 지표를 보려면 귀하의 앱을 선택하세요. 다음 쿼리를 사용하여 NRQL로 데이터를 볼 수도 있습니다.

```sql
SELECT firstPaint
FROM PageView
```
