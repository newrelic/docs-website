우리의 <InlinePopover type="browser"/>모니터링되는 사이트의 각 페이지 헤드 섹션에 작은 "로더" 스크립트를 배치합니다. 이 JavaScript 계측은 다음을 위해 설계되었습니다.

* 프런트엔드 성능에 대한 최대 가시성을 제공하고, 모든 브라우저에서 페이지를 로드하는 데 걸리는 시간을 정확하게 측정하고, JavaScript 오류 및 AJAX 호출에 대한 세부 정보를 보고합니다.
* 검색 엔진 순위, 인덱싱, 크롤링 효율성 또는 기타 SEO 관련 문제를 포함하여 전체 페이지 로드 시간 및 검색 엔진 최적화(SEO)에 최소한의 영향을 미칩니다.

## 인라인 자바스크립트의 효율성 [#why-javascript]

브라우저 모니터링을 위한 가장 효과적인 방법은 모니터링되는 페이지의 헤드에 최소한의 JavaScript 계측 코드 인라인을 포함하는 것입니다. 그런 다음 이 코드는 페이지 로드가 완료된 후 필요한 코드의 나머지 부분을 검색합니다.

다른 브라우저 모니터링 방법에는 페이지 본문 끝에 JavaScript 코드가 포함되거나 외부 스크립트를 단독으로 사용하는 방법이 있습니다. 그러나 이러한 방법은 최종 사용자의 경험에 대한 가시성을 제한할 수 있습니다.

## SEO에 미치는 영향 [#seo-impact]

성능 테스트 결과에 따르면 브라우저 모니터링의 JavaScript를 사용하는 것은 페이지 로드 시간에 미미한 영향을 미칩니다. 또한 사용자 또는 검색 엔진이 사이트와 상호 작용하는 방식에 부정적인 영향을 미치지 않습니다.

웹사이트 소유자에 대한 Google의 일관된 권장 사항은 사용자에게 가치가 있고 검색 엔진 크롤러가 액세스할 수 있는 사이트를 구축하는 것입니다. Google 순위는 가장 관련성 높은 정보와 최고의 사용자 경험을 제공하는 사이트를 선호합니다.

브라우저 모니터링은 다음과 같은 성능 병목 현상을 식별하여 사용자 경험을 개선하는 데 도움이 됩니다.

* [느린 페이지 로드](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity)
* [문제가 있는 JavaScript 오류](/docs/errors-inbox/browser-tab)
* [긴 AJAX 호출](/docs/browser/new-relic-browser/browser-pro-features/ajax-dashboard-identifying-time-consuming-calls)

Javascript 오류를 식별하는 것은 AJAX가 많은 웹 애플리케이션에서 제공하는 콘텐츠에 액세스하기 위해 크롤링하는 웹사이트에서 점점 더 많은 Google 봇이 JavaScript 코드를 실행하고 있기 때문에 특히 유용합니다. 이전에는 사람만 볼 수 있었던 JavaScript 오류(예: 깨진 버튼)는 Google 봇이 사이트와 성공적으로 상호작용할 수 있는지 여부에 영향을 줄 수 있습니다.

검색 엔진에서 참조하는 SEO 및 트래픽이 많은 비즈니스에서 매우 중요하다는 것을 이해합니다. 브라우저 모니터링을 효과적으로 사용하면 성능과 사용자 경험을 개선하여 Google 및 기타 검색 엔진에서 사이트 순위를 높일 수도 있습니다.
