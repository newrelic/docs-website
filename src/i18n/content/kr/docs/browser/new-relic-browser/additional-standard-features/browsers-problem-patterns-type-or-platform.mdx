브라우저 모니터링의 **브라우저** 페이지는 Google Chrome, Mozilla Firefox, Microsoft Internet Explorer 및 Apple Safari와 같이 사용하는 브라우저를 기반으로 최종 사용자의 앱 경험에 대한 정보를 제공합니다. 이 페이지에는 다음이 포함됩니다.

* 처리량별 상위 브라우저(분당 페이지 수 또는 **ppm** )
* 플랫폼 유형(모바일, 태블릿, 데스크톱)별 평균 페이지 로드 시간

드릴다운 차트는 선택한 브라우저 유형을 버전별로 분류하기도 합니다. 예를 들어 Chrome 31, 32, 33 등입니다. 이렇게 하면 페이지 로드 타이밍 문제가 특정 브라우저 유형 또는 플랫폼과 관련이 있는지 또는 문제가 더 널리 퍼져 있는지 여부를 빠르게 결정할 수 있습니다.

## 특정 브라우저 세부 정보 보기 [#browser_details]

드릴다운 세부 정보에는 버전별 분석 및 평균 페이지 로드 시간이 포함됩니다.

<img
  title="browser_example.png"
  alt="browser_example.png"
  src={browserExample}
/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Browsers > (앱 선택) > Browsers > (브라우저 유형 선택):** 이 예는 선택된 특정 브라우저 유형(Firefox)을 보여줍니다. 가장 많은 수의 사용자가 버전 42.0을 사용하므로 범례에서 이 레이블(또는 다른 레이블)을 숨기면 다른 버전에 대한 세부 정보를 쉽게 검사할 수 있습니다.
</figcaption>

[평균 페이지 로드 시간](/docs/browser/new-relic-browser/page-load-timing/page-load-timing-process) 에 대한 색상으로 구분된 세그먼트는 다음과 같습니다.

* **요청 큐잉** (검은색): 웹 서버와 애플리케이션 코드 사이의 대기 시간입니다. 큰 숫자는 사용 중인 애플리케이션 서버를 나타냅니다.
* **웹 애플리케이션** (보라색): 애플리케이션 코드에서 소요된 시간.
* **네트워크** (갈색): 네트워크 대기 시간 또는 요청이 인터넷을 통해 왕복하는 데 걸리는 시간입니다.
* **DOM 처리** (노란색): 브라우저에서 HTML을 구문 분석 및 해석하고 자산을 검색합니다. 브라우저의 DOMContentLoaded 이벤트에 의해 측정됩니다.
* **페이지 렌더링** (파란색): 브라우저에서 HTML을 표시하고 인라인 JavaScript를 실행하고 이미지를 로드합니다. 브라우저의 Load 이벤트에 의해 측정됩니다.

**참고:** [복사/붙여넣기 방법을 사용하여 배포된](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser) 앱의 경우 브라우저에는 [네트워크](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process#h3-network) 시간에 웹 앱 및 대기열 시간이 포함됩니다.
