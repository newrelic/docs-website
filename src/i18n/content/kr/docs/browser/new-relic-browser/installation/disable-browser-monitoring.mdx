브라우저를 사용하여 애플리케이션을 모니터링하는 경우 앱 또는 특정 페이지에 대한 모니터링을 수동으로 비활성화할 수 있습니다. 끄는 이유 <InlinePopover type="browser"/>특정 페이지에는 다음이 포함됩니다.

* 일부 페이지는 모니터링과 관련이 없을 수 있습니다.
* 특정 페이지만 모니터링하려고 합니다.
* 문제 해결을 위해 브라우저 스크립트를 제거하려고 합니다.

## 특정 페이지/URL 모니터링 비활성화 [#disable]

특정 페이지 또는 URL에서 브라우저를 비활성화하려면 배포 방법에 따라 절차를 따르십시오.

## 도메인 모니터링 비활성화 [#domains]

특정 도메인 또는 하위 도메인에 대한 모니터링을 끄려면 브라우저 [도메인 조건](/docs/browser/new-relic-browser/configuration/monitor-or-block-specific-domains-subdomains) 을 업데이트하십시오.

## 브라우저 에이전트 비활성화 [#disable-agent]

[복사/붙여넣기 배포 방법](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app) 을 사용하고 브라우저 앱에 대한 데이터 수집을 중지하려면 모니터링하지 않을 페이지에서 브라우저 JavaScript 스니펫을 제거하기만 하면 됩니다. [APM 에이전트](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app) 를 사용하여 브라우저 에이전트를 설치한 경우 일부 브라우저 모니터링 기능을 개별적으로 끄거나 브라우저를 완전히 비활성화할 수 있습니다.

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) 으로 이동하여 Browser > (브라우저 앱 선택) > 설정 > Settings > Application settings** 합니다.

2. [Pro 계정 수준 기능](/docs/browser/new-relic-browser/getting-started/new-relic-browser) 만 비활성화하려면 **Lite** 를 선택합니다.

3. 브라우저 모니터링을 완전히 끄려면 **끄기** 를 선택합니다.

4. **애플리케이션 설정 저장을** 선택합니다.

5. 해당하는 경우 컴퓨터를 다시 시작하십시오.

   <InlinePopover type="apm"/>

   대리인.

브라우저 모니터링이 꺼져 있으면 에이전트는 모니터링 스크립트가 있는 페이지를 계측하지 않습니다.
