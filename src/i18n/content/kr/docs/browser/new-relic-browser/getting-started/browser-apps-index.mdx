[브라우저 모니터링](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) 을 시작하려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 으로 이동하여 페이지 상단에서 브라우저를 선택하십시오. 이미 앱을 추가한 경우 브라우저 애플리케이션 색인이 자동으로 나타납니다.

브라우저 색인 페이지는 브라우저에서 모니터링 중인 기존 앱 목록을 제공합니다. 추가 앱용 [브라우저 에이전트를 설치하는](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) 시작점이기도 합니다.

* [페이지 로드 시간](/docs/browser/new-relic-browser/dashboard-details/page-views-dashboard) , [SPA 응답 시간](/docs/browser/single-page-app-monitoring/use-spa-data/view-spa-data-new-relic-browser) 및 처리량(분당 페이지 수)
* [AJAX 응답 시간](/docs/browser/new-relic-browser/dashboard-details/ajax-dashboard) 및 처리량(분당 요청 수)
* [자바스크립트 오류](/docs/browser/new-relic-browser/dashboard-details/javascript-errors-dashboard)

## 브라우저 앱 이름 [#app-name]

[APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm) 에서도 모니터링하는 [앱은 APM의 앱 설정에서](/docs/apm/new-relic-apm/installation-configuration/naming-your-application) 이름을 가져옵니다. APM이 아닌 브라우저에서 모니터링하는 앱의 경우 해당 이름은 [브라우저를 설치할](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) 때 제공된 이름입니다.

* 브라우저 앱의 [요약 페이지](/docs/browser/new-relic-browser/dashboard-details/browser-overview-dashboard) 를 보려면: 브라우저 앱 색인에서 해당 이름을 선택합니다.
* 브라우저 앱의 이름을 변경하려면: 절차에 따라 원래 배포 방법을 기반으로 [앱 이름](/docs/browser/new-relic-browser/installation-configuration/rename-browser-apps) 을 변경합니다.

## 앱 상태 [#status]

색상으로 구분된 [상태 표시기](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status) 는 각 앱의 현재 상태를 보여줍니다.

* **녹색:** 브라우저 모니터링이 활성화되었으며 앱이 정상입니다.
* **노란색:** 경고 경고는 앱에 중요하지 않은 문제가 있음을 나타냅니다. 예를 들어 최종 사용자 [Apdex 점수](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) 또는 오류율이 관련 경고 정책에 대해 정의된 임계값을 초과했습니다.
* **빨간색:** 치명적 경고는 앱에서 문제가 에스컬레이션되었음을 나타냅니다.
* **회색:** 현재 앱에 대해 보고되는 데이터가 없습니다.

현재 상태에 대한 간략한 요약을 보려면 앱의 색상이 지정된 상태 표시줄 위로 마우스를 가져갑니다.
