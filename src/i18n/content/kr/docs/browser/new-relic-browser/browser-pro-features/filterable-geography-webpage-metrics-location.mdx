New Relic 브라우저 모니터링을 위해 필터링 가능한 지리 UI 페이지는 전 세계 도시, 지역 및 국가의 프론트 엔드 경험에 대한 색상으로 구분된 성능 정보와 함께 세계 보기를 제공합니다.

지도에는 지역별 사용자 데이터가 표시되며 장치 유형 정보와 함께 트래픽 및 오류 핫스팟을 시각화할 수 있습니다. 이 수준의 시각적 세부 정보는 IT 및 운영 전문가가 피어링 계약 및 CDN 사용에 대한 비즈니스 결정을 내리는 데 특히 유용합니다.

사용 가능한 데이터를 최대한 활용하는 방법에 대한 몇 가지 팁은 [필터 예](#filter) 를 참조하세요.

## 위치별 성능 데이터 보기 [#apdex-by-region]

필터링 가능한 지역 기능을 사용하려면 [Pro 또는 Pro+SPA 에이전트 유형](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent/#agent-types) 이 필요합니다.

<img
  title="Filterable geography UI"
  alt="Screenshot of Filterable geography UI in New Relic"
  src={screenBrowserFilterGeography}
/>

위치별로 성능 정보를 보거나 정렬하려면:

1. **[one.newrelic.com](https://one.newrelic.com) > Browser > (앱 선택) > 필터링 가능한 지역으로** 이동합니다.

2. 드롭다운을 사용하여 페이지 로드 또는 고유 세션 수, 다양한 프로세스의 평균 시간, 가장 큰 콘텐츠가 포함된 페인트, 첫 번째 입력 지연, 누적 레이아웃 이동 등과 같이 보고 싶은 성능 또는 사용 데이터 유형을 선택합니다.

3. [특정 지역](#functions) 으로 드릴다운하려면 지리 지도에서 아무 지역이나 마우스를 올리거나 선택하거나 지도 오른쪽에 있는 목록에서 실적이 가장 낮은 10개 위치 중 하나를 선택합니다.

4. 특정 실적 데이터를 보려면 유입경로 아래의 [속성](#filter) 중 하나를 선택하십시오.

   <Icon name="fe-filter"/>

   상.

5. 페이지 로드 성능 및 기록 성능에 대한 비교 데이터를 보려면 **맵 보기** 탭의 맵 아래에 있는 차트를 검토하십시오.

6. 데이터의 추가 차트를 보려면 **주요 속성 차트** 탭을 클릭하십시오.

**주요 속성 차트** 탭의 차트는 선택한 위치에 대한 비교 성능 세부 정보로 자동 새로 고침됩니다. 이 차트는 특정 위치, 브라우저, 장치, URL 또는 브라우저 버전과 관련된 성능 문제에 대한 통찰력을 제공합니다.

## 지도 기능 사용 [#functions]

## 성능 데이터 필터링 [#filter]

다양한 범주의 속성별로 데이터를 필터링하고 그룹화할 수 있습니다. 사용 가능한 속성 목록을 보려면 유입경로 <Icon name="fe-filter"/>상.

필터링된 결과는 표준 [지도 기능](#functions) 을 사용하여 추가 세부 정보를 탐색할 수 있는 지리적 지도에 나타납니다. 여기 몇 가지 예가 있어요.

## 수집된 데이터에 영향을 미치는 요소 [#collected-data]

페이지 로드 타이밍 정보를 캡처하기 위해 브라우저 모니터링 에이전트는 대부분의(전부는 아니지만) 브라우저가 지원하는 [Navigation Timing API](/docs/browser/new-relic-browser/browser-agent-apis/navigation-timing-api) 를 사용합니다. 브라우저가 Navigation Timing API를 지원하지 않는 경우 UI 차트 및 데이터 포인트는 DNS 조회, 연결, 보안 핸드셰이크 등에 대해 수신하는 네트워크 데이터만 표시할 수 있습니다. 이는 앱의 전체 트래픽의 100%가 아닐 수 있습니다.

또한 방화벽은 최종 사용자에 대해 수집된 지리적 데이터에 영향을 미칠 수 있습니다.
