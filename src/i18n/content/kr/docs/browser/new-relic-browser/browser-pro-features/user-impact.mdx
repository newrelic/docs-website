<InlinePopover type="browser" />의 <DNT>**User impact**</DNT> 페이지는 기술적 성능 문제가 사용자의 실제 경험에 어떤 영향을 미치는지 이해하는 데 도움이 됩니다. 이 페이지는 정량적 사용자 성능 지표와 정성적 행동 분석을 연결하여 인사이트에 대한 사용자 불만 패턴과 기술적 문제와의 상관 관계를 제공합니다.

사용자 영향 페이지는 크게 두 가지 분석 영역으로 구성됩니다.

* <DNT>Frustration metrics</DNT>
* <DNT>Performance risk</DNT>

각각은 사용자 경험 문제와 그 기술적 근본 원인을 이해하는 데 있어 서로 다른 관점을 제공합니다.

**필수 조건:**

* 사용자 불편 신호 및 성능 위험 데이터를 수집하려면 브라우저 에이전트 버전 1.269 이상이 필요합니다.
* 이 페이지에 링크된 세션 리플레이에 액세스하려면 [세션 리플레이를](/docs/browser/browser-monitoring/browser-pro-features/session-replay/get-started/) 활성화해야 합니다.

## 접근 권한 사용자 영향 분석 [#access]

사용자 영향 페이지를 보려면 다음을 참조하세요.

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app)** &gt; **Users** &gt; **User impact**</DNT> 으)로 이동합니다.
2. 탭에서 <DNT>**Frustration metrics**</DNT> 또는 <DNT>**Performance risk**</DNT> 중 하나를 선택하세요.

<Callout variant="tip">
  [페이지 조회](/docs/browser/new-relic-browser/browser-pro-features/page-views-examine-page-performance) 페이지에서 <DNT>**Analyze frustration signals**</DNT> 또는 <DNT>**Analyze performance risk**</DNT> 링크를 클릭하여 바로 이 페이지로 이동할 수도 있습니다.
</Callout>

## 좌절감을 분석하다 [#frustration-metrics]

&apos;좌절감&apos; 탭은 사용자가 작업을 완료하는 데 있어 혼란, 짜증 또는 어려움을 나타내는 행동 지표에 중점을 둡니다. 이 분석을 통해 사용자가 가장 어려움을 겪는 부분을 파악하고 실제 사용자 불편 사항을 기반으로 개선 사항의 우선순위를 정할 수 있습니다.

&apos;좌절감&apos; 탭에는 다음 섹션이 포함되어 있습니다.

* 분노 클릭: 사용자가 반응이 없거나 손상된 UI 요소에 대한 불만으로 인해 빠르고 반복적으로 클릭하는 행동을 보이는 페이지 및 요소를 식별합니다.
* 가장 많은 영향을 받는 페이지: 가장 많은 불편 신호를 발생시키는 특정 페이지를 강조 표시하여 영향력이 큰 영역에 집중할 수 있도록 합니다.
* 영향을 받은 페이지 요소: 사용자가 가장 불만을 느끼는 특정 UI 요소가 무엇인지 세분화된 인사이트를 제공합니다.

## 성능 지표 분석 [#performance-impact]

성능 위험 탭에서는 측정 가능한 성능 지표와 사용자 환경 문제와의 상관 관계에 중점을 두고 기술적 성능 문제가 사용자 세션에 어떤 영향을 미치는지 조사합니다.

성능 위험 탭에는 다음 섹션이 포함됩니다.

* Web Vitals: 열악한 코어 웹 바이탈(LCP, INP, CLS)이 사용자 세션 및 경험에 얼마나 영향을 미치는지 분석합니다.
* 오류 분석: 자바스크립트 오류 패턴과 사용자 세션에 미치는 영향을 조사합니다.
* 가장 큰 영향을 받는 페이지: 성능 관련 사용자 영향이 가장 큰 페이지를 식별합니다.
* 지리적 분석: 다양한 지리적 위치에 따른 성과 위험을 조사하여 지역적 문제를 파악합니다.
