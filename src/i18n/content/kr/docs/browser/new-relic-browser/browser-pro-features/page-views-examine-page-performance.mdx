<InlinePopover type="browser" />의 <DNT>**Page views**</DNT> 페이지는 인사이트를 사이트 페이지의 성능과 사용자 환경에 제공합니다. 정렬 옵션을 사용하면 페이지 로드 시간 비율, 평균 페이지 로드 시간 또는 처리량에 대한 세부 정보를 찾을 수 있습니다. 또한 사용자 좌절 신호 및 성능 영향 지수를 보고, 세션 리플레이에 액세스하고, 조회한 상위 웹페이지에 대한 자세한 정보를 탐색할 수도 있습니다.

당사의 브라우저 기능에는 [단일 페이지 앱(SPA) 모니터링이](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring) 포함됩니다. SPA 모니터링을 선택한 경우 SPA <DNT>**Page views**</DNT> UI가 달라집니다. 원하는 보기(SPA 또는 표준 브라우저) 간에 전환할 수 있습니다.

## 요약 정보 보기 [#dashboard]

브라우저 모니터링은 페이지 보기를 [URL](/docs/browser/new-relic-browser/getting-started/url-whitelists-grouping-browser-metrics)로 나열합니다. 이렇게 하면 특정 프런트엔드 페이지 보기를 쉽게 식별할 수 있습니다. 예: `/browse/plans`.

<img title="Page views UI overview" alt="A screenshot of the Page views UI in New Relic" src="/images/browser_screenshot-full_pageviews-overview.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT>: 페이지 로드 시간, 처리량, top 프로세서 및 트레이스에 대한 자세한 정보를 제공하는 페이지입니다.
</figcaption>

최종 사용자가 앱에 대해 보고 있는 페이지에 대한 자세한 정보를 보려면:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT> 으)로 이동합니다.
2. 해당하는 경우 SPA 또는 표준 브라우저 앱에 대한 페이지 보기를 전환합니다.
3. 페이지 로드 타이밍 정보에 대한 보기 유형을 [차트(기본값), 히스토그램 또는 백분위수](/docs/apm/apm-ui-pages/monitoring/response-time-chart-types-apm-browser/) 로 선택합니다.

## 사용자 영향 지표 이해 [#user-impact-metrics]

페이지 조회 페이지에는 성능 문제가 사용자 경험에 미치는 영향을 파악하는 데 도움이 되는 사용자 영향 창이 포함되어 있습니다.

### 좌절감 신호 [#frustration-signals]

<DNT>**Frustration signals**</DNT> 은 선택한 기간 동안 불만 표시기가 포함된 사용자 세션의 비율을 나타냅니다. 좌절감을 나타내는 신호는 다음과 같습니다.

* **분노 클릭**: 사용자가 1초 이내에 동일한 요소를 4회 이상 빠르게 클릭하는 행위
* **데드 클릭**: 2초 이내에 감지 가능한 변화가 발생하지 않는 클릭
* **오류 클릭**: 2초 이내에 JavaScript 오류가 발생하는 클릭

### 성능 위험 [#performance-impact]

<DNT>**Performance risk**</DNT> 은 선택한 기간 동안 성능 문제로 영향을 받은 사용자 세션의 비율을 나타냅니다. 이 기능은 자바스크립트 오류에 대한 처리되지 않은 예외와 애플리케이션 오류를 표시합니다.

### 세션 리플레이 샘플링 고려 사항 [#sampling-considerations]

사용자 영향 카드는 세션에 따라 조정됩니다. 리플레이 설정:

<CollapserGroup>
  <Collapser id="sr-enabled" title="세션 리플레이가 표준 샘플링으로 활성화되었습니다.">
    세션 리플레이가 기본 갱값, 경계(일반 샘플링의 경우 10%, 오류 샘플링의 경우 100%) 이상의 샘플링 속도로 활성화되면 두 카드 모두 세션 리플레이 개수와 재생 세션에 대한 직접 링크가 포함된 전체 지표를 표시합니다.
  </Collapser>

  <Collapser id="sr-low-sampling" title="낮은 샘플링으로 활성화된 세션 리플레이">
    샘플링 속도가 권장 속도보다 낮으면 카드에 샘플링 속도를 높이라는 알림이 표시됩니다. &quot;샘플링 속도 높이기&quot; 링크를 클릭하여 브라우저 애플리케이션 설정으로 이동하세요.
  </Collapser>

  <Collapser id="sr-disabled" title="세션 리플레이가 비활성화되었습니다.">
    세션 리플레이가 비활성화된 경우, 카드에는 리플레이 데이터 없이 세션 횟수만 표시됩니다. <DNT>See where your users are struggling</DNT> 과 기능을 활성화하기 위한 <DNT>Turn on session replays</DNT> 링크가 포함된 알림이 표시됩니다.
  </Collapser>
</CollapserGroup>

## 성능 세부 정보 검사 [#view\_details][#view_details]

페이지 보기 요약에서 URL을 클릭하면 페이지 로드, [APM](/docs/apm/apm-ui-pages/features/request-queuing-tracking-front-end-time/) 또는 [분산 추적](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui) 의 프런트엔드 및 백엔드 타이밍, 처리량, [브라우저 유형](/docs/browser/new-relic-browser/additional-standard-features/browsers-problem-patterns-type-or-platform) 및 [세션 추적](/docs/browser/browser-monitoring/browser-pro-features/session-traces-explore-webpages-life-cycle) 에 대한 성능 세부 정보를 볼 수 있습니다.

<img title="Page view details" alt="A screenshot of the page view details UI in New Relic" src="/images/browser_screenshot-full_page-load-performance.webp" />

여기에서 다음을 수행할 수도 있습니다.

* 어제 및 지난 주와 페이지 로드 및 처리량 성능을 비교합니다.
* 웹 트랜잭션에 대한 앱 응답 시간에 대한 응답 시간을 봅니다.
