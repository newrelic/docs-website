브라우저 모니터링의 **페이지 보기** 페이지는 사이트 페이지의 성능에 대한 통찰력을 제공합니다. 정렬 옵션을 사용하면 페이지 로드 시간 비율, 평균 페이지 로드 시간 또는 처리량에 대한 세부 정보를 찾을 수 있습니다. 조회된 상위 웹 페이지, 타이밍 및 처리량 세부 정보, 브라우저 추적 또는 세션 추적에 대한 링크에 대한 자세한 정보도 제공됩니다.

당사의 브라우저 기능에는 [단일 페이지 앱(SPA) 모니터링이](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring) 포함됩니다. SPA 모니터링을 선택한 경우 SPA **페이지 보기** UI가 달라집니다. 원하는 보기(SPA 또는 표준 브라우저) 간에 전환할 수 있습니다.

## 요약 정보 보기 [#dashboard]

브라우저 모니터링은 페이지 보기를 [URL](/docs/browser/new-relic-browser/getting-started/url-whitelists-grouping-browser-metrics) 로 나열합니다. 이렇게 하면 특정 프런트 엔드 페이지 보기를 쉽게 식별할 수 있습니다. 예: `/browse/plans` .

  title="Page views UI overview"
  alt="A screenshot of the Page views UI in New Relic"

/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (앱 선택) > Page views** : 이 페이지는 페이지 로드 시간, 처리량, 상위 트랜잭션 및 추적에 대한 자세한 정보를 제공합니다.
</figcaption>

최종 사용자가 앱에 대해 보고 있는 페이지에 대한 자세한 정보를 보려면:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (앱 선택) > Page views** 로 이동합니다.
2. 해당하는 경우 SPA 또는 표준 브라우저 앱에 대한 페이지 보기를 전환합니다.
3. 페이지 로드 타이밍 정보에 대한 보기 유형을 [차트(기본값), 히스토그램 또는 백분위수](/docs/apm/apm-ui-pages/monitoring/response-time-chart-types-apm-browser/) 로 선택합니다.

## 성능 세부 정보 검사 [#view_details]

페이지 보기 요약에서 URL을 클릭하면 페이지 로드, [APM](/docs/apm/apm-ui-pages/features/request-queuing-tracking-front-end-time/) 또는 [분산 추적](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui) 의 프런트엔드 및 백엔드 타이밍, 처리량, [브라우저 유형](/docs/browser/new-relic-browser/additional-standard-features/browsers-problem-patterns-type-or-platform) 및 [세션 추적](/docs/browser/browser-monitoring/browser-pro-features/session-traces-explore-webpages-life-cycle) 에 대한 성능 세부 정보를 볼 수 있습니다.

<img
  title="Page view details"
  alt="A screenshot of the page view details UI in New Relic"
  src={browserPageLoadPerformance}
/>

여기에서 다음을 수행할 수도 있습니다.

* 어제 및 지난 주와 페이지 로드 및 처리량 성능을 비교합니다.
* 웹 트랜잭션에 대한 앱 응답 시간에 대한 응답 시간을 봅니다.
