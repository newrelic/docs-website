New Relic의 **브라우저** UI는 최종 사용자가 사용하는 브라우저(예: Google Chrome, Mozilla Firefox, Microsoft Internet Explorer, Apple Safari, Microsoft Edge)에 따라 앱 사용 경험에 대한 정보를 제공합니다. 이 페이지에는 다음이 포함됩니다.

* 처리량(분당 페이지 조회수 또는 `ppm` ) 및 페이지 로드 시간별 상위 브라우저
* 최대 내용 페인트(LCP), 첫 번째 입력 지연(FID), 누적 레이아웃 전환 점수(CSL) 등을 포함한 핵심 Web Vitals 데이터
* [JS 오류 UI](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors/) 의 자세한 내용에 대한 링크가 포함된 오류별 상위 JavaScript 오류
* 기기 유형별 필터링 가능한 데이터(모바일, 태블릿, 데스크톱)

드릴다운 차트는 선택한 브라우저 유형을 버전별로 분류하기도 합니다. 예를 들어 Chrome 14, 47, 72 등 이렇게 하면 페이지 로드 타이밍 문제가 특정 브라우저 유형 또는 장치와 관련이 있는지 또는 문제가 더 널리 퍼져 있는지 여부를 빠르게 확인할 수 있습니다.

브라우저 정보 보기: **[one.newrelic.com](https://one.newrelic.com) > Browser > (앱 선택) >Browsers** 로 이동합니다.

여기에서 다음과 같은 질문에 답하는 데 도움이 되는 데이터를 볼 수 있습니다.

* [페이지 로드 타이밍 프로세스](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 의 스파이크는 어디에 있습니까?
* 시간이 지남에 따라 반복되는 패턴이 있습니까?
* 평균 페이지 로드 시간과 처리량이 허용 가능한 범위 내에 있습니까?
* 특정 브라우저 버전과 관련된 문제입니까?
* 상위 [JavaScript 오류](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors/) 는 코드의 문제, 잠재적인 보안 취약성 또는 기타 문제를 나타냅니까?
* 이 브라우저 유형의 경우 이러한 성능 문제가 첫 번째 페인트, 첫 번째 콘텐츠가 포함된 페인트, 가장 큰 콘텐츠가 포함된 페인트, 창 로드 및 첫 번째 상호 작용에 대한 Core Web Vitals 점수에 어떤 영향을 줍니까?
* 이러한 문제는 다른 장치 유형(데스크톱, 모바일, 태블릿) 또는 다른 브라우저 유형과 어떻게 비교됩니까?

이 데이터를 사용하여 문제를 해결할 때 다른 사람들과 협력하십시오. 예를 들어 이 데이터를 이미지 또는 차트 URL로 공유하거나 대시보드에 추가합니다. 향후 문제를 방지하기 위해 경고를 생성할 수 있습니다.
