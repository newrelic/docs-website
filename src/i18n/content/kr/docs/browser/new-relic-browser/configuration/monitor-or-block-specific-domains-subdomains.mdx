[브라우저](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) 를 사용하면 브라우저 앱이 모니터링하는 도메인 또는 하위 도메인을 선택할 수 있습니다. 이는 중요하다고 생각하는 애플리케이션 활동으로만 모니터링을 제한하는 데 유용합니다. 예를 들어 [복사/붙여넣기 방법](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) 을 사용하여 브라우저 앱을 배포하는 경우 도메인 조건을 설정하여 [브라우저 모니터링 라이선스 키 및 앱 ID](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id) 의 보안을 보호할 수 있습니다.

<Callout variant="tip">
  도메인 조건은 보고된 페이지의 URL을 기반으로 데이터만 필터링합니다. 도메인 조건 필터는 New Relic에 보고된 AJAX 데이터에는 적용되지 않습니다. `AjaxRequest` 이벤트를 필터링하려면 [구성 문서를 읽어보세요](/docs/browser/new-relic-browser/configuration/filter-ajax-request-events/) .
</Callout>

## 도메인 조건 구성 [#configure]

도메인 조건 설정을 보거나 편집하려면:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (앱 선택) > Settings > Application settings > Domain conditions**으로 이동합니다.

2. 도메인 조건이 없으면 도메인 조건 **활성화** 를 선택합니다. 조건이 있는 경우 조건 **편집** 을 선택합니다.

3. 허용된 도메인 목록을 작성하거나 차단/거부된 도메인 목록을 작성하는 옵션을 선택합니다.

4. 도메인 조건을 선택합니다(최대 10개 조건).

5. **시작** 조건으로 도메인 조건을 선택한 다음 하위 도메인이 있는 URL을 입력합니다(최대 10개 조건).

   <figcaption>
     **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (브라우저 앱 선택) > Settings > Domain conditions:** 이 기능을 사용하여 중요한 앱 활동으로만 모니터링을 제한하거나 브라우저 라이선스 키 및 앱 ID에 대한 보안을 강화할 수 있습니다.

6. 도메인 조건 설정을 검토하고 확인합니다.

미리보기 텍스트 상자는 설정한 조건과 일치하는 도메인을 실시간으로 표시합니다. 또한 제안된 설정이 데이터에 미칠 영향에 대한 예상도 볼 수 있습니다.

## 도메인 조건 끄기 [#disable]

도메인 조건을 비활성화하고 모든 도메인 모니터링으로 돌아가려면:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (앱 선택) > Settings > Domain conditions** 으로 이동합니다.
2. **도메인 조건 활성화** 옵션을 **끄기** 로 설정합니다.
