[에이전트 버전 1211](https://docs.newrelic.com/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/) 이상에서는 페이지의 모든 네트워크 요청이 `AjaxRequest` 이벤트로 기록됩니다. **애플리케이션 설정** 페이지에서 거부 목록 구성 옵션을 사용하여 이벤트를 기록하는 요청을 필터링할 수 있습니다.

## 거부 목록 사용

다음 세 가지 방법으로 요청을 차단할 수 있습니다.

* 모든 `AjaxRequest` 이벤트의 기록을 차단하려면 별표 `*` 를 와일드카드로 추가하십시오.
* 도메인에 대한 `AjaxRequest` 이벤트 기록을 차단하려면 도메인 이름만 입력하세요. 예시: `example.com`
* 특정 도메인 및 경로에 대한 `AjaxRequest` 이벤트의 기록을 차단하려면 도메인 및 경로를 입력하세요. 예시: `example.com/path`

URL의 프로토콜, 포트, 검색 및 해시는 거부 목록에서 무시됩니다.

## 거부 목록에 액세스

애플리케이션이 이벤트 생성에서 필터링할 URL의 거부 목록을 업데이트하려면 앱 설정 페이지로 이동하십시오.

1. **[one.newrelic.com > All capabilties](https://one.newrelic.com/all-capabilities) > Browser > (앱 선택) > Settings > Application settings** 으로 이동합니다.

2. **Ajax 요청 거부 목록** 아래에서 앱에 적용할 필터를 추가합니다.

3. **애플리케이션 설정 저장** 을 선택하여 에이전트 구성을 업데이트합니다.

4. [브라우저 에이전트를 다시 배포합니다](/docs/browser/new-relic-browser/installation/upgrade-browser-agent) (연결된

   <InlinePopover type="apm"/>

   에이전트 또는 복사/붙여넣기 브라우저 설치 업데이트).

## 확인

추가한 필터가 예상대로 작동하는지 확인하려면 필터와 일치하는 `AjaxRequest` 이벤트에 대해 NRQL 쿼리를 실행하세요.

```sql
FROM AjaxRequest SELECT * WHERE requestUrl LIKE `%example.com%`
```
