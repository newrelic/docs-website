브라우저에 추가하는 데 사용한 [배포 옵션](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) 에 따라 적절한 절차에 따라 브라우저 앱의 이름을 바꿉니다.

## APM에서 모니터링하는 앱 이름 바꾸기 [#apm-app]

이미 APM에서 모니터링 중인 브라우저에 앱을 추가한 경우 브라우저에 표시되는 앱 이름은 <InlinePopover type="apm"/>UI. 앱 이름을 바꾸려면 다음 옵션 중 하나를 사용하십시오.

## JavaScript 스니펫을 통해 앱 이름 바꾸기 [#javascript]

이 정보는 브라우저 에이전트의 JavaScript 스니펫을 다음에 삽입한 경우에 적용됩니다.

* [복사/붙여넣기 방식](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser#copy-paste-app) 을 통해 배포된 APM에서 모니터링하는 앱
* 단일 페이지 앱( [SPA](/docs/browser/single-page-app-monitoring/get-started/add-apps-single-page-app-monitoring) )
* APM에서도 모니터링하지 않는 모든 독립형 브라우저 앱

기존 앱에서 JavaScript 스니펫을 직접 편집할 수 없습니다. 대신 다른 이름으로 앱을 사용하려면 다음 단계를 따르세요.

1. [복사/붙여넣기 방법을 사용하여 새 앱을 만들려면](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser#copy-paste-app) 표준 절차를 따르십시오. 새 이름을 제공하고 JavaScript 스니펫을 생성하십시오.
2. 새 JavaScript 스니펫을 복사합니다.
3. 웹 페이지에서 원래 앱의 JavaScript를 새 앱 이름에 대해 생성된 JavaScript로 바꿉니다.
4. 독립 실행형 앱에 [여러 이름](/docs/agents/manage-apm-agents/app-naming/use-multiple-names-app) 이 있는 경우 구성 스크립트 [의 사용자 지정 앱 ID를 새 앱에 복사합니다](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id) .

[데이터 보존 일정](/docs/accounts-partnerships/accounts/account-billing-usage/data-retention) 에 따라 New Relic은 이전 이름으로 수집된 이력 데이터를 유지하고 새 이름으로 데이터를 수집 및 집계합니다.

## New Relic REST API(v2)를 통해 앱 이름 바꾸기 [#rest-api]

이 정보는 APM에서도 모니터링하는 브라우저 앱에만 적용됩니다. 복사/붙여넣기 방법을 사용하여 브라우저의 JavaScript 스니펫을 앱에 삽입한 경우 REST API를 통해 앱 이름을 변경할 수 없습니다.

REST API 또는 [API Explorer](https://rpm.newrelic.com/api/explore/applications/update) [를 통해 APM 앱 별칭을 변경하려면](/docs/apis/rest-api-v2/application-examples-v2/change-alias-your-application-v2) 표준 절차를 따르십시오. New Relic은 브라우저를 포함한 New Relic UI의 모든 위치에서 앱의 표시 이름을 자동으로 변경합니다.
