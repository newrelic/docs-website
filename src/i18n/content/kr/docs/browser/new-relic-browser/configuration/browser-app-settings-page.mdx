### 브라우저 세션 저장소 [#browser-session-storage]

브라우저 에이전트 내에서 세션 트레이스와 세션 리플레이 데이터를 수집하기 위한 제어 기능을 통합했습니다. 이제 이 설정 페이지의 <DNT>**Access browser local storage**</DNT> 토글에서 모든 세션 수준 데이터를 직접 관리할 수 있습니다.

{/* * **Access browser local storage toggle**: This unified control determines how session-level data (including session IDs and other session-level attributes) is collected.
    * **When turned ON (default)**: The New Relic browser agent uses the browser's persistent storage mechanisms to collect session trace and session replay data.
    * **When turned OFF**: The New Relic browser agent immediately halts its use of the browser's persistent storage. This action prevents the collection of all session trace and session replay data, giving you direct control over the browser's handling of session-related information and enhancing your ability to meet data privacy and compliance requirements. */}

<Callout variant="important">
  브라우저 에이전트 버전 1.303.0부터 개인정보 보호 및 동의 기능을 간소화하기 위해 **Access local storage** \[로컬 저장소 액세스] 토글이 UI에서 제거됩니다.

  * 브라우저 에이전트 데이터 수집 및 사용자 동의 관리에 권장되는 방법은 동의 모드 설정을 이용하는 것입니다. 자세한 내용은 [동의서를](/docs/browser/new-relic-browser/configuration/consent-mode/) 참조하십시오.

  * 현재 **Access local storage** \[로컬 저장소 액세스] 토글이 `OFF` 으로 설정되어 있는 경우 동의 모드, 세션 리플레이 및 세션 트레이 토글도 비활성화되어 있음을 알 수 있습니다. 이러한 기능을 활성화하려면 [NerdGraph API를](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-browser-application) 통해 `cookie_enabled` 속성을 `true` 로 설정해야 합니다. 이 작업을 수행하면 UI 의 토글이 다시 활성화되어 동의 모드, 세션 리플레이 및 세션 트레이스를 올바르게 구성할 수 있습니다.
</Callout>

## 브라우저 API [#apis]

또한 브라우저 에이전트 계측을 사용자 정의하여 사용자 지정 추적 및 이벤트를 캡처하고, 필요에 맞게 특정 오류를 보고하고, 최적의 성능을 위해 데이터 수집 타이밍을 미세 조정할 수 있습니다. 시작하려면 [브라우저 API 사용을](/docs/browser/new-relic-browser/browser-apis/using-browser-apis/) 참조하세요.
