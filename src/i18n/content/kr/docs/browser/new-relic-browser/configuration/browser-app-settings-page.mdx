### 브라우저 세션 저장소 [#browser-session-storage]

브라우저 에이전트 내에서 세션 트레이스와 세션 리플레이 데이터를 수집하기 위한 제어 기능을 통합했습니다. 이제 이 설정 페이지의 <DNT>**Access browser local storage**</DNT> 토글에서 모든 세션 수준 데이터를 직접 관리할 수 있습니다.

/\* \* \*\*브라우저 로컬 저장소 토글 액세스\*\*: 이 통합 제어는 세션 수준 데이터(세션 ID 및 기타 세션 수준 속성 포함)가 수집되는 방식을 결정합니다. \* \*\*켜진 경우(기본값)\*\*: 뉴렐릭 브라우저 에이전트는 브라우저의 영구 저장 메커니즘을 사용하여 세션 트레이스 및 세션 리플레이 데이터를 수집합니다. \* \*\*끄면\*\*: 뉴렐릭 브라우저 에이전트가 브라우저의 영구 저장소 사용을 즉시 중단합니다. 이 작업을 수행하면 모든 세션 트레이 및 세션 리플레이 데이터 수집이 방지되어 브라우저에서 세션 관련 정보를 처리하는 방식을 직접 제어할 수 있으며, 데이터 개인 정보 보호 및 규정 준수 요구 사항을 충족하는 능력이 향상됩니다. \*/

<Callout variant="important">
  브라우저 에이전트 버전 1.303.0부터 개인정보 보호 및 동의 기능을 간소화하기 위해 **Access local storage** \[로컬 저장소 액세스] 토글이 UI에서 제거됩니다.

  * 브라우저 에이전트 데이터 수집 및 사용자 동의를 관리하는 데 권장되는 방법은 동의 모드 구성을 사용하는 것입니다. 우리는 모든 사용자에게 이 새로운 기능을 채택할 것을 강력히 권장합니다. 자세한 내용은 [동의 문서를](/docs/browser/new-relic-browser/configuration/consent-mode/) 참조하세요.

  * 현재 **Access local storage** \[로컬 저장소 액세스] 토글이 `OFF` 으로 설정되어 있는 경우 동의 모드, 세션 리플레이 및 세션 트레이 토글도 비활성화되어 있음을 알 수 있습니다. 이러한 기능을 활성화하려면 [NerdGraph API를](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-browser-application) 통해 `cookie_enabled` 속성을 `true` 로 설정해야 합니다. 이 작업을 수행하면 UI 의 토글이 다시 활성화되어 동의 모드, 세션 리플레이 및 세션 트레이스를 올바르게 구성할 수 있습니다.
</Callout>

/\* &lt;DNT&gt;\*\*애플리케이션 설정\*\*&lt;/DNT&gt; 페이지로 이동하려면: 1. &lt;DNT&gt;\*\*\[one.newrelic.com &gt; All capabilities]\(https\://one.newrelic.com/all-capabilities)&gt; Browser\*\* に移動して、アプリを選択します。 2. 왼쪽 탐색 메뉴에서 &lt;DNT&gt;\*\*Settings\*\*&lt;/DNT&gt; 까지 아래로 스크롤하고 &lt;DNT&gt;\*\*Application settings\*\*&lt;/DNT&gt;을 클릭합니다. \*/

## 브라우저 API [#apis]

또한 브라우저 에이전트 계측을 사용자 정의하여 사용자 지정 추적 및 이벤트를 캡처하고, 필요에 맞게 특정 오류를 보고하고, 최적의 성능을 위해 데이터 수집 타이밍을 미세 조정할 수 있습니다. 시작하려면 [브라우저 API 사용을](/docs/browser/new-relic-browser/browser-apis/using-browser-apis/) 참조하세요.
