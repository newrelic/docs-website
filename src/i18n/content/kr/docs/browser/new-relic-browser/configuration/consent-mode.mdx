## 이 기능은 무슨 역할을 하나요?

이를 통해 웹이 에이전트에 의해 데이터 모니터를 수집, 전송 및 처리하는 방법에 대한 사용자의 개인 정보 보호 정책 옵트인을 통합할 수 있습니다. 이 기능을 활성화하면 사용자가 브라우저 에이전트가 네트워크를 통해 데이터를 전송하는 데 동의할지 여부를 결정할 수 있습니다.

브라우저 에이전트의 init 개체에 `browser_consent_mode` 이라는 새로운 속성이 추가되었습니다. 이 설정은 에이전트에서 동의 모드를 사용할지 여부를 제어합니다. 브라우저 Lite, 브라우저 Pro 또는 Pro + SPA 에이전트 [버전 1.303.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.303.0) 이상에서 사용할 수 있습니다.

## 어떻게 사용하나요?

이 기능을 사용하려면 앱 설정 페이지를 방문하여 `"Turn on consent"` 토글을 켜야 합니다. 이 설정은 기본적으로 꺼져 있습니다.

NerdGraph를 사용하면 설정을 켜고 끌 수 있습니다. 이렇게 하려면 `browser_consent_mode` 속성에 대해 `browserSettings` graphQL 개체에 대한 뮤테이션 쿼리를 실행하고 `enabled` `true` 또는 `false` 로 설정합니다.

웹 애플리케이션에 개인정보 보호정책이나 옵트인 모달을 통합한 경우 사용자가 다음 중 하나를 클릭하면

1. “네, 동의합니다.”

   `consent(accept?: boolean)` 이라는 새로운 API를 사용해야 합니다. [동의() API](/docs/browser/new-relic-browser/browser-apis/consent) 호출할 때, accept 시범은 기본적으로 true입니다.

   **autoStart: true**를 사용하는 브라우저는 동의가 제공된 후 뉴렐릭 기능으로 데이터를 보냅니다.

   **autoStart: false** 가 설정된 브라우저 기능은 [start() API](/docs/browser/new-relic-browser/browser-apis/start) 함수를 사용하여 호출해야 네트워크를 통해 데이터를 전송하는 지연된 에이전트 기능을 시작할 수 있습니다.

2. &quot;아니요, 동의하지 않습니다.&quot;

   기본값이 true이므로 false 인수를 기록해 두고 `consent(false)` 호출해야 합니다. 에이전트는 네트워크를 통해 해당 사용자에 대한 데이터를 전송하지 않습니다.

## 추가 고려 사항

* `browser_consent_mode` 활성화된 경우 최대 \~30초와 최대 1MB의 데이터가 최종 사용자의 브라우저에 버퍼링되고 사용자가 동의를 선택하고 동의 API 호출될 때까지 뉴렐릭으로 전송되지 않습니다. 이러한 제약으로 인해 동의 API 호출될 때 일부 데이터가 삭제되어 뉴렐릭으로 전송되지 않을 수 있습니다.

* [세션 저장이](/docs/browser/new-relic-browser/configuration/browser-app-settings-page/#browser-session-storage) 활성화된 경우 동의 API로 전달된 결정은 여러 페이지 간에 유지됩니다.

* [세션 저장소가](/docs/browser/new-relic-browser/configuration/browser-app-settings-page/#browser-session-storage) 비활성화된 경우, 모든 페이지 로드 시 동의 API를 호출해야 합니다.
