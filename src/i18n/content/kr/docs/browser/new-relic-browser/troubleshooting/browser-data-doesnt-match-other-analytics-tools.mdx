## 문제

<InlinePopover type="browser" />에서는 다른 분석 도구에서 사용하는 데이터와는 다른 데이터를 볼 수 있습니다. 일반적인 증상은 다음과 같습니다.

* 브라우저는 예상보다 적은 [페이지 조회수](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity) 를 기록합니다.
* 사용자에 대한 브라우저의 [지리적 분포](/docs/browser/new-relic-browser/additional-standard-features/geography-webpage-performance-location) 는 예상 사용자 기반의 위치와 다릅니다.
* 브라우저는 다른 곳에서 볼 수 있는 것과 다른 수의 [활성 세션](/docs/browser/new-relic-browser/page-load-timing-resources/session-tracking) 을 표시합니다.

## 해결책

브라우저 모니터링을 위해 다음 문제 해결 단계를 따르십시오.

1. 브라우저 모니터링이 애플리케이션의 모든 페이지에 제대로 설치되었는지 확인하십시오.
2. JavaScript 스니펫이 누락되었거나 사용자가 New Relic에 데이터를 보낼 수 없는 경우 [브라우저 모니터링 설치 문제 해결](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation) 단계를 따르십시오.
3. 단일 페이지 스타일의 애플리케이션이 있고 경로 변경을 뷰로 볼 것으로 기대하는 경우, 초기 페이지 로드와 경로 변경에 대한 통합된 뷰를 제공하는 [브라우저 SPA 모니터링을](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring) 사용하는 것을 고려하세요.

브라우저 모니터링이 제대로 작동하고 있다면 뉴렐릭이 다른 분석 도구와는 다르게 정보를 수집하고 제공하기 때문에 다른 결과가 보일 수 있습니다. 매칭의 경우, `JavascriptErrors` 과 `AjaxRequest` 은 개별적으로 모니터링되며 해당 데이터는 에이전트가 이를 관찰하는 시점과 수집할 수 있거나 수집하기로 결정한 내용에 따라 달라집니다. `BrowserInteraction` 의 경우 자체 로그 로그를 사용하고, `PageViewTiming` 및 `PageView` 아래의 Vitals 및 인덱스의 경우 Chrome 팀이 소유한 web-vitals 라이브러리가 해당 데이터의 소스입니다.

데이터 수집 방법에 대한 자세한 내용은 다음을 참조하세요.

* [페이지 로드 타이밍 프로세스](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process)
* [브라우저 모니터링을 위한 계측](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-browser-monitoring)
* [SPA 데이터 수집 이해](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection)
