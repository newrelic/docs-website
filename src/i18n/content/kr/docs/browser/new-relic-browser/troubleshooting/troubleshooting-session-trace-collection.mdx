## 문제

브라우저 애플리케이션에 대한 [세션 추적](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle) 이 표시되지 않습니다.

## 해결책

애플리케이션이 [APM으로 계측되고](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation) [Pro 브라우저 에이전트와 연결된 기타 기능](/docs/browser/new-relic-browser/browser-pro-features) 이 작동하는 경우 다음 단계를 따르십시오.

1. [응용 프로그램에 여러 이름을 사용하고](/docs/apm/new-relic-apm/installation-configuration/using-multiple-names-app) 있는지 확인하십시오. 세션 추적은 기본 애플리케이션에만 기록됩니다.

2. 여러 이름을 사용하는 경우

   <InlinePopover type="apm"/>

   앱이 먼저 나열되는지 확인하기 위해 애플리케이션 이름을 지정합니다.

3. 애플리케이션이 먼저 나열되지 않으면 대신 먼저 나열된 애플리케이션 이름에서 세션 추적을 찾으십시오.

세션 추적 데이터가 표시되지 않는 다른 이유는 다음과 같습니다.

* 최종 사용자는 [Resource Timing API](http://www.w3.org/TR/resource-timing/) 를 지원하는 브라우저를 사용하지 않습니다.
* 최종 사용자는 `bam.nr-data.net` 의 `/resources` 엔드포인트에 데이터를 게시할 수 없습니다.
* 최종 사용자의 클라이언트는 필요한 [세션 쿠키를](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking) 허용하지 않습니다.
