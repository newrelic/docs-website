## 문제

사이트에 유효하지 않은 URL에 대해 Google 검색 대시보드에 404 오류가 표시됩니다. 이러한 경로에 대한 웹 서버 로그에 404 오류가 표시될 수도 있습니다. 사이트를 검색한 후 HTML에 삽입되는 브라우저 에이전트 로더 스크립트에 사이트 경로가 문자열로 존재한다는 것을 알게 됩니다.

## 원인

브라우저 에이전트는 코드에서 문자열 리터럴을 사용하지만 로더 문자열을 문자열 리터럴로 래핑할 수 있는 코드의 가능한 문제를 방지하기 위해 빌드 프로세스 중에 문자열 연결로 변환합니다. 경우에 따라 슬래시 `/`로 시작하는 문자열이 될 수 있습니다. 이는 Google이 에이전트 로더 스크립트가 포함된 페이지의 색인을 생성할 때 이러한 문자열을 사이트의 잠재적 경로로 저장하고 색인을 생성하려고 시도함을 의미합니다.

## 솔루션

이것은 사이트 관리자에게 드문 시나리오가 아니며 단순히 브라우저 에이전트에 연결된 것도 아닙니다. 사이트 HTML에 문자열이 표시되어 Google 검색에서 문자열을 잠재적인 경로로 인식하는 데에는 여러 가지 이유가 있을 수 있습니다. 다음은 다른 사람들이 이 문제를 어떻게 해결했는지 보여주는 몇 가지 리소스입니다.

* [Google이 **NEXT_DATA**의 URL이라고 생각하는 항목을 크롤링하지 못하도록 차단하려면 어떻게 해야 하나요?](https://stackoverflow.com/questions/75672103/how-can-i-block-google-from-crawling-things-it-thinks-are-urls-from-next-data)
* [Google은 상대 경로로 JavaScript 문자열을 따릅니다. - 404 오류가 발생합니다.](https://webmasters.stackexchange.com/questions/50848/google-follows-javascript-string-as-relative-path-produces-404-error)

사용 가능한 인터넷 리소스에 따르면 이러한 404 오류는 사이트 순위나 인덱싱에 영향을 미치지 않습니다. 무시해도 됩니다. 그러나 여전히 우려되는 경우 [Google 검색 지원 커뮤니티](https://support.google.com/webmasters/community?hl=en) 에 연락하여 추가 피드백과 도움을 받을 수 있습니다.
