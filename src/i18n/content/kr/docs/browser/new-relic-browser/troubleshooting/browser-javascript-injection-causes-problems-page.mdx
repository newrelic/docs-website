## 문제

브라우저 모니터링은 마크업 또는 페이지 로드에 문제를 일으키는 위치에 JavaScript 계측을 삽입하고 있습니다.

## 해결책

New Relic 에이전트는 최적의 위치에 JavaScript를 삽입하려고 시도하지만 문제가 발생하면 다음 문제 해결 단계를 시도하십시오.

1. 브라우저 모니터링을 [비활성화하여 브라우저 모니터링](/docs/browser/new-relic-browser/installation/disable-browser-monitoring/)이 원인인지 확인하십시오.

2. 브라우저 모니터링을 비활성화하여 문제가 해결되면 페이지의 소스 코드를 조사하십시오. 브라우저 모니터링을 다시 활성화하고 다음과 같은 가능한 문제를 확인하십시오.

   * 페이지(iFrame이 아님)에 있는 둘 이상의 `<html>` 태그 인스턴스.
   * `<head>` 또는 `<body>` 태그의 인스턴스가 두 개 이상이거나 `<head>` 또는 `<body>` 태그가 누락되었습니다.
   * 페이지의 JavaScript 내에서 `<head>` 또는 `<body>` 태그가 발생합니다.
   * [http://validator.w3.org](http://validator.w3.org) 를 사용하여 페이지 소스를 확인하고 심각한 마크업 유효성 문제 또는 문자 인코딩 문제를 확인하십시오.

3. 페이지 구조를 수정하거나 [, 해당 페이지에 대한 브라우저 모니터링을 비활성화](/docs/browser/new-relic-browser/installation-configuration/disable-browser-monitoring-specific-pages) 하거나, 스크립트 삽입을 완전히 제어하기 위해 [에이전트 API를 통한 수동 계측으로](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring#agent-instrumentation) 전환하여 발견한 충돌을 제거하십시오.

4. 다른 상황이 발생하면 [New Relic 지원](https://support.newrelic.com/) 에 연락하여 페이지 또는 해당 소스 코드에 대한 액세스 가능한 링크를 제공하십시오. 링크 또는 소스 코드에서 브라우저 모니터링이 활성화되어 있는지 확인하십시오.
