## 문제

브라우저 콘솔을 사용하여 자세한 오류 메시지 환경을 보고 싶지만 브라우저 모니터링이 활성화된 경우 브라우저 콘솔에 자세한 오류 추적이 표시되지 않습니다.

## 해결책

브라우저 모니터링은 오류를 기록하기 위해 캡처하고 다시 던져야 하지만 일부 브라우저 버전은 다시 발생한 오류에 대한 스택 추적을 표시하지 않습니다. 이 문제를 해결하려면 로컬 브라우저에서 브라우저의 개발 모드를 활성화하십시오.

[브라우저 모니터링 에이전트 버전 593 이상](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) 에는 콘솔에서 활성화할 수 있는 이 동작에 대한 해결 방법이 포함되어 있습니다. 다음 절차는 Google Chrome용이지만 다른 브라우저에도 적용할 수 있습니다.

1. Chrome에서 메뉴 아이콘을 선택한 다음 **추가 도구 > JavaScript 콘솔** 을 선택합니다.

2. JavaScript 콘솔에서 로컬 저장소의 `nr_flags` 에 대해 다음 값을 설정합니다.

   ```js
   localStorage.setItem('__nr_flags', 'dev')
   ```

3. JavaScript 오류를 확인하려는 페이지를 새로고침하세요.

4. 개발 모드가 활성화되었는지 확인하려면 콘솔의 브라우저 에이전트 로그에서 `NR AGENT IN DEVELOPMENT MODE` 메시지를 확인하십시오.

5. 사이트를 탐색한 다음 추적하려는 오류를 트리거합니다.

개발 모드를 비활성화하려면 콘솔에서 다음 명령을 실행합니다.

```js
localStorage.removeItem('__nr_flags')
```
