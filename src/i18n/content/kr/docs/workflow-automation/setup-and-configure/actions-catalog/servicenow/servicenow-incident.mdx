이 페이지는 워크플로우 자동화 액션 카탈로그에서 사용할 수 있는 ServiceNow 인시던트 액션에 대한 참조를 제공합니다. 이러한 작업을 통해 ServiceNow에서 인시던트를 생성하고 관리할 수 있습니다.

## 전제 조건

워크플로우 자동화에서 ServiceNow 작업을 사용하기 전에 다음이 준비되어 있는지 확인하십시오:

* 적절한 권한이 있는 ServiceNow 인스턴스.
* 인증을 위한 ServiceNow REST API 키(`x-sn-apikey` 헤더로 전달됨).
* ServiceNow 인스턴스에서 인시던트를 생성하고 관리하는 데 필요한 권한입니다.

작업 입력에 `apiKey` 을(를) 제공해야 합니다. 사용자를 인증하는 데 사용됩니다 ― 키는 요청 헤더 `x-sn-apikey`의 일부로 전달됩니다.

## API 키 인증 설정

자세한 내용은 ServiceNow 문서에서 [API 키 인증 프로필을 사용하여 REST API 액세스 정책 생성](https://www.servicenow.com/docs/r/platform-security/authentication/create-api-access-policy.html) 을 참조하세요.

ServiceNow에서 REST API 키를 생성하고 구성하려면:

<Steps>
  <Step>
    ### 인바운드 인증 프로필 생성

    1. <DNT>**System Web Services &gt; API Access Policies &gt; Inbound Authentication Profile**</DNT> 으로 이동합니다.
    2. 새 **API 키 인증 프로필**을 생성합니다.
    3. 파라미터 이름 `x-sn-apikey`을(를) 사용하여 **Auth Parameter** 를 HTTP 헤더(권장)로 정의합니다.
  </Step>

  <Step>
    ### REST API 키 생성

    1. <DNT>**System Web Services &gt; API Access Policies &gt; REST API Key**</DNT> 으로 이동합니다.
    2. 적절한 역할 및 권한을 가진 전용 서비스 계정과 연결된 새 키를 생성하십시오.
    3. 생성된 토큰 값을 저장하고 복사하십시오. 이는 비밀 키입니다.
  </Step>

  <Step>
    ### API 액세스 정책 구성

    1. <DNT>**System Web Services &gt; API Access Policies &gt; REST API Access Policies**</DNT> 으로 이동합니다.
    2. 새 정책을 생성하고, 타겟 REST API(예: **Table API** 또는 **Scripted REST API**)를 선택합니다.
    3. 이 정책의 관련 목록에 **Inbound Authentication Profile** \[인바운드 인증 프로필] 을 추가합니다.
  </Step>
</Steps>
