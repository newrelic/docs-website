워크플로우 자동화는 코드 없이 인시던트 전투 및 전투 관리를 자동화합니다. 알림에 응답하고, AWS 및 Slack과 같은 도구와 통합하며, 운영 작업을 자동으로 실행하는 도구를 구축하세요.

## 요구 사항

워크플로우를 사용하려면 다음이 필요합니다.

* 뉴렐릭 계정
* 고급 컴퓨트(aCCU) 가격 플랜

<Callout variant="important">
  고급 컴퓨트(aCCU)에는 자동화 및 비밀 관리 기능이 포함되어 있습니다. aCCU를 보유하고 있는지 확실하지 않은 경우 [계정 설정을](https://one.newrelic.com/admin-portal/plan-info/home) 확인하거나 계정 담당자에게 문의하십시오.
</Callout>

## 주요 이점

* **더 빠른 인시던트 대응**: 알림 실행 시 EC2의 크기를 자동으로 조정하고, 구현을 롤백하거나, 배포하거나, 서비스를 다시 시작합니다. 수동 개입이 필요하지 않습니다.
* **도구 연결**: AWS, Slack, Jira, ServiceNow 및 모든 HTTP API 와 통합하여 팀이 작업하는 곳으로 풍부한 공지 데이터를 전송하세요.
* **승인 게이트 추가**: 중요한 부분에 인적 감독을 포함합니다. 프로덕션을 확장하거나 롤백을 실행하기 전에 팀 확인을 요청합니다.
* **복잡성 처리**: 조건 논리, 루프, 데이터 변환을 사용하여 고유한 운영 프로세스를 자동화합니다.

      1. [EC2 자동 크기 조정 예제를](/docs/workflow-automation/create-a-workflow-automation/create-your-own) 검토하세요.

      4. 점진적으로 구축하고 각 단계를 테스트합니다.
    </TabsPageItem>

    <TabsPageItem id="setup">

      **AWS 흐름 흐름**: [AWS 자격 증명(IAM 역할, 사용자 또는 세션)을 설정합니다](/docs/workflow-automation/setup-and-configure/set-up-aws-credentials).

      **공지**: [대상(Slack, 이메일, 웹훅)을 생성하세요](/docs/workflow-automation/setup-and-configure/create-destinations).

## 트리거 흐름

세 가지 방법으로 워크플로우를 시작할 수 있습니다:

* **알림에서**: 공지 조건 위반 시 자동으로 플로우를 시작합니다.
* **온디맨드 방식**: 필요할 때 API를 통해 워크플로우를 수동으로 실행합니다.
* **예약 실행**: cron 표현식을 사용하여 지정된 시간에 자동으로 실행하세요.

공지로 인해 발생하는 워크플로우에 대해서는 [공지 보내기를](/docs/workflow-automation/setup-and-configure/create-destinations#trigger-from-alerts) 참조하세요.
