## 통사론

```
NewRelic.setMaxEventPoolSize(int $maxSize)
```

이벤트 풀의 최대 크기를 설정합니다.

## 요구 사항

에이전트 버전 5.0.0 이상.

## 설명

기본적으로 모바일 모니터링은 이벤트 수집 주기당 최대 1,000개의 이벤트를 수집하며 기본적으로 600초입니다. 이 방법은 다음 수확 주기까지 메모리에 저장된 이벤트 풀의 최대 크기를 제어합니다. 풀 크기 제한에 도달하면 New Relic Android 에이전트는 이벤트 풀이 다음 수확 주기로 전송될 때까지 일부 이전 및 일부 새 이벤트를 버리고 이벤트 샘플링을 시작합니다. 이 방법을 사용하면 해당 이벤트 풀의 최대 크기를 재정의할 수 있습니다. 풀 크기 제한에 도달하면 New Relic Android 에이전트는 이벤트 풀이 다음 수확 주기로 전송될 때까지 일부 이전 및 일부 새 이벤트를 버리고 [이벤트 샘플링](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling) 을 시작합니다.

* 이벤트 수집 주기의 기본값은 600초입니다.

이벤트 수집 주기의 길이를 변경할 수 있는 [`setMaxEventBufferTime()`](/docs/mobile-monitoring/new-relic-mobile-android/android-sdk-api/setmaxeventbuffertime-android-sdk-api) 도 참조하세요.

<Callout variant="important">
  많은 수의 이벤트를 보고하거나 이벤트를 너무 자주 보고하면 앱 성능에 영향을 줄 수 있습니다.
</Callout>

이 API를 사용하는 방법에 대한 컨텍스트는 [사용자 지정 속성 및 이벤트 보내기](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/work-android-sdk-api#attributes-events-insights) 를 참조하세요.

## 반환 값

성공하면 `true` , 실패하면 `false` 를 반환합니다.

## 예

### 이벤트 풀의 최대 크기를 1000으로 설정

```
boolean poolSizeSet = NewRelic.setMaxEventPoolSize(1000);
```
