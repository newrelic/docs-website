## 통사론

```
setMaxEventPoolSize:(unsigned int)size;
NewRelic.setMaxEventPoolSize(unsigned int $size)
```

이벤트 풀의 최대 크기를 설정합니다.

## 요구 사항

새 값이 항상 적용되도록 하려면 이 API 호출을 `applicationDidBecomeActive` iOS 수명 주기 메서드에 배치합니다.

모든 에이전트 버전과 호환됩니다.

## 설명

기본적으로 iOS 에이전트는 수집 [주기](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#harvest-cycle) 당 최대 1000개의 이벤트를 수집합니다. 이 방법은 다음 수확 주기까지 메모리에 저장된 이벤트 풀의 최대 크기를 제어합니다. 풀 크기 제한에 도달하면 New Relic Android 에이전트는 이벤트 풀이 다음 수확 주기로 전송될 때까지 일부 이전 및 일부 새 이벤트를 버리고 [이벤트 샘플링](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling) 을 시작합니다. 이 방법을 사용하면 해당 이벤트 풀의 최대 크기를 재정의할 수 있습니다. `Agent.start()` 이 호출된 후 이 값을 설정해야 합니다.

* 이벤트 수집 주기의 기본값은 600초입니다.

이벤트 수집 주기의 길이를 변경할 수 있는 [`setMaxEventBufferTime()`](/docs/mobile-monitoring/new-relic-mobile-ios/ios-sdk-api/setmaxeventbuffertime-ios-sdk-api) 도 참조하세요.

이 API를 사용하는 방법에 대한 컨텍스트는 사용자 정의 속성 및 이벤트 전송에 대한 문서를 참조하십시오.

* [오브젝티브-C](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/work-ios-sdk-api#objc-custom-att-events)
* [빠른](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/work-ios-sdk-api#swift-custom-att-events)

## 반환 값

성공적으로 기록된 경우 `true` 을 반환하고 기록되지 않은 경우 `false` 을 반환합니다.

## 예

### 오브젝티브-C [#obj-c]

**방법:**

```
+ (void) setMaxEventPoolSize:(unsigned int)size;
```

**예시:**

```
BOOL poolSizeSet = [NewRelic setMaxEventPoolSize:1000];
```

### 빠른

**방법:**

```
NewRelic.setMaxEventPoolSize(size: UInt32)
```

**예시:**

```
let poolSizeSet = NewRelic.setMaxEventPoolSize(1000)
```
