세션 리플레이를 수집하기 시작하면 공통적인 패턴과 사용자 경험 마찰이 발생하는 영역을 식별하여 디지털 경험을 개선하는 데 사용할 수 있습니다. 또한 오류나 예상치 못한 동작이 발생하기 전에 사용자가 취한 단계를 검토하여 문제를 신속하게 진단하고 해결하는 데에도 도움이 됩니다.

## 세션 리플레이 페이지에서 리플레이 보기 [#replay-page]

세션 리플레이 페이지는 선택한 기간 동안 수집된 모든 리플레이 목록을 제공합니다. 기본적으로, 우리는 최소 5초 이상 지속된 리플레이를 목록으로 필터링합니다. 이 표는 오류 수와 같은 유용한 데이터 포인트를 제공하며, 필터를 사용하면 문제를 해결하는 데 도움이 되는 세션으로 목록을 좁힐 수 있습니다.

<img title="Session Replay listing page" alt="Session Replay listing page" src="/images/mobile-session-replay-listing.webp" />

<figcaption>
  **Mobile &gt; \[귀하의\_앱] &gt; Session replay** 에서 다시보기를 보고 필터링하세요.
</figcaption>

예를 들어, 특정 지역의 리플레이를 보려면 `countryCode` 필터를 추가하여 현지화가 예상대로 작동하는지 확인할 수 있고, 호환성 문제가 있을 수 있는 특정 휴대폰 모델의 리플레이를 보려면 `deviceModel` 필터를 추가할 수 있습니다.

**세션 리플레이 페이지에 액세스하려면:**

뉴렐릭 웹 인터페이스에서:

1. **[one.newrelic.com &gt; All Capabilities &gt; Mobile](https://one.newrelic.com/all-capabilities)** 로 이동합니다.
2. 모바일 앱을 선택하세요.
3. 왼쪽 메뉴에서 아래로 스크롤하여 **Session replay** \[세션 재생을] 클릭합니다.
4. 리플레이를 보려면 행을 클릭하세요.

## 오류 및 성능 문제가 있는 리플레이 보기 [#errors-issues]

에러스 인박스(errors inbox) 페이지에서:

1. 왼쪽 모바일 메뉴에서 **Errors** \[오류 를] 클릭합니다.
2. 오류에 첨부된 재생을 보려면 **Triage** \[분류] 및 **Group errors** \[그룹 오류] 페이지를 클릭하세요.

<img title="Session Replay in errors experiences" alt="Session Replay integration with error experiences showing a replay timeline with error markers" src="/images/mobile-session-replay-errors.webp" />

리플레이가 캡처되는 동안 사용자가 문제를 겪은 경우, 충돌, 처리된 예외 및 ANR 환경의 오류 발생 내역에서 리플레이 미리보기를 사용할 수 있습니다.

미리보기에서는 문제가 발생하기 5-10초 전의 모습이 표시되고, 치명적이지 않은 문제의 경우 문제가 발생한 후 5-10초 동안의 모습이 표시됩니다. 리플레이에서 오류가 발생한 정확한 시점을 보여주기 위해 타임라인에 아이콘이 표시됩니다. 이를 통해 문제를 일으킨 사용자 작업을 신속하게 평가하고 최종 사용자에게 미치는 영향의 심각성을 파악할 수 있습니다.

미리보기만으로 문제가 해결되지 않으면 **View full session replay** \[전체 세션 보기] 버튼을 클릭하여 캡처된 다른 텔레메트리 데이터와 동기화된 전체 세션을 확인하세요.

## UI 이해 [#ui]

UI에는 비활성 건너뛰기 또는 0.5배속에서 8배속과 같이 다른 일반적인 비디오 플레이어와 매우 유사해 보이는 몇 가지 요소가 있습니다. 또한 처리된 예외, HTTP 오류, 애플리케이션 응답 없음 이벤트 및 크래시에 대한 마커를 포함하여 앞으로 건너뛸 수 있는 특정 시점도 찾을 수 있습니다.

**Skip Inactivity** \[비활성 건너뛰기] 기능은 페이지나 뷰의 비활성이 아닌 사용자 비활성 기간을 건너뜁니다.

* 빨간색 마커는 충돌을 나타냅니다.
* 불 아이콘은 해당 기간 동안 99개 이상의 오류 또는 충돌 이벤트가 발생했음을 의미합니다.
* 지속 시간 선의 빨간색 대각선은 ANR이 발생하기 전에 리플레이가 종료되었음을 의미합니다.
* 지속 시간 라인의 주황색 대각선은 ANR이 발생하기 전에 리플레이가 종료되었음을 의미합니다.

전체 진단 컨텍스트를 위해 세션 리플레이 UI 내에 로그가 있습니다. 세션 리플레이가 (표준 샘플링, 오류 모드 샘플링 또는 수동 API를 통해) 활성화되어 작동 중인 경우, 로깅 기능이 전역적으로 활성화되어 있다면 해당 세션이 원래 로깅을 위해 샘플링되지 않았더라도 에이전트는 해당 세션에 대한 원격 로그도 캡처합니다.
