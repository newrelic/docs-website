모바일 모니터링은 Android 및 iOS 앱용 애플리케이션 코드를 계측하고 사용자 상호 작용을 서비스할 때 코드를 모니터링하는 엔진을 제공합니다. 이는 시간이 많이 소요되는 사용자 활동을 분석하는 데 유용합니다. Android 또는 iOS 앱용 SDK API를 설정한 후 상호 작용 페이지에서 자세한 정보를 볼 수 있습니다.

## 수단

Android 및 iOS 앱의 상호 작용을 계측하기 위한 SDK API를 제공합니다. 여기에는 다음이 포함됩니다.

* Android: 활동, AsyncTask 스레딩 및 데이터베이스 성능, 이미지 로드, JSON 구문 분석 및 기타 공통 API
* iOS: UIViewControllers, GCD dispatch_async 및 CoreData, UIImage, NSJSON 및 기타 일반 API의 성능

자세한 내용 [은 Android SDK API](/docs/mobile-sdk-api/android-api) [작업 및 iOS SDK API 작업](/docs/mobile-sdk-api/ios-api) 을 참조하십시오.

## 상호 작용 페이지 보기 [#interactions-dashboard]

상호 작용 페이지에는 가장 비용이 많이 드는 5가지 상호 작용(기본 스레드에서 실행되는 코드 및 작업자 스레드에서 실행되는 코드 포함)에 대한 분석 차트와 가장 느린 추적 목록이 포함되어 있습니다.

상호작용 페이지를 보려면:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile >** (앱 선택) **> More views > Interactions** 으로 이동합니다.
2. 자세한 정보로 드릴다운하려면 상호 작용을 선택하거나 해당 [플랫폼 UI 기능을](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)사용하십시오.

<Callout variant="tip">
  [모바일 앱 개요](/docs/mobile-monitoring-ui/mobile-apps-dashboard) 페이지에서 **상호 작용** 으로 바로 이동하려면 **가장 느린 상호 작용** 차트의 제목을 선택하거나 상호 작용을 선택하여 자세한 정보를 봅니다.
</Callout>

## 분석 테이블 세부 정보 보기 [#breakdown-table]

계측된 모든 방법의 전체 목록을 표시하려면 선택한 상호 작용에서 **분석 테이블** 링크를 선택합니다. 측정항목에 대해 사용자 지정 계측을 사용하는 경우 클래스 및 메서드 이름에 대한 범주는 **사용자 지정** 을 나타냅니다. 추적된 각 방법에 대해 상호 작용 분석 테이블에는 다음이 표시됩니다.

* 스레드 유형(전경 또는 배경)
* 클래스 및 메서드 이름
* 이 방법에 소요된 총 실행 시간의 백분율
* 상호작용당 이 메서드에 대한 평균 호출 수
* 이 메서드에 대한 각 호출의 전체 평균 실행 시간

## 상호 작용 타임라인 보기 [#interactions-timeline]

각 상호 작용 추적 기간은 새 상호 작용이 시작될 때까지 추적을 기록합니다.

모바일 앱의 상호 작용 타임라인을 보려면 다음 단계를 따르세요.

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) >App > Interactions** 으로 이동합니다.
2. 가장 **느린 추적** 테이블에서 추적에 대한 **활동** 또는 **기간** 링크를 선택하십시오.

또는:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) > App > Interactions > (상호 작용 선택)** 으로 이동합니다.
2. 가장 **느린 추적** 테이블에서 추적에 대한 **기간** 링크를 선택하십시오.

타임라인 위로 마우스를 가져가면 아래에 있는 메서드, 응용 프로그램에서 사용하는 메모리 및 활성 네트워크 요청에 대한 정보가 나타납니다. 행 레이블은 각 메서드가 실행 중인 스레드를 나타냅니다. 메소드 내에서 실행 흐름의 폭포수를 보려면 메소드 블록을 선택하십시오.

상호 작용 타임라인은 또한 앱의 기기 수준 데이터에 대한 통찰력을 얻는 데 도움이 되는 **기기별 메트릭** 을 제공합니다. 사용 메트릭은 앱이 최종 사용자의 기기와 상호 작용하는 방식을 보여줍니다. 운영 체제 메트릭 외에도 사용량 메트릭에는 데이터베이스, CPU 및 메모리가 포함됩니다.

* 온디바이스 사용량 메트릭을 보려면 타임라인 위로 마우스를 가져갑니다.
* 자세한 정보를 드릴다운하려면 타임라인에서 아무 위치나 선택합니다.
* 데이터에 대한 컨텍스트를 제공하려면 **이전 상호작용** 링크를 선택하십시오.
