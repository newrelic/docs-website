**Summary**\[요약] 페이지는 모바일 앱의 작동 스냅샷을 제공합니다. 모든 차트, 표 및 통계는 현재 선택된 [기간](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) 및 애플리케이션 버전에 대한 데이터를 보여줍니다.

## 주요 앱 측정항목 [#dsahboard-metrics]

요약 페이지는 5가지 주요 앱 측정항목을 캡처합니다.

  <thead>

      </td>

      </td>

        New Relic은 모바일 세션을 앱이 화면에 나타날 때 시작하여 앱이 백그라운드로 전송될 때 끝나는 것으로 정의합니다.

      </td>

        상위 5개는 각 호스트에 대한 완료된 요청 수를 기반으로 계산됩니다.

      </td>
    </tr>

## 요약 페이지 보기 [#summary-page]

모바일 앱의 요약 페이지를 보려면:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택)으로** 이동합니다.
2. 모바일 앱의 다른 페이지를 보려면 요약 페이지 또는 모바일 메뉴에서 링크를 선택하십시오.

## 권장 알림 조건 추가 [#recommended-alert-conditions]

모바일 **Summary** \[요약] 페이지를 통해 권장 알림 조건을 엔터티에 추가할 수 있습니다.

발견되지 않은 엔터티를 더 보려면 [경고 적용 범위 격차 및 조건 권장 사항](/docs/alerts-applied-intelligence/applied-intelligence/alert-coverage-gaps/) 페이지를 참조하십시오.

권장 알림 조건을 추가하려면 다음 단계를 따르세요.

1. 모바일 모니터링 **Summary**\[요약] 페이지를 엽니다.

2. **Add recommended conditions**\[권장 조건 추가 를]클릭합니다.

   <img
     width="80%;"
     style={{align: "left"}}
     title="Mobile summary page"
     alt="Screenshot of the New Relic mobile summary page"
     src={mobileAlertConditions}
   />


3. 여러 권장 경보 조건을 선택하십시오.

   <img
     width="70%;"
     style={{align: "left"}}
     title="Mobile recommended alert conditions"
     alt="Mobile recommended alert conditions"
     src={mobileAddAlert}
   />

4. **Next**클릭합니다.

5. 알림을 받을 정책을 선택하세요. 기존 항목을 선택하거나 [새로 만들](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name)수 있습니다.

6. **Activate conditions**\[조건 활성화를]클릭합니다.
