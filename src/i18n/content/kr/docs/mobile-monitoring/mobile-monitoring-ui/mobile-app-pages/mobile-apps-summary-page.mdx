**요약** 페이지는 모바일 앱의 운영 스냅샷을 제공합니다. 모든 차트, 테이블 및 통계에는 현재 선택한 [기간](/docs/apm/new-relic-apm/ui-functions/time-picker-setting-time-periods-view-data) 및 애플리케이션 버전에 대한 데이터가 표시됩니다.

## 주요 앱 측정항목 [#dsahboard-metrics]

요약 페이지는 5가지 주요 앱 측정항목을 캡처합니다.

<table>
  <thead>
    <tr>
      <th width={200}>
        미터법
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        **애플리케이션 충돌률**
      </td>

      <td>
        시간 경과에 따른 충돌 세션 수를 모든 세션의 백분율로 표시하고 앱 버전별로 분류합니다. 평균 충돌 세션 비율, 총 충돌 횟수, 기간 동안 이러한 충돌의 영향을 받은 고유 사용자 수가 오른쪽 상단에 표시됩니다.
      </td>
    </tr>

    <tr>
      <td>
        **앱 실행**
      </td>

      <td>
        일정 기간 동안 모니터링되는 앱 세션 시작 수를 차트로 표시합니다.

        New Relic은 모바일 세션을 앱이 화면에 나타날 때 시작하여 앱이 백그라운드로 전송될 때 끝나는 것으로 정의합니다.
      </td>
    </tr>

    <tr>
      <td>
        **HTTP 오류/네트워크 오류**
      </td>

      <td>
        **http 상태 코드 오류** (400 이상)를 발생시키는 네트워크 요청 수를 완료된 모든 요청의 백분율로 표시하고 **실패한 네트워크 호출** 수(예: 연결 실패)를 완료된 요청의 백분율로 표시합니다.
      </td>
    </tr>

    <tr>
      <td>
        **HTTP 응답 시간**
      </td>

      <td>
        앱이 통신하는 상위 5개 호스트 각각에서 완료된 모든 http 요청의 평균 응답 시간을 차트로 표시합니다.

        상위 5개는 각 호스트에 대한 완료된 요청 수를 기반으로 계산됩니다.
      </td>
    </tr>

    <tr>
      <td>
        **빈번한 상호 작용**
      </td>

      <td>
        앱에서 가장 일반적으로 실행되는 5가지 상호 작용 각각에 대한 주요 성능 메트릭을 제공합니다.
      </td>
    </tr>
  </tbody>
</table>

## 요약 페이지 보기 [#summary-page]

모바일 앱의 요약 페이지를 보려면:

1. **[one.newrelic.com](https://one.newrelic.com) > Mobile > (앱 선택)으로** 이동합니다.
2. 모바일 앱의 다른 페이지를 보려면 요약 페이지 또는 모바일 메뉴에서 링크를 선택하십시오.
