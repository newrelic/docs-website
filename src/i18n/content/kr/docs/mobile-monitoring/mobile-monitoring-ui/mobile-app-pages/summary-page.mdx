요약 페이지는 모바일 앱의 운영 스냅샷을 제공합니다. 모든 차트, 표, 통계에는 지정된 [기간](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) 과 앱 버전에 대한 데이터가 표시됩니다.

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app):**</DNT> 요약 페이지는 높은 수준의 성능 데이터와 특정 지표 또는 문제를 자세히 살펴볼 수 있는 랜딩 기반을 제공합니다.
</figcaption>

요약 페이지에는 다음과 같은 5가지 주요 성과 지표가 나와 있습니다.

## 요약 페이지 보기 [#summary-page]

1. [one.newrelic.com > All capabilities \[모든 기능\]](https://one.newrelic.com/all-capabilities) > Mobile \[모바일] 이동합니다.
2. 모바일 앱을 선택하세요. 요약 페이지는 기본적으로 로드됩니다.

## (권장) 모바일 모니터링 알림 설정 [#alerts]

느린 네트워크 요청 및 충돌률 증가와 같은 주요 모바일 문제에 대한 알림을 생성하는 것이 좋습니다. 요약 페이지에서 이러한 알림을 쉽게 설정하려면 다음을 수행하세요.

1. <DNT>**Add recommended conditions**</DNT> 을(를) 클릭합니다.

   <img
     width="80%;"
     style={{align: "left"}}
     title="Mobile Summary page"
     alt="Screenshot of the New Relic mobile Summary page"
     src="/images/mobile_screenshot-full_mobile-alert-conditions.webp"
   />


2. 여러 권장 경보 조건을 선택하십시오.

   <img
     width="70%;"
     style={{align: "left"}}
     title="Mobile recommended alert conditions"
     alt="Mobile recommended alert conditions"
     src="/images/mobile_screenshot-crop_mobile-add-alert.webp"
   />

3. <DNT>**Next**</DNT> 을(를) 클릭합니다.

4. 알림을 받으려면 정책을 선택하세요. 기존 정책을 선택하거나 [새 정책을 생성](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name) 할 수 있습니다.

5. <DNT>**Activate conditions**</DNT> 을(를) 클릭합니다.
