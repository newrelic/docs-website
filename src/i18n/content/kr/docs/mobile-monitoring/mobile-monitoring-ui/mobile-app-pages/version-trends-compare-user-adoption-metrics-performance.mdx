릴리스 버전 페이지 <InlinePopover type="mobile"/>다양한 버전에 걸친 모바일 애플리케이션 성능에 대한 포괄적인 개요를 제공합니다. 충돌 발생률, 사용자 채택 및 전반적인 성능 점수에 대한 실시간 통찰력을 제공하므로 정보에 입각한 결정을 내려 앱의 안정성과 사용자 경험을 향상시킬 수 있습니다.

## 요구 사항

모바일 앱의 릴리스 버전 정보를 수집하려면 에이전트 버전이 다음과 같은지 확인하세요.

* [Android 에이전트 v6.5.0 이상용](/docs/release-notes/mobile-release-notes/android-release-notes/) New Relic 모바일 에이전트
* [iOS 에이전트 v7.0.0 이상용](/docs/release-notes/mobile-release-notes/ios-release-notes/) New Relic 모바일 에이전트

## 릴리스 버전 페이지 보기 [#view-release-version]

릴리스 버전 페이지를 보려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Users > Release versions** 으로 이동하세요.

## 측정항목 이해 [#understanding-metrics]

* **Crash Free Users** \[충돌 없는 사용자]: 각 앱 버전에 대해 충돌을 경험한 사용자의 비율을 쉽게 추적하여 추세를 파악하고 버그 수정 우선순위를 지정할 수 있습니다.
* **User Adoption** \[사용자 채택]: 사용자가 새로운 앱 버전을 어떻게 채택하는지 모니터링하여 업데이트 및 기능의 성공 여부를 측정할 수 있습니다.
* **Overall Performance Score** \[전체 성능 점수]: 충돌 발생률과 HTTP 오류 없음 비율을 사용하여 계산된 객관적인 성능 점수를 통해 앱 상태에 대한 전체적인 보기를 얻습니다. 앱 버전에 배포가 활성화되면 릴리스 버전은 각 버전의 릴리스 날짜를 표시하여 한 단계 더 발전합니다. 이 추가 기능은 지표에 대한 추가 컨텍스트를 제공하여 성능 변화를 특정 출시 날짜와 연관시킬 수 있습니다.

## 버전 요약 페이지 [#version-summary-page]

버전 요약 페이지에서는 각 특정 앱 버전에 대한 심층적인 통찰력과 지표에 액세스할 수 있습니다. 이 페이지는 다양한 필수 지표를 제시하여 애플리케이션 성능에 대한 포괄적인 보기를 제공합니다. 이해하기 쉬운 시각화와 자세한 차트를 갖춘 버전 요약 페이지를 통해 핵심 성과 지표를 분석하고 정보에 입각한 결정을 내려 앱의 사용자 경험을 최적화할 수 있습니다. 해당 버전과 관련된 "요약 보기" 또는 "세부 정보 보기" 버튼을 클릭하면 특정 앱 버전의 버전 요약 페이지에 액세스할 수 있습니다. 표시된 측정항목:

* **App Launches** \[앱 실행 횟수]: 앱 버전이 실행된 총 횟수입니다.
* **Average Interaction Time** \[평균 상호 작용 시간]: 사용자가 앱과 상호 작용하는 데 소비한 평균 시간입니다.
* **Average Memory Used** \[사용된 평균 메모리]: 앱이 사용하는 평균 메모리 양입니다.
* **Average Response Time** \[평균 응답 시간]: 앱이 사용자 작업에 응답하는 데 걸리는 평균 시간입니다.
* **Crash Rate** \[충돌 발생률]: 충돌을 초래한 앱 실행의 비율입니다.
* **Error Rate** \[오류율]: 오류로 이어지는 앱 상호작용의 비율입니다.
* **HTTP Error Rate** \[HTTP 오류율]: 오류가 발생한 HTTP 요청의 비율입니다.
* **Install Count** \[설치 횟수]: 앱 버전의 총 설치 횟수입니다.
* **Network Failure Rate** \[네트워크 실패율]: 실패한 네트워크 요청의 비율입니다.
* **Request Count** \[요청 수]: 앱에서 수행한 총 HTTP 요청 수입니다.
* **Total Users** \[총 사용자]: 앱 버전과 상호작용한 총 고유 사용자 수입니다.
* **Upgrade Count** \[업그레이드 횟수]: 이 앱 버전으로 업그레이드한 총 사용자 수입니다. 위에 나열된 각 지표에 대해 버전 요약 페이지에는 동적 시계열 차트가 포함되어 있어 시간 경과에 따른 지표 성능을 추적할 수 있습니다. 차트의 추세, 급증 또는 하락을 관찰하면 의사 결정에 영향을 미칠 수 있는 패턴과 상관 관계를 빠르게 식별할 수 있습니다.

## 버전 추세 데이터 쿼리 [#insights]

[NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) 을 사용하여 쿼리할 때 버전 추세 정보를 분석하는 데 도움이 되는 두 가지 속성을 제공합니다.

* 세션에 새 설치 또는 새 업그레이드가 포함된 시기를 추적하려면 `install` 을 사용합니다. 이 속성은 신규 설치에 대해 `true` 을 기록합니다.
* 업그레이드가 감지되었을 때 모바일 앱의 마지막 버전을 추적하려면 `upgradeFrom` 사용하세요. 이러한 속성을 사용하려면 [Android 에이전트](/docs/release-notes/mobile-release-notes/android-release-notes/) 또는 [iOS 에이전트](/docs/release-notes/mobile-release-notes/ios-release-notes/) 버전이 해당 속성을 지원하는지 확인하세요.
