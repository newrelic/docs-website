모바일 앱의 상태를 이해하면 팀이 새 버전마다 더 똑똑하고 효율적인 선택을 할 수 있습니다. 모바일 앱의 변경 사항을 추적하기 위해 New Relic은 모바일 모니터링 UI에서 버전 추세 데이터를 제공합니다. 여기에서 버전 추세 보고서를 사용하여 최신 버전의 모바일 앱에서 성능을 비교할 수 있습니다. 각 버전에 대한 개선, 수정 및 저하의 영향을 분석하는 데 필요한 주요 데이터 포인트에 액세스할 수 있습니다. 사용량, 채택 및 기술 지표를 비교할 수 있습니다.

## 요구 사항

모바일 앱의 버전 추세 정보를 수집하려면 에이전트 버전이 다음과 같은지 확인하십시오.

* [Android 에이전트 v6.5.0 이상용](/docs/release-notes/mobile-release-notes/android-release-notes/) New Relic 모바일 에이전트
* [iOS 에이전트 v7.0.0 이상용](/docs/release-notes/mobile-release-notes/ios-release-notes/) New Relic 모바일 에이전트

## 버전 추세 세부 정보 이해 [#details]

모바일 앱의 새 버전을 출시한 후 버전 추세 보고서를 사용하여 성능과 사용자 채택을 비교할 수 있습니다. 비정상 종료율 및 HTTP 오류와 같은 주요 지표를 통해 앱의 상태를 측정할 수도 있습니다.

## 버전 추세 데이터 쿼리 [#insights]

[NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) 을 사용하여 쿼리할 때 버전 추세 정보를 분석하는 데 도움이 되는 두 가지 속성을 제공합니다.

* 세션에 새 설치 또는 새 업그레이드가 포함된 시기를 추적하려면 `install` 을 사용합니다. 이 속성은 신규 설치에 대해 `true` 을 기록합니다.
* 업그레이드가 감지될 때 모바일 앱의 마지막 버전을 추적하려면 `upgradeFrom` 을 사용합니다.

이러한 속성을 사용하려면 [Android 에이전트](/docs/release-notes/mobile-release-notes/android-release-notes/) 또는 [iOS 에이전트](/docs/release-notes/mobile-release-notes/ios-release-notes/) 버전이 해당 속성을 지원하는지 확인하십시오.
