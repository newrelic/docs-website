## 릴리스 버전 페이지 보기 [#view-release-versions-page]

1. \*\*[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile로 이동합니다.
2. 모바일 앱을 선택하세요.
3. 왼쪽 메뉴에서 **Release versions** \[릴리스 버전 을] 클릭합니다.

<Callout variant="tip">
  데이터가 보이지 않나요? 다음을 시도해 보세요:

  * 새 버전이 출시되기 전의 데이터를 포함하도록 시간 필터를 조정하세요.
  * 모바일 에이전트가 iOS 에이전트 버전 7.0.0 이상 또는 Android 에이전트 버전 6.5.0 이상을 사용하고 있는지 확인하세요.
</Callout>

## 특정 지표를 자세히 살펴보세요. [#drill-down]

릴리스 버전 페이지에는 각 앱 버전에 대한 상위 수준 지표가 표시됩니다.

* **Overall performance score** \[전체 성능 점수]: 전체 고객 충돌률과 HTTP 오류 없음 비율을 기준으로 계산됩니다.
* **Crash free user %** \[충돌 방지 사용자 %]: 특정 버전에서 충돌을 경험하지 않은 사용자의 비율입니다.
* **User adoption** \[사용자 채택]: 특정 버전을 활용하는 사용자의 비율입니다.

**View summary** \[요약 보기] 또는 **View details** \[세부 정보 보기를] 클릭하여 앱 버전에 대한 특정 지수를 확장하세요. 이해하기 쉬운 시각화와 상세한 차트를 갖춘 버전 요약 페이지를 통해 핵심 성과 지표(KPI)를 분석하고 정보에 입각한 결정을 내려 앱의 사용자 경험을 최적화할 수 있습니다.

위에 나열된 각 지표에 대해 릴리스 버전 페이지에는 동적 시계열 차트가 포함되어 있어 시간 경과에 따른 지표 성과를 추적할 수 있습니다. 차트의 추세, 급증 또는 하락을 관찰하면 의사 결정에 영향을 미칠 수 있는 패턴과 상관 관계를 빠르게 식별할 수 있습니다.

## 알림 생성 [#create-alerts]

이제 앱 성능을 확인했으므로 주요 성능 변화에 대한 알림을 설정해야 합니다.

지표를 클릭한 후 차트 메뉴에서 **Create alert condition** \[공지 조건 생성을] 클릭하고 UI의 단계를 따르세요.
