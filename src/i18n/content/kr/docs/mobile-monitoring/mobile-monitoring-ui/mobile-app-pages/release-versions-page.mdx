최근 앱 출시가 어떻게 진행되고 있는지 궁금하십니까? 릴리스 버전 페이지를 사용하면 최신 릴리스의 성공을 분석하는 데 사용할 수 있는 다양한 릴리스 버전 간의 충돌률, 사용자 참여 및 전반적인 성능 지표를 볼 수 있습니다.

## 릴리스 버전 페이지 보기 [#view-release-versions-page]

1. [one.newrelic.com > All capabilities \[모든 기능\]](https://one.newrelic.com/all-capabilities) > Mobile \[모바일] 이동합니다.

2. 모바일 앱을 선택하세요.

3. 왼쪽 메뉴에서

   <DNT>
     **Release versions**
   </DNT>

   클릭합니다.

<Callout variant="tip">
  데이터가 보이지 않나요? 다음을 시도해 보세요:

  * 새 버전이 출시되기 전의 데이터를 포함하도록 시간 필터를 조정하세요.
  * 모바일 에이전트가 iOS 에이전트 버전 7.0.0 이상 또는 Android 에이전트 버전 6.5.0 이상을 사용하고 있는지 확인하세요.
</Callout>

## 특정 지표를 자세히 살펴보세요. [#drill-down]

릴리스 버전 페이지에는 각 앱 버전에 대한 상위 수준 지표가 표시됩니다.

* <DNT>
    **Overall performance score**
  </DNT>

  : 전체적인 고객충돌률과 HTTP 무오류율을 기준으로 계산됩니다.

* <DNT>
    **Crash free user %**
  </DNT>

  : 특정 버전에서 충돌을 경험하지 않은 사용자의 비율입니다.

* <DNT>
    **User adoption**
  </DNT>

  : 특정 버전으로 업데이트하거나 설치한 사용자의 비율입니다.

앱 버전에 대한 특정 지표로 확장하려면 <DNT>**View summary**</DNT> 또는 <DNT>**View details**</DNT> 클릭하세요. 이해하기 쉬운 시각화와 상세한 차트를 갖춘 버전 요약 페이지를 통해 핵심 성과 지표(KPI)를 분석하고 정보에 입각한 결정을 내려 앱의 사용자 환경을 최적화할 수 있습니다.

<img
  title="Details of release versions"
  alt="Screenshot of release versions detailed view"
  src="/images/mobile_screenshot-full_release-versions-details.webp"
/>

위에 나열된 각 지표에 대해 릴리스 버전 페이지에는 동적 시계열 차트가 포함되어 있어 시간 경과에 따른 지표 성과를 추적할 수 있습니다. 차트의 추세, 급증 또는 하락을 관찰하면 의사 결정에 영향을 미칠 수 있는 패턴과 상관 관계를 빠르게 식별할 수 있습니다.

## 알림 생성 [#create-alerts]
