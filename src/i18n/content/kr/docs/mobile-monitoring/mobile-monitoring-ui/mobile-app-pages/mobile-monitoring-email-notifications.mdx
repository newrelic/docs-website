뉴렐릭의 <InlinePopover type="mobile"/>는 다음을 포함한 여러 가지 이메일 보고서를 제공합니다.

* [릴리스 알림](#version-trends) (버전 동향 이메일이라고도 함)
* [충돌 보고서 이메일](#crash-reports)

## 새 버전 성능에 대한 이메일 받기 [#version-trends]

뉴렐릭의 모바일 예측은 [<DNT>**Release versions**</DNT>](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/release-versions-page) 페이지를 제공합니다. 버전 동향 이메일(일명 릴리스 공지)을 활성화할 수도 있습니다. 이러한 보고서는 앱의 진행 상황, 채택 및 기타 추세를 공유, 추적 및 검토할 수 있는 편리한 커뮤니케이션 도구를 제공합니다. 각 이메일에는 자세한 내용을 볼 수 있는 <DNT>**Version trends**</DNT> UI에 대한 링크가 포함되어 있습니다.

정기적으로 보고서를 이메일로 보내드립니다.

### 버전 추세 이메일 설정 편집 [#version-trend-ui]

버전 추세 이메일 설정을 구성하는 방법은 사용 중인 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

<Callout variant="tip">
  사용자 모델을 빠르게 확인하려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 <DNT>**User preferences**</DNT> 클릭하세요. 거기에 모바일 이메일 설정이 표시되면 원래 사용자 모델을 사용하고 있는 것입니다.
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-version-trends"
    title="버전 이메일(원래 사용자 모델)"
  >
    모니터링되는 모든 모바일 앱에 대한 버전 동향 이메일 보고서를 선택하거나 선택 해제하려면:

    1. <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select a mobile app) > App > Version trends**</DNT> 으)로 이동합니다.

    2. <DNT>**Enable version trends email notifications**</DNT> 링크를 선택하고 기본 설정을 편집하세요.

       특정 모바일 앱에 대한 이메일 보고서를 구성하려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT> 으로 이동하여 환경설정을 수정하세요.
  </Collapser>

  <Collapser
    id="new-relic-one-user-version-trends"
    title="버전 이메일(최신 사용자 모델)"
  >
    특정 모바일 앱에 대해 수신하는 이메일 보고서를 구성하려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT> 으로 이동하여 출시 공지 이메일 설정을 수정하세요.
  </Collapser>
</CollapserGroup>

## 충돌 보고서 이메일 [#crash-reports]

모바일 앱에 새로운 종류의 충돌이 발생하면 무엇이 잘못되었는지 알고 싶어 향후 이러한 유형의 오류를 방지할 수 있습니다. 충돌 보고서 이메일을 사용하면 고유한 충돌이 감지될 때 알림을 받습니다.

충돌 보고서 이메일 설정을 구성하는 방법은 사용 중인 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

<Callout variant="tip">
  사용자 모델을 빠르게 확인하려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 <DNT>**User preferences**</DNT> 클릭하세요. 거기에 모바일 이메일 설정이 표시되면 원래 사용자 모델을 사용하고 있는 것입니다.
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-crash-emails"
    title="충돌 이메일(원래 사용자 모델)"
  >
    여러 앱에 걸쳐 계정에 대한 충돌 보고서 이메일 설정을 구성하려면 다음 단계를 따르세요. <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**</DNT>)로 이동하세요.

    모니터링되는 특정 모바일 앱에 대한 충돌 보고서 이메일 설정을 구성하려면:

    1. <DNT>
         **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**
       </DNT>

       으)로 이동합니다.

    2. <DNT>
         **Mobile crash reporting**
       </DNT>

       섹션에서 모바일 앱 이름 위에 마우스를 올리면

       <DNT>
         **Edit application settings**
       </DNT>

       표시됩니다. 선택하세요.

    3. 그러면 앱의

       <DNT>
         [**Settings**](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)
       </DNT>

       [UI 페이지](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings) 로 이동됩니다.

       <DNT>
         **Enable crash report emails for this application**
       </DNT>

       확인란을 전환합니다.
  </Collapser>

  <Collapser
    id="new-relic-one-user-crash-emails"
    title="충돌 이메일(최신 사용자 모델)"
  >
    특정 모바일 앱에 대한 오류 이메일 보고서 수신 여부를 구성하려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Settings > Application**</DNT> 으로 이동하여 오류 보고서 이메일 설정을 수정하세요.
  </Collapser>
</CollapserGroup>
