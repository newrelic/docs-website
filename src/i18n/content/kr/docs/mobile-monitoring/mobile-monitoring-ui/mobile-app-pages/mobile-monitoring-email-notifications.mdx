New Relic의 모바일 모니터링은 다음을 포함한 여러 이메일 보고서를 제공합니다.

* [릴리스 알림](#version-trends) (버전 동향 이메일이라고도 함)
* [충돌 보고서 이메일](#crash-reports)

## 버전 동향 이메일 [#version-trends]

New Relic의 모바일 모니터링은 [**버전 동향** UI](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/version-trends-compare-user-adoption-metrics-performance) 를 제공합니다. 버전 동향 이메일(일명 릴리스 알림)을 활성화할 수도 있습니다. 이러한 보고서는 앱의 진행 상황, 채택 및 기타 추세를 공유, 추적 및 검토할 수 있는 편리한 커뮤니케이션 도구를 제공합니다. 각 이메일에는 자세한 내용을 볼 수 있는 **버전 추세** UI에 대한 링크가 포함되어 있습니다.

### 버전 추세 이메일 설정 편집 [#version-trend-ui]

버전 추세 이메일 설정을 구성하는 방법은 사용 중인 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

<Callout variant="tip">
  사용자 모델을 빠르게 확인하려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 **사용자 기본 설정** 을 클릭합니다. 거기에 모바일 이메일 설정이 보이면 원래 사용자 모델에 있는 것입니다.
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-version-trends"
    title="버전 이메일(원래 사용자 모델)"
  >
    모니터링되는 모든 모바일 앱에 대한 버전 동향 이메일 보고서를 선택하거나 선택 해제하려면:

    1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (모바일 앱 선택) > App > Version trends** 로 이동합니다.

    2. **버전 동향 이메일 알림 활성화** 링크를 선택하고 기본 설정을 편집합니다.

       특정 모바일 앱에 대한 이메일 보고서를 구성하려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (앱 선택) > Settings > Application** 으로 이동하고 기본 설정을 편집합니다.
  </Collapser>

  <Collapser
    id="new-relic-one-user-version-trends"
    title="버전 이메일(최신 사용자 모델)"
  >
    특정 모바일 앱에 대해 수신하는 이메일 보고서를 구성하려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (앱 선택) > Settings > Application** 으로 이동하고 릴리스 알림 이메일 설정을 편집합니다.
  </Collapser>
</CollapserGroup>

## 충돌 보고서 이메일 [#crash-reports]

모바일 앱에 새로운 종류의 충돌이 발생하면 무엇이 잘못되었는지 알고 싶어 향후 이러한 유형의 오류를 방지할 수 있습니다. 충돌 보고서 이메일을 사용하면 고유한 충돌이 감지될 때 알림을 받습니다.

충돌 보고서 이메일 설정을 구성하는 방법은 사용 중인 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

<Callout variant="tip">
  사용자 모델을 빠르게 확인하려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 **사용자 기본 설정** 을 클릭합니다. 거기에 모바일 이메일 설정이 보이면 원래 사용자 모델에 있는 것입니다.
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-crash-emails"
    title="충돌 이메일(원래 사용자 모델)"
  >
    여러 앱에서 계정에 대한 충돌 보고서 이메일 설정을 구성하려면: **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(사용자 메뉴)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences** 으로 이동합니다.

    모니터링되는 특정 모바일 앱에 대한 충돌 보고서 이메일 설정을 구성하려면:

    1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(사용자 메뉴)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences** 으로 이동합니다.
    2. **모바일 충돌 보고** 섹션에서 모바일 앱 이름 위로 마우스를 이동하면 **애플리케이션 설정 편집** 이 표시됩니다. 그것을 선택하십시오.
    3. 앱의[**설정** UI 페이지](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings) 로 이동합니다. **이 애플리케이션에 대한 충돌 보고서 이메일 활성화** 확인란을 전환합니다.
  </Collapser>

  <Collapser
    id="new-relic-one-user-crash-emails"
    title="충돌 이메일(최신 사용자 모델)"
  >
    특정 모바일 앱에 대한 충돌 이메일 보고서를 수신할지 여부를 구성하려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) > Settings > Application**으로 이동하고 충돌 보고서 이메일 설정을 편집합니다.
  </Collapser>
</CollapserGroup>
