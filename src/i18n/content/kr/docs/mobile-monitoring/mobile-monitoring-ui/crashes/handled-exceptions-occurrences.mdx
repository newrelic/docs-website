[**처리된 예외** 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/handled-exceptions-analyze-trends-prevent-crashes) 를 사용하여 패턴 및 추세를 탐색한 후에는 특정 예외 발생에 대한 세부 정보를 드릴다운할 수 있습니다. New Relic이 기본적으로 모니터링하는 이벤트 또는 [사용자가 생성한 사용자 지정 이벤트일 수 있습니다](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile#custom-events) [.](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/introduction-mobile-handled-exceptions#hex-api)

## 발생 세부정보 보기 [#hex-occurrences]

처리된 예외의 각 발생에 대한 세부 정보를 보려면 다음을 수행합니다.

1. **[one.newrelic.com](https://one.newrelic.com) > Mobile > (앱 선택) > Exceptions > Handled exceptions** 로 이동합니다.
2. **상위 5개 예외 위치** 테이블에서 임의의 행에서 처리된 예외를 선택합니다.

**발생** 페이지에는 장치 유형 또는 운영 체제별 분석 데이터, 영향을 받는 사용자 수, 선택한 기간의 총 발생 횟수, 속성 등을 포함하여 선택한 예외에 대한 세부 정보가 표시됩니다.

## 기호화되지 않은 스택 추적 [#unsymbolicated]

New Relic은 처리된 예외에 대해 기호화되지 않은 스택 추적을 저장하지 않습니다. 이것은 [충돌 이벤트 추적](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail) 과 다릅니다. 새로 처리된 예외가 기호화되므로 필요에 따라 나중에 [dSYM을 업로드할](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/upload-dsyms-bitcode-apps) 수 있습니다.
