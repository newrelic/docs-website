그만큼 <InlinePopover type="mobile"/>**crash event trail** \[충돌 이벤트 추적은] 구독 수준의 [데이터 보존 정책](/docs/accounts-partnerships/accounts/account-billing-usage/data-retention) 에 따라 모바일 앱의 충돌로 이어지는 이벤트를 보여줍니다. 이는 기본적으로 New Relic이 모니터링하는 이벤트이거나 사용자 정의 이벤트일 수 있습니다. 이 문서에서는 [충돌 이벤트 추적이 무엇인지](#what-is) , [어떻게 사용하는지](#use) 설명합니다.

## 모바일 앱이 다운되기 전에 이벤트 보기 [#what-is]

모바일 앱이 충돌하고 이유를 모를 때 충돌 직전에 무슨 일이 일어났는지 연구할 수 있습니다. 충돌 이벤트 추적은 이러한 이벤트를 보여주므로 충돌로 이어지는 "이동 경로"를 추적하고 실패 원인을 진단할 수 있습니다.

충돌 이벤트 추적은 충돌로 이어지는 모든 모바일 [이벤트 유형을](/docs/insights/explore-add-data/custom-events/insert-query-custom-mobile-app-events-attributes#event-definition) 보여줍니다. [레코드 탐색경로 SDK를](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-breadcrumb/) 사용하여 비정상 종료를 진단하는 데 도움이 될 것으로 생각되는 앱 활동을 추적하는 사용자 정의 `MobileBreadcrumb` 이벤트를 생성할 수 있습니다.

충돌 이벤트 추적에서 `MobileHandledException` 이벤트를 사용하여 디버깅을 도울 수도 있습니다. iOS 및 Android [`recordHandledException`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/) SDK 메서드를 사용하여 애플리케이션에서 예외가 처리되는 위치에 주석을 답니다. 이러한 이벤트는 충돌 이벤트 트레일에 자동으로 나타납니다.

사용자 지정 데이터로 충돌 이벤트 추적에 주석을 추가하는 방법에 대한 자세한 내용은 [모바일 모니터링에 사용자 지정 데이터 추가](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile) 를 참조하십시오.

## 이벤트 추적 사용 [#use]

충돌 이벤트 추적을 사용하려면:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (모바일 앱 선택) > Crash analysis** 으로 이동합니다.
2. **충돌 분석** 페이지의 오른쪽 하단에서 충돌 유형을 선택합니다.
3. **크래시 세부 정보** 페이지의 스택 추적 아래에서 **이벤트 트레일 을**선택합니다.
4. 충돌 원인에 대한 단서를 찾으려면 충돌 유형으로 이어지는 이벤트를 연구하십시오.
5. 이벤트 속성에 대한 세부 정보를 확장하려면 이벤트를 선택합니다.
6. New Relic에서 이벤트 추적 결과를 보려면 **Insights에서 세션 열기** 를 선택합니다.
7. 동일한 충돌 유형의 발생을 스크롤하려면 이벤트 추적의 왼쪽 및 오른쪽 화살표를 사용하십시오.

충돌 분석 도구를 최대한 활용하려면 다음을 사용하세요.

* 맞춤 [`MobileBreadcrumb`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-breadcrumb/) 또는 [`MobileHandledException`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/) 이벤트를 생성하는 모바일 SDK
* [`MobileRequest` 이벤트 활성화](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/analyze-network-requests-using-mobilerequest-event-data)
* [**충돌 분석** 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes)
