<Callout variant="important">
  모바일 **지도** UI는 2020년 12월 22일부터 더 이상 사용되지 않습니다. [서비스 지도](/docs/understand-dependencies/understand-system-dependencies/service-maps/introduction-service-maps) 는 각 모바일 엔터티의 왼쪽 탐색에서 사용할 수 있으며 아키텍처 표현을 시각화하고 사용자 지정하는 더 나은 방법입니다. 자세한 내용은 [Explorers Hub 게시물](https://discuss.newrelic.com/t/important-upcoming-changes-to-capabilities-across-browser-php-python-mobile-and-android-agents/123951) 을 참조하세요.
</Callout>

지도는 모바일 앱 또는 해당 서비스의 성능 문제를 찾는 데 도움이 됩니다. 이를 통해 앱과 다른 서비스의 관계 및 각 서비스가 다른 서비스에 미치는 영향을 명확하게 파악할 수 있습니다. 한 서비스가 실패하면 어떤 다른 서비스가 영향을 받는지 한 눈에 알 수 있습니다.

## 모바일 앱 서비스 지도 보기 [#map_procedures]

모바일 앱 및 관련 서비스를 아키텍처 맵으로 보려면 **[one.newrelic.com](https://one.newrelic.com) > Mobile > (모바일 앱 선택) > 모니터 > Monitor > Service map** 이동합니다. 자세한 내용은 [서비스 맵 설명서](/docs/understand-dependencies/understand-system-dependencies/service-maps/how-use-service-maps) 를 참조하십시오.

더 이상 사용되지 않는 모바일 **지도** 페이지를 사용해야 하는 경우 다음 단계를 따르세요.

1. **[one.newrelic.com](https://one.newrelic.com) > Mobile > (앱 선택) > Network > Map** 로 이동합니다.
2. 서비스에 대한 [HTTP 요청 세부 정보](/docs/mobile-monitoring-ui/http-requests-dashboard) 를 보려면 해당 이름을 선택합니다.
3. 서비스와 관련하여 [APM에서 모니터링하는 앱](/docs/applications-menu/applications-overview) 에 대한 세부 정보를 보려면 연결된 호스트 이름 아래에서 서비스 이름을 선택합니다.
4. 처리량 세부정보를 차트로 보려면 서비스 이름 아래에 있는 아이콘이나 `cpm` 막대를 선택하세요.
5. 서비스에 대한 자세한 메트릭을 보려면 처리량 차트 위로 마우스를 가져갑니다.

모바일 모니터링을 위한 **지도** 페이지는 모바일 앱과 앱이 사용하는 서비스에 대한 아키텍처 보기를 제공합니다.
