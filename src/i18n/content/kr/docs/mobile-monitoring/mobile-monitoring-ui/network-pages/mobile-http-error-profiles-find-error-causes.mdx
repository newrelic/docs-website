## HTTP 오류 프로필 보기 [#view]

모바일 모니터링 오류 프로필을 보려면:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile >** (앱 선택) **> More views > HTTP errors**로 이동합니다.
2. HTTP 오류 프로필 세부 정보를 보려면: HTTP 오류 UI 페이지 하단에서 테이블 위의 **HTTP 오류 프로필** 탭을 선택한 다음 HTTP 오류 프로필을 선택하여 확장합니다.

## HTTP 오류 프로필이란 무엇입니까? [#about]

**HTTP 오류 프로필** 은 성공적인 HTTP 요청(MobileRequest 이벤트)과 비교할 때 HTTP 오류 값(MobileRequestError 이벤트) 간에 발견되는 잠재적으로 의미 있는 상관 관계입니다. 이러한 상관 관계는 HTTP 오류의 원인과 기여 요인을 조사하는 데 도움이 됩니다.

## HTTP 오류 프로필 데이터 필터링 [#filter]

HTTP 오류 프로필 및 해당 비교 페이지 보기를 생성하는 데 사용되는 데이터를 추가로 필터링하려면 다음 [HTTP 오류 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/http-errors-network-failure-analysis#group-sort-filters) 기능을 사용하십시오.

* **시간 선택기** : 데이터를 생성하는 데 사용되는 시간 범위를 선택합니다.
* **필터** : 필터를 사용하여 선택한 속성 값이 있는 HTTP 오류 프로필 및 페이지 보기만 표시합니다. 여러 속성 값을 선택할 수 있습니다.

열 설명:

* 첫 번째 열은 **오류** 열입니다. 성공적인 HTTP 요청과 비교할 때 가장 편차가 큰 속성 값을 보여줍니다. 덜 중요한 속성 값은 **기타** 범주에 집계됩니다. 이 예에서 OS 버전 6.0, 5.1, 7.0 및 7.1.1은 Chrome 사용자에 대한 HTTP 오류의 100%를 구성합니다.

* **오류** 열에는 첫 번째 열의 속성 값이 있는 MobileRequestError 이벤트의 백분율이 표시됩니다. 이 열은 선택한 [시간 범위 및 필터 세트](#filter) 의 모든 HTTP 오류를 나타내는 최대 100%를 추가합니다. 이 예에서 HTTP 오류의 상당 부분은 Galaxy OS 버전 7.1.1에서 발생합니다. 이 문제를 더 조사하려면 OS 버전 7.1.1 데이터만 표시하도록 필터링할 수 있습니다.

* **요청** 열에는 첫 번째 열의 속성 값이 있는 MobileRequest 이벤트(성공적인 HTTP 요청)의 백분율이 표시됩니다. 이 예에서 요청의 14% 미만이 OS 버전 6의 사용자에 의해 이루어지지만 HTTP 오류의 24%는 해당 버전의 사용자에 의해 생성됩니다.

* **차이** 열에는 **오류** 열과 **요청** 열 간의 백분율 차이가 표시됩니다. 이 예에서

  <Icon name="nr-caret-up"/>

  11%는 요청보다 HTTP 오류 이벤트에 대한 속성이 11% 포인트 높음을 나타냅니다.
