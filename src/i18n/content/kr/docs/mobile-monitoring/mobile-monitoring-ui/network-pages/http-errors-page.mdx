## HTTP 오류 페이지 살펴보기 [#ui-interactions]

<DNT>**HTTP errors**</DNT> 페이지에서 HTTP 요청 및 네트워크 오류를 조사합니다.

* 표준 페이지 기능을 사용하여

  <DNT>
    **Errors and failures**
  </DNT>

  차트의 추세를 찾아보세요.

* 데이터를 그룹화, 정렬 및 필터링하여 특정 유형의 오류 및 실패를 대상으로 지정합니다.

* HTTP 오류 프로필을 사용하여 요청 오류에서 이상을 찾으십시오.

* 세부 정보를 보려면 오류 또는 실패를 선택하십시오.

중요한 서비스의 오류 유형에 초점을 맞춘 [NRQL 경고](/docs/alerts/new-relic-alerts/defining-conditions/create-alert-conditions-nrql-queries) 를 정의하거나 [앱 데이터를 쿼리할](#insights) 수도 있습니다.

## HTTP 오류 프로필 [#error-profiles]

오류 프로필은 HTTP 오류 이벤트에 대한 다양한 값의 빈도 차이에 대한 시각적 세부 정보를 제공합니다. 각 [속성](/docs/new-relic-solutions/get-started/glossary/#attribute) 에 대해 오류 프로필에는 다음이 포함됩니다.

* 가장 편차가 큰 값에 대해 오류 속성이 어떻게 분포되어 있는지 보여주는 히트맵
* 오류 속성의 분포를 다른 오류의 분포와 비교하는 레이블

<img
  title="HTTP error profiles"
  alt="Screenshot of http error profiles"
  src="/images/mobile_screenshot-full_http-error-profiles.webp"
/>

이렇게 하면 모바일 애플리케이션의 HTTP 오류를 해결할 때 추측을 더 많이 할 수 있습니다. 오류를 안전하게 무시할지 아니면 새로운 배포, 코드 변경, 고객 커뮤니케이션 또는 기타 작업으로 오류 해결을 시도해야 하는지 더 쉽게 결정할 수 있습니다.

## 단일 오류로 드릴다운 [#error-summary]

오류 또는 실패에 대한 세부 정보를 보려면 상위 20개 오류 목록에서 요청 URL을 선택하세요. 오류 요약 페이지에서 버전 정보, 요청 속성, 응답 본문을 보고 요청 URL에 대한 오류 유형 분석을 확인할 수 있습니다.

## 추적 세부정보 가져오기 [#trace-details]

모바일 모니터링은 400 또는 500 수준 상태 코드를 반환하는 HTTP 요청에서 응답 세부 정보를 캡처합니다. 또한 Android 앱에서 생성된 오류 메시지에는 스택 추적이 포함됩니다.

<DNT>**Errors**</DNT> 페이지의 오류 트레이스에 대한 세부정보를 보려면 해당 요청 URL 링크를 선택하세요. 여기에서 다음을 수행할 수 있습니다.

* 응답 본문을 봅니다.
* 이메일로 다른 사람과 오류 세부 정보를 공유하십시오.

오류 차트는 선택한 모바일 앱의 [요약](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/mobile-apps-summary-page/) 페이지에도 표시됩니다.

## 알 수 없는 오류 또는 URL 오류 문제 해결 [#troubleshoot-unknown-url]

모바일 에이전트는 예외 유형 목록을 유지 관리합니다. 경우에 따라 애플리케이션에서 발생하는 사용자 정의 예외가 이 목록에 포함되지 않습니다. 이 경우 모바일 <DNT>**Errors**</DNT> 페이지에 `Unknown` 나타날 수 있습니다.

오류 목록에서 `Unknown` 을(를) 찾고 누락된 예외 유형을 조사하는 데 도움이 필요한 경우 [support.newrelic.com](https://support.newrelic.com "링크가 새 창에서 열립니다.") 에서 지원을 받으십시오.
