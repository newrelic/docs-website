[모바일 모니터링](/docs/mobile-monitoring) 에는 모바일 앱과 관련된 HTTP 요청과 이러한 네트워크 호출이 성능에 미치는 영향을 더 잘 이해하는 데 도움이 되는 **HTTP 요청** UI 페이지가 있습니다.

이 문서에서는 엔터프라이즈 수준 **HTTP 요청** 페이지에 대해 설명합니다. 기업이 아닌 계정에는 [기존 HTTP 요청 페이지가 표시](#non-enterprise-http-requests) 됩니다.

## HTTP 요청 페이지 찾기 및 사용 [#find-page]

모바일 모니터링의 **HTTP 요청** 페이지를 보려면:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) > Network > HTTP requests** 으로 이동합니다.
2. [표준 페이지 기능](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) 을 사용하여 HTTP 분석 차트에서 추세를 찾으십시오.
3. 데이터를 [그룹화, 정렬 및 필터링](#groups-sort-filter) 하여 특정 요청 및 응답 속성을 대상으로 지정합니다.

## HTTP 요청 데이터 이해 [#understand-data]

가장 중요한 HTTP 요청 정보를 찾을 수 있는 곳은 다음과 같습니다.

<CollapserGroup>
  <Collapser
    id="response-time-chart"
    title="응답 시간 차트"
  >
    응답 시간 차트는 앱의 네트워크 호출이 백분위수에서 어떻게 수행되는지 보여줍니다. 이를 사용하여 평균 응답 시간을 첫 번째, 50번째, 99번째 백분위수와 비교합니다. 백분위수를 사용하면 평균 응답 시간을 예상보다 높게 만들 수 있는 이상값을 필터링할 수 있습니다.
  </Collapser>

  <Collapser
    id="errors-network-failures-chart"
    title="HTTP 오류 및 네트워크 오류 차트"
  >
    이 차트는 앱에서 발생한 실패한 네트워크 호출을 보여줍니다. 오류 및 실패에 대한 자세한 내용을 보려면 차트 제목을 선택하여 [**HTTP 오류** 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/network-errors-http-error-network-failure-analysis) 로 이동하십시오.
  </Collapser>

  <Collapser
    id="total-requests-chart"
    title="총 요청"
  >
    **총 요청** 별로 정렬하여 가장 자주 사용되는 네트워크 요청을 식별합니다. 이것이 도움이 될 수 있는 이유는 가장 느린 네트워크 호출은 드물게만 사용되는 반면, 더 자주 사용되는 요청은 가장 느리지 않더라도 최적화할 가치가 있기 때문입니다.
  </Collapser>
</CollapserGroup>

비 엔터프라이즈 **HTTP 요청** UI 페이지에 대한 설명은 [레거시 HTTP 요청](#non-enterprise-http-requests) 을 참조하십시오.

## HTTP 요청 데이터 보기 및 공유 [#insights]

Insights에서 **HTTP 요청** 차트를 보려면:

1. 고르다

   <Icon name="fe-more-horizontal"/>

   모든 차트에 대해.

2. **View query > View in Insights** 선택합니다.

3. 선택 사항: 데이터를 대시보드에 추가하거나 [영구 링크를 사용하여 공유합니다](/docs/data-analysis/user-interface-functions/share-your-data/permalink) .

4. 요청 데이터를 더 자세히 알아보려면 [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) 이벤트 및 [속성](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes) 을 쿼리하세요.

## 기존 HTTP 요청 UI 페이지 보기 [#non-enterprise-http-requests]

엔터프라이즈 수준 구독이 없는 계정에는 다른 **HTTP 요청** UI 페이지가 표시됩니다.

<CollapserGroup>
  <Collapser
    id="non-enterprise"
    title="기존 HTTP 요청 UI 보기"
  >
    상위 5개 도메인을 보거나 특정 HTTP 요청에 대한 세부 정보로 드릴다운하려면:

    1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > (앱 선택) > Network > HTTP requests** 으로 이동합니다.
    2. 선택 사항: **정렬** 기준 및 **1% 미만 처리량 숨기기** 옵션을 선택합니다.
    3. 앱의 모든 요청을 보거나 숨기려면 모두 **확장** 또는 **모두 축소** 를 선택합니다.
    4. 특정 호스트 또는 HTTP 요청(요청 시간, 평균 처리량 및 데이터 전송 포함)에 대한 세부 정보를 보려면 해당 이름을 선택합니다.
