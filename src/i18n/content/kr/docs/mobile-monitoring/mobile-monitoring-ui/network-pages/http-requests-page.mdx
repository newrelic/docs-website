<DNT>**HTTP requests**</DNT> UI 페이지는 모바일 앱과 관련된 HTTP 요청과 이러한 네트워크 호출이 성능에 어떤 영향을 미치는지 더 잘 이해하는 데 도움이 됩니다.

## HTTP 요청 페이지 보기 [#find-page]

<DNT>**HTTP requests**</DNT> 페이지를 보려면:

1. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Network > HTTP requests**
   </DNT>

   으)로 이동합니다.

2. [표준 UI 기능을](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) 사용하여 HTTP 분석 차트의 추세를 찾아보세요.

3. 데이터를 [그룹화, 정렬 및 필터링](#groups-sort-filter) 하여 특정 요청 및 응답 속성을 대상으로 지정합니다.

## HTTP 요청 데이터 이해 [#understand-data]

가장 중요한 HTTP 요청 데이터를 찾을 수 있는 장소는 다음과 같습니다.

<CollapserGroup>
  <Collapser
    id="response-time-chart"
    title="응답 시간 차트"
  >
    응답 시간 차트는 앱의 네트워크 호출이 백분위수에서 어떻게 수행되는지 보여줍니다. 이를 사용하여 평균 응답 시간을 첫 번째, 50번째, 99번째 백분위수와 비교합니다. 백분위수를 사용하면 평균 응답 시간을 예상보다 높게 만들 수 있는 이상값을 필터링할 수 있습니다.
  </Collapser>

  <Collapser
    id="errors-network-failures-chart"
    title="HTTP 오류 및 네트워크 오류 차트"
  >
    이 차트는 앱에서 겪고 있는 실패한 네트워크 호출을 보여줍니다. 오류 및 실패에 대한 자세한 내용을 보려면 차트 제목을 선택하여 [<DNT>**HTTP errors**</DNT> 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/network-errors-http-error-network-failure-analysis) 로 이동하세요.
  </Collapser>

  <Collapser
    id="total-requests-chart"
    title="총 요청"
  >
    가장 자주 사용되는 네트워크 요청을 식별하려면 <DNT>**Total requests**</DNT> 으로 정렬하세요. 이것이 도움이 될 수 있는 이유는 가장 느린 네트워크 호출이 거의 사용되지 않는 반면, 더 자주 사용되는 요청은 가장 느리지 않더라도 최적화할 가치가 더 높을 수 있기 때문입니다.
  </Collapser>
</CollapserGroup>

비엔터프라이즈 <DNT>**HTTP requests**</DNT> UI 페이지에 대한 설명은 [구형 HTTP 요청을](#non-enterprise-http-requests) 참조하세요.

## HTTP 요청 데이터 보기 및 공유 [#insights]

인사이트에서 <DNT>**HTTP requests**</DNT> 차트를 보려면:

1. 고르다

   <Icon name="fe-more-horizontal"/>

   모든 차트에 대해.

2. <DNT>
     **View query > View in Insights**
   </DNT>

   선택합니다.

3. 선택 사항: 데이터를 대시보드에 추가하거나 [영구 링크를 사용하여 공유합니다](/docs/data-analysis/user-interface-functions/share-your-data/permalink) .

4. 요청 데이터를 더 자세히 알아보려면 [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) 이벤트 및 [속성](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes) 을 쿼리하세요.
