모바일 모니터링에는 다음을 통해 최상위 버전을 포함하여 애플리케이션 버전 간의 채택 및 성능에 대한 비교 분석이 포함됩니다.

* 상호작용 시간
* 활성 세션
* 오류율

**버전** 페이지에는 생성된 날짜, 평균 메모리, 평균 CPU, 분당 평균 세션, 활성 앱당 분당 평균 요청 수(RPM)별로 각 버전을 비교하는 표도 포함되어 있습니다. 특정 버전에 대한 추가 세부 정보로 드릴다운할 수도 있습니다.

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) > Usage > Versions:** **버전** 분석에는 모바일 앱 사용에 대한 색상 코드 차트와 모바일 앱 버전 및 평균 메모리, CPU, 활성 사용자 및 네트워크 RPM(분당 요청).
</figcaption>

**버전** 페이지는 감지된 앱의 모든 버전 목록과 함께 지난 7일 동안 활성화된 모든 버전에 대한 개요 정보를 제공합니다. 비교 분석을 보려면:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) > Usage > Versions** 으로 이동합니다.
2. 기간을 선택하려면 New Relic 메뉴 표시줄 아래의 [**time picker**](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)\[시간 선택기를] 사용하십시오.
3. 선택 사항: **정렬 기준** 옵션을 선택합니다.
4. 특정 **버전** 에 대한 세부정보만 보려면 해당 이름을 선택합니다.

**Versions**\[버전] 페이지는 선택한 기간에 활성화된 앱의 모든 버전 목록을 제공합니다. 자세한 정보를 드릴다운하려면 적용 가능한 [플랫폼 UI 기능을](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)사용하십시오.

## 드릴다운 세부 정보 보기 [#details]

<img
  title="screen versions details.png"
  alt="screen versions details.png"
  src={mobileVersionDetails}
/>

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (앱 선택) > Usage > Versions > (선택한 버전):** 다음은 선택한 버전에 대한 세부 정보의 예입니다.
</figcaption>

세부 정보 페이지는 선택한 버전을 참조 버전(최신 또는 인기 있는 버전)과 비교하는 방법 및 앱의 다른 버전 평균에 대한 자세한 정보를 제공합니다. 시계열은 오류율, 응답 시간, 활성 세션 및 메모리 사용량에 대한 비교를 보여줍니다.

세부 정보 페이지를 종료하려면 **닫기** ( **X** ) 버튼을 선택합니다.

## 버전 번호를 얻는 방법 [#how-obtain-version]

모바일 모니터링이 버전 번호를 얻는 방법은 플랫폼에 따라 다릅니다.

<CollapserGroup>
  <Collapser
    id="android-ver-num"
    title="Android"
  >
    [Android 에이전트](/docs/mobile-monitoring/new-relic-mobile-android/get-started/android-installation-configuration) 는 매니페스트의 `android:versionName` 속성에서 버전 정보를 가져옵니다.
  </Collapser>

  <Collapser
    id="ios-ver-num"
    title="iOS"
  >
    iOS 에이전트는 `CFBundleShortVersionString` 및 `CFBundleVersion` 속성을 모두 사용하여 앱 버전을 가져옵니다. 에이전트는 iOS API를 통해 해당 속성에 액세스합니다. `info.plist` 파일을 읽어서 가져오지 않습니다.
  </Collapser>
</CollapserGroup>
