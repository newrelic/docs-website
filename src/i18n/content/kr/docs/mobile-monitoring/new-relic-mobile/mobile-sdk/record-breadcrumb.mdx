      ## 통사론 [#syntax]

      ### 자바 [#java]

      ```java
      NewRelic.recordBreadcrumb(
                  string $Name,
                  map<string, object> $eventAttributes
              )
      ```

      ### 코틀린 [#kotlin]

      ```kotlin
      NewRelic.recordBreadcrumb(
                  breadcrumbName: String?,
                  attributes: MutableMap<String?, Any?>?
              )
      ```

      ## 설명 [#description]

      이 호출은 NRQL 및 [충돌 이벤트 추적](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail) 으로 쿼리할 수 있는 `MobileBreadcrumb` 이벤트를 생성하고 기록합니다. 모바일 이동 경로는 충돌 분석에 유용합니다. 충돌 문제 해결에 도움이 될 수 있는 앱 활동을 위해 생성되어야 합니다.

      선택한 맞춤 속성 외에도 이벤트에는 관련 [세션 속성](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list) 도 있습니다. [`setAttribute`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/create-attribute/) 사용과 달리 탐색경로 이벤트에 속성을 추가하면 해당 이벤트에만 속성이 추가됩니다. 이는 세션 속성이 아닙니다.

      ## 반환 값 [#return-values]

      이벤트가 성공적으로 기록되면 `true` 을 반환하고 그렇지 않으면 `false` 을 반환합니다.

      ## 예 [#examples]

      다음은 간단한 이동 경로 이벤트 호출의 예입니다.

      ### 자바 [#java]

      ```java
      Map attributes = new HashMap();
      attributes.put("attributeName1", "value1");
      attributes.put("attributeName1", 2);
      boolean eventRecorded = NewRelic.recordBreadcrumb("Name", attributes);
      ```

      ### 코틀린 [#kotlin]

      ```kotlin
      val map = mutableMapOf<String,Any>("attributeName1" to "value1", "attributeName1" to 2)
      val eventRecorded = NewRelic.recordBreadcrumb("Name", map)
      ```

      다음은 실제 값이 포함된 이동 경로 이벤트의 예입니다.

      ### 자바 [#java]

      ```java
      Map attributes = new HashMap();
      attributes.put("button", "sign-in");
      attributes.put("location", "LaunchFragment");

      NewRelic.recordBreadcrumb("user tapped sign in button", attributes);
      ```

      ### 코틀린 [#kotlin]

      ```kotlin
      val attributes = mutableMapOf<String,Any>("button" to "sign-in", "location" to "LaunchFragment")
      NewRelic.recordBreadcrumb("user tapped sign in button", attributes)
      ```

      선택한 맞춤 속성 외에도 이벤트에는 관련 [세션 속성](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list) 도 있습니다. [`setAttribute`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/create-attribute/) 사용과 달리 탐색경로 이벤트에 속성을 추가하면 해당 이벤트에만 속성이 추가됩니다. 이는 세션 속성이 아닙니다.
