      ## 통사론 [#syntax]

      ### 자바 [#java]

      ```java
      NewRelic.withApplicationVersion(string $appVersion)
      ```

      ### 코틀린 [#kotlin]

      ```kotlin
      NewRelic.withApplicationVersion(appVersion: String?)
      ```

      ## 설명 [#description]

      기본적으로 New Relic Android 에이전트는 `AndroidManifest.xml` 또는 `build.gradle` 에서 앱 버전을 가져옵니다. 이 메서드를 사용하면 해당 값을 재정의하고 앱 버전을 전달하여 앱 버전을 설정할 수 있습니다. 앱 버전은 New Relic UI에 드롭다운으로 표시되며 이를 사용하여 상호 작용을 정렬할 수 있습니다.

      ## 예시 [#example]

      ### 자바 [#java]

      ```java
      NewRelic.withApplicationToken("").withApplicationVersion("1.23.4").start(this.getApplication());
      ```

      ### 코틀린 [#kotlin]

      ```kotlin
      NewRelic.withApplicationToken("").withApplicationVersion("1.23.4").start(applicationContext)
      ```

      ## 통사론 [#syntax]

      ### 목표-c [#objc]

      ```objectivec
      setApplicationVersion:(NSString *_Nonnull)versionString;
      ```

      ### 빠른 [#swift]

      ```swift
      NewRelic.setApplicationVersion:(NSString*)versionString;
      ```

      ## 설명 [#description]

      기본적으로 New Relic은 빌드 번호를 보고할 때 `CFBundleShortVersionString` 를 사용합니다. 그러나 `startWithApplicationToken` 호출하기 전에 이 메서드를 호출하여 보고된 빌드 번호를 재정의할 수 있습니다.
