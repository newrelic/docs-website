      ## 통사론 [#syntax]

      ### 자바 [#java]

      ```java
      NewRelic.setMaxEventPoolSize(int $maxSize)
      ```

      ### 코틀린 [#kotlin]

      ```kotlin
      NewRelic.setMaxEventPoolSize(maxSize: Int)
      ```

      ## 설명 [#description]

      이벤트 풀의 최대 크기를 설정합니다.

      기본적으로 <InlinePopover type="mobile"/>이벤트 하베스트당 최대 1,000개의 이벤트를 수집하며, 기본적으로 길이는 600초입니다. 이 방법은 다음 하베스트 주기까지 메모리에 저장되는 이벤트 풀의 최대 크기를 제어합니다. 풀 크기 제한에 도달하면 뉴렐릭 Android 에이전트는 [이벤트 샘플링을](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling) 시작하고 이벤트 풀이 다음 하베스트 서버와 함께 전송될 때까지 일부 이전 이벤트와 새 이벤트를 삭제합니다. 이 방법을 사용하면 해당 이벤트 풀의 최대 크기를 재정의할 수 있습니다.

      이벤트 수집 주기의 기본값은 600초입니다. 이벤트 수집 주기의 길이를 변경하려면 [최대 이벤트 버퍼 시간 설정을](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-buffer-time) 참조하십시오.

      <Callout variant="important">
        많은 수의 이벤트를 보고하거나 이벤트를 너무 자주 보고하면 앱 성능에 영향을 줄 수 있습니다.
      </Callout>

      ## 통사론 [#syntax]

      ### 오브젝티브-C [#objc]

      ```objectivec
      + (void) setMaxEventPoolSize:(unsigned int)size;
      ```


      ## 설명 [#description]

      이벤트 풀의 최대 크기를 설정합니다.

      기본적으로 iOS 에이전트는 [수확 주기](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#harvest-cycle)당 최대 1000개의 이벤트를 수집합니다. 이 방법은 다음 수집 주기까지 메모리에 저장되는 이벤트 풀의 최대 크기를 제어합니다. 풀 크기 제한에 도달하면 New Relic iOS 에이전트는 이벤트 풀을 다음 수확 주기와 함께 전송할 때까지 [이벤트 샘플링을](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling)시작하여 이전 및 일부 새 이벤트를 버립니다. 이 방법을 사용하면 해당 이벤트 풀의 최대 크기를 재정의할 수 있습니다. `Agent.start()` 호출된 후에 이 값을 설정해야 합니다.

      이벤트 수집 주기의 기본값은 600초입니다. 이벤트 수집 주기의 길이를 변경하려면 [최대 이벤트 버퍼 시간 설정](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-buffer-time/) 도 참조하십시오.

      새 값이 항상 적용되도록 하려면 이 API 호출을 `applicationDidBecomeActive` iOS 수명 주기 메서드에 배치합니다.
