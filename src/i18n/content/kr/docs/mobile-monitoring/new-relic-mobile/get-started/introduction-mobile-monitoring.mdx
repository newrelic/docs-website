## 앱 성능을 가속화할 수 있는 기회 파악 [#speed-performance]

* [대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) 를 만들어 성능 데이터를 쿼리, 시각화 및 공유합니다.
* [처리된 예외](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/introduction-mobile-handled-exceptions) 를 식별하여 워크플로를 간소화합니다.
* [충돌 프로필](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes/#profiles) 과 [HTTP 프로필을](/docs/errors-inbox/group-request-errors/#error-profiles) 사용하여 세션, 장치, 지리적 위치, 운영 시스템, 통신사, 요청 및 응답 등에 대한 추세를 확인하세요.
* 실행 시간, 오류 등에 대한 [경고 알림](/docs/mobile-monitoring/new-relic-mobile/getting-started/alert-information-new-relic-mobile) 을 설정합니다.
* [사용자 정의 속성 및 이벤트](/docs/insights/insights-data-sources/custom-events/insert-custom-events-attributes-mobile-data) 를 정의하여 조직에 가장 중요한 모바일 데이터에 대한 구체적인 추가 세부 정보를 얻으십시오.

## 백엔드 문제 모니터링 [#backend]

모바일 모니터링 UI의 [네트워크 페이지를](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/analyze-network-requests-using-mobilerequest-event-data/) 사용하여 HTTP 요청, 오류 및 기타 네트워크 문제를 통해 표면화되는 문제를 식별할 수 있습니다. 또한 [데이터를 조회](/docs/nrql/nrql-examples/nrql-query-examples-mobile-monitoring/) 하고 맞춤설정을 생성하여 <InlinePopover type="dashboards" />백앤드 API 개발자와 같은 관련 팀과 공유할 수도 있습니다.
