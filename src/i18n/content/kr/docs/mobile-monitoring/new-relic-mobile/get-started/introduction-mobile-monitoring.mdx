New Relic의 모바일 모니터링 기능을 사용하면 [Android](/docs/mobile-monitoring/new-relic-mobile-android/get-started/introduction-new-relic-mobile-android) 및 [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/get-started/introduction-new-relic-mobile-ios) 애플리케이션 성능을 분석하고 충돌 문제를 해결하는 방법에 대한 더 깊은 가시성을 얻을 수 있습니다. 또한 예기치 않은 지연에 대해 HTTP 및 기타 네트워크 성능을 검사할 수 있습니다. 그러면 백엔드 팀과 보다 효율적으로 협업하는 데 도움이 됩니다.

New Relic 계정을 만들고(영원히 무료입니다!) Android, iOS 또는 Cordova 환경에 대한 [모바일 모니터링을 설치](#new_users) 한 후 환경에 맞게 모바일 모니터링을 구성할 수 있습니다.

## 모바일 모니터링을 사용하는 방법 [#features]

New Relic에는 고객이 모바일 장치에서 New Relic을 사용할 수 있도록 iOS 및 Android 앱을 만드는 자체 모바일 개발자가 있습니다. 기능을 배포하는 동안 Android 앱에서 극단적인 경우 충돌을 일으키는 작은 버그가 발생했습니다. 이것은 우리가 새로운 기능을 생성, 테스트 및 배포한 후에 발생했습니다.

우리는 New Relic에서 일하기 때문에 모니터링 도구를 사용하여 모바일 앱을 분석하고 문제를 해결합니다. 모바일 모니터링의 힘을 탐구하는 방법에 대한 아이디어를 제공하기 위해 다음은 당사 모바일 엔지니어가 이 작은 버그를 추적하는 데 사용하는 방법의 예입니다.

## 프론트엔드 및 백엔드 성능 향상 [#backend]

충돌 분석을 위한 도구 이상인 당사의 모바일 모니터링 기능은 앱 성능을 가속화할 기회를 식별하는 데 도움이 될 수 있습니다. 예를 들어 다음을 수행할 수 있습니다.

* [대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) 를 만들어 성능 데이터를 쿼리, 시각화 및 공유합니다.
* [처리된 예외](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/introduction-mobile-handled-exceptions) 를 식별하여 워크플로를 간소화합니다.
* [충돌 프로필](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes) 및 [HTTP 프로필](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/mobile-http-error-profiles-find-error-causes) 을 사용하여 세션, 장치, 지리적 위치, 운영 체제, 이동통신사, 요청 및 응답 등의 추세를 확인합니다.
* 실행 시간, 오류 등에 대한 [경고 알림](/docs/mobile-monitoring/new-relic-mobile/getting-started/alert-information-new-relic-mobile) 을 설정합니다.
* [사용자 정의 속성 및 이벤트](/docs/insights/insights-data-sources/custom-events/insert-custom-events-attributes-mobile-data) 를 정의하여 조직에 가장 중요한 모바일 데이터에 대한 구체적인 추가 세부 정보를 얻으십시오.

우수한 모바일 앱은 제품 정보 요청, 사용자 인증 또는 미디어 콘텐츠 스트리밍과 같은 기능을 강화하기 위해 백엔드 시스템에 점점 더 의존하고 있습니다. 그러나 일부 백엔드 문제는 다음을 포함하여 최종 사용자에게 영향을 미치는 업스트림 모바일 앱 충돌, 정지 및 기능 손상으로 이어질 수 있습니다.

* 변경되거나 버전이 지정되지 않은 API 엔드포인트
* 잘못된 페이로드
* API 시간 초과
* 느리거나 더 이상 사용되지 않는 API 엔드포인트

모바일 모니터링 UI의 [네트워크 페이지를](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages) 사용하여 HTTP 요청, 오류 및 기타 네트워크 문제를 통해 표면화되는 문제를 식별할 수 있습니다. [데이터를 쿼리](/docs/using-new-relic/data/understand-data/query-new-relic-data)할 수도 있습니다. 그런 다음 생성 <InlinePopover type="dashboards"/>이 정보를 백엔드 API 개발자 팀 및 조직의 다른 사람들과 공유하여 모바일 앱의 전반적인 최종 사용자 경험을 개선하는 데 도움을 줍니다.

또한 고객의 전반적인 디지털 경험을 개선할 수 있습니다.

* 최종 사용자가 모바일 앱과 [웹 브라우저](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) 를 경험하는 방식을 빠르게 파악할 수 있습니다.
* [실제](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) 또는 [시뮬레이션된](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) 최종 사용자 행동을 분석하여 로드 시간, 가용성 및 오류와 같은 메트릭을 연마하는 동시에 이러한 디지털 경험을 가능하게 하는 백엔드 서비스를 기반으로 합니다.
