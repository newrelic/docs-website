## 애플리케이션 토큰 보기 [#application-token]

귀하의 모바일 앱의 데이터는 뉴렐릭에 귀하의 앱 데이터를 인증하는 고유한 40자 16진수 문자열입니다. 귀하가 모니터링하는 각 모바일 앱에는 미국과 EU 데이터 센터 모두에 대한 고유한 의미가 있습니다.

모바일 앱의 모든 데이터는 모바일 장치에 설치된 앱 이름에 관계없이 New Relic 모바일 모니터링 UI의 동일한 위치에 보고됩니다. Flutter 또는 React Native와 같은 하이브리드 프레임워크를 사용하고 iOS 및 Android 플랫폼 모두에 배포하는 경우 iOS용 토큰과 Android용 토큰을 하나씩 사용해야 합니다.

애플리케이션 토큰을 보려면:

1. <DNT>
     **[one.newrelic.com > All Capabilities](https://one.newrelic.com/all-capabilities)**
   </DNT>

   으로 이동하여

   <DNT>
     **Mobile**
   </DNT>

   클릭하고 모니터링되는 앱을 선택합니다.

2. <DNT>
     **Application**
   </DNT>

   또는

   <DNT>
     **Installation**
   </DNT>

   선택합니다.

## 서버 측 구성 [#configure-server-side-settings]

서버 측 모바일 에이전트 설정은 사용자 장치에서 모바일 에이전트 구현, 배포의 동작을 원격으로 관리하고 조정하는 메커니즘을 제공합니다. 이 기능을 사용하면 새로운 앱 스토어를 배포하지 않고도 모니터링 설정을 동적으로 제어할 수 있습니다.

현재 **Application** \[애플리케이션] 페이지에서 다음 설정을 원격으로 수정할 수 있습니다.

* ANR(애플리케이션 응답 없음) 오류 추적(Android만 해당): ANR 오류 원인을 추적하고 보고할 수 있습니다. 이 기능은 기본적으로 활성화되어 있으며 Android 에이전트 7.4.1 이상이 필요합니다.

  <Callout variant="important">
    서버 측 설정은 항상 로컬 에이전트 설정보다 우선합니다. 동시에 활성화된 기기가 100,000대 미만인 모바일 앱의 경우 최종 사용자 기기에 변경 사항을 적용하는 데 최대 10분이 걸릴 수 있습니다. 활성 장치가 100,000개가 넘는 앱의 경우 몇 시간이 걸릴 수 있습니다.
  </Callout>

### UI에서 구성 [#ui-configuration]

UI 에서 모니터 앱에 대한 서버 측 설정 설정에 액세스하려면 <DNT>**[one.newrelic.com > All Capabilities](https://one.newrelic.com/all-capabilities)**</DNT> > <DNT>**Mobile**</DNT> (앱 선택) > <DNT>**Settings**</DNT> > <DNT>**Application**</DNT>로 이동하세요.
