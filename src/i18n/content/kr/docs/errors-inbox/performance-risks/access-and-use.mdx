## 성능 위험 수신함에 액세스 [#access]

계정 및 엔티티 수준에서 <DNT>Performance Risks inbox</DNT> 에 액세스할 수 있습니다:

* [엔티티 수준 액세스](#access-entity)
* [계정 수준 액세스](#access-account)

### 엔티티 수준 액세스 [#access-entity]

특정 엔티티의 <DNT>Performance Risks inbox</DNT> 에 액세스하여 해당 엔티티 및 관련 텔레메트리 데이터와 관련된 성능 문제를 확인할 수 있습니다. 이를 통해 특정 애플리케이션이나 서비스의 성능에 집중하고, 이것이 최종 사용자에게 어떤 영향을 미칠 수 있는지 이해할 수 있습니다.

**엔티티에 대한 <DNT>Performance Risks inbox</DNT> 에 액세스하려면:**

1. <DNT>[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services</DNT> 으)로 이동합니다.
2. 필요한 엔티티를 선택하십시오.
3. 엔티티 탐색 창의 <DNT>**Triage**</DNT> 섹션에서 <DNT>**Performance Risks**</DNT> 을(를) 클릭합니다.

### 계정 수준 액세스 [#access-account]

또한 계정 수준에서 <DNT>Performance Risks inbox</DNT> 에 액세스하여 계정 내 모든 엔티티 전반의 성능 문제에 대한 전체적인 뷰를 확인할 수 있습니다. 이를 통해 여러 애플리케이션이나 서비스에 영향을 미칠 수 있는 공통적인 성능 문제를 식별하고, 이에 따라 최적화 작업의 우선순위를 지정할 수 있습니다.

**계정의 <DNT>Performance Risks inbox</DNT> 에 액세스하려면:**

1. <DNT>[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox</DNT> 으)로 이동합니다.
2. 엔티티 탐색 창에서 <DNT>**Performance Risks**</DNT>을(를) 클릭합니다.

## 성능 위험 보기 및 분석 [#view-analyze]

<DNT>Performance Risks inbox</DNT>에 액세스하면 감지된 문제를 두 가지 방법으로 볼 수 있습니다: <DNT>**Triage**</DNT> 보기 및 <DNT>**Group performance risks**</DNT> 보기. 두 뷰 모두 동일한 기본 데이터를 표시하지만 이를 분석하는 다른 방법을 제공합니다.

### 분류 [#triage]

<DNT>**Triage**</DNT> 뷰는 유사한 성능 위험을 자동으로 처리하고 함께 그룹화합니다. 유사한 쿼리는 컴퓨트되고, 저장되며, 단일 성능 위험 그룹으로 병합됩니다. 예를 들어, `IN` 절의 변형과 같이 동일한 패턴을 공유하는 여러 쿼리는 개별 항목으로 나타나는 대신 하나의 항목으로 그룹화됩니다. 이를 통해 각 발생 건을 개별적으로 검토하지 않고도 가장 큰 영향을 미치는 문제를 더 쉽게 식별하고 우선순위를 지정할 수 있습니다.

애플리케이션 전반의 패턴을 빠르게 식별하기 위해 성능 문제의 사전 그룹화된 요약이 필요한 경우 <DNT>**Triage**</DNT> 뷰를 사용하세요.

<img title="Screenshot of the Triage view in Performance Risks inbox showing pre-grouped performance issues." alt="Screenshot of the Triage view in Performance Risks inbox showing pre-grouped performance issues." src="/images/performancerisks_triage.webp" />

### 그룹 성능 위험 [#group]

<DNT>**Group performance risks**</DNT> 뷰는 사전 컴퓨트된 그룹화 없이 원시 성능 위험 데이터를 표시합니다. 정규화된 쿼리 표현(예: 쉼표 및 물음표 플레이스홀더)이 표시되며, 다음과 같은 다양한 속성을 기반으로 데이터를 필터링하고 그룹화할 수 있습니다:

* 성능 위험의 영향을 받는 사용자
* 호출을 생성하는 서비스
* 쿼리를 트리거하는 API

조사와 관련된 특정 속성을 기반으로 성능 위험을 분석해야 하는 경우 <DNT>**Group performance risks**</DNT> 뷰를 사용하십시오.

<img title="Screenshot of the Group performance risks view in Performance Risks inbox showing raw performance risk data." alt="Screenshot of the Group performance risks view in Performance Risks inbox showing raw performance risk data." src="/images/performancerisks_grpperrisk.webp" />

## 임계값 설정 [#set-thresholds]

<DNT>Performance Risks inbox</DNT> 구성 가능한 임계값을 사용하여 성능 이상 징후를 자동으로 감지합니다. 계정 및 엔티티 수준 모두에서 이러한 임계값을 업데이트하여 감지 기준이 특정 성능 모니터링 요구 사항 및 우선순위에 부합하도록 할 수 있습니다. 이러한 임계값은 무엇이 성능 위험에 해당하는지 결정하며, 애플리케이션에서 가장 관련성이 높고 조치 가능한 성능 문제에 대한 알림을 받을 수 있도록 도와줍니다.

계정 또는 엔티티 수준에서 성능 위험 임계값을 설정할 수 있습니다:

* [계정별 임계값](#account-thresholds)
* [엔티티 수준 임계값](#entity-thresholds)

### 계정 수준 임계값을 설정합니다. [#account-thresholds]

계정 수준에서 임계값을 설정하면 계정 내의 모든 엔티티에 적용되는 기본 성능 위험 감지 기준을 설정할 수 있습니다. 이는 일관된 모니터링 표준을 유지하고 모든 애플리케이션이 동일한 성능 벤치마크를 기준으로 평가되도록 보장하는 데 유용합니다.

**계정 수준에서 임계값을 설정하려면:**

<CollapserGroup>
  <Collapser id="new-account-thresholds" title="새로운 계정 수준 임계값">
    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox &gt; Performance Thresholds**.</DNT> 으)로 이동합니다.
    2. **Accounts** \[계정] 탭에서 <DNT>**+ New**</DNT>을(를) 클릭합니다.
    3. <DNT>**Account**</DNT> 드롭다운 메뉴에서 새 임계값을 생성할 계정을 선택합니다.
    4. 각 분석기에 필요한 제한을 설정하십시오.
    5. 저장하려면 <DNT>**Submit**</DNT> 을(를) 클릭합니다.
  </Collapser>

  <Collapser id="edit-account-thresholds" title="기존 계정 수준 임계값 편집">
    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox &gt; Performance Thresholds**.</DNT> 으)로 이동합니다.
    2. **Accounts** \[계정] 탭에서 업데이트하려는 계정 옆에 있는 <DNT>**Edit**</DNT> 버튼을 클릭합니다.
    3. 각 분석기에 필요한 제한을 설정하십시오.
    4. 저장하려면 <DNT>**Submit**</DNT> 을(를) 클릭합니다.

    <Callout variant="tip">
      임계값을 수정하면, 누가 변경했는지 추적할 수 있도록 사용자 이름이 표시됩니다.
    </Callout>
  </Collapser>
</CollapserGroup>

### 엔티티 수준 임계값 설정 [#entity-thresholds]

엔티티 수준에서 임계값을 설정하면 고유한 성능 특성이나 요구 사항이 있을 수 있는 특정 애플리케이션 또는 서비스에 대한 성능 위험 감지 기준을 사용자 지정할 수 있습니다. 이는 다양한 유형의 애플리케이션, 환경 또는 비즈니스 우선순위에 맞게 모니터링을 맞춤화하는 데 특히 유용합니다.

**엔티티 수준에서 임계값을 설정하려면:**

다음 방법 중 하나로 특정 엔티티에 대한 임계값을 설정할 수 있습니다:

<CollapserGroup>
  <Collapser id="entity-triage" title="성능 위험 페이지에서">
    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services**.</DNT> 으)로 이동합니다.
    2. 필요한 엔티티를 선택하십시오.
    3. 엔티티 탐색 창의 <DNT>**Triage**</DNT> 섹션에서 <DNT>**Performance Risks**</DNT> 을(를) 클릭합니다.
    4. 오른쪽 상단 코너에 있는 <DNT>**Adjust performance thresholds**</DNT> 을(를) 클릭합니다.
    5. 각 분석기에 필요한 제한을 설정하십시오.
    6. 저장하려면 <DNT>**Submit**</DNT> 을(를) 클릭합니다.

    <Callout variant="tip">

  </Collapser>

  <Collapser id="entity-page" title="성능 임계값 페이지에서">

## 뉴렐릭 MCP 서버에서 성능 위험 수신함 도구를 사용하십시오. [#using]

[뉴렐릭 MCP 서버](/docs/agentic-ai/mcp/overview/)에서 <DNT>Performance Risks inbox</DNT> 도구(`list_performance_risks()`)에 액세스할 수도 있습니다. 이 도구는 특정 엔티티에 대해 감지된 모든 성능 위험을 위험 유형, 심각도, 발생 횟수와 같은 세부 정보와 함께 나열합니다. 이 도구를 사용하여 프로그래밍 방식으로 성능 위험 데이터를 가져올 수 있습니다.
