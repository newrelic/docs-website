로그 UI를 사용하여 다음을 수행합니다.

* 로그에서 흥미롭거나 중요한 패턴을 찾습니다.
* 특정 로그 줄 주변에 더 많은 컨텍스트를 조사하십시오.
* 필터 및 구문 분석 규칙을 사용하여 로깅 데이터를 탐색하고 조작합니다.
* 데이터를 쿼리하여 맞춤형 시각화를 생성하거나 대시보드에 추가
* 조직의 로그 데이터를 구성하고 데이터 파티션으로 쿼리 성능을 최적화하세요.
* 방지하려는 문제에 대한 경고 조건을 설정합니다.
* 이메일을 통해 정기적으로 쿼리 결과를 받으려면 [자동화된 NRQL 검색을 예약하세요](#workflow-search).

새로운 기능과 개선 사항에 대한 최신 정보를 얻으려면 [Logs 릴리스 정보에 대한 RSS 피드를](/docs/release-notes/logs-release-notes) 구독하십시오.

  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT>: 로그를 탐색하고 관리하려면 왼쪽 탐색 메뉴를 사용하세요. 자세한 정보를 보려면 로그 줄을 클릭하세요.

## 로그 UI 찾기 [#find-ui]

로그 UI를 찾으려면 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT> 에서 <DNT>**Logs**</DNT> 선택합니다.

## 로그 데이터 탐색 [#ui-workflow]

로그 UI의 왼쪽 탐색을 모든 로그, 속성, 패턴, 라이브 테일 로깅 및 쿼리를 통한 쉬운 워크플로우로 사용하십시오. 데이터 삭제 또는 구문 분석, 데이터 파티션 생성 및 경고 설정을 통해 로그 데이터를 관리합니다. [난독화 표현식 및 규칙을](/docs/logs/ui-data/obfuscation-ui)사용하여 로그의 민감한 데이터를 해시하거나 마스킹합니다. 중앙 탐색에서 특정 로그 및 해당 속성에 대한 자세한 정보를 얻으십시오.

로깅 데이터를 탐색하려면 이 기본 워크플로를 따르세요.

<CollapserGroup>
  <Collapser id="workflow-path" title="1. 로그 UI로 이동합니다.">
    <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT> 의 로그 UI로 이동합니다.

    [EU 지역 데이터센터를](/docs/using-new-relic/welcome-new-relic/get-started/our-eu-us-region-data-centers/) 사용하는 경우 <DNT>**[one.eu.newrelic.com](https://one.eu.newrelic.com) &gt; Logs**</DNT> 로 이동하세요.

    [뉴렐릭 탐색 메뉴를 맞춤설정](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/#custom-navigation) 하지 않은 경우 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT> 으로 이동하여 <DNT>**Browse data**</DNT> 클릭하고 <DNT>**Logs**</DNT> 선택하세요.
  </Collapser>

  <Collapser id="workflow-patterns" title="2. 패턴을 찾으십시오.">
    <img title="Logs patterns UI" alt="Screenshot of Log patterns UI" src="/images/logs_screenshot-crop_patterns.webp" />

    * 로그 메시지에서 의심스러운 급증이나 감소를 확인하려면 왼쪽 탐색 메뉴에서 <DNT>**[Patterns](/docs/logs/ui-data/find-unusual-logs-log-patterns/)**</DNT> 클릭하세요.
    * 특정 기간의 로그를 보려면 차트에서 해당 지점을 클릭(또는 영역을 클릭하고 드래그)하거나 시간 선택기를 사용하세요.
  </Collapser>

  <Collapser id="workflow-focus" title="3. 초점을 좁히십시오.">
    왼쪽 탐색 메뉴에는 초기 검색 결과의 범위를 좁히거나 이상값을 빠르게 찾는 데 도움이 되는 옵션이 포함되어 있습니다.

    * 어디서부터 시작해야 할지 잘 모르겠으면 왼쪽 탐색 메뉴에서 <DNT>**Attributes**</DNT> 클릭한 다음 필요에 따라 추가 값을 선택하세요. 예를 들어, `hostname` 속성 아래 나열된 호스트가 다른 호스트보다 훨씬 더 많은 오류 메시지를 생성하는 경우 해당 값을 선택하여 검색에 적용하십시오.
    * 로그 메시지를 더 쉽게 쿼리하고 이해할 수 있도록 [기본 제공 구문 분석 규칙](/docs/logs/ui-data/built-log-parsing-rules/) 을 사용하거나 특정 값과 일치하는 속성에 대한 [고유 구문 분석 규칙을 만드십시오](/docs/logs/ui-data/parsing/) .
    * 수집된 로그 데이터의 양을 관리하고 더 적은 수의 로그를 저장하려면 필요하지 않은 데이터 수집을 방지하는 [삭제 필터 규칙](/docs/logs/ui-data/drop-data-drop-filter-rules/) 을 만듭니다.
  </Collapser>

  <Collapser id="workflow-details" title="4. 로그 세부 정보를 검사합니다.">
    <img title="Logs details UI" alt="Screenshot of Log details UI including severity level" src="/images/logs_crop_attributes_ui_log_detail_side_panel.webp" />

    해당하는 경우 로그 메시지는 심각도 수준( `INFO` , `DEBUG` 등)을 나타냅니다. 세부 정보를 속성 테이블 또는 JSON으로 보려면 로그 메시지를 선택하십시오.

    * 로그 메시지에 포함된 속성을 보려면 로그 라인을 클릭하십시오.
    * 로그 세부 정보의 특정 값과 관련된 문제를 해결하는 데 도움이 되도록 속성 세부 정보에 대한 [주변 로그를 표시](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs) 합니다.
    * 매우 긴 메시지에서 더 자세한 정보를 얻으려면 [blob](/docs/logs/ui-data/long-logs-blobs/) 으로 저장된 데이터를 확장하십시오.
  </Collapser>

  <Collapser id="workflow-search" title="5. 쿼리 및 검색.">
    기본적으로 로그 UI에는 모든 로그가 표시되지만 키워드나 구로 검색하여 원하는 결과를 찾을 수도 있습니다. 예를 들어:

    ```
    process failed
    ```

    자동 완성 드롭다운이 있는 검색 필드를 사용하여 속성, 연산자 및 값을 선택할 수도 있습니다. 예를 들어:

    ```
    service_name equals my service
    ```

    쿼리가 필요한 세부 정보에 집중할 수 있도록 하려면 다음 중 하나를 수행하여 속성을 추가하거나 제거합니다.

    * 왼쪽 탐색 메뉴에서 <DNT>**Attributes**</DNT> 선택하고 원하는 값을 선택한 다음 <DNT>**Query logs**</DNT> 클릭합니다.

    * 로그 표에서 <DNT>**+**</DNT> 아이콘을 클릭하여 속성을 추가한 다음 이를 클릭하여 쿼리에 추가합니다.

      Lucene 구문 검색창에서 NRQL 쿼리로 전환하려면 검색창 옆에 있는 <DNT>**NRQL**</DNT> 버튼을 클릭하세요. NRQL과 Lucene 사이에는 직접적인 동등성이 없습니다. 이는 NRQL 쿼리가 종종 더 간단하고 복잡하지 않다는 것을 의미하지만 이는 동일한 데이터로 쿼리를 작성하는 데 유용한 시작을 제공합니다.

      로그 쿼리에 대한 자세한 내용은 [로그 쿼리 구문 docs 를](/docs/logs/ui-data/query-syntax-logs/)참조하세요.

      **검색 일정을 예약하세요**

      <Callout title="시사">
        이 기능은 아직 개발 중이지만 꼭 사용해 보시기 바랍니다!

        이 기능은 현재 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 미리보기 프로그램의 일부로 제공됩니다.
      </Callout>

      로그 UI에서 쿼리를 작성하고 실행한 후 자동으로 실행되도록 예약하고 이메일로 결과를 받을 수 있습니다.

      1. <DNT>**All logs**</DNT> 페이지에서 쿼리를 작성하고 실행합니다.

      2. 검색을 예약하려면 다음 옵션 중 하나를 사용하세요.

         * 오른쪽 상단 모서리를 클릭하세요 <Icon name="fe-more-horizontal" /> <DNT>**Schedule search**</DNT> 선택하세요.
         * 쿼리 추세 차트 아래에서 시계 아이콘을 클릭하세요.

      3. 공지 설정, 일정 및 출력 형식을 사용하여 [예약된 NRQL 검색을](/docs/nrql/using-nrql/schedule-nrql-searches) 구성하세요.

      예약된 모든 검색을 보고 관리하려면 <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Scheduled searches**</DNT> 로 이동하세요. [예약된 검색 관리](/docs/nrql/using-nrql/schedule-nrql-searches/#manage) 에 대해 자세히 알아보세요.
  </Collapser>

  <Collapser id="query-builder" title="6. NRDB 쿼리 빌더를 사용한 고급 로그 분석.">
    <img title="Advanced log analysis with NRDB query builder" alt="Screenshot of link to NRDB query builder" src="/images/logs_crop_nrql_button.webp" />

    [NRDB 쿼리 빌더를](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/) 사용하면 필터를 잃지 않고 로그 이벤트에 대한 NRQL 쿼리를 작성할 수 있습니다. 로그 검색창 오른쪽에 있는 <DNT>**NRQL**</DNT> 버튼을 클릭한 후 쿼리를 시작하세요.
  </Collapser>

  <Collapser id="workflow-related" title="7. 관련 로그를 가져옵니다.">
    가장 유용한 로그에 집중하면 다음 작업에 도움이 됩니다.

    * <DNT>**Optimizing performance:**</DNT> 계정 내에서 데이터를 구성하고 쿼리 성능을 최적화하려면 [데이터 파티션 규칙](/docs/logs/ui-data/data-partitions/) 을 만드세요.
    * <DNT>**Controlling data access:**</DNT> 어떤 사용자가 특정 로그 파티션에 액세스할 수 있는지 제한하려면 [데이터 액세스 제어를](/docs/accounts/accounts-billing/new-relic-one-user-management/data-access-control) 사용하여 사용자 그룹에 대한 정책을 만듭니다.
    * <DNT>**Reviewing deployments:**</DNT> 시스템이 배포 또는 기타 앱 변경에 어떻게 응답하는지 즉시 확인하려면 [실시간 추적 로깅](/docs/logs/troubleshooting/view-log-messages-real-time-live-tail) 을 활성화하십시오.
    * <DNT>**Bypassing unrelated details:**</DNT> 특정 값에 대한 모든 로그를 보려면 선택한 로그에 대한 <DNT>**Log details**</DNT> 의 속성 목록을 검토한 후 필요에 따라 속성을 계속 추가하거나 제거하세요.
    * <DNT>**Finding the root cause:**</DNT> 문제가 발생하기 전의 근본 원인이나 이벤트 이후의 영향을 파악하려면 <DNT>**Log details**</DNT> 에서 <DNT>**...**</DNT> 클릭하여 [주변 로그를 표시하세요](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs).
    * <DNT>**Getting more context:**</DNT> 앱 및 호스트에 대한 다른 텔레메트리 데이터와 관련된 로그를 보려면 [로그 인 컨텍스트를](/docs/logs/logs-context/logs-in-context) 사용하세요.
  </Collapser>

  <Collapser id="traces-in-context-overview" title="8. 분산 추적에 대해 자세히 알아보십시오.">
    필터를 사용하여 로그 집합의 범위를 좁힌 다음 특정 로그를 열면 관련 분산 추적을 볼 수 있습니다. 분산 추적을 설정하고 로그와 관련된 샘플링된 추적이 있는 한 이를 볼 수 있는 옵션이 표시됩니다. 기본 분산 추적 페이지로 이동하지 않고 추적 정보를 보는 빠른 방법입니다.

    <DNT>**Log details**</DNT> 창을 열고 <DNT>**Distributed trace**</DNT> 섹션이 표시되면 폭포 보기에서 스팬 트레이스를 볼 수 있는 두 가지 옵션이 있습니다.

    * 트레이스 이름을 직접 클릭하거나 오른쪽에 화살표가 있는 아이콘을 클릭하면 오류가 있는 트레이스 범위를 강조 표시하는 워터폴 포커스 보기가 열립니다.

      <img style={{ align: "left",width: "85%" }} title="Screenshot showing how to reach the log details page" alt="Screenshot showing how to reach the log details page" src="/images/logs_crop_traces_in_context.webp" />

    * 또는 <DNT>**Explore**</DNT> 클릭하여 모든 범위를 클릭할 수 있는 필터링되지 않은 폭포식 구조를 열 수 있습니다.

      <Callout variant="tip">
        추적 폭포 사용에 대한 도움이 필요한 경우 [분산 추적 UI 이해 및 사용을](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui/)참조하세요.
      </Callout>
  </Collapser>

    핵심 New Relic UI 기능을 사용하여 [데이터(특정 계정, 시간 범위, 지표 및 이벤트, 쿼리 빌더 등)를 탐색](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/) 하고 차트와 데이터를 공유하고 대시보드에 추가하는 등의 작업을 수행할 수 있습니다. 자세한 내용은 이 문서의 [예시를](#examples) 참조하세요.

## 내보내다 [#export]

공유할 로그의 하위 집합을 내보내려면 사용자는 로그 UI의 전용 내보내기 기능을 활용할 수 있습니다. 이 기능을 사용하면 CSV 또는 JSON 형식으로 최대 5000개의 로그 레코드를 내보낼 수 있습니다. 로그 기록은 현재 브라우저의 `/Downloads/` 폴더에 있는 컴퓨터에 로컬로 다운로드됩니다.

## 사용자 정의 가능한 테이블 뷰 [#customizable-table-views]

<DNT>Table settings</DNT> 옵션을 사용하여 로그 데이터와 상호 작용하는 방법을 사용자 정의합니다.

<img title="Logs Table Settings menu" alt="Screenshot of Table Settings menu" src="/images/logs_crop_table_settings_menu.webp" />

### 로그 보기 설정

로그 항목이 표에 표시되는 방식을 선택하세요.

* **요약:** 각 로그 항목을 한 줄에 표시하여 대용량 로그를 빠르게 검색할 수 있습니다.
* **확장됨:** 모든 세부 정보에 즉시 액세스할 수 있도록 기본적으로 각 로그 항목의 전체 내용을 표시합니다.

### 로그 세부 정보 탐색 설정

개별 로그 항목의 세부 정보를 보는 방법을 선택하세요.

* **사이드 패널:** 로그 항목을 선택하면 오른쪽에 사이드 패널이 열립니다. 패널에는 컨텍스트에 따른 로그를 포함한 전체 로그 세부 정보가 표시됩니다.

<img title="Logs side panel detail view" alt="Screenshot of log side panel detail view" src="/images/logs_crop_side_panel_detail_view_expanded.webp" />

* **표 행:** 표 내에서 로그 항목을 확장하여 전체 로그 내용을 보려면 로그 항목을 선택하세요. 로그 메시지에 JSON 개체가 포함되어 있으면 더 쉽게 분석할 수 있도록 자동으로 포맷됩니다. 또는 로그 행 위에 마우스를 올려놓고 <DNT>**Show log detail**</DNT> 클릭하여 측면 패널을 엽니다.

## 권한 [#permissions]

사용자 유형별 권한에 대한 자세한 내용은 [사용자 유형을](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/#logs-capabilities)참조하십시오.

`NRQL drop rules` 기능을 제한하고 삭제 필터 생성을 제한하는 맞춤 역할을 만들 수 있습니다. 필요한 경우 기본 사용자에게 제한을 추가할 수도 있습니다. [삭제 규칙에 대해 자세히 알아보세요](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions#capability-definitions).

## 보기 저장 [#saved-views]

저장된 보기에 로그 쿼리, 테이블 구성, 시간 범위 및 속성 그룹화를 저장하면 나중에 빠르게 돌아갈 수 있습니다. 보기를 구성한 후 로그 분석 보기를 저장하려면:

1. <DNT>**Your logs**</DNT> 섹션의 UI 페이지에서 오른쪽에 있는 <DNT>**Saved views**</DNT> 클릭한 다음 <DNT>**Create new**</DNT> 클릭합니다.
2. 현재 보기에 이름을 지정합니다.
3. 현재 보기에서 저장하려는 측면을 선택합니다.
4. 팀원의 액세스 권한을 선택합니다.

## 예 [#examples]

다음은 로그 UI를 사용하여 자세한 정보를 얻는 방법에 대한 몇 가지 예입니다.

<CollapserGroup>
  <Collapser id="alert" title="로그 데이터에서 경고를 만듭니다.">
    로그 UI에서 직접 알림 조건을 만들 수 있습니다.

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT> 으)로 이동합니다.

    2. 경고하려는 결과를 검색하십시오. 예: `service_name:"your service" "fatal error"` .

    3. 벨을 클릭하세요 <Icon name="fe-bell-off" /> 쿼리 바 옆에 있는 아이콘.

    4. 절차를 완료하십시오

    5. 경고 조건을 강화할 NRQL 쿼리를 검토합니다.

       공지 조건을 저장한 후 <InlinePopover type="alerts" />UI에서 볼 수 있으며 필요에 따라 추가로 변경할 수 있습니다.
  </Collapser>

  <Collapser id="dashboard" title="대시보드에 로그 차트를 추가합니다.">
    <DNT>**Logs**</DNT> UI에서 로그 데이터를 대시보드에 [테이블 위젯](/docs/query-your-data/explore-query-data/use-charts/chart-types/#widget-table) 으로 추가할 수 있습니다. 지표와 이벤트를 사용하여 로그 테이블을 만들거나 뉴렐릭의 쿼리 빌더를 만들 수도 있습니다.

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT> 으)로 이동합니다.
    2. 오른쪽에서 <DNT>**Saved views**</DNT> 클릭하고 저장된 보기를 선택합니다. 또는 플롯하려는 결과를 검색하십시오. 예를 들어, `service_name:"checkout service" "process failed"`.
    3. <DNT>**Add to dashboard**</DNT> 클릭한 다음 세부정보를 입력하여 로그 차트를 기존 대시보드 또는 새 대시보드에 테이블로 추가합니다.

    그런 다음 대시보드에서 다음을 수행할 수 있습니다.

    * 복사, 편집, 삭제 등과 같은 표준 대시보드 위젯 기능을 사용합니다.
    * 로그 행을 클릭하면 해당 행에 대한 세부 정보가 표시됩니다.
    * 열을 더 추가하려면 쿼리를 업데이트하세요.
    * 사용 가능한 다른 계정의 로그 데이터를 쿼리하고 대시보드에 더 많은 차트(예: 비교 데이터)를 추가합니다.
    * 추가 문제 해결, 해결을 위해 <DNT>**Logs**</DNT> UI로 직접 이동하려면 <DNT>**Open in logs**</DNT> 클릭하세요.
  </Collapser>

  <Collapser id="troubleshoot-error" title="오류를 해결합니다(컨텍스트에 로그인).">
    앱에서 오류가 발생했을 때 호스트에서 무슨 일이 일어났는지 더 잘 이해하려면 [컨텍스트에서 로그](/docs/logs/logs-context/configure-logs-context-apm-agents/) 를 볼 수 있어야 합니다. 그런 다음 관련 오류를 해결하려면:

    1. <DNT>**APM &gt; Errors inbox**</DNT> 또는 <DNT>**APM &gt; (select an app) &gt; Events &gt; Error analytics**</DNT> 로 이동하여 오류 트레이스를 선택하세요.

    2. 오류 트레이스 세부정보에서 <DNT>**Open in logs**</DNT> 클릭합니다.

    3. 관련 로그 세부 정보를 찾습니다.

    4. 오류를 생성하는 호스트를 식별하려면 원하는 항목에 대해 <DNT>**...**</DNT> 클릭한 다음 <DNT>**Show surrounding logs**</DNT> 클릭합니다.

       <img title="APM errors inbox UI with logs" alt="Screenshot of APM errors inbox UI with logs" src="/images/logs_screenshot-crop_surrounding-logs.webp" />
  </Collapser>

  <Collapser id="troubleshoot-latency" title="지연 문제 해결(컨텍스트 로그인)">
    이러한 방식으로 지연 문제를 해결하려면 [컨텍스트에서 로그](/docs/logs/logs-context/configure-logs-context-apm-agents/) 를 볼 수 있어야 합니다. 그런 다음 성능이 눈에 띄게 느려졌을 때 시스템이 어떻게 작동했는지 더 잘 이해하려면 다음을 수행하십시오.

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; services &gt; (select an app)**</DNT> 으)로 이동합니다.

    2. 앱의 <DNT>**Summary**</DNT> 페이지에서 <DNT>**Distributed tracing**</DNT> 클릭한 다음 특히 느린 트레이스를 선택합니다.

    3. 트레이스 <DNT>**Details**</DNT> 에서 <DNT>**Logs**</DNT> 클릭합니다.

       <img title="APM distributed tracing UI with logs" alt="Screenshot of APM distributed tracing UI with logs" src="/images/logs_crop_distributed_tracing_logs_ui.webp" />
  </Collapser>
</CollapserGroup>
