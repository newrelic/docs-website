**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**에서 뉴렐릭의 **로그** UI를 사용하여 몇 초 만에 로그 데이터를 빠르게 검색할 수 있습니다. 각 로그는 `log_summary` 열에 사용 가능한 속성을 나열합니다. 추가 세부 정보를 살펴보려면 강조 표시된 속성을 클릭합니다.

시작할 준비가 되셨습니까?아직 계정이 없으시면, 아래에서 무료 뉴렐릭 계정을 생성해 지금 바로 데이터 모니터링을 시작하십시오.<InlineSignup/>

## 쿼리 구조 [#query-structure]

**로그**를 사용하면 `new` 및 `relic` 같은 간단한 키워드나 `"new relic agent"` 같은 구를 검색 필드에 직접 입력하여 로그 데이터를 검색할 수 있습니다.

일반 용어는 로그의 `message` 속성에 대한 "포함" 검색입니다. 예를 들어, `"new relic agent"`는 더 자세한 `message: "*New Relic Agent*"`와 동일합니다.

다른 속성을 검색하려면 `source: "*new relic agent*"` 같이 속성 앞에 용어를 붙입니다. 자세한 내용은 아래의 [일반 연산자](#general-operators)를 참조하십시오.

키워드나 구를 연산자와 결합하여 보다 복잡한 쿼리를 구성할 수도 있습니다.

<Callout variant="tip">
  뉴렐릭의 로그 쿼리는 Lucene 쿼리 언어를 기반으로 하며 이 문서에 나열된 모든 Lucene 기능을 지원합니다. (만약 Lucene 기능이 나열되지 않으면 지원하지 않는 것입니다.) 몇 가지 유용한 예제를 보려면 이 [Lucene 튜토리얼](https://www.lucenetutorial.com/lucene-query-syntax.html)을 확인하십시오.
</Callout>

        `status: *202*`

        `status: "*/log/v1 202*"`

## 텍스트로 검색 [#query-text]

보다 구체적인 쿼리 결과를 반환하려면 텍스트 검색을 사용하여 키워드나 구를 결합합니다.

### 텍스트 연산자 [#text-operators]

로그 쿼리 구문은 다음 텍스트 연산자를 허용합니다.

    <tr>

      </td>

    <tr>

## 속성으로 검색 [#query-attributes]

속성 검색을 사용하여 쿼리 결과를 특정 속성이나 필드로 좁힐 수 있습니다.

### 일반 연산자 [#general-operators]

다음 연산자는 모든 유형의 속성에서 사용할 수 있습니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        Condition(조건)
      </th>

      <th>
        일반 연산자 예
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Equal `:`
      </td>

      <td>
        속성이 지정된 키워드와 동일한 경우 로그 결과를 검색합니다. 예: `hostname` 필드는 `chi`와 같습니다.

        `hostname:chi`
      </td>
    </tr>

    <tr>
      <td>
        Does not equal `- :`
      </td>

      <td>
        속성이 지정된 키워드와 일치하지 않는 로그 결과를 검색합니다. 예: `hostname` 필드가 `chi`와 같지 않습니다.

        `-hostname:chi`
      </td>
    </tr>

    <tr>
      <td>
        Contains `*`
      </td>

      <td>
        속성에 지정된 키워드가 포함된 로그 결과를 검색합니다. 예: 필드 `hostname`에는 `chi`가 포함됩니다.

        `hostname:*chi*`
      </td>
    </tr>

    <tr>
      <td>
        Does not contain `- *`
      </td>

      <td>
        속성에 지정된 키워드가 포함되지 않은 로그 결과를 검색합니다. 예: `hostname` 필드에 &#x7B; `chi` }가 포함되지 않습니다.

        `-hostname:*chi*`
      </td>
    </tr>

    <tr>
      <td>
        Starts with `*`
      </td>

      <td>
        속성이 지정된 지정된 키워드로 시작하는 로그 결과를 검색합니다. 예: `hostname` 필드는 `chi` }로 시작합니다.

        `hostname:chi*`
      </td>
    </tr>

    <tr>
      <td>
        Ends with `*`
      </td>

      <td>
        지정된 키워드로 속성이 끝나는 로그 결과를 검색합니다. 예: `hostname` 필드는 `chi` }로 끝납니다.

        `hostname:*chi`
      </td>
    </tr>

    <tr>
      <td>
        가지다
      </td>

      <td>
        지정된 필드가 있는 로그 결과를 검색합니다. 예: `user_name` 필드가 있습니다.

        `has:user_name`
      </td>
    </tr>

    <tr>
      <td>
        잃어버린
      </td>

      <td>
        지정된 필드가 누락된 로그 결과를 검색합니다. 예: `user_name` 필드가 누락되었습니다.

        `missing:user_name`
      </td>
    </tr>
  </tbody>
</table>

## NRQL 쿼리 확인 및 대시보드에 추가 [#create-dashboard]

로그 차트의 NRQL 버전을 보려면 해당 차트에서 줄임표 메뉴를 클릭하고 **View query **를 클릭합니다. 그러면 해당 차트를 생성한 NRQL의 뷰가 열립니다. 여기에서 쿼리를 수정하거나 대시보드에 추가할 수 있습니다.

Lucene 구문 쿼리에서 NRQL 쿼리로 전환하려면 쿼리 빌더 오른쪽에 있는 [**NRQL** 버튼](/docs/logs/ui-data/use-logs-ui#workflow-search)을 클릭합니다.

예를 들어 Apache 로그에서 503 응답 코드를 검색하고, `FACET`을 사용해 NRQL 쿼리로 변환하고, 파이 차트로 보고, 그 차트를 대시보드에 추가할 수 있습니다. 방법은 다음과 같습니다.

1. **모든 로그** UI 페이지에서 쿼리를 사용하여 503 응답 코드가 있는 모든 Apache 로그를 검색합니다. `logtype:apache response:503`
2. 로그 차트 타원 메뉴에서 **View query**를 클릭합니다.
3. 쿼리에서 `SINCE <timestamp> UNTIL <timestamp> TIMESERIES MAX` `FACET verb`로 바꾸고 **Run**을 클릭합니다. 메서드(동사)별로 그룹화된 응답 코드가 있는 표가 표시됩니다.
4. 차트 유형을 **Pie**로 변경합니다.
5. **Add to dashboard**를 클릭하고 단계를 완료합니다.
