라이브 아카이브 기능은 자주 쿼리하지 않을 로그 데이터의 장기 저장을 제공하여 저장 비용을 낮추고 해당 데이터에 대한 쿼리에 대한 추가 비용을 제공합니다. 전체 플랫폼 사용자는 라이브 아카이브를 사용할 수 있습니다. 라이브 아카이브 기능에는 다음 세 가지 청구 요소가 포함됩니다.

* <DNT>**Data ingest**</DNT>: 데이터 수집은 로그가 라이브 아카이브에 저장되었는지 여부에 관계없이 로그에 대해 한 번만 청구됩니다. 라이브 아카이브에 저장된 로그에는 추가 데이터 수집 비용이 없습니다.
* <DNT>**Compute**</DNT>: 라이브 아카이브 데이터 쿼리에 따라 컴퓨팅 단위(CCU)를 기준으로 청구서를 받게 됩니다.
* <DNT>**Storage**</DNT>: 이는 아래에 설명된 측정 단위인 GB 월로 관리됩니다.

<Callout variant="tip">
  라이브 아카이브를 사용하고 요금을 청구받으려면 사용량 기반 가격 책정 모델을 사용해야 합니다. 사용자 및 데이터 외에도 라이브 아카이브는 [GB Months](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions#gb-months) 및 [Advanced 컴퓨팅 용량 단위(CCU)](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions#compute-capacity-units) 로도 측정됩니다.
</Callout>

## 라이브 아카이브 스토리지로 간주되는 것은 무엇입니까? [#what-counts]

New Relic 가격 계획의 맥락에서 라이브 아카이브 스토리지는 장기간 저장된 데이터, 라이브 아카이브 보존, [모든 로그 데이터에 대해 설정된 데이터 보존](/docs/data-apis/manage-data/manage-data-retention/#adjust-retention) 에 추가하여 또는 정기적으로 저장된 데이터를 측정한 것입니다. 데이터 보존.

정규 데이터 수집 기간에 라이브 아카이브 보유 기간을 합산하면 전체 보유 기간이 됩니다. 로그의 정규 데이터 보존 기간에는 기본 보존 기간(존 데이터 옵션의 경우 30일, Data Plus의 경우 120일)과 추가 보존 기간(“연장 보존”)이 모두 포함됩니다.

예를 들어, 고객이 로그에 대해 3개월의 정기 데이터 보존 기간을 갖고 라이브 아카이브의 로그에 대해 9개월의 총 보존 기간을 설정한 경우 라이브 아카이브 보존 기간은 다음과 같습니다.

## 저장용량은 어떻게 계산하나요? [#calculate]

추가 기능인 Live Archive를 구매하면 스토리지가 GB/월 단위로 측정됩니다. GB 월은 매일 계산됩니다.

특정 날짜(UTC)가 끝날 때를 기준으로 라이브 아카이브에 저장된 바이트 수를 계산합니다. 라이브 아카이브에 저장된 바이트 수를 계산하기 위해 전 총 보존 일수부터 시작하여 시간을 기준으로 (총 보존 - 일반 데이터 보존) 일 동안 수집된 GB를 합산합니다. 그런 다음 해당 바이트 수를 해당 월의 일수로 나눕니다. 한 달 동안의 총 GB 월 수를 얻으려면 해당 월의 날짜에 대한 GB 월 수를 더하면 됩니다.

**Example** \[예]: 고객이 로그의 총 보존 기간을 60일로 구성하고 정기적인 데이터 보존 기간을 30일로 설정했다고 가정해 보겠습니다. 즉, 로그는 라이브 아카이브 스토리지에서 60일 중 마지막 30일을 보내고 총 60일이 지나면 로그는 라이브 아카이브에서 만료되어 더 이상 보관되지 않는다는 의미입니다. 고객이 몇 달 동안 데이터를 수집한 경우 하루에 1GB의 로그가 꾸준히 수집되면 30GB는 일반 로그 스토리지에 저장되고 나머지 30GB는 라이브 아카이브에 저장됩니다. 매일 1GB가 표준 스토리지에 수집되고 새로운 1GB가 라이브 아카이브에서 만료되어 삭제됩니다.

이 예의 목적을 위해 9월에는 다음을 수행합니다.

* 라이브 아카이브에 저장된 30GB를 계산하고
* 9번째 달은 30일이므로 1일에 1GB 월(30GB/30일)의 저장 공간 사용량을 기록합니다.
* 30일 Live Archives의 &quot;버킷&quot;이 한 달 내내 30GB로 가득 차 있기 때문에 9개월의 각 날짜에 대해 동일한 측정을 반복합니다.

각 1GB 월의 모든 날짜 측정값을 합산하면 10월 1일까지 총 30GB 월이 됩니다. 따라서 이 고객은 9월 한 달 동안 30GB의 월별 스토리지를 사용했습니다.

## 데이터는 언제 볼 수 있나요? [#when]

라이브 아카이브 데이터는 다음 조건이 충족된 후에만 액세스할 수 있습니다.

* **라이브 아카이브 기능 활성화**: 뉴렐릭 계정 내에서 라이브 아카이브 기능을 명시적으로 활성화했습니다.
* **로깅 보존 기간 충족**: 데이터가 처음 로그인된 이후 계정의 라이브 아카이브 보존 기간에 지정된 기간이 경과되었습니다.

예를 들어 1월 1일에 라이브 아카이브 기능을 활성화하고 라이브 아카이브 보존 기간을 30일로 설정하면 라이브 아카이브 데이터는 2월 1일 이후에 제공됩니다. 로깅 보존 기간이 60일, 90일 또는 120일로 설정된 경우 이 기능을 사용하려면 해당 보존 기간이 지날 때까지 기다려야 합니다.

## 기타 계산 세부 정보 [#other-calc-details]

계산의 몇 가지 추가 측면은 다음과 같습니다.

* 가격 책정 모델의 맥락에서 GB는 10억 바이트로 정의됩니다.
* 월별 라이브 아카이브 스토리지(GB/월)는 가장 가까운 정수로 반올림됩니다. 예를 들어 조직에서 한 달 동안 99.9GB 월을 사용한 경우 이는 100GB 월로 계산됩니다.
* 사용 등급의 특정 날짜에 대한 보존은 UTC 날짜에 설정된 마지막 값으로 간주됩니다. 일반 데이터 보존과 라이브 아카이브 스토리지 모두에 적용됩니다.
* 데이터가 사용 파이프라인에 들어갈 때의 타임스탬프는 사용량을 계산하는 데 사용되며, 로그 타임스탬프나 이전 날짜 또는 앞으로 날짜가 지정된 이벤트 타임스탬프가 아닙니다.
