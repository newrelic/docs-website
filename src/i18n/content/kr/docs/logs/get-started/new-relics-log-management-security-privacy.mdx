## 로그 난독화 [#log-obfuscation]

추가 필터링을 포함하여 뉴렐릭으로 보내는 데이터는 사용하는 [로그 전달자](/docs/logs/forward-logs/enable-log-management-new-relic) 의 설정에 따라 제어됩니다. 귀하의 데이터를 복제할 수 있는 권한은 귀하가 통제하므로 조직의 보안 지침에 따라 마스크, 난독화 또는 민감한 데이터 전송을 방지하세요.

또한 다음과 같은 로그 난독화 기능을 제공합니다.

* 귀하의 모든 뉴렐릭 로그 데이터에 대해 기본적으로 제공되는 자동 로그인 난독화입니다.
* 사용자 정의 난독화 규칙을 설정할 수 있는 로그인 난독화 기능입니다.

다음으로 이 두 가지에 대해 더 자세히 설명하겠습니다.

### 자동 로그 난독화 [#auto-obfuscation]

난독화 기능은 기본적으로 모든 뉴렐릭 계정에 활성화되어 있습니다. 신용카드나 주민등록번호 등의 항목에 나타나는 마스크 번호 패턴을 자동으로 제공하는 기능입니다. 구분 기호로 사용할 수 있는 공백 및 하이픈을 포함한 모든 정수는 `X`문자열로 대체됩니다. 이 자동 난독화는 `message` 필드에만 적용됩니다.

신용카드로 보이는 번호(13-16자리)는 `XXXXXXXXXXXXXXXX` 으로 난독화됩니다. 예를 들어:

* 하이픈이 포함된 숫자(예: `4111-1111-1111-1111`
* 공백이 있는 숫자(예: `4111 1111 1111 1111`
* 13자리(Visa), 14자리(Diner&apos;s Club), 15자리(American Express, JCB) 또는 16자리 숫자(Visa, Mastercard, Discover, JCB) `4111111111111111`

사회보장번호로 보이는 하이픈이 포함된 9자리 숫자(예: `123-45-6789`)는 `XXXXXXXXX` 로 난독화됩니다. `123 45 6789` 와 같이 공백이 있는 9자리 숫자나 `12-345-67-89` 과 같이 다른 패턴의 하이픈은 자동으로 난독화되지 않습니다.

자동 난독화를 선택 해제해야 하는 경우 [지원팀에 문의하세요](https://support.newrelic.com).

### 사용자 정의 로그 난독화 [#custom-obfuscation]

[Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus) 와 함께 사용할 수 있는 로그인 난독화 기능을 사용하면 데이터가 뉴렐릭에 도착할 때 데이터의 일부를 난독화하는 규칙을 설정할 수 있습니다. 이렇게 하면 모든 애플리케이션에서 민감한 데이터를 추적하고 소스에서 수정하려고 할 필요 없이 더 쉽고 안전해집니다. 자세한 내용은 [로그 난독화를](/docs/logs/ui-data/obfuscation-ui) 참조하세요.

## 데이터 접근 제어 [#data-access-control]

로그 난독화는 민감한 데이터를 보호하는 데 도움이 되지만, 특정 로그 데이터에 액세스할 수 있는 사용자를 제어해야 할 수도 있습니다. [데이터 액세스 제어를](/docs/accounts/accounts-billing/new-relic-one-user-management/data-access-control) 사용하면 특정 [로그 파티션](/docs/logs/ui-data/data-partitions/) 에 대한 사용자 액세스를 제한할 수 있습니다.

이 기능은 다음과 같은 경우에 유용합니다.

* 특정 팀만 접근해야 하는 민감한 정보가 포함된 로그 데이터입니다.
* 여러 팀이 각자의 서비스나 애플리케이션과 관련된 로그만 확인해야 합니다.
* 특정 로그 데이터에 누가 접근할 수 있는지 제한하는 규정 준수 요구 사항입니다.

데이터 접근 제어는 [역할 기반 접근 제어](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 와 함께 작동합니다. 역할은 사용자가 액세스할 수 있는 기능을 제어하는 반면, 데이터 액세스 제어는 사용자가 볼 수 있는 로그 파티션을 지정합니다.
