당사의 로그 관리 솔루션은 애플리케이션 및 인프라 성능 데이터(이벤트, 오류, 추적 등)에 대한 심층적인 가시성을 제공하여 MTTR(평균 해결 시간)을 줄이고 프로덕션 인시던트 문제를 신속하게 해결합니다.

## 문제를 더 빨리 찾고 컨텍스트 전환을 줄입니다. [#logs-definition]

로그 관리는 로그 데이터를 나머지 애플리케이션 및 인프라 데이터와 연결하는 방법을 제공합니다. 도구 간 전환으로 인해 컨텍스트를 잃지 않고 문제의 근본 원인을 빠르게 찾을 수 있습니다. 예를 들어 다음을 수행할 수 있습니다.

* 앱 및 호스트의 문제 컨텍스트에서 로그를 제공하는 New Relic UI에서 관련 데이터를 가져옵니다.
* 로그 UI를 사용하여 로그 데이터에 대해 자세히 알아보고 사용자 지정 차트, 대시보드 및 알림을 만드십시오.
* 로그를 즉시 검색합니다.
* 다른 도구 간에 전환하지 않고 성능 문제를 해결합니다.
* 한 곳에서 모든 것을 시각화하십시오.

## 로깅 데이터 가져오기 [#integrate-logs]

로그 데이터를 New Relic에 전달하려면 다음 옵션 중 하나를 사용할 수 있습니다.

* 앱의 APM 에이전트 [가 컨텍스트의 APM 로그를](/docs/apm/new-relic-apm/getting-started/get-started-logs-context) 사용하여 New Relic에 직접 로그 데이터를 전달하도록 합니다. 추가 타사 소프트웨어를 설치하거나 유지 관리할 필요가 없습니다!
* 앱 및 호스트의 로그를 위한 경량 데이터 수집기로 [인프라 모니터링을 위한 설치 안내를](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent) 사용하십시오.
* Amazon, Microsoft, Fluentd, Fluent Bit, Kubernetes, Logstash 등을 포함한 다양한 [로그 전달 플러그인](/docs/logs/forward-logs/enable-log-management-new-relic/) 중에서 선택하십시오.
* [Log API](/docs/logs/log-api/introduction-log-api/) 또는 [TCP 끝점](/docs/logs/log-api/use-tcp-endpoint-forward-logs-new-relic) 을 사용하여 로그 데이터를 보냅니다.
* OpenTelemetry SDK를 사용하여 앱에서 [OpenTelemetry 수집기](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/best-practices/opentelemetry-best-practices-logs) 로 로그를 보내면 OTLP를 통해 이를 New Relic으로 전달할 수 있습니다.

## New Relic에서 로깅 데이터 보기 [#find-data]

UI 또는 API로 로깅 데이터를 탐색할 수 있습니다.

* [one.newrelic.com](https://one.newrelic.com) 의 새로운 유물 UI
* 해당하는 경우 EU 지역 데이터 센터용 UI: [one.eu.newrelic.com](https://one.eu.newrelic.com)

`Log` 데이터 유형을 쿼리할 수도 있습니다. 예를 들어 NRQL을 사용하여 다음을 실행합니다.

```
SELECT * FROM Log
```

또한 GraphQL 형식 API인 NerdGraph를 사용하여 필요한 정확한 데이터를 요청할 수 있습니다.

## 사용자 액세스 [#requirements]

로그 관리 기능에 대한 액세스는 사용자 유형에 따라 제한됩니다. 자세한 내용은 [사용자 유형 기능](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#user-type-capabilities) 을 참조하십시오.

[New Relic One 사용자 모델 사용자](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 경우 일부 로그 기능은 여러 [로그 관련 기능](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#capabilities) 에 의해 관리됩니다.

## 무엇 향후 계획 [#what-next]

로그 관리 솔루션을 시작할 준비가 되셨습니까?

1. 아직 계정이 없다면 [New Relic 계정을 만드](https://newrelic.com/signup) 십시오. 영원히 무료입니다.
2. [컨텍스트에서 APM 로그](/docs/apm/new-relic-apm/getting-started/get-started-logs-context) , [인프라 에이전트](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/) 또는 기타 [로그 전달 솔루션](/docs/logs/forward-logs/enable-log-management-new-relic/) 을 사용하여 로그 관리를 활성화하십시오.
3. 오류, 분산 추적, 호스트 등을 포함 [하여 플랫폼 전체에서 관련 로그 데이터를](/docs/apm/new-relic-apm/getting-started/get-started-logs-context#response-time-example) 탐색합니다.
4. [로그 UI](/docs/logs/ui-data/use-logs-ui/) 를 모든 로그, 속성, 패턴, 라이브 테일 로깅 및 쿼리를 통해 쉬운 워크플로로 사용하십시오. 경고를 추가하고, 데이터를 쿼리하고, 대시보드를 만듭니다.
