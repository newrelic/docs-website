CloudFormation 그리드를 구현, 배포하면 S3 버킷에 트리거가 생성되고 CloudWatch 로그 그룹에 필터가 생성됩니다. 스택을 삭제하면 이러한 트리거와 필터도 제거됩니다.

S3 버킷에서 트리거를 수동으로 생성하거나 CloudWatch 로그 그룹에서 필터를 생성하고 대상을 뉴럴릭에서 생성한 Lambda 함수로 설정하는 경우에도 지원됩니다. 하지만 우리는 이런 접근 방식을 권장하지 않습니다. 스택을 삭제하더라도 수동으로 생성한 트리거나 필터는 제거되지 않습니다. 스택에 해당 트리거나 필터를 삭제할 권한이 없기 때문입니다.

이 지침은 효과적인 리소스 관리를 보장하고 스택 삭제 후에 의도치 않은 트리거나 필터가 남아 있는 것을 방지하는 데 중요합니다.
