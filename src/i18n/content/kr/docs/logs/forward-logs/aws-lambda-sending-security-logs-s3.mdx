[AWS 서버리스 애플리케이션 리포지토리](https://serverlessrepo.aws.amazon.com/applications) 에서 쉽게 배포할 수 있는 서버리스 앱 `newrelic-securitylake-s3-processor-LogForwarder`을(를) 사용하여 [Amazon Security Lake 로그를](https://docs.aws.amazon.com/security-lake/latest/userguide/internal-sources.html)New Relic으로 보낼 수 있습니다.

Security Lake에서 New Relic으로 로그를 전달하면 로그 데이터를 수집, 처리, 탐색, 쿼리 및 경고할 수 있는 향상된 로그 관리 기능이 제공됩니다.

## 서버리스 애플리케이션 설치 [#install-app]

Serverless 앱을 설치하여 Security Lake 로그를 New Relic에 전달하려면:

1. 당신이 가지고 있는지 확인

   <InlinePopover type="licenseKey"/>

   .

2. 브라우저에서 [AWS Serverless Application Repository를](https://serverlessrepo.aws.amazon.com/applications) 엽니다.

   <img title="AWS Lambda - Select region" alt="AWS Lambda - Select region" src={serverlessAWSLambdaSelectRegion}/>

3. `newrelic` 을 검색하고 **사용자 지정 IAM 역할 또는 리소스 정책을 생성하는 앱 표시를 선택하여** `newrelic-securitylake-s3-processor-LogForwarder` 을 찾습니다.

4. `newrelic-securitylake-s3-processor-LogForwarder` 세부정보를 클릭하고 **배포** 를 클릭합니다.

5. 필요한 매개 변수를 설정하려면 앱 지침을 따르십시오.

* `SecurityLakeSQSArn`
* `SecurityLakeSubscriberRoleArn`
* `ExternalID`
* `NRLicenseKey`

보안 레이크 공급자로부터 `SecurityLakeSQSArn` 및 `SecurityLakeSubscriberRoleArn` 를 받습니다.

이벤트 소스 지역은 Lambda 지역과 일치해야 합니다.
