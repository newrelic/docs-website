[Fluent Bit](https://fluentbit.io/) 에서 이미 로그 데이터를 모니터링하고 있는 경우 [Fluent Bit 출력 플러그인](https://github.com/newrelic/newrelic-fluent-bit-output) 을 사용하여 New Relic에서 로그 데이터를 전달하고 강화할 수 있습니다. 이 플러그인은 [DaemonSet](https://kubernetes.io/docs/concepts/workloads/controllers/daemonset/) 형태로 [Kubernetes](https://kubernetes.io/) 클러스터에 설치할 수 있는 독립 실행형 Docker 이미지로도 제공됩니다. 이것을 Kubernetes 플러그인이라고 합니다.

Kubernetes 로그를 New Relic으로 전달하면 로그 데이터를 수집, 처리, 탐색, 쿼리 및 경고하는 향상된 로그 관리 기능이 제공됩니다.

## 로그 관리를 위해 Kubernetes 활성화 [#enable-process]

Fluent Bit용 Kubernetes 플러그인을 사용하여 로그를 New Relic에 전달하려면:

1. 다음 사항이 있는지 확인하십시오.

* A [New Relic 라이센스 키](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)
* [Kubernetes](https://kubernetes.io/docs/setup/) 클러스터 배포됨

2. GitHub의 절차에 [따라 Kubernetes 플러그인을 설치하십시오](https://github.com/newrelic/helm-charts/tree/master/charts/newrelic-logging#installation) .
3. 선택 사항: 플러그인을 투구 차트로 설치하는 경우[다양한 구성](https://github.com/newrelic/helm-charts/tree/master/charts/newrelic-logging#configuration) 을 설정할 수 있습니다. 그러나 대부분의 사용자에게 유효하므로 표준 설정을 권장합니다.
4. 일부 트래픽을 생성하고 몇 분 동안 대기한 다음 데이터에 대해 [계정을 확인](#find-data) 하십시오.

## 로그 데이터 보기 [#find-data]

모든 것이 올바르게 구성되고 데이터가 수집되는 경우 다음 두 위치에서 데이터 로그가 표시되어야 합니다.

* [New Relic의 로그 UI](https://one.newrelic.com/launcher/logger.log-launcher)
* [NRQL 조회](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data)를 실행하는 새 도구. 예를 들어, 다음과 같은 쿼리를 실행할 수 있습니다.

```
SELECT * FROM Log
```

로그 관리 기능을 활성화한 후에도 데이터가 나타나지 않으면 [표준 로그 문제 해결 절차](/docs/logs/log-management/troubleshooting/no-log-data-appears-ui/) 를 따르십시오.

## 다음은 뭐지? [#what-next]

[로그 UI](/docs/logs/log-management/ui-data/use-logs-ui/) 를 사용하여 플랫폼 전반에 걸쳐 로깅 데이터를 탐색하십시오.

* [컨텍스트에서 로그](/docs/logs/enable-log-management-new-relic/configure-logs-context/configure-logs-context-apm-agents/) 기능을 사용하여 로그를 전달하여 애플리케이션 및 플랫폼 성능 데이터 모두에 대한 가시성을 확보하십시오.
* [경보](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/)를 설정하십시오.
* [데이터를 조회](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/) 하고 [대시보드를 작성](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/)하십시오.

## 로그 전달 사용 안함 [#disable]

로그 전달 기능을 비활성화하려면 [Kubernetes 설명서](https://github.com/newrelic/helm-charts/tree/master/charts/newrelic-logging#installation) 의 표준 절차를 따르십시오. New Relic에서는 다른 작업을 수행할 필요가 없습니다.
