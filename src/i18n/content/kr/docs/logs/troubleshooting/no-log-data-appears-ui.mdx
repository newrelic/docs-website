## 문제

New Relic에서 로그 관리 기능을 활성화한 후 약 5분 후에 로그 UI에 데이터가 표시되지 않습니다.

## 해결책

일부 로그 페이로드를 보내고 5분 정도 기다린 후에도 데이터가 나타나지 않으면 다음을 시도하십시오.

<table>
  <thead>
    <tr>
      <th>
        로그 문제 해결
      </th>

      <th>
        코멘트
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        데이터 액세스
      </td>

      <td>
        [기능 및 데이터에 대한 액세스에 영향을 미치는 요소를](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data/) 참조하십시오.
      </td>
    </tr>

    <tr>
      <td>
        호환성
      </td>

      <td>
        [호환되는 로그 전달자](/docs/logs/forward-logs/) 를 설치했는지 확인하십시오.
      </td>
    </tr>

    <tr>
      <td>
        상태 코드
      </td>

      <td>
        New Relic 로그 수집 끝점에서 반환되는 응답 상태 코드를 확인합니다. 예를 들어 다음이 표시될 수 있습니다.

        ```
        HTTP Error 403: Forbidden. Review your license key.
        ```

        이 오류는 잘못된 보안 키를 사용하고 있음을 의미합니다. New Relic은 로그 전달을 활성화하기 위해 [라이센스 키](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key) 가 필요합니다.

        `HTTP 202` 응답은 성공을 나타냅니다.
      </td>
    </tr>

    <tr>
      <td>
        오류
      </td>

      <td>
        [`NrIntegrationErrors` 이벤트](/docs/telemetry-data-platform/manage-data/nrintegrationerror/) 를 사용하여 쿼리를 실행하여 로깅과 관련된 오류가 있는지 확인합니다. 예를 들어 다음과 같은 메시지를 찾습니다.

        ```
        Error unmarshalling message payload
        ```
      </td>
    </tr>

    <tr>
      <td>
        질문 `Log`
      </td>

      <td>
        `Log` 데이터 유형을 쿼리해 보세요.

        ```
        SELECT * FROM Log
        ```

        쿼리 빌더에 데이터가 표시되지 않으면 로그 UI에 데이터가 표시되지 않습니다. 자세한 내용은 New Relic의 [데이터 쿼리 옵션](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/) 에 대한 설명서를 참조하십시오.
      </td>
    </tr>
  </tbody>
</table>
