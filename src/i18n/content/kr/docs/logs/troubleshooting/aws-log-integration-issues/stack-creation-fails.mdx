## 문제

다양한 이유로 CloudFormation 스택을 만드는 데 실패할 수 있습니다.

## 해결책

가능한 이유와 해결책:

* CloudFormation 스택을 생성하는 동안 선택한 IAM 역할에 충분한 [권한이](https://docs.aws.amazon.com/ARG/latest/userguide/gettingstarted-prereqs-permissions.html) 없을 수 있습니다.

* IAM 역할을 선택하지 않은 경우 계정에 필요한 [권한이](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#logs) 있는지 확인하세요.

* 다음과 같은 실패 메시지가 나타나면 해당 이름을 가진 기존 리소스를 삭제하거나 이름을 바꾼 다음 다시 시도하세요.

  `Resource type {resource-name} with identifier {resource-identifier-name} already exists.`

* 중첩된 CloudFormation 스택의 출력을 확인하여 잠재적인 오류를 파악합니다.

* 선택한 S3 버킷에 객체 생성 이벤트에 대한 이벤트 공지가 없는지 확인하세요.

* 선택한 CloudWatch 로그 그룹에 유사한 구독 필터가 없는지 확인하세요.
