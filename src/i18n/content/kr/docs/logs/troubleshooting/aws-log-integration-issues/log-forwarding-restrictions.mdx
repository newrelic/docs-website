뉴렐릭은 S3 또는 Firehose를 통해 로그를 수집할 때 로그 포워딩을 위한 AWS 통합에 다음과 같은 제한 사항이 있습니다.

### S3에서 AWS 로그를 수집하려면 [#s3-restrictions]

* 로그 파일 크기는 80MB보다 작아야 합니다.
* 단일 로그 줄의 크기는 8MB보다 작아야 합니다.
* 1mb와 8mb 사이의 로그 줄은 여러 개의 메시지로 분할되어 뉴렐릭으로 전달됩니다.
* 뉴렐릭은 현재 클라이언트 JSON이나 로그 라인 구문 분석을 지원하지 않습니다.
* 뉴렐릭은 현재 로그 파일에 대해 `Gzip` 및 `Bzip2` 압축 형식만 지원합니다. 다른 압축 형식의 파일을 압축 해제된 파일로 처리합니다.

### CloudWatch에서 로그를 수집하려면 [#cloudwatch-restrictions]

* 뉴렐릭은 현재 클라이언트 JSON이나 로그 라인 구문 분석을 지원하지 않습니다.
* 뉴렐릭은 현재 구문 분석 논리에 [Lambda 레이어](https://github.com/newrelic/newrelic-lambda-layers) 만 지원합니다.
