뉴렐릭 AI는 사용자의 시스템과 뉴렐릭 플랫폼에 대한 이해를 돕기 위해 LLM(LLM)과 뉴렐릭의 데이터 플랫폼을 사용합니다. 일반 언어를 사용하여 질문하고, 문제를 해결하고, 스프레드시트 데이터를 탐색할 수 있습니다. 뉴렐릭 AI는 다음과 같은 도움을 드립니다.

* 스택을 이해하세요
* 문제 해결
* 주소 중단
* 운영 효율성 향상

채팅, NRQL 콘솔, UI 도구, GitHub Copilot, Google Gemini Code Assist와 같은 통합을 통해 상호 작용할 수 있습니다. 이 시스템은 엔지니어링 작업의 효율적이고 명확한 실행을 지원합니다.

뉴렐릭 AI는 3단계 프로세스를 통해 작업을 간소화합니다.

1. &quot;높은 오류율을 보여주세요&quot;와 같은 프롬프트를 입력하면 AI는 추론 기능을 사용하여 질문을 해석합니다.
2. 그런 다음 뉴렐릭 계정에서 관련 지표 또는 로그인을 가져와 분석합니다.
3. 마지막으로, 이해하기 쉬운 설명, 요약 또는 표를 생성하여 명확한 답변을 제공합니다.

/\* 예를 들어, &quot;어제 이상표시를 찾았나요?&quot;라고 묻는 경우, 뉴렐릭 AI는 데이터에 대한 쿼리를 작성하고 실행하고, 추론을 통해 급증하는 부분을 식별한 다음, 사람이 읽을 수 있는 요약을 생성합니다. \*/

<img title="ask ai demo" alt="A gif showing how to use New Relic AI in the UI" src="/images/ai_screenshot-full_ask-ai-demo.gif" />

/\* ## 뉴렐릭 AI 활성화 \[#enable] 뉴렐릭 AI는 아직 미리보기 단계입니다. 미리보기에서 뉴렐릭 AI를 사용하려면 모든 페이지의 오른쪽 상단에 있는 \*\*Ask AI\*\*를 클릭한 다음, \*\*Get started\*\*를 클릭하세요. 귀하는 \[생성형 AI 특정 약관]\(https\://newrelic.com/termsandconditions/service-specific)을 검토하게 될 것입니다. 귀하의 조직이 이전에 \[뉴렐릭의 사전 출시 정책]\(/docs/licenses/license-information/referenced-policies/뉴렐릭-pre-release-policy/)을 수락하지 않은 경우 여기에서 검토할 수 있습니다. 취소하거나 \*\*Accept\*\*을 클릭할 수 있습니다. 이러한 약관에 동의하면 해당 약관이 귀하의 모든 계정에 적용됩니다. \*/

## AI와 채팅 [#ask-AI]

언제든지 대화형 채팅 환경을 이용해 뉴렐릭 AI와 협력할 수 있습니다. 대부분의 뉴렐릭 플랫폼 페이지에서 오른쪽 상단에 있는 <DNT>**Ask AI**</DNT> 버튼을 클릭하기만 하면 됩니다.

우리의 AI 비서는 일반적인 질문과 자연어를 사용하여 엔지니어와 소통하도록 훈련되었습니다. 온라인 상점을 운영하고 있는데 `Checkout service` 의 응답 시간이 지연되고 있다는 알림을 받기 시작했다고 가정해 보겠습니다. 빠른 해결이 필요하신 경우, 뉴렐릭 AI에게 문의하시면 됩니다. 뉴렐릭 AI는 시스템에 무엇이 손상되었는지 알려주고 차트와 표를 통해 데이터를 탐색하는 데 도움을 줄 수 있습니다.

<img title="New Relic AI NRQL " alt="A screenshot showing how to query New Relic AI" src="/images/ai_screenshot-full_new-relic-ai-nrql-query.gif" />

## 오류 로그 분석에 대한 도움 받기 [#errors]

오류 로그는 분석하기 어려울 수 있습니다. 뉴렐릭 AI는 오류에 대해 자세히 알아보고, 관련 정보를 쿼리하고, 다음 단계에 대해 질문하는 데 도움이 됩니다.

UI 의 모든 로그인 패널에서 직접 로그인 오류에 관해 뉴렐릭 AI와 채팅할 수 있습니다. 그러나 방금 시작한 경우 다음 경로를 따를 수 있습니다.

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT> &gt; <DNT>**Logs**</DNT> 로 이동합니다.
2. <DNT>**All logs**</DNT> 을(를) 클릭합니다.
3. `error` 을(를) 검색하세요.
4. 결과 로그 줄을 클릭하고 로그 세부 정보 패널 상단에 있는 <DNT>**Explain this error**</DNT> 버튼을 찾으세요. AI 조수가 오류 요약을 제공합니다.
   <img title="new relic ai log errors" alt="a screenshot depicting log errors explained by New Relic AI" src="/images/ai_screenshot-crop_new-relic-ai-log-errors.webp" />

AI 도우미에게 스택에 대해 알리고 후속 질문을 하여 추가 조사를 진행할 수 있습니다. 예를 들어 다음과 같이 질문할 수 있습니다.

```
How often does this error occur? What's the error rate? Is it 1% or 50%?
```

```
Which hosts are throwing these errors? Is it across all of my hosts, or is it just affecting one or two of them?
```

## 오류 받은 편지함에서 스택 추적 분석 [#stack-traces]

AI 도우미를 사용하여 스택 추적을 분석하면 컨텍스트 전환이 줄어들고 분류 및 분석 속도가 빨라집니다. 뉴렐릭 AI를 사용하여 그리드 추적을 조사하려면:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT> &gt; <DNT>**Errors inbox**</DNT> 로 이동합니다.
2. 워크로드를 선택하세요.
3. **Stack trace panel** \[스택 추적 패널] 에 포함된 <DNT>**Explain this error**</DNT> 버튼을 찾으세요. 뉴렐릭 AI가 오류 요약을 제공합니다.
   <img title="explain this error" alt="A screenshot showing how to use New Relic AI to investigate stack traces. " src="/images/ai_screenshot-full_explain-this-error-stack-traces.webp" />

후속 질문을 통해 더 깊이 파고들 수 있습니다. 예를 들어 다음과 같이 말할 수 있습니다.

```
I have an Azure environment and am having issues with Redis being periodically unavailable. Can you tell me more about this error? How can I diagnose this issue? 
```

## 경고 공백 채우기 [#coverage-gaps]

팀과 시스템이 발전함에 따라 잠재적인 문제를 미리 대비하는 것이 중요합니다. 알림은 조기 경보 시스템이지만 최신 상태를 유지하는 것이 어려울 수 있습니다. 뉴렐릭 AI는 중요한 공지를 놓칠 수 있는 영역을 식별하는 데 도움이 될 수 있습니다.

뉴렐릭 AI와 채팅하고 물어보세요:

```
Show me all of my entities that are not covered by an alert. 
```

```
Show me my OpenTelemetry services that are not covered by an alert.
```

## 합성 모니터 만들기 [#synthetic]

신세틱 모니터링은 문제가 고객에게 영향을 미치기 전에 사전에 파악하고 해결하는 데 도움이 됩니다. 이제 AI 어시스턴트와 채팅을 통해 신세틱스 모니터를 생성할 수 있습니다. 뉴렐릭 AI를 사용하여 선택한 위치에서 핑 확인 모니터를 만들고 원하는 속도로 URL의 가용성을 확인하세요.

예를 들어:

```
Create a synthetic monitor to check the URL 'newrelic.com' every 10 minutes and name it: 'my awesome monitor'.
```

뉴렐릭 AI로 핑 확인 모니터를 생성하려면:

1. **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)** **All Entities** 로 이동합니다.
2. 오른쪽 상단에서 **Ask AI** \[AI에게 물어보기를] 클릭하세요.
3. AI에게 귀하의 레이더를 기반으로 핑 확인 모니터를 생성하라고 지시하세요.
4. **Create monitor** \[모니터 생성 을] 클릭합니다.
5. 설정을 확인하거나 변경하려면 **Go to monitor** \[모니터로 이동을] 선택하세요.
6. 고급 설정을 원하거나 다른 유형의 신세틱스 모니터에 액세스하려면 **Configure other monitors** \[다른 모니터 구성을] 선택하세요.

## 지원 티켓 상태 확인 [#support]

해결 시간을 단축하려면 뉴렐릭 AI에 지원 지원 요약 요약을 요청하세요. 솔루션과 과거 문제를 재검토하여 관리를 간소화하세요. 또한 지원 케이스가 열려 있는지 또는 해결된 후 닫힐 수 있는지 확인할 수 있습니다.

AI와 채팅하고 질문하세요.

```
Summarize my support ticket(s).
```

```
What is the status of my support ticket(s)?
```

## 대시보드의 이해하기 [#dashboards]

대시보드는 매우 유용한 도구입니다. 하지만 소유하지 않은 대시보드에서 스프레드시트를 조사하는 경우 모든 미묘한 차이를 이해하지 못할 수도 있습니다. 뉴렐릭 AI는 개인 데이터 분석가가 되어 핵심 정보를 자동으로 요약하고 뉴렐릭 UI 내에서 직접 숨겨진 인사이트를 찾아냅니다.

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT> &gt; <DNT>**Dashboards**</DNT> 로 이동합니다.
2. 분석할 대시보드를 선택하세요.
3. 오른쪽 상단에서 **Ask AI** \[AI에게 물어보기를] 클릭하세요.

기본적인 질문이라도 복잡한 데이터를 명확히 하는 데 도움이 되는 자세한 요약을 제공할 수 있습니다. 다음과 같이 질문해 보세요.

```
What is on this dashboard?
```

```
Summarize this dashboard for me. 
```

## 접근 및 데이터 관리

[사용자 정의 역할 및 그룹을](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) 사용하여 뉴렐릭 AI에 대한 액세스를 제한합니다. 뉴렐릭 AI는 모든 표준 뉴렐릭 RBAC 역할에 대해 기본적으로 활성화되어 있으며 해당 권한을 편집할 수 없지만, 특정 액세스 설정으로 사용자 지정 역할을 만들어 사용자 액세스를 제한할 수 있습니다.

## 정확성과 보안 [#secure-data]

당사의 AI 도우미는 다양한 입력을 사용하여 응답을 개선하고 프롬프트에 가장 적합한 제안을 제공하려고 노력합니다. 예를 들어, 제품 설명서, 쿼리 예제의 광범위한 라이브러리, 텔테이블 데이터와 같은 입력을 사용합니다.

뉴렐릭의 보안 제어에 대한 자세한 내용은 [보안 정책을](/docs/licenses/license-information/referenced-policies/security-policy/) 참조하세요.

<Callout variant="important">
  뉴렐릭 AI는 생성형 AI에 의해 구동되며 생성형 AI의 [서비스별 약관이](https://newrelic.com/termsandconditions/service-specific) 적용됩니다.
</Callout>

## 문제점 해결 [#troubleshoot]

뉴렐릭 AI를 사용하는 동안 이러한 오류 메시지 중 하나가 발생하면 다음 문제 해결 단계를 참조하세요.

**귀하의 사용자는 이 계정의 뉴렐릭 AI에 액세스할 수 없습니다. 다른 계정을 사용해 보거나 뉴렐릭 관리자에게 문의하세요.**

* 문제: 현재 사용 중인 뉴렐릭 계정에 뉴렐릭 AI가 활성화되어 있지 않거나, 계정 내의 사용자 역할에 뉴렐릭 AI에 액세스할 수 있는 권한이 없습니다.
* 해결 방법: 이 기능이 활성화된 다른 뉴렐릭 계정에서 뉴렐릭 AI에 액세스해 보세요. 액세스 권한이 필요하다고 생각되면 뉴렐릭 계정 관리자에게 연락하여 귀하의 계정에 뉴렐릭 AI를 활성화하도록 요청하거나 [사용자 역할 권한을](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) 조정하세요.

**NRAI는 현재 HIPAA 또는 FedRAMP 계정에서는 사용할 수 없습니다. 다른 계정을 시도해 보세요.**

* 문제: 규제 제한으로 인해 뉴렐릭 AI는 현재 HIPAA 또는 FedRAMP 준수 계정에서 사용할 수 없습니다.
* 해결 방법: 뉴렐릭 AI를 사용하려면 HIPAA 또는 FedRAMP 규정 준수 요구 사항이 적용되지 않는 다른 뉴렐릭 계정에서 액세스하세요.

## 피드백을 주세요 [#feedback]
