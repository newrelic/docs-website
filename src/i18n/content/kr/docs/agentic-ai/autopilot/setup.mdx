사전 요구 사항, 권한 및 공지 통합을 포함하여 조직을 위해 <DNT>Autopilot</DNT> 을(를) 설정하고 구성하는 방법을 알아보세요.

## 전제 조건 [#prerequisites]

<DNT>Autopilot</DNT>을(를) 설정하기 전에 다음 요구 사항을 충족하는지 확인합니다:

{/* ### Advanced Compute pricing

You must opt your account into Advanced Compute pricing to use <DNT>Autopilot</DNT>. If you want to add this SKU, contact your New Relic account team. */}

### 기능 제어 관리자

You can turn <DNT>Autopilot</DNT> on or off for your entire organization using [Feature control manager](/docs/accounts/accounts-billing/new-relic-one-user-management/feature-control-manager). Only an **Organization Manager** can edit these settings. After you turn on <DNT>Autopilot</DNT> in <DNT>Feature control manager</DNT>, you still need to configure it in the <DNT>AI</DNT> panel using the steps below.

### 관리자 권한(설정용)

사용자에게 <DNT>Autopilot</DNT>을(를) 사용할 권한을 부여하려면 사용자는 **Organization Manager** \[조직 관리자] 및 **Authentication Domain Manager** \[인증 도메인 관리자]여야 합니다.

<Callout variant="tip">
  If you don&apos;t have these permissions, you&apos;ll get the message: &quot;This looks like a job for an admin.&quot; Contact your organization manager to enable the agent for you.
</Callout>

### 사용자 권한(액세스용)

To use <DNT>Autopilot</DNT>, users must be [basic, full, or core users](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/).

사용자에게 <DNT>Autopilot</DNT> 사용 권한을 부여하려면 관리자 또는 인증 도메인 관리자가 사용자에게 다음 조직 수준 기능이 포함된 역할을 할당해야 합니다:

* **<DNT>Autopilot</DNT>** - 읽기, 기타(질문, 자동화에 추가)
* **뉴렐릭 MCP 서버** ― 읽기

### Required opt-ins

In addition to turning on <DNT>Autopilot</DNT> in [Feature control manager](#prerequisites), you must opt into the following:

* [MCP 서버](/docs/agentic-ai/mcp/overview/)
* [교차 계정 대상](/docs/apis/nerdgraph/examples/cross-account-destinations/overview/)

<Callout variant="tip">
  Cross account destinations is only required if you want to set up alerts with <DNT>Autopilot</DNT>. Without it, <DNT>Autopilot</DNT> will remain partially configured.
</Callout>

## Autopilot 설정 [#setup]

온보딩 흐름을 사용하여 다음을 자동으로 설정합니다:

* 에이전트를 나타내는 시스템 ID
* 기본 사용에 필요한 권한
* 알림 대상

<Callout variant="tip">
  You can configure only one <DNT>Autopilot</DNT> agent per organization.
</Callout>

<Callout variant="important">
  **시스템 아이덴티티란 무엇인가요?** 시스템 ID는 에이전트가 사용자 계정 없이 다른 서비스에 직접 인증할 수 있도록 하는 비인간 ID입니다. 시스템 ID의 범위는 단일 뉴렐릭 조직으로 지정됩니다.
</Callout>

<DNT>Autopilot</DNT>을(를) 온보딩하려면:

1. 이동 <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>

2. In the upper right, select <DNT>**Ask AI**</DNT> to open the <DNT>AI</DNT> panel. Select the **three-dot menu**, then select <DNT>**Configure**</DNT>. The same menu also has a <DNT>**What&apos;s new**</DNT> option, where you can find release notes.

3. <DNT>**Agents**</DNT> 아래에서 <DNT>**Autopilot**</DNT> 선택합니다.

4. <DNT>**Agent name**</DNT> 필드에 에이전트 이름을 입력합니다. 시스템은 자동으로 <DNT>`Autopilot -`</DNT> 접두사로 추가하고 이 이름을 시스템 ID 및 액세스 관리 구성 요소에 적용합니다.

5. 다운 드롭 메뉴에서 에이전트가 분석할 계정을 선택하세요.

   <Callout variant="important">
     You can add an unlimited number of accounts for <DNT>Autopilot</DNT> to analyze. Adding more accounts allows the agent to better identify connections and surface insights across your stack. However, the agent&apos;s analysis will be visible to any users with <DNT>Autopilot</DNT> interaction access, regardless of their direct access to those accounts.
   </Callout>

6. (Optional) [Connect Slack](/docs/agentic-ai/autopilot/connect-to-slack) to give <DNT>Autopilot</DNT> access to your team&apos;s conversations as additional context.

7. **Permissions** \[권한] 요약을 검토하십시오. 이 시스템은 사용자가 선택한 계정에 <DNT>Autopilot</DNT> 표준 읽기 전용 액세스 권한을 부여합니다.

   <Callout variant="important">
     관리 사용자는 에이전트에 제공되는 계정 및 권한을 수정할 수 있습니다. 표준 읽기 전용 역할에서 권한을 축소하면 <DNT>Autopilot</DNT> 이(가) 액세스할 수 있는 텔레메트리 데이터가 제한되어 분석이 불완전해지거나, 이상 징후를 놓치거나, 근본 원인 식별 기능이 저하될 수 있습니다.
   </Callout>

8. <DNT>**Add Agent**</DNT> 클릭하세요. 시스템 설정이 완료되는 데 몇 분 정도 소요됩니다.

<Callout variant="tip">
  Creating the alert destination does not automatically assign any alert conditions or policies to <DNT>Autopilot</DNT>. Refer to [Integrate with alerts](#alerts) to configure that.
</Callout>

<Callout variant="tip">
  If the <DNT>**Configure**</DNT> tab shows &quot;Partially configured&quot; next to <DNT>Autopilot</DNT>, open the configuration panel and select <DNT>**Update Agent**</DNT> to complete setup.
</Callout>

## 알림과 통합 [#alerts]

Connect <DNT>Autopilot</DNT> to your alert workflows so that <DNT>Autopilot</DNT> insights are automatically attached to every triggered alert or issue.

### 공지흐름 구성

<Callout variant="tip">
  **Phased approach recommendation:** Start with a small subset of critical or high-priority alert conditions or policies to validate performance and confirm the agent works well for your environment, then expand more broadly. <DNT>Autopilot</DNT> allows up to 100 requests per hour per organization, so avoid connecting it to noisy, low-priority alerts that could use up that budget.
</Callout>

알림과 함께 <DNT>Autopilot</DNT> 을(를) 설정하려면:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Issues &amp; Activity &gt; Notification Workflows &gt; Add a workflow**</DNT> 으)로 이동합니다.
2. 워크플로우 설정의 <DNT>**Notify**</DNT> 섹션에서 <DNT>**AI agent**</DNT> 선택하세요.
3. 온보딩 중에 생성된 <DNT>Autopilot</DNT> 목적지를 선택하고 <DNT>**Save message**</DNT> 클릭합니다.

<Callout variant="tip">
  **The alert prompt is editable.** The <DNT>Autopilot</DNT> destination uses a default prompt, &quot;Show me what&apos;s wrong, check for root cause, show me health charts.&quot; You can edit this prompt freely to focus the analysis on what matters for that workflow.
</Callout>

설정이 완료되면 최근에 발생한 공지 또는 이슈를 열면 해당 항목에 대한 <DNT>Autopilot</DNT> 요약이 자동으로 표시됩니다.
