## OAuth 흐름이 시작되지 않습니다

* 브라우저가 `localhost` 에 액세스할 수 있는지 확인하세요.
* 방화벽 설정을 확인하세요.
* 도구가 `OAuth` URL에 액세스할 수 있는지 확인하세요.

## API 키가 작동하지 않습니다

* 키 형식을 확인합니다( `NRAK-` 으로 시작).
* 뉴렐릭에서 주요 권한을 확인하세요.
* 키가 만료되지 않았는지 확인하세요.

## MCP 서버가 감지되지 않았습니다

* 애플리케이션을 완전히 다시 시작하세요.
* `JSON` 구문 유효성을 확인하세요.
* 파일 경로와 권한을 확인하세요.

## mcp-remote 프록시 문제

* `Node.js` 설치되어 있는지 확인하세요.
* 인터넷 연결을 확인하세요.
* npx가 `mcp-remote` 패키지에 액세스할 수 있는지 확인하세요.

## 사용자 정의 그룹에 대한 액세스가 거부되었습니다.

### 문제

뉴렐릭 AI MCP 서버에 성공적으로 연결하거나 인증할 수 없으며 다음 오류 메시지가 나타납니다. `Missing required capabilities to access MCP server. Please ensure you have the necessary permissions.` 이는 일반적으로 조직 수준 권한이 없는 사용자 지정 사용자 그룹의 구성원인 경우 발생합니다.

### 원인

Your group lacks an organization-scoped role with permission to read the MCP server. Access to this organization-wide feature must be explicitly granted to your group via an organization-scoped role.

### 해결

`Authentication Domain Manager` 역할이 있는 조직 관리자는 다음 두 가지 솔루션 중 하나를 구현하여 액세스 권한을 부여해야 합니다.

1. Grant your group a role with permission to read the MCP server:

   * 관리자는 사용자가 속한 사용자 지정 그룹에 다음 역할 중 하나를 추가해야 합니다: `Organization Read Only`, `Organization Manager` 또는 `Organization Product admin`.
   * The administrator can create a [custom organization-scoped role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) and assign it to a group you belong to via an [organization-coped grant](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#access-grants). This custom role must include the `New Relic MCP Server` read permission.

2. Add you to an authorized group:

   * The administrator must add you to an existing user group that already has an organization-scoped role with permission to read the MCP server.

사용자 관리 개념, 역할, 조직 부여에 대한 자세한 내용은 [뉴렐릭 사용자 관리 문서](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 를 참조하세요. 조직에서 사용자 관리를 위해 SCIM(System for Cross-Domain Identity Management)을 사용하는 경우 이러한 그룹 및 역할 조정은 외부 ID 공급자(IdP) 내에서 수행해야 합니다.

## 오류 메시지

* **연결이 거부되었습니다**. 도구의 URL과 네트워크 연결을 확인하세요.
* **인증 필요**: API 키가 여전히 활성 상태이고 올바르게 입력되었는지 확인하거나 `OAuth` 흐름을 완료하세요.
* **잘못된 클라이언트**: `OAuth` 설정 문제, `mcp-remote` 프록시를 시도하세요.
* **`Time.UnmarshalJSON`**: `OAuth` 에 대해 `mcp-remote` 프록시를 사용합니다.
