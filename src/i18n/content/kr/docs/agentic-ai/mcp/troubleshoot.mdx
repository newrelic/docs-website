## OAuth 흐름이 시작되지 않습니다

* 브라우저가 `localhost` 에 액세스할 수 있는지 확인하세요.
* 방화벽 설정을 확인하세요.
* 도구가 `OAuth` URL에 액세스할 수 있는지 확인하세요.

## API 키가 작동하지 않습니다

* 키 형식을 확인합니다( `NRAK-` 으로 시작).
* 뉴렐릭에서 주요 권한을 확인하세요.
* 키가 만료되지 않았는지 확인하세요.

## MCP 서버가 감지되지 않았습니다

* 애플리케이션을 완전히 다시 시작하세요.
* `JSON` 구문 유효성을 확인하세요.
* 파일 경로와 권한을 확인하세요.

## mcp-remote 프록시 문제

* `Node.js` 설치되어 있는지 확인하세요.
* 인터넷 연결을 확인하세요.
* npx가 `mcp-remote` 패키지에 액세스할 수 있는지 확인하세요.

## 사용자 정의 그룹에 대한 액세스가 거부되었습니다.

### 문제

뉴렐릭 AI MCP 서버에 성공적으로 연결하거나 인증할 수 없으며 다음 오류 메시지가 나타납니다. `Missing required capabilities to access MCP server. Please ensure you have the necessary permissions.` 이는 일반적으로 조직 수준 권한이 없는 사용자 지정 사용자 그룹의 구성원인 경우 발생합니다.

### 원인

사용자 지정 그룹에는 MCP 서버 엔드포인트를 사용하는 데 필요한 조직 수준 액세스 제어나 역할이 없습니다. 이 조직 전체 기능에 대한 액세스는 해당 그룹이나 사용자 역할에 명시적으로 부여되어야 합니다.

### 해결

The administrator responsible for your organization&apos;s user access must perform the grant. This is typically a user with the **Core** or **Full Platform** user type who has the `Authentication Domain Manager` or `Tenant Manager` role. This administrator must implement one of the following two solutions to grant you access:

1. 그룹에 조직 수준 역할 부여:
   * The administrator must add one of the following roles to the custom group you belong to: `Organization Read Only`, `Organization Manager`, or `Organization Product admin`.
2. 승인된 그룹으로 이동:
   * The administrator must add you to an existing user group that already has one of the specific required organization-level roles listed earlier.

사용자 관리 개념, 역할, 조직 부여에 대한 자세한 내용은 [뉴렐릭 사용자 관리 문서](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 를 참조하세요. 조직에서 사용자 관리를 위해 SCIM(System for Cross-Domain Identity Management)을 사용하는 경우 이러한 그룹 및 역할 조정은 외부 ID 공급자(IdP) 내에서 수행해야 합니다.

## 오류 메시지

* **연결이 거부되었습니다**. 도구의 URL과 네트워크 연결을 확인하세요.
* **인증 필요**: API 키가 여전히 활성 상태이고 올바르게 입력되었는지 확인하거나 `OAuth` 흐름을 완료하세요.
* **잘못된 클라이언트**: `OAuth` 설정 문제, `mcp-remote` 프록시를 시도하세요.
* **`Time.UnmarshalJSON`**: `OAuth` 에 대해 `mcp-remote` 프록시를 사용합니다.
