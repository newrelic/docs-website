New Relic의 REST API Explorer(v2)를 사용하면 사용 가능한 모든 API 엔드포인트에 대한 요청을 쉽게 테스트하고 보낼 수 있습니다. API 호출 유형(응용 프로그램, 브라우저 등)에 대해 선택한 기능을 선택하면 사용자 인터페이스에서 요구 사항을 확인하고 매개 변수 값을 테스트할 수 있는 대화형 양식을 제공합니다. 또한 API 값에 대한 문서의 라이브 소스를 제공합니다.

## API 키 요구사항 [#api-key]

[API Explorer를](https://rpm.newrelic.com/api/explore) 사용하려면 먼저 API 액세스를 활성화하고 계정에 대한 [사용자 API 키를](/docs/apis/rest-api-v2/requirements/rest-api-key) 생성해야 합니다.

팁:

* API 탐색기를 사용하면 UI 상단에 [사용자 API 키를](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 삽입할 수 있으며 해당 키는 탐색기의 <DNT>**Request**</DNT> 및 <DNT>**Parameters**</DNT> 섹션에 자동으로 나타납니다.

## API 탐색기 사용 [#use-api-explorer]

1. <DNT>**[API Explorer](https://api.newrelic.com/docs/)**</DNT> 로 이동합니다.
2. <DNT>**Servers**</DNT> 드롭다운 메뉴에서 US 또는 EU 기반 API URL을 선택하세요.
3. <DNT>**Authorize**</DNT> 클릭하고 사용자 API 키를 입력한 후 <DNT>**Authorize**</DNT> 다시 클릭합니다.
4. 사용 가능한 API 함수 중 하나를 확장하세요: <DNT>**GET**</DNT>, <DNT>**PUT**</DNT>, <DNT>**DELETE**</DNT>.
5. (선택 사항) API 호출에 대한 <DNT>**Parameters**</DNT> 값을 추가하여 응답을 필터링합니다(v2에 대한 설명과 요구 사항은 UI를 참조하세요).
6. <DNT>**Media type**</DNT> 드롭다운 메뉴에서 요청에 대한 형식을 JSON 또는 XML 중에서 선택하세요.
7. <DNT>**Try it out**</DNT> 클릭한 다음 <DNT>**Execute**</DNT> 클릭하세요.
