New Relic의 REST API Explorer(v2)를 사용하면 사용 가능한 모든 API 엔드포인트에 대한 요청을 쉽게 테스트하고 보낼 수 있습니다. API 호출 유형(응용 프로그램, 브라우저 등)에 대해 선택한 기능을 선택하면 사용자 인터페이스에서 요구 사항을 확인하고 매개 변수 값을 테스트할 수 있는 대화형 양식을 제공합니다. 또한 API 값에 대한 문서의 라이브 소스를 제공합니다.

## API 키 요구사항 [#api-key]

[API 탐색기](https://rpm.newrelic.com/api/explore) 를 사용하려면 API 액세스가 활성화되어야 하고 계정에 대한 [API 키](/docs/apis/rest-api-v2/requirements/rest-api-key) 가 생성되어야 합니다.

<Callout variant="important">
  다음을 사용하는 것이 좋습니다. <InlinePopover type="userKey"/>, [REST API 키](/docs/apis/intro-apis/new-relic-api-keys/#rest-api-key)가 아닌 사용자 키에 더 적은 제한이 있기 때문입니다.
</Callout>

팁:

* New Relic에 로그인한 경우 API 탐색기를 사용할 때 UI 상단에서 [API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 를 선택할 수 있으며 해당 키는 탐색기의 **요청** 및 **매개변수** 섹션에 자동으로 나타납니다.
* New Relic에 로그인하지 않은 경우 [API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 를 API Explorer의 **Parameters** 에 붙여넣을 수 있습니다.

## API 탐색기에 액세스 [#accessing]

New Relic API 탐색기를 사용하려면:

1. **[rpm.newrelic.com/api/explore](https://rpm.newrelic.com/api/explore)** 로 이동합니다.
2. API 탐색기의 메뉴 표시줄에 있는 드롭다운 목록에서 앱의 계정 이름을 선택합니다.
3. 사이드바에서 기능(애플리케이션, 브라우저 등)과 사용 가능한 API [기능](/docs/apm/apis/api-explorer-v2/parts-api-explorer#functions) ( **GET** , **PUT** , **DELETE** )을 선택합니다.
4. API 호출에 대한 다른 **매개변수** 값을 입력합니다. (v2에 대한 설명 및 요구 사항은 UI를 참조하십시오.)
5. 요청 **형식** (JSON 또는 XML)을 선택합니다.
6. **요청 보내기** 를 선택합니다.
