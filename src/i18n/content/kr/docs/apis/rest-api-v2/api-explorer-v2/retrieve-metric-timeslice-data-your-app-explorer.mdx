New Relic의 REST API Explorer(v2)를 사용하여 애플리케이션에 대한 [메트릭 타임슬라이스 데이터](/docs/telemetry-data-platform/ingest-manage-data/understand-data/new-relic-data-types#timeslice-data) 를 가져올 때 다음이 필요합니다.

* [API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)
* [앱의 ID](#app_id)
* [메트릭 이름](#view_metrics) 및 값을 포함하여 앱에 사용할 수 있는 메트릭 타임슬라이스 데이터 유형

## 앱 ID 보기 [#app_id]

[앱 ID](/docs/apm/apis/requirements/identification-code) 를 보려면 다음 단계를 따르세요.

1. New Relic REST [API Explorer](https://rpm.newrelic.com/api/explore "링크가 새 창에서 열립니다.") 에서 **Applications > GET List** 를 선택하십시오.
2. New Relic에 로그인하지 않은 경우 앱에 대한 [API 키](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing) 를 제공하십시오.
3. 선택 사항: **Applications > List** 에서 `name` , `ids` 또는 `language` 필터 값을 입력합니다.
4. **요청 보내기** 를 선택합니다.
5. **응답** 섹션에서 앱의 `id` 을 복사합니다.

[메트릭 이름](#view_metrics) 을 보려면 절차를 계속하십시오.

## 앱의 측정항목 이름 보기 [#view_metrics]

앱에 사용할 수 있는 메트릭 타임슬라이스 데이터를 보려면 다음 단계를 따르세요.

1. New Relic REST API Explorer에서 **Applications > GET Metric Names** 을 선택하십시오.
2. **Applications > Metric Names** 에서 [애플리케이션 ID](/docs/apm/apis/requirements/identification-code) 를 입력하거나 붙여넣습니다.
3. 특정 또는 부분 측정항목 `name` 을 입력하거나 사용 가능한 측정항목 이름 및 값 목록을 보려면 비워 둡니다.
4. **요청 보내기** 를 선택합니다.
5. **응답** 섹션에서 앱의 사용 가능한 `metrics` 을 검토합니다.

[메트릭](#app_metric_data) 타임슬라이스 [데이터](#app_metric_data) 를 가져오는 절차를 계속합니다.

## 앱에 대한 메트릭 타임슬라이스 데이터 가져오기 [#app_metric_data]

앱의 메트릭 값을 가져오려면 **메트릭 이름** 결과에서 값을 복사하고 **메트릭 타임슬라이스 데이터** 필드에 붙여넣습니다.

1. New Relic REST API Explorer에서 **Applications > GET Metric Data** 가져오기를 선택합니다.
2. **Applications > Metric Data** 에서 [애플리케이션 ID](/docs/apm/apis/requirements/identification-code) 를 입력하거나 붙여넣습니다.
3. 앱에 대해 하나 이상의 `names` ( **GET Metric Names** )를 입력하거나 붙여넣습니다. 새 줄에서 각 이름을 시작합니다.
4. **요청 보내기** 를 선택합니다.
5. **응답** 섹션에서 앱의 사용 가능한 `metric_data` 을 검토합니다.

**응답** 섹션을 검토한 후 조정을 계속하고 **요청 보내기** 를 다시 선택할 수 있습니다. 예를 들어:

* 측정항목 타임슬라이스 데이터의 [시간 범위](/docs/apm/apis/api-v2-examples/specifying-time-range-api-v2) 를 지정하려면 `from` 및 `to` 값을 입력합니다.
* 시간 범위에 대한 데이터의 [평균](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) 을 생성하려면 `summarize` 을 `true` 로 설정합니다. 또는 지정한 범위의 각 기간에 대한 데이터를 유지하려면 `summarize` 를 공백으로 둡니다.
