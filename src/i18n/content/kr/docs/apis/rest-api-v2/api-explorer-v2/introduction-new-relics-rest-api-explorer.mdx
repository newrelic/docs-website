New Relic은 [New Relic REST API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2) 를 비롯한 [여러 API](/docs/apis/getting-started/introduction-new-relic-apis) 를 제공합니다. 이 문서에서는 관리 사용자와 [API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 가 있는 사용자가 다음을 수행할 수 있는 [REST API 탐색기](https://api.newrelic.com) 를 소개합니다.

* 사용 가능한 REST API 끝점을 찾습니다.
* 사용자 인터페이스(API 탐색기) 내에서 REST API와 상호 작용합니다.
* 문서의 라이브 소스를 보십시오.
* API 작업에 대한 컬 명령을 가져옵니다.
* API 탐색기의 URL을 복사하여 붙여넣어 구성된 API 호출을 동료와 공유합니다.

이를 통해 솔루션을 신속하게 검색하고 API 호출을 자체 소프트웨어 구성 요소에 추가하기 전에 테스트할 수 있습니다.

인프라 <InlinePopover type="alerts"/>에 대한 REST API 호출은 API 탐색기에서 사용할 수 없습니다.

## 특징

뉴렐릭의 API 탐색기에는 선택한 계정에 대한 대화형 사용자 인터페이스가 포함되어 있습니다. API 탐색기 UI에는 API 호출 유형(<DNT>**Applications**</DNT>, <DNT>**Users**</DNT> 등)과 사용 가능한 함수(예: <DNT>**GET metric data**</DNT>, <DNT>**PUT (update) applications**</DNT>, <DNT>**DELETE applications**</DNT> 등)가 나열됩니다.

<DNT>**Parameters**</DNT> 값을 입력하면 요청을 보내기 전에 구문을 테스트하고 확인할 수 있도록 해당 값이 <DNT>**Request**</DNT> 에 자동으로 나타납니다. UI 필수 필드, 필드 설명, 해당 유형(정수, 실행, 부울 등) 및 위치(경로, 쿼리 등)를 나타냅니다.

API 키 요구사항에 대한 정보는 [REST API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 를 참조하십시오.

## API 버전 1과의 차이점 [#v1]

이 API 탐색기는 New Relic 내부 및 외부 데이터에 중점을 둔 [New Relic REST API 버전 2](/docs/apm/apis/requirements/new-relic-rest-api-v2-getting-started) 에만 적용됩니다. 버전 2는 New Relic의 더 이상 사용되지 않는 REST API 버전 1을 대체합니다.

버전 2와 버전 1 사이에는 몇 가지 차이점이 있습니다.

* 데이터의 이름은 다를 수 있습니다.
* v2의 일부 cURL 명령은 v1과 다릅니다.

<Callout variant="tip">
  New Relic 에이전트는 다른 API를 사용하며 API 탐색기를 통해 액세스할 수 없습니다.
</Callout>
