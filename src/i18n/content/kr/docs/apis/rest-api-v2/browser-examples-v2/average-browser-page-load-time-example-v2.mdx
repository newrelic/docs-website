평균 브라우저 [페이지 로드 시간](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) (또는 응답 시간)은 [<DNT>**Summary**</DNT> 페이지](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance#viewing) 의 기본 차트 오른쪽 상단에 표시됩니다. 이 값은 <DNT>**Browser**</DNT> 로 표시되며 평균 페이지 로드 시간(초)을 표시합니다.

## 평균 페이지 로드 시간 [#h2-average]

<Callout variant="important">
  예제에서는 뉴렐릭의 REST API v2를 활용하지만, 지표 타임슬라이스 쿼리 실행에는 [NRQL 함수를](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/) 사용하는 것이 좋습니다. 각 API 값은 동등한 NRQL 함수에 매핑될 수 있습니다. 이러한 API 예제를 기반으로 NRQL 쿼리를 만드는 방법을 알아보려면 [설명서를](/docs/apis/rest-api-v2/migrate-to-nrql/) 참조하세요.
</Callout>

[시간 범위](/docs/apm/apis/requirements/specifying-time-range-api-v2) (기본값은 마지막 30분)에 대한 [평균](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) 을 얻으려면 다음 명령을 사용하십시오. 이 예의 `$APP_ID` 및 `$API_KEY` 변수를 특정 [애플리케이션 ID](/docs/apm/apis/requirements/identification-code) 및 [REST API 키로](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing) 대체해야 합니다.

```sh
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json" \ 
     -H "X-Api-Key:$API_KEY" -i \ 
     -d 'names[]=EndUser&values[]=average_response_time&summarize=true'
```
