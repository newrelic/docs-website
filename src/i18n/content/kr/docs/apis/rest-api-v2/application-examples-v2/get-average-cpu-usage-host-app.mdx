New Relic REST API(v2)를 사용하여 단일 호스트에서 애플리케이션의 평균 CPU 사용량을 얻을 수 있습니다. 이 값은 **호스트** 테이블의 [APM **요약** 페이지](/docs/apm/applications-menu/monitoring/apm-overview-page) 에 있는 **CPU 사용량** 열에 나타납니다.

## 호스트의 CPU 사용량 가져오기

앱 호스트 중 하나에 대한 평균 CPU 사용량을 얻으려면 단일 명령을 사용하여 `values[]=percent` }과 함께 &#x7B; `names[]=CPU/User+Time` 메트릭을 가져옵니다. 이 예에서는 [기본 기간](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2) (지난 30분)의 시간 범위를 보여줍니다.

```bash
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/hosts/$HOST_ID/metrics/data.json" \
     -H "Api-Key:$API_KEY" -i \
     -d 'names[]=CPU/User+Time&values[]=percent&summarize=true'
```

* `$APP_ID` 및 `$API_KEY` 을 [앱 ID](/docs/apm/apis/requirements/identification-code) 및 [API 키로](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 바꿉니다.
* `$HOST_ID` 을 대상 호스트의 [호스트 ID](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_host_id) 로 바꿉니다.

추가 정보:

* 자세한 [시계열 데이터](/docs/apis/rest-api-v2/requirements/calculating-average-metric-values-summarize) 를 얻으려면 `summarize=true` 을(를) 제거하세요.
* 다른 [시간 범위](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2) 를 지정하십시오.

<Callout variant="tip">
  [New Relic API Explorer(v2)](https://rpm.newrelic.com/api/explore) 에서 동일한 정보를 얻으려면 [**Application Hosts > GET Metric Data**](https://rpm.newrelic.com/api/explore/application_hosts/data) 를 선택하고 [API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 를 포함하십시오. 해당 필드에 [애플리케이션 ID](/docs/apm/apis/requirements/identification-code) , [호스트 ID](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_host_id) , `names[]=CPU/User Time` 및 `values[]=percent` 측정항목을 추가합니다.
</Callout>

## 전체 앱의 CPU 사용량 가져오기 [#api-call]

애플리케이션의 평균 CPU 사용량을 얻으려면 이전 예와 유사한 명령을 사용하지만 `host_id` 에 대한 참조는 사용하지 않습니다.

```bash
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json" \
     -H "Api-Key:$API_KEY" -i \
     -d 'names[]=CPU/User+Time&values[]=percent&summarize=true'
```

* `$APP_ID` 및 `$API_KEY` 을 [앱 ID](/docs/apm/apis/requirements/identification-code) 및 [API 키로](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 바꿉니다.

추가 정보:

* 자세한 [시계열 데이터](/docs/apis/rest-api-v2/requirements/calculating-average-metric-values-summarize) 를 얻으려면 `summarize=true` 을(를) 제거하세요.
* 다른 [시간 범위](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2) 를 지정하십시오.

<Callout variant="tip">
  [New Relic API Explorer(v2)](https://rpm.newrelic.com/api/explore) 에서 동일한 정보를 얻으려면 [**Applications > GET Metric Data**](https://rpm.newrelic.com/api/explore/applications/data) 를 선택하고 [API 키](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) 를 포함하십시오. 적절한 필드에 [애플리케이션 ID](/docs/apm/apis/requirements/identification-code) 와 `names[]=CPU/User Time` 및 `values[]=percent` 측정항목을 추가합니다.
</Callout>
