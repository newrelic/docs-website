이 접근 방식은 파트너(및 파트너 관계 구조를 사용하여 여러 계정을 관리하는 엔터프라이즈 규모 고객)가 파트너 자격 증명을 사용하여 모든 [REST](https://api.newrelic.com/) API( v2) 통화. 파트너가 아닌 API 사용자에게는 새로운 기능을 제공하지 않습니다.

New Relic REST API(v2)에서 엔드포인트를 호출할 때 <InlinePopover type="userKey"/>, New Relic 은 설명된 [인증 방법 을](/docs/apis/rest-api-v2/requirements/admins-api-key-partnerships#Partners)사용할 것을 제안 합니다 .

## 사용자 정의 헤더 및 API 호출 [#Partners]

API를 호출할 때 [파트너 API 키](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference#the-api-key) , [파트너 ID](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference#partner-id) 및 [계정 ID](/docs/accounts-partnerships/accounts/account-setup/account-id) 와 함께 다음 사용자 정의 헤더를 사용하십시오.

* `X-Api-Key:YOUR_PARTNER_ACCOUNT_API_KEY`
* `NewRelic-Partner-Id:YOUR_PARTNER_ID`
* `NewRelic-Account-Id:YOUR_CUSTOMERS_ACCOUNT_ID`

이 양식에서 REST API(v2)에 대한 인증을 제공하면 고객 계정에서 모든 API 작업을 완료할 수 있습니다. 이는 일부 API 호출에 관리자의 API 키가 필요하다는 고객 요구 사항을 우회합니다.

이 인증 방법은 REST API 키만 필요한 API 끝점에서도 작동합니다. 이 새로운 인증 방법은 관리 사용자의 API 키가 필요한 끝점뿐만 아니라 모든 끝점에서 작동합니다.

## 알려진 제한 사항

이 파트너 전용 인증 방법은 [New Relic REST API(v2)](https://api.newrelic.com/) 에서만 작동합니다. 다음에서는 작동 **하지 않습니다** .
