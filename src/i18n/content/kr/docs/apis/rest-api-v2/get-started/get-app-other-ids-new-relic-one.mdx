[New Relic REST API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2) 를 사용할 때 일부 ID 자리 표시자(예: `$APP_ID` , `$KEYTX_ID` , `$COMPONENT_ID` 등)를 앱, 트랜잭션 또는 기타 구성요소의 ID로 바꿔야 합니다.

New Relic UI를 사용하여 이러한 ID 중 일부를 얻을 수 있습니다. [REST API Explorer](/docs/apis/rest-api-v2/get-started/list-application-id-host-id-instance-id) 를 사용하여 ID 목록을 얻을 수도 있습니다.

## UI를 사용하여 ID 찾기 [#ui]

UI에는 메타데이터 패널의 애플리케이션 및 기타 구성 요소의 ID가 포함됩니다.

<img
  title="Metadata panel in NR1"
  alt="Metadata panel in NR1"
  src={img0MetadataPanelInNr1}
/>

<figcaption>
  클릭 <Icon name="fe-info"/>애플리케이션 이름 옆에 있는 아이콘을 클릭하여 메타데이터 패널을 엽니다.
</figcaption>

여기 몇 가지 예가 있어요.

<CollapserGroup>
  <Collapser
    id="apm"
    title="APM: 앱 ID"
  >
    New Relic UI에서 `$APP_ID` 을 찾으려면:

    1. [one.newrelic.com](https://one.newrelic.com) 에서 **APM** 을 클릭하고 애플리케이션을 선택합니다.

    2. 클릭 <Icon name="fe-tag"/>애플리케이션 이름 옆에 있는 아이콘을 클릭하여 메타데이터 패널을 엽니다.

    3. **앱 ID** 를 찾습니다.

       REST API 호출에서 `$APP_ID` 을 이 번호로 대체하십시오.
  </Collapser>

  <Collapser
    id="browser"
    title="브라우저 앱 ID"
  >
    New Relic UI에서 `$APP_ID` 을 찾으려면:

    1. [one.newrelic.com](https://one.newrelic.com) 에서 **브라우저** 를 클릭하고 모니터링되는 앱을 선택합니다.

    2. 클릭 <Icon name="fe-tag"/>애플리케이션 이름 옆에 있는 아이콘을 클릭하여 메타데이터 패널을 엽니다.

    3. **앱 ID** 를 찾습니다.

       REST API 호출에서 `$APP_ID` 을 이 번호로 대체하십시오.
  </Collapser>

  <Collapser
    id="mobile"
    title="모바일 앱 ID"
  >
    New Relic UI에서 `$MOBILE_ID` 을 찾으려면:

    1. [one.newrelic.com](https://one.newrelic.com) 에서 **모바일** 을 클릭하고 모니터링되는 앱을 선택합니다.

    2. 클릭 <Icon name="fe-tag"/>애플리케이션 이름 옆에 있는 아이콘을 클릭하여 메타데이터 패널을 엽니다.

    3. **앱 ID** 를 찾습니다.

       REST API 호출에서 `$MOBILE_ID` 을 이 번호로 대체하십시오.
  </Collapser>
</CollapserGroup>
