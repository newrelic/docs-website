New Relic은 다음과 같은 용도로 사용할 수 있는 다양한 API 및 SDK를 제공합니다.

* New Relic에 데이터를 보냅니다.
* New Relic에서 데이터를 검색합니다.
* 설정을 보고 구성합니다.

API 키를 찾고 계십니까? [API 키를](/docs/apis/intro-apis/new-relic-api-keys) 참조하십시오.

## 데이터 수집용 API [#data-type-apis]

데이터 수집 API는 [데이터 보고를 위한 많은 솔루션](https://newrelic.com/instant-observability) 중 일부입니다. API는 직접 사용할 수 있지만 많은 데이터 보고 도구의 기본 수집 경로이기도 합니다. New Relic에 데이터 보고를 막 시작했다면 [Install New Relic](/docs/new-relic-solutions/new-relic-one/install-configure/install-new-relic) 에서 시작하는 것이 좋습니다.

## NerdGraph(GraphQL) [#graphql]

[NerdGraph](/docs/apis/graphql-api/getting-started/introduction-new-relic-graphql-api) 는 New Relic 데이터를 쿼리하고, 계정 정보를 쿼리하고, 다양한 기능 구성을 만드는 데 권장하는 API입니다. 무엇을 할 수 있는지 알아 [보려면 NerdGraph 자습서](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#tutorials) 를 확인하십시오.

NerdGraph는 최신 API이며 이전 API 중 일부를 한 곳에 모으려는 시도입니다. 아직 NerdGraph로 수행할 수 없는 [REST API](#rest-api) 로 수행할 수 있는 몇 가지 기능이 있으며 이것이 일부 New Relic 조직이 여전히 REST API를 사용하는 이유입니다.

## REST API [#rest-api]

우리의 [REST API](/docs/apis/rest-api-v2) 는 쿼리 및 구성을 위한 이전 API이며 [NerdGraph](#graphql) 가 교체하는 중입니다. 우리는 REST API에 대해 최소한의 지속적인 작업을 하고 있습니다. REST API에는 아직 NerdGraph에 없는 몇 가지 구성 기능이 있지만 우선 NerdGraph를 사용하는 것이 좋습니다. NerdGraph에 주의를 기울이고 기능을 추가할 것이기 때문입니다.
