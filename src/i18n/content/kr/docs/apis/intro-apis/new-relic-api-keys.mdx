뉴렐릭의 모니터링 솔루션과 API는 API 키를 사용해 아이덴티티를 인증하고 확인합니다. 이 키를 사용하면 조직에서 승인된 사람만 뉴렐릭에 데이터를 보고하고, 해당 데이터에 액세스하며, 기능을 구성할 수 있습니다. 기본 키는 라이선스 키(데이터 보고용)와 <InlinePopover type="userKey"/>(GraphQL API인 NerdGraph 작업용)입니다.

API 키로 시작하려면:

1. 아직 [계정이 없으시다면, 먼저 무료 계정을 신청](https://newrelic.com/signup)합니다.
2. [에이전트 및 통합](https://newrelic.com/instant-observability/?category=infrastructure-and-os)을 사용하여 뉴렐릭으로 데이터를 수집하기 시작합니다.
3. 데이터를 바로 관찰할 수 있도록 [API keys UI 페이지](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher)에서 API 키를 생성하고 관리합니다.

## API 키 보기 및 관리 [#keys-ui]

[API keys UI 페이지](https://one.newrelic.com/api-keys)에서 대부분의 API 키를 확인하고 관리할 수 있습니다. 키를 찾으려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings/#user-menu)에서 **API 키**를 클릭합니다. 또한 다음이 가능합니다.

* [NerdGraph 탐색기를 사용](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph#explorer)하여 사용자 키 확인 및 생성
* [NerdGraph API](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys)를 사용해 라이선스 키, 브라우저 키 및 사용자 키를 프로그래밍 가능한 방식으로 관리

[계정 ID](/docs/accounts/accounts-billing/account-setup/account-id)는 뉴렐릭에 데이터를 보고할 때 종종 필요한 또 다른 식별 번호입니다.

## UI에서의 키 복사 방법 [#copy-keys]

[API keys UI 페이지](https://one.newrelic.com/api-keys)에서 기존 API 키를 복사할 수도 있습니다. UI에서 사용된 언어에 대한 세부 정보는 다음과 같습니다.

* **Copy key(키** **복사) vs. Copy key ID(키 ID 복사)** : 키 복사 옵션은 키의 값을 복사합니다. 거의 모든 경우에서 키 복사 옵션을 사용합니다. 키 ID 옵션은 API를 통해 키 객체를 참조하는 데 필요한 키의 ID를 복사합니다.
* **Copy truncated key(잘린 키 복사)** : 이 옵션은 본인 소유가 아닌 키에 사용할 수 있습니다. 키에서 몇 자릿수만 복사하므로 키의 내부 추적이나 지원팀과의 대화에서 유용할 수 있습니다.

## API 키의 보안 유지 [#security-practices]

API 키가 잘못된 사람의 손에 들어가면 보안 위험이 발생할 수 있습니다. 예를 들어:

* 귀하의

  <InlinePopover type="licenseKey"/>

  를 가진 누군가가 귀하의 계정에 임의로 데이터를 보낼 수 있습니다.

* 팀원의 사용자 키 중 하나를 가진 사람이 뉴렐릭 데이터를 보고 뉴렐릭 계정을 변경할 수 있습니다.

비밀번호 및 기타 민감한 정보와 마찬가지로 API 키를 안전하게 취급해야 합니다. 몇 가지 권장 사항:

* 라이선스 키와 브라우저 키의 경우 키 순환 전략을 수립하는 것이 좋습니다. 새 키를 만들고 정해진 일정에 따라 이전 키를 삭제하십시오. 고려 사항:

  * 계정과 연결된 원래 키는 삭제할 수 없으므로 강력한 보안 전략을 구현하려면 나중에 삭제할 수 있는 추가 키를 생성해야 합니다.
  * 모바일 앱 토큰에는 적용되지 않습니다. 토큰을 삭제하거나 추가 토큰을 만들 수 없습니다.

* <InlinePopover type="userKey"/>

  의 경우:

  * 팀원들에게 사용자 키를 안전하게 유지하도록 지시합니다.
  * 구성원이 조직을 떠날 때 [기본 사용자](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)이더라도 뉴렐릭에서 삭제합니다.

## API 키에 대해 자세히 알아보기 [#key-details]

API 키를 생성하거나 관리하려면 [one.newrelic.com/launcher/api-keys-ui.api-keys-launcher](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) 또는 [NerdGraph API](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys)의 UI를 사용합니다. 자세한 내용을 보려면 키를 선택하십시오.

<CollapserGroup>
  <Collapser
    id="license-key"
    title="라이선스 키"
  >
    데이터 수집에 사용되는 주요 키를 라이선스 키라고 합니다. API 키 UI 및 [NerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys)에서 이 키는 때때로 `ingest - license`로 참조됩니다.

    거의 모든 뉴렐릭 데이터 수집에는 라이선스 키가 필요합니다. 브라우저 모니터링 데이터(브라우저 키 사용) 및 모바일 모니터링 데이터(모바일 앱 토큰 사용)는 예외입니다.

    라이선스 키는 뉴렐릭 계정과 연결된 40자의 16진수 문자열입니다. 뉴렐릭에 처음 [가입](/docs/subscriptions/creating-your-new-relic-account)하면 단일 계정과 자체 라이선스 키가 있는 조직이 생성됩니다. 더 많은 계정이 추가되면 각 계정은 자체 라이선스 키로 시작합니다. 계정에 대해 원래 생성된 라이선스 키는 삭제할 수 없지만 관리 및 삭제할 수 있는 추가 라이선스 키를 생성할 수 있으며, 이는 키 순환과 같은 보안 관행을 구현하는 데 유용합니다. 계정의 기존 라이선스 키를 삭제해야 하는 경우, [지원팀에 문의](https://support.newrelic.com)하시기 바랍니다.

    사용자가 라이선스 키를 보거나 관리할 수 없도록 제한하려면 해당 기능이 없는 역할을 할당합니다: [기존 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#add-on) \| [새로운 사용자 모델](/docs/accounts/accounts-billing/new-relic-one-user-management/user-capabilities)
  </Collapser>

  <Collapser
    id="browser-key"
    title="브라우저 키"
  >
    브라우저 모니터링은 라이선스 키가 아닌 브라우저 키를 사용하여 데이터를 보고합니다. 브라우저 키는 [브라우저 모니터링 에이전트](/docs/browser/new-relic-browser/getting-started/introduction-browser-monitoring)의 데이터를 계정에 연결하는 데 사용됩니다.

    계정 생성 시 생성된 원래 브라우저 키는 관리하거나 삭제할 수 없지만, 새 브라우저 키를 생성하고 해당 키를 삭제할 수 있습니다. 계정의 첫 번째 브라우저 키를 삭제하는 데 도움이 필요할 경우, [지원팀에 문의](https://support.newrelic.com)하시기 바랍니다.
  </Collapser>

  <Collapser
    id="mobile-app-token"
    title="모바일 앱 토큰"
  >
    모바일 모니터링은 라이선스 키가 아닌 모바일 앱 토큰을 사용하여 데이터를 보고합니다. 자세한 내용은 [모바일 앱 토큰](/docs/mobile-monitoring/new-relic-mobile/maintenance/viewing-your-application-token)을 참조하세요.
  </Collapser>

  <Collapser
    id="user-key"
    title="사용자 키"
  >
    [NerdGraph](/docs/apis/intro-apis/introduction-new-relic-apis/#graphql) 및 [REST API](/docs/apis/intro-apis/introduction-new-relic-apis/#rest-api)를 사용하려면 "개인 API 키"라고도 하는 뉴렐릭 사용자 키가 필요합니다.

    <InlinePopover type="userKey"/>는 특정 뉴렐릭사용자에게 연결되어 있으며 양도할 수 없습니다. 사용자 키를 사용하면 키와 연결된 특정 계정뿐만 아니라 액세스 권한이 부여된 모든 계정에 대해 쿼리를 수행할 수 있습니다. 뉴렐릭 사용자가 뉴렐릭에서 삭제되면 사용자 키도 비활성화되어 작동하지 않습니다.

    여러 계정에 대한 사용자 액세스를 제공하더라도 사용자 키는 단일 특정 계정에 연결됩니다. 계정이 삭제되면 해당 계정과 연결된 모든 사용자 키가 더 이상 작동하지 않는다는 것을 의미합니다. 또한 REST API 호출은 사용자 키에 연결된 계정의 컨텍스트로 제한됩니다.

    사용자가 사용자 키를 확인하거나 관리하지 못하도록 하려면 해당 기능이 없는 역할을 할당해야 합니다. [기존 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#add-on) \| [새로운 사용자 모델](/docs/accounts/accounts-billing/new-relic-one-user-management/user-capabilities)
  </Collapser>
</CollapserGroup>

          이 키는 사용 중인 상태이지만, 동일한 작업 등에 사용할 수 있는 <InlinePopover type="licenseKey"/>를 사용할 것을 권합니다.

        이 키는 [Event API](/docs/insights/insights-data-sources/custom-data/insert-custom-events-insights-api), [Log API](/docs/enable-new-relic-logs-http-input), [Metric API](/docs/introduction-new-relic-metric-api) 및 [Trace API](/docs/apm/distributed-tracing/trace-api/introduction-new-relic-trace-api) 또는 이러한 API를 사용하는 통합을 통해 데이터를 수집하는 데 사용됩니다.

        가용성 및 액세스에 대한 팁:

        * 이러한 키는 특정 사용자가 아닌 계정과 연결되어 있으므로 키에 대한 액세스 권한이 있는 계정의 모든 사용자가 이를 사용할 수 있습니다.

        * 보안을 위한 모범 사례로 다른 애플리케이션 또는 다른 데이터 원본에 대해 다른 Insights 삽입 키를 사용하는 것이 좋습니다.

          Insights 삽입 키를 찾고 관리하려면, [user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)에서 **API keys**를 클릭합니다. ([API 키 페이지에 대한 직접 링크](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) 가져오기) 그런 다음 **Insights insert keys**를 클릭합니다.
      </Collapser>

      >
        관리자 키는 더 이상 사용되지 않는 이전의 API 키 중 하나입니다. 2020년 12월 4일부로 기존의 모든 관리자 키는 사용자 키로 마이그레이션되었습니다.

        관리자 키를 사용하고 있었다면, 해당 키를 활성 상태로 유지하기 위해 아무 작업도 수행할 필요가 없습니다. API 키 UI를 통해 자동으로 액세스할 수 있고 사용자 키로 레이블이 지정되며 동일한 권한이 부여됩니다. 동일한 워크플로우를 통해 사용자 키처럼 관리할 수 있습니다.

        마이그레이션된 모든 관리자 키에는 키 테이블의 **Migrated from an admin user key(관리자 키에서 마이그레이션됨)** 이라는 메모가 있습니다.
      </Collapser>

      <Collapser
        id="rest-api-key"
        title="REST API 키(권장하지 않음)"
      >
        REST API 키는 REST [API](/docs/apis/rest-api-v2/get-started/introduction-new-relic-rest-api-v2)를 사용하기 위한 이전 키입니다. **이제 REST API 키 대신 사용자 키를 사용하는 것이 좋습니다.** 사용자는 계정별이 아니라 사용자별로 다르므로 조직에서 팀 구성원의 액세스를 더 잘 제어할 수 있습니다. 또한 REST API 대신 최신 API인 [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph)를 사용하는 것이 좋습니다.

        고려해야 할 사항:

        * 각 뉴렐릭 계정에는 REST API 키가 **하나만** 있을 수 있습니다.

        * REST API 키 대신 사용자 키를 사용하는 것이 좋습니다.

        * 가능하면 REST API보다 [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph)를 사용하는 것이 좋습니다.

        * 관리자 수준의 사용자 권한이 필요합니다. REST API 키 또는 REST API 탐색기에 대한 액세스 권한이 없는 경우 권한이 없기 때문일 수 있습니다. 뉴렐릭 계정 관리자에게 문의하거나, 대신 사용자 키를 사용하세요.

          REST API 키를 찾고 관리하려면 [user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)에서 **API keys**를 클릭합니다. ([API 키 페이지에 대한 직접 링크](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) 가져오기) 그런 다음 **REST API key**를 클릭합니다. API 키를 구성하거나 삭제하기 전에 올바른 계정에 대해 수행하고 있는지 확인하시기 바랍니다.
      </Collapser>
