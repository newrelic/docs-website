NerdGraph는 뉴렐릭 데이터를 쿼리하고 일부 뉴렐릭 기능을 구성할 수 있는 GraphQL 포맷 API입니다. [무료 뉴렐릭 계정을 신청](https://newrelic.com/signup)하고 모니터링 서비스를 [설치한](/docs/using-new-relic/cross-product-functions/install-configure/install-new-relic/) 후 NerdGraph를 시작할 수 있습니다.

## 너드그래프란? [#overview]

NerdGraph는 여러 뉴렐릭 [API](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/) 중 하나입니다. NerdGraph는 뉴렐릭 데이터를 쿼리하고 일부 특정 구성을 수행하는 데 권장되는 API입니다. ([기능에 대해 자세히 알아보십시오.](#tutorials)) NerdGraph는 뉴렐릭의 다양한 API 및 마이크로서비스에서 데이터를 반환하기 위한 단일 API 인터페이스를 제공합니다. 시간이 지남에 따라 다른 구성 기능들이 NerdGraph에 추가될 예정입니다.

<Callout variant="important">
  NerdGraph는 데이터 수집에 사용되지 않습니다. 이를 위해 [데이터 수집 API](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/#data-type-apis) 를 사용합니다.
</Callout>

NerdGraph는 오버페칭이나 언더페칭 없이 필요한 데이터를 정확히 요청할 수 있는 오픈 소스 API 형식인 [GraphQL](https://graphql.org) 을 사용하여 구축되었습니다.

## 요구 사항 [#requirements]

시작하기 전에:

* 뉴렐릭 계정이 필요하며 해당 계정을 사용하면 쿼리 및 변형에 포함해야 하는 API <InlinePopover type="userKey"/>에 액세스할 수 있습니다.
* 사용자 유형 및 할당된 역할은 NerdGraph 권한에 영향을 줄 수 있습니다. 자세한 내용은 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data#user-permissions) 를 참조하십시오.

## 끝점 [#endpoints]

사용 가능한 API 엔드포인트는 [데이터센터가 위치한 지역](/docs/accounts/accounts-billing/account-setup/choose-your-data-center)에 따라 다릅니다.

* 메인 엔드포인트: `https://api.newrelic.com/graphql`
* EU 데이터 센터를 사용하는 계정의 엔드포인트: `https://api.eu.newrelic.com/graphql`

엔드포인트에 액세스하려면 코드에서 호출하거나, [NerdGraph API 탐색기를](/docs/apis/nerdgraph/get-started/nerdgraph-explorer) 사용하거나, 다음과 유사한 `curl` 명령을 만들 수 있습니다.

```bash
curl -X POST https://api.newrelic.com/graphql \
     -H 'Content-Type: application/json' \
     -H 'API-Key: YOUR_NEW_RELIC_USER_KEY' \
     -d '{ "query":  "{ requestContext { userId apiKey } }" }'
```

## NerdGraph API 탐색기 사용 [#explorer]

NerdGraph API를 시험 사용하는 손쉬운 방법은 NerdGraph API 탐색기를 사용하는 것입니다. 이 도구를 시작하는 데 도움이 필요하면 [NerdGraph API 탐색기 튜토리얼](/docs/apis/nerdgraph/get-started/nerdgraph-explorer)을 참조하십시오.

## NerdGraph 기능 [#tutorials]

NerdGraph 기능은 두 가지 주요 범주로 나눌 수 있습니다.

* <DNT>**Querying data.**</DNT> 프로그래밍 방식의 워크플로우에서 사용하거나 사용자 지정 데이터 시각화를 위한 [뉴렐릭 앱을](https://developer.newrelic.com/build-apps) 구축하는 등 다양한 목적으로 데이터를 가져올 수 있습니다.
* <DNT>**Configuring features.**</DNT> 다양한 설정을 사용할 수 있으며 시간이 지남에 따라 더 많은 설정이 추가될 예정입니다. 태그 추가, 워크로드 구성, &amp;quot;골든 메트릭&amp;quot; 사용자 지정 등의 작업을 수행할 수 있습니다.

NerdGraph를 사용하여 광범위한 데이터를 반환하고 구성할 수 있습니다. NerdGraph가 무엇을 할 수 있는지 확인하는 한 가지 방법은 쿼리 작성을 시작하고 [NerdGraph 탐색기](/docs/apis/nerdgraph/get-started/nerdgraph-explorer)를 사용하여 스키마를 탐색하는 것입니다.

* [모니터링 대상에 대한 데이터 가져오기](/docs/apis/nerdgraph/examples/nerdgraph-entities-api-tutorial)
* [NRQL(저희 쿼리 언어)을 사용하여 데이터](/docs/apis/nerdgraph/examples/nerdgraph-nrql-tutorial) 쿼리

## NerdGraph 용어 [#terminology]

다음은 NerdGraph가 사용하는 API 형식인 GraphQL에서 유래한 용어입니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        용어
      </th>

      <th>
        정의
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        쿼리 및 돌연변이
      </td>

      <td>
        GraphQL 작업에는 두 가지 클래스가 있습니다.

        * [쿼리](https://graphql.org/learn/queries) 는 데이터를 가져오는 데만 사용되는 기본 요청입니다. 이러한 쿼리는 정적이 아니므로 필요에 따라 더 많은 데이터 또는 더 적은 데이터를 요청할 수 있습니다. 각 쿼리에 대해 [스키마](https://graphql.org/learn/schema) 에서 지원하는 한 검색하려는 데이터를 정확히 지정할 수 있습니다.
        * [변형](https://graphql.org/learn/queries/#mutations) 은 리소스 생성 또는 구성 변경과 같은 작업을 수행하는 요청입니다. 돌연변이에는 `mutation` 키워드와 돌연변이의 `name` 가 필요합니다.
      </td>
    </tr>

    <tr>
      <td>
        유형
      </td>

      <td>
        GraphQL의 데이터는 유형으로 구성됩니다. 유형은 [스칼라](https://graphql.org/learn/schema/#scalar-types) (문자열, 숫자 또는 부울과 같은) 또는 객체 유형일 수 있습니다.

        [개체 유형](https://graphql.org/learn/schema/#object-types-and-fields) 은 필드 모음으로 구성된 사용자 정의 유형입니다. 예를 들어, `User` 이라는 객체 유형은 시스템의 사용자를 나타낼 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        들
      </td>

      <td>
        [필드](https://graphql.org/learn/schema/#object-types-and-fields) 는 쿼리할 수 있는 개체 유형에 대한 정보를 나타냅니다. 필드는 스칼라, 목록 또는 개체일 수 있습니다. 예를 들어, `User` 객체 유형에는 `name` 이라는 문자열 필드가 있을 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        상호 작용
      </td>

      <td>
        [인터페이스](https://graphql.org/learn/schema/#interfaces) 는 다른 객체 유형이 `implement` 할 수 있는 공통 필드 모음을 나타내는 추상 유형입니다.
      </td>
    </tr>
  </tbody>
</table>
