[NerdGraph](https://api.newrelic.com/graphiql)는 뉴렐릭 데이터를 쿼리하고 일부 뉴렐릭 기능을 구성할 수 있도록 해주는 GraphQL 포맷 API입니다. [무료 뉴렐릭 계정을 신청](https://newrelic.com/signup)하고 모니터링 서비스를 [설치](/docs/using-new-relic/cross-product-functions/install-configure/install-new-relic/)한 후 NerdGraph를 시작할 수 있습니다.

## NerdGraph란? [#overview]

뉴렐릭에는 여러 [API](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/)가 있습니다. NerdGraph는 뉴렐릭 데이터를 쿼리하고 일부 특정 구성을 수행하는 데 권장되는 API입니다. ([기능에 대해 자세히 알아보십시오.](#tutorials)) NerdGraph는 뉴렐릭의 다양한 API와 마이크로서비스에서 데이터를 반환하기 위한 단일 API 인터페이스를 제공합니다. 향후 여러 다른 구성 기능이 NerdGraph에 추가될 예정입니다.

<Callout variant="important">
  NerdGraph는 데이터 인제스트에 사용되지 않습니다. 이를 위해서는 [데이터 인제스트 API](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/#data-type-apis)를 사용해야 합니다.
</Callout>

NerdGraph는 오버페칭이나 언더페칭 없이 필요한 데이터를 정확히 요청할 수 있는 오픈소스 API 포맷인 [GraphQL](https://graphql.org)을 사용하여 구축되었습니다.

## 요구 사항 및 엔드포인트 [#authentication]

NerdGraph 요구 사항에 대한 참고 사항:

* NerdGraph에는 뉴렐릭 [사용자 키](/docs/apis/intro-apis/new-relic-api-keys/#user-api-key)가 필요합니다. 키는 [GraphiQL 탐색기](#explorer)에서 생성하고 액세스할 수 있습니다.
* 사용자 유형 및 할당된 역할은 NerdGraph 권한에 영향을 줄 수 있습니다. 보다 자세한 내용은 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data#user-permissions)를 참조하십시오.

엔드포인트는 [데이터센터가 위치한 지역](/docs/accounts/accounts-billing/account-setup/choose-your-data-center)에 따라 다릅니다.

* 메인 엔드포인트: [https://api.newrelic.com/graphiql](https://api.newrelic.com/graphiql)
* EU 데이터센터를 사용하는 계정의 엔드포인트: [https://api.eu.newrelic.com/graphiql](https://api.eu.newrelic.com/graphiql)

엔드포인트에 액세스하려면 [NerdGraph 탐색기](#explorer)를 사용하거나 다음과 같은 curl 명령으로 직접 액세스할 수 있습니다.

```
curl -X POST https://api.newrelic.com/graphql \
-H 'Content-Type: application/json' \
-H 'API-Key: YOUR_NEW_RELIC_USER_KEY' \
-d '{ "query":  "{ requestContext { userId apiKey } }" } '
```

## GraphiQL 탐색기 사용 [#explorer]

NerdGraph 탐색기를 사용해 NerdGraph 스키마를 탐색하고 API를 호출할 수 있습니다.

* 미국 데이터센터: [https://api.newrelic.com/graphiql](https://api.newrelic.com/graphiql)
* EU 데이터센터: [https://api.eu.newrelic.com/graphiql](https://api.eu.newrelic.com/graphiql)

탐색기 사용에 대한 팁은 [NerdGraph 탐색기 튜토리얼](/docs/apis/nerdgraph/get-started/nerdgraph-explorer)을 참조하십시오.

## NerdGraph 기능 [#tutorials]

NerdGraph 기능은 두 가지 주요 카테고리로 나눌 수 있습니다.

* **쿼리 데이터.** 프로그래매틱 방식의 워크플로우에서 사용하거나 커스텀 데이터 시각화를 위해 [뉴렐릭 앱](https://developer.newrelic.com/build-apps)을 구축하는 등 다양한 목적으로 데이터를 가져올 수 있습니다.
* **기능 구성** 다양한 구성을 사용할 수 있으며, 앞으로 더 많은 구성이 추가될 예정입니다. 태그 추가, 워크로드 구성 또는 ‘골든 메트릭’ 맞춤화 등의 작업을 수행할 수 있습니다.

NerdGraph를 사용하여 광범위한 데이터를 반환하고 구성할 수 있습니다. NerdGraph가 무엇을 할 수 있는지 확인하는 한 가지 방법은 쿼리 작성을 시작하고 [NerdGraph 탐색기](/docs/apis/nerdgraph/get-started/nerdgraph-explorer)를 사용하여 스키마를 탐색하는 것입니다.

* [모니터링된 엔터티에 대한 데이터 가져오기](/docs/apis/nerdgraph/examples/nerdgraph-entities-api-tutorial)
* [NRQL(뉴렐릭의 쿼리 언어)을 사용하여 데이터 쿼리](/docs/apis/nerdgraph/examples/nerdgraph-nrql-tutorial)
* [대시보드 생성](/docs/apis/nerdgraph/examples/create-widgets-dashboards-api)

## NerdGraph 용어 [#terminology]

다음은 NerdGraph가 사용하는 API 포맷인 GraphQL에서 사용되는 용어입니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        용어
      </th>

      <th>
        정의
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        쿼리 및 뮤테이션
      </td>

      <td>
        GraphQL 연산에는 두 가지 클래스가 있습니다.

        * [쿼리](https://graphql.org/learn/queries)는 데이터를 가져오는 데만 사용되는 기본 요청입니다. 이러한 쿼리는 고정된 것이 아닙니다. 즉, 필요에 따라 더 많은 데이터 또는 더 적은 데이터를 요청할 수 있습니다. [스키마](https://graphql.org/learn/schema)에서 지원만 된다면, 각 쿼리에 대해 검색하려는 데이터를 정확히 지정할 수 있습니다.
        * [뮤테이션(Mutations)](https://graphql.org/learn/queries/#mutations)은 리소스 생성 또는 구성 변경 같은 작업을 수행하는 요청입니다. 뮤테이션은 `mutation` 키워드와 뮤테이션의 `name`이 필요합니다.
      </td>
    </tr>

    <tr>
      <td>
        Type
      </td>

      <td>
        GraphQL의 데이터는 유형별로 정리됩니다. 유형은 [스칼라](https://graphql.org/learn/schema/#scalar-types)(문자열, 숫자 또는 부울 등) 또는 객체 유형일 수 있습니다.

        [객체 유형](https://graphql.org/learn/schema/#object-types-and-fields)은 필드 모음으로 구성된 커스텀 유형입니다. 예를 들어, `User`라는 객체 유형은 시스템의 사용자를 나타낼 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        필드
      </td>

      <td>
        [필드](https://graphql.org/learn/schema/#object-types-and-fields)는 쿼리할 수 있는 객체 유형에 대한 정보를 나타냅니다. 필드는 스칼라, 목록 또는 객체일 수 있습니다. 예를 들어, `User` 객체 유형에는 `name`이라는 문자열 필드가 있을 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        인터페이스
      </td>

      <td>
        [인터페이스](https://graphql.org/learn/schema/#interfaces)는 다른 객체 유형이 `implement` 할 수 있는 공통 필드 모음을 나타내는 추상 유형입니다.
      </td>
    </tr>
  </tbody>
</table>
