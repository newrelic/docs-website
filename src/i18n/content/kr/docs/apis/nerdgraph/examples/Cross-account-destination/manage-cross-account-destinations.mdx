<DNT>NerdGraph API</DNT> 을 사용하여 <DNT>cross account destinations</DNT> 생성, 테스트, 나열, 업데이트 및 삭제합니다. <DNT>Cross account destinations</DNT> 사용하면 조직 수준에서 공지 관리를 중앙 집중화할 수 있습니다.

## 목적지 만들기 [#create-destination]

목적지를 생성하려면 목적지 유형별로 서로 다른 입력값을 제공해야 합니다.

<Callout variant="important">
  대상이 생성될 위치를 나타내는 스코프 객체를 제공해야 합니다. 스코프 객체에는 다음이 필요합니다.

  * id: 계정 또는 조직의 ID입니다.
  * 유형: 범위 유형(계정 또는 조직).

  양방향 통합을 허용하는 선택적 two\_way\_integration 속성이 통합에 제공됩니다.
</Callout>

        destination: {
          type: JIRA
          name: "Destination Name"
          auth: {
            type: BASIC
            basic: { user: YOUR_EMAIL, password: YOUR_PASSWORD }
          }
          properties: [
            { key: "url", value: "https://YOUR_INSTANCE.atlassian.net" }
