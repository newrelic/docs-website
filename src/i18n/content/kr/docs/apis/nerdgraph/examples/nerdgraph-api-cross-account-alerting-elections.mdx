조직 내 계정 간 데이터 공유를 허용하려면 해당 계정에서 &apos;계정 간 알림&apos; 기능을 선택해야 합니다. NerdGraph API Explorer를 사용하여 교차 계정 알림을 활성화하여 계정 설정을 업데이트하는 방법에 대한 다음 지침을 따르세요.

<Callout variant="important">
  &quot;데이터 공유 액세스 제어&quot; 권한이 있는 사용자만 데이터 공유를 위한 교차 계정 선거를 업데이트할 수 있습니다. 기본적으로 이 권한은 &quot;조직 관리자&quot; 역할이 있는 사용자에게만 부여됩니다.
</Callout>

## 교차 계정 알림 선거를 업데이트하는 단계

<Steps>
  <Step>
    업데이트할 계정 ID 목록을 수집합니다. API는 최대 **20개의 계정 ID를** 허용합니다.
  </Step>

  <Step>
    `true` 선거를 활성화할지, `false` 비활성화할지 표시하세요.
  </Step>

  <Step>
    NerdGraph API 탐색기를 사용하여 계정 선택을 업데이트합니다.
  </Step>
</Steps>

돌연변이의 예는 다음과 같습니다.

```graphql
mutation {
  alertsUpdateCrossAccountElections(
    accountIds: [YOUR_1ST_ACCOUNT_ID, YOUR_2ND_ACCOUNT_ID, YOUR_3RD_ACCOUNT_ID],
    electionStatus: true
  ) {
    electionStatus
  }
}
```
