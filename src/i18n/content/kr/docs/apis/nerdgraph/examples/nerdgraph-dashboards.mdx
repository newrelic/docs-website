[NerdGraph API를](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) 사용하여 <InlinePopover type="dashboards" />생성하고 관리할 수 있습니다.

## 개요 [#starting-out]

사용자 지정 대시보드 기능에 대한 소개 [는 대시보드 문서](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) 를 참조하십시오.

NerdGraph를 사용하여 대시보드를 구성할 때 대시보드는 모니터링되는 앱, 호스트 및 서비스와 같이 엔터티로 간주되는 다른 항목과 유사하게 고유한 엔터티 ID가 있는 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic) 로 간주된다는 점을 이해하는 데 도움이 됩니다.

대시보드에서 위젯 및 차트를 추가하고 구성하는 방법은 [차트 및 기타 위젯 구성](/docs/apis/nerdgraph/examples/create-widgets-dashboards-api) 을 참조하십시오.

## 대시보드 CRUD 작업 [#crud-operations]

이 문서에서는 NerdGraph API를 사용하여 대시보드(CRUD)를 만들고, 읽고, 업데이트하고, 삭제하는 방법을 설명합니다. 이러한 작업은 대시보드 전체를 수정합니다.

### 대시보드 만들기 [#create-dashboard]

대시보드에는 최소한 한 페이지가 필요합니다. 하나 이상의 페이지로 구성된 대시보드를 만들 수 있으며, 각 페이지에는 하나 이상의 위젯이 있을 수 있습니다.

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. NerdGraph의 대시보드 API 사용하여 새로운 대시보드와 해당 설정을 만드세요.
3. `name`, `permissions` 등 필수 필드와 최소한 한 페이지를 모두 포함해야 합니다.

### 대시보드 읽기 [#read-dashboard]

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. `actor > entity()` 사용하여 콘솔 GUID로 대시보드를 찾으세요.
3. NerdGraph의 대시보드 API 사용하여 기존 대시보드와 해당 설정을 읽을 수 있습니다.

이 예에서 `DASHBOARD_GUID` 읽으려는 대시보드의 실제 GUID로 바꾸세요.

```graphql
query GetDashboardEntityQuery {
	actor{
		entity(guid:"DASHBOARD_GUID"){
			...on DashboardEntity {
				guid
				name
				description
				createdAt
				updatedAt
				owner {
					email
					userId
				}
				permissions
				pages {
					guid
					name
					description
					createdAt
					updatedAt
					widgets {
						id
						visualization {
							id
						}
						layout {
							column
							row
							height
							width
						}
						title
						linkedEntities {
							guid
						}
						rawConfiguration
					}
				}
				variables {
					name
					items {
						title
						value
					}
					defaultValues {
						value {
							string
						}
					}
					nrqlQuery {
						accountIds
						query
					}
					options {
						excluded
						ignoreTimeRange
						showApplyAction
					}
					title
					type
					isMultiSelection
					replacementStrategy
				}
			}
		}
	}
}
```

검색하려는 정보에 따라 쿼리의 필드를 수정할 수 있습니다.

### 대시보드 업데이트 [#update-dashboard]

대시보드를 업데이트하려면, 하나의 요소만 업데이트하는 경우에도 모든 페이지와 개념을 포함하여 대시보드의 완전한 설정을 제공해야 합니다. 업데이트 작업은 대시보드의 내용을 완전히 교체하는 것입니다.

<Callout variant="important">
  대시보드를 업데이트할 때 페이지 GUID 또는 위젯 ID가 제공되지 않으면 기존 페이지 또는 위젯이 대시보드에서 제거되고 뮤테이션에 지정된 새 페이지 또는 위젯으로 교체됩니다.
</Callout>

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. NerdGraph의 대시보드 API 사용하여 대시 [보드 읽기](#read-dashboard) 섹션에 표시된 대로 해당 GUID로 대시보드를 읽어 기존 대시보드 설정을 가져옵니다.
3. 대시보드 설정에서 업데이트하려는 필드를 수정하세요.
4. NerdGraph의 대시보드 API 사용하여 수정된 설정으로 기존 대시보드를 업데이트하세요.

<Callout variant="important">
  위젯에서 `Facet Linking` 사용하는 경우 대시보드를 읽을 때 필드는 `linkedEntities { guid }` 이지만 대시보드를 업데이트할 때는 `linkedEntityGuids: [ "GUID" ]` 사용해야 합니다.
</Callout>

### 대시보드 삭제 [#delete-dashboard]

대시보드를 삭제하려면 삭제하려는 대시보드의 GUID를 제공해야 합니다. 이 작업은 대시보드를 복구할 수 있는 논리적 삭제를 실행합니다.

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. NerdGraph의 대시보드 API 사용하여 해당 GUID로 대시보드를 삭제하세요.
3. `status` 및 `errors` 체크하여 삭제를 확인하세요.

<CollapserGroup>
  <Collapser id="mutation-delete-dashboard" title="대시보드 삭제">
    ```graphql
    mutation {
      dashboardDelete(guid: "DASHBOARD_GUID") {
        status
        errors {
          type
          description
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>

#### 대시보드 삭제 취소 [#undelete-dashboard]

대시보드 GUID가 주어지면 이전에 삭제된 대시보드를 복구할 수 있습니다. 사용자 정의 태그는 복구할 수 없습니다.

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. NerdGraph의 대시보드 API 사용하여 편집 GUID로 대시보드 삭제를 취소하세요.
3. `errors` 이 있으면 체크하여 삭제 취소를 확인하세요.

## 대시보드 페이지 작업 [#page-operations]

이 작업은 대시보드의 특정 페이지를 수정합니다.

### 대시보드 페이지 업데이트 [#update-dashboard-page]

대시보드 페이지 구성 GUID가 주어지면 기존 대시보드의 한 페이지를 업데이트할 수 있습니다. 메타데이터에서 설명 설정까지 완전한 업데이트된 대시보드 페이지 요소를 지정해야 합니다.

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. NerdGraph의 대시보드 API 사용하여 대시 [보드 읽기](#read-dashboard) 섹션에 표시된 대로 해당 GUID로 대시보드를 읽어 기존 대시보드 설정을 가져옵니다.
3. 대시보드에서 업데이트할 페이지를 식별하여 추출합니다.
4. 페이지에서 업데이트하려는 필드를 수정합니다.
5. `dashboardUpdatePage()` 사용하여 페이지를 수정하세요.
6. `errors` 이 있으면 체크하세요.

<Callout variant="important">
  페이지를 업데이트할 때 위젯 ID가 제공되지 않으면 기존 위젯이 대시보드에서 제거되고 뮤테이션에 지정된 새 위젯으로 교체됩니다.
</Callout>

<Callout variant="tip">
  * `id` 필드 없이 `widgets` 시스템에 위젯을 포함시켜 페이지에 새 위젯을 추가할 수 있습니다.
  * `widgets` 구조에서 위젯을 생략하여 페이지에서 위젯을 제거할 수 있습니다.
</Callout>

### 페이지의 위젯을 업데이트합니다 [#update-widgets-in-page]

대시보드 페이지 GUID가 주어지면 대시보드 페이지의 기존 세트를 업데이트할 수 있습니다. 업데이트할 위젯 세트와 그에 대한 전체 설정을 지정해야 합니다.

<Callout variant="important">
  이 작업으로는 페이지에 위젯을 추가하거나 제거할 수 없습니다. 위젯을 추가하거나 제거하려면 [대시보드 페이지 업데이트](#update-dashboard-page) 작업을 사용하세요.
</Callout>

1. api.newrelic.com/graphiql에서 [NerdGraph](https://api.newrelic.com/graphiql) GraphiQL 탐색기로 이동합니다.
2. NerdGraph의 대시보드 API 사용하여 대시 [보드 읽기](#read-dashboard) 섹션에 표시된 대로 해당 GUID로 대시보드를 읽어 기존 대시보드 설정을 가져옵니다.
3. 대시보드에서 업데이트하려는 페이지의 위젯을 식별하고 추출합니다.
4. 위젯에서 업데이트하려는 필드를 수정합니다.
5. `dashboardUpdateWidgetsInPage()` 사용하여 위젯을 수정하세요.
6. `errors` 이 있으면 체크하세요.

```graphql
mutation UpdateWidgetsInPage($pageGuid: EntityGuid!, $widgets: [DashboardUpdateWidgetInput!]!) {
  dashboardUpdateWidgetsInPage(guid: $pageGuid, widgets: $widgets) {
    errors {
      type
      description
    }
  }
}
```

이 돌연변이에 사용할 변수는 다음과 같습니다.

```json lineHighlight=2,8

      "title": "Updated Total Transaction Count",
      "linkedEntityGuids": [],
      "rawConfiguration": {
        "nrqlQueries": [
          {
            "accountIds": [1],
            "query": "SELECT count(*) FROM Transaction"
          }
        ]
      }
    }
  ]
}
```

## 일등 시민으로서의 오류 [#errors-first-class]

모든 대시보드 변형은 실행 시 오류를 요청하는 방법을 제공합니다. 즉, 예상되는 잠재적 문제를 감지하기 위해 대시보드 변형을 수행하고 응답을 확인할 수 있습니다. 모든 오류에는 문제의 원인을 식별하는 데 도움이 되는 유형과 설명이 있습니다.

<CollapserGroup>
  <Collapser id="errors-first-class" title="모든 돌연변이 응답의 일부인 오류">
    ```graphql lineHighlight=2
    mutation {
      dashboardMutation(guid: "MY_EXISTING_DASHBOARD_GUID") {
        mutationResult {
          result
        }
        errors {
          description
          type
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>

이러한 오류는 우리가 미리 알고 있는 예상 오류임을 명심하십시오. 또한 표준 [GraphQL 오류 필드](https://graphql.org/learn/serving-over-http/#response) 에 반환될 예기치 않은 오류가 있는지 확인해야 합니다.
