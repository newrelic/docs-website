이 문서는 구성 예제를 위한 장소입니다. <InlinePopover type="apm"/>NerdGraph API를 사용하는 에이전트.

## 서버 측 구성 구성 [#server-side-config]

참고 <InlinePopover type="apm"/>에이전트가 NerdGraph를 통해 변경된 구성 값을 사용하려면 [서버 측 구성을](/docs/apm/agents/manage-apm-agents/configuration/server-side-agent-configuration#requirements) 활성화해야 합니다.

요구 사항은 [서버 측 구성 요구](/docs/apm/agents/manage-apm-agents/configuration/server-side-agent-configuration#requirements) 사항을 참조하십시오.

다음은 지정된 엔터티에 대한 서버 측 구성 설정의 상태를 반환하는 쿼리의 예입니다.

```
query ExampleReadQuery {
  actor {
    entity(guid:"ZjY1ODgxfEFQTXxBUFBYSUNBVElPTnz0ODEwMTY3NzZ") {
      ...on ApmApplicationEntity {
        apmSettings {
          apmConfig {
            useServerSideConfig
          }
        }
      }
    }
  }
}
```

다음은 [서버 측 구성](/docs/apm/agents/manage-apm-agents/configuration/server-side-agent-configuration) 을 비활성화하는 예입니다. `guid` 은 배열을 사용하므로 여러 항목을 업데이트하려는 경우에 유용할 수 있습니다.

```
mutation ExampleUpdateQuery {
  agentApplicationSettingsUpdate(guid: "ZjY1ODgxfEFQTXxBUFBYSUNBVElPTnz0ODEwMTY3NzZ", settings: {apmConfig: {useServerSideConfig: false}}) {
    apmSettings {
      apmConfig {
        useServerSideConfig
      }
    }
    errors {
      description
      errorClass
      field
    }
  }
}
```

엔터티의 GUID를 찾는 방법은 [엔터티 데이터 찾기](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/#find) 를 참조하세요.

## 설정 검색 [#get-settings]

다음은 엔터티의 트랜잭션 추적기 설정을 반환하는 예입니다.

```
query ExampleReadQuery {
  actor {
    entity(guid:"ZjY1ODgxfEFQTXxBUFBYSUNBVElPTnz0ODEwMTY3NzZ") {
    ... on ApmApplicationEntity {
        guid
        name
        apmSettings {
          transactionTracer {
            enabled
            explainEnabled
            explainThresholdType
            explainThresholdValue
          }
        }
      }
    }
  }
}
```
