## 비디오 개요 페이지 UI

**All Platforms &gt; Video Overview** 페이지에서는 다양한 플랫폼 전반의 비디오 성능에 대한 광범위한 보기를 제공합니다. 이는 모든 모니터에 대한 비디오 성능 지표에 대한 포괄적인 개요를 제공합니다. 비디오 개요 페이지에는 다음 정보가 포함되어 있습니다.

<img title="overview of the media summary page" alt="A gif depicting an overview of the media summary page" src="/images/video-overview-page.gif" />

모든 플랫폼에서 비디오 에이전트의 비디오 개요 페이지를 보려면 <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Streaming Video &amp; Ads &gt; Video Overview**</DNT> 로 이동하세요.

## 필터 기능 사용하기 [#using-the-filter-functionality]

기본적으로 **Video Overview** \[비디오 개요] 페이지에는 비디오 에이전트가 포함된 모든 소스에 대한 비디오 재생 성능 데이터가 표시됩니다. 필터 바를 사용하면 모든 속성이나 값을 검색할 수 있습니다. 예를 들어, 콘텐츠 제목, 장치 ID, 사용자 ID, 플랫폼 등으로 데이터를 필터링할 수 있습니다.

## 차원 KPI

플랫폼 및 비디오 플레이어 이름을 포함한 다양한 트리거를 활용하여 트래픽을 심층적으로 탐색하고 분석합니다. 고객의 시청 습관에 대한 정보를 얻으세요. 여기에는 특정 콘텐츠를 시청하는 고객의 비율, 비디오를 시청하는 해상도, 장치의 운영 체제 버전에 대한 자세한 정보가 포함됩니다.

* **플랫폼**: Roku, 브라우저, Android 또는 iOS 장치와 같은 장치 유형입니다.

* **OS 버전**: 장치의 운영 시스템 버전입니다. Android 또는 iOS 기기를 사용하는 경우 운영 시스템의 버전이 표시됩니다.

* **비디오 플레이어 이름**: 비디오를 재생하는 데 사용된 비디오 플레이어의 이름입니다.

* **ISP**: 시청자가 사용하는 인터넷 서비스 공급업체(ISP)입니다.

* **지역**: 시청자의 지리적 위치.

* **비디오 해상도**: 시청자가 재생하는 비디오의 해상도입니다.

## 스트리밍 추가 시스템

기본적으로 **Streaming Entities** 페이지에는 해당 계정과 관련된 모든 Insights가 표시됩니다. 하지만 새로운 스트리밍을 생성하고 싶다면:

1. **+ Add streaming entity** \[+ 스트리밍 추가 버튼] 을 클릭하세요.
2. 모니터링하려는 앱 유형을 선택하세요. [비디오 에이전트 설치를](/docs/streaming-video-&-ads/installation/install-the-streaming-video-&-ads-agent/#installation)참조하세요
