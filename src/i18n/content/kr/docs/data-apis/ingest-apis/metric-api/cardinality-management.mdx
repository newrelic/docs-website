</Callout>

기수란 집합 내 고유한 요소의 수, 특히 하루 기간 내에 주어진 지표에 대해 관찰된 고유한 속성 조합의 수를 나타냅니다. 이 데이터는 매일 UTC 자정에 재설정됩니다. 고유한 조합이 많아질수록 기수가 증가하므로 데이터 제한에 영향을 미칠 수 있습니다. 데이터 사용을 최적화하고 제한 문제를 방지하려면 기수를 이해하고 관리하는 것이 중요합니다.

뉴렐릭 관리자는 데이터 수집을 감독하여 지정된 한도 내에서 유지되도록 합니다. <DNT>**Cardinality Management**</DNT> 데이터 사용량을 자세히 보여주는 그래프와 표를 제공하므로, 사용량이 한계에 도달하면 조치를 취할 수 있습니다.

**<DNT>Cardinality Management</DNT>** UI를 보려면 **<DNT>[one.newrelic.com](https://one.newrelic.com)</DNT> &amp;gt; (사용자 이름 클릭) &amp;gt; <DNT>Administration &gt; Cardinality Management</DNT>** 로 이동하세요.

## 계정 기수 분석 [#account-breakdown]

조직에 여러 계정이 있는 경우 각 계정의 카디널리티 세부 정보를 볼 수 있습니다. 계정을 전환하려면 오른쪽 상단에 있는 계정 선택기를 사용하세요. 계정에 액세스할 수 있으면 UI에서 계정 기수 정보를 볼 수 있습니다.

### 시간 경과에 따른 사용 [#usage-over-time]

<DNT>**Cardinality Managemen**t</DNT> 의 랜딩 페이지에서 <DNT>**Usage over time**</DNT> 차트를 볼 수 있습니다. 차트는 계정 한도와 비교한 계정 기수 사용량을 표시합니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the usage over time chart" alt="Screenshot showing the usage over time chart" src="/images/apis-and-data_screenshot-crop_usage-over-time.webp" />

<Callout variant="tip">
  기수 사용 값이 변경되지 않으면 새로운 데이터 포인트가 보고되지 않습니다. 그러면 빈 데이터 포인트가 보일 수 있습니다. 이것은 데이터 손실이 아닙니다.
</Callout>

시계열을 조정하려면 오른쪽 상단 모서리에 있는 타임 피커를 사용하세요. 이 변경 사항은 차트에 있는 시계열에만 영향을 미치며 뒤에 나오는 메트릭 표에는 영향을 미치지 않습니다.

### 메트릭 테이블 [#metrics-table]

해당일에 선택한 계정의 카디널리티 상위 20개 지표를 확인하세요. <DNT>**Metrics**</DNT> 테이블에서 메트릭 이름별로 구성된 계정 기수에 대한 세부 정보를 확인하세요.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the metrics table" alt="Screenshot showing the metrics table" src="/images/apis-and-data_screenshot-crop_metrics-table.webp" />

모든 메트릭에는 100,000의 기본 카디널리티 제한이 있습니다. 요구 사항에 따라 [메트릭별 제한을 조정할](#per-metric-limit-adjustment) 수 있습니다.

## 메트릭 카디널리티 분석 [#metric-cardinality-breakdown]

<DNT>**Metrics**</DNT> 테이블에서 메트릭을 선택하면 전용 <DNT>**Cardinality Management**</DNT> 페이지에서 자세한 기수 정보를 볼 수 있습니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the Cardinality Management page" alt="Screenshot showing the Cardinality Management page" src="/images/apis-and-data_screenshot-crop_metric-cardinality-breakdown.webp" />

### 바마켓

지표별 보기에서 두 개의 바 차트를 사용하여 현재 기수 사용량을 확인할 수 있습니다.

* <DNT>**Metric cardinality budget**</DNT> (제한): 메트릭에 특정한 기수 사용
* <DNT>**Account cardinality budget**</DNT> (제한): 계정의 전체 기수 사용량

### 시간 경과에 따른 사용(표적용) [#usage-over-time-metrics]

<DNT>**Usage over time**</DNT> 차트는 해당 메트릭의 한도와 비교한 메트릭 카디널리티 사용량을 표시합니다. 이 개별 지표의 기수 데이터의 다양한 기간을 보려면 타임테이블 피커를 조정할 수 있습니다.

<Callout variant="tip">
  지표가 기수 제한을 초과하는 경우에도 뉴렐릭은 여전히 지표에 대한 정보를 채우지만, 집계된 데이터 대신 원시 데이터를 사용하므로 약간 느릴 수 있습니다.
</Callout>

### 속성 테이블 [#attributes-table]

<DNT>**Attributes**</DNT> 테이블에는 지표의 카디널리티에 크게 기여하는 상위 20개 속성 목록이 있습니다. 특정 속성의 값을 보려면 속성 행의 오른쪽에 있는 옵션 메뉴를 클릭하고 <DNT>**View attribute values**</DNT> 선택하세요.

## 계정 정리 규칙 [#account-pruning-rules]

계정 기수를 한도 내로 유지하기 위한 일반적인 방법은 중요하지 않은 속성에 대한 정리 규칙을 구현하여 불필요한 데이터를 제거하는 것입니다.

### 계정 정리 규칙 보기 [#view-pruning-rules]

계정 정리 규칙을 보려면:

1. 계정의 **Cardinality Management** \[기수 관리] 페이지로 이동합니다.
2. **Metrics** \[메트릭] 테이블에서 <DNT>**View Pruning Rules**</DNT> 클릭합니다. 해당 계정과 관련된 모든 가지치기 규칙이 스크롤 가능한 목록으로 나타납니다.

<Callout variant="tip">
  계정에 가지치기 규칙이 없으면 <DNT>**View Pruning Rules**</DNT> 버튼이 나타나지 않습니다.
</Callout>

### 가지치기 규칙 만들기 [#create-pruning-rule]

가지치기 규칙을 만들려면:

1. 계정의 **Cardinality Management** \[기수 관리] 페이지로 이동합니다.

2. **Metrics** \[지표] 표에서 가지치기 규칙을 만들려는 지표를 선택합니다. 메트릭의 **Cardinality Management** \[카디널리티 관리] 페이지가 열립니다.

3. **Attributes** \[속성] 테이블로 가서 정리하려는 속성을 선택합니다.

   <img style={{ align: 'left', maxWidth: '90%' }} title="Screenshot showing how to select attributes for pruning rule" alt="Screenshot showing how to select attributes for pruning rule" src="/images/apis-and-data_screenshot-crop_select-attributes-for-pruning-rule.webp" />

4. <DNT>**Create pruning rule**</DNT> 클릭하세요. 팝업이 나타나서 가지치기 규칙이 지표의 기수에 어떤 영향을 미치는지에 대한 세부 정보를 제공합니다.

<img style={{ align: 'left' }} title="Screenshot of page to name the pruning rule" alt="Screenshot of page to name the pruning rule" src="/images/apis-and-data_screenshot-crop_name-pruning-rule.webp" />

3. 규칙 이름이나 설명을 입력하세요.

4. 가지치기 규칙을 만들려면 <DNT>**Submit**</DNT> 을 클릭하세요.

<Callout variant="important">
  가지치기 규칙은 즉시 적용됩니다. 규칙을 만들기 전에 데이터를 정리할 것인지 확인하세요. 가지치기 규칙을 만든 후에는 삭제할 수 있지만, 규칙을 제거하기 전에 일부 데이터가 가지치기될 수 있습니다.
</Callout>

### 가지치기 규칙 삭제 [#delete-pruning-rule]

가지치기 규칙을 삭제하려면:

1. 계정의 **Cardinality Management** \[기수 관리] 페이지로 이동합니다.
2. **Metrics** \[메트릭] 테이블에서 <DNT>**View Pruning Rules**</DNT> 선택합니다.
3. 해당 가지치기 규칙으로 스크롤하여 삭제하세요.

## 메트릭당 제한 조정 [#per-metric-limit-adjustment]

특정 메트릭의 모든 속성이 중요하고 카디널리티 한도에 접근하는 경우, 필요에 맞게 메트릭별 한도를 조정할 수 있습니다. 일반적으로 Indicator의 기본 최대 카디널리티는 100,000이지만, Indicator당 최대 100만까지 늘릴 수 있습니다.

<Callout variant="tip">
  계정의 전체 기수 한도는 변경되지 않습니다. 따라서 신중하게 고려하지 않고 모든 메트릭 한도를 100만으로 설정하면 계정의 기수 용량이 빠르게 소진될 수 있습니다.
</Callout>

### 메트릭당 한도 편집 [#edit-per-metric-limit]

1. 특정 메트릭의 <DNT>**Cardinality Management**</DNT> 페이지로 이동합니다.
2. <DNT>**Adjust metric budget**</DNT> 선택합니다.
3. 귀하의 요구 사항에 맞게 한도 값을 업데이트하세요. 100만 이하인지 확인하세요.

업데이트된 한도는 몇 분 내에 <DNT>**Cardinality Management**</DNT> 페이지에 나타납니다.
