인증 도메인 관리자로서 사용자에게 계정 및 메트릭 기수 관리 액세스 권한을 부여하려면 다음 단계를 수행합니다. 사용자의 특정 요구 사항을 확인하고 그에 따라 설정을 진행하세요. 사용자 관리에 대한 자세한 내용은 [사용자 관리 튜토리얼을](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/) 참조하세요.

<Steps>
  <Step>
    ### 사용자 정의 역할에 권한 추가 [#permissions-to-custom-role]

    다음 권한으로 [새 사용자 지정 역할을 만들](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) 거나 기존 역할에 권한을 추가합니다. 사용자에게 문의하여 필요한 액세스 수준을 확인하세요.

    * `Manage account cardinality`: 계정의 기수 예산을 조정합니다.

      * `View`: 계정의 기수 예산을 볼 수 있습니다.
      * `Modify`: 계정의 기수 예산을 업데이트할 수 있습니다.

    * `Manage metric cardinality`: 메트릭의 기수 예산을 조정합니다.

      * `View`: 메트릭의 기수 예산을 볼 수 있습니다.
      * `Modify`: 메트릭의 기수 예산을 업데이트할 수 있습니다.

    * `Manage pruning rules`: 메트릭에 대한 정리 규칙을 관리합니다.

      * `View`: 메트릭의 정리 규칙을 볼 수 있습니다.
      * `Modify`: 메트릭에 대한 정리 규칙을 만들고 삭제할 수 있습니다.

    * `Customer limits management`: 기수를 포함한 계정의 다양한 기능 한도를 조정합니다.

      * `View`: 기능에 대해 사용 가능한 제한 사항을 확인할 수 있습니다.
      * `Modify`: 기능의 한계를 조정할 수 있습니다.
  </Step>

  <Step>
    ### 사용자 역할을 추가하고 그룹에 계정을 추가하세요 [#group-access]

    기수 관리를 위해 다음과 같이 [새 그룹을 만들](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#group-access) 거나 기존 그룹을 관리하세요.

    1. **[New Relic Administration](https://one.newrelic.com/admin-portal) &gt; Access Management &gt; Groups** 으로 이동합니다.
    2. <DNT>**Create a new group**</DNT> 클릭하거나 <DNT>**Manage group access**</DNT> 선택하세요. <Icon name="fe-more-horizontal" /> 기존 그룹의 메뉴.
    3. 새 그룹의 경우 이름을 입력하고 <DNT>**Create group**</DNT> 을 클릭하세요. 해당 그룹의 <DNT>**Admin**</DNT> 창이 열립니다.
    4. <DNT>**Members**</DNT> 에서 사용자를 추가합니다.
    5. <DNT>**Account access**</DNT> 에서 사용자 지정 역할을 기수 관리를 위한 원하는 계정과 연결합니다.
    6. 기수 관리에 여러 계정이 관련된 경우 <DNT>**Account access**</DNT> 에서 그룹 내의 각 관련 계정에 대한 사용자 지정 역할을 포함합니다.
  </Step>
</Steps>
