기수란 집합 내 고유한 요소의 수, 특히 하루 동안 주어진 지표에 대해 관찰된 고유한 속성 조합의 수를 나타냅니다. 이 데이터는 매일 UTC 자정에 재설정됩니다. 고유한 조합이 많아질수록 기수가 증가하므로 데이터 제한에 영향을 미칠 수 있습니다. 카디널리티가 어떻게 작동하는지 이해하려면 [이해 및 쿼리 카디널리티 지수](/docs/data-apis/ingest-apis/metric-api/NRQL-high-cardinality-metrics/) 페이지를 참조하세요.

뉴렐릭 관리자는 데이터 수집을 감독하여 지정된 예산 내에서 데이터 수집이 이루어지도록 합니다. <DNT>**Cardinality Management**</DNT> 데이터 사용량을 자세히 보여주는 그래프와 표를 제공하여, 사용량이 한계에 도달하면 조치를 취할 수 있도록 도와줍니다.

**<DNT>Cardinality Management</DNT>** UI를 보려면 **<DNT>[one.newrelic.com](https://one.newrelic.com)</DNT> &gt; (사용자 이름 클릭) &gt; <DNT>Administration &gt; Cardinality Management</DNT>** 로 이동하세요.

## 계정 기수 모니터링 [#account-breakdown]

조직에 여러 계정이 있는 경우 각 계정의 카디널리티 세부 정보를 볼 수 있습니다. 계정을 전환하려면 오른쪽 상단에 있는 계정 선택기를 사용하세요. 계정에 액세스할 수 있으면 UI에서 계정 기수 정보를 볼 수 있습니다.

### 시간 경과에 따른 사용 [#usage-over-time]

<DNT>**Cardinality Management**</DNT> 의 랜딩 페이지에서 <DNT>**Usage over time**</DNT> 차트를 볼 수 있습니다. 차트는 해당 계정의 예산과 비교한 계정 기수 사용량을 표시합니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the usage over time chart" alt="Screenshot showing the usage over time chart" src="/images/apis-and-data_screenshot-crop_usage-over-time.webp" />

<Callout variant="tip">
  기수 사용 값이 변경되지 않으면 새로운 데이터 포인트가 보고되지 않습니다. 그러면 빈 데이터 포인트가 보일 수 있습니다. 이것은 데이터 손실이 아닙니다.
</Callout>

시계열을 조정하려면 오른쪽 상단 모서리에 있는 타임 피커를 사용하세요. 이 변경 사항은 차트에 있는 시계열에만 영향을 미치며 뒤에 나오는 메트릭 표에는 영향을 미치지 않습니다.

### 메트릭 테이블 [#metrics-table]

해당일에 선택한 계정의 카디널리티 상위 20개 지표를 확인하세요. <DNT>**Metrics**</DNT> 테이블에서 메트릭 이름별로 구성된 계정 기수에 대한 세부 정보를 확인하세요.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the metrics table" alt="Screenshot showing the metrics table" src="/images/apis-and-data_screenshot-crop_metrics-table.webp" />

모든 지표에는 100,000의 기본 기수 예산이 있습니다. 귀하의 요구 사항에 따라 [메트릭별 예산을 조정할](#per-metric-limit-adjustment) 수 있습니다.

## 메트릭 카디널리티 모니터링 [#metric-cardinality-breakdown]

<DNT>**Metrics**</DNT> 테이블에서 메트릭을 선택하면 전용 <DNT>**Cardinality Management**</DNT> 페이지에서 자세한 기수 정보를 볼 수 있습니다.

<img style={{ align: 'left',maxWidth: '90%' }} title="Screenshot showing the Cardinality Management page" alt="Screenshot showing the Cardinality Management page" src="/images/apis-and-data_screenshot-crop_metric-cardinality-breakdown.webp" />

### 바마켓

지표별 보기에서 두 개의 바 차트를 사용하여 현재 기수 사용량을 확인할 수 있습니다.

* <DNT>**Metric cardinality budget**</DNT>: 메트릭에 특정한 기수 사용
* <DNT>**Account cardinality budget**</DNT>: 계정의 전체 기수 사용량

### 시간 경과에 따른 사용(표적용) [#usage-over-time-metrics]

<DNT>**Usage over time**</DNT> 차트는 해당 메트릭의 예산과 비교한 메트릭 카디널리티 사용량을 표시합니다. 타임 피커를 조정하면 이 개별 지표의 기수 데이터의 다양한 기간을 볼 수 있습니다.

<Callout variant="tip">
  지표가 기수 예산을 초과하는 경우에도 뉴렐릭은 지표에 대한 정보를 계속 채우지만, 집계된 데이터 대신 원시 데이터를 사용하므로 약간 더 느릴 수 있습니다.
</Callout>

### 속성 테이블 [#attributes-table]

<DNT>**Attributes**</DNT> 테이블에는 지표의 카디널리티에 크게 기여하는 상위 20개 속성 목록이 있습니다. 특정 속성의 값을 보려면 속성 행의 오른쪽에 있는 옵션 메뉴를 클릭하고 <DNT>**View attribute values**</DNT> 선택하세요.
