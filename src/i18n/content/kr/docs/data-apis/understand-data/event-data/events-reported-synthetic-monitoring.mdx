[New Relic의 종합 모니터링은](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) 일부 UI 디스플레이에 표시되고 [쿼리 및 차트 작성](/docs/using-new-relic/data/understand-data/query-new-relic-data) 에도 사용할 수 있는 [이벤트 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 를 보고합니다. 속성을 보려면 다음 표에서 이벤트 이름을 선택하십시오.

  <thead>
    <tr>
      <th style={{ width: "300px" }}>
        이벤트
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="syntheticcheck-error">
        [`SyntheticCheck`](/attribute-dictionary/?event=SyntheticCheck)
      </td>

      <td>
        `SyntheticCheck` 특정 모니터의 한 번 실행에서 메트릭을 반환합니다. 이러한 메트릭에는 모니터에 대한 기간 정보, 모니터 확인 위치, 요청 및 응답 헤더 크기, 모니터 유형, 타임스탬프가 포함됩니다.

        합성 모니터가 검사를 실행할 때마다 검사에 대한 세부정보가 `SyntheticCheck` 이벤트 유형에 캡처됩니다. `SyntheticCheck` 이벤트에는 상태, 모니터 유형, 요청 및 응답 헤더 크기와 같은 가시성을 제공하기 위한 검사와 관련된 세부정보가 포함됩니다.
      </td>
    </tr>

    <tr>
      <td id="syntheticrequest">
        [`SyntheticRequest`](/attribute-dictionary/?event=SyntheticRequest)
      </td>

      <td>
        `SyntheticRequest` 검사 중에 이루어진 개별 HTTP 요청의 결과를 반환합니다. 수집된 데이터에는 작업 정보, 위치, 요청 콘텐츠 유형, 기간 정보, 요청 크기 및 페이지 로드 정보가 포함됩니다.

        단순하거나 스크립트로 작성된 각 모니터 검사를 통해 검사 중에 이루어진 각 개별 HTTP 요청을 캡처합니다. HTTP 세부정보는 `SyntheticCheck` 이벤트 유형보다 더 세분화된 수준에서 캡처됩니다.
      </td>
    </tr>

    <tr>
      <td>
        [`SyntheticPrivateLocationStatus`](/attribute-dictionary/?event=SyntheticsPrivateLocationStatus)
      </td>

      <td>
        개인 위치에서 실행되는 모든 모니터 검사는 해당 개인 위치에 대한 용량 세부 정보를 트리거합니다. 이러한 세부정보는 `SyntheticPrivateLocationStatus` 이벤트에서 캡처됩니다. 이를 통해 개인 위치의 용량과 워크로드를 지원하기 위해 추가 미니언이 필요한지 여부에 대한 가시성을 제공합니다.
      </td>
    </tr>

    <tr>
      <td id="synthetic-private-minion">
        [`SyntheticPrivateMinion`](/attribute-dictionary/?event=SyntheticsPrivateMinion)
      </td>

      <td>
        방화벽 내부와 같은 비공개 위치가 있는 경우 `SyntheticPrivateMinion` 이벤트를 사용하여 해당 위치에 대한 정보를 볼 수 있습니다.

        실행 중인 각 비공개 미니언은 30초마다 상태 세부정보를 `SyntheticPrivateMinion` 에 보냅니다. 이를 통해 해당 위치에서 실행 중인 개인 미니언의 상태를 이해할 수 있습니다.
      </td>
    </tr>
  </tbody>
</table>
