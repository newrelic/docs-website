[APM](/docs/apm/new-relic-apm/getting-started) 은 일부 UI 디스플레이에 나타나는 [이벤트 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 를 보고하며 [쿼리 및 차트 작성](/docs/using-new-relic/data/understand-data/query-new-relic-data) 에도 사용할 수 있습니다. 속성을 보려면 다음 표에서 이벤트 이름을 선택하십시오.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        이벤트
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="transaction-event">
        [`Transaction`](/attribute-dictionary/?event=Transaction)
      </td>

      <td>
        트랜잭션은 소프트웨어 애플리케이션의 논리적 작업 단위입니다. `Transaction` 이벤트에는 앱, 데이터베이스 호출, 트랜잭션 기간 및 발생할 수 있는 오류에 대한 정보가 포함됩니다.
      </td>
    </tr>

    <tr>
      <td id="transactionerror-event">
        [`TransactionError`](/attribute-dictionary/?event=TransactionError)
      </td>

      <td>
        트랜잭션 오류는 요청이 해당 요청을 채우기 위해 취한 코드 경로에서 예외를 throw할 때 발생합니다. [오류를 예상대로 수집, 무시 또는 표시](/docs/agents/manage-apm-agents/agent-data/manage-errors-apm-collect-ignore-or-mark-expected) 할지 여부를 지정할 수 있으므로 트랜잭션 오류 수는 트랜잭션 수와 동일하지 않습니다.
      </td>
    </tr>

    <tr>
      <td id="span-event">
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        Span 이벤트는 New Relic 에이전트 또는 기타 추적 계측 도구가 분산 추적의 일부인 계측 작업을 수행할 때 생성됩니다. [span 의 정의를](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#span) 참조하십시오.
      </td>
    </tr>
  </tbody>
</table>
