는 일부 UI 디스플레이에 표시되고 [쿼리 및](/docs/using-new-relic/data/understand-data/query-new-relic-data) [<InlinePopover type="mobile"/>](/docs/mobile-monitoring/new-relic-mobile/getting-started/introduction-new-relic-mobile)차트 작성 에도 [사용할 수 있는 이벤트 데이터를](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 보고합니다. 해당 속성을 보려면 다음 표에서 이벤트 이름을 선택하십시오.

  <tbody>
    <tr>
      <td>
        [`Mobile`](/attribute-dictionary/?event=Mobile)
      </td>

      <td>
        `Mobile` 이벤트는 충돌이 발생하거나 상호작용이 종료되거나 1초 동안 실행된 경우 또는 앱이 닫히거나 백그라운드에서 실행되거나 10분 동안 실행된 후 세션이 완료되는 경우 생성됩니다. `Mobile` 이벤트는 한 때 유일한 이벤트 유형이었고 모든 이벤트에 대해 생성되었지만 지금은 몇 가지 특수 이벤트 유형이 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileApplicationExit`](/attribute-dictionary/?event=mobileApplicationExit)
      </td>

      <td>
        충돌이나 메모리 부족, ANR 등과 같은 기타 이유로 인해 앱이 종료되면 `MobileApplicationExit` 이벤트가 생성됩니다. 이 이벤트는 앱 종료와 관련된 상황에 대한 귀중한 인사이트를 제공하며 앱 종료 이유, 종료 코드, 설명과 같은 속성을 포함합니다.

        현재 `MobileApplicationExit` 이벤트는 Android 모바일 애플리케이션에 대해서만 보고됩니다. Android의 `ApplicationExitInfo` API를 활용하여 앱이 종료되는 다양한 이유를 모니터링하고 보고합니다.
      </td>
    </tr>

    <tr>
      <td>
        `MobileBreadcrumb`
      </td>

      <td>
        이 이벤트에 대해 나열된 속성이 없습니다. 속성에는 [세션 속성](#session-list) 과 [추가된 사용자 정의 속성이](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile)포함됩니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileCrash`](/attribute-dictionary/?event=MobileCrash)
      </td>

      <td>
        `MobileCrash` 이벤트는 앱이 다운될 때 생성됩니다. `MobileCrash` 에는 충돌 줄 번호, 클래스 및 충돌 메시지와 같은 속성이 포함됩니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileHandledException`](/attribute-dictionary/?event=MobileHandledException)
      </td>

      <td>
        `MobileHandledException` 예외가 포착되면 전송되며 [`recordHandledException` API 호출을](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/) 사용하여 New Relic에 보고되는 치명적이지 않은 예외에 사용됩니다.

        예외는 스택 추적을 포함하여 이 이벤트의 쿼리와 [<DNT>**Handled exceptions**</DNT> UI 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/handled-exceptions-analyze-trends-prevent-crashes) 에 표시됩니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequest`](/attribute-dictionary/?event=MobileRequest)
      </td>

      <td>
        HTTP 요청이 성공적으로 완료되면 `MobileRequest` 이벤트가 생성되어 응답 코드가 400 미만이 됩니다.

        New Relic `MobileRequest` 데이터는 기본적으로 다음에 대해 활성화됩니다.

        * [안드로이드 버전 5.15.2](/docs/release-notes/mobile-release-notes/android-release-notes/android-5152) 이상

        * [iOS 버전 6.0.0](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-600) 이상

          이전 버전의 경우 Android 버전 5.14.0 또는 iOS 버전 5.14.0부터 해당 기능을 활성화해야 합니다. 최신 [Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/upgrade-android-agent) 또는 [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/installation/upgrade-ios-agent) 버전으로 업그레이드하거나 [Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-configuration-feature-flags) 또는 [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/ios-agent-configuration-feature-flags#networkRequest) 구성 설정을 사용하여 `NetworkRequests` 기능 플래그를 활성화하세요.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequestError`](/attribute-dictionary/?event=MobileRequestError)
      </td>

      <td>
        `MobileRequestError` 은 HTTP 오류 또는 네트워크 오류에 사용됩니다. HTTP 오류는 상태 코드가 400보다 큰 HTTP 요청입니다. 네트워크 오류는 응답이 없는 HTTP 요청입니다. HTTP 요청이 완료되면 이벤트가 전송됩니다.
      </td>
    </tr>

    <tr id="session-list">
      <td>
        [`MobileSession`](/attribute-dictionary/?event=MobileSession)
      </td>

      <td>
        `MobileSession` 이벤트는 앱이 닫히거나 백그라운드에서 실행되거나 10분 동안 활성 사용이 경과하면 전송됩니다. 다른 모바일 모니터링 이벤트에서 사용하는 일반 세션 데이터의 소스입니다. `MobileSession` 은 기기 유형, 기기 OS 및 지리적 정보와 같은 속성을 캡처합니다.
      </td>
    </tr>

    <tr>
      <td>
        `MobileUserAction`
      </td>

      <td>
        이 이벤트에 대해 나열된 속성이 없습니다. 속성에는 [세션 속성](#session-list) 과 [추가된 사용자 정의 속성이](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile)포함됩니다.

        이 이벤트는 분산 추적 기능이 활성화된 경우 일부 사용자 작업으로 인해 에이전트가 시작/포그라운드로 돌아가거나 중지/백그라운드로 돌아가게 되었다고 가정할 때 기록됩니다.
      </td>
    </tr>

    <tr>
      <td>
        맞춤 이벤트
      </td>

      <td>
        이 이벤트 유형은 사용자 정의 대시보드이므로 속성 목록이 없습니다. 속성에는 [세션 속성](#session-list) 과 추가한 모든 [사용자 정의 속성이](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile) 포함됩니다.
      </td>
    </tr>
  </tbody>
</table>
