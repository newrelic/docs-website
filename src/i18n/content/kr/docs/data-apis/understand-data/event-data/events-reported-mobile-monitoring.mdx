[New Relic의 모바일 모니터링은](/docs/mobile-monitoring/new-relic-mobile/getting-started/introduction-new-relic-mobile) 일부 UI 디스플레이에 표시되고 [쿼리 및 차트 작성](/docs/using-new-relic/data/understand-data/query-new-relic-data) 에도 사용할 수 있는 [이벤트 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 를 보고합니다. 속성을 보려면 다음 표에서 이벤트 이름을 선택하십시오.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        이벤트
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`Mobile`](/attribute-dictionary/?event=Mobile)
      </td>

      <td>
        `Mobile` 이벤트는 충돌이 발생하거나 상호작용이 종료되거나 1초 동안 실행된 경우 또는 앱이 닫히거나 백그라운드에서 실행되거나 10분 동안 실행된 후 세션이 완료되는 경우 생성됩니다. `Mobile` 이벤트는 한 때 유일한 이벤트 유형이었고 모든 이벤트에 대해 생성되었지만 지금은 몇 가지 특수 이벤트 유형이 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        `MobileBreadcrumb`
      </td>

      <td>
        이 이벤트에 대해 나열된 속성이 없습니다. 속성에는 [세션 속성](#session-list) 과 [추가된 사용자 정의 속성이](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile)포함됩니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileCrash`](/attribute-dictionary/?event=MobileCrash)
      </td>

      <td>
        `MobileCrash` 이벤트는 앱이 다운될 때 생성됩니다. `MobileCrash` 에는 충돌 줄 번호, 클래스 및 충돌 메시지와 같은 속성이 포함됩니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileHandledException`](/attribute-dictionary/?event=MobileHandledException)
      </td>

      <td>
        `MobileHandledException` 예외가 포착될 때 전송되고 [Android](/docs/mobile-monitoring/new-relic-mobile-android/android-sdk-api/recordhandledexception-android-sdk-api) 또는 [iOS용](/docs/mobile-monitoring/new-relic-mobile-ios/ios-sdk-api/record-handled-exception) `recordHandledException` API 호출을 사용하여 New Relic에 보고된 치명적이지 않은 예외에 사용됩니다.

        예외는 스택 추적을 포함하여 이 이벤트의 쿼리와 [**처리된 예외** UI 페이지](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/handled-exceptions-analyze-trends-prevent-crashes) 에서 볼 수 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequest`](/attribute-dictionary/?event=MobileRequest)
      </td>

      <td>
        HTTP 요청이 성공적으로 완료되면 `MobileRequest` 이벤트가 생성되어 응답 코드가 400 미만이 됩니다.

        New Relic `MobileRequest` 데이터는 기본적으로 다음에 대해 활성화됩니다.

        * [안드로이드 버전 5.15.2](/docs/release-notes/mobile-release-notes/android-release-notes/android-5152) 이상

        * [iOS 버전 6.0.0](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-600) 이상

          이전 버전의 경우 Android 버전 5.14.0 또는 iOS 버전 5.14.0부터 기능을 활성화해야 합니다. 최신 [Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/upgrade-new-relic-mobiles-android-sdk) 또는 [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/installation/upgrade-new-relic-mobiles-ios-sdk) 버전으로 업그레이드하거나 [Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-configuration-feature-flags) 또는 [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/ios-agent-configuration-feature-flags#networkRequest) 구성 설정을 사용하여 `NetworkRequests` 기능 플래그를 활성화하세요.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequestError`](/attribute-dictionary/?event=MobileRequestError)
      </td>

      <td>
        `MobileRequestError` 은 HTTP 오류 또는 네트워크 오류에 사용됩니다. HTTP 오류는 상태 코드가 400보다 큰 HTTP 요청입니다. 네트워크 오류는 응답이 없는 HTTP 요청입니다. HTTP 요청이 완료되면 이벤트가 전송됩니다.
      </td>
    </tr>

    <tr id="session-list">
      <td>
        [`MobileSession`](/attribute-dictionary/?event=MobileSession)
      </td>

      <td>
        `MobileSession` 이벤트는 앱이 닫히거나 백그라운드에서 실행되거나 10분 동안 활성 사용이 경과하면 전송됩니다. 다른 모바일 모니터링 이벤트에서 사용하는 일반 세션 데이터의 소스입니다. `MobileSession` 은 기기 유형, 기기 OS 및 지리적 정보와 같은 속성을 캡처합니다.
      </td>
    </tr>

    <tr>
      <td>
        맞춤 이벤트
      </td>

      <td>
        이 이벤트 유형은 사용자 정의 이벤트이므로 속성 목록이 없습니다. 속성에는 [세션 속성](#session-list) 과 [사용자가 추가한 모든 사용자 정의 속성](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile) 이 포함됩니다.
      </td>
    </tr>
  </tbody>
</table>
