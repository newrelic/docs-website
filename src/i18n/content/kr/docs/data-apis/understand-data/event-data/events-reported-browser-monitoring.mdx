[<InlinePopover type="browser" />](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)은 UI에 표시되는 [이벤트 데이터를](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 보고하며, [쿼리 및 차트 작성이](/docs/using-new-relic/data/understand-data/query-new-relic-data) 가능합니다.

속성을 보려면 다음 표에서 이벤트 이름을 선택하십시오.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        이벤트
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`PageView`](/attribute-dictionary/?event=PageView)
      </td>

      <td>
        `PageView` 각 브라우저 페이지 로드에 대한 지리적 및 타이밍 데이터를 추적합니다.
      </td>
    </tr>

    <tr>
      <td>
        [`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming)
      </td>

      <td>
        `PageViewTiming` 사용 가능한 즉시 각 데이터 포인트를 별도의 이벤트로 보냅니다. 이는 비동기식 동적 페이지에 유용합니다.

        이 이벤트를 사용하려면 브라우저 에이전트 버전이 1149 이상이어야 합니다. 자세한 내용은 [`PageViewTiming` 쿼리 예](/docs/browser/new-relic-browser/page-load-timing-resources/pageviewtiming-async-or-dynamic-page-details) 를 참조하세요.
      </td>
    </tr>

    <tr>
      <td>
        [`PageAction`](/attribute-dictionary/?event=PageAction)
      </td>

      <td>
        `PageAction` 이벤트는 페이지에서 수행된 각 작업에 대한 데이터를 추적합니다. 해당 속성은 앱 및 지리적 데이터는 물론 사용자의 브라우저 크기, 세션 ID, 참조 및 페이지 URL에 대한 데이터를 제공합니다.
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserInteraction`](/attribute-dictionary/?event=BrowserInteraction)
      </td>

      <td>
        `BrowserInteraction` 여러 PageView 속성과 단일 페이지 앱(SPA)에 특정한 속성이 포함되어 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        [`AjaxRequest`](/attribute-dictionary/?attribute_name=&events_tids%5B%5D=8302&event=AjaxRequest)
      </td>

      <td>
        `AjaxRequest` 이벤트는 Ajax 호출이 수행될 때마다 생성됩니다. `BrowserInteraction` 이벤트 중에 Ajax 호출이 발생하면 브라우저 상호작용과 관련된 추가 속성으로 장식됩니다. 이벤트 속성은 지리적 및 브라우저 정보를 추적합니다.
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserTiming` (더 이상 사용되지 않음)](/attribute-dictionary/?event=BrowserTiming)
      </td>

      <td>
        `BrowserTiming` 사용자 정의 [createTracer](/docs/browser/new-relic-browser/browser-agent-spa-api/createtracer-browser-spa-api) SPA API 메소드를 사용하여 시작된 브라우저에 대한 SPA 타이밍 데이터를 캡처하는 사용자 정의 대시보드입니다. `BrowserTiming` 에는 다른 이벤트, 특히 `AjaxRequest` 에서 사용되는 것과 동일한 속성이 많이 포함되어 있습니다. *해당 방법은 더 이상 SPA v2+의 일부로 사용되지 않으므로 이 사용자 정의 대시보드와 해당 방법은 더 이상 사용되지 않으며 사용해서는 안 됩니다.*
      </td>
    </tr>

    <tr>
      <td>
        [`JavaScriptError`](/attribute-dictionary/?event=JavaScriptError)
      </td>

      <td>
        자바스크립트 오류가 발생하면 `JavaScriptError` 이벤트가 생성됩니다. `JavaScriptError` 은 자바스크립트 오류 데이터와 브라우저 상호작용 데이터를 캡처합니다. 자세한 내용은 브라우저 UI의 [JS 오류 페이지](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-examine-errors-over-time) 와 브라우저 및 SPA 이벤트에 대한 [예제 NRQL 쿼리](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-browser-single-page-app-data#javascriptError-example) 를 참조하세요.
      </td>
    </tr>

    <tr>
      <td>
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        `Span` 데이터는 [분산 추적](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing)에 대해 보고됩니다.
      </td>
    </tr>
  </tbody>
</table>

이 데이터가 보존되는 기간에 대한 자세한 내용은 [데이터 보존을](/docs/data-apis/manage-data/manage-data-retention)참조하십시오.
