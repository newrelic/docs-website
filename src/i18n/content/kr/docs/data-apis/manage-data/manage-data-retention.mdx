[데이터 관리 허브](https://one.newrelic.com/data-management-hub) 에 있는 **데이터 보존** UI 페이지에서 다양한 유형의 데이터가 저장되는 기간을 검토하고 관리할 수 있습니다. 민감한 정보를 보호하기 위해 일부 데이터 유형에 대해 더 짧은 보존 기간을 원할 수도 있고 더 오랜 기간 동안 데이터와 추세를 보고 분석할 수 있도록 더 긴 보존 기간을 원할 수도 있습니다.

## 장기 분석 및 규정 준수를 위해 데이터 보존 기간 연장

데이터 보존 기간을 연장하면 모든 소스에서 모든 지표, 이벤트, 로그 및 추적에 대한 장기 분석, 시각화 및 경고를 수행할 수 있습니다. 그러나 비용, 성능 및 경우에 따라 규정 준수를 위해 해당 데이터를 관리하는 것이 중요합니다. 당사의 데이터 관리 허브는 데이터의 출처를 이해 및 제어하고 저장 내용과 기간을 조정하는 데 필요한 도구를 제공합니다.

New Relic은 2023년 3월 31일부터 데이터 보존 제한을 적용하기 시작합니다. 귀하의 데이터 보존이 명시된 한도를 초과하면 해당하는 경우 기본 또는 계약된 데이터 보존으로 돌아가고 명시되거나 계약된 한도를 초과하는 데이터 보존은 제거됩니다. 영향을 받는 데이터에는 2023년 3월 31일 전후에 수집된 모든 데이터가 포함되며 대시보드, 데이터 분석, 감사 및 보고 요구 사항에 사용하기 위한 용도도 포함됩니다.

명시된 제한을 초과하여 데이터 보존을 조정하면 연장된 보존을 반영하기 위해 새로운 청구 변경 사항으로 보존을 업그레이드하라는 메시지가 [데이터 관리 허브](/docs/data-apis/manage-data/manage-your-data)에서 표시됩니다.

<Callout variant="tip">
  [Data Plus 옵션](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus) 을 사용하면 최대 90일의 추가 보존(대부분의 데이터 유형에 대해) 및 더 긴 쿼리 기간, FedRAMP 및 HIPAA 규정 준수, 추가 보안 기능 등과 같은 기타 엔터프라이즈급 기능을 얻을 수 있습니다.
</Callout>

## 요구 사항 [#requirements]

데이터 보존을 구성하려면 청구 관리자 권한이 있는 Pro 또는 Enterprise 버전이 필요합니다. 이 기준을 충족하지 않는 계정 특성은 일부 New Relic 조직이 자체적으로 보존을 조정하지 못하도록 차단합니다. 청구 관리자 계정의 Pro 또는 Enterprise 조직이고 보존을 편집할 수 없는 경우 New Relic 계정 담당자에게 문의하십시오.

[Data Plus 옵션](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus) 은 최대 90일의 추가 보존 기간을 제공합니다.

보존 보기 및 편집을 위한 권한 관련 요구 사항:

* [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) :

  * 전체 조직에 대해 더 긴 데이터 보존 계약을 설정하려면 [**결제** 관리 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) 이 필요합니다.
  * 특정 유형의 데이터 또는 특정 계정에 대한 계약 범위 내에서 보존을 조정하려면 [**Insights 이벤트 보존** 기능](/docs/accounts/accounts-billing/new-relic-one-user-management/user-capabilities/#data-retention) 이 필요합니다.

* [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) : **Insights 이벤트 보존** 기능이 필요합니다.

## 다양한 데이터 유형에 대한 보존 세부정보 [#data-details]

이 섹션에서는 변경할 수 없는 보존 설정이 있는 일부 데이터 유형을 포함하여 몇 가지 다른 유형의 데이터에 대한 세부 정보를 제공합니다.

<CollapserGroup>
  <Collapser
    id="dimensional-metrics"
    title="차원 측정항목(Metric API)"
  >
    원시 [차원 측정항목](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) 데이터( `Metric` 데이터 유형)는 몇 가지 다른 시간 범위에 대해 집계(롤업)되며 이러한 집계는 다른 기간 동안 유지됩니다. 원시 차원 메트릭 데이터 요소는 30일 동안 저장됩니다. 집계된 모든 데이터 포인트(1분에 집계 시작)는 13개월 동안 유지됩니다.

    원시 데이터 수집만 사용량 계산에 포함되며 집계된 데이터는 계산에 포함되지 않습니다.

    측정항목 보존 기간은 편집할 수 없습니다.
  </Collapser>

  <Collapser
    id="metric-timeslice"
    title="메트릭 타임슬라이스 데이터(APM, 브라우저, 모바일)"
  >
    [메트릭 타임슬라이스 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types#timeslice-data) 는 APM(사용자 지정 메트릭 및 기본 메트릭 모두), 모바일 및 브라우저 에이전트에 의해 생성됩니다. 메트릭 타임슬라이스는 메트릭 이름, 메트릭이 나타내는 시간 세그먼트("타임슬라이스") 및 숫자 값(측정)의 세 부분으로 구성됩니다. APM, 브라우저 및 모바일 에이전트는 오류율, 대역폭 사용량 및 가비지 수집 시간과 같은 다양한 성능 메트릭에 대해 분당 수천 개의 메트릭 타임슬라이스를 수집할 수 있습니다.

    메트릭 타임슬라이스 데이터는 다음과 같이 집계됩니다.

    * 8일 동안 저장된 처음 - 1분 롤업

    * 1분 롤업은 90일 동안 유지되는 1시간 롤업에도 추가됩니다.

      초기 원시 데이터 수집만 사용량 계산에 포함되며 집계된 데이터는 계산에 포함되지 않습니다.

      일부 주요 메트릭은 무기한 저장됩니다. 제품별 주요 측정항목을 알아보려면 계속 읽으십시오.

      다음은 무기한 저장되는 주요 메트릭 타임슬라이스 데이터입니다.

## 데이터 보존 편집 [#adjust-retention]

보존을 보고 편집하려면 [데이터 관리 허브](https://one.newrelic.com/data-management-hub) 로 이동하여 **데이터 보존** 을 클릭하고 데이터 **보존 편집** 을 클릭하십시오. 데이터 보존을 편집하거나 Data Plus로 업그레이드하기 위한 옵션이 표시됩니다. 표시되지 않으면 [요구 사항을 검토하고](#requirements) New Relic 계정 담당자에게 문의하십시오.

리텐션 증가 옵션:

* 아직 전환하지 않은 경우 훨씬 더 긴 데이터 보존 및 기타 이점을 제공하는 [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus) 로 전환하십시오.
* Data Plus 또는 원본 데이터 옵션의 경우: 데이터 보존을 추가합니다. 조직의 30일 추가 데이터 보존 비용은 매월 수집된 GB당 0.05 USD입니다.

리텐션 증가에 대한 중요 사항:

* [**데이터 보존** UI](#find-ui) 에 나열된 모든 데이터 유형에 대해 전반적으로 보존이 증가합니다. 특정 데이터 유형에 대해서만 늘릴 수 없습니다.
* 더 높은 보존 기간을 활성화하면 특정 데이터 소스에 대한 보존을 줄여도 청구에 영향을 미치지 않습니다. 데이터 수집 및 청구를 줄이는 방법에 대해 알아보려면 [데이터 수집 관리](/docs/data-apis/manage-data/manage-data-coming-new-relic) 를 참조하십시오.
* 보관을 조정하면 적용되는 데 최대 24시간이 걸릴 수 있습니다.

### 보존 감소 [#reduce-retention]

데이터 원본의 보존 기간을 줄이면 해당 유형의 기존 데이터가 즉시 사라지지 않습니다. 해당 데이터는 최대 몇 주 동안 조회에 표시되고 반환될 수 있습니다. 실수로 데이터를 삭제하지 않도록 보호 장치가 마련되어 있지만 이는 보존 값을 줄이는 것이 [데이터를 신속하게 제거](#deleting-data) 하는 좋은 전략이 아님을 의미하기도 합니다.

데이터 수집을 줄이는 옵션에 대해 알아보려면 [데이터 수집 관리](/docs/data-apis/manage-data/manage-data-coming-new-relic) 를 참조하십시오.

## 기존 데이터 삭제 [#deleting-data]

[원격 측정 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types) 가 New Relic 데이터베이스에 보고되고 쿼리에 사용할 수 있게 되면 해당 데이터를 편집하거나 삭제할 수 없습니다. 이것은 New Relic의 속도와 성능을 최적화하는 의도적인 설계 결정입니다. 데이터는 [데이터 보존 기간](#adjust-retention) 이 끝나면 만료됩니다. 이미 보고된 데이터를 삭제하려면 New Relic 담당자가 있는 경우 담당자에게 문의하십시오.

## API를 통한 보존 관리 [#api-examples]

UI에서 보존 설정을 변경하는 것 외에도 [NerdGraph API](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) 를 사용할 수 있습니다. 몇 가지 예시 NerdGraph 요청이 아래에 포함되어 있습니다. 데이터 보존 변경 사항이 적용되려면 최대 24시간이 걸릴 수 있습니다.

API를 사용하여 보존을 조정하기 전에 보존 [조정에 대한 규칙을 검토하십시오](#adjust-retention) .

<CollapserGroup>
  <Collapser
    id="api-list-namespaces"
    title="사용자 지정 가능한 보존 이벤트 네임스페이스 나열"
  >
    계정에 대한 사용자 정의 가능한 보존 이벤트 이름 공간을 나열하려면:

    ```graphql
    query {
     actor {
       account(id: YOUR_ACCOUNT_ID) {
         dataManagement {
           customizableRetention {
             eventNamespaces {
               namespace
             }
           }
         }
       }
     }
    }
    ```
  </Collapser>

  <Collapser
    id="api-list-rules"
    title="계정의 활성 규칙 나열"
  >
    계정의 활성 규칙을 나열하려면 다음을 수행합니다.

    ```graphql
    query {
     actor {
       account(id: YOUR_ACCOUNT_ID) {
         dataManagement {
           eventRetentionRules {
             id
             deletedAt
             deletedById
             createdAt
             createdById
             retentionInDays
             namespace
           }
         }
       }
     }
    }
    ```
  </Collapser>

  <Collapser
    id="api-show-active-rule"
    title="계정/네임스페이스 쌍에 대한 활성 규칙 표시"
  >
    특정 계정/네임스페이스 쌍에 대한 활성 규칙을 표시하려면:

    ```graphql
    query {
     actor {
       account(id: YOUR_ACCOUNT_ID) {
         dataManagement {
           eventRetentionRule(namespace: "APM") {
             id
             deletedById
             deletedAt
             createdById
             createdAt
             retentionInDays
             namespace
           }
         }
       }
     }
    }
    ```
  </Collapser>

  <Collapser
    id="api-create-new-rule"
    title="새 규칙 만들기"
  >
    새 규칙을 생성하려면:

    ```graphql
    mutation {
     dataManagementCreateEventRetentionRule(accountId: YOUR_ACCOUNT_ID, namespace: "APM", retentionInDays: 8) {
       id
       deletedById
       deletedAt
       createdById
       createdAt
       retentionInDays
       namespace
     }
    }
    ```
  </Collapser>

  <Collapser
    id="api-delete-rule"
    title="기존 규칙 삭제"
  >
    하위 계정에 적용된 규칙을 삭제하면 해당 하위 계정은 지정된 네임스페이스에 대한 상위 계정의 현재 보존 값을 적용합니다. 상위 계정에 적용된 규칙을 삭제하면 지정된 네임스페이스에 기본 보존 값이 다시 적용됩니다. 기존 규칙을 삭제하려면:

    ```graphql
    mutation {
     dataManagementDeleteEventRetentionRule(accountId: YOUR_ACCOUNT_ID, namespace: "APM") {
       id
       deletedById
       deletedAt
       createdById
       createdAt
       retentionInDays
       namespace
     }
    }
    ```
  </Collapser>

  <Collapser
    id="api-delete-rule"
    title="기존 하위 계정 규칙을 다른 사용자에게 복사"
  >
    하위 계정을 템플릿 또는 원본 계정으로 지정하고 다른 대상 하위 계정이 원본 하위 계정과 동일한 네임스페이스 보존 값을 채택하도록 할 수 있습니다. 참고: 모든 하위 계정은 동일한 상위 계정에 있어야 합니다. 기존 보존 세트를 복사하려면 다음을 수행합니다.

    ```graphql
    mutation {
      dataManagementCopyRetentions(
        sourceAccountId: SOURCE_ACCOUNT_ID,
        destinationAccountIds: [DESTINATION_ACCOUNT_ID_1, DESTINATION_ACCOUNT_ID_2, DESTINATION_ACCOUNT_ID_3])
      {
        success
        failure
      }
    }
    ```

    성공 및 실패는 원본 계정에서 규칙을 복사하는 데 성공한(또는 실패한) 대상 계정 ID 목록을 반환합니다.
  </Collapser>
</CollapserGroup>
