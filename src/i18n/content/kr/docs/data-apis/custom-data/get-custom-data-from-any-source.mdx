New Relic을 최대한 활용하려면 맞춤 데이터가 필요하거나 보고해야 할 수 있습니다. 모든 유형의 데이터를 New Relic으로 가져오는 데 필요한 도구를 제공합니다.

## 먼저 더 선별된 솔루션을 확인하세요. [#options]

New Relic 제품은 많은 데이터를 자동으로 보고하므로 추가 구성이 필요하지 않습니다!APM, 브라우저 모니터링, 모바일 모니터링, 인프라 모니터링 또는 다양한 [통합 및 빠른 시작과](https://newrelic.com/instant-observability) 같은 모니터링 솔루션을 사용하면 기본적으로 다양한 유형의 성능 데이터를 수신합니다.

그러나 기본적으로 수집되지 않는 데이터를 New Relic으로 가져올 수 있습니다. 에이전트 설치가 필요 없는 API 기반 솔루션을 원할 수도 있습니다. 다른 분석 서비스의 원격 측정 데이터를 New Relic으로 가져오고 싶을 수도 있습니다. 아래에서 New Relic에 더 많은 사용자 지정 데이터 집합을 가져오는 몇 가지 방법을 설명합니다.

## 에이전트 API로 사용자 지정 데이터 보고 [#agent-apis]

New Relic 에이전트(APM, 브라우저 또는 모바일)를 사용하는 경우 사용자 지정 데이터를 보내도록 구성할 수 있습니다. 자세히 알아보려면 사용 중인 에이전트에 대한 문서를 참조하세요.

## 맞춤형 인프라 데이터 보고 [#flex-integration]

인프라 통합이 귀하의 요구 사항을 충족하지 못하는 경우 [Flex 통합](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) 을 사용하여 모든 유형의 인프라 관련 메트릭 또는 구성 설정을 New Relic에 보고할 수 있습니다.

중대형 기업 조직이고 [SAP®](https://www.sap.com/) 기술을 사용하여 비즈니스 운영 및 고객 관계를 관리하려는 경우 [SAP 통합](/docs/data-apis/custom-data/sap-integration) 을 사용할 수 있습니다.

## 오픈 소스 원격 측정 서비스 및 사양의 데이터 보고 [#third-party-telemetry]

[빠른 시작 페이지](https://newrelic.com/instant-observability) 에서 모든 통합을 검토할 수 있습니다. 다음은 타사, 오픈 소스 원격 측정 서비스 및 사양과의 가장 인기 있는 통합 중 일부입니다.

## 맞춤 속성 및 이벤트 보고 [#events]

New Relic이 보고한 일부 이벤트를 사용자 지정 메타데이터로 장식할 수 있습니다. 이에 대한 자세한 내용은 [사용자 정의 속성](/docs/data-apis/custom-data/custom-events/collect-custom-attributes) 을 참조하십시오.

[Event API](/docs/data-apis/ingest-apis/event-api/introduction-event-api) 를 사용하여 완전히 맞춤화된 이벤트를 New Relic에 보고할 수도 있습니다.

그런 다음 사용자 정의 속성 및 이벤트를 쿼리하고 차트로 작성할 수 있습니다.

## 맞춤 추적 데이터 보고 [#tracing]

우리의 [추적 API를](/docs/distributed-tracing/trace-api/introduction-trace-api) 사용하면 분산 추적 데이터를 New Relic으로 보내고 여러 소스의 추적 데이터를 한 곳에서 통합할 수 있습니다. 추적 데이터는 두 가지 형식으로 허용됩니다.

* 지퍼 형식
* New Relic 형식(Zipkin 형식 데이터가 없으면 이것을 사용)

## 맞춤 로그 데이터 보고 [#logs]

기존 [로깅 통합](/docs/logs/get-started/get-started-log-management) 이 요구 사항을 충족하지 않는 경우 [Log API](/docs/logs/log-api/introduction-log-api) 를 사용하여 임의의 로그 데이터를 New Relic에 보낼 수 있습니다.

## 맞춤형 데이터 시각화 생성 [#visualizations]

사용자 지정 쿼리, 차트 및 대시보드 구축에 관심이 있으십니까? 쿼리를 시작하려면 [NRQL 소개를](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) 참조하세요. 쿼리를 만들 때 결과 차트를 가져와 [대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) 에 추가할 수 있습니다.

차트보다 더 많은 기능과 사용자 정의가 필요한 경우 <InlinePopover type="dashboards"/>허용하면 나만의 [New Relic 앱을](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application)만들거나 [앱 카탈로그](https://opensource.newrelic.com/nerdpacks)에서 기존 앱을 사용자 지정할 수도 있습니다. New Relic 앱은 플랫폼에 상주하며 팀이나 대중과 공유할 수 있습니다.
