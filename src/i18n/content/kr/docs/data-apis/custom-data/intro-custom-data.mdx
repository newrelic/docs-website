New Relic을 최대한 활용하려면 환경과 조직이 직면한 고유한 문제에 대한 통찰력을 얻는 데 도움이 되는 사용자 지정 데이터를 보고해야 하거나 보고해야 할 수 있습니다.

## 사용자 지정 데이터에 대한 옵션 [#options]

다양한 유형의 데이터를 가져오는 다양한 [에이전트와 빠른 시작](https://newrelic.com/instant-observability) 이 있습니다. 그러나 기본적으로 보고되지 않는 데이터를 가져와야 할 수도 있습니다.

사용자 지정 데이터 보고 옵션은 다음과 같습니다.

* [OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/introduction-opentelemetry-new-relic) , [Prometheus](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic) , [Grafana](/docs/more-integrations/grafana-integrations/get-started/grafana-support-prometheus-promql) ,[DropWizard](/docs/more-integrations/open-source-telemetry-integrations/dropwizard/dropwizard-reporter) , [Micrometer](/docs/more-integrations/open-source-telemetry-integrations/micrometer/micrometer-metrics-registry) 를 포함한 타사 원격 측정 서비스 및 사양에 대한 통합. [모든 솔루션을 찾아보십시오.](https://newrelic.com/instant-observability)
* 사용자 지정 데이터를 보내도록 에이전트를 구성합니다. APM 에이전트, 브라우저 에이전트 또는 모바일 에이전트 중 하나를 사용하는 경우 사용자 지정 데이터를 보내도록 구성할 수 있습니다. 자세히 알아보려면 사용 중인 도구에 대한 문서를 참조하세요.
* 언어별 [Telemetry SDK](/docs/telemetry-data-platform/ingest-manage-data/ingest-apis/telemetry-sdks-report-custom-telemetry-data) 를 사용하여 원격 측정 데이터 보고를 위한 고유한 솔루션을 만드십시오. 또는 기본 API인 [Metric API](/docs/telemetry-data-platform/ingest-manage-data/ingest-apis/introduction-metric-api) , [Trace API](/docs/distributed-tracing/trace-api/introduction-trace-api) , [Event API](/docs/telemetry-data-platform/ingest-manage-data/ingest-apis/introduction-event-api) , [Log API](/docs/logs/log-management/log-api/introduction-log-api) 를 직접 사용하십시오.
* 운영 체제 또는 네트워크의 데이터를 보고합니다. 인프라 에이전트를 사용하는 경우 간단한 구성 파일을 사용하여 데이터를 보고할 수 있는 [Flex 통합](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) 을 사용할 수 있습니다.
* [New Relic 앱](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application) 을 빌드하거나 [앱 카탈로그](https://opensource.newrelic.com/nerdpacks) 에서 기존 앱을 사용자 지정합니다. 우리 플랫폼에 상주하고 팀이나 대중과 공유할 수 있고 고유한 JavaScript UI 기능을 사용하는 앱을 빌드할 수 있습니다. New Relic 데이터를 분석하거나 원하는 데이터를 가져올 수 있습니다.
