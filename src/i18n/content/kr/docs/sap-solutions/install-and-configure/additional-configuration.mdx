각 모니터 시스템에 대해 이 설정 작업을 완료하십시오.

## 업로드 권한 역할 [#upload-authorization-role]

1. <DNT>**SAP transaction**</DNT> `PFCG`을(를) 사용합니다.

2. <DNT>**Role**</DNT> &gt; <DNT>**Upload**</DNT> 로 이동하여 설치 패키지에서 <DNT>**\_NEWR\_DATA\_PROVIDER.SAP**</DNT> 파일을 업로드합니다.

3. `/NEWR/DATA_PROVIDER` 역할이 로드되는지 확인합니다.

   <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

4. <img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP">를 선택합니다.</img>

   역할이 로드된 후 아이콘.

5. <DNT>**Authorizations**</DNT> 탭에서 다음을 선택하세요.<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   <DNT>**Change Authorization Data**</DNT>를 수행한 다음, 다음을 사용하여 활성화합니다.<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />아이콘입니다.

   <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

6. 프로필 이름이 제안되면 녹색 체크 아이콘을 선택합니다.

7. 상태 표시줄에 <DNT>**Profile(s) were updated**</DNT> 이(가) 나타나는지 확인합니다.

## 뉴렐릭 RFC 사용자 ID 생성 [#create-rfc-user-id]

<DNT>**authorization profile**</DNT>을(를) 활성화한 후 트랜잭션 `SU01` 을(를) 사용하여 새 <DNT>**RFC user ID**</DNT> (예: `NR4SAPDP`)을(를) 생성합니다. <DNT>**user type**</DNT> 을(를) <DNT>**System**</DNT> 또는 <DNT>**Communication**</DNT> (으)로 설정하고 역할 `/NEWR/DATA_PROVIDER`을(를) 추가합니다.

<img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />

## 모니터링되는 시스템에 대한 RFC 대상 생성 [#create-rfc-for-monitored-systems]

각 모니터 시스템에 대해 트랜잭션 `SM59` 을(를) 사용하여 <DNT>**RFC destination**</DNT> 을(를) 생성합니다.

1. <DNT>**ABAP Connections**</DNT> 를 선택한 다음 <DNT>**Create**</DNT> 아이콘을 클릭하여 새로운 <DNT>**RFC destination**</DNT>를 설정합니다.

2. 기술 <DNT>**RFC Destination**</DNT> 이름을 입력합니다. 다음을 권장합니다: `NR_<SYSID>CLNT<CLNT#>`

3. <DNT>**Connection Type**</DNT> 을 `3 - ABAP Connection` 로 설정합니다.

4. <DNT>**Technical Settings**</DNT> 탭에서 다음을 입력합니다:

   * <DNT>**Target Host**</DNT> (정규화된 도메인 또는 IP) 및 <DNT>**Instance No.**</DNT>
   * <DNT>**Gateway Host**</DNT> 그리고 필요한 경우 <DNT>**Service**</DNT>.

   <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

5. <DNT>**Logon &amp; Security**</DNT> 탭에서 <DNT>**RFC user**</DNT> (`NR4SAPDP`)와 비밀번호를 입력합니다.

6. <DNT>**Save**</DNT> 선택합니다.

7. <DNT>**Utility &gt; Test &gt; Connection Test**</DNT> 및 <DNT>**Authorization Test**</DNT>로 이동합니다.

   * <DNT>**Expected result**</DNT>: 두 테스트 모두 성공 상태를 보여줍니다.

## HTTP 대상을 뉴렐릭 cloud로 설정

트랜잭션 `SM59` 을 사용하여 <DNT>**HTTPS connections**</DNT> 을 <DNT>**New Relic API endpoints**</DNT>에 설정합니다.

<Callout variant="important">
  <DNT>**Events**</DNT>, <DNT>**Logs**</DNT>, <DNT>**Metrics**</DNT>, <DNT>**Traces**</DNT> 에 대해 개별 <DNT>**RFC destinations**</DNT> 을(를) 각각 생성하십시오.
</Callout>

1. <DNT>**Destination Name**</DNT> 을(를) 입력하세요(예: `NR_API_EVENT`).

2. <DNT>**Connection Type**</DNT> 을(를) `G` (외부 서버에 대한 HTTP 연결)으로 설정합니다.

3. <DNT>**Target Host**</DNT>, <DNT>**Service No.**</DNT> 을(를) <DNT>**443**</DNT> (으)로 입력하고(<DNT>**HTTPS**</DNT>의 경우 [표준 포트](/docs/sap-solutions/reference/network-accessibility) 참조), <DNT>**Path Prefix**</DNT> 를 입력합니다(자세한 내용은 [설치 준비 체크리스트](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness) 참조).

4. 프록시가 필요한 경우 <DNT>**Technical Settings**</DNT> 탭에서 구성하세요.

5. <DNT>**Security Options**</DNT> 탭에서 <DNT>**SSL**</DNT>에 대한 <DNT>**Active**</DNT> 옵션을 선택합니다.

   <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />

6. 필요한 경우 [뉴렐릭](https://one.newrelic.com) 에서 <DNT>**root certificate**</DNT> 을(를) 다운로드하고 트랜잭션 `STRUST`을(를) 사용하여 <DNT>**SAP**</DNT> 에 추가합니다.

7. <DNT>**Connection Test**</DNT> 을(를) 선택하여 연결을 확인합니다.

   * <DNT>**Expected result**</DNT>: 연결에 성공하면 로그온 데이터를 입력하라는 프롬프트가 표시됩니다.
   * <DNT>**If test times out**</DNT>: 방화벽 설정을 확인하세요.

8. 모든 <DNT>**New Relic API endpoints**</DNT>에 대해 이 절차를 반복합니다.

## 데이터베이스 연결 생성 [#create-database-connection]

트랜잭션 `DBACOCKPIT` 을 사용하여 원격 <DNT>**HANA Database connections**</DNT>을(를) 정의합니다.

1. <DNT>**Database Connections**</DNT> 을 두 번 클릭하고 <DNT>**Add**</DNT>를 선택합니다.

   <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

2. <DNT>**Connection Name**</DNT> 를 입력하세요(<DNT>**SAP system ID**</DNT>와 동일한 것이 좋습니다).

3. <DNT>**Database System**</DNT> 로 <DNT>**HANA Database**</DNT> 선택합니다.

4. <DNT>**Database user ID**</DNT> (예: `NRAGENT`)와 비밀번호를 입력합니다.

5. <DNT>**Database Host**</DNT> 및 <DNT>**SQL Port**</DNT>을(를) 입력하세요.

   <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

6. <DNT>**Save**</DNT>를 선택하고, <DNT>**Test**</DNT> 를 선택하여 연결을 확인합니다.

   * <DNT>**Expected result**</DNT>: 연결 테스트가 성공 상태를 표시합니다.
