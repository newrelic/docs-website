**NRM4SAP** 의 ABAP 구성 요소는 SAP 추가 기능으로 패키징되어 있으며 **클라이언트 000** 에서 트랜잭션 **SAINT를** 사용하여 설치할 수 있습니다.

## 애드온 설치 파일 [#add-on-installation-file]

* **초기 설치**: 로컬 PC의 폴더에 애드온 파일 `K-xxxAOIINNEWR.SAR` 다운로드하세요.
* **이전 버전에서 업그레이드**: `K-xxxAOUINNEWR.SAP` 이라는 파일을 사용하세요.

## SAP 추가 기능 파일 설치 [#install-sap-add-on-file]

1. 설정 및 시스템 관리 작업을 위한 기본 클라이언트인 **클라이언트 000** 에 로그인합니다.
2. SAP GUI 명령 필드에 `SAINT` 입력하고 **Enter를** 클릭합니다.
3. **Installation Package** &gt; **Load Packages** 로 이동합니다.
4. 다음 방법 중 하나를 선택하세요.

<CollapserGroup>
  <Collapser id="sar-archive-from-frontend" title="프런트엔드의 SAR 아카이브">
    1. 압축을 푼 설치 파일을 찾아서 엽니다.
    2. **압축 파일 `K-xxxAOIINNEWR.SAR`의 내용** 팝업에서 **압축 해제를** 클릭합니다.

    <img title="Content of the compressed file" alt="A screenshot displaying the Content of the compressed file" src="/images/content-of-compressed-file-1.webp" width="" />

    3. **파일 K-xxxAOIINNEWR.SAR이 성공적으로 전송되고 압축 해제되었으며 삭제되었습니다라는** 메시지가 나타납니다.
  </Collapser>
</CollapserGroup>

<CollapserGroup>
  <Collapser id="sar-archive-from-application-server" title="애플리케이션 서버의 SAR 아카이브">
    1. 애플리케이션 서버(`/usr/sap/trans/download`)의 trans 디렉토리 아래에 다운로드 디렉토리가 있는지 확인하세요.
    2. 추가 기능 파일을 이 디렉토리에 복사합니다.
  </Collapser>
</CollapserGroup>

<CollapserGroup>
  <Collapser id="eps-files-from-application-server" title="애플리케이션 서버의 EPS 파일">
    1. 먼저 `SAR` 파일을 `PAT` 파일로 압축을 풉니다. OS(이 경우 Linux)에서 아래 명령을 실행합니다.

    ```sql
    cd /usr/sap/trans
    cp /tmp/K-300AOIINNEWR.SAR /usr/sap/trans
    SAPCAR -xvf K-300AOIINNEWR.SAR
    ```

    2. 팝업에서 **EPS 받은 편지함에서 OCS 패키지를 업로드하시겠습니까? 에서** **예를** 선택합니다. OCS 파일이 오류 없이 업로드되었는지 확인하세요.
  </Collapser>
</CollapserGroup>

5. **Start** \[시작을] 클릭하세요.

6. **시스템에 설치할 수 있는 추가 기능 패키지를** 선택하고 **계속을** 클릭합니다.

7. **대기열에 수정 조정 운송을 추가하시겠습니까?라는** 메시지가 나타나면 **아니요를** 클릭합니다.

   <img title="Do you want to add Modification Adjustment Transports to the queue?, Click No" alt="A screenshot displaying question, Do you want to add Modification Adjustment Transports to the queue?" src="/images/modify-adjustement-transports.webp" width="700" />

<Callout variant="important">
  조정 운송이 지침과 함께 포장 목록에 제공된 경우 **예를** 클릭합니다.
</Callout>

8. 클릭<img src="/images/green-tick-button.webp" class="inline" style={{height: "25px", width: "30px"}} alt="green check button" title="Green check button" />다음 화면으로 넘어가려면 버튼을 클릭하세요.

<Callout variant="important">
  **OCS 패키지 분해 중 오류** 페이지가 나타나면 **무시를** 클릭합니다.
</Callout>

9. 패키지가 설치되면 마침 **을** 클릭합니다. **런타임 분석 전송** 팝업이 나타납니다. 다음을 선택할 수 있습니다.

   * **보내기**: 로컬 브라우저 창에서 런타임 분석 데이터가 포함된 피드백 양식을 엽니다. 이 양식을 사용하여 SAP로 데이터를 보낼 수 있습니다.
   * **표시**: 런타임 분석을 먼저 표시한 다음 피드백 양식을 로컬 브라우저 창으로 보냅니다.
   * **보내지 않음**: 런타임 분석 데이터를 전송하지 않고 대화 상자를 닫습니다.

10. **설치된 구성 요소** 탭에서 설치가 성공했는지 확인하세요.

11. **클라이언트 000** 에서 로그아웃합니다.

<Callout variant="tip">
  SAP GUI 명령 필드에 **/nex를** 입력하면 시스템에서 로그아웃할 수 있습니다.
</Callout>
