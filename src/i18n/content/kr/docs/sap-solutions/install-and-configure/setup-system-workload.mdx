## 시스템 워크로드 설정

1. <DNT>**All entities &gt; Workloads &gt; + Create a workload**</DNT>으로 이동합니다. 자세한 내용은 [워크로드 생성 및 정의](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads/)를 참조하십시오.

2. <DNT>**workload**</DNT> 의 이름을 입력하고 드롭다운 메뉴 목록에서 계정을 선택합니다.

3. <DNT>**Entity Type = All**</DNT> 옆에 있는 <DNT>**+**</DNT> 아이콘을 선택하고 필터 표시줄에 `sap.system = {SAP System SID}` 를 입력합니다.

   <img title="SAP System" alt="A screenshot displaying the SAP System filter" src="/images/sap-system.webp" />

4. <DNT>**workload definition**</DNT>에 쿼리를 추가하려면 <DNT>**+ Add this query**</DNT> 를 선택합니다.

5. <DNT>**Create a workload**</DNT> 선택합니다.

새로운 <DNT>**workload**</DNT> 이(가) <DNT>**All entities**</DNT> 에 표시되며 <DNT>**SAP system**</DNT>(으)로 태그된 모든 엔티티를 포함합니다.

## 서비스 팩 설치

<DNT>**Service packs**</DNT> `K-<Version Number>SP<SP Number>IN<Software Component>.SAR` 명명 규칙을 사용하는 `.SAR` 파일로 전달됩니다.

1. 트랜잭션 `SPAM`을(를) 사용하여 <DNT>**service pack**</DNT> 을(를) 설치합니다.
2. <DNT>**client 000**</DNT>에서 새 <DNT>**configuration template**</DNT> 을(를) 로드합니다.
3. <DNT>**Client 000**</DNT>에서 <DNT>**Install New Release and Patch**</DNT> &gt; <DNT>**Copy Template Tables**</DNT> (으)로 이동합니다.
4. <DNT>**Install New Release and Patch**</DNT> &gt; <DNT>**Generate Configuration for New Data Providers**</DNT>의 단계에 따라 새로운 <DNT>**data provider configurations**</DNT> 을(를) 생성합니다.
5. <DNT>**Install New Release and Patch**</DNT> &gt; <DNT>**Verify Installation Completeness**</DNT> 선택하여 설치를 확인합니다.
