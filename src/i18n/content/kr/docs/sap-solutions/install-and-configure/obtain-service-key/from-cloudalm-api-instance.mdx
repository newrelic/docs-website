이 가이드는 SAP Cloud 애플리케이션 수명주기 관리(CALM) API 인스턴스에서 서비스 키를 확보하는 방법을 보여줍니다. SAP Cloud ALM 모니터링을 구성하려면 이 서비스 키가 필요합니다.

## 전제 조건

* 적절한 권한이 있는 <DNT>**CALM subaccount**</DNT> 에 대한 액세스
* 다음에 대한 귀하의 <DNT>**BTP (Business Technology Platform) administrator**</DNT> 의 확인 <DNT>**Space**</DNT>
* [Managing Scopes](https://help.sap.com/docs/cloud-alm/apis/managing-scopes)에서 식별된 필수 범위

1. <DNT>**CALM (Cloud Application Lifecycle Management) subaccount**</DNT>에 로그인한 다음 <DNT>**Instances and Subscriptions**</DNT>을 클릭합니다.

2. 오른쪽 상단의 <DNT>**Create**</DNT> 를 클릭하세요.

3. 팝업 화면에서 다음을 선택하세요.

   * <DNT>**Service**</DNT>: <DNT>**SAP Cloud ALM API**</DNT>
   * <DNT>**Plan**</DNT>: <DNT>**Standard**</DNT>
   * <DNT>**Runtime Environment**</DNT>: <DNT>**Cloud Foundry**</DNT>
   * <DNT>**Space**</DNT>: 귀하의 담당자에게 확인하세요 <DNT>**BTP administrator**</DNT>
   * <DNT>**Instance Name**</DNT>: 조직 명명 규칙을 따릅니다

   <img title="SAP Cloud ALM API" alt="The Create Instance dialog showing Service, Plan, Runtime Environment, Space, and Instance Name fields for SAP Cloud ALM API" src="/images/sap-cloud-alm-api.webp" width="600" />

4. <DNT>**Next**</DNT> 을(를) 클릭합니다.

5. 교정 탭 아래의 교정 이름과 함께 [범위 관리](https://help.sap.com/docs/cloud-alm/apis/managing-scopes) 에서 필요한 범위를 검토하고 삽입하세요.

   <img title="Manage Scopes" alt="The parameters tab showing the required scopes configuration for the CALM instance" src="/images/manage-scope.webp" width="600" />

6. <DNT>**Next**</DNT> 을(를) 클릭하여 세부 정보를 검토하세요.

   <img title="Review Details" alt="The review screen showing the summary of the SAP Cloud ALM API instance configuration before creation" src="/images/cloud-alm-new-instance.webp" width="600" />

7. <DNT>**Create**</DNT> 을(를) 클릭합니다.

8. 서비스를 만든 후 <DNT>**Instances**</DNT> 탭에서 해당 서비스를 찾으세요.

9. 새로 생성된 인스턴스의 행에 있는 줄임표(`...`)를 클릭한 다음 <DNT>**Create Service Key**</DNT>을(를) 선택합니다.

10. <DNT>**Service Key Name**</DNT> 를 입력하고 <DNT>**Key Type**</DNT>: `ClientId/Secret`를 선택합니다.

11. <DNT>**Create**</DNT> 을(를) 클릭합니다.

12. <DNT>**service key**</DNT> 행에 있는 줄임표(`...`)를 클릭한 다음 <DNT>**Download**</DNT>을(를) 선택합니다.

<DNT>**service key file**</DNT> 은(는) <DNT>**JSON file**</DNT>(으)로 다운로드됩니다. 안전하게 보관하십시오. 모니터링 설정을 구성하려면 이 파일의 자격 증명이 필요합니다.
