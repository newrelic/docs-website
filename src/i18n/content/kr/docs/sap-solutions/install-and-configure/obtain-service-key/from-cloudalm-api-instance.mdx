이 가이드는 SAP Cloud 애플리케이션 수명주기 관리(CALM) API 인스턴스에서 서비스 키를 확보하는 방법을 보여줍니다. SAP Cloud ALM 모니터링을 구성하려면 이 서비스 키가 필요합니다.

## 전제 조건

* 적절한 권한이 있는 CALM 하위 계정에 대한 액세스
* Space에 대한 BTP(비즈니스 테크놀로지 플랫폼) 관리자의 확인
* [Managing Scopes](https://help.sap.com/docs/cloud-alm/apis/managing-scopes)에서 식별된 필수 범위

1. **CALM (Cloud Application Lifecycle Management) subaccount** \[CALM(클라우드 애플리케이션 수명 주기 관리) 하위 계정]에 로그인한 다음 **Instances and Subscriptions** \[인스턴스 및 구독]을 클릭합니다.

2. 오른쪽 상단의 **Create**\[만들기]를 클릭하세요.

3. 팝업 화면에서 다음을 선택하세요.

   * **서비스**: SAP Cloud ALM API
   * **플랜**: 표준
   * **런타임 환경**: Cloud Foundry
   * **공간**: BTP 관리자에게 문의하세요
   * **인스턴스 이름**: 조직의 명명 규칙을 따르세요.

   <img title="SAP Cloud ALM API" alt="The Create Instance dialog showing Service, Plan, Runtime Environment, Space, and Instance Name fields for SAP Cloud ALM API" src="/images/sap-cloud-alm-api.webp" width="600" />

4. **Next**클릭합니다.

5. 교정 탭 아래의 교정 이름과 함께 [범위 관리](https://help.sap.com/docs/cloud-alm/apis/managing-scopes) 에서 필요한 범위를 검토하고 삽입하세요.

   <img title="Manage Scopes" alt="The parameters tab showing the required scopes configuration for the CALM instance" src="/images/manage-scope.webp" width="600" />

6. **Next** \[다음을] 클릭하여 세부 정보를 검토하세요.

   <img title="Review Details" alt="The review screen showing the summary of the SAP Cloud ALM API instance configuration before creation" src="/images/cloud-alm-new-instance.webp" width="600" />

7. **Create** \[만들기 를] 클릭합니다.

8. 서비스를 만든 후 **Instances** \[인스턴스] 탭에서 해당 서비스를 찾으세요.

9. 새로 생성된 인스턴스의 행에 있는 줄임표(`...`)를 클릭한 다음 **Create Service Key** \[서비스 키 만들기]를 선택합니다.

10. **Service Key Name** \[서비스 키 이름]을 입력하고 **Key Type** \[키 유형]을 선택합니다: `ClientId/Secret`.

11. **Create** \[만들기 를] 클릭합니다.

12. 서비스 키 행에 있는 줄임표(`...`)를 클릭한 다음 **Download** \[다운로드]를 선택합니다.

서비스 키 파일이 JSON 파일로 다운로드됩니다. 안전하게 보관하십시오. 모니터링 설정을 구성하려면 이 파일의 자격 증명이 필요합니다.
