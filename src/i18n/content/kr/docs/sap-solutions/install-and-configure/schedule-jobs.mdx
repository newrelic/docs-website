## RFC 검사 프로그램 일정 [#rfc-check]

`/NEWR/REGISTRY_RFC_CHECK` 1～3분마다 예약하여 데이터 수집 작업보다 더 자주 실행되도록 합니다. 작업 이름 `NR_RFC_CHECK` 을 사용하세요.

## ABAP 데이터 수집 관리자 일정 [#abap-collection-manager]

다음 데이터 제공자 그룹에 초점을 맞춘 변형을 사용하여 세 가지 데이터 수집 작업을 예약합니다. `SP03` 이전에 컬렉션 관리자 작업을 예약하는 방법에 대한 지침은 [SP03 이전의 변형 및 작업을](/docs/sap-solutions/additional-resources/variants-and-jobs-before-sp03) 참조하세요. 일정을 예약하기 위해 데이터 제공자를 세 그룹으로 나눕니다.

* **트레이스**: 이 그룹에는 성능 데이터베이스(STAD) 관련 데이터 공급자가 있습니다.

  * `NR.SAP.TRANSACTION` (트레이스 데이터 공급자), `NR.SAP.TRANSACTION.COMPLETE` (병렬 배경 RFC 프로세스를 갖춘 완전한 성능 데이터베이스 컬렉션), `NR.SAP.SYSTEM.PROCESS`, `NR_SAP_PROGRAM`, `NR_SAP_WORKLOAD_COMPLETE` 및 `NR.SAP.WORKLOAD.COMPLETE` 와 같은 데이터베이스 성능 관련 공급자를 포함합니다.
  * `NR.SAP.SYSTEM.PROCESS` `SP02` 이후에는 `NR_SAP_PROGRAM` 일시 중단됩니다.

* **IDOC**: 프로덕션 시스템에서 IDOC 수집에는 상당한 시간이 걸릴 수 있습니다.
  * `NR_SAP_IDOC` (주요 IDOC 상태 이벤트 데이터 공급자), `NR_SAP_IDOC_COMPLETE` (BG RFC 수집을 통한 완전한 IDOC 검색), `NR_SAP_IDOC_Configuration` (IDOC 데이터 공급자 성능 개선을 위한 IDOC 마스터 데이터 다운로드)와 같은 IDOC 관련 공급자가 포함됩니다.

* **시스템**: 나머지 데이터 제공자.

## 일정 재제출 프로그램 [#resubmit-program]

1. 전송 오류가 발생한 경우 뉴렐릭 계정에 대해 실패한 전송을 다시 제출하기 위해 프로그램 `/NEWR/RESEND_FAILED_API_TRANS` 에 대해 **DEFAULT** 라는 이름의 변형을 만듭니다.

2. 다음 값을 지정하세요.

   * [**New Relic 계정 아이디**](/docs/accounts/accounts-billing/account-setup/account-id)
   * **HTTP 응답 코드**: `000` 및 `413`, 그리고 `200` ～ `299` 범위를 제외합니다.

   <img title="Resend failed API transactions" alt="A screenshot displaying the resend failed API transactions" src="/images/resend-failed-api-transaction.webp" width="700" />

3. HTTP 응답 코드 `500` 등의 실패한 전송을 다시 제출하고 전송 로그 항목을 주기적으로 삭제하기 위해 15분마다 실행되도록 이 작업을 예약합니다.
