이 가이드에서는 뉴렐릭 모니터링 for SAP Solutions의 데이터 수집 프로세스를 모니터링하는 방법을 보여줍니다. 여러 방법을 사용하여 데이터 수집이 올바르게 작동하는지 확인할 수 있습니다.

## 전제 조건

* 적절한 트랜잭션 권한이 있는 SAP 시스템 액세스
* 뉴렐릭 모니터링 for SAP Solutions 설치 및 구성됨

## 런타임 상태 및 전송 로그

**Data Provider Registry** \[데이터 공급자 레지스트리] (트랜잭션 `/NEWR/REGISTRY`)에서 각 데이터 공급자의 **Runtime Status** \[런타임 상태] 및 **Transmission Log** \[전송 로그]를 확인할 수 있습니다. 전송 로그는 각 전송의 상태와 시간을 보여줍니다.

## 작업 상태 및 스풀 확인

1. 트랜잭션 `SM37` 을 사용하여 예약된 작업의 상태를 확인합니다. **ABAP Program Name** \[ABAP 프로그램 이름] 필드에 `/NEWR/*` 입력하고 **Execute** \[실행을] 클릭합니다.
2. 실행 기간을 검토하십시오. 비정상적으로 긴 실행 시간은 잠재적인 문제가 있음을 나타냅니다.
3. 겹치는 작업이 있는지 확인하고 필요에 따라 일정을 조정합니다. 작업 세부 정보에서 작업 스풀에 액세스하여 각 데이터 공급자의 결과를 확인하십시오.

## 애플리케이션 로그 확인

데이터 수집 프로세스는 애플리케이션 로그(트랜잭션 `SLG1`)에 기록됩니다. 애플리케이션 로그 객체 `/NEWR/DP`을(를) 사용하여 쿼리합니다.

## SAP 데이터 수집 모니터링 대시보드

데이터 수집 메트릭은 자체 모니터링을 위해 뉴렐릭에도 전송됩니다. 기본 제공되는 **SAP Data Collection Monitoring** \[SAP 데이터 수집 모니터링] 대시보드를 사용하여 각 데이터 공급자에 대한 데이터 수집 프로세스를 모니터링합니다.
