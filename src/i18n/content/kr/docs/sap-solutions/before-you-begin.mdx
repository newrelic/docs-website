설치 전에 다음 전제 조건을 완료하십시오:

## 전제 조건 체크리스트

### New Relic 계정

* **계정 ID**: [계정 ID 확인하기](/docs/accounts/accounts-billing/account-setup/account-id)
* **라이선스 키**: 인증 및 데이터 액세스를 위해 뉴렐릭 <InlinePopover type="licenseKey" />을(를) 가져옵니다.
* **데이터센터 지역**: [데이터센터 위치(미국 또는 EU)를 선택](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) 하고 계정을 구성합니다

### SAP 시스템 요구사항

* **소프트웨어 버전**: SAP 시스템이 필수 소프트웨어 구성 요소 릴리스 수준을 충족하는지 확인합니다. 시스템 특정 요구 사항은 [데이터 공급자 참조](/docs/sap-solutions/monitor/data-providers-reference) 를 참조하십시오
* **모니터 시스템**: SAP\_BASIS 740+, Kernel 753 Patch 602+, ST-PI 740 SP16+
* **Central monitoring system** \[중앙 모니터링 시스템]: SAP\_BASIS 740+, ST-PI 740 SP14+

### 네트워크 액세스

* 뉴렐릭 엔드포인트에 대한 HTTPS 액세스([엔드포인트 URL](#network-endpoints) 참조)
* 방화벽 규칙 구성됨
* 프록시 설정 구성됨(필요한 경우)

### BTP 모니터링(해당하는 경우)

BTP 계정을 모니터링하는 경우 다음 중 하나를 확인하세요:

* BTP 모니터링을 위해 구성된 CloudALM 계정
* Process Integration Runtime 서비스 구독(통합 모니터링에 필요)

<Callout variant="important">
  보안 또는 네트워크 팀의 승인이나 설정 지원이 필요할 수 있습니다. 설치하기 전에 이 단계를 완료하고 [설치 준비 체크리스트](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness)를 검토하십시오.
</Callout>

## 모니터 시스템 준비

에이전트리스 아키텍처는 데이터를 검색하기 위해 RFC 연결을 사용합니다.

### 버전 요구사항

필수 소프트웨어 버전을 확인합니다:

* **SAP\_BASIS**: 릴리스 740 이상
* **커널**: 릴리스 753 패치 602 이상
* **ST-PI**: 릴리스 740 SP16 이상

### 사용자 설정

* [권한 부여 역할을 업로드합니다.](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* RFC 연결을 위해 업로드된 역할을 사용하여 [시스템 사용자 생성](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id)

### HANA 데이터베이스 설정(해당하는 경우)

HANA 데이터베이스를 모니터링하는 경우, Catalog Read 및 모니터링 액세스 권한이 있는 HANA 데이터베이스 사용자 `NRAGENT` 을(를) 생성합니다:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

SQL 연결을 위해 HANA 서버 호스트 및 테넌트 DB 포트를 수집합니다.

### 기타 데이터베이스

다른 데이터베이스의 경우, ABAP 프로그램 `RSDBKPIMON`을(를) 실행하여 표준 CCMS 데이터베이스 KPI를 활성화합니다.

버전이 목록에 없는 경우, 추가 패키지에 대해 뉴렐릭 계정 담당자에게 문의하세요.

### 시스템 요구 사항

* 최소 20개의 다이얼로그 워크 프로세스와 10개의 백그라운드 워크 프로세스
* 뉴렐릭 데이터센터에 대한 아웃바운드 HTTPS 액세스(위의 엔드포인트 URL 참조)

### 네트워크 설정

* **프록시 설정** (필요한 경우):

  * 서버 도메인 이름/IP 주소 및 포트
  * 프록시 액세스 자격 증명(인증이 필요한 경우)

* **방화벽 정책**: 뉴렐릭 엔드포인트에 대한 HTTPS 트래픽 허용

### RFC 대상

각 모니터 시스템에 대해 트랜잭션 `SM59` 과 원격 시스템 ID를 사용하여 RFC 대상을 생성하고 연결을 테스트합니다. [모니터 시스템에 대한 RFC 대상 생성](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems)을 참조하십시오.

<Callout variant="important">
  RFC 사용자는 **/NEWR/DATA\_PROVIDER** 역할이 할당되어 있어야 합니다.
</Callout>

### 원격 HANA 데이터베이스 모니터링(해당하는 경우)

원격 HANA 데이터베이스를 모니터하려면:

* 중앙 모니터링 시스템이 HANA 서버 및 테넌트 DB 포트에 액세스할 수 있는지 확인하십시오.

* 비 S4HANA 중앙 모니터링 시스템의 경우, 다음 구성 요소 중 하나를 설치하십시오:

  * HANA 클라이언트 v2.0(HANA DB 버전과 호환됨)

  * HANA 클라이언트 라이브러리 파일:

    * `libSQLDBCHDB.so` (리눅스)
    * `libSQLDBCHDB.dll` (윈도우즈)
    * SAP Kernel 폴더에서 라이브러리를 찾아 SID Admin ID(예: `sd1adm`)에 소유권을 할당하세요

* 각 원격 HANA 데이터베이스에 대해 트랜잭션 `SAP DBACOCKPIT` 을 사용하여 DB 연결을 생성합니다. [DB 연결 만들기](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)를 참조하세요.

## BTP 모니터링 준비(해당하는 경우)

BTP 계정을 모니터링하려면, BTP 계정을 모니터링하도록 설정된 CloudALM 인스턴스 또는 Process Integration Runtime 서비스에 대한 구독(통합 모니터링에만 해당)이 필요합니다. 연결 인증을 위해 서비스 키가 필요합니다. [서비스 키 검색 방법](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime)을 참조하십시오.

서비스 키를 사용하여 [외부 HTTP 대상](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) 을 생성하고 API 데이터 제공자의 API 호스트로 사용합니다.
