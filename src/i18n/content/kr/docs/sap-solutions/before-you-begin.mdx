SAP 솔루션에 뉴렐릭 Monitoring을 사용하기 전에 다음 전제 조건이 충족되었는지 확인하세요.

* **뉴렐릭 계정 ID**: 뉴렐릭 계정을 식별하기 위해 [계정 ID를](/docs/accounts/accounts-billing/account-setup/account-id) 얻으세요.

* **인스턴스 키**: 인증 및 데이터 접근에 필요한 뉴렐릭 <InlinePopover type="licenseKey" />을(를) 확보합니다.

* **데이터센터 지역**: [선호하는 데이터센터 위치(미국 또는 EU)를 결정](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) 하고 그에 따라 계정을 구성하세요.

* **SAP 요구 시스템 사항**: 귀하의 SAP 시스템이 SAP 솔루션에 대한 뉴렐릭 모니터링에서 지정한 필수 소프트웨어 구성 요소 릴리스 수준을 충족하는지 확인하세요. 시스템 유형별 사용 가능한 데이터 공급자에 대한 자세한 내용은 [데이터 공급자 참조를](/docs/sap-solutions/additional-resources/data-providers-reference) 참조하십시오.

* **모니터링 BTP 계정**: BTP 계정이 모니터링 설정의 일부인 경우 다음 중 하나를 사용할 수 있어야 합니다.

  * **CloudALM 계정**: CloudALM 계정은 BTP 계정을 모니터링하도록 구성해야 합니다.
  * **프로세스 통합 런타임 서비스**: 특히 통합 모니터링에 필요한 모니터 BTP 계정의 프로세스 통합 런타임 서비스에 대한 구독을 확인하세요.

  <Callout variant="important">
    이러한 전제 조건을 충족하려면 보안 또는 네트워크 팀의 승인이나 설정 지원이 필요할 수 있습니다. 설치를 시작하기 전에 이러한 단계가 완료되었는지 확인하고, [설치 준비 체크리스트를](/docs/sap-solutions/additional-resources/pre-installation-checklist/#checklist-for-installation-readiness) 확인하여 모든 환경이 올바르게 설정되었는지 확인하세요.
  </Callout>

## 모니터링 시스템 [#monitored-systems]

우리의 에이전트리스는 **RFC 연결을** 사용하여 데이터를 검색합니다. 안전한 연결을 보장하려면 이러한 연결에 대한 특정 권한 프로필이 있는 SAP 시스템 사용자를 만들어야 합니다.

* `SAP_BASIS` 이 **릴리스 740 이상** 인지, `Kernel` 이 **릴리스 753 패치 602 이상** 인지, `ST-PI` 가 **릴리스 740 SP 16 이상** 인지 확인하세요.

* **권한 역할** [`NEWR_DATA_PROVIDER`](/docs/sap-solutions/additional-resources/additional-configuration/#upload-authorization-role) 을 업로드하세요.

* 업로드된 역할을 가진 시스템 사용자를 RFC 연결용으로 생성합니다. [뉴렐릭 RFC 사용자 ID 생성 을](/docs/sap-solutions/additional-resources/additional-configuration/#create-rfc-user-id) 참조하세요.

* **HANA 데이터베이스를** 모니터링하는 경우 다음 SQL 명령을 사용하여 **카탈로그 읽기** 및 **모니터링** 액세스 권한이 있는 **HANA 데이터베이스 사용자 NRAGENT를** 만듭니다.

  ```sql
    CREATE ROLE NR_AGENT_ROLE;
    GRANT CATALOG READ TO NR_AGENT_ROLE;
    GRANT MONITORING TO NR_AGENT_ROLE;
  ```

  ```sql
    CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
    GRANT NR_AGENT_ROLE TO NRAGENT;
  ```

* SQL 연결을 위해 **HANA 서버 호스트** 와 **테넌트 DB 포트를** 수집합니다.

* 다른 데이터베이스의 경우 ABAP 프로그램 `RSDBKPIMON` 을 실행하여 표준 **CCMS 데이터베이스 KPI를** 활성화합니다.

* 필요한 경우 뉴렐릭에 액세스하기 위한 프록시 서버 정책을 조정하세요. 프록시 설정이 구성되었는지 확인하세요.

  * 서버 도메인 이름/IP 주소 및 포트.
  * 인증이 필요한 경우 프록시 액세스 자격 증명.

* 차단된 경우 HTTPS 트래픽을 뉴렐릭으로 허용하도록 정책을 수정하세요.

  <Callout variant="tip">
    이러한 연결은 **Central Monitoring System** \[중앙 모니터링 시스템]에서 **SAP Transaction SM59** \[SAP 트랜잭션 SM59]를 사용하여 설정하고 테스트할 수 있습니다.자세한 내용은 [뉴렐릭 클라우드로 HTTP 대상 설정하기](/docs/sap-solutions/additional-resources/additional-configuration/#set-http-destination-to-new-relic-cloud) 를 참조하십시오.
  </Callout>

* 지정된 각 모니터링 시스템에 대해 트랜잭션 `SM59` 과 원격 시스템 ID를 사용하여 **RFC Destination** \[RFC 대상을] 생성하고 연결을 테스트합니다. 자세한 내용은 [모니터링 대상 시스템에 대한 RFC 대상 생성](/docs/sap-solutions/additional-resources/additional-configuration/#create-rfc-for-monitored-systems) 문서를 참조하십시오.

  <Callout variant="tip">
    RFC 사용자에게 `/NEWR/DATA_PROVIDER` 역할이 할당되었는지 확인하세요.
  </Callout>

* 원격 HANA 데이터베이스를 모니터링하려면 다음 사항을 확인하세요.

  * 중앙 모니터링 시스템은 HANA 서버와 테넌트 DB 포트에 액세스할 수 있어야 합니다.

  * 중앙 모니터링 시스템이 `S4HANA` 시스템이 아닌 경우 다음 구성 요소 중 하나가 설치되어 있는지 확인하세요.

    * HANA 클라이언트 v2.0(버전은 HANA DB와 호환되어야 함)

    * HANA 클라이언트 라이브러리 파일:

      * `libSQLDBCHDB.so` (리눅스)
      * `libSQLDBCHDB.dll` (윈도우즈)
      * 라이브러리는 SAP 커널 폴더에 있어야 하며 SID 관리자 ID(예: sd1adm)가 소유해야 합니다.

  * 각 원격 HANA 데이터베이스에 대해 트랜잭션 `SAP DBACOCKPIT` 사용하여 DB 연결을 생성합니다. 자세한 내용은 [데이터베이스 연결 생성 문서](/docs/sap-solutions/additional-resources/additional-configuration/#create-database-connection) 를 참조하십시오.

## BTP 계정 모니터링

BTP 계정을 모니터링하려면 BTP 계정을 모니터링하도록 CloudALM을 설정하거나 Process 통합 런타임 서비스에 대한 구독이 필요합니다(통합 모니터링에만 적용 가능). 연결 인증에는 서비스 키가 필요합니다. 자세한 내용은 [서비스 키를 검색하는 방법을](/docs/sap-solutions/additional-resources/obtain-service-key/process-integration-runtime) 참조하세요.

서비스 키는 [외부 HTTP 대상을](/docs/sap-solutions/additional-resources/additional-configuration/#create-authentication-endpoint) 생성하는 데에도 사용되며 API 데이터 제공자를 위한 API 호스트 역할을 합니다.
