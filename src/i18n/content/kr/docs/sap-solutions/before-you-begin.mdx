설치 전에 다음 전제 조건을 완료하십시오:

## 전제 조건 체크리스트

### New Relic 계정

* <DNT>**Account ID**</DNT>: [계정 ID 확인](/docs/accounts/accounts-billing/account-setup/account-id)
* <DNT>**License Key**</DNT>: 인증 및 데이터 액세스를 위한 뉴렐릭 <InlinePopover type="licenseKey" />가져오기
* <DNT>**Data Center Region**</DNT>: [데이터센터 위치(미국 또는 EU)를 선택](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) 하고 계정을 구성합니다

### SAP 시스템 요구사항

* <DNT>**Software versions**</DNT>: <DNT>**SAP systems**</DNT> 이(가) 필수 소프트웨어 구성 요소 릴리스 수준을 충족하는지 확인하십시오. 시스템별 요구 사항은 [데이터 공급자 참조](/docs/sap-solutions/monitor/data-providers-reference) 를 참조하십시오.
* <DNT>**Monitored systems**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**Kernel 753 Patch 602+**</DNT>, <DNT>**ST-PI 740 SP16+**</DNT>
* <DNT>**Central monitoring system**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**ST-PI 740 SP14+**</DNT>

### 네트워크 액세스

* <DNT>**HTTPS access**</DNT> <DNT>**New Relic endpoints**</DNT> (으)로([엔드포인트 URL](#network-endpoints) 참조)
* <DNT>**Firewall rules**</DNT> 구성됨
* <DNT>**Proxy settings**</DNT> 구성됨(필요한 경우)

### BTP 모니터링(해당하는 경우)

<DNT>**BTP accounts**</DNT>을(를) 모니터링하는 경우 다음 중 하나를 확인합니다:

* <DNT>**CloudALM account**</DNT> 용으로 구성됨 <DNT>**BTP monitoring**</DNT>
* <DNT>**Process Integration Runtime**</DNT> 서비스 구독(통합 모니터링에 필요)

<Callout variant="important">
  보안 또는 네트워크 팀의 승인이나 설정 지원이 필요할 수 있습니다. 설치하기 전에 이 단계를 완료하고 [설치 준비 체크리스트](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness)를 검토하십시오.
</Callout>

## 모니터 시스템 준비

에이전트리스 아키텍처는 데이터를 검색하기 위해 RFC 연결을 사용합니다.

### 버전 요구사항

필수 소프트웨어 버전을 확인합니다:

* <DNT>**SAP\_BASIS**</DNT>: 릴리스 <DNT>**740**</DNT> 이상
* <DNT>**Kernel**</DNT>: 릴리스 <DNT>**753 Patch 602**</DNT> 이상
* <DNT>**ST-PI**</DNT>: 릴리스 <DNT>**740 SP16**</DNT> 이상

### 사용자 설정

* [권한 부여 역할을 업로드합니다.](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* 다음을 위해 업로드된 역할을 사용하여 [시스템 사용자 만들기](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) <DNT>**RFC connections**</DNT>

### HANA 데이터베이스 설정(해당하는 경우)

<DNT>**HANA database**</DNT>을(를) 모니터링하는 경우 <DNT>**Catalog Read**</DNT> 및 <DNT>**Monitoring**</DNT> 액세스 권한이 있는 <DNT>**HANA database user**</DNT> `NRAGENT` 을(를) 생성합니다:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

<DNT>**SQL connection**</DNT>에 대한 <DNT>**HANA server host**</DNT> 및 <DNT>**tenant DB port**</DNT> 을(를) 수집합니다.

### 기타 데이터베이스

다른 데이터베이스의 경우, <DNT>**ABAP program**</DNT> `RSDBKPIMON`를 실행하여 표준 <DNT>**CCMS database KPIs**</DNT> 을 활성화합니다.

버전이 목록에 없는 경우, 추가 패키지에 대해 뉴렐릭 계정 담당자에게 문의하세요.

### 시스템 요구 사항

* 최소 <DNT>**20 dialog work processes**</DNT> 및 <DNT>**10 background work processes**</DNT>
* <DNT>**New Relic data center**</DNT> 로의 아웃바운드 <DNT>**HTTPS access**</DNT> (위의 엔드포인트 URL 참조)

### 네트워크 설정

* <DNT>**Proxy settings**</DNT> (필요한 경우):

  * 서버 도메인 이름/IP 주소 및 포트
  * 프록시 액세스 자격 증명(인증이 필요한 경우)

* <DNT>**Firewall policies**</DNT>: <DNT>**HTTPS traffic**</DNT> 이(가) 다음을 수행하도록 허용 <DNT>**New Relic endpoints**</DNT>

### RFC 대상

각 모니터 시스템에 대해 트랜잭션 `SM59` 과 원격 시스템 ID를 사용하여 <DNT>**RFC destinations**</DNT> 을(를) 생성하고 연결을 테스트합니다. [모니터 시스템에 대한 RFC 대상 생성](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems)을 참조하십시오.

<Callout variant="important">
  <DNT>**RFC user**</DNT> 에는 <DNT>**/NEWR/DATA\_PROVIDER**</DNT> 역할이 할당되어 있어야 합니다.
</Callout>

### 원격 HANA 데이터베이스 모니터링(해당하는 경우)

원격 <DNT>**HANA databases**</DNT>을(를) 모니터링하려면:

* 중앙 모니터링 시스템이 <DNT>**HANA server**</DNT> 에 액세스할 수 있는지 확인하고 <DNT>**tenant DB port**</DNT>

* 비 S4HANA 중앙 모니터링 시스템의 경우, 다음 구성 요소 중 하나를 설치하십시오:

  * <DNT>**HANA Client v2.0**</DNT> (<DNT>**HANA DB version**</DNT>와(과) 호환 가능)

  * <DNT>**HANA Client Lib file**</DNT>:

    * `libSQLDBCHDB.so` (리눅스)
    * `libSQLDBCHDB.dll` (윈도우즈)
    * <DNT>**SAP Kernel folder**</DNT> 에서 라이브러리를 찾아 <DNT>**SID Admin ID**</DNT> (예: `sd1adm`)에 소유권을 할당합니다.

* 각 원격 <DNT>**HANA database**</DNT>에 대해 트랜잭션 `SAP DBACOCKPIT` 을 사용하여 <DNT>**DB connection**</DNT>를 생성합니다. [DB 연결 만들기](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)를 참조하세요.

## BTP 모니터링 준비(해당하는 경우)

<DNT>**BTP account**</DNT>을(를) 모니터하려면 <DNT>**BTP account**</DNT>을(를) 모니터하도록 설정된 <DNT>**CloudALM instance**</DNT> 또는 <DNT>**Process Integration Runtime**</DNT> 서비스에 대한 구독이 필요합니다(통합 모니터링에만 해당). 연결 인증을 위해 <DNT>**Service keys**</DNT> 이(가) 필요합니다. [서비스 키 검색 방법](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime)을 참조하십시오.

<DNT>**service keys**</DNT> 을(를) 사용하여 [외부 HTTP 대상](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) 을 생성하고 <DNT>**API data providers**</DNT>에 대한 <DNT>**API host**</DNT> (으)로 사용합니다.
