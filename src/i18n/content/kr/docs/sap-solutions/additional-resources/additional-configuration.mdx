이 섹션에서는 시스템 설정에 대한 추가 지침을 제공합니다.

<Steps>
  <Step>
    ## 업로드 권한 역할 [#upload-authorization-role]

    1. `SAP transaction PFCG` 을(를) 사용합니다.

    2. **Role** &gt; **Upload** 로 이동하여 설치 패키지에서 **\_NEWR\_DATA\_PROVIDER.SAP** 파일을 업로드합니다.

    3. 역할 `/NEWR/DATA_PROVIDER` 이 로드될지 확인하세요.

       <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

    4. 클릭<img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

       로드되면 아이콘이 표시됩니다.

    5. **Authorizations** \[권한] 탭에서 다음을 선택하세요.<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

       **Change Authorization Data** \[권한 데이터 변경] 및 활성화를 사용하여<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />아이콘입니다.

    <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

    6. 프로필 이름이 제안되면 녹색 확인 아이콘을 클릭하세요.
    7. **Profile(s) were updated** \[프로필이 업데이트되었습니다라는] 메시지가 상태 표시줄에 나타나는지 확인하세요.
  </Step>

  <Step>
    ## 뉴렐릭 RFC 사용자 ID 생성 [#create-rfc-user-id]

    권한 프로필을 활성화한 후 트랜잭션 `SU01` 사용하여 새 RFC 사용자 ID(예: `NR4SAPDP`)를 생성합니다. 사용자 유형을 **System** \[시스템] 또는 **Communication** \[통신] 으로 설정하고 역할 `/NEWR/DATA_PROVIDER` 추가합니다.

    <img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />
  </Step>

  <Step>
    ## 모니터링되는 시스템에 대한 RFC 대상 생성 [#create-rfc-for-monitored-systems]

    `SM59` 트랜잭션을 사용하여 모니터링되는 각 시스템에 대한 RFC 대상을 생성합니다.

    1. **ABAP Connections** \[ABAP 연결을] 선택하고 **Create** \[만들기] 아이콘을 클릭하여 새로운 RFC 대상을 설정합니다.

    2. 기술 **RFC Destination** \[RFC 대상] 이름을 입력합니다. RFC 대상의 이름을 다음과 같이 지정하는 것이 좋습니다. `NR_<SYSID>CLNT<CLNT#>`

    3. **Connection Type** \[연결 유형을] `3 - ABAP Connection` 으로 설정합니다.

    4. **Technical Settings** \[기술 설정] 탭에서 다음을 입력합니다.

       * **Target Host** \[대상 호스트]\(정규화된 도메인 또는 IP) 및 **Instance No.** \[인스턴스 번호]
       * 필요한 경우 **Gateway Host** \[게이트웨이 호스트] 와 **Service** \[서비스를] 지정하세요.

    <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

    5. **Logon &amp; Security** \[로그온 및 보안] 탭에서 RFC 사용자(`NR4SAPDP`)와 비밀번호를 입력합니다.
    6. **Save** \[저장] 을 클릭합니다.
    7. 연결이 성공했는지 확인하려면 **Utility &gt; Test &gt; Connection Test** 및 **Authorization Test**\[권한 부여 테스트] 로 이동하세요.

    <Callout variant="important">
      테스트 결과가 성공적인지 확인하세요.
    </Callout>
  </Step>

  <Step>
    ## HTTP 대상을 뉴렐릭 cloud로 설정

    그래픽 `SM59` 사용하여 뉴렐릭 API 엔드포인트에 대한 HTTPS 연결을 설정하세요.

    <Callout variant="important">
      이벤트, 로그인, 지표, 트레이스에 대한 개별 RFC 대상을 별도로 생성해야 합니다.
    </Callout>

    1. **Destination Name**\[목적지 이름을] 입력하세요(예: `NR_API_EVENT`).
    2. **Connection Type** \[연결 유형을] `G` (외부 서버에 대한 HTTP 연결)으로 설정합니다.
    3. **Target Host** \[대상 호스트], **Service No.** \[서비스 번호]\(443)(HTTPS의 경우 [표준 포트](/docs/sap-solutions/additional-resources/network-accessibility) 참조), 그리고 **경로 접두사**를 입력하십시오(자세한 내용은 [설치 준비 체크리스트](/docs/sap-solutions/additional-resources/pre-installation-checklist/#checklist-for-installation-readiness) 참조).
    4. 프록시가 필요한 경우 **Technical Settings** \[기술 설정] 탭에서 구성하세요.
    5. **Security Options** \[보안 옵션] 탭에서 SSL에 대한 **Active** \[활성] 라디오 버튼을 선택합니다.
       <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />
    6. 필요한 경우 [뉴렐릭](https://one.newrelic.com) 에서 루트 인증서를 다운로드하고 론 `STRUST` 을 사용하여 SAP에 추가하세요.
    7. **Connection Test** \[연결 테스트를] 사용하여 연결을 확인하세요. 연결에 성공하면 **Enter Logon Data** \[로그인 데이터를 입력하라는] 메시지가 표시됩니다. 테스트 시간이 초과되면 방화벽 설정을 확인하세요.
    8. 모든 뉴렐릭 API 엔드포인트에 대해 이 절차를 반복합니다.
  </Step>

  <Step>
    ## BTP/CloudALM 인증 엔드포인트 생성 [#create-authentication-endpoint]

    서비스에 따라 아래 지침을 따르세요.

    <CollapserGroup>
      <Collapser id="btp-process-integration-runtime" title="BTP 프로세스 통합 런타임">
        1. 다운로드한 **Service Key** \[서비스 키] 에서 `clientId`, `clientsecret` 및 `tokenurl` 를 확인하세요.
        2. 웹 브라우저에서 `tokenurl` 으로 이동하여 필요한 인증서(`Digicert root, global, and SAP on demand`)를 다운로드하세요.

        <img title="BTP process integration runtime" alt="A screenshot displaying the connection setup for BTP process integration runtime" src="/images/btp-process-integration.webp" width="600" />

        3. `STRUST` 을 사용하여 세 개의 인증서를 하나씩 업로드하고 **Add to Certificate List** \[인증서 목록에 추가를] 클릭합니다.
        4. **저장** 을 클릭합니다.
        5. 트랜잭션 `SMICM` 을 사용하고 **Administration &gt; ICM &gt; Exit Soft &gt; Global** 로 이동하여 모든 ICM 프로세스를 다시 시작합니다.

        <img title="Administration ICM" alt="A screenshot displaying the Administration ICM" src="/images/administration-icm.webp" width="600" />

        6. `SM59` 트랜잭션을 사용하여 **HTTP Connections to External Server** \[외부 서버에 대한 HTTP 연결] 에서 새로운 RFC 연결을 생성합니다.
        7. tokenurl을 호스트와 경로 접두사로 분할합니다. 경로 접두사에 `?grant_type=client_credentials` 추가합니다.
        8. **Service No.** \[서비스 번호를] `443` 로 설정합니다.

        <img title="Service No." alt="A screenshot displaying the Service No." src="/images/service-no.webp" width="600" />

        9. **Login &amp; Security** \[로그인 및 보안] 탭에서 **Basic Authentication** \[기본 인증을] 선택하고 사용자 이름과 비밀번호로 `clientId` 과 `clientsecret` 입력합니다.
        10. **Secure Protocol** \[보안 프로토콜] 상태를 **Active** \[활성] 으로 설정하고 구성을 저장합니다.
        11. **Connection Test** \[연결 테스트를] 실행합니다.
      </Collapser>

      <Collapser id="cloud-alm-api" title="클라우드ALM API">
        1. 다운로드한 **Service Key** \[서비스 키] 에서 `clientId`, `clientsecret` 및 `url` 를 확인하세요.
        2. 웹 브라우저에서 URL로 이동하여 **Digicert root**, **global** 및 **SAP on demand** 인증서를 다운로드하세요.

        <img title="CloudALM API" alt="A screenshot displaying the connection setup for CloudALM API" src="/images/cloudalm-api.webp" width="600" />

        3. `STRUST` 을 사용하여 세 개의 인증서를 하나씩 업로드하고 **Add to Certificate List** \[인증서 목록에 추가를] 클릭합니다.
        4. **저장** 을 클릭합니다.
        5. 트랜잭션 `SMICM` 을 사용하고 **Administration &gt; ICM &gt; Exit Soft &gt; Global** 로 이동하여 모든 ICM 프로세스를 다시 시작합니다.

        <img title="Administration ICM" alt="A screenshot displaying the Administration ICM" src="/images/administration-icm.webp" width="600" />

        6. `SM59` 트랜잭션을 사용하여 새로운 RFC 연결을 생성합니다.
        7. **Target host** \[대상 호스트]에 URL을 입력하고 **Service No.** \[서비스 번호]를 `443`으로 설정합니다.**Path Prefix** \[경로 접두사]를 `/oauth/token?grant_type=client_credentials`로 설정합니다.

        <img title="SM59 Configuration" alt="A screenshot displaying the SM59 configuration" src="/images/sm59-configuration.webp" width="600" />

        8. **Login &amp; Security** \[로그인 및 보안] 탭에서 **Basic Authentication** \[기본 인증을] 선택하고 사용자 이름과 비밀번호로 `clientId` 과 `clientsecret` 입력합니다.
        9. **Secure Protocol** \[보안 프로토콜] 상태를 **Active** \[활성] 으로 설정하고 구성을 저장합니다.
        10. **Connection Test** \[연결 테스트를] 실행합니다.
      </Collapser>
    </CollapserGroup>
  </Step>

  <Step>
    ## 데이터베이스 연결 생성 [#create-database-connection]

    트랜잭션 `DBACOCKPIT` 을 사용하여 원격 **HANA Database** \[HANA 데이터베이스] 연결을 정의합니다.

    1. **Database Connections** \[데이터베이스 연결을] 두 번 클릭하고 **Add** \[추가를] 클릭합니다.

       <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

    2. **Connection Name** \[연결 이름을] 입력하세요(SAP 시스템 ID와 동일한 것이 좋습니다).

    3. **Database System** \[데이터베이스 시스템] 으로 **HANA Database** \[HANA 데이터베이스를] 선택합니다.

    4. **Database user ID** \[데이터베이스 사용자 ID] (예: `NRAGENT`)와 비밀번호를 입력하세요.

    5. **Database Host** \[데이터베이스 호스트] 와 **SQL Port**\[SQL 포트를] 입력하세요.

       <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

    6. 데이터를 저장하고 **Test** \[테스트를] 클릭하여 연결을 확인하세요.
  </Step>
</Steps>
