## 런타임 상태 및 전송 로그

**Data Provider Registry** \[데이터 공급자 레지스트리] (트랜잭션 `/NEWR/REGISTRY`)에서 다음을 확인할 수 있습니다.<img src="/images/runtimestatus.webp" class="inline" style={{height: "40px", width: "120px"}} alt="RuntimeStatus icon" title="RuntimeStatus" />

그리고<img src="/images/transmissionlog.webp" class="inline" style={{height: "40px", width: "120px"}} alt="transmissionlog" title="Transmissionlog.webp" />각 데이터 제공자별로. 전송 로그는 각 전송의 상태와 시간을 보여줍니다.

## 작업 상태 및 스풀 확인

1. 트랜잭션 `SM37` 을 사용하여 예약된 작업의 상태를 확인합니다. **ABAP Program Name** \[ABAP 프로그램 이름] 필드에 `/NEWR/*` 입력하고 **Execute** \[실행을] 클릭합니다.
2. 실행 기간에 주의하세요. 비정상적으로 긴 시간은 잠재적인 문제를 나타냅니다.
3. 중복되는 작업이 있는지 확인하고 필요에 따라 일정을 조정하세요. 작업 스풀은 각 데이터 제공자에 대한 결과를 제공합니다.

## 애플리케이션 로그 확인

데이터 수집 프로세스는 로그로그(트랜잭션 SLG1)에서 로그됩니다. 애플리케이션 로그 개체 `/NEWR/DP` 를 사용한 쿼리.

## SAP 데이터 수집 모니터링 대시보드

데이터 수집 지표는 자체 모니터링을 위해 뉴렐릭에도 전송됩니다. 기본 제공되는 [**SAP Data Collection Monitoring** \[SAP 데이터 수집 모니터링\]]() 대시보드를 사용하여 각 데이터 공급자에 대한 데이터 수집 프로세스를 모니터링합니다.
