이 문서에서는 공지 이벤트를 종료하는 다양한 방법을 설명합니다.

## 공지이벤트 자동 종료 방법 [#close-automatic]

공지 이벤트는 해당 조건의 해리값, 경계에 표시된 기간 동안 목표 신호가 위반하지 않은 상태로 돌아오면 자동으로 종료됩니다. 이 대기 시간을 [복구 기간](/docs/using-new-relic/welcome-new-relic/get-started/glossary#recovery-period) 이라고 합니다.

예를 들어, 위반 행위가 &quot;5분 내에 Apdex 점수가 0.80 미만인 경우가 한 번 이상 발생&quot;인 경우, Apdex 점수가 5분 연속으로 0.80 이상이 되면 공지 이벤트가 자동으로 종료됩니다. &quot;최소 x분 동안&quot;에도 동일하게 적용됩니다. 즉, x분 동안 위반 행위가 없어야 공지 이벤트가 자동으로 종료됩니다.

공지 이벤트가 자동으로 종료되는 경우:

1. 마감 타임스탬프는 복구 기간 시작으로 소급됩니다.
2. 평가는 이전 공지 이벤트가 종료된 시점부터 재설정되어 다시 시작됩니다.

모든 조건에는 오래 지속되는 공지 이벤트를 자동으로 강제 종료하는 [공지 이벤트 시간 제한](#time-limit) 설정이 있습니다.

## 오래 지속되는 공지 이벤트의 시간 제한을 설정하세요. [#time-limit]

공지 이벤트 시간 제한 설정은 선택한 일/시간이 지나면 오래 지속되는 공지 이벤트를 자동으로 강제 종료합니다. 이는 사라질 때 자동으로 닫히지 않는 지속적인 알림 이벤트를 발생시키는 일시적인 요소에 가장 유용합니다.

<DNT>
  **Limits and Defaults**
</DNT>

* 모든 공지 이벤트에는 공지 이벤트 시간 제한이 적용됩니다. 대부분의 공지 조건에서는 이 필드를 편집할 수 있습니다.
* 기본값은 구성 중에 지정하지 않으면 3일(인프라 조건의 경우 24시간)입니다.
* 인프라가 아닌 조건에 대한 공지이벤트 시간 제한은 최소 5분, 최대 30일까지 설정할 수 있습니다. 어떤 이유로든 30일 후에도 여전히 신호가 위반되는 경우 공지 이벤트가 종료되고 새로운 공지 이벤트가 열립니다. 인프라 조건은 1, 2, 4, 8, 12, 24, 48 또는 72시간으로 설정할 수 있습니다.

<Callout variant="tip">
  이 설정은 [비활성 이슈 설정](/docs/alerts/admin/user-settings/#auto-close-issue-window) 과 관련이 있습니다.

  두 설정의 시간 간격이 다를 경우, 저희 시스템은 설정과 관계없이 더 짧은 시간 간격을 사용합니다. 예를 들어, 이슈 마감/개시 알림 이벤트 시간 설정이 2일이고 비활성 이슈 시간 설정이 3일인 경우, 저희 시스템은 이슈를 마감하기 전에 2일을 기다립니다.
</Callout>

<DNT>
  **Examples:**
</DNT>

* 공지 이벤트 시간 제한을 12시간으로 설정했습니다. 해당 공지 이벤트가 12시간 동안 지속되면 12시간 후에 종료되고 해당 공지에 대한 조건 평가가 초기화됩니다.
* JVM 에 CPU 스파이크가 발생하여 공지 이벤트가 생성됩니다. 그러면 JVM이 충돌하고 새로운 JVM으로 교체됩니다. 공지 이벤트 시간 제한을 설정하지 않은 경우 충돌이 발생한 JVM의 공지 이벤트는 절대 닫히지 않습니다.
