## 문제 검색 및 필터링 [#search-filters]

검색창 자유 텍스트를 사용하여 모든 문제를 쉽게 검색할 수 있습니다. 이 검색 표시줄을 사용하여 다음을 검색할 수 있습니다.

* 문제 이름
* 문제 ID
* 정책 이름
* 조건 이름
* 엔티티 이름
* 엔티티 ID
* 우선순위

## 일괄 작업 [#bulk-actions]

문제 수수료 목록에서 여러 문제를 선택하고 <DNT>**Acknowledge issues**</DNT> 또는 <DNT>**Close issues**</DNT> 작업을 선택하여 해당 문제를 직접 확인하거나 닫을 수 있습니다.

</DNT>

<Callout variant="important">
  공지 조건은 현재 **warning** \[경고] 와 **critical** \[심각의] 두 가지 우선순위 수준을 지원합니다. 이러한 수준은 생성되는 공지 이벤트의 우선순위를 결정합니다.

  * 공지 조건에 대한 **warning** \[경고] 값, 주석은 우선순위가 높은 공지 이벤트를 생성합니다.
  * **Critical** \[중요한] 노인값, 릴레이는 중요한 우선순위 공지 이벤트를 생성합니다.

  이슈의 우선순위는 해당 이슈를 구성하는 가장 중요한 공지 이벤트의 우선순위에서 결정됩니다. 예를 들어, 높음/경고 공지 이벤트가 4개이고 중요한 알림 이벤트가 1개인 문제에는 **critical** \[중요] 우선순위가 할당됩니다.
</Callout>

필터 아이콘을 클릭하면 문제 속성이나 관련 태그를 기준으로 필터링할 수 있습니다.

* 문제 상태(생성됨, 활성 또는 닫힘)
* 문제가 인정됨(참 또는 거짓)
* 문제가 숨겨졌습니다(참 또는 거짓).
* 상관된 문제(참 또는 거짓)
* 문제 우선순위(낮음, 중간, 높음 또는 중요)
* 문제 소스(NR 경고, NR 이상 또는 REST API)
* 실재
* 경고 정책
* 경보 조건
* 태그(문제와 관련된 태그 검색)

## 이슈 페이지 [#issue-summary]

<DNT>**Issue**</DNT> 페이지에서는 먼저 문제를 이해하고 해결 시간을 최소화할 수 있도록 인사이트에 대한 결론을 제공합니다.

여기에서 문제 페이지 사용자 인터페이스를 안내하는 빠른 데모를 만들었습니다.

### 이슈 페이지 정보 [#about-issue-page]

<Collapser id="issue page details" title="이슈 페이지 정보">
  <DNT>**Issue page**</DNT> 에는 다음 섹션이 포함되어 있습니다.

  * <DNT>**Issue payload:**</DNT> 이를 통해 문제 페이로드 세부 정보를 제공하고 버튼 클릭 한 번으로 페이로드를 복사할 수 있습니다.

  * <DNT>**Issue destinations:**</DNT> 문제 이름 아래에 목적지를 표시하는 알림이라는 카테고리가 표시됩니다. 목적지 위로 마우스를 가져가면 목적지에 대한 자세한 프리젠테이션과 열려 있는 지원 케이스에 대한 링크가 표시됩니다.

  * <DNT>**Issue duration:**</DNT> 문제 이름 상단에 문제가 지속된 기간이 표시됩니다.

  * <DNT>**Number of alert events:**</DNT> 공지 이벤트 활동 섹션 상단에는 해당 이슈 내 총 공지 이벤트 개수가 표시됩니다.

  * <DNT>**Alert event list:**</DNT> 공지 이벤트 활동 섹션의 왼쪽에는 공지 이벤트 활동 목록이 표시됩니다. 여기에는 우선 순위, 상태, 공지 이벤트 이름, 생성 날짜 및 시간, 기간 등 각 공지 이벤트에 대한 가장 관련성이 높은 정보가 표시됩니다. 또한 기간, 음소거된 공지 이벤트, 최신순, 오래된 공지순, 중요순, 낮은 공지이벤트순으로 목록을 정렬할 수도 있습니다. 마지막으로, <DNT>**Show open only**</DNT> 버튼을 활성화하면 열린 공지 이벤트만 필터링할 수 있습니다.

  * <DNT>**Alert event activity chosen:**</DNT> 공지이벤트 활동 목록에서 공지이벤트를 클릭하면 오른쪽의 공지이벤트가 전체 보기 모드로 열립니다. 여기에는 공지문, 공지조건, 공지조건 유형과 같은 정보가 포함됩니다.

  * <DNT>**Alert event graph:**</DNT> 공지 이벤트 활동 목록에서 공지 이벤트를 클릭하면 전체 범위의 공지 이벤트 차트가 열리므로 성능 저하, 공지 이벤트 및 복구 기간을 더 잘 시각화할 수 있습니다.

  * <DNT>**User actions (above the chart):**</DNT> 공지 이벤트 활동 목록에서 공지 이벤트를 클릭하면 수행할 수 있는 특정 작업이 표시됩니다.

    * 이 조건에 Runbook URL이 정의되어 있으면 여기에 Runbook URL이 표시됩니다.
    * 주요 이벤트 시간 창(모든 유형)이 있는 필기 요약 페이지로 리디렉션되는 부분 개요 버튼입니다.
    * <DNT>**See errors**</DNT> 버튼은 오류 수신함(errors inbox)에 동일한 오류 유형과 관련된 오류 그룹이 있고, 해당 오류 그룹이 공지 이벤트와 동일한 시간대에 발생한 경우에만 나타납니다.
    * 줄임표 아이콘 <DNT>**...**</DNT> 에는 특정 공지 이벤트를 닫는 옵션(해당 공지 이벤트가 1개 이상인 경우에만 해당)과 공지 이벤트 페이로드를 표시하는 드롭다운 메뉴 메뉴가 있습니다.

  * <DNT>**Alert event entity section (below the chart):**</DNT> 공지이벤트 활동 목록에서 해당 공지이벤트를 클릭하시면 다음과 같은 내용을 보실 수 있습니다.

    * 해당 공지 이벤트의 영향을 받은 부분, 부분 유형 및 계정 이름이 속해 있습니다.

    * 태그(엔터티 &amp; 조건 태그) 및 해당 값의 목록입니다. 모든 태그를 표시하려면 <DNT>**Show all**</DNT> 버튼을 클릭하세요.

## 검시 [#postmortem-intro]

사후 분석은 팀이 공지 이벤트에 대응하고 해결할 때 무엇이 효과가 있었고 무엇이 효과가 없었는지 분석하기 위해 사용하는 회고적 프로세스입니다.

뉴렐릭 플랫폼에서 사후 분석 기능은 공지 이벤트와 관련된 데이터를 자동으로 수집하는 도구로, 팀이 향후 공지 이벤트에 대한 향상된 대응을 위해 분석 및 작업 항목에 집중할 수 있도록 해줍니다.

부검에는 다음이 포함됩니다.

* 설명을 포함한 공지사항의 기록
* 공지사항 타임라인
* 공지이벤트의 영향
* 공지이벤트의 근본 원인
* 팀에서 취한 완화 조치
* 향후 공지사항 재발 방지를 위한 후속 조치 사항

사후 분석 생성에 대한 자세한 단계를 보거나 연습 데모를 보려면 [사후 분석 문서](/docs/alerts-applied-intelligence/applied-intelligence/postmortems-applied-intelligence/) 페이지를 방문하십시오.

## 영향을 받는 엔터티 및 문제 맵

<img title="The issue entity map view." alt="A screenshot of the issue entity map view." src="/images/accounts_screenshot-crop_issue-map-final.webp" />

<figcaption>
  엔티티 위로 마우스를 가져가면 엔티티에 대한 자세한 정보를 볼 수 있습니다.
</figcaption>

<DNT>**Impacted entities**</DNT> 섹션에서는 둘 이상의 엔터티와 관련된 모든 문제에 대해 문제 맵을 사용할 수 있습니다. [지도에는](/docs/new-relic-one/use-new-relic-one/ui-data/maps-in-context/) 영향을 받는 엔터티뿐만 아니라 해당 엔터티와 직접 관련된 서비스 및 리소스도 표시됩니다.

<DNT>**Impacted entities:**</DNT> 엔티티란 모니터링할 수 있는 데이터를 가진 모든 것을 의미합니다. 특히 뉴렐릭 소스의 공지 이벤트에 초점을 맞춰 부분을 추출하고 요약을 제공합니다. 각각의 부분은 독특합니다. 엔티티를 목록이나 지도에서 확인할 수 있습니다.

문제의 데이터에 따라 이 네 가지 섹션이 모두 각 문제에 대해 함께 표시되거나 별도로 표시될 수 있습니다. 엔티티 옆에 있는 3개의 점을 클릭하여 다음과 같은 옵션이 있는 드롭다운 메뉴를 열기만 하면 됩니다.

* <DNT>**See dashboards**</DNT> 계정의 사용자가 엔터티와 관련하여 생성한 [대시보드를](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 보고 상호 작용하는 데 도움이 됩니다. 다양한 위젯을 구동하기 위해 실행한 쿼리는 가능할 때마다 자동으로 엔터티에 매핑되며 빠른 액세스 및 검색을 위해 여기에 다시 표시됩니다.
* <DNT>**Entity view**</DNT> 작업의 이상 수행 페이지가 열립니다. 이는 사전 탐지를 위해 설정된 애플리케이션에만 사용할 수 있습니다.
* <DNT>**deployment events**</DNT> 에는 배포와 관련 배포라는 두 가지 유형이 있습니다. <DNT>**[Show all deployments](/whats-new/applied-intelligence-deployment-events-issue-feed)**</DNT> 클릭하여 모든 배포 이벤트가 도착했을 때 확인하거나 특정 배포를 클릭하여 해당 <InlinePopover type="apm" />배포 페이지를 확인하세요. [APM 배포 페이지에는](/docs/change-tracking/change-tracking-view-analyze/#deployment-details) 최근 배포와 이것이 최종 사용자 및 앱 서버의 Apdex 점수, 응답 시간, 처리량 및 오류에 미치는 영향이 나열되어 있습니다. 이 섹션은 뉴렐릭이 배포가 포함된 영향을 받은 엔터티에서 구성을 식별한 경우에만 표시됩니다.
