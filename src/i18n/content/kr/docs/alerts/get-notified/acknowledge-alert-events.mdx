[문제를](/docs/new-relic-solutions/get-started/glossary/#issue) 확인하면 문제의 소유권을 가져가고 있음을 나타냅니다. 이것은 알림을 받은 다른 사람들에게 당신이 책임을 졌다는 것을 알리는 데 유용합니다.

## &quot;인정&quot;의 의미 [#ack-role]

문제를 인정할 때 소유하기로 동의하는 책임에 대한 조직의 지침을 따르십시오. 문제를 인정함으로써 귀하의 이름에 플래그를 지정하고 다른 사람에게 알립니다.

<Callout variant="tip">
  공지 처리를 위한 조직의 표준 운영 절차에 대한 빠른 참조를 제공하기 위해 공지 사항 [에 대한 레버 설정, 조건에 런북 URL을](/docs/alerts/create-alert/condition-details/provide-runbook-instructions-alert-activity) 포함할 수 있습니다. UI 에는 <DNT>**alert events**</DNT> 페이지와 이메일 공지에 런북 URL이 포함됩니다.
</Callout>

어떤 문제를 인정할 수 있는 사람은 오직 한 명뿐입니다. 일단 인정하면 되돌릴 수 없습니다. 다른 사람이 동일한 문제를 확인하려고 시도하면(예: 이메일 공지를 통해), 문제가 확인되었음을 보여주는 [문제의 <DNT>**History**</DNT> 페이지가](/docs/alerts/alert-event-management/view-alert-event-details#view-events) 나타납니다.

## 문제를 인정하는 방법 [#acknowledge-issue]

사용자 인터페이스를 통해 [<DNT>**Issues**</DNT>](/docs/alerts/alert-event-management/view-alert-event-details) 인덱스 또는 선택한 이슈 자체에서 이슈를 보고 승인할 수 있습니다. <DNT>**Issues**</DNT> 페이지는 이슈 기록에 대한 세부 정보를 제공하므로 이슈의 이벤트 체인과 타임라인을 더 쉽게 분석할 수 있습니다.

사용자 인터페이스에서 문제를 확인하려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Issues &amp; Activity**</DNT> 로 이동합니다. <DNT>**Issues**</DNT> 탭을 클릭하세요.
2. 선택한 문제 행에서 다음을 클릭하세요. <Icon name="fe-more-horizontal" /> 아이콘을 클릭하고 <DNT>**Acknowledge this issue**</DNT> 선택하세요.

/\* &lt;img title=&quot;알림 페이지: 이 문제를 승인하세요&quot; alt=&quot;알림 페이지: 이 문제를 승인하세요&quot; src=&quot;/images/alerts\_screenshot-full-acknowledge-this-issue.webp&quot; /&gt; \*/

3. 옵션을 선택하면 <DNT>**Actions taken**</DNT> 열에 할당된 사용자의 아바타가 표시됩니다. 사용자 이름을 확인하려면 아바타 위에 마우스를 올리세요.

   /\* &lt;img width=&quot;50%;&quot; title=&quot;알림 페이지: 조치 내역&quot; alt=&quot;알림 페이지: 조치 내역&quot; src=&quot;/images/alerts\_screenshot-crop\_acknowledged-by.webp&quot; /&gt; \*/

공지를 수신하도록 설정된 모든 사람에게 자동으로 통보됩니다. 또한 UI는 자동으로 귀하의 이름(또는 해당되는 경우 아바타)을 <DNT>**Actions taken**</DNT> 열에 추가합니다.

## 문제를 승인 취소하고 닫는 방법 [#unacknowledge-issue]

사용자 인터페이스에서 문제 확인을 취소하려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Issues &amp; Activity**</DNT> 로 이동합니다. <DNT>**Issues**</DNT> 탭을 클릭하세요.

2. 선택한 문제 행에서 할당된 사용자가 있는 특정 문제를 찾습니다.

3. 클릭하세요 <Icon name="fe-more-horizontal" /> 아이콘을 클릭하고 <DNT>**Unacknowledge this issue**</DNT> 또는 <DNT>**Close this issue**</DNT> 선택하세요.

   /\* &lt;img width=&quot;50%;&quot; title=&quot;알림 페이지: 이 문제 승인 취소&quot; alt=&quot;알림 페이지: 이 문제 승인 취소&quot; src=&quot;/images/alerts\_screenshot-crop\_unacknowledge-this-issue.webp&quot; /&gt; \*/

## 문제 필터링 [#filter-issues]

문제를 필터링하려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Issues &amp; Activity**</DNT> 으)로 이동합니다.

2. 일부 열을 사전순으로 정렬할 수 있습니다. 모든 열을 정렬할 수 있는 것은 아닙니다.

3. 필터 아이콘을 사용할 수도 있습니다.

   /\* &lt;img title=&quot;알림 페이지: 문제 필터 아이콘&quot; alt=&quot;알림 페이지: 문제 필터 아이콘&quot; src=&quot;/images/alerts\_screenshot-crop\_filter-issues.webp&quot; /&gt; \*/

## 사용할 수 있는 다른 옵션 [#ack-email-mobile]

추가 방법(채널)을 사용하여 알림을 확인할 수 있습니다. 예를 들어:

* 귀하의 공지 채널에 이메일이 포함되어 있는 경우 이메일의 <DNT>**Acknowledge**</DNT> 링크를 선택하여 공지를 수신하는 즉시 확인할 수 있습니다.
* iOS 또는 Android 모바일 앱을 등록하고 계정에 연결했다면 정책에서 모바일을 알림 채널로 추가할 수 있습니다. 예를 들어 Android 사용자는 알림 표시줄을 사용하여 경고를 확인할 수 있습니다. New Relic 앱을 열 필요가 없습니다.
