이 페이지의 정보를 최대한 활용하려면 다음 용어의 의미를 이해하는 것이 좋습니다.

* <DNT>**Issues**</DNT>이슈는 동일한 근본적인 문제와 관련된 공지 이벤트들의 그룹입니다. 새로운 공지 이벤트가 생성되면 공지 이벤트 인텔리전스가 새로운 이슈를 열고 다른 공개 이슈의 가능한 상관 관계를 평가할 수 있습니다. 자세한 내용은 [이슈 및 공지 이벤트 관리 및 대응을](/docs/alerts/alert-event-management/Issues-and-alert-event-management-and-response/) 참조하세요.

* <DNT>**Alert events**</DNT>. 공지 이벤트는 시간이 지남에 따라 시스템에 영향을 미치는 증상을 설명하는 이벤트입니다. 자세한 내용은 [공지이벤트 세부정보 보기를](/docs/alerts/alert-event-management/view-alert-event-details/) 참고하세요.

* <DNT>**Conditions**</DNT>. 조건은 모니터 데이터 소스와 공지 이벤트로 간주되는 해당 데이터 소스의 동작을 설명합니다. 자세한 내용은 [&apos;조건 생성&apos;을](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/) 참조하세요.

* <DNT>**Policies**</DNT>. 정책은 하나 이상의 조건으로 구성된 그룹입니다. 자세한 내용은 [정책 관리를](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/) 참조하세요.

이슈 유형에 따라 비즈니스 운영에 지장을 주는 공지 이벤트에 대한 알림을 받는 시점이 결정됩니다. 이러한 공지 이벤트는 귀하의 공지 조건을 충족할 때 발생합니다. 예를 들어 CPU 초과, 신규 모니터링 충돌 또는 느린 웹 앱 응답 시간에 대한 공지 조건을 가질 수 있습니다.

문제를 생성하는 정책 조건을 제어할 수 있는 경우 진행 중이거나 반복되는 중요한 문제에 대한 알림을 받을 수 있습니다. 이렇게 하면 불필요한 알림을 받지 않고 소음을 피할 수 있습니다.

정책, 조건 또는 조건과 신호 조합에 따라 문제 기본 설정을 지정하는 방법을 알아보세요.

## 중요 및 경고 공지사항 [#alert-event-types]

정책 조건이 충족되면 두 가지 임계값 유형 중에서 선택합니다.

* 크리티컬(빨간색)
* 경고(노란색)

정책 조건이 허용 또는 제한을 위반하면 알림 이벤트가 열리고 효율적인 대응을 위해 상세 정보가 포함된 이슈 레코드가 생성됩니다. 이슈 기록에는 개시 공지 이벤트 이후에 발생하는 다른 공지 이벤트도 포함될 수 있습니다.

문제가 열렸을 때, 문제가 승인된 경우, 문제가 종료될 때 적절하게 구성된 워크플로의 모든 대상뿐만 아니라 정책의 모든 알림 채널로 알림이 전송됩니다.

### 정책당 하나의 문제(기본값) [#preference-policy]

정책 및 해당 조건의 범위가 집중된 경우(예: 모니터링 1개) <DNT>**One issue per policy**</DNT> 설정을 선택하세요.

이슈 오픈 이후 추가적인 조건으로 공지 이벤트가 오픈되면, 해당 이슈에 공지 이벤트가 추가됩니다. 선택한 정책에 대해서는 <DNT>**Issues &amp; activity**</DNT> 페이지에 하나의 문제만 표시됩니다. [이슈 생성 환경설정](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#issue-creation) 에 대해 자세히 알아보세요.

<figcaption>
  문제 설정에서 기본 옵션 <DNT>**One issue per policy**</DNT> 을 선택하면 해당 정책의 조건에 따라 열린 모든 공지 이벤트가 동일한 문제로 그룹화됩니다.
</figcaption>

### 조건당 하나의 문제 [#preference-condition]

개별 문제 기록이 특정 조건에 초점을 맞추도록 하려면 <DNT>**One issue per condition**</DNT> 옵션을 선택합니다. 정책에 여러 조건이 있는 경우 <DNT>**Issues &amp; activity**</DNT> 페이지에서 각 조건에 대한 별도의 문제 기록을 볼 수 있습니다. [이슈 생성 기본 설정](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#issue-creation) 에 대해 자세히 알아보세요.

* 신호가 조건의 희소가치, 경계를 위반하는 경우 공지 이벤트가 열리고 이슈 기록이 생성됩니다.
* 동일한 신호가 다른 조건의 옛값을 위반하면 또 다른 공지 이벤트가 열리고 또 다른 이슈 기록이 생성됩니다.
* 이슈가 발생한 후 동일한 조건에 대해 추가 신호가 알림 이벤트를 열면, 이러한 알림 이벤트는 해당 조건에 대해 이미 열린 이슈 레코드에 추가됩니다.

<img width="40%;" class="center" title="A diagram showing how the <DNT>**One issue per condition**</DNT> option works" alt="A diagram showing how the <DNT>**One issue per condition**</DNT> option works" src="/images/alerts_diagram_by-condition.webp" />

<figcaption>
  이슈 기본 설정으로 옵션 <DNT>**One issue per condition**</DNT> 을 선택하면 특정 조건에 대한 모든 신호별 공지 이벤트가 별도의 이슈로 그룹화됩니다.
</figcaption>

### 조건 및 신호당 하나의 문제 [#preference-signal]

<DNT>**One issue per condition and signal**</DNT> 옵션은 문제 레코드 생성을 위한 가장 세부적인 수준입니다. 인프라 전체에서 발생하는 모든 것을 면밀히 모니터링하려면 이 문제 기본 설정을 선택하십시오. [이슈 생성 기본 설정](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#issue-creation) 에 대해 자세히 알아보세요.

<img width="40%;" class="center" title="A diagram showing how the <DNT>**One issue per condition and signal**</DNT> option works" alt="A diagram showing how the <DNT>**One issue per per condition and signal**</DNT> option works" src="/images/alerts_diagram_by-signal.webp" />

<figcaption>
  이슈 선호도에서 <DNT>**One issue per condition and signal**</DNT> 선택하면 동일한 조건과 신호를 공유하는 공지 이벤트가 하나의 이슈로 그룹화됩니다. 이슈가 발생한 후 조건에 대한 동일한 신호가 다른 공지 이벤트를 열면(예: 동일한 신호에 대해 경고 공지 이벤트가 열렸다가 잠시 후에 중요한 알림 이벤트가 이어질 수 있음) 해당 새 공지 이벤트가 동일한 이슈에 추가됩니다. 선택한 조건/신호 조합에 대해 <DNT>**Issues &amp; activity**</DNT> 페이지에는 하나의 문제만 표시됩니다.
</figcaption>
