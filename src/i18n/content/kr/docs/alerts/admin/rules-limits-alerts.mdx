## NRDB 경고 쿼리는 분당 데이터 포인트와 일치했습니다. [#query-limit]

[](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/understand-technical-concepts/streaming-alerts-key-terms-concepts)알림 조건 `Matched data points per minute` 제한은 New Relic [계정](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure)의 알림 쿼리에 대해 일치하는 데이터 포인트 의 총 비율에 적용됩니다.

이 한도를 초과하면 요금이 한도 아래로 떨어질 때까지 영향을 받는 계정에 대한 조건을 만들거나 업데이트할 수 없습니다. 기존 공지 조건은 영향을 받지 **않습니다** .

[한도 UI](/docs/data-apis/manage-data/view-system-limits) 에서 일치하는 데이터 포인트와 한도 인시던트를 볼 수 있습니다.

처리량이 가장 많은 조건을 이해하려면 다음과 같은 쿼리를 수행할 수 있습니다.

```sql
FROM NrAiSignal
SELECT sum(aggregatedDataPointsCount) AS 'alert matched data points'
FACET conditionId
```

일치하는 데이터 포인트를 최적화하기 위한 몇 가지 팁:

* [슬라이딩 윈도우를](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions/#sliding-window-aggregation)사용하는 경우 이로 인해 데이터 포인트 수가 크게 증가할 수 있습니다. 데이터 포인트 수를 줄이려면 더 긴 집계 기간을 사용할 수 있습니다.
* `WHERE` 절을 사용하여 경고되는 데이터의 양을 줄입니다. 경우에 따라 `FACET` 대신 `WHERE` 사용하면 더 효율적인 알림을 생성할 수 있습니다.
* 유사한 경고를 결합합니다. 유사한 경고 조건이 여러 개 있는 경우 결합된 필터로 함께 그룹화하는 것이 좋습니다.

한도 증가를 요청하려면 New Relic 계정 담당자에게 문의하십시오.

[슬라이딩 윈도우를](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/create-smoother-charts-sliding-windows) 사용하면 데이터 포인트 수가 크게 증가할 수 있습니다. 생성되는 데이터 포인트 수를 줄이려면 더 긴 기간의 슬라이딩 윈도우 집계를 사용하는 것이 좋습니다.

## 공지 쿼리 분당 스캔 작업 수 [#query-scan-limit]

공지 조건 `Alert query scan operations per minute` 제한은 수집된 이벤트에 대한 쿼리 스캔 작업의 총 속도에 적용됩니다. 쿼리 스캔 작업은 수집된 이벤트를 뉴렐릭 [계정](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) 에 등록된 공지 쿼리와 일치시키기 위해 뉴렐릭 파이프라인에서 수행되는 작업입니다.

이 한도를 초과하면 요금이 한도 아래로 떨어질 때까지 영향을 받는 계정에 대한 조건을 만들거나 업데이트할 수 없습니다. 기존 공지 조건은 영향을 받지 **않습니다** .

[제한 UI](/docs/data-apis/manage-data/view-system-limits) 에서 쿼리 스캔 작업과 모든 제한을 확인할 수 있습니다.

이벤트를 공지 쿼리와 일치시킬 때 쿼리가 참조하는 [데이터 유형](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/#what-you-can-query) 의 모든 이벤트를 검사해야 합니다. 다음은 지정된 데이터 유형에서 이벤트를 줄이는 몇 가지 일반적인 방법입니다(이렇게 하면 공지 쿼리 검사 작업이 감소합니다).

* 로그인 데이터에 대해 경고할 때 [로그인 파티션을](/docs/tutorial-manage-large-log-volume/organize-large-logs/) 사용하여 공지 쿼리에 대해 검색되는 로그인을 제한하세요.

* 맞춤형 대시보드에 대해 경고할 때 더 큰 맞춤형 대시보드 유형을 분할하세요.

* 소니 이벤트에 대해 경고하는 대신 맞춤형 대시보드를 사용하세요.

* 데이터를 집계하기 위해 [지표를 만듭니다](/docs/data-apis/convert-to-metrics/analyze-monitor-data-trends-metrics/) .

* 가능하면 프로세서 이벤트에 대해 경고하는 대신 [인덱스 타임슬라이스 쿼리를](/docs/data-apis/understand-data/metric-data/query-apm-metric-timeslice-data-nrql/) 사용하십시오.

위의 팁 외에도 사용되지 않거나 필요하지 않은 공지 쿼리(공지 조건)를 정리하면 쿼리 검색 작업 수가 줄어듭니다.

한도 증가를 요청하려면 New Relic 계정 담당자에게 문의하십시오.

참고: 뉴렐릭 서비스 중단 중에는 알림을 일시적으로 이용할 수 없습니다.

## 제한의 허용 가능한 사용

뉴렐릭의 [허용 가능한 사용 정책](/docs/licenses/license-information/general-usage-licenses/acceptable-use-policy) 에 따라, 우리의 컴퓨팅을 압도하는 공지 조건은 뉴렐릭 관리자에 의해 비활성화될 수 있습니다.

뉴렐릭 관리자가 조건을 비활성화했는지 확인하기 위해 자체 알림을 설정하려면 [NrIntegrationError](/docs/data-apis/manage-data/nrintegrationerror/#query-alert) 예제를 참조하세요.
