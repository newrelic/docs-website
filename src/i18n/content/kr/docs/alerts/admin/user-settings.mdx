뉴렐릭 알림의 경우 유예 기간 설정을 통해 문제에 대한 알림을 받기 전까지 경과해야 하는 시간을 제어할 수 있습니다. 비활성 문제 설정은 비활성 문제가 자동으로 닫힐 때까지 기다리는 시간을 제어합니다.

이 설정은 계정으로 들어오는 모든 데이터에 적용됩니다.

알림 설정을 찾으려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Alerts > General**</DNT> 으로 이동하세요.

## 결정을 사용하는 계정에 대한 유예 기간 정책 [#grace-period]

상관 관계 논리의 후보인 문제의 경우 유예 기간을 설정할 수 있습니다. 유예 기간은 이슈가 생성된 시점과 해당 이슈가 알림을 생성하는 시점 사이의 시간입니다. 유예 기간은 불필요한 알림 수를 줄이는 데 사용됩니다.

각 우선 순위 수준에 대해 유예 기간을 0분에서 20분까지 설정할 수 있습니다. 유예 기간이 길수록 시스템에서 새로운 활동을 분석하고 과거 활동과 연관시키는 데 더 많은 시간이 필요합니다.

보낼 알림이 있는 경우 유예 기간이 경과하면 알림이 전송됩니다.

등급 기간 기본값은 다음과 같습니다.

* 낮음: 15분
* 중간: 10분
* 높음: 5분
* 크리티컬: 1분

<DNT>
  **Note: Grace period is eligible only for users who use correlations**
</DNT>

## 자동 닫기 문제 창 [#auto-close-issue-window]

자동 종료 문제 설정은 비활성 문제가 자동으로 종료되기 전에 경과되는 시간을 제어합니다.범위는 1-30일이며 기본값은 3일입니다.

<Callout variant="tip">
  이 설정은 각 조건에 대한 ["... 이후 미결 사건 닫기" 설정](/docs/alerts-applied-intelligence/new-relic-alerts/alert-incidents/how-alert-condition-incidents-are-closed/#time-limit) 과 관련됩니다.

  이 두 설정의 기간이 다른 경우 당사 시스템은 설정에 관계없이 더 짧은 기간을 사용합니다. 예를 들어, 열린 인시던트 종료 시간 설정이 3일이고 비활성 이슈 시간 설정이 2일인 경우 이슈(모든 관련 인시던트 포함)가 2일 후에 종료됩니다.
</Callout>

## 비활성 문제 [#inactiv-issue]

문제가 비활성화되면 시스템에서 집계되거나 상관되지 않습니다. 예를 들어 비활성 문제와 유사한 새 문제가 발생하면 비활성 문제와 상관 관계가 있는 것이 아니라 새 문제로 처리됩니다.

이슈가 5,000건의 인시던트를 수집하면 비활성 상태로 간주됩니다.

30일이 지나면 비활성 문제가 자동으로 종료됩니다.
