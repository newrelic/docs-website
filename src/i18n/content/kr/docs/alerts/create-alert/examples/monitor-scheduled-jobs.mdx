[중요한 임계값](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/advanced-techniques/set-thresholds-alert-condition) 및 [신호 손실 설정](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions#signal-loss) 을 사용하여 예약된 작업을 모니터링하고 발생하지 않을 때 알림을 받을 수 있습니다.

예를 들어 시스템에 24시간마다 유지 관리가 예약되어 있는 경우 해당 유지 관리가 수행되지 않는 시점을 알고 싶습니다.

이 기술은 최대 신호 손실 임계값 시간인 48시간보다 더 자주 발생하는 정기적으로 예약된 작업에 대해서만 작동합니다.

## 당신이 알아야 할 것은

이 절차를 최대한 활용하려면 [조건을 만드는](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) 방법을 알아야 합니다.

## 작동 원리

일반적으로 조건은 인시던트를 트리거하기 위해 중요한 레버 값을 사용합니다. 하지만 신호가 끊어져도 인지던트가 작동할 수 있습니다. 예약된 작업을 모니터링하고 해당 작업이 발생하지 않을 경우 알림을 받으려면 작업 일정에 맞게 신호 손실 타이머를 설정하세요.

## 예약된 작업 모니터링

1. 모니터링할 작업과 관련된 [조건을 만듭니다](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) .
2. <DNT>**+ Add lost signal threshold**</DNT> 클릭한 다음 모니터링되는 작업의 예약된 주기보다 길게 신호 만료 기간을 설정합니다.
3. <DNT>**Add lost signal threshold**</DNT> 확인하세요.
4. 저장하기 전에 조건의 이름을 지정했는지 확인하십시오.
