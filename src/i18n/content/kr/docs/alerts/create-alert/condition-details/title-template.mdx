공지 이벤트가 조건에 따라 열릴 때 제목 템플릿이 사용됩니다. 기본 제목을 덮어씁니다.

## 제목 템플릿을 사용하는 이유는 무엇입니까? [#why-titles]

공지조건은 공지이벤트 생성의 기반이 됩니다. 안내 모드를 사용하시든 차트나 정책에서 조건을 직접 작성하시든, 고유한 제목 템플릿을 사용하는 것은 공지 조건을 생성하는 마지막 단계입니다.

제목 템플릿 사용은 선택 사항이지만 사용을 권장합니다. 공지 조건은 귀하가 감시하고자 하는 오래된 값, 릴레이의 집합을 정의합니다. 해당 레버 값 중 하나라도 위반되면 공지 이벤트가 생성됩니다. 의미 있는 제목 템플릿을 사용하면 문제를 정확히 파악하고 장애를 더 빠르게 해결할 수 있습니다.

## 시작하기 전에 [#begin]

제목 템플릿을 사용하기 전에 알아두어야 할 몇 가지 주요 사항은 다음과 같습니다.

* **호환성**: 타이틀 템플릿 기능은 NRQL 조건에서만 사용할 수 있습니다.
* **글자 수 제한**: 타이틀 템플릿의 최대 길이는 256자입니다.
* **형식**: 제목 템플릿은 유효한 [핸들바](https://handlebarsjs.com/guide/) 형식을 따라야 합니다.
* **분실된 신호 공지 이벤트**: 분실된 신호로 인해 발생하는 공지 이벤트의 경우 항상 기본 제목이 사용됩니다. 신호 손실에 대한 자세한 내용은 [신호 손실 및 공백 복구를](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/) 참조하십시오.

## 제목 템플릿 이해 [#example]

예를 들어 전자상거래 사이트를 운영하고 있고, 네트워크 오류가 많이 발생할 때 공지 이벤트를 트리거하는 공지 조건을 만들었다고 가정해 보겠습니다. 제목 템플릿을 사용하면 이 조건에서 공지 이벤트를 쉽게 식별할 수 있습니다.

이 공지 조건의 이름을 `High Network Errors` 로 지정하셨습니다. 하지만 공지 이벤트 제목에 부품 및 우선순위에 대한 추가 정보가 반영되기를 원합니다.

**예** :

```handlebars
{{conditionName}} Affecting {{entity.name}} - {{entity.type}} - {{priority}}
```

이 템플릿은 다음과 같은 공지 이벤트 제목을 생성합니다.

```
High Network Errors Affecting Entity123 - Kubernetes Pod - Critical
```

## 제목 템플릿 사용 [#use-titles]

UI를 통하거나 GraphQL을 사용하는 두 가지 방법으로 제목 템플릿과 상호 작용할 수 있습니다. 방법을 알아보려면 다음 단계를 따르세요.

<CollapserGroup>
  <Collapser id="template-ui" title="UI에서 제목 템플릿 만들기">
    1. NRQL 조건 빌더로 이동합니다. 지침은 [공지 조건을](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/alert-conditions/) 참조하세요.
    2. NRQL 조건을 만듭니다.
    3. 마지막 단계인 **Add details** \[세부 정보 추가] 에서 **Title template** \[제목 템플릿] 필드에 제목 템플릿을 입력합니다.
    4. 조건을 저장합니다.
  </Collapser>

  <Collapser id="template-ui" title="GraphQL에서 제목 템플릿 생성">
    GraphQL에서 제목 템플릿 필드는 `titleTemplate` 이라고 합니다. 다음은 제목 템플릿을 추가하기 위해 NRQL 정적 조건을 업데이트하는 변형의 예입니다.

    ```graphql
    mutation {
      alertsNrqlConditionStaticUpdate(
        accountId: 123456
        id: 456789
        condition: {titleTemplate: "{{conditionName}} has triggered an incident"}
      ) {
        titleTemplate
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## 속성을 사용하여 조건을 추가로 정의하세요. [#attributes]

더 많은 컨텍스트를 추가하기 위해 제목 템플릿에 공지 이벤트 속성의 하위 집합을 포함할 수 있습니다.

방법은 다음과 같습니다.

* **`{{AN_ATTRIBUTE}}`형식을 사용합니다**. 예를 들어:

  ```handlebars
  {{conditionName}} has triggered an alert event
  ```

* **태그 속성**: 태그의 경우 `{{tags.TAG_NAME}}` 형식을 사용합니다.

* **패싯 절 태그**: NRQL 조건이 `FACET` 절을 사용하는 경우 `tags` 형식을 사용하여 제목 템플릿에서 해당 값을 사용할 수 있습니다. 예를 들어 NRQL 쿼리에 `FACET hostName, cluster` 포함된 경우 다음을 사용할 수 있습니다.

  ```handlebars
  Host {{tags.hostName}}, cluster {{tags.cluster}} alert event
  ```

태그 이름에는 공백을 포함할 수 없지만, 확장된 값에는 공백을 포함할 수 있습니다. 사용 가능한 속성의 전체 목록은 [공지 이벤트 속성을](/docs/alerts/create-alert/condition-details/alert-event-attributes) 참조하세요.
