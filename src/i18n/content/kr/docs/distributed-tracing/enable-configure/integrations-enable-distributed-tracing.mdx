원격 측정 도구 Kamon, OpenTelemetry 또는 AWS X-Ray를 사용하는 경우 원격 측정 통합을 통해 해당 데이터를 New Relic으로 가져올 수 있습니다.

## 샘플링 고려 사항 [#sampling]

분산 시스템은 많은 추적 데이터를 생성할 수 있으므로 원격 분석 도구는 데이터 샘플링(필터링)에 의존합니다. 추적 데이터를 보고하는 [원격 분석 통합](/docs/integrations/open-source-telemetry-integrations) 을 설치할 때 [무한 추적](https://newrelic.com/products/edge-infinite-tracing) 을 활성화하는 옵션이 있습니다.

무한 추적을 선택하면 원격 분석 도구에서 샘플링을 구성하는 방법에 영향을 미칩니다.

* **무한 추적을 사용하지 않는 표준 설치:** 표준 설치에서는 원격 측정 도구가 추적 데이터를 보내기 전에 추적 데이터를 샘플링하기를 원한다고 가정합니다. (추적 데이터가 Trace API 제한을 초과하는 경우 추가 샘플링을 수행할 수도 있습니다.)
* **무한 추적으로 설치:** 무한 추적(읽기 [요구 사항](/docs/understand-dependencies/distributed-tracing/infinite-tracing/introduction-infinite-tracing#requirements) )을 선택하면 원격 측정 도구의 샘플링이 100%로 설정되어 해당 도구의 모든 추적 데이터가 당사로 전송된다고 가정합니다. 추적 관찰자는 [꼬리 기반 샘플링](/docs/understand-dependencies/distributed-tracing/get-started/how-new-relic-distributed-tracing-works#tail-based) 을 사용하여 가장 중요하고 실행 가능한 추적을 선택하고 해당 데이터는 Trace API를 통해 수집됩니다.
