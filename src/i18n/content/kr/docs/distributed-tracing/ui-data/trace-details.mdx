      * 엔터티에 1초 동안 병렬로 실행되는 3개의 프로세스가 있고 그 중 어느 것도 외부 호출을 하지 않은 경우 추적에 1초의 대기 시간이 기여하는 것으로 표시됩니다.

      폭포 보기에는 관심 있는 범위를 더 쉽게 찾을 수 있는 여러 컨트롤이 있습니다.

      * <DNT>**Standard/Manual**</DNT>: 표준 모드에서는 각 프로세스의 첫 번째 범위를 포함하는 반확장 보기가 제공됩니다. 수동 모드에서는 확장한 것만 볼 수 있습니다.

        * 표준 또는 수동 모드에서 <DNT>**Collapse all**</DNT> 클릭하면 폭포식 구조가 사용 중인 모드의 기본값으로 재설정됩니다.

      * <DNT>**Focus on**</DNT>스팬에 오류가 있거나 이상한 부분이 있으면 이러한 유형 각각에 대한 확인란이 표시됩니다. 이 옵션을 선택하면 워터폴 뷰에서 다른 모든 유형의 스팬이 축소되고 색상으로 구분된 오류 또는 비정상 스팬만 표시됩니다. 이는 문제가 있는 구간을 빠르게 찾는 방법입니다.

## 스팬 세부 정보 창 [#span-details]

스팬을 선택하면 스팬 세부 정보가 포함된 창이 열립니다. 이러한 세부 정보는 성능 문제를 해결하는 데 도움이 될 수 있습니다. 이 페이지에는 세 개의 탭이 있습니다.

* <DNT>**Performance**</DNT>차트를 통해 해당 스팬 작업의 평균 지속 시간과 처리량은 물론, 이 특정 스팬의 성능이 평균과 어떻게 비교되는지도 확인할 수 있습니다.
* <DNT>**Attributes**</DNT>: [데이터 사전](/attribute-dictionary/?dataSource=Distributed+Tracing&event=Span) 에서 속성에 대해 자세히 알아볼 수 있습니다.
* <DNT>**Details**</DNT>: [스팬의 전체 이름](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui/#prettified-span-names) , 데이터 소스 등의 세부정보를 볼 수 있습니다.
* <DNT>**Span links**</DNT>해당 스팬에 [링크가](#span-links) 포함되어 있으면 이 탭이 표시됩니다. 다른 경로의 범위와의 인과 관계를 보여줍니다. [span 링크에 대해 자세히 알아보세요](#span-links).

스팬이 표시하는 내용은 스팬 유형을 기반으로 합니다. 예를 들어 데이터 저장소 범위의 세부 정보에는 데이터베이스 쿼리가 포함됩니다. 추적 구조 및 범위 속성이 결정되는 방법에 대한 자세한 내용은 [추적 구조를](/docs/understand-dependencies/distributed-tracing/get-started/how-new-relic-distributed-tracing-works#trace-structure) 참조하세요.

## 스팬 링크 이해하기 [#span-links]

스팬 링크는 서로 다른 트레이스에 걸쳐 있는 스팬 간의 인과 관계를 이해하는 데 도움이 됩니다. 트레이스가 통신 또는 장기 실행 프로세스로 인해 분할되는 경우 스팬 링크를 사용하면 관련 트레이스 간을 탐색하고 서버의 전체 그림을 볼 수 있습니다.

### 트레이스가 분할되는 경우 [#why-traces-split]

현대의 분산 시스템에서는 다음과 같은 시나리오에서 트레이스가 분리되고 연결이 끊긴 트레이스로 나타날 수 있습니다.

* <DNT>**Long-running workflows**</DNT>: 트레이스 기간 제한(헤드 기반의 경우 90초 이상, Infinite Tracing의 경우 20초 이상)을 초과하거나 장기간 실행되는 백그라운드 작업 또는 워크플로우입니다.

메시지를 추적할 때 일반적으로 생산자와 소비자의 트레이스를 연결하는 스팬 링크를 사용하는 것이 좋습니다. 예시로는 다음과 같은 것들이 있습니다.

* <DNT>**General asynchronous message queues**</DNT>서비스가 메시지를 큐(예: AWS SQS, RabbitMQ 또는 Kafka)에 게시하면 다른 서비스가 나중에 해당 메시지를 소비합니다.
* <DNT>**Batch-processing**</DNT>: 여러 메시지를 일괄 처리하는 작업자, 각 메시지는 서로 다른 전송원에서 전송됨
* <DNT>**Fan-in pattern**</DNT>여러 서비스가 독립적으로 메시지를 생성하고, 하나의 소비자가 이를 취합합니다.
* <DNT>**Fan-out pattern**</DNT>: 여러 소비자가 하나의 메시지에서 소비 - 각각 자체 트레이스 시작

이렇게 관계가 분리되면 전통적인 부모-자녀 관계로는 전체적인 상황을 파악할 수 없습니다. Span 링크는 명확한 인과 관계를 생성함으로써 이러한 격차를 해소합니다.

### span 링크란 무엇인가요? [#what-are-span-links]

스팬 링크는 직접적인 부모-자식 연결이 없는 스팬 간의 인과 관계를 생성하는 OpenTelemetry 속성입니다. 이를 통해 다음을 수행할 수 있습니다.

* 소비 영역에서 생산 영역으로 다시 이동합니다.
* 전송이 분할되는 경우에도 전체 요청 흐름을 이해하세요.
* 비동기 경계를 넘나드는 디버깅 문제

스팬 링크에는 트레이스 ID와 관련 스팬의 스팬 ID가 포함되어 있어 뉴렐릭이 트레이스를 연결할 수 있습니다.

### span 링크를 사용하여 트레이 식별 [#identify-span-links]

거래 상세 정보 UI 에서 스팬 링크가 포함된 거래는 여러 가지 방식으로 표시됩니다.

1. <DNT>**Span links filter badge**</DNT>: 트레이스 필터 바에는 링크가 있는 스팬의 수를 나타내는 배지가 표시됩니다(예: `span links (2)`).
2. <DNT>**Span count indicator**</DNT>스팬 목록을 볼 때, 각 스팬에 링크가 있는 경우 해당 스팬의 링크 수가 표시됩니다.

<img title="Distributed tracing span links filter" alt="Screenshot showing the span links filter badge in the trace details UI" src="/images/distributed-tracing-span-links-filter-badge.webp" />

<figcaption>
  스팬 링크 필터 배지는 트레이스에 링크가 있는 스팬이 포함된 경우 트레이스 필터 막대에 나타납니다.
</figcaption>

### 스팬 링크 보기 [#view-span-links]

스팬 링크를 보고 탐색하려면 다음을 참조하세요.

1. 트레이스 세부정보 페이지의 스팬 링크 드롭다운 메뉴에서 스팬을 선택하세요.

2. 스팬 세부 정보 창에서 <DNT>**Span links**</DNT> 탭을 클릭합니다.

3. 다음과 같은 정보가 포함된 연결된 트레이 목록이 표시됩니다.

   * <DNT>**Direction badge**</DNT>: `Forward` 시간상 나중에 발생한 기간(후속 기간)을 나타내고, `Backward` 시간상 이전에 발생한 기간(선행 기간)을 나타냅니다.
   * <DNT>**Trace ID**</DNT>연결된 트레이스의 ID입니다(표시를 위해 일부 생략됨).
   * <DNT>**Timestamp**</DNT>연결된 스팬이 발생했을 때.
   * <DNT>**Duration**</DNT>연결된 구간이 이동하는 데 걸린 시간입니다.
   * <DNT>**Errors**</DNT>: 연결된 전송의 오류 수.

4. 연결된 트레이를 클릭하면 해당 트레이로 이동합니다. 브라우저의 뒤로 가기 버튼을 사용하여 원래 페이지로 돌아가세요.

<img title="Span links tab showing linked traces" alt="Screenshot showing the Span links tab with forward and backward linked traces" src="/images/distributed-tracing-span-links-tab-detail.webp" />

<figcaption>
  Span 링크 탭에는 방향 배지(앞/뒤), 트레이스 ID, 타임스탬프, 소요 시간 및 오류 횟수와 함께 연결된 트레이가 표시됩니다.
</figcaption>

<Callout variant="tip">
  <DNT>**Understanding forward vs. backward links**</DNT>

  * <DNT>**Forward links**</DNT>현재 기간의 결과로 발생한 트레이스를 가리키세요. 예시: 이 스팬에서 메시지가 게시되었고 나중에 다른 트레이스에서 처리되었습니다.
  * <DNT>**Backward links**</DNT>: 현재 스팬을 발생시킨 트레이스를 가리킵니다. 예시: 이 스팬은 이전 트레이스에서 게시된 메시지를 처리하고 있습니다.
</Callout>

### span 링크가 표시되지 않을 때 [#span-links-troubleshooting]

span 링크가 표시될 것으로 예상했지만 나타나지 않는 경우 다음 사항을 확인하십시오.

<img title="Empty state when trace has no span links" alt="Screenshot showing the empty state message when a trace has no span links" src="/images/distributed-tracing-span-links-empty-state.webp" />

<figcaption>
  태그에 스팬 링크가 없으면 스팬 링크 탭에 이 메시지가 표시됩니다.
</figcaption>

<CollapserGroup>
  <Collapser id="trace-context-not-propagated" title="트레이스 컨텍스트가 전파되지 않음">
    스팬 링크에는 메시징 시스템을 통한 적절한 트레이스 실행이 필요합니다.

    * 메시지 헤더에 W3C 추적 컨텍스트 헤더(`traceparent`, `tracestate`)가 포함되어 있는지 확인하십시오.
    * 메시징 라이브러리 또는 프레임워크가 컨텍스트 전파를 지원하는지 확인하십시오.
    * 수동으로 계측할 경우 메시지에서 컨텍스트를 추출하고 스팬 링크를 프로그래밍 방식으로 생성해야 합니다.

    구현 예시는 [OpenTelemetry 스팬 링크 모범 사례를](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/#span-links) 참조하세요.
  </Collapser>

  <Collapser id="sampling" title="샘플링 고려 사항">
    스팬 링크가 나타나려면 원본 트레이스와 연결된 트레이스 모두에서 샘플링이 이루어져야 합니다.

    * 샘플링 과정에서 두 트레이스 중 하나라도 누락되면 스팬 링크가 표시되지 않습니다.
    * 중요한 트레이스를 보존하는 헤드 기반 또는 테일 기반 전략을 사용하는 것을 고려하십시오.
    * 스팬 링크가 있는 중요한 스프레드우의 경우 샘플링 속도를 조정해야 할 수 있습니다.
  </Collapser>

  <Collapser id="data-processing" title="데이터가 아직 처리되지 않았습니다">
    데이터 전송 후 스팬 링크가 나타나는 시점 사이에 지연이 발생할 수 있습니다.

    * 데이터 처리 및 색인 생성에 1～2분 정도 소요됩니다.
    * 연결된 두 개의 트랜스가 모두 성공적으로 주입되었는지 확인하십시오.
    * 데이터 사용량 제한이나 요금 제한에 걸리지 않는지 확인하세요.
  </Collapser>
</CollapserGroup>

측정, 로그에서 스팬 링크를 구현하는 방법에 대한 자세한 내용은 [트레이스에 대한OpenTelemetry 모범 사례 가이드를](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/#span-links) 참조하세요.

## 스팬 속성 [#span-attributes]

`Span` 데이터에 대해 자세히 알아보려면:

* [쿼리 빌더를](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 사용하여 스팬 데이터를 탐색합니다.
* 스팬 데이터에 첨부된 기본 속성을 보려면 [데이터 사전](/attribute-dictionary/?event=Span) 을 사용하십시오.
* 이러한 특성을 사용하는 NRQL 쿼리에 대한 도움말은 이 [예제 쿼리를](/docs/apm/distributed-tracing/ui-data/example-insights-queries-distributed-trace-data) 확인하세요.

## 관련 로그 보기 [#view-your-logs]

[컨텍스트 기능의 로그를](/docs/logs/logs-context/configure-logs-context-apm-agents/) 사용하는 경우 추적에 연결된 모든 로그를 볼 수 있습니다.

1. 추적을 클릭하여 추적 세부 정보 페이지로 이동합니다.
2. 스팬 세부정보 페이지의 왼쪽 상단에서 <DNT>**Logs**</DNT> 탭을 클릭합니다.
3. 개별 로그 메시지와 관련된 자세한 내용은 메시지를 직접 클릭하십시오.
