시작하기 전에 [샘플링 유형에 대한 문서를](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works) 검토하여 표준 분산 추적을 사용할지 [무한 추적을](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing) 사용할지 결정하는 것이 좋습니다. 또한, 현재 뉴렐릭 <InlinePopover type="apm"/>에이전트를 사용하고 있고 방사 추적을 활성화하려는 경우 [계획 가이드를](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide) 참조하세요.

분산 추적을 설정하려면 다음 세 가지 일반적인 단계를 완료합니다.

<img
  title="Diagram showing the three steps to set up distributed tracing."
  alt="Diagram showing the three steps to set up distributed tracing"
  src="/images/distributed-tracing_diagram_high-level-dist-tr.webp"
/>

1. <DNT>**Identify services:**</DNT> 이 요청을 완료하는 데 사용되는 엔드포인트, 서비스, 언어 및 시스템을 식별하고 기록해 둡니다. 다음 단계에서 이 정보가 필요합니다. 다음과 같은 환경 다이어그램이 있는 경우 이를 사용하여 요청을 처리하는 서비스 목록을 생성할 수 있습니다.

   <img
     title="A sample environment diagram showing how you might identify services."
     alt="A sample environment diagram showing how you might identify services."
     src="/images/distributed-tracing_screenshot-full_distributed-tracing-map.webp"
   />

2. <DNT>**Instrument services:**</DNT> 트레이스 데이터를 보낼 수 있도록 식별하는 각 서비스를 로그합니다. <InlinePopover type="apm"/>에이전트, 용량 추가 서비스와 같은 일부 도구는 자동으로 서비스를 제공하지만 다른 도구에서는 서비스에 일부 코드를 삽입해야 합니다. 측정, 반환 단계를 보려면 아래 아이콘을 클릭하세요.
