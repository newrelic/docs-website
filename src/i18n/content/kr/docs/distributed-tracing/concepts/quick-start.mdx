시작하기 전에 표준 분산 추적 또는 [무한 추적을](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing) 사용할지 결정하기 전에 [샘플링 유형에 대한 문서를](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works)검토하는 것이 좋습니다. 또한 현재 New Relic을 사용하고 계시다면 <InlinePopover type="apm"/>분산 추적을 활성화하려면 [계획 가이드 를](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide)참조하십시오.

시작할 준비가 되셨습니까? 아직 하지 않았다면 아래에서 무료 New Relic 계정을 만들어 오늘 데이터 모니터링을 시작하십시오.

<InlineSignup/>

분산 추적을 설정하려면 다음 세 가지 일반적인 단계를 완료합니다.

<img
  title="Diagram showing the three steps to set up distributed tracing."
  alt="Diagram showing the three steps to set up distributed tracing"
  src={distributedtracingHighLevelDistTr}
/>

1. **서비스 식별:** 이 요청을 완료하는 데 사용되는 엔드포인트, 서비스, 언어 및 시스템을 식별하고 기록해 둡니다(다음 단계에서 이 정보가 필요함). 다음과 같은 환경 다이어그램이 있는 경우 이를 사용하여 요청을 처리하는 서비스 목록을 만들 수 있습니다.

<img
  title="A sample environment diagram showing how you might identify services."
  alt="A sample environment diagram showing how you might identify services."
  src={distributedtracingDistributedTracingMap}
/>

2. **Instrument services:** 추적 데이터를 보낼 수 있도록 식별하는 각 서비스를 계측합니다. 다음과 같은 일부 도구 <InlinePopover type="apm"/>다른 도구는 서비스에 일부 코드를 삽입해야 하는 반면 에이전트, 계측 서비스는 자동으로 제공됩니다. 계측 단계를 보려면 아래 아이콘을 클릭하십시오.
