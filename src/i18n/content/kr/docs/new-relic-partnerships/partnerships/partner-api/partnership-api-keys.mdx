[파트너십 API](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) 키는 New Relic에서 파트너십 계정으로 호출을 인증하는 데 사용됩니다. API 키로 작업할 때 중요한 차이점은 계정 대 파트너 관계 소유자입니다. 파트너쉽 소유자는 계정에 대한 데이터를 관리하는 데 사용되는 특별한 형태의 계정입니다.

## 요구 사항 [#requirements]

**New Relic 담당자가 사용해야 한다고 특별히 지시하지 않는 한 Partnership API는 조직에서 사용할 수 없습니다.**

요구 사항 [은 파트너십 API 소개를](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) 참조하세요.

## 라이센스 키

계정을 생성할 때 XML로 반환되는 키입니다. 에이전트가 계정을 식별하는 데 사용합니다.

## 파트너 API 키 [#partner-key]

파트너십 API를 사용할 때 파트너십 소유자 계정의 REST API 키를 사용해야 합니다. 이것은 `x-api-key: <api key>` 이 표시되는 API 호출에 사용되는 키입니다.

## API 키 찾기 [#locating-keys]

키를 보거나 재생성하려면 파트너십 소유자 자격 증명을 사용하여 New Relic 사이트에 로그인한 다음 **Partnerships > Edit settings** 으로 이동하십시오. 키는 페이지 하단에 나열됩니다.

<Callout variant="caution">
  재생성된 키는 되돌릴 수 없습니다.
</Callout>

## 보안 모범 사례 [#security]

**권장 사항:** New Relic 사이트에 액세스하는 데 사용되는 키는 보안 모범 사례로 정기적으로 재생성해야 합니다. 키 재생성 변경은 즉시 적용됩니다.

키를 코드에 하드 코딩하고 새 키가 포함된 새 통합 버전을 프로덕션으로 푸시하려는 경우 키 재생성과 사이트 업데이트 간에 조정해야 합니다. 이렇게 하면 가동 중지 시간을 최소화하는 데 도움이 됩니다. 이 상황에서는 Show Account API를 사용하여 이러한 키를 동적으로 가져오는 것을 고려하십시오.
