## 머리글 및 바닥글 [#headers-footers]

머리글과 바닥글을 구현하려면:

1. &lt;div> 태그로 묶인 콘텐츠로 구성된 머리글 및/또는 바닥글 페이지를 만듭니다. 페이지에 CSS를 가져오지 않거나 인라인 CSS가 포함되어 있지 않은지 확인하십시오.

2. 머리글과 바닥글에 필요한 CSS를 가져와 별도의 CSS 파일에서 사용할 수 있도록 합니다. 예를 들어, New Relic의 표준 스타일 정의를 무시하지 않으려면 `yourcompany_` 과 같은 사용자 지정 문자열을 다양한 스타일 이름 앞에 추가합니다.

3. 파트너가 호스팅하는 공개적으로 액세스 가능한 위치에 3개의 파일을 게시합니다. 고객에게 혼합 콘텐츠 경고가 표시되지 않도록 https를 사용합니다. 예를 들어:

   ```
   https://yourcompany.com/newrelic_header.htm
   https://yourcompany.com/newrelic_footer.htm
   https://yourcompany.com/newrelic_styles.css
   ```

4. 머리글, 바닥글 및 CSS 페이지 위치에 대한 URL을 New Relic **Partnership** 콘솔의 해당 필드에 입력합니다.

<Callout variant="tip">
  헤더를 보려면 파트너십의 계정 중 하나에 로그인하십시오.
</Callout>

## 레이아웃 및 스타일 [#styles]

머리글 및 바닥글 레이아웃과 스타일은 각 페이지 출력에 삽입하기 위해 New Relic의 수집기 서버에 로드 및 캐시됩니다. 파트너 사이트의 레이아웃 업데이트는 매시간 자동으로 새로 고쳐집니다.

파트너 웹사이트에 대한 사용자 지정 링크가 레이아웃에 추가될 수 있습니다. 싱글 사인온이 활성화된 경우 파트너는 레이아웃에서 사용하기 위해 고객별 정보를 New Relic 사용자 세션에 삽입할 수 있습니다. 자세한 내용은 [싱글 사인온 및 액세스 제어를](/docs/accounts-partnerships/partnerships/partner-integration-guide/single-sign-access-control) 참조하십시오.

## 하위 도메인

파트너는 선택적으로 사용자 지정 하위 도메인을 표시할 수 있습니다. 예를 들어 **yourcompany.newrelic.com** , New Relic 계정에 액세스하는 고객. 이를 준비하려면 New Relic의 Business Enablement 팀에 문의하십시오.
