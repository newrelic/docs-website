당사의 파트너십 API를 사용하면 New Relic 파트너 및 [고객 파트너십](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) 으로 설정된 New Relic 계정이 계정, 사용자 및 구독 관련 설정을 관리할 수 있습니다.

## 요구 사항 [#requirements]

**Partnership API 사용에는 몇 가지 주요 제한 사항이 있습니다.** New Relic 조직은 New Relic 담당자가 특별히 사용하도록 지시하지 않는 한 이 API를 사용해서는 안 됩니다. 제한 사항을 이해하려면 전체 요구 사항 섹션을 읽으십시오.

Partnership API는 두 가지 유형의 New Relic 계정, 즉 [파트너](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) (관리 서비스 공급자 및 New Relic 리셀러) 및 [고객 파트너십](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) (파트너십 계정 구조에 대한 액세스 권한이 부여된 대규모 조직)에서 사용할 수 있습니다.

파트너십 API 호출에는 파트너십 소유자 계정의 REST API 키와 파트너 ID 모두를 사용한 인증이 필요합니다.

진정한 New Relic 파트너(관리 서비스 제공자 또는 리셀러)인 계정은 Partnership API 사용에 제한이 없습니다. [고객 파트너십](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) 으로 설정된 New Relic 조직은 다음과 같은 제한이 있습니다.

### 고객 파트너십 제한 [#customer-restrictions]

New Relic 조직이 [고객 파트너십](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) 으로 설정된 경우 조직이 최신 [가격 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) 또는 최신 [사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 을 사용하는 경우 몇 가지 제한 사항이 있습니다. 세부:

* **사용자 모델.** New Relic 조직에 [최신 사용자 모델의 사용자](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 가 있는 경우 더 이상 계정 또는 사용자를 관리하는 Partnership API 호출을 사용할 수 없습니다. 대신 사용할 항목에 대한 세부정보는 다음과 같습니다.

  * Partnership API [사용자 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-user-object) 를 사용하는 대신 [NerdGraph를 통해 사용자를 관리합니다](/docs/apis/nerdgraph/examples/nerdgraph-manage-users) .
  * Partnership API [계정 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-account-object) 또는 [하위 계정 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-sub-account-object) 를 사용하는 대신 [NerdGraph를 통해 계정을 관리합니다](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph) .

* **가격 책정 모델.** 최신 [사용량 기반 가격 책정 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) 을 사용 중인 경우 [원래 가격 책정 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) 을 제어하는 API 호출을 사용할 수 없습니다. API 영향은 다음과 같습니다.

  * [구독 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-subscription-object/) 를 사용할 수 없습니다.
  * [NerdGraph 프로비저닝 API를](/docs/apis/nerdgraph/tutorials/provisions-your-subscriptions-nerdgraph/) 사용할 수 없습니다.

가격 책정 모델 또는 사용자 모델을 결정하려면 [가격 책정 및 사용자 모델 변경 사항 개요를](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model) 참조하세요.

## 할 수 있는 일 [#functions]

다음은 이 API로 수행할 수 있는 작업에 대한 개요입니다.

**계정:**

[요구 사항](#requirements) 을 충족하는 일부 조직은 [계정 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-account-object/) 를 사용하여 계정을 만들고 업데이트할 수 있습니다. 이를 사용하여 다음을 수행할 수 있습니다.

* 새로 만들기
* 보여 주다
* 업데이트
* 취소
* 사용량 표시
* 기본 관리자 설정
* 구독 설정

하위 계정을 만들기 위한 [하위 계정 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-sub-account-object) 도 있습니다.

**사용자:**

[요구 사항을 충족하는](#customer-restrictions) 일부 조직에서는 [사용자 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-user-object) 를 사용하여 다음을 수행할 수 있습니다.

* 계정에 사용자 액세스 추가
* 계정에서 사용자 액세스 제거

**신청:**

[요구 사항을 충족](#customer-restrictions) 하고 원래 제품 가격 책정 모델을 사용하는 조직은[구독 개체](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-subscription-object) 를 사용하여 다양한 구독 관련 특성을 구성할 수 있습니다.

### 비밀번호 요구사항 [#password]

계정 생성을 위해 전달된 비밀번호에는 다음과 같은 요구 사항이 있습니다.

* 8-50자 길이
* 문자, 숫자 또는 특수 문자만 포함
* 공백을 포함할 수 없습니다.
* 최소 1글자를 포함해야 합니다.
* 1개 이상의 숫자 또는 특수 문자를 포함해야 합니다.
