우리는 신세틱 모니터링을 사용하여 전 세계의 연결을 시뮬레이션합니다. 소스 포인트와 출력 사이에 중단이 있는 경우 신세틱 검사가 실패하고 서비스 수준 목표(SLO) 점수에 영향을 미치며 매우 중요한 알림이 트리거됩니다. 우리는 이것을 입력 성능이라고 부릅니다.

기본적으로 "업타임"이라고도 하는 애플리케이션에 연결하는 능력으로 입력 성능을 측정합니다. 아래에서 입력 성능 SLI를 설정하는 단계와 시도해 볼 수 있는 NRQL 쿼리 예시를 확인할 수 있습니다.

<Steps>
  <Step>
    ## 당신의 신세틱 검사를 만들어보세요

    가장 일반적인 입력 성능 서비스 레벨은 "연결성" 또는 "업타임"이라고도 합니다. 상태 API 엔드포인트에 대한 간단한 확인 또는 URL 로드입니다. 신세틱 모니터링 서비스를 이용하면 이 두 가지를 모두 수행할 수 있습니다. 서비스 레벨 표시기를 생성하기 전에 [간단한 브라우저 모니터 추가](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#simple) 및 [스크립트 API 테스트 추가를](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#complex) 참조하여 아직 수행하지 않은 경우 데이터 보고를 시작하는 방법을 알아보세요.
  </Step>

  <Step>
    ## 서비스 레벨 지표 만들기

    데이터 보고가 완료되면 서비스 수준 관리 서비스를 사용하여 다음 절차에 따라 입력 지표 및 목표를 생성할 수 있습니다.

    1. [부품 탐색기](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts#find) 에서 **Service levels** 선택하세요.

    2. **+ Add a service level indicator** \[+ 서비스 레벨 표시기 추가 를] 클릭합니다. **Add a service level** \[서비스 레벨 추가] 버튼이 표시되지 않으면 뉴렐릭 관리자에게 권한을 문의하세요.

    3. 아래 스크린샷과 같이 부문 유형을 `Synthetic monitors` 으로 필터링합니다.

       <img
         alt="Filter entity types in service levels"
         src="/images/apm_screenshot-crop_add-input-service-level.webp"
       />

    4. 목록에서 신세틱스 모니터를 찾아 클릭하세요. 그러면 왼쪽 패널의 **Continue** 버튼이 활성화됩니다. **Continue** 을 클릭합니다.

    5. **성공** 서비스 수준에 대한 권장 설정 버튼이 표시됩니다(아래 참조). 클릭하세요.

       <img
         alt="Synthetics input service level guided flow"
         src="/images/apm_screenshot-crop_synthetics-setup.webp"
       />

    6. 필요에 따라 태그, 제목 및 설명을 적절하게 변경합니다.

    7. **저장** 을 클릭합니다.
  </Step>
</Steps>
