## 설정 전용 모드(Flux 제외)

* 이제 클러스터에 Flux 컨트롤러가 없어도 파이프라인 컨트롤 게이트웨이를 설치하고 운영할 수 있습니다. 이 모드에서 뉴렐릭은 UI를 통해 파이프라인 설정(샘플링, 필터링, 변환)을 자동으로 계속 관리하는 반면, 사용자는 Helm을 통해 게이트웨이 인프라(확장, 버전)를 수동으로 관리합니다.
* 이는 권한 범위를 cluster-admin에서 네임스페이스 범위의 ConfigMap 액세스로만 축소하므로, 보안이 제한되고 규정 준수 중심의 환경에 적합합니다.
* 내장된 config-watcher 사이드카는 UI에서 푸시된 설정 변경 사항을 자동으로 감지하고 게이트웨이 파드를 다시 시작합니다 ― 외부 오퍼레이터(예: Stakater Reloader)가 필요하지 않습니다.
* 설치 지침은 [Flux 없이 게이트웨이 설치](/docs/new-relic-control/pipeline-control/gateway/install-gateway-without-flux/)를 참조하십시오.

## DaemonSet 배포 모드

* HPA를 사용하는 배포 대신 Kubernetes DaemonSet(노드당 하나의 파드)으로 게이트웨이를 배포하기 위한 지원이 추가되었습니다. 이는 노드 수준의 텔레메트리 처리 또는 일관된 노드당 리소스 할당이 필요한 환경에 유용합니다.
* Helm 값에서 `daemonset.enabled: true` 을(를) 설정하여 DaemonSet 모드를 활성화합니다. DaemonSet 모드는 설정 전용 설치를 위한 config-watcher 사이드카 및 `customConfigMap` 도 지원합니다.
* DaemonSet 및 배포 모드는 상호 배타적입니다. 차트는 조건부 가드를 사용하여 한 번에 하나의 모드만 활성화되도록 합니다.
