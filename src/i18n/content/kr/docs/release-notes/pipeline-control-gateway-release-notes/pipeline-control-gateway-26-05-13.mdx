## 다중 플릿 지원

* 이제 단일 조직 내에서 여러 게이트웨이 플릿을 생성하고 관리할 수 있습니다. 이를 통해 개별 비즈니스 단위 또는 환경(예: 프로덕션과 스테이징)에서 독립적인 샘플링, 필터링 및 변환 규칙을 유지할 수 있습니다.
* 파이프라인 컨트롤 UI의 새로운 플릿 선택기 드롭다운 메뉴를 사용하여 여러 설정 간에 쉽게 이동할 수 있습니다.

### 업데이트된 YAML 스키마

* 신호별 중첩(예: `logs:`, `spans:`)을 직접 규칙 어레이로 대체하는 간소화된 설정으로 마이그레이션하여, 설정을 더 평면적이고 읽기 쉽게 만들었습니다.
* 필터 프로세서의 경우, 새 스키마는 명시적인 `context` 필드를 지원합니다. 이를 통해 **메트릭 데이터 포인트** 및 **스팬 이벤트**와 같은 데이터 하위 유형을 더 정밀하게 타겟팅할 수 있습니다.

### 개선된 배포 UX

* 새로운 배포 페이지는 이제 배포 이름과 설명에 대한 기본값을 자동으로 채워주어, 변경 사항을 적용하는 데 필요한 수동 단계를 크게 줄여줍니다.
* 변환 프로세서에 대한 좌우 설정 차이점 미리보기를 추가했습니다. 이를 통해 배포를 완료하기 전에 새로운 YAML 로직을 현재 버전과 비교할 수 있습니다.
