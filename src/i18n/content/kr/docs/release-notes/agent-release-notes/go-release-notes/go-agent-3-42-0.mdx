## 3.42.0

### 추가됨

* 프로세서 이벤트, Custom Insights 이벤트, 오류 이벤트 및 로그 이벤트에 대해 저장된 최대 샘플을 완전히 제어할 수 있도록 `ConfigTransactionEventsMaxSamplesStored`및 `ConfigErrorCollectorMaxSamplesStored` 을 추가했습니다.
* nrlambda에서 `events.APIGatewayProxyResponse` 에서 헤더를 추출할 때 `MultiValueHeaders` 속성에 대한 지원이 추가되었습니다.
  * 이 솔루션에 기여해 주신 커뮤니티 멤버 @rittneje에게 감사드립니다.

### 결정된

* 사용되지 않는 변수를 제거하고 `nrpxg5` 통합에서 interface를 any로 대체하여 현대화했습니다.
* 오류 이벤트가 예상됨을 올바르게 표시하지 않는 버그를 수정했습니다.
  * 이 솔루션에 기여해 주신 커뮤니티 멤버 @driimus에게 감사드립니다.
* v1.0.2를 사용하기 위해 `nrwriter` 통합을 범프했습니다.
  * 이 사실을 발견해 주신 커뮤니티 멤버 @hiicharm에게 감사드립니다.

### 지지 성명

우리는 최신 버전의 Go 언어를 사용합니다. 최소한 Go 팀에서 지원하는 것보다 오래된 Go 버전은 사용해서는 안 됩니다. 지원되는 Go 에이전트 버전과 타사 구성 요소에 대한 자세한 내용은 [Go 에이전트 EOL 정책을](/docs/apm/agents/go-agent/get-started/go-agent-eol-policy/) 참조하세요.
