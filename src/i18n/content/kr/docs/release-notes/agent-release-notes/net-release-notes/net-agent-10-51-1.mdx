### 알아채다

* 사용자 지정 공급업체 이름으로 `RecordDatastoreSegment()` 을(를) 사용하는 고객은 메트릭 이름이 `Datastore/*/Other/*` 에서 `Datastore/*/<VendorName>/*`(으)로 변경되는 것을 확인할 수 있습니다. 이전 Other 기반 메트릭 이름을 참조하는 모든 대시보드, 알림 또는 쿼리를 업데이트하십시오. ([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))

### 수정 사항

* `RecordDatastoreSegment()` API는 호출자가 제공한 벤더 이름을 무시하지 않고 올바르게 사용합니다. ([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))
* 트랜잭션이 수집되기 전에 모든 StackExchange.Redis 세그먼트가 생성되어 트랜잭션에 추가되었는지 확인합니다([#3574](https://github.com/newrelic/newrelic-dotnet-agent/issues/3574))([4152ea2](https://github.com/newrelic/newrelic-dotnet-agent/commit/4152ea25b23318c68d372447c905c622ac5ac868))
* 9.7.0 서명 변경 사항을 지원하도록 MySQL.Data 계측 업데이트 ([#3562](https://github.com/newrelic/newrelic-dotnet-agent/issues/3562)) ([1e40e44](https://github.com/newrelic/newrelic-dotnet-agent/commit/1e40e44e89ab3f5c1af540a9aa30c6e1fa6a30e3))

* 특히 오래된 에이전트를 사용하는 경우 [구형 에이전트 업데이트](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net) 에 대한 주요 변경 사항 및 절차 목록을 검토하세요.

[에이전트에게 최신 정보를 제공하는 방법](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent) 에 대해 자세히 알아보세요. 에이전트 릴리스 및 지원 날짜에 대한 자세한 내용은 뉴렐릭 .NET 에이전트 [EOL 정책 문서를](/docs/apm/agents/net-agent/getting-started/net-agent-eol-policy) 참조하세요.
