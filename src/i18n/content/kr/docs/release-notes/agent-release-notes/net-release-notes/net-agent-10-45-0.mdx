### 새로운 기능

* [OpenTelemetry 구현을](https://github.com/open-telemetry/opentelemetry-dotnet/blob/main/src/OpenTelemetry/Trace/Sampler/TraceIdRatioBasedSampler.cs) 기반으로 하는 새로운 분산 추적 샘플러인 TraceId 비율 기반 샘플러를 추가합니다. 이 샘플러는 TraceId(무작위 GUID)와 구성된 샘플링 비율(샘플링될 트레이스의 목표 백분율)을 사용하여 샘플링 결정을 내립니다. 자세한 내용은 [OpenTelemetry 사양을](https://opentelemetry.io/docs/specs/otel/trace/tracestate-probability-sampling/) 참조하세요. ([#3211](https://github.com/newrelic/newrelic-dotnet-agent/issues/3211)) ([63ca603](https://github.com/newrelic/newrelic-dotnet-agent/commit/63ca603ca6cb86110431413c03afbf8a8c8e9367))

### 수정 사항

* MSSQL 연결 문자열 구문 분석은 예외를 발생시키고 데이터 저장 측정, 소스([#3263](https://github.com/newrelic/newrelic-dotnet-agent/issues/3263))([cdf000b](https://github.com/newrelic/newrelic-dotnet-agent/commit/cdf000b73fe4beaa73fa4bfe3d338f8eb6698cf1))를 비활성화할 수 있습니다.
* Kafka &quot;Consume&quot; 측정, 리소스 관련 문제를 해결하여 자동 측정, 로그측정 측정이 사용자 정의와 함께 작동하는지 확인하세요. ([#3257](https://github.com/newrelic/newrelic-dotnet-agent/issues/3257)) ([1dbd1c5](https://github.com/newrelic/newrelic-dotnet-agent/commit/1dbd1c51aeaf6e2909d23b078029ee78b721b3bc))

### 에이전트 업데이트

* [.NET 에이전트를 업데이트](/docs/agents/net-agent/installation-configuration/update-net-agent) 하려면 표준 절차를 따르세요.
* 특히 오래된 에이전트를 사용하는 경우 [구형 에이전트 업데이트](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net) 에 대한 주요 변경 사항 및 절차 목록을 검토하세요.

최신 에이전트 버전이 출시되면 즉시 업데이트하는 것이 좋습니다. 최신 버전으로 업그레이드할 수 없는 경우, 에이전트를 90일 이내 버전으로 업데이트하세요. [에이전트에게 최신 정보를 제공하는 방법](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent) 에 대해 자세히 알아보세요. 에이전트 릴리스 및 지원 날짜에 대한 자세한 내용은 뉴렐릭 .NET 에이전트 [EOL 정책 문서를](/docs/apm/agents/net-agent/getting-started/net-agent-eol-policy) 참조하세요.
