## 포함된 패치 파일

이번 V3.0 SP12 릴리스에는 하나의 파일이 포함되어 있습니다.

* K-300SP12NEWR.SAR

## 실행 절차

### 서비스팩 설치 및 설정

* **설치 장소:** 이 서비스 팩을 중앙 모니터링 시스템의 클라이언트 000에 구현, 배포합니다.
* **설정 위치:** 중앙 모니터링 시스템의 비즈니스 클라이언트에서 모든 후속 설정 작업을 수행합니다.
* **자세한 설치 지침:** 단계별 설치 절차는 **설치 가이드 V3.0** (부록 VIII: 서비스 팩 설치) 또는 **관리자 가이드 V3.0** (섹션: 서비스 팩 설치)을 참조하십시오.

## 이번 패치의 업데이트 내용

SP12는 버그 수정 및 새로운 기능에 대한 뉴럴릭 노트 모음입니다.

### NR 1344

`NR_SAP_AIF_ERROR` 에 대한 새로운 보고서 `AIF_TABLES` 을(를) 사용하면 `/AIF/STD_IDX_TBL` 이외의 테이블에서 오류 메시지를 검색할 수 있습니다.

### NR 12-02

API 기반 데이터 제공업체의 문제점을 수정하고 새로운 기능을 추가합니다.

* API 대상에 도달할 수 없을 때 `/NEWR/API_DATA_COLLECTOR` 에서 발생하던 코드 덤프 오류를 수정했습니다.
* CALM Landscape 데이터 추출 시 발생하던 리소스 오류를 수정했습니다.
* BTP 메시지 처리 로그 추출 시 발생하던 리소스 오류를 수정했습니다.
* 모니터링 그룹 CALM에 속한 모든 데이터 공급자에 대해 통화 ID로 구성된 서비스 ID(LMSID)에 대한 유효성 검사가 추가되었습니다.
* 데이터 수집 API 호출 전에 인증서 유효성 검사 기능을 추가했습니다.

### NR 12-03

컬렉션 관리자에 대한 컬렉션 지연 및 고정 데이터 수집 기간을 구현하고 레지스트리의 통화 ID를 통한 테스트를 수행합니다.

### NR 12-05

다음과 같은 개선 사항을 구현합니다.

* CALM 모니터링 그룹의 데이터 제공자를 위한 페이지네이션 지원
* RFC 검사 프로그램이 단일 RFC 대상을 검사할 수 있도록 하는 새로운 프로그램 템플릿
* 단일 통화 ID에 대해서만 데이터 공급자 테스트를 강제로 수행하도록 구성하는 설정입니다.

다음 문제를 해결합니다:

* 수금 관리자 출력에 발신자 번호가 표시되지 않습니다.
* 자격 증명 유효성 검사 문제 `NR_CF_AUDIT_EVENTS`

### NR 12-06

새로운 데이터 공급자 `NR_SAP_ABAP_WS_COMPLETE` ABAP 웹 서비스 이벤트를 수집합니다.

### 추가 버그 수정

위의 사항 외에도 SP12는 다음과 같은 문제를 해결합니다.

* 테이블 검색 데이터 공급자 함수 모듈 `/NEWR/RFC_EVENTS_COLLECTOR` 의 타임스탬프 문제를 수정했습니다. `/NEWR/RFC_EVENTS_COLLECTOR2`
* 트레이스 레코드에 대한 연결을 용이하게 하기 위해 IDoc 이벤트에 프로세서 ID를 추가했습니다.

## 수동 설정 업데이트

### NR\_SAP\_AIF\_ERROR에 새로운 형식을 추가합니다.

새로운 `AIF_TABLES` 보고서를 구성하려면 다음을 수행하십시오.

1. 첨부된 전송 정보를 중앙 모니터링 시스템에 적용하십시오.
2. 프로세서 `/n/NEWR/REGISTRY` 로 이동합니다.
3. 데이터 공급자 `NR_SAP_AIF_ERROR` 선택하고 **Configure** \[구성]을 클릭합니다.
4. 파장, `AIF_TABLES` 을 추가하고 구성된 라벨스페이스/인터페이스/버전 조합에 대해 `/AIF/T_IDX_TBL` 에 지정된 테이블 이름을 입력합니다.

<Callout variant="tip">
  `AIF_TABLES` 이 비어 있거나 지정되지 않은 경우 인덱스 테이블은 기본적으로 표준 단일 인덱스 테이블 `/AIF/STD_IDX_TBL` 로 설정됩니다.
</Callout>

### 수집 지연 및 고정 수집 기간을 구성합니다.

유동적인 컬렉션 경계와 데이터베이스 지연 시간으로 인해 발생하는 데이터 누락 문제를 해결하기 위해 데이터 공급자 레지스트리에 두 개의 새로운 설정 필드가 추가되었습니다.

**데이터 공급자 레지스트리에 새 필드 추가:**

* **DATA\_COLLECTION\_DELAY:** 수집 타임스탬프에서 차감할 의도적인 지연 시간(분)을 지정합니다. 이 버퍼는 데이터베이스 업데이트로 인해 늦게 도착하는 데이터가 수집 기간이 종료되기 전에 캡처되도록 합니다.
* **FIXED\_SCHEDULES:** 고정 수집 기간을 활성화하는 플래그입니다. `X` 으로 설정하면 일관되고 예측 가능한 수집 기간을 만들기 위해 수집 타임스탬프에서 초 단위가 제거됩니다.

**작동 방식:**

**고정 일정** (`FIXED_SCHEDULES = 'X'`):

* 수집 타임스탬프는 초 단위를 제거하여 정규화됩니다.
* 예: 12:12:23은 12:12:00으로 바뀝니다.
* 모든 컬렉션에 걸쳐 일관되고 정렬된 컬렉션 경계를 만듭니다.

**데이터 수집 지연** (`DATA_COLLECTION_DELAY > 0`):

* 데이터 수집 시작 및 종료 타임스탬프에서 지정된 지연 시간(분)을 뺍니다.
* 데이터베이스 업데이트 누락을 포착하기 위해 수집 기간을 뒤로 이동합니다.
* 예시: 2분 지연 변경 시 10:00:00 - 10:05:00이 09:58:00 - 10:03:00으로 변경됩니다.

## ABAP 비즈니스 시스템을 위한 새로운 데이터 제공자

### NR\_SAP\_ABAP\_WS\_COMPLETE

모니터링 대상 시스템 전반에 걸쳐 웹 서비스 메시지 헤더와 데이터(태그) 레코드를 레코드 수 제한 없이 병렬로 처리합니다.

**구성:**

* **유형:** 이벤트
* **의사소통 담당자:** N/A
* **데이터 제공자 기능:** `/NEWR/WS_MESSAGES_ARFC`
* **데이터 공급자 RFC 활성화됨:** 예
* **마무리 기능:** `/NEWR/WS_EVENTS_FINISHER`
* **모니터링 그룹:** `SAP_APPL`

### 기능 개요

이 데이터 제공업체는 사전 활성화되어 있습니다. 필요한 조치:

* **`ADAPTER_TYPE`**

  * **목적:** ABAP 웹 서비스 메시지 헤더에서 선택할 어댑터 유형을 지정합니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 테이블 필드는 `;` 으로 구분되며, 여러 줄에 걸쳐 입력할 수 있습니다.

* **`DESTINATION`**

  * **목적:** RFC 대상 지정
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** RFC 목적지

* **`DPC_COLL_TIME`**

  * **목적:** 시간 간격 측정
  * **표시:** 포용적인 내용만
  * **옵션:** BT
  * **값:** 시스템에서 생성된 시작 타임스탬프 및 종료 타임스탬프

* **`IB_IF_NAME`**

  * **목적:** ABAP 웹 서비스 메시지 헤더에서 선택할 수신자 인터페이스 이름입니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 수신자 인터페이스 이름은 `;` 으로 구분되며, 여러 줄로 입력할 수 있습니다.

* **`IB_PARTY`**

  * **목적:** 수신자는 ABAP 웹 서비스 메시지 헤더에서 선택됩니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 수신자 정보는 `;` 으로 구분되며, 여러 줄로 입력할 수 있습니다.

* **`JOIN_CONDITION`**

  * **목적:** 추가 처리를 위해 테이블 `/NEWR/JOIN_COND` 의 조인 레코드를 지정합니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 제목은 `;` 으로 구분되며, 여러 줄로 입력할 수 있습니다.

* **`NR_ACCID`**

  * **목적:** 뉴렐릭 계정 ID 지정
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 뉴렐릭 계정 ID

* **`OB_IF_NAME`**

  * **목적:** ABAP 웹 서비스 메시지 헤더에서 선택할 송신자 인터페이스 이름입니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 송신자 인터페이스 이름은 `;` 으로 구분되며, 여러 줄로 입력할 수 있습니다.

* **`OB_PARTY`**

  * **목적:** 발신자는 ABAP 웹 서비스 메시지 헤더에서 선택됩니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 발신자 정보는 `;` 으로 구분되며, 여러 줄로 입력할 수 있습니다.

* **`SETTING_SWITCH`**

  * **목적:** 분산 처리에 사용되는 옵션을 지정합니다.

  * **표시:** 포용적인 내용만

  * **옵션:** EQ

  * **값:** 분산 프로세스를 제어하기 위한 여러 옵션이 `;` 으로 구분되어 있습니다.

    * **HEADONLY=**

      * `SPACE` (기본값): 헤더와 태그 데이터를 모두 제공합니다(태그 이름이 `TAG_LIST` 매개변수에 지정된 경우).
      * `X`: 태그 이름이 `TAG_LIST` 에 지정된 경우에도 헤더 데이터만 제공합니다.

    * **FIRSTONLY=**

      * `X` (기본값): 데이터 제공자는 `TAG_LIST` 매개변수, 변수에 나열된 각 태그의 첫 번째 발생에서만 데이터를 추출합니다.
      * `SPACE`: 데이터 제공자는 `TAG_LIST` 서식에 나열된 각 태그의 모든 항목에서 데이터를 추출합니다.

    * **FREE%=nn:** 모니터링 시스템에서 사용 가능한 작업 프로세스의 백분율을 지정하여 원하는 여유 작업 프로세스 제한을 결정합니다.

      * 현재 모니터링 시스템의 사용 가능한 작업 프로세스 수가 이 원하는 사용 가능한 작업 프로세스 제한과 같거나 크면 분산 처리를 계속합니다.

      * 그렇지 않으면 1초간 기다린 후 다음 중 하나가 발생할 때까지 위의 확인을 반복합니다.

        * 배포 처리가 계속될 수 있습니다.
        * 총 대기 시간이 지정된 최대 대기 시간을 초과합니다(`MWAIT#=nn` 참조).

      * 이 옵션의 기본값은 30%입니다.

    * **MINFT#=nn:** 최소 사용 가능한 작업 프로세스 제한을 지정합니다.

      * 모니터링 시스템에서 계산된 원하는 자유 작업 프로세스 수가 이 최소 임계값보다 낮아지면 시스템은 추가 처리를 시도하기 전에 1초 동안 일시 중지됩니다.
      * 이 옵션의 기본값은 3입니다.

    * **MWAIT#=nn:** 최대 대기 시간(초)을 지정합니다.

      * 총 대기 시간(참조 `FREE%=nn`)이 지정된 최대값을 초과하면 시스템 리소스를 절약하기 위해 배포 프로세스가 취소됩니다.
      * 이 옵션의 기본값은 60입니다.

    * **MREC#=nnnnn:** **함수 모듈** `/SDF/IMA_DC_WS_ABAP` 을 호출할 때마다 검색되는 **웹 서비스 메시지 헤더** 레코드의 개수 제한을 지정합니다.
      * 이 옵션의 기본값은 10000입니다.

    * **JSEND#=nnnn:** 뉴렐릭 서버로 전송되는 결과 속성 레코드의 개수 제한을 지정합니다.
      * 이 옵션의 기본값은 5000입니다.

    * **USER=user:** 각 모니터 시스템에 대해 RFC 사용자 이름을 지정하여 해당 시스템에서 현재 사용 중인 총 작업 프로세스 수를 검색합니다. `user` 은 SAP 사용자 ID입니다.

    * **DPWP#=nn:** **최대 작업 프로세스 제한을** 지정합니다.

      * 분산 처리 중: 현재 모니터링 시스템에서 사용 가능한 작업 프로세스 수가 이 최대 한도 이하인 경우 분산 처리를 계속합니다.
      * 그렇지 않으면 1초간 기다린 후 다시 확인하도록 루프를 돌립니다.
      * 분산 처리가 진행될 수 있거나 총 대기 시간이 지정된 최대 대기 시간(`MWAIT#=nn`)을 초과할 때까지 이 루프를 계속합니다.
      * 참고: 이 검사는 사용자 시도가 초기 값이 아닌 값으로 지정된 경우에만 실행됩니다.
      * 이 옵션의 기본값은 8입니다.

* **`STATE_GROUP`**

  * **목적:** ABAP 웹 서비스 메시지 헤더에서 선택할 처리 상태 그룹. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 처리 상태 그룹은 `;` 으로 구분되며 여러 줄로 입력할 수 있습니다.

* **`SYS_ID`**

  * **목적:** 시스템 ID 확인
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 시스템 ID

* **`TAG_LIST`**

  * **목적:** ABAP 웹 서비스 메시지 데이터에서 선택하여 메시지 데이터 출력에 표시할 태그 이름을 지정합니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 테이블 필드는 `;` 으로 구분되며, 여러 줄에 걸쳐 입력할 수 있습니다.

* **`USERNAME`**

  * **목적:** ABAP 웹 서비스 메시지 헤더(`CREATED_BY`)에서 메시지를 생성한 사용자 이름을 선택합니다. 함수 모듈 `/SDF/IMA_DC_WS_ABAP` 의 필터 옵션입니다.
  * **표시:** 포용적인 내용만
  * **옵션:** EQ
  * **값:** 사용자 이름은 `;` 으로 구분되며, 여러 줄로 입력할 수 있습니다.
