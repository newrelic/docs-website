## 버그 수정

* 세션 리플레이 이미지 압축이 메인 스레드에서 실행되던 성능 문제를 수정했습니다. 이제 비동기적으로 실행되어 캡처 중 오버헤드를 줄입니다.

* 세션 리플레이에서 점진적인 뷰 추가를 자연스러운 뷰 순서로 내보내어 렌더링 정확도 문제를 해결했습니다.

* 너무 오래 대기열에 있는 데이터를 삭제하는 TTL(Time-To-Live)을 적용하여 오래된 오프라인 페이로드가 전송될 수 있는 문제를 수정했습니다.

* 종료 수집 중에 세션 속성이 손실될 수 있는 문제를 해결했습니다.

* 앱 종료 시 버퍼링된 이벤트가 삭제되는 문제를 수정하여, 데이터 손실을 방지하기 위해 최종 수집 시 전송되도록 보장합니다.

* 기기 회전 중 전환 시 세션 연속성이 손실될 수 있는 타이밍 문제를 수정했습니다.

* HTTP 헤더 처리를 스레드로부터 안전하게 만들어 동시 액세스 시 발생할 수 있는 오류를 해결했습니다.

* 시작 타임스탬프가 설정되지 않은 경우 앱 시작 메트릭이 잘못 계산될 수 있는 문제를 수정했습니다.

* 활성 네트워크 연결 없이 에이전트가 크래시를 보고하려고 시도하여 오프라인 디바이스에서 불필요한 오류를 발생시키는 문제를 해결했습니다.

* `stop()`의 ABI 불일치로 인해 종료 중 발생할 수 있는 네이티브(NDK) 크래시를 수정했습니다.

* DexGuard APK를 직접 생성하는 빌드에 대한 ProGuard 맵 업로드를 수정했습니다.

* 맵 업로드 중 Gradle 설정 캐시 호환성을 개선했습니다.

* 호스트 애플리케이션과의 충돌을 방지하기 위해 Gradle 플러그인에 번들로 제공되는 `commons-io` 의존성을 재배치했습니다.

  지지 성명

  최소 3개월마다 에이전트를 업데이트하는 것이 좋습니다. 모바일 모니터링 에이전트 EOL 정책에서 안드로이드 에이전트 지원에 대한 구체적인 정책 및 날짜를 확인하세요.
