## 공지

안드로이드용 세션 리플레이가 프리뷰에서 정식 버전으로 전환되었습니다. 세션의 충실한 시각적 재현을 통해 사용자의 모바일 경험에 대한 완벽한 가시성을 확보하세요. 세션 리플레이에 대해 자세히 알아보기.

## 새로운 기능

* 세션 리플레이 중에 특정 뷰가 캡처되는 것을 프로그래밍 방식으로 차단하는 기능을 추가하여, 기본 마스킹 규칙을 넘어 기록되는 내용에 대한 세분화된 제어를 제공합니다.
* 이제 에이전트는 4시간 연속 사용 후 자동으로 새 세션을 생성하여 장기 실행 세션 추적을 개선합니다.
* 이제 로그 포워딩은 세션 리플레이와 샘플링 결정을 조정하여 리플레이 중인 세션에 대한 로그가 캡처되도록 보장합니다.

## 버그 수정

* 에이전트가 활성 네트워크 연결 없이 로그를 전달하려고 시도하여 오프라인 디바이스에서 불필요한 오류가 발생하는 문제를 해결했습니다.

* HttpsURLConnection에서 스트림을 읽거나 닫을 때 발생할 수 있는 RuntimeException을 수정했습니다.

* 세션 처리 중 발생할 수 있는 널 포인터 예외를 해결했습니다.

* 하베스트 주기가 완료되기 전에 세션 로테이션이 발생하여 전환 중 데이터 손실을 유발할 수 있는 타이밍 문제를 수정했습니다.

* 세션 로테이션 중 세션 ID가 null이 될 수 있어 데이터가 잘못 속성으로 지정되는 문제를 해결했습니다.

* 세션 리플레이 캡처에서 여러 렌더링 정확도 문제를 해결했습니다.

* 세션 리플레이 캡처 중 로컬 마스킹 설정이 텍스트 요소에 올바르게 적용되지 않던 문제를 수정했습니다.
