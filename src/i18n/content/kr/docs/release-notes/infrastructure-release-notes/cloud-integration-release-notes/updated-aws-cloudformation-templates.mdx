### 업데이트

* 우리는 [지표 스트림 통합을](https://docs.newrelic.com/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/via-cloudformation-cwms/) 위한 AWS CloudFormation 템플릿에 대한 업데이트를 출시했습니다. 이 업데이트는 스택을 업데이트하고 최신 AWS Lambda Python 런타임을 사용하는 것을 방해할 수 있는 심각한 문제를 해결합니다.

* 기존 통합에 수정 사항을 적용하려면 아래 단계에 따라 CloudFormation 스택을 업데이트하세요.

  1. **CloudFormation으로 이동합니다**. AWS Management Console에 로그인하고 **CloudFormation** 서비스로 이동합니다.

  2. **그리드 선택**: 뉴렐릭 지표 Streams 그리드(종종 `NewRelic-Metric-Streams-*` 로 명명됨)를 찾아서 선택한 다음 **Update Stack** \[업데이트 그리드] 드롭다운 메뉴를 클릭하세요. 옵션에서 **Make a direct update** \[직접 업데이트를] 선택하세요. 자세한 내용은 [스택을 직접 업데이트하는 방법에 대한 AWS 설명서를](https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/using-cfn-updating-stacks-direct.html) 참조하세요.

  3. **템플릿 바꾸기**: 업데이트 화면에서 **현재 템플릿 바꾸기를** 선택 **Replace current template** \[해야 합니다].

  **참고**: &quot;Use existing template&quot; \[&quot;기존 템플릿 사용&quot;] 옵션은 사용하지 마세요. 필요한 수정 사항이 적용되지 않습니다.

  4. **새 템플릿 URL 제공**: *템플릿 지정* 섹션에서 **Amazon S3 URL** 옵션을 선택하고 다음 링크를 붙여넣습니다.

  `https://nr-downloads-main.s3.amazonaws.com/cloud_integrations/aws/cloudformation/newrelic-cloudformation-mstreams.yml`

  5. **확인 및 업데이트**: 모범 사례 및 그리드 옵션 페이지를 통해 **Next** \[다음을] 클릭하세요. 최종 검토 페이지에서 맨 아래로 스크롤하여 **AWS CloudFormation이 IAM 리소스를 생성할 수 있음을 확인** 하는 상자를 선택하고 **Update stack** \[스택 업데이트를] 클릭합니다.

  **Events** \[이벤트] 탭에서 진행 상황을 모니터링할 수 있습니다. 스택 상태가 `UPDATE_COMPLETE` 으로 변경되면 업데이트가 완료됩니다.
