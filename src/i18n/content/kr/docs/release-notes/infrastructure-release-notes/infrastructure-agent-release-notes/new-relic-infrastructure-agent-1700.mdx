에이전트의 새로운 버전이 출시되었습니다. [인프라 에이전트를 업데이트](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent) 하려면 표준 절차를 따르세요. 뉴렐릭은 에이전트를 정기적으로, 그리고 최소 3개월마다 업그레이드할 것을 권장합니다. 이번 릴리스부터 지원되는 가장 오래된 버전은 [안드로이드 에이전트 1.58.0](https://docs.newrelic.com/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1580/) 입니다.

## 변경됨

* feat(oci): [https://github.com/newrelic/infrastructure-agent/pull/2121](https://github.com/newrelic/infrastructure-agent/pull/2121)에서 OCI에서 실행되는 호스트에 작업 클라우드별 태그를 추가합니다.

  버전 1.70.0부터 뉴렐릭 인프라 에이전트는 이제 OCI(데이터베이스 관리)에서 실행 중인 VM에 대한 메타데이터를 자동으로 수집합니다. 에이전트는 다음과 같은 세부 정보를 수집합니다.

  * 이름 표시하기
  * 호스트 아이디
  * oci.availabilityDomain
  * oci.region
  * oci.compartmentId
  * oci.shape
  * oci.imageId

  이 데이터 데이터를 부분 검색의 태그로 사용하거나 NRQL 쿼리의 속성으로 사용할 수 있습니다. `host.id` 속성은 OCI용 뉴렐릭 클라우드 통합에서도 cloud 메타데이터를 올바른 호스트에 연결하는 데 사용됩니다.

  이 기능은 다른 cloud 제공업체의 기존와 일관성을 유지하기 위해 기본적으로 활성화되어 있습니다. 이 기능을 비활성화해야 하는 경우 구성 설정 [disable\_cloud\_metadata](https://docs.newrelic.com/docs/infrastructure/infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#disable-cloud-metadata) 또는 [disable\_cloud\_instance\_id를](https://docs.newrelic.com/docs/infrastructure/infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#disable-cloud-instance-id) 사용할 수 있습니다.

  *배경*: 이미 수년 동안 AWS, Azure 또는 GCP에서 실행되는 VM 에 설치된 뉴렐릭 인프라 에이전트에는 동등한 세부 정보 수집이 있었으며 이 동작은 기본적으로 활성화되어 있습니다. 다양한 클라우드 간 일관성을 위해 OCI 기능은 동일한 설정 항목과 동일한 기본 동작을 사용합니다.
