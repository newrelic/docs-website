## 버전 1.312.0

### 특징

#### MFE 로그를 자동으로 감지합니다

`Log` 이벤트에 대한 자동 마이크로 프런트엔드(MFE) 속성 부여 지원을 추가합니다. 이전에는 `Log` 이벤트가 컨테이너 브라우저 애플리케이션에 의해 전역적으로 *자동으로* 캡처되었습니다. 이 변경으로 에이전트는 주어진 로그 메소드(예: console.log, 콘솔 정보, `register` API에 등록된 MFE 모듈의 호출 스택 서명에 대한 등)입니다. 트리거 에이전트 등록을 담당하는 파일이 트리거되는 로그 메서드의 호출 그리드 서명에서 발견되면 해당 이벤트는 컨테이너 대신 MFE에 대한 속성이 됩니다.

#### MFE AJAX 자동 감지

`AjaxRequest` 이벤트에 대한 자동 마이크로 프런트엔드(MFE) 속성 부여 지원을 추가합니다. 이전에는 `AjaxRequest` 이벤트가 컨테이너 브라우저 애플리케이션에 의해 전역적으로만 캡처되었습니다. 이 변경을 통해 에이전트는 AJAX 호출의 호출 스택 서명을 `register` API에 등록된 MFE 모듈의 호출 스택 서명과 교차 비교합니다. 트리거 에이전트 등록을 담당하는 파일이 트리거되는 AJAX 요청의 호출 그리드 서명에서 발견되면 해당 이벤트는 컨테이너 대신 MFE에 대한 속성이 됩니다.

### 버그 수정

#### 세션 리플레이가 Safari에서 실행되는 Angular 앱에서 작동하지 않습니다.

Safari에서 실행되는 Angular 앱에서 스냅샷 리플레이가 초기 스냅샷 이후의 녹화를 캡처하지 못하는 문제를 수정했습니다.

#### 원형 대체 기능을 되돌리기

JSON.stringify 작업에 적용된 순환 대체 방식은 `scroll` 이벤트 핸들러와 같이 복잡하고 중첩된 객체가 빠르게 연속적으로 생성되는 특정 사례에서 지연을 유발하는 것으로 나타났습니다. 이 방법은 성능 병목현상, 병목지점을 감소시키기 위해 더 빠르고 간단한 작업으로 되돌아갔습니다.

## 지지 성명

뉴렐릭은 에이전트를 정기적으로 업그레이드하여 최신 기능과 성능 이점을 얻을 것을 권장합니다. 이전 릴리스는 지원 [종료](https://docs.newrelic.com/docs/browser/browser-monitoring/getting-started/browser-agent-eol-policy/) 시점부터 더 이상 지원되지 않습니다. 출시일은 에이전트 버전의 원래 출판일을 반영합니다.

새로운 브라우저 에이전트 릴리스는 일정 기간 동안 작은 단계로 고객에게 출시됩니다. 이로 인해, 귀하의 계정에서 해당 릴리스에 접근할 수 있는 날짜가 원래 게시 날짜와 일치하지 않을 수 있습니다. 자세한 내용은 이 [상태 대시보드를](https://newrelic.github.io/newrelic-browser-agent-release/) 참조하세요.

당사의 [브라우저 지원 정책](https://docs.newrelic.com/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring/#browser-types) 에 따라 브라우저 에이전트 v1.312.0은 Chrome 136-146, Edge 136-146, Safari 18-26 및 Firefox 138-148 버전 범위에서 작동하도록 개발 및 테스트되었습니다. 모바일 장치의 경우 v1.312.0이 Android OS 16 및 iOS Safari 18-26.2에 대해 구축 및 테스트되었습니다.
