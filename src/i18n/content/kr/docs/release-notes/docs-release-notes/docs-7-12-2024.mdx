### 새로운 문서

* 새로운 분기형 [Kubernetes 모니터링 통합](/install/kubernetes/) 설치 문서는 필요한 특정 설치 정보를 찾는 데 도움이 됩니다.
* 아파치 Airflow와 Snowflake를 사용하여 [dbt 클라우드](/docs/more-integrations/dbt-cloud/dbt-cloud-integration) 데이터를 설치, 구성 및 관리하세요.
* 새로운 [Kubernetes 에이전트 운영자는](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator) APM 에이전트 측정을 자동화하므로 더 이상 수동으로 관리할 필요가 없습니다. 현재 공개 미리보기 중입니다.
* iOS 에이전트는 [모바일 로그](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs) 및 [watchOS 측정, 쿠키를](/docs/mobile-monitoring/new-relic-mobile-ios/watchOS/instrument-watchos) 뉴렐릭으로 보내기 위한 지원을 추가했습니다.
* [AWS Lambda에 데이터가 표시되지 않는 문제 해결 방법은](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-no-data-appears) UI에 AWS Lambda 데이터가 나타나지 않을 때의 해결 방법을 설명합니다.
* [Android 에이전트 기본 충돌 보고는](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-native-crash-reporting) 충돌 보고 기능을 활성화, 구성 및 디버그하는 방법을 설명합니다.
* [NRQL 공지 조건의 제목 템플릿은](/docs/alerts/create-alert/condition-details/title-template) 제목 템플릿을 사용하여 알림 이벤트에 대한 의미 있는 제목을 만드는 방법을 설명합니다.
* [OpenLLMetry가 포함된 Traceloop LLM 옵저버빌리티는](/docs/opentelemetry/get-started/traceloop-llm-observability/traceloop-llm-observability-intro) 이 OpenTelemetry기반 도구를 사용하여 AI 성능을 모니터링하는 방법을 설명합니다.

### 주요 변경 사항

* [APM-계측된](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes) 및 [OpenTelemetry-계측화된](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) 구성을 연결하기 위한 콘텐츠 및 스타일 업데이트로 가독성이 향상되고 이 설정을 더 쉽게 할 수 있습니다. 또한 [Kubernetes 통합](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration) 에 대한 소개를 스타일과 명확성을 개선하기 위해 업데이트했습니다.
* [데이터 수집 청구 및 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices) 등에서 GB당 가격 변경 사항을 반영하도록 문서를 업데이트했습니다.
* 데이터 사전에 새로운 [`MobileApplicationExit`](/attribute-dictionary/?dataSource=Mobile&event=MobileApplicationExit) 속성을 추가했습니다.
* [.NET 에이전트 설치를](/install/dotnet/?deployment=azure&azure=azuresiteextension) 위해 Azure 관련 설치 정보를 추가했습니다.
* 2024년 7월 3일부로 당사의 [Kubernetes 통합 기능은](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements/#compatibility) Kubernetes v1.25 이하 버전에 대한 지원을 중단했습니다.
* [뉴렐릭 AI를 만나보세요. 옵저버빌리티 도우미인 뉴렐릭](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-ai) AI를 최대한 활용하는 방법을 설명하는 대폭 수정된 안내문입니다.
* 새로운 `is_integrations_only` 설정에 대한 구성 정보로 [인프라 에이전트 구성 설정이](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#is_integrations_only) 업데이트되었습니다. OpenTelemetry 수집기 또는 Prometheus 노드 내보내기와 같은 에이전트 에이전트가 아닌 소스에서 호스트를 리소스로 사용하는 경우 이 설정을 활성화합니다.

### 사소한 변경 사항

* 새로운 Kubernetes 에이전트 오퍼레이터와 관련된 다양한 업데이트 사항입니다.
* OpenTelemetry 문서 업데이트를 통해 일부 깨진 앵커 링크가 수정되었고, [OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp/#payload-limits) 에 타임아웃 설정 정보가 추가되었으며, 기타 일관성 개선 사항이 적용되었습니다.
* [AWS PrivateLink를](/docs/data-apis/custom-data/aws-privatelink/#regions-zones) 업데이트하여 여러 리전을 지원한다는 내용을 명확히 했습니다.
* [Node.js 에이전트 호환성 표를](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#system) 업데이트하여 Node.js 버전 22 지원을 추가했습니다.
* .NET 에이전트 Azure 문제 해결, 해결 문서를 [문제 해결, 해결 범주](/docs/apm/agents/net-agent/azure-installation/azure-cloud-services-no-data-appears/) 로 이동했습니다.
* [/docs/data-apis/ingest-apis/security-data-api](/docs/data-apis/ingest-apis/security-data-api/#req-fields) 에서 `issueId` 업데이트하고 `entityGuid` 추가했습니다.
* [Cacti 통합](/docs/infrastructure/host-integrations/host-integrations-list/cacti-integration/) 의 스니펫 코드 오류를 수정했습니다.
* .NET Lambda 레이어가 필요하지 않음을 명확히 하기 위해 [레이어 없는 측정, Lambda 모니터링에 대한 로그를](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-layerless/#dotnet) 업데이트했습니다.
* 특정 상황에서 에이전트를 끄는 방법을 설명하기 위해 [루비 에이전트가 시작될 때의 제어가](/docs/apm/agents/ruby-agent/troubleshooting/control-when-ruby-agent-starts) 업데이트되었습니다.
* 저희는 문서 사이트 전체에서 페이지에 삽입된 가입 양식을 모두 제거했습니다.
* [Temporal 자체 호스팅 모니터링 통합](/docs/infrastructure/host-integrations/host-integrations-list/temporal-monitoring-integration/#temporal-logs-configuration) 에서 Temporal 로그를 구성하고 전달하는 방법에 대한 지침을 추가했습니다.
* CentOS 7에 대한 [인프라](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/#os), [SNMP 성능](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#linux), [네트워크 syslog](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#linux) 및 [네트워크 흐름 데이터](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#linux) 모니터링 지원은 2024년 6월 30일에 종료되었습니다.
* NerdGraph의 기본 쿼리 시간 초과(5초)를 사용하는 [NRQL에서 속도 제한을](/docs/nrql/using-nrql/rate-limits-nrql-queries/#query-duration) 업데이트했습니다.
* 새로운 스크린샷 및 서버 측 설정 세부 정보에 [응답하지 않는 기능이](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding) 업데이트되었습니다.
* [부하 에이전트의 CPU 사용 공간 문제 문제 해결](/docs/infrastructure/new-relic-infrastructure/troubleshooting/reduce-infrastructure-agents-cpu-footprint) 가이드에서 `disable_all_plugins` 에 대한 구문 오류가 수정되었습니다.
* PHP용 CodeStream [앱 레벨 지표](/docs/codestream/observability/code-level-metrics/#requirements) 에는 PHP Intelephense 확장이 필요합니다.
* [뉴렐릭의 제거 기능](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/#wait-longer) 에서는 뉴렐릭이 12시간 이상 해당 앱으로부터 데이터를 받지 못한 경우 해당 앱을 삭제할 수 있음을 명확히 했습니다.
* Kubernetes를 사용한 Reduce 데이터 수집 및 데이터 거버넌스 관련 문서를 단일 [Reduce 데이터 수집](/docs/kubernetes-pixie/kubernetes-integration/installation/reduce-ingest) 페이지로 통합했습니다.
* 현재 지원하는 몇 가지 새로운AWS 리전과 해당 이미지를 포함하여 [로그 포워딩에 대한 AWS FireLens 플러그인을](/docs/logs/forward-logs/aws-firelens-plugin-log-forwarding/#firelens-plugin) 업데이트했습니다.
* [공지 이벤트 흐름을](/docs/alerts/get-notified/alert-event-workflows/#navigate-notification-log) 현재 공지 로그인 열로 업데이트했습니다.
* 당사의 잔류 에이전트 버전 8.13.0 이상이 HikariCP 2.4.0을 최신 버전으로 지원함을 알려드리기 위해 [잔류 에이전트에 대한 호환성 및 요구 사항이](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#JDBC) 업데이트되었습니다. (이 모듈은 HikariCP 배양기 모듈을 대체합니다.)
* .NET 에이전트는 이제 [`Elastic.Clients.Elasticsearch`](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#app-frameworks-core) 버전 8.14.5를 지원합니다.
