### 새로운 문서

* 뉴렐릭 eBPF 에이전트가 수집하는 모든 속성에 대한 포괄적인 문서를 제공하기 위해 [eBPF 속성 참조](/docs/ebpf/attributes-reference) 를 추가했습니다.
* 전체 자동화 없이 파이프라인 컨트롤 게이트웨이를 배포하기 위한 지침을 제공하는 [Flux 없이 게이트웨이 설치](/docs/new-relic-control/pipeline-control/gateway/install-gateway-without-flux) 를 추가하여, 사용자가 Pipeline 설정을 위해 UI를 계속 사용하면서 Kubernetes 도구를 사용하여 게이트웨이 인프라를 관리할 수 있도록 했습니다.

### 주요 변경 사항

* Fluxless 설치 모드 지원 및 개선된 설정 가이드를 포함하도록 [파이프라인 컨트롤 게이트웨이 문서](/docs/new-relic-control/pipeline-control/gateway/overview) 를 업데이트했습니다.
* Flux CD 종속성 없이 에이전트 컨트롤을 배포하기 위한 Fluxless 설치 지침으로 [에이전트 컨트롤 설정](/docs/new-relic-control/agent-control/setup) 문서를 업데이트했습니다.
* 향상된 OpenTelemetry 설정 예제 및 배포 옵션으로 [ECS EC2 모니터링](/docs/opentelemetry/integrations/ecs-monitoring/ecs-ec2) 및 [ECS Fargate 모니터링](/docs/opentelemetry/integrations/ecs-monitoring/ecs-fargate) 문서를 업데이트했습니다.
* MariaDB 호환성 업데이트, 새로운 설정 옵션, 개선된 쿼리 성능 메트릭 가이드를 포함하여 MySQL 및 MariaDB 통합 설치 마법사를 업데이트했습니다.

### 사소한 변경 사항

* SPA 상호작용 추적을 위한 새로운 옵션과 자세한 반환 값 설명으로 [브라우저 API 상호작용](/docs/browser/new-relic-browser/browser-apis/interaction) 문서를 업데이트했습니다.
* 기본 샘플러와 네이티브 샘플러 간의 차이점을 명확히 하기 위해 [Node.js VM 측정](/docs/apm/agents/nodejs-agent/extend-your-instrumentation/nodejs-vm-measurements) 문서를 업데이트했습니다.
* 사용자 이름에 대한 이름 필드 유효성 검사 요구 사항을 반영하여 [사용자 관리 UI 및 작업](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks) 을 업데이트했습니다.
* 탐색을 개선하고 지식 기반 통합 기능을 명확히 하기 위해 [지식 통합 개요](/docs/agentic-ai/knowledge-integration/overview) 를 업데이트했습니다.
* 단순화된 설치 지침으로 [Node.js 에이전트 설치](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent) 문서를 업데이트했습니다.
* `messaging.operation` 이(가) `NULL`일 때 `OtelMessagingConsumer1_24` 대체 동작을 사용하도록 [OpenTelemetry APM UI](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-ui) 를 업데이트했습니다.
* [컨텍스트의 AWS 로그](/docs/logs/logs-context/aws-logs-in-context)에서 더 이상 사용되지 않는 RDS 인스턴스 로그 지원을 제거했습니다.
