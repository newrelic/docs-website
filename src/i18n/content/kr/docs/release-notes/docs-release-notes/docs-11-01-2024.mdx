### 새로운 문서

* [PHP 에이전트 오버헤드 감소 팁은](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips/) 시스템에서 에이전트의 오버헤드를 줄이기 위해 변경할 수 있는 구성 설정을 나열합니다.

### 주요 변경 사항

* [Build your ownIntegration: 뉴럴릭 Flex](/docs/infrastructure/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) 재구성하고 다시 작성하여 명확성과 완성도를 높였습니다.
* [Scala 측정, 계측](/docs/apm/agents/java-agent/frameworks/scala-installation-java) 에 [Scala Futures](/docs/apm/agents/java-agent/frameworks/scala-installation-java/#futures) 섹션을 추가했습니다.
* [파트 관리 시작하기 ](/docs/vulnerability-management/overview), [파트 우선순위 이해 ](/docs/vulnerability-management/understanding-prioritization)및 [보안 팀으로 파트 관리를 ](/docs/vulnerability-management/security-workflow)포함하여 세 가지 파트 관리 항목을 업데이트했습니다.
* [`cancelAsyncQuery`](/docs/apis/nerdgraph/examples/async-queries-nrql-tutorial/#example-cancel-query) 변이 사용 방법에 대한 정보를 추가했습니다.

### 사소한 변경 사항

* Lambda 함수와 호환되는 뉴렐릭의 끌어 및 루비 이미지에 대한 링크가 업데이트되었습니다.
* [Create NRQL 공지 조건](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions) 에서 독립형 신호 손실 쿼리의 비용이 많이 드는 특성을 언급했습니다.
* 업데이트된 [뉴럴릭 CLI 옵션](/docs/new-relic-solutions/build-nr-ui/newrelic-cli/#nr-cli-commands-options).
* [PHP 에이전트 오버헤드 감소 팁](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips) 에 대한 문제 설명을 다듬어 성능 오버헤드를 명확히 했습니다.
* [거래 제한](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works/#trace-limits) 설명란을 수정하여 이제 제한 UI 에서 이러한 제한을 확인할 수 있음을 명시했습니다.
* [`NrAiIncident` 이벤트 속성](/docs/alerts/create-alert/condition-details/alert-event-attributes/#NrAiIncident-attributes) 에서 `description` 및 `incidentId` 속성 설명을 업데이트했습니다.
* 필요한 새로운 루비 확장을 반영하도록 [코드 레벨 지표를](/docs/codestream/observability/code-level-metrics) 업데이트했습니다.
* 뉴렐릭이 이제 Kubernetes 1.26 대신 Kubernetes 1.31을 지원함을 나타내기 위해 Kubernetes 모니터링 문서를 업데이트했습니다.
* [메시지](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#messaging-core) 테이블에 새로운 `AWSSDK.DynamoDBv2` 측정, 계측 항목을 추가했습니다.
* Node.js 에이전트의 [호환성 요구 사항이](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) 업데이트되었습니다.
* [타입 변환](/docs/nrql/nrql-syntax-clauses-functions/#type-conversion)시 `numeric()` 함수를 사용하지 말아야 할 경우를 명확히 했습니다.
* 공지 쿼리 등록에 대한 새로운 제한에 대한 정보가 추가되었습니다.
