### 새로운 문서

* 두 가지 새로운 GA 기능에 대한 [서비스 수준 관리](/docs/service-level-management/intro-slm) 문서를 추가했습니다: SLI 계산에서 불량 이벤트를 제외하는 계획된 다운타임을 예약하기 위한 유지 관리 Windows, 그리고 속성별로 SLO 준수 및 오류 예산을 세분화하기 위한 Facet Compliance.

### 주요 변경 사항

* 주요 변경 사항, 제거된 계측 및 업데이트된 최소 지원 버전을 포함하여 v14 마이그레이션 가이드를 [Node.js 에이전트 업데이트](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent)에 추가했습니다.
* v14에서 `@apollo/server` 계측이 독립형 플러그인에서 기존 계측으로 마이그레이션된 것을 반영하도록 [Node.js용 Apollo Server 플러그인](/docs/apm/agents/nodejs-agent/extend-your-instrumentation/apollo-server-plugin-nodejs) 을 재구성했습니다.
* 새로운 설정 옵션 및 설치 단계를 포함한 v1.4.1 릴리스 콘텐츠로 [Kubernetes용 뉴렐릭 eBPF 에이전트 설치](/docs/ebpf/k8s-installation) 및 [Linux 호스트용 뉴렐릭 eBPF 에이전트 설치](/docs/ebpf/linux-installation) 를 업데이트했습니다.

### 사소한 변경 사항

* [대시보드](/docs/query-your-data/explore-query-data/dashboards/share-dashboards-publicly/) 및 [차트](/docs/query-your-data/explore-query-data/dashboards/share-charts-publicly/)의 공개 공유를 위한 전용 페이지를 제공하도록 대시보드 및 차트 공유 문서를 재구성했습니다.
* 성능 위험 수신함에서 성능 위험 그룹을 가져오기 위해 [MCP 도구 참조](/docs/agentic-ai/mcp/tool-reference) 에 `list_entity_performance_risk_groups` 도구를 추가했습니다.
* 추가 컨텍스트에 대한 중요한 콜아웃을 포함하여 [setMaxEventPoolSize](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-pool-size) 모바일 SDK 참조를 업데이트했습니다.
* 최신 지원 프레임워크 버전을 반영하여 [.NET 에이전트 호환성 요구 사항](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) 을 업데이트했습니다.
* 현재 대시보드 이름을 반영하도록 [NGINX 통합](/docs/opentelemetry/integrations/nginx/find-and-query-your-data) 및 [NGINX Plus 통합](/docs/opentelemetry/integrations/nginx-plus/nginx-plus-otel) 문서에서 nginx 대시보드 이름을 수정했습니다.
* 모든 전체 플랫폼 사용자에게 추가 비용 없이 SAI 기능이 포함된다는 점을 명확히 하기 위해 [뉴렐릭 One 가격 책정 및 청구](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) 와 [서비스 아키텍처 인텔리전스 시작하기](/docs/service-architecture-intelligence/getting-started) 를 업데이트했습니다.
* Chrome 버전 참조를 &quot;최신&quot;으로 일원화하도록 [새로운 런타임](/docs/synthetics/synthetic-monitoring/using-monitors/new-runtime), [모니터 런타임 관리](/docs/synthetics/synthetic-monitoring/using-monitors/manage-monitor-runtimes) 및 [런타임 업그레이드 문제 진단, 해결](/docs/synthetics/synthetic-monitoring/troubleshooting/runtime-upgrade-troubleshooting) 을 업데이트했습니다.
* 런타임 전환에 대한 특정 EOL 날짜를 포함하도록 [신세틱 모니터링 런타임 전환 가이드](/docs/synthetics/synthetic-monitoring/private-locations/runtime-transition-guide) 를 업데이트했습니다.
* 내부 앱 이름 참조를 제거하여 [NRQL 구문, 절 및 함수](/docs/nrql/nrql-syntax-clauses-functions) 의 포맷 문제를 수정했습니다.
* [Node.js 에이전트 업데이트](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent)에서 코드 블록 들여쓰기를 수정했습니다.

### 새로운 게시물은 무엇인가요?

* 서비스 수준 관리에서 예약된 유지 관리 Windows 및 패싯 기반 SLO 규정 준수 추적의 정식 출시(GA)를 알리기 위해 [SLM 업데이트: 유지 관리 Windows 및 패싯 규정 준수 정식 출시](/whats-new/2026/05/whats-new-05-18-slm-maintenance-windows-facet-compliance) 를 추가했습니다.

    * 크로스 애플리케이션 추적(CAT) 및 라이선스, 애플리케이션, 보안 정책(LASP) 지원을 제거했습니다.

    * `@nestjs/cli`, `@nestjs/core`, `next`, `bluebird`, `mysql2` 및 `cassandra-driver`에 대해 지원되는 최소 버전을 업데이트했습니다.
    * `config.distributed_tracing.exclude_newrelic_header` 기본값이 `true`(으)로 업데이트되었습니다.

  * [Node API 런타임 rc1.21](/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.21):

  * [노드 브라우저 런타임 rc1.21](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.21):

    * CVE-2026-3832, CVE-2026-3833, CVE-2026-5260 등을 포함한 여러 보안 취약점을 수정했습니다.
