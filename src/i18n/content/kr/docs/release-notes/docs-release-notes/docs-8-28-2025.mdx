### 새로운 문서

* 사용자가 지표 관련 지표에 접근하는 것을 제어하고 지표 수집 한도에 대한 권한을 [관리하기 위해 지표를 관리하는 액세스 관리 기능이](/docs/data-apis/ingest-apis/metric-api/cardinality-access-management/) 추가되었습니다.
* 데이터 수집을 최적화하고 예상치 못한 비용을 피하기 위해 지표 기수 사용을 추적하고 모니터링하는 [모니터 기수 기능이](/docs/data-apis/ingest-apis/metric-api/monitor-cardinality/) 추가되었습니다.

### 사소한 변경 사항

* ebpf에 대한 [새로운 설정 방법인](/docs/ebpf/linux-installation/#config-params) `DROP_DATA_FOR_ENTITY` 을 추가했습니다.

* [Lambda 확장 v2.3.22](/docs/release-notes/lambda-extension-release-notes/lambda-extension-2.3.22)

  * `APM` 연결 재시도 논리를 추가했습니다.
  * AWS 방화벽 문제를 해결하기 위해 `godebug` 을 추가했습니다.
  * `Node.js` 런타임에 대한 로그에서 `requestId` 추출을 추가했습니다.
  * `--enable-license-key-secret` 에 대한 Readme가 업데이트되었습니다.

* [Lambda 확장 v2.3.23](/docs/release-notes/lambda-extension-release-notes/lambda-extension-2.3.23)

  * 로컬 테스트를 위해 `nodejs` 테스트 레이어를 빌드하는 옵션이 추가되었습니다.
  * 보안 취약점 `[CVE-2020-8911, CVE-2020-8912, CVE-2025-0913, CVE-2025-4673]` 을 해결하기 위해 AWS SDK 및 Go 버전을 업데이트했습니다.

* [루비 에이전트 v9.21.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-9-21-0)

  * Serverless APM 모드에서는 이제 트랜잭션 이름에 이벤트 소스 이름이 접두사로 포함됩니다. 이 향상된 기능은 트리거링 이벤트 소스를 명확하게 표시하여 프로세서의 식별 및 추적 가능성을 뉴렐릭 UI 에서 향상시킵니다.
  * 스팬 초기화를 위한 스레드 추적 논리의 변경 사항을 되돌렸습니다. 이 변경 사항은 스팬이 시작되는 스레드를 추적하는 이전 동작을 복원하여 업데이트된 논리에서 발생한 문제를 해결합니다.
