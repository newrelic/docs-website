### 새로운 문서

* [Amazon MemoryDB 통합은](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-memorydb-monitoring-integration) 뉴렐릭으로 통합을 활성화하는 방법을 설명합니다. 통합을 사용하면 Amazon MemoryDB 클러스터에 대한 텔레메트리 데이터를 수집하고, 수신 데이터를 쿼리하고, 대시보드를 구축할 수 있습니다.
* [애플리케이션 로그에 사용자 지정 태그(레이블) 추가하기에서는](/docs/logs/logs-context/custom-tags-agent-forwarder-logs) 에이전트가 전달하는 로그에 사용자 지정 레이블을 추가할 수 있다고 설명합니다.

### 주요 변경 사항

* 명확성과 구성을 위해 [AWS 통합 문서를](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations) 업데이트했습니다. 주요 변경 사항에는 CloudWatch Metric Streams를 주요 AWS 통합 방법으로 강조하고, 탐색 기능을 개선하고 통합 및 콘텐츠 간소화를 통해 중복을 줄이며, 가독성을 높이기 위해 내용을 수정하는 것이 포함됩니다.
* [사후 분석 보고서 작성](/docs/alerts/alert-event-management/postmortems-applied-intelligence/#create-postmortem) 절차를 업데이트하여 정확성을 높였습니다.
* 뉴렐릭의 2024년 연례 평가를 바탕으로 [인증, 표준 및 규정](/docs/security/security-privacy/compliance/regulatory-audits-new-relic-services) 에 대한 준수 관련 업데이트를 진행했습니다.
* .NET 에이전트에 대한 .NET 9 및 ASP.NET Core 9 호환성을 추가하고, 지원되지 않는 .NET 버전에 대한 지원 입장을 명확히 하고, [.NET 에이전트 대응 및 요구 사항](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) 에서 오래된 .NET 프레임워크 참조를 제거했습니다.
* 명확성을 위해 [로그인 포워딩에 대한Fluent Bit 플러그인을](/docs/logs/forward-logs/fluent-bit-plugin-log-forwarding) 다시 작성했습니다.
* 정확성을 위해 [CodeStream 을](/docs/codestream/codestream-settings/account-settings) 업데이트했습니다.
* cloud 공급자와 APM 의 관계를 생성하기 위해 [Node.js 에이전트 변수](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#cloud_config) 세부 정보를 추가했습니다.
* [Azure 이벤트 허브 생성](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export/#create-event-hub) 단계를 업데이트했습니다.

### 사소한 변경 사항

* [사용자 정의 태그(레이블)를 로그 로그에 추가](/docs/logs/logs-context/custom-tags-agent-forwarder-logs) 에서 .NET 및 루비 에이전트 버전을 업데이트했습니다.
* Unity [모바일 로그](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs) 에서 로깅 API를 사용하는 방법을 보여주는 예시를 제공했습니다.
* [.NET 에이전트 호환성 및 요구 사항이](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) 업데이트되었습니다.
* [Node.js 에이전트 호환성 및 요구 사항이](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) 업데이트되었습니다.
* [관측 이벤트](/docs/infrastructure/infrastructure-data/default-infra-data) 의 네트워크 장치 포함 및 NetworkSample 생성에 대한 기본 동작 및 설정 지침이 업데이트되었습니다.
* `localStorage` API의 이름을 수정하고 세션 추적이 작동하지 않을 수 있는 경우에 대한 정보를 [세션 추적 섹션](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking) 에 업데이트했습니다.
* Node.js 에이전트 구성 세부 정보를 업데이트하여 애플리케이션 로그에 레이블이 추가되도록 [설정](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels) 하거나 [제외했습니다](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels-exclude).
