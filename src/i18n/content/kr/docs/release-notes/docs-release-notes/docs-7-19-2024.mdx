### 주요 변경 사항

* [.NET 에이전트의 분기 설치](/install/dotnet/) 에 대한 최종 업데이트 및 개선 사항을 적용하여 원하는 정보를 더 쉽게 찾고 앱 모니터링을 설정할 수 있도록 했습니다.
* [NRQL 참조](/docs/nrql/nrql-syntax-clauses-functions/#func-histogram) 에는 추가 인수와 기본값이 추가되었으며, [히스토그램 응답 시간](histogram-response-times-defaults) 도 업데이트되어 이러한 활력자 기능과 관련된 쿼리를 더 쉽게 작성할 수 있게 되었습니다.
* 저희 UI에서는 `Add data` `Integrations & Agents` 로 대체했습니다. 이러한 변경 사항을 반영하여 문서 사이트를 전면 개편했습니다.
* [NerdGraph](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-account) 및 [NerdGraph 튜토리얼 &quot;계정 보기, 생성, 이름 변경 또는 취소&quot;](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph) 에서 계정 취소와 조직 삭제의 차이점을 명확히 했습니다.

### 사소한 변경 사항

* [차트 및 대시보드: 데이터 시각화에](/docs/nrql/get-started/charts-and-dashboards-with-nrql/#performance-improvement-for-dashboard-and-charts) 대시보드 및 차트 성능을 개선하는 방법에 대한 팁이 추가되었습니다.
* [인프라 에이전트 구성 설정](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#process-container-decoration) 에 `process_container_decoration` 활성화된 경우 에이전트가 프로세스 샘플에 컨테이너 정보를 추가하는 방법을 설명하는 섹션이 추가되었습니다.
* [.NET Core에 대한 .NET 성능 및 요구 사항](/docs/apm/agents/net-agent/configuration/net-agent-configuration/) 의 호환 버전 정보가 단순화되었습니다.
* 고보안 모드가 활성화된 경우 설정 옵션이 비활성화될 때 이를 알리는 참고 사항을 여러 Python APM 설정 문서에 추가했습니다.
* [이슈 및 공지 이벤트 관리 및 대응](/docs/alerts/alert-event-management/Issues-and-alert-event-management-and-response/#search-filters) 에 경고 및 중요 알림 이벤트가 생성되는 시기를 명시하는 메모가 추가되었습니다.
* [컨테이너화되지 않은 Lambda 함수](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/instrument-your-own/) 의 상단 다이어그램을 업데이트했습니다.
* &quot;RightScale 사용자 및 뉴렐릭&quot;을 삭제했습니다. RightScale은 오래전에 해체된 파트너십이었습니다.
* ServiceNow ITSM은 구형 알림 공지 통합임을 참고하세요. [ServiceNow에서 인증한 통합 기능을](/docs/alerts/get-notified/notification-integrations/#servicenow-certified) 사용하십시오.
* [Kubernetes 통합 버전 2](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/k8s-version2/overview/) 관련 문서를 모두 통합하여 사이트 내 하나의 하위 카테고리로 이동했습니다. Kubernetes 버전 3이 최신 버전이므로 해당 버전을 사용하시는 것을 권장합니다.
* 기존 키는 더 이상 사용되지 않으므로 [REST API 키를 변경할](/docs/apis/intro-apis/new-relic-api-keys/#rotate-rest-api-key) 수 없습니다. REST API 키를 사용 중이시라면 [해당 키를 삭제하고 다른 키를](/docs/apis/intro-apis/new-relic-api-keys/#rotate-keys) 사용하세요.
