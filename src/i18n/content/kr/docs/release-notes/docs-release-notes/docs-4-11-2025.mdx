* [브라우저 로그](/docs/browser/browser-monitoring/browser-pro-features/browser-logs/overview) 캡처 브라우저 콘솔 로그를 통해 옵저버빌리티를 향상하고 디버깅을 단순화하며 사용자 경험을 향상할 수 있습니다.

* [AWS 로그인 컨텍스트는](/docs/logs/logs-context/aws-logs-in-context) AWS 환경 내에서 문제 해결, 처리 효율성 및 시스템 옵저버빌리티를 향상하는 포괄적인 통합 로그인 관리 솔루션을 제공합니다.
* [에이전트 제어를](/docs/new-relic-control/agent-control/reinstrumentation) 사용하여 현재 측정값을 관리하고 다시 설치하세요.
* [Agentic Control은](/docs/new-relic-control/agent-control/overview) 출력 수준 측정, 출력량 모니터의 상태를 원격으로 구성, 업데이트 및 관리하기 위해 플루릿과 통신하여 OpenTelemetry 에이전트의 다른 뉴렐릭 및 뉴렐릭 배포를 관리하는 경량 에이전트 감독자입니다.
* [Pipeline 제어를](/docs/new-relic-control/pipeline-control/overview) 사용하면 모니터 부분에서 뉴렐릭으로의 데이터 흐름을 관리할 수 있습니다.
* [도커 컨테이너](/docs/infrastructure/host-integrations/host-integrations-list/docker-container-monitoring-integration) 통합 모니터 및 도커 컨테이너의 지표를 뉴렐릭으로 보고합니다.

### 주요 변경 사항

* 에 대해 [지원되는](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships) 관계 로 [Amazon,CloudWatch Metric Streams](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams) [ElastiCache(Memcached),](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached) [Amazon MQ(ActiveMQ) 를 추가했습니다.](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) OpenTelemetry

* CloudWatch Metric Streams [Amazon Managed Kafka(MSK)를](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-managed-kafka-msk) 생성하기 위해 Amazon MSK와 함께 추가했습니다. 이러한 경고는 `aws.clusterName` 또는 `aws.kafka.ClusterName`, `aws.awsRegion` 또는 `aws.region`, `aws.topic` 또는 `aws.kafka.Topic` 가 있는 태그입니다.

* 버전 8.10의 지표 변경 사항에 대한 [Solr 문제 해결, 해결](/docs/apm/agents/java-agent/troubleshooting/java-solr-troubleshooting) 지침이 추가되었습니다.

* 뉴렐릭은 다음 IP 범위 사용을 중단했습니다.

  * 3.145.244.128/25
  * 3.77.79.0/25
  * 3.27.118.128/25
  * 20.51.136.0/25
  * 4.197.217.128/25
  * 18.246.82.0/25
  * 158.177.65.64/29
  * 159.122.103.184/29
  * 161.156.125.32/28

  2025년 5월 1일부터 이러한 범위는 cloud 제공업체가 당사가 통제할 수 없는 목적으로 다른 고객에게 재할당할 수 있습니다. [네트워크](/docs/new-relic-solutions/get-started/networks) 설정을 이에 맞게 업데이트하세요.

* 뉴렐릭과 함께 Secrets Manager를 사용하여 문제에 대한 문제 해결,[볼륨 키를 검색 하는 문서를 추가했습니다.](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/retrieve-license) AWS

* 문제 해결, [Lambda 함수에 대한 처리 문서가 HTML로 표시되지 않는 문제를](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-not-instrumented-lambda-function) 추가했습니다.

* [각 지역에 여러 AWS 계정을 추가하는 것과](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-unable-to-add-multiple-aws-accounts/) 관련된 문제에 대한 문제 진단, 해결 문서를 추가했습니다.

* [뉴렐릭과 함께 AWS Secrets Manager를 사용하여](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-using-aws-secrets-manager/) 문제 해결, 해결 문서를 추가했습니다.

* [FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp) 에 뉴렐릭 Agentic 통합이 추가되었습니다.

* [뉴렐릭의 EC2 Image Builder 통합](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-ec2-image-builder-integration) 에 대한 새 문서를 추가했습니다.

* 에 추가했습니다.[Dashboards](/docs/codestream/observability/dashboards) CodeStream

### 사소한 변경

* 메시지 전송을 위한 분산 추적은 버전 2.1.0부터 지원됩니다. [프레임워크와 흰 모니터](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#frameworks) 의 최신 소식을 확인하세요.
* AWS Glue Spark ETL 작업에는 [AWS Glue 모니터링 통합](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-glue-monitoring-integration) 의 끌어올리기 Shell 작업에 대한 제한 사항이 있습니다.
* 현재 [매출세가](/docs/accounts/accounts/billing/view-or-change-account-tax-information) 부과되는 지역에 캐나다(브리티시 컬럼비아주와 서스캐처원주), 시카고, 컬럼비아 특별구, 일리노이주, 버몬트주를 추가했습니다.
* 영감된 Azure Functions에 대한 Windows 옵션을 [Azure Functions 렐릭 .NET 에이전트로](/docs/serverless-function-monitoring/azure-function-monitoring/install-serverless-azure-monitoring/#instrument-your-azure-functions-with-the-new-relic-net-agent) 업데이트했습니다.
* 뉴렐릭 컨테이너의 계층화 및 SDK 측정, 이제 `com.amazonaws:aws-lambda-java-events` 버전 3.0.0과 호환됩니다. 그리고 나중에.
* [간단한 외부 GUID와 계측되지 않은 GUID가](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/#handling-legacy-externals-and-uninstrumented-entities-exceptions-in-entity-relationships) 모두 나타날 수 있는 예외적인 사례를 추가했습니다.
* 서비스 맵 페이지에서 바로 [새로운 맵](/docs/service-architecture-intelligence/maps/advanced-maps) 환경에 온보딩하기 위한 지침이 추가되었습니다.
* [지원되는 시간](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings) 섹션을 업데이트했습니다.
* [dSYM](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/upload-dsyms-bitcode-apps) 파일은 이제 각각 1GB로 제한됩니다.
* [AWS Lambda에 대한 추가 환경 변수](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda/#nodejs) 로 `NODE_OPTIONS` 추가했습니다.
* 에이전트 설정 [및 에이전트](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file) [전달 로그 레이블 정보에](/docs/apm/agents/java-agent/configuration/java-agent-config-file-template) 로그 레이블을 추가했습니다.
* 필요할 때 전체 키를 관리하고 얻는 방법에 대한 세부 정보를 담은 [뉴렐릭 API 키](/docs/apis/intro-apis/new-relic-api-keys) 섹션을 업데이트했습니다.
* [군대 에이전트 8.18.0](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) 의 사용자는 `newrelic.yml` 파일에서 SQS 측정, 로그를 수동으로 활성화해야 합니다.
* `NEW_RELIC_USE_ESM` 환경 변수는 Node.js [Lambda 레이어](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda) 의 향후 버전에서 제거될 예정입니다.
* [정책 한도당 조건을](/docs/alerts/admin/rules-limits-alerts) 500개에서 1000개로 업데이트했습니다.
* 2025년 3월 3일 월요일부터 [Kubernetes 통합은](/eol/2025/03/deprecation-notice-v1.27-and-lower) Kubernetes v1.27 및 이전 버전을 더 이상 지원하지 않습니다. Kubernetes 통합 v3.40.1 이상은 Kubernetes 버전 1.28 이상과만 호환됩니다.
* [인어 다이어그램 지원](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#markdown) 정보를 복구했습니다.
* [Advanced 및 Core 컴퓨터 가격 책정 계획](/docs/alerts/create-alert/create-alert-condition/alert-conditions) 에 대한 관련 가격 세부 정보가 추가되었습니다.
* 특수 문자 지원에 대한 정보를 포함하도록 [이메일 계정 설정 문서를](/docs/accounts/accounts/account-maintenance/account-email-settings/#change-email) 업데이트했습니다.
* 동일한 클러스터에 여러 Agent Control 설치가 지원되지 않음을 명확히 하기 위해 [Agent Control 문서를](/docs/new-relic-control/agent-control/overview/) 업데이트했습니다.
* 무한 추적을 위한 배치 및 압축 설정을 포함하도록 [Node.js 에이전트 구성 문서를](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration) 업데이트했습니다.
* `plugin.type:"syslog-newrelic"` 속성을 검색하여 로그를 찾는 방법에 대한 정보를 포함하도록 [TCP 엔드포인트 사용 문서를](/docs/logs/log-api/use-tcp-endpoint-forward-logs-new-relic) 업데이트했습니다.
* [Agentic AI 문서](/docs/agentic-ai/agentic-integration/introduction-agentic-ai) 에 새로운 액세스 및 데이터 관리 섹션을 추가했습니다.
* [TCP 엔드포인트 문서](/docs/logs/log-api/use-tcp-endpoint-forward-logs-new-relic) 에서 로그 API의 제한 사항을 추가했습니다.
* 구현, 배포 후 파드를 다시 시작하는 것의 중요성을 강조하기 위해 [Kubernetes 문서를](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes) 업데이트했습니다.
* [Azure Functions](/docs/serverless-function-monitoring/azure-function-monitoring/install-serverless-azure-monitoring) 에 대한 환경 변수를 업데이트했습니다.
* 명확성을 높이기 위해 [뉴렐릭 OTLP 속성 처리 문서를](/docs/opentelemetry/best-practices/opentelemetry-otlp-attributes-processing-update) 업데이트했습니다.
