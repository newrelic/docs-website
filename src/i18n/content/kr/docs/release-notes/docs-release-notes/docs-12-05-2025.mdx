### 새로운 문서

* Berkeley Packet Filter 모니터링 기능을 확장하기 위한 포괄적인 지침을 제공하기 위해 [eBPF eAPM 및 eBPF 네트워크 모니터링](/docs/ebpf/overview) 설명서를 추가했습니다.
* APM + Lambda Serverless 융합에 대한 사용자 안내를 위해 [APM 경험 설명서에 업그레이드를](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/upgrade-to-apm-experience) 추가했습니다.

### 주요 변경 사항

* 브라우저 및 모바일 애플리케이션을 위한 향상된 오류 추적 기능을 포함하도록 광범위한 개정을 거쳐 [영향을 받은 세션 및 장치 메트릭 문서가](/docs/errors-inbox/error-impacted) 업데이트되었습니다.

* 최신 호환성 정보를 담은 [Node.js 호환성 보고서가](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) 업데이트되었습니다.

* 현재 프레임워크 지원 세부 정보로 [대응 대응 문서가](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent) 업데이트되었습니다.

* 최신 프레임워크 호환성 정보를 사용하여 [.NET 호환성 문서를](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/) 업데이트했습니다.

* 개선된 에이전트 설정 안내를 위해 [루비 설정 문서가](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration) 업데이트되었습니다.

* 다양한 플랫폼 전반의 영향 지표에 대한 포괄적인 적용 범위와 조직 개선을 위해 [오류 사용자 영향 문서가](/docs/errors-inbox/error-impacted) 업데이트되었습니다.

  * 세션 및 영향을 받은 기기의 메트릭을 포함한 향상된 [브라우저 오류 추적](/docs/errors-inbox/browser-tab) 문서.
  * 영향을 받은 세션 및 장치에 대한 향상된 [모바일 오류 추적](/docs/errors-inbox/mobile-tab) 문서입니다.

* 모니터링 모니터링을 위한 새로운 IP 주소 범위로 [신세틱 모니터링 퍼블릭 미니언 IP가](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips) 업데이트되었습니다.

### 사소한 변경 사항

* [데이터 보존 문서](/docs/data-apis/manage-data/manage-data-retention) 에 AI 모니터링 네임스페이스 보존 값을 추가했습니다.
* 시뮬레이션과 실시간 상관관계에 대한 지침을 명확히 하기 위해 [의사 결정 문서를](/docs/alerts/organize-alerts/change-applied-intelligence-correlation-logic-decisions) 업데이트했습니다.
* 향상된 정확도 정보를 통해 [uniqueCount 함수](/docs/nrql/nrql-syntax-clauses-functions/) 에 대한 해시 충돌 문서가 수정되었습니다.
