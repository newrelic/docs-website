### 새로운 문서

* OpenTelemetry 사용하여RabbitMQ 모니터링하기 위한 지침을 제공하기 위해 [RabbitMQ OpenTelemetry 측정 측정을](/docs/opentelemetry/integrations/rabbitmq/overview) 추가했습니다.
* OpenTelemetry 를 사용하여 도커 컨테이너 모니터링에 대한 지침을 제공하기 위해 [도커 OpenTelemetry 모니터링을](/docs/opentelemetry/integrations/docker-monitoring/overview) 추가했습니다.
* 사용자 지정 코드 없이 로그를 파싱하는 방법에 대한 지침을 제공하기 위해 [노코드 파싱](/docs/logs/ui-data/parsing) 문서를 추가했습니다.
* 텔메트리 데이터 수집 최적화에 대한 지침을 제공하기 위해 [적응형 텔메트리 프로세서(ATP)](/docs/opentelemetry/nrdot/atp/overview) 문서가 추가되었습니다.
* Azure 자동화 작업에 대한 지침을 제공하기 위해 [Azure 실행 작업 카탈로그가](/docs/workflow-automation/setup-and-configuration/actions-catalog/azure/azure-execute-api) 추가되었습니다.
* [Bitmovin 및 Shaka](/docs/release-notes/streaming-browser-release-notes) 플레이어 통합 관련 미디어 스트리밍 릴리스 노트를 추가했습니다.

### 주요 변경 사항

* 답변 제목 및 마크다운 답변에 대한 예시가 포함된 [대시보드 템플릿 변수가](/docs/query-your-data/explore-query-data/dashboards/dashboard-template-variables) 업데이트되었습니다.
* OpenTelemetry에서 Prometheus 통합으로 마이그레이션하기 위해 [OpenTelemetry Kafka Strimzi를](/docs/opentelemetry/integrations/kafka/kubernetes-strimzi) 업데이트했습니다.
* NRDOT 수집기 설정으로 [OpenTelemetry Kafka 문서를](/docs/opentelemetry/integrations/kafka/overview) 업데이트했습니다.
* 일관성 향상을 위해 &apos;인시던트&apos;를 &apos;공지 이벤트&apos; 용어로 대체하도록 알림 문서를 업데이트했습니다.

### 새로운 게시물은 무엇인가요?

* [노코드 파싱 기능의 정식 출시를 알리기 위해 &apos;새로운 기능&apos; 게시물을](/whats-new) 추가했습니다.

### 사소한 변경 사항

* [iOS 에이전트 문서](/docs/mobile-monitoring/new-relic-mobile-ios/get-started/introduction-new-relic-mobile-ios) 에 스위프트 및 오브젝티브-C 코드 샘플을 추가했습니다.

* NGINX OpenTelemetry 기본 최적화 설정을 [NGINX OTel 문서](/docs/opentelemetry/integrations/nginx/nginx-otel-overview) 에 추가했습니다.
* Kubernetes용 [APM 애플리케이션 연결](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes) 지침이 업데이트되었습니다.
* [글로벌 기술 지원 제공 관련](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings) 문서가 업데이트되었습니다.
* 최신 프레임워크 지원을 포함하여 [.NET 에이전트 호환성 문서를](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) 업데이트했습니다.
* 최신 모듈 지원을 반영하여 [Node.js 에이전트 호환성 문서를](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) 업데이트했습니다.
