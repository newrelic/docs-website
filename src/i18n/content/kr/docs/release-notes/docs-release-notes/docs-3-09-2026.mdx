### 새로운 문서

* AI 기반 SRE 기능을 활용하는 데 필요한 포괄적인 지침을 제공하기 위해 [SRE 에이전트 개요를](/docs/agentic-ai/sre-agent/overview) 추가했습니다.
* [SRE 에이전트 설정 부분을](/docs/agentic-ai/sre-agent/sre-agent-setup) 추가하여 SRE 에이전트 구성에 대한 단계별 지침을 제공합니다.
* SRE 에이전트 기능 활용에 대한 지침을 제공하기 위해 [&quot;SRE 에이전트 사용&quot; 섹션을](/docs/agentic-ai/sre-agent/use-sre-agent) 추가했습니다.
* [AI 에이전트 성능 모니터링 및 분석에 대한 지침을 제공하기 위해 View AI 에이전트가](/docs/ai-monitoring/explore-ai-data/view-ai-agents) 추가되었습니다.
* Microsoft SQL Server 데이터베이스 모니터링에 대한 포괄적인 지침을 제공하기 위해 [OpenTelemetry를 사용한 MSSQL 모니터링 기능을](/docs/opentelemetry/database/otel-mssql) 추가했습니다.
* 계획 작업 데이터베이스에 대한 포괄적인 지침을 제공하기 위해 [OpenTelemetry사용하여 작업 데이터베이스 모델링을](/docs/opentelemetry/database/otel-oracledb) 추가했습니다.
* [데이터베이스 서비스 식별 및 구성에 대한 지침을 제공하기 위해 데이터베이스 서비스 식별이](/docs/opentelemetry/database/database-service-identification) 추가되었습니다.

### 주요 변경 사항

* CLI, GitHub Actions 및 NerdGraph에 대한 새로운 설정 섹션을 포함하여 향상된 조직 및 유용성을 위해 재구성된 [변경 추적 문서입니다](/docs/change-tracking/overview).
* 업데이트된 [변경 추적 보기 및 데이터 분석 기능은](/docs/change-tracking/view-analyze-data) 명확성을 개선하고 새로운 시각화 기능을 추가하는 등 대대적인 수정을 거쳤습니다.
* 쿼리 구현, 배포 데이터에 대한 포괄적인 예제와 개선된 지침을 사용하여 [추적 쿼리 데이터를](/docs/change-tracking/query-data) 업데이트했습니다.
* 여러 AWS 계정을 관리하는 방법에 대한 명확한 지침을 제공하기 위해 [AWS 다중 계정 관련 문서를](/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/aws-multi-account) 개편했습니다.
* [eBPF 네트워크가](/docs/ebpf/network-metrics) 업데이트되었으며, 문서화가 대폭 개선되고 새로운 모니터링 기능이 추가되었습니다.
* eBPF를 사용한 향상된 애플리케이션 성능 모니터링을 위해 [eBPF APM 통합 관련](/docs/ebpf/ebpf-apm) 문서를 추가했습니다.

### 사소한 변경 사항

* 구현, 배포 및 설정에 대한 향상된 권장 사항으로 [eBPF 모범 사례가](/docs/ebpf/best-practices) 업데이트되었습니다.
* 명확한 설치 단계로 [eBPF Kubernetes 설치가](/docs/ebpf/k8s-installation) 업데이트되었습니다.
* 추가 설정 옵션으로 [eBPF Linux 설치가](/docs/ebpf/linux-installation) 업데이트되었습니다.
* 기능 설명이 강화된 업데이트된 [eBPF 개요](/docs/ebpf/overview) 입니다.
* [eBPF 요구사항이](/docs/ebpf/requirements) 업데이트되었으며, 시스템 필수 조건이 추가되었습니다.
* [SAP 릴리스 노트](/docs/release-notes/sap-release-notes) 에 포괄적인 릴리스 이력 및 패치 정보를 추가했습니다.
* [API 키 용어를](/docs/apis/intro-apis/new-relic-api-keys) &quot;회전&quot;에서 &quot;교체&quot;로 업데이트하여 명확성을 높였습니다.
* 계절성 설정 설정에 [이상하게 감지 예시를](/docs/alerts/create-alert/set-thresholds/anomaly-detection) 추가했습니다.
* 더 이상 사용되지 않는 HTTP/RPC 페이지를 제거하여 [Collector 옵저버빌리티를](/docs/opentelemetry/collector-observability/collector-observability) 업데이트했습니다.
* 팀 계층 구조 및 관리자 관리 예시를 포함하여 [NerdGraph 팀 튜토리얼이](/docs/apis/nerdgraph/examples/nerdgraph-teams-tutorial) 업데이트되었습니다.
* 향상된 설정 지침으로 [VMware Tanzu .NET 빌드팩 통합이](/docs/infrastructure/other-infrastructure-integrations/cloudfoundry-integrations/vmware-tanzu-dotnet-buildpack-integration) 업데이트되었습니다.
* 수정된 설정 세부 정보로 [VMware Tanzu Firehose Nozzle 통합이](/docs/infrastructure/other-infrastructure-integrations/cloudfoundry-integrations/vmware-tanzu-firehose-nozzle-integration) 업데이트되었습니다.
* 서식과 명확성을 개선하여 [워크플로우 공지 자동화 예제를](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) 업데이트했습니다.
* 예제와 문서가 향상된 업데이트된 [자동화 정의 스키마입니다](/docs/workflow-automation/workflow-automation-apis/definition-schema).
* 개선된 안내로 [워크플로우 자동화 범례를](/docs/workflow-automation/limitations-and-faq/workflow-best-practices) 업데이트했습니다.
* 수정된 이벤트 유형 코드로 [Kubernetes 비용 속성이](/docs/cci/financial-intelligence/nrql-for-cci) 업데이트되었습니다.
* [인증 도메인 문서](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 에서 세션 시간 초과 설정 관련 문서를 삭제하고, 세션 시간 초과 구성 지침을 추가했습니다.
* 시스템 제한 정보를 사용하여 [자동화 제한을](/docs/workflow-automation/limitations-and-faq/workflow-limits) 업데이트했습니다.
* 사용 중단 일정과 함께 [해당 Kubernetes 이미지에 대한 EOL 공지를](/eol/2026/03/eol-03-31-26-legacy-k8s-images) 추가했습니다.
