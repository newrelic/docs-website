### 새로운 문서

* [메트릭 및 이벤트 탐색기](/eol/2025/05/metrics-and-event-explorer/) 의 수명 종료 날짜가 2025년 6월 16일로 발표되었습니다.
* 안드로이드 에이전트를 통해 [고객 바이오리조트를 생성하는 지침이](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-custom-Interaction/) 추가되었습니다.

### 주요 변경 사항

* [Node.js 에이전트 설정](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#filename) 에 `NEW_RELIC_CONFIG_FILENAME` 환경 변수 재정의를 추가했습니다.
* 추적 부모에 트레이스 플래그가 설정된 경우 원격 부모와 함께 트랜잭션 샘플링 동작을 관리하는 루비 에이전트의 사역 처리 [설정](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/#distributed_tracing-sampler-remote_parent_sampled) 에 설정을 추가했습니다.
* [AI 모니터링에 대한 호환성 및 요구 사항](/docs/ai-monitoring/compatibility-requirements-ai-monitoring/) 페이지에 끌어 당기는 저항에 대해 지원되는 라이브러리로 Google Gen AI SDK를 추가했습니다.
* [공지 조건 페이지](/docs/alerts/create-alert/create-alert-condition/alert-conditions/#disable-health-status-reporting) 에 공지 공지 발생 시 일정 상태의 자동 변경을 방지할 수 있는 기능이 추가되었습니다.
* [로그 UI사용](/docs/logs/ui-data/use-logs-ui/#customizable-table-views) 페이지에서 로그 테이블 뷰 사용자 정의 기능이 향상되었습니다.
* 에이전트 컨트롤을 설치하거나 작동할 때 발생하는 일반적인 문제를 해결하기 위해 [NRDiag를 사용하는 방법](/docs/new-relic-control/agent-control/troubleshooting/#nrdiag) 에 대한 정보를 추가했습니다.
* JSON IP 범위 목록에서 이전 [합성 모니터 퍼블릭 미니언 IP](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips/#dynamic-ip-addresses) 범위를 제거하기 위해 수명 종료 날짜를 2025년 5월 21일로 연장했습니다.
* Linux 환경에서만 인프라 에이전트 및 온호스트 통합 지원 ARM64를 반영하도록 [ARM64/Graviton 및 x86\_64용 에이전트 지원](/docs/apm/agents/manage-apm-agents/configuration/support-arm-graviton-x86-64/) 문서를 업데이트했습니다.
* [사용자 삭제](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/#delete-users) Dashboards 시 삭제되는 자산 목록에서 제거했습니다.
* 뉴렐릭 플랫폼의 [ANR(기능 응답 없음) UI](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding/#view-anr-new-relic) 에 대한 새로운 사용자 기능에 대한 정보를 추가했습니다.
* [](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/eligibility-guidelines-new-relic-nonprofit-program/#exceptions)뉴렐릭의 Observability for Good 프로그램에 대한 자격 지침이 업데이트되었습니다.

### 사소한 변경 사항

* REST API v2 예제에 대한 대체 NerdGraph 또는 NRQL 함수 참조가 추가되었습니다.
* [Node.js 에이전트의 호환성 및 요구 사항이](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/) 업데이트되었습니다.
* [끌어당김 에이전트의 호환성 및 요구 사항을](/docs/apm/agents/python-agent/getting-started/compatibility-requirements-python-agent/) 업데이트했습니다.
* [.NET 에이전트의 호환성 및 요구 사항을](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/) 업데이트합니다.
* [FedRAMP 인증을 받지 않은 서비스](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp/#not-scope) 에 대한 소개 페이지에 대한 링크를 추가했습니다.
* [](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/upload-dsyms-bitcode-apps/#mobile-ui)뉴렐릭 UI 에서 dSYM 업로드 버튼의 가시성과 관련된 조건을 추가했습니다.
* `Microsoft.DocumentDB/mongoClusters` 이 Azure Cosmos DB(Document DB) 모니터링 통합에서 지원되지 않는다는 사실을 반영하여 [설명서를](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-cosmos-db-document-db-monitoring-integration/#metric-data) 업데이트했습니다.

### 릴리즈 정보

새로운 기능과 릴리스에 대한 정보를 알아보려면 &apos;새로운 소식&apos; 게시물을 확인하세요.

* [로그 테이블 설정 기능](/whats-new/2025/05/whats-new-05-06-log-table-settings/) 에 대한 새로운 소식입니다.

최신 릴리스에 대한 최신 정보를 받아보세요.

* [.NET 에이전트 v10.41.0](/docs/release-notes/agent-release-notes/net-release-notes/net-agent-10-41-0/):

  * 들어오는 W3C traceparent `sampled` 플래그가 .NET 에이전트 샘플링 결정에 영향을 미치도록 허용합니다.
  * Asp..NET Core 6+ 브라우저에서 null 참조 예외가 수정되었습니다.

* [루비 v9.19.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-9-19-0/):

  * 모든 스팬에 스레드 ID를 속성으로 추가했습니다.
  * W3C TraceContext 트레이스 플래그에 대한 지원이 추가되었습니다.
  * 기본적으로 Transaction 이벤트에 request.uri가 포함됩니다.
  * perform\_all\_later를 사용할 때 활성 작업 측정 및 과학적 오류를 수정했습니다.

* [끌어당기는 에이전트 v10.11.0](/docs/release-notes/agent-release-notes/python-release-notes/python-agent-101100/):

  * Google 생성형 AI SDK를 통해 Google Gemini에 대한 지원이 추가되었습니다.
  * 통계적으로 새로운 AWS Kinesis 메서드.

* [끌어당기는 에이전트 v10.12.0](/docs/release-notes/agent-release-notes/python-release-notes/python-agent-101200/):

  * 레디스 v6.0.0에 대한 지원이 추가되었습니다.
  * OpenTelemetry-proto 패키지와의 충돌이 해결되었습니다.

* [NRDOT v10.12.0](/docs/release-notes/nrdot-release-notes/nrdot-2025-05-06/):

  * 호스트와 k8s 배포판에 라우팅 커넥터를 추가했습니다.
  * otel 구성 요소를 0.125.0으로 업데이트했습니다.
  * 모든 호스트 파이프라인에 memory\_limiter를 추가합니다.
  * `nr-k8s-otel-collector 0.8.10` 와 동기화된 구성입니다.

* [Go 에이전트 v3.39.0](/docs/release-notes/agent-release-notes/go-release-notes/go-agent-3-39-0/):

  * Go Fiber 프레임워크에 nrfiber 통합이 추가되었습니다.
  * 새로운 API로 nrslog 예제를 업데이트했습니다.
  * nrgin 미들웨어에 선택적 경로 필터링 기능을 추가했습니다.
  * nrreddis 통합에 ConfigDatastoreKeysEnabled를 추가하여 데이터스토어 작업과 함께 키 이름을 보고할 수 있게 되었습니다.
  * nrslog의 속성 값에 대한 시간 개체 지원이 추가되었습니다.
  * 로그 메시지에서 링크 메타데이터 위치가 고정되었습니다.

* [브라우저 에이전트 v1.290.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.290.0/):

  * Bundler 도구 내보내기 도구가 추가되었습니다.
  * 집계가 끝나기 전에 UserAction currentUrl을 설정합니다.
  * 지연 로딩 최적화가 개선되었습니다.
  * 분할된 API 코드.

* [노드 브라우저 런타임 v3.0.30](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-3.0.30/):

  * 내부 Axios를 1.8.2로 업그레이드했습니다.

* [PHP v11.9.0.23](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-11-9-0-23/):

  * Drupal 11.1에 도입된 객체 지향 후크 구현의 자동 측정, 자동 로그를 추가합니다.
  * PHP 8.0 이상에 대한 사용자 함수 측정, 트랜잭션의 성능을 개선합니다.

* [Kubernetes 통합 v3.39.1](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-3-39-1/):

  * Kubernetes 패키지를 `v0.33.0` 로 업데이트했습니다.
