### 새로운 문서

* [FOSSA 통합은](/docs/vulnerability-management/integrations/fossa/) 이 새로운 통합 기능이 오픈 소스 라이브러리의 악용 가능한 취약점을 보여줌으로써 애플리케이션 보안 강화에 어떻게 도움이 되는지 설명합니다.

### 주요 변경 사항

* [인프라 문서 카테고리를](/docs/infrastructure/introduction-infra-monitoring/) 재구성하고 URL을 저장소 내 위치와 일치시켰습니다. 이러한 문서를 더 쉽게 찾고 작업할 수 있도록 하는 몇 가지 비공개 작업입니다.
* [.NET API](/docs/apm/agents/net-agent/net-agent-api/net-agent-api/) 를 대폭 개편하여 더 이상 사용되지 않는 정보를 제거하고, 일부 콘텐츠를 재구성하고, 예제 코드를 업데이트하고, API의 기능에 대한 자세한 설명을 추가하고, 샘플 코드 링크를 추가하고, 분산 추적 API에 ID 태그를 추가했습니다.

### 사소한 변경 사항

* [.NET 에이전트 호환성 및 요구 사항](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#database-core) 에 DynamoDB를 추가했습니다.
* 태그 `hostStatus: shutdown` 은 신호 상실 공지 이벤트를 방지합니다. [NRQL 공지 조건 생성](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/) 및 [신호 손실, 갭 채우기에 대한 NerdGraph 튜토리얼을](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/) 업데이트했습니다.
* 개선된 데이터 집계 기능을 통해 [기본 인프라 모니터링 데이터를](/docs/infrastructure/infrastructure-data/default-infra-data/) 업데이트했습니다.
* [Kubernetes에 대한 사용 중단 알림을](/eol/2024/10/deprecation-notice-v1.26-and-lower/) 추가했습니다.
* [&quot;호스트가 보고하지 않음&quot; 조건 생성 시 인프라](/docs/infrastructure/infrastructure-alerts/create-infrastructure-host-not-reporting-condition/) 의 가독성이 향상되었습니다.
* [레이어리스 측정, 계측](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/enable-serverless-monitoring-aws-lambda-layerless/) 에서 ES 모듈에 대한 예제와 확장 전용 레이어 설치를 위한 새로운 단계를 추가했습니다.
* [AI 모니터링 소개글을](/docs/ai-monitoring/intro-to-ai-monitoring/) 새로 작성하고 스크린샷을 업데이트했습니다.
