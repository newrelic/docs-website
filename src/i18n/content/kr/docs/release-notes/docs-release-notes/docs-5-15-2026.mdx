### 새로운 문서

* N+1 쿼리, 느린 SQL 및 기타 코딩 안티패턴을 감지하는 새로운 퍼블릭 프리뷰에 대한 [성능 위험 인박스](/docs/errors-inbox/performance-risks/introduction) 문서를 추가했습니다.
* 생성, 가져오기, 목록, 해결 및 업데이트 작업을 포함하는 워크플로우 자동화를 위한 [ServiceNow 인시던트 작업](/docs/workflow-automation/setup-and-configure/actions-catalog/servicenow/servicenow-incident) 을 추가했습니다.

### 주요 변경 사항

* 플랫 `rules` 어레이와 명시적 `context` 필드가 있는 간소화된 v2.2.0 YAML 스키마를 반영하도록 [파이프라인 컨트롤 게이트웨이 프로세서 문서](/docs/new-relic-control/pipeline-control/gateway/yaml-overview) 를 업데이트했습니다.
* 새로운 OpenTelemetry `traces.include` 및 `traces.exclude` 설정을 포함하여 v13.0.0 릴리스에 대한 [파이썬 에이전트 설정](/docs/apm/agents/python-agent/configuration/python-agent-configuration) 및 트레이스 API 참조를 업데이트했습니다.
* <DNT>**Fix manually**</DNT> 및 <DNT>**Fix with AI**</DNT> 워크플로우를 위한 새로워진 탭 레이아웃으로 [신세틱스 런타임 업그레이드 UI](/docs/synthetics/synthetic-monitoring/using-monitors/runtime-upgrade-ui) 를 업데이트했습니다.

### 사소한 변경 사항

* 에이전트의 트랜잭션 및 스팬 API와 함께 `IAgent` 을(를) 안전하게 사용하는 방법을 설명하는 의존성 주입에 대한 참고 사항을 [.NET 에이전트 API](/docs/apm/agents/net-agent/net-agent-api/net-agent-api) 에 추가했습니다.
* v3.2.0+ ARM 템플릿용 [Azure 이벤트 허브 로그 포워딩](/docs/logs/forward-logs/azure-log-forwarding) 에 `Managed Identity` 인증 옵션을 추가했습니다.
* [ECS EC2](/docs/opentelemetry/integrations/ecs-monitoring/ecs-ec2) 및 [ECS Fargate](/docs/opentelemetry/integrations/ecs-monitoring/ecs-fargate) OpenTelemetry 모니터링 가이드에 `aws.cluster.arn` 속성 설정을 추가했습니다.
* [루비 에이전트 지원 프레임워크](/docs/apm/agents/ruby-agent/getting-started/ruby-agent-requirements-supported-frameworks)에 로깅, Rails 이벤트 로거 및 시맨틱 로거 라이브러리 지원을 추가했습니다.
* [루비 에이전트 설정](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration)에 `active_record_use_table_name` 및 Dalli 5.0 항목을 추가했습니다.
* `runtimeTypeVersion: "22.20.0"` 을(를) 권장하고 이전 Node.js 런타임에 대한 2026년 11월 18일 강제 업그레이드를 표시하도록 [신세틱스 NerdGraph API 예제](/docs/apis/nerdgraph/examples/synthetics-api/scripted-api-monitor) 를 업데이트했습니다.
* 어떤 APM 이벤트 유형이 분산 트레이스 네임스페이스에 속하지 않는지 명확히 하기 위해 [데이터 관리](/docs/data-apis/manage-data/manage-data-retention) 에 APM 네임스페이스 축소기를 추가했습니다.
* [워크플로우 자동화 정의 스키마](/docs/workflow-automation/workflow-automation-apis/definition-schema)에 `jq` 표현식이 포함된 `for` 루프 예제를 추가했습니다.
* 새로운 `JSON`, `CSV`, `PSV` 및 `TSV` 쿼리 결과 형식과 `success` 선택기 예제로 [뉴렐릭 공지 작업](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) 을 업데이트했습니다.
* [보안 게시판 개요](/docs/security/security-privacy/information-security/security-bulletins) 를 신세틱 모니터링의 `CVE-2026-31431` 에 대한 새로운 [NR26-05](/docs/security/new-relic-security/security-bulletins/security-bulletin-nr26-05) 항목으로 업데이트했습니다.
* Local System, Active Directory, Microsoft Entra ID 및 gMSA 계정을 포함한 Windows 서비스 로그온 옵션으로 [인프라 에이전트 요구 사항](/docs/infrastructure/infrastructure-agent/requirements-infrastructure-agent) 을 업데이트했습니다.
* 최신 지원 패키지 버전으로 [Node.js 에이전트 호환성 보고서](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) 를 업데이트했습니다.

### 새로운 게시물은 무엇인가요?

* N+1 쿼리, 느린 SQL, 과도하고 순차적인 데이터베이스 쿼리, 느린 HTTP requests 및 대용량 HTTP 페이로드를 감지하기 위한 무료 공개 프리뷰 액세스를 발표하기 위해 [성능 위험 수신함이 이제 공개 프리뷰로 제공됩니다](/whats-new/2026/05/whats-new-05-14-performance-risks-inbox) 를 추가했습니다.
