현재 New Relic은 v1 및 v3의 두 가지 합성 모니터링 API 버전을 지원합니다. 버전 3은 2016년 10월에 릴리스되었습니다. **버전 1은 더 이상 사용되지 않으며** 결국 더 이상 사용할 수 없게 됩니다. 종료 날짜는 발표되지 않았습니다. 그러나 v1에 대한 추가 개발이나 수정은 없습니다.

<Callout variant="caution">
  권장 사항: [v3 합성 API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api) 를 사용하여 새 모니터를 만들고 v1 스크립트를 v3 스크립트로 마이그레이션하십시오.
</Callout>

New Relic의 종합 모니터링에 REST API를 사용하려면 [API 액세스를 활성화하고 계정 설정에서 Admin User의 API 키를 생성해야](/docs/apis/rest-api-v2/requirements/api-keys#creating) 합니다. 그런 다음 명령줄을 통해 표준 API 호출을 수행할 수 있습니다.

## 단순 및 스크립트 모니터 관리 [#simple-monitor-calls]

합성 REST API를 호출하려면 관리자의 API 키를 사용해야 합니다. 계정의 REST API 키가 작동하지 않습니다.

<Callout variant="caution">
  합성 REST API는 계정의 요청 속도를 초당 3개의 요청으로 제한합니다. 이 임계값을 초과하는 요청은 429 응답 코드를 반환합니다.
</Callout>

다음 예에서는 curl 명령을 보여줍니다.

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="get-all-monitors"
    title="모든 모니터 가져오기"
  >
    계정에 대한 New Relic의 모든 모니터 목록을 보려면 `https://synthetics.newrelic.com/synthetics/api/v1/monitors` 에 GET 요청을 보내십시오. 예를 들어:

    ```
    curl -v \
         -H 'X-Api-Key:{Admin_User_Key}' https://synthetics.newrelic.com/synthetics/api/v1/monitors
    ```

    성공적인 요청은 **200 OK** 응답을 반환합니다. [반환된 데이터](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api#api-attributes) 는 다음 형식의 JSON 개체입니다.

    ```
    {
      "count": integer,
      "monitors": [
        {
          "id": UUID,
          "name": string,
          "type": string,
          "frequency": integer,
          "uri": string,
          "locations": array of strings,
          "status": string,
          "slaThreshold": double,
          "userId": integer,
          "apiVersion": string
        }
      ]
    }
    ```
  </Collapser>

  <Collapser
    className="freq-link"
    id="get-specific-monitor"
    title="특정 모니터 가져오기"
  >
    New Relic에서 단일 기존 모니터를 보려면 `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}` 에 GET 요청을 보냅니다. 다음 예에서 &#x7B;id}를 특정 모니터 ID로 바꿉니다.

    ```
    curl -v \
         -H 'X-Api-Key:{Admin_User_Key}' https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}
    ```

    성공적인 요청은 **200 OK** 응답을 반환합니다. [반환된 데이터](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api#api-attributes) 는 다음 형식의 JSON 개체입니다.

    ```
    {
      "id": UUID,
      "name": string,
      "type": string,
      "frequency": integer,
      "uri": string,
      "locations": array of strings,
      "status": string,
      "slaThreshold": double,
      "userId": integer,
      "apiVersion": string
    }
    ```

    잘못된 모니터 ID는 오류 **404** 를 반환합니다. 찾을 수 없음: 지정된 모니터가 존재하지 않습니다.
  </Collapser>

  <Collapser
    className="freq-link"
    id="add-simple-ping-monitor"
    title="모니터 만들기"
  >
    New Relic의 계정에 새 모니터를 추가하려면 모니터를 설명하는 JSON 페이로드와 함께 POST 요청을 `https://synthetics.newrelic.com/synthetics/api/v1/monitors` 에 보냅니다.

    ```
    {
      "name": string [required],
      "type": string (SIMPLE, BROWSER, SCRIPT_API, SCRIPT_BROWSER) [required],
      "frequency": integer (minutes) [required, must be one of 1, 5, 10, 15, 30, 60, 360, 720, or 1440],
      "uri": string [required for SIMPLE and BROWSER type],
      "locations": array of strings (send a GET request to https://synthetics.newrelic.com/synthetics/api/v1/locations to get a list of valid locations) [at least one required],
      "status": string (ENABLED, MUTED, DISABLED) [required],  
      "slaThreshold": double, 
    }
    ```

    또한 REST API를 통해 **스크립팅된 모니터를 추가** 하려면 [추가 API 엔드포인트](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitor-scripts-rest-api) 를 호출하여 방금 생성한 모니터에 대한 스크립트를 보냅니다.

    다음 예의 [Synthetics REST API 속성](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api) 을 특정 값으로 바꿉니다.

    ```
    curl -v \
         -X POST -H 'X-Api-Key:{Admin_User_Key}' \
         -H 'Content-Type: application/json' https://synthetics.newrelic.com/synthetics/api/v1/monitors \
         -d '{  "name" : "monitor1",  "frequency" : 15, "uri" : "http://my-uri.com",   "locations" : [ "AWS_US_WEST_1" ],   "type" : "browser"}'
    ```

    요청이 성공하면 `location` 헤더에 새로 생성된 모니터의 URI가 지정된 **201 Created** 응답이 반환됩니다. 가능한 오류 코드는 다음과 같습니다.

    * **400 잘못된 요청** : 하나 이상의 모니터 값이 잘못되었거나 요청 형식이 잘못되었습니다. 예를 들어 빈도가 범위를 벗어났거나 지정된 위치 중 하나 이상이 유효하지 않습니다(응답 본문의 오류 메시지 참조).
    * **402 지불 필요** : 모니터를 만들면 예약된 수표가 계정의 구매 수표 한도를 초과하여 증가합니다.
  </Collapser>

  <Collapser
    className="freq-link"
    id="update-monitor"
    title="기존 모니터 업데이트"
  >
    New Relic에서 기존 모니터를 업데이트하려면 `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}` 에 PUT 요청을 보냅니다. 또한 스크립팅된 모니터의 경우 절차에 [따라 BASE64로 인코딩된 스크립트를 업데이트하십시오](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitor-scripts-rest-api) .

    다음 예에서 &#x7B;id}를 특정 모니터 ID로 바꾸고 [Synthetics REST API 속성](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api) 을 특정 값으로 바꾸십시오.

    ```
    curl -v \
         -X PUT -H 'X-Api-Key:{Admin_User_Key}' \
         -H 'Content-Type: application/json' https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id} \
         -d '{  "name" : "updated monitor name", "type" : "simple", "frequency" : 15, "uri" : "http://my-uri.com/",   "locations" : [ "AWS_US_WEST_1" ],   "status" : "enabled", "slaThreshold": "7.0" }'
    ```

    PUT 요청은 대상 엔터티를 대체하기 위한 것이므로 [새 모니터를 생성](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#add-simple-ping-monitor) 할 때 JSON 페이로드에 필요한 모든 속성은 기존 모니터를 업데이트할 때도 필요합니다.

    요청이 성공하면 본문이 비어 있는 **204 No Content** 응답이 반환됩니다. 가능한 오류 코드는 다음과 같습니다.

    * **400 잘못된 요청** : 하나 이상의 모니터 값이 잘못되었거나 요청 형식이 잘못되었습니다. 예를 들어 빈도가 범위를 벗어났거나 지정된 위치 중 하나 이상이 유효하지 않습니다(응답 본문의 오류 메시지 참조).
    * **404 Not Found** : 지정한 모니터가 존재하지 않습니다.
  </Collapser>

  <Collapser
    className="freq-link"
    id="delete-monitor"
    title="기존 모니터 삭제"
  >
    New Relic에서 기존 모니터를 삭제하려면 `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}` 에 DELETE 요청을 보냅니다. 다음 예에서 [&#x7B;id}](/docs/apis/synthetics-rest-api/monitor-examples/payload-attributes-synthetics-rest-api) 를 특정 모니터 ID로 바꿉니다.

    ```
    curl -v \
    ```

    요청이 성공하면 본문이 비어 있는 **204 No Content** 응답이 반환됩니다. 실패한 요청은 **404 Not Found** : 지정된 모니터가 존재하지 않는다는 응답을 반환합니다.
  </Collapser>

## 스크립팅된 모니터 관리 [#scripted-monitor-calls]

일반 API 외에도 **스크립팅된 브라우저** ( `SCRIPT_BROWSER` ) 및 **API 테스트** ( `SCRIPT_API` ) 모니터 유형에 대한 여러 API 메소드가 있습니다.

이 예는 curl 명령을 보여줍니다.

<CollapserGroup>
  <Collapser
    id="get-all-monitors"
    title="모니터 스크립트 가져오기"
  >
    계정에 대한 New Relic의 특정 `SCRIPT_BROWSER` 또는 `SCRIPT_API` 모니터와 연결된 스크립트를 보려면 `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}/script` 에 GET 요청을 보냅니다. &#x7B;id}를 특정 모니터 ID로 바꿉니다. 예를 들어:

    ```
    curl -v
         -H 'X-Api-Key: {Admin_User_Key}' 
         https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}/script
    ```

    성공적인 요청은 **200 OK** 응답을 반환합니다. 반환된 데이터는 다음 형식의 JSON 개체입니다.

    ```
    {
      "scriptText": BASE64 encoded string
    }
    ```

    가능한 오류 코드는 다음과 같습니다.

    * **403 금지됨:** 지정된 모니터가 SCRIPT_BROWSER 또는 SCRIPT_API 유형이 아닙니다.
    * **404 Not Found:** 지정된 모니터가 존재하지 않거나 모니터와 연관된 스크립트가 존재하지 않습니다.
  </Collapser>

  <Collapser
    id="procedure"
    title="스크립트 모니터 추가"
  >
    REST API를 사용하여 새 스크립팅된 모니터를 New Relic에 추가하려면:

    1. [표준 API 절차에 따라 새 모니터를 추가](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#add-simple-ping-monitor) 하고 `type` 을 `SCRIPT_BROWSER` 또는 `SCRIPT_API` 로 식별합니다.

    2. 스크립트의 BASE64 인코딩 버전으로 새 모니터를 `${MONITOR_UUID}/script` 엔드포인트로 업데이트합니다.

       자세한 내용은 [예제](#sample-script) 를 참조하십시오.
  </Collapser>

  <Collapser
    id="update-monitor-script"
    title="모니터 스크립트 업데이트"
  >
    계정에 대한 New Relic의 특정 `SCRIPT_BROWSER` 또는 `SCRIPT_API` 모니터와 연결된 스크립트를 업데이트하려면 `scriptText` (필수)가 포함된 JSON 페이로드와 함께 PUT 요청을 `https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}/script` 에 보냅니다. `scriptLocations` 데이터는 [**검증된 스크립트 실행**](/docs/synthetics/new-relic-synthetics/private-locations/verified-script-execution-private-locations) 이 켜진 비공개 위치에만 필요합니다.

    HMAC 문자열을 생성하는 데 사용되는 비밀번호는 개인 위치에 대해 설정된 비밀번호와 일치해야 합니다. HMAC 문자열을 생성할 때 SHA256 알고리즘을 사용합니다.

    ```
    {
      "scriptText": BASE64 encoded String,
      "scriptLocations": [
        {
          "name": Location name,
          "hmac" BASE64 encoded String of SHA256 HMAC for location
        }
      ]
    }
    ```

    &#x7B;id}를 특정 모니터 ID로 바꿉니다. 다음은 스크립트의 예입니다.

    ```
    var assert = require('assert');
    assert.equal('1', '1');
    ```

    이 예에서는 `password` 을 `scriptLocation` 의 비밀번호로 사용합니다.

    ```
    curl -v
    	-H 'X-Api-Key: '{Admin_User_Key}'
    	-H 'content-type: application/json'
    	https://synthetics.newrelic.com/synthetics/api/v1/monitors/{id}/script
    	-d '{ "scriptText": "dmFyIGFzc2VydCA9IHJlcXVpcmUoJ2Fzc2VydCcpOw0KYXNzZXJ0LmVxdWFsKCcxJywgJzEnKTs=","scriptLocations": [ { "name": "my_vse_enabled_location", "hmac": "MjhiNGE4MjVlMDE1N2M4NDQ4MjNjNDFkZDEyYTRjMmUzZDE3NGJlNjU0MWFmOTJlMzNiODExOGU2ZjhkZTY4"} ]}'
    ```

    요청이 성공하면 본문이 비어 있는 **204 No Content** 응답이 반환됩니다. 가능한 오류 코드는 다음과 같습니다.

    * **400 잘못된 요청:** `scriptText` 또는 `hmac` 에 대한 잘못된 BASE64 인코딩 문자열입니다.
    * **403 금지됨:** 지정된 모니터가 SCRIPT_BROWSER 또는 SCRIPT_API 유형이 아닙니다.
    * **404 Not Found:** 지정한 모니터가 존재하지 않습니다.
  </Collapser>
</CollapserGroup>

## 스크립팅된 브라우저 예제

다음은 New Relic의 REST API와 bash 스크립트를 사용하여 스크립팅된 브라우저 모니터를 만드는 예입니다.

<CollapserGroup>
  <Collapser
    id="sample-script"
    title="스크립팅된 브라우저 API 예제"
  >
    다음 예는 스크립팅된 브라우저 모니터를 생성하기 위한 curl 명령을 보여줍니다.

    * 스크립트 상단에서 변수를 특정 값으로 바꿉니다.

    * `scriptfile` 변수의 경우 생성할 스크립트의 파일 이름을 식별합니다. 다음은 예에서 사용하기 위해 `sample_synth_script.js` 로 저장할 수 있는 샘플 스크립트입니다.

      ```
      var assert = require("assert");
      $browser.get("http://example.com").then(function(){ 
        // Check the H1 title matches "Example Domain" 
        return $browser.findElement($driver.By.css("h1")).then(function(element){ 
          return element.getText().then(function(text){ 
            assert.equal("Example Domain", text, "Page H1 title did not match"); 
          }); 
        }); 
      }).then(function(){ 
        // Check that the external link matches "http://www.iana.org/domains/example" 
        return $browser.findElement($driver.By.css("div > p > a")).then(function(element){ 
          return element.getAttribute("href").then(function(link){ 
            assert.equal("http://www.iana.org/domains/example", link, "More information link did not match"); 
          }); 
        }); 
      });
      ```
  </Collapser>

  <Collapser
    id="bash-script"
    title="Bash 스크립트 예제"
  >
    이 예는 `SCRIPTED_BROWSER` 모니터를 생성할 bash 스크립트를 보여줍니다.

    ```
    #!/bin/bash

    # Admin API key from your account settings
    adminAPIKey=''
    # Other attributes found at https://docs.newrelic.com/docs/apis/synthetics-rest-api/monitor-examples/attributes-synthetics-rest-api#api-attributes
    monitorName='Test API Script'
    monitorType='SCRIPT_BROWSER'
    frequency=1440
    locations='"AWS_US_WEST_1", "AWS_US_EAST_1"'
    slaThreshold=7.0
    # Location of the file with your script
    scriptfile=sample_synth_script.js

    # Test that the script file exists (does not validate content)
    if [ -e "$scriptfile" ]
    then
      script=$(cat "$scriptfile")

      payload="{  \"name\" : \"$monitorName\", \"frequency\" : $frequency,    \"locations\" : [ $locations ],   \"status\" : \"ENABLED\",  \"type\" : \"$monitorType\", \"slaThreshold\" : $slaThreshold, \"uri\":\"\"}"
      echo "Creating monitor"

      # Make cURL call to API and parse response headers to get monitor UUID
      shopt -s extglob # Required to trim whitespace; see below
      while IFS=':' read key value; do
        # trim whitespace in "value"
        value=${value##+([[:space:]])}; value=${value%%+([[:space:]])}
        case "$key" in
            location) LOCATION="$value"
                    ;;
            HTTP*) read PROTO STATUS MSG <<< "$key{$value:+:$value}"
                    ;;
        esac
      done < <(curl -sS -i  -X POST -H "X-Api-Key:$adminAPIKey" -H 'Content-Type: application/json' https://synthetics.newrelic.com/synthetics/api/v1/monitors -d "$payload")

      # Validate monitor creation & add script unless it failed
      if [ $STATUS = 201 ]; then
        echo "Monitor created, $LOCATION "
        echo "Uploading script"
          # base64 encode script
          encoded=`echo "$script" | base64`
          scriptPayload='{"scriptText":"'$encoded'"}'
            curl -s -X PUT -H "X-Api-Key:$adminAPIKey" -H 'Content-Type: application/json' "$LOCATION/script" -d $scriptPayload
            echo "Script uploaded"
      else
        echo "Monitor creation failed"
      fi

    else
      echo "script file not found, not creating monitor"
    fi
    ```

## 특정 모니터 끝점 [#specific-monitor]

특정 모니터에 대해 REST API를 호출할 때 `monitor_uuid` 을 엔드포인트의 일부로 포함합니다. `monitor_uuid` 은 URL의 일부인 GUID입니다. 예를 들어 선택한 [합성 모니터](/docs/synthetics/new-relic-synthetics/pages/synthetics-overview-page-view-monitors-performance) 에는 다음 URL이 있습니다.

```
https://synthetics.newrelic.com/accounts/nnnn/monitors/ab123-c456d-e78-90123-f45g
```

`monitor_uuid` 은 `/monitors/` 다음에 오는 값입니다.
