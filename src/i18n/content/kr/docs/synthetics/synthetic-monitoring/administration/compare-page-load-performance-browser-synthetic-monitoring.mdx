[종합 모니터링](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) 은 웹사이트, 중요한 비즈니스 트랜잭션 및 API 엔드포인트를 정기적으로 확인하여 최적의 성능을 측정합니다. [브라우저 모니터링](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) 은 장치, 브라우저 및 연결 속도의 모든 변형에서 웹 페이지 성능의 실제 결과를 추적합니다. 함께 사용하면 실제 사용자(브라우저) 상호 작용과 합성 모니터 간의 직접적인 페이지 로드 시간 비교를 제공합니다.

## 실적 및 추세 비교 [#features]

New Relic의 비교 차트 기능은 다음을 제공하여 운영 관리자와 팀을 돕습니다.

* 페이지 로드 시간에 대한 벤치마크
* 사이트를 최적화할 위치를 계획하는 데 도움이 되는 추가 정보
* 모니터링 기능 간에 전환할 필요 없이 합성 추세와 실제 브라우저 성능을 비교합니다.

예를 들어 페이지 중단 동안 합성 모니터링 추세를 실제 브라우저 모니터링 비교와 비교하여 문제가 Synthetics UI에도 표시되는지 또는 New Relic 외부의 변수로 인해 발생하는지 확인할 수 있습니다. 이렇게 하면 조치를 취해야 할 위치를 보다 효율적으로 알 수 있습니다.

## 필요한 것 [#requirements]

비교 차트 기능에는 다음이 필요합니다.

* [브라우저 SPA 에이전트](/docs/browser/single-page-app-monitoring/get-started/spa-compatibility-requirements) ( [버전 885](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) 이상)로 브라우저 모니터링이 활성화되었습니다.
* 브라우저 에이전트가 모니터링하는 동일한 URL에 대한 하나 이상의 테스트가 있는 [합성 브라우저 또는 스크립트된 모니터](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors) 입니다.

합성 또는 브라우저 모니터링에서 비교 차트 기능을 활성화한 후에는 추가 설정이 필요하지 않습니다. New Relic이 일치하는 URL을 찾으면 비교 차트 기능이 나타납니다.

## 비교 데이터 활성화 [#enable]

[합성](#enable-synthetics) 또는 [브라우저](#enable-browser) 모니터링에서 비교 차트 기능을 활성화한 후에는 추가 설정이 필요하지 않습니다. 비교 차트 기능은 합성 또는 브라우저 모니터링이 일치하는 URL을 찾을 때마다 나타납니다.

## 비교 데이터 보기 [#view-ui]

합성 모니터링이 브라우저 모니터링과 URL 일치를 식별하고 비교할 수 있을 때마다 비교 차트 기능이 나타납니다. New Relic 기능 간에 전환할 필요 없이 [종합 모니터링 **요약** 페이지](/docs/synthetics/new-relic-synthetics/pages/synthetics-overview-page-view-monitors-performance) 또는 [브라우저 모니터링 **페이지 보기** 페이지](/docs/browser/new-relic-browser/additional-standard-features/page-views-understanding-your-sites-popularity) (브라우저 앱 또는 단일 페이지 앱용)에서 브라우저와 종합의 성능을 비교할 수 있습니다.

선택한 [데이터 소스](#data-sources) 를 기반으로 요약이 표시됩니다.

* 브라우저(실제 사용자) 페이지 보기와 사용자가 보기 권한이 있는 합성 모니터에서 찾은 일치하는 URL 간의 전체 속도 백분율 비교
* 브라우저 롤업 URL과 일치하는 합성 모니터에서 발견된 URL 수

비교 차트 요약을 보려면:

1. 합성 모니터링 UI에서: **[one.newrelic.com](https://one.newrelic.com) > Synthetics > (모니터 선택) > Summary** 이동합니다.

   또는

   브라우저 모니터링 UI에서: **[one.newrelic.com](https://one.newrelic.com) > Browser > (브라우저 앱 선택) > Page views** 로 이동한 다음 종합 모니터링으로 모니터링되는 페이지를 선택합니다.

2. 전체 속도 백분율 및 일치하는 URL 수에 대한 비교 차트 기능의 요약을 검토합니다.

3. 추가 세부 정보를 보려면 요약의 오른쪽 화살표를 선택하십시오. <Icon name="fe-chevron-right"/>상.

어떤 경우에는 브라우저 에이전트가 `BrowserInteraction` 이벤트를 수집하기 전에 단순 브라우저 모니터가 종료됩니다. 이 경우 UI에 비교 차트 데이터가 표시되지 않습니다. 이 문제를 해결하려면 대신 스크립팅된 브라우저 모니터를 만들고 페이지가 로드된 후 대기(절전) 호출을 추가하십시오. 다음은 예입니다.

```
$browser.get('https://www.mywebsite.com').then(function(){
return $browser.sleep(1000);
})
```

## 비교 데이터 숨기기 또는 반환 [#hide]

비교 차트 기능을 숨기려면 줄임표를 선택하십시오. <Icon name="fe-more-horizontal"/>상.

* 표시되도록 유지하되 페이지 상단에서 멀리 **이동하려면 하단으로 이동 을** 선택합니다.
* 페이지의 원래 위치로 되돌리려면 **맨 위로 이동 을** 선택합니다.
* 페이지에서 제거하려면 **모든 합성/브라우저 비교 숨기기** 를 선택합니다.
* 제거한 후 페이지로 되돌리려면 표준 절차에 따라 [비교 차트를 활성화](#enable) 하십시오.
