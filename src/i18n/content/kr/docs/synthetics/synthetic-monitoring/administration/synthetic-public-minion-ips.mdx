New Relic은 공개 미니언을 사용하여 [합성 모니터](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors) 를 실행합니다. 이 미니언은 전 세계의 다양한 데이터 센터에 배치되어 모니터 실행을 담당합니다.

이 때문에 방화벽이 동기화 네트워크 요청을 허용하는지 확인하십시오. 미니언 IP 주소는 JSON 형식으로 공용으로 사용할 수 있으므로 쉽게 읽고 구문 분석할 수 있다.

<Callout variant="important">
  두 환경에서 요청을 보낼 수 있으므로 허용 목록 또는 관련 구성에 [IP 범위 목록](#dynamic-ip-addresses) 의 모든 범위가 포함되어 있는지 확인하십시오.
</Callout>

<DNT>**Optional steps**</DNT>:

* 이러한 파일을 기반으로 허용 목록에 대한 규칙을 자동화할 수 있습니다.
* [표준 헤더를](/docs/synthetics/new-relic-synthetics/administration/identify-synthetics-requests-your-app) 사용하거나 [핑이나 단순 브라우저 모니터](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors/#simple), [스크립트 API 모니터](/docs/synthetics/synthetic-monitoring/scripting-monitors/write-synthetic-api-tests/#request-options) 또는 [펼쳐보기 브라우저](/docs/synthetics/synthetic-monitoring/scripting-monitors/synthetic-scripted-browser-reference-monitor-versions-chrome-100#structure) 에 사용자 정의 헤더를 추가하여 신세틱스 모니터 요청을 식별할 수 있습니다.

## IP 주소는 개인 데이터가 아닙니다. [#not-personal-data]

미니언즈는 서버에 배치되며 에이전트는 비개인 신임 정보를 사용하여 활성화됩니다. 서버에서 실행되는 미니언즈와 관련된 IP 주소는 데이터 보호 및 개인정보 보호법에 따라 개인 데이터가 아닙니다. 자세한 정보는 [합성 보안 문서](/docs/synthetics/new-relic-synthetics/getting-started/security-new-relic-synthetics)를 참조하십시오.

## IP 주소에 대한 매일 JSON 목록 [#dynamic-ip-addresses]

해제된 위치의 IP 주소는 변경될 수 있습니다. 변경이 필요한 경우 이메일을 통해 변경 전에 사전에 고객에게 알리도록 노력할 것입니다. [지원 포럼에서](https://discuss.newrelic.com/) 업데이트를 확인할 수도 있습니다. 나열된 IP 범위는 New Relic에서 사용하도록 예약되어 있으며 다른 사람이 사용할 수 없습니다.
