표준 합성 모니터링의 WebDriverJS 기반 스크립팅 환경 외에도 New Relic은 [스크립팅된 브라우저](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) 및 API 테스트를 향상시키기 위해 널리 사용되는 많은 Node.js 모듈을 지원합니다.

## 선택적 모듈 가져오기 [#import-procedure]

지원되는 모듈을 가져오려면 해당 모듈에 대한 표준 가져오기 절차를 따르십시오. 예를 들어:

* **crypto-js** 를 가져오려면 다음을 호출하십시오.

  ```
  var crypto = require('crypto-js');
  ```

* **chai** 를 가져오려면 다음을 호출하십시오.

  ```
  var assert = require('chai').assert;
  ```

## 지원되는 타사 모듈

서로 다른 합성 스크립팅된 모니터 버전은 서로 다른 타사 모듈을 지원합니다. 다른 모니터 버전에서 지원하는 모듈에 대한 정보는 [모니터 버전 비교 표](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment) 를 참조하십시오.

## 지원되지 않는 모듈

[컨테이너화된 개인 미니언](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms) 에서 모든 노드 모듈을 사용할 수 있습니다. 컨테이너화된 개인 미니언에 대한 사용자 지정 모듈을 구성하는 방법에 대한 지침은 [CPM 구성](/docs/synthetics/new-relic-synthetics/private-locations/containerized-private-minion-cpm-configuration#custom-modules) 을 참조하십시오.

지원되지 않는 모듈은 작동이 보장되지 않으며 지원 대상이 아닙니다.
