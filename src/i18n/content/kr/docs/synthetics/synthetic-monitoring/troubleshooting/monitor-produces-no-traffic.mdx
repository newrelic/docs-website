## 문제

New Relic 합성 모니터링 API 테스트 또는 스크립트된 브라우저 모니터가 원하는 대로 실행되는 것처럼 보이지만 항상 다음 오류를 반환합니다.

`NetworkError: Monitor produced no traffic`

## 해결책

합성 모니터링을 사용하면 Node.js 네트워크 모듈을 가져올 수 있지만 [스크립트로 작성된 브라우저 모니터](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) ( `$browser` ) 및 [API 테스트](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-api-tests) ( `$http` )에 대한 네트워크 트래픽을 캡처하려면 기본 제공 개체를 사용해야 합니다. 이를 통해 New Relic은 네트워크 요청을 추적하고 리소스 로드 시간, 상태 코드 및 기타 세부 정보를 보고할 수 있습니다.

`request` 과 같은 라이브러리가 수동으로 필요하고 스크립트 단계 내에서만 해당 모듈을 사용하는 경우 합성이 네트워크 트래픽을 캡처할 수 없습니다. 이것이 네트워크 오류가 표시되는 이유입니다.

이 문제를 해결하려면 라이브러리를 수동으로 가져오지 마십시오. 대신 동일한 기능을 제공하는 내장 객체를 사용하십시오.

내장 개체에서 사용할 수 없는 몇 가지 기능이 있을 수 있습니다. 이러한 경우 기본 제공 개체를 사용하여 단일 또는 자리 표시자 요청을 만들고 가져온 모듈로 나머지 모니터 단계를 완료할 수 있습니다. 이것은 네트워크 트래픽 결과가 모니터가 수행하는 전체 작업을 반영하지 않기 때문에 권장되는 솔루션이 아닙니다. 그러나 이 오류가 해결되고 모니터가 성공적으로 실행될 수 있습니다.
