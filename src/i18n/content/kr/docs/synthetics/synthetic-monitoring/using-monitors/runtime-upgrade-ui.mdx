뉴렐릭은 2026년에 [이전 런타임에 대한 지원을 종료](/eol/2025/11/eol-chrome-140-node-22-synthetics/) 합니다. 신세틱 모니터를 지원되는 최신 런타임으로 마이그레이션하는 과정을 간소화하기 위해, 사용자의 조치가 필요하지 않은 모든 모니터를 자동 업그레이드했습니다. 여전히 주의가 필요한 남은 모니터를 확인하려면 런타임 업그레이드 도구를 사용하십시오.

## 전제 조건 [#prerequisites]

모니터 업그레이드를 시작하기 전에 다음 요구 사항을 충족하는지 확인하십시오:

* **일괄 런타임 업그레이드 권한:** UI를 통해 일괄 업데이트를 적용하는 데 필요합니다. [제품 관리자](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) 는 기본적으로 이 권한을 가집니다. [커스텀 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics)에 추가할 수도 있습니다.
* **AI 권한(<DNT>Fix with AI</DNT> 전용):** `nrai_ccu` 또는 `nrai_discount_usage` 권한이 필요합니다. FedRAMP 계정은 이 환경에서 차단됩니다. 권한을 활성화하려면 뉴렐릭 계정 팀에 문의하세요.

## 남은 모니터를 업그레이드하십시오 [#upgrade-options]

주의가 필요한 각 모니터에 대해 두 가지 경로 중 하나를 선택할 수 있습니다:

<CollapserGroup>
  <Collapser id="fix-manually" title="수동 해결">
    AI 생성에 의존하는 대신 <DNT>**Fix manually**</DNT> 옵션을 사용하여 모니터 스크립트를 작성하고 편집하세요. 모니터를 수동으로 업데이트하려면:

    1. <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT> 으)로 이동합니다.

    2. 모니터를 찾아 <DNT>**Fix manually**</DNT>을(를) 클릭합니다.

    3. <DNT>**Fix script manually**</DNT> 창에서 새 런타임 환경에 맞게 스크립트 코드를 업데이트합니다.

    4. <DNT>**Close and start validation**</DNT>을 클릭합니다. 시스템이 업데이트된 스크립트를 검증하는 동안 모니터가 <DNT>**Monitors we&apos;re validating**</DNT> 테이블로 이동합니다.

       <Callout title="메모">
         검증에 실패하면 모니터 옆의 **⋯** 을 클릭하고, 메뉴에서 <DNT>**Fix manually**</DNT> 을(를) 선택하고 스크립트를 업데이트한 다음 <DNT>**Close and start validation**</DNT> 을(를) 다시 클릭합니다.
       </Callout>

    5. 유효성 검사가 성공하고 상태가 <DNT>**Fixed**</DNT>(으)로 표시되면 <DNT>**Ready to update**</DNT> 탭을 클릭합니다.

    6. 검증된 모니터 옆의 확인란을 선택하고 <DNT>**Upgrade selected**</DNT>을(를) 클릭합니다.

    완료되면 확인 공지가 나타나고 모니터가 <DNT>**Updated**</DNT> 탭으로 이동하여 성공적인 업그레이드를 확인할 수 있습니다.
  </Collapser>

  <Collapser id="fix-with-ai" title="AI로 수정">
    <DNT>New Relic AI (NRAI)</DNT> 이(가) 업데이트된 스크립트를 생성하도록 <DNT>**Fix with AI**</DNT> 옵션을 사용합니다. AI 지원 스크립트 생성은 주로 런타임 업그레이드 중에 발생하는 `timeout` 오류를 수정하는 데 맞춰져 있습니다. 다른 오류 유형의 경우, 스크립트 자체를 [수동으로](#fix-manually) 변경해야 할 가능성이 높습니다.

    AI를 사용하여 모니터를 업데이트하려면:

    1. <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT> 으)로 이동합니다.
    2. 모니터를 찾아 <DNT>**Fix with AI**</DNT>을(를) 클릭합니다.
    3. AI가 제안한 스크립트를 검토하세요. 원본 코드에 적용된 특정 변경 사항을 보려면 <DNT>**Show comparison**</DNT> 을(를) 클릭하여 나란히 표시되는 다른 뷰를 엽니다.
    4. <DNT>**Continue to validation**</DNT>을 클릭합니다. 시스템은 새 스크립트를 테스트하고 오류 없이 실행되는지 확인하기 위해 모니터를 일시적으로 검증 상태로 둡니다.
    5. 스크립트가 검증된 후 <DNT>**Ready to update**</DNT> 탭을 클릭합니다.
    6. 검증된 모니터 옆의 확인란을 선택하고 <DNT>**Upgrade selected**</DNT>을(를) 클릭합니다.

    완료되면 확인 공지가 나타나고 모니터가 <DNT>**Updated**</DNT> 탭으로 이동하여 성공적인 업그레이드를 확인할 수 있습니다.
  </Collapser>
</CollapserGroup>

한 번에 여러 업그레이드를 실행할 수 있으므로, 시스템이 각 스크립트의 유효성 검사를 완료할 때까지 기다리지 않고 여러 모니터에서 병렬로 작업할 수 있습니다.
