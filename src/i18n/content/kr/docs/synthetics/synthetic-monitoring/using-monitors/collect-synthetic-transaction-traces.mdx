합성 트랜잭션 추적은 [합성 결과](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results) 를 [APM 트랜잭션](/docs/apm/traces/transaction-traces/transaction-traces) 추적에 연결합니다. 종합 결과는 각 종합 검사에서 브라우저 측 세부 정보를 캡처하지만 트랜잭션 추적은 앱 서버의 활동을 캡처합니다. 합성 모니터를 APM에 연결하면 모든 합성 실행의 양면을 볼 수 있습니다.

<Callout variant="important">
  합성 모니터링 및 <InlinePopover type="apm"/>이러한 연결을 만들려면 트랜잭션 추적에 대해 동일한 계정에 있어야 합니다.
</Callout>

## 합성 트랜잭션 추적 사용 [#enabling]

가상 트랜잭션 추적은 다음에서 기본적으로 활성화됩니다. <InlinePopover type="apm"/>에이전트 버전:

* **자바** : 3.14.0 이상
* **.NET** : 3.10.43.0 이상
* **Node.js** : 1.16.1 이상
* **PHP** : 4.18.0.89 이상
* **Python** : 2.40.0.34 이상
* **루비** : 3.9.7 이상

## 합성 트랜잭션 추적 보기 [#viewing]

가상 트랜잭션 추적을 보려면 가상 트랜잭션 추적을 활성화한 웹 사이트를 확인하는 모니터를 만드십시오. 그런 다음 Synthetics UI의 **요약** 페이지에서 위치 차트의 점을 클릭하여 결과 세부 정보를 엽니다. 트랜잭션 추적에 대한 링크를 얻으려면 타임라인 차트를 가리킵니다.

New Relic은 애플리케이션당 1분에 최대 20개의 가상 트랜잭션 추적을 수집합니다. 앱이 분당 20개 이상의 합성 검사를 수신하는 경우 모든 합성 결과에 합성 추적이 포함되지는 않습니다.
