신세틱 모니터링은 모든 <DNT>**ping**</DNT> 모니터 검사를 자동으로 기록하여 모든 실행에 대한 로드 시간과 응답 크기를 확인할 수 있습니다.

[탐색기](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer-view-performance-across-apps-services-hosts) 와 선택한 핑 모니터의 <DNT>**Summary**</DNT> 및 <DNT>**Results**</DNT> 페이지를 사용하여 다음을 수행하십시오.

* 로드 타이밍, 응답 및 요청 헤더, 기타 세부 정보를 보려면 리소스를 선택하십시오.
* 이러한 세부 정보를 사용하여 문제를 찾고 성능 문제를 진단합니다.

<Callout variant="tip">
  <DNT>**simple**</DNT> 또는 <DNT>**scripted**</DNT> 모니터에 대한 자세한 내용은 [단순 또는 스크립팅된 모니터 결과 보기를](/docs/synthetics/new-relic-synthetics/using-monitors/view-ping-monitor-results) 참조하세요.
</Callout>

## 핑 모니터 결과 보기 [#finding]

<DNT>**ping**</DNT> 모니터 결과의 전체 목록을 보려면 다음을 참조하십시오.

1. <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> 으)로 이동합니다.
2. 쿼리 `monitorType=ping` 을 사용하여 <DNT>**Monitor Type**</DNT> 으로 정렬합니다.
3. 페이지 로드 시간 및 가용성과 같은 모니터에 대한 특정 정보를 보려면 핑 모니터를 선택하여 선택한 모니터의 <DNT>**Summary**</DNT> 및 <DNT>**Results**</DNT> 페이지에 액세스하십시오.

<img title="Entitiy monitor list" alt="entitiy_ping_monitor_page.png" src="/images/synthetic_screenshot-full_view-ping-results.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> (모니터 선택): 선택한 핑 모니터의 로드 시간 및 총 로드 크기를 포함한 요약 정보를 봅니다.
</figcaption>

<CollapserGroup>
  <Collapser id="page-resources" title="페이지 리소스에 대한 세부정보 가져오기">
    특정 핑 모니터 체크를 클릭하여 <DNT>**Result Details**</DNT> 보기로 이동하세요.

    결과 세부 정보 보기에서 다음을 수행할 수 있습니다.

    * 각 페이지의 리소스가 로드된 정확한 순서를 확인하세요.
    * 각 요소를 로드하는 데 걸린 시간을 확인하세요.
    * HTTP 상태 코드, [타이밍 정보](#viewing-logs), 응답 헤더 및 요청 헤더를 포함한 자세한 범위를 참조하십시오.
  </Collapser>

  <Collapser id="transaction-traces" title="트랜잭션 추적 보기">
    1. 보려는 핑 모니터에 대해[합성 트랜잭션 추적을 활성화](/docs/synthetics/new-relic-synthetics/using-monitors/collect-synthetic-transaction-traces) 했는지 확인하십시오.
    2. <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> 으로 이동 &gt; (핑 모니터를 선택하세요).
    3. <DNT>**Timeline**</DNT> 위에 마우스 커서를 올리고 <DNT>**APM transaction trace**</DNT> 클릭하세요.
  </Collapser>

  <Collapser id="sharing" title="결과 공유">
    브라우저의 주소 표시줄에서 고유한 URL을 복사합니다. 예를 들어:

    ```
    https://one.newrelic.com/launcher/nr1-core.explorer#launcher=7890wxyz-7c6c-4786-94bc-31d58fc91a73
    ```

    그런 다음 이 URL을 New Relic 계정 데이터에 액세스할 수 있는 다른 사람과 공유할 수 있습니다.
  </Collapser>

  <Collapser id="quickly-switch" title="다른 모니터에 빠르게 액세스">
    1. 화면 상단에서 현재 모니터의 이름을 클릭하여 드롭다운 메뉴를 엽니다.
    2. 최근 모니터 목록에서 선택하거나 검색 필드에 이름을 입력하여 특정 모니터를 검색합니다.
  </Collapser>
</CollapserGroup>

## 타이밍 세부정보 [#viewing-logs]

일부 모니터 유형의 경우 전체 모니터 확인 기간이 개별 페이지 요청 기간보다 깁니다. 이는 일부 브라우저 동작이 개별적으로 측정되지 않지만 여전히 총 검사 시간에 포함되기 때문입니다.

측정되지 않은 행동의 예는 다음과 같습니다.

* 자바스크립트 상호작용
* 리소스 미리 가져오기 및 우선 순위 지정
* DNS 사전 확인
* TCP 사전 연결
* 페이지 사전 렌더링
