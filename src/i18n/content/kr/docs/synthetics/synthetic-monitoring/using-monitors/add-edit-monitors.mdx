합성 모니터는 웹사이트를 모니터링하고 각 확인을 자세히 기록하는 API 확인 또는 가상 브라우저 인스턴스입니다. 모니터는 다음을 수행할 수 있습니다.

* 개요 또는 요약을 포함하여 집계 수치를 캡처합니다.
* 다운타임 공지 이벤트 및 페이지 리소스에 대한 자세한 통계를 제공합니다.
* 모니터 성능을 자세히 설명하는 사용자 지정 응답 코드를 수집합니다.

설치하지 않고 합성 모니터를 만들 수 있습니다. 시작하려면:

1. [New Relic 계정을](https://newrelic.com/signup) 만드세요.
2. <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT> 으로 이동하여 **Create a monitor** \[모니터 생성을] 클릭합니다.
3. 만들고 싶은 모니터를 선택하세요. 처음이라면 핑이나 스텝 모니터를 만들어 시작하는 것이 좋습니다.

## 모니터 만들기 [#adding-monitors]

[합성 모니터](/docs/synthetics/synthetic-monitoring/using-monitors/intro-synthetic-monitoring/#types-of-synthetic-monitors)중 하나(또는 여러 개!)로 웹 앱을 모니터링할 준비가 되셨습니까? 아래 단계는 모든 모니터에 대한 프로세스를 안내합니다.

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT> 으로 이동하여 **Create a monitor** \[모니터 생성을] 클릭합니다.
</figcaption>

<CollapserGroup>
  <Collapser id="simple" title="핑 또는 간단한 브라우저 모니터 추가">
    1. 이동 <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT>
    2. 핑 모니터의 경우 <DNT>**Availability**</DNT> 타일을 클릭하고 간단한 브라우저 모니터의 경우 <DNT>**Page load performance**</DNT> 클릭합니다.
    3. 선택 사항: [경고 알림](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics) 을 설정합니다.

    잠시 기다린 후 [<DNT>**Monitors**</DNT> 인덱스](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-summary) 에서 모니터를 확인하세요.

    ping 또는 간단한 브라우저 모니터에 대한 몇 가지 추가 구성 옵션이 있습니다. 두 모니터 유형 모두에 대해 다음을 설정할 수 있습니다.

    * [응답 유효성 검사를](#response-validation) 위한 하위 문자열 모니터링을 활성화하는 <DNT>**validation string**</DNT> 입니다.

    * <DNT>**Custom Headers**</DNT> 귀하의 모니터가 제출한 요청과 함께 제공됩니다.

    * <DNT>**Verify SSL**</DNT> SSL 인증서 체인의 유효성을 검사합니다. 다음 구문을 실행하여 복제할 수 있습니다.

      ```sh
      openssl s_client -servername YOUR_HOSTNAME -connect YOUR_HOSTNAME:443 -CApath /etc/ssl/certs -verify_hostname YOUR_HOSTNAME > /dev/null
      ```

      0이 아닌 종료 코드가 반환되면 모니터가 실패합니다.

    ping 모니터만 있으면 다음을 설정하도록 선택할 수 있습니다.

    * 기본 `HEAD` 요청을 건너뛰고 대신 `GET` 동사를 사용하는 <DNT>**Bypass HEAD request**</DNT> 옵션입니다.
    * <DNT>**Redirect is Failure**</DNT>, 새 URL로의 리디렉션을 따르지 않고 리디렉션을 실패로 분류합니다.
    * <DNT>**Custom Headers**</DNT> ping 및 간단한 브라우저 모니터에 추가할 수 있습니다. 이 헤더는 모니터가 제출한 요청에 추가됩니다.

    간단한 브라우저 모니터만 사용하는 경우 Chrome이나 Firefox 중에서 선택하고 장치 유형, 화면 방향 등의 [장치 에뮬레이션](/docs/synthetics/synthetic-monitoring/using-monitors/device-emulation) 설정을 활성화할 수 있습니다.
  </Collapser>

  <Collapser id="complex" title="스크립팅된 브라우저 또는 API 테스트 모니터 추가">
    1. <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT> 으)로 이동합니다.

    2. [모니터 유형](#setting-type) 및 [이름](#setting-name) 을 지정하십시오.

    3. [기간을](#setting-frequency) 선택하여 각 위치에서 모니터를 실행할 빈도를 결정하십시오.

    4. 선택 사항: 하나 이상의 브라우저 유형을 선택하고 [장치 에뮬레이션](/docs/synthetics/synthetic-monitoring/using-monitors/device-emulation) 설정을 활성화합니다. 브라우저 유형과 장치 유형 및 화면 방향 포함은 멀티미디어 디스플레이에서 사용할 수 있습니다.

    5. 모니터를 실행할 [위치](#setting-location) 를 선택하십시오.

    6. 선택 사항: [경고 알림](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics) 을 설정합니다.

    7. <DNT>**Next: Write your script**</DNT> 선택하여 [<DNT>**scripted browser**</DNT>](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) 또는 [<DNT>**API test**</DNT>](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests) 에 대한 스크립트를 만든 다음 <DNT>**Validate**</DNT> 선택하여 구문을 확인하세요.

       <Callout variant="tip">
         복잡한 스크립트의 경우 유효성 검사에 최대 1분이 소요될 수 있습니다.
       </Callout>

    8. 확인하려면 <DNT>**Create my monitor**</DNT> 선택하세요.

       몇 분 정도 기다린 후 [<DNT>**Monitors**</DNT> 색인](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-monitors-dashboard-access-your-monitors) 에서 모니터를 확인하세요.
  </Collapser>

  <Collapser id="stepmonitor" title="단계 모니터 추가">
    1. <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT> 으)로 이동합니다.

    2. [모니터 유형](#setting-type) 으로 단계 모니터를 선택합니다.

    3. [이름을](#setting-name) 지정하고 [기간을](#setting-frequency) 선택하여 각 위치에서 모니터를 실행할 빈도를 결정하십시오.

    4. 선택 사항: 하나 이상의 브라우저 유형을 선택하고 [장치 에뮬레이션](/docs/synthetics/synthetic-monitoring/using-monitors/device-emulation) 설정을 활성화합니다. 브라우저 유형과 장치 유형 및 화면 방향 포함은 멀티미디어 디스플레이에서 사용할 수 있습니다.

    5. 모니터를 실행할 [위치](#setting-location) 를 선택하십시오.

    6. UI 하단의 사전 구성된 단계에서 선택하여 모니터를 구축합니다.

       * URL로 이동
       * 텍스트 입력
       * 요소 클릭
       * 어설션 텍스트
       * 요소 주장
       * 자격 증명 확보

       UI 오른쪽에 있는 지침을 사용하여 CSS 클래스, HTML ID, 링크 텍스트 또는 XPath별로 요소를 찾을 수 있습니다.

    7. 확인하려면 <DNT>**Save monitor**</DNT> 선택하세요.

       트래픽을 생성하고 몇 분 정도 기다린 후 [<DNT>**Monitors**</DNT> 색인](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-monitors-dashboard-access-your-monitors) 에서 모니터를 확인하세요.
  </Collapser>

  <Collapser className="freq-link" id="certificate" title="인증서 확인 모니터 추가">

    7. 모니터를 실행할 [위치를](#setting-location) 선택한 다음 <DNT>**Save monitor**</DNT> 클릭하여 확인합니다.

       트래픽을 생성하고 몇 분 정도 기다린 후 [<DNT>**Monitors**</DNT> 색인](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-monitors-dashboard-access-your-monitors) 에서 모니터를 확인하세요.

한 번에 하나 이상의 모니터를 배포하고 싶으십니까?[NerdGraph API 합성 모니터링 튜토리얼](/docs/apis/nerdgraph/examples/nerdgraph-synthetics-tutorial) 을 확인하는 것이 좋습니다.API 호출로 모니터 생성을 자동화할 수 있습니다.

[합성 모니터링 REST API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api) 를 사용하여 모니터를 생성할 수도 있습니다.REST API의 경우 `GET` 요청을 수행하여 선택한 합성 모니터의 구성 세부정보를 검색합니다.해당 데이터를 검색한 후 `POST` 을 사용하여 선택한 모니터의 &quot;사본&quot;을 만듭니다.

## 모니터 편집 [#editing-monitors]

<img src="/images/synthetic_screenshot-full_edit-monitor-from-general.webp" title="Edit a monitor page" alt="A screenshot that shows the New Relic UI when you're editing your monitor." />

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets) &gt; General**</DNT>: 큐렐릭이 항상 가장 적절한 데이터를 받을 수 있도록 모니터를 최신 상태로 유지하세요.
</figcaption>

모니터가 생성된 후에는 모니터 [유형](#setting-type) 을 변경할 수 없지만 다른 모니터 설정은 편집할 수 있습니다.

1. <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> 으로 이동한 다음 편집하려는 모니터를 선택하세요.

2. 사이드 메뉴에서 링크를 선택하여 다음 [설정](#settings) 을 변경합니다.

   * [이름](#setting-name), URL, [위치](#setting-location), [빈도](#setting-frequency) 및 고급 옵션을 수정하려면 <DNT>**Settings &gt; General**</DNT> 선택하세요.
   * <DNT>**Scripted browser**</DNT> 및 <DNT>**API test**</DNT> 모니터의 경우 <DNT>**Settings &gt; Script**</DNT> 선택하여 모니터 펼쳐보기를 편집하세요.
   * 신세틱 모니터링 알림을 보려면 <DNT>**Manage alerts**</DNT> 클릭하세요.

3. 확인하려면 <DNT>**Save changes**</DNT> 선택하세요.

<Callout variant="important">
  모니터를 편집할 때 [새로운 합성 런타임](/docs/synthetics/synthetic-monitoring/getting-started/new-runtime) 으로 변환하십시오. [새 공용 IP 범위](/docs/synthetics/synthetic-monitoring/administration/new-synthetic-public-minion-ips) 의 네트워크 요청을 수락하도록 허용 목록을 업데이트해야 합니다.
</Callout>

## 모니터 삭제 [#deleting-monitors]

<img src="/images/synthetic_screenshot-crop_delete-monitor.webp" title="Edit a monitor page to delete monitor" alt="A screenshot of the Edit a monitor page with a red arrow that shows where someone should look to delete a monitor." />

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets) &gt; General**</DNT>: 모니터 편집 페이지에서 신세틱스 모니터를 삭제하세요.
</figcaption>

모니터를 삭제하려면 [관리자 권한](/docs/synthetics/synthetic-monitoring/administration/user-roles-synthetic-monitoring/) 이 있어야 합니다.

모니터를 삭제하려면:

1. <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> 으로 이동한 다음 편집하려는 모니터를 선택하세요.
2. <DNT>**General**</DNT> 클릭한 다음 모니터 삭제 버튼을 클릭합니다.

[NerdGraph API](/docs/apis/nerdgraph/examples/nerdgraph-synthetics-tutorial) 또는 [종합 모니터링 REST API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#delete-monitor) 를 사용하여 모니터를 삭제할 수도 있습니다.

## 모니터 설정 [#settings]

모니터를 구성할 때 다음 설정을 사용할 수 있습니다.

<CollapserGroup>
  <Collapser id="setting-type" title="유형">
    생성할 모니터 유형을 선택합니다. 모니터가 생성된 후에는 모니터 [유형](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) 을 변경할 수 없습니다.

    * <DNT>**Ping**</DNT>: 가용성을 모니터링할 단일 URL을 지정합니다. 뉴렐릭은 HEAD 또는 GET 요청을 통해 이 URL을 확인합니다. 이 모니터에 대해 구성할 수 없는 [시간 제한](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) 은 60초입니다.
    * <DNT>**Simple browser**</DNT>: 실제 브라우저를 통해 모니터링할 단일 URL을 지정합니다. 각 [빈도 간격](#setting-frequency) 마다 뉴렐릭은 셀레니엄 기반 Chrome 또는 Firefox 브라우저를 통해 이 URL을 확인합니다. 이 모니터의 구성 불가능한 [시간 초과](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers/#waiting-elements) 는 60초입니다.
    * <DNT>**Scripted browser**</DNT>: 셀레니엄 기반의 Chrome 또는 Firefox 브라우저를 구동하기 위한 [펼쳐보기를 만듭니다](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) . 브라우저는 스크립트의 각 단계를 따라 복잡한 동작이 예상대로 작동하는지 확인합니다(예: 웹사이트를 검색한 다음, 검색 결과 중 하나를 클릭하는 것). 이 모니터의 구성 불가능한 [시간 초과는](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) 180초입니다.
    * <DNT>**API test**</DNT>: API 엔드포인트가 올바르게 작동하는지 확인하기 위해 API 스크립트를 만듭니다. 자세한 내용은 [API 테스트 작성을](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests) 참조하세요. 이 모니터에 대해 구성할 수 없는 시간 제한은 180초입니다.
    * <DNT>**Step monitor**</DNT>: 셀레니엄 기반 Chrome 또는 Firefox 브라우저를 구동하기 위해 하나 이상의 단계를 구성하는 코드 없는 옵션입니다. 브라우저는 각 단계를 따라 복잡한 동작이 예상대로 작동하는지 확인합니다(예: 웹사이트를 검색한 다음, 검색 결과 중 하나를 클릭하는 것). 이 모니터의 구성 불가능한 [시간 초과는](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) 180초입니다.
    * <DNT>**Certificate check**</DNT>: SSL 인증서가 구성 가능한 일수 또는 그 이하로 만료되는지 확인합니다.
    * <DNT>**Broken links**</DNT>: URL에서 발견된 모든 링크를 테스트하여 성공적인 HTTP 응답 코드로 응답하는지 확인합니다.
  </Collapser>

  <Collapser id="setting-name" title="모니터 이름">
    모니터의 이름을 정의합니다. 모니터 이름에는 인코딩되지 않은 꺾쇠 괄호( `<` 및 `>` )가 포함될 수 없습니다. 모니터 이름에 꺾쇠 괄호를 포함하려면 UI 또는 API에서 HTML 괄호 엔터티(`<`の場合は `&lt;`、`>` の場合は `&gt;` )로 인코딩합니다.
  </Collapser>

  <Collapser id="setting-location" title="위치">
    모니터를 실행할 위치를 선택하십시오. 애플리케이션을 전 세계 사용자가 사용할 수 있도록 더 많은 위치를 선택하세요. [개인 위치](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations) 가 있는 경우 여기에도 나열됩니다. [Synthetics API `location` 엔드포인트](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#list-locations) 를 사용하여 계정의 유효한 위치 목록을 검색할 수 있습니다.

    모니터는 각 [주파수 간격](#setting-frequency) 동안 선택한 각 위치에서 하나의 검사를 실행합니다. 예를 들어, 3개의 위치를 선택하고 15분의 빈도를 정의하면 모니터는 각 15분 동안 3개의 검사를 실행합니다(또는 월별 8,640개의 검사).
  </Collapser>

  <Collapser id="setting-frequency" title="빈도">
    모니터 실행 빈도를 분, 시간 또는 1일 단위로 선택합니다. 이 빈도는 각 [위치](#setting-location) 에 적용됩니다. 예를 들어, 세 개의 위치를 선택하고 빈도를 <DNT>**15 minutes**</DNT> 으로 선택하면 모니터는 평균 5분마다, 매 15분 동안 3번의 확인(또는 월 8,640번의 확인)을 실행합니다.
  </Collapser>

  <Collapser id="response-validation" title="응답 검증(선택 사항)">
    페이지 DOM에서 검색할 텍스트를 지정합니다. 단순 브라우저 또는 핑 [모니터 유형](/docs/synthetics/synthetic-monitoring/getting-started/types-synthetic-monitors/#types-monitors) 을 사용하는 경우 페이지 로드에 1MB(10^6바이트) 제한이 있습니다.
  </Collapser>

  <Collapser id="apdex-t" title="Apdex T(선택 사항)">
    모니터의 <DNT>**tolerable**</DNT> 응답 시간에 대한 레버 값을 지정하십시오. 기본값은 7초(7000ms)입니다. 자세한 내용은 [SLA 보고서 지표 이해를](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-aggregate-monitor-metrics/#understanding) 참조하세요.
  </Collapser>
</CollapserGroup>
