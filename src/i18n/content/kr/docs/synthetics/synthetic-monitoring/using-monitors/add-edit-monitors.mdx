합성 모니터는 웹사이트를 모니터링하는 API 검사 또는 가상 브라우저 인스턴스로, [각 검사를 자세히](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results) 기록합니다. 또한 다음을 수행할 수 있습니다.

* 핑 모니터에 [대한 개요](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-overview-dashboard-view-monitors-performance) 또는 [요약](/docs/synthetics/new-relic-synthetics/pages/summary-page) 을 포함하여 집계 숫자를 캡처합니다.
* 각 [페이지 리소스](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-resources-dashboard-understand-load-times) 및 [다운타임 인시던트](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-downtimes-dashboard-troubleshoot-downtime) 에 대한 자세한 통계를 제공합니다.
* 모니터 실행에 대한 자세한 내용은 [사용자 정의 응답 코드](/docs/synthetics/new-relic-synthetics/using-monitors/synthetics-response-codes) 를 수집하십시오.

[New Relic 계정을 만든](https://newrelic.com/signup) 후 환경에 맞게 합성 모니터를 추가하고 구성할 수 있습니다. (영원히 무료입니다!)

<Callout variant="important">
  모니터를 편집할 때 [새로운 합성 런타임](/docs/synthetics/synthetic-monitoring/getting-started/new-runtime) 으로 변환하십시오. [새 공용 IP 범위](/docs/synthetics/synthetic-monitoring/administration/new-synthetic-public-minion-ips) 의 네트워크 요청을 수락하도록 허용 목록을 업데이트해야 합니다.
</Callout>

## 모니터 추가 [#adding-monitors]

여러 [유형의 합성 모니터](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors) 를 추가할 수 있습니다.

* Ping 모니터는 웹 사이트가 응답하는지 확인하고 간단한 브라우저 모니터는 실제 브라우저를 보내 웹 사이트를 확인합니다.
* 더 복잡한 모니터링의 경우 스크립트로 작성된 브라우저 모니터는 특정 리소스가 있는지 확인하는 반면 API 테스트는 API 끝점을 확인합니다.

새 모니터의 기본 시간 초과는 180초입니다.

단계 모니터를 사용하여 코드를 작성하지 않고 복잡한 워크플로 유효성 검사 스크립트를 작성하는 방법을 배우려면 이 짧은 YouTube 동영상(약 3분 20분)을 시청하십시오.

    title="핑 또는 간단한 브라우저 모니터 추가"
  >
    1. [**one.newrelic.com**](http://one.newrelic.com/) 으로 이동 **> Synthetics > <Icon name="fe-plus-circle"/>Create monitor** .

    2. [모니터 유형](#setting-type) , [이름](#setting-name) 및 URL(유효한 HTTP 또는 HTTPS URL일 수 있음)을 지정하십시오.

    3. 선택 사항: **유효성 검사 문자열** 또는 **고급 옵션을** 추가합니다.

       * ping 및 단순 브라우저에 **유효성 검사 문자열** 을 사용할 수 있습니다. 이 옵션은 [응답 유효성 검사](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors#response-validation) 를 위한 하위 문자열 모니터링을 활성화합니다.

       * ping 및 단순 브라우저에서 **SSL** 을 사용할 수 있는지 확인합니다. 이 옵션은 SSL 인증서 체인의 유효성을 확인합니다. 다음 구문을 실행하여 복제할 수 있습니다.

         ```
         openssl s_client -servername {<var>YOUR_HOSTNAME</var>} -connect {<var>YOUR_HOSTNAME</var>}:443 -CApath /etc/ssl/certs -verify_hostname {<var>YOUR_HOSTNAME</var>} > /dev/null
         ```

         0이 아닌 종료 코드가 반환되면 모니터가 실패합니다.

       * **Bypass HEAD 요청** 옵션은 ping에 사용할 수 있습니다. 이 옵션은 기본 `HEAD` 요청을 건너뛰고 대신 ping 확인과 함께 `GET` 동사를 사용합니다.

       * **리디렉션은 ping에 실패** 할 수 있습니다. Redirect is Failure가 활성화되었을 때 리디렉션 결과가 발생하면 New Relic Synthetics는 리디렉션을 따라 결과 URL을 확인하는 대신 결과를 실패로 분류합니다.

    4. 모니터를 실행할 [위치](#setting-location) 를 선택하십시오.

    5. 빈도를 선택하여 각 위치에서 모니터를 실행하는 [빈도](#setting-frequency) 를 결정하십시오.

    6. 선택 사항: [경고 알림](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics) 을 설정합니다.

    7. **내 모니터 만들기** 를 선택하여 확인합니다.

       약간의 트래픽을 생성하고 몇 분 정도 기다린 다음 [**모니터** 인덱스](/docs/new-relic-one-monitors-index) 에서 모니터를 확인하십시오.
  </Collapser>

## 모니터 편집 [#editing-monitors]

모니터가 생성된 후에는 모니터 [유형](#setting-type) 을 변경할 수 없지만 다른 모니터 설정은 편집할 수 있습니다.

1. [**one.newrelic.com**](http://one.newrelic.com/) **> Synthetics** 의 **모니터** 탭에서 편집하려는 모니터를 선택합니다.

2. 사이드 메뉴에서 링크를 선택하여 다음 [설정](#settings) 을 변경합니다.

   * **Settings > General** 을 선택하여 [이름](#setting-name) , [Apdex T](#setting-apdex) , URL, [위치](#setting-location) , [빈도](#setting-frequency) 및 고급 옵션을 편집합니다.
   * **스크립팅된 브라우저** 및 **API 테스트** 모니터의 경우 **Settings > Script** 를 선택하여 모니터 스크립트를 편집하십시오.
   * 종합 모니터링 경고의 경우 경고 **관리** 를 클릭합니다.

3. **변경 사항 저장** 을 선택하여 확인합니다.

## 모니터 삭제 [#deleting-monitors]

모니터를 삭제하려면 [관리자 권한](/docs/synthetics/synthetic-monitoring/administration/user-roles-synthetic-monitoring/) 이 있어야 합니다.

모니터를 삭제하려면:

1. [**one.newrelic.com**](http://one.newrelic.com/) **> Synthetics** 의 **모니터** 탭에서 편집하려는 모니터를 선택합니다.
2. 선택한 모니터에서 **Settings > General** 을 선택합니다.
3. 페이지 하단으로 스크롤하여 휴지통 아이콘을 선택합니다.

[NerdGraph API](/docs/apis/nerdgraph/examples/nerdgraph-synthetics-tutorial) 또는 [Synthetics REST API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#delete-monitor) 를 사용하여 모니터를 삭제할 수도 있습니다.

## 모니터 설정 [#settings]

모니터를 구성할 때 다음 설정을 사용할 수 있습니다.

<CollapserGroup>
  <Collapser
    id="setting-type"
    title="유형"
  >
    생성할 모니터 유형을 선택합니다. 모니터가 생성된 후에는 모니터 [유형](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) 을 변경할 수 없습니다.

    * **Ping** : 가용성을 모니터링할 단일 URL을 지정합니다. New Relic은 HEAD 또는 GET 요청을 통해 이 URL을 확인합니다. 이 모니터에 대해 구성할 수 없는 [제한 시간](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) 은 60초입니다.
    * **단순 브라우저** : 실제 브라우저를 통해 모니터링할 단일 URL을 지정합니다. 각 [빈도 간격마다](#setting-frequency) New Relic은 Selenium 기반 Google Chrome 브라우저를 통해 이 URL을 확인합니다. 이 모니터에 대해 구성할 수 없는 [제한 시간](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) 은 60초입니다.
    * **스크립팅된 브라우저** : Selenium 기반 Google Chrome 브라우저를 구동 [하는 스크립트를 만듭니다](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) . 브라우저는 스크립트의 각 단계를 따라 복잡한 동작이 예상대로 작동하는지 확인합니다(예: 웹사이트 검색 후 검색 결과 중 하나 클릭). 이 모니터에 대해 구성할 수 없는 [제한 시간](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) 은 180초입니다.
    * **API 테스트** : API 엔드포인트가 올바르게 작동하는지 확인하기 위해 API 스크립트를 작성하십시오. 자세한 내용은 [API 테스트 작성](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests) 을 참조하십시오. 이 모니터에 대해 구성할 수 없는 제한 시간은 180초입니다.
  </Collapser>

  <Collapser
    id="setting-name"
    title="모니터 이름"
  >
    모니터의 이름을 정의합니다. 모니터 이름에는 인코딩되지 않은 꺾쇠 괄호( `< >` )가 포함될 수 없습니다. 모니터 이름에 꺾쇠 괄호를 포함하려면 UI 또는 API에서 HTML 괄호 엔터티( `&lt; &gt;` )로 인코딩합니다.
  </Collapser>

  <Collapser
    id="setting-location"
    title="위치"
  >
    모니터를 실행할 위치를 선택하십시오. 애플리케이션을 전 세계 사용자가 사용할 수 있도록 더 많은 위치를 선택하세요. [개인 위치](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations) 가 있는 경우 여기에도 나열됩니다. [Synthetics API `location` 엔드포인트](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#list-locations) 를 사용하여 계정의 유효한 위치 목록을 검색할 수 있습니다.

    모니터는 각 [주파수 간격](#setting-frequency) 동안 선택한 각 위치에서 하나의 검사를 실행합니다. 예를 들어, 3개의 위치를 선택하고 15분의 빈도를 정의하면 모니터는 각 15분 동안 3개의 검사를 실행합니다(또는 월별 8,640개의 검사).
  </Collapser>

  <Collapser
    id="setting-frequency"
    title="빈도"
  >
    모니터 실행 빈도를 분, 시간 또는 1일 단위로 선택합니다. 이 빈도는 각 [위치](#setting-location) 에 적용됩니다. 예를 들어, 3개의 위치와 **15분** 의 빈도를 선택하면 모니터는 각 15분 기간(또는 월별 8,640개의 검사)에서 평균 5분마다 3개의 검사를 실행합니다.
  </Collapser>

  <Collapser
    id="setting-alerts"
    title="경고"
  >
    모니터 실패 시 경고를 수신할 이메일 주소를 지정하십시오. 또는 더 많은 알림 옵션을 위해 기존 경고 정책에 모니터를 연결합니다. 자세한 내용 [은 종합 모니터링 경고를](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics) 참조하십시오.
  </Collapser>

  <Collapser
    id="setting-apdex"
    title="Apdex T"
  >
    이 모니터에 대한 [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) 를 사용자 정의하십시오. 이 설정은 새 모니터를 생성할 때가 아니라 기존 모니터에 대한 설정을 편집할 때만 사용할 수 있습니다. [SLA 보고서에서 보다 정확한 Apdex 점수를](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-sla-report-dashboard-aggregate-metrics-all-monitors) 얻으려면 Apdex T를 기본 7초에서 변경하십시오. 예를 들어, [스크립트로 작성된 브라우저](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) 가 매우 긴 경우 일반적인 완료 시간을 더 가깝게 반영하기 위해 Apdex T를 15초로 조정할 수 있습니다. 유사하게, 간단한 브라우저 검사를 위한 좋은 Apdex T는 2초에 불과할 수 있습니다.
  </Collapser>

  <Collapser
    id="response-validation"
    title="응답 검증(선택 사항)"
  >
    페이지에서 검색할 텍스트를 지정합니다. 단순 브라우저 또는 ping [모니터 유형](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) 을 사용하는 경우 페이지 로드에 1MB(10^6바이트) 제한이 있습니다.
  </Collapser>
</CollapserGroup>
