## SLA 보고서 측정항목 이해 [#understanding]

SLA 보고서를 사용하여 계정 전체의 **SLA 보고서** 페이지에서 단일 모니터 또는 모든 모니터에 대해 집계된 성능 메트릭을 봅니다. SLA 보고서에는 다음 측정항목이 포함됩니다.

* **지속 시간** : 모든 모니터 결과의 평균 지속 시간입니다.

* **가동 시간** : 성공적으로 종료된 모든 모니터 결과의 백분율입니다. 예를 들어 모니터 A는 하루에 50번, 모니터 B는 하루에 150번 확인할 수 있습니다. 모니터 A가 50개 중 29개 성공하고 모니터 B가 150개 중 148개 성공했다면 **가동 시간** 은 88.5가 됩니다.

  (29+148)/(50+150)=88.5

  개별 SLA 보고서의 경우 가동 시간 점수에는 선택한 모니터만 포함됩니다.

* **Apdex** : 모든 모니터의 평균 [Apdex입니다](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) . 모니터에는 Synthetics 모니터를 생성하거나 업데이트할 때 수정할 수 있는 7초의 기본 [Apdex T가](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) 있습니다. 실망스러운 결과를 정의하는 [Apdex F](/docs/apm/new-relic-apm/getting-started/glossary#apdex_f) 는 항상 Apdex T의 4배입니다. Apdex에 대한 자세한 내용은 [Apdex: 사용자 만족도 측정을](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) 참조하세요. 다른 Apdex T 값을 사용하여 합성 모니터에 대한 Apdex 점수를 보려면 [NRQL apdex 함수](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/#func-apdex) 및 [SyntheticCheck 이벤트를](/attribute-dictionary/?event=SyntheticCheck) 사용하십시오.

  개별 SLA 보고서의 경우 Apdex 점수에는 선택한 모니터만 포함됩니다.

* **% Satisfied** : ["만족스러운" 시간](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction#bullet-satisfied) 에 완료된 모니터 결과의 백분율입니다. [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) 이하에서 완료되는 모니터 결과를 만족 시간으로 정의합니다.

* **% Toleration** : ["허용 가능한" 시간](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction#bullet-tolerating) 내에 완료되는 모니터 결과의 백분율. 허용 가능한 시간은 [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) 보다 크지만 [Apdex F](/docs/apm/new-relic-apm/getting-started/glossary#apdex_f) ( [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) 의 4배)보다 작습니다.

* **% Frustrated** : ["실망스러운" 시간](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction#bullet-tolerating) 에 완료되는 모니터 결과의 백분율입니다. 답답한 시간은 [Apdex F](/docs/apm/new-relic-apm/getting-started/glossary#apdex_f) 보다 큽니다( [Apdex T의](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) 4배).

계정 전체 SLA 보고서에는 모든 모니터 유형(ping, 단순 브라우저, 스크립트 브라우저 및 API 테스트)이 포함됩니다.

## SLA 값 생성 [#generating_sla_in_insights]

SLA 보고서의 값은 사용 가능한 종합 모니터링 데이터에 대한 NRQL 쿼리에서 생성됩니다. 이러한 값을 쉽게 다시 만들고 필요에 맞게 쿼리를 수정할 수 있습니다.

이 쿼리는 평균 기간, apdex 및 가동 시간을 반환합니다. 아래에 강조 표시되고 설명된 변수를 값으로 대체하십시오.

```sql
SELECT average(duration), apdex(duration, t:), percentage(count(*), WHERE result='SUCCESS') FROM SyntheticCheck FACET timeperiod
```

[NRQL 쿼리는](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) 기본적으로 지난 데이터 시간에 대해 쿼리합니다. 데이터 범위를 넓히려면 쿼리 끝에 [`SINCE` 절을](/docs/insights/new-relic-insights/using-new-relic-query-language/nrql-reference#sel-since) 포함해야 합니다.

**예 #1: 지난 주에 대한 일일 보고서**

지난 주에 대한 일일 보고서를 생성하려면 `SINCE 1 week ago` 을 추가합니다.

```sql
SELECT average(duration), apdex(duration, t:), percentage(count(*), WHERE result='SUCCESS') 
FROM SyntheticCheck FACET dateOf(timestamp) SINCE 1 week ago
```

**예 #2: 특정 모니터에 대한 보고서**

결과 범위를 특정 모니터로 지정하려면 특정 모니터 이름을 포함하도록 아래 쿼리를 편집할 수 있습니다.

```sql
SELECT average(duration), apdex(duration, t:), percentage(count(*), WHERE result='SUCCESS') 
FROM SyntheticCheck FACET timeperiod WHERE monitorName = 'mymonitorname'
```

**예 #3: 다중 모니터에 대한 보고서**

결과 범위를 모니터 모음으로 지정하려면 다음을 수행합니다.

```sql
SELECT average(duration), apdex(duration, t:), percentage(count(*), WHERE result='SUCCESS') 
FROM SyntheticCheck FACET timeperiod WHERE monitorName IN ('mymonitor1', 'mymonitor2', 'mymonitor3')
```
