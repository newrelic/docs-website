<DNT>**Resources**</DNT> 페이지를 사용하여 웹사이트의 각 리소스가 로드 시간에 미치는 영향을 파악하세요. [리소스 로드 시간](#resources-load-time), [타사에서 소요된 시간](#third-parties) 및 [비정상 응답 코드](#response-codes) 그래프를 사용하여 전체적인 개요를 파악하고, [리소스 목록을](#resources-list) 사용하여 대용량 CSS 파일이나 이미지와 같이 영향이 큰 리소스를 식별하십시오.

단일 리소스에 대한 성능 정보를 집계하는 [자세한 리소스 메트릭](#detailed-metrics) 으로 특정 리소스를 조사할 수도 있습니다. 성능 문제를 정확히 찾아내기 위해 정확한 요청 및 응답 헤더를 사용하여 [해당 리소스의 한 로드를](#resource-results) 더 자세히 살펴보십시오.

모니터의 <DNT>**Resources**</DNT> 페이지에 접속하려면 <DNT>**[one.newrelic.com](https://one.newrelic.com/synthetics-nerdlets) &gt; Synthetic monitoring &gt; (select a monitor) &gt; Resources**</DNT> 로 이동하세요.

## 리소스 메트릭 이해 [#understanding]

<DNT>**Resources**</DNT> 페이지를 사용하여 웹사이트의 각 리소스가 성능에 미치는 영향을 파악하세요.

* <DNT>**Resources list**</DNT>: <DNT>**average download time**</DNT> 에서 모니터링하는 웹사이트의 리소스를 나열합니다. 이 목록을 활용하여 다운로드 시간을 단축함으로써 웹사이트 성능을 최적화하세요. <DNT>**Search all resources**</DNT> 필드에서 특정 리소스를 검색하여 해당 리소스의 로드 시간을 확인할 수도 있습니다. [개별 리소스를 보려면](#individual) 리소스를 선택하세요.
* <DNT>**Load time**</DNT>: 모든 리소스의 평균 로드 시간을 리소스 유형(예: `jpeg`, `json` 또는 `javascript`)별로 분류하여 그래프로 표시합니다. 이 그래프를 사용하여 웹사이트 로딩 시간이 시간에 따라 어떻게 변하는지 파악하세요.
* <DNT>**Time spent by third parties**</DNT>웹 글꼴, 내장된 트위터 타임라인, 외부에서 호스팅되는 이미지 등 모든 타사 리소스의 평균 로드 시간을 그래프로 보여줍니다. 이 그래프를 사용하여 타사 웹사이트가 웹사이트 성능에 영향을 미치는지 확인하세요.
* <DNT>**Irregular response codes**</DNT>모니터가 수신한 부정적인 HTTP 응답 코드([HTTP 상태 코드](http://en.wikipedia.org/wiki/List_of_HTTP_status_codes) 라고도 함)의 수를 그래프로 나타냅니다. 이 그래프에는 300(리디렉션), 400(클라이언트 오류) 및 500(서버 오류) 범위의 모든 응답이 포함되어 있습니다. 100이나 200 범위의 응답은 계산에 포함되지 않습니다. 신세틱스는 또한 여러 개의 [사용자 지정 응답 코드를](/docs/synthetics/new-relic-synthetics/using-monitors/synthetics-response-codes) 수집합니다.

이러한 메트릭을 분석하여 문제가 되는 리소스를 추적하고 성능을 개선할 수 있습니다. 예를 들어 최적화되지 않은 로고 이미지는 사이트의 모든 방문자에게 속도 저하를 일으킬 수 있는 반면 미국 기반 서버에서만 제공되는 추적기는 미국 이외의 방문자에게 성능 저하를 초래할 수 있습니다.

## 자세한 리소스 측정항목 보기 [#detailed-metrics]

사이트의 각 리소스에 대한 자세한 참조 정보도 확인할 수 있습니다. 자세한 범위를 보려면 [특정 리소스를 찾](#locate-resource) 거나 <DNT>**Resources**</DNT> 페이지에서 흥미로운 리소스를 선택하세요. 리소스를 선택한 후에는 해당 리소스에 대한 개요 메트릭을 보거나 [개별 리소스 부하를 볼](#resource-results) 수 있습니다.

* <DNT>**Max duration**</DNT>, <DNT>**Min duration**</DNT>, <DNT>**Avg duration**</DNT>: [선택한 기간](#time-frame) 동안 이 리소스에 대한 로드 시간 메트릭입니다.
* <DNT>**For 50% of requests**</DNT>: requests 의 <DNT>**fastest**</DNT> %에 대한 최대 다운로드 시간입니다. 이는 해당 리소스에 대한 requests 의 50%가 지정된 시간 내에 또는 그보다 빠르게 완료되었음을 의미합니다.
* <DNT>**For 95% of requests**</DNT>: requests 의 <DNT>**fastest**</DNT> %에 대한 최대 다운로드 시간입니다. 이는 해당 리소스에 대한 requests 의 95%가 지정된 시간 내에 또는 그보다 더 빨리 완료되었음을 의미합니다.
* <DNT>**Average load time**</DNT>[선택한 기간](#time-frame) 동안 해당 리소스의 평균 로드 시간을 그래프로 표시합니다. 이 그래프를 사용하여 웹사이트 최적화(예: 이미지 크기 축소 또는 CSS 최소화)가 제대로 작동하는지 확인할 수 있습니다.

[차트 요소를 숨기거나 표시할 수도 있고](#hide-unhide), 위치별로 필터링하거나 [리소스 결과](#resource-results) 목록을 <DNT>**Download time**</DNT> 또는 <DNT>**DNS lookup**</DNT> 기준으로 정렬할 수도 있습니다. 리소스 이름 위에 마우스를 올리면 전체 경로를 볼 수 있습니다(동적으로 생성된 리소스 이름이 매우 긴 경우에 유용합니다).

## 개별 리소스 로드 보기 [#resource-results]

특정 리소스에 대한 [자세한 대역폭을](#detailed-metrics) 확인한 후, 개별 리소스 부하를 선택하여 성능 저하 공지 이벤트를 심층적으로 이해할 수 있습니다. 그러면 전체 연결 시간 분석 결과를 확인하여 문제가 발생한 위치를 파악하고 요청 및 응답 헤더를 검토할 수 있습니다.

<img title="screen-synthetics-individual-resource-results.png" alt="screen-synthetics-individual-resource-results.png" src="/images/synthetic_screenshot-full_find-resource-load-details.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; (모니터 선택) &gt; <DNT>**Resources**</DNT> &gt; (리소스 선택) &gt; (리소스 부하 선택): 개별 리소스 부하를 선택하면 연결 시간, 요청 헤더 및 응답 헤더에 대한 자세한 분석을 볼 수 있습니다. [HTTP Archive 1.2 사양](http://www.softwareishard.com/blog/har-12-spec/#timings) 에서 가져온 네트워크 타이밍 구조입니다.
</figcaption>
