<DNT>**Failures**</DNT> 페이지를 사용하여 다운타임 공지 이벤트 또는 기타 오류를 찾아 문제를 해결하세요. 다운타임 공지 이벤트는 모니터가 완전히 실행되지 않을 때마다 발생합니다. 예를 들어, GET 요청이 실패하면 ping 모니터가 &quot;다운&quot;되고, 펼쳐의 일부라도 실행에 실패하면 멀티미디어 브라우저 모니터가 &quot;다운&quot;됩니다. 흥미로운 다운타임을 찾은 후 이를 선택하여 [해당 다운타임 공지 이벤트의 자세한 결과를 확인](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) 하고 문제를 해결하세요.

## 실패 페이지 보기 [#viewing]

모니터의 오류를 확인하려면: <DNT>**[one.newrelic.com &gt;Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; (모니터 선택) &gt; <DNT>**Failures**</DNT>.

<img title="Synthetic monitoring - Failures page" alt="Synthetic monitoring - Failures page" src="/images/synthetic_screenshot-full_failure-page.webp" />

<figcaption>
  오류 위에 마우스 커서를 올리면 해당 오류에 대한 빠른 정보를 얻을 수 있습니다. 점을 클릭하면 오류에 대한 자세한 보고서를 열 수 있습니다. [실패한 모니터를 다시 확인](/docs/synthetics/synthetic-monitoring/using-monitors/recheck-failed-monitors) 하려면 <DNT>**Run check**</DNT> 클릭할 수도 있습니다.
</figcaption>

## 개별 다운타임 보기 [#individual]

개별 다운타임 공지이벤트를 선택하시면 더 자세히 보실 수 있습니다. 특정 오류 유형에 따라 다운타임 결과에는 서버 오류 메시지(예: <DNT>**Server replied with &quot;HTTP 500&quot; error**</DNT>)만 포함되거나 전체 또는 부분 [워터폴 보기가](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) 포함될 수 있습니다.

다운타임 결과에는 모니터링의 일부만 올바르게 실행되었을 때의 워터폴 차트가 포함됩니다. 예를 들어, [301 리디렉션은](http://en.wikipedia.org/wiki/HTTP_301) 오류가 발생한 웹 페이지로 연결될 수 있습니다. 브라우저는 리디렉션을 올바르게 실행하지만, 대상 웹 페이지에서 오류가 발생합니다. 다운타임 공지 이벤트 문제를 해결하려면 오류 메시지 또는 폭포를 사용하세요.
