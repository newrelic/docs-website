**장애** 페이지를 사용하여 다운타임 인시던트 또는 기타 오류를 찾아 문제를 해결할 수 있습니다. 다운타임 사고는 모니터가 완전히 실행되지 않을 때마다 발생합니다. 예를 들어, GET 요청이 실패하면 ping 모니터가 "다운"되고 스크립트의 일부가 실행에 실패하면 스크립팅된 브라우저 모니터가 "다운"됩니다. 흥미로운 다운타임을 찾은 후 선택 [하여 해당 다운타임 사고의 자세한 결과를 보고](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) 문제를 해결합니다.

## 실패 페이지 보기 [#viewing]

모니터 장애에 액세스하려면 **[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)** > (모니터 선택) > **Failures**.

## 개별 다운타임 보기 [#individual]

개별 다운타임 인시던트를 선택하여 더 자세히 볼 수 있습니다. 특정 오류에 따라 다운타임 결과에는 서버 오류 메시지(예: **서버가 "HTTP 500" 오류로 응답함** ) 또는 전체 또는 부분 [폭포식 보기](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) 만 포함될 수 있습니다.

다운타임 결과에는 모니터의 일부만 올바르게 실행될 때 폭포수가 포함됩니다. 예를 들어, [301 리디렉션](http://en.wikipedia.org/wiki/HTTP_301) 은 실패한 웹 페이지로 연결될 수 있습니다. 브라우저는 리디렉션을 올바르게 실행하지만 대상 웹 페이지는 오류를 반환합니다. 오류 메시지 또는 폭포수를 사용하여 다운타임 문제를 해결합니다.
