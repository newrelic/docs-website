종합 모니터링의 **요약** 페이지는 웹사이트 성능을 한 눈에 이해할 수 있도록 합니다. 요청/응답 시간, 연결 시간, 오류 등의 추세를 확인할 수 있습니다. 필터를 사용하여 관심 있는 속성 또는 값으로 데이터 범위를 좁히거나 [실패한 모니터를 다시 확인](/docs/synthetics/synthetic-monitoring/using-monitors/recheck-failed-monitors) 하십시오.

## 요약 페이지 보기 [#viewing]

모니터의 **요약** 페이지에 액세스하려면: **[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)** > (모니터 선택)으로 이동합니다.

<img
  title="screen-synthetics-overview"
  alt="screen-synthetics-overview"
  src={syntheticsummaryPage}
/>

<figcaption>
  **요약** 페이지는 웹사이트 성능에 대한 높은 수준의 보기를 제공하고 위치 확인, 경고 위반 및 모니터의 메타데이터에 대한 바로 가기를 제공합니다.
</figcaption>

## 요약 페이지 이해 [#understanding]

**요약** 페이지를 사용하여 웹사이트의 성능을 이해하십시오.

  <thead>
    <tr>

      </td>

      <td>

        표시되는 기간은 모니터의 주파수에 따라 다릅니다.

        * 1분 = 30분
        * 5분 = 1.5시간
        * 15분 = 7.5시간
        * 30분 = 15시간
        * 1시간 = 30시간
        * 6시간 = 7.5일
        * 12시간 = 15일
        * 24시간 = 30일
      </td>

    <tr>
      <td id="load-time">

      </td>

      <td>

        * 1분 = 30분
        * 5분 = 1.5시간
        * 15분 = 7.5시간
        * 30분 = 15시간
        * 1시간 = 30시간
        * 6시간 = 7.5일
        * 12시간 = 15일
        * 24시간 = 30일
      </td>
    </tr>

    <tr>
      <td id="request-response-times">
        요청/응답 시간
      </td>

      <td>
        이것은 ping 모니터가 응답을 보내고 받는 데 걸린 시간을 나타내며 보낸 시간, 대기 시간, 수신 시간으로 나뉩니다.

        기간은 평균, 중앙값 또는 95번째 백분위수로 표시할 수 있습니다. 자세한 내용은 [백분위수: 순위 데이터](/docs/using-new-relic/user-interface-functions/view-your-data/percentiles-compare-ranked-data#options) 를 참조하십시오.

        이것은 **ping** 모니터 결과에만 사용할 수 있습니다.
      </td>
    </tr>

    <tr>
      <td id="connection-times">
        연결 시간
      </td>

      <td>
        이것은 이 모니터에 대해 설정될 네트워크 연결에 대한 분석을 보여주며 DNS, SSL 및 연결 지속 시간으로 분류됩니다.

        기간은 평균, 중앙값 또는 95번째 백분위수로 표시할 수 있습니다. 자세한 내용은 [백분위수: 순위 데이터](/docs/using-new-relic/user-interface-functions/view-your-data/percentiles-compare-ranked-data#options) 를 참조하십시오.

        이것은 **ping** 모니터 결과에만 사용할 수 있습니다.
      </td>
    </tr>

    <tr>
      <td id="non-200-response-codes">
        200이 아닌 응답 코드
      </td>

      <td>
        이것은 이 모니터를 실행한 결과로 수신된 모든(200이 아닌) 오류 응답 코드를 보여줍니다.

        이것은 **ping** 모니터 결과에만 사용할 수 있습니다.
      </td>
    </tr>

    <tr>
      <td id="activity-panel">
        활동 패널
      </td>

      <td>
        다음을 포함하여 모니터와 관련된 모든 최근 활동을 표시합니다.

        * 모니터 이름
        * 열린 위반 횟수
        * 최근 공개 및 종료 위반 및 감사 이벤트를 포함한 모니터 활동.
        * 메타데이터 및 [태그](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) .
        * 모니터링되는 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic) : 이 모니터가 실행될 때 관찰되는 모든 New Relic 엔터티이며 엔터티 경고 상태를 포함합니다. 엔터티 상태를 선택하면 모니터링되는 엔터티 요약 페이지로 이동합니다.
      </td>
    </tr>
  </tbody>
</table>

## 페이지 기능 사용 [#using]

**요약** 페이지는 다음 기능을 지원합니다.

        * 요청/응답 시간

        * 연결 시간

        * 200이 아닌 응답 코드

</Callout>

  </thead>

  <tbody>
    <tr>
      <td id="load-time">
        로드 시간 차트
      </td>

      <td>
        이것은 각 모니터 [위치](/docs/synthetics/new-relic-synthetics/using-monitors/adding-editing-monitors#setting-location)에 대한 로드 시간을 보여줍니다. [짧은 시간 프레임(예: 30분)을 선택](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) 하면 합성 모니터링이 각 위치에서 모니터를 실행할 때마다 정확한 로드 시간을 표시합니다. 더 긴 시간 프레임에서 합성 모니터는 해상도를 조정하여 평균을 표시합니다.

        위치의 가시성을 전환하려면 해당 레이블을 선택합니다.
      </td>
    </tr>

    <tr>
      <td id="availability">
        유효성
      </td>

      <td>
        이것은 성공한 모니터 실행의 백분율을 표시합니다. 스크립팅된 모니터의 경우 실행이 성공으로 간주되려면 전체 스크립트가 완료되어야 합니다.

        선택한 모니터의 **가용성** 차트 위의 줄임표 <Icon name="fe-more-horizontal"/>아이콘이 나타납니다. 실제 사용자( [브라우저 모니터링](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) ) 상호 작용과 합성 모니터 간의 직접적인 페이지 로드 시간 비교를 위해 New Relic의 [비교 차트 기능](/docs/synthetics/new-relic-synthetics/administration/compare-page-load-performance-browser-synthetics) 을 사용하십시오.
      </td>
    </tr>

    <tr>
      <td id="average-load-size">
        평균 부하 크기
      </td>

      <td>
        이 차트는 모니터를 실행할 때마다 소비된 데이터의 양(킬로바이트 또는 메가바이트)을 보여줍니다. 스크립팅된 모니터의 경우 전체 스크립트가 평균 로드 크기 그래프에 포함됩니다. 예를 들어, 세 페이지를 로드하는 모니터는 세 페이지 모두에 있는 모든 자산의 크기를 측정합니다.

        사이트가 정적인 경우 평균 로드 크기는 매우 일관적입니다. 뉴스 사이트와 같은 동적 사이트의 경우 페이지 콘텐츠가 변경됨에 따라 평균 로드 크기가 달라집니다.
      </td>
    </tr>

    <tr>
      <td id="slowest-results">
        가장 느린 결과
      </td>

      <td>
        선택한 시간 프레임에 대해 가장 느린 5개의 결과가 나열됩니다. [실적 분석을 보려면](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) 결과를 선택하세요.
      </td>
    </tr>

    <tr>
      <td id="monitor-downtimes">
        다운타임 모니터링
      </td>

      <td>
        여기에는 가장 최근의 모니터 다운타임 인시던트가 나열됩니다. [추가 세부 정보를 보려면](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) 다운타임 인시던트를 선택하십시오.
      </td>
    </tr>
