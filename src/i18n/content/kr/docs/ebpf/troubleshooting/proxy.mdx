뉴렐릭 <DNT>eBPF agent</DNT> 이(가) 예상대로 프록시를 통해 텔레메트리를 라우팅하지 않는 경우, 이 페이지를 사용하여 일반적인 오류를 진단하십시오. 설정 지침 및 설정 참조는 [뉴렐릭 <DNT>eBPF agent</DNT>용 프록시 설정](/docs/ebpf/proxy-configuration)을 참조하십시오.

## 일반적인 오류 [#common-errors]

<CollapserGroup>
  <Collapser id="error-400" title="HTTP 프록시가 응답 코드 400을 반환했습니다.">
    **문제:** 프록시가 HTTP <DNT>`CONNECT`</DNT> 메서드를 지원하지 않습니다.

    **해결 방법:** <DNT>`CONNECT`</DNT>지원 프록시를 사용하십시오. [지원되는 프록시 서버](/docs/ebpf/proxy-configuration#supported-proxies)를 참조하세요.
  </Collapser>

  <Collapser id="error-https-scheme" title="프록시 URI에서 “https” 스킴이 지원되지 않습니다">
    **문제:** gRPC는 <DNT>`https://`</DNT> 을(를) 프록시 체계로 지원하지 않습니다. 이는 gRPC 제한 사항이지, 에이전트 제한이 아닙니다.

    **해결 방법:** 프록시 체계를 <DNT>`http`</DNT> (예: <DNT>`otlpProxy.scheme: "http"`</DNT>)(으)로 설정합니다. 사용자의 텔레메트리 데이터는 여전히 프록시 터널을 통해 종단 간 TLS 암호화된 상태로 전송됩니다. 프록시 자체에 대한 연결만 평문입니다.
  </Collapser>

  <Collapser id="error-socket-closed" title="소켓이 닫혔거나 연결이 거부되었습니다">
    **문제:** 프록시 서버가 실행 중이 아니거나 호스트 또는 포트가 올바르지 않습니다.

    **해결 방법:** [<DNT>`curl`</DNT> 테스트 명령어](/docs/ebpf/proxy-configuration#test-connectivity)를 사용하여 프록시에 연결할 수 있는지 확인합니다.
  </Collapser>

  <Collapser id="error-connect-aborted" title="프록시 CONNECT 중단됨">
    **문제:** 방화벽 규칙, ACL 또는 지원되지 않는 대상 때문에 프록시가 <DNT>`CONNECT`</DNT> 요청을 거부했습니다.

    **해결 방법:** 프록시 서버 로그를 확인하여 에이전트의 소스 IP에서 포트 `443` 에 대한 <DNT>`CONNECT`</DNT> requests가 허용되는지 확인하십시오.
  </Collapser>

  <Collapser id="error-tls-required" title="프록시 설정 무시됨: gRPC HTTP CONNECT 프록시에는 TLS가 필요합니다">
    **문제:** 프록시가 구성되어 있지만 TLS가 비활성화되어 있습니다(예: <DNT>`TLS_ENABLED=false`</DNT>). gRPC의 HTTP CONNECT 프록시 구현은 OTLP 엔드포인트에 대한 연결이 TLS를 사용하는 경우에만 작동합니다.

    **해결 방법:** TLS가 활성화되어 있는지 확인합니다(기본적으로 활성화되어 있습니다). 그러면 에이전트는 구성된 프록시를 사용합니다.
  </Collapser>

  <Collapser id="error-credential-chars" title="프록시 사용자 이름에 특수 문자 또는 대문자가 포함되어 있습니다.">
    **문제:** 프록시 사용자 이름 또는 비밀번호에 gRPC의 URI 파서가 허용하지 않는 문자가 포함되어 있습니다. [자격 증명 문자 제한](/docs/ebpf/proxy-configuration#credential-limitations)을 참조하세요.

    **해결 방법:** 소문자 영숫자 사용자 이름과 문자, 숫자 및 허용된 특수 문자만 사용하는 비밀번호를 가진 프록시 계정을 사용하십시오.
  </Collapser>
</CollapserGroup>
