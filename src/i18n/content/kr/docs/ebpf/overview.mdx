뉴렐릭 eBPF 옵저버빌리티는 새로운 가시성 계층을 제공하여 현대적이고 복잡한 네트워크와 환경을 모니터링하는 데 도움이 됩니다.

## eBPF 에이전트는 어떤 가치를 제공합니까? [#value]

뉴렐릭 eBPF 옵저버빌리티는 Kubernetes 및 Linux 호스트 환경 전반에서 통합된 제로코드 가시성을 제공합니다. Linux 커널의 확장된 Berkeley Packet Filter(eBPF) 기술을 사용하여 자동화되고 언어에 구애받지 않는 인사이트를 제공합니다. 이 접근 방식은 수동 측정, 계측 또는 코드 변경의 필요성을 없애 운영 오버헤드를 줄입니다.

## 사용 사례 [#use-cases]

이 솔루션은 플랫폼 엔지니어가 직면한 중요한 과제를 해결합니다.

* **더 빠른 문제 해결, 해결:** 외관 및 네트워크 인사이트와 통합 텔레메트리의 상관관계를 확인하세요. 이를 통해 성능 문제, 특히 기본 시스템에서 발생하는 문제를 빠르게 진단할 수 있습니다.

* **자동 검색:** 제로 측정, 리소스로 자동화된 옵저버빌리티를 받으세요. eBPF 에이전트는 실행 중인 모든 서비스와 해당 서비스의 의존성/종속성을 지속적으로 식별하고 매핑하여 기능 언어에 대한 사전 지식 없이도 새롭거나 알려지지 않은 워크로드에 대한 아키텍처 뷰를 제공합니다.

* **통합 모니터링:** 다양한 환경에서 어떤 언어로든 작성된 모니터 기능입니다. 이를 통해 뉴렐릭 사용자 인터페이스 내에서 직접 일관된 가시성과 서비스의 다양한 성능 분석을 제공합니다.

## 타격 [#impact]

최소한의 리소스 오버헤드로 더 심층적인 옵저버빌리티를 달성하고 문제를 더 빠르게 해결할 수 있으므로 모델링이 성능에 영향을 미치지 않도록 할 수 있습니다. 에이전트는 시스템 역학에 대한 포괄적인 이해를 위해 고유한 커널 수준 및 네트워크 세부 정보를 제공합니다.

eBPF 에이전트 작동 방식 \[#how-it-works] 뉴렐릭 eBPF 에이전트는 Kubernetes 에서 helm 차트를 사용하고 Linux 호스트에서 Debian/RPM 패키지를 사용하여 배포합니다. Kubernetes에서는 \`daemonset\`으로 실행되고 Linux 호스트에서는 시스템 서비스로 실행됩니다. 에이전트를 설치하면 해당 환경에서 실행 중인 모든 서비스와 애플리케이션을 자동으로 감지합니다. 이러한 부분은 eBPF 부분 아래 뉴렐릭의 APM &amp; Service UI 에 통합되어 있습니다. 뉴렐릭 eBPF 에이전트에는 두 가지 모드가 있습니다. \* \*\*검색 모드:\*\* 활성 모니터링 없이 환경에서 자동으로 검색합니다. 이 모드는 초기 설정이나 덜 방해가 되는 모니터링에 적합합니다. \* eAPM 모드:\*\* 시스템 역학에 대한 포괄적인 이해를 위해 적극적으로 모델링하고 하위 레벨 지표를 수집하여 더 깊은 수준의 옵저버빌리티를 잠금 해제합니다. ## eBPF가 APM 보완하는 방식 \[#ebpf-APM] \*/

## eBPF 에이전트를 설정하는 방법 [#setup]

Linux 호스트에 뉴렐릭 eBPF 에이전트를 설치하고 쿠버네티스 클러스터를 설치하면 전체 시스템 상태를 모니터링할 수 있습니다. eBPF 에이전트는 코드 변경이나 구현, 배포할 필요 없이 언어별 에이전트에 대한 심층적인 가시성을 제공합니다.
