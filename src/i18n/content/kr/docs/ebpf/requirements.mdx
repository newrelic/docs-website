      ### Kubernetes 환경 [#kubernetes-environments]

      다음과 같은 Kubernetes 환경이 지원됩니다.

      * Amazon Elastic Kubernetes Service(EKS)
      * 구글 쿠버네티스 엔진(GKE)
      * Azure Kubernetes 서비스(AKS)
      * 레드햇 오픈시프트
      * Microk8s

      ### 컨테이너 런타임 지원 [#container-runtime]

      지원되는 컨테이너 런타임은 다음과 같습니다.

      * 도커 기본 런타임(runc)
      * 컨테이너d
      * 크리오
      * 포드만
      * 유키
      * 크런
    </TabsPageItem>

    <TabsPageItem id="linux">
      ### 커널 요구 사항 [#kernel-requirements]

      * 리눅스 커널 버전:

        * `x86-64` (amd64): `v5.8` 이상
        * `arm64` (aarch64): `v5.8` 이상

      <Callout variant="important">
        일부 리눅스 배포판에서는 최신 OS 버전에서 이전 커널 버전을 실행할 수 있습니다. [아래 목록에](#supported-linux-distributions) 배포판이 포함되어 있더라도 커널 버전이 `5.8` 보다 낮으면 eBPF 에이전트가 작동하지 않습니다.
      </Callout>

      ### 지원되는 Linux 배포판 [#supported-linux-distributions]

      다음 리눅스 배포판이 지원됩니다.

      * CentOS `9` 이상
      * Debian `11` 이상
      * Red Hat Enterprise Linux `8.1` 이상
      * Ubuntu `20.04` 이상
      * Amazon Linux `2` 및 `2023`
      * Bottle 로켓 `1.51.0` 이상
      * 알마리눅스 `9.7`

      ### 시스템 요구 사항 [#system-requirements]

      * 루트 권한(설치 및 작동에 필요)
      * 최소 `2 GB` 의 RAM
    </TabsPageItem>
  </TabsPages>
</Tabs>

### 추가 프로토콜 지원 [#protocol-support]

eBPF 에이전트는 다음 프로토콜 모니터링도 지원합니다.

* HTTP

* 아파치 스리프트

* gRPC

* Cassandra

* 카프카

* AMQP

  <Callout variant="tip">
    Cassandra, Kafka 및 AMQP의 경우 eBPF 에이전트는 세그먼트 연결을 지원하지 않습니다.
  </Callout>

<Callout variant="tip">
  eBPF 모니터링은 언어에 구애받지 않으며 모든 구조에 대해 지연 시간 및 처리량 지표를 제공합니다. 하지만, [세그먼트 연결은](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm/#segments) 언어에 따라 다르며 현재 PHP 및 Node.js 에서는 지원되지 않습니다.
</Callout>

{/* ## Querying eBPF data [#querying-data]

Once the eBPF agent is deployed, telemetry data flows to New Relic and can be queried using NRQL. Here are common query patterns:

### Query spans

FROM Span

### Query metrics

Including TCP stats and performance metrics:

FROM Metric

<Callout variant="tip">

### Entity naming

The eBPF agent automatically generates entity names differently depending on the environment:

* **In hosts or Docker**: Names are a combination of the process name, its directory or container ID, and the listening port. For example, `ruby:/home/ubuntu/app:[5678]` or `java:f4aead533895:[8080]`.

* **In Kubernetes**: Names are derived from the service name, for example, `mysql-database-service`.

## Data access in New Relic [#data-access]

eBPF data is accessible through multiple New Relic interfaces:

1. **APM & Services UI**: Go to **[one.newrelic.com](https://one.newrelic.com) > APM & Services** and filter by `instrumentation.name = nr_ebpf`

2. **NRQL queries**: Use the query examples above to explore your data

3. **OpenTelemetry UI**: eBPF data integrates with New Relic's OpenTelemetry experience
