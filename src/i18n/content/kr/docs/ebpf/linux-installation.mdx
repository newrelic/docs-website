Linux 호스트에 뉴렐릭 eBPF 에이전트를 설치하여 전체 시스템 상태를 모니터링할 수 있습니다. eBPF 에이전트는 코드 변경이나 구현, 배포할 필요 없이 언어별 에이전트에 대한 심층적인 가시성을 제공합니다.

## 뉴렐릭 eBPF 에이전트 설치 [#install-linux]

<Steps>
  <Step>
    ### 시작하기 전에 [#requirements]

    Linux 호스트에 대한 [eBPF 호환성 및 요구 사항](/docs/ebpf/requirements#linux) 문서에 설명된 전제 조건을 충족해야 합니다.
  </Step>

  <Step>
    ### eBPF 에이전트 설치 [#install]

    eBPF 에이전트를 설치하려면:

    1. New Relic 계정에 로그인하세요.

    2. <DNT>**[left navigation pane &gt; + Integration &amp; Agents &gt; eBPF Agent](https://onenr.io/0oR8XWW36RG)**</DNT> 으)로 이동합니다.

    3. <DNT>Select an account</DNT> 화면에서 eBPF 에이전트를 설치할 계정을 선택하고 <DNT>**Continue**</DNT> 클릭합니다.

    4. <DNT>Select an installation method</DNT> 페이지에서 <DNT>**On a host**</DNT> 선택하고 <DNT>**Continue**</DNT> 클릭합니다.

    5. <DNT>Enter your user key</DNT> 화면에서 다음 옵션 중 하나를 선택한 다음 <DNT>**Continue**</DNT> 클릭하십시오.

       * <DNT>**Use an existing key**</DNT>이미 사용자 키가 있는 경우 사용자 키를 제공하십시오. 자세한 내용은 [사용자 키를](/docs/apis/intro-apis/new-relic-api-keys/#user-key) 참조하십시오.
       * <DNT>**Create a new key**</DNT>사용자 키가 없으면 <DNT>**Create a new key**</DNT> 클릭하여 생성하세요.

    6. <DNT>Configure the host integration</DNT> 화면에서 호스트에 대한 구현, 배포 이름을 입력한 다음 <DNT>**Continue**</DNT> 클릭합니다.

    7. <DNT>Install the agent</DNT> 화면에서 터미널에 제공된 명령어를 실행하여 Debian 패키지를 통해 eBPF 에이전트를 설치하십시오.

       1. 표시된 명령을 복사하여 붙여넣어 Debian 호스트에 eBPF 에이전트를 설치합니다.

       2. 설치를 확인하려면 다음 명령을 실행하세요.

          ```bash
          sudo systemctl status newrelic-ebpf-agent
          ```

       성공적으로 설치하면 이러한 서비스 상태가 활성(실행 중)으로 표시됩니다.

       <Callout variant="tip">
         <DNT>`/etc/newrelic-ebpf-agent/newrelic-ebpf-agent.yaml`</DNT> 에서 사용 가능한 <DNT>`newrelic-ebpf-agent.yaml`</DNT> 파일을 편집하여 eBPF 에이전트 설정을 사용자 정의할 수 있습니다. 설정 시위에 대한 자세한 내용은 [설정 시위를](#config-params) 참조하세요.
       </Callout>
  </Step>

  <Step>
    ### 뉴렐릭에서 eBPF 데이터에 액세스 [#access-data]

    eBPF 에이전트가 설치되면 Linux 호스트에서 자동으로 데이터를 수집하기 시작합니다. 이 데이터는 뉴렐릭의 OpenTelemetry UI 에서 액세스할 수 있습니다. 뉴렐릭 OpenTelemetry UI 에 대한 자세한 내용은 [OpenTelemetry APM UI](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-ui) 참조하세요.

    **뉴렐릭에서 eBPF 데이터를 보려면:**

    1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; APM &amp; Services**</DNT> 으)로 이동합니다.
    2. 검색 배너에서 검색 기준을 <DNT>`instrumentation.name = nr_ebpf`</DNT> 으로 설정합니다.
       <img style={{ align: 'left' }} title="eBPF filters" alt="eBPF filter for eBPF data in New Relic OpenTelemetry UI" src="/images/ebpf_filters.webp" />
  </Step>
</Steps>

## eBPF 에이전트 업그레이드 [#upgrade]

Linux 호스트에서 eBPF 에이전트를 업그레이드하려면 뉴렐릭 UI 에서 설치 명령을 다시 실행하세요. 이 프로세스를 실행하면 새로운 에이전트 버전이 설치되고, 이전 버전은 자동으로 제거됩니다.

<Callout variant="important">
  패키지 매니저를 통한 업그레이드는 지원되지 않습니다. 성공적인 업그레이드를 위해서는 뉴렐릭 UI 에서 설치 명령을 사용해야 합니다.
</Callout>

최신 설치 명령을 받으려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; + Integration &amp; Agents &gt; eBPF Agent**</DNT> 으)로 이동합니다.

2. 업데이트된 명령어를 얻으려면 설치 절차를 따르십시오.

3. 호스트에서 해당 명령어를 실행하여 에이전트를 업그레이드하십시오.

4. 아래 명령어를 사용하여 ebpf 에이전트 서비스를 다시 시작하십시오.

   ```bash
   sudo systemctl restart newrelic-ebpf-agent
   ```

## 구성 매개변수 [#config-params]

`newrelic-ebpf-agent.yaml` 파일에는 다음과 같은 설정 매개변수가 포함되어 있습니다.

<Callout variant="tip">
  eBPF 에이전트는 환경에 따라 다음과 같이 이름을 자동으로 다르게 생성합니다.

  * 호스트나 도커에서 이러한 이름은 프로세스 이름, 디렉토리 또는 컨테이너 ID, 수신 포트의 조합입니다. 예를 들어, `ruby:/home/ubuntu/app:[5678]` 또는 `java:f4aead533895:[8080]`.

  * Kubernetes에서 이러한 이름은 `mysql-database-service` 과 같은 서비스 이름에서 파생됩니다.

  애플리케이션에 사용자 정의 이름 지정:

  * Kubernetes 및 온호스트 제작 모두에 대해 `NEW_RELIC_APP_NAME` 환경 변수를 설정하여 맞춤 이름을 할당할 수 있습니다.
</Callout>

```bash
sudo apt-get purge newrelic-ebpf-agent
```
