New Relic 고객 지원팀에 문제를 보고할 때 로그 파일을 포함하는 것이 도움이 되는 경우가 많습니다. 로그 파일을 얻으려면 IDE에 대한 다음 지침을 따르세요.

## 제트브레인 [#jetbrains]

1. 문제를 재현합니다.
2. IDE에서 **Help > Collect Logs and Diagnostic Data** 으로 이동합니다. 그러면 새로 생성된 zip 파일이 표시되어야 하는 파인더가 열립니다.

## VS 코드 [#vscode]

1. 문제를 재현합니다.
2. VS Code에서 **출력** 보기를 열고( **보기: 명령 팔레트에서 출력을 토글** ) 오른쪽 상단의 드롭다운 메뉴에서 **CodeStream(에이전트)** 을 선택합니다. 표시되는 모든 출력을 복사하고 텍스트 파일에 저장합니다.

## 비주얼 스튜디오 [#visual-studio]

1. **Tools > Options > CodeStream** 으로 이동하고 로그 수준이 `Debug` 이상으로 설정되어 있는지 확인합니다. `Info` , `Errors` 또는 `Silent` 로 설정된 경우 `Debug` 로 변경한 다음 Visual Studio를 다시 시작합니다.
2. 문제를 재현합니다.
3. `%localappdata%\CodeStream\Logs` }에서 `vs-extension.log` 및 `vs-agent.log` 를 가져오고 &#x7B; `%AppData%\Microsoft\VisualStudio\16.0_<RandomText>` (Visual Studio 2019의 경우) 또는 `%AppData%\Microsoft\VisualStudio\17.0_<RandomText>` (Visual Studio 2022의 경우)에서 `ActivityLog.xml` 를 가져옵니다.
