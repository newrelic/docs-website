코드 블록이 포함된 메시지를 게시하면 다음 Git 관련 경고 중 하나가 표시될 수 있습니다.

## 저장소가 Git에서 관리되지 않거나 Git을 찾을 수 없습니다. [#not-found]

기술적으로 요구 사항은 아니지만 Code Stream 기능의 주요 요소는 Git 또는 GitHub와 같은 Git 호스팅 서비스에서 관리하는 리포지토리에 따라 달라집니다. Git을 사용하지 않는 경우 CodeStream은 생성한 주석과 문제를 소스 파일의 적절한 코드 블록에 연결할 수 없습니다.

이 오류는 Git이 PATH에 없음을 의미할 수도 있습니다. 그렇다면 PATH에 추가한 다음 IDE를 다시 시작하십시오.

## 저장소에 원격 URL이 없습니다. [#no-remote-url]

원격 URL은 CodeStream이 메시지의 코드 블록을 IDE의 해당 리포지토리에 있는 적절한 파일 및 위치에 연결할 수 있도록 하는 일부입니다. CodeStream에서 동료가 귀하가 댓글이나 문제를 생성한 저장소와 동일한 저장소를 열었는지 식별할 수 없으면 댓글이나 문제를 볼 수 없습니다.

명령줄에서 `git remote -v` 을 사용하여 리포지토리에 대해 구성된 원격 URL이 있는지 확인합니다. 원격 URL이 없으면 올바른 URL을 찾은 다음 `git remote add origin <remote URL>` 을 통해 원격 URL을 지정합니다.

## 저장되지 않은 파일 [#unsaved]

Git에서 관리하지 않는 파일과 마찬가지로 CodeStream은 저장되지 않은 새 파일의 코드 블록에 주석이나 문제를 연결할 수 없습니다. 첫 번째 댓글이나 문제를 만들기 전에 파일을 저장하세요.
