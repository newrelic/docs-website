JetBrains IDE를 다시 시작하거나 새 프로젝트를 열 때마다 New Relic CodeStream에 로그인해야 하는 경우 IDE가 로컬 암호 금고에서 인증 토큰을 저장 및/또는 검색하지 않을 수 있습니다.

[이 JetBrains 기사](https://www.jetbrains.com/help/idea/reference-ide-settings-password-safe.html) 는 올바르게 구성되었는지 확인하는 방법에 대한 세부 정보를 제공합니다.

예를 들어, Mac을 사용하는 경우 아래 스크린샷에 표시된 것처럼 <DNT>**in native Keychain**</DNT> 옵션을 사용하고 싶을 것입니다.

<img
  alt="Passwords"
  src="/images/codestream_screenshot-crop_jetbrains-password-keychain.webp"
/>

기본 키체인이 이미 선택되어 있는 경우 처음에 거부했던 IDE 키체인 액세스를 허용했을 수도 있습니다. 그렇다면 <DNT>**Keychain Access**</DNT>k 앱에 들어가서 수정하시면 됩니다.
