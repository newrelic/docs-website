New Relic CodeStream 확장 프로그램에 연결 문제가 발생하는 경우 네트워크 프록시 뒤에 있고 CodeStream이 함께 작동하도록 구성되지 않았기 때문일 수 있습니다. IDE에서 CodeStream의 설정을 찾으려면:

* VS Code:

  <DNT>
    **Settings**
  </DNT>

  으로 이동하여 CodeStream을 검색합니다.

* Visual Studio:

  <DNT>
    **Tools > Options > CodeStream**
  </DNT>

  으로 이동합니다.

* JetBrains:

  <DNT>
    **Settings/Preferences > Tools > CodeStream**
  </DNT>

  로 이동하세요.

<DNT>**Proxy Support**</DNT> 설정을 찾아 <DNT>**on**</DNT> 로 설정되어 있는지 확인하세요. CodeStream 확장 프로그램에서는 SSL/TLS 인증서가 합법적인 인증 기관에서 온 것으로 인식되지 않으므로 엄격한 SSL 검사도 비활성화해야 합니다.

VS Code에서 이러한 설정은 다음과 같습니다.

IDE에 프록시 지원이 구성된 경우 CodeStream은 먼저 해당 설정을 상속하려고 시도합니다. 그렇지 않으면 CodeStream은 운영 체제/환경에서 프록시 설정을 상속합니다.

설정을 변경한 후 IDE를 다시 시작하십시오.
