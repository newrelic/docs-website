CodeStream과 WSL(Linux용 Windows 하위 시스템)을 사용하려면 다음 단계를 따르세요.

1. Windows 컴퓨터와 WSL 배포판에 Git이 설치되어 있는지 확인하세요. 두 가지 간에 설치된 버전이 다를 수 있지만, WSL 배포판에 설치된 버전과 Windows 버전이 일치하는지 확인하는 것이 가장 좋습니다.

2. git 리포지터리를 WSL 배포판(예: `~/code/YOUR_PROJECT_FOLDER`)에 복제합니다.

3. Windows에서는 디렉토리를 `safe` 으로 표시하여 git이 이 UNC 경로를 통해 통신할 수 있도록 허용합니다. Windows 터미널에서 이 명령을 사용하여 `YOUR_DISTRO`, `YOUR_USERNAME`, `YOUR_PROJECT_FOLDER` 설정에 맞게 바꿔서 이 작업을 수행하세요.

   ```shell
   git config --global --add safe.directory '//wsl.localhost/YOUR_DISTRO/home/YOUR_USERNAME/code/YOUR_PROJECT_FOLDER' 
   ```

4. IDE에서 WSL UNC 경로를 사용하여 프로젝트를 엽니다. 예를 들어, Ubuntu를 사용하고 사용자 이름이 `codestream` 이고 프로젝트 폴더가 `Amazing_App` 인 경우 UNC 경로는 `//wsl.localhost/Ubuntu/home/codestream/code/Amazing_App` 가 됩니다.

<Callout variant="tip">
  Windows용 Git은 WSL에서 로컬로 복제된 리포지터리에 요청을 하는 데만 사용되며 원격(예: GitHub, GitLab 등)과 통신할 필요가 없습니다.
</Callout>

이러한 단계를 완료한 후에도 문제가 발생하면 [로그 파일을 수집](/docs/codestream/troubleshooting/client-logs) 하고 뉴렐릭 고객 지원팀에 문의하여 추가 조사를 받으세요.
