New Relic CodeStream을 팀의 Slack 채널에 연결합니다.

댓글, 문제 또는 피드백 요청을 게시하면 팀원들에게 활동 피드 및 잠재적으로 이메일을 통해 알림이 전송됩니다. 그러나 때로는 Slack에도 공유하고 싶을 수 있습니다. 이를 통해 아직 CodeStream에 가입하지 않았거나 IDE에서 많은 시간을 보내지 않은 사람들에게 다가갈 수 있습니다.

## Slack에 공유 [#share]

의견, 문제 또는 피드백 요청을 게시할 때 양식 하단에 있는 Slack 링크를 클릭하면 웹으로 이동하여 Slack 작업 공간 중 하나로 인증할 수 있습니다.

기본적으로 CodeStream은 사용자가 마지막으로 공유했는지 여부와 위치를 기억하고 다음 번에 양식은 기본적으로 동일한 선택 항목으로 설정됩니다. 현재 있는 리포지토리에 따라 사용할 기본 채널을 선택하려면 게시물 상단의 톱니바퀴 메뉴를 클릭하여 팝업한 다음 각 리포지토리의 기본 동작을 설정합니다.

팀원이 작성한 게시물을 포함하여 기존 게시물을 Slack에 공유할 수도 있습니다. 댓글, 문제 또는 피드백 요청으로 이동하여 줄임표 메뉴에서 **공유** 옵션을 찾으세요.

게시물이 공유된 위치를 추적 **하려면 공유** 대상 섹션을 찾으세요. 항목을 클릭하면 Slack에서 해당 대화로 바로 이동합니다.

## Slack에서 참여 [#participate]

Slack에 공유된 의견, 문제 또는 피드백 요청은 단순한 알림보다 훨씬 강력합니다. 팀원들은 Slack에서 바로 토론에 참여할 수 있습니다.

<img
  alt="Codemark on Slack"
  src={shareOnSlack1}
/>

### 토론 보기 및 답장 [#discussions]

팀원은 **토론 보기 및 회신** 을 클릭하여 회신을 추가할 수 있습니다. 해당 회신은 CodeStream에 게시됩니다. 전체 토론을 따라잡을 수 있도록 모든 이전 응답도 표시됩니다.

<img
  src={replyFromSlack}
  alt="Reply from Slack"
  title="Reply from Slack"
/>

Slack을 통해 토론에 참여하기 위해 팀원이 CodeStream 사용자일 필요는 없습니다.

### IDE에서 열기 [#open-idea]

IDE에서 **열기를 클릭하면 IDE** 내에서 바로 코드와 토론을 볼 수 있습니다. 먼저 어떤 IDE를 열 것인지 선택하는 CodeStream 웹 페이지로 이동합니다. CodeStream은 다음에 동일한 리포지토리에서 토론을 볼 때 선택 항목을 기억합니다. 그런 다음 IDE의 적절한 소스 파일로 이동하고 관련 코드 블록으로 스크롤되며 토론이 CodeStream 창에 표시됩니다. IDE에서 해당 리포지토리를 열지 않은 경우 CodeStream이 자동으로 소스 파일을 엽니다.

JetBrains IDE를 사용하는 경우 CodeStream이 IDE에 딥 링크할 수 있도록 [도구 상자 앱](https://www.jetbrains.com/toolbox-app/) 을 설치합니다.

### GitHub(또는 Bitbucket 또는 GitLab)에서 열기 [#open-github]

코드 블록이 GitHub, Bitbucket 또는 GitLab에서 호스팅되는 리포지토리에서 가져온 경우 이 버튼을 누르면 해당 호스팅 서비스의 해당 코드 블록으로 이동합니다.

## 코드마크 및 피드백 요청 해결 [#resolve]

코드마크 또는 피드백 요청이 해결, 승인, 거부 또는 다시 열리면 응답이 Slack의 해당 대화에 자동으로 게시됩니다.

<img
  alt="Resolved on Slack"
  src={resolvedOnSlack}
/>

많은 조직에는 Slack을 통해 토론에 참여하거나 최소한 수동적으로 모니터링하는 비개발자가 있으며, 이러한 알림은 토론이나 피드백 요청이 해결될 때 이러한 사람들을 루프 상태로 유지하는 데 도움이 됩니다.
