New Relic CodeStream을 팀의 Slack 작업 공간에 연결합니다.

Slack에 코드마크 또는 피드백 요청을 공유하는 것은 아직 CodeStream에 가입하지 않았거나 IDE에서 많은 시간을 보내지 않은 사람들에게 다가갈 수 있는 좋은 방법입니다. Slack에서 직접 토론에 참여할 수도 있습니다.

## Slack에 공유 [#share]

코드마크 또는 피드백 요청을 게시할 때 양식 하단에 있는 Slack 링크를 클릭하면 웹으로 이동하여 Slack 작업 공간 중 하나로 인증할 수 있습니다.

작업 공간에 연결되면 드롭다운 목록에서 공유할 채널을 선택합니다.

다이렉트 메시지로 공유할 수도 있습니다. 대화에 포함할 각 사람의 이름을 클릭합니다.

Slack에 공유할 때 CodeStream의 봇이 대화에 포함되며 이를 통해 플랫폼 간에 응답을 동기화할 수 있습니다. 다이렉트 메시지의 경우 선택된 참가자와 CodeStream의 봇을 포함하는 대화를 기술적으로 생성한다는 점에 유의하십시오. Slack의 다이렉트 메시지 목록에서 대화를 찾을 때 이 점을 염두에 두십시오.

기본적으로 CodeStream은 사용자가 마지막으로 공유한 위치를 기억하고 양식은 다음 번에 동일한 선택 항목으로 기본 설정됩니다. 선택한 코드 블록과 연결된 저장소를 기반으로 사용할 기본 채널을 설정하려면 상단의 톱니바퀴 메뉴를 클릭합니다.

이미 공유되지 않은 기존 게시물을 Slack에 공유할 수도 있습니다. 코드마크 또는 피드백 요청으로 이동하여 줄임표 메뉴에서 **공유** 옵션을 찾으세요. 코드마크 또는 피드백 요청 디스플레이의 **공유** 대상 섹션은 공유 여부와 공유 위치를 알려줍니다. 대화 이름을 클릭하여 Slack에서 엽니다.

## Slack에서 참여 [#participate]

Slack에 공유할 때 가장 좋은 점은 팀원들이 Slack에서 바로 대화에 참여할 수 있다는 것입니다. CodeStream 조직에 속하지 않은 팀원도 포함됩니다. 다른 Slack 게시물과 마찬가지로 "스레드에서 회신"을 클릭하여 회신을 추가하면 회신이 CodeStream에도 표시됩니다.

마찬가지로 CodeStream을 통해 추가된 응답은 자동으로 Slack의 스레드에 추가되어 예상되는 모든 알림을 트리거합니다. 이제 크로스 플랫폼 협업입니다!

### IDE에서 열기 [#open-idea]

**IDE에서 열기를** 클릭하면 IDE 내에서 코드와 토론을 모두 볼 수 있습니다. 먼저 열 IDE를 선택하는 CodeStream 웹 페이지로 이동합니다. CodeStream은 다음에 동일한 리포지토리에서 토론을 볼 때 선택 항목을 기억합니다. 그런 다음 IDE의 적절한 소스 파일로 이동하고 관련 코드 블록으로 스크롤되며 토론이 CodeStream 창에 표시됩니다. IDE에서 해당 리포지토리를 열지 않은 경우 CodeStream이 자동으로 소스 파일을 엽니다(이전에 해당 리포지토리를 열고 CodeStream이 설치된 위치에서 찾을 수 있도록 가정).

JetBrains IDE를 사용하는 경우 CodeStream이 IDE에 딥 링크할 수 있도록 [도구 상자 앱](https://www.jetbrains.com/toolbox-app/) 을 설치합니다.

### GitHub(또는 Bitbucket 또는 GitLab)에서 열기 [#open-github]

코드 블록이 GitHub, Bitbucket 또는 GitLab에서 호스팅되는 리포지토리에서 가져온 경우 이 버튼을 누르면 해당 호스팅 서비스의 해당 코드 블록으로 이동합니다.
