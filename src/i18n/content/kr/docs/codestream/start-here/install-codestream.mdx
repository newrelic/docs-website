New Relic CodeStream은 보다 자연스럽고 상황에 맞는 방식으로 코드를 쉽게 논의하고 검토할 수 있는 개발자 협업 플랫폼입니다. New Relic에 연결되면 IDE에서 직접 응용 프로그램 오류에 대해 공동 작업을 수행합니다.

아직 [가입하지 않았다면 New Relic CodeStream을 최대한 활용할 수 있도록 무료 New Relic 계정에](https://newrelic.com/signup) 가입하십시오.

## 도구 연결 [#connect-tools]

GitHub, GitLab 또는 Bitbucket에서 pull 요청을 만들고 검토합니다. Jira, Trello 및 기타 문제 추적기에서 문제를 생성합니다. New Relic에 보고된 오류를 조사하십시오. Slack 또는 Microsoft Teams에서 코드 토론을 공유하세요. CodeStream은 매일 사용하는 도구를 IDE에서 함께 제공합니다.

CodeStream 창 상단에서 얼굴 사진을 클릭한 다음 **통합** 을 클릭하여 모든 도구를 CodeStream에 연결합니다.

  title="Some of the tools you can connect to CodeStream, such as Bitbucket and GitHUb."

<Callout variant="tip">
  CodeStream을 설치한 후 New Relic에 연결하려면 New [Relic 사용자 키](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) 가 필요합니다. 여기로 이동 [하여 사용자 키를 찾거나 만드는 방법에 대해 자세히 알아보세요](/docs/apis/intro-apis/new-relic-api-keys/#user-api-key) .
</Callout>

다른 사람의 코드를 이해하려고 하거나 방금 작성한 코드에 대한 도움을 받으려는 경우 코드를 선택하고 댓글 버튼을 클릭하여 질문하십시오.

## 진행 중인 작업에 대한 피드백 받기 [#get-feedback]

**+** 메뉴를 클릭한 다음 개발 주기 중 언제든지 **피드백 요청** 을 클릭하세요. 진행 중인 일부 작업(커밋되지 않은 코드 포함)을 빠르게 살펴보거나 완료된 작업에 대한 공식 검토를 할 수 있습니다.

<img
  title="The Request Feedback highlighted in the + Create menu."
  alt="A screenshot of the Request Feedback highlighted in the + Create menu."
  src={qsFeedbackRequest2}
/>

팀원들은 분기를 전환하거나 자신의 작업을 따로 보관할 필요 없이 IDE에서 바로 변경 사항을 검토할 수 있습니다. [피드백 요청에 대해 자세히 알아보세요.](/docs/codestream/how-use-codestream/feedback-requests)

## 풀 리퀘스트 생성 또는 검토 [#pull-request]

CodeStream 사이드바에서 **풀 요청** 섹션을 찾아 열려 있는 풀 요청을 검토하십시오. 풀 요청을 선택(또는 URL에서 로드)하여 IDE에서 바로 완전한 GitHub 경험을 얻으십시오.

<img
  title="A GitHub pull request in CodeStream."
  alt="A screenshot of a GitHub pull request in CodeStream."
  src={qsPullRequest}
/>

GitHub, GitLab 또는 Bitbucket에서 pull 요청을 생성할 수 있지만 pull 요청 검토 지원은 현재 GitHub에서만 사용할 수 있습니다. [끌어오기 요청에 대해 자세히 알아보세요.](/docs/codestream/how-use-codestream/pull-requests/)
