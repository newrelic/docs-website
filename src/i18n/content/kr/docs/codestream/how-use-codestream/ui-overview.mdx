    ## 사이드바 개요 [#overview]

    사이드바 섹션에 대한 간략한 개요는 다음과 같습니다.

    * **[관찰 가능성](/docs/codestream/codestream-ui-overview/observability-section)** : New Relic에서 할당된 오류를 추적하고 IDE에서 연 리포지토리에서 최근 오류를 발견합니다.

    * **[끌어오기 요청](/docs/codestream/codestream-ui-overview/pull-requests-section)** : 팀에서 GitHub 또는 GitHub Enterprise를 사용하여 코드를 호스팅하는 경우 여기에 나열된 모든 열려 있는 끌어오기 요청을 볼 수 있습니다. 검토 또는 편집을 시작하려면 특정 풀 리퀘스트를 클릭하십시오.

    * **[피드백 요청](/docs/codestream/codestream-ui-overview/feedback-requests-section)** : 피드백 요청을 할당받았거나 다른 사람에게 피드백을 요청한 경우 여기에 나열된 항목을 찾으세요.

    * **[코드마크](/docs/codestream/codestream-ui-overview/codemarks-section)** : 코드마크는 코드베이스에 주석을 추가하는 토론입니다. 코드마크는 풀 요청, 피드백 요청 또는 임시 코드 주석/문제를 통해 생성됩니다. 현재 저장소의 모든 코드마크는 참조용으로 나열됩니다.

    * **[문제](/docs/codestream/codestream-ui-overview/issues-section)** : 여러 서비스에 걸쳐 있는 모든 미해결 문제를 한 곳에서 볼 수 있습니다. 특정 문제를 클릭하여 상태를 업데이트하고, 작업을 수행할 기능 분기를 만들고, Slack에서 상태를 업데이트합니다.

    * **[CI/CD](/docs/codestream/how-use-codestream/cicd)** : 현재 리포지토리 및 브랜치를 기반으로 빌드 상태를 확인합니다.

      CodeStream 사이드바는 완전히 사용자 지정할 수 있습니다.

    * 섹션을 드래그 앤 드롭하여 재정렬합니다.

    * 섹션 구분선을 클릭하고 끌어 양쪽 섹션의 크기를 조정합니다.

    * 필요에 따라 각 섹션을 확장하거나 축소합니다.

    * 각 섹션의 최대화 버튼을 클릭하여 전체 화면을 채웁니다. 이것은 더 긴 목록을 볼 때 유용합니다. 다시 클릭하면 이전 보기로 돌아갑니다.

      ## 사용자 이름 메뉴 [#username]

      <img
        title="The username menu items."
        alt="A screenshot of the username menu items."
        src={codestreamUserMenu}
      />

      <figcaption>
        사용자 이름 메뉴는 계정, 조직, 알림 수신 방법 및 표시되는 섹션을 관리하는 옵션을 제공합니다.
      </figcaption>

      다음은 각 메뉴 항목에 대한 설명입니다.

    * **계정** : 프로필을 봅니다. 프로필 사진, 이메일, 사용자 이름 및 계정의 전체 이름을 사용자 지정하기 위한 다양한 옵션이 포함되어 있습니다.

    * **보기** : 보고 싶지 않은 섹션을 선택 취소합니다.

    * **알림** : 알림을 받는 방법과 내용을 관리합니다.

    * **조직 관리자** : 조직 설정을 관리합니다. 또한 데이터를 내보내고 조직을 삭제합니다.

    * **조직 전환** : 둘 이상의 조직에 속한 경우 사용합니다.

    * **통합** : CodeStream을 사용하는 코드 호스트, 문제 및 메시징 공급자에 연결합니다.

    * **New Relic 설정** : New Relic 계정을 CodeStream에 연결하여 New Relic CodeStream의 관찰 도구를 최대한 활용하십시오.

    * **피드백** : 피드백이 있습니까? GitHub 문제를 작성하십시오.

    * **도움말** : 문서, 비디오 라이브러리, 키 바인딩, CodeStream 워크플로, 새로운 기능 및 문제 보고에 대한 링크입니다.

      ## 헤더 메뉴 항목 [#header-menu]

      <img
        title="The CodeStream header menu."
        alt="A screenshot of the CodeStream header menu."
        src={codestreamGlobalOptionsButtons}
      />

      <figcaption>
        헤더 메뉴 항목은 조직에서 콘텐츠를 만들고 검색하기 위한 다양한 옵션을 제공합니다. 왼쪽에서 오른쪽으로 **만들기** , **활동 피드** , **내 조직** , **필터 및 검색** .
      </figcaption>

    * **+** (작성): 클릭하여 코드 주석/문제를 생성하거나 변경 사항에 대한 피드백을 요청하거나 풀 요청을 생성합니다.

    * **[활동 피드](/docs/codestream/codestream-ui-overview/activity-feed)** : 활동 피드는 새로운 코드 주석/문제 및 피드백 요청에 대해 알려줄 뿐만 아니라 기존에 대한 응답도 알려줍니다.

    * **[내 조직](/docs/codestream/codestream-ui-overview/myorganization)** : CodeStream 조직에 누가 있는지 확인하고, 새 구성원을 초대하고, 비난 지도를 만듭니다.

    * **[필터 및 검색](/docs/codestream/codestream-ui-overview/filter-and-search)** : 필터 검색 도구를 사용하면 팀의 코드 주석, 문제 및 기능 요청 모음을 원하는 대로 분할할 수 있습니다.
  </Collapser>

  <Collapser
    className="freq-link"
    id="ui-activity"
    title="활동 피드"
  >
    활동 피드는 팀 동료가 게시한 새 댓글, 문제 및 피드백 요청이나 기존 토론에 대한 새 답글에 대해 알아볼 수 있는 확실한 장소입니다. 활동 피드에 있지 않을 때 활동 피드 아이콘에서 배지를 찾아 새로운 것이 있는지 여부를 항상 알 수 있습니다. 흰색 점이 있는 파란색 배지는 새로운 토론이나 응답이 있음을 의미합니다.

    <img
      title="badge"
      alt="This image shows an activity badge on an icon."
      src={codestreamActivityFeedBadgeIcon}
    />

    <figcaption>
      내부에 숫자가 있는 파란색 배지는 내가 언급된 곳에 새로운 토론이나 답장이 있음을 의미합니다.
    </figcaption>

    IDE의 상태 표시줄에 있는 CodeStream 섹션은 또한 피드에 새 메시지가 있을 때 이를 알려줍니다. 사용자 이름 오른쪽에 있는 점은 새 메시지가 있음을 의미하고 숫자는 새 멘션이 있음을 의미합니다.

    <img
      title="ide-status"
      alt="This image shows how CodeStream appears in your IDE's status bar"
      src={codestreamStatusBarMentions}
    />

    활동 피드 상단의 필터는 피드에 표시되는 내용을 제어합니다.

    * **내 조직** 의 모든 사용자의 활동: 조직의 모든 활동을 봅니다. 소규모 조직에만 권장합니다.

    * **내 IDE에서 열린 코드와 관련된 활동(기본값)** : IDE에서 연 리포지토리의 코드와 관련된 토론을 참조하세요.

    * **선택한 폴더의 코드와 관련된 활동** : 팀이 monorepos 내에서 작업하는 경우 이 옵션을 사용합니다. 현재 선택된 폴더의 코드와 관련된 토론을 보십시오.

      필터 설정에 관계없이 @멘션되면 항상 알림을 받습니다.
  </Collapser>

    당사의 필터 및 검색 도구를 사용하면 조직의 의견, 문제 및 피드백 요청 모음을 적절하다고 생각하는 대로 분할할 수 있습니다. 기본적으로 귀하에게 할당된 모든 미해결 문제 또는 피드백 요청을 볼 수 있으며 그 뒤에 모든 미해결 문제 및 피드백 요청이 표시됩니다.

    <img
      title="filter"
      alt="A screenshot showing the filter and search tool."
      src={codestreamFilterandSearchBar}
    />

    필터 및 검색 도구를 사용하여 코드스트림 구성을 선택하고 특정 댓글, 문제 또는 피드백 요청을 찾습니다.

    * 자세한 내용을 보려면 목록의 항목 위로 마우스를 가져갑니다.

    * 사전 정의된 강력한 필터의 이점을 활용하여 작업을 쉽게 만드십시오.

    * 고급 검색 구문을 사용하여 자신만의 검색을 만들고 검색 상자 오른쪽에 있는 책갈피 아이콘을 통해 나중에 사용할 수 있도록 저장할 수 있습니다.

    다른 사람이 작성했거나 다른 사람이 수정한 코드로 작업하는 것이 일상적인 일이라는 것은 거의 불가피합니다. CodeStream은 편집기의 각 코드 줄 끝에 비난 정보를 표시하므로 해당 코드를 마지막으로 건드린 사람을 쉽게 찾을 수 있습니다.

    <img
      title="blame-hovers"
      alt="This image shows a hover with blame information for the current line of code"
      src={codestreamBlameHoverExample}
    />

    인라인 주석 위로 마우스를 가져가면 작성자, 커밋, 관련 pull 요청 및 피드백 요청에 대한 자세한 정보를 볼 수 있습니다. 또한 주어진 코드 줄에 대한 토론이나 버그 보고를 신속하게 시작할 수 있습니다.

    GitLens 또는 GitToolBox 확장이 설치된 경우 비난 정보가 표시되지 않습니다. IDE 설정의 CodeStream 섹션에서 비난 호버를 끌 수도 있습니다.
