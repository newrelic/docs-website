개발자가 개발 프로세스에서 더 일찍, 더 자주 코드에 관해 이야기하도록 하면 팀에 큰 결과를 가져올 수 있습니다. 대화는 더 나은 성능과 더 높은 품질의 코드를 제공하고 개발자를 더 행복하게 만듭니다. New Relic CodeStream을 사용하면 코드에 대한 논의가 그 어느 때보다 쉬워졌습니다.

질문, 제안, 문서, 버그 신고 등 코드를 선택하고 **Add comment** \[댓글 추가] 또는 **Create issue** \[이슈 생성을] 클릭하고 선택적으로 Slack이나 Teams에 공유하세요. 이러한 모든 토론은 나중에 활용할 수 있도록 참조하는 코드 블록에 저장되고 고정됩니다. 팀에 합류하는 새로운 개발자일 수도 있고, 다른 사람의 코드에 있는 버그를 수정하려는 개발자일 수도 있고, 심지어 6개월 전에 변경한 이유를 기억하려는 여러분일 수도 있습니다. 어떤 경우이든 CodeStream은 상황에 맞는 토론을 제시하여 코드를 이해하는 데 도움을 줍니다.

시간이 지남에 따라 파일이 변경되더라도 주석은 코드에 연결된 상태로 유지됩니다. 코드 블록 위에 새로운 코드 줄을 추가하거나, 코드를 편집하거나, 전체 블록을 잘라내어 파일의 다른 섹션에 붙여넣으면 변경 사항에 따라 주석이 움직이는 것을 볼 수 있습니다.

## 댓글 또는 이슈 생성 [#create]

편집기에서 코드 블록을 선택한 다음 선택 항목 옆에 있는 CodeStream 창에서 **Add comment** \[주석 추가] 또는 **Create issue** \[문제 생성] 아이콘을 클릭합니다.

IntelliJ와 같은 JetBrains IDE를 사용하는 경우 코드 블록을 선택할 때 편집기 여백에 나타나는 **+** 버튼을 클릭하세요.

CodeStream 창이 닫혀 있거나 보기에 없는 경우에도 전구 또는 상황에 맞는 메뉴의 CodeStream 옵션을 통해 코드에 대해 논의할 수 있습니다. CodeStream 창 상단에서 **+** 메뉴를 찾을 수도 있습니다.

## Slack 또는 Microsoft Teams에 공유 [#sharing]

CodeStream에는 [Slack 통합](/docs/codestream/code-discussion/slack-integration) 과 [Microsoft Teams 통합이](/docs/codestream/code-discussion/msteams-integration) 모두 포함되어 있어 IDE에서 많은 시간을 보내지 않거나 아직 CodeStream을 사용하지 않는 팀원에게 쉽게 접근할 수 있습니다. Slack 통합을 사용하면 팀원이 Slack에서 직접 응답할 수도 있습니다.

## 이슈 생성 [#issues]

Slack 및 Teams와의 공유 외에도 CodeStream은 다양한 문제 추적 서비스와도 통합됩니다. 팀에서 Asana, Azure DevOps, Bitbucket(클라우드), GitHub(클라우드 또는 온프레미스), GitLab(클라우드 또는 자체 관리형), Jira(클라우드 또는 서버), Linear, Shortcut, Trello 또는 YouTrack(클라우드)을 사용하는 경우 문제 추적을 위해 CodeStream에서 직접 해당 서비스 중 하나에 대한 문제를 생성할 수 있습니다. 이슈 생성 시 양식 상단의 드롭다운에서 사용하는 서비스를 선택하세요.

## 적절한 사람들을 토론에 참여시키십시오. [#mentions]

댓글이나 문제를 생성하면 CodeStream은 댓글을 달고 있는 코드를 가장 최근에 터치한 사람들을 자동으로 언급합니다. 귀하의 질문에 답변할 수 있는 가장 적합한 사람들일 수 있지만, 물론 해당 언급을 제거하고 적절한 경우 다른 사람을 수동으로 언급할 수도 있습니다.

## 다른 버전의 코드로 작업 [#code-versions]

어쩌면 기능 분기에 있거나 로컬 변경 사항이 있거나 단순히 한동안 작업하지 않았을 수도 있습니다. 당신이 보고 있는 코드가 팀원이 보고 있는 코드와 다를 수 있는 이유는 셀 수 없이 많습니다. 결과적으로 토론에서 참조된 코드가 로컬에 있는 코드와 일치하지 않는 경우가 많습니다.

CodeStream은 이러한 상황을 인식하고 코드 블록의 원본 버전, 현재 버전 및 차이점을 포함합니다.

<img
  title="version diff"
  alt="A screenshot showing a code diff."
  src={codestreamCodemarkWithCodeDiff}
/>

CodeStream을 사용하면 편집기에 방금 입력하고 아직 커밋하지 않은 코드라도 언제든지 소스 파일의 모든 코드 행에 대해 논의할 수 있습니다. CodeStream을 사용하면 개발 프로세스의 가장 초기 단계에서 코드에 대해 논의할 수 있습니다.
