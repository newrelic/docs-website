New Relic CodeStream을 사용하면 개발자가 IDE에 관찰 가능성을 가져오고 일상의 일부로 만들어 자신이 담당하는 코드가 프로덕션 환경에서 어떻게 수행되고 있는지 확인할 수 있습니다.

CodeStream 에 표시되는 옵저버빌리티 데이터는 IDE에서 열려 있는 현재와 관련이 있습니다. 해당 사용자가 New Relic의 서비스와 연결되어 있지 않은 경우, 귀하는 [해당 연결을 만드는](/docs/codestream/observability/repo-association) 역할을 맡게 됩니다. 그렇지 않으면 코드에서 구축된 서비스가 어떻게 수행되고 있는지에 대한 포괄적인 보기를 얻게 됩니다.

각 서비스에 대해 다음 성능 데이터가 표시될 수 있습니다.

* <DNT>
    **Golden metrics**
  </DNT>

  : 서비스가 어떻게 수행되고 있는지를 나타내는 주요 신호입니다. 지표는 서비스 유형(APM, 브라우저, 모바일, OTel, Lambda 기술)에 따라 다릅니다.

* <DNT>
    **Issues**
  </DNT>

  : 해당 서비스에 대해 구성된 공지 조건을 기반으로 하는 모든 미해결 문제(APM 에만 해당).

* <DNT>
    **Service-level objectives**
  </DNT>

  : 구성된 서비스 수준 목표(SLO)(APM, 브라우저, OTel)에 대해 서비스가 어떻게 수행되는지에 대한 요약입니다.

* <DNT>
    **Transaction performance**
  </DNT>

  : 마지막 릴리스 이후 어떤 트랜잭션에서 성능 문제가 발생하는지 확인하세요. [트랜잭션 성능에 대해 자세히 알아보세요](/docs/codestream/observability/transactions) (APM만 해당).

* <DNT>
    **Errors**
  </DNT>

  : 귀하가 담당하는 코드에서 최근 발생한 오류입니다. 오류를 클릭하면 스택 추적을 단계별로 진행하고 팀원과 조사에 협력할 수 있습니다. [오류 조사(APM, 브라우저)에 대해 자세히 알아보세요](/docs/codestream/observability/error-investigation) .

* <DNT>
    **Vulnerabilities**
  </DNT>

  : 교정 방법에 대한 정보를 포함한 패키지 수준의 교육입니다(APM 만 해당).

* <DNT>
    **Related services**
  </DNT>

  : 서비스에서 호출하거나 호출하는 모든 서비스(각 서비스에 대한 골든 메트릭을 볼 수 있는 기능 포함) 귀하의 서비스가 더 큰 그림(APM, 브라우저, 모바일, OTel)에 어떻게 부합하는지 이해하십시오.

* <DNT>
    **Logs**
  </DNT>

  : 컨텍스트 전환을 피하기 위해 IDE에서 바로 로그를 검색합니다. [로그 검색(APM, OTel)에 대해 자세히 알아보세요](/docs/codestream/observability/log-search) .

트리에서 서비스 이름 위로 마우스를 가져가면 지구본을 클릭하여 New Relic의 서비스 요약 페이지를 엽니다.

CodeStream에서 옵저빌리버티 기능을 사용하려면 New Relic 사용자 유형의 [핵심 사용자 또는 전체 플랫폼이](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/#user-type-capabilities) 있거나 New Relic 컴퓨팅 가격 모델을 사용해야 합니다.
