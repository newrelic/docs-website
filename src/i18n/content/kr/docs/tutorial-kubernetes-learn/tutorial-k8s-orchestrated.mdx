오케스트레이션된 구성 요소는 필요에 따라 포드를 위아래로 회전시키는 전체 배포로 구성됩니다. Kubernetes 시스템에서 개별 배포에는 특정 구성이 있습니다. 예를 들어 배포는 개별 컨테이너에서 애플리케이션의 4개 인스턴스를 실행할 수 있습니다. 배포는 해당 할당량을 충족할 때까지 해당 애플리케이션이 포함된 포드를 가동합니다. 포드가 실패하면 지정된 숫자를 계속 충족하기 위해 새 포드를 가동합니다.

수십 또는 수백 개의 배포가 있고 각각의 구성이 기억나지 않을 수도 있는 경우 문제가 발생합니다.

다음 단계는 배포를 모니터링하는 일반적인 전략을 안내합니다.

<Steps>
  <Step>
    ## 개요 대시보드로 이동

    **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Kubernetes > Overview Dashboard** 로 이동합니다.
  </Step>

  <Step>
    ## 배포 분류

    비정상 배포 타일과 배포 횟수 타일을 살펴보세요.

    두 가지를 비교하여 클러스터의 비정상 배포 비율을 측정하세요. 비정상 배포에는 포드가 없거나 사용할 수 없습니다. 이는 일반적으로 배포가 해당 Pod를 스핀업할 수 없음을 의미합니다.

    자세히 살펴보겠습니다.

    <img
      title="tutorialOverviewdashboard"
      alt="The main overview dashboard for the kubernetes capability"
      src="/images/tutorial_screenshot-full_overview-deployments.webp"
    />
  </Step>

  <Step>
    ### 보류 중인 포드와 실패한 포드 식별

    조금 아래로 스크롤하여 **Pending and Failed Pods** 테이블을 찾습니다. 이 표에는 어떤 이유로든 실패했거나 보류 중인 모든 포드가 표시됩니다. 시스템의 기본 상태에 따라 Pod가 어느 정도 실패하는 것은 정상입니다.

    당신이 찾고 있는 것은 반복적으로 실패하는 포드입니다. 필요에 따라 차트를 아래로 스크롤합니다.

    <img
      title="tutorialOverviewFailed"
      alt="The main overview dashboard for the kubernetes capability focusing on the failed pods section"
      src="/images/tutorial_screenshot-crop_failed-pods.webp"
    />

    정기적으로 실패하는 Pod를 식별한 후에는 해당 Pod에 대한 배포 구성 문제를 해결하세요.
  </Step>
</Steps>
