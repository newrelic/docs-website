APM에서는 사용자 지정 수준의 모니터링을 원하거나 자체 [경고 정책](/docs/apm/transactions/key-transactions/view-key-transactions-alert-information) 을 원할 때 트랜잭션을 [키 트랜잭션](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events) 으로 만들 수 있습니다.

대부분의 New Relic 에이전트의 경우 키 트랜잭션은 모든 유형의 트랜잭션이 될 수 있습니다. PHP, Node.js, Go 에이전트의 경우 **웹 트랜잭션** 만 키 트랜잭션으로 전환될 수 있습니다.

## 주요 거래 생성 [#create]

주요 거래를 생성하려면:

1. **[one.newrelic.com](https://one.newrelic.com) > More > Key transactions** 로 이동하여 [주요 거래 색인](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events#find) 을 확인하십시오.
2. **추가** 를 선택한 다음 애플리케이션 및 트랜잭션을 선택합니다.

## 키 트랜잭션 이름 바꾸기 [#rename]

이름을 바꾸는 방법은 다음과 같습니다.

1. **[one.newrelic.com](https://one.newrelic.com) > More > Key transactions** 로 이동합니다.

2. 인덱스에서 기어를 선택합니다.

   <Icon name="fe-settings"/>

   이름을 바꾸려는 키 트랜잭션의 아이콘입니다.

3. 트랜잭션의 새 이름을 입력한 다음 저장합니다.

## Apdex 설정 변경 [#apdex]

[웹 트랜잭션](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#transaction) 인 경우 키 트랜잭션에 대한 사용자 지정 [Apdex 임계값](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) 을 설정할 수 있습니다. (Apdex는 [웹이 아닌 트랜잭션](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#non-web-transaction) 에는 적용되지 않습니다.) 이는 트랜잭션이 다른 트랜잭션보다 빠르거나 느릴 것으로 예상되는 경우에 유용합니다.

<Callout variant="important">
  주요 트랜잭션 Apdex 설정은 해당 앱에 대해 설정된 T 값을 재정의합니다. 예를 들어, 주요 트랜잭션은 **만족** 으로 보고될 수 있지만 응답 시간이 동일한 동일한 앱의 다른 트랜잭션은 **좌절** 될 수 있음을 의미할 수 있습니다.
</Callout>

주요 트랜잭션의 Apdex 임계값을 변경하려면:

1. **[one.newrelic.com](https://one.newrelic.com) > More > Key transactions** 로 이동합니다.

2. 선택

   <Icon name="fe-settings"/>

   편집하려는 거래의 아이콘입니다.

3. **Apdex T** 제목 아래에 새 값(초)을 입력한 다음 저장합니다.

Apdex 임계값은 [주요 트랜잭션 추적](/docs/apm/transactions/transaction-traces/configure-transaction-traces#key-txns) 이 생성되는 방식에 영향을 줍니다.

## 주요 거래 삭제 [#delete]

주요 거래를 삭제하려면:

1. **[one.newrelic.com](https://one.newrelic.com) > More > Key transactions** 로 이동합니다.

2. 기어 선택

   <Icon name="fe-settings"/>

   삭제하려는 주요 거래의 아이콘입니다.

3. **추적 중지를** 선택합니다.

New Relic은 삭제된 키 트랜잭션과 관련된 모든 경고 및 이벤트를 자동으로 삭제합니다.
