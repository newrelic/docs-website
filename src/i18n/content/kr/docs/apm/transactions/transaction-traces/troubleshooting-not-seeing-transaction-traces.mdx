## 문제

APM UI에 트랜잭션 추적이 표시되지 않습니다.

## 해결책

[APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm) 에서 [트랜잭션 추적](/docs/apm/transactions/transaction-traces/transaction-traces) 이 표시되지 않는 경우 몇 가지 이유가 있을 수 있습니다.

<CollapserGroup>
  <Collapser
    id="disabled"
    title="거래 추적이 비활성화되었습니다."
  >
    트랜잭션 추적은 기본적으로 활성화되어 있지만 설정이 수동으로 비활성화되었을 수 있습니다. 이 문제를 해결하려면 트랜잭션 추적 설정을 활성화하도록 설정을 다시 변경하십시오.
  </Collapser>

  <Collapser
    id="criteria"
    title="거래가 추적 기준을 충족하지 않습니다."
  >
    거래가 [추적 기준을](/docs/apm/transactions/transaction-traces/transaction-traces#basics) 충족하지 않습니다. 예를 들어:

    * 4배 Apdex 임계값 또는 사용자 지정 초 수 임계값보다 더 빠르게 응답합니다.
    * 1분이라는 긴 수확 주기 동안 가장 느린 추적으로 선택되지 않습니다.
  </Collapser>

  <Collapser
    id="app"
    title="앱 활동이 트랜잭션으로 캡처되지 않습니다."
  >
    APM [트랜잭션 페이지](/docs/apm/applications-menu/monitoring/transactions-page)에 나열된 트랜잭션이 표시되지 않으면 APM 에이전트가 활동을 트랜잭션으로 캡처하지 않는 것입니다. 이 상황에서는 작업을 트랜잭션으로 모니터링하도록 [사용자 정의 계측을](/docs/apm/agents/manage-apm-agents/agent-data/custom-instrumentation/) 설정해야 합니다.
  </Collapser>

  <Collapser
    id="brackets"
    title="사용자 지정 트랜잭션 이름에는 대괄호가 포함됩니다."
  >
    [사용자 정의 계측을](/docs/apm/agents/manage-apm-agents/agent-data/custom-instrumentation/) 사용하여 트랜잭션을 수동으로 생성한 경우 New Relic의 명명 규칙을 따라야 합니다.

    <Callout variant="important">
      거래 이름 끝에 대괄호 `[suffix]` 를 사용하지 마십시오. New Relic은 자동으로 이름에서 괄호를 제거합니다. 대신 필요한 경우 괄호 `(suffix)` 또는 기타 기호를 사용하십시오.
    </Callout>
  </Collapser>

  <Collapser
    id="no-traces"
    title="모든 거래에 대한 추적이 표시되지 않습니다."
  >
    트랜잭션이 임계값을 확실히 초과하도록 **임계** 값을 일시적으로 `0.0001` 으로 변경합니다. 트랜잭션 추적 수신을 시작한 후에는 이를 원래 설정으로 되돌려야 합니다.
  </Collapser>
</CollapserGroup>
