APM에서 <DNT>**Databases**</DNT> 페이지에는 애플리케이션의 데이터베이스와 캐시 데이터가 표시됩니다. 페이지에는 개별 데이터베이스 트랜잭션이 정렬 가능한 테이블로 표시되고 작업, 처리량 및 응답 시간이 차트로 표시됩니다. [느린 쿼리](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) 가 보고된 경우 트랜잭션을 선택할 때 표시됩니다.

데이터베이스 정보를 세 부분으로 나눕니다.

* 모든 데이터베이스 개요(메인 페이지)

  * 실제 시간 기준 상위 5개 데이터베이스 작업

  * 데이터베이스 응답 시간

  * 데이터베이스 처리량(분당 작업 요청 또는

    <DNT>
      **rpm**
    </DNT>

    )

* 상위 데이터베이스 작업(하위 페이지)

  * 쿼리 시간
  * 처리량
  * 발신자의 시간 소비

* 모든 데이터베이스 작업이 포함된 테이블(하위 페이지)

## 데이터베이스 공급업체별 필터링 작업 [#db_filtering]

사용 가능한 경우: 데이터베이스 작업 목록 위에 있는 버튼을 사용하여 특정 데이터베이스 공급업체로 [필터링](#availablity) 할 수 있습니다.

* 데이터베이스 공급업체별로 콘텐츠를 필터링하려면 해당 이름을 선택합니다.

* 필터를 제거하려면

  <DNT>
    **All**
  </DNT>

  선택합니다.

## 상위 데이터베이스 작업 검토 [#top_db_operations]

기본 <DNT>**Databases**</DNT> 페이지에는 다양한 방법으로 정렬할 수 있는 주요 데이터베이스 작업 목록이 표시됩니다.

작업에 대한 세부 정보를 보려면 레코드 위로 마우스를 이동하거나 클릭하여 추가 차트를 볼 수 있습니다.

기록을 클릭하면 <DNT>**Time consumption by caller**</DNT> 차트를 볼 수 있습니다. 그러면 선택한 작업을 호출하는 트랜잭션 목록이 표시되며, 각 호출자가 소비한 시간의 백분율을 나타내는 색상 막대가 표시됩니다. 호출자 트랜잭션에 대한 <DNT>**Transactions**</DNT> 페이지를 보려면 차트에서 해당 트랜잭션에 대한 색상 막대를 선택하세요.

## 모든 데이터베이스 작업 표시 [#db_displaying]

[사용 가능한 경우](#availability) 모든 데이터베이스 작업을 표시하려면 다음을 수행합니다.

* 작업 목록 하단에서

  <DNT>
    **Show all database operations table...**
  </DNT>

  링크를 선택합니다.

* 열을 기준으로 테이블을 정렬하려면 열 머리글을 선택합니다.

* 작업 목록을 필터링하려면 표 상단의 검색 상자를 사용하십시오.

* 특정 작업에 대한 세부 정보를 보려면 작업 이름을 선택합니다.

## 흔적 삭제 [#db_deleting]

앱의 모든 추적을 영구적으로 삭제하려면 다음 단계를 따르세요.

1. 데이터베이스 작업 목록 아래에서

   <DNT>
     **Delete all traces**
   </DNT>

   버튼을 선택합니다.

2. 확인 프롬프트를 선택합니다.

<Callout variant="tip">
  일반 레코드 정리를 수행하는 경우 [트랜잭션 추적](/docs/traces/deleting-transaction-traces) 또는 [오류 추적](/docs/apm/applications-menu/events/viewing-apm-errors-error-traces) 을 삭제할 수도 있습니다. 이렇게 하면 다른 애플리케이션 데이터(예: Apdex, 배포 정보 등)를 유지하면서 잠재적으로 민감한 데이터를 제거할 수 있습니다.
</Callout>

## 에이전트 버전에 따른 추가 기능 [#availablity]

<DNT>**Databases**</DNT> 페이지의 추가 기능은 다음 에이전트 버전에서 사용할 수 있습니다. 자세한 내용은 [뉴렐릭 에이전트 업데이트를](/docs/agents/manage-apm-agents/installation/update-new-relic-agent) 참조하세요.

* 1.4 이상으로 이동
* 자바 3.14.0 이상
* .NET 4.1.134 이상
* Node.js 1.17.0 이상
* PHP 4.19.0 이상
* 파이썬 2.44.0 이상
* 루비 3.11.0 이상

이러한 기능은 다음과 같습니다.

* 여러 데이터베이스 및 캐시 유형
* 작업 목록, 차트 및 느린 쿼리의 다양한 유형에 대한 레이블
* 특정 데이터베이스 공급업체 또는 캐시 유형에 대한 필터링 작업 및 차트
* 모든 데이터베이스 작업 테이블 보기
* [서비스 맵](/docs/apm/applications-menu/monitoring/service-maps) 에서 데이터베이스를 보는 기능

또한 APM 에이전트가 [호스트 및 인스턴스 수준 세부 정보](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues) 를 지원하는 경우 쿼리 세부 정보와 쿼리가 실행된 특정 인스턴스를 모두 볼 수 있습니다.
