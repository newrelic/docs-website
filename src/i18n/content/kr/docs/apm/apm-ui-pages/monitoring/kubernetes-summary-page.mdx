## Kubernetes 클러스터 배포 모니터링 [#monitor-kubernetes]

APM의 Kubernetes 페이지는 애플리케이션에 대한 다음 정보를 제공합니다.

<CollapserGroup>
  <Collapser id="deployment-summary" title="배포 요약">
    클러스터 이름, 배포 및 실행 중인 네임스페이스를 볼 수 있습니다. 배포를 클릭하면 슬라이드 아웃 패널에서 배포에 대한 세부 정보를 볼 수 있습니다. 슬라이드 아웃 패널에서는 지정된 시간 범위 내의 로그 수, 오류 로그 비율 등의 관련 정보가 포함된 간단한 개요를 볼 수 있습니다.

    <img title="Kubernetes page - Mini overview" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-apm-mini-overview.webp" />

    <DNT>**Instances**</DNT> 드롭다운에서 서비스의 단일 파드 제외를 선택할 수 있습니다.

    <img title="Kubernetes page - Deployment summary" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-header.webp" />

    멀티 클러스터 앱(여러 클러스터에 걸쳐 있는 앱)이 있는 경우 <DNT>**Kubernetes**</DNT> 페이지에 모든 클러스터 요약이 표시됩니다. 특정 클러스터를 분리하려면 <DNT>**Cluster**</DNT> 드롭다운 목록에서 선택하세요. 모든 클러스터 보기로 돌아가려면 <DNT>**All clusters**</DNT> 클릭하세요.

    <img title="Kubernetes page - Current deployment" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-current-deploy.webp" />

    <img width="70%;" title="Kubernetes page - Performance summary" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-performance.webp" />
  </Collapser>

  <Collapser id="pods-containers" title="포드 및 컨테이너">

    또한, <DNT>**Logs**</DNT> 버튼을 사용하면 해당 패드에 대한 컨텍스트 내 로그인을 볼 수 있습니다.

    * CPU 사용량(코어)
    * CPU 조절(%)
    * 메모리 사용량(바이트)
    * 수신된 네트워크(KBps)
    * 전송된 네트워크(KBps)
    * 네트워크 오류(초당)
    * 스토리지 사용량(바이트)
    * 스토리지 활용도(%)
    * 컨테이너 재시작 횟수
    * 서비스 처리량(분당 요청)
    * 서비스 응답 시간(ms)
    * 서비스 오류율(%)

## Kubernetes에 애플리케이션 연결 [#link-app]

당사의 독점 에이전트나 OpenTelemetry를 사용하든, Kubernetes 메타데이터를 표면화하고 이를 APM 에이전트에 연결할 수 있습니다. <InlinePopover type="apm" />는 Kubernetes 메타데이터를 사용하여 해당 APM 애플리케이션과 관련된 특정 Kubernetes 배포 정보를 표시합니다.

### 뉴렐릭 독점분석 [#nr-proprietary]

[뉴렐릭의 Kubernetes 통합을 설치](/install/kubernetes/) 하면 뉴렐릭의 감지 데이터 표시가 `nri-metadata-injection` 매개변수를 통해 자동으로 작동됩니다. Kubernetes 시스템에서 설정한 `MutatingAdmissionWebhook`은 필요한 환경 변수로 태그를 처리합니다. 여기에서 이러한 변수에서 수집된 데이터는 APM 메트릭과 엔터티로 전달되어 귀중한 맥락을 제공하여 풍부하게 됩니다.

자세한 내용은 [애플리케이션을 Kubernetes에 연결하는](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-your-applications-kubernetes/) 방법을 참조하세요.

Kubernetes 요약 페이지를 표시하려면 APM 서비스 엔터티에 다음 속성이 필요합니다.

* `k8s.clusterName`
* `k8s.namespaceName`
* `k8s.deploymentName`

### OpenTelemetry 계측 [#otel-instrumentation]

[OpenTelemetry 수집기는](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro/) Kubernetes 대시보드를 통해 APM 텔레메트리를 강화하는 Kubernetes 속성 프로세서를 제공합니다.

1. 배포 매니페스트에서 환경 변수를 정의해야 합니다.

2. 이 환경 Kubernetes APM 변수를 사용하여 적절한 모니터링 데이터를 검색하려면 수집기 설정을 조정하세요.

결과적으로 모든 APM 메트릭과 엔터티에는 `K8sattributes` 프로세서 덕분에 Kubernetes 메타데이터가 포함되게 됩니다. 자세한 내용은 [OpenTelemetry 애플리케이션을 Kubernetes에 연결하는](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes/) 방법을 참조하세요.

Kubernetes 요약 페이지를 표시하려면 APM 서비스 엔터티에 다음 속성이 필요합니다.

* `k8s.cluster.name`
* `k8s.deployment.name`
* `k8s.namespace.name`

<Callout variant="important">
  Kubernetes 요약 페이지는 측정, 로그 제공자(뉴렐릭 및 OpenTelemetry)가 혼합되어 작동합니다. 그러나 Kubernetes가 OpenTelemetry 에이전트에 의해 모니터링되고 APM이 New Relic 에이전트에 의해 모니터링되는 경우 <Link to="https://github.com/newrelic/k8s-metadata-injection/tree/main/charts/nri-metadata-injection">nri-metadata-injection</Link>&lt;/span&gt;을 설치하여 제대로 작동하는지 확인하십시오. APM 메트릭에 필요한 Kubernetes 메타데이터를 제공합니다.
</Callout>
