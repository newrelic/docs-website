APM **요약** 페이지는 [웹 트랜잭션](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm) 및 비웹 [트랜잭션](/docs/apm/transactions/intro-transactions/monitor-background-processes-other-non-web-transactions) , [Apdex](/docs/site/apdex-measuring-user-satisfaction) 점수, CPU 사용량, 처리량(분당 요청 수 또는 **rpm** ), 트랜잭션 시간, 오류율, 애플리케이션 활동 및 [호스트](/docs/accounts-partnerships/accounts/account-billing-usage/compute-unit-pricing-host-pricing#understand) 를 포함하여 선택한 앱에 대한 일반 정보를 제공합니다.

<Callout variant="tip">
  모든 애플리케이션과 서비스에 대한 높은 수준의 개요를 보려면 [New Relic 탐색기](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer) 를 사용하십시오.
</Callout>

## 앱의 요약 페이지 보기 [#overview]

**요약** 페이지에 도달하려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) >APM & Services > (앱 선택)**으로 이동합니다. [모든 엔터티를](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/) 사용하여 모든 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)즉, 애플리케이션 및 호스트에서 모든 요소의 사용자 지정 그룹화에 이르기까지 데이터를 보고하는 식별할 수 있는 모든 항목에 액세스합니다.

자세한 내용은 [핵심 UI 구성 요소 탐색](/docs/new-relic-one/use-new-relic-one/get-started/new-relic-one-core-ui-components) 에 대한 설명서를 참조하십시오.

<Callout
  variant="important"
  title="EOL 공지"
>
  2022년 6월 30일부터 UI에서 켜고 끌 수 있는 기본 APM **요약** 페이지를 비롯한 여러 기능에 대한 지원이 중단됩니다. 이 전환을 쉽게 준비하는 방법을 포함한 자세한 내용은 [지원 포럼 게시물](https://discuss.newrelic.com/t/q1-bulk-eol-announcement-fy23/181744) 을 참조하십시오.
</Callout>

## 리소스에 앱 성능 연결 [#infra-server]

APM **요약** 페이지에는 다음을 포함하여 [호스트](/docs/accounts-partnerships/accounts/account-billing-usage/compute-unit-pricing-host-pricing#understand) 에 있는 앱 인스턴스에 대한 평균이 포함된 테이블이 표시됩니다.

* Apdex
* 응답 시간
* 처리량
* 오류율
* CPU 사용량
* 메모리

<Callout variant="tip">
  CPU 사용량 백분율은 애플리케이션이 하나의 CPU 코어에서 실행되는 것처럼 계산됩니다. 이 계산에 대한 자세한 내용은 [CPU 사용량이 100%를 초과함을](/docs/apm/new-relic-apm/troubleshooting/cpu-usage-over-100#h2-apps) 참조하십시오.
</Callout>


            이렇게 하면 컨테이너가 UTS Linux 네임스페이스를 기본 호스트와 공유하게 됩니다. 그러나 이 집합을 사용하면 권한 있는 컨테이너가 호스트의 호스트 이름을 변경할 수 있습니다.
          </td>
        </tr>
