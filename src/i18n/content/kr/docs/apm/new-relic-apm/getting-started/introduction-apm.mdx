애플리케이션 성능 모니터링의 약자인 APM은 모든 애플리케이션과 마이크로서비스에 대한 통합 모니터링 서비스를 제공합니다. 최신 스택의 수백여 개 종속성부터 간단한 웹 트랜잭션 시간과 앱 처리량에 이르기까지 모든 것을 모니터링합니다. 사전 구축된 커스텀 대시보드를 통해 메트릭, 이벤트, 로그 및 트랜잭션을 모니터링하여 앱의 상태를 실시간으로 추적할 수 있습니다.

뉴렐릭의 APM 서비스는 에이전트 중 하나를 설치할 때 코드를 자동으로 계측하여 앱에서 필요한 정확한 항목을 모니터링할 수 있는 유연성을 제공합니다.

* 조기 액세스 소프트웨어가 아직 불안정합니까? 사용자에게 영향을 미치기 전에, 에러 인박스로 이러한 오류를 사전에 모니터링하고 해결하십시오.
* 사용자가 웹 앱의 속도에 대해 자주 언급하고 있고, 그러한 피드백을 정량화하고 싶으신가요? Apdex를 한눈에 모니터링하여 만족도를 측정하십시오.
* 로그를 저장할 장소가 필요하십니까? 뉴렐릭 에이전트는 자동으로 로그를 수집합니다.
* 최신 스택의 수십 또는 수백 개의 종속성을 어떻게 추적해야 할까요? 관계 맵과 외부 서비스를 통해 추적할 수 있습니다.

단일 통합 모니터링 솔루션으로 시간을 절약하고 싶으신가요? APM을 시작하려면 로고를 클릭하십시오. 몇 분 이내에 가능합니다!

## 작동 원리 [#how-works]

뉴렐릭은 많은 언어 에이전트 중 하나를 사용해 코드 수준에서 애플리케이션을 계측합니다. 이러한 에이전트는 애플리케이션에서 다양한 메트릭을 수집하고 이를 뉴렐릭APM으로 보내 사전 구축된 대시보드에서 모니터링할 수 있도록 합니다.

## APM 설치 [#get-started]

몇 가지 짧은 단계로 APM을 시작하십시오.

1. 뉴렐릭 계정을 [신청](https://newrelic.com/signup/)하십시오.
2. 앱용 언어 에이전트를 설치합니다.

이제 애플리케이션에 약간의 트래픽을 생성하고 계정에 로그인하기만 하면 됩니다. 몇 분 안에 데이터 흐름이 보이기 시작합니다!

몇 분을 기다린 후에도 데이터가 나타나지 않으면 APM 에이전트에 대한 [문제 해결 팁](/docs/agents/manage-apm-agents/troubleshooting/not-seeing-data)을 따르십시오.

APM을 사용하기 전에 먼저 비용을 추정하고 싶으신가요? [비용 견적 리소스](https://newrelic.com/blog/nerdlog/estimate-data-cost)를 참조하세요.

## 데이터 사용 방법 [#use-case]

뉴렐릭이 앱에서 데이터를 수신하는 순간 앱의 기본 상태를 모니터링할 수 있습니다. 뉴렐릭 UI 의 <DNT>**APM Summary**</DNT> 페이지에 기본 <InlinePopover type="dashboards"/>사항을 파악하고 맞춤 설정 없이도 앱의 성능을 빠르게 이해할 수 있습니다.

[Explorer](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/)를 사용하면 앱을 포함한 전체 소프트웨어 스택에 액세스하여 관찰하고, 성능 데이터와 알림 상태를 한 눈에 보며, 관계를 확인할 수 있습니다. 뉴렐릭은 모든 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/), 즉 데이터를 보고하는 식별할 수 있는 모든 항목을 모니터링할 수 있는 간단하면서도 강력한 시각적 툴을 제공합니다. 뉴렐릭 에코시스템에서 엔터티는 애플리케이션, 호스트, 컨테이너 또는 데이터베이스 서비스 같은 기본 구성 요소를 포함하지만 이러한 요소의 커스텀 그룹을 참조할 수도 있습니다. [고유한 엔터티를 생성](https://github.com/newrelic/entity-definitions#entity-definitions)할 수도 있습니다.
