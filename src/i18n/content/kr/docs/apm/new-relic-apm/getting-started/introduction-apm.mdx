현대의 애플리케이션들은 복잡합니다. 여러 서비스, 데이터베이스, API 및 의존성/종속성을 관리하며 사용자에게 빠르고 안정적인 경험을 제공하기 위해 애쓰고 있지만, 문제가 발생했을 때 근본 원인을 찾는 것은 모래사장에서 바늘을 찾는 것처럼 느껴질 수 있습니다.

## 애플리케이션 성능 문제로 야기되는 비용

올바르게 모니터링되지 않으면 애플리케이션 문제는 비즈니스에 상당한 영향을 미칠 수 있습니다.

* **사용자 경험 저하** - 페이지 로딩 속도가 느리면 모바일 사용자 53%는 단 3초 만에 이탈합니다
* **수익 손실 위험** - 100밀리초의 레이턴시가 발생할 때마다 전환율이 7% 감소합니다.
* **팀 생산성 저하** - 개발자들은 기능 개발이 아니라 문제 해결에 75%의 시간을 소비합니다.
* **인시던트 악화** - 팀이 아니라 사용자가 발견한 문제로 인해 평판과 신뢰가 손상됩니다.

## 애플리케이션 성능 혁신

뉴렐릭 APM은 애플리케이션 성능을 선제적으로 관리하는 데 필요한 가시성과 인사이트를 제공합니다.

**사용자보다 먼저 문제를 발견할 수 있습니다.**

* 지능형 알림과 실시간 오류 추적
* 머신 러닝을 활용한 선제적 이상 감지
* 성능 저하 시 즉시 알림 제공

**몇 시간이 아닌 몇 분 안에 문제를 정확히 파악할 수 있습니다.**

* 속도 저하 위치를 파악해주는 세부적인 트랜잭션 트레이스
* 병목 지점을 식별해주는 데이터베이스 쿼리 분석
* 성능 핫스팟을 표시해주는 코드 레벨 인사이트

**확신을 갖고 최적화할 수 있습니다.**

* Apdex 점수로 사용자 만족도 정량화
* 성능 기준선을 설정해 시간에 따른 개선 사항 추적
* A/B 테스트 영향 측정

## APM 모니터링의 작동 원리

APM 모니터링은 생각보다 간단합니다.

1. **경량 에이전트 설치** - 복잡한 변경 없이 코드를 자동 계측하는 소용량 라이브러리를 애플리케이션에 추가합니다.
2. **자동 데이터 수집** - 에이전트가 최소한의 오버헤드(&lt; 3% 성능 영향)로 성능 지표, 오류 및 사용자의 상호 작용을 추적합니다.
3. **실시간 분석** - 뉴렐릭으로 데이터가 유입되면 AI 알고리즘이 패턴, 이상 징후, 추세를 감지합니다.
4. **실행 가능한 인사이트** - 사전 구축된 대시보드와 알림으로 가장 중요한 사항을 즉시 파악할 수 있습니다.

전체 스택에서 포괄적인 옵저버빌리티를 확보할 수 있고 애플리케이션들은 정상적으로 계속 실행됩니다.

## 몇 분 안에 시작할 수 있습니다.

지금 바로 애플리케이션 모니터링을 혁신하세요. 언어를 선택하면 몇 분 안에 성능에 대한 인사이트를 확인할 수 있습니다.

<Callout variant="tip">
  **무료 계정** - 월 최대 100GB의 데이터를 무료로 모니터링할 수 있습니다. 신용카드는 필요하지 않습니다.
</Callout>

**설치 후 기대할 수 있는 이점:**

1. **즉각적인 인사이트** - 배포 후 2-3분 내에 성능 데이터를 확인할 수 있습니다.
2. **자동 기준선 설정** - APM이 애플리케이션의 정상적인 동작 패턴을 학습합니다.
3. **스마트 알림** - 사용자 경험에 실질적인 영향을 미치는 문제가 발생했을 때만 알림을 받습니다.
4. **안내되는 최적화** - 성능 향상을 위해 구체적인 권장 사항이 제공됩니다.

<Callout variant="important">
  **시작하는 데 도움이 필요하신가요?** 각 에이전트에는 단계별 설치 가이드와 문제 해결 지원이 포함되어 있습니다. 대부분의 팀은 15분 이내에 설치하고 사용을 시작합니다.
</Callout>

## 즉각적인 혜택

에이전트가 설치되고 데이터 전송이 시작되면 다음 기능에 액세스할 수 있습니다.

**APM 요약 대시보드**

* 응답 시간 트렌드 및 처리량 메트릭
* 스택 추적을 통한 오류율 추적
* 데이터베이스 쿼리 성능 분석
* 외부 서비스 호출 모니터링

**고급 기능**

* 모든 애플리케이션 및 인프라에 대한 [풀스택 옵저버빌리티](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/)
