뉴렐릭 APM 에이전트는 이제 OpenTelemetry (OTel) API를 지원하므로 뉴렐릭의 강력한 APM 기능에 대한 풀 액세스를 유지하면서 빼기 공급 업체 비종속 OpenTelemetry API를 사용하여 리소스를 축소할 수 있습니다. 이 기능을 사용하면 뉴렐릭이 제공하는 인사이트 및 기능의 깊이를 희생하지 않고도 이식 가능한 측정, 로그 코드를 작성할 수 있는 유연성을 얻을 수 있습니다.

## 혜택 [#benefits]

뉴렐릭 에이전트와 함께 OpenTelemetry API 지원을 사용하면 다음과 같은 몇 가지 이점을 얻을 수 있습니다.

* **업체 비종속 측정, 수신**: 감시 플랫폼에서 작동하는 표준 OpenTelemetry API를 사용하여 사용자 정의 측정 작성
* **마이그레이션이 필요하지 않습니다**. 현재 측정값을 교체하지 않고 기존 뉴렐릭 에이전트에서 OTel API 지원을 활성화합니다.
* **점진적 채택**: 기존 뉴렐릭 측정, 리소스가 계속 작동하는 동안 새로운 사용자 정의 측정을 위해 OpenTelemetry API를 사용합니다.
* **혼합 모드 호환성**: 뉴렐릭 에이전트, OpenTelemetry API 또는 순수 OTel 에이전트를 사용하여 서비스 전반에 걸쳐 requests 원활하게 추적합니다.
* **커뮤니티 측정, 리소스**: 프레임워크 및 라이브러리에 대한 기본 OpenTelemetry 측정, 리소스 활용

## 이 기능이 제공하는 것 [#what-it-provides]

뉴렐릭 에이전트에서 OpenTelemetry API 지원을 활성화하면 에이전트는 다음을 수행할 수 있습니다.

* OpenTelemetry API를 사용하여 생성된 스팬, 트레이스, 메트릭 및 로그를 캡처합니다.
* OTel에서 생성된 텔레메트리를 뉴렐릭 APM 트레이스에 원활하게 통합합니다.
* 뉴렐릭의 내장 측정, 로그와 충돌하지 않는 기본 OpenTelemetry 측정 측정 지원
* 혼합된 뉴렐릭 및 OpenTelemetry 권한화된 서비스 전체에서 트레이스 컨텍스트를 유지 관리합니다.

<Callout variant="important">
  이 기능은 **기본적으로 비활성화되어** 있습니다. 에이전트 설정을 통해 명시적으로 활성화해야 합니다.
</Callout>

### 혼합 환경에서의 샘플링

이제 모든 기기가 OpenTelemetry 표준에 맞춰 **트랜스 비율 샘플링을** 지원하며, 다양한 환경에서 일관된 샘플링 로직을 보장합니다.

**주요 이점**:

* 뉴렐릭 및 OTel-계측된 서비스 전반에 걸쳐 일관된 샘플링 결정
* OTel SDK에서 상위 트레이스 결정 사항 보존

적응형 샘플링 작동 방식, 샘플링 강제, 요원별 목표 및 설정 옵션에 대한 자세한 내용은 [꺾이는 추적 샘플링을](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works/#trace-origin-sampling) 참조하세요.

## 지원되는 측정, 로그 유형 [#instrumentation-types]

### 네이티브 측정, 로그(자동)

네이티브 OpenTelemetry 측정, 로그는 기본적으로 지원됩니다. 이러한 측정, 로그는 유지 관리 담당자가 라이브러리 또는 프레임워크에 직접 구축하고 OpenTelemetry API 지원을 활성화하면 자동으로 작동합니다.

사용 가능한 구체적인 네이티브 측정, 도구는 언어 에이전트에 따라 다릅니다. 각 언어에 대해 지원되는 측정, 로그에 대한 자세한 내용은 [설정 섹션을](#configuration) 참조하세요.

<Callout variant="important">
  **알려진 제한 사항**: 일부 네이티브 측정, 계측은 OpenTelemetry의 내장 측정, 계측과 비교하여 스팬 이름이나 범위 이름이 약간 다르게 생성될 수 있습니다. 이는 OpenTelemetry 의 의미 규칙 차이 때문입니다(예: Elasticsearch, RabbitMQ).
</Callout>

### OpenTelemetry API를 사용한 맞춤형 측정

표준 OpenTelemetry API를 사용하여 맞춤형 측정을 작성할 수 있습니다.

* **트레이스**: OpenTelemetry Tracing API사용하여 사용자 지정 스팬을 생성하고 속성을 추가합니다.
* **지표**: OpenTelemetry 지표 API 사용하여 지표 지표를 기록합니다(지원되는 경우).
* **로그**: OpenTelemetry 로그 API 사용하여 로그 쓰기(지원되는 경우)

해당 언어에서 이러한 API를 사용하는 방법에 대한 자세한 내용은 [OpenTelemetry 설명서를](https://opentelemetry.io/docs/instrumentation/) 참조하십시오.

## OpenTelemetry API 지원을 활성화하는 방법 [#enable]

OpenTelemetry API 지원은 기본적으로 비활성화되어 있습니다. 에이전트 설정을 통해 해당 기능을 활성화하세요.

<CollapserGroup>
  <Collapser id="java-config" title="자바 에이전트 구성">
    ### 전제 조건

    필수 OpenTelemetry 의존성/종속성을 제작에 추가하세요.

    ```groovy
    implementation(platform("io.opentelemetry:opentelemetry-bom:1.44.1"))
    implementation("io.opentelemetry:opentelemetry-sdk-extension-autoconfigure")
    implementation("io.opentelemetry:opentelemetry-exporter-otlp")
    ```

    ### OpenTelemetry API 지원을 활성화합니다.

    **1. OpenTelemetry SDK 자동 구성을 활성화합니다.**

    다음 방법 중 하나를 사용하십시오.

    시스템 속성:

    ```bash
    -Dotel.java.global-autoconfigure.enabled=true
    ```

    환경 변수:

    ```bash
    export OTEL_JAVA_GLOBAL_AUTOCONFIGURE_ENABLED=true
    ```

    **2. 뉴럴릭 OpenTelemetry 지원 활성화**

    시스템 속성:

    ```bash
    -Dnewrelic.config.opentelemetry.enabled=true
    ```

    환경 변수:

    ```bash
    NEW_RELIC_OPENTELEMETRY_ENABLED=true
    ```

    YAML 설정(`newrelic.yml`):

    ```yaml
    opentelemetry:
      enabled: true
    ```

    <Callout variant="important">
      로그 에이전트는 OpenTelemetry 트레이스, 메트릭, 로그 API를 사용하여 사용자가 내보낸 신호를 캡처합니다. 이 제품은 OpenTelemetry 측정, 계측 기능을 자동으로 활성화하거나 제공하지 **않습니다**. OpenTelemetry 자동 측정, 자동 로그는 지원되지 않습니다.
    </Callout>

    ### 지원되는 측정, 로그 유형

    #### 수동 계측

    OpenTelemetry 트레이스, 지표 또는 로그 API를 사용하는 사용자 정의 측정은 에이전트에 의해 자동으로 캡처됩니다.

    #### 라이브러리 및 네이티브 측정, 로그

    OpenTelemetry 독립 실행형 측정 라이브러리, 리소스 및 네이티브 측정 리소스, 라이브러리에 직접 내장되어 뉴렐릭 에이전트와 함께 작동합니다. 하지만 OpenTelemetry 바이트코드 기반 자동 측정, 자동 리소스는 뉴렐릭 에이전트에 의해 감지되지 않습니다.
  </Collapser>

  <Collapser id="nodejs-config" title="Node.js 에이전트 구성">
    ### OpenTelemetry API 지원을 활성화합니다.

    `newrelic.js` 설정 파일에서 OpenTelemetry API 지원을 활성화하세요.

    ```javascript
    exports.config = {
      // ... other config
      opentelemetry: {
        enabled: true,
        traces: {
          enabled: true
        }
      }
      // ... other config
    }
    ```

    <Callout variant="important">
      Node.js 에이전트는 OpenTelemetry 측정, 리소스를 자동으로 활성화 **하지 않습니다**. 귀하가 내보낸 텔넷리를 캡처하기 위해 OpenTelemetry API만 추가했습니다.
    </Callout>

    ### 지원되는 OpenTelemetry API

    OpenTelemetry 지원이 활성화되면 Node.js 에이전트는 다음을 완벽하게 지원합니다.

    #### 트레이스 API

    OpenTelemetry Tracing API를 사용하여 사용자 지정 스팬을 생성하고 속성을 추가합니다. [OpenTelemetry Node.js 추적 예제를](https://github.com/newrelic/newrelic-node-examples/tree/4a3332d55e4455dca0f8eeba34a87e4a513a679a/opentelemetry-example) 참조하세요.

    #### 메트릭 API

    지표 API는 Node.js 고객에게 **API 유용한 기능입니다**. 이는 자주 요청되었던 기능인 차원 메트릭을 포함하도록 메트릭 수집 기능을 확장합니다.

    OpenTelemetry 지표 API 사용하여 사용자 정의 지표를 기록합니다. [OpenTelemetry Node.js 메트릭 예제를](https://github.com/newrelic/newrelic-node-examples/tree/4a3332d55e4455dca0f8eeba34a87e4a513a679a/opentelemetry-example) 참조하세요.

    #### 로그 API

    JavaScript OpenTelemetry 로그 API 아직 개발 중이지만 Node.js 에이전트가 이를 지원합니다. API 사용하여 뉴렐릭으로 전송될 로그를 생성할 수 있습니다.

    [OpenTelemetry Node.js 로그 예제를](https://github.com/newrelic/newrelic-node-examples/tree/4a3332d55e4455dca0f8eeba34a87e4a513a679a/opentelemetry-example) 참조하세요.

    ### 지원되는 측정, 로그 유형

    #### 수동 측정, 로그(권장)

    OpenTelemetry API 사용하여 인력을 추가하는 고객은 에이전트가 해당 데이터를 올바르게 처리하고 배송한다는 것을 알게 될 것입니다.

    #### 네이티브 및 라이브러리 측정, 로그(제한적)

    OpenTelemetry 측정 측정 목록은 Node.js 에이전트의 지원 모듈과 상당히 중복됩니다. 아직 뉴렐릭 에이전트 측정에서 다루지 않은 로그는 다음과 같습니다.

    * 에이전트 내부와의 충돌
    * 고객 수요가 충분하지 않습니다.

    <Callout variant="caution">
      저희는 공식 OpenTelemetry 목록 외의 OpenTelemetry 측정 및 계측 도구를 테스트하지 않았으므로 현재로서는 특정 도구를 추천해 드릴 수 없습니다.
    </Callout>

    ### 알려진 제한 사항

    * **자동 측정, 도구 없음**: 에이전트는 OpenTelemetry 측정, 로그를 자동으로 활성화하지 않습니다. 고객은 OpenTelemetry API를 사용하여 코드를 수동으로 업로드해야 합니다.
  </Collapser>

  <Collapser id="dotnet-config" title=".NET 에이전트 구성">
    ### OpenTelemetry API 지원을 활성화합니다.

    환경 변수 또는 `newrelic.config` XML 설정 파일을 사용하여 OpenTelemetry API 지원을 활성화하십시오.

    **환경 변수 사용:**

    ```bash
    NEW_RELIC_OPENTELEMETRY_ENABLED=true
    ```

    **newrelic.config 사용:**

    ```xml
    <configuration>
      <openTelemetry enabled="true" />
    </configuration>
    ```

    <Callout variant="tip">
      OpenTelemetry 지원이 활성화되면 **트레이스** 와 **범위는** 모두 기본적으로 활성화됩니다.
    </Callout>

    ### 지원되는 네이티브 측정, 로그

    OpenTelemetry API 지원을 활성화하면 .NET 에이전트는 OpenTelemetry-계측팀에서 텔레메트리를 캡처할 수 있습니다.

    #### 트레이스 측정, 실행

    에이전트는 네이티브 OpenTelemetry 추적 측정, 리소스를 지원합니다. OpenTelemetry Tracing API 사용하여 작성된 맞춤형 측정도 지원됩니다.

    #### 측정값, 사용률

    지표가 활성화되면 에이전트는 OpenTelemetry 메트릭 측정, 라이브러리 및 프레임워크를 지원합니다. OpenTelemetry 지표 API 사용하여 작성된 맞춤형 측정도 지원됩니다.

    #### 로그 측정, 사용

    .NET 에이전트는 이미 로그 데이터용 OpenTelemetry API 된 로깅 라이브러리를 포함하고 있습니다. OpenTelemetry API 지원을 활성화할 때 로그를 **변경할 필요는 없습니다**. 이전 버전과의 호환성을 위해, 로그아웃, 로그아웃은 이전과 같이 계속 작동합니다.
  </Collapser>

  <Collapser id="python-config" title="Python 에이전트 구성">
    ### OpenTelemetry API 지원을 활성화합니다.

    `newrelic.ini` 설정 파일 또는 환경 변수를 사용하여 OpenTelemetry API 지원을 활성화하세요.

    **newrelic.ini 사용:**

    ```ini
    [newrelic]
    opentelemetry.enabled = true
    ```

    **환경 변수 사용:**

    ```bash
    NEW_RELIC_OPENTELEMETRY_ENABLED=true
    ```

    <Callout variant="important">
      OpenTelemetry 지원이 활성화되면 에이전트는 `opentelemetry-api` 설치되어 있는지 자동으로 감지하고 지원되는 라이브러리에 대한 OpenTelemetry 측정, 로그를 사용하도록 전환합니다.
    </Callout>

    ### 작동 원리

    OpenTelemetry 지원이 활성화되고 에이전트가 `opentelemetry-api` 설치되어 있음을 감지하면 다음과 같습니다.

    1. **네이티브 측정, 리소스**: 네이티브 OpenTelemetry 측정, 리소스가 내장된 라이브러리의 경우 에이전트는 자동으로 뉴렐릭 후크를 비활성화하고 대신 네이티브 OpenTelemetry 측정, 리소스를 사용합니다.

    2. **측정, 리소스**: OpenTelemetry 측정, 리소스 라이브러리가 감지되면 에이전트는 뉴렐릭의 측정, 리소스 후크 대신 자동으로 OpenTelemetry 측정 라이브러리를 사용합니다.

    현재 사용 가능한 수축 수축, 리소스 목록은 [공식 OpenTelemetry 수축 수축, 리소스 목록을](https://opentelemetry.io/ecosystem/registry/?language=python&component=instrumentation) 참조하세요.

    <Callout variant="caution">
      일부 측정, 로그(예: AWS 및 gRPC 라이브러리)는 계속해서 뉴렐릭의 측정, 로그를 사용합니다. 네이티브 측정이 포함된 AI 라이브러리는 현재 명시적으로 비활성화되어 있으며 향후 릴리스에서 지원될 예정입니다.
    </Callout>

    ### 수동 계측

    OpenTelemetry Tracing API 사용하여 작성된 모든 맞춤형 측정은 에이전트에 의해 캡처됩니다.

    **예시:**

    ```python
    from opentelemetry import trace

    tracer = trace.get_tracer(__name__)

    with tracer.start_as_current_span("my-custom-span") as span:
        span.set_attribute("custom.attribute", "value")
        # Your code here
    ```

    ### 지원되는 기능

    #### 트레이스 API

    ✅ **완벽 지원**: OpenTelemetry Tracing API를 사용하여 사용자 지정 스팬을 생성하고 속성을 추가할 수 있습니다.

    #### 메트릭 API

    ❌ **아직 지원되지 않음**: 끌어당김 에이전트에서는 OpenTelemetry 지표 API 활성화되지 않습니다.

    #### 로그 API

    ✅ **이미 작동함**: 에이전트가 이미 로깅 모듈을 지원합니다. 파이썬용 OpenTelemetry 로깅 API는 존재하지 않습니다(SDK만 있습니다). OpenTelemetry에서 로깅을 위해 설정하는 것은 로깅 활성화 여부와 데이터를 전송할 엔드포인트입니다.

    ### 작동 원리

    OpenTelemetry 지원이 활성화된 경우:

    1. **에이전트는 시스템을 검색하여** OpenTelemetry 측정, 로그 라이브러리가 설치되어 있는지 확인합니다.
    2. **자동 전환**: 에이전트가 발견되면 뉴렐릭 후크 대신 OpenTelemetry 라이브러리의 측정, 로그를 사용합니다.
    3. **원활한 통합**: OpenTelemetry 스팬이 블루렐릭 트레이스에 통합되었습니다.

    ### 알려진 제한 사항

    * **지원되지 않음**: OpenTelemetry API 아직 활성화되지 않았습니다.
    * **일부 AWS 및 gRPC 라이브러리 제외**: 계속해서 뉴렐릭 측정, 로그를 사용할 예정입니다.
    * **AI 라이브러리 비활성화됨**: 현재 AI 라이브러리에 대한 네이티브 OpenTelemetry 측정, 로그가 비활성화되어 있습니다.
    * **포함/제외 설정**: 세부 조정 옵션은 향후 릴리스에서 제공될 예정이며, 현재는 활성화/비활성화 기능만 작동합니다.
  </Collapser>

  <Collapser id="ruby-config" title="Ruby 에이전트 구성">
    ### OpenTelemetry API 지원을 활성화합니다.

    루비 에이전트에서 OpenTelemetry API 지원을 활성화하려면 `newrelic.yml` 설정 파일에 다음을 추가하세요.

    ```yaml
    opentelemetry:
      enabled: true
    ```

    또는 환경 변수를 설정하세요.

    ```bash
    NEW_RELIC_OPENTELEMETRY_ENABLED=true
    ```

    <Callout variant="important">
      OpenTelemetry 지원이 활성화되면 에이전트는 `opentelemetry-api` 설치되어 있는지 자동으로 감지하고 지원되는 라이브러리에 대한 OpenTelemetry 측정, 로그를 사용하도록 전환합니다.
    </Callout>

    ### 수동 계측

    OpenTelemetry Tracing API 사용하여 작성된 모든 맞춤형 측정은 에이전트에 의해 캡처됩니다.

    **예시:**

    ```ruby
    require 'opentelemetry'

    APP_TRACER = OpenTelemetry.tracer_provider.tracer('my-app', '1.0.0')

    APP_TRACER.in_span("my-custom-span") do |span|
      span.set_attribute("test.attribute", "value")
      # Your code here
    end
    ```

    ### 지원되는 기능

    #### 트레이스 API

    ✅ **지원됨**: OpenTelemetry Tracing API를 사용하여 사용자 지정 스팬을 생성하고 속성을 추가할 수 있습니다.

    #### 메트릭 API

    ❌ **지원되지 않음**: OpenTelemetry API

    #### 로그 API

    ❌ **지원되지 않음**: OpenTelemetry 로그 API

    #### 스팬 링크

    ❌ **지원되지 않음**: 스팬 링크

    #### Spans의 이벤트

    ❌ **지원되지 않음**: 스팬의 이벤트

    ### 작동 원리

    OpenTelemetry 지원이 활성화되면 뉴렐릭 루비 에이전트는 다음을 수행합니다.

    1. OpenTelemetry 추적 API에 대한 호출을 **수신합니다.**
    2. 해당 호출을 뉴럴릭 데이터 모델로 **변환합니다**.
    3. OpenTelemetry 범위를 블루렐릭 트레이스에 **통합합니다.**

    <Callout variant="information">
      기본적으로 OpenTelemetry 측정, 리소스는 라이브러리에 대해서도 제외됩니다. 여기에는 [OpenTelemetry-ruby-contrib](https://github.com/open-telemetry/opentelemetry-ruby-contrib) 의 기본적으로 수용체인 라이브러리 및 측정, 로그 패키지가 포함됩니다.
    </Callout>

    ### 알려진 제한 사항

    * **API 지원 제한**: 이번 릴리스에서는 TRAS API 만 지원됩니다.
    * **지표 및 기록**: 해당사항 없음
    * **Span 링크 및 이벤트**: 현재 지원되지 않습니다.
  </Collapser>
</CollapserGroup>

## 제대로 작동하는지 확인하십시오. [#verify]

OpenTelemetry API 지원을 활성화한 후 정상적으로 작동하는지 확인하십시오.

1. 업데이트된 에이전트 설정으로 구현하고 배포합니다.

2. 맞춤형 측정을 사용하는 경우 코드가 OpenTelemetry API를 사용하여 범위/메트릭을 생성하는지 확인하세요.

3. 애플리케이션으로 트래픽을 유도하세요.

4. 뉴렐릭 APM UI 확인하세요.

   * **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; services &gt; (앱 선택)**로 이동하세요.
   * 트레이스에 통합된 OpenTelemetry API에 의해 생성된 범위를 보려면 분리 트레이스를 확인하세요.

### 의미론적 관습의 차이

일부 OpenTelemetry 측정 도구는 뉴렐릭의 내장 측정 도구와 비교하여 범위 및 지표에 대해 다른 명명 규칙을 사용합니다.

* **Elasticsearch**: 스팬 이름과 작업 이름은 대소문자 및 길이가 다를 수 있습니다.
* **RabbitMQ**: 분산 추적 방식이 다릅니다 (스팬 링크 vs. 부모/자식 관계).

<Callout variant="tip">
  이러한 차이점은 특정 범위 또는 지표 이름에 의존하는 기존 대시보드 및 공지 조건에 영향을 미칠 수 있습니다. 이러한 서비스에 대해 뉴렐릭 측정, 렌더링에서 OpenTelemetry 측정, 렌더링으로 전환하는 경우 쿼리를 검토하고 업데이트하세요.
</Callout>

### 에이전트별 제한사항

* **끌어오기**: 트레이스 API 만 지원됩니다. 지표 및 로그 API를 사용할 수 없습니다.
* **루비**: 트레이스 API 만 지원됩니다. 지표 및 로그 API를 사용할 수 없습니다.
* **.NET**: OpenTelemetry 활성화는 차원 데이터 수집 비용을 증가시킬 수 있습니다.

### OpenTelemetry 스팬이 트레이스에 나타나지 않음

OpenTelemetry API 지원을 활성화했지만 추적 기록에 OTel에서 생성된 스팬이 표시되지 않는 경우:

<CollapserGroup>
  <Collapser id="verify-enabled" title="OpenTelemetry 지원이 활성화되어 있는지 확인하십시오.">
    OpenTelemetry 지원이 활성화되어 있는지 확인하려면 에이전트 설정을 확인하세요.

    * 환경 변수 또는 설정 파일을 검토하세요.
    * OpenTelemetry 지원이 활성화되었는지 확인하려면 에이전트 로그를 확인하십시오.
    * 설정을 변경한 후에는 애플리케이션을 다시 시작했는지 확인하십시오.
  </Collapser>

  <Collapser id="check-version" title="에이전트 버전을 확인하세요">
    OpenTelemetry API를 지원하는 최소 필수 에이전트 버전이 실행 중인지 확인하십시오. 위의 [지원 언어 표를](#supported-languages) 참조하십시오.
  </Collapser>
</CollapserGroup>
