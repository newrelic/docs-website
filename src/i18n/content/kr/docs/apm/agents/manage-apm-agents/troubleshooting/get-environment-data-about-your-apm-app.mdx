## 문제

다음과 같이 애플리케이션 환경에 대한 정보를 쉽게 얻을 수 있는 방법이 필요합니다.

* 특정 에이전트 버전이 사용하는 서비스
* 에이전트 버전이 오래된 앱
* 어떤 호스트가 어떤 에이전트를 사용하는지
* 특정 에이전트 설정에 적용되는 값

## 해결책

에이전트에 적용되는 서비스 및 설정을 확인하거나 특정 서비스 또는 설정을 사용하는 에이전트를 보려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 으로 이동하십시오. 목록에서 앱을 클릭한 다음 **Settings > Environment > Environment snapshot** 을 클릭합니다.

**환경 스냅샷** 페이지는 환경 데이터가 포함된 에이전트의 페이로드를 연결하는 데 도움이 되므로 어떤 모니터링 데이터가 기록되는지 알 수 있습니다. 이는 또한 서비스 전반에서 불일치를 보다 쉽게 식별하고 업데이트해야 할 항목을 결정하는 데 도움이 됩니다.

다음 예에서 사용자는 목록의 두 번째 에이전트 위로 마우스를 이동합니다. 이 에이전트에 대한 대부분의 설정은 다른 것과 일치하지만 논리적 프로세서, 물리적 코어 및 Ruby 패치 수준에 대한 특정 호스트 설정을 쉽게 볼 수 있습니다. 또한 이 에이전트는 이전 Ruby 버전을 사용하고 있습니다.

**환경 스냅샷** 페이지의 정보는 사용하는 APM 에이전트에 따라 다릅니다. 예를 들어 Ruby 에이전트를 사용하는 앱은 gem에 대한 설정을 표시하고 Java 에이전트를 사용하는 앱은 jar에 대한 설정을 표시합니다.
