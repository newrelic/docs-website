Kubernetes 관련 환경 변수를 수집하고 이를 트랜잭션 이벤트에 추가하여 Kubernetes 클러스터와 서비스 간의 컨텍스트를 제공하도록 에이전트를 구성할 수 있습니다. 이점에 대한 자세한 내용은 [이 블로그 게시물](https://blog.newrelic.com/engineering/monitoring-application-performance-in-kubernetes/) 을 참조하십시오.

에이전트 호환성 및 설치 지침에 대한 자세한 내용은 [Kubernetes 설명서](/docs/integrations/kubernetes-integration/metadata-injection/kubernetes-apm-metadata-injection) 를 참조하십시오.
