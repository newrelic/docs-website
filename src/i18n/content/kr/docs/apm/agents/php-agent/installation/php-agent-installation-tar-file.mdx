지원되는 프로세서의 모든 지원 운영 시스템에 압축된 tar 파일을 통해 뉴렐릭의 PHP 에이전트를 설치할 수 있습니다. 그러나 x86_64 기반 RedHat 또는 CentOS의 경우 [RPM 패키지를](/docs/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos/) 사용하는 것이 더 일반적입니다. x86_64의 Ubuntu 또는 Debian의 경우 [Debian 패키지를](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian) 사용하는 것이 더 일반적입니다.

## tar 배포판 다운로드 [#download]

다른 설치 옵션과 달리 tar 아카이브는 특별한 저장소 설정이 필요하지 않습니다. 아카이브를 다운로드하고 이 지침을 따르기만 하면 됩니다.

<DNT>**[download.newrelic.com/php_agent/release/](https://download.newrelic.com/php_agent/release/)**</DNT> 에서 적절한 tar 배포 파일을 다운로드합니다.

* GNU libc를 사용하는 Linux(대부분의 Linux 배포판)의 경우 `newrelic-php5-X.X.X.X-linux.tar.gz` 다운로드하세요.
* musl libc가 있는 Linux(Alpine Linux)의 경우 `newrelic-php5-X.X.X.X-linux-musl.tar.gz`(를) 다운로드하세요.

<Callout variant="tip">
  PHP 에이전트의 패키지 이름은 `newrelic-php5` 입니다. 패키지 이름은 PHP 5를 참조하지만 이 패키지는 [지원되는 모든 PHP 버전](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release) 에서 작동합니다.
</Callout>

## 설치 또는 업데이트 [#install]

tar 파일을 설치하는 경우 처음 설치하는 단계와 업데이트하는 단계는 동일합니다. XXXX를 올바른 New Relic 버전으로 바꾸십시오. 에이전트를 설치하거나 업데이트하려면:

1. [New Relic 웹사이트](https://download.newrelic.com/php_agent/release/) 에서 적절한 tar 파일을 다운로드하고 편리한 위치의 로컬 디스크에 저장합니다. `/tmp` 폴더를 사용하지 마십시오.

2. 아카이브 압축을 풀고 압축을 풉니다.

   ```bash
   gzip -dc newrelic-php5-X.X.X.X-OS.tar.gz | tar xf -
   ```

3. 새로 생성된 디렉토리로 변경:

   ```bash
   cd newrelic-php5-X.X.X.X-OS
   ```

4. 설치 스크립트를 실행합니다.

   ```bash
   ./newrelic-install
   ```

5. 기본 [응용 프로그램 이름](/docs/site/naming-your-application) 을 의미 있는 이름으로 변경합니다.

6. 웹 서버(Apache, Nginx, PHP-FPM 등)를 다시 시작합니다.

7. 애플리케이션에서 데이터를 보낼 때까지 몇 분 정도 기다린 다음 [UI에서 앱의 성능을 확인합니다](/docs/apm/applications-menu/monitoring/apm-overview-page) .
