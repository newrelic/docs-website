## 문제

다음과 같은 작업 후에 웹 서버(Apache, PHP-FPM, Nginx 등)를 다시 시작해야 하는 이유 또는 시기를 잘 모르는 경우 다음을 수행합니다.

* PHP 에이전트 [설치](/docs/agents/php-agent/installation/php-agent-installation-overview) 또는 [업그레이드](/docs/agents/php-agent/installation/upgrading-php-agent)
* [PHP를 주요 릴리스 버전으로 업데이트](/docs/agents/php-agent/troubleshooting/agent-stops-working-after-updating-php)
* `newrelic.ini` 또는 `newrelic.cfg` 파일에서 에이전트 [구성 설정](/docs/agents/php-agent/configuration/php-agent-configuration) 변경
* 데몬의 [외부 시작 모드](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#selecting-external) 활용

## 원인

PHP 에이전트는 PHP 확장입니다. 웹 서버가 시작을 하며 PHP를 로드할 때, PHP 에이전트도 로드하고 `newrelic.ini` 구성 파일을 구문 분석합니다. 웹 서버는 구성 파일을 다시 로드하고 구문 분석하라는 지시가 있을 때까지 그 구성을 사용합니다. 다시 시작하면 웹 서버에 이를 수행하라는 메시지가 표시됩니다.

다음과 같은 경우 웹 서버를 다시 시작해야 합니다.

<CollapserGroup>
  <Collapser
    id="config"
    title="구성 파일을 수정합니다."
  >
    `newrelic.ini` 또는 <DNT>**`newrelic.cfg`**</DNT> 파일의 구성 변경 사항은 웹 서버를 다시 시작할 때까지 [적용되지 않습니다.](/docs/agents/php-agent/troubleshooting/ini-settings-not-taking-effect-immediately) (예: 앱 이름 지정, 로그 생성, 라이선스 키 변경)
  </Collapser>

  <Collapser
    id="package-manager-auto-update"
    title="패키지 관리자는 웹 서버를 다시 시작하지 않고 에이전트를 자동 업데이트합니다."
  >
    <Callout variant="tip">
      이러한 상황을 방지하려면 패키지 관리자에서 `newrelic-php5*`에 대한 예외를 설정해야 합니다.
    </Callout>

    패키지 관리자가 자동으로 [에이전트를 업데이트하는](/docs/agents/php-agent/installation/upgrading-php-agent) 경우 웹 서버를 다시 시작해야 합니다. 다시 시작하지 않고 에이전트를 업데이트하면 에이전트가 보고를 중지할 수 있습니다.

    이 문제는 자동 업데이트가 활성화되고 데몬이 새 버전으로 업데이트될 때 발생합니다. 그러면 새 데몬은 아직 이전 버전을 실행 중인 에이전트와 통신할 수 없습니다.

    이를 방지하려면 업데이트 후에 새 에이전트를 시작해야 합니다. 이 프로세스는 복잡합니다. 전체 에이전트를 다시 시작하려면 웹 서버 소프트웨어도 다시 시작해야 하기 때문입니다.
  </Collapser>

  <Collapser
    id="daemon-external-mode"
    title="데몬은 외부 모드에서 시작하도록 설정됩니다."
  >
    에이전트가 [외부 시작 모드](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#selecting-external)인 경우 [수동으로 데몬](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#daemon-external)을 시작하고 에이전트가 보고할 에이전트를 다시 시작해야 합니다.
  </Collapser>

  <Collapser
    id="full-restart"
    title={<>Apache가 <InlineCode>apachectl restart</InlineCode> 명령으로 완전히 다시 시작되지 않습니다.</>}
  >
    `apachectl restart` 명령을 사용하면 웹 서버가 항상 완전하게 다시 시작되는 것은 아니기 때문에, 웹 서버가 에이전트의 구성 파일을 다시 구문 분석하지 않습니다. 다음을 사용할 것을 권장합니다.

    ```
    apachectl stop
    apachectl start
    ```
  </Collapser>
</CollapserGroup>
