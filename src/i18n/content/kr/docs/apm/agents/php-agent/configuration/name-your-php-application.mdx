[New Relic for PHP](/docs/agents/php-agent/getting-started/introduction-new-relic-php) 는 New Relic에 보고되고 표시되는 애플리케이션 이름을 변경하는 여러 방법을 제공합니다. 이 문서에서는 이러한 이름 지정 옵션에 대한 개요를 제공합니다.

APM 앱 이름 지정(PHP 전용 아님)에 대한 일반적인 개요는 애플리케이션 이름 지정을 참조 [하세요](/docs/agents/manage-apm-agents/app-naming/name-your-application) .

## 앱 이름 설정 또는 변경 [#naming-options]

New Relic은 이름으로 애플리케이션을 구별합니다. 응용 프로그램이 보고하는 이름을 변경하면 New Relic은 새 응용 프로그램 엔터티를 만들고 새 엔터티와 이전 엔터티 간의 데이터 연속성을 끊습니다. 데이터 연속성을 손상시키지 않고 애플리케이션의 이름을 변경하려면 UI alias-change 옵션을 사용하십시오.

New Relic에서 PHP 애플리케이션의 이름을 변경하는 방법은 다음과 같습니다.

* [`newrelic.appname` 을 사용하여 `newrelic.ini` 파일 구성](/docs/agents/php-agent/configuration/php-agent-configuration#inivar-appname) : 가장 일반적으로 사용되는 권장 옵션입니다.
* [디렉토리별 INI 설정 설정](/docs/agents/php-agent/configuration/php-directory-ini-settings) : 하나의 웹 서버가 여러 애플리케이션을 제공하고 이에 대해 다른 설정이나 이름을 사용하려는 경우에 유용합니다. (PHP-FPM에서 [Apache](/docs/agents/php-agent/configuration/php-directory-ini-settings#perdir-apache) , [PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#php-fpm_per-dir) 및 [NGINX에](/docs/agents/php-agent/configuration/php-directory-ini-settings#PHP-FPM_nginix) 대한 절차가 문서화되어 있습니다.)
* [Change alias via UI](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-alias) : 보고되는 기본 앱 이름이 아닌 UI의 별칭만 변경합니다. (이 옵션은 APM에서 데이터 연속성을 깨뜨리지 않습니다.)
* [API를 통한 변경](/docs/agents/php-agent/php-agent-api/newrelic_set_appname) : 애플리케이션 코드를 수정할 수 있는 액세스 권한이 있고 직접 계측해도 되며 이 호출의 추적 연속성 의미를 수용할 수 있는 옵션입니다.

## 여러 앱 이름 사용 [#multiple]

여러 앱 이름을 사용하여 APM에서 동일한 이름으로 여러 애플리케이션의 데이터를 집계할 수 있습니다(데이터 "롤업"이라고도 함). 여러 앱 이름이 작동하는 방식에 대한 자세한 내용은 여러 앱 이름 [사용](/docs/agents/manage-apm-agents/app-naming/use-multiple-names-app) 을 참조하세요.

PHP의 경우 최대 3개의 애플리케이션 이름을 설정할 수 있습니다. 기본 응용 프로그램 이름은 첫 번째 이름이고 두 번째 및 세 번째 이름은 보다 일반적인 데이터 집계 범주에 사용됩니다.
