New Relic으로 바로 PHP 애플리케이션 모니터링을 시작하고 싶으십니까?이 가이드를 사용하여 애플리케이션 데이터를 New Relic으로 보내기 시작하는 가장 좋은 경로를 찾으십시오.

시작하기 전에 자세히 알아보시겠습니까?[APM](/docs/apm/new-relic-apm/getting-started/introduction-apm) 소개 및 [PHP용 New Relic 소개](/docs/apm/agents/php-agent/getting-started/introduction-new-relic-php) 를 시작하는 것이 좋습니다.

## 무엇을 모니터링하시겠습니까? [#what]

모니터링 대상에 따라 PHP 에이전트를 설치할 수 있는 몇 가지 경로가 있습니다.

<CollapserGroup>
  <Collapser id="docker" title="Docker 컨테이너에 배포된 앱">
    PHP 에이전트를 사용하여 Docker 앱을 계측하기 위한 몇 가지 옵션이 있습니다.

    * 데몬과 PHP 에이전트를 다른 컨테이너에 설치하려면 [안내 설치를](https://one.newrelic.com/marketplace/install-data-source?account=4355438&state=b92c9282-c3e5-cacc-95c8-f70051e7b57c) 통해 설치하세요. 여전히 데몬을 생성해야 하는 경우 [도커 및 기타 컨테이너 환경](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#daemon-container)에서 추가 단계를 따르세요.
    * 동일한 컨테이너에 에이전트와 데몬을 설정하려면 [다음 단계](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#install-same-container) 를 따르십시오.여기에는 PHP 에이전트용 [tar 아카이브](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file/) 다운로드가 포함됩니다.
  </Collapser>

  <Collapser id="redhat-centos" title="RedHat, CentOS 또는 AWS Linux와 함께 배포된 앱">
    * 저희 [안내 설치는](https://one.newrelic.com/marketplace/install-data-source?account=4355438&state=b92c9282-c3e5-cacc-95c8-f70051e7b57c) 매뉴얼 설치와 마찬가지로 에이전트를 설치하기 위해 `yum` 사용합니다. 안내 설치는 설치 과정의 어느 단계에 있는지에 대한 가시성과 확실성을 높이고 싶을 때 좋은 옵션입니다.
    * [수동 설치](/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos#manager-tarball) 는 동일한 프로세스를 안내하지만 명령줄에서 모든 단계를 관리할 수 있습니다.
  </Collapser>

  <Collapser id="debian-ubuntu" title="Debian 또는 Ubuntu와 함께 배포된 앱">
    * 저희 [안내 설치는](https://one.newrelic.com/marketplace/install-data-source?account=4355438&state=b92c9282-c3e5-cacc-95c8-f70051e7b57c) 매뉴얼 설치와 마찬가지로 에이전트를 설치하기 위해 `apt` 사용합니다. 안내 설치는 설치 과정의 어느 단계에 있는지에 대한 가시성과 확실성을 높이고 싶을 때 좋은 옵션입니다.
    * [수동 설치](/docs/apm/agents/php-agent/installation/php-agent-installation-ubuntu-debian) 는 동일한 프로세스를 안내하지만 명령줄에서 모든 단계를 관리할 수 있습니다.
  </Collapser>

  <Collapser id="tar" title="다른 Unix 버전과 함께 배포된 앱(tar 아카이브 사용)">
    다른 버전의 Unix를 사용하는 경우 [PHP tar 아카이브](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file) 를 사용하여 에이전트를 설치하십시오.
  </Collapser>

  <Collapser id="hosting-providers" title="AWS, Heroku, Magento 또는 기타 호스팅 제공업체에서 호스팅되는 앱">
    New Relic과의 파트너십을 통해 웹 앱이 타사 제공업체에서 호스팅되는 경우 PHP 에이전트를 실행할 수 있습니다.

    * AWS 호스팅 앱의 경우 [Amazon CloudWatch Metric Streams 통합](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup) 을 설치합니다.New Relic 및 AWS 통합에 대해 알아보려면 AWS 통합 [소개 를](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations/) 읽으십시오.
    * Heroku에서 호스팅하는 PHP 앱의 경우 [New Relic 추가 기능을](/docs/apm/agents/php-agent/advanced-installation/php-agent-heroku/#installing) 설치합니다.
    * Magento의 판매자인 경우 [다음 단계](/docs/accounts/install-new-relic/partner-based-installation/log-install-new-relic-partners/#magento) 에 따라 Magento 설치에 PHP 에이전트를 배포하십시오.
  </Collapser>
</CollapserGroup>
