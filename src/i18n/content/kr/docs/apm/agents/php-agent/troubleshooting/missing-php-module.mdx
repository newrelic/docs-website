## 문제

**newrelic** 모듈 섹션은 `phpinfo()` 에 표시되지 않습니다.

## 해결책

`phpinfo()` 페이지에서 누락된 `newrelic` 모듈 설명 섹션은 **`newrelic.so`** 파일이 확장 디렉토리에 설치되지 않았을 수 있음을 나타냅니다. New Relic PHP 모듈이 존재하는지 확인하려면:

1. [`phpinfo()` 페이지](/docs/php/php-kb-103) 에서 **PHP Core** 섹션을 찾습니다.
2. PHP 확장이 로드되는 디렉토리를 식별하려면 `extension_dir` 설정의 로컬 값을 찾으십시오.
3. `newrelic.so` 이 `extension_dir` 에 의해 지정된 디렉토리에 있는지 확인하십시오.

확장 디렉토리에 `newrelic.so` 파일이 없으면 다음 중 하나를 수행합니다.

* New Relic의 [비표준 PHP 설치 절차를](/docs/php/php-agent-installation-non-standard-php) 따르십시오.

  또는

* 32비트의 경우 `/usr/lib/newrelic-php5/agent/x32` 또는 64비트의 경우 `/usr/lib/newrelic-php5/agent/x64` 에서 올바른 버전으로 수동으로 연결합니다. 또는 tar 파일에서 설치한 경우 tar 파일을 추출한 **에이전트** 디렉터리에서 복사하거나 링크합니다.
