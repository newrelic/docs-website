New Relic PHP 에이전트를 제거하는 방법을 배우려면 계속 읽으십시오. 에이전트를 일시적으로 비활성화하는 방법에 대한 지침은 에이전트 [비활성화](/docs/agents/manage-apm-agents/installation/disable-apm-agent#php) 를 참조하십시오.

이 문서에서는 다음 시스템에서 [New Relic PHP 에이전트](/docs/agents/php-agent/getting-started/new-relic-php) 를 제거하는 방법을 설명합니다.

* 레드햇과 센트OS
* 우분투와 데비안
* Tar 아카이브( [지원되는 모든 시스템](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems) )

## 에이전트 제거 [#install-overview]

PHP 애플리케이션에서 New Relic을 제거할 때 New Relic 에이전트를 제거하고 연결된 New Relic 구성 파일도 제거해야 합니다.

시스템에서 PHP 에이전트를 제거하려면:

1. 시스템에서 에이전트 파일을 제거합니다.

   <CollapserGroup>
     <Collapser
       id="remove-redhat-centos"
       title="레드햇 또는 센트OS"
     >
       옵션 없이 `newrelic-install` 스크립트를 실행하고 스크립트를 호출할 때 기본 메뉴에서 제거 옵션을 선택합니다.
     </Collapser>

     <Collapser
       id="remove-ubuntu-debian"
       title="우분투 또는 데비안"
     >
       `apt-get` 을 사용하여 에이전트를 제거합니다.

       ```bash
       apt-get remove newrelic-php5
       ```
     </Collapser>
   </CollapserGroup>

2. 나머지 New Relic 구성 파일을 제거하십시오. `purge` 옵션과 함께 `newrelic-install` 또는 `apt-get remove` 명령을 사용합니다.

   ```bash
   newrelic-install purge
   ```

   또는

   ```bash
   apt-get remove --purge newrelic
   ```

3. 패키지 관리자를 사용하는 경우 해당 관리자와 관련된 파일을 제거합니다.

   <CollapserGroup>
     <Collapser
       id="uninstall-redhat-centos"
       title="레드햇 또는 센트OS"
     >
       `yum` 사용:

       ```bash
       sudo yum remove newrelic-php5
       ```

       **`rpm` 사용(32비트)** :

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (YOUR VERSION #)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```

       **`rpm` (64비트)** 사용:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (VERSION NUMBER)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="uninstall-ubuntu-debian"
       title="우분투 또는 데비안"
     >
       `apt-get` 사용:

       ```bash
       sudo apt-get purge newrelic-php5
       ```

       **`dpkg` 사용(32비트)** :

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_i386.deb newrelic-php5_X.X.X.X_i386.deb
       ```

       **`dpkg` (64비트)** 사용:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_amd64.deb newrelic-php5_X.X.X.X_amd64.deb
       ```
     </Collapser>
   </CollapserGroup>

4. 웹 서버(Apache, Nginx, PHP-FPM 등)를 다시 시작합니다.
