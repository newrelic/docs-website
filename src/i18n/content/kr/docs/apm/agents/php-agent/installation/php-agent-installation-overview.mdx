## PHP 구성 요소 [#components]

[PHP 에이전트](/docs/agents/php-agent/getting-started/new-relic-php) 는 두 가지 기본 구성 요소로 구성됩니다.

* 애플리케이션에서 데이터를 수집하는 PHP 확장
* New Relic에 데이터를 전송하는 로컬 프록시 데몬

웹 서버, 운영 체제, 시스템 권한, PHP 설치 자체, 패키지 관리자 등을 포함한 시스템 구성 구성 요소는 에이전트가 그 일부이기 때문에 설치 및 구성 중에 모두 중요한 요소입니다.

지원되는 PHP 릴리스, 프레임워크 및 기타 시스템 요구 사항에 대한 자세한 내용은 [PHP 호환성 및 요구](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) 사항을 참조하십시오.

## 에이전트 설치 [#install-overview]

이 일반 프로세스에 따라 시스템에 에이전트를 설치하십시오. 에이전트 설치 패키지 또는 tar 아카이브에는 일부 설치 작업을 자동화하는 대화형 [**`newrelic-install`** 스크립트](/docs/agents/php-agent/installation/newrelic-install-script) 가 포함되어 있습니다.

<Callout variant="tip">
  애플리케이션이 자주 비활성 상태인 경우 애플리케이션 또는 사이트와 상호 작용하여 일부 데이터를 생성하는 것이 좋습니다. 이렇게 하면 에이전트가 처음에 연결되는지 확인하는 데 도움이 됩니다.
</Callout>

1. 아직 없는 경우 [새 리릭 계정을 작성하십시오](https://newrelic.com/signup). 무료입니다. 영원히요.

2. 시스템이 적절한 [시스템 권한](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#permissions) 을 포함하여 [에이전트의 요구 사항](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) 을 충족하는지 확인합니다.

3. **계정 설정**에서 <InlinePopover type="licenseKey"/>정보.

4. 시스템에 에이전트 패키지 또는 tar 아카이브를 설치합니다.

   <CollapserGroup>
     <Collapser
       id="install-redhat-centos"
       title="레드햇 또는 센트OS"
     >
       RedHat 또는 CentOS에 에이전트를 설치하는 가장 일반적인 방법은 설치 패키지( **.rpm** )와 패키지 관리자를 사용하는 것입니다. [패키지 관리자](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#tell-rpm) 에게 New Relic 저장소에서 가져오도록 지시하거나 New Relic 웹 사이트에서 직접 [**.rpm** 파일을 다운로드](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#download-rpm) 할 수 있습니다.

       원하는 패키지 관리자( `yum` 또는 `rpm` )로 [패키지를 설치](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos) 한 다음 **newrelic-install** 스크립트를 실행합니다.
     </Collapser>

     <Collapser
       id="intall-ubuntu-debian"
       title="우분투 또는 데비안"
     >
       `apt-get` 또는 `dpkg` 명령을 사용 [하여 패키지를 설치](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian) 합니다. 일반적으로 **newrelic-install** 스크립트를 실행할 필요는 없습니다.
     </Collapser>

     <Collapser
       id="install-tar"
       title="타르 아카이브"
     >
       **.tar** 아카이브 방법은 [지원되는 모든 운영 체제](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems) 에서 사용할 수 있는 가장 일반적인 방법입니다. 다음 상황 중 하나에 해당하는 경우 시스템용 [압축 tar 아카이브를 다운로드하십시오](/docs/agents/php-agent/installation/php-agent-installation-tar-file#download) .

       * 귀하의 운영 체제는 여기에 나열되지 않습니다.

       * 패키지 관리자를 사용하고 있지 않습니다.

       * 다른 방법은 성공하지 못합니다.

         [지원되는 플랫폼](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) 에 에이전트 [의 tar 아카이브를 설치합니다](/docs/agents/php-agent/installation/php-agent-installation-tar-file#install) .
     </Collapser>
   </CollapserGroup>

5. 기본 [응용 프로그램 이름](/docs/site/naming-your-application) 을 의미 있는 이름으로 변경합니다.

6. 옵션: 다른 에이전트 [구성](/docs/agents/php-agent/configuration/php-agent-configuration) 설정을 변경하여 설치를 추가로 사용자 정의하십시오.

7. 웹 서버(Apache, Nginx, PHP-FPM 등)를 다시 시작합니다.

8. **권장 사항:** PHP 에이전트가 시작되도록 하려면, 특히 애플리케이션에 자주 활동하지 않는 경우 몇 초 동안 앱을 사용하여 일부 데이터를 생성하십시오.

9. 애플리케이션이 New Relic에 데이터를 보낼 때까지 몇 분 정도 기다립니다.

그런 다음 [New Relic UI 에서 앱의 성능을 확인하세요](/docs/apm/applications-menu/monitoring/apm-overview-page) .

## APM 및 인프라 데이터에 대한 로그 보기 [#logs-context]

또한 로그와 애플리케이션의 데이터를 함께 가져와 더 쉽고 빠르게 문제를 해결할 수 있습니다. [컨텍스트의 로그를](/docs/logs/logs-context/configure-logs-context-php/) 사용하면 앱의 UI에서 직접 오류 및 추적과 관련된 로그 메시지를 볼 수 있습니다. Kubernetes 클러스터와 같은 [인프라 데이터](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/) 컨텍스트에서 로그를 볼 수도 있습니다. 다른 UI 페이지로 전환할 필요가 없습니다.
