우리의 PHP 에이전트는 귀하의 코드를 자동으로 분석하여 모델링을 시작할 수 있습니다. x86_64에 Amazon Linux 2, 또는 CentOS를 사용하여 뉴렐릭의 용 PHP 에이전트를 설치하려면 APM RedHat 다음 절차를 따르세요. aarch64 아키텍처에 설치하려면 [tar 파일](/docs/php/php-agent-installation-tar-files) 방법을 사용하세요. <DNT>**Exception:**</DNT> 이전 버전이 설치되어 있는 경우 대신 [에이전트를 업그레이드하세요](/docs/agents/php-agent/installation/upgrading-php-agent) .

## 에이전트 설치 [#install]

New Relic의 PHP 에이전트에 대한 패키지 이름은 PHP 5를 참조하지만 패키지는 [지원되는 모든 PHP 버전](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release) 에서 작동합니다.

1. <InlinePopover type="licenseKey"/>에 액세스할 수 있는지 확인하세요.

2. 다음 방법 중 하나를 사용하여 설치 패키지를 가져옵니다.

   <CollapserGroup>
     <Collapser
       id="tell-rpm"
       title={<>패키지 관리자( <InlineCode>
         rpm
       </InlineCode> )에게 New Relic 저장소에 대해 알립니다.</>}
     >
       ```bash
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-3.noarch.rpm
       ```
     </Collapser>

     <Collapser
       id="download-rpm"
       title="New Relic에서 rpm 파일 다운로드"
     >
       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```
       * ```
         newrelic-daemon-X.X.X.X-1.x86_64.rpm
         ```
       * ```
         newrelic-php5-X.X.X.X-1.x86_64.rpm
         ```
     </Collapser>
   </CollapserGroup>

3. 선호하는 패키지 관리자를 사용하여 에이전트와 데몬을 설치합니다.

   <CollapserGroup>
     <Collapser
       id="manager-yum"
       title="냠"
     >
       ```bash
       sudo yum install newrelic-php5
       ```

       PHP용 New Relic을 처음 설치할 때 yum은 New Relic 공개 키를 수락하라는 메시지를 표시합니다. New Relic의 키 ID는 `548C16BF` 입니다.
     </Collapser>

     <Collapser
       id="manager-rpm"
       title="rpm"
     >
       다음 명령을 실행할 때 `X.X.X.X` 을 최신 [PHP 에이전트 버전 번호](/docs/release-notes/agent-release-notes/php-release-notes) 로 바꿉니다.

       ```bash
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="manager-tarball"
       title="타르볼"
     >
       <DNT>**yum**</DNT> 및 <DNT>**rpm**</DNT> 이 호스트 구성에서 작동하지 않으면 [바이너리 tarball에서 설치하세요](/docs/php/php-agent-installation-tar-files).
     </Collapser>
   </CollapserGroup>

4. `newrelic-install` 스크립트를 실행하고 지침을 따릅니다.

   ```bash
   sudo newrelic-install install
   ```

5. 웹 서버(Apache, NGINX, PHP-FPM 등)를 다시 시작합니다.

6. 애플리케이션에 대한 트래픽을 생성하고 New Relic에 데이터를 보낼 때까지 몇 분 정도 기다립니다. 그런 다음 [UI에서 앱의 성능을 확인합니다](/docs/apm/applications-menu/monitoring/apm-overview-page) .
