## 문제

New Relic for PHP 설치에 문제가 발생했지만 원인이 무엇인지 확실하지 않습니다.

## 해결책

New Relic 로그를 모니터링하면 문제 진단에 도움이 될 수 있습니다. New Relic PHP 에이전트와 데몬에는 자체 로그가 있습니다. 로그를 모니터링하려면:

1. 두 로그에서 콘텐츠를 모두 확인합니다.

   * <DNT>
       **[Default agent log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-logfile)**
     </DNT>

     : `/var/log/newrelic/php_agent.log`

   * <DNT>
       **[Default daemon log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)**
     </DNT>

     : `/var/log/newrelic/newrelic-daemon.log`

2. 에이전트의 로그 수준을 `verbosedebug` 으로 설정하고 데몬의 로그 수준을 `debug` 로 설정합니다. 자세한 내용은:

   * [에이전트](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)

     <DNT>
       [**loglevel**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
     </DNT>

     [(를) 다음으로 설정](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)

     <DNT>
       [**verbosedebug**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
     </DNT>

   * [데몬](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

     <DNT>
       [**loglevel**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)
     </DNT>

     [을(를) 다음으로 설정](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

     <DNT>
       [**debug**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)
     </DNT>

3. 앱을 다시 시작합니다.

4. 적절한 값으로 설정된 로그 수준으로 5-10분의 로그 시간을 수집합니다.

5. 로그를 검토하고 정보를 사용하여 문제를 진단하십시오.

6. 문제 해결을 마치면 로그 수준을 재설정하십시오. `verbosedebug` 또는 `debug` 에 보관하지 마십시오.

<Callout variant="caution">
  로그 파일의 크기를 확인하십시오. 애플리케이션에 따라 매우 빠르게 성장할 수 있습니다.
</Callout>
