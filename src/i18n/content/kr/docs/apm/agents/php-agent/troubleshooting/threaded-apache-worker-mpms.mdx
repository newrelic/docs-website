## 문제

New Relic PHP 에이전트는 스레딩과 관련된 Apache MPM(다중 처리 모델)을 지원하지 않습니다.

## 해결책

PHP 언어 개발 팀 [은 Apache와 함께 스레드 MPM을 사용하는 것을 권장하지](http://www.php.net/manual/en/install.unix.apache2.php "링크가 새 창에서 열립니다.") 않으며 유사한 안정성 이유로 PHP 에이전트에서 스레드 MPM에 대한 지원을 구현하지 않기로 결정했습니다. 여기에는 **작업자** 및 **이벤트** MPM이 포함됩니다.

기본 **비스레드 프리포크 MPM이 성능상의 이유로 귀하의 요구 사항에 적합하지 않은 경우 mod_fcgid** **및** **php-fpm** 을 사용하여 대신 FastCGI를 통해 PHP를 사용하는 것이 좋습니다.
