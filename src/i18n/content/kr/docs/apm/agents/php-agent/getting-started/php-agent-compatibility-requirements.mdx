New Relic for PHP를 [설치](/docs/apm/agents/php-agent/installation) 하기 전에 시스템이 해당 요구 사항을 충족하는지 확인하십시오.

아직 계정이 없다면 [New Relic 계정을 만드](https://newrelic.com/signup) 십시오. 영원히 무료입니다.

## PHP 릴리스 [#php-release]

New Relic은 PHP 버전 5.5, 5.6, 7.0, 7.1, 7.2, 7.3, 7.4, 8.0, 8.1을 지원합니다.

<Callout variant="important">
  호환성 참고: PHP 8.0 또는 8.1이 New Relic 에이전트와 같은 관찰 가능성 확장을 감지하면 PHP는 Just-In-Time 컴파일을 비활성화합니다.

  PHP 8.1 지원에는 Fiber가 포함되지 않습니다.
</Callout>

* [지원되는 PHP 릴리스](http://php.net/supported-versions.php) , 특히 7.4, 8.0 및 8.1을 사용하는 것이 좋습니다.
* PHP 5.1 지원은 에이전트 릴리스 4.0에서 더 이상 사용되지 않으며 릴리스 4.5에서 제거되었습니다.
* PHP 5.2 지원은 에이전트 릴리스 6.8에서 더 이상 사용되지 않으며 릴리스 7.0에서 제거되었습니다.
* PHP 5.3 및 PHP 5.4 지원은 에이전트 릴리스 [9.15](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-9150293) 에서 더 이상 사용되지 않으며 릴리스 [9.17](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-9170300/) 에서 제거되었습니다.
* PHP 버전 5.5, 5.6, 7.0, 7.1, 7.2, 7.3, 7.4에 대한 32비트 지원은 릴리스 [9.19](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-9190309) 에서 더 이상 사용되지 않습니다.
* 릴리스 [9.19](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-9190309) 는 ZTS 빌드를 지원하는 마지막 릴리스입니다. 앞으로 ZTS 빌드가 제공되지 않을 수 있으며 지원이 코드베이스에서 완전히 제거될 수 있습니다.

## 권한 [#permissions]

* 설치: 대부분의 설치에는 루트 액세스 [가 필요](/docs/agents/php-agent/troubleshooting/determing-permissions-requirements) 합니다.
* 실행 중: 루트 액세스 [가 필요하지 않습니다](/docs/agents/php-agent/troubleshooting/determing-permissions-requirements) .

## 라이센스 키 [#license_key]

모든 설치에는 New Relic [라이센스 키](/docs/subscriptions/license-key) 가 필요합니다. 이것은 New Relic이 귀하의 계정에 [가입할](https://newrelic.com/signup) 때 제공하는 40자의 16진수 문자열입니다. 자세한 내용은 PHP INI 파일의 PHP [설치 스크립트](/docs/php/the-newrelic-install-script) 및 [라이선스 키 설정](/docs/php/php-agent-phpini-settings#inivar-license) 을 참조하십시오.

## 기타 APM 소프트웨어 [#other-monitoring-software]

귀하의 애플리케이션이 당사 에이전트 이외의 다른 애플리케이션 성능 모니터링(APM) 소프트웨어를 사용하는 경우 당사 에이전트가 올바르게 작동한다고 보장할 수 없으며 기술 지원을 제공할 수 없습니다. 자세한 내용은 [다른 모니터링 소프트웨어 사용 시 오류를](/docs/apm/new-relic-apm/troubleshooting/errors-while-using-new-relic-apm-alongside-other-apm-software) 참조하십시오.

* `newrelic.ini` }에서 `newrelic.datastore_tracer.instance_reporting.enabled` 을 `false` 로 설정합니다.
* `newrelic.datastore_tracer.database_name_reporting.enabled = false` 을 사용하여 데이터베이스 이름을 생략합니다.

현재 New Relic 에이전트에 대해 나열되지 않은 데이터 저장소에서 인스턴스 수준 정보를 요청하려면 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으십시오.

## 보안 요구 사항 [#security]

[데이터 수집을 위한 표준 보안 수단으로](/docs/accounts-partnerships/accounts/security/data-security) 앱 서버는 SHA-2(256비트)를 지원해야 합니다. SHA-1은 지원되지 않습니다.
