## 문제

New Relic은 PHP 에이전트를 설치한 후 웹 서버를 다시 시작한 후 몇 분 이내에 데이터를 보고하지 않습니다.

## 해결책

`phpinfo()` 을 사용하여 에이전트가 성공적으로 설치되었는지 확인합니다.

1. 다음을 포함하는 웹사이트 내에 간단한 `.php` 웹페이지를 만듭니다.

   ```php
   <?php phpinfo(); ?>
   ```

2. 웹 브라우저에서 이 페이지로 이동하고 모듈의 알파벳순 목록을 스크롤하여 `newrelic` 을(를) 찾습니다. 이 목록에 `newrelic` 이 표시되지 않으면 비표준 버전의 PHP를 사용 중이거나 New Relic을 설치한 후 웹 서버(또는 다른 PHP 호스트)를 다시 시작하지 않은 것입니다.

3. 웹 서버를 다시 시작하여 모듈 목록에 `newrelic` 이 나타나는지 확인하십시오.

4. 모듈 목록에 `newrelic` 이 나타나지 않으면 [확장 디렉토리를 확인하여](/docs/php/php-kb-104) [PHP의 비표준 버전이](/docs/php/php-agent-installation-non-standard-php) 있는지 확인하십시오.

5. 모듈 목록에 `newrelic` 가 표시되면 `license.key` 항목에 모듈의 처음과 마지막 몇 자가 표시되는지 확인하십시오.<InlinePopover type="licenseKey"/>

6. `license.key` 이(가) 표시되지 않으면 [로드된 구성 파일 디렉토리를 확인하십시오.](/docs/agents/php-agent/troubleshooting/checking-loaded-configuration-files-directory)
