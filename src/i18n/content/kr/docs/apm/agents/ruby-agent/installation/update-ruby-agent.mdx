New Relic의 최신 기능, 향상된 기능 및 중요한 보안 패치를 최대한 활용하려면 Ruby 에이전트를 최신 버전으로 업데이트하는 것이 좋습니다. 특정 에이전트 업데이트에 대한 추가 정보는 [Ruby 에이전트 릴리스 정보](/docs/release-notes/agent-release-notes/ruby-release-notes) 를 참조하십시오.

## Ruby 에이전트 버전 확인 [#determine-versions]

앱이 현재 사용하는 Ruby 에이전트 버전을 확인하려면 다음을 수행하십시오.

1. New Relic의[계정 드롭다운](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#account-dropdown) 에서 **Account settings > Connected agents** 를 선택합니다.
2. 선택 사항: **가장 오래된 에이전트 버전** 을 기준으로 정렬합니다.

## 에이전트 업데이트 [#Updating_the_Agent]

Ruby 에이전트를 업데이트하려면:

* 번들러 사용: 다음 명령을 실행합니다.

  ```
  bundle update newrelic_rpm
  ```

* 번들러를 사용하지 않음: 다음 명령을 실행합니다.

  ```
  gem update newrelic_rpm
  ```

## 악기 JRuby [#jruby]

올바른 계측을 위해:

* JRuby 버전 1.7 이상에서는 New Relic Ruby 에이전트 버전 [3.6.8.164 - 3.18.1 을](/docs/releases/ruby) 사용하세요.
* JRuby 버전 9 이상에서는 New Relic Ruby 에이전트 버전 [3.6.8.164 이상](/docs/releases/ruby) 을 사용합니다.

그렇지 않고 애플리케이션에 트랜잭션이 없고 JRuby를 사용하는 경우 CPU 메트릭을 볼 수 없습니다. 이는 `Process.times` 이(가) CPU 경과 시간 대신 벽시계 시간을 반환한 JRuby의 버그 때문이었습니다. 이 버그는 New Relic의 Ruby 에이전트 3.6.8.164 및 JRuby 1.7.0에서 수정되었습니다.

## Ruby 에이전트 버전이 지원되지 않음 [#eol-versions]

**수명 종료 알림:** 2015년 1월 26일부터 New Relic은 **3.9.6** 이전 Ruby 에이전트 버전의 데이터를 더 이상 허용하지 않습니다.

이러한 에이전트 버전은 New Relic의 데이터 수집 서비스와 통신할 때 오래된 프로토콜을 사용합니다. 또한, 이러한 버전 중 다수에는 민감한 데이터를 New Relic 수집기로 잘못 보낼 수 있는 잠재적인 보안 문제가 포함되어 있습니다.
