## 문제

[New Relic Ruby 에이전트를 설치하고](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation) 약간의 트래픽을 생성하고 5분을 기다린 후 New Relic UI에 데이터가 표시되지 않습니다.

## 해결책

누락된 데이터 문제를 해결하려면:

1. 구성 파일을 확인하십시오.

   * 구성 파일이 애플리케이션 디렉토리 내의 `config/newrelic.yml` 에 설치되어 있는지 확인합니다.

   * 구성 파일이 유효한 YML인지 확인하십시오.

   * <InlinePopover type="licenseKey"/>

     이 미리 입력된 이 파일의 깨끗한 사본을 다운로드하려면

     <DNT>
       **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (user menu) > Account settings > Download a clean configuration file**
     </DNT>

     로 이동하세요.

2. 웹 서버를 다시 시작하십시오.

3. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 사용하여 문제를 자동으로 식별해 보십시오.

4. 애플리케이션 서버 프로세스를 다시 시작할 때 표시되는 오류 또는 경고가 있는지 `log/newrelic_agent.log` 에 있는 에이전트의 로그 파일을 검사하십시오. 이 로그 파일이 없거나 기록되지 않는 경우 로그 [파일 없음](/docs/agents/ruby-agent/troubleshooting/no-log-file-ruby) 을 참조하십시오.

5. `INFO : Reporting to:` 다음에 URL이 오는 항목에 대한 New Relic 에이전트 로그를 확인하십시오. URL이 New Relic UI에서 보고 있는 계정 및 애플리케이션과 일치하는지 확인하십시오.

6. New Relic 에이전트가 배포할 환경에서 시작하도록 구성되어 있는지 확인합니다. 자세한 내용 [은 Ruby 에이전트 시작 시간 제어를](/docs/agents/ruby-agent/troubleshooting/controlling-when-ruby-agent-starts) 참조하십시오.

7. `info` 수준에서 명백한 오류가 없으면 [자세한 디버그 로그를 생성](/docs/agents/ruby-agent/troubleshooting/generating-logs-troubleshooting-ruby) 합니다.

8. <DNT>**newrelic_rpm**</DNT> gem의 [최신 버전으로 업데이트](/docs/agents/ruby-agent/installation-configuration/updating-deprecated-ruby-agents#update) 해 보세요.

9. Rails가 아닌 다른 것을 사용하는 경우 개발 또는 프로덕션과 같이 `newrelic.yml` 파일에 나타나는 환경으로 `RACK_ENV` 을 설정합니다.
