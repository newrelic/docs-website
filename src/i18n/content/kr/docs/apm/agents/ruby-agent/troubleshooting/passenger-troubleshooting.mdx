Ruby 에이전트 및 Passenger를 사용할 때 일반적인 문제에 대한 다음 문제 해결 팁을 읽으십시오.

<CollapserGroup>
  <Collapser
    id="agent"
    title="에이전트 다시 시작"
  >
    승객 `tmp/restart.txt` 파일을 편집하기만 하면 새 에이전트를 선택하지 않습니다. 에이전트를 업데이트하고 Passenger를 다시 시작할 때 변경 사항을 적용하려면 Apache를 하드 다시 시작해야 합니다.
  </Collapser>

  <Collapser
    id="logs"
    title="누락된 로그 파일"
  >
    승객의 경우 로그 파일은 `newrelic_agent.passenger.log` 입니다. 권장 사항: Apache 사용자의 경우 전체 `/log` 디렉터리에 대한 쓰기 권한을 제공하십시오.
  </Collapser>

  <Collapser
    id="developer_mode"
    title="개발자 모드"
  >
    개발자 모드에서 Passenger를 사용할 때 때때로 요청 목록이 사라지거나 일부 요청이 누락된 것처럼 보일 수 있습니다. 또한 요청을 선택할 때 추적을 사용할 수 없다는 오류 메시지가 표시되어 다시 시작했을 수 있습니다.

    이는 Passenger가 둘 이상의 핸들러 프로세스를 생성할 때 발생합니다. 트랜잭션 추적은 프로세스 중 하나에 들어가고 다른 프로세스에 연결하면 볼 수 없습니다.

    해결 방법은 [PassengerMaxPoolSize](http://www.modrails.com/documentation/Users%20guide%20Apache.html#_passengermaxpoolsize_lt_integer_gt) 매개변수를 사용하여 최대 인스턴스 수를 1로 설정하는 것입니다. 또한 여러 번 새로 고침하면 요청을 다시 받을 수 있는 좋은 기회가 있습니다.
  </Collapser>
</CollapserGroup>
