자세한 디버그 로깅은 표준 Ruby 에이전트 설치 문제를 해결하는 데 도움이 될 수 있습니다.

<Callout variant="important">
  Heroku를 사용하는 경우 [Ruby 에이전트 및 Heroku](/docs/agents/ruby-agent/miscellaneous/ruby-agent-heroku#logging) 를 참조하세요.
</Callout>

## 로그 파일 생성 [#generating]

자세한 New Relic for Ruby 로그 파일을 생성하려면:

1. 편집기에서 `newrelic.yml` 파일을 엽니다.

2. [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level) 을 `debug` 로 변경하고 파일을 저장합니다. 들여쓰기를 수정하지 않았는지 확인하십시오.

   <Callout variant="important">
     `debug` 매우 자세한 로깅 수준입니다. 디버그 수준 로그를 생성한 후 이전 `log_level` 로 돌아가야 합니다.
   </Callout>

3. 웹 애플리케이션을 다시 시작합니다.

4. 앱에 약간의 트래픽을 생성하고 약 5분 동안 실행합니다.

5. 로그 파일에 오류가 있는지 검사하십시오. 기본적으로 로그 파일은 앱의 로그 폴더에 있으며 이름은 `newrelic_agent.log` 입니다.

6. 로그 파일을 New Relic Support에 보내는 경우 `newrelic.yml` , Gemfile 및 `Gemfile.lock` 과 함께 로그 파일을 지원 티켓에 첨부하십시오.

7. `newrelic.yml` 을 다시 수정하고 [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level) 을 이전 설정으로 변경합니다. 파일을 저장합니다.
