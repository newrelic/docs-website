[뉴렐릭의 루비 에이전트를 설치하기](/docs/agents/ruby-agent/installation-and-configuration/ruby-agent-installation) 전에 시스템이 호환 가능한 운영 시스템, 보안 프로토콜, 루비 버전 및 지원되는 프레임워크/라이브러리에 대한 요구 사항을 충족하는지 확인하세요.

아직 계정이 없다면 먼저 [New Relic 계정을 만드세요](https://newrelic.com/signup) . 영원히 무료입니다.

## 운영체제 [#os]

루비 에이전트는 Linux, Solaris, FreeBSD, macOS 등 UNIX와 유사한 운영 시스템을 지원합니다. Windows 공식적으로 운영 환경을 지원하지 않지만 개발에는 작동할 수 있습니다.

## 보안 요구 사항 [#security]

* **데이터 수집:** [데이터 수집을 위한 표준 보안 조치](/docs/accounts-partnerships/accounts/security/data-security) 로, 앱 서버는 뉴렐릭과의 SSL 통신을 위해 SHA-2(256비트)를 지원해야 합니다. SHA-1은 지원되지 않습니다.
* **TLS:** 뉴렐릭은 에이전트에서 들어오는 모든 연결에 TLS 1.2 이상이 필요합니다.

## 지원되는 버전에 대한 참고 사항 [#notes\_on\_supported\_versions][#notes_on_supported_versions]

* 해당 에이전트는 명시적으로 나열된 것보다 더 다양한 버전(이전 또는 최신)과 호환될 수 있습니다. 그러나 나열된 버전은 가장 엄격한 테스트를 거쳤습니다.
* 새로 지원되는 버전이나 지원 중단에 대한 최신 정보는 항상 [루비 에이전트 릴리스 노트를](/docs/release-notes/agent-release-notes/ruby-release-notes) 참조하세요.
* 라이브러리가 나열되지 않은 경우, 리소스 *인* 기본 기술(예: Rack 에 구축된 프레임워크 또는 표준 데이터베이스 어댑터를 사용하는 ORM)에 의존하는 경우 여전히 수용체일 수 있습니다. 중요한 미등록 라이브러리의 경우 사용자 [정의 측정을](/docs/apm/agents/ruby-agent/api-guides/ruby-custom-instrumentation) 탐색하거나 [기능 요청을 제출할](https://github.com/newrelic/newrelic-ruby-agent/issues/new?template=feature_request.md) 수 있습니다.

여기에 언급되지 않은 특정 버전이나 라이브러리에 대한 질문이 있는 경우 [Explorers Hub 커뮤니티 포럼을](https://forum.newrelic.com/) 방문하세요.

## 기타 APM 소프트웨어 [#other\_monitoring\_software][#other_monitoring_software]

귀하의 애플리케이션이 당사 에이전트 외에 다른 APM(애플리케이션 성능 모니터링) 소프트웨어를 사용하는 경우 당사는 당사 에이전트가 올바르게 작동한다고 보장할 수 없으며 기술 지원을 제공할 수 없습니다. 자세한 내용은 [다른 모니터링 소프트웨어 사용 시 오류를](/docs/apm/new-relic-apm/troubleshooting/errors-while-using-new-relic-apm-alongside-other-apm-software) 참조하십시오.
