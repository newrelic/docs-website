## 문제

웹 애플리케이션에서 [Solr platform](http://lucene.apache.org/solr/) \[Solr 플랫폼을] 사용하고 있지만 Solr 데이터가 <InlinePopover type="apm"/>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (Java 앱 선택) > Monitor** 앱의 UI.

## 해결책

Solr가 올바르게 [구성](/docs/agents/java-agent/getting-started/compatibility-requirements-java-agent#other) 되고 데이터가 New Relic에 보고되는지 확인하려면 해당하는 경우 다음을 수행하십시오.

* **JMX:** `<jmx/>` 태그를 사용하여 `solrconfig.xml` 의 Solr 구성에서 JMX가 활성화되어 있는지 확인합니다. `solrconfig.xml` 파일은 Solr 자체에 영향을 미치는 매개변수가 가장 많은 구성 파일입니다.
* **Jetty:** Jetty 사용자인 경우 Jetty[용 JMX를 수동으로 활성화](https://wiki.eclipse.org/Jetty/Tutorial/JMX#Enabling_JMXConnectorServer_for_Remote_Access) 하십시오.
* **Solr 메트릭:** JMX 콘솔에서 Solr 메트릭을 확인하십시오. Solr 캐시 및 업데이트는 MBean에서 가져옵니다.
