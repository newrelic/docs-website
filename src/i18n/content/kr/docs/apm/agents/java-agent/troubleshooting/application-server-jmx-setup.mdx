## 문제

APM 에서 [<DNT>**JVMs**</DNT> 페이지](/docs/agents/java-agent/features/jvm-metrics-page) 의 <DNT>**Threads**</DNT> 탭에는 활성, 유휴 및 총 스레드와 같이 제작 서버에서 지원하는 다양한 JMX 지표가 표시됩니다. <DNT>**Threads**</DNT> 탭에서 지표를 보려면 기능 서버에서 JMX 보고를 활성화해야 합니다.

JMX 보고가 활성화되지 않은 경우 JMX를 활성화하기 위한 애플리케이션 서버 관련 문서 링크가 포함된 빈 <DNT>**Threads**</DNT> 탭이 표시됩니다.

## 해결책

이 문제를 해결하려면 응용 프로그램 서버에 따라 다음 섹션을 읽으십시오.

<CollapserGroup>
  <Collapser
    id="JBoss"
    title="제이보스"
  >
    JMX에 대한 로컬 지원은 기본적으로 JBoss 애플리케이션 서버에서 활성화됩니다. 기본 설정을 변경한 경우 다음을 참조하십시오.

    * [제이보스 EAP 6.0](https://access.redhat.com/site/documentation/en-US/JBoss_Enterprise_Application_Platform/6/html/Administration_and_Configuration_Guide/index.html)
    * [제이보스 AS 7.1](https://docs.jboss.org/author/display/AS71/Admin%20Guide.html#8094211_AdminGuide-JMX)
    * [제이보스 AS 6](https://docs.jboss.org/jbossas/6/Admin_Console_Guide/en-US/html/)
  </Collapser>

  <Collapser
    id="Jetty"
    title="둑"
  >
    Jetty 애플리케이션 서버에서 JMX에 대한 로컬 지원을 활성화하려면 <DNT>**etc/jetty-jmx.xml**</DNT> 파일을 사용하여 MBeanContainer를 구성하세요. 자세한 내용은 다음을 참조하세요.

    * [부두 7 및 8](http://wiki.eclipse.org/Jetty/Tutorial/JMX)
    * [제티 9](http://www.eclipse.org/jetty/documentation/current/jmx-chapter.html)
  </Collapser>

  <Collapser
    id="Glassfish"
    title="유리고기"
  >
    Glassfish 특수 서버에서는 JMX에 대한 로컬 지원이 기본적으로 활성화되어 있지만 실제로 메트릭 정보를 제공하는 MBean은 기본적으로 `OFF` 입니다. 특수 서버에서 적절한 지표를 수집하려면 다음 구성 요소를 <DNT>**Monitoring Service**</DNT> 에 설정해야 합니다.

    * [Glassfish 4.x](https://glassfish.java.net/docs/4.0/administration-guide.pdf) : 웹 컨테이너, HTTP 서비스 및 트랜잭션 서비스 구성 요소는 HIGH 모니터링 수준으로 설정되어야 합니다.
    * [Glassfish 3.1](http://docs.oracle.com/cd/E18930_01/html/821-2416/ablur.html) : 웹 컨테이너, HTTP 서비스 및 트랜잭션 서비스 구성 요소는 HIGH 모니터링 수준으로 설정되어야 합니다.
    * [Glassfish 3.0](http://docs.oracle.com/cd/E19798-01/821-1751/ablvr/index.html) : 웹 컨테이너, 스레드 풀 및 트랜잭션 서비스 구성 요소는 LOW 모니터링 수준으로 설정해야 합니다.
    * [Glassfish 2.1](http://docs.oracle.com/cd/E19879-01/820-4335/ablur/index.html) : 웹 컨테이너, 스레드 풀 및 트랜잭션 서비스 구성 요소는 LOW 모니터링 수준으로 설정해야 합니다.
  </Collapser>
