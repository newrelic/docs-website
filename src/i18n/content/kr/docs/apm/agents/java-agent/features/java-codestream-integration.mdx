코드 수준 메트릭을 통해 [New Relic CodeStream 확장을](/docs/codestream/how-use-codestream/performance-monitoring#code-level) 사용하는 개발자는 코드의 개별 메서드와 함께 IDE에서 상황에 맞게 표시된 APM 데이터를 볼 수 있습니다. 이를 통해 개발자는 코드를 작성하고 검토할 때 성능 문제를 보다 적극적으로 해결할 수 있습니다.

## 시작하기

먼저 지원되는 IDE에 New Relic CodeStream 확장을 [설치](/docs/codestream/start-here/install-codestream) 하고 로그인합니다.

<Callout variant="important">
  코드 수준 메트릭은 Java 에이전트 버전 7.10.0 이상에서 사용할 수 있으며 버전 7.11.0 이상에서는 기본적으로 활성화됩니다. 활성화 또는 비활성화 방법에 대한 지침은 [Java 에이전트 구성: 구성 파일을](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file/#code-level-metrics) 참조하십시오.
</Callout>

## 에이전트 속성

Java 에이전트는 다음 속성을 보고하고 애플리케이션에서 생성된 범위에 연결합니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        **이름**
      </th>

      <th>
        **설명**
      </th>

      <th>
        **예시**
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `code.function`
      </td>

      <td>
        계측된 함수의 이름(Java 메서드)입니다.
      </td>

      <td>
        데이터 검색
      </td>
    </tr>

    <tr>
      <td>
        `code.namespace`
      </td>

      <td>
        `code.function` 이 정의된 네임스페이스(정규화된 클래스 이름)
      </td>

      <td>
        com.example.MyService
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="important">
  API 메서드 `NewRelic.getAgent().getTransaction().startSegment()` 를 사용하여 생성된 스팬에는 이러한 속성이 포함되지 않습니다. 이러한 속성을 포함하는 범위를 만들려면 `@Trace` 으로 메서드에 주석을 달거나 XML 계측을 사용하세요.
</Callout>
