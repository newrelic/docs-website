이 문서에서는 프레임워크의 JVM에 `-javaagent` 인수를 전달하는 방법을 설명합니다. 이 설치 단계는에이전트가 앱에 포함되도록 합니다. 모든 앱 서버에 대해 `newrelic.jar` 파일의 전체 경로를 전달해야 합니다.

이 문서는 인수를 전달하는 방법에 대한 참조용일 뿐입니다. 보다 자세한 설치 절차는 [Java 에이전트 설치](/docs/agents/java-agent/installation/java-agent-manual-installation)를 참조하세요.

## ColdFusion [#Installing_on_ColdFusion][#Installing_on_ColdFusion]

ColdFusion에서 `-javaagent` 인수를 전달하려면:

1. ColdFusion 서버를 시작하고 ColdFusion 관리 콘솔로 이동합니다.

2. 왼쪽 메뉴에서 <DNT>**SERVER SETTINGS > Java and JVM**</DNT>을 선택합니다.

3. 에이전트 API를 사용하는 경우: <DNT>**ColdFusion Class Path**</DNT> 필드에서 `newrelic-api.jar`로의 경로를 지정합니다.

4. <DNT>**JVM Arguments**</DNT> 필드에 `-javaagent` 인수를 추가합니다.

   ```
   -javaagent:/full/path/to/newrelic.jar
   ```

5. <DNT>**Submit Changes**</DNT>를 선택한 다음 ColdFusion 서버를 다시 시작합니다.

## Geronimo [#Installing_on_Geronimo][#Installing_on_Geronimo]

Geronimo에서 `-javaagent` 인수를 전달하려면 시작 명령을 실행할 때 `JAVA_OPTS` 환경 변수에서 뉴렐릭 에이전트 jar를 참조하세요.

```sh
export JAVA_OPTS="$JAVA_OPTS -javaagent:/full/path/to/newrelic.jar" && geronimo run
```

## Glassfish [#Installing_on_Glassfish][#Installing_on_Glassfish]

Glassfish에서 `-javaagent` 인수를 전달하려면:

1. Glassfish 콘솔에서 <DNT>**Application Server > JVM Settings > JVM options**</DNT>를 선택합니다.

2. JVM 옵션 페이지에서 <DNT>**Add JVM option**</DNT>을 선택합니다.

3. `-javaagent` 인수에 대한 항목을 추가합니다.

   ```
   -javaagent:/full/path/to/newrelic.jar
   ```

4. Glassfish를 저장하고 다시 시작합니다.

Glassfish가 시작되지 않으면 `-javaagent` 인수가 올바르게 설정되지 않았을 수 있습니다. `domain.xml` 파일을 수정하여 서버 JVM 인수를 변경할 수 있습니다.

<Callout variant="important">
  Glassfish 2.1의 버그로 인해 부트스트랩 클래스 로더(뉴렐릭 에이전트)의 클래스는 Java 로깅 API를 사용하지 못합니다. 이는 2.1.1 이상 릴리스에서 수정된 것으로 보입니다.
</Callout>

## Grails [#Installing_on_Grails][#Installing_on_Grails]

Grails에서 `-javaagent` 인수를 전달하려면:

<CollapserGroup>
  <Collapser
    id="grails-run-app"
    title="run-app으로 패스"
  >
    1. 압축을 푼 Grails 버전으로 시작합니다.

    2. 다음 명령을 실행합니다.

       ```sh
       grails -noreloading -javaagent:/full/path/to/newrelic.jar run-app
       ```
  </Collapser>

  <Collapser
    id="grails-run-war"
    title="run-war 패스"

       ```
       grails-app/conf/BuildConfig.groovy
       ```

    2. JVM 인수 행을 추가하거나 편집합니다.

       ```ini
       grails.tomcat.jvmArgs = ["-javaagent:/full/path/to/newrelic.jar"]
       ```
  </Collapser>
</CollapserGroup>

  <Collapser

  >
    JBoss 버전 6.x EAP 및 7.0 AS 이상에 대해 도메인 모드를 사용합니다.

## Play [#Installing_on_Play][#Installing_on_Play]

Play에서 `-javaagent` 인수를 전달하려면:

<CollapserGroup>
  <Collapser
    id="play-1-2-4"
    title="Play 1.2.4"
  >
    Play 애플리케이션을 실행할 때 `-javaagent` 인수를 추가합니다.

    ```sh
    play run your_app_name -javaagent:/full/path/to/newrelic.jar
    ```
  </Collapser>

  <Collapser
    id="play-2.0"
    title="Play 2.0"
  >
    1. `start` 스크립트가 포함된 <DNT>**unzipped**</DNT> 배포로 시작합니다.

       ```sh
       play clean dist && unzip dist/*.zip
       ```

    2. Play 앱을 시작할 때 `-javaagent` 인수를 추가합니다.

       ```sh
       cd unzipped/folder; chmod a+x start; ./start -javaagent:/full/path/to/newrelic.jar
       ```
  </Collapser>

  <Collapser
    id="play-2-2"
    title="Play 2.2"
  >
    1. `start` 스크립트가 포함된 <DNT>**unzipped**</DNT> 배포로 시작합니다.

       ```sh
       play clean dist && unzip target/directory/universal/*.zip
       ```

    2. Play 앱을 시작할 때 `-J-javaagent` 인수를 추가합니다.

       ```sh
       cd unzipped/folder; ./bin/scriptname -J-javaagent:/full/path/to/newrelic.jar
       ```

       <Callout variant="tip">
         페이지 로드 타이밍을 활성화하려면 [Play 2.2를 사용한 수동 계측](/docs/agents/java-agent/instrumentation/page-load-timing-java#manual-play-2-2)을 참조하십시오.
       </Callout>
  </Collapser>

  <Collapser
    id="play-2-3-etc"
    title="Play 2.3, 2.4, 2.5"
  >
    1. `start` 스크립트가 포함된 <DNT>**unzipped**</DNT> 배포로 시작합니다.

       ```sh
       activator clean dist && unzip target/directory/universal/*.zip
       ```

    2. Play 앱을 시작할 때 `-J-javaagent` 인수를 추가합니다.

       ```sh
       cd unzipped/folder; ./bin/scriptname -J-javaagent:/full/path/to/newrelic.jar
       ```

    3. Play 2.4에서 Typesafe Activator를 사용하는 경우 다음 줄을 `build.sbt`에 추가합니다.

       ```scala
       javaOptions ++= Seq("-javaagent:/full/path/to/newrelic.jar")
       ```
  </Collapser>
</CollapserGroup>

## Resin [#Installing_on_Resin][#Installing_on_Resin]

Java 8

* 옵션 1: 서버의 `resin.properties` 파일에 있는 `jvmargs` 속성을 사용해 `javaagent` 인수를 추가합니다.
  ```properties
  jvm_args  : -javaagent:/full/path/to/newrelic.jar
  ```
* 옵션 2: `conf/resin.conf` 또는 `conf/resin.xml` 파일의 `<jvm-args>` 섹션에 추가함으로써 `-javaagent` 인수를 지정합니다.
  ```xml
  <jvm-arg>-javaagent:/full/path/to/newrelic.jar</jvm-arg>
  ```

Java 9 이상

Java 9에 도입된 모듈 시스템은 `-javaagent` 속성이 `conf/resin.conf` 또는 `conf/resin.xml` 파일에 추가되면 예외 `NoClassDefFoundError: java/sql/SQLException`이 발생할 수 있습니다. Java 9 이상을 사용하는 경우 `-javaagent` 속성이 해당 파일에 포함되어 있지 않은지 확인하세요.

Resin은 다음 옵션 중 하나를 사용하는 Java 에이전트와 함께 Java 9 이상에서 실행할 수 있습니다.

* 옵션 1: 서버의 `resin.properties` 파일에 있는 `jvmargs` 속성을 사용해 `javaagent` 인수를 추가합니다.

  ```properties
  jvm_args  : -javaagent:/full/path/to/newrelic.jar
  ```

  그러면 Resin 서버가 `./bin/resin.sh start`로 실행될 수 있습니다.

* 옵션 2: 명령줄에서 `-javaagent` 속성을 사용하여 Resin Jar를 실행합니다.

  ```sh
  java -javaagent:/path/to/newrelic.jar -jar lib/resin.jar start
  ```

<Callout variant="important">
  Java 9 이상을 사용하는 경우 에이전트가 Windows에서 Resin과 함께 작동하지 않습니다.
</Callout>

## Spring Boot

Spring Boot에서 `-javaagent` 인수를 전달하려면 이를 앱을 시작하는 명령줄에 추가합니다. `-jar` 인수 앞에 추가해야 합니다.

```sh
java -javaagent:/full/path/to/newrelic.jar -jar app.jar
```

## Tanuki Wrapper

Tanuki Wrapper에서 `-javaagent` 인수를 전달하려면 래퍼 옵션 `wrapper.conf`을 추가합니다. 아래 줄에서 이 파일에서 사용하지 않는 번호를 `XXX`로 대체합니다.

```ini
wrapper.java.additional.XXX=-javaagent:/full/path/to/newrelic.jar
```

<Callout variant="tip">
  Linux 시스템에서는 이 값을 설정할 때 따옴표가 필요하지 않습니다. 이는 운영 체제마다 다를 수 있습니다.
</Callout>

## Tomcat [#Installing_on_Tomcat][#Installing_on_Tomcat]

Tomcat에서 `-javaagent` 인수를 전달하려면:

<CollapserGroup>
  <Collapser
    id="tomcat-setenvsh"
    title="setenv.sh 사용"
  >
    아직 존재하지 않는 경우 `CATALINA_BASE/bin/setenv.sh` 스크립트를 만듭니다. `CATALINA_OPTS` 환경 변수를 사용하여 뉴렐릭 에이전트를 사용하도록 `setenv.sh` 스크립트를 구성합니다.

    ```sh
    export CATALINA_OPTS="$CATALINA_OPTS -javaagent:/full/path/to/newrelic.jar"
    ```
  </Collapser>

  <Collapser
    id="tomcat-setenvbat"
    title="setenv.bat 사용"
  >
    아직 존재하지 않는 경우 `CATALINA_BASE/bin/setenv.bat` 스크립트를 만듭니다. `CATALINA_OPTS` 환경 변수를 사용하여 뉴렐릭 에이전트를 사용하도록 `setenv.bat` 스크립트를 구성합니다.

    ```batch
    SET "CATALINA_OPTS=%CATALINA_OPTS% -javaagent:/full/path/to/newrelic.jar"
    ```
  </Collapser>

  <Collapser
    id="tomcat-catalinash"
    title="catalina.sh 사용(권장하지 않음)"
  >
    <Callout variant="tip">
      이 스크립트에서는 변수를 설정하지 않는 것이 좋습니다. 대신 `CATALINA_BASE/bin`의 스크립트 `setenv.sh`에 넣어 맞춤화된 설정을 별도로 유지합니다.
    </Callout>

    `JAVA_OPTS` 환경 변수를 사용하여 뉴렐릭 에이전트를 사용하도록 `catalina.sh` 파일을 구성합니다.

    ```sh
    export JAVA_OPTS="$JAVA_OPTS -javaagent:/full/path/to/newrelic.jar"
    ```
  </Collapser>

  <Collapser
    id="tomcat-catalinabat"
    title="catalina.bat 사용(권장하지 않음)"
  >
    <Callout variant="tip">
      이 스크립트에서는 변수를 설정하지 않는 것이 좋습니다. 대신 `CATALINA_BASE/bin`의 스크립트 `setenv.bat`에 넣어 맞춤화된 설정을 별도로 유지합니다.
    </Callout>

    `catalina.bat`를 사용하여 Tomcat을 시작하는 경우 파일 상단 근처에 `JAVA_OPTS` 변수를 설정합니다.

    ```batch
    SET JAVA_OPTS=%JAVA_OPTS% -javaagent:/full/path/to/newrelic.jar
    ```
  </Collapser>

  <Collapser
    id="tomcat-windows"
    title="Windows 사용"
  >
    1. <DNT>**Start > Apache Tomcat X.Y.Z. > Configure Tomcat > Java**</DNT>을 선택합니다.

    2. <DNT>**Java Options**</DNT> 텍스트 상자에 인수를 입력합니다. 경로 구분 기호로 슬래시(`/`)를 사용합니다. Tomcat 6의 경우 `-javaagent` 인수 뒤에 줄 바꿈을 추가합니다.

       ```
       -javaagent:/full/path/to/newrelic.jar
       ```

    3. <DNT>**Apply**</DNT>를 선택한 다음 Tomcat을 다시 시작합니다.
  </Collapser>

  <Collapser
    id="apache-commons-daemon"
    title="Apache Commons 데몬 사용"
  >
    Tomcat 6에 포함된 Apache Commons Daemon(jsvc) 버전은 뉴렐릭에서 사용하는 `-javaagent` 인수를 지원하지 않습니다. 그러나 트렁크 소스의 jsvc 데몬 빌드는 `-X` 접두사를 통해 `-javaagent` 인수를 지원합니다. [문제를 추적하는 Apache 버그](https://issues.apache.org/jira/browse/DAEMON-84)를 참조하십시오.

    Apache Commons 소스 리포지토리에 수정 방법이 있습니다. 보다 자세한 내용은:

    * [Apache 소스 리포지토리 문서](https://commons.apache.org/svninfo.html)를 참조하세요.
    * [SVN 소스 리포지토리](https://subversion.apache.org/)로 바로 이동합니다.
  </Collapser>
</CollapserGroup>

## WebLogic [#Installing_on_WebLogic][#Installing_on_WebLogic]

WebLogic에서 `-javaagent` 인수를 전달하려면:

<CollapserGroup>
  <Collapser
    id="weblogic-nix"
    title="Linux 또는 macOS의 관리 서버"
  >
    1. 도메인의 `bin` 디렉터리에 있는 `startWebLogic.sh` 파일을 수정합니다.

    2. 파일 시작 부분에 다음을 추가합니다.

       ```sh
       export JAVA_OPTIONS="$JAVA_OPTIONS -javaagent:/full/path/to/newrelic.jar"
       ```
  </Collapser>

  <Collapser

    1. 도메인의 `bin` 디렉터리에 있는 `startWebLogic.bat` 파일을 수정합니다.

    2. 파일 시작 부분에 다음을 추가합니다.

       ```
  </Collapser>

    관리 서버 인스턴스의 경우 Linux/macOS 또는 Windows 지침을 따르십시오. 관리 콘솔을 사용해서 관리 서버 인스턴스를 설치할 수는 없습니다.

    매니지드 서버 인스턴스의 경우 관리 콘솔을 사용합니다.

       ```

## WebSphere [#Installing_on_Websphere][#Installing_on_Websphere]

WebSphere에서 `-javaagent` 인수를 전달하려면:

1. 관리 콘솔에서 <DNT>**Servers > Application servers > (select a server) > Configuration > Service Infrastructure > Java and Process Management**</DNT>를 선택합니다.

2. <DNT>**Process Definition > Additional Properties**</DNT>를 선택한 다음 <DNT>**Java Virtual Machine**</DNT>를 클릭합니다.

3. <DNT>**Generic JVM arguments**</DNT> 필드에서 `newrelic.jar` 파일에 대한 `-javaagent` 인수를 추가합니다.

   ```
   -javaagent:/full/path/to/newrelic.jar
   ```

4. <DNT>**Apply**</DNT>를 선택한 다음 <DNT>**Save**</DNT>를 클릭합니다.

5. 서버를 다시 시작합니다.

보다 자세한 정보는 [WebSphere PMI 메트릭 수집](/docs/agents/java-agent/features/jvm-metrics-page#default-pmi)에 대한 문서를 참조하십시오.

## WebSphere Community [#Installing_on_Websphere_Community_Edition][#Installing_on_Websphere_Community_Edition]

WebSphere Community Edition에서 `-javaagent` 인수를 전달하려면 시작 명령을 실행할 때 `JAVA_OPTS` 환경 변수에서 뉴렐릭 에이전트 jar를 참조합니다.

```sh
export JAVA_OPTS="$JAVA_OPTS -javaagent:/full/path/to/newrelic.jar" && startup
```

## WebSphere Liberty Profile [#Installing_on_WebSphere_Liberty_Profile][#Installing_on_WebSphere_Liberty_Profile]

WebSphere Liberty Profile에서 `-javaagent` 인수를 전달하려면:

1. `${server.config.dir}/jvm.options`를 수정합니다.

2. `newrelic.jar` 파일을 가리키도록 `-javaagent` 인수를 추가합니다.

   ```
   -javaagent:/full/path/to/newrelic.jar
   ```

3. 서버를 다시 시작합니다.
