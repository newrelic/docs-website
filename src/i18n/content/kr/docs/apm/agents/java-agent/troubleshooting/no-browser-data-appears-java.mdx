## 문제

당신 [은 <InlinePopover type="browser"/>](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent)Java 앱용 에이전트 이지만 앱이 [브라우저 UI](/docs/browser/new-relic-browser/getting-started/browser-overview-page-website-performance-summary) 에 데이터를 보고하지 않습니다.

## 해결책

애플리케이션 서버가 Jasper JSP 컴파일러를 사용하지 않는 경우(예: WebSphere 또는 WebLogic을 사용하는 경우) 자동 계측을 사용할 수 없습니다. 대신 [브라우저를 수동으로 계측](/docs/java/page-load-timing-in-java#manual_instrumentation) 해야 합니다.

앱 서버가 JSP 컴파일러를 사용하지만 여전히 브라우저 데이터를 받지 못하는 경우 구성 충돌이 있을 수 있습니다. New Relic의 Java 에이전트를 사용하면 `newrelic.yml` 구성 파일에서 또는 `web.xml` `display-name` 를 사용하여 자동으로 앱 이름을 설정할 수 있습니다.

앱이 올바른 이름으로 보고되는지 확인하려면 다음 단계를 따르세요.

1. 일시적으로 [`finest` 레벨 로그를](/docs/agents/java-agent/troubleshooting/generating-logs-troubleshooting) 생성합니다.
2. `"enabled for"` 에 대한 로그 파일을 검색합니다. [브라우저 타이밍 데이터](/docs/agents/java-agent/instrumentation/page-load-timing-java) 를 보고하는 앱 목록에 앱 이름이 나타나는지 확인합니다.
3. 나열된 앱 이름이 예상 이름과 일치하지 않으면 `newrelic.yml` 및 `web.xml` `display-name` 설정에서 동일한 앱 이름을 설정하고 있는지 확인하십시오.
