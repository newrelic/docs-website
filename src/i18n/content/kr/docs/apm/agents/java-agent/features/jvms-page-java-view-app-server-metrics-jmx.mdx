New Relic의 Java 에이전트를 사용하여 JMX를 통해 가장 일반적인 애플리케이션 서버에서 제공하는 추가 메트릭을 수집하고 볼 수 있습니다. 수집된 메트릭은 앱 서버에 따라 다르며 스레드 풀, HTTP 세션 및 트랜잭션의 측정값을 포함합니다.

## JVM 측정항목 보기 [#dashboard]

애플리케이션 서버는 APM의 **JVM** 페이지에 나타나는 다양한 메트릭을 수집하고 보고합니다. (Heroku를 통해 New Relic을 사용하는 애플리케이션의 경우 **JVM** 페이지의 이름은 **[Instances](/docs/accounts-partnerships/partnerships/heroku-new-relic/heroku-instances-dashboard)** 입니다.)

최소한 각 앱 서버는 메모리에서 메트릭 데이터를 수집하고 보고합니다. JMX 메트릭 활성화에 대한 지침은 앱 서버의 설명서를 참조하십시오.

JVM 측정 단위를 보려면 다음을 수행합니다.

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (앱 선택) > Monitoring > JVMs > (JVM 선택)** 으로 이동합니다.
2. 선택한 JVM의 **메모리** 페이지에서 New Relic의 표준 [사용자 인터페이스 기능](/docs/accounts-partnerships/education/getting-started-new-relic/standard-dashboard-features) 을 사용하여 자세한 정보를 드릴다운합니다.
3. 다른 측정항목을 보려면 [사용 가능한 탭](#dashboard-tabs) 을 선택합니다.
4. 선택 사항: [스레드 프로파일러를 사용하여 스레드](/docs/applications-menu/thread-profiler) 실행 데이터의 병목 현상을 찾으려면 **이 JVM 프로파일링** 을 선택하십시오.

## 탭에서 사용할 수 있는 측정항목 [#tab-metrics]

다음은 JVM 페이지에서 사용할 수 있는 메트릭 요약입니다. 앱 서버가 특정 메트릭을 지원하지 않는 경우 해당 탭은 JVM 페이지에 표시되지 않습니다.

        * **JVM에 대한 힙이 아닌 메모리 풀 사용량:** 사용된 코드 캐시 및 사용된 CMS Perm Gen(MB)입니다.
        * **가비지 컬렉션:** 가비지 컬렉션 CPU 시간입니다.

      </td>
    </tr>

    <tr>
      <td>
        앱 서버 트랜잭션

      <td>

        * **생성된 트랜잭션:** 앱 서버당 생성된 총 트랜잭션 수의 변화입니다. 일부 앱 서버는 최상위 트랜잭션과 중첩 트랜잭션을 분리합니다.
        * **완료된 트랜잭션:** 앱 서버당 완료된 트랜잭션의 총 수의 변화입니다. 일부 앱 서버는 중단된 트랜잭션과 커밋된 완료된 트랜잭션으로 구분합니다.

## Tomcat을 사용하여 Spring Boot 2.2 이상에서 활성화

Spring Boot 2.2 이상에서 실행할 때 New Relic이 Tomcat JMX 측정항목에 액세스할 수 있도록 하려면 시스템 속성 `server.tomcat.mbeanregistry.enabled` 을 `true` 로 설정해야 합니다. [Spring Boot 2.2 릴리스 정보](https://github.com/spring-projects/spring-boot/wiki/Spring-Boot-2.2-Release-Notes#tomcats-mbean-registry) 를 참조하십시오.

## WebSphere PMI 메트릭 사용 [#default-pmi]

New Relic이 WebSphere PMI 메트릭에 액세스할 수 있도록 하려면 필요한 통계 세트를 모니터링하도록 WebSphere를 구성해야 합니다.

<CollapserGroup>
  <Collapser
    id="enable-all-stat-sets"
    title="모든 통계 세트 활성화(권장)"
  >
    가장 간단한 접근 방식은 모든 통계 집합의 수집을 활성화하는 것입니다.

    1. Java 2 Security를 사용하는 경우 [New Relic에 적절한 권한을 부여](/docs/agents/java-agent/installation/install-java-agent-java-2-security) 했는지 확인하십시오.
    2. WebSphere 관리 콘솔에서 **Monitoring and Tuning > PMI (Performance Monitoring Infrastructure) > (사용자의 서버 이름)** 을 선택하십시오.

    4. **현재 모니터링되는 통계 집합** 섹션에서 **모두** 를 선택한 다음 변경 사항을 저장합니다.
  </Collapser>

    title="사용자 정의 통계 세트 활성화(고급)"
  >
    New Relic이 기본적으로 모니터링하는 PMI 메트릭만 수집할 수도 있습니다. 일부 PMI 지표만 수집하려면:

    1. Java 2 Security를 사용하는 경우 [New Relic에 적절한 권한을 부여](/docs/agents/java-agent/installation/install-java-agent-java-2-security) 했는지 확인하십시오.

    2. WebSphere 관리 콘솔에서 **모니터링 및 조정 > PMI(Performance Monitoring Infrastructure) > (선택한 앱 서버)** 를 선택하십시오.

    3. **PMI(Performance Monitoring Infrastructure) 활성화** 확인란을 선택합니다.

    4. **사용자 지정** 을 선택합니다. 사용자 지정 통계 집합 목록에서 다음 측정항목을 활성화합니다.

       * **ThreadPool** 범주에서 **ActiveCount** 및 **PoolSize** 확인란을 선택합니다.
       * **Servlet Session Manager** 범주에서 **ActiveCount** 및 **LiveCount** 확인란을 선택합니다.
       * **트랜잭션 관리자** 범주에서 **CommittedCount** , **RolledbackCount** 및 **GlobalTimeoutCount** 확인란을 선택합니다.

    5. **메시지** 패널 **에서 마스터 구성에 직접 저장을** 선택합니다.
  </Collapser>
</CollapserGroup>
