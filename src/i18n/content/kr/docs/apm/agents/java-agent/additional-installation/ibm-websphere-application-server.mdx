IBM WebSphere Application Server를 사용하는 경우 New Relic의 Java 에이전트를 구성하는 방법에 대해 설명합니다. [호환 가능한 버전](#compatible-versions) 의 경우 New Relic의 절차에 따라 [WebSphere에 Java 에이전트를 설치하십시오](/docs/agents/java-agent/installation/include-java-agent-jvm-argument#Installing_on_Websphere) .

    <tr id="compatible-versions">
      <td>
        호환되는 IBM JVM 버전
      </td>

      <td>
        New Relic은 [Java 에이전트와 호환되는](/docs/agents/java-agent/getting-started/compatibility-requirements-java-agent) 모든 버전의 WebSphere를 지원합니다. 그러나 특정 버전의 IBM JVM은 IBM JVM [의 알려진 문제](http://www-01.ibm.com/support/docview.wss?uid=swg1IV25688) 로 인해 Java 에이전트와 호환되지 않습니다.

        New Relic의 Java 에이전트는 다음 주요 버전의 IBM JVM과 호환됩니다.

        * 8: 모든 버전
      </td>
    </tr>

    <tr id="security">
      <td>
        자바 2 보안
      </td>

      <td>
        Java 2 Security 및 WebSphere를 사용하는 경우 Java 에이전트가 제대로 실행되기 전에 [추가 권한](/docs/agents/java-agent/installation/install-java-agent-java-2-security#websphere-java-2) 을 부여해야 합니다.
      </td>
    </tr>

    <tr id="browser-monitoring">
      <td>
        브라우저 모니터링
      </td>

      <td>
        WebSphere에서 Java 에이전트를 실행할 때 브라우저 모니터링을 사용하려면 브라우저 모니터링을 [수동으로 활성화](/docs/agents/java-agent/instrumentation/new-relic-browser-java-agent#manual_instrumentation) 해야 합니다.
      </td>
    </tr>

    <tr id="pmi-metrics">
      <td>
        WebSphere PMI 메트릭
      </td>

      <td>
        [추가 WebSphere PMI 메트릭을 캡처](/docs/agents/java-agent/features/jvm-metrics-page#default-pmi) 하도록 Java 에이전트를 구성할 수 있습니다. 이러한 메트릭은 New Relic **JVM 메트릭** 페이지에 나타납니다.
      </td>
    </tr>

      </td>
    </tr>
