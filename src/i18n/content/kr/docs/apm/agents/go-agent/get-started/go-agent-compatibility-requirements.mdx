아직 계정이 없다면 [New Relic 계정을 만드](https://newrelic.com/signup) 십시오. 영원히 무료입니다. [New Relic for Go를 설치](/docs/agents/go-agent/installation/install-new-relic-go) 하기 전에 시스템이 이러한 요구 사항을 충족하는지 확인하십시오.

## 골랑 버전 [#golang-release]

New Relic은 [Golang 1.19 이상을](https://go.dev/doc/devel/release) 지원합니다.

## 운영 환경 [#operating-systems]

에이전트는 <DNT>Linux</DNT>, <DNT>macOS</DNT>, <DNT>Windows</DNT> 지원합니다. [Google App Engine(GAE) 유연한 환경](/docs/agents/go-agent/get-started/install-new-relic-go-agent-gae-flexible-environment) 에서도 Go 에이전트를 사용할 수 있습니다.

## 통합 [#frameworks]

다음 [통합 패키지](https://godoc.org/github.com/newrelic/go-agent/_integrations) 는 기본 [newrelic](https://godoc.org/github.com/newrelic/go-agent) 패키지를 확장하여 다른 프레임워크 및 라이브러리를 지원합니다. 프레임워크와 데이터베이스에 통합 패키지가 없는 경우에도 [newrelic](https://godoc.org/github.com/newrelic/go-agent) 패키지 기본 요소를 사용하여 계측할 수 있습니다. 이러한 기본 요소를 사용하여 데이터베이스를 계측하는 방법에 대한 자세한 내용은 [GitHub에서 Go 에이전트의 데이터 저장소 세그먼트 설명서를](https://github.com/newrelic/go-agent/blob/master/GUIDE.md#datastore-segments) 참조하세요.

[GitHub의 nrgin 예제에](https://github.com/newrelic/go-agent/blob/master/v3/integrations/nrgin/example/main.go) 표시된 대로 다음 통합 패키지를 [newrelic](https://godoc.org/github.com/newrelic/go-agent/v3/newrelic) 패키지와 함께 가져와야 합니다.

## 데이터베이스 및 인스턴스 수준 성능

뉴렐릭은 [다양한 데이터베이스 및 데이터베이스 드라이버에 대한 세부 정보를](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues) 수집합니다. <InlinePopover type="apm" />의 특정 인스턴스와 데이터베이스 정보 유형을 볼 수 있는 기능은 에이전트 버전에 따라 다릅니다.

New Relic Go 에이전트 [버전 1.4 이상](/docs/release-notes/agent-release-notes/go-release-notes/go-agent-14) 은 모든 데이터베이스 드라이버에 대한 인스턴스 세부 정보를 지원합니다.
