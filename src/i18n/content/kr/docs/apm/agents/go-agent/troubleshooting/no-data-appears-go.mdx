## 문제

Golang 앱용 [New Relic Go 에이전트를 설치했습니다](/docs/agents/go-agent/get-started/install-new-relic-go) . 약간의 트래픽을 생성하고 몇 분을 기다린 후에도 앱은 여전히 New Relic에서 데이터를 보고하지 않습니다.

## 해결책

웹 애플리케이션에 요청을 보낸 후 2-3분 내에 데이터가 <InlinePopover type="apm"/>UI에 표시됩니다. 여전히 앱의 [<DNT>**Summary**</DNT> 페이지](/docs/apm/applications-menu/monitoring/apm-overview-page) 에 데이터가 표시되지 않으면 다음 문제 해결 팁을 시도해 보세요.

1. Go 에이전트 구성의 [`newrelic.Config` 구조체](/docs/agents/go-agent/instrumentation/go-agent-configuration) 에서 [앱 이름](/docs/agents/go-agent/instrumentation/go-agent-configuration#app-name) 과 [라이선스 키](/docs/agents/go-agent/instrumentation/go-agent-configuration#license) 를 확인합니다.
2. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 사용하여 문제를 자동으로 식별해 보십시오.
3. [`newrelic.WrapHandleFunc` 또는 `Application.StartTransaction` 및 `Transaction.End`](/docs/agents/go-agent/get-started/instrument-go-transactions) 를 사용하여 앱의 거래를 계측했는지 확인하십시오.
4. 웹 서버를 다시 시작하십시오.
5. [Go Logger 인터페이스를 사용하여 `Debug` 수준 로그를 생성](/docs/agents/go-agent/instrumentation/go-agent-logging) 하고 해당 로그에 오류가 있는지 확인합니다.
6. 필요한 경우 [최신 New Relic Go 에이전트 릴리스](/docs/release-notes/agent-release-notes/go-release-notes) 로 업그레이드하십시오.
