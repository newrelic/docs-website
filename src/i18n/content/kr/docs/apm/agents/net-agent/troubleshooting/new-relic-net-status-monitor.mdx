  .NET 에이전트 릴리스 6.20부터 New Relic의 .NET 상태 모니터는 더 이상 사용 **되지 않으며** 더 이상 사용할 수 없습니다. [.NET 에이전트 버전 6.20 이상의](/docs/release-notes/agent-release-notes/net-release-notes) 경우 [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 사용합니다.
</Callout>

상태 모니터는 더 이상 사용되지 않는 New Relic의 .NET 에이전트용 진단 및 상태 도구입니다. 다음을 수행하도록 설계되었습니다.

* 서버에서 실행 중인 .NET 에이전트의 현재 상태에 대한 가시성을 제공합니다.
* 일반적인 설치 문제를 해결하는 데 도움이 됩니다.
* newrelic.com에서 계측된 각 애플리케이션에 대한 빠른 링크를 제공하십시오.
* 계측된 각 애플리케이션에 대한 New Relic 로그 파일에 대한 빠른 액세스를 제공합니다.

추가 도움이 필요하면 New Relic .NET 에이전트의 문제 해결 설명서를 참조하십시오.

## 모니터링되는 앱의 상태 보기 [#viewing]

<Callout variant="important">
  .NET 에이전트 릴리스 6.20부터 New Relic의 .NET 상태 모니터는 더 이상 사용할 수 없습니다. [.NET 에이전트 버전 6.20 이상의](/docs/release-notes/agent-release-notes/net-release-notes) 경우 [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 사용합니다.
</Callout>

상태 모니터는 현재 .NET 에이전트가 계측하고 있는 애플리케이션과 각 애플리케이션의 상태에 대한 가시성을 제공합니다. 최근에 트래픽이 발생하지 않는 한 애플리케이션에 대한 데이터가 표시되지 않습니다. 시스템 리소스를 절약하기 위해 상태 모니터가 자동으로 새로 고쳐지지 않습니다.

상태 모니터를 시작하려면:

1. Windows **시작** 메뉴 또는 **시작** 화면에서 `Status monitor` 을(를) 검색합니다.

   또는

2. Windows 작업 표시줄 상태 영역(시스템 트레이)에서 New Relic 아이콘을 마우스 오른쪽 버튼으로 클릭한 다음 **모니터링되는 애플리케이션** 을 선택합니다.

3. 응용 프로그램이 계측되고 있는지 확인하려면 해당 확인란이 모두 녹색인지 확인하십시오.

4. 특정 응용 프로그램을 선택한 후 사용 가능한 옵션 중 하나를 선택하여 New Relic에서 앱을 보고, 로그 파일을 보고, 오류가 발견된 경우 [설치를 복구](#repairing) 하거나, 새로 고침한 다음 모니터링되는 프로세스 및 응용 프로그램의 최신 상태를 봅니다.

## 수리 설치 [#repairing]

New Relic의 .NET 에이전트 설치는 강력하고 사용하기 쉽습니다. 그러나 이전 설치가 최신 버전과 충돌할 수 있는 특정 시나리오가 있습니다. 일반적인 문제를 해결하려면 설치를 복구하십시오.

1. 상태 모니터의 앱 목록에서 앱을 선택한 다음 **설치 복구** 를 선택합니다.

   또는

   Windows 시스템 트레이에서 New Relic 아이콘을 선택한 다음 **Tools > Repair installation** 를 선택합니다.

2. 설치 복구가 완료된 후: 명령줄에서 `IISRESET` 을(를) 수행합니다.
