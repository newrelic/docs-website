코드 수준 지표를 통해 [New Relic CodeStream 확장을](/docs/codestream/observability/code-level-metrics) 사용하는 개발자는 코드의 개별 메서드와 함께 IDE에 상황에 따라 표시되는 APM 데이터를 볼 수 있습니다. 이를 통해 개발자는 코드를 작성하고 검토할 때 성능 문제를 보다 적극적으로 해결할 수 있습니다.

## 시작하다 [#get-started]

먼저 지원되는 IDE에 New Relic CodeStream 확장을 [설치](/docs/codestream/start-here/install-codestream) 하고 로그인합니다.

<Callout variant="important">
  .NET 에이전트 버전 10.2.0 릴리스에서는 코드 수준 메트릭이 기본적으로 활성화되었습니다.

  코드 수준 메트릭은 [.NET 에이전트 버전 9.9.0 이상](/docs/release-notes/agent-release-notes/net-release-notes) 에서 사용할 수 있으며 기본적으로 비활성화되어 있습니다. 이 구성을 변경하려면 [설명서를](/docs/apm/agents/net-agent/configuration/net-agent-configuration#code_level_metrics)확인하십시오.
</Callout>

## 에이전트 속성 [#attributes]

.NET 에이전트는 다음 속성을 보고하고 애플리케이션에서 생성한 범위에 연결합니다.

<Callout variant="important">
  경우에 따라 에이전트는 애플리케이션 내에서 호출되고 IDE 내에서 표시되지 않는 기본 프레임워크 코드(데이터 저장소, 외부 호출 등)를 직접 래핑합니다.
</Callout>
