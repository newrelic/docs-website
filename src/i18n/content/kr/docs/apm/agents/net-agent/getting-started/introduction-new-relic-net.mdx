애플리케이션 성능 모니터링을 위한 .NET 에이전트를 사용하여 다음을 수행할 수 있습니다.

* [APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm) 을 사용 [하여 앱에 대한 높은 수준의 개요](/docs/apm/applications-menu/monitoring/apm-overview-page) 를 확인하고 [트랜잭션 추적](/docs/apm/transactions/transaction-traces/transaction-traces) , [데이터베이스 쿼리](/docs/apm/transactions/transaction-traces/sql-statements) 및 [오류](/docs/apm/applications-menu/events/view-apm-error-analytics) 와 같은 코드 수준 세부 정보를 확인 [하고 대규모 분산 시스템의 활동을 추적합니다](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) .
* 앱이 원활하게 실행되고 있는지 확인하기 위해 [경고](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) 에서 사전 알림을 받습니다.
* [쿼리 빌더](/docs/query-your-data/explore-query-data/query-builder/use-advanced-nrql-mode-specify-data) 를 사용하여 데이터를 쿼리하고 해당 데이터로 사용자 지정[대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-new-relic-one-dashboards) 를 만듭니다.
* 앱 호스트 환경의 성능을 보려면 [인프라 모니터링](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server) 을 설치하세요.

## .NET Framework 및 .NET Core 모두 지원 [#core-features]

New Relic의 .NET 에이전트는 [.NET Framework](/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent) 와 . [NET Core](/docs/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core) , VB.NET, C# 및 CLI와 같은 모든 .NET 호환 언어와 함께 작동합니다.

.NET Core에 대한 에이전트 지원은 Microsoft .NET Core의 호환성, 속도, 확장된 API 기능 및 플랫폼 간 기능을 활용합니다. 에이전트는 2.0 이전의 Microsoft .NET Core 버전을 지원 **하지 않습니다** .

.NET에 대한 New Relic의 지원을 통해 다음과 같은 동적 또는 분산 환경에서 앱을 모니터링할 수 있습니다.

* 클라우드 관리 서버 VM 이미지
* 호스트 VM 서버
* 마이크로소프트 애저 앱 서비스
* 자체 호스팅 Windows 및 Linux 시스템
* AWS EC2 VM

## 설치 안내를 통해 Windows에 에이전트 설치 [#guided-install]

시작할 준비가 되셨습니까? 안내 설치 버튼을 클릭합니다. 계정에서 [EU 데이터 센터](/docs/using-new-relic/welcome-new-relic/get-started/our-eu-us-region-data-centers/) 를 통해 데이터를 보고하는 경우 EU Guided install을 클릭합니다.

## 에이전트 구성 [#configuration]

에이전트 구성의 가장 중요한 부분은 [앱에 설명이 포함된 이름을 지정하는 것](/docs/agents/net-agent/installation-configuration/name-your-net-application) 입니다. New Relic [은 여러 앱 또는 호스트가 있는 경우 이 앱 이름을 사용하여 메트릭을 집계](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name) 합니다. 에이전트에는 설치를 추가로 사용자 지정할 수 있는 다양한 [구성 옵션](/docs/agents/net-agent/installation-configuration/net-agent-configuration) 도 포함되어 있습니다.

## APM 및 인프라 데이터에 대한 로그 보기 [#logs-context]

또한 로그와 애플리케이션의 데이터를 함께 가져와 문제 해결을 더 쉽고 빠르게 할 수 있습니다. [컨텍스트의 로그를](/logs/logs-context/net-configure-logs-context-all/) 사용하면 앱의 UI에서 직접 오류 및 추적과 관련된 로그 메시지를 볼 수 있습니다. Kubernetes 클러스터와 같은 [인프라 데이터](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/) 컨텍스트에서 로그를 볼 수도 있습니다. 다른 UI 페이지로 전환할 필요가 없습니다.

## 소스 코드 확인 [#source-code]

.NET 에이전트는 오픈 소스 소프트웨어입니다. 즉 [, 소스 코드를 탐색하고](https://github.com/newrelic/newrelic-dotnet-agent) 개선 사항을 보내거나 고유한 포크를 만들어 빌드할 수 있습니다. 자세한 내용은 [README](https://github.com/newrelic/newrelic-dotnet-agent/blob/main/README.md) 를 참조하십시오.
