## 문제

[Microsoft Azure Web App](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps) 용 New Relic의 .NET 에이전트( `NewRelic.Profiler.dll` ) 새 버전을 배포할 때 오류 메시지가 나타나 프로세스를 중지하거나 다시 시작할 때까지 계속할 수 없습니다.

## 해결책

일반적으로 프로세스가 중지된 다음 배포 중에 New Relic이 사이트 루트( `D:\Home\site\wwwroot\newrelic` 에 있음)에 포장하는 자산으로 자산을 덮어씁니다. 문제를 해결하려면 다음 해결 방법 중 하나를 시도해 보십시오.

<CollapserGroup>
  <Collapser
    id="stop-dll-deploy"
    title="파일 릴리스 중지"
  >
    이 솔루션은 기존 버전 위에 새 .NET 에이전트 버전을 배포하려는 경우 Azure Web App에 사용할 수 있습니다.

    1. 웹 앱을 중지하여 `.dll` 파일을 해제하세요.
    2. 업데이트가 진행되도록 허용합니다.
    3. 배포가 완료되면 인스턴스를 다시 시작합니다.
  </Collapser>

  <Collapser
    id="site-extension"
    title="새 유물 사이트 확장 사용"
  >
    [New Relic Site Extension](https://www.siteextensions.net/packages/NewRelic.Azure.WebSites/) 을 사용하여 Azure Web App에 New Relic의 .NET 에이전트를 설치할 수 있습니다. (확장 프로그램은 한 번만 설치하면 됩니다.)

    1. **https&#x3A;//yoursite.scm.azurewebsites.net/** 으로 이동합니다. **사이트 확장** 을 선택합니다.

    2. **갤러리** 에서 추가를 선택합니다. <Icon name="fe-plus"/>아이콘을 클릭한 다음 **새 유물** 을 선택합니다.

    3. 지침에 [따라 구성 키를 웹 앱에 수동으로 추가하고](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps) 요구 사항에 가장 적합한 옵션을 사용합니다.

       사이트 확장을 업데이트하려면 [Azure Portal](https://portal.azure.com) 에서 **WebApps > (사이트 선택) > All Settings > Extensions > New Relic** 을 선택합니다.
  </Collapser>
</CollapserGroup>
