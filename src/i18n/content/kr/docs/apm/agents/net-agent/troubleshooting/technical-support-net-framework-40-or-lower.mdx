<Callout variant="important">
  뉴렐릭은 .NET 프레임워크 4.0 이하를 사용하는 고객을 위해 <DNT>**limited technical support**</DNT> 제공합니다.
</Callout>

이 지원 수준에는 다음 영역이 예상대로 구성되었는지 확인하는 것이 포함됩니다.

* 에이전트 구성
* 환경 설정
* 네트워크 액세스
* 권한 설정

수동으로 수행해야 하는 몇 가지 검사를 제외하고 대부분의 문제 해결 검사는 [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 사용하여 수행할 수 있습니다. New Relic Diagnostics 검사에서 문제가 발견되면 추가 문제 해결 및 잠재적 해결 방법에 대한 정보가 터미널에 나타납니다.

## 에이전트 버전 요구 사항 [#agent-version]

New Relic .NET Framework 에이전트를 최대한 활용하려면 New Relic의 [호환성 및 요구 사항](/docs/agents/net-agent/getting-started/compatibility-requirements-net-agent) 지침을 따라야 합니다.

<Callout variant="important">
  .NET 프레임워크 버전 4.0 이하에서 실행하려면 뉴렐릭 .NET 에이전트 버전 <DNT>**lower than 7.0**</DNT> 이 필요합니다.
</Callout>

[아키텍처](/docs/agents/net-agent/getting-started/compatibility-requirements-net-agent#architecture) 에 적합한 설치 패키지를 다운로드하고 실행합니다.

* [32비트 설치 프로그램](https://download.newrelic.com/dot_net_agent/6.x_release/NewRelicDotNetAgent_x86.msi)
* [64비트 설치 프로그램](https://download.newrelic.com/dot_net_agent/6.x_release/NewRelicDotNetAgent_x64.msi)

## 에이전트 구성 [#config]

기본 설치 및 구성 문제를 해결하기 위한 제한된 기술 지원에는 다음이 포함됩니다.

1. .NET 에이전트가 있고 유효한 구성 파일이 있는지 확인하십시오.
2. 에이전트가 설치된 경우 에이전트의 코어 `.dll` 를 확인하십시오.
3. 구성 파일이 유효한 경우 예상되는 요소를 확인하십시오. 예를 들어 `.xsd` 에 명시된 표준을 준수하는지 수동 확인을 포함합니다.

## 수단

계측이 가능한 경우 문제 해결을 위한 제한된 기술 지원에는 다음이 포함됩니다.

1. WAS 레지스트리 키와 필수 값이 존재하고 유효한지 확인하십시오.
2. W3SVC 레지스트리 키와 필수 값이 존재하고 유효한지 확인하십시오.
3. ENV(비IIS/전역 환경 변수) reg 키와 필수 값이 존재하고 유효한지 확인하십시오.
4. 이러한 키 검사를 통해 모니터링할 수 있는 앱 유형을 결정하여 계측이 가능한지 확인합니다.
5. 프로세스에 필요한 환경 변수가 로드되었는지 확인합니다. 예를 들어, [Microsoft Sysinternals Process Explorer](https://docs.microsoft.com/en-us/sysinternals/downloads/process-explorer) 와 같은 도구를 사용하십시오.

## 웹 앱 [#web-apps]

일부 검사에는 높은 권한이 필요합니다. 최상의 결과를 얻으려면 관리 셸에서 [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 실행하십시오.

.NET 에이전트가 설치된 웹 앱 실행 문제를 해결하기 위한 제한된 기술 지원에는 다음이 포함됩니다.

1. 실행 중인 W3WP 프로세스를 확인하고 `pid` 을(를) 가져옵니다.
2. 계측에 필요한 `.dll` 키를 확인하십시오. 예를 들어 W3WP 프로세스가 있는 경우 New Relic 프로파일러 `.dll` 및 `mscorelib.dll` 가 프로세스에 로드되었는지 확인합니다.

## IIS에서 호스팅되지 않는 응용 프로그램 [#non-IIS-apps]

IIS에서 호스팅되지 않는 앱 문제를 해결하기 위한 제한된 기술 지원에는 다음이 포함됩니다.

1. MSI를 사용할 때 `Instrument all` 옵션이 활성화되었는지 확인하십시오.
2. 사용자 정의 계측 파일이 생성되었고 올바른 위치에 있으며 정확한지 확인하십시오. 해당하는 경우 `.xsd` 에 명시된 표준을 준수하는지 수동 확인을 포함합니다.

## 네트워크 액세스 및 권한 [#access]

시스템 네트워크 액세스 및 권한 문제를 해결하기 위한 제한된 기술 지원은 모든 에이전트 버전에 적용되며 다음이 포함됩니다.

1. 시스템 네트워크 액세스 및 권한을 확인하십시오.
2. 수집기 테스트를 실행합니다. 예를 들어, URL `https://collector.newrelic.com/status/mongrel` 이 작동 중이고 서버가 이에 연결할 수 있는지 확인하십시오.
3. 구성 파일에 프록시 설정이 포함되어 있는지, 프록시 정보가 New Relic Diagnostics에 전달되었는지, 이러한 설정이 New Relic 수집기에 대한 연결을 허용하는지 확인하십시오.
4. New Relic Diagnostics가 `admin` 으)로 실행 중인지 확인합니다.

New Relic Diagnostics는 다른 테스트의 정확성을 보장하기 위해 `admin` 으로 실행되어야 합니다. 또한 .NET 에이전트를 설치하려면 `admin` 권한이 필요합니다.

## 에이전트 로그 [#logs]

.NET 에이전트 로그에서 제공되는 문제 해결 정보에 대한 제한된 기술 지원에는 다음이 포함됩니다.

1. 사용 가능한 에이전트 로그를 수집합니다.
2. 로그 레벨을 확인하십시오.
3. 로그에 기록된 알려진 문제를 확인합니다.

로그 파일 문제를 해결하려면 [New Relic Support](https://support.newrelic.com/) 와 직접 작업해야 할 수 있습니다.

## 프레임워크 버전

제한된 기술 지원은 사용 중인 .NET 에이전트 버전에 따라 다릅니다. New Relic Support는 시스템에 설치된 .NET 에이전트의 프레임워크 버전을 확인합니다.

## OpenRasta 요청의 전체 URL [#openrasta]

OpenRasta 계측을 사용하는 경우 계측 요청에서 전체 URL을 캡처할 수 있습니다. 그 결과 민감한 정보를 포함할 수 있는 쿼리 문자열이 수집될 수 있습니다. 보안 게시판 [NR19-01](/docs/using-new-relic/new-relic-security/security-bulletins/security-bulletin-nr19-01) 을 참조하십시오.
