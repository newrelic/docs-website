이 문서에는 ASP.NET Core 애플리케이션을 모니터링하기 위해 New Relic의 .NET 에이전트를 설치하기 위한 지침이 포함되어 있습니다. 다음은 [Windows](/docs/agents/net-agent/installation/install-net-agent-windows) 및 [Linux](/docs/agents/net-agent/installation/install-net-agent-linux) 의 표준 설치 절차를 보완하는 추가 주의 사항 및 지침입니다.

## .NET Core를 대상으로 하는 ASP.NET Core 앱 [#core-targeting-core]

New Relic은 IIS에서 호스팅되는 .NET Core를 대상으로 하는 ASP.NET Core 앱을 자동으로 계측합니다.

IIS를 사용 **하지 않는** 응용 프로그램의 경우 해당되는 경우 [Windows](/docs/agents/net-agent/other-installation/install-net-agent-aspnet-core-targeting-net-framework-windows) 또는 [Linux](/docs/agents/net-agent/installation/install-net-agent-linux) 용 표준 설치 절차를 따르십시오. 또한 애플리케이션에 대한 에이전트를 활성화해야 하며 모니터링이 자동으로 발생하지 않는 경우 사용자 지정 계측을 설정해야 합니다.

* 앱 활성화: [Windows](/docs/agents/net-agent/installation/install-net-agent-windows#enabling-the-agent) 및 [Linux용](/docs/agents/net-agent/installation/install-net-agent-linux#enable-agent) 지침을 참조하세요.
* 사용자 지정 계측: Kestral을 사용하지 않거나 다른 이유로 기본적으로 계측이 수행되지 않는 경우 [앱을 사용자 지정 계측해야 합니다](/docs/agents/net-agent/custom-instrumentation/introduction-net-custom-instrumentation) .

## .NET Framework를 대상으로 하는 ASP.NET Core 앱 [#core-targeting-framework]

IIS에서 호스팅되지 않는 애플리케이션 또는 .NET Framework를 대상으로 하는 ASP.NET Core 앱의 [경우 Windows용 표준 설치 절차를](/docs/agents/net-agent/installation/install-net-agent-windows#msi) 따르세요. 또한 [.NET 에이전트를 명시적으로 활성화](/docs/agents/net-agent/installation/install-net-agent-windows#enabling-the-agent) 하려면 다음 절차를 따라야 합니다.
