이 문서에서는 .NET 에이전트를 설치하고 WCF 응용 프로그램을 모니터링하는 방법을 설명합니다.

## WCF 애플리케이션용 설치 [#wcf]

IIS 호스팅 WCF 앱에 .NET 에이전트를 설치하려면:

1. [IIS 호스팅 앱에 대한 다음 지침을](/install/dotnet/?deployment=windowsInstall&docker=noDocker) 사용하여 .NET 에이전트를 설치합니다.

   <Callout variant="important">
     자체 호스팅 WCF 애플리케이션(사용자 지정 Windows 서비스, 콘솔 앱 또는 Windows Forms 앱으로 호스팅됨)의 경우 에이전트도 활성화해야 합니다. [비 IIS 호스팅 앱에 대한 지침](/install/dotnet/?deployment=windowsInstallNonIis&docker=noDocker) 에서 **에이전트 활성화** 단계를 참조하십시오.
   </Callout>

2. [각 WCF 앱에 설명이 포함된 이름을 지정](/docs/agents/net-agent/installation-and-configuration/naming-your-net-application#naming-non-iis) 해야 합니다.

3. 모니터링할 이벤트 유형에 따라 WCF 응용 프로그램에서 ASP.NET 호환성 모드를 활성화 또는 비활성화합니다.

   <CollapserGroup>
     <Collapser
       id="mode-enabled"
       title="ASP.NET 파이프라인 이벤트 및 WCF 이벤트"
     >
       ASP.NET 호환 모드가 **활성화** 되면 .NET 에이전트는 ASP.NET 파이프라인 이벤트와 WCF 이벤트를 계측합니다. 이 모드에서 에이전트는 애플리케이션 스택의 전체 보기를 제공합니다.
     </Collapser>

     <Collapser
       id="mode-disabled"
       title="WCF 이벤트만"
     >
       ASP.NET 호환 모드가 **비활성화** 되면 .NET 에이전트는 WCF 이벤트만 계측합니다. WCF 트랜잭션 외부에서 수행되는 에이전트 API 호출과 같이 ASP.NET 파이프라인의 일부로 발생하는 모든 활동은 캡처되지 않습니다.
     </Collapser>
   </CollapserGroup>

애플리케이션이 트래픽을 수신하는 경우 데이터가 몇 분 이내에 표시되어야 합니다. 표시되지 않으면 [데이터가 표시되지 않음](/docs/agents/net-agent/troubleshooting/no-data-appears-net) 을 참조하십시오.
