디스크 사용량을 고려하면서 로그에 대한 필요성의 균형을 맞추기 위해 각 .NET 에이전트는 로그 롤링을 사용하여 디스크 사용량을 250MB로 제한합니다. 에이전트는 먼저 `newrelic_agent_UNIQUENAME.log` 파일에 기록하고 파일이 존재하지 않는 경우 생성합니다. 파일 크기가 50MB에 도달하면 에이전트는 다음을 수행합니다.

1. 새 로그 파일을 만듭니다.
2. 각 기존 로그 파일을 순차적으로 번호가 매겨진 새 이름(최대 4개의 아카이브 파일)으로 롤링합니다.
3. 네 번째 아카이브를 삭제합니다.

로그 파일을 롤링하려면 이전 `newrelic_agent_UNIQUENAME.log` 이 새 `newrelic_agent_UNIQUENAME.log(1)` 이(가) 됩니다. 그런 다음 이전 `newrelic_agent_UNIQUENAME.log(1)` 가 새 `newrelic_agent_UNIQUENAME.log(2)` 이 되는 식입니다. 이전 `newrelic_agent_UNIQUENAME.log(4)` 이(가) 삭제됩니다.
