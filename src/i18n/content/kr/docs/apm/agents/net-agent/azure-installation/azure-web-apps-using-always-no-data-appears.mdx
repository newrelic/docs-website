## 문제

[Microsoft Azure Web Apps](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps) 를 통해 New Relic의 .NET 에이전트를 설치했으며 Azure의 `Always On` 설정을 사용하고 있습니다. 일부 트래픽을 생성하고 앱에 대한 데이터 수집이 시작될 때까지 몇 분을 기다린 후에도 데이터는 여전히 APM 사용자 인터페이스에 표시되지 않습니다.

## 해결책

.NET 에이전트 버전 [8.22.181.0](/docs/release-notes/agent-release-notes/net-release-notes/net-agent-8221810) 부터 Azure의 `Always On` 설정은 에이전트를 방해하지 않고 활성화될 수 있습니다. `Always On` 을 사용 [하려면 에이전트를 업데이트하기](/docs/agents/net-agent/azure-installation/install-net-agent-azure-web-apps) 만 하면 됩니다.

## .NET Framework 4.0 이하의 해결 방법 [#4-0-framework]

귀하의 에이전트가 목표 [.NET 프레임워크 4.0 이하인](/docs/agents/net-agent/troubleshooting/technical-support-net-framework-40-or-lower) 경우 Azure의 [`Always On` 설정이](https://azure.microsoft.com/en-us/documentation/articles/web-sites-create-web-jobs/) 방해를 받게 됩니다. 이 기능은 Azure의 <DNT>**Basic**</DNT> 및 <DNT>**Standard**</DNT> 모드에서만 사용할 수 있습니다.

`Always On` 설정은 `APP_POOL_ID` 환경 변수가 설정되지 않도록 합니다. New Relic의 .NET 에이전트는 앱 이름을 가져오기 위해 이 환경 변수가 필요합니다. 에이전트가 응용 프로그램 이름을 결정할 수 없으면 시작되지 않습니다.

여전히 .NET Framework 4.0 이하를 사용하는 경우 다음 해결 방법을 시도하십시오.

Azure Portal에서 Microsoft의 <DNT>**Standard**</DNT> 또는 <DNT>**Basic**</DNT> 티어를 사용하는 경우 `Always On` 설정이 비활성화되어 있는지 확인하세요.

1. Azure Portal에서

   <DNT>
     **(your app) > Settings > Application Settings**
   </DNT>

   선택합니다.

2. `Always On` 토글을 `Off` 으로 설정합니다.

시도해 볼 수 있는 다른 옵션은 다음과 같습니다. 그래도 작동하지 않으면 <DNT>**must**</DNT> [`Always On` 설정을 끄세요](#4-0-framework).

<CollapserGroup>
  <Collapser
    id="name-app"
    title="Kudu를 사용하여 앱 이름을 지정하십시오."
  >
    Kudu를 사용하여 로컬 <DNT>**`newrelic.config`**</DNT> 에서 애플리케이션 이름을 지정하려면 다음 안내를 따르세요.

    1. Microsoft Azure 포털에서 <DNT>**(your app) > Tools > Kudu**</DNT> 선택한 다음 <DNT>**Go**</DNT> 선택합니다.

    2. <DNT>**Debug Console > CMD**</DNT> 선택합니다.

    3. <DNT>**Site > wwwroot > newrelic**</DNT> 선택합니다.

    4. 편집 선택 <Icon name="fe-edit-2"/>`newrelic.config` 아이콘을 클릭하고 다음을 추가합니다.

       ```xml
       <application>
         <name>Your app's name</name>
       </application>
       ```

    5. 애플리케이션을 저장하고 다시 시작합니다.
  </Collapser>

  <Collapser
    id="name-app-web-config"
    title="app/web.config를 사용하여 앱 이름을 지정합니다."
  >
    <Callout variant="important">
      이 옵션은 임시 수정 사항일 뿐입니다. .NET 에이전트가 간헐적으로 보고를 중지할 수 있습니다.
    </Callout>

    `app/web.config` 파일에서 애플리케이션의 이름을 지정하려면:

    1. `appSettings` 에 다음 키를 추가합니다.

       ```xml
       <appSettings>
         <add key="NewRelic.AppName" value="Your app's name" />
       </appSettings>
       ```

    2. 인스턴스를 다시 시작합니다.
  </Collapser>

  <Collapser
    id="pinger"
    title="종합 모니터링을 사용하여 웹사이트를 모니터링합니다."
  >
    `Always On` 사용량에 따라 [종합 모니터링](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) 을 사용하여 웹사이트를 활성 상태로 유지할 수 있습니다.

    * `Always On` 설정을 사용하여

      <DNT>
        **are not**
      </DNT>

      하는 경우 신세틱 모니터링 무료 [핑어 모니터를](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors#monitor-type-ping) 사용해 볼 수 있습니다.

    * `Always On` 설정을 사용하여

      <DNT>
        **are**
      </DNT>

      하는 경우 신세틱 모니터링은 작업자 역할에 대한 엔드포인트에 도달할 수 있어야 합니다. 그렇지 않은 경우 [신세틱스 스크립트 모니터를](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers) 사용하여 사이트에서 실행해 볼 수 있습니다.
  </Collapser>
</CollapserGroup>
