코드 수준 메트릭을 통해 [New Relic CodeStream 확장을](/docs/codestream/how-use-codestream/performance-monitoring#code-level) 사용하는 개발자는 코드의 개별 메서드와 함께 IDE에서 상황에 맞게 표시된 APM 데이터를 볼 수 있습니다. 이를 통해 개발자는 코드를 작성하고 검토할 때 성능 문제를 보다 적극적으로 해결할 수 있습니다.

## 시작하다 [#get-started]

먼저 지원되는 IDE에 New Relic CodeStream 확장을 [설치](/docs/codestream/start-here/install-codestream) 하고 로그인합니다.

<Callout variant="important">
  코드 수준 메트릭은 Node.js 에이전트 버전 9.7.5 이상에서 사용할 수 있으며 버전 10.5.0 이상에서는 기본적으로 활성화됩니다. 이 구성을 변경하려면 [설명서를](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration#code-level-metrics) 확인하십시오.
</Callout>

## 에이전트 속성 [#attributes]

Node.js 에이전트는 애플리케이션의 [웹 프레임워크](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent#support) 에서 생성된 대부분의 범위에 다음 특성을 보고하고 연결합니다.

<Callout variant="important">
  경우에 따라 에이전트는 애플리케이션 내에서 호출되고 IDE 내에는 표시되지 않는 기본 프레임워크 코드를 직접 래핑합니다.
</Callout>
