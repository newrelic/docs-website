Node.js 에이전트는 [npm(Node Package Manager) 저장소](https://npmjs.org/package/newrelic) 와 [GitHub](https://github.com/newrelic/node-newrelic) 에서 공개적으로 사용할 수 있습니다. Node.js 에이전트를 설치하기 전에 애플리케이션이 다음 시스템 요구 사항을 충족하는지 확인하십시오.

아직 없는 경우 [새 리릭 계정을 작성하십시오](https://newrelic.com/signup). 무료입니다. 영원히요.

## Node.js 12 오류 [#node12-errors]

Node.js 12의 경우 다음 변경 사항이 Node.js 에이전트에 영향을 줍니다.

처리되지 않은 거부를 초래하는 오류는 거부된 약속이 생성될 때 활성 상태였던 트랜잭션으로 범위가 지정되지 않습니다. 이는 init async 후크를 트리거하는 프라미스가 더 이상 프라미스 랩 인스턴스를 통해 전달되지 않기 때문입니다. 이렇게 하면 지정된 약속 거부를 예약된 트랜잭션과 연결하는 연결이 끊어집니다.

## 지원되는 Node.js 프레임워크 [#supported-frameworks]

* [익스프레스](https://www.npmjs.com/package/express) 4.6.0 이상
* [수정하다](https://www.npmjs.com/package/restify)
* [연결하다](https://www.npmjs.com/package/connect)
* [하피](https://www.npmjs.com/package/hapi)
* [Koa](https://www.npmjs.com/package/koa) 2.0.0 이상(에이전트와 함께 로드된 [외부 모듈](https://github.com/newrelic/node-newrelic-koa) )
* [금식하다](https://www.npmjs.com/package/fastify)
* [Next.js](https://www.npmjs.com/package/next) 12.0.9 이상

기본 라우터와 함께 지원되는 프레임워크를 사용하는 경우 Node.js 에이전트는 이러한 프레임워크의 경로 이름을 있는 그대로 읽을 수 있습니다. 그러나 프레임워크에서 제공하는 것보다 더 구체적인 이름을 원하면 New Relic이 [Node.js 트랜잭션 명명 API](/docs/nodejs/nodejs-transaction-naming-api) 와 함께 제공하는 도구 중 하나 이상을 사용할 수 있습니다.

<Callout title="EOL 공지">
  2021년 11월에 여러 기능에 대한 지원이 중단됩니다. 여기에는 Node.js 에이전트용 Hapi 19.2 이전 버전과 Oracle 드라이버 패키지가 포함됩니다. 이 전환을 쉽게 준비하는 방법을 비롯한 자세한 내용은 [Explorers Hub 게시물](https://discuss.newrelic.com/t/important-upcoming-changes-to-capabilities-and-support-across-node-agents-suggested-pagerduty-responders-incident-workflows-and-kubernetes-instrumentation/164481?u=dholloran) 을 참조하세요.
</Callout>

## 프로세스 관리자

일반적으로 Node.js의 시작, 중지 및 다시 시작을 처리하는 프로세스 관리자(예: [Forever](https://www.npmjs.com/package/forever) )는 Node.js 에이전트와 호환되어야 합니다. [PM2](https://www.npmjs.com/package/pm2) 를 사용하는 경우 지원되는 최소 PM2 버전은 2.0입니다.

## 보안 요구 사항

[데이터 수집을 위한 표준 보안 수단으로](/docs/accounts-partnerships/accounts/security/data-security) 앱 서버는 SHA-2(256비트)를 지원해야 합니다. SHA-1은 지원되지 않습니다.
