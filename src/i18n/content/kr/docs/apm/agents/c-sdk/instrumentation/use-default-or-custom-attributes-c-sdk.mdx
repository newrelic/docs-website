New Relic에서 **속성** 은 [이벤트](/docs/accounts-partnerships/getting-started-new-relic/glossary#event) 또는 [트랜잭션](/docs/accounts-partnerships/getting-started-new-relic/glossary#transaction) 의 속성을 결정하는 정보를 포함하는 키-값 쌍입니다. 이러한 키-값 쌍을 사용하면 애플리케이션에 대한 통찰력을 높이고 [데이터를 쿼리할 수 있습니다](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder) .

## 속성 보기 및 사용 [#locations]

C 애플리케이션에 대한 [기본 APM 속성](/docs/insights/new-relic-insights/decorating-events/apm-default-attributes-insights#transaction-event) 과 [사용자 정의 속성](/docs/agents/manage-apm-agents/agent-data/collect-custom-attributes) 은 모두 다음 위치에 나타납니다.

* APM [트랜잭션 추적](/docs/apm/transactions/transaction-traces/introduction-transaction-traces) 및 [오류 분석](/docs/apm/applications-menu/error-analytics/introduction-error-analytics)
* [APM 이벤트](/docs/insights/explore-data/attributes/apm-default-attributes-insights)

## C 관련 속성 [#attributes]

<Callout variant="important">
  사용자 정의 속성을 생성하기 전에 [NRQL에서 사용하는 New Relic의 예약 용어](/docs/insights/insights-data-sources/custom-data/insights-custom-data-requirements-limits#reserved-words) 목록을 검토하십시오. 그렇지 않으면 예기치 않은 결과가 발생할 수 있습니다.
</Callout>

C 애플리케이션에 사용자 정의 속성을 추가하려면 [C SDK API의 속성 함수](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#abfe6f64a8eec7d60d8588f8969781d34) 중 하나를 호출하십시오. 예: `newrelic_add_attribute_double()` . 사용자 정의 속성의 키 이름은 함수를 호출할 때 지정하는 항목에 따라 다릅니다.
