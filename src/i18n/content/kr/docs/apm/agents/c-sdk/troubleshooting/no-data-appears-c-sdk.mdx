  2022년 4월부터 C SDK 기능을 지원하지 않습니다. 자세한 내용은 [지원 포럼 게시물 을](https://discuss.newrelic.com/t/q1-bulk-eol-announcement-fy23/181744)참조하십시오.

## 문제

[New Relic C SDK를 설치하고](/docs/install-c-agent-compile-link-your-code) 앱에 대한 트래픽을 생성하고 몇 분을 기다렸지만 여전히 APM에 데이터 보고가 표시되지 않습니다.

## 해결책

이러한 문제 해결 팁은 가장 일반적인 문제에서 가장 일반적이지 않은 문제 순으로 대략적으로 정렬되어 있습니다.

1. 오류에 대한 [C SDK 및 데몬 로그 파일](/docs/generate-logs-troubleshooting-c-sdk) 의 출력을 확인하십시오.

2. 계측된 애플리케이션을 시작하면서 [데몬 프로세스(`newrelic-daemon`)가](/docs/install-c-sdk-compile-link-your-code#daemon) 호출되도록 <DNT>**before**</DNT> 실행 중인지 확인하세요.

3. [C SDK 설정](/docs/c-sdk-configuration) 에서 앱 이름과 <InlinePopover type="licenseKey"/>확인하세요.

4. 다음과 같이 전화하십시오.

   * [`newrelic_create_app()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a61dd90439ae3cc5060021f6ab4701132)
   * 해당하는 경우 [`newrelic_start_web_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#adbf7c1fa57482f6e0a7f291e0b5ec80f) 또는 [`newrelic_start_non_web_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a88648cc287f8d7e371139dc3809b7afb)
   * [`newrelic_end_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#acbd88933d620c21b7b19f12b6f042da8)

   또한 이러한 호출이 `false` 또는 `NULL` 을 반환하지 않는지 확인합니다.

5. 필요한 경우 [`newrelic_version()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a8aec87e54af38e466fc5f715ef591d10) 을 사용하여 C SDK 버전을 확인하고 C [SDK 라이브러리를 최신 릴리스로 업데이트합니다](/docs/update-your-c-sdk-library) .
