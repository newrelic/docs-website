## 통사론

```py
newrelic.agent.capture_request_params(flag=True)
```

트랜잭션의 쿼리 문자열 매개변수를 캡처할 수 있습니다.

## 설명

이 호출을 통해 웹 트랜잭션의 쿼리 문자열 매개변수를 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 으로 캡처할 수 있습니다. 이 컬렉션은 보안상의 이유로 기본적으로 차단되어 있습니다. 대신 전체 앱에 대해 쿼리 문자열 매개변수 수집을 활성화하려면 [`record_sql`](/docs/agents/python-agent/installation-configuration/python-agent-configuration#txn-tracer-sql) 구성 설정을 사용하세요.

[높은 보안 모드](/docs/accounts-partnerships/accounts/security/high-security) 는 활성 상태인 경우 이 호출을 무시하며, 이 경우 이 호출은 효과가 없습니다. 이 호출은 에이전트 [구성](/docs/agents/python-agent/installation-configuration/python-agent-configuration) 의 다른 관련 설정보다 우선합니다.

<Callout variant="tip">
  WSGI 환경 사전에서 쿼리 문자열 캡처를 활성화할 수도 있습니다. 이렇게 하려면 WSGI 서버에서 대상 WSGI 앱으로 전달한 WSGI 환경 사전의 특정 요청에 대해 `newrelic.capture_request_params` 키에 플래그를 지정합니다.
</Callout>
