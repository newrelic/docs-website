## 표준 설치 [#install]

이 지침은 [Python 에이전트가 자동으로 계측하는 프레임워크의 응용 프로그램을](/docs/agents/python-agent/getting-started/instrumented-python-packages) 위한 것입니다.

* 웹이 아닌 Python 스크립트, 사용자 정의 백그라운드 작업 또는 자동 계측되지 않는 프레임워크의 애플리케이션을 모니터링하려면 [웹이 아닌 스크립트 또는 기능 모니터링](/docs/agents/python-agent/supported-features/python-background-tasks) 에 대한 설명서를 참조하십시오.
* Docker를 사용하는 경우 각 컨테이너 내에 에이전트를 설치합니다.

[자동 계측 프레임워크](/docs/agents/python-agent/getting-started/instrumented-python-packages) 에 애플리케이션용 New Relic Python 에이전트를 설치하려면 다음을 수행합니다.

1. 아직 없는 경우 [새 리릭 계정을 작성하십시오](https://newrelic.com/signup). 무료입니다. 영원히요.

2. New Relic [라이센스 키](/docs/subscriptions/license-key) 가 있는지 확인하십시오. (아직 New Relic 계정이 없다면 [가입하여](https://newrelic.com/signup) 계정을 만드세요.)

3. 다음을 실행하여 **pip** 를 사용하여 **newrelic** Python 패키지를 설치합니다.

   ```
   pip install newrelic
   ```

4. 다음 명령을 실행하여 에이전트 구성 파일을 생성합니다.

   ```
   newrelic-admin generate-config <var>YOUR_LICENSE_KEY</var> newrelic.ini
   ```

5. 웹 애플리케이션 또는 WSGI 서버가 시작되는 방식을 제어하는 경우 일반적인 시작 명령 옵션 앞에 다음 New Relic 관리 스크립트 명령을 사용합니다.

   ```
   NEW_RELIC_CONFIG_FILE=newrelic.ini newrelic-admin run-program <var>YOUR_COMMAND_OPTIONS</var>
   ```

   이 방법으로 관리 스크립트를 실행할 수 없는 경우 추가 컨텍스트 및 기타 설치 옵션에 대한 [고급 설치 지침](/docs/agents/python-agent/installation-configuration/python-agent-installation) 을 읽으십시오.

6. Python 웹 애플리케이션 또는 WSGI 서버를 시작하거나 다시 시작합니다. 5분 정도 기다렸다가 New Relic UI로 이동하여 데이터를 확인하세요.

자세한 설치 절차 및 유용한 컨텍스트는 [Python 에이전트 설치](/docs/agents/python-agent/installation-configuration/python-agent-installation) 를 참조하십시오.

## APM 및 인프라 데이터에 대한 로그 보기 [#logs-context]

또한 로그와 애플리케이션의 데이터를 함께 가져와 더 쉽고 빠르게 문제를 해결할 수 있습니다. [컨텍스트의 로그를](/docs/logs/logs-context/configure-logs-context-python/) 사용하면 앱의 UI에서 직접 오류 및 추적과 관련된 로그 메시지를 볼 수 있습니다. Kubernetes 클러스터와 같은 [인프라 데이터](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/) 컨텍스트에서 로그를 볼 수도 있습니다. 다른 UI 페이지로 전환할 필요가 없습니다.
