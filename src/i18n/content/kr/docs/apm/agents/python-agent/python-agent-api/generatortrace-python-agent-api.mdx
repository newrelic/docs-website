## 통사론

```py
newrelic.agent.generator_trace(name=None, group=None, label=None, params=None, terminal=False)
```

기본적으로 계측되지 않은 제너레이터를 계측하는 데 사용됩니다.

## 설명

`generator_trace` 제너레이터 함수에 추가하는 데코레이터입니다. 이 데코레이터를 추가하면 추가적인 [트랜잭션](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) 정보(트랜잭션 트레이스 정보 포함)를 수집할 수 있습니다.

`generator_trace` 데코레이터는 에이전트 버전 2.102.0.85 이상에서 제너레이터 및 코루틴에 사용할 수 있습니다. 이러한 객체의 타이밍은 소비가 시작될 때 시작되며, 객체가 소진되거나 범위를 벗어날 때 종료됩니다.

정적 및 클래스 메서드를 포함하여 기존 데코레이터와 함께 데코레이터를 사용할 수 있습니다. 여러 데코레이터를 중첩할 때 `generator_trace` 데코레이터를 가장 바깥쪽 데코레이터로 사용하세요.

## 대체 호출 양식 [#alternate]

데코레이터를 사용할 수 없는 설정의 경우 다음 대체 호출 양식을 사용할 수 있습니다.

### 래퍼

트레이스하려는 특정 제너레이터가 어디에 있는지 미리 알고 있다면 `generator_trace` 데코레이터를 사용할 수 있습니다. 하지만 트레이스해야 하는 모든 제너레이터를 <DNT>**don&apos;t**</DNT> 알지 못하는 경우(예: 라우팅 시스템의 일부로 동적으로 조회되는 경우), 등록 시점이나 호출 시점에 제너레이터를 래핑하려면 `GeneratorTraceWrapper` 을(를) 사용해야 합니다.

### 경로 기반 래핑 [#path-based]

`wrap_generator_trace` 선언된 코드 외부에서 제너레이터를 래핑하는 데 사용됩니다. 예를 들어: 수정하고 싶지 않은 라이브러리 코드를 계측하는 데 이를 사용할 수 있습니다.

이러한 호출 형식 간의 차이점에 대한 자세한 내용은 다른 [호출](/docs/python-agent-api-different-call-forms) 형식 을 참조하십시오.

## 예

### `generator_trace` [#trace\_example][#trace_example]

`generator_trace` 데코레이터 사용 예:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
def gen():
    for i in range(5):
        yield i
```

비동기 제너레이터와 함께 `generator_trace` 데코레이터를 사용하는 예:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
async def gen():
    for i in range(5):
        yield i
```

### 래퍼 예 [#trace-wrap-example]

`GeneratorTraceWrapper` 사용 예:

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen)
```

메트릭 이름을 엔드포인트 이름으로 지정하려는 경우(호출되는 생성기의 식별자를 기반으로 메트릭 이름을 지정하는 대신), `GeneratorTraceWrapper` 객체가 생성될 때 사용할 이름과 대체 메트릭 경로 접두사를 제공할 수 있습니다.

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name="gen_5", group="Python/Generator")
```

메트릭 이름을 동적으로(호출되는 생성자의 식별자를 기반으로 메트릭 이름을 지정하는 대신) 지정하려면, `GeneratorTraceWrapper` 객체가 생성될 때 이름을 생성자로 제공할 수 있습니다.

```py
import newrelic.agent

def gen(x):
    for i in range(x):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name = lambda x: f"gen_{x}")
```
