Python 에이전트는 [PostgresSQL](https://www.postgresql.org/) 용 [asyncpg](https://pypi.org/project/asyncpg/) 데이터베이스 클라이언트 모듈을 사용할 때 데이터베이스 트랜잭션 기록을 지원합니다. 이 기능은 데이터베이스 쿼리의 타이밍, 데이터베이스 쿼리에 대한 SQL 문 및 긴 데이터베이스 쿼리에 대한 스택 추적을 캡처합니다.

Python 에이전트를 [asyncpg](https://pypi.org/project/asyncpg/) 를 사용하는 앱과 통합하기 위한 요구 사항 및 팁에 대해 읽어보세요. 일반 설치 지침으로 돌아가려면 [Python 에이전트 설치](/docs/agents/python-agent/installation-configuration/python-agent-installation) 로 이동하십시오.

## 관리 스크립트로 자동 초기화 [#wrapper-script]

`python app.py` 으로 앱을 시작하고 Python 에이전트 **버전 5.20.0.149** 이상을 사용하는 경우 권장되는 [관리 스크립트 통합 방법](/docs/agents/python-agent/installation-configuration/python-agent-installation#integration) 을 사용할 수 있습니다. 예를 들면 다음과 같습니다.

```
NEW_RELIC_CONFIG_FILE=path/to/newrelic.ini newrelic-admin run-program python path_to_app
```

## Python 에이전트 API 사용 [#api]

[Python 에이전트 API](/docs/agents/python-agent/python-agent-api) 를 사용하여 모든 ASGI 서버 또는 프레임워크를 모니터링할 수 있습니다. 이렇게 하려면 [ASGI 애플리케이션 진입점](/docs/agents/python-agent/python-agent-api/asgiapplication-python-agent-api) 을 표시하고 [트랜잭션 이름](/docs/agents/python-agent/python-agent-api/settransactionname-python-agent-api) 을 설정하십시오.

## 이벤트 루프 진단 지원 [#event-loops]

Python 에이전트는 `asyncio` 이벤트 루프 진단을 지원합니다. 자세한 내용은 [Python 이벤트 루프 진단](/docs/agents/python-agent/supported-features/python-event-loop-diagnostics) 을 참조하세요.
