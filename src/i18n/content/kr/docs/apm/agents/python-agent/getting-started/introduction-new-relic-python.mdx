Python 에이전트는 Python 애플리케이션을 모니터링하여 [성능 문제](#monitor-performance) 를 식별하고 해결하는 데 도움을 줍니다. 또한 성능 모니터링을 확장하여 [비즈니스 데이터](#business-data) 를 수집 및 분석하여 고객 경험을 개선하고 데이터 기반 비즈니스 결정을 내릴 수 있습니다. [사용자 지정 계측 및 API](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation) 에 대한 유연한 옵션을 통해 Python 에이전트는 앱에서 필요한 데이터를 사용자 지정하기 위한 여러 빌딩 블록을 제공합니다.

Python은 Django, Gunicorn, WSGI, CherryPy, uWSGI 등을 포함한 다양한 [웹 프레임워크 및 호스팅 메커니즘과](/docs/agents/python-agent/getting-started/instrumented-python-packages) 함께 작동합니다. [Google App Engine 가변형 환경](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment) 에 Python 에이전트를 설치할 수도 있습니다.

## 앱 성능 모니터링 [#monitor-performance]

Python 에이전트를 [설치](#installation) 하면 앱에 대한 데이터 수집이 시작됩니다. [New Relic UI](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one) 에서 데이터를 차트 및 테이블로 볼 수 있습니다.

<DNT>
  **View the big picture of your app:**
</DNT>

* 앱의 [Apdex(사용자 만족도)](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) 를 모니터링합니다.
* [요약 페이지](/docs/apm/applications-menu/monitoring/apm-overview-page) 를 사용하여 앱에 대한 높은 수준의 요약을 얻으십시오.
* [분산 추적](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) 을 활성화하여 많은 서비스가 있는 아키텍처 전반의 활동을 확인합니다.
* [인프라 모니터링](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure) 을 설치하고 앱에 대한 자세한 서버/호스트 데이터를 봅니다.

* 중요한 메트릭에 대한 [사용자 지정 대시보드](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards) 를 만듭니다.
* 사용자에게 영향을 [미치기](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) 전에 오류나 문제가 발생하면 팀에 알립니다.
* [배포 후](/docs/apm/applications-menu/events/deployments-page) 성능 보기 .

## Python 에이전트 설치 [#installation]

Python 에이전트를 설치하기 전에 시스템이 [시스템 요구 사항](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent) 을 충족하는지 확인하십시오. 또한 [New Relic 계정](https://newrelic.com/signup) 을 만들어야 합니다.

Django, WSGI 및 Gunicorn을 포함하여 즉시 사용 가능한 여러 웹 프레임워크 및 라이브러리를 지원합니다. [지원되는 웹 프레임워크](/docs/agents/python-agent/getting-started/instrumented-python-packages) 중 하나를 사용하면 설치가 쉽습니다. 지원되지 않는 프레임워크를 사용하는 경우 프로세스에 앱 코드 및/또는 웹 서버 파일에 몇 가지 추가 작업이 포함됩니다.

대부분의 설정에서 작동하는 빠르고 간단한 설치 프로세스를 위해 다음의 간단한 단계를 따르십시오.

1. Python 패키지를 다운로드하여 설치합니다.
2. 구성 파일을 만듭니다.
3. Python 에이전트를 애플리케이션과 통합합니다.

## 웹이 아닌 스크립트, 백그라운드 작업 및 기능 모니터링 [#non-web]

또한 Python 에이전트를 사용하면 웹이 아닌 스크립트, 작업자 프로세스, 작업 및 기능을 모니터링할 수 있습니다. 웹이 아닌 트랜잭션에 대한 설치 프로세스는 웹 앱에 사용되는 프로세스와 비슷하지만 한 가지 큰 차이점이 있습니다. [설치 지침](/docs/agents/python-agent/getting-started/python-agent-quick-start) 에 설명된 표준 통합 프로세스를 거치는 대신 원하는 기능을 수동으로 "래핑"합니다. 감시 장치. 자세한 내용은 [웹이 아닌 작업 및 프로세스](/docs/agents/python-agent/supported-features/python-background-tasks) 를 참조하십시오. Celery 작업 모니터링에 대한 지침은 [Celery 백그라운드 작업](/docs/agents/python-agent/back-end-services/python-agent-celery) 을 참조하십시오.

## 설치 후 다음은 무엇입니까? [#instrumentation]

에이전트를 시작하고 실행하면 다음 단계가 제안됩니다.

* 에서 데이터를 탐색하고 사용자 인터페이스에 익숙해집니다.
* [다른 관찰 가능성 솔루션](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability) 및 [APM 페이지](/docs/apm) 에서 문서를 읽어보십시오.
* [애플리케이션의 이름](/docs/agents/manage-apm-agents/app-naming/rename-your-application) 또는 기타 [구성 옵션을](/docs/agents/python-agent/installation-configuration/python-agent-configuration) 변경합니다.
* 기본적으로 모니터링되지 않는 애플리케이션 활동에 대한 [사용자 지정 계측](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation) 설정에 대해 알아보세요.
* [Python Telemetry SDK](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic) 를 고려하십시오.

## 문제점 해결

설치 프로세스를 완료한 후 5분 이내에 [APM UI](/docs/apm/applications-menu/monitoring/apm-overview-page) 에 데이터가 표시되어야 합니다. 그렇지 않은 경우 다음 문제 해결 리소스를 사용하십시오.

* 데이터가 표시되지 않으면 다음 [문제 해결 단계](/docs/agents/python-agent/troubleshooting/no-data-appears-python) 를 따르세요.
* 새 호스트에서 Python 에이전트를 설치하거나 실행할 때 문제가 발생 [하면 패키지가 올바르게 설치되었는지 테스트하고](/docs/agents/python-agent/installation-configuration/testing-python-agent) New Relic의 데이터 수집기 서비스에 연결할 수 있는지 테스트하십시오.
* 다른 문제에 대해서는 [문제 해결 문서의 전체 목록을](/docs/agents/python-agent/troubleshooting/) 참조하십시오.
