뉴렐릭의 Python 에이전트는 Python 애플리케이션을 모니터링하여 [성능 문제](#monitor-performance)를 식별하고 해결하는 데 도움을 줍니다. 또한 성능 모니터링을 확장하여 [비즈니스 데이터](#business-data)를 수집 및 분석하여 고객 경험을 개선하고 데이터 기반 비즈니스 결정을 내릴 수 있습니다. [커스텀 계측 및 API](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation)에 대한 유연한 옵션을 통해, Python 에이전트는 앱에서 필요한 데이터를 맞춤화하는 데 필요한 다양한 구성 요소를 제공합니다.

Python은 Django, Gunicorn, WSGI, CherryPy, uWSGI 등을 포함한 다양한 [웹 프레임워크 및 호스팅 메커니즘](/docs/agents/python-agent/getting-started/instrumented-python-packages)과 함께 작동합니다. [Google App Engine 가변형 환경](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment)에 Python 에이전트를 설치할 수도 있습니다.

## 앱 성능 모니터링 [#monitor-performance]

Python 에이전트를 [설치](#installation)하면 앱에 대한 데이터 수집이 시작됩니다. [뉴렐릭 UI](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)에서 데이터를 차트와 테이블로 확인할 수 있습니다.

**앱에 대한 개괄적인 정보 확인:**

* 앱의 [Apdex(사용자 만족도)](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) 모니터링
* [요약 페이지](/docs/apm/applications-menu/monitoring/apm-overview-page)에서 앱에 대한 대략적인 요약 확인
* [분산 추적](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)을 활성화하여 많은 서비스가 있는 아키텍처 전반에서 활동 확인
* [인프라 모니터링](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure)을 설치하고 앱에 대한 자세한 서버/호스트 데이터 확인

**오류 및 문제의 신속한 발견:**

* 비즈니스와 관련된 [주요 트랜잭션](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events) 추적
* 중요한 메트릭에 대한 [커스텀 대시보드](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards) 생성
* 오류나 문제가 발생하면 사용자에게 영향을 미치기 전에 팀에게 [알림](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) 전송
* [배포 후](/docs/apm/applications-menu/events/deployments-page) 성능 확인

**성능 세부 정보 분석:**

* 코드 수준의 [트랜잭션 트레이스](/docs/apm/transactions/transaction-traces/transaction-traces) 검토
* [데이터베이스 쿼리 트레이스](/apm/transactions/transaction-traces/troubleshoot-sql-statements) 검토
* [오류 추적](/docs/apm/applications-menu/events/view-apm-error-analytics) 검토
* [스레드 프로파일러 세션](/docs/apm/applications-menu/events/thread-profiler-tool)을 사용하여 샘플링된 스레드의 자세한 스택 트레이스를 확인할 수 있습니다.

**APM 및 인프라 데이터에 대한 로그 보기:**

로그와 애플리케이션의 데이터를 함께 가져와 더 쉽고 빠르게 문제를 해결할 수 있습니다. 다른 UI 페이지로 전환할 필요가 없습니다.

* [문맥적 로그](/docs/logs/logs-context/configure-logs-context-python/)를 사용하면 앱의 UI에서 직접 오류 및 트레이스와 관련된 로그 메시지를 볼 수 있습니다.
* 쿠버네티스 클러스터 같은 [인프라 데이터](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)의 문맥적 로그를 확인할 수도 있습니다.

## Python 에이전트 설치 [#installation]

Python 에이전트를 설치하기 전에 시스템이 [다음 요구 사항](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent)을 충족하는지 확인하십시오. 또한 [뉴렐릭 계정](https://newrelic.com/signup)이 필요합니다.

뉴렐릭은 Django, WSGI 및 Gunicorn 등 즉시 사용 가능한 여러 웹 프레임워크 및 라이브러리를 지원합니다. [지원되는 웹 프레임워크](/docs/agents/python-agent/getting-started/instrumented-python-packages) 중 하나를 사용하면 쉽게 설치할 수 있습니다. 지원되지 않는 프레임워크를 사용하는 경우 프로세스에 앱 코드 및/또는 웹 서버 파일에 몇 가지를 추가해야 합니다.

대부분의 설정에서 작동하는 빠르고 간단한 설치 프로세스의 경우, 다음의 간단한 단계를 따르십시오.

1. Python 패키지를 다운로드하여 설치합니다.
2. 구성(config) 파일을 생성합니다.
3. Python 에이전트를 애플리케이션과 통합합니다.

## 비웹 스크립트, 백그라운드 작업 및 기능 모니터링 [#non-web]

또한 Python 에이전트는 비웹 스크립트, 작업자 프로세스, 작업 및 기능을 모니터링할 수 있도록 해줍니다. 비웹 트랜잭션에 대한 설치 프로세스는 웹 앱에 사용되는 프로세스와 비슷하지만, 한 가지 큰 차이점이 있습니다. [설치 지침](/docs/agents/python-agent/getting-started/python-agent-quick-start)에 설명된 표준 통합 프로세스를 거치는 대신 원하는 기능을 수동으로 '래핑'합니다. 보다 자세한 내용은 [비웹 작업 및 프로세스](/docs/agents/python-agent/supported-features/python-background-tasks)를 참조하십시오. Celery 작업 모니터링에 대한 지침은 [Celery 백그라운드 작업](/docs/agents/python-agent/back-end-services/python-agent-celery)을 참조하십시오.

## 설치 후 무엇을 해야 하나요? [#instrumentation]

에이전트를 시작하고 실행하면 아래와 같은 다음 단계가 제안됩니다.

* 데이터를 탐색하고 사용자 인터페이스에 익숙해집니다.
* [다른 옵저버빌리티 솔루션](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability) 및 [APM 페이지](/docs/apm)에 관한 뉴렐릭의 문서를 참조하십시오.
* [애플리케이션의 이름](/docs/agents/manage-apm-agents/app-naming/rename-your-application) 또는 기타 [설정 옵션](/docs/agents/python-agent/installation-configuration/python-agent-configuration)을 변경합니다.
* 기본적으로 모니터링되지 않는 애플리케이션 활동에 대한 [커스텀 계측](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation)을 설정하는 방법을 알아봅니다.
* [Python Telemetry SDK](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic)를 고려해보십시오.

## 문제 해결

설치 프로세스를 완료한 후 5분 이내에 [APM UI](/docs/apm/applications-menu/monitoring/apm-overview-page)에 데이터가 표시되어야 합니다. 그렇지 않은 경우 다음 문제 해결 자료를 사용하십시오.

* 데이터가 표시되지 않으면 다음 [문제 해결 단계](/docs/agents/python-agent/troubleshooting/no-data-appears-python)를 따릅니다.
* 새 호스트에 Python 에이전트를 설치하거나 실행할 때 문제가 발생[하면 패키지가 올바르게 설치되었는지 테스트하고](/docs/agents/python-agent/installation-configuration/testing-python-agent) 뉴렐릭의 데이터 수집기 서비스에 연결할 수 있는지 테스트합니다.
* 다른 문제의 경우, [전체 문제 해결 문서 목록](/docs/agents/python-agent/troubleshooting/)을 참조합니다.
