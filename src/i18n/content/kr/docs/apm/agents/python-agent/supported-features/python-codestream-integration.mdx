[New Relic CodeStream](/docs/codestream/how-use-codestream/performance-monitoring#code-level) 을 Python 에이전트와 통합하여 상황에 맞는 APM 데이터를 IDE에 직접 표시합니다. 이 통합을 통해 편집기에서 코드 수준 프로덕션 원격 분석을 작성 및 검토 코드로 시각화합니다.

## 시작하기

먼저 지원되는 IDE에 New Relic CodeStream 확장을 [설치](/docs/codestream/start-here/install-codestream) 하고 로그인합니다.

<Callout variant="important">
  New Relic Code Level Metrics 통합은 Python 에이전트 버전 7.10.0.175 이상에서 사용할 수 있으며 기본적으로 활성화되어 있습니다. 이 구성을 변경하려면 설명서를 확인하십시오.
</Callout>

## 에이전트 속성

Python 에이전트는 애플리케이션에서 생성된 범위에 다음 속성을 보고하고 연결합니다.

<table>
  <thead>
    <tr>
      <th>
        **이름**
      </th>

      <th>
        **설명**
      </th>

      <th>
        **예시**
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `code.function`
      </td>

      <td>
        계측된 함수의 이름
      </td>

      <td>
        validate_credentials
      </td>
    </tr>

    <tr>
      <td>
        `code.filepath`
      </td>

      <td>
        `code.function` 이 정의된 소스 코드 파일의 절대 경로
      </td>

      <td>
        /usr/local/apps/test-app/main.py
      </td>
    </tr>

    <tr>
      <td>
        `code.lineno`
      </td>

      <td>

      </td>

        639
      </td>
    </tr>

      <td>
        `code.namespace`
      </td>

      <td>
        `code.function` 이 정의된 네임스페이스(클래스/모듈 이름)
      </td>

      <td>

      </td>
    </tr>
  </tbody>

<Callout variant="important">
  애플리케이션 코드의 모든 기능이 위의 속성으로 계측되는 것은 아닙니다. 경우에 따라 에이전트는 애플리케이션 내에서 호출되고 IDE 내에서 표시되지 않는 기본 프레임워크 코드(예: Django, Flask 등)를 직접 래핑합니다.
</Callout>
