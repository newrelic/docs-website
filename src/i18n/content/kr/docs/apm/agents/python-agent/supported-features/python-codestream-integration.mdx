코드 수준 메트릭을 통해 [New Relic CodeStream 확장을](/docs/codestream/how-use-codestream/performance-monitoring#code-level) 사용하는 개발자는 코드의 개별 메서드와 함께 IDE에서 상황에 맞게 표시된 APM 데이터를 볼 수 있습니다. 이를 통해 개발자는 코드를 작성하고 검토할 때 성능 문제를 보다 적극적으로 해결할 수 있습니다.

## 시작하기

먼저 지원되는 IDE에 New Relic CodeStream 확장을 [설치](/docs/codestream/start-here/install-codestream) 하고 로그인합니다.

<Callout variant="important">
  코드 수준 지표는 Python 에이전트 버전 7.10.0.175 이상에서 사용할 수 있으며 기본적으로 활성화되어 있습니다. 이 구성을 변경하려면 [설명서를](/docs/apm/agents/python-agent/configuration/python-agent-configuration/#codestream-integration)확인하십시오.
</Callout>

## 에이전트 속성

Python 에이전트는 애플리케이션에서 생성된 범위에 다음 속성을 보고하고 연결합니다.

<table>
  <thead>
    <tr>
      <th>
        **이름**
      </th>

      <th>
        **설명**
      </th>

      <th>
        **예시**
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `code.function`
      </td>

      <td>
        계측된 함수의 이름

      <td>
        validate_credentials
      </td>
    </tr>

    <tr>
      <td>
        `code.filepath`
      </td>

      <td>

      </td>

      <td>
        /usr/local/apps/test-app/main.py
      </td>
    </tr>

    <tr>
      <td>

      </td>

      </td>

      </td>
    </tr>
  </tbody>

<Callout variant="important">
  애플리케이션 코드의 모든 기능이 위의 속성으로 계측되는 것은 아닙니다. 경우에 따라 에이전트는 애플리케이션 내에서 호출되고 IDE 내에서 표시되지 않는 기본 프레임워크 코드(예: Django, Flask 등)를 직접 래핑합니다.
</Callout>
