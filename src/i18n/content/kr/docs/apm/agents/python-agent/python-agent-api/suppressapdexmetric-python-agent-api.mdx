## 통사론

```py
newrelic.agent.suppress_apdex_metric(flag=True)
```

현재 트랜잭션이 Apdex에 기여하는 것을 방지합니다.

## 설명

이 호출은 웹 트랜잭션에 대한 [Apdex 지표](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) 생성을 억제합니다. 장기 실행 트랜잭션이 애플리케이션의 평균 Apdex 점수에 자주 영향을 미치는 경우 이 작업을 수행할 수 있습니다.

이전에 억제된 트랜잭션의 억제를 해제하려면 플래그를 `False` 으로 설정합니다.

<Callout variant="tip">
  WSGI 환경 사전에서 Apdex 메트릭을 억제할 수도 있습니다. 이렇게 하려면 WSGI 서버가 모니터링 중인 WSGI 애플리케이션으로 전달한 WSGI 환경 사전의 특정 요청에 대해 `newrelic.suppress_apdex_metric` 키를 설정합니다.
</Callout>

## 예

### 특정 거래에 대해 Apdex 끄기 [#apdex-example]

앱에 평균 Apdex를 낮추는 자주 실행되는 기능이 있는 경우 트랜잭션이 생성되는 위치에서 `suppress_apdex_metric` 를 호출할 수 있습니다.

```py
import newrelic.agent
newrelic.agent.suppress_apdex_metric()
```
