이 문서에서는 Python 에이전트를 `Apache/mod_wsgi` 또는 `mod_wsgi-express` 을 사용하는 앱과 통합하기 위한 요구사항 및 팁을 설명합니다. 일반 설치 지침으로 돌아가려면 [Python 에이전트 설치](/docs/agents/python-agent/installation-configuration/python-agent-installation) 로 이동하십시오.

## Apache/mod_wsgi 사용 [#use-apache-mod-wsgi]

Python 에이전트는 [`Apache/mod_wsgi`](http://www.modwsgi.org/) 사용을 지원합니다. Apache/mod_wsgi를 사용할 때 [Python 애플리케이션과의 수동 통합에](/docs/agents/python-agent/installation-and-configuration/python-agent-integration#manual-integration) 대한 표준 지침을 따르십시오.

<Callout variant="caution">
  명령줄을 통해 admin 스크립트를 실행하는 권장 통합 절차는 Apache/mod_wsgi와 함께 사용할 수 없습니다.
</Callout>

## mod_wsgi-express 사용 [#use-mod-wsgi-express]

`mod_wsgi-express` 이라는 새로운 기능이 `mod_wsgi` 버전 4.1.0에 도입되었으며 에이전트를 통합하는 대체 방법을 제공합니다.

WSGI 애플리케이션과 함께 `mod_wsgi-express` 을 사용하는 경우 앱의 시작 명령과 함께 스크립트의 `--with-newrelic` 옵션을 사용할 수 있습니다. `mod_wsgi-express` 스크립트와 `--with-newrelic` 플래그는 `newrelic-admin` 스크립트와 다르지만 서버 실행을 래핑하도록 애플리케이션을 구성하는 동일한 목적을 수행합니다.

`--with-newrelic` 옵션을 사용할 때 Python 에이전트가 설치되어 있고 `newrelic.ini` 구성 파일을 생성했는지 확인하십시오.

예를 들어 Django와 함께 에이전트를 사용하려면 다음과 유사한 명령이 필요할 수 있습니다.

```
NEW_RELIC_CONFIG_FILE=newrelic.ini mod_wsgi-express start-server <var>mysite</var>/wsgi.py --with-newrelic
```
