## 통사론

```py
newrelic.agent.memcache_trace(command, async_wrapper=None)
```

기본적으로 계측되지 않는 함수에 대해 memcache 명령 함수를 계측하는 데 사용됩니다.

## 설명

`memcache_trace` memcache 명령 함수에 추가하는 데코레이터입니다. 이 데코레이터를 추가하면 추가적인 [트랜잭션](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) 정보(트랜잭션 트레이스 정보 포함)를 수집할 수 있습니다.

`memcache_trace` 데코레이터는 에이전트 버전 2.102.0.85 이상에서 함수에 사용할 수 있습니다. 이러한 객체의 타이밍은 소비가 시작될 때 시작되며, 객체가 소진되거나 범위를 벗어날 때 종료됩니다.

정적 및 클래스 메서드를 포함하여 기존 데코레이터와 함께 데코레이터를 사용할 수 있습니다. 여러 데코레이터를 중첩할 때 `memcache_trace` 데코레이터를 가장 바깥쪽 데코레이터로 사용하세요.

## 대체 호출 양식 [#alternate]

데코레이터를 사용할 수 없는 설정의 경우 다음 대체 호출 양식을 사용할 수 있습니다.

### 래퍼

트레이스에 추가하려는 특정 기능이 어디에 있는지 미리 알고 있다면 `memcache_trace` 데코레이터를 사용할 수 있습니다. 그러나 트레이스가 되어야 하는 모든 함수를 <DNT>**don&apos;t**</DNT> 알고 있는 경우(예를 들어 라우팅 시스템의 일부로 동적으로 조회되는 경우) `MemcacheTraceWrapper` 를 사용하여 트레이스에서 함수를 래핑해야 합니다. 등록시 또는 전화시.

### 경로 기반 래핑 [#path-based]

`wrap_memcache_trace` 함수가 선언된 코드 외부에서 함수를 래핑하는 데 사용됩니다. 예를 들어 수정하고 싶지 않은 라이브러리 코드를 계측하는 데 사용할 수 있습니다.

이러한 호출 형식 간의 차이점에 대한 자세한 내용은 다른 [호출](/docs/python-agent-api-different-call-forms) 형식 을 참조하십시오.

## 예

### `memcache_trace` [#trace\_example][#trace_example]

`memcache_trace` 데코레이터 사용 예:

```py
import newrelic.agent

@newrelic.agent.memcache_trace("get")
def get(id):
    ...
```

### 래퍼 예 [#trace-wrap-example]

`MemcacheTraceWrapper` 사용 예:

```py
import newrelic.agent

def get(id):
    ...

get = newrelic.agent.MemcacheTraceWrapper(get, command="get")
```

명령의 값을 동적으로 반환하려면 `MemcacheTraceWrapper` 객체가 생성될 때 명령을 함수로 제공할 수 있습니다.

```py
import newrelic.agent

def get(id):
    ...

get = newrelic.agent.MemcacheTraceWrapper(get, command=lambda id: f"get:{id}")
```
