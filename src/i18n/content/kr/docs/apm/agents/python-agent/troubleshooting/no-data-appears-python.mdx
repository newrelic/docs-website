## 문제

Python 에이전트를 설치하고 약간의 트래픽을 생성하고 5분을 기다린 후 데이터가 New Relic UI에 표시되지 않습니다.

## 해결책

다음은 누락된 데이터 문제를 해결하기 위한 몇 가지 팁이며 문제가 될 가능성이 있는 대략적인 순서로 배치되어 있습니다.

1. 애플리케이션과 동일한 환경에 에이전트를 설치했는지 확인하십시오. 가상 환경을 사용하는 경우 거기에 설치하십시오.

2. 에이전트가 최신 상태인지 확인하세요.

   <DNT>
     **pip**
   </DNT>

   또는

   <DNT>
     **easy_install**
   </DNT>

   사용하지 않고 뉴렐릭 사이트에서 다운로드한 경우 버전 번호를 [현재 릴리스](https://download.newrelic.com/python_agent/release/) 와 비교하세요.

3. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) 를 사용하여 문제를 자동으로 식별해 보십시오.

4. [Python 에이전트 테스트](/docs/python/testing-the-python-agent) 를 실행하여 데이터가 New Relic UI에 도달할 수 있는지 확인하십시오.

5. [호환성 및 요구 사항](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent) 을 충족하는지 확인하십시오. 사용 중인 [웹 프레임워크 또는 웹 서버](/docs/agents/python-agent/hosting-mechanisms) 에 대한 특별한 요구 사항이 있는지 확인합니다.

6. `newrelic-admin generate-config` 명령으로 생성된 설정 파일을 검토하여 올바른

   <InlinePopover type="licenseKey"/>

   포함되어 있는지 확인하세요.

7. [문제 해결을 위한 자세한 로그를 생성](/docs/agents/python-agent/troubleshooting/generating-logs-troubleshooting) 합니다. 그런 다음 로그 파일에서 오류 메시지를 확인하십시오. 로그 파일이 없으면 해당 위치에 쓸 수 있는 권한이 있는지 확인하십시오.

또한 데이터가 표시되지 않고 앱 활성화 또는 재활성화에 대한 경고 메시지가 표시되면 [활성화 문제 해결 절차](/docs/agents/python-agent/troubleshooting/activate-application-warning-python) 를 따르십시오.
