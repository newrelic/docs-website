## 누가 Teams에 액세스할 수 있나요? [#teams-permissions]

모든 사용자가 항상 팀 소유권을 파악할 수 있도록 팀은 계정 수준이 아닌 조직 수준에서 관리됩니다.

* \*\*읽기 액세스: \*\*모든 사용자는 연락처 정보, 팀 구성원 및 기타 관련 세부 정보와 같은 메타데이터를 포함하여 Teams에 대한 읽기 액세스 권한을 갖습니다.
* \*\*팀 수정: \*\* **Organization Product Admin** \[조직 제품 관리자] 역할이 있는 사용자는 팀을 수정할 수 있습니다. 이는 현재 **All Product Admin role** \[모든 제품 관리자 역할을] 갖고 있는 모든 사용자에게 할당된 새로운 역할입니다.
* \*\*팀 생성 및 삭제:\*\* **조직 관리자** 또는 **인증 관리자** 역할을 가진 사용자는 팀을 생성하고 삭제할 수 있습니다.

<Callout variant="tip" title="팁">
  조직 관리자 또는 인증 관리자 역할을 부여하지 않고도 추가 사용자가 팀을 만들고 삭제할 수 있도록 하려면 사용자 [지정 역할을](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#create-role) 만들 수 있습니다. 권한이 계정이 아닌 조직에 적용되는지 확인하세요.
</Callout>

## 팀 만들기 [#create-teams]

{/* You have three options for creating Teams in the UI: manually, from existing user groups or from your GitHub teams.  */}

<Callout variant="tip">
  프로그래밍 방식으로 팀을 만드는 것을 선호한다면 [NerdGraph API를](/docs/apis/nerdgraph/examples/nerdgraph-teams-tutorial) 사용하여 팀을 만들 수도 있습니다.
</Callout>

### 수동으로 팀 생성 [#create-manually]

아직 다른 곳에 팀 및 서비스 소유권 데이터가 없는 경우 이 옵션을 권장합니다.

1. **All Capabilities &gt; Teams** \[모든 기능 &gt; 팀] 으로 이동합니다.
2. **Build manually** \[수동으로 빌드를] 클릭합니다.
   <img title="Screenshot showing how to start the manual process" alt="Screenshot showing how to start the manual process" src="/images/teams_screenshot-crop_create-manually.webp" />
3. 팀 이름을 입력하세요.
4. *(선택 사항)* **설명** 과 **별칭을** 추가하세요.
5. *(선택 사항)* **팀 구성원을** 추가하세요.
6. **상위 또는 하위 팀 설정(계층 구조):** 조직 구조 내에 이 팀을 배치하려면 구성 메뉴에서 상위 팀 또는 하위 팀을 선택하십시오.
   * **참고:** 특정 **Team Hub** \[팀 허브] 내의 **Settings** \[설정] 탭을 통해 나중에 이러한 관계를 관리할 수도 있습니다.
7. **Create team** \[팀 만들기를] 클릭합니다.

### GitHub 팀에서 팀을 생성하세요 [#create-from-github]

뉴렐릭 팀을 기존 GitHub 팀과 동기화 상태로 유지하려면 이 옵션을 권장합니다.

1. **All Capabilities &gt; Teams** \[모든 기능 &gt; 팀] 으로 이동합니다.
2. **Use a Github integration** \[Github 통합 사용 을] 클릭하세요. GitHub 통합 설정 페이지로 리디렉션됩니다.

<img title="Screenshot showing the GitHub option" alt="Screenshot showing the GitHub option" src="/images/create_teams_github.webp" />

### ID 공급자를 통해 Teams를 자동으로 생성하세요.

팀 관리를 간소화하고 조직 구조에 맞추기 위해 Okta 또는 Active Directory와 같은 ID 공급자(IdP)에서 팀을 자동으로 생성하고 동기화할 수 있습니다.

**전제 조건**

* `authentication_domain.read.groups` 권한이 포함된 사용자 역할이 할당되었는지 확인하십시오.

IdP에서 자동 팀 생성을 설정하려면 다음 단계를 따르십시오.

1. **IdP에서 그룹을 설정하세요.**

   * 뉴렐릭에서 관리하려는 팀을 나타내는 그룹을 IdP에 생성하는 것으로 시작하세요. 예를 들어 team\_backend, team\_frontend 또는 team\_data와 같은 그룹을 만들 수 있습니다.

2. **SCIM을 통해 그룹을 동기화합니다.**

   * SCIM 프로토콜을 사용하여 IdP 그룹을 뉴렐릭과 동기화하세요. 이를 통해 뉴럴릭은 이러한 그룹을 동적으로 인식하고 관리할 수 있습니다.

3. **뉴렐릭에서 팀 생성을 구성하세요. 그룹 동기화가 완료되면 구성이 완료됩니다.**

   1. 뉴럴릭 UI 에서 **All Capabilities &gt; Teams** 으로 이동하세요.
   2. **Fetch groups from external sources** \[외부 소스에서 그룹 가져오기]를 클릭하세요.

   <img title="Screenshot showing the Idp option" alt="Screenshot showing the Idp option" src="/images/create_teams_Idp.webp" />

   3. 그룹을 팀으로 변환하려면 **Automate with a rule** \[규칙을 사용하여 자동화]를 선택하세요.
   4. 그룹 이름과 일치하는 규칙을 추가하세요. 예를 들어, &apos;team\_&apos;으로 시작하는 모든 그룹 이름에 적용되는 규칙을 만드세요.
   5. **규칙 만들기를** 클릭하세요. 뉴렐릭 UI 에서 팀을 확인할 수 있습니다.

   참고: 그룹 이름에 명확한 패턴이 없는 경우 규칙을 사용하는 대신 관련 그룹을 수동으로 선택할 수 있습니다.

   설정을 완료하면 뉴렐릭이 자동으로 규칙에 맞는 팀을 생성하고 관리하며, 변경 사항이 발생할 때마다 IdP와 동기화하므로 수동으로 업데이트할 필요가 없습니다.

## 팀 계층 구조 관리

팀 계층 구조를 사용하면 회사의 실제 조직 구조를 반영하는 부모-자식 구조로 팀을 구성할 수 있습니다.

### 인간관계를 관리하는 방법

현재 계층 구조는 수동으로 생성하거나 API를 통해 생성된 팀에 대해서만 지원되므로 이러한 관계는 다음 두 가지 방법으로 관리할 수 있습니다.

1. **팀 생성 시:** 수동으로 팀을 구성할 때는 **상위 팀** 및 **하위 팀** 드롭다운 메뉴를 사용하세요.
2. **기존 팀:** 아무 팀의 **팀 허브** 로 이동하여 **설정** 탭에서 **상위 팀** 또는 **하위 팀** 필드를 업데이트하세요.

<Callout variant="tip" title="팁">
  한 팀은 여러 개의 **하위 팀을** 가질 수 있지만, **상위 팀은** 하나만 가질 수 있습니다. 팀의 상위 팀을 변경하면 해당 팀은 조직 트리의 새 분기로 완전히 이동됩니다.
</Callout>

### 계층 수준 사용자 지정

기본적으로 계층 구조 레벨은 &quot;레벨 1&quot;, &quot;레벨 2&quot; 등으로 표시됩니다. 조직의 용어와 더 잘 일치시키려면 **계층 설정** 에서 이러한 레벨의 이름을 &quot;부서&quot;, &quot;팀&quot; 등으로 변경할 수 있습니다.

## 팀 소유권 자동화 [#assign-ownership]

뉴럴릭에서 팀에 권한을 할당하는 것은 명확한 책임 소재를 확립하고 효과적인 협업을 가능하게 하는 데 핵심적인 요소입니다. 이를 수행하는 가장 간단하고 효율적인 방법 중 하나는 레이블(태그)을 참조 설정 파일에 직접 적용하는 것입니다.

<Steps>
  <Step>
    ### 당신의 에이전트 설정 파일에 태그를 지정하세요 [#tag-entities]

    특정 팀을 소유한 팀을 지정하려면 에이전트 설정에 팀 레이블을 추가하는 것부터 시작하세요. 예를 들어 저항 에이전트에서는 다음을 추가할 수 있습니다.

    ```
    <labels>team:Backend</labels>
    ```

    이 태그는 분리를 “백앤드” 팀과 연결합니다.
  </Step>

  <Step>
    ### 팀에 엔티티 할당 자동화 [#entity-assignment]

    각 팀에 태그 할당을 자동화하려면:

    1. **All Capabilities &gt; Teams &gt; Automation** 로 이동하세요.
    2. **엔티티 소유권** 기능을 활성화하세요. 활성화되면 뉴렐릭은 팀 태그가 일치하는 모든 대상을 해당 팀에 자동으로 할당합니다.
  </Step>

  <Step>
    ### (선택 사항) 팀 별칭 사용 [#team-aliases]

    조직에서 동일한 팀에 대해 서로 다른 명칭(예: &quot;백앤드&quot;와 &quot;BE&quot;)을 사용하는 경우, 일관성을 유지하기 위해 별칭을 설정할 수 있습니다. 그렇게 하려면:

    1. 해당 팀의 **팀 허브** 페이지를 엽니다.
    2. **설정** 섹션으로 이동하세요.
    3. 대체 태그 값을 동일한 팀에 매핑하는 별칭을 추가합니다. 별칭을 설정하면 &quot;team:백앤드&quot; 또는 &quot;team:BE&quot;가 포함된 분리 태그가 모두 &quot;백앤드&quot; 팀에 할당됩니다.

    <Callout variant="tip">
      소유권을 나타내는 다른 입력된 태그도 사용할 수 있습니다. 이를 통해 유연성이 향상되고 조직의 현재 태그 지정 규칙과의 일관성이 보장됩니다.
    </Callout>
  </Step>
</Steps>

### 수동으로 소유권을 지정합니다 [#manual-ownership]

소유권을 나타내는 태그가 설정되어 있지 않은 경우, 다음 위치에서 수동으로 소유권을 지정할 수 있습니다.

#### 팀 허브 페이지에서 [#ownership-via-hub]

팀 허브에서 소유권을 할당하려면 다음 단계를 따르세요.

1. 팀 허브 페이지로 이동하여 **&apos;엔티티 관리&apos;를** 클릭합니다.

2. 할당할 엔티티를 선택하세요.

3. **저장** 을 클릭합니다.

#### 엔티티 페이지에서 [#entity-pages]

또한 모든 엔티티의 요약 페이지 헤더에서 직접 소유자를 지정할 수 있습니다.

<img title="Screenshot showing teams from entity page" alt="Screenshot showing teams from entity page" src="/images/teams_entity_summary_page.webp" />

#### 카탈로그에서 [#entity-headers]

팀에 아직 연결되지 않은 소유권이 없는 카탈로그 항목에 소유권을 할당할 수 있습니다.
