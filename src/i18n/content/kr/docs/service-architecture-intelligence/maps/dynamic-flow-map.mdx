<DNT>**Dynamic Flow Map**</DNT> 집계된 트레이스 데이터의 맵 뷰를 제공하는 시각화 도구입니다. 이는 초점 노드라고 불리는 특정 서비스와 관련된 성능 신호를 강조 표시합니다. 이 지도는 추적 데이터를 사용하여 트레이스에 참여하는 서비스와 초점 노드 간의 관계를 표시하고 상류와 하류 한국경제의 성과 변화를 상관관계로 나타냅니다. 이는 기능 및 서비스의 병목현상, 병목지점을 식별하고 근본 원인을 찾아내는 데 도움이 됩니다.

## 중요한 이유 [#why-it-matters]

<DNT>**Dynamic Flow Map**</DNT> 기능 이상 작용을 시각화하고 식별할 수 있도록 하여 해결 시간(MTTR)을 줄이는 데 도움이 됩니다. 상류 및 하류 추적 성능 데이터를 상호 연관시키면 다양한 서비스가 어떻게 상호 작용하고 서로의 성능에 영향을 미치는지 이해하는 데 도움이 됩니다.

지도 구성 요소와 기능에 대해 알아보려면 [<DNT>**Maps**</DNT>](/docs/service-architecture-intelligence/maps/advanced-maps) 참조하세요.

<Callout variant="important">
  <DNT>**Dynamic Flow Map**</DNT> 작동하려면 [<DNT>**Distributed tracing**</DNT>](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) 필요하며 모든 APM 에이전트에서 기본적으로 활성화되어 있습니다. 데이터가 표시되지 않으면 <DNT>**Distributed tracing**</DNT> 초점 노드와 관련 페이지에 대해 활성화되어 있는지 확인하고 추적 데이터가 뉴렐릭에 나타나는지 확인하세요.
</Callout>

## 동적 흐름 맵 탐색 [#explore-dynamic-flow-map]

<DNT>**Dynamic Flow Map**</DNT> 에 액세스하려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> 에서 모든 [서비스, 인프라 또는 팀 맵을](/docs/service-architecture-intelligence/maps/advanced-maps) 엽니다.
2. <DNT>**Dynamic Flow Map**</DNT> 열려면 노드를 가리키고 <DNT>**View dynamic flow**</DNT> 클릭하세요.

[<DNT>**Transactions 360**</DNT>](/docs/apm/transactions/workload-performance-monitoring/transaction-workloads/) 에서도 <DNT>**Dynamic Flow Map**</DNT> 에 액세스할 수 있습니다. 특정 거래에 관련된 서비스와 각 서비스에 걸리는 시간을 보여주는 맵을 볼 수 있습니다.

<DNT>**Dynamic Flow Map**</DNT> 은 초점 노드와 주변 인터페이스 사이의 관계를 보여줍니다. 상류 서비스는 모서리가 초점 노드를 향하여 가리키는 반면, 하류 서비스는 모서리가 초점 노드에서 멀어지는 방향으로 표시됩니다. 다음과 같이 지도와 상호 작용하여 성능 추세, 이상신호 및 기타 세부 정보를 볼 수 있습니다.

<CollapserGroup>
  <Collapser id="anomalies" title="이상신호 분석">
    이상 징후는 다음과 같이 서비스 노드 옆에 색상으로 구분된 모서리와 점으로 표시됩니다.

    * **블루에지**: 서비스 간 지연시간 문제.
    * **빨간색 가장자리**: 서비스 간 신호 오류.
    * **파란색과 빨간색 줄무늬 가장자리**: 지연시간 문제와 서비스 간 신호 오류입니다.
    * **서비스 노드 옆의 파란색 점**: 서비스에 지연시간 문제가 있습니다.
    * **서비스 노드 옆의 빨간색 점**: 서비스에 신호 오류가 있습니다.

    노드 옆에 이상소 점이 있지만, 이 노드와 초점 노드를 연결하는 모서리가 색상으로 구분되어 있지 않은 경우, 노드 간의 이상소가 상관관계가 없음을 나타냅니다. [성능 추세](#performance-trends) 에서 지표 시계열 차트를 검토하면 이상징후가 노드에서 서로 다른 시간에 발생했음을 확인할 수 있습니다.
  </Collapser>

  <Collapser id="performance-trends" title="성과 추세 보기">
    <DNT>**Dynamic Flow Map**</DNT> 에서 초점 노드와 연결된 노드 간의 성능 추세를 보려면 주변 노드를 클릭합니다. 노드 간의 성능 추세는 오른쪽 패널에 나타나며 성능 시계열 그래프를 보여줍니다.

    다음과 같이 성능 추세를 분석하고 노드를 관리할 수 있습니다.

    * 선택한 노드 간의 패턴을 식별합니다.
    * 시계열 차트 아래에 있는 노드를 선택하여 차트와 상호 작용합니다.
    * 선택한 노드 사이의 모든 트레이스를 봅니다.
    * 특정 팀에 노드 소유권을 할당합니다.
    * 노드에 태그를 추가합니다.
  </Collapser>

  <Collapser id="investigate-issues" title="문제를 조사하다">
    <DNT>**Dynamic Flow Map**</DNT> 에서 초점 노드와 상관관계가 있는 성능 문제가 있는 노드를 식별한 후 해당 노드 간의 트레이스를 조사할 수 있습니다. 트레이스를 보려면 오른쪽 패널에서 **See all traces** \[모든 트레이스 보기를] 클릭하세요. 트레이스를 분석하면 성능 문제의 근본 원인을 파악하고 이를 해결하기 위한 필요한 조치를 취할 수 있습니다.
  </Collapser>
</CollapserGroup>
