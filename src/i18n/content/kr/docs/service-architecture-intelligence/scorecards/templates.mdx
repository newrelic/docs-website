스코어카드 템플릿 라이브러리는 엔지니어링 및 비즈니스 사용 사례를 처리하는 간소화된 방법을 제공합니다. 라이브러리는 대기업의 모범 사례를 활용하여 일반적인 엔지니어링 과제에 대한 솔루션을 제공합니다.

라이브러리의 각 템플릿은 귀하의 조직의 표준에 맞게 완벽하게 사용자 정의할 수 있습니다. 쿼리, 규칙, 보안, 한계를 수정하여 스코어카드를 사용자의 요구에 맞게 조정할 수 있습니다.

## 스코어카드 템플릿

<CollapserGroup>
  <Collapser id="product-readiness" title="생산 준비">
    **목적:** 정의된 목표, 공지 설정, 최근 구현, 배포 활동을 확인하여 서비스가 프로덕션 구현, 배포 준비가 되었는지 확인합니다.

    **중요성:** 구현, 배포 중에 다운타임을 최소화하고 높은 서비스 품질을 보장합니다.

    **필수 조건:**

    * [뉴렐릭 변경 추적](/docs/change-tracking/change-tracking-introduction)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)

    **주요 규칙:**

    * **정의된 SLO:** 성능 및 신뢰성에 대한 명확한 뺄셈, 가치를 설정합니다.
    * **APM 서비스 알림 설정:** 조치 문제 감지에 대한 알림이 설정되어 있는지 확인합니다.
    * **최근 구현, 배포:** 지난 달 내에 성공적인 구현, 배포를 확인합니다.
  </Collapser>

  <Collapser id="ownership-documentation" title="소유권 및 문서화">
    **목적:** 팀 소유권, 태그, 코드 저장소 및 문서의 가시성을 평가합니다.

    **필수 조건:**

    * [뉴렐릭 Teams](/docs/service-architecture-intelligence/teams/teams)
    * [GitHub 통합](/docs/service-architecture-intelligence/github-integration) (해당되는 경우)

    **주요 규칙:**

    * **팀 소유권:** 소유권 태그를 통해 서비스를 책임 팀에 연결합니다.
    * **정의된 코드 리포지터리:** 연관된 코드를 확인합니다.
  </Collapser>

  <Collapser id="operational-readiness" title="작전 준비 상태">
    **목적:** 운영 절차에 대한 런북의 가용성을 검증하여 공지사항을 처리할 수 있도록 시스템이 준비되었는지 확인합니다.

    **중요성:** 투명성을 높이고 공지 이벤트 해결에 도움이 됩니다.

    **전제조건:** [뉴렐릭 알림](/docs/alerts/overview)

    **주요 규칙:**

    * **팀 소유권:** 소유권 태그를 통해 서비스를 책임 팀에 연결합니다.
    * **정의된 코드 리포지터리:** 연관된 코드를 확인합니다.
  </Collapser>

  <Collapser id="web-vitals" title="웹 바이탈">
    **목적:** 로딩 속도, 상호작용성, 시각적 안정성 등 사용자 경험에 영향을 미치는 주요 성능 지표를 평가합니다.

    **중요성:** 빠른 로드 시간, 반응성 있는 상호작용, 안정적인 레이아웃을 보장하여 사용자 경험을 최적화합니다.

    **전제조건:** [뉴렐릭 브라우저 모델링](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)

    **주요 규칙:**

    * **가장 큰 콘텐츠 페인트(LCP):** 주요 콘텐츠 로드 속도를 측정합니다.
    * **FID(입력 지연):** 사용자에 대한 반응성을 평가합니다.
    * **누적 레이아웃 이동(CLS):** 최소한의 레이아웃 이동을 보장합니다.
  </Collapser>

  <Collapser id="aws-reliability" title="AWS 안정성">
    **목적:** AWS 표준에 따라 안정성을 평가하여 cloud 운영의 보안과 중복성을 보장합니다.

    **중요성:** 클라우드 인프라를 보호하고 운영 연속성을 보장합니다.

    **전제 조건:** [뉴렐릭 AWS 모니터링](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations)

    **주요 규칙:**

    * **EFS 암호화 확인:** EFS(Elastic File System) 볼륨의 암호화를 확인합니다.
    * **RDS 인스턴스 백업 보존:** 관계형 데이터베이스 서비스(RDS)의 백업 보존 기간을 확인합니다.
    * **RDS 삭제 보호:** 삭제 보호를 확인합니다.
    * **RDS MultiAZ 존재:** 여러 가용성 영역(MultiAZ)에 걸쳐 구현 및 배포를 보장합니다.
    * **RDS 자동 하위 버전 업그레이드 확인:** 자동 하위 버전 업그레이드를 확인합니다.
  </Collapser>

  <Collapser id="dora-metrics" title="DevOps 연구 및 평가(DORA) 지표">
    **목적:** 핵심 성과 지표(KPI)를 활용하여 자원 배분 및 프로세스 개선 사항을 평가합니다.

    **중요성:** 소프트웨어 제공을 정량화하고, 전략적 의사 결정을 지원합니다.

    **필수 조건:**

    * [변경 추적](/docs/change-tracking/change-tracking-introduction)
    * [New Relic Alerts](/docs/alerts/overview)

    **주요 규칙:**

    * **구현, 배포 빈도:** 성공적인 구현, 배포 비율을 추적합니다.
    * **복구 평균 시간:** 장애로부터 복구하는 데 걸리는 시간을 측정합니다.
  </Collapser>
</CollapserGroup>
