<DNT>Repository catalog</DNT> 은 조직 내 저장소 관리를 중앙 집중화하고 간소화하도록 설계된 도구입니다. 모든 저장소에 대한 통합된 보기를 제공함으로써 팀 간의 장벽을 허물고 팀 간 협업을 강화합니다. 이를 통해 개발자, 사이트 개발자(SRE) 및 기타 이해관계자가 기존 마커를 효율적으로 식별하고, 목적을 이해하고, 소유권 또는 상태를 결정할 수 있습니다. 이러한 중앙 집중식 접근 방식은 코드 저장소의 검색, 접근 및 관리를 간소화하여 조직 전체의 협업, 책임감 및 생산성을 향상시킵니다.

## 주요 특징들 [#features]

* 중앙 집중식 저장소 검색: 신속하게 식별합니다. 저장소는 서비스, 프로젝트 또는 <DNT>[Teams](/docs/service-architecture-intelligence/teams/teams)</DNT> 과 관련되며 검색 또는 상담에 소요되는 시간을 최소화합니다.

* 서비스 및 개체 탐색: 포함 또는 서비스에서 대상 정의로 또는 그 반대로 원활하게 탐색하여 코드 의존성/종속성 및 감시를 이해하는 데 도움을 줍니다.

* 소유권 및 책임성: 저장소 소유자를 쉽게 식별하여 직접적인 쿼리를 지원하고 팀 내 책임성을 강화합니다.

* 로그 Insights: 공개 풀 요청 및 구현, 배포 이력 등의 적용 활동에 대한 세부 지표에 액세스하여 소홀히 된 프로젝트를 식별하고 리소스 할당을 최적화합니다.

* 관리 및 운영: 태그 또는 팀 할당과 같은 대량 작업을 수행하여 저장소를 효율적으로 관리하고 조직 표준을 유지합니다.

## 저장소 카탈로그에 액세스하세요 [#access-repository]

**필수 조건:**

* 뉴렐릭 계정에서 카탈로그 기능을 활성화하세요. 자세한 내용은 [카탈로그 활성화를](/docs/service-architecture-intelligence/catalogs/catalogs/#enable-catalogs) 참조하세요.

**저장소 카탈로그 인벤토리에 액세스하려면:**

1. 뉴렐릭 플랫폼에 로그인하세요.

2. **왼쪽 탐색 창 &gt; Catalogs &gt; Repository catalog** 로 이동합니다.

### 새로운 데이터 추가 [#add-new-data]

<DNT>Repository catalog</DNT> 에 추가 데이터를 추가하려면 오른쪽 상단의 <DNT>**+ Add GitHub repositories**</DNT> 버튼을 클릭하세요. 화면에 나타나는 지시에 따라 저장소에 데이터를 추가하세요. 교체 추가에 대한 자세한 내용은 [GitHub 통합을](/docs/service-architecture-intelligence/github-integration) 참조하세요.

### 한정 [#limitations]

* <DNT>Repository catalog</DNT> 페이지에서는 현재 데이터 정렬이 지원되지 않습니다.
