<Callout variant="important">
  Agent Control과 뉴렐릭 Control이 이제 Kubernetes 에서 **일반적으로 사용 가능해** 졌습니다! Linux 호스트 및 Windows 호스트에 대한 지원은 **공개 미리 보기** 프로그램에 포함되어 있습니다, [사전 출시 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy)에 따라.
</Callout>

## Kubernetes 마이그레이션

쿠네티스 클러스터가 이미 뉴렐릭으로 구성된 경우 에이전트 Control에서 관리하도록 에이전트를 전환하여 해당 관리 기능을 사용할 수 있습니다. 마이그레이션 프로세스에서는 기존 에이전트 설치를 제거하고 플릿 컨트롤을 통해 중앙에서 구성된 에이전트 제어 관리 설치로 교체합니다.

<Callout variant="important" title="원격 설정이 항상 승리합니다">
  에이전트 Control은 에이전트 설정에 대한 신뢰할 수 있는 소스로 플릭스를 사용합니다. 에이전트 설정(버전, 차트 값, 리소스 제한)은 로컬 설치 `values.yaml` 가 아닌 플릿위험 원격 설정에서 정의되어야 합니다. 두 설정이 모두 존재하는 경우 원격 설정이 우선 적용되어 로컬 값을 재정의합니다.
</Callout>

<Callout variant="tip" title="모범 사례">
  데이터 연속성을 보장하려면 에이전트 제어 설치 중에 동일한 클러스터 이름과 동일한 뉴렐릭 계정을 사용하세요. 지원이 가능할 때까지 클러스터의 Agent Control에서 아직 지원되지 않는 에이전트 또는 구성 요소를 유지할 수 있습니다.
</Callout>

### 1단계. 기존 설정을 불러오세요.

먼저, Agent Control로 관리할 수 있는 기존 에이전트를 식별하십시오.

에이전트 제어는 에이전트 제어 에이전트 유형에 매핑된 `nri-bundle` Helm 차트에서 다음 에이전트를 구현하고 배포하고 관리할 수 있습니다.

* `newrelic-infrastructure`, `nri-kube-events`, `kube-state-metrics` 및 `nri-kube-events`: 뉴렐릭 인프라 에이전트 유형으로 관리됩니다.
* `newrelic-prometheus-configurator`: 뉴렐릭 Prometheus 에이전트 형태로 관리됩니다.
* `newrelic-logging`Fluent Bit 에이전트 유형으로 관리됩니다.
* `nr-k8s-otel-collector`: 뉴렐릭 OpenTelemetry Collector 에이전트 형태로 관리됩니다.

Helm을 사용하여 에이전트를 설치한 경우 다음 명령을 실행하여 클러스터에서 설치 구성을 검색합니다.

```sh
# Find releases in a namespace (adjust as needed)
helm list --namespace <namespace-name>

# Get the values for a specific release
$ helm get values <release-name> --namespace <namespace-name>
```

설치 시 저장해둔 원래 설정 파일이 있다면 해당 파일을 바로 사용할 수 있습니다. Kubernetes 매니페스트(kubectl 또는 Kustomize)를 통해 클러스터를 해제한 경우 각 에이전트에 설정된 옵션을 기록해 두십시오. 이를 플릿 제어 원격 설정으로 다시 생성하게 됩니다.

### 2단계. 기존 에이전트를 제거합니다.

에이전트 컨트롤을 설치하기 전에 클러스터에서 기존 에이전트를 제거하세요.

Helm을 설치하려면 `helm uninstall` 명령어를 사용하십시오.

```shell
$ helm uninstall <release-name> -n <namespace>
```

예를 들어, `nri-bundle` 을 제거하려면 다음과 같이 하세요.

```shell
$ helm uninstall nri-bundle -n newrelic
```

### 3단계. 에이전트 컨트롤을 설치합니다.

[에이전트 제어 안내 설치를](/docs/new-relic-control/agent-control/setup/) 시작하고 생성된 `values.yaml` 을(를) 다운로드합니다. 이 파일에는 에이전트 제어 감독자 설정(인증서, 플릿 ID 및 관리할 에이전트 유형)만 포함되어 있습니다. **여기에 에이전트별 차트 값을 추가하지 마세요**. 플릿 컨트롤 원격 설정에 속합니다.

Helm 설치의 경우 일반적으로 다음을 실행합니다.

```shell
$ helm upgrade --install agent-control-bootstrap -n newrelic newrelic/agent-control-bootstrap --create-namespace --values my_values.yaml
```

### 4단계. 플릿 위험에서 에이전트 설정을 다시 만듭니다.

에이전트 Control이 설치되고 플릿 제어에 연결되면, 이전 설정으로 플릿의 원격 설정을 생성하거나 업데이트하여 에이전트를 배포합니다. 이곳은 모든 에이전트 설정에 대한 권위 있는 장소입니다.

#### 예시: nri-bundle 설치 마이그레이션

다음은 기존 `nri-bundle` Helm 값을 플릿 위험 원격 설정으로 변환하는 방법을 보여줍니다.

**기존 `nri-bundle` 설정:**

```yaml
global:
  cluster: test-migration
  licenseKey: ***
kube-state-metrics:
  enabled: false
newrelic-prometheus-agent:
  enabled: true
newrelic-infrastructure:
  enabled: true
  kubelet:
    tolerations:
      - operator: "Exists"
        effect: "NoSchedule"
      - operator: "Exists"
        effect: "NoExecute"
      - operator: "Exists"
        key: "MyToleration"
  ksm:
    enabled: false
  common:
    config:
      interval: 29s
newrelic-logging:
  enabled: true
  image:
    tag: "latest"
  resources:
    limits:
      cpu: 200m
    requests:
      cpu: 200m
nri-kube-events:
  enabled: true
  customAttributes:
    test_tag_label: test_tag_value
```

**동등한 플릿 컨트롤 원격 설정:**

플릿 위험에서 이를 귀하의 차량에 대한 원격 설정으로 설정하세요. Agent Control이 자동으로 적용해 드립니다.

```yaml
logs:
  chart_version: "1.25.1"
  chart_values:
    newrelic-logging:
      image:
        tag: "latest"
      resources:
        limits:
          cpu: 200m
        requests:
          cpu: 200m
infrastructure:
  chart_version: "5.0.109"
  chart_values:
    newrelic-infrastructure:
      kubelet:
        tolerations:
          - operator: "Exists"
            effect: "NoSchedule"
          - operator: "Exists"
            effect: "NoExecute"
          - operator: "Exists"
            key: "MyToleration"
      ksm:
        enabled: false
      common:
        config:
          interval: 29s
    nri-kube-events:
      customAttributes:
        test_tag_label: test_tag_value
prometheus:
  chart_version: "1.15.4"
```

## 리눅스 마이그레이션

<Callout variant="important">
  Agent Control은 Linux에서 기존 에이전트의 자동 마이그레이션을 지원하지 않습니다. Linux 호스트에 뉴럴릭 인프라가 이미 설치되어 있는 경우, Control을 설치하기 전에 수동으로 제거해야 합니다. Agent Control은 플릿 위험을 통해 OCI 호환 패키지를 통해 에이전트를 배포하며 기존 `newrelic-infra` 패키지 설치를 대체하거나 상속하지 않습니다.
</Callout>

Linux의 독립형 군인 설치에서 Agent Control로 마이그레이션하려면:

1. **기존 설정을 기록해 두세요.** 제거하기 전에 에이전트 설정 사본을 `/etc/newrelic-infra/newrelic-infra.yml` 에 저장하고 모든 온-호스트 통합 구성은 `/etc/newrelic-infra/integrations.d/` 에 저장하세요. Agent Control을 설치한 후 플릿 확률 원격 설정으로 다시 생성해야 합니다.

2. **기존 인프라 에이전트를 제거합니다.** 사용 중인 배포판에 맞는 제거 단계를 따르십시오.

   <Tabs>
     <TabsBar>
       <TabsBarItem id="linux-uninstall-debian">데비안, 우분투,</TabsBarItem> <TabsBarItem id="linux-uninstall-suse">SLES,</TabsBarItem> <TabsBarItem id="linux-uninstall-redhat">아마존 리눅스, 센토오스, RHEL</TabsBarItem>
     </TabsBar>

     <TabsPages>
       <TabsPageItem id="linux-uninstall-debian">
         ```bash
         sudo apt-get remove newrelic-infra -y
         ```
       </TabsPageItem>

       <TabsPageItem id="linux-uninstall-suse">
         ```bash
         sudo zypper -n remove newrelic-infra
         ```
       </TabsPageItem>

       <TabsPageItem id="linux-uninstall-redhat">
         ```bash
         sudo yum remove newrelic-infra -y
         ```
       </TabsPageItem>
     </TabsPages>
   </Tabs>

3. **에이전트 제어를 설치합니다.** 안내 설치를 참고하여 [Linux 설치 단계를](/docs/new-relic-control/agent-control/setup/) 따르고, **&quot;Host - Linux&quot;** 유형의 플릿을 선택하거나 생성했는지 확인하십시오.

4. **플로릿 컨트롤에서 에이전트 설정을 다시 만드세요.** 에이전트 Control이 설치되고 연결된 후, 플릿 컨트롤을 통해 에이전트 에이전트를 구현하고 배포하고 원격 설정을 통해 이전 설정을 적용합니다. 설정 형식에 대한 지침은 [Linux 설정을](/docs/new-relic-control/agent-control/configuration/#linux-configuration) 참조하세요.

사용자 정의 대시보드 및 알림 에이전트의 데이터 참조는 동일한 계정 및 호스트 이름을 사용하는 한 계속 작동합니다.

## Windows 마이그레이션

<Callout variant="important">
  Agent Control은 Windows 에서 기존 에이전트의 자동 마이그레이션을 지원하지 않습니다. Windows 호스트에 뉴럴릭 인프라가 이미 설치되어 있는 경우, 컨트롤을 설치하기 전에 수동으로 제거해야 합니다. APM 에이전트는 Windows의 에이전트 제어 시스템에서 관리되지 않으므로 설치된 상태로 유지할 수 있습니다.
</Callout>

Windows 의 독립형 군인 설치에서 에이전트 제어로 마이그레이션하려면:

1. **기존 인프라 에이전트를 제거합니다.** **Add or Remove Programs** \[프로그램 추가/제거] (또는 PowerShell 사용)를 열고 뉴렐릭 인프라 기구를 제거하십시오. 또는 제거 프로그램을 실행하세요.

   ```powershell
   # Example using the MSI uninstaller
   msiexec /x {PRODUCT_CODE} /quiet
   ```

2. **기존 설정을 기록해 두세요.** 제거하기 전에 에이전트 설정(일반적으로 `C:\Program Files\New Relic\newrelic-infra\newrelic-infra.yml`) 사본과 모든 온-호스트 통합 구성을 `C:\Program Files\New Relic\newrelic-infra\integrations.d\` 에 저장하세요. Agent Control을 설치한 후 플릿 확률 원격 설정으로 다시 생성해야 합니다.

3. **에이전트 제어를 설치합니다.** 안내 설치를 사용하여 [Windows 설치 단계를](/docs/new-relic-control/agent-control/setup/) 따르고 **&quot;Host - Windows&quot;** 유형의 플릿을 선택하거나 생성했는지 확인하십시오.

4. **플로릿 컨트롤에서 에이전트 설정을 다시 만드세요.** 에이전트 Control이 설치되고 연결된 후, 플릿 컨트롤을 통해 에이전트 에이전트를 구현하고 배포하고 원격 설정을 통해 이전 설정을 적용합니다. 설정 형식에 대한 지침은 [Windows 설정을](/docs/new-relic-control/agent-control/configuration/#windows-configuration) 참조하세요.

사용자 정의 대시보드 및 알림 에이전트의 데이터 참조는 동일한 계정 및 호스트 이름을 사용하는 한 계속 작동합니다.
