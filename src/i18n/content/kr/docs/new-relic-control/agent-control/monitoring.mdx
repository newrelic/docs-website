## 개요

에이전트 Control은 로컬 감독자로 설계되었으며 뉴렐릭 Control은 중앙 집중식 모니터링 경험을 제공합니다. 에이전트 제어는 관리되는 에이전트의 클러스터 감독자 역할을 합니다. 주요 역할은 에이전트가 올바르게 구현, 배포 및 실행되는지 확인하는 것입니다. 이 가이드에서는 관리형 에이전트를 모니터링하는 방법을 자세히 설명합니다. 함대의 상태를 전체적으로 파악하려면 뉴렐릭 제어 UI 사용할 수 있습니다. 심층적인 문제 해결, 문제 해결 및 유효성 검사를 위해 호스트나 클러스터에서 직접 검사를 수행할 수 있습니다.

## 뉴렐릭 Control의 중앙 집중식 모니터링

Agent Control 솔루션의 궁극적인 목표는 뉴렐릭 UI 내에서 포괄적이고 중앙 집중화된 모니터링 경험을 제공하는 것입니다. 이렇게 하면 상태 확인을 위해 수동으로 kubectl 명령어를 실행할 필요가 없어지고 전체 서버를 통합된 화면에서 확인할 수 있습니다. 이 경험은 크리에이 컨트롤에서 지원하는 모든 환경에서 이용할 수 있습니다.

중앙 집중식 보기는 에이전트의 전용 대시보드(뉴렐릭)에 직접 연결됩니다. 이를 통해 플릿 상태 모니터링에서 개별 에이전트가 수집한 특정 지표, 로그 및 트레이스를 조사하는 것으로 빠르게 전환할 수 있습니다. 이는 상위 관리부터 심층적인 옵저버빌리티까지 원활한 워크플로우를 제공합니다.

## 쿠버네티스 모니터링

이 지침은 Kubernetes 환경에서 실행되는 KeyControl에 적용됩니다.

### On-클러스터 상태 점검 및 문제 해결, 해결

이러한 검사는 표준 Kubernetes 도구와 명령을 사용하여 수행됩니다.

문제 진단, 문제 해결 시 kubectl 및 로컬 상태 엔드포인트를 사용하여 에이전트 제어 및 관리되는 에이전트의 상태를 직접 검사할 수 있습니다.

### 패드 상태 확인 및 로그인

표준 Kubernetes 명령을 사용하여 에이전트 제어 감독자와 해당 하위 에이전트 패드가 예상대로 실행되고 있는지 확인할 수 있습니다.

```shell
# Check Agent Control pods
kubectl get pods -n newrelic-agent-control

# Check sub-agent pods (e.g., in the 'newrelic' namespace)
kubectl get pods -n newrelic

# Inspect logs for the Agent Control supervisor
kubectl logs <agent-control-pod-name> -n newrelic-agent-control
```

### 로컬 상태 엔드포인트를 사용하세요

에이전트 제어는 감독자와 관리되는 모든 에이전트 모두의 상태 점검을 제공하는 로컬 상태 엔드포인트를 노출합니다.

1. 포트 전달: 터미널을 열고 다음 명령을 실행하여 로컬 포트를 agent-control 서버로 전달합니다.

```shell
kubectl port-forward <agent-control-pod-name> 51200:51200 -n newrelic-agent-control
```

2. 상태 요청: 새 터미널에서 curl을 사용하여 상태 엔드포인트를 쿼리합니다.

```shell
curl localhost:51200/status
```

출력은 구현, 배포 세부 정보, 에이전트 버전 및 감지된 오류를 포함하여 자세한 상태를 제공하는 JSON 응답입니다.
