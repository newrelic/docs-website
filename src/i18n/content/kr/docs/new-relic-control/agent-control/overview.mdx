Agent Control은 OpenTelemetry 에이전트의 다른 뉴렐릭 및 뉴렐릭 배포판을 관리하는 경량 에이전트 감독자입니다. 이는 플릿 컨트롤과 통신하여 Kubernetes 에서 대시보드 수준 측정, 리소스의 상태를 원격으로 구성, 업데이트 및 모니터링합니다.

## 전제 조건

쿠버네티스 클러스터에 배포하다 에이전트 제어를 시작하기 전에 다음 전제 조건을 충족하는지 확인하세요.

* **Helm 3:** Helm 버전 3이 컴퓨터에 설치되어 있어야 합니다. 설치 지침은 [Helm 설치를](https://helm.sh/docs/intro/install) 참조하세요.
* **뉴렐릭 클러스터 키:** 귀하의 뉴렐릭 계정에 텔넷리를 보고하려면 클러스터 키가 필요합니다.
* **쿠버네티스 클러스터 이름:** 쿠버네티스 클러스터의 이름을 준비하세요. 설치 과정에서 참조하게 될 겁니다.
* **인증 관리자:** 에이전트 제어를 설정하고 플릿 위험에 연결하려면 인증 관리자 역할이 있어야 합니다. 자세한 내용은 [사용자 관리 개념을](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings) 참조하세요.
* **기존 측정, 도구:** 쿠버네티스 클러스터가 이미 뉴렐릭 측정 도구로 구성된 경우 에이전트 Control을 설치하기 전에 기존 측정, 도구를 제거해야 합니다. 에이전트 제어를 설치한 후에는 플릿 컨트롤을 사용하여 클러스터에서 에이전트 에이전트 및 측정값을 설치하고 관리할 수 있습니다. 지침은 [에이전트 제어를 사용하여 기존 측정 관리, 리소스 관리](/docs/new-relic-control/agent-control/reinstrumentation) 를 참조하세요.

<Callout variant="caution" title="경고">
  동일한 클러스터에 여러 설치 에이전트 제어가 지원되지 않습니다.
</Callout>

## 호환성

Agent Control은 다음과 호환됩니다:

* 마지막 3개의 Kubernetes 마이너 릴리스
* Minikube, Kind, Amazon EKS, Azure AKS 및 Google GKE

뉴렐릭 Kubernetes 에이전트 에이전트 및 뉴렐릭 OpenTelemetry (NRDOT) 수집기와 관련된 시스템 요구 사항은 다음을 참조하세요.

* [Kubernetes 통합: 호환성 및 요구 사항](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements)
* [Kubernetes용 OpenTelemetry](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel)
