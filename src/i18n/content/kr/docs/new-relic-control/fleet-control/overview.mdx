  플릿 컨트롤은 다음과 같은 통합 관리를 제공합니다.

  * **쿠버네티스 클러스터** - 정식 출시(GA)
  * **Linux 호스트** - 공개 미리보기
  * **Windows 호스트** - 공개 미리보기 🆕 **신규**

  세 가지 어셈블리 유형 모두 동일하고 일관적인 플릿 컨트롤 경험을 공유합니다. 지원되는 모든 및 해당 환경의 전체 목록은 [유형 호환성 문서를](/docs/new-relic-control/agent-control/agent-types) 참조하십시오.

  공개 미리보기 기능은 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 제공됩니다.
</Callout>

<DNT>Fleet Control</DNT> 뉴렐릭 에이전트에 대한 중앙 집중식 원격 관리를 제공하여 규모에 맞게 전체 측정, 로그 수명 주기를 관리하는 데 도움을 줍니다. 쿠버네티스 기능 클러스터에서 에이전트를 관리하기 위해 일반적으로 사용 가능(GA)됩니다. Linux 및 Windows 호스트에서 에이전트를 관리하는 기능은 현재 공개 미리보기 단계에 있습니다.

<DNT>Fleet Control</DNT> 사용하면 다음을 수행할 수 있습니다.

* **에이전트 작업을 중앙 집중화하고 수동 작업을 줄입니다.** 모든 에이전트와 통합을 한 곳에서 설치, 모니터링, 구성 및 업데이트합니다.
* **대량 관리를 위한 플릿 사용자 정의:** 측정, 로그 요구 사항별로 에이전트를 그룹화하고 몇 번의 클릭만으로 전체 플릿에 대한 에이전트 버전을 쉽게 업그레이드할 수 있습니다. 스크립트가 필요 없습니다.
* **텔레메트리 공백 제거:** 환경 전반에 걸쳐 확장되는 하나의 경량 수퍼바이저인 에이전트 Control을 사용하여 쿠버네티스 클러스터에서 에이전트를 관리하세요.

<DNT>Fleet Control</DNT> 으로 이동하려면 <DNT>**All capabilities → New Relic Control → Fleets**</DNT> 로 이동하세요. 핀을 사용하여 왼쪽 탐색 모음에 뉴렐릭 컨트롤을 추가하여 쉽게 접근할 수 있습니다.

## <DNT>Fleet Control</DNT> 사용하는 이유는 무엇인가요?

대규모 환경에서 측정 및 계측을 관리하는 것은 상당한 어려움을 초래할 수 있습니다. <DNT>Fleet Control</DNT> 중앙 집중식 제어 평면을 제공하여 이러한 문제를 해결하도록 설계되었습니다.

* **오래된 에이전트 방지:** 에이전트를 최신 상태로 유지하여 보안 위험과 비호환성을 방지하고 최신 혁신을 활용할 수 있도록 하세요.
* **설정 드리프트 방지:** 모든 부분에서 일관된 설정을 보장하여 숨겨진 옵저버빌리티 격차와 규정 준수 문제를 제거합니다.
* **손상된 측정, 리소스 수정:** 측정, 리소스 관련 문제를 신속하게 식별하고 해결하여 옵저버빌리티 데이터에 대한 신뢰를 유지합니다.
* **수동 작업 감소:** 에이전트와 통합을 관리하기 위해 일회성 스크립트와 수동 업데이트에 의존하지 마세요.

## 함대란 무엇인가요?

<DNT>Fleet Control</DNT> 에서 플릿은 동일한 측정, 로그 설정을 공유하는 관리되는 부분(예: 쿠버네티스 클러스터 또는 호스트)의 동종 컬렉션입니다. Fleet은 관리형 회의의 논리적 그룹화 역할을 하여 효율적인 관리와 규모에 따른 일관된 옵저버빌리티 설정을 가능하게 합니다.

함대는 세 가지 주요 구성 요소로 구성됩니다.

* **활성 구현, 배포:** 함대에 적용된 최신 작업 세트의 요약입니다. 구현, 배포에는 에이전트의 설정을 설치, 삭제 또는 업데이트하는 것과 같은 하나 이상의 작업이 포함될 수 있으며 해당 변경 사항의 최종 결과를 반영합니다.
* **에이전트 및 설정:** 플릿의 관리되는 부분에서 로그아웃되는 에이전트 및 설정입니다.
* **관리되는 섹션:** 함대에 가입된 모든 할당된 관리되는 섹션입니다.

### 관리되는 진리 이해

<DNT>Fleet Control</DNT> 컨텍스트에서 관리되는 부분은 쿠버네티스 클러스터, Linux 호스트 또는 뉴렐릭 측정을 실행하는 Windows 호스트이며, <DNT>Agent Control</DNT> 에서 관리됩니다. 각 집합은 특정 관리 유형(쿠버네티스 클러스터, Linux 호스트 또는 Windows 호스트)과 연결되어 있으며 집합 내의 모든 부분은 해당 유형과 일치해야 합니다. 에이전트 설정은 함대 수준에서 지정되며 함대의 모든 관리되는 설정은 동일한 설정을 공유합니다.

* **관리형 영역:** 쿠버네티스 클러스터 또는 Agent Control이 설치된 호스트입니다. 이는 해당 개체가 <DNT>Fleet Control</DNT> 에 의해 관리될 수 있음을 의미하지만, 아직 함대에 연결되지 않았을 수도 있습니다.
* **할당된 관리 대상 엔티티:** 플릿에 합류한 관리 대상 엔티티입니다. 이 개체는 현재 <DNT>Fleet Control</DNT> 에 의해 적극적으로 관리되고 있으며, 해당 설정은 소속된 함대에 의해 결정됩니다.
* **할당되지 않은 관리 엔티티:** 현재 어떤 플릿에도 소속되지 않은 관리 엔티티입니다. 해당 장치에는 사용자 제어 기능이 설치되어 관리가 가능하지만, 현재는 자체 로컬 설정을 사용하여 작동 중입니다.

<Callout title="중요" variant="important">
  Agent Control을 설치하여 관리형으로 만들기 전에 플릿에 쿠버네티스 클러스터나 호스트를 추가할 수 없습니다.
</Callout>

## 주요 특징들

* **중앙 집중식 관리:** 쿠버네티스 클러스터 및 호스트 플릿에서 실행되는 에이전트를 관리하기 위한 통합 플랫폼을 제공하여 규모에 맞게 측정, 리소스 전체를 관리할 수 있습니다.
* **대시보드 가시성:** 플릿 상태, 에이전트 상태, 구현, 배포 진행률 및 설정 오류에 대한 명확한 보기를 통해 측정 상태, 로그를 관리합니다.
* **유연한 플릿 조직:** 다양한 측정, 인력 표준, 제작 환경 또는 팀별 요구 사항을 나타내기 위해 고유한 플릿을 생성합니다. 이 구조는 모든 설정을 관리하는 중앙 집중식 옵저버빌리티 팀과 개별 팀이 세분화된 권한을 사용하여 특정 함대를 소유하고 관리하는 분산형 모델을 모두 지원합니다.
* **원격 설정:** 환경의 다양한 측면을 모니터링하고 지표, 이벤트, 로그 및 트레이스(MELT)를 포함한 주요 데이터 포인트를 보고하는 뉴렐릭 에이전트 및 통합의 원격 설정을 지원합니다.

## 전제 조건

<DNT>Fleet Control</DNT> 설정하기 전에 시스템이 다음 필수 조건을 충족하는지 확인하십시오.

* **호스트용 공개 미리 보기 참여:** Linux 또는 Windows 호스트에서 에이전트를 관리하기 위해 <DNT>Fleet Control</DNT> 사용하려면 이 특정 기능에 대한 공개 미리 보기 프로그램에 참여해야 합니다. 쿠버네티스 클러스터 관리 에이전트는 일반적으로 사용 가능하며 사전 동의가 필요하지 않습니다.
* **에이전트 제어:** 관리하려는 쿠버네티스 클러스터 또는 호스트에 [에이전트 제어가](/docs/new-relic-control/agent-control/setup) 설치되어 있어야 합니다. 이는 모든 <DNT>Fleet Control</DNT> 기능에 필수적인 구성 요소입니다.

## 필수 권한

**조직 범위 권한:** <DNT>Fleet Control</DNT> 내에서 플릿을 생성하고 편집하려면 **Organization Manager** \[조직 관리자] 역할(제품에서 관리 설정으로 언급됨)이 필요합니다. 또는 차량 관리 권한이 있는 [사용자 지정 역할을 생성](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#create-role) 할 수도 있습니다.
