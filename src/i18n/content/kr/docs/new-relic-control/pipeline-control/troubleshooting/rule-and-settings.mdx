## 문제: 잘못된 NRQL 규칙

규칙 구현하다, 배포하다에 잘못된 NRQL 구문이 포함되어 있습니다.

## 해결책

* 실패한 규칙 등록 이벤트를 찾아 게이트웨이 패드의 로그인을 확인하세요.
* `DELETE` 대신 `SELECT` 을 사용하여 쿼리더 빌더에서 NRQL 검증한 다음 규칙을 편집하고 다시 구현하다, 배포하다.
* 자세한 내용은 NRQL 문서를 참조하세요.

## 문제: 잘못된 구성 맵 구현, 배포

오류 설정이 포함된 `ConfigMap` 이(가) 업데이트되었습니다.

## 해결책

* 새로운 구현, 배포 파드가 다시 시작되지 않는지 확인하세요.
* 이전 작업 설정으로 돌아갑니다.

## 문제: K8 구현, 배포 작업을 완료할 수 없음

최신 구성 변경 사항이 적용된 새로운 패드는 리소스 고갈로 인해 예약할 수 없습니다.

## 해결책

* 건강에 해로운 클러스터에 대한 게이트웨이 요약 페이지를 살펴보세요. Kubernetes UI 사용하여 파드 이벤트 및 보류 중인 작업을 확인하세요.
* 노드 풀 크기를 늘리거나 패드에 대한 리소스 제한(CPU 및 메모리) requests 확인하세요.
* 활성 노드 수에 대한 cloud 공급자 제한을 확인하세요.

## 문제: 데이터 스키마 불일치

이 규칙은 게이트웨이와 다른 스키마를 가진 NRDB의 데이터에 대해 작성됩니다.

## 해결책

* 스키마 호환성을 보장하기 위해 각 MELT 유형에 대한 설명서 지침을 따르세요.
