## 문제: 텔레메트리 프로듀서와 게이트웨이 간 DNS 확인 문제

DNS 확인 오류로 인해 에이전트가 게이트웨이에 연결할 수 없습니다.

## 해결책

* 에이전트 로그에 액세스(UI 또는 직접)하고 연결 오류를 검색합니다.
* 전시 및 네트워크 토폴로지에 따라 DNS 설정을 조정하려면 네트워크 관리자에게 문의하세요. 자세한 내용은 [에이전트 문서를](/docs/apm/new-relic-apm/getting-started/introduction-apm/) 참조하세요.

## 문제: 에이전트 게이트웨이 SSL 인증서 문제

에이전트와 게이트웨이 사이의 SSL 인증 설정에 문제가 있습니다.

## 해결책

* SSL 연결 오류가 있는지 에이전트 로그를 확인하세요.
* 인프라와 네트워크 토폴로지를 고려하여 SSL 인증서가 올바르게 구성되고 유효한지 확인하세요. 자세한 내용은 [에이전트 문서를](/docs/apm/new-relic-apm/getting-started/introduction-apm/) 참조하세요.

## 문제: 게이트웨이 구성 오류

데이터가 게이트웨이에 도달했지만 뉴렐릭에 게시하지 못했습니다.

## 해결책

* 발신 요청 및 오류 메트릭을 확인하세요.
* 실패한 규칙 메트릭을 검토합니다.
* 시작되지 않는 패드의 로그를 검사합니다.
* 게이트웨이 설정을 올바르게 하고 모든 패드가 작동하는지 확인하세요.

## 문제: 지원되지 않는 텔레메트리 프로듀서/프로토콜

지원되지 않는 API나 프로토콜에서 데이터가 전송되었습니다.

## 해결책

* 뉴렐릭과의 호환성을 확인하세요. 잘못된 API 클라이언트는 `501` 상태 코드를 반환합니다.
* 지원을 위해 기능 요청을 제출하세요.
* 데이터를 뉴렐릭으로 직접 보내도록 텔레메트리 생산자를 구성합니다.

## 문제 : 파괴적인 규칙

모든 데이터를 삭제하여 뉴렐릭에 도달하지 못하게 하는 규칙이 있습니다.

## 해결책

* 낙하 데이터 지표를 확인하세요.
* 데이터 흐름을 허용하기 위해 파괴적인 규칙을 수정하거나 제거합니다.

## 문제: 데이터 누락

섭취 후 뉴렐릭 백앤드에서 데이터가 누락되었습니다.

## 해결책

* 오류 지표를 검토하고 클라이언트 측 시간 초과를 확인하세요.
* 자원 고갈 징후를 평가합니다.
* [뉴렐릭 상태를](https://status.newrelic.com/) 확인하세요.
* 영향을 받은 시간 동안 게이트웨이 로그를 조사합니다.
