## 문제: 유효하지 않은 볼륨 키

게이트웨이가 잘못된 라이선스 키로 구성되었거나 라이선스 키가 교체되었습니다.

## 해결책

* 문제를 확인하려면 게이트웨이 로그에 직접 접근하세요. 내부 모니터링 파이프라인과 사용량 내보내기 도구에서 403 오류를 찾아보지만, 에이전트 데이터는 찾아보지 않습니다.
* 인스턴스 키가 유효하고 올바르게 구성되었는지 확인하세요. 필요한 경우 키를 업데이트하세요.

## 문제: 게이트웨이 모니터링 데이터에 영향을 미치는 클라우드 규칙

게이트웨이에서 생성된 priv은 cloud 규칙에 의해 의도치 않게 삭제됩니다.

## 해결책

* cloud 규칙의 사용 데이터를 검토하여 의도치 않은 삭제가 있는지 확인하세요.
* 게이트웨이 지표가 삭제되지 않도록 cloud 규칙 설정을 수정합니다.

## 문제: 메트릭 API에서 속도 제한이 발생합니다.

Indicator API 에 대한 요청 제한을 초과했을 수 있으며, 이로 인해 게이트웨이에서 후속 requests 429 응답 코드로 실패할 수 있습니다.

## 해결책

* 계정에서 OpenTelemetry 메트릭 API와 관련된 속도 제한 이벤트를 확인하세요.
* 요청 속도 관리 및 조정에 대한 지침은 속도 제한 문서를 참조하세요.
