파이프라인 컨트롤은 뉴렐릭 Control의 데이터 관리 구성 요소입니다. 이는 가치가 낮은 데이터를 뉴렐릭 플랫폼에 수집하기 전에 필터링하고 삭제하는 데 도움이 될 수 있습니다.

파이프라인 지배는 파이프라인 지배 게이트웨이(간단히 게이트웨이라고 함)와 파이프라인 지배 cloud 규칙이라는 두 가지 하위 구성 요소로 구성됩니다.

게이트웨이는 네트워크를 떠나기 전에 데이터나 속성을 필터링하도록 설계되었습니다. cloud 규칙은 뉴렐릭 cloud 내에서 해당 처리를 수행합니다. 두 시나리오 모두에서 파이프라인 제어는 동일한 NRQL 쿼리 구문을 사용하여 데이터를 처리합니다.

게이트웨이를 사용하려면 먼저 사용자 환경에 게이트웨이를 설정해야 합니다. cloud 규칙에는 설정이 필요하지 않습니다.

파이프라인 취소 비용을 이해하려면 [파이프라인 취소 비용을](/docs/new-relic-control/pipeline-control/costs) 참조하세요.

## 전제 조건

시작하기 전에 다음이 필요합니다.

* 조직 제품 관리자 및 조직 관리자 권한이 있는 뉴렐릭 계정

<Callout variant="important" title="중요한">
  조직 제품 관리자 및 조직 관리자 사용자 역할에는 다음이 필요합니다.

  * 게이트웨이 설치
  * 게이트웨이에 클러스터 추가
  * 게이트웨이 업데이트(예: 새로운 규칙 및 설정 변경)
  * 구현하다, 배포하다 게이트웨이 설정
  * 게이트웨이 규칙 보기
</Callout>

* Kubernetes 운영 및 관리에 대한 지식
* 쿠버네티스 클러스터 관리자는 클러스터 내에서 구현하고, 배포하고, 리소스를 관리할 수 있습니다.
* 네트워크 내에서 개인 DNS 호스트명을 생성하고 관리할 수 있는 권한
* DNS 레코드를 생성하고 관리할 수 있는 권한
* Helm이 로컬에 설치됨
* 파이프라인 규칙을 생성하고 관리하기 위한 NRQL 이해

<Callout variant="important">
  * 통풍 조절 규칙은 규칙을 만든 순간부터 도착하는 데이터에만 적용되며, [이미 수집된](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-data-retention#data-deletion) 데이터는 삭제하지 않습니다.
  * 규칙을 만들 때는 해당 규칙이 귀하가 정한 조건을 충족하는 데이터를 정확하게 식별하고 삭제하는지 확인해야 합니다. 또한 귀하는 규칙을 모니터링하고, 귀하가 뉴렐릭에 공개하는 데이터에 대해서도 책임을 져야 합니다.
</Callout>
