파이프라인 컨트롤은 뉴렐릭 Control의 데이터 관리 구성 요소입니다. 이 서비스는 텔레메트리 데이터를 필터링, 변환 및 관리하여 비용을 절감하고 신호 품질을 개선하며 규정을 준수하도록 도와줍니다.

파이프라인은 독립적으로 또는 함께 사용할 수 있는 두 가지 처리 계층을 제공합니다.

## 파이프라인 제어 게이트웨이

**게이트웨이 규칙은** 인프라에서 실행됩니다.

* 데이터가 **네트워크를 떠나기 전에** 처리합니다(송신 비용 절감).
* 조직 전체 설정(조직의 모든 계정에 규칙이 적용됨)
* 완전한 변환, 필터링 및 샘플링 기능
* 쿠버네티스 클러스터 구현, 배포 필요)를 관리합니다.
* 게이트웨이에서 수신된 데이터 용량을 기준으로 요금이 부과됩니다.

게이트웨이는 다음과 같은 상황에서 가장 유용합니다:

* 데이터가 인프라를 떠나기 전에 필터링하여 네트워크 송출 비용을 줄입니다.
* 모든 MELT 유형에 대한 더욱 풍부한 데이터 변환 기능(예: 로그를 분석하고, 속성을 정규화하고, 메타데이터를 추가합니다.
* 규정 준수 요건을 충족하기 위해 민감한 데이터가 네트워크를 벗어나기 전에 삭제하거나 난독화합니다.
* 더 많은 양의 데이터를 처리하기 위해 게이트웨이가 cloud 보다 저렴한 규칙이 적용됩니다.

게이트웨이에 대한 자세한 내용은 [게이트웨이 개요를](/docs/new-relic-control/pipeline-control/gateway/overview) 참조하십시오.

## 파이프라인 통제 cloud 규칙

데이터가 뉴렐릭 엔드포인트에 도달한 후 뉴렐릭의 렌더링에서 **클라우드 규칙이** 실행됩니다.

* **뉴렐릭에 도달한 후** 데이터 처리(전송 관리가 필요하지 않음)
* 계정별 규칙 (각 규칙은 하나의 뉴렐릭 계정에만 적용됩니다)
* NRQL 기반 데이터 및 속성 필터링을 위한 삭제 규칙
* 인프라 설정이나 관리가 필요 없습니다.
* 데이터 전송량(aCCU당 7.5GB)과 스캔된 수십억 개의 이벤트(aCCU당 2.237 BSE)를 기준으로 요금이 부과됩니다.

클라우드 규칙은 다음과 같은 경우에 가장 효과적입니다.

* 게이트웨이 인프라 관리 없이 간단한 드롭 규칙을 원합니다.
* 귀사의 처리 요구 사항은 크지 않으므로 본격적인 게이트웨이 구현이 필요하지 않습니다.
* 전체 수집량 대비 필터링 비율이 낮을수록 cloud 규칙 기반 가격 책정이 더 비용 효율적일 수 있습니다.

자세히 알아보기: [클라우드 규칙 생성](/docs/new-relic-control/pipeline-control/cloud-rules/create-pipeline-rules)

## 두 가지를 함께 사용하면

게이트웨이 규칙과 cloud 규칙을 동시에 사용할 수도 있습니다.

* 델메트리 소스(예: 에이전트는 게이트웨이를 통해 라우팅할지 여부를 다르게 구성할 수 있습니다.
* 심층 방어 전략은 민감한 데이터가 절대적으로 삭제되도록 보장합니다.
* 게이트웨이를 기본 처리에 사용하면서 cloud API/UI 통해 일부 규칙을 편리하게 관리할 수 있습니다.

## 비교표

| 기능              | 게이트웨이 규칙                    | 클라우드 규칙                      |
| --------------- | --------------------------- | ---------------------------- |
| **처리 위치**       | 귀하의 장치(Kubernetes)          | 뉴렐릭 cloud                    |
| **처리될 때**       | 데이터가 네트워크를 떠나기 전에           | 데이터가 뉴렐릭에 도달한 후              |
| **설정 필요**       | 예(쿠버네티스 클러스터, Helm 차트)      | 아니요                          |
| **설정 방법**       | YAML 또는 UI                  | UI 또는 NerdGraph API(NRQL 기반) |
| **규칙 범위**       | 조직 전체                       | 계정별                          |
| **기능**          | 변환, 데이터/속성 삭제, 샘플 평가        | 데이터/속성 삭제                    |
| **지원되는 데이터 유형** | 지표, 이벤트, 로그, 트레이스           | 지표, 이벤트, 로그, 트레이스            |
| **가장 좋은**       | 대용량 처리, 변환, 송출 비용 절감, 규정 준수 | 간단한 드롭 규칙, 낮은 볼륨, 인프라 관리 불필요 |

자세한 가격 정보 및 비용 계산은 [파이프라인 위험 비용을](/docs/new-relic-control/pipeline-control/costs) 참조하세요.

## 전제 조건

### cloud 규칙의 경우

* 파이프라인 위험 cloud 규칙을 관리하기 위한 RBAC 권한이 있는 역할을 맡은 뉴렐릭 계정
* NRQL에 대한 이해를 바탕으로 필터링 규칙을 생성합니다.

### 게이트웨이용

* 조직 제품 관리자 또는 조직 관리자 권한이 있는 뉴렐릭 계정

<Callout variant="important" title="중요한">
  조직 제품 관리자 또는 조직 관리자 사용자 역할에는 다음 사항이 필요합니다.

  * 게이트웨이 설치
  * 게이트웨이에 클러스터 추가
  * 구현하다, 배포하다 게이트웨이 설정
  * 게이트웨이 규칙 보기
</Callout>

* Kubernetes 운영 및 관리에 대한 지식
* 쿠버네티스 클러스터 관리자는 클러스터 내에서 구현하고, 배포하고, 리소스를 관리할 수 있습니다.
* 네트워크 내에서 개인 DNS 호스트명을 생성하고 관리할 수 있는 권한
* DNS 레코드를 생성하고 관리할 수 있는 권한
* Helm이 로컬에 설치됨

<Callout variant="important">
  * 통풍 조절 규칙은 규칙을 만든 순간부터 도착하는 데이터에만 적용되며, [이미 수집된](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-data-retention#data-deletion) 데이터는 삭제하지 않습니다.
  * 규칙을 만들 때는 해당 규칙이 귀하가 정한 조건을 충족하는 데이터를 정확하게 식별하고 삭제하는지 확인해야 합니다. 또한 귀하는 규칙을 모니터링하고, 귀하가 뉴렐릭에 공개하는 데이터에 대해서도 책임을 져야 합니다.
</Callout>
