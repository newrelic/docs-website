[NRQL(New Relic 쿼리 언어)](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/) 을 사용하여 쉽게 경고 조건을 생성합니다.

조건은 알림을 받고자 하는 내용을 설명하며 쿼리와 임계값의 두 가지 속성만 필요합니다. 쿼리는 사용자에게 중요한 데이터를 정의하고 알림을 받을 대상을 결정합니다. 임계값은 알림을 받을 시기를 결정합니다.

NRQL 쿼리 및 임계값을 사용하여 첫 번째 경고 조건을 작성하려면 다음 단계를 따르십시오. 완료되면 작동 중인 경고 상태가 됩니다.

시작할 준비가 되셨습니까? [New Relic 계정](https://newrelic.com/signup) 이 있는지 확인하십시오. 영원히 무료입니다!

## 1단계: 쿼리 작성 [#step1-nrql]

NRQL 쿼리를 사용하여 환경이 수행되는 방식에 대한 데이터를 반환할 수 있습니다. 그런 다음 해당 쿼리에서 조건을 만들 수 있습니다.

이 예에서는 각 호스트의 CPU 사용률(쿼리)에 대한 조건이 생성되어 CPU 사용률이 0.7%(임계값)를 초과할 때 알림을 트리거합니다. 일반적으로 1% 미만의 임계값은 사용하지 않지만 이 예에서는 임계값을 설정하는 방법을 보여줍니다.

UI 상단에서[NRQL 쿼리 빌더](/docs/query-your-data/explore-query-data/query-builder/use-advanced-nrql-mode-query-data/) 를 찾습니다.

</figcaption>

조건은 각 호스트의 CPU 사용률을 보여줍니다. 이 NRQL 쿼리 또는 모니터링 대상과 관련된 유사한 쿼리를 사용하여 조건을 생성합니다.

</figcaption>

유효한 NRQL 쿼리를 생성했으면 실행합니다. 그런 다음 **경고 만들기** 를 클릭합니다.

다른 유용한 쿼리를 생성하기 위한 [NRQL 구문](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/#syntax) 에 대해 자세히 알아보세요.

## 3단계: 조건 임계값 설정 [#step3-nrql]

중요 및 경고 임계값을 설정하여 환경 성능에 대한 알림을 받는 시기를 결정합니다.

* 경고 조건에 대한 임계 임계값이 필요합니다.
* [경고 임계값](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/set-thresholds-alert-condition/#threshold-levels) 은 선택 사항입니다. 이러한 임계값은 인시던트를 생성하거나 임계 임계값처럼 사용자에게 알리지 않습니다. 임계 임계값이 인시던트를 열고 사용자에게 알리는 경우 나중에 생성된 경고 임계값 위반이 보고서에 포함됩니다.

**조건 임계값 설정** 섹션의 이 예에서 중요 임계값(빨간색)은 최소 5분 동안 CPU 사용률이 0.7%를 초과하는 호스트를 확인합니다. 호스트의 CPU 사용률이 최소 5분 동안 0.67%를 초과하는 경우를 표시하는 경고 임계값(노란색)도 추가됩니다.
