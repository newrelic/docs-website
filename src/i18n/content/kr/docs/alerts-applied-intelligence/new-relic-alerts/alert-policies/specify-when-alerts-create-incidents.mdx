이슈는 하나 이상의 인시던트의 모음입니다.

문제는 비즈니스를 방해하는 사고에 대해 알림을 받는 시점을 결정합니다. 이러한 사고는 주어진 경고 조건(예: CPU 초과, 종합 모니터링 충돌 또는 느린 웹 앱 응답 시간)이 충족될 때 발생합니다.

문제를 생성하는 정책 조건을 제어합니다. 이렇게 하면 중요한 진행 중이거나 반복되는 문제에 대해 알림을 받고 불필요한 알림을 받지 않습니다.

정책, 조건 또는 조건과 신호 조합에 따라 문제 기본 설정을 지정하는 방법을 알아보세요.

## 중대 및 경고 위반 [#incident-types]

정책 조건이 충족되면 두 가지 임계값 유형 중에서 선택합니다.

* 크리티컬(빨간색)
* 경고(노란색)

정책 조건이 위험 임계값을 위반하는 경우 효율적으로 대응하는 데 도움이 되는 세부 정보와 함께 문제 레코드를 생성할 수 있습니다. 문제 레코드에는 개시 임계 임계값 인시던트 이후에 발생한 모든 경고 인시던트도 포함됩니다.

문제가 열렸을 때, 문제가 승인된 경우, 문제가 종료될 때 적절하게 구성된 워크플로의 모든 대상뿐만 아니라 정책의 모든 알림 채널로 알림이 전송됩니다.

## 인시던트 기본 설정 선택 [#preference-procedures]

기본적으로 각 정책에 대해 단일 문제 레코드가 생성됩니다.

선택한 정책에 대한 문제 기본 설정을 변경하려면:

1. **[one.newrelic.com](https://one.newrelic.com/ "링크가 새 창에서 열립니다.")** 상단 탐색에서 **경고 및 AI** 를 클릭한 다음 **경고 조건(정책)** 을 클릭한 다음 ( 정책 **선택** ) 을 클릭합니다.
2. **이슈 생성 기본 설정** 을 클릭하고 사용 가능한 [옵션](#preference-options) 을 선택한 다음 저장합니다.

필요에 따라 각 정책에 대해 이 단계를 반복합니다.

## 정책당 하나의 문제(기본값) [#preference-policy]

<img
  title="A diagram showing how the By policy issue preference works"
  alt="A diagram showing how the By policy issue preference works"
  src={byPolicy}
/>

<figcaption>
  문제 기본 설정으로 기본값( **정책당 하나의 문제** )을 선택하면 모든 조건 인시던트를 동일한 문제로 그룹화합니다.
</figcaption>

이슈가 열린 후 추가 조건이 인시던트를 열면 인시던트가 동일한 이슈에 추가됩니다. 선택한 정책에 대한 문제 및 활동 페이지에는 하나의 문제만 나타납니다.

정책 및 해당 조건에 초점이 맞춰진 범위(예: 하나의 엔터티 모니터링)가 있는 경우 **정책당 하나의 문제** 기본 설정을 선택합니다. [문제 기본 설정 모범 사례](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#incident-practices) 에 대해 자세히 알아보세요.

## 조건당 하나의 문제 [#preference-condition]

<img
  title="A diagram showing how the By condition issue preference works"
  alt="A diagram showing how the By condition issue preference works"
  src={byCondition}
/>

<figcaption>
  문제 기본 설정으로 **조건당 하나의 문제** 를 선택하면 특정 조건에 대한 대상별로 모든 문제를 별도의 문제로 그룹화합니다.
</figcaption>

개별 문제 레코드가 특정 조건에 초점을 맞추도록 하려면 **조건당 하나의 문제** 옵션을 선택합니다. 정책에 여러 조건이 있는 경우 각 조건에 대한 별도의 문제 레코드가 문제 및 활동 페이지에 나타납니다.

* 대상이 경보 정책의 조건을 위반하는 경우 인시던트가 열리고 문제 레코드가 생성됩니다.
* 동일한 대상이 경고 정책에 대한 다른 조건을 위반하는 경우 다른 인시던트가 열리고 다른 문제 레코드가 생성됩니다.
* 추가 대상이 문제가 열린 후 동일한 조건에서 사건을 열면 이러한 사건은 해당 조건에 대해 이미 열려 있는 문제 레코드에 추가됩니다.

[문제 기본 설정 모범 사례](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#incident-practices) 에 대해 자세히 알아보세요.

## 사건당 하나의 문제 [#preference-target]

<img
  title="Alert policy issue preferences"
  alt="Alert policy issue preferences"
  src={bySignal}
/>

<figcaption>
  문제 기본 설정으로 **문제당 하나의 문제** 를 선택하면 문제가 전혀 그룹화되지 않습니다. 정책에서 발생하는 모든 사건에 대해 새로운 문제가 생성됩니다.
</figcaption>

**문제당 하나의 문제** 옵션은 문제 레코드를 생성하기 위한 가장 세부적인 수준입니다. 인프라 전반에서 발생하는 모든 것을 면밀히 모니터링하려는 경우 이 문제 기본 설정을 선택합니다.
