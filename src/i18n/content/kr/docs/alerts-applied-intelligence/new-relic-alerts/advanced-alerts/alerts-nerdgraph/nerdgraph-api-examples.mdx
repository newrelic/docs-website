GraphQL 형식의 [NerdGraph API](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) 를 사용하여 프로그래밍 방식으로 정책, 조건 및 뮤팅 규칙을 관리할 수 있습니다. 이것은 New Relic 내에서 또는 이전 [REST API](/docs/alerts/rest-api-alerts/new-relic-alerts-rest-api/rest-api-calls-new-relic-alerts) 를 사용하여 관리하는 강력한 대안입니다.

## NerdGraph로 관리할 수 있는 경고 기능 [#what-you-can-do]

경고 관련 NerdGraph 문서에 대한 링크는 다음과 같습니다.

* [정책 관리](/docs/alerts/alerts-nerdgraph/nerdgraph-examples/nerdgraph-api-alerts-policies)
* [NRQL 조건 사용](/docs/alerts/alerts-nerdgraph/nerdgraph-examples/nerdgraph-api-alerts-nrql-conditions)
* [알림을 억제하는 음소거 규칙 추가](/docs/alerts/new-relic-alerts/managing-notification-channels/muting-rules-suppress-notifications)
* [알림 채널 관리](/docs/alerts-applied-intelligence/new-relic-alerts/alerts-nerdgraph/nerdgraph-api-notification-channels)
* [신호 손실 및 간격 채우기 사용자 지정](/docs/alerts-applied-intelligence/new-relic-alerts/alerts-nerdgraph/nerdgraph-api-loss-signal-gap-filling)

경고 쿼리 및 변형을 발견하는 가장 쉬운 방법은 NerdGraph API 탐색기를 사용하는 것입니다.
