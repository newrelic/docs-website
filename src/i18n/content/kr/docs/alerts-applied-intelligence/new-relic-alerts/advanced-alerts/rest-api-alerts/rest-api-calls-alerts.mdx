우리의 REST API는 New Relic 경고 설정을 프로그래밍 방식으로 구성하기 위한 New Relic의 원래 API입니다( [우리가 선호하는 API인 NerdGraph에 대해 알아보십시오](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/alerts-nerdgraph/nerdgraph-api-examples) ).

[REST API Explorer](/docs/apm/apis/api-explorer-v2/getting-started-new-relics-api-explorer) 에는 사용 가능한 각 API 호출에 대한 컬 요청 형식, 사용 가능한 매개변수, 잠재적 응답 상태 코드 및 JSON 응답 구조도 포함됩니다. [UI에서 경고 조건을 만들](/docs/alerts/new-relic-alerts/defining-conditions/define-alert-conditions) 수도 있습니다.

## REST API를 사용하기 전에 [#before-start]

REST API를 사용하여 경고 데이터를 관리하기 전에 [경고 및 적용된 인텔리전스를 위한 API 소개 를](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/alerts-api/intro-alerts-api) 읽으십시오.

인프라 알림 [은 인프라 모니터링 알림을 위한 REST API를](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/rest-api-calls-new-relic-infrastructure-alerts) 참조하세요.

또한 아직 하지 않았다면 아래에서 무료 New Relic 계정을 만들어 오늘 데이터 모니터링을 시작하십시오.

## REST API에서 제외된 조건 [#excluded]

이러한 유형의 조건에는 API에서 사용 가능한 엔드포인트가 없습니다.

* APM: 웹 트랜잭션 백분위수, 레이블 및 이상을 대상으로 하는 조건
* NRQL: 이상

        }
      }'

## APM, 브라우저, 모바일에 대한 조건 [#conditions]

이러한 API 기능에는 정책 [조건](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions) 을 생성, 업데이트, 삭제 또는 나열할 수 있는 API 탐색기에 대한 링크가 포함됩니다. 이러한 호출은 다음 유형의 엔터티가 있는 조건에 대한 것입니다.

* APM: 앱 및 주요 거래
* 브라우저: 앱 및 주요 거래
* 모바일: 모바일 앱

**Exception:** 다음 <InlinePopover type="apm"/>이 API 엔드포인트에서 **not available** 조건:

* 조건 타겟팅 라벨(동적 타겟팅)
* 웹 트랜잭션 백분위수 조건
* 이상 조건

<Callout variant="important">
  API Explorer는 [외부 서비스](#ext-services-conditions) (APM 및 모바일) 및 [종합 모니터링](#) 을 포함하여 **별도의 엔드포인트** 를 사용하여 다른 유형의 조건에 대한 정보를 제공합니다. 검색 또는 업데이트 시 모든 유형의 경고 조건을 고려하십시오. 계정이 EU 데이터 센터의 데이터를 호스팅하는 경우 [EU 지역 계정에 적절한 API 엔드포인트를](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#endpoints) 사용하고 있는지 확인하십시오.

    * [사용자 키](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)

    * 사건 ID

      **[API Explorer](https://rpm.newrelic.com/api/explore/alerts_incidents/close) > Alerts Incidents > PUT > Close**

      ```
      curl -X PUT 'https://api.newrelic.com/v2/alerts_incidents/{id}/close.json' \
           -H "X-Api-Key:<a href='/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key'>$API_KEY</a>" -i \
           -H 'Content-Type: application/json'
      ```

## 경보 엔티티 조건

이러한 API 기능에는 경고 정책에 대한 조건에서 엔터티를 나열, 추가 및 제거할 수 있는 API 탐색기에 대한 링크가 포함됩니다.

<Callout variant="important">
  조직이 EU 데이터 센터에서 데이터를 호스팅하는 경우 [EU 지역 계정에 적절한 API 끝점을](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#endpoints)사용하고 있는지 확인하십시오.
</Callout>

<CollapserGroup>
  <Collapser
    id="list-condition-by-entity"
    title="항목별 조건 나열"
  >
    APM, 브라우저, 모바일 또는 키 트랜잭션의 [엔터티](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) 에 적용되는 [조건](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition) 을 보려면 API 호출에 다음 값을 포함합니다.

    * [사용자 키](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key) 또는 [REST API 키](/docs/apis/rest-api-v2/requirements/rest-api-key)

    * `entity_id` 모니터링할 특정 [엔터티(경고 대상)](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) 입니다.

    * `entity_type` , 다음 중 하나여야 합니다.

      * 애플리케이션
      * 브라우저응용 프로그램
      * 모바일 애플리케이션
      * 키 트랜잭션

      **[API Explorer](https://rpm.newrelic.com/api/explore/alerts_entity_conditions/list) > Alerts Entity Conditions > GET > list**

      ```
      curl -X GET 'https://api.newrelic.com/v2/alerts_entity_conditions/$ENTITY_ID.json' \
           -H "X-Api-Key:<a href='/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key'>$API_KEY</a>" -i \
           -G -d 'entity_type=$ENTITY_TYPE'
      ```
  </Collapser>

    id="add-entity"

    * [사용자 키](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)

    * `entity_id` 모니터링할 [항목(경고 대상)](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) 입니다. 의 숫자 ID입니다. <InlinePopover type="apm"/>애플리케이션, 브라우저 앱, 키 트랜잭션 또는 모바일 앱.

    * `condition_id` ( **[API Explorer](https://rpm.newrelic.com/api/explore/alerts_conditions/list) > Alerts Conditions > GET > List** 에서 사용 가능)

    * `entity_type` , 다음 중 하나여야 합니다.

      * 애플리케이션
      * 브라우저응용 프로그램
      * 모바일 애플리케이션
      * 키 트랜잭션

      **[API Explorer](https://rpm.newrelic.com/api/explore/alerts_entity_conditions/add) > Alerts Entity Conditions > PUT > Add**

      ```
      curl -X PUT 'https://api.newrelic.com/v2/alerts_entity_conditions/$ENTITY_ID.json' \
           -H "X-Api-Key:<a href='/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key'>$API_KEY</a>" -i \
           -H 'Content-Type: application/json' \
           -G -d 'entity_type=$ENTITY_TYPE&condition_id=$CONDITION_ID'
      ```
  </Collapser>
