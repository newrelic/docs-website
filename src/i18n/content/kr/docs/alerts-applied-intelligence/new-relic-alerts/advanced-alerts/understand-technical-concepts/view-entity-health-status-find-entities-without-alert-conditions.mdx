경고를 사용하면 엔터티(알림 대상)에 연결된 하나 이상의 조건이 있는지 쉽게 알 수 있습니다.

* **예인** 경우 선택한 인덱스(APM, 브라우저 등)의 상태 표시기가 현재 상태로 색상으로 구분됩니다. 현재 경고 위반의 요약을 보려면 해당 상태 표시기에 마우스를 가져갑니다.
* **아니오** 인 경우 선택한 인덱스의 상태 표시기가 회색으로 나타납니다.

<Callout variant="important">
  조건과 정책이 함께 작동하는 방식에 대한 자세한 내용은 [중요 개념 소개 를](/docs/alerts/new-relic-alerts/getting-started/alert-policy-workflow#terminology) 참조하십시오.
</Callout>

## 예외

건강 상태 표시기는 다음에 적용되지 않습니다.

* 인프라 엔터티
* [대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-new-relic-one-dashboards)
* 레이블의 대상이 되는 엔티티

## 색상으로 구분된 건강 상태 [#colors]

## 예: 조건이 없는 앱 [#example-v3-no-policies]

다음은 어떤 조건과도 연결 **되지 않은** APM 인덱스에 나열된 앱의 예입니다. 색상으로 구분된 상태는 밝은 회색으로 해당 엔터티에 대해 설정된 경고 조건이 없음을 나타냅니다.

<img
  title="032715crop-events-no-v3.png"
  alt="032715crop-events-no-v3.png"
  src={img4032715CropEventsNoV3Png}
/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com/)** 으로 이동한 다음 **탐색기** 를 클릭합니다. 이 예는 현재 경고 조건과 연결되지 않은 앱을 보여줍니다. 회색 상태 아이콘은 조건이 없음을 의미합니다.
</figcaption>

기존 조건에 추가하거나 새 조건을 생성하려면 [표준 절차](/docs/alerts/new-relic-alerts/getting-started/alert-policy-workflow#workflow) 를 따르십시오.

## 예: 조건이 있는 앱 [#example-v3-yes-policies]

다음은 하나 이상의 조건과 연결된 APM 인덱스 **에** 나열된 앱의 예입니다. 데이터를 수집 중이기 때문에 색상으로 구분된 상태는 녹색이며 현재 경고(노란색) 또는 위험(빨간색) 위반이 없습니다.

<img
  title="032715crop-events-green.png"

<figcaption>

</figcaption>

<Callout variant="tip">
  이 엔터티뿐만 아니라 모든 제품에서 현재 진행 중인 [인시던트](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/explore-incident-history) 를 나열하는 인덱스를 보려면 **모든 위반 보기** 를 선택합니다.
</Callout>
