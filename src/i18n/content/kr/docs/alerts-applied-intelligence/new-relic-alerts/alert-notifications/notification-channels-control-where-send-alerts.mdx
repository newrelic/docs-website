[알림](/docs/alerts) 을 사용하여 알림 채널을 설정하고 해당 채널을 [정책](/docs/alerts/new-relic-alerts/getting-started/alert-policy-workflow#terminology) 에 연결할 수 있습니다. 선택한 채널은 적절한 담당자가 사고에 대해 알림을 받을 수 있는 빠르고 일관된 방법을 제공합니다. 예를 들어 알림을 통해 사건에 대한 차트를 포함하여 컨텍스트를 제공하고 팀과 공유할 수 있습니다.

Alerts는 웹훅, Slack 룸, 이메일 등을 포함한 여러 알림 채널을 제공합니다. 인시던트가 열리거나 확인되거나 종료되면 알림 채널을 통해 알림을 받게 됩니다. 이 문서에서는 사용 가능한 알림 채널과 설정 방법에 대해 설명합니다.

<Callout variant="tip">
  이 문서는 경고 알림에 관한 것입니다. 마케팅 이메일, 주간 보고서 및 공지 사항을 포함하여 다른 New Relic 이메일 구독 취소에 대한 일반적인 정보는 New Relic 이메일 [구독 취소 를](/docs/accounts/accounts/account-maintenance/unsubscribe-new-relic-emails) 참조하십시오.
</Callout>

## 알림 채널 보기 [#view-channels]

계정의 모든 알림 채널을 보려면: **[one.newrelic.com](https://one.newrelic.com/ "링크가 새 창에서 열립니다.")** 으로 이동하여 상단 탐색 메뉴에서 **Alerts & AI** 를 클릭한 다음 **Channels** 를 클릭하십시오.

## 알림 채널 추가 또는 제거 [#add-channel]

새 알림 채널을 설정하려면:

1. **알림 채널에서** **새 알림 채널** 을 클릭합니다.
2. [채널 유형을](#channel-types) 선택하고 필요한 기타 단계를 완료하십시오.

알림 정책 또는 채널을 추가하거나 제거하려면:

* 특정 [알림 채널](#view-channels) 을 선택하고 **경고 정책** 을 선택한 다음 정책을 추가하거나 제거합니다.

  또는

* 특정 [정책](/docs/alerts/new-relic-alerts/configuring-alert-policies/create-or-rename-alert-policy#alert-policy-view) 을 선택하고 **알림 채널** 을 선택한 다음 채널을 추가하거나 제거합니다.

다음은 사용 가능한 알림 채널 유형입니다.

    **요구 사항** : 이 기능은 New Relic One 사용자 모델의 사용자는 사용할 수 없으며 원래 사용자 모델의 사용자만 사용할 수 있습니다( [사용자 모델 차이점](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 에 대해 알아보기). 해결 방법으로 [이메일 알림 채널](#email) 을 사용할 수 있습니다.

    **사용자** 알림 채널을 사용하여 기존 계정 팀 구성원 및 관리자를 선택합니다. **사용자** 목록을 보거나 [경고 정책에 사용자를 추가](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/view-or-update-user-email-channels) 하려면: **[one.newrelic.com](https://one.newrelic.com/ "링크가 새 창에서 열립니다.")** 으로 이동하여 상단 탐색 메뉴에서 **경고 및 AI** 를 클릭한 다음 **채널** 을 클릭합니다.

    사용자가 [New Relic 모바일 앱을 구성한](/docs/mobile-apps/new-relic-mobile-apps/authentication-alerts/alerting-new-relic-mobile-apps/) 경우 사용자 알림 채널은 또한 사용자의 등록된 모바일 장치 중 하나로 [푸시 알림](#mobile-push) 을 보냅니다.

    계정에 하나 이상의 [하위 계정](/docs/accounts/install-new-relic/account-setup/manage-apps-or-users-sub-accounts) 이 있는 경우 알림 채널에는 [현재 선택된 상위 또는 하위 계정](/docs/accounts/install-new-relic/account-setup/use-multiple-accounts) 의 사용자만 포함됩니다.
  </Collapser>

## 모바일 푸시 알림 수신 [#mobile-push]

<Callout variant="important">
  모바일 푸시 알림을 받으려면 장치가 등록되어 **(계정) > User preferences** 에 나열되어 있어야 합니다. 장치가 **사용자 기본 설정** 에 나열되지 않으면 앱에서 로그아웃했다가 다시 로그인한 다음 목록에 있는지 다시 확인하세요.
</Callout>

모바일 푸시 알림을 수신하려면:

1. 모바일 앱을 통해 New Relic 계정에 한 번 이상 로그인하여 장치가 등록되었는지 확인하십시오.
2. 경고 정책 [에 사용자 채널을 추가합니다](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/view-or-update-user-email-channels#account-users) .
3. 장치에 대한 푸시 알림을 **켜** 십시오.

## 경고 알림 확인 [#ack-notification]

계정의 모든 사용자는 사용자 인터페이스 또는 이메일 알림을 통해 [알림](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/acknowledge-alert-incidents) 을 확인할 수 있습니다.

New Relic에서 사건을 승인하면 [PagerDuty](#pagerduty) 에서 관련된 모든 사건도 승인합니다.
