  경고 **조건** 위반으로 **인해 사고** 가 발생하는 방식, 사고로 **인해 문제** 가 발생하는 방식 및 **워크플로** 의 논리가 알림을 생성하는 방식을 보여주는 다이어그램입니다.  

</figcaption>

위반을 구성하고 **인시던트** 를 트리거하는 것을 정의하는 경고 **조건** 을 구성하여 원격 분석 데이터에 대한 경고를 설정합니다. 예를 들어 호스트의 CPU가 80%를 초과하는 경우 인시던트를 여는 경고 조건을 만들 수 있습니다. 인시던트가 발생하면 실제로 알림이 발생하는 **문제** 가 발생합니다.

조건은 경고 **정책** 내에 있습니다. 정책에는 해당 정책의 모든 조건을 제어하는 상위 수준 설정이 포함되어 있습니다. 한 가지 설정은 여러 인시던트를 단일 문제로 그룹화할 수 있는 방법을 제어하는 **문제 생성 기본** 설정입니다. 예를 들어, 모든 인시던트에 고유한 문제가 있다고 결정하거나, 조건별로 그룹화하거나, 정책별로 그룹화할 수 있습니다. 문제를 더 큰 문제로 그룹화하는 논리적 작업인 기본 제공 **의사결정** 이 있습니다(사용자 정의 결정을 작성할 수도 있음).

문제는 차례로 문제가 알림으로 전환되는 방식을 제어할 수 있는 **워크플로** 를 트리거할 수 있습니다. **목적지** 는 알림을 받는 서비스입니다. Slack, PagerDuty, ServiceNow와 같은 일반적인 서비스와 간단한 웹 후크도 지원합니다. 사용자가 중복 대상을 생성하지 않도록 대상 설정을 주의 깊게 관리하는 것이 좋습니다.
