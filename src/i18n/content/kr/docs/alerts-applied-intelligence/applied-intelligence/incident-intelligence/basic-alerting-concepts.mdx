위반을 구성하고 **인시던트** 를 트리거하는 것을 정의하는 경고 **조건** 을 구성하여 원격 분석 데이터에 대한 경고를 설정합니다. 예를 들어 호스트의 CPU가 80%를 초과하는 경우 인시던트를 여는 경고 조건을 생성할 수 있습니다. 인시던트가 발생하면 실제로 알림이 발생하는 **문제** 가 발생합니다. 문제는 증상의 근본적인 문제를 설명하는 **사건** 그룹입니다. 새로운 인시던트가 생성되면 이슈가 생성됩니다. 인시던트는 시간 경과에 따른 시스템의 "증상"을 설명하는 **이벤트** 그룹입니다. 이러한 증상은 New Relic 경고 조건 또는 외부 모니터링 도구에 의해 감지됩니다. 이벤트는 New Relic 경고 조건 또는 외부 모니터링 시스템에 의해 정의된 상태 변경 또는 트리거를 나타냅니다. 이벤트에는 영향을 받는 엔터티에 대한 정보가 포함되어 있습니다.

조건은 경고 **정책** 내에 있습니다. 정책에는 해당 정책의 모든 조건을 제어하는 상위 수준 설정이 포함되어 있습니다. 한 가지 설정은 여러 인시던트를 단일 문제로 그룹화할 수 있는 방법을 제어하는 **문제 생성 기본** 설정입니다. 예를 들어 모든 인시던트에 고유한 문제가 있다고 결정하거나 조건별로 그룹화하거나 정책별로 그룹화할 수 있습니다. 문제를 더 큰 문제로 그룹화하는 논리적 작업인 기본 제공 **결정** 이 있습니다(사용자 정의 결정을 만들 수도 있음). 결정을 활성화한 경우 새로 생성된 문제와 상관 관계가 있는지 확인하기 위해 다른 문제가 평가됩니다. 그렇다면 상관된 문제를 병합하여 노이즈를 줄이고 근본 원인 분석을 용이하게 합니다.

문제는 차례로 문제가 알림으로 전환되는 방식을 제어할 수 있는 **워크플로** 를 트리거할 수 있습니다. **목적지** 는 알림을 받는 서비스입니다. Slack, PagerDuty, ServiceNow와 같은 일반적인 서비스와 간단한 웹 후크도 지원합니다. 사용자가 중복 대상을 생성하지 않도록 대상 설정을 주의 깊게 관리하는 것이 좋습니다.
