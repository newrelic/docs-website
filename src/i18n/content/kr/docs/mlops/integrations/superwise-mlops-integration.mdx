Superwise에는 사용자가 Superwise가 모니터링하는 AI 상태를 New Relic의 중앙 집중식 운영 보기에 추가할 수 있도록 New Relic과 [모델 성능 모니터링](/docs/alerts-applied-intelligence/mlops/get-started/intro-mlops/) 통합이 포함되어 있습니다.

## New Relic을 사용하여 Superwise에서 데이터 마이그레이션 [#integrate-superwise]

Superwise의 KPI 및 인시던트 통합은 드리프트, 활동, 인시던트 및 구성한 사용자 지정 KPI를 포함하여 즉시 사용 가능한 모델 KPI의 결과를 New Relic으로 직접 간소화합니다. 모든 사용 사례, 논리, 세분화, 임계값 및 민감도에 맞게 조정할 수 있는 오작동하는 모델에 대한 즉각적인 개요를 얻을 수 있습니다.

Superwise에서 New Relic으로 데이터를 수신하려면 다음을 수행해야 합니다.

1. **New Relic 계정에 로그인합니다.** **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > + Add more data** 로 이동합니다.

2. **Superwise 클릭: 검색 창에** **Superwise** 를 입력합니다.

3. **계정 ID 선택:** Superwise가 통합할 계정 ID를 선택합니다.

4. **두 개의 액세스 토큰을 만듭니다.**

   * 계정 ID를 선택하면 New Relic 통합 화면이 나타납니다. **모델 메트릭 데이터**아래에서 **키 만들기를**클릭합니다. 이것은

   * 또한 선택적으로 **Incident Insights** 에서 인증 토큰을 복사할 수 있습니다. 향후 단계를 위해 New Relic 페이지를 열어 두십시오.

5. **Superwise 포털에 로그인: 이제 Superwise 포털에 로그인** 하고 **알림 채널 설정** 으로 이동합니다. **New Relic** 을 선택하고 채널 이름을 만듭니다.

6. **Superwise에서 키와 토큰을 복사하여 붙여넣기:** New Relic 통합 대시보드로 돌아가 **복사** 아이콘을 클릭하여 생성한 키와 선택적 토큰을 복사합니다. Superwise 포털의 **New Relic** 채널에서 **Model Metrics Data** 아래에 **API 키** 를 붙여넣고 **Incidents** 아래에 **인증 토큰** 을 붙여넣습니다. **저장** 을 클릭하여 완료합니다.

7. **토큰 확인:** Superwise 채널 포털에서 **테스트** 버튼을 클릭하여 두 토큰이 모두 제대로 작동하는지 확인합니다. 확인을 나타내기 위해 **녹색** 또는 **빨간색** 상태 상자가 나타나야 합니다.

## New Relic 대시보드에서 Superwise 모델 보기 및 탐색

사용자가 Superwise에서 New Relic 통합을 구성하면 표준 모델 메트릭이 New Relic으로 전송되고 사용자는 New Relic 내에서 모델 [관찰 가능성 대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 를 받습니다. 또한 사용자는 특정 모델 메트릭 및 인시던트 정책을 구성하고 비즈니스 컨텍스트에 맞게 조정된 모델 관찰 가능성을 위해 이를 New Relic으로 보낼 수 있습니다.

1. **통합 대시보드로 이동:** 토큰을 테스트하고 통합이 올바르게 설정되었는지 확인한 후 New Relic 통합 대시보드로 돌아가서 **데이터 보기** 를 클릭합니다. 그러면 Superwise에서 제공하는 자동 생성 대시보드로 리디렉션됩니다.
2. **Superwise의 대시보드를 분석합니다.** New Relic의 Superwise 대시보드에는 3개의 차트가 있습니다.

* **모델 활동** 차트는 활성 모델 수, 활동 및 필터링된 기간 동안의 총 예측 수를 표시합니다.
* **모델 입력 드리프트** 감지 차트는 드리프트 중인 모델을 표시하며 0에서 100 사이의 스케일링된 모델 드리프트 레벨러를 사용하여 재교육이 필요할 수 있습니다.
* **인시던트 인사이트** 차트는 미해결 인시던트가 있는 모델 수, 인시던트가 배포되는 방식 및 모델 인시던트 자체에 대한 자세한 세부 정보를 표시합니다.

3. **Superwise에서 메트릭 및 인시던트 사용자 지정:** Teams는 Superwise 플랫폼에서 모든 모델 메트릭 및 인시던트 유형을 쉽게 사용자 지정할 수 있습니다. 그런 다음 **Metrics** 아래의 New Relic **Browser** 탭에서 이러한 메트릭을 검색할 수 있습니다. 이렇게 하려면 [Superwise 문서 4단계의 지침을](https://docs.superwise.ai/docs/new-relic#4-customize-metrics-and-incidents-you-would-like-to-send) 따르세요.
4. **경고 알림 설정:** 대시보드를 만들고 나면 데이터에 대한 경고를 받을 수 있습니다. 차트에서 NRQL 경고 조건을 생성하려면 차트 위젯을 클릭한 **다음 경고 조건 생성** 을 클릭합니다. 조건의 이름을 지정하고 사용자 정의한 후에는 기존 정책에 추가하거나 새 정책을 생성할 수 있습니다.
5. **알림 받기:** 알림 조건을 만든 후에 알림 방법을 선택할 수 있습니다. [알림 채널을 설정하는 방법](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/) 에 대한 문서를 참조하세요.
6. **인시던트 상관 관계:** 알림 외에도 인시던트 인텔리전스를 사용하여 인시던트의 상관 관계를 지정할 수 있습니다. [의사결정을 사용하여 사건을 연관시키는 방법](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/change-applied-intelligence-correlation-logic-decisions/) 에 대한 문서를 참조하십시오.

이제 Superwise와 New Relic을 성공적으로 통합했습니다. 새로 생성된 경고는 이제 New Relic 경고와 상관 관계가 있으며 새로 보고된 예측에 대한 데이터를 볼 수 있습니다.
