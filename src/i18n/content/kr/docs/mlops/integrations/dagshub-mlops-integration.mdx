DagsHub는 데이터 과학자와 기계 학습 엔지니어가 데이터, 모델, 실험 및 코드의 버전을 지정하고 동기화할 수 있는 플랫폼입니다. 이를 통해 귀하와 귀하의 팀은 작업을 쉽게 공유, 검토 및 재사용하여 기계 학습을 위한 GitHub와 같은 경험을 제공할 수 있습니다. DagsHub는 인기 있는 오픈 소스 도구 및 형식을 기반으로 하므로 New Relic과 같이 이미 사용 중인 도구와 쉽게 통합할 수 있습니다.

New Relic에는 실시간으로 기계 학습 교육 지표를 모니터링하고 분석하기 위해 DagsHub와 통합된 [모델 성능 모니터링](/docs/alerts-applied-intelligence/mlops/get-started/intro-mlops/) 이 포함되어 있습니다.

## DagsHub를 New Relic과 통합 [#integrate]

DagsHub와 New Relic을 통합하면 기계 학습 훈련 지표를 실시간으로 분석하고 모니터링할 수 있습니다. [New Relic 대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 에서 메트릭을 시각화하고, 사용자 지정 메트릭을 만들고, 훈련 실행 중에 발생하는 이벤트와 사건을 알리도록 [경고](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/) 를 설정할 수 있습니다.

DagsHub에서 New Relic으로 데이터를 수신하려면 계정 번호와 다음과 같이 얻을 수 있는 특수 키가 필요합니다.

1. **New Relic 계정에 로그인합니다**. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > +Add more data로** 으로 이동합니다.
2. **DagsHub 클릭: 검색 창에** DagsHub를 입력합니다.
3. DagsHub와 통합할 **계정 ID를 선택합니다** .
4. **액세스 토큰 만들기** : 계정 ID를 선택한 후 **실시간 교육 측정항목** 아래 **에서 API 키 만들기** 를 클릭합니다. 이것은 새로운 원격 분석 API 키가 됩니다. 향후 단계를 위해 New Relic 페이지를 열어 두십시오.
5. **DagsHub 포털에 로그인합니다. 로그인** 한 후 작업 중인 리포지토리를 선택합니다. **설정** 을 클릭하고 왼쪽 메뉴에서 **통합** 버튼을 선택합니다. **New Relic** 타일을 클릭합니다. 실시간 실험 모니터링으로 프로젝트를 생성하는 방법을 알아 [보려면 DagsHub MLflow 설명서 페이지](https://dagshub.com/docs/integration_guide/mlflow_tracking/) 를 읽어보십시오.
6. **DagsHub에 토큰을 복사하여 붙여넣습니다** . New Relic 통합 대시보드로 돌아가서 삽입 키 옆에 있는 **복사** 아이콘을 클릭하여 생성한 토큰을 복사합니다. DagsHub 포털에서 **New Relic Insight 삽입** 키 아래에 삽입 키를 붙여넣고 **다음** 을 클릭하여 완료합니다. 토큰이 확인되고 확인 화면이 나타납니다.

## New Relic 대시보드에서 DagsHub 모델 보기 및 탐색 [#view-explore]

DagsHub에서 New Relic 통합을 구성하면 실시간으로 DagsHub에 기록된 모든 교육 지표가 New Relic으로 전송됩니다.

1. **DagsHub 통합 대시보드** 로 이동: 토큰을 테스트하고 통합이 올바르게 설정되었는지 확인했으면 New Relic 통합 대시보드로 돌아가서 **데이터 보기** 를 클릭합니다. DagsHub에서 제공하는 자동 생성된 New Relic 대시보드로 리디렉션됩니다.
2. **DagsHub 대시보드 분석** : DagsHub 대시보드에는 각 리포지토리별로 그룹화된 모델 손실을 표시하는 **리포지토리별 손실 메트릭** 하나의 차트가 포함되어 있습니다. 훈련한 모든 모델에서 작동합니다.

3. **사용자 지정 지표 표시**: DagsHub에서 제공되는 교육 지표를 쉽게 사용자 지정할 수 있습니다. NRQL 사용 및 데이터 추적을 위한 쿼리 생성에 대한 자세한 내용은 [NRQL 참조 를](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/)참조하십시오.
4. **경고 알림 설정** : 대시보드 내에서 차트를 생성하면 [데이터에 대한 경고를 받을](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/) 수 있습니다. 차트에서 NRQL 경고 조건을 생성하려면 **...** 차트 메뉴를 클릭한 **다음 경고 조건 생성** 을 클릭합니다. 조건의 이름을 지정하고 사용자 정의한 후에는 기존 정책에 추가하거나 새 정책을 생성할 수 있습니다.
5. **알림 받기** : 알림 조건을 만든 후에 알림 방법을 선택할 수 있습니다. [알림 채널을 설정하는 방법](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/) 에 대한 문서를 참조하세요.

이제 New Relic을 DagsHub와 성공적으로 통합했습니다. 새로 생성된 경고는 New Relic 경고와 상관 관계가 있으며 새로 보고된 예측에 대한 데이터를 볼 수 있습니다.
