Aporia는 데이터 과학 팀이 프로덕션 환경에서 기계 학습 모델을 모니터링할 수 있는 빠르고 쉽고 안전한 방법입니다. Aporia를 사용하면 팀은 데이터 드리프트, 예상치 못한 편향, 데이터 무결성 문제 및 성능 저하와 같은 문제를 조기에 감지하기 위해 실시간 경고를 수신하기 위해 몇 분 만에 맞춤형 모니터를 구축할 수 있습니다. Aporia에는 추가 조사 및 근본 원인 분석을 가능하게 하는 조사 도구 상자도 있습니다. [아포리아에 대해 자세히 알아보세요](https://www.aporia.com) .

Aporia에는 MLOps의 전체 모델 관리를 제공하기 위한 뉴렐릭과의 통합이 포함되어 있으며, 뉴렐릭 내의 사용자 정의된 <InlinePopover type="dashboards"/>사용하여 레이블 조사를 표시합니다.

## New Relic과 Aporia 통합 [#integrate-aporia]

Aporia를 사용하면 Aporia의 모니터에서 생성된 <InlinePopover type="alerts"/>뉴렐릭의 인시던트 인텔리전스 엔진 및 예측 데이터에 연결하여 모델에 대한 뉴렐릭의 포괄적인 모니터링 대시보드를 생성할 수 있습니다.

1. <DoNotTranslate>**Login into Aporia's console**</DoNotTranslate>

   : 왼쪽 탐색 모음에서

   <DoNotTranslate>**Integrations**</DoNotTranslate>

   클릭하고

   <DoNotTranslate>**New Relic**</DoNotTranslate>

   선택합니다.

2. <DoNotTranslate>**Login into your New Relic account:**</DoNotTranslate>

   <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > +Add more data**</DoNotTranslate>

   으)로 이동합니다.

3. <DoNotTranslate>**Click on Aporia:**</DoNotTranslate>

   검색창에

   <DoNotTranslate>**Aporia**</DoNotTranslate>

   를 입력합니다.

4. <DoNotTranslate>**Select your account ID:**</DoNotTranslate>

   Aporia와 통합할 계정 ID를 선택하세요.

5. <DoNotTranslate>**Get a New Relic API key:**</DoNotTranslate>

   Aporia 아이콘을 클릭한 후 1단계를 따르세요.

   <DoNotTranslate>**Prediction data & model quality alerts**</DoNotTranslate>

   아래에서

   <DoNotTranslate>**Select**</DoNotTranslate>

   클릭하여 기존 키를 선택하거나

   <DoNotTranslate>**Create API key**</DoNotTranslate>

   클릭합니다. 이로 인해 뉴렐릭에 데이터를 보고하는 데 사용되는 뉴렐릭

   <InlinePopover type="licenseKey"/>

   이 생성되었습니다.

6. <DoNotTranslate>**Paste the key in Aporia:**</DoNotTranslate>

   이제 Aporia의 대시보드로 이동하여 뉴렐릭 통합 화면 아래에 있는

   <DoNotTranslate>**New Relic insert token**</DoNotTranslate>

   아래에 키를 붙여넣으세요. 붙여넣은 후

   <DoNotTranslate>**Save**</DoNotTranslate>

   ) 클릭하세요.

7. <DoNotTranslate>**Verify the key:**</DoNotTranslate>

   Aporia 대시보드에서

   <DoNotTranslate>**Verify token**</DoNotTranslate>

   버튼을 클릭하여 키가 제대로 작동하는지 확인합니다. 상태를 나타내기 위해

   <DoNotTranslate>**Green**</DoNotTranslate>

   확인 표시 또는

   <DoNotTranslate>**Red**</DoNotTranslate>

   오류 표시가 나타나야 합니다.

## Aporia로 기계 학습 모델 모니터링

이제 New Relic과 Aporia를 통합했으므로 Aporia에서 생성된 자동화된 차트가 포함된 [대시보드를](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 사용하여 데이터를 모니터링할 수 있습니다.

1. <DoNotTranslate>**Go to the integration dashboard:**</DoNotTranslate> 의미를 확인하고 통합이 올바르게 설정되었음을 확인한 후 뉴렐릭 통합 대시보드로 돌아가서 <DoNotTranslate>**See your data**</DoNotTranslate> 클릭하세요. 그러면 뉴렐릭의 Aporia에 보고된 데이터를 표시하는 자동 생성 대시보드로 리디렉션됩니다.

2. <DoNotTranslate>**Analyze Aporia's dashboard.**</DoNotTranslate> 뉴렐릭으로 채워진 Aporia의 대시보드에는 6개의 차트가 포함되어 있습니다.

   * <DoNotTranslate>**Most active models**</DoNotTranslate>

     차트에는 선택한 기간에 예측을 보고한 다양한 모델이 표시됩니다.

   * <DoNotTranslate>**Most active model versions**</DoNotTranslate>

     차트에는 선택한 기간에 예측을 보고한 다양한 버전이 표시됩니다.

   * <DoNotTranslate>**Model inferences**</DoNotTranslate>

     그래프는 각 모델 및 버전에 대해 보고된 고유 예측 수를 표시합니다.

   * <DoNotTranslate>**Average numeric inferences**</DoNotTranslate>

     차트에는 각 모델 및 버전에 대해 보고된 평균 수치 예측 값이 표시됩니다.

   * <DoNotTranslate>**Numeric inferences heatmaps**</DoNotTranslate>

     차트에는 각 모델 및 버전에 대해 보고된 숫자 예측의 히스토그램이 표시됩니다.

   * <DoNotTranslate>**Categorical inferences**</DoNotTranslate>

     차트에는 각 모델 및 버전에 대해 보고된 다양한 고유값과 범주형 예측 빈도가 표시됩니다.

   <img
     title="An image of the Aporia dashboard"
     alt="An image of the Aporia dashboard."
     src={modelperformancemonitoringAporiaDashboard}
   />

3. <DoNotTranslate>**Filter data:**</DoNotTranslate> <DoNotTranslate>**…**</DoNotTranslate> 버튼을 클릭하고 <DoNotTranslate>**Edit**</DoNotTranslate> 클릭합니다. 오른쪽 탐색 모음의 <DoNotTranslate>**User as filter**</DoNotTranslate> 아래에서 <DoNotTranslate>**Filter the current dashboard**</DoNotTranslate> 활성화하고 <DoNotTranslate>**Save**</DoNotTranslate> 클릭합니다.

4. <DoNotTranslate>**Setup alert notifications:**</DoNotTranslate> 대시보드를 만들고 나면 데이터에 대한 알림을 받을 수 있습니다. 차트에서 NRQL 공지 조건을 생성하려면 차트를 클릭한 다음 <DoNotTranslate>**Create alert condition**</DoNotTranslate>을 클릭하세요. 조건의 이름을 지정하고 사용자 정의한 후에는 이를 기존 정책에 추가하거나 새 정책을 생성할 수 있습니다.

   <img
     title="Set up notifications for Aporia"
     alt="Set up notifications for Aporia."
     src={modelperformancemonitoringAporiaCreateAlert}
   />

5. <DoNotTranslate>**Get notified:**</DoNotTranslate> 공지조건을 만든 후에는 알림을 받을 방법을 선택할 수 있습니다. [공지 채널 설정 방법](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts) 에 대한 문서를 참조하세요.

6. <DoNotTranslate>**Correlate your incidents:**</DoNotTranslate> 공지 외에도 인시던트 인텔리전스를 사용하여 인시던트를 연관시킬 수 있습니다. [의사결정을 사용하여 인시던트를 연관시키는 방법](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/change-applied-intelligence-correlation-logic-decisions) 에 대한 문서를 참조하세요.
