Mona는 프로덕션의 전체 AI 시스템에서 데이터를 수집하고 분석하여 기본 및 사용자 지정 메트릭으로 모델 품질을 평가할 수 있도록 합니다. 데이터 드리프트, 모델 편향 또는 데이터 무결성 문제가 데이터의 특정 세그먼트에서만 발생하더라도 실행 가능한 통찰력을 통해 감지할 수 있습니다.

Mona에는 New Relic과 통합된 [모델 성능 모니터링](/docs/alerts-applied-intelligence/mlops/get-started/intro-mlops/) 이 포함되어 있으므로 Mona가 모니터링하는 AI의 상태를 New Relic의 중앙 집중식 운영 보기에 추가할 수 있습니다.

## Mona에서 New Relic으로 데이터 마이그레이션 [#migrate-data]

Mona를 사용하면 **[NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/)** 에서 나중에 사용할 수 있도록 처리된 컨텍스트 데이터를 New Relic과 자동으로 공유할 수 있습니다. Mona에서 New Relic으로 데이터를 수신하려면 계정 번호와 특수 키가 필요하며 다음과 같이 얻을 수 있습니다.

1. **New Relic 계정에 로그인:** **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > +Add more data**.

2. **Mona 선택:** 검색 창에 **Mona** 를 입력합니다.

3. **계정 ID 복사:** Mona와 통합할 계정 ID를 선택합니다. **계속** 을 클릭합니다. 이제 **복사** 아이콘을 사용하여 계정 ID를 복사하십시오.

4. **Mona 포털에서 New Relic 선택: 이제 Mona 포털** 에 로그인하고 구성 페이지로 이동하여 관련 컨텍스트 클래스를 클릭합니다. 그런 다음 **통합** 탭으로 이동하여 **New Relic** 을 선택합니다.

5. **계정 ID 붙여넣기:** New Relic에서 복사한 계정 ID를 Mona의 계정 ID 필드에 붙여넣습니다.

6. **API 키 선택 또는 생성:** 이제 New Relic으로 돌아가서 New Relic 통합 페이지에서 처리된

   <InlinePopover type="licenseKey"/>

   (또는 기존 키 선택) **처리된 컨텍스트 데이터**아래에서 . Mona로 돌아가서 **Ingest license key** 필드에 이 키를 입력합니다.

7. **통합 저장:** **저장** 을 클릭하면 데이터가 Mona와 New Relic 모두에 통합됩니다.

## New Relic에서 활성 Mona 통찰력으로 대시보드 만들기 [#create-dashboards]

Mona는 활성 통찰력이 포함된 [New Relic 대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 를 자동으로 생성합니다. 활성화하려면 다음 단계를 따르세요.

1. **Mona에서 알림 추가:** New Relic에서 통합을 설정했으면 컨텍스트 프로젝트 내에서 Mona의 **알림** 탭으로 이동합니다. 오른쪽 상단에 **있는 알림 추가를** 클릭하고 관련 필드를 계속 입력합니다.
2. **사용자 API 키 가져오기:** New Relic 통합 페이지로 돌아가서 **활성 인사이트** 아래 **에서 API 키를 선택하거나 생성합니다** . 이 키를 복사합니다.
3. **Mona에 키 붙여넣기** : Mona 알림 설정 탭에서 **New Relic 대시보드** 아래에 키를 붙여넣습니다. **저장** 을 클릭합니다.

## Mona Insights를 New Relic 사고 인텔리전스에 연결 [#connect-insights]

New Relic의 **[사고 인텔리전스](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/get-started-incident-intelligence/#get-started)** 는 사고의 상관 관계를 파악하고 환경의 소음을 줄이는 데 도움이 됩니다. 모든 사건, 출처 및 관련 이벤트에 대한 개요를 제공합니다. Mona 인사이트를 New Relic에 보내면 이를 다른 소스의 다른 문제와 연결하고 모델을 더 명확하게 이해하고 이해할 수 있습니다.

1. **Mona에서 알림 추가:** New Relic에서 통합을 설정했으면 컨텍스트 프로젝트 내에서 Mona의 **알림** 탭으로 이동합니다. 오른쪽 상단에 **있는 알림 추가를** 클릭하고 관련 필드를 계속 입력합니다.
2. **확인란 클릭:** Mona 알림 설정 탭에서 **New Relic 사건 인텔리전스 포함** 확인란을 선택합니다.
3. **저장 및 알림 받기:** 알림 설정 페이지 하단의 저장을 클릭하세요. 이제 다음에 Mona가 통찰력을 찾으면 Mona에서 알림을 받고 New Relic에서 알림을 받게 되며 전용 [New Relic 대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 에서 직접 통찰력을 볼 수 있습니다.

<Callout variant="Sending Older Insights">
  New Relic의 사고 정보는 지난 24시간 동안 처음 나타난 새로운 통찰력만 받을 수 있습니다. 그 이전의 인사이트는 New Relic에서 허용하지 않습니다.
</Callout>

## New Relic 대시보드에서 Mona 모델 보기 및 탐색 [#view-explore]

Mona에서 New Relic 통합을 구성하면 표준 모델 메트릭이 New Relic으로 전송됩니다. 모델 관찰 가능성 대시보드를 보고 비즈니스 컨텍스트에 맞게 조정된 모델 관찰 가능성에 대한 특정 모델 메트릭 및 인시던트 정책을 구성하십시오.

1. **Mona 통합 대시보드로 이동합니다.** 통합 및 통찰력을 설정했으면 New Relic 통합 페이지로 돌아가 **데이터 보기** 를 클릭합니다. 그러면 Mona에서 제공하는 자동 생성 대시보드로 리디렉션됩니다.
2. **Mona의 대시보드 분석:** 초기 대시보드는 컨텍스트 클래스당 데이터 흐름으로 채워집니다. 또한 **New Relic 대시보드에서 활성 통찰력 보기** 링크를 클릭하여 Mona 통찰력을 볼 수 있습니다. 초기에 통합한 데이터를 기반으로 Mona에서 자동으로 생성한 동적 AI 데이터를 표시합니다.

<img
  title="Anayze Mona dashboards"

/>

3. **경고 알림 설정:** 대시보드에서 일부 차트를 생성하면 데이터에 대한 경고를 받을 수 있습니다. 차트에서 NRQL [경고 조건](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/) 을 생성하려면 차트 위젯 **...** 메뉴를 클릭한 **다음 경고 조건 생성** 을 클릭합니다. 조건의 이름을 지정하고 사용자 정의한 후에는 기존 정책에 추가하거나 새 정책을 생성할 수 있습니다.
4. **알림 받기:** 알림 조건을 만든 후에 알림 방법을 선택할 수 있습니다. [알림 채널을 설정하는 방법](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/) 에 대한 문서를 참조하세요.
5. **인시던트 상관 관계:** 알림 외에도 인시던트 인텔리전스를 사용하여 인시던트의 상관 관계를 지정할 수 있습니다. [의사결정을 사용하여 사건을 연관시키는 방법](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/change-applied-intelligence-correlation-logic-decisions/) 에 대한 문서를 참조하십시오.

이제 New Relic을 Mona와 성공적으로 통합했습니다. 새로 생성된 Mona 경고는 이제 New Relic 경고와 상관 관계가 있으며 가장 최근에 보고된 예측에 대한 데이터를 볼 수 있습니다.
