여기에서는 New Relic 모바일 앱 계정을 인증하는 방법과 모바일 장치에서 사용자를 추가하거나 제거하는 방법에 대해 설명합니다.

## 사용자 인증 [#authentication]

New Relic 계정에 따라 New Relic 모바일 앱을 설치할 때 추가 설치 또는 인증 단계가 필요할 수 있습니다.

  <thead>
    <tr>

      </th>

      <th>
        **추가 요구 사항**
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        신규 사용자
      </td>

      <td>
        아직 New Relic 계정이 없는 경우:

        1. 데스크탑 웹 브라우저에서 [New Relic 계정을 만드](https://newrelic.com/signup) 십시오.

        2. 적절한 New Relic 에이전트를 사용 [하여 애플리케이션을 설치합니다](/docs/site/installing-the-agent) .

           새 계정 설정의 일부로 비밀번호 재설정 링크가 포함된 이메일을 받게 됩니다. 모바일 앱의 경우 비밀번호 재설정 링크는 20분 후에 만료됩니다.
      </td>
    </tr>

    <tr>
      <td>

      </td>

      <td>
        추가 요구 사항 없음 로그인하면 애플리케이션, 호스트 및 주요 트랜잭션이 자동으로 나타납니다.
      </td>
    </tr>

    <tr>
      <td>
        New Relic 파트너 계정이 있는 사용자
      </td>

      <td>
        파트너에 따라 다른 [인증 프로세스](/docs/ios/ios-with-partners) 를 완료해야 할 수 있습니다.

        **Azure Store 사용자:** Azure Storefront와 New Relic 간의 긴밀한 통합으로 인해 Azure Storefront 사용자는 New Relic Android 또는 iOS 앱에서 계정에 액세스할 수 없습니다.
      </td>
    </tr>

    <tr>
      <td>
        SAML-SSO 사용 계정이 있는 사용자
      </td>

      <td>
        New Relic 모바일 앱에 로그인하면 세션이 자동으로 웹 브라우저로 리디렉션됩니다. 여기에서 New Relic SAML-SSO 계정에 로그인할 수 있습니다.

        모바일 장치에서 SAML-SSO 계정을 사용할 때 오류가 표시되면 데스크톱 웹 브라우저로 **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 에 로그인할 수 있는지 확인하십시오.

        * 그렇지 않은 경우 관리자에게 문의하십시오.
        * 그렇다면 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으십시오.
      </td>
    </tr>
  </tbody>
</table>

  <Collapser
    id="switch-accounts"
    title="계정 간 전환"
  >
    사용자 이름과 연결된 계정 간에 전환하려면:

    1. **사용자** 메뉴에서 사용자 이름을 선택합니다.
    2. 계정 이름을 선택합니다.
  </Collapser>

  <Collapser
    id="Remove a username"
    title="사용자 이름 제거 또는 다시 추가"
  >
    이 장치에서 특정 사용자 이름을 제거하려면:

    1. **사용자** 메뉴에서 **로그아웃** 을 선택합니다.

    2. 이 장치에서 사용자를 제거하려면 사용자의 빨간색 마이너스를 선택하십시오. <Icon style={{color: 'red'}} name="fe-minus-circle"/> 상.

    3. 사용자의 **로그아웃** 아이콘을 선택합니다.

       사용자를 다시 추가하려면 해당 사용자 이름으로 다시 로그인하세요.
  </Collapser>
</CollapserGroup>
