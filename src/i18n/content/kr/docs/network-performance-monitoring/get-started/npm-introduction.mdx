시스템 성능이 저하되면 코드, 인프라 또는 기본 네트워크로 인한 것인지 알아야 합니다. 그리고 당신은 당신의 노력에 집중할 수 있도록 빨리 알아야합니다. 네트워크 모니터링을 사용하면 애플리케이션, 인프라, 디지털 경험 및 네트워크 데이터를 모두 한 곳에서 상호 연관시키고 분석할 수 있으며 네트워크 성능과 전체 시스템 성능이 서로 어떤 영향을 미치는지 이해할 수 있습니다.

## 네트워크 데이터 유형 [#network-data-types]

다음 유형의 네트워크 데이터를 모니터링할 수 있습니다.

* **SNMP를 통한 장치 성능: SNMP** (Simple Network Management Protocol)는 네트워크 장치 간에 관리 정보를 교환하기 위한 응용 프로그램 계층 프로토콜입니다. SNMP 데이터를 [New Relic](https://one.newrelic.com) 으로 보내려면 [SNMP 데이터 모니터링을 설정](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring) 하십시오.
* **네트워크 시스템 로그** : 변경된 사항, 오류 및 기타 장치 이벤트에 대한 통찰력을 제공하여 '왜?' 성능 원격 측정 변경 뒤에 있습니다. 네트워크 syslog 데이터를 보내려면 [네트워크 syslog 모니터링을 설정](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring) 하십시오.
* **클라우드 흐름 로그** : 소스 및 대상 IP 주소 및 포트, 두 끝점을 모두 설명하는 메타데이터, 주요 네트워크 성능 원격 측정, 특정 통신 시도 상태를 캡처하여 호스트와 다른 서비스 간의 통신을 자세히 설명합니다.[클라우드 흐름 로그 모니터링을 설정합니다](/docs/network-performance-monitoring/setup-performance-monitoring/aws-vpc-flow-monitoring) .

<Callout title="베타 기능">
  Cloud Flow 로그는 아직 베타 버전입니다.
</Callout>

* **네트워크 흐름 로그** : 온프레미스 네트워크의 네트워크 인터페이스로 들어오고 나가는 IP 트래픽에 대한 정보를 캡처합니다. 네트워크 흐름 데이터를 [New Relic](https://one.newrelic.com) 으로 보내려면[네트워크 흐름 데이터 모니터링을 설정](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring) 하십시오.

<Callout variant="important">
  네트워크에 대한 더 나은 가시성을 위해 SNMP 및 네트워크 흐름 데이터를 모두 구성하는 것이 좋습니다. 이렇게 하면 네트워크 문제를 해결하고 최적화하기 위한 성능 메트릭과 트래픽 패턴이 모두 제공됩니다.
</Callout>
