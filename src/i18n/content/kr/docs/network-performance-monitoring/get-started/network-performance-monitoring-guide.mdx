당신은 SNMP 지원 장치 네트워크에 의존하는 운영 엔지니어입니다. 귀하의 팀은 소프트웨어 및 인프라 모니터링을 위해 이미 New Relic의 기능을 사용하고 있지만 네트워크 장치도 모니터링해야 할 때입니다.

이 트랙에서는 Kentik의 ktranslate 라이브러리를 사용하여 네트워크 장치를 자동으로 검색하고, 데이터를 수집하고, 해당 데이터를 New Relic으로 보내 네트워크 성능을 인프라, 애플리케이션 및 디지털 경험과 연관시킵니다. 그런 다음 New Relic을 사용하여 네트워크 상태를 모니터링합니다.

## 목표

* 네트워크 장치에서 SNMP 데이터를 폴링하여 New Relic으로 전송
* 네트워크 흐름 데이터를 샘플링하여 New Relic으로 전송
* 네트워크 장치 모니터링을 위한 New Relic 대시보드 생성
* 디바이스를 논리적으로 그룹화하고 이상 감지를 설정하기 위한 New Relic 워크로드 생성
* 새로운 데이터를 사용하여 네트워크 내 행동 이해

## 요구 사항

* 무료 New Relic 계정
* 전체 플랫폼 사용자 또는 Nerdpack 수정 사용자 권한이 있는 핵심 사용자

## 숙제

잘하셨어요! ktranslate 및 New Relic을 사용하여 네트워크 장치를 모니터링하는 방법을 익히셨으므로 이제 실제 네트워크 모니터링을 준비하면서 스스로 할 수 있는 몇 가지 작업이 있습니다.

* New Relic의 [네트워크 모니터링을 최대한 활용하는 방법을 알아보려면 "네트워크 모니터링 시작하기](/docs/network-performance-monitoring/get-started/npm-introduction/)"를 읽어보십시오.
* ktranslate 작동 방식을 알아보려면 [ktranslate GitHub 리포지토리를](https://github.com/kentik/ktranslate) 방문하십시오.
* New Relic 엔터티와 자신의 엔터티를 합성하는 방법에 대해 자세히 알아보려면 "[엔터티란 무엇입니까?](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)"를 읽으십시오. 이렇게 하면 **Kentik 기본** 엔터티에서 더 많은 마일리지를 얻을 수 있습니다.
