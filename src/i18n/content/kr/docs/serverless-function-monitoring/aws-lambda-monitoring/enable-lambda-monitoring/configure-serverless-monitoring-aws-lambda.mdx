[AWS Lambda에 대한 서버리스 모니터링을 활성화](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-aws-lambda-monitoring) 한 후 추가 구성을 추가하여 데이터를 미세 조정할 수 있습니다.

## 무한 추적 활성화 [#infinite-tracing]

Lambda 계층을 사용하여 함수를 설정하는 경우 분산 추적이 기본적으로 포함됩니다.

Lambda 함수가 [Infinite Tracing](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing) 이라는 꼬리 기반 분산 추적을 사용하는 서비스와 통신하는 경우 Lambda 함수에 대해 이 기능을 활성화해야 합니다.이렇게 하면 Infinite Tracing에서 추적하는 나머지 범위에 Lambda 범위가 포함됩니다.

Lambda 함수에 대해 이를 설정하려면 추적 관찰자를 생성하고 범위를 보고해야 하는 Lambda 함수(데이터 소스)를 선택하는 [무한 추적 단계](/docs/distributed-tracing/infinite-tracing/set-trace-observer) 를 참조하십시오.

## 알림 설정 [#alerts]

저희의 <InlinePopover type="alerts"/>APM 리소스, 롸듸 프로세서, NRQL 쿼리 등을 포함하여 뉴렐릭으로 모니터링할 수 있는 모든 것에 대한 공지를 얻을 수 있습니다. [알림 문서를](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/) 참조하세요.

## 맞춤 이벤트 추가 [#custom-events]

기본적으로 제공되는 데이터 외에도 고유한 [이벤트](/docs/using-new-relic/welcome-new-relic/get-started/glossary#event) 또는 [속성](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute) 을 설정할 수도 있습니다.

이러한 언어별 설정에 대한 자세한 내용은 [AWS Lambda에서 사용자 지정 속성 및 이벤트 구성](https://discuss.newrelic.com/t/configuring-custom-attributes-and-events-in-aws-lambda/96043) 을 참조하십시오.
