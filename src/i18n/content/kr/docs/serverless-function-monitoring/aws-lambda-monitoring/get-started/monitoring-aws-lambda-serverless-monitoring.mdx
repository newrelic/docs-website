서버리스 [AWS Lambda](https://aws.amazon.com/lambda/) 기능에 대한 심층적인 성능 모니터링을 제공합니다. 기능, 작동 방식 및 요구 사항에 대해 알아보려면 계속 읽으십시오.

**이 Lambda 모니터링 기능은 CloudWatch 데이터만 사용하는 [Lambda 모니터링 통합](/docs/integrations/amazon-integrations/aws-integrations-list/aws-lambda-monitoring-integration) 과 다릅니다** . 서버리스 Lambda 모니터링은 더 광범위하며 CloudWatch 데이터와 코드 수준 계측을 모두 사용하여 보다 심층적인 모니터링을 제공합니다.

## 중요한 이유 [#why]

코드 변경에 의존하지 않고 서버리스 기능에 대한 모니터링 및 관찰 가능성을 신속하게 계측해야 합니까? Lambda 계층은 Lambda 함수의 가장 자세한 동작에 대한 통합된 가시성을 제공하므로 서버리스 애플리케이션에서 무슨 일이 일어나고 있는지 이해할 수 있습니다.

이 모든 정보를 사용하여 시스템 문제를 해결하고 기능을 최적화하여 서버리스 아키텍처에서 더 빠르게 작동하고 자신 있게 전달할 수 있습니다. 계측 시간을 줄이고 구축에 더 많은 시간을 할애하십시오.

[Lambda 모니터링](/docs/serverless-function-monitoring/aws-lambda-monitoring/ui-data/understand-lambda-monitoring-ui#find-data) 은 다음을 제공합니다.

* 자세한 기간, 콜드 스타트, 예외 및 역추적과 같은 성능 데이터를 포함하여 Lambda 함수의 **모든 호출** .
* API Gateway, ALB SNS, SQS, DynamoDB 등과 같이 각 AWS Lambda 호출을 트리거한 이벤트에 대한 컨텍스트와 속성을 제공하는 AWS Lambda **이벤트 소스 정보** .
* [분산 추적](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) 을 통해 **전체 에코시스템** 에 대한 가시성: Lambda로 이어지는 요청 경로를 확인하고 다른 분산 추적에서 Lambda 범위를 확인합니다.
* 메트릭, 속성 및 추적 데이터와 함께 **전체 호출 및 기능 수준 로그** 를 제공하는 컨텍스트에 로그인합니다.
* **Inventoried tags and metadata** \[인벤토리 태그 및 메타데이터]. AWS 엔터티에서 정보를 검색하여 팀 또는 기능 구성의 특정 메타데이터 속성 또는 호출 자체를 필터링하고 패싯할 수 있는 기능을 제공합니다.
* 쿼리를 작성할 필요 없이 속성 또는 기능별로 [대시보드 및 차트](/docs/serverless-function-monitoring/aws-lambda-monitoring/ui-data/understand-lambda-monitoring-ui#chart-data) 를 즉시 패싯하여 사용자 지정 데이터를 탐색하는 **자동 패싯 빌더** 로 데이터를 쉽게 쿼리할 수 있습니다.
* 팀이 콜드 스타트를 줄일 수 있는 [AWS Snapstart](https://docs.aws.amazon.com/lambda/latest/dg/snapstart.html) 에 대한 지원을 제공합니다. New Relic 서버리스 모니터링은 AWS의 스냅스타트 기능이 애플리케이션 성능에 미치는 영향을 분석하는 데 도움이 됩니다. New Relic은 생성된 새로운 람다를 감지하고 각 함수의 호출 및 기간에 대한 완전한 가시성을 제공합니다. New Relic이 콜드 스타트를 식별하면 팀은 자세한 추적을 사용하여 도구에서 함수의 구성을 검사하고 런타임 및 리전이 AWS snapstart에서 지원되는지 확인할 수 있습니다.

Lambda 데이터는 자체 [전용 UI](/docs/lambda-monitoring-ui) 에 표시됩니다.
