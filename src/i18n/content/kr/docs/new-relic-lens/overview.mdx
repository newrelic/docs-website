  이 기능은 아직 개발 중이지만 꼭 사용해 보시기 바랍니다!

  이 기능은 현재 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 미리보기 프로그램의 일부로 제공됩니다.
</Callout>

뉴롤릭 사용자로서 귀하는 여러 데이터베이스, 데이터 웨어하우스 및 스프레드시트에 저장된 데이터와 함께 스프레드시트 데이터를 분석해야 할 수도 있습니다. 각 소스에서 데이터를 수동으로 내보내고 오프라인에서 운영자 데이터와 결합하는 것은 시간이 많이 걸리고 생산성을 저하시킵니다. 또한 규정 준수, 보안 또는 운영상의 요구 사항으로 인해 모든 데이터를 단일 데이터베이스로 통합할 수 없는 경우가 많습니다.

<DNT>New Relic Lens</DNT> 이러한 과제들을 해결합니다. <DNT>Lens</DNT> 은 뉴렐릭 내의 통합 쿼리 플랫폼으로, 뉴렐릭에서 직접 여러 데이터 소스를 연결하고 쿼리할 수 있습니다. <DNT>Lens</DNT> 에 ANSI SQL을 사용하여 쿼리를 작성하고 다양한 소스의 데이터를 분석하고 상관 관계를 파악하기 위해 교차 데이터베이스 조인을 수행합니다. <DNT>Lens</DNT> 각 데이터 소스에 계산 작업을 푸시하고, 각 데이터 소스는 쿼리의 해당 부분을 처리하고 결과만 반환합니다. <DNT>Lens</DNT> 그런 다음 이러한 결과를 결합하여 외부 소스에서 원시 데이터를 수집하지 않음으로써 데이터 전송을 최소화하고 보안을 유지합니다.

## 작동 원리 [#how-it-works]

<DNT>Lens</DNT> 사용하는 데에는 다음 단계가 포함됩니다.

<Steps>
  <Step>
    ### 렌즈 활성화

    <DNT>Lens</DNT> 공개 미리보기 버전으로 제공됩니다. <DNT>Organization Manager</DNT> 에 문의하여 <DNT>**[Previews &amp; Trials](https://one.newrelic.com/admin-portal/promotion-management/home)**</DNT> 페이지에서 참여 여부를 확인하세요.
  </Step>

  <Step>
    ### 권한 확인

    <DNT>Organization Manager</DNT> 인 경우 기본적으로 모든 <DNT>Lens</DNT> 기능에 액세스하는 데 필요한 권한이 있습니다. <DNT>Organization Manager</DNT> 가 아닌 경우 [필요한 권한이 있는 사용자 지정 역할을 만들](/docs/new-relic-lens/overview/#permissions) 려면 <DNT>Authentication Domain Manager</DNT> 에 문의하세요.
  </Step>

  <Step>
    ### 설정 <DNT>connectors</DNT>

    <DNT>Lens</DNT> NRDB 데이터용 내장 함수 <DNT>system connectors</DNT> 이 포함되어 있습니다. 필요한 권한이 있으면 [<DNT>data connectors</DNT>](/docs/new-relic-lens/set-up-connectors) 외부 데이터 소스에 연결하도록 설정할 수 있습니다.

    * **클라우드 데이터 웨어하우스**: <DNT>Snowflake</DNT>, <DNT>Redshift</DNT>, <DNT>ClickHouse</DNT>
    * **관계형 데이터베이스**: <DNT>PostgreSQL</DNT>, <DNT>MySQL</DNT>, <DNT>SQL Server</DNT>
    * **문서 데이터베이스**: <DNT>MongoDB</DNT>, <DNT>Elasticsearch</DNT>
    * **스프레드시트**: <DNT>Google Sheets</DNT>
    * **데이터 레이크**: <DNT>Iceberg</DNT>
    * **측정 및 검토**: <DNT>Prometheus</DNT>, <DNT>AWS CloudWatch</DNT>
  </Step>

  <Step>
    ### 데이터 쿼리

    테이블을 참조하고 소스 전체에서 데이터를 조인하려면 `connection_name.schema.table` 형식을 사용하여 [쿼리 빌더, Notebooks 또는 사용자 정의 대시보드](/docs/new-relic-lens/query-data) 에 SQL 쿼리를 작성하세요.
  </Step>

  <Step>
    ### <DNT>materialized views</DNT> 생성 (선택 사항)

    자주 실행되는 복잡한 쿼리의 경우 비용을 절감하고 성능을 향상시키기 위해 결과를 [<DNT>materialized views</DNT>](/docs/new-relic-lens/materialized-views) 으로 저장할 수 있습니다.
  </Step>
</Steps>

## 사용 사례 [#use-cases]

Lens는 다양한 데이터 분석 시나리오를 지원합니다.

* **예산 분석**: 할당된 예산 대비 지출을 추적하기 위해 <DNT>NRDB</DNT> 의 뉴렐릭 활용 데이터를 <DNT>Snowflake</DNT> 에 저장된 예산 데이터와 비교합니다.
* **데이터베이스 최적화**: 여러 수동 내보내기/가져오기 단계(한 데이터베이스에서 다른 데이터베이스로)를 필요한 모든 데이터를 결합하는 단일 SQL 쿼리로 대체하여 비용을 절감하고 효율성을 향상시킵니다.
* **지역별 데이터 통합**: 다양한 데이터 소스의 데이터를 통합하여 여러 지리적 지역의 사용자 트래픽을 추적합니다.
* **<DNT>materialized views</DNT>이용한 성능 최적화**: 경영진 대시보드는 5개 이상의 데이터베이스에서 데이터를 통합하고 매주 새로 고쳐집니다. 다중 데이터베이스 `JOIN` 쿼리의 결과를 <DNT>materialized view</DNT> 에 저장함으로써 누군가가 대시보드를 로드할 때마다 계산 비용이 많이 드는 쿼리를 실행하는 것을 방지할 수 있습니다. 이를 통해 쿼리 비용이 절감되고 페이지 로딩 속도가 크게 향상됩니다.

## Lens를 사용할 수 있는 곳 [#where-to-use]

뉴렐릭 플랫폼의 다음 영역에서 Lens를 사용할 수 있습니다.

* **쿼리 빌더**: 쿼리 빌더에서 SQL 쿼리를 직접 작성하고 실행합니다.
* <DNT>**Notebooks**</DNT>노트북에 SQL 블록을 생성하여 쿼리와 시각화, 마크다운 문서를 결합하세요.
* **고객 Dashboards**: 개인 대시보드에 SQL 기반 차트를 추가하세요.

<Callout variant="important">
  SQL 기반 차트는 공개 대시보드에서 지원되지 않습니다.
</Callout>

## 사용자 역할 및 권한 [#permissions]

<DNT>Connector</DNT> 및 <DNT>materialized view</DNT> 액세스는 특정 RBAC 권한에 의해 제어됩니다. <DNT>**Organization Manager**</DNT> 역할을 가진 사용자는 기본적으로 모든 <DNT>Lens</DNT> 관련 RBAC 권한을 갖습니다.

다른 사용자의 경우 <DNT>**Authentication Domain Manager**</DNT> 은 필요한 <DNT>Lens</DNT> 권한을 가진 [NerdGraph를 통해 조직 범위의 사용자 지정 역할을 생성](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#organization-scoped-role) 해야 합니다. 역할을 생성할 때 아래 표의 권한 ID를 `permissionIds` 에 추가하세요.

<DNT>
  /* UI-based permission assignment (for future use when UI support is available): For other users, the &lt;DNT&gt;**Authentication Domain Manager**&lt;/DNT&gt; needs to create an [organization-scoped custom role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) with the required &lt;DNT&gt;Lens&lt;/DNT&gt; permissions from the &lt;DNT&gt;**Query Your Data**&lt;/DNT&gt; section. check... */
</DNT>
