**쿼리 빌더**, <DNT>**Notebooks**</DNT> 및 **사용자 정의 대시보드** 에서 <DNT>Lens</DNT> 쿼리를 작성할 수 있습니다. 이 페이지에서는 각 플랫폼에서 데이터를 조회하는 방법을 설명합니다.

<Callout variant="tip">
  데이터 소스를 쿼리할 때 <DNT>Lens</DNT> 외부 소스에서 가져올 수 있는 행 수를 제한하지 않습니다. 그러나 <DNT>NRDB</DNT> 부터 <DNT>Lens</DNT> 까지의 쿼리는 <DNT>NRQL</DNT> 과 동일한 5,000행 제한을 유지합니다.
</Callout>

## 전제 조건 [#prerequisites]

시작하기 전에:

* 쿼리 데이터 소스에 필요한 [권한이](/docs/new-relic-lens/overview#permissions) 있는지 확인하십시오.
* 외부 데이터 소스를 쿼리해야 하는 경우 [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors) 이 설정되어 있는지 확인하십시오.

## 쿼리 빌더에서 쿼리 [#query-builder]

1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> 으)로 이동합니다.
2. 페이지 하단의 서랍에서 <DNT>**Query your data**</DNT> 콘솔을 엽니다.
3. 쿼리 편집기에서 쿼리 언어로 <DNT>**Lens data (SQL)**</DNT> 선택합니다.
4. NRDB 데이터를 조회하려면 계정 선택기에서 계정을 선택하세요. 기타 <DNT>data connectors</DNT> 은 조직 수준에서 사용 가능하며 계정 선택이 필요하지 않습니다.
5. [테이블 참조 형식을](#table-format) 사용하여 SQL 쿼리를 작성하세요. `connection_name.schema.table`
6. <DNT>**Run**</DNT> 클릭하세요. 차트는 사용자의 쿼리 결과를 기반으로 표시됩니다.

## 쿼리를 저장하세요 <DNT>Notebooks</DNT> [#notebooks]

[노트북은](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/) 쿼리, 시각화 및 마크다운 문서를 결합한 공유 가능한 문서입니다.

1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> 으)로 이동합니다.
2. 페이지 하단의 서랍에서 <DNT>**Query your data**</DNT> 콘솔을 엽니다.
3. <DNT>**Notebooks**</DNT> 옆의 토글을 활성화하세요. 페이지 하단에 <DNT>**+ SQL**</DNT> 버튼이 나타납니다.
4. <DNT>**+ SQL**</DNT> 클릭하여 SQL 블록을 추가하세요.
5. NRDB 데이터를 조회하려면 계정 선택기에서 계정을 선택하세요. 기타 <DNT>data connectors</DNT> 은 조직 수준에서 사용 가능하며 계정 선택이 필요하지 않습니다.
6. [테이블 참조 형식을](#table-format) 사용하여 SQL 쿼리를 작성하세요. `connection_name.schema.table`
7. <DNT>**Run**</DNT> 클릭하세요. 차트는 사용자의 쿼리 결과를 기반으로 표시됩니다.
8. <DNT>**Save**</DNT> 을(를) 클릭합니다.

더 자세한 안내는 [노트북 설명서를](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/) 참조하세요.

## 맞춤형 대시보드의 쿼리 [#dashboards]

사용자 [정의 대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) 에 SQL 기반 차트를 추가할 수 있습니다.

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**</DNT> 으)로 이동합니다.
2. [새 대시보드를 만들](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create-new) 거나 기존 대시보드를 엽니다.
3. <DNT>**+ Add widget**</DNT> 클릭하고 차트 유형을 선택하세요.
4. 쿼리 편집기에서 쿼리 언어로 <DNT>**Lens data (SQL)**</DNT> 선택합니다.
5. NRDB 데이터를 조회하려면 계정 선택기에서 계정을 선택하세요. 기타 <DNT>data connectors</DNT> 은 조직 수준에서 사용 가능하며 계정 선택이 필요하지 않습니다.
6. [테이블 참조 형식을](#table-format) 사용하여 SQL 쿼리를 작성하세요. `connection_name.schema.table`
7. <DNT>**Run**</DNT> 클릭하세요. 차트는 사용자의 쿼리 결과를 기반으로 표시됩니다.
8. <DNT>**Add to dashboard**</DNT> 클릭하세요. 자세한 지침은 [맞춤형 대시보드 설명서를](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 참조하세요.

<Callout variant="important">
  SQL 기반 차트는 비공개 대시보드에서만 지원됩니다. 공개 대시보드는 SQL 쿼리를 지원하지 않습니다.
</Callout>

## SQL 구문 [#syntax]

<DNT>Lens</DNT> ANSI SQL을 사용하여 데이터 소스를 쿼리하므로 데이터베이스별 쿼리 언어를 배울 필요가 없습니다.

### 테이블 참조 형식 [#table-format]

<DNT>Lens</DNT> 에 쿼리를 작성할 때는 다음 형식을 사용하여 테이블을 참조하십시오.

```
connection_name.schema.table
```

예를 들어:

* `telemetry.Transaction` - NRDB(텔레메트리)의 프로세서 테이블 <DNT>connector</DNT>
* `snowflake.sales.orders` - &quot;sales&quot; 스키마에 있는 &quot;snowflake&quot;라는 이름의 <DNT>Snowflake</DNT> <DNT>connector</DNT> 에서 orders 테이블을 쿼리합니다.
* `gsheets.default.employees` - &quot;gsheets&quot;라는 이름의 <DNT>Google Sheets</DNT> <DNT>connector</DNT> 을 쿼리합니다.

### 지원되는 SQL 문 [#supported-sql]

<DNT>Lens</DNT> 표준 ANSI SQL 문을 지원합니다.

* `SELECT` - 하나 이상의 테이블에서 가져온 데이터
* `JOIN` - 여러 테이블의 데이터를 결합합니다(데이터베이스 간 조인 포함).
* `WHERE` - 검색 결과 필터링
* `GROUP BY` - 집계 데이터
* `ORDER BY` - 결과 정렬
* `LIMIT` - 반환되는 행 수를 제한합니다.
* `WITH` (공통 테이블 표현식) - 복잡한 쿼리를 위한 공통 테이블 표현식을 정의합니다.

### 성능 향상 팁 [#performance-tips]

* **필터를 일찍 사용하세요**: `WHERE` 절을 적용하여 처리되는 데이터 양을 줄이세요.
* **결과 제한**: 데이터 탐색 시 반환되는 행 수를 제한하려면 `LIMIT` 사용하세요.
* **<DNT>materialized views</DNT>고려하세요**: 자주 실행되는 복잡한 쿼리의 경우 <DNT>materialized view</DNT> 을 생성하세요.
* **조인 최적화**: 대규모 테이블을 조인할 때, 가능하면 조인 전에 데이터를 필터링하십시오.

## NRQL과의 쿼리 구문 차이점 및 예시 [#syntax-differences]

<DNT>Lens</DNT> NRQL 대신 ANSI SQL을 사용합니다. 다음 예시는 주요 차이점과 일반적인 쿼리 패턴을 보여줍니다.

<CollapserGroup>
  <Collapser id="groupby" title="`FACET` 대신 `GROUP BY`를 사용하세요.">
    결과를 그룹화할 때는 `GROUP BY` 사용하고 그룹화된 열을 `SELECT` 문에 포함시키세요.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    FACET enduser.id
    ```

    **SQL:**

    ```sql
    SELECT "enduser.id", count(*)
    FROM telemetry.PageViewTiming
    GROUP BY "enduser.id"
    ```
  </Collapser>

  <Collapser id="where-timestamp" title="`SINCE` 대신 `타임스탬프`와 함께 `WHERE`를 사용하세요.">
    `timestamp` 열과 날짜 간격을 사용하여 `WHERE` 시간 범위를 필터링합니다.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    SINCE 1 day ago
    ```

    **SQL:**

    ```sql
    SELECT count(*)
    FROM telemetry.PageViewTiming
    WHERE timestamp > current_date - INTERVAL '1' DAY
    ```
  </Collapser>

  <Collapser id="timeseries" title="에포크 형식의 시계열">
    시계열 시각화를 생성하려면 첫 번째 열에 에포크 형식의 날짜가 포함되도록 쿼리를 구성하십시오. 날짜를 변환하려면 `to_unixtime()` 사용하세요.

    ```sql
    WITH orders AS (
      SELECT
        to_unixtime(date(date_parse(order_date, '%Y-%m-%d'))) as dt,
        count(*) as orders
      FROM gsheets.default.orders
      GROUP BY 1
    )
    SELECT *
    FROM orders
    WHERE dt >= to_unixtime(date('2025-10-19'))
      AND dt <= to_unixtime(date('2025-10-21'))
    ORDER BY dt ASC
    ```
  </Collapser>

  <Collapser id="cross-db-join" title="교차 데이터베이스 `JOIN`">
    표준 SQL `JOIN` 구문을 사용하여 여러 소스의 데이터를 단일 쿼리로 결합합니다.

    ```sql
    WITH sf AS (
      SELECT DISTINCT user_id, assigned_success_manager
      FROM demogorgon.default.relibank_opportunity_matrix
    )
    SELECT
      sf.assigned_success_manager,
      MAX(nrdb.largestContentfulPaint)
    FROM telemetry.PageViewTiming nrdb
    LEFT JOIN sf ON nrdb."enduser.id" = sf.user_id
    WHERE timingName = 'largestContentfulPaint'
      AND nrdb.timestamp > current_date - INTERVAL '1' DAY
    GROUP BY sf.assigned_success_manager
    ```
  </Collapser>

  <Collapser id="query-mv" title="구체화된 뷰를 쿼리합니다.">
    `mv.` 접두사를 사용하여 <DNT>materialized views</DNT> 참조하십시오.

    ```sql
    SELECT *
    FROM mv.weekly_performance_summary
    WHERE week_number = 42
    ```

    <DNT>materialized views</DNT> 다른 데이터 소스와 결합할 수 있습니다.

    ```sql
    SELECT
      mv.store_id,
      mv.total_sales,
      s.store_name,
      s.region
    FROM mv.store_sales_mv mv
    JOIN snowflake.retail.stores s ON mv.store_id = s.store_id
    WHERE mv.total_sales > 10000
    ```
  </Collapser>
</CollapserGroup>
