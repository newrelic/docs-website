규모가 크고 계산 비용이 많이 드는 쿼리의 경우 결과를 <DNT>materialized views</DNT> in <DNT>Lens</DNT> 로 저장할 수 있습니다. <DNT>materialized view</DNT> 쿼리의 미리 계산된 결과를 저장하므로 매번 복잡한 쿼리를 실행하는 대신 저장된 데이터를 쿼리할 수 있습니다.

## 이점 <DNT>materialized views</DNT> [#benefits]

<DNT>Materialized views</DNT> 도와드리겠습니다:

* **쿼리 비용 절감**: 외부 데이터 소스에서 동일한 데이터를 반복적으로 가져오는 것을 방지하여 스캔 또는 전송된 데이터에 따른 비용을 줄일 수 있습니다.
* **성능 향상**: 매번 여러 데이터베이스에 걸쳐 복잡한 조인을 실행하는 대신 미리 계산된 결과를 쿼리하세요.
* **대시보드 단순화**: 다중 데이터베이스 쿼리의 결과를 저장하고 대시보드에서 간단히 참조하십시오.

## 전제 조건 [#prerequisites]

시작하기 전에:

* <DNT>materialized views</DNT> 생성하고 관리하는 데 필요한 [권한이](/docs/new-relic-lens/overview#permissions) 있는지 확인하십시오.
* 외부 데이터 소스를 쿼리해야 하는 경우 [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors) 이 설정되어 있는지 확인하십시오.

## 생성하세요 <DNT>materialized view</DNT> [#create]

<DNT>materialized view</DNT> 을 생성하려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT> 으)로 이동합니다.
2. <DNT>**Create materialized view**</DNT> 을(를) 클릭합니다.
3. <DNT>materialized view</DNT> 에 고유한 이름을 입력하세요.
4. NRDB 데이터를 조회하려면 계정 선택기에서 계정을 선택하세요. 이 계정을 통해 NRDB 데이터를 외부 데이터 소스와 결합할 수 있습니다.
5. [테이블 참조 형식을](/docs/new-relic-lens/query-data#table-format) 사용하여 SQL 쿼리를 작성하세요. `connection_name.schema.table`
6. <DNT>**Create**</DNT> 을(를) 클릭합니다.

<DNT>materialized view</DNT> 이 목록에 나타납니다. 쿼리의 복잡성에 따라 결과를 저장하는 데 시간이 다소 걸릴 수 있습니다.

## 새로 고침 <DNT>materialized view</DNT> [#refresh]

<DNT>Materialized views</DNT> 데이터가 생성되었거나 마지막으로 갱신되었을 때의 스냅샷을 포함합니다. 최신 데이터를 얻으려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT> 으)로 이동합니다.
2. 새로 고칠 <DNT>materialized view</DNT> 을 찾으세요.
3. 클릭하세요 <Icon name="fe-more-horizontal" /> 메뉴.
4. <DNT>**Refresh**</DNT> 선택합니다.

뷰는 원래 쿼리를 다시 실행하고 저장된 결과를 최신 데이터로 업데이트합니다.

<Callout variant="important">
  <DNT>materialized views</DNT> 새로 고침 빈도를 결정할 때 데이터의 최신성 요구 사항을 고려하십시오. 복잡한 쿼리를 사용하여 뷰를 자주 새로 고치면 비용이 증가할 수 있습니다.
</Callout>

## 쿼리 <DNT>materialized view</DNT> [#query]

<DNT>materialized view</DNT> 테이블은 <DNT>Lens</DNT> 의 다른 테이블과 마찬가지로 쿼리할 수 있습니다.

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Query your data**</DNT> 으)로 이동합니다.

2. <DNT>**Notebooks**</DNT> 옆의 토글을 활성화하고 <DNT>**+ SQL**</DNT> 클릭하여 SQL 블록을 추가하세요.

3. <DNT>materialized view</DNT> 을 참조할 때는 `mv.` 접두사 뒤에 뷰 이름을 붙이세요.

   ```sql
   SELECT * FROM mv.your_materialized_view_name
   ```

4. 쿼리를 실행합니다.

<DNT>materialized view</DNT> 에 추가 필터, 조인 또는 집계를 적용할 수 있습니다.

```sql
SELECT store_id, transaction_count
FROM mv.top_stores_mv
WHERE transaction_count > 1000
ORDER BY transaction_count DESC
```

<DNT>materialized view</DNT> 은 미리 계산된 결과를 포함하는 테이블 역할을 하므로 원래의 복잡한 쿼리를 다시 실행하지 않고도 추가 분석을 수행할 수 있습니다.

## 삭제 <DNT>materialized view</DNT> [#delete]

<DNT>materialized view</DNT> 을 삭제하려면 더 이상 다음이 필요하지 않습니다.

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT> 으)로 이동합니다.
2. 삭제하려는 <DNT>materialized view</DNT> 을 찾으세요.
3. 클릭하세요 <Icon name="fe-more-horizontal" /> 메뉴.
4. <DNT>**Delete**</DNT> 선택합니다.
5. 삭제를 확인하세요.

<Callout variant="caution">
  <DNT>materialized view</DNT> 을 삭제하면 영구적입니다. 삭제된 뷰를 참조하는 모든 쿼리 또는 대시보드는 실패합니다.
</Callout>

교차 데이터베이스 조인을 포함한 쿼리 예제는 [쿼리 구체화된 뷰를](/docs/new-relic-lens/query-data/#query-mv) 참조하세요.
