## <DNT>Data connectors</DNT> [#data-connectors]

<DNT>Data connectors</DNT> <DNT>Lens</DNT> 데이터베이스, 데이터 웨어하우스, 스프레드시트와 같은 외부 데이터 소스에 연결합니다. <DNT>connectors</DNT> 생성, 편집 또는 삭제하려면 [특정 RBAC 권한이](/docs/new-relic-lens/overview#permissions) 필요합니다.

### 접근 제어 [#access-control]

<DNT>Lens</DNT> <DNT>connectors</DNT> 쿼리할 수 있는 사람을 제어하는 두 가지 방법을 제공합니다.

* **RBAC 권한**: <DNT>**Lens connectors**</DNT> RBAC 권한을 가진 사용자는 조직의 모든 <DNT>connectors</DNT> 조회할 수 있습니다.
* **세부적인 접근 제어**: 광범위한 RBAC 권한이 없는 사용자는 <DNT>connectors</DNT> 적인 접근 권한이 부여된 경우 특정 <DNT>connectors</DNT> 계속 쿼리할 수 있습니다.

### 생성하세요 <DNT>connector</DNT> [#create-connector]

<DNT>data connector</DNT> 을 설정하려면:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Connectors**</DNT> 으)로 이동합니다.

2. <DNT>**Create a connector**</DNT> 을(를) 클릭합니다.

3. 드롭다운 메뉴에서 <DNT>connector</DNT> 유형을 선택하세요.

4. 선택한 <DNT>connector</DNT> 유형에 필요한 연결 정보를 입력하십시오.

5. RBAC 권한이 없는 사용자 또는 그룹에게 이 커넥터에 대한 액세스 권한을 부여하려면 <DNT>**Access control**</DNT> 섹션에서 세부적인 액세스 제어를 구성하십시오.

   1. 사용자 또는 그룹이 포함된 <DNT>**Auth domain**</DNT> 을 선택하세요.
   2. <DNT>**User or group**</DNT> 을 선택합니다.
   3. <DNT>**Lens Viewer**</DNT>역할을 선택하세요.
   4. 사용자 또는 그룹을 추가하려면 <DNT>**Add**</DNT> 클릭하고 위의 단계를 반복하십시오.

6. <DNT>**Create**</DNT> 을(를) 클릭합니다.

<DNT>connector</DNT> 을 생성한 후:

* 스키마를 보거나 <DNT>connector</DNT> 을 삭제하려면 다음을 선택하세요. <Icon name="fe-more-horizontal" /> <DNT>connector</DNT> 행의 메뉴입니다.
* 액세스 제어 설정을 업데이트하려면 <DNT>connector</DNT> 을 선택하고 액세스 권한을 추가 또는 제거한 다음 변경 사항을 저장하세요.
* <DNT>connector</DNT> 설정을 업데이트하려면 <DNT>connector</DNT> 을 삭제하고 새 정보로 다시 생성하세요.

    ### 승인하다 <DNT>Lens</DNT> [#grant-iceberg-access]

    <DNT>Lens</DNT> 이 Iceberg 테이블을 쿼리할 수 있도록 하려면 AWS 계정에 Glue 데이터 카탈로그 및 S3 버킷에 대한 읽기 액세스 권한을 부여하는 IAM 정책 및 역할을 생성하세요.

    <Callout variant="important">
      Glue 데이터 카탈로그와 Iceberg 데이터가 저장된 S3 버킷은 동일한 AWS 리전에 있어야 합니다.
    </Callout>

    <Steps>
      <Step>
        ### IAM 정책을 생성하세요 [#create-iceberg-policy]

        <DNT>Lens</DNT> 이 Glue 메타데이터와 S3의 데이터 파일을 읽을 수 있도록 다음과 같은 권한을 가진 [IAM 정책을 생성하십시오](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html). `{REGION}`, `{ACCOUNT_ID}`, `{DATABASE_NAME}`, `{S3_BUCKET_NAME}` 여러분의 값으로 바꾸세요. 입력한 정책 이름을 기록해 두십시오. 이 정책을 IAM 역할에 연결하려면 해당 이름이 필요합니다.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Action": [
                "glue:GetDatabase",
                "glue:GetTable",
                "glue:GetTables",
                "glue:SearchTables",
                "glue:GetPartition",
                "glue:GetPartitions",
                "glue:BatchGetPartition"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:catalog",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:database/{DATABASE_NAME}",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:table/*"
              ]
            },
            {
              "Action": [
                "s3:GetObject",
                "s3:ListBucket",
                "s3:GetBucketLocation"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:s3:::{S3_BUCKET_NAME}",
                "arn:aws:s3:::{S3_BUCKET_NAME}/*"
              ]
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### 신뢰 관계를 가진 IAM 역할을 생성합니다. [#create-iceberg-role]

        [IAM 역할을 생성하고](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) <DNT>Lens</DNT> 서비스 계정이 고유한 외부 ID를 사용하여 해당 역할을 수임할 수 있도록 신뢰 관계를 구성합니다. 역할을 설정할 때 다음 값을 사용하십시오.

        * **신뢰할 수 있는 엔티티 유형**: AWS 계정
        * **계정 ID**: `017663287629` (뉴렐릭의 AWS 계정)
        * **외부 ID 필수**: 활성화됨. 원하는 고유한 문자열을 입력하고 <DNT>Lens</DNT> <DNT>connector</DNT> 설정에 저장하세요.
        * **권한 정책**: 이전 단계에서 생성한 정책 이름을 검색하여 선택합니다.

        신뢰 정책은 다음 형식과 일치해야 합니다. `<YOUR_EXTERNAL_ID>` 입력한 외부 ID로 바꾸세요.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        역할을 생성한 후 역할 요약 페이지를 열고 **Role ARN** \[역할 ARN]을 복사합니다. <DNT>Lens</DNT> <DNT>connector</DNT> 을 구성할 때 이 ARN과 외부 ID가 필요합니다.
      </Step>

    ### 승인하다 <DNT>Lens</DNT> [#grant-cloudwatch-access]

    <DNT>Lens</DNT> 이 CloudWatch를 쿼리할 수 있도록 하려면 AWS 계정에 CloudWatch에 대한 읽기 액세스 권한을 부여하는 IAM 정책 및 역할을 생성하세요.

    <Steps>
      <Step>
        ### IAM 정책을 생성하세요 [#create-cloudwatch-policy]

        <DNT>Lens</DNT> 이 AWS 계정에서 CloudWatch를 읽을 수 있도록 다음 권한이 포함된 [IAM 정책을 생성하세요](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html). 입력한 정책 이름을 기록해 두십시오. 이 정책을 IAM 역할에 연결하려면 해당 이름이 필요합니다.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Action": [
                "cloudwatch:ListMetrics",
                "cloudwatch:GetMetricData",
                "cloudwatch:GetMetricStatistics",
                "tag:GetResources"
              ],
              "Resource": "*"
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### 신뢰 관계를 가진 IAM 역할을 생성합니다. [#create-cloudwatch-role]

        [IAM 역할을 생성하고](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) <DNT>Lens</DNT> 서비스 계정이 고유한 외부 ID를 사용하여 해당 역할을 수임할 수 있도록 신뢰 관계를 구성합니다. 역할을 설정할 때 다음 값을 사용하십시오.

        * **신뢰할 수 있는 엔티티 유형**: AWS 계정
        * **계정 ID**: `017663287629` (뉴렐릭의 AWS 계정)
        * **외부 ID 필수**: 활성화됨. 원하는 고유한 문자열을 입력하고 <DNT>Lens</DNT> <DNT>connector</DNT> 설정에 저장하세요.
        * **권한 정책**: 이전 단계에서 생성한 정책 이름을 검색하여 선택합니다.

        신뢰 정책은 다음 형식과 일치해야 합니다. `<YOUR_EXTERNAL_ID>` 입력한 외부 ID로 바꾸세요.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        역할을 생성한 후 역할 요약 페이지를 열고 **Role ARN** \[역할 ARN]을 복사합니다. <DNT>Lens</DNT> <DNT>connector</DNT> 을 구성할 때 이 ARN과 외부 ID가 필요합니다.
      </Step>
    </Steps>
  </Collapser>
