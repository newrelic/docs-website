    New Relic은 보안, 데이터 개인 정보 보호 및 규정 준수를 보장하기 위해 무엇을 합니까? 그리고 무엇을 할 수 있습니까?

    다음은 데이터의 기밀성, 무결성 및 가용성을 보장하는 데 사용할 수 있는 몇 가지 리소스입니다. 예를 들어, New Relic은 [귀하와 귀하의 고객에 대한 개인 정보가 저장 및 사용되는 방식을 제어](/docs/using-new-relic/new-relic-security/security/new-relic-personal-data-requests) 할 수 있도록 개인 정보 보호법 및 규정을 준수하는 프로세스를 개발했습니다.

    연결된 세상에서 정보 보안은 중요합니다. 정책, 자격 증명, 감사 및 기타 리소스에 대한 자세한 내용은 [회사 보안 정책](/docs/licenses/license-information/referenced-policies/security-policy) 및 [New Relic 보안 웹사이트](https://newrelic.com/security) 를 참조하십시오. 추가 질문이 있는 경우 New Relic 계정 담당자에게 문의하세요.
  </HeroContent>

    미국 및 EU의 [Tier III, SOC2 인증 데이터 센터](https://newrelic.com/security) 는 특정 [보존 기간 동안](/docs/using-new-relic/welcome-new-relic/getting-started/data-retention-components) 데이터 처리 및 저장 [규정 준수를 보장합니다](/docs/using-new-relic/new-relic-security/security/data-privacy-new-relic#legal) .우리는 특정 기준을 충족하는 계정에 대해 [FedRAMP Authorized Moderate](/docs/security/security-privacy/compliance/data-encryption) 입니다.HIPAA가 적용되고 New Relic을 사용하는 고객을 위해 [HIPAA 계정 활성화](/docs/security/security-privacy/compliance/hipaa-readiness-new-relic) 를 제공합니다.또한 Amazon Web Services(AWS)에서 호스팅되는 New Relic 플랫폼에 대해 [HITRUST 인증 상태](/docs/security/security-privacy/compliance/hitrust/) 를 받았습니다.
