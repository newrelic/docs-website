New Relic은 고객과 데이터의 보안을 위해 최선을 다하고 있습니다. 우리는 조정된 공개 프로그램을 통해 보안 연구원과 협력하는 것이 보안 목표를 달성하기 위한 중요한 수단이라고 믿습니다.

당사 제품 또는 웹사이트 중 하나에서 보안 취약점을 발견했다고 생각하는 경우 이 문서에 설명된 대로 조정 공개 프로그램을 통해 신고해 주셔서 감사합니다. 다른 문제가 있는 경우 [다른 이메일 또는 계정 문제](#customer-issues) 에 대한 정보를 참조하세요.

## 조정된 공개 프로그램 [#coordinated\_disclosure\_program][#coordinated_disclosure_program]

뉴렐릭은 연구원들이 보안 취약점을 최대한 쉽게 보고할 수 있도록 기존 버그 바운티 플랫폼과 협력하고 있습니다. 이러한 문제를 찾는 데 드는 노력을 인정하여 적격한 신고에 포상금을 제공할 수 있습니다.

뉴렐릭은 공식적으로 조정된 공개 프로그램을 BugCrowd로 전환했습니다.

조정 공개 프로그램에 참여하려면:

1. 보안 테스트를 시작하기 전에 Bugcrowd Researcher 이용 약관에 동의하고 [Bugcrowd 정책을](https://bugcrowd.com/engagements/newrelic-mbb-og-public) 숙지하고 준수해야 합니다.
2. 허용된 테스트 활동에만 사용할 수 있는 뉴렐릭 계정을 만들고 해당 [서비스 약관을](https://newrelic.com/termsandconditions/terms) 준수해야 합니다.

## 고객 보안 문제 [#customer-issues]

New Relic 고객이고 암호 또는 계정 문제가 있는 경우 조정 공개 프로그램을 사용하지 마십시오. 대신 [비밀번호, 이메일 및 로그인 문제에](/docs/accounts/accounts-billing/account-setup/troubleshoot-new-relics-password-email-address-login-problems) 대한 표준 문제 해결 절차를 따르십시오.
