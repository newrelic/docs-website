## 요약

Windows 인프라 에이전트용 보안 업데이트는 하드 코딩된 파일 경로로 인해 악의적인 사용자가 에이전트 구성을 무시할 수 있는 문제를 수정합니다.

**출시일:** 2018년 6월 18일

**취약점 식별자:** NR18-10

**우선 순위:** 높음

## 취약점 정보 [#vuln-info]

New Relic Windows Infrastructure 에이전트는 특정 위치에서 구성 파일을 찾습니다. 악의적인 사용자가 구성 파일을 무시할 수 있는 Unix 전용 파일 경로가 Windows 에이전트에 포함되었습니다. 이 보안 업데이트를 통해 Windows 인프라 에이전트는 더 이상 Unix 특정 위치를 찾지 않습니다.

### 완화 요인 [#factors]

* 이 취약점은 Windows 시스템에만 존재합니다.
* 이 취약점은 시스템에 대한 기존 액세스가 필요합니다.

## 해결 방법

해당 없음

## New Relic에 보안 취약점 보고 [#report]

New Relic은 고객과 고객 데이터의 보안을 위해 최선을 다하고 있습니다. 당사 제품이나 웹사이트 중 하나에서 보안 취약점을 발견했다고 생각되시면 New Relic의 통합 공개 프로그램에 신고해 주셔서 감사합니다. 자세한 내용은 [보안 취약점 보고](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities) 를 참조하십시오.
