## 요약

.NET 에이전트용 보안 업데이트는 Microsoft SQL Server를 계측할 때 특정 문자 조합이 쿼리 난독화를 방지할 수 있는 문제를 수정합니다.

**출시일:** 2019년 4월 22일

**취약점 식별자:** NR19-03

**우선순위:** 중간

## 취약점 정보 [#vuln-info]

SQL 쿼리를 난독화하도록 구성된 경우 .NET 에이전트는 난독화 기능을 통해 모든 쿼리를 전달합니다. Microsoft SQL Server가 쿼리를 삭제하는 방식으로 인해 이 난독화에 실패할 수 있는 특정 문자 조합이 생성될 수 있습니다.

### 완화 요인 [#factors]

이 취약점은 Microsoft SQL Server를 계측하고 쿼리 난독화를 활성화한 경우에만 존재합니다.

## 해결 방법

* 데이터베이스 계측을 비활성화합니다.
* [최신 New Relic .NET 에이전트로 업데이트하십시오.](/docs/agents/net-agent/installation/update-net-agent)

## New Relic에 보안 취약점 보고 [#report]

New Relic은 고객과 고객 데이터의 보안을 위해 최선을 다하고 있습니다. 당사 제품이나 웹사이트 중 하나에서 보안 취약점을 발견했다고 생각되시면 New Relic의 통합 공개 프로그램에 신고해 주셔서 감사합니다. 자세한 내용은 [보안 취약점 보고](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities) 를 참조하십시오.
