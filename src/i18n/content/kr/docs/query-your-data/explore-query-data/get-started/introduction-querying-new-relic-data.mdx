[New Relic](http://one.newrelic.com) 은 전체 시스템의 모든 데이터에 액세스할 수 있는 강력한 관찰 가능성 플랫폼입니다. 우리는 기본적으로 애플리케이션, 서비스, 호스트 등을 포함하여 모니터링할 수 있는 데이터가 있는 식별할 수 있는 모든 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/) 에서 보낸 원격 측정 데이터를 호스팅합니다. 이름을 지정하세요!

선별된 [대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 를 통해 데이터를 볼 수 있는 즉시 사용 가능한 경험을 제공하는 동안 [UI](#query-ui) 또는 [API를](#query-apis) 포함하여 여러 가지 방법으로 데이터에 대한 액세스를 조정하고 가시성을 사용자 정의할 수 있습니다.

<Callout variant="important">
  New Relic에 저장된 데이터를 더 잘 이해하려면 [데이터 유형](/docs/using-new-relic/data/understand-data/new-relic-data-types) 을 참조하십시오.
</Callout>

## UI에서 데이터 찾아보기 [#browse-data]

[우리 플랫폼](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one) 은 NRQL이나 쿼리 언어에 대한 지식이 필요하지 않은 여러 경험을 제공합니다. UI에서 **데이터 쿼리** 버튼으로 이동하거나 데이터 **찾아보기** 드롭다운을 클릭한 다음 탐색할 데이터 유형(메트릭, 이벤트, 로그 및 추적)을 선택합니다.

* [이벤트](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 및 [메트릭](/docs/using-new-relic/data/understand-data/new-relic-data-types#metrics) 의 경우 직관적인 데이터 [탐색기](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) 인 데이터 탐색기를 사용하여 시각화를 만듭니다. 탐색기에서 [쿼리 작성기](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) 로 전환하여 쿼리를 보고 구체화할 수 있습니다.
* [분산 추적 쿼리](/docs/understand-dependencies/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one) : 추적 쿼리를 위한 특수 UI입니다.
* [로그 쿼리](/docs/logs/new-relic-logs/ui-data/query-syntax-logs) : New Relic에서 [로그 관리 데이터](/docs/logs/new-relic-logs/get-started/introduction-new-relic-logs) 를 쿼리하기 위한 특수 UI입니다.

## UI의 쿼리 데이터 [#query-ui]

데이터 검색 이상의 작업을 수행할 준비가 되었다면 전면적인 배우가 되어 New Relic UI에서 쿼리를 개인화하십시오. [New Relic 쿼리 언어](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/) 또는 [PromQL 스타일 쿼리 언어](https://prometheus.io/docs/prometheus/latest/querying/basics/) 를 포함한 쿼리 언어를 사용하여 완전한 유연성으로 쿼리를 편집합니다. 예를 들어, 더 많은 [`WHERE` 절](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/#sel-where) 을 추가하고, 반환된 값을 수정하고, [다른 유형의 시각화](/docs/query-your-data/explore-query-data/use-charts/chart-types/) 로 변경할 수 있습니다.

<Callout variant="tip">
  언어 쿼리가 처음이신가요? [데이터 탐색기](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) 에서 데이터 탐색을 시작한 다음 [쿼리 작성기](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) 로 전환하여 작성한 쿼리를 확인하고 구체화합니다.
</Callout>

자체 쿼리를 작성하여 데이터를 검색하고 차트를 작성하는 두 가지 방법이 있습니다.

* [NRQL 모드의 쿼리 빌더](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) : NRQL( [New Relic 쿼리 언어](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) )을 사용한 쿼리, 대부분의 UI 경험을 구축하는 데 사용하는 것과 동일한 언어, New Relic에서 데이터를 쿼리하는 가장 고급 방법.
* [PromQL 스타일 모드의 쿼리 빌더](/docs/query-your-data/explore-query-data/chart-builder/use-advanced-promql-mode-specify-data) : PromQL 스타일 쿼리를 사용하여 기본 쿼리를 작성합니다.

## API를 통한 쿼리 데이터 [#query-apis]

New Relic 플랫폼에 들어가는 것이 옵션이 아닌 경우 API를 사용하여 New Relic에서 데이터를 검색하고 쿼리할 수 있습니다. 예를 들어 [NerdGraph](/docs/apis/graphql-api/get-started/introduction-new-relic-nerdgraph) (GraphQL API)를 사용하여 [NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) (쿼리 언어) 쿼리를 실행할 수 있습니다.

자세한 내용 [은 New Relic API 소개를](/docs/apis/intro-apis/introduction-new-relic-apis/) 참조하세요.
