[대시보드를](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 사용하여 시스템 상태를 시각화 및 추적하고, 문제를 해결하고, 추가 단계를 계획할 수 있습니다. 다양한 가져오기 및 내보내기 옵션을 사용하여 팀, 고객 또는 기타 이해관계자와 모든 정보를 공유하는 방법을 알아보세요.

## 대시보드를 JSON으로 가져오기 [#import-json]

코드 레벨에서 기본 대시보드를 수정하려는 경우 대시보드를 JSON으로 가져올 수 있습니다. 예를 들어, 두 개의 기존 대시보드를 병합하여 맞춤형 대시보드를 생성하려는 경우 기본 대시보드의 JSON을 수정한 다음 이를 다시 뉴렐릭으로 가져올 수 있습니다.

대시보드를 JSON으로 가져오려면 다음 안내를 따르세요.

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT> 으)로 이동합니다.

2. <DNT>**...**</DNT> 아이콘을 클릭한 다음 <DNT>**Copy JSON**</DNT> 선택하여 기존 대시보드의 JSON을 엽니다.

3. <DNT>**Dashboards**</DNT> 페이지로 돌아갑니다.

4. 오른쪽 상단에서 <DNT>**Import dashboard**</DNT>를 클릭합니다.

5. JSON 코드를 붙여넣으세요.

6. 대시보드에 대한 계정 및 권한 설정을 선택합니다. 계정을 설정한 후에는 변경할 수 없지만 [권한은 언제든지 변경할](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-settings) 수 있습니다.

7. <DNT>**Save**</DNT> 을(를) 클릭합니다.

## 대시보드를 PDF로 내보내기 [#export-dashboards-pdf]

대시보드를 PDF 파일로 내보내려면 대시보드 오른쪽 상단에서 <DNT>**...**</DNT> 아이콘을 클릭하고 <DNT>**Export dashboard as PDF**</DNT> 선택합니다.

UI를 통해 대시보드를 PDF로 내보낼 때 다음 사항을 고려하십시오:

* [사용자 정의 시각화는](/docs/query-your-data/explore-query-data/dashboards/add-custom-visualizations-your-dashboards/) 지원되지 않습니다.

사용자 지정 크기 및 템플릿 변수 재정의와 같은 추가 옵션을 사용하여 PDF 또는 PNG로 내보내려면 [NerdGraph API](#export-api)를 사용하십시오.

## NerdGraph API를 통해 대시보드 및 차트 내보내기 [#export-api]

NerdGraph API를 사용하여 대시보드 및 개별 차트 위젯을 스냅샷 이미지로 프로그래밍 방식으로 내보냅니다. API는 사용자 지정 차원, 템플릿 변수 재정의, 엔티티 필터 등 UI에서 사용할 수 없는 옵션을 지원합니다.

### 대시보드를 PDF 또는 PNG로 내보내기 [#api-dashboard-snapshot]

`dashboardCreateSnapshotUrl` 뮤테이션은 전체 대시보드 페이지에 대한 스냅샷 URL을 생성합니다.

<Callout variant="tip">
  `guid` 파라미터에는 대시보드 GUID가 아닌 **페이지** GUID가 필요합니다. 대시보드에는 여러 페이지가 있을 수 있습니다. 페이지 GUID를 찾으려면 `entitySearch` 쿼리를 사용하거나 대시보드 페이지를 열고 <DNT>**...**</DNT> 메뉴에서 <DNT>**Manage JSON**</DNT> 을(를) 선택합니다.

  ```graphql
  {
    actor {
      entitySearch(query: "parentId ='YOUR_DASHBOARD_GUID' AND tags.isDashboardPage = 'true'") {
        results {
          entities {
            guid
            name
          }
        }
      }
    }
  }
  ```
</Callout>

**뮤테이션 스키마:**

```graphql
mutation {
  dashboardCreateSnapshotUrl(
    guid: EntityGuid!
    params: SnapshotUrlInput
  ): String
}

input SnapshotUrlInput {
  timeWindow: SnapshotUrlTimeWindowInput
  display: SnapshotUrlDisplayInput
  variables: [SnapshotUrlVariableInput!]
  filter: String
  format: SnapshotUrlFormat
}

input SnapshotUrlTimeWindowInput {
  beginTime: EpochMilliseconds
  endTime: EpochMilliseconds
  duration: Milliseconds
}

input SnapshotUrlDisplayInput {
  width: Int
  height: Int
}

input SnapshotUrlVariableInput {
  name: String!
  values: [String!]!
}

enum SnapshotUrlFormat {
  PDF
  PNG
}
```

### 차트 이미지 가져오기 [#api-widget-snapshot]

`dashboardWidgetCreateSnapshotUrl` 뮤테이션은 단일 차트 위젯에 대한 PNG 스냅샷 URL을 생성합니다. 위젯이 대시보드에 존재할 필요는 없습니다 ― 인라인으로 정의할 수 있으므로 프로그래밍 방식으로 차트 이미지를 생성하는 데 유용합니다.

**뮤테이션 스키마:**

```graphql
mutation {
  dashboardWidgetCreateSnapshotUrl(
    widget: DeclarativeUiWidget!
  ): WidgetSnapshotResult
}

type WidgetSnapshotResult {
  url: String!
}
```

`widget` 파라미터는 `DeclarativeUiWidget` 을(를) 허용합니다 — 세 개의 중첩된 계층이 있는 JSON 객체:

```
declarative/widget envelope
└── widget node        ← title and widget-level props
    └── visualization  ← visualization type and configuration
```

```json
{
  "type": "declarative/widget",
  "version": 1,
  "content": {
    "type": "widget",
    "props": {
      "title": "My Chart Title"
    },
    "content": {
      "type": "visualization",
      "id": "viz.line",
      "props": {
        // visualization-specific config (nrqlQueries, colors, thresholds, etc.)
      }
    }
  }
}
```

JSON으로 내보낸 대시보드에서 위젯 정의를 빌드하는 경우 다음 필드 매핑을 사용하십시오:

<Callout variant="tip">
  NerdGraph의 구형 `actor.account.nrql.staticChartUrl` 필드는 계속 사용할 수 있지만 임계값, 색상 형식 지정 및 최신 차트 유형과 같은 최신 시각화 기능을 지원하지 않습니다. 앞으로는 `dashboardWidgetCreateSnapshotUrl` 방식을 권장합니다.
</Callout>

<Callout variant="important">
  `dashboardWidgetCreateSnapshotUrl` 에서 생성된 스냅샷 URL은 PNG 이미지이며, 인증 없이 공개적으로 액세스할 수 있고 3개월 후에 만료됩니다.
</Callout>

## 차트를 CSV로 내보내기 [#export-charts-csv]

대시보드의 표 차트를 CSV 파일로 내보내려면 차트 오른쪽 상단에서 <DNT>**...**</DNT> 클릭하고 <DNT>**Export as CSV**</DNT> 선택합니다.

<img title="Export chart as CSV" alt="Screenshot of how to export a table chart as a CSV file" src="/images/dashboards_screenshot-crop_export-csv.webp" />

차트를 CSV로 내보낼 때 다음 사항을 고려하세요.

* 데이터는 항상 UTC 시간으로 내보내집니다.
* 내보내기는 다음과 같은 집계 기능을 지원하지 않습니다. `average()`
* 내보내기는 `COMPARE_WITH` 절을 지원하지 않습니다.
* 일부 시나리오에서는 UI가 내보낸 CSV에 복제되지 않은 데이터의 추가 형식을 수행할 수 있으며 이로 인해 약간의 불일치가 발생할 수 있습니다(예: -1.1E-17이 UI에서는 0으로 렌더링되지만 CSV에서는 렌더링되지 않음).
* 교차 계정 쿼리의 결과를 내보낼 수 없습니다.
