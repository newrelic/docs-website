[New Relic 대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) 에 액세스하여 차트 메뉴에서 직접 차트를 생성 또는 관리하고, 대시보드 레이아웃을 사용자 지정하고, 표시 모드를 조정하거나, 데이터를 내보낼 수 있습니다.

대시보드를 사용자 정의하고 차트를 작성했으면 데이터 탐색을 위한 고급 시각화 기능 및 도구를 사용하여 데이터를 연관시키고 분석하십시오.

## 대시보드 사용자 지정 [#dash-manage]

대시보드 페이지를 열려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards**</DNT> 으로 이동하세요.

<img
  title="Dashboards index"
  alt="Dashboards index"
  src="/images/dashboards_screenshot-full_dashboard-index.webp;"
/>

<figcaption>
  대시보드 색인을 열려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards** </DNT>으로 이동하세요.
</figcaption>

대시보드를 클릭하여 차트를 엽니다. 대시보드 페이지에는 다음 기능이 포함되어 있습니다.

<CollapserGroup>
  <Collapser
    id="dash-clone"
    title="대시보드 복제"
  >
    권한 수준에 관계없이 대시보드를 복제할 수 있습니다. 다음을 클릭하세요. <Icon name="fe-more-horizontal"/> 오른쪽 모서리에 있는 아이콘을 클릭하고 <DNT>**Duplicate dashboard**</DNT> 선택합니다.

    복제된 대시보드의 이름은 원본 대시보드와 동일하게 이름 뒤에 "복사"라는 단어가 붙습니다. 예를 들어 `This is my dashboard` 라는 대시보드를 복제하면 복제본은 `This is my dashboard copy` 이라고 합니다. 이름을 변경하고, 연결된 계정을 선택하고, 복제할 때 권한을 추가할 수 있습니다. <DNT>**Create**</DNT> 클릭하면 복제된 대시보드에 액세스할 수 있는 팝업 메시지가 화면에 표시됩니다. 언제든지 권한과 같은 대시보드의 이름과 기타 속성을 편집할 수 있습니다.

    대시보드가 자동으로 복사되고 중복 항목이 [인덱스](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#get-started) 에 추가됩니다. 복제본에는 [<DNT>**Public - Read and write**</DNT> 권한이](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-permissions) 있습니다.
  </Collapser>

  <Collapser
    id="dash-edit"
    title="대시보드 수정"
  >
    [권한이](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-permissions) 있는 경우 대시보드를 편집할 수 있습니다.

    편집 사용 <Icon name="fe-edit-2"/>버튼:

    * 대시보드의 퍼머링크를 복사합니다.

    * 대시보드의 이름을 바꿉니다. [이름은 검색 가능](#dashboards-search)하므로 대시보드를 쉽게 찾는 데 도움이 되는 단어를 사용하여 의미 있는 이름을 지정하는 것이 좋습니다.

    * <DNT>
        **+ Add widget**
      </DNT>

      버튼을 클릭하여 새 콘텐츠를 만듭니다. [쿼리 빌더를](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/) 사용하여 새 차트를 추가하거나 [마크다운 편집기를](#markdown) 사용하여 텍스트, 이미지 또는 링크를 추가하세요.

    * [차트](/docs/query-your-data/explore-query-data/use-charts/use-your-charts/)크기 조정 및 재정렬 . 차트를 이동하고 대시보드의 아무 곳에나 배치하여 설정한 레이아웃이 기본 설정에 맞도록 할 수 있습니다. 관련성이 더 높은 차트를 맨 위에 놓거나 덜 사용하는 차트를 모서리에 놓습니다. 최대 12개의 차트 열을 설정할 수 있습니다.
  </Collapser>

    PDF 또는 이미지로 내보내면 차트 유형에 따라 표시가 변경됩니다.

    * <DNT>**Show time picker**</DNT>. 표시되는 데이터의 [시간 범위를](#dash-time-picker) 변경하려면 이 옵션을 사용합니다.

    * <DNT>**Cycle to the next page every 20 seconds**</DNT>. 대시보드에 [여러 페이지가](#add-pages) 포함된 경우 이 옵션이 표시됩니다.

## 대시보드에 새 콘텐츠 추가 [#dash-create-chart]

대시보드에 새 콘텐츠를 추가하는 몇 가지 방법은 다음과 같습니다.

* 편집을 클릭하세요

  <Icon name="fe-edit-2"/>

  버튼을 클릭한 다음

  <DNT>
    **Add widget**
  </DNT>

  버튼을 클릭하여 새 콘텐츠를 만듭니다.

* [메트릭 및 이벤트](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) 와 [쿼리 빌더](/docs/introduction-chart-builder) 에서 .

* [쿼리 빌더](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 에 액세스하거나 [마크다운 편집기를](#dash-create-chart) 사용하여 콘텐츠(예: 텍스트, 링크, 이미지)를 추가하려면 기본 대시보드 페이지 또는 편집 모드에서 액세스할 수 있는

  <DNT>
    **+ Add widget**
  </DNT>

  버튼을 사용하세요.

* 대시보드에서 [기존 차트를 복사합니다](#dash-manage-charts) .

새 콘텐츠를 추가하는 데 문제가 있는 경우 [오류 메시지를 확인하세요](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/#chart-error-messages) .

### Markdown 편집기를 사용하여 사용자 정의 콘텐츠 추가 [#markdown]

Markdown 편집기를 사용하여 사용자 지정 콘텐츠로 대시보드에 컨텍스트와 감각을 추가하세요. 이 기능을 사용하면 서식이 지정된 텍스트, 체크리스트, 이미지, 코드 스니펫 등을 추가할 수 있습니다. 이를 통해 공유 대시보드를 쉽게 설명하고, 유용한 하이퍼링크를 추가하고, 작업 목록을 만들고, 필요한 다른 풍부한 콘텐츠를 만들 수 있습니다.

마크다운 편집기에는 콘텐츠를 입력하는 <DNT>**Markdown**</DNT> 창과 콘텐츠를 볼 수 있는 <DNT>**Preview**</DNT> 창이 포함되어 있습니다. Markdown 구문 옵션에 대한 자세한 내용은 [GitHub의 Markdown 가이드를](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax) 참조하세요.

줄임표를 클릭하여 기존 콘텐츠를 편집할 수도 있습니다. <Icon name="fe-more-horizontal"/> Markdown 위젯에서 아이콘을 클릭하고 <DNT>**Edit**</DNT> 선택합니다.

## 페이지로 대시보드 구성 [#add-pages]

여러 페이지를 사용하여 대시보드 데이터를 다양한 보기로 구성할 수 있습니다. 해당 대시보드에 페이지를 더 추가하면 대시보드 UI 상단에 있는 탭을 사용하여 이러한 페이지에 액세스할 수 있습니다.

<img
  title="Dashboard with multiple pages"
  alt="Dashboard with multiple pages"
  src="/images/dashboards_screenshot-full_multi-page-dashboard.webp;"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards**</DNT>: 대시보드 상단의 탭으로 표시되는 여러 페이지로 구성된 대시보드의 예입니다. 대시보드에 페이지를 추가하고, 기존 페이지를 복사하고, 페이지 탭을 새 위치로 끌어서 놓을 수 있습니다.
</figcaption>

이 기능을 사용하여 관련 대시보드 보기를 그룹화할 수 있습니다. 이는 특정 프로젝트, 팀 또는 주제와 관련된 많은 데이터 및 차트를 집계할 때 유용합니다.

예를 들어 모바일 앱 팀은 국가별 앱 성능에 중점을 둔 대시보드를 구축할 수 있습니다. 첫 번째 대시보드 페이지는 특정 국가에 초점을 맞춘 다른 페이지와 함께 모든 국가의 성과에 대한 개요일 수 있습니다.

대시보드를 연결하는 다른 기능이 있습니다.

* [Markdown 텍스트가 포함된 위젯](#markdown) 을 만들어 특정 UI 페이지 또는 대시보드에 대한 직접 링크를 추가합니다.
* [패싯 필터링](/docs/dashboards/new-relic-one-dashboards/manage-your-dashboard/filter-new-relic-one-dashboards-facets) 을 사용하여 다른 대시보드에 자동으로 연결하고 필터링하는 링크를 생성합니다.
* 대시보드 검색을 사용하여 유사한 이름의 대시보드를 찾습니다. 이를 활용하기 위해 대시보드 이름에 팀 또는 프로젝트별 단어/구를 추가할 수 있습니다.

### 대시보드에 페이지 추가 및 편집 [#add-pages-edit]

대시보드에서 페이지를 추가하거나 편집하려면:

1. 신규 또는 기존 대시보드에서 다음을 선택하여 편집 모드로 들어갑니다.

   <Icon name="fe-edit-2"/>

   상. 대시보드의 권한에 따라 이 아이콘이 표시되거나 표시되지 않습니다.

2. 새 페이지 추가:

   * 빈 페이지를 추가하려면

     <DNT>
       **Add a page**
     </DNT>

     선택합니다.

   * 페이지 이름 옆에 있는 드롭다운 메뉴를 클릭하고

     <DNT>
       **Duplicate Page**
     </DNT>

     선택하여 기존 페이지를 복제합니다.

3. 편집 모드에서 새 페이지에 위젯을 추가하고, 페이지 탭을 새 위치로 끌어다 놓고, 기타 [대시보드 편집 작업을](#dash-manage-charts)수행할 수 있습니다.

4. 완료되면

   <DNT>
     **Done editing**
   </DNT>

   선택합니다.

## 차트 및 마크다운 콘텐츠 관리 [#dash-manage-charts]

* 차트를 전체 화면으로 확장합니다.
* [차트를 이미지나 링크로 공유하세요](#dash-export) .
* 테이블 차트의 경우에만 `.csv` 파일로 내보냅니다. 이 파일을 Microsoft Excel 또는 Google 스프레드시트와 같은 다른 앱으로 가져와 추가 분석을 수행할 수 있습니다.

* 차트를 복제합니다.
* 차트를 삭제합니다.

## 차트 필터링 및 구체화 [#dash-search]

쿼리 조건을 시각적으로 나타내는 필터링 기능을 사용하여 표시되는 정보의 범위를 좁힐 수 있습니다.

* <DNT>
    **filter bar**
  </DNT>

  를 사용하여 보려는 값이나 속성을 선택하고 차트에서 나머지 요소를 제거합니다.

* <DNT>
    **advanced filter bar**
  </DNT>

  를 열어 부울 연산자(예: `=`, `!=`, `CONTAINS`, `EXCLUDES` 등)에 액세스하고 데이터 필터링을 위한 복합 및 복잡한 조건을 추가합니다.

필터를 적용하면 대시보드에는 선택한 요소와 관련된 데이터만 표시됩니다. 작은 카운터는 한 번에 적용되는 필터 수를 나타냅니다.

기본 보기로 돌아가려면 필터 옆에 있는 작은 십자가를 클릭하여 제거합니다.

## 차트 범례를 사용하여 필터링 [#filter-legend]

범례가 있는 차트에서 범례를 클릭하면 해당 계열만 표시되고 나머지 계열은 차트에서 제거됩니다. 이렇게 하면 분석할 데이터를 분리하는 데 도움이 됩니다.

반대로 하려면 CMD(Mac의 경우) 또는 CTRL(Windows의 경우)을 사용합니다. 이렇게 하면 선택한 시리즈가 제거되고 나머지는 유지됩니다.

## 패싯을 사용하여 대시보드 필터링 [#filter-facets]

차트의 NRQL 쿼리에 `FACET` 절이 포함된 경우 패싯 속성을 사용하여 현재 대시보드 또는 다른 관련 대시보드를 필터링할 수 있습니다. 자세한 내용 [은 패싯으로 필터링을](/docs/dashboards/new-relic-one-dashboards/manage-your-dashboard/filter-new-relic-one-dashboards-facets) 참조하십시오.

## 시간 선택기를 사용하여 시간 설정 조정 [#dash-time-picker]

기본적으로 대시보드의 각 차트는 [쿼리 빌더](/docs/introduction-chart-builder)에서 생성될 때 지정된 기간 동안의 데이터를 표시합니다. 그러나 [시간 선택기를](/docs/new-relic-solutions/get-started/glossary/#time-picker) 사용하여 표시되는 데이터의 시간 범위를 변경하고 모든 차트에 대해 동일한 범위를 설정할 수 있습니다. 특정 기간에 발생한 일을 관찰하기 위해 데이터 범위를 좁혀야 하는 경우 인시던트 문제를 해결하는 동안 특히 유용합니다.

<img
  width="50%;"
  title="Time picker on the Dashboard page"
  alt="Time picker on the Dashboard page"
  src="/images/dashboards_screenshot-crop_time-picker.webp;"
/>

새로 고침 빈도는 보고 있는 기간의 지속 시간에 따라 다릅니다. 자세한 내용과 예는 [차트 새로 고침 간격](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-refresh-intervals) 을 참조하십시오.

시간 범위를 변경하려면:

* 드롭다운 메뉴에서 사용 가능한 옵션 중 하나를 선택합니다(마지막 `5 minutes` 에서 마지막 `6 months` 까지).
* 사용자 지정 메뉴를 사용하여 특정 시작 및 종료 타임스탬프로 시간 범위를 사용자 지정합니다.

시간 선택기를 사용할 때 차트의 시간 범위를 변경하지 않으려면 다음을 수행하십시오.

1. 다음을 클릭하세요.

   <Icon name="fe-more-horizontal"/>

   아이콘을 클릭하고

   <DNT>
     **Edit**
   </DNT>

   선택합니다.

2. 맞춤설정 UI에서

   <DNT>
     **Dashboard Options**
   </DNT>

   메뉴를 엽니다.

3. 필요에 따라

   <DNT>
     **Ignore time picker**
   </DNT>

   토글을 켜거나 끄도록 설정합니다.

## 데이터 공유 [#data-share]

지원되는 시각화에서 <DNT>**Get chart link**</DNT> 옵션을 사용하여 라이브 차트 URL을 생성할 수 있는 기능을 가진 조직의 모든 사용자에게 익숙할 것입니다. 이제 잠재적인 데이터 노출을 방지하기 위해 관리자는 라이브 차트 URL을 생성할 수 있는 사람과 생성할 수 없는 사람을 제어할 수 있습니다. 이 권한은 이제 아래 설명된 <DNT>**Live Chart URL (individual)**</DNT> 및 <DNT>**Live Chart URL (all)**</DNT> 기능 모두에 의해 제어됩니다. 이를 사용하면 개별 사용자가 공유 가능한 URL을 생성할 수 있도록 계정에 권한을 할당할 수 있습니다.

중단을 최소화하기 위해 이 기능을 활성화하려는 고객은 2024년 4월 30일 이전에 맞춤 역할을 만들어 사용자에게 할당할 수 있습니다. 2024년 4월 30일 이후에 이 기능이 정식 출시되고 해당 기능이 활성화됩니다.

<Callout variant="important">
  라이브 차트 URL을 생성하는 기능은 2024년 4월 30일에 <DNT>HIPAA</DNT> 및 <DNT>FedRAMP</DNT> 계정에서 제거됩니다.
</Callout>

### 라이브 차트 URL(개별) [#live-chart-individual]

이 기능을 사용하여 사용자 작업 범위를 사용자가 생성하는 실제 URL로 제한합니다.

* 보기: 사용자는 자신이 생성한 라이브 차트 URL을 볼 수 있습니다.
* 수정: 사용자는 라이브 차트 URL을 생성할 수 있습니다.
* 삭제: 사용자는 자신이 생성한 라이브 차트 URL을 삭제할 수 있습니다.

### 라이브 차트 URL(전체) [#live-chart-all]

이 기능을 사용하면 계정의 모든 사용자가 생성한 실제 URL에 대한 사용자 작업 범위를 늘릴 수 있습니다.

* 보기: 사용자는 계정의 모든 사용자가 생성한 모든 라이브 차트 URL을 볼 수 있습니다.
* 삭제: 사용자는 계정의 모든 사용자가 생성한 라이브 차트 URL을 삭제할 수 있습니다.

새로운 기능은 [라이브 차트 URL API](/docs/apis/nerdgraph/examples/manage-live-chart-urls-via-api) 에도 적용됩니다.
