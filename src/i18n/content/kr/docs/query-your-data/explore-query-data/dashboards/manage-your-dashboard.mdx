[New Relic 대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) 에 액세스하여 차트 메뉴에서 직접 차트를 생성 또는 관리하고, 대시보드 레이아웃을 사용자 지정하고, 표시 모드를 조정하거나, 데이터를 내보낼 수 있습니다.

대시보드를 사용자 정의하고 차트를 작성했으면 데이터 탐색을 위한 고급 시각화 기능 및 도구를 사용하여 데이터를 연관시키고 분석하십시오.

## 대시보드 사용자 지정 [#dash-manage]

**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards** 로 이동하여 대시보드 페이지를 엽니다.

</figcaption>

대시보드를 클릭하여 차트를 엽니다. 대시보드 페이지에는 다음 기능이 포함되어 있습니다.

    복제된 대시보드는 원래 대시보드 뒤에 "복사"라는 단어가 붙는 것과 같이 이름이 지정됩니다. 예를 들어 이름이 `This is my dashboard` 인 대시보드를 복제하면 복제본의 이름은 `This is my dashboard copy` 입니다. 이름을 변경하고 연결된 계정을 선택하고 복제할 때 권한을 추가할 수 있습니다. **만들기 를**클릭하면 복제된 대시보드에 액세스하라는 팝업 메시지가 화면에 표시됩니다. 언제든지 권한과 같은 대시보드의 이름 및 기타 속성을 편집할 수 있습니다.

## 대시보드에 새 콘텐츠 추가 [#dash-create-chart]

대시보드에 새 콘텐츠를 추가하는 몇 가지 방법은 다음과 같습니다.

* [메트릭 및 이벤트](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) 와 [쿼리 빌더](/docs/introduction-chart-builder) 에서 .
* **+ 대시보드에 추가** 버튼(기본 대시보드 페이지 또는 편집 모드에서 액세스 가능)을 사용하여 [쿼리 작성기](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 에 액세스하거나 [Markdown 편집기](#dash-create-chart) 를 사용하여 콘텐츠(예: 텍스트, 링크 또는 이미지)를 추가합니다.
* 대시보드에서 [기존 차트를 복사합니다](#dash-manage-charts) .

새 콘텐츠를 추가하는 데 문제가 있는 경우 [오류 메시지를 확인하세요](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/#chart-error-messages) .

### Markdown 편집기를 사용하여 사용자 정의 콘텐츠 추가 [#markdown]

Markdown 편집기를 사용하여 사용자 지정 콘텐츠로 대시보드에 컨텍스트와 감각을 추가하세요. 이 기능을 사용하면 서식이 지정된 텍스트, 체크리스트, 이미지, 코드 스니펫 등을 추가할 수 있습니다. 이를 통해 공유 대시보드를 쉽게 설명하고, 유용한 하이퍼링크를 추가하고, 작업 목록을 만들고, 필요한 다른 풍부한 콘텐츠를 만들 수 있습니다.

마크다운 편집기에는 콘텐츠를 입력할 수 있는 **마크다운** 창과 콘텐츠를 볼 수 있는 **미리보기** 창이 있습니다. Markdown 구문 옵션에 대한 자세한 내용은 [GitHub의 Markdown 가이드](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax) 를 참조하십시오.

줄임표를 클릭하여 기존 콘텐츠를 편집할 수도 있습니다. <Icon name="fe-more-horizontal"/>Markdown 위젯의 아이콘을 클릭하고 **편집** 을 선택합니다.

## 페이지로 대시보드 구성 [#add-pages]

여러 페이지를 사용하여 대시보드 데이터를 다양한 보기로 구성할 수 있습니다. 해당 대시보드에 페이지를 더 추가하면 대시보드 UI 상단에 있는 탭을 사용하여 이러한 페이지에 액세스할 수 있습니다.

<img
  title="Dashboard with multiple pages"
  alt="Dashboard with multiple pages"
  src={dashboardsMultiPageDashboard}
/>

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards**: 이것은 대시보드 상단의 탭으로 표시되는 여러 페이지가 있는 대시보드의 예입니다. 대시보드에 페이지를 추가하고, 기존 페이지를 복사하고, 페이지 탭을 새 위치로 끌어다 놓을 수 있습니다.
</figcaption>

이 기능을 사용하여 관련 대시보드 보기를 그룹화할 수 있습니다. 이는 특정 프로젝트, 팀 또는 주제와 관련된 많은 데이터 및 차트를 집계할 때 유용합니다.

예를 들어 모바일 앱 팀은 국가별 앱 성능에 중점을 둔 대시보드를 구축할 수 있습니다. 첫 번째 대시보드 페이지는 특정 국가에 초점을 맞춘 다른 페이지와 함께 모든 국가의 성과에 대한 개요일 수 있습니다.

대시보드를 연결하는 다른 기능이 있습니다.

* [Markdown 텍스트가 포함된 위젯](#markdown) 을 만들어 특정 UI 페이지 또는 대시보드에 대한 직접 링크를 추가합니다.
* [패싯 필터링](/docs/dashboards/new-relic-one-dashboards/manage-your-dashboard/filter-new-relic-one-dashboards-facets) 을 사용하여 다른 대시보드에 자동으로 연결하고 필터링하는 링크를 생성합니다.
* 대시보드 검색을 사용하여 유사한 이름의 대시보드를 찾습니다. 이를 활용하기 위해 대시보드 이름에 팀 또는 프로젝트별 단어/구를 추가할 수 있습니다.

### 대시보드에 페이지 추가 및 편집 [#add-pages-edit]

대시보드에서 페이지를 추가하거나 편집하려면:

1. 신규 또는 기존 대시보드에서 다음을 선택하여 편집 모드로 들어갑니다.

   <Icon name="fe-edit-2"/>

   상. 대시보드의 권한에 따라 이 아이콘이 표시되거나 표시되지 않습니다.

2. 새 페이지 추가:

   * **페이지 추가를** 선택하여 빈 페이지를 추가합니다.
   * 페이지 이름 옆에 있는 드롭다운을 클릭하고 **페이지 복제 를**선택하여 기존 페이지를 복제합니다.

3. 편집 모드에서 새 페이지에 위젯을 추가하고, 페이지 탭을 새 위치로 끌어다 놓고, 기타 [대시보드 편집 작업을](#dash-manage-charts)수행할 수 있습니다.

4. 완료되면 **수정 완료 를**선택합니다.

## 차트 필터링 및 구체화 [#dash-search]

쿼리 조건을 시각적으로 나타내는 필터링 기능을 사용하여 표시되는 정보의 범위를 좁힐 수 있습니다.

* **필터 막대** 를 사용하여 보려는 값이나 속성을 선택하고 차트에서 나머지 요소를 제거합니다.
* **고급 필터 표시줄** 을 열어 부울 연산자(예: `=` , `!=` , `CONTAINS` , `EXCLUDES` 등)에 액세스하고 데이터 필터링을 위한 복합 및 복합 조건을 추가합니다.

필터를 적용하면 대시보드에는 선택한 요소와 관련된 데이터만 표시됩니다. 작은 카운터는 한 번에 적용되는 필터 수를 나타냅니다.

기본 보기로 돌아가려면 필터 옆에 있는 작은 십자가를 클릭하여 제거합니다.

## 차트 범례를 사용하여 필터링 [#filter-legend]

범례가 있는 차트에서 범례를 클릭하면 해당 계열만 표시되고 나머지 계열은 차트에서 제거됩니다. 이렇게 하면 분석할 데이터를 분리하는 데 도움이 됩니다.

반대로 하려면 CMD(Mac의 경우) 또는 CTRL(Windows의 경우)을 사용합니다. 이렇게 하면 선택한 시리즈가 제거되고 나머지는 유지됩니다.

## 패싯을 사용하여 대시보드 필터링 [#filter-facets]

차트의 NRQL 쿼리에 `FACET` 절이 포함된 경우 패싯 속성을 사용하여 현재 대시보드 또는 다른 관련 대시보드를 필터링할 수 있습니다. 자세한 내용 [은 패싯으로 필터링을](/docs/dashboards/new-relic-one-dashboards/manage-your-dashboard/filter-new-relic-one-dashboards-facets) 참조하십시오.

## 시간 선택기를 사용하여 시간 설정 조정 [#dash-time-picker]

기본적으로 대시보드의 각 차트는 [쿼리 빌더](/docs/introduction-chart-builder)에서 생성될 때 지정된 기간 동안의 데이터를 표시합니다. 그러나 [시간 선택기를](/docs/new-relic-solutions/get-started/glossary/#time-picker) 사용하여 표시되는 데이터의 시간 범위를 변경하고 모든 차트에 대해 동일한 범위를 설정할 수 있습니다. 특정 기간에 발생한 일을 관찰하기 위해 데이터 범위를 좁혀야 하는 경우 인시던트 문제를 해결하는 동안 특히 유용합니다.

<img
  width="50%;"
  title="Time picker on the Dashboard page"
  alt="Time picker on the Dashboard page"
  src={dashboardsTimePicker}
/>

새로 고침 빈도는 보고 있는 기간의 지속 시간에 따라 다릅니다. 자세한 내용과 예는 [차트 새로 고침 간격](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-refresh-intervals) 을 참조하십시오.

시간 범위를 변경하려면:

* 드롭다운 메뉴에서 사용 가능한 옵션 중 하나를 선택합니다(마지막 `5 minutes` 에서 마지막 `6 months` 까지).
* 사용자 지정 메뉴를 사용하여 특정 시작 및 종료 타임스탬프로 시간 범위를 사용자 지정합니다.

시간 선택기를 사용할 때 차트의 시간 범위를 변경하지 않으려면 다음을 수행하십시오.

1. 클릭

   <Icon name="fe-more-horizontal"/>

   아이콘을 클릭하고 **수정을**선택합니다.

2. 사용자 지정 UI에서 **대시보드 옵션** 메뉴를 엽니다.

3. 필요에 따라 **무시 시간 선택기** 토글을 켜거나 끕니다.
