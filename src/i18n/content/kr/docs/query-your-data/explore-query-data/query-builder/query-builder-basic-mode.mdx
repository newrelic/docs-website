**기본 모드** 에서 [쿼리 빌더](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 를 사용하여 쿼리 언어인 NRQL을 사용하지 않고도 차트를 생성할 수 있습니다. 기본 모드는 쿼리 생성 프로세스를 안내하는 데 도움이 됩니다. 원시 데이터의 소스를 선택하고 필터를 적용하고 다른 기술을 사용하여 차트의 데이터 범위를 좁힐 수 있습니다.

<Callout variant="important">
  2021년 9월 1일부로 쿼리 빌더의 기본 모드 지원이 중단됩니다. 대신 New Relic에서 데이터 탐색기의 사용자 친화적이고 직관적인 기능을 사용할 수 있습니다. 이 전환을 쉽게 준비하는 방법을 비롯한 자세한 내용은 [Explorers Hub 게시물](https://discuss.newrelic.com/t/important-upcoming-changes-to-support-and-capabilities-across-browser-node-js-agent-query-builder-net-agent-apm-errors-distributed-tracing/153373) 을 참조하세요.
</Callout>

## 데이터 형식

쿼리 빌더 기본 모드에는 두 가지 옵션이 있는 **데이터 유형** 선택기가 있습니다.

* **이벤트:** 이 컨텍스트에서 이는 [이벤트](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) , [로그](/docs/using-new-relic/data/understand-data/new-relic-data-types#log-data) 및 [추적 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types#trace-data) 를 포함하여 [`Metric-` 유형](/docs/using-new-relic/data/understand-data/new-relic-data-types#dimensional-metrics) 이 아닌 모든 데이터를 나타냅니다.
* **측정항목:** 차원 [`Metric` 데이터 유형](/docs/using-new-relic/data/understand-data/new-relic-data-types#dimensional-metrics) 을 나타냅니다. 이것을 사용하여 [일부 유형의 메트릭 타임슬라이스 데이터를 쿼리](/docs/query-data/nrql-new-relic-query-language/nrql-query-tutorials/query-metric-timeslice-data-nrql) 할 수도 있습니다. 다른 유형의 측정항목에 대한 자세한 내용은 [데이터 유형](/docs/using-new-relic/data/understand-data/new-relic-data-types#metrics-new-relic) 을 참조하세요.

## 기본 모드 사용 예 [#example]

이 예는 기본 모드에서 차트를 생성하는 방법을 보여줍니다.

<CollapserGroup id="basic-mode-tutorial">
  <Collapser
    id="step-1"
    title={<strong>1단계: 차트의 데이터 소스 선택</strong>}
  >
    차트에서 보려는 데이터를 지정하여 시작합니다. 다음 **으로 차트 보기** 상자를 클릭하여 이벤트 유형, 속성 및 속성에 대해 수행할 기능을 선택하십시오.

    [이벤트 데이터 사전](/attribute-dictionary) 을 사용하여 단일 페이지에서 이벤트 유형 및 해당 속성에 대한 정보를 볼 수 있습니다. 이벤트 또는 속성에 대한 정보가 포함된 도구 설명을 보려면 그 아래에 점선이 있는 용어 위로 마우스를 가져갑니다.

    다음은 이벤트 데이터 사전을 사용하여 데이터를 지정한 결과입니다.

    * **이벤트 유형** . [`Transaction`](/attribute-dictionary/?event=Transaction) 이벤트 유형은 사용자가 페이지의 버튼을 클릭하는 것과 같이 사용자가 웹사이트에 있는 동안 발생하는 일을 설명하는 다양한 데이터를 측정합니다.

    * **속성** . `name` 속성은 모든 트랜잭션에 대한 정보를 저장합니다.

    * **기능** . `unique_count` 함수를 선택하여 기간 동안 발생한 모든 트랜잭션의 수를 가져옵니다.

      이제 기본 모드에 선택 항목이 표시됩니다.

      <img
        title="querybuilder02.png"
        alt="querybuilder02.png"
        src={querybuilder02}
      />

      <figcaption>
        **[one.newrelic.com](https://one.newrelic.com) > Query builder > Basic >** (이벤트 및 속성 지정)
      </figcaption>

      데이터를 지정하면 차트가 업데이트되어 지정한 데이터의 결과가 표시됩니다.

      지금까지 지정된 정보를 기반으로 기본 시간 프레임인 30분 동안의 총 트랜잭션 수를 보여주는 차트를 볼 수 있습니다. 이 합계에는 트랜잭션이 성공적으로 완료되었는지 또는 오류가 있었는지에 관계없이 모든 트랜잭션이 포함됩니다.

      <img
        title="crop-404-single-line.png"
        alt="crop-404-single-line.png"
        src={crop404SingleLine0}
      />

      <figcaption>
        **[one.newrelic.com](https://one.newrelic.com) > Query builder > Basic >** (이벤트 및 속성 지정)
      </figcaption>
  </Collapser>

  <Collapser
    id="step-2"
    title={<strong>2단계: 데이터 필터링</strong>}
  >
    다음 단계는 해당 트랜잭션 중 **404 페이지를 찾을 수 없음** 오류가 발생한 트랜잭션을 확인하는 것입니다. [`Transaction` 이벤트에 대한 데이터 사전을](/attribute-dictionary/?event=Transaction) 보면 이 이벤트에 `http.statusCode` 이라는 속성도 포함되어 있음을 알 수 있습니다.

    * **페이지를 찾을 수 없음** 오류가 발생한 트랜잭션만 표시하도록 결과 범위를 좁힙니다. 이 필터를 만들려면 **결과 범위 좁히기** 상자를 사용하세요. `http.statusCode = 404` .

    * 404 오류를 발생시키는 앱의 이름을 보고 싶기 때문에 **패싯 기준** 상자를 사용하여 `appName` ( `Transaction` 이벤트 유형에 대한 속성이기도 함)별로 결과를 확인합니다. `appName` 에 의한 패싯은 차트를 업데이트하여 애플리케이션 이름별로 총 404 오류 수를 분류합니다. 이를 통해 404 오류가 발생한 앱을 알 수 있습니다.

      이제 차트에 각각 고유한 색상이 있는 각 앱에 대한 선이 있는 [꺾은선형 차트](/docs/insights/use-insights-ui/manage-dashboards/chart-types#widget-linechart) 가 표시됩니다.

      <img
        title="crop-basic-example-chart.png"
        alt="crop-basic-example-chart.png"
        src={cropBasicExampleChart0}
      />

      <figcaption>
        **[one.newrelic.com](https://one.newrelic.com) > Query builder > Basic >** (이벤트 및 속성 지정) **>** (필터 및 패싯 적용)
      </figcaption>
  </Collapser>

  <Collapser
    id="step-3"
    title={<strong>3단계: 시간 범위 및 제한 조정</strong>}
  >
    **페이지를 찾을 수 없음** 오류가 가장 많은 5개 앱에 집중하기로 결정했습니다. **한도** 필드의 기본값은 10이며, 이는 차트에 가장 관련성이 높은 10개의 수익이 표시됨을 의미합니다. 그 값을 5로 변경합니다.

    고객 지원팀에서는 두 시간이 조금 넘는 시간 동안 이러한 오류에 대한 전화를 받았다고 말했습니다. 고객이 지원에 전화한 시간 동안 오류를 볼 수 있도록 **시간 범위** 를 지난 30분에서 마지막 3시간으로 변경하기로 결정합니다.

    필요한 데이터를 정확히 볼 수 있도록 데이터 세트를 확보했으므로 차트의 모양에 주의를 기울일 수 있습니다.
  </Collapser>

  <Collapser
    id="step-4"
    title={<strong>4단계: 차트 사용자 지정</strong>}
  >
    타임라인 보기보다 총 오류 수에 더 관심이 있으므로 [차트 유형](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-types) 을 [막대 차트](/docs/insights/use-insights-ui/manage-dashboards/chart-types#widget-barchart) 로 변경합니다.

    <img

    <figcaption>
      **one.newrelic.com > Query builder > Basic >** (이벤트 및 속성 지정) **>** (필터 및 패싯 적용) **>** (시간 범위 및 제한 사용자 지정) **>** (차트 유형 사용자 지정)
    </figcaption>
  </Collapser>
</CollapserGroup>
