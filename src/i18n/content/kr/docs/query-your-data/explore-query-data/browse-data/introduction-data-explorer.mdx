<DNT>**Metrics and events**</DNT> 사용하여 모든 데이터를 시각적으로 탐색할 수 있으며 [NRQL 형식 쿼리를](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) 작성하는 방법을 알 필요가 없습니다.

더 자세한 검색을 수행해야 합니까? [데이터 쿼리](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/)방법을 읽어보십시오. 아직 하지 않았다면 아래에서 무료 New Relic 계정을 만들어 오늘 데이터 모니터링을 시작하십시오.

## 중요한 이유 [#why-matters]

당신은 시스템 상태를 조사하시나요? 리소스를 계획하고, 감지하고 대응하고, 잘못된 동작을 해결해야 합니까? 메트릭과 이벤트를 사용하면 NRQL을 사용하거나 쿼리를 작성하지 않고도 시각적 메뉴를 통해 원하는 데이터를 쉽게 식별하고 가져오고 시각화할 수 있습니다.

<DNT>**Metrics and events**</DNT> 사용하면 뉴렐릭 데이터베이스(NRDB)에 저장된 모든 데이터에 빠르고 직관적으로 액세스할 수 있습니다. 그런 다음 <DNT>**Dimensions**</DNT> 선택하여 해당 데이터의 특정 차원에 따라 측면을 지정할 수 있습니다.

측정항목 및 이벤트가 도움이 되는 기타 사항은 다음과 같습니다.

* 원시 데이터에서 진화, 분포 등에 대한 통찰력을 제공하는 다양한 시각화에 이르기까지 다양한 관점에서 데이터를 확인하십시오.
* 필터를 사용하여 데이터를 드릴다운합니다.
* 선 및 영역 차트에 예측을 추가합니다( [NRQL 예측](/docs/query-your-data/explore-query-data/use-charts/nrql-predictions) 공개 미리 보기에서 사용 가능).
* [대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) 에 검색을 추가합니다.
* [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) 작동 방식 이해: 메트릭 및 이벤트는 쿼리 작성 방식을 보여줍니다.

## 탐색할 수 있는 데이터 유형 [#data-types]

<DNT>**Metrics and events**</DNT> 컨텍스트에서 `event` 뉴렐릭 데이터베이스(NRDB)에 저장된 모든 개체를 나타냅니다. 메트릭과 이벤트를 통해 다음을 탐색할 수 있습니다.

* 측정항목:

  * [차원 측정항목](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) ( `Metric` 데이터 유형).
  * [지표슬라이스 데이터](/docs/data-apis/understand-data/new-relic-data-types/#timeslice-data): <InlinePopover type="apm" />에이전트, 모바일 에이전트 및 <InlinePopover type="browser" />에이전트가 이 데이터 유형을 보고합니다. 이러한 유형의 데이터를 탐색하려면 해당 에이전트 중 하나의 IF모니터를 선택해야 하며, 그러면 해당 옵션이 표시됩니다.

* 이벤트: [이벤트 데이터 유형](/docs/data-apis/understand-data/new-relic-data-types/#events-new-relic) .

* 로그: `Log` 데이터 유형( [로그 UI](/docs/logs/ui-data/use-logs-ui) 를 통해서도 탐색할 수 있음).

* 추적: `Span` 데이터 유형( [추적 UI](/docs/distributed-tracing/ui-data/query-distributed-trace-data) 를 통해 탐색할 수도 있음).

## 데이터 탐색 [#explore-data]

데이터 탐색기에 액세스하려면 <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; Metrics and events**</DNT> 로 이동하세요.

메트릭 및 이벤트는 다음으로 구성됩니다.

* 상단의 범위 지정 섹션. `metrics` 또는 `events` 및 `account` 중에서 선택할 수 있습니다. 메트릭을 선택하면 엔터티별로 필터링할 수 있습니다.
* 왼쪽의 데이터 검색 영역. `metrics` 를 선택하면 옵션은 `metric` 및 `dimensions` 입니다. `events` 을 선택하면 옵션은 `event type` , `plot` 및 `dimensions` 입니다.
* 작업 공간. 이 영역에는 선택한 결과가 표시됩니다.

<img title="Metrics and events view" alt="Metrics and events view" src="/images/queries-nrql_screenshot-full_metrics-events-data-explorer.webp" />

</figcaption>

메트릭 및 이벤트를 사용하려면:

1. 범위를 정의합니다. 데이터 유형(지표 또는 이벤트), 계정 및 엔터티를 선택합니다.

2. [시간 선택기를](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)사용하여 시간 범위를 선택합니다.

3. 왼쪽에 있는 블록을 사용하여 사용 가능한 데이터를 탐색하고 검색을 구성합니다. 블록당 하나의 요소만 선택할 수 있습니다. 블록을 검색할 수 있습니다.

## 탐색 시각화 및 개선 [#workspace]

오른쪽 작업 공간에는 탐색 결과가 표시됩니다.

작업 공간에서 다음을 볼 수 있습니다.

    NRQL에 익숙하지 않은 경우 이 영역을 확인하여 쿼리 빌드에 대해 알아보세요.

## 맞춤 측정항목을 만든 후 시각화 [#visualize-metric]

측정항목 탐색기를 통해 맞춤 측정항목을 시각화하려면 다음 단계를 따르세요.

1. 데이터 탐색기에 액세스하려면 <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; Metrics and events**</DNT> 로 이동하세요.

2. 법인을 선택하세요.

3. <DNT>**See timeslice metrics for `your entity`**</DNT> 링크를 클릭하세요.

## 사용 사례 [#uses-cases]

메트릭 및 이벤트를 사용하는 방법과 시기를 알아보려면 다음 예를 참조하십시오.

<CollapserGroup>
  <Collapser className="freq-link" id="use-case-validate-data" title="새로운 데이터 발견">
    <DNT>
      **I&apos;ve just connected new instrumentation and want to see if new data is available.**
    </DNT>

    1. 새 데이터를 생성하는 계정 및 이벤트 또는 측정항목을 선택합니다.
