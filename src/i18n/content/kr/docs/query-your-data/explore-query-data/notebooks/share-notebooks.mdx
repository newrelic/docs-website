노트북은 협업을 지원하도록 설계되었습니다. 데이터 중심 문서를 팀원들과 쉽게 공유하여 인사이트를 배포하고, 공동으로 조사를 하거나, 재사용 가능한 분석 템플릿을 만들 수 있습니다.

## 노트북 공유 개요

뉴렐릭 조직 내에서 생성된 모든 노트북은 노트북 기능에 액세스할 수 있는 조직의 다른 구성원들이 자동으로 사용할 수 있습니다. 이를 통해 팀 전체에서 원활한 협업이 가능해 집니다.

<Callout variant="important">
  현재 노트북에는 세부적인 권한 설정 기능이 없습니다. 귀하의 조직 내에서 노트북 기능에 액세스할 수 있는 사람은 누구나 모든 노트북을 보고, 수정하고, 삭제할 수 있습니다. 향후 릴리스에서는 더 세부적인 권한 설정이 도입될 예정입니다.
</Callout>

## 노트북 URL 공유

특정 노트북을 공유하는 가장 간단한 방법은 URL을 공유하는 것입니다.

1. 공유하려는 노트북을 엽니다.
2. 브라우저의 주소창에서 URL을 복사합니다.
3. 이메일, 채팅 또는 선호하는 커뮤니케이션 도구를 통해 팀원과 URL을 공유합니다.

(뉴렐릭 조직에 접근할 수 있는) 팀원이 URL 클릭하면 바로 노트북으로 이동합니다.

## 공유 노트북 찾기

팀원이 만든 노트북을 검색하려면:

1. **[one.newrelic.com](https://one.newrelic.com)**으로 이동합니다.
2. **Query your data** 콘솔을 엽니다.
3. **Notebooks**를 클릭하면 노트북 인덱스가 표시됩니다.
4. 조직의 모든 노트북에서 찾아보거나 검색 기능을 사용해 원하는 노트북을 찾습니다.

### 노트북 검색

노트북 인덱스의 검색 창에 다음을 입력해 특정 노트북을 찾을 수 있습니다.

* 노트북 이름
* 노트북 제목의 키워드
* 노트북 내 콘텐츠(검색 가능한 경우)

### 노트북 정렬

노트북 목록을 다음 기준으로 정렬할 수 있습니다.

* **Name**: 알파벳순
* **Last modified**: 최신 업데이트 순
* **Created date**: 최신 또는 오래된 순
* **Favorites**: 별표가 표시된 노트북을 맨 위에 표시

## 협업 모범 사례

### 명명 규칙

팀원들이 노트북의 목적을 이해하는데 도움이 되도록 내용을 설명하는 명확한 이름을 사용합니다.

* **좋음**: &quot;API 성능 조사 - 2024년 10월&quot;
* **좋음**: &quot;주간 오류율 분석 템플릿&quot;
* **좋음**: &quot;고객 이탈 분석 - 3분기 실적&quot;
* **나쁨**: &quot;내 노트북&quot; 또는 &quot;테스트&quot;

### 조직 전략

#### 템플릿 생성

보편적인 분석 작업을 위해 재사용 가능한 노트북 템플릿을 구축할 수 있습니다.

1. 표준 쿼리와 구조로 노트북 생성
2. 사용자 정의 가능한 요소에 변수 사용
3. 팀원을 위한 마크다운 지침 추가
4. 명확하게 템플릿이라는 이름 지정(예: &quot;템플릿: 조사&quot;)

#### 분석 문서화

마크다운 블록을 사용해 맥락을 제공할 수 있습니다.

* 각 쿼리의 목적을 설명합니다.
* 분석 방법론을 문서화합니다.
* 관련 리소스에 대한 링크를 포함합니다.
* 결론과 다음 단계를 추가합니다.

#### 설명이 포함된 시각화 사용

* 차트에 명확한 제목을 추가합니다.
* 일관된 색상 테마를 사용합니다.
* 축 레이블과 단위를 포함합니다.
* 관련성이 있는 경우 한도 또는 기준 표시기를 추가합니다.

## 협업 워크플로우

### 인시던트 대응

인시던트 조사를 위한 협업용 노트북을 생성할 수 있습니다.

1. 표준 인시던트 템플릿으로 시작합니다.
2. 조사가 진행되면서 필요한 팀원을 추가합니다.
3. 마크다운 블록을 사용해 실시간으로 문서 결과를 확인합니다.
4. 관련 쿼리와 시각화를 포함합니다.
5. 근본 원인과 해결 단계를 요약합니다.

### 주간/월간 보고

반복적으로 사용되는 분석 노트북을 구축할 수 있습니다.

1. 표준 메트릭을 사용하여 노트북 템플릿을 생성합니다.
2. 시간 범위 및 필터에 변수를 사용합니다.
3. 팀과 정기적인 검토 일정을 수립합니다.
4. 날짜별 이름으로 완료된 보고서를 보관합니다.

### 지식 공유

일회성 조사를 재사용 가능한 지식으로 전환할 수 있습니다.

1. 세부 분석 노트북을 생성합니다.
2. 마크다운에 방법론과 추론을 포함합니다.
3. 발견된 특이 사항을 설명하는 댓글을 추가합니다.
4. 향후 참고할 수 있도록 다른 팀들과 공유합니다.

## 버전 제어 및 기록

<Callout variant="tip">
  현재 노트북에는 버전 기록 기능이 내장되어 있지 않습니다. 이전 버전을 보존하려면 노트북 내용을 변경하기 전에 사본을 만드는 것이 좋습니다.
</Callout>

버전 제어를 하려면:

1. **주요 변경 사항 전에 복제**: 중요한 수정 사항을 적용하기 전에 노트북 사본을 만듭니다.
2. **설명이 담긴 이름 사용**: 노트북 이름에 버전 번호 또는 날짜를 포함합니다.
3. **문서 변경 사항**: 마크다운 블록을 사용하여 무엇이 언제 변경되었는지 기록합니다.
4. **이전 버전 보관**: 오래된 버전을 별도의 폴더로 옮기거나 다른 명명 규칙을 적용합니다.

## 공유 관련 문제 해결

### 팀원에게 노트북이 보이지 않습니다.

팀원이 노트북을 볼 수 없는 경우:

1. **조직 액세스 확인**: 동일한 뉴렐릭 조직에 속해 있는지 확인합니다.
2. **기능 액세스 확인**: 노트북 미리 보기 기능에 액세스할 수 있는지 확인합니다.
3. **직접 URL 시도**: 인덱스에서 검색하도록 만드는 대신 노트북 URL을 공유합니다.

### 공유 노트북 사용 시 성능

크고 복잡한 노트북은 모든 사용자에게 느리게 로딩될 수 있습니다.

1. **쿼리 최적화**: 적절한 시간 범위와 필터를 사용합니다.
2. **결과 세트 제한**: NRQL 쿼리에서 `LIMIT` 절을 사용합니다.
3. **분할 고려**: 사이즈가 큰 노트북을 더 작은 문서로 분할합니다.
