</Callout>

우리의 <DNT>**Instrumentation**</DNT> 편집기를 사용하면 .NET 앱 사용자가 기능 구성을 다시 시작할 필요 없이 뉴렐릭 사용자 인터페이스에서 사용자 정의를 구현할 수 있습니다. 이 편집자는:

* 승인된 사용자가 로컬 XML 파일 대신 New Relic UI에서 사용자 정의 계측을 편집할 수 있습니다.
* 사용자 지정 계측 스키마에 대해 유효성을 검사합니다.
* 애플리케이션이 실행되는 애플리케이션 코드나 호스트에 액세스할 수 없는 고객을 위한 옵션을 제공합니다.

<Callout variant="important">
  라이브 계측을 배포하면 잠시 동안 더 높은 오버헤드가 발생합니다. 이로 인해 몇 초 동안 애플리케이션 요청이 눈에 띄게 느려질 수 있습니다.
</Callout>

<DNT>**Instrumentation**</DNT> 편집기를 사용하려면:

1. <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM**</DNT> 로 이동합니다. 앱을 찾은 다음 <DNT>**Settings > Instrumentation**</DNT>(를) 클릭하세요.

2. [로컬 .xml 계측 파일 편집](/docs/agents/net-agent/custom-instrumentation/create-transactions-xml-net) 및 [계측 세부정보 추가](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net) 에 대해 설명된 것과 동일한 구문을 사용하여 XML을 제공하려면 <DNT>**Instrumentation**</DNT> 편집기를 사용하세요. UI에서 다음 옵션 중 하나를 사용하세요.

   * XML 코드를 직접 입력합니다.

   * 코드를 복사하여 클립보드에서 붙여넣습니다.

   * <DNT>
       **Download Sample XML**
     </DNT>

     클릭하여 시작점으로 샘플 계측 파일을 만듭니다.

   * .xml 업로드

     <DNT>
       **Choose file**
     </DNT>

     선택하여 파일을 저장합니다. 예를 보려면 [MyInstrumentation.xml 예를](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net#sample) 참조하세요.

3. <DNT>**Instrumentation**</DNT> 편집기는 입력할 때 XML 코드의 유효성을 검사합니다. 편집기 외부에서 XML 유효성 검사를 수행하려면 <DNT>**Download XSD**</DNT> 선택하고 즐겨 사용하는 XML 유효성 검사 도구와 함께 XSD 파일을 사용하여 문제를 진단하십시오.

4. 변경 사항을 구현, 배포하려면 <DNT>**Submit**</DNT> 선택합니다.

## 계측 결과 보기 [#view]

.NET 에이전트는 몇 [하베스트 배포](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#harvest-cycle) (일반적으로 몇 분) 내에 새로운 감수를 시작합니다. <DNT>**Instrumentation**</DNT> UI의 <DNT>**Current instrumentation**</DNT> 섹션에는 적용된 변경사항의 상태가 표시됩니다.

<DNT>**Instrumentation**</DNT> 편집기의 XML은 에이전트를 실행하는 머신의 로컬 계측 파일에 있는 XML과 결합됩니다. 에이전트는 에이전트나 애플리케이션을 다시 시작할 필요 없이 주기적으로 최종(결합된) XML의 변경 사항을 찾아 동적으로 선택합니다.
