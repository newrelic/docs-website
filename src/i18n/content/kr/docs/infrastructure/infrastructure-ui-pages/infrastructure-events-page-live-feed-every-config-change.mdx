인프라 모니터링 UI에서 **Events** \[이벤트] UI 페이지는 인벤토리 변경 이벤트, 구성 변경 및 로그 분석 이벤트를 포함하여 중요한 시스템 및 호스트 활동의 라이브 피드입니다. 이벤트 피드는 이러한 이벤트와 시스템 성능 간의 상관 관계를 이해하는 데 도움이 됩니다. 이벤트를 검색하고 필터링하여 인프라 문제를 감지하고 복구하는 평균 시간을 단축합니다.

<Callout variant="tip">
  여기서 `event` 는 [이벤트 데이터 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 과 다른 인프라별 의미를 가집니다.
</Callout>

**[one.newrelic.com > 모든 기능](https://one.newrelic.com/all-capabilities) > 인프라 > 이벤트로** 이동하여 **Events** \[이벤트] 페이지를 찾으십시오.

</figcaption>

## 이벤트 타임라인 [#event-timeline]

이벤트 차트는 표시된 측정항목과 동일한 시간 범위 내에서 발생하는 인프라 [이벤트](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page) 의 스냅샷을 제공합니다. 마우스를 올리면 색상으로 구분된 범주별 이벤트 수를 확인하여 비정상적인 스파이크를 조사할 수 있습니다.

<img
  title="infra-events-timeline.png"
  alt="infra-events-timeline.png"
  src={infrastructureEventsTimeline}
/>

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**: 이벤트 타임라인은 표시된 지표에 대해 동일한 기간에 발생하는 이벤트의 패턴을 보여줍니다.
</figcaption>

인프라 페이지의 차트와 타임라인을 비교하면 생태계의 문제를 빠르게 찾아낼 수 있습니다. 예를 들어 대규모 CPU 스파이크가 발생하면 해당 시간 범위의 이벤트 타임라인을 클릭하여 이를 유발한 이벤트를 찾을 수 있습니다. 거기에서 실제 문제를 밝혀내기 위해 더 깊이 파고들 수 있습니다.
