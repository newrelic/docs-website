New Relic의 인프라 모니터링에서 **이벤트** UI 페이지는 인벤토리 변경 이벤트, 구성 변경 및 로그 분석 이벤트를 포함하여 중요한 시스템 및 호스트 활동의 라이브 피드입니다. 이벤트 피드는 이러한 이벤트와 시스템 성능 간의 상관 관계를 이해하는 데 도움이 됩니다. 이벤트를 검색하고 필터링하여 인프라 문제를 감지하고 복구하는 평균 시간을 줄입니다.

<Callout variant="tip">
  "이벤트"의 이러한 사용은 인프라 모니터링에만 해당되며 보다 일반적인 [이벤트 데이터 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 과는 별도의 개념입니다.
</Callout>

**[one.newrelic.com](http://one.newrelic.com) > Infrastructure > Events** 로 이동하여 **이벤트** 페이지에 액세스할 수 있습니다.

New Relic은 환경의 각 변경 사항을 이해할 수 있도록 다양한 변경 이벤트를 수집합니다.

        인벤토리 이벤트에는 다음이 포함될 수 있습니다.

        * **커널** (모듈 및 구성 포함): 추가, 수정 또는 삭제할 수 있습니다.

        * **서비스** : 시작, 중지 또는 다시 시작할 수 있습니다.
        * **세션** (사용자 포함): 연결 또는 연결 해제할 수 있습니다.

## 이벤트 페이지 기능 [#search]

라이브 이벤트 피드를 보려면 **[one.newrelic.com](http://one.newrelic.com) > Infrastructure > Events** 로 이동합니다. **이벤트** 페이지에는 선택한 시간 범위 내에서 발생하는 이벤트의 스냅샷을 제공하는 [히트맵](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/events-heatmap-examine-patterns-time-range) 이 포함되어 있습니다.

## 차트 데이터 속성 [#attributes]

**이벤트** 페이지를 채우는 데 사용되는 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 에 대한 설명은 [InfrastructureEvent 속성](https://docs.newrelic.com/attribute-dictionary/?event=InfrastructureEvent) 을 참조하십시오.
