우리의 레이아웃 모니터링 UI 에서 <DNT>**Events**</DNT> UI 페이지는 인벤토리 변경 이벤트, 설정 변경, 로그 분석 이벤트를 비롯한 중요한 시스템 및 호스트 활동의 라이브 피드입니다. 이벤트 피드는 이러한 이벤트와 시스템 성능 간의 상관 관계를 이해하는 데 도움이 됩니다. 이벤트를 검색하고 필터링하여 인프라 문제를 감지하고 복구하는 데 소요되는 평균 시간을 줄입니다.

<Callout variant="tip">
  여기서 `event` 는 [이벤트 데이터 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 과 다른 인프라별 의미를 가집니다.
</Callout>

<DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**</DNT> 로 이동하여 <DNT>**Events**</DNT> 페이지를 찾으세요.

</figcaption>

## 이벤트 타임라인 [#event-timeline]

이벤트 차트는 표시된 측정항목과 동일한 시간 범위 내에서 발생하는 인프라 [이벤트](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page) 의 스냅샷을 제공합니다. 마우스를 올리면 색상으로 구분된 범주별 이벤트 수를 확인하여 비정상적인 스파이크를 조사할 수 있습니다.

<img
  title="infra-events-timeline.png"
  alt="infra-events-timeline.png"
  src="/images/infrastructure_screenshot-crop_events-timeline.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**</DNT>: 이벤트 타임라인은 표시된 지표에 대해 동일한 시간대에 발생한 이벤트의 패턴을 보여줍니다.
</figcaption>

인프라 페이지의 차트와 타임라인을 비교하면 생태계의 문제를 빠르게 찾아낼 수 있습니다. 예를 들어 대규모 CPU 스파이크가 발생하면 해당 시간 범위의 이벤트 타임라인을 클릭하여 이를 유발한 이벤트를 찾을 수 있습니다. 거기에서 실제 문제를 밝혀내기 위해 더 깊이 파고들 수 있습니다.

        * <DNT>
            **Kernel**

            **Metadata**

          (호스트에 대한 다양한 추가 정보 포함) : 추가, 수정, 삭제할 수 있습니다.

            **Packages**
          </DNT>

          : 설치, 제거 또는 수정될 수 있습니다.

        * <DNT>
            **Services**
          </DNT>

          : 시작, 중지 또는 다시 시작할 수 있습니다.

        * <DNT>
            **Sessions**
          </DNT>

          (사용자 포함): 연결하거나 연결 해제할 수 있습니다.
