인프라 모니터링 도구의 2022년 재런칭은 사용자가 [플랫폼](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one/) 과 상호 작용하는 방식에 일관성을 추가합니다.

이전에는 상단 탐색 모음에서 인프라 항목을 클릭하면 [탐색기](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/) 외부로 리디렉션되어 탐색기 및 Lookout과 같은 기능이 누락되었습니다. 이제 인프라 또는 호스트를 클릭하면 **Explorer의 목록 보기에** 있는 모든 호스트 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/) 목록으로 이동하고 **인프라 이벤트가 활동 스트림에 표시됩니다** . 이 릴리스의 일부로 인벤토리 환경이 변경되지 않았습니다.

짧은 비디오(6분 77초)로 새로운 인프라 모니터링 기능에 대해 자세히 알아보십시오.

## 새 인프라 UI에 액세스 [#access-new-ui]

새로운 인프라 모니터링 확인: [one.newrelic.com](https://one.newrelic.com) 으로 이동하여 상단 탐색 모음에서 **인프라** 를 클릭한 다음 **호스트(새로 만들기)** 를 선택합니다. 또는 **탐색기** 로 이동하여 **호스트** 를 선택합니다.

<img
  title="New infrastructure UI"
  alt="The new infrastructure hosts UI"
  src={pegasusUiFeb2022}
/>

인프라 탐색기 목록 보기는 다음 패널로 구성됩니다.

* **시스템의 엔티티** 는 왼쪽에 있습니다.

* 가운데에 있는 **Estate View** 패널.

  * 한 번에 최대 25개의 엔터티를 선택하여 함께 분석할 수 있는 기능을 제공합니다. 관찰하려는 호스트를 선택하고 **선택한 항목 보기** 버튼을 눌러 새 [분석 nerdlet](#infra-analysis-nerdlet) 으로 이동합니다.
  * 목록 보기에서 목록의 엔터티 이름을 클릭하여 개별 엔터티 요약으로 직접 이동할 수 있습니다.
  * 엔터티의 오른쪽 끝에 있는 **...** 를 클릭하면 엔터티의 태그 및 기타 관련 정보를 볼 수 있습니다.

* 오른쪽의 **활동 스트림** .

</Callout>

### 엔티티는 골든 메트릭을 기반으로 표시됩니다. [#infra-entity-golden-metrics]

새 엔터티 목록은 New Relic 전체에서 보기 시작하는 [골든 메트릭](/docs/apis/nerdgraph/examples/golden-metrics-entities-nerdgraph-api-tutorial/) 으로 구동됩니다. 이러한 인프라 엔터티 유형에 대한 Lookout을 보도록 선택하면 동일한 메트릭 세트가 표시됩니다. 이러한 새 메트릭은 이전 UI의 이전 요약 메트릭을 대체합니다.

### 향상된 정렬 및 필터링 기능이 있는 보기 [#infra-entity-filter]

열 값을 정렬하기 위해 2,000개 미만의 엔터티 제한을 제거했습니다. 10,000개 이상의 항목이 있는 경우 성능에 약간의 영향을 미칠 수 있습니다.

정렬 외에도 엔터티의 메트릭 값을 필터링하여 주의가 필요한 엔터티를 정확히 파악할 수 있습니다. 예를 들어 CPU가 80% 이상인 엔터티만 필터링할 수 있으므로 해당 자산에 문제가 있는지 빠르게 식별할 수 있습니다. 필터링은 현재 하나의 열로 제한되어 있지만 향후 릴리스에서는 여러 열로 확장될 예정입니다.

<img
  title="Analysis nerdlet"

  src={nerdlet4Tabs}

데이터를 탐색할 수 있는 4개의 탭이 있습니다.

* [체계](#nerdlet-system)
* [회로망](#nerdlet-network)
* [프로세스](#nerdlet-processes)

</Callout>

### 시스템 탭 [#nerdlet-system]

시스템 탭에는 세 개의 패널이 있습니다.

**왼쪽 패널: 선택한 엔티티 목록**

상단 드롭다운에서 이러한 엔터티의 주요 측정항목 간에 전환합니다.

<img
  title="Filter by metrics"
  alt="Filter by metrics"
  src={systemEntityDropdown}
/>

엔터티에 대한 자세한 내용을 보려면 엔터티 행 위로 마우스를 가져갑니다.

<img
  title="Entity details"
  alt="See details on your entity"
  src={newHostsMenu}
/>

* **로그 보기** 를 클릭하여 컨텍스트 정보의 로그가 있는 새 탭을 엽니다. 로그를 활성화하려면 [로그 관리 문서를](/docs/logs/forward-logs/enable-log-management-new-relic/) 참조하십시오. 호스트 분석 패널이 여전히 제자리에 있는 탭으로 돌아갈 수 있습니다.
* 호스트의 경우에만 **맵** 아이콘을 사용하여 특정 호스트 엔터티에 대한 자동 매핑 기능을 시작합니다. 이 보기는 스택의 다른 엔터티에 대한 호스트의 관계를 시각적으로 보여줍니다.

<img
  title="Auto-map feature"
  alt="Auto-map feature"
  src={nerdletAutomap}
/>

* 해당 개별 엔터티에 대한 요약 페이지로 리디렉션할 **호스트 보기** 라는 **돋보기** 아이콘을 선택합니다.
* 왼쪽 패널에서 엔터티를 선택하면 해당 엔터티의 선도 각 차트에서 강조 표시됩니다.
* **눈** 아이콘을 클릭하여 목록에서 엔티티를 숨깁니다. 왼쪽 패널에서 엔터티를 숨기도록 선택한 경우 해당 엔터티는 가운데 패널의 차트에서 제거됩니다.

**중간 패널: 골든 메트릭 비교 차트**

골든 메트릭 비교 차트는 선택한 엔터티에 대한 [골든 메트릭 신호](https://github.com/newrelic/entity-definitions) 를 나타냅니다. 기본 차트에는 CPU 사용량, 메모리 사용량, 스토리지 사용량, 네트워크 전송 트래픽 및 네트워크 수신 트래픽이 포함됩니다.

분석을 극대화하려면:

* 왼쪽 패널에서 엔터티를 숨기도록 선택한 경우 해당 엔터티는 차트에서 제거됩니다.
* 왼쪽 패널에서 엔터티가 강조 표시되면 해당 엔터티의 선도 각 차트에서 강조 표시됩니다. 또는 선을 강조 표시하면 엔터티도 왼쪽에 강조 표시됩니다.
* 툴팁 위에 마우스를 올려 놓으면 각 차트의 데이터 포인트에 대한 세부 정보를 확인하십시오.

**오른쪽 패널: 관련 항목**

New Relic에서 관계를 이해하는 것은 복잡할 수 있지만 문제를 해결하려고 할 때 매우 강력합니다. 호스트가 오작동을 하고 있고 클릭을 많이 하지 않고도 호스트가 영향을 미칠 수 있는 기타 사항을 알고 싶어한다고 상상해 보십시오. [관련 엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/#related-entities) 패널은 왼쪽 패널에서 선택과 관련된 엔터티를 표시하여 해당 엔터티에 대한 일반적인 황금 신호를 볼 수 있도록 합니다. 연결된 APM 서비스가 느린 응답 시간, 처리량 감소 또는 높은 오류율을 경험하고 있는지 한 눈에 확인하십시오!

<img
  title="See related entities"
  alt="See related entities"
  src={systemRelatedEntities}
/>

* 새 보기를 처음 입력하면 해당 엔터티의 경고 상태별로 정렬된 관련 엔터티 패널이 표시됩니다. 심각한 위반이 있는 서비스가 있으면 바로 맨 위에 표시됩니다!
* 왼쪽 패널에서 엔터티를 가리키면 항목과 연결된 관련 엔터티가 오른쪽에 강조 표시됩니다. 또한 왼쪽에 있는 항목을 클릭하면 오른쪽 패널이 재정렬되어 연관된 관련 엔터티를 오른쪽 열의 상단으로 범프합니다.

### 네트워크 탭 [#nerdlet-network]

**네트워크** 탭은 네트워크 전체에서 개별 리소스의 상태와 성능에 대한 실시간 가시성을 제공합니다. 기본 차트는 전송 바이트, 전송 오류, 전송 패킷, 엔터티 목록과 해당 인터페이스 정보를 보여줍니다.

<img
  title="The network tab"
  alt="The network tab"
  src={networkTab}
/>

개별 인터페이스에 대한 세부 정보는 다음을 수행하는 데 도움이 될 수 있습니다.

* 특정 리소스 간의 부하 분산을 비교합니다.
* 유사한 호스트 간의 용량 패턴에서 예상치 못한 차이를 식별합니다.
* New Relic이 선택한 기간 동안 자동으로 표시하는 상위 5개 네트워크 오류를 평가하십시오.

이 실시간 네트워크 데이터는 구매 결정을 내리기 전에 네트워크를 재구성하거나 부하를 재조정하거나 기타 예방적 유지 관리 조치를 취하여 오류를 해결할지 여부를 결정하는 데 도움이 될 수 있습니다.

### 프로세스 탭 [#nerdlet-processes]

<Callout variant="important">
  기본적으로 인프라 에이전트는 안내 설치를 사용하지 않는 한 [운영 체제의 프로세스에 대한 데이터를](/attribute-dictionary/?event=ProcessSample) 보내지 않습니다. 프로세스 데이터 세트 [`enable_process_metrics`](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/) 을(를) `true` 으)로 보내기를 활성화하려면 . 모니터링하려는 프로세스를 미세 조정하려면 [`include_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#include-matching-metrics) 를 구성하십시오.
</Callout>

**프로세스** 탭을 사용하여 호스트에서 실행 중인 프로세스에 대한 정보를 얻고 프로세스 메트릭에 대한 경고를 설정합니다. **프로세스** 탭에는 개별 프로세스 또는 프로세스 그룹에 대한 CPU 백분율, I/O 바이트 및 메모리 사용량과 같은 데이터가 표시됩니다.

**프로세스** 페이지 차트를 채우는 데 사용되는 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 에 대한 기술적인 설명은 [ProcessSample 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#processsample-attributes) 을 참조하십시오.

<Callout variant="tip">
  UI에서 프로세스를 시작, 중지 또는 다시 시작할 수 없습니다. 대신 표준 절차에 따라 인프라 에이전트를 수동으로 [시작, 중지, 확인 또는 다시 시작](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status) 합니다.
</Callout>

### 스토리지 탭 [#nerdlet-storage]

**저장소** 탭을 사용하면 장치의 전체 사용률, 디스크 사용량 또는 I/O 작업을 포함하여 리소스의 용량과 효율성을 모니터링할 수 있습니다. 이를 통해 다음을 수행할 수 있습니다.

* 예상치 못한 패턴을 조사하십시오. 예를 들어 I/O 작업을 균일하게 처리하지 않는 미러링된 시스템의 클러스터입니다.
* 디스크 공간이 완전히 소진되기 전에 사용량 수준을 모니터링하십시오.
* 하나 이상의 호스트에 문제가 있는 경우 알림 조건을 설정합니다. 예를 들어 읽기 또는 쓰기 작업을 위한 긴 처리 시간, 백분율 또는 바이트를 기반으로 한 디스크 가용성 또는 사용률 등
* 일반 처리 요청보다 높거나 낮은 호스트를 재배포할 위치에 대해 정보에 입각한 결정을 내립니다.
* 비상 사태가 발생하기 전에 데이터를 사용하여 인프라 예산에 대한 추가 또는 변경 계획을 세우십시오.

스토리지 페이지 차트를 채우는 데 사용되는 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 에 대한 기술적인 설명은 [StorageSample 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#storagesample-attributes) 을 참조하십시오.
