**호스트** 페이지를 사용하여 리소스 사용량, 네트워크 및 프로세스 성능, 컨테이너에 대한 데이터와 같은 중요한 성능 메트릭을 더 잘 이해할 수 있습니다. 속성별로 호스트를 필터링하고 해당 성능을 인벤토리 변경 이벤트와 비교하여 근본 원인을 파악할 수 있습니다. 문제를 방지하는 데 도움이 되도록 경고 알림을 설정할 수도 있습니다.

호스트 정보를 보려면 **[one.newrelic.com](http://one.newrelic.com) > Infrastructure > Hosts** 로 이동한 후 다음 탭 중 하나를 클릭하십시오.

* **[시스템](#system-tab)** : 호스트 성능 개요
* **[네트워크](#network-tab)** : 네트워크 인터페이스에 대한 대역폭 및 오류 데이터
* **[프로세스](#processes-tab)** : 개별 또는 프로세스 그룹에 대한 CPU 백분율, I/O 바이트 및 메모리 사용량에 대한 데이터
* **[저장소](#storage-tab)** : 장치의 전체 사용률, 디스크 사용량 또는 I/O 작업을 포함한 리소스의 용량 및 효율성
* **[Docker 컨테이너](#containers-tab)** : 개별 또는 컨테이너 그룹에 대한 CPU 백분율, I/O 바이트 및 메모리 사용량

## 시스템 탭

다음은 **호스트** 페이지에 표시되는 기본 그래프입니다. 추가 차트 옵션을 보려면 드롭다운을 선택하십시오. <Icon name="fe-chevron-down"/>모든 차트에 대해.

* **CPU %** : **시스템** 탭에서 CPU 백분율은 `SystemSample` 이벤트의 일부인 파생된 메트릭입니다. CPU 백분율은 New Relic에서 수집하지 않지만 여러 다른 메트릭에서 파생됩니다. 특히 [`cpuPercent`](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#systemsample-cpu-percent) 속성은 `cpuUserPercent` , `cpuSystemPercent` , `cpuIoWaitPercent` 및 `cpuStealPercent` 의 집계입니다.
* **부하 평균 5분** : 지난 5분 동안 시스템에서 작업을 수행하기 위해 대기 중인 작업 수의 평균을 나타냅니다.
* **메모리 여유 %** : 여유 메모리 바이트의 양을 사용된 메모리 바이트의 양과 비교합니다.

다양한 차트 메트릭에 대한 설명은 [기본 인프라 속성 및 이벤트](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#systemsample-attributes) 를 참조하십시오.

차트 데이터 조정 기능은 다음과 같습니다.

* **다른 호스트 그룹 선택:** [호스트 필터](/docs/infrastructure/new-relic-infrastructure/filter-group/filter-sets-organize-your-infrastructure-hosts) 를 사용합니다.
* **시간 범위 변경:** 차트 위의 시간 선택기를 사용하거나 차트에서 시간 범위를 끌어서 선택합니다.
* **차트의 요소 수 변경:** **차트 표시** 드롭다운을 사용합니다.
* **호스트 주문에 사용되는 데이터 변경:** 차트 아래의 표를 사용하여 다양한 메트릭별로 차트 요소를 정렬합니다. 예를 들어, 최고에서 최저로 `CPU user %` 또는 최저에서 최고로 `Disk free` .

### APM 및 인프라 데이터 [#apm]

인프라 데이터와 연결된 APM 데이터가 있는 경우 **호스트** 페이지에서 여러 가지 방법으로 APM 데이터에 액세스할 수 있습니다.

* [호스트 필터](/docs/infrastructure/new-relic-infrastructure/filter-group/filter-sets-organize-your-infrastructure-hosts) 를 사용하여 특정 애플리케이션을 실행하는 호스트를 필터링합니다.
* 차트 아래의 호스트 목록에서 **애플리케이션** 열을 선택하여 특정 애플리케이션을 필터링합니다.
* 차트 이름 옆의 차트 선택기 드롭다운에서 애플리케이션 관련 차트 중 하나를 선택합니다.

<Callout variant="important">
  인프라 모니터링의 APM 차트에는 다른 인프라 차트처럼 **쿼리 보기** 또는 **경고 생성** 옵션이 없습니다.
</Callout>

APM과 인프라 모니터링을 함께 사용하는 방법에 대한 자세한 내용은 [인프라의 APM 데이터를](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure) 참조하세요.

## 네트워크 탭

**네트워크** 페이지는 네트워크 전반에 걸쳐 개별 호스트, 웹 서버 또는 기타 리소스 그룹의 상태와 성능에 대한 실시간 가시성을 제공합니다. 기본 차트는 패킷별 대역폭 메트릭, 바이트별 대역폭 및 초당 오류를 표시합니다.

개별 인터페이스에 대한 세부 정보는 다음을 수행하는 데 도움이 될 수 있습니다.

* 네트워크 전체 또는 선택한 [필터 세트](/docs/scopes-organize-your-infrastructure) 에 대한 포화 수준을 검사합니다.
* 특정 리소스 간의 부하 분산을 비교합니다.
* 유사한 호스트 간의 용량 패턴에서 예상치 못한 차이를 식별합니다.
* New Relic이 선택한 기간 동안 자동으로 표시하는 상위 5개 네트워크 오류를 평가하십시오.

이 실시간 네트워크 데이터는 구매 결정을 내리기 전에 네트워크를 재구성하거나 부하를 재조정하거나 기타 예방적 유지 관리 조치를 취하여 오류를 해결할지 여부를 결정하는 데 도움이 될 수 있습니다.

**네트워크** 탭에서 네트워크 인터페이스에 대한 대역폭 및 오류 데이터를 볼 수 있습니다. **네트워크** 페이지에는 표시된 측정항목과 동일한 시간 범위 내에서 발생하는 이벤트의 스냅샷을 제공하는 [**이벤트** 히트맵](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/events-heatmap-examine-patterns-time-range) 이 포함되어 있습니다.

**네트워크** 페이지 차트를 채우는 데 사용되는 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 에 대한 기술적인 설명은 [NetworkSample 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#networksample-attributes) 을 참조하십시오.

## 프로세스 탭

<Callout variant="important">
  기본적으로 인프라 에이전트는 안내 설치를 사용하지 않는 한 [운영 체제의 프로세스에 대한 데이터를](/attribute-dictionary/?event=ProcessSample) 보내지 않습니다. 프로세스 데이터 세트 [`enable_process_metrics`](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/) 을(를) `true` 으)로 보내기를 활성화하려면 . 모니터링하려는 프로세스를 미세 조정하려면 [`include_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#include-matching-metrics) 를 구성하십시오.
</Callout>

**프로세스** 탭을 사용하여 호스트에서 실행 중인 프로세스에 대한 정보를 얻고 프로세스 메트릭에 대한 경고를 설정합니다. **프로세스** 탭에는 개별 프로세스 또는 프로세스 그룹에 대한 CPU 백분율, I/O 바이트 및 메모리 사용량과 같은 데이터가 표시됩니다.

이 페이지에는 표시된 측정항목과 동일한 시간 범위 내에서 발생하는 이벤트의 스냅샷을 제공하는 [이벤트 히트맵](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/events-heatmap-examine-patterns-time-range) 도 포함되어 있습니다. 이 정보를 사용하여:

* 성능 문제를 일으키는 프로세스를 정확히 찾아냅니다.
* CPU 백분율 및 메모리 사용량과 같은 조건에 대한 경고를 생성합니다.

**프로세스** 페이지에서 CPU 백분율은 호스트가 아닌 개별 프로세스로 범위가 지정됩니다. 이 때문에 CPU 백분율 메트릭은 전체 시스템의 리소스를 고려하지 않습니다. 대신 각 프로세스가 사용하는 단일 CPU 코어의 양을 보여줍니다.

**예시**

다음은 문제를 정확히 찾아내고 경고를 설정하는 방법의 예입니다.

1. [**호스트** 페이지](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-compute-page-measure-resource-usage) 에서 클러스터의 CPU 백분율이 정상보다 훨씬 높다는 것을 알 수 있습니다.
2. **프로세스** 페이지로 이동하여 해당 클러스터의 프로세스로 필터링합니다.
3. 동일한 프로세스의 여러 인스턴스에 과도한 CPU 백분율이 있음을 알 수 있습니다.
4. 문제를 수정한 후 해당 프로세스 유형에 대한 CPU 백분율이 특정 임계값을 초과할 때 트리거되는 경고 조건을 생성하기로 결정합니다.

**프로세스** 페이지 차트를 채우는 데 사용되는 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 에 대한 기술적인 설명은 [ProcessSample 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#processsample-attributes) 을 참조하십시오.

## 스토리지 탭

**저장소** 페이지에서는 장치의 전체 사용률, 디스크 사용량 또는 I/O 작업을 포함하여 리소스의 용량과 효율성을 모니터링할 수 있습니다. 이를 통해 다음을 수행할 수 있습니다.

* 예상치 못한 패턴을 조사하십시오. 예를 들어 I/O 작업을 균일하게 처리하지 않는 미러링된 시스템의 클러스터입니다.
* 디스크 공간이 완전히 소진되기 전에 사용량 수준을 모니터링하십시오.
* 하나 이상의 호스트에 문제가 있는 경우 알림 조건을 설정합니다. 예를 들어 읽기 또는 쓰기 작업을 위한 긴 처리 시간, 백분율 또는 바이트를 기반으로 한 디스크 가용성 또는 사용률 등
* 일반 처리 요청보다 높거나 낮은 호스트를 재배포할 위치에 대해 정보에 입각한 결정을 내립니다.
* 비상 사태가 발생하기 전에 데이터를 사용하여 인프라 예산에 대한 추가 또는 변경 계획을 세우십시오.

**스토리지** 페이지에는 표시된 메트릭과 동일한 시간 범위 내에서 발생하는 이벤트의 스냅샷을 제공하는 [**이벤트** 히트맵](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/events-heatmap-examine-patterns-time-range) 이 포함되어 있습니다.

스토리지 페이지 차트를 채우는 데 사용되는 [속성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 에 대한 기술적인 설명은 [StorageSample 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#storagesample-attributes) 을 참조하십시오.

## Docker 컨테이너 탭 [#containers-tab]

**Docker 컨테이너** 탭을 사용하여 호스트에서 실행 중인 컨테이너에 대한 정보를 얻고 컨테이너 지표에 대한 경고를 설정합니다. **Docker 컨테이너** 탭에는 개별 컨테이너 또는 컨테이너 그룹에 대한 CPU 백분율, I/O 바이트 및 메모리 사용량과 같은 데이터가 표시됩니다.

이 페이지에는 표시된 측정항목과 동일한 시간 범위 내에서 발생하는 이벤트의 스냅샷을 제공하는 [이벤트 히트맵](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/events-heatmap-examine-patterns-time-range) 도 포함되어 있습니다. 이 정보를 사용하여:

* 성능 문제가 있는 컨테이너를 식별합니다.
* CPU 백분율 및 메모리 사용량과 같은 조건에 대한 경고를 생성합니다.
