New Relic의 인프라 모니터링을 사용하면 현재 New Relic으로 모니터링 중인 컨텍스트 내에서 직접 [경고 조건](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-condition) 을 생성할 수 있습니다.

예를 들어 [필터 표시줄을](/docs/new-relic-solutions/new-relic-one/core-concepts/search-filter-entities) 사용하여 특정 호스트 집합으로 필터링했는데 문제가 발견되면 인프라 UI에서 경고를 생성할 수 있습니다. 경고를 생성하려면 저장된 필터 보기를 선택하고 보고 있는 차트에서 직접 경고 조건을 조정합니다. 이를 통해 환경에 대한 알림 시스템을 사전 예방적으로 관리하고 모니터링할 수 있습니다.

인프라 에이전트로 경보 조건을 시험해보고 싶으십니까? 무료 [로 New Relic 계정을 만드](https://newrelic.com/signup) 십시오! 신용 카드가 필요하지 않습니다.

## 인프라에 대한 경고 조건 생성 [#create-condition]

공지조건은 공지 [정책](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-policy) 에 적용됩니다. 기존 정책을 선택하거나 포털 UI 에서 공지 이메일을 사용하여 새 정책을 생성할 수 있습니다. 다른 유형의 [공지 채널을](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts) [](/docs/alerts/new-relic-alerts/configuring-alert-policies/name-or-rename-alert-policy)사용하려면 <InlinePopover type="alerts" />UI 내에서 새 정책을 만드세요.

<Callout id="rest-api-alert-limit" variant="important">
  전망 REST API 활성 <DNT>**and**</DNT> 비활성화 조건을 모두 포함하여 3,700개의 공지 조건으로 제한됩니다. 직접 사용하든 를 통해 API UI 사용하든 3,700개의 공지 조건 제한을 초과하는 추가 공지 조건을 추가하라는 모든 요청을 거부합니다.
</Callout>

알림 정책에 인프라 알림 조건을 추가하려면:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure**</DNT> 로 이동합니다. 공지하려는 차트 위에 마우스를 올려놓고, 줄임표를 선택하세요. <Icon name="fe-more-horizontal" /> 아이콘을 클릭한 다음 <DNT>**Create alert condition**</DNT> 선택하세요.

2. 의미 있는 조건 이름을 입력합니다.

3. <DNT>**Alert type**</DNT> 을 선택하거나 [예시를](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerting-examples) 참조하여 선택할 유형을 결정하세요.

4. 개별 필터를 만들거나 [엔터티 필터 표시줄](/docs/new-relic-solutions/new-relic-one/core-concepts/search-filter-entities) 에서 모든 필터를 복사하여 경고 조건에서 사용할 호스트를 식별합니다.

5. 공지 공지를 트리거하기 위해 <DNT>**Critical**</DNT> (필수) 및 <DNT>**Warning**</DNT> (선택 사항, 사용 가능한 경우) [값을](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-threshold) 정의합니다.

6. 선택사항: 조건 기준을 사전에 생성하지만 현재 공지 공지를 받지 않으려면 [<DNT>**Enabled**</DNT> 확인란 옵션을](/docs/infrastructure/new-relic-infrastructure/configuration/infrastructure-alerts-add-edit-or-view-host-alert-information#enabled) 선택 취소하세요.

7. 새 조건에 대한 기존 정책을 선택합니다.

   또는

   새 정책을 만들고 경고 알림에 대한 전자 메일을 식별하는 옵션을 선택합니다.

8. 선택 사항: [Runbook URL](#runbook-url-infrastructure) 을 추가합니다.

9. 선택 사항: 일정 시간 후 열림 알림 이벤트가 자동으로 닫히도록 [<DNT>**Close open alert events after**</DNT>](#alert-event-time-limit) 시간 제한을 설정합니다(인프라 조건의 경우 기본값은 24시간입니다).

10. <DNT>**Create**</DNT> 선택합니다.

<Callout variant="important">
  New Relic이 지난 60분 동안 클라우드 통합 서비스의 속성을 받지 못한 경우 이를 &quot;자동 속성&quot;이라고 하며 UI에서 경고 조건으로 사용할 수 없습니다. 이 상황에서 [API](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/rest-api-calls-new-relic-infrastructure-alerts/) 를 사용하여 자동 속성에 대한 경고 조건을 생성할 수 있습니다.
</Callout>

## 기타 인프라 경보 조건 방법 [#methods]

다음과 같은 다른 방법을 사용하여 인프라 경고 조건을 생성할 수도 있습니다.

<CollapserGroup>
  <Collapser className="freq-link" id="alerts-ui" title="알림 UI 사용">
    <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Alert policies &gt; New alert policy &gt; Create new condition**</DNT> 으로 이동한 다음 제품으로 <DNT>**Infrastructure**</DNT> 선택합니다.
  </Collapser>

  <Collapser className="freq-link" id="infrastructure-ui" title="인프라 UI 사용">
    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Hosts**</DNT> 으)로 이동합니다.
    2. 공지하려는 차트 위에 마우스를 올려놓고, 줄임표를 선택하세요. <Icon name="fe-more-horizontal" /> 아이콘을 클릭한 다음 <DNT>**Create alert**</DNT> 선택하세요.
  </Collapser>

  <Collapser className="freq-link" id="integration" title="통합을 위한 인프라 설정 사용">
    <Callout variant="tip">
      이 방법을 사용하여 인프라 통합에 대한 경고 조건을 생성합니다.
    </Callout>

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Settings &gt; Alerts**</DNT> 으로 이동한 다음 <DNT>**Create alert condition**</DNT> 클릭합니다.

    2. 경고 조건의 이름을 지정하고 설명합니다.

    3. <DNT>**Integrations**</DNT> 공지 유형을 클릭한 후 사용하려는 통합 데이터 소스를 선택하세요.

    4. 조건을 특정 속성으로 제한하려면 <DNT>**Filter entities**</DNT> 드롭다운 메뉴를 사용하세요.

    5. <DNT>**Define thresholds**</DNT> 드롭다운 메뉴를 사용하여 조건의 레버 값을 정의한 후 <DNT>**Create**</DNT> 클릭하세요.

       구성 설정은 선택 사항입니다. 나중에 언제든지 업데이트할 수 있습니다.
  </Collapser>
</CollapserGroup>

## 호스트 경고 이벤트 보기 [#view-alerts]

해당 정책의 [공지채널](#notification) 에 포함된 누구나 공지공지를 직접 수신할 수 있습니다. 또한 귀하의 뉴렐릭 계정에 대한 권한이 있는 사람은 누구나 사용자 인터페이스를 통해 피규어 공지 이벤트 및 개별 공지 이벤트를 볼 수 있습니다.

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Events**</DNT> 으)로 이동합니다.
2. 호스트 또는 시간 프레임을 변경하려면 검색 창, 엔티티 필터 막대 또는 시간 기능을 사용하십시오.
3. <DNT>**Events**</DNT> 목록에서 공지 이벤트를 선택합니다.
4. 해당 공지사항에 대한 [알림의 상세정보를 확인](/docs/alerts/alert-event-management/view-alert-event-details) 하시려면 해당 링크를 선택하세요.

## 호스트 경고 정보 업데이트 또는 삭제 [#maintain-alerts]

호스트 경고 정보를 편집, 비활성화(또는 다시 활성화) 또는 삭제하려면:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Settings &gt; Alerts**</DNT> 으)로 이동합니다.
2. 선택사항: 하나 이상의 공지 조건을 찾으려면 검색 창이나 <DNT>**Select all**</DNT> 확인란을 사용하세요.
3. 사용 가능한 기능 중에서 선택하세요 <Icon name="fe-edit-2" /> 편집, 비활성화, 활성화 또는 <Icon name="fe-trash-2" /> 선택한 조건을 삭제합니다.

## 설명 추가 [#description]

다음과 같은 공지 조건 유형에 대해 <DNT>**Description**</DNT> 필드를 사용할 수 있습니다.

* NRQL 조건: [NerdGraph API](/docs/alerts/alerts-nerdgraph/nerdgraph-examples/nerdgraph-api-alerts-nrql-conditions) 를 사용하여 설명을 추가합니다.
* 인프라 조건: UI 또는 REST API를 사용하여 설명을 추가합니다.

공지 조건의 <DNT>**Description**</DNT> 필드에 배치한 텍스트는 관련 공지 이벤트 및 공지로 다운스트림으로 전달됩니다. 설명은 다음과 같은 여러 용도로 사용될 수 있습니다.

* 경고 조건의 원인을 캡처합니다.
* 모니터링되는 신호를 정의합니다.
* 다음 단계를 정의합니다.
* 다운스트림 시스템에 메타데이터를 추가합니다.

템플릿 대체를 사용하여 연결된 공지 이벤트 이벤트의 속성 값을 삽입할 수 있습니다. 템플릿 형식은 `{{attributeName}}` 입니다. 설명을 만들 때 사용할 수 있는 속성은 [공지 이벤트 속성을](/docs/alerts/create-alert/condition-details/alert-event-attributes) 참조하세요.

사용 가능한 속성 중 하나는 특수 `{{tag.*}}` 속성입니다. 이 속성 접두사는 대상 신호에 포함된 태그 값 또는 대상 신호와 연결된 엔터티 태그에 액세스하는 데 사용됩니다.

귀하의 공지 이벤트와 관련된 분류 태그가 있는 경우, 분류 태그 이름을 사용하여 접근할 수 있습니다. 이것의 예로는 `{{tag.aws.awsRegion}}` 있습니다. 사용 가능한 간판 태그는 공지이벤트에 포함되어 표시되며, 공지이벤트 조회 시 표시됩니다.

이 필드의 최대 문자 크기는 4,000자입니다.

## Runbook URL 추가 또는 편집 [#runbook-url-infrastructure]

[공지 조건 생성 과정에는](#create-condition) 런북 지침에 대한 URL을 설정하는 옵션이 포함되어 있습니다. 이를 통해 공지 이벤트 처리 관련 정보 또는 표준 절차에 연결할 수 있습니다. 링크를 추가하거나 업데이트하기 전에 유효한 URL을 사용하고 있는지 확인하십시오.

경고 조건의 Runbook URL을 추가, 업데이트 또는 삭제하려면:

1. 공지 조건을 선택하고 <DNT>**Runbook URL**</DNT> 링크를 변경하세요.
2. 조건을 저장합니다.

저장하려면 URL이 유효한 URL이어야 합니다.

## 공지이벤트 기한 [#alert-event-time-limit]

공지 이벤트 시간 제한을 사용하면 공지 이벤트가 강제 종료될 기간을 정의할 수 있습니다. 기본적으로 공지 이벤트 시간 제한은 인프라 조건에 따라 24시간입니다.

공지 조건의 공지 이벤트 시간 제한을 추가하거나 업데이트하려면:

1. 공지사항을 선택하고, 공지이벤트 시간 제한을 변경하세요.
2. 조건을 저장합니다.

## 너무 긴 NRQL 쿼리를 생성하는 경고 조건 [#alert-nrql-error]

인프라에 대해 생성된 경고 조건은 비하인드 스토리 NRQL 쿼리에 의존하며 NRQL 쿼리에는 4096자 제한이 있습니다. 즉, 조건이 많은 요소(예: 많은 호스트 또는 많은 태그 포함)를 필터링하는 매우 복잡한 NRQL 쿼리를 생성하는 경우 이 제한을 초과하고 조건이 실패했다는 오류 메시지를 표시합니다.

이 문제를 해결하려면 경고 조건에서 사용 중인 요소의 수를 줄이십시오. 예를 들어:
