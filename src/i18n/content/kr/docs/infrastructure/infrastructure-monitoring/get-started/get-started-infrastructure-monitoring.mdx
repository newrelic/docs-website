뉴렐릭을 사용하면 소프트웨어 인프라를 모니터링하여 안정적인 시스템을 제공할 수 있습니다. 인프라 모니터링을 사용하면, 클라우드, 전용 호스트, 오케스트레이션된 환경에서 실행되는 컨테이너 등 모든 환경에서 상태와 성능 결과를 개선할 수 있습니다.

## 하나의 에이전트, 다양한 기능 [#overview]

인프라 모니터링 기능에 액세스하려면 인프라 에이전트가 필요합니다. 설치된 에이전트는 온프레미스 또는 가상화된 호스트의 시스템 리소스 및 프로세스에 대한 성능 데이터와 상태 데이터를 수집합니다. 여기에서, 뉴렐릭이 지원하는 [온호스트 통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) 중 하나와 페어링하거나 [로그를 전달](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent)하도록 에이전트를 구성할 수 있습니다.

<img
  title="Agents and New Relic"
  alt="Agents and New Relic"
  src="/images/infrastructure_diagram_angets-and-nr.webp"
/>

운영 팀은 인프라 모니터링을 사용해 단일 데이터센터는 물론 수천 개의 Amazon, Google Cloud 또는 Azure 인스턴스와 복잡한 하이브리드 시스템까지 심층적인 가시성을 확보할 수 있습니다. 아키텍처와 상관 없이 뉴렐릭의 인프라 모니터링 기능은 시스템 문제를 해결하고 개선하는 데 도움을 줍니다. 다음은 뉴렐릭의 몇 가지 솔루션입니다.

* Linux, macOS 및 Windows 운영 체제용 [인프라 에이전트](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent)
* [쿠버네티스](/install/kubernetes) 및 [Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/get-started-auto-telemetry-pixie) 통합
* Amazon, Azure 및 Google Cloud Platform용 [클라우드 통합](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations/#cloud-integrations)
* [Prometheus 통합](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* MySQL, NGINX, Cassandra, Kafka 등 다양한 서비스에 대한 [온호스트 통합](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations/#on-host-integrations) 온호스트 통합의 대부분은 인프라 에이전트와 함께 설치되고 데이터 전송을 위해 함께 작동합니다.

## 실행 가능한 데이터로 MTTR 감소 [#why-it-matters]

실시간 메트릭과 분석은 호스트 성능의 변경 사항을 설정 변경 사항에 연결하여 [평균 해결 시간](https://newrelic.com/devops/how-to-reduce-mttr)(MTTR)을 줄여줍니다. 다음은 인프라 데이터가 어떻게 도움이 되는지 보여주는 예입니다.

<DNT>**Inventory**</DNT> 페이지는 자산에 포함된 모든 인프라 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic)에 중점을 둡니다.

<img
  title="New Relic - Infrastructure monitoring - Inventory"
  alt="New Relic - Infrastructure monitoring - Inventory"
  src="/images/infrastructure_screenshot-full_inventory-page.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com](https://one.newrelic.com) > All Capabilities > Infrastructure monitoring > Inventory**</DNT>: 특정 패키지, 구성 또는 시작 스크립트가 포함된 호스트를 정확히 찾기 위해 전체 시스템을 검색합니다.
</figcaption>

<DNT>**Events**</DNT> 페이지에서 구성 변경, 재시작, SSH 세션 및 기타 [주요 이벤트](/docs/infrastructure-events-page#types)의 변경 사항을 추적할 수 있습니다. 실시간 피드는 전체 인프라에 대한 변경 로그를 제공합니다.

뉴렐릭의 솔루션은 데이터를 [안전하게](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-security) 수집하고 표시하여 실시간 모니터링을 지원합니다.

## 인프라 에이전트 설치 [#install-infra-agent]

인프라 모니터링에 액세스하려면 먼저 에이전트를 설치해야 합니다. Linux, macOS 및 Windows 시스템에 인프라 에이전트를 설치할 수 있습니다. 고유한 옵저빌리버티 요구 사항에 맞게 시스템에 에이전트를 몇 가지 방법으로 구현할 수 있습니다. [인프라 에이전트 설치 방법 선택](/docs/infrastructure/infrastructure-monitoring/get-started/choose-infra-install-method)을 검토하여 사용 사례에 적합한 옵션을 결정하시기 바랍니다.

대부분의 솔루션은 인프라 에이전트를 먼저 설치하고 데이터를 전송하는 방식으로 작동합니다.

## 뉴렐릭의 인프라 서비스 통합 [#aws]

뉴렐릭의 인프라 통합은 보편적인 서비스와 시스템의 메트릭에 액세스할 수 있게 해줍니다. 통합에는 다음이 포함됩니다.

* 클라우드 플랫폼 통합: [Amazon](/docs/infrastructure/amazon-integrations/aws-integrations-list), [Google Cloud Platform](/docs/integrations/google-cloud-platform-integrations) 및 [Microsoft Azure](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list)
* 뉴렐릭의 [쿠버네티스 통합](/install/kubernetes) 및 [Pixie 통합](/docs/kubernetes-pixie/auto-telemetry-pixie/get-started-auto-telemetry-pixie)
* NGINX, Cassandra, MySQL 등과 같은 [온호스트 통합](/docs/infrastructure/host-integrations/host-integrations-list)은 인프라 에이전트와 함께 작동합니다.
* [뉴렐릭 Flex](/docs/introduction-new-relics-flex-integration)를 사용해 자체적인 통합을 구축할 수 있습니다.

## 인프라 및 앱 데이터에 대한 로그 보기 [#logs-context]

로그와 인프라 데이터를 함께 가져와 더 쉽고 빠르게 문제를 해결하세요! 문맥적 로그를 사용하여 쿠버네티스 클러스터 같은 인프라 데이터의 로그를 볼 수 있습니다. 필요한 만큼 구성 파일을 추가할 수 있습니다. 이 파일들은 로그 메타데이터를 뉴렐릭 플랫폼으로 푸시하는 소스 역할을 합니다.

또한 오류 및 트레이스와 관련된 로그 메시지를 앱이나 호스트의 UI 에서 직접 확인할 수도 있습니다. 자세한 내용은 [문맥적 APM 로그](/docs/apm/new-relic-apm/getting-started/get-started-logs-context)를 참조하십시오.

사용 중인 <InlinePopover type="apm"/>에이전트가 아직 컨텍스트 솔루션의 자동 로그를 지원하지 않는 경우, 컨텍스트 솔루션의 수동 로그를 계속 사용하고 인프라 에이전트 또는 기타 [지원되는 서드파티 로그 전달자를](/docs/logs/forward-logs/enable-log-management-new-relic) [통해 로그를 전달](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)할 수 있습니다.
