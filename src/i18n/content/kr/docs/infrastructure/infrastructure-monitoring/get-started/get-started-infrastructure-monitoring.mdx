뉴렐릭은 클라우드 또는 전용 호스트에서 실행되는 서비스에서 오케스트레이션된 환경에서 실행되는 컨테이너에 이르기까지 전체 인프라에 대한 유연하고 동적인 옵저버빌리티를 제공합니다. 모든 호스트의 상태와 성능을 애플리케이션 컨텍스트, 로그 및 구성 변경에 연결할 수 있습니다.

인프라 모니터링을 통해 최신 운영 팀은 데이터 센터에서 수천 개의 Amazon, Google Cloud 또는 Azure 인스턴스에 이르기까지 복잡한 하이브리드 시스템에 대한 완전한 옵저버빌리티를 확보할 수 있습니다.

인프라 모니터링 솔루션과 시작하는 방법에 대해 보다 자세히 알아보겠습니다. 아직 계정이 없으시면, 아래에서 무료 뉴렐릭 계정을 생성해 지금 바로 데이터 모니터링을 시작하십시오.

## 뉴렐릭의 인프라 모니터링 솔루션 [#infrastructure-solutions]

뉴렐릭이 보유한 광범위한 인프라 모니터링 솔루션에 대한 개괄적인 정보입니다.

* Linux, macOS 및 Windows 운영 체제용 [인프라 에이전트](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent)
* [쿠버네티스](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure) 및 [Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/get-started-auto-telemetry-pixie) 통합
* Amazon, Azure 및 Google Cloud Platform용 [클라우드 통합](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations/#cloud-integrations)
* [Prometheus 통합](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* MySQL, NGINX, Cassandra, Kafka 등 다양한 서비스에 대한 [온호스트 통합](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations/#on-host-integrations) 온호스트 통합의 대부분은 인프라 에이전트와 함께 설치되고 데이터 전송을 위해 함께 작동합니다.

## 실행 가능한 데이터로 MTTR 감소 [#why-it-matters]

실시간 메트릭과 분석은 호스트 성능의 변경 사항을 구성 변경 사항과 연결하여 [평균 해결 시간](https://newrelic.com/devops/how-to-reduce-mttr)(MTTR)을 줄여줍니다. **인벤토리** 페이지에서 전체 자산을 검색하여 특정 패키지, 구성 또는 시작 스크립트가 포함된 호스트를 정확히 찾을 수 있습니다.

<img
  title="New Relic - Infrastructure monitoring - Inventory"
  alt="New Relic - Infrastructure monitoring - Inventory"
  src={infrastructureInventory}
/>

<figcaption>
  **인벤토리** 페이지는 자산의 모든 인프라 [엔터티](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic)에 중점을 둡니다.
</figcaption>

**이벤트** 페이지에서 구성 변경, 재시작, SSH 세션 및 기타 [주요 이벤트](/docs/infrastructure-events-page#types)의 변경 사항을 추적할 수 있습니다. 실시간 피드는 전체 인프라에 대한 변경 로그를 제공합니다. 뉴렐릭의 솔루션은 5초 내에 데이터를 [안전하게](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-security) 수집하고 표시하므로 모니터링이 실시간에 가깝습니다.

  title="New Relic - Infrastructure monitoring - Events"
  alt="New Relic - Infrastructure monitoring - Events"
  src={infrastructureEvents}
/>

<figcaption>
  **이벤트** 페이지에는 호스트에서 일어나는 모든 일에 대한 실시간 피드가 포함되어 있습니다.
</figcaption>

인프라 모니터링 툴을 사용하여 다음을 수행할 수도 있습니다.

* [데이터를 쿼리하여](/docs/using-new-relic/data/understand-data/query-new-relic-data) [인프라 이벤트](/docs/default-infrastructure-events-attributes)에 대해 자세히 분석하고 팀과 공유할 수 있는 [커스텀 대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards)를 구축합니다.
* [인프라 모니터링과 APM 데이터의 연결](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure) 덕분에, 서버 측이든 애플리케이션 측이든 문제가 발생한 위치에 관계없이 성능 문제를 해결할 수 있습니다.
* 관련 인프라 차트에서 직접 [알림 설정을 생성, 확인 또는 업데이트할 수 있습니다](/docs/add-edit-or-view-host-alert-information). 예를 들어 알람 조건을[ 보고하지 않는 호스트](/docs/infrastructure/new-relic-infrastructure/configuration/create-infrastructure-host-not-reporting-condition)를 생성할 수 있습니다.

## 인프라 및 앱 데이터에 대한 로그 보기 [#logs-context]

로그와 인프라 데이터를 함께 가져와 더 쉽고 빠르게 문제를 해결하세요! 문맥적 로그를 사용하여 쿠버네티스 클러스터 같은 인프라 데이터의 로그를 볼 수 있습니다. 필요한 만큼 구성 파일을 추가할 수 있습니다. 이 파일들은 로그 메타데이터를 뉴렐릭 플랫폼으로 푸시하는 소스 역할을 합니다.

앱 또는 호스트의 UI에서 직접 오류 및 트레이스와 관련된 로그 메시지를 볼 수도 있습니다. 뉴렐릭에서 다른 UI 페이지로 전환할 필요가 없습니다! 보다 자세한 내용은 [문맥적 APM 로그](/docs/apm/new-relic-apm/getting-started/get-started-logs-context)를 참조하세요.

사용 중인 <InlinePopover type="apm"/>에이전트가 뉴렐릭의 자동 문맥적 로그 솔루션을 아직 지원하지 않지만, 수동 문맥적 로그 솔루션을 계속 사용하며 인프라 에이전트 또는 [지원되는 다른 서드파티 로그 전달자](/docs/logs/forward-logs/enable-log-management-new-relic)[를 통해 로그를 전달할](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)수 있습니다.

## 인프라 에이전트 설치 [#install]

[안내 설치](#quick)를 사용하지 않고 인프라 에이전트를 설치하는 가장 간단한 방법 [은 패키지 매니저](/docs/infrastructure/install-configure-manage-infrastructure/linux-installation/install-infrastructure-linux-using-package-manager)(Linux) 또는 [MSI 인스톨러](/docs/infrastructure/install-configure-manage-infrastructure/windows-installation/install-infrastructure-windows-server-using-msi-installer)(Windows)를 사용하는 것입니다.

아직 뉴렐릭 계정이 없거나 단계별 절차를 따르길 원하시면, [Linux](/docs/infrastructure/install-configure-manage-infrastructure/linux-installation/install-infrastructure-linux-using-package-manager) \| [Windows](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server) \| [Elastic Beanstalk](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-aws-elastic-beanstalk) \| [Ansible](/docs/infrastructure/new-relic-infrastructure/config-management-tools/configure-new-relic-infrastructure-using-ansible) \| [Chef](/docs/infrastructure/new-relic-infrastructure/config-management-tools/configure-new-relic-infrastructure-using-chef) \| [Puppet](/docs/infrastructure/new-relic-infrastructure/config-management-tools/configure-new-relic-infrastructure-puppet)용 에이전트 설치에 대한 튜토리얼을 참조하세요.

## 뉴렐릭의 인프라 서비스 통합 [#aws]

뉴렐릭의 인프라 통합은 보편적인 서비스와 시스템의 메트릭에 액세스할 수 있게 해줍니다. 통합에는 다음이 포함됩니다.

* 클라우드 플랫폼 통합: [Amazon](/docs/infrastructure/amazon-integrations/aws-integrations-list), [Google Cloud Platform](/docs/integrations/google-cloud-platform-integrations) 및 [Microsoft Azure](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list)
* 뉴렐릭의 [쿠버네티스 통합](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure) 및 [Pixie 통합](/docs/kubernetes-pixie/auto-telemetry-pixie/get-started-auto-telemetry-pixie)
* NGINX, Cassandra, MySQL 등과 같은 [온호스트 통합](/docs/infrastructure/host-integrations/host-integrations-list)은 인프라 에이전트와 함께 작동합니다.
* [뉴렐릭 Flex](/docs/introduction-new-relics-flex-integration)를 사용해 자체적인 통합을 구축할 수 있습니다.
