ECS 통합은 [Amazon ECS](https://docs.aws.amazon.com/ecs/index.html) 환경의 성능 데이터를 보고하고 표시합니다. ECS 통합은 다른 통합과 잘 작동하므로 [ECS에서 실행되는 서비스를 모니터링](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs) 할 수도 있습니다.

## 특징 [#features]

[Amazon Elastic Container Service](https://docs.aws.amazon.com/ecs/index.html) (ECS)는 다음에서 Docker 컨테이너를 쉽게 실행, 중지 및 관리할 수 있는 확장 가능한 컨테이너 관리 서비스입니다.

* Amazon EC2 클러스터
* [AWS 파게이트](https://aws.amazon.com/fargate/)
* [ECS Anywhere](https://aws.amazon.com/ecs/anywhere/)

ECS 통합은 ECS 객체의 메트릭을 보고하여 기본 컨테이너 인스턴스와 컨테이너 계층을 계측합니다. 통합을 통해 ECS 인스턴스, 작업, 서비스 및 컨테이너에 대한 통찰력을 얻을 수 있습니다.

기능은 다음과 같습니다.

* ECS 환경에 대한 즉각적인 통찰력을 위해 사전 구축된 대시보드에서 데이터를 봅니다.
* 자동으로 보고된 데이터에서 [쿼리 빌더](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 에서 고유한 쿼리와 차트를 만듭니다.
* ECS 데이터에 대한 [경고 조건](/docs/ecs-integration-recommended-alert-conditions) 을 생성합니다.
* [New Relic Explorer](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer-view-performance-across-apps-services-hosts) 를 사용하여 엔티티를 탐색하십시오.

<Callout variant="important">
  New Relic은 또한 호스트 내 통합과 다른 데이터 세트를 보고하는 [ECS 클라우드 통합](/docs/integrations/amazon-integrations/aws-integrations-list/aws-ecsecr-monitoring-integration) 을 제공합니다. 완전한 ECS 모니터링을 위해 두 통합을 모두 활성화하는 것이 좋습니다.
</Callout>

## 호환성 및 요구 사항 [#requirements]

요구 사항:

* Amazon ECS 컨테이너 에이전트 1.21.0 이상.
* 윈도우는 지원하지 않습니다.
* 이 통합은 [인프라 에이전트](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure) 및 [Docker 계측](/docs/infrastructure/install-configure-manage-infrastructure/docker-installation/docker-instrumentation-infrastructure) 을 사용합니다. 해당 시스템의 적용 가능한 요구 사항 및 제한 사항이 적용됩니다.

## 설치 [#install]

설치하려면 [설치 통합 문서](/docs/install-ecs-integration) 를 참조하십시오.

## 소스 코드 확인 [#source-code]

이 통합은 오픈 소스 소프트웨어입니다. 즉 [, 소스 코드를 탐색하고](https://github.com/newrelic/nri-ecs "링크가 새 창에서 열립니다.") 개선 사항을 보내거나 고유한 포크를 만들어 빌드할 수 있습니다.
