이 문서에서는 VMware Tanzu용 뉴렐릭 .NET 확장 빌드팩을 설명하고 타일 설치 방법에 대한 지침을 제공합니다. 타일은 뉴렐릭 에이전트를 ..NET Core 또는 .NET 프레임워크 앱에 바인딩하여 VMware Tanzu 환경에서 모니터링할 수 있습니다.

VMware Tanzu용 뉴렐릭 .NET 확장 빌드팩을 사용하면 ..NET Core 및 프레임워크) 앱을 뉴렐릭 .NET 에이전트에 바인딩할 수 있습니다. 이를 통해 이러한 앱의 상태와 성능을 모니터링하고, 에이전트에서 캡처한 데이터를 분석하고, 추가로 캡처된 에이전트 데이터를 VMware Tanzu 콘솔 지표 및 [뉴렐릭 Firehose Nozzle](https://support.broadcom.com/group/ecx/productdownloads?subfamily=New%20Relic%20Nozzle%20for%20VMware%20Tanzu) 에서 수집한 이벤트와 상호 연관시킬 수 있습니다.

## 전제 조건 [#prereqs]

이 제품은 테스트를 거쳤으며 v3.0 이하의 VMware Tanzu OpsManager 버전 및 Tanzu Application Service 5.0과 호환됩니다.

뉴렐릭 VMware Tanzu용 .NET 확장 빌드팩에는 다음이 필요합니다.

* 라이센스 키가 있는 활성 New Relic 계정. 이는 .NET 앱을 New Relic .NET 에이전트에 바인딩하는 데 사용됩니다.
* 앱 매니페스트에서 다중 빌드팩을 사용하려면 cf CLI v6.38 이상이 필요합니다. 매니페스트에 빌드팩을 추가하는 방법에 대한 일반적인 정보는 Cloud Foundry 설명서에서 [여러 빌드팩을 사용하여 애플리케이션 푸시를](https://docs.pivotal.io/application-service/buildpacks/use-multiple-buildpacks.html) 참조하세요.
* .NET Core 확장을 사용하려면 dotnet core buildpack 2.1.5가 필요합니다. 또는 나중에.
* .NET HWC 확장을 사용하려면 HWC 빌드팩 3.0.3이 필요합니다. 또는 나중에.
* Broadcom의 HWC builpack [알려진 문제 및 해결 방법](https://knowledge.broadcom.com/external/article?articleNumber=408736)

## 설치 [#install]

VMware Tanzu용 뉴렐릭 .NET 확장 빌드팩은 Ops Manager의 타일을 통해 설치할 수 있습니다. 또는 cf 명령줄 인터페이스(CLI) 명령 `cf create-buildpack` 을 사용하여 `.Pivotal` 파일을 추출하고 개별 확장 빌드팩을 설치할 수 있습니다.

앱 모니터링을 시작한 후 뉴렐릭 경고 하위 시스템을 사용하여 .NET 에이전트에서 수집한 지표를 기반으로 <InlinePopover type="alerts" />설정할 수 있습니다.

VMware Tanzu용 뉴렐릭 .NET 확장 빌드팩은 타일 설정에 따라 다음 빌드팩 중 하나 이상을 설치합니다(총 8개의 확장 빌드팩).

* 뉴렐릭 Ubuntu Jammy 1.\* 그리드에서 실행되는 ..NET Core 전용 ..NET Core 확장 빌드팩입니다. 이 확장 빌드팩은 캐시되지 않습니다.
* 뉴렐릭 연결이 끊긴(격리된) VMware Tanzu 배포의 Ubuntu Jammy 1.\*에서 실행되는 ..NET Core 확장용 ..NET Core 확장 캐시 빌드팩입니다. 이는 외부 세계에 접근할 수 없는 Air-Gapped 환경을 지원하기 위한 것입니다.
* Windows 2019 그리드에서 실행되는 .NET 프레임워크용 HWC 확장 빌드팩 3개. 이 확장 빌드팩은 캐시되지 않습니다.
* 3 뉴렐릭 연결이 끊긴(격리된) VMware Tanzu 배포의 Windows 2019 그리드에서 실행되는 .NET 프레임워크용 HWC 확장 캐시 빌드팩. 이는 외부 세계에 접근할 수 없는 Air-Gapped 환경을 지원하기 위한 것입니다.

모든 빌드팩은 Cloud Foundry의 다중 빌드팩 접근 방식을 사용하며 표준 .NET Core 빌드팩 또는 HWC 빌드팩을 앱 매니페스트 또는 `cf push` 명령줄에서 빌드팩 체인의 마지막 빌드팩으로 지정해야 합니다.

<Callout variant="important">
  .NET Core 및 .NET Framework 모두에 대한 이 확장 빌드팩의 캐시된 버전에는 New Relic .NET Agents 버전이 포함되어 있습니다. `10.43.x`
</Callout>
