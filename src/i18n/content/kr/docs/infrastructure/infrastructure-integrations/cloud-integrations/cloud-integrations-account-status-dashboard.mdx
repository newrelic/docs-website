New Relic 인프라 모니터링에 연결하는 각 클라우드 계정(예: [AWS](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) 또는 [Azure](/docs/integrations/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations) )에는 당사 플랫폼이 해당 클라우드 서비스 공급자의 데이터를 처리하는 방법을 보여주는 **계정 상태 대시보드** 가 있습니다.

## 중요한 이유 [#reasons-to-use]

이 대시보드를 모니터링하는 이유는 다음과 같습니다.

* **API 호출 사용을 이해합니다.** 클라우드 API를 사용하여 메트릭 및 인벤토리 데이터를 수집합니다. 대시보드에는 해당 API 호출의 수와 빈도가 표시됩니다. API 호출에 비용이 들 수 있고 API 사용량이 특정 사용량 수준에서 제한될 수 있는 두 가지 이유로 이를 모니터링할 수 있습니다. 사용량을 줄이려면 모니터링하는 구성 요소의 수 또는 모니터링 빈도를 줄이도록 [통합 설정을 구성](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 할 수 있습니다.

* **누락된 데이터 또는 기타 데이터 문제를 해결합니다.** 대시보드는 다음을 포함하여 통합 보고 데이터에 영향을 주는 정보를 표시합니다.

  * 권한 오류 또는 할당량 소진 오류와 같이 New Relic 수집 데이터에 영향을 주는 오류입니다.
  * 폴링 간격 변경 또는 태그 수집이 비활성화 또는 활성화되는 것과 같은 통합 구성 변경.

## 대시보드 데이터 이해 [#understand-data]

계정 상태 대시보드에 표시되는 특정 데이터는 클라우드 서비스 제공업체에 따라 다릅니다. 일반적인 차트에는 다음이 포함됩니다.

* **데이터 업데이트** : 측정항목 데이터 또는 인벤토리 데이터에 대한 업데이트를 표시합니다(업데이트가 `1` 값으로 표시됨).
* **계정 변경** : 통합 작동 방식에 영향을 주는 작업입니다. 예: 클라우드 계정 이름 변경 또는 연결 해제, 폴링 간격 변경 및 기타 구성 옵션.
* **데이터 신선도** : 각 통합에 대해 수집된 마지막 데이터 포인트의 타임스탬프입니다.
* **가져오기 오류** : 데이터 수집에 문제가 있음을 나타냅니다. (클라우드 제공자 측의 문제일 수 있습니다.)

차트 데이터를 더 잘 이해하기 위한 옵션은 다음과 같습니다.

* 차트 위로 마우스를 가져갑니다.

  <Icon name="fe-help-circle"/>

  아이콘을 클릭하면 차트 설명이 표시됩니다(사용 가능한 경우).

* [차트의 기본 NRQL 쿼리를 봅니다](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform).

## 계정 상태 대시보드 찾기 [#find-dashboard]

클라우드 서비스 공급자의 계정 상태 대시보드를 찾으려면:

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure** 에서 클라우드 서비스 공급자(예: **AWS**)를 선택합니다.
2. 클라우드 계정에 대한 **계정 상태 대시보드** 를 선택합니다.
