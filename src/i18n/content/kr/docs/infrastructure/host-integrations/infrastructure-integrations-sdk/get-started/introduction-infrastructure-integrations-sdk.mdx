New Relic을 사용하여 고유한 통합을 생성할 수 있는 몇 가지 방법이 있습니다.

* 일반 원격 측정(메트릭, 추적) 솔루션: [원격 측정 SDK](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic) 를 사용합니다.
* 자체 JavaScript UI 기능을 사용하는 맞춤형 [New Relic 앱](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application) 을 빌드하십시오.
* New Relic Infrastructure가 있는 경우 경량 [Flex 통합 도구](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) (권장)를 사용하거나 Integrations SDK를 사용하여 완전한 인프라 [온-호스트 통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) 을 구축할 수 있습니다.

<Callout variant="important">
  New Relic은 향후 온-호스트 통합을 위해 Prometheus와 같은 오픈 소스 표준에 의존하도록 전환하고 있습니다. 인프라 SDK는 이러한 전환의 기반이지만 이 SDK와 관련된 일부 자습서 및 도구는 최신 개발 사항에 따라 최신 상태가 아닐 수 있습니다.
</Callout>

## 통합 SDK란 무엇입니까? [#what-is-it]

인프라 통합 SDK를 사용하면 호스트 또는 서비스의 사용자 지정 데이터를 보고하는 [호스트 내 통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) 을 구축할 수 있습니다. 그런 다음 해당 데이터는 New Relic Infrastructure에서 찾을 수 있으며 사용자 지정 쿼리 및 차트를 만드는 데 사용할 수 있습니다.

## 호스트 내 통합으로 어떤 데이터를 보고할 수 있습니까? [#data-types]

Integrations SDK를 사용하여 통합을 빌드할 때 데이터를 보고하는 **엔터티** 를 할당합니다. New Relic 용어에서 **엔티티** 는 New Relic에 데이터를 보고하는 것을 의미하는 의도적으로 모호한 용어입니다(예: 로컬 호스트, 로드 밸런서 또는 데이터베이스).

단일 통합은 여러 엔터티의 데이터를 보고할 수 있으므로 둘 이상의 서비스 또는 호스트 인스턴스에서 데이터를 보고할 수 있습니다.

엔터티가 생성할 수 있는 데이터에는 세 가지 유형이 있습니다.

* **메트릭스** : 메트릭 데이터는 수치 측정 데이터에 사용됩니다. 예: 대기열에 있는 요청 수 또는 분당 데이터베이스 적중 수. 사용자 지정 통합의 메트릭 데이터를 쿼리하고 대시보드를 만드는 데 사용할 수 있습니다.
* **인벤토리** : 라이브 시스템 상태 및 구성 정보. 이 데이터는 인프라 [인벤토리 UI 페이지](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-inventory-page-search-your-entire-infrastructure) 에 표시됩니다.
* **이벤트** : 이벤트는 시스템의 중요한 활동을 기록하는 데 사용됩니다. 예: 서비스 시작 또는 생성 중인 새 테이블. 이벤트 데이터는 인프라 [이벤트 UI 페이지](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page-live-feed-every-config-change) 에 표시됩니다.

## 사용자 지정 통합 만들기 [#create]

통합 SDK를 사용하여 통합을 생성하려면 다음 리소스를 사용하세요.

* [Go 언어 빌드 도구 및 자습서를](/docs/infrastructure/integrations-sdk/get-started/infrastructure-integration-building-resources) 참조하세요. 이 튜토리얼은 Go에서 Redis 통합을 만드는 과정을 안내합니다. (참고: Go는 필수가 아니며 추가 빌드 도구를 제공하는 언어일 뿐입니다. 자세한 내용은 [통합 SDK 요구 사항](/docs/integrations/integrations-sdk/getting-started/compatibility-requirements-infrastructure-integrations-sdk) 을 참조하십시오.)
* 통합 생성에 필요한 파일을 설명하는 [통합 파일 구조](/docs/infrastructure/integrations-sdk/get-started/create-activate-infrastructure-integration) 문서를 참조하십시오.
