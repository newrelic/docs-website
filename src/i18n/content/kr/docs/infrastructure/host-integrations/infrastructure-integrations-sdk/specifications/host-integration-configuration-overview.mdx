온 [호스트 통합은](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations) 인프라 에이전트로 데이터를 보내고, 인프라 에이전트는 해당 데이터를 New Relic으로 보냅니다. 통합이 에이전트와 상호 작용하는 방식은 각 통합의 구성에 의해 제어됩니다.

이것이 적용되는 통합에 대한 자세한 내용은 [온호스트 통합 설치를](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations) 참조하십시오.

## 구성 작동 방식 개요 [#overview]

호스트 내 통합 중 일부는 [인프라 에이전트](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent) 에 의해 실행되는 외부 프로그램입니다. 각 통합은 [특정 서비스를](/docs/integrations/host-integrations/host-integrations-list) 모니터링합니다. 통합에는 최소한 다음 파일이 포함됩니다.

* 에이전트가 예상하는 JSON 형식의 다양한 [데이터 유형](/docs/integrations/new-relic-integrations/get-started/introduction-infrastructure-integrations#data-types) 을 내보내는 [실행 파일](/docs/infrastructure/infrastructure-integrations/custom-integrations/create-integration-executable-file)
* 하나 이상의 YAML 형식 구성 파일(예: [Apache 통합 구성)](/docs/integrations/host-integrations/host-integrations-list/kafka-monitoring-integration#config) . (형식 문제를 피하기 위해 사용하기 전에 YAML 구성 파일을 [린트](http://www.yamllint.com/) 하는 것이 좋습니다.)

특정 온-호스트 통합의 구성 외에도 [인프라 에이전트의 구성](/docs/infrastructure/install-infrastructure-agent/configuration/configure-infrastructure-agent) 을 편집할 수도 있습니다.

## 구성 파일 위치 [#file-location]

표준 온-호스트 통합 설치의 경우 구성은 인프라 에이전트의 디렉토리에 있습니다. 에이전트 [는 자체 구성 파일의 설정에](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#plugin-dir) 따라 이 구성 위치를 결정합니다.

일부 구현의 경우 통합 구성이 다른 위치에 있습니다. 예를 들어:

* [Kubernetes](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes) 에서 실행되는 서비스: 구성은 Kubernetes 통합 구성 파일에 있습니다.
* [Amazon ECS](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs) 에서 실행되는 서비스: 구성은 AWS 콘솔에 배치됩니다.

## 구성 형식 [#config-formats]

호스트 내 통합은 두 가지 구성 형식을 사용합니다.

* <DNT>
    **Standard**
  </DNT>

  : 2019년 12월부터 [인프라 에이전트 버전 1.8.0은](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) 일부 통합에서 사용되는 새로운 형식을 지원하기 시작했습니다. 이 형식은 단일 설정 파일을 사용하며 기타 개선 사항을 제공합니다. 자세한 내용은 [표준 설정 을](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-standard-configuration-format) 참조하세요.

* <DNT>
    **Legacy**
  </DNT>

  : 대부분의 전체호스트 통합에서 사용하는 형식입니다. 이 설정은 정의 파일과 설정 파일이라는 두 가지 파일을 사용합니다. 자세한 내용은 [구형 설정을](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-legacy-configuration-format/) 참조하세요.
