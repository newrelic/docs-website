New Relic Infrastructure [Integrations SDK](/docs/infrastructure/integrations/intro-infrastructure-integrations) 를 사용하면 맞춤형 호스트 및 서비스 데이터를 보고하기 위한 [온-호스트 통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) 을 생성할 수 있습니다. 이 문서에서는 호스트 통합에 필요한 파일을 설명합니다. New Relic은 [Go 언어 통합 구축 도구와 튜토리얼](/docs/infrastructure/integrations-sdk/tools-resources/infrastructure-integration-building-resources) 도 제공합니다.

## 통합 파일 [#overview]

호스트 내 통합을 구축하기 전에 [호환성 및 요구 사항](/docs/integrations/integrations-sdk/getting-started/compatibility-requirements-infrastructure-integrations-sdk) 을 충족하는지 확인하십시오.

통합에는 최소한 다음 파일이 필요합니다.

* 인프라 에이전트에서 예상하는 형식으로 JSON 데이터를 내보내는 모든 언어로 작성된 [실행 파일](/docs/infrastructure/infrastructure-integrations/custom-integrations/create-integration-executable-file)
* [구성 파일](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-standard-configuration-format)

Go 언어 빌드 도구 및 이러한 파일을 만들기 위한 자습서는 [빌드 리소스](/docs/integrations/integrations-sdk/getting-started/integrations-tutorials-build-resources) 를 참조하세요.
