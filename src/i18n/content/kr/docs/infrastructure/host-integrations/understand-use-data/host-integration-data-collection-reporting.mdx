[New Relic 온-호스트 통합이](/docs/infrastructure/host-integrations/getting-started/introduction-host-integrations) 어떻게 데이터를 수집하고 New Relic에 보고하는지 알아보려면 계속 읽으십시오.

## 데이터 수집 및 보고 프로세스 [#data-collection]

호스트 내 인프라 통합이 New Relic에 데이터를 보내는 방법은 다음과 같습니다.

1. 시작 시 인프라 에이전트는 통합의 정의 파일이 포함된 디렉토리를 스캔합니다.
2. 인프라 에이전트는 정의 파일에 정의된 모든 통합 실행 파일을 등록합니다.
3. 에이전트는 통합 구성 파일에 대한 전용 디렉터리를 검색합니다.
4. 이러한 구성 파일이 인프라 에이전트에 등록된 통합을 지정하는 경우 에이전트는 통합을 설정하고 일정을 잡습니다.
5. 예약된 간격(기본값은 15초)에 에이전트는 통합에서 데이터를 수집하고 전송을 위해 준비합니다. 60초마다 해당 데이터를 다른 인프라 데이터와 함께 New Relic으로 보냅니다.
6. 성공적인 수집 통과 후 통합 실행 파일이 종료됩니다.

## 파일 구조 및 사양 [#file-structure]

New Relic on-host 통합의 파일 구조를 이해하면 통합을 사용자 정의하고, [데이터를 이해 및 사용하고](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) , 문제를 해결하는 데 도움이 될 수 있습니다.

호스트 내 통합은 일련의 오픈 소스 사양을 준수하므로 누구나 호스트 내 통합 인프라를 구축할 수 있습니다. 이러한 파일 사양에 대한 설명은 [파일](/docs/integrations/integrations-sdk/file-specifications) 사양 을 참조하십시오.
