## 문제

인프라 에이전트에 구성 매개변수가 정의되어 있고 해당 매개변수를 [온호스트 통합](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations) 중 하나로 전달하려고 합니다.

## 해결책

인프라 에이전트의 [구성 파일](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#passthrough_-environment) 에 `passthrough_environment` 변수를 추가하여 에이전트에 설정된 구성 매개변수를 온호스트 통합에 전달할 수 있습니다.

인프라 에이전트 버전 1.24.1부터 `passthrough_environment` 변수 이름에 대한 정규식을 지원합니다.

예를 들어 인프라 에이전트에 구성된 프록시 옵션을 통합에 전달하려면 인프라 구성 파일에 다음 구성을 추가합니다.

```
passthrough_environment:
  - HTTPS_PROXY
  - HTTP_PROXY
  - MY_REDIS_PASSWORD
  - NRIA_.*
```

인프라 에이전트 버전 1.14.0부터 아래와 같이 통합 구성 파일의 일부로 환경 변수를 정의할 수도 있습니다(`passthrough_environment` 구성도 필요함).

```
integration_name: com.newrelic.redis
instances:
  - name: redis-metrics
    command: metrics
    arguments:
      hostname: localhost
      port: 6379
      password: {{MY_REDIS_PASSWORD}}
```
