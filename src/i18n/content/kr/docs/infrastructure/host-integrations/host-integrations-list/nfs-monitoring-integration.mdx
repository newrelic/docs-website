당사의 NFS [통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) 은 귀하의 NFS 서비스에서 인벤토리 및 메트릭을 수집하여 당사 플랫폼으로 전송합니다. 여기서 귀하는 디스크 사용량 데이터를 볼 수 있으므로 문제의 원인을 보다 쉽게 찾을 수 있습니다.

통합을 설치하고 수집하는 데이터를 보려면 계속 읽으십시오.

## 호환성 및 요구 사항 [#req]

통합은 NFS 버전 3 또는 4와 호환됩니다.

통합을 설치하기 전에 다음 요구 사항을 충족하는지 확인하십시오.

* 호스트(NFS 클라이언트)에 [인프라 에이전트 , 최소 버전 1.5.40 설치](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic)
* [인프라 에이전트와 호환되는](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure#operating-systems) Linux 배포판

## 설치 및 활성화 [#install]

NFS 통합은 기본적으로 이미 활성화되어 설치되어 있습니다. 따라서 즉시 사용할 수 있으므로 설치 또는 활성화 요구 사항이 없습니다. [샘플 변수](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#samples-variables) 를 참조하고 자세한 정보는 NFS를 검색하십시오.

## 데이터 찾기 및 사용 [#find-and-use]

통합 데이터를 찾으려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Third-party services** 로 이동하고 **NFS 통합** 링크를 선택하십시오.

[데이터를 쿼리할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 때 NFS 데이터는 `NFSSample` [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 에 연결됩니다.

데이터를 찾고 사용하는 방법에 대한 자세한 내용은 [통합 데이터 이해 및 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 을 참조하십시오.

## 통합 비활성화 [#disable]

NFS 메트릭 스크랩을 방지하려면 인프라 에이전트 [구성 파일](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#metrics-nfs-sample-rate) 에서 `metrics_nfs_sample_rate` `-1` 로 설정합니다.
