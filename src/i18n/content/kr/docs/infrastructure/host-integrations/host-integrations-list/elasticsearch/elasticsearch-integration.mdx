Elasticsearch 통합은 [Elasticsearch](https://www.elastic.co/) 클러스터에서 인벤토리와 지표를 수집하여 Elasticsearch 환경의 상태를 볼 수 있는 플랫폼으로 보냅니다. 클러스터, 노드 및 인덱스 수준에서 메트릭을 수집하므로 문제의 원인을 더 쉽게 찾을 수 있습니다.

Elasticsearch 모니터링 통합을 설치하려면 다음 단계를 실행하십시오.

1. [통합을 설치하고 활성화합니다](#install) .
2. [통합을 구성합니다](#config) .
3. [데이터를 찾아 사용합니다](#find-and-use) .
4. 선택적으로 [Elasticsearch의 구성 설정](/docs/infrastructure/host-integrations/host-integrations-list/elasticsearch/elasticsearch-config) 을 참조하십시오.

### Elasticsearch 버전 [#elasticsearch-versions]

우리의 통합은 Elasticsearch 7.x에서 8.X까지 호환됩니다.

## 통합 설치 및 활성화 [#install]

Elasticsearch 통합을 설치하려면 환경에 대한 지침을 따르십시오.

### 리눅스 설치 [#linux]

1. [통합 설치](/docs/install-integrations-package) 지침을 따르고 `INTEGRATION_FILE_NAME` 변수를 `nri-elasticsearch` 로 바꿉니다.

2. 다음을 실행하여 디렉토리를 통합 구성 폴더로 변경하십시오.

   ```shell
   cd /etc/newrelic-infra/integrations.d
   ```

3. 다음을 실행하여 샘플 구성 파일을 복사합니다.

   ```shell
   sudo cp elasticsearch-config.yml.sample elasticsearch-config.yml
   ```

4. 즐겨 사용하는 편집기로 `elasticsearch-config.yml` 구성 파일을 편집합니다. 몇 가지 [훌륭한 구성 파일 예제](#examples) 를 확인하십시오.

5. 인프라 에이전트를 다시 시작하십시오. [다양한 Linux 환경에서 인프라 에이전트를 다시 시작하는](/docs/infrastructure/install-infrastructure-agent/manage-your-agent/start-stop-restart-infrastructure-agent/#linux) 방법을 참조하십시오.

6. 자동 Elasticsearch 오류 로그 구문 분석 및 전달을 활성화하려면 `elasticsearch-log.yml.example` 파일을 `elasticsearch-log.yml` 에 복사(또는 이름 변경)하십시오. 에이전트를 다시 시작할 필요가 없습니다.

**예시**

```shell
sudo cp /etc/newrelic-infra/logging.d/elasticsearch-log.yml.example /etc/newrelic-infra/logging.d/elasticsearch-log.yml
```

## 통합 구성 [#config]

설치 방법에 따라 통합을 구성하는 몇 가지 방법이 있습니다.

* 통해 활성화된 경우

  <img style={{ width: '32px', height: '32px'}} class="inline" title="Kubernetes" alt="Kubernetes" src={osk8}/>

  Kubernetes, [Kubernetes에서 실행 중인 모니터링 서비스 ](/docs/monitor-service-running-kubernetes)참조.

* 통해 활성화된 경우

  <img style={{ width: '32px', height: '32px'}} class="inline" title="ECS" alt="ECS" src={osEcs}/>

  Amazon ECS, [ECS에서 실행되는 모니터링 서비스 참조 ](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs).

* 호스트에 설치된 경우 통합의 YAML 구성 파일 `elasticsearch-config.yml` 에서 구성을 편집합니다. 통합의 YAML 형식 구성은 필수 로그인 자격 증명을 배치하고 데이터 수집 방법을 구성할 수 있는 곳입니다. 변경하는 옵션은 설정 및 기본 설정에 따라 다릅니다. 구성 파일에는 `interval` , `timeout` , `inventory_source` 와 같은 모든 통합에 적용할 수 있는 공통 설정이 있습니다. 이러한 공통 설정에 대한 모든 내용을 읽으려면 [구성 형식](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-newer-configuration-format/#configuration-basics) 문서를 참조하십시오.

<Callout variant="important">
  레거시 구성 또는 정의 파일을 계속 사용하는 경우 [표준 구성 형식](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-standard-configuration-format/) 을 확인하십시오.
</Callout>

## Elasticsearch-config.yml 샘플 파일 [#examples]

<CollapserGroup>
  <Collapser
    id="basic-config"
    title="기본 구성"
  >
    다음은 localhost에서 메트릭 및 인벤토리를 수집하는 데 사용되는 기본 구성입니다.

    ```yaml
    integrations:
      - name: nri-elasticsearch
        env:
          HOSTNAME: localhost
          PORT: 9200
          USERNAME: elasticsearch_user
          PASSWORD: elasticsearch_password
          CONFIG_PATH: /etc/elasticsearch/elasticsearch.yml
        interval: 15s
        labels:
          environment: production
        inventory_source: config/elasticsearch
    ```
  </Collapser>

  <Collapser
    id="basic-intervals"
    title="메트릭 및 인벤토리 간격이 다른 기본 구성"
  >
    이 구성은 15초마다 메트릭을 수집하고 60초마다 인벤토리를 수집합니다.

    ```yaml
    integrations:
      - name: nri-elasticsearch
        env:
          METRICS: true
          HOSTNAME: localhost
          PORT: 9200
          USERNAME: elasticsearch_user
          PASSWORD: elasticsearch_password
          REMOTE_MONITORING: true
        interval: 15s
        labels:
          environment: production

      - name: nri-elasticsearch
        env:
          INVENTORY: true
          HOSTNAME: localhost
          PORT: 9200
          USERNAME: elasticsearch_user
          PASSWORD: elasticsearch_password
          CONFIG_PATH: /etc/elasticsearch/elasticsearch.yml
        interval: 60s
        labels:
          environment: production
        inventory_source: config/elasticsearch
    ```
  </Collapser>

  <Collapser
    id="envvar-replacement"
    title="환경 변수 교체"
  >
    이 구성에서는 환경 변수 `ELASTIC_HOST` 를 사용하여 통합의 HOSTNAME 설정을 채웁니다.

    ```yaml
    integrations:
      - name: nri-elasticsearch
        env:
          METRICS: "true"
          HOSTNAME: {{ELASTIC_HOST}}
          PORT: 9200
          USERNAME: elasticsearch_user
          PASSWORD: elasticsearch_password
        interval: 15s
        labels:
          env: production
          role: load_balancer
    ```
  </Collapser>

  <Collapser
    id="multi-instance"
    title="클러스터 모니터링"
  >
    통합은 모든 클러스터 메트릭을 스크랩하기 전에 표시되는 기본 구성을 사용하여 노드 중 하나에 연결된 클러스터에서 모든 메트릭을 수집할 수 있습니다.

    통합을 구성하는 또 다른 방법은 모든 인스턴스의 스크랩을 구성하되 스크랩된 인스턴스가 마스터인 경우에만 통합이 측정항목을 수집하도록 하는 `MASTER_ONLY` 매개변수를 활성화하는 것입니다. 이를 통해 모든 노드에 대해 동일한 구성을 공유하고 데이터 중복을 피할 수 있습니다.  
    각 노드의 구성 파일은 다음과 같아야 합니다.

    ```yaml
    integrations:
      - name: nri-elasticsearch
        env:
          HOSTNAME: localhost
          PORT: 9200
          USERNAME: elasticsearch_user
          PASSWORD: elasticsearch_password
          MASTER_ONLY: "true"
          CONFIG_PATH: /etc/elasticsearch/elasticsearch.yml
        interval: 15s
        labels:
          environment: production
        inventory_source: config/elasticsearch
    ```
  </Collapser>
</CollapserGroup>
