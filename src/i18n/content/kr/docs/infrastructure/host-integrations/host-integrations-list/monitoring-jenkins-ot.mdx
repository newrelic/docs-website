작업 및 파이프라인 실행을 [분산 추적](/docs/distributed-tracing/concepts/introduction-distributed-tracing) 으로 시각화하여 [OpenTelemetry 플러그인](https://plugins.jenkins.io/opentelemetry) 으로 Jenkins를 모니터링합니다. 또한 [빠른 시작](https://newrelic.com/instant-observability/jenkins) 을 설치하여 Jenkins 파이프라인을 모니터링하기 위해 사전 빌드된 대시보드를 얻을 수 있습니다.

## 전제 조건 [#prerequisites]

먼저 Jenkins에서 OpenTelemetry 플러그인을 설치해야 합니다.

1. 젠킨스에 로그인합니다.

2. 대시보드에서

   <DNT>
     **Manage Jenkins**
   </DNT>

   클릭합니다.

3. 시스템 설정에서

   <DNT>
     **Plugins**
   </DNT>

   클릭합니다.

4. <DNT>
     **Available plugins**
   </DNT>

   탭을 클릭한 다음

   <DNT>
     **OpenTelemetry**
   </DNT>

   ) 검색하세요.

5. <DNT>
     **OpenTelemetry**
   </DNT>

   확인란을 선택하고

   <DNT>
     **Install without restart**
   </DNT>

   선택합니다.

6. 설치가 완료되면

   <DNT>
     **Manage Jenkins**
   </DNT>

   클릭합니다.

7. <DNT>
     **System Configuration**
   </DNT>

   아래에서

   <DNT>
     **System**
   </DNT>

   클릭합니다.

8. 아래로 스크롤하여

   <DNT>
     **OpenTelemetry**
   </DNT>

   섹션을 확인하세요. 표시되지 않으면 Jenkins를 다시 시작하세요.

## 구성 [#configuration]

뉴렐릭으로 데이터를 보내려면 [Jenkins 플러그인을 구성하려면](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-setup/#note-endpoints) 뉴렐릭 OTLP 엔드포인트 와 가 필요합니다.<InlinePopover type="licenseKey"/> OpenTelemetry

<img
  title="Screenshot showing Jenkins OpenTelemetry configuration"
  alt="Screenshot showing Jenkins OpenTelemetry configuration"
  src="/images/opentelemetry_screenshot-full_integrations-jenkins-02.webp"
/>

1. OTLP 끝점을 입력합니다. 예: `https://otlp.nr-data.net:4317` .

2. 인증을 위해

   <DNT>
     **Header Authentication**
   </DNT>

   선택합니다.

   <DNT>
     **Header Name**
   </DNT>

   필드에

   <DNT>
     **api-key**
   </DNT>

   입력합니다. 비.

   <DNT>
     **Header Value**
   </DNT>

   필드에 뉴렐릭 수집 인스턴스 키가 포함된 비밀 텍스트를 입력합니다.

3. 변경 사항을 저장합니다.

뉴렐릭 클러스터 키로 생성된 비밀 텍스트가 없으면 <DNT>**+ Add**</DNT> 버튼을 클릭하고 <DNT>**Jenkins Credentials Provider**</DNT> 선택하여 생성하세요. <DNT>**kind**</DNT> 및 <DNT>**secret**</DNT> 을 제외하고 기본값을 유지할 수 있으며 <DNT>**description**</DNT> 는 선택 사항입니다.

<img
  title="Screenshot showing Jenkins Credential Provider"
  alt="Screenshot showing Jenkins Credential Provider"
  src="/images/opentelemetry_screenshot-full_integrations-jenkins-03.webp"
/>

1. <DNT>
     **Kind**
   </DNT>

   드롭다운 메뉴에서

   <DNT>
     **Secret text**
   </DNT>

   선택하세요.

2. <DNT>
     **secret**
   </DNT>

   필드에 뉴렐릭 수집 인스턴스 키를 입력합니다.

3. 선택적으로 비밀 텍스트의 용도를 기록하는 설명을 추가합니다.

## 확인 [#validation]

New Relic에서 Jenkins 데이터를 보려면 작업을 실행하거나 새 파이프라인을 만드십시오. 파이프라인을 구축하는 방법은 다음과 같습니다.

1. Jenkins에 로그인하고

   <DNT>
     **New Item**
   </DNT>

   클릭합니다.

2. 항목 이름을 입력하고

   <DNT>
     **Pipeline**
   </DNT>

   클릭한 다음

   <DNT>
     **OK**
   </DNT>

   클릭합니다.

3. <DNT>
     **Pipeline**
   </DNT>

   섹션까지 맨 아래로 스크롤합니다.

4. 파이프라인 펼쳐보기를 사용하고

   <DNT>
     **try sample Pipeline...**
   </DNT>

   드롭다운 메뉴에서 옵션을 선택하세요.

5. 저장을 클릭합니다.

6. 새로 생성된 파이프라인에서

   <DNT>
     **Build Now**
   </DNT>

   클릭합니다.

7. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > Services - OpenTelemetry > jenkins**
   </DNT>

   에 도착했습니다.

8. 작업 및 파이프라인 실행을 시각화하려면

   <DNT>
     **Distributed tracing**
   </DNT>

   클릭하세요.

9. Jenkins 콘솔 로그를 보려면

   <DNT>
     **Logs**
   </DNT>

   클릭하세요. 로그가 없으면 환경 변수 `OTEL_LOGS_EXPORTER="otlp"` 이 설정되어 있는지 확인하세요.

## Jenkins 빠른 시작 대시보드 설치 [#quickstart]

Jenkins 파이프라인 데이터를 New Relic으로 보낸 후에는 [New Relic Instant Observability](http://newrelic.com/instant-observability) 에서 미리 빌드된 대시보드를 사용하여 작업 및 파이프라인 실행을 쉽게 모니터링할 수도 있습니다. 통합 보기에서 주요 메트릭을 보려면 미리 빌드된 대시보드로 몇 분 만에 시작하세요.

1. [뉴렐릭 Instant 옵저버빌리티](https://newrelic.com/instant-observability/jenkins) 의 Jenkins 퀵스타트로 이동하여

   <DNT>
     **+ Install now**
   </DNT>

   클릭하세요.

2. 계정을 선택하고

   <DNT>
     **Begin installation**
   </DNT>

   클릭합니다.

3. 이미 [유효성 검사를](#validation) 완료했다면

   <DNT>
     **done**
   </DNT>

   선택하여 다음 단계로 이동하세요.

4. 퀵스타트 구현은 귀하의 계정에 대한 리소스를 배포합니다. 대시보드로 이동하려면

   <DNT>
     **see your data**
   </DNT>

   클릭하세요.
