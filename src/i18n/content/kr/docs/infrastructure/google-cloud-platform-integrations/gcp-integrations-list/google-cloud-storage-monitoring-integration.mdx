New Relic은 Google Cloud Storage 데이터를 New Relic에 보고하기 위한 [통합](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 을 제공합니다. [이 통합을 인프라 모니터링에 연결](#connect) 하는 방법과 New Relic이 이 통합에 대해 보고하는 [메트릭 데이터](#metrics) 에 대해 알아보십시오.

## 특징

Google Cloud Storage는 웹사이트 콘텐츠를 제공하고, 보관 및 재해 복구를 위해 데이터를 저장하고, 직접 다운로드를 통해 데이터 개체를 배포하는 데 사용할 수 있는 Google Cloud Platform 서비스입니다. Google Cloud Storage 통합을 통해 다음 기능에 액세스할 수 있습니다.

* Google Cloud Storage에서 저장하고 검색하는 데이터에 대한 차트와 정보를 봅니다.
* 자동으로 캡처된 [데이터](#metrics) 에서 [사용자 지정 쿼리 및 차트를 만듭니다](/docs/using-new-relic/data/understand-data/query-new-relic-data) .
* 통합 페이지에서 직접 Google Cloud Storage 데이터에 대한 [알림](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#integration) 을 설정합니다.

## 통합 활성화 [#activate]

통합을 활성화하려면 표준 절차에 [따라 GCP 서비스를 New Relic에 연결하십시오](/docs/connect-google-cloud-platform-services-infrastructure) .

## 폴링 빈도 [#polling]

New Relic은 5분의 폴링 간격을 기반으로 Google Cloud Storage 서비스를 쿼리합니다.

## 데이터 찾기 및 사용 [#find-data]

통합을 New Relic에 연결하고 몇 분 정도 기다리면 New Relic UI에 데이터가 나타납니다. 대시보드 및 알림 설정을 포함한 [통합 데이터를 찾아 사용](/docs/telemetry-data-platform/understand-data/new-relic-data-types/) 하려면 [**one.newrelic.com**](https://one.newrelic.com) **> Infrastructure > GCP > Google Cloud Storage** 로 이동합니다.

통합을 위한 사용자 정의 대시보드를 만들려면 공급자 값이 `GcpStorageBucket` 인 `GcpStorageBucketSample` 이벤트 유형에 대한 [쿼리](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights) 를 만듭니다.

## 측정항목 데이터 [#metrics]

통합은 `method` 및 `response_code` 의 모든 값에 대한 [측정항목 데이터](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics) 를 보고합니다.

* `response_code`: 요청의 응답 코드입니다.
* `method`: 호출된 API 메서드의 이름입니다.

New Relic이 Google Cloud Storage 통합에서 수신하는 측정항목 데이터에는 다음이 포함됩니다.
