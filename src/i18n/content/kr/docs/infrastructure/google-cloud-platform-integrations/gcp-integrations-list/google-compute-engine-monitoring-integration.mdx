New Relic의 Compute Engine 통합을 사용하여 호스트의 성능 및 상태에 대한 종합적인 실시간 보기를 얻을 수 있습니다.

[Google Compute Engine](https://cloud.google.com/compute) 과의 통합은 Google 인프라에서 호스팅되는 인스턴스(가상 머신)에 대한 메타데이터를 보고합니다. [New Relic 인프라 모니터링을](/docs/infrastructure) 사용하여 GCP 인스턴스 데이터를 모니터링하고 경고할 수 있으며 New Relic에서 커스텀 쿼리 및 차트를 만들 수 있습니다.

## 통합 활성화 [#activate]

통합을 활성화하려면 표준 절차에 [따라 GCP 서비스를 New Relic에 연결하십시오](/docs/connect-google-cloud-platform-services-infrastructure) .

<Callout variant="important">
  해당 호스트에서 [지표를](/docs/infrastructure/manage-your-data/data-instrumentation/default-infrastructure-monitoring-data/#infrastructure-events) 보려면 각 GCE 호스트에 에이전트 에이전트를 설치해야 합니다. Google Cloud 프로젝트를 연결하면 New Relic 이 지역, 유형, 태그 등 GCE 메타데이터에 액세스할 수 있습니다.
</Callout>

## 폴링 빈도 [#polling]

New Relic은 통합에 따라 달라지는 폴링 간격에 따라 GCP 서비스를 쿼리합니다. Google Compute Engine 통합의 폴링 간격은 5분입니다.

## 데이터 찾기 및 사용 [#find-data]

통합을 활성화하고 몇 분 정도 기다리면( [폴링 빈도](#polling) 에 따라) New Relic UI 에 데이터가 표시됩니다. <InlinePopover type="dashboards"/>및 공지 설정에 대한 링크를 포함하여 [데이터를 찾고 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 하려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > GCP > (select an integration)**</DNT>)로 이동하세요.
