[New Relic](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 과 [GCP(Google Cloud Platform)](https://cloud.google.com/) 의 통합에는 [Google Cloud Pub/Sub](https://cloud.google.com/pubsub/) 데이터를 New Relic에 보고하는 통합이 포함됩니다. 이 문서에서는 GCP Pub/Sub 통합을 활성화하는 방법과 보고될 데이터를 설명합니다.

## 특징

Google Cloud [스트림 분석 솔루션](https://cloud.google.com/solutions/stream-analytics) 의 일부인 Cloud Pub/Sub 서비스는 빠른 데이터 처리 및 분석을 위해 이벤트 스트림을 수집 및 제공합니다.

## 통합 활성화 [#activate]

통합을 활성화하려면 표준 절차에 [따라 GCP 서비스를 New Relic에 연결하십시오](/docs/connect-google-cloud-platform-services-infrastructure) .

## 폴링 빈도 [#polling]

New Relic 통합은 통합에 따라 달라지는 폴링 간격에 따라 GCP 서비스를 쿼리합니다. Google Cloud Pub/Sub의 폴링 빈도는 5분입니다. 분해능은 1분에 1 데이터 포인트입니다.

## 데이터 찾기 및 사용 [#find-data]

통합을 활성화하고 몇 분 정도 기다리면( [폴링 빈도](#polling)기준) New Relic UI에 데이터가 나타납니다. 귀하의 데이터에 대한 링크를 포함하여 [귀하의 데이터를 찾고 사용하기](/docs/infrastructure/integrations/find-use-infrastructure-integration-data)위해 <InlinePopover type="dashboards"/>및 알림 설정은 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > GCP > (통합 선택)**으로 이동합니다.

## 측정항목 데이터 [#metrics]

Pub/Sub 주제는 메시지 피드를 나타내는 명명된 엔터티이고 구독은 특정 주제의 메시지 대상을 나타내는 명명된 엔터티입니다.

New Relic에서 GCP Pub/Sub 통합에 대한 [측정항목 데이터](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics) 를 보려면 `GcpPubSubTopicSample` 및 `GcpPubSubSubscriptionSample` [이벤트](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 와 관련 속성에 대한 [NRQL 쿼리](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights#nrql) 를 만듭니다.
