New Relic과 [GCP(Google Cloud Platform)](https://cloud.google.com/) 의 통합에는 [Google BigQuery](https://cloud.google.com/bigquery/) 데이터를 New Relic에 보고하는 통합이 포함됩니다. 이 문서에서는 Google BigQuery 통합을 활성화하는 방법과 보고할 수 있는 데이터를 설명합니다.

## 특징

Google BigQuery는 관리할 인프라가 없는 데이터 웨어하우스입니다. BigQuery를 사용하면 데이터 스키마를 지정하거나 서비스에서 자동으로 검색하도록 할 수 있습니다. SQL 쿼리를 사용하여 데이터를 탐색할 수 있습니다.

## 통합 활성화 [#activate]

통합을 활성화하려면 표준 절차에 [따라 GCP 서비스를 New Relic에 연결하십시오](/docs/connect-google-cloud-platform-services-infrastructure) .

## 폴링 빈도 [#polling]

New Relic 통합은 통합에 따라 달라지는 폴링 간격에 따라 GCP 서비스를 쿼리합니다. Google BigQuery 통합의 폴링 빈도는 5분입니다. 해상도에 대한 정보는 [메트릭 데이터 세부사항](#metrics) 을 참조하십시오.

## 테이블 측정항목 활성화 [#table-metrics]

BigQuery 테이블 측정항목 수집은 기본적으로 사용 중지되어 있습니다. 이는 [GcpBigQueryTableSample](#table-sample) 아래에 나열된 측정항목에 영향을 줍니다. 이러한 측정항목을 수집하려면 BigQuery 통합의 구성 메뉴에서 '테이블 측정항목 사용' 설정을 사용 설정하세요.

## 데이터 찾기 및 사용 [#find-data]

통합을 활성화하고 몇 분 정도 기다리면( [폴링 빈도](#polling)기준) New Relic UI에 데이터가 나타납니다. 귀하의 데이터에 대한 링크를 포함하여 [귀하의 데이터를 찾고 사용하기](/docs/infrastructure/integrations/find-use-infrastructure-integration-data)위해 <InlinePopover type="dashboards"/>및 알림 설정은 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > GCP > (통합 선택)**으로 이동합니다.

## 측정항목 데이터 [#metrics]

New Relic에서 Google BigQuery 통합에 대한 [측정항목 데이터](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics) 를 보려면 `GcpBigQueryProjectSample` , `GcpBigQueryDatasetSample` 및 `GcpBigQueryTableSample` [이벤트](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#event) 와 해당 속성에 대한 [NRQL 쿼리를 만듭니다](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights#nrql) .
