[New Relic](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 과 [GCP(Google Cloud Platform)](https://cloud.google.com) 의 통합에는 [Google Cloud Spanner](https://cloud.google.com/spanner/) 데이터를 New Relic에 보고하는 통합이 포함됩니다. 이 문서에서는 GCP Cloud Spanner 통합을 활성화하는 방법과 보고할 수 있는 데이터를 설명합니다.

## 특징

Google Cloud Spanner 서비스는 클라우드용으로 구축된 전 세계적으로 분산된 관계형 데이터베이스 서비스입니다. 개발자는 [Google Spanner 콘솔](https://cloud.google.com/spanner/docs/quickstart-console) 을 사용하여 Cloud Spanner 인스턴스를 만들고, 스키마를 추가하고, 데이터를 작성 및 수정하고, 쿼리를 실행할 수 있습니다.

## 통합 활성화 [#activate]

통합을 활성화하려면 표준 절차에 [따라 GCP 서비스를 New Relic에 연결하십시오](/docs/connect-google-cloud-platform-services-infrastructure) .

## 폴링 빈도 [#polling]

New Relic 통합은 통합에 따라 달라지는 폴링 간격에 따라 GCP 서비스를 쿼리합니다. Google Cloud Spanner의 폴링 빈도는 5분입니다. 분해능은 1분에 1 데이터 포인트입니다.

## 데이터 찾기 및 사용 [#find-data]

통합을 활성화하고 몇 분 정도 기다리면( [폴링 빈도](#polling) 기준) 데이터가 New Relic UI에 나타납니다. 대시보드 및 경고 설정에 대한 링크를 포함 [하여 데이터를 찾고 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 하려면 [**one.newrelic.com**](http://one.newrelic.com) **> Infrastructure > GCP > (통합 선택)으로** 이동하십시오.

## 측정항목 데이터 [#metrics]

New Relic에서 GCP Spanner 통합에 대한 [측정항목 데이터](/docs/integrations/new-relic-integrations/getting-started/#metric) 를 보려면 `GcpSpannerInstanceSample` 및 `GcpSpannerDatabaseSample` [이벤트](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 와 관련 속성에 대한 [NRQL 쿼리](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights#nrql) 를 만듭니다.

### GcpSpanner데이터베이스샘플 [#gcp-spanner-database-sample]

New Relic에서 `GcpSpannerDatabaseSample` 이벤트를 쿼리하여 다음 속성에 대한 데이터를 봅니다.

<dd>
  <table>
    <thead>
      <tr>
        <th style={{ width: "300px" }}>
          기인하다
        </th>

        <th>
          설명
        </th>
      </tr>
    </thead>

    <tbody>
      <tr>
        <td>
          `api.ReceivedBytes`
        </td>

        <td>
          Cloud Spanner에서 수신한 압축되지 않은 요청 바이트입니다.
        </td>
      </tr>

      <tr>
        <td>
          `api.Requests`
        </td>

        <td>
          Cloud Spanner API 요청 비율입니다.
        </td>
      </tr>

      <tr>
        <td>
          `api.RequestLatencies`
        </td>

        <td>
          데이터베이스에 대한 서버 요청 대기 시간 분포. 여기에는 Cloud Spanner 백엔드 및 API 레이어의 요청 처리 지연 시간이 포함됩니다. 여기에는 클라이언트와 서버 간의 네트워크 또는 역방향 프록시 오버헤드가 포함되지 않습니다.
        </td>
      </tr>

      <tr>
        <td>
          `api.SentBytes`
        </td>

        <td>
          Cloud Spanner에서 보낸 압축되지 않은 응답 바이트입니다.
        </td>
      </tr>
    </tbody>
  </table>
</dd>
