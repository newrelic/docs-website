Grafana에서 뉴렐릭으로 대시보드 변환 REST API를 사용하여 기존 Grafana 대시보드를 뉴렐릭 네이티브 대시보드로 변환할 수 있습니다. 이 API는 현재 Grafana 플러그인을 사용하여 뉴렐릭 데이터를 시각화하는 고객이 Grafana에서 뉴렐릭으로 마이그레이션하는 데 도움을 줄 수 있습니다. 옵저버빌리티 툴을 통합하거나, 전체 모니터링 설정을 뉴렐릭으로 마이그레이션하거나, 기존 Grafana 설정과 함께 뉴렐릭 대시보드를 테스트하려는 경우, 이 API를 사용하면 더 쉽게 시작할 수 있습니다.

이 API는 뉴렐릭의 PromQL 변환 로직을 사용하여 쿼리를 변환합니다. 지원되는 PromQL 기능 및 함수 목록을 보려면 [PromQL 기능 지원](/docs/infrastructure/prometheus-integrations/view-query-data/supported-promql-features)을 참조하십시오.

## 전제 조건 [#prerequisites]

이 API를 사용하기 전에 다음이 필요합니다:

* [뉴렐릭 사용자 API 키](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)
* JSON으로 내보낸 Grafana 대시보드(Grafana에서: Settings → JSON Model → Copy to Clipboard)
* 뉴렐릭 계정 ID(기본 계정을 재정의하려는 경우)

## API 엔드포인트에 요청 [#api-endpoints]

Grafana 대시보드를 뉴렐릭 대시보드로 변환하려면, 요청 본문에 변환하려는 대시보드를 포함시켜 해당 리전의 적절한 엔드포인트로 `POST` 요청을 보냅니다.

* US 고객의 경우: `https://prometheus-api.newrelic.com/api/v1/translate/dashboard/grafana`
* EU 고객의 경우: `https://prometheus-api.eu.newrelic.com/api/v1/translate/dashboard/grafana`

### 인증 [#authentication]

모든 API 호출에는 [사용자 API 키](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)가 필요합니다. 요청의 `x-api-key` 헤더에 이 키를 포함합니다.

### 요청 본문 [#request-body]

요청 본문은 다음 형식의 JSON 객체여야 합니다:

```
{
  "accountId": int [optional],
  "dashboard": object [required]
}
```

여기서 `dashboard`는 변환하려는 Grafana 대시보드의 JSON 객체이고, `accountId` 는 변환된 대시보드가 데이터를 쿼리할 계정입니다. `accountId`가 포함되지 않은 경우, 변환된 대시보드는 요청을 하는 사용자의 계정에서 데이터를 쿼리합니다.

### 응답 형식 [#response-format]

API는 변환된 뉴렐릭 대시보드가 포함된 JSON 객체를 반환합니다:

```json
{
  "dashboard": {
    "name": "Translated Dashboard",
    "permissions": "PUBLIC_READ_WRITE",
    "pages": [...]
    }
}
```

## 변환 오류 문제 해결 [#troubleshooting]

위젯이나 쿼리를 완전히 변환하지 못하게 만드는 문제를 발견하면, 변환기는 무엇이 잘못되었는지 설명하고 해결책을 제안하는 특수 마크다운 위젯인 &apos;Action Items(작업 항목)&apos;를 생성합니다.

## 예 [#examples]

<CollapserGroup>
  <Collapser id="dashboard-translation-basic" title="예: Grafana 대시보드를 변환하기 위한 기본 API 호출">
    ```bash
    export NR_USER_KEY="your_user_key"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --slurpfile dashboard "${JSON_FILE}" '{dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-account-override" title="예: 계정 ID 재정의를 사용한 대시보드 변환">
    대시보드에서 데이터를 쿼리할 계정을 재정의하려면 요청의 페이로드에 `accountId: int`를 포함할 수 있습니다. 이렇게 하면 대시보드의 모든 쿼리가 지정된 계정에서 데이터를 쿼리하게 됩니다.

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-and-upload" title="예: NerdGraph API를 사용하여 변환된 대시보드를 뉴렐릭에 업로드">
    <Callout variant="tip">
      NerdGraph를 사용하는 방법과 지역 엔드포인트를 찾는 방법에 대한 자세한 내용은 [뉴렐릭 NerdGraph 소개](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph)를 참조하십시오.
    </Callout>

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)
    export NERDGRAPH_ENDPOINT="US/EU Regional Nerdgraph endpoint"

    NR_DASHBOARD=$(curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard')


    curl -X POST ${NERDGRAPH_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "API-Key: ${NR_USER_KEY}" \
    -d '{
    "query": "mutation($accountId: Int!, $dashboard: DashboardInput!) { dashboardCreate(accountId: $accountId, dashboard: $dashboard) { entityResult { guid name } errors { description } } }",
    "variables": {
    "accountId": '${ACCOUNT_ID}',
    "dashboard": '${NR_DASHBOARD}'
    }
    }'
    ```
  </Collapser>
</CollapserGroup>
