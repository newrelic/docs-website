인프라 에이전트를 통해 데이터를 보고하는 [호스트 내 통합](/docs/integrations/host-integrations/get-started/introduction-host-integrations) 또는 인프라 에이전트를 사용하는 경우 데이터 보고를 구성하는 여러 가지 방법이 있습니다. 다음은 데이터 보고를 관리하기 위한 두 가지 일반적인 옵션입니다.

* [프로세스 메트릭 활성화/비활성화](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#enable-process-metrics)
* [보고할 특정 속성 선택](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#include-matching-metrics)

다른 에이전트 구성 옵션은 [구성](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings) 을 참조하십시오.

인프라 통합의 경우 데이터 보고 빈도를 변경할 수도 있습니다.

* [호스트 내 통합](/docs/infrastructure/host-integrations/host-integrations-list) 의 경우: 특정 통합의 `interval` [구성](/docs/integrations/integrations-sdk/file-specifications/host-integration-configuration-overview) 설정을 사용합니다.
* 클라우드 통합의 경우(AWS, Azure, Google Cloud): [폴링 빈도](/docs/integrations/new-relic-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations) 를 편집합니다.

일반적인 New Relic 데이터 관리에 대한 자세한 내용은 데이터 관리를 참조 [하십시오](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-your-data) .
