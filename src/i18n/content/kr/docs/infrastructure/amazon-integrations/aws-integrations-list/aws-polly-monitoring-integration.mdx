[New Relic 인프라 통합에는](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) Polly 지표를 New Relic에 보고하는 Amazon Polly 통합이 포함됩니다.

## 특징 [#features]

이 통합을 통해 [Amazon Polly](https://docs.aws.amazon.com/polly) 서비스에서 원격 측정 데이터를 수집하고 New Relic으로 보낼 수 있습니다. 서비스를 모니터링하고, 들어오는 데이터를 쿼리하고, 대시보드를 구축하여 모든 것을 한 눈에 관찰하세요.

## 보고서 데이터 [#report-data]

이 데이터를 보고하는 데는 두 가지 옵션이 있습니다. 권장되는 경로는 [Amazon CloudWatch Metric Streams 통합을](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream)설정하는 것입니다. 또는 [폴링에 의존하는 이전 Amazon 통합을](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations)사용할 수 있습니다.

## 데이터 찾기 및 사용 [#find-use-data]

이 통합의 데이터는 `Metric` 데이터 유형에 연결됩니다. UI에서 이 데이터를 찾으려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics and events** 로 이동하고 `aws.polly`에 대해 필터링합니다.

## 데이터 사용에 대한 팁 [#use-data]

개발 중인 변경 사항을 알리는 경고를 생성할 수 있습니다. 예를 들어 중요하거나 치명적인 오류를 관련 당사자에게 알리도록 경고를 설정할 수 있습니다. [알림 만들기에 대해 자세히 알아보세요](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts).
