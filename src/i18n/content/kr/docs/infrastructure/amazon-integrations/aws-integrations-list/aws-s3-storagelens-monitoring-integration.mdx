<Callout variant="important">
  [AWS CloudWatch Metric Streams 통합](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-metric-stream/) 을 활성화하여 사용자 지정 네임스페이스를 포함하여 AWS 서비스의 모든 CloudWatch 지표를 모니터링합니다. 개별 통합은 더 이상 권장되는 옵션이 아닙니다.
</Callout>

[뉴렐릭 인프라 통합](/docs/infrastructure/introduction-infra-monitoring) 에는 Amazon S3 데이터를 뉴렐릭에 보고하기 위한 통합이 포함됩니다. 이 문서에서는 통합을 활성화하는 방법과 보고되는 데이터에 대해 설명합니다.

## 특징 [#features]

[Amazon Simple Storage Service](https://aws.amazon.com/s3) (Amazon S3)는 개발자와 IT 팀에 안전하고 내구성이 뛰어나며 확장성이 뛰어난 클라우드 스토리지를 제공합니다.

뉴렐릭의 Amazon S3 스토리지 렌즈 통합을 통해 보고되는 데이터에는 S3 버킷 크기, 버킷 객체 수, GET 요청, POST 요청 및 기타 [지표가](#metrics) 포함됩니다. S3 데이터는 사전 구축된 <InlinePopover type="dashboards" />에서 사용할 수 있으며 [사용자 정의 쿼리 및 차트를](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights) 만들 수도 있습니다. S3 데이터의 변경 사항을 알리기 위해 공지 조건을 생성할 수도 있습니다.

## 통합 활성화 [#activate]

<Callout variant="important">
  [요청 및 데이터 전송](https://docs.aws.amazon.com/AmazonS3/latest/user-guide/configure-metrics.html) 지표는 프리미엄이며 AWS 통해 별도로 지불됩니다. CloudWatch 요금 정보는 Amazon의 [S3 향상된 모니터링](https://docs.aws.amazon.com/AmazonS3/latest/dev/cloudwatch-monitoring.html) 설명서를 참조하세요.
</Callout>

이 통합을 활성화하려면 [AWS 서비스를 New Relic에 연결](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure) 하기 위한 표준 절차를 따르십시오.

## 데이터 찾기 및 사용 [#find-data]

통합 데이터를 찾으려면 <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; AWS**</DNT> 으로 이동하여 S3 통합 링크 중 하나를 선택하세요.

검색 찾기 표시줄에서 `storagelens` 검색하여 지표 탐색기에서 [데이터를 쿼리하고 탐색](/docs/using-new-relic/data/understand-data/query-new-relic-data) 할 수 있습니다.

데이터 사용 방법에 대한 자세한 내용은 [통합 데이터 이해](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 를 참조하십시오.

## 측정항목 데이터 [#metrics]

이 통합은 Amazon S3 스토리지 렌즈 지표를 수집합니다. 이러한 지표에 대한 자세한 내용은 [Amazon의 S3 Storage Lens 설명서를](https://docs.aws.amazon.com/AmazonS3/latest/userguide/storage_lens_basics_metrics_recommendations.html) 참조하세요.
