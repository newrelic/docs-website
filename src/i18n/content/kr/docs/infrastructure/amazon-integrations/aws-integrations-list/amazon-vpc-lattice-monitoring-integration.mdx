New Relic은 [Amazon VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/what-is-vpc-lattice.html)에 대한 통합을 제공합니다.

## 특징 [#about]

[Amazon VPC Lattice는](https://aws.amazon.com/vpc/lattice/) 여러 계정과 Virtual Private Cloud(VPC)에서 애플리케이션 서비스를 연결, 보호 및 모니터링하는 데 사용하는 완전관리형 애플리케이션 네트워킹 서비스입니다.

New Relic은 Amazon VPC Lattice에 대한 두 가지 모니터링 통합인 [CloudWatch 메트릭](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) 과 [액세스 로그를](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html)지원합니다.

Cloudwatch 지표의 경우 통합을 한 번만 설정하면 `AWS/VPCLattice`포함하여 사용 가능한 모든 AWS 네임스페이스가 포함됩니다. VPC Lattice에 대한 CloudWatch 지표는 Lattice 서비스와 대상 그룹 간의 트래픽을 따라잡습니다.

<Callout variant="important">
  [AWS CloudWatch Metric Streams 통합](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-metric-stream/) 을 활성화하여 사용자 지정 네임스페이스를 포함하여 AWS 서비스의 모든 CloudWatch 지표를 모니터링합니다. 개별 통합은 더 이상 권장되는 옵션이 아닙니다.
</Callout>

## 액세스 로그 통합 활성화 [#activate-logs-integration]

<Callout variant="important">
  Kinesis Firehose를 사용하려면 Kinesis Data Firehose 전송 스트림을 생성해야 합니다. 그렇지 않은 경우 [AWS 콘솔](https://console.aws.amazon.com/firehose) 에서 Amazon Kinesis Firehose 기능으로 이동하고 `Destination: New Relic`를 사용하여 전송 스트림을 생성하여 생성할 수 있습니다.

  [Amazon Kinesis Data Firehose 전송 스트림 생성에 대해 자세히 알아보십시오.](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html?icmpid=docs_console_unmapped)
</Callout>

Amazon VPC Lattice에 대한 액세스 로그 통합은 [Amazon Kinesis Firehose를](https://aws.amazon.com/kinesis/data-firehose/)사용합니다. 서비스 네트워크에 모니터링을 추가하거나 추적하려는 서비스에 개별적으로 모니터링을 추가하여 Lattice 서비스에 대한 액세스 로그 모니터링을 활성화할 수 있습니다.

이 통합을 설정하려면:

1. AWS 콘솔에서 [AWS VPC 서비스](https://console.aws.amazon.com/vpc)로 이동합니다.
2. 왼쪽 패널의 **VPC Lattice** 드롭다운에서 모니터링을 활성화하는 데 사용할 옵션에 따라 `Service Networks` 또는 `Services`을 선택합니다.
3. 모니터링할 서비스 또는 서비스 네트워크를 선택합니다.
4. `Monitoring` 탭을 선택합니다.
5. `Edit access logs` 선택합니다.
6. `Access logs` 모니터링을 켭니다.
7. `Kinesis Data Firehose delivery stream`라고 표시된 확인란을 선택합니다.
8. New Relic을 가리키도록 구성된 드롭다운에서 전송 스트림을 선택합니다.
9. 변경 사항을 저장하다.

## 액세스 로그 데이터 찾기 및 사용 [#find-log-data]

`ServiceNetworkArn LIKE '%vpc-lattice%'`필터를 사용하여 로그 UI에서 데이터를 쿼리하고 탐색할 수 있습니다.

전송 중인 로그 데이터에 대한 자세한 정보를 얻으려면 [VPC Lattice에 대한 액세스 로그를](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html) 살펴보십시오.

## CloudWatch 지표 데이터 찾기 및 사용 [#find-metric-data]

[메트릭 쿼리 탐색기를](https://docs.newrelic.com/docs/data-apis/understand-data/metric-data/query-metric-data-type/)통해 CloudWatch 데이터를 쿼리하고 탐색할 수 있습니다.

전송되는 지표 데이터에 대한 자세한 정보를 얻으려면 [VPC Lattice용 CloudWatch 지표](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) 에 대한 세부 정보를 확인하십시오.
