[당사의 인프라 통합](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 에는 Amazon Web Services Simple Notification Service(SNS) 데이터를 New Relic에 보고하기 위한 통합이 포함됩니다. 이 문서에서는 통합을 활성화하는 방법과 보고된 데이터를 설명합니다.

## 특징 [#features]

[AWS Simple Notification Service(SNS)](https://aws.amazon.com/sns/) 는 관리형 푸시 알림 서비스입니다. New Relic의 AWS SNS 통합을 통해 New Relic에 보고되는 데이터에는 다음이 포함됩니다.

* 보낸 메시지 및 실패한 수
* 구독 확인 및 삭제됨
* 추가 [측정항목](#metrics)

SNS 데이터는 New Relic의 사전 구축된 대시보드에서 사용할 수 있습니다. [사용자 지정 쿼리 및 차트](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights) 를 생성하고 SNS 데이터의 변경 사항을 알리는 [알림 조건](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information) 을 생성할 수도 있습니다.

## 통합 활성화 [#activate]

이 통합을 활성화하려면 [AWS 서비스를 New Relic에 연결](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure) 하기 위한 표준 절차를 따르십시오.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다. AWS SNS 통합에 대한 기본 [폴링](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) 정보에는 다음이 포함됩니다.

* New Relic 폴링 간격: 5분
* AWS SNS 데이터 간격: 5분

## 데이터 찾기 및 사용 [#find-data]

[통합 데이터](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 를 탐색하려면:

1. [**one.newrelic.com**](http://one.newrelic.com) **> Infrastructure > AWS** 로 이동합니다.
2. 사용 가능한 SNS 통합 링크를 선택합니다.

`provider` 값이 `SnsTopic` 인 `QueueSample` [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 을 사용 [하여 데이터를 쿼리하고 탐색할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.
