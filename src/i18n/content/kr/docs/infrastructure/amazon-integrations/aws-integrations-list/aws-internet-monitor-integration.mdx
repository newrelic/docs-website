AWS CloudWatch Internet Monitor 통합은 CloudWatch Internet Monitor 데이터를 New Relic 플랫폼에 보고합니다.

## 특징 [#features]

이 통합을 통해 원격 측정 데이터를 수집하고 [CloudWatch Internet Monitor](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-InternetMonitor.html) 에서 New Relic으로 보낼 수 있습니다. 일단 통합되면 서비스를 모니터링하고, 들어오는 데이터를 쿼리하고, 로그를 모니터링하고, 사전 구축된 대시보드를 사용하여 모든 것을 한 눈에 관찰할 수 있습니다.

## 메트릭 데이터 가져오기 [#get-metrics-data]

이 데이터를 보고하는 데는 두 가지 옵션이 있습니다. 권장되는 경로는 [Amazon CloudWatch Metric Streams 통합을](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream)설정하는 것입니다. 또는 [폴링에 의존하는 이전 Amazon 통합을](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations)사용할 수 있습니다.

## 로그 데이터 가져오기 [#get-logs-data]

[로그 수집 Lambda를](/docs/logs/forward-logs/aws-lambda-sending-cloudwatch-logs) 사용하여 로그 데이터를 New Relic에 보고할 수 있습니다. Lambda에 [트리거를 추가하고](/docs/logs/forward-logs/aws-lambda-sending-cloudwatch-logs/#create-trigger) `country`, `city`, `subdivision` 및 `metro` 수준에서 `internet-monitor` 에 해당하는 로그 그룹을 선택해야 합니다.

## 데이터 찾기 및 사용 [#find-use-data]

이 통합의 메트릭 데이터는 `Metric` 데이터 유형에 첨부됩니다. 데이터를 보려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Metrics and events**</DNT> 로 이동하여 `aws.internetmonitor` 로 필터링하세요.
