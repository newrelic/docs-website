[New Relic](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 에는 Amazon Kinesis Data Firehose 데이터 수집을 위한 통합이 포함되어 있습니다. 이 문서에서는 이 통합을 활성화하는 방법과 보고할 수 있는 데이터에 대해 설명합니다.

## 특징 [#features]

[Amazon Kinesis Data Firehose](https://aws.amazon.com/kinesis/firehose/) 는 스트리밍 데이터를 캡처하고 로드하는 간단한 방법을 제공합니다. 스트리밍 데이터를 캡처, 변환 및 Amazon Kinesis Data Analytics, Amazon S3, Amazon Redshift 및 Amazon Elasticsearch Service로 로드하여 거의 실시간으로 분석할 수 있습니다.

New Relic의 Kinesis Data Firehose 통합은 인덱싱된 레코드 및 바이트, AWS 서비스에 복사된 데이터 수, 레코드의 수명 및 최신성, 기타 [지표 데이터](#metrics) 와 같은 데이터를 보고합니다.

## 통합 활성화 [#activate]

이 통합을 활성화하려면 [AWS 서비스를 New Relic에 연결](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure) 하기 위한 표준 절차를 따르십시오.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

Amazon Kinesis Data Firehose 통합을 위한 [기본 폴링](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) :

* New Relic 폴링 간격: 5분
* Amazon CloudWatch 데이터 간격: 1분

## 데이터 찾기 및 사용 [#find-data]

통합 데이터를 찾으려면 [**one.newrelic.com**](http://one.newrelic.com) **> Infrastructure > AWS** 로 이동하여 Kinesis Data Firehose 통합 링크 중 하나를 선택하십시오.

`provider` 값이 `KinesisDeliveryStream` 인 `QueueSample` [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 을 사용 [하여 데이터를 쿼리하고 탐색할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

데이터 사용 방법에 대한 자세한 내용은 [통합 데이터 이해 및 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 을 참조하십시오.
