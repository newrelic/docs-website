[New Relic 인프라 통합](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 에는 AWS Security Lake 통합이 포함되어 있으므로 보안 로그 데이터를 New Relic으로 보낼 수 있습니다.

통합을 사용하여 [Security Lake](https://aws.amazon.com/security-lake/) 에서 원격 측정 데이터를 수집하고 New Relic으로 보냅니다. 이 통합을 사용하여 서비스를 모니터링하고, 들어오는 데이터를 쿼리하고, 대시보드를 구축하여 모든 것을 한 눈에 관찰할 수 있습니다.

## 통합 활성화 [#activate]

이 통합을 활성화하려면 S3 로그 전달자를 설정하십시오. 쉽고 편리하게 저희 응용 프로그램을 사용하는 것이 좋지만 직접 설정할 수도 있습니다.

<Callout variant="info">
  Security Lake 모니터링 설정에는 두 가지 옵션이 있습니다. 반복 단계를 피하기 위해 여러 지역을 통합하거나 지역별로 설정할 수 있습니다. 자세한 내용은 [여러 지역 관리를](https://docs.aws.amazon.com/security-lake/latest/userguide/manage-regions.html)참조하세요.
</Callout>

## 전제 조건 [#prerequisites]

이 통합을 활성화하기 전에 먼저 Security Lake에 대해 다음 단계가 완료되었는지 확인하십시오.

1. Amazon Security Lake [시작 안내서](https://docs.aws.amazon.com/security-lake/latest/userguide/getting-started.html)에 언급된 단계를 완료하십시오.
2. [Security Lake 구독자 안내서](https://docs.aws.amazon.com/security-lake/latest/userguide/subscriber-data-access.html#prereqs-creating-subscriber)의 단계에 따라 데이터 액세스 권한이 있는 구독자를 설정하기 위한 전제 조건을 완료하십시오.

## 구성 단계 [#steps]

수행할 단계에 대한 개요는 다음과 같습니다.

1. [Security Lake 구독자를 만듭니다](#create-subscriber).
2. [전용 Amazon Security Lake 로그 전달자를 설치합니다](#install-log-forwarder).
3. [로그 데이터를 찾아 사용하십시오](#logs).

## Amazon Security Lake 로그 전달자 설치 [#install-log-forwarder]

로그 전달자를 설치하려면:

1. 브라우저에서 [AWS Serverless Application Repository를](https://serverlessrepo.aws.amazon.com/applications) 엽니다.

   <img title="AWS Lambda - Select region" alt="AWS Lambda - Select region" src={serverlessAWSLambdaSelectRegion}/>

2. `newrelic` 를 검색하고 **Show apps that create custom IAM roles or resource policies**\[맞춤 IAM 역할 또는 리소스 정책을 생성하는 앱 표시를] 선택하여 `newrelic-securitylake-s3-processor-LogForwarder`을 찾습니다.

3. `newrelic-securitylake-s3-processor-LogForwarder` 세부정보를 클릭하고 **Deploy**\[배포를]클릭합니다.

4. 이전 단계의 `AWS role ID` ARN을 복사하여 `SecurityLakeSubscriberRoleArn` 필드에 붙여넣습니다.

5. 이전 단계의 `Subscription endpoint` ARN을 복사하여 `SecurityLakeSubscriberRoleArn` 필드에 붙여넣습니다.

6. 이전 단계에서 추가한 `ExternalID` 를 입력합니다.

7. 당신의

   <InlinePopover type="licenseKey"/>

   `NRLicenseKey` 필드에 입력합니다.

8. 확인하고 **Deploy**\[배포를]선택합니다.

이에 대한 자세한 내용은 [Amazon Security Lake 로그 전달자 문서를](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3)참조하십시오.

## 로그 데이터 찾기 및 사용 [#logs]

New Relic에서 로그를 찾으려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Logs** 이동하고 **Attributes** \[속성을] `product.name` 로 설정한 다음 원하는 로그 소스를 선택하십시오.

현재 지원되는 로그 소스는 다음과 같습니다.

* [루트 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [VPC 흐름](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [클라우드 트레일](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [보안 허브](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)

<Callout variant="info">
  Amazon Security Lake는 로그에 [OCSF 스키마를](https://schema.ocsf.io/) 사용합니다.
</Callout>
