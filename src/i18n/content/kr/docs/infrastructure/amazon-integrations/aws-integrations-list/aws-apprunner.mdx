[New Relic의 인프라 통합에는](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) Amazon App Runner 데이터를 New Relic에 보고하기 위한 통합이 포함됩니다. 이 문서에서는 통합을 활성화하는 방법과 보고되는 데이터에 대해 설명합니다.

## 특징 [#features]

이 통합을 사용하면 Amazon App Runner 컨테이너에서 실행되는 앱과 서비스에서 텔레메트리 데이터를 수집하고 보낼 수 있습니다. 서비스를 모니터링하고 나면 수신 데이터를 쿼리하고 <InlinePopover type="dashboards"/>을 빌드하여 모든 것을 한눈에 관찰할 수 있습니다.

## 통합 활성화 [#activate]

이 통합을 활성화하려면 [Connect AWS services to New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure) 문서의 표준 절차를 따르십시오.

## 구성 및 폴링 [#config-poll]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

AWS 통합에 대한 [기본 폴링 정보](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) :

* New Relic 폴링 간격: 5분
* Amazon CloudWatch 데이터 간격: 1분 또는 5분

## 데이터 찾기 및 사용 [#find-data]

통합 데이터를 찾으려면:

* 로그의 경우:

  <DNT>
    **[one.newrelic.com ](https://one.newrelic.com)> Logs > Attributes > AWS**
  </DNT>

  으로 이동하여

  <DNT>
    **logGroup**
  </DNT>

  및/또는 선택

  <DNT>
    **logStream**
  </DNT>

* 지표의 경우:

  <DNT>
    **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Metrics and events**
  </DNT>

  으로 이동하여 다음을 기준으로 필터링 `aws.apprunner`
