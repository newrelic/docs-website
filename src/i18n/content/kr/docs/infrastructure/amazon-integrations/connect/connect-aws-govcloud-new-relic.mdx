[AWS GovCloud(US)](https://aws.amazon.com/govcloud-us/) 리전은 미국(연방, 주 및 지방 기관), 교육 기관 및 지원 에코시스템의 특정 규제 요구 사항을 해결하도록 설계되었습니다. 클라우드에서 민감한 데이터와 규제 대상 워크로드를 호스팅하도록 설계된 격리된 AWS 리전으로, 고객이 미국 정부 규정 준수 요구 사항을 지원할 수 있도록 지원합니다. [사용 가능한 AWS 서비스](https://aws.amazon.com/govcloud-us/details/) 세트는 AWS 에코시스템의 하위 집합입니다.

New Relic은 가장 중요한 서비스를 GovCloud에 배포할 수 있는 자신감을 제공하여 [플랫폼](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one) 에서 전체 생태계를 모니터링하고 관찰할 수 있도록 합니다.

AWS CloudWatch 지표 스트림 기능은 GovCloud 리전에서 사용할 수 없습니다.

<Callout variant="tip">
  Terraform을 사용하여 클라우드 통합을 활성화하는 프로세스를 자동화할 수 있습니다. [Terraform 공식 문서 사이트](https://registry.terraform.io/providers/newrelic/newrelic/latest/docs/guides/cloud_integrations_guide) 에서 방법을 읽어보세요.
</Callout>

## AWS GovCloud를 New Relic에 연결

New Relic AWS 통합으로 Amazon 데이터 수신을 시작하려면 Amazon 계정을 New Relic에 연결하십시오.

1. [자격 증명을](#govcloud-credentials) 얻습니다.
2. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > GovCloud** 로 이동합니다.
3. **AWS GovCloud 계정 추가** 를 클릭합니다.
4. AWS 계정에 이름을 지정하고 계정 연결을 위한 자격 증명을 제공한 다음 **제출** 을 클릭합니다.
5. New Relic 인프라 통합으로 모니터링할 Amazon Web Services를 선택한 다음 **저장** 을 클릭합니다.

## New Relic에 대한 GovCloud 자격 증명을 얻는 방법 [#govcloud-credentials]

1. [IAM 콘솔](https://console.amazonaws-us-gov.com/iam/home#/users) 에서 **사용자 추가** 를 클릭합니다.

2. **사용자 이름** 에 **NewRelicInfrastructure-Integrations** 를 입력하십시오. **AWS 액세스 유형** 선택에서 **프로그래밍 방식 액세스** 를 선택합니다.

   <img
     title="AWS-add-user.png"
     alt="Add a user in the AWS IAM console."
     src={infrastructureAwsAddUser}
   />

   <figcaption>
     **AWS IAM console > Add user** 추가 : **NewRelicInfrastructure-Integrations** 를 사용자로 추가합니다.
   </figcaption>

3. 정책 연결: **ReadOnlyAccess** 검색 , **ReadOnlyAccess** 라는 정책의 확인란을 선택한 후 **다음: 태그** 를 클릭합니다(태그 추가는 선택 사항임). 또는 [자체 관리형 정책을 생성](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) 하고 모니터링하려는 AWS 서비스에 따라 New Relic에 부여하는 권한을 제한할 수 있습니다.

   <img
     title="AWS-set-permissions.png"
     alt="AWS-set-permissions.png"
     src={infrastructureAwsSetPermissions}
   />

   <figcaption>
     **AAWS IAM console > Add user > Set permissions** : **ReadOnlyAccess** 를 선택합니다.
   </figcaption>

4. **태그** 페이지에서 **다음: 검토** 를 클릭합니다. 사용자 세부 정보 요약을 검토하고 사용자 **만들기** 를 클릭합니다.

   <img
     title="AWS-create-user.png"
     alt="In IAM console, create the user."
     src={infrastructureAwsCreateUser}
   />

   <figcaption>
     **AWS IAM console > Add user > Set permissions > Tags > Review** : 새 사용자 정보가 정확한지 확인합니다.
   </figcaption>

5. 사용자가 성공적으로 생성되어야 합니다. **.csv 다운로드** 버튼을 클릭하여 사용자 보안 자격 증명을 다운로드한 다음 **닫기** 를 클릭합니다.
