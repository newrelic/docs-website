## 문제

AWS 통합을 설정하고 [이를 New Relic에 연결](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) 했지만[AWS 대시보드](/docs/integrations/amazon-integrations/get-started/introduction-aws-integrations#insights) 에서 데이터를 볼 수 없습니다.

## 해결책

측정항목을 검색할 수 [있는 올바른 권한을 부여](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies) 했는지 확인하세요.

이를 확인하려면 통합의 **계정 상태 대시보드** 로 이동하여 보고된 오류가 없는지 확인하십시오.

<img
  title="AWS account status dashboard"
  alt="Screenshot showing the account status dashboard."
  src={infrastructureAwsAccount}
/>

계정 상태 대시보드에서 **권한 오류** 대시보드를 확인하고 데이터 원본에 대한 메서드 오류가 있는지 확인합니다.

<img
  title="Permission errors in the AWS account status dashboard"
  alt="Screenshot showing permissions errors in the account status dashboard."
  src={infrastructureAwsPermissionErrors}
/>

오류가 보고되면 권한 정책 문서(표준 AWS `Read-Only Access` 정책이 아닌 경우)에 [사용자 지정 권한](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies) 문서에 따라 오류 대시보드에 보고된 메서드가 있는지 확인하십시오.

<Callout variant="important">
  일부 방법은 통합과 관련이 없으며 **[모든 통합에 대해 부여되어야 합니다](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies#all-permissions)** .
</Callout>
