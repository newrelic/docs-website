## 문제

인프라 에이전트를 설치한 다음 Amazon 계정을 여기에 연결했습니다. 몇 분을 기다린 후에도 인프라 모니터링 UI에서 또는 데이터를 쿼리할 때 AWS 통합에 대한 데이터가 여전히 표시되지 않습니다.

## 해결책

10분 이상 기다린 후에도 데이터가 표시되지 않으면 다음 문제 해결 제안을 시도해 보세요.

1. 인프라 에이전트가 [설치되어 제대로 실행되고](/docs/infrastructure/new-relic-infrastructure/troubleshooting/no-data-appears-infrastructure) 있는지 확인하십시오. **먼저** 인프라 에이전트를 설치한 다음 AWS 계정에 연결해야 합니다.

   <Callout variant="tip">
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events** 또는 **Infrastructure > Settings > Agents** 에서 속성 이름이 표시되지만 데이터는 표시되지 않는 경우 인프라 에이전트를 설치하지 않았을 수 있습니다.
   </Callout>

2. 계정 상태 대시보드에 표시되는 오류를 수정하십시오. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS** 로 이동하고 각 계정에 대한 **계정 상태 대시보드를** 선택합니다.

   **역할 오류** 또는 **권한 오류** 보고가 표시되면 [통합 및 관리형 정책](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) 을 참조하고 계정에서 관련 데이터를 읽는 데 필요한 권한을 New Relic에 부여했는지 확인하세요.

   다른 오류 보고가 표시되면 [AWS IAM 콘솔](https://console.aws.amazon.com/iam/home?#/roles) 로 이동하여 AWS ARN에 대한 설정이 다음과 일치하는지 확인하십시오.

3. 데이터의 [올바른 위치를 찾고](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) 있는지 확인하십시오. [메트릭 및 이벤트](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) 에서 모든 통합 데이터를 찾을 수 있습니다.
