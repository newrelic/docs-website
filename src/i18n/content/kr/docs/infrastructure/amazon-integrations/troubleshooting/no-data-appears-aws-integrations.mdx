## 문제

인프라 에이전트를 설치한 다음 Amazon 계정을 여기에 연결했습니다. 몇 분을 기다린 후에도 인프라 모니터링 UI에서 또는 데이터를 쿼리할 때 AWS 통합에 대한 데이터가 여전히 표시되지 않습니다.

## 해결책

10분 이상 기다린 후에도 데이터가 표시되지 않으면 다음 문제 해결 제안을 시도해 보세요.

1. 인프라 에이전트가 [설치되어 제대로 실행되고 있는지](/docs/infrastructure/new-relic-infrastructure/troubleshooting/no-data-appears-infrastructure) 확인하십시오. 인프라 에이전트 <DNT>**first**</DNT> 설치한 다음 AWS 계정에 연결해야 합니다.

   <Callout variant="tip">
     속성 이름은 표시되지만 <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Events**</DNT> 또는 <DNT>**Infrastructure &gt; Settings &gt; Agents**</DNT> 에 데이터가 없으면 인프라 에이전트를 설치하지 않았을 수 있습니다.
   </Callout>

2. 계정 상태 대시보드에 표시되는 오류를 수정하세요. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; AWS**</DNT> 으로 이동하여 각 계정에 대해 <DNT>**Account status dashboard**</DNT> 을 선택하세요.

   <DNT>**Role errors**</DNT> 또는 <DNT>**Permission errors**</DNT> 보고가 표시되면 [통합 및 관리형 정책을](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) 참조하고 뉴렐릭에게 계정에서 관련 데이터를 읽는 데 필요한 권한을 부여했는지 확인하세요.

   다른 오류 보고가 표시되면 [AWS IAM 콘솔](https://console.aws.amazon.com/iam/home?#/roles) 로 이동하여 AWS ARN에 대한 설정이 다음과 일치하는지 확인하십시오.
