## 문제

AWS에서 RDS Enhanced Monitoring, VPC 흐름 로그 통합 또는 AWS Lambda에 대한 모니터링의 초기 버전(알파 및 베타)으로 로그를 푸시하기 위해 `NewRelic-log-ingestion` , 람다 함수를 사용하고 있습니다. 작동하지 않거나 부분 데이터를 보내고 있습니다.

## 해결책

2.1 이전의 `NewRelic-log-ingestion` 람다 버전은 더 이상 사용되지 않습니다. AWS Serverless Repository에 게시된 최신 람다로 업데이트합니다. [New Relic CLI](https://developer.newrelic.com/build-tools/new-relic-one-applications/cli) 를 사용하거나 수동으로 업데이트할 수 있습니다.

권장 사항: 로그 수집 Lambda 함수를 업데이트하는 주요 방법은 뉴렐릭 CLI를 사용하여 함수를 업데이트하는 [표준 절차를](/docs/serverless-function-monitoring/aws-lambda-monitoring/get-started/update-lambda-monitoring) 따르는 것입니다.

기능을 수동으로 업데이트할 수도 있습니다.

1. 경고에서 오탐 방지: [UI 절차](/docs/alerts/new-relic-alerts/configuring-alert-policies/update-or-disable-policies-conditions#condition-on-off) 또는 [API 절차](/alerts/rest-api-alerts/new-relic-alerts-rest-api/disable-enable-alerts-conditions-using-api) 에 따라 AWS Lambda, RDS Enhanced Monitoring 및 VPC Flow Logs와의 모니터링 통합과 관련된 모든 경고 조건을 비활성화합니다.
2. 람다의 오래된 람다 버전 제거: AWS Lambda 콘솔로 이동하여 `newrelic-log-ingestion` 을(를) 제거합니다. 이렇게 하면 다음 단계가 완료될 때까지 RDS Enhanced Monitoring 및 VPC Flow Logs 통합이 중지됩니다.
3. 서비스 다시 활성화: [RDS Enhanced Monitoring](/docs/integrations/amazon-integrations/aws-integrations-list/aws-rds-enhanced-monitoring-integration#activate) 또는 [VPC Flow Logs](/docs/integrations/amazon-integrations/aws-integrations-list/aws-vpc-flow-logs-monitoring-integration#enable) 의 지침을 따르거나 [New Relic Lambda로 스트리밍하도록 CloudWatch 로그를 구성](/docs/serverless-function-monitoring/aws-lambda-monitoring/get-started/enable-new-relic-monitoring-aws-lambda#stream-logs) 하는 단계를 따릅니다.
4. 데이터가 새 람다를 통해 흐르고 있는지 확인하십시오.
