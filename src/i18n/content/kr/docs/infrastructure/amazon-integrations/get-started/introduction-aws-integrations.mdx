뉴렐릭은 Amazon Web Services (AWS) 환경에 대한 성능 데이터를 수집하는 여러 가지 방법을 제공합니다. 뉴렐릭을 AWS 와 연결한 후에는 시스템의 AWS 텔레메트리에 액세스하여 AWS 그리드의 상태를 면밀히 모니터링할 수 있습니다. 우리의 AWS 옵저버빌리티 솔루션은 [Amazon CloudWatch Metric Streams 통합을](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations/#monitor-aws) 지원하여 지표를 얻고, [AWS 서비스 쿼리를 뉴렐릭으로 전달하여](/install/aws-logs/) 뉴렐릭에 로그인을 가져옵니다. 이 문서에서는 AWS 모니터링을 시작하는 방법을 설명하고 AWS 비용에 대한 개요를 제공합니다.

## CloudWatch Metric Streams 통합으로 시작하세요 [#monitor-aws]

AWS 옵저버빌리티 솔루션은 Amazon CloudWatch Metric Streams 통합으로,AWS [Amazon CloudWatch Metric Streams](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Metric-Streams.html) 통해 전체 리소스에 대한 데이터를 수집합니다. 대부분의 사용 사례에서 [Amazon CloudWatch Metric Streams 통합을 설치](/install/aws-cloudwatch) 하여 AWS 스택 모니터링을 시작합니다.

CloudWatch Metric Streams 통합은 AWS 대시보드에 대한 최상의 데이터를 제공하지만 [CloudWatch Metric Streams에서 지원하지 않는 통합을](/docs/infrastructure/amazon-integrations/connect/connect-aws-new-relic-infrastructure-monitoring/#integrations-not-replaced-streams) 사용하는 경우 통합 API 폴링 솔루션을 사용해야 할 수 있습니다. 현재 지원되는 통합에 대해 API 폴링을 사용 중인 경우 [데이터를 Amazon CloudWatch Metric Streams로 마이그레이션하는](/install/aws-cloudwatch/?task=migrate) 것이 좋습니다.

<Callout variant="tip">
  AWS 스택에 미국 연방, 주 및 지방 기관에 대한 특정 규정 요구 사항이 있거나 교육 기관인 경우 [AWS GovCloud에 대한 절차를 따르세요](/docs/infrastructure/amazon-integrations/connect/connect-aws-govcloud-new-relic).
</Callout>

## EC2 인스턴스 및 Amazon EKS에 대한 데이터 수집 [#additional-tools]

Amazon CloudWatch Metric Streams 통합을 사용하면 EC2 인스턴스와 Amazon Elastic Kubernetes Service(EKS) 환경에 대한 데이터를 수집할 수 있습니다. 더욱 세부적인 데이터가 필요한 경우 CloudWatch Metric Streams 통합 위에 추가 통합을 설치하여 가시성을 더욱 높일 수 있습니다.

* [EC2 호스트에 인프라 에이전트를 설치](/docs/infrastructure/amazon-integrations/connect/ec2-quickstart) 하거나 [AWS System Manager Distributor를 할당하여 인프라 에이전트를 설치하면](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-sys-dist) EC2 인스턴스를 모니터링할 수 있습니다.
* 쿠버네티스 클러스터를 모니터링하려면 [뉴렐릭 EKS 애드온 설치 절차를 따르세요](/docs/infrastructure/amazon-integrations/connect/eks-add-on).

## 통합을 위한 AWS 비용 계산 [#aws-billing]

이 섹션에서는 AWS 통합이 AWS 청구에 어떤 영향을 미치는지 설명하는 관련 AWS 문서로 이동합니다.

* [AWS CloudWatch 가격 책정 페이지에서는](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Metric-Streams.html)메트릭 업데이트 수에 따라 AWS CloudWatch Metric Streams 가격을 정의합니다.
* 때때로 AWS 에서 뉴렐릭으로의 전달 방법으로 AWS Kinesis Data Firehose를 사용하므로 [AWS Firehose 가격 페이지를](https://aws.amazon.com/kinesis/data-firehose/pricing/) 검토하는 것이 좋습니다.
* AWS Config를 사용하여 리소스 관리자로 CloudWatch 지표를 강화하면 기록된 설정 항목 수에 따라 요금이 부과됩니다. 자세한 내용은 [AWS Config 가격 페이지를](https://aws.amazon.com/config/pricing/) 참조하세요.
* [AWS 리소스 그룹 태그 API](https://docs.aws.amazon.com/resourcegroupstagging/latest/APIReference/overview.html) 연관된 AWS 리소스에 정의된 사용자 정의 태그로 MintWorks를 강화합니다. 이는 폴링과 메트릭 스트림 모두에 영향을 미칩니다.

뉴렐릭은 대신 API 폴링 AWS API 통합을 사용할 때 Amazon CloudWatch 사용하여 서비스에 대한 데이터를 CloudWatch Metric Streams 캡처합니다. 일부 상황에서는 CloudWatch API에 대한 호출 수가 AWS에서 부여한 <DNT>**1 million free**</DNT> 한도를 초과하여 [CloudWatch 요금이 증가](/docs/infrastructure/amazon-integrations/troubleshooting/cloudwatch-billing-increase) 할 수 있습니다. API 폴링을 통해 추가 통합을 활성화하거나, 해당 통합에 AWS 리소스를 추가하거나, 해당 통합을 더 많은 지역으로 확장할 때 이러한 상황이 발생할 수 있습니다.

AWS 일부 서비스에 대해 향상된 모니터링을 제공하여 더 높은 빈도로 더 많은 지표를 수집할 수 있도록 합니다. 자세한 내용은 [RDS 향상된 모니터링 비용을](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_Monitoring.OS.overview.html#USER_Monitoring.OS.cost) 확인하세요.

## 지역 가용성 [#region-availability]

대부분의 AWS 서비스는 클라우드 리소스와 실제 간의 데이터 지연 시간을 줄이기 위해 지역별 엔드포인트를 제공합니다. 뉴렐릭은 중국을 제외한 모든 [AWS 리전](http://docs.aws.amazon.com/general/latest/gr/rande.html) 에 위치한 서비스 및 엔드포인트로부터 모델링 데이터를 얻을 수 있습니다.
