`newrelic-infra-ctl` 유틸리티를 사용하여 인프라 에이전트의 문제를 해결할 수 있습니다. 이 바이너리는 `newrelic-infra` 기본 바이너리와 동일한 디렉토리의 설치 패키지에도 포함되어 있습니다.

`newrelic-infra-ctl` 명령을 수신하면 에이전트는 다음을 수행합니다.

* 5분 동안 [자세한 로그](/docs/infrastructure/new-relic-infrastructure/troubleshooting/generate-logs-troubleshooting-infrastructure) 를 활성화한 다음 로그 수준을 이전 설정으로 되돌립니다.
* 모든 에이전트 구성 옵션을 기록합니다.
* 모든 통합 구성 옵션을 기록합니다.
* 로드된 모든 통합에 대해 상태 확인을 실행합니다. (상태 확인은 추가 로그 및 출력 유효성 검사와의 통합을 즉시 실행하는 것입니다.)

## Linux `newrelic-infra-ctl` [#binary-ctl-linux]

Linux 시스템에서 문제 해결 바이너리는 [패키지 관리자](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-linux) 또는 [tarball 지원](/docs/infrastructure/install-configure-infrastructure/linux-installation/assisted-install-infrastructure-linux) 설치 방법 모두에서 사용할 수 있는 `/usr/bin/newrelic-infra-ctl` 입니다.

<Callout variant="important">
  Linux에서 실행할 때 `newrelic-infra-ctl` 은 [루트 사용자 또는](/docs/infrastructure/install-configure-infrastructure/linux-installation/linux-agent-running-modes) `newrelic-infra` 프로세스를 실행하는 동일한 사용자가 실행해야 합니다.
</Callout>

`newrelic-infra-ctl` 바이너리는 호스트에서 실행 중인 에이전트 프로세스를 자동으로 감지할 수 있습니다. 에이전트가 컨테이너 내부에서 실행 중인지 여부도 감지할 수 있습니다.

## 윈도우 `newrelic-infra-ctl` [#binary-ctl-windows]

Windows에서 MSI 설치 프로그램을 사용하는 경우 문제 해결 바이너리는 `C:\Program Files\New Relic\newrelic-infra\newrelic-infra-ctl.exe` 입니다.

<Callout variant="important">
  Windows에서 에이전트 프로세스는 항상 자동으로 감지됩니다. `pid` 또는 `cid` 에 종속되지 않습니다.
</Callout>
