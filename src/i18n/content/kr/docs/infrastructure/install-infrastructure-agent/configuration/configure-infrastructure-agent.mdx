인프라 에이전트는 다양한 방식으로 구성할 수 있습니다. 기본 방법은 `newrelic-infra.yml` 파일에서 [구성 변수](/docs/infrastructure/install-configure-infrastructure/configuration/infrastructure-configuration-settings) 를 편집하는 것입니다.

구성 파일 템플릿은 [인프라 구성 파일 템플릿](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 에서 사용할 수 있습니다. [환경 변수](#Environment_Variables) 를 설정하여 구성 파일을 재정의하거나 Ansible 또는 Puppet과 같은 [구성 관리 도구](#config-mgmt) 를 사용할 수도 있습니다.

인프라 에이전트와 작동하는 호스트 내 통합을 구성하려면 [특정 통합 문서](/docs/integrations/host-integrations/get-started/introduction-host-integrations) 를 참조하십시오.

인프라 에이전트를 사용해 보고 싶으십니까? 무료 [로 New Relic 계정을 만드](https://newrelic.com/signup) 십시오! 신용 카드가 필요하지 않습니다.

## 구성 방법 및 우선 순위 [#precedence]

인프라 에이전트를 구성하는 기본 방법은 `newrelic-infra.yml` 구성 파일입니다. 컨테이너화된 환경에 이상적인 환경 변수를 설정하여 구성 파일을 재정의할 수 있습니다.

각 구성 방법에 대한 자세한 설명은 다음과 같습니다.

<CollapserGroup>
  <Collapser
    id="config-file"
    title={<>구성 파일( <InlineCode>newrelic-infra.yml</InlineCode> ) 위치 및 설명</>}
  >
    인프라 에이전트를 구성하려면 `newrelic-infra.yml` 파일을 사용합니다. 기본 위치는 다음과 같습니다.

    * 리눅스: `/etc/newrelic-infra.yml`

    * Windows: `C:\Program Files\New Relic\newrelic-infra\newrelic-infra.yml`

      샘플 구성 파일은 [인프라 구성 파일 템플릿](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 을 참조하십시오.
  </Collapser>

  <Collapser
    id="Environment_Variables"
    title="환경 변수 구문 및 설명"
  >
    환경 변수를 설정하려면 구성 파일의 변수 이름을 모두 대문자로 사용하고 `NRIA_` 접두사를 사용하십시오. 예: `NRIA_LICENSE_KEY` . 컨테이너화된 환경에서만 환경 변수를 사용하십시오.

    [인프라 구성 파일 템플릿](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 에 설명된 옵션에는 `Env var` 레이블이 지정된 해당 환경 변수가 포함됩니다. 환경 변수는 [인프라 구성 설정](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings) 에도 나열됩니다.

    인프라 에이전트 1.14.0부터 환경 변수는 `{{MY_ENV_VAR}}` 표기법을 사용하여 구성 파일에 정의할 수도 있습니다. 예를 들면 다음과 같습니다.

    ```
    license_key: {{MY_LICENSE_KEY}}
    ```
  </Collapser>

  <Collapser
    id="passthrough"
    title="호스트 내 통합과 함께 환경 변수 패스스루 사용"
  >
    환경 변수를 사용 [하여 호스트 내 통합](/docs/infrastructure/host-integrations/host-integrations-list) 을 위한 구성 설정을 제어할 수 있으며, 이는 인프라 에이전트에 전달할 수 있습니다. 각 온-호스트 통합에 사용할 수 있는 변수는 각 통합에 대한 설명서에 나열되어 있으며 다음 방법 중 하나를 통해 설정할 수 있습니다.

    인프라 에이전트 1.24.1 passthrough_environment부터 변수 이름에 대한 정규식을 지원합니다.

    * **구성 파일 내부에서:** [인프라 구성 파일](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 에 항목을 추가하여 에이전트가 설정한 환경 변수를 가져올 수 있도록 합니다.

      ```
      passthrough_environment:
        - HTTPS_PROXY
        - HTTP_PROXY
        - HTTP.*
        - .* 
      ```

    * **명령줄에서:** 다른 환경 변수를 사용하여 이러한 설정을 전달합니다.

      ```
      NRIA_PASSTHROUGH_ENVIRONMENT="HTTPS_PROXY,HTTP_PROXY"
      ```

      각 통합에는 인수와 이름이 정확히 동일한 환경 변수를 정의하고 전달하여 덮어쓸 수 있는 변수 세트가 있습니다(예: `redis` 포트 인수를 대체하는 PORT 환경 변수).

      인프라 에이전트 1.14.0부터 `{{}}` 표기법을 사용하여 통합 구성 파일에서 환경 변수를 정의할 수 있습니다. 예를 들어:

      ```
      integration_name: com.newrelic.redis
      instances:
        - name: redis-metrics
          command: metrics
          arguments:
            hostname: localhost
            port: 6379
            password: {{MY_REDIS_PASSWORD}}
      ```

      <Callout variant="tip">
        `newrelic-infra` 서비스가 호스트에 정의된 모든 사용자 정의 환경 변수에 대한 가시성을 가지고 있는지 확인하십시오. [systemd와 같은 서비스 관리자는 기본적으로 모든 환경 변수를 설정하지 않으며](https://unix.stackexchange.com/questions/455261/how-to-set-environmental-variable-in-systemd-service) 추가 단계가 필요할 수 있습니다.
      </Callout>
  </Collapser>
</CollapserGroup>

<Callout variant="important">
  구성 설정을 더 잘 이해하려면 각 변수에 대한 자세한 설명이 포함된 [인프라 구성 설정](/docs/infrastructure/install-configure-infrastructure/configuration/infrastructure-configuration-settings) 을 참조하십시오.
</Callout>

## 구성 파일 구조 [#config_structure]

`newrelic-infra.yml` 편집 시 [YAML 형식](https://en.wikipedia.org/wiki/YAML) 을 준수하고 다음 규칙을 준수해야 합니다.

* 구성 옵션의 대소문자를 수정하지 마십시오. YAML 키는 대소문자를 구분합니다.
* 들여쓰기 수준을 준수합니다. 모든 들여쓰기는 일반적으로 두 개의 공백 문자로 비슷한 증분으로 되어 있습니다. 파일의 동일한 스탠자에 있는 데이터는 동일한 수준의 들여쓰기를 사용해야 합니다. 추가 2개의 공백으로 하위 스탠자를 들여씁니다( [사용자 정의 속성](/docs/infrastructure/install-configure-infrastructure/configuration/infrastructure-configuration-settings#custom-attributes) 및 [네트워크 인터페이스 필터](/docs/infrastructure/install-configure-infrastructure/configuration/infrastructure-configuration-settings#network-interface-filters) 의 예 참조).

`newrelic-infra.yml` 템플릿은 [인프라 에이전트 저장소](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 에서 사용할 수 있습니다. 인프라 에이전트와 함께 파일을 사용하기 전에 [유효성 검사기](http://www.yamllint.com/) 를 사용하여 구문이 정확한지 확인하십시오.

<Callout variant="important">
  설정을 변경한 후에는 항상 [에이전트 또는 웹 서버를 다시 시작하십시오](/docs/infrastructure-start-stop-restart-check-agent-status) .
</Callout>
