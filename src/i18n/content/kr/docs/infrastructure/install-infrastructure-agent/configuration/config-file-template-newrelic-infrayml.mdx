인프라 에이전트는 [구성](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent) 을 위해 `newrelic-infra.yml` 파일을 읽습니다. [GitHub의 인프라 에이전트 리포지토리](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 에서 최신 구성 템플릿을 찾을 수 있습니다.

<Callout variant="tip">
  형식 문제를 피하기 위해 YAML 구성 파일을 사용하기 전에 [린트](http://www.yamllint.com/) 하는 것이 좋습니다.
</Callout>

각 설정에 대한 자세한 내용은 [인프라 구성 설정](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings) 을 참조하십시오.

<Callout variant="important">
  설정을 변경한 후에는 항상 [에이전트 또는 웹 서버를 다시 시작하십시오](/docs/infrastructure-start-stop-restart-check-agent-status) .
</Callout>
