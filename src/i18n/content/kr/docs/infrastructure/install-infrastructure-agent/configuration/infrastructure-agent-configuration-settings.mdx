인프라 에이전트에는 [동작을 미세 조정하기 위한 많은 구성 설정](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent) 세트가 있습니다. 여기에서 우리는:

* 모든 구성 옵션을 나열합니다(YAML 및 환경 변수 이름 모두).
* 설정이 무엇을 하고 언제 사용하는지 설명합니다.
* 변수 유형과 기본값(있는 경우)을 제공합니다.
* 해당되는 경우 최소 필수 에이전트 버전을 나열합니다.

아직도 New Relic 계정이 없으신가요? 영원히 무료로 [가입하세요](https://newrelic.com/signup) .

## 시작하다 [#requirements]

[New Relic 계정](https://newrelic.com/signup) (영구 무료)을 만들고 인프라 에이전트를 [설치](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent/) 한 후 환경에 맞게 인프라 에이전트를 구성할 수 있습니다.

[`license_key`](#license-key) 만 필수 설정입니다.

이러한 모든 변수를 사용하는 방법에 대한 예 [는 GitHub의 샘플 구성 템플릿을](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example) 참조하십시오.

## 클라우드 변수

에이전트가 클라우드 인스턴스에서 실행 중인 경우 에이전트는 클라우드 유형을 감지하고 기본 [메타데이터](/docs/integrations/new-relic-integrations/getting-started/understand-use-data-infrastructure-integrations#overview) 를 가져오려고 합니다. [클라우드 공급자 계정을 New Relic과 연결할](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations#cloud) 때 확장된 클라우드 메타데이터(사용자 지정 리소스 태그 포함)로 메트릭을 보강할 수도 있습니다.

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="custom-attributes"
    title="custom_attributes"
  >
    사용자 정의 속성은 인프라 에이전트의 데이터에 주석을 추가하는 데 사용되는 키-값 쌍(다른 도구의 태그와 유사)입니다. 이 메타데이터를 사용 [하여 필터 세트를 만들고](/docs/scopes-organize-your-infrastructure) [결과를 그룹화](/docs/infrastructure-group-results-specific-attributes) 하고 데이터에 주석을 추가할 수 있습니다. 예를 들어, 시스템 환경(스테이징 또는 프로덕션), 시스템이 호스팅하는 서비스(예: 로그인 서비스) 또는 해당 시스템을 담당하는 팀을 나타낼 수 있습니다.

    <Callout variant="tip">
      에이전트는 [Amazon Elastic Compute Cloud(Amazon EC2) 태그](/docs/default-infrastructure-events-attributes#aws-ec2-attributes) 를 포함하여 [기본 속성](/docs/default-infrastructure-events-attributes) 의 일부로 환경에 대한 많은 세부 정보를 수집합니다.
    </Callout>

    <table>
      <thead>
        <tr>
          <th style={{ width: "250px" }}>
            YML 옵션 이름
          </th>

          <th style={{ width: "250px" }}>
            환경 변수
          </th>

          <th>
            유형
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `custom_attributes`
          </td>

          <td>
            `NRIA_CUSTOM_ATTRIBUTES`
          </td>

          <td>
            지도\[문자열]인터페이스&#x7B;}
          </td>
        </tr>
      </tbody>
    </table>

    사용자 정의 속성 목록을 사용하여 이 에이전트 인스턴스의 데이터에 주석을 추가하십시오. `KEY: VALUE` 에서와 같이 콜론 `:` 으로 키와 값을 구분하고 각 키-값 쌍을 줄 바꿈으로 구분합니다. 키는 슬래시 `/` 를 제외한 모든 유효한 YAML이 될 수 있습니다. 값은 공백을 포함한 모든 YAML 문자열이 될 수 있습니다.

    YAML 속성의 예:

    ```
    custom_attributes:
      environment: production
      service: login service
      team: alpha-team
    ```

    환경 변수의 예:

    ```
    NRIA_CUSTOM_ATTRIBUTES='{"<var>customAttribute_1</var>":"<var>SOME_ATTRIBUTE</var>","<var>customAttribute_2</var>": "<var>SOME_ATTRIBUTE_2</var>"}'
    ```

    <Callout variant="important">

  <Collapser
    className="freq-link"
    id="include-matching-metrics"
    title="include_matching_metrics"
  >
    <Callout variant="important">
      현재 이 설정은 운영 체제의 프로세스 메트릭에만 적용됩니다.
    </Callout>

    메트릭 [속성](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/query-infrastructure-dimensional-metrics-nrql#naming-conventions) 값을 기반으로 메트릭 데이터 전송을 제한할 수 있는 `include_matching_metrics` 을 구성하여 New Relic으로 전송되는 데이터의 양을 제어할 수 있습니다.

    메트릭 속성에 대한 리터럴 또는 부분 값을 정의하여 메트릭 데이터를 포함합니다. 예를 들어, `process.name` 이 `^java` [정규식](https://github.com/google/re2/wiki/Syntax) 과 일치하는 모든 프로세스의 `host.process.cpuPercent` 를 보내도록 선택할 수 있습니다.

    이 예에서는 실행 파일과 이름을 사용하여 프로세스 메트릭을 포함합니다.

    ```
    include_matching_metrics: # You can combine attributes from different metrics
        process.name:
          - regex “^java”    # Include all processes starting with "java"
        process.executable:
          - “/usr/bin/python2”              # Include the Python 2.x executable
          - regex “\\System32\\svchost”     # Include all svchost executables
    ```

    값에 명령줄 인수를 포함해야 하는 경우 [`strip_command_line`](#strip-command-line) 을 false로 설정합니다(인프라 에이전트는 기본적으로 비밀 누출을 방지하기 위해 CLI 인수를 제거합니다).

    `include_matching_metrics` 을 [Kubernetes 통합](/docs/integrations/kubernetes-integration) 을 위한 환경 변수로 구성하려면 `env:` 객체 내의 매니페스트에 추가합니다.

    ```
    env:
        - name: NRIA_INCLUDE_MATCHING_METRICS
          value: |
            process.name:
              - regex "^java"
            process.executable:
              - "/usr/bin/python2"
              - regex "\\System32\\svchost"
    ```
