뉴렐릭의 인프라 모니터링 에이전트는 [호스트에 대한 데이터](/docs/infrastructure/manage-your-data/data-instrumentation/default-infrastructure-events)를 수집하는 [경량](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead) 실행 파일입니다. 또한 [호스트 통합](/docs/integrations/infrastructure-integrations/get-started/introduction-infrastructure-integrations)에서 뉴렐릭으로 데이터와 로그 분석을 위한 로그 [데이터](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent)를 전달합니다.

인프라 모니터링 에이전트는 현재 많은 Linux 배포판, Windows 및 macOS에서 실행할 수 있습니다. 설정 및 요구 사항에 따라 에이전트를 설치하고 배포하는 방법은 여러 가지가 있습니다. 이 문서에서는 인프라 모니터링 에이전트의 작동 방식과 설치 방법에 대해 설명합니다. 뉴렐릭 계정이 필요하므로 계정이 없으시다면, 지금 영구 무료 계정을 [신청하십시오](https://newrelic.com/signup).

## 빠른 시작: 설치 안내 사용 [#quick]

## 인프라 모니터링 에이전트 설치 [#install]

### Linux [#Linux]

뉴렐릭 계정이 없으면 설치 안내가 작동하지 않습니다. 절차를 수동으로 따르려면 [튜토리얼을 참조](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/)하십시오.

### Windows 서버 및 10 [#Windows]

뉴렐릭 계정이 없으면 설치 안내가 작동하지 않습니다. MSI 설치 프로그램을 사용하여 수동으로 절차를 따르려면 [튜토리얼을 참조](/docs/infrastructure/install-configure-manage-infrastructure/windows-installation/install-infrastructure-windows-server-using-msi-installer)하십시오.

## 하나의 에이전트, 다양한 기능 [#overview]

인프라 모니터링 에이전트는 시스템 리소스 및 활성화된 호스트(온프레미스 또는 가상화)의 프로세스에 대한 성능 및 상태 데이터를 수집합니다. 동시에 두 가지 유형의 데이터 즉, [온호스트 통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations)에 의해 수집되는 핵심 서비스 메트릭과 [로그](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent)에 대한 전달자 역할을 합니다.

호스트에서 실행 중인 핵심 서비스에 대한 데이터를 수집하려면 먼저 [인프라 모니터링 에이전트](#install)를 설치한 다음 [온호스트 통합을 설치하거나 활성화해야 합니다](/docs/integrations/host-integrations/installation/install-infrastructure-host-integrations).

## 소스 코드 확인 [#source-code]

인프라 모니터링 에이전트는 오픈소스 소프트웨어입니다. 즉[, 소스 코드를 탐색하고](https://github.com/newrelic/infrastructure-agent) [개선 사항을 보내](https://github.com/newrelic/infrastructure-agent/blob/master/CONTRIBUTING.md)거나 고유한 포크를 만들어 빌드할 수 있습니다. 자세한 내용은 [README](https://github.com/newrelic/infrastructure-agent/blob/master/README.md)를 참조하십시오.

## 다음 단계

인프라 모니터링 에이전트를 설치한 후:

* [에이전트를 구성](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent)하거나 [구성 템플릿](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example)을 수정하는 방법을 알아보십시오.
* [호스트 통합](/docs/integrations/host-integrations/getting-started/introduction-host-integrations)(예: Apache 또는 MySQL)을 설치합니다.
* [인프라 에이전트를 사용하여 로그 전달을 활성화](/docs/logs/enable-new-relic-logs/1-enable-logs/forward-your-logs-using-new-relic-infrastructure)합니다.
* [에이전트 관리](/docs/infrastructure/install-infrastructure-agent/manage-your-agent) 방법을 알아보십시오.
