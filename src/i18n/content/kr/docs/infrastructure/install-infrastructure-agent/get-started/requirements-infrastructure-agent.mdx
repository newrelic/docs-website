[인프라 에이전트를 설치](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic) 하기 전에 시스템과 구성한 호스트 내 통합이 요구 사항을 충족하는지 확인하십시오. 또한 New Relic 계정이 필요합니다. [무료로 가입하세요](https://newrelic.com/signup) . 신용 카드가 필요하지 않습니다.

## 프로세서 아키텍처 [#processors]

인프라 에이전트는 다음 프로세서 아키텍처를 지원합니다.

* **Linux** : x86 프로세서 아키텍처용 64비트(64비트 패키지 관리자 및 종속성도 필요)
* **Windows** : x86 프로세서 아키텍처의 경우 32비트 및 64비트 모두
* **ARM** : [AWS Graviton 2 프로세서](https://aws.amazon.com/ec2/graviton/) 를 포함한 arm64 아키텍처는 호환되는 Linux 운영 체제에서 지원됩니다. 호스트 내 통합도 지원됩니다(Oracle 통합 제외).
* **macOS** : 64비트 x86 프로세서(ARM 프로세서는 아직 지원되지 않음).

## 고유한 호스트 이름 [#hostname]

인프라 에이전트는 호스트 이름을 사용하여 각 서버를 고유하게 식별합니다. 단일 호스트 이름으로 여러 서버를 결합하는 부정확한 메트릭을 방지하려면 모니터링되는 각 서버에 고유한 호스트 이름이 있는지 확인하십시오.

* 선택적 [`display_name`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#display_name) 설정을 사용하여 기본 호스트 이름을 재정의할 수 있습니다.
* 이름이 `localhost` 인 서버는 기본 이름이고 본질적으로 고유하지 않기 때문에 보고되지 않습니다.

## 권한

인프라 에이전트에는 다음 권한이 필요합니다.

* **Linux** : 기본적으로 [에이전트는 root 로 실행 및 설치됩니다](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#agent-mode-intro) . 권한 있는 실행 모드 또는 [권한](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-privileged) [없는](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-unprivileged) 실행 모드를 선택할 수도 있습니다.
* **Windows** : 에이전트는 관리자 계정으로 설치해야 하며 실행하려면 관리자 권한이 필요합니다.
* **macOS** : 에이전트는 모든 사용자 계정에서 설치할 수 있습니다.

## 도서관

[에이전트 버전 1.1.19 이상의 경우](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) 인프라를 설치하려면 `libcap` 라이브러리가 필요합니다. 배포의 공식 리포지토리에서 사용할 수 있습니다.

## 네트워크 액세스 [#network]

New Relic에 데이터를 보고하려면 인프라 에이전트가 [특정 도메인 및 포트](/docs/apm/new-relic-apm/getting-started/networks#infrastructure) 에 대한 아웃바운드 액세스 권한이 있어야 합니다. 시스템에서 이러한 도메인에 연결하기 위해 프록시가 필요한 경우 [`proxy`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#proxy) 설정을 사용합니다.

## 컨테이너 소프트웨어 [#container]

인프라 에이전트는 호스트 서버에 설치될 때 [Docker 컨테이너를 계측](/docs/infrastructure-docker-instrumentation) 합니다. Docker 버전 1.12 이상을 지원합니다.

## CPU, 메모리 및 디스크 사용량 [#resources]

인프라 에이전트는 상당히 가볍습니다. 일반적인 CPU, 메모리 및 디스크 사용량은 [에이전트 성능 오버헤드](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead) 페이지를 참조하십시오. 지원되는 파일 시스템에 대한 자세한 정보는 [스토리지 샘플 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#storagesample-attributes) 을 참조하십시오.
