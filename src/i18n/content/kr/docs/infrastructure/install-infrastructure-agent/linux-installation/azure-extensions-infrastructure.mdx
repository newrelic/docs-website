[Microsoft Azure 확장](https://docs.microsoft.com/en-us/azure/virtual-machines/extensions/overview) 을 사용하여 [인프라](/docs/welcome-new-relic-infrastructure) 에이전트를 설치할 수 있습니다.

<CollapserGroup>
  <Collapser
    id="what-is-extension"
    title="Azure 확장이란 무엇입니까?"
  >
    Azure VM(가상 머신) 확장은 Azure VM에서 배포 후 구성 및 자동화 작업을 제공하는 작은 애플리케이션입니다. 예를 들어 가상 머신에 소프트웨어 설치, 바이러스 백신 보호 또는 스크립트 실행이 필요한 경우 VM 확장을 사용할 수 있습니다. Azure CLI, PowerShell, Azure Resource Manager 템플릿 및 Azure Portal은 모두 Azure VM 확장을 실행할 수 있습니다. 확장은 새 VM 배포와 함께 번들로 제공되거나 기존 시스템에 대해 실행할 수 있습니다.
  </Collapser>
</CollapserGroup>

New Relic은 이러한 확장을 사용하여 인프라 에이전트의 설치를 용이하게 합니다. 이렇게 하면 에이전트를 수동으로 업데이트할 필요가 없으며, 더 중요한 것은 [Azure 확장 집합을](https://docs.microsoft.com/en-us/azure/virtual-machine-scale-sets/overview)사용하여 에이전트를 자동으로 확장합니다.

## Azure 확장을 사용하여 인프라 설치 [#install-infra-azure-extensions]

지원되는 [운영 체제](https://docs.newrelic.com/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/)에서 Azure 확장을 사용하여 에이전트를 설치하려면 .

1. 확장 집합을 사용하여 Azure VM, Azure 확장 집합 또는 Azure Service Fabric으로 이동하고 **확장** 탭을 선택합니다.

   <img
     title="azure_ext01.png"
     alt="azure_ext01.png"
     src={infrastructureAzureExtensionsTab}
   />

2. **New Relic Infrastructure** 확장을 선택합니다.

   <img
     title="azure_ext02.png"
     alt="azure_ext02.png"
     src={infrastructureAzureNrInfrastructureExtension}
   />

3. 지침에 따라 <InlinePopover type="licenseKey" /> 를 추가합니다.

   <img
     title="azure_ext03.png"
     alt="azure_ext03.png"
     src={infrastructureAzureNrLicenseKey}
   />

4. 가도 돼!
