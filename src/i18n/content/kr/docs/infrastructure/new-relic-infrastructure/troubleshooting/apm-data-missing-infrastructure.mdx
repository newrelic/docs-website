## 문제

APM 및 인프라 에이전트가 동일한 호스트에 설치되고 동일한 New Relic 라이센스 키를 사용하는 경우 APM 데이터가 **인프라** 에 나타나야 하며 그 반대의 경우도 마찬가지입니다. 이 APM-인프라 연결이 표시되지 않으면 다음 문제 해결 팁을 따르십시오.

## 해결책

[APM/인프라 통합](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure) 을 완료했지만 **인프라** 에 APM 데이터가 표시되지 않는 경우 다음 문제 해결 절차를 시도하십시오.

<CollapserGroup>
  <Collapser
    id="restart-app-server"
    title="앱 서버를 다시 시작합니다."
  >
    몇 주 또는 몇 달 동안 APM 모니터링 애플리케이션을 다시 시작하지 않은 경우 인프라 및 APM의 데이터 스트림이 연결되지 않을 수 있습니다.

    1. 앱 서버를 다시 시작합니다.
    2. 앱에 대한 트래픽을 생성합니다.
    3. 잠시 기다렸다가 [인프라 모니터링에서 APM 데이터](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure) 를 확인합니다.
  </Collapser>

  <Collapser
    id="hostnames"
    title="호스트 이름이 APM과 인프라에서 동일한지 확인하십시오."
  >
    APM 및 인프라 모니터링에서 호스트 이름이 다른 경우 New Relic은 데이터를 통합할 수 없습니다. 이 문제의 일반적인 원인 중 하나는 기본 호스트 이름이 다르기 때문입니다. 예를 들어 인프라 모니터링은 호스트의 FQDN(예: `myhost1.example.com` )을 사용하는 반면 APM은 호스트 이름(예: `myhost1` )을 사용합니다.

    1. **\[** .newrelic.com **]( [http://one.newrelic.com](http://one.newrelic.com) ) > APM > (앱 선택) 으로** 이동 [**합니다**](http://one.newrelic.com) .
    2. 앱의 APM **개요** 페이지에서 앱과 연결된 호스트 이름을 확인합니다. 인프라 모니터링에서 해당 이름을 동일한 호스트 이름과 비교하십시오.
    3. 이름이 다른 경우 FQDN과 일치 [하도록 APM 에이전트 호스트의 `display_name`](/docs/agents/manage-apm-agents/configuration/add-rename-remove-hosts#display_name) 를 설정하거나 APM에 설정된 것과 일치하도록 [인프라의 호스트 `display_name` 를](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#display_name) 설정합니다.
  </Collapser>

  <Collapser
    id="gap-fqdn-issue"
    title="교체 호스트 FQDN 인식 문제를 확인합니다."
  >
    APM-인프라 통합이 이전에 작동했지만 중지된 경우 서버가 동일한 FQDN을 가진 다른 서버로 교체되었을 수 있습니다. 일정 시간 동안 두 서버가 동시에 존재했다면 New Relic은 새 서버를 자동으로 인식하지 못합니다. 그러면 APM과 인프라 데이터 간의 연결이 끊어집니다.

    * 이 문제를 해결하려면 [support.newrelic.com](https://support.newrelic.com/tickets/new) 에서 도움을 받으십시오.
    * 이 문제를 방지하려면 기존 서버를 중단하고 새 서버를 만드는 사이에 시간 간격이 있는지 확인하십시오.
  </Collapser>
</CollapserGroup>
