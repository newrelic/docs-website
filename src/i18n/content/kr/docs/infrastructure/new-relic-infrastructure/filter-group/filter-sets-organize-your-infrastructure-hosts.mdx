New Relic 인프라 모니터링을 사용하면 필터를 필터 세트로 결합하여 가장 중요한 기준에 따라 호스트를 구성할 수 있습니다. 귀하의 계정에는 보고하는 수백 개의 호스트가 있을 수 있으므로 효과적인 문제 해결을 위해서는 중요한 사항에 집중하는 것이 중요합니다. 필터 세트를 효율적으로 사용하는 방법을 배우려면 이 짧은 비디오(약 2분 50초)를 시청하십시오.

## 카테고리 내 패턴 보기 [#benefits]

사용 가능한 [속성](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute) 또는 [태그](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) 를 사용하여 필터 세트를 생성할 수 있습니다.

예를 들어 인프라를 다음과 같은 범주로 구성할 수 있습니다.

* 지역
* 운영 체제 버전
* Docker 컨테이너와 연결된 호스트
* 테스트 환경

필터 세트를 계정의 다른 사람들과 공유할 수 있으며 필터 세트에 있는 각 호스트의 색상으로 구분된 [상태](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status) 를 확인하여 인프라 문제를 빠르게 식별할 수 있습니다.

## 필터 세트 생성 [#create]

기본 인프라 필터 세트는 **모든 호스트** 이며 필터 세트를 생성하기 위한 템플릿 역할을 합니다.

필터 세트를 생성하려면:

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure** 로 이동하고 **Hosts** , **Inventory** 또는 **Events** 를 클릭합니다.

2. **All hosts** 가 왼쪽 사이드바에 표시되지 않으면 **Saved filter sets > All hosts** 를 선택하여 해당 필터 세트를 엽니다.

3. **모든 호스트** 에서 호스트 **필터링** 을 클릭합니다.

4. 목록에서 항목을 클릭하면 값 목록이 표시됩니다.

5. **포함** 또는 **제외** 를 클릭합니다( [필터 집합 논리](#logic) 참조).

6. 값을 개별적으로 클릭하거나 여러 값과 일치하도록 텍스트를 입력합니다.

7. 원하는 필터 세트를 얻을 때까지 필터를 계속 추가합니다.

8. 필터 이름을 지정하려면

   <Icon name="fe-edit-2"/>

   아이콘을 클릭하고 이름을 입력하고 **저장** 을 클릭합니다.

## 필터 세트 편집 [#edit]

기존 필터 세트를 변경하려면:

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure** 로 이동하고 **Hosts** , **Inventory** 또는 **Events** 를 클릭합니다.
2. 사이드바에서 **저장된 필터 세트** 를 클릭하여 목록을 엽니다.
3. 스크롤하거나 검색어를 입력하여 필터 세트를 찾습니다.
4. 필터 세트를 클릭하여 엽니다.
5. 사이드바에서 옵션을 클릭하여 필터 세트를 업데이트한 다음 저장합니다.

## 필터 세트 삭제 [#delete]

기본 **모든 호스트** 를 제외하고 저장된 필터 세트를 삭제할 수 있습니다.

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure** **> Settings > Filter sets** 로 이동합니다.

2. 딸깍 하는 소리

   <Icon name="fe-trash-2"/>

   필터 세트를 삭제합니다.

## 그룹화와 필터 세트 결합 [#grouping]

일부 페이지에서는 [**그룹화**](/docs/infrastructure-group-results-specific-attributes) 기준을 사용하여 특정 속성별로 차트 결과를 그룹화할 수 있습니다. 예를 들어 **호스트** 페이지에서 `awsRegion` 별로 그룹화하여 CPU 사용량이 가장 높은 AWS 리전을 표시할 수 있습니다.

그룹화는 선택한 필터 세트에 적용됩니다. 필터 세트를 그룹화와 결합하여 자세한 시스템 정보를 빠르게 찾을 수 있습니다. 이러한 도구를 사용하여 문제를 해결하는 예는 [필터 세트 및 그룹화 결합](/docs/infrastructure-group-results-specific-attributes#scoping-and-grouping) 을 참조하십시오.

## 필터 세트에서 경고로 필터 복사 [#alerts]

경고 조건을 만들 때 필터를 개별적으로 작성하거나 필터 세트의 모든 필터를 새 경고 조건으로 복사할 수 있습니다. 이것은 일부 필터로 새 경고 조건을 채우는 빠른 바로 가기입니다.

<Callout variant="important">
  필터 집합 필터를 새 경고 조건에 복사하면 이러한 필터가 더 이상 필터 집합에 연결되지 않습니다. 필터 세트를 변경해도 경고 필터는 영향을 받지 않습니다.
</Callout>

필터 세트 필터를 새 경고 조건으로 복사하려면:

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure** 로 이동하고 **Hosts** , **Inventory** 또는 **Events** 를 클릭합니다.
2. 사이드바에서 **저장된 필터 세트** 를 클릭하여 목록을 엽니다.
3. 스크롤하거나 검색어를 입력하여 필터 세트를 찾습니다.
4. 필터 세트를 클릭하여 엽니다.
5. 차트 위에 마우스를 놓고 클릭하십시오. **<Icon name="fe-more-horizontal"/>> Create alert** .
6. 경고 조건 이름을 입력합니다.
7. 필요에 따라 필터를 조정하십시오.
8. 나머지 경고 필드를 완료합니다( [경고 조건 생성](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#create-condition) 참조).

## 필터 세트 로직 [#logic]

### 그리고/또는

필터 세트는 백그라운드에서 논리 연산자 `AND` 및 `OR` 를 사용하여 데이터를 결합합니다.

* 여러 속성 또는 태그의 값을 클릭하면 `AND` 으로 결합됩니다.
* 속성 또는 태그 내에서 값을 클릭하면 `OR` 으로 결합됩니다.

필터 결과는 다음 두 가지 모두에 해당하는 호스트를 표시합니다.

* 선택한 인프라 에이전트 버전 중 하나를 포함하는 호스트
* 선택한 AWS 가용 영역 중 하나의 호스트
