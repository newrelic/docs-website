인프라 모니터링 도구에서 **Group by** 기능을 사용하여 특정 속성별로 차트 결과를 그룹화할 수 있습니다. 예를 들어 호스트 페이지에서 CPU 사용량이 가장 높은 AWS 리전을 `awsRegion` 으로 그룹화하여 표시할 수 있습니다.

**Group by** 는 일부 [인프라 모니터링 UI 페이지](/docs/infrastructure/infrastructure-ui-pages/infrastructure-ui/infrastructure-hosts-page) 의 상단 근처에서 사용할 수 있습니다.

## 특정 속성별로 차트 그룹화 [#grouping]

일부 인프라 페이지에서는 **Group by** 를 사용하여 호스트 이름, 엔터티 ID 또는 AWS 리전과 같은 특정 속성별로 페이지 결과 및 차트를 그룹화할 수 있습니다. 그룹화 기준으로 사용할 수 있는 속성은 시스템 설정에 따라 다릅니다. 여기에는 다음이 포함될 수 있습니다.

* [기본](/docs/infrastructure-events-attributes) 인프라 속성
* [사용자 정의 속성](/docs/infrastructure-configure-your-agent#attributes)
* [APM 관련 속성](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure#filter-by-app)

특정 속성별로 인프라 결과를 그룹화하려면 다음을 수행합니다.

1. 이 기능이 있는 페이지에서 **그룹화** 기준(시간 선택기 옆에 있음)을 선택합니다.
2. 드롭다운에서 그룹화 기준으로 사용할 속성을 선택합니다.

## 필터 세트 및 그룹화 결합 [#scoping-and-grouping]

그룹화는 선택한 모든 **[필터 세트](/docs/infrastructure-organize-your-resources-scopes)** 에 적용됩니다. 필터 세트를 **Group by** 와 결합하여 상세한 시스템 정보를 빠르게 찾을 수 있습니다.
