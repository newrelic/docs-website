New Relic의 인프라 모니터링을 사용하면 현재 New Relic으로 모니터링 중인 컨텍스트 내에서 직접 [경고 조건](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-condition) 을 생성할 수 있습니다.

예를 들어 [필터 세트](/docs/infrastructure/new-relic-infrastructure/filter-group/organize-infrastructure-hosts-filter-sets) 를 모니터링하고 문제를 발견하면 경고를 직접 만들 수 있으며 [경고](/docs/alerts/new-relic-alerts/getting-started/introduction-new-relic-alerts) 에서 다시 만들 필요가 없습니다. 경고를 생성하려면 즉시 필터 세트를 선택하고 보고 있는 차트에서 직접 경고 조건을 조정하십시오. 이를 통해 환경에 대한 경고 시스템을 사전에 관리하고 모니터링할 수 있습니다. 모든 경고 위반은 필터 세트 내의 엔터티별로 생성됩니다.

인프라 에이전트로 경보 조건을 시험해보고 싶으십니까? 무료 [로 New Relic 계정을 만드](https://newrelic.com/signup) 십시오! 신용 카드가 필요하지 않습니다.

## 인프라에 대한 경고 조건 생성 [#create-condition]

경고 조건은 경고 [정책](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-policy) 에 적용됩니다. 인프라 모니터링 UI에서 이메일 알림을 사용하여 기존 정책을 선택하거나 새 정책을 생성할 수 있습니다. 다른 유형의 [알림 채널](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts) 을 사용하려면 알림 UI 내에서 [새 정책](/docs/alerts/new-relic-alerts/configuring-alert-policies/name-or-rename-alert-policy) 을 만드세요.

<Callout
  id="rest-api-alert-limit"
  variant="important"
>
  Infrastructure REST API는 활성 **및** 비활성화 조건을 모두 포함하여 3,700개의 경고 조건으로 제한됩니다. API는 직접 사용하든 UI를 통해 사용하든 3,700 경고 조건 제한을 초과하는 추가 경고 조건을 추가하라는 모든 요청을 거부합니다.
</Callout>

알림 정책에 인프라 알림 조건을 추가하려면:

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure** 로 이동한 다음 인프라 모니터링 페이지( [**Hosts**](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-compute-page-measure-resource-usage) , [**Processes**](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-processes-page-inspect-process-performance) , [**Network**](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-network-page-measure-compare-capacity) 또는 [**Storage**](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-storage-page-evaluate-disk-usage-efficiency) ) 중 하나를 선택합니다. 경고하려는 차트 위로 마우스를 가져간 다음 타원을 선택합니다. <Icon name="fe-more-horizontal"/>아이콘을 클릭한 다음 **Create alert** 을 선택합니다.

2. 의미 있는 조건 이름을 입력합니다.

3. **경보 유형** 을 선택하거나 [예](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerting-examples) 를 참조하여 선택할 유형을 결정하십시오.

4. 개별 필터를 생성하거나 [필터 세트](/docs/infrastructure/new-relic-infrastructure/filter-group/filter-sets-organize-your-infrastructure-hosts) 에서 모든 필터를 복사하여 경고 조건에서 사용할 호스트를 식별합니다.

   <Callout variant="important">
     필터 이면의 규칙에 대한 자세한 내용은 [필터 집합 논리](/docs/infrastructure/new-relic-infrastructure/filter-group/filter-sets-organize-your-infrastructure-hosts#logic) 를 참조하세요.
   </Callout>

5. 경고 알림을 트리거하기 위한 **위험** (필수) 및 **경고** (사용 가능한 경우 선택) [임계값](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-threshold) 을 정의합니다.

6. 선택 사항: 조건 기준을 사전에 생성하지만 지금은 경고 알림을 받지 않으려면 [**사용** 확인란 옵션](/docs/infrastructure/new-relic-infrastructure/configuration/infrastructure-alerts-add-edit-or-view-host-alert-information#enabled) 을 끕니다.

7. 새 조건에 대한 기존 정책을 선택합니다.

   또는

   새 정책을 만들고 경고 알림에 대한 전자 메일을 식별하는 옵션을 선택합니다.

8. 선택 사항: [Runbook URL](#runbook-url-infrastructure) 을 추가합니다.

9. 선택 사항: [위반에 대한 **위반 시간 제한** 을](#violation-time-limit) 설정합니다(기본값은 24시간).

10. **만들기** 를 선택합니다.

<Callout variant="important">
  New Relic이 지난 60분 동안 클라우드 통합 서비스의 속성을 받지 못한 경우 이를 "자동 속성"이라고 하며 UI에서 경고 조건으로 사용할 수 없습니다. 이 상황에서 [API](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/rest-api-calls-new-relic-infrastructure-alerts/) 를 사용하여 자동 속성에 대한 경고 조건을 생성할 수 있습니다.
</Callout>

## 호스트 경고 이벤트 보기 [#view-alerts]

정책의 [알림 채널](#notification) 에 포함된 모든 사용자는 경고 알림을 직접 받습니다. 또한 New Relic 계정에 대한 권한이 있는 사람은 누구나 사용자 인터페이스를 통해 인프라 경고 사건 및 개별 위반을 볼 수 있습니다.

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure >** **Events** 로 이동합니다.
2. 호스트 또는 시간 프레임을 변경하려면 검색 창, **필터 세트** 또는 **시간** 기능을 사용하십시오.
3. **이벤트** 목록에서 경고 위반을 선택합니다.
4. 선택한 위반에 대한 [경고에서 자세한 정보를 보려면](/docs/alerts/new-relic-alerts/reviewing-alert-incidents/view-incident-violation-details) 링크를 선택하십시오.

## 호스트 경고 정보 업데이트 또는 삭제 [#maintain-alerts]

호스트 경고 정보를 편집, 비활성화(또는 다시 활성화) 또는 삭제하려면:

1. **[one.newrelic.com](http://one.newrelic.com) > Infrastructure >** **Settings > Alerts** 로 이동합니다.

2. 선택 사항: 검색 창을 사용하거나 **모두** 선택 확인란을 사용하여 하나 이상의 경고 조건을 찾습니다.

3. 사용 가능한 기능을 선택하여

   <Icon name="fe-edit-2"/>

   편집, 비활성화, 활성화 또는

   <Icon name="fe-trash-2"/>

   선택한 조건을 삭제합니다.

## 설명 추가 [#description]

**설명** 필드는 다음 경고 조건 유형에 사용할 수 있습니다.

* NRQL 조건: [NerdGraph API](/docs/alerts/alerts-nerdgraph/nerdgraph-examples/nerdgraph-api-alerts-nrql-conditions) 를 사용하여 설명을 추가합니다.
* 인프라 조건: UI 또는 REST API를 사용하여 설명을 추가합니다.

경고 조건의 **설명** 필드에 입력한 텍스트는 관련 위반 및 알림으로 다운스트림으로 전달됩니다. 설명은 다음을 포함하여 여러 목적으로 사용될 수 있습니다.

* 경고 조건의 원인을 캡처합니다.
* 모니터링되는 신호를 정의합니다.
* 다음 단계를 정의합니다.
* 다운스트림 시스템에 메타데이터를 추가합니다.

템플리트 대체를 사용하여 연관된 위반 이벤트의 속성 값을 삽입할 수 있습니다. 템플릿 형식은 `{{attributeName}}` 입니다. 설명을 작성할 때 사용할 수 있는 속성은 [위반 이벤트 속성](/docs/alerts/new-relic-alerts/defining-conditions/violation-event-attributes) 을 참조하십시오.

사용 가능한 속성 중 하나는 특수 `{{tag.*}}` 속성입니다. 이 속성 접두사는 대상 신호에 포함된 태그 값 또는 대상 신호와 연결된 엔터티 태그에 액세스하는 데 사용됩니다.

위반과 관련된 엔터티 태그가 있는 경우 엔터티 태그 이름을 사용하여 액세스할 수 있습니다. 이에 대한 예는 `{{tag.aws.awsRegion}}` 입니다. 엔티티 태그를 사용할 수 있는 경우 위반에 포함된 것을 볼 수 있으며 인시던트의 위반을 볼 때 표시됩니다.

이 필드의 최대 문자 크기는 4,000자입니다.

## Runbook URL 추가 또는 편집 [#runbook-url-infrastructure]

[경고 조건 생성 프로세스](#create-condition) 에는 Runbook 지침에 대한 URL을 설정하는 옵션이 포함됩니다. 이를 통해 위반 처리를 위한 정보 또는 표준 절차에 연결할 수 있습니다. 링크를 추가하거나 업데이트하기 전에 유효한 URL을 사용하고 있는지 확인하십시오.

경고 조건의 Runbook URL을 추가, 업데이트 또는 삭제하려면:

1. 경고 조건을 선택하고 **Runbook URL** 링크를 변경합니다.
2. 조건을 저장합니다.

저장하려면 URL이 유효한 URL이어야 합니다.

## 위반에 대한 위반 시간 제한 [#violation-time-limit]

위반 시간 제한을 통해 위반이 강제 종료되는 기간을 정의할 수 있습니다. 기본적으로 위반 시간 제한은 24시간입니다.

경고 조건의 위반 시간 제한을 추가하거나 업데이트하려면:

1. 경고 조건을 선택하고 위반 시간 제한을 변경합니다.
2. 조건을 저장합니다.

## 너무 긴 NRQL 쿼리를 생성하는 경고 조건 [#alert-nrql-error]

인프라에 대해 생성된 경고 조건은 비하인드 스토리 NRQL 쿼리에 의존하며 NRQL 쿼리에는 4096자 제한이 있습니다. 즉, 조건이 많은 요소(예: 많은 호스트 또는 많은 태그 포함)를 필터링하는 매우 복잡한 NRQL 쿼리를 생성하는 경우 이 제한을 초과하고 조건이 실패했다는 오류 메시지를 표시합니다.

이 문제를 해결하려면 경고 조건에서 사용 중인 요소의 수를 줄이십시오. 예를 들어:

  <thead>
    <tr>

        문제
      </th>

      <th>
        해결책
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        호스트
      </td>

      <td>
        * 조건을 실패하게 만든 많은 수의 호스트를 입력한 경우 호스트 수를 줄이십시오.
        * 하위 문자열을 사용하여 호스트를 대상으로 지정합니다. 예를 들어 `prod-host-01` , `prod-host-02` 및 `prod-host-03` 를 타겟팅하는 대신 이름에 `prod-host-0` 이 있는 모든 호스트를 타겟팅하면 됩니다.
      </td>
    </tr>

    <tr>
      <td>
        엔티티
      </td>

      <td>
        * 대상으로 지정하려는 엔터티에 적용되는 특정 속성을 대상으로 알림 조건을 편집합니다.
        * 대상으로 지정하려는 엔터티에 대한 사용자 지정 속성을 만들고 경고 조건에서 해당 속성을 사용합니다.
