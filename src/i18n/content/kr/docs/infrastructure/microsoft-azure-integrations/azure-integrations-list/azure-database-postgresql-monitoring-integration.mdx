[New Relic의 통합](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 에는 [Microsoft Azure Database for PostgreSQL](https://docs.microsoft.com/en-gb/azure/postgresql/) 메트릭 및 기타 데이터를 New Relic에 보고하기 위한 통합이 포함됩니다. 이 문서에서는 통합을 활성화하는 방법과 보고된 데이터를 설명합니다.

## 특징

New Relic은 완전히 관리되는 엔터프라이즈 지원 PostgreSQL 커뮤니티 데이터베이스를 서비스로 제공하는 Azure Database for PostgreSQL 서비스에서 데이터베이스 데이터를 수집합니다. 이 서비스는 고가용성, 탄력적 확장, 자동 백업, 저장 및 이동 데이터 보호를 제공합니다.

New Relic을 사용하여 다음을 수행할 수 있습니다.

* 미리 빌드된 New Relic 대시보드에서 Azure Database for PostgreSQL 데이터를 봅니다.
* [맞춤 쿼리를 실행하고 데이터를 시각화합니다](/docs/using-new-relic/data/understand-data/query-new-relic-data) .
* 데이터 변경 사항을 알리는 경고 조건을 만듭니다.

## 통합 활성화 [#activate]

[New Relic에서 Azure 서비스를 활성화](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 하려면 표준 절차를 따르세요.

## 구성 및 폴링 [#polling]

New Relic은 통합에 따라 달라지는 기본[폴링](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) 간격에 따라 Azure Database 서비스를 쿼리합니다. Azure Database for PostgreSQL 통합의 경우:

* 폴링 간격: 5분(최대 권장 폴링 빈도: 1시간)
* 해상도: 분당 1 데이터 포인트

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

## 데이터 보기 및 사용 [#find-and-use]

[통합 데이터를 탐색](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 하려면: <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure > (select an integration)**</DNT> 으)로 이동하세요.

공급자 값이 `AzurePostgreSqlServer` 인 `AzurePostgreSqlServerSample` 이벤트 유형을 사용 [하여 데이터를 쿼리하고 탐색할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.
