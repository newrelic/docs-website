New Relic Azure Monitor 통합을 통해 Microsoft Azure 서비스에 대한 데이터를 모니터링하고 New Relic에 보고합니다.

Azure Monitor 통합은 과거 Azure 솔루션을 개선합니다.

* 적용 범위: Azure가 새 Azure 서비스의 데이터를 포함하여 Azure Monitor API 엔드포인트에 추가하는 순간 새 메트릭을 사용할 수 있습니다.
* 항상 최신 상태 유지: 기존 메트릭의 개선 사항은 New Relic에 자동으로 표시됩니다.
* 더 빠른 메트릭: 새로운 통합을 통해 고객은 최대 1개의 폴링 간격을 정의하여 대부분의 Azure Monitor 네임스페이스에 대한 평균 수집 시간을 줄일 수 있습니다.

## 특징

Azure Monitor 통합을 통해 지원되는 모든 Azure 서비스에서 모든 Azure 메트릭을 모니터링하세요.

New Relic Azure Monitor 통합을 활성화하면 New Relic이 구성된 폴링 간격에 따라 Azure 플랫폼 서비스를 쿼리합니다. 이렇게 하면 다음 기능이 모두 활성화됩니다.

* [Azure Monitor에서 지원하는 모든 Azure 서비스](https://learn.microsoft.com/en-us/azure/azure-monitor/essentials/metrics-supported) 의 메트릭 수집
* 쿼리, 경고 및 데이터 탐색기에 대한 기본 [차원 메트릭](/docs/data-apis/understand-data/new-relic-data-types/#metrics-new-relic) 환경입니다.
* 골든 메트릭 및 기본 제공 대시보드가 있는 [New Relic 탐색기](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/) 의 Azure [엔터티](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/) .
* Azure 리소스에 정의된 사용자 지정 태그로 장식된 메트릭
* 더 빠른 [폴링 간격](/docs/infrastructure/microsoft-azure-integrations/getting-started/polling-intervals-azure-integrations/) 사용 가능(최대 `1min` ), 지표에 대한 더 빠른 시간 확보 가능
* 그룹, 유형 및 태그별로 모니터링 리소스를 필터링하는 기능

## 기존 Azure 통합보다 Azure Monitor가 필요한 이유는 무엇입니까?

Azure Monitor 통합 전에 Azure를 모니터링하려면 메트릭 및 메타데이터를 검색하기 위해 서비스별 Azure API가 필요했습니다.

반대로 새로운 **Azure Monitor** 통합은 [사용 가능한 모든 Azure Monitor 메트릭](https://learn.microsoft.com/en-us/azure/azure-monitor/essentials/metrics-supported) 을 검색하여 Azure 지원을 강화하고 새로운 Azure 서비스 및 메트릭의 가용성을 가속화합니다.

## 요구 사항

Azure 통합 활성화 프로세스를 수행하려면 다음이 필요합니다.

* New Relic 계정. 하나도 없어? 무료 가입! 신용 카드가 필요하지 않습니다.
* Azure에서 New Relic 애플리케이션 및 키를 만듭니다.
* 모니터링하려는 Azure 서비스에 대한 액세스 권한을 이 애플리케이션에 부여합니다.
* 통합 UI에 필요한 정보를 배치합니다.

## Azure Monitor 통합 설정

1. [Azure 통합 활성화](/docs/infrastructure/microsoft-azure-integrations/get-started/activate-azure-integrations/) 설명서를 참조하세요. 그러면 Azure 통합 탭에서 설정하기 위해 **Azure Monitor** 통합이 표시됩니다.
2. New Relic이 앱에 대한 데이터 수집을 시작할 때까지 몇 분 정도 기다리십시오.

통합을 활성화하면 다음을 권장합니다.

* 먼저 테스트 환경에서 새로운 통합을 검증하고 테스트하는 것부터 시작하십시오.

## 통합 설정

폴링 빈도:

* `Metrics polling interval`: New Relic이 Azure Monitor를 통해 리소스에서 메트릭을 가져오는 빈도
* `Metadata and tags polling interval`: New Relic이 Azure Monitor를 통해 Azure 리소스에서 메타데이터 및 태그를 가져오는 빈도

데이터 수집 및 필터:

* `Limit to resource types`: 지정된 리소스 유형과 일치하는 리소스에서만 데이터를 가져옵니다(활성화되고 비어 있지 않은 경우).
* `Limit to resource group`: 지정된 리소스 그룹과 일치하는 리소스에서만 데이터를 가져옵니다(활성화되고 비어 있지 않은 경우).
* `Resource tags to include`: 지정된 태그로 태그가 지정된 리소스에서만 데이터를 가져옵니다(활성화되고 비어 있지 않은 경우).
* `Resouce tags to exclude`: 지정된 태그로 태그가 지정된 리소스에서 데이터 삭제(우선 순위가 가장 높음)

    Azure Monitor를 사용하여 수집된 메트릭:

    * `summary` 유형의 [차원 메트릭](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) 으로 New Relic에 저장됩니다.

    * 엔터티에 매핑되지 않은 경우 원시 차원 메트릭으로 사용할 수 있습니다.

    Azure Monitor 통합으로 쉽게 마이그레이션하기 위해 기존 Azure 통합에서 쿼리 및 메트릭 이름 지정을 계속 사용할 수 있습니다.

    모든 Azure Monitor 메트릭에는 다음 특성이 있습니다.

    * `collector.name`: `azure-monitor`

    * `instrumentation.provider`: `azure`

    * `azure.region`

    * `azure.resourceId`

    * `azure.resourceType`

    * `azure.resourceId`

      예를 들어 지난 1시간 동안 Azure Monitor에서 받은 메트릭 업데이트의 양을 보려면 다음 쿼리를 실행할 수 있습니다.

    ```
    FROM Metric SELECT count(*) WHERE collector.name = 'azure-monitor' TIMESERIES SINCE 1 hour ago
    ```

    사용자 지정 메트릭은 현재 Azure Monitor 통합에서 지원되지 않습니다.

    ```

    SELECT max(`azure.storage.storageaccounts.SuccessServerLatency`)
    WHERE collector.name = 'azure-monitor'
    FACET azure.region, azure.resourceId
    ```

    그런 다음 경고가 데이터를 올바르게 처리하는지 확인하려면 고급 신호 설정을 구성하십시오. Azure Monitor가 특정 지연으로 서비스에서 메트릭을 수신하기 때문에 이러한 설정이 필요합니다.

    * **집계 기간** : 집계 기간이 1분인 경우 평가 오프셋을 8분으로 설정하는 것이 좋습니다.

## Azure Monitor 통합으로 마이그레이션

개별 Azure 통합에서 Azure Monitor 통합으로 마이그레이션할 때 유의해야 할 몇 가지 사항이 있습니다.

* Azure Monitor 통합을 사용하도록 설정하면 모든 리소스에 대해 별도의 새 엔터티가 생성됩니다. Azure Polling 통합에서 생성된 엔터티는 그대로 유지됩니다. 즉, 대시보드, 경고 및 해당 엔터티를 참조하는 기타 기능을 업데이트해야 합니다.
* 이전 엔터티는 24시간 동안 사용할 수 있습니다.
