[인프라 모니터링](/docs/infrastructure) 은 Azure Functions 서비스에서 New Relic으로 데이터를 보고하는 [Microsoft Azure Functions](https://azure.microsoft.com/en-us/services/functions/) 에 대한 통합을 제공합니다. 이 문서에서는 이 통합을 활성화하는 방법과 캡처할 수 있는 데이터에 대해 설명합니다.

## 특징

Azure Functions 통합은 함수 실행 수, 보내고 받은 바이트 수, HTTP 오류 수와 같은 Functions 서비스에 대한 메트릭 데이터를 보고합니다. 또한 서비스의 상태 및 구성에 대한 데이터를 수집합니다.

[인프라 모니터링](/docs/infrastructure) 을 통해 Azure Functions 데이터를 모니터링하고 경고할 수 있으며 [사용자 지정 쿼리 및 차트 대시보드를 만들](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

## 통합 활성화 [#requirements]

이 통합을 활성화하려면 [인프라 모니터링을 위해 Azure 서비스를 활성화](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 하는 표준 절차를 따르세요.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

Azure Functions 통합에 대한 기본 [폴링](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) 정보:

* 폴링 간격: 5분
* 해상도: 분당 1 데이터 포인트

## 데이터 찾기 및 사용 [#find-and-use]

통합 데이터를 찾으려면 **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 으로 이동하십시오. 상단 탐색에서 **인프라** 를 클릭한 다음 **Azure** 를 클릭하고 통합을 찾습니다.

`AzureFunctionsAppSample` [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 을 사용 [하여 데이터를 쿼리하고 탐색할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

통합 데이터를 찾고 사용하는 방법에 대한 자세한 내용은 데이터 [이해 및 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 을 참조하십시오.

## 기타 시스템 데이터 [#other-attributes]

Azure Functions 통합은 서비스 및 해당 구성에 대한 다음 [특성](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) 도 수집합니다.

* `availabilityState`
* `defaultHostName`
* `enabled`
* `isPremiumApp`
* `javaVersion`
* `netFrameworkVersion`
* `nodeVersion`
* `operatingSystem`
* `phpVersion`
* `pythonVersion`
* `platformArchitecture`
* `state`
* `usageState`
