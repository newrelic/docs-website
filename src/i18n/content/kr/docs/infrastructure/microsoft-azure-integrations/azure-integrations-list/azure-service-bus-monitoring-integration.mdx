[New Relic 인프라 모니터링](/docs/infrastructure) 은 Azure Service Bus 서비스에서 New Relic으로 데이터를 보고하는 [Microsoft Azure Service Bus](https://docs.microsoft.com/en-us/azure/service-bus-messaging/service-bus-messaging-overview) 에 대한 통합을 제공합니다. 이 문서에서는 이 통합을 활성화하는 방법과 캡처할 수 있는 데이터에 대해 설명합니다.

## 특징

Azure Service Bus에 대한 New Relic의 통합은 대기열의 메시지 수, 배달 못한 편지 메시지 수 및 대기열의 바이트 크기와 같은 Service Bus 서비스에 대한 메트릭 데이터를 보고합니다. 또한 서비스의 상태 및 구성에 대한 데이터를 수집합니다.

[New Relic](/docs/infrastructure) 에서 Azure Service Bus 데이터를 모니터링하고 경고할 수 있으며 [사용자 지정 쿼리 및 차트 대시보드를 만들](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

## 통합 활성화 [#requirements]

이 통합을 [활성화하려면 표준 절차에 따라 New Relic에서 Azure 서비스](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 를 활성화합니다.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

Service Bus 통합에 대한 기본 [폴링](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) 정보:

* 폴링 간격: 5분
* 해상도: 분당 1 데이터 포인트

## 데이터 찾기 및 사용 [#find-and-use]

통합 데이터를 찾으려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**</DNT> 으로 이동하여 통합을 찾으세요.

다음 [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 을 사용 [하여 데이터를 쿼리하고 탐색할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

* 네임스페이스 데이터: [`AzureServiceBusNamespaceSample`](#namespace)
* 대기열 데이터: [`AzureServiceBusQueueSample`](#queue)
* 구독 데이터: [`AzureServiceBusSubscriptionSample`](#subscription)
* 주제 데이터: [`AzureServiceBusTopicSample`](#topic)

통합 데이터를 찾고 사용하는 방법에 대한 자세한 내용은 데이터 [이해 및 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 을 참조하십시오.

## 측정항목 데이터 [#metrics]
