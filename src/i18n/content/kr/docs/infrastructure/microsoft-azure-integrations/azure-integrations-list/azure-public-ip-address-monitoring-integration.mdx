New Relic에는 [Microsoft Azure 공용 IP 주소](https://learn.microsoft.com/en-us/azure/virtual-network/ip-services/public-ip-addresses) 메트릭 및 기타 데이터를 New Relic에 보고하기 위한 통합 이 [포함되어 있습니다.](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations) 이 문서는 통합을 활성화하는 방법과 보고된 데이터를 설명합니다.

## 특징

New Relic은 Azure 공용 IP 주소 서비스에 대해 Azure Monitor에서 메트릭 데이터를 수집합니다. Azure 공용 IP 주소는 인터넷을 통해 Azure에서 실행되는 리소스와 통신할 수 있도록 하는 데 사용됩니다. 일반적으로 가상 머신, 로드 밸런서, VPN 게이트웨이와 같은 리소스에 할당되며 외부 클라이언트가 이러한 리소스에 액세스할 수 있는 방법을 제공합니다.

New Relic을 사용하여 다음을 수행할 수 있습니다.

* 미리 빌드된 대시보드에서 Azure 공용 IP 주소 데이터를 봅니다.
* [사용자 지정 쿼리를 실행하고 데이터를 시각화합니다](/docs/infrastructure/integrations-getting-started/get-started/use-integration-data-new-relic-insights).
* 데이터 변경 사항을 알리는 경고 조건을 만듭니다.

## 통합 활성화 [#activate]

표준 [Azure Monitor 통합 절차](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor) 에 따라 New Relic에서 Azure 서비스를 활성화하세요.

## 구성 및 폴링 [#polling]

[구성 옵션을](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#settings)사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

New Relic은 기본 [폴링](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#settings) 간격에 따라 Azure Monitor 통합을 통해 공용 IP 주소 서비스를 쿼리합니다.

## 데이터 찾기 및 사용 [#find-and-use]

[통합 데이터를 탐색](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 하려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure > (통합 선택)** 으로 이동하십시오.
