[New Relic의 통합](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 에는 [Microsoft Azure SQL Database](https://azure.microsoft.com/en-us/services/sql-database/) 메트릭 및 기타 데이터를 New Relic에 보고하기 위한 통합이 포함됩니다. 이 문서에서는 통합을 활성화하는 방법과 보고된 데이터를 설명합니다.

## 특징

New Relic은 Azure의 완전 관리형 관계형 클라우드 데이터베이스 서비스에서 데이터베이스 데이터를 수집합니다. Azure SQL은 자체 리소스 집합이 있는 단일 데이터베이스와 리소스 집합을 공유하는 탄력적 풀을 제공합니다. 둘 다 Azure SQL Database 논리 서버와 연결됩니다. 데이터베이스가 방화벽으로 보호되는 경우 이전 시점으로 복제 및 복원할 수 있습니다.

New Relic을 사용하여 다음을 수행할 수 있습니다.

* 미리 빌드된 대시보드에서 Azure SQL 데이터베이스 데이터를 봅니다.
* [맞춤 쿼리를 실행하고 데이터를 시각화합니다](/docs/using-new-relic/data/understand-data/query-new-relic-data) .
* 데이터 변경 사항을 알리는 경고 조건을 만듭니다.

## 통합 활성화 [#activate]

통합을 [활성화하려면 표준 절차에 따라 New Relic에서 Azure 서비스](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 를 활성화합니다.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

New Relic은 통합에 따라 달라지는 기본[폴링](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) 간격에 따라 Azure Database 서비스를 쿼리합니다. Azure SQL Database 통합에 대한 폴링 빈도:

* 폴링 간격: 5분
* 해상도: 분당 1 데이터 포인트
