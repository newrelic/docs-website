[New Relic에는 Azure Firewall 데이터를 당사 플랫폼에 보고하기 위한 통합](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 이 포함되어 있습니다. 여기에서는 통합을 활성화하는 방법과 통합에서 수집하는 데이터에 대해 설명합니다.

## 통합 활성화 [#activate]

통합을 [활성화하려면 표준 절차에 따라 Azure 서비스](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 를 활성화합니다.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

Azure Firewalls 통합에 대한 기본 [폴링](/docs/infrastructure/microsoft-azure-integrations/getting-started/azure-integration-polling) 정보:

* New Relic 폴링 간격: 5분

## 데이터 찾기 및 사용 [#find-data]

New Relic에서 통합 데이터를 찾으려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure** 로 이동하고 통합을 선택합니다.

데이터는 다음 [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 에 첨부됩니다.
