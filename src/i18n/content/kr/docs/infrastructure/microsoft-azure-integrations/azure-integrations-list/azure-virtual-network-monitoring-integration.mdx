[New Relic 인프라 모니터링](/docs/infrastructure) 은 Virtual Network 서비스의 데이터를 New Relic에 보고하는 [Microsoft Azure의 Virtual Network](https://docs.microsoft.com/en-us/azure/virtual-network/virtual-networks-overview) 에 대한 통합을 제공합니다. 이 문서에서는 이 통합을 활성화하는 방법과 캡처할 수 있는 데이터에 대해 설명합니다.

## 특징

Azure Virtual Network에 대한 New Relic의 통합은 초당 삭제된 패킷 또는 초당 전달된 바이트와 같은 가상 네트워크(VNet)에 대한 메트릭 데이터를 보고합니다. 또한 계정의 상태 및 구성에 대한 데이터를 수집합니다.

[New Relic](/docs/infrastructure) 에서 가상 네트워크 데이터를 모니터링하고 경고할 수 있으며 [사용자 지정 쿼리 및 차트 대시보드를 만들](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

## 통합 활성화 [#requirements]

이 통합을 [활성화하려면 표준 절차에 따라 New Relic에서 Azure 서비스](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 를 활성화합니다.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

가상 네트워크 통합에 대한 기본 [폴링](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) 정보:

* 폴링 간격: 메트릭의 경우 1분, 인벤토리 5분
* 해상도: 분당 1 데이터 포인트

## 데이터 찾기 및 사용 [#find-and-use]

통합 데이터를 찾으려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure** 로 이동하여 통합을 찾으십시오.

`AzureVirtualNetworksPublicIpAddressSample` [이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 을 사용 [하여 데이터를 쿼리하고 탐색할](/docs/using-new-relic/data/understand-data/query-new-relic-data) 수 있습니다.

통합 데이터를 찾고 사용하는 방법에 대한 자세한 내용은 데이터 [이해 및 사용](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 을 참조하십시오.
