[New Relic 인프라 모니터링](/docs/infrastructure) 은 Azure Apps Service에서 New Relic 플랫폼으로 데이터를 보고하는 [Microsoft Azure App Service](https://azure.microsoft.com/en-us/services/app-service/) 에 대한 통합을 제공합니다. 이 문서에서는 이 통합을 활성화하는 방법과 캡처할 수 있는 데이터에 대해 설명합니다.

## 특징

Azure App Service는 완전히 관리되는 플랫폼에서 웹 애플리케이션, REST API 및 모바일 백엔드를 호스팅하고 실행하기 위한 서비스입니다. New Relic의 Azure App Service 통합은 다음을 포함하여 웹 앱에 대한 메트릭 데이터를 보고합니다.

* 수신된 요청 수
* HTTP 응답 및 오류 수

또한 서비스의 상태 및 구성에 대한 데이터를 수집합니다. New Relic을 사용하여 다음을 수행할 수 있습니다.

* 미리 빌드된 인프라 대시보드에서 Azure Storage 데이터를 봅니다.
* 사용자 지정 쿼리를 실행 [하고 New Relic UI](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights) 에서 데이터를 시각화합니다.
* 스토리지 데이터의 변경 사항을 알리는 경고 조건을 만듭니다.

## 통합 활성화 [#activate]

통합을 [활성화하려면 표준 절차에 따라 New Relic에서 Azure 서비스](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 를 활성화합니다.

## 구성 및 폴링 [#polling]

[구성 옵션](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 을 사용하여 폴링 빈도를 변경하고 데이터를 필터링할 수 있습니다.

New Relic은 통합에 따라 달라지는 기본[폴링](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) 간격에 따라 Azure App Services를 쿼리합니다. Azure App Services 통합에 대한 폴링 빈도:

* 폴링 간격: 5분
* 해상도: 분당 1 데이터 포인트

## 데이터 찾기 및 사용 [#find-and-use]

[통합 데이터를 탐색](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 하려면 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Azure**</DNT> 로 이동하세요.

[이벤트 유형](/docs/data-apis/understand-data/new-relic-data-types/#event-data) 은 다음과 같습니다.

* `AzureAppServiceWebAppSample`: 계정의 각 Azure App Service Web Apps를 나타냅니다.
* `AzureAppServiceHostNameSample`: 웹 앱에 액세스하는 데 사용할 수 있는 웹 앱에 바인딩된 각 도메인 이름을 나타냅니다.
* `AzureAppServicePlan:` 웹앱을 실행할 컴퓨팅 리소스 집합을 나타냅니다.

## 측정항목 데이터 [#metrics]

Azure App Service 통합에서 보고한 측정항목을 보려면 `AzureAppServiceWebAppSample` 또는 `AzureAppServicePlanSample` 이벤트 유형을 쿼리합니다. 각 메트릭과 연결된 메타데이터를 사용하여 보고되는 데이터를 [필터링하고 패싯](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) 합니다. 자세한 메트릭 정보는 [Azure 지원 메트릭](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-excluding-functions) 설명서를 참조하세요.
