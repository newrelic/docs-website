New Relic의 [Azure 통합](/docs/infrastructure/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations) 은 통합과 관련된 폴링 간격에 따라 Azure 서비스를 쿼리합니다.

폴링 간격은 통합 서비스와 관련된 <DNT>**every**</DNT> Azure 엔터티에 적용됩니다. 예를 들어 13개의 CosmosDB 인스턴스가 있는 경우 13개의 인스턴스 각각은 5분마다 폴링됩니다.

## 투표 데이터 보기 [#account-status]

[Azure 통합을 활성화하면](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) New Relic은 Azure에서 데이터 폴링을 시작하고 UI를 통해 데이터에 액세스할 수 있도록 합니다. 다른 New Relic 기능에서 가져온 추가 데이터와 함께 Azure 데이터를 쿼리할 수 있습니다. 특정 통합 또는 계정 전체에 대한 대시보드 데이터를 볼 수도 있습니다.

Azure 통합에 대한 폴링 간격, API 호출 및 기타 데이터의 시각화:

1. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**
   </DNT>

   으)로 이동합니다.

2. 특정 통합에 대한 데이터를 보려면 통합 행에 대한

   <DNT>
     **Dashboards**
   </DNT>

   링크를 선택합니다.

## New Relic 폴링 간격 [#aws-integrations]

폴링 및 해결에 대한 자세한 내용은 [Azure 통합 목록](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/) 섹션에서 특정 통합에 대한 설명서를 참조하세요.
