[Microsoft Azure](https://azure.microsoft.com/) 통합을 사용하면 Azure 서비스에 대한 데이터를 모니터링하고 New Relic에 보고하여 전체 아키텍처를 한 곳에서 종합적으로 볼 수 있습니다.

Azure 통합은 Azure [에 대한 APM의 .NET 지원](/docs/agents/net-agent/azure-installation) 과 동일하지 않습니다.

## 요구 사항 [#permissions]

특정 통합에 대한 요구 사항은 [Azure 통합](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list) 문서를 확인하세요.

New Relic은 Azure Government에 있거나 [클래식 배포 모델](https://docs.microsoft.com/en-us/azure/azure-resource-manager/resource-manager-deployment-model) 을 통해 생성된 리소스에서 모니터링 데이터를 가져올 수 없습니다.

## 특징 [#features]

[Azure 통합을 활성화](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) 하면 New Relic은 정기적인 [폴링 간격](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) 에 따라 Azure 플랫폼 서비스를 쿼리하기 시작합니다. 통합 UI를 사용하여 다음을 수행할 수 있습니다.

* 에코시스템을 변경할 때 자동으로 확장되는 [통합 대시보드](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 에서 성능 데이터를 봅니다.
* 경고로 [경고 조건](/docs/infrastructure/new-relic-infrastructure/configuration/infrastructure-alerts-add-edit-or-view-host-alert-information#integration) 을 관리합니다.
* [데이터를 쿼리합니다](/docs/using-new-relic/data/understand-data/query-new-relic-data) .

## 비용 고려 사항 [#cost-considerations]

Microsoft Azure와 New Relic의 통합 비용을 평가할 때 Azure의 Monitor 가격을 고려하십시오. [Azure 가격 책정 문서](https://azure.microsoft.com/en-us/pricing/details/monitor/#pricing) 에서 "메트릭 쿼리" 비용 항목을 참조하세요.

* 가격은 월별 API 호출 수를 기준으로 합니다.
* 다양한 Azure 서비스에 대해 New Relic이 수행한 API 호출의 추정치는 **Infrastructure > Azure > Azure Status Dashboard** 아래의 UI에서 볼 수 있습니다.
