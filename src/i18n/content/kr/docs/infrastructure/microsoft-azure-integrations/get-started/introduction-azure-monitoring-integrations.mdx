[Microsoft Azure](https://azure.microsoft.com) 애플리케이션 및 서비스를 모니터링할 수 있는 몇 가지 방법이 있습니다. 다음은 이러한 오퍼링에 대한 개요와 이 오퍼링과 다른 오퍼링을 사용하는 이유입니다.

* Azure의 New Relic: [Azure Native New Relic Service](/docs/infrastructure/microsoft-azure-integrations/get-started/azure-native) 를 사용하면 Azure Portal에서 직접 Azure에 구축된 New Relic을 배포할 수 있습니다. 이 옵션은 아래 두 기능(및 Java 앱 모니터링)에 해당하는 기능을 제공하며 Azure Portal에서 직접 구성할 수 있습니다.
* Azure 통합:[Azure 인프라 통합](/docs/infrastructure/microsoft-azure-integrations/get-started/activate-azure-integrations) 은 수십 개의 Azure 서비스를 지원합니다.
* .NET APM: [.NET APM 에이전트](/install/dotnet) 를 설치하여 Azure 애플리케이션을 모니터링하고 자세한 원격 측정 데이터를 받을 수 있습니다.
