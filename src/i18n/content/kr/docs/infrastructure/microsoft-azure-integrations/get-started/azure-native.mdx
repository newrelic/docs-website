팀이 Azure를 통해 배포하고 소프트웨어 및 클라우드 서비스를 구현하는 경우 [Azure 네이티브 뉴렐릭 서비스를 사용하면](https://azuremarketplace.microsoft.com/en-us/marketplace/apps/newrelicinc1635200720692.newrelic_liftr_payg) Azure-뉴렐릭 통합을 관리하고, 클러스터 에이전트와 .NET 및 Java <InlinePopover type="apm"/>에이전트를 모두 관리 및 설치할 수 있습니다. Azure Portal에서 직접.

Azure Native New Relic Service를 사용하면 다음을 수행할 수 있습니다.

* Azure에 구축된 기본 통합 서비스로 New Relic을 쉽게 온보딩하고 사용할 수 있습니다.
* New Relic을 포함하여 Azure에서 사용하는 모든 리소스에 대한 단일 청구서를 받습니다.
* New Relic에 대한 Azure 구독 활동 및 리소스 로그를 자동으로 모니터링합니다.
* New Relic을 사용하여 Azure 메트릭을 자동으로 모니터링합니다.
* 단일 환경을 사용하여 가상 머신 및 앱 서비스에서 New Relic 에이전트를 설치 및 제거합니다.

또한 [500개 이상의 통합](https://newrelic.com/instant-observability) 을 사용하여 Azure와 관련되지 않은 다른 모니터링에 New Relic을 사용할 수 있습니다.

<Callout variant="important">
  이 서비스는 Azure에서 New Relic 데이터를 호스팅하도록 선택한 고객에게만 제공됩니다.
</Callout>

## New Relic에 로그인 [#login-newrelic]

New Relic 리소스를 만들고 Azure에서 데이터를 보내고 있음을 확인했으면 New Relic으로 이동할 수 있습니다.

New Relic에 로그인하려면:

1. 새 리소스에 대한 Azure

   <DNT>
     **Overview**
   </DNT>

   페이지에서

   <DNT>
     **Login to New Relic account**
   </DNT>

   클릭합니다.

2. 새로운 비밀번호를 생성해야 합니다. 이렇게 하려면

   <DNT>
     **Forgot your password**
   </DNT>

   클릭하세요.

3. 이메일 주소를 입력하세요. Azure 계정에 사용하는 것과 동일해야 합니다.

## 데이터 센터에 대한 참고 사항 [#datacenters]

Azure Native New Relic Service는 데이터를 Microsoft의 Azure 데이터 센터에 직접 저장합니다. 현재 Azure 네이티브 서비스는 Azure의 미국 동부 지역에 모든 데이터를 저장합니다.

New Relic의 로그 패턴 서비스는 Amazon Web Services에서만 운영됩니다. New Relic 로그 패턴을 사용함으로써 귀하는 New Relic이 귀하의 데이터를 Amazon Web Services로 이동하고 처리할 수 있다는 데 동의합니다. 이는 New Relic의 Azure 데이터 센터 또는 Amazon Web Services 데이터 센터에 데이터를 저장하는지 여부에 관계없이 적용됩니다.
