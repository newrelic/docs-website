팀이 Azure를 통해 소프트웨어 및 클라우드 서비스를 배포하는 경우 Azure Native New Relic Service를 사용하면 Azure Marketplace를 통해 직접 New Relic을 배포하고 관리할 수 있습니다.

* 무료 New Relic 계정을 만들거나 Azure Portal에서 직접 New Relic을 구매하세요.
* Azure Marketplace를 통해 직접 청구 관리
* 자동화된 설정 흐름을 통해 APM 및 인프라 에이전트 배포
* 대화형 대시보드를 통해 모든 리소스에 대한 메트릭 및 로그 관리

## 모니터링할 수 있는 항목

Azure Native New Relic 서비스를 통해 다음 에이전트를 배포할 수 있습니다.

APM 에이전트:

* .그물
* 자바

인프라 에이전트:

* 윈도우
* Linux

## Azure Portal에서 New Relic 리소스 만들기

1. Azure Marketplace에서 Azure Native New Relic Service를 구독합니다.
2. Azure Portal에서 New Relic 리소스를 만듭니다.
3. **New Relic 리소스 만들기를** 선택하여 새 New Relic 계정을 만들어 Azure 데이터를 모니터링하거나 **기존 New Relic 리소스** 를 연결하여 Azure 데이터를 기존 New Relic 계정에 연결합니다.
4. 표시되는 프롬프트를 완료하고 **검토 + 만들기** 를 선택합니다.
5. **리소스 구성** 탭에서 서비스가 지표 및 로그를 보고하는 방법을 구성합니다.
6. New Relic이 Azure Portal의 **모니터링되는 리소스** 탭에서 데이터를 수신하는지 확인합니다. 녹색 확인 표시가 있는 리소스는 New Relic으로 데이터를 보내고 있습니다.

## New Relic에서 데이터 보기

New Relic 리소스를 만들고 Azure에서 데이터를 보내고 있음을 확인했으면 New Relic으로 이동할 수 있습니다.

New Relic에 로그인하려면:

1. 새 자원의 **개요** 페이지 **에서 New Relic 계정에 로그인 을** 클릭하십시오.
2. 새 암호를 만들어야 합니다. 이렇게 하려면 **비밀번호 분실** 을 클릭합니다.
3. 이메일 주소를 입력하세요. Azure 계정에 사용하는 것과 동일해야 합니다.

New Relic에 들어가면 먼저 살펴봐야 할 몇 가지 팁이 있습니다.

1. [APM 데이터 보기](/docs/apm/apm-ui-pages/monitoring/apm-summary-page-view-transaction-apdex-usage-data)
2. [인프라 데이터 보기](/docs/infrastructure/infrastructure-ui-pages/infrastructure-ui-entities)
3. [로그 보기 및 분석](/docs/logs/ui-data/use-logs-ui/)
4. [알림 설정](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition)

### 데이터 센터에 대한 참고 사항

Azure Native New Relic Service는 데이터를 Microsoft의 Azure 데이터 센터에 직접 저장합니다. 현재 Azure 네이티브 서비스는 Azure의 미국 동부 지역에 모든 데이터를 저장합니다.

New Relic의 로그 패턴 서비스는 Amazon Web Services에서만 운영됩니다. New Relic 로그 패턴을 사용함으로써 귀하는 New Relic이 귀하의 데이터를 Amazon Web Services로 이동하고 처리할 수 있다는 데 동의합니다. 이는 New Relic의 Azure 데이터 센터 또는 Amazon Web Services 데이터 센터에 데이터를 저장하는지 여부에 관계없이 적용됩니다.
