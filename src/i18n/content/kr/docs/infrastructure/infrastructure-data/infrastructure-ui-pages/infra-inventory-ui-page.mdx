뉴렐릭은 시스템 모듈, 설정 파일, 메타데이터, 패키지, 서비스, 사용자 세션 등을 포함하여 호스트별 시스템 설정에 대한 자세한 정보를 수집할 수 있습니다. <DNT>**Inventory**</DNT> 페이지에서는 각 호스트의 설정에 대한 필터링 및 검색 가능한 보기를 제공합니다.

<DNT>**Inventory**</DNT> 페이지를 사용하여 다음을 수행하세요.

* 버전 업데이트가 모든 호스트에 성공적으로 적용되었는지 확인하십시오.
* 호스트 간의 버전 불일치를 감사합니다.
* 보안 취약점을 수정하기 위해 업데이트가 필요한 호스트를 빠르게 식별합니다.

## 인벤토리 페이지 찾기 [#find]

인벤토리 데이터를 보고 검색하려면: <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Inventory**</DNT> 으)로 이동하세요.

## 이 페이지에는 어떤 데이터가 표시됩니까? [#data]

이 페이지에 데이터를 표시하려면 [인프라 에이전트가](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent) 설치되어 있어야 합니다.

## 페이지 기능 [#page-functions]

호스트의 특정 항목에 대한 정보를 찾으려면 <DNT>**Inventory**</DNT> 페이지 기능을 사용하세요.

<CollapserGroup>
  <Collapser id="search" title="인벤토리 검색">
    검색 기능을 사용하여 인벤토리 항목을 검색합니다.

    예를 들어 OpenSSL과 관련된 정보를 찾으려면 `openssl` 검색하세요. 검색어는 인벤토리 항목 이름과 일치합니다.

    <Callout variant="tip">
      시스템에서 사용 중인 Fluent Bit 버전을 찾으려면 **Search Inventory** \[인벤토리 검색] 검색 창에 `fluent-bit` 입력한 후 **&gt;**를 클릭하여 검색 결과를 확장하세요.
    </Callout>
  </Collapser>

  <Collapser id="view" title="인벤토리 항목 세부 정보 보기">
    인벤토리 항목 세부정보는 뉴렐릭 [인벤토리 항목 이름](#item-naming) 에 따라 상주하는 각 호스트에 대한 호스트 및 시스템 정보를 제공합니다. 다른 호스트에 동일한 항목의 다른 버전이 있는 경우 뉴렐릭은 이를 감지하고 인벤토리 페이지에 <DNT>**variant hosts**</DNT> 라벨을 사용하여 플래그를 지정하고 각 버전을 실행하는 각 호스트를 나열합니다.

    항목 세부 정보는 해당 소스에서 지시하는 속성(키/값 쌍)입니다. 특정 속성은 일반적으로 시간이 지남에 따라 안정적이지만 새로운 속성이 추가되거나 더 이상 사용되지 않을 수 있습니다. 속성은 각 인벤토리 항목의 핵심인 중요한 메타데이터를 전달합니다.

    일반적인 인벤토리 항목 속성은 다음과 같습니다.

    * 변형 호스트(호스트 이름)
    * 아키텍처
    * 설명
    * 필수적인
    * 우선순위
    * 상태
    * 버전
  </Collapser>

  <Collapser id="alerts" title="호스트의 경고 임계값 인시던트 보기">
    한 명 이상의 호스트의 [공지 레버 값, 릴레이 인시던트를](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-threshold) 보려면 호스트의 <DNT>**Critical**</DNT> 선택하세요. <Icon name="fe-x-circle" /> 아이콘 또는 <DNT>**Warning**</DNT> <Icon name="fe-alert-triangle" /> 상.
  </Collapser>
</CollapserGroup>

## 인벤토리 데이터 보존 [#data-retention]

인벤토리 데이터는 실시간입니다. 호스트가 보고를 중지하면 인벤토리 데이터가 최대 24시간 동안 계속 표시됩니다.

## 차트 데이터 속성 [#attributes]

<DNT>**Inventory**</DNT> 페이지를 채우는 데 사용되는 속성에 대한 기술적인 설명은 [기본 인프라 속성 및 이벤트를](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#shared-attributes) 참조하세요. 여기에는 [운영 시스템별 공통 이벤트](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#events-os) 요약이 포함됩니다.

## 필터 세트 [#filter-sets]

필터 세트 UI 구성요소는 호스트를 필터링하고 저장된 보기를 생성하는 데 사용됩니다. <DNT>**It is only available on the Events and Inventory UI pages and is in the process of being deprecated.**</DNT> [저장된 보기](/docs/infrastructure/infrastructure-ui-pages/infra-hosts-ui-page#ui-features) 기능으로 대체되었습니다. 이 시점부터 생성하는 모든 필터 세트는 저장된 보기 기능으로 마이그레이션되지 않습니다.

<CollapserGroup>
  <Collapser id="filter-set" title="필터 세트 사용 방법">
    New Relic 인프라 모니터링을 사용하면 필터를 필터 세트로 결합하여 가장 중요한 기준에 따라 호스트를 구성할 수 있습니다. 귀하의 계정에는 수백 명의 호스트가 보고할 수 있으므로 문제를 효과적으로 해결할 수 있으려면 중요한 것에 집중하는 것이 중요합니다.

    ### 카테고리 내 패턴 보기 [#benefits]

    사용 가능한 [속성](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute) 또는 [태그](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) 를 사용하여 필터 세트를 생성할 수 있습니다.

    예를 들어 인프라를 다음과 같은 범주로 구성할 수 있습니다.

    * 지역
    * 운영 체제 버전
    * Docker 컨테이너와 연결된 호스트
    * 테스트 환경

    필터 세트를 계정의 다른 사람들과 공유할 수 있으며 필터 세트에 있는 각 호스트의 색상으로 구분된 [상태](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status) 를 확인하여 인프라 문제를 빠르게 식별할 수 있습니다.

    ### 필터 세트 생성 [#create]

    기본 인프라 필터 세트는 <DNT>**All hosts**</DNT> 이며 필터 세트를 생성하기 위한 템플릿 역할을 합니다.

    필터 세트를 생성하려면:

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure**</DNT> 으로 이동하여 <DNT>**Inventory**</DNT> 또는 <DNT>**Events**</DNT> 클릭합니다.
    2. 왼쪽 사이드바에 <DNT>**All hosts**</DNT> 표시되지 않으면 <DNT>**Saved filter sets &gt; All hosts**</DNT> 선택하여 해당 필터 세트를 엽니다.
    3. <DNT>**All hosts**</DNT> 에서 <DNT>**Filter hosts**</DNT> 클릭합니다.
    4. 목록에서 항목을 클릭하면 값 목록이 표시됩니다.
    5. <DNT>**Include**</DNT> 또는 <DNT>**Exclude**</DNT> 클릭합니다( [필터 설정 논리](#logic) 참조).
    6. 값을 개별적으로 클릭하거나 여러 값과 일치하도록 텍스트를 입력합니다.
    7. 원하는 필터 세트를 얻을 때까지 필터를 계속 추가합니다.
    8. 필터 이름을 지정하려면 <Icon name="fe-edit-2" /> 아이콘을 클릭하고 이름을 입력한 후 <DNT>**Save**</DNT> 클릭합니다.

    ### 필터 세트 편집 [#edit]

    기존 필터 세트를 변경하려면:

    1. 사이드바에서 <DNT>**Saved filter sets**</DNT> 클릭하여 목록을 엽니다.
    2. 스크롤하거나 검색어를 입력하여 필터 세트를 찾습니다.
    3. 필터 세트를 클릭하여 엽니다.
    4. 사이드바에서 옵션을 클릭하여 필터 세트를 업데이트한 다음 저장합니다.

    ### 필터 세트 로직 [#logic]

    필터 세트를 생성할 때 결과를 좁히는 속성 및/또는 태그 목록을 생성합니다. 이 섹션에서는 필터 세트가 목록에 다양한 규칙을 적용하는 방법을 설명합니다.

    #### 그리고/또는

    필터 세트는 백그라운드에서 논리 연산자 `AND` 및 `OR` 를 사용하여 데이터를 결합합니다.

    * 여러 속성 또는 태그의 값을 클릭하면 `AND` 으로 결합됩니다.
    * 속성 또는 태그 내에서 값을 클릭하면 `OR` 으로 결합됩니다.

    필터 결과는 다음 두 가지 모두에 해당하는 호스트를 표시합니다.

    * 선택한 인프라 에이전트 버전 중 하나를 포함하는 호스트
    * 선택한 AWS 가용 영역 중 하나의 호스트
