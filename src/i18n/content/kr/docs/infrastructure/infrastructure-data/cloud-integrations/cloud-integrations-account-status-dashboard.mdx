뉴클레릭 클러스터 모니터링(예: [AWS](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) 또는 [Azure](/docs/integrations/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations))에 연결하는 각 클라우드 계정에는 우리 플랫폼이 해당 클라우드 서비스 공급업체의 데이터를 처리하는 방법을 보여주는 <DNT>**account status dashboard**</DNT> 가 있습니다.

## 중요한 이유 [#reasons-to-use]

이 대시보드를 모니터링하는 이유는 다음과 같습니다.

* <DNT>**Understand API call usage.**</DNT> 우리는 클라우드 API 사용하여 지표 및 지표 데이터를 수집합니다. 대시보드에는 해당 API 호출의 개수와 빈도가 표시됩니다. 두 가지 이유로 이를 모니터링해야 할 수 있습니다. API 호출은 비용이 들 수 있고 API 사용은 특정 사용 수준에서 제한될 수 있습니다. 사용량을 줄이려면 [통합 설정을 구성하여](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) 모니터하는 구성 요소 수나 모니터하는 빈도를 줄일 수 있습니다.

* <DNT>**Troubleshoot missing data or other data issues.**</DNT> 대시보드에는 다음을 포함하여 통합이 데이터를 보고하는 방식에 영향을 미치는 정보가 표시됩니다.

  * 권한 오류 또는 할당량 소진 오류와 같이 New Relic 수집 데이터에 영향을 주는 오류입니다.
  * 폴링 간격 변경 또는 태그 수집이 비활성화 또는 활성화되는 것과 같은 통합 구성 변경.

## 대시보드 데이터 이해 [#understand-data]

계정 상태 대시보드에 표시되는 특정 데이터는 클라우드 서비스 제공업체에 따라 다릅니다. 일반적인 차트에는 다음이 포함됩니다.

* <DNT>**Data updates**</DNT>: 지표 데이터 또는 지표 데이터에 대한 업데이트를 표시합니다( `1` 값으로 표시되는 업데이트).
* <DNT>**Account changes**</DNT>: 통합 작동 방식에 영향을 미치는 작업입니다. 예: 클라우드 계정 이름 변경 또는 연결 해제, 폴링 처리 변경 및 기타 설정 옵션.
* <DNT>**Data freshness**</DNT>: 각 통합에 대해 수집된 마지막 데이터 포인트의 타임스탬프입니다.
* <DNT>**Fetching errors**</DNT>: 데이터 수집에 문제가 있음을 나타냅니다. (클라우드 제공자 측의 문제일 수도 있습니다.)

차트 데이터를 더 잘 이해하기 위한 옵션은 다음과 같습니다.

* 차트 위에 마우스를 올려 놓으세요 <Icon name="fe-help-circle" /> 차트 설명을 보려면 아이콘을 클릭하세요(가능한 경우).
* [차트의 기본 NRQL 쿼리를 봅니다](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform).

## 계정 상태 대시보드 찾기 [#find-dashboard]

클라우드 서비스 공급자의 계정 상태 대시보드를 찾으려면:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure**</DNT> 에서 클라우드 서비스 공급업체를 선택합니다(예: <DNT>**AWS**</DNT>).
2. 클라우드 계정으로 <DNT>**Account status dashboard**</DNT> 선택합니다.
