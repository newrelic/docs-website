Pixie [엔터티를](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/) 사용하여 골든 메트릭과 특정 프로토콜을 사용하여 특정 서비스로 전송된 개별 전신 요청 샘플을 확인합니다. Pixie 엔터티는 Pixie에서 자동으로 추적한 데이터로 구동됩니다. 자세히 알아보려면 아래에서 프로토콜을 선택하세요.

* [DNS](#dns)
* [MySQL](#mysql)

* [AMQP](#amqp)

## DNS [#dns]

**DNS - Pixie** 엔터티 목록에서 서비스를 선택하여 해당 서비스로 전송된 모든 DNS 요청에 대한 응답 시간, 오류율 및 처리량의 시계열 그래프를 확인합니다. 샘플링된 DNS 요청 테이블을 검사하여 이 서비스에서 수신한 전신 DNS 요청 및 응답을 확인합니다.

Pixie에서 자동으로 추적하는 DNS 요청을 사용하여 다음을 수행할 수 있습니다.

* 어떤 서비스가 DNS 요청을 하고 있는지 신속하게 확인합니다.
* 높은 수준의 DNS 응답 시간 및 처리량 정보를 얻습니다.
* DNS 서버 간의 처리량 불균형을 관찰하십시오.

## 카프카 [#kafka]

## AMQP [#amqp]

**AMQP - Pixie** 엔터티 목록에서 서비스를 선택하여 해당 서비스로 전송된 모든 AMQP 메시지에 대한 응답 시간, 오류율 및 처리량의 시계열 그래프를 확인합니다.

Pixie에서 추적하는 AMQP 메시지를 사용하여 다음을 수행할 수 있습니다.

* 서비스 및 포드당 응답 시간, 오류율 및 처리량을 확인하세요.
* AMQP 소비자 및 생산자 목록을 참조하십시오.
* AMQP 방법별 응답 시간을 참조하십시오.
* 추적된 AMQP 메시지의 본문을 검사합니다.
