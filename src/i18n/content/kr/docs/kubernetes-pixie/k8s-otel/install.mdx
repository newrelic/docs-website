OpenTelemetry 사용하여 쿠버네티스 클러스터를 모니터링할 수 있습니다. OpenTelemetry는 텔레메트리 데이터를 수집하여 뉴렐릭으로 보내는 표준화된 방법을 제공합니다. 이 통합을 통해 지표, 이벤트 및 로그를 포함하여 인사이트를 Kubernetes 환경에 추가할 수 있습니다.

뉴렐릭은 [Kubernetes용 OpenTelemetry (NRDOT)의 뉴렐릭 배포판](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s) 사용을 권장합니다. 이 배포판은 뉴렐릭과 원활하게 작동하도록 사전 구성되어 있으며 쿠버네티스 클러스터를 효과적으로 모니터링하는 데 필요한 모든 구성 요소와 설정이 포함되어 있습니다. 하지만 자체 OpenTelemetry Collector를 사용하려면 Kubernetes 모니터링에 필요한 구성 요소가 포함되어 있는지 확인해야 합니다.

## 요구 사항 [#requirements]

Kubernetes 텔리메트리 데이터를 뉴렐릭으로 보내려면 OpenTelemetry Collector 가 필요합니다. 저희 [NRDOT는](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s) 귀하의 쿠버네티스 클러스터를 자동으로 모니터링하도록 구성되어 있습니다. 이는 [`nr-k8s-otel-collector`](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector) Helm 차트를 통해 필요한 모든 구성 요소를 구현하고 배포합니다.

다른 OpenTelemetry Collector를 선택하는 경우, 포괄적인 Kubernetes 모니터링에 필요한 모든 핵심 구성 요소가 포함되어 있는지 확인하십시오.

### 지원되는 환경 [#supported-envs]

Kubernetes 위한 OpenTelemetry 모니터링은 다양한 구현, 배포 환경 전반에 걸쳐 강력한 지원을 제공합니다. 지원되는 플랫폼은 다음과 같습니다.

* **클라우드 공급업체:**

  * 아마존 EKS
  * 마이크로소프트 AKS
  * 구글 GKE
  * 오라클 OKE
  * 레드햇 오픈시프트

* **온프레미스 클러스터:** 온프레미스 쿠버네티스 클러스터를 지원합니다.

* **Kubernetes 버전:** 지원은 각 벤더에서 현재 지원하는 Kubernetes 버전에 맞춰져 있어 이러한 환경 전반에 걸쳐 호환성과 효과적인 모니터링 솔루션을 보장합니다.

<Callout variant="important">
  Windows 노드는 지원되지 않습니다.
</Callout>

## OpenTelemetry로 쿠버네티스 클러스터를 설치하세요 [#install]

Kubernetes용 OpenTelemetry는 다음 방법 중 하나를 사용하여 설치할 수 있습니다.

<Callout variant="tip">
  고급 설정 옵션에 대해서는 [OpenTelemetry Kubernetes의 고급 설정을](/docs/kubernetes-pixie/k8s-otel/advance-config) 참조하십시오.
</Callout>

<CollapserGroup>
  <Collapser id="helm-install" title="Helm 설치">
    ## Helm 설치 [#helm-install]

    Helm 설치 방법은 Kubernetes 에서 구현하다, 배포하다 OpenTelemetry 에 권장되는 접근 방식입니다. 이 방법은 구현, 배포 프로세스를 단순화하고 쉬운 설정 관리를 허용합니다. NRDOT 또는 자체 OpenTelemetry Collector를 사용할 수 있습니다.

    뉴렐릭은 뉴렐릭과 원활하게 연동되도록 사전 구성된 NRDOT for Kubernetes 사용을 권장합니다. 이 배포판에는 쿠버네티스 클러스터를 효과적으로 모니터링하는 데 필요한 모든 구성 요소와 설정이 포함되어 있습니다. 하지만 자체 OpenTelemetry Collector를 사용하려면 Kubernetes 모니터링에 필요한 구성 요소가 포함되어 있는지 확인해야 합니다.

    **Helm 차트를 설치하려면 다음 단계를 따르세요.**

    1. [Helm 차트 `values.yaml` 파일을](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector/values.yaml#L20-L24) 다운로드하고 특정 요구 사항에 맞게 수정하십시오.

       * 클러스터 이름과 <InlinePopover type="licenseKey" />은 필수입니다.

       * 자세한 내용은 [설정 시위를](#config-params) 참조하세요.

    2. `values.yaml` 파일을 사용하여 [Helm 차트를](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector) 설치하세요.

       ```shell
       helm repo add newrelic https://helm-charts.newrelic.com
       helm upgrade nr-k8s-otel-collector newrelic/nr-k8s-otel-collector -f your-custom-values.yaml -n newrelic --create-namespace --install
       ```

    3. 파드가 성공적으로 시작되었는지 확인하십시오.

       ```shell
           kubectl get pods -n newrelic --watch
       ```

       `newrelic` 네임스페이스에서 `nr-k8s-otel-collector-<hash>` 과 같은 이름을 가진 파드를 볼 수 있을 것입니다.

    4. 다음 쿼리를 실행하여 뉴렐릭이 메트릭, 이벤트, 로그 등 필요한 데이터를 수신하는지 확인하세요. 자세한 내용은 [쿼리 빌더 소개를](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/) 참조하십시오.

       ```sql
       FROM Metric SELECT * WHERE k8s.cluster.name='<CLUSTER_NAME>'
       FROM OtlpInfrastructureEvent SELECT * WHERE k8s.cluster.name='<CLUSTER_NAME>'
       FROM Log SELECT * WHERE k8s.cluster.name='<CLUSTER_NAME>'
       ```

    <Callout variant="tip">
      Kubernetes 텔레메트리 데이터를 수집하기 위해 다른 OpenTelemetry Collector 사용하려면 `values.yaml` 파일의 `image` 설정을 업데이트하여 원하는 OpenTelemetry Collector 지정하십시오.

      ```yaml
      image:
          repository: otel/opentelemetry-collector-contrib # Example: Using the contrib distro
          tag: "latest" # Or a specific stable version like "0.98.0"
      ```
    </Callout>
  </Collapser>

  <Collapser id="manifest-install" title="매니페스트 설치">
    Manifest 설치 방식은 Helm을 사용하지 않고도 Kubernetes용 OpenTelemetry를 설정할 수 있는 방법을 제공합니다. 이 방법은 직접적인 설정을 선호하거나 수동 설정이 필요한 특정 요구 사항이 있는 사용자에게 적합합니다.

    **매니페스트 설치를 수행하려면 다음 단계를 따르세요.**

    1. `nr-k8s-otel-collector` 의 [렌더링된 예제 디렉터리](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector/examples/k8s/rendered) 내용을 로컬 작업 공간으로 복사합니다. 이 디렉터리에는 Kubernetes 매니페스트 파일이 포함되어 있습니다.

    2. 로컬 `rendered` 디렉터리에 있는 [`secret.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/secret.yaml) 파일을 업데이트하세요. `<Your Base64 encoded License key>` Base64로 인코딩된 뉴렐릭 볼륨 키로 바꾸세요.

       ```yaml
       data:
           licenseKey: <Your Base64 encoded License key>
       ```

       라이선스 키를 `Base64` 으로 인코딩하려면:

       * **Linux/macOS:** `echo -n "YOUR_LICENSE_KEY" | base64` 사용하세요.
       * **Windows(PowerShell):** `[System.Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes("YOUR_LICENSE_KEY"))` 사용하세요.

    3. 로컬 `rendered` 디렉터리 내의 [`daemonset-configmap.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/daemonset-configmap.yaml) 및 [`deployment-configmap.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/deployment-configmap.yaml) 파일 모두에서 클러스터 이름을 수동으로 업데이트하십시오. `k8s.cluster.name` 의 인스턴스를 찾고 `<cluster_name>` 원하는 클러스터 이름으로 바꿉니다.

       ```yaml
       - key: k8s.cluster.name
           action: upsert
           value: <cluster_name>
       ```

    4. 이러한 필수 필드를 업데이트한 후 `kubectl` 을 사용하여 `newrelic` 지우스페이스 및 구현하다, 클러스터에 매니페스트를 배포합니다.

       ```bash
       kubectl create namespace newrelic
       kubectl apply -n newrelic -R -f rendered
       ```

    <Callout variant="tip">
      구현하다, Helm 없이 배포하는 경우 렌더링된 매니페스트를 통해 구성 요소를 구현하고 배포하는 경우 접두사(예: `example-` 가 포함될 수 있습니다. 이 접두사는 일반적으로 Helm 릴리스 이름을 기반으로 동적 명명을 위해 Helm 차트 템플릿에서 사용됩니다. 보다 깔끔한 명명 규칙을 선호하는 경우, 적용하기 전에 매니페스트 파일에서 이러한 접두사를 직접 조정할 수 있습니다.
    </Callout>
  </Collapser>
</CollapserGroup>

## 제거 [#uninstall]

### Helm 제거 [#helm-uninstall]

OpenTelemetry 로 쿠버네티스 클러스터 모니터링을 중지하려면 다음 명령을 실행하세요.

```shell
helm uninstall nr-k8s-otel-collector -n newrelic
```

### 매니페스트 제거 [#manifests-uninstall]

OpenTelemetry 로 쿠버네티스 클러스터 모니터링을 중지하려면 다음 명령을 실행하세요.

```shell
kubectl delete -R -f rendered
kubectl delete namespaces newrelic
```
