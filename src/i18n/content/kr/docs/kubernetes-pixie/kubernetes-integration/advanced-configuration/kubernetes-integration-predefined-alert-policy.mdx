계정에서 처음으로 New Relic Kubernetes 통합을 배포할 때 기본 [경고 조건](/docs/alerts/new-relic-alerts/defining-conditions/define-alert-conditions) 집합을 계정에 배포합니다. **Kubernetes 기본 경고 정책** 이라는 사전 정의된 경고 정책에는 원치 않는 알림을 피하기 위해 기본적으로 [알림 채널](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-control-where-send-alerts) 이 없습니다.

알림 조건의 임계값은 환경에 맞게 사용자 지정할 수 있으며 알림을 보내도록 알림 정책을 업데이트할 수 있습니다. 자세한 내용은 [인프라 경고 설명서](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information) 를 참조하십시오.
