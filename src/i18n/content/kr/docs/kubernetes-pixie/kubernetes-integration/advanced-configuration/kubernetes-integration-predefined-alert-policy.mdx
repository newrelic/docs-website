계정에서 처음으로 New Relic Kubernetes 통합을 배포할 때 기본 [경고 조건](/docs/alerts/new-relic-alerts/defining-conditions/define-alert-conditions) 세트를 계정에 배포합니다. **Kubernetes 기본 경고 정책** 이라는 사전 정의된 경고 정책에는 원치 않는 알림을 방지하기 위한 [알림 채널이](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-control-where-send-alerts) 기본적으로 없습니다.

경고 조건에 대한 임계값을 사용자 정의할 수 있을 뿐만 아니라 알림을 보내도록 정책을 업데이트할 수도 있습니다. 자세한 내용은 [인프라 경고 설명서를](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information) 참조하세요.
