Kubernetes는 배포 자동화, 확장 및 컨테이너화된 애플리케이션 관리를 위한 오픈 소스 도구입니다. 뉴렐릭 Kubernetes 모니터링 통합은 쿠버네티스 클러스터 및 워크로드가 온프레미스 또는 클라우드에서 호스팅되는지 여부에 관계없이 무슨 일이 일어나고 있는지 빠르고 쉽게 확인할 수 있는 방법을 제공합니다.

[Kubernetes 통합을 구축하고 실행](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure/) 하면 Kubernetes에서 실행되는 모든 서비스를 한곳에서 [즉시 사용 가능한 Kubernetes 환경](/docs/integrations/kubernetes-integration/cluster-explorer/kubernetes-cluster-explorer) 으로 모니터링할 수 있습니다. 제어 평면의 상단부터 단일 패드에서 실행되는 시스템까지 모든 것을 보여 주므로, 회사가 어떻게 작동하는지 핵심적으로 파악할 수 있습니다.

<img title="New Relic - Kubernetes cluster explorer" alt="New Relic - Kubernetes cluster explorer" src="/images/kubernetes_screenshot-crop_cluster-explorer.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes**</DNT> 으로 이동하여 클러스터를 선택하세요. 클러스터 탐색기는 대규모로 실행할 때 Kubernetes에 필요한 모든 것을 충족하는 편리한 올인원 솔루션입니다.
</figcaption>

## 뉴렐릭으로 Kubernetes 예측 [#monitoring-k8s-nr]

Kubernetes를 관리하는 것은 어려울 수 있습니다. 몇 분 만에 컨테이너가 생성 및 삭제되고, 앱이 충돌하고, 리소스가 예상치 못한 방식으로 사용되는 등 항상 많은 일이 일어나고 있습니다. 우리의 통합은 온프레미스, 클라우드, 하이브리드 설정 등 Kubernetes를 다양한 방식으로 사용하는 방법을 파악하는 데 도움이 됩니다.

뉴렐릭으로 무엇을 할 수 있는지 살펴보세요:

* 클러스터에서 지표를 관리하고 수집하려면 [Kubernetes 제어 평면을](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/configure-control-plane-monitoring) 모니터링하세요.

* [Kubernetes 로그를 뉴렐릭으로 전달하여](/docs/logs/forward-logs/kubernetes-plugin-log-forwarding/) 로그 데이터를 수집, 처리, 탐색, 조회, 공지함으로써 로그 데이터를 관리할 수 있는 더 많은 방법을 제공합니다.

* 자체 차트를 작성하고 컨테이너 오케스트레이션 계층에서 통합이 수집하는 모든 [Kubernetes 데이터를](/docs/integrations/kubernetes-integration/understand-use-data/find-use-your-kubernetes-data) [쿼리하세요 .](/docs/using-new-relic/data/understand-data/query-new-relic-data) 이를 통해 노드, 지우스페이스, 배포, 복제본 세트, 파드 및 컨테이너에 더 많은 인사이트를 제공합니다.

* [APM 데이터를 Kubernetes에 연결하여](/docs/integrations/kubernetes-integration/link-your-applications/link-your-applications-kubernetes) 웹 및 모바일 앱의 성능을 측정하세요. 요청률, 처리량, 오류율, 가용성 등을 추적할 수 있습니다.

* Apache, NGINX, Cassandra 등과 같이 [Kubernetes에서 실행되는 서비스를 모니터링합니다](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes). [Kubernetes에서 Redis를 모니터링하는 방법에 대한 튜토리얼을](/docs/integrations/kubernetes-integration/link-apps-services/tutorial-monitor-redis-running-kubernetes) 확인하세요.

* [Prometheus 에이전트](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic/#Agent) 가 제공하는 서비스 검색 덕분에 클러스터의 모든 워크로드에서 Prometheus 메트릭을 스크랩합니다.

* 통합에는 [미리 정의된 공지 정책 세트가](/docs/kubernetes-pixie/kubernetes-integration/installation/predefined-alert-policy) 포함되어 있지만,[ Kubernetes ](/docs/kubernetes-pixie/kubernetes-integration/installation/create-alerts)Kubernetes 데이터를 기반으로 알림을 만들고 수정하거나 [권장되는 공지 정책 세트를 추가할](/docs/kubernetes-pixie/kubernetes-integration/installation/recommended-alert-policies) 수 있습니다.

* 모든 Kubernetes 이벤트를 확인하세요. [Kubernetes 이벤트 통합은](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration) 쿠버네티스 클러스터에서 발생하는 이벤트를 감시하고 해당 이벤트를 뉴렐릭으로 보냅니다. 그런 다음 클러스터 탐색기에서 이벤트 데이터를 시각화할 수 있습니다. [뉴렐릭 Kubernetes 통합 설치](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure/) 후 기본으로 설치됩니다.

* [팟(Pod) 자동 확장을 트리거하는](/docs/journey-demand/autoscale-your-infra) Kubernetes에 데이터를 보고하여 피크 수요 이벤트와 같이 수요가 급증할 때 항상 리소스를 사용할 수 있습니다.

  <img title="New Relic - Kubernetes Events" alt="New Relic - Kubernetes Events" src="/images/kubernetes_screenshot-full_events.gif" />

  <figcaption>
    <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes cluster explorer &gt; Events**</DNT>: 모든 Kubernetes 이벤트를 검색 및 필터링하고 애플리케이션 로그 및 인프라 데이터를 자세히 살펴보세요.
  </figcaption>

이러한 기능은 New Relic이 [계측된 호스트에서 실행되는 컨테이너화된 프로세스에](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/docker-instrumentation-infrastructure) 대해 이미 보고한 데이터에 추가됩니다.
