New Relic의 Kubernetes 클러스터 탐색기는[Kubernetes 통합](/docs/integrations/kubernetes-integration/get-started/introduction-kubernetes-integration) 으로 수집된 데이터를 사용하여 제어 평면에서 노드 및 포드에 이르기까지 클러스터 상태를 표시합니다. 각 엔터티의 상태를 확인하고, 로그를 탐색하고, 앱의 성능을 확인할 수 있습니다. [이벤트 통합](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration) 을 사용하면 클러스터에서 발생하는 모든 것이 표시되고 [로그 플러그인](/docs/logs/enable-logs/enable-logs/kubernetes-plugin-logs) 을 사용하여 가져온 로그도 사용할 수 있습니다.

## 클러스터 탐색기를 만나보세요 [#cluster-explorer-use]

클러스터 탐색기는 가장 관련성이 높은 클러스터 데이터를 배의 바퀴 모양으로 차트에 표시하며 이는 Kubernetes 로고이기도 합니다.

* 외부 링: 클러스터의 최대 24개 노드를 포함하며 경보 양에 따라 가장 관련성이 높습니다. 각 노드 위로 마우스를 가져가면 리소스 사용량과 사용된 할당 가능한 포드의 비율을 확인할 수 있습니다.

* 내부 링: 포드 포함(

  ) 각 노드. 활성 경고가 있는 포드는 세 번째 가장 안쪽 링에 표시되고 보류 중이거나 실행할 수 없는 포드는 중앙에 있습니다.

리소스 사용량에 대한 빠른 개요를 보려면 각 노드 또는 포드 위로 마우스를 가져갑니다. 각 노드와 포드를 클릭하여 시간 경과에 따른 리소스 사용량을 보거나 상태 및 활성 경고에 대한 추가 정보를 얻을 수 있습니다. [색상](/docs/alerts/new-relic-alerts/configuring-alert-policies/identify-entities-without-alert-policies) 은 [사전 정의된 경고 조건](/docs/integrations/kubernetes-integration/kubernetes-events/kubernetes-integration-predefined-alert-policy) 을 기반으로 합니다. 노란색 포드에는 활성 경고 경고가 있고 빨간색 포드에는 활성 위험 경고가 있습니다.

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Kubernetes** : 상태 및 상태에 대한 추가 정보를 얻고 애플리케이션 데이터와 추적, 로그 및 이벤트에 대해 더 자세히 알아보려면 포드를 클릭하십시오.
</figcaption>

노드를 클릭하면 다음 데이터가 표시됩니다.

* 포드 통계
* 할당 가능한 양에 대한 CPU, 메모리 및 스토리지 사용량
* 할당 가능한 포드 양에 대해 노드에서 사용하는 포드의 양

[활성화한 통합 및 기능](/docs/integrations/kubernetes-integration/get-started/introduction-kubernetes-integration) 에 따라 각 포드에 대해 다음을 볼 수 있습니다.

* 네임스페이스 및 배포를 포함한 포드 상태 및 메타데이터
* 컨테이너 상태 및 통계
* 활성 경고(경고 및 위험 모두)
* [해당 포드에서 발생한 Kubernetes 이벤트](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration)
* APM 데이터 및 추적(APM 데이터를 [연결](/docs/integrations/kubernetes-integration/link-your-applications/link-your-applications-kubernetes) 한 경우)
* [New Relic의 로그 관리를 위해 Kubernetes 플러그인을](/docs/logs/enable-logs/enable-logs/kubernetes-plugin-logs) 사용하여 수집된 포드 및 컨테이너 로그에 대한 링크

클러스터 및[제어 평면](/docs/integrations/kubernetes-integration/installation/configure-control-plane-monitoring) 통계는 항상 왼쪽에 표시됩니다.

### 클러스터 대시보드

클러스터 대시보드는 **Kubernetes 대시보드** 를 클릭하여 클러스터 탐색기에서 언제든지 액세스할 수 있습니다. Kubernetes 클러스터에 대해 선별된 대시보드 환경을 제공합니다.

<img
  title="New Relic - Cluster explorer - Dashboard detail"

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Kubernetes > Kubernetes dashboard** : Kubernetes 클러스터 탐색기에서 Kubernetes 대시보드에 액세스할 수 있습니다. 유용한 Kubernetes 메트릭 데이터를 보여줍니다.
</figcaption>

### 클러스터 탐색기 노드 테이블 [#cluster-explorer-table]

클러스터 탐색기 아래에는 클러스터, 네임스페이스 또는 배포의 모든 노드를 보여주는 노드 테이블 **이** 있습니다. 다른 모든 사용 지표와 마찬가지로 이 표는 [할당 가능한 리소스](https://kubernetes.io/docs/tasks/administer-cluster/reserve-compute-resources/#node-allocatable) 에 대한 소비를 보여줍니다.

## 클러스터 데이터 검색 및 필터링 [#search-filter]

클러스터 탐색기에서 데이터 보기를 수정하는 주요 방법은 상단 표시줄을 사용하여 특정 속성이나 값을 검색하는 것입니다. [Kubernetes 통합에서 수집한 모든 속성과 값](/docs/integrations/kubernetes-integration/understand-use-data/understand-use-data) 을 결합하여 클러스터 보기의 범위를 좁힐 수 있습니다.

<img
  title="New Relic  - Cluster explorer - Search"
  alt="New Relic  - Cluster explorer - Search"
  src={nr1ClusterExplorerSearch}
/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Kubernetes cluster explorer** : 모든 Kubernetes 클러스터의 속성 및 데이터 포인트를 사용하여 클러스터 탐색기 보기를 필터링할 수 있습니다.
</figcaption>

오른쪽 상단 모서리에 있는 시간 선택기를 사용하여 시간 프레임을 변경할 수도 있습니다. **자동 새로 고침** 상자는 클러스터 탐색기를 60초마다 새로 고쳐지는 실시간 대시보드로 바꿉니다.

<img
  title="New Relic - Cluster explorer - Time picker and Auto-refresh"

  src={nr1ClusterExplorerPickerRefresh}
/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Kubernetes cluster explorer** : 시간 선택기를 사용하여 미리 정의된 여러 시간 범위를 선택할 수 있습니다. 1분마다 데이터를 다시 로드하려면 자동 새로 고침 상자를 선택합니다.
</figcaption>

## Kubernetes 이벤트 찾아보기

[Kubernetes 이벤트 통합](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration) 을 활성화한 경우 **이벤트** 탭을 클릭하여 경고에서 일반 이벤트에 이르기까지 클러스터에서 발생한 모든 것을 탐색할 수 있습니다. 설정하려면 [설치 마법사의](/docs/integrations/kubernetes-integration/installation/kubernetes-installation-configuration#install-wizard) 3단계에서 **Kubernetes 이벤트** 상자를 선택하거나 [지침](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration#install) 을 따르세요.
