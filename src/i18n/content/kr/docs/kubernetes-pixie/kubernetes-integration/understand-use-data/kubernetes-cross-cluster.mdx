최신 Kubernetes 환경은 복잡한 **다중 클러스터 플릿** 으로 발전했지만 기존의 옵저버빌리티 도구는 개별 클러스터에 대해 분리되고 사일로화된 보기만 제공하는 경우가 많습니다. 이 Kubernetes 크로스 클러스터 UI는 다중 클러스터의 복잡성을 전체 클러스터에 걸쳐 명확하게 파악할 수 있도록 통합된 명령 센터를 제공합니다.

이러한 통합된 관점은 플랫폼 엔지니어와 SRE에게 다음과 같은 도움을 줍니다.

* 뉴렐릭 에이전트 또는 OpenTelemetry 통해 클러스터 모니터와 호환되는 단일 대시보드에 **전체 옵저버빌리티를 통합합니다**.
* 가이드형 분류를 통해 **근본 원인 분석을 가속화하세요**.
* 전체 항공기단에서 낭비되는 자원을 파악하여 **비용을 절감하십시오**.
* **개발자에게 애플리케이션 중심적인 관점을 제공하여 셀프 서비스를 지원합니다**.

## 새로운 UI에 접속하세요 [#access]

뉴렐릭 플랫폼의 왼쪽 탐색에 있는 Kubernetes 옵션에서 새로운 Kubernetes 크로스 클러스터 UI 액세스할 수 있습니다.

Kubernetes 클러스터 간 UI에 액세스하려면 다음 단계를 따르세요.

1. 이동 <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes**</DNT>
2. 페이지 오른쪽 상단에 있는 **Try it out** \[직접 사용해 보기] 버튼을 클릭하세요.

Kubernetes 클러스터 간 UI는 전체 Kubernetes 클러스터에 대한 개괄적인 정보와 심층 분석 기능을 모두 제공하도록 구성되어 있습니다. UI 의 주요 구성 요소는 다음과 같습니다.

* **피드백 버튼**: 사용자 UI 사용 경험에 대한 피드백을 간편하게 공유할 수 있습니다.

* **엔티티 필터**: **클러스터 또는 노드 수준** 에서 **태그와 값을** 사용하여 문제를 필터링하고 분류합니다.
  * **Cluster 필터**: UI 를 단일 또는 더 작은 쿠버네티스 클러스터 세트로 분리하는 필터입니다.

* **색상별 성과표**: 전체 함대에서 주의가 필요한 가장 중요한 문제를 파악하기 위해 관련성이 높은 수준의 지표를 표시합니다.

  * 스코어카드를 클릭하면 해당 컨텍스트에 따라 **테이블이** 정렬되어 가장 중요한 문제가 있는 노드와 노드가 맨 위에 표시됩니다.
  * 또한 선택한 시간 범위에 걸쳐 변화 추이를 **선 그래프** 로 시각화할 수 있습니다.

* **선 그래프**: 선택한 기간 동안 **의 변화 추이를** 보여주어 추세 분석에 활용할 수 있습니다.

* **표**: 함대의 모든 분리 목록을 표시하고 5개의 기본 탭 아래 각 클러스터에 대한 **자세한 인덱스 데이터를** 제공합니다.

  * **건강 집중**: 이 표는 &quot;건강에 좋지 않은&quot; 항목만 나열하는 필터를 제공하여 정말 필요한 부분에 집중할 수 있도록 도와줍니다.
  * **검색**: 텍스트 검색창을 사용하면 입력한 **텍스트를** 기준으로 테이블 행을 필터링할 수 있습니다.
  * **색상 코드**: 점수표와 마찬가지로 표 셀은 특정 **심각도 기준을** 충족하는 문제를 빠르게 강조하기 위해 색상 코드(노란색, 빨간색)로 표시됩니다.

## 환자 분류를 위한 상세 분석 기능 [#drill-down]

크로스 클러스터 UI **가이드 분류를** 통해 근본 원인 분석을 가속화하도록 특별히 설계되었습니다. 이러한 심층 조사의 주요 메커니즘은 전체 함대의 관점에서 단일 함대의 관점으로 드릴다운할 수 있는 능력입니다.

* **스코어카드에서 클러스터 순위로:** **컬러 스코어카드** (예: Unhealthy 파드)를 클릭하면 클러스터 테이블이 즉시 다시 정렬됩니다. 이렇게 하면 특정 구간에서 성능이 가장 저조한 구간이 맨 위에 표시되므로 전체 차량에서 문제가 가장 심각한 부분을 파악할 수 있습니다.
* **테이블 지표에서 Kubernetes 네비게이터까지:** 특정 클러스터 또는 노드에 대한 테이블의 셀 내에서 지표 값을 클릭하면 **Kubernetes 네비게이터가** 자동으로 설치됩니다. 탐색기의 보기 화면은 관심 있는 특정 클러스터 및 메트릭에 맞게 자동으로 **사전 정의/필터링** 되므로 심층 분석을 통해 조사를 계속할 수 있습니다.
