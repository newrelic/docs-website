## 문제

New Relic의 Kubernetes 통합을 위한 [설치 절차](/docs/kubernetes-monitoring-integration#install) 를 완료했지만 New Relic 계정에 Kubernetes 데이터가 표시되지 않습니다.

## 해결책

[인프라 에이전트를 배포하고](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent/#install) [Kubernetes 설치 절차](/docs/kubernetes-monitoring-integration#install) 를 완료했지만 Kubernetes 대시보드에 정보가 표시되지 않는 경우 다음 단계를 따르세요.

1. 이 기능에 대한 액세스 권한이 있는지 확인하세요. [기능 및 데이터에 대한 액세스에 영향을 미치는 요소를](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data) 참조하십시오.

2. 배포 파일을 실행하기 전에 배포 파일에서 `CLUSTER_NAME` 및 `NRI_LICENSE_KEY` 변수를 구성했는지 확인합니다. 자세한 내용은 [설치 절차](/docs/integrations/host-integrations/host-integrations-list/kubernetes-monitoring-integration#install) 를 참조하십시오.

3. 다음을 실행하여 배포가 성공했는지 확인합니다.

   ```
   kubectl get pods --all-namespaces -o wide | grep newrelic
   ```

4. [인프라 에이전트에서 로그를 검색하고](/docs/integrations/kubernetes-integration/troubleshooting/get-logs-version) [오류 메시지](/docs/integrations/host-integrations/troubleshooting/kubernetes-integration-troubleshooting-error-messages) 가 없는지 확인합니다.
