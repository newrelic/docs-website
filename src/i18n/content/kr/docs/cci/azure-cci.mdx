Azure 청구의 완전한 비용 예측 및 최적화를 위해 Azure 비용 인텔리전스에 연결하세요. 서비스와 리소스 그룹 전반의 지출 패턴, 리소스 사용, 비용 할당에 대한 실시간 가시성을 확보하세요. Azure용 클라우드 비용 인텔리전스를 사용하면 다음을 수행할 수 있습니다.

* **리소스 사용 최적화** - 구독 및 리소스 그룹 전체에서 할당을 조정하기 위해 활용도가 낮거나 비용이 많이 드는 Azure 서비스와 VM을 찾습니다.
* **비용 제어** - 더 나은 재무 결정을 위해 컴퓨트, 스토리지, 네트워킹 및 데이터베이스 서비스를 포함하여 Azure 환경의 특정 비용 동인을 확인하세요.
* **효율성 향상** - Azure 리소스와 예산을 효율적으로 사용하기 위해 비즈니스 목표에 맞는 리소스 관리 전략에 집중
* **다중 구독 비용 모니터링** - 하나의 대시보드에서 여러 Azure 구독 및 관리 그룹의 지출을 추적합니다.
* **비용 책임성 활성화** - Azure 태그 및 리소스 그룹을 사용하여 부서 수준 비용 할당 및 청구 거부 구현
* **예산을 사전에 관리하세요** - Azure 지출에 대한 알림을 설정하여 예산 초과를 방지하고 재정 통제를 유지하세요.

## 전제 조건

Azure 클라우드 비용 인텔리전스에 연결하기 전에 다음 사항을 확인하세요.

* 활성 청구 계정이 있는 **Azure 구독**.
* 모니터링하려는 구독에 대한 **기여자 또는 소유자 역할입니다**.
* **Azure Cost Management 기능에 액세스합니다**.
* 구독 또는 관리 그룹에 대한 **청구 독자 권한입니다**.
* **Azure 청구 내보내기 구성 완료**: 이를 통해 연결 프로세스에 필요한 스토리지 계정, 컨테이너 및 디렉터리 경로가 설정됩니다. 자세한 내용은 [Azure 청구 내보내기 설정을](#azure-billing-export-setup) 참조하세요.

    이제 Azure 청구 내보내기 설정을 완료했으므로 Azure 청구를 Newrellic에 연결할 준비가 되었습니다. 먼저 뉴렐릭 계정을 선택하세요.

  <Step>
    ### 청구 데이터에 대한 액세스 권한을 부여하세요

    이제 Azure 연결 세부 정보를 제공했으므로 뉴렐릭에는 스토리지 컨테이너에서 청구 데이터를 읽을 수 있는 권한이 필요합니다. 이 단계에서는 뉴렐릭 서비스 주체(**msazurecostdata**)가 청구 내보내기에 안전하게 액세스할 수 있도록 승인합니다.

    1. 뉴렐릭 인터페이스에서 **Initiate Connection** \[연결 시작] 버튼을 클릭하세요.
    2. **Microsoft 인증** 흐름으로 리디렉션됩니다.

    <Callout variant="tip">
      이 단계에서는 Azure 관리자 승인이 필요합니다.
    </Callout>

    3. 인증 요청을 검토하고 수락합니다.
    4. 수락 시 작은 파일(0KB)이 귀하의 시스템에 다운로드됩니다. 이 파일은 안전하게 삭제할 수 있습니다.
  </Step>

  <Step>
    ### 서비스 주체가 추가되었는지 확인하세요

    인증 요청을 수락한 후 뉴렐릭 서비스 주체가 Azure 환경에 성공적으로 추가되었는지 확인하세요.

    1. **Microsoft Entra 계정** 에서 **Enterprise Applications** \[엔터프라이즈 애플리케이션] 으로 이동합니다.
    2. 서비스 주체 **msazurecostdata**가 목록에 나타나는지 확인합니다.

    <Callout variant="tip">
      엔터프라이즈 애플리케이션을 보는 데 도움이 필요하면 [Microsoft Entra 설명서를](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/view-applications-portal) 참조하세요.
    </Callout>
  </Step>

  <Step>
    ### 서비스 주체에게 읽기 권한 할당

    청구 내보내기에 대한 읽기 전용 액세스를 위해 서비스 주체에 \*\*Storage Blob Data Reader\*\* 역할을 할당하여 스토리지 계정에 대한 액세스와 IAM 역할 할당을 수정할 수 있는 권한을 보장합니다.

    1. 청구 내보내기가 게시된 저장소 계정 컨테이너를 엽니다. 왼쪽 탐색 메뉴에서 **Access Control (IAM)** \[액세스 제어(IAM)]를 선택합니다.
    2. **Role Assignments** \[역할 할당] 탭에서 **Add** \[추가] 또는 **Add role assignment** \[역할 할당 추가를] 클릭합니다.
    3. 역할 목록에서 **Storage Blob Data Reader**를 선택하여 청구 내보내기에 대한 읽기 전용 액세스 권한을 제공합니다.
    4. **Add Members** \[멤버 추가] 섹션에서 `msazurecostdata` (검증된 서비스 주체)를 검색하여 선택합니다.
    5. 역할 할당 설정을 검토합니다. **Review + Assign** \[검토 + 할당을] 클릭하고 역할 할당 목록에 `msazurecostdata` 이 나타나는지 확인합니다.
  </Step>

  <Step>
    ### 연결을 확인하세요

    위의 모든 단계를 완료한 후 다음 사항을 확인하세요.

    * 서비스 주체 **msazurecostdata**는 Enterprise에 표시됩니다.
    * 서비스 주체에 **Storage Blob Data Reader** 역할이 할당되었습니다.
    * 역할 할당은 올바른 스토리지 계정 컨테이너에 적용됩니다.
    * 뉴렐릭 인터페이스에서 연결 테스트가 성공적으로 완료되었습니다.

    <Callout variant="tip">
      **보안 참고 사항**: 서비스 주체는 청구 데이터에 대해 읽기 전용 액세스 권한만 갖습니다. 어떠한 데이터도 쓰거나, 수정하거나, 삭제할 수 없습니다. 액세스는 귀하가 지정한 청구 내보내기 컨테이너로만 제한됩니다.
    </Callout>
  </Step>

  <Step>
    ### 무엇 향후 계획

    * 뉴렐릭이 Azure 청구 내보내기 처리를 시작합니다.
    * 초기 데이터 처리에는 최대 12시간이 소요될 수 있습니다.
    * 처리가 완료되면 비용 개요 대시보드에 비용 데이터가 표시됩니다.

    <Callout variant="important">
      처음으로 데이터를 처리하는 데 최대 12시간이 걸릴 수 있습니다.
    </Callout>
  </Step>

  </Step>
