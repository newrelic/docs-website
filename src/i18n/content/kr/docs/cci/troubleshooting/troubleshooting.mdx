설치 중에 문제가 발생하면 아래에서 해결 방법을 찾아보세요.

## 보고서에서 누락된 데이터

비용 사용량 보고서(CUR), 실시간 비용 추정 또는 Kubernetes 비용 할당 데이터가 표시되지 않는 경우:

* **사유:** 필수 요건 미충족

* **해결책:**

  1. 모든 설치 필수 조건이 완료되었는지 확인하십시오.
  2. 설정 구성을 확인하세요
  3. 문제가 지속되면 [클라우드 분석 인텔리전스 지원팀](mailto:cci-support@newrelic.com) 에 문의하세요.

## 설정 중에 연결이 실패합니다.

* **이유:** 불안정한 네트워크 연결
* **해결 방법:** 안정적인 인터넷 연결을 확인하고 설정을 다시 시도하십시오. 문제가 지속되면 네트워크 관리자에게 방화벽 제한 사항을 확인하십시오.

## 권한이 부족합니다

* **이유:** 사용자에게 필요한 권한이 없습니다.
* **해결 방법:** AWS 와 뉴렐릭에 관리자 권한 또는 필요한 IAM 정책이 있는지 확인하십시오. 필요한 경우 관리자에게 문의하십시오.

## ARN 설정 오류

* **메시지:** &quot;입력한 ARN은 AWS 계정에 대한 올바른 액세스를 허용하지 않습니다. ARN을 확인하고 다시 시도하세요.&quot;
* **이유:** 잘못된 ARN 설정
* **해결 방법:** ARN이 필수 설정과 일치하는지 확인하십시오. 오류를 수정하고 설정을 다시 시도하세요.

## 외부 ID로 인한 보안 위험

* **메시지:** &quot;보안 위험: 입력한 ARN에는 광범위한 권한이 있습니다. 모든 &apos;외부 ID&apos;를 사용하여 역할을 맡을 수 있습니다. AWS 역할을 생성하는 동안 제공된 외부 ID &apos;%s&apos;를 externalId로 추가하세요.&quot;
* **이유:** 지나치게 관대한 외부 ID 정책
* **해결 방법:** 제공된 외부 ID를 externalId로 사용하도록 AWS 역할 설정을 업데이트합니다.

## S3 버킷을 찾을 수 없습니다.

* **이유:** S3 버킷이 존재하지 않거나 버킷 이름이 잘못되었습니다.
* **해결 방법:** 버킷 이름이 AWS 계정 이름과 일치하는지 확인하세요. S3 경로 접두사가 `cost-and-usage-reports` 인지 확인하십시오. 맞춤법을 확인하고, 필요한 경우 버킷을 생성하거나 이름을 수정하세요.

## 중복된 연결 이름

* **메시지:** &quot;%s 클라우드 분석 인텔리전스 AWS 연결 이름이 이미 존재합니다. 새로운 연결 이름을 입력하세요.&quot;
* **이유:** 연결 이름이 이미 사용 중입니다.
* **해결 방법:** 고유한 연결 이름을 선택하고 다시 시도하십시오.

## 중복 지급인 계정 설정

* **메시지:** &quot;%s AWS Payer 계정이 이미 이 뉴렐릭 계정에 연결되어 있습니다.&quot;
* **이유:** AWS 결제 계정이 이미 구성되어 있습니다.
* **해결 방법:** 결제자 계정이 이미 연결되어 있는지 확인하십시오. 필요한 경우 다른 결제 계정을 사용하십시오.

## 추가 지원

문제가 지속되면 [클라우드 컴퓨팅 인텔리전스 지원팀](mailto:cci-support@newrelic.com) 에 문의하세요.

## 클라우드폼 설정

**Create CFT Stack** \[CFT 스택 생성에] 실패하는 경우:

1. 선택한 스택의 Events \[이벤트] 탭/열에서 Status \[상태] 행을 확인하세요.
2. **CREATE\_FAILED** 오류를 찾아서 해당 상태 이유 메시지를 뉴렐릭에게 공유해 주세요.
3. **CloudWatch** 로 이동하여 **Cloudwatch logs** \[CloudWatch 로그] 에서 cloud 스택 이름을 선택하고 **View in Logs console** \[로그 콘솔에서 보기]를 클릭합니다.
4. **Log Streams** \[로그인 Streams] 아래에서 로그인을 열고 로그인을 뉴렐릭 지원팀과 공유하세요.
