AWS, Azure 및 GCP 전반에 걸쳐 cloud 공급자 연결의 상태를 통합하고 상태 기반 작업을 통해 연결을 관리합니다.

<Callout variant="important">
  cloud 공급자 통합을 하나 이상 성공적으로 완료하면 통합 탭에 액세스할 수 있습니다.
</Callout>

## 액세스 통합

1. <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> 으로 이동 &gt; <DNT>**All Capabilities**</DNT> 선택 &gt; <DNT>**Cloud Cost Intelligence**</DNT>.
2. 모든 cloud 비용 통합을 보려면 <DNT>**Integrations**</DNT> 탭을 선택하세요.
3. cloud 제공업체 계정에 대한 새 통합을 추가하려면 <DNT>**+ Add new integrations**</DNT> 을(를) 선택하세요.
4. cloud 공급업체 간 통합 상태를 평가하려면 <DNT>**Status**</DNT> 열을 확인하세요.

## 통합 상태를 파악합니다.

각 통합에는 다음 상태 표시기 중 하나가 표시됩니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        상태
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**PENDING**</DNT>
      </td>

      <td>
        통합이 생성되었으며 CCI의 검증을 기다리고 있습니다. 새 통합이 처음 설정될 때 할당되는 초기 상태입니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**IN PROGRESS**</DNT>
      </td>

      <td>
        연결 설정이 진행 중이며, 초기 구성 및 인증이 설정되고 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**VALIDATED**</DNT>
      </td>

      <td>
        연결 인증이 성공적으로 완료되었고 모든 필수 검사를 통과했습니다. 스토리지 계정에 첫 번째 비용 파일이 게시될 때까지 기다리고 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**RUNNING**</DNT>
      </td>

      <td>
        통합 기능이 완전히 작동 중이며 비용 데이터를 적극적으로 수집하고 있습니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**FAILED**</DNT>
      </td>

      <td>
        초기 설정 또는 CUR 보고서 유효성 검사 중 연결에 오류가 발생하여 주의가 필요합니다. 오류 세부 정보 및 해결 단계를 보려면 연결 이름을 클릭하세요.

        <Callout variant="important">
          <DNT>**FAILED**</DNT> 연결이 <DNT>**RUNNING**</DNT> 상태에 도달하기 전, 초기 통합 설정 단계에만 적용됩니다.
        </Callout>
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**ERROR**</DNT>
      </td>

      <td>
        이전에 <DNT>**RUNNING**</DNT> 연결에 통합 문제가 발생했습니다. 초기 설정 중에 발생하는 <DNT>**FAILED**</DNT>과(와) 달리, <DNT>**ERROR**</DNT> 은(는) 이미 활성화된 연결에 문제가 있음을 나타냅니다. 오류 세부 정보 및 해결 단계를 보려면 연결 이름을 클릭하세요.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**PAUSED**</DNT>
      </td>

      <td>
        데이터 수집이 일시 중단되었습니다. 연결이 <DNT>**RUNNING**</DNT> 상태에 도달한 후 일시 중지하고 나중에 재개할 수 있습니다.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  통합이 <DNT>**RUNNING**</DNT> 상태로 유지되도록 <DNT>**Status**</DNT> 열을 정기적으로 모니터링하십시오. 비용 데이터 수집 갭을 방지하려면 <DNT>**FAILED**</DNT> 또는 <DNT>**ERROR**</DNT> 통합을 신속하게 처리하고, 활성화를 앞당기려면 <DNT>**IN PROGRESS**</DNT> 단계에서 유효성 검사 요구 사항을 검토하십시오.
</Callout>

## 통합 관리

통합 옆에 있는 점 세 개 아이콘을 클릭하여 사용 가능한 작업에 액세스하세요. 표시되는 작업은 현재 통합 상태에 따라 다릅니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        동작
      </th>

      <th style={{ width: "150px" }}>
        이용 가능합니다
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Resume**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> 통합(통합이 성공적으로 생성되었지만 CUR 보고서 유효성 검사에 실패한 경우에만) 및 <DNT>**PAUSED**</DNT> 통합
      </td>

      <td>
        데이터 수집을 재개하거나 연결 설정 프로세스를 재시도합니다. 유효성 검사 오류로 인해 통합 자체가 생성되지 않은 <DNT>**FAILED**</DNT> 통합의 경우 재개 옵션을 사용할 수 없습니다. <DNT>**PAUSED**</DNT> 통합의 경우, 재개 시 활성 데이터 수집이 복원됩니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> 통합
      </td>

      <td>
        설정 정보를 유지하면서 데이터 수집을 일시적으로 중단합니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> 통합 (<DNT>**RUNNING**</DNT> 상태에 도달하기 전)
      </td>

      <td>
        통합 기능을 영구적으로 제거하고 새로운 연결 설정을 가능하게 합니다.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  통합 설정을 유지하려면 일시 중단 시 <DNT>**Delete**</DNT> 대신 <DNT>**Pause**</DNT> 을(를) 사용합니다. <DNT>**Resume**</DNT> 을(를) 사용하여 <DNT>**PAUSED**</DNT> 통합을 복원하거나 연결이 생성되었지만 CUR 보고서 유효성 검사에 실패한 <DNT>**FAILED**</DNT> 통합을 재시도합니다.
</Callout>
