AWS, Azure 및 GCP 전반에 걸쳐 cloud 공급자 연결의 상태를 통합하고 상태 기반 작업을 통해 연결을 관리합니다.

<Callout variant="important">
  cloud 공급자 통합을 하나 이상 성공적으로 완료하면 통합 탭에 액세스할 수 있습니다.
</Callout>

## 액세스 통합

1. <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> 으로 이동 &gt; <DNT>**All Capabilities**</DNT> 선택 &gt; <DNT>**Cloud Cost Intelligence**</DNT>.
2. Select the <DNT>**Integrations**</DNT> tab to view all cloud cost integrations.
3. Select the <DNT>**+ Add new integrations**</DNT> to add a new integration for a cloud provider account.
4. Check the <DNT>**Status**</DNT> column to assess integration health across your cloud providers.

## 통합 상태를 파악합니다.

각 통합에는 다음 상태 표시기 중 하나가 표시됩니다.

<Callout variant="tip">
  Monitor the <DNT>**Status**</DNT> column regularly to ensure integrations remain <DNT>**RUNNING**</DNT>. Address <DNT>**FAILED**</DNT> or <DNT>**ERROR**</DNT> integrations promptly to avoid gaps in cost data collection, and review validation requirements during the <DNT>**IN PROGRESS**</DNT> phase to expedite activation.
</Callout>

## 통합 관리

통합 옆에 있는 점 세 개 아이콘을 클릭하여 사용 가능한 작업에 액세스하세요. 표시되는 작업은 현재 통합 상태에 따라 다릅니다.

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        동작
      </th>

      <th style={{ width: "150px" }}>
        이용 가능합니다
      </th>

      <th>
        설명
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Resume**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> integrations (only where the integration was created successfully but CUR report validation failed) and <DNT>**PAUSED**</DNT> integrations
      </td>

      <td>
        Resumes data collection or reattempts the connection setup process. For <DNT>**FAILED**</DNT> integrations where the integration itself was not created due to a validation error, the resume option is not available. For <DNT>**PAUSED**</DNT> integrations, resume restores active data collection.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> 통합
      </td>

      <td>
        설정 정보를 유지하면서 데이터 수집을 일시적으로 중단합니다.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> integrations (prior to reaching <DNT>**RUNNING**</DNT> status)
      </td>

      <td>
        통합 기능을 영구적으로 제거하고 새로운 연결 설정을 가능하게 합니다.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Use <DNT>**Pause**</DNT> rather than <DNT>**Delete**</DNT> for temporary suspensions to preserve your integration configuration. Use <DNT>**Resume**</DNT> to restore a <DNT>**PAUSED**</DNT> integration or retry a <DNT>**FAILED**</DNT> integration where the connection was created but CUR report validation failed.
</Callout>
