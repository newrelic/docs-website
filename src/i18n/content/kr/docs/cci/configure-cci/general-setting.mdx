  이 기능은 현재 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 미리보기 프로그램의 일부로 제공됩니다.
</Callout>

클라우드 해상 인텔리전스는 자원 비용 할당을 위해 CUR(Cost and Usage Reports)의 기본 태그와 함께 사용자 정의 태그를 지원합니다. 자세한 비용 분석을 위해 태그를 필터링하고 그룹화하세요. Kubernetes 환경에서 클러스터와 리소스에 레이블을 적용합니다. 태그 선택다운 드롭 메뉴는 CUR 데이터와 `K8SContainerSample` 이벤트를 기반으로 합니다.

## 설정 구성

<Steps>
  <Step>
    ### 일반 설정으로 이동

    <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT> &gt; **General** 탭으로 이동하세요.
  </Step>

  <Step>
    ### 태그 구성

    비용 및 사용량 파일에서 리소스 태그와 Kubernetes 레이블을 선택합니다.

    1. **Choose Resource tags for cost allocation** \[비용 할당 목록에서 리소스 태그를 선택하여] 서비스별 cloud 비용을 분류하세요.
    2. **Choose Kubernetes tags for cost mapping** \[비용 매핑을 위한 Kubernetes 태그 선택] 목록에서 Kubernetes 레이블을 선택하여 비용을 분류합니다.
  </Step>

  <Step>
    ### 텔레메트리 데이터 액세스 관리

    1. **Telemetry Data** \[텔레메트리 데이터] 탭을 선택합니다.
    2. **Select accounts for telemetry data** \[텔레메트리 데이터에 대한 계정 선택] 목록에서 텔레메트리 데이터에 대한 계정을 선택합니다.

    <Callout variant="important">
      앞으로 비용 추정 및 K8s 할당은 스프레드시트 데이터를 통해 지원됩니다. 활용 지표를 위해 기존 텔메트리 파이프라인을 재사용합니다. CUR 파일 수집 외에는 새로운 데이터가 수집되지 않았습니다.
    </Callout>
  </Step>
</Steps>
