  이 기능은 아직 개발 중이지만 꼭 사용해 보시기 바랍니다!

  이 기능은 현재 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 미리보기 프로그램의 일부로 제공됩니다.
</Callout>

클라우드 해상 인텔리전스는 기본 태그와 함께 사용자 정의 태그를 지원하며, 리소스 비용 할당을 위해 비용 및 사용 보고서(CUR)에 통합됩니다. 이를 통해 태그를 필터링하고 그룹화하여 세부적인 비용 분석을 수행할 수 있습니다. Kubernetes 환경에서는 클러스터와 리소스에 레이블을 적용할 수 있습니다. 태그 선택을 위한 드롭다운 메뉴는 CUR 데이터와 `K8SContainerSample` 이벤트를 기반으로 하여 관리 및 분석을 간소화합니다.

## 일반 설정 구성

<Steps>
  <Step>
    ## 일반 설정으로 이동

    1. <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT> 으로 이동합니다.
    2. 왼쪽 탐색 메뉴에서 **General** \[일반] 탭을 선택하세요.
  </Step>

  <Step>
    ## 태그 구성

    비용 및 사용량 파일을 통해 얻은 값에서 원하는 리소스 태그와 Kubernetes 레이블을 선택하여 관리 및 분석을 간소화합니다.

    1. 서비스별로 cloud 비용을 분류하고 분석하려면 **Choose Resource tags for cost allocation** \[비용 할당을 위한 리소스 태그 선택] 목록에서 리소스 태그를 선택합니다.
    2. 서비스별로 클라우드 비용을 분류하고 분석하려면 **Choose Kubernetes tags for cost mapping** \[비용 매핑을 위한 Kubernetes 태그 선택] 목록에서 Kubernetes 레이블을 선택합니다.
  </Step>

  <Step>
    ## 텔레메트리 데이터 액세스 관리

    1. 비용 분석을 위해 텔리메트리 데이터를 가져오려는 계정에 대한 정보를 요청하려면 **Telemetry Data** \[텔리메트리 데이터] 탭을 선택하세요.
    2. 비용 분석을 위한 텔레메트리 데이터를 보고 관리하려면 **Select accounts for telemetry data** \[텔레메트리 데이터에 대한 계정 선택] 목록에서 하나 이상의 계정을 선택하십시오.

      <Callout variant="important">
        * 향후 비용 추정은 텔레메트리 데이터를 통해 지원됩니다.
        * K8s 할당은 텔레메트리 데이터를 통해 지원됩니다.
        * 기존 텔레메트리 파이프라인을 재사용하여 활용도 지표를 얻고, CUR 파일 수집을 제외하고는 새로운 데이터를 수집하지 않습니다.
      </Callout>
  </Step>
</Steps>
