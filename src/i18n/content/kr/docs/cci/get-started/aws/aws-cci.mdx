통합 청구, 비용 및 사용량 보고서(CUR), ID 및 액세스 관리(IAM) 역할 및 정책, 그리고 필요한 AWS 리전을 구성하여 클라우드 비용 인텔리전스를 위한 AWS 환경을 설정하십시오.

## 전제 조건

두 가지 설치 방법 중 하나를 시작하기 전에 다음 사항을 확인하십시오.

### 모든 설정에 필수입니다.

* **통합 결제가 포함된AWS 계정**: [AWS 조직에 대한 통합 결제를 설정합니다](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html). 이는 조직 내 모든 계정의 비용 데이터를 집계하여 전체 AWS 인프라에 걸쳐 포괄적인 비용 분석을 가능하게 하는 데 필요합니다.
* **IAM 권한**: 바구니, 목표 AWS 계정에 대한 관리 액세스는 구현하다, 배포하다 리소스, S3 버킷 생성, IAM 정책 및 IAM 역할에 필요합니다.

### 자동 설정에만 필요합니다.

* **뉴렐릭 API 키**: 그리드 생성을 위한 유효한 API 키를 생성합니다. CCI 통합을 위해 전용 키를 생성하는 것을 권장합니다. 뉴렐릭 UI: [사용자 메뉴 &gt; API Keys](https://onenr.io/07wklp68lRL) 에서 이를 생성합니다.

## 설치 방법을 선택하세요

클라우드 솔루션 인텔리전스는 AWS 통합을 설정하는 두 가지 방법을 제공합니다.

* [**CloudFormation을 사용하여 AWS 자동화(권장)**](/docs/cci/get-started/aws/automated-setup-cloudformation): 필요한 모든 AWS 리소스를 자동으로 프로비저닝하는 사전 구성된 CloudFormation 스택을 신속하게 구현하고 배포합니다. 최소한의 수동 설정으로 빠른 온보딩이 가능한 최적의 솔루션입니다.
* [**AWS 계정을 수동으로 통합하세요**](/docs/cci/get-started/aws/manual-integrate-aws-account). 각 AWS 리소스를 수동으로 구성하여 설정 프로세스를 완벽하게 제어할 수 있습니다. 특정 규정 준수 또는 맞춤 설정 요구 사항이 있는 조직에 가장 적합합니다.
