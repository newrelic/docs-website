통합 청구, 비용 및 사용 보고서(CUR), Identity and Access Management(IAM) 역할 및 정책, 필요한 AWS 리전을 구성하여 <DNT>**Cloud Cost Intelligence**</DNT> 을(를) 위한 AWS 환경을 설정합니다.

## 전제 조건

두 가지 설치 방법 중 하나를 시작하기 전에 다음 사항을 확인하십시오.

### 모든 설정에 필수입니다.

* <DNT>**AWS account with consolidated billing**</DNT>: [AWS 조직의 통합 결제 설정](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html). 이는 조직의 모든 계정에서 비용 데이터를 집계하는 데 필요하며, 이를 통해 전체 AWS 인프라에 대한 포괄적인 비용 분석이 가능해집니다.
* <DNT>**IAM Privileges**</DNT>: 리소스를 배포하고, S3 버킷, IAM 정책 및 IAM 역할을 생성하려면 타겟 AWS 계정에 대한 관리자 액세스 권한이 필요합니다.

### 자동 설정에만 필요합니다.

* <DNT>**New Relic API Key**</DNT>: 스택 생성을 위한 유효한 API 키를 생성합니다. <DNT>**CCI**</DNT> 통합을 위해 전용 키를 생성하는 것을 권장합니다. 뉴렐릭 UI: [사용자 메뉴 &gt; API Keys](https://onenr.io/07wklp68lRL)에서 이를 생성합니다.

## 설치 방법을 선택하세요

<DNT>**Cloud Cost Intelligence**</DNT> AWS 통합을 설정하는 두 가지 방법을 제공합니다:

* <DNT>[**Automate AWS with CloudFormation (Recommended)**](/docs/cci/get-started/aws/automated-setup-cloudformation)</DNT>: 필요한 모든 AWS 리소스를 자동으로 프로비저닝하는 사전 구성된 CloudFormation 스택을 빠르게 배포합니다. 최소한의 수동 설정으로 더 빠른 온보딩에 가장 적합합니다.
* <DNT>[**Manually integrate your AWS account**](/docs/cci/get-started/aws/manual-integrate-aws-account)</DNT>: 설정 프로세스를 완전히 제어하려면 각 AWS 리소스를 수동으로 구성합니다. 특정 규정 준수 또는 커스터마이징 요구 사항이 있는 조직에 가장 적합합니다.
