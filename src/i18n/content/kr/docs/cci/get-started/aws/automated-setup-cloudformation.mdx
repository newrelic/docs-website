자동화된 AWS CloudFormation 템플릿을 사용하여 <DNT>**Cloud Cost Intelligence**</DNT> 설정을 가속화하세요. 사전 구성된 스택은 S3 버킷 설정, CUR 내보내기 설정, [Trusted Advisor 설정 확인](/docs/cci/financial-intelligence/cci-recommendations/#aws-trusted-advisor) 및 필요한 IAM 역할과 정책을 포함하여 필요한 AWS 환경을 자동으로 프로비저닝합니다. 배포하기 전에 AWS CloudFormation 콘솔에서 모든 리소스 권한을 검토하십시오.

## 비용 고려 사항

이 템플릿은 CUR 데이터 수집을 위한 S3 버킷을 자동으로 생성합니다. 기본 지역은 `us-east-2` 이지만, CloudFormation 요구 사항에 맞춰 이를 변경할 수 있습니다.

## 설치 지침

<Steps>
  <Step>
    ### 클라우드 컴퓨팅 인텔리전스에 액세스하세요

    <DNT>[**one.newrelic.com**](https://one.newrelic.com/)</DNT> <DNT>**Integrations &amp; Agents**</DNT> <DNT>**Cloud Cost Intelligence - AWS**</DNT>(으)로 이동합니다.
  </Step>

  <Step>
    ### CloudFormation 자동화를 선택하세요

    <DNT>**Automate AWS with CloudFormation (Recommended)**</DNT> 선택하고 <DNT>**Continue**</DNT> 클릭합니다.
  </Step>

  <Step>
    ### AWS의 CloudFormation

    1. 연결 이름을 확인하고 뉴렐릭 데이터 지역을 선택하세요.
    2. <DNT>**Launch your CloudFormation in AWS**</DNT> 을(를) 클릭합니다.
    3. 프롬프트가 표시되면 <DNT>**Admin role**</DNT> 로 AWS 계정에 로그인합니다. AWS 콘솔에서 <DNT>**CloudFormation Stacks**</DNT> 화면이 열립니다.
  </Step>

  <Step>
    ### 구성해야 합니다.

    <DNT>**Create Stack**</DNT> 양식에서 다음과 같은 파라미터를 구성합니다:

    <DNT>**Pre-populated parameters (do not modify):**</DNT>

    * <DNT>**New Relic Account ID**</DNT>: 미리 채워짐. 변경하지 마십시오.
    * <DNT>**New Relic Data Region**</DNT>: 미리 채워짐. 변경하지 마십시오.
    * <DNT>**External ID**</DNT>: 미리 채워짐. 변경하지 마십시오.

    <DNT>**Required parameters:**</DNT>

    * <DNT>**NewRelicAPIKey**</DNT>: 뉴렐릭 API 키를 입력하세요. New Relic UI에서 다음 경로를 통해 생성하십시오: 사용자 메뉴 &gt; API Keys.
    * <DNT>**Stack Name**</DNT>: 기본값이 제공됩니다. 필요한 경우 편집하되, 이름이 모든 AWS CloudFormation 스택에서 전역적으로 고유한지 확인하세요. 기본값을 유지하는 것이 좋습니다.
    * <DNT>**Bucket Name**</DNT>: 기본값이 제공됩니다. 필요한 경우 편집하되, 이름이 모든 S3 버킷에서 전역적으로 고유한지 확인합니다.
    * <DNT>**Connection Name**</DNT>: 기본값이 제공됩니다. 필요한 경우 명명 규칙에 맞게 편집하세요.

    <DNT>**Optional parameters:**</DNT>

    * <DNT>**Grant Access to Trusted Advisor**</DNT>: 기본값은 <DNT>**Yes**</DNT>입니다. 활성화하면 계정에 <DNT>**Trusted Advisor**</DNT> 비용 인사이트가 활성화되어 있는지 확인하고 추천 사항을 가져올 수 있습니다. <DNT>**No**</DNT> 을(를) 선택하거나 <DNT>**Trusted Advisor**</DNT> 이(가) 활성화되어 있지 않으면 <DNT>**Trusted Advisor**</DNT> 추천을 사용할 수 없습니다.

    <Callout variant="important">
      <DNT>**CCI**</DNT> 네이티브 권장 사항은 <DNT>**Trusted Advisor**</DNT> 와(과) 독립적이며 이 설정과 관계없이 작동합니다.
    </Callout>

    다른 모든 규격은 선택 사항이며 필요에 따라 구성할 수 있습니다.
  </Step>

  <Step>
    ### 스택을 생성합니다

    <DNT>**I acknowledge that AWS CloudFormation might create IAM resources**</DNT> 선택하고 <DNT>**Create stack**</DNT> 클릭합니다.
  </Step>

  <Step>
    ### 스택 생성 확인

    스택 생성이 성공적으로 완료되면 상태에 <DNT>**CREATE\_COMPLETE**</DNT>이(가) 표시됩니다.
  </Step>

  <Step>
    ### CUR 보고서 생성 여부를 확인합니다.

    24시간 기다린 후 AWS 버킷에 parquet 형식의 CUR 보고서가 생성되었는지 확인하십시오.

    <Callout variant="important">
      아래 경로는 예시입니다. 실제 경로에는 서로 다른 타임스탬프, UUID 및 청구 기간이 적용됩니다.

      <DNT>**Amazon S3**</DNT> &gt; <DNT>**Buckets**</DNT> &gt; <DNT>**bucket\_name**</DNT> &gt; <DNT>**cost-and-usage-reports/**</DNT> &gt; <DNT>**bucket\_name-cci/data**</DNT> &gt; <DNT>**BILLING\_PERIOD=2025-12/**</DNT> &gt; <DNT>**2025-12-24T03:05:19.886Z-a2e23b2a-588a-4305-97fd-be712f555b75/**</DNT> &gt; <DNT>**bucket\_name-cci-00001.snappy.parquet**</DNT>
    </Callout>

    통합 설정이 완료되었습니다.
  </Step>
</Steps>

## 제한 사항

자동화된 <DNT>**CloudFormation**</DNT> 템플릿은 다음을 수행하지 않습니다:

* 컴퓨트 또는 네트워크 리소스를 생성하거나 구성합니다.
* 추가 AWS 서비스를 활성화하거나 지원 수준을 업데이트하세요.
* 로그, 트레이스 및 기타 지표 수집 프로세스를 자동화합니다.
* 쓰기 권한 역할 또는 정책을 생성하세요.

## 구성된 정책 및 권한

자동화된 스택은 다음과 같은 액세스 수준을 구성합니다.

* <DNT>**Billing Data Access**</DNT>: 비용 및 사용 보고서를 저장하기 위해 생성된 특정 S3 버킷에 대한 읽기 전용 권한.
* <DNT>**Service Pricing Access**</DNT>: 사용량을 공개 요금과 연관시키기 위한 AWS Pricing API에 대한 읽기 전용 액세스.
* <DNT>**Configuration Automation**</DNT>: 설정 단계에서 CUR 내보내기를 생성하고 통합 상태를 확인하려면 제한된 권한이 필요합니다.
* <DNT>**Optional Optimization Access**</DNT>: (선택한 경우) 비용 절감 권장 사항을 표시하기 위한 AWS Trusted Advisor 읽기 전용 액세스 권한.
