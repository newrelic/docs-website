  이 기능은 현재 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 미리보기 프로그램의 일부로 제공됩니다.
</Callout>

AWS CloudFormation 자동화 템플릿을 사용하여 클라우드 비용 인텔리전스 설정을 가속화하세요. 사전 구성된 스택은 S3 버킷 구성, CUR 내보내기 설정, [Trusted Advisor 설정 확인](/docs/cci/financial-intelligence/cci-recommendations/#aws-trusted-advisor), 필요한 IAM 역할 및 정책을 포함하여 필요한 AWS 환경을 자동으로 프로비저닝합니다. 구현 및 배포 전에 AWS CloudFormation 콘솔에서 모든 리소스 권한을 검토하십시오.

## 비용 고려 사항

이 템플릿은 CUR 데이터 수집을 위한 S3 버킷을 자동으로 생성합니다. 기본 지역은 `us-east-2` 이지만, CloudFormation 요구 사항에 맞춰 이를 변경할 수 있습니다.

## 설치 지침

<Steps>
  <Step>
    ### 클라우드 컴퓨팅 인텔리전스에 액세스하세요

    <DNT>[**one.newrelic.com**](https://one.newrelic.com/) &gt; **Integrations &amp; Agents** &gt; **Cloud Cost Intelligence - AWS**</DNT> 으로 이동합니다.
  </Step>

  <Step>
    ### CloudFormation 자동화를 선택하세요

    **Automate AWS with CloudFormation (Recommended)** \[CloudFormation을 사용하여 AWS 자동화(권장)]를 선택하고 **Continue** \[계속을] 클릭합니다.
  </Step>

  <Step>
    ### AWS의 CloudFormation

    1. 연결 이름을 확인하고 뉴렐릭 데이터 지역을 선택하세요.
    2. **Launch your CloudFormation in AWS** \[AWS에서 CloudFormation을 시작하려면] 클릭하세요.
    3. 로그인 시 **관리자 역할** 로 AWS 계정에 로그인합니다. AWS 콘솔에서 **CloudFormation Stacks** \[CloudFormation 스택] 화면이 열립니다.
  </Step>

  <Step>
    ### 구성해야 합니다.

    **Create Stack** \[Create 그리드] 양식에서 다음과 같은 트리거를 구성합니다.

    **미리 채워진 모범 사례, 수정하지 마세요:**

    * **뉴렐릭 계정 ID**: 미리 입력되어 있습니다. 변경하지 마세요.
    * **뉴렐릭 데이터 영역**: 사전 채워짐. 변경하지 마세요.
    * **외부 ID**: 미리 입력되어 있습니다. 변경하지 마세요.

    **필요한 조치:**

    * **NewRelicAPIKey**: 뉴렐릭 API 키를 입력하세요. 뉴렐릭 UI: 사용자 메뉴 &gt; API Keys에서 이를 생성합니다.
    * **스택 이름**: 기본값이 제공됩니다. 필요한 경우 수정하되, 모든 AWS CloudFormation 스택에서 이름이 전역적으로 고유한지 확인하십시오. 기본값을 유지하는 것을 권장합니다.
    * **버킷 이름**: 기본값이 제공됩니다. 필요에 따라 수정하되, 모든 S3 버킷에서 이름이 전역적으로 고유한지 확인하십시오.
    * **연결 이름**: 기본값이 제공됩니다. 명명 규칙에 맞게 수정하십시오.

    **선택적 조치:**

    * **Trusted Advisor에 액세스 권한 부여**: 기본값은 **Yes** \[예] 입니다. 활성화하면 계정에 **Trusted Advisor** 비용 인사이트가 활성화되어 있는지 확인하고 추천 사항을 가져올 수 있습니다. **No** \[아니요]를 선택하거나 **Trusted Advisor** \[신뢰할 수 있는 조언자] 가 활성화되어 있지 않으면 **Trusted Advisor** \[신뢰할 수 있는 조언자] 추천을 사용할 수 없습니다.

    <Callout variant="important">
      CCI 네이티브 추천 기능은 **Trusted Advisor** 와 독립적이며 해당 설정과 관계없이 작동합니다.
    </Callout>

    다른 모든 규격은 선택 사항이며 필요에 따라 구성할 수 있습니다.
  </Step>

  <Step>
    ### 스택을 생성합니다

    **I acknowledge that AWS CloudFormation might create IAM resources** \[AWS CloudFormation이 IAM 리소스를 생성할 수 있음을 인정합니다] 를 선택하고 **Create stack** \[스택 생성]을 클릭합니다.
  </Step>

  <Step>
    ### 스택 생성 확인

    스택 생성이 성공적으로 완료되면 상태에 **CREATE\_COMPLETE**가 표시됩니다.
  </Step>

  <Step>
    ### CUR 보고서 생성 여부를 확인합니다.

    24시간 기다린 후 AWS 버킷에 parquet 형식의 CUR 보고서가 생성되었는지 확인하십시오.

    <Callout variant="important">
      아래 경로는 예시입니다. 실제 경로에는 서로 다른 타임스탬프, UUID 및 청구 기간이 적용됩니다.

      > **Amazon S3** &gt; **Buckets** &gt; **bucket\_name** &gt; **cost-and-usage-reports/** &gt; **bucket\_name-cci/data** &gt; **BILLING\_PERIOD=2025-12/** &gt; **2025-12-24T03:05:19.886Z-a2e23b2a-588a-4305-97fd-be712f555b75/** &gt; **bucket\_name-cci-00001.snappy.parquet**
    </Callout>

    통합 설정이 완료되었습니다.
  </Step>
</Steps>

## 제한 사항

자동화된 CloudFormation 템플릿은 다음을 수행하지 않습니다.

* 컴퓨트 또는 네트워크 리소스를 생성하거나 구성합니다.
* 추가 AWS 서비스를 활성화하거나 지원 수준을 업데이트하세요.
* 로그, 트레이스 및 기타 지표 수집 프로세스를 자동화합니다.
* 쓰기 권한 역할 또는 정책을 생성하세요.

## 구성된 정책 및 권한

자동화된 스택은 다음과 같은 액세스 수준을 구성합니다.

* **청구 데이터 접근 권한**: 비용 및 사용량 보고서 저장을 위해 생성된 특정 S3 버킷에 대한 읽기 전용 권한.
* **서비스 가격 액세스**: 사용량을 공개 요금과 비교하기 위해 AWS 가격 API에 대한 읽기 전용 액세스 권한이 필요합니다.
* **구성 자동화**: CUR 내보내기를 생성하고 설정 단계에서 통합 상태를 확인하는 데 필요한 권한이 제한적입니다.
* **선택적 최적화 액세스**: (선택 시) 비용 절감 권장 사항을 확인하기 위해 AWS Trusted Advisor에 대한 읽기 전용 액세스 권한이 제공됩니다.
