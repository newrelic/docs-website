필요한 AWS 리소스 및 설정을 구성하여 AWS 계정을 클라우드 분석 인텔리전스와 수동으로 통합하세요. 이 방법은 각 설정 단계를 완벽하게 제어할 수 있도록 해줍니다.

## 설치 지침

<Callout variant="important">
  For the following steps, sign in to the <DNT>**AWS Management Console**</DNT> and remain signed in throughout the process.
</Callout>

<Steps>
  <Step>
    ### 비용 및 사용량 보고서용 S3 버킷을 생성합니다. [#create-s3-bucket]

    AWS 비용 및 사용량 보고서(CUR)를 저장할 S3 버킷을 생성합니다. `us-east-2` 영역을 사용하시는 것을 권장합니다.

    1. Navigate to <DNT>**S3 service**</DNT> and click <DNT>**Create bucket**</DNT>.

    2. 고유한 버킷 이름을 입력하세요(예: `<your-company-name>-billing`).

    3. 버킷 설정 구성:

       * 버전 관리 활성화
       * 공개 액세스 비활성화(권장)
       * 기본 암호화 활성화(권장)

    4. <DNT>**Tags (Optional)**</DNT>: Add any tags if needed for your organization&apos;s resource management and click <DNT>**Create bucket**</DNT>.

    <Callout variant="important">
      다음 단계를 위해 버킷 이름을 기억해 두세요.
    </Callout>
  </Step>

  <Step>
    ### AWS 비용 및 사용량 보고서 구성 [#create-cur]

    Create a <DNT>**Cost and Usage Report**</DNT> to be delivered to your S3 bucket:

    1. Navigate to <DNT>**Billing Dashboard**</DNT> &gt; <DNT>**Cost &amp; Usage Analysis**</DNT> &gt; <DNT>**Data Exports**</DNT>, then click <DNT>**Create**</DNT>.

    2. 보고서 설정 구성:

    </Callout>

  <Step>
    ### CCI 액세스를 위한 IAM 정책을 생성합니다. [#create-iam-policy]

    CCI가 CUR 데이터에 접근할 수 있도록 권한을 부여하는 ID 및 액세스 관리(IAM) 정책을 생성하십시오.

    1. Navigate to <DNT>**IAM service**</DNT> &gt; <DNT>**Policies**</DNT> and click <DNT>**Create policy**</DNT>.
    2. Switch to the <DNT>**JSON editor**</DNT> and paste the following policy:

    ```json
    {
    "Version": "2012-10-17",
    "Statement": [
        {
        "Action": ["s3:Get*", "s3:List*"],
        "Effect": "Allow",
        "Resource": [
            "arn:aws:s3:::<bucket-name>/*",
            "arn:aws:s3:::<bucket-name>"
        ],
        "Sid": "AccessMasterPayerBillingBucket"
        },
        {
        "Effect": "Allow",
        "Action": [
            "pricing:DescribeServices",
            "pricing:GetAttributeValues",
            "pricing:GetProducts",
            "pricing:GetPriceListFileUrl",
            "pricing:ListPriceLists"
        ],
        "Resource": ["*"]
        }
    ]
    }
    ```

    <Callout variant="important">
      [비용 및 사용량 보고서용 S3 버킷 생성](#create-s3-bucket) 에서 얻은 S3 버킷 이름으로 `bucket-name` 바꾸세요.
    </Callout>

    3. <DNT>**Next**</DNT> 을(를) 클릭합니다.
    4. Name the policy `CCI-Access-Policy` (or your preferred name), add a description, and click <DNT>**Create policy**</DNT>.
  </Step>

  <Step>
    ### CCI가 사용할 수 있는 IAM 역할을 생성합니다. [#create-iam-role]

    CCI가 CUR 데이터에 액세스하도록 하는 IAM 역할을 만듭니다.

    1. Navigate to <DNT>**IAM service**</DNT> &gt; <DNT>**Roles**</DNT> and click <DNT>**Create role**</DNT>.
    2. Select <DNT>**Custom Trust Policy**</DNT> in the trusted entity type section and paste the following role definition:
       ```json
       {
       "Version": "2012-10-17",
       "Statement": [
           {
           "Effect": "Allow",
           "Principal": {
               "AWS": "arn:aws:iam::207192125115:user/cci_global_user"
           },
           "Action": "sts:AssumeRole",
           "Condition": {
               "StringEquals": {
               "sts:ExternalId": "<externalID>"
               }
           }
           }
       ]
       }
       ```

    <Callout variant="important" title="메모">
      `externalId` 은 CCI 통합을 시작할 때 생성됩니다.
    </Callout>

    3. <DNT>**Next**</DNT> 을(를) 클릭합니다.
    4. Attach the policy created in [Create IAM policy for CCI access](#create-iam-policy) and click <DNT>**Next**</DNT>.
    5. Name the role `CCI-Access-Role` (or your preferred name), add a description (optional), and click <DNT>**Create role**</DNT>.
    6. Note the <DNT>**Role ARN (Amazon Resource Name)**</DNT> from the role summary page.

    <Callout variant="important">
      You&apos;ll need the <DNT>**Role ARN**</DNT> during the <DNT>**CCI integration**</DNT>.
    </Callout>
  </Step>

  <Step>
    ### AWS 계정을 수동으로 통합하세요 [#access-integration]

    1. <DNT>[**one.newrelic.com**](https://one.newrelic.com/) &gt; **Integrations &amp; Agents** &gt; **Cloud Cost Intelligence - AWS**</DNT> 으로 이동합니다.
    2. <DNT>**Manually integrate your AWS account**</DNT> 선택하고 <DNT>**Continue**</DNT> 클릭합니다.
  </Step>

  <Step>
    ### 연결 세부 정보 제공 [#provide-connection-details]

    필수 정보를 입력하세요:

    * <DNT>**Connection name**</DNT>: Default name provided. Edit for custom name.
    * <DNT>**Cross-account IAM role ARN**</DNT>: Enter the IAM role ARN (Amazon Resource Name) to allow Cloud Cost Intelligence to access your S3 bucket (format: `arn:aws:iam::<account-id>:role/<role-name>`).
    * <DNT>**External ID**</DNT>: Copy the external ID and add it to your IAM role trust policy in AWS.
    * <DNT>**Data export name**</DNT>: Enter the name of the data export file. For report `cci-ingestion-test`, the path is `cost-and-usage-reports/cci-ingestion-test/data`.
    * <DNT>**S3 bucket name**</DNT>: Enter the exact bucket name. Use `us-east-2` region for optimal performance.
  </Step>

  <Step>
    ### 신뢰 정책을 구성합니다. [#configure-trust-policy]

    IAM 역할 신뢰 정책에 뉴렐릭 전역 사용자(`arn:aws:iam::207192125115:user/cci_global_user`)를 추가합니다.

    ```json
    {
      "Version": "2012-10-17",
      "Statement": [
        {
          "Effect": "Allow",
          "Principal": {
            "AWS": "arn:aws:iam::207192125115:user/cci_global_user"
          },
          "Action": "sts:AssumeRole",
          "Condition": {
            "StringEquals": {
              "sts:ExternalId": "<externalID>"
            }
          }
        }
      ]
    }
    ```

    <Callout variant="important">
      [연결 세부 정보 제공](#provide-connection-details) 에서 외부 ID를 `sts:ExternalId` 으로 설정합니다. 기본적으로 활성화되지 않은 리전(예: `ap-east-1`, `eu-south-1`)에 대해 AWS 보안 토큰 서비스(STS)를 활성화합니다.
    </Callout>
  </Step>

  <Step>
    ### 연결 테스트 [#test-connection]

    Click <DNT>**Test connection**</DNT> to verify the connection to your AWS environment.

    <Callout variant="important">
      초기 데이터 처리는 설정 완료 후 몇 시간이 소요됩니다.
    </Callout>
  </Step>
</Steps>

### Kubernetes 비용 할당 [#k8s-cost-allocation]

* [Kubernetes 통합을 설치하세요](https://docs.newrelic.com/install/kubernetes/). AWS Fargate 통합은 지원되지 않습니다.
* Beyond standard <DNT>**Kubernetes labels (`app.kubernetes.io/name`, `app.kubernetes.io/instance`, `app.kubernetes.io/component`, and `app.kubernetes.io/part-of`)**</DNT>, add custom labels such as <DNT>**`environment`**</DNT> and <DNT>**`team`**</DNT>, and custom resource tags such as <DNT>**`project`**</DNT> and <DNT>**`costCenter`**</DNT> for better cost segmentation and analysis.

## 권장 사항

* AWS 계정에서 [비용 할당 태그를](https://docs.aws.amazon.com/tag-editor/latest/userguide/tagging.html) 구성하세요. 효과적인 비용 필터링 및 그룹화를 위해 AWS 리소스 전반에 걸쳐 최소 두 개 이상의 사용자 지정 리소스 태그를 일관되게 적용하십시오.
* 최적의 성능과 비용 효율성을 위해 `us-east-2` 영역의 S3 버킷을 사용하십시오.

## 접근 제어 권고

지정된 뉴렐릭 계정/조직에 액세스할 수 있는 모든 사용자는 AWS 클라우드 비용 데이터를 볼 수 있습니다. 내부 접근 제어 정책에 맞는 적절한 뉴렐릭 계정/조직을 선택하십시오.

<Callout variant="important">
  <DNT>**S3 bucket access**</DNT>

  뉴렐릭은 S3 버킷에 대한 읽기 권한만 필요하며, 이는 단일 IAM 역할로 제한됩니다. 데이터를 보호하려면 이 접근 수준을 유지하세요.
</Callout>
