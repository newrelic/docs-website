<Callout title="시사">
  이 기능은 아직 개발 중이지만 꼭 사용해 보시기 바랍니다!

  이 기능은 현재 [출시 전 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy) 에 따라 미리보기 프로그램의 일부로 제공됩니다.
</Callout>

Azure 청구서를 클라우드 비용 인텔리전스에 연결하여 cloud 지출을 모니터링하고 최적화하세요. 서비스 및 자원 그룹 전반에 걸친 지출 패턴, 자원 활용 현황 및 비용 배분 내역을 확인하세요.

주요 기능:

* **자원 최적화** - 활용도가 낮거나 비용이 많이 드는 서비스를 파악하고 할당량을 조정합니다.
* **비용 제어** - 컴퓨트, 스토리지, 네트워킹 및 데이터베이스 전반의 비용 동인을 추적합니다.
* **다중 구독 모니터링** - 구독 및 관리 그룹 전반의 지출 보기
* **비용 책임성** - Azure 태그 및 리소스 그룹을 사용하여 비용 청구 활성화
* **예산 관리** - 지출 금액, 제한 및 한도에 대한 알림 설정

## 시작하기 전에

필요한 것:

* 활성 결제 계정이 있는 Azure 구독
* 구독에 대한 기여자 또는 소유자 역할
* Azure 비용 관리 액세스
* 구독 또는 관리 그룹에 대한 청구서 읽기 권한

## 설치 지침

<Callout variant="important">
  **이미 FOCUS 내보내기 파일이 있으신가요?** 기존에 **Cost and Usage (FOCUS)** \[비용 및 사용량(FOCUS)] 형식, 일별 빈도, Parquet 압축 방식의 청구서 내보내기 파일이 있는 경우 해당 파일을 사용할 수 있습니다. 그렇지 않으면 [기존 내보내기 확인](#check-existing-export), [청구 내보내기 생성](#create-billing-export) 및 [저장소 설정 구성](#configure-storage-settings) 을 따라 새 내보내기를 생성하십시오.
</Callout>

    ### 기존 내보내기를 확인하세요 [#check-existing-export]

    필요한 형식의 청구서 내보내기 파일이 이미 있는지 확인하십시오.

    1. Azure 포털에서 **Cost Management &amp; Billing** \[비용 관리 및 청구] 로 이동합니다.
    2. **Billing Scope** \[청구 범위] 를 선택한 다음 **Settings** \[설정을] 확장하고 **Exports** \[내보내기]를 클릭합니다.
    3. **Cost and Usage (FOCUS)** 형식과 **Parquet** 압축을 사용한 내보내기 목록을 검토하십시오.

    * **일치하는 내보내기 파일이 없는 경우**: [청구 내보내기 생성을](#create-billing-export) 계속 진행하십시오.

  <Step>
    ### 접근 권한 부여 [#authorize-access]

    뉴렐릭 서비스 주체(**msazurecostdata**)가 청구 데이터에 액세스할 수 있도록 승인하십시오.

    1. **Initiate Connection** \[연결 시작]을 클릭하세요.
    2. **Microsoft Authentication** 절차를 완료하세요.

    <Callout variant="important">
      Azure 관리자 승인이 필요합니다.
    </Callout>

    3. 인증 요청을 수락합니다.
    4. 작은 파일(0KB)이 다운로드됩니다. 이 파일을 삭제하셔도 됩니다.
  </Step>

  <Step>
    ### 서비스 주체를 확인하세요 [#confirm-service-principal]

    서비스 주체가 추가되었는지 확인하십시오.

    1. **Microsoft Entra Account** 의 **Enterprise Applications** \[엔터프라이즈 애플리케이션] 으로 이동하세요.
    2. **msazurecostdata**가 목록에 나타나는지 확인하십시오.

    <Callout variant="important">
      엔터프라이즈 애플리케이션을 보는 데 도움이 필요하면 [Microsoft Entra 설명서를](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/view-applications-portal) 참조하세요.
    </Callout>
  </Step>

  <Step>
    ### 권한 할당 [#assign-permissions]

    읽기 전용 액세스를 위해 **Storage Blob Data Reader** 역할을 할당합니다.

    1. 스토리지 계정 컨테이너를 열고 **Access Control (IAM)** \[액세스 제어(IAM)]를 선택합니다.
    2. **Add** &gt; **Add role assignment** 클릭합니다.
    3. **Storage Blob Data Reade**r \[스토리지 Blob 데이터 판독기]를 선택하십시오.
    4. `msazurecostdata` 검색하여 선택하세요.
    5. **Review + Assign** \[검토 + 할당을] 클릭하고 역할 할당에 `msazurecostdata` 표시되는지 확인하세요.
  </Step>

  <Step>
    ### 설정 확인 [#verify-setup]

    확인하다:

    * **msazurecostdata는** **Enterprise Applications** \[엔터프라이즈 애플리케이션]에 나타납니다.

    * 서비스 주체는 **Storage Blob Data Reader** \[스토리지 Blob 데이터 읽기] 권한을 가지고 있습니다.

    * 역할은 올바른 저장 컨테이너에 적용됩니다.

    * 뉴렐릭에서 연결 테스트가 성공했습니다.

      <Callout variant="important">
        **보안**: 서비스 주체는 청구 데이터에 대한 읽기 전용 액세스 권한만 갖습니다. 데이터를 쓰거나, 수정하거나, 삭제할 수 없습니다.
      </Callout>
  </Step>

  </Step>

  </Step>
