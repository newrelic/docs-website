클라우드 컴퓨팅 인텔리전스는 cloud 비용에 대한 가시성과 관리를 제공하도록 설계되었습니다. 클라우드 컴퓨팅 인텔리전스 AI 모니터링은 비용 가시성과 어셈블리 옵저버빌리티 간의 격차를 해소하여 cloud 지출에 대한 전체적인 보기를 제공합니다.

예를 들어, 대기업이 클라우드 서비스 사용을 빠르게 확장함에 따라 cloud 비용을 추적하고 관리하는 일이 점점 더 복잡해질 수 있습니다. 클라우드 비용 인텔리전스는 귀하의 자금이 어디에 할당되는지 인사이트를 제공하여 지원합니다. 이를 통해 잠재적으로 비용을 절감할 수 있는 분야를 파악하는 데 도움이 될 수 있습니다. cloud 지출을 실제 사용량에 맞게 조정하면 전략적 성장 이니셔티브를 계속 지원하는 동시에 예산을 최적화할 수 있습니다. 이러한 접근 방식은 귀하가 투자한 모든 달러가 회사의 전반적인 성공에 기여하도록 보장하는 데 도움이 됩니다.

<Callout variant="important">
  현재는 의 cloud 비용만 AWS 지원됩니다.
</Callout>

## 주요 특징들

### 포괄적인 비용 가시성

* **범주별 분석:** 애플리케이션, 서비스, 소유 팀, 지역별로 세분화된 cloud 비용을 확인하세요.
* **추세 분석:** 월별 비용 추세, 평균 비용, 주요 비용 요인을 파악하세요.

### Kubernetes 비용 할당

Kubernetes 관련 비용에 대한 가시성을 확보하세요. 우리는 뉴렐릭의 텔레메트리 데이터를 활용하여 제작 이름별로 비용을 분류합니다.

### 실시간 비용 추정

클라우드 비용 인텔리전스는 텔레메트리 데이터와 과거 비용 패턴을 활용하여 리소스 프로비저닝 후 1시간 이내에 변경 사항을 반영하는 시간별 비용 업데이트를 제공합니다. 청구 데이터와 통합하면 지난주 평균 단가와 최근 48시간 동안의 뉴렐릭 텔레메트리 데이터를 기반으로 비용을 추정할 수 있습니다. 이러한 통합이 없으면 추정치는 잠재적으로 더 높은 표준 단위 가격으로 기본 설정됩니다.

가장 정확한 추정을 위해서는 텔레메트리를 통합하고 최소 7일간의 이력 데이터를 유지하세요. 이 데이터는 동일한 인스턴스에 대한 비용을 계산하는 데 도움이 됩니다. 이러한 데이터가 없는 경우 추정치는 cloud 공급업체가 사용하는 표준적인 사용량 기반 지불 모델인 주문형 가격 책정으로 회귀합니다.

<Callout variant="important">
  **교차 계정 데이터 수집**<br /> 클라우드 비용 인텔리전스의 범위는 사용자이며 뉴렐릭 사용자 자격 증명과 관련된 모든 계정에서 스프레드시트 데이터를 수집합니다. 즉 클라우드 인텔 뉴리전스에 대해 단일 뉴렐릭 계정을 사용하더라도 귀하가 액세스할 수 있는 추가 뉴렐릭 계정의 텔레메트리 데이터도 해당 단일 뉴렐릭 계정 내에서 수집되고 표시됩니다. 사용자 계정 권한이 내부 정책에 부합하는지 확인하고, 보고 싶은 뉴렐릭 계정에 대한 데이터 액세스를 허용하세요.
</Callout>

## 클라우드 솔루션 인텔리전스가 스토리지 솔루션을 통해 cloud 비용을 회수하는 방법 [#fetch-costs]

클라우드 비용 인텔리전스는 설정 및 통합이 완료되면 지정된 S3 버킷에 있는 AWS 비용 및 사용 보고서(CUR)에 자동으로 액세스합니다. 이 과정은 귀하의 승인과 접근 권한 제공으로 시작됩니다. 클라우드 비용 인텔리전스는 데이터를 지정된 S3 버킷으로 내보낸 후에만 보고서를 가져올 수 있다는 점에 유의하는 것이 중요합니다. [필수 구성 요소](/docs/cci/getting-started/prerequisites) 에 지정된 대로 S3 경로 접두사가 cost-and-usage-reports로 설정되어 있는지 확인하세요.

일반적으로 AWS CUR을 하루에 세 번 업데이트하고, 클라우드는 설정 후 1시간에 한 번씩 S3 버킷에서 CUR을 검색합니다. 클라우드 비용 인텔리전스는 AWS 업데이트를 제공하고 그 직후 클라우드 비용 인텔리전스가 이를 검색하면 귀하의 최신 CUR이 표시됩니다.

<Callout variant="important">
  **귀하의 데이터 개인정보 보호**<br /> CUR은 클라우드 서비스 사용 및 비용에 대한 데이터로, 개인 데이터는 포함되어서는 안 됩니다. CUR을 S3 버킷에 추가하고 뉴렐릭에 업로드하기 전에 전송하는 데이터에 개인 식별 정보나 민감한 정보가 없는지 확인하세요. 현재 CUR 데이터 삭제는 지원되지 않습니다. 개인 식별 정보나 민감한 정보를 사용하여 CUR을 생성하지 않는 것이 좋습니다.

  **데이터 수집:** 당사의 현재 데이터 삭제 정책은 데이터를 최대 5년 동안 유지하는 것입니다. 2년 후, 데이터는 비용이 저렴하고 검색 시간이 긴 저장 옵션인 AWS Glacier로 이동됩니다.

  **액세스 보기:** 설치 중에 귀하가 선택한 뉴렐릭 계정에 액세스할 수 있는 사람은 누구나 해양 인텔리전스 내의 모든 데이터를 볼 수 있다는 점에 유의하십시오. RBAC 및 액세스 정책 요구 사항에 맞는 적절한 액세스 권한이 있는 계정을 선택하세요.
</Callout>

### 데이터 분석 및 보고

클라우드 인텔리전스는 CUR 데이터를 검색한 후 이 정보를 처리 및 분석하여 대시보드에 실행 가능한 인사이트를 제시합니다. cloud 사용에 따른 재정적 영향을 명확하게 시각화하여 비용 요인과 최적화 가능한 영역을 파악할 수 있습니다.
