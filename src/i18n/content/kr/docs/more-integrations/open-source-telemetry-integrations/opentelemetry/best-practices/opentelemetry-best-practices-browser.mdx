<Callout variant="caution">
  원격 분석을 브라우저 애플리케이션에서 New Relic의 OTLP 끝점으로 직접 보내지 않는 것이 좋습니다. 이렇게 하면 라이선스 키가 노출될 수 있기 때문입니다.
</Callout>

OpenTelemetry로 실제 사용자 모니터링(RUM)을 활성화하기 위해 브라우저, 모바일 및 기타 클라이언트 측 응용 프로그램을 계측하는 지원은 현재 제한적이며 활발히 개발 중입니다. OpenTelemetry API 또는 OpenTelemetry JS를 사용하여 브라우저에서 실행되는 애플리케이션을 계측할 수 있지만, 쉽게 얻을 수 있는 New Relic 라이센스 키 또는 기타 민감한 데이터를 포함하는 것은 결코 안전하지 않습니다.

노출된 키와 관련된 위험을 완화하기 위해 다음 사례를 계층화할 수 있습니다.

* 애플리케이션과 New Relic의 OTLP 엔드포인트 간에 OpenTelemetry Collector를 실행하여 클라이언트 브라우저에 키가 노출되는 것을 방지하세요.
* 악의적으로 사용되는 경우 쉽게 취소될 수 있는 브라우저 애플리케이션에서 원격 측정 데이터를 수집하기 위한 전용 라이선스 키를 생성합니다.
* 사용 제한 또는 청구 문제에 직면할 가능성을 포함하여 노출된 키와 관련된 많은 위험을 이해합니다.
* 노출된 키의 잠재적인 부정적인 영향을 제한하기 위해 OTLP를 통해 전송된 브라우저 원격 측정 데이터에 대해 별도의 격리된 계정을 사용합니다.
