경고는 원격 측정이 특정 임계값을 초과하는 경우 자신과 다른 사람에게 알리는 데 사용할 수 있는 도구입니다. 임계값은 생성한 조건에 의해 정의되며 이러한 조건을 정책으로 그룹화할 수 있습니다.

귀하의 역할에 따라 조건을 사용하여 귀하와 귀하의 팀이 통지를 받는 위반을 정의하거나 귀하가 해당 위반에 대응하고 있으며 위반을 일으킨 조건을 알고 싶어할 수 있습니다.

## 경고 조건 만들기 [#create-alerts]

조건은 NRQL 쿼리를 사용하여 알림을 받을 대상을 정의합니다. 조건 UI를 사용하여 위반 및 해당 알림을 생성하는 특정 임계값을 설정합니다. NRQL 쿼리를 처음부터 작성하는 것은 어려울 수 있지만 New Relic 차트에서 조건을 생성하는 경우 차트를 생성하는 데 사용된 NRQL로 조건을 채웁니다. 그런 다음 원하는 경우 쿼리를 수정한 다음 조건 임계값을 설정할 수 있습니다.

이를 시도하는 방법은 다음과 같습니다.

1. 차트의 오른쪽 상단에서 줄임표 버튼('...')을 클릭한 **다음 경고 조건 만들기** 를 클릭합니다.

   <img
     src={opentelemetryCreateAlertCondition}
     title="Create alert condition"
     alt="Create alert condition"
   />

   * 차트에 단일 메트릭이 표시되면 슬라이드 아웃 창을 사용하여 NRQL 쿼리가 미리 채워진 상태로 새 경고 조건을 생성합니다.
   * 차트에 여러 쿼리가 표시되는 경우 사용할 쿼리를 선택하라는 메시지가 표시됩니다.

2. 조건 생성 양식에서 기존 정책을 선택하거나 새 정책을 생성할 수 있습니다. 정책은 조건이 경고를 트리거할 때 알림을 받는 방법을 결정합니다.

경고 생성에 대한 추가 도움말은 [첫 번째 경고 생성](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition) 을 참조하십시오.

## 현재 알림 보기 [#see-alerts-list]

위반 사항에 대응할 책임이 있는 경우 각 서비스에 대해 어떤 경고가 설정되어 있는지 알고 싶을 수도 있습니다. UI에서 모든 조건 및 관련 정책 목록을 빠르게 가져올 수 있습니다.

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 에서 **모든 엔티티** 를 클릭합니다.

2. 왼쪽 탐색 창에서 **서비스 - OpenTelemetry** 를 선택합니다.

3. 조사할 서비스 이름을 클릭합니다.

4. **Settings**아래의 왼쪽 창에서 **Alert conditions**클릭합니다.

   <img src={opentelemetryAlertsConditionList} title="Screenshot showing the Alert conditions page for Opentelemetry" alt="Screenshot showing the Alert conditions page for Opentelemetry"/>

**경고 조건** 페이지에서 경고에 대한 세부 정보를 볼 수 있을 뿐만 아니라 경고를 추가할 수도 있습니다. 임계값이 임계값에 가까워지면 알림을 받을 수 있도록 이 작업을 수행할 수 있습니다. **임계값** 열에서 **경고 생성** 을 클릭하면 경고 임계값을 추가할 수 있는 경고 조건 편집기로 직접 이동합니다.

경고 생성에 대한 추가 도움말은 [첫 번째 경고 생성](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition) 을 참조하십시오.
