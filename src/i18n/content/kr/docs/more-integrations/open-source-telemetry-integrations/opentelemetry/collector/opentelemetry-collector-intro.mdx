## 언제 수집기를 사용합니까? [#when]

대부분의 경우 수집기는 선택적 구성 요소이지만 다양한 이유로 필요할 수 있습니다.

* 예를 들어 앱을 호스팅하는 서버에서 CPU 사용량을 캡처하는 등 인프라에 대한 메트릭을 수집합니다.
* 다양한 형식(Jaeger, Prometheus, Fluent Bit 등)으로 신호를 수집합니다.
* 여러 관측 가능성 백엔드로 데이터를 보냅니다.
* 관측 가능성 백엔드로 내보내기 전에 측정항목을 변환하거나 스팬의 이름을 변경하세요.
* 리소스 집약적 서비스를 실행할 때 네트워크를 통해 데이터를 보내는 작업을 오프로드합니다.
* 추가 메타데이터로 원격 분석(메트릭, 추적 및 로그)을 강화합니다.
* OpenTelemetry 계측 앱을 Kubernetes 메타데이터에 연결합니다.
