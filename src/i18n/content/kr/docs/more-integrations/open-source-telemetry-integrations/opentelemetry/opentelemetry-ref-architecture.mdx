New Relic을 사용하여 OpenTelemetry를 구현하는 지침이 필요한 경우 이 참조 아키텍처는 소프트웨어 개발자, DevOps, 엔터프라이즈 설계자 및 기술 관리자가 프로젝트에서 협업하는 데 도움이 될 것입니다. New Relic으로 OpenTelemetry를 구현하는 일반적인 방법을 조감도로 보여주는 템플릿입니다.

다이어그램은 OpenTelemetry, Prometheus, Jaeger 및 New Relic에서 원격 분석을 수집하는 구현을 보여줍니다. <InlinePopover type="apm"/>대리인. 세 가지 서비스(A, B 및 C)는 측정항목, 추적 및 로그를 전송하도록 구성됩니다.

OpenTelemetry에서 수집한 데이터의 경우 OTLP 엔드포인트를 통해 New Relic으로 직접 보내거나 선택적 OpenTelemetry Collector로 먼저 보낼 수 있습니다. 수집기를 사용하는 경우 원격 분석이 백엔드(데이터 분석용 Kafka 및 종단 간 관찰 가능성용 New Relic)로 전송되기 전에 데이터 파이프라인을 집계하고 향상시킵니다.
