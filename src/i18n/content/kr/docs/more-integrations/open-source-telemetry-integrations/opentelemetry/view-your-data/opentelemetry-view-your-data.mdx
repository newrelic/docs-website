## OpenTelemetry 데이터에 대한 규칙 [#conventions]

OpenTelemetry 프로젝트는 관측 가능성 도구에서 원격 측정을 일관되게 해석할 수 있도록 [메트릭](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/metrics/semantic_conventions) , [추적](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/trace/semantic_conventions) , [로그](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/logs/semantic_conventions) 및 [이벤트](https://github.com/open-telemetry/opentelemetry-specification/blob/main/specification/logs/semantic_conventions/events.md) 에 대한 의미론적 규칙을 지정합니다. New Relic은 이러한 규칙을 사용하여 UI에 원격 측정 데이터를 표시합니다.

OpenTelemetry 자동 계측 에이전트 및 SDK는 일반적으로 시맨틱 규칙을 따르지만 프로젝트가 빠르게 개발 중이므로 규칙에 맞게 계측을 조정해야 할 수도 있습니다.

데이터가 UI에서 예상한 대로 표시되지 않으면 관련 UI 페이지의 설명서를 참조하세요. 여기에서 해당 페이지에서 사용된 의미 체계에 대한 참조를 찾을 수 있습니다.
