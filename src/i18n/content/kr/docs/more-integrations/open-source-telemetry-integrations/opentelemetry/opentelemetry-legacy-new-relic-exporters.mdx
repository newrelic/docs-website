New Relic은 New Relic의 독점 프로토콜을 통해 OpenTelemetry 데이터를 보내는 데 사용되는 여러 내보내기 도구를 게시했습니다. 이러한 내보내기는 OpenTelemetry 프로토콜을 사용하여 New Relic에 데이터를 보내기 위해 더 이상 사용되지 않습니다.

<Callout variant="important">
  수출업체는 참조용으로 여기에 나열됩니다. 이러한 수출업체에서 마이그레이션하려면 [빠른 시작 가이드](/docs/integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-quick-start) 를 따르십시오.
</Callout>

## New Relic 언어 내보내기 [#newrelic-language-exporters]

위에서 언급했듯이 언어별 내보내기는 더 이상 사용되지 않습니다. 이러한 내보내기를 통해 애플리케이션은 OpenTelemetry 데이터를 New Relic으로 직접 내보낼 수 있습니다. [New Relic OTLP 끝점](#native-otlp) 은 응용 프로그램이 New Relic의 독점 끝점으로 데이터를 내보낼 수 있도록 하기 때문에 이 전략을 구식으로 만듭니다.

수출업체 및 유지보수 현황은 다음과 같습니다.

* [이동](https://github.com/newrelic/opentelemetry-exporter-go) : 보관됨
* [자바](https://github.com/newrelic/opentelemetry-exporter-java) : 아카이브
* [파이썬](https://github.com/newrelic/opentelemetry-exporter-python) : 보관됨
* [.NET](https://github.com/newrelic/newrelic-telemetry-sdk-dotnet/tree/main/src/NewRelic.OpenTelemetry) : 아카이브
* [수집기](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/v0.50.0/exporter/newrelicexporter) : 2022년 5월 10일부로 제거되었습니다.

## New Relic 내보내기를 위한 엔드포인트 구성 [#h2-change-endpoints]

데이터를 보내는 New Relic 엔드포인트를 변경할 수 있습니다.

### 미국 엔드포인트 [#default-endpoints]

기본적으로 New Relic OpenTelemetry 내보내기는 다음 미국 데이터 센터로 데이터를 보냅니다.

* 스팬: [https://trace-api.newrelic.com/trace/v1](https://trace-api.newrelic.com/trace/v1)
* 메트릭: [https://metric-api.newrelic.com/metric/v1](https://metric-api.newrelic.com/metric/v1)
* 로그: [https://log-api.newrelic.com/log/v1](https://log-api.newrelic.com/log/v1)

EU 지역으로 데이터를 보내거나 무한 추적을 사용하려면 이러한 기본 끝점을 재정의해야 할 수 있습니다.

### EU 지역 엔드포인트 [#eu-region]

EU 지역에 있는 New Relic의 엔드포인트에 원격 측정 데이터를 보내려면 다음을 사용하세요.

<Callout variant="tip">
  이러한 URL은 무한 추적에 적용되지 않습니다.
</Callout>

* 스팬: [https://trace-api.eu.newrelic.com/trace/v1](https://trace-api.eu.newrelic.com/trace/v1)
* 메트릭: [https://metric-api.eu.newrelic.com/metric/v1](https://metric-api.eu.newrelic.com/metric/v1)
* 로그: [https://log-api.eu.newrelic.com/log/v1](https://log-api.eu.newrelic.com/log/v1)
