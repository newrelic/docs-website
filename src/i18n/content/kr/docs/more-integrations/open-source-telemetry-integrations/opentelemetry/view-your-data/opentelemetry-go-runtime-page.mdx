OpenTelemetry 데이터를 전송하고 UI에서 서비스(엔터티)를 연 후 **런타임 이동을** 클릭하여 언어 런타임과 관련된 비정상적이거나 비정상적인 성능 패턴이 있는 서비스 인스턴스를 식별할 수 있습니다.

<Callout variant="important">
  이 보기에서 사용되는 런타임 메트릭을 수집하려면 별도의 계측 패키지를 설치해야 합니다. 아래의 [런타임 측정항목 세부정보를](#metric-details) 참조하세요.
</Callout>

응답 시간, 처리량, 오류율, 가비지 수집 시간 및 메모리 사용량과 같은 주요 메트릭 요약을 기반으로 비교할 여러 서비스 인스턴스를 선택할 수 있습니다. 그런 다음 시계열 차트를 사용하여 모든 인스턴스의 지표를 비교하여 문제를 파악하고 가비지 수집, 고루틴 또는 기타 런타임 리소스가 관련되어 있는지 확인할 수 있습니다.

다음은 일반적인 워크플로입니다.

1. **Go 런타임 을** 클릭합니다.

2. 요약된 상태 지표 표를 사용하여 흥미로운 Go 런타임 인스턴스를 찾으십시오.

   * 필터 표시줄을 사용하여 검색 범위를 좁힙니다.
   * 예를 들어 가비지 수집 시간이 더 긴 인스턴스와 같은 이상값을 찾기 위해 정렬합니다.

3. 흥미로운 Go 런타임 인스턴스를 선택합니다.

4. **비교를** 클릭하여 인스턴스별로 분류된 상태 및 런타임 지표의 표시를 확인합니다.

<CollapserGroup>
  <Collapser

    title="런타임 측정항목 세부정보"
  >
    특정 인스턴스로 드릴다운하거나 여러 인스턴스를 비교할 때 UI는 Go 런타임 메트릭 데이터로 구동되는 차트를 표시합니다. [여기에서](https://pkg.go.dev/go.opentelemetry.io/contrib/instrumentation/runtime) Go 런타임 측정항목을 생성하는 도구를 찾을 수 있습니다.
  </Collapser>

  <Collapser
    className="freq-link"
    id="metrics-or-spans"
    title="측정항목 또는 범위"
  >
    표와 차트에서 응답 시간, 처리량, 오류율과 같은 황금 신호(상태 지표라고도 함)를 강화하기 위해 [지표 또는 범위를 사용하도록 선택할](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-find-entities/#metric-span-toggle) 수 있습니다.

    가비지 수집, 메모리 사용량 및 고루틴과 같은 런타임 메트릭은 스팬이 아닌 메트릭 데이터에서만 사용할 수 있습니다. 따라서 이 토글 스위치의 영향을 받지 않습니다.
  </Collapser>

  <Collapser
    className="freq-link"
    id="config-steps"
    title="데이터가 표시되는지 확인하는 방법"
  >
    데이터가 이 섹션에 표시되려면 Go 런타임 인스턴스 목록을 렌더링하기 위한 `service.instance.id` 리소스 속성이 있는지 확인하세요.
  </Collapser>
</CollapserGroup>
