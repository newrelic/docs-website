속성에 대한 New Relic의 제한은 OTLP 소스 데이터를 포함한 모든 소스의 데이터에 적용됩니다. 다른 제한 사항은 [지표 속성 제한](/docs/data-apis/ingest-apis/metric-api/metric-api-limits-restricted-attributes/) 및 [이벤트 속성 제한을](/docs/data-apis/ingest-apis/event-api/introduction-event-api/#limits) 참조하세요.

* 속성 이름의 최대 길이: 255자
* 속성 값의 최대 길이: 4095자
* 속성 배열 값의 최대 크기: 64개 항목

범위 속성이 제한을 초과하는 경우 [범위 제한 환경 변수](https://github.com/open-telemetry/opentelemetry-specification/blob/d6bcc0cb072d8d6f6ced856f1f23c451648a3caa/specification/sdk-environment-variables.md#span-limits-) 를 사용하여 최대 길이를 구성할 수 있습니다. [리소스 속성](https://github.com/open-telemetry/opentelemetry-specification/blob/d6bcc0cb072d8d6f6ced856f1f23c451648a3caa/specification/sdk-environment-variables.md#general-sdk-configuration) 이 위반자인 경우 `OTEL_RESOURCE_ATTRIBUTES=<offending-attribute>=unset` 를 설정하여 이를 재정의할 수 있습니다.

모든 언어 SDK가 범위 제한 환경 변수를 지원하는 것은 아니므로 사용 중인 언어에 따라 다릅니다. 언어가 이를 지원하지 않는 경우 OpenTelemetry 커뮤니티에서 해결할 수 있도록 해당 언어 SDK 리포지토리에서 GitHub 문제를 여는 것이 좋습니다.
