요청당 전송할 수 있는 데이터 양을 최대화하려면 모든 OTLP 내보내기에서 압축을 활성화하는 것이 좋습니다. New Relic은 허용되는 최대 페이로드 크기가 1MB(10^6바이트)인 gRPC 또는 HTTP를 통해 전송되는 OTLP 페이로드에 대해 `gzip` 및 `zstd` 압축을 지원합니다.

## Otel 수집기 구성

OTel 수집기의 OTLP 내보내기에 대한 기본 압축은 현재 `gzip` 이며, `zstd` 압축이 더 효율적인 경우가 많습니다. 벤치마크 정보에 대한 자세한 내용은 [압축 비교를](https://github.com/open-telemetry/opentelemetry-collector/blob/main/config/configgrpc/README.md#compression-comparison) 참조하세요.

수집기에서 `zstd` 활성화하려면 수집기 내보내기 구성을 조정하세요.

```
exporters:
  otlphttp:
    ...
    compression: zstd

```

다른 압축 형식 지원을 확인하려면 [CNCF Slack 채널](https://cloud-native.slack.com/archives/C024DRQ63UP) 을 통해 알려주시기 바랍니다. [여기서 CNCF의 Slack 계정에 가입](https://slack.cncf.io/) 할 수 있습니다.
