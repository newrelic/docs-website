요청당 보낼 수 있는 데이터의 양을 최대화하려면 모든 OTLP 내보내기에서 압축을 활성화하는 것이 좋습니다. New Relic은 gRPC 또는 HTTP를 통해 전송되는 OTLP 페이로드에 대해 gzip 압축을 지원합니다. 최대 허용 페이로드 크기는 1MB(10^6바이트)입니다.

지원을 원하는 다른 압축 형식이 있으면 [CNCF Slack 채널](https://cloud-native.slack.com/archives/C024DRQ63UP) 에서 알려주십시오. [여기에서 CNCF의 Slack 계정에 가입](https://slack.cncf.io/) 할 수 있습니다.
