OpenTelemetry 및 New Relic을 시작하기 위한 다양한 옵션이 있습니다. 자체 앱, 호스트 또는 Kubernetes 클러스터의 모니터링으로 바로 이동하거나 일부 데모 앱을 설정하여 OpenTelemetry 및 New Relic을 실험할 수 있습니다. OpenTelemetry의 기능을 활용하여 데이터를 수집하고 New Relic으로 보내는 데 도움이 되는 다양한 지침이 있습니다.

## 나만의 앱 또는 서비스 설정 [#own-app]

OpenTelemetry에 익숙하고 고유한 코드로 작업할 준비가 된 경우 OpenTelemetry로 [고유한 앱 또는 서비스 설정 을](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/get-started/opentelemetry-set-up-your-app) 확인하십시오. 이 단계는 원격 측정을 수집하고 New Relic으로 내보내는 데 도움이 됩니다.

## 호스트 모니터링 [#otel-infra]

OpenTelemetry Collector를 사용하여 호스트를 모니터링할 수 있습니다. [호스트 모니터링을 위한 Collector를](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/collector/opentelemetry-collector-infra-hosts)참조하십시오.

## Kubernetes 모니터링 [#otel-kubernetes]

OpenTelemetry Collector로 Kubernetes 클러스터를 모니터링합니다. [Kubernetes에 OpenTelemetry 계측 애플리케이션 링크](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes/)의 지침을 참조하십시오.

## 데모 Java 앱 사용해 보기 [#demo-app-java]

자체 앱 설정으로 바로 이동하는 대신 OpenTelemetry 및 New Relic을 사용하여 데모 Java 앱을 스핀업할 수 있습니다.

OpenTelemetry에 익숙해지는 데 도움이 되는 세 가지 [Java 자습서가](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/get-started/opentelemetry-tutorial-java) 있습니다.

* New Relic으로 데이터를 전송하기 위해 빠르게 회전할 수 있는 사전 계측된 앱을 검사합니다.
* OpenTelemetry Java 에이전트를 사용하여 데모 앱을 자동으로 계측합니다.
* 데모 앱을 수동으로 계측하고 확장 프로그램을 사용하여 SDK를 구성합니다.

## 마스터 클래스에서 OpenTelemetry 개념 알아보기 [#otel-masterclass]

OpenTelemetry의 개념에 대해 자세히 알아보고 작은 데모 앱에 적용하려면 [마스터 클래스](https://developer.newrelic.com/opentelemetry-masterclass) 를 살펴보십시오. .NET 또는 Python에 대한 실습 자습서를 사용해 볼 수 있습니다.
