[Micrometer](https://micrometer.io/) 는 가장 널리 사용되는 모니터링 시스템에 대해 계측 클라이언트에 대한 간단한 외관을 제공하므로 공급업체에 종속되지 않고 JVM 기반 애플리케이션 코드를 계측할 수 있습니다.

New Relic의 Micrometer 메트릭 레지스트리는 Micrometer 원격 측정 데이터를 New Relic 계정으로 보냅니다. 하나가 없습니까? [무료로 가입하세요](https://newrelic.com/signup) . 신용 카드가 필요하지 않습니다.

Micrometer 통합을 위한 리소스:

* [읽어보기](https://github.com/newrelic/micrometer-registry-newrelic)
* [통합 사양](https://github.com/newrelic/newrelic-exporter-specs/tree/master/micrometer)

## 데이터 찾기

데이터를 찾으려면 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > All entities** 로 이동하십시오. 이름으로 서비스를 검색하십시오. Micrometer Spring Boot 계측에서 제공하는 가장 유용한 메트릭 중 일부에 대한 차트가 포함된 즉시 사용 가능한 개요를 제공합니다.

기본 개요를 시작점으로 사용하여 고유한 대시보드를 구축할 수 있습니다. 차트에서 `...` 아이콘을 클릭하여 해당 차트를 대시보드에 추가하거나 차트 뒤의 쿼리를 확인하여 수정하고 대시보드에 추가합니다.

New Relic에 있는 데이터를 쿼리하는 방법에 대한 자세한 내용은 데이터 [쿼리](/docs/using-new-relic/data/understand-data/query-new-relic-data) 및 [NRQL 소개를](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) 참조하세요.
