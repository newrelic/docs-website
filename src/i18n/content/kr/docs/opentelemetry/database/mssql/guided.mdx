<Callout title="시사">
  이 기능은 아직 개발 중이지만 꼭 사용해 보시기 바랍니다!

  이 기능은 현재 [사전 출시 정책](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy/) 에 따라 미리보기의 일부로 제공됩니다.
</Callout>

뉴렐릭 플랫폼의 안내 설치를 사용하여 Microsoft SQL Server 모니터링을 위한 NRDOT Collector를 설치할 수 있으며, 이는 뉴렐릭의 UI를 통해 플랫폼 안내 설정을 제공합니다. 빠른 시작을 위해 권장되는 방법입니다.

<Steps>
  <Step>
    ### 전제 조건 [#prerequisites]

    환경이 [호환성 및 필수 조건](/docs/opentelemetry/database/mssql/compatibility)을 충족하는지 확인하세요.
  </Step>

  <Step>
    ### 안내 설치 [#guided]

    안내 설치를 사용하여 NRDOT Collector를 설치하려면:

    1. [뉴렐릭 계정](https://one.newrelic.com/)에 로그인하세요.

    2. <DNT>**left navigation pane &gt; + Integration &amp; Agents &gt; MSSQL (OpenTelemetry)**</DNT> 으)로 이동합니다.

    3. <DNT>Select an account</DNT> 화면에서 MSSQL 통합을 설치할 계정을 선택하고 <DNT>**Continue**</DNT>을(를) 클릭합니다.

    4. <DNT>Enter your credential</DNT> 화면에서 다음 옵션 중 하나를 선택한 다음 **Continue** \[계속을] 클릭합니다:

       * **기존 키 사용**: 이미 라이선스 키가 있는 경우, 라이선스 키를 제공하십시오. 자세한 내용은 [사용자 키를](/docs/apis/intro-apis/new-relic-api-keys/#user-key) 참조하세요.
       * **새 키 만들기**: 라이선스 키가 없으면 **Create a new key** \[새 키 만들기를] 클릭하여 키를 만듭니다.

    5. <DNT>Select platform</DNT> 화면에서 MSSQL 통합을 설치할 플랫폼을 선택하고 <DNT>**Continue**</DNT>을(를) 클릭합니다.

    6. 화면의 지시에 따라 선택한 플랫폼에 대한 통합을 설정하세요.

    7. 설치 후 <DNT>Test your integration</DNT> 화면에서 **Test connection** \[연결 테스트]을 클릭하여 NRDOT Collector가 SQL Server 인스턴스에 연결하고 뉴렐릭으로 데이터를 전송할 수 있는지 확인합니다.
  </Step>

  <Step>
    ### (선택 사항) APM-데이터베이스 상관 관계 설정 [#correlation]

    애플리케이션 성능과 데이터베이스 작업 간의 상관관계를 파악하려면 데이터베이스 서비스 식별을 설정할 수 있습니다. 이를 통해 특정 데이터베이스워크로드 기능을 생성하는 기능을 정확하게 확인할 수 있습니다. 자세한 내용은 뉴렐릭에서 APM- 데이터베이스 상관 관계를 가져오기 위한 [데이터베이스 서비스 식별 설정을](/docs/opentelemetry/database/database-service-identification/) 참조하세요.

    <Callout variant="important">
      APM 에서 데이터베이스 성능 데이터를 보려면 두 가지 모두 동일한 뉴렐릭 계정에 있어야 합니다. 만약 데이터가 서로 다른 뉴럴릭 계정에 있다면, 데이터를 보려면 두 계정 모두에 접근 권한이 있어야 합니다.
    </Callout>
  </Step>

  <Step>
    ### (선택 사항) 비밀 관리 구성 [#rds-secret-management]

    비밀 관리 기능을 사용하면 데이터베이스 자격 증명과 같은 중요한 정보를 안전하게 관리할 수 있습니다. 이렇게 하면 구성 파일에 민감한 데이터를 하드코딩하는 것을 방지하여 모니터링 설정의 보안을 강화할 수 있습니다. 자세한 내용은 [비밀 관리 부분](/docs/opentelemetry/database/database-service-identification/#secret-management) 을 참조하십시오.
  </Step>

  <Step>
    ### 데이터 찾기 및 사용 [#find]

    데이터가 수집되면 뉴렐릭의 UI 통해 포괄적인 SQL Server 데이터베이스 모니터링에 액세스할 수 있습니다.

    뉴렐릭에서 SQL Server 데이터베이스를 찾으려면 다음을 수행하세요.

    1. <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT> 으)로 이동합니다.
    2. **<DNT>Entity type</DNT>** 드롭다운 메뉴에서 <DNT>**MSSQL instance**</DNT>을(를) 선택한 다음 <DNT>**Apply**</DNT>을(를) 클릭합니다.
    3. 모임 목록에서 SQL Server 데이터베이스를 선택합니다.
  </Step>
</Steps>
