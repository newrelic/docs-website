AWS RDS for SQL Server 인스턴스를 모니터링하려면 Windows 환경에서 NRDOT 수집기를 사용하여 Microsoft SQL Server 모니터링을 설정합니다.

RDS SQL Server 보안 설정에 따라 세 가지 인증 방법 중 하나를 선택하여 NRDOT Collector를 RDS 인스턴스에 연결할 수 있습니다:

        <Step>
          ### NRDOT Collector설정하세요 [#setup-sql]

          RDS SQL Server 인스턴스에 연결할 수 있는 EC2 Windows 인스턴스 또는 로컬 Windows 머신에 NRDOT Collector를 설치합니다.

          <Callout variant="important">
            RDS 모니터링의 경우, 최적의 네트워크 연결과 더 낮은 지연시간을 위해 RDS 인스턴스와 동일한 VPC에 있는 EC2 인스턴스에 NRDOT Collector를 설치하세요.
          </Callout>

          1. PowerShell을 사용하여 NRDOT Collector를 다운로드하고 설치합니다:

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

          2. 보안 그룹이 포트 1433(또는 사용자 지정 포트)에서 NRDOT Collector 호스트로부터 RDS 인스턴스로의 인바운드 연결을 허용하는지 확인합니다.
        </Step>

        <Step>
          ### 데이터베이스 사용자 구성 [#configure-user-sql]

          관리자 자격 증명과 함께 SQL Server Management Studio(SSMS) 또는 sqlcmd를 사용하여 RDS SQL Server 인스턴스에 연결합니다.

          다음 스크립트를 실행하여 적절한 권한을 가진 `newrelic` 모니터링 사용자를 생성하세요:

          ```sql
          USE [master];
          GO
          CREATE LOGIN [newrelic] WITH PASSWORD = '<PASSWORD>';
          GO

          GRANT VIEW SERVER STATE TO [newrelic];
          GRANT VIEW ANY DEFINITION TO [newrelic];
          GRANT VIEW ANY DATABASE TO [newrelic];
          GO

          DECLARE @name SYSNAME;
          DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
          SELECT [name]
          FROM [master].[sys].[databases]
          WHERE [name] NOT IN ('master', 'msdb', 'model', 'rdsadmin', 'distribution')
          AND [state] = 0;
          OPEN db_cursor;
          FETCH NEXT FROM db_cursor INTO @name;
          WHILE @@FETCH_STATUS = 0
          BEGIN
            BEGIN TRY
              EXEC('USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''newrelic'')
              BEGIN
                CREATE USER [newrelic] FOR LOGIN [newrelic];
              END;
              GRANT VIEW DATABASE STATE TO [newrelic];');
            END TRY
            BEGIN CATCH
              PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
            END CATCH
            FETCH NEXT FROM db_cursor INTO @name;
          END
          CLOSE db_cursor;
          DEALLOCATE db_cursor;
          GO
          ```

          <Callout variant="tip">
            **RDS의 경우:** RDS 마스터 사용자 이름(또는 `rds_superuser` 역할이 있는 사용자)을 사용하여 이 명령을 실행합니다. 스크립트는 AWS에서 예약한 `rdsadmin` 데이터베이스를 자동으로 제외합니다.
          </Callout>
        </Step>

          ## NRDOT Collector구성 [#configure-sql]

          모니터링 요구 사항에 따라 설정 유형을 선택하십시오:

          <CollapserGroup>
            <Collapser id="standard" title="표준 설정">
              이 설정은 핵심 메트릭을 사용한 필수 데이터베이스 모니터링에 중점을 둡니다.

              <Callout variant="tip">
                **RDS 엔드포인트 형식:** RDS 엔드포인트는 `your-db-instance.xxxxxxxxxxxx.region.rds.amazonaws.com` 형식입니다(예: `mydb-instance.c9akciq32.us-east-1.rds.amazonaws.com`). AWS RDS 콘솔의 &quot;Connectivity &amp; security&quot;에서 이를 찾으세요.
              </Callout>

              1. **설정 파일을 만듭니다.** 파일 이름은 `mssql-config.yaml`이어야 합니다. PowerShell에서 관리자로 다음 명령을 실행하십시오:

                 ```powershell
                 New-Item -Path "C:\Program Files\nrdot-collector\mssql-config.yaml" -ItemType File
                 ```

              2. 이전 단계에서 생성한 **mssql-config.yaml 파일에 아래 설정 내용을 추가합니다**.

              ```yaml
              receivers:
                nrsqlserver:
                  collection_interval: 15s
                  username: <YOUR_USERNAME>
                  password: <YOUR_PASSWORD>
                  server: <YOUR_SERVER>
                  port: <YOUR_PORT>

                  # Enable comprehensive database metrics
                  metrics:
                    sqlserver.database.count:
                      enabled: true
                    sqlserver.database.io:
                      enabled: true
                    sqlserver.database.latency:
                      enabled: true
                    sqlserver.database.operations:
                      enabled: true
                    sqlserver.database.tempdb.space:
                      enabled: true
                    sqlserver.database.tempdb.version_store.size:
                      enabled: true
                    sqlserver.deadlock.rate:
                      enabled: true
                    sqlserver.os.wait.duration:
                      enabled: true
                    sqlserver.processes.blocked:
                      enabled: true
                    sqlserver.memory.grants.pending.count:
                      enabled: true
                    sqlserver.database.file.size:
                      enabled: true
                    sqlserver.memory.area:
                      enabled: true

                  # Enable top query and query sample log collection
                  events:
                    db.server.query_sample:
                      enabled: true
                    db.server.top_query:
                      enabled: true

                  # Top query collection configuration
                  top_query_collection:
                    lookback_time: 60s
                    max_query_sample_count: 1000
                    top_query_count: 250
                    collection_interval: 60s

                  collect_full_query_text: true
                  allowed_comment_keys:
                    - nr_service_guid

                  # Query sample collection configuration
                  query_sample_collection:
                    max_rows_per_query: 100

              processors:
                memory_limiter:
                  check_interval: ${env:NR_MEM_LIMITER_CHECK_INTERVAL:-1s}
                  limit_mib:       ${env:NR_MEM_LIMITER_LIMIT_MIB:-200}
                  spike_limit_mib: ${env:NR_MEM_LIMITER_SPIKE_MIB:-50}

                batch:

              exporters:
                otlphttp:
                  endpoint: <YOUR_OTLP_ENDPOINT>
                  headers:
                    api-key: <YOUR_LICENSE_KEY>
                  tls:
                    insecure: false
                  compression: gzip

              service:
                telemetry:
                  metrics:
                    level: none

                pipelines:
                  metrics:
                    receivers: [nrsqlserver]
                    processors: [memory_limiter, batch]
                    exporters: [otlphttp]

                  logs:
                    receivers: [nrsqlserver]
                    processors: [memory_limiter, batch]
                    exporters: [otlphttp]

              ```

          ### 구성 매개변수

          다음 표는 nrsqlserver 수신기의 주요 설정 파라미터를 설명합니다:

          | 매개변수                   | 설명                                                                                                                 |
          | ---------------------- | ------------------------------------------------------------------------------------------------------------------ |
          | `<YOUR_USERNAME>`      | 인증을 위해 데이터베이스 사용자 이름을 입력하세요.                                                                                       |
          | `<YOUR_PASSWORD>`      | 인증을 위해 데이터베이스 비밀번호를 입력하세요.                                                                                         |
          | `<YOUR_SERVER>`        | SQL Server 호스트명 또는 IP 주소를 입력하십시오.                                                                                  |
          | `<YOUR_PORT>`          | SQL Server 포트 번호를 입력합니다. 기본값은 1433입니다.                                                                             |
          | `<YOUR_OTLP_ENDPOINT>` | 뉴렐릭 OTLP 엔드포인트를 입력하세요. 자세한 내용은 [뉴렐릭 OTLP 엔드포인트 문서를](/docs/opentelemetry/best-practices/opentelemetry-otlp/) 참조하세요. |
          | `<YOUR_LICENSE_KEY>`   | 귀하의 블루 클러스터 키를 입력하세요.                                                                                              |
          | `collection_interval`  | 데이터베이스 메트릭을 수집하는 간격(초)입니다. 기본: 15s                                                                                 |

          ### Microsoft SQL Server 활성화 [#enable-sql-server-domain]

        <Step>
          ### READ 액세스 권한 부여 [#grant-permissions-domain]

          다음 명령문을 사용하여 모든 데이터베이스에서 도메인 사용자에게 READ 액세스 권한을 부여합니다:

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
                AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''DomainName\DomainUser'')
                BEGIN
                  CREATE USER [DomainName\DomainUser] FOR LOGIN [DomainName\DomainUser];
                END;
                GRANT VIEW DATABASE STATE TO [DomainName\DomainUser];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="tip">
            **RDS의 경우:** 스크립트는 AWS에서 예약한 `rdsadmin` 데이터베이스를 자동으로 제외합니다. `DomainName\DomainUser` 을(를) 실제 도메인 자격 증명으로 바꿔야 합니다.
          </Callout>

          ### NRDOT Collector 설정 [#setup-domain]

          동일한 도메인의 일부이며 RDS SQL Server 인스턴스에 연결할 수 있는 도메인에 가입된 EC2 Windows 인스턴스 또는 로컬 Windows 머신에 NRDOT Collector를 설치합니다.

             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue

             # Configure service to use domain account

             </Callout>

          3. 보안 그룹이 포트 1433(또는 사용자 지정 포트)에서 NRDOT Collector 호스트로부터 RDS 인스턴스로의 인바운드 연결을 허용하는지 확인합니다.

        <Step>
          ### READ 액세스 권한 부여 [#grant-read-access-rds-gmsa]

          다음 명령문을 사용하여 모든 데이터베이스에서 gMSA 계정에 READ 액세스 권한을 부여합니다:

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
              AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''<DOMAIN>\<gMSA_NAME>$'')
                BEGIN
                  CREATE USER [<DOMAIN>\<gMSA_NAME>$] FOR LOGIN [<DOMAIN>\<gMSA_NAME>$];
                END;
                GRANT VIEW DATABASE STATE TO [<DOMAIN>\<gMSA_NAME>$];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="important">
            `DomainName\gMSA` 을(를) 실제 도메인 이름 및 gMSA 계정 이름(예: `CORP\SQL-gMSA$`)으로 바꿉니다. 이 스크립트는 시스템 데이터베이스 및 RDS 전용 데이터베이스를 제외한 모든 사용자 데이터베이스에 gMSA 계정용 데이터베이스 사용자를 생성합니다.

          다음 표는 nrsqlserver 수신기의 주요 설정 파라미터를 설명합니다:

          | 매개변수                   | 설명                                                                                                                 |
          | ---------------------- | ------------------------------------------------------------------------------------------------------------------ |
          | `<YOUR_RDS_ENDPOINT>`  | RDS SQL Server 엔드포인트를 입력하세요                                                                                        |
          | `<YOUR_PORT>`          | SQL Server 포트 번호를 입력합니다. 기본값은 1433입니다.                                                                             |
          | `<YOUR_OTLP_ENDPOINT>` | 뉴렐릭 OTLP 엔드포인트를 입력하세요. 자세한 내용은 [뉴렐릭 OTLP 엔드포인트 문서를](/docs/opentelemetry/best-practices/opentelemetry-otlp/) 참조하세요. |

                max_concurrent_queries: 4  # default is 4 if omitted or set to 0

          ### Windows 서비스 레지스트리 업데이트 [#update-registry-gmsa]

          새 mssql-config.yaml 파일을 사용하도록 Windows 서비스 레지스트리를 업데이트합니다.

          **방법 1: PowerShell**

          관리자 권한으로 PowerShell에서 다음 명령을 실행합니다:

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **방법 2: 명령 프롬프트(대안)**

          PowerShell을 사용할 수 없는 경우 대신 명령 프롬프트를 사용할 수 있습니다:

          1. 시작 메뉴를 열고 다음을 입력합니다 `cmd`
          2. 명령 프롬프트를 마우스 오른쪽 버튼으로 클릭하고 **Run as administrator** \[관리자 권한으로 실행]을 선택합니다
          3. 검은색 CMD 창에 이 명령을 그대로 복사하여 붙여넣고 Enter를 누릅니다:

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
        </Step>

          3. 모임 목록에서 SQL Server 데이터베이스를 선택합니다.
