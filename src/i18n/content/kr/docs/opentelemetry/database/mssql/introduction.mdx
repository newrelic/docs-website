데이터베이스 메트릭, 쿼리 성능 및 시스템 상태에 대한 포괄적인 인사이트를 제공하는 뉴렐릭 Distribution of OpenTelemetry(NRDOT)를 사용하여 Microsoft SQL Server 데이터베이스 성능을 모니터링하세요.

이 NRDOT 기반 접근 방식은 모니터링을 위한 OpenTelemetry 데이터베이스 표준을 활용하여 기존 온호스트 통합(OHI)을 보완하고 기존 옵저버빌리티 그리드와 통합을 더 쉽게 만듭니다.

## SQL Server용 NRDOT를 설정하는 방법 [#configure]

<Steps>
  <Step>
    ### 전제 조건 [#prerequisites]

    환경이 [호환성 및 필수 조건](/docs/opentelemetry/database/mssql/compatibility)을 충족하는지 확인하세요.
  </Step>

  <Step>
    ### NRDOT Collector 설치 [#installation]

    인프라에 따라 설치 경로를 선택하세요:

    * **안내 설치**: 뉴렐릭의 UI를 통한 플랫폼 안내 설정(빠른 시작을 위해 권장됨)

    * **자체 호스팅 환경**: 자체 인프라에 수동 설치

      * **[Windows 자체 호스팅](/docs/opentelemetry/database/mssql/windows-hosted)**
      * **[Linux 자체 호스팅](/docs/opentelemetry/database/mssql/linux-hosted)**

    * **AWS RDS 환경**: 클라우드 관리형 SQL Server 모니터링

      * **[Windows RDS 모니터링](/docs/opentelemetry/database/mssql/windows-rds)**
      * **[Linux RDS 모니터링](/docs/opentelemetry/database/mssql/linux-rds)**
  </Step>

  <Step>
    ### 데이터 찾기 및 사용 [#find]

    데이터가 수집되면 뉴렐릭의 UI 통해 포괄적인 SQL Server 데이터베이스 모니터링에 액세스할 수 있습니다.

    뉴렐릭에서 SQL Server 데이터베이스를 찾으려면 다음을 수행하세요.

    1. <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT> 으)로 이동합니다.
    2. **<DNT>Entity type</DNT>** 드롭다운 메뉴에서 <DNT>**MSSQL instance**</DNT>을(를) 선택한 다음 <DNT>**Apply**</DNT>을(를) 클릭합니다.
    3. 모임 목록에서 SQL Server 데이터베이스를 선택합니다.
  </Step>
</Steps>

## 무엇 향후 계획

NRDOT을 사용하여 SQL Server 데이터베이스 모니터링을 설정한 후:

* [사용자 정의 대시보드를 만들어](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) 데이터베이스 지표를 시각화하세요.
* 중요한 데이터베이스 성능에 대한 [알림 설정](/docs/alerts/create-alert/create-alert-condition/alert-conditions/), 참조 값
* NRDOT 수집기가 수집한 [사용 가능한 메트릭 탐색](/docs/opentelemetry/database/mssql/metrics-reference)
* 모니터링 설정에 대한 [일반적인 문제 해결](/docs/opentelemetry/database/mssql/troubleshooting)
