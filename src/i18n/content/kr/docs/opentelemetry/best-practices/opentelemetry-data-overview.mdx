OpenTelemetry [OTLP를](https://opentelemetry.io/docs/specs/otlp/) 통해 뉴렐릭과 통합될 수 있는 풍부한 리소스 및 도구 생태계를 갖추고 있습니다. 이 문서에서는 뉴렐릭이 OTLP 데이터를 수신, 처리, 수집하는 방법을 설명합니다. 일반적인 통합 패턴에 대한 자세한 내용은 [OpenTelemetry 시작하기 및 뉴렐릭을](/docs/opentelemetry/get-started/opentelemetry-get-started-intro/) 참조하세요.

[뉴렐릭 OTLP 엔드포인트는](/docs/opentelemetry/best-practices/opentelemetry-otlp/) OpenTelemetry [문제 해결](/docs/opentelemetry/best-practices/opentelemetry-otlp-troubleshooting/) 정보와 함께 뉴렐릭과의 모든 통합과 관련된 OTLP 설정 요구 사항을 설명합니다.

OTLP는 트레이스, 메트릭, 로그 신호로 구분되며 모두 리소스의 개념을 포함합니다. 각 처리 및 수집 방법에 대한 자세한 내용은 다음 페이지를 참조하세요.

* [뉴렐릭의 OpenTelemetry 트레이스](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/)
* [뉴렐릭의 OpenTelemetry 지표](/docs/opentelemetry/best-practices/opentelemetry-best-practices-metrics/)
* [뉴렐릭의 OpenTelemetry 로그](/docs/opentelemetry/best-practices/opentelemetry-best-practices-logs/)
* [뉴렐릭의 OpenTelemetry 리소스](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/)

뉴렐릭 및 관리 도구의 OpenTelemetry 데이터 볼륨에 대한 자세한 내용은 [OpenTelemetry 데이터 수집 볼륨 관리를](/docs/opentelemetry/best-practices/opentelemetry-manage-data-ingest-volume/) 참조하세요.

모든 OTLP 데이터는 [뉴렐릭 플랫폼](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/) 에 수집됩니다. 여기에는 다음을 포함하여 문제를 식별하고 진단하는 데 도움이 되는 많은 강력한 범용 옵저버빌리티 도구가 포함되어 있습니다.

* [알림](/docs/alerts-applied-intelligence/overview/): 신호가 청년값을 초과하면 알림을 받습니다.
* [사용자 정의 쿼리](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/): 임시 데이터 분석 및 탐색을 위한 사용자 정의 NRQL 쿼리를 작성합니다.
* [대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/): 애플리케이션의 고유한 요구 사항을 반영하는 사용자 정의 데이터 시각화를 만듭니다.
