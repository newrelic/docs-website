OpenTelemetry [OTLP( OpenTelemetry Protocol)](https://opentelemetry.io/docs/specs/otlp/) 를 통해 뉴렐릭과 원활하게 통합될 수 있는 측정, 보고서 및 HTML 폼리를 위한 다용도 비용 시스템을 제공합니다. 이 가이드는 뉴렐릭이 OTLP 데이터를 처리하고 수집하는 방법에 대한 개요를 제공하여 시스템을 효율적으로 모니터링하고 관찰할 수 있도록 도와줍니다.

## 시작하다 [#get-started]

OpenTelemetry 뉴렐릭과 통합하는 방법에 대해 자세히 알아보려면 [OpenTelemetry 및 뉴렐릭 시작하기를](/docs/opentelemetry/get-started/opentelemetry-get-started-intro/) 참조하세요. 이 페이지에서는 일반적인 통합 패턴과 모범 사례에 대해 다룹니다.

## 뉴렐릭을 위한 OTLP 구성 [#configure]

자세한 OTLP 설정 요구 사항 및 문제 해결 지침은 다음 페이지를 방문하세요.

* [뉴렐릭 OTLP 엔드포인트는](/docs/opentelemetry/best-practices/opentelemetry-otlp/) OpenTelemetry 데이터를 뉴렐릭에 통합하기 위한 필수 설정 단계를 안내합니다.
* [뉴렐릭 OTLP 문제 해결은](/docs/opentelemetry/best-practices/opentelemetry-otlp-troubleshooting/) 로그 확인, API 키, 연결 문제 등 데이터 수집과 관련된 일반적인 문제를 해결하는 데 도움이 됩니다.

### OpenTelemetry의 신호 유형 [#signals]

OTLP는 트레이스, 메트릭, 로그 신호로 구분되며, 이는 모두 리소스 개념을 포함합니다. 다음 페이지에서는 각 지표가 뉴렐릭에서 처리되고 수집되는 방법에 대한 세부 정보를 제공합니다.

* [뉴렐릭의 OpenTelemetry 트레이스](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/)
* [뉴렐릭의 OpenTelemetry 지표](/docs/opentelemetry/best-practices/opentelemetry-best-practices-metrics/)
* [뉴렐릭의 OpenTelemetry 로그](/docs/opentelemetry/best-practices/opentelemetry-best-practices-logs/)
* [뉴렐릭의 OpenTelemetry 리소스](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/)

## 데이터 볼륨 관리 [#manage-volume]

OpenTelemetry 데이터 볼륨을 관리하고 수집 비용을 제어하려면 [OpenTelemetry 데이터 수집 볼륨 관리를](/docs/opentelemetry/best-practices/opentelemetry-manage-data-ingest-volume/) 참조하세요. 이 페이지에서는 효율적인 리소스 사용을 보장하기 위해 데이터 볼륨을 최적화하는 전략을 다룹니다.

## 뉴렐릭의 플랫폼을 이용하세요 [#platform]

OpenTelemetry 데이터가 [뉴렐릭 플랫폼](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/) 에 수집되면 다음과 같은 옵저버빌리티 도구 중 일부를 활용할 수 있습니다.

* [알림](/docs/alerts/overview/): 신호가 청년값을 초과하면 알림을 받습니다.
* [사용자 정의 쿼리](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/): 임시 데이터 분석 및 탐색을 위해 사용자 정의 NRQL 쿼리를 작성합니다.
* [대시보드](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/): 애플리케이션의 고유한 요구 사항을 반영하는 사용자 정의 데이터 시각화를 만듭니다.
