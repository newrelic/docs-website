## OpenTelemetry로 NGINX를 모니터링해야 하는 이유는 무엇일까요? [#why-monitor]

NGINX 모델링은 웹 성능과 안정성을 유지하는 데 필수적입니다. OpenTelemetry 이러한 핵심 데이터를 수집하고 분석하기 위한 표준화된 접근 방식을 제공합니다.

### 주요 이점

* **성능 최적화**: 병목 현상 식별, 병목지점, 응답 시간 추적, 데이터 중심 확장 결정
* **사전 예방적 모니터링**: 사용자에게 영향이 미치기 전에 알림을 받고 문제를 조기에 감지하세요.
* **운영 인사이트**: 트래픽 패턴 파악, SLA 준수 추적, 감사 기록 유지
* **미래 지향적**: 오픈소스 OpenTelemetry 표준, 공급업체 비종속 통합

### 사용 사례

전자상거래 웹사이트, API 게이트웨이 또는 콘텐츠 전송 플랫폼을 운영하든 관계없이 NGINX 모니터링은 빠르고 안정적인 웹 서비스를 유지하는 데 도움이 됩니다. 결제 과정 속도가 느려질 때 알림을 받고, API 엔드포인트에 과부하가 걸렸을 때 이를 감지하거나, 문제가 발생하는 지역을 파악하세요. 이러한 모니터링은 고객 거래를 처리하는 웹사이트, 모바일 앱 백앤드, 마이크로서비스를 제외하고 다운타임이나 느린 성능이 사용자와 비즈니스에 직접적인 영향을 미치는 모든 웹사이트에 필수적입니다.

## 사용 가능한 지표 [#metrics]

OpenTelemetry Collector 연결 통계, 요청 횟수 및 서버 상태 표시기를 포함한 주요 NGINX 성능 지표를 자동으로 수집합니다.

**수집된 주요 지표:**

* **연결 범위** - 활성, 승인됨, 처리된 연결 및 해당 상태
* **요청 빈도** - 시간 경과에 따른 총 requests 및 요청률
* **서버 상태** - 연결 효율성 및 성능 지표

**전체 지표 및 속성 참조:** 온호스트 및 Kubernetes 구현에 대한 자세한 설명, 지표 유형 및 리소스 속성은 [NGINX OpenTelemetry 지표 및 속성 참조를](/docs/opentelemetry/nginx/nginx-otel-metrics-reference/) 참조하세요.
