Amazon ECS 환경에 대한 OpenTelemetry 모니터링을 설정한 후, 뉴렐릭의 인프라 및 쿼리 인터페이스에서 데이터를 찾고 쿼리할 수 있습니다. 이 가이드는 ECS 메트릭을 찾고 모니터링 및 문제 진단, 해결을 위한 의미 있는 쿼리를 생성하는 데 도움이 됩니다.

## ECS 데이터를 찾을 수 있는 위치 [#find-data]

ECS 모니터링 데이터는 메트릭 유형과 ECS 시작 유형에 따라 뉴렐릭 내 여러 위치에 표시됩니다.

### 인프라 모니터링 [#infrastructure-view]

ECS 인프라를 보려면 <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Infrastructure**</DNT> (으)로 이동하세요:

#### EC2 기반 ECS의 경우

1. 호스트 뷰: 호스트별로 필터링하여 ECS 컨테이너를 실행 중인 EC2 인스턴스를 확인하세요.
2. 컨테이너 뷰: ECS 작업 내에서 개별 컨테이너 성능을 확인하세요

#### ECS Fargate의 경우

1. 컨테이너 뷰: Fargate 컨테이너 메트릭의 기본 위치

<Callout variant="tip">
  필터 옵션을 사용하여 특정 클러스터, 서비스 또는 컨테이너 이름으로 범위를 좁혀 필요한 데이터를 빠르게 찾습니다.
</Callout>

### 지표 및 이벤트 [#metrics-events]

**<DNT>[one.newrelic.com](https://one.newrelic.com)</DNT> &gt; Query your data** 로 이동하여 다음을 수행합니다:

* 커스텀 차트 및 대시보드 생성
* 특정 메트릭 임계값에 따라 알림 설정

## 쿼리 예시 [#query-examples]

### 컨테이너 CPU 사용률 모니터링

```sql
SELECT average(container.cpu.utilization)
FROM Metric
WHERE ClusterName = 'your-cluster-name'
AND ServiceName = 'your-service-name'
TIMESERIES AUTO
FACET container.name
```

### 메모리 사용량 추세 추적

```sql
SELECT max(container.memory.usage.total) / 1024 / 1024 as 'Memory (MB)'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET ServiceName
```

### 호스트 레벨 메트릭 모니터링(EC2 전용)

```sql
SELECT average(1 - system.cpu.utilization) * 100 AS 'CPU used %',
       average(system.memory.utilization) * 100 AS 'Memory used %'
FROM Metric
WHERE aws.ecs.launchtype = 'ec2'
AND aws.ecs.cluster.arn = 'your-cluster-arn'
TIMESERIES AUTO
```

## 데이터 갭 문제 해결 [#troubleshooting]

예상되는 데이터가 표시되지 않는 경우:

1. **수집기 상태 확인**: OpenTelemetry 수집기의 컨테이너 로그를 확인합니다.
2. **네트워크 연결 확인**: 뉴렐릭에 대한 아웃바운드 HTTPS 액세스를 확인하세요.
3. **라이선스 키 확인**: SSM 파라미터 스토어에 라이선스 키가 올바르게 구성되었는지 확인하세요.
4. **작업 정의 검토**: 수집기 컨테이너가 적절한 리소스 할당으로 실행 중인지 확인합니다.
