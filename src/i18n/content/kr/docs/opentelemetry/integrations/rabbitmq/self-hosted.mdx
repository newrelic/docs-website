[NRDOT Collector](https://github.com/newrelic/nrdot-collector-releases/releases/latest) (권장) 또는 [OpenTelemetry Collector](https://github.com/open-telemetry/opentelemetry-collector-contrib) 사용하여 Linux 호스트에서 실행 중인 RabbitMQ 모니터링하여 지표 및 텔메트리 데이터를 뉴렐릭으로 보냅니다. 이 종합 가이드는 Debian, Ubuntu, RHEL 및 CentOS 시스템의 설치, 설정 및 검증 과정을 단계별로 안내합니다.

## 설치 단계 [#install]

자체 호스팅 RabbitMQ 인스턴스에 대한 모니터링을 설정하려면 다음 단계를 따르십시오.

        <Step>
          ### 시작하기 전에 [#requirements-nrdot]

          사용 환경이 다음 요구 사항을 충족하는지 확인하십시오.

          <CollapserGroup>
            <Collapser id="rabbitmq-reqs" title="RabbitMQ 요구 사항">
              * **버전**: RabbitMQ 3.8 이상
              * **관리 플러그인**: 메트릭 API를 노출하려면 반드시 활성화해야 합니다.
              * **사용자 계정**: 관리 API접근할 수 있는 권한을 가진 관리 사용자
            </Collapser>

            <Collapser id="system-reqs" title="시스템 요구 사항">
              * **운영 시스템**: Debian, Ubuntu, RHEL, CentOS
              * **셸 접근 권한**: sudo 권한이 있는 사용자
              * **네트워크**: 뉴렐릭의 [OTLP 엔드포인트](https://docs.newrelic.com/docs/opentelemetry/best-practices/opentelemetry-otlp/#configure-endpoint-port-protocol)에 대한 아웃바운드 HTTPS 연결
              * **리소스**: 수집기 작동을 위해 최소 512MB RAM과 1개의 CPU 코어가 필요합니다.
            </Collapser>

            <Collapser id="newrelic-reqs" title="뉴렐릭 요구 사항">
              * **계정**: 활성 뉴렐릭 계정
              * 귀하의 계정에 대한 **클러스터** [키](/docs/apis/intro-apis/new-relic-api-keys/#license-key)
            </Collapser>

            <Collapser id="software-reqs" title="소프트웨어 요구 사항">
              * **NRDOT Collector**: [최신 릴리스](https://github.com/newrelic/nrdot-collector-releases/releases/latest) (뉴렐릭의 OpenTelemetry 배포판)
              * **시스템 패키지**: `wget`, `systemd` (일반적으로 사전 설치됨)
            </Collapser>
          </CollapserGroup>

          <Callout variant="tip">
            검증 단계에서 오류가 발생하면 누락된 구성 요소를 설치한 후 계속 진행하십시오. NRDOT Collector 설치에 도움이 필요하신가요? nrdot-수집기-releases의 [설치 섹션을](https://github.com/newrelic/nrdot-collector-releases/blob/main/distributions/README.md#installation) 확인하세요.

            참고: 설치 중 사용 `export collector_distro="nrdot-collector"`
          </Callout>

        <Step>
          ### RabbitMQ 관리 플러그인을 활성화하세요. [#enable-plugin]

          관리 플러그인은 OpenTelemetry Collector가 수집하는 API를 제공합니다.

          ```bash
          sudo rabbitmq-plugins enable rabbitmq_management
          ```

          제대로 작동하는지 확인하세요:

          ```bash
          curl -I -u admin:password http://localhost:15672/api/overview
          ```

          응답에 `HTTP/1.1 200 OK` 표시되어야 합니다.
        </Step>

        <Step>
          ### NRDOT Collector설치하세요 [#install-nrdot-collector]

          [NRDOT Collector 릴리스 페이지](https://github.com/newrelic/nrdot-collector-releases/releases/latest) 에서 호스트 운영 시스템에 맞는 NRDOT Collector 다운로드하여 설치하십시오. 배포판별 지침은 [설치 가이드를](https://github.com/newrelic/nrdot-collector-releases/blob/main/distributions/README.md#installation) 참조하십시오.

          참고: 설치 중 사용 `export collector_distro="nrdot-collector"`
        </Step>

        <Step>
          ### 수집기 구성 [#configure-nrdot]

          `/etc/nrdot-collector/rabbitmq-config.yaml` 에서 수집기 설정 파일을 만듭니다.

          ```yaml
          receivers:
            rabbitmq:
              endpoint: http://localhost:15672
              username: admin
              password: password
              collection_interval: 30s
              metrics:
                # Queue Metrics (essential for message flow and backlog)
                rabbitmq.consumer.count:
                  enabled: true
                rabbitmq.message.delivered:
                  enabled: true
                rabbitmq.message.published:
                  enabled: true
                rabbitmq.message.acknowledged:
                  enabled: true
                rabbitmq.message.dropped:
                  enabled: true
                rabbitmq.message.current:
                  enabled: true # Crucial for monitoring queue backlog, includes 'ready' and 'unacknowledged' states

                # Node Health Metrics (critical for server resource monitoring)
                rabbitmq.node.disk_free:
                  enabled: true
                rabbitmq.node.disk_free_limit:
                  enabled: true
                rabbitmq.node.disk_free_alarm:
                  enabled: true
                rabbitmq.node.mem_used:
                  enabled: true
                rabbitmq.node.mem_limit:
                  enabled: true
                rabbitmq.node.mem_alarm:
                  enabled: true
                rabbitmq.node.mem_used_details.rate:
                  enabled: true
                rabbitmq.node.fd_used:
                  enabled: true
                rabbitmq.node.fd_total:
                  enabled: true
                rabbitmq.node.sockets_used:
                  enabled: true
                rabbitmq.node.sockets_total:
                  enabled: true
                rabbitmq.node.proc_used:
                  enabled: true
                rabbitmq.node.proc_total:
                  enabled: true
                rabbitmq.node.uptime:
                  enabled: true
                rabbitmq.node.run_queue:
                  enabled: true
                rabbitmq.node.processors:
                  enabled: true
                rabbitmq.node.context_switches_details.rate:
                  enabled: true
                rabbitmq.node.gc_num_details.rate:
                  enabled: true
                rabbitmq.node.gc_bytes_reclaimed_details.rate:
                  enabled: true

                # I/O Metrics (important for understanding disk and network activity)
                rabbitmq.node.io_read_count_details.rate:
                  enabled: true
                rabbitmq.node.io_read_bytes_details.rate:
                  enabled: true
                rabbitmq.node.io_read_avg_time_details.rate:
                  enabled: true
                rabbitmq.node.io_write_count_details.rate:
                  enabled: true
                rabbitmq.node.io_write_bytes_details.rate:
                  enabled: true
                rabbitmq.node.io_write_avg_time_details.rate:
                  enabled: true
                rabbitmq.node.io_sync_count_details.rate:
                  enabled: true
                rabbitmq.node.io_sync_avg_time_details.rate:
                  enabled: true
                rabbitmq.node.io_seek_count_details.rate:
                  enabled: true
                rabbitmq.node.io_seek_avg_time_details.rate:
                  enabled: true
                rabbitmq.node.io_reopen_count_details.rate:
                  enabled: true

                # Mnesia and Store Metrics (for internal database and message storage)
                rabbitmq.node.mnesia_ram_tx_count_details.rate:
                  enabled: true
                rabbitmq.node.mnesia_disk_tx_count_details.rate:
                  enabled: true
                rabbitmq.node.msg_store_read_count_details.rate:
                  enabled: true
                rabbitmq.node.msg_store_write_count_details.rate:
                  enabled: true
                rabbitmq.node.queue_index_write_count_details.rate:
                  enabled: true
                rabbitmq.node.queue_index_read_count_details.rate:
                  enabled: true

                # Connection/Channel/Queue Lifecycle Metrics
                rabbitmq.node.connection_created_details.rate:
                  enabled: true
                rabbitmq.node.connection_closed_details.rate:
                  enabled: true
                rabbitmq.node.channel_created_details.rate:
                  enabled: true
                rabbitmq.node.channel_closed_details.rate:
                  enabled: true
                rabbitmq.node.queue_declared_details.rate:
                  enabled: true
                rabbitmq.node.queue_created_details.rate:
                  enabled: true
                rabbitmq.node.queue_deleted_details.rate:
                  enabled: true

          # processors: Process data before exporting.
          processors:
            resourcedetection:
              detectors: [system]
              system:
                resource_attributes:
                  host.name:
                    enabled: true
                  host.id:
                    enabled: true
            resource:
              attributes:
                - key: instrumentation.provider
                  value: opentelemetry
                  action: upsert
                - key: rabbitmq.deployment.name
                  value: my-rabbitmq-server  # Replace with your server name
                  action: upsert
            batch:
              send_batch_size: 1024
              timeout: 30s
            transform/metadata_nullify:
              metric_statements:
                - context: metric
                  statements:
                    - set(description, "")
                    - set(unit, "")

          exporters:
            otlp_http:
              endpoint: ${env:OTEL_EXPORTER_OTLP_ENDPOINT:-https://otlp.nr-data.net}
              headers:
                api-key: ${env:NEW_RELIC_LICENSE_KEY}

          service:
            pipelines:
              metrics:
                receivers: [rabbitmq]
                processors: [resourcedetection, resource, batch, transform/metadata_nullify]
                exporters: [otlp_http]
          ```

          설정에서 다음 값을 업데이트하세요.

          * `endpoint`RabbitMQ 관리 API URL (기본값: `http://localhost:15672`)
          * `username` 및 `password`: RabbitMQ 자격 증명
          * `rabbitmq.deployment.name`이 RabbitMQ 인스턴스에 대한 고유한 이름입니다.
          * `collection_interval`메트릭을 수집하는 빈도(기본값: 30초)
        </Step>

        <Step>
          ### 환경을 구성하고 구성 경로를 업데이트합니다. [#set-environment-nrdot]

          RabbitMQ 설정 파일을 사용하도록 NRDOT Collector 구성을 업데이트하고 OTLP 엔드포인트를 설정하십시오.

          <Callout variant="important">
            라이선스 키는 설치 중에 `/etc/nrdot-collector/nrdot-collector.conf` 에 이미 구성되었습니다. 설정 파일 경로와 OTLP 엔드포인트만 업데이트하면 됩니다.
          </Callout>

          귀하의 뉴렐릭 지역을 기준으로 OTLP 포인트를 결정하세요. 엔드포인트의 전체 목록은 [엔드포인트, 포트 및 프로토콜 구성을](/docs/opentelemetry/best-practices/opentelemetry-otlp/#configure-endpoint-port-protocol) 참조하세요.

          ```bash
          export collector_distro="nrdot-collector"
          export otlp_endpoint="https://otlp.nr-data.net"  # Replace with your region's endpoint

          # Update the config file path to point to rabbitmq-config.yaml
          sudo sed -i 's|OTELCOL_OPTIONS="--config=/etc/nrdot-collector/config.yaml"|OTELCOL_OPTIONS="--config=/etc/nrdot-collector/rabbitmq-config.yaml"|' /etc/${collector_distro}/${collector_distro}.conf

          # Add the OTLP endpoint
          echo "OTEL_EXPORTER_OTLP_ENDPOINT=${otlp_endpoint}" | sudo tee -a /etc/${collector_distro}/${collector_distro}.conf > /dev/null
          ```

          systemd 다시 로드하고 수집기를 재시작하세요.

          ```bash
          sudo systemctl daemon-reload
          sudo systemctl restart nrdot-collector.service
          ```
        </Step>

        <Step>
          ### 뉴렐릭에서 데이터를 확인하세요 [#verify-nrdot]

          서비스 상태를 확인하세요:

          ```bash
          sudo systemctl status nrdot-collector.service
          ```

          출력 결과에 `active (running)` 표시되어야 합니다.

          최근 로그 보기:

          ```bash
          sudo journalctl -u nrdot-collector.service -n 50 --no-pager
          ```

          데이터 수집이 성공적으로 완료되었음을 나타내는 메시지를 찾아보세요.

          ```
          INFO    RabbitmqReceiver        Successfully scraped rabbitmq metrics
          ```

          뉴렐릭에서 지표를 확인하세요.

          데이터가 나타날 때까지 2-3분 정도 기다린 후 [쿼리 빌더](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder) 에서 다음 쿼리를 실행하세요.

          ```sql
          SELECT count(*)
          FROM Metric
          WHERE metricName LIKE 'rabbitmq.%'
            AND instrumentation.provider = 'opentelemetry'
            AND rabbitmq.deployment.name = 'my-rabbitmq-server'
          FACET metricName
          SINCE 10 minutes ago
          ```

          다음과 같은 메트릭 이름이 표시될 것입니다.

          * `rabbitmq.queue.count` - 대기열 수
          * `rabbitmq.queue.message.count` - 큐 전체의 총 메시지 수
          * `rabbitmq.connection.count` - 활성 연결
          * `rabbitmq.consumer.count` - 적극적인 소비자

          <Callout variant="tip">
            5분 후에도 데이터가 표시되지 않으면 아래 [문제 해결, 해결 섹션을](#troubleshoot) 확인하세요.
          </Callout>
        </Step>

        <Step>
          ### (선택 사항) RabbitMQ 로그를 뉴렐릭으로 전달합니다. [#logs]

          메트릭 외에도 포괄적인 옵저버빌리티를 위해 RabbitMQ 로그 로그를 뉴렐릭으로 전달할 수 있습니다. 이는 지표와 로그 전반에 걸쳐 문제를 연관시키는 데 도움이 됩니다.

          <Callout variant="important">
            로그 포워딩에는 추가 설정 및 시스템 권한이 필요합니다. 진행하기 전에 필요한 접근 권한이 있는지 확인하십시오.
          </Callout>

          **`/etc/nrdot-collector/rabbitmq-config.yaml`업데이트하여** 파일 로그 수신기를 추가하세요.

          ```yaml
          receivers:
            rabbitmq:
              # ... existing rabbitmq config ...

            # Add filelog receiver for RabbitMQ logs
            filelog/rabbitmq:
              include:
                - /var/log/rabbitmq/*.log
                - /var/log/rabbitmq/**/*.log
              include_file_path: true
              include_file_name: false
              operators:
                - type: regex_parser
                  regex: '^(?P<timestamp>\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}\.\d{3}) \[(?P<level>\w+)\] <(?P<pid>[^>]+)> (?P<message>.*)$'
                  timestamp:
                    parse_from: attributes.timestamp
                    layout: '%Y-%m-%d %H:%M:%S.%L'
                  severity:
                    parse_from: attributes.severity

          processors:
            # ... existing processors ...

            transform/rabbitmq_logs:
              log_statements:
                - context: resource
                  statements:
                    - set(attributes["rabbitmq.display.name"], Concat(["server", attributes["rabbitmq.deployment.name"]], ":"))

          service:
            pipelines:
              metrics:
                receivers: [rabbitmq]
                processors: [resourcedetection, resource, batch, transform/metadata_nullify]
                exporters: [otlp_http]

              logs/rabbitmq:
                receivers: [filelog]
                processors: [batch, resource, transform/rabbitmq_logs]
                exporters: [otlp_http]
          ```

          수집기가 로그를 읽을 수 있는 권한을 부여하십시오. 한 가지 방법을 선택하세요:

          rabbitmq 그룹에 추가:

          ```bash
          sudo usermod -a -G rabbitmq nrdot-collector
          ```

          또는 파일 권한을 설정하세요.

          ```bash
          sudo chmod 644 /var/log/rabbitmq/*.log
          ```

          수집기를 재시작하세요:

          ```bash
          sudo systemctl restart nrdot-collector.service
          ```

          뉴렐릭에서 로그가 수집되고 있는지 확인하세요.

          ```sql
          SELECT count(*)
          FROM Log
          WHERE service.name = 'rabbitmq'
            AND rabbitmq.deployment.name = 'my-rabbitmq-server'
          SINCE 10 minutes ago
          ```

## 데이터 찾기 [#find-data]

몇 분 후 RabbitMQ 창이 뉴렐릭에 나타날 것입니다. 뉴럴릭 UI 의 다양한 보기에서 RabbitMQ 대역폭을 탐색하는 방법에 대한 자세한 지침은 [데이터 찾기 및 검색을](/docs/opentelemetry/integrations/rabbitmq/find-and-query-data) 참조하세요.

NRQL을 사용하여 데이터를 쿼리할 수도 있습니다.

```sql
FROM Metric SELECT * WHERE rabbitmq.deployment.name = 'my-rabbitmq-server'
```

## 문제점 해결 [#troubleshoot]

<CollapserGroup>
  <Collapser id="no-data" title="뉴릭에 데이터가 나타나지 않습니다">
    설치 후 뉴렐릭에 지표가 표시되지 않는 경우:

    **1. 관리 플러그인이 활성화되어 있는지 확인하십시오.**

    ```bash
    sudo rabbitmq-plugins list | grep management
    ```

    `[E*] rabbitmq_management` 이 표시되면 활성화되어 실행 중인 것입니다.

    **2. 관리 API를 테스트합니다.**

    ```bash
    curl -u admin:password http://localhost:15672/api/overview
    ```

    이 명령은 RabbitMQ 인스턴스에 대한 JSON 데이터를 반환합니다. 그렇지 않다면 관리 플러그인이 제대로 작동하지 않는 것입니다.

    **3. 수집기 로그에서 오류를 확인하십시오.**

    ```bash
    sudo journalctl -u nrdot-collector.service -f
    ```

    다음 내용을 찾아보세요:

    * 인증 오류(잘못된 사용자 이름/비밀번호)
    * 연결 시간 초과(네트워크 문제)
    * 구문 분석 오류 (설정 구문 문제)

    **4. 수집기 서비스가 실행 중인지 확인합니다.**

    ```bash
    sudo systemctl status nrdot-collector.service
    ```

    `active (running)` 표시해야 합니다. 그렇지 않다면 로그를 확인하여 시작 실패 원인을 파악하십시오.

    **5. 라이선스 키를 확인하세요:**

    ```bash
    sudo cat /etc/nrdot-collector/nrdot-collector.conf
    ```

    `NEW_RELIC_LICENSE_KEY` 이 올바르게 설정되었는지, 불필요한 공백이나 따옴표가 없는지 확인하십시오.
  </Collapser>

  <Collapser id="connection-refused" title="연결 거부 오류">
    수집기 로그에 `connection refused` 표시되면:

    **RabbitMQ가 15672번 포트에서 수신 대기 중인지 확인하십시오.**

    ```bash
    sudo netstat -tlnp | grep 15672
    ```

    RabbitMQ가 수신 대기 중인 것을 확인할 수 있을 것입니다. 그렇지 않다면 관리 플러그인이 활성화되어 있지 않거나 예상 포트에 바인딩되어 있지 않은 것입니다.

    **방화벽 규칙을 확인하세요:**

    ```bash
    sudo iptables -L -n | grep 15672
    ```

    방화벽 규칙으로 인해 15672번 포트가 차단되지 않았는지 확인하십시오.

    **엔드포인트 설정 확인:**\
    `/etc/nrdot-collector/rabbitmq-config.yaml` 의 `endpoint` 이 RabbitMQ 설정(호스트명, 포트)과 일치하는지 확인하십시오.
  </Collapser>

  <Collapser id="auth-failures" title="인증 실패(401 권한 없음)">
    `401 Unauthorized` 오류가 발생하는 경우:

    **자격 증명을 확인하세요:**\
    `/etc/nrdot-collector/rabbitmq-config.yaml` 에 있는 사용자 이름과 비밀번호가 RabbitMQ 사용자 이름과 일치하는지 확인하십시오.

    **관리자 권한이 있는지 확인하십시오.**

    ```bash
    sudo rabbitmqctl list_users
    ```

    사용자 계정에는 `[administrator]` 태그가 있어야 합니다. 그렇지 않다면 관리자 권한을 부여하십시오.

    ```bash
    sudo rabbitmqctl set_user_tags your-username administrator
    ```
  </Collapser>

  <Collapser id="high-memory" title="메모리 또는 CPU 사용량이 높습니다.">
    수집기가 과도한 자원을 소비하는 경우:

    **배치 크기를 줄이세요:**\
    `batch` 프로세서에서 `send_batch_size` 512에서 256 이하로 줄이십시오.

    **수집 간격을 늘리세요:**\
    스크래핑 빈도를 낮추려면 `collection_interval` 30초에서 60초 이상으로 변경하세요.

    **불필요한 기능을 비활성화합니다.**\
    `rabbitmq` 수신기 설정에서는 필요하지 않은 지표입니다.

    예시:

    ```yaml
    receivers:
      rabbitmq:
        collection_interval: 60s  # Increased
        metrics:
          rabbitmq.queue.message.count:
            enabled: true
          # Disable less critical metrics
          # rabbitmq.queue.count:
          #   enabled: false
    ```
  </Collapser>

  <Collapser id="log-permissions" title="로그에 대한 권한이 거부되었습니다.">
    로그 수집 시 권한 오류가 발생하는 경우:

    **파일 접근 테스트:**

    ```bash
    sudo -u nrdot-collector cat /var/log/rabbitmq/rabbit@$(hostname).log
    ```

    이 과정이 실패하면 수집기 사용자는 로그 파일을 읽을 수 없습니다.

    **파일 권한을 확인하세요:**

    ```bash
    ls -la /var/log/rabbitmq/
    ```

    **권한 수정:**\
    Rabbitmq 그룹에 수집기 사용자를 추가합니다.

    ```bash
    sudo usermod -a -G rabbitmq nrdot-collector
    sudo systemctl restart nrdot-collector.service
    ```
  </Collapser>
</CollapserGroup>
