## IBM MQ лӘЁлӢҲн„°л§Ғ м„Өм • [#setup]

мҲҳм§‘кё°лҘј л°°нҸ¬н•ҳкі  IBM MQ л©”нҠёлҰӯмқ„ лүҙл җлҰӯмңјлЎң м „мҶЎн•ҳкё° мӢңмһ‘н•ҳл Өл©ҙ лӢӨмқҢ лӢЁкі„лҘј л”°лҘҙмӢӯмӢңмҳӨ:

<Steps>
  <Step>
    ### лүҙл җлҰӯ мһҗкІ© мҰқлӘ… л№„л°Җмқ„ мғқм„ұн•©лӢҲлӢӨ. [#create-secret]

    лүҙл җлҰӯ мһҗкІ© мҰқлӘ…мқ„ м•Ҳм „н•ҳкІҢ м ҖмһҘн•ҳкё° мң„н•ҙ Kubernetes Secretмқ„ мғқм„ұн•©лӢҲлӢӨ. мҲҳм§‘кё°лҠ” лҹ°нғҖмһ„м—җ мқҙлҹ¬н•ң к°’мқ„ мқҪм–ҙ лҜјк°җн•ң м •ліҙк°Җ м„Өм • нҢҢмқјм—җ нҸ¬н•Ёлҗҳм§Җ м•ҠлҸ„лЎқ н•©лӢҲлӢӨ.

    1. `ibmmq` л„Өмһ„мҠӨнҺҳмқҙмҠӨк°Җ мЎҙмһ¬н•ҳлҠ”м§Җ нҷ•мқён•©лӢҲлӢӨ:

    ```bash
    kubectl get namespace ibmmq >/dev/null 2>&1 || kubectl create namespace ibmmq
    ```

    2. мһҗкІ© мҰқлӘ… мӢңнҒ¬лҰҝмқ„ мғқм„ұн•ҳкі , `<YOUR_LICENSE_KEY>` мқ„(лҘј) мӢӨм ң лқјмқҙм„ мҠӨ нӮӨлЎң л°”кҝүлӢҲлӢӨ:

    ```bash
    kubectl create secret generic newrelic-otlp-secret \
      --namespace ibmmq \
      --from-literal=NEW_RELIC_LICENSE_KEY="<YOUR_LICENSE_KEY>" \
      --from-literal=NEW_RELIC_OTLP_ENDPOINT="https://otlp.nr-data.net:4318" \
      --dry-run=client -o yaml | kubectl apply -f -
    ```

    EU кі„м •мқҳ кІҪмҡ° м—”л“ңнҸ¬мқёнҠё к°’мңјлЎң `https://otlp.eu01.nr-data.net:4318` мқ„(лҘј) мӮ¬мҡ©н•©лӢҲлӢӨ.
  </Step>

  <Step>
    ### мҲҳм§‘кё° Helm к°’ кө¬м„ұ [#configure-values]

    мҲҳм§‘кё° м„Өм •мқҙ нҸ¬н•Ёлҗң лЎңм»¬ `values.yaml` нҢҢмқјмқ„ мғқм„ұн•©лӢҲлӢӨ. мқҙ нҢҢмқјм—җлҠ” OpenTelemetry Helm м°ЁнҠёлҘј нҶөн•ҙ мҲҳм§‘кё°лҘј л°°нҸ¬н•ҳлҠ” лҚ° н•„мҡ”н•ң лӘЁл“  м„Өм •мқҙ нҸ¬н•Ёлҗҳм–ҙ мһҲмҠөлӢҲлӢӨ.

    <Callout variant="caution">
      мҙҲкё° л°°нҸ¬ нӣ„ `TARGET_NAME` мқ„(лҘј) ліҖкІҪн•ҳм§Җ л§ҲмӢӯмӢңмҳӨ. мқҙ к°’мқҖ лӘЁл“  `IBMMQ_MANAGER` л°Ҹ `IBMMQ_QUEUE` м—”нӢ°нӢ° GUIDмқҳ мІ« лІҲм§ё м„ёк·ёлЁјнҠёлҘј нҳ•м„ұн•©лӢҲлӢӨ. мқҙлҘј ліҖкІҪн•ҳл©ҙ мғҲлЎңмҡҙ м—”нӢ°нӢ°к°Җ мғқм„ұлҗҳкі  кё°мЎҙ м—”нӢ°нӢ°к°Җ кі лҰҪлҗҳм–ҙ лҢҖмӢңліҙл“ң л°Ҹ м•ҢлҰјмқҙ мҶҗмғҒлҗ©лӢҲлӢӨ.
    </Callout>

    <CollapserGroup>
      <Collapser id="nrdot" title="NRDOT Collectorмқҳ кІҪмҡ°(к¶ҢмһҘ)">
        `TARGET_NAME` л°Ҹ `IBMMQ_CLUSTER_NAME` мқ„(лҘј) нҷҳкІҪмқҳ к°’мңјлЎң л°”кҫёмӢӯмӢңмҳӨ:

        ```yaml
        mode: deployment
        replicaCount: 1   # Must stay 1 вҖ” kubernetes_sd scrapes every target from every replica.
                          # For HA, use the OpenTelemetry TargetAllocator to shard targets.

        image:
          repository: newrelic/nrdot-collector
          tag: "latest"   # Pin a specific tested version for production
          pullPolicy: Always

        # --- RBAC for kubernetes_sd_configs(role: pod) ---
        serviceAccount:
          create: true
          name: ""

        clusterRole:
          create: true
          rules:
            - apiGroups: [""]
              resources:
                - pods
                - nodes
                - nodes/proxy
                - nodes/metrics
                - services
                - endpoints
              verbs: ["get", "list", "watch"]
            - nonResourceURLs: ["/metrics"]
              verbs: ["get"]

        # --- New Relic credentials from the Secret (never hardcoded) ---
        extraEnvsFrom:
          - secretRef:
              name: newrelic-otlp-secret

        # TARGET_NAME is the STABLE first segment of every IBMMQ entity GUID.
        # Keep it constant across redeploys вҖ” changing it orphans existing entities.
        extraEnvs:
          - name: TARGET_NAME
            value: "prod-mq-cluster"   # <-- change to your stable cluster identifier
          - name: IBMMQ_CLUSTER_NAME
            value: "prod-mq"           # <-- change to your cluster name

        resources:
          limits:
            cpu: 200m
            memory: 512Mi   # must stay above memory_limiter limit_mib so the soft limiter trips before the kubelet OOM-kills
          requests:
            cpu: 100m
            memory: 256Mi

        # This Collector only scrapes outbound and exports outbound.
        # Disabling the Service prevents Kubernetes from rejecting a zero-port Service spec.
        # Self-metrics on :8888 are reachable via kubectl port-forward to the pod directly.
        service:
          enabled: false

        # Disable chart default receivers/ports вҖ” only prometheus and health_check are used.
        ports:
          otlp:
            enabled: false
          otlp-http:
            enabled: false
          jaeger-compact:
            enabled: false
          jaeger-thrift:
            enabled: false
          jaeger-grpc:
            enabled: false
          zipkin:
            enabled: false
          metrics:
            enabled: false

        # Disable chart presets вҖ” the full config is supplied below.
        presets:
          kubernetesAttributes:
            enabled: false
          hostMetrics:
            enabled: false
          logsCollection:
            enabled: false
          kubeletMetrics:
            enabled: false

        # --- Full Collector configuration ---
        config:
          extensions:
            health_check:
              endpoint: 0.0.0.0:13133

          receivers:
            # One auto-discovery receiver discovers every pod in the ibmmq namespace
            # that carries prometheus.io/scrape=true and scrapes podIP:<port><path>.
            # Adding a new QM = deploy another annotated pod; no collector change needed.
            prometheus/ibmmq:
              config:
                scrape_configs:
                  - job_name: 'ibmmq-k8s-pods'
                    scrape_interval: 60s
                    scrape_timeout: 15s
                    kubernetes_sd_configs:
                      - role: pod
                        namespaces:
                          names:
                            - ibmmq
                    relabel_configs:
                      # (1) Keep only pods that opted into scraping
                      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
                        action: keep
                        regex: "true"
                      # (2) Skip non-Running pods (reduces target churn)
                      - source_labels: [__meta_kubernetes_pod_phase]
                        action: drop
                        regex: (Pending|Succeeded|Failed|Unknown)
                      # (3) Metrics path from annotation (defaults to /metrics)
                      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_path]
                        action: replace
                        target_label: __metrics_path__
                        regex: (.+)
                      # (4) __address__ = podIP:<prometheus.io/port>
                      #     $$ escapes to literal $ so Prometheus relabeling receives $1:$2
                      - source_labels:
                          - __meta_kubernetes_pod_ip
                          - __meta_kubernetes_pod_annotation_prometheus_io_port
                        action: replace
                        target_label: __address__
                        regex: (.+);(.+)
                        replacement: $$1:$$2
                      # (5) Stable entity GUID first segment (constant across all QMs)
                      - target_label: targetName
                        replacement: ${env:TARGET_NAME}
                      # (6) Cluster tag on IBMMQ_MANAGER entities
                      - target_label: clusterName
                        replacement: ${env:IBMMQ_CLUSTER_NAME}
                      # (7) Cosmetic namespace label (debug/visibility only)
                      - source_labels: [__meta_kubernetes_namespace]
                        action: replace
                        target_label: k8s_namespace
                      # (8) Cosmetic pod label (debug/visibility only)
                      - source_labels: [__meta_kubernetes_pod_name]
                        action: replace
                        target_label: k8s_pod

          processors:
            # Drop go_* / process_* / promhttp_* / scrape_* overhead metrics
            filter/ibmmq-overhead:
              metrics:
                exclude:
                  match_type: regexp
                  metric_names:
                    - "^go_.*"
                    - "^process_.*"
                    - "^promhttp_.*"
                    - "^scrape_.*"

            # Exclude high-cardinality SYSTEM.* and AMQ.* internal queues
            filter/ibmmq-queues:
              metrics:
                datapoint:
                  - 'attributes["queue"] != nil and IsMatch(attributes["queue"], "^SYSTEM\\.(ADMIN\\.|MQSC\\.|DEFAULT\\.|AUTH\\.|CHANNEL\\.|CHLAUTH\\.|CICS\\.|SYNCPOINT\\.|INTERNAL\\.|PENDING\\.|PROTECTION\\.|BROKER\\.|AMQP\\.|DOTNET\\.|REST\\.|RETAINED\\.|SELECTION\\.|DURABLE\\.|HIERARCHY\\.|DDELAY\\.)")'
                  - 'attributes["queue"] != nil and IsMatch(attributes["queue"], "^SYSTEM\\.CLUSTER\\.(COMMAND|HISTORY)\\.QUEUE$")'
                  - 'attributes["queue"] != nil and IsMatch(attributes["queue"], "^AMQ\\.")'

            # Detect host/cloud resource attributes
            resourcedetection:
              detectors: [env, ec2, gcp, azure, system, k8snode]
              system:
                resource_attributes:
                  host.name:
                    enabled: true
                  host.id:
                    enabled: true

            # Strip OTel auto-attributes that would land metrics on the wrong entity
            # NOTE: qmgr and queue labels are NOT touched вҖ” entity synthesis depends on them
            transform/ibmmq-cleanup:
              metric_statements:
                - context: resource
                  statements:
                    - delete_key(attributes, "server.address")
                    - delete_key(attributes, "server.port")
                    - delete_key(attributes, "url.scheme")
                - context: datapoint
                  statements:
                    # Rename injected identity labels to OTel dotted form (entity synthesis
                    # keys on target.name / cluster.name). qmgr / queue stay raw.
                    - set(attributes["target.name"], attributes["targetName"]) where attributes["targetName"] != nil
                    - delete_key(attributes, "targetName")
                    - set(attributes["cluster.name"], attributes["clusterName"]) where attributes["clusterName"] != nil
                    - delete_key(attributes, "clusterName")
                    - delete_key(attributes, "instance")
                    - delete_key(attributes, "job")

            memory_limiter/ibmmq:
              check_interval: 1s
              limit_mib: 400
              spike_limit_mib: 100

            # Convert Prometheus monotonic counters to delta (required for NR rate metrics)
            cumulativetodelta/ibmmq: {}

            batch/ibmmq:
              send_batch_size: 1000
              timeout: 200ms

          exporters:
            otlphttp/ibmmq:
              endpoint: ${env:NEW_RELIC_OTLP_ENDPOINT}
              headers:
                api-key: ${env:NEW_RELIC_LICENSE_KEY}

          service:
            extensions: [health_check]
            # Expose collector self-telemetry on :8888 for kubectl port-forward inspection.
            # This is separate from the inbound Service (which is disabled above).
            telemetry:
              metrics:
                level: detailed
                readers:
                  - pull:
                      exporter:
                        prometheus:
                          host: 0.0.0.0
                          port: 8888
            pipelines:
              metrics/ibmmq:
                receivers: [prometheus/ibmmq]
                processors:
                  - filter/ibmmq-overhead
                  - filter/ibmmq-queues
                  - resourcedetection
                  - transform/ibmmq-cleanup
                  - memory_limiter/ibmmq
                  - cumulativetodelta/ibmmq
                  - batch/ibmmq
                exporters: [otlphttp/ibmmq]
        ```

                          - __meta_kubernetes_pod_ip
                          - __meta_kubernetes_pod_annotation_prometheus_io_port
                        action: replace
                        target_label: __address__
                        regex: (.+);(.+)
                        replacement: $$1:$$2
                      # (5) Stable entity GUID first segment (constant across all QMs)
                      - target_label: targetName
                        replacement: ${env:TARGET_NAME}
                      # (6) Cluster tag on IBMMQ_MANAGER entities
                      - target_label: clusterName
                        replacement: ${env:IBMMQ_CLUSTER_NAME}
                      # (7) Cosmetic namespace label (debug/visibility only)
                      - source_labels: [__meta_kubernetes_namespace]
                        action: replace
                        target_label: k8s_namespace
                      # (8) Cosmetic pod label (debug/visibility only)
                      - source_labels: [__meta_kubernetes_pod_name]
                        action: replace
                        target_label: k8s_pod

          processors:
            # Drop go_* / process_* / promhttp_* / scrape_* overhead metrics
            filter/ibmmq-overhead:
              metrics:
                exclude:
                  match_type: regexp
                  metric_names:
                    - "^go_.*"
                    - "^process_.*"
                    - "^promhttp_.*"
                    - "^scrape_.*"

            # Exclude high-cardinality SYSTEM.* and AMQ.* internal queues
            filter/ibmmq-queues:
              metrics:
                datapoint:
                  - 'attributes["queue"] != nil and IsMatch(attributes["queue"], "^SYSTEM\\.(ADMIN\\.|MQSC\\.|DEFAULT\\.|AUTH\\.|CHANNEL\\.|CHLAUTH\\.|CICS\\.|SYNCPOINT\\.|INTERNAL\\.|PENDING\\.|PROTECTION\\.|BROKER\\.|AMQP\\.|DOTNET\\.|REST\\.|RETAINED\\.|SELECTION\\.|DURABLE\\.|HIERARCHY\\.|DDELAY\\.)")'

                  - 'attributes["queue"] != nil and IsMatch(attributes["queue"], "^AMQ\\.")'

            # Detect host/cloud resource attributes
            resourcedetection:
              detectors: [env, ec2, gcp, azure, system, k8snode]
              system:
                resource_attributes:
                  host.name:
                    enabled: true
                  host.id:
                    enabled: true

            # Strip OTel auto-attributes that would land metrics on the wrong entity
            # NOTE: qmgr and queue labels are NOT touched вҖ” entity synthesis depends on them
            transform/ibmmq-cleanup:
              metric_statements:
                - context: resource
                  statements:
                    - delete_key(attributes, "server.address")
                    - delete_key(attributes, "server.port")
                    - delete_key(attributes, "url.scheme")
                - context: datapoint
                  statements:
                    # Rename injected identity labels to OTel dotted form (entity synthesis
                    # keys on target.name / cluster.name). qmgr / queue stay raw.
                    - set(attributes["target.name"], attributes["targetName"]) where attributes["targetName"] != nil
                    - delete_key(attributes, "targetName")
                    - set(attributes["cluster.name"], attributes["clusterName"]) where attributes["clusterName"] != nil
                    - delete_key(attributes, "clusterName")
                    - delete_key(attributes, "instance")
                    - delete_key(attributes, "job")

            memory_limiter/ibmmq:
              check_interval: 1s
              limit_mib: 400
              spike_limit_mib: 100

            # Convert Prometheus monotonic counters to delta (required for NR rate metrics)
            cumulativetodelta/ibmmq: {}

            batch/ibmmq:
              send_batch_size: 1000
              timeout: 200ms

          exporters:
            otlphttp/ibmmq:
              endpoint: ${env:NEW_RELIC_OTLP_ENDPOINT}
              headers:
                api-key: ${env:NEW_RELIC_LICENSE_KEY}

          service:
            extensions: [health_check]
            # Expose collector self-telemetry on :8888 for kubectl port-forward inspection.
            # This is separate from the inbound Service (which is disabled above).
            telemetry:
              metrics:
                level: detailed
                readers:
                  - pull:
                      exporter:
                        prometheus:
                          host: 0.0.0.0
                          port: 8888
            pipelines:
              metrics/ibmmq:
                receivers: [prometheus/ibmmq]
                processors:
                  - filter/ibmmq-overhead
                  - filter/ibmmq-queues
                  - resourcedetection
                  - transform/ibmmq-cleanup
                  - memory_limiter/ibmmq
                  - cumulativetodelta/ibmmq
                  - batch/ibmmq
                exporters: [otlphttp/ibmmq]

        мһҗлҸҷ кІҖмғү лҰ¬мӢңлІ„лҠ” `ibmmq` л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җм„ң **нҢҢл“ң н…ңн”ҢлҰҝ** (л°°нҸ¬ лҳҗлҠ” StatefulSet к°қмІҙ мһҗмІҙк°Җ м•„лӢҲлқј `.spec.template.metadata.annotations`м—җ мһҲмқҢ)м—җ мқҙ м„ё к°Җм§Җ мЈјм„қмқҙ мһҲлҠ” нҢҢл“ңлҘј м°ҫмҠөлӢҲлӢӨ:

        ```yaml
        metadata:
          annotations:
            prometheus.io/scrape: "true"
            prometheus.io/port: "9157"
            prometheus.io/path: "/metrics"
        ```

        **`prometheus.io/scrape: "true"`** мқҖ(лҠ”) кё°ліё кІҢмқҙнҠёмһ…лӢҲлӢӨ. relabel к·ңм№ҷмқҖ мқҙ мЈјм„қм—җ `action: keep` мқ„(лҘј) мӮ¬мҡ©н•©лӢҲлӢӨ. мҰү, мқҙ мЈјм„қмқҙ м •нҷ•нһҲ `"true"` л¬ёмһҗм—ҙлЎң м„Өм •лҗҳм§Җ м•ҠмқҖ нҢҢл“ңлҠ” мҠӨнҒ¬лһ© мӢңлҸ„ м „м—җ нғҖкІҹ лӘ©лЎқм—җм„ң мһҗлҸҷмңјлЎң мӮӯм ңлҗ©лӢҲлӢӨ. `True`, `yes`(мңј)лЎң м„Өм •н•ҳкұ°лӮҳ мҷ„м „нһҲ мғқлһөн•ҳл©ҙ лӘЁл‘җ нҢҢл“ңк°Җ мҠӨнҒ¬лһ©лҗҳм§Җ м•ҠмҠөлӢҲлӢӨ. мқҙ мЈјм„қмқҖ м„ңл№„мҠӨ лҳҗлҠ” мқёк·ёл ҲмҠӨ к°қмІҙк°Җ м•„лӢҢ нҢҢл“ң н…ңн”ҢлҰҝм—җ мһҲм–ҙм•ј н•©лӢҲлӢӨ.

        **`prometheus.io/port: "9157"`** мқҖ(лҠ”) нҢҢл“ңмқҳ IPм—җм„ң м—°кІ°н•  нҸ¬нҠёлҘј Collectorм—җ м•Ңл ӨмӨҚлӢҲлӢӨ. мқҙлҠ” mq-metric-samples мқөмҠӨнҸ¬н„° мӮ¬мқҙл“ңм№ҙк°Җ мӢӨм ңлЎң мҲҳмӢ  лҢҖкё° мӨ‘мқё нҸ¬нҠёмҷҖ мқјм№ҳн•ҙм•ј н•©лӢҲлӢӨ. кё°ліё IBM MQ Prometheus мқөмҠӨнҸ¬н„° нҸ¬нҠёлҠ” `9157`мһ…лӢҲлӢӨ. мқөмҠӨнҸ¬н„°к°Җ лӢӨлҘё нҸ¬нҠём—җм„ң мҲҳмӢ  лҢҖкё°н•ҳлҸ„лЎқ м„Өм •лҗң кІҪмҡ°, мқҙ мЈјм„қмқ„ мқјм№ҳн•ҳлҸ„лЎқ ліҖкІҪн•ҳм„ёмҡ” вҖ• лӢӨлҘё м„Өм • ліҖкІҪмқҖ н•„мҡ”н•ҳм§Җ м•ҠмҠөлӢҲлӢӨ. CollectorлҠ” мқҙ мЈјмҶҢлҘј нҷ•мқён•ҳкё° мң„н•ҙ Kubernetes `Service` мқ„(лҘј) мӮ¬мҡ©н•ҳм§Җ м•Ҡмңјл©°, м„ңл№„мҠӨ лЎңл“ң л°ёлҹ°мӢұмқ„ мҡ°нҡҢн•ҳм—¬ нҢҢл“ң IPм—җ м§Ғм ‘ м—°кІ°н•©лӢҲлӢӨ. мҰү, мЈјм„қмқҖ м„ңл№„мҠӨ нҸ¬нҠёк°Җ м•„лӢҢ мӢӨм ң м»Ён…Ңмқҙл„Ҳ нҸ¬нҠёлҘј л°ҳмҳҒн•ҙм•ј н•©лӢҲлӢӨ.

        **`prometheus.io/path: "/metrics"`** мқҖ(лҠ”) Collectorм—җ мҡ”мІӯн•  HTTP кІҪлЎңлҘј м•Ңл ӨмӨҚлӢҲлӢӨ. IBM MQ Prometheus мқөмҠӨнҸ¬н„°мқҳ кё°ліёк°’мқҖ `/metrics`мһ…лӢҲлӢӨ. мқөмҠӨнҸ¬н„°к°Җ лӢӨлҘё кІҪлЎңм—җм„ң л©”нҠёлҰӯмқ„ м ңкіөн•ҳлҠ” кІҪмҡ° мқҙ мЈјм„қмқ„ м—…лҚ°мқҙнҠён•ҳм„ёмҡ”. мЈјм„қмқҙ м—Ҷмңјл©ҙ л Ҳмқҙлё” мһ¬м§Җм • к·ңм№ҷмқҙ мӢӨн–үлҗҳм§Җ м•Ҡкі  кІҪлЎңк°Җ м–ҙмЁҢл“  `/metrics` (мңј)лЎң кё°ліё м„Өм •лҗҳм§Җл§Ң вҖ• м•”мӢңм Ғмқё кІғліҙлӢӨ лӘ…мӢңм Ғмқё кІғмқҙ мўӢмҠөлӢҲлӢӨ.

        **нҢҢл“ң н…ңн”ҢлҰҝмқҳ мЈјм„қ л°°м№ҳ.** мқҙлҹ¬н•ң мЈјм„қмқҖ л°°нҸ¬, StatefulSet лҳҗлҠ” нҢҢл“ң л§ӨлӢҲнҺҳмҠӨнҠёмқҳ `spec.template.metadata.annotations` м—җ мҶҚн•©лӢҲлӢӨ вҖ• л°°нҸ¬/StatefulSet мһҗмІҙмқҳ мөңмғҒмң„ `metadata.annotations` м—җлҠ” мҶҚн•ҳм§Җ м•ҠмҠөлӢҲлӢӨ. `kubernetes_sd` лҠ” м»ЁнҠёлЎӨлҹ¬ л©”нғҖлҚ°мқҙн„°к°Җ м•„лӢҢ нҢҢл“ң мҲҳмӨҖ л©”нғҖлҚ°мқҙн„°лҘј мқҪмҠөлӢҲлӢӨ. нқ”нһҲ н•ҳлҠ” мӢӨмҲҳлҠ” л°°нҸ¬ мһҗмІҙмқҳ л©”нғҖлҚ°мқҙн„° лё”лЎқм—җ мЈјм„қмқ„ л°°м№ҳн•ҳлҠ” кІғмңјлЎң, мқҙлЎң мқён•ҙ Collectorк°Җ нғҖкІҹмқ„ н•ҳлӮҳлҸ„ л°ңкІ¬н•ҳм§Җ лӘ»н•ҳкІҢ лҗ©лӢҲлӢӨ.

        **`qmgr` л Ҳмқҙлё” мҡ”кө¬ мӮ¬н•ӯ.** mq-metric-samples мқөмҠӨнҸ¬н„°лҠ” нҒҗ кҙҖлҰ¬мһҗ мқҙлҰ„(мҳҲ: `qmgr="QM1"`)мңјлЎң м„Өм •лҗң `qmgr` л Ҳмқҙлё”кіј н•Ёк»ҳ л©”нҠёлҰӯмқ„ лӮҙліҙлӮҙм•ј н•©лӢҲлӢӨ. мқҙкІғмқҖ н‘ңмӨҖ IBM MQ Prometheus мқөмҠӨнҸ¬н„°мқҳ кё°ліё лҸҷмһ‘мһ…лӢҲлӢӨ вҖ• мқҙ л Ҳмқҙлё”мқ„ м–өм ңн•ҳкұ°лӮҳ мқҙлҰ„мқ„ л°”кҫём§Җ л§ҲмӢӯмӢңмҳӨ. лүҙл җлҰӯмқҳ м—”нӢ°нӢ° н•©м„ұ к·ңм№ҷмқҖ `qmgr` мқ„(лҘј) мқҪм–ҙ л©”нҠёлҰӯмқ„ мІЁл¶Җн•  `IBMMQ_MANAGER` м—”нӢ°нӢ°лҘј кІ°м •н•©лӢҲлӢӨ. мқҙ л Ҳмқҙлё”мқҙ лҲ„лқҪлҗҳкұ°лӮҳ мқҙлҰ„мқҙ ліҖкІҪлҗҳл©ҙ л©”нҠёлҰӯмқҙ лүҙл җлҰӯм—җ лҸ„м°©н•ҳм§Җл§Ң `IBMMQ_MANAGER` м—”нӢ°нӢ°к°Җ н•©м„ұлҗҳм§Җ м•Ҡмңјл©° IBM MQ лҢҖмӢңліҙл“ңм—җ л©”нҠёлҰӯмқҙ н‘ңмӢңлҗҳм§Җ м•ҠмҠөлӢҲлӢӨ.

        **`queue` л Ҳмқҙлё” мҡ”кө¬ мӮ¬н•ӯ.** нҒҗ мҲҳмӨҖ л©”нҠёлҰӯмқҳ кІҪмҡ° мқөмҠӨнҸ¬н„°лҠ” нҒҗ мқҙлҰ„кіј н•Ёк»ҳ `queue` л Ҳмқҙлё”мқ„ лӮҙліҙлӮҙм•ј н•©лӢҲлӢӨ. мқҙкІғмқҖ лҳҗн•ң кё°ліё лҸҷмһ‘мһ…лӢҲлӢӨ. лүҙл җлҰӯмқҖ `(target.name, qmgr, queue)` нҠёлҰ¬н”Ңм—җм„ң `IBMMQ_QUEUE` м—”нӢ°нӢ°лҘј н•©м„ұн•©лӢҲлӢӨ. `queue` л Ҳмқҙлё”мқ„ н•„н„°л§Ғн•ҳл©ҙ лӘЁл“  нҒҗ л©”нҠёлҰӯмқҙ лӢЁмқј мқөлӘ… м—”нӢ°нӢ°лЎң м¶•мҶҢлҗ©лӢҲлӢӨ.

        **мқөмҠӨнҸ¬н„° л Ҳмқҙлё”м—җм„ң к°Җм ёмҳЁ QMлі„ ID.** CollectorлҠ” нҒҗ кҙҖлҰ¬мһҗк°Җ лӘҮ к°ңлӮҳ мЎҙмһ¬н•ҳлҠ”м§Җ лҳҗлҠ” мқҙлҰ„мқҙ л¬ҙм—Үмқём§Җ м•Ң н•„мҡ”к°Җ м—ҶмҠөлӢҲлӢӨ. к°Ғ нҢҢл“ңмқҳ мқөмҠӨнҸ¬н„°лҠ” лӮҙліҙлӮҙлҠ” лӘЁл“  л©”нҠёлҰӯм—җ `qmgr` л Ҳмқҙлё”мқ„ нҸ¬н•Ён•ҳл©°, CollectorлҠ” н•ҙлӢ№ л Ҳмқҙлё”мқ„ ліҖкІҪн•ҳм§Җ м•Ҡкі  к·ёлҢҖлЎң м „лӢ¬н•©лӢҲлӢӨ. лӢЁмқј нҢҢмқҙн”„лқјмқёмқҖ лӘЁл“  QMмқҙ м–ҙлҠҗ Collector нғҖкІҹм—җм„ң мҷ”лҠ”м§Җк°Җ м•„лӢҲлқј мһҗмІҙ л©”нҠёлҰӯ л Ҳмқҙлё”м—җ мқҳн•ҙ мҷ„м „нһҲ кө¬лі„лҗҳкё° л•Ңл¬ём—җ лӘЁл“  QMмқ„ мІҳлҰ¬н•©лӢҲлӢӨ. л”°лқјм„ң нҒҗ кҙҖлҰ¬мһҗлҘј м¶”к°Җн•ҳлҠ” кІғмқҖ мЈјм„қмқҙ лӢ¬лҰ° лӢӨлҘё нҢҢл“ңлҘј л°°нҸ¬н•ҳлҠ” кІғмқј лҝҗмһ…лӢҲлӢӨ вҖ• `values.yaml` ліҖкІҪ, Helm м—…к·ёл Ҳмқҙл“ң, Collector мһ¬мӢңмһ‘мқҙ м—ҶмҠөлӢҲлӢӨ.

        **мӢӨн–ү мӨ‘мқё нҢҢл“ңм—җм„ң мЈјм„қ нҷ•мқё:**

        ```bash
        kubectl -n ibmmq get pod <your-qm-pod-name> -o jsonpath='{.metadata.annotations}' | python3 -m json.tool
        ```

        `prometheus.io/scrape`, `prometheus.io/port` л°Ҹ `prometheus.io/path` к°Җ лӘЁл‘җ мЎҙмһ¬н•ҳкі  мҳ¬л°”лҘё к°’мңјлЎң м„Өм •лҗҳм—ҲлҠ”м§Җ нҷ•мқён•©лӢҲлӢӨ.
      </Collapser>
    </CollapserGroup>
  </Step>

  <Step>
    ### HelmмңјлЎң мҲҳм§‘кё° м„Өм№ҳ [#install-helm]

    OpenTelemetry Helm м ҖмһҘмҶҢлҘј м¶”к°Җн•ҳкі  мқҙм „ лӢЁкі„м—җм„ң л§Ңл“  `values.yaml` мқ„(лҘј) мӮ¬мҡ©н•ҳм—¬ `ibmmq` л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ мҲҳм§‘кё°лҘј м„Өм№ҳн•©лӢҲлӢӨ:

    ```bash
    helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts
    helm repo update

    helm upgrade --install ibmmq-collector open-telemetry/opentelemetry-collector \
      --namespace ibmmq \
      --create-namespace \
      --values values.yaml
    ```

    мҲҳм§‘кё° нҢҢл“ңк°Җ `1/1 Running`м—җ лҸ„лӢ¬н•ҳм§Җ лӘ»н•ҳлҠ” кІҪмҡ°, м•„лһҳмқҳ [л¬ём ң м§„лӢЁ, н•ҙкІ°](#troubleshooting) мқ„ м°ёмЎ°н•ҳм„ёмҡ”.
  </Step>

  <Step>
    ### л°°нҸ¬ нҷ•мқё [#verify]

    мҲҳм§‘кё° нҢҢл“ңк°Җ мӢӨн–ү мӨ‘мқём§Җ нҷ•мқён•©лӢҲлӢӨ:

    ```bash
    kubectl -n ibmmq rollout status deploy/ibmmq-collector-opentelemetry-collector --timeout=180s
    kubectl -n ibmmq get pods
    ```

    мҲҳм§‘кё° нҢҢл“ңм—җ `1/1 Running`мқҙ(к°Җ) н‘ңмӢңлҗҳм–ҙм•ј н•©лӢҲлӢӨ. мӢӨм ңлЎң нҢҢл“ңлҘј кІҖмғүн•ҳкі  мҠӨнҒ¬лһ©н•ҳлҠ”м§Җ нҷ•мқён•ҳл Өл©ҙ мһҗмІҙ н…”л Ҳл©”нҠёлҰ¬ м—”л“ңнҸ¬мқёнҠё(мқёл°”мҡҙл“ң м„ңл№„мҠӨк°Җ л№„нҷңм„ұнҷ”лҗҳм–ҙ мһҲмңјлҜҖлЎң `:8888`м—җ л…ём¶ңлҗҳкі  `port-forward` мқ„(лҘј) нҶөн•ҙм„ңл§Ң м—°кІ° к°ҖлҠҘ)лҘј нҸ¬нҠё нҸ¬мӣҢл”©н•ҳкі  н—Ҳмҡ©/лӮҙліҙлӮё м№ҙмҡҙн„°лҘј нҷ•мқён•©лӢҲлӢӨ:

    ```bash
    kubectl -n ibmmq port-forward deploy/ibmmq-collector-opentelemetry-collector 8888:8888 &
    curl -s http://localhost:8888/metrics | \
      grep -E 'otelcol_(receiver_accepted|exporter_sent|exporter_send_failed)_metric_points'
    kill %1 2>/dev/null
    ```

    `otelcol_receiver_accepted_metric_points` 0ліҙлӢӨ нҒ° к°’мқҖ мҲҳм§‘кё°к°Җ мЈјм„қмқҙ лӢ¬лҰ° мӢӨн–ү мӨ‘мқё нҢҢл“ңлҘј н•ҳлӮҳ мқҙмғҒ м°ҫм•„ мҠӨнҒ¬лһ©н–ҲмқҢмқ„ нҷ•мқён•©лӢҲлӢӨ, `otelcol_exporter_send_failed_metric_points` мқҖ(лҠ”) `0` м—җ мң м§Җлҗҳм–ҙм•ј н•©лӢҲлӢӨ(0мқҙ м•„лӢҢ к°’мқҖ OTLP м—°кІ° лҳҗлҠ” мһҗкІ© мҰқлӘ… л¬ём ңлҘј лӮҳнғҖлғ„).

    к·ёлҹ° лӢӨмқҢ [лҚ°мқҙн„° м°ҫкё° л°Ҹ мҝјлҰ¬](/docs/opentelemetry/integrations/ibm-mq/view-data)мқҳ нҷ•мқё мҝјлҰ¬лҘј мӮ¬мҡ©н•ҳм—¬ лүҙл җлҰӯм—җм„ң IBM MQ л©”нҠёлҰӯ л°Ҹ м—”нӢ°нӢ°лҘј нҷ•мқён•©лӢҲлӢӨ. мғҒнғң к°’мқҳ мқҳлҜёлҠ” [MQ мғҒнғң мҪ”л“ң м°ёмЎ°](/docs/opentelemetry/integrations/ibm-mq/metrics#status-codes)лҘј нҷ•мқён•ҳм„ёмҡ”.
  </Step>

  <Step>
    ## New Relicм—җм„ң лҚ°мқҙн„° ліҙкё° [#find-data]

    мҲҳм§‘кё° нҢҢл“ңк°Җ мӢӨн–үлҗҳкі  л©”нҠёлҰӯмқҙ нқҗлҘҙкё° мӢңмһ‘н•ҳл©ҙ лүҙл җлҰӯм—җм„ң нҒҗ кҙҖлҰ¬мһҗк°Җ `IBMMQ_MANAGER` м—”нӢ°нӢ°лЎң н‘ңмӢңлҗҳкі  н•ҙлӢ№ нҒҗк°Җ н•ҳмң„ `IBMMQ_QUEUE` м—”нӢ°нӢ°лЎң н‘ңмӢңлҗ©лӢҲлӢӨ. лҚ°мқҙн„° м°ҫкё°, мҝјлҰ¬ мӢӨн–ү, лҢҖмӢңліҙл“ң л°Ҹ м•ҢлҰј м„Өм •м—җ лҢҖн•ң мһҗм„ён•ң лӮҙмҡ©мқҖ [лҚ°мқҙн„° ліҙкё° л°Ҹ мҝјлҰ¬](/docs/opentelemetry/integrations/ibm-mq/view-data)лҘј м°ёмЎ°н•ҳм„ёмҡ”.
  </Step>
