## 레디스 데이터 보기 [#find-and-use]

수집기가 실행되고 데이터를 전송하면 뉴렐릭에서 레디스 메트릭을 볼 수 있습니다:

1. 이동 <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Integrations &amp; Agents**</DNT>
2. 검색 <DNT>**Redis (OpenTelemetry)**</DNT>
3. **Installation plan** \[설치 계획]에서 설치 단계를 보려면 **See installation docs** \[설치 문서 보기]를 클릭합니다. 그렇지 않으면 **Skip this step** \[이 단계 건너뛰기]를 클릭합니다.
4. <DNT>**See your data** </DNT>을(를) 클릭합니다.

레디스 상태, 성능 메트릭 및 리소스 사용량을 보여주는 대시보드가 표시되어야 합니다. Kubernetes에서 레디스를 모니터링하는 경우 대시보드에 Kubernetes 관련 컨텍스트가 포함됩니다.

또는

1. 이동 <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Integrations &amp; Agents**</DNT>
2. **All** 탭에서 **대시보드**를 클릭합니다. 레디스 OTel을 검색하면 대시보드를 볼 수 있습니다.

또는

1. 왼쪽 패널에서 <DNT>**Dashboards** </DNT>클릭하세요.
2. 레디스 OTel 대시보드를 검색하면 대시보드를 볼 수 있습니다.

<Callout variant="tip">
  **데이터가 보이지 않나요?** 데이터가 표시되는 데 몇 분 정도 소요될 수 있습니다. 10분이 지나도 지표가 표시되지 않으면 수집기 로그를 확인하고 설정을 확인하세요.
</Callout>

## 알림 설정 [#alerts]

사전 예방적 모니터링과 알림 기능을 통해 문제가 사용자에게 영향을 미치기 전에 문제를 파악할 수 있습니다. 뉴렐릭에서 공지 조건을 생성하려면:

1. 이동 <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Alerts** &gt; **Alert Conditions**</DNT>
2. **Create condition** \[조건 생성]을클릭하세요
3. [**Guided mode**](/docs/alerts/create-alert/create-alert-condition/alert-conditions/) \[안내 모드] 또는 [**NRQL**](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/) 쿼리 빌더를 사용하여 공지를 구성합니다

강력한 레디스 모니터링을 위해 아래의 공지 설정이 권장됩니다:

### 중요 알림 [#essential-alerts]

이 알림은 데이터 손실이나 서비스 중단을 유발할 수 있는 중요한 레디스 상태 문제를 모니터합니다:

<CollapserGroup>
  <Collapser id="high-memory" title="높은 메모리 사용률 공지">
    `redis.maxmemory` 대비 메트릭 `redis.memory.used` 이(가) 최소 5분 동안 90%를 초과합니다. 메모리가 고갈되면 레디스는 쓰기 명령 수락을 중지합니다.
  </Collapser>

  <Collapser id="rejected-connections" title="거부된 연결 공지">
    메트릭 `redis.connections.rejected` 이(가) 최소 1분 동안 0보다 큽니다. `maxclients` 한도에 도달하면 레디스는 새 연결을 거부합니다.
  </Collapser>

  <Collapser id="replicas-dropped" title="연결된 복제본 감소 공지">
    메트릭 `redis.slaves.connected` 이(가) 최소 2분 동안 예상 기준 아래로 떨어집니다. 복제본 오류는 읽기 용량을 줄이고 데이터 내구성을 손상시킵니다.
  </Collapser>

  <Collapser id="high-evictions" title="높은 축출률 공지">
    `redis.keys.evicted` 의 비율이 최소 5분 동안 초당 1,000을 초과합니다. 높은 축출률은 레디스가 메모리 압박을 받고 있음을 의미합니다.
  </Collapser>
</CollapserGroup>

### 추가 모니터링 알림 [#additional-alerts]

이러한 알림은 성능 및 운영 문제를 모니터링하는 데 도움이 됩니다.

<CollapserGroup>
  <Collapser id="low-hit-ratio" title="낮은 캐시 히트 비율 공지">
    적중률(`redis.keyspace.hits` /(`redis.keyspace.hits` + `redis.keyspace.misses`))이 최소 5분 동안 50% 미만입니다.
  </Collapser>

  <Collapser id="high-fragmentation" title="높은 단편화 비율 공지">
    메트릭 `redis.memory.fragmentation_ratio` 이(가) 최소 10분 동안 1.5를 초과합니다. 과도한 메모리 단편화는 리소스를 낭비합니다.
  </Collapser>

  <Collapser id="call-latency" title="APM 공지의 레디스 호출 지연시간">
    최소 5분 동안 `db.system = redis` 이(가) 10ms를 초과하는 평균 스팬 기간입니다. 애플리케이션 서비스의 레디스 호출 지연시간이 길면 레디스 또는 네트워크 문제일 수 있습니다.
  </Collapser>
</CollapserGroup>
