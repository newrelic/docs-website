## OpenTelemetry로 HAProxy를 모니터링하는 이유 [#why-monitor]

로드 밸런서 성능, 백앤드 서버 상태 및 애플리케이션 안정성을 유지하려면 HAProxy를 모니터링하세요. OpenTelemetry는 이 데이터를 수집하고 분석하기 위한 표준화된 공급업체 비종속 접근 방식을 제공합니다.

### 주요 이점

* **차원 메트릭**: 풍부한 메타데이터(프록시 이름, 서비스 이름, 서버 주소)를 통해 유연한 그룹화 및 필터링 가능
* **공급업체 비종속 수집**: 오픈 소스 OpenTelemetry 표준은 종속을 방지하고 기존 옵저버빌리티 파이프라인과 통합됩니다.
* **유연한 수집 방법**: NRDOT 또는 OTel Collector Contrib를 통해 배포되는 haproxyreceiver(CSV 통계) 또는 Prometheus 스크래핑 중에서 선택합니다.
* **Kubernetes에서의 자동 검색**: 수집기는 각 인스턴스에 대한 수동 설정 없이 HAProxy 파드를 자동으로 검색합니다.
* **백앤드 상태 가시성**: 전체 로드 밸런서 플릿에서 백앤드 서버 상태, 대기열 깊이 및 연결 오류를 추적합니다.

### 사용 사례

HAProxy를 로드 밸런서, 리버스 프록시 또는 API 게이트웨이로 실행하든 관계없이 이 모니터링은 다음을 지원합니다:

* 백앤드 서버가 다운될 때 알림 받기
* 세션 한도에 도달하는 시점 감지
* 오류 급증을 겪고 있는 프런트엔드를 식별합니다.
* 서버 풀 전체의 지연시간 증가를 확인합니다

이 통합은 고가용성 배포, 마이크로서비스 아키텍처 및 HAProxy가 사용자 트래픽의 중요 경로에 있는 모든 환경을 위한 것입니다.

## 지원되는 플랫폼 [#supported-platforms]

* 아마존 리눅스
* Red Hat Enterprise Linux (RHEL)
* CentOS
* 우분투
* 데비안

## 수집 옵션 [#collection-options]

다음 방법 중 하나를 사용하여 HAProxy 메트릭을 수집할 수 있습니다:

* **NRDOT Collector** (권장): OpenTelemetry의 뉴렐릭 배포판 ― 사전 구성된 HAProxy 모니터링이 포함된 안내 설치(자체 호스팅) 또는 Helm 차트(Kubernetes)를 통한 자동 설정
* **OTel Collector Contrib**: [haproxyreceiver](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/haproxyreceiver) 를 사용한 수동 설정 ― HAProxy 통계 엔드포인트에서 메트릭을 수집합니다.
* **Prometheus Receiver**: 이미 HAProxy의 내장 [Prometheus 엔드포인트](https://www.haproxy.com/blog/haproxy-exposes-a-prometheus-metrics-endpoint) 를 사용 중인 환경의 경우(HAProxy 2.0+)

## 사용 가능한 지표 [#metrics]

haproxyreceiver는 HAProxy 통계 엔드포인트에서 최대 27개의 메트릭을 수집합니다. 각 메트릭에는 프런트엔드별, 백앤드별 및 서버별 가시성을 위한 `haproxy.proxy_name` 및 `haproxy.service_name` 과 같은 차원 속성이 포함됩니다.

**수집된 주요 지표:**

* **세션 메트릭**: 현재 세션, 세션 비율, 프런트엔드/백앤드/서버당 평균 세션 시간
* **요청 메트릭**: 총 requests, requests 비율, 거부된 requests, requests 오류, 대기 중인 requests
* **연결 메트릭**: 연결 오류, 재시도, 연결 비율, 총 연결
* **처리량 메트릭**: 프런트엔드, 백앤드 및 개별 서버당 수신/송신 바이트
* **백앤드 상태**: 활성/백업 서버 수, 실패한 상태 확인, 다운타임

전체 메트릭 및 속성 참조는 [HAProxy OTel 메트릭 참조](/docs/opentelemetry/integrations/haproxy/metrics-reference/)를 확인하세요.
