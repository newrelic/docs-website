[OpenTelemetry 수집기는](https://opentelemetry.io/docs/collector/) 벤더에 구애받지 않는 텔레메트리 데이터를 수신, 처리 및 내보내기 위한 도구입니다. 수집기 요구 사항과 설정은 다양하지만 다양한 수신기와 프로세서가 함께 제공되어 인텔리전스 모니터링에 인기가 높습니다. 데이터 처리를 위해 수집기를 사용하는 것도 일반적이지만 이 문서에서는 모니터링 사용 사례에 중점을 둡니다. 자세한 내용은 [데이터 처리에 대한OpenTelemetry 수집기를](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) 참조하세요.

## 텔리메트리 수신 중 [#infrastructure-receivers]

## OpenTelemetry APM과 인프라 상호 연결 [#infrastructure-correlation]

다음 예에서는 수집기를 사용하여 [APM OpenTelemetry](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro) 와 Insight 데이터를 연관시키는 방법을 보여줍니다.

* [호스트 리포지터리 예시](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/host-monitoring)

일반적인 패턴은 OTLP를 통해 데이터를 뉴럴릭으로 내보내기 전에 리소스 속성 형태의 추가 환경 컨텍스트로 APM 텔레메트리를 감지하고 강화하는 프로세서로 수집기를 구성하는 것입니다. 뉴렐릭은 이 상관 데이터를 감지하고 리소스를 APM 통해 과 관측 엔터티 [간의](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources) 관계를 구성합니다.

## Kubernetes 모니터링을 위한 OpenTelemetry Collector [#k8s-monitoring]

Collector 요구 사항과 설정은 다양하지만 다양한 수신기와 프로세서가 포함되어 있어 Kubernetes Monitoring에 널리 사용됩니다. 필요한 텔레메트리를 모두 얻으려면 각기 다른 권한 요구 사항이 있는 여러 수신기를 사용하여 다양한 Kubernetes API를 스크래핑해야 하므로 작업을 간편하게 수행할 수 있도록 후드 아래에서 수집기 [nrdot-수집기-k8s](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s) 의 사용자 지정 배포를 사용하는 헬름 차트를 만들었습니다. 시작하는 방법에 대한 자세한 내용은 [Kubernetes용 OpenTelemetry를](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/) 참조하세요.
