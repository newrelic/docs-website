OpenTelemetry 프로젝트는 [API 및 SDK 사양](https://opentelemetry.io/docs/specs/otel/), [계측](https://opentelemetry.io/docs/concepts/instrumentation/), [수집기](https://opentelemetry.io/docs/collector/), [의미 규칙](https://opentelemetry.io/docs/specs/semconv/), [OTLP](https://opentelemetry.io/docs/specs/otlp/) 프로토콜 등 그 범위가 매우 넓습니다. 뉴렐릭은 OpenTelemetry에 최고 수준의 지원을 제공하며 OpenTelemetry 도구를 쉽게 사용할 수 있게 해주는 다양한 플랫폼 기능을 갖추고 있습니다.

이 문서는 뉴렐릭과 일반적인 OpenTelemetry 통합의 패턴들에 대해 설명합니다. 가능한 모든 사례가 포함되지는 않았지만, OpenTelemetry가 주로 사용되는 방식과 이러한 작업 흐름 위에 구현되는 주요 뉴렐릭 플랫폼 기능들을 이해하는 데 도움이 될 것입니다.

[OpenTelemetry APM 모델링](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro)은 OpenTelemetry를 사용해 [뉴렐릭 에이전트를 사용한 APM 모니터링](/docs/apm/new-relic-apm/getting-started/introduction-apm/)과 유사한 APM 모니터링을 수행하는 방법을 설명합니다.

[인프라 모니터링을 위한 OpenTelemetry 수집기](/docs/opentelemetry/get-started/collector-infra-monitoring/opentelemetry-collector-infra-intro)에는 인프라 타겟 스크래핑/폴링, OpenTelemetry APM 데이터와 인프라 간의 상호 연관 등 인프라 모니터링에 OpenTelemetry 수집기를 사용하는 방법을 설명하는 다양한 예가 포함되어 있습니다.

[데이터 처리를 위한 OpenTelemetry 수집기](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro)는 텔레메트리 데이터를 수신, 처리(강화, 변환, 샘플링, 필터링 등), 내보내기 위한 일반적인 도구로 OpenTelemetry 수집기를 사용할 때 필요한 기본적인 요소들을 설명합니다.
