New Relic의 파트너 중 한 곳을 통해 New Relic에 가입한 경우 New Relic을 사용하려면 로그인하기 위해 추가 SSO(Single Sign-On) 절차를 따라야 할 수 있습니다. 배포 프로세스를 완료하는 데 특별한 단계가 필요할 수도 있습니다. 이 문서는 New Relic 파트너를 통해 생성된 계정에 대한 기본 로그인 절차와 해당되는 추가 리소스에 대한 링크를 제공합니다.

## New Relic 계정 생성(모든 파트너) [#partner-account]

New Relic 파트너로부터 특별 제안을 받은 경우 파트너의 링크를 따라 직접 New Relic 계정을 만드십시오.

## New Relic에 로그인 및 배포(대부분의 파트너) [#partner-users]

계정을 확인하고, New Relic에 로그인하고, 배포 프로세스를 완료할 수 있는 링크가 포함된 이메일 확인 메시지를 받게 됩니다.

## New Relic(특별 파트너) 로그인 및 배포 [#partner-users]

이러한 파트너는 New Relic 계정을 사용하기 위해 특정 로그인 및 배포 절차가 필요합니다.

<Callout variant="tip">
  여기에 나열되지 않은 파트너를 통해 New Relic 계정에 가입한 경우 [표준 절차에 따라](#partner-users) New Relic에 로그인하고 배포하세요.
</Callout>

### A - M 파트너 [#a-m-partners]

<CollapserGroup>
  <Collapser

    앱용으로 AWS를 설치할 것인지 호스트용으로 설치할 것인지에 따라 다른 설치 및 페이지 보기 절차가 적용됩니다. 자세한 내용은 [Amazon Web Services(AWS) 사용자](/docs/accounts-partnerships/partnerships/partner-based-installation/amazon-web-services-aws-users) 를 참조하십시오.
  </Collapser>

  <Collapser
    id="appharbor"
    title="구글 클라우드 플랫폼(GCP)"
  >
    [Google Cloud Platform (GCP) 파트너](https://newrelic.com/partner/gcp) 로서 뉴렐릭은 [App Engine 가변형 환경을](https://cloud.google.com/appengine/docs/flexible) 지원합니다. 각 <InlinePopover type="apm"/>에이전트에는 [GAE Flex 환경에 대한 자체 요구사항이](/docs/accounts-partnerships/partnerships/google-cloud-platform-gcp/google-app-engine-environment) 있습니다. 뉴렐릭 에이전트를 설치한 후 [APM](/docs/apm/new-relic-apm/getting-started/introduction-new-relic-apm) 으로 앱 성능을 볼 수 있습니다.
  </Collapser>

  <Collapser
    id="heroku"
    title="헤로쿠"
  >
    [Heroku](https://www.heroku.com/) 는 웹 호스팅이 가능한 PaaS(Platform as a Service) 솔루션입니다. 뉴렐릭 Heroku 추가 기능을 사용하면 <InlinePopover type="apm"/>및 <InlinePopover type="browser"/>의 지표로 Heroku 확장할 수 있습니다. 뉴렐릭 추가 기능은 Java, Node.js, PHP, 파이썬, 루비. Heroku 와 함께 뉴렐릭을 설치하고 사용하는 방법에 대한 자세한 내용은 다음을 참조하세요.

    * [Heroku 개발자 센터 문서](https://devcenter.heroku.com/articles/newrelic "링크가 새 창에서 열립니다.")
    * [Heroku 설치에 대한 New Relic의 문서](/docs/accounts-partnerships/partnerships/heroku-new-relic/heroku-installing-new-relic)
  </Collapser>

  <Collapser
    id="magento"
    title="마젠토"
  >
    Magento는 New Relic과 파트너 관계를 맺어 판매자가 데이터 및 즉시 사용 가능한 도구에 더 빠르게 액세스할 수 있도록 했습니다. [New Relic Reporting Extension](http://www.magentocommerce.com/magento-connect/new-relic-reporting.html) 은 전자 상거래 비즈니스가 데이터 기반 의사 결정을 내리는 데 도움이 됩니다.

    PHP 앱용 [Magento](http://newrelic.com/magento) 로 New Relic 계정을 생성하려면:

    1. Magento 판매 솔루션 제공업체를 통해 [가입](https://newrelic.com/signup?funnel=magento-ce&partner=Magento+CE&product_id=Standard&promo_code=MAGCE30PRO) 하세요.

    2. New Relic 계정에 가입한 후 Magento 설치 [에 PHP 에이전트를 배포](/docs/agents/php-agent/installation/php-agent-installation-overview) 합니다.

       몇 분 정도 기다린 후 [APM <DNT>**Overview**</DNT> 페이지](/docs/apm/applications-menu/monitoring/applications-overview-dashboard) 에서 앱 성능을 확인하세요.
  </Collapser>
</CollapserGroup>

### N - Z 파트너 [#n-z-partners]

<CollapserGroup>
  <Collapser
    id="network-redux"
    title="판테온"
  >
    [Pantheon](https://pantheon.io/features/new-relic) 을 통해 New Relic 계정을 만들고 New Relic에 로그인하려면:

    1. [뉴렐릭이 있는 Pantheon의 파트너 페이지](https://pantheon.io/features/new-relic) 에서 <DNT>**sign up**</DNT> 링크를 선택하세요.

    2. 온라인 지침에 따라 설치 프로세스를 완료하십시오.

    3. New Relic이 앱에 대한 데이터 수집을 시작할 때까지 기다리세요.

       몇 분 후에 [APM <DNT>**Overview**</DNT> 페이지](/docs/apm/applications-menu/monitoring/applications-overview-dashboard) 에서 앱 성능을 확인할 수 있습니다. 자세한 내용은 [Pantheon 문서 를](https://pantheon.io/docs/new-relic/) 참조하세요.
  </Collapser>

  <Collapser
    id="rackspace"
    title="Rackspace 클라우드 도구"
  >
    [Rackspace Cloud Tools](https://www.rackspace.com) 를 통해 계정을 만들고 New Relic에 로그인하려면 다음 기본 절차를 따르세요. 자세한 내용은 [Rackspace Cloud Load Balancer 플러그인](/docs/accounts-partnerships/accounts/install-new-relic/partner-based-installation/rackspace-cloud-load-balancer-plugin) 을 참조하십시오.

    1. cloudtools.rackspace.com/myapps에서 [CloudTools](https://cloudtools.rackspace.com/myapps) 계정에 로그인합니다.

    2. <DNT>**Use Your Applications**</DNT> 에서 <DNT>**New Relic**</DNT> 선택합니다.

    3. 에이전트 언어(Java, .NET, PHP, Python, Ruby)를 선택한 다음 온라인 지침에 따라 설치 프로세스를 완료합니다.

    4. New Relic이 앱에 대한 데이터 수집을 시작할 때까지 기다리세요.

       몇 분 정도 기다린 후 [APM <DNT>**Overview**</DNT> 페이지](/docs/apm/applications-menu/monitoring/applications-overview-dashboard) 에서 앱 성능을 확인하세요.
  </Collapser>

  <Collapser
    id="wordpress-w3-edge"
    title="W3 엣지와 워드프레스"
  >
    <Callout variant="tip">
      [W3 Total Cache](https://wordpress.org/plugins/w3-total-cache/) 를 통해 계정을 생성하고 New Relic에 로그인하려면 API 키만 받는 것이 아니라 W3 PHP 에이전트를 설치하고 실행해야 합니다.
    </Callout>

    구현하다, 배포하다 에이전트에는 다음을 포함하여 <DNT>**administrative access**</DNT> 필요합니다.

    * SSH/명령줄 액세스

    * New Relic 설치 프로그램을 실행하고 웹 서버를 다시 시작하기 위한 루트 또는 sudo 액세스

      이코노미, 공유 호스팅 또는 블로거 패키지를 실행 중이거나 계정 권한이 무엇인지 확실하지 않은 경우 호스팅 제공업체에 문의하십시오. 예를 들어 WordPress 사이트가 공유 서버에 있는 경우 New Relic을 설치하지 못할 수 있습니다. 공급자는 계정 권한에 대한 정보를 제공하거나 필요한 경우 계정을 업그레이드할 수 있어야 합니다.

      W3 Edge/Wordpress 사용자를 위한 New Relic 플러그인을 설치하려면:

    1. [wordpress.org/support/bb-login.php](https://wordpress.org/support/bb-login.php) 에서 WordPress 계정에 로그인합니다.

    2. [New Relic의 지침](/docs/agents/php-agent/installation/wordpress-users-new-relic) 에 따라 설치 프로세스를 완료하거나 [W3TC 지원](http://wordpress.org/support/plugin/w3-total-cache) 에 도움을 요청하십시오.

    3. New Relic이 데이터 수집을 시작할 때까지 기다리십시오.

       에이전트를 배포한 후 몇 분 안에 앱에 대한 성능 데이터가 APM의 [애플리케이션 개요 페이지](/docs/apm/applications-menu/monitoring/applications-overview-dashboard) 에 나타납니다.

       <Callout variant="tip">
         [WordPress](https://wordpress.org/plugins/rt-newrelic-browser/) 는 [브라우저 모니터링](/docs/agents/php-agent/installation/wordpress-users-new-relic#browser) 을 위한 플러그인도 제공합니다.
       </Callout>
  </Collapser>
