수집 및 청구 가능 사용자의 기본 청구 요소 외에도 선택적 청구 가능 추가 기능을 사용하여 New Relic 사용 경험을 향상할 수도 있습니다.

* <DNT>
    **Compute Add On**
  </DNT>

  : 이 추가 기능은 대화형 애플리케이션 보안 테스트, 라이브 아카이브 및 기타 제공되는 추가 기능에 대해 실행하는 쿼리에 대한 사용량 기반 청구에 적용됩니다. [컴퓨팅 단위(CCU)(CCU)를](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions#compute-capacity-unit) 기준으로 사용량을 측정합니다.

* <DNT>
    **EU Data Center for New Relic - Data, and New Relic - Data Plus**
  </DNT>

  : 이 추가 기능은 유럽 연합을 데이터 지역으로 선택하는 경우 적용됩니다.

* <DNT>
    **Extended Retention for New Relic - Data, and New Relic - Data Plus**
  </DNT>

  : 이 추가 기능은 데이터 보관 기본 기간을 초과하는 경우 적용됩니다. 이는 로그뿐만 아니라 모든 데이터에 적용되며, 소규모 쿼리를 많이 수행하거나 대용량 데이터에 대해 쿼리를 수행해야 하는 경우 좋은 옵션입니다.

* <DNT>
    **Live Archives**
  </DNT>

  : 로그 저장 기간을 표준 보존보다 최소 90일 이상 연장합니다. 이 추가 기능은 자주 쿼리하지 않는 로그를 보관해야 하는 경우에 가장 적합합니다. 라이브 아카이브에는 컴퓨트 애드온도 필요합니다.

* <DNT>
    **New Relic Synthetic Checks**
  </DNT>

  : 이 추가 기능은 수표가 신세틱 모니터 검사의 기본 수를 초과하는 경우 적용됩니다.

* <DNT>
    **Vulnerability Management**
  </DNT>

  : 이 기능은 데이터 플러스에 포함되어 있지만, 데이터 플러스 없이 취약점 관리 기능을 사용하는 경우 추가 기능이 적용됩니다.

이러한 선택적 추가 기능을 시작하려면 New Relic 계정 팀에 문의하세요.
