[가격 책정 모델](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) 에서 청구 가능한 사용자 수는 청구 요소입니다.

## 청구 가능한 사용자란 무엇입니까? [#billable-users]

청구 가능 사용자는 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 이 코어 사용자 또는 전체 플랫폼 사용자인 사용자입니다. 기본 사용자는 무료입니다.

사용자가 청구할 수 있는 이유에 대한 기타 세부정보:

* [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 사용자의 경우: 사용자는 New Relic 조직에 청구 가능한 사용자로 추가될 때 청구 가능합니다. 청구 가능 사용자가 New Relic에 로그인한 적이 없고 UI에 `Pending invite` 태그가 있더라도 여전히 청구 가능합니다.
* [기본 사용자 청구 버전](#pricing-versions) 의 경우: 청구 가능한 사용자에 대한 월별 요금이 있으며 비례 배분되지 않습니다. 즉, 사용자를 한 달 내내 청구 가능한 사용자 유형으로 유지하든 해당 월 동안 1분만 유지하든 비용은 동일합니다. [사용자 계산에 대해 자세히 알아보세요.](#user-count)
* [Standard 에디션](https://newrelic.com/pricing) 은 한 명의 무료 전체 플랫폼 사용자와 함께 제공됩니다.

## UI에서 사용자 보기 [#ui]

UI에서 청구 가능한 사용자 수를 확인하는 방법은 [청구 관련 UI](/docs/accounts/accounts-billing/general-account-settings/introduction-account-settings) 를 참조하십시오. 조직에 여러 계정이 있는 경우 조직의 모든 사용자를 보려면 기본 보고 계정(일반적으로 조직에서 생성된 첫 번째 계정)에 있어야 합니다.

UI에서 사용자를 관리하는 방법은 [사용자 관리](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks) 를 참조하십시오.

## 사용자 청구 버전 [#pricing-versions]

사용자 요금 청구 방법에는 두 가지 버전이 있습니다.

* **기본 사용자 청구 버전입니다.** 이것은 최신 버전입니다. 한 달에 청구 가능한 총 사용자 수를 기준으로 요금이 청구됩니다. 이것은 새로 생성된 모든 New Relic 조직이 가지고 있는 버전이며 귀하의 조직이 가지고 있을 가능성이 있는 버전입니다.
* **원래 사용자 청구 버전입니다.** 이것은 일부 조직에서 여전히 가지고 있는 이전 버전입니다.

조직의 버전에 따라 사용자 청구 방식과 일부 사용자 다운그레이드 규칙이 결정됩니다. 이러한 규칙은 아래에서 자세히 설명합니다.

사용 중인 버전이 확실하지 않습니까? 다음은 몇 가지 팁입니다.

* [결제 UI](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/billing-ui) 의 **요금제 요약** 페이지에 `monthly provisioned users` 가 표시되면 원래 버전을 사용 중인 것입니다.
* 인보이스에 `New Relic One - Pro Users` 또는 `New Relic One - Enterprise Users` 이(가) 표시되면 원래 버전을 사용 중인 것입니다. 여기에 사용자 유형(예: `New Relic - Pro Full Platform Users` )이 포함되어 있으면 최신 버전을 사용하고 있는 것입니다.

## 기본 사용자 결제 버전 [#primary-version]

[사용량 UI](/docs/accounts/accounts-billing/general-account-settings/introduction-account-settings/#pricing) 를 사용하여 청구 가능한 사용자 수에 대한 개요를 확인할 수 있습니다. UI가 제공하는 것보다 더 자세한 정보가 필요한 경우 [사용 관련 NRQL 쿼리](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/usage-queries-alerts) 를 실행할 수도 있습니다.

    <Callout variant="important">

    한 달에 청구 가능한 조직의 사용자 수를 결정하기 위해 해당 월에 전체 플랫폼 사용자 또는 핵심 사용자의 **청구 가능한 사용자 유형** 을 가진 사용자를 계산합니다.사용자의 **청구 가능한 사용자 유형** 은 한 달 동안 사용자가 설정된 가장 높은 사용자 유형으로 정의됩니다.UTC 시간대를 사용하여 달력 월의 시작과 끝을 정의합니다.

    이것이 실제로 어떻게 작동하는지에 대한 예: 사용자가 달력 월 중 어느 시점에서든 전체 플랫폼 사용자로 설정된 경우 해당 월에 청구 가능한 사용자 유형은 "전체 플랫폼 사용자"이며 다음과 같은 경우에도 변경되지 않습니다. 그들은 그 달 말에 다운그레이드합니다. 이는 해당 사용자가 전체 플랫폼 사용자로 잠시 변경되더라도 해당됩니다.

    청구 가능한 사용자를 추가하거나 사용자 유형을 변경할 계획이라면 이러한 규칙을 염두에 두어야 합니다. 몇 가지 팁:

    * 청구 가능한 사용자를 추가하거나 사용자를 업그레이드하려는 경우 월초에 수행하도록 선택할 수 있습니다.
    * 사용자를 다운그레이드하려면 월말에 다운그레이드할 수 있습니다.

    고유 사용자 수는 이메일 주소로 결정됩니다. 조직에 동일한 이메일 주소를 가진 여러 사용자 레코드가 있는 경우 청구 목적으로 해당 사용자 레코드는 단일 사용자로 계산되고 해당 사용자의 청구 가능 사용자 유형은 해당 월에 할당된 가장 높은 사용자 유형이 됩니다.

    [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 사용자: 관리자가 청구 가능한 사용자로 추가한 사용자는 청구할 수 있습니다. 청구 가능한 사용자가 New Relic에 로그인한 적이 없고 UI에 `Pending invite` 태그가 있더라도 여전히 청구 가능합니다. ( [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 의 사용자는 `Pending invite` 명의 사용자에게 청구할 수 없습니다.)

    사용자 추가 및 초대에 대한 자세한 내용은 사용자 [추가](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#add-users) 를 참조하십시오.

    청구 가능한 사용자의 비용은 조직의 [가격 책정 버전](https://newrelic.com/pricing) (Standard, Pro 또는 Enterprise) 또는 New Relic과의 맞춤 거래에 따라 다릅니다.

    New Relic 조직에 처음 요금이 청구되기 시작하면 청구 가능한 사용자 수는 시작한 달의 시기를 기준으로 일할 계산됩니다. 조직에서 구독을 취소하면 지난 달에 대해서도 일할 계산이 적용됩니다.
  </Collapser>

    전체 플랫폼 사용자를 다운그레이드할 수 있는 횟수와 관련된 규칙은 [사용량 계획](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing#usage-plans) 이 종량제인지 약정 계약인지에 따라 다릅니다.

    #### 종량제 다운그레이드 규칙

    [종량제 사용 계획](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing#usage-plans) 의 경우 사용자 다운그레이드를 제한하는 규칙은 없지만 사용자를 업그레이드하거나 다운그레이드하기로 결정할 때 [청구 영향](#user-count) 이 영향을 미칠 수 있습니다.

    #### 약정 계약 다운그레이드 규칙

    사용자를 업그레이드하거나 다운그레이드하기 전에 [결제에 미치는 영향](#user-count) 을 이해해야 합니다.

    [약정 계약](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#usage-plans) 의 경우 사용자가 전체 플랫폼 사용자에서 하위 사용자 유형으로 몇 번이나 다운그레이드될 수 있는지에 관한 규칙이 있습니다. (코어 사용자와 기본 사용자 간에 사용자가 얼마나 자주 변경되는지에 대한 규칙은 없습니다.)

    New Relic 조직의 계약 연도(아래에 정의됨) 동안 전체 플랫폼 사용자가 하위 사용자 유형으로 변경되었다가 다시 전체 플랫폼 사용자로 두 번 변경되면 해당 사용자는 해당 계약 연도의 나머지 기간 동안 전체 플랫폼 사용자로 청구됩니다. , 사용자 유형 조정에 관계없이.

    계약 연도는 계약 시작 시점 또는 해당 시점의 기념일에 시작하는 1년으로 정의됩니다. 조직에서 다른 요금제를 시작하고 이 버전의 요금제로 전환한 경우 사용자 유형 다운그레이드 규칙은 선택한 시점부터 구독 기간이 갱신될 때까지 또는 해당하는 경우 약정 기간의 연간 기념일까지 적용됩니다. , 둘 중 더 빠른 것.

일부 조직에서는 청구 가능한 사용자에 대한 계층형 가격 책정에 액세스할 수 있습니다. 이에 대한 자세한 내용은 [계층형 가격 책정](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing#tiered-pricing) 을 참조하십시오.

## 원래 사용자 결제 버전 [#original-version]

[원래 사용자 청구 버전](#pricing-versions) 에 대한 세부 정보는 다음과 같습니다.

<CollapserGroup>
  <Collapser
    id="original-user-count"
    title="청구 가능한 사용자 계산 방법"
  >
    다음 규칙은 원래 [사용자 결제 버전](#pricing-versions) 의 조직에만 적용됩니다.

    * **청구 가능한 사용자를 결정하는 방법입니다.** 역월의 경우 해당 월의 청구 가능한 사용자 수 계산을 기준으로 조직에 요금이 청구됩니다.
    * **첫 달과 마지막 달에 비례 배분합니다.** 청구 가능한 사용자 수는 New Relic 조직이 구독을 시작한 시점 또는 사용자가 전체 플랫폼 사용자가 된 시점(전체 플랫폼 사용자로 추가되거나 사용자로 전환된 시점)을 기준으로 일할 계산됩니다.
    * **사용자는 프로비저닝될 때 청구 가능합니다.** 사용자는 New Relic에서 청구 가능한 사용자 유형으로 설정되는 순간 청구 가능한 것으로 간주됩니다. 이는 해당 사용자가 New Relic에 로그인했거나 사용한 적이 있는지 여부에 관계없이 적용됩니다.
    * **이메일 주소를 기반으로 한 사용자 수입니다.** 동일한 이메일 주소를 가진 조직에 여러 사용자 레코드가 있는 경우 청구 목적으로 단일 사용자로 계산됩니다.
    * **원래 사용자 모델에 대한 경고입니다.** 조직에 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 사용자가 있는 경우: 사용자가 조직의 한 계정에서는 기본 사용자로 설정되고 다른 계정에서는 청구 가능 사용자로 설정된 경우 청구 가능 사용자 상태가 우선합니다.
  </Collapser>

  <Collapser
    id="original-user-downgrade-rules"
    title="사용자 다운그레이드 규칙"
  >
    다음 사용자 다운그레이드 규칙은 [원래 사용자 결제 버전](#pricing-versions) 의 조직에 적용됩니다.

    사용자 유형은 사용자의 기대되는 New Relic 의무 및 책임에 따라 상당히 장기적인 설정을 의미합니다. 이러한 이유로 전체 플랫폼 사용자는 12개월 동안 최대 2번만 다운그레이드될 수 있습니다. 사용자의 사용자 유형이 이 허용된 변경 수보다 많이 변경된 경우 New Relic은 해당 사용자에게 전체 플랫폼 사용자로 청구할 수 있습니다.
  </Collapser>
