[사용량 기반 가격 책정 모델](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) 의 경우 청구 가능한 사용자 수가 청구 요소입니다.

## 청구 가능한 사용자는 무엇입니까? [#billable-users]

청구 가능 사용자는 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 이 코어 사용자 또는 전체 플랫폼 사용자인 사용자입니다. 기본 사용자는 무료입니다.

[Standard 에디션](https://newrelic.com/pricing) 은 한 명의 무료 전체 플랫폼 사용자와 함께 제공됩니다.

## UI에서 사용자 수 보기 [#ui]

UI에서 청구 가능한 사용자 수를 보는 방법은 [청구 관련 UI](/docs/accounts/accounts-billing/general-account-settings/introduction-account-settings) 를 참조하세요.

## 가격 책정 버전: 핵심 사용자 유무 [#pricing-versions]

사용량 기반 가격 책정 모델에는 두 가지 버전이 있습니다.

* 핵심 사용자를 포함하는 **기본 사용량 기반 가격 책정 버전** 입니다. 이것은 2022년 1월 12일에 출시되었습니다. 이 버전은 세 번째 사용자 유형인 [핵심 사용자](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 에 대한 액세스 권한을 제공합니다. 또한 다른 [청구 계산](#user-count) 및 [사용자 다운그레이드 규칙](#user-downgrade-rules) 이 있습니다.
* **"핵심 사용자가 없는" 버전.** New Relic 조직이 2022년 1월 12일 이전에 존재했고 최신 버전으로 전환하지 않은 경우 이것이 귀하의 버전입니다. 이 버전의 조직은 기본 사용자와 전체 플랫폼 사용자만 추가할 수 있습니다. 핵심 사용자에 대한 액세스 권한이 없습니다. 이 버전의 고객이 핵심 사용자 버전으로 전환함에 따라 이 버전은 점점 더 사용되지 않습니다. [이 버전에 대한 규칙을](#non-core) 참조하십시오.

사용 중인 버전 확인을 위한 팁:

* 조직에 핵심 사용자가 있거나 핵심 사용자를 추가할 수 있는 기능이 있는 경우 기본 버전을 사용하고 있는 것입니다. 기본 사용자와 전체 플랫폼 사용자만 추가할 수 있다면 "핵심 사용자가 없는 New Relic One" 버전을 사용하고 있는 것입니다. [**사용자 관리** UI](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/#where) 로 이동하여 사용자를 볼 수 있습니다.
* 버전을 확인하는 또 다른 방법은 [**플랜 관리** UI](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/billing-ui) 로 이동하는 것입니다. "New Relic One - 사용자"가 표시되면 "핵심 사용자가 없는 New Relic One" 가격 책정 버전을 사용 중임을 의미합니다.

이러한 버전 간의 차이점에 대한 자세한 내용은 [핵심 사용자 릴리스](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/core-users-release) 를 참조하십시오.

## 계층형 가격 책정 [#tiered-pricing]

일부 Pro 및 Enterprise 에디션 조직은 청구 가능한 사용자(전체 플랫폼 사용자)에 대한 계층형 가격 책정에 액세스할 수 있습니다.가격과 계층 수준은 조직의 특정 계약에 따라 다릅니다.다음은 조직의 전체 플랫폼 사용자에게 계층화된 가격 책정이 작동하는 방식의 예입니다.

* 가격 x의 사용자 #1-10
* 가격 y의 사용자 #11-20(x보다 낮음)
* 가격 z의 사용자 #21-30(y보다 낮음)

하위 계층의 청구 가능한 사용자는 후속 사용자를 위해 잠금 해제된 다른 저렴한 계층의 수에 관계없이 해당 계층의 비용으로 계속 청구할 수 있습니다.위의 예에서 고객이 해당 월에 29명의 전체 플랫폼 사용자를 보유하고 있는 경우 해당 월에 청구 가능한 전체 플랫폼 사용자는 a) 10명의 사용자에 가격 x를 곱하고 b) 10명의 사용자에 y를 곱한 값 c) 9명의 사용자에 가격 z를 곱합니다.

단계별 가격 책정에 액세스하려면 계정 담당자에게 문의하세요.

## 기본 가격 책정 버전에 대한 사용자 계산(코어 사용자 포함) [#user-count]

<Callout variant="important">
  이 규칙은 [기본 가격 책정 버전](#pricing-versions) (코어 사용자가 있는 버전)을 사용하는 조직에 적용됩니다.
</Callout>

[사용량 UI](/docs/accounts/accounts-billing/general-account-settings/introduction-account-settings/#pricing) 를 사용하여 청구 가능한 사용자 수에 대한 개요를 확인할 수 있습니다. UI가 제공하는 것보다 더 자세한 정보가 필요한 경우 [사용 관련 NRQL 쿼리](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/usage-queries-alerts) 를 실행할 수도 있습니다.

한 달 동안 조직의 청구 가능한 사용자 수를 결정하기 위해 해당 월에 전체 플랫폼 사용자 또는 핵심 사용자의 **청구 가능한 사용자 유형** 을 가진 사용자를 계산합니다. 사용자의 **청구 가능한 사용자 유형** 은 한 달 동안 사용자가 설정된 가장 높은 사용자 유형으로 정의됩니다. UTC 시간대를 사용하여 달력 월의 시작과 끝을 정의합니다.

이것이 실제로 어떻게 작동하는지에 대한 예: 사용자가 달력 월 중 어느 시점에서든 전체 플랫폼 사용자로 설정된 경우 해당 월에 청구 가능한 사용자 유형은 "전체 플랫폼 사용자"이며 다음과 같은 경우에도 변경되지 않습니다. 그들은 그 달 말에 다운그레이드합니다. 이는 해당 사용자가 전체 플랫폼 사용자로 잠시 변경되더라도 해당됩니다.

청구 가능한 사용자를 추가하거나 사용자 유형을 변경할 계획이라면 이러한 규칙을 염두에 두어야 합니다. 몇 가지 팁:

* 청구 가능한 사용자를 추가하거나 사용자를 업그레이드하려는 경우 월초에 수행하도록 선택할 수 있습니다.
* 사용자를 다운그레이드하려면 월말에 다운그레이드할 수 있습니다.

고유 사용자 수는 이메일 주소로 결정됩니다. 조직에 동일한 이메일 주소를 가진 여러 사용자 레코드가 있는 경우 청구 목적으로 해당 사용자 레코드는 단일 사용자로 계산되고 해당 사용자의 청구 가능 사용자 유형은 해당 월에 할당된 가장 높은 사용자 유형이 됩니다.

청구 가능한 사용자의 비용은 조직의 [가격 책정 버전](https://newrelic.com/pricing) (Standard, Pro 또는 Enterprise) 또는 New Relic과의 사용자 지정 거래에 따라 다릅니다.

New Relic 조직에 처음 요금이 청구되기 시작하면 청구 가능한 사용자 수는 시작한 달의 시기를 기준으로 일할 계산됩니다. 조직에서 구독을 취소하면 지난 달에 대해서도 일할 계산이 적용됩니다.

## 핵심 사용자가 없는 가격 책정 버전에 대한 규칙 [#non-core]

다음 규칙은 [**핵심 사용자가 없는** 가격 책정 버전의](#pricing-versions) 조직에만 적용됩니다.

* **청구 가능한 사용자가 결정되는 방법.** 달력 월의 경우 해당 월의 전체 플랫폼 사용자 수를 계산하여 조직에 비용이 청구됩니다.
* **첫 달과 마지막 달에 비례 배분합니다.** 청구 가능한 사용자 수는 New Relic 조직이 구독을 시작한 시점 또는 사용자가 전체 플랫폼 사용자가 된 시점(전체 플랫폼 사용자로 추가되거나 사용자로 전환된 시점)을 기준으로 일할 계산됩니다.
* **사용자는 프로비저닝될 때 청구 가능합니다.** 사용자는 New Relic에서 청구 가능한 사용자 유형으로 설정되는 순간 청구 가능한 것으로 간주됩니다. 이는 해당 사용자가 New Relic에 로그인했거나 사용한 적이 있는지 여부에 관계없이 적용됩니다.
* **이메일 주소를 기반으로 한 사용자 수입니다.** 동일한 이메일 주소를 가진 조직에 여러 사용자 레코드가 있는 경우 청구 목적으로 단일 사용자로 계산됩니다.
* **원래 사용자 모델에 대한 경고입니다.** 조직에 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 사용자가 있는 경우: 사용자가 조직의 한 계정에서는 기본 사용자로 설정되고 다른 계정에서는 청구 가능 사용자로 설정된 경우 청구 가능 사용자 상태가 우선합니다.

사용자 다운그레이드 규칙은 아래를 참조하십시오.

    title="사용자 다운그레이드 규칙"

    다음 사용자 다운그레이드 규칙은 [핵심 사용자가 없는 가격 책정 버전](#pricing-versions) 의 조직에 적용됩니다.

    사용자 유형은 사용자의 기대되는 New Relic 의무 및 책임에 따라 상당히 장기적인 설정을 의미합니다. 이러한 이유로 전체 플랫폼 사용자는 12개월 동안 최대 2번만 다운그레이드될 수 있습니다. 사용자의 사용자 유형이 이 허용된 변경 수보다 많이 변경된 경우 New Relic은 해당 사용자에게 전체 플랫폼 사용자로 청구할 수 있습니다.
