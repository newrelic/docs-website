## 가격 모델 [#pricing-model]

우리의 가격 모델은 기존 제품 기능을 간략히 설명하고 풀스택 옵저버빌리티의 경우 Data + 사용자 또는 Data + Core 컴퓨트, Intelligent 옵저버빌리티의 Advanced 컴퓨트와 같은 가능한 청구 옵션을 나타냅니다. 이러한 가격 책정 모델을 이해하면 비즈니스 요구 사항에 가장 적합한 가격을 선택하는 데 도움이 됩니다.

<Callout title="메모">
  [핵심 컴퓨트](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#compute-capacity-unit) 기준 요금제 모델은 사용자별 라이선스 장벽 없이 전체 엔지니어링 조직에 무제한 플랫폼 액세스 및 풀스택 옵저버빌리티 기능을 제공합니다. Core 컴퓨팅은 현재 미리보기 단계이므로 가격 모델을 시도하고 싶다면 뉴렐릭 계정 담당자에게 문의하세요.
</Callout>

<CollapserGroup>
  <Collapser id="data" title="데이터">
    뉴렐릭이 수집한 데이터 양을 [GB 단위](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#gb-ingested) 로 나타낸 것을 말합니다. GB Ingested에 대해 다음과 같이 두 가지 옵션을 제공합니다.

    * <b>원본 데이터:</b> 뉴렐릭의 기존 데이터 옵션은 기본적으로 최소 8일 동안의 데이터 수집과 분당 최대 200억 개의 데이터 포인트에 대한 쿼리 제한을 제공합니다.

    * <b>Data Plus:</b> 뉴렐릭의 Data Plus 옵션은 최대 90일 동안의 확장된 데이터 보존 및 분당 600억 개의 데이터 포인트로 증가된 최대 쿼리 한도와 같은 고급 기능을 제공합니다.

    데이터 유형에 관계없이 수집된 GB 양에 따라 요금을 청구하므로 청구서를 이해하기 쉽습니다. 매달 100GB의 수집 용량을 무료로 이용할 수 있습니다. 수집된 GB를 추적하려면 [사용 요약](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/usage-ui) 페이지를 사용하거나 [쿼리를 실행하세요](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/usage-queries-alerts).

    ### 청구 가능한 데이터로 간주되는 것은 무엇입니까? [#usage-calculation]

    일반적으로 &quot;수집된 데이터&quot;는 다양한 데이터 트리밍 및 데이터 변환 규칙을 적용한 후 귀사가 뉴렐릭에 저장한 데이터를 의미합니다. 다시 말해, 뉴렐릭에 전송되는 원시 데이터의 양이 중요한 것이 아니라, 실제로 저장되는 데이터의 크기가 중요합니다.

    일반적으로, 당사 기능에서 활용되는 데이터는 수집된 GB에 포함되지 않습니다. GB Ingested에 포함되지 않는 기능 및 기능 관련 데이터의 예는 다음과 같습니다.

    * [기본 알림 기능](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/alerts-concepts-workflow). 예를 들어, 이러한 기능을 생성하는 데 사용되는 이벤트인 `NrAiIncident` 은 수집된 GB에 포함되지 않습니다.
    * 집계된 원시 데이터(예: [집계된 사고 데이터](/docs/data-apis/manage-data/manage-data-retention/#infrastructure-data) 또는 [차원 지표 데이터](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics))로 생성된 데이터입니다.
    * [골든 등급](/docs/apis/nerdgraph/examples/golden-metrics-entities-nerdgraph-api-tutorial) ( 골든 등급에 대한 자세한 내용은 [제외 정의를](https://github.com/newrelic/entity-definitions) 참조하세요).
    * 신세틱 검토. 신세틱스 청구, [신세틱스 한도 및 가격](docs/synthetics/synthetic-monitoring/getting-started/monitor-limits/) 에 대한 자세한 내용을 확인하세요.
    * 사용량 및 청구 추적 데이터(예: `NrConsumption`).
    * 계정 관리와 관련된 데이터(예: `NrIntegrationError`, `NrAuditEvent`).

    월별 수집된 GB 수는 가장 가까운 정수로 반올림됩니다. 예를 들어, 조직에서 한 달에 100.9GB를 사용했다면 100GB로 계산됩니다.
  </Collapser>

  <Collapser id="user" title="사용자">
    뉴렐릭에서 사용자는 플랫폼에 접근하고 사용할 수 있는 개인입니다. 청구 가능한 사용자 수는 청구 요소입니다.

    ### 사용자: 청구 및 규칙 [#billable-users]

    사용자는 뉴렐릭 조직에 청구 가능 사용자로 추가되는 순간 청구 가능합니다. 뉴렐릭 사용자의 유형은 다음과 같습니다.

    * **기본 사용자:** 기본 사용자는 요금이 부과되지 않으며, 옵저버빌리티 도구를 설정하고, 데이터에 대해 쿼리를 실행하고, 최대 7일 동안 맞춤 대시보드(퀵스타트 대시보드)를 사용할 수 있습니다. 또한 몇 가지 기본 알림 기능도 사용할 수 있습니다. 하지만 APM UI, 브라우저 모니터링 UI, 모바일 UI 등 큐레이션된 경험은 사용할 수 없습니다. 접속 자체에는 직접적인 비용이 발생하지 않지만 CCU 추가 기능을 포함한 사용량 기반 기능에 대한 요금을 청구할 수 있습니다.

    * **핵심 사용자:** 핵심 사용자로서 귀하는 뉴렐릭 CodeStream, 뿌스 인박스(errors inbox) 및 로그 관리 UI 와 같은 강력한 개발자 중심 기능에 대한 액세스와 함께 모든 기본 사용자 기능을 얻을 수 있습니다.

    * **전체 플랫폼 사용자:** 전체 플랫폼 사용자는 다양한 플랫폼 기능에 액세스할 수 있습니다. 여기에는 APM, 레이아웃 모니터링, 레이아웃 모니터링, 모바일 모니터링, 신세틱 모니터링 등 더욱 엄선된 옵저버빌리티 UI 경험이 포함됩니다.

    자세한 내용은 [사용자 유형을](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/) 참조하세요.

    <CollapserGroup>
      <Collapser id="user-count" title="청구 가능한 사용자 계산 방법">
        청구 대상 사용자에 대한 청구는 달력 월별로 이루어집니다. 한 달 동안 조직의 청구 가능 사용자 수를 결정하기 위해 해당 월 동안 전체 플랫폼 사용자 또는 핵심 사용자 중 하나인 <DNT>**billable user type**</DNT> 보유한 사용자 수를 계산합니다. 사용자의 <DNT>**billable user type**</DNT> 달력 월 동안 사용자가 설정된 가장 높은 사용자 유형으로 정의됩니다. 우리는 UTC 시간대를 사용하여 달력 월의 시작과 끝을 정의합니다.

        예를 들어, 사용자가 달력 월 중 어느 시점에든 전체 플랫폼 사용자로 설정된 경우 해당 월의 청구 가능 사용자 유형은 `full platform user` 이며, 해당 월 후반에 다운그레이드하더라도 변경되지 않습니다. 이는 해당 사용자가 잠시 동안 전체 플랫폼 사용자로 변경된 경우에도 마찬가지입니다. 변경 사항은 다음 달부터 적용되며, 이후 다시 변경하지 않는 한 해당 사용자는 전체 플랫폼 사용자로 간주되지 않습니다.

        청구 가능한 사용자를 추가하거나 사용자의 사용자 유형을 변경하려는 경우 다음 팁을 염두에 두세요.

        * 청구 가능한 사용자를 추가하거나 사용자를 업그레이드하려면 월초에 하는 것이 가장 좋습니다.
        * 사용자를 다운그레이드하려면 아마도 월말에 다운그레이드하는 것이 좋습니다.
        * 우리는 UTC 시간을 사용하여 한 달의 시작과 끝을 결정합니다. 즉, 예를 들어 태평양 표준시를 사용하고 그 달의 마지막 날에 사용자 등급을 낮추려면 태평양 표준시 기준 오후 5시까지 변경해야 합니다.

        동일한 이메일 주소를 가진 조직 내 사용자는 한 명의 사용자로만 계산됩니다. 자세한 내용은 문서의 [사용자 추적](/docs/accounts/accounts-billing/account-setup/multiple-logins-found#user-records) 섹션을 참조하세요.

        [최신 사용자 모델을](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 사용하는 경우 관리자가 사용자를 청구 가능 사용자로 추가하면 해당 사용자가 청구 가능합니다. 청구 가능한 사용자가 뉴렐릭에 로그인한 적이 없고 UI 에 `Pending invite` 태그가 있더라도 여전히 청구 가능합니다( [원래 사용자 모델을](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 사용하는 경우 `Pending invite` 사용자는 청구 가능하지 않습니다).

        사용자 추가 및 초대에 대한 자세한 내용은 문서의 [사용자 추가](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#add-users) 섹션을 참조하세요.

        청구 가능한 사용자 비용은 조직의 [가격 책정 버전](https://newrelic.com/pricing) (Standard, Pro 또는 Enterprise) 또는 Newrellic에서 설정한 사용자 지정 가격에 따라 달라집니다.

        뉴렐릭 조직이 처음 청구되기 시작하면 해당 월의 시작 날짜를 기준으로 청구 가능한 사용자 수를 비례 배분합니다. 구독을 취소하는 경우에도 마지막 달 요금은 일할 계산됩니다.
      </Collapser>

      <Collapser id="user-downgrade-rules" title="사용자 다운그레이드 규칙">
        전체 플랫폼 사용자를 다운그레이드할 수 있는 횟수와 관련된 규칙은 [사용 계획이](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing#usage-plans) 종량제(PAYG)인지 약정 계약인지에 따라 다릅니다.

        ### 종량제 다운그레이드 규칙

        사용량에 따른 요금제(PAYG)의 경우, 사용자 다운그레이드를 제한하는 규칙이 없습니다. [청구 계산은](#user-count) 사용자를 업그레이드하거나 다운그레이드할 시기를 결정하는 데 영향을 미칠 수 있습니다.

        ### 약정 계약 다운그레이드 규칙

        사용자를 업그레이드하거나 다운그레이드하기 전에 [사용자 결제 방식을](#user-count)이해해야 합니다.

        약정된 계약을 맺은 조직의 경우, 전체 플랫폼 사용자의 다운그레이드 및 업그레이드를 규정하는 규칙은 다음과 같습니다.

        <DNT>**If a Full Platform user is downgraded to a lower user type in a later month and then returned to a Full Platform user in a later month, and this downgrade/upgrade cycle happens twice in a contract year, that user will be billed as a Full Platform user for the remainder of the year, regardless of edits to their user type.**</DNT> [사용자 청구 방식](#user-count) 으로 인해 한 달 내에 발생하는 사용자 유형 변경을 관리하는 규칙은 없습니다. 한 달 동안 사용자의 청구 가능 사용자 유형은 해당 월에 설정된 가장 높은 사용자 유형입니다.
      </Collapser>
    </CollapserGroup>

    ### 사용자 및 비용 관리 [#manage]

    사용자를 관리하려면 문서의 [사용자 관리](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks) 섹션을 참조하세요. [사용 UI를](/docs/accounts/accounts-billing/general-account-settings/introduction-account-settings/#pricing) 사용하면 청구 가능한 사용자 수에 대한 개요를 얻을 수 있습니다. UI가 제공하는 것보다 더 자세한 정보가 필요한 경우 [사용 관련 NRQL 쿼리를](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/usage-queries-alerts) 실행할 수도 있습니다.
  </Collapser>

  <Collapser id="core-compute-capacity-unit" title="코어컴퓨트">
    Core 컴퓨트는 풀스택 옵션저버빌리티 기능을 위한 무제한 사용자 요금제입니다. 청구서는 쿼리, 알림, API 호출, 페이지 로드 등의 작업을 성공적으로 실행하는 데 사용된 [CCU](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#compute-capacity-unit) (핵심 CCU)를 기준으로 합니다. 가격 모델 표는 사용자가 Core Comput을 대체 가격 모델로 선택할 수 있는 모든 기능을 나타냅니다.
  </Collapser>

  <Collapser id="advance-compute-capacity-unit" title="고급 컴퓨터">
    Advanced 컴퓨트는 Data + 사용자 또는 Data + Core Compute 요금제에 추가 기능으로 Intelligent 옵저버빌리티 기능에 대한 기본 요금제를 제공합니다. 이 추가 기능은 Advanced [컴퓨팅 용량 단위(CCU)](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#compute-capacity-unit) (aCCUs)로 측정됩니다. 고급 CCU 사용량에 따라 요금이 청구됩니다.

    풀스택 옵저버빌리티 기능이 Intelligent 옵저버빌리티 기능과 결합된 경우 해당 CCU(Advanced 컴퓨팅 용량 단위)가 청구됩니다.

    <Callout variant="tip">
      고급 컴퓨팅 플랜에서 CodeStream 사용하면 사용자 유형에 관계없이 고급 CCU 요금이 부과됩니다. 즉, 기본 사용자는 고급 컴퓨트 기능에 액세스할 수 있으며 고급 CCU에 대한 요금이 부과됩니다.
    </Callout>
  </Collapser>
</CollapserGroup>

### 추가 기능에 대한 청구 [#add-on-billing]

데이터 및 청구 가능한 사용자의 기본 청구 요소 외에도 선택적 청구 가능한 추가 기능을 사용하여 뉴렐릭을 사용한 경험을 더욱 풍부하게 만들 수 있습니다.

* **Advanced Compute**: Data + 사용자 또는 Data + Core Compute 요금제에 이 추가 기능을 추가하면 Intelligent 옵저버빌리티 기능에 액세스할 수 있습니다.
* **Original Data 또는 Data Plus용 EU 데이터센터**: 이 추가 기능은 사용 가능한 경우 데이터 지역으로 유럽 연합을 선택할 때 데이터 옵션(Original Data 또는 Data Plus)에 적용됩니다.
* **Extended Retention for Original Data or Data Plus \[원본 데이터 또는 데이터 플러스의 연장 보존]**: 이 추가 기능은 데이터가 보존되는 기본 기간을 초과한 경우 적용됩니다. 이는 로그뿐만 아니라 모든 데이터에 적용되며, 많은 양의 작은 쿼리를 실행해야 하거나 대량의 데이터에 대한 쿼리를 실행해야 하는 경우 좋은 옵션입니다.
* **Live Archives \[라이브 아카이브]**: 로그 보관 기간을 최대 7년까지 연장합니다. 라이브 아카이브에는 고급 컴퓨팅도 필요합니다.
* **New Relic Synthetics Checks**: 이 추가 기능은 귀하의 수표가 신세틱 모니터링 검사의 기본 수를 초과하는 경우 적용됩니다.
