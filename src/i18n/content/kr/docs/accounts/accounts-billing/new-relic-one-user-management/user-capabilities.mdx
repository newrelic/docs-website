사용자 관리 공간에서 **기능** 은 사용자 지정 역할에 추가할 수 있도록 제공한 New Relic으로 수행할 수 있는 세분화된 작업입니다.이러한 기능은 사전 구축 [표준 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) 에도 연결됩니다.기능의 예는 APM 앱 설정을 보거나 경고 조건을 수정하거나 데이터 보존 설정을 관리하는 기능입니다.

역할에 어떤 기능이 있는지 알아보려면 사용자 관리 UI로 이동하여 특정 역할을 확인하세요.이 UI의 위치는 사용자가 속한 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

* 최신 사용자 모델: **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 에서 , **[사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Administration > Access management > Roles** 을 클릭합니다.
* 원래 사용자 모델: **[one.newrelic.com](https://one.newrelic.com/all-capabilities)** 에서, **[사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)> Account settings > Users and roles** 클릭합니다.

## 기능이란 무엇입니까? [#capabilities-roles]

모든 권한을 가진 New Relic 전체 플랫폼 사용자(예: **Admin** 그룹의 사용자)는 플랫폼의 모든 기능을 사용할 수 있습니다. New Relic에서 수행할 수 있는 몇 가지 작업은 역할 **기능** 으로 제공되었습니다. 사용자 지정 역할에서 추가하거나 제거할 수 있으며 이를 사용하여 기본적으로 사용 가능한 [표준 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) 을 구분할 수 있습니다. 우리가 표시하고 선택할 수 있는 기능은 조직이 제어하는 데 유용하다고 생각하는 기능입니다.

우리가 표시 **하지 않고** 선택할 수 있는 New Relic 기능이 많이 있습니다. 예를 들어 모든 사용자로 액세스할 수 있고 기능으로 제한되지 않는 다양한 UI 페이지가 있습니다. 다른 예: **조직** [관리 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) 이 있는 그룹의 사용자는 조직 수준 설정을 구성할 수 있으며 해당 권한은 사용자 지정 역할에 추가할 수 없습니다.

## 기능에 대한 중요 사항 [#important-points]

다음은 역할 기능에 대한 몇 가지 다른 중요한 사항입니다.

* **사용자의 사용자 유형도 액세스를 허용해야 합니다.** New Relic 기능에 대한 사용자의 액세스는 사용자 유형과 할당된 역할에 의해 관리됩니다. 자세한 내용은 [User access](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#user-type-groups-relation) \[사용자 액세스를]참조하십시오.

* **일부 기능은 기능이 겹칩니다.** 이것이 UI에서 일부 기능 확인란을 선택하면 다른 상자를 자동으로 선택하거나 선택 취소하는 이유입니다.

* **기능은 데이터 쿼리에 영향을 주지 않습니다.** 대부분의 기능은 데이터 쿼리가 아닌 New Relic UI 및 API 경험에 적용됩니다. 예를 들어, 귀하의 능력으로 인해

  <InlinePopover type="apm"/>

  UI에서 해당 계정에 대한 액세스 권한이 있는 경우 여전히 APM 데이터를 쿼리할 수 있습니다. 일부 프로젝트 또는 사용자에 대해 더 확고한 데이터 경계가 필요한 경우 데이터를 [다른 계정](/docs/accounts/accounts-billing/account-structure/add-accounts)으로 분류할 수 있습니다.

사용자 권한이 제어되는 주요 방법에 대해 자세히 알아보려면 [사용자 관리 개념](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#user-type-groups-relation) 을 참조하세요.

## 사전 구축된 역할 [#pre-built-roles]

사전 구축된 역할에는 다양한 기능 그룹이 있습니다.사전 구축된 역할의 작동 방식은 사용 중인 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

* 새로운 사용자 모델: [역할에 대해 알아](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 보세요.
* 원래 사용자 모델: [역할에 대해 알아](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 보세요.

      선호됩니다).
