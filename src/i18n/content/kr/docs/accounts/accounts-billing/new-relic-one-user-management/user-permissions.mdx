뉴렐릭 사용자 관리 컨텍스트에서 <DNT>**permission**</DNT> 은 사용자 정의 역할에 추가할 수 있도록 제공한 뉴렐릭으로 수행할 수 있는 세부적인 작업입니다. 이러한 권한은 사전 빌드된 [표준 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) 에도 연결되어 있습니다. 권한의 예는 APM 앱 설정을 보거나 공지 조건을 수정하거나 데이터 보존 설정을 관리하는 기능입니다.

역할에 어떤 권한이 있는지 알아보려면 사용자 관리 UI로 이동하여 특정 역할을 확인하세요. 이 UI를 찾으려면: <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT> 에서 오른쪽 하단에 있는 [사용자 메뉴를](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 클릭한 후 <DNT>**Administration > Access management > Roles**</DNT> 로 이동합니다.

## 권한이란 무엇입니까? [#permissions-roles]

제한이 없는 뉴렐릭 전체 플랫폼 사용자(예: <DNT>**Admin**</DNT> 그룹의 사용자)는 플랫폼의 모든 기능을 사용할 수 있습니다. 뉴렐릭에서 수행할 수 있는 작업 중 일부는 <DNT>**permissions**</DNT> 로 제공됩니다. 사용자 정의 역할에서 이를 추가하거나 제거할 수 있으며, 우리는 또한 이러한 권한을 사용하여 [표준 역할을](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) 구별합니다. 우리가 표시하고 선택할 수 있는 권한은 일반적인 사용자 관리 작업에 가장 유용할 것으로 생각되는 권한입니다.

우리가 <DNT>**don't**</DNT> 표시하고 선택할 수 있도록 하는 뉴켈릭 기능이 많이 있습니다. 예를 들어, 모든 사용자로서 액세스할 수 있고 우리가 노출하는 권한에 의해 통제되지 않는 다양한 UI 페이지가 있습니다. 또 다른 예: <DNT>**Organization**</DNT> [관리 설정이](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) 있는 그룹의 사용자는 조직 수준 설정을 구성할 수 있으며 해당 권한은 사용자 정의 역할에 추가하는 데 사용할 수 없습니다.

<Callout variant="tip">
  권한은 때때로 <DNT>**capabilities**</DNT> 이라고도 합니다.
</Callout>

## 권한에 대한 중요 사항 [#important-points]

권한에 대한 다른 중요한 사항은 다음과 같습니다.

* <DNT>**A user's user type must also allow access.**</DNT> 뉴켈릭 기능에 대한 사용자의 액세스는 사용자 유형과 할당된 역할에 따라 결정됩니다. 이에 대한 자세한 내용은 [사용자 액세스를](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 참조하세요.
* <DNT>**Some permissions overlap in functionality.**</DNT> 이것이 UI에서 일부 권한 확인란을 선택하면 자동으로 다른 상자를 선택하거나 선택 취소하는 이유입니다.
* <DNT>**Permissions don't affect querying of data.**</DNT> 대부분의 권한은 뉴렐릭 UI 및 API 경험에 적용되며 데이터 쿼리에는 적용되지 않습니다. 예를 들어 권한으로 인해 <InlinePopover type="apm"/>UI에 액세스할 수 없는 경우에도 해당 계정에 대한 액세스 권한이 있으면 APM 데이터를 쿼리할 수 있습니다. 일부 프로젝트나 사용자에 대해 더 확고한 데이터 경계가 필요한 경우 데이터를 [여러 계정](/docs/accounts/accounts-billing/account-structure/add-accounts) 으로 분류할 수 있습니다.

사용자 권한을 제어하는 주요 방법에 대해 자세히 알아보려면 [사용자 관리 개념을](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 참조하세요.

## 사전 구축된 역할 [#pre-built-roles]

사전 구축된 역할에는 다양한 권한 그룹이 있습니다. 사전 구축된 역할의 작동 방식은 현재 사용 중인 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

* 최신 사용자 모델(대부분의 사용자): [역할에 대해 알아보세요](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).
* 원래 사용자 모델: [역할에 대해 알아](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 보세요.

## 권한 정의 [#permission-definitions]

UI로 이동하여 사전 구축된 각 역할에 대한 권한을 볼 수 있습니다. [UI](https://one.newrelic.com) 왼쪽 하단에서 이름을 클릭하여 사용자 메뉴를 연 다음 **Administration > Access management > Roles** 로 이동합니다. 다음 역할에 대해 권한 목록을 사용할 수 있습니다.

* 모든 제품 관리자
* 표준 사용자
* 읽기 전용

특정 권한에 대해 자세히 알아보려면 아래 카테고리를 선택하거나 이 문서에서 원하는 특정 용어를 검색해 보세요.

    <DNT>**Insights**</DNT> 은 사용자 정의 데이터 수집, 사용자 정의 쿼리, 사용자 정의 차트 및 사용자 정의 대시보드와 관련된 기능을 갖춘 제품의 원래 이름입니다. 권한은 다음과 같습니다:

    * <DNT>**Any dashboard**</DNT>: 계정의 대시보드를 삭제하는 기능과 관련이 있습니다.

    * <DNT>**Data sources**</DNT>: 뉴렐릭에 보고된 데이터를 제어할 수 있는 현재 사용되지 않는 UI와 관련이 있습니다.

    * <DNT>**Events to metrics**</DNT>: 이는 다음을 관리합니다.

      * [이벤트-메트릭 규칙 및 데이터](/docs/data-apis/convert-to-metrics/analyze-monitor-data-trends-metrics)
      * [서비스 수준 지표 및 목표](/docs/service-level-management/create-slm)

    * <DNT>**Insert keys**</DNT>: 대부분 더 이상 사용되지 않는 [인사이트 삽입 키](/docs/apis/intro-apis/new-relic-api-keys/#insights-insert-key) 와 관련됩니다( [라이선스 키](/docs/apis/intro-apis/new-relic-api-keys/#overview-keys) 가 선호됩니다).

    * <DNT>**NRQL drop rules**</DNT>: [삭제 규칙을 사용하여 데이터 삭제](/docs/data-apis/manage-data/drop-data-using-nerdgraph) 와 관련됩니다.

    * <DNT>**Query keys**</DNT>: 대부분 더 이상 사용되지 않는 [인사이트 쿼리 키](/docs/apis/intro-apis/new-relic-api-keys/#insights-query-key) 와 관련됩니다( <InlinePopover type="userKey"/>이 선호됨).
  </Collapser>

    * <DNT>**Nerdpacks**</DNT>: [뉴켈릭 앱](/docs/new-relic-solutions/new-relic-one/build-custom-new-relic-one-application) 과 관련이 있습니다.

    * <DNT>**Pixie live data**</DNT>: [쿠버네티스 클러스터 탐색기](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/kubernetes-cluster-explorer) 에서 실시간 디버깅 데이터에 접근할 수 있습니다.
    * <DNT>**Repositories**</DNT>: 뉴켈릭-모니터링 대상 생성 및 삭제와 관련됩니다( [뉴켈릭 CodeStream](/docs/codestream/start-here/what-is-codestream) 과 같은 기능에서 사용됨).
    * <DNT>**Tags**</DNT>: 플랫폼 [태그](/docs/new-relic-solutions/new-relic-one/core-concepts/use-tags-help-organize-find-your-data) 와 관련이 있습니다.
    * <DNT>**Workloads**</DNT>: [워크로드](/docs/new-relic-solutions/new-relic-one/workloads/workloads-isolate-resolve-incidents-faster) 와 관련이 있습니다.
