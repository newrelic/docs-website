이 문서에서는 **사용자 유형**을 정의하는 방법, 각 사용자 유형별로 사용할 수 있는 기능 및 사용자 유형을 결정하는 방법을 배울 수 있습니다.

비용이 청구되는 사용자 수를 계산하는 방법에 대해 알고 싶으신가요? [뉴렐릭 요금제](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)를 참조하십시오. 아직 계정이 없으시다면, 지금 [뉴렐릭의 기간 제한없는 무료 계정을 신청](https://newrelic.com/signup)하십시오.

## "사용자 유형"이란? [#user-type-defined]

사용자의 **사용자 유형**은 사용자가 이론적으로 액세스할 수 있는 뉴렐릭 기능 집합을 결정합니다. 실제로 사용자에게는 다양한 방식으로 기능을 제한하는 [역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts)이 할당되는 경우가 많지만 사용자 유형은 이론적인 기능의 최대 집합을 나타냅니다.

세 가지 사용자 유형이 있습니다.

* **기본 사용자**: 몇 가지 기본적이지만 강력한 뉴렐릭 플랫폼 기능 사용.
* **핵심 사용자** ([제한된 가용성](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/core-users-release#availability)): 기본 사용자보다 더 많은 기능 사용.
* **전체 플랫폼 사용자**: 모든 기능 사용.

사용할 수 있는 기능에 대한 자세한 내용으로 건너뛰려면 [Capabilities(기능)](#user-type-capabilities)을 참조하십시오. 사용자 유형의 개념에 대한 자세한 정보는 아래와 같습니다.

[사용량 기반 요금제](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)에서 기본 사용자는 무료이며, 코어 사용자와 전체 플랫폼 사용자는 요금이 청구됩니다. 사용자 수는 요금 청구에 고려되는 요소이므로 [사용자 요금 계산 방법](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)을 이해하는 것이 중요합니다.

[뉴렐릭 사용자를 추가](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model/#add-users)해야 하는 경우 주요 결정 요인 중 하나는 사용자 유형을 지정하는 것입니다. 처음에 확실하지 않은 경우는 먼저 기본 사용자로 추가하고 나중에 사용자를 업그레이드할 수 있습니다. 사용자 유형을 조정하는 방법은 [사용자 유형 관리](#manage-user-type)를 참조하십시오.

사용자 유형은 향후 몇 개월 또는 그 이상 동안 사용자가 기대하는 뉴렐릭의 책임에 기반한 장기적인 설정입니다. 이러한 의도는 [청구 계산 및 다운그레이드 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)에 반영되어 있습니다. 사용자 액세스를 더 자주 또는 더 세부적으로 조정하려면 [역할](/docs/accounts/accounts-billing/new-relic-one-user-management/groups-roles-capabilities)을 사용해야 합니다.

## 사용자 유형을 선택하는 방법 [#choose-user-type]

[사용량 기반 요금제를 사용하는 조직의](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models) 경우 사용자 유형이 청구 요소입니다. 사용자 유형을 결정하기 전에 [사용자 관련 계산 및 다운그레이드 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)을 이해하는 것이 좋습니다. 자세한 기능 비교는 [기능 표](#user-type-comparison-table)를 참조하세요.

다음은 사용자 유형을 결정하기 위한 몇 가지 팁입니다.

**전체 플랫폼 사용자 요건:**

* 애플리케이션 개발 수명 주기의 개발, 테스트, 배포 및 유지 관리 단계에서 핵심적인 역할을 합니다.
* 정기적으로 코드를 중단/수정합니다. 워크플로우 분류, 문제 해결 또는 팀의 사용자 및 역할 관리를 담당합니다.
* 데브옵스 사례(예: 버전 제어 시스템 및 CI/CD 구현)가 있습니다.
* 뉴렐릭의 큐레이션된 대시보드와 경험을 사용해야 합니다. 즉, 뉴렐릭 플랫폼에 대한 전체 액세스 권한이 필요합니다.
* 사용자 및/또는 청구서를 관리할 수 있어야 합니다.

**[코어 사용자](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/core-users-release) 요건:**

* 안정성과 가동 시간 유지가 주요 업무가 아닌 개발자입니다.

* 전체 플랫폼 액세스가 필요하지 않지만 다음과 같은 코어 사용자에게 제공되는 특정 기능의 이점을 누릴 수 있습니다.

  * [뉴렐릭 CodeStream](/docs/codestream/start-here/what-is-codestream)을 사용하여 IDE에서 직접 코드 문제 디버깅
  * [에러 인박스](/docs/errors-inbox/errors-inbox)를 사용하여 스택 전체의 오류를 한 곳에서 확인
  * [로그 UI](/docs/logs/ui-data/use-logs-ui)를 사용하여 로그에서 문제와 패턴 발견
  * [뉴렐릭 앱을 구축 및 배포](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/build-custom-new-relic-one-application)하고 [앱 카탈로그](https://opensource.newrelic.com/nerdpacks)의 앱을 사용하는 기능

**기본 사용자 요건:**

* 애플리케이션 개발 수명주기의 계획 단계에서 핵심적인 역할을 합니다.
* 뉴렐릭 툴을 사용 및 구성하여 데이터를 뉴렐릭으로 가져오고 이러한 데이터에 대한 알림을 액세스, 설정 및 사용합니다. (워크플로우 분류, 문제 해결 또는 팀의 사용자 및 역할 관리에 대해 반드시 책임이 있는 것은 아님)
* 미래 계획(예: 고위 임원)을 위한 높은 수준의 분석 및 비즈니스 메트릭을 보길 원합니다.
* 뉴렐릭의 큐레이션된 경험과 대시보드를 사용할 필요가 없지만, 자신만의 커스텀 쿼리와 데이터 차트를 생성할 수 있는 기능을 통해 이점을 얻을 수 있습니다. 즉, 플랫폼에 대한 전체 액세스 권한이 필요하지 않습니다.
* 사용자 또는 요금을 관리하지 않습니다.
