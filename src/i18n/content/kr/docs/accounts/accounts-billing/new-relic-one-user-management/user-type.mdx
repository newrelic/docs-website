이 문서에서는 사용자 유형(<DNT>**user type**</DNT>) 정의 방법, 각 사용자 유형이 액세스할 수 있는 기능, 사용자 유형을 결정하는 방법에 대해 알아봅니다.

비용이 청구되는 사용자 수를 계산하는 방법에 대해 알고 싶으신가요? [뉴렐릭 요금제](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)를 참조하십시오. 아직 계정이 없으시다면, 지금 [뉴렐릭의 기간 제한없는 무료 계정을 신청](https://newrelic.com/signup)하십시오.

## 사용자 유형이란? [#user-type-defined]

뉴렐릭 사용자의 <DNT>**user type**</DNT>에 따라 액세스할 수 있는 뉴렐릭 권한의 최대 집합이 결정됩니다. 사용자 유형은 향후 몇 개월 또는 그 이상 동안 사용자의 예상되는 책임을 기반으로 하는 상당히 장기적인 설정입니다.

사용자의 사용자 유형 선택은 주로 청구와 관련된 결정입니다. 코어 사용자와 전체 플랫폼 사용자는 요금이 청구될 수 있지만 기본 사용자는 그렇지 않습니다. 팀 구성원들이 뉴렐릭을 사용해 어떤 가치를 제공할지 조직이 가진 기대치가 관건입니다. (청구에 대한 자세한 내용은 [사용자 청구](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)를 참조하십시오.)

사용자 유형을 사용자의 권한을 제어하는 방법으로 사용해서는 **안됩니다**. 뉴렐릭이 각 사용자 유형별로 사용 가능한 권한을 조정하는 경우가 있기 때문입니다. 권한에 대한 액세스를 제한하려면 [역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts)을 사용해야 합니다.

세 가지 사용자 유형이 있습니다.

* <DNT>**Basic user**</DNT>(기본 사용자): 몇 가지 기본적이지만 강력한 뉴렐릭 플랫폼 기능에 액세스합니다.
* <DNT>**Core user**</DNT>(코어 사용자): 기본 사용자보다 더 많은 기능에 액세스합니다.
* <DNT>**Full platform user**</DNT>(전체 플랫폼 사용자): 모든 기능에 액세스할 수 있습니다.

[뉴렐릭 사용자를 추가](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model/#add-users)해야 하는 경우 주요 결정 요인 중 하나는 사용자 유형을 지정하는 것입니다. 처음에 확실하지 않은 경우는 먼저 기본 사용자로 추가하고 나중에 사용자를 업그레이드할 수 있습니다. 사용자 유형을 조정하는 방법은 [사용자 유형 관리](#manage-user-type)를 참조하십시오.

## 사용자 유형을 선택하는 방법 [#choose-user-type]

사용자 유형을 결정하기 전에 [청구에 미치는 영향과 다운그레이드 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)을 이해하시기 바랍니다. 보다 자세한 비교는 [사용자 유형 액세스 비교표](#user-type-comparison-table)를 참조하십시오.

다음은 사용자 유형을 결정하기 위한 몇 가지 팁입니다.

<DNT>
  **Reasons to make someone a full platform user:**
</DNT>

* 커스텀 차트를 쿼리하고 생성하는 기능뿐만 아니라 뉴렐릭 플랫폼(선별된 대시보드 및 기타 경험)에 대한 전체 액세스 권한이 필요합니다.
* 애플리케이션 개발 수명 주기의 개발, 테스트, 배포 및 유지 관리 단계에서 핵심적인 역할을 합니다.
* 정기적으로 코드를 중단/수정합니다. 워크플로우 분류, 문제 해결 또는 팀의 사용자 및 역할 관리를 담당합니다.
* 데브옵스 사례(예: 버전 제어 시스템 및 CI/CD 구현)가 있습니다.

<DNT>
  **Reasons to make someone a core user:**
</DNT>

* 안정성과 가동 시간이 주요 임무가 아닌 개발자입니다.

* 전체 플랫폼 액세스가 필요하지 않지만 코어 사용자에게 제공되는 다음과 같은 특정 기능의 이점을 누릴 수 있습니다.

  * [뉴렐릭 CodeStream](/docs/codestream/start-here/what-is-codestream)을 사용하여 IDE에서 직접 코드 문제 디버깅
  * [에러스 인박스(errors inbox)](/docs/errors-inbox/errors-inbox)를 사용해 전체 스택의 오류를 한 곳에서 확인할 수 있습니다.
  * [로그 UI](/docs/logs/ui-data/use-logs-ui)를 사용하여 로그에서 문제와 패턴 발견
  * [뉴렐릭 앱 카탈로그](https://opensource.newrelic.com/nerdpacks)의 앱 사용

* 사용자 및/또는 청구를 관리할 필요가 있습니다. (코어 및 전체 플랫폼 사용자 모두 가능)

<DNT>
  **Reasons to make someone a basic user:**
</DNT>

* 선별한 경험과 <InlinePopover type="dashboards" />를 사용하기 위해 전체 플랫폼 액세스가 필요하지는 않지만 커스텀 쿼리와 데이터 차트를 생성하면 다양한 이점을 얻을 수 있습니다.
* 애플리케이션 개발 수명주기의 계획 단계에서 핵심적인 역할을 합니다.
* 뉴렐릭 툴을 사용 및 구성하고 해당 데이터에 대한 <InlinePopover type="alerts" />에 액세스, 구성 및 사용이 가능하지만, 반드시 워크플로우 분류, 문제 해결, 또는 팀의 사용자 및 역할 관리를 담당하지는 않습니다.
* 미래 계획을 수립하기 위해 높은 수준의 분석 및 비즈니스 메트릭을 보길 원합니다.(예: 고위 임원)
* 기본 사용자는 무료이지만, 추가 기능인 고급 컴퓨팅에 포함되어 요금이 부과되는 항목에 액세스할 수 있도록 권한을 부여할 수 있습니다. CodeStream 같은 특정 기능의 CCU 사용에 대해서만 비용이 부과됩니다.
* 사용자 또는 요금을 관리하지 않습니다.

## 사용자 유형 액세스 비교표 [#user-type-comparison-table]

다음은 각 사용자 유형이 액세스할 수 있는 기능을 자세히 비교한 것입니다. 이 표의 중요 사항:

* 이 표는 [뉴렐릭 요금 페이지](https://newrelic.com/pricing)에서 가져온 것입니다. 이 표를 확인하려면 <DNT>**User costs**</DNT>를 방문해 <DNT>**View permissions**</DNT>를 클릭하면 됩니다.
* 많은 기능들이 기본 데이터가 아니라 UI 경험에 액세스해야 사용할 수 있습니다. 모든 사용자들은 액세스 권한이 있는 계정의 모든 데이터를 쿼리할 수 있으며 커스텀 차트를 만들고 확인할 수 있습니다. 예를 들어 기본 사용자는 <InlinePopover type="apm" />데이터, 브라우저 모니터링 데이터 등에 액세스할 수 있지만 선별된 UI 환경에는 액세스할 수 없습니다.
* 사용자 유형은 장기적인 설정입니다. 사용자 유형과 역할 모두 뉴렐릭의 기능들에 대한 액세스를 제어합니다. [사용자 액세스 요소에 대해 자세히 알아보십시오](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## 사용자 유형 및 역할 [#user-type-and-roles]

다음은 사용자 유형과 [역할 기반 액세스](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts)의 차이점에 대한 설명입니다.

* 사용자의 <DNT>**[user type](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)**</DNT>: 어떤 사용자 유형을 적용하느냐는 팀 구성원이 뉴렐릭으로 무엇을 하기를 기대하는지, 그리고 그 작업에서 얼마나 많은 가치를 얻을 것으로 기대하는지가 중요한 문제입니다. 주로 청구와 관련된 결정입니다. 사용자가 액세스할 수 있는 최대 허용 권한을 설정합니다. 사용자 유형은 사용자의 액세스 및 권한을 제어하는 데 사용되지 **않습니다**. 이를 위해서는 역할(role)을 사용해야 합니다.
* 사용자의 <DNT>**roles**</DNT>: 역할은 사용자의 액세스를 제어합니다. 역할은 뉴렐릭에서 특정 작업(예: APM 설정 수정)을 수행할 수 있는 액세스 권한을 부여하는 권한(<DNT>**permissions**</DNT>)으로 구성됩니다. 역할은 [사용자 그룹](#groups)에 적용해 할당하며, 조직 내 하나 이상의 계정에 존재할 수 있습니다.

뉴렐릭 사용자는 a) 사용자 유형과 b) 역할 권한의 조합을 통해 뉴렐릭의 기능을 사용할 수 있는 권한이 부여됩니다. 뉴렐릭 사용자가 무언가에 액세스하려면 사용자 유형과 할당된 역할이 해당 액세스를 허용해야 합니다.

예를 들어, 기본 사용자에게 [<DNT>**All product admin**</DNT>](#standard-roles) (기본 그룹 <DNT>**User**</DNT> 및 <DNT>**Admin**</DNT> 모두 가지고 있음)과 같이 광범위한 뉴렐릭 액세스 권한을 가진 역할이 할당되었다고 가정해 보겠습니다. 이 사용자는 사용자 유형이 기본 사용자이기 때문에 코어 사용자 또는 해당 역할을 가진 전체 플랫폼 사용자가 액세스할 수 있는 많은 기능을 사용할 수 없습니다. 더 많은 액세스 권한을 얻으려면 이 기본 사용자는 코어 사용자 또는 전체 플랫폼 사용자가 되어야 합니다.

또 다른 예로, 전체 플랫폼 사용자에게 특정 계정(예: [<DNT>**Read only**</DNT>](#standard-roles))에 제한적인 역할이 할당되어 있다고 가정해 보겠습니다. 전체 플랫폼 사용자는 이론적으로 뉴렐릭의 모든 기능에 액세스할 수 있지만, 이 경우 해당 계정에 할당된 역할로 인해 액세스가 크게 제한됩니다. 더 많은 액세스 권한을 얻으려면 할당된 역할을 변경해야 합니다(예: 다른 그룹에 할당하거나 그룹에 할당된 역할 조정).

역할 및 그룹에 대한 자세한 내용은 [사용자 관리 개념](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts)을 참조하십시오.
