## 사용자 관리 UI [#where]

New Relic 조직의 사용자 관리 옵션을 찾으려면 [계정 드롭다운](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown) 에서 **관리** 를 선택합니다. 두 개의 사용자 관리 UI 페이지가 있습니다.

* **사용자 관리** ( [UI 링크](https://one.newrelic.com/admin-portal/organizations/users-list) ): 사용자를 추가하고, 사용자 유형을 업데이트하고, 사용자에게 할당된 그룹을 관리하고, 사용자 업그레이드 요청을 승인하는 데 사용합니다.
* **액세스 관리** ( [UI 링크](https://one.newrelic.com/admin-portal/organizations/organization-detail) ): 이 UI를 사용하여 그룹 및 역할을 보고, 만들고, 관리합니다. SAML SSO 및 SCIM 프로비저닝을 설정하려면 인증 도메인 UI를 사용하십시오.

UI 사용에 대한 팁은 [일반 작업](#workflow) 및 [사용자 관리 비디오](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos) 를 참조하십시오.

## 요구 사항 [#requirements]

사용자 관리 권한이 무엇인지 확인하는 가장 쉬운 방법은 [사용자 관리 UI](#where) 로 이동하여 액세스 권한이 있는 항목을 확인하는 것입니다.

일부 사용자 관리 요구 사항 및 제한 사항:

* 이 문서에서는 [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 에서 사용자를 관리하는 방법을 설명합니다.

* 그룹과 역할을 추가하고 편집하려면 조직에 [Pro 또는 Enterprise](https://newrelic.com/pricing) 버전이 있어야 합니다.

* 권한 관련 요구 사항:

  * 핵심 사용자 또는 전체 플랫폼 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 이 필요합니다.
  * 대부분의 사용자 관리 기능에는 조직 및 인증 도메인 관련 [관리 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) 이 필요합니다.

* 구성 충돌을 방지하려면 조직에서 한 번에 한 사람만 사용자를 관리하도록 하십시오.

    사용자를 추가하려면 [**사용자 관리** UI](#where) 로 이동하여 사용자 **추가** 를 선택합니다.

    참고:[자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 ID 제공업체의 사용자만 추가할 수 있습니다.
  </Collapser>

  <Collapser
    id="edit-user-type"
    title="사용자의 사용자 유형 수정"
  >
    사용자의 사용자 유형을 변경하기 전에 다음 사항을 이해하는 것이 좋습니다.

    * [사용자의 사용자 유형을 결정하는 방법](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#choose-user-type)
    * [청구 가능한 사용자 계산 방법](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)
    * [사용자 다운그레이드 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#user-downgrade-rules)
    * [자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 [사용자 유형을 관리하기 위한 다른 옵션이 있습니다](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade) .

    한 명 이상의 사용자의 사용자 유형을 변경하려면:

    1. [**사용자 관리** UI](#where) 에서 사용자 유형을 수정하려는 사용자의 확인란을 클릭합니다.
    2. 사용자 선택을 시작하면 **유형 편집** 에 대한 옵션이 나타납니다.

    특정 사용자를 클릭하여 사용자 유형 및 그룹을 편집할 수도 있습니다.

    <img
      title="Edit user type in the UI"

    />

    사용자가 사용자 유형을 업그레이드하는 방법을 관리하려면 [인증 도메인 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade) 을 참조하십시오.
  </Collapser>

  <Collapser
    id="bulk-edit-users"
    title="사용자 일괄 수정"
  >
    한 번에 여러 사용자의 설정을 편집하려면:

    1. [**사용자 관리** UI](#where) 로 이동하여 편집하려는 그룹 또는 사용자 유형의 사용자에 대한 확인란을 선택합니다.
    2. 편집 옵션은 UI 상단에 나타납니다.

    참고:[자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 ID 공급업체에서 그룹을 편집해야 합니다.
  </Collapser>

  <Collapser
    id="delete-users"
    title="사용자 삭제"
  >
    사용자를 삭제하려면 [**사용자 관리** UI](#where) 로 이동하여 한 명 이상의 사용자에 대한 확인란을 선택합니다. 그런 다음 **사용자 삭제 를** 클릭합니다.
  </Collapser>

  <Collapser
    id="export-users"

    또는 일부 사용자를 내보내려면 먼저 내보낼 사용자를 선택한 다음 다운로드 버튼을 누르십시오.
  </Collapser>

    * [사용자 관리](#where) UI에서 해당 설정이 활성화된 기본 [**Admin** 그룹](/docs/accounts/accounts-billing/new-relic-one-pricing-users/users-roles#groups) 에 사용자를 추가할 수 있습니다.

      또는

    * 사용자 지정 그룹을 만들고 해당 설정을 지정할 수 있습니다. 일반적인 사용자 관리 작업에 대한 자습서는 [사용자 관리 자습서](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model) 를 참조하십시오.
