## 사용자 관리 UI [#where]

[New Relic One 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 사용자의 경우 사용자 관리 옵션을 찾으려면 다음을 수행하십시오. [계정 드롭다운](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown) 에서 **관리** 를 선택하십시오. 두 개의 사용자 관리 UI 페이지가 있습니다.

* **사용자 관리** : 사용자 추가, [사용자 유형 업데이트](#edit-user-type) , 사용자에게 할당된 그룹 관리, 사용자 업그레이드 요청 승인에 사용합니다.
* **액세스 관리** : 이 UI를 사용하여 그룹, 역할 및 액세스 권한을 생성하고 관리합니다. 인증 도메인 UI를 사용하여 SAML SSO 및 SCIM 프로비저닝을 설정합니다.

사용자 관리 UI의 작동 동영상을 보고 싶으십니까? [사용자 관리 비디오](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos) 를 참조하십시오.

<Callout variant="important">
  이러한 UI 페이지가 표시되지 않는 경우 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 을 사용 중이거나 [필요한 역할](#requirements) 이 없기 때문일 수 있습니다.
</Callout>

## 요구 사항 [#requirements]

사용자 관리 권한이 무엇인지 확인하는 가장 쉬운 방법은 [사용자 관리 UI](#where) 로 이동하여 액세스 권한이 있는 항목을 확인하는 것입니다.

일부 사용자 관리 요구 사항 및 제한 사항:

* 이 문서는 [New Relic One 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 의 사용자 관리에 관한 것입니다.

* 그룹, 역할 및 액세스 권한을 관리하려면 조직이 [Pro 또는 Enterprise](https://newrelic.com/pricing) 버전이어야 합니다.

* 권한 관련 요구 사항:

  * 핵심 사용자 또는 전체 플랫폼 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 이 필요합니다.
  * 대부분의 사용자 관리 기능에는 [**인증 도메인 관리자** 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#standard-roles) 이 필요하고 일부는 [**조직 관리자** 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#standard-roles) 이 필요합니다. 자세한 내용은 [역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#standard-roles) 을 참조하십시오.

* 구성 충돌을 방지하려면 조직에서 한 번에 한 사람만 사용자를 관리하도록 하십시오.

* New Relic 사용자는 자신의 로그인 정보를 다른 사람과 공유할 수 없습니다. New Relic 사용자는 최대 3개의 동시 활성 세션 또는 주어진 시간에 3개의 고유한 IP 주소를 사용할 수 있습니다.
