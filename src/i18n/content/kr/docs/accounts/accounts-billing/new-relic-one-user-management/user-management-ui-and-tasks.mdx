## 사용자 관리 UI [#where]

New Relic 조직의 사용자 관리 옵션을 찾으려면: [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 에서 **관리** 를 선택합니다. 여기에서 찾을 수 있는 사용자 관리 관련 UI 페이지는 다음과 같습니다.

* **사용자 관리** ( [UI 링크](https://one.newrelic.com/admin-portal/organizations/users-list) ): 사용자 추가, 사용자 유형 업데이트, 사용자 그룹 관리, 사용자 업그레이드 요청 승인 및 세션 설정 제어에 사용합니다.
* **액세스 관리** ( [UI 링크](https://one.newrelic.com/admin-portal/organizations/organization-detail) ): 이 UI를 사용하여 [그룹 및 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts) 을 보고 만들고 관리합니다.
* **API 키** : [API 키 UI](/docs/apis/intro-apis/new-relic-api-keys) 를 사용하여 사용자의 API 키를 보고 관리합니다.
* **도메인 캡처** : 도메인 캡처를 사용하여 사용자가 New Relic에 가입하려고 할 때 올바른 위치로 라우팅하십시오. [도메인 캡처에 대해 자세히 알아보세요.](/docs/accounts/accounts-billing/account-setup/domain-capture)
* **인증 도메인** ( [UI에 대한 링크](https://one.newrelic.com/admin-portal/centralized-admin-user-auto-provisioning/home) ): 이를 사용하여 사용자가 추가되는 방식(New Relic 또는 ID 공급자에서), 로그인 방식(수동 또는 SAML SSO를 통해) 등을 제어합니다. 자세한 내용은 [인증 도메인](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 을 참조하십시오.

사용자 관리에 대한 팁은 [일반 작업](#workflow) 및 [사용자 관리 비디오](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos) 를 참조하십시오.

## 요구 사항 [#requirements]

사용자 관리 권한이 무엇인지 확인하는 가장 쉬운 방법은 [사용자 관리 UI](#where) 로 이동하여 액세스 권한이 있는 항목을 확인하는 것입니다.

일부 사용자 관리 요구 사항 및 제한 사항:

* 이 문서에서는 [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 에서 사용자를 관리하는 방법을 설명합니다.

* 그룹과 역할을 추가하고 편집하려면 조직에 [Pro 또는 Enterprise](https://newrelic.com/pricing) 버전이 있어야 합니다.

* 권한 관련 요구 사항:

  * 핵심 사용자 또는 전체 플랫폼 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 이 필요합니다.
  * 대부분의 사용자 관리 기능에는 조직 및 인증 도메인 관련 [관리 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) 이 필요합니다.

* 구성 충돌을 방지하려면 조직에서 한 번에 한 사람만 사용자를 관리하도록 하십시오.

## 사용자 관리 UI [#user-mgmt]

다음은 **사용자 관리** UI의 스크린샷입니다. 아래에서 열 헤더에 대해 설명하겠습니다.

<img
  title="User management UI"
  alt="User management UI"
  src={accountsUserMgmtUi}
/>

* **이름** : 사용자의 이름입니다.

* **이메일** : 사용자의 이메일 주소입니다. 표시될 수 있는 메시지에 대한 몇 가지 참고 사항:

  * `Pending`: 이 사용자는 New Relic에 추가되었지만 아직 처음으로 로그인하지 않았습니다. `Pending` 상태에서도 사용자는 여전히 프로비저닝된 것으로 간주됩니다. 즉, 청구 가능한 사용자 유형이 있는 경우 청구가 가능합니다.

  * <img src={accountsPendingInviteApproval} class="inline" style={{height: "30px", width: "30px"}} alt="Pending invite approval icon" title="Pending invite approval icon"/>

    이 아이콘은 `Pending invite approval` 를 나타냅니다. 이러한 사용자는 아직 청구할 수 없습니다. 팀원을 초대하는 방법에 대한 자세한 내용은 [사용자 추가](#add-users) 를 참조하십시오.

* **유형** : 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 입니다.

* **그룹** : 사용자가 속한 [그룹](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 입니다.

* **Last active**: 사용자가 New Relic에 마지막으로 로그인한 날짜입니다. 사용자의 API 활동을 추적하지 않습니다.

  <InlinePopover type="userKey"/>

  . `Last active` 사용자를 더 낮은 사용자 유형으로 다운그레이드할지 결정하는 데 유용할 수 있습니다.

* **ID** : 사용자의 New Relic이 부여한 ID.

## 일반적인 사용자 관리 작업에 대한 팁 [#workflow]

수행할 수 있는 몇 가지 사용자 관리 작업은 다음과 같습니다.

<CollapserGroup>
  <Collapser
    id="add-users"
    title="사용자 추가"
  >
    다음과 같은 몇 가지 방법으로 사용자를 추가할 수 있습니다.

    * **사용자 추가** : [**사용자 관리** UI](#where) 로 이동하여 사용자 **추가** 를 선택합니다. 사용자를 New Relic에 직접 추가하는 것은 사용자가 `Pending` 이고 아직 New Relic에 로그인하지 않았는지 여부에 관계없이 청구 가능한 사용자 유형을 가진 사용자에게 즉시 청구가 가능함을 의미합니다.

    * **사용자 초대**: [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 **사용자 추가를**클릭합니다. 사용자 초대에 대한 세부 정보:

      * [**인증 도메인** 권한](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings) 이 있는 경우 사용자를 직접 추가할 수 있는 **사용자 추가** UI로 이동합니다.
      * 해당 권한이 없는 경우 팀원을 초대할 수 있습니다. 이것은 New Relic에 추가되는 해당 사용자의 관리자에 의한 검토를 요청합니다. 검토되지 않은 대기 중인 초대 [는 사용자 관리 UI에서 `Pending invite approval`](#user-mgmt) 로 표시됩니다. 관리자가 초대된 사용자를 검토하면 사용자 **추가** 워크플로우를 완료하고 해당 사용자는 New Relic에 추가된 것으로 간주됩니다.

    사용자 추가에 대한 기타 참고 사항:

    * 또한 [NerdGraph API를 사용하여 사용자를 추가하고 관리할 수](/docs/apis/nerdgraph/examples/nerdgraph-manage-users) 있습니다.
    * [자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 ID 제공업체의 사용자만 추가할 수 있습니다.
  </Collapser>

    * [사용자의 사용자 유형을 결정하는 방법](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#choose-user-type)
    * [청구 가능한 사용자 계산 방법](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing)
    * [사용자 다운그레이드 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#user-downgrade-rules)
    * [자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 [사용자 유형을 관리하기 위한 다른 옵션이 있습니다](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade) .

    UI에서 한 명 이상의 사용자의 사용자 유형을 변경하려면:

    1. [**사용자 관리** UI](#where) 에서 사용자 유형을 수정하려는 사용자의 확인란을 클릭합니다.
    2. 사용자 선택을 시작하면 **유형 편집** 에 대한 옵션이 나타납니다.

    특정 사용자를 클릭하여 사용자 유형 및 그룹을 편집할 수도 있습니다.

    API를 사용하여 사용자 정보를 관리하는 방법 [은 NerdGraph로 사용자 관리](/docs/apis/nerdgraph/examples/nerdgraph-manage-users) 를 참조하십시오.

    2. 편집 옵션은 UI 상단에 나타납니다.
