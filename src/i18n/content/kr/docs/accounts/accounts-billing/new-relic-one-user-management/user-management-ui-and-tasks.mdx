## 사용자 관리 UI [#where]

뉴렐릭 조직의 사용자 관리 옵션을 찾으려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 에서 <DNT>**Administration**</DNT> 선택하세요. 여기에서 찾을 수 있는 사용자 관리 관련 UI 페이지는 다음과 같습니다.

* <DNT>**User management**</DNT> ([ UI링크](https://one.newrelic.com/admin-portal/organizations/users-list)): 사용자 추가, 사용자 유형 업데이트, 사용자 그룹 관리, 사용자 업그레이드 요청 승인, 세션 설정 제어에 사용됩니다.

* <DNT>**Invitation approval**</DNT>: 사용자를 뉴렐릭에 추가하도록 승인하는 곳입니다. 이러한 사용자는 아직 프로비저닝된 것으로 간주되지 않으며 요금이 청구되지 않습니다.

* <DNT>**Access management**</DNT> ([UI 링크](https://one.newrelic.com/admin-portal/organizations/organization-detail)): RBAC를 관리하기 위한 기본 인터페이스로, 4개의 전용 탭으로 구성됩니다.

  * <DNT>**Groups**</DNT>: 사용자 그룹을 생성 및 관리하고, 그룹에 사용자를 할당합니다.
  * <DNT>**Accounts**</DNT>: 조직 내에서 계정을 만들고 관리합니다.
  * <DNT>**Roles**</DNT>: 명확한 범위 선택(조직, 계정 또는 과정 수준)을 통해 사용자 정의 역할을 만들고 관리합니다.
  * <DNT>**Access Grants**</DNT>: 특정 목적, 목표에 따라 그룹을 역할에 연결하는 액세스 권한 부여를 만듭니다.

* <DNT>**API keys**</DNT>: [API 키 UI를](/docs/apis/intro-apis/new-relic-api-keys) 사용하여 사용자의 API 키를 보고 관리합니다.

* <DNT>**Domain capture**</DNT>: 사용자가 뉴렐릭에 가입하려고 할 때 캡처를 사용하여 사용자를 올바른 위치로 라우팅하세요. [도메인 캡처에 대해 자세히 알아보세요.](/docs/accounts/accounts-billing/account-setup/domain-capture)

* <DNT>**Authentication domains**</DNT> ([ UI링크](https://one.newrelic.com/admin-portal/centralized-admin-user-auto-provisioning/home)): 이를 사용하여 사용자를 추가하는 방법(뉴렐릭 또는 ID 공급자에서), 로그인하는 방법(수동 또는 SAML SSO 통해) 등을 제어합니다. 자세한 내용은 [인증 도메인을](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 참조하세요.

사용자 관리에 대한 팁은 [일반 작업](#workflow) 및 [사용자 관리 비디오](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos) 를 참조하십시오.

## 요구 사항 [#requirements]

사용자 관리 권한이 무엇인지 확인하는 가장 쉬운 방법은 [사용자 관리 UI](#where) 로 이동하여 액세스 권한이 있는 항목을 확인하는 것입니다.

일부 사용자 관리 요구 사항 및 제한 사항:

* 그룹과 역할을 추가하고 편집하려면 조직에 [Pro 또는 Enterprise](https://newrelic.com/pricing) 버전이 있어야 합니다.

* 권한 관련 요구 사항:

  * 핵심 사용자 또는 전체 플랫폼 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 이 필요합니다.
  * 대부분의 사용자 관리 권한에는 인증 도메인 관리 권한이 있는 조직 범위 역할이 필요합니다.

* 구성 충돌을 방지하려면 조직에서 한 번에 한 사람만 사용자를 관리하도록 하십시오.

## UI 세부정보 [#user-mgmt]

다음은 <DNT>**User management**</DNT> UI의 스크린샷입니다. 아래에서 열 헤더에 대해 설명하겠습니다.

<img title="User management UI" alt="User management UI" src="/images/accounts_screenshot-crop_user-mgmt-ui.webp" />

* <DNT>**Name**</DNT>: 사용자의 이름입니다.
* <DNT>**Email**</DNT>: 사용자의 이메일 주소입니다. `Pending` 사용자의 경우: 이 사용자는 추가되었지만 아직 이메일을 확인하지 않았습니다. `Pending` 상태에서도 사용자는 프로비저닝된 것으로 간주되므로 핵심 또는 전체 플랫폼 사용자인 경우 비용이 청구됩니다.
* <DNT>**Type**</DNT>: 사용자의 [사용자 유형입니다](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type).
* <DNT>**Last active**</DNT>: 사용자가 플랫폼에 마지막으로 로그인한 날짜입니다. (이는 <InlinePopover type="userKey" />가 필요한 API 사용을 추적하지 **않습니다** .) 이는 사용자를 더 낮은 사용자 유형으로 다운그레이드할지 결정하는 데 유용할 수 있습니다.
* <DNT>**ID**</DNT>: 사용자에게 뉴렐릭이 부여된 ID입니다.

사용자가 속한 그룹에 대한 정보를 보려면 UI에서 TSV 파일을 다운로드하거나 [NerdGraph를 사용할](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#query-user-groups) 수 있습니다.

## 일반적인 사용자 관리 작업에 대한 팁 [#workflow]

수행할 수 있는 몇 가지 사용자 관리 작업은 다음과 같습니다.

<CollapserGroup>
  <Collapser id="learn-about-users" title="사용자에 대해 알아보기">
    사용자에 대해 알아보려면 <DNT>**User management UI**</DNT> 으로 이동하여 특정 사용자를 자세히 조사하세요. 또한 <DNT>**Access management**</DNT> UI로 이동하여 그룹, 즉 해당 그룹이 갖고 있는 역할과 각 그룹의 사용자를 검사할 수도 있습니다.

    모든 사용자와 그룹 구성원을 보려면 [NerdGraph를 사용할](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups#query-user-groups)수 있습니다.
  </Collapser>

  <Collapser id="add-users" title="사용자 추가">
    New Relic에 사용자를 직접 추가한다는 것은 해당 사용자가 `Pending` 이고 아직 New Relic에 로그인하지 않았는지 여부에 관계없이 청구 가능한 사용자 유형의 사용자가 즉시 청구 가능하다는 것을 의미합니다. 사용자 관리 UI를 통해 사용자를 추가하는 옵션은 다음과 같습니다.

    * [<DNT>**User management**</DNT> UI](#where) 에서 <DNT>**Add user**</DNT> 선택합니다.
    * [<DNT>**Access management UI**</DNT>](#where) 에서 그룹을 생성하거나 편집할 때 해당 그룹에 사용자를 추가합니다.

    <DNT>**Invite users**</DNT> 기능을 사용할 수도 있습니다. 이를 사용하려면 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 <DNT>**Add user**</DNT> 클릭하세요. 사용자 초대에 대한 세부정보:

    * 인증 도메인 관리 권한이 있는 조직 범위 역할이 있는 경우 <DNT>**Add users**</DNT> UI로 이동하여 사용자를 직접 추가할 수 있습니다.
    * 해당 권한이 없으면 팀원을 초대할 수 있습니다. 뉴렐릭에 추가된 사용자의 관리자에게 검토를 요청합니다. 검토되지 않은 대기 중인 초대는 [사용자 관리 UI에서`Pending invite approval` ](#user-mgmt)으로 표시됩니다. 관리자가 초대된 사용자를 검토하면 <DNT>**Add user**</DNT> 워크플로우가 완성되고 해당 사용자는 뉴렐릭에 추가된 것으로 간주됩니다.

    사용자 추가에 관한 몇 가지 중요한 사항은 다음과 같습니다.

    * [NerdGraph API를 사용하여 사용자를 관리](/docs/apis/nerdgraph/examples/nerdgraph-manage-users) 할 수도 있습니다.
    * [자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 ID 제공업체의 사용자만 추가할 수 있습니다.
  </Collapser>

  <Collapser id="edit-user-type" title="사용자 유형 편집">
    사용자의 사용자 유형을 변경하기 전에 다음 사항을 이해하는 것이 좋습니다.

    * [사용자 유형을 결정하는 방법](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#choose-user-type).
    * 새 사용자를 추가하거나 사용자 유형을 조정하는 방법에 대한 팁을 포함하여[사용자 계산 방법](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing).
    * [사용자 다운그레이드 규칙](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#user-downgrade-rules).
    * [자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign)사용 ? [사용자 유형을 관리하기 위한 다른 옵션이](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade)있습니다.

    UI에서 사용자 유형을 변경하려면:

    1. [<DNT>**User management**</DNT> UI](#where) 에서 편집하려는 사용자 유형의 확인란을 클릭합니다.
    2. 사용자 선택을 시작하면 <DNT>**Edit type**</DNT> 에 대한 옵션이 나타납니다.

    특정 사용자를 클릭하여 사용자 유형 및 그룹을 편집할 수도 있습니다.

    API를 사용하여 사용자 정보를 관리하는 방법 [은 NerdGraph로 사용자 관리](/docs/apis/nerdgraph/examples/nerdgraph-manage-users) 를 참조하십시오.

    사용자가 사용자 유형을 업그레이드하는 방법을 관리하려면 [인증 도메인 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade) 을 참조하십시오.
  </Collapser>

  <Collapser id="bulk-edit-users" title="사용자 일괄 수정">
    한 번에 여러 사용자의 설정을 편집하려면:

    1. [<DNT>**User management**</DNT> UI](#where) 로 이동하여 편집하려는 그룹 또는 사용자 유형의 사용자에 대한 확인란을 선택합니다.
    2. 편집 옵션은 UI 상단에 나타납니다.

    참고:[자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용하는 경우 ID 공급업체에서 그룹을 편집해야 합니다.
  </Collapser>

  <Collapser id="bulk-import-users" title="대량 가져오기 사용자">
    여러 사용자를 뉴렐릭으로 한 번에 가져오려면 다음 옵션이 있습니다.

    * **SCIM 프로비저닝(지속적인 관리에 권장)**: Azure AD, Okta 또는 OneLogin 과 같은 ID 공급자에서 사용자 및 그룹을 자동으로 뉴렐릭과 동기화하도록 [자동화된 사용자 관리를](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign/#how-to) 설정합니다. 이는 대량으로 사용자를 관리하고 동기화를 유지하려는 조직에 가장 적합한 옵션입니다. [Pro 또는 에디션 에디션이](https://newrelic.com/pricing) 필요합니다.

    * **SCIM API (일회성 대량 가져오기 또는 맞춤 통합용)**: [SCIM API](/docs/accounts/accounts/automated-user-management/scim-support-automated-user-management/#create-user) 사용하여 프로그래밍 방식으로 사용자를 생성합니다.

    * **NerdGraph API(스크립트된 가져오기용)**: [NerdGraph API를](/docs/apis/nerdgraph/examples/nerdgraph-manage-users/#create-users) 사용하여 사용자를 개별적으로 생성하고, 대량 작업을 위해 스크립팅할 수 있습니다.

    <Callout variant="tip">
      뉴렐릭은 현재 대량 사용자 가져오기에 대한 직접 CSV 또는 스프레드시트 업로드를 지원하지 않습니다. 대량 수입의 경우 위에 나열된 자동화된 방법 중 하나를 사용하세요.
    </Callout>
  </Collapser>

  <Collapser id="add-user-managers" title="다른 사용자를 관리할 사용자 지정">
    사용자에게 다른 사용자를 관리할 수 있는 기능을 제공하는 데는 두 가지 옵션이 있습니다.

    * 인증이 있는 조직 범위 역할 관리 권한은 사용자, 그룹 및 인증 관리와 관련된 조직 전반의 기능을 부여합니다.

    * [그룹 관리자](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#group-admin) 역할은 훨씬 더 제한적입니다. 즉, 사용자에게 특정 그룹의 사용자를 추가 및 제거할 수 있는 기능을 제공합니다.

      조직 전체에 사용자 관리 권한을 부여하려면 조직 범위 역할과 해당 권한이 있는 기본 <DNT>**Admin**</DNT> 그룹에 사용자를 추가할 수 있습니다. 사용자 정의 그룹을 만들고 인증 도메인 관리 권한이 있는 조직 범위 역할을 할당할 수도 있습니다.

      일반적인 사용자 관리 작업에 대한 튜토리얼은 [사용자 관리 튜토리얼을](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial) 참조하세요.
  </Collapser>

  <Collapser id="delete-users" title="사용자 삭제">
    사용자를 삭제하려면 [<DNT>**User management**</DNT> UI](#where) 로 이동하여 한 명 이상의 사용자에 대한 확인란을 선택하세요. 그런 다음 <DNT>**Delete users**</DNT>) 클릭하세요.

    사용자를 삭제하면 다음 사용자 자산이 제거됩니다.

    * 즐겨찾기
    * 주간 이메일 설정
    * 이메일 수신 동의/선택 해제 기본 설정
    * 사용자별 [사용자 키](/docs/apis/intro-apis/new-relic-api-keys/#user-api-key)
    * 뉴렐릭 앱 [NerdStorage 데이터](/docs/new-relic-solutions/build-nr-ui/nerdstorage)
  </Collapser>

  <Collapser id="export-users" title="사용자 스프레드시트 내보내기">
    모든 사용자 목록 및 세부정보(이름, 이메일 주소, 현재 사용자 유형, 그룹, 사용자 ID)를 내보내려면: [<DNT>**User management**</DNT> UI](#where) 에서 <DNT>**Add user**</DNT> 버튼 옆에 있는 다운로드 아이콘을 클릭합니다. 해당 인증 도메인의 모든 사용자 목록을 다운로드하는 옵션이 있습니다.

    또는 일부 사용자를 내보내려면 먼저 내보낼 사용자를 선택한 다음 다운로드 버튼을 누르십시오.
  </Collapser>

  <Collapser id="access-grants" title="사용자에게 계정 및 역할에 대한 액세스 권한 부여">
    [사용자 관리 튜토리얼](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model) 을 참조하십시오.
  </Collapser>

  <Collapser id="new-group" title="새 사용자 지정 그룹 및 역할 만들기">
    [사용자 관리 튜토리얼](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model) 을 참조하십시오.
  </Collapser>

  <Collapser id="new-group" title="SAML SSO 및/또는 SCIM 프로비저닝 설정">
    SAML SSO 사용하여 단일 로그인 ID 공급자를 사용하여 뉴렐릭에 로그인할 수 있습니다. 대량 사용자 관리의 경우 SCIM 프로비저닝(자동화된 사용자 관리라고도 함)을 사용하는 것이 좋습니다. 이를 통해 ID 공급자에서 사용자 및 그룹을 자동으로 뉴렐릭과 동기화할 수 있습니다.

    SCIM 프로비저닝은 대량 사용자 관리에 다음과 같은 이점을 제공합니다.

    * ID 공급자로부터 대량의 사용자를 자동으로 가져와 동기화합니다.
    * 사용자 정보를 자동으로 최신 상태로 유지
    * ID 공급자를 통해 중앙에서 사용자 및 그룹을 관리하세요.

    SAML SSO 또는 SCIM 프로비저닝을 설정하려면 [SAML SSO 또는 SCIM 시작하기를](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-saml-scim) 참조하세요. 이러한 기능을 사용하려면 [Pro 또는 부품 버전이](https://newrelic.com/pricing) 필요합니다.
  </Collapser>
