New Relic 사용자가 New Relic에 추가되는 방식, 관리 방식 및 로그인 방식에 대한 자동 제어를 설정하기 위해 다음 기능을 제공합니다.

* **SAML SSO** : 이를 통해 사용자는 기본 이메일 + 비밀번호를 사용하는 대신 싱글 사인온(SSO) ID 제공자 서비스를 사용하여 New Relic에 로그인할 수 있습니다.

  * 요구 사항: [Pro 또는 Enterprise 에디션](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).

* 자동화된 사용자 관리라고도 하는 **SCIM** 프로비저닝: SCIM 프로비저닝을 사용하면 조직에서 ID 제공자 서비스를 사용하여 사용자가 New Relic에 추가되고 업데이트되는 방식을 자동화할 수 있습니다. SCIM을 사용하는 조직은 거의 항상 SAML SSO를 사용합니다.

  * 요구 사항: [Pro 또는 Enterprise 에디션](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions), 이를 사용하면 최신 사용자 모델에 사용자 레코드가 추가됩니다(자세한 내용은 아래 참조).

이러한 기능을 활성화하기 전에 이러한 기능에 영향을 미치는 중요한 요소인 사용자가 속한 사용자 모델을 이해하는 것이 중요합니다.2020년 [에는 새롭고 향상된 사용자 모델을 출시했습니다](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) .처음에는 주로 신규 고객에게 이 신규 사용자 모델의 사용자가 있었지만 이제는 [사용자 마이그레이션 절차](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 를 통해 일부 이전 고객이 사용할 수 있습니다.시간이 지나면 모든 New Relic 사용자가 이 모델로 마이그레이션됩니다.

SAML 및 SCIM을 활성화하는 방법에 대한 정보를 보려면 아래에서 사용자 모델을 선택하십시오.현재 사용 중인 모델이 확실하지 않은 경우 [사용자 모델 개요 를](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 참조하세요.

  <Collapser
    id="original-account-model"
    title="원래 사용자 모델"
  >
    조직이 원래 사용자 모델을 사용하는 경우 가능한 경우 [셀프 서비스 마이그레이션 절차](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 를 통해 사용자를 마이그레이션하는 것이 좋습니다.요구 사항을 충족하지 않지만 SAML 및/또는 SCIM을 활성화하려면 New Relic 계정 담당자에게 문의하세요.

    다음은 몇 가지 팁과 문서 링크입니다.

    * SAML: 원래 사용자 모델을 유지하고 싶거나 유지해야 하고 새 모델로 마이그레이션할 수 없는 경우 [원래 SAML 문서](/docs/accounts/accounts/saml-single-sign/saml-service-providers) 를 참조하세요.
    * SCIM + SAML: SCIM 프로비저닝은 최신 사용자 모델에서 사용자 레코드를 생성합니다.이것이 SCIM을 사용하기 전에 먼저 사용자를 새 사용자 모델로 마이그레이션하는 것이 좋습니다.새 모델로 마이그레이션하는 방법에 대한 자세한 내용은 [사용자 마이그레이션](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 을 참조하십시오.
  </Collapser>
