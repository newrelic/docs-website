New Relic 사용자가 New Relic에 추가되는 방식, 관리 방식 및 로그인 방식에 대한 자동 제어를 설정하기 위해 다음 기능을 제공합니다.

* <DNT>**SAML SSO**</DNT>: 이를 통해 사용자는 기본 이메일 + 비밀번호를 사용하는 대신 싱글 사인온(SSO) ID 제공자 서비스를 사용하여 뉴렐릭에 로그인할 수 있습니다. [유료 버전](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) 이 필요합니다.
* <DNT>**SCIM provisioning**</DNT>, 자동화된 사용자 관리라고도 함: SCIM 프로비저닝을 통해 조직은 ID 제공자 서비스를 사용하여 뉴렐릭에서 사용자가 추가되고 업데이트되는 방식을 자동화할 수 있습니다. SCIM을 사용하는 조직은 거의 항상 SAML SSO를 사용합니다. [Pro 또는 에디션 에디션이](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) 필요합니다.

SAML 및 SCIM을 활성화하는 방법에 대한 자세한 내용은 [인증 도메인 UI를](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 사용하여 SAML 및 SCIM을 설정합니다. Okta, OneLogin 및 Azure에는 전용 뉴렐릭 앱이 있습니다. 이러한 앱을 사용하는 경우 **New Relic by account**가 아닌 **New Relic by Organization** 이라는 앱을 선택해야 합니다.
