New Relic 사용자는 동일한 이메일 주소를 사용하여 둘 이상의 사용자 기록을 보유할 수 있습니다. 다음과 같은 경우가 있습니다.

* 둘 이상의 [New Relic 조직](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) 에 대한 액세스
* New Relic 조직에서 둘 이상의 [인증 도메인](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 에 대한 액세스

## 사용자 레코드 간 전환 [#switch]

New Relic 로그인 시 [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 로 이동하여 동일한 이메일 주소를 가진 다른 사용자 기록으로 전환할 수 있습니다. 메뉴에서 사용자 레코드를 선택하면 해당 사용자 레코드에 로그인하라는 메시지가 표시됩니다(New Relic에 직접 또는 ID 공급자를 통해).

이메일 주소가 다른 사용자 기록에 액세스하려면 로그아웃했다가 다시 로그인해야 합니다.

## "여러 로그인을 찾았습니다" 메시지 [#login-message]

이메일이 여러 사용자 레코드와 연결된 경우 로그인할 때 `Multiple logins found` 메시지가 표시되고 이메일을 확인하라는 메시지가 표시됩니다. 이는 여러 New Relic 조직 또는 조직의 여러 [인증 도메인](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 에 대한 액세스 권한이 있는 경우 발생할 수 있습니다.

<figcaption>
  동일한 이메일 주소에 대해 두 개 이상의 사용자 기록이 있는 경우 로그인하는 동안 이에 대한 메시지를 받게 됩니다.
</figcaption>

위와 같이 로그인 페이지에서 사용자 기록이 `Original` 으로 표시되면 사용자 기록이 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 있음을 의미합니다.

이메일 주소를 확인하면 액세스할 사용자 레코드를 선택할 수 있습니다. 이메일 주소를 확인하지 않으면 New Relic은 이메일 주소와 연결된 가장 최근에 생성된 사용자 기록으로 로그인을 시도합니다.

로그인할 때마다 이메일을 확인하지 않으려면 로그인 화면에서 **나 기억하기** 확인란을 클릭하십시오.

로그인 문제에 대한 자세한 내용은 로그인 문제 [해결](/docs/accounts/accounts-billing/account-setup/troubleshoot-new-relics-password-email-address-login-problems) 을 참조하십시오.

## 사용자 기록 삭제 [#delete]

실수로 New Relic에 가입한 [경우 조직 삭제](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-unused-org) 를 참조하세요.

사용자 기록을 삭제하려면 [사용자 관리 UI](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks) 를 사용하세요.
