일부 특정 로그인 문제에 대한 문제 해결 팁이 포함된 이 문서입니다. New Relic 기능 및 데이터에 대한 액세스에 영향을 미치는 요소에 대한 개요는 액세스에 영향을 미치는 [요소](/docs/factors-affecting-access-features-data) 를 참조하십시오. (아직 New Relic 계정이 없으신가요? [여기에서 가입하세요](https://newrelic.com/signup) . 무료입니다!)

## 계정 액세스 문제 [#account-access]

이미 New Relic에 로그인했지만 예상한 계정이나 데이터가 표시되지 않으면 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data#account-access) 를 참조하세요.

## 비밀번호 솔루션 [#solution]

New Relic에 로그인하고 New Relic에서 정보를 받으려면 유효한 이메일 주소와 [유효한 비밀번호](/docs/accounts-partnerships/accounts/account-maintenance/change-passwords-user-preferences#requirements) 가 필요합니다. 계정을 성공적으로 만들 수 없거나 암호 또는 기타 로그인 문제가 있는 경우 이러한 문제 해결 제안을 검토하여 해결 방법을 확인하십시오.

<CollapserGroup>
  <Collapser
    id="forgot-password"
    title="비밀번호를 잊어버렸거나 비밀번호가 작동하지 않습니다."
  >
    New Relic 계정 비밀번호를 잊어버렸거나 비밀번호가 작동하지 않는 경우 New Relic에 비밀번호 재설정을 요청하세요. 여러 New Relic 로그인에 액세스할 수 있는 경우 사용 중인 로그인 방법에 대해 올바른 비밀번호를 사용하고 있는지도 확인하십시오.

    1. [New Relic의 로그인 페이지](https://login.newrelic.com/login) 로 이동합니다.

    2. **비밀번호 찾기를** 선택합니다.

    3. 계정 이메일 주소를 입력하고 **내 재설정 링크 보내기** 를 선택합니다.

    4. New Relic으로부터 확인 이메일을 받으면 비밀번호 재설정 링크를 선택하고 [New Relic의 비밀번호 요구 사항](/docs/accounts-partnerships/accounts/account-maintenance/change-passwords-user-preferences#requirements) 에 따라 비밀번호 재설정 프로세스를 완료하십시오.

       <Callout variant="important">
         비밀번호 재설정 링크는 12시간 후에 만료됩니다. New Relic에서 이메일을 빨리 받지 못하면 [스팸 필터](/docs/accounts-partnerships/accounts/account-setup/create-your-new-relic-account#email-whitelist) 를 확인하고 조직의 이메일 관리자에게 문제 해결 제안을 문의하고 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으십시오.
       </Callout>
  </Collapser>

  <Collapser
    id="reset-password"
    title="암호를 재설정."
  >
    자신의 비밀번호를 잊어버렸거나 계정 이메일에 대한 비밀번호 재설정을 요청해야 하는 경우 New Relic의 [셀프 서비스 옵션을](#forgot-password) 사용할 수 있습니다. 관리자 수준의 사용자는 다른 사용자의 비밀번호를 재설정할 수 없습니다. 다른 사람의 비밀번호를 재설정해야 하는 경우 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으십시오.
  </Collapser>

  <Collapser
    id="account-login-errors"
    title="비밀번호 오류 메시지."
  >
    가입 절차를 완료했지만 비밀번호 오류 메시지로 인해 계정에 로그인할 수 없는 경우 [support.newrelic.com](https://support.newrelic.com/home) 에서 지원을 받으십시오.
  </Collapser>

  <Collapser
    id="expired-password"
    title="비밀번호 재설정 링크가 만료되었습니다."
  >
    비밀번호를 변경하고 싶지만 비밀번호 재설정 링크가 만료되었다는 메시지가 표시되면 개인 브라우저를 사용하거나 브라우저 캐시 및 쿠키를 지워보세요.

    <Callout variant="important">
      비밀번호 재설정 링크는 12시간 후에 만료됩니다. 재설정 링크를 선택한 후 New Relic에서 이메일을 빨리 받지 못하면 [스팸 필터](/docs/accounts-partnerships/accounts/account-setup/create-your-new-relic-account#email-whitelist) 를 확인하고 조직의 이메일 관리자에게 문제 해결 제안을 문의하고 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으십시오.
    </Callout>
  </Collapser>

  <Collapser
    id="existing-user"
    title="다른 New Relic 계정에 기존 사용자를 추가했습니다."
  >
    기존 New Relic 계정 사용자이고 [다른 계정에 추가된](/docs/accounts/install-new-relic/account-setup/use-multiple-accounts) 경우 새 비밀번호를 만들 필요가 없습니다. 기존 비밀번호는 각 계정에 대해 작동하며 로그인 후 [계정 간에 전환](/docs/accounts/install-new-relic/account-setup/use-multiple-accounts#switch-accounts) 할 수 있습니다.
  </Collapser>
</CollapserGroup>

## 이메일 솔루션 [#solution-email]

New Relic에 로그인하고 New Relic에서 정보를 받으려면 유효한 이메일 주소와 [유효한 비밀번호](/docs/accounts-partnerships/accounts/account-maintenance/change-passwords-user-preferences#requirements) 가 필요합니다. 계정을 성공적으로 만들 수 없거나 이메일 또는 기타 로그인 문제가 있는 경우 이 문제 해결 제안을 검토하여 해결 방법을 찾으십시오.

<CollapserGroup>
  <Collapser
    id="no-confirmation-email"
    title="새 계정 확인 이메일을 받지 못했습니다."
  >
    처음 계정을 만들 때 New Relic은 설정 프로세스를 완료하고 로그인할 수 있도록 확인 이메일을 보냅니다. 원래 계정 확인 이메일을 찾을 수 없는 경우:

    1. new Relic의 로그인 페이지( **[login.newrelic.com/login](https://login.newrelic.com/login)** )로 이동합니다.

    2. **비밀번호 찾기를** 선택합니다.

    3. 계정 이메일 주소를 입력하고 **내 비밀번호 보내기** 를 선택합니다.

    4. New Relic의 시스템에서 이메일 메시지를 반환하면 해당 링크를 선택하여 계정을 다시 확인하십시오.

       New Relic에서 이메일을 받지 못한 경우:

    * [스팸 필터](/docs/accounts-partnerships/accounts/account-setup/create-your-new-relic-account#email-whitelist) 를 확인하십시오. 해당하는 경우 [이메일 허용 목록에 New Relic을 추가합니다](/docs/accounts/accounts/account-maintenance/account-email-settings#email-whitelist) .
    * [support.newrelic.com](https://support.newrelic.com/home) 에서 지원을 받으세요.
  </Collapser>

  <Collapser
    id="forgot-email"
    title="이메일 주소가 작동하지 않거나 이메일 오류 메시지가 표시됩니다."
  >
    가입 절차를 완료했지만 이메일 또는 비밀번호 오류 메시지로 인해 계정에 로그인할 수 없는 경우 [support.newrelic.com](https://support.newrelic.com/home) 에서 지원을 받으십시오.
  </Collapser>

  <Collapser
    id="account-exists"
    title="귀하의 이메일 계정이 이미 존재한다는 메시지를 받았습니다."
  >
    새 계정을 만들려고 하는데 이메일 계정이 이미 있다는 메시지를 받으면 [support.newrelic.com](https://support.newrelic.com/home) 에서 지원을 받으십시오.
  </Collapser>

  <Collapser
    id="link-redirect"
    title="이메일 링크가 비밀번호 재설정 페이지로 리디렉션되지 않습니다."
  >
    비밀번호 재설정 링크를 사용했지만 비밀번호 재설정 페이지로 리디렉션되지 않으면 다음 문제를 해결하십시오.

    * 비밀번호 재설정 링크가 만료되었습니다. New Relic 웹사이트에서 [새 링크를 요청](#forgot-password) 해야 합니다.

    * 캐싱 문제가 발생했습니다. 링크를 다시 시도하기 전에 브라우징 캐시를 지우거나 개인 브라우징 창을 사용하십시오.

      <Callout variant="tip">
        **시크릿 모드** 라고도 하는 **개인** 정보 탐색은 탐색 기록 및 웹 캐시를 비활성화하는 개인 정보 보호 기능입니다. 시크릿 브라우징 창을 열려면 대부분의 브라우징 응용 프로그램에 대해 Windows의 경우 키보드 단축키 `CTRL+Shift+N` , Mac의 경우 `Command+Shift+N` 를 사용할 수 있습니다.
      </Callout>

      그래도 문제가 해결되지 않으면 [support.newrelic.com](https://support.newrelic.com/home) 에서 지원을 받으십시오.
  </Collapser>

  <Collapser
    id="verify-email"
    title="로그인 시 이메일 확인을 요청합니다."
  >
    이메일이 여러 계정과 연결된 경우 로그인하는 동안 이메일을 확인할 수 있는 옵션이 제공됩니다. 이렇게 하면 액세스할 계정을 선택할 수 있습니다. 이메일 주소를 확인하지 않으면 New Relic은 이메일 주소와 연결된 가장 최근에 생성된 사용자 기록으로 로그인을 시도합니다.

    로그인할 때마다 이메일을 확인하지 않으려면 로그인 화면에서 **나 기억하기** 확인란을 클릭하십시오.
  </Collapser>
</CollapserGroup>

## 일반 시스템 솔루션 [#solution-system]

일부 일반적인 시스템 문제 및 솔루션:

<CollapserGroup>
  <Collapser
    id="active-session-limit"
    title="활성 세션 제한에 도달했습니다."
  >
    New Relic 사용자는 주어진 시간에 최대 3개의 동시 활성 세션을 사용할 수 있습니다. 이 컨텍스트에서 세션은 New Relic 플랫폼에 대한 고유한 로그인을 나타냅니다. 세션은 해당 세션에 할당된 토큰으로 추적됩니다. 한도를 초과하면 로그인 시 `Active session limit reached` 메시지가 표시되고 열려 있는 세션을 닫는 옵션이 표시됩니다.

    사용자 세션을 제어하는 방법은 [세션 설정](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more#session-mgmt) 을 참조하십시오.
  </Collapser>

  <Collapser
    id="two-week-option"
    title="SAML 싱글 사인온(SSO) 문제로 인해 로그인이 차단됩니다."
  >
    조직에서 SAML SSO(Single Sign On) 솔루션을 사용하는 경우 로그인할 때 **암호** 필드를 건너뛸 수 있습니다. 암호를 재설정해야 하는 경우 조직의 시스템 관리자나 해당하는 IT 부서에 문의하십시오.

    최근에 SAML SSO 설정을 활성화하거나 변경한 관리자가 로그인할 수 없는 경우 구성에 문제가 있을 수 있습니다.[최신 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 을 사용하는 고객은 특별한 복구 흐름을 사용하여 문제를 해결할 수 있습니다.[login.newrelic.com/recovery_access를](http://login.newrelic.com/recovery_access) 방문하십시오.SSO를 우회하고 조직에 대한 일회성 액세스 권한을 얻습니다.
  </Collapser>

  <Collapser
    id="failure-errors"
    title="시스템 오류 오류로 인해 로그인이 불가능합니다."
  >
    가입을 시도하는 동안 실패 오류가 발생하면 타사 암호 관리자가 New Relic의 스팸 트랩을 트리거할 수 있습니다. 이 문제를 해결하려면 다음 솔루션을 시도하십시오.

    * 암호 관리자를 우회하십시오.
    * 다른 브라우저를 사용하여 New Relic에 가입하세요.
    * [support.newrelic.com](https://support.newrelic.com/home) 에서 지원을 받으세요.
  </Collapser>
</CollapserGroup>

## 모바일 장치 솔루션 [#mobile-errors]

모바일 장치에서 로그인할 수 없는 경우 새 사용자 인증 기간이 만료되었을 수 있습니다. 모바일 기기에 대한 New Relic의 확인 메시지를 받은 후 20분 이내에 이 프로세스를 완료해야 합니다.

이 문제를 해결하려면 장치로 보낼 다른 확인 메시지를 요청하십시오. 또한, New Relic 계정에 따라 [iOS](/docs/mobile-apps/new-relic-ios-app/features/ios-user-settings-authentication) 또는 [Android](/docs/mobile-apps/new-relic-android-app/features/android-authentication) 앱 계정에 추가 설치 또는 인증 단계가 필요할 수 있습니다.
