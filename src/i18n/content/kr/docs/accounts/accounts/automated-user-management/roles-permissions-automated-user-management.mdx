[자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 통해 ID 공급자(예: OneLogin 또는 Okta)의 사용자 및 그룹이 New Relic과 동기화됩니다.

## 그룹 작동 방식 [#group-example]

[자동화된 사용자 관리](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) 를 사용할 때 ID 공급자 서비스에서 사용자 그룹을 만듭니다. 나중에 New Relic 측에서 "이 New Relic 계정에서 이 New Relic 역할에 대한 [액세스](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 권한을 이 그룹에 부여합니다."

  ID 공급자의 사용자 그룹이 특정 계정의 특정 역할(기본 또는 고객)에 그룹 액세스 권한을 할당하는 New Relic의 [액세스 권한 부여](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 에 매핑되는 방법의 예입니다.

위의 예에서는 Engineers 그룹에 중점을 두겠습니다. SCIM을 통해 New Relic은 두 개의 사용자 기록과 그들이 Engineers 그룹에 속한 정보를 받게 됩니다. 이 예에서 구성은 다음과 같습니다. 두 명의 엔지니어 사용자는 계정 1에서 [**읽기 전용** 역할](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) , 계정 2에서 [**표준 사용자**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) 역할, 계정 3에서 [**모든 제품 관리자**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) 역할을 갖습니다.

이 액세스는 New Relic에서 세 가지 [액세스 권한](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 을 생성하여 부여됩니다.

* 계정 1의 **엔지니어** 그룹에 **읽기 전용** 역할을 할당하는 액세스 권한 부여
* 계정 2의 **Engineers** 그룹에 역할 **Standard 사용자** 를 할당하는 액세스 권한 부여
* **모든 제품 관리자** 역할을 계정 3의 **엔지니어** 그룹에 할당하는 액세스 권한 부여

액세스 권한 부여의 작동 방식에 대한 자세한 내용은 [사용자 관리 개념](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 을 참조하세요.

새 그룹 및 사용자 지정 역할 생성에 대한 자습서는 [사용자 관리 자습서](/docs/accounts/accounts-billing/new-relic-one-user-management/tutorial-add-new-user-groups-roles-new-relic-one-user-model/) 를 참조하십시오.
