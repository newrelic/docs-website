New Relic은 다양한 기간 동안 다양한 유형의 데이터를 저장합니다. 데이터 유형의 보존 기간은 제품, 구독 수준 및 기능에 따라 다릅니다.

## 데이터 보존 UI [#find]

데이터 보존을 이해하고 사용 가능한 옵션을 확인하는 가장 좋은 방법은 UI로 이동하는 것입니다. 이것이 권장되는 한 가지 이유는 조직에서 설정을 편집했으며 더 이상 여기에 설명된 기본 설정이 아닐 수 있기 때문입니다.

**데이터 보존** UI 페이지는 [데이터 관리 허브](/docs/data-apis/manage-data/manage-your-data) 에 있습니다. 보존 **편집** 을 클릭하여 보존 값을 조정합니다.

## 데이터 편집 및 삭제 제한 [#edit-limits]

[원격 측정 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types) (이벤트, 메트릭, 로그, 추적)가 New Relic에 보고되고 쿼리에 사용할 수 있게 되면 해당 데이터를 편집하거나 삭제할 수 없습니다. 이것은 New Relic의 속도와 성능을 최적화하는 의도적인 설계 결정입니다. 데이터는 [데이터 보존](/docs/manage-stored-data) 이 종료되면 만료됩니다.

제거해야 하는 원치 않는 데이터나 민감한 데이터를 New Relic에 보낸 경우 New Relic의 계정 담당자에게 문의하거나 [support.newrelic.com](https://support.newrelic.com/) 에서 지원을 받으십시오.

    ## 통합 컴퓨팅 유닛 이벤트 제한 [#event-limit]

    월별 컴퓨팅 단위당 통합 이벤트에 대한 2275 제한은 **전체** 인프라 통합 이벤트에 대한 제한입니다. 각 통합의 이벤트에 대해 2275개의 제한이 없습니다.

    이 한도에 대한 추가 세부정보 및 설명:

    * 이 제한은 모든 통합(클라우드 통합 및 호스트 내 통합)의 모든 이벤트에 적용됩니다. 이벤트는 모두 동일하게 처리됩니다.
    * 인프라 에이전트가 수신한 기본 데이터는 계산 단위당 2275 이벤트 제한에 포함되지 않습니다.
    * 한도를 초과하면 이 한도가 적용되지 않습니다. 한도를 초과하는 경우 조직의 통합에 대해 가장 비용 효율적인 가격을 얻을 수 있도록 가격 옵션을 검토합니다.
  </Collapser>

    * APM, 브라우저, 모바일 및 서버리스 제품
    * 에이전트 API 또는 [이벤트 API](/docs/insights/insights-data-sources/custom-data/introduction-event-api) 에서 가져온 [사용자 지정 이벤트](/docs/insights/insights-data-sources/custom-data/report-custom-event-data)

    Insights 구독이 관리하는 데이터 보존을 보려면 [사용량 UI](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing#usage-ui) 로 이동하여 **Insights usage** \[nsights 사용량을] 선택합니다.

    Insights Pro 구독을 사용하면 [유연한 보관을 사용하여 이벤트 데이터가 보관되는 기간을 사용자 지정할](/docs/accounts/accounts/subscription-pricing/flexible-retention-define-how-long-new-relic-retains-event-data) 수 있습니다. 이렇게 하면 필요한 기간 동안 필요한 데이터만 보관할 수 있습니다.

    <CollapserGroup>
      <Collapser
        id="retention-calculation"
        title="저장된 이벤트 수 계산 방법"
      >
        Insights Pro 구독에 대해 기본적으로 저장된 이벤트 수가 어떻게 계산되는지에 대한 설명입니다. ( [유연한 보관](/docs/accounts/accounts/subscription-pricing/flexible-retention-define-how-long-new-relic-retains-event-data) 기능을 사용하면 보관 기간을 보다 세부적으로 제어할 수 있습니다.)

        저장된 이벤트는 1) 시간 경과에 따라 저장된 총 이벤트(주당 생성된 이벤트를 기반으로 계산) 및 2) 사용 가능한 데이터 보존 기간을 기반으로 계산됩니다. 이 방정식은 다음과 같이 나타낼 수 있습니다.

        ```
        events stored = (events generated per week) * (weeks of retention)
        ```

        Insights Pro 구독은 주어진 데이터 보존 기간과 해당 보존 기간 동안 주어진 이벤트 수를 제공합니다.

        예를 들어:

        * (주당 트랜잭션 2억 개) \* (보존 기간 4주) = Insights에 저장된 이벤트 8억 개
        * (주당 1,600만 트랜잭션) \* (50주 보존) = Insights에 저장된 8억 이벤트

        Insights Pro 구독의 경우 데이터는 볼륨이 아닌 보존 기간에 따라 제거됩니다. 보존 기간이 지나면 시스템에서 삭제됩니다.

        예: Insights 라이선스가 4주 보존 기간의 8억 이벤트에 대한 것인 경우 데이터는 4주가 지난 후에 제거되기 시작합니다. 구독 수준을 초과하는 데이터의 일시적인 급증은 계속 기록되지만 일관된 초과분은 구독 수준을 업그레이드하거나 수집된 데이터를 줄여 해결해야 합니다.

        [Insights Pro 구독](#insights-pro) 이 없는 고객의 경우 New Relic은 분당 호스트당 4,000개 이하의 이벤트로 이벤트를 조절하거나 다운샘플링할 수 있습니다.
      </Collapser>

      <Collapser
        id="usage-overage"
        title="Insights Pro 이벤트 초과분 예시"
      >
        이 예에서는 4주 동안 8억 개의 이벤트(주당 2억 개의 이벤트 비율)에 대한 라이선스가 있는 Insights Pro 구독이 있습니다. APM Pro, Browser Pro 및 Mobile Enterprise가 있습니다. 구독을 통해 다섯 번째 주의 데이터가 추가되어 계획에 저장된 총 10억 개의 이벤트가 발생합니다.

        * 9억 7,500만 이벤트를 사용하고 있다면 보유 기간 **이 끝나지 않았습니다** .
        * 12억 5천만 개의 이벤트를 사용하고 있다면 보유 기간 **을** 초과한 것입니다.
      </Collapser>
    </CollapserGroup>
  </Collapser>
