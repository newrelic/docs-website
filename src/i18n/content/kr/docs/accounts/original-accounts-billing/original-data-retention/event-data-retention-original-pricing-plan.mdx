[원래 가격 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models) 을 사용하는 조직의 경우 다양한 제품에서 광범위한 [이벤트 데이터](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) 를 보고합니다. 제품마다 데이터 보존 기간이 다르고 이벤트 데이터 보존을 연장하는 방법도 다릅니다. [유연한 이벤트 보존](#flex-retention) 을 통해 이벤트 데이터 보존 기간을 사용자 정의할 수 있습니다.

## 데이터 보존 UI [#find]

데이터 보존을 이해하고 사용 가능한 옵션을 확인하는 가장 좋은 방법은 UI로 이동하는 것입니다. 이것이 권장되는 한 가지 이유는 조직에서 설정을 편집했으며 더 이상 여기에 설명된 기본 설정이 아닐 수 있기 때문입니다.

**데이터 보존** UI 페이지는 [데이터 관리 허브](/docs/data-apis/manage-data/manage-your-data) 에 있습니다.

## 이벤트 데이터 보존 개요 [#overview-event-retention]

원래 가격 모델의 경우 모든 New Relic 제품 구독에는 다양한 유형의 데이터가 보존되는 기간을 제어하는 특정 수준의 [데이터 보존](/docs/accounts/accounts/data-management/data-retention-components#products) 이 제공됩니다.

다음은 다양한 제품 구독이 이벤트 데이터 보존에 미치는 영향에 대한 몇 가지 예입니다.

* 무료/라이트 APM 구독: 기본 보고 이벤트는 1일 동안 사용 가능합니다. 사용 가능한 맞춤 이벤트가 없습니다.
* Pro APM 구독: 기본 보고 이벤트는 8일 동안 사용 가능합니다. 1일 동안 사용할 수 있는 [사용자 지정 이벤트](/docs/insights/insights-data-sources/custom-data/report-custom-event-data) ( [Insights Pro](#insights-pro) 로 연장 가능).

## 인사이트 프로

원래 가격 모델을 사용하는 조직의 경우 유료 Insights 구독은 다음에 대한 이벤트 데이터 보존 연장을 관리합니다.

* APM, 브라우저, 모바일 및 서버리스 제품
* 에이전트 API 또는 [이벤트 API](/docs/insights/insights-data-sources/custom-data/introduction-event-api) 에서 가져온 [사용자 지정 이벤트](/docs/insights/insights-data-sources/custom-data/report-custom-event-data)

Insights 구독에 의해 관리되는 데이터 보존을 보려면 [사용 UI](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/billing-ui) 로 이동하여 **Insights 사용** 을 선택합니다.

Insights Pro 구독을 사용하면 [유연한 보관을 사용하여 이벤트 데이터가 보관되는 기간을 사용자 지정할](/docs/accounts/accounts/subscription-pricing/flexible-retention-define-how-long-new-relic-retains-event-data) 수 있습니다. 이렇게 하면 필요한 기간 동안 필요한 데이터만 보관할 수 있습니다.

<CollapserGroup>
  <Collapser
    id="retention-calculation"
    title="저장된 이벤트 수 계산 방법"
  >
    Insights Pro 구독에 대해 기본적으로 저장된 이벤트 수가 어떻게 계산되는지에 대한 설명입니다. ( [유연한 보관](/docs/accounts/accounts/subscription-pricing/flexible-retention-define-how-long-new-relic-retains-event-data) 기능을 사용하면 보관 기간을 보다 세부적으로 제어할 수 있습니다.)

    저장된 이벤트는 1) 시간 경과에 따라 저장된 총 이벤트(주당 생성된 이벤트를 기반으로 계산) 및 2) 사용 가능한 데이터 보존 기간을 기반으로 계산됩니다. 이 방정식은 다음과 같이 나타낼 수 있습니다.

    ```
    events stored = (events generated per week) * (weeks of retention)
    ```

    Insights Pro 구독은 주어진 데이터 보존 기간과 해당 보존 기간 동안 주어진 이벤트 수를 제공합니다.

    예를 들어:

    * (주당 트랜잭션 2억 개) \* (보존 기간 4주) = Insights에 저장된 이벤트 8억 개
    * (주당 1,600만 트랜잭션) \* (50주 보존) = Insights에 저장된 8억 이벤트

    Insights Pro 구독의 경우 데이터는 볼륨이 아닌 보존 기간에 따라 제거됩니다. 보존 기간이 지나면 시스템에서 삭제됩니다.

    예: Insights 라이선스가 4주 보존 기간의 8억 이벤트에 대한 것인 경우 데이터는 4주가 지난 후에 제거되기 시작합니다. 구독 수준을 초과하는 데이터의 일시적인 급증은 계속 기록되지만 일관된 초과분은 구독 수준을 업그레이드하거나 수집된 데이터를 줄여 해결해야 합니다.

    [Insights Pro 구독](#insights-pro) 이 없는 고객의 경우 New Relic은 분당 호스트당 4,000개 이하의 이벤트로 이벤트를 조절하거나 다운샘플링할 수 있습니다.
  </Collapser>

  <Collapser
    id="usage-overage"
    title="Insights Pro 이벤트 초과분 예시"
  >
    이 예에서는 4주 동안 8억 개의 이벤트(주당 2억 개의 이벤트 비율)에 대한 라이선스가 있는 Insights Pro 구독이 있습니다. APM Pro, Browser Pro 및 Mobile Enterprise가 있습니다. 구독을 통해 다섯 번째 주의 데이터가 추가되어 계획에 저장된 총 10억 개의 이벤트가 발생합니다.

    * 9억 7,500만 이벤트를 사용하고 있다면 보유 기간 **이 끝나지 않았습니다** .
    * 12억 5천만 개의 이벤트를 사용하고 있다면 보유 기간 **을** 초과한 것입니다.
  </Collapser>

    3. **[Insights.newrelic.com](https://insights.newrelic.com/) > 데이터 관리 > 요약** 으로 이동합니다.

    5. 적절한 스위치를 켜거나 끈 다음 저장하십시오.

       `Transaction` 을 켜거나 끄면 보고 에이전트가 스스로 다시 시작됩니다.

       이벤트 보고 구성에 대한 자세한 내용은 [이벤트 데이터 보존](/docs/insights/use-insights-ui/manage-account-data/extend-event-data-retention) 을 참조하십시오.

## 유연한 데이터 보존 [#flex-retention]

원래 가격 모델을 사용하는 조직의 경우 [Insights Pro](#insights-pro) 를 구독하면 일부 유형의 [이벤트 데이터](/docs/insights/use-insights-ui/manage-account-data/insights-data-retention) 를 보존하는 방법을 정의할 수 **있는 유연한 보존** 에 액세스할 수 있습니다. 이를 통해 필요한 이벤트 데이터만 필요한 기간 동안 유지할 수 있습니다. [UI](#ui-overview) 또는 [GraphQL API를](#api-examples) 통해 유연한 유지를 관리할 수 있습니다.

이 기능을 사용하기 위한 요구 사항:

* [Insights Pro 구독](https://newrelic.com/infrastructure/pricing) 또는 이에 상응하는 평가판.
* [Insights Pro 구독이 적용되는 이벤트](#insights-pro) 에만 적용됩니다.
* 이 기능을 사용하려면 계정 [소유자](/docs/accounts/accounts/roles-permissions/users-roles#roles) 또는 계정의 데이터 보존 [추가 기능 관리자](/docs/accounts/accounts/roles-permissions/users-roles#add-on) 여야 합니다.
