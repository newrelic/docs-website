<Callout variant="important">
  이 문서는 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 의 사용자가 있는 조직에 적용됩니다.
</Callout>

2020년 7월 30일부터 New Relic에 새로 가입하면 새로운 사용자 모델을 사용하게 됩니다.아직 [사용자를 새 모델로 마이그레이션](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 하지 않은 이전 고객은 여전히 원래 사용자 모델에 있습니다.더 많은 New Relic 고객이 새 모델로 마이그레이션됨에 따라 원래 사용자 모델은 점점 더 사용되지 않습니다.

다음은 원래 사용자 모델의 몇 가지 중요한 측면과 중요한 문서에 대한 링크입니다.

* 우리의 원래 사용자 모델은 조직/고객 사이에 확고한 경계가 없었고 훨씬 더 사용자 중심적이었습니다. 이는 사용자에게 다른 조직의 데이터에 대한 액세스 권한을 부여하고 한 번의 로그인으로 해당 데이터에 액세스할 수 있음을 의미했습니다. [이전 모델과 새 모델의 차이점에 대해 자세히 알아보세요](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) .
* 원래 사용자 모델에서 계정에 대한 사용자의 액세스는 a) 특정 계정에 추가되거나 b) 상위 계정에 대한 액세스 권한이 있고 해당 계정의 하위 계정에 대한 액세스 상속을 기반으로 합니다.[원래 사용자 관리 문서](/docs/accounts/original-accounts-billing/original-users-roles) 를 참조하십시오.[상위/하위 계정의 작동 방식에 대한 설명을](/docs/accounts/original-accounts-billing/original-users-roles/mastersub-account-structure) 참조하십시오.
