New Relic에는 두 가지 사용자 모델이 있습니다. **원래 사용자** 모델이라고 하는 이전 사용자 모델과 2020년 7월에 출시된 최신 사용자 모델입니다.

## 필요한 문서 찾기 [#docs]

현재 사용 중인 사용자 모델은 사용자와 역할 및 계정에 대한 액세스를 관리하는 방법에 영향을 줍니다( [사용자 모델의 차이점](#differences) 참조). 두 사용자 모델에 대해 서로 다른 문서 세트가 있습니다.

* [원래 사용자 모델용 문서](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model)
* [최신 사용자 모델용 문서](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-managing-users)

사용 중인 사용자 모델이 확실하지 않습니까? [사용자 모델 결정](#determine-user-model) 을 참조하십시오.

## 사용자 관리 UI [#manage-users]

사용자 관리 UI 사용 방법:

* 최신 사용자 모델을 사용하는 사용자의 경우: [사용자 관리 를](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#where) 참조하십시오.
* 원래 사용자 모델의 사용자: [one.newrelic.com](https://one.newrelic.com) ,[계정 드롭다운](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown) 을 클릭하고 **계정 설정** 을 클릭한 다음 **사용자 및 역할** 을 클릭합니다.자세한 내용은 [원래 사용자](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) 를 참조하십시오.

사용 중인 사용자 모델이 확실하지 않습니까? [사용자 모델 결정](#determine-user-model) 을 참조하십시오.

## 사용자 모델 결정 [#determine-user-model]

New Relic에 로그인하면 해당 로그인과 관련된 사용자 레코드가 한 사용자 모델에 있거나 다른 모델에 있습니다.사용 중인 사용자 모델을 확인하려면 [계정 드롭다운](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown) 에서 **사용자 기본 설정** 을 선택합니다.아래 UI가 표시되면 최신 사용자 모델을 사용하고 있는 것입니다.더 많은 콘텐츠가 표시된 다른 UI가 표시되면 원래 사용자 모델에 있는 것입니다( [원래 사용자 문서](/docs/accounts/original-accounts-billing/original-users-roles) 참조).

최신 사용자 모델을 사용하는 사용자를 관리할 수 있는지 확인하려면 사용자 [관리](/docs/accounts/accounts-billing/new-relic-one-pricing-users/add-manage-users-groups-roles#where) 를 참조하십시오.

사용자 모델은 가격 모델과 무관합니다. 사용자 모델이 가격과 어떤 관련이 있는지는 [가격 및 사용자 모델 표](#pricing-user-table) 를 참조하십시오.

## 사용자 모델 비교 [#differences]

New Relic에 로그인하면 로그인과 관련된 사용자 기록이 한 사용자 모델에 있거나 다른 모델에 있습니다. 다음은 사용자 모델 간의 차이점에 대한 설명입니다.

* **최신 사용자 모델** ([문서 참조](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-managing-users) ): 이 새롭고 [향상된](https://www.newrelic.com/blog/how-to-relic/user-management-and-access-security) 사용자 모델은 2020년 7월에 출시되었습니다.새로운 New Relic 가입을 위한 기본 사용자 모델입니다.다음은 원래 사용자 모델과의 주요 차이점입니다.

  * 모든 계정과 사용자는 최상위 [조직](/docs/new-relic-account-structure) 에 포함됩니다. 이를 통해 조직은 사용자와 사용자가 액세스할 수 있는 항목을 보다 강력하게 제어할 수 있습니다. 이것의 한 가지 영향은 여러 New Relic 조직에서 일하는 사용자가 [여러 로그인](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data/#account-access) 과 연결된 이메일 주소를 가질 수 있다는 것입니다.
  * 이 모델의 사용자는 권한 및 계정 액세스를 관리하기 위해 다른 시스템을 사용합니다. [액세스 권한 부여](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 를 사용하여 사용자에게 특정 계정의 특정 역할에 대한 액세스 권한을 할당합니다.

* **원래 사용자 모델** ( [문서 참조](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) ): 새 모델과 다른 이 이전 모델의 일부 측면:

  * 사용자에 대한 조직 수준의 제어가 많지 않았습니다. 예를 들어, New Relic 사용자는 단일 로그인을 사용하여 액세스 권한이 부여된 여러 계정에 액세스할 수 있었습니다.
  * 원래 사용자 모델의 사용자는 다른 [사용자 관리 시스템](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) 을 가집니다.

새 모델의 이점에 대해 자세히 알아보려면 사용자 모델 변경 사항 및 [새 계정 구조에 대한 설명에](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) [대한 블로그 게시물](https://www.newrelic.com/blog/how-to-relic/user-management-and-access-security) 을 참조하세요. 영향 및 제한 사항은 [기능 영향](#limitations) 을 참조하십시오.

## 최신 사용자 모델에 대한 요구 사항 [#requirements]

다음은 최신 모델을 사용하는 조직을 결정하는 요소입니다.

* 2020년 7월 30일 이후에 가입한 모든 New Relic 조직에는 이 모델의 사용자가 있습니다(또한 [최신 가격 모델](#pricing-plans) 도 있음).
* 일부 오래된 New Relic 조직에서는 New Relic 또는 [사용자 마이그레이션 절차](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 를 사용하여 사용자를 새 모델로 마이그레이션했습니다.
* 파트너 계정(리셀러, 관리 서비스 제공업체) 및 [파트너십 계정 구조](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions) 를 사용하는 조직은 아직 사용자를 새 모델로 마이그레이션할 수 없습니다.

## 사용자 모델의 기능 영향 [#limitations]

새로운 사용자 모델은 [많은 이점을 제공](https://newrelic.com/blog/how-to-relic/user-management-and-access-security) 합니다. 다음은 전환을 결정하기 전에 알아야 할 몇 가지 기능 영향입니다.

* 조직/계정의 더 많은 분리: 여러 New Relic 계정과 연결된 이메일 주소가 있는 경우 로그아웃했다가 다시 로그인해야 할 수 [있습니다. 계정 액세스에 대해 자세히 알아보세요.](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data/#account-access)
* 클래식 경고 알림(새 [알림](/docs/alerts-applied-intelligence/notifications/intro-notifications) 아님): 새 모델의 [사용자는 경고 알림을 받을 사용자를 선택할](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/#user) 때 옵션으로 표시되지 않습니다. 그 결과 이러한 사용자는 [New Relic 모바일 앱 푸시 알림](/docs/mobile-apps/new-relic-mobile-apps/authentication-alerts/alerting-new-relic-mobile-apps) 을 받지 못합니다. 해결 방법으로 [이메일 주소](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/#email) 를 알림 채널로 대신 사용할 수 있습니다.
* REST API를 사용하여 [사용자 목록을 생성](/docs/apis/rest-api-v2/account-examples-v2/listing-users-your-account/) 할 수 없습니다(원래 모델 사용자만). 대신 [사용자 관리 UI](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#where) 를 사용합니다.

## 사용자 모델과 가격 모델의 관계 [#relation-to-pricing]

사용자 모델은 두 가지 가격 책정 모델과 직접적인 관련이 없습니다. 이들 간의 관계에 대한 정보는 [가격 책정 모델 및 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models/#pricing-user-table) 을 참조하십시오.
