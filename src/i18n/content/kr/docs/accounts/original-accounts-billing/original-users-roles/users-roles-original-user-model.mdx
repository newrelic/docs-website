[원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 사용자: 이 모델에서 사용자 역할 및 권한이 작동하는 방식, 사용자를 추가하고 관리하는 방법.

## 요구 사항 [#who-is-doc-for]

문서의 이 섹션에서는 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에서 사용자를 관리하는 방법을 보여줍니다. [<DNT>**Users and roles**</DNT> UI](#view-access) 에서 사용자를 볼 수 있다면 해당 사용자는 원래 사용자 모델에 있는 것입니다.

New Relic 사용자가 가질 수 있는 세션과 IP 주소 수에는 제한이 있습니다. 보다 자세한 내용은 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data#session-limits)를 참조하십시오.

## 새로운 사용자 모델에 대한 업데이트 [#migrate]

2020년 7월, 뉴렐릭은 [조직과 사용자를 관리하는 측면에서 많은 이점](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)을 제공하는 [새로운 사용자 모델](https://newrelic.com/blog/how-to-relic/user-management-and-access-security)을 발표했습니다. 처음에는 신규 가입에 한해 새로운 사용자 모델을 제공했지만, 시간이 지나면서 이전 고객을 새 모델로 마이그레이션했습니다. 일부 기존 고객은 [사용자 마이그레이션을 자체적으로 처리할 수](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 있습니다. 기존 모델이 완전히 사용되지 않을 때까지, 사용자를 새 모델로 마이그레이션하는 작업이 계속될 것입니다.

새 사용자 모델을 사용하면 영향을 받는 것 중 하나는 사용자가 동일한 이메일과 연결된 여러 로그인을 가질 수 있다는 것입니다. 예를 들어, 여러 조직에 대한 액세스 권한이 있는 사용자(예: 계약업체)가 사용자 기록을 한 조직의 새 사용자 모델로 업데이트하면 기존 로그인 방법과 기록, 새로운 모델의 사용자 기록을 모두 보유하는 경우가 생길 수 있습니다. 이로 인해, 사용자가 뉴렐릭에 로그인되어 원하는 계정을 찾지 못하는 수가 있습니다. 이에 대한 자세한 내용은 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data/#account-access)를 참조하십시오.

## UI에서 사용자 보기 및 관리 [#view-access]

<DNT>**Users and roles**</DNT> UI [원래 사용자 모델](#who-is-doc-for) 에서 사용자를 관리하는 데 사용됩니다. 이 UI를 찾으려면 [사용자 메뉴를](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 클릭하고 <DNT>**Account settings**</DNT> 클릭한 다음 <DNT>**Users and roles**</DNT> 클릭합니다.

최신 사용량 기반 요금 모델을 사용 중인 경우 몇 가지 [중요한 고려 사항](#full-user-count)이 있습니다.

[New Relic REST API](/docs/apis/rest-api-v2/account-examples-v2/listing-users-your-account) 를 사용하여 계정의 사용자 목록을 가져올 수도 있습니다.

다음은 몇 가지 일반적인 사용자 관리 작업 및 시나리오에 대한 지침 및 고려 사항입니다.

  <Collapser
    id="adding_users"
    title="새 사용자 추가"
  >
    <Callout variant="tip">
      <DNT>
        **Owner or Admins**

    여러 계정이 있는 조직에 [원래 사용자 모델](#who-is-doc-for) 의 사용자를 추가하는 경우 [계정 구조가 사용자가 액세스할 수 있는 항목에 영향을 미치](/docs/accounts/original-accounts-billing/original-users-roles/parent-child-account-structure) 므로 사용자를 추가할 계정에 대해 생각하는 것이 좋습니다.

    New Relic 계정에 새 사용자를 추가하려면:

    1. 사용자를 추가하려는 계정의 경우 다음으로 이동하세요. <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles > Users**</DNT>.

    2. 오른쪽 상단에서 <DNT>**<Icon name="fe-plus-circle"/> New user**</DNT> 클릭합니다.

    3. 적절한 이름과 이메일 주소를 입력합니다.

    4. [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)을 선택합니다. [사용량 기반 요금 모델을 사용](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)하는 경우 사용자 유형은 청구 요소입니다.

    5. 기본 역할을 관리자, 사용자 또는 제한됨으로 선택합니다.

    6. <DNT>**Add user**</DNT> 선택합니다.

       새 사용자는 New Relic에서 자동으로 이메일 알림을 받게 됩니다.

       <Callout variant="important">
         New Relic은 사용자당 최대 1,000개의 계정을 권장합니다. 추가 계정으로 인해 일부 New Relic 기능에 대한 액세스가 제한될 수 있습니다.
       </Callout>
  </Collapser>

  <Collapser
    id="full-user-count"
    title="청구 가능한 사용자 관리"
  >
    [사용량 기반 요금 모델](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/)을 사용하는 경우 사용자의 사용자 유형이 청구 요소입니다.

    조직에 여러 계정이 있는 경우 원래 사용자 모델에서 청구 가능한 사용자를 관리할 때 다음과 같은 몇 가지 중요한 고려 사항이 있습니다.

    * 단일 UI에서 조직의 모든 사용자를 볼 수는 없습니다. 모든 사용자를 검토하고 수정하는 것이 목표인 경우(예: 청구 목적으로 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 수정) 각 개별 계정의 <DNT>**Users and roles**</DNT> UI 로 이동해야 합니다. (사용자를 보다 쉽게 관리할 수 있는 능력은 [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 의 이점 중 하나입니다.)

    * 위의 사항으로 인해 <DNT>**View your usage**</DNT> UI에 표시되는 청구 가능 사용자 수가 특정 계정의 <DNT>**Users and roles**</DNT> UI로 이동할 때 표시되는 사용자 수와 일치하지 않을 수 있습니다.

    * 해당 계정이 <DNT>**Account settings**</DNT> UI에서 '활성' 또는 '비활성'(보고 데이터 아님)으로 표시되어 있는지 여부에 관계없이 조직 내 모든 계정의 청구 가능 사용자에게 비용이 청구될 수 있습니다.

      사용자 유형 편집에 대한 팁은 사용자 유형 [관리](#update-user-type) 를 참조하십시오.
  </Collapser>

  <Collapser
    id="update-user-type"
    title="사용자 유형 및 사용자 업그레이드 관리"
  >
    사용자 유형 관리를 위한 몇 가지 중요한 팁:

    * [사용량 기반 요금 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models)을 사용하는 경우 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)이 청구 요소입니다.

    * 사용자를 상위 사용자 유형으로 업그레이드하면(UI 또는 업그레이드 요청 이메일을 통해) 조직의 모든 계정에서 해당 사용자 유형이 변경됩니다.

      사용자의 사용자 유형을 업데이트하려면:

    1. 이동: <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles > Users**</DNT>.

    2. 사용자를 선택하고 유형을 편집하거나 [여러 사용자의 유형을 일괄 업데이트합니다](#bulk). 조직에 여러 계정이 있는 경우 단일 UI 위치에서 모든 사용자를 볼 수는 없습니다. 여러 계정의 사용자를 관리하려면 각 계정의 <DNT>**Users and roles**</DNT> UI로 이동해야 합니다.

       업그레이드를 요청한 사용자를 보고 승인하려면:

    3. 이동: <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles**</DNT>.

    4. <DNT>**Upgrade requests**</DNT> 클릭합니다. 조직에 여러 계정이 있는 경우 단일 UI 위치에서 모든 사용자를 볼 수 없습니다. 여러 계정의 사용자를 관리하려면 각 계정에 대해 <DNT>**Users and roles**</DNT> UI로 이동해야 합니다.

       사용자 업그레이드 방법을 제어하려면 <DNT>**Users and roles**</DNT> UI에서 <DNT>**Access requests**</DNT> 선택하면 됩니다. 두 가지 옵션이 있습니다:

    * <DNT>**Automatic approval**</DNT>: 이 옵션을 사용하면 사용자가 자신의 사용자 유형을 자동으로 즉시 업그레이드할 수 있습니다. 이 옵션을 사용하면 사용자가 문제를 더 쉽게 해결할 수 있습니다.

    * <DNT>**Require review**</DNT>: 이 옵션을 사용하면 사용자가 업그레이드를 요청할 때 관리자가 공지 이메일을 받게 되며 관리자는 이를 먼저 업그레이드해야 합니다. 공지 이메일이나 <DNT>**Users and roles**</DNT> UI 의 사용자 기록을 통해 승인할 수 있습니다.

      사용자 유형에 대한 자세한 내용은 [사용자](#user-type) 유형을 참조하십시오.
  </Collapser>

    조직에서 [SAML Single Sign On 을](/docs/accounts-partnerships/accounts/saml-single-sign/saml-service-providers) 구현한 경우 <DNT>**Pending users**</DNT> 목록이 있을 수 있습니다. [SAML 활성화 계정에 추가](/docs/accounts-partnerships/accounts/saml-single-sign/adding-users-saml-accounts) 되었지만 아직 확인되지 않은 사용자입니다.

    [새로운 사용량 기반 요금 모델](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)을 사용하며 기존 사용자 모델을 사용하는 사용자가 있는 조직의 경우, `Pending invite` 사용자는 요금 청구 대상이 아닙니다. (새로운 사용자 모델을 사용하는 경우 `Pending invite` 사용자는 청구 가능합니다.)
  </Collapser>

    1. 이동: <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles > Users**</DNT>.

    2. 사람의 이름을 선택합니다.

    3. <DNT>**Roles and capabilities**</DNT> 아래에서 기본 역할을 관리 사용자 또는 제한됨으로 선택합니다.

       계정 소유자는 [소유자 역할을 업데이트](#owner) 해야 합니다.

    New Relic 계정에서 사용자를 제거하려면:

    1. 이동: <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles > Users**</DNT>.

    2. 업데이트하려는 사람의 이름을 클릭합니다.

    3. <DNT>**<Icon name="fe-trash-2"/> Delete User**</DNT> 을(를) 클릭합니다.

       <Callout variant="tip">
         [CSV 파일을 통해](#bulk) 대량으로 사용자를 추가, 업데이트 또는 삭제할 수도 있습니다.

  <Collapser
    id="owner"
    title="계정 소유자 업데이트"
  >
    소유자 역할 개념은 원래 사용자 모델의 사용자에게만 존재합니다.

    New Relic 계정은 한 번에 하나의 소유자 [역할](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) 만 가질 수 있습니다. 현재 계정에 대한 관리자 역할이 있는 사람으로 역할을 변경하려면 현재 계정 소유자여야 합니다. [현재 소유자를 사용할 수 없는](#no-owner) 경우 New Relic에서 계정 담당자에게 문의하거나 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으십시오.

    할당된 소유자 역할을 삭제하거나 제거할 수 없습니다. 그러나 계정에 하나 이상의 관리자 역할이 있는 경우 소유자를 관리자로 변경할 수 있습니다.

    1. 이동: <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Users and roles**</DNT>.

    2. 활성 사용자 목록 위에서 <Icon name="fe-repeat"/> <DNT>**Change owner**</DNT>. 계정에 관리자가 없으면 이 버튼을 사용할 수 없습니다.

    3. 현재 계정에 대한 관리자 역할이 있는 사람을 선택하십시오.

    4. 변경 사항을 적용하려면 페이지를 새로고침하세요.

       이전 소유자 역할은 자동으로 관리자 역할로 변경됩니다.

       현재 할당된 소유자가 누구인지 확인하려면:

    5. 이동: <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Users and roles**</DNT>.

    6. 계정 소유자를 찾으려면 <DNT>**Base role**</DNT> 열을 확인하세요.

       그만큼 <Icon name="fe-repeat"/> <DNT>**Change owner**</DNT> 버튼은 현재 계정 소유자에게만 표시됩니다. 현재 소유자가 역할을 변경할 수 없는 경우(예: 해당 사람이 더 이상 조직에 속하지 않는 경우) 뉴렐릭의 계정 담당자에게 문의하거나 [support.newrelic.com](https://support.newrelic.com) 에서 지원을 받으세요.
  </Collapser>
</CollapserGroup>

## 사용자 유형 [#user-type]

[사용량 기반 요금 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models)을 사용하는 조직의 경우 사용자 유형이 청구 요소입니다.

사용자의 사용자 유형에 따라 뉴렐릭 기능에 최대한 허용되는 액세스가 결정됩니다. 실제로 사용자에게는 다양한 방식으로 권한을 제한하는 [역할](#roles)이 할당되는 경우가 많지만, 사용자 유형은 이론적인 최대 액세스 권한을 나타냅니다. 자세한 내용은 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)을 참조하십시오.

조직의 사용자가 다른 계정에서 다른 사용자 유형으로 설정된 경우 해당 사용자는 가장 높은 사용자 유형이 무엇이든 상관없이 간주됩니다.

사용자 유형을 편집하는 방법은 [사용자 유형 관리](#update-user-type) 를 참조하십시오.

## 추가 역할 [#add-on]

추가 기능 역할을 사용하면 사용자에게 보다 구체적이고 세부적인 액세스 권한을 부여할 수 있습니다. 사용자 또는 제한된 사용자 추가 기능 관리자 액세스 권한을 부여하면 해당 기능 카테고리에 대해 <DNT>**Admin**</DNT> 권한과 동등한 권한이 부여됩니다. 다른 모든 뉴렐릭 제품에 대한 사용자 또는 키보드 사용자 권한은 계속 보유하게 됩니다. 예를 들어, 회사의 소프트웨어 엔지니어를 사용자로 만들고 그들에게 <DNT>**APM manager**</DNT> 역할을 할당하여 <InlinePopover type="apm"/>권한을 부여할 수 있습니다.

## 대량 사용자 관리 [#bulk]

<DNT>**Bulk user actions**</DNT> 기능을 사용하면 여러 사용자를 한 번에 추가, 업데이트 또는 삭제할 수 있습니다. 이는 다음과 같은 경우에 도움이 될 수 있습니다.

* 여러 명의 새 직원이 시작될 때 역할 추가
* 여러 직원이 떠날 때 역할 삭제
* 여러 직원에게 관리자 역할 부여

### 사용자 일괄 업데이트 [#update]

대량 사용자 작업을 수행하기 위한 몇 가지 중요한 규칙 및 권장 사항:

* 자신의 역할이나 소유자 역할은 업데이트할 수 없습니다.
* 기존 사용자의 이메일 주소나 이름은 수정할 수 없습니다.
* 기존 사용자를 삭제했다가 다시 추가하는 방식으로 편집하는 것은 피해야 합니다. 이는 의도하지 않은 결과를 초래할 수 있기 때문입니다(예: 원래 사용자와 연결된 API 키가 손실됨).

새 사용자 역할을 추가하려면 기존 사용자 역할을 업데이트하거나 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 의 사용자에 대한 사용자 역할을 삭제하십시오.

1. <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles**</DNT> 으로 이동하여 URL 끝에 `/bulk_actions` 추가하세요.

   예시 URL:

   ```
   https://account.newrelic.com/accounts/123456789/users/bulk_actions
   ```

2. <DNT>**Backup CSV file**</DNT> 을(를) 다운로드합니다. 백업 파일을 다운로드하면 변경이 이루어지기 전에 계정의 사용자 기록이 유지되며 실수로 제거될 수 있는 사용자를 쉽게 다시 추가할 수 있습니다.

3. <DNT>**CSV of users**</DNT> 또는 <DNT>**CSV template**</DNT> 을 다운로드합니다. 각 대량 작업(추가, 업데이트 또는 삭제)에는 자체 CSV 파일이 필요합니다. 뉴렐릭은 계좌 번호, 날짜, 수행 중인 일괄 작업과 함께 파일을 저장할 것을 권장합니다. 예를 들어: `account_123456789_delete_users_2018-06-29`

4. 선택한 일괄 작업을 적용할 역할의 사용자로만 시트를 채웁니다. 역할을 변경하지 않으려는 사용자를 스프레드시트에서 제거합니다.
