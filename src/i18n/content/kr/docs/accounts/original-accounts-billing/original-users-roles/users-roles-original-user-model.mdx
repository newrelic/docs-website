[기존 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models)의 사용자: 이 모델에서 사용자 역할 및 권한이 작동하는 방식과 사용자를 추가하고 관리하는 방법.

## 요구 사항 [#who-is-doc-for]

문서의 이 섹션에서는 [기존 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models)에서 사용자를 관리하는 방법을 보여줍니다. [**사용자 및 역할** UI](#view-access)에서 사용자를 볼 수 있는 경우, 해당 사용자는 기존 사용자 모델에 있는 것입니다.

뉴렐릭 사용자가 가질 수 있는 세션과 IP 주소 수에는 제한이 있습니다. 보다 자세한 내용은 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data#session-limits)를 참조하십시오.

## 새로운 사용자 모델에 대한 업데이트 [#migrate]

2020년 7월, 뉴렐릭은 [조직과 사용자를 관리하는 측면에서 많은 이점](https://newrelic.com/blog/how-to-relic/user-management-and-access-security)을 제공하는 [새로운 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)을 발표했습니다. 처음에는 신규 가입에 한해 새로운 사용자 모델을 제공했지만, 시간이 지나면서 이전 고객을 새 모델로 마이그레이션했습니다. 일부 기존 고객은 [사용자 마이그레이션을 자체적으로 처리할 수](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) 있습니다. 기존 모델이 완전히 사용되지 않을 때까지, 사용자를 새 모델로 마이그레이션하는 작업이 계속될 것입니다.

새 사용자 모델을 사용하면 영향을 받는 것 중 하나는 사용자가 동일한 이메일과 연결된 여러 로그인을 가질 수 있다는 것입니다. 예를 들어, 여러 조직에 대한 액세스 권한이 있는 사용자(예: 계약업체)가 사용자 기록을 한 조직의 새 사용자 모델로 업데이트하면 기존 로그인 방법과 기록, 새로운 모델의 사용자 기록을 모두 보유하는 경우가 생길 수 있습니다. 이로 인해, 사용자가 뉴렐릭에 로그인되어 원하는 계정을 찾지 못하는 수가 있습니다. 이에 대한 자세한 내용은 [액세스에 영향을 미치는 요소](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data/#account-access)를 참조하십시오.

## UI에서 사용자 보기 및 관리 [#view-access]

**사용자 및 역할** UI는 [기존 사용자 모델](#who-is-doc-for)에서 사용자를 관리하는 데 사용됩니다. 이 UI를 찾으려면 [user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), **Account settings** , **Users and roles**를 차례로 클릭합니다.

최신 사용량 기반 요금 모델을 사용 중인 경우 몇 가지 [중요한 고려 사항](#full-user-count)이 있습니다.

[뉴렐릭 REST API](/docs/apis/rest-api-v2/account-examples-v2/listing-users-your-account)를 사용하여 계정의 사용자 목록을 가져올 수도 있습니다.

다음은 몇 가지 일반적인 사용자 관리 작업 및 시나리오에 대한 지침 및 고려 사항입니다.

    2. 오른쪽 상단에서**<Icon name="fe-plus-circle"/>New user**를 클릭합니다.

    3. 적조항한 이름과 이메일 주소를 입력합니다.

    4. [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)을 선택합니다. [사용량 기반 요금 모델을 사용](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)하는 경우 사용자 유형은 청구 요소입니다.

    5. 기본 역할로 Admin, User, 또는 Restricted 중 하나를 선택합니다.

    6. **Add user**를 선택합니다.

       새 사용자는 뉴렐릭에서 자동으로 이메일 알림을 받게 됩니다.

       <Callout variant="important">
         뉴렐릭은 사용자당 최대 1,000개의 계정을 권장합니다. 추가 계정으로 인해 일부 뉴렐릭 기능에 대한 액세스가 제한될 수 있습니다.
       </Callout>
  </Collapser>

  <Collapser
    id="full-user-count"
    title="청구 가능한 사용자 관리"
  >
    [사용량 기반 요금 모델](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/)을 사용하는 경우 사용자의 사용자 유형이 청구 요소입니다.

    조직에 여러 계정이 있는 경우, 원래 사용자 모델에서 청구 가능한 사용자를 관리할 때 다음과 같은 몇 가지 중요한 고려 사항이 있습니다.

    * 단일 UI에서 조직의 모든 사용자를 볼 수는 없습니다. 모든 사용자를 검토하고 편집하는 것이 목표인 경우(예: 청구 목적으로 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) 수정), 각 개별 계정의 **사용자 및 역할** UI로 이동해야 합니다. (사용자를 보다 쉽게 관리할 수 있는 기능은 [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models)의 이점 중 하나입니다.)

    * 위의 이유로, **사용량 보기** UI에 표시되는 청구 가능한 사용자 수가 특정 계정의 **사용자 및 역할** UI로 이동할 때 표시되는 사용자 수와 일치하지 않을 수 있습니다.

    * **계정 설정** UI에서 해당 계정이 "활성" 또는 "비활성"(보고 데이터 아님)으로 표시되는지 여부에 관계없이, 조직의 모든 계정에서 청구 가능한 사용자에 대해 비용이 청구될 수 있습니다.

      사용자 유형 수정에 대한 팁은 사용자 유형 [관리](#update-user-type)를 참조하십시오.
  </Collapser>

  <Collapser
    id="update-user-type"
    title="사용자 유형 및 사용자 업그레이드 관리"
  >
    사용자 유형 관리를 위한 몇 가지 중요한 팁:

    * [사용량 기반 요금 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models)을 사용하는 경우 사용자의 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)이 청구 요소입니다.

    * 사용자를 상위 사용자 유형으로 업그레이드하면(UI 또는 업그레이드 요청 이메일을 통해) 조직의 모든 계정에서 해당 사용자 유형이 변경됩니다.

      사용자의 사용자 유형을 업데이트하려면:

    1. **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles > Users**로 이동합니다.

    2. 사용자를 선택하고 유형을 수정하거나 [여러 사용자의 유형을 일괄 업데이트](#bulk)합니다. 조직에 여러 계정이 있는 경우, 단일 UI 위치에서 모든 사용자를 볼 수는 없습니다. 다른 계정의 사용자를 관리하려면 각 계정의 **사용자 및 역할** UI로 이동해야 합니다.

       업그레이드를 요청한 사용자를 확인하고 승인하려면:

    3. **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles**로 이동합니다.

    4. **Upgrade requests**를 클릭합니다. 조직에 여러 계정이 있는 경우 단일 UI 위치에서 모든 사용자를 볼 수는 없습니다. 다른 계정의 사용자를 관리하려면 각 계정의 **사용자 및 역할** UI로 이동해야 합니다.

       사용자 업그레이드 방법을 제어하려면 **사용자 및 역할** UI에서 **Access requests**를 선택할 수 있습니다. 두 가지 옵션이 있습니다.

    * **자동 승인**: 이 옵션을 사용하면 사용자 유형을 자동으로 즉시 업그레이드할 수 있습니다. 이 옵션을 사용하면 사용자가 문제를 보다 쉽게 해결할 수 있습니다.

    * **검토 필요**: 이 옵션을 사용하면 사용자가 업그레이드를 요청하고 관리자가 먼저 업그레이드해야 할 때 관리자에게 알림 이메일이 전송됩니다. 알림 이메일이나 **사용자 및 역할** UI의 사용자 기록에서 승인할 수 있습니다.

      사용자 유형에 대한 자세한 내용은 [사용자 유형](#user-type)을 참조하십시오.
  </Collapser>

    소유자 역할 개념은 기존 사용자 모델의 사용자에게만 존재합니다.

    뉴렐릭 계정에는 한 번에 하나의 소유자 [역할](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) 만 있을 수 있습니다. 현재 계정에 대한 관리자 역할로 역할을 변경하려면 현재 계정 소유자여야 합니다. [현재 소유자가 변경을 할 수 없는](#no-owner) 경우, 뉴렐릭의 계정 담당자에게 문의하거나 [support.newrelic.com](https://support.newrelic.com)에서 지원을 요청하시기 바랍니다.

    할당된 소유자 역할을 삭제하거나 제거할 수 없습니다. 그러나 계정에 하나 이상의 관리자 역할이 있는 경우, 소유자를 관리자로 변경할 수 있습니다.

    1. **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Users and roles**로 이동합니다.

    2. Active users 목록 위에서<Icon name="fe-repeat"/>**Change owner**를 선택합니다. 계정에 Admin이 없으면 이 버튼을 사용할 수 없습니다.

    3. 현재 계정에 대한 관리자 역할을 가진 사용자를 선택합니다.

    4. 변경 사항을 적용하려면 페이지를 새로고침합니다.

       이전 소유자 역할이 자동으로 관리자 역할로 변경됩니다.

       현재 할당된 소유자가 누구인지 확인하려면:

    5. **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Users and roles**로 이동합니다.

    6. **기본 역할** 열을 보고 계정 소유자를 찾습니다.

       <Icon name="fe-repeat"/>**Change owner** 버튼은 현재 계정 소유자에게만 표시됩니다. 현재 소유자가 역할을 변경할 수 없는 경우(예: 해당 사람이 더 이상 조직에 없는 경우), 뉴렐릭의 계정 담당자에게 문의하거나 [support.newrelic.com](https://support.newrelic.com)에서 지원을 요청하십시오.
  </Collapser>

## 사용자 유형 [#user-type]

[사용량 기반 요금 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models)을 사용하는 조직의 경우 사용자 유형이 청구 요소입니다.

사용자의 사용자 유형에 기반해 최대 허용되는 뉴렐릭 기능 세트가 결정됩니다. 실제로 사용자에게는 다양한 방식으로 기능을 제한하는 [역할](#roles)이 할당되는 경우가 많지만, 사용자 유형은 이론적으로 사용 가능한 기능의 최대 세트를 나타냅니다. 자세한 내용은 [사용자 유형](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type)을 참조하십시오.

조직의 사용자가 다른 계정에서는 다른 사용자 유형으로 설정된 경우, 해당 사용자는 가장 높은 사용자 유형으로 간주됩니다.

사용자 유형을 수정하는 방법은 [사용자 유형 관리](#update-user-type)를 참조하십시오.

## 애드온 역할 [#add-on]

애드온(Add-on) 역할을 사용하면 사용자들에게 보다 구체적이고 세분화된 액세스 권한을 부여할 수 있습니다. 사용자(User) 또는 제한된 사용자(Restricted User) 애드온 관리자 액세스 권한을 부여하면, 해당 기능 카테고리에 대한 **관리** 기능과 동등한 권한이 부여됩니다. 이 사용자들은 다른 모든 뉴렐릭 제품에 대해 사용자 또는 제한된 사용자 기능을 계속 유지하게 됩니다. 예를 들어 회사의 소프트웨어 엔지니어를 사용자로 지정하고 **APM 관리자** 역할을 할당하면 <InlinePopover type="apm"/>기능을 부여할 수 있습니다.

    관리되는 애드온 역할은 기본적으로 각 뉴렐릭 제품에 사용할 수 있습니다. 사용자의 관리 역할을 추가[하면 할당된 제품에 대한 Admin 수준 권한이 부여됩니다](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles). 이 역할은 수정하거나 삭제할 수 없습니다. 계정의 사용자 또는 제한된 사용자에 대해 관리되는 애드온 역할을 할당하려면:

## 일괄 사용자 관리 [#bulk]

**일괄 사용자 작업** 기능을 사용하면, 한 번에 여러 사용자를 추가, 업데이트 또는 삭제할 수 있습니다. 다음과 같은 경우에 도움이 될 수 있습니다.

* 여러 명의 새 직원이 근무를 시작할 때 역할 추가
* 여러 직원이 퇴직한 경우 역할 삭제
* 여러 직원에게 관리자 역할 부여

### 사용자 일괄 업데이트 [#update]

일괄 사용자 작업을 수행하기 위한 몇 가지 중요한 규칙 및 권장 사항:

* 자신의 역할이나 소유자 역할은 업데이트할 수 없습니다.
* 기존 사용자의 이메일 주소나 이름은 수정할 수 없습니다.
* 기존 사용자를 삭제했다가 다시 추가하는 방식으로 수정하는 것은 피해야 합니다. 이는 의도하지 않은 결과를 초래할 수 있기 때문입니다. (예: 원래 사용자와 연결된 API 키 손실)

새 사용자 역할을 추가하려면 기존 사용자 역할을 업데이트하거나 [기존 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)의 사용자에 대한 사용자 역할을 삭제하십시오.

1. **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Users and roles**로 이동한 후 URL의 마지막 부분에 `/bulk_actions`를 추가합니다.

   예시 URL:

   ```
   https://account.newrelic.com/accounts/123456789/users/bulk_actions
   ```

2. **백업 CSV 파일**을 다운로드합니다. 백업 파일을 다운로드하면 변경 사항이 적용되기 전에 계정의 사용자 기록이 유지되고 실수로 제거될 수 있는 사용자를 쉽게 다시 추가할 수 있습니다.

3. **사용자**의 CSV 또는 **CSV 템플릿**을 다운로드합니다. 각 일괄 작업(추가, 업데이트 또는 삭제)에는 자체 CSV 파일이 필요합니다. 계정 번호, 날짜 및 수행 중인 일괄 작업과 함께 파일을 저장할 것을 권장합니다. 예를 들어: `account_123456789_delete_users_2018-06-29`

4. 선택한 일괄 작업을 적용할 역할의 사용자로만 시트를 채웁니다. 역할을 변경하지 않으려는 사용자를 스프레드시트에서 제거합니다.

</Callout>

    1. 백업 CSV 파일을 엽니다.
    2. 역할이 수정될 사용자로 백업 CSV 파일을 채웁니다.
    3. 새 CSV 파일에 대한 CSV 작업을 선택합니다. **추가**, **업데이트** 또는 **삭제**할 수 있습니다.
    4. 새 CSV를 업로드하고 **변경 사항 저장**을 선택합니다.
