New Relic 조직은 부모/자식 계정 구조를 가질 수 있습니다. 이는 기본적으로 사용자가 [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) 인 조직과 관련이 있지만 두 사용자 모델의 사용자가 있는 조직에도 적용될 수 있습니다. 상위/하위 구조가 적용되는 주요 차이점은 다음과 같습니다.

* 원래 사용자 모델: 원래 사용자 모델은 부모/자식 계정 구조를 사용합니다. 여기서 하위 계정은 계정의 사용자 액세스와 관련된 규칙을 포함하여 상위 계정의 설정을 상속합니다. 우리의 [파트너십 계정](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions) 도 이 모델을 사용합니다. 조직에 이 모델의 사용자가 있는 경우 해당 사용자는 [자녀 계정을 만들](#creating) 수 있습니다. 이 모델에 사용자가 있는 경우 [아래를 계속 읽고](#hierarchy) 이 계정 구조가 어떻게 작동하는지 알아보세요.
* 새로운 사용자 모델: 새롭고 향상된 사용자 모델.이 모델의 사용자가 있는 조직은 상위/하위 계정 구조를 가질 수 있지만 더 이상 계정에 대한 사용자 액세스를 관리하는 것과 관련이 없으며 주로 일부 기능에 대한 계정 관계를 결정하고 조직 수준 청구를 집계하는 데 사용됩니다.새로운 사용자 모델에서 계정 액세스가 작동하는 방식에 대한 자세한 내용은 [계정 구조](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) 를 참조하세요.

## 상위 및 하위 계정 계층 [#hierarchy]

[원래 사용자 모델](#requirements) 의 사용자가 있는 조직의 경우 해당 계정은 상위/하위 계정 구조를 가질 수 있으며 각 하위 계정은 단일 상위 계정에 연결됩니다. 각 계정에는 보고하는 여러 앱이 있을 수 있습니다. 상위 계정에도 보고하는 애플리케이션이 있을 수 있지만 하위 계정 앱에서 보낸 모든 데이터가 상위 계정으로 롤업되는 것은 아닙니다.

UI에서 계정 간에 전환하는 방법을 알아보려면 [계정 액세스](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data#account-access) 를 참조하세요.

상위 및 하위 계정이 EU 지역 계정과 함께 확장되는 방식을 이해하려면 [EU 지역 계정 계층 구조](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#global-account-hierarchy) 를 참조하십시오.

## 하위 계정용 라이선스 키 [#keys]

New Relic 조직에서 기본 New Relic 계정은 자체 [라이선스 키](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key) 가 있는 상위 계정 역할을 합니다. 각 하위 계정에는 고유한 개별 라이선스 키가 있습니다. 각 애플리케이션은 하나의 라이센스 키만 사용할 수 있으므로 각 애플리케이션은 하나의 계정에만 보고할 수 있습니다.

## 자녀 계정 만들기 [#creating]

New Relic 조직에 계정을 추가하려면 Pro 또는 Enterprise [버전이](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions)있어야 합니다.

계정을 추가하는 방법은 사용자가 속한 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

* 최신 사용자 모델의 사용자: [계정 구조](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure/#new-model) 를 참조하십시오.

* 원래 사용자 모델의 사용자: 다음 지침을 따르십시오.

  1. 이동: **[사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Summary** .
  2. **계정 추가** 를 클릭합니다.
  3. 자녀 계정 이름을 입력한 다음 **이 계정 추가** 를 선택합니다.
  4. 하위 계정에 사용자를 추가하는 절차를 계속 진행합니다.

## 계정에 사용자 추가 [#users]

둘 이상의 계정이 있는 New Relic 조직을 보유하려면 Pro 또는 Enterprise [에디션이](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions)있어야 합니다.

조직의 하위 계정에 사용자를 추가하는 방법은 사용자가 속한 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

* 최신 사용자 모델: 특정 계정에 대한 액세스 권한이 있는 그룹에 사용자를 할당해야 합니다. 자세한 내용은 [사용자 관리 개념](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts) 을 참조하세요.

* 원래 사용자 모델: 일반적으로 상위 계정의 사용자에 대한 권한은 하위 계정의 해당 사용자에 대해 자동으로 상속됩니다. 사용자를 추가하려면:

  1. 이동: **[사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Summary** .
  2. 하위 계정 목록에서 하위 계정을 선택합니다.
  3. 하위 계정의 **활성 사용자** 섹션에서 **사용자 추가** 를 선택하고 사용자의 이메일, 이름, 역할 및 직위를 적절하게 입력한 다음 **이 사용자 추가** 를 선택합니다.
  4. 선택 사항: 하위 계정 [의 소유자 또는 관리자 역할을 적절하게 변경합니다](/docs/accounts-partnerships/accounts/account-setup/adding-updating-users) .

## SAML SSO로 계정 설정 [#saml]

SAML SSO를 설정하는 방법은 사용자가 속한 [사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 에 따라 다릅니다.

* 원래 사용자 모델: 상위 계정 수준에서 [SAML SSO를 설정](/docs/accounts-partnerships/accounts/saml-single-sign/setting-sso) 하면 하위 계정 사용자도 SSO를 사용하여 로그인할 수 있습니다. 상위 수준에서 SAML SSO를 설정 **하지 않은** 경우 하위 계정 수준에서 설정할 수 있습니다. 그러나 두 수준 모두에서 SAML SSO를 설정 **하지 마십시오** . 자세한 내용 [은 여러 계정으로 SAML 구성](/docs/accounts-partnerships/accounts/saml-single-sign/configuring-saml-multiple-accounts) 을 참조하십시오.
* 최신 사용자 모델: [SAML SSO](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) 를 참조하십시오.

## 자녀 계정 애플리케이션 업데이트 [#subaccount_apps]

**요구 사항:**

* 프로 또는 엔터프라이즈 [에디션](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).
* [원래 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 을 사용 중이어야 합니다( [최신 사용자 모델](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) 을 사용 중이고 계정을 추가하려면 계정 담당자에게 문의하세요).
* 소유자 또는 관리자여야 합니다.

앱의 구성 파일을 변경하여 한 계정에서 다른 계정으로 이동하면 New Relic은 이를 새 하위 계정의 새 앱으로 취급합니다. New Relic은 앱의 과거 데이터를 이동하지 않습니다.

자녀 계정용 새 앱을 설치하려면 다음 단계를 따르세요.

1. 상위 계정을 선택했는지 확인하십시오. [계정 전환기](/docs/new-relic-solutions/get-started/glossary/#account-switcher) 에서 원하는 계정을 선택하십시오.
2. [사용자 메뉴](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) 에서 **계정 설정** 을 선택합니다.
3. 새 응용 프로그램을 설치하려면 표준 절차를 따르십시오.
