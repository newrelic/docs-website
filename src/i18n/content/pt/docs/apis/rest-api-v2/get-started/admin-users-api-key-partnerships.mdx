Essa abordagem adiciona um novo método de autenticação à [API REST New Relic (v2)](https://api.newrelic.com/) voltada para os clientes que permite que parceiros (e clientes em escala empresarial que usam a estrutura de parceria para gerenciar suas múltiplas contas) usem suas credenciais de parceria para autorizar todas as API REST ( v2) chamadas. Ele não oferece novas funcionalidades para usuários da API não parceiros.

Ao chamar o endpoint na API REST da New Relic (v2) que exige <InlinePopover type="userKey"/>, a New Relic sugere que você use o [método de autenticação](/docs/apis/rest-api-v2/requirements/admins-api-key-partnerships#Partners) descrito .

## Cabeçalhos personalizados e chamada de API [#Partners]

Ao fazer uma chamada de API, use os seguintes cabeçalhos personalizados com sua [chave de API de parceiro](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference#the-api-key), [ID de parceiro](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference#partner-id) e [ID de conta](/docs/accounts-partnerships/accounts/account-setup/account-id):

* `X-Api-Key:YOUR_PARTNER_ACCOUNT_API_KEY`
* `NewRelic-Partner-Id:YOUR_PARTNER_ID`
* `NewRelic-Account-Id:YOUR_CUSTOMERS_ACCOUNT_ID`

Fornecer autenticação à API REST (v2) neste formulário permite concluir qualquer operação de API nas contas dos seus clientes. Isso ignora o requisito do cliente de que algumas chamadas de API exigem uma chave de API do usuário administrador.

Este método de autenticação também funciona para endpoints de API que requerem apenas a chave de API REST. Este novo método de autenticação funciona para todos os endpoints, não apenas aqueles que requerem a chave de API do usuário administrador.

## Limitações conhecidas

Este método de autenticação somente para parceiros funcionará apenas com a [API REST da New Relic (v2)](https://api.newrelic.com/). Ele <DNT>**does not**</DNT> funciona com o seguinte:

* [API de implantação](/docs/apm/new-relic-apm/maintenance/deployment-notifications#api)
* [API de infraestrutura para alerta](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/rest-api-calls-new-relic-infrastructure-alerts)
* [API de parceiro](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference)
* [API de monitoramento sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-rest-api)
