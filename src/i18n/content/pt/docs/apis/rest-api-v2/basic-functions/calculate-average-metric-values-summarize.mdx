Por padrão, a chamada de API REST do New Relic retorna uma série de valores de dados métricos com base em [restrições de intervalo de tempo](/docs/apm/apis/requirements/extracting-metric-data#time). Para obter a média desses valores, inclua `&summarize=true` na sua chamada de API. Por exemplo:

```
curl -X GET "https://api.newrelic.com/v2/applications/${APP_ID}/metrics/data.json" \
     -H "Api-Key:<a href="/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key">$API_KEY</a>" -i \
     -d 'names[]=Agent/MetricsReported/count&from=2014-08-11T14:42:00%2B08:00&to=2014-08-11T15:12:00%2B08:00&summarize=true'
```

Se você tiver uma [conta na região da UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center), o URL acima começa com `https://api.eu.newrelic.com`.

<Callout variant="tip">
  Para evitar o resumo dos dados, omita `summarize` na sua chamada de API. Você não precisa especificar `&summarize=false`.
</Callout>
