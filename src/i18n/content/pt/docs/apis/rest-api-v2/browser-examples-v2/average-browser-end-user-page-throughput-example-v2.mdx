As taxas médias de transferência do browser aparecem no canto superior direito do gráfico <DNT>**Throughput by browser**</DNT> na sua [página<DNT>**Summary**</DNT> ](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance#viewing). Esta é a média de páginas por minuto (PPM) por tipo de browser no intervalo de tempo mostrado no gráfico.

## Taxas médias de transferência de páginas [#h2-average]

<Callout variant="important">
  Embora os exemplos utilizem New Relic API REST v2 da , recomendamos o uso de [NRQL funções](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/) para executar consulta métrica de fração de tempo. Cada valor de API pode ser mapeado para uma função NRQL equivalente. Para saber como criar uma consulta NRQL com base nesses exemplos API , consulte nossa [documentação](/docs/apis/rest-api-v2/migrate-to-nrql/).
</Callout>

Para obter a [média](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) do [intervalo de tempo](/docs/apm/apis/requirements/specifying-time-range-api-v2) (o padrão são os últimos 30 minutos), use o seguinte comando. Certifique-se de substituir as variáveis `$APP_ID` e `$API_KEY` neste exemplo pelo [ID do aplicativo](/docs/apm/apis/requirements/identification-code) específico e [pela chave de API REST](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing).

```sh
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json" \
     -H "X-Api-Key:$API_KEY" -i \
     -d 'names[]=EndUser&values[]=requests_per_minute&summarize=true'
```
