O REST API Explorer (v2) da New Relic facilita o teste e o envio de solicitações para qualquer endpoint de API disponível. Depois de selecionar sua escolha de funções para o tipo de chamada de API (aplicativo, browser, etc.), a interface do usuário fornece um formulário interativo para visualizar requisitos e testar seus valores de parâmetros. Ele também fornece uma fonte ativa de documentação sobre os valores da API.

## Requisitos de chave de API [#api-key]

Antes de poder usar o [API Explorer](https://rpm.newrelic.com/api/explore), o acesso à API deve ser ativado e uma [chave de API](/docs/apis/rest-api-v2/requirements/rest-api-key) deve ser gerada para sua conta.

<Callout variant="important">
  Recomendamos usar <InlinePopover type="userKey" />, não a [chave de API REST](/docs/apis/intro-apis/new-relic-api-keys/#rest-api-key), porque a chave do usuário tem menos restrições.
</Callout>

Pontas:

* Se você estiver conectado ao New Relic, ao usar o API Explorer você poderá escolher sua [chave de API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) na parte superior da interface e essa chave aparecerá automaticamente nas seções <DNT>**Request**</DNT> e <DNT>**Parameters**</DNT> do Explorer.
* Se você não estiver conectado ao New Relic, poderá colar sua [chave de API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) no <DNT>**Parameters**</DNT> do API Explorer.

## Use o API Explorer [#use-api-explorer]

1. Vá para o <DNT>**[API Explorer](https://api.newrelic.com/docs/)**</DNT>.
2. No dropdown <DNT>**Servers**</DNT>, escolha uma URL API baseada nos EUA ou na UE.
3. Clique em <DNT>**Authorize**</DNT>, insira sua chave de API de usuário e clique em <DNT>**Authorize**</DNT> novamente.
4. Expanda uma das funções de API disponíveis: <DNT>**GET**</DNT>, <DNT>**PUT**</DNT>, <DNT>**DELETE**</DNT>.
5. (Opcional) Adicione valores <DNT>**Parameters**</DNT> para sua chamada de API para filtrar a resposta (consulte a interface para obter descrições e requisitos para a v2.)
6. No dropdown <DNT>**Media type**</DNT>, escolha um formato para sua solicitação: JSON ou XML.
7. Clique em <DNT>**Try it out**</DNT> e depois em <DNT>**Execute**</DNT>.
