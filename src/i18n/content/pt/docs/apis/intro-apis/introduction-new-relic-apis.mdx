A New Relic oferece uma variedade de API e SDKs que você pode usar para:

* Envie dados para a New Relic.
* Recuperar dados do New Relic.
* Visualize e defina as configurações.

Procurando por chave de API? Consulte [chave da API](/docs/apis/intro-apis/new-relic-api-keys).

## API para ingestão de dados [#data-type-apis]

Nossa API de ingestão de dados é algumas de nossas [muitas soluções para geração de relatórios de dados](https://newrelic.com/instant-observability). Nossa API pode ser usada diretamente, mas também é a rota de ingestão subjacente para muitas de nossas ferramentas de relatório de dados. Se você está apenas começando a relatar dados para o New Relic, recomendamos começar em [Instalar o New Relic](/docs/new-relic-solutions/new-relic-one/install-configure/install-new-relic).

## NerdGraph (GraphQL) [#graphql]

[NerdGraph](/docs/apis/graphql-api/getting-started/introduction-new-relic-graphql-api) é a API que recomendamos para consultar dados do New Relic, consultar informações da conta e fazer uma série de configurações de recursos. Para saber o que você pode fazer, confira [os tutoriais do NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#tutorials).

NerdGraph é nossa mais nova API e é uma tentativa de reunir em um só lugar algumas de nossas APIs mais antigas.

## API REST [#rest-api]

Nossa [API REST V2](/docs/apis/rest-api-v2) é nossa API mais antiga para consulta e configuração, que [o NerdGraph](#graphql) está em processo de substituição. Estamos fazendo um trabalho mínimo em andamento na API REST. Sua primeira escolha deve ser usar o NerdGraph porque é nele que concentraremos nossa atenção e adicionaremos recursos daqui para frente.

## API por recurso [#product-apis]

As ferramentas e recursos da New Relic, como APM, monitoramento de infraestrutura, <InlinePopover type="browser" />e alerta, são frequentemente usados juntos e, às vezes, podem se sobrepor em funcionalidade. É por isso que várias API podem ser relevantes para cada área. Algumas funcionalidades da API dependerão do seu [acesso ao recurso e aos dados](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data).
