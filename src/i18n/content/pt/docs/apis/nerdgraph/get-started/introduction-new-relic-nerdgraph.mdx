NerdGraph é nossa API no formato GraphQL que permite consultar dados do New Relic e configurar alguns recursos do New Relic. Depois de [se inscrever para uma conta gratuita da New Relic](https://newrelic.com/signup) e [instalar](/docs/using-new-relic/cross-product-functions/install-configure/install-new-relic/) qualquer um de nossos serviços de monitoramento, você pode começar a usar o NerdGraph.

## O que é NerdGraph? [#overview]

NerdGraph é uma das várias [API](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/) da New Relic. NerdGraph é a API que recomendamos para consultar dados do New Relic e realizar algumas configurações específicas ([saiba mais sobre recurso](#tutorials)). NerdGraph fornece uma interface API única para retornar dados de várias APIs e microsserviços da New Relic. Com o tempo, outros recursos de configuração serão adicionados ao NerdGraph.

<Callout variant="important">
  NerdGraph não é usado para ingestão de dados. Para isso, você usaria nossa [API de ingestão de dados](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/#data-type-apis).
</Callout>

NerdGraph é construído usando [GraphQL](https://graphql.org), que é um formato de API de código aberto que permite solicitar exatamente os dados necessários, sem busca excessiva ou insuficiente.

## Requisitos [#requirements]

Antes de começar:

* Você precisa de uma conta New Relic e, com essa conta, poderá acessar sua API <InlinePopover type="userKey"/>que precisa incluir com consultas e mutações.
* O tipo de usuário e as funções atribuídas podem afetar suas permissões do NerdGraph. Para obter mais detalhes, consulte [Fatores que afetam o acesso](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data#user-permissions).

## Ponto final [#endpoints]

Os endpoints da API que você usa dependem da [região do seu data center](/docs/accounts/accounts-billing/account-setup/choose-your-data-center):

* Endpoint principal: `https://api.newrelic.com/graphql`
* Endpoint para contas que usam data center da UE: `https://api.eu.newrelic.com/graphql`

Para acessar o endpoint, você pode fazer chamadas a partir do seu código, usar o [NerdGraph API Explorer](/docs/apis/nerdgraph/get-started/nerdgraph-explorer) ou fazer um comando `curl` semelhante a este:

```bash
curl -X POST https://api.newrelic.com/graphql \
     -H 'Content-Type: application/json' \
     -H 'API-Key: YOUR_NEW_RELIC_USER_KEY' \
     -d '{ "query":  "{ requestContext { userId apiKey } }" }'
```

## Use o explorador de API NerdGraph [#explorer]

Uma maneira fácil de experimentar a API NerdGraph é usar o NerdGraph API Explorer. Para obter ajuda para começar a usar esta ferramenta, consulte [Tutorial do NerdGraph API Explorer](/docs/apis/nerdgraph/get-started/nerdgraph-explorer).

## Funcionalidade NerdGraph [#tutorials]

A funcionalidade do NerdGraph pode ser dividida em duas categorias principais:

* <DNT>**Querying data.**</DNT> Você pode buscar dados para diversas finalidades, incluindo usá-los em um fluxo de trabalho programático ou criar um [aplicativo New Relic](https://developer.newrelic.com/build-apps) para visualizações de dados personalizadas.
* <DNT>**Configuring features.**</DNT> Há uma variedade de configurações disponíveis e mais serão adicionadas ao longo do tempo. Você pode fazer coisas como adicionar tags, configurar carga de trabalho ou personalizar &amp;quot;métricas clássicas&amp;quot;.

Você pode usar o NerdGraph para retornar e configurar uma ampla variedade de dados. Uma maneira de ver o que o NerdGraph pode fazer é começar a construir a consulta e explorar o esquema usando o [explorador NerdGraph](/docs/apis/nerdgraph/get-started/nerdgraph-explorer).

## Terminologia NerdGraph [#terminology]

A seguir estão os termos originados do GraphQL, o formato de API que o NerdGraph usa.

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        Prazo
      </th>

      <th>
        Definição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Consulta e mutações
      </td>

      <td>
        Existem duas classes de operações GraphQL:

        * [consulta](https://graphql.org/learn/queries) são solicitações básicas usadas apenas para buscar dados. Estas consultas não são estáticas, o que significa que você pode solicitar mais ou menos dados, dependendo da sua necessidade. Para cada consulta, você pode especificar exatamente quais dados deseja recuperar, desde que sejam suportados pelo [esquema](https://graphql.org/learn/schema).
        * [Mutações](https://graphql.org/learn/queries/#mutations) são solicitações que executam uma ação, como criar um recurso ou alterar configuração. As mutações requerem a palavra-chave `mutation`, bem como o `name` da mutação.
      </td>
    </tr>

    <tr>
      <td>
        Tipo
      </td>

      <td>
        Os dados no GraphQL são organizados em tipos. Os tipos podem ser [escalares](https://graphql.org/learn/schema/#scalar-types) (como strings, números ou booleanos) ou tipos de objetos.

        Um [tipo de objeto](https://graphql.org/learn/schema/#object-types-and-fields) é um tipo personalizado composto por uma coleção de campos. Por exemplo, um tipo de objeto chamado `User` pode representar um usuário em um sistema.
      </td>
    </tr>

    <tr>
      <td>
        Campo
      </td>

      <td>
        Um [campo](https://graphql.org/learn/schema/#object-types-and-fields) representa uma informação sobre um tipo de objeto que pode ser consultado. Os campos podem ser escalares, listas ou objetos. Por exemplo, um tipo de objeto `User` poderia ter um campo de string chamado `name`.
      </td>
    </tr>

    <tr>
      <td>
        Interface
      </td>

      <td>
        Uma [interface](https://graphql.org/learn/schema/#interfaces) é um tipo abstrato que representa uma coleção de campos comuns que outros tipos de objetos podem `implement`.
      </td>
    </tr>
  </tbody>
</table>
