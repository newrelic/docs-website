Use o <DNT>NerdGraph API</DNT> para criar, testar, listar, atualizar e excluir <DNT>cross account destinations</DNT>. <DNT>Cross account destinations</DNT> permitem centralizar o gerenciamento de notificações no nível da organização.

## Crie um destino [#create-destination]

Para criar um destino, diferentes entradas devem ser fornecidas para cada tipo de destino.

<Callout variant="important">
  Você deve fornecer um objeto de escopo identificando onde o destino será criado. O objeto de escopo requer:

  * id: O ID da Conta ou Organização.
  * type: O tipo de escopo (Conta ou Organização).

  Uma propriedade opcional two\_way\_integration está disponível para integrações que permitem integração bidirecional.
</Callout>

        **Gerar a URL do OAuth**

        1. Abra o <DNT>[NerdGraph Explorer](https://api.newrelic.com/graphiql)</DNT> e cole a seguinte consulta no editor de consultas. Esta consulta gera uma URL OAuth do Slack exclusiva para sua organização.

           ```graphql
           {
             actor {
               organization {
                 aiNotifications {
                   oAuthUrl(type: SLACK) {
                     url
                     transactionId
                     error {
                       description
                       details
                       type
                     }
                   }
                 }
               }
             }
           }
           ```

        2. Clique em <DNT>**Play** (Execute Query)</DNT> na parte superior do explorador.
      </Step>

      <Step>
        **Acesse o link de autenticação**

        1. Assim que a consulta for executada, você verá uma resposta JSON no lado direito da tela contendo uma URL.
        2. Localize o campo `url` na resposta.
        3. Copie toda a string da URL (tudo dentro das aspas).
        4. Abra uma nova aba no seu navegador e cole a URL.
      </Step>

      <Step>
        **Concluir autorização do Slack**

        1. O link leva você para uma página de autorização do Slack.
        2. Se solicitado, faça login no workspace do Slack que você deseja conectar.
        3. Revise as permissões solicitadas pela New Relic.
        4. Para autorizar a conexão, clique em <DNT>**Allow**</DNT>.
        5. Você deve ser redirecionado de volta para uma página de &quot;Sucesso&quot; da New Relic.
      </Step>

      <Step>
        **Verifique o destino**

        Após a autenticação OAuth bem-sucedida, o destino é criado automaticamente no nível da organização, tornando-o disponível em todas as contas da sua organização.

        1. Navegue até <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Alerts &gt; Destinations**</DNT>.
        2. Na lista <DNT>**Destinations**</DNT>, agora você deve ver uma nova entrada para o workspace do Slack que você acabou de autenticar.
      </Step>
    </Steps>
