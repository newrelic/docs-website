<DNT>Autopilot</DNT> A memória permite que <DNT>Autopilot</DNT> retenha o contexto entre conversas e controle quem pode ler, criar, atualizar ou excluir esse contexto armazenado.

## Acesso padrão à memória [#default-access]

O New Relic fornece acesso padrão à memória por meio destas funções padrão:

### Acesso com escopo de conta

<table>
  <thead>
    <tr>
      <th>
        Ação
      </th>

      <th>
        Função necessária
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Ler memória
      </td>

      <td>
        Somente leitura, Usuário padrão ou Usuário completo (qualquer nível de usuário)
      </td>
    </tr>

    <tr>
      <td>
        Criar ou modificar memória
      </td>

      <td>
        Administrador de todos os produtos
      </td>
    </tr>

    <tr>
      <td>
        Excluir memória
      </td>

      <td>
        Administrador de todos os produtos
      </td>
    </tr>
  </tbody>
</table>

### Acesso com escopo de organização

<table>
  <thead>
    <tr>
      <th>
        Ação
      </th>

      <th>
        Função necessária
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Ler memória em toda a organização
      </td>

      <td>
        Somente leitura da organização, Gerente da organização ou Administrador de produto da organização
      </td>
    </tr>

    <tr>
      <td>
        Criar ou modificar memória em toda a organização
      </td>

      <td>
        Gerenciador de Organização
      </td>
    </tr>

    <tr>
      <td>
        Excluir memória em toda a organização
      </td>

      <td>
        Gerenciador de Organização
      </td>
    </tr>
  </tbody>
</table>

Você pode precisar de uma função personalizada se as permissões padrão não atenderem às necessidades da sua organização.

## Criar uma função personalizada via API NerdGraph [#create-role]

Cada uma das seguintes chamadas de API lida com uma parte da criação e atribuição de uma função personalizada para a memória. Expanda cada uma para ver seus parâmetros de entrada e exemplos de solicitação e resposta.

<CollapserGroup>
  <Collapser id="find-permission-ids" title="Encontre os IDs de permissão necessários para a memória">
    Use a consulta `customerAdministration` para recuperar uma lista de recursos, seus IDs de permissão e informações relacionadas. Filtre por recurso para obter as permissões de memória.

    **Parâmetro de entrada**

    <table>
      <thead>
        <tr>
          <th>
            Parâmetro
          </th>

          <th>
            Tipo de dados
          </th>

          <th>
            Descrição
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `eq`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            Defina o valor como 

            `Memory`

             (com escopo de conta) ou 

            `Memory For Organization`

             (com escopo de organização) para recuperar as permissões correspondentes.
          </td>
        </tr>
      </tbody>
    </table>

    **Exemplo de requisição — com escopo de conta**

    ```graphql
    {
      customerAdministration {
        permissions(filter: { feature: { eq: "Memory" } }) {
          items {
            feature
            category
            id
            product
          }
        }
      }
    }
    ```

    **Resposta de amostra**

    ```json
    {
      "data": {
        "customerAdministration": {
          "permissions": {
            "items": [
              {
                "category": "READ",
                "feature": "Memory",
                "id": "xxxxx",
                "product": "New Relic AI"
              },
              {
                "category": "MODIFY",
                "feature": "Memory",
                "id": "xxxxxx",
                "product": "New Relic AI"
              },
              {
                "category": "DELETE",
                "feature": "Memory",
                "id": "xxxxxx",
                "product": "New Relic AI"
              }
            ]
          }
        }
      }
    }
    ```

    Execute a mesma consulta com `feature: { eq: "Memory For Organization" }` para recuperar os IDs de permissão com escopo de organização (READ, MODIFY, DELETE para memória para organização).

    Na resposta, copie os IDs de permissão em que `feature` é `Memory` ou `Memory For Organization`, correspondendo ao nível de acesso que você deseja conceder. Você precisará desses IDs para criar a função personalizada.
  </Collapser>

  <Collapser id="retrieve-organization-id" title="Recupere o ID da sua organização">
    Recupere o ID da sua organização, que você usará em mutações subsequentes. Para uma função com escopo de conta, recupere também o ID da conta.

    **Solicitação de amostra**

    ```graphql
    {
      actor {
        organization {
          id
        }
        accounts {
          id
          name
        }
      }
    }
    ```

    **Resposta de amostra**

    ```json
    {
      "data": {
        "actor": {
          "organization": {
            "id": "YOUR_ORGANIZATION_ID"
          },
          "accounts": [
            {
              "id": "YOUR_ACCOUNT_ID",
              "name": "Your Account Name"
            }
          ]
        }
      }
    }
    ```

    Copie o(s) ID(s) de que você precisa da resposta. Você os usará para criar a função personalizada.
  </Collapser>

  <Collapser id="create-custom-role" title="Criar a função personalizada">
    Use a mutação `customRoleCreate` para criar sua função personalizada para o gerenciamento de memória.

    **Parâmetro de entrada**

    <table>
      <thead>
        <tr>
          <th>
            Parâmetro
          </th>

          <th>
            Tipo de dados
          </th>

          <th>
            Descrição
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `id`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            O ID da organização (com escopo de organização) ou o ID da conta (com escopo de conta) da chamada anterior.
          </td>
        </tr>

        <tr>
          <td>
            `type`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            Defina como 

            `organization`

             para uma função com escopo de organização ou 

            `account`

             para uma função com escopo de conta.
          </td>
        </tr>

        <tr>
          <td>
            `name`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            O nome de exibição para a função personalizada.
          </td>
        </tr>

        <tr>
          <td>
            `permissionIds`

             (Obrigatório)
          </td>

          <td>
            Matriz de inteiros
          </td>

          <td>
            Os IDs de permissão de memória identificados acima.
          </td>
        </tr>

        <tr>
          <td>
            `scope`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            Defina como 

            `organization`

             ou 

            `account`

            , correspondendo a 

            `type`

            .
          </td>
        </tr>
      </tbody>
    </table>

    **Exemplo de solicitação — função com escopo de conta**

    ```graphql
    mutation {
      customRoleCreate(
        container: { id: "YOUR_ACCOUNT_ID", type: "account" }
        name: "Memory manager"
        permissionIds: [xxxxx, xxxxx]
        scope: "account"
      ) {
        id
      }
    }
    ```

    **Exemplo de solicitação — função com escopo de organização**

    ```graphql
    mutation {
      customRoleCreate(
        container: { id: "YOUR_ORGANIZATION_ID", type: "organization" }
        name: "Memory manager (org-wide)"
        permissionIds: [xxxxx, xxxxx]
        scope: "organization"
      ) {
        id
      }
    }
    ```

    **Resposta de amostra**

    ```json
    {
      "data": {
        "customRoleCreate": {
          "id": 9999999
        }
      }
    }
    ```

    Salve o ID da função retornado — você precisará dele para atribuir essa função a um grupo de usuários.
  </Collapser>

  <Collapser id="add-role-to-group" title="Adicione a função personalizada a um grupo de usuários">
    Após criar a função personalizada, atribua-a a um grupo de usuários.

    **Recuperar IDs de grupo**

    Use a consulta `customerAdministration` para obter uma lista de grupos de usuários disponíveis.

    **Parâmetro de entrada**

    <table>
      <thead>
        <tr>
          <th>
            Parâmetro
          </th>

          <th>
            Tipo de dados
          </th>

          <th>
            Descrição
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `id`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            ID da sua organização.
          </td>
        </tr>
      </tbody>
    </table>

    **Solicitação de amostra**

    ```graphql
    {
      customerAdministration {
        groups(filter: { organizationId: { eq: "YOUR_ORGANIZATION_ID" } }) {
          nextCursor
          items {
            id
            name
            users {
              items {
                id
                email
              }
            }
          }
        }
      }
    }
    ```

    Na resposta, copie o ID do grupo para o qual você deseja atribuir a função de memória.

    **Atribua a função ao grupo**

    Use a mutação `authorizationManagementGrantAccess` para atribuir a função personalizada a um grupo de usuários.

    **Parâmetro de entrada**

    <table>
      <thead>
        <tr>
          <th>
            Parâmetro
          </th>

          <th>
            Tipo de dados
          </th>

          <th>
            Descrição
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `roleId`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            O ID da função personalizada da chamada anterior.
          </td>
        </tr>

        <tr>
          <td>
            `groupId`

             (Obrigatório)
          </td>

          <td>
            Corda
          </td>

          <td>
            O ID do grupo obtido acima.
          </td>
        </tr>
      </tbody>
    </table>

    **Exemplo de solicitação — função com escopo de conta**

    ```graphql
    mutation {
      authorizationManagementGrantAccess(
        grantAccessOptions: {
          accountAccessGrants: { accountId: "YOUR_ACCOUNT_ID", roleId: "YOUR_ROLE_ID" }
        }
        groupId: "YOUR_GROUP_ID"
      ) {
        roles {
          id
          name
          accountId
          roleId
          groupId
          displayName
        }
      }
    }
    ```

    **Exemplo de solicitação — função com escopo de organização**

    ```graphql
    mutation {
      authorizationManagementGrantAccess(
        grantAccessOptions: {
          organizationAccessGrants: { roleId: "YOUR_ROLE_ID" }
        }
        groupId: "YOUR_GROUP_ID"
      ) {
        roles {
          id
          name
          organizationId
          roleId
          groupId
          displayName
        }
      }
    }
    ```

    **Resposta de amostra**

    ```json
    {
      "data": {
        "authorizationManagementGrantAccess": {
          "roles": [
            {
              "displayName": "Memory manager",
              "groupId": null,
              "id": "99999999",
              "name": "memory_manager",
              "organizationId": "YOUR_ORGANIZATION_ID",
              "roleId": 99999
            }
          ]
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>
