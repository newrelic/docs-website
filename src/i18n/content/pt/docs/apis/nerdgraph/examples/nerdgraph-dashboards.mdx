Você pode usar [nossa API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) para criar e gerenciar <InlinePopover type="dashboards" />.

## Visão geral [#starting-out]

Para obter uma introdução ao nosso recurso personalizado do painel, consulte [a documentação do painel](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Ao usar o NerdGraph para configurar o painel, é útil entender que nossos painéis são considerados [entidade](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic), que possuem seus próprios IDs de entidade, semelhantes a outras coisas que consideramos entidade, como aplicativos de monitoramento, hosts e serviços.

Para saber como adicionar e configurar widgets e gráficos em um dashboard, consulte [Configurar gráficos e outros widgets](/docs/apis/nerdgraph/examples/create-widgets-dashboards-api).

## Operações CRUD dashboard [#crud-operations]

Este documento explica como usar nossa API NerdGraph para criar, ler, atualizar e excluir dashboards (CRUD). Essas operações modificam todo o dashboard.

### Crie um dashboard [#create-dashboard]

Um dashboard requer pelo menos uma página. Você pode criar um dashboard com uma ou mais páginas, e cada página pode ter um ou mais widgets.

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use API dashboard do NerdGraph para criar um novo dashboard e sua configuração.
3. Certifique-se de incluir todos os campos obrigatórios, como `name`, `permissions` e pelo menos uma página.

<CollapserGroup>
  <Collapser id="mutation-create-dashboard" title="Dashboard com uma página, um widget e uma variável">
    ```graphql
    mutation {
    dashboardCreate(
      accountId: 1
      dashboard: {
        name: "My New Dashboard"
        description: "This is my new dashboard created via NerdGraph"
        permissions: PUBLIC_READ_WRITE
        pages: [
          {
            name: "Page 1"
            description: "This is the first page"
            widgets: [
              {
                visualization: { id: "viz.billboard" }
                layout: { column: 1, row: 1, height: 3, width: 4 }
                title: "Total Transaction Count"
                rawConfiguration: {
                  nrqlQueries: [
                    {
                      accountIds: [1],
                      query: "SELECT count(*) FROM Transaction"
                    }
                  ]
                }
              }
            ]
          }
        ]
        variables: [
          {
            name: "nrql_variable"
            title: "countries"
            type: NRQL
            nrqlQuery: {
              accountIds: [1],
              query: "SELECT uniqueCount(countryCode) FROM PageView"
            }
            replacementStrategy: STRING
          }
        ]
      }
    ) {
    entityResult {
      guid
      name
      description
      createdAt
      updatedAt
      owner {
        email
        userId
      }
      permissions
      pages {
        guid
        name
        description
        createdAt
        updatedAt
        widgets {
          id
          visualization { id }
          layout { column row height width }
          title
          linkedEntities { guid }
          rawConfiguration
        }
      }
      variables {
        name
        items {
          title
          value
        }
        defaultValues {
          value {
            string
           }
        }
        nrqlQuery {
          accountIds
          query
        }
        options {
          excluded
          ignoreTimeRange
          showApplyAction
        }

      type
      description
    }
    }
    ```

### Ler um dashboard [#read-dashboard]

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use `actor > entity()` para encontrar o dashboard pelo GUID da entidade.
3. Use API dashboard do NerdGraph para ler o dashboard existente e sua configuração.

Neste exemplo, substitua `DASHBOARD_GUID` pelo GUID real do dashboard que você deseja ler:

```graphql
query GetDashboardEntityQuery {
	actor{
		entity(guid:"DASHBOARD_GUID"){
			...on DashboardEntity {
				guid
				name
				description
				createdAt
				updatedAt
				owner {
					email
					userId
				}
				permissions
				pages {
					guid
					name
					description
					createdAt
					updatedAt
					widgets {
						id
						visualization {
							id
						}
						layout {
							column
							row
							height
							width
						}
						title
						linkedEntities {
							guid
						}
						rawConfiguration
					}
				}
				variables {
					name
					items {
						title
						value
					}
					defaultValues {
						value {
							string
						}
					}
					nrqlQuery {
						accountIds
						query
					}
					options {
						excluded
						ignoreTimeRange
						showApplyAction
					}
					title
					type
					isMultiSelection
					replacementStrategy
				}
			}
		}
	}
}
```

Dependendo das informações que você deseja recuperar, você pode modificar os campos na consulta.

### Atualizar um dashboard [#update-dashboard]

Para atualizar um dashboard, você precisa fornecer a configuração completa do dashboard, incluindo todas as suas páginas e widgets, mesmo que esteja atualizando apenas um elemento. A operação de atualização é uma substituição completa do conteúdo do dashboard.

<Callout variant="important">
  Ao atualizar um dashboard, se o GUID da página ou o ID widget não forem fornecidos, as páginas ou widgets existentes serão removidos do dashboard e substituídos pelos novos especificados na mutação.
</Callout>

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use API dashboard do NerdGraph para obter a configuração dashboard existente usando a leitura do dashboard pelo GUID da entidade, conforme mostrado na seção [Ler um dashboard](#read-dashboard).
3. Modifique os campos que deseja atualizar na configuração dashboard.
4. Use API dashboard do NerdGraph para atualizar o dashboard existente com a configuração modificada.

<Callout variant="important">
  Se você estiver usando `Facet Linking` no seu widget, ao ler o dashboard o campo será `linkedEntities { guid }`, mas ao atualizar o dashboard você precisará usar `linkedEntityGuids: [ "GUID" ]`.
</Callout>

### Excluir um dashboard [#delete-dashboard]

Para excluir um dashboard, você precisa fornecer o GUID da entidade do dashboard que deseja excluir. Esta operação executa uma exclusão lógica que permite recuperar seu dashboard.

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use API do dashboard do NerdGraph para excluir o dashboard pelo GUID da entidade.
3. Confirme a exclusão marcando `status` e `errors`.

<CollapserGroup>
  <Collapser id="mutation-delete-dashboard" title="Excluir um dashboard">
    ```graphql
    mutation {
      dashboardDelete(guid: "DASHBOARD_GUID") {
        status
        errors {
          type
          description
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>

#### Recuperar um dashboard [#undelete-dashboard]

Você pode recuperar um dashboard excluído anteriormente, fornecendo um GUID de entidade dashboard. A tag personalizada não pode ser recuperada.

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use API dashboard do NerdGraph para recuperar o dashboard excluído pelo GUID da entidade.
3. Confirme a recuperação marcando `errors` se houver.

<CollapserGroup>
  <Collapser id="mutation-undelete-dashboard" title="Recuperar um dashboard">
    ```graphql
    mutation {
      dashboardUndelete(guid: "DASHBOARD_GUID") {
        errors {
          type
          description
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## Operações da página dashboard [#page-operations]

Esta operação modifica uma página específica de um dashboard.

### Atualizar uma página dashboard [#update-dashboard-page]

Você pode atualizar uma página de um dashboard existente, dado um GUID de entidade de página de dashboard. Você precisa especificar o elemento de página dashboard completo e atualizado, desde os metadados até a configuração widget.

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use API dashboard do NerdGraph para obter a configuração dashboard existente usando a leitura do dashboard pelo GUID da entidade, conforme mostrado na seção [Ler um dashboard](#read-dashboard).
3. Identifique e extraia a página que você deseja atualizar do dashboard.
4. Modifique os campos que deseja atualizar na página.
5. Use `dashboardUpdatePage()` para modificar a página.
6. Marque `errors` se houver.

<Callout variant="important">
  Ao atualizar uma página, se os IDs widget não forem fornecidos, o widget existente será removido do dashboard e substituído pelos novos especificados na mutação.
</Callout>

<Callout variant="tip">
  * Você pode adicionar um novo widget a uma página incluindo-o na matriz `widgets` sem um campo `id`.
  * Você pode remover um widget de uma página omitindo-o da matriz `widgets`.
</Callout>

<CollapserGroup>

    ```graphql

        errors {
          type

        }
      }
    }
    ```

### Atualiza o widget em uma página [#update-widgets-in-page]

Você pode atualizar um conjunto de widgets existentes de uma página de dashboard, dado um GUID de entidade de página de dashboard. Você precisa especificar o conjunto de widgets a serem atualizados e sua configuração completa.

<Callout variant="important">
  Esta operação não permite adicionar ou remover widgets de uma página. Para adicionar ou remover um widget, use a operação [Atualizar uma página dashboard ](#update-dashboard-page).
</Callout>

1. Vá para o explorador NerdGraph GraphiQL em [api.newrelic.com/graphiql](https://api.newrelic.com/graphiql).
2. Use API dashboard do NerdGraph para obter a configuração dashboard existente usando a leitura do dashboard pelo GUID da entidade, conforme mostrado na seção [Ler um dashboard](#read-dashboard).
3. Identifique e extraia o widget na página que você deseja atualizar do dashboard.
4. Modifique os campos que deseja atualizar no widget.
5. Use `dashboardUpdateWidgetsInPage()` para modificar o widget.
6. Marque `errors` se houver.

```graphql
mutation UpdateWidgetsInPage($pageGuid: EntityGuid!, $widgets: [DashboardUpdateWidgetInput!]!) {
  dashboardUpdateWidgetsInPage(guid: $pageGuid, widgets: $widgets) {
    errors {
      type
      description
    }
  }
}
```

E aqui estão as variáveis para usar com esta mutação:

```json lineHighlight=2,8
{

    {
```

## Painel entre contas [#cross-account]

Com o NerdGraph, você pode [criar consultas de dados de mais de uma conta New Relic](/docs/apis/nerdgraph/examples/nerdgraph-nrql-tutorial#cross-account-query). Você também pode criar um dashboard usando dados de várias contas adicionando IDs de contas à matriz `accountIds`.

Aqui está um exemplo de criação de um dashboard entre contas:

<CollapserGroup>
  <Collapser id="cross-account-dashboard" title="Dashboard com consulta entre contas">
    ```graphql lineHighlight=21,37,53
    mutation {
      dashboardCreate(
        accountId: 1
        dashboard: {
          name: "Cross account queries - NerdGraph API - Cross Account Test1"
          description: null
          permissions: PUBLIC_READ_WRITE
          pages: [
            {
              name: "Page 1"
              description: null
              widgets: [
                {
                  visualization: { id: "viz.billboard" }
                  layout: { column: 1, row: 1, height: 3, width: 4 }
                  title: "Cross Account Total Transaction Count"
                  rawConfiguration: {
                    legend: { enabled: true }
                    nrqlQueries: [
                      {
                        accountIds: [1, 1606862, 2212585]
                        query: "SELECT count(*) FROM Transaction"
                      }
                    ]
                    yAxisLeft: { zero: true }
                  }
                  linkedEntityGuids: null
                }
                {
                  visualization: { id: "viz.pie" }
                  layout: { column: 5, row: 1, height: 3, width: 4 }
                  title: "Cross Account Pie"
                  rawConfiguration: {
                    legend: { enabled: true }
                    nrqlQueries: [
                      {
                        accountIds: [1, 1606862, 2212585]
                        query: "SELECT count(*) FROM Transaction FACET accountId()"
                      }
                    ]
                    yAxisLeft: { zero: true }
                  }
    ```

## Erros como cidadãos de primeira classe [#errors-first-class]

Todas as mutações dashboard oferecem uma maneira de solicitar erros ao serem executadas. Isso significa que você pode realizar alterações dashboard e verificar a resposta para detectar possíveis problemas esperados. Cada erro tem um tipo e uma descrição para ajudá-lo a identificar qual é a origem do problema.

<CollapserGroup>
  <Collapser id="errors-first-class" title="Erros como parte de cada resposta de mutação">
    ```graphql lineHighlight=2
    mutation {
      dashboardMutation(guid: "MY_EXISTING_DASHBOARD_GUID") {
        mutationResult {
          result
        }
        errors {
          description
          type
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>

Tenha em mente que estes são erros esperados dos quais estamos cientes antecipadamente. Você também deve verificar se há erros inesperados que serão retornados no [campo de erros padrão do GraphQL](https://graphql.org/learn/serving-over-http/#response).
