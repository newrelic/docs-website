Configurar <InlinePopover type="alerts"/>para notificá-lo sobre problemas com sua stack é uma das coisas mais importantes que você pode fazer para monitor seus dados. Mas criar e gerenciar alertas muitas vezes parece cansativo. Nos tutoriais a seguir, orientaremos você nas três maneiras principais de criar alertas com o New Relic. Também mostraremos como usar esses alertas para coletar informações e usar esses dados para melhorar sua stack. ## O que são alertas? [#alerts-what]

Alerta são condições que, quando atendidas, enviam uma notificação sobre o evento desencadeador. Você pode receber notificações para qualquer número de pessoas em qualquer número de equipes. Quando configurado corretamente, o alerta permite resolver um problema com mais eficiência, enviando notificações diretas aos grupos que podem resolver o problema. Existem cinco partes principais para alertar:

* <DoNotTranslate>**Incidents**</DoNotTranslate>

  são eventos em que um limite de condição é violado. Exemplos comuns de incidente são quando a latência ultrapassa um determinado limite, uma página não carrega ou ocorre um erro de JavaScript

* <DoNotTranslate>**Issues**</DoNotTranslate>

  são uma coleção de um ou mais incidentes que fazem com que uma notificação seja enviada

* <DoNotTranslate>**Conditions**</DoNotTranslate>

  são objetos de configuração que você configura para abrir um incidente

* <DoNotTranslate>**Policies**</DoNotTranslate>

  são agrupamentos de condições onde você define como será notificado quando ocorrer um incidente

* <DoNotTranslate>**Notifications**</DoNotTranslate>

  são as mensagens que você recebe quando um problema é aberto, confirmado ou fechado

<Callout variant="tip">
  Qual é a diferença entre <DoNotTranslate>**Incidents**</DoNotTranslate> e <DoNotTranslate>**Issues**</DoNotTranslate>?

  Um <DoNotTranslate>**incident**</DoNotTranslate> é um evento individual que detalha um sintoma de um problema, enquanto um <DoNotTranslate>**issue**</DoNotTranslate> é um agrupamento de incidentes que descreve um problema maior no seu sistema.
</Callout>

## Onde estão os alertas em New Relic? [#alerts-where]

No New Relic, você geralmente gerencia seu alerta em [<DoNotTranslate>**one.newrelic.com > Alerts & AI**</DoNotTranslate>](https://one.newrelic.com/alerts-ai). Esta seção da plataforma tem uma página <DoNotTranslate>**Overview**</DoNotTranslate> que fornece detalhamentos visuais de todos os seus alertas, bem como uma página <DoNotTranslate>**Alert conditions (policies)**</DoNotTranslate> que fornece uma lista de todas as suas políticas de alertas, bem como um local para criar novas.
