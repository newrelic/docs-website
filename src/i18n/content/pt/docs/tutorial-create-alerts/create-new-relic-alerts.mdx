Configurar <InlinePopover type="alerts"/>para notificá-lo sobre problemas com sua stack é uma das coisas mais importantes que você pode fazer para monitor seus dados. Mas criar e gerenciar alertas muitas vezes parece cansativo. Nos tutoriais a seguir, orientaremos você nas três maneiras principais de criar alertas com o New Relic. Também mostraremos como usar esses alertas para coletar informações e usar esses dados para melhorar sua stack. Se você é completamente novo no alerta, temos alguns conceitos básicos e terminologia de alerta para aprender antes de começar.

## O que são alertas? [#alerts-what]

Alerta são condições que, quando atendidas, enviam uma notificação sobre o evento desencadeador. Você pode receber notificações para qualquer número de pessoas em qualquer número de equipes. Quando configurado corretamente, o alerta permite resolver um problema com mais eficiência, enviando notificações diretas aos grupos que podem resolver o problema. Existem cinco partes principais para alertar:

* <DNT>
    **Incidents**
  </DNT>

  são eventos em que um limite de condição é violado. Exemplos comuns de incidente são quando a latência ultrapassa um determinado limite, uma página não carrega ou ocorre um erro de JavaScript

* <DNT>
    **Issues**
  </DNT>

  são uma coleção de um ou mais incidentes que fazem com que uma notificação seja enviada

* <DNT>
    **Conditions**
  </DNT>

  são objetos de configuração que você configura para abrir um incidente

* <DNT>
    **Policies**
  </DNT>

  são agrupamentos de condições onde você define como será notificado quando ocorrer um incidente

* <DNT>
    **Notifications**
  </DNT>

  são as mensagens que você recebe quando um problema é aberto, confirmado ou fechado

<Callout variant="tip">
  Qual é a diferença entre <DNT>**Incidents**</DNT> e <DNT>**Issues**</DNT>?

  Um <DNT>**incident**</DNT> é um evento individual que detalha um sintoma de um problema, enquanto um <DNT>**issue**</DNT> é um agrupamento de incidentes que descreve um problema maior no seu sistema.
</Callout>

## Onde estão os alertas em New Relic? [#alerts-where]

No New Relic, você geralmente gerencia seu alerta em [<DNT>**one.newrelic.com > Alerts**</DNT>](https://one.newrelic.com/alerts-ai). Esta seção da plataforma tem uma página <DNT>**Overview**</DNT> que fornece detalhamentos visuais de todos os seus alertas, bem como uma página <DNT>**Alert conditions (policies)**</DNT> que fornece uma lista de todas as suas políticas de alertas, bem como um local para criar novas.
