Recomendamos que você revise [nossos documentos sobre tipos de amostragem](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works) antes de começar para determinar se deseja usar distributed tracing padrão ou [o rastreamento infinito](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing). Além disso, se você estiver usando o agente <InlinePopover type="apm"/>New Relic e quiser ativar distributed tracing, consulte nosso [guia de planejamento](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide).

Pronto para começar? Se ainda não o fez, crie sua conta New Relic gratuita abaixo para começar a monitorar seus dados hoje mesmo.

<InlineSignup/>

Para configurar distributed tracing, você concluirá três etapas gerais:

<img
  title="Diagram showing the three steps to set up distributed tracing."
  alt="Diagram showing the three steps to set up distributed tracing"
  src={distributedtracingHighLevelDistTr}
/>

1. <DoNotTranslate>**Identify services:**</DoNotTranslate> Identifique e anote o endpoint, os serviços, os idiomas e os sistemas usados para concluir essa solicitação (você precisará dessas informações na próxima etapa). Se você tiver um diagrama de ambiente como o seguinte, poderá usá-lo para criar uma lista de serviços que tratam de solicitações:

   <img
     title="A sample environment diagram showing how you might identify services."
     alt="A sample environment diagram showing how you might identify services."
     src={distributedtracingDistributedTracingMap}
   />

2. <DoNotTranslate>**Instrument services:**</DoNotTranslate> Instrumento cada serviço que você identifica para que ele possa enviar seus dados trace . Algumas ferramentas, como o <InlinePopover type="apm"/>agente, o instrumento atende automaticamente, enquanto outras ferramentas exigem a inserção de algum código nos serviços.
