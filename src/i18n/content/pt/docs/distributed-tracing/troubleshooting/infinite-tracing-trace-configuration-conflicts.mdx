## Problema

Os conflitos de rastreamento com configuração são causados quando você habilita [o Rastreamento Infinito](/docs/understand-dependencies/distributed-tracing/infinite-tracing/introduction-infinite-tracing) para um ou mais serviços em um trace, mas outros serviços nesse trace têm nossas soluções distributed tracing padrão (não-Rastreamento Infinito) habilitadas. Você pode resolver esse problema ativando o rastreamento infinito para todos os serviços em um trace.

Um conflito de configuração de rastreamento pode causar problemas como:

* Rastreamento fragmentado e extensões órfãs.

* Métrica de rastreamento infinito incompleta devido à não consideração da amostragem padrão de rastreamento.

* Discrepâncias para métricas como contagem de span, contagem de serviços, durações e contagem de erros.

* Resultados de pesquisa confusos. Por exemplo, para intervalos de rastreamento padrão que enviam cabeçalhos para serviços de instrumento de rastreamento infinito, esses intervalos podem aparecer na lista trace , mas não na visualização em cascata de trace .

* Rastreamento ausente para aplicativos de browser de monitoramento e aplicativos móveis. Como o Infinite Tracing ainda não está disponível para

  <InlinePopover type="mobile"/>

  ou

  <InlinePopover type="browser"/>

  , os períodos desses serviços não aparecerão na cascata de trace quando fizerem solicitações para serviços habilitados para Infinite Tracing.

Para rastreamento com conflito de configuração, exibiremos apenas os dados do Rastreamento Infinito na interface porque esses dados são de qualidade superior. (Todos os dados de rastreamento podem ser consultados via NRQL.)

Você pode usar nosso [aplicativo de status de instrumentação](https://one.newrelic.com/dt-setup-assistant) para ter uma visão geral de sua instrumentação trace . Para encontrar isso, acesse: <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Apps > Distributed tracing instrumentation status**</DNT>.
