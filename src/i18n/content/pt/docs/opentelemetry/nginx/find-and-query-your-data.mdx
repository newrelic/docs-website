## Visualize seus dados do NGINX no New Relic [#find-data]

Depois que a configuração estiver concluída e os dados estiverem fluindo, você pode visualizar suas métricas e logs do NGINX no New Relic e criar alertas personalizados.

**Método 1: Através de Integrações e Agentes**

1. Acesse [one.newrelic.com](https://one.newrelic.com) &gt; **Integrations &amp; Agents**
2. Clique em **Dashboards**
3. Encontre e clique no dashboard **NGINX OTel overview**
4. Selecione sua conta e clique em **View dashboard**

**Método 2: Através do monitoramento da infraestrutura**

1. Acesse [one.newrelic.com](https://one.newrelic.com) &gt; **On-host integrations**
2. Pesquise seu servidor NGINX por nome de implantação (por exemplo, `production-web-01`)
3. Clique no seu servidor para ver métricas detalhadas

## Consulte seus dados com NRQL

Você pode criar consultas personalizadas para analisar seus dados do NGINX:

**Exemplo de consulta:**

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## Solucionar problemas de visibilidade de dados

Se você não vir dados:

1. Aguarde um pouco - a ingestão inicial de dados pode levar alguns minutos
2. Verifique o intervalo de tempo - Certifique-se de estar visualizando dados recentes
3. Verifique o nome da implantação - Use o nome exato da sua configuração do Collector
4. Verifique se há dados com NRQL:
   ```sql
   FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
   ```

Se os dados parecerem intermitentes:

1. Verifique os logs do Collector em busca de erros:
   ```bash
   sudo journalctl -u otelcol-contrib -n 50
   ```
2. Verifique se o NGINX está gerando tráfego - Teste com `curl http://localhost`
3. Verifique a autenticação - Certifique-se de que a chave de licença e o endpoint estejam corretos
