## Por que monitorar o NGINX com OpenTelemetry? [#why-monitor]

O monitoramento NGINX é essencial para manter o desempenho e a confiabilidade do aplicativo web. O OpenTelemetry fornece uma abordagem padronizada para coletar e analisar esses dados críticos.

### Principais benefícios

* **Otimização de desempenho**: identifique gargalos, acompanhe os tempos de resposta, tome decisões de dimensionamento baseadas em dados
* **Monitoramento proativo**: Receba alertas antes do impacto do usuário, detecte problemas precocemente
* **Insights operacionais**: Entenda os padrões de tráfego, acompanhe a conformidade com o SLA, mantenha trilhas de auditoria
* **À prova de futuro**: padrão OpenTelemetry de código aberto, integração neutra em relação ao fornecedor

### Caso de uso

Se você está executando um site de comércio eletrônico, um gateway de API ou uma plataforma de entrega de conteúdo, o monitoramento NGINX ajuda você a manter serviços web rápidos e confiáveis. Seja notificado quando seu processo de checkout ficar lento, detecte quando seus endpoints de API estiverem sobrecarregados ou identifique quais regiões geográficas estão enfrentando problemas. Este monitoramento é essencial para sites que lidam com transações de clientes, backends de aplicativos móveis, arquiteturas de microsserviços e qualquer aplicativo web em que o tempo de inatividade ou o desempenho lento impactam diretamente seus usuários e negócios.

## Métricas disponíveis [#metrics]

O coletor OpenTelemetry coleta automaticamente as principais métricas de desempenho do NGINX, incluindo estatísticas de conexão, contagem de solicitações e indicadores de integridade do servidor.

**Métricas principais coletadas:**

* **Métricas de conexão** - Conexões ativas, aceitas, tratadas e seus estados
* **Métricas de solicitação** - Total de solicitações e taxas de solicitação ao longo do tempo
* **Integridade do servidor** - Eficiência de conexão e indicadores de desempenho

**Para referência completa de métricas e atributos:** Consulte [Referência de métricas e atributos do NGINX OpenTelemetry](/docs/opentelemetry/nginx/nginx-otel-metrics-reference/) para descrições detalhadas, tipos de métricas e atributos de recursos para implantações on-host e Kubernetes.
