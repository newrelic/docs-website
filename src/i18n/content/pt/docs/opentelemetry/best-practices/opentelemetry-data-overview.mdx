O OpenTelemetry oferece um ecossistema versátil para instrumentação e telemetria, que pode ser integrado perfeitamente ao New Relic por meio do [OpenTelemetry Protocol (OTLP)](https://opentelemetry.io/docs/specs/otlp/). Este guia fornece uma visão geral de como o New Relic processa e ingere dados OTLP para ajudá-lo a monitorar e observar seus sistemas de forma eficiente.

## Iniciar [#get-started]

Para saber mais sobre como integrar o OpenTelemetry com o New Relic, consulte [Introdução ao OpenTelemetry e ao New Relic](/docs/opentelemetry/get-started/opentelemetry-get-started-intro/). Esta página aborda padrões comuns de integração e práticas recomendadas.

## Configurar OTLP para New Relic [#configure]

Para obter requisitos detalhados de configuração do OTLP e orientações sobre resolução de problemas, visite as seguintes páginas:

* [O endpoint OTLP do New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/) orienta você nas etapas essenciais de configuração para integrar dados do OpenTelemetry ao New Relic.
* [A resolução de problemasNew Relic OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp-troubleshooting/) ajuda você a corrigir problemas comuns com ingestão de dados, como verificação de log, chave de API e problemas de conexão.

### Tipos de sinais em OpenTelemetry [#signals]

O OTLP é dividido em sinais de trace, métricos e log , todos os quais incluem o conceito de recursos. As páginas a seguir fornecem detalhes sobre como cada métrica é processada e ingerida no New Relic:

* [Traces de OpenTelemetry no New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/)
* [Métricas de OpenTelemetry em New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-metrics/)
* [Registro OpenTelemetry no New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-logs/)
* [Recursos OpenTelemetry no New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/)

## Gerenciar volume de dados [#manage-volume]

Para gerenciar o volume de dados do OpenTelemetry e controlar os custos de ingestão, consulte [Gerenciar volume de ingestão de dados do OpenTelemetry](/docs/opentelemetry/best-practices/opentelemetry-manage-data-ingest-volume/). Esta página aborda estratégias para otimizar o volume de dados para garantir o uso eficiente dos recursos.

## Use a plataforma da New Relic [#platform]

Depois que os dados do OpenTelemetry forem inseridos na [plataforma New Relic](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/), você poderá aproveitar algumas dessas ferramentas de observabilidade:

* [alertas](/docs/alerts/overview/): Seja informado quando um sinal exceder um limite.
* [Consultoria personalizada](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/): Escreva uma consultoria NRQL personalizada para análise e exploração de dados ad hoc.
* [Dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/): crie visualizações de dados personalizadas que refletem os requisitos exclusivos do seu aplicativo.
