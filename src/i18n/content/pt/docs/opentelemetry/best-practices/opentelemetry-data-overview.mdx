OpenTelemetry possui um rico ecossistema de instrumentação e ferramentas que podem ser integrados ao New Relic via [OTLP](https://opentelemetry.io/docs/specs/otlp/). Esta documentação descreve como a New Relic recebe, processa e ingere dados OTLP. Consulte [Introdução ao OpenTelemetry e New Relic](/docs/opentelemetry/get-started/opentelemetry-get-started-intro/) para obter informações sobre padrões de integração comuns.

[New Relic OTLP endpoint](/docs/opentelemetry/best-practices/opentelemetry-otlp/) descreve os requisitos de configuração OpenTelemetry New Relicdo OTLP relevantes para qualquer integração com , juntamente com [informações sobre resolução de problemas .](/docs/opentelemetry/best-practices/opentelemetry-otlp-troubleshooting/)

O OTLP é dividido em sinais trace, métricos e log , todos os quais incluem o conceito de recursos. Consulte as páginas a seguir para obter detalhes sobre como cada um é processado e ingerido:

* [Traces de OpenTelemetry no New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/)
* [Métricas de OpenTelemetry em New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-metrics/)
* [Registro OpenTelemetry no New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-logs/)
* [Recursos OpenTelemetry no New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/)

Para obter detalhes sobre o volume de dados do OpenTelemetry no New Relic e ferramentas de gerenciamento, consulte [Gerenciar o volume de ingestão de dados do OpenTelemetry](/docs/opentelemetry/best-practices/opentelemetry-manage-data-ingest-volume/).

Todos os dados OTLP são inseridos na [plataforma New Relic](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/), que inclui muitas ferramentas poderosas de observabilidade de uso geral para ajudar você a identificar e diagnosticar problemas, incluindo:

* [Alertas](/docs/alerts-applied-intelligence/overview/): Seja informado quando um sinal ultrapassar um limite.
* [Consulta personalizada](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/): escreva uma consulta NRQL personalizada para análise e exploração de dados ad hoc.
* [Dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/): crie visualizações de dados personalizadas que refletem os requisitos exclusivos do seu aplicativo.
