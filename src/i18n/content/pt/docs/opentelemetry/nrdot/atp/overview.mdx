O Adaptive Telemetry Processor (ATP) é um processador inteligente de filtragem de métricas para o Coletor da New Relic Distribution for OpenTelemetry (NRDOT). O ATP monitora dinamicamente as métricas de sistema e processo, filtrando a telemetria de baixo valor com base em limiares configuráveis, enquanto preserva a visibilidade crítica durante eventos de desempenho.

## Por que o ATP é importante

Abordagens tradicionais de monitoramento criam vários desafios:

* Sobrecarga de dados: Dados de telemetria de alto volume de todos os processos aumentam os custos sem valor proporcional
* Lacunas de visibilidade: A filtragem sem estado cria &quot;lacunas de dados&quot; onde os processos aparecem e desaparecem constantemente da telemetria à medida que oscilam em torno dos limites
* Falta de contexto: Processos importantes podem ser perdidos durante picos de recursos breves, mas críticos
* Ajuste manual: Limiares estáticos exigem ajustes constantes conforme as condições do sistema mudam

## O que é ATP

Em vez de filtragem simples baseada em limites, o ATP combina:

* Amostragem baseada em processos: Monitore processos específicos conforme sua necessidade.
* Pontuação composta multimétrica: Combine várias métricas com pesos configuráveis para uma avaliação holística
* Ajuste dinâmico de limites: Adapte automaticamente os limites com base em linhas de base históricas
* Detecção de anomalias: Detecte alterações repentinas nas métricas e preserve dados anômalos
* Gerenciamento de estado persistente: Mantenha o contexto entre reinicializações do coletor

## Como funciona

O ATP processa dados de telemetria por meio de um sistema de filtragem inteligente:

* Identificação de processos: Monitora todos os processos para padrões de uso de recursos
* Avaliação de limiar: Compara o uso atual com limiares configurados ou dinâmicos
* Rastreamento de retenção: Mantém o estado para processos que excedem os limites
* Amostragem inteligente: Aplica lógica avançada, incluindo detecção de anomalias e avaliação multimétrica
* Encaminhamento de dados: Envia apenas dados de telemetria relevantes para a New Relic
