A Distribuição New Relic do OpenTelemetry Collector (NRDOT) é uma distribuição pré-construída do OpenTelemetry [Collector](https://opentelemetry.io/docs/collector/) suportada pela New Relic. O NRDOT é projetado para funcionar perfeitamente com os produtos e serviços da New Relic, e é a maneira recomendada de coletar, processar e exportar dados de telemetria do OpenTelemetry para uso com a New Relic. A New Relic fornece uma única distribuição chamada `nrdot-collector` que se destina a funcionar com inúmeras configurações, cada uma integrando-se à ingestão da New Relic e impulsionando casos de uso específicos de domínio.

Além disso, usuários avançados podem criar distribuições personalizadas com componentes adicionais usando o [OCB](https://opentelemetry.io/docs/collector/custom-collector/) se os componentes [incluídos na NRDOT](https://github.com/newrelic/nrdot-collector-releases/blob/main/distributions/nrdot-collector/manifest.yaml) não atenderem às suas necessidades.

<Callout variant="tip">
  Há cenários em que você pode querer implantar o coletor de uma forma menos opinativa. Recomendamos ler nossa documentação sobre [processamento do coletor](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) para entender como configurar o coletor para seu caso de uso específico.
</Callout>

Antes de se aprofundar em um de nossos casos de uso, recomendamos familiarizar-se com os conceitos básicos do OpenTelemetry Collector para entender como nossas configurações são estruturadas e como personalizá-las de acordo com suas necessidades.

## Implantação [#deployments]

O coletor em si é um único binário que pode ser implantado de várias maneiras. As estratégias típicas de implantação são:

* Como agente: implante no mesmo host do aplicativo.
* Como gateway: implantar como ponto central para coleta de telemetria.

Cada caso de uso do NRDOT vem com uma implantação específica em mente e, embora as configurações possam ser personalizadas para atender às suas necessidades, recomendamos seguir as recomendações na respectiva documentação, a menos que você tenha um forte motivo para divergir delas.

Informações adicionais sobre padrões de implantação podem ser encontradas na [documentação do OpenTelemetry Collector](https://opentelemetry.io/docs/collector/deployment/).

## instalação [#installation]

Para instalar o NRDOT, comece selecionando um [caso de uso](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector#use-cases) e siga as instruções correspondentes.
