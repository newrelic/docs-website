A New Relic Distribution of OpenTelemetry Collector (NRDOT) é uma coleção de distribuições pré-criadas do OpenTelemetry [Collector](https://opentelemetry.io/docs/collector/) suportadas pela New Relic. O NRDOT foi projetado para funcionar perfeitamente com os produtos e serviços da New Relic e é a maneira recomendada de coletar, processar e exportar dados de telemetria abertos para uso com a New Relic. Cada distribuição vem com configurações pré-configuradas para integração com o New Relic Ingest.

Em vez de fornecer uma distribuição única e abrangente do OpenTelemetry Collector, nos concentramos em fornecer distribuições que são:

* Leve
* Limitado em escopo
* Adaptado para experiências específicas do New Relic
* Pré-configurado para reportar ao endpoint OTLP da New Relic
* Suportado

Essa abordagem oferece ao usuário uma experiência simplificada e bem testada, permitindo melhorias mais rápidas em nossas distribuições. Cada distribuição é projetada para um caso de uso específico New Relic , ajudando o usuário a integrar facilmente OpenTelemetry com New Relic. Monitoramos com IA para sobreposição mínima entre distribuições, permitindo que elas trabalhem junto com outros coletores, se necessário. Além disso, usuários avançados podem criar distribuições personalizadas com componentes adicionais usando [OCB](https://opentelemetry.io/docs/collector/custom-collector/).

<Callout variant="tip">
  Há cenários em que você pode querer implantar o coletor de uma forma menos opinativa. Recomendamos ler nossa documentação sobre [processamento do coletor](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) para entender como configurar o coletor para seu caso de uso específico.
</Callout>

Antes de mergulhar em uma de nossas distribuições, recomendamos que você se familiarize com os conceitos básicos do OpenTelemetry Collector para entender como nossas configurações são estruturadas e como personalizá-las de acordo com suas necessidades.

## Implantação [#deployments]

O coletor em si é um único binário que pode ser implantado de várias maneiras. As estratégias típicas de implantação são:

* Como agente: implante no mesmo host do aplicativo.
* Como gateway: implantar como ponto central para coleta de telemetria.

Cada distribuição do NRDOT é configurada imediatamente com uma implantação específica em mente, mas pode ser personalizada para atender às suas necessidades fornecendo configurações personalizadas. Informações adicionais sobre padrões de implantação podem ser encontradas na [documentação do OpenTelemetry Collector](https://opentelemetry.io/docs/collector/deployment/).
