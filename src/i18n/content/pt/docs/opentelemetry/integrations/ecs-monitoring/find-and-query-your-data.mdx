Após configurar o monitoramento do OpenTelemetry para o seu ambiente do Amazon ECS, você pode encontrar e consultar seus dados nas interfaces de infraestrutura e consulta do New Relic. Este guia ajuda você a localizar suas métricas do ECS e criar consultas significativas para monitoramento e resolução de problemas.

## Onde encontrar seus dados do ECS [#find-data]

Seus dados de monitoramento do ECS aparecem em vários locais no New Relic, dependendo do tipo de métrica e do seu tipo de lançamento do ECS.

### Monitoramento de infraestrutura [#infrastructure-view]

Navegue até <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Infrastructure**</DNT> para visualizar sua infraestrutura ECS:

#### Para ECS no EC2

1. Visualização de hosts: filtre por host para ver suas instâncias de EC2 executando contêineres do ECS
2. Visualização de contêineres: veja o desempenho individual do contêiner em suas tarefas do ECS

#### Para ECS Fargate

1. Visualização de contêiner: local principal para métricas de contêiner do Fargate

<Callout variant="tip">
  Use as opções de filtro para restringir a clusters, serviços ou nomes de contêiner específicos para encontrar rapidamente os dados de que você precisa.
</Callout>

### Métricas e eventos [#metrics-events]

Acesse **<DNT>[one.newrelic.com](https://one.newrelic.com)</DNT> &gt; Query your data** para:

* Criar gráficos e dashboards personalizados
* Configure alertas com base em limites específicos de métrica

## Exemplos de consulta [#query-examples]

### Monitorar a utilização de CPU do contêiner

```sql
SELECT average(container.cpu.utilization)
FROM Metric
WHERE ClusterName = 'your-cluster-name'
AND ServiceName = 'your-service-name'
TIMESERIES AUTO
FACET container.name
```

### Acompanhe as tendências de uso de memória

```sql
SELECT max(container.memory.usage.total) / 1024 / 1024 as 'Memory (MB)'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET ServiceName
```

### Monitorar métricas de nível de host (apenas EC2)

```sql
SELECT average(1 - system.cpu.utilization) * 100 AS 'CPU used %',
       average(system.memory.utilization) * 100 AS 'Memory used %'
FROM Metric
WHERE aws.ecs.launchtype = 'ec2'
AND aws.ecs.cluster.arn = 'your-cluster-arn'
TIMESERIES AUTO
```

## Resolução de problemas de lacunas de dados [#troubleshooting]

Se você não vir os dados esperados:

1. **Verifique o status do coletor**: verifique os logs do contêiner do coletor OpenTelemetry
2. **Confirme a conectividade de rede**: garanta o acesso HTTPS de saída para a New Relic
3. **Verifique a chave de licença**: verifique se a chave de licença está configurada corretamente no SSM Parameter Store
4. **Revise a definição da tarefa**: confirme se o contêiner do coletor está em execução com a alocação de recursos adequada
