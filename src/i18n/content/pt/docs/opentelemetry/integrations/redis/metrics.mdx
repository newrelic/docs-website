## Referência completa de métricas

A integração coleta mais de 30 métricas de instância em comandos, memória, keyspace, conexões e replicação — além de 9 métricas específicas de cluster para implantações do Redis Cluster. Expanda as seções abaixo para obter especificações detalhadas das métricas.

<Callout variant="tip">
  Para o catálogo upstream completo, consulte a [documentação do redisreceiver](https://github.com/open-telemetry/opentelemetry-collector-contrib/blob/main/receiver/redisreceiver/documentation.md) do OpenTelemetry.
</Callout>

## Métrica de cluster

Essas métricas estão disponíveis apenas durante o monitoramento do Redis no modo Cluster. Elas exigem a ativação das métricas do cluster no receptor do Redis (todas as métricas do cluster estão desativadas por padrão) ou o uso da abordagem do receptor do Prometheus com `redis_exporter --is-cluster`.

<table>
  <thead>
    <tr>
      <th style={{ width: "260px" }}>
        Métrica
      </th>

      <th>
        Descrição
      </th>

      <th style={{ width: "100px" }}>
        Tipo
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `redis.cluster.state`
      </td>

      <td>
        Status de saúde geral do cluster. 1 = OK (todos os slots servidos), 0 = FAIL (um ou mais slots indisponíveis).
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.slots_assigned`
      </td>

      <td>
        Número de slots de hash mapeados para nós. Deve ser 16.384 para um cluster totalmente operacional.
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.slots_ok`
      </td>

      <td>
        Número de slots que atendem ativamente a requests sem problemas.
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.slots_pfail`
      </td>

      <td>
        Número de slots no estado PFAIL (falha provável) — suspeita-se que um nó esteja inacessível, mas ainda não confirmado.
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.slots_fail`
      </td>

      <td>
        Número de slots em estado FAIL confirmado — NÃO atendendo ao tráfego.
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.known_nodes`
      </td>

      <td>
        Número total de nós (mestres + réplicas) visíveis no cluster.
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.node.count`
      </td>

      <td>
        Número de nós mestre servindo atualmente slots de hash.
      </td>

      <td>
        Medidor
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.stats_messages_sent`
      </td>

      <td>
        Total de mensagens do barramento do cluster enviadas (ping, pong, meet, fail). O tráfego constante indica um protocolo gossip saudável.
      </td>

      <td>
        Cumulativo
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.stats_messages_received`
      </td>

      <td>
        Total de mensagens do barramento do cluster recebidas de nós pares. Uma queda repentina indica uma possível partição de rede.
      </td>

      <td>
        Cumulativo
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Todos os nós em um Cluster do Redis relatam métricas do cluster idênticas (`CLUSTER INFO` retorna os mesmos dados de qualquer nó). Você pode ativar o monitoramento do cluster em todos os nós para resiliência ou em um único nó para minimizar dados redundantes. Para a abordagem do receptor do Prometheus, use `redis_exporter --is-cluster`, que coleta automaticamente essas métricas.
</Callout>
