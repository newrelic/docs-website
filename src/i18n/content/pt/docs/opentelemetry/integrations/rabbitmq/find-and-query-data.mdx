Assim que o monitoramento do RabbitMQ for configurado com o OpenTelemetry, suas métricas aparecerão em vários locais no New Relic. Este guia mostra onde encontrar seus dados e como consultá-los de forma eficaz.

## Navegue para seus dados do RabbitMQ [#find-data]

Suas métricas do RabbitMQ aparecem em vários locais da plataforma New Relic, cada um otimizado para diferentes casos de uso:

### 1. Explorador de entidades

Ideal para: Verificações rápidas de integridade e relacionamentos de entidades

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; On host integrations**</DNT>
2. Visualize entidades RabbitMQ criadas automaticamente a partir de suas métricas

### 2. Painéis pré-construídos

Ideal para: Monitoramento e visualização abrangentes

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards &gt; Recommended dashboards (View all)**</DNT>
2. Pesquisar por <DNT>**&quot;OpenTelemetry RabbitMQ&quot;**</DNT>

### 3. Serviços de terceiros

Ideal para: Monitoramento focado em integração

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; Infrastructure &gt; Third-party services**</DNT>
2. Pesquise por <DNT>**&quot;RabbitMQ&quot;**</DNT> para encontrar rapidamente suas instâncias monitoradas

## Consulte seus dados do RabbitMQ [#query-data]

Todas as métricas do RabbitMQ incluem o atributo `instrumentation.provider = 'opentelemetry'`. Aqui estão algumas consultas úteis:

### Visualizar todas as métricas disponíveis

```sql
SELECT count(*)
FROM Metric
WHERE metricName LIKE 'rabbitmq.%'
  AND instrumentation.provider = 'opentelemetry'
FACET metricName
SINCE 10 minutes ago
```

### Monitorar profundidades de fila

```sql
SELECT latest(rabbitmq.queue.message.count)
FROM Metric
WHERE instrumentation.provider = 'opentelemetry'
FACET rabbitmq.queue.name
TIMESERIES
```

### Acompanhar o lag do consumidor

```sql
SELECT latest(rabbitmq.queue.message.unacknowledged)
FROM Metric
WHERE instrumentation.provider = 'opentelemetry'
  AND rabbitmq.queue.name = 'your-queue-name'
TIMESERIES
```
