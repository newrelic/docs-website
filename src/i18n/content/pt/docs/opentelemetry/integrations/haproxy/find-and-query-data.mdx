Depois de configurar o monitoramento do HAProxy com o OpenTelemetry, seus dados aparecem no New Relic em alguns minutos. Esta página mostra como encontrar suas entidades do HAProxy, explorar métricas e escrever consultas NRQL.

## Encontre seus dados do HAProxy [#find-data]

<CollapserGroup>
  <Collapser id="method1-entities" title="Use o explorador de entidade">
    1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities**</DNT>.
    2. Pesquise suas entidades do HAProxy por nome ou filtre por tipo de entidade `HAPROXYINSTANCE`.
    3. Selecione uma entidade para visualizar a página de resumo com métricas clássicas (sessões por segundo, requests por segundo, erros de conexão).
  </Collapser>

  <Collapser id="method2-dashboard" title="Use Integração & agente">
    1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Integrations &amp; Agents**</DNT>.
    2. Clique em <DNT>**Dashboards**</DNT>.
    3. Pesquise <DNT>**HAProxy (OpenTelemetry Collector)**</DNT> e selecione-o.
    4. Clique em <DNT>**View dashboard**</DNT> para ver visualizações pré-construídas para suas métricas do HAProxy.
  </Collapser>

  <Collapser id="method3-query" title="Use o criador de consulta">
    1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Query your data**</DNT>.
    2. Execute uma consulta para encontrar seus dados do HAProxy:
       ```sql
       FROM Metric SELECT uniques(metricName)
       WHERE metricName LIKE 'haproxy.%'
       SINCE 1 hour ago
       ```
  </Collapser>
</CollapserGroup>

## Consulte seus dados com NRQL [#query-data]

Use estes exemplos de consultas NRQL para analisar seus dados do HAProxy:

```sql
-- Sessions per second across frontends
FROM Metric SELECT average(haproxy.sessions.rate)
WHERE haproxy.service_name = 'FRONTEND'
FACET haproxy.proxy_name
TIMESERIES

-- Error rates by backend
FROM Metric SELECT rate(sum(haproxy.responses.errors), 1 minute) AS 'Errors/min'
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name
TIMESERIES

-- Bytes throughput (in + out)
FROM Metric SELECT rate(sum(haproxy.bytes.input), 1 minute) AS 'Bytes In/min',
  rate(sum(haproxy.bytes.output), 1 minute) AS 'Bytes Out/min'
FACET haproxy.proxy_name
TIMESERIES

-- Backend server health
FROM Metric SELECT latest(haproxy.active) AS 'Active Servers',
  latest(haproxy.backup) AS 'Backup Servers'
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name

-- Request queue depth (indicates backend saturation)
FROM Metric SELECT max(haproxy.requests.queued)
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name
TIMESERIES
```

## Solucionar problemas de visibilidade de dados [#troubleshoot]

<CollapserGroup>
  <Collapser id="troubleshoot-no-data" title="Nenhum dado aparece no New Relic">
    * Aguarde alguns minutos para a ingestão inicial de dados.
    * Verifique o intervalo de tempo — certifique-se de estar visualizando dados recentes (últimos 30 minutos).
    * Consulte quaisquer métricas do HAProxy:
      ```sql
      FROM Metric SELECT count(*) WHERE metricName LIKE 'haproxy.%' SINCE 30 minutes ago
      ```
    * Verifique se o coletor está em execução:
      ```bash
      # Self-hosted
      sudo systemctl status otelcol-contrib
      # or
      sudo systemctl status nrdot-collector

      # Kubernetes
      kubectl get pods -n newrelic
      kubectl logs -n newrelic -l app.kubernetes.io/name=nr-k8s-otel-collector
      ```
    * Verifique se o endpoint de estatísticas do HAProxy está acessível a partir do coletor.
  </Collapser>

  <Collapser id="troubleshoot-intermittent-data" title="Os dados parecem intermitentes">
    * Verifique os logs do coletor em busca de erros:
      ```bash
      # Self-hosted
      sudo journalctl -u otelcol-contrib -n 50 --no-pager

      # Kubernetes
      kubectl logs -n newrelic deploy/nr-k8s-otel-collector-deployment --tail=50
      ```
    * Verifique se o endpoint de estatísticas do HAProxy está consistentemente acessível.
    * Verifique se há problemas de conectividade de rede entre o coletor e o endpoint OTLP da New Relic.
    * Certifique-se de que o HAProxy não esteja limitando a taxa de requests do endpoint de estatísticas.
  </Collapser>
</CollapserGroup>
