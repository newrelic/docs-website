Esta integração do Kubernetes descobre automaticamente os pods HAProxy em seu cluster e coleta métricas sem configuração manual para cada instância. Ele utiliza o [haproxyreceiver](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/haproxyreceiver) e o [receivercreator](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/receivercreator) OpenTelemetry para monitorar dinamicamente as sessões do HAProxy, requests, erros e integridade do servidor backend em seu ambiente em contêineres.

<Callout variant="tip">
  Executando HAProxy em hosts Linux? Consulte o [guia de instalação auto-hospedada](/docs/opentelemetry/integrations/haproxy/self-hosted) para implantações em máquinas virtuais ou bare metal.
</Callout>

## Antes de você começar [#prerequisites]

Certifique-se de ter:

* [Chave de licença](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)da New Relic válida
* Pods do HAProxy com o [endpoint de estatísticas habilitado](https://www.haproxy.com/documentation/haproxy-configuration-manual/latest/#4-stats%20enable) (normalmente na porta 8404)
* Um rótulo em cada pod HAProxy para descoberta (por exemplo, `app: haproxy`)
* Se você estiver usando o método de instalação do Helm, conclua a [instalação base do Kubernetes OpenTelemetry Helm](/docs/kubernetes-pixie/k8s-otel/install/#helm-install)

        <Collapser id="nrdot-k8s-helm-install" title="Instalação via Helm">
          1. Adicione as seguintes seções sob [`deployment.extraConfig`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/values.yaml) no seu `values.yaml`:

          <Callout variant="important">
            Em `deployment.extraConfig`, `pipelines:` deve ser definido no nível raiz — não aninhado sob `service:`. O modelo do Helm chart mapeia `extraConfig.pipelines` para `service.pipelines`, e `extraConfig.service.extensions` para `service.extensions`.
          </Callout>

          Extensões:

          ```yaml
          extensions:
            k8s_observer:
              auth_type: serviceAccount
              observe_pods: true
              observe_nodes: false
          ```

          Variáveis de ambiente (adicione em `deployment.envs`):

          ```yaml
          deployment:
            envs:
              - name: OTEL_RESOURCE_ATTRIBUTES
                value: "host.id=$(KUBE_NODE_NAME)"
          ```

          <Callout variant="important">
            O atributo de recurso `host.id` é necessário para a síntese de entidade. As imagens de contêiner não têm `/etc/machine-id`, então você deve injetá-lo via `OTEL_RESOURCE_ATTRIBUTES`. O chart do Helm já define `KUBE_NODE_NAME` a partir da Downward API do Kubernetes.
          </Callout>

          Receptores:

          ```yaml
          receivers:
            receiver_creator/haproxy:
              watch_observers: [k8s_observer]
              receivers:
                haproxy:
                  rule: type == "pod" && labels["app"] == "haproxy"
                  config:
                    endpoint: 'http://`endpoint`:8404/stats'
                    collection_interval: 30s
          ```

          Processadores:

          ```yaml
          processors:
            resource/cluster:
              attributes:
                - key: k8s.cluster.name
                  value: "<YOUR_CLUSTER_NAME>"
                  action: insert

            transform/haproxy:
              metric_statements:
                - context: resource
                  statements:
                    - set(attributes["haproxy.display.name"], Concat([
                        "server",
                        "k8s",
                        attributes["k8s.cluster.name"],
                        attributes["k8s.namespace.name"],
                        "pod",
                        attributes["k8s.pod.name"],
                        "haproxy",
                        "8404"
                      ], ":"))

            transform/metadata_nullify:
              metric_statements:
                - context: metric
                  statements:
                    - set(description, "")
                    - set(unit, "")
          ```

          Extensões de serviço:

          ```yaml
          service:
            extensions: [k8s_observer]
          ```

          Pipelines:

          ```yaml
          pipelines:
            metrics/haproxy:
              receivers: [receiver_creator/haproxy]
              processors: [batch, resource/cluster, transform/haproxy, transform/metadata_nullify, memory_limiter]
              exporters: [otlphttp/newrelic]
          ```

        <Collapser id="nrdot-k8s-manifest-install" title="Instalação via manifesto">
          Após concluir a [instalação base do manifesto do OpenTelemetry para Kubernetes](/docs/kubernetes-pixie/k8s-otel/install/#manifest-install), configure o monitoramento do HAProxy:

          1. Atualize a imagem do coletor para usar o coletor NRDOT.

             No arquivo [`deployment.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/deployment.yaml) no seu diretório local `rendered`, atualize a imagem para:

             ```yaml
             image: newrelic/nrdot-collector:latest
             ```

          2. Monte o `/etc/machine-id` do host no contêiner do coletor adicionando-os à especificação de implantação:

             ```yaml
             volumeMounts:
               - name: machine-id
                 mountPath: /etc/machine-id
                 readOnly: true
             volumes:
               - name: machine-id
                 hostPath:
                   path: /etc/machine-id
                   type: File
             ```

             <Callout variant="important">
               O atributo de recurso `host.id` é necessário para a síntese da entidade. O processador `resourcedetection` lê `/etc/machine-id` para preenchê-lo. Sem essa montagem de volume, `host.id` não será definido e as entidades não serão criadas.
             </Callout>

          3. Substitua o conteúdo em [`deployment-configmap.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/deployment-configmap.yaml) em `deployment-config.yaml: |` por:

             ```yaml
             extensions:
               health_check:
                 endpoint: 0.0.0.0:13133
               k8s_observer:
                 auth_type: serviceAccount
                 observe_pods: true
                 observe_nodes: false

             receivers:
               receiver_creator/haproxy:
                 watch_observers: [k8s_observer]
                 receivers:
                   haproxy:
                     rule: type == "pod" && labels["app"] == "haproxy"
                     config:
                       endpoint: 'http://`endpoint`:8404/stats'
                       collection_interval: 30s

             processors:
               resourcedetection:
                 detectors: [env, system]
                 system:
                   resource_attributes:
                     host.name:
                       enabled: true
                     host.id:
                       enabled: true
                     os.type:
                       enabled: true

               batch:
                 send_batch_max_size: 1000
                 timeout: 30s
                 send_batch_size: 800

               memory_limiter:
                 check_interval: 1s
                 limit_percentage: 80
                 spike_limit_percentage: 25

               resource/cluster:
                 attributes:
                   - key: k8s.cluster.name
                     value: "<YOUR_CLUSTER_NAME>"
                     action: insert

               transform/haproxy:
                 metric_statements:
                   - context: resource
                     statements:
                       - set(attributes["haproxy.display.name"], Concat([
                           "server",
                           "k8s",
                           attributes["k8s.cluster.name"],
                           attributes["k8s.namespace.name"],
                           "pod",
                           attributes["k8s.pod.name"],
                           "haproxy",
                           "8404"
                         ], ":"))

               transform/metadata_nullify:
                 metric_statements:
                   - context: metric
                     statements:
                       - set(description, "")
                       - set(unit, "")

             exporters:
               otlp_http/newrelic:
                 endpoint: "<YOUR_NEWRELIC_OTLP_ENDPOINT>"
                 headers:
                   api-key: ${env:NR_LICENSE_KEY}

             service:
               extensions: [health_check, k8s_observer]
               pipelines:
                 metrics/haproxy:
                   receivers: [receiver_creator/haproxy]
                   processors: [resourcedetection, batch, resource/cluster, transform/haproxy, transform/metadata_nullify, memory_limiter]
                   exporters: [otlp_http/newrelic]
             ```

          4. Exclua o DaemonSet e a implantação do kube-state-metrics (não são necessários para o monitoramento apenas do HAProxy):

             ```bash
             kubectl delete daemonset nr-k8s-otel-collector-daemonset -n newrelic
             kubectl delete deployment nr-k8s-otel-collector-kube-state-metrics -n newrelic
             ```

          5. Reinicie a implantação do coletor para aplicar suas alterações:

             ```bash
             kubectl rollout restart deployment nr-k8s-otel-collector-deployment -n newrelic
             ```

        <Collapser id="otelcontrib-helm-install" title="Instalação via Helm">
          1. Crie um arquivo `values.yaml` com a seguinte configuração de monitoramento do HAProxy:

             ```yaml

                 - apiGroups: [""]

                   verbs: ["get", "list", "watch"]
                 - apiGroups: ["apps"]
                   resources: ["replicasets"]
                   verbs: ["get", "list", "watch"]

             extraVolumes:
               - name: machine-id
                 hostPath:
                   path: /etc/machine-id
                   type: File

             extraVolumeMounts:
               - name: machine-id
                 mountPath: /etc/machine-id
                 readOnly: true

             config:
               extensions:
                 health_check:
                   endpoint: ${env:MY_POD_IP}:13133
                 k8s_observer:
                   auth_type: serviceAccount
                   observe_pods: true
                   observe_nodes: false

               receivers:
                 receiver_creator/haproxy:
                   watch_observers: [k8s_observer]

                   timeout: 30s

                   check_interval: 1s
             ```

        <Collapser id="otelcontrib-manifest-install" title="Instalação via manifesto">
          Use esta opção se você não puder usar o Helm ou preferir gerenciar os recursos do Kubernetes diretamente com `kubectl`.

          1. Crie um segredo com sua chave de licença da New Relic:

             ```bash
             kubectl create namespace newrelic
             kubectl create secret generic otel-collector-haproxy-secret \
               --namespace newrelic \
               --from-literal=licenseKey='<YOUR_NEW_RELIC_LICENSE_KEY>'
             ```

          2. Salve o seguinte manifesto em um arquivo (por exemplo, `otel-collector-haproxy.yaml`):

             ```yaml
             ---
             apiVersion: v1
             kind: ServiceAccount
             metadata:
               name: otel-collector-haproxy
               namespace: newrelic
               labels:
                 app: otel-collector-haproxy
             ---
             apiVersion: rbac.authorization.k8s.io/v1
             kind: ClusterRole
             metadata:
               name: otel-collector-haproxy
               labels:
                 app: otel-collector-haproxy
             rules:
               - apiGroups: [""]
                 resources: ["pods", "nodes"]
                 verbs: ["get", "list", "watch"]
               - apiGroups: ["apps"]
                 resources: ["replicasets"]
                 verbs: ["get", "list", "watch"]
             ---
             apiVersion: rbac.authorization.k8s.io/v1
             kind: ClusterRoleBinding
             metadata:
               name: otel-collector-haproxy
               labels:
                 app: otel-collector-haproxy
             roleRef:
               apiGroup: rbac.authorization.k8s.io
               kind: ClusterRole
               name: otel-collector-haproxy
             subjects:
               - kind: ServiceAccount
                 name: otel-collector-haproxy
                 namespace: newrelic
             ---
             apiVersion: v1
             kind: ConfigMap
             metadata:
               name: otel-collector-haproxy-config
               namespace: newrelic
               labels:
                 app: otel-collector-haproxy
             data:
               collector-config.yaml: |
                 extensions:
                   health_check:
                     endpoint: 0.0.0.0:13133
                   k8s_observer:
                     auth_type: serviceAccount
                     observe_pods: true
                     observe_nodes: false

                 receivers:
                   receiver_creator/haproxy:
                     watch_observers: [k8s_observer]
                     receivers:
                       haproxy:
                         rule: type == "pod" && labels["app"] == "haproxy"
                         config:
                           endpoint: 'http://`endpoint`:8404/stats'
                           collection_interval: 30s

                 processors:
                   resourcedetection:
                     detectors: [env, system]
                     system:
                       resource_attributes:
                         host.name:
                           enabled: true
                         host.id:
                           enabled: true
                         os.type:
                           enabled: true

                   batch:
                     send_batch_size: 1024
                     timeout: 30s

                   memory_limiter:
                     check_interval: 1s
                     limit_percentage: 80
                     spike_limit_percentage: 25

                   resource/cluster:
                     attributes:
                       - key: k8s.cluster.name
                         value: "<YOUR_CLUSTER_NAME>"
                         action: insert

                   transform/haproxy:
                     metric_statements:
                       - context: resource
                         statements:
                           - set(attributes["haproxy.display.name"], Concat(["server", "k8s", attributes["k8s.cluster.name"], attributes["k8s.namespace.name"], "pod", attributes["k8s.pod.name"], "haproxy", "8404"], ":"))

                   transform/metadata_nullify:
                     metric_statements:
                       - context: metric
                         statements:
                           - set(description, "")
                           - set(unit, "")

                 exporters:
                   otlp_http/newrelic:
                     endpoint: "<YOUR_NEWRELIC_OTLP_ENDPOINT>"
                     headers:
                       api-key: ${env:NR_LICENSE_KEY}

                 service:
                   extensions: [health_check, k8s_observer]
                   pipelines:
                     metrics/haproxy:
                       receivers: [receiver_creator/haproxy]
                       processors: [resourcedetection, batch, resource/cluster, transform/haproxy, transform/metadata_nullify, memory_limiter]
                       exporters: [otlp_http/newrelic]
             ---
             apiVersion: apps/v1
             kind: Deployment
             metadata:
               name: otel-collector-haproxy
               namespace: newrelic
               labels:
                 app: otel-collector-haproxy
             spec:
               replicas: 1
               selector:
                 matchLabels:
                   app: otel-collector-haproxy
               template:
                 metadata:
                   labels:
                     app: otel-collector-haproxy
                 spec:
                   serviceAccountName: otel-collector-haproxy
                   containers:
                   - name: otel-collector
                     image: otel/opentelemetry-collector-contrib:latest
                     command:
                     - "/otelcol-contrib"
                     - "--config=/conf/collector-config.yaml"
                     env:
                     - name: NR_LICENSE_KEY
                       valueFrom:
                         secretKeyRef:
                           name: otel-collector-haproxy-secret
                           key: licenseKey
                     - name: GOGC
                       value: "80"
                     ports:
                     - name: health
                       containerPort: 13133
                       protocol: TCP
                     resources:
                       limits:
                         cpu: 500m
                         memory: 512Mi
                       requests:
                         cpu: 100m
                         memory: 256Mi
                     volumeMounts:
                     - name: config
                       mountPath: /conf
                     - name: machine-id
                       mountPath: /etc/machine-id
                       readOnly: true
                   volumes:
                   - name: config
                     configMap:
                       name: otel-collector-haproxy-config
                       items:
                       - key: collector-config.yaml
                         path: collector-config.yaml
                   - name: machine-id
                     hostPath:
                       path: /etc/machine-id
                       type: File
             ```

             <Callout variant="important">
               A montagem de volume `/etc/machine-id` é necessária para a síntese de entidade. O processador `resourcedetection` lê este arquivo para preencher o atributo de recurso `host.id`. Sem isso, as entidades não serão criadas no New Relic.
             </Callout>

          3. Aplique o manifesto:

             ```bash
             kubectl apply -f otel-collector-haproxy.yaml
             ```

          4. Verifique se o coletor está em execução e descobrindo os pods do HAProxy:

             ```bash
             kubectl get pods -n newrelic -l app=otel-collector-haproxy
             kubectl logs -n newrelic -l app=otel-collector-haproxy
             ```

             Você deve ver linhas de log mostrando `starting receiver` com os IPs do seu pod do HAProxy.

## Visualize seus dados no New Relic [#find-data]

1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities**</DNT>.

## Resolução de problemas [#troubleshooting]

<CollapserGroup>
  <Collapser id="no-data" title="Nenhum dado aparece no New Relic">
    * Verifique se os pods do HAProxy têm o rótulo correto (por exemplo, `app: haproxy`): `kubectl get pods -l app=haproxy`
    * Verifique se o endpoint de estatísticas responde: `kubectl exec -it <haproxy-pod> -- curl -s http://localhost:8404/stats`
    * Revise os logs do coletor em busca de erros de descoberta: `kubectl logs -n newrelic -l app.kubernetes.io/name=nr-k8s-otel-collector -f` (NRDOT) ou `kubectl logs -n newrelic -l app=otel-collector-haproxy -f` (manifesto do OTel Contrib)
    * Certifique-se de que o ServiceAccount do coletor tenha permissões de `get`, `list` e `watch` nos pods.
  </Collapser>

  <Collapser id="single-entity" title="Múltiplos pods HAProxy são mostrados como uma entidade">
    Cada pod do HAProxy deve criar uma entidade separada no New Relic. Se você vir apenas uma entidade:

    * Verifique se o coletor tem `host.id` definido. Em ambientes em contêineres, adicione isso à implantação do coletor:
      ```yaml
      env:
        - name: K8S_NODE_NAME
          valueFrom:
            fieldRef:
              fieldPath: spec.nodeName
        - name: OTEL_RESOURCE_ATTRIBUTES
          value: "host.id=$(K8S_NODE_NAME)"
      ```
    * Confirme diferentes valores de `haproxy.addr` por pod consultando: `FROM Metric SELECT uniques(haproxy.addr) WHERE metricName LIKE 'haproxy.%' SINCE 10 minutes ago`
  </Collapser>

  <Collapser id="stats-unreachable" title="O Collector não consegue alcançar o endpoint de estatísticas do HAProxy">
    * O `receiver_creator` usa o IP do pod (não `localhost`) para alcançar o HAProxy. Certifique-se de que o frontend de estatísticas esteja vinculado a `*:8404` (todas as interfaces), não a `127.0.0.1:8404`.
    * Verifique as políticas de rede que podem bloquear o tráfego entre o pod do coletor e os pods do HAProxy.
  </Collapser>
</CollapserGroup>
