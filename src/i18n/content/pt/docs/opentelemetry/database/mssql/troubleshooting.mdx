Solucionar problemas comuns que você pode encontrar ao monitorar seu Microsoft SQL Server com o coletor NRDOT.

## Problemas comuns

<CollapserGroup>
  <Collapser id="network-connectivity" title="Problemas de conectividade de rede">
    Se você encontrar problemas de conectividade de rede com o NRDOT Collector, execute os seguintes comandos para testar a conectividade com sua instância do SQL Server na porta 1433:

    * Para Windows, execute:

      ```shell
      Test-NetConnection -ComputerName <YOUR_IP_ADDRESS> -Port <YOUR_PORT_NUMBER>
      ```

    * Para Linux, execute:

      ```bash
      nc -zv <YOUR_IP_ADDRESS> <YOUR_PORT_NUMBER>
      ```
  </Collapser>

  <Collapser id="ssl" title="Problemas de conexão SSL">
    Para se conectar ao Microsoft SQL Server usando SSL sem validação do certificado, certifique-se de que o parâmetro `enable_ssl` esteja definido como `true` e o parâmetro `trust_server_certificate` esteja definido como `true` na configuração do seu NRDOT Collector.

    ```yaml
    enable_ssl: true                                                                                                                            
    trust_server_certificate: true
    ```
  </Collapser>
</CollapserGroup>
