        <Step>
          ### Configurar o Coletor NRDOT [#setup-sql]

          Instale o NRDOT Collector em uma instância do EC2 do Windows ou em uma máquina local do Windows que possa se conectar à sua instância do RDS SQL Server.

          <Callout variant="important">
            Para o monitoramento do RDS, instale o NRDOT Collector em uma instância de EC2 na mesma VPC que a sua instância do RDS para obter conectividade de rede ideal e menor latência.
          </Callout>

          1. Baixe e instale o NRDOT Collector usando o PowerShell:

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

          2. Certifique-se de que seu grupo de segurança permita conexões de entrada do host do NRDOT Collector para sua instância do RDS na porta 1433 (ou sua porta personalizada).
        </Step>

        <Step>
          ### Configurar usuário do banco de dados [#configure-user-sql]

          Conecte-se à sua instância do RDS SQL Server usando o SQL Server Management Studio (SSMS) ou sqlcmd com suas credenciais de administrador.

          Execute o seguinte script para criar o usuário de monitoramento `newrelic` com as permissões apropriadas:

          ```sql
          USE [master];
          GO
          CREATE LOGIN [newrelic] WITH PASSWORD = '<PASSWORD>';
          GO

          GRANT VIEW SERVER STATE TO [newrelic];
          GRANT VIEW ANY DEFINITION TO [newrelic];
          GRANT VIEW ANY DATABASE TO [newrelic];
          GO

          DECLARE @name SYSNAME;
          DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
          SELECT [name]
          FROM [master].[sys].[databases]
          WHERE [name] NOT IN ('master', 'msdb', 'model', 'rdsadmin', 'distribution')
          AND [state] = 0;
          OPEN db_cursor;
          FETCH NEXT FROM db_cursor INTO @name;
          WHILE @@FETCH_STATUS = 0
          BEGIN
            BEGIN TRY
              EXEC('USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''newrelic'')
              BEGIN
                CREATE USER [newrelic] FOR LOGIN [newrelic];
              END;
              GRANT VIEW DATABASE STATE TO [newrelic];');
            END TRY
            BEGIN CATCH
              PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
            END CATCH
            FETCH NEXT FROM db_cursor INTO @name;
          END
          CLOSE db_cursor;
          DEALLOCATE db_cursor;
          GO
          ```

          <Callout variant="tip">
            **Para RDS:** Use seu nome de usuário mestre do RDS (ou um usuário com a função `rds_superuser`) para executar estes comandos. O script exclui automaticamente o banco de dados `rdsadmin`, que é reservado pela AWS.
          </Callout>
        </Step>

          ### Parâmetro de configuração

          A tabela a seguir descreve os principais parâmetros de configuração para o receptor nrsqlserver:

          | Parâmetro              | Descrição                                                                                                                                                                      |
          | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
          | `<YOUR_USERNAME>`      | Insira seu nome de usuário do banco de dados para autenticação                                                                                                                 |
          | `<YOUR_PASSWORD>`      | Insira a senha do seu banco de dados para autenticação                                                                                                                         |
          | `<YOUR_SERVER>`        | Insira o nome do host ou endereço IP do SQL Server                                                                                                                             |
          | `<YOUR_PORT>`          | Insira o número da porta do SQL Server. O valor padrão é 1433.                                                                                                                 |
          | `<YOUR_OTLP_ENDPOINT>` | Insira o endpoint OTLP da New Relic. Para mais informações, consulte a [documentação dos endpoints OTLP da New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/). |
          | `<YOUR_LICENSE_KEY>`   | Insira sua chave de licença do New Relic.                                                                                                                                      |
          | `collection_interval`  | Intervalo em segundos para coletar métricas de banco de dados. Padrão: 15s                                                                                                     |

          <Callout variant="tip">
            **Nota:** a opção `max_concurrent_queries` controla quantas consultas de métrica o receiver executa em paralelo no SQL Server durante cada ciclo de scrape.

            ```yaml
            receivers:
              nrsqlserver:
                max_concurrent_queries: 4  # default is 4 if omitted or set to 0
            ```
          </Callout>

          <Callout variant="tip">
            **Observação:** se você habilitar todas as métricas disponíveis, aumente o valor de `limit_mib` (por exemplo, 1.000) para evitar que o limitador de memória descarte dados.
          </Callout>

        <Step>
          ### Atualizar Registro de Serviço do Windows [#update-registry-sql]

          Atualize o Registro de Serviço do Windows para usar o novo arquivo mssql-config.yaml.

          **Método 1: PowerShell**

          Execute o seguinte comando no PowerShell como administrador:

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **Método 2: prompt de comando (alternativa)**

          Se o PowerShell não estiver disponível, você pode usar o prompt de comando em vez disso:

          1. Abra o menu Iniciar, digite `cmd`
          2. Clique com o botão direito do mouse no prompt de comando e escolha **Run as administrator**
          3. Copie e cole este comando exato na janela preta do CMD e pressione Enter:

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
        </Step>

        <Step>
          ### Habilite seu Microsoft SQL Server [#enable-sql-server-domain]

          Conecte-se à sua instância do RDS SQL Server usando o SQL Server Management Studio (SSMS) ou sqlcmd com suas credenciais de administrador.

          Execute o seguinte script para criar um login de domínio do Windows e conceder as permissões necessárias:

          ```sql
          USE master;
          CREATE LOGIN [DomainName\DomainUser] FROM WINDOWS;
          GRANT VIEW ANY DATABASE   TO [DomainName\DomainUser];
          GRANT VIEW SERVER STATE TO [DomainName\DomainUser];
          GRANT VIEW ANY DEFINITION TO [DomainName\DomainUser];
          GO
          ```

          <Callout variant="important">
            Substitua `DomainName\DomainUser` pelo seu domínio e nome de usuário reais (por exemplo, `CORP\SQLMonitor`).
          </Callout>

        <Step>
          ### Conceda privilégios de acesso de LEITURA [#grant-permissions-domain]

          Use as seguintes instruções para conceder privilégios de acesso READ ao usuário do domínio em todos os bancos de dados:

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
                AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''DomainName\DomainUser'')
                BEGIN
                  CREATE USER [DomainName\DomainUser] FOR LOGIN [DomainName\DomainUser];
                END;
                GRANT VIEW DATABASE STATE TO [DomainName\DomainUser];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="tip">
            **Para RDS:** O script exclui automaticamente o banco de dados `rdsadmin` que é reservado pela AWS. Certifique-se de substituir `DomainName\DomainUser` pelas suas credenciais de domínio reais.
          </Callout>
        </Step>

        <Step>
          ### Configurar o NRDOT Collector [#setup-domain]

          Instale o NRDOT Collector em uma instância de EC2 do Windows ingressada no domínio ou em uma máquina local do Windows que faça parte do mesmo domínio e possa se conectar à sua instância do SQL Server do RDS.

          1. Baixe e instale o NRDOT Collector usando o PowerShell:

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

             ```powershell
             # Configure service to use domain account
             # The space after obj= and password= is required syntax for sc.exe
             sc.exe config "nrdot-collector" obj= "DomainName\DomainUser" password= "<password>"

             # Restart the service
             Restart-Service nrdot-collector
             ```

             Verifique a identidade do serviço:

             ```powershell
             Get-WmiObject Win32_Service -Filter "Name='nrdot-collector'" | Select Name, State, StartName

             ```

             <Callout variant="important">
               Substitua `DomainName\DomainUser` pelo seu domínio e nome de usuário reais (por exemplo, `CORP\SQLMonitor`) e `<password>` pela senha do usuário do domínio.
             </Callout>

          3. Certifique-se de que seu grupo de segurança permita conexões de entrada do host do NRDOT Collector para sua instância do RDS na porta 1433 (ou sua porta personalizada).

          **Requisitos adicionais para gMSA (Group Managed Service Account) com RDS:**

          * Ambiente do Active Directory com suporte a gMSA

          * Instância do EC2 do Windows ingressada no domínio autorizada a usar a conta gMSA
          * Grupo de segurança configurado para permitir conexões na porta do SQL Server (padrão 1433)
        </Step>

        <Step>
          ### Habilite seu Microsoft SQL Server [#enable-sql-server-gmsa]

          Conecte-se à sua instância do RDS SQL Server e execute os seguintes comandos SQL para criar um login para a conta gMSA e conceder as permissões necessárias:

          ```sql
            USE master;
            CREATE LOGIN [DomainName\gMSA] FROM WINDOWS;
            GRANT VIEW ANY DATABASE TO [DomainName\gMSA];
            GRANT VIEW SERVER STATE TO [DomainName\gMSA];
            GRANT VIEW ANY DEFINITION TO [DomainName\gMSA];
          ```

          <Callout variant="important">
            Substitua `DomainName\gMSA` pelo seu nome de domínio e nome de conta gMSA reais (por exemplo, `CORP\SQL-gMSA$`). Observe que as contas gMSA geralmente terminam com um símbolo `$`.
          </Callout>
        </Step>

        <Step>
          ### Conceda privilégios de acesso de LEITURA [#grant-read-access-rds-gmsa]

          Use as seguintes instruções para conceder privilégios de acesso READ à conta gMSA em todos os bancos de dados:

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
              AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''<DOMAIN>\<gMSA_NAME>$'')
                BEGIN
                  CREATE USER [<DOMAIN>\<gMSA_NAME>$] FOR LOGIN [<DOMAIN>\<gMSA_NAME>$];
                END;
                GRANT VIEW DATABASE STATE TO [<DOMAIN>\<gMSA_NAME>$];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="important">
            Substitua `DomainName\gMSA` pelo seu nome de domínio real e nome da conta gMSA (por exemplo, `CORP\SQL-gMSA$`). Este script cria um usuário de banco de dados para a conta gMSA em todos os bancos de dados de usuários, excluindo bancos de dados do sistema e bancos de dados específicos do RDS.
          </Callout>
        </Step>

        <Step>
          ### Configurar o NRDOT Collector [#setup-gmsa]

          Instale o NRDOT Collector em uma instância do Windows no EC2 ingressada no domínio e autorizada a usar a conta gMSA.

          <Callout variant="important">
            Para RDS com gMSA, o NRDOT Collector **deve** ser instalado em uma instância de EC2 do Windows associada ao domínio que tenha permissões para usar a conta gMSA. Certifique-se de que seu grupo de segurança permita conexões com o RDS na porta 1433.

          ### Parâmetro de configuração

          A tabela a seguir descreve os principais parâmetros de configuração para o receptor nrsqlserver:

          | Parâmetro              | Descrição                                                                                                                                                                      |
          | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
          | `<YOUR_RDS_ENDPOINT>`  | Insira o endpoint do seu RDS SQL Server                                                                                                                                        |
          | `<YOUR_PORT>`          | Insira o número da porta do SQL Server. O valor padrão é 1433.                                                                                                                 |
          | `<YOUR_OTLP_ENDPOINT>` | Insira o endpoint OTLP da New Relic. Para mais informações, consulte a [documentação dos endpoints OTLP da New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/). |
          | `<YOUR_LICENSE_KEY>`   | Insira sua chave de licença do New Relic.                                                                                                                                      |
          | `collection_interval`  | Intervalo em segundos para coletar métricas de banco de dados. Padrão: 15s                                                                                                     |

          <Callout variant="tip">
            A opção `max_concurrent_queries` controla quantas consultas de métrica o receiver executa em paralelo no SQL Server durante cada ciclo de scrape.

            ```yaml
            receivers:
              nrsqlserver:
                max_concurrent_queries: 4  # default is 4 if omitted or set to 0
            ```
          </Callout>

          ### Atualizar Registro de Serviço do Windows [#update-registry-gmsa]

          Atualize o Registro de Serviço do Windows para usar o novo arquivo mssql-config.yaml.

          **Método 1: PowerShell**

          Execute o seguinte comando no PowerShell como administrador:

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **Método 2: prompt de comando (alternativa)**

          Se o PowerShell não estiver disponível, você pode usar o prompt de comando em vez disso:

          1. Abra o menu Iniciar, digite `cmd`
          2. Clique com o botão direito do mouse no prompt de comando e escolha **Run as administrator**
          3. Copie e cole este comando exato na janela preta do CMD e pressione Enter:

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
