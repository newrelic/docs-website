Você pode instalar o NRDOT Collector para o monitoramento do Microsoft SQL Server usando a instalação guiada da plataforma New Relic, que fornece uma configuração guiada pela plataforma por meio da interface da New Relic. Esta é a abordagem recomendada para inícios rápidos.

<Steps>
  <Step>
    ### Pré-requisitos [#prerequisites]

    Certifique-se de que seu ambiente atenda à nossa [compatibilidade e pré-requisitos](/docs/opentelemetry/database/mssql/compatibility).
  </Step>

  <Step>
    ### Instalação guiada [#guided]

    Para instalar o NRDOT Collector usando a instalação guiada:

    1. Entre na sua [conta New Relic](https://one.newrelic.com/).

    2. Vá para <DNT>**left navigation pane &gt; + Integration &amp; Agents &gt; MSSQL (OpenTelemetry)**</DNT>.

    3. Na tela <DNT>Select an account</DNT>, selecione a conta na qual deseja instalar a integração MSSQL e clique em <DNT>**Continue**</DNT>.

    4. Na tela <DNT>Enter your credential</DNT>, selecione uma das seguintes opções e clique em **Continue**:

       * **Use uma chave existente**: se você já possui uma chave de licença, forneça a chave de licença. Para mais informações, consulte [Chaves de usuário](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
       * **Criar uma nova chave**: se você não tiver uma chave de licença, clique em **Create a new key** para criar uma.

    5. Na tela <DNT>Select platform</DNT>, selecione a plataforma para instalar a integração MSSQL e clique em <DNT>**Continue**</DNT>.

    6. Siga as instruções na tela para configurar a integração para a plataforma selecionada.

    7. Após a instalação, na tela <DNT>Test your integration</DNT>, clique em **Testar conexão** para verificar se o NRDOT Collector consegue se conectar à sua instância do SQL Server e enviar dados para o New Relic.
  </Step>

  <Step>
    ### (Opcional) Configurar a correlação entre APM e banco de dados [#correlation]

    Para correlacionar o desempenho da sua aplicação com as operações de banco de dados, você pode configurar a identificação de serviço de banco de dados. Este recurso permite visualizar exatamente quais aplicativos estão gerando workloads específicas de banco de dados. Para mais informações, consulte [configurar a identificação de serviço de banco de dados](/docs/opentelemetry/database/database-service-identification/) para obter a correlação APM-banco de dados no New Relic.

    <Callout variant="important">
      Para visualizar dados de desempenho do banco de dados no APM, ambas as entidades devem estar na mesma conta New Relic. Se as entidades estiverem em contas diferentes da New Relic, você deve ter acesso a ambas as contas para visualizar os dados.
    </Callout>
  </Step>

  <Step>
    ### (Opcional) Configurar o gerenciamento de segredos [#rds-secret-management]

    O recurso de gerenciamento de segredos permite gerenciar com segurança informações confidenciais, como credenciais de banco de dados. Isso ajuda a aprimorar a segurança do seu ambiente de monitoramento ao evitar a inserção direta de dados sensíveis nos arquivos de configuração. Para mais informações, consulte [gerenciamento de segredos](/docs/opentelemetry/database/database-service-identification/#secret-management).
  </Step>

  <Step>
    ### Encontre e use seus dados [#find]

    Assim que seus dados estiverem sendo coletados, você poderá acessar o monitoramento abrangente do banco de dados SQL Server por meio da interface do New Relic.

    Para encontrar sua entidade de banco de dados SQL Server no New Relic:

    1. Vá para <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>.
    2. Na dropdown **<DNT>Entity type</DNT>**, selecione <DNT>**MSSQL instance**</DNT> e clique em <DNT>**Apply**</DNT>.
    3. Selecione seu banco de dados SQL Server na lista de entidades.
  </Step>
</Steps>
