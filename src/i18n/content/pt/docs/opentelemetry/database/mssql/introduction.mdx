Monitore o desempenho do banco de dados Microsoft SQL Server usando a New Relic Distribution of OpenTelemetry (NRDOT) com insights abrangentes sobre métricas de banco de dados, desempenho de consultas e saúde do sistema.

Essa abordagem baseada em NRDOT complementa nossa Integração On-Host (OHI) existente, utilizando padrões OpenTelemetry para monitoramento de banco de dados, facilitando a integração com sua stack de observabilidade existente.

## Como você pode configurar o NRDOT para SQL Server [#configure]

<Steps>
  <Step>
    ### Pré-requisitos [#prerequisites]

    Certifique-se de que seu ambiente atenda à nossa [compatibilidade e pré-requisitos](/docs/opentelemetry/database/mssql/compatibility).
  </Step>

  <Step>
    ### Instalação do NRDOT Collector [#installation]

    Escolha seu caminho de instalação com base em sua infraestrutura:

    * **Instalação guiada**: configuração guiada pela plataforma por meio da interface da New Relic (recomendado para inícios rápidos)

    * **Ambientes auto-hospedados**: instalação manual em sua própria infraestrutura

      * **[Windows auto-hospedado](/docs/opentelemetry/database/mssql/windows-hosted)**
      * **[Linux auto-hospedado](/docs/opentelemetry/database/mssql/linux-hosted)**

    * **Ambientes AWS RDS**: monitoramento do SQL Server gerenciado na nuvem

      * **[Monitoramento do Windows RDS](/docs/opentelemetry/database/mssql/windows-rds)**
      * **[Monitoramento do Linux RDS](/docs/opentelemetry/database/mssql/linux-rds)**
  </Step>

  <Step>
    ### Encontre e use seus dados [#find]

    Assim que seus dados estiverem sendo coletados, você poderá acessar o monitoramento abrangente do banco de dados SQL Server por meio da interface do New Relic.

    Para encontrar sua entidade de banco de dados SQL Server no New Relic:

    1. Vá para <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>.
    2. Na dropdown **<DNT>Entity type</DNT>**, selecione <DNT>**MSSQL instance**</DNT> e clique em <DNT>**Apply**</DNT>.
    3. Selecione seu banco de dados SQL Server na lista de entidades.
  </Step>
</Steps>
