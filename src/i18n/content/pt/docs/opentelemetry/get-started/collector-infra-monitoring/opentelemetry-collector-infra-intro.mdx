O [coletorOpenTelemetry ](https://opentelemetry.io/docs/collector/)é uma ferramenta independente de fornecedor para receber, processar e exportar dados de telemetria. Embora os requisitos e a configuração do coletor variem, ele vem com uma variedade de receptores e processadores que o tornam popular para monitoramento de infraestrutura. Também é comum usar o coletor para processamento de dados, mas esta documentação foca em casos de uso de monitoramento de infraestrutura. Consulte [ColetorOpenTelemetry para processamento de dados](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) para obter mais informações.

## Correlacionando infraestrutura com OpenTelemetry APM [#infrastructure-correlation]

O exemplo a seguir demonstra o uso do coletor para correlacionar [APM OpenTelemetry](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro) com dados de infraestrutura:

* [Exemplo de repositório de hosts](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/host-monitoring)

O padrão geral é configurar o coletor com um processador que detecta e enriquece a telemetria APM com contexto de ambiente adicional na forma de atributo de recurso, antes de exportar os dados para New Relic via OTLP. New Relic detecta esses dados de correlação e constrói relacionamentos entre APM e a infraestrutura da entidade por meio [de recursos](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources).

## OpenTelemetry Collector para monitoramento do Kubernetes [#k8s-monitoring]

Embora os requisitos e a configuração do Collector variem, ele vem com uma variedade de receptores e processadores que o tornam popular para monitoramento do Kubernetes. Para obter toda a telemetria necessária, é preciso extrair várias APIs Kubernetes usando vários receptores com diferentes requisitos de permissão. Portanto, para simplificar as coisas para você, criamos um gráfico do Helm que usa nossa distribuição personalizada do coletor [nrdot-coletor-k8s](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s) nos bastidores. Para mais detalhes sobre como começar, consulte [OpenTelemetry para Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/).
