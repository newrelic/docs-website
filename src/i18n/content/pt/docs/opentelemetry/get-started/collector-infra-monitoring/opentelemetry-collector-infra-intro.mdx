O [coletor OpenTelemetry](https://opentelemetry.io/docs/collector/) é uma ferramenta independente de fornecedor para receber, processar e exportar dados de telemetria. Embora os requisitos e a configuração do coletor variem, ele vem com uma variedade de receptores e processadores que o tornam popular para monitoramento de infraestrutura. Também é comum usar o coletor para processamento de dados, mas esta documentação se concentra em casos de uso de monitoramento de infraestrutura. Consulte [ColetorOpenTelemetry para processamento de dados](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) para obter mais informações.

## Recebendo telemetria de infraestrutura [#infrastructure-receivers]

Os exemplos a seguir demonstram o uso do coletor para monitor diversos componentes de infraestrutura. O padrão geral é configurar o coletor com um receptor de tecnologia específica ou de uso geral (ou seja, prometheus) para extrair ou pesquisar dados de um destino de infraestrutura e exportar os dados para o New Relic via OTLP.

* [HCP Consul](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/hcp-consul)
* [HiveMQ](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/hivemq)
* [Anfitriões](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/host-monitoring)
* [Nuvem Confluente Kafka](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/confluentcloud)
* [Prometeu](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/prometheus)
* [Redis](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/redis)
* [Singlestore](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/singlestore)
* [Squid](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/squid)
* [EstatísticasD](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/statsd)

## Correlacionando infraestrutura com OpenTelemetry APM [#infrastructure-correlation]

Os exemplos a seguir demonstram o uso do coletor para correlacionar [dados do APM OpenTelemetry](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro) com a infraestrutura. O padrão geral é configurar o coletor com um processador que detecte e enriqueça a telemetria APM com contexto de ambiente adicional na forma de atributo de recursos, antes de exportar os dados para o New Relic via OTLP. New Relic detecta esses dados de correlação e constrói [relacionamentos entre APM e a entidade de infraestrutura](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources).
