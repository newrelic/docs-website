O projeto OpenTelemetry tem um escopo amplo incluindo [especificaçõesAPI e SDK](https://opentelemetry.io/docs/specs/otel/), [instrumentação](https://opentelemetry.io/docs/concepts/instrumentation/), [coletor](https://opentelemetry.io/docs/collector/), [convenções semânticas](https://opentelemetry.io/docs/specs/semconv/), protocolo [OTLP](https://opentelemetry.io/docs/specs/otlp/) e muito mais. New Relic AI Monitoring terá suporte de primeira classe para OpenTelemetry e possui uma ampla variedade de recursos de plataforma que facilitam o uso de ferramentas OpenTelemetry .

Esta documentação é uma coleção de padrões comuns de integração do OpenTelemetry com o New Relic. Não é um conjunto exaustivo de todos os usos possíveis, mas deve dar uma ideia das maneiras como a maioria das pessoas usa o OpenTelemetry e os tipos de recursos da plataforma New Relic criados com base nesses fluxos de trabalho.

[OpenTelemetry APM monitoramento](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro) discute o uso do OpenTelemetry para realizar monitoramento APM semelhante ao [monitoramentoAPM com agente New Relic ](/docs/apm/new-relic-apm/getting-started/introduction-apm/).

[OpenTelemetry coletor para monitoramento de infraestrutura](/docs/opentelemetry/get-started/collector-infra-monitoring/opentelemetry-collector-infra-intro) contém uma variedade de exemplos de uso do OpenTelemetry coletor para monitoramento de infraestrutura, incluindo scraping/polling infraestrutura destino e correlação de dados OpenTelemetry APM com infraestrutura.

[OpenTelemetry coletor para processamento de dados](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) discute os fundamentos do uso do OpenTelemetry coletor como uma ferramenta geral para receber, processar (enriquecer, transformar, amostrar, filtrar, etc) e exportar dados de telemetria.
