O [OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) é uma ferramenta independente de fornecedor para receber, processar e exportar dados de telemetria. Embora os requisitos e a configuração do Collector variem, ele vem com uma variedade de receptores e processadores que o tornam popular para monitoramento do Kubernetes. Para obter toda a telemetria necessária, é preciso extrair várias APIs Kubernetes usando vários receptores com diferentes requisitos de permissão. Portanto, para simplificar as coisas para você, criamos um gráfico do Helm que usa nossa distribuição personalizada do coletor [nrdot-coletor-k8s](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s) nos bastidores. Para mais detalhes sobre como começar, consulte [OpenTelemetry para Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/).
