Se você decidiu criar um aplicativo personalizado ou modificar um de nossos aplicativos de código aberto, precisará de algumas ferramentas essenciais:

* A [interface de linha de comando (CLI) do New Relic](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==).
* Uma chave de API, que você obtém ao fazer download da CLI.

Dependendo do que você deseja fazer com seu aplicativo, você pode ter algumas configurações e configurações adicionais. Este guia cobre:

* Baixando o New Relic CLI para construir ou modificar aplicativos
* Contribua com conteúdo para este site

## Antes de você começar

Antes de começar, recomendamos primeiro ler sobre [permissões](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps).

Para começar a construir, você deve ter:

* Uma [conta no github](https://github.com/) . Embora não seja estritamente necessário para criar aplicativos, uma conta GitHub permite que você [baixe e personalize nossos aplicativos de código aberto](https://opensource.newrelic.com/explore-projects) e contribua com um [projeto de código aberto](https://opensource.newrelic.com/contributing/).
* Uma [conta de desenvolvedor New Relic ](https://newrelic.com/signup/?partner=Developer+Edition). Caso ainda não tenha uma, você pode obter uma conta de teste grátis para desenvolvimento do aplicativo New Relic .
* [npm](https://www.npmjs.com/get-npm). Se você instalou [o Node.js](https://nodejs.org/en/download/), então você já tem `npm`, que é usado para compartilhar, reutilizar e atualizar o código JavaScript. Isso é necessário para trabalhar com componentes React que são a framework dos aplicativos New Relic e deste site.

### Uma nota sobre suporte

Construir um aplicativo New Relic é o mesmo que construir qualquer aplicativo JavaScript/React. Oferecemos suporte para ajudar com nossas ferramentas de construção ([nossa CLI](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==) e [biblioteca SDK)](/docs/new-relic-solutions/build-nr-ui/sdk-component/intro-to-sdk). No entanto, não oferecemos suporte para dúvidas ou problemas básicos de codificação em JavaScript ou React.

Para perguntas e respostas comuns sobre construção, consulte a [página do Explorers Hub sobre construção na New Relic](https://discuss.newrelic.com/c/build-on-new-relic).

## Prepare-se para criar ou modificar aplicativos

<Steps>
  <Step>
    ### Baixe a CLI e a chave da API

    Na [página do aplicativo Build New Relic ](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==), conclua as etapas de **Quick start** para fazer download.

    Essas seis etapas de início rápido fornecem uma chave de API para uso no desenvolvimento de aplicativos e a CLI do New Relic para criar e implantar aplicativos. No final do Início rápido, você terá um projeto composto pelo seguinte:

    * Um **Nerdpack**. O pacote que contém todos os arquivos exigidos pelo seu aplicativo. Ele contém dois tipos de arquivos que você personaliza para criar seu aplicativo: Nerdlets e o inicializador.
    * Um ou mais arquivos **Nerdlet** . Uma visualização ou janela de interface específica. Um Nerdlet é um pacote React JavaScript que inclui um arquivo `index.js` , um `stylesheet` e um arquivo `config` no formato JSON. Ele pode conter qualquer funcionalidade JS (gráficos, campos interativos, dicas de ferramentas, etc.).
    * Um arquivo `launcher` : esta é a base para o inicializador, que é usado para abrir seu aplicativo no New Relic depois de publicá-lo.
  </Step>

  <Step>
    ### Comece a construir

    Se você estiver pronto para programar, `cd` acesse seu Nerdpack e comece.

    Se você quiser saber mais sobre como criar aplicativos, experimente estes guias passo a passo:

    * [Construa um aplicativo &quot;Olá, mundo!&quot;](/docs/new-relic-solutions/tutorials/build-hello-world-app) mostra como criar um pequeno aplicativo, publicá-lo no New Relic e compartilhá-lo com outras pessoas assinando contas nele.
    * [O mapa de visualizações de página por região](/docs/new-relic-solutions/tutorials/map-pageviews-region/) orienta você nas etapas para criar um dos nossos populares aplicativos de código aberto. Você aprende a adicionar uma consulta personalizada a um aplicativo e visualizá-la em uma tabela e, em seguida, adicionar esses dados a um mapa.
  </Step>
</Steps>

## Contribua com o site `developer.newrelic.com`

Este site é de código aberto e gostaríamos de receber sua opinião. Crie uma pull request se encontrar um erro que você sabe como corrigir. Deixe-nos um problema no GitHub se você encontrar algumas lacunas de conteúdo nas quais deseja que trabalhemos. Ou escreva um guia totalmente novo, se tiver um que gostaria de compartilhar. Continue lendo para saber como.

<Steps>
  <Step>
    ### Bifurque o repositório `developer-website` GitHub

    A bifurcação do repositório permite que você trabalhe em sua própria cópia dos arquivos `developer.newrelic.com` e [crie o site localmente](https://github.com/newrelic/developer-website#developernewreliccom). Ele também nos permite gerenciar com mais facilidade as solicitações pull recebidas.

    1. Vá para a página [do site dos desenvolvedores](https://github.com/newrelic/developer-website) no GitHub.
    2. Selecione o botão **Fork** no canto superior direito da página.
    3. Escolha a conta para a qual deseja fazer o fork e aguarde alguns segundos enquanto o fork é criado.

    [Sincronize regularmente](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/syncing-a-fork) para manter seu fork atualizado com alterações e adições ao upstream do branch principal.
  </Step>

  <Step>
    ### Faça uma solicitação de recurso ou documentação

    1. Em qualquer página, selecione o botão **GitHub** na parte superior da página.
    2. Selecione o tipo de alteração desejada.
    3. Preencha o formulário GitHub.
  </Step>

  <Step>
    ### Contribua com um novo guia

    Confira nossas [diretrizes para contribuidores](https://github.com/newrelic/developer-website/blob/master/CONTRIBUTING.md#creating-new-pages), que orientarão você durante o processo.
  </Step>
</Steps>
