Na maioria das vezes, os comandos da CLI `nr1` simplesmente funcionarão, sem exigir nenhuma configuração extra. Você poderá criar Nerdpacks, Nerdlets, lançadores e visualizações sem problemas. Mas às vezes você precisa de algo especial para sua configuração.

Neste guia, você aprenderá como habilitar a configuração avançada para seu Nerdpack.

## Antes de você começar

Certifique-se de ter a versão mais recente da CLI `nr1` :

```sh
nr1 update
```

## Estenda a configuração do seu webpack

Você pode usar nosso arquivo `.extended-webpackrc.js` para estender seu webpack para configuração. Isso permite que você adicione seus próprios aliases, script e configuração ao servir ou implantar localmente seu aplicativo.

## Adicione certificados extras

Ao trabalhar com `nr1` atrás de um proxy corporativo, você poderá ver um `RequestError`:

```sh
nr1 create --type nerdpack --name new-nerdpack
[output] {error}RequestError: Error:{normal} self signed certificate in certificate chain
```

O Node usa sua própria cadeia de certificados. Portanto, se estiver recebendo este erro, você precisará adicionar seu certificado corporativo às CAs "raiz" do Node:

```sh
export NODE_EXTRA_CA_CERTS=[cert]
```
