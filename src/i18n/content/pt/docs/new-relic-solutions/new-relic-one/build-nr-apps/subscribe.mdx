Assine um Nerdpack para usá-lo na plataforma New Relic. Cancele a assinatura dos Nerdpacks que você não usa mais.

## Atualize sua CLI

Antes de executar qualquer comando, certifique-se de ter a versão mais recente da CLI:

```shell
nr1 update
```

## Verifique suas permissões

Para assinar um Nerdpack, você precisa de:

* Acesso à conta que o publicou
* As [permissões necessárias para gerenciar Nerdpacks](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps)

## Publique seu Nerdpack

Você precisa publicar os Nerdpacks criados antes de assiná-los:

1. [Publique seu Nerdpack](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish)
2. [Tag seu Nerdpack](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) se ele for publicado e ainda não tiver tag

## Assine um Nerdpack

Assine um Nerdpack publicado, com a interface da Web ou a CLI `nr1` , para usá-lo em sua conta. A assinatura permite que você selecione a versão da tag que deseja usar, bem como a conta com a qual deseja se inscrever. Isso permite que você controle quais Nerdpacks você usa em sua conta.

<Callout variant="tip">
  Quando a tag que você assinou é atualizada, o New Relic atualiza seu Nerdpack automaticamente.
</Callout>

<Steps>
  <Step>
    Na página inicial do [New Relic](https://one.newrelic.com) , vá para <DNT>**+ Integrations & Agents**</DNT> na barra de navegação superior.
  </Step>

  <Step>
    Encontre seu Nerdpack publicado usando o filtro de aplicativos ou a barra de pesquisa. Todos os Nerdpacks publicados são mostrados aqui. No entanto, aquele que você está procurando pode ter sido filtrado. Ajuste o filtro para encontrar o Nerdpack que você procura.
  </Step>

  <Step>
    Selecione o Nerdpack que deseja adicionar à sua conta New Relic.
  </Step>

  <Step>
    Clique em **Add this app**. Este botão mostra **Manage access** se você já estiver inscrito no Nerdpack.
  </Step>

  <Step>
    Selecione a conta para assinar e a versão da tag para assinar. Em seguida, atualize sua(s) conta(s).

    <Callout variant="tip">
      Na interface da web e em alguns locais da CLI, nos referimos à **tags** como **channels**. Esta é uma relíquia de uma implementação anterior de versionamento do Nerdpack. Ignorando detalhes históricos e de implementação, considere-os sinônimos.
    </Callout>

    Agora você e os membros das contas com as quais se inscreveu podem lançar o Nerdpack da New Relic.
  </Step>

  <Step>
    Se você assinou um aplicativo, encontre-o em **Your apps**. Se você assinou uma visualização, encontre-a em **Custom visualizations**.
  </Step>
</Steps>

## Inscreva-se na linha de comando

Se você publicou seu próprio Nerdpack, pode ser mais rápido assiná-lo com a CLI `nr1` .

<Steps>
  <Step>
    Navegue até o diretório raiz do seu Nerdpack:

    ```shell
    cd my-awesome-nerdpack
    ```

    Estas instruções referem-se ao seu Nerdpack como `my-awesome-nerdpack`.
  </Step>

  <Step>
    Certifique-se de que seu Nerdpack foi publicado e marque:

    ```shell
    nr1 nerdpack:info
    [output] Id: {purple}ab123c45-678d-9012-efg3-45hi6jkl7890
    [output] Region: {muted}us
    [output] Account ID: 1234567
    [output] Local version: 1.0.0
    [output] Subscription Model: OWNER_AND_ALLOWED
    [output] Version Count: 1
    [output]
    [output] Version  Date               Tags
    [output] -------  -----------------  ------
    [output] 1.0.0    {muted}a few seconds ago  {green}STABLE
    ```

    Você deverá ver pelo menos uma versão [publicada](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish) com uma [tag](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag). Não precisa ser `STABLE`. Você pode assinar qualquer versão da tag.
  </Step>

  <Step>
    Assine seu Nerdpack:

    ```shell
    nr1 nerdpack:subscribe --channel=STABLE
    [output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}STABLE{normal} channel
    ```

    Aqui, você se inscreveu na tag `STABLE` . Você também pode assinar as tags `DEV` e `BETA` na CLI, mas o comando assinatura não permite tags arbitrárias. Para assinar qualquer outra versão da tag, você precisa usar a interface web.

    Semelhante à interface web, o termo **channel** `nr1 nerdpack:subscribe` é uma referência a uma implementação mais antiga das versões do Nerdpack.

    <Callout
      variant="tip"
      title="Detalhe técnico"
    >
      `nr1 nerdpack:subscribe` é um alias para `nr1 subscription:set`.

      Leia a documentação do [`nr1 subscription:set`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionset) para saber mais. Ou veja a página de ajuda `nr1` :

      ```shell
      nr1 subscription:set --help
      ```
    </Callout>
  </Step>
</Steps>

## Cancelar assinatura de um Nerdpack

Cancele a assinatura de um Nerdpack para removê-lo da sua conta.

<Steps>
  <Step>
    No [New Relic](https://one.newrelic.com), vá para <DNT>**Apps**</DNT> na barra de navegação superior.
  </Step>

  <Step>
    Clique no ícone “i” ao lado do Nerdpack que você assinou. Isso abre a página de detalhes.
  </Step>

  <Step>
    Clique em **Manage access**.
  </Step>

  <Step>
    Desmarque as contas que você deseja cancelar. Em seguida, atualize essas contas.
  </Step>
</Steps>

## Cancelar inscrição usando a linha de comando

Cancele rapidamente a assinatura do seu perfil CLI do Nerdpacks.

<Steps>
  <Step>
    Navegue até o diretório raiz do seu Nerdpack:

    ```shell
    cd my-awesome-nerdpack
    ```
  </Step>

  <Step>
    Veja a assinatura do seu perfil:

    ```shell
    nr1 subscription:list
    [output] {muted}Listing subscribed deployed nerdpacks on your account:
    [output] Version  UUID                                  Name              Date         User
    [output] -------  ------------------------------------  ----------------  -----------  -------
    [output] 0.6.4    {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal}  Account Maturity  {muted}a month ago{normal}  2774472
    ```
  </Step>

  <Step>
    Cancele a assinatura do Nerdpack com o UUID:

    ```shell
    nr1 nerdpack:unsubscribe --nerdpack-id=384de6e5-8e57-4d56-9d13-299ee7ae8641
    [output] Unsubscribed account {green}1234567{normal} from the nerdpack {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal} on the {green}STABLE{normal} channel.
    ```

    <Callout variant="tip">
      `nr1 nerdpack:unsubscribe` é um alias para `nr1 subscription:unset`.

      Leia a documentação do [`nr1 subscription:unset`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionunset) para saber mais. Ou veja a página de ajuda `nr1` :

      ```sh
      nr1 subscription:unset --help
      ```
    </Callout>
  </Step>
</Steps>

## Resolva problemas de acesso ao seu Nerdpack publicado

Às vezes, ao assinar um Nerdpack, você pode ter problemas. Se você não consegue assinar ou vê assinatura duplicada em sua conta, considere algumas soluções comuns para resolver esses problemas.

### Tag sua versão do Nerdpack

Ao assinar um Nerdpack, você escolhe uma tag específica. Portanto, se o Nerdpack que você está tentando assinar não estiver marcado ou não tiver a tag que você está almejando, [tag](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) sua versão de lançamento:

```shell
nr1 nerdpack:subscribe --channel=DEV
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}DEV{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
[output]
nr1 nerdpack:tag --tag=DEV
[output] {success}✔  Tagged {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} version {green}1.0.0{normal} as {green}DEV{normal}.
[output]
nr1 nerdpack:subscribe --channel=DEV
[output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}DEV{normal} channel.
```

Se você se inscrever sem um argumento `--channel` , o canal padrão será `STABLE`, portanto, certifique-se de ter uma versão `STABLE` neste caso:

```shell
nr1 nerdpack:subscribe
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}STABLE{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
```

### Assine com a interface da web

Ao se inscrever na CLI, você passa um argumento `--channel` para selecionar uma tag. Um **channel** é, para todos os efeitos, uma **tag**. Este termo é uma relíquia de uma implementação anterior de versionamento do Nerdpack.

No entanto, embora os termos sejam sinônimos, o uso de `--channel` é limitado na CLI. Ao assinar com um `--channel`, você só pode escolher entre `DEV`, `BETA` e `STABLE`. Então, para assinar uma tag diferente, use a interface web:

```shell
nr1 nerdpack:subscribe --channel=QA
[output] {error}›{normal}   Error: Expected --channel=QA to be one of: {green}DEV{normal}, {green}BETA{normal}, {green}STABLE
[output] {error}›{normal}   See more help with --help
[output] {error}›{normal}   Code: UNKNOWN
```
