Este guia inclui algumas idéias sobre como melhorar o tempo de operação da web.

## 1. Investigue as verificações sintéticas

Investigue e resolva falhas nas verificações sintéticas quando elas ocorrerem. Dicas sobre como fazer isso:

* Verifique vários locais.
* Configure um alerta para notificá-lo quando vários locais falharem.

## 2. Crie carga de trabalho

Crie cargas de trabalho que correlacionem o monitor Sintético com aplicativos, serviços e infraestrutura do Browser. Inclua um link para a workload no runbook de alerta. Você também pode encontrar a workload correlacionada para um Sintético usando a pesquisa global.

Certifique-se de que <InlinePopover type="alerts"/>estão configurados para cada nível da sua workload. Desta forma você pode ver a saúde de cada um dos níveis em uma única visualização. Isso economizará seu tempo na resolução de problemas. Você não precisa criar notificação de alerta para cada nível para se beneficiar desta visualização.

## 3. Investigue primeiro as interrupções no nível mais baixo

Quando ocorrer uma interrupção, comece a investigar no nível mais baixo que está alertando. Por exemplo, se você perceber que tem um problema de infraestrutura e um problema de JavaScript, investigue a infraestrutura antes do JavaScript, a menos que você tenha uma segunda pessoa ou equipe para quem possa delegar isso.

Utilize as ferramentas que estão à sua disposição para resolução de problemas:

* Certifique-se de que distributed tracing esteja ativado para

  <InlinePopover type="browser"/>

  e também para

  <InlinePopover type="apm"/>

  .

* Use a interface do Browser para entender o que está acontecendo no nível do usuário final.

* Use o Lookout para ajudá-lo a entender o que está causando a oscilação ou problemas recorrentes.
