Entenda os requisitos para gerenciar e usar Nerdpacks no New Relic.

## Visão geral das capacidades [#capabilities]

Sua capacidade de gerenciar e usar Nerdpacks é afetada pelo seu usuário New Relic :

* Modelo do usuário (modelo original ou mais recente)
* Tipo de usuário
* Funções atribuídas

Esteja você no [modelo de usuário original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) ou [no modelo de usuário mais recente](/docs/accounts/accounts-billing/new-relic-one-user-management/new-relic-one-user-model-understand-user-structure/), revise as tabelas abaixo para obter um resumo dos recursos do seu Nerdpack.

## Permissões do Nerdpack: modelo de usuário mais recente [#nerdpack-permissions]

Aprenda as diferenças entre como o usuário básico, o usuário de plataforma completa e o usuário principal em [nosso modelo de usuário mais recente](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) podem usar e gerenciar Nerdpacks.

### Usuário básico [#basic-users-new-users]

Se você é um usuário básico em nosso modelo de usuário mais recente, seus recursos do Nerdpack estão limitados. Você só pode criar e servir Nerdpacks localmente. Para publicar, tag, assinar e usar seus Nerdpacks, um administrador deve atualizá-lo para um usuário de plataforma completo ou usuário principal.

<Callout variant="tip">
  Se você for um usuário básico, geralmente não poderá usar nenhum Nerdpacks. No entanto, existem alguns Nerdlets mantidos pela New Relic que você pode usar. Eles são raros e atualmente não há como distingui-los em nossa interface web. E mesmo que você possa usar um Nerdlet em um Nerdpack, você nunca poderá usar visualizações personalizadas. Para isso, você deve estar atualizado para um usuário completo da plataforma ou um usuário principal.
</Callout>

### Usuário principal e usuário completo da plataforma [#core-full-platform-users-new]

Se você é um usuário principal ou um usuário completo da plataforma em nosso modelo de usuário mais recente, você tem o privilégio de **Nerdpacks "modify"** ou não.

O privilégio **Nerdpacks "modify"** é necessário para publicar, assinar e tag Nerdpacks. Portanto, se você não tiver o privilégio de **Nerdpacks "modify"** , você só poderá criar e servir Nerdpacks localmente e usar Nerdpacks nos quais suas contas já foram assinadas.

Para publicar, tag ou assinar seu Nerdpack, um administrador deve conceder a você o recurso de **Nerdpacks "modify"** .

<Callout variant="tip">
  A capacidade de **Nerdpacks "modify"** está incluída nos grupos **User** e **Admin** , os únicos grupos disponíveis por padrão. Portanto, na maioria dos casos, você poderá gerenciar Nerdpacks como um usuário completo da plataforma ou como usuário principal. No entanto, se você for um usuário completo da plataforma ou um usuário principal atribuído a um grupo personalizado que não inclui o privilégio de **Nerdpacks "modify"** , você não poderá gerenciar Nerdpacks.
</Callout>

## Permissões do Nerdpack: modelo de usuário original [#nerdpack-permissions-original]

Aprenda as diferenças entre como usuário básico, usuário completo, usuário principal e administradores em nosso [modelo de usuário original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/) podem usar e gerenciar Nerdpacks.

### Usuário básico [#basic-users-original]

Se você é um usuário básico em nosso modelo de usuário original, seus recursos do Nerdpack estão limitados. Você só pode criar e servir Nerdpacks localmente. Para publicar, tag, assinar e usar seus Nerdpacks, um administrador deve atualizá-lo para um usuário completo.

<Callout variant="tip">
  Se você for um usuário básico, geralmente não poderá usar nenhum Nerdpacks. No entanto, existem alguns Nerdlets mantidos pela New Relic que você pode usar. Eles são raros e atualmente não há como distingui-los em nossa interface web. E mesmo que você possa usar um Nerdlet em um Nerdpack, você nunca poderá usar visualizações personalizadas. Para isso, você deve estar atualizado para um usuário da plataforma principal ou completa.
</Callout>

### Usuário principal e usuário completo da plataforma [#core-full-platform-users-original]

Se você é um usuário principal ou completo da plataforma em nosso modelo de usuário original, você tem a função complementar **Nerdpack Manager** ou não.

A função **Nerdpack Manager** é necessária para publicar, assinar e tag Nerdpacks. Portanto, se você não tem a função **Nerdpack Manager** , você só pode criar e servir Nerdpacks localmente e usar Nerdpacks que suas contas já tenham assinado.

Para publicar, tag ou assinar um Nerdpack, um administrador deve atualizá-lo para um usuário principal ou usuário de plataforma completo ou conceder a você a função complementar **Nerdpack Manager** .

### Proprietário/usuário administrador [#owner-admin-users]

Se você for proprietário ou usuário administrador, poderá executar qualquer um dos recursos do Nerdpack. Você pode criar, veicular, publicar, tag, assinar e usar todo e qualquer item do Nerdpack, tanto Nerdlets quanto visualizações. Isso inclui Nerdpacks criados pela New Relic ou por uma de suas contas.

## Capacidades de conta pai/filho [#parent-child]

Algumas organizações New Relic possuem [estruturas de contas pai/filho](/docs/accounts/original-accounts-billing/original-users-roles/parent-child-account-structure). Para essas organizações, as contas dos filhos reportam os dados às contas dos pais. Esta hierarquia organizacional afeta os recursos do Nerdpack.
