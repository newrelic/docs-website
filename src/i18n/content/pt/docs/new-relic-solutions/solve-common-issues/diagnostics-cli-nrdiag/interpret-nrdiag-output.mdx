Depois de executar a CLI de diagnóstico em seu terminal, você verá os resultados de cada tarefa à medida que forem concluídas. As tarefas que resultam em um código de status `Warning` ou `Failure` log detalhes adicionais sobre possíveis problemas encontrados durante a execução, juntamente com sugestões de resolução de problemas e links relevantes para documentação. Se você usar o flag [-attach ou -chave de API](/docs/new-relic-solutions/solve-common-issues/diagnostics-cli-nrdiag/pass-command-line-options-nrdiag) , poderá visualizar seus resultados na [interface](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/interpret-nrdiag-output/#diag-in-nr1)

## Saída de arquivo

A CLI de diagnóstico gera três arquivos:

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        Arquivos de saída de diagnóstico
      </th>

      <th>
        Comentários
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `nrdiag-output.zip`
      </td>

      <td>
        Uma estrutura de pastas simplificada com um ou mais arquivos de configuração e qualquer log existente do New Relic. Anexe este arquivo à sua conta New Relic. Este arquivo também contém uma cópia de `nrdiag-output.json`.
      </td>
    </tr>

    <tr>
      <td>
        `nrdiag-output.json`
      </td>

      <td>
        Saída dos testes individuais. Anexe esta saída à sua conta New Relic.
      </td>
    </tr>

    <tr>
      <td>
        `nrdiag-filelist.txt`
      </td>

      <td>
        Uma lista de arquivos encontrados. O arquivo `nrdiag-output.zip` que você anexa à sua conta New Relic inclui automaticamente esta lista. Você não precisa anexar este arquivo `.txt` separadamente.
      </td>
    </tr>
  </tbody>
</table>

## Códigos de status de resultado [#status-codes]

A CLI de diagnóstico retorna os seguintes códigos de status após a execução:

<table>
  <thead>
    <tr>
      <th style={{ width: "125px" }}>
        Código de status
      </th>

      <th>
        Definição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `Success`
      </td>

      <td>
        Tarefa executada com sucesso sem problemas detectados.
      </td>
    </tr>

    <tr>
      <td>
        `Warning`
      </td>

      <td>
        Tarefa executada com sucesso. No entanto, possíveis problemas detectados.
      </td>
    </tr>

    <tr>
      <td>
        `Failure`
      </td>

      <td>
        A tarefa falhou com problemas detectados.
      </td>
    </tr>

    <tr>
      <td>
        `Error`
      </td>

      <td>
        Tarefa incapaz de ser executada. Isso pode ser devido a problemas de permissão.
      </td>
    </tr>

    <tr>
      <td>
        `None`
      </td>

      <td>
        A tarefa foi considerada irrelevante para o ambiente detectado e não foi executada.
      </td>
    </tr>
  </tbody>
</table>

## Saída CLI de diagnóstico no New Relic [#diag-in-nr1]

<img
  title="Diagnostics CLI Output application in New Relic"
  alt="Diagnostics CLI Output application in New Relic"
  src="/images/solutions_screenshot-crop_nerdlet-diag.webp"
/>

Você pode examinar a saída `nrdiag-output.json` na interface. A conta selecionada está associada à chave de licença validada pela CLI de diagnóstico ou pela chave de API que você forneceu para upload.

1. Clique no ponto de interrogação no canto superior direito [da interface](https://one.newrelic.com).

2. Selecione a guia

   <DNT>
     **Diagnostics Home**
   </DNT>

   .

3. Na seção Saída da CLI de diagnóstico, clique em

   <DNT>
     **View Run Details**
   </DNT>

   .

4. Isso o levará à sua última corrida. Você pode selecionar execuções anteriores no lado esquerdo.

5. Para cada execução, você verá os status de cada tarefa, com avisos e falhas como foco principal. Você também pode ver qualquer saída de log ou script encontrada pela CLI.
