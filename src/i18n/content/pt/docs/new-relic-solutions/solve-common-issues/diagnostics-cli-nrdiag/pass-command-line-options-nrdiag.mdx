Para usar as seguintes opções de linha de comando com a CLI de diagnóstico:

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Opção
      </th>

      <th>
        Uso
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `-a`

        `-attach`
      </td>

      <td>
        Anexe para upload automático para uma conta New Relic. O tamanho máximo de upload é de 4 GB. Isso usa um <InlinePopover type="licenseKey"/>validado do seu ambiente.
      </td>
    </tr>

    <tr>
      <td>
        `-api-key`
      </td>

      <td>
        Chave de usuário da API a ser usada ao enviar resultados para sua conta New relic. O tamanho máximo de upload é de 4 GB. Isso ignora o uso de uma chave de licença e usa seu <InlinePopover type="userKey"/>para anexar resultados.
      </td>
    </tr>

    <tr>
      <td>
        `-r`

        `-region`
      </td>

      <td>
        A [região](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) em que sua conta New Relic está hospedada. Valores aceitos: UE ou EUA. Não diferencia maiúsculas de minúsculas. Padrão: EUA
      </td>
    </tr>

    <tr>
      <td id="browser-check">
        `-browser-url STRING`
      </td>

      <td>
        CLI de diagnóstico versão 1.1.9 ou mais alto

        Quando invocado, executará apenas verificações de diagnóstico relacionadas a <InlinePopover type="browser"/>. Este comando verifica se o agente de monitoramento de Browser do New Relic está presente e retorna a versão do agente, o método de injeção ([via <InlinePopover type="apm"/>](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app)ou [via copiar/colar](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app)) e o tipo de carregador (Pro, Lite, SPA). Para ser usado para fornecer detalhes ao suporte da New Relic em caso de resolução de problemas em sites da intranet.
      </td>
    </tr>

    <tr>
      <td>
        `-c STRING`

        `-config-file STRING`
      </td>

      <td>
        Substitua o local padrão do arquivo de configuração do agente. Pode ser usado para especificar uma pasta a ser pesquisada além das pastas padrão ou um caminho para um arquivo de configuração específico.
      </td>
    </tr>

    <tr>
      <td>
        `-filter STRING`
      </td>

      <td>
        Filtre os resultados da tarefa por status de resultado. Aceita uma lista separada por vírgulas. Aceita `Success`, `Warning`, `Failure`, `Error`, `None` ou `Info`. Sintaxe de exemplo:

        ```
        "Success,Warning,Failure"
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-h`

        `-help`
      </td>

      <td>
        Exibir lista completa de opções de linha de comando. Para listar todas as tarefas a serem executadas, use `-h tasks`. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-include STRING`
      </td>

      <td>
        Inclua um arquivo ou diretório (incluindo subdiretórios) em `nrdiag-output.zip`. Limite de 4 GB. Use o sinalizador `-a` ou `-api-key` para fazer upload dos resultados para o New Relic.

        Sintaxe de exemplo:

        ```
        -include ./path/to/file
        ```

        ```
        -include \some\path\
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-interactive`
      </td>

      <td>
        Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-output-path STRING`
      </td>

      <td>
        Especifica um diretório de saída diferente para gravar os resultados de `nrdiag-output.zip`, `nrdiag-output.json` e `nrdiag-filelist.txt`. O local padrão é `./`.
      </td>
    </tr>

    <tr>
      <td>
        `-o`

        `-override STRING`
      </td>

      <td>
        Passe argumentos para substituir quando solicitado pelo suporte da New Relic. Formato: `identifier.property=value`.

        Sintaxe de exemplo:

        ```
        -override Java/Config/Agent.Status=Success
        ```

        ```
        -o Base/Config/Validate.agentlanguage=PHP
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-p STRING`

        `-proxy STRING`
      </td>

      <td>
        Fornece proxy para ser usado em tarefas de conexão HTTP. Pode ser HTTP ou HTTPS. O proxy deve estar no formato `http(s)://proxyIp:proxy`. Se a CLI de diagnóstico encontrar um proxy no arquivo de configuração do agente, ele usará esse proxy por padrão. Na maioria dos casos, `port` não é necessário. Se a CLI de diagnóstico encontrar um proxy no arquivo de configuração do agente, ele usará esse proxy por padrão.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-pw STRING`
      </td>

      <td>
        Senha do proxy, se necessário. Se a CLI de diagnóstico encontrar um proxy no arquivo de configuração do agente, ele usará esse proxy por padrão.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-user STRING`
      </td>

      <td>
        Nome de usuário proxy, se necessário. Se a CLI de diagnóstico encontrar um proxy no arquivo de configuração do agente, ele usará esse proxy por padrão.
      </td>
    </tr>

    <tr>
      <td>
        `-q`
      </td>

      <td>
        A saída silenciosa imprime apenas os resultados de alto nível e não a saída explicativa. Suprime avisos de adição de arquivo se `-y` também for usado. Não contradiz `-v`. Os filtros de inclusão são ignorados. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-qq`
      </td>

      <td>
        A saída muito silenciosa imprime apenas uma única linha de resumo para saída (implica `q`). Suprime avisos de adição de arquivo se `-y` também for usado. Não contradiz `-v`. Os filtros de inclusão são ignorados. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-skip-version-check`
      </td>

      <td>
        Ignore a verificação automática de uma versão mais recente do aplicativo. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-s STRING`

        `-suites STRING`
      </td>

      <td>
        Execute um [conjunto](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag/#suites), uma coleção de tarefas que destinam produtos ou problemas específicos. Para especificar vários conjuntos, separe-os com vírgulas. Para obter uma lista de todas as suítes, execute:

        ```
        ./nrdiag -h suites
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-t STRING`

        `-tasks STRING`
      </td>

      <td>
        Execute apenas um subconjunto de tarefas, por agente ou por tipo de tarefa. Para especificar diversas tarefas, separe-as com vírgulas e/ou com um curinga `*`. Para obter uma lista de todas as tarefas, execute:

        ```
        ./nrdiag -h tasks
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-usage-opt-out`
      </td>

      <td>
        Recuse-se a enviar dados anônimos de uso da ferramenta Diagnostics CLI para a New Relic para esta execução. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-list-scripts`
      </td>

      <td>
        Listar script disponível.
      </td>
    </tr>

    <tr>
      <td>
        `-script STRING`
      </td>

      <td>
        Visualize o script especificado. Use com `-run` para executar o script.
      </td>
    </tr>

    <tr>
      <td>
        `-run`
      </td>

      <td>
        Use com `-script` para executar o script.
      </td>
    </tr>

    <tr>
      <td>
        `-script-flags`
      </td>

      <td>
        Use com `-run -script` para passar sinalizadores de linha de comando para o script.
      </td>
    </tr>

    <tr>
      <td>
        `-v`

        `-verbose`
      </td>

      <td>
        Exibir log detalhado durante a execução da verificação. Desativado por padrão. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-version`
      </td>

      <td>
        Exibe a versão atual da CLI de diagnóstico. Isso também solicita a verificação de uma versão mais recente e o download se uma versão mais recente estiver disponível. Tem precedência sobre `-skip-version-check`. Tipo: bool
      </td>
    </tr>

    <tr>
      <td>
        `-y`

        `-yes`
      </td>

      <td>
        Responda `yes` a qualquer prompt que surgir durante a execução. Desativado por padrão. Tipo: bool
      </td>
    </tr>
  </tbody>
</table>
