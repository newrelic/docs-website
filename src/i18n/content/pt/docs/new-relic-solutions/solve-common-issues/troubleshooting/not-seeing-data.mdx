## Problema

Você não vê dados na interface após [instalar um agente New Relic](/docs/agents/manage-apm-agents/installation/install-agent).

## Solução

Você deve começar a ver os dados alguns minutos após instalar um agente New Relic e gerar tráfego para seu aplicativo. Caso não veja os dados, você pode usar o utilitário [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) para identificar automaticamente problemas comuns. Para obter dicas adicionais sobre resolução de problemas, consulte a documentação específica do agente.

Além disso, você pode tentar estas etapas de resolução de problemas que se aplicam a todos os agentes APM:

<CollapserGroup>
  <Collapser
    id="deleted_apps"
    title="Aplicativo excluído ou renomeado no APM"
  >
    Um aplicativo precisa parar de relatar dados por pelo menos uma hora antes que você possa reutilizar esse nome. Ele também precisa se reconectar ao coletor New Relic (ser reiniciado) antes que novos dados sejam aceitos.

    O aplicativo permanece no cache do coletor por uma hora antes de ser liberado. Durante esse período, ele será marcado como “excluído”, portanto nenhum dado novo será aceito. Além disso, os dados são associados a um aplicativo em execução que foi excluído até que o agente seja reiniciado.

    Para mais informações, veja:

    * [Nomeie seu aplicativo](/docs/site/naming-your-application)
    * [Use vários nomes para um aplicativo](/docs/site/using-multiple-names-for-an-app)
  </Collapser>

  <Collapser
    id="data_collection"
    title="Sem conexão com o coletor"
  >
    Seu aplicativo não será afetado se o agente da New Relic não conseguir se conectar ao [coletor](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#collector). Os dados continuam a ser coletados e carregados assim que a conexão for restaurada.

    Enquanto a rede estiver inoperante ou o coletor indisponível, você poderá ver lacunas onde faltam dados na CPU do APM e nos gráficos de memória. O agente continuará tentando se reconectar e, quando conseguir, você verá novamente os dados aparecendo na interface do usuário.

    Enquanto o agente não consegue se comunicar com o coletor, ele ainda está coletando dados. Assim que conseguir se conectar novamente, ele fará upload dos dados e preencherá o segmento ausente para que não haja qualquer confusão sobre se seu aplicativo estava inativo ou simplesmente não reportava dados. Para economizar memória, os dados serão agregados e calculados a média ao longo do período, de modo que você verá barras planas e gráficos durante o período em que não foi possível se comunicar com o coletor.
  </Collapser>
</CollapserGroup>
