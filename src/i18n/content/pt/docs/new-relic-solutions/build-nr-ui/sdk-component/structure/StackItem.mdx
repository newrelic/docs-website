Elemento filho do componente `<Stack>` .

O `<Stack>` pode alterar a altura, largura e ordem do item stack com base no layout selecionado.

### Uso

```js
import { StackItem } from 'nr1'
```

### Props

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>Nó</h5> <h5>OBRIGATÓRIO</h5>
      </td>

      <td>
        Conteúdo a ser exibido dentro do item stack .
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>corda</h5>
      </td>

      <td>
        Acrescenta nomes de classes ao componente. Deve ser usado apenas para fins de posicionamento e espaçamento.
      </td>
    </tr>

    <tr>
      <td>
        `grow` <h5>boleano</h5>
      </td>

      <td>
        Se `true`, o item preencherá o espaço restante. Para que funcione, você precisará estender sua stack para ocupar todo o espaço disponível na direção de seus itens. Isso pode ser conseguido usando os adereços `fullWidth` ou `fullHeight` da stack, dependendo de sua direção.
      </td>
    </tr>

    <tr>
      <td>
        `shrink` <h5>boleano</h5>
      </td>

      <td>
        Se `true`, o item será reduzido, se necessário.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objeto</h5>
      </td>

      <td>
        Estilo embutido para estilo personalizado.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>corda</h5>
      </td>

      <td>
        Adiciona um atributo `data-test-id` . Use-o para destino do componente em testes unitários e E2E. Para que um ID de teste seja válido, prefixe-o com seu ID nerdpack, seguido por um ponto. Por exemplo, `my-nerdpack.some-element`.

        **Nota:** Você pode não ver `data-test-id` atributo pois eles são removidos do DOM, para depurá-los passe um parâmetro de consulta `e2e-test` para a URL.
      </td>
    </tr>

</table>
