Ao criar um aplicativo New Relic, é importante visualizá-lo na plataforma. Isso ajuda você a projetar, implementar e depurar seu aplicativo no ambiente em que ele será publicado. Com `nr1`, você pode lançar um servidor de desenvolvimento que hospede seu aplicativo para que New Relic possa apresentá-lo a você.

<Steps>
  <Step>
    De dentro do diretório raiz do Nerdpack, sirva seu Nerdpack:

    ```bash
    nr1 nerdpack:serve
    ```

    Quando o Nerdpack for compilado com sucesso e seu aplicativo estiver pronto para visualização, você verá uma mensagem com um link para o New Relic:

    ```bash
    [output] {green}✔{plain}  Server ready! Test it at: {purple}https://one.newrelic.com/?nerdpacks=local
    [output] {blue}↩{plain}  Server will reload automatically if you modify any file!
    ```

    Observe o parâmetro de consulta anexado `nerdpacks=local`. Este parâmetro de consulta diz ao New Relic para mostrar Nerdpacks servidos localmente.

    <Callout variant="tip">
      Se você não encontrar seu aplicativo na plataforma New Relic, certifique-se de incluir `nerdpacks=local` em sua string de consulta!
    </Callout>
  </Step>

  <Step>
    Navegue até o URL fornecido. Na página inicial, selecione **Apps** para visualizar o aplicativo New Relic.
  </Step>

  <Step>
    Na seção **Your apps**, encontre seu launcher, chamado "teste A/B Launcher".
  </Step>

  <Step>
    Selecione seu launcher para ver seu Nerdlet root e sua mensagem de boas-vindas padrão.
  </Step>
</Steps>

Parabéns, você atendeu seu primeiro aplicativo New Relic!

Observe, na saída do comando, que o servidor é recarregado quando você altera os arquivos do seu Nerdpack. Experimente atualizando `nerdlets/ab-test-nerdlet/index.js`:

```js fileName=nerdlets/ab-test-nerdlet/index.js lineHighlight=5
import React from 'react';

export default class AbTestNerdletNerdlet extends React.Component {
    render() {
        return <h1>A/B test results</h1>;
    }
}
```

Seu aplicativo é atualizado automaticamente para mostrar o novo título.
