Antes de prosseguir com a criação de um início rápido, você precisa entender sua estrutura de diretórios.

Para começar, navegue até o repositório GitHub de início rápido do New Relic e faça um fork. Em seguida, clone seu próprio repositório em sua máquina local usando `git clone` e abra-o no IDE de sua preferência.

Na raiz do projeto clonado, os diretórios na pasta `\_template` espelham os diretórios no nível superior do repositório e contêm os modelos para os respectivos blocos de construção de observabilidade no início rápido.

Com a atualização recente na estrutura dos guias de início rápido, o diretório `\_template` agora tem a seguinte estrutura:

```
alert-policies/
    example-alert-policy/
        baseline-alert.yml
        static-alert.yml

dashboards/
    example-dashboard/
        example-dashboard.json
        example-dashboard.png

data-sources/
    example-data-source/
        config.yml
        logo.svg

quickstarts/
    example-quickstart/
        config.yml
        logo.svg
```

Cada um desses diretórios contém modelos que você pode usar para criar entidade para o seu início rápido. A estrutura das novas contribuições deve corresponder à estrutura vista aqui. Consulte os dashboards e alertas associados nos guias de início rápido `config.yml`.

Para construir um início rápido, primeiro você precisa adicionar sua entidade como dashboard e alertas ao início rápido.
