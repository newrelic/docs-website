Imagine que você desenvolveu um novo banco de dados chamado FlashDB, o banco de dados mais leve e rápido do espaço. Seu banco de dados já é muito popular entre os desenvolvedores. Você sabe que New Relic oferece ótimas ferramentas de observabilidade, então você recomenda que seu usuário use New Relic para obter insights de sua stack. Até agora, seu usuário precisava configurar manualmente o painel e os alertas no New Relic. Recentemente, você aprendeu sobre a observabilidade (I/O) instantânea New Relic , que fornece blocos de construção de observabilidade para obter valor de seus dados com mais rapidez.

Com a crescente popularidade do seu banco de dados, você decide ajudar o usuário do FlashDB a observá-lo de forma rápida e fácil no New Relic.

Para atingir seu objetivo, contribua com um início rápido para New Relic I/O para que outros usuários possam monitor rapidamente o FlashDB.

## Observabilidade instantânea [#instant-observability]

[A observabilidade instantânea (I/O) New Relic ](https://newrelic.com/instant-observability)é um ecossistema unificado de recursos de observabilidade prontos para uso que ajudam você a obter valor imediato de seus dados. Ele permite que você instrumente, monitor e analise sua pilha de tecnologia, evitando o fardo da configuração manual.

Com o New Relic I/O, você:

* Comece mais rápido
* Reduza o trabalho da instrumentação manual
* Aprenda com especialistas em observabilidade

<Video id="c9zprczTXj8" type="youtube" />

Você pode escolher entre centenas de guias de início rápido que reúnem os blocos de construção necessários para instrumentação e monitoramento de sua pilha de tecnologia, instalá-los com um clique e iniciar o monitoramento.

## Início rápido [#quickstart]

Com New Relic, você pode coletar, monitor e relatar dados de diferentes fontes. Você pode usar o painel para visualizar, explorar, personalizar, compreender e correlacionar os dados coletados usando diferentes gráficos fáceis de usar. Você também pode definir políticas de alertas para monitor métricas de desempenho chave e receber notificações de anomalias.

Tradicionalmente, o usuário FlashDB teria que gastar horas cruciais de desenvolvimento criando painéis, alertas e outras entidades para monitor seu banco de dados. Mas com os guias de início rápido, você pode fazer esse trabalho para eles. Os guias de início rápido fornecem valor imediato para seu caso de uso específico. Eles incluem:

* Instruções claras para instrumentar seus serviços
* Blocos de construção de observabilidade como dashboard e alertas

Tudo isso está disponível por meio do New Relic I/O, onde os desenvolvedores, parceiros e clientes da New Relic contribuem com suas soluções práticas recomendadas. Vocês, como desenvolvedores do FlashDB, são um ótimo candidato para contribuir com um início rápido para ajudar seu usuário monitor rapidamente sua instância.

## Objetivos [#objectives]

* Envie dados de telemetria para New Relic

* Crie dashboard e alertas

* Crie um início rápido

  * Adicione seu dashboard
  * Adicione seus alertas
  * Adicionar uma fonte de dados
  * Contribuir com início rápido para GitHub

## Pré-requisito [#prerequisite]

* Crie uma [conta gratuita da New Relic](https://login.newrelic.com/login)
* Instale [Python](https://www.python.org/downloads/)
