Antes de criar um início rápido, você precisa ativar seus serviços de demonstração.

## Ative os serviços de demonstração [#spin-demo-services]

Clone o repositório do laboratório do GitHub:

```bash
    git clone https://github.com/newrelic-experimental/build-a-quickstart-lab.git
```

Mude para o diretório de demonstração, chamado `flashDB`.

```bash

cd build-a-quickstart-lab/flashDB

```

Este diretório contém código para dois serviços importantes:

* Um programa python que imita o banco de dados e também fornece funções para operações de criação, leitura, atualização e exclusão (CRUD).
* Um serviço de simulador que gera tráfego de banco de dados fictício para que você não precise realizar operações CRUD manualmente

Execute o seguinte comando para executar seus serviços:

```Python
python simulator.py
```

Aqui, você executa o simulador que gera tráfego simulado em seu banco de dados. Assim que o simulador estiver instalado e funcionando, você verá a seguinte saída em seu terminal:

```bash animate
python simulator.py
[output] Writing...
[output] try_send
[output] Writing...
[output] try_send
[output] Reading...
[output] try_send
[output] Reading...
[output] try_send
[output] Writing...
[output] try_send
[output] Writing...
[output] try_send
[output] Reading...
```

Em seguida, você usa [telemetria](https://github.com/newrelic/newrelic-telemetry-sdk-python) para enviar dados de telemetria do seu serviço flashDB para New Relic.
