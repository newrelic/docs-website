O primeiro passo no processo de construção de um [início rápido](https://opensource.newrelic.com/projects/newrelic/newrelic-quickstarts) é instrumentar sua biblioteca ou configurar sua plataforma para enviar dados de telemetria para New Relic. Depois de configurar isso, o usuário do seu produto poderá começar a ver esses dados em sua conta.

Configurar seu produto para permitir que seu usuário envie facilmente dados de seu produto para New Relic é importante porque o início rápido que você oferecerá a eles será usado em painéis, alertas ou outros recursos que você fornecer. Assim que o usuário puder ver os dados, ele poderá usar o início rápido para visualizá-los e obter insights acionáveis em questão de minutos.

## Tipos de dados de telemetria [#telemetry-data-types]

Você precisa enviar pelo menos um tipo de dados de telemetria da sua biblioteca ou plataforma para o New Relic para criar um início rápido eficaz. New Relic oferece suporte [a quatro tipos de dados de telemetria](/docs/data-apis/understand-data/new-relic-data-types):

* [Métrica](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-metrics): métricas são medidas agregadas do desempenho e comportamento do seu sistema. Se o seu produto for um banco de dados, você poderá enviar métricas como utilização de CPU, utilização de memória e taxas de transferência de consulta. Observe que as métricas geralmente são usadas apenas se você deseja limitar a quantidade de dados enviados para New Relic. Note-se que muitas métricas, tais como taxas de erros e taxas de transferência, podem ser calculadas por agregação de eventos.
* [Evento](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-events): evento captura coisas que ocorrem em seu produto. Se sua plataforma automatizar a implantação do aplicativo, você poderá gerar um evento sempre que um trabalho for executado. Se seu aplicativo verificar vulnerabilidades de segurança, você poderá gerar um evento sempre que detectar uma.
* [Logs](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-logs): logs são registros de texto baseados em tempo que ajudam seu usuário a ver o que está acontecendo em seu sistema.
* [Trace](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/send-traces): o rastreamento captura detalhes de uma única solicitação à medida que ela se move através de um sistema.
