As fontes de dados representam um único tipo de instrumentação, como um agente, um atributo sobre uma transação, uma integração de provedor de nuvem, uma integração de terceiros, etc.

Existem dois tipos de fontes de dados:

* CORE
* COMMUNITY

As fontes de dados CORE são fornecidas pela New Relic One e não existem neste repositório, as fontes de dados COMMUNITY são definidas no repositório de início rápido.

As fontes de dados COMMUNITY residem no diretório `data-sources/`. Neste procedimento, você aprende como adicionar uma fonte de dados ao início rápido.

<Steps>
  <Step>
    Se ainda não o fez, bifurque o repositório [de início rápido do New Relic](https://github.com/newrelic/newrelic-quickstarts) e clone-o em sua máquina local.
  </Step>

  <Step>
    Abra seu projeto no IDE de sua preferência e navegue até o diretório `\_template`.
  </Step>

  <Step>
    Aqui, copie o diretório `data-sources/example-data-source` e seu conteúdo para o diretório `data-sources` no nível raiz. Renomeie o diretório como `flashdb`.

    Este diretório contém um arquivo YAML de amostra para criar uma fonte de dados.
  </Step>

  <Step>
    Navegue até o arquivo `datasources/flashdb/config.yml` e adicione detalhes da sua fonte de dados. Para sua referência, a fonte de dados flashdb deve ser semelhante à seguinte:

    ```yml
    id: flashdb
    displayName: FlashDB
    description: |
      FlashDB is the fastest and lightest database in the universe. First, it can handle a large number of concurrent users without slowing down. Second, it has a very low latency, so queries are processed quickly. And finally, it scales easily as your data grows.
      This quickstart allows users to monitor their database to ensure peak performance. It also notifies you if there's any problem the database. 

      The flashdb quickstart includes: 
      ## a dashboard 
        where users can easily monitor:
        - Database methods
        - Average response time
        - Errors
        - Database size
        - Cache hits
        - Keys
      ## alert conditions
        where users are notified on:
        - slow read responses
        - low cache hit 
    icon: logo.svg
    install:
      primary:
        link:
          url: https://www.newrelic.com
    keywords:
      - infrastructure
      - database
    categoryTerms:
      - database
    ```
  </Step>

  <Step>
    Adicione um logotipo para sua fonte de dados. Pode ser `.svg` ou `.png`.
  </Step>
</Steps>
