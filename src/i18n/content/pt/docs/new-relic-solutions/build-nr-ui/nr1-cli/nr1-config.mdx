## `nr1 config:delete` [#nr1-configdelete]

### Remover uma configuração [#nr1-configdelete-remove]

Remove o valor de uma configuração específica.

### Uso [#nr1-configdelete-usage]

`$ nr1 config:delete OPTION`

### Opções [#nr1-configdelete-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        Isto é **necessário**. A chave da configuração.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        O perfil de autenticação que você deseja usar.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        Se presente, esta configuração só será aplicada durante a execução com o perfil especificado.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Adiciona informações extras à saída.
      </td>
    </tr>
  </tbody>
</table>

### Apelido [#nr1-configdelete-aliases]

* `nr1 config:remove`
* `nr1 config:rm`

### Exemplos [#nr1-configdelete-exmples]

* `$ nr1 config:delete --key=proxyHttp`

## `nr1 config:set` [#nr1-configset]

### Definir uma configuração [#nr1-configset-set]

Define um valor de configuração específico, dada uma chave de configuração. Por padrão, o comando prompt um novo valor após fornecer uma chave, mas você também pode usar a opção `--k, --key=KEY` para pular esta etapa.

### Uso [#nr1-configset-usage]

`$ nr1 config:set OPTION`

### Opções [#nr1-configset-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        Isto é **necessário**. A chave da configuração.
      </td>
    </tr>

    <tr>
      <td>
        `-V, --value=VALUE`
      </td>

      <td>
        O valor da configuração.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        O perfil de autenticação que você deseja usar
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        Se presente, esta configuração só será aplicada durante a execução com o perfil especificado.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Adiciona informações extras à saída.
      </td>
    </tr>
  </tbody>
</table>

### Exemplos [#nr1-configset-examples]

* `$ nr1 config:set --key=proxyEnabled`
* `$ nr1 config:set --key=proxyEnabled --value=ENABLED`
