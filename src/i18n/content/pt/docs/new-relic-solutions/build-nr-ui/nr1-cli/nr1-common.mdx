## preenchimento automático nr1 [#nr1-autocomplete]

Por padrão, o comando exibe as instruções de preenchimento automático para `zsh`. Se você quiser instruções para o bash, execute `nr1 autocomplete bash`.

### Uso

`$ nr1 autocomplete`

### Argumentos

<table>
  <tbody>
    <tr>
      <td>
        `SHELL`
      </td>

      <td>
        O tipo de shell para o qual você deseja instruções.
      </td>
    </tr>
  </tbody>
</table>

### Opções

<table>
  <tbody>
    <tr>
      <td>
        `-r, --refresh-cache`
      </td>

      <td>
        Atualiza o cache e ignora a exibição de instruções.
      </td>
    </tr>
  </tbody>
</table>

### Exemplos

* `$ nr1 autocomplete`
* `$ nr1 autocomplete zsh`
* `$ nr1 autocomplete bash`
* `$ nr1 autocomplete --refresh-cache`

## nr1 criar [#nr1-create]

Cria um novo componente a partir do nosso modelo, seja um Nerdpack, Nerdlet, launcher ou catálogo. A CLI orientará você nesse processo.

Para saber mais sobre Nerdpacks e sua estrutura de arquivos, consulte [Estrutura de arquivos Nerdpack](/docs/new-relic-solutions/build-nr-ui/nerdpack-file-structure). Para mais informações sobre como configurar seus Nerdpacks, consulte nossos [comandos CLI do Nerdpack](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-nerdpack).

### Uso

`$ nr1 create`

### Opções

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        Se presente, substitui os arquivos existentes sem perguntar.
      </td>
    </tr>

    <tr>
      <td>
        `-n, --name=NAME`
      </td>

      <td>
        Nomeia o componente.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --type=TYPE`
      </td>

      <td>
        Especifica o tipo de componente.
      </td>
    </tr>

    <tr>
      <td>
        `--path=PATH`
      </td>

      <td>
        A rota para o componente.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        O perfil de autenticação que você deseja usar.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Adiciona informações extras à saída.
      </td>
    </tr>
  </tbody>
</table>

## ajuda nr1 [#nr1-help]

Mostra todos os comandos nr1 por padrão. Para obter detalhes sobre um comando específico, execute `nr1 help COMMAND_NAME`.

### Uso

`$ nr1 help`

### Argumentos

<table>
  <tbody>
    <tr>
      <td>
        `COMMAND_NAME`
      </td>

      <td>
        O nome de um comando específico.
      </td>
    </tr>
  </tbody>
</table>

### Exemplos

* `$ nr1 help`
* `$ nr1 help nerdpack`
* `$ nr1 help nerdpack:deploy`

## nr1 NRQL [#nr1-nrql]

Busca dados do banco de dados usando uma consulta NRQL .

Para saber mais sobre NRQL e como usá-lo, consulte nossa [documentação sobre NRQL](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/).

### Uso

`$ nr1 nrql OPTION ...`

### Opções

<table>
  <tbody>
    <tr>
      <td>
        `-a, --account=ACCOUNT`
      </td>

      <td>
        Isto é **necessário**. O ID da conta do usuário.
      </td>
    </tr>

    <tr>
      <td>
        `-q, --query=QUERY`
      </td>

      <td>
        Isto é **necessário**. A consulta NRQL a ser executada.
      </td>
    </tr>

    <tr>
      <td>
        `-u, --ugly`
      </td>

      <td>
        Exibe o conteúdo sem tabulações ou espaços.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        O perfil de autenticação que você deseja usar.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Adiciona informações extras à saída.
      </td>
    </tr>
  </tbody>
</table>

## Perfis nr1 [#nr1-profiles]

Exibe uma lista de comandos que você pode usar para gerenciar seus perfis. Execute `nr1 help profiles:COMMAND` para saber mais sobre seus usos específicos. Você pode ter mais de um perfil, o que é útil para executar comandos em várias contas New Relic.

Para saber mais sobre como configurar perfis, consulte [nosso workshop no Github](https://github.com/newrelic/nr1-workshop/blob/master/lab-cli/INSTRUCTIONS.md).

### Uso

`$ nr1 profiles:COMMAND`

### Comandos

<table>
  <tbody>
    <tr>
      <td>
        `profiles:add`
      </td>

      <td>
        Adiciona um novo perfil às chaves dos seus perfis.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:default`
      </td>

      <td>
        Escolhe qual perfil deve ser padrão.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:list`
      </td>

      <td>
        Lista os perfis em seu chaveiro.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:remove`
      </td>

      <td>
        Remove um perfil do seu chaveiro.
      </td>
    </tr>
  </tbody>
</table>

## atualização nr1 [#nr1-update]

Atualizações para a versão mais recente da CLI. Você pode especificar qual canal atualizar, se desejar.

### Uso

`$ nr1 update`

### Argumentos

<table>
  <tbody>
    <tr>
      <td>
        `CHANNEL`
      </td>

      <td>
        O nome de um canal específico.
      </td>
    </tr>
  </tbody>
</table>

### Exemplos

* `$ nr1 update`
* `$ nr1 update somechannel`
