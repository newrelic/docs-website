No APM, a página <DNT>**Databases**</DNT> mostra o banco de dados e os dados de cache de um aplicativo. A página mostra a transação individual do banco de dados como uma tabela classificável e as operações, taxas de transferência e tempo de resposta como gráficos. Se [consultas lentas](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) foram reportadas, elas serão mostradas quando você selecionar uma transação.

Dividimos as informações do banco de dados em três partes:

* Visão geral de todo o banco de dados (página principal)

  * As cinco principais operações de banco de dados por tempo transcorrido

  * Tempo de resposta do banco de dados

  * Taxas de transferência de banco de dados (solicitações de operação por minuto ou

    <DNT>
      **rpm**
    </DNT>

    )

* Principais operações de banco de dados (subpágina)

  * Tempo de consulta
  * Taxas de transferência
  * Consumo de tempo por chamador

* Tabela com todas as operações do banco de dados (subpágina)

## Navegação [#database-page]

Escolha uma das seguintes opções para abrir a página:

* Vá para

  <DNT>
    **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > All entities > (select an app) > Monitor > Databases**
  </DNT>

  .

* Vá para

  <DNT>
    **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select an app) > Monitor > Databases**
  </DNT>

  .

<Callout variant="tip">
  Para obter uma visão geral de alto nível de todos os seus aplicativos e serviços, use nosso [explorador de entidades](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer).
</Callout>

## Filtrar operações por fornecedor de banco de dados [#db_filtering]

Se disponível: Acima da lista de operações de banco de dados, os botões permitem [filtrar](#availablity) até um fornecedor de banco de dados específico.

* Para filtrar o conteúdo por fornecedor de banco de dados, selecione seu nome.

* Para remover o filtro, selecione

  <DNT>
    **All**
  </DNT>

  .

## Revise as principais operações do banco de dados [#top_db_operations]

A página principal <DNT>**Databases**</DNT> mostra uma lista das principais operações do banco de dados que você pode classificar de diversas maneiras.

Para obter detalhes sobre uma operação, você pode passar o mouse sobre o registro ou clicar nele para ver gráficos adicionais.

Ao clicar em um registro, você pode ver o gráfico <DNT>**Time consumption by caller**</DNT> . Apresenta uma lista das transações que ligam para a operação selecionada, com barras coloridas indicando a porcentagem de tempo consumido por cada chamador. Para visualizar a página <DNT>**Transactions**</DNT> de uma transação do chamador, selecione a barra colorida dessa transação no gráfico.

## Exibir todas as operações do banco de dados [#db_displaying]

Para mostrar todas as operações do banco de dados [, se disponíveis](#availability):

* Na parte inferior da lista de operações, selecione o link

  <DNT>
    **Show all database operations table...**
  </DNT>

  .

* Para classificar a tabela por qualquer coluna, selecione o cabeçalho da coluna.

* Para filtrar a lista de operações, use a caixa de pesquisa na parte superior da tabela.

* Para visualizar detalhes de uma operação específica, selecione o nome da operação.

## Excluir rastreamento [#db_deleting]

Para excluir permanentemente todos os rastreamentos do seu aplicativo:

1. Selecione o botão

   <DNT>
     **Delete all traces**
   </DNT>

   abaixo da lista de operações do banco de dados.

2. Selecione o prompt de confirmação.

<Callout variant="tip">
  Se você estiver fazendo uma limpeza geral de registros, também poderá excluir [o trace da transação](/docs/traces/deleting-transaction-traces) ou [o error trace](/docs/apm/applications-menu/events/viewing-apm-errors-error-traces). Isso removerá dados potencialmente confidenciais, mantendo os outros dados do aplicativo (como Apdex, informações de implantação, etc.).
</Callout>

## Recurso adicional baseado na versão do agente [#availablity]

Recursos adicionais na página <DNT>**Databases**</DNT> estão disponíveis nas seguintes versões do agente. Para obter mais informações, consulte [Atualizar o agente New Relic](/docs/agents/manage-apm-agents/installation/update-new-relic-agent).

* Vá para 1.4 ou superior
* Java 3.14.0 ou superior
* .NET 4.1.134 ou mais alto
* Node.js 1.17.0 ou superior
* PHP 4.19.0 ou superior
* Python 2.44.0 ou superior
* Ruby 3.11.0 ou superior

Esses recursos incluem:

* Vários tipos de banco de dados e cache
* Etiquetas para diferentes tipos na lista de operações, gráficos e consulta lenta
* Filtrando operações e gráficos para um fornecedor de banco de dados ou tipo de cache específico
* Visualizando uma tabela de todas as operações do banco de dados
* Capacidade de visualizar o banco de dados em [mapas de serviços](/docs/apm/applications-menu/monitoring/service-maps)

Além disso, se o seu agente APM oferecer suporte a [detalhes em nível de host e de instância](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues), você poderá visualizar os detalhes da consulta e a instância específica onde a consulta foi executada.
