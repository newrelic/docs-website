## Problema

Você precisa de uma maneira fácil de obter informações sobre o ambiente do seu aplicativo, como:

* Quais serviços uma determinada versão do agente está usando
* Quais aplicativos têm versões de agente desatualizadas
* Quais hosts usam qual agente
* Quais valores se aplicam a configurações específicas do agente

## Solução

Para ver quais serviços e configurações se aplicam a um agente ou qual agente usa um serviço ou configuração específico, acesse <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>. Clique no seu aplicativo na lista e depois clique em <DNT>**Settings > Environment > Environment snapshot**</DNT>.

A página <DNT>**Environment snapshot**</DNT> ajuda você a conectar cargas do agente que incluem dados do ambiente, para que você tenha uma noção de quais dados de monitoramento são registrados. Isso também ajuda a identificar mais facilmente discrepâncias em seus serviços e a determinar o que pode precisar ser atualizado.

No exemplo a seguir, o usuário passa o mouse sobre o segundo agente da lista. A maioria das configurações desse agente é consistente com outras, mas é fácil ver configurações específicas do host para processadores lógicos, núcleos físicos e níveis de patch Ruby. Observe também que este agente está usando uma versão anterior do Ruby.

As informações na página <DNT>**Environment snapshot**</DNT> dependem do agente APM que você usa. Por exemplo, aplicativos que usam o agente Ruby mostram configurações para gems, enquanto aplicativos que usam o agente Java mostram configurações para jars.
