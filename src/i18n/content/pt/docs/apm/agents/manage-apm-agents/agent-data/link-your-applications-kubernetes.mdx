Agente pode ser configurado para coletar variáveis de ambiente relacionadas ao Kubernetes e adicioná-las ao evento Transaction para fornecer contexto entre seu cluster do Kubernetes e seus serviços. Para obter detalhes sobre os benefícios, consulte [esta postagem do blog](https://blog.newrelic.com/engineering/monitoring-application-performance-in-kubernetes/).

Para mais informações sobre compatibilidade do agente e instruções de instalação, consulte nossa [documentação do Kubernetes](/docs/integrations/kubernetes-integration/metadata-injection/kubernetes-apm-metadata-injection).
