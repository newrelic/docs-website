## Problema

Você está usando a [plataforma Solr](http://lucene.apache.org/solr/) em seu aplicativo da web, mas os dados do Solr não aparecem na interface do usuário <InlinePopover type="apm"/>do seu aplicativo de <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select a Java app) > Monitor**</DNT>.

## Solução

Para garantir que o Solr esteja [configurado](/docs/agents/java-agent/getting-started/compatibility-requirements-java-agent#other) corretamente e que os dados sejam reportados ao New Relic, faça o seguinte conforme aplicável:

* <DNT>
    **JMX:**
  </DNT>

  Certifique-se de que o JMX esteja ativado na configuração do Solr em `solrconfig.xml` usando a tag `<jmx/>` . O arquivo `solrconfig.xml` é o arquivo de configuração com mais parâmetros que afetam o próprio Solr.

* <DNT>
    **Jetty:**
  </DNT>

  Se você for um usuário Jetty, [ative manualmente o JMX para Jetty](https://wiki.eclipse.org/Jetty/Tutorial/JMX#Enabling_JMXConnectorServer_for_Remote_Access).

* <DNT>
    **Solr metrics:**
  </DNT>

  Verifique no console JMX a métrica do Solr. O cache e as atualizações do Solr vêm do MBeans.
