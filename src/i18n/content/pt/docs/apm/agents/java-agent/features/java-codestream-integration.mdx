O nível do código métrico permite que os desenvolvedores que usam a [extensão New Relic CodeStream](/docs/codestream/observability/code-level-metrics) vejam os dados APM exibidos contextualmente em seu IDE, juntamente com métodos individuais no código. Isso permite que os desenvolvedores sejam mais proativos na abordagem de problemas de desempenho à medida que escrevem e revisam o código.

## Começando

Primeiro, [instale](/docs/codestream/start-here/install-codestream) a extensão New Relic CodeStream no IDE compatível de sua escolha e login.

<Callout variant="important">
  Nível do código métrica estão disponíveis para o agente Java versão 7.10.0 ou superior e habilitados por padrão para a versão 7.11.0 e superior. Consulte [Configuração do agente Java: arquivo de configuração para obter](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file/#code-level-metrics) instruções sobre como ativá-lo ou desativá-lo.
</Callout>

## Atributo do agente

O agente Java relata e anexa o seguinte atributo aos spans produzidos pelo seu aplicativo:

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        <DNT>
          **Name**
        </DNT>
      </th>

      <th>
        <DNT>
          **Description**
        </DNT>
      </th>

      <th>
        <DNT>
          **Example**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `code.function`
      </td>

      <td>
        O nome da função instrumentada (método Java).
      </td>

      <td>
        retrieveData
      </td>
    </tr>

    <tr>
      <td>
        `code.namespace`
      </td>

      <td>
        O namespace (nome de classe totalmente qualificado) no qual `code.function` é definido
      </td>

      <td>
        com.example.MyService
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="important">
  Os spans criados usando o método API `NewRelic.getAgent().getTransaction().startSegment()` não conterão esses atributos. Para criar um intervalo que contenha esses atributos, anote um método com `@Trace` ou use instrumentação XML.
</Callout>
