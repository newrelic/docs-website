## Problema

Seu aplicativo usa o agente Java da New Relic e você se depara com um [problema de agrupamento métrico](/docs/agents/manage-apm-agents/troubleshooting/metric-grouping-issues).

## Solução

Experimente estes procedimentos de resolução de problemas antes que as questões de agrupamento métrico se tornem um problema muito grande.

<CollapserGroup>
  <Collapser
    id="reconfigure-java"
    title="Certifique-se de que a nomenclatura baseada em componentes esteja habilitada"
  >
    Por padrão, os [nomes dos agentes Java são tratados](/docs/agents/java-agent/instrumentation/naming-web-transactions) com base em sua framework e nos diversos componentes de sua aplicação. No entanto, se o agente Java estiver configurado para desabilitar a nomenclatura baseada em componentes, ele não será capaz de nomear adequadamente essas transações. Isto muitas vezes levará a um problema de agrupamento métrico.

    Para solucionar problemas de configuração do agente Java: Se [a nomenclatura baseada em componentes](/docs/agents/java-agent/instrumentation/naming-web-transactions#reverting) tiver sido desativada, reative-a.
  </Collapser>

  <Collapser
    id="upgrade"
    title="Atualizar o agente Java"
  >
    Alguns problemas de agrupamento métrico podem ser resolvidos simplesmente [atualizando seu agente New Relic](/docs/agents/java-agent/installation/upgrade-java-agent). A menos que você tenha um bom motivo para não fazê-lo, a New Relic recomenda sempre atualizar para a versão mais recente.

    Para solucionar problemas da versão do agente Java, consulte as [notas sobre a versão do agente Java](/docs/release-notes/agent-release-notes/java-release-notes). As notas de versão resumem correções e alterações, incluindo situações em que problemas de agrupamento métrico relacionados a agentes foram corrigidos.
  </Collapser>

  <Collapser
    id="manually-name-transactions"
    title="Nomear transação manualmente"
  >
    Se a versão do seu agente Java estiver atualizada e configurada corretamente, mas você ainda tiver um problema de agrupamento métrico, pode ser necessário [nomear manualmente sua transação](/docs/agents/java-agent/instrumentation/naming-web-transactions). Para ajudá-lo a começar, consulte [o programa de exemplo da New Relic usando a API Java](/docs/agents/java-agent/custom-instrumentation/java-agent-api-example-program).

    A [API do agente Java](/docs/agents/java-agent/custom-instrumentation/java-agent-api) permite [definir um nome de transação](http://newrelic.github.io/java-agent-api/javadoc/com/newrelic/api/agent/NewRelic.html#setTransactionName%28java.lang.String,%20java.lang.String%29) em seu aplicativo. Isso ajuda a eliminar nomes e IDs exclusivos em nomes de métricas.
  </Collapser>

  <Collapser
    id="resolve-limit"
    title="Resolver limite de métrica colocado pela New Relic"
  >
    Depois de detectar um problema de agrupamento métrico, o New Relic aplica automaticamente um limite métrico ao seu aplicativo. Isso evita que novos nomes de métricas entrem em seu aplicativo.

    O limite métrico evita que o problema se agrave. No entanto, antes que o New Relic possa remover o limite, a causa do problema de agrupamento métrico deve ser resolvida.

    Depois que o problema subjacente for gerenciado, [o suporte da New Relic](https://support.newrelic.com) verificará seu aplicativo para garantir que:

    * Seu agente Java atualizado não envia mais as métricas antigas que estavam causando problemas de agrupamento.

    * Seus novos [nomes de transação](/docs/agents/java-agent/instrumentation/naming-web-transactions) excluem adequadamente as partes exclusivas do nome.

      Após confirmar que o problema de agrupamento de métricas foi resolvido, o suporte da New Relic atualizará manualmente seu aplicativo para que novos nomes de métricas sejam novamente permitidos.
  </Collapser>
</CollapserGroup>
