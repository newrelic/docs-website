Você pode usar o agente Java da New Relic para coletar e visualizar métricas adicionais fornecidas pelos servidores de aplicativos mais comuns via JMX. As métricas coletadas variam de acordo com o servidor do aplicativo e incluem medições de pools de threads, sessões HTTP e transações.

## Ver métricas JVM [#dashboard]

Os servidores de aplicativos coletam e reportam diferentes métricas, que aparecem na página <DNT>**JVMs**</DNT> do APM. (Para aplicativos que usam New Relic via Heroku, a página <DNT>**JVMs**</DNT> é chamada <DNT>**[Instances](/docs/accounts-partnerships/partnerships/heroku-new-relic/heroku-instances-dashboard)**</DNT>.)

No mínimo, cada servidor de aplicativos coleta e reporta dados métricos na memória. Para obter instruções sobre como ativar a métrica JMX, consulte a documentação do seu servidor de aplicativos.

Para visualizar métricas JVM:

1. Vá para

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select an app) > Monitoring > JVMs > (select a JVM)**
   </DNT>

   .

2. Na página

   <DNT>
     **Memory**
   </DNT>

   da JVM selecionada, use qualquer uma das [funções padrão da interface do usuário](/docs/accounts-partnerships/education/getting-started-new-relic/standard-dashboard-features) do New Relic para detalhar informações detalhadas.

3. Para visualizar outras métricas, selecione qualquer [aba disponível](#dashboard-tabs).

4. Opcional: para usar o [criador de perfil de encadeamento](/docs/applications-menu/thread-profiler) para procurar gargalo nos dados de execução de encadeamentos, selecione

   <DNT>
     **Profile this JVM**
   </DNT>

   .

## Habilite o Spring Boot 2.2 e superior com Tomcat

Para permitir que o New Relic acesse a métrica JMX do Tomcat ao executar com Spring Boot 2.2 e superior, a propriedade do sistema `server.tomcat.mbeanregistry.enabled` deve ser definida como `true`. Consulte as [notas de versão do Spring Boot 2.2](https://github.com/spring-projects/spring-boot/wiki/Spring-Boot-2.2-Release-Notes#tomcats-mbean-registry).

## Ativar métrica do WebSphere PMI [#default-pmi]

Para permitir que o New Relic acesse a métrica do WebSphere PMI, será necessário configurar o WebSphere para monitor os conjuntos de estatísticas necessários.

<CollapserGroup>
  <Collapser
    id="enable-all-stat-sets"
    title="Habilitar todos os conjuntos de estatísticas (recomendado)"
  >
    A abordagem mais simples é permitir a coleta de todos os conjuntos estatísticos:

    1. Se você estiver usando o Java 2 Security, certifique-se de ter [concedido as permissões adequadas ao New Relic](/docs/agents/java-agent/installation/install-java-agent-java-2-security).

    2. No console de administração do WebSphere, selecione

       <DNT>
         **Monitoring and Tuning > Performance Monitoring Infrastructure (PMI) > (your server name)**
       </DNT>

       .

    3. Marque a caixa de seleção

       <DNT>
         **Enable Performance Monitoring Infrastructure (PMI)**
       </DNT>

       .

    4. Na seção

       <DNT>
         **Currently monitored statistic set**
       </DNT>

       , selecione

       <DNT>
         **All**
       </DNT>

       e salve suas alterações.
  </Collapser>

  <Collapser
    id="enable-custom-stat-sets"
    title="Habilitar conjuntos de estatísticas personalizadas (avançado)"
  >
    Você também pode coletar apenas a métrica do PMI que o New Relic monitora por padrão. Para coletar apenas algumas métricas do PMI:

    1. Se você estiver usando o Java 2 Security, certifique-se de ter [concedido as permissões adequadas ao New Relic](/docs/agents/java-agent/installation/install-java-agent-java-2-security).

    2. No console de administração do WebSphere, selecione

       <DNT>
         **Monitoring and Tuning > Performance Monitoring Infrastructure (PMI) > (selected app server)**
       </DNT>

       .

    3. Marque a caixa de seleção

       <DNT>
         **Enable Performance Monitoring Infrastructure (PMI)**
       </DNT>

       .

    4. Selecione

       <DNT>
         **Custom**
       </DNT>

       . Na lista de conjuntos de estatísticas personalizadas, habilite a seguinte métrica:

       * Na categoria

         <DNT>
           **ThreadPool**
         </DNT>

         , marque as caixas de seleção

         <DNT>
           **ActiveCount**
         </DNT>

         e

         <DNT>
           **PoolSize**
         </DNT>

         .

       * Na categoria

         <DNT>
           **Servlet Session Manager**
         </DNT>

         , marque as caixas de seleção

         <DNT>
           **ActiveCount**
         </DNT>

         e

         <DNT>
           **LiveCount**
         </DNT>

         .

       * Na categoria

         <DNT>
           **Transaction Manager**
         </DNT>

         , marque as caixas de seleção

         <DNT>
           **CommittedCount**
         </DNT>

         ,

         <DNT>
           **RolledbackCount**
         </DNT>

         e

         <DNT>
           **GlobalTimeoutCount**
         </DNT>

         .

    5. No painel

       <DNT>
         **Messages**
       </DNT>

       , selecione

       <DNT>
         **Save directly to the master configuration**
       </DNT>

       .
  </Collapser>
</CollapserGroup>
