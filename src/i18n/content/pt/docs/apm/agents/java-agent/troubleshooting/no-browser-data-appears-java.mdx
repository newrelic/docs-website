## Problema

Você [instalou o agente <InlinePopover type="browser"/>](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent)para seu aplicativo Java, mas o aplicativo não está reportando dados na [interface do browser](/docs/browser/new-relic-browser/getting-started/browser-overview-page-website-performance-summary).

## Solução

Se o seu servidor de aplicativos não usar o compilador Jasper JSP (por exemplo, você usa WebSphere ou WebLogic), a instrumentação automática não estará disponível. Em vez disso, você deve [manualmente instrumento browser](/docs/java/page-load-timing-in-java#manual_instrumentation).

Se o seu servidor de aplicativos usar o compilador JSP e você ainda não receber dados do browser, poderá haver um conflito de configuração. O agente Java da New Relic permite que você defina um nome de aplicativo no arquivo de configuração `newrelic.yml` ou automaticamente com seu `web.xml` `display-name`.

Para verificar se seu aplicativo está reportando o nome correto:

1. Gere temporariamente [o log de nível`finest` ](/docs/agents/java-agent/troubleshooting/generating-logs-troubleshooting).
2. Procure no arquivo de log por `"enabled for"`. Verifique se o nome do seu aplicativo aparece na lista de aplicativos que informam [dados de tempo do browser](/docs/agents/java-agent/instrumentation/page-load-timing-java).
3. Se o nome do aplicativo listado não corresponder ao nome esperado, verifique se você está definindo o mesmo nome do aplicativo em `newrelic.yml` e na configuração `web.xml` `display-name` .
