## Problema

No APM, a guia <DNT>**Threads**</DNT> na [página<DNT>**JVMs**</DNT> ](/docs/agents/java-agent/features/jvm-metrics-page)mostra diversas métricas JMX suportadas pelo servidor do seu aplicativo, como threads ativos, inativos e totais. Para ver a métrica na aba <DNT>**Threads**</DNT> , você deve habilitar o relatório JMX no servidor do seu aplicativo.

Se o relatório JMX não estiver ativado, você verá uma guia <DNT>**Threads**</DNT> vazia com um link para a documentação relevante do servidor do seu aplicativo para ativar o JMX.

## Solução

Para resolver o problema, leia as seções a seguir, dependendo do servidor do seu aplicativo:

<CollapserGroup>
  <Collapser
    id="JBoss"
    title="JBoss"
  >
    O suporte local para JMX está ativado nos servidores do aplicativo JBoss por padrão. Se você alterou as configurações padrão, consulte:

    * [JBoss Enterprise Application Plataform 6.0](https://access.redhat.com/site/documentation/en-US/JBoss_Enterprise_Application_Platform/6/html/Administration_and_Configuration_Guide/index.html)
    * [JBoss AS 7.1](https://docs.jboss.org/author/display/AS71/Admin%20Guide.html#8094211_AdminGuide-JMX)
    * [JBoss AS 6](https://docs.jboss.org/jbossas/6/Admin_Console_Guide/en-US/html/)
  </Collapser>

  <Collapser
    id="Jetty"
    title="Jetty"
  >
    Para ativar o suporte local para JMX em servidores de aplicativos Jetty, configure o MBeanContainer com o arquivo <DNT>**etc/jetty-jmx.xml**</DNT> . Para mais informações, veja:

    * [Jetty 7 e 8](http://wiki.eclipse.org/Jetty/Tutorial/JMX)
    * [Jetty 9](http://www.eclipse.org/jetty/documentation/current/jmx-chapter.html)
  </Collapser>

  <Collapser
    id="Glassfish"
    title="Glassfish"
  >
    Embora o suporte local para JMX esteja habilitado nos servidores do aplicativo Glassfish por padrão, os MBeans que realmente fornecem informações de métrica são padronizados como `OFF`. Para que a métrica adequada seja coletada pelo aplicativo servidor, os seguintes componentes devem ser configurados no <DNT>**Monitoring Service**</DNT>:

    * [Glassfish 4.x](https://glassfish.java.net/docs/4.0/administration-guide.pdf): Os componentes do contêiner Web, serviço HTTP e serviço de transação devem estar configurados para o nível de monitoramento ALTO.
    * [Glassfish 3.1](http://docs.oracle.com/cd/E18930_01/html/821-2416/ablur.html): Os componentes do contêiner Web, serviço HTTP e serviço de transação devem estar configurados para o nível de monitoramento ALTO.
    * [Glassfish 3.0](http://docs.oracle.com/cd/E19798-01/821-1751/ablvr/index.html): Os componentes Web contêiner, Thread Pool e serviço de transação devem estar configurados para o nível de monitoramento BAIXO.
    * [Glassfish 2.1](http://docs.oracle.com/cd/E19879-01/820-4335/ablur/index.html): Os componentes Web contêiner, Thread Pool e serviço de transação devem ser configurados para o nível de monitoramento BAIXO.
  </Collapser>

  <Collapser
    id="Resin"
