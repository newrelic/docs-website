## Problema

Você atualizou o PHP para uma [versão compatível](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release), mas o agente PHP parou de funcionar ou de gerar relatórios.

## Solução

Se você atualizar a instalação do PHP para outra versão principal (por exemplo, 5.3 a 5.4), deverá reinstalar o agente PHP. A atualização do PHP para versões secundárias não requer a reinstalação do agente.

Para reinstalar o agente, siga as [instruções de instalação](/docs/agents/php-agent/installation/php-agent-installation-overview) da sua plataforma ou use o comando `newrelic-install install` para iniciar o <DNT>**[newrelic-install script](/docs/agents/php-agent/advanced-installation/using-newrelic-install-script)**</DNT>.

Se você estiver usando um gerenciador de pacote (por exemplo, `apt-get upgrade`) para atualizar o PHP sem especificar pacotes, você também poderá [atualizar o agente PHP da New Relic](/docs/agents/php-agent/installation/upgrading-php-agent) no processo. Para garantir que a atualização seja aplicada com êxito:

1. Reinicie seu servidor web (Apache, Nginx, PHP-FPM, etc.).
2. Aguarde cinco minutos para ver se o agente começa a reportar.
3. Se o agente ainda não estiver reportando, [verifique a configuração do daemon](/docs/agents/php-agent/troubleshooting/verifying-php-daemon). Se estiver configurado para iniciar no modo externo, você deverá [iniciar manualmente o daemon](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#daemon-external) para que o agente comece a reportar.
