[O New Relic para PHP](/docs/agents/php-agent/getting-started/introduction-new-relic-php) oferece várias maneiras de alterar o nome do aplicativo relatado e exibido no New Relic. Este documento fornece uma visão geral dessas opções de nomenclatura.

Para obter uma visão geral da nomenclatura de aplicativos APM (não específica do PHP), consulte [Nomeie seu aplicativo](/docs/agents/manage-apm-agents/app-naming/name-your-application).

## Definir ou alterar o nome do aplicativo [#naming-options]

A New Relic diferencia os aplicativos pelo nome. Quando você altera o nome relatado por um aplicativo, o New Relic cria uma nova entidade de aplicativo e interrompe a continuidade dos dados entre a entidade nova e a antiga. Para alterar o nome de um aplicativo sem interromper a continuidade dos dados, use a opção interface alias-change.

Aqui estão as maneiras de alterar o nome do seu aplicativo PHP no New Relic:

* [Configure o arquivo `newrelic.ini` usando `newrelic.appname`](/docs/agents/php-agent/configuration/php-agent-configuration#inivar-appname): a opção recomendada e mais comumente usada.
* [Definir configurações INI por diretório](/docs/agents/php-agent/configuration/php-directory-ini-settings): útil quando você tem um servidor web servindo vários aplicativos e deseja ter configurações ou nomes diferentes para eles. (Os procedimentos para [Apache](/docs/agents/php-agent/configuration/php-directory-ini-settings#perdir-apache), [PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#php-fpm_per-dir) e [NGINX em PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#PHP-FPM_nginix) estão documentados.)
* [Alterar alias via interface](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-alias): altera apenas o alias na interface, e não o nome do aplicativo subjacente que está sendo relatado. (Esta opção não interrompe a continuidade dos dados no APM.)
* [Alterar via API](/docs/agents/php-agent/php-agent-api/newrelic_set_appname): uma opção se você tiver acesso para modificar o código do seu aplicativo, concordar em instrumentá-lo diretamente e considerar aceitáveis as implicações de continuidade trace desta chamada.

## Use vários nomes de aplicativos [#multiple]

Você pode usar vários nomes de aplicativos para agregar dados de vários aplicativos com o mesmo nome no APM (às vezes chamado de "acumulação" de seus dados). Para saber mais sobre como funcionam vários nomes de aplicativos, consulte [Usar vários nomes de aplicativos](/docs/agents/manage-apm-agents/app-naming/use-multiple-names-app).

Para PHP, você pode configurar até três nomes de aplicativos. O nome do aplicativo principal é o primeiro, e o segundo e o terceiro nomes são usados para as categorias de agregação de dados mais gerais.
