Continue lendo para saber como desinstalar o agente PHP New Relic. Para obter instruções sobre como desabilitar temporariamente o agente, consulte [Desabilitar o agente](/docs/agents/manage-apm-agents/installation/disable-apm-agent#php).

Este documento explica como desinstalar o [agente PHP New Relic](/docs/agents/php-agent/getting-started/new-relic-php) para os seguintes sistemas:

* <DNT>RedHat</DNT> e <DNT>CentOS</DNT>
* <DNT>Ubuntu</DNT> e <DNT>Debian</DNT>
* Arquivo Tar (qualquer [sistema suportado](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems))

## Desinstalando o agente [#install-overview]

Ao desinstalar o New Relic de um aplicativo PHP, você deve remover o agente New Relic e também remover os arquivos de configuração associados do New Relic.

Para desinstalar o agente PHP do seu sistema:

1. Remova os arquivos do agente do seu sistema:

   <CollapserGroup>
     <Collapser
       id="remove-redhat-centos"
       title="RedHat ou CentOS"
     >
       Execute o script `newrelic-install` sem opções e selecione a opção de desinstalação no menu principal ao invocar o script.
     </Collapser>

     <Collapser
       id="remove-ubuntu-debian"
       title="Ubuntu ou Debian"
     >
       Use `apt-get` para remover o agente:

       ```bash
       apt-get remove newrelic-php5
       ```
     </Collapser>
   </CollapserGroup>

2. Remova todos os arquivos de configuração restantes do New Relic. Use os comandos `newrelic-install` ou `apt-get remove` com a opção `purge` .

   ```bash
   newrelic-install purge
   ```

   OU

   ```bash
   apt-get remove --purge newrelic
   ```

3. Se você usa um gerenciador de pacote, remova os arquivos específicos do gerenciador:

   <CollapserGroup>
     <Collapser
       id="uninstall-redhat-centos"
       title="RedHat ou CentOS"
     >
       Usando `yum`:

       ```bash
       sudo yum remove newrelic-php5
       ```

       Usando <DNT>**`rpm` (32-bit)**</DNT>:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (YOUR VERSION #)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```

       Usando <DNT>**`rpm` (64-bit)**</DNT>:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (VERSION NUMBER)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="uninstall-ubuntu-debian"
       title="Ubuntu ou Debian"
     >
       Usando `apt-get`:

       ```bash
       sudo apt-get purge newrelic-php5
       ```

       Usando <DNT>**`dpkg` (32-bit)**</DNT>:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_i386.deb newrelic-php5_X.X.X.X_i386.deb
       ```

       Usando <DNT>**`dpkg` (64-bit)**</DNT>:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_amd64.deb newrelic-php5_X.X.X.X_amd64.deb
       ```
     </Collapser>
   </CollapserGroup>

4. Reinicie seus servidores web (Apache, Nginx, PHP-FPM, etc.).
