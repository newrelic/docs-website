## Problema

Você instalou o agente PHP New Relic e gerou algum tráfego, mas o agente não envia a primeira transação.

## Solução

Este é o comportamento esperado. Quando o [daemon PHP](/docs/agents/php-agent/getting-started/new-relic-daemon-processes) estiver em seu estado inicial, <DNT>**two**</DNT> a transação deverá ocorrer antes de começar a reportar:

* Com a primeira transação, o daemon PHP adiciona seu aplicativo recém-relatado à sua própria tabela de informações do aplicativo.
* Depois que o aplicativo é adicionado, o daemon PHP reporta a segunda transação ao New Relic.

Recomendação: execute um processo php trivial, como `phpinfo()`, antes de executar os processos que deseja relatar.

## Causa

Este é o comportamento esperado pelo daemon PHP, que inicialmente requer duas transações para serem reportadas.
