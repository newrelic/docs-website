## Problema

Depois de [instalar o agente PHP do New Relic](/docs/agents/php-agent/installation/php-agent-installation-overview), gerar algum tráfego e esperar pelo menos cinco minutos, nenhum dado aparece na interface do usuário do New Relic.

## Solução

Se nenhum dado aparecer depois de você gerar tráfego para seu aplicativo e esperar pelo menos cinco minutos, use [o New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) para detectar automaticamente problemas comuns e sugerir resolução de problemas. Se isso não resolver o problema, tente o seguinte:

        Se você estiver usando uma versão não padrão do PHP, siga os [procedimentos avançados de instalação](/docs/php/php-agent-installation-non-standard-php) para garantir que o instalador padrão possa encontrar sua versão do PHP.
      </td>
    </tr>

        Certifique-se de que seus aplicativos tenham um [nome descritivo e exclusivo.](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name) Por exemplo, se você tiver vários aplicativos com o mesmo nome, como o nome do aplicativo padrão `PHP Application`, os dados de cada um desses aplicativos serão acumulados no nome do aplicativo padrão e poderá parecer que um aplicativo individual não está gerando relatórios.
      </td>
    </tr>

    <tr>
      <td>
        Servidor web
      </td>

      <td>
        Reinicie seu servidor web (Apache, Nginx, PHP-FPM, etc.) e aguarde alguns minutos para que os dados apareçam.
      </td>
    </tr>

    <tr>
      <td>
        `phpinfo()`
      </td>

      <td>
        Marque `phpinfo()` para [verificar se você instalou o agente PHP](/docs/agents/php-agent/troubleshooting/using-phpinfo-verify-php) e se o <InlinePopover type="licenseKey"/>nele está correto.
      </td>
    </tr>

    <tr>
      <td>
        SELinux
      </td>

      <td>
        Se o seu sistema usa SELinux, [configure o SELinux](/docs/agents/php-agent/troubleshooting/data-stops-reporting-while-using-selinux#solution) para funcionar com o agente PHP.
      </td>
    </tr>

    <tr>
      <td>
        Arquivo de log
      </td>

      <td>
        1. Verifique se tanto o agente quanto o daemon estão [gravando em seu arquivo de log](/docs/agents/php-agent/troubleshooting/generating-logs-troubleshooting-php).
        2. Caso contrário, [reinstale o agente](/docs/agents/php-agent/installation/php-agent-installation-overview).
        3. Verifique as permissões do arquivo de log para garantir que o proprietário do log seja o mesmo do usuário do New Relic.
      </td>
