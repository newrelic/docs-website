Quer começar a monitorar sua aplicação PHP com New Relic agora mesmo? Use este guia para encontrar o melhor caminho para começar a enviar os dados do seu aplicativo para a New Relic.

Quer saber mais antes de começar? [Introdução ao APM](/docs/apm/new-relic-apm/getting-started/introduction-apm) e [Introdução ao New Relic para PHP](/docs/apm/agents/php-agent/getting-started/introduction-new-relic-php) são um bom lugar para começar.

## O que você deseja monitor? [#what]

Temos alguns caminhos para instalação do agente PHP, dependendo do que você está monitorando.

<CollapserGroup>
  <Collapser
    id="docker"
    title="Aplicativo implantado em um contêiner docker"
  >
    Você tem algumas opções para instrumentar seu aplicativo docker com o agente PHP.

    * Instale através de nossa [instalação guiada](https://one.newrelic.com/marketplace?state=38549f24-54f0-9e80-e910-0c42a9b493b1) caso queira seu daemon e agente PHP em contêineres diferentes. Se você ainda precisar criar seu daemon, siga as etapas adicionais no [Docker e em outros ambientes de contêiner](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#daemon-container)
    * Para configurar o agente e o daemon no mesmo contêiner, siga [estas etapas](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#install-same-container). Isso envolve o download do [arquivo tar](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file/) para o agente PHP.
  </Collapser>

  <Collapser
    id="redhat-centos"
    title="Implantação de aplicativo com RedHat, CentOS ou AWS Linux"
  >
    * Nossa [instalação guiada](https://one.newrelic.com/marketplace?state=38549f24-54f0-9e80-e910-0c42a9b493b1) usa `yum` para instalar o agente, assim como nossa instalação manual. A instalação guiada é uma ótima opção se você deseja mais visibilidade e certeza sobre onde está o processo de instalação.
    * Nossa [instalação manual](/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos#manager-tarball) orienta você no mesmo processo, mas permite gerenciar cada etapa a partir da linha de comando.
  </Collapser>

  <Collapser
    id="debian-ubuntu"
    title="Implantar aplicativo com Debian ou Ubuntu"
  >
    * Nossa [instalação guiada](https://one.newrelic.com/marketplace?state=38549f24-54f0-9e80-e910-0c42a9b493b1) usa `apt` para instalar o agente, assim como nossa instalação manual. A instalação guiada é uma ótima opção se você deseja mais visibilidade e certeza sobre onde está o processo de instalação.
    * Nossa [instalação manual](/docs/apm/agents/php-agent/installation/php-agent-installation-ubuntu-debian) orienta você no mesmo processo, mas permite gerenciar cada etapa a partir da linha de comando.
  </Collapser>

  <Collapser
    id="tar"
    title="Implantar aplicativo com outros sabores Unix (usando arquivo tar)"
  >
    Se você estiver usando uma versão diferente do Unix, instale o agente usando nosso [arquivo PHP tar](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file).
  </Collapser>

  <Collapser
    id="lambda"
    title="Implantação de aplicativo com AWS Lambda"
  >
    Se seu aplicativo for implantado com AWS Lambda, siga nossas [etapas autoguiadas](https://one.newrelic.com/marketplace?state=3daeff79-c8e3-4375-f0e0-57d417ec07ab) para instrumentar seu aplicativo com o agente PHP.
  </Collapser>

  <Collapser
    id="hosting-providers"
    title="Aplicativo hospedado em AWS, Heroku, Magento ou outros provedores de hospedagem"
  >
    Por meio de parcerias com a New Relic, você pode executar o agente PHP se seu aplicativo web estiver hospedado em um provedor terceirizado.

    * Para aplicativos hospedados na AWS, instale a [integração do Amazon CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup). Para saber mais sobre a New Relic e a integração da AWS, leia [Introdução à integração da AWS](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations/).
    * Para um aplicativo PHP hospedado no Heroku, instale o [complemento New Relic](/docs/apm/agents/php-agent/advanced-installation/php-agent-heroku/#installing)
    * Se você é um comerciante do Magento, [siga estas etapas](/docs/accounts/install-new-relic/partner-based-installation/log-install-new-relic-partners/#magento) para implantar o agente PHP em sua instalação do Magento.
  </Collapser>
</CollapserGroup>
