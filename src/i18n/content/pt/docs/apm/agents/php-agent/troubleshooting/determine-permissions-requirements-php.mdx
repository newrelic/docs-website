## Problema

Você precisa determinar quais permissões de sistema você precisa para instalar e usar o agente PHP New Relic.

## Solução

Para a maioria dos casos de uso, você precisará de acesso root para instalar o New Relic for PHP. Isso ocorre porque o processo de instalação requer privilégios disponíveis apenas para o usuário root. Esses privilégios incluem:

* Acesso de leitura/gravação ao diretório de extensões PHP

* Acesso de leitura/gravação aos dados de configuração do PHP, no diretório de configuração do PHP (preferencial) ou em

  <DNT>
    **php.ini**
  </DNT>

  .

* Capacidade de reiniciar o servidor web ou pool de processos PHP.

A execução do agente não requer acesso root. O agente precisa de acesso de leitura/gravação aos seguintes arquivos (a localização desses arquivos pode ser [personalizada nas configurações do agente PHP](/docs/php/php-agent-phpini-settings)):

* [Um arquivo no qual armazenar o arquivo de log do agente](/docs/php/php-agent-phpini-settings#inivar-daemon-logfile)
* [Um arquivo no qual armazenar o endpoint do soquete do agente](/docs/php/php-agent-phpini-settings#inivar-daemon-port)
* [Um arquivo no qual registrar o ID do processo do agente](/docs/php/php-agent-phpini-settings#inivar-daemon-pidfile)
