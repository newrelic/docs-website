## Problema

<DoNotTranslate>**newrelic**</DoNotTranslate> a seção do módulo não aparece em `phpinfo()`.

## Solução

Uma seção de descrição do módulo `newrelic` ausente na página `phpinfo()` indica que o arquivo <DoNotTranslate>**`newrelic.so`**</DoNotTranslate> pode não ter sido instalado no diretório de extensões. Para determinar se o módulo New Relic PHP existe:

1. Na sua [página`phpinfo()` ](/docs/php/php-kb-103), procure a seção

   <DoNotTranslate>**PHP Core**</DoNotTranslate>

   .

2. Para identificar o diretório de onde suas extensões PHP são carregadas, encontre o valor local da configuração `extension_dir` .

3. Verifique se `newrelic.so` está presente no diretório especificado por `extension_dir`.

Se o arquivo `newrelic.so` não estiver presente em seu diretório de extensões:

* Siga [os procedimentos de instalação PHP não padrão](/docs/php/php-agent-installation-non-standard-php) da New Relic.

  OU

* Vincule manualmente a versão correta de `/usr/lib/newrelic-php5/agent/x32` para 32 bits ou `/usr/lib/newrelic-php5/agent/x64` para 64 bits. Ou, se você instalou a partir de um arquivo tar, copie ou crie um link do diretório <DoNotTranslate>**agent**</DoNotTranslate> onde você extraiu o arquivo tar.
