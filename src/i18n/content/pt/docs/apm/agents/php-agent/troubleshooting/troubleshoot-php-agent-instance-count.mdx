## Problema

Na página [<DNT>**Summary**</DNT>do APM](/docs/apm/applications-menu/monitoring/apm-overview-dashboard) do agente PHP, qual é a contagem de instâncias do agente PHP na tabela <DNT>**Hosts**</DNT> ? Por que esses números mudam e por que parece haver uma inconsistência neles?

## Causa

Cada servidor listado na página <DNT>**Summary**</DNT> inclui o número de instância; por exemplo, <DNT>**42 app instances**</DNT>. A contagem de instâncias indica o número máximo de descritores de arquivos abertos usados pelo agente PHP na última coleta relatada. Esses descritores de arquivo são soquetes de E/S abertos e refletem os canais de comunicação agente/daemon.

O número de instâncias aumentará se a linha de comando, o plano de fundo ou processos externos estiverem em execução. O número também pode ser influenciado por pesadas taxas de transferência. Por exemplo, um servidor Apache pode ser configurado para aumentar o pool de conexões para acomodar o aumento de carga.

Uma diferença nos valores esperados entre servidores pode refletir tarefas em segundo plano executadas em um servidor, mas não no outro. Uma contagem inesperada também pode indicar um balanceador de carga configurado incorretamente.
