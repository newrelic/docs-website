Em geral, os planos básicos de hospedagem não permitem a instalação padrão do agente PHP da New Relic, porque são necessárias permissões de administrador (root) para instalá-lo. Seu provedor de hospedagem pode permitir ou fornecer a instalação de aplicativos/estruturas da web comuns (como WordPress, Drupal, Joomla, etc.), mas a instalação do agente PHP ainda requer permissões não permitidas em muitos planos básicos.

Se necessário, entre em contato com o desenvolvedor/provedor de hospedagem do seu site para obter assistência técnica.

## Serviços de hospedagem compartilhada [#hosts]

Em geral, provedores de hospedagem compartilhada (que não fornecem acesso root à linha de comando) e provedores de hospedagem que não possuem sistema operacional tradicional (como PaaS) não permitem que o usuário instale o agente New Relic através de canais típicos. Por exemplo, esses provedores de hospedagem **não** oferecem suporte à integração New Relic para determinado nível de assinatura:

* DreamHost: hospedagem compartilhada na web
* Blog hospedado em WordPress.com
* GoDaddy: Economia, Deluxe, Ultimate, Hospedagem de Blogs WordPress
* Hospedagem HostGator

Antes de instalar o agente PHP da New Relic, certifique-se de que seu nível de serviço de hospedagem seja compatível com a New Relic. Certifique-se também de ter permissões de administrador para procedimentos de instalação a partir da linha de comando no nível raiz.

## Dicas de instalação e resolução de problemas [#installation]

Para ajudar a garantir que seu aplicativo possa ser integrado com êxito ao New Relic:

1. Certifique-se de que seu sistema atenda [aos requisitos de compatibilidade PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) da New Relic.

   <Callout variant="important">
     O sistema operacional Windows não é compatível.
   </Callout>

2. Siga [os procedimentos de instalação do PHP](/docs/agents/php-agent/installation/php-agent-installation-overview) da New Relic.

3. Se o seu sistema atender aos requisitos de compatibilidade do New Relic, mas a instalação não for bem-sucedida, reúna as seguintes informações:

   * Etapas exatas que você tentou
   * Resultados, incluindo mensagens de sucesso ou falha
   * Seu sistema operacional e versão (por exemplo, Ubuntu Linux versão 12.04 de 64 bits)
   * Seu provedor de serviço de hospedagem e nível de assinatura

4. Se aplicável, entre em contato com os desenvolvedores do seu site para obter assistência.
