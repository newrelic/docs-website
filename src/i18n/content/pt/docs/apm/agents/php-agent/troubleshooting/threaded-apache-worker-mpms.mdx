## Problema

O agente PHP New Relic não oferece suporte a nenhum modelo de multiprocessamento (MPM) do Apache que envolva threading.

## Solução

A equipe de desenvolvimento da linguagem PHP [desencoraja o uso de um MPM encadeado com o Apache](http://www.php.net/manual/en/install.unix.apache2.php "O link abre em uma nova janela") e optamos por não implementar suporte para MPMs encadeados no agente PHP por motivos de estabilidade semelhantes. Isso inclui os MPMs <DNT>**worker**</DNT> e <DNT>**event**</DNT> .

Se o MPM <DNT>**prefork**</DNT> padrão não encadeado não for adequado às suas necessidades por motivos de desempenho, sugerimos usar <DNT>**mod_fcgid**</DNT> e <DNT>**php-fpm**</DNT> para usar PHP via FastCGI.
