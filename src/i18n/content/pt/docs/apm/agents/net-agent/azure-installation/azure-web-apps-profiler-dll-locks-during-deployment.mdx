## Problema

Ao implantar uma nova versão do agente .NET da New Relic (`NewRelic.Profiler.dll`) para seu [aplicativo Web do Microsoft Azure](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps), uma mensagem de erro impede que você continue até interromper ou reiniciar o processo.

## Solução

Normalmente, o processo é interrompido e, em seguida, os ativos são substituídos pelos ativos que o New Relic empacota na raiz do site (localizada em `D:\Home\site\wwwroot\newrelic`) durante uma implantação. Para resolver o problema, tente uma destas soluções.

<CollapserGroup>
  <Collapser
    id="stop-dll-deploy"
    title="Impedir a liberação do arquivo"
  >
    Esta solução está disponível para o seu Aplicativo Web do Azure se você estiver tentando implantar uma nova versão do agente .NET sobre uma versão existente:

    1. Pare seu aplicativo da web para liberar o arquivo `.dll` .
    2. Permita que a atualização prossiga.
    3. Assim que o implante for concluído, reinicie a instância.
  </Collapser>

  <Collapser
    id="site-extension"
    title="Use a extensão do site New Relic"
  >
    Você pode instalar o agente .NET da New Relic em seu aplicativo Web do Azure usando a [extensão de site da New Relic](https://www.siteextensions.net/packages/NewRelic.Azure.WebSites/). (Você só precisará instalar a extensão uma vez.)

    1. Navegue até <DNT>**https&#x3A;//yoursite.scm.azurewebsites.net/**</DNT> e selecione <DNT>**Site Extensions**</DNT>.

    2. Em <DNT>**Gallery**</DNT>, selecione adicionar <Icon name="fe-plus"/> ícone e selecione <DNT>**New Relic**</DNT>.

    3. Siga as instruções para [adicionar manualmente as chaves de configuração](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps) ao seu aplicativo Web e use a opção que melhor atenda às suas necessidades.

       Para atualizar a extensão do site: No [Portal do Azure](https://portal.azure.com), selecione <DNT>**WebApps > (select your site) > All Settings > Extensions > New Relic**</DNT>.
  </Collapser>
</CollapserGroup>
