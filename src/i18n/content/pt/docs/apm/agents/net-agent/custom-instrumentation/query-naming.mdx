New Relic .NET agente instrumento e agrupa a consulta SQL da sua aplicação. Normalmente, o agente .NET atribuirá automaticamente um nome a uma consulta com base nas propriedades da consulta. Como alternativa, você pode atribuir um nome personalizado a uma consulta.

## Como nomear uma consulta [#name-query]

Todas as versões SQL suportadas pelo nosso agente .NET podem receber um nome personalizado.

Para dar um nome a uma consulta, adicione um comentário ao texto da consulta com um formato como `/* NewRelicQueryName: myQueryName */`. Aqui está um exemplo de como adicionar isso ao código de um aplicativo:

```cs
var connection = new MySqlConnection(mySqlConnectionString);
var command = new MySqlCommand("/* NewRelicQueryName: My query name */ SELECT date FROM dates", connection);
```

## Encontre consulta nomeada [#find-named-query]

Seu banco de dados e dados de consulta podem ser encontrados na [página da interface do usuário<DNT>**Databases**</DNT> ](/docs/apm/apm-ui-pages/monitoring/databases-page-view-operations-throughput-response-time).
