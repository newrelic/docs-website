O agente .NET da New Relic oferece suporte tanto ao .NET Framework quanto ao ..NET Core. Este documento descreve a compatibilidade e o suporte para tempos de execução, estrutura e biblioteca do .NET.

O agente inclui instrumentação integrada para algumas das partes mais populares do ecossistema .NET, incluindo estrutura, banco de dados e sistemas de enfileiramento de mensagens. Depois de fazer [download e instalar](/install/dotnet) o agente, ele será executado no processo do monitor. O agente não cria um processo ou serviço separado.

Para framework e biblioteca que não são [instrumentados automaticamente](#instrumented-features), você pode estender o agente com [instrumentação personalizada .NET](/docs/agents/net-agent/custom-instrumentation/introduction-net-custom-instrumentation).

Quer experimentar nosso agente .NET? [Crie uma conta New Relic](https://newrelic.com/signup) gratuitamente! Não é necessário cartão de crédito.

          O agente .NET não coleta dados sobre processos do servidor de armazenamento de dados. Ele coleta apenas dados do uso da biblioteca cliente do armazenamento de dados. Para monitorar diretamente os processos do servidor de armazenamento de dados, use o agente New Relic Infrastructure com [integração no host](/docs/infrastructure/host-integrations/get-started/introduction-host-integrations/).

          Por padrão, o agente .NET não captura parâmetros SQL para procedimentos armazenados ou consultas parametrizadas em um trace de consulta. Para visualizar estes parâmetros SQL, deve-se habilitar o recurso de coleta no [agente de configuração](/docs/apm/agents/net-agent/configuration/net-agent-configuration/#datastore-tracer-query-parameters).

          A coleta [de detalhes da instância](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues) para datastores compatíveis está habilitada por padrão. Para solicitar informações em nível de instância de armazenamentos de dados não listados atualmente, obtenha suporte em [support.newrelic.com](https://support.newrelic.com).

          Se seu armazenamento de dados não estiver listado aqui, você poderá adicionar instrumentação personalizada usando o método `RecordDatastoreSegment` na [API do agente .NET](/docs/apm/agents/net-agent/net-agent-api/net-agent-api/#ITransaction).

      ## Recurso indisponível [#unavailable-core]

      Os seguintes recursos não estão disponíveis para o agente .NET:

      * O agente .NET não suporta [trim implantação e executáveis independentes](https://docs.microsoft.com/en-us/dotnet/core/deploying/trim-self-contained), porque o compilador pode potencialmente cortar assemblies dos quais o agente depende.
      * [O Infinite Tracing](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing) não é compatível com Alpine Linux devido a um [problema de compatibilidade GRPC](https://github.com/grpc/grpc/issues/21446). Consulte [este problema do agente](https://github.com/newrelic/newrelic-dotnet-agent/issues/289) para obter mais informações.
      * O agente .NET não suporta [a implantação Native Ahead of Time (AOT)](https://learn.microsoft.com/en-us/dotnet/core/deploying/native-aot/?tabs=net8plus%2Cwindows) para o aplicativo .NET porque a compilação just-in-time (JIT) é necessária para que o agente funcione corretamente.

Para executar um aplicativo no .NET Framework versão 4.0 ou inferior, você deve executar uma versão do New Relic .NET Agent inferior a 7.0. Para obter mais informações e procedimentos de download, consulte [Suporte técnico para .NET Framework 4.0 ou inferior](/docs/agents/net-agent/troubleshooting/technical-support-net-frameworks-40-or-earlier).

Se o seu aplicativo .NET não durar tanto, você pode definir o atributo `sendDataOnExit` do elemento `service` como `true` no arquivo `newrelic.config` .
        </Collapser>

        <Collapser className="freq-link" id="app-web-servers-framework" title="Servidores de aplicativos/web">
          Você deve usar um destes servidores de aplicativo/web:

          * IIS
          * OWIN auto-hospedado
          * WCF auto-hospedado
          * Francelho
          * Kestrel com proxy reverso do IIS via AspNetCoreModule
          * Kestrel com proxy reverso do IIS via AspNetCoreModuleV2

        </Collapser>

        </Collapser>

          * Seja membro do grupo administrador.

          * No sistema operacional mais recente, forneça credenciais de elevação quando solicitado.

          <DNT>**Recommendation:**</DNT>

          * Restrinja as permissões para o arquivo `newrelic.config` e conceda acesso de leitura/gravação somente ao proprietário do processo monitor.

          * Revise as permissões do log criado pelo agente para minimizar o número de usuários com acesso e seus privilégios.
        </Collapser>

        </Collapser>

Executar nosso agente .NET junto com outro software de monitoramento resultará em um [conflito de perfil](/docs/apm/agents/net-agent/troubleshooting/profiler-conflicts/).
