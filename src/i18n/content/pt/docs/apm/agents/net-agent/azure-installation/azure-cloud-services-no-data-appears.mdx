## Problema

Você instalou o agente .NET da New Relic por meio [dos serviços na nuvem do Microsoft Azure](/docs/agents/net-agent/azure-installation/install-app-azure-cloud-services) e, em seguida, instalou o pacote NuGet. Depois de gerar algum tráfego e esperar alguns minutos para que os dados comecem a ser coletados para seu aplicativo, os dados ainda não aparecem na interface do usuário do APM.

## Solução

Para que o agente .NET seja iniciado e anexado ao seu aplicativo, pelo menos uma métrica deve ser coletada. Se você tiver uma chamada externa ou uma chamada de banco de dados desse instrumento New Relic, seu aplicativo reportará dados para sua [página APM <DNT>**Summary**</DNT> ](/docs/apm/applications-menu/monitoring/apm-overview-page). Além disso, informações sobre chamadas externas aparecerão nos [mapas de serviços](/docs/data-analysis/user-interface-functions/view-your-data/service-maps-visualize-monitor-apps-entire-architecture) do seu aplicativo.

Sem instrumentação personalizada, as funções `Worker` relatam apenas chamadas de banco de dados e chamadas externas. instrumentado a `Worker` Role é semelhante a [instrumentado a custom application](/docs/agents/net-agent/instrumentation/instrumenting-non-iis-apps).) Para visualizar outros detalhes, você deve coletar métricas personalizadas utilizando [a API .NET](/docs/agents/net-agent/features/net-agent-api) da New Relic e visualizá-las em um [dashboard personalizado](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards).
