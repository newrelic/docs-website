Para equilibrar a necessidade de log enquanto considera o uso do disco, cada agente .NET limitará o uso do disco a 250 MB usando a rolagem log . O agente primeiro log no arquivo `newrelic_agent_UNIQUENAME.log` e criará o arquivo se ele não existir. Quando o arquivo atingir 50 MB, o agente irá:

1. Crie um novo arquivo de log.
2. Transfira cada arquivo de log existente para um novo nome numerado sequencialmente (até quatro arquivos arquivados).
3. Exclua o quarto arquivo.

Para rolar o arquivo de log, o antigo `newrelic_agent_UNIQUENAME.log` vira o novo `newrelic_agent_UNIQUENAME.log(1)`. Então, o antigo `newrelic_agent_UNIQUENAME.log(1)` se torna o novo `newrelic_agent_UNIQUENAME.log(2)` e assim por diante. O antigo `newrelic_agent_UNIQUENAME.log(4)` foi excluído.
