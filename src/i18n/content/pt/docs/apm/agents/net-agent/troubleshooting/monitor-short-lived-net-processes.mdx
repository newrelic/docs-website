## Problema

Você tem um processo .NET que é executado por menos de 60 segundos e não está vendo os dados relatados para esse processo.

## Solução

O APM [ciclo de coleta](/docs/new-relic-solutions/get-started/glossary/#harvest-cycle) é de 60 segundos. Por esse motivo, processos executados por períodos mais curtos podem não reportar dados. Para corrigir isso, use os [elementos de serviço](/docs/apm/agents/net-agent/configuration/net-agent-configuration/#service) `sendDataOnExit` e `sendDataOnExitThreshold` em seu [newrelic.config](/docs/apm/agents/net-agent/configuration/net-agent-configuration/#config-options-precedence). Você deve definir o valor `sendDataOnExitThreshold` para um tempo menor do que o processo leva para ser concluído.
