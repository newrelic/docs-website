No New Relic, <DNT>**attributes**</DNT> são pares de valores principais contendo informações que determinam as propriedades de um [evento](/docs/accounts-partnerships/getting-started-new-relic/glossary#event) ou [transação](/docs/accounts-partnerships/getting-started-new-relic/glossary#transaction). Esses pares de valor principal podem ajudá-lo a obter maiores insights sobre sua aplicação e [consultar seus dados](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder).

## Ver e usar atributo [#locations]

Tanto [o atributo APM padrão](/docs/insights/new-relic-insights/decorating-events/apm-default-attributes-insights#transaction-event) quanto [o atributo personalizado](/docs/agents/manage-apm-agents/agent-data/collect-custom-attributes) para sua aplicação C aparecem em:

* APM [trace da transação](/docs/apm/transactions/transaction-traces/introduction-transaction-traces) e [análise de erros](/docs/apm/applications-menu/error-analytics/introduction-error-analytics)
* [Evento APM](/docs/insights/explore-data/attributes/apm-default-attributes-insights)

## Atributo específico C [#attributes]

<Callout variant="important">
  Antes de criar um atributo personalizado, revise a lista de [termos reservados da New Relic usados pelo NRQL](/docs/insights/insights-data-sources/custom-data/insights-custom-data-requirements-limits#reserved-words). Caso contrário, poderão ocorrer resultados inesperados.
</Callout>

Para adicionar atributo personalizado à sua aplicação C, chame uma das [funções atributo na API C SDK](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#abfe6f64a8eec7d60d8588f8969781d34); por exemplo, `newrelic_add_attribute_double()`. O nome da chave do seu atributo personalizado depende do que você especifica ao chamar a função.
