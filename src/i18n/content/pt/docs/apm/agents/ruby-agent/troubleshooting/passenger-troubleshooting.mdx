Leia as dicas de resolução de problemas a seguir para problemas comuns ao usar o agente Ruby e o Passenger.

<CollapserGroup>
  <Collapser
    id="agent"
    title="Reiniciando o agente"
  >
    Se tudo o que você fizer for editar o arquivo do Passageiro `tmp/restart.txt` , ele não atenderá o novo agente. Ao atualizar o agente e reiniciar o Passenger, você deve reiniciar o Apache para que a alteração entre em vigor.
  </Collapser>

  <Collapser
    id="logs"
    title="Arquivo de log ausente"
  >
    Para Passageiro, o arquivo de log é `newrelic_agent.passenger.log`. Recomendação: Para o usuário Apache, forneça permissões de gravação para todo o diretório `/log`.
  </Collapser>

  <Collapser
    id="developer_mode"
    title="Modo desenvolvedor"
  >
    Ao usar o Passenger no modo desenvolvedor, às vezes você pode perceber que a lista de solicitações pode desaparecer, ou que parece que faltam algumas solicitações. Além disso, ao selecionar uma solicitação, você poderá ver uma mensagem de erro indicando que o trace não está disponível, sugerindo que talvez você tenha reiniciado.

    Isso ocorre quando o Passenger gera mais de um processo manipulador. O trace da transação é colocado em um dos processos, e se você se conectar ao outro, não verá.

    A solução alternativa é definir o número máximo de instância como 1 usando o parâmetro [PassengerMaxPoolSize](http://www.modrails.com/documentation/Users%20guide%20Apache.html#_passengermaxpoolsize_lt_integer_gt) . Além disso, se você atualizar várias vezes, terá uma boa chance de recuperar as solicitações.
  </Collapser>
</CollapserGroup>
