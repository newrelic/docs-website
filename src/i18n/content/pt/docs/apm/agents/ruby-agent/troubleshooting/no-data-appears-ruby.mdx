## Problema

Depois de [instalar o agente Ruby do New Relic](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation), gerar algum tráfego e esperar cinco minutos, nenhum dado aparece na interface do usuário do New Relic.

## Solução

Para solucionar problemas de dados ausentes:

1. Verifique seu arquivo de configuração:

   * Certifique-se de que seu arquivo de configuração esteja instalado em `config/newrelic.yml` no diretório do seu aplicativo.

   * Certifique-se de que seu arquivo de configuração seja YML válido.

   * Para fazer download de uma cópia limpa deste arquivo com seu

     <InlinePopover type="licenseKey"/>

     pré-preenchido, vá para

     <DNT>
       **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (user menu) > Account settings > Download a clean configuration file**
     </DNT>

     .

2. Reinicie seu servidor web.

3. Use [o New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) para tentar identificar automaticamente o problema.

4. Examine o arquivo de log do agente em `log/newrelic_agent.log` em busca de erros ou avisos que aparecem ao reiniciar os processos do servidor do aplicativo. Se esse arquivo de log não estiver presente ou não estiver sendo gravado, consulte [Nenhum arquivo de log](/docs/agents/ruby-agent/troubleshooting/no-log-file-ruby).

5. Verifique no log do agente New Relic se há uma entrada que tenha `INFO : Reporting to:` seguida por um URL. Certifique-se de que o URL corresponda à conta e ao aplicativo que você está visualizando na interface do New Relic.

6. Certifique-se de que o agente New Relic esteja configurado para iniciar no ambiente em que você está implantado. Para obter mais informações, consulte [Controlando quando o agente Ruby é iniciado](/docs/agents/ruby-agent/troubleshooting/controlling-when-ruby-agent-starts).

7. Se nenhum erro for óbvio no nível `info` , [gere um log de depuração detalhado](/docs/agents/ruby-agent/troubleshooting/generating-logs-troubleshooting-ruby).

8. Tente [atualizar para a versão mais recente](/docs/agents/ruby-agent/installation-configuration/updating-deprecated-ruby-agents#update) do gem <DNT>**newrelic_rpm**</DNT> .

9. Se você estiver usando algo diferente do Rails, defina `RACK_ENV` para um ambiente que aparece no arquivo `newrelic.yml` , como desenvolvimento ou produção.
