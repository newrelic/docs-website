Quer começar a monitorar sua aplicação Node.js com New Relic agora mesmo? Use este guia para encontrar o melhor caminho para começar a enviar os dados do seu aplicativo para a New Relic.

Quer saber mais antes de começar? [Introdução ao APM](/docs/apm/new-relic-apm/getting-started/introduction-apm) e [Introdução ao New Relic para Node.js](/docs/apm/agents/nodejs-agent/getting-started/introduction-new-relic-nodejs) são um bom lugar para começar. Se quiser saber mais sobre o processo geral de instalação, consulte nossa [visão geral da instalação do Node.js.](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent)

## O que você deseja monitor? [#what]

<Callout variant="tip">
  Seu aplicativo está rodando em um cluster do Kubernetes? Experimente nosso método de instalação usando o [operador agenteKubernetes ](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator/).
</Callout>

Temos alguns caminhos para instalar o agente Node.js, dependendo do que você está monitorando.

<CollapserGroup>
  <Collapser
    id="docker"
    title="Aplicativo implantado em um contêiner docker"
  >
    Para instalar o agente em um contêiner Docker, siga [estas etapas autoguiadas](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent-docker) usando a linha de comando.

    Este processo orienta você na adição do agente a `package.json` e na solicitação do New Relic no nível do aplicativo. No final, você adicionará sua chave de licença e o nome do aplicativo a `docker run` para que o agente seja executado com seu aplicativo docker .
  </Collapser>

  <Collapser
    id="server"
    title="Implantar aplicativo em um servidor"
  >
    Temos algumas opções diferentes para aplicativos hospedados em um servidor.

    * Nossa [instalação guiada](https://one.newrelic.com/marketplace/install-data-source?state=f535d53b-5b84-cd48-80cf-61704ad02e29) divide a instalação em etapas. Requer alguma interação com a linha de comando.

    * Nosso documento [Instalar o agente Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent) usa exclusivamente a linha de comando. Você pode visualizar o mesmo processo em nosso repositório [de agente Node.js.](https://github.com/newrelic/node-newrelic#installation)

    * Para aplicativos que usam o gerenciador de processos PM2, siga a [instalação guiada](https://one.newrelic.com/marketplace/install-data-source?state=ba2c3ecb-defb-e99a-c4be-47d786690d2a). Se você estiver usando um gerenciador de processos que não seja PM2, escolha a opção de instalação guiada ou manual.

      Quando estiver pronto para monitor seu aplicativo, adicione `-r newrelic` ao comando de inicialização do seu aplicativo para exigir o agente antes de outros módulos, por exemplo `node -r newrelic your-program.js`. Isso é especialmente importante se você não tiver acesso root ao seu aplicativo.
  </Collapser>

  <Collapser
    id="lambda"
    title="Implantação de aplicativo com AWS Lambda"
  >
    Para instalar um agente para sua função AWS do Lambda, nossas [etapas autoguiadas](https://one.newrelic.com/marketplace/install-data-source?state=9aa3d004-6f13-1477-604e-b6a2d6bb992d) fornecem um script de configuração que configura automaticamente a AWS para você.

    Caso você tenha preferência pela instalação manual via linha de comando, atualmente temos a opção de [instrumentação legada para monitoramento Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-legacy/#node).
  </Collapser>
</CollapserGroup>
