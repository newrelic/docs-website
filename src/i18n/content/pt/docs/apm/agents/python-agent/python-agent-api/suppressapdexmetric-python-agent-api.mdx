## Sintaxe

```py
newrelic.agent.suppress_apdex_metric(flag=True)
```

Impede que a transação atual contribua para o Apdex.

## Descrição

Esta chamada suprime a geração da [métrica Apdex](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) para uma transação da web. Você pode querer fazer isso quando uma transação de longa duração afeta frequentemente a pontuação média do Apdex do seu aplicativo.

Para cancelar a supressão de uma transação suprimida anteriormente, defina o sinalizador como `False`.

<Callout variant="tip">
  Você também pode suprimir a métrica Apdex no dicionário de ambiente WSGI. Para fazer isso, defina a chave `newrelic.suppress_apdex_metric` para a solicitação específica no dicionário de ambiente WSGI transmitido pelo servidor WSGI para o aplicativo WSGI que está sendo monitorado.
</Callout>

## Valores de retorno

Nenhum.

## Exemplos

### Desative o Apdex para transações específicas [#apdex-example]

Se você tiver funções frequentes e de longa execução em seu aplicativo que estão fazendo com que seu Apdex médio caia, você pode chamar `suppress_apdex_metric` onde a transação está sendo gerada:

```py
import newrelic.agent
newrelic.agent.suppress_apdex_metric()
```
