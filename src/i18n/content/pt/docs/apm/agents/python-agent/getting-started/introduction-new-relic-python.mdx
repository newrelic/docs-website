Nosso agente Python monitora seu aplicativo Python para ajudá-lo a identificar e resolver [problemas de desempenho](#monitor-performance). Você também pode estender seu monitoramento de desempenho para coletar e analisar [dados de negócios](#business-data) para ajudá-lo a melhorar a experiência dos clientes e tomar decisões de negócios orientadas por dados. Com opções flexíveis de [instrumentação personalizada e API](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation), o agente Python oferece vários blocos de construção para personalizar os dados que você precisa do seu aplicativo.

Nosso Python funciona com uma ampla variedade de [estruturas web e mecanismos de hospedagem](/docs/agents/python-agent/getting-started/instrumented-python-packages), incluindo Django, Gunicorn, WSGI, CherryPy, uWSGI e muito mais. Você também pode instalar o agente Python em um [ambiente flexível do Google App Engine](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment).

## monitor o desempenho do aplicativo [#monitor-performance]

Depois de [instalar](#installation) o agente Python, ele começa a coletar dados sobre seu aplicativo. Você pode visualizar os dados como gráficos e tabelas na [interface do New Relic](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one).

<DNT>
  **View the big picture of your app:**
</DNT>

* monitor [o Apdex (satisfação do usuário)](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) do seu aplicativo.
* Obtenha um resumo de alto nível do seu aplicativo com [a página Resumo](/docs/apm/applications-menu/monitoring/apm-overview-page).
* Habilite [distributed tracing](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) para ver a atividade em uma arquitetura com muitos serviços.
* Instale [o monitoramento de infraestrutura](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure) e visualize dados detalhados de servidor/host para seu aplicativo.

<DNT>
  **Find errors and problems quickly:**
</DNT>

* Acompanhe [o principal da transação](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events) específico para o seu negócio.
* Crie [dashboard personalizado](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards) para métricas importantes.
* [alerte](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) sua equipe quando ocorrer um erro ou problema antes que afete seu usuário.
* Veja o desempenho [após uma implantação](/docs/apm/applications-menu/events/deployments-page).

<DNT>
  **Drill down into performance details:**
</DNT>

* Examine o nível do código [trace da transação](/docs/apm/transactions/transaction-traces/transaction-traces).
* Examine [consulta ao banco de dados trace](/apm/transactions/transaction-traces/troubleshoot-sql-statements).
* Examine [o rastreamento de erros](/docs/apm/applications-menu/events/view-apm-error-analytics).
* Use [sessões do thread profiler](/docs/apm/applications-menu/events/thread-profiler-tool) para ver o rastreamento de pilha detalhado de threads de amostra

<DNT>
  **View logs for your APM and infrastructure data:**
</DNT>

Reúna os dados do seu log e da aplicação para tornar a resolução de problemas mais fácil e rápida. Não há necessidade de mudar para outra página de interface.

* Com [os logs contextualizados](/docs/logs/logs-context/configure-logs-context-python/), você pode ver a mensagem do log relacionada aos seus erros e rastrear diretamente na interface do seu aplicativo.
* Você também pode ver o logs contextualizados dos [dados da sua infraestrutura](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), como cluster do Kubernetes.

## Instale o agente Python [#installation]

Antes de instalar o agente Python, certifique-se de que seu sistema atenda aos [requisitos do sistema](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent). Você também deve criar uma [conta New Relic](https://newrelic.com/signup).

Oferecemos suporte imediato a uma série de estruturas e bibliotecas da web, incluindo Django, WSGI e Gunicorn. Se você usar uma das [estruturas da web suportadas](/docs/agents/python-agent/getting-started/instrumented-python-packages), a instalação será fácil. Se você usar uma framework não compatível , o processo envolverá algumas adições ao código do seu aplicativo e/ou aos arquivos do servidor web.

Para um processo de instalação rápido e simples que funcionará para a maioria das configurações, siga estas etapas simples:

1. Baixe e instale o pacote Python.
2. Crie um arquivo de configuração.
3. Integre o agente Python ao seu aplicativo.

## monitor scripts não-web, tarefas em segundo plano e funções [#non-web]

O agente Python também permite monitor scripts não web, processos de trabalho, tarefas e funções. O processo de instalação dessas transações fora da web é semelhante ao usado para um aplicativo web, com uma grande diferença: em vez de passar pelo processo de integração padrão descrito nas [instruções de instalação](/docs/agents/python-agent/getting-started/python-agent-quick-start), você "embrulharia" manualmente qualquer função que desejasse monitor. Para obter mais informações, consulte [Tarefas e processos não web](/docs/agents/python-agent/supported-features/python-background-tasks). Para obter instruções sobre tarefas de monitoramento do Celery, consulte [Tarefas em segundo plano do Celery](/docs/agents/python-agent/back-end-services/python-agent-celery).

## O que vem a seguir após a instalação? [#instrumentation]

Depois de colocar o agente em funcionamento, algumas próximas etapas sugeridas são:

* Explore seus dados e sinta-se confortável com a interface do usuário.
* Leia nossos documentos sobre [nossas outras soluções de observabilidade](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability) e a [página APM](/docs/apm).
* [Altere o nome do seu aplicativo](/docs/agents/manage-apm-agents/app-naming/rename-your-application) ou outras [opções de configuração](/docs/agents/python-agent/installation-configuration/python-agent-configuration).
* Saiba mais sobre como configurar [a instrumentação personalizada](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation) para atividades de aplicativos não monitoradas por padrão.
* Considere o [SDK de telemetria do Python](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic).

## Resolução de problemas

Depois de concluir o processo de instalação, seus dados deverão aparecer na [interface do APM](/docs/apm/applications-menu/monitoring/apm-overview-page) em cinco minutos. Caso contrário, use estes recursos de resolução de problemas:

* Se nenhum dado aparecer, siga estas [etapas de resolução de problemas](/docs/agents/python-agent/troubleshooting/no-data-appears-python).
* Se você tiver problemas ao instalar ou executar o agente Python em um novo host, [teste se o pacote está instalado corretamente](/docs/agents/python-agent/installation-configuration/testing-python-agent) e se pode entrar em contato com o serviço de coleta de dados da New Relic.
* Para outros problemas, consulte a [lista completa da documentação de resolução de problemas](/docs/agents/python-agent/troubleshooting/).
