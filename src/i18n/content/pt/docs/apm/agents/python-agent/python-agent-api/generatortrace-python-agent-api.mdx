## Sintaxe

```py
newrelic.agent.generator_trace(name=None, group=None, label=None, params=None, terminal=False)
```

Usado para instrumentar geradores que não são instrumentados por padrão.

## Descrição

`generator_trace` é um decorador para adicionar a funções geradoras. Adicionar este decorator permite coletar informações adicionais de [transação](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) (incluindo informações de trace da transação).

O decorador `generator_trace` pode ser usado em geradores e corrotinas com a versão 2.102.0.85 ou superior do agente. A medição de tempo desses objetos começa quando o consumo é iniciado e termina quando o objeto se esgota ou sai de escopo.

Você pode usar o decorador em conjunto com decoradores existentes, incluindo aqueles para métodos estáticos e de classe. Ao aninhar vários decoradores, tenha o decorador `generator_trace` como o decorador mais externo.

## Formulários de chamada alternativos [#alternate]

Para configurações onde você não pode usar o decorador, estes formulários de chamada alternativos estão disponíveis:

### O invólucro

Se você souber com antecedência onde estão os geradores específicos que deseja fazer o trace, você poderia usar o decorador `generator_trace`. No entanto, se você <DNT>**don&apos;t**</DNT> conhecer todos os geradores que precisam de trace (por exemplo, se eles estiverem sendo buscados dinamicamente como parte de um sistema de roteamento), então você deve usar o `GeneratorTraceWrapper` para encapsular o gerador no momento do registro ou no momento da chamada.

### Quebra automática baseada em caminho [#path-based]

`wrap_generator_trace` é usado para envolver geradores fora do código em que são declarados. Por exemplo: você pode usar isso para instrumentar o código da biblioteca que não deseja modificar.

Para obter mais informações sobre as diferenças entre esses formatos de chamada, consulte [Diferentes formatos de chamada](/docs/python-agent-api-different-call-forms).

## Exemplos

### `generator_trace` [#trace\_example][#trace_example]

Um exemplo de uso do decorador `generator_trace` :

```py
import newrelic.agent

@newrelic.agent.generator_trace()
def gen():
    for i in range(5):
        yield i
```

Um exemplo de uso do decorador `generator_trace` com um gerador assíncrono:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
async def gen():
    for i in range(5):
        yield i
```

### Exemplo de wrapper [#trace-wrap-example]

Um exemplo de uso do `GeneratorTraceWrapper`:

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen)
```

Se você quiser nomear a métrica com o nome do endpoint (em vez de nomear a métrica com base no identificador do gerador que está sendo chamado), você pode fornecer o nome a ser usado mais um prefixo de caminho de métrica alternativo quando o objeto `GeneratorTraceWrapper` for criado.

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name="gen_5", group="Python/Generator")
```

Se você quiser nomear a métrica dinamicamente (em vez de nomear a métrica com base no identificador do gerador sendo chamado), você pode fornecer o nome como um gerador quando o objeto `GeneratorTraceWrapper` for criado.

```py
import newrelic.agent

def gen(x):
    for i in range(x):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name = lambda x: f"gen_{x}")
```
