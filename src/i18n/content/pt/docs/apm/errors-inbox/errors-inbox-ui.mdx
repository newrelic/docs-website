Errors Inbox é um local único para detectar, fazer a triagem e tomar medidas proativas em relação a todos os erros antes que eles afetem os clientes. Receba alertas sempre que surgir um erro crítico com impacto nos clientes através do seu canal de comunicação preferido.

Errors Inbox está incorporada à <InlinePopover type="apm"/>interface para que você possa resolver erros mais rapidamente. Os erros são agrupados para reduzir o ruído e a colaboração entre as equipes é fácil com visibilidade compartilhada dos mesmos dados de erro.

<Callout variant="important">
  Errors Inbox não está disponível na região da UE.
</Callout>

## Como fazer triagem de erros no APM

Nenhuma configuração ou carga de trabalho adicional é necessária para fazer a triagem de erros no APM. Basta navegar até o serviço APM que deseja visualizar e seguir as etapas abaixo:

1. Na tela de visão geral da interface do APM, clique na opção

   <DNT>
     **Errors inbox**
   </DNT>

   na navegação à esquerda na seção de triagem.

2. Esta visualização Errors Inbox agrupa erros relacionados especificamente ao serviço APM que você está visualizando. Você pode ver todos os detalhes do erro e também fazer a triagem do erro diretamente na interface do APM.

3. Clique nos detalhes de um grupo de erros para ver o stack trace, o atributo de logs contextualizados e outras informações sem navegar para outras telas ou serviços.

4. Clique em

   <DNT>
     **Go to global inbox**
   </DNT>

   para obter uma experiência mais ampla Errors Inbox , que agrupa erros em toda a stack, incluindo erros

   <InlinePopover type="browser"/>

   e

   <InlinePopover type="mobile"/>

   . O New Relic encontra automaticamente uma workload que inclui o serviço APM que você estava procurando ou solicita que você crie uma nova workload.

Usar Errors Inbox no APM oferece uma camada adicional de conveniência, permitindo fazer a triagem de erros sem trocar de tela.

## Conecte uma caixa de entrada ao Slack

Quando conectado ao Slack, grupos de erros novos e recriados serão enviados para um canal do Slack segundos após sua ocorrência. Isso permite que sua equipe identifique rapidamente quaisquer novos erros ou regressões e os resolva rapidamente com links diretos para o stack trace.

1. Se o seu espaço de trabalho Slack não tiver o [aplicativo New Relic](https://newrelic.slack.com/apps/AP92KQJS3-new-relic?tab=more_info) instalado, faça isso primeiro.

2. Em uma caixa de entrada, selecione o ícone Configurações da caixa de entrada (parece uma engrenagem) no canto superior direito.

3. Alterne o botão Slack para

   <DNT>
     **on**
   </DNT>

   se for

   <DNT>
     **off**
   </DNT>

   .

4. Se nenhum espaço de trabalho estiver disponível, clique no botão "+" para ativar o Slack com uma autenticação do Slack com um clique.

5. Uma vez autenticado, você poderá selecionar um espaço de trabalho e um canal específico para enviar a notificação.

6. Clique em “Testar” para garantir que as mensagens estão sendo enviadas para o canal correto.
