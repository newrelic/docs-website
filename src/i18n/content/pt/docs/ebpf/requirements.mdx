      ### Distribuições Linux [#linux-distributions]

      * Versões do kernel Linux `5.4` e posteriores

      * `arm64` ou `x86-64` arquitetura

      * Privilégios de root (acesso sudo necessário para instalação e operação)

      * Um mínimo de `2 GB` de RAM

      * Compatível com as seguintes distribuições Linux:

        * Debian `11` e posterior
        * Ubuntu `20.04` e posterior
        * CentOS Stream `9` e posterior
        * RHEL `8` e posterior

/\* ## Consultando dados do eBPF \[#querying-data] Uma vez implantado o agente eBPF, os dados de telemetria fluem para o New Relic e podem ser consultados usando NRQL. Aqui estão os padrões de consulta comuns: ### consulta spans Incluindo requests de corpo inteiro e dados trace : \`\`\`sql FROM Span SELECT \* WHERE deployment-name = &apos;&lt;deployment-name&gt;&apos; LIMIT MAX \`\`\` ### consulta métrica Incluindo estatísticas TCP e métrica de desempenho: \`\`\`sql FROM métrica SELECT \* WHERE deployment-name = &apos;&lt;deployment-name&gt;&apos; LIMIT MAX \`\`\` &lt;Callout variant=&quot;tip&quot;&gt; Substituir \`&lt;deployment-name&gt;\` pelo nome real da implantação que você configurou durante a instalação. &lt;/Callout&gt; ### Nomenclatura de entidades O agente eBPF gera automaticamente nomes de entidades de forma diferente dependendo do ambiente: \* \*\*Em hosts ou docker: Os nomes são uma combinação do nome do processo, seu diretório ou ID de contêiner e a porta de escuta. Por exemplo, \`ruby:/home/ubuntu/app:\[5678]\` ou \`java:f4aead533895:\[8080]\`. \* \*\*No Kubernetes: Os nomes são derivados do nome do serviço, por exemplo, \`mysql-database-service\`. ## Acesso a dados no New Relic \[#data-access] Os dados do eBPF são acessíveis por meio de várias interfaces New Relic : 1. \*\*Interface APM &amp; Services\*\*: Acesse \*\*\[one.newrelic.com]\(https\://one.newrelic.com) &gt; APM &amp; Services\*\* e filtre por \`instrumentation.name = nr\_ebpf\` 2. Consulta NRQL: Use os exemplos de consulta acima para explorar seus dados 3. Interface OpenTelemetry : Os dados do eBPF se integram à experiência OpenTelemetry da New Relic Para obter mais informações sobre como acessar seus dados, consulte \[Interface OpenTelemetry APM ]\(/docs/OpenTelemetry/get-started/APM-monitoramento/OpenTelemetry-APM-interface). \*/
