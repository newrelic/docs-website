Se o New Relic <DNT>eBPF agent</DNT> não estiver roteando a telemetria pelo seu proxy como esperado, use esta página para diagnosticar erros comuns. Para obter instruções de instalação e referência de configuração, consulte [Configurar um Proxy para o New Relic <DNT>eBPF agent</DNT>](/docs/ebpf/proxy-configuration).

## Erros comuns [#common-errors]

<CollapserGroup>
  <Collapser id="error-400" title="Proxy HTTP retornou o código de resposta 400">
    **Problema:** o proxy não suporta o método HTTP <DNT>`CONNECT`</DNT>.

    **Solução:** use um proxy compatível com <DNT>`CONNECT`</DNT>. Consulte [Servidores proxy suportados](/docs/ebpf/proxy-configuration#supported-proxies).
  </Collapser>

  <Collapser id="error-https-scheme" title="Esquema 'https' não suportado no URI do proxy">
    **Problema:** o gRPC não suporta <DNT>`https://`</DNT> como esquema de proxy. Esta é uma limitação do gRPC, não uma restrição do agente.

    **Solução:** defina o esquema de proxy para <DNT>`http`</DNT> (por exemplo, <DNT>`otlpProxy.scheme: "http"`</DNT>). Seus dados de telemetria ainda trafegam criptografados por TLS de ponta a ponta pelo túnel de proxy. Apenas a conexão com o próprio proxy é em texto simples.
  </Collapser>

  <Collapser id="error-socket-closed" title="Socket fechado ou conexão recusada">
    **Problema:** o servidor proxy não está em execução, ou o host ou a porta estão incorretos.

    **Solução:** verifique se o proxy está acessível usando o [comando de teste<DNT>`curl`</DNT> ](/docs/ebpf/proxy-configuration#test-connectivity).
  </Collapser>

  <Collapser id="error-connect-aborted" title="Proxy CONNECT abortado">
    **Problema:** o proxy rejeitou a requisição <DNT>`CONNECT`</DNT> devido a uma regra de firewall, ACL ou destino não suportado.

    **Solução:** verifique os logs do seu servidor proxy e confirme se as requests <DNT>`CONNECT`</DNT> para a porta `443` são permitidas a partir do IP de origem do agente.
  </Collapser>

  <Collapser id="error-tls-required" title="Configuração de proxy ignorada: proxy gRPC HTTP CONNECT requer TLS">
    **Problema:** um proxy está configurado, mas o TLS está desativado (por exemplo, <DNT>`TLS_ENABLED=false`</DNT>). A implementação de proxy HTTP CONNECT do gRPC só funciona quando a conexão com o endpoint OTLP usa TLS.

    **Solução:** certifique-se de que o TLS esteja habilitado (ele está habilitado por padrão). O agente então usará o proxy configurado.
  </Collapser>

  <Collapser id="error-credential-chars" title="O nome de usuário do proxy contém caracteres especiais ou maiúsculos">
    **Problema:** seu nome de usuário ou senha do proxy contém caracteres que o analisador de URI do gRPC não aceita. Consulte [Limitações de caracteres de credenciais](/docs/ebpf/proxy-configuration#credential-limitations).

    **Solução:** use uma conta proxy com um nome de usuário alfanumérico em letras minúsculas e uma senha que utilize apenas letras, dígitos e os caracteres especiais permitidos.
  </Collapser>
</CollapserGroup>
