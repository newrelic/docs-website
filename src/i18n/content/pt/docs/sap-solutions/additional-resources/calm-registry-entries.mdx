Configure entradas de registro (Call IDs) para provedores de dados do CloudALM para habilitar o monitoramento de serviços do SAP Cloud ALM.

## Visão geral

Este programa utilitário fornece funcionalidade para adicionar, alterar, excluir e gerar automaticamente configurações para provedores de dados CloudALM.

* <DNT>**Transaction Code**</DNT>: <DNT>**/NEWR/GEN\_DP\_CALLID**</DNT>
* <DNT>**Program Name**</DNT>: <DNT>**/NEWR/GENERATE\_CONFIG\_CALLID**</DNT>
* <DNT>**Configuration Table**</DNT>: <DNT>**/NEWR/CALLID\_CNF**</DNT>

## Pré-requisitos

Certifique-se de que o provedor de dados <DNT>**NR\_CALM\_LANDSCAPE**</DNT> esteja ativo e coletando dados antes de executar este utilitário.

## Procedimento de configuração

<Steps>
  <Step>
    ## Configurar mapeamento de provedor de dados e tipo de serviço

    1. Execute a transação `/NEWR/GEN_DP_CALLID`.
    2. Clique no <DNT>**DP/CALM Services Type Maintenance Link**</DNT>.
    3. Selecione os tipos de serviço que você deseja monitorar.
    4. Vincule cada provedor de dados ao sistema SAP ou serviço adequado configurado no SAP Cloud ALM.
    5. Salve a configuração.

    <Callout variant="tip">
      <DNT>**Service type**</DNT> representa <DNT>**SAP systems**</DNT> ou <DNT>**services**</DNT> habilitado no seu ambiente <DNT>**SAP Cloud ALM**</DNT>.
    </Callout>
  </Step>

  <Step>
    ## Adicionar novos registros de ID de chamada

    1. Na tela de seleção, insira o provedor de dados e o tipo de serviço que você precisa configurar.
    2. Selecione o botão de opção <DNT>**Add new (Call ID) records to Registry**</DNT>.
    3. Clique em <DNT>**Execute**</DNT>.
    4. O sistema exibe todos os <DNT>**service type instances**</DNT> configurados em <DNT>**Cloud ALM**</DNT>, cada um com um <DNT>**LMS ID**</DNT> exclusivo.
    5. Selecione as instâncias que você deseja monitorar.
    6. Exclua quaisquer entradas não necessárias.
    7. Clique em <DNT>**Update and save configuration**</DNT>.
  </Step>

  <Step>
    ## Verifique a configuração

    1. Vá para a transação `/NEWR/REGISTRY`.
    2. Selecione <DNT>**data provider**</DNT>.
    3. Clique em <DNT>**Configure**</DNT> para visualizar o <DNT>**Call ID entries**</DNT> salvo.
    4. Clique em <DNT>**Test**</DNT> para verificar a configuração.

    **Resultado esperado**: o teste de conexão exibe status bem-sucedido.
  </Step>
</Steps>

## Opções de manutenção

### Atualizar entradas existentes

Se você precisar alterar ou excluir <DNT>**Call ID records**</DNT> existentes:

1. Execute a transação `/NEWR/GEN_DP_CALLID`.
2. Selecione <DNT>**Copy/Change/Delete existing (Call ID) records in Registry**</DNT>.
3. Clique em <DNT>**Execute**</DNT>.
4. Faça as alterações necessárias.
5. Salve a configuração.

### Limpar todas as entradas

Para remover todos os <DNT>**Call ID records**</DNT> (para configuração inicial ou atualizações de Service Pack):

1. Execute a transação `/NEWR/GEN_DP_CALLID`.
2. Selecione <DNT>**Backup &amp; delete all existing (Call ID) records in Registry**</DNT>.
3. Clique em <DNT>**Execute**</DNT>.

<Callout variant="important">
  Isso cria um backup antes da exclusão. Use esta opção apenas durante a configuração inicial ou quando instruído durante uma atualização de service pack.
</Callout>

### Atualizar definições de serviço

Se você não conseguir localizar um <DNT>**service type**</DNT> ou <DNT>**LMS ID**</DNT>:

1. Clique em <DNT>**Retrieve new services (LMSID) setup in Cloud ALM**</DNT>.
2. Isso busca os serviços mais recentes configurados no seu <DNT>**Cloud ALM account**</DNT>.
