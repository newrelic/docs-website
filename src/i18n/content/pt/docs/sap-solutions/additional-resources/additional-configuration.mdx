Esta seção fornece instruções adicionais para configurar o sistema.

<Steps>
  <Step>
    ## Função de autorização de upload [#upload-authorization-role]

    1. Usar `SAP transaction PFCG`.

    2. Vá para **Role** &gt; **Upload** e carregue o arquivo **\_NEWR\_DATA\_PROVIDER.SAP** do pacote de instalação.

    3. Confirme se a função `/NEWR/DATA_PROVIDER` deve ser carregada.

       <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

    4. Clique no<img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

       ícone uma vez carregado.

    5. Na aba **Authorizations**, selecione<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

       **Change Authorization Data** e ativar usando<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />ícone.

    <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

    6. Se um nome de perfil for sugerido, clique no ícone de verificação verde.
    7. Verifique se a mensagem **Profile(s) were updated** aparece na barra de status.
  </Step>

  <Step>
    ## Criar New Relic [#create-rfc-user-id]

    Após ativar o perfil de autorização, use a transação `SU01` para criar um novo ID de usuário RFC (por exemplo, `NR4SAPDP`). Defina o tipo de usuário como **System** ou **Communication** e adicione a função `/NEWR/DATA_PROVIDER`.

    <img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />
  </Step>

  <Step>
    ## Crie destinos RFC para sistemas monitorados [#create-rfc-for-monitored-systems]

    Use a transação `SM59` para criar um destino RFC para cada sistema monitorado.

    1. Selecione **ABAP Connections** e clique no ícone **Create** para configurar um novo destino RFC.

    2. Insira um nome descritivo para o **RFC Destination**. Recomendamos nomear o destino RFC como `NR_<SYSID>CLNT<CLNT#>`

    3. Defina o **Connection Type** como `3 - ABAP Connection`.

    4. Na aba **Technical Settings**, insira:

       * **Target Host** (domínio ou IP totalmente qualificado) e **Instance No.**.
       * Especifique o **Gateway Host** e o **Service**, se necessário.

    <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

    5. Na guia **Logon &amp; Security**, insira o usuário RFC (`NR4SAPDP`) e a senha.
    6. Clique em **Save**.
    7. Vá para **Utility &gt; Test &gt; Connection Test** e **Authorization Test** para confirmar uma conexão bem-sucedida.

    <Callout variant="important">
      Certifique-se de que os resultados dos testes sejam bem-sucedidos.
    </Callout>
  </Step>

  <Step>
    ## Definir destino HTTP para a cloud New Relic

    Use a transação `SM59` para configurar conexões HTTPS com os pontos de extremidade da API do New Relic.

    <Callout variant="important">
      Certifique-se de criar um destino RFC individual para evento, logs, métrica e rastreamento separadamente.
    </Callout>

    1. Digite um **Destination Name** (por exemplo, `NR_API_EVENT`).
    2. Defina o **Connection Type** como `G` (Conexões HTTP para Servidor Externo).
    3. Digite o **Target Host**, **Service No.** como (443) (consulte [a porta padrão](/docs/sap-solutions/additional-resources/network-accessibility) para HTTPS) e **o Path Prefix** (consulte [a lista de verificação de prontidão para instalação](/docs/sap-solutions/additional-resources/installation-readiness) para obter detalhes).
    4. Se um proxy for necessário, configure-o na guia **Technical Settings**.
    5. Na guia **Security Options**, marque o botão de opção **Active** para SSL.
       <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />
    6. Se necessário, baixe o certificado raiz do [New Relic](https://one.newrelic.com) e adicione-o ao SAP usando a transação `STRUST`.
    7. Use o **Connection Test** para verificar a conexão. Uma conexão bem-sucedida prompt que você **Enter Logon Data**. Se o tempo do teste expirar, verifique as configurações do firewall.
    8. Repita este procedimento para todos os endpoints da API New Relic.
  </Step>

  <Step>
    ## Criar endpointde autenticação BTP/CloudALM [#create-authentication-endpoint]

    Siga as instruções abaixo com base no serviço:

    <CollapserGroup>
      <Collapser id="btp-process-integration-runtime" title="Tempo de execução de integração do processo BTP">
        1. Observe `clientId`, `clientsecret` e `tokenurl` da **Service Key** baixada.
        2. Acesse `tokenurl` em um navegador da web e baixe os certificados necessários (`Digicert root, global, and SAP on demand`).

        <img title="BTP process integration runtime" alt="A screenshot displaying the connection setup for BTP process integration runtime" src="/images/btp-process-integration.webp" width="600" />

        3. Use a transação `STRUST` para carregar os três certificados um por um e clique em **Add to Certificate List**.
        4. Clique em **Save**.
        5. Use a transação `SMICM` e navegue até **Administration &gt; ICM &gt; Exit Soft &gt; Global** para reiniciar todos os processos do ICM.

        <img title="Administration ICM" alt="A screenshot displaying the Administration ICM" src="/images/administration-icm.webp" width="600" />

        6. Use a transação `SM59` para criar uma nova conexão RFC em **HTTP Connections to External Server**.
        7. Divida o tokenurl no prefixo do host e do caminho. Adicione o prefixo do caminho com `?grant_type=client_credentials`.
        8. Defina o **Service No.** como `443`.

        <img title="Service No." alt="A screenshot displaying the Service No." src="/images/service-no.webp" width="600" />

        9. Na guia **Login &amp; Security**, selecione **Basic Authentication** e insira `clientId` e `clientsecret` como nome de usuário e senha.
        10. Defina o status do **Secure Protocol** como **Active** e salve a configuração.
        11. Execute um **Connection Test**.
      </Collapser>

      <Collapser id="cloud-alm-api" title="API CloudALM">
        1. Observe `clientId`, `clientsecret` e `url` da **Service Key** baixada.
        2. Acesse a URL em um navegador da web e baixe os certificados **Digicert root**, **global** e **SAP on demand** da Digicert.

        <img title="CloudALM API" alt="A screenshot displaying the connection setup for CloudALM API" src="/images/cloudalm-api.webp" width="600" />

        3. Use a transação `STRUST` para carregar os três certificados um por um e clique em **Add to Certificate List**.
        4. Clique em **Save**.
        5. Use a transação `SMICM` e navegue até **Administration &gt; ICM &gt; Exit Soft &gt; Global** para reiniciar todos os processos do ICM.

        <img title="Administration ICM" alt="A screenshot displaying the Administration ICM" src="/images/administration-icm.webp" width="600" />

        6. Use a transação `SM59` para criar uma nova conexão RFC.
        7. Insira a URL como **Target host** e defina o **Service No.** como `443`. Defina o **Path Prefix** como `/oauth/token?grant_type=client_credentials`.

        <img title="SM59 Configuration" alt="A screenshot displaying the SM59 configuration" src="/images/sm59-configuration.webp" width="600" />

        8. Na guia **Login &amp; Security**, selecione **Basic Authentication** e insira `clientId` e `clientsecret` como nome de usuário e senha.
        9. Defina o status do **Secure Protocol** como **Active** e salve a configuração.
        10. Execute um **Connection Test**.
      </Collapser>
    </CollapserGroup>
  </Step>

  <Step>
    ## Criar conexão com o banco de dados [#create-database-connection]

    Use a transação `DBACOCKPIT` para definir as conexões remotas do **HANA Database**.

    1. Clique duas vezes em **Database Connections** e clique em **Add**.

       <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

    2. Insira um **Connection Name** (de preferência o mesmo que o ID do sistema SAP).

    3. Selecione **HANA Database** como o **Database System**.

    4. Digite o **Database user ID** (por exemplo, `NRAGENT`) e a senha.

    5. Insira o **Database Host** e a **SQL Port**.

       <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

    6. Salve os dados e clique em **Test** para verificar a conexão.
  </Step>
</Steps>
