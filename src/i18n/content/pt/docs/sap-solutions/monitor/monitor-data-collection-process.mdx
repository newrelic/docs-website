Este guia mostra como monitorar o processo de coleta de dados do New Relic Monitoring for SAP Solutions. Você pode usar vários métodos para verificar se a coleta de dados está funcionando corretamente.

## Pré-requisitos

* Acesso ao sistema SAP com as permissões de transação apropriadas
* New Relic Monitoring for SAP Solutions instalado e configurado

## Status de tempo de execução e log de transmissão

No **Data Provider Registry** (transação `/NEWR/REGISTRY`), você pode verificar o **Runtime Status** e o **Transmission Log** para cada provedor de dados. O log de transmissão mostra o status e a hora de cada transmissão.

## Verifique o status do trabalho e o spool

1. Use a transação `SM37` para verificar o status dos trabalhos agendados. Digite `/NEWR/*` no campo **ABAP Program Name** e clique em **Execute**.
2. Revise a duração da execução. Um tempo de execução anormalmente longo indica um possível problema.
3. Verifique se há jobs sobrepostos e ajuste o agendamento conforme necessário. Acesse o spool do job a partir dos detalhes do job para visualizar o resultado de cada provedor de dados.

## Verifique o log do aplicativo

Os processos de coleta de dados são registrados no log do aplicativo (transação `SLG1`). Consulte com o objeto de log de aplicativo `/NEWR/DP`.

## Dashboard de monitoramento de coleta de dados SAP

A métrica de coleta de dados também é enviada ao New Relic para automonitoramento. Use o dashboard **SAP Data Collection Monitoring** pronto para uso para monitorar os processos de coleta de dados para cada provedor de dados.
