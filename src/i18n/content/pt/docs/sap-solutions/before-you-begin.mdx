Conclua estes pré-requisitos antes da instalação:

## Lista de verificação de pré-requisitos

### Conta New Relic

* <DNT>**Account ID**</DNT>: [obter o ID da sua conta](/docs/accounts/accounts-billing/account-setup/account-id)
* <DNT>**License Key**</DNT>: obtenha uma <InlinePopover type="licenseKey" />da New Relic para autenticação e acesso a dados
* <DNT>**Data Center Region**</DNT>: [escolha a localização do seu data center (EUA ou UE)](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) e configure sua conta

### Requisito do sistema SAP

* <DNT>**Software versions**</DNT>: Verifique se seus <DNT>**SAP systems**</DNT> atendem aos níveis de versão do componente de software exigidos. Consulte a [Referência de provedores de dados](/docs/sap-solutions/monitor/data-providers-reference) para requisitos específicos do sistema
* <DNT>**Monitored systems**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**Kernel 753 Patch 602+**</DNT>, <DNT>**ST-PI 740 SP16+**</DNT>
* <DNT>**Central monitoring system**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**ST-PI 740 SP14+**</DNT>

### Acesso à rede

* <DNT>**HTTPS access**</DNT> para <DNT>**New Relic endpoints**</DNT> (consulte as [URLs do endpoint](#network-endpoints))
* <DNT>**Firewall rules**</DNT> configurado
* <DNT>**Proxy settings**</DNT> configurado (se necessário)

### Monitoramento do BTP (se aplicável)

Se estiver monitorando <DNT>**BTP accounts**</DNT>, certifique-se de um dos seguintes:

* <DNT>**CloudALM account**</DNT> configurado para <DNT>**BTP monitoring**</DNT>
* <DNT>**Process Integration Runtime**</DNT> assinatura de serviço (necessária para o monitoramento de integração)

<Callout variant="important">
  Você pode precisar de aprovação ou assistência de configuração das suas equipes de segurança ou de rede. Conclua estas etapas antes da instalação e revise a [lista de verificação de prontidão para instalação](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness).
</Callout>

## Preparar sistemas monitorados

A arquitetura sem agente usa conexões RFC para recuperar dados.

### Requisitos de versão

Verifique as versões de software necessárias:

* <DNT>**SAP\_BASIS**</DNT>: Release <DNT>**740**</DNT> ou superior
* <DNT>**Kernel**</DNT>: Release <DNT>**753 Patch 602**</DNT> ou superior
* <DNT>**ST-PI**</DNT>: Release <DNT>**740 SP16**</DNT> ou superior

### Configuração do usuário

* [Carregar a função de autorização](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* [Crie um usuário do sistema](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) com a função carregada para <DNT>**RFC connections**</DNT>

### Configuração do banco de dados HANA (se aplicável)

Se realizar o monitoramento de um <DNT>**HANA database**</DNT>, crie um <DNT>**HANA database user**</DNT> `NRAGENT` com acesso <DNT>**Catalog Read**</DNT> e <DNT>**Monitoring**</DNT>:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

Colete o <DNT>**HANA server host**</DNT> e o <DNT>**tenant DB port**</DNT> para <DNT>**SQL connection**</DNT>.

### Outros bancos de dados

Para outros bancos de dados, ative o <DNT>**CCMS database KPIs**</DNT> padrão executando o <DNT>**ABAP program**</DNT> `RSDBKPIMON`.

Se a sua versão não estiver listada, entre em contato com seu executivo de contas da New Relic para obter um pacote suplementar.

### Requisito do sistema

* Pelo menos <DNT>**20 dialog work processes**</DNT> e <DNT>**10 background work processes**</DNT>
* <DNT>**HTTPS access**</DNT> de saída para o <DNT>**New Relic data center**</DNT> (veja as URLs do endpoint acima)

### Configuração de rede

* <DNT>**Proxy settings**</DNT> (se obrigatório):

  * Nome de domínio/endereço IP e porta do servidor
  * Credenciais de acesso do proxy (se a autenticação for necessária)

* <DNT>**Firewall policies**</DNT>: Permite que <DNT>**HTTPS traffic**</DNT> <DNT>**New Relic endpoints**</DNT>

### Destinos RFC

Para cada sistema monitorado, use a transação `SM59` e o ID do sistema remoto para criar <DNT>**RFC destinations**</DNT> e testar conexões. Consulte [Criar destinos RFC para sistemas monitorados](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems).

<Callout variant="important">
  O <DNT>**RFC user**</DNT> deve ter a função <DNT>**/NEWR/DATA\_PROVIDER**</DNT> atribuída.
</Callout>

### Monitoramento do banco de dados HANA remoto (se aplicável)

Para monitorar <DNT>**HANA databases**</DNT> remoto:

* Certifique-se de que o sistema central de monitoramento tenha acesso ao <DNT>**HANA server**</DNT> e <DNT>**tenant DB port**</DNT>

* Para sistemas de monitoramento central não S4HANA, instale um destes componentes:

  * <DNT>**HANA Client v2.0**</DNT> (compatível com o <DNT>**HANA DB version**</DNT>)

  * <DNT>**HANA Client Lib file**</DNT>:

    * `libSQLDBCHDB.so` (Linux)
    * `libSQLDBCHDB.dll` (Windows)
    * Localize a biblioteca no <DNT>**SAP Kernel folder**</DNT> e atribua a propriedade ao <DNT>**SID Admin ID**</DNT> (por exemplo, `sd1adm`)

* Para cada <DNT>**HANA database**</DNT> remoto, use a transação `SAP DBACOCKPIT` para criar um <DNT>**DB connection**</DNT>. Veja [Criar conexão de banco de dados](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)

## Preparar o monitoramento do BTP (se aplicável)

Para monitorar um <DNT>**BTP account**</DNT>, você precisa de um <DNT>**CloudALM instance**</DNT> configurado para monitorar o <DNT>**BTP account**</DNT> ou de uma assinatura do serviço <DNT>**Process Integration Runtime**</DNT> (aplicável apenas ao monitoramento de integração). <DNT>**Service keys**</DNT> são necessários para a autenticação da conexão. Consulte [Como recuperar chaves de serviço](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).

Use <DNT>**service keys**</DNT> para criar [destinos HTTP externos](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) e como o <DNT>**API host**</DNT> para <DNT>**API data providers**</DNT>.
