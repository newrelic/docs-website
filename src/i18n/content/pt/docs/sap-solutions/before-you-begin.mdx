Conclua estes pré-requisitos antes da instalação:

## Lista de verificação de pré-requisitos

### Conta New Relic

* **ID da conta**: [obtenha o ID da sua conta](/docs/accounts/accounts-billing/account-setup/account-id)
* **Chave de licença**: obtenha uma <InlinePopover type="licenseKey" />da New Relic para autenticação e acesso a dados
* **Data Center Region**: [escolha a localização do seu data center (EUA ou UE)](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) e configure sua conta

### Requisito do sistema SAP

* **Versões de software**: verifique se seus sistemas SAP atendem aos níveis de versão do componente de software exigidos. Consulte a [Referência de provedores de dados](/docs/sap-solutions/monitor/data-providers-reference) para requisitos específicos do sistema
* **Sistemas monitorados**: SAP\_BASIS 740+, Kernel 753 Patch 602+, ST-PI 740 SP16+
* **Sistema central de monitoramento**: SAP\_BASIS 740+, ST-PI 740 SP14+

### Acesso à rede

* Acesso HTTPS aos endpoints da New Relic (veja as [URLs dos endpoints](#network-endpoints))
* Regras de firewall configuradas
* Configurações de proxy configuradas (se necessário)

### Monitoramento do BTP (se aplicável)

Se estiver monitorando contas BTP, certifique-se de um dos seguintes:

* Conta CloudALM configurada para o monitoramento do BTP
* Assinatura do serviço Process Integration Runtime (necessária para o monitoramento de integração)

<Callout variant="important">
  Você pode precisar de aprovação ou assistência de configuração das suas equipes de segurança ou de rede. Conclua estas etapas antes da instalação e revise a [lista de verificação de prontidão para instalação](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness).
</Callout>

## Preparar sistemas monitorados

A arquitetura sem agente usa conexões RFC para recuperar dados.

### Requisitos de versão

Verifique as versões de software necessárias:

* **SAP\_BASIS**: versão 740 ou superior
* **Kernel**: versão 753 Patch 602 ou superior
* **ST-PI**: release 740 SP16 ou superior

### Configuração do usuário

* [Carregar a função de autorização](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* [Crie um usuário do sistema](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) com a função carregada para conexões RFC

### Configuração do banco de dados HANA (se aplicável)

Se estiver monitorando um banco de dados HANA, crie um usuário `NRAGENT` do banco de dados HANA com acesso de Catalog Read e monitoramento:

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

Colete o host do servidor HANA e a porta do banco de dados do locatário para conexão SQL.

### Outros bancos de dados

Para outros bancos de dados, ative os KPIs padrão de banco de dados do CCMS executando o programa ABAP `RSDBKPIMON`.

Se a sua versão não estiver listada, entre em contato com seu executivo de contas da New Relic para obter um pacote suplementar.

### Requisito do sistema

* Pelo menos 20 processos de trabalho de diálogo e 10 processos de trabalho em segundo plano
* Acesso HTTPS de saída ao data center da New Relic (veja as URLs de endpoint acima)

### Configuração de rede

* **Configurações de proxy** (se necessário):

  * Nome de domínio/endereço IP e porta do servidor
  * Credenciais de acesso do proxy (se a autenticação for necessária)

* **Políticas de firewall**: permita o tráfego HTTPS para os endpoints da New Relic

### Destinos RFC

Para cada sistema monitorado, use a transação `SM59` e o ID do sistema remoto para criar destinos RFC e testar conexões. Consulte [Criar destinos RFC para sistemas monitorados](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems).

<Callout variant="important">
  O usuário RFC deve ter o papel **/NEWR/DATA\_PROVIDER** atribuído.
</Callout>

### Monitoramento do banco de dados HANA remoto (se aplicável)

Para monitorar bancos de dados HANA remotos:

* Certifique-se de que o sistema central de monitoramento tenha acesso ao servidor HANA e à porta do banco de dados do locatário.

* Para sistemas de monitoramento central não S4HANA, instale um destes componentes:

  * HANA Client v2.0 (compatível com a versão do HANA DB)

  * Arquivo de biblioteca do cliente HANA:

    * `libSQLDBCHDB.so` (Linux)
    * `libSQLDBCHDB.dll` (Windows)
    * Localize a biblioteca na pasta do SAP Kernel e atribua a propriedade ao ID de administrador do SID (p. ex., `sd1adm`)

* Para cada banco de dados HANA remoto, use a transação `SAP DBACOCKPIT` para criar uma conexão de banco de dados. Veja [Criar conexão de banco de dados](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)

## Preparar o monitoramento do BTP (se aplicável)

Para monitorar uma conta BTP, você precisa de uma instância do CloudALM configurada para monitorar a conta BTP, ou de uma assinatura do serviço Process Integration Runtime (aplicável apenas ao monitoramento de integração). As chaves de serviço são necessárias para a autenticação da conexão. Consulte [Como recuperar chaves de serviço](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).

Use chaves de serviço para criar [destinos HTTP externos](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) e como o host da API para provedores de dados da API.
