O componente ABAP do **NRM4SAP** é empacotado como um complemento SAP e pode ser instalado usando a transação **SAINT** no **client 000**.

## Arquivo de instalação do complemento [#add-on-installation-file]

* **Instalação inicial**: Baixe o arquivo complementar `K-xxxAOIINNEWR.SAR` para uma pasta no seu PC local.
* **Atualização da versão anterior**: use o arquivo chamado `K-xxxAOUINNEWR.SAP`.

## Instalar arquivo complementar SAP [#install-sap-add-on-file]

1. Efetue login no **client 000**, que é o cliente padrão para tarefas de configuração e administração do sistema.
2. Digite `SAINT` no campo de comando da GUI do SAP e clique em **Enter**.
3. Navegue até **Installation Package** &gt; **Load Packages**.
4. Escolha um dos seguintes métodos:

<CollapserGroup>
  <Collapser id="sar-archive-from-frontend" title="Arquivo SAR do frontend">
    1. Navegue e abra o arquivo de instalação descompactado.
    2. No pop-up **Content of the compressed file `K-xxxAOIINNEWR.SAR`**, clique em **Decompress**.

    <img title="Content of the compressed file" alt="A screenshot displaying the Content of the compressed file" src="/images/content-of-compressed-file-1.webp" width="" />

    3. A mensagem **File K-xxxAOIINNEWR.SAR sucessfully transferred, decompressed, and deleted** é exibida.
  </Collapser>
</CollapserGroup>

<CollapserGroup>
  <Collapser id="sar-archive-from-application-server" title="Arquivo SAR do aplicativo servidor">
    1. Certifique-se de que haja um diretório de download no diretório trans no servidor do aplicativo (`/usr/sap/trans/download`).
    2. Copie o arquivo complementar para este diretório.
  </Collapser>
</CollapserGroup>

<CollapserGroup>
  <Collapser id="eps-files-from-application-server" title="Arquivos EPS do aplicativo servidor">
    1. Primeiro, descompacte o arquivo `SAR` no arquivo `PAT`. Execute o comando abaixo no sistema operacional (Linux neste caso):

    ```sql
    cd /usr/sap/trans
    cp /tmp/K-300AOIINNEWR.SAR /usr/sap/trans
    SAPCAR -xvf K-300AOIINNEWR.SAR
    ```

    2. No pop-up, **Do you want to upload OCS Packages from the EPS inbox?**, selecione **Yes**. Certifique-se de que o arquivo OCS seja carregado sem erros.
  </Collapser>
</CollapserGroup>

5. Clique em **Start**.

6. Selecione os **Installable Add-on packages for your system** e clique em **Continue**.

7. Quando perguntado **Do you want to add Modification Adjustment Transports to the queue?**, clique em **No**.

   <img title="Do you want to add Modification Adjustment Transports to the queue?, Click No" alt="A screenshot displaying question, Do you want to add Modification Adjustment Transports to the queue?" src="/images/modify-adjustement-transports.webp" width="700" />

<Callout variant="important">
  Clique em **Yes** se os transportes de ajuste forem fornecidos na lista de embalagens com instruções.
</Callout>

8. Clique no<img src="/images/green-tick-button.webp" class="inline" style={{height: "25px", width: "30px"}} alt="green check button" title="Green check button" />botão para prosseguir para a próxima tela.

<Callout variant="important">
  Se a página **Errors during Disassembling OCS Packages** aparecer, clique em **Ignore**.
</Callout>

9. Depois que o pacote estiver instalado, clique em **Finish**. A janela pop-up **Sending the runtime analysis** é exibida; você pode escolher:

   * **Send**: abre um formulário de feedback com os dados da análise de tempo de execução em uma janela do navegador local. Você pode usar este formulário para enviar dados ao SAP.
   * **Display**: exibe primeiro a análise de tempo de execução e depois envia o formulário de feedback em uma janela do navegador local.
   * **Do not send**: sai da caixa de diálogo sem enviar os dados de análise de tempo de execução.

10. Garanta o sucesso da instalação na aba **Installed Components**.

11. Sair do **Client 000**.

<Callout variant="tip">
  Você pode digitar **/nex** no campo de comando da GUI do SAP para sair do sistema.
</Callout>
