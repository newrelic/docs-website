## Configurar workload do sistema

1. Vá para <DNT>**All entities &gt; Workloads &gt; + Create a workload**</DNT>. Para mais informações, consulte [Criar e definir workloads](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads/).

2. Insira um nome para o seu <DNT>**workload**</DNT> e selecione uma conta na lista dropdown.

3. Ao lado de <DNT>**Entity Type = All**</DNT>, selecione o ícone <DNT>**+**</DNT> e insira `sap.system = {SAP System SID}` na barra de filtro.

   <img title="SAP System" alt="A screenshot displaying the SAP System filter" src="/images/sap-system.webp" />

4. Selecione <DNT>**+ Add this query**</DNT> para adicionar a consulta ao seu <DNT>**workload definition**</DNT>.

5. Selecione <DNT>**Create a workload**</DNT>.

Seu novo <DNT>**workload**</DNT> aparece em <DNT>**All entities**</DNT> e inclui todas as entidades com a tag do seu <DNT>**SAP system**</DNT>.

## Instalar service pack

<DNT>**Service packs**</DNT> são entregues como arquivos `.SAR` com a convenção de nomenclatura `K-<Version Number>SP<SP Number>IN<Software Component>.SAR`.

1. Instale o <DNT>**service pack**</DNT> usando a transação `SPAM`.
2. Carregue o novo <DNT>**configuration template**</DNT> de <DNT>**client 000**</DNT>.
3. Navegue até <DNT>**Install New Release and Patch**</DNT> &gt; <DNT>**Copy Template Tables**</DNT> a partir de <DNT>**Client 000**</DNT>.
4. Gere o novo <DNT>**data provider configurations**</DNT> seguindo as etapas em <DNT>**Install New Release and Patch**</DNT> &gt; <DNT>**Generate Configuration for New Data Providers**</DNT>.
5. Selecione <DNT>**Install New Release and Patch**</DNT> &gt; <DNT>**Verify Installation Completeness**</DNT> para verificar a instalação.
