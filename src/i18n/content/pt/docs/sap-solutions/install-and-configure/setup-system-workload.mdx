## Configurar workload do sistema

1. Acesse **All entities &gt; Workloads &gt; + Create a workload**. Para mais informações, consulte [Criar e definir workloads](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads/).

2. Insira um nome para o seu workload e selecione uma conta na lista dropdown.

3. Ao lado de **Entity Type = All**, selecione o ícone **+** e insira `sap.system = {SAP System SID}` na barra de filtro.

   <img title="SAP System" alt="A screenshot displaying the SAP System filter" src="/images/sap-system.webp" />

4. Selecione **+ Add this query** para adicionar a consulta à sua definição de workload.

5. Selecione **Create a workload**.

Sua nova workload aparece em **All entities** e inclui todas as entidades com a tag do seu sistema SAP.

## Instalar service pack

Os service packs são entregues como arquivos `.SAR` com a convenção de nomenclatura `K-<Version Number>SP<SP Number>IN<Software Component>.SAR`.

1. Instale o service pack usando a transação `SPAM`.
2. Carregue o novo modelo de configuração do cliente 000.
3. Navegue até **Install New Release and Patch** &gt; **Copy Template Tables** do cliente 000.
4. Gere a nova configuração do provedor de dados seguindo as etapas em **Install New Release and Patch** &gt; **Generate Configuration for New Data Providers**.
5. Selecione **Install New Release and Patch** &gt; **Verify Installation Completeness** para verificar a instalação.
