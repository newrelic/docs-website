Conclua estas tarefas de configuração para cada sistema monitorado.

## Função de autorização de upload [#upload-authorization-role]

1. Use <DNT>**SAP transaction**</DNT> `PFCG`.

2. Vá para <DNT>**Role**</DNT> &gt; <DNT>**Upload**</DNT> e carregue o arquivo <DNT>**\_NEWR\_DATA\_PROVIDER.SAP**</DNT> do pacote de instalação.

3. Confirme que a função `/NEWR/DATA_PROVIDER` carrega.

   <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

4. Selecione o<img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   ícone após o carregamento da função.

5. Na aba <DNT>**Authorizations**</DNT>, selecione<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   <DNT>**Change Authorization Data**</DNT> e, em seguida, ative-o usando o<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />ícone.

   <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

6. Se um nome de perfil for sugerido, selecione o ícone de check verde.

7. Verifique se <DNT>**Profile(s) were updated**</DNT> aparece na barra de status.

## Criar New Relic [#create-rfc-user-id]

Após ativar o <DNT>**authorization profile**</DNT>, use a transação `SU01` para criar um novo <DNT>**RFC user ID**</DNT> (por exemplo, `NR4SAPDP`). Defina o <DNT>**user type**</DNT> como <DNT>**System**</DNT> ou <DNT>**Communication**</DNT> e adicione a função `/NEWR/DATA_PROVIDER`.

<img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />

## Crie destinos RFC para sistemas monitorados [#create-rfc-for-monitored-systems]

Use a transação `SM59` para criar um <DNT>**RFC destination**</DNT> para cada sistema monitorado.

1. Selecione <DNT>**ABAP Connections**</DNT> e, em seguida, selecione o ícone <DNT>**Create**</DNT> para configurar um novo <DNT>**RFC destination**</DNT>.

2. Insira um nome descritivo para <DNT>**RFC Destination**</DNT>. Recomendamos: `NR_<SYSID>CLNT<CLNT#>`

3. Defina <DNT>**Connection Type**</DNT> como `3 - ABAP Connection`.

4. Na aba <DNT>**Technical Settings**</DNT>, insira:

   * <DNT>**Target Host**</DNT> (domínio ou IP totalmente qualificado) e <DNT>**Instance No.**</DNT>
   * <DNT>**Gateway Host**</DNT> e <DNT>**Service**</DNT> se necessário.

   <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

5. Na guia <DNT>**Logon &amp; Security**</DNT>, insira o <DNT>**RFC user**</DNT> (`NR4SAPDP`) e a senha.

6. Selecione <DNT>**Save**</DNT>.

7. Vá para <DNT>**Utility &gt; Test &gt; Connection Test**</DNT> e <DNT>**Authorization Test**</DNT>.

   * <DNT>**Expected result**</DNT>: Ambos os testes mostram status de sucesso.

## Definir destino HTTP para a cloud New Relic

Use a transação `SM59` para configurar <DNT>**HTTPS connections**</DNT> para o <DNT>**New Relic API endpoints**</DNT>.

<Callout variant="important">
  Crie <DNT>**RFC destinations**</DNT> individuais para <DNT>**Events**</DNT>, <DNT>**Logs**</DNT>, <DNT>**Metrics**</DNT> e <DNT>**Traces**</DNT> separadamente.
</Callout>

1. Insira um <DNT>**Destination Name**</DNT> (por exemplo, `NR_API_EVENT`).

2. Defina <DNT>**Connection Type**</DNT> como `G` (Conexões HTTP para Servidor Externo).

3. Insira <DNT>**Target Host**</DNT>, <DNT>**Service No.**</DNT> como <DNT>**443**</DNT> (consulte [porta padrão](/docs/sap-solutions/reference/network-accessibility) para <DNT>**HTTPS**</DNT>) e <DNT>**Path Prefix**</DNT> (consulte [lista de verificação de prontidão para instalação](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness) para obter detalhes).

4. Se você precisar de um proxy, configure-o na guia <DNT>**Technical Settings**</DNT>.

5. Na aba <DNT>**Security Options**</DNT>, selecione a opção <DNT>**Active**</DNT> para <DNT>**SSL**</DNT>.

   <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />

6. Se necessário, baixe o <DNT>**root certificate**</DNT> da [New Relic](https://one.newrelic.com) e adicione-o a <DNT>**SAP**</DNT> usando a transação `STRUST`.

7. Selecione <DNT>**Connection Test**</DNT> para verificar a conexão.

   * <DNT>**Expected result**</DNT>: Uma conexão bem-sucedida prompt você a inserir dados de logon.
   * <DNT>**If test times out**</DNT>: Verifique as configurações do firewall.

8. Repita este procedimento para todos os <DNT>**New Relic API endpoints**</DNT>.

## Criar conexão com o banco de dados [#create-database-connection]

Use a transação `DBACOCKPIT` para definir o <DNT>**HANA Database connections**</DNT> remoto.

1. Clique duas vezes em <DNT>**Database Connections**</DNT> e selecione <DNT>**Add**</DNT>.

   <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

2. Insira um <DNT>**Connection Name**</DNT> (de preferência o mesmo que o <DNT>**SAP system ID**</DNT>).

3. Selecione <DNT>**HANA Database**</DNT> como <DNT>**Database System**</DNT>.

4. Digite <DNT>**Database user ID**</DNT> (por exemplo, `NRAGENT`) e a senha.

5. Insira <DNT>**Database Host**</DNT> e <DNT>**SQL Port**</DNT>.

   <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

6. Selecione <DNT>**Save**</DNT> e, em seguida, selecione <DNT>**Test**</DNT> para verificar a conexão.

   * <DNT>**Expected result**</DNT>: O teste de conexão mostra status bem-sucedido.
