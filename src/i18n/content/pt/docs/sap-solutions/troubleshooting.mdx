## Solucionar problemas de disponibilidade do servidor

Os provedores de dados `NR_SAP_INSTANCE_AVAILABILITY` e `NR_SAP_SERVER_LIST` dependem de chamadas SOAP para serviços da web `SAPCONTROL`. Para que isso funcione, o perfil `DEFAULT.PFL` nos servidores monitorados deve ser configurado.

1. Confirme se a seguinte linha existe no arquivo `DEFAULT.PFL` :

`service/protectedwebmethods = SDEFAULT`

2. Se esta linha estiver faltando, adicione o seguinte parâmetro:

`service/protectedwebmethods = SDEFAULT -GetQueueStatistic -ABAPGetWPTable -EnqGetStatistic -GetProcessList -GetEnvironment -GetVersionInfo -GetSystemInstanceList`

3. Reinicie o serviço SAP com os comandos `stopsap sapstartsrv` e `startsap sapstartsrv`.

4. Confirme se a configuração funciona executando o comando `sapcontrol -prot GSOAP_HTTP -nr 00 -function GetSystemInstanceList`.
