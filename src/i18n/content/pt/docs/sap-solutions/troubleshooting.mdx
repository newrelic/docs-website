## Provedores de dados de disponibilidade do servidor não estão funcionando

**Sintoma**: os provedores de dados `NR_SAP_INSTANCE_AVAILABILITY` e `NR_SAP_SERVER_LIST` falham ao coletar dados.

**Causa**: o perfil `DEFAULT.PFL` nos servidores monitorados não está configurado para serviços web do SAPCONTROL.

**Solução**:

1. Abra o arquivo `DEFAULT.PFL` e verifique se esta linha existe:

`service/protectedwebmethods = SDEFAULT`

2. Se a linha estiver ausente, adicione o seguinte parâmetro:

`service/protectedwebmethods = SDEFAULT -GetQueueStatistic -ABAPGetWPTable -EnqGetStatistic -GetProcessList -GetEnvironment -GetVersionInfo -GetSystemInstanceList`

3. Reinicie o serviço SAP:

```yaml
stopsap sapstartsrv
startsap sapstartsrv
```

4. Verifique a configuração:

```yaml
sapcontrol -prot GSOAP_HTTP -nr 00 -function GetSystemInstanceList
```

**Resultado esperado**: o comando retorna a lista de instâncias do sistema sem erros.
