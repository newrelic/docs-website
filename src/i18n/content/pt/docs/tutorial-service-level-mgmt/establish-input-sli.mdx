Utilizamos monitoramento sintético para simular conexões de todo o mundo. Se houver uma interrupção entre o ponto de origem e a saída, as verificações sintéticas falharão, afetarão as pontuações dos seus objetivos de nível de serviço e acionarão muito alerta crítico. Chamamos isso de desempenho de entrada.

Basicamente, você mede o desempenho de entrada como a capacidade de conexão com sua aplicação, também chamada de “tempo de operação”. Você pode encontrar as etapas para configurar SLIs de desempenho de entrada abaixo, bem como exemplos de consulta NRQL para você experimentar.

<Steps>
  <Step>
    ## Crie suas verificações sintéticas

    O nível de serviço de desempenho de entrada mais comum é muitas vezes referido como “conectividade” ou “tempo de operação”. É uma simples verificação de endpoints de integridade da API ou carregamento de um URL. Você pode fazer ambos usando nosso serviço de monitoramento sintético. Antes de criar seu indicador de nível de serviço, consulte [Adicionar monitor de Browser simples](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#simple) e [Adicionar teste de API com script](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#complex) para saber como começar a relatar dados, caso ainda não tenha feito isso.
  </Step>

  <Step>
    ## Crie seu indicador de nível de serviço

    Depois de ter o relatório de dados, você pode usar o serviço de gerenciamento a nível de serviço para criar um indicador de entrada e objetivo usando o seguinte procedimento:

    1. No [explorador de entidades](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts#find), selecione **Service levels**.

    2. Clique em **+ Add a service level indicator**. Se você não vir o botão **Add a service level** , verifique com o administrador do New Relic sobre suas permissões.

    3. Filtre seus tipos de entidade para `Synthetic monitors`, como na captura de tela abaixo:

       <img
         alt="Filter entity types in service levels"
         src="/images/apm_screenshot-crop_add-input-service-level.webp"
       />

    4. Encontre o seu monitor Sintético na lista e clique nele. Isso ativa o botão **Continue** no painel esquerdo. Clique em **Continue**.

    5. Você verá um botão com as configurações recomendadas para um nível de serviço **Success** (mostrado abaixo). Clique.

       <img
         alt="Synthetics input service level guided flow"
         src="/images/apm_screenshot-crop_synthetics-setup.webp"
       />

    6. Faça as alterações apropriadas na tag, título e descrição conforme necessário.

    7. Clique em **Save**.
  </Step>
</Steps>
