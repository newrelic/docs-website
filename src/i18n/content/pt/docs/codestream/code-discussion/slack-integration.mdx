Compartilhar um código de código ou solicitação de feedback com o Slack é uma ótima maneira de alcançar pessoas que ainda não aderiram ao CodeStream ou que talvez não passem muito tempo em seu IDE. Eles podem até participar da discussão diretamente do Slack.

## Compartilhar no Slack [#share]

Ao postar um código ou solicitação de feedback, clicar no link do Slack na parte inferior do formulário o levará à web para se autenticar em um de seus espaços de trabalho do Slack.

/>

Depois de se conectar a um espaço de trabalho, selecione o canal com o qual deseja compartilhar na lista dropdown .

  src={codestreamChooseSlackChannel}
/>

Você também pode compartilhar em uma mensagem direta. Clique nos nomes de cada pessoa para incluir na conversa.

<img
  title="Slack channels"
  alt="An image showing direct message selection."
  src={codestreamChooseSlackDirectMessage}
/>

Ao compartilhar no Slack, o bot do CodeStream é incluído na conversa, e é isso que permite que as respostas sejam sincronizadas entre a plataforma. No caso de mensagens diretas, observe que você está tecnicamente criando uma conversa que inclui os participantes selecionados, bem como o bot do CodeStream. Lembre-se disso ao procurar a conversa na sua lista de mensagens diretas no Slack.

Por padrão, o CodeStream lembrará onde você compartilhou pela última vez e o formulário assumirá a mesma seleção na próxima vez. Clique no menu de engrenagem na parte superior se desejar definir um canal padrão para usar com base no repositório associado ao bloco de código selecionado.

Você também pode compartilhar uma postagem existente no Slack, desde que ela ainda não tenha sido compartilhada. Acesse um código de código ou solicitação de feedback e procure a opção <DoNotTranslate>**Share**</DoNotTranslate> no menu de reticências. A seção <DoNotTranslate>**Shared To**</DoNotTranslate> na exibição de um código ou solicitação de feedback informará se e onde ele foi compartilhado. Clique no nome da conversa para abri-la no Slack.

### Abrir no IDE [#open-idea]

Clique em <DoNotTranslate>**Open in IDE**</DoNotTranslate> para visualizar o código e a discussão diretamente no seu IDE. Primeiro, você será levado a uma página da Web do CodeStream, onde escolherá qual IDE abrir. CodeStream lembra sua seleção na próxima vez que você visualizar uma discussão do mesmo repositório. Em seguida, você será levado ao arquivo de origem apropriado em seu IDE, rolado até o bloco de código relevante, com a discussão exibida no painel CodeStream. Se você não tiver esse repositório aberto em seu IDE, o CodeStream abrirá automaticamente o arquivo de origem para você (supondo que você tenha aberto esse repositório anteriormente, com o CodeStream instalado, para que saibamos onde encontrá-lo).

Se você estiver usando um IDE JetBrains, instale o [aplicativo Toolbox](https://www.jetbrains.com/toolbox-app/) para que o CodeStream possa fazer um link direto para o IDE.

### Abra no GitHub (ou Bitbucket ou GitLab) [#open-github]

Se o bloco de código for de um repositório hospedado no GitHub, Bitbucket ou GitLab, este botão o levará ao bloco de código correspondente nesse serviço de hospedagem.
