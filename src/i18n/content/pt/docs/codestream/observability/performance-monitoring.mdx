O New Relic CodeStream permite que os desenvolvedores vejam o desempenho do código pelo qual são responsáveis na produção, trazendo observabilidade para o IDE e tornando-o parte de sua rotina diária.

Os dados de observabilidade exibidos no CodeStream são contextuais ao repositório que você tem aberto em seu IDE. Se esses repositórios não estiverem associados a serviços no New Relic, você será solicitado a [fazer essas associações](/docs/codestream/observability/repo-association). Caso contrário, você terá uma visão abrangente do desempenho dos serviços criados a partir do seu código.

Para cada serviço, os seguintes dados de desempenho poderão ser exibidos:

* <DNT>
    **Golden metrics**
  </DNT>

  : os principais sinais que indicam o desempenho do seu serviço. As métricas variam em função do tipo de serviço (APM, Browser, Mobile, OTel, função do Lambda).

* <DNT>
    **Issues**
  </DNT>

  : Quaisquer problemas em aberto com base na condição do alerta configurada para o serviço (somente APM).

* <DNT>
    **Service-level objectives**
  </DNT>

  : um resumo do desempenho do serviço em relação a quaisquer objetivos de nível de serviço configurados (APM, Browser, OTel).

* <DNT>
    **Transaction performance**
  </DNT>

  : veja quais transações estão apresentando problemas de desempenho desde seu último lançamento. [Saiba mais sobre desempenho de transação](/docs/codestream/observability/transactions) (somente APM).

* <DNT>
    **Errors**
  </DNT>

  : erros recentes ocorridos no código pelo qual você é responsável. Clique em um erro para percorrer o stack trace e colaborar com colegas de equipe na investigação. [Saiba mais sobre investigação de erros](/docs/codestream/observability/error-investigation) (APM, Browser).

* <DNT>
    **Vulnerabilities**
  </DNT>

  : vulnerabilidades em nível de pacote, incluindo informações sobre como remediar (somente APM).

* <DNT>
    **Related services**
  </DNT>

  : Todos os serviços que ligam ou são chamados pelo seu serviço, podendo ver as métricas clássicas de cada um deles. Entenda como seu serviço se enquadra no cenário geral (APM, Browser, mobile, OTel).

* <DNT>
    **Logs**
  </DNT>

  : pesquise o log diretamente do seu IDE para evitar a troca de contexto. [Saiba mais sobre pesquisa de log](/docs/codestream/observability/log-search) (APM, OTel).

Ao passar o mouse sobre o nome de um serviço na árvore, clique no globo para abrir a página de resumo do serviço no New Relic.

Para usar o recurso de observabilidade no CodeStream, observe que você deve ter um tipo de usuário New Relic de [usuário principal ou plataforma completa](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/#user-type-capabilities) ou deve estar no modelo de preços New Relic Compute.
