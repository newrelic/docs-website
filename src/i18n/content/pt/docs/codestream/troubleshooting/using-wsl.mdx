Para usar o CodeStream e o WSL (Windows Subsystem for Linux), siga estas etapas:

1. Certifique-se de que o Git esteja instalado na máquina Windows, bem como na sua distribuição WSL. Embora as versões instaladas entre os dois possam ser diferentes, é melhor garantir que a versão do Windows corresponda à que está instalada na sua distribuição WSL.

2. Clone seu repositório git na sua distribuição WSL (por exemplo, `~/code/YOUR_PROJECT_FOLDER`).

3. No Windows, permita que o git se comunique por este caminho UNC marcando o diretório como `safe`. Faça isso no seu terminal do Windows com este comando, substituindo `YOUR_DISTRO`, `YOUR_USERNAME` e `YOUR_PROJECT_FOLDER` para corresponder à sua configuração:

   ```shell
   git config --global --add safe.directory '//wsl.localhost/YOUR_DISTRO/home/YOUR_USERNAME/code/YOUR_PROJECT_FOLDER' 
   ```

4. No seu IDE, abra o projeto usando o caminho WSL UNC. Por exemplo, se você tivesse o Ubuntu, seu nome de usuário fosse `codestream` e a pasta do projeto fosse `Amazing_App`, seu caminho UNC seria `//wsl.localhost/Ubuntu/home/codestream/code/Amazing_App`.

<Callout variant="tip">
  O Git no Windows é usado apenas para fazer solicitações ao seu repositório clonado localmente no WSL e não precisa se comunicar com o remoto (por exemplo, GitHub, GitLab, etc.).
</Callout>

Se você tiver problemas após concluir essas etapas, [colete seu arquivo de log](/docs/codestream/troubleshooting/client-logs) e entre em contato com o suporte ao cliente New Relic para uma investigação mais aprofundada.
