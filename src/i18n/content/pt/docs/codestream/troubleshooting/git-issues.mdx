Ao postar uma mensagem com um bloco de código, você poderá encontrar um dos seguintes avisos relacionados ao Git.

## O repositório não é gerenciado pelo Git ou o Git não pode ser encontrado [#not-found]

Embora tecnicamente não seja um requisito, os principais elementos da funcionalidade do Code Stream dependem de seu repositório ser gerenciado pelo Git ou por um serviço de hospedagem Git como o GitHub. Se você não estiver usando o Git, o CodeStream não conseguirá conectar os comentários e problemas criados aos blocos de código apropriados nos arquivos de origem.

Este erro também pode significar que o Git não está no seu PATH. Nesse caso, adicione-o ao seu PATH e reinicie o seu IDE.

## O repositório não tem um URL remoto [#no-remote-url]

A URL remota é parte do que permite ao CodeStream vincular o bloco de código da sua mensagem ao arquivo e local apropriado no repositório correspondente em seu IDE. Se o CodeStream não conseguir identificar o repositório que seu colega de equipe abriu como o mesmo no qual você criou seu comentário ou problema, eles não poderão ver o comentário ou problema.

Na linha de comando, use `git remote -v` para confirmar se você tem ou não um URL remoto configurado para seu repositório. Se não houver URL remoto, descubra o URL correto e especifique o URL remoto por meio de `git remote add origin <remote URL>`.

## Arquivo não salvo [#unsaved]

Assim como acontece com os arquivos não gerenciados pelo Git, o CodeStream não será capaz de vincular um comentário ou problema a um bloco de código em um arquivo novo e não salvo. Vá em frente e salve o arquivo antes de criar seu primeiro comentário ou problema.
