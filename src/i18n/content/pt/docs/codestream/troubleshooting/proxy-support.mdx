Se você estiver enfrentando problemas de conectividade com a extensão New Relic CodeStream, pode ser porque você está atrás de um proxy de rede e o CodeStream não está configurado para funcionar com ele. Para encontrar as configurações do CodeStream em seu IDE:

* Código VS: vá para

  <DNT>
    **Settings**
  </DNT>

  e pesquise CodeStream.

* Visual Studio: vá para

  <DNT>
    **Tools > Options > CodeStream**
  </DNT>

  .

* JetBrains: Vá para

  <DNT>
    **Settings/Preferences > Tools > CodeStream**
  </DNT>

  .

Procure a configuração <DNT>**Proxy Support**</DNT> e certifique-se de que esteja definida como <DNT>**on**</DNT>. Você também precisará desabilitar a verificação estrita de SSL, pois sua extensão CodeStream não verá os certificados SSL/TLS como provenientes de uma autoridade de certificação legítima.

Esta é a aparência dessas configurações no VS Code.

<img
  alt="Proxy Settings"
  src="/images/codestream_screenshot-crop_vscode-proxy-settings.webp"
/>

Se você tiver suporte a proxy configurado em seu IDE, o CodeStream tentará primeiro herdar essas configurações. Caso contrário, o CodeStream herdará as configurações de proxy do seu sistema operacional/ambiente.

Após alterar suas configurações, reinicie seu IDE.
