Ao criar problemas do GitLab a partir do CodeStream, você pode receber um erro `404 Not Found` . URLs de grupo/projeto do GitLab são gerados com uma barra codificada que não é tratada pelo proxy reverso apache2 padrão. Para permitir que o apache2 resolva com êxito esses URLs, adicione o seguinte à configuração do apache2:

* Adicione o AllowEncodedSlashes
* Adicione `nocanon` ao final da sua linha ProxyPass

Por exemplo:

```
AllowEncodedSlashes NoDecode
ProxyPass / http://33.211.55.3:9000/ nocanon
```
