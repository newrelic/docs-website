Como gerente de domínio de autenticação, execute as seguintes etapas para conceder acesso a um usuário para gerenciar a conta e a cardinalidade métrica. Verifique com o usuário suas necessidades específicas e faça a configuração de acordo. Para saber mais sobre gerenciamento de usuários, consulte [tutorial de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/).

<Steps>
  <Step>
    ### Adicionar permissões a uma função personalizada [#permissions-to-custom-role]

    [Crie uma nova função personalizada](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) com as seguintes permissões ou adicione-as a uma função existente. Verifique com o usuário para entender o nível de acesso necessário.

    * `Manage account cardinality`: Ajuste o orçamento de cardinalidade da conta.

      * `View`: Pode ver o orçamento de cardinalidade da conta.
      * `Modify`: Pode atualizar o orçamento de cardinalidade da conta.

    * `Manage metric cardinality`: Ajuste o orçamento de cardinalidade da métrica.

      * `View`:Pode ver o orçamento de cardinalidade da métrica.
      * `Modify`: Pode atualizar o orçamento de cardinalidade da métrica.

    * `Manage pruning rules`: Gerenciar regras de poda para métrica.

      * `View`:Pode ver as regras de poda da métrica.
      * `Modify`: Pode criar e excluir regras de poda para a métrica.

    * `Customer limits management`: Ajuste os limites de diferentes recursos na conta, incluindo a cardinalidade.

      * `View`:Pode ver os limites disponíveis para os recursos.
      * `Modify`:Pode ajustar os limites das capacidades.
  </Step>

  <Step>
    ### Adicionar perfil do usuário e conta a um grupo [#group-access]

    Para o gerenciamento de cardinalidade, [crie um novo grupo](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#group-access) ou gerencie um grupo existente da seguinte maneira:

    1. Navegue até **[New Relic Administration](https://one.newrelic.com/admin-portal) &gt; Access Management &gt; Groups**.
    2. Clique em <DNT>**Create a new group**</DNT> ou selecione <DNT>**Manage group access**</DNT> no <Icon name="fe-more-horizontal" /> menu de um grupo existente.
    3. Para um novo grupo, forneça um nome e clique em <DNT>**Create group**</DNT>. O painel <DNT>**Admin**</DNT> do grupo é aberto.
    4. Em <DNT>**Members**</DNT>, adicione o usuário.
    5. Em <DNT>**Account access**</DNT>, associe a função personalizada à conta desejada para gerenciamento de cardinalidade.
    6. Se o gerenciamento de cardinalidade envolver várias contas, em <DNT>**Account access**</DNT>, inclua a função personalizada para cada conta relevante dentro do grupo.
  </Step>
</Steps>
