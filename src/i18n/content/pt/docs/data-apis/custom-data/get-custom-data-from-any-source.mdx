Para aproveitar ao máximo o New Relic, você pode precisar ou desejar relatar dados personalizados. Fornecemos as ferramentas necessárias para inserir qualquer tipo de dados no New Relic.

## Confira nossas soluções selecionadas [#options]

Os produtos New Relic reportam muitos dados automaticamente, sem necessidade de configuração adicional! Ao usar soluções de monitoramento como <InlinePopover type="apm"/>, <InlinePopover type="browser"/>, <InlinePopover type="mobile"/>, monitoramento de infraestrutura ou qualquer uma de nossas diversas [integrações e guias de início rápido](https://newrelic.com/instant-observability), por padrão você recebe vários tipos de dados de desempenho.

Mas você pode querer trazer para o New Relic dados que não são coletados por padrão. Talvez você queira uma solução baseada em API que não exija a instalação de um agente. Talvez você queira trazer dados de telemetria de outro serviço de análise para o New Relic. Abaixo, descreveremos várias maneiras de obter mais conjuntos de dados personalizados no New Relic.

## Reportar dados customizados com API do agente [#agent-apis]

Se estiver usando o agente New Relic (APM, browser ou mobile), você poderá configurá-los para enviar dados personalizados. Para saber mais, consulte a documentação do agente que você está usando.

## Relatar dados de infraestrutura personalizados [#flex-integration]

Se nossas integrações de infraestrutura não atenderem às suas necessidades, você pode usar nossa [integração Flex](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) para reportar qualquer tipo de métrica ou configuração relacionada à infraestrutura para a New Relic.

Se você é uma organização empresarial de médio a grande porte e deseja usar a tecnologia [SAP ®](https://www.sap.com/) para gerenciar suas operações comerciais e relações com clientes, pode usar nossa [integração SAP](/docs/data-apis/custom-data/sap-integration).

## Relate dados de serviços e especificações de telemetria de código aberto [#third-party-telemetry]

Você pode revisar toda a nossa integração em [nossa página de início rápido](https://newrelic.com/instant-observability). Aqui estão algumas de nossas integrações mais populares com especificações e serviços de telemetria de código aberto de terceiros:

* [OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/introduction-opentelemetry-new-relic)
* [Prometeu](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* [Zipkin](/docs/distributed-tracing/trace-api/report-zipkin-format-traces-trace-api)
* [EstatísticasD](/docs/more-integrations/open-source-telemetry-integrations/statsd/statsd-monitoring-integration)
* [Grafana](/docs/more-integrations/grafana-integrations/get-started/grafana-support-prometheus-promql)
* [DropWizard](/docs/more-integrations/open-source-telemetry-integrations/dropwizard/dropwizard-reporter)
* [Micrômetro](/docs/more-integrations/open-source-telemetry-integrations/micrometer/micrometer-metrics-registry)

## Report atributo personalizado e evento [#events]

Você pode decorar algum evento relatado pelo New Relic com metadados personalizados. Para mais informações, veja [atributo personalizado](/docs/data-apis/custom-data/custom-events/collect-custom-attributes).

Você também pode reportar eventos inteiramente personalizados para o New Relic com nossa [API de eventos](/docs/data-apis/ingest-apis/event-api/introduction-event-api).

Atributo personalizado e evento podem então ser consultados e mapeados.

## Relatar dados trace personalizados [#tracing]

Nossa [API Trace](/docs/distributed-tracing/trace-api/introduction-trace-api) permite enviar dados distributed tracing para a New Relic e consolidar dados trace de diversas fontes em um só lugar. Aceitamos dados trace em dois formatos:

* Formato Zipkin
* New Relic (se você não tiver dados no formato Zipkin, você usaria este)

## Relatar dados de log personalizados [#logs]

Se nossa [integração de registro](/docs/logs/get-started/get-started-log-management) existente não atender às suas necessidades, você poderá usar nossa [API Log ](/docs/logs/log-api/introduction-log-api)para enviar quaisquer dados de registro arbitrários para a New Relic.

## Crie visualizações de dados personalizadas [#visualizations]

Interessado em criar consultas, gráficos e painéis personalizados? Para começar a consultar, consulte [Introdução ao NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language). Ao fazer uma consulta, você pode pegar o gráfico resultante e adicioná-lo a [um dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Se precisar de mais capacidade e personalização do que nossos gráficos e <InlinePopover type="dashboards"/>permitem, você também pode criar seu próprio [aplicativo New Relic](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application) ou personalizar um existente [no catálogo de aplicativos](https://opensource.newrelic.com/nerdpacks). Os aplicativos New Relic estão disponíveis em nossa plataforma e podem ser compartilhados com sua equipe ou com o público.
