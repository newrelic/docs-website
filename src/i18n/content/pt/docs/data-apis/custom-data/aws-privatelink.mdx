Você pode enviar dados de telemetria da sua nuvem privada virtual (VPC) da AWS para o New Relic por meio [do AWS PrivateLink](https://aws.amazon.com/privatelink).

## Visão geral [#overview]

O envio de dados do New Relic por meio do AWS PrivateLink pode:

* Reduza seus custos de saída da AWS
* Melhore a segurança mantendo seus dados na rede Amazon

PrivateLink funciona para enviar [dados de telemetria](/docs/data-apis/understand-data/new-relic-data-types) para uma variedade de [endpoints](#endpoints) New Relic. Ele não pode ser usado para nossa [API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) ou outras APIs que não sejam de ingestão.

Para usar esse recurso, você precisará configurar um [endpoint de interface](#create) em sua VPC que seu agente e integração local do New Relic usará para rotear dados para o New Relic por meio do AWS PrivateLink.

## Requisitos [#requirements]

PrivateLink requer a opção [New Relic Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus) . Se você tentar enviar dados sem o Data Plus, o tráfego enviado via PrivateLink será rejeitado e retornado com um erro 402.

Um serviço de endpoint está disponível somente na região onde foi criado ([consulte a documentação da Amazon](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)) e pode ser acessado de outras regiões usando peering inter-regional ou aproveitando ([Conectividade entre regiões para AWS PrivateLink](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)).

Com o peering de VPC e a conectividade entre regiões, observe que o cliente é responsável pelos custos de transferência de dados entre regiões.

Isso significa que se a sua VPC estiver em uma das regiões compatíveis (`us-east-1`, `us-east-2` ou `us-west-2` para contas dos EUA; `eu-central-1` para contas da UE; `ap-northeast-1` para contas do JP), você só precisará criar o endpoint interno da VPC necessário, conforme descrito abaixo. Se a sua VPC estiver em uma região fora das regiões compatíveis, recomendamos usar a [Conectividade entre regiões para o AWS PrivateLink](#cross-region) ou você pode [configurar uma conexão de emparelhamento](#peering).

## Conectividade entre regiões [#cross-region]

A New Relic oferece suporte à [conectividade entre regiões do AWS PrivateLink](https://aws.amazon.com/blogs/networking-and-content-delivery/introducing-cross-region-connectivity-for-aws-privatelink/), que permite que você se conecte aos endpoints da New Relic de qualquer região da AWS sem configurar o emparelhamento de VPC. Essa é a abordagem recomendada quando sua VPC está em uma região fora das listadas em [Regiões e zonas suportadas](#regions-zones).

A conectividade entre regiões é suportada a partir de todas as regiões da AWS para os endpoints de data center dos EUA, UE e JP da New Relic, **exceto** `ap-southeast-6` (Nova Zelândia) e `ap-east-2` (Taipé).

## Configurar uma conexão de peering [#peering]

O emparelhamento da VPC não é mais necessário para se conectar a partir de regiões fora das [regiões suportadas](#regions-zones). Em vez disso, recomendamos o uso da [Conectividade entre regiões para o AWS PrivateLink](#cross-region). Se você ainda preferir usar o emparelhamento da VPC, observe que os custos de transferência de dados entre regiões são de sua responsabilidade.

### Pareamento de VPC [#vpc-peering]

Siga [as instruções para criar conexões de emparelhamento de VPC entre regiões](https://docs.aws.amazon.com/vpc/latest/peering/create-vpc-peering-connection.html), especificando o ID de VPC das conexões de endpoint que você criou.

### Espreitamento do gateway de trânsito [#transit-gateway-peering]

Siga [as instruções para criar um Route 53 PHZ e compartilhá-lo entre VPCs](https://aws.amazon.com/blogs/networking-and-content-delivery/integrating-aws-transit-gateway-with-aws-privatelink-and-amazon-route-53-resolver), especificando o ID do VPC das conexões de endpoint que você criou.
