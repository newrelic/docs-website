Por padrão, os produtos New Relic relatam uma variedade de dados usados em nossos gráficos de interface e que estão disponíveis para [consulta](/docs/using-new-relic/data/understand-data/query-new-relic-data). Nossos produtos não transmitirão informações confidenciais sem serem explicitamente instrumentos para fazê-lo.

## Evento e atributo padrão [#default-events]

Nossos produtos reportam um conjunto de [evento e atributo padrão](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products). Nunca enviaremos parâmetro de solicitação ou qualquer outro atributo que não esteja no conjunto padrão, a menos que alguém tenha habilitado isso explicitamente via configuração.

## Ajuste os dados relatados [#default-security]

Ao avaliar as configurações de segurança de um produto New Relic, revise o [evento padrão e o atributo](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products). O atributo padrão não contém dados confidenciais. Em geral, são simplesmente os dados necessários para um monitoramento eficaz do desempenho. Nossos produtos não enviam outros dados, a menos que você altere as configurações de segurança padrão.

Dependendo dos seus requisitos, uma ou ambas as situações podem ser aplicadas:

* Se a lista padrão contiver dados que o preocupam, você poderá desativar a coleta desses atributos. Para saber como editar isso, consulte a documentação do produto que você está usando.
* Se você precisar enviar um atributo não informado por padrão, você pode [habilitar o envio desses atributos](/docs/insights/insights-data-sources/custom-data/send-custom-data-insights). Nesse caso, não use [o modo de alta segurança](/docs/accounts-partnerships/accounts/security/high-security): isso desabilitará a capacidade de coletar atributo personalizado.
