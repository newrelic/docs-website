[O APM](/docs/apm/new-relic-apm/getting-started) relata [dados de eventos](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) que aparecem em algumas telas da interface e também estão disponíveis para [consulta e criação de gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data). Selecione um nome de evento na tabela a seguir para ver seu atributo.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Evento
      </th>

      <th>
        Descrição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="transaction-event">
        [`Transaction`](/attribute-dictionary/?event=Transaction)
      </td>

      <td>
        Uma transação é uma unidade lógica de trabalho em um aplicativo de software. O evento `Transaction` inclui informações sobre o aplicativo, chamadas de banco de dados, a duração da transação e quaisquer erros que possam ocorrer.
      </td>
    </tr>

    <tr>
      <td id="transactionerror-event">
        [`TransactionError`](/attribute-dictionary/?event=TransactionError)
      </td>

      <td>
        Erros de transação ocorrem quando uma solicitação lança uma exceção no caminho do código usado para preencher essa solicitação. O número de erros de transação não é igual ao número de transações, pois você pode especificar se deseja [coletar, ignorar ou marcar os erros como esperado](/docs/agents/manage-apm-agents/agent-data/manage-errors-apm-collect-ignore-or-mark-expected).
      </td>
    </tr>

    <tr>
      <td id="span-event">
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        Os eventos Span são criados quando o agente New Relic, ou outras ferramentas de instrumentação de rastreamento, executa operações de instrumentos que fazem parte de um distributed trace. Veja a [definição de span](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#span).
      </td>
    </tr>
  </tbody>
</table>
