Nosso [<InlinePopover type="browser" />](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)relata [dados de eventos](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) exibidos em nossa interface e disponíveis para [consulta e gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

Selecione um nome de evento na tabela a seguir para ver seu atributo.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Evento
      </th>

      <th>
        Descrição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`PageView`](/attribute-dictionary/?event=PageView)
      </td>

      <td>
        `PageView` rastreia dados geográficos e de tempo para cada carregamento de página do browser.
      </td>
    </tr>

    <tr>
      <td>
        [`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming)
      </td>

      <td>
        `PageViewTiming` envia cada ponto de dados como um evento separado assim que estiver disponível. Isto é útil para páginas assíncronas e dinâmicas.

        Para usar este evento, você deve ter o agente browser versão 1149 ou superior. Para obter mais informações, consulte os [exemplos de consulta`PageViewTiming` ](/docs/browser/new-relic-browser/page-load-timing-resources/pageviewtiming-async-or-dynamic-page-details).
      </td>
    </tr>

    <tr>
      <td>
        [`PageAction`](/attribute-dictionary/?event=PageAction)
      </td>

      <td>
        `PageAction` O evento rastreia dados sobre cada ação executada em uma página. Seu atributo fornece dados geográficos e de aplicativos, bem como dados sobre as dimensões do browser do usuário, IDs de sessão e URLs de referência e de página.
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserInteraction`](/attribute-dictionary/?event=BrowserInteraction)
      </td>

      <td>
        `BrowserInteraction` contém vários atributos PageView, bem como atributos específicos para aplicativos de página única (SPA).
      </td>
    </tr>

    <tr>
      <td>
        [`AjaxRequest`](/attribute-dictionary/?attribute_name=&events_tids%5B%5D=8302&event=AjaxRequest)
      </td>

      <td>
        Um evento `AjaxRequest` é criado sempre que uma chamada Ajax é feita. Quando uma chamada Ajax acontece durante um evento `BrowserInteraction` , ela é decorada com atributos adicionais relacionados à interação do browser. O atributo do evento rastreia informações geográficas e do browser.
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserTiming` (descontinuada)](/attribute-dictionary/?event=BrowserTiming)
      </td>

      <td>
        `BrowserTiming` é um evento personalizado que captura dados de tempo do SPA para interação do navegador iniciada usando o método customizado [createTracer](/docs/browser/new-relic-browser/browser-agent-spa-api/createtracer-browser-spa-api) SPA API. `BrowserTiming` contém muitos dos mesmos atributos usados por outros eventos, especialmente `AjaxRequest`. *Esse método não é mais usado como parte do SPA v2+, portanto, ele e este evento personalizado estão obsoletos e não devem ser usados.*
      </td>
    </tr>

    <tr>
      <td>
        [`JavaScriptError`](/attribute-dictionary/?event=JavaScriptError)
      </td>

      <td>
        Quando ocorre um erro de JavaScript, um evento `JavaScriptError` é criado. `JavaScriptError` captura dados de erros de JavaScript, bem como dados de interação do browser. Para mais informações veja a [página de erros JS](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-examine-errors-over-time) na interface do browser e o [exemplo de consulta NRQL](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-browser-single-page-app-data#javascriptError-example) para browser e evento SPA.
      </td>
    </tr>

    <tr>
      <td>
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        `Span` os dados são relatados para [distributed tracing](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing).
      </td>
    </tr>

    <tr>
      <td>
        [`UserAction`](/attribute-dictionary/?event=UserAction)
      </td>

      <td>
        `UserAction` O evento é capturado como resultado da interação do usuário com o aplicativo web. Este evento inclui informações sobre a ação e o identificador de destino DOM, juntamente com vários atributos padrão, como aplicativo e dados geográficos.
      </td>
    </tr>
  </tbody>
</table>

Para obter detalhes sobre por quanto tempo esses dados são retidos, consulte [retenção de dados](/docs/data-apis/manage-data/manage-data-retention).
