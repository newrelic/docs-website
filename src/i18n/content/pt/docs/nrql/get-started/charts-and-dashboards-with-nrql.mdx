consulta ajuda você a encontrar as informações que você precisa, e os quadros e gráficos baseados nessas consultas tornam os dados úteis para sua organização. Gráficos e painéis fornecem a base da visualização de dados, oferecendo diversas opções que permitem criar fluxos de dados visualmente atraentes e de fácil compreensão. Você pode aproveitar ao máximo seus dados entendendo como a consulta NRQL leva aos gráficos e ao painel.

Para concluir a escrita de uma consulta NRQL, você deve criar um gráfico como etapa final. O New Relic oferece [várias maneiras](/docs/query-your-data/explore-query-data/use-charts/chart-types) de visualizar seus dados, e você pode fazer muito com seus gráficos depois de criá-los. Depois de escrever sua consulta NRQL, você pode:

* Use seu gráfico como início de um [novo dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create) para monitor várias consultas de NRQL em um único local.
* [Adicione seu gráfico a um painel existente](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#add-pages-edit) para adicionar outra consulta NRQL ao seu monitoramento.
* [Compartilhe seu gráfico](/docs/query-your-data/explore-query-data/use-charts/use-your-charts#share-charts) com outras pessoas copiando o link permanente ou baixando-o como PDF (ou CSV, se aplicável) para permitir que outras pessoas vejam os dados de sua consulta.
* Edite a consulta NRQL atrás do gráfico para ajudar a ajustar os dados que você está vendo.
* [Crie um alerta](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition#create-simple-alert) a partir de sua consulta NRQL para ajudar a notificá-lo de forma proativa quando surgirem problemas.

Os gráficos são ótimos por si só, mas eles realmente brilham quando você os compila para criar um painel, sobre o qual você pode aprender mais nas seções abaixo.

Embora os gráficos sejam ótimos para gerar relatórios sobre peças individuais, os painéis são a melhor maneira de obter uma visão muito mais ampla dos seus dados: se os gráficos são peças de um quebra-cabeça, os painéis são o quebra-cabeça concluído. dashboard são altamente configuráveis e você pode usá-los para mostrar quantos fluxos desejar. Você pode:

* [Adicione gráficos](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#dash-create-chart) aos seus dashboards usando o criador de consulta.
* Adicione seu próprio [texto, imagens ou links](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#markdown) (usando Markdown) ao seu painel.
* Mova os gráficos em um dashboard para garantir que os dados mais importantes estejam onde você deseja.
* [Compartilhe](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data#dashboards) seu dashboard por e-mail ou crie um link permanente para seu dashboard.
* [Importe](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-import) ou [exporte](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#dash-json) seu dashboard como JSON ou exporte seu dashboard como PDF.
* [Duplique](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#dash-clone) o painel para usar como base para um totalmente novo.
* [Edite](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#add-pages-edit) os componentes do seu painel para adicionar novos fluxos de dados, modificar os existentes ou remover os mais antigos que não são mais relevantes.

## Dicas para melhorar o desempenho dashboard e do gráfico [#performance-improvement-for-dashboard-and-charts]

Se os gráficos do New Relic no painel não estiverem apresentando desempenho ideal, considere as dicas a seguir para melhorar o desempenho:

* **Usar visualização de tabela:** para exibir grandes conjuntos de dados, use a visualização de tabela em vez de visualizações de barra ou outdoor.
* **Visualizações separadas:** distribua suas visualizações em várias páginas. Um dashboard com inúmeras visualizações em uma única página pode causar problemas de desempenho, principalmente se alguns gráficos atualizam ou processam frequentemente grandes conjuntos de dados.
* **Dividir consulta NRQL :** Divida grandes consultas NRQL em consultas menores e mais gerenciáveis.
* **Limite os resultados da consulta:** como último recurso, evite usar `LIMIT MAX`. Em vez disso, especifique um limite que atenda aos seus requisitos.
