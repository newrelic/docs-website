Você pode escrever consultas <DNT>Lens</DNT> no **Query Builder**, <DNT>**Notebooks**</DNT> e em **dashboards personalizados**. Esta página explica como consultar dados em cada plataforma.

<Callout variant="tip">
  Ao consultar fontes de dados, o <DNT>Lens</DNT> não restringe o número de linhas que você pode recuperar de fontes externas. No entanto, consultas ao <DNT>NRDB</DNT> por meio do <DNT>Lens</DNT> mantêm o mesmo limite de 5.000 linhas do <DNT>NRQL</DNT>.
</Callout>

## Pré-requisitos [#prerequisites]

Antes de começar:

* Verifique se você tem as [permissões](/docs/new-relic-lens/overview#permissions) necessárias para consultar fontes de dados.
* Se você precisar consultar fontes de dados externas, certifique-se de que os [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors) estejam configurados.

## Consulta no construtor de consultas [#query-builder]

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
2. A partir da gaveta na parte inferior da página, abra o console <DNT>**Query your data**</DNT>.
3. No editor de consultas, selecione <DNT>**Lens data (SQL)**</DNT> como a linguagem de consulta.
4. Se você deseja consultar dados do NRDB, selecione a conta no seletor de contas. Outros <DNT>data connectors</DNT> estão disponíveis no nível da organização e não exigem seleção de conta.
5. Escreva sua consulta SQL usando o [formato de referência de tabela](#table-format): `connection_name.schema.table`
6. Clique em <DNT>**Run**</DNT>. O gráfico é renderizado com base nos resultados da sua consulta.

## Salvar consulta em <DNT>Notebooks</DNT> [#notebooks]

[Notebooks](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/) são documentos compartilháveis que combinam consultas, visualizações e documentação em markdown.

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
2. A partir da gaveta na parte inferior da página, abra o console <DNT>**Query your data**</DNT>.
3. Ative a chave ao lado de <DNT>**Notebooks**</DNT>. Na parte inferior da página, o botão <DNT>**+ SQL**</DNT> aparece.
4. Clique em <DNT>**+ SQL**</DNT> para adicionar um bloco SQL.
5. Se você deseja consultar dados do NRDB, selecione a conta no seletor de contas. Outros <DNT>data connectors</DNT> estão disponíveis no nível da organização e não exigem seleção de conta.
6. Escreva sua consulta SQL usando o [formato de referência de tabela](#table-format): `connection_name.schema.table`
7. Clique em <DNT>**Run**</DNT>. O gráfico é renderizado com base nos resultados da sua consulta.
8. Clique em <DNT>**Save**</DNT>.

Para mais orientações, consulte a [documentação do Notebooks](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/).

## Consulta em dashboards personalizados [#dashboards]

Você pode adicionar gráficos baseados em SQL aos seus [dashboards personalizados](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards):

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**</DNT>.
2. [Crie um novo dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create-new) ou abra um existente.
3. Clique em <DNT>**+ Add widget**</DNT> e selecione um tipo de gráfico.
4. No editor de consultas, selecione <DNT>**Lens data (SQL)**</DNT> como a linguagem de consulta.
5. Se você deseja consultar dados do NRDB, selecione a conta no seletor de contas. Outros <DNT>data connectors</DNT> estão disponíveis no nível da organização e não exigem seleção de conta.
6. Escreva sua consulta SQL usando o [formato de referência de tabela](#table-format): `connection_name.schema.table`
7. Clique em <DNT>**Run**</DNT>. O gráfico é renderizado com base nos resultados da sua consulta.
8. Clique em <DNT>**Add to dashboard**</DNT>. Para obter mais orientações, consulte a [documentação de dashboards personalizados](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/).

<Callout variant="important">
  Gráficos baseados em SQL são suportados apenas em dashboards privados. Dashboards públicos não suportam consultas SQL.
</Callout>

## Sintaxe SQL [#syntax]

<DNT>Lens</DNT> usa ANSI SQL para consultar fontes de dados, para que você não precise aprender linguagens de consulta específicas de banco de dados.

### Formato de referência de tabela [#table-format]

Ao escrever consultas em <DNT>Lens</DNT>, referencie as tabelas usando o formato:

```
connection_name.schema.table
```

Por exemplo:

* `telemetry.Transaction` - Consulte a tabela Transaction da telemetria (NRDB) <DNT>connector</DNT>
* `snowflake.sales.orders` - Consulte a tabela de pedidos de um <DNT>Snowflake</DNT> <DNT>connector</DNT> com o nome &quot;snowflake&quot; no esquema &quot;sales&quot;
* `gsheets.default.employees` - Consulte um <DNT>Google Sheets</DNT> <DNT>connector</DNT> chamado &quot;gsheets&quot;

### Instruções SQL suportadas [#supported-sql]

<DNT>Lens</DNT> suporta instruções padrão ANSI SQL, incluindo:

* `SELECT` - Consultar dados de uma ou mais tabelas
* `JOIN` - Combine dados de várias tabelas (incluindo junções entre bancos de dados)
* `WHERE` - Filtrar resultados
* `GROUP BY` - Agregar dados
* `ORDER BY` - Ordenar resultados
* `LIMIT` - Restrinja o número de linhas retornadas
* `WITH` (CTEs) - Defina expressões de tabela comuns para consultas complexas

### Dicas de desempenho [#performance-tips]

* **Use filtros antecipadamente**: Aplique cláusulas `WHERE` para reduzir a quantidade de dados processados.
* **Limitar resultados**: Use `LIMIT` para restringir as linhas retornadas ao explorar dados.
* **Considere <DNT>materialized views</DNT>**: Para consultas complexas executadas com frequência, crie um <DNT>materialized view</DNT>.
* **Otimize junções**: Ao unir tabelas grandes, filtre os dados antes da junção quando possível.

## Diferenças de sintaxe de consulta do NRQL e exemplos [#syntax-differences]

<DNT>Lens</DNT> usa ANSI SQL em vez de NRQL. Os exemplos a seguir mostram as principais diferenças e padrões de consulta comuns.

<CollapserGroup>
  <Collapser id="groupby" title="Use `GROUP BY` em vez de `FACET`">
    Ao agrupar resultados, use `GROUP BY` e inclua a coluna agrupada na sua instrução `SELECT`.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    FACET enduser.id
    ```

    **SQL:**

    ```sql
    SELECT "enduser.id", count(*)
    FROM telemetry.PageViewTiming
    GROUP BY "enduser.id"
    ```
  </Collapser>

  <Collapser id="where-timestamp" title="Use `WHERE` com `timestamp` em vez de `SINCE`">
    Filtre intervalos de tempo usando `WHERE` com a coluna `timestamp` e intervalos de datas.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    SINCE 1 day ago
    ```

    **SQL:**

    ```sql
    SELECT count(*)
    FROM telemetry.PageViewTiming
    WHERE timestamp > current_date - INTERVAL '1' DAY
    ```
  </Collapser>

  <Collapser id="timeseries" title="Séries temporais com formato epoch">
    Para criar visualizações de séries temporais, estruture sua consulta para que a primeira coluna contenha datas no formato epoch. Use `to_unixtime()` para converter datas.

    ```sql
    WITH orders AS (
      SELECT
        to_unixtime(date(date_parse(order_date, '%Y-%m-%d'))) as dt,
        count(*) as orders
      FROM gsheets.default.orders
      GROUP BY 1
    )
    SELECT *
    FROM orders
    WHERE dt >= to_unixtime(date('2025-10-19'))
      AND dt <= to_unixtime(date('2025-10-21'))
    ORDER BY dt ASC
    ```
  </Collapser>

  <Collapser id="cross-db-join" title="`JOIN` entre bancos de dados">
    Use a sintaxe padrão SQL `JOIN` para combinar dados de múltiplas fontes em uma única consulta.

    ```sql
    WITH sf AS (
      SELECT DISTINCT user_id, assigned_success_manager
      FROM demogorgon.default.relibank_opportunity_matrix
    )
    SELECT
      sf.assigned_success_manager,
      MAX(nrdb.largestContentfulPaint)
    FROM telemetry.PageViewTiming nrdb
    LEFT JOIN sf ON nrdb."enduser.id" = sf.user_id
    WHERE timingName = 'largestContentfulPaint'
      AND nrdb.timestamp > current_date - INTERVAL '1' DAY
    GROUP BY sf.assigned_success_manager
    ```
  </Collapser>

  <Collapser id="query-mv" title="Consultar visualizações materializadas">
    Referencie <DNT>materialized views</DNT> usando o prefixo `mv.`:

    ```sql
    SELECT *
    FROM mv.weekly_performance_summary
    WHERE week_number = 42
    ```

    Você pode combinar <DNT>materialized views</DNT> com outras fontes de dados:

    ```sql
    SELECT
      mv.store_id,
      mv.total_sales,
      s.store_name,
      s.region
    FROM mv.store_sales_mv mv
    JOIN snowflake.retail.stores s ON mv.store_id = s.store_id
    WHERE mv.total_sales > 10000
    ```
  </Collapser>
</CollapserGroup>
