Como usuário do New Relic, você pode precisar analisar seus dados de telemetria juntamente com dados armazenados em vários bancos de dados, data warehouses e planilhas. Exportar dados manualmente de cada fonte e combiná-los com seus dados de observabilidade offline leva tempo e reduz a produtividade. Além disso, muitas vezes você não pode consolidar todos os dados em um único banco de dados devido a requisitos de conformidade, segurança ou operacionais.

<DNT>New Relic Lens</DNT> aborda esses desafios. O <DNT>Lens</DNT> é uma plataforma de consulta unificada no New Relic que permite conectar e consultar várias fontes de dados diretamente do New Relic. Escreva uma consulta no <DNT>Lens</DNT> usando ANSI SQL e realize junções entre bancos de dados para analisar e correlacionar dados de várias fontes. <DNT>Lens</DNT> envia o trabalho computacional para cada fonte de dados, que processa sua parte da consulta e retorna apenas os resultados. O <DNT>Lens</DNT> então combina esses resultados, minimizando a transferência de dados e mantendo a segurança ao nunca ingerir dados brutos de suas fontes externas.

## Como funciona [#how-it-works]

O uso do <DNT>Lens</DNT> envolve as seguintes etapas:

<Steps>
  <Step>
    ### Habilitar Lens

    <DNT>Lens</DNT> está disponível em visualização pública. Verifique com seu <DNT>Organization Manager</DNT> para fazer a adesão na página <DNT>**[Previews &amp; Trials](https://one.newrelic.com/admin-portal/promotion-management/home)**</DNT>.
  </Step>

  <Step>
    ### Verifique as permissões

    Se você for um <DNT>Organization Manager</DNT>, você tem as permissões necessárias para acessar todos os recursos do <DNT>Lens</DNT> por padrão. Se você não for um <DNT>Organization Manager</DNT>, entre em contato com seu <DNT>Authentication Domain Manager</DNT> para [criar uma função personalizada com as permissões necessárias](/docs/new-relic-lens/overview/#permissions).
  </Step>

  <Step>
    ### Configurar <DNT>connectors</DNT>

    <DNT>Lens</DNT> inclui <DNT>system connectors</DNT> integrado para dados do NRDB. Com as permissões necessárias, você pode configurar [<DNT>data connectors</DNT>](/docs/new-relic-lens/set-up-connectors) para se conectar a fontes de dados externas:

    * **Data warehouses na nuvem**: <DNT>Snowflake</DNT>, <DNT>Redshift</DNT>, <DNT>ClickHouse</DNT>
    * **Bancos de dados relacionais**: <DNT>PostgreSQL</DNT>, <DNT>MySQL</DNT>, <DNT>SQL Server</DNT>
    * **Bancos de dados de documentos**: <DNT>MongoDB</DNT>, <DNT>Elasticsearch</DNT>
    * **Planilhas**: <DNT>Google Sheets</DNT>
    * **Data lakes**: <DNT>Iceberg</DNT>
    * **Métricas e monitoramento**: <DNT>Prometheus</DNT>, <DNT>AWS CloudWatch</DNT>
  </Step>

  <Step>
    ### Consultar dados

    Escreva consultas SQL no [Query Builder, Notebooks ou dashboards personalizados](/docs/new-relic-lens/query-data) usando o formato `connection_name.schema.table` para referenciar tabelas e unir dados de várias fontes.
  </Step>

  <Step>
    ### Criar <DNT>materialized views</DNT> (opcional)

    Para consultas complexas executadas com frequência, você pode salvar os resultados como [<DNT>materialized views</DNT>](/docs/new-relic-lens/materialized-views) para reduzir custos e melhorar o desempenho.
  </Step>
</Steps>

## Casos de uso [#use-cases]

O Lens permite vários cenários de análise de dados:

* **Análise de orçamento**: Compare os dados de utilização do New Relic de <DNT>NRDB</DNT> com os dados de orçamento armazenados em <DNT>Snowflake</DNT> para monitorar os gastos em relação aos orçamentos alocados.
* **Otimização de fluxo de trabalho**: Substitua várias etapas manuais de exportação/importação (de um banco de dados para outro) por uma única consulta SQL que une todos os dados necessários, reduzindo custos e melhorando a eficiência.
* **Agregue dados entre regiões**: Monitore o tráfego de usuários em várias regiões geográficas unindo dados de diferentes fontes de dados.
* **Otimização de desempenho com <DNT>materialized views</DNT>**: Um dashboard executivo combina dados de mais de 5 bancos de dados e atualiza semanalmente. Ao armazenar os resultados da consulta `JOIN` de múltiplos bancos de dados em um <DNT>materialized view</DNT>, você evita executar a consulta computacionalmente custosa toda vez que alguém carrega o dashboard. Isso reduz os custos de consulta e melhora significativamente a velocidade de carregamento da página.

## Onde você pode usar o Lens [#where-to-use]

Você pode usar o Lens nas seguintes áreas da plataforma New Relic:

* **Construtor de consultas**: Escreva e execute consultas SQL diretamente no construtor de consultas.
* <DNT>**Notebooks**</DNT>: Crie blocos SQL em notebooks para combinar consultas com visualizações e documentação em markdown.
* **Dashboards de Cliente**: Adicione gráficos baseados em SQL aos seus dashboards privados.

<Callout variant="important">
  Gráficos baseados em SQL não são suportados em dashboards públicos.
</Callout>

## Funções e permissões de usuário [#permissions]

<DNT>Connector</DNT> e o acesso a <DNT>materialized view</DNT> é controlado por permissões RBAC específicas. Usuários com a função <DNT>**Organization Manager**</DNT> têm todas as permissões de RBAC relacionadas a <DNT>Lens</DNT> por padrão.

Para outros usuários, o <DNT>**Authentication Domain Manager**</DNT> precisa [criar uma função personalizada com escopo de organização via NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#organization-scoped-role) com as permissões de <DNT>Lens</DNT> necessárias. Ao criar a função, adicione os IDs de permissão das tabelas abaixo ao array `permissionIds`.

<DNT>
  /* UI-based permission assignment (for future use when UI support is available): For other users, the &lt;DNT&gt;**Authentication Domain Manager**&lt;/DNT&gt; needs to create an [organization-scoped custom role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) with the required &lt;DNT&gt;Lens&lt;/DNT&gt; permissions from the &lt;DNT&gt;**Query Your Data**&lt;/DNT&gt; section. check... */
</DNT>
