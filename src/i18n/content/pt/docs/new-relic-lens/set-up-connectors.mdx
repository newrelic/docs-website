## <DNT>Data connectors</DNT> [#data-connectors]

<DNT>Data connectors</DNT> conecte <DNT>Lens</DNT> a fontes de dados externas, como bancos de dados, data warehouses e planilhas. Para criar, editar ou excluir <DNT>connectors</DNT>, você precisa de [permissões RBAC específicas](/docs/new-relic-lens/overview#permissions).

### Controle de acesso [#access-control]

<DNT>Lens</DNT> oferece duas maneiras de controlar quem pode consultar <DNT>connectors</DNT>:

* **Permissões RBAC**: Usuários com permissões RBAC <DNT>**Lens connectors**</DNT> podem consultar todos os <DNT>connectors</DNT> na organização.
* **Controle de acesso granular**: Usuários sem permissões RBAC amplas ainda podem consultar <DNT>connectors</DNT> específicos se tiverem acesso granular a esses <DNT>connectors</DNT>.

### Criar um <DNT>connector</DNT> [#create-connector]

Para configurar um <DNT>data connector</DNT>:

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Connectors**</DNT>.

2. Clique em <DNT>**Create a connector**</DNT>.

3. Selecione o seu tipo de <DNT>connector</DNT> no menu suspenso.

4. Insira as informações de conexão necessárias para o tipo de <DNT>connector</DNT> selecionado.

5. Para conceder acesso a este conector a usuários ou grupos que não possuem permissões RBAC, configure o controle de acesso granular na seção <DNT>**Access control**</DNT>:

   1. Selecione o <DNT>**Auth domain**</DNT> que contém os usuários ou grupos.
   2. Selecione <DNT>**User or group**</DNT>.
   3. Selecione a função<DNT>**Lens Viewer**</DNT>.
   4. Para adicionar mais usuários ou grupos, clique em <DNT>**Add**</DNT> e repita as etapas acima.

6. Clique em <DNT>**Create**</DNT>.

Depois de criar um <DNT>connector</DNT>:

* Para visualizar o esquema ou excluir o <DNT>connector</DNT>, selecione o menu <Icon name="fe-more-horizontal" /> na linha <DNT>connector</DNT>.
* Para atualizar as configurações de controle de acesso, selecione o <DNT>connector</DNT>, adicione ou remova concessões de acesso e salve suas alterações.
* Para atualizar a configuração do <DNT>connector</DNT>, exclua e recrie o <DNT>connector</DNT> com os novos detalhes.

    ### Gere a chave de credenciais da conta de serviço [#generate-credentials-key]

    Para autenticar <DNT>Lens</DNT> com <DNT>Google Sheets</DNT>, você precisa criar uma conta de serviço do <DNT>Google Cloud</DNT> e gerar uma chave JSON.

    <Steps>
      <Step>
        ### Criar uma conta de serviço [#create-service-account]

        Para criar uma conta de serviço <DNT>Google Cloud</DNT>, siga as instruções em [Criar contas de serviço](https://cloud.google.com/iam/docs/service-accounts-create).
      </Step>

      <Step>
        ### Configurar permissões para criação de chaves [#set-up-permissions]

        Para criar chaves para sua conta de serviço, você precisa das permissões adequadas. Para informações sobre as permissões necessárias, consulte [Permissões necessárias](https://cloud.google.com/iam/docs/keys-create-delete#required-permissions).
      </Step>

      <Step>
        ### Criar uma chave JSON [#create-json-key]

        Para gerar uma chave JSON para sua conta de serviço, siga as etapas em [Criar uma chave de conta de serviço](https://cloud.google.com/iam/docs/keys-create-delete#creating).
      </Step>

      <Step>
        ### Converta a chave JSON para base64 [#convert-json-key-to-base64]

        Para usar a chave JSON em <DNT>Lens</DNT>, converta todo o conteúdo do arquivo JSON para codificação base64 e salve-o em um arquivo de texto.
      </Step>
    </Steps>

    ### Configurar o acesso ao Google Sheets [#setup-gsheets-access]

    Após gerar a chave de credenciais, configure o acesso ao seu Google Sheets.

    <Steps>
      <Step>
        ### Compartilhe suas planilhas com a conta de serviço [#share-sheets-with-service-account]

        Para permitir que o <DNT>Lens</DNT> acesse seus dados, conceda permissões de leitura ao endereço de e-mail da conta de serviço para cada Planilha Google que você deseja conectar.
      </Step>

      <Step>
        ### Criar uma planilha de metadados [#create-metadata-sheet]

        Para gerenciar as planilhas disponíveis para <DNT>Lens</DNT>, crie uma nova planilha do Google Sheets com as quatro colunas a seguir:

        * **Nome da planilha**: O nome de cada planilha que você deseja acessar com <DNT>Lens</DNT>. Todas as planilhas listadas devem ser acessíveis pela conta de serviço.
        * **Sheetid**: O identificador exclusivo de cada planilha. Você pode encontrar isso na URL da planilha: `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`.
        * **Proprietário**: O proprietário da planilha. Use `Trino` como o valor padrão.
        * **Notas**: Quaisquer notas ou descrições adicionais para a planilha.
      </Step>

      <Step>
        ### Compartilhe a planilha de metadados [#share-metadata-sheet]

        Para permitir que <DNT>Lens</DNT> leia os metadados, conceda permissão de leitura à sua conta de serviço para a planilha de metadados.
      </Step>

      <Step>
        ### Copie o ID da planilha de metadados [#copy-metadata-sheet-id]

        Para configurar o <DNT>connector</DNT>, copie o ID da planilha da URL da planilha de metadados. O ID da planilha aparece na URL como: `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`.
      </Step>
    </Steps>

    ### Conceder <DNT>Lens</DNT> [#grant-iceberg-access]

    Para permitir que o <DNT>Lens</DNT> consulte suas tabelas Iceberg, crie uma política e uma função do IAM em sua conta da AWS que concedam acesso de leitura ao seu Glue Data Catalog e ao bucket do S3.

    <Callout variant="important">
      O Glue Data Catalog e o bucket S3 contendo seus dados Iceberg devem estar na mesma região da AWS.
    </Callout>

    <Steps>
      <Step>
        ### Criar uma política do IAM [#create-iceberg-policy]

        [Crie uma política do IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) com as seguintes permissões para permitir que <DNT>Lens</DNT> leia os metadados do Glue e os arquivos de dados no S3. Substitua `{REGION}`, `{ACCOUNT_ID}`, `{DATABASE_NAME}` e `{S3_BUCKET_NAME}` pelos seus valores. Anote o nome da política que você inserir, pois você precisará dele para anexar esta política à função do IAM.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Action": [
                "glue:GetDatabase",
                "glue:GetTable",
                "glue:GetTables",
                "glue:SearchTables",
                "glue:GetPartition",
                "glue:GetPartitions",
                "glue:BatchGetPartition"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:catalog",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:database/{DATABASE_NAME}",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:table/*"
              ]
            },
            {
              "Action": [
                "s3:GetObject",
                "s3:ListBucket",
                "s3:GetBucketLocation"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:s3:::{S3_BUCKET_NAME}",
                "arn:aws:s3:::{S3_BUCKET_NAME}/*"
              ]
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### Crie uma função do IAM com uma relação de confiança [#create-iceberg-role]

        [Crie uma função do IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) e configure a relação de confiança para permitir que a conta de serviço <DNT>Lens</DNT> a assuma usando um ID externo exclusivo. Ao configurar a função, use estes valores:

        * **Tipo de entidade confiável**: Conta da AWS
        * **ID da conta**: `017663287629` (conta da AWS da New Relic)
        * **Exigir ID externo**: Habilitado. Insira uma string exclusiva de sua escolha e salve-a para a configuração de <DNT>Lens</DNT> <DNT>connector</DNT>.
        * **Política de permissões**: Pesquise o nome da política que você criou na etapa anterior e selecione-a.

        A política de confiança deve corresponder ao seguinte formato. Substitua `<YOUR_EXTERNAL_ID>` pelo ID externo que você inseriu.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        Após criar a função, abra a página de resumo da função e copie o **Role ARN**. Você precisará deste ARN e do ID externo ao configurar o <DNT>Lens</DNT> <DNT>connector</DNT>.
      </Step>
    </Steps>
  </Collapser>

    ### Conceder <DNT>Lens</DNT> [#grant-cloudwatch-access]

    Para permitir que o <DNT>Lens</DNT> consulte suas métricas do CloudWatch, crie uma política e uma função do IAM na sua conta da AWS que concedam acesso de leitura ao CloudWatch.

    <Steps>
      <Step>
        ### Criar uma política do IAM [#create-cloudwatch-policy]

        [Crie uma política do IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) com as seguintes permissões para permitir que <DNT>Lens</DNT> leia métricas do CloudWatch da sua conta da AWS. Anote o nome da política que você inserir, pois você precisará dele para anexar esta política à função do IAM.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Action": [
                "cloudwatch:ListMetrics",
                "cloudwatch:GetMetricData",
                "cloudwatch:GetMetricStatistics",
                "tag:GetResources"
              ],
              "Resource": "*"
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### Crie uma função do IAM com uma relação de confiança [#create-cloudwatch-role]

        [Crie uma função do IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) e configure a relação de confiança para permitir que a conta de serviço <DNT>Lens</DNT> a assuma usando um ID externo exclusivo. Ao configurar a função, use estes valores:

        * **Tipo de entidade confiável**: Conta da AWS
        * **ID da conta**: `017663287629` (conta da AWS da New Relic)
        * **Exigir ID externo**: Habilitado. Insira uma string exclusiva de sua escolha e salve-a para a configuração de <DNT>Lens</DNT> <DNT>connector</DNT>.
        * **Política de permissões**: Pesquise o nome da política que você criou na etapa anterior e selecione-a.

        A política de confiança deve corresponder ao seguinte formato. Substitua `<YOUR_EXTERNAL_ID>` pelo ID externo que você inseriu.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        Após criar a função, abra a página de resumo da função e copie o **Role ARN**. Você precisará deste ARN e do ID externo ao configurar o <DNT>Lens</DNT> <DNT>connector</DNT>.
      </Step>
    </Steps>
  </Collapser>
