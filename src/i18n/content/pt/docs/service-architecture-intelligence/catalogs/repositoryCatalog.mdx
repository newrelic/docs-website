O <DNT>Repository catalog</DNT> é uma ferramenta desenvolvida para centralizar e agilizar a gestão do repositório dentro de uma organização. Ao fornecer uma visão unificada de todo o repositório, ele aprimora a colaboração entre equipes e simplifica a descoberta, o acesso e o gerenciamento do repositório de código.

O <DNT>Repository catalog</DNT> foi projetado para quebrar silos entre equipes e aumentar a colaboração entre equipes. Ele permite que desenvolvedores, engenheiros de confiabilidade do site (SREs) e outras partes interessadas identifiquem com eficiência o repositório existente, entendam sua finalidade e determinem a propriedade ou status. Essa abordagem centralizada leva a uma melhor colaboração, responsabilização e produtividade em toda a organização.

## Recurso chave [#features]

* **Descoberta de repositório centralizado:** identifique rapidamente o repositório relevante para serviços, projetos ou <DNT>[Teams](/docs/service-architecture-intelligence/teams/teams)</DNT>, minimizando o tempo gasto pesquisando ou consultando o colega.

* **Navegação de serviços e repositórios:** navegue facilmente do aplicativo ou serviços até seu repositório definidor e vice-versa, auxiliando na compreensão da dependência e infraestrutura do código.

* **Propriedade e responsabilidade:** identifique facilmente os proprietários do repositório, apoiando a consultoria direta e aumentando a responsabilidade dentro das equipes.

* **Insights do repositório:** Acesse métricas detalhadas sobre a atividade do repositório, como open pull request e histórico de implantação, para identificar projetos negligenciados e otimizar a alocação de recursos.

* **Gerenciamento e operações:** execute operações em massa, como atribuição de tags ou equipes, para gerenciar o repositório com eficiência e manter os padrões organizacionais.

## Acesse o catálogo do repositório [#access-repository]

**Pré-requisitos:**

* Habilite os Catálogos na sua conta New Relic. Para obter mais informações, consulte [Habilitar os catálogos](/docs/service-architecture-intelligence/catalogs/catalogs/#enable-catalogs).

**Para acessar o inventário do catálogo do repositório:**

1. Entre na sua plataforma New Relic.

2. Vá para o **painel de navegação esquerdo &gt; Catalogs &gt; Repository catalog**.

   <img title="Repository catalogs" alt="RepositoryCatalog" src="/images/new-relic-repository-catalog.webp" />

### Adicionar novos dados [#add-new-data]

Para adicionar dados adicionais ao seu <DNT>Repository catalog</DNT> , clique no botão **+ Add GitHub repositories** no canto superior direito. Siga as instruções na tela para adicionar dados ao seu repositório. Para obter informações adicionais sobre como adicionar repositório, consulte [GitHub integração.](/docs/service-architecture-intelligence/github-integration)

### Limitações [#limitations]

* Não há suporte para classificação de dados na página <DNT>Repository catalog</DNT> .
* Em certos cenários, as visualizações salvas podem não funcionar conforme o esperado no <DNT>Repository catalog</DNT>.
