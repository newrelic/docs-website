A integração do New Relic GitHub importa usuários e equipes da sua conta do GitHub para aumentar a visibilidade do seu [recurso de inteligência de arquitetura do Service](/docs/service-architecture-intelligence/getting-started) na plataforma New Relic .

Essa integração agiliza o processo de integração de usuários e equipes, simplifica o gerenciamento de funções e esclarece o mapeamento de propriedade de entidade, reduzindo a configuração, o tempo e o esforço de configuração. Esta integração do AI Monitoring melhora o gerenciamento e a visibilidade de [catálogos](/docs/service-architecture-intelligence/catalogs/catalogs) e [scorecards](/docs/service-architecture-intelligence/scorecards/getting-started) no New Relic. Para obter mais informações, consulte [Capacidade de inteligência de arquitetura de serviço](/docs/service-architecture-intelligence/getting-started).

**Pré-requisitos:**

* Entre na sua conta do GitHub que você planeja integrar com o New Relic.
* Você deve ser o gerente da organização ou o gerente do domínio de autenticação.

**Para configurar a integração do GitHub:**

1. Acesse **[one.newrelic.com &gt; + Integration &amp; Agents &gt; GitHub integration](https://one.newrelic.com/marketplace/install-data-source?state=9306060d-b674-b245-083e-ff8d42765e0d)**.

2. Na tela **Select an account** , mantenha a seleção de conta padrão e clique em **Continue**.

3. Na tela de **Integration setup** :

   1. Para se conectar à sua conta do GitHub, clique em **Install**. A observabilidade New Relic é aberta no GitHub Marketplace.

   2. Conclua a instalação do aplicativo em sua organização GitHub para recuperar dados relacionados ao usuário, equipes e repositório. Após a conclusão da instalação, você será redirecionado para a plataforma New Relic.

   3. Clique em **First time sync**.

   4. *(Opcional)* Clique em **On-demand sync** para sincronizar manualmente os dados.

      <Callout variant="tip" title="dica">
        Você pode sincronizar manualmente os dados uma vez a cada 4 horas. O botão de **On-demand sync** permanece desabilitado se a sincronização tiver ocorrido nas últimas 4 horas.
      </Callout>

   5. Depois de visualizar a mensagem Sincronização iniciada, clique em **Continue**. A tela de **GitHub Integration** exibirá a contagem de equipes e repositório, atualizando a cada 5 segundos. Aguarde alguns minutos para a importação completa de todos os dados.

      <img title="GitHub integration" alt="GitHub integration" src="/images/github-integration-progress.webp" />

4. Na tela de **GitHub integration** :

   * Clique em **Go to Teams** para visualizar as informações das equipes importadas no [Teams](/docs/service-architecture-intelligence/teams/teams).

   <Callout variant="tip" title="dica">
     Para atribuir automaticamente o repositório do GitHub às suas equipes, crie uma propriedade personalizada `teamOwningRepo` no GitHub com o nome da equipe como seu valor de string. A integração atribuirá automaticamente o repositório às respectivas equipes com base na propriedade personalizada.
   </Callout>
