A organização New Relic pode ter uma estrutura de conta pai/filho. Isto é relevante principalmente para organização cujos usuários estão em nosso [modelo de usuário original](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), mas pode se aplicar à organização com usuários em ambos os modelos de usuário. Aqui estão as principais diferenças em como a estrutura pai/filho se aplica a eles:

* Modelo de usuário original: nosso modelo de usuário original utiliza uma estrutura de conta pai/filho, onde as contas filho herdam configurações das contas pai, incluindo regras relacionadas ao acesso do usuário às contas. Nossas [contas de parceria](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions) também usam esse modelo. Se sua organização tiver usuários neste modelo, esses usuários poderão [criar contas filhas](#creating). Se você possui usuário neste modelo, [continue lendo abaixo](#hierarchy) para saber como funciona essa estrutura de conta.
* Modelo de usuário mais recente: Nosso modelo de usuário novo e aprimorado. Organizações com usuário neste modelo podem ter uma estrutura de conta pai/filho, mas ela não é mais relevante para gerenciar o acesso do usuário às contas e é usada principalmente para determinar relacionamentos de contas para algum recurso e para agregar faturamento em nível de organização. Para obter mais informações sobre como funciona o acesso à conta para o modelo de usuário mais recente, consulte [Estrutura da conta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure).

## Hierarquia de contas pai e filho [#hierarchy]

Para organização com usuários em nosso [modelo de usuário original](#requirements), suas contas podem ter uma estrutura de conta pai/filho, com cada conta filho anexada a uma única conta pai. Cada conta pode ter vários aplicativos reportando a ela. A conta pai também pode ter relatórios de aplicativos, mas nem todos os dados enviados dos aplicativos da conta infantil são acumulados na conta pai.

Para saber como alternar entre contas na interface, consulte [Acesso à conta](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data#account-access).

Para entender como as contas principais e secundárias são dimensionadas com as contas da região da UE, consulte [Hierarquia de contas da região da UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#global-account-hierarchy).

<Callout variant="tip">
  Se você tiver uma estrutura de conta de parceria, consulte [Contas de parceria](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions).
</Callout>

## Chave de licença para contas infantis [#keys]

Em uma organização New Relic, sua conta principal da New Relic funciona como conta principal com sua própria [chave de licença](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key). Cada conta infantil possui sua própria chave de licença individual. Cada aplicativo pode usar apenas uma chave de licença, portanto, cada aplicativo pode reportar apenas a uma conta.

## Criar contas infantis [#creating]

Para adicionar contas à sua organização New Relic, você deve ter [a edição](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) Pro ou Enterprise.

Como adicionar uma conta depende do [modelo de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) em que seus usuários estão:

* Usuários no modelo de usuário mais recente: consulte [Estrutura da conta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure/#new-model).

* Usuários no modelo usuário original: Siga estas instruções:

  1. Vá para:

     <DNT>
       **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Summary**
     </DNT>

     .

  2. Clique em

     <DNT>
       **Add account**
     </DNT>

     .

  3. Digite o nome da conta filha e selecione

     <DNT>
       **Add this account**
     </DNT>

     .

  4. Continue com os procedimentos para adicionar usuário à conta infantil.

## Adicionar usuário às contas [#users]

Para ter uma organização New Relic com mais de uma conta, você deve ter [a edição](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) Pro ou Enterprise.

Como adicionar um usuário a uma conta infantil em sua organização depende do [modelo de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) em que seu usuário está:

* Modelo de usuário mais recente: um usuário deve ser atribuído a um grupo com acesso à conta específica. Para saber mais, consulte [conceitos de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts).

* Modelo de usuário original: em geral, as permissões de um usuário em uma conta pai são herdadas automaticamente para esse usuário em contas filhas. Para adicionar usuário:

  1. Vá para:

     <DNT>
       **[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > Account settings > Account > Summary**
     </DNT>

     .

  2. Na lista de contas infantis, selecione a conta infantil.

  3. Na seção

     <DNT>
       **Active users**
     </DNT>

     da conta secundária, selecione

     <DNT>
       **Add user**
     </DNT>

     , preencha o e-mail, nome, função e cargo do usuário conforme apropriado e selecione

     <DNT>
       **Add this user**
     </DNT>

     .

  4. Opcional: [altere as funções de Proprietário ou Administrador](/docs/accounts-partnerships/accounts/account-setup/adding-updating-users) da conta filha conforme apropriado.

## Configurar contas com SAML SSO [#saml]

Como configurar o SAML SSO dependerá do [modelo de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) em que seus usuários estão:

* Modelo de usuário original: se você [configurar o SSO SAML](/docs/accounts-partnerships/accounts/saml-single-sign/setting-sso) no nível da conta pai, os usuários da sua conta filha também poderão fazer login usando o SSO. Se você

  <DNT>
    **don't**
  </DNT>

  definir o SSO SAML no nível pai, poderá defini-lo no nível da conta filha. No entanto,

  <DNT>
    **do not**
  </DNT>

  defina o SSO SAML em ambos os níveis. Para obter mais informações, consulte [Configurando SAML com diversas contas](/docs/accounts-partnerships/accounts/saml-single-sign/configuring-saml-multiple-accounts).

* Modelo de usuário mais recente: consulte [SAML SSO](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more).

## Atualizar aplicativo de conta infantil [#subaccount_apps]

<DNT>
  **Requirements:**
</DNT>

* [Edição](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions) Pro ou Enterprise.
* Você deve estar em nosso [modelo de usuário original](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) (se estiver em [nosso modelo de usuário mais recente](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) e quiser adicionar contas, fale com seu representante de conta).
* Deve ser proprietário ou administrador.

Quando você altera o arquivo de configuração do aplicativo para movê-lo de uma conta para outra, o New Relic o trata como um novo aplicativo na nova conta infantil. A New Relic não move nenhum dado histórico para o app.

Para instalar um novo aplicativo para uma conta infantil:

1. Certifique-se de ter selecionado a conta pai: No [alternador de contas](/docs/new-relic-solutions/get-started/glossary/#account-switcher), selecione a conta desejada.

2. No [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), selecione

   <DNT>
     **Account settings**
   </DNT>

   .

3. Siga os procedimentos padrão para instalar o novo aplicativo.
