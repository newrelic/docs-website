A New Relic tem dois modelos de usuário: um modelo de usuário mais antigo, conhecido como <DNT>**original user model**</DNT> , e nosso modelo de usuário mais recente, lançado em julho de 2020.

## Determine seu modelo de usuário [#determine-user-model]

Quando você log no New Relic, o registro do usuário associado a esse login está em um modelo de usuário ou em outro. Para determinar em qual modelo de usuário você está:

* Vá para o canto inferior esquerdo e clique no [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings).

* Se você vir uma guia

  <DNT>
    **Administration**
  </DNT>

  , você está em nosso modelo de usuário mais recente. Se você vir uma guia

  <DNT>
    **Account settings**
  </DNT>

  , você está em nosso modelo de usuário original.

## Encontre os documentos certos [#docs]

O modelo de usuário em que você está afeta como seus usuários são gerenciados ([leia sobre as diferenças do modelo de usuário](#differences)). Temos diferentes conjuntos de documentos para os dois modelos de usuário:

* [Documentos para nosso modelo de usuário original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model)
* [Documentos para nosso modelo de usuário mais recente](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-managing-users)

Não tem certeza de qual modelo de usuário você está? Consulte [Determinar o modelo do usuário](#determine-user-model).

## Interface de gerenciamento de usuários [#manage-users]

Para saber como usar a interface de gerenciamento de usuários:

* Para usuário em nosso modelo de usuário mais recente: consulte [Gerenciar usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#where).

* Para usuários em nosso modelo usuário original: em

  <DNT>
    **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**
  </DNT>

  , clique no [menu usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), clique em

  <DNT>
    **Account settings**
  </DNT>

  e depois clique em

  <DNT>
    **Users and roles**
  </DNT>

  . Para obter informações, consulte [Usuário original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/).

## Comparação de modelo de usuário [#differences]

Quando você log no New Relic, o registro de usuário associado ao seu login está em um modelo de usuário ou em outro. Aqui está uma explicação das diferenças entre os modelos de usuário:

* <DNT>
    **Newer user model**
  </DNT>

  ([ver documentos](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-managing-users)): este modelo de usuário mais novo e [aprimorado](https://www.newrelic.com/blog/how-to-relic/user-management-and-access-security) foi lançado em julho de 2020. É o modelo de usuário padrão para novas inscrições no New Relic. Aqui estão as principais diferenças em relação ao modelo de usuário original:

  * Todas as suas contas e usuários estão contidos em uma [organização](/docs/new-relic-account-structure) de nível superior. Isso dá à organização um controle mais forte sobre o gerenciamento do usuário e o que ele pode acessar. Um impacto disso é que o usuário que trabalha com múltiplas organizações da New Relic pode ter um endereço de e-mail associado a [vários logins](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data/#account-access).
  * O usuário neste modelo possui um sistema diferente para gerenciar permissões e acesso à conta. [Saiba como funciona o acesso do usuário.](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#understand-concepts)

* <DNT>
    **Original user model**
  </DNT>

  ([ver documentos](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/)): alguns aspectos deste modelo antigo que são diferentes do nosso modelo mais recente:

  * Não havia tanto controle no nível da organização sobre o usuário. Por exemplo, um usuário do New Relic tinha a capacidade de acessar várias contas às quais tinha acesso usando um único login.
  * Os usuários no modelo usuário original possuem um [sistema de gerenciamento de usuário](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model) diferente.

Para saber mais sobre os benefícios do novo modelo, consulte nossa [postagem no blog sobre alterações no modelo do usuário](https://www.newrelic.com/blog/how-to-relic/user-management-and-access-security) e uma [explicação da nova estrutura da conta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure). Para impactos e limitações, consulte [Impactos do recurso](#limitations).

## Requisitos para nosso modelo de usuário mais recente [#requirements]

Aqui estão os fatores que regem qual organização tem o usuário no modelo mais recente:

* Todas as organizações da New Relic que se inscreveram após 30 de julho de 2020 possuem usuário neste modelo (e também possuem o [modelo de precificação mais recente](#pricing-plans)).
* Algumas organizações mais antigas da New Relic tiveram seus usuários migrados para o novo modelo pela New Relic ou usando o [procedimento de migração de usuário](/docs/accounts/original-accounts-billing/original-users-roles/user-migration).
* Alguns parceiros New Relic (por exemplo, revendedores, provedores de serviço gerenciados) tiveram seus usuários migrados para o novo modelo. (Se você for uma organização New Relic com usuário no modelo original e precisar de multilocação, entre em contato com seu representante de conta para obter mais informações.)

## Impactos do recurso do modelo do usuário [#limitations]

O novo modelo de usuário oferece [muitos benefícios](https://newrelic.com/blog/how-to-relic/user-management-and-access-security). Aqui estão alguns impactos de recursos que você pode gostar de saber antes de decidir mudar:

* Para notificação de alerta clássica (não nova [notificação](/docs/alerts-applied-intelligence/notifications/intro-notifications)): o usuário no novo modelo não aparecerá como opção ao [escolher usuário para receber notificação de alerta](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/#user). O resultado disso é que esses usuários não receberão [notificação push do aplicativo New Relic Mobile ](/docs/mobile-apps/new-relic-mobile-apps/authentication-alerts/alerting-new-relic-mobile-apps). Como solução alternativa, você pode usar o [endereço de e-mail](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/#email) como canal de notificação.
* Você não pode usar a API REST para [gerar uma lista de usuários](/docs/apis/rest-api-v2/account-examples-v2/listing-users-your-account/) (somente usuário modelo original). Em vez disso, você usaria a [interface de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks#where).

## Relação entre modelo de usuário e modelo de precificação [#relation-to-pricing]

O modelo de usuário não está diretamente relacionado aos nossos dois modelos de precificação. Para obter informações sobre como eles se relacionam, consulte [Modelo de preços e modelo de usuário](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models#pricing-user-table).
