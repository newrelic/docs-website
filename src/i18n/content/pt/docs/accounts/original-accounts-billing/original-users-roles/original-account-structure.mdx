<Callout variant="important">
  Este documento se aplica à organização com usuários em nosso [modelo de usuário original](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).
</Callout>

Desde 30 de julho de 2020, novas inscrições no New Relic têm usuários em nosso modelo de usuário mais recente. Clientes mais antigos que ainda não [migraram seus usuários para o novo modelo](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) ainda estão em nosso modelo de usuário original. O modelo de usuário original será cada vez mais obsoleto à medida que mais clientes da New Relic migrarem para o novo modelo.

Aqui estão alguns aspectos importantes do modelo de usuário original e links para documentos importantes:

* Nosso modelo de usuário original não tinha limites firmes entre organização/clientes e era muito mais centrado no usuário. Isto significava que um utilizador poderia ter acesso a dados de diferentes organizações e aceder a esses dados a partir de um único login. [Leia mais sobre as diferenças entre o modelo antigo e o novo](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).
* No modelo de usuário original, o acesso de um usuário às contas é baseado em: a) ser adicionado a uma conta específica ou b) ter acesso a uma conta pai e herdar o acesso às contas filhas dessa conta. Veja nossos [documentos originais de gerenciamento de usuários](/docs/accounts/original-accounts-billing/original-users-roles). Veja uma [explicação de como funcionam as contas pai/filho](/docs/accounts/original-accounts-billing/original-users-roles/mastersub-account-structure).
