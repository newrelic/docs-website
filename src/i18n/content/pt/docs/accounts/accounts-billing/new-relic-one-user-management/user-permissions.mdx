Em um contexto de gerenciamento de usuários do New Relic, um <DoNotTranslate>**permission**</DoNotTranslate> é algo granular que você pode fazer com o New Relic que disponibilizamos para adição a uma função personalizada. Essas permissões também estão anexadas às nossas [funções padrão](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles) pré-construídas. Exemplos de permissões são: a capacidade de visualizar as configurações do aplicativo APM, ou modificar a condição do alerta, ou gerenciar configurações de retenção de dados.

Para saber quais permissões uma função possui, acesse a interface de gerenciamento do usuário e visualize uma função específica. Para encontrar esta interface: Em <DoNotTranslate>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DoNotTranslate>, clique no [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) no canto inferior direito e vá para: <DoNotTranslate>**Administration > Access management > Roles**</DoNotTranslate>.

## O que são permissões? [#permissions-roles]

Um usuário da plataforma completa New Relic sem limitações (por exemplo, um usuário do grupo <DoNotTranslate>**Admin**</DoNotTranslate> ) pode utilizar todos os recursos da plataforma. Disponibilizamos algumas das coisas que você pode fazer no New Relic como <DoNotTranslate>**permissions**</DoNotTranslate>. Você pode adicioná-los ou removê-los de uma função personalizada, e também usamos essas permissões para diferenciar nossas [funções padrão](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles). As permissões que tornamos visíveis e disponíveis para seleção são aquelas que acreditamos que você achará mais úteis para tarefas comuns de gerenciamento de usuários.

Há muitas funcionalidades do New Relic que <DoNotTranslate>**don't**</DoNotTranslate> tornamos visíveis e disponíveis para seleção. Por exemplo, existem várias páginas de interface que você pode acessar como qualquer usuário e que não são controladas pelas permissões que expomos. Outro exemplo: um usuário em um grupo com [configurações de administração](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) <DoNotTranslate>**Organization**</DoNotTranslate> pode definir configurações no nível da organização e esse poder não está disponível para adição a uma função personalizada.

<Callout variant="tip">
  Às vezes, as permissões também podem ser chamadas de <DoNotTranslate>**capabilities**</DoNotTranslate>.
</Callout>

## Pontos importantes sobre permissões [#important-points]

Aqui estão alguns outros pontos importantes sobre permissões:

* <DoNotTranslate>**A user's user type must also allow access.**</DoNotTranslate>

  O acesso de um usuário ao recurso New Relic é governado pelo tipo de usuário e pelas funções atribuídas. Para obter mais informações, consulte [Acesso do usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

* <DoNotTranslate>**Some permissions overlap in functionality.**</DoNotTranslate>

  É por isso que selecionar algumas caixas de seleção de permissões na interface marcará ou desmarcará automaticamente outras caixas.

* <DoNotTranslate>**Permissions don't affect querying of data.**</DoNotTranslate>

  A maioria das permissões se aplica à interface do New Relic e às experiências de API, e não à consulta de dados. Por exemplo, se suas permissões restringirem o acesso à interface

  <InlinePopover type="apm"/>

  , você ainda poderá consultar os dados do APM se tiver acesso a essa conta. Se você precisar de limites de dados mais firmes para alguns projetos ou usuários, poderá segmentar seus dados em [contas diferentes](/docs/accounts/accounts-billing/account-structure/add-accounts).

Para saber mais sobre as principais formas de controle das permissões dos usuários, consulte [Conceitos de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Nossos papéis pré-construídos [#pre-built-roles]

Nossas funções predefinidas possuem vários agrupamentos de permissões. O modo como nossas funções predefinidas funcionam é diferente dependendo de qual dos nossos [modelos de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) você está:

* Nosso modelo de usuário mais recente (a maioria dos usuários): [aprenda sobre funções](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).
* Nosso modelo de usuário original: [aprenda sobre funções](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model).

## Definições de permissão [#permission-definitions]

Você pode acessar a interface para visualizar as permissões para cada uma de nossas funções predefinidas. No canto inferior esquerdo da [interface](https://one.newrelic.com), clique no seu nome para abrir o menu do usuário e depois vá em **Administration > Access management > Roles**. Listas de permissões estão disponíveis para as seguintes funções:

* Todos administradores de produtos
* Usuário padrão
* Somente leitura
