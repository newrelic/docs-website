No contexto de gerenciamento de usuários New Relic, uma permissão é uma tarefa específica que você pode realizar com New Relic. Diversas permissões estão incluídas em nossas funções pré-configuradas (por exemplo, consulte nossas [funções padrão](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles)). Aqui estão alguns exemplos de permissões:

* A capacidade de visualizar as configurações do aplicativo APM.
* Modificar condição do alerta
* Gerenciar configurações de retenção de dados
* Criar domínio de autenticação a nível organizacional
* Implantar frotas individuais em recursos específicos

Você pode criar funções personalizadas em três escopos diferentes e adicionar qualquer número de permissões a elas:

* **permissões no escopo da organização**: Para funções administrativas em toda a organização (gerenciamento de identidade e acesso, New Relic One, chave de API, segurança, etc.)
* **Permissões com escopo de conta**: Para recursos da plataforma dentro de contas específicas (APM, Browser, infraestrutura, alertas, etc.)
* **Permissões com escopo de entidade**: Para acesso granular a recursos específicos.

Para saber quais permissões uma função possui, acesse a interface de gerenciamento do usuário e visualize uma função específica. Para encontrar esta interface: Em <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, clique no [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) no canto inferior direito e vá para: <DNT>**Administration &gt; Access management &gt; Roles**</DNT>.

## Pontos importantes sobre permissões [#important-points]

Um usuário completo da plataforma New Relic sem limitações (por exemplo, um usuário no grupo <DNT>**Admin**</DNT>) é capaz de usar todos os recursos da plataforma. Muitas, mas não todas, das tarefas que você pode realizar no New Relic estão disponíveis como <DNT>permissions</DNT>. Você pode adicionar ou remover essas permissões de uma função personalizada, e também usamos essas permissões para diferenciar nossas [funções padrão](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles). As permissões que tornamos visíveis e disponíveis para seleção são aquelas que acreditamos serem mais úteis para tarefas comuns de gerenciamento de usuários.

Existem muitas funcionalidades do New Relic que não tornamos visíveis e disponíveis como permissões que você pode selecionar. Por exemplo, existem várias páginas da interface do usuário que você pode acessar como qualquer usuário e que não são restritas pelas permissões que expomos.

<Callout variant="tip">
  Às vezes, as permissões também podem ser chamadas de <DNT>**capabilities**</DNT>.
</Callout>

Aqui estão alguns outros pontos importantes sobre permissões:

* <DNT>**A user&apos;s user type must also allow access.**</DNT> O acesso de um usuário ao recurso New Relic é governado pelo tipo de usuário e pelas funções atribuídas. Para obter mais informações, consulte [Acesso do usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).
* <DNT>**Some permissions overlap in functionality.**</DNT> É por isso que selecionar algumas caixas de seleção de permissões na interface marcará ou desmarcará automaticamente outras caixas.
* <DNT>**Permissions don&apos;t affect querying of data.**</DNT> A maioria das permissões se aplica à interface do New Relic e às experiências de API, e não à consulta de dados. Por exemplo, se suas permissões restringirem o acesso à interface <InlinePopover type="apm" />, você ainda poderá consultar os dados do APM se tiver acesso a essa conta. Se você precisar de limites de dados mais firmes para alguns projetos ou usuários, poderá segmentar seus dados em [contas diferentes](/docs/accounts/accounts-billing/account-structure/add-accounts).

Para saber mais sobre as principais formas de controle das permissões dos usuários, consulte [Conceitos de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Nossos papéis pré-construídos [#pre-built-roles]

Nossos perfis predefinidos possuem diversos agrupamentos de permissões. Veja [mais sobre funções](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Definições de permissão [#permission-definitions]

Você pode acessar a interface do usuário para visualizar as permissões de cada uma de nossas funções predefinidas. No canto inferior esquerdo da [interface](https://one.newrelic.com) do usuário, clique no seu nome para abrir o menu do usuário e, em seguida, acesse **Administration &gt; Access management &gt; Roles**.

A interface do usuário fornece informações detalhadas sobre as permissões para todas as funções, incluindo:

* Todos administradores de produtos
* Usuário padrão
* Somente leitura
* Todas as funções personalizadas em sua organização

A interface de gerenciamento de acesso exibe os detalhes de permissão mais recentes para cada função.
