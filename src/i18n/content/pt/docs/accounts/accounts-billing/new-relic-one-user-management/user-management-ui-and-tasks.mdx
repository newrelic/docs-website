Este documento explica como [encontrar e usar a interface de gerenciamento de usuários](#where) e como executar algumas [tarefas comuns de gerenciamento de usuários](#workflow).

## Interface de gerenciamento de usuários [#where]

Para encontrar as opções de gerenciamento de usuários da sua organização New Relic: No [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), selecione <DNT>**Administration**</DNT>. Aqui estão as páginas da interface relacionadas ao gerenciamento de usuários que você pode encontrar lá:

* <DNT>
    **User management**
  </DNT>

  ([link para interface](https://one.newrelic.com/admin-portal/organizations/users-list)): Use isto para adicionar usuários, atualizar o tipo de usuário, gerenciar grupos de usuários, aprovar solicitações de atualização de usuários e controlar configurações de sessão.

* <DNT>
    **Invitation approval**
  </DNT>

  : É aqui que você pode aprovar a adição do usuário ao New Relic. Esses usuários ainda não são considerados provisionados e não são faturáveis.

* <DNT>
    **Access management**
  </DNT>

  ([link para interface](https://one.newrelic.com/admin-portal/organizations/organization-detail)): Use esta interface para visualizar, criar e gerenciar [grupos e funções](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

* <DNT>
    **API keys**
  </DNT>

  : Utilize a [interface da chave de API](/docs/apis/intro-apis/new-relic-api-keys) para visualizar e gerenciar a chave de API do seu usuário.

* <DNT>
    **Domain capture**
  </DNT>

  : Use a captura de domínio para direcionar seu usuário ao lugar certo quando ele tentar se inscrever no New Relic. [Saiba mais sobre captura de domínio.](/docs/accounts/accounts-billing/account-setup/domain-capture)

* <DNT>
    **Authentication domains**
  </DNT>

  ([link para interface](https://one.newrelic.com/admin-portal/centralized-admin-user-auto-provisioning/home)): Use isto para controlar como seus usuários são adicionados (do New Relic ou de um provedor de identidade), como eles log (manualmente ou via SAML SSO) e muito mais. Para obter detalhes, consulte [Domínio de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more).

Para dicas sobre como gerenciar usuários, consulte [Tarefas comuns](#workflow) e nossos [vídeos sobre gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-mgmt-videos).

## Requisitos [#requirements]

A maneira mais fácil de ver quais são as permissões do seu usuário de gerenciamento é acessar a [interface de gerenciamento do usuário](#where) e ver a que você tem acesso.

Alguns requisitos e restrições de gerenciamento de usuários:

* Para adicionar e editar grupos e funções, sua organização deve ter a edição [Pro ou Enterprise](https://newrelic.com/pricing) .

* Requisitos relacionados às permissões:

  * Requer um [tipo de usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) de usuário principal ou usuário de plataforma completa.
  * A maioria das permissões de gerenciamento de usuários exige [configurações de administração](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) relacionadas ao domínio de autenticação e organização.

* Para evitar conflitos de configuração, tente garantir que apenas uma pessoa na sua organização gerencie o usuário por vez.

## Detalhes da interface [#user-mgmt]

Aqui está uma captura de tela da interface <DNT>**User management**</DNT> . Explicaremos os cabeçalhos das colunas abaixo.

<img
  title="User management UI"
  alt="User management UI"
  src="/images/accounts_screenshot-crop_user-mgmt-ui.webp"
/>

* <DNT>
    **Name**
  </DNT>

  : o nome do usuário.

* <DNT>
    **Email**
  </DNT>

  : o endereço de e-mail do usuário. Para usuários com `Pending`: Este usuário foi adicionado, mas ainda não verificou seu e-mail. Mesmo no estado `Pending` , um usuário é considerado provisionado e, portanto, faturável se for um usuário principal ou completo da plataforma.

* <DNT>
    **Type**
  </DNT>

  : O [tipo de usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) do usuário.

* <DNT>
    **Last active**
  </DNT>

  : a última data em que um usuário fez login na plataforma. (Isso **não** rastreia o uso de nossa API que exige um

  <InlinePopover type="userKey"/>

  .) Isso pode ser útil para decidir fazer o downgrade de um usuário para um tipo de usuário inferior.

* <DNT>
    **ID**
  </DNT>

  : ID atribuído ao New Relic de um usuário.

Para obter informações sobre os grupos em que o usuário está, você pode baixar um arquivo TSV da interface ou [usar o NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#query-user-groups).

## Dicas sobre tarefas comuns de gerenciamento de usuários [#workflow]

Aqui estão algumas tarefas de gerenciamento de usuários que você pode querer realizar:

<CollapserGroup>
  <Collapser
    id="learn-about-users"
    title="Saiba mais sobre o usuário"
  >
    Para saber mais sobre seu usuário, acesse <DNT>**User management UI**</DNT> e examine o usuário específico para obter detalhes. Você também pode acessar a interface <DNT>**Access management**</DNT> para examinar grupos: as funções que esses grupos possuem e o usuário em cada grupo.

    Para ver todos os seus usuários e suas participações em grupos, você pode [usar NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups#query-user-groups).
  </Collapser>

  <Collapser
    id="add-users"
    title="Adicionar usuário"
  >
    Observe que adicionar usuário diretamente ao New Relic significa que usuários com um tipo de usuário faturável são imediatamente faturáveis, independentemente de serem `Pending` e ainda não terem feito login no New Relic. As opções para adicionar usuários por meio da interface de gerenciamento de usuários incluem:

    * Na [interface](#where)

      <DNT>
        [**User management**](#where)
      </DNT>

      [](#where), selecione

      <DNT>
        **Add user**
      </DNT>

      .

    * No

      <DNT>
        [**Access management UI**](#where)
      </DNT>

      , ao criar ou editar um grupo, adicione um usuário a esse grupo.

      Você também pode usar o recurso <DNT>**Invite users**</DNT> . Para usar isso, acesse o [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) e clique em <DNT>**Add user**</DNT>. Detalhes sobre como convidar o usuário:

    * Se você tiver [permissões](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings)

      <DNT>
        [**Authentication domain**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings)
      </DNT>

      [](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings), será direcionado para a interface

      <DNT>
        **Add users**
      </DNT>

      , onde você poderá adicionar o usuário diretamente.

    * Se você não tiver essa permissão, poderá convidar um colega de equipe. Isso solicita a revisão por um administrador do usuário que está sendo adicionado ao New Relic. Os convites pendentes que não foram revisados são marcados como [`Pending invite approval` na interface de gerenciamento do usuário](#user-mgmt). Quando um administrador analisa um usuário convidado, ele conclui o fluxo de trabalho

      <DNT>
        **Add user**
      </DNT>

      e esse usuário é considerado adicionado ao New Relic.

      Alguns pontos importantes sobre a adição de usuário:

    * Você também pode usar [nossa API NerdGraph para gerenciar usuários](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).

    * Se você estiver usando [o gerenciamento automatizado de usuários](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign), só poderá adicionar usuários do seu provedor de identidade.
  </Collapser>

  <Collapser
    id="edit-user-type"
    title="Editar o tipo de um usuário"
  >
    Antes de alterar o tipo de usuário de um usuário, recomendamos que você entenda:

    * [Como decidir sobre um tipo de usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#choose-user-type).
    * [Como os usuários são calculados](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing), incluindo dicas sobre como adicionar novos usuários ou ajustar seu tipo de usuário.
    * [Regras de downgrade do usuário](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#user-downgrade-rules).
    * Usando [gerenciamento automatizado de usuários](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign)? Você tem [outras opções para gerenciar o tipo de usuário](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).

      Para alterar o tipo de usuário de usuários na interface:

    1. Na [interface](#where)

       <DNT>
         [**User management**](#where)
       </DNT>

       [](#where), clique nas caixas de seleção dos usuários cujo tipo de usuário você deseja editar.

    2. Assim que você começar a selecionar o usuário, uma opção aparecerá para

       <DNT>
         **Edit type**
       </DNT>

       .

       Você também pode editar o tipo de usuário e o grupo de um usuário específico clicando nesse usuário.

       Para saber como usar nossa API para gerenciar informações de usuários, consulte [Gerenciar usuários com NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).

       Para gerenciar como os usuários atualizam seu tipo de usuário, consulte as [configurações do domínio de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#user-upgrade).
  </Collapser>

  <Collapser
    id="bulk-edit-users"
    title="Editar usuário em massa"
  >
    Para editar as configurações de vários usuários ao mesmo tempo:

    1. Acesse a [interface](#where)

       <DNT>
         [**User management**](#where)
       </DNT>

       [](#where)e marque as caixas de seleção dos usuários cujo grupo ou tipo de usuário você deseja editar.

    2. As opções de edição aparecerão na parte superior da interface.

       Observação: se você estiver usando [o gerenciamento automatizado de usuários](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign), os grupos deverão ser editados no seu provedor de identidade.
  </Collapser>

  <Collapser
    id="add-user-managers"
    title="Atribuir usuário para gerenciar outro usuário"
  >
    Existem duas opções para dar ao seu usuário a capacidade de gerenciar outro usuário:

    * As [configurações de domínio de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) concedem ao usuário recursos em toda a organização relacionados ao gerenciamento de usuários, grupos e domínios de autenticação.

    * A função [de administrador do grupo](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#group-admin) é muito mais restrita: dá ao usuário a capacidade de adicionar e remover usuários de um grupo específico.

      Para fornecer recursos de gerenciamento de usuários para toda a organização, você pode adicionar um usuário ao grupo <DNT>**Admin**</DNT> padrão, que tem essas configurações ativadas. Você também pode criar um grupo personalizado e definir essas configurações.

      Para obter um tutorial sobre tarefas comuns de gerenciamento de usuários, consulte o [tutorial de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial).
  </Collapser>

  <Collapser
    id="delete-users"
    title="Excluir usuário"
  >
    Para excluir um usuário: acesse a [interface do usuário<DNT>**User management**</DNT> ](#where)e marque as caixas de seleção de um ou mais usuários. Em seguida, clique em <DNT>**Delete users**</DNT>.
  </Collapser>

  <Collapser
    id="export-users"
    title="Exportar uma planilha do usuário"
  >
    Para exportar uma lista de todos os usuários e seus detalhes (nomes, endereços de e-mail, tipo de usuário atual, grupos, ID do usuário): Na [interface do usuário<DNT>**User management**</DNT> ](#where), clique no ícone de download ao lado do botão <DNT>**Add user**</DNT> . Existe a opção de baixar uma lista de todos os usuários desse domínio de autenticação.

    Alternativamente, para exportar um subconjunto de usuário, primeiro selecione o usuário que deseja exportar e, em seguida, clique no botão de download.
  </Collapser>

    id="access-grants"

  </Collapser>

    id="new-group"
