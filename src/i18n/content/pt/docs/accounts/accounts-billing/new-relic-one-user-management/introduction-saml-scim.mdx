Para configurar controles automáticos sobre como seus usuários New Relic são adicionados ao New Relic, como são gerenciados e como fazem log , oferecemos estes recursos:

* <DNT>**SAML SSO**</DNT>: permite que seu usuário use um serviço de provedor de identidade de logon único (SSO) para login no New Relic, em vez de usar o e-mail + senha padrão. Requer uma [edição paga](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).
* <DNT>**SCIM provisioning**</DNT>, também conhecido como gerenciamento automatizado de usuários: o provisionamento SCIM permite que a organização use seu serviço de provedor de identidade para automatizar a forma como seus usuários são adicionados e atualizados no New Relic. organização que usa SCIM quase sempre usará SAML SSO. Requer [Pro ou edição Enterprise](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).

Para obter informações sobre como habilitar o SAML e o SCIM, você usará a [interface do usuário do domínio de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) para configurar o SAML e o SCIM. Observe que Okta, OneLogin e Azure têm aplicativos New Relic dedicados: se você usá-los, deverá escolher aquele chamado **New Relic by organization**, e não **New Relic by account**.
