Para configurar controles automáticos sobre como seus usuários New Relic são adicionados ao New Relic, como são gerenciados e como fazem log , oferecemos estes recursos:

* <DNT>
    **SAML SSO**
  </DNT>

  : permite que seu usuário use um serviço de provedor de identidade de logon único (SSO) para login no New Relic, em vez de usar o e-mail + senha padrão. Requer uma [edição paga](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).

* <DNT>
    **SCIM provisioning**
  </DNT>

  , também conhecido como gerenciamento automatizado de usuários: o provisionamento SCIM permite que a organização use seu serviço de provedor de identidade para automatizar a forma como seus usuários são adicionados e atualizados no New Relic. organização que usa SCIM quase sempre usará SAML SSO. Requer [Pro ou edição Enterprise](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions).

Para obter informações sobre como habilitar SAML e SCIM, escolha seu modelo de usuário abaixo. Se você não tiver certeza de qual está usando, consulte [Visão geral dos modelos de usuário](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).

<CollapserGroup>
  <Collapser
    id="new-account-model"
    title="Nosso modelo de usuário mais recente"
  >
    Para usuários em [nosso modelo de usuário mais recente](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), você usará a [interface do domínio de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) para configurar SAML e SCIM. Observe que Okta, OneLogin e Azure têm aplicativos New Relic dedicados: se você usá-los, deverá escolher aquele chamado "New Relic por organização" e não "New Relic por conta".
  </Collapser>

  <Collapser
    id="original-account-model"
    title="Modelo de usuário original"
  >
    Se sua organização estiver no modelo de usuário original, recomendamos migrar seus usuários por meio do [procedimento de migração de autoatendimento](/docs/accounts/original-accounts-billing/original-users-roles/user-migration) , se possível. Se você não atende aos requisitos, mas deseja ativar SAML e/ou SCIM, fale com seu representante de conta da New Relic.

    Aqui estão algumas dicas e links de documentos:

    * SAML: se você quiser ou precisar permanecer no modelo de usuário original e não puder migrar para o novo modelo, consulte nossos [documentos SAML originais](/docs/accounts/accounts/saml-single-sign/saml-service-providers).
    * SCIM + SAML: o provisionamento SCIM cria registros de usuário em nosso modelo de usuário mais recente. É por isso que recomendamos que você primeiro migre seus usuários para o novo modelo de usuário antes de usar o SCIM. Para saber mais sobre como migrar para o novo modelo, consulte [Migrar usuário](/docs/accounts/original-accounts-billing/original-users-roles/user-migration).
  </Collapser>
</CollapserGroup>
