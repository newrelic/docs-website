Como usuário da New Relic, você pode ter mais de um registro de usuário usando o mesmo endereço de e-mail. Este seria o caso se você tivesse:

* Acesso a mais de uma [organização New Relic](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure)
* Acesso a mais de um [domínio de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) em uma organização New Relic

## Como contamos e rastreamos usuários [#user-records]

Quando um usuário é criado no New Relic, esse usuário recebe um ID de usuário. Para encontrar o ID de um usuário, você pode acessar a [interface<DNT>**User management**</DNT> ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks)e procurar as informações de um usuário específico.

<Callout variant="tip">
  Uma maneira de encontrar seu próprio ID de usuário é inspecionar o código-fonte para um carregamento de página da interface do New Relic e pesquisar `userId` nessa página.
</Callout>

Para determinar o número de usuários em uma organização New Relic para fins de cobrança, contamos os IDs dos usuários e, em seguida, eliminamos a duplicação dos usuários que possuem os mesmos endereços de e-mail.

Ao atualizar seu endereço de e-mail no New Relic, isso não causará cobrança dupla. Se você estiver atualizando seu email e tiver vários registros de usuário em uma organização, certifique-se de atualizá-lo para todos os registros de usuário.

## Alternar entre registros de usuário [#switch]

Ao log no New Relic, você pode acessar o [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) para alternar para outros registros de usuário que tenham o mesmo endereço de e-mail. Ao escolher um registro de usuário no menu, você será solicitado a login nesse registro de usuário (diretamente na New Relic ou por meio de seu provedor de identidade).

Para acessar registros de usuários que possuem um endereço de e-mail diferente, você deve log e log novamente.

## Mensagem "Vários logins encontrados" [#login-message]

Se seu e-mail estiver associado a vários registros de usuário, você verá uma mensagem `Multiple logins found` ao fazer login e será solicitado a verificar seu e-mail. Isso pode acontecer se você tiver acesso a diversas organizações New Relic ou a diversos [domínios de autenticação](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) em uma organização.

<figcaption>
  Quando você tiver dois ou mais registros de usuário para o mesmo endereço de e-mail, receberá uma mensagem sobre isso durante o login.
</figcaption>

Quando um registro de usuário é marcado como `Original` na página de login, conforme mostrado acima, isso significa que o registro do usuário está em nosso [modelo de usuário original](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models).

A verificação do seu endereço de e-mail permitirá que você escolha qual registro de usuário acessar. Se você não verificar seu endereço de e-mail, a New Relic tentará log com o registro de usuário criado mais recentemente associado ao seu endereço de e-mail.

Para evitar a necessidade de verificar seu e-mail em cada login, clique na caixa de seleção <DNT>**Remember me**</DNT> na tela de login.

Para obter mais informações sobre problemas de login, consulte [Solucionar problemas de login](/docs/accounts/accounts-billing/account-setup/troubleshoot-new-relics-password-email-address-login-problems).

## Excluir registros de usuário [#delete]

Se você se inscreveu acidentalmente no New Relic, consulte [Excluir uma organização](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-unused-org).

Se desejar remover um registro de usuário, utilize a [interface de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks).
