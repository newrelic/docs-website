Além dos principais fatores de cobrança de usuário ingerido e faturável, você também pode usar complementos faturáveis opcionais para aprimorar sua experiência com a New Relic:

* <DoNotTranslate>**Compute Add On**</DoNotTranslate>

  : este complemento se aplica ao faturamento baseado no uso de consultas executadas em testes de segurança de aplicativos interativos, Live Archives e outros complementos disponibilizados. Ele mede seu uso com base na [unidade de capacidade de computação (CCUs)](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions#compute-capacity-unit).

* <DoNotTranslate>**EU Data Center for New Relic - Data, and New Relic - Data Plus**</DoNotTranslate>

  : este complemento se aplica quando você seleciona a União Europeia como sua região de dados, conforme disponível.

* <DoNotTranslate>**Extended Retention for New Relic - Data, and New Relic - Data Plus**</DoNotTranslate>

  : este complemento se aplica se você exceder o período padrão de retenção de seus dados. Isso se aplica a todos os seus dados - não apenas ao log - e é uma boa opção se você precisar fazer muitas pequenas consultas ou consultar grandes volumes de dados.

* <DoNotTranslate>**Live Archives**</DoNotTranslate>

  : estenda a duração do armazenamento de log em pelo menos 90 dias a mais do que a retenção padrão. Este complemento é melhor se você precisar manter registros que você consulta com pouca frequência. Live Archives também requer complemento de computador.

* <DoNotTranslate>**New Relic Synthetic Checks**</DoNotTranslate>

  : Este complemento se aplica caso seus verificações excedam o número padrão de verificações do monitoramento sintético.

* <DoNotTranslate>**Vulnerability Management**</DoNotTranslate>

  : esse recurso está incluído no Data Plus, mas o complemento se aplica se você usar o recurso Gerenciamento de vulnerabilidades sem o Data Plus.

Para começar a usar qualquer um desses complementos opcionais, entre em contato com a equipe da sua conta New Relic.
