Cada conta de nuvem vinculada ao monitoramento de infraestrutura New Relic (por exemplo, [AWS](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) ou [Azure](/docs/integrations/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations)) tem um <DNT>**account status dashboard**</DNT> que mostra como nossa plataforma está lidando com dados desse provedor de serviço de nuvem.

## Por que isso importa [#reasons-to-use]

Os motivos para monitor este dashboard incluem:

* <DNT>**Understand API call usage.**</DNT> Utilizamos API em nuvem para coletar métricas e dados de inventário. O painel mostra contagens e frequências para essas chamadas de API. Você pode querer monitor -los por dois motivos: a chamada de API pode custar dinheiro e o uso API pode ser limitado em determinados níveis de uso. Para diminuir o uso, você pode [definir configurações de integração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) para reduzir o número de componentes monitor ou a frequência com que eles são monitorados.

* <DNT>**Troubleshoot missing data or other data issues.**</DNT> O dashboard exibe informações que afetam a forma como a integração reporta os dados, incluindo:

  * Erros que afetam a coleta de dados do New Relic, como erros de permissão ou erros de esgotamento de cota.
  * Alterações na configuração de integração, como alterações no intervalo de sondagem ou coleta de tagsendo desativada ou ativada.

## Entenda os dados dashboard [#understand-data]

Os dados específicos exibidos no dashboard de status da conta serão diferentes de acordo com o provedor de serviço de nuvem. Os gráficos comuns incluem:

* <DNT>**Data updates**</DNT>: Mostra atualizações de dados métricos ou dados de inventário (atualizações mostradas como um valor `1` ).
* <DNT>**Account changes**</DNT>: Ações que afetam o funcionamento da integração. Por exemplo: renomear ou desvincular uma conta na nuvem, alterar intervalo de sondagem e outras opções de configuração.
* <DNT>**Data freshness**</DNT>: timestamp do último dado coletado para cada integração.
* <DNT>**Fetching errors**</DNT>: indicam problemas na coleta de dados. (Isso pode ser um problema do lado do provedor de nuvem.)

As opções para melhor compreensão dos dados do gráfico incluem:

* Passe o mouse sobre um gráfico <Icon name="fe-help-circle" /> ícone para ver uma descrição do gráfico (se disponível).
* [Veja a consulta NRQL subjacente do gráfico](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform).

## Encontre dashboard de status da conta [#find-dashboard]

Para encontrar o dashboard de status da conta de um provedor de serviço em nuvem:

1. Em <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure**</DNT> e selecione um provedor de serviço de nuvem (por exemplo, <DNT>**AWS**</DNT>).
2. Selecione <DNT>**Account status dashboard**</DNT> para a conta na nuvem.
