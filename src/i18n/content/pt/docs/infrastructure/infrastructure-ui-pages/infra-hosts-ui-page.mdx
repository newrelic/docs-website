## Um exemplo de caso de uso [#use-case]

A página <DoNotTranslate>**Hosts**</DoNotTranslate> é o local ideal para investigar problemas em sua infraestrutura. Aqui está um exemplo de fluxo de trabalho comum que nossos clientes usam para diagnosticar um problema:

1. Na [página de resumo do APM](/docs/apm/agents/manage-apm-agents/agent-data/triage-run-diagnostics), você percebe um problema com seu serviço Kafka. taxas de transferência caiu e muitos erros estão presentes.

2. Com base nos dados de infraestrutura que você vê no APM, você acha que é um problema de host, então clica na página de infraestrutura

   <DoNotTranslate>**Hosts**</DoNotTranslate>

   .

3. A página

   <DoNotTranslate>**Hosts**</DoNotTranslate>

   filtra automaticamente para seu serviço Kafka. Você vê que metade dos hosts que executam esse serviço atingiram o limite máximo de sua% de CPU.

4. Se você suspeitar que uma alteração na configuração do host pode ser a origem do seu problema, acesse a guia

   <DoNotTranslate>**Inventory**</DoNotTranslate>

   e verifique as configurações do seu sistema por host.

Existem alguns recursos de interface importantes disponíveis na interface <DoNotTranslate>**Hosts**</DoNotTranslate> . Entender como usá-los ajudará você a aproveitar ao máximo essa interface.

* <DoNotTranslate>**A.**</DoNotTranslate>

  A barra de filtro: Filtre os hosts que você deseja ver. Para dicas, consulte [Barra de filtros](/docs/new-relic-solutions/new-relic-one/core-concepts/search-filter-entities).

* <DoNotTranslate>**B.**</DoNotTranslate>

  Visualizações salvas: ao criar uma visualização desejada (que pode incluir filtros e seleções personalizadas de gráficos e tabelas), clique em

  <DoNotTranslate>**Save**</DoNotTranslate>

  .

* <DoNotTranslate>**C.**</DoNotTranslate>

  Opções de gráfico: você pode alternar para métricas diferentes para cada gráfico e clicar em `...` para ver [mais opções de gráfico](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/#data-analysis), incluindo obter um link compartilhável e criar um alerta para essa métrica.

* <DoNotTranslate>**D.**</DoNotTranslate>

  Agrupar por: Use para agrupar os resultados do gráfico por atributo específico. Por exemplo, você pode exibir as regiões da AWS com maior uso de CPU agrupadas por `awsRegion`.

* <DoNotTranslate>**E.**</DoNotTranslate>

  Filtro de atributo: permite filtrar seus hosts para aqueles com um valor [de atributo](/docs/new-relic-solutions/get-started/glossary/#attribute) específico. Por exemplo,

* <DoNotTranslate>**F.**</DoNotTranslate>

  Tabela de hosts: Por padrão, 25 hosts são selecionados para exibição. Use as colunas da tabela e o filtro atributo para explorar os hosts e selecionar diferentes hosts para exibição. Você pode exportar a tabela como CSV.

* <DoNotTranslate>**G.**</DoNotTranslate>

  Detalhes do host: para um host específico na tabela de hosts, clique no botão

  <Icon name="fe-more-horizontal"/>

    Aqui estão os gráficos padrão mostrados na página <DoNotTranslate>**System**</DoNotTranslate> :

    * <DoNotTranslate>**CPU %**</DoNotTranslate>

      : esta é uma métrica derivada de diversas outras métricas no evento `SystemSample` . Especificamente, o atributo [`cpuPercent`](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#systemsample-cpu-percent) é uma agregação de `cpuUserPercent`, `cpuSystemPercent`, `cpuIoWaitPercent` e `cpuStealPercent`.

    * <DoNotTranslate>**Load average five minute**</DoNotTranslate>

      : representa uma média do número de tarefas aguardando para trabalhar em um sistema nos últimos 5 minutos.

    * <DoNotTranslate>**Memory free %**</DoNotTranslate>

      : compara a quantidade de bytes de memória livres com a quantidade de bytes de memória usados.

    Para obter detalhes sobre os dados usados para gerar esses gráficos, consulte [SystemSample](attribute-dictionary/?event=SystemSample).
  </Collapser>

Os gráficos padrão mostram métricas de largura de banda por pacote, largura de banda por bytes e erros por segundo.

    Detalhes sobre interfaces individuais podem ajudá-lo:

    * Examine os níveis de saturação em sua rede ou em hosts selecionados.
    * Compare equilíbrios de carga entre recursos específicos.
    * Identifique diferenças inesperadas nos padrões de capacidade entre hosts semelhantes.
    * Avalie os cinco principais erros de rede detectados.

    Esses dados de rede em tempo real podem ajudá-lo a determinar se é necessário resolver erros reconfigurando sua rede, reequilibrando as cargas ou tomando outras ações de manutenção preventiva antes de precisar tomar uma decisão de compra.

    Para obter detalhes sobre os dados usados para gerar esses gráficos, consulte [NetworkSample](/attribute-dictionary/?event=NetworkSample).
  </Collapser>

  <Collapser
    id="processes"
    title="Interface de processos"
  >
    <Callout variant="important">
      Por padrão, o agente de infraestrutura não envia [dados sobre os processos do sistema operacional](/attribute-dictionary/?event=ProcessSample) , a menos que você use a instalação guiada. Para habilitar o envio do processo conjunto de dados [`enable_process_metrics`](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/) para `true`. Para ajustar quais processos você deseja monitor, configure [`include_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#include-matching-metrics).
    </Callout>

    Use a guia <DoNotTranslate>**Processes**</DoNotTranslate> para obter informações sobre processos em execução em seus hosts e para definir alertas sobre métricas de processos. A guia <DoNotTranslate>**Processes**</DoNotTranslate> mostra dados como porcentagem de CPU, bytes de E/S e uso de memória para processos individuais ou agrupamentos de processos.

    <img
      title="Hosts Processes page"
      alt="Hosts Processes page"
      src={infrastructureHostNewProcesses}
    />

    <figcaption>
      <DoNotTranslate>
        **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Hosts > Processes**
      </DoNotTranslate>
    </figcaption>

    Na guia <DoNotTranslate>**Processes**</DoNotTranslate> , o percentual de CPU tem como escopo processos individuais, e não hosts. Por causa disso, a métrica percentual de CPU não leva em consideração os recursos de todo o sistema. Em vez disso, mostra quanto de um único núcleo de CPU cada processo está ocupando.

    Para obter detalhes sobre os dados usados para gerar esses gráficos, consulte [ProcessSample](/attribute-dictionary/?event=ProcessSample).

    Aqui está um exemplo de como identificar um problema e definir um alerta:

    1. Você percebe na página

       <DoNotTranslate>**Hosts**</DoNotTranslate>

       que um cluster tem uma porcentagem de CPU muito maior que o normal.

    2. Você acessa a página

       <DoNotTranslate>**Processes**</DoNotTranslate>

       e filtra os processos desse cluster.

    3. Você percebe que várias instâncias do mesmo processo apresentam porcentagem excessiva de CPU.

    4. Depois de corrigir o problema, você decide criar uma condição do alerta que é acionada quando a porcentagem de CPU para esse tipo de processo excede um determinado limite.

    <Callout variant="tip">
      Você não pode iniciar, parar ou reiniciar processos a partir da interface. Em vez disso, siga os procedimentos padrão para [iniciar, parar, verificar ou reiniciar](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status) o agente de infraestrutura manualmente.
    </Callout>

    A guia <DoNotTranslate>**Storage**</DoNotTranslate> permite monitor a capacidade e a eficiência dos seus recursos, incluindo a utilização geral dos seus dispositivos, o uso do disco ou as operações de E/S. Isso pode ajudá-lo a:

    * Examine padrões inesperados; por exemplo, um cluster de máquinas espelhadas que não processam tarefas de E/S de maneira uniforme.
    * Monitor os níveis de uso antes que o espaço em disco se esgote completamente.
    * Defina condição do alerta para notificá-lo sobre problemas com um ou mais hosts; por exemplo, longos tempos de processamento para operações de leitura ou gravação, disponibilidade ou utilização do disco com base em porcentagem ou bytes, etc.
    * Tome decisões informadas sobre onde redistribuir hosts com solicitações de processamento superiores ou inferiores ao normal.
    * Use dados para ajudar a planejar acréscimos ou alterações em seu orçamento de infraestrutura antes que surja uma emergência.
