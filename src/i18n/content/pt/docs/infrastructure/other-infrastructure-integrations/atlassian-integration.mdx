New Relic oferece uma integração com o Atlassian Jira que sincroniza seu problema New Relic e informações de incidentes com o Jira. Esses dados aparecem no Jira no [recurso de incidente](https://support.atlassian.com/jira-software-cloud/docs/what-is-the-incidents-feature/) e permitem conectá-los aos problemas do Jira para melhorar o fluxo de trabalho da interrupção. As seções abordam como configurar essa integração e as práticas recomendadas para usá-la.

## Compartilhamento de dados [#data-sharing]

Ao instalar o aplicativo New Relic para Jira do Atlassian Marketplace, você habilita a integração para executar estas ações:

* Leia entidade, incidente e problemas do Jira.
* Escreva entidade, incidente e problemas para Jira.
* Exclua entidade, incidente e problemas do New Relic e do Jira para espelhar o relatório de dados com cada entidade.

<Callout variant="caution">
  Essa integração exporá seus problemas New Relic e informações de incidentes a todos os usuários no espaço de trabalho Jira relacionado. O usuário Jira neste espaço de trabalho pode não ter acesso ao New Relic e, mesmo que tenha, pode não ter acesso às mesmas contas New Relic que seu usuário chave de API.

  Se você tiver informações de conta da New Relic para ocultar, use um [usuário de integração](/docs/more-integrations/best-practices-integration/#integration-user-leverage) que não tenha acesso às suas contas confidenciais.
</Callout>

## instalação [#installation]

Siga estas etapas para integrar o New Relic ao Jira.

<Steps>
  <Step>
    ## Instale o aplicativo New Relic para Jira [#install-nr-jira-app]

    Você precisa instalar o aplicativo New Relic para Jira para sincronizar seus dados entre o New Relic e o Atlassian. Lembre-se de que você deve ter permissão de administrador do site Jira para instalar o aplicativo Marketplace. Acesse [Atlassian Marketplace](https://marketplace.atlassian.com/apps/1233828/new-relic-for-jira?tab=overview&hosting=cloud) e clique no botão **Install**.

    <Callout variant="tip">
      Se você não tiver permissão de administrador do site, poderá solicitá-la ao seu administrador. Recomendamos que você entre em contato com o administrador do site Jira para concluir o processo de instalação do aplicativo.
    </Callout>
  </Step>

  <Step>
    ## Crie uma chave de API New Relic [#create-new-relic-key]

    Depois de instalar o aplicativo New Relic for Jira, você precisará criar uma [chave de API New Relic](/docs/apis/intro-apis/new-relic-api-keys/). O administrador do site Jira e o administrador do New Relic decidirão sobre as contas expostas ao Jira. O usuário que tiver o acesso correto à conta New Relic poderá gerar uma chave de API New Relic. A chave de API New Relic tem acesso às mesmas contas do usuário que as gerou.

    Crie e gerencie sua chave de API na página [New Relic de interface de chave de API](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) ou na página de log [API token](https://id.atlassian.com/manage-profile/security/api-tokens) da Atlassian e clique em **Create API token**.
  </Step>

  <Step>
    ## Aplicar uma chave de API New Relic [#apply-new-relic-key]

    Depois de criar uma chave de API, você poderá aplicá-la à primeira página do aplicativo de conexão New Relic.

    Cole seu token da API New Relic no aplicativo New Relic Atlassian Marketplace e clique em **Connect**.

    <img width="50%;" title="Apply a New Relic API key" alt="Apply a New Relic API key" src="/images/opentelemetry_screenshot-crop_connect-nr-jira.webp" />
  </Step>

  <Step>
    ## Selecione seus serviços New Relic [#select-nr-services]

    Depois que o aplicativo New Relic for Jira estiver conectado por meio de uma chave de API, você verá uma lista de todas as entidades associadas às contas nessa chave de API. Selecione o recurso incidente na navegação, depois conecte qualquer entidade que deseja expor no Jira.

    Aprenda como [conectar e remover serviços no Jira](https://support.atlassian.com/jira-software-cloud/docs/connect-jira-service-management-services-to-jira-software/).

    <img title="Jira incident management page" alt="Jira incident management page" src="/images/opentelemetry_screenshot-full_jira-incident-mgt.webp" />
  </Step>

  <Step>
    ## Veja os problemas e incidentes New Relic no Atlassian Jira [#see-issues-incidents]

    Depois de selecionar sua entidade New Relic , o aplicativo começa a sincronizar os dados do New Relic com o Atlassian Jira. Abra a guia incidente para ver uma lista de incidentes detectados.
  </Step>
</Steps>

## Habilitar recurso incidente [#enable-incidents-feature]

Se você tiver permissões de administrador do projeto, siga estas etapas para ativar o recurso de incidente:

1. Na barra lateral do seu projeto, vá para <DNT>**Project settings &gt; Features**</DNT>.

2. Ative ou desative o recurso <DNT>**Incidents**</DNT>.

3. Depois de ativado, selecione <DNT>**Incidents**</DNT> na barra lateral do seu projeto para usar o recurso.

Consulte a página [do recurso incidente](https://support.atlassian.com/jira-software-cloud/docs/what-is-the-incidents-feature/) na documentação do Suporte da Atlassian para obter mais informações.

## Remover uma conexão entre New Relic e Jira [#remove-connection]

Você pode interromper a sincronização de dados do New Relic e remover os dados sincronizados a qualquer momento. Se você tiver permissões de administrador do site JIRA, poderá fazer isso na página de configuração do New Relic no JIRA. Você pode selecionar sua organização conectada e clicar em <DNT>**Delete**</DNT>. Assim que o processo for concluído, você não verá mais o incidente New Relic e emitirá informações no incidente recurso no Jira.

## Conecte-se a uma organização diferente [#different-organization]

Para se conectar a uma organização diferente ou alterar o acesso à conta fornecido pela chave de API, acesse a página de configuração do New Relic no Jira.

Ao selecionar <DNT>**Connect another New Relic organization**</DNT>, você precisará fornecer uma chave de API New Relic. Adicionar uma nova chave excluirá sua conexão atual, incluindo todos os dados sincronizados com o Atlassian. Em seguida, iniciará um novo processo de sincronização entre o acesso da conta e da organização da nova chave de API.
