Se sua equipe implanta software e serviços na nuvem através do Azure, nosso [Azure Native New Relic Service](https://azuremarketplace.microsoft.com/en-us/marketplace/apps/newrelicinc1635200720692.newrelic_liftr_payg) permite que você gerencie a integração Azure-New Relic e gerencie e instale nosso agente de infraestrutura e nosso agente .NET e Java <InlinePopover type="apm"/>, tudo diretamente do o Portal do Azure.

O serviço Azure Native New Relic permite:

* Integre e use facilmente o New Relic como um serviço nativamente integrado criado no Azure.
* Obtenha uma fatura única para todos os recursos que você consome no Azure, incluindo o New Relic.
* Monitor automaticamente a atividade de assinatura do Azure e o log de recursos do New Relic.
* Monitor automaticamente as métricas do Azure usando New Relic.
* Use uma experiência única para instalar e desinstalar o agente New Relic na máquina virtual e nos serviços de aplicativos.

Além disso, você pode usar New Relic para outros monitoramentos não relacionados ao Azure que possa precisar, usando qualquer uma de [nossas mais de 500 integrações](https://newrelic.com/instant-observability).

<Callout variant="important">
  Este serviço está disponível apenas para clientes que optam por que o Azure hospede seus dados do New Relic.
</Callout>

Para obter documentos detalhados, consulte o [site de documentos do Microsoft Azure](https://learn.microsoft.com/en-us/azure/partner-solutions/new-relic/new-relic-overview). Aqui está um breve resumo de como configurar este serviço:

1. No Azure Marketplace, assine o serviço Azure Native New Relic.

2. No Portal do Azure, crie um recurso New Relic.

3. Selecione

   <DNT>
     **Create a New Relic resource**
   </DNT>

   para criar uma nova conta New Relic ou você pode

   <DNT>
     **Link an existing New Relic resource**
   </DNT>

   para vincular a um recurso existente.

4. Preencha o prompt exibido e selecione

   <DNT>
     **Review + create**
   </DNT>

   .

5. Configure como seu serviço reporta métricas e faça login na aba

   <DNT>
     **Resource configuration**
   </DNT>

   .

6. Vá para

   <DNT>
     **App services**
   </DNT>

   para ativar nosso agente

   <InlinePopover type="apm"/>

   . Vá para

   <DNT>
     **Virtual machines**
   </DNT>

   para ativar nosso agente de infraestrutura.

7. Verifique se o New Relic está recebendo dados na guia

   <DNT>
     **Monitored resources**
   </DNT>

   do Portal do Azure. Os recursos com uma marca de seleção verde estão enviando dados para o New Relic.

Para obter detalhes sobre a configuração, consulte o [site de documentos do Microsoft Azure](https://learn.microsoft.com/en-us/azure/partner-solutions/new-relic/new-relic-overview)

## Log na New Relic [#login-newrelic]

Depois de criar seu recurso New Relic e verificar se o Azure está enviando dados para ele, você poderá entrar no New Relic.

Para log no New Relic:

1. Na página

   <DNT>
     **Overview**
   </DNT>

   do Azure para o novo recurso, clique em

   <DNT>
     **Login to New Relic account**
   </DNT>

   .

2. Você precisará criar uma nova senha. Para fazer isso, clique em

   <DNT>
     **Forgot your password**
   </DNT>

   .

3. Insira o seu endereço de email. Deve ser o mesmo que você usa para sua conta do Azure.

## Nota sobre data center [#datacenters]

O Azure Native New Relic Service armazena seus dados diretamente no data center Azure da Microsoft. Atualmente, o serviço Azure Native armazena todos os dados na região Leste dos EUA do Azure.

O serviço Padrões de log da New Relic opera exclusivamente na Amazon Web Services. Ao usar os padrões do New Relic Logs, você concorda que New Relic pode mover e processar seus dados na Amazon Web Services. Isso se aplica quer você armazene seus dados no data center Azure da New Relic ou em nosso data center Amazon Web Services.
