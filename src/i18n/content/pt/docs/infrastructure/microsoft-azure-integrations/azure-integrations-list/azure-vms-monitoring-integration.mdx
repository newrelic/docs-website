[O monitoramento de infraestrutura New Relic ](/docs/infrastructure)fornece uma integração para [a Microsoft máquina virtual do Azure (VM)](https://docs.microsoft.com/en-us/azure/virtual-machines/) que reporta dados do seu serviço Azure VM para a New Relic. Este documento explica como ativar esta integração e descreve os dados que podem ser capturados.

## Recurso

A integração da New Relic para máquina virtual do Azure reporta dados sobre o seu serviço de VM, como o ID VM , o tamanho VM , o conjunto de disponibilidade e o nome da região. Você pode monitor e alertar sobre seus dados de VM do Azure a partir [New Relic](/docs/infrastructure) e pode [criar consultas personalizadas e painéis de gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Requisitos

Os requisitos incluem:

* [Agente New Relic Infrastructure instalado](/docs/infrastructure/new-relic-infrastructure/installation) com [agente de infraestrutura versão 1.0.775](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) ou superior. ([Atualize o agente de infraestrutura](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent).)
* [Integração New Relic Azure ativada](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations)

## Ativar integração [#requirements]

Para habilitar essa integração siga os procedimentos padrão para [ativar seu serviço Azure no New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

<Callout variant="important">
  Você deve instalar o agente de infraestrutura em cada VM para ver as métricas desse host. Conectar sua assinatura do Azure permite que a New Relic acesse os metadados da VM.
</Callout>

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informações [de polling](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) padrão para a integração da máquina virtual:

* Intervalo de sondagem: 5 minutos
* Resolução: 1 ponto de dados por minuto

## Encontre e use dados [#find-and-use]

Para encontrar seus dados de integração, acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**</DNT> e procure a integração.

Você pode [consultar e explorar seus dados](/docs/using-new-relic/data/understand-data/query-new-relic-data) usando o [tipo de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureVirtualMachineSample` . O valor `provider` é `AzureVirtualMachine`.

Para obter mais informações sobre como localizar e usar dados de integração, consulte [Compreender e usar dados](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Outros dados do sistema [#other-attributes]

A máquina virtual do Azure integração também coleta o seguinte [atributo](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) sobre o serviço e sua configuração:

* Região
* Zona de disponibilidade
* Tipo de instância
* ID da instância

## Resolução de problemas

Se você usar <DNT>**Host not reporting**</DNT> condição do alerta, a importação de metadados do Azure para sua VM alterará o registro de hosts e resultará em notificação de alerta falsa. Para evitar falso positivo:

1. Para cada instância do Azure, [desative](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#maintain-alerts) sua

   <DNT>
     **Host not reporting**
   </DNT>

   condição de alerta.

2. [Atualizar o agente de infraestrutura](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent) para todas as instâncias que utilizaram esta condição de alerta.

3. [Reative](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#maintain-alerts) cada

   <DNT>
     **Host not reporting**
   </DNT>

   condição do alerta.
