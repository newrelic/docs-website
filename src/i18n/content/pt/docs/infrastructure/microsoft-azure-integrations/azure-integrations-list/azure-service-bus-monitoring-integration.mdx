[O monitoramento de infraestrutura New Relic ](/docs/infrastructure)fornece uma integração para [Microsoft Azure Service Bus](https://docs.microsoft.com/en-us/azure/service-bus-messaging/service-bus-messaging-overview) que reporta dados do seu serviço Azure Service Bus para a New Relic. Este documento explica como ativar esta integração e descreve os dados que podem ser capturados.

## Recurso

A integração da New Relic para Azure Service Bus reporta dados métricos sobre o seu serviço Service Bus, como o número de mensagens na fila, o número de mensagens mortas e o tamanho de bytes da fila. Também coleta dados sobre o status e a configuração do serviço.

Você pode monitor e alertar sobre os dados do Azure Service Bus a partir do [New Relic](/docs/infrastructure) e pode [criar consultas personalizadas e painéis de gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Ativar integração [#requirements]

Para habilitar essa integração siga os procedimentos padrão para [ativar seu serviço Azure no New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informações [de sondagem](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) padrão para a integração do Service Bus:

* Intervalo de sondagem: 5 minutos
* Resolução: 1 ponto de dados por minuto

## Encontre e use dados [#find-and-use]

Para encontrar seus dados de integração, acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Azure**</DNT> e procure a integração.

Você pode [consultar e explorar seus dados](/docs/using-new-relic/data/understand-data/query-new-relic-data) usando estes [tipos de eventos](/docs/data-apis/understand-data/new-relic-data-types/#event-data):

* Dados de namespace: [`AzureServiceBusNamespaceSample`](#namespace)
* Dados da fila: [`AzureServiceBusQueueSample`](#queue)
* Dados de assinatura: [`AzureServiceBusSubscriptionSample`](#subscription)
* Dados do tópico: [`AzureServiceBusTopicSample`](#topic)

Para obter mais informações sobre como localizar e usar dados de integração, consulte [Compreender e usar dados](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Dados métricos [#metrics]

Existem vários conjuntos de métricas reportados pela integração do Azure Service Bus. Para obter mais informações sobre como os dados são estruturados e relatados ao New Relic, consulte [Compreender e usar dados de integração](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric).
