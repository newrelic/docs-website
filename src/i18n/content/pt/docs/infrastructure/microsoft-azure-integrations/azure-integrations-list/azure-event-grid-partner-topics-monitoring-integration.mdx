[A integração da New Relic](/docs/infrastructure/introduction-infra-monitoring) inclui uma integração para reportar sua métrica [de eventos Grid Partner TopicsMicrosoft Azure ](https://azure.microsoft.com/en-us/products/event-grid/)e outros dados para a New Relic. Este documento explica como ativar a integração e descreve os dados relatados.

## Recurso

Um tópico de parceiro no evento Grid representa um ou mais eventos publicados pelos serviços do Azure, como o Azure Storage e o Azure Event Hubs. Por exemplo, um tópico de parceiro pode representar todos os eventos de blob ou apenas eventos de blob criados e blob excluídos publicados para uma conta de armazenamento específica.

Usando o New Relic, você pode:

* Visualize os dados do Azure Event Grid Partner Topics no painel pré-criado.
* Execute [uma consulta personalizada e visualize os dados](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).
* Crie condição do alerta para notificá-lo sobre alterações nos dados.

## Ativar integração [#activate]

Siga [o procedimento padrão de integração Azure Monitor ](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor)para ativar seu serviço Azure no monitoramento de infraestrutura New Relic .

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#settings).

New Relic consulta seus serviços do Azure Event Grid Partner Topics por meio da integração Azure Monitor de acordo com um intervalo [de pesquisa](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#settings) padrão.

## Encontre e use dados [#find-and-use]

Para [explorar seus dados de integração](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), acesse <DNT>**[one.newrelic.com/infra](https://one.newrelic.com/infra) &gt; Azure &gt; (select an integration)**</DNT>.
