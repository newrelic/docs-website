Nosso [monitoramento de infraestrutura](/docs/infrastructure) fornece uma integração para [o Microsoft Azure Functions](https://azure.microsoft.com/en-us/services/functions/) que reporta dados do seu serviço Azure Functions para a New Relic. Este documento explica como ativar esta integração e descreve os dados que podem ser capturados.

## Recurso

A nossa integração Azure Functions reporta dados métricos sobre o seu serviço Functions, como o número de execuções de funções, bytes enviados e recebidos e contagens de erros HTTP. Também coleta dados sobre o status e a configuração do serviço.

Você pode monitor e alertar sobre seus dados Azure Functions com nosso [monitoramento de infraestrutura](/docs/infrastructure), além de [criar consultas personalizadas e dashboards gráficos](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Ativar integração [#requirements]

Para permitir esta integração siga os procedimentos padrão para [ativar o seu serviço Azure para monitoramento de infraestrutura](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informações [de sondagem](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) padrão para a integração do Azure Functions:

* Intervalo de sondagem: 5 minutos
* Resolução: 1 ponto de dados por minuto

## Encontre e use dados [#find-and-use]

Para encontrar seus dados de integração, vá para <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, na navegação superior clique em <DNT>**Infrastructure**</DNT>, depois clique em <DNT>**Azure**</DNT> e procure a integração.

Você pode [consultar e explorar seus dados](/docs/using-new-relic/data/understand-data/query-new-relic-data) usando o [tipo de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureFunctionsAppSample` .

Para obter mais informações sobre como localizar e usar dados de integração, consulte [Compreender e usar dados](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Dados métricos [#metrics]

Para visualizar as métricas reportadas pela integração do Azure Functions , consulte o tipo de evento `AzureFunctionsAppSample`. Use os metadados associados a cada métrica para [filtrar e facetar](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) os dados que estão sendo relatados. Para obter informações métricas detalhadas, consulte a documentação [métrica suportada pelo Azure](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-functions) .

## Outros dados do sistema [#other-attributes]

A integração do Azure Functions também coleta estes [atributos](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) sobre o serviço e sua configuração:

* `availabilityState`
* `defaultHostName`
* `enabled`
* `isPremiumApp`
* `javaVersion`
* `netFrameworkVersion`
* `nodeVersion`
* `operatingSystem`
* `phpVersion`
* `pythonVersion`
* `platformArchitecture`
* `state`
* `usageState`
