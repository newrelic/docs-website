[O monitoramento de infraestrutura New Relic ](/docs/infrastructure)fornece uma integração para [Microsoft Azure App Service](https://azure.microsoft.com/en-us/services/app-service/) que reporta dados do Azure Apps Service para a plataforma New Relic . Este documento explica como ativar esta integração e descreve os dados que podem ser capturados.

## Recurso

Um Azure App Service é um serviço para hospedar e executar aplicativos web, API REST e back-end móvel em uma plataforma totalmente gerenciada. A integração da New Relic para o Azure App Service reporta dados métricos sobre as suas aplicações Web, incluindo:

* Número de solicitações recebidas
* Número de respostas e erros HTTP

Também coleta dados sobre o status e a configuração do serviço. Usando o New Relic, você pode:

* Visualize os dados do Azure Storage em um painel de infraestrutura pré-construído.
* Execute uma consulta personalizada e visualize os dados na [interface New Relic ](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).
* Crie condição do alerta para notificá-lo sobre alterações nos dados de armazenamento.

## Ativar integração [#activate]

Para habilitar a integração siga os procedimentos padrão para [ativar seu serviço Azure no New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

A New Relic consulta seus Serviços de Aplicativos do Azure de acordo com um intervalo [de sondagem](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) padrão, que varia dependendo da integração. Frequência de sondagem para a integração dos Serviços de Aplicações Azure:

* Intervalo de sondagem: 5 minutos
* Resolução: 1 ponto de dados por minuto

## Encontre e use dados [#find-and-use]

Para [explorar seus dados de integração](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Azure**</DNT>.

Os [tipos de eventos](/docs/data-apis/understand-data/new-relic-data-types/#event-data) são:

* `AzureAppServiceWebAppSample`: representa cada um dos aplicativos Web do serviço de aplicativo do Azure na conta.
* `AzureAppServiceHostNameSample`: representa cada um dos nomes de domínio vinculados a um aplicativo Web que pode ser usado para acessá-lo.
* `AzureAppServicePlan:` Representa um conjunto de recursos computacionais para execução de um aplicativo Web.

## Dados métricos [#metrics]

Para visualizar as métricas reportadas pela integração do Azure App Service, consulte os tipos de evento `AzureAppServiceWebAppSample` ou `AzureAppServicePlanSample` . Use os metadados associados a cada métrica para [filtrar e facetar](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) os dados que estão sendo relatados. Para obter informações métricas detalhadas, consulte a documentação [métrica suportada pelo Azure](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-excluding-functions) .
