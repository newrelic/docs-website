## Problema

Você configurou sua AWS integração e [conectou-a ao New Relic](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure), mas não consegue ver nenhum dado no [AWS dashboard](/docs/integrations/amazon-integrations/get-started/introduction-aws-integrations#insights).

## Solução

Certifique-se de ter [nos concedido as permissões corretas](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies) para recuperar sua métrica.

Para verificar isso, acesse <DNT>**Account status dashboard**</DNT> da sua integração para verificar se nenhum erro está sendo relatado:

<img
  title="AWS account status dashboard"
  alt="Screenshot showing the account status dashboard."
  src="/images/infrastructure_screenshot-crop_aws-account.webp"
/>

No dashboard de status da conta, verifique o dashboard <DNT>**Permission Errors**</DNT> e veja se há erros de método para alguma das suas fontes de dados:

<img
  title="Permission errors in the AWS account status dashboard"
  alt="Screenshot showing permissions errors in the account status dashboard."
  src="/images/infrastructure_screenshot-crop_aws-permission-errors.webp"
/>

Se erros forem relatados, certifique-se de que seu documento de política de permissões (se não for a política padrão AWS `Read-Only Access`) tenha os métodos relatados no dashboard de erros de acordo com nossa documentação [de permissões personalizadas](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies) .

<Callout variant="important">
  Alguns métodos não são específicos de integração e <DNT>**[must be granted for all integrations](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies#all-permissions)**</DNT>.
</Callout>
