[A integração New Relic Infrastructure ](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)inclui uma integração AWS [Elastic Beanstalk](https://aws.amazon.com/elasticbeanstalk) para relatar seus dados do Beanstalk para New Relic. Este documento explica o recurso de integração, como ativá-lo e quais dados podem ser reportados.

AWS Elastic Beanstalk é um serviço dinâmico que permite fácil implantação e escalabilidade para seu aplicativo. O “aplicativo” no AWS Beanstalk é o ambiente. Um "aplicativo" pode ter vários ambientes associados.

## Requisitos [#requirements]

AWS Elastic Beanstalk enviará algumas métricas básicas para o Amazon CloudWatch por padrão. Adicionalmente, você pode habilitar relatórios de saúde aprimorados como métrica personalizada; ativar essas métricas pode adicionar cobranças adicionais à definição de preço da sua conta do Amazon CloudWatch. Para habilitar métricas aprimoradas do CloudWatch, consulte [Publicação de métricas personalizadas do Amazon CloudWatch para um ambiente](http://docs.aws.amazon.com/elasticbeanstalk/latest/dg/health-enhanced-cloudwatch.html).

## Ativar integração [#activate]

Por padrão, nenhuma métrica é selecionada nas configurações do Elastic Beanstalk. Você deve selecioná-los para o ambiente <DNT>**each**</DNT> para gerar métricas na interface New Relic . Se nenhuma métrica for selecionada, nenhum dado será reportado à New Relic.

Para ativar esta integração:

1. Siga as instruções em [Conectar serviços da AWS ao New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

2. Atualize as configurações do Elastic Beanstalk para selecionar qual métrica (o padrão é nenhuma métrica) observar para o ambiente

   <DNT>
     **each**
   </DNT>

   : Na interface AWS Elastic Beanstalk, vá para

   <DNT>
     **Configuration > Health**
   </DNT>

   e configure a métrica personalizada do CloudWatch para relatórios de integridade.

Depois de selecionar a métrica que deseja relatar para cada ambiente, o Elastic Beanstalk gerará dados a partir do momento em que os objetos de dados `instance` e `environment` forem criados.

## Configuração e polling [#polling]

Você pode alterar a [frequência de pesquisa](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informações de pesquisa padrão para a integração do AWS Elastic Beanstalk:

* Intervalo de sondagem New Relic : 5 minutos
* Intervalo de dados do Amazon CloudWatch: 1 minuto

## Visualizar e usar dados [#find-data]

Para [visualizar e usar seus dados de integração](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS**</DNT> e selecione um dos links de integração do AWS Elastic Beanstalk.
