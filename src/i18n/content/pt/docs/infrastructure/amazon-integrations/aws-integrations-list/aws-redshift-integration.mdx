[A integração New Relic Infrastructure ](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)inclui uma integração para relatar seus dados do Amazon Redshift para New Relic. Este documento explica como ativar a integração e descreve os dados reportados.

## Recurso [#features]

[O Amazon Redshift](https://aws.amazon.com/redshift) é um data warehouse totalmente gerenciado que torna simples e econômica a análise de todos os seus dados usando SQL padrão e as ferramentas existentes do Amazon Business Intelligence (BI). A integração Redshift da New Relic reúne dados métricos e de configuração para o banco de dados relacional associado à sua conta Amazon Redshift.

Os dados reportados incluem bytes recebidos e bytes transmitidos, status de saúde, conexões de banco de dados, latência e outras [métricas](#metrics)). Seus dados do Redshift estão disponíveis em <InlinePopover type="dashboards"/>pré-construído e você também pode criar [consultas e gráficos personalizados](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).

## Ativar integração [#activate]

Para habilitar essa integração, siga os procedimentos padrão para [conectar os serviços da AWS ao New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

Para permitir que o New Relic acesse dados do Redshift, sua conta da AWS deve ter a política <DNT>**ReadOnly**</DNT> associada a ela ou as seguintes permissões devem ser concedidas à sua conta do New Relic:

* `describeClusters`
* `describeClusterParameters`

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informações [de pesquisa](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) padrão para a integração do Amazon RedShift:

* Intervalo de sondagem New Relic : 5 minutos
* Intervalo de dados do Amazon CloudWatch: 1 minuto

## Encontre e use dados [#find-data]

Para encontrar seus dados de integração, acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS**</DNT> e selecione um dos links de integração do Redshift.

Você pode [consultar e explorar seus dados](/docs/using-new-relic/data/understand-data/query-new-relic-data) usando o [tipo de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `DatastoreSample` , com um valor `provider` de `RedshiftNode` para nós do Redshift ou `RedshiftCluster` para cluster do Redshift.

Para saber mais sobre como usar seus dados, consulte [Compreender e usar dados de integração](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Dimensões métricas do Amazon Redshift [#dimensions]

Os dados do Amazon Redshift podem ser filtrados em qualquer uma das seguintes [dimensões](http://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/rs-metricscollected.html#rs-metric-dimensions):

<table>
  <thead>
    <tr>
      <th width={285}>
        Dimensão
      </th>

      <th>
        Descrição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `NodeID`
      </td>

      <td>
        Filtra dados solicitados específicos dos nós de um cluster. `NodeID` será `Leader`, `Shared` ou `Compute-N`, onde `N` é 0, 1 etc. para o número de nós no cluster.

        `Shared` significa que o cluster possui apenas um nó. O nó `Leader` e o nó `Compute` são combinados.
      </td>
    </tr>

    <tr>
      <td>
        `ClusterIdentifier`
      </td>

      <td>
        Filtra dados solicitados específicos do cluster. métricas específicas do cluster incluem `HealthStatus`, `MaintenanceMode` e `DatabaseConnections`.

        Em geral, as métricas para esta dimensão (por exemplo, `ReadIOPS`) que também são métricas de nós representam um agregado dos dados métricos do nó. Tome cuidado ao interpretar essas métricas, pois elas agregam o comportamento dos nós `Leader` e `Compute` .
      </td>
    </tr>
  </tbody>
</table>
