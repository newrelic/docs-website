Agora você pode usar o AWS [Application Migration Service](https://docs.aws.amazon.com/mgn/latest/ug/what-is-application-migration-service.html) para instalar o agente New Relic em servidores que estão migrando para AWS.

## Configurar [#setup]

Para que a AWS instale o [agente da New Relic Infrastructure](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/), você precisará criar um modelo de pós-lançamento por meio do console da AWS. Siga estes passos:

1. No [console da AWS](https://console.aws.amazon.com/mgn/home), abra <DNT>**Application Migration Services** </DNT>.
2. Abra o servidor de origem desejado na lista disponível.
3. Clique na aba **Post-launch settings** \[Configurações pós-lançamento].
4. Selecione **Edit &gt; Install New Relic agent** e adicione seu ID de conta e chave de licença.

## monitoramento seus dados [#monitor-data]

Assim que a configuração estiver concluída, seu aplicativo AWS será monitorado pelo agente de infraestrutura New Relic . Para encontrar seus dados no New Relic:

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com/) &gt; All entities**</DNT>.
2. Na seção de entidade **Hosts** , localize sua instância.
