New Relic integra-se com Amazon Web Services (AWS) para relatar seus dados [do Amazon Cognito](https://aws.amazon.com/vpc/lattice/) métrica e outros dados ao New Relic.

Este documento explica como ativar a integração e descreve os dados relatados.

## Recurso [#features]

O Amazon VPC Lattice é um serviço de rede de aplicativos totalmente gerenciado que você usa para conectar, proteger e monitor os serviços do seu aplicativo em várias contas e nuvens privadas virtuais (VPC).

New Relic suporta duas integração de monitoramento para Amazon VPC Lattice: [métricas do CloudWatch](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) e [log de acesso](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html).

Para métricas do CloudWatch, você só precisa configurar a integração uma vez e todos os namespaces AWS disponíveis, incluindo `AWS/VPCLattice`, serão incluídos. As métricas do CloudWatch for VPC Lattice acompanham o tráfego entre seus serviços Lattice e grupos de destino.

## Ativar integração [#activate]

Essa integração está disponível por meio do CloudWatch MetricStreams.

Para habilitar essa integração, veja como [conectar serviços da AWS ao New Relic via CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/).

## Ativar integração de log do Lattice [#activate-logs-integration]

<Callout variant="important">
  Para usar o Kinesis Firehose, você deve ter um fluxo de entrega do Kinesis Data Firehose criado. Caso contrário, você poderá criar um acessando o recurso Amazon Kinesis Firehose no [console da AWS](https://console.aws.amazon.com/firehose) e criando um fluxo de entrega com `Destination: New Relic`.

  [Saiba mais sobre como criar um fluxo de entrega do Amazon Kinesis Data Firehose.](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html?icmpid=docs_console_unmapped)
</Callout>

A integração do log de acesso para Amazon VPC Lattice usa [Amazon Kinesis Firehose](https://aws.amazon.com/kinesis/data-firehose/). Você pode habilitar o monitoramento de log de acesso para seus Serviços Lattice adicionando monitoramento à rede de serviços ou individualmente nos serviços que deseja rastrear.

Para configurar esta integração:

1. No console AWS, navegue até [AWS VPC Service](https://console.aws.amazon.com/vpc).
2. No painel esquerdo, no dropdown <DNT>**VPC Lattice**</DNT>, selecione `Service Networks` OU `Services`, dependendo de qual opção você deseja usar para ativar o monitoramento.
3. Selecione o serviço ou rede de serviços que você gostaria de monitor.
4. Selecione a guia `Monitoring` .
5. Selecione `Edit access logs`.
6. Ative o monitoramento `Access logs` .
7. Marque a caixa de seleção denominada `Kinesis Data Firehose delivery stream`.
8. Selecione um fluxo de entrega no dropdown configurado para apontar para New Relic.
9. Salve as alterações.

## Encontre e use dados de log de acesso [#find-log-data]

Você pode consultar e explorar seus dados em nossa interface de log usando este filtro: `ServiceNetworkArn LIKE '%vpc-lattice%'`.

Explore [Log de acesso para VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html) para obter mais informações sobre os dados log que estão sendo enviados.

## Encontre e use dados de métricas do CloudWatch [#find-metric-data]

Você pode consultar e explorar os dados do CloudWatch através [do nosso explorador de consulta métrica](/docs/data-apis/understand-data/metric-data/query-metric-data-type/).

Confira os detalhes das [métricas do CloudWatch for VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) para obter mais informações sobre os dados métricos que estão sendo enviados.
