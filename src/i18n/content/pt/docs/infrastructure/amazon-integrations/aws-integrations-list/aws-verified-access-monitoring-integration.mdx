New Relic fornece uma integração para seu [log de acesso verificado AWS ](https://docs.aws.amazon.com/verified-access/latest/ug/what-is-verified-access.html).

## Recurso [#about]

[AWS Verified Access](https://aws.amazon.com/verified-access) é uma ferramenta que fornece acesso seguro a aplicativos corporativos sem a necessidade de VPN.

Sempre que alguém tenta acessar seu aplicativo corporativo, um log é criado e pode ser encaminhado para a New Relic por meio [do AWS Kinesis Firehose](https://aws.amazon.com/kinesis/firehose).

Essa integração monitora sua atividade de acesso verificado, incluindo os principais endereços IP negados, contagem geral de atividades negadas e concedidas e muito mais.

## Ative a integração do Verified Access [#activate-integration]

<Callout variant="important">
  Para usar o Kinesis Firehose, você deve ter um fluxo de entrega do Kinesis Data Firehose já criado. Caso contrário, você pode criar um acessando o recurso Amazon Kinesis Firehose no [console da AWS](https://console.aws.amazon.com/firehose) e criando um fluxo de entrega com `Destination: New Relic`.

  [Saiba mais sobre como criar um fluxo de entrega do Amazon Kinesis Data Firehose.](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html?icmpid=docs_console_unmapped)
</Callout>

Para configurar esta integração:

1. No console AWS, navegue até [AWS VPC Service](https://console.aws.amazon.com/vpc).

2. No painel esquerdo, no dropdown

   <DNT>
     **AWS Verified Access**
   </DNT>

   , selecione `Verified Access instances`.

3. Selecione o ID da instância que você deseja configurar.

4. Selecione a guia `Verified Access instance logging configuration` .

5. Selecione `Modify Verified Access instance logging configuration`.

6. Selecione o botão de alternância `Deliver to Amazon Kinesis Data Firehose` .

7. Selecione um fluxo de entrega configurado para apontar para o New Relic.

## Encontre e use dados [#find-data]

Você pode consultar e explorar seus dados em [nossa interface de log](/docs/logs/ui-data/use-logs-ui), usando um filtro de `metadata.product.name = Verified Access`.
