A New Relic oferece uma integração para relatar seus dados [do AWS Direct Connect](https://aws.amazon.com/directconnect/) . Este documento explica como ativar esta integração e descreve os dados que podem ser reportados.

## Ativar integração [#activate]

Para habilitar essa integração, siga os procedimentos padrão para [conectar os serviços da AWS ao New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informações [de pesquisa](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) padrão para a integração do AWS Direct Connect:

* Intervalo de sondagem New Relic : 5 minutos
* Intervalo de dados do Amazon CloudWatch: 1 minuto

## Dados métricos [#metrics]

Essa integração coleta dados do AWS Direct Connect para Connection.

### Dados de conexão Direct Connect

      <td>

      </td>

      <td>
        Contar
      </td>

      <td>
        O estado da conexão. `0` indica PARA BAIXO e `1` indica PARA CIMA.

      <td>
        `connectionBpsEgress`

    </tr>

      <td>

      </td>

      <td>
        BitsPorSegundo
      </td>

      <td>
        A taxa de bits dos dados de entrada para o lado AWS da conexão. O número relatado é o agregado durante o período especificado (5 minutos por padrão, 1 minuto no mínimo).
      </td>
    </tr>

    <tr>
      <td>
        `connectionPpsEgress`
      </td>

      <td>
        Contagem por segundo
      </td>

      <td>
        A taxa de pacotes para dados de saída do lado AWS da conexão. O número relatado é o agregado durante o período especificado (5 minutos por padrão, 1 minuto no mínimo).
      </td>
    </tr>

    <tr>
      <td>
        `connectionPpsIngress`
      </td>

      <td>
        Contagem por segundo
      </td>

      <td>
        A taxa de pacotes para dados de entrada no lado AWS da conexão. O número relatado é o agregado durante o período especificado (5 minutos por padrão, 1 minuto no mínimo).
      </td>
    </tr>

    <tr>
      <td>
        `connectionCRCErrorCount`
      </td>

      <td>
        Contar
      </td>

      <td>
        O número de vezes que erros de verificação de redundância cíclica (CRC) são observados para os dados recebidos no dispositivo AWS. Essa métrica é um contador que pode permanecer o mesmo por um longo período de tempo porque só é redefinido quando o dispositivo é reinicializado. Se esse valor não aumentar, não haverá ocorrência de erros.
      </td>
    </tr>

    <tr>
      <td>
        `connectionLightLevelTx`
      </td>

        Contar
      </td>

      <td>
        Indica a integridade da conexão de fibra para tráfego de saída (saída) do lado AWS da conexão. Esta métrica está disponível apenas para conexões com velocidades de porta de 10 Gbps.
      </td>
    </tr>

    <tr>
      <td>
        `connectionLightLevelRx`
      </td>

      <td>
        Contar
      </td>

      <td>
        Indica a integridade da conexão de fibra para tráfego de entrada (entrada) no lado AWS da conexão. Esta métrica está disponível apenas para conexões com velocidades de porta de 10 Gbps.
      </td>
    </tr>
