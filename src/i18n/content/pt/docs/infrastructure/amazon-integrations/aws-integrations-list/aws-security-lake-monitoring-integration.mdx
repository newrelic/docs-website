[A integração da New Relic Infrastructure](/docs/infrastructure/introduction-infra-monitoring) inclui uma integração para AWS Security Lake, permitindo que você envie seus dados de log de segurança para o New Relic.

Colete e envie dados de telemetria para New Relic do [Security Lake](https://aws.amazon.com/security-lake/) usando nossa integração. Você pode usar esta integração para monitor seus serviços, consultar dados recebidos e construir um painel para observar tudo rapidamente.

## Ativar integração [#activate]

Para viabilizar essa integração, configure um S3 alocador de logs. Sugerimos usar nosso aplicativo de encaminhamento Serverless para maior facilidade e conveniência, mas você também pode definir o seu próprio.

<Callout variant="tip">
  Você tem duas opções para configuração de monitoramento do Security Lake. Você pode consolidar várias regiões para evitar a repetição de etapas ou pode configurá-lo por região. Para obter mais detalhes, consulte [gerenciamento de diversas regiões](https://docs.aws.amazon.com/security-lake/latest/userguide/manage-regions.html).
</Callout>

## Pré-requisitos [#prerequisites]

Antes de habilitar essa integração, primeiro certifique-se de que estas etapas sejam concluídas para o Security Lake:

1. Conclua as etapas mencionadas no [Guia de conceitos básicos](https://docs.aws.amazon.com/security-lake/latest/userguide/getting-started.html) do Amazon Security Lake.
2. Conclua os pré-requisitos para configurar um assinante com acesso a dados seguindo as etapas no [guia do assinante do Security Lake](https://docs.aws.amazon.com/security-lake/latest/userguide/subscriber-data-access.html#prereqs-creating-subscriber).

## Etapas de configuração [#steps]

Esta é uma visão geral das etapas que você executará:

1. [Crie um assinante do Security Lake](#create-subscriber).
2. [Instale nosso Amazon Security Lake dedicado direcionador de logs](#install-log-forwarder).
3. [Encontre e use seus dados log ](#logs).

## Crie um assinante do Security Lake [#create-subscriber]

1. Navegue até o recurso Security Lake em seu [console AWS](console.aws.amazon.com/securitylake).

2. Selecione <DNT>**Subscribers**</DNT> e selecione <DNT>**Create Subscriber**</DNT>.

3. Dê um nome ao assinante e selecione uma região.

4. Selecione quais fontes log e eventos você deseja enviar para o New Relic.

5. Preencha os detalhes restantes conforme descrito abaixo:

## Instale nosso Amazon Security Lake direcionador de logs [#install-log-forwarder]

Para instalar o direcionador de logs:

1. Abra o [AWS Serverless Application Repository](https://serverlessrepo.aws.amazon.com/applications) em seu Browser.
   <img title="AWS Lambda - Select region" alt="AWS Lambda - Select region" src="/images/serverless_screenshot-crop_AWS-Lambda-select-region.webp" />
2. Pesquise `newrelic` e marque <DNT>**Show apps that create custom IAM roles or resource policies**</DNT> para encontrar `newrelic-securitylake-s3-processor-LogForwarder`.
3. Clique nos detalhes `newrelic-securitylake-s3-processor-LogForwarder` e clique em <DNT>**Deploy**</DNT>.
4. Copie/cole o ARN `AWS role ID` da etapa anterior no campo `SecurityLakeSubscriberRoleArn` .
5. Copie/cole o ARN `Subscription endpoint` da etapa anterior no campo `SecurityLakeSubscriberRoleArn` .
6. Insira o `ExternalID` que você adicionou na etapa anterior.
7. Insira seu <InlinePopover type="licenseKey" />no campo `NRLicenseKey` .
8. Confirme e selecione <DNT>**Deploy**</DNT>.

Para obter mais detalhes sobre isso, consulte [nossos documentos do Amazon Security Lake direcionador de logs](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3).

## Encontre e use dados log [#logs]

Para encontrar seu logon no New Relic, vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT> e defina <DNT>**Attributes**</DNT> como `product.name` e escolha a origem log desejada.

As seguintes fontes de log são atualmente suportadas:

* [ROTA 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [FLUXO DE VPC](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [CLOUDTRAIL](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [CENTRO DE SEGURANÇA](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)

<Callout variant="tip">
  O Amazon Security Lake usa o [esquema OCSF](https://schema.ocsf.io/) para seu log.
</Callout>
