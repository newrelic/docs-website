[O New Relic](/docs/infrastructure/introduction-infra-monitoring) inclui uma integração para coletar seus dados do Amazon Kinesis Data Firehose. Este documento explica como ativar essa integração e descreve os dados que podem ser relatados.

## Recurso [#features]

[O Amazon Kinesis Data Firehose](https://aws.amazon.com/kinesis/firehose/) oferece uma maneira simples de capturar e carregar dados de streaming. Ele pode capturar, transformar e carregar dados de streaming no Amazon Kinesis Data Analytics, Amazon S3, Amazon Redshift e Amazon Elasticsearch Service, permitindo análises quase em tempo real.

A integração Kinesis Data Firehose da New Relic relata dados como registros e bytes indexados, contagens de dados copiados para serviços da AWS, idade e atualização dos registros e outros [dados métricos](#metrics).

## Ativar integração [#activate]

Para habilitar essa integração, siga os procedimentos padrão para [conectar os serviços da AWS ao New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

## Configuração e polling [#polling]

Você pode alterar a frequência de pesquisa e filtrar dados usando [opções de configuração](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

[Pesquisa padrão](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) para a integração do Amazon Kinesis Data Firehose:

* Intervalo de sondagem New Relic : 5 minutos
* Intervalo de dados do Amazon CloudWatch: 1 minuto

## Encontre e use dados [#find-data]

Para encontrar seus dados de integração, acesse <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; AWS**</DNT> e selecione um dos links de integração do Kinesis Data Firehose.

Você pode [consultar e explorar seus dados](/docs/using-new-relic/data/understand-data/query-new-relic-data) usando o [tipo de evento](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `QueueSample` , com um valor `provider` de `KinesisDeliveryStream`.

Para saber mais sobre como usar seus dados, consulte [Compreender e usar dados de integração](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).
