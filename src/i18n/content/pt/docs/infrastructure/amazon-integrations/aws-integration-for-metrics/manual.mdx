Para uma integração perfeita da sua conta <DNT>AWS</DNT> com o New Relic, a [integração via <DNT>CloudFormation</DNT> e CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/via-cloudformation) é a abordagem recomendada. Se você não puder usar os modelos <DNT>CloudFormation</DNT> e não quiser usar <DNT>Terraform</DNT>, siga este procedimento para integrar manualmente a partir do console <DNT>AWS</DNT>.

<Callout variant="tip">
  * Se você gerencia várias contas <DNT>AWS</DNT>, [conecte cada conta a uma única conta New Relic](/docs/infrastructure/amazon-integrations/manage-aws-data/aws-multi-account).
  * Se você gerencia várias regiões dentro dessas contas, crie um [Kinesis Data <DNT>Firehose</DNT>](#create-firehose) separado para cada região.
</Callout>

<Steps>
  <Step>
    ## Pré-requisitos [#prerequisites]

    Antes de começar a integrar sua conta <DNT>AWS</DNT> com o New Relic, certifique-se de atender aos seguintes pré-requisitos:

    * **Uma [Ingest - chave de licença e uma chave de usuário](https://one.newrelic.com/api-keys)do New Relic**: você precisará dessas chaves para conectar seus serviços <DNT>AWS</DNT> à sua conta New Relic.
    * **[Permissões](https://docs.aws.amazon.com/ARG/latest/userguide/gettingstarted-prereqs-permissions.html)** para implantar novos recursos <DNT>AWS</DNT> e **[funções do IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles.html)**: você precisará dessa permissão para sua conta <DNT>AWS</DNT>.
    * **[Permissões](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#infrastructure)** para integração na nuvem: você precisará dessas permissões para sua conta New Relic.
  </Step>

  <Step>
    ## Criar um Kinesis Data <DNT>Firehose</DNT> [#create-firehose]

    No console <DNT>AWS</DNT>, defina os seguintes parâmetros de destino para o fluxo de entrega:

    * Fonte: <DNT>Direct PUT</DNT> ou outras fontes

    * Destino: New Relic

    * Transformação de dados: Desativado

    * Conversão de formato de registro: Desativado

    * Certifique-se de que as seguintes configurações estejam definidas:

      * Configuração do New Relic (configurações de destino):

        * URL do endpoint HTTP:

          * Datacenter dos EUA: `https://aws-api.newrelic.com/cloudwatch-metrics/v1`
          * Datacenter da UE: `https://aws-api.eu01.nr-data.net/cloudwatch-metrics/v1`
          * Datacenter do JP: `https://aws-api.jp.nr-data.net/cloudwatch-metrics/v1`

        * Chave de API: Digite sua<InlinePopover type="licenseKey" />

        * Codificação de conteúdo: `GZIP`

        * Duração da nova tentativa: `60`

        * Condições de buffer do New Relic:

          * Tamanho do buffer: `1 MB`
          * Intervalo de buffer: `60 (seconds)`

      * Modo de backup S3: somente dados com falha

      * Bucket de backup do S3: selecione um bucket ou crie um novo para armazenar métricas que falharam ao serem enviadas.

      * Configurações avançadas:
        * Acesso ao serviço: forneça permissões para a função do IAM:
          * Criar ou atualizar função do IAM
  </Step>

  <Step>
    ## Criar um CloudWatch Metric Streams [#create-metric-stream]

    1. No console do <DNT>AWS</DNT>, vá para <DNT>**CloudWatch service**</DNT>, selecione <DNT>**Streams**</DNT> no menu <DNT>**Metrics**</DNT> e clique em <DNT>**Create metric stream**</DNT>.
    2. Para controlar quais serviços do <DNT>AWS</DNT> enviam métrica para o New Relic, configure filtros de inclusão e exclusão.
    3. Na lista de fluxos de entrega do <DNT>Firehose</DNT>, selecione o fluxo que você criou e insira um nome significativo para o fluxo de métricas, por exemplo, `newrelic-metric-stream`.
    4. No campo de formato de saída, selecione `Open Telemetry 0.7` ou `Open Telemetry 1.0`. JSON não é suportado.
  </Step>

  <Step>
    ## Conecte seu <DNT>AWS</DNT> [#connect-aws-account]

    1. Na New Relic, vá para [<DNT>**Infrastructure &gt; AWS**</DNT>](https://one.newrelic.com/infra/infrastructure/cloud-aws).

    2. Selecione a conta correta.

    3. Clique em <DNT>**Set up integration**</DNT>.

    4. Em cada tela, preencha os campos que se aplicam a você. Para esta integração, escolha as seguintes opções:

       * <DNT>**Choose Data type(s)**</DNT>: <DNT>Metrics</DNT>
       * <DNT>**Choose a setup method**</DNT>: <DNT>Manually integrate your AWS account</DNT>

    5. Na tela <DNT>**Select services**</DNT>, alterne <DNT>**Show resources not supported by CloudWatch Metric Streams**</DNT> se necessário, selecione os serviços que deseja monitorar e continue.

    6. Na tela <DNT>**Choose data integration type**</DNT>, escolha entre as seguintes opções:

       * **Monitoramento <DNT>AWS</DNT> em tempo real (Recomendado)**: Esta opção é para os serviços que suportam CloudWatch Metric Streams.
       * **API Polling**: esta opção é para serviços não suportados pelo CloudWatch Metric Streams.

    7. Da página <DNT>**Create role**</DNT> para a página <DNT>**Configure budgets policy**</DNT>, siga as instruções na tela e conclua as etapas no console <DNT>AWS</DNT>.
  </Step>

  <Step>
    ## Valide seus dados [#validate]

    Para confirmar que você está recebendo dados do Metric Streams:

    1. No New Relic, vá para <DNT>**[one.newrelic.com &gt; All capabilities &gt; Infrastructure &gt; AWS](https://one.newrelic.com/infra/infrastructure/cloud-aws)**</DNT>, pesquise sua conta <DNT>AWS</DNT> conectada e procure por contas **Stream**.
    2. No dashboard de status da conta, confirme se New Relic está recebendo dados métricos, incluindo erros e o número de namespace e métrica ingeridos.
    3. Para explorar métricas específicas, escreva consultas para os conjuntos de métricas que você deseja analisar.

    Pode levar alguns minutos para que New Relic detecte novos recursos e os sintetize como entidade.

    <Callout variant="tip">
      <DNT>AWS</DNT> As métricas do CloudWatch para serviços globais, como <DNT>AWS</DNT> Billing, estão disponíveis somente na região <DNT>**us-east-1**</DNT>. Certifique-se de que haja um CloudWatch Metric Streams ativo configurado nessa região.
    </Callout>
  </Step>
</Steps>
