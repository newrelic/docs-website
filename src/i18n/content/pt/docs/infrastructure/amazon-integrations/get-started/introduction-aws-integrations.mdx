A integração da Amazon permite monitor seus dados AWS em diversos recursos New Relic . Habilitar a integração do [AWS CloudWatch Metric Streams](/docs/integrations/amazon-integrations/aws-integrations-list/aws-metric-stream) é a solução recomendada para monitor todas as métricas do CloudWatch de todos os serviços da AWS (incluindo namespace personalizado). Além disso, integrações adicionais estão disponíveis para obter visibilidade estendida dos principais serviços da AWS, além das informações disponíveis no CloudWatch. Para uma referência completa das métricas suportadas, verifique as métricas disponíveis do CloudWatch para cada serviço nas páginas de documentação AWS .

## Conecte AWS e New Relic [#connect]

Para obter dados da AWS, siga o procedimento para [conectar a AWS ao New Relic](/docs/infrastructure/amazon-integrations/getting-started/connect-aws-services-infrastructure).

Integrações <DNT>**API Polling**</DNT> adicionais podem ser habilitadas no AWS CloudWatch Metric Streams para extrair dados que não estão disponíveis como métricas do CloudWatch. As seguintes integrações não são substituídas pelas correntes métricas:

* AWS CloudTrail
* AWS Health
* AWS Trusted Advisor
* AWS X-Ray

Por fim, outras integrações podem exigir configuração adicional em sua conta AWS :

* [AWS VPC Flow Logs](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-vpc-flow-logs-monitoring-integration/)
* [AWS CloudFormation](https://github.com/newrelic/cloudformation-partner-integration)

## Custos de integração e AWS [#cloudwatch-billing]

Tenha em mente os seguintes itens:

* A definição de preço AWS CloudWatch Metric Streams é definida com base no número de atualizações métricas. Para obter informações atualizadas sobre preços, consulte [Preços do AWS CloudWatch](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-Metric-Streams.html).
* AWS Kinesis Data Firehose é usado como método de entrega. Para obter detalhes, consulte a [página de definição de preço do AWS Firehose](https://aws.amazon.com/kinesis/data-firehose/pricing/).
* O AWS Config pode ser habilitado opcionalmente em sua conta da AWS e usado para enriquecer métricas do CloudWatch com metadados de recursos. Com o AWS Config, você é cobrado com base no número de itens de configuração registrados. Consulte a [página de definição de preço do AWS Config](https://aws.amazon.com/config/pricing/) para obter detalhes.
* [AWS de tag Resource Groups API](https://docs.aws.amazon.com/resourcegroupstagging/latest/APIReference/overview.html) é usada para enriquecer a métrica com a tag personalizada definida nos AWS recursos associados (aplicável a fluxos de pesquisa e métrica).
* Se a integração de polling estiver habilitada (em vez de fluxos de métricas), New Relic usará a API do Amazon CloudWatch para obter métricas dos serviços da AWS que você monitor. O número de chamadas para a API do CloudWatch aumenta à medida que você permite mais integração. Adicione recursos AWS a essas integrações ou dimensione-as em mais regiões. Isso pode fazer com que as solicitações à API do CloudWatch excedam os <DNT>**1 million free**</DNT> limites concedidos pela AWS e [aumentem sua fatura do CloudWatch](/docs/infrastructure/amazon-integrations/troubleshooting/cloudwatch-billing-increase).
* AWS oferece monitoramento aprimorado para alguns de seus serviços o que permite maior métrica, com maior frequência. Por exemplo, consulte [Custos de monitoramento aprimorado do RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/USER_Monitoring.OS.overview.html#USER_Monitoring.OS.cost).

## Visualize seus dados da AWS [#insights]

Depois de concluir a configuração, os dados dos seus serviços da AWS serão reportados diretamente para a New Relic. A entidade AWS para muitos serviços está listada em nosso [explorador de entidade](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer). métrica e evento aparecem em [métrica e evento](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/). Os dados AWS também são visíveis na interface da infraestrutura.

Para visualizar seus dados da AWS:

1. Vá para <DNT>\*\*[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities)</DNT>.
2. Pesquise <DNT>**Amazon Web Services**</DNT> e clique no bloco.

Você pode visualizar e reutilizar consultas NRQL no painel <InlinePopover type="dashboards"/>pré-configurado e no painel <DNT>**Events explorer**</DNT>. Isso permite que você adapte a consulta às suas necessidades específicas.

## Disponibilidade de região [#availability]

A maioria dos serviços da AWS oferece endpoint regional para reduzir a latência de dados entre os recursos da nuvem e o aplicativo. New Relic pode obter dados de monitoramento de serviços e endpoints localizados em todas as [regiões AWS ](http://docs.aws.amazon.com/general/latest/gr/rande.html), exceto China.
