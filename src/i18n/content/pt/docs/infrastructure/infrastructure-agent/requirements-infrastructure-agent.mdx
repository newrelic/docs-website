Antes de [instalar nosso agente de infraestrutura](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic), certifique-se de que seu sistema e qualquer integração no host que você configurar atendam aos requisitos. Você também precisa de uma conta New Relic. [Cadastre-se gratuitamente](https://newrelic.com/signup). Não é necessário cartão de crédito.

## Processador arquitetura [#processors]

O agente de infraestrutura suporta estas arquiteturas de processador:

* <DNT>**Linux**</DNT>: 64 bits para processador x86 arquitetura (também requer gerenciador de pacote e dependência de 64 bits)
* <DNT>**Windows**</DNT>: 32 e 64 bits para arquitetura de processador x86
* <DNT>**ARM**</DNT>: a arquitetura arm64 incluindo [o processador AWS Graviton 2](https://aws.amazon.com/ec2/graviton/) é compatível com sistemas operacionais Linux compatíveis. integração no host também são suportadas (com exceção da integração Oracle).
* <DNT>**macOS**</DNT>: x86 de 64 bits e silício da Apple.

## Sistema operacional [#os]

O agente de infraestrutura suporta estes sistemas operacionais até o fim da vida útil do fabricante. O agente ainda pode ser compatível com versões mais antigas, mas consulte as instruções passo a passo para cada sistema operacional se a instalação guiada não estiver mais disponível para uma versão específica do sistema operacional.

## Nome exclusivo do host [#hostname]

O agente de infraestrutura usa o nome do host para identificar exclusivamente cada servidor. Para evitar métricas imprecisas de combinar vários servidores sob um único nome do host, certifique-se de que cada servidor monitor tenha um nome do host exclusivo.

* Você pode usar a configuração opcional [`display_name`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#display_name) para substituir o nome do host padrão.
* Os servidores denominados `localhost` não são relatados porque esse é um nome padrão e inerentemente não exclusivo.

## Permissões

O agente de infraestrutura requer estas permissões:

* <DNT>**Linux**</DNT>: por padrão, [o agente é executado e instalado como root](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#agent-mode-intro). Você também pode selecionar modos de [execução](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-unprivileged) [com](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-privileged) ou sem privilégios .
* <DNT>**Windows**</DNT>: o agente deve ser instalado a partir de uma conta de administrador e requer privilégios de administrador para ser executado.
* <DNT>**macOS**</DNT>: o agente pode ser instalado a partir de qualquer conta de usuário.

## Bibliotecas

Para [versões de agente 1.1.19 ou superior](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes), você precisa da biblioteca `libcap` para instalar. Está disponível no repositório oficial da sua distribuição.

## Acesso à rede [#network]

Para reportar dados à New Relic, nosso agente de infraestrutura deve ter acesso de saída a [determinados domínios e portas](/docs/apm/new-relic-apm/getting-started/networks#infrastructure). Se o seu sistema precisar de um proxy para se conectar a esses domínios, use a configuração [`proxy`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#proxy) .

## Software de contêiner [#container]

O agente de infraestrutura [instrumento Docker contêiner](/docs/infrastructure-docker-instrumentation) quando instalado no servidor host. Oferecemos suporte às versões 1.12 ou superiores Docker .

## Uso de CPU, memória e disco [#resources]

O agente de infraestrutura é bastante leve. Para uso típico de CPU, memória e disco, consulte nossa página sobre [sobrecarga de desempenho do agente](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead). Para obter mais informações sobre sistemas de arquivos suportados, consulte [Atributo de exemplo de armazenamento](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#storagesample-attributes).
