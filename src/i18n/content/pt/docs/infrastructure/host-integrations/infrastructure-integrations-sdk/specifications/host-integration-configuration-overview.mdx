Nossa [integração no host](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations) envia dados para o agente de infraestrutura, que por sua vez envia esses dados para a New Relic. A forma como a integração interage com o agente é controlada pela configuração de cada integração.

Para obter mais informações sobre a que integração isso se aplica, consulte [integração no host instalação](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations).

## Visão geral de como funciona a configuração [#overview]

Algumas de nossas integração no host são programas externos executados pelo [agente de infraestrutura](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent). Cada integração monitora um [serviço específico](/docs/integrations/host-integrations/host-integrations-list). Uma integração possui, no mínimo, estes arquivos:

* Um [executável](/docs/infrastructure/infrastructure-integrations/custom-integrations/create-integration-executable-file) que exporta vários [tipos de dados](/docs/integrations/new-relic-integrations/get-started/introduction-infrastructure-integrations#data-types) em formato JSON esperado pelo agente
* Um ou mais arquivos de configuração no formato YAML (por exemplo, a [configuração de integração do Apache)](/docs/integrations/host-integrations/host-integrations-list/kafka-monitoring-integration#config). (Recomendamos [lintar](http://www.yamllint.com/) os arquivos de configuração YAML antes de usar para evitar problemas de formatação.)

Observe que, além da configuração específica da integração no host, você também pode editar a [configuração do agente de infraestrutura](/docs/infrastructure/install-infrastructure-agent/configuration/configure-infrastructure-agent).

## Localização do arquivo de configuração [#file-location]

Com integração padrão no host instalação, a configuração está localizada no diretório do agente de infraestrutura. O agente determina esse local de configuração por meio de uma [configuração em seu próprio arquivo de configuração](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#plugin-dir).

Para algumas implementações, a configuração da integração estará localizada em outro lugar. Por exemplo:

* Serviços em execução no [Kubernetes](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes): a configuração está localizada no arquivo de configuração de integração do Kubernetes.
* Serviços em execução no [Amazon ECS](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs): a configuração é colocada no console AWS.

## Formatos de configuração [#config-formats]

Integração no host usa dois formatos de configuração:

* <DNT>
    **Standard**
  </DNT>

  : A partir de dezembro de 2019, [o agente de infraestrutura versão 1.8.0](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) passou a suportar um novo formato utilizado por alguma integração. Este formato usa um único arquivo de configuração e oferece outras melhorias. Para obter mais detalhes, consulte [Configuração padrão](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-standard-configuration-format).

* <DNT>
    **Legacy**
  </DNT>

  : Este é o formato utilizado pela maioria das integração no host. Esta configuração usa dois arquivos: um arquivo de definição e um arquivo de configuração. Para obter mais detalhes, consulte [configuração de legado](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-legacy-configuration-format/).
