O [SDK de integração de infraestrutura](/docs/infrastructure/integrations-sdk/get-started/intro-infrastructure-integrations-sdk) permite construir uma integração personalizada no host. Este documento explica as ferramentas de construção e os recursos disponíveis para construir uma integração no host com nossas ferramentas de linguagem Go.

## Tutorial de integração [#tutorial]

<Callout variant="important">
  O tutorial a seguir é baseado em integração usando o protocolo SDK integração v3.  
    Encontre mais informações sobre o protocolo de integração v4 no [repositório Github](https://github.com/newrelic/infra-integrations-sdk).
</Callout>

O [tutorial de construção de integração de linguagem Go no GitHub](https://github.com/newrelic/infra-integrations-sdk/blob/master/docs/tutorial.md) fornece procedimentos passo a passo para construir uma integração de linguagem Go que reporte dados Redis.

O tutorial mostra como criar uma integração usando a linha de comando do Linux, mas você pode usar as mesmas técnicas para uma integração do Windows com uma instalação Go padrão e PowerShell. O comando `make` não funcionará com o PowerShell, mas você pode usar os comandos Go contidos nele como um guia para criar sua integração.

<Callout variant="tip">
  Você pode criar uma integração no host em qualquer idioma, mas Go é o idioma que New Relic usa para sua própria integração e ferramentas de construção. Para criar uma integração em outro idioma, siga as [estruturas de arquivos de integração e os requisitos de saída JSON](/docs/integrations/integrations-sdk/getting-started/integration-file-structure-activation).
</Callout>

## Pacote de construção de integração de idiomas Go [#package]

O tutorial se baseia em um [pacote de biblioteca de construção de integração da linguagem New Relic Go](https://godoc.org/github.com/newrelic/infra-integrations-sdk), que fornece um conjunto de funções e estruturas de dados úteis do Go. O pacote oferece ferramentas que:

* Gere uma estrutura de integração "scaffold" com todos os campos obrigatórios.
* Leia valores de argumentos de linha de comando ou variáveis de ambiente.
* Gere e imprima dados JSON em `stdout`.

Para obter informações sobre formatos de arquivo e especificações de saída JSON, consulte [Requisitos de arquivo](/docs/integrations/integrations-sdk/getting-started/integration-file-structure-activation).
