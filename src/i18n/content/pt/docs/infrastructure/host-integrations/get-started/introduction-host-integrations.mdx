Nossa integração no host permite monitor uma variedade de serviços de terceiros.

## O que são integração no host? [#overview]

Na New Relic, <DNT>**on-host integrations**</DNT> são o que chamamos de nossa integração que funciona em conjunto com nosso agente de infraestrutura. Essas integrações coletam dados de serviços suportados e usam o agente de infraestrutura para enviar esses dados para a New Relic.

Algumas de nossas integração no host são construídas usando [nosso Flex integração](/docs/infrastructure/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration), que você também pode usar para construir sua própria integração personalizada e leve.

## Dados relatados [#data]

Integração no host report métrica e dados de inventário. Algumas integrações têm a opção de ativar relatórios log . Detalhes sobre os dados reportados estão na documentação específica de cada integração.

## Iniciar [#get-started]

Para começar a integração no host: Se você já possui New Relic, acesse a [página<DNT>**Integrations & Agents**</DNT> ](https://one.newrelic.com/marketplace)e pesquise o serviço que deseja monitor.

Ou você pode:

1. Navegue pela lista de integração nesta seção.
2. Revise os requisitos de uma integração e siga os procedimentos para instalá-la e configurá-la.

Para navegar por toda integração, consulte nossa [página de observabilidade instantânea](https://newrelic.com/instant-observability).

## Recurso [#features]

Depois de instalar e ativar uma integração, você poderá:

* [Filtre e analise](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) os dados métricos e de configuração na [interface de infraestrutura](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure).
* Crie [consultas e gráficos personalizados](/docs/using-new-relic/data/understand-data/query-new-relic-data) dos seus dados de integração.
* Crie [condição do alerta](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-condition) para monitor problemas de desempenho do seu serviço

## Monitor configuração remota e multilocatário [#multiple-instances]

Algumas configurações podem usar elementos de terceiros, como banco de dados, que residem em hosts remotos e não acessíveis. Nossa integração pode buscar dados de hosts ou servidores locais e remotos, garantindo um monitoramento contínuo do seu serviço. Eles também oferecem suporte à configuração em cluster multilocatário. Ao abstrair o serviço do host, múltiplas entidades podem ser monitoradas como instância remota.

Para ativar monitoramento remoto e multilocação na integração Apache, Cassandra, MySQL, NGINX e Redis, utilize o [parâmetro`remote_monitoring` ](/docs/infrastructure/host-integrations/understand-use-data/remote-monitoring-host-integrations).
