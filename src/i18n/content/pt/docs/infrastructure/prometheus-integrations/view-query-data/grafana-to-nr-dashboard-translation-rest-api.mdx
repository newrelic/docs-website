Você pode usar a API REST de tradução de dashboard do Grafana para o New Relic para converter seus dashboards existentes do Grafana em dashboards nativos do New Relic. Esta API pode ajudar na migração do Grafana para a New Relic para clientes que atualmente usam o plug-in do Grafana para visualizar seus dados da New Relic. Seja para consolidar suas ferramentas de observabilidade, migrar toda a sua configuração de monitoramento para o New Relic ou simplesmente testar os dashboards do New Relic junto com sua configuração existente do Grafana, esta API torna mais fácil começar.

Esta API usa a lógica de tradução PromQL da New Relic para traduzir suas consultas. Para ver uma lista de recursos e funções suportados do PromQL, consulte [Suporte a recursos do PromQL](/docs/infrastructure/prometheus-integrations/view-query-data/supported-promql-features).

## Pré-requisitos [#prerequisites]

Antes de usar esta API, você precisará de:

* Uma [chave de API de usuário da New Relic](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)
* Seu dashboard do Grafana exportado como JSON (no Grafana: Configurações → Modelo JSON → Copiar para a área de transferência)
* Seu ID da conta New Relic (se você quiser substituir a conta padrão)

## Faça uma requisição aos endpoints da API [#api-endpoints]

Para traduzir um dashboard do Grafana para um dashboard do New Relic, envie uma requisição `POST` para o endpoint apropriado para sua região com o dashboard que você deseja traduzir no corpo da requisição.

* Para os EUA, `https://prometheus-api.newrelic.com/api/v1/translate/dashboard/grafana`
* Para a UE, `https://prometheus-api.eu.newrelic.com/api/v1/translate/dashboard/grafana`

### Autenticação [#authentication]

Todas as chamadas de API exigem uma [chave de API de usuário](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key). Inclua esta chave no cabeçalho `x-api-key` da sua solicitação.

### Corpo da requisição [#request-body]

O corpo da requisição deve ser um objeto JSON com o seguinte formato:

```
{
  "accountId": int [optional],
  "dashboard": object [required]
}
```

Onde `dashboard` é o objeto JSON do dashboard do Grafana que você deseja traduzir, e `accountId` é a conta da qual você deseja que seu dashboard traduzido consulte dados. Se `accountId` não for incluído, o dashboard traduzido consultará dados da conta do usuário que faz a solicitação.

### Formato de resposta [#response-format]

A API retorna um objeto JSON contendo o dashboard New Relic traduzido:

```json
{
  "dashboard": {
    "name": "Translated Dashboard",
    "permissions": "PUBLIC_READ_WRITE",
    "pages": [...]
    }
}
```

## Exemplos [#examples]

<CollapserGroup>
  <Collapser id="dashboard-translation-basic" title="Exemplo: chamada de API básica para traduzir um dashboard do Grafana">
    ```bash
    export NR_USER_KEY="your_user_key"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --slurpfile dashboard "${JSON_FILE}" '{dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-account-override" title="Exemplo: traduzir dashboard com override de ID da conta">
    Se você deseja substituir a conta da qual seu dashboard consultará dados, você pode incluir `accountId: int` na carga da solicitação. Isso fará com que todas as consultas no dashboard consultem dados da conta especificada.

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-and-upload" title="Exemplo: fazer upload de dashboard traduzido para o New Relic usando a API NerdGraph">
    <Callout variant="tip">
      Para mais informações sobre como usar o NerdGraph e encontrar seu endpoint regional, consulte [Introdução ao New Relic NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).
    </Callout>

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)
    export NERDGRAPH_ENDPOINT="US/EU Regional Nerdgraph endpoint"

    NR_DASHBOARD=$(curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard')


    curl -X POST ${NERDGRAPH_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "API-Key: ${NR_USER_KEY}" \
    -d '{
    "query": "mutation($accountId: Int!, $dashboard: DashboardInput!) { dashboardCreate(accountId: $accountId, dashboard: $dashboard) { entityResult { guid name } errors { description } } }",
    "variables": {
    "accountId": '${ACCOUNT_ID}',
    "dashboard": '${NR_DASHBOARD}'
    }
    }'
    ```
  </Collapser>
</CollapserGroup>
