Converta seus dashboards existentes do Grafana em dashboards nativos do New Relic usando nossa API REST de tradução. Seja para consolidar suas ferramentas de observabilidade, migrar toda a sua configuração de monitoramento para o New Relic ou simplesmente testar os dashboards do New Relic junto com sua configuração existente do Grafana, esta API facilita o início.

Esta API usa a lógica de tradução PromQL da New Relic para traduzir suas consultas. Para ver uma lista de recursos e funções PromQL suportados, consulte o [guia de suporte de recursos PromQL](/docs/infrastructure/prometheus-integrations/view-query-data/supported-promql-features).

## Pré-requisitos [#prerequisites]

Antes de usar esta API, você precisará de:

* Uma [chave de API de usuário da New Relic](/docs/apis/intro-apis/new-relic-api-keys/#overview-keys)
* Seu [ID da conta](/docs/accounts/accounts-billing/account-structure/account-id)da New Relic

## Faça uma requisição aos endpoints da API [#api-endpoints]

Para traduzir um dashboard do Grafana para um dashboard do New Relic, envie uma requisição `POST` para o endpoint apropriado para sua região com o dashboard que você deseja traduzir no corpo da requisição.

* NÓS: `https://prometheus-api.newrelic.com/api/v1/translate/dashboard/grafana`
* UE: `https://prometheus-api.eu.newrelic.com/api/v1/translate/dashboard/grafana`
* JP: `https://prometheus-api.jp.newrelic.com/api/v1/translate/dashboard/grafana`

## Referência de API [#api-reference]

### Autenticação [#authentication]

Todas as chamadas de API exigem uma [chave de API de usuário](/docs/apis/intro-apis/new-relic-api-keys/#overview-keys). Inclua esta chave no cabeçalho `api-key` da sua solicitação.

### Corpo da requisição [#request-body]

O corpo da solicitação deve ser um objeto JSON com o seguinte formato:

```json
{
  "accountId": int [optional],
  "dashboard": object [required]
}
```

Onde dashboard é o objeto JSON do dashboard do Grafana que você deseja traduzir. O campo accountId é opcional — se omitido, o dashboard traduzido consultará dados da conta do usuário que faz a solicitação.

### Formato de resposta [#response-format]

A API retorna um objeto JSON contendo o dashboard New Relic traduzido:

```json
{
  "dashboard": {
    "name": "Translated Dashboard",
    "permissions": "PUBLIC_READ_WRITE",
    "pages": [...]
    }
}
```

## Exemplos [#examples]

<CollapserGroup>
  <Collapser id="dashboard-translation-basic" title="Exemplo: chamada de API básica para traduzir um dashboard do Grafana">
    ```bash
    export NR_USER_KEY="YOUR_NEW_RELIC_USER_API_KEY"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<TRANSLATOR_ENDPOINT_FOR_YOUR_REGION>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --slurpfile dashboard "${JSON_FILE}" '{dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-account-override" title="Exemplo: traduzir dashboard com override de ID da conta">
    Se você deseja substituir a conta da qual seu dashboard consultará dados, você pode incluir `accountId: int` na carga da solicitação. Isso fará com que todas as consultas no dashboard consultem dados da conta especificada.

    ```bash
    export NR_USER_KEY="YOUR_NEW_RELIC_USER_API_KEY"
    export ACCOUNT_ID="YOUR_NEW_RELIC_STAGING_ACCOUNT_ID"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<TRANSLATOR_ENDPOINT_FOR_YOUR_REGION>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-and-upload" title="Exemplo: fazer upload de dashboard traduzido para o New Relic usando a API NerdGraph">
    <Callout variant="tip">
      Para mais informações sobre como usar o NerdGraph e encontrar seu endpoint regional, consulte [Introdução ao New Relic NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).
    </Callout>

    ```bash
    export NR_USER_KEY="YOUR_NEW_RELIC_USER_API_KEY"
    export ACCOUNT_ID="YOUR_NEW_RELIC_STAGING_ACCOUNT_ID"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<TRANSLATOR_ENDPOINT_FOR_YOUR_REGION>" # (see API endpoints section above)
    export NERDGRAPH_ENDPOINT="<NERDGRAPH_ENDPOINT_FOR_YOUR_REGION>"

    NR_DASHBOARD=$(curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard')


    curl -X POST ${NERDGRAPH_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "API-Key: ${NR_USER_KEY}" \
    -d '{
    "query": "mutation($accountId: Int!, $dashboard: DashboardInput!) { dashboardCreate(accountId: $accountId, dashboard: $dashboard) { entityResult { guid name } errors { description } } }",
    "variables": {
    "accountId": '${ACCOUNT_ID}',
    "dashboard": '${NR_DASHBOARD}'
    }
    }'
    ```
  </Collapser>
</CollapserGroup>
