O agente de monitoramento de infraestrutura da New Relic é um arquivo executável [leve](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead) que coleta dados sobre seus hosts. Ele também relata dados de [alguns serviços de terceiros](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations), se habilitado, e também [registra dados](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent).

Atualmente, o agente de monitoramento de infraestrutura pode ser executado em muitas distribuições Linux, Windows e macOS. Existem várias maneiras de instalar e implantar o agente, dependendo da sua configuração e necessidades. Este documento descreve como funciona o agente de monitoramento de infraestrutura e como instalá-lo. Você precisará de uma conta New Relic, então, se ainda não tiver uma, [crie-a](https://newrelic.com/signup) gratuitamente, para sempre.

## Início rápido: use nossa instalação guiada [#quick]

A maneira mais rápida de começar a usar nosso agente de monitoramento de infraestrutura é por meio de nossa instalação guiada. Nossa instalação guiada não apenas instala o agente de infraestrutura, mas também tenta descobrir serviços e fontes de log em seu ambiente e recomenda quais instrumentos usar.

Pronto para começar? Clique em um botão com base na [região do seu data center New Relic](/docs/accounts/accounts-billing/account-setup/choose-your-data-center).

## Instale o agente de monitoramento de infraestrutura [#install]

### Linux [#Linux]

Se você ainda não tem uma conta New Relic, a instalação guiada não funcionará. Caso queira seguir o procedimento manualmente, [veja nosso tutorial](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/).

### Windows [#Windows]

Se você ainda não tem uma conta New Relic, a instalação guiada não funcionará. Se você quiser seguir o procedimento manualmente usando nosso instalador MSI, [consulte nosso tutorial](/docs/infrastructure/install-configure-manage-infrastructure/windows-installation/install-infrastructure-windows-server-using-msi-installer).

## Um agente, muitos recursos [#overview]

Nosso agente de monitoramento de infraestrutura coleta dados de desempenho e integridade sobre os recursos e processos do sistema do host onde está habilitado (no local ou virtualizado). Ao mesmo tempo, atua como encaminhador de dois tipos de dados: métricas de serviços core, que são coletadas pela [integração no host](/docs/integrations/host-integrations/getting-started/introduction-host-integrations), e [logs](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent).

Se desejar coletar dados sobre os serviços principais em execução no seu host, será necessário [instalar primeiro o agente de monitoramento de infraestrutura](#install) e, em seguida [, instalar ou ativar a integração no host](/docs/integrations/host-integrations/installation/install-infrastructure-host-integrations).

## Verifique o código-fonte [#source-code]

O agente de monitoramento de infraestrutura é um software de código aberto. Isso significa que você pode [navegar pelo código-fonte](https://github.com/newrelic/infrastructure-agent) e [enviar melhorias](https://github.com/newrelic/infrastructure-agent/blob/master/CONTRIBUTING.md) ou criar seu próprio fork e construí-lo. Para obter mais informações, consulte o [README](https://github.com/newrelic/infrastructure-agent/blob/master/README.md).

## Qual é o próximo

Depois de instalar o agente de monitoramento de infraestrutura:

* Aprenda como [configurar o agente](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent) ou editar o [modelo de configuração](https://github.com/newrelic/infrastructure-agent/blob/master/assets/examples/infrastructure/newrelic-infra-template.yml.example).
* Instale [integração no host](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) (por exemplo, Apache ou MySQL).
* [Habilite o encaminhamento de logs usando o agente de infraestrutura](/docs/logs/enable-new-relic-logs/1-enable-logs/forward-your-logs-using-new-relic-infrastructure).
* Aprenda como [gerenciar o agente](/docs/infrastructure/install-infrastructure-agent/manage-your-agent).
