[A desinstalação do agente de infraestrutura](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations) não afeta diretamente nenhuma [integração de sua infraestrutura](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure): Se você desinstalar o agente, sua integração permanecerá. Da mesma forma, se você [desabilitar ou desinstalar sua integração](#uninstall-integrations), o agente de infraestrutura permanecerá.

Para desinstalar qualquer uma das suas integrações, siga o procedimento correspondente ao tipo de integração.

## Integração sem host [#uninstall-on-host]

Se você utilizou o pacote integração, [consulte as instruções do pacote integração](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations#uninstall-package).

  <Collapser
    id="uninstall-kubernetes"
    title="Kubernetes"
  >
    Certifique-se de usar o contexto apropriado na máquina onde executará o Helm e `kubectl`:

    Você pode verificar os contextos disponíveis com:

    ```
    kubectl config get-contexts
    ```

    E mude para o contexto desejado usando:

    ```
    kubectl config use-context CONTEXT_NAME
    ```

    Se você usou o Helm para instalar a integração do Kubernetes, basta desinstalá-lo usando o Helm:

    ```
    helm uninstall --namespace NAMESPACE_USED_DURING_INSTALLATION RELEASE_NAME
    ```

    Se você instalou a integração do Kubernetes usando um manifesto, use o mesmo manifesto para desinstalá-la:

    ```
    kubectl delete -f newrelic-infrastructure.yaml
    ```
  </Collapser>

## Afastando-se do pacote de integração [#uninstall-package]

Embora ainda seja possível utilizar o pacote integração, recomendamos removê-lo completamente e trabalhar com integração individualmente.

O último pacote integração contém as seguintes versões da integração:

* Apache 1.1.2
* Cassandra 2.0.3
* MySQL 1.1.5
* Nginx 1.0.2
* Redis 1.0.1

Se você remover o pacote integração e quiser continuar usando o pacote integração no host, será necessário instalá-los um por um. Para desinstalar o pacote e reinstalar sua integração:

1. Remova o pacote de integração seguindo estas instruções. Os arquivos de configuração da integração antiga não serão excluídos, portanto você não precisará configurá-los novamente.
