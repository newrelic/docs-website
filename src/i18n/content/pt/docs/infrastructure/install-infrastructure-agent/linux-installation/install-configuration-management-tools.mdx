Você tem várias maneiras de começar a usar nosso [agente de infraestrutura](/docs/infrastructure/infrastructure-monitoring/get-started/get-started-infrastructure-monitoring/):

* Através de nossa [instalação guiada](/docs/new-relic-one/use-new-relic-one/cross-product-functions/install-configure/new-relic-guided-install-overview/).
* Seguindo as instruções passo a passo para [instalar o agente manualmente](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux#manual-install).
* Usando ferramentas de gerenciamento de configuração.

Para instalar o agente de infraestrutura com uma ferramenta de gerenciamento de configuração, clique no bloco abaixo:

## Instalar usando arquivos tarball [#custom-setups-tarball]

Para <DNT>**custom**</DNT> cenários de configuração, você pode instalar o agente de infraestrutura usando [nossos arquivos tarball](https://download.newrelic.com/infrastructure_agent/binaries/linux/) nos modos [assistido](/docs/infrastructure/install-configure-manage-infrastructure/linux-installation/tarball-assisted-install-infrastructure-linux) ou [manual](/docs/infrastructure/install-configure-manage-infrastructure/linux-installation/tarball-manual-install-infrastructure-linux) .

Isto é especialmente útil quando você precisa adaptar as configurações de instalação padrão ao seu ambiente, ou instalar o agente de infraestrutura em distribuições que não possuem o pacote `newrelic-infra` em seu repositório.

<Callout variant="important">
  Observe que a instalação customizada do agente de infraestrutura usando arquivos tarball não é oficialmente suportada.
</Callout>
