Se você usar o agente de infraestrutura ou [integração no host](/docs/integrations/host-integrations/get-started/introduction-host-integrations) que reporta dados por meio do agente de infraestrutura, há diversas maneiras de configurar o relatório de dados. Aqui estão duas opções comuns para gerenciar relatório de dados:

* [Habilitar/desabilitar métrica de processo](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#enable-process-metrics)
* [Selecione um atributo específico para reportar](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#include-matching-metrics)

Para outras opções de configuração do agente, consulte [Configuração](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings).

Para nossa integração de infraestrutura, você também pode alterar a frequência do relatório de dados:

* Para [integração no host](/docs/infrastructure/host-integrations/host-integrations-list): use uma configuração [de configuração](/docs/integrations/integrations-sdk/file-specifications/host-integration-configuration-overview) `interval` de integração específica.
* Para integração na nuvem (AWS, Azure, Google Cloud): edite a [frequência de polling](/docs/integrations/new-relic-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations).

Para obter mais informações sobre o gerenciamento de dados da New Relic em geral, consulte [Gerenciar dados](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-your-data).
