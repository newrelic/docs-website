## Visão geral

O Agent Control oferece uma maneira centralizada e contínua de gerenciar todo o ciclo de vida dos seus agentes, incluindo atualizações. Em vez de atualizar manualmente os agentes em cada host ou cluster, você define o estado desejado no Fleet Control e o Agent Control cuida automaticamente do resto.

O processo de atualização do agente Control e do agente que ele gerencia foi projetado para ser automatizado e não disruptivo.

### Principais benefícios do processo de atualização com o New Relic Control

* Controle centralizado: gerencie todas as versões do agente para toda a sua frota a partir de um único local.
* Não disruptivo: As atualizações são realizadas usando estratégias de atualização gradual para evitar tempo de inatividade.
* Automatizado: depois de definir uma nova versão, todo o processo de implantação e reconciliação é realizado automaticamente.
* Resiliente: O sistema foi projetado para ser resiliente a atualizações com falha e pode retornar ao último estado bom conhecido se ocorrer um problema.

## Atualizando seus Agentes gerenciados

Você pode atualizar qualquer agente gerenciado pelo Agent Control simplesmente alterando sua versão na configuração remota dentro do Fleet Control. Essa abordagem declarativa garante consistência e confiabilidade em toda a sua frota.

* **Edite a configuração:** Navegue até a configuração da sua frota no Fleet Control.
* **Atualize a versão:** Localize o agente que você deseja atualizar (por exemplo, `infrastructure`) e altere o valor de `chart_version` para o novo número da versão.
* **Implantar as Alterações:** Salve e implante a configuração atualizada.

O Agent Control detectará automaticamente a alteração e realizará uma atualização gradual sem interrupções, garantindo que seus novos agentes estejam em execução e íntegros antes de substituir os antigos.

### Exemplo: Atualizando o agente de infraestrutura do New Relic

Este exemplo mostra como atualizar o agente de infraestrutura para uma nova versão.

```shell
# Remote configuration for a single agent
chart_version: "5.0.109"  # Change this value
chart_values:
  newrelic-infrastructure:
    enableProcessMetrics: true
```

## Atualizando o Agent Control

### Kubernetes

As instruções a seguir se aplicam ao Agent Control em execução em um cluster Kubernetes.

O Agente Control é capaz de se atualizar remotamente. Isso garante que o supervisor em execução no seu cluster esteja sempre atualizado com os últimos recursos, melhorias de desempenho e patches de segurança.

O processo para atualizar o supervisor do Agent Control é o mesmo de qualquer outro agente gerenciado: você especifica a nova versão na configuração e implanta as alterações a partir do Fleet Control.

1. Edite a configuração do Agent Control: No Fleet Control, acesse a configuração do próprio Agent Control.
2. Atualizar o `chart_version`: Altere o `chart_version` para a nova versão desejada.
3. implantar as Alterações: Salve e implante a configuração.

Esse processo aciona uma atualização gradual dos pods do Agent Control no seu cluster. As verificações de integridade integradas do Kubernetes garantem que a nova versão esteja saudável antes que a antiga seja encerrada para uma atualização sem interrupções.

### Hosts Windows

O Agent Control no Windows é atualizado executando novamente o script de instalação fornecido na instalação guiada. O script baixa a versão mais recente e substitui o executável existente, preservando seus arquivos de configuração.

1. Abra o PowerShell com privilégios de administrador.

2. Execute o script de instalação da instalação guiada para a nova versão.

3. Verifique a atualização conferindo o status do serviço:

   ```powershell
   Get-Service -Name newrelic-agent-control
   Invoke-WebRequest -Uri "http://localhost:51200/status" -UseBasicParsing
   ```

Agentes gerenciados (Infrastructure Agent, NRDOT) são atualizados por meio do Fleet Control da mesma forma que em outras plataformas — atualize a versão desejada do agente na configuração da sua frota e implante as alterações.

### Iniciar, parar e reiniciar o Controle do Agente [#start-stop-restart]

O Agent Control inicia automaticamente após você executar o script de instalação.

#### Linux

Para Linux, certifique-se de usar o comando correto para o seu sistema de inicialização. Selecione `start`, `stop`, `restart` ou `status` conforme apropriado.

Aqui está um exemplo para `systemd`:

```shell
sudo systemctl <start|stop|restart|status> newrelic-agent-control
```

#### Windows

Execute estes comandos no PowerShell com privilégios de administrador:

```powershell
Start-Service -Name newrelic-agent-control
Stop-Service -Name newrelic-agent-control
Restart-Service -Name newrelic-agent-control
Get-Service -Name newrelic-agent-control
```
