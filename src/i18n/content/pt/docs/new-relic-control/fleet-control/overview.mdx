O Controle de Agentes fornece comando centralizado e remoto sobre frotas de cluster gerenciado para gerenciamento do ciclo de vida da instrumentação em escala. Use o Controle de Agentes para:

* **Centralize as operações do agente e reduza o trabalho manual:** instale, monitore, configure e atualize todos os agentes e integrações em um só lugar.
* **Personalize frotas para gerenciamento em massa:** agrupe agentes por necessidade de instrumentação e atualize facilmente versões de agentes para frotas inteiras com apenas alguns cliques. Não é necessário nenhum script.
* **Elimine lacunas de telemetria:** gerencie o agente no cluster do Kubernetes com um supervisor leve — o Agent Control — que pode ser dimensionado em todo o seu ambiente.

Para navegar até Controle de Agentes, vá para **Capabilities → New Relic Control → Fleets**. Você pode usar o pin para adicionar o New Relic Control à barra de navegação do lado esquerdo para facilitar o acesso.

## O que é uma frota?

No Controle de Agentes, uma frota é uma coleção homogênea de entidades gerenciadas (como cluster do Kubernetes) que compartilham configuração de instrumentação idêntica. As frotas servem como agrupamentos lógicos de entidades gerenciadas, permitindo gerenciamento simplificado e configuração de observabilidade consistente em escala.

Uma frota é composta por três componentes principais:

* **Implantação ativa:** O resumo da implantação mais recente

* **Configuração:** A configuração que está sendo aplicada ao seu cluster do Kubernetes

* **entidade:** Todos os clusters do Kubernetes dentro da frota

### Compreendendo a entidade gerenciada

No contexto do Controle de Agentes, uma entidade gerenciada refere-se a um cluster do Kubernetes executando a infraestrutura New Relic e instrumentação de logging gerenciada pelo Agent Control. A configuração do agente é especificada no nível da frota e cada cluster do Kubernetes na frota compartilha a mesma configuração que outros clusters naquela frota.

<Callout title="importante">
  Não é possível adicionar um cluster do Kubernetes a uma frota antes de instalar o Agent Control nesse cluster.
</Callout>

## Recurso chave

* **Gerenciamento centralizado:** oferece uma plataforma unificada para administrar agentes em execução em frotas de cluster do Kubernetes, permitindo que você gerencie a instrumentação em toda a sua infraestrutura em escala.
* **Agrupamento e propriedade:** funciona como um mecanismo de agrupamento e um limite de propriedade, permitindo que diferentes equipes gerenciem suas respectivas frotas de forma independente, mantendo práticas de observabilidade consistentes.
* **Configuração remota:** Suporta configuração remota de vários agentes. Para mais detalhes, veja [escopo](/docs/new-relic-control/getting-started/#scope).

Esses agentes são responsáveis por monitorar vários aspectos do seu ambiente e relatar pontos de dados importantes, incluindo métrica, eventos, logs e rastreamento (MELT).

## Pré-requisitos

Antes de configurar o Controle de Agentes, certifique-se de que seu sistema atende aos seguintes pré-requisitos:

* **Aceitar:** Como o Agent Control e o Controle de Agentes estão em versão prévia pública, você precisa aceitar para usar o produto.
* **Agent Control:** Certifique-se de ter instalado [o Agent Control](/docs/new-relic-control/agent-control/setup) no cluster do Kubernetes que você planeja adicionar às suas frotas.

## Permissões necessárias

**Permissões no escopo da organização:** A função Gerente de organização (referida como configuração Administrativa no produto) é necessária para criar e editar frotas no Controle de Agentes.
