<DNT>Fleet Control</DNT> fornece gerenciamento remoto e centralizado para seus agentes New Relic, ajudando a gerenciar todo o ciclo de vida da instrumentação em escala. Este recurso está disponível para o público geral (GA) para gerenciar agentes em clusters Kubernetes. O suporte para gerenciar agentes em hosts Linux e Windows está atualmente em visualização pública.

Com <DNT>Fleet Control</DNT>, você pode:

* **Centralize as operações do agente e reduza o trabalho manual:** instale, monitore, configure e atualize todos os agentes e integrações em um só lugar.
* **Personalize frotas para gerenciamento em massa:** agrupe agentes por necessidade de instrumentação e atualize facilmente versões de agentes para frotas inteiras com apenas alguns cliques. Não é necessário nenhum script.
* **Elimine lacunas de telemetria:** gerencie o agente no cluster do Kubernetes com um supervisor leve — o Agent Control — que pode ser dimensionado em todo o seu ambiente.

Para navegar até <DNT>Fleet Control</DNT>, vá para <DNT>**All capabilities → New Relic Control → Fleets**</DNT>. Você pode usar o alfinete para adicionar o New Relic Control à barra de navegação lateral esquerda para facilitar o acesso.

## Por que usar <DNT>Fleet Control</DNT>?

Gerenciar a instrumentação em um grande ambiente pode levar a desafios significativos. O <DNT>Fleet Control</DNT> foi projetado para resolver esses problemas, fornecendo um plano de controle centralizado para:

* **Evite agentes desatualizados:** mantenha o agente atualizado para evitar riscos de segurança, incompatibilidades e garantir que você possa aproveitar as últimas inovações New Relic.
* **Evite desvios de configuração:** elimine lacunas ocultas de observabilidade e problemas de conformidade garantindo configuração consistente em toda a sua entidade.
* **Corrija instrumentação quebrada:** identifique e resolva rapidamente problemas com sua instrumentação para manter a confiança em seus dados de observabilidade.
* **Reduza o esforço manual:** pare de depender de scripts únicos e atualizações manuais para gerenciar seu agente e integração.

## O que é uma frota?

Em <DNT>Fleet Control</DNT>, uma frota é uma coleção homogênea de entidades gerenciadas (como clusters Kubernetes ou hosts) que compartilham configurações de instrumentação idênticas. As Frotas servem como agrupamentos lógicos de entidades gerenciadas, permitindo um gerenciamento simplificado e uma configuração de observabilidade consistente em escala.

Uma frota é composta por três componentes principais:

* **Implantação ativa:** Um resumo do último conjunto de ações aplicadas à frota. Uma implantação pode incluir uma ou mais ações, como instalar, excluir ou atualizar a configuração de um agente, e reflete o resultado final dessas alterações.
* **Agentes e Configurações:** Os agentes e configurações que estão sendo instrumentados nas entidades gerenciadas na frota.
* **Entidade gerenciada:** todas as entidades gerenciadas atribuídas que estão associadas à frota.

### Compreendendo a entidade gerenciada

No contexto de <DNT>Fleet Control</DNT>, uma entidade gerenciada é um cluster Kubernetes, host Linux ou host Windows executando instrumentação da New Relic gerenciada por <DNT>Agent Control</DNT>. Cada frota é associada a um tipo específico de entidade gerenciada (Cluster Kubernetes, Host Linux ou Host Windows), e todas as entidades dentro de uma frota devem corresponder a esse tipo. A configuração do agente é especificada no nível da frota, e cada entidade gerenciada na frota compartilha a mesma configuração.

* **Entidade gerenciada:** Um cluster Kubernetes ou host que possui o Agent Control instalado. Isso significa que a entidade pode ser gerenciada pelo <DNT>Fleet Control</DNT>, mas não está necessariamente vinculada a uma frota ainda.
* **Entidade gerenciada atribuída:** Uma entidade gerenciada que foi associada a uma frota. Esta entidade agora é gerenciada ativamente por <DNT>Fleet Control</DNT>, e sua configuração é determinada pela frota à qual pertence.
* **Entidade gerenciada não atribuída:** Uma entidade gerenciada que não está atualmente vinculada a nenhuma frota. Embora tenha o Agent Control instalado e possa ser gerenciado, ele está operando atualmente usando sua própria configuração local.

<Callout title="Importante" variant="important">
  Não é possível adicionar um cluster do Kubernetes ou host a uma frota antes de instalar o Agente Control para torná-lo uma entidade gerenciada.
</Callout>

## Recurso chave

* **Gerenciamento centralizado:** oferece uma plataforma unificada para administrar agentes em execução em frotas de clusters do Kubernetes e hosts, permitindo que você gerencie a instrumentação em toda a sua infraestrutura em escala.
* **Visibilidade em tempo real:** gerencie a integridade da sua instrumentação com uma visão clara da integridade da frota, status do agente, progresso da implantação e erros de configuração.
* **Organização flexível de frota:** Crie frotas distintas para representar diferentes padrões de instrumentação, ambientes de aplicativos ou necessidades específicas da equipe. Essa estrutura oferece suporte tanto a equipes de observabilidade centralizadas que gerenciam todas as configurações quanto a modelos descentralizados nos quais equipes individuais possuem e gerenciam suas frotas específicas usando permissões refinadas.
* **Configuração remota:** Suporta configuração remota do agente e integração New Relic responsável por monitorar diversos aspectos do seu ambiente e reportar os principais pontos de dados, incluindo métricas, eventos, logs e trace (MELT).

## Pré-requisitos

Antes de configurar <DNT>Fleet Control</DNT>, certifique-se de que seu sistema atenda aos seguintes pré-requisitos:

* **Adesão à Pré-visualização Pública para hosts:** Para usar <DNT>Fleet Control</DNT> para gerenciar agentes em hosts Linux ou Windows, você deve aderir ao programa de pré-visualização pública para este recurso específico. O gerenciamento de agentes em clusters Kubernetes está em disponibilidade geral e não requer ativação.
* **Agent Control:** Você deve ter o [Agent Control](/docs/new-relic-control/agent-control/setup) instalado em qualquer cluster Kubernetes ou host que você planeja gerenciar. Este é um componente necessário para toda a funcionalidade do <DNT>Fleet Control</DNT>.

## Permissões necessárias

**Permissões no escopo da organização:** A função de **Organization Manager** (referida como uma configuração administrativa no produto) é necessária para criar e editar frotas no <DNT>Fleet Control</DNT>. Alternativamente, você pode [criar uma função personalizada](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#create-role) com permissões de gerenciamento de frota.
