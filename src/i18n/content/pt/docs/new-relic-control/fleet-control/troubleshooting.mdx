Este documento descreve as etapas para solucionar problemas comuns durante a instalação ou operação do <DNT>Fleet Control</DNT>.

## Solução de problemas de implantações com eventos de implantação

<DNT>Fleet Control</DNT> fornece uma aba <DNT>deployment events</DNT> que oferece visibilidade granular de cada evento que ocorre durante uma implantação. Esta é a principal ferramenta para solucionar problemas de implantação e entender as causas raiz.

### Acessar Eventos de Implantação

Há duas maneiras de acessar a guia Eventos de Implantação:

**Do histórico de implantação:**

1. Navegue até a aba <DNT>deployments</DNT> em uma frota específica.
2. Clique na linha da implantação que deseja investigar.
3. Na visualização de resumo da implantação, clique na aba <DNT>Deployment Events</DNT>.

**Após criar uma nova implantação:**

1. Após iniciar uma implantação, a visualização de progresso será concluída e transicionará para a visualização de resumo da implantação.
2. Nesta visualização de resumo, clique na aba <DNT>Deployment Events</DNT> para ver imediatamente eventos para solução de problemas ou verificação.

### Entendendo a tabela de Eventos de Implantação

A tabela <DNT>deployment events</DNT> exibe dados atômicos em nível de evento para cada ação realizada durante uma implantação. A tabela inclui as seguintes colunas:

* <DNT>**Timestamp:**</DNT> Quando o evento ocorreu.
* <DNT>**Status:**</DNT> O status do evento (por exemplo, <DNT>Completed, Config Applied, Applying</DNT>).
* <DNT>**Managed entity name:**</DNT> O nome do cluster ou host onde o evento ocorreu.
* <DNT>**Agent name:**</DNT> O agente específico envolvido no evento.
* <DNT>**Agent type:**</DNT> O tipo de agente (por exemplo, <DNT>New Relic Infrastructure, Fluent Bit</DNT>).
* <DNT>**Event:**</DNT> O tipo de evento que ocorreu.
* <DNT>**Details:**</DNT> Informações adicionais sobre o evento, muitas vezes incluindo mensagens de erro para eventos com falha.
* <DNT>**Duration:**</DNT> Quanto tempo o evento levou para ser concluído.

### Filtragem e pesquisa de eventos

A aba <DNT>deployment events</DNT> oferece várias maneiras de filtrar eventos:

* <DNT>**Search:**</DNT> Use a barra de pesquisa para encontrar eventos específicos por palavra-chave.
* <DNT>**Filter by entities:**</DNT> Filtre para mostrar eventos de uma entidade gerenciada específica. Isso é especialmente útil quando você tem uma grande frota (por exemplo, 100 clusters) e apenas uma ou algumas entidades falharam.
* <DNT>**Filter by status:**</DNT> Filtre eventos pelo status (por exemplo, <DNT>Completed, Config Applied, Applying</DNT>) para focar em eventos bem-sucedidos ou problemáticos.

### Solução de problemas avançada com NRQL

Para uma solução de problemas mais avançada, você pode acessar a consulta NRQL subjacente que preenche a tabela Eventos de implantação:

1. Na aba <DNT>Deployment Events</DNT>, localize o botão à direita do cabeçalho da tabela de eventos.
2. Clique neste botão para abrir uma aba de consulta NRQL no <DNT>Data Explorer</DNT>.
3. A consulta será pré-preenchida com o ID da implantação e exibirá colunas adicionais com informações mais detalhadas, incluindo vários IDs e metadados.
4. Você pode modificar esta consulta, se necessário, para investigar o problema mais a fundo.
5. Use os recursos padrão do <DNT>Data Explorer&apos;s</DNT>, como alternar entre as visualizações em formato de tabela e JSON.
