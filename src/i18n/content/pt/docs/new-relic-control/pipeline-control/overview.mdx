O Controle de Pipeline é o componente de gerenciamento de dados do New Relic Control. Ele pode ajudar a filtrar e descartar dados de baixo valor antes que eles sejam inseridos na plataforma New Relic.

O Controle de Pipeline consiste em dois subcomponentes:Pipeline o gateway Controle de (simplesmente chamado de gateway) e as Pipeline cloud regras Controle de .

O gateway foi projetado para filtrar dados ou atributos antes que eles saiam da sua rede. As cloud regras realizam esse processamento na New Relic cloud do . Em ambos os cenários, o Controle de Pipeline usa uma sintaxe de consulta NRQL idêntica para processar dados.

Para usar o gateway, você precisa primeiro configurá-lo em seu ambiente. As regras cloud não envolvem configuração.

Para entender os custos associados ao uso do Controle de Pipeline, consulte [Custos do Controle de Pipeline ](/docs/new-relic-control/pipeline-control/costs).

## Pré-requisitos

Antes de começar, você deve ter:

* Uma conta New Relic com permissões de administrador de produto da organização e gerente da organização

<Callout variant="important" title="importante">
  O administrador de produto da organização e o gerente de organização do perfil do usuário estão obrigados a:

  * Instalar gateway
  * Adicionar cluster a um gateway
  * Atualizar gateway (por exemplo, novas regras e alterações de configurações)
  * implantar uma configuração de gateway
  * Ver regras do gateway
</Callout>

* Familiaridade com operações e gerenciamento do Kubernetes
* cluster do Kubernetes acesso de administrador para implantar e gerenciar recursos dentro do seu cluster
* Permissões para criar e gerenciar nomes DNS privados para hospedar em sua rede
* Permissões para criar e gerenciar registros DNS
* Helm instalado localmente
* Compreensão do NRQL para criar e gerenciar regras de pipeline

<Callout variant="important">
  * As regras de Controle de Pipeline só se aplicam aos dados que chegam a partir do momento em que você cria a regra; elas não excluem dados que [já foram ingeridos](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-data-retention#data-deletion).
  * Ao criar regras, você é responsável por garantir que elas identifiquem e descartem com precisão os dados que atendem às condições que você estabeleceu. Você também é responsável por monitorar a regra, bem como os dados que você divulga à New Relic.
</Callout>
