O Pipeline Control é o componente de gerenciamento de dados do New Relic Control. Ajuda você a filtrar, transformar e gerenciar dados de telemetria para reduzir custos, melhorar a qualidade do sinal e manter a conformidade.

O Pipeline Control oferece duas camadas de processamento que você pode usar independentemente ou em conjunto:

## Gateway de controle de pipeline

**Regras de Gateway** são executadas na sua infraestrutura:

* Processe dados **antes que saiam da sua rede** (reduz custos de saída)
* Configuração em toda a organização (as regras se aplicam a todas as contas na sua organização)
* Capacidades completas de transformação, filtragem e amostragem
* Você gerencia a infraestrutura (requer implantação de cluster Kubernetes)
* Cobrado com base no volume de dados recebido no gateway

O gateway é ideal quando você precisa:

* Para reduzir os custos de saída de rede filtrando dados antes que saiam da sua infraestrutura
* Capacidades de transformação de dados mais ricas para todos os tipos de MELT (p. ex. analisar logs, normalizar atributos, enriquecer com metadados)
* Para descartar ou ofuscar dados sensíveis antes que saiam da sua rede para requisitos de conformidade
* Para processar maiores volumes de dados, onde o gateway é mais barato que as regras de nuvem

Saiba mais sobre o gateway na [Visão geral do gateway](/docs/new-relic-control/pipeline-control/gateway/overview).

## Regras de nuvem do Pipeline Control

**Regras de nuvem** são executadas na infraestrutura da New Relic após os dados chegarem aos endpoints da New Relic:

* Processe dados **após chegarem ao New Relic** (sem necessidade de gerenciamento de infraestrutura)
* Regras específicas da conta (cada regra se aplica a uma única conta New Relic)
* Regras de descarte baseadas em NRQL para filtrar dados e atributos
* Nenhuma configuração ou gerenciamento de infraestrutura necessários
* Cobrado com base em dados descartados (7,5 GB por aCCU) mais bilhões de eventos escaneados (2,237 BSE por aCCU)

As regras de nuvem são melhores quando:

* Você quer regras de descarte simples sem gerenciar a infraestrutura de gateway
* Seus requisitos de processamento são modestos e não justificam uma implantação de gateway em grande escala
* Filtragem de menor volume como uma % da sua ingestão total, onde a precificação de regras de nuvem pode ser mais econômica

Saiba mais: [Criar regras de nuvem](/docs/new-relic-control/pipeline-control/cloud-rules/create-pipeline-rules)

## Usando ambos em conjunto

Você também pode usar regras de gateway e nuvem simultaneamente:

* Fontes de telemetria (por exemplo, agentes) podem ser configurados de forma diferente para rotear através do seu gateway ou não
* A estratégia de defesa em profundidade garante que dados sensíveis sejam absolutamente descartados
* Conveniente para gerenciar algumas regras via API/UI na nuvem enquanto utiliza o gateway para processamento primário

## Tabela de comparação

| Recurso                       | Regras do Gateway                                                                      | Regras de Cloud                                                               |
| ----------------------------- | -------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| **Local de processamento**    | Sua infraestrutura (Kubernetes)                                                        | Nuvem da New Relic                                                            |
| **Quando processa**           | Antes de os dados saírem da sua rede                                                   | Depois que os dados chegam ao New Relic                                       |
| **Configuração necessária**   | Sim (cluster Kubernetes, Helm chart)                                                   | Não                                                                           |
| **Método de configuração**    | YAML ou UI                                                                             | UI ou API NerdGraph (baseada em NRQL)                                         |
| **Escopo da regra**           | Em toda a organização                                                                  | Específico da conta                                                           |
| **Capacidades**               | Transformar, descartar dados/atributos, amostragem por taxa                            | Descartar dados/atributos                                                     |
| **Tipos de dados suportados** | Métricas, Eventos, Logs, Traces                                                        | Métricas, Eventos, Logs, Traces                                               |
| **Melhor para**               | Processamento de alto volume, transformações, redução de custos de saída, conformidade | Regras de descarte simples, baixo volume, sem gerenciamento de infraestrutura |

Para informações detalhadas sobre preços e cálculos de custos, consulte [Custos do Pipeline Control](/docs/new-relic-control/pipeline-control/costs).

## Pré-requisitos

### Para regras de nuvem

* Uma conta da New Relic em uma função com permissões RBAC para gerenciar regras de nuvem do Pipeline Control
* Entendimento de NRQL para criar regras de filtragem

### Para gateway

* Uma conta da New Relic com permissões de Admin de produtos da organização ou Gerente da organização

<Callout variant="important" title="importante">
  As funções de usuário Administrador de Produtos da Organização ou Gerente da Organização são necessárias para:

  * Instalar gateway
  * Adicionar cluster a um gateway
  * implantar uma configuração de gateway
  * Ver regras do gateway
</Callout>

* Familiaridade com operações e gerenciamento do Kubernetes
* cluster do Kubernetes acesso de administrador para implantar e gerenciar recursos dentro do seu cluster
* Permissões para criar e gerenciar nomes DNS privados para hospedar em sua rede
* Permissões para criar e gerenciar registros DNS
* Helm instalado localmente

<Callout variant="important">
  * As regras de Controle de Pipeline só se aplicam aos dados que chegam a partir do momento em que você cria a regra; elas não excluem dados que [já foram ingeridos](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-data-retention#data-deletion).
  * Ao criar regras, você é responsável por garantir que elas identifiquem e descartem com precisão os dados que atendem às condições que você estabeleceu. Você também é responsável por monitorar a regra, bem como os dados que você divulga à New Relic.
</Callout>
