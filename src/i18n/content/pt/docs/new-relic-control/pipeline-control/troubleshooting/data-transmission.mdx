/\* Se você estiver enfrentando problemas com dados não recebidos no New Relic, este guia ajudará você a diagnosticar e resolver problemas comuns. Siga os passos abaixo com base no problema específico que você encontrar: \*/

## Problema: Problemas de resolução de DNS entre o produtor de telemetria e o gateway

O agente não consegue se conectar ao gateway devido a erros de resolução de DNS.

## Solução

* Acesse os logs do agente (via interface do usuário ou diretamente) e pesquise erros de conexão.
* Consulte seu administrador de rede para ajustar a configuração de DNS com base em sua infraestrutura e topologia de rede. Consulte [a documentação do agente](/docs/apm/new-relic-apm/getting-started/introduction-apm/) para obter mais detalhes.

## Problema: Problemas com o certificado SSL do gateway do agente

Há um problema com a configuração do certificado SSL entre o agente e o gateway.

## Solução

* Verifique os logs do agente em busca de erros de conexão SSL.
* Certifique-se de que o certificado SSL esteja configurado corretamente e válido, considerando sua infraestrutura e topologia de rede. Consulte [a documentação do agente](/docs/apm/new-relic-apm/getting-started/introduction-apm/) para obter mais detalhes.

## Problema: Configuração incorreta do gateway

Os dados chegam ao gateway, mas não são publicados no New Relic.

## Solução

* Verifique a solicitação de saída e o erro métrico.
* Revisar regra métrica com falha.
* Inspecione os logs do pod que não estão iniciando.
* Corrija as configurações do gateway e garanta que todos os pods estejam operacionais.

## Problema: Produtor/protocolo de telemetria não suportado

Os dados são enviados de uma API ou protocolo não suportado.

## Solução

* Verifique a compatibilidade com o New Relic. O cliente de API incorreto retornará um código de status `501` .
* Envie uma solicitação de recurso para suporte.
* Configure o produtor de telemetria para enviar dados diretamente para o New Relic.

## Problema: Regra destrutiva

Uma regra está descartando todos os dados, impedindo que eles cheguem ao New Relic.

## Solução

* Verifique os dados de queda métrica.
* Modifique ou remova a regra destrutiva para permitir o fluxo de dados.

## Problema: Dados ausentes

Dados estão faltando no backend do New Relic após a ingestão.

## Solução

* Revise a métrica de erro e verifique se há tempos limite no lado do cliente.
* Avalie os sinais de esgotamento de recursos.
* Verifique [o status do New Relic](https://status.newrelic.com/).
* Examine os logs do gateway durante o tempo afetado
