/\* Se você implantou uma regra ou alteração de configuração no seu gateway e ela não está surtindo o efeito esperado, este guia ajudará você a diagnosticar e resolver problemas comuns. Siga os passos abaixo com base no problema específico que você encontrar: \*/

## Problema: Regra NRQL inválida

A regra implantar contém sintaxe NRQL inválida.

## Solução

* Procure por eventos de log de regras com falha e verifique os logs do pod do gateway.
* Valide o NRQL no criador de consulta usando `SELECT` em vez de `DELETE`, edite a regra e reimplantar.
* Consulte a documentação do NRQL para obter orientação.

## Problema: Configuração errada de implantação do mapa

Um `ConfigMap` contendo o aplicativo de configuração foi atualizado com um erro.

## Solução

* Verifique se o novo pod de implantação falhou ao reiniciar.
* Reverta para a configuração de trabalho anterior.

## Problema: O trabalho de implantação do K8s não pode ser concluído

Não é possível agendar novos pods com as últimas alterações de configuração devido ao esgotamento de recursos.

## Solução

* Veja a página de resumo do gateway para cluster não íntegro. Use a interface Kubernetes para visualizar eventos pod e trabalhos pendentes.
* Aumente o tamanho do pool de nós ou verifique as requests de limite de recursos (CPU e memória) para o pod.
* Verifique os limites do provedor cloud para o número ativo de nós.

## Problema: incompatibilidade de esquema de dados

A regra é escrita em dados no NRDB com um esquema diferente do que no gateway.

## Solução

* Siga as instruções da documentação específicas para cada tipo de MELT para garantir a compatibilidade do esquema.
