/\* Se você estiver recebendo dados de telemetria de produtores enviando para o gateway, mas nenhum dado de monitoramento do próprio gateway, este guia ajudará você a diagnosticar e resolver problemas comuns. Siga os passos abaixo com base no problema específico que você encontrar: \*/

## Problema: Chave de licença inválida

O gateway está configurado com uma chave de licença inválida ou que foi rotacionada.

## Solução

* Acesse os logs do gateway diretamente para verificar o problema. Procure por erros 403 no pipeline de monitoramento interno e no exportador de uso, mas não por dados do agente.
* Certifique-se de que a chave de licença seja válida e configurada corretamente. Atualize a chave se necessário.

## Problema: Regra de nuvem afetando dados de monitoramento de gateway

métrica produzida pelo gateway está sendo descartada involuntariamente por uma regra cloud .

## Solução

* Revise os dados de uso das regras cloud para identificar quaisquer quedas não intencionais.
* Modifique a configuração da regra cloud para impedir que a métrica do gateway seja descartada.

## Problema: Estou com limitação de taxa na APImétrica

Você pode ter excedido o limite de solicitações em relação à API métrica, fazendo com que requests subsequentes do gateway falhem com códigos de resposta 429.

## Solução

* Verifique se há eventos de limitação de taxa relacionados à OpenTelemetry métrica API em sua conta.
* Consulte a documentação de limitação de taxa para obter orientações sobre como gerenciar e ajustar taxas de solicitação.
