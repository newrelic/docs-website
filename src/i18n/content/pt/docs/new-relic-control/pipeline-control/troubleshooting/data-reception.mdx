/\* Se você não estiver recebendo dados de telemetria de produtores configurados para enviar ao gateway, mas o gateway estiver produzindo seus próprios dados de monitoramento, este guia ajudará você a diagnosticar e resolver problemas comuns. Siga os passos abaixo com base no problema específico que você encontrar: \*/

## Problema: Configuração incorreta do produtor de telemetria

O produtor de telemetria está configurado incorretamente, resultando em nenhum dado de telemetria sendo enviado ao gateway.

## Solução

* Acesse os logs do produtor para identificar erros de configuração.
* Consulte a documentação apropriada da API, do agente ou do produtor de telemetria para obter as etapas de configuração corretas.

## Problema: Estou com limitação de taxa

Você está atingindo limites de taxa em seus dados de telemetria.

## Solução

* Verifique os códigos de resposta do cliente HTTP do gateway para códigos de status 429 recebidos da API do New Relic.
* Revise o evento criado na sua conta indicando limitação de taxa.
* Consulte a [documentação de limitação de taxa](https://docs.newrelic.com/search-results/?query=rate%20limiting&page=2) para obter orientações sobre como gerenciar e ajustar taxas de dados de telemetria.
