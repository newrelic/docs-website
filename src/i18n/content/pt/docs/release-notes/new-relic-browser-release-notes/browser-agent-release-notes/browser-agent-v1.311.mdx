## v1.311.0

### Recurso

#### remoção do agentIdentifier

Removidas as referências com escopo por identificador de agente para simplificar a arquitetura do agente. Todas as referências agora são armazenadas diretamente na referência do agente, eliminando a necessidade de mesclar referências do agente com objetos de escopo via chaves de pesquisa.

#### Atribuir a URL da página original a todos os nós PageViewTiming

Atribuição de `pageUrl` expandida para todas as temporizações, com base no trabalho anterior para a LCP. Os eventos `PageViewTiming` agora reportam a URL original de navegação hard para todos os tipos de nó, garantindo a consistência dos dados para SPA mesmo que a URL mude antes da coleta.

#### melhorar a lógica de temporização do MFE

Aprimorada a capacidade do agente de identificar o script MFE correto que chama a API de registro. Essa melhoria resulta em informações de tempo mais precisas para ambientes MFE.

### Correções de bugs

#### Exigir IDs de string para novos registros de MFE

Resolvido um problema em que IDs não string causavam erros durante a síntese de entidade MFE. O agente agora impõe IDs baseados em string e emite um aviso se um tipo de dados diferente for fornecido.

#### O substituidor circular removeu objetos reutilizados das coletas

Corrigido um problema em que o método interno de stringify removia ponteiros compartilhados entre payloads de coleta, o que ocasionalmente causava a ausência de atributos `params` nas coletas de erro.

#### Atualizar RegExp para evitar problema de build do webpack não minificado

O caractere `$` foi substituído pelo seu equivalente hexadecimal em expressões regulares. Isso evita que o Webpack substitua incorretamente o caractere durante o processo de build, o que anteriormente causava falhas no agente em builds não minificados.

## Declaração de apoio

New Relic recomenda que você atualize o agente regularmente para garantir que esteja obtendo os benefícios mais recentes de recursos e desempenho. Versões mais antigas não terão mais suporte quando chegarem [ao fim de sua vida útil](https://docs.newrelic.com/docs/browser/browser-monitoring/getting-started/browser-agent-eol-policy/). As datas de lançamento refletem a data de publicação original da versão do agente.

Novos lançamentos de agente do browser são disponibilizados aos clientes em pequenas etapas ao longo de um período de tempo. Por esse motivo, a data em que o lançamento fica acessível à sua conta pode não corresponder à data de publicação original. Consulte este [dashboard de status](https://newrelic.github.io/newrelic-browser-agent-release/) para obter mais informações.

Em conformidade com nossa [política de suporte ao navegador](https://docs.newrelic.com/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring/#browser-types), a v1.311.0 do agente do Browser foi desenvolvida e testada para estes navegadores e faixas de versão: Chrome 134-144, Edge 134-144, Safari 18-26 e Firefox 137-147. Para dispositivos móveis, a v1.311.0 foi criada e testada para Android OS 16 e iOS Safari 18-26.
