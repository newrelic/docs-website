## v1.313.0

### Recurso

#### Detectar automaticamente erros do MFE

Adicionar suporte para atribuição automática de Micro Frontend (MFE) de eventos `JavaScriptError`. Anteriormente, os eventos `JavaScriptError` eram capturados apenas globalmente pelo aplicativo de navegador do contêiner. Com essa alteração, o agente compara o stack trace da instância de Error lançada com a assinatura da pilha de chamadas do módulo MFE que se registrou na API `register`. Se o arquivo responsável pelo registro no agente do browser for encontrado no stack trace da instância de Error lançada, esse evento será atribuído ao MFE em vez do contêiner.

#### remover &quot;withCredentials&quot;

Remova todo o uso de `withCredentials` ou `credentials: 'include'` das coletas do agente, pois os cookies não são usados no agente desde antes da [v1220](https://github.com/newrelic/newrelic-browser-agent/releases/tag/v1220).

#### Melhorar as heurísticas de MicroFrontEndTiming

Melhorar a capacidade de detectar tempos de execução de script para MFE, resultando em métricas `timeToLoad` mais precisas.

#### Reportar violações de CSP como novos eventos

As violações de Content Security Policy (CSP) que aparecem agora são capturadas e reportadas como um novo tipo de evento `SecurityPolicyViolation` ao consumidor do Page insights. Eles se tornam consultáveis através do NRQL.

#### Aplicar requisitos legais e de conformidade à API de registro

Certifique-se de que a API de registro atenda aos requisitos da empresa antes de expô-la para visualização limitada.

### Correções de bugs

#### `trace.lastTimestamp` ruim no Safari + caso extremo de navegação de retorno

Corrige uma condição de corrida em que a navegação de voltar do navegador Safari (BFCache) poderia fazer com que um timestamp absoluto fosse registrado no início do recurso de trace da sessão, resultando em um valor `trace.lastTimestamp` anormal e uma mistura de timestamp relativo e absoluto em uma única carga.

## Declaração de apoio

New Relic recomenda que você atualize o agente regularmente para garantir que esteja obtendo os benefícios mais recentes de recursos e desempenho. Versões mais antigas não terão mais suporte quando chegarem [ao fim de sua vida útil](https://docs.newrelic.com/docs/browser/browser-monitoring/getting-started/browser-agent-eol-policy/). As datas de lançamento refletem a data de publicação original da versão do agente.

Novos lançamentos de agente do browser são disponibilizados aos clientes em pequenas etapas ao longo de um período de tempo. Por esse motivo, a data em que o lançamento fica acessível à sua conta pode não corresponder à data de publicação original. Consulte este [dashboard de status](https://newrelic.github.io/newrelic-browser-agent-release/) para obter mais informações.

De acordo com a nossa [política de suporte a navegadores](https://docs.newrelic.com/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring/#browser-types), a v1.313.0 do agente do Browser foi desenvolvida para e testada nestes navegadores e intervalos de versões: Chrome 136–146, Edge 136–146, Safari 18–26 e Firefox 138–148. Para dispositivos móveis, a v1.313.0 foi compilada e testada para Android OS 16 e iOS Safari 18-26.2.
