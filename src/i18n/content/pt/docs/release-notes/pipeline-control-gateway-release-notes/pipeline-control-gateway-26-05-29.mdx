## Modo somente configuração (sem Flux)

* Agora você pode instalar e operar o gateway do Controle de Pipeline sem a necessidade de controladores Flux no seu cluster. Neste modo, o New Relic continua a gerenciar a configuração do seu pipeline (amostragem, filtragem, transformações) automaticamente pela interface, enquanto você gerencia a infraestrutura do gateway (escalonamento, versões) manualmente via Helm.
* Isso reduz o escopo de permissões de cluster-admin apenas para acesso a ConfigMap com escopo de namespace, tornando-o adequado para ambientes com restrições de segurança e orientados à conformidade.
* Um sidecar config-watcher integrado detecta automaticamente as alterações de configuração enviadas pela interface e reinicia os pods do gateway — não são necessários operadores externos (como o Stakater Reloader).
* Para obter instruções de instalação, consulte [Instalar o gateway sem o Flux](/docs/new-relic-control/pipeline-control/gateway/install-gateway-without-flux/).

## Modo de implantação de DaemonSet

* Adicionado suporte para implantar o gateway como um DaemonSet do Kubernetes (um pod por nó) em vez de uma implantação com HPA. Isso é útil para ambientes que exigem processamento de telemetria no nível do nó ou alocação consistente de recursos por nó.
* Habilite o modo DaemonSet definindo `daemonset.enabled: true` nos seus valores do Helm. O modo DaemonSet também suporta o sidecar config-watcher e `customConfigMap` para instalações apenas de configuração.
* Os modos DaemonSet e implantação são mutuamente exclusivos — o chart usa guardas condicionais para garantir que apenas um modo esteja ativo por vez.
