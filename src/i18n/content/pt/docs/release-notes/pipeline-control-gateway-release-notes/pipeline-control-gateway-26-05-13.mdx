## Suporte a múltiplas frotas

* Agora você pode criar e gerenciar várias frotas de gateway em uma única organização. Isso permite que unidades de negócios ou ambientes separados (por exemplo, prod vs. preparação) mantenham regras independentes de amostragem, filtragem e transformação.
* Navegue facilmente entre diferentes configurações usando o novo dropdown seletor de frota na interface do Controle de Pipeline.

### Esquema YAML atualizado

* Migrado para uma configuração simplificada que remove o aninhamento específico de sinal (por exemplo, `logs:`, `spans:`) em favor de uma matriz direta de regras, tornando a configuração mais plana e fácil de ler.
* Para processadores de filtro, o novo schema suporta um campo `context` explícito. Isso permite um direcionamento mais preciso de subtipos de dados, como **pontos de dados de métrica** e **eventos de span**.

### UX de implantação aprimorada

* A nova página de implantação agora preenche automaticamente os valores padrão para o nome e a descrição da implantação, reduzindo significativamente as etapas manuais necessárias para enviar alterações.
* Adicionada uma pré-visualização lado a lado do diff de configuração para processadores de transformação. Isso permite que você compare sua nova lógica YAML com a versão atual antes de finalizar uma implantação.
