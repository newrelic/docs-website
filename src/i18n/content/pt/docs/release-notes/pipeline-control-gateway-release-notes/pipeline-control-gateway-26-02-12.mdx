## Notas de versão do gateway do Pipeline Control - v2.0.0

### Atualizado para o OpenTelemetry Collector v0.139.0

Esta versão é baseada no OpenTelemetry Collector v0.139.0, trazendo as mais recentes melhorias de estabilidade e recursos do projeto upstream. Esta atualização habilita novas capacidades de processamento de dados não disponíveis em versões anteriores do gateway.

### Novos recursos do processador

Adicionados três novos processadores que lhe dão controle sobre seus dados de telemetria antes que saiam da sua infraestrutura:

* **Processador de amostragem**: Reduza o volume de dados com regras de amostragem probabilísticas e condicionais
* **Processador de filtro**: Descarte registros inteiros ou atributos específicos com base em condições booleanas OTTL
* **Processador Transform**: Adicione, modifique ou exclua atributos usando a Linguagem de Transformação do OpenTelemetry (OTTL)

Esses processadores podem ser encadeados para criar pipelines sofisticados de processamento de dados para métricas, eventos, logs e traces.

### Opções de configuração aprimoradas

* **UI redesenhada**: Nova interface baseada em formulários para criar e gerenciar regras de processador sem escrever YAML
* **Configuração YAML aprimorada**: Suporte para configuração de processador baseada em OTTL

### Estrutura de documentação melhorada

* Guias abrangentes para cada tipo de processador com exemplos de OTTL e casos de uso
* Distinção clara entre regras de gateway (na sua infraestrutura) e regras de nuvem (na infraestrutura da New Relic)
* Recursos expandidos de solução de problemas para instalação do gateway, monitoramento de integridade e problemas de fluxo de dados
* Arquitetura da informação reorganizada para facilitar a navegação
