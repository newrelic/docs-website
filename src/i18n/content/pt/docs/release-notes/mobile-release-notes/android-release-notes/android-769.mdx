## Correções de bugs

* Foi resolvido um problema de vazamento de memória em que os ouvintes trace não estavam sendo removidos corretamente após o aplicativo alternar entre primeiro e segundo plano várias vezes.
* Corrigido o problema com a sincronização de threads que ocasionalmente fazia com que os dados não fossem reportados ao New Relic.
* Foram resolvidos problemas relacionados aos relatórios de logs durante ANRs para melhorar a estabilidade e o desempenho.
* Corrigidos problemas de classes duplicadas relacionadas à biblioteca Curtains para garantir compatibilidade e evitar conflitos.
* Foi resolvido um problema de exceção de ponteiro nulo no método setUserId para aumentar a robustez.
* Problemas corrigidos com o Retrofit Interceptor em que os URLs não eram relatados corretamente se a solicitação falhasse devido a erros de rede no dispositivo.

## Declaração de apoio

Recomendamos atualizar o agente pelo menos a cada três meses. Encontre políticas e datas específicas para suporte ao agente Android na [Política de EOL de monitoramento de agente móvel](/docs/mobile-monitoring/new-relic-mobile/get-started/mobile-agents-eol-policy/#android-eol).
