## Novos recursos e melhorias

* Novos tipos de evento introduzidos

  * **VideoAction**: Este tipo de evento representa ações padrão relacionadas à reprodução e interação de vídeo.
  * **VideoErrorAction**: Este tipo de evento captura erros que ocorrem durante a reprodução de vídeo, fornecendo insights detalhados para resolução de problemas.
  * **VideoAdAction**: tipo de evento dedicado para rastrear ações relacionadas a anúncios em vídeo, garantindo melhor granularidade e relatórios.
  * **VideoCustomAction**: Um tipo de evento flexível para ações personalizadas e interação dentro do aplicativo de vídeo.

## Depreciações

* `PageAction`
  * O tipo de evento `PageAction` foi descontinuado. O usuário deve fazer a transição para o uso dos tipos de eventos mais recentes que oferecem recursos de rastreamento aprimorados e especificidade.

## Novo atributo introduzido

* Introduzimos um conjunto de novos atributos para aprimorar o rastreamento e a análise de dados. Esses atributos fornecem contexto e detalhes adicionais para os novos tipos de eventos; consulte a [documentação do modelo de dados](https://github.com/newrelic/video-html5-js/blob/v3.0.0/DATAMODEL.md) para mais detalhes.

## Impacto e ações necessárias

* **Transição de PageAction**: os usuários que atualmente rastreiam `PageAction` eventos devem adaptar suas implementações para utilizar os novos tipos de eventos introduzidos. Essa mudança proporciona maior especificidade e escopo no rastreamento de ações em interações de vídeo.

* **Atualizar implementações**: Os integradores devem atualizar seus sistemas para aproveitar os novos atributos introduzidos com o lançamento, que fornecem contexto e detalhes mais ricos para o rastreamento de eventos.

* **Testes e validação**: Recomendamos testar a implementação de novos atributos e tipos de evento para garantir a compatibilidade e a transmissão correta de dados.
