## Procedimentos de implementação

### Instalação e configuração do Service Pack

* **Onde instalar:** Implante este service pack no mandante 000 do seu Central Monitoring System.
* **Local de configuração:** Execute todas as tarefas de configuração subsequentes no cliente de negócios do seu Central Monitoring System.
* **Instruções completas:** Para procedimentos de instalação passo a passo, consulte o **Guia de Instalação V3.0** (Apêndice VIII: Instalar Service Pack) ou o **Guia do Administrador V3.0** (Seção: Instalar Service Pack).

## Atualizações neste patch

O SP12 é uma coleção de Notas da New Relic abordando correções de bugs e novos recursos:

### NR 1344

Novo parâmetro `AIF_TABLES` para `NR_SAP_AIF_ERROR` permite recuperar mensagens de erro de tabelas diferentes de `/AIF/STD_IDX_TBL`.

### NR 12-02

Corrige problemas e adiciona novos recursos para provedores de dados baseados em API:

* Corrigido o dump de código em `/NEWR/API_DATA_COLLECTOR` quando o destino da API não está acessível
* Corrigida falha de recurso ao extrair dados do CALM Landscape
* Corrigida falha de recurso ao extrair o BTP Message Processing Log
* Adicionada validação para o ID de Serviço (LMSID) configurado como CALL ID para todos os provedores de dados no grupo de monitoramento CALM
* Adicionada verificação de validade do certificado antes de chamar a API de Coleta de Dados

### NR 12-03

Implementa atraso na coleta e janela fixa de coleta de dados para o gerenciador de coleta e testes por ID de chamada no registro.

### NR 12-05

Implementa as seguintes melhorias:

* Suporte à paginação para provedores de dados no grupo de monitoramento CALM
* Novo parâmetro de programa para permitir que o programa de verificação de RFC verifique um único destino RFC
* Configuração para forçar o teste do provedor de dados apenas para um único ID de chamada

Corrige os seguintes problemas:

* ID de chamada em branco na saída do gerenciador de coleta
* Problema de validação de credencial para `NR_CF_AUDIT_EVENTS`

### NR 12-06

Novo provedor de dados `NR_SAP_ABAP_WS_COMPLETE` coleta eventos de web service ABAP.

### Correções de bugs adicionais

Além das notas acima, o SP12 também corrige os seguintes problemas:

* Corrigido problema de timestamp com módulos de função do provedor de dados de recuperação de tabelas `/NEWR/RFC_EVENTS_COLLECTOR` e `/NEWR/RFC_EVENTS_COLLECTOR2`
* Adicionado o ID da transação aos eventos de IDoc para facilitar a vinculação aos registros de rastreamento

## Atualizações manuais de configuração

### Adicionar novo parâmetro ao NR\_SAP\_AIF\_ERROR

Para configurar o novo parâmetro `AIF_TABLES`:

1. Aplique o transporte incluído ao Central Monitoring System.
2. Vá para a transação `/n/NEWR/REGISTRY`.
3. Selecione o provedor de dados `NR_SAP_AIF_ERROR` e clique em **Configure**.
4. Adicione o parâmetro `AIF_TABLES` e insira o nome da tabela especificado em `/AIF/T_IDX_TBL` para a combinação de namespace/interface/versão configurada.

<Callout variant="tip">
  Se `AIF_TABLES` for deixado em branco ou não for especificado, a tabela de índice assume como padrão a tabela de índice único padrão `/AIF/STD_IDX_TBL`.
</Callout>

### Configure o atraso de coleta e a janela de coleta fixa

Dois novos campos de configuração foram adicionados ao Registro de Provedores de Dados para resolver lacunas de dados causadas por limites de coleta flutuantes e latência do banco de dados.

**Novos campos no registro do provedor de dados:**

* **DATA\_COLLECTION\_DELAY:** Especifica um atraso intencional em minutos para subtrair dos carimbos de data/hora de coleta. Este buffer garante que dados de atualizações do banco de dados que chegam com atraso sejam capturados antes que a janela de coleta feche.
* **FIXED\_SCHEDULES:** Flag para habilitar períodos de coleta fixos. Quando definido como `X`, os segundos são removidos dos timestamps de coleta para criar janelas de coleta consistentes e previsíveis.

**Como funciona:**

**Horários Fixos** (`FIXED_SCHEDULES = 'X'`):

* Os carimbos de data/hora de coleta são normalizados removendo os segundos.
* Exemplo: 12:12:23 torna-se 12:12:00.
* Cria limites de coleta consistentes e alinhados em todas as coleções.

**Atraso na coleta de dados** (`DATA_COLLECTION_DELAY > 0`):

* Subtrai o atraso especificado em minutos dos carimbos de data/hora de início e fim da coleta de dados.
* Desloca a janela de coleta para trás para capturar atualizações de banco de dados perdidas.
* Exemplo: um atraso de 2 minutos altera 10:00:00 - 10:05:00 para 09:58:00 - 10:03:00.

## Novos provedores de dados para sistemas de negócios ABAP

### NR\_SAP\_ABAP\_WS\_COMPLETE

Processa cabeçalhos de mensagens de web service e registros de dados (tag) em paralelo nos sistemas monitorados, sem limitações de registros.

**Configuração:**

* **Tipo:** Evento
* **Comunicador:** N/A
* **Função do provedor de dados:** `/NEWR/WS_MESSAGES_ARFC`
* **Provedor de dados RFC habilitado:** Sim
* **Função de finalização:** `/NEWR/WS_EVENTS_FINISHER`
* **Grupo de monitoramento:** `SAP_APPL`

### Visão geral funcional

Este provedor de dados vem pré-ativado. Parâmetros obrigatórios:

* **`ADAPTER_TYPE`**

  * **Objetivo:** Especificar o tipo de adaptador a ser selecionado do cabeçalho da mensagem do serviço web ABAP. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Campos da tabela separados por `;`, podem ser múltiplas linhas

* **`DESTINATION`**

  * **Objetivo:** Especificar o destino RFC
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Destino RFC

* **`DPC_COLL_TIME`**

  * **Objetivo:** Determinar o intervalo de tempo
  * **Sinal:** Apenas inclusivo
  * **Opção:** BT
  * **Valores:** Timestamp de início e timestamp de fim gerados pelo sistema

* **`IB_IF_NAME`**

  * **Objetivo:** Nome da interface do receptor a ser selecionado do cabeçalho da mensagem do serviço web ABAP. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Nomes das interfaces do receptor separados por `;`, pode ter várias linhas

* **`IB_PARTY`**

  * **Objetivo:** Parte receptora a ser selecionada do cabeçalho da mensagem de serviço web ABAP. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Parte receptora separada por `;`, pode ter várias linhas

* **`JOIN_CONDITION`**

  * **Objetivo:** Especificar registros de junção para a tabela `/NEWR/JOIN_COND` para processamento posterior
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Títulos de exibição separados por `;`, podem ser várias linhas

* **`NR_ACCID`**

  * **Objetivo:** Especificar o ID da conta New Relic
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** ID da conta New Relic

* **`OB_IF_NAME`**

  * **Objetivo:** Nome da interface do remetente a ser selecionado no cabeçalho da mensagem do serviço web ABAP. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Nomes de interfaces de remetente separados por `;`, pode ter várias linhas

* **`OB_PARTY`**

  * **Objetivo:** Parte emissora a ser selecionada do cabeçalho da mensagem de serviço web ABAP. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Partes remetentes separadas por `;`, pode ter várias linhas

* **`SETTING_SWITCH`**

  * **Objetivo:** Especificar opções usadas pelo processamento distribuído

  * **Sinal:** Apenas inclusivo

  * **Opção:** EQ

  * **Valores:** Várias opções separadas por `;` para controlar o processo distribuído:

    * **HEADONLY=**

      * `SPACE` (padrão): Fornece dados de cabeçalho e de TAG (se os nomes das TAGs forem especificados no parâmetro `TAG_LIST`).
      * `X`: Fornece apenas dados de cabeçalho mesmo que nomes de TAG sejam especificados no parâmetro `TAG_LIST`.

    * **FIRSTONLY=**

      * `X` (padrão): O provedor de dados extrai dados exclusivamente da primeira ocorrência de cada TAG listada no parâmetro `TAG_LIST`.
      * `SPACE`: O provedor de dados extrai dados de todas as ocorrências de cada TAG listada no parâmetro `TAG_LIST`.

    * **FREE%=nn:** Especifique a porcentagem de Processos de Trabalho disponíveis no sistema de monitoramento para determinar a Limitação de Processos de Trabalho Livres Desejada:

      * Se o número de Processos de Trabalho livres no sistema de monitoramento atual for igual ou superior a esta Limitação de Processos de Trabalho Livres Desejados, continue o processamento de distribuição.

      * Caso contrário, aguarde 1 segundo e repita a verificação acima até que:

        * O processamento de distribuição pode continuar, ou
        * O tempo total de espera excede o Tempo Máximo de Espera especificado (consulte `MWAIT#=nn`).

      * O valor padrão para esta opção é 30(%).

    * **MINFT#=nn:** Especifique a limitação mínima de processos de trabalho livres:

      * Se os Processos de Trabalho Livres Desejados contados pelo sistema de monitoramento ficarem abaixo deste limite mínimo, o sistema fará uma pausa de 1 segundo antes de tentar processamento adicional.
      * O valor padrão para esta opção é 3.

    * **MWAIT#=nn:** Especifique o tempo máximo de espera (em segundos).

      * Se o tempo total de espera (consulte `FREE%=nn`) exceder o máximo especificado, o processo de distribuição será cancelado para conservar recursos do sistema.
      * O valor padrão para esta opção é 60.

    * **MREC#=nnnnn:** Especifique o limite de número de registros de **Cabeçalho da Mensagem do Web Service** recuperados cada vez ao chamar o **Módulo de Função** `/SDF/IMA_DC_WS_ABAP`.
      * O valor padrão para esta opção é 10000.

    * **JSEND#=nnnn:** Especifique o limite do número de registros de atributos de resultado enviados ao servidor da New Relic a cada vez.
      * O valor padrão para esta opção é 5000.

    * **USER=user:** Especifique o nome de usuário RFC para cada sistema monitorado para obter o número total de processos de trabalho atualmente utilizados por esse sistema. O `user` é o ID de usuário SAP.

    * **DPWP#=nn:** Especifique o **Limite Máximo de Processos de Trabalho**.

      * Durante o processamento distribuído: Se o número de Processos de Trabalho livres no sistema de monitoramento atual for menor ou igual a este limite máximo, continue o processamento distribuído.
      * Caso contrário, aguarde 1 segundo e volte para verificar novamente.
      * Continue este loop até que o processamento distribuído possa prosseguir ou o tempo total de espera exceda o Tempo Máximo de Espera especificado (`MWAIT#=nn`).
      * Nota: Esta verificação é executada apenas quando o parâmetro USER é especificado com valores não iniciais.
      * O valor padrão para esta opção é 8.

* **`STATE_GROUP`**

  * **Objetivo:** Grupo de status de processamento a ser selecionado do cabeçalho da mensagem do web service ABAP. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Grupos de status de processamento separados por `;`, podem ser várias linhas

* **`SYS_ID`**

  * **Objetivo:** Determinar o ID do sistema
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** ID do sistema

* **`TAG_LIST`**

  * **Objetivo:** Especificar nomes de TAG a serem selecionados dos dados de mensagem do web service ABAP e exibidos como saída nos dados da mensagem. Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Campos da tabela separados por `;`, podem ser múltiplas linhas

* **`USERNAME`**

  * **Objetivo:** Nomes de usuário que criaram a mensagem a serem selecionados do cabeçalho da mensagem do serviço web ABAP (`CREATED_BY`). Opção de filtro do módulo de função `/SDF/IMA_DC_WS_ABAP`.
  * **Sinal:** Apenas inclusivo
  * **Opção:** EQ
  * **Valores:** Nomes de usuário separados por `;`, podem ser várias linhas
