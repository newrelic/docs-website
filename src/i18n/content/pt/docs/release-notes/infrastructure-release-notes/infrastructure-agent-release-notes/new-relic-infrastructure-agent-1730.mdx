Uma nova versão do agente foi lançada. Siga os procedimentos padrão para [atualizar o agente de infraestrutura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). A New Relic recomenda que você atualize o agente regularmente e, no mínimo, a cada 3 meses. A partir desta versão, a versão mais antiga suportada é o [Infrastructure agent 1.63.0](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1630/).

### Adicionado

* feat:Suporte para conta de serviço com privilégios mínimos do NewRelic Infrastructure Agent no Windows ([#2152](https://github.com/newrelic/infrastructure-agent/pull/2152))

  Esta versão introduz suporte abrangente para executar o agente de infraestrutura sob contas de serviço com privilégios mínimos no Windows:

  * A atualização preserva a conta de serviço: Na atualização, o instalador agora detecta e preserva a conta de logon de serviço existente (por exemplo, um domínio personalizado ou uma conta gMSA) em vez de redefini-la para LocalSystem. As permissões no diretório de dados do aplicativo são automaticamente concedidas novamente à conta preservada.

  * A instalação limpa usa `LocalSystem` por padrão: Novas instalações continuam a registrar o serviço em `LocalSystem`. Use `set-service-account.ps1` (novo script complementar) pós-instalação para mudar para uma conta de menor privilégio. Suporta contas locais, contas de domínio padrão e Contas de Serviço Gerenciadas de Grupo (gMSA). Também suporta as credenciais armazenadas no Gerenciador de Credenciais do Windows.

  * Segurança de rollback: Um arquivo marcador é gravado antes da criação do serviço para que a ação de rollback do MSI remova apenas os serviços criados pela execução atual — ela não excluirá um serviço preexistente durante uma atualização com falha.

  * Log de depuração: A instalação agora grava um log de depuração com carimbo de data/hora (`newrelic_installer_debug.log`), facilitando a solução de problemas.

  * Impacto: Sem alterações de quebra para instalações existentes usando a conta SYSTEM padrão. Os novos parâmetros são totalmente opcionais.
