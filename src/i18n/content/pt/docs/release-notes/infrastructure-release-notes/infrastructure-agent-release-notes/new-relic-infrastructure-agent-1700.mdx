Uma nova versão do agente foi lançada. Siga os procedimentos padrão para [atualizar o agente de infraestrutura](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). A New Relic recomenda que você atualize o agente regularmente e, no mínimo, a cada 3 meses. A partir deste lançamento, a versão mais antiga suportada é [agente de infraestrutura 1.58.0](https://docs.newrelic.com/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1580/).

## Mudado

* feat(oci): adicionar tag específica Oracle Cloud aos hosts em execução no OCI em [https://github.com/newrelic/infrastructure-agent/pull/2121](https://github.com/newrelic/infrastructure-agent/pull/2121)

  A partir da versão 1.70.0, o agente New Relic Infrastructure agora coleta automaticamente metadados para VMs em execução na infraestrutura de nuvem (OCI) Oracle. O agente coleta os seguintes detalhes:

  * displayName
  * host.id
  * oci.availabilityDomain
  * oci.region
  * oci.compartmentId
  * oci.shape
  * oci.imageId

  Você pode usar esses metadados como tag para pesquisa de entidade ou como atributo na consulta NRQL. O atributo `host.id` também é usado pela New Relic integração na nuvem para OCI para anexar metadados cloud à entidade hospedeira correta.

  Este recurso é habilitado por padrão para garantir consistência com o SQL existente em outros provedores cloud. Se precisar desabilitar esse recurso, você pode usar as definições de configuração [disable\_cloud\_metadata](https://docs.newrelic.com/docs/infrastructure/infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#disable-cloud-metadata) ou [disable\_cloud\_instance\_id](https://docs.newrelic.com/docs/infrastructure/infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#disable-cloud-instance-id).

  *Contexto*: Há muitos anos, o agente New Relic Infrastructure instalado em uma VM em execução no AWS, Azure ou GCP tinha a coleta de detalhes de metadados equivalente, e esse comportamento é habilitado por padrão. Para consistência entre diferentes nuvens, o recurso OCI está usando os mesmos itens de configuração e o mesmo comportamento padrão.
