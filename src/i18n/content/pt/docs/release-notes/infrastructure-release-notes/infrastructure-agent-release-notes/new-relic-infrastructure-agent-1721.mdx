Uma nova versão do agente foi lançada. Siga os procedimentos padrão para [atualizar o agente de infraestrutura](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). A New Relic recomenda que você atualize o agente regularmente e, no mínimo, a cada 3 meses. Até o momento desta versão, a versão mais antiga suportada é [o agente de infraestrutura 1.58.1](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1581/).

## Adicionado

* Adicionada a opção de configuração `logging_http_client_timeout` para controlar o tempo limite do cliente HTTP (em segundos) para o plugin de saída fluent-bit do New Relic em [#2167](https://github.com/newrelic/infrastructure-agent/pull/2167)
* Adicionado o esquema de upload para RHEL10 em [#2171](https://github.com/newrelic/infrastructure-agent/pull/2171)

## Mudado

* Definir a flag `win_network_interface_v2` como verdadeira (true) por padrão em [#2173](https://github.com/newrelic/infrastructure-agent/pull/2173)
* Versão mínima atualizada para pkg-tests em [#2172](https://github.com/newrelic/infrastructure-agent/pull/2172)

## Segurança

* Abordada a vulnerabilidade crítica - CVE-2025-15467 e a vulnerabilidade alta - CVE-2025-69419
