### Atualizar

* Lançamos uma atualização para o modelo AWS CloudFormation para nossa [integração do Métrica Streams](https://docs.newrelic.com/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/via-cloudformation-cwms/). Esta atualização resolve um problema crítico que poderia impedir que a stack fosse atualizada e usasse os tempos de execução Python modernos AWS Lambda.

* Para aplicar a correção à sua integração existente, siga as etapas abaixo para atualizar sua stack do CloudFormation:

  1. **Navegue até o CloudFormation**: faça login no seu AWS Management Console e acesse o serviço **CloudFormation**.

  2. **Selecione sua pilha**: encontre e selecione sua stack New Relic métrica Streams (geralmente chamada de `NewRelic-Metric-Streams-*`) e clique dropdown **Update Stack** \[Atualizar pilha]. Nas opções, selecione **Make a direct update** \[Fazer uma atualização direta]. Para obter mais informações, consulte a [documentaçãoAWS sobre atualização direta do stack](https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/using-cfn-updating-stacks-direct.html).

  3. **Substituir modelo**: na tela de atualização, você **deve** selecionar **Replace current template** \[Substituir modelo atual].

  **OBSERVAÇÃO**: Não use a opção &quot;Usar modelo existente&quot;, pois ela não aplicará as correções necessárias.

  4. **Forneça a URL do novo modelo**: na seção *Especificar modelo*, escolha a opção **Amazon S3 URL** e cole o seguinte link:

  `https://nr-downloads-main.s3.amazonaws.com/cloud_integrations/aws/cloudformation/newrelic-cloudformation-mstreams.yml`

  5. **Confirmar e atualizar**: clique em **Next** \[Avançar] nas páginas de parâmetros e opções stack. Na página de revisão final, role até o final, marque a caixa para **confirmar que AWS CloudFormation pode criar recursos do IAM** e clique em **Update stack** \[Atualizar stack].

  Você pode monitorar o progresso na aba **de eventos**. A atualização é concluída quando o status stack muda para `UPDATE_COMPLETE`.
