### Novos documentos

* Um novo documento de instalação ramificado da [integração de monitoramento do Kubernetes](/install/kubernetes/) ajuda você a encontrar as informações de instalação específicas de que precisa.
* Instale, configure e monitore seus dados do [dbt Cloud](/docs/more-integrations/dbt-cloud/dbt-cloud-integration) em execução no Apache Airflow e usando o Snowflake.
* Nosso novo [operador de agente do Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator) automatiza a instrumentação do agente APM para que você não precise mais gerenciá-los manualmente. Agora em visualização pública.
* O agente iOS adicionou suporte para o envio de [logs de mobile](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs) e [instrumentação do watchOS](/docs/mobile-monitoring/new-relic-mobile-ios/watchOS/instrument-watchos) para a New Relic.
* [Solucionar problemas de dados que não aparecem para o AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-no-data-appears) descreve soluções para quando você não vê seus dados do AWS Lambda na interface.
* [Relatório de falhas nativas do agente Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-native-crash-reporting) descreve como ativar, configurar e depurar o relator de falhas.
* [Modelo de título para condições de alerta NRQL](/docs/alerts/create-alert/condition-details/title-template) descreve como usar modelos de título para criar títulos significativos para seus eventos de alerta.
* [Observabilidade de LLM do Traceloop com OpenLLMetry](/docs/opentelemetry/get-started/traceloop-llm-observability/traceloop-llm-observability-intro) descreve como usar essa ferramenta baseada em OpenTelemetry para monitorar o desempenho do seu modelo de IA.

### Grandes mudanças

* Atualizações de conteúdo e estilo na vinculação de aplicações [instrumentadas com APM](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes) e [instrumentadas com OpenTelemetry](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) melhoram a legibilidade e facilitam a configuração. Também atualizamos nossa introdução à [integração com o Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration) para melhorar o estilo e a clareza.
* Atualizamos nossa documentação para refletir a alteração de preço por GB, por exemplo, em [faturamento e regras de ingestão de dados](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices).
* Adicionados novos atributos [`MobileApplicationExit`](/attribute-dictionary/?dataSource=Mobile&event=MobileApplicationExit) ao nosso dicionário de dados.
* Adicionadas informações de instalação específicas do Azure para [instalar o agente .NET](/install/dotnet/?deployment=azure&azure=azuresiteextension).
* Em 3 de julho de 2024, nossa [integração com Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements/#compatibility) encerrou o suporte ao Kubernetes v1.25 e anteriores.
* Uma grande reescrita de [Conheça a New Relic AI, seu assistente de observabilidade](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-ai) descreve como aproveitar ao máximo nossas ferramentas de IA.
* Atualizamos as [configurações do agente de infraestrutura](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#is_integrations_only) com informações de configuração sobre a nova configuração `is_integrations_only`. Habilite esta configuração se você estiver instrumentando um host a partir de uma fonte diferente do agente de infraestrutura, como um coletor OpenTelemetry ou um Prometheus node exporter.

### Pequenas mudanças

* Várias atualizações relacionadas ao novo operador do agente do Kubernetes.
* Atualizações na documentação do OpenTelemetry corrigiram alguns links âncora quebrados, adicionaram informações de configuração de timeout ao [endpoint OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp/#payload-limits) e outras melhorias de consistência.
* Atualizado o [AWS PrivateLink](/docs/data-apis/custom-data/aws-privatelink/#regions-zones) para esclarecer que oferecemos suporte a várias regiões
* Atualizamos nossa [tabela de compatibilidade do agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#system) para listar o suporte ao Node.js versão 22.
* Movemos nossos documentos de resolução de problemas do agente .NET no Azure para a [categoria de resolução de problemas](/docs/apm/agents/net-agent/azure-installation/azure-cloud-services-no-data-appears/).
* Atualizado `issueId` e adicionado `entityGuid` em [/docs/data-apis/ingest-apis/security-data-api](/docs/data-apis/ingest-apis/security-data-api/#req-fields).
* Corrigido erro em trecho de código na [integração do Cacti](/docs/infrastructure/host-integrations/host-integrations-list/cacti-integration/).
* Atualizada a [instrumentação sem camadas para monitoramento do Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-layerless/#dotnet) para esclarecer que o Lambda .NET não requer camadas.
* Atualizado [controlar quando o agente Ruby inicia](/docs/apm/agents/ruby-agent/troubleshooting/control-when-ruby-agent-starts) para descrever como desativar o agente em contextos específicos.
* Removemos os formulários de inscrição inline de todo o nosso site de documentação.
* Adicionadas instruções sobre como configurar e encaminhar logs do Temporal na [integração de monitoramento do Temporal auto-hospedado](/docs/infrastructure/host-integrations/host-integrations-list/temporal-monitoring-integration/#temporal-logs-configuration).
* O suporte ao monitoramento de [Infraestrutura](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/#os), [desempenho de SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#linux), [syslog de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#linux) e [dados de fluxo de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#linux) para o CentOS 7 terminou em 30 de junho de 2024.
* [Limites de taxa com NRQL](/docs/nrql/using-nrql/rate-limits-nrql-queries/#query-duration) atualizados com o tempo limite de consulta padrão de 5 segundos do NerdGraph.
* Atualizado [aplicativo não respondendo](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding) com novas capturas de tela e detalhes de configuração do lado do servidor.
* Corrigido erro de sintaxe para `disable_all_plugins` no guia de resolução de problemas [reduzir o consumo de CPU do agente de infraestrutura](/docs/infrastructure/new-relic-infrastructure/troubleshooting/reduce-infrastructure-agents-cpu-footprint).
* As [métricas em nível de código](/docs/codestream/observability/code-level-metrics/#requirements) do CodeStream para PHP requerem a extensão PHP Intelephense.
* Em [remover aplicativos do New Relic](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/#wait-longer), esclarecemos que você pode excluir um aplicativo quando o New Relic não tiver recebido nenhum dado desse aplicativo por 12 horas ou mais.
* Unificamos nossa documentação de redução de ingestão de dados e governança de dados do Kubernetes em uma única página de [redução de dados ingeridos](/docs/kubernetes-pixie/kubernetes-integration/installation/reduce-ingest).
* Atualizado o [plugin AWS FireLens para encaminhamento de logs](/docs/logs/forward-logs/aws-firelens-plugin-log-forwarding/#firelens-plugin) com várias novas regiões da AWS que agora suportamos e seus repositórios de imagens.
* Atualizados os [fluxos de trabalho de eventos de alerta](/docs/alerts/get-notified/alert-event-workflows/#navigate-notification-log) com as colunas atuais do log de notificação.
* Atualizamos a [compatibilidade e requisitos do agente Java](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#JDBC) para informar que nosso agente Java versão 8.13.0 ou superior suporta HikariCP 2.4.0 até a versão mais recente. (Isso substitui o módulo incubador HikariCP.)
* O agente .NET agora suporta o [`Elastic.Clients.Elasticsearch`](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#app-frameworks-core) versão 8.14.5.
