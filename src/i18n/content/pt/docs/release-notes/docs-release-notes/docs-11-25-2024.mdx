### Novos documentos

* [Integração do Amazon MemoryDB](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-memorydb-monitoring-integration) descreve como ativar a integração com o New Relic. A integração permite coletar dados de telemetria sobre seus clusters do Amazon MemoryDB, consultar os dados recebidos e criar dashboards.
* [Adicionar tags (rótulos) personalizadas aos logs da aplicação](/docs/logs/logs-context/custom-tags-agent-forwarder-logs) explica que você pode adicionar rótulos personalizados aos logs encaminhados pelo agente.

### Grandes mudanças

* Atualizamos a [documentação de integrações da AWS](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations) para clareza e organização. As principais mudanças incluem enfatizar os fluxos de métricas do CloudWatch como o principal método de integração da AWS, aprimorar a navegação e reduzir a redundância ao consolidar e simplificar o conteúdo, além de fazer edições de texto para melhor legibilidade.
* Atualizamos as etapas para [criar um postmortem](/docs/alerts/alert-event-management/postmortems-applied-intelligence/#create-postmortem) para maior precisão.
* Atualizações relacionadas à conformidade feitas em [Certificações, padrões e regulamentações](/docs/security/security-privacy/compliance/regulatory-audits-new-relic-services) com base nas avaliações anuais de 2024 da New Relic.
* Adicionada compatibilidade com .NET 9 e ASP.NET Core 9 para o agente .NET, esclarecida a posição de suporte para versões do .NET fora de suporte e removidas referências desatualizadas ao .NET Framework em [Agente .NET: compatibilidade e requisitos](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements).
* Reescrito o [plugin do Fluent Bit para encaminhamento de logs](/docs/logs/forward-logs/fluent-bit-plugin-log-forwarding) para maior clareza.
* [Notificações do CodeStream](/docs/codestream/codestream-settings/account-settings) atualizadas para precisão.
* Adicionados os detalhes da [variável do agente Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#cloud_config) para criar um relacionamento entre provedores de nuvem e aplicações APM.
* Atualizamos os passos para [criar um Azure Event Hub](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export/#create-event-hub).

### Pequenas mudanças

* Atualizadas as versões dos agentes .NET e Ruby em [Adicionar tags personalizadas (labels) aos logs de aplicação](/docs/logs/logs-context/custom-tags-agent-forwarder-logs).
* Fornecido um exemplo para mostrar o uso de APIs de log com Unity em [Logs do Mobile](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs).
* [Compatibilidade e requisitos do agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) atualizados.
* [Compatibilidade e requisitos do agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) atualizados.
* Comportamento padrão e orientações de configuração atualizados para a inclusão de dispositivos de rede e geração de NetworkSample em [eventos de infraestrutura](/docs/infrastructure/infrastructure-data/default-infra-data).
* Corrigido o nome da API `localStorage` e atualizadas as informações sobre quando o rastreamento de sessão pode não funcionar em [Rastreamento de sessão](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking).
* Detalhes de configuração do agente Node.js atualizados para [ativar](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels) e [excluir](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels-exclude) a adição de rótulos aos logs da aplicação.
