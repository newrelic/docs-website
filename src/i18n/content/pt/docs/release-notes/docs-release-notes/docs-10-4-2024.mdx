### Novos documentos

* O [Gerenciamento de cardinalidade](/docs/data-apis/ingest-apis/metric-api/cardinality-management) oferece ferramentas administrativas para gerenciar e revisar o uso de cardinalidade da sua organização.

### Grandes mudanças

* Instruções aprimoradas sobre [rastreamento de usuários impactados com o errors inbox](/docs/errors-inbox/error-users-impacted/) com uma nova captura de tela e uma tabela descrevendo os vários métodos de implementação.
* Atualizados [Vincular aplicativos instrumentados com OpenTelemetry ao Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) e [Explorar seu cluster Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/kubernetes-cluster-explorer) com informações atuais de configuração e instalação do OpenTelemetry.
* Reescrevemos [Regras de silenciamento: Suprimir notificações](/docs/alerts/get-notified/muting-rules-suppress-notifications) para descrever como criar, gerenciar, suprimir e revisar seus eventos de alerta e problemas.
* Reorganizamos nossa documentação de [solução de problemas do Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/troubleshooting/overview) para melhorar a navegação e a facilidade de localização.
* Revisamos nossa documentação do [OpenTelemetry](/docs/opentelemetry/opentelemetry-introduction) para clareza e completude.
* Adicionadas instruções para configurar a API de runtime do Composer para detecção de pacotes de gerenciamento de vulnerabilidades na [configuração do agente PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-vulnerability-management).

### Pequenas mudanças

* Atualizamos alguns documentos para informar que os gráficos de billboard não serão renderizados corretamente quando compartilhados via links, imagens exportadas e PDFs.
* Corrigidos vários documentos onde estávamos misturando comandos Linux com nossos comandos de shell do Windows.
* Removidos alguns documentos legados e adicionados redirecionamentos para suas versões atuais.
* Atualizamos as [Condições de alerta](/docs/alerts/create-alert/create-alert-condition/alert-conditions) para a interface atual.
* Adicionados novos atributos de [`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming) ao nosso dicionário de dados.
* Documentação de monitoramento de desempenho de rede atualizada relacionada à API de polling da Meraki e às configurações padrão para uso de ICMP com o KTranslate.
* Adicionado `NewRelic.Labels` às [configurações do .NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration).
* Em [Rastrear usuários impactados com o errors inbox](/docs/errors-inbox/error-users-impacted/#set-users), foi adicionada a chamada de API `newrelic_set_user_id` do PHP.
* Adicionados novos valores de tipo de evento proibidos em [Requisitos e limites de dados para dados de eventos personalizados](/docs/data-apis/custom-data/custom-events/data-requirements-limits-custom-event-data): prefixos de string `EntityRelationship`, `RelationshipUpdate` e `Relationship[0-9]`.
* Em [Primeiros passos com o New Relic e o Terraform](/docs/infrastructure-as-code/terraform/terraform-intro), descrevemos como usar o CodeStream com o Terraform.
