### Novos documentos

* [O tutorial do NerdGraph: Gerenciando as regras de normalização métrica](/docs/apis/nerdgraph/examples/nerdgraph-metric-normalization-rule) fornece orientações sobre o uso do NerdGraph para gerenciar regras de normalização métrica.
* [A biblioteca de modelos de scorecards](/docs/service-architecture-intelligence/scorecards/templates) possui modelos personalizáveis para casos de uso de engenharia e negócios, baseados em práticas empresariais recomendadas para enfrentar desafios comuns.
* [O mapeamento métricoNew Relic OTel K8](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel-metrics) descreve como as métricas Kubernetes OpenTelemetry são mapeadas para as métricas New Relic Kubernetes .

### Grandes mudanças

* Adicionado `sampler.remote_parent_sampled` na [configuração do agente Java : Arquivo de configuração](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file/#dt-sampler-remote-parent-sampled) para definir como o agente deve manipular intervalos de amostragem quando seu intervalo de amostragem de uma entidade upstream for amostrado.
* Atualize o [provisionamento do SCIM](/docs/accounts/accounts-billing/new-relic-one-user-management/introduction-saml-scim) para especificar que agora ele requer um <DNT>**Paid Edition**</DNT> para gerenciamento automatizado de usuários no New Relic.
* O documento [Cloud Rule APIs](/docs/new-relic-control/pipeline-control/cloud-rules-api/#drop-attributes-on-dimensional-metric-rollups) para controle de pipeline foi atualizado para incluir informações sobre como remover atributos de rollups de métricas dimensionais.
* Adicionada consulta para acompanhar eventos de login e logout na página [Monitoramento de Alterações de Conta](/docs/accounts/accounts/account-maintenance/query-account-audit-logs-nrauditevent/#user-login) .

### Pequenas mudanças

* Atualizada a [versão de compatibilidade do agente .Net](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#framework).
* Foi removida a limitação do PingOne no [gerenciamento automatizado de usuários](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) , que anteriormente não era suportada devido a restrições de provisionamento de grupo.
* Foi adicionada uma limitação na página [Detecção de anomalias,](/docs/alerts/create-alert/set-thresholds/anomaly-detection/) especificando que não há suporte para sazonalidade mensal ou anual devido a limites computacionais.
* Atualizado o [relatório de compatibilidade do agente Node.js.](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)
* Adicionadas configurações de compactação e processamento em lote para rastreamento infinito ao documento [de configuração do agente Node.js.](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#infinite-tracing-batching)
