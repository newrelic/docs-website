### Novos documentos

* Adicionada a documentação da [Caixa de Entrada de Riscos de Desempenho](/docs/errors-inbox/performance-risks/introduction) para o novo Preview Público que detecta consultas N+1, SQL lento e outros antipadrões de codificação.
* Adicionadas [ações de incidente do ServiceNow](/docs/workflow-automation/setup-and-configure/actions-catalog/servicenow/servicenow-incident) para automação de fluxo de trabalho, abrangendo operações de criação, obtenção, listagem, resolução e atualização.

### Grandes mudanças

* Atualizamos a [documentação do processador de gateway do Controle de Pipeline](/docs/new-relic-control/pipeline-control/gateway/yaml-overview) para refletir o esquema YAML v2.2.0 simplificado com uma matriz `rules` plana e um campo `context` explícito.
* Atualizadas a [configuração do agente Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration) e as referências da API de trace para a versão v13.0.0, incluindo as novas configurações `traces.include` e `traces.exclude` do OpenTelemetry.
* Atualizada a [interface de atualização do runtime do Synthetics](/docs/synthetics/synthetic-monitoring/using-monitors/runtime-upgrade-ui) com um layout de abas renovado para os fluxos de trabalho <DNT>**Fix manually**</DNT> e <DNT>**Fix with AI**</DNT>.

### Pequenas mudanças

* Adicionadas notas sobre injeção de dependência à [API do agente .NET](/docs/apm/agents/net-agent/net-agent-api/net-agent-api) explicando como usar `IAgent` com segurança com as APIs de transação e span do agente.
* Adicionada a opção de autenticação `Managed Identity` ao [encaminhamento de logs do Hub de Eventos do Azure](/docs/logs/forward-logs/azure-log-forwarding) para modelos ARM v3.2.0+.
* Adicionada a configuração do atributo `aws.cluster.arn` aos guias de monitoramento do OpenTelemetry para [ECS EC2](/docs/opentelemetry/integrations/ecs-monitoring/ecs-ec2) e [ECS Fargate](/docs/opentelemetry/integrations/ecs-monitoring/ecs-fargate).
* Adicionado suporte às bibliotecas Logging, Rails Event Logger e Semantic Logger aos [frameworks suportados pelo agente Ruby](/docs/apm/agents/ruby-agent/getting-started/ruby-agent-requirements-supported-frameworks).
* Adicionadas as entradas `active_record_use_table_name` e Dalli 5.0 à [configuração do agente Ruby](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration).
* Atualizados os [exemplos da API NerdGraph de Sintéticos](/docs/apis/nerdgraph/examples/synthetics-api/scripted-api-monitor) para recomendar `runtimeTypeVersion: "22.20.0"` e sinalizar a atualização forçada de 18 de novembro de 2026 para runtimes mais antigos do Node.js.
* Adicionado um bloco expansível de namespace do APM em [Gerenciar seus dados](/docs/data-apis/manage-data/manage-data-retention) esclarecendo quais tipos de evento do APM ficam fora do namespace de distributed trace.
* Adicionado um exemplo de loop `for` com expressão `jq` ao [Esquema de Definição de Automação de Fluxo de Trabalho](/docs/workflow-automation/workflow-automation-apis/definition-schema).
* Atualizada a [ação de notificação do New Relic](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) com novos formatos de resultado de consulta `JSON`, `CSV`, `PSV` e `TSV` e um exemplo de seletor `success`.
* Atualizamos a [Visão Geral dos Boletins de Segurança](/docs/security/security-privacy/information-security/security-bulletins) com a nova entrada [NR26-05](/docs/security/new-relic-security/security-bulletins/security-bulletin-nr26-05) para `CVE-2026-31431` no monitoramento sintético.
* Atualizamos os [requisitos do agente de infraestrutura](/docs/infrastructure/infrastructure-agent/requirements-infrastructure-agent) com opções de logon de serviço do Windows, incluindo contas Local System, Active Directory, Microsoft Entra ID e gMSA.
* Atualizado o [relatório de compatibilidade do agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) com as versões suportadas mais recentes dos pacotes.

### Publicações de novidades

* Adicionado [A Caixa de entrada de riscos de desempenho agora está em Visualização Pública](/whats-new/2026/05/whats-new-05-14-performance-risks-inbox) para anunciar o acesso gratuito à Visualização Pública para detectar consultas N+1, SQL lento, consultas ao banco de dados excessivas e sequenciais, requests HTTP lentos e grandes cargas HTTP.

### Notas de versão

* Fique por dentro dos nossos últimos lançamentos:

  * [Agente Python v13.0.0](/docs/release-notes/agent-release-notes/python-release-notes/python-agent-130000):

    * Removido o suporte para funções do Lambda em Python no arquivo de configuração (segurança).
    * O cabeçalho de distributed tracing `newrelic` foi excluído por padrão; apenas W3C `traceparent` e `tracestate` são enviados.
    * Adicionada instrumentação de streaming da Anthropic e de streaming de resposta do Gemini.
    * Adicionado o atributo `http.statusCode` aos eventos de transação; descontinuado o `response.status`.
