### Novos documentos

* [Gerenciamento de acesso adicionado para gerenciar a cardinalidade](/docs/data-apis/ingest-apis/metric-api/cardinality-access-management/) para controlar o acesso do usuário à métrica relacionada à cardinalidade e gerenciar permissões para limites de ingestão de métrica.
* [Cardinalidade do monitor](/docs/data-apis/ingest-apis/metric-api/monitor-cardinality/) adicionada para rastrear e monitorar o uso da cardinalidade métrica para otimizar a ingestão de dados e evitar custos inesperados.

### Pequenas mudanças

* Adicionado [novo parâmetro de configuração](/docs/ebpf/linux-installation/#config-params) `DROP_DATA_FOR_ENTITY` para ebpf.

* [Agente Ruby v9.21.0](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-9-21-0)

  * No modo APM Serverless, os nomes de transação agora incluem o nome da fonte do evento como um prefixo. Esse aprimoramento melhora a identificação e a rastreabilidade da transação na interface New Relic ao indicar claramente a origem do evento de acionamento.
  * Alterações revertidas na lógica de rastreamento de threads para inicialização de intervalo. Essa alteração restaura o comportamento anterior de rastrear o thread onde os intervalos começam, resolvendo problemas que surgiram da lógica atualizada.
