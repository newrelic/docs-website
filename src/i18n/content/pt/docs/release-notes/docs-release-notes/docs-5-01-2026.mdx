### Novos documentos

* Adicionada a suíte de documentação do [Mobile Session Replay](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/introduction/), agora com disponibilidade geral, com guias abrangentes cobrindo instalação, configuração de privacidade, recursos avançados, visualização de reproduções e resolução de problemas para Android e iOS.
* Adicionado [Atualizar monitores de API com script para o Node.js 22](/docs/synthetics/synthetic-monitoring/scripting-monitors/synthetics-scripted-api-monitors-to-node-22/) para fornecer instruções de migração passo a passo para mover monitores de API com script do Node.js 16 para o Node.js 22.

### Grandes mudanças

* Documentação da Inteligência de Custos na Nuvem atualizada para disponibilidade geral, incluindo as novas páginas [Preços do CCI](/docs/cci/cci-pricing/), [Condições do Alerta para a Inteligência de Custos na Nuvem](/docs/cci/configure-cci/cci-alerts/) e [Custos de IA](/docs/cci/financial-intelligence/ai-costs/), além de conteúdo expandido de configuração de integrações e perguntas frequentes.
* Documentação do [New Relic Teams](/docs/service-architecture-intelligence/teams/teams/) reestruturada com um novo guia [Configurar a Propriedade da Entidade](/docs/service-architecture-intelligence/teams/team-ownership/) e conteúdo de [Gerenciar Equipes](/docs/service-architecture-intelligence/teams/manage-teams/) atualizado para melhor navegação e clareza.
* Atualizada a documentação do agente eBPF para instalação no [Kubernetes](/docs/ebpf/k8s-installation/) e no [Linux](/docs/ebpf/linux-installation/) com orientações de migração da configuração YAML e alterações de campos introduzidas na v1.4.0.
* [Otimize o uso do seu computador (compute)](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/compute-optimizer/) atualizado com orientação expandida do dashboard, novas capturas de tela e recomendações de otimização aprimoradas.
* Atualizada a [Interface OpenTelemetry APM](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-ui/) com uma nova seção sobre monitoramento de erros na experiência do OpenTelemetry APM.
