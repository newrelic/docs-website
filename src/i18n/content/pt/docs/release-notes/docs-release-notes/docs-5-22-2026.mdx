### Novos documentos

* Adicionada a documentação de [gerenciamento a nível de serviço](/docs/service-level-management/intro-slm) para dois novos recursos em GA: Maintenance Windows para agendar um período de inatividade planejado que exclui eventos ruins dos cálculos de SLI, e Facet Compliance para detalhar a conformidade de SLO e os orçamentos de erro por atributo.

### Grandes mudanças

* Adicionadas orientações de migração para a v14 em [Atualizar o agente Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent), incluindo alterações com quebra de compatibilidade, instrumentação removida e versões mínimas suportadas atualizadas.
* Reestruturado o [plug-in do Apollo Server para Node.js](/docs/apm/agents/nodejs-agent/extend-your-instrumentation/apollo-server-plugin-nodejs) para refletir a migração da instrumentação do `@apollo/server` de um plug-in independente para a instrumentação tradicional na v14.
* Atualizamos [Instalar o agente eBPF da New Relic para Kubernetes](/docs/ebpf/k8s-installation) e [Instalar o agente eBPF da New Relic para hosts Linux](/docs/ebpf/linux-installation) com o conteúdo da versão v1.4.1, incluindo novas opções de configuração e etapas de instalação.

### Pequenas mudanças

* Reestruturamos a documentação de compartilhamento de dashboard e gráficos para ter páginas dedicadas ao compartilhamento público de [dashboards](/docs/query-your-data/explore-query-data/dashboards/share-dashboards-publicly/) e [gráficos](/docs/query-your-data/explore-query-data/dashboards/share-charts-publicly/).
* Adicionada a ferramenta `list_entity_performance_risk_groups` à [referência de ferramentas MCP](/docs/agentic-ai/mcp/tool-reference) para buscar grupos de risco de desempenho da Caixa de entrada de riscos de desempenho.
* Atualizada a referência do Mobile SDK [setMaxEventPoolSize](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-pool-size) com um aviso importante para contexto adicional.
* Atualizados os [requisitos de compatibilidade do agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) com as versões de framework suportadas mais recentes.
* Corrigidos os nomes dos dashboards do nginx na documentação da [integração do NGINX](/docs/opentelemetry/integrations/nginx/find-and-query-your-data) e da [integração do NGINX Plus](/docs/opentelemetry/integrations/nginx-plus/nginx-plus-otel) para refletir os nomes atuais dos dashboards.
* Atualizamos [Preços e faturamento do New Relic One](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) e [Introdução à Inteligência de Arquitetura de Serviços](/docs/service-architecture-intelligence/getting-started) para esclarecer que os recursos do SAI estão incluídos sem custo adicional para todos os usuários da plataforma completa.
* Atualizamos [Novo runtime](/docs/synthetics/synthetic-monitoring/using-monitors/new-runtime), [Gerenciar runtimes de monitor](/docs/synthetics/synthetic-monitoring/using-monitors/manage-monitor-runtimes) e [Resolução de problemas de atualização de runtime](/docs/synthetics/synthetic-monitoring/troubleshooting/runtime-upgrade-troubleshooting) para centralizar as referências de versão do Chrome em &quot;Latest&quot;.
* Atualizado o [Guia de transição de runtime do monitoramento sintético](/docs/synthetics/synthetic-monitoring/private-locations/runtime-transition-guide) com datas específicas de EOL para transições de runtime.
* Corrigido um problema de formatação em [Sintaxe, cláusulas e funções do NRQL](/docs/nrql/nrql-syntax-clauses-functions) ao remover uma referência a um nome de aplicativo interno.
* Corrigida a indentação do bloco de código em [Atualizar o agente Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent).

### Publicações de novidades

* Adicionado [Atualização do SLM: Janelas de manutenção e conformidade de facetas agora estão em Disponibilidade Geral](/whats-new/2026/05/whats-new-05-18-slm-maintenance-windows-facet-compliance) para anunciar a disponibilidade geral (GA) de janelas de manutenção programadas e o rastreamento de conformidade de SLO baseado em facetas no gerenciamento a nível de serviço.

### Notas de versão

* Fique por dentro dos nossos últimos lançamentos:

  * [Agente Node.js v14.0.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-0-0):

    * Suporte removido para Node.js 20 e `fastify` 3.x (breaking change).
    * Removido o Cross Application Tracing (CAT) e o suporte a políticas de licença, aplicativo e segurança (LASP).
    * Migrada a instrumentação `@apollo/server` do pacote `@newrelic/apollo-server-plugin` para a instrumentação tradicional.
    * Removida a instrumentação para `koa-route` e `koa-router`.
    * Versões mínimas suportadas atualizadas para `@nestjs/cli`, `@nestjs/core`, `next`, `bluebird`, `mysql2` e `cassandra-driver`.
    * Padrão de `config.distributed_tracing.exclude_newrelic_header` atualizado para `true`.

  * [Runtime da API Node rc1.21](/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.21):

    * Corrigidas múltiplas vulnerabilidades de segurança, incluindo CVE-2024-50004, CVE-2024-58096, CVE-2025-37926, CVE-2025-38201 e outras.

  * [Runtime do Node browser rc1.21](/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.21):

    * Chrome atualizado de 148.0.7778.167-1 para 148.0.7778.178-1.
    * Corrigidas múltiplas vulnerabilidades de segurança, incluindo CVE-2026-3832, CVE-2026-3833, CVE-2026-5260 e outras.

  * [Integração do Kubernetes v4.1.1](/docs/release-notes/infrastructure-release-notes/kubernetes-integration-release-notes/kubernetes-integration-4-1-1):

    * Lançado como parte do chart `newrelic-infrastructure-4.1.1` e `nri-bundle-7.0.12`.

  * [Agente de infraestrutura v1.75.0](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1750):

    * Adicionado suporte ao endpoint OCI IMDS v2.
    * Atualizadas as dependências `nri-docker`, `nri-winservices`, `nri-prometheus` e `nri-flex`.
    * Versão do Go atualizada para 1.26.3.

  * [Plug-in de saída do Logstash v2.0.0](/docs/release-notes/logstash-release-notes/logstash-26-03-05):

    * Alterado para o cliente HTTP Manticore com reutilização de conexão para melhor desempenho.

    * Testes de integração atualizados para validar a entrega de logs ao New Relic.
