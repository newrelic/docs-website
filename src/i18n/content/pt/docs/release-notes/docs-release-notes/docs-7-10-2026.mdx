### Novos documentos

* Adicionada a [integração do IBM MQ OpenTelemetry](/docs/opentelemetry/integrations/ibm-mq/overview) para o monitoramento do IBM MQ com OpenTelemetry, abrangendo configurações de host, Kubernetes, métricas e resolução de problemas.
* Adicionada a [integração do Redis OpenTelemetry](/docs/opentelemetry/integrations/redis/overview) para monitoramento do Redis com OpenTelemetry, cobrindo configurações de Docker, host e Kubernetes.
* Adicionada a [integração do HAProxy com OpenTelemetry](/docs/opentelemetry/integrations/haproxy/overview) para monitoramento do HAProxy com OpenTelemetry, cobrindo configurações auto-hospedadas e Kubernetes.
* Adicionado o [monitoramento de log de fluxo da VNet do Azure](/docs/network-performance-monitoring/setup-performance-monitoring/cloud-flow-logs/azure-vnet-flow-log-monitoring) para encaminhar e consultar dados de log de fluxo da rede virtual do Azure.

### Grandes mudanças

* Os [destinos entre contas](/docs/apis/nerdgraph/examples/cross-account-destinations/overview) agora estão em disponibilidade geral.

### Pequenas mudanças

* Adicionada a documentação do parâmetro `blocks` à [ação de automação de fluxo de trabalho](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification) `sendSlack` para o envio de mensagens do Slack Block Kit, juntamente com uma tabela de tipos de blocos suportados.
* Atualizados os [requisitos de compatibilidade do agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) com o relatório de compatibilidade automatizado mais recente.
* Atualizamos os exemplos de [instalação do Docker no Linux](/install/dotnet/?deployment=linux&docker=yesDocker) e [instalação do Docker no Windows](/install/dotnet/?deployment=windowsInstall&docker=yesDocker) para o agente .NET com instruções para instalar uma versão específica em vez da mais recente.
* Corrigido o intervalo de atualização de associação do [workload inteligente](/docs/new-relic-solutions/new-relic-one/workloads/create-intelligent-workload) de 5 minutos para 10 minutos, observando que está sujeito a alterações durante a visualização pública.
* Esclarecido no [intervalo de tempo da consulta NRQL](/docs/nrql/using-nrql/query-time-range) que a inclusão de datapoint de métrica bruta depende do horário de início de um datapoint, não do seu horário de término.
* Atualizada a [configuração de status do workload](/docs/new-relic-solutions/new-relic-one/workloads/workload-status-configuration) para esclarecer quais métodos de status estão disponíveis para workloads Standard versus Intelligent.
* Adicionada orientação para remover um usuário de um domínio de autenticação ou organização em [vários registros de usuário](/docs/accounts/accounts-billing/account-setup/multiple-user-records).
* Atualizados os [requisitos de compatibilidade do agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements) para esclarecer o suporte e as limitações do IBM DB2 para .NET Core e .NET framework.
* Adicionado suporte para Ubuntu 26.04 (Resolute) ao [encaminhamento de logs por meio do agente de infraestrutura](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent), via Fluent Bit 5.0.6.

### EOLs

* [Fim da vida útil para o monitoramento do Drupal 7 e 9 no agente PHP](/eol/2026/07/eol-07-09-26-drupal-7-9-monitoring): o suporte termina em 15 de outubro de 2026.
* [Fim da vida útil do suporte ao runtime do Node.js 20](/eol/2026/07/eol-07-09-26-nodejs-20-runtime): os agentes da New Relic não oferecem mais suporte ao runtime do Node.js 20, que atingiu o fim da vida útil em 30 de abril de 2026.
