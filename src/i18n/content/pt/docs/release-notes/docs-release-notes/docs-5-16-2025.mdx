### Novos documentos

* Anunciado 16 de junho de 2025 como a data de fim da vida útil do [Metrica e Event Explorer](/eol/2025/05/metrics-and-event-explorer/).
* Adicionadas [instruções para criação de interação de clientes](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-custom-Interaction/) via agente Android .

### Grandes mudanças

* Adicionada a substituição da variável de ambiente `NEW_RELIC_CONFIG_FILENAME` à [configuração do agente Node.js.](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#filename)
* Foi adicionada uma Ruby [distributed tracing configuração à configuração](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/#distributed_tracing-sampler-remote_parent_sampled) do agente que gerencia o comportamento de amostragem de transações com um pai remoto quando o trace sinalizador é definido no traceparent.
* Adicionado o Google Gen AI SDK como biblioteca suportada para o agente Python na página [Compatibilidade e requisitos para monitoramento de IA](/docs/ai-monitoring/compatibility-requirements-ai-monitoring/) .
* Adicionado recurso na [página condição do alerta](/docs/alerts/create-alert/create-alert-condition/alert-conditions/#disable-health-status-reporting) que permite evitar alterações automáticas no status de saúde da entidade quando ocorre uma notificação de alerta.
* Melhorias adicionadas na personalização da visualização da tabela de logs na página [da interface do usuário Usar logs](/docs/logs/ui-data/use-logs-ui/#customizable-table-views) .
* Adicionadas informações sobre [o uso do NRDiag](/docs/new-relic-control/agent-control/troubleshooting/#nrdiag) para solucionar problemas comuns ao instalar ou operar o Agente Control.
* Prorrogou a data de fim da vida útil para 21 de maio de 2025 para remover intervalos [de IP públicos mais antigos do minion Sintético](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips/#dynamic-ip-addresses) da lista de intervalos de IP JSON.
* Atualizado o documento [Suporte do agente para ARM64/Graviton e x86\_64](/docs/apm/agents/manage-apm-agents/configuration/support-arm-graviton-x86-64/) para refletir que a infraestrutura do agente e a integração no host oferecem suporte a ARM64 exclusivamente em ambientes Linux.
* Dashboards removidos da lista de ativos que são excluídos quando [o usuário os exclui](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/#delete-users).
* Adicionadas informações sobre novos recursos de usuário para a [interface do usuário do aplicativo não está respondendo (ANR)](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding/#view-anr-new-relic) na plataforma New Relic.
* [Diretrizes de elegibilidade](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/eligibility-guidelines-new-relic-nonprofit-program/#exceptions) atualizadas para o programa Observability for Good da New Relic.

### Pequenas mudanças

* Adicionadas referências alternativas de função NerdGraph ou NRQL para os exemplos da API REST v2.
* Atualizada a [compatibilidade e os requisitos para o agente Node.js.](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/)
* Atualizada a [compatibilidade e os requisitos para o agente Python](/docs/apm/agents/python-agent/getting-started/compatibility-requirements-python-agent/).
* Atualizar a [compatibilidade e os requisitos para o agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/).
* Adicionados links para as páginas introdutórias para [serviços que não são autorizados pelo FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp/#not-scope).
* Adicionada uma condição referente à [visibilidade do botão Carregar dSYM](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/upload-dsyms-bitcode-apps/#mobile-ui) na interface do usuário do New Relic.
* [Documentação](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-cosmos-db-document-db-monitoring-integration/#metric-data) atualizada para refletir que `Microsoft.DocumentDB/mongoClusters` não são suportados na integração de monitoramento do Azure Cosmos DB (Document DB).
