### Novos documentos

* [Integração com o FOSSA](/docs/vulnerability-management/integrations/fossa/) descreve como essa nova integração apoia suas iniciativas de segurança de aplicações ao exibir vulnerabilidades exploráveis em suas bibliotecas de código aberto.

### Grandes mudanças

* Reorganizamos a [categoria de documentação de infraestrutura](/docs/infrastructure/introduction-infra-monitoring/) e alinhamos as URLs com sua localização no repositório. Algum trabalho de bastidores que facilita encontrar e trabalhar nesta documentação.
* Uma reescrita significativa da [API do agente .NET](/docs/apm/agents/net-agent/net-agent-api/net-agent-api/) removeu informações obsoletas, reorganizou parte do conteúdo, atualizou o código de exemplo, adicionou mais detalhes sobre o que as APIs fazem, adicionou um link para nosso repositório de código de exemplo e adicionou tags de ID às APIs de rastreamento distribuído.

### Pequenas mudanças

* Adicionado o DynamoDB à [compatibilidade e requisitos do agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#database-core).
* A tag `hostStatus: shutdown` impede a abertura de um evento de alerta de perda de sinal. Atualizamos [criar condições de alerta NRQL](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/) e o [tutorial do NerdGraph sobre perda de sinal e preenchimento de lacunas](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/).
* Atualizamos os [dados padrão de monitoramento de infraestrutura](/docs/infrastructure/infrastructure-data/default-infra-data/) com nossa agregação de dados aprimorada.
* Adicionado um [aviso de descontinuação para Kubernetes](/eol/2024/10/deprecation-notice-v1.26-and-lower/).
* Legibilidade aprimorada em [criar condições de infraestrutura de &quot;host não reportando&quot;](/docs/infrastructure/infrastructure-alerts/create-infrastructure-host-not-reporting-condition/).
* Na [instrumentação sem camadas](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/enable-serverless-monitoring-aws-lambda-layerless/), foram adicionados exemplos para o Módulo ES e novos passos para instalar a Camada Extension Only.
* Reescrevemos a introdução da [introdução ao monitoramento de IA](/docs/ai-monitoring/intro-to-ai-monitoring/) e atualizamos as capturas de tela.
