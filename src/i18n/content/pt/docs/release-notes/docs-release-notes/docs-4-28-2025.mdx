### Grandes mudanças

* Adicionado o atributo [trace.id](/attribute-dictionary/?event=AjaxRequest&attribute=trace.id) ao evento `AjaxRequest` que atua como um identificador distinto para requests individuais, garantindo sua diferenciação entre limites interprocessuais e intraprocessuais.

* A seção de integração do AmazonQ foi removida da documentação do New Relic.

* Atualizou a [cláusula PREDICT](/docs/nrql/nrql-syntax-clauses-functions/##sel-prediction) com os comportamentos padrões.

* O agente de vídeo e todos os documentos correspondentes foram removidos da documentação do New Relic .

* Marcado o [tutorial do NerdGraph para canal de alerta de notificação](/docs/apis/nerdgraph/examples/nerdgraph-api-alert-notification-channels/) como obsoleto.

* Foi descontinuado o suporte para [cobertura de testes](/docs/infrastructure/infrastructure-agent/linux-installation/instrument-container/#test-coverage) de ponta a ponta (E2E) no sistema operacional utilizando o Control Group v1.

* Adicionado novo estado de investigação na [seção de alerta](/docs/alerts/alerts-ai-overview-page/).

### Pequenas mudanças

* Atualizou a [configuração do agente .Net](/docs/apm/agents/net-agent/configuration/net-agent-configuration/).

* O [arquivo de configuração do agente Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration/) foi atualizado para incluir &apos;variável de ambiente&apos; por meio das configurações.

* Seção [de downloads do agente](/docs/new-relic-solutions/get-started/networks/#agent-download) atualizada para acomodar a configuração de IP alterada do New Relic.

* Atualizado o PHP compatível do agente e os requisitos para [o Amazon DynamoDB](/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/#databases).

* Adicionado o Windows Server 2025 como sistema operacional suportado para o [agente de infraestrutura](/docs/infrastructure/infrastructure-agent/requirements-infrastructure-agent/).

* Atualizado o [relatório de compatibilidade do agente Node.js.](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)
