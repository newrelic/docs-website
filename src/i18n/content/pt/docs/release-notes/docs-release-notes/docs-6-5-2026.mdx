### Novos documentos

* Adicionada [documentação de integração do Elasticsearch com o OpenTelemetry](/docs/opentelemetry/integrations/elasticsearch/elasticsearch-otel-integration-overview/) abrangente, cobrindo a instalação para ambientes standalone e Kubernetes usando o New Relic OTel Collector e o Prometheus.
* Adicionada nova [documentação de integração do OTel Kafka para ambientes autogerenciados do Kubernetes](/docs/opentelemetry/integrations/kafka/kubernetes-self-managed/) junto com a documentação expandida para monitoramento do Kafka auto-hospedado com Prometheus.

### Grandes mudanças

* Documentação de permissões da Inteligência de Arquitetura de Serviços (SAI) atualizada para refletir que Catálogos e Mapas não exigem mais funções adicionais:

  * [Catálogo de aplicativos](/docs/service-architecture-intelligence/catalogs/applications-catalog/) — esclarecido que não são necessárias funções adicionais para acessar o catálogo de aplicativos.
  * [Catálogo de repositório](/docs/service-architecture-intelligence/catalogs/repository-catalog/) — esclarecido que não são necessárias funções adicionais para acessar o catálogo de repositório.
  * [Mapas avançados](/docs/service-architecture-intelligence/maps/advanced-maps/) — orientações de permissões atualizadas e detalhes de configuração expandidos.

* Atualizados os [requisitos de compatibilidade e permissão do AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/compatibility-requirement-lambda-monitoring/) com detalhes expandidos sobre as permissões do IAM exigidas para o monitoramento do Lambda, incluindo permissões adicionais necessárias para instrumentação e coleta de dados.

* Atualizamos a [documentação de instrumentação do SQS e do Lambda do agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-approaches-lambda/) para corrigir as limitações do distributed tracing e o comportamento da instrumentação do .NET Lambda.

* Adicionado um [vídeo tutorial à documentação de Monitoramento de Alterações do NerdGraph](/docs/change-tracking/config/nerdgraph/) explicando como configurar eventos de Monitoramento de Alterações.

* Adicionado o Ruby 3.4 como um runtime suportado na [documentação de instrumentação em camadas do AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/introduction-lambda/).

* Atualizamos a [documentação do conector AWS do New Relic Lens](/docs/new-relic-lens/set-up-connectors/) para documentar o prefixo `nrlens-` obrigatório para a nomenclatura de conectores.

* Atualizamos a [interface de gerenciamento de usuários e tarefas](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/) para substituir nomes de domínio de exemplo por valores de espaço reservado em conformidade.

### Pequenas mudanças

* Atualizados os [requisitos de compatibilidade do agente .NET](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/) com as versões compatíveis verificadas mais recentes para vários pacotes instrumentados, incluindo Microsoft.Azure.Cosmos, EnyimMemcachedCore e AWSSDK.DynamoDBv2.
* Atualizamos a página de [requisitos de compatibilidade do AI Monitoring](/docs/ai-monitoring/compatibility-requirements-ai-monitoring/) para direcionar os leitores à [página de compatibilidade do agente Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/) para obter a lista completa de bibliotecas de AI Monitoring suportadas.
* Atualizamos a [documentação de Inteligência de Custos na Nuvem do GCP](/docs/cci/get-started/gcp-cci/) para adicionar um link de navegação às páginas de configurações gerais e de integração para facilitar o acesso após a configuração.
* Atualizamos a [página de conformidade com o FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp/) para remover o New Relic eAPM da lista de serviços não autorizados pelo FedRAMP.
* Pequena melhoria no documento [migrar de API Polling para CWMS](/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/migrate-from-api-polling-to-cwms/).

### Notas de versão

Fique por dentro dos nossos últimos lançamentos:

* [Agente Node.js v14.1.0](/docs/release-notes/agent-release-notes/nodejs-release-notes/node-agent-14-1-0/):
