### Novos documentos

* [Dicas de redução de sobrecarga do agente PHP](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips/) lista configurações que você pode alterar para reduzir a sobrecarga do agente em seu sistema.

### Grandes mudanças

* Reorganizado e reescrito [Crie sua própria integração: New Relic Flex](/docs/infrastructure/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) para clareza e completude.
* Adicionada a seção [Scala Futures](/docs/apm/agents/java-agent/frameworks/scala-installation-java/#futures) na [instrumentação do Scala](/docs/apm/agents/java-agent/frameworks/scala-installation-java).
* Três tópicos sobre Gerenciamento de vulnerabilidades foram atualizados, incluindo [Introdução ao Gerenciamento de vulnerabilidades ](/docs/vulnerability-management/overview), [Entendendo a priorização de vulnerabilidades ](/docs/vulnerability-management/understanding-prioritization)e [Gerencie vulnerabilidades como uma equipe de segurança ](/docs/vulnerability-management/security-workflow).
* Adicionadas informações sobre como usar a mutação [`cancelAsyncQuery`](/docs/apis/nerdgraph/examples/async-queries-nrql-tutorial/#example-cancel-query).

### Pequenas mudanças

* Links atualizados para as imagens Python e Ruby da New Relic compatíveis com sua função Lambda.
* Destacada a natureza custosa das consultas de perda de sinal independentes em [Criar condições de alerta NRQL](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions).
* Atualizamos as [opções da CLI do New Relic](/docs/new-relic-solutions/build-nr-ui/newrelic-cli/#nr-cli-commands-options).
* Refinada a declaração do problema em [dicas de redução de sobrecarga do agente PHP](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips) para esclarecer a sobrecarga de desempenho.
* A descrição dos [limites de rastreamento](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works/#trace-limits) foi modificada para indicar que agora você pode visualizar esses limites na interface de limites.
* Atualizadas as descrições dos atributos `description` e `incidentId` nos [atributos de evento`NrAiIncident` ](/docs/alerts/create-alert/condition-details/alert-event-attributes/#NrAiIncident-attributes).
* [Métricas de nível de código](/docs/codestream/observability/code-level-metrics) atualizadas para refletir a nova extensão Ruby necessária.
* Atualizada a documentação de monitoramento do Kubernetes para indicar que a New Relic agora suporta o Kubernetes 1.31 em vez do 1.26.
* Adicionada uma entrada para a nova instrumentação de `AWSSDK.DynamoDBv2` na tabela de [mensagens](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#messaging-core).
* [Requisitos de compatibilidade](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent) atualizados para o agente Node.js.
* Esclarecido quando não usar a função `numeric()` em [Conversão de tipo](/docs/nrql/nrql-syntax-clauses-functions/#type-conversion)
* Adicionadas informações sobre um novo limite no registro de consultas de alerta em
