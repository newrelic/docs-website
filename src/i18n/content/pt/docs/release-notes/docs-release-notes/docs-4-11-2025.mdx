### Novo documento

* [AWS SQS fila de mensagens instrumentação](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues) suporta cabeçalhos distributed trace para mensagens SQS a partir da versão 2.1.0, com orientação sobre instrumentação personalizada para operações de recebimento de mensagens.
* [A ferramenta alerta Intelligence](/docs/agentic-ai/tools/alert-intelligence-tool) aprimora a capacidade de avaliar a saúde e o desempenho do aplicativo por meio da integração agente com nossos parceiros.
* [O streaming gerenciado do Confluent Cloud para Apache Kafka](/docs/infrastructure/other-infrastructure-integrations/confluent-cloud-integration) fornece orientação sobre o monitoramento do cluster e dos tópicos do Kafka, além de configurar alertas para o Confluent Cloud.
* [O resumoNew Relic AI para Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/nrai-summary) resume e mapeia os dados da trilha de eventos do Session Replay, permitindo uma melhor compreensão dos erros e acertos nas jornadas do usuário do browser.
* [A inteligência de resposta com a New Relic AI](/docs/alerts/alert-event-management/response-intelligence-ai) reduz o tempo de resolução de problemas e eventos de alerta ao utilizar a New Relic AI com inteligência de resposta.
* [Os logsBrowser ](/docs/browser/browser-monitoring/browser-pro-features/browser-logs/overview)capturam logs reais do console do browser para melhorar a observabilidade, simplificar a depuração e melhorar a experiência do usuário.
* [A introdução ao Transação 360](/docs/apm/transactions/workload-performance-monitoring/introdution) oferece uma abordagem revolucionária ao aplicativo de monitoramento e resolução de problemas, capturando e correlacionando automaticamente toda entidade e dados relacionados a problemas ou questões específicas.
* [AWS Logs in Context](/docs/logs/logs-context/aws-logs-in-context) fornece uma solução abrangente e integrada de gerenciamento de logs que melhora a eficiência da resolução de problemas e a observabilidade do sistema em ambientes AWS .
* Gerencie e reinstale sua instrumentação atual com [o agente Control](/docs/new-relic-control/agent-control/reinstrumentation).
* [O Agentic Control](/docs/new-relic-control/agent-control/overview) é um supervisor de agente leve que gerencia outras distribuições New Relic e New Relic do agente OpenTelemetry , comunicando-se com o Controle de Agentes para configurar, atualizar e monitorar remotamente a integridade da instrumentação em nível de infraestrutura.
* [O controlePipeline ](/docs/new-relic-control/pipeline-control/overview)permite que você gerencie o fluxo de dados da sua entidade monitorada para New Relic.
* [Docker contêiner](/docs/infrastructure/host-integrations/host-integrations-list/docker-container-monitoring-integration) integração monitora e reporta métricas do Docker contêiner para New Relic.
* Funções de [monitoramento de respostaLambda ](/docs/serverless-function-monitoring/aws-lambda-monitoring/response-stream)no aplicativo Node.js para obter insights reais sobre o desempenho e o comportamento de cada invocação.

### Grandes mudanças

* Adicionados [Amazon CloudWatch Metric Streams](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams), [ElastiCache (Memcached)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached) e [Amazon MQ (ActiveMQ)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) como [relacionamentos de entidade suportados](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships) para OpenTelemetry.

* Adicionado Amazon CloudWatch Metric Streams com [Amazon Managed Kafka (MSK)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-managed-kafka-msk) para gerar entidade MSK no New Relic. Essas entidades são marcadas com `aws.clusterName` ou `aws.kafka.ClusterName`, `aws.awsRegion` ou `aws.region` e `aws.topic` ou `aws.kafka.Topic`.

* Adicionadas instruções adicionais [Solr resolução de problemas](/docs/apm/agents/java-agent/troubleshooting/java-solr-troubleshooting) para as alterações métricas na versão 8.10.

* A New Relic descontinuou o uso dos seguintes intervalos de IP:

  * 3.145.244.128/25
  * 3.77.79.0/25
  * 3.27.118.128/25
  * 20.51.136.0/25
  * 4.197.217.128/25
  * 18.246.82.0/25
  * 158.177.65.64/29
  * 159.122.103.184/29
  * 161.156.125.32/28

  A partir de 1º de maio de 2025, esses intervalos podem ser realocados pelo provedor cloud para outros clientes para fins além do nosso controle. Atualize sua configuração [de rede](/docs/new-relic-solutions/get-started/networks) adequadamente.

* Adicionado um documento de resolução de problemas para [recuperar a chave de licença New Relic ](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/retrieve-license)para problemas usando AWS Secrets Manager com New Relic.

* Adicionado um documento de resolução de problemas para [a função do Lambda que não aparece como instrumentado](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-not-instrumented-lambda-function).

* Adicionado um documento de resolução de problemas para problemas [ao adicionar várias contas AWS com respectivas regiões](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-unable-to-add-multiple-aws-accounts/).

* Adicionado um documento de resolução de problemas para problemas [ao usar AWS Secrets Manager com New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-using-aws-secrets-manager/).

* Adicionada integração New Relic Agentic ao [FedRAMP](/docs/security/security-privacy/compliance/certificates-standards-regulations/fedramp).

* o rastreamento distribuído para envio de mensagens é compatível com a versão 2.1.0 para o que há de mais recente no [framework e biblioteca](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#frameworks) para monitoramento Java .

* Adicionados casos excepcionais onde tanto [externos legados quanto GUIDs não instrumentados](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/#handling-legacy-externals-and-uninstrumented-entities-exceptions-in-entity-relationships) podem aparecer.

* A partir de segunda-feira, 3 de março de 2025, nossa [integração com o Kubernetes](/eol/2025/03/deprecation-notice-v1.27-and-lower) não oferecerá mais suporte ao Kubernetes v1.27 e versões anteriores.
