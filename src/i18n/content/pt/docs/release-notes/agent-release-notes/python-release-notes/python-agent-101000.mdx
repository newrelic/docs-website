## Notas

Esta versão do agente Python adiciona suporte para async [elasticsearch](https://pypi.org/project/elasticsearch/), corrige um problema de análise de configuração do graphql, corrige um bug na instrumentação [Kombu](https://pypi.org/project/kombu/) e reativa a instrumentação Kombu.

Instale o agente usando `easy_install/pip/distribute` por meio do [Índice de Pacotes Python](https://pypi.python.org/pypi/newrelic) ou baixe-o diretamente do [site de download do New Relic](https://download.newrelic.com/python_agent/release).

## Novo recurso

* Adicionar suporte assíncrono para elasticsearch

  * Adiciona suporte para [elasticsearch](https://pypi.org/project/elasticsearch/) assíncrono. Obrigado [fagnerzulin](https://github.com/fagnerzulin) pela contribuição!

## Correções de bugs

* Adicionar análise da configuração do graphql

  * Anteriormente, quando a configuração `instrumentation.graphql.capture_introspection_queries` era definida no arquivo de configuração, essa configuração não entrava em vigor. Isso foi corrigido.

* Corrigir travamento no Kombu ao executar com Sentry

  * Corrige uma falha na instrumentação [do Kombu](https://pypi.org/project/kombu/) onde um `TypeError` era gerado pelo módulo `inspect` . Esse bug ocorreu devido a um conflito de encapsulamento entre New Relic e Sentry. A instrumentação Kombu também foi reativada.
