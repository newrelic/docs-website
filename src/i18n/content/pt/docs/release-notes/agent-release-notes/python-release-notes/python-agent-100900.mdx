## Notas

Esta versão do agente Python adiciona nomes de atributos de span de armazenamento de dados compatíveis [OpenTelemetry](https://github.com/open-telemetry/semantic-conventions/blob/v1.24.0/docs/database/database-spans.md) , corrige um bug na instrumentação [LangChain](https://pypi.org/project/langchain/) e desativa temporariamente a instrumentação [Kombu](https://pypi.org/project/kombu/) .

Instale o agente usando `easy_install/pip/distribute` por meio do [Índice de Pacotes Python](https://pypi.python.org/pypi/newrelic) ou baixe-o diretamente do [site de download do New Relic](https://download.newrelic.com/python_agent/release).

## Novo recurso

* Adicionar armazenamento de dados compatível com OpenTelemetry span atributo

  * O atributo de armazenamento de dados span agora é mais consistente com as convenções de nomenclatura do [OpenTelemetry](https://github.com/open-telemetry/semantic-conventions/blob/v1.24.0/docs/database/database-spans.md)

## Correções de bugs

* Corrigir instrumentação LangChain para tipos de resposta de string

  * Anteriormente, quando [o LangChain](https://pypi.org/project/langchain/) retornava respostas do tipo string, o agente capturava apenas o primeiro caractere da string. Isso foi corrigido.

* Desativar instrumentação Kombu

  * A instrumentação [Kombu](https://pypi.org/project/kombu/) existente no agente está travando com um `TypeError` sendo gerado pelo módulo `inspect` . Esta instrumentação foi temporariamente desabilitada e será reabilitada em uma versão futura assim que a causa raiz da falha for identificada e corrigida.
