## 3.42.0

### Adicionado

* Adicionados `ConfigTransactionEventsMaxSamplesStored`e `ConfigErrorCollectorMaxSamplesStored` permitindo controle total do máximo de amostras armazenadas para evento de transação, evento Insights Personalizados, evento de Erro e evento de log
* Adicionado suporte para a propriedade `MultiValueHeaders` ao extrair os cabeçalhos de `events.APIGatewayProxyResponse` em nrlambda.
  * Obrigado ao membro da comunidade @rittneje por contribuir para esta solução

### Fixo

* Variáveis não utilizadas foram removidas e a integração foi modernizada substituindo a interface por qualquer outra na integração `nrpxg5`
* Corrigido um bug em que o evento de erro não marcava corretamente o erro esperado.
  * Agradecemos ao membro da comunidade @driimus por sua contribuição para esta solução.
* Integração `nrwriter` para usar v1.0.2
  * Agradecemos ao membro da comunidade @hiicharm por ter notado isso.

### Declaração de apoio

Utilizamos a versão mais recente da linguagem Go. No mínimo, você não deve usar nenhuma versão do Go anterior àquela suportada pela própria equipe do Go. Consulte a [Política de Fim de Vida do Agente Go](/docs/apm/agents/go-agent/get-started/go-agent-eol-policy/) para obter detalhes sobre as versões suportadas do agente Go e componentes de terceiros.
