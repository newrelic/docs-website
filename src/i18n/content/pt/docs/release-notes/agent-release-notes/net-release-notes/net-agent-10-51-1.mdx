### Perceber

* Clientes que usam `RecordDatastoreSegment()` com um nome de fornecedor personalizado verão os nomes das métricas mudarem de `Datastore/*/Other/*` para `Datastore/*/<VendorName>/*`. Atualize quaisquer dashboards, alertas ou consultas que referenciem os antigos nomes de métrica baseados em Other. ([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))

### Correções

* `RecordDatastoreSegment()` A API usará corretamente o nome do fornecedor fornecido pelo chamador em vez de ignorá-lo. ([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))
* Garantir que todos os segmentos do StackExchange.Redis sejam criados e adicionados à transação antes de ser coletada ([#3574](https://github.com/newrelic/newrelic-dotnet-agent/issues/3574)) ([4152ea2](https://github.com/newrelic/newrelic-dotnet-agent/commit/4152ea25b23318c68d372447c905c622ac5ac868))
