### Novo recurso

* Adicione um novo amostrador de rastreamento distribuído, o TraceId Ratio-Based Sampler, que é baseado na [implementaçãoOpenTelemetry ](https://github.com/open-telemetry/opentelemetry-dotnet/blob/main/src/OpenTelemetry/Trace/Sampler/TraceIdRatioBasedSampler.cs). Este amostrador toma sua decisão de amostragem usando o TraceId (um GUID aleatório) e uma taxa de amostragem configurada (uma porcentagem de traço a ser amostrada). Para mais informações, consulte a [especificação OpenTelemetry](https://opentelemetry.io/docs/specs/otel/trace/tracestate-probability-sampling/). ([#3211](https://github.com/newrelic/newrelic-dotnet-agent/issues/3211)) ([63ca603](https://github.com/newrelic/newrelic-dotnet-agent/commit/63ca603ca6cb86110431413c03afbf8a8c8e9367))

### Correções

* A análise da string de conexão MSSQL pode lançar exceções e desabilitar o armazenamento de dados instrumentação ([#3263](https://github.com/newrelic/newrelic-dotnet-agent/issues/3263)) ([cdf000b](https://github.com/newrelic/newrelic-dotnet-agent/commit/cdf000b73fe4beaa73fa4bfe3d338f8eb6698cf1))
* Resolva problemas com a instrumentação &quot;Consume&quot; do Kafka para garantir que a instrumentação automática funcione em conjunto com a instrumentação personalizada. ([#3257](https://github.com/newrelic/newrelic-dotnet-agent/issues/3257)) ([1dbd1c5](https://github.com/newrelic/newrelic-dotnet-agent/commit/1dbd1c51aeaf6e2909d23b078029ee78b721b3bc))

### Atualizando seu agente

* Siga os procedimentos padrão para [atualizar o agente .NET](/docs/agents/net-agent/installation-configuration/update-net-agent).
* Se você estiver usando um agente particularmente antigo, revise a lista de principais alterações e procedimentos para [atualizar o legado .NET agente](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net).

Recomendamos atualizar para a versão mais recente do agente assim que estiver disponível. Se você não puder atualizar para a versão mais recente, atualize seu agente para uma versão com no máximo 90 dias.
