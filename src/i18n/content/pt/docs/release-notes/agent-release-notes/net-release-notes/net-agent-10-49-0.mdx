### Novo recurso

* O Agente Híbrido .NET fornece uma solução de observabilidade preparada para o futuro com suporte integrado para as APIs de Tracing e Métricas do OpenTelemetry. Os logs do OpenTelemetry continuarão sendo suportados por meio da integração existente do Microsoft.Extensions.Logging. Esta versão traz recursos críticos do OpenTelemetry, incluindo Span Links, Eventos em Spans e algoritmos de amostragem unificados. Ele foi projetado para preencher a lacuna para equipes .NET que buscam adotar os padrões OpenTelemetry, mantendo a retrocompatibilidade com seus dashboards existentes do New Relic. ([72a0414](https://github.com/newrelic/newrelic-dotnet-agent/commit/72a0414d49273d412cddc11dcb329b30680aa8f0))

### Atualizando seu agente

* Siga os procedimentos padrão para [atualizar o agente .NET](/docs/agents/net-agent/installation-configuration/update-net-agent).
* Se você estiver usando um agente particularmente antigo, revise a lista de principais alterações e procedimentos para [atualizar o legado .NET agente](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net).

Recomendamos atualizar para a versão mais recente do agente assim que estiver disponível. Se você não puder atualizar para a versão mais recente, atualize seu agente para uma versão com no máximo 90 dias. Saiba mais sobre [como manter seu agente atualizado](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent).
