## Novos recursos e melhorias

* Novos tipos de eventos introduzidos

  * **VideoAction**: Este tipo de evento representa ações padrão relacionadas à reprodução e interação de vídeo.
  * **VideoErrorAction**: Este tipo de evento captura erros que ocorrem durante a reprodução de vídeo, fornecendo insights detalhados para resolução de problemas.
  * **VideoAdAction**: tipo de evento dedicado para rastrear ações relacionadas a anúncios em vídeo, garantindo melhor granularidade e relatórios.
  * **VideoCustomAction**: Um tipo de evento flexível para ações personalizadas e interação dentro do aplicativo de vídeo.
  * **ConnectedDeviceSystem**: Este tipo de evento monitora eventos gerais de nível de sistema.

## Depreciações

* RokuVideo
  * O tipo de evento `RokuVideo` foi descontinuado. O usuário deve fazer a transição para o uso dos tipos de eventos mais recentes que oferecem recursos de rastreamento aprimorados e especificidade.
* Sistema Roku
  * O tipo de evento `RokuSystem` foi descontinuado. O usuário deve fazer a transição para o uso dos tipos de eventos mais recentes que oferecem recursos de rastreamento aprimorados e especificidade.

## Novo atributo introduzido

* Introduzimos um conjunto de novos atributos para melhorar o rastreamento e a análise de dados. Esses atributos fornecem contexto e detalhes adicionais para os novos tipos de eventos. Consulte a [documentação do modelo de dados](https://github.com/newrelic/video-agent-roku/blob/4.0.0/DATAMODEL.md) para obter mais detalhes.

## Impacto e ações necessárias

* **Transição do RokuVideo**: os usuários que atualmente rastreiam `RokuVideo` eventos devem adaptar suas implementações para utilizar os novos tipos de eventos introduzidos. Essa mudança proporciona maior especificidade e escopo no rastreamento de ações em interações de vídeo.
* **Atualizações de implementações**: os integradores devem atualizar seus sistemas para aproveitar os novos atributos introduzidos com o lançamento, que fornecem contexto e detalhes mais ricos para o rastreamento de eventos.
* **Teste e validação**: recomendamos testar a implementação de novos tipos de atributos e eventos para garantir compatibilidade e transmissão correta de dados.
