<Callout variant="tip">
  Este guia é para uma migração única de versões anteriores à 3.xx para a versão 3.xx. Se você já está na versão 3.xx e precisa atualizar para uma versão mais recente, siga os procedimentos padrão de atualização para a sua plataforma.
</Callout>

Para tornar a migração de versões anteriores o mais fácil possível, desenvolvemos uma camada de compatibilidade que traduz a maioria das opções de configuração para seus novos equivalentes. Esta camada de compatibilidade é temporária e será removida no futuro. Recomendamos que você leia este guia cuidadosamente e migre a configuração sob supervisão humana.

## Migração para a versão 3.xx

Para experimentar o novo vista customizada, você deve migrar para a versão 3.xx do agente de mídia.

### Complicações do tipo evento duplo

Existe a possibilidade de que, após a migração, alguns dos seus usuários finais não tenham optado pela atualização do aplicativo, ou muito poucos o tenham feito. Durante esse período intermediário, você poderá experimentar tipos de eventos duplos, o que pode ser complicado. Para gerenciar isso:

* Reúna todas as consultas e mapeamentos de ações.
* Informe ao seu usuário final que durante esse período intermediário, ele poderá ver dados em vários eventos e como eles estão relacionados.

<Callout variant="important">
  Isso se aplica ao agente móvel. O browser será atualizado automaticamente.
</Callout>

### Visualizando dados pós-migração

Após a migração, se você quiser ver os dados do mês anterior, recomendamos que você use algum dashboard personalizado ou consulte:

```sql
SELECT * FROM pageAction
```

<Callout variant="important">
  Com definições de eventos nomeados, eles perdem a capacidade dashboard de consultar os dados antigos.
</Callout>

### evento Definição de mapeamento

O mapeamento de tipo de evento entre as versões antigas e novas é específico do cliente com base na sua implementação. Para garantir uma transição tranquila e manter a integridade dos dados, entre em contato com o suporte ao cliente da New Relic para um plano de migração personalizado.

Recomendamos que você siga cuidadosamente este guia e garanta que todas as configurações sejam migradas sob supervisão. A camada de compatibilidade é uma solução temporária e é importante fazer a transição para a nova configuração o mais rápido possível.
