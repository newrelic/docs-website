Quando você configura os dashboards <DNT>**Video Overview**</DNT> ou <DNT>**Ad Overview**</DNT>, a New Relic salva suas configurações no seu perfil de usuário. Na próxima vez que você retornar a qualquer dashboard, sua configuração será restaurada para que você não precise começar do zero.

As preferências são salvas por usuário e por tipo de dashboard. Suas configurações de <DNT>**Video Overview**</DNT> nunca afetam suas configurações de <DNT>**Ad Overview**</DNT>, e vice-versa.

## Como as preferências são salvas [#how-saving-works]

As alterações são gravadas de forma assíncrona em segundo plano para que o dashboard permaneça responsivo durante o salvamento. Para reduzir gravações desnecessárias quando você faz ajustes rápidos, os salvamentos são agrupados brevemente em lotes.

As preferências salvas persistem em:

* Atualizações do Browser
* Logout e login
* Browsers ou dispositivos diferentes, desde que você esteja conectado com a mesma conta

## Comportamento padrão [#defaults]

Se você não tiver preferências salvas para um dashboard (por exemplo, em sua primeira visita), o dashboard será aberto com a configuração padrão do produto.

## Redefinir para os padrões [#reset-to-defaults]

Um controle <DNT>**Reset to default view**</DNT> permite limpar suas preferências salvas e retornar o dashboard aos padrões originais do produto.

<Callout variant="tip">
  A redefinição afeta apenas o dashboard que você está visualizando no momento. Redefinir as preferências de <DNT>**Video Overview**</DNT> não afeta suas preferências de <DNT>**Ad Overview**</DNT>.
</Callout>

## Tratamento de erros [#error-handling]

Se a recuperação de preferência falhar quando o dashboard carregar, o dashboard retornará à configuração padrão do produto e exibirá uma notificação de erro. Seu fluxo de trabalho não é bloqueado.

Se o salvamento de uma preferência falhar, o dashboard continuará funcionando normalmente.

Se uma métrica, dimensão ou faceta salva anteriormente não estiver mais disponível (por exemplo, porque foi descontinuada), o dashboard a substitui por um padrão razoável para essa configuração, como a primeira opção disponível, para que nenhum widget fique vazio ou quebrado.
