## Detectar automaticamente os logs do browser [#automatic-log]

O logging automático coleta mensagens de log emitidas pelo browser `console` para ajudar você a maximizar a observabilidade do seu aplicativo frontend.

Os logs do navegador são rastreados por padrão no nível `WARN` para os agentes <DNT>Pro</DNT> e <DNT>Pro+SPA</DNT>, mas não estão disponíveis para o agente Lite browser. Recomendamos que você primeiro confirme se está usando o agente browser <DNT>**Pro**</DNT> ou <DNT>**Pro + SPA**</DNT>.

<Callout variant="important">
  Certas configurações, como ativar ou desativar a coleta de logs do navegador e ajustar as taxas de amostragem ou a verbosidade do log, são configurações do lado do servidor. Depois de adicionar o snippet do agente às suas páginas, você não precisa reimplantar o script para alterar essas configurações. No entanto, as configurações do lado do servidor não têm efeito se o recurso não estiver incluído no script implantado. Isso se aplica à versão `1.259.0` ou posterior do agente do navegador.
</Callout>

## Ativar a coleta automática de logs (para aplicativos de navegador existentes) [#enable-automatic-log]

Melhore a observabilidade e os recursos de depuração do seu aplicativo seguindo estas instruções simples de configuração.

<Steps>
  <Step>
    Acesse **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Selecione seu aplicativo de browser.
  </Step>

  <Step>
    No menu à esquerda, clique em **Application settings**.
  </Step>

  <Step>
    Certifique-se de que o agente browser <DNT>**Pro**</DNT> ou <DNT>**Pro + SPA**</DNT> esteja selecionado.
  </Step>

  <Step>
    Alternar **Automatically collect console logs** para **ON** para coletar mensagens enviadas ao console do navegador. <br />Alternar **Collect logs manually** para **ON** para coletar logs enviados programaticamente via `newrelic.log()` ou `newrelic.wrapLogger()`.
  </Step>

  <Step>
    Edite a taxa de amostragem para especificar a porcentagem de sessões de usuário para coletar logs. Você pode inserir um valor entre 0 e 100.

    <Callout variant="important">
      A taxa de amostragem padrão para logging do navegador é definida como 100% das sessões do usuário. Você pode definir essa taxa de amostragem da sessão entre `0%` e `100%`. Por exemplo, definir a taxa de amostragem da sessão como `50%` resultará na coleta de eventos de log de aproximadamente metade das sessões de usuário selecionadas aleatoriamente.
    </Callout>
  </Step>

  <Step>
    Selecione o nível de verbosidade do log do navegador desejado. O nível de verbosidade padrão é definido como **WARN**. Os níveis que você pode escolher são:

    * `ERROR` - coletar logs com nível `ERROR`
    * `WARN` - coletar logs com nível `WARN` ou `ERROR`
    * `INFO` - coletar logs com nível `INFO`, `WARN` ou `ERROR`
    * `DEBUG` - coletar logs com nível `DEBUG`, `INFO`, `WARN` ou `ERROR`
    * `TRACE` - coletar logs com nível `TRACE`, `DEBUG`, `INFO`, `WARN` ou `ERROR`

    <br />

    <Callout variant="important">
      Os dados passados pelos métodos do console podem passar por serialização e [ofuscação](/docs/browser/new-relic-browser/configuration/obfuscate-browser-agent-data). Dependendo do tamanho e da frequência, isso pode impactar negativamente o desempenho do aplicativo, bem como os custos de dados. Em geral, NÃO é recomendado passar objetos grandes ou grandes quantidades de dados para métodos de console.
    </Callout>
  </Step>
</Steps>

## Desativar logs do browser [#disable-browser-logs]

Para desabilitar os logs do browser para um aplicativo de browser existente:

<Steps>
  <Step>
    Acesse **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Selecione seu aplicativo de browser.
  </Step>

  <Step>
    No menu à esquerda, clique em **Application settings**.
  </Step>

  <Step>
    Alternar **Automatically collect console logs** e **Collect logs manually** para **OFF**.
  </Step>
</Steps>

## Consumo de dados

Os logs seguem o mesmo preço de consumo dos outros bytes do seu browser. A quantidade de bytes produzidos depende da contagem e do comprimento das mensagens.

O recurso de logging automático elimina a necessidade de chamar a API do browser `newrelic.log` ou `newrelic.wrapLogger`, exceto ao adicionar atributo personalizado ao registro de evento.
