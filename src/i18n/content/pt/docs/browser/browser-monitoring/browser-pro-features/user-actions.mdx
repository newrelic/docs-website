As ações do usuário no monitoramento do browser ajudam você a entender o comportamento do usuário, como onde o usuário clica, rola e interage com seu aplicativo da web. Você pode usar ações do usuário para vincular ações, ajudando a obter insights sobre onde seu aplicativo da web cria atrito e aumenta a frustração do usuário.

As ações do usuário são rastreadas por padrão no Pro e Pro+SPA Agent, mas não estão disponíveis no Lite browser Agent. Recomendamos que você primeiro confirme se está usando o agente Pro ou Pro+SPA:

Vá para <DNT>**[one.newrelic.com &gt; All Capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT> (selecione seu aplicativo de browser) &gt; <DNT>**Application settings**</DNT>. Nesta página, verifique se o browser do agente <DNT>**Pro**</DNT> ou <DNT>**Pro + SPA**</DNT> está selecionado. Certifique-se de que você está na versão 1.268.0 ou superior.

## Ver ações do usuário [#view-events]

Quando um usuário carrega uma página da web, nosso browser agente configura ouvintes de eventos para o evento de interação chave. Quando o agente do browser observa essas ações, ele pode coletar o evento gerado e encaminhá-lo para New Relic.

As ações do usuário podem colher estes eventos:

* auxclick
* desfoque (somente janela)
* clique
* cópia
* foco (somente janela)
* tecla pressionada
* colar
* fim da rolagem

Para visualizar as ações do usuário capturadas pelo agente do browser, crie uma consulta NRQL para o tipo de evento `UserAction` no [criador de consulta](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/). Aqui está um exemplo de uma consulta básica:

```sql
FROM UserAction 
SELECT *
```

Por padrão, os dados de eventos do browser são armazenados por oito dias, mas a retenção real de dados depende da sua conta. Para mais detalhes sobre ações do usuário, consulte o [Dicionário de dados](/attribute-dictionary/).

## Consumo de dados [#data-consumption]

As ações do usuário são agregadas sequencialmente, o que significa que se várias ações forem executadas em rápida sucessão no mesmo destino DOM , elas serão agrupadas como uma única ação do usuário. Isso mantém os custos de consumo baixos e ajuda você a entender como os usuários interagem com seu aplicativo web.

As ações do usuário seguem o mesmo preço de consumo dos outros bytes do seu browser. A quantidade de bytes produzidos depende da contagem, duração e níveis de atividade do usuário nas sessões.
