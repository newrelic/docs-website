Nosso [<InlinePopover type="browser"/>](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser)fornece insights sobre como seu aplicativo ou site se comporta quando é carregado em um browser da web. O browser registra apenas dados de desempenho, conforme explicado neste documento. Ele **não** registra nenhum dado usado ou armazenado pelo aplicativo monitor, a menos que você o configure explicitamente para fazer isso.

Para obter mais informações sobre as medidas de segurança da New Relic, consulte nossa [documentação de segurança e privacidade](/docs/using-new-relic/new-relic-security/security/security-matters-data-privacy-new-relic) ou visite o [site de segurança da New Relic](https://newrelic.com/security).

## Dados relatados [#data]

O browser relata muitos tipos diferentes de dados para ajudá-lo a analisar o desempenho do seu site. Ele <DNT>**only**</DNT> reporta dados de visualização de página, a menos que você tenha se inscrito no [Pro recurso](http://newrelic.com/browser-monitoring/pricing). Você também pode [ativar a funcionalidade](/docs/browser/new-relic-browser/welcome-new-relic-browser/new-relic-browser#enhanced-features) para solicitações AJAX, erros de JavaScript e rastreamento da sessão.

Para a maioria dos tipos de dados, o browser [transmite os dados de forma segura](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-page-load-timing#transmission) usando criptografia HTTPS. O agente browser transmite dados para [o coletor](/docs/accounts-partnerships/getting-started-new-relic/glossary#collector) da New Relic usando o domínio `bam.nr-data.net` ou `bam-cell.nr-data.net`.

Aqui está um resumo dos tipos de dados reportados pelo monitoramento de Browser.

<CollapserGroup>
  <Collapser
    id="page-view-data"
    title="Dados de visualização de página"
  >
    Esses dados são relatados uma vez por visualização de página e consistem em:

    * Dados de tempo de carregamento da página

    * Nome do controlador de aplicativo do lado do servidor que atendeu a página, se disponível (ofuscado na página e durante a transmissão)

    * Parâmetro personalizado adicional definido pelo controlador de aplicativo do lado do servidor, se disponível (ofuscado na página e durante a transmissão)

    * Parâmetro personalizado adicional definido pela [API do agente browser](/docs/browser/new-relic-browser/browser-agent-apis/report-data-events-browser-agent-api), se definido antes do carregamento da página

      Essas informações aparecem na [página<DNT>**Page views**</DNT> ](/docs/browser/new-relic-browser/dashboard-details/page-views-dashboard). Por motivos de segurança de dados, o browser não registra nem coleta strings de consulta de URL.

      Os dados do lado do servidor só podem ser coletados quando o host também é instrumento da New Relic e a instrumentação de monitoramento do Browser é injetada pelo agente. Para obter mais informações sobre como coletamos e apresentamos esses dados, consulte [instrumentação para tempo de carregamento da página](/docs/browser/new-relic-browser/page-load-timing/instrumentation-page-load-timing).
  </Collapser>

  <Collapser
    id="ajax-data"
    title="Dados de tempo AJAX"
  >
    Quando ativado, o browser reporta periodicamente dados de tempo AJAX até que o usuário saia ou feche a página. (O New Relic filtra automaticamente todas as solicitações AJAX que demoram mais de dois minutos.) Os dados incluem:

    * Nome do host, portas e caminhos (mas não parâmetro de pesquisa/consulta) de URLs de solicitação AJAX

    * Código de status HTTP das respostas

    * Tamanho em bytes do corpo da mensagem de solicitação

    * Nome do controlador de aplicativo do lado do servidor que atende a solicitação AJAX e os dados de temporização do lado do servidor (ofuscados na página e durante a transmissão), quando a instrumentação do browser é injetada pelo agente New Relic

    * Dados de tempo para a transação AJAX

    * Dados de tempo para o retorno de chamada AJAX

      Essas informações aparecem na [página<DNT>**AJAX**</DNT> ](/docs/browser/new-relic-browser/browser-pro-features/ajax-dashboard-identifying-time-consuming-calls).
  </Collapser>

  <Collapser
    id="js-errors-data"
    title="Dados de erro de JavaScript"
  >
    Quando ativado, o browser reporta periodicamente dados sobre cada erro que ocorre na página até que o usuário saia ou feche a página. Essas informações aparecem na [página<DNT>**JavaScript errors**</DNT> ](/docs/browser/new-relic-browser/browser-pro-features/js-errors-dashboard-examining-errors-over-time).

    Para cada erro, os dados incluem:

    * Classe de exceção do erro

    * Mensagem de erro contendo texto arbitrário

    * Stack trace do erro, que pode conter nomes de funções e URLs do script que causa o erro

      Mensagem de erro normalmente não contém nenhuma informação confidencial ou sensível. No entanto, é possível que as mensagens sejam construídas propositalmente com informações confidenciais. Antes de ativar o relatório de erros de JavaScript, certifique-se de que seu site não exponha nenhuma informação sensível na mensagem de erro.
  </Collapser>

  <Collapser
    id="session-trace-data"
    title="Rastrear dados da sessão"
  >
    Quando ativado, o browser relata periodicamente dados sobre os detalhes do ciclo de vida de uma única página, incluindo interação do usuário, carregamentos de AJAX e erros de JavaScript, até que o usuário saia ou feche a página. O New Relic interrompe automaticamente a gravação de mais dados após dez minutos. Os dados incluem:

    * Detalhes do tempo de carregamento de ativos

    * Interação do usuário, como rolagem, mouse e clique

    * Tempo de erro de JavaScript e outras informações de erro de JavaScript

    * Evento Javascript disparado

      Trace da sessão são capturados aleatoriamente a uma taxa fixa entre as visualizações de página do monitor. as informações de rastreamento da sessão aparecem na [página<DNT>**Session traces**</DNT> ](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle).
  </Collapser>

  <Collapser
    id="spa-data"
    title="Dados do SPA"
  >
    Se você usar o monitoramento de aplicativo de página única (SPA) do browser, o New Relic reportará os seguintes dados uma vez por carregamento de página ou alteração de rota.

    * Dados do browser para [visualizações de páginas](#page-view-data), [tempo AJAX](#ajax-data), [erros de JavaScript](#js-errors-data) e [rastreamento da sessão](#session-trace-data)

    * Aplicar fragmentos de hash associados a alterações de rota SPA

    * Parâmetro personalizado adicional adicionado da [API SPA](/docs/browser/new-relic-browser/browser-agent-apis/new-relic-spa-api)

      Quando o monitoramento do SPA for ativado, essas informações aparecerão na [página<DNT>**Page views**</DNT> ](/docs/browser/single-page-app-monitoring/browser-ui/view-spa-data-new-relic-browser).

      Os dados do lado do servidor só podem ser coletados quando o host também é instrumento da New Relic, e a instrumentação de monitoramento do Browser é injetada pelo agente. Para obter mais informações sobre como coletamos e apresentamos esses dados, consulte [instrumentação para tempo de carregamento da página](/docs/browser/new-relic-browser/page-load-timing/instrumentation-page-load-timing).
  </Collapser>

## Sequências de consulta de URL [#url-strings]

O agente browser usa o atributo HTTP `referer` para rastrear URLs de páginas. Às vezes, os URLs podem conter dados de consulta potencialmente confidenciais inseridos pelo usuário (por exemplo, o nome de um usuário). Por motivos de segurança de dados, o browser não registra nem coleta strings de consulta de URL.

## Endereço IP do visitante [#visitors-ip]

O browser usa o endereço IP do visitante para enriquecer os dados para segmentação adicional do visitante. Detalhes como [`ASN` e `geoID`](/docs/browser/new-relic-browser/browser-pro-features/filterable-geography-webpage-metrics-location#metrics) são mapeados para dados do browser a partir do endereço IP. Por motivos de segurança de dados, o browser não retém o endereço IP do visitante para geração de relatórios. O endereço IP é obtido no cabeçalho HTTP da solicitação ao [coletor](/docs/accounts-partnerships/getting-started-new-relic/glossary#collector) New Relic.

A New Relic não retém o endereço IP do visitante após o mapeamento do atributo. O valor do endereço IP é substituído dentro de 24 horas após a coleta dos dados.

## Tipos de browser [#types]

Browser determina o tipo de browser a partir do cabeçalho `User-Agent` e a localização geográfica com base no endereço IP do browser. A New Relic não retém o endereço IP, apenas o país e a região associados aos dados de desempenho.

Essas informações aparecem na [página<DNT>**Geography**</DNT> ](/docs/browser/new-relic-browser/additional-standard-features/geography-webpage-performance-location)do aplicativo selecionado. Além disso, detalhes sobre tipos específicos de browser aparecem na [página<DNT>**Browsers**</DNT> ](/docs/browser/new-relic-browser/additional-standard-features/browsers-problem-patterns-type-or-location)do aplicativo selecionado.

## Acesso CDN [#traces]

O tempo de carregamento da página requer acesso à rede de entrega de conteúdo (CDN), onde o arquivo JavaScript do utilitário New Relic (`nr.js`) está hospedado. O nome de domínio do arquivo (`js-agent.newrelic.com`) permanece estático, mas o número no caminho (versão) pode mudar periodicamente.

Uma tag `script` é injetada pelo agente New Relic (ou colada na página da Web para aplicativos independentes) que faz referência ao JavaScript na CDN, que é então carregada pelo browser. O JavaScript carregado coleta e reporta a métrica dinamicamente para qualquer um dos domínios `bam.nr-data.net` ou `bam-cell.nr-data.net`.

<Callout variant="important">
  Se o seu usuário final estiver atrás de um firewall ou proxy e não tiver acesso ao CDN ou às [redes](/docs/apm/new-relic-apm/getting-started/networks) da New Relic (incluindo `bam.nr-data.net` e `bam-cell.nr-data.net`), o monitoramento do browser não funcionará.
</Callout>

## Biscoitos

<Callout variant="important">
  Para o agente atual (versão 1217 e superior), o uso de cookies foi descontinuado.
</Callout>

O monitoramento de Browser cria [cookies](/docs/browser/new-relic-browser/page-load-timing-resources/new-relic-cookies) no browser do usuário final. Se o usuário tiver os cookies desabilitados, o tempo de carregamento da página (às vezes chamado de monitoramento de usuário real ou RUM) não será capaz de rastrear as sessões adequadamente. Além disso, se o usuário tiver um browser mais antigo que não oferece suporte à [API de especificação de tempo de navegação](/docs/browser/new-relic-browser/browser-agent-apis/navigation-timing-api), o tempo de carregamento da página não será capaz de rastrear o tempo de resposta com tanta precisão.

Os cookies da New Relic gerados pelo browser agente anterior à versão 995 não podem conter o atributo `secure` . Isso ocorre porque [a transmissão de dados](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-page-load-timing#transmission) de tempo de carregamento da página em versões anteriores à versão 995 usa HTTP quando a página é HTTP, mas usa HTTPS quando a página é HTTPS. Todas as versões do browser agente acima da versão 995 sempre usarão o sinalizador `secure` para cookies e transmitirão por HTTPS.

Os dados JavaScript e AJAX podem conter informações mais confidenciais, por isso são <DNT>**always**</DNT> transmitidos por HTTPS. A transmissão destes cookies através de HTTP ou o acesso aos mesmos através de JavaScript não representa um risco de segurança significativo, porque os cookies não são utilizados para tomar decisões de segurança ou permitir o acesso a uma conta. Eles são usados apenas para coletar dados de desempenho, com quaisquer dados identificáveis ofuscados.

Para clientes sujeitos a diretrizes especiais para coleta de cookies, como aquelas sob as [Diretrizes GDPR/PECR ICO](https://ico.org.uk/for-organisations/guide-to-pecr/cookies-and-similar-technologies/) da UE, agora oferecemos a opção de desativar a coleta de cookies para seu aplicativo. Consulte nossas [notas de versão do agente browser v1169](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1169) para obter mais informações.

<Callout variant="important">
  Se o seu site usa [P3P](http://www.w3.org/p3p/), ele deve estar configurado para permitir esses cookies.
</Callout>

## Solicitações JSONP [#jsonp]

As métricas de tempo de carregamento da página são relatadas à New Relic usando um script `GET`, também conhecido como solicitação JSONP. O script `GET` retorna um valor que é posteriormente armazenado em um cookie e usado para acionar a captura trace .
