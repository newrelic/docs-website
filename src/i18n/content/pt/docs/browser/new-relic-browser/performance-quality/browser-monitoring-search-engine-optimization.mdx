Nosso <InlinePopover type="browser"/>coloca um pequeno script "carregador" na seção principal de cada página em um site monitor. Esta instrumentação JavaScript foi projetada para:

* Fornece visibilidade máxima do desempenho do front-end, cronometra com precisão quanto tempo leva para as páginas carregarem em todos os browsers e relata detalhes de erros de JavaScript e chamadas AJAX.
* Têm impacto mínimo no ritmo geral de carregamento da página e na otimização do mecanismo de pesquisa (SEO), incluindo classificação do mecanismo de pesquisa, indexação, eficiência de rastreamento ou outras preocupações relacionadas ao SEO.

## Eficiência do JavaScript embutido [#why-javascript]

O método mais eficaz para monitoramento do browser é incluir uma quantidade mínima de código de instrumentação JavaScript embutido no cabeçalho da página do monitor. Esse código recupera o restante do código necessário após o carregamento da página.

Outros métodos de monitoramento do Browser incluem código JavaScript no final do corpo da página ou utilizando exclusivamente um script externo. No entanto, esses métodos podem limitar a visibilidade da experiência do usuário final.

## Impacto no SEO [#seo-impact]

Os resultados dos testes de desempenho indicam que o uso do monitoramento de JavaScript do browser tem um efeito insignificante no tempo de carregamento da página. Além disso, não tem impacto negativo na forma como o usuário ou os mecanismos de busca interagem com o seu site.

A recomendação consistente do Google aos proprietários de sites é construir um site que seja valioso para o usuário e acessível aos rastreadores dos mecanismos de pesquisa. As classificações do Google favorecem sites que fornecem as informações mais relevantes e a melhor experiência do usuário.

O monitoramento do Browser pode ajudá-lo a melhorar a experiência do usuário, identificando gargalos de desempenho, incluindo:

* [Carregamento lento da página](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity)
* [Erros problemáticos de JavaScript](/docs/errors-inbox/browser-tab)
* [Chamadas AJAX longas](/docs/browser/new-relic-browser/browser-pro-features/ajax-dashboard-identifying-time-consuming-calls)

Identificar erros de Javascript é especialmente útil porque os bots do Google estão cada vez mais executando o código JavaScript em sites que rastreiam para acessar o conteúdo fornecido por aplicativos da web com uso pesado de AJAX. Um erro de JavaScript que anteriormente era visível apenas para o usuário humano (por exemplo, um botão quebrado) pode afetar a capacidade dos bots do Google interagirem com sucesso com seu site.

Entendemos que o SEO e o tráfego encaminhado pelos mecanismos de pesquisa são extremamente importantes para muitas empresas. Quando usado de forma eficaz, o monitoramento de browser pode até aumentar a classificação de um site no Google e em outros mecanismos de busca, melhorando o desempenho e a experiência do usuário.
