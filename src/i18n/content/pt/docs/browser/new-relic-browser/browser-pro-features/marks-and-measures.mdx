[Marcas](https://developer.mozilla.org/en-US/docs/Web/API/Performance/mark) e [medidas](https://developer.mozilla.org/en-US/docs/Web/API/Performance/measure) são APIs padrão do browser que ajudam você a observar e relatar o desempenho de suas páginas da web. Esses eventos genéricos e nativos permitem que você meça a duração de qualquer tarefa ou processo em seu aplicativo. Com o monitoramento de browser New Relic, agora você pode rastrear marcas e medidas automaticamente e armazená-las como `BrowserPerformance` evento, proporcionando maior visibilidade do desempenho real do seu site.

## Por que usar o monitoramento de marcas e medidas?

O rastreamento automático de marcas e medidas com New Relic fornece insights práticos sobre o desempenho do seu aplicativo web. Ao capturar esses eventos, você pode:

* Identifique áreas lentas ou problemáticas no seu código.
* Medir o impacto de um novo recurso ou implantação.
* Compare o desempenho em diferentes jornadas do usuário.
* Crie dashboards personalizados e alertas baseados em dados reais do usuário.

## Pré-requisitos [#prerequisites]

* Um agente browser **Pro** ou **Pro + SPA**, versão 1.272.0 ou superior.

<Callout variant="important">
  A detecção automática de marcas e medidas não está disponível para o agente browser Lite.
</Callout>

## Habilitar coleta automática [#enable-collection]

Melhore a observabilidade e os recursos de depuração do seu aplicativo seguindo estas instruções simples de configuração.

<Steps>
  <Step>
    Acesse **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Selecione seu aplicativo de browser.
  </Step>

  <Step>
    No menu à esquerda, clique em **Application settings**.
  </Step>

  <Step>
    Certifique-se de que o browser do agente **Pro** ou **Pro + SPA** esteja selecionado.
  </Step>

  <Step>
    Para coletar marcas automaticamente, ative **Capture marks**. Para coletar medidas automaticamente, ative **Capture measures**. Para coletar detalhes de marcas ou medidas observadas como atributo personalizado, ative **Add details**.
  </Step>
</Steps>

## Desativar coleta de marcas e medidas [#disable-marks-and-measures]

Para desabilitar marcas e/ou medidas para um aplicativo de browser existente:

<Steps>
  <Step>
    Acesse **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Selecione seu aplicativo de browser.
  </Step>

  <Step>
    No menu à esquerda, clique em **Application settings**.
  </Step>

  <Step>
    Para desabilitar marcas, desative **Capture marks**. Para desabilitar medidas, desative **Capture measures**.
  </Step>
</Steps>

## Consumo de dados

`BrowserPerformance` evento segue o mesmo preço de consumo dos outros bytes do seu navegador. A quantidade de bytes produzidos depende da contagem e do atributo dos dados.

Uma vez ativado, o agente capturará automaticamente marcas e medidas conforme elas ocorrem no seu aplicativo.

## Encontre seus dados no New Relic [#find-data]

Os dados de marcas e medidas são armazenados como evento `BrowserPerformance` no New Relic, denotados como `mark` ou `measure` no campo `entryType`. Consulte o [dicionário de dados](https://docs.newrelic.com/attribute-dictionary/?event=BrowserPerformance) para obter mais informações sobre os campos disponíveis para uso com `BrowserPerformance` evento. Para um dashboard rápido e útil, veja este [início rápido](https://newrelic.com/instant-observability/browser-marks-and-measures), caso contrário, você pode consultar e visualizar esses dados para obter insights sobre o desempenho do seu aplicativo.

**Encontre as medidas mais lentas por duração média**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET entryName 
LIMIT 10 
ORDER BY average(entryDuration) DESC
```

**Conte o número de marcas e medidas por página**

```nrql
FROM BrowserPerformance 
SELECT count(*) 
WHERE entryType IN ('mark', 'measure') 
FACET pageUrl
```

**Veja a duração média da medida por navegador**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET userAgentName
```

**Duração das medidas de acompanhamento por país**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET countryCode
```

**Encontre todos os nomes de medidas exclusivos registrados nas últimas 24 horas**

```nrql
FROM BrowserPerformance 
SELECT uniqueCount(entryName) 
WHERE entryType = 'measure' 
SINCE 24 hours ago
```

**Compare durações de medidas para desktop e dispositivos móveis**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET deviceType
```

**Listar todas as medidas para uma sessão específica**

```nrql
FROM BrowserPerformance 
SELECT * 
WHERE entryType = 'measure' 
AND session = 'abcdefghijklmnop'
```

Use estas consultas ou crie as suas próprias para criar dashboards personalizados, configurar alertas e monitorar o desempenho real do seu aplicativo web com precisão.
