A página <DNT>**User impact**</DNT> em <InlinePopover type="browser" />ajuda você a entender como os problemas de desempenho técnico afetam a experiência real de seus usuários. Esta página faz a ponte entre métricas de desempenho quantitativas e análise qualitativa do comportamento do usuário, fornecendo insights sobre os padrões de frustração do usuário e sua correlação com problemas técnicos.

A página de User impact consiste em duas áreas principais de análise:

* <DNT>Frustration metrics</DNT>
* <DNT>Performance risk</DNT>

Cada um fornece diferentes lentes para entender os problemas de experiência do usuário e suas causas técnicas.

**Pré-requisito:**

* A versão 1.269 ou posterior do agente Browser é necessária para capturar sinais de frustração e dados de risco de desempenho.
* [Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/get-started/) deve estar habilitado para acessar as reproduções de sessão vinculadas a esta página.

## Acessar Análise de impacto do usuário [#access]

Para visualizar a página de User impact:

1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app)** &gt; **Users** &gt; **User impact**</DNT>.
2. Selecione <DNT>**Frustration metrics**</DNT> ou <DNT>**Performance risk**</DNT> na guia.

<Callout variant="tip">
  Você também pode navegar aqui diretamente da página [Page views](/docs/browser/new-relic-browser/browser-pro-features/page-views-examine-page-performance) clicando em <DNT>**Analyze frustration signals**</DNT> ou no link <DNT>**Analyze performance risk**</DNT>.
</Callout>

## Analisar métricas de frustração [#frustration-metrics]

A guia Frustration metrics se concentra em indicadores comportamentais do usuário que sinalizam confusão, aborrecimento ou dificuldade em concluir tarefas. Essa análise ajuda você a identificar onde os usuários mais lutam e priorizar as correções com base nos pontos problemáticos reais do usuário.

A guia Frustration metrics inclui as seguintes seções:

* Clique de raiva: identifica páginas e elementos onde os usuários exibem comportamento de clique rápido e repetido, indicando frustração com componentes de interface do usuário (UI) que não respondem ou estão quebrados.
* Páginas mais afetadas: Destaca as páginas específicas que geram mais sinais de frustração, permitindo que você se concentre em áreas de alto impacto.
* Principais elementos de página afetados: fornece informações detalhadas sobre quais elementos específicos da interface do usuário estão causando mais frustração ao usuário.

## Analisar métrica de desempenho [#performance-impact]

A guia Performance risk examina como os problemas de desempenho técnico afetam as sessões do usuário, com foco em métricas de desempenho mensuráveis e sua correlação com problemas de experiência do usuário.

A guia Performance risk inclui as seguintes seções:

* Web vitals: analisa como os principais web vitals ruins (LCP, INP, CLS) impactam as sessões e a experiência do usuário.
* Análise de erros: Investiga padrões de erros JavaScript e seu efeito nas sessões do usuário.
* Páginas mais afetadas: Identifica quais páginas apresentam o maior impacto no usuário relacionado ao desempenho.
* Análise geográfica: examina o risco de desempenho em diferentes locais geográficos para identificar problemas regionais.
