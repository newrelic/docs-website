Se você estiver usando o browser para monitor seu aplicativo, poderá desativar manualmente o monitoramento do seu aplicativo ou de páginas específicas. Os motivos para desativar <InlinePopover type="browser"/>em determinadas páginas incluem:

* Algumas páginas podem não ser relevantes para monitoramento.
* Você deseja monitor apenas páginas específicas.
* Você deseja remover o script do browser para fins de resolução de problemas.

## Desabilitar o monitoramento de páginas/URLs específicos [#disable]

Para desativar o browser em uma página ou URL específico, siga os procedimentos com base no seu método de implantação:

<CollapserGroup>
  <Collapser
    id="agent"
    title="Implantar com agente APM selecionado"
  >
    Se você usou o agente <InlinePopover type="apm"/>para inserir automaticamente seu trecho JavaScript, poderá desativar a injeção apenas para essas páginas específicas usando a API ou o arquivo de configuração do agente de linguagem APM. Para obter mais informações, consulte as instruções do seu agente:

    * Go (atualmente não aplicável)
    * [Java](/docs/agents/java-agent/instrumentation/page-load-timing-java#disable-auto-specific)
    * [.NET](/docs/agents/net-agent/features/page-load-timing-net#disable_instrumentation_pageview)
    * [Node.js](/docs/agents/nodejs-agent/supported-features/page-load-timing-nodejs#disabling): Este agente não insere JavaScript automaticamente. Para desativar o monitoramento em determinadas páginas, basta remover a chamada de API dessas páginas.
    * [PHP](/docs/agents/php-agent/features/new-relic-browser-php-agent#disable)
    * [Python](/docs/agents/python-agent/supported-features/page-load-timing-python#disabling_instrumentation)
    * [Ruby](/docs/agents/ruby-agent/installation-configuration/ignoring-specific-transactions#page-load-timing-rum)
  </Collapser>

## Desabilitar monitoramento de domínio [#domains]

Para desativar o monitoramento de domínios ou subdomínios específicos, atualize [as condições de domínio](/docs/browser/new-relic-browser/configuration/monitor-or-block-specific-domains-subdomains) do seu browser.

## Desativar agente browser [#disable-agent]

Se você usou o [método de implantação copiar/colar](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app) e quiser parar de coletar dados para um aplicativo de browser, basta remover o trecho JavaScript do browser das páginas que você não deseja monitor. Se você usou o [agente APM](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app) para instalar o agente browser, você pode desligar alguns dos recursos de monitoramento do browser individualmente, ou você pode desabilitar o browser completamente.

1. Vá para

   <DNT>
     **[one.newrelic.com](https://one.newrelic.com/all-capabilities), click on Browser > (select a browser app) > Settings > Application settings**
   </DNT>

   .

2. Para desativar apenas o [recurso de nível de conta Pro](/docs/browser/new-relic-browser/getting-started/new-relic-browser), selecione

   <DNT>
     **Lite**
   </DNT>

   .

3. Para desativar completamente o monitoramento do browser, selecione

   <DNT>
     **Off**
   </DNT>

   .

4. Selecione

   <DNT>
     **Save application settings**
   </DNT>

   .

5. Se aplicável, reinicie seu agente

   <InlinePopover type="apm"/>

   .

Quando o monitoramento do Browser estiver desligado, o agente não instrumentará páginas com script de monitoramento.
