## Problema

Você deseja usar o console do seu navegador para visualizar mensagens de erro detalhadas do ambiente, mas quando <InlinePopover type="browser"/>está ativo, o console do seu navegador não exibe rastreamento de erro detalhado.

## Solução

O monitoramento de navegador deve capturar e relançar erros para registrá-los, mas algumas versões do navegador não exibem stack trace para erros relançados. Para resolver este problema, habilite o modo de desenvolvimento do navegador em seu navegador local.

[As versões 593 ou superiores do agente de monitoramento do navegador](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) incluem uma solução alternativa para esse comportamento, que pode ser habilitada no console. O procedimento a seguir é para o Google Chrome, mas você pode adaptá-lo para outros navegadores.

1. No Chrome, selecione o ícone do menu e selecione <DoNotTranslate>**More tools > JavaScript Console**</DoNotTranslate>.

2. No console JavaScript, defina o seguinte valor para `nr_flags` no armazenamento local:

   ```js
   localStorage.setItem('__nr_flags', 'dev')
   ```

3. Atualize a página onde deseja ver seus erros de JavaScript.

4. Para confirmar se o modo de desenvolvimento está ativo, verifique o log do navegador do agente do console em busca da mensagem `NR AGENT IN DEVELOPMENT MODE`.

5. Navegue no site e acione os erros que deseja rastrear.

Para desabilitar o modo de desenvolvimento, execute o seguinte comando no console:

```js
localStorage.removeItem('__nr_flags')
```
