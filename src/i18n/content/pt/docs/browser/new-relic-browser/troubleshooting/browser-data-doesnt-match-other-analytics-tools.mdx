## Problema

Você vê dados diferentes em <InlinePopover type="browser" />dos de outras ferramentas de análise que usa. Os sintomas comuns incluem:

* O browser registra menos [visualizações de página](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity) do que o esperado.
* [A distribuição geográfica](/docs/browser/new-relic-browser/additional-standard-features/geography-webpage-performance-location) do browser para o seu usuário difere das localizações da sua base de usuários esperada.
* O browser mostra um número diferente de [sessões ativas](/docs/browser/new-relic-browser/page-load-timing-resources/session-tracking) do que você vê em outros lugares.

## Solução

Siga estas etapas de resolução de problemas para monitoramento do browser:

1. Verifique se o monitoramento do Browser está instalado corretamente em todas as páginas da sua aplicação.
2. Caso o trecho JavaScript esteja faltando, ou caso o usuário não consiga enviar dados para o New Relic, siga os passos para [resolução de problemas de monitoramento de instalação do Browser](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation).
3. Se você tem um aplicativo de estilo de página única e espera ver alterações de rota como visualizações, considere usar [o monitoramento do browser de SPA](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring), que fornece uma visualização integrada dos carregamentos de página iniciais e das alterações de rota.

Se o monitoramento do browser estiver funcionando corretamente, você poderá ver resultados diferentes porque New Relic coleta e apresenta informações de forma diferente de outras ferramentas de análise. Por exemplo, `JavascriptErrors` e `AjaxRequest` são monitorados individualmente e seus dados dependem de quando nosso agente os observa e o que podemos ou decidimos capturar. Para `BrowserInteraction`, usamos nossa própria instrumentação personalizada e, para sinais vitais e métricas em `PageViewTiming` e `PageView`, a biblioteca web-vitals de propriedade da equipe do Chrome é a fonte desses dados.

Para obter mais informações sobre como coletamos dados, consulte:

* [Processo de tempo de carregamento da página](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process)
* [Instrumentação para monitoramento de Browser](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-browser-monitoring)
* [Entenda a coleta de dados do SPA](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection)
