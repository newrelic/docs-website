## Problema

Você está vendo erros 404 no dashboard da Pesquisa Google para URLs que não são válidos para o seu site. Você também pode ver erros 404 no log do servidor web para esses caminhos. Depois de pesquisar em seu site, você descobre que o caminho do site está presente como uma string no script do carregador do agente browser que está sendo injetado em seu HTML.

## Causa

O agente browser faz uso de literais de string no código, mas os converte em concatenação de string durante o processo de construção para evitar possíveis problemas com código que podem agrupar a string do carregador em um literal de string. Em alguns casos, isso pode levar a strings que começam com uma barra `/`. Isso significa que quando o Google indexar a página que contém o script do carregador do agente, ele armazenará essas strings como caminhos potenciais para o seu site e tentará indexá-las.

## Soluções

Este não é um cenário incomum para o administrador do site e não está vinculado apenas ao agente browser. Pode haver vários motivos para o aparecimento de strings no HTML do seu site e fazer com que a Pesquisa Google as perceba como um caminho potencial. Aqui estão alguns recursos que demonstram como outros abordaram essa preocupação:

* [Como posso impedir que o Google rastreie itens que ele considera URLs de **NEXT_DATA**?](https://stackoverflow.com/questions/75672103/how-can-i-block-google-from-crawling-things-it-thinks-are-urls-from-next-data)
* [O Google segue a string JavaScript como caminho relativo - produz erro 404](https://webmasters.stackexchange.com/questions/50848/google-follows-javascript-string-as-relative-path-produces-404-error)

Os recursos disponíveis na Internet indicam que esses erros 404 não afetarão a classificação ou indexação do seu site. Eles podem ser ignorados com segurança. No entanto, se ainda estiver preocupado, você pode entrar em contato com a [Comunidade de suporte da Pesquisa Google](https://support.google.com/webmasters/community?hl=en) para obter feedback e ajuda adicionais.
