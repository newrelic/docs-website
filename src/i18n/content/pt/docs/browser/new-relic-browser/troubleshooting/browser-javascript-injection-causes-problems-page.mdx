## Problema

Nosso <InlinePopover type="browser"/>está inserindo sua instrumentação JavaScript em um local que causa problemas de marcação ou carregamento de página.

## Solução

O agente New Relic tenta injetar o JavaScript no local ideal, mas se você encontrar problemas, tente estas etapas de resolução de problemas.

1. Verifique se o monitoramento do Browser é a causa [desativando o monitoramento do Browser](/docs/browser/new-relic-browser/installation/disable-browser-monitoring/).

2. Se desabilitar o monitoramento do browser resolver o problema, investigue o código-fonte da página. Reative o monitoramento do browser e verifique os seguintes possíveis problemas:

   * Mais de uma instância da tag `<html>` na página (não em um iFrame).
   * Mais de uma instância da tag `<head>` ou `<body>` ou uma tag `<head>` ou `<body>` ausente.
   * Ocorrência de qualquer tag `<head>` ou `<body>` no JavaScript da página.
   * Use [http://validator.w3.org](http://validator.w3.org) para verificar a origem da sua página e verificar se há sérios problemas de validade de marcação ou codificação de caracteres.

3. Elimine quaisquer conflitos que você encontrar revisando a estrutura da página, [desativando o monitoramento do browser para aquela página](/docs/browser/new-relic-browser/installation-configuration/disable-browser-monitoring-specific-pages) ou alternando [manualmente para o instrumento via API do agente](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring#agent-instrumentation) para controle total sobre a injeção de script.

4. Se você encontrar outras situações, entre em contato com [o suporte da New Relic](https://support.newrelic.com/) e forneça um link acessível para sua página ou o código-fonte dela. Certifique-se de que o monitoramento do browser esteja habilitado em seu link ou código fonte.
