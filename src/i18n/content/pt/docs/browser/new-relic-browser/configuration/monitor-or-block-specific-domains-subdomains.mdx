[browser](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) permite escolher quais domínios ou subdomínios um aplicativo de browser monitora. Isto é útil para limitar o monitoramento apenas à atividade do aplicativo que você considera importante. Por exemplo, se você usar o [método copiar/colar](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) para implantar seu aplicativo de browser, poderá definir condições de domínio para proteger a segurança de sua [<InlinePopover type="browser"/>chave de licença e ID do aplicativo](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id).

<Callout variant="tip">
  As condições do domínio filtram apenas os dados com base no URL da página relatada. Os filtros de condição de domínio não se aplicam aos dados AJAX relatados à New Relic. Para filtrar o evento `AjaxRequest` , leia [nossos documentos de configuração](/docs/browser/new-relic-browser/configuration/filter-ajax-request-events/).
</Callout>

## Configurar condições de domínio [#configure]

Para visualizar ou editar as configurações de condição do seu domínio:

1. Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Application settings > Domain conditions**</DNT>.

2. Se não houver condições de domínio em vigor, selecione <DNT>**Enable domain conditions**</DNT>. Se existirem condições, selecione <DNT>**Edit conditions**</DNT>.

3. Selecione a opção de criar uma lista de domínios permitidos ou de criar uma lista de domínios bloqueados/negados.

4. Selecione as condições do domínio (máximo de 10 condições).

5. Selecione as condições do domínio com a condição <DNT>**Starts with**</DNT> e insira o URL com o subdomínio (máximo de 10 condições).

   <img
     title="browser_domain_conditions.png"
     alt="browser_domain_conditions.png"
     src="/images/browser_screenshot-full_domain-settings.webp"
   />

   <figcaption>
     <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select a browser app) > Settings > Domain conditions:**</DNT> Use esse recurso para limitar o monitoramento apenas às atividades importantes do aplicativo ou para aumentar a segurança da chave de licença do browser e do ID do aplicativo.
   </figcaption>

6. Revise e confirme as configurações de condição do seu domínio.

Uma caixa de texto de visualização exibe em tempo real quais domínios correspondem às condições que você definiu. Você também verá uma estimativa do impacto que as configurações propostas terão nos seus dados.

## Desative as condições de domínio [#disable]

Para desabilitar as condições de domínio e retornar ao monitoramento de todos os domínios:

1. Vá para

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Domain conditions**
   </DNT>

   .

2. Defina a opção

   <DNT>
     **Enable domain conditions**
   </DNT>

   como

   <DNT>
     **Off**
   </DNT>

   .
