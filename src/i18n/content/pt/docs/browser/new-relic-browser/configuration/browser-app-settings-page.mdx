### Armazenamento de sessão Browser [#browser-session-storage]

Reforçamos o controle para coleta de dados de rastreamento de sessão e repetição de sessão do agente do browser. Agora você pode gerenciar todos os dados de nível de sessão diretamente do botão <DNT>**Access browser local storage**</DNT> nesta página de configurações.

* **Alternância para acesso ao armazenamento local do navegador**: esse controle unificado determina como os dados em nível de sessão (incluindo IDs de sessão e outros atributos em nível de sessão) são coletados.

  * **Quando ativado (padrão)**: O agente do browser do New Relic Browser usa os mecanismos de armazenamento persistente do navegador para coletar dados trace de sessão e repetição de sessão.
  * **Quando DESLIGADO**: O agente do browser do New Relic Browser interrompe imediatamente o uso do armazenamento persistente do navegador. Esta ação impede a coleta de todos os dados trace de sessão e repetição de sessão, dando a você controle direto sobre o manuseio de informações relacionadas à sessão pelo navegador e melhorando sua capacidade de atender aos requisitos de privacidade e conformidade de dados.

Para acessar a página <DNT>**Application settings**</DNT> :

1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT> e selecione um aplicativo.
2. Na navegação esquerda, role para baixo até <DNT>**Settings**</DNT> e clique em <DNT>**Application settings**</DNT>.

Você também pode personalizar a instrumentação de agente do seu browser para capturar rastreamentos e eventos personalizados, relatar erros específicos adaptados às suas necessidades e ajustar o tempo de coleta de dados para obter desempenho ideal.
