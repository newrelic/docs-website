### Armazenamento de sessão Browser [#browser-session-storage]

Reforçamos o controle para coleta de dados de rastreamento de sessão e repetição de sessão do agente do browser. Agora você pode gerenciar todos os dados de nível de sessão diretamente do botão <DNT>**Access browser local storage**</DNT> nesta página de configurações.

/\* \* \*\*Alternar acesso ao armazenamento local do navegador\*\*: Este controle unificado determina como os dados em nível de sessão (incluindo IDs de sessão e outros atributos em nível de sessão) são coletados. \* \*\*Quando ativado (padrão)\*\*: O agente do Browser de New Relic usa os mecanismos de armazenamento persistente do navegador para coletar dados trace da sessão e reprodução da sessão. \* \*\*Quando DESATIVADO\*\*: O agente do Browser de New Relic interrompe imediatamente o uso do armazenamento persistente do navegador. Essa ação impede a coleta de todos os dados trace de sessão e reprodução de sessão, dando a você controle direto sobre como o navegador lida com as informações relacionadas à sessão e aprimorando sua capacidade de atender aos requisitos de privacidade e conformidade de dados. \*/

<Callout variant="important">
  A partir da versão 1.303.0 do agente do Browser , a opção **Access local storage** está sendo removida da interface para simplificar nossos recursos de privacidade e consentimento.

  * A maneira recomendada de gerenciar a coleta de dados do agente do browser e o consentimento do usuário é por meio da configuração do modo de consentimento. Para obter mais informações, consulte [os documentos de consentimento](/docs/browser/new-relic-browser/configuration/consent-mode/).

  * Se você tiver a opção **Access local storage** definida como `OFF`, verá que as opções Modo de consentimento, Reprodução de sessão e trace de sessão também estarão inativas. Para habilitar esses recursos, você deve definir o atributo `cookie_enabled` para `true` por meio [APINerdGraph](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-browser-application). Essa ação reativará as opções na interface, permitindo que você configure corretamente o Modo de consentimento, a Reprodução de sessão e trace da sessão.
</Callout>

## API do browser [#apis]

Você também pode personalizar a instrumentação de agente do seu browser para capturar rastreamentos e eventos personalizados, relatar erros específicos adaptados às suas necessidades e ajustar o tempo de coleta de dados para obter desempenho ideal. Consulte [Usando API do browser](/docs/browser/new-relic-browser/browser-apis/using-browser-apis/) para começar.
