Na [versão 1211 do agente](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1211/) ou superior, todas as solicitações de rede feitas por uma página são registradas como eventos `AjaxRequest`. Você pode usar as opções de configuração da lista de negação na página <DNT>**Application settings**</DNT> para filtrar quais solicitações registram eventos.

## Usando a lista de negações

Por padrão, registramos eventos `AjaxRequest` para todas as solicitações de rede feitas pelo seu aplicativo.

* Para parar de registrar todos os eventos, adicione um único caractere curinga `*` (asterisco).
  * Exemplo: `*`
* Para parar de registrar eventos para um site completo, adicione o nome do domínio.
  * Exemplo: `mywebsite.com`
* Para parar de registrar eventos para uma página específica, adicione o domínio e o caminho.
  * Exemplo: `mywebsite.com/photos`
* Começando com a [versão 1.307.0 do agente](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.307.0/), em qualquer lugar que encontrarmos um `*` (asterisco), corresponderemos a qualquer comprimento de string.
  * Exemplo: `mywebsite.com/shop/*` (para um caminho), `*.mywebsite.com` (para um subdomínio), `mywebsite.com/*/shop/*/checkout` (para strings de URL intermediárias).

O protocolo, a porta, a pesquisa e o hash de aplicação de uma URL são ignorados pela lista de negações.

## Acesse a lista de negações

Para atualizar a lista de URLs negados que seu aplicativo filtrará na criação do evento, acesse a página de configurações do aplicativo:

1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Settings &gt; Application settings**</DNT>.
2. Em <DNT>**Ajax Request Deny List**</DNT>, adicione os filtros que você gostaria de aplicar ao seu aplicativo.
3. Selecione <DNT>**Save application settings**</DNT> para atualizar a configuração do agente.
4. [Reimplante o agente browser](/docs/browser/new-relic-browser/installation/upgrade-browser-agent) (reiniciando o agente <InlinePopover type="apm" />associado ou atualizando a instalação de copiar/colar do browser).

## Validação

Para validar se os filtros adicionados funcionam conforme o esperado, execute uma consulta NRQL para `AjaxRequest` evento correspondente ao seu filtro.

```sql
FROM AjaxRequest SELECT * WHERE requestUrl LIKE `%example.com%`
```
