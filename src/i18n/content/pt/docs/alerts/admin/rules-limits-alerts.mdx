## Consulta de alerta NRDB com pontos de dados correspondentes por minuto [#query-limit]

O limite de condição do alerta `Matched data points per minute` se aplica à taxa total de [pontos de dados](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/understand-technical-concepts/streaming-alerts-key-terms-concepts) correspondentes para a consulta de alerta em uma [conta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure) New Relic .

Se esse limite for excedido, você não poderá criar ou atualizar condições para a conta afetada até que a taxa fique abaixo do limite. As condições de alerta existentes **não são** afetadas.

Você pode ver seus pontos de dados correspondentes e qualquer incidente de limite na [interface de limites](/docs/data-apis/manage-data/view-system-limits).

Para entender quais condições estão levando ao maior número de taxas de transferência, você pode realizar uma consulta como:

```sql
FROM NrAiSignal
SELECT sum(aggregatedDataPointsCount) AS 'alert matched data points'
FACET conditionId
```

Algumas dicas sobre como otimizar seus pontos de dados correspondentes:

* Se você estiver usando [janelas deslizantes](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions/#sliding-window-aggregation), observe que isso pode aumentar significativamente o número de pontos de dados. Para diminuir o número de pontos de dados, você pode usar uma duração de agregação mais longa.
* Use cláusulas `WHERE` para reduzir a quantidade de dados que estão sendo alertados. Usar `WHERE` em vez de `FACET` pode produzir alertas mais eficientes em alguns casos.
* Combine alerta semelhante. Se você tiver várias condições de alerta semelhantes, considere agrupá-las com filtros combinados.

Para solicitar um aumento de limite, fale com seu representante de conta New Relic.

Observe que o uso [de janelas deslizantes](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/create-smoother-charts-sliding-windows) pode aumentar significativamente o número de pontos de dados. Considere usar uma duração mais longa de agregação de janela deslizante para reduzir o número de pontos de dados produzidos.

## Operações de verificação de consulta de alerta por minuto [#query-scan-limit]

O limite da condição do alerta `Alert query scan operations per minute` aplica-se à taxa total de operações de verificação de consulta no evento ingerido. Uma operação de verificação de consulta é o trabalho realizado pelo New Relic pipeline para corresponder o evento ingerido ao alerta de consulta registrado em uma New Relic [conta](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure).

Se esse limite for excedido, você não poderá criar ou atualizar condições para a conta afetada até que a taxa fique abaixo do limite. As condições de alerta existentes **não são** afetadas.

Você pode ver suas operações de digitalização de consulta e qualquer incidente de limite na [interface de limites](/docs/data-apis/manage-data/view-system-limits).

Ao corresponder evento a alerta consulta, todos os eventos do [tipo de dados](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/#what-you-can-query) aos quais a consulta faz referência devem ser examinados. Aqui estão algumas maneiras comuns de ter menos eventos em um determinado tipo de dados (o que diminuirá as operações de verificação de alerta de consulta):

* Ao alertar sobre dados de log, use [partições de log ](/docs/tutorial-manage-large-log-volume/organize-large-logs/)para limitar quais logs estão sendo verificados em busca de alerta consulta.

* Ao alertar sobre evento personalizado, divida tipos maiores de eventos personalizados.

* Utilize evento personalizado em vez de alertar sobre evento de transação.

* [Crie métricas](/docs/data-apis/convert-to-metrics/analyze-monitor-data-trends-metrics/) para agregar dados.

* Utilize [métrica de fração de tempo consulta](/docs/data-apis/understand-data/metric-data/query-apm-metric-timeslice-data-nrql/) quando possível em vez de alertar sobre evento de transação.

Além das dicas acima, limpar qualquer alerta consulta (condição do alerta) não utilizado ou desnecessário diminuirá o número de operações de verificação de consulta.

Para solicitar um aumento de limite, fale com seu representante de conta New Relic.

Observação: durante uma interrupção do serviço do New Relic, os alertas podem ficar temporariamente indisponíveis.

## Uso aceitável de limites

De acordo com New Relic [a política de uso aceitável](/docs/licenses/license-information/general-usage-licenses/acceptable-use-policy) da , condições de alerta que sobrecarreguem nossa infraestrutura podem ser desabilitadas pelo New Relic administrador .

Para configurar seus próprios alertas para determinar se uma condição foi desabilitada pelo administrador New Relic , consulte os exemplos de [NrIntegrationError](/docs/data-apis/manage-data/nrintegrationerror/#query-alert) .
