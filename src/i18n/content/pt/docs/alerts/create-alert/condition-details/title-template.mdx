Um modelo de título é usado quando eventos de alerta são abertos pela condição. Substitui o título padrão.

## Por que usar um modelo de título? [#why-titles]

Uma condição de alerta é a base para a criação de eventos de alerta. Seja usando nosso modo guiado ou criando uma condição a partir de um gráfico ou política, usar um modelo de título exclusivo é a etapa final para criar uma condição de alerta.

O uso do modelo de título é opcional, mas nós o recomendamos. Uma condição de alerta define um conjunto de limites que você deseja monitorar. Se algum desses limites for excedido, um evento de alerta é criado. Modelos de título significativos ajudam a identificar problemas e resolver interrupções mais rapidamente.

## Antes de você começar [#begin]

Aqui estão algumas coisas importantes a serem observadas antes de usar nosso modelo de título:

* **Compatibilidade**: o recurso de modelo de título está disponível apenas para condições NRQL.
* **Limite de caracteres**: o comprimento máximo de um modelo de título é de 256 caracteres.
* **Formatação**: O modelo de título deve seguir um formato [de guiador](https://handlebarsjs.com/guide/) válido.
* **Eventos de alerta de sinal perdido**: Para eventos de alerta acionados por um sinal perdido, o título padrão será sempre usado. Para saber mais sobre perda de sinal, consulte [Perda de sinal e preenchimento de lacunas](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/).

## Entenda o modelo de título [#example]

Digamos que você administre um site de e-commerce e tenha criado uma condição de alerta para acionar um evento de alerta quando os erros de rede estiverem altos. Você pode usar um modelo de título para tornar os eventos de alerta desta condição facilmente identificáveis.

Você nomeou esta condição de alerta `High Network Errors`. Mas você quer que o título do evento de alerta reflita mais informações sobre a entidade e a prioridade.

**Exemplo**:

```handlebars
{{conditionName}} Affecting {{entity.name}} - {{entity.type}} - {{priority}}
```

Este modelo geraria o seguinte título de evento de alerta:

```
High Network Errors Affecting Entity123 - Kubernetes Pod - Critical
```

## Use o modelo de título [#use-titles]

Você pode interagir com o modelo de título de duas maneiras: através da interface ou usando GraphQL. Siga estas etapas para saber como.

<CollapserGroup>
  <Collapser id="template-ui" title="Crie um modelo de título na interface">
    1. Navegue até o construtor de condições NRQL. Para obter instruções, consulte [condição do alerta](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/alert-conditions/).
    2. Crie sua condição NRQL.
    3. Na etapa final, **Add details**, insira seu modelo de título no campo **Title template** .
    4. Salve a condição.
  </Collapser>

  <Collapser id="template-ui" title="Crie um modelo de título no GraphQL">
    No GraphQL, o campo do modelo de título é chamado `titleTemplate`. Aqui está um exemplo de mutação para atualizar uma condição estática NRQL para adicionar um modelo de título:

    ```graphql
    mutation {
      alertsNrqlConditionStaticUpdate(
        accountId: 123456
        id: 456789
        condition: {titleTemplate: "{{conditionName}} has triggered an incident"}
      ) {
        titleTemplate
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## Use atributo para definir melhor sua condição [#attributes]

Você pode incluir um subconjunto de atributos de eventos de alerta em seus modelos de título para adicionar mais contexto.

Veja como:

* **Use o formato `{{AN_ATTRIBUTE}}`**. Por exemplo:

  ```handlebars
  {{conditionName}} has triggered an alert event
  ```

* **Tag atributos**: Para tag, use o formato `{{tags.TAG_NAME}}`.

* **Tag de cláusula de faceta**: se uma condição NRQL usar uma cláusula `FACET`, você poderá usar a formatação `tags` para usar esses valores em seu modelo de título. Por exemplo, se a consulta NRQL incluísse `FACET hostName, cluster`, você poderia usar isto:

  ```handlebars
  Host {{tags.hostName}}, cluster {{tags.cluster}} alert event
  ```

Nomes de tags não podem conter espaços, mas os valores expandidos podem. Para uma lista completa de atributos disponíveis, consulte [Atributos de evento de alerta](/docs/alerts/create-alert/condition-details/alert-event-attributes).
