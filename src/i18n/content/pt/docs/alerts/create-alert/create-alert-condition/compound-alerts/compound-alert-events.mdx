Quando a expressão de acionamento de uma condição do alerta composta é avaliada como verdadeira, o New Relic abre um evento de alerta composto. O evento inclui um link para a URL do seu runbook. Para visualizar violações de limite específicas para cada condição do componente, clique nos eventos de alerta individuais.

## Eventos de Alerta e Correlação de Problemas [#alert-events-correlation]

As condições do alerta compostas criam vários eventos de alerta relacionados:

* <DNT>**Multiple alert events**</DNT>: uma condição do alerta composta cria seu próprio evento de alerta composto. Por padrão, cada condição do componente também cria seus próprios eventos de alerta quando viola seus limites. Para reduzir o ruído, você pode silenciar esses eventos a partir da configuração da condição do alerta. Se você silenciar os eventos de alerta de uma condição do componente, as tags dessa condição não aparecerão no evento de alerta composto.
* <DNT>**Correlating to the same issue**</DNT>: para garantir que todos os eventos de alerta relacionados (tanto compostos quanto de componentes) sejam agrupados no mesmo problema, configure suas condições do componente para usar a mesma política que sua condição do alerta composta. Isso permite que a correlação de problemas do New Relic os agrupe.
* <DNT>**Viewing related events**</DNT>: a partir de um evento de alerta composto, você pode navegar até os eventos de alerta de componentes individuais para ver as violações de limite e as métricas de cada condição.
