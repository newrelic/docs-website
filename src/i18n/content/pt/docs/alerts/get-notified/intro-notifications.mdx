As equipes de DevOps têm muito a monitor à medida que os sistemas se tornam cada vez mais complexos e pode ser difícil acompanhar o que está acontecendo e onde. notificação é um conjunto de ferramentas que permite levar as informações certas às pessoas certas o mais rápido possível. As notificações enviam notificação do evento para serviços de terceiros, como Slack, Jira, ServiceNow e e-mail. Você também pode usar webhooks para enviar seus dados para qualquer serviço de terceiros compatível.

## Como funciona a notificação [#configurations]

Depois de configurar sua condição de alerta e políticas para identificar comportamentos irregulares em seu sistema, é importante notificar as pessoas certas de sua equipe sobre quaisquer problemas potenciais. A New Relic criou fluxo de trabalho para que você possa controlar quando e onde deseja receber notificações sobre problemas. Isso permite que você forneça as informações corretas à pessoa ou equipe relevante. fluxo de trabalho também oferece a oportunidade de enriquecer sua notificação com dados adicionais da New Relic.

Fluxo de trabalho utiliza destinos e canais para enviar notificações de alerta para sua equipe. Você precisa configurar destinos e canais para que o fluxo de trabalho funcione perfeitamente.

Os destinos indicam _para onde e como suas informações estão sendo enviadas_. Por exemplo, digamos que você queira ter certeza de que sempre que a New Relic identificar qualquer latência em sua implantação recente, ela notificará um canal da equipe do Slack. Você criaria um destino para definir o onde (slack) e o como (qualquer chave de API ou token oauth necessário para enviá-lo).

Os canais definem _o que_ está sendo enviado. No cenário acima, digamos que você queira que a mensagem no canal do Team Slack seja "_latência identificada no aplicativo x_ ". Você configuraria um canal para criar a mensagem.

### Fluxo de trabalho [#workflows]

O primeiro passo no gerenciamento de notificações é criar um fluxo de trabalho. fluxo de trabalho são os sistemas que você utiliza para controlar sua notificação. Nos fluxos de trabalho você pode criar filtros para decidir quando o fluxo de trabalho será executado, criar canais e adicionar enriquecimentos. Enriquecimentos são dados que serão buscados no NRDB e aparecerão no evento de saída.

Saiba mais sobre como criar fluxos de trabalho em nossa [documentação sobre fluxo de trabalho](/docs/alerts-applied-intelligence/applied-intelligence/incident-workflows/incident-workflows/).

### Destinos [#destinations]

Depois que um fluxo de trabalho for criado, você poderá criar seus destinos. Use destinos para integrar, enviar notificações e compartilhar dados entre a New Relic e seus sistemas de terceiros. Um destino contém o local para onde a notificação será enviada. Os destinos são reutilizáveis e podem ser usados em produtos New Relic.

Saiba mais sobre como criar destinos em nossa [documentação de destinos](/docs/alerts-applied-intelligence/notifications/destinations/).

### Canais [#channels]

Depois que um fluxo de trabalho for criado, você poderá configurar o eventual evento de notificação e mapear os dados do New Relic para seus serviços de terceiros usando um canal, também conhecido como modelo de mensagem. Um canal permite que você personalize seus dados de eventos de notificação antes de serem enviados para seu destino de terceiros.

Saiba mais sobre como criar canais em nossa [documentação de canais](/docs/alerts-applied-intelligence/notifications/message-templates/).

## API [#api]

Você também pode usar nossa API NerdGraph para visualizar e configurar [destinos](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-destinations) e [canais de notificação](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-channels).

## Segurança [#security]

As credenciais usadas para destinos e canal de notificação são armazenadas de forma segura de acordo com [os padrões de segurança da New Relic](/docs/security/security-privacy/compliance/data-encryption).
