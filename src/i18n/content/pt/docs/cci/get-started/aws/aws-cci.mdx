Configure seu ambiente AWS para o Cloud Cost Intelligence configurando o faturamento consolidado, os Relatórios de custos e uso (CUR), as funções e políticas do Identity and Access Management (IAM) e as regiões da AWS necessárias.

## Pré-requisitos

Certifique-se de ter o seguinte antes de iniciar qualquer um dos métodos de configuração:

### Obrigatório para todas as configurações

* **Conta da AWS com faturamento consolidado**: [Configurar o faturamento consolidado para o AWS Organizations](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html). Isso é necessário para agregar dados de custo de todas as contas da sua organização, possibilitando uma análise de custos abrangente em toda a sua infraestrutura da AWS.
* **Privilégios do IAM**: O acesso administrativo à conta da AWS de destino é necessário para implantar recursos, criar buckets do S3, políticas do IAM e funções do IAM.

### Necessário apenas para configuração automatizada

* **Chave de API da New Relic**: Gere uma chave de API válida para a criação da stack. Recomendamos criar uma chave dedicada para a integração CCI. Gere isso na interface do New Relic: [Menu do usuário &gt; API Keys](https://onenr.io/07wklp68lRL).

## Escolha seu método de configuração

O Cloud Cost Intelligence oferece duas maneiras de configurar a integração com a AWS:

* [**Automatize a AWS com o CloudFormation (Recomendado)**](/docs/cci/get-started/aws/automated-setup-cloudformation): Implante rapidamente uma pilha do CloudFormation pré-configurada que provisiona automaticamente todos os recursos necessários da AWS. Melhor para uma integração mais rápida com configuração manual mínima.
* [**Integre manualmente sua conta da AWS**](/docs/cci/get-started/aws/manual-integrate-aws-account): Configure manualmente cada recurso da AWS para ter controle total sobre o processo de configuração. Melhor para organizações com requisitos específicos de conformidade ou personalização.
