Configure seu ambiente AWS para <DNT>**Cloud Cost Intelligence**</DNT> configurando o faturamento consolidado, os Relatórios de custos e uso (CUR), as funções e políticas do Identity and Access Management (IAM) e as regiões da AWS necessárias.

## Pré-requisitos

Certifique-se de ter o seguinte antes de iniciar qualquer um dos métodos de configuração:

### Obrigatório para todas as configurações

* <DNT>**AWS account with consolidated billing**</DNT>: [Configurar o faturamento consolidado para o AWS Organizations](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html). Isso é necessário para agregar dados de custo de todas as contas da sua organização, possibilitando uma análise de custos abrangente em toda a sua infraestrutura da AWS.
* <DNT>**IAM Privileges**</DNT>: O acesso administrativo à conta da AWS de destino é necessário para implantar recursos, criar buckets do S3, políticas do IAM e funções do IAM.

### Necessário apenas para configuração automatizada

* <DNT>**New Relic API Key**</DNT>: gere uma chave de API válida para a criação da stack. Recomendamos criar uma chave dedicada para a integração <DNT>**CCI**</DNT>. Gere isso na interface do New Relic: [menu do usuário &gt; API Keys](https://onenr.io/07wklp68lRL).

## Escolha seu método de configuração

<DNT>**Cloud Cost Intelligence**</DNT> oferece duas maneiras de configurar a integração com a AWS:

* <DNT>[**Automate AWS with CloudFormation (Recommended)**](/docs/cci/get-started/aws/automated-setup-cloudformation)</DNT>: implante rapidamente uma stack do CloudFormation pré-configurada que provisiona automaticamente todos os recursos necessários da AWS. Melhor para uma integração mais rápida com configuração manual mínima.
* <DNT>[**Manually integrate your AWS account**](/docs/cci/get-started/aws/manual-integrate-aws-account)</DNT>: configure manualmente cada recurso da AWS para ter controle total sobre o processo de configuração. Melhor para organizações com requisitos específicos de conformidade ou personalização.
