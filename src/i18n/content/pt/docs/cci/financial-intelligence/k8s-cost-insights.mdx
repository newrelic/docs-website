Acesse insights específicos para seus ambientes Kubernetes por meio da página <DNT>**Kubernetes Costs**</DNT>. Este recurso utiliza os dados de telemetria do New Relic para fornecer uma visão detalhada dos custos associados ao seu cluster do Kubernetes.

Use a página <DNT>**Kubernetes Costs**</DNT> para visualizar os custos incorridos por cada aplicativo ao longo do tempo e uma análise detalhada dos custos.

<img title="Kubernetes Cost Insights highlighted on the Cloud Cost Intelligence Kubernetes Cost Insights page" alt="Kubernetes Cost Insights highlighted on the Cloud Cost Intelligence Kubernetes Cost Insights page" src="/images/k8s-cci.webp" />

## Filtros e personalização

### Filtros de provedor de nuvem

Para filtrar seus dados de custo por provedor de cloud, selecione uma opção no menu dropdown no canto superior esquerdo da página:

* <DNT>**All**</DNT>: Todos os provedores de cloud
* <DNT>**AWS**</DNT>: apenas Amazon Web Services
* <DNT>**Azure**</DNT>: apenas Microsoft Azure

### Seletor de intervalo de datas

No canto superior direito da página, use o seletor de intervalo de datas para:

* Escolha um período de tempo predefinido
* Defina datas de início e fim personalizadas
* Visualize os custos no seu fuso horário preferido (UTC exibido por padrão)

### Filtro de custos de engenharia

Se a sua organização usa tags de alocação de custos, você pode filtrar pela tag <DNT>**Engineering costs**</DNT> para restringir a análise a equipes, projetos ou centros de custo específicos marcados com tag no seu ambiente de cloud.

## Resumo

A seção Summary exibe quatro principais métricas de custos para o período selecionado:

* <DNT>**Total cost**</DNT>: O gasto agregado para todos os clusters do Kubernetes com a variação percentual em comparação com o período anterior
* <DNT>**Average cost (Daily)**</DNT>: O gasto médio diário com variação percentual
* <DNT>**Unused cost**</DNT>: O custo dos recursos que foram alocados, mas não estão recebendo requests, com variação percentual
* <DNT>**Unallocated cost**</DNT>: O custo dos recursos do cluster que não foram alocados a nenhuma fonte, com variação percentual

## Insights

A seção Insights fornece dois widgets de visualização para ajudar você a analisar seus custos do Kubernetes.

### Tendências de Custos

O widget <DNT>**Cost Trends**</DNT> exibe um gráfico de linhas mostrando como seus custos do Kubernetes mudam ao longo do tempo. Use as opções de visualização para alternar entre as visualizações <DNT>**Hourly**</DNT>, <DNT>**Daily**</DNT>, <DNT>**Weekly**</DNT> ou <DNT>**Monthly**</DNT>.

### Top 20

O widget <DNT>**Top 20**</DNT> exibe suas áreas de maior custo usando uma visualização em gráfico de árvore. Por padrão, ele mostra os 20 principais clusters por custo. Você pode personalizar a view usando:

* <DNT>**Dropdown filter**</DNT>: Selecione o que exibir (<DNT>**Cluster name, by cost**</DNT>)
* <DNT>**Group by**</DNT>: Escolha dimensões para agrupar dados de custo
* <DNT>**View by**</DNT>: Alternar entre <DNT>**Tree chart**</DNT> e outros formatos de visualização

## Cost &amp; Usage Analysis

A seção <DNT>**Cost &amp; Usage Analysis**</DNT> apresenta uma tabela detalhada com uma análise do uso de recursos e custos por nome do cluster:

* <DNT>**Cluster name**</DNT>: O nome do seu cluster do Kubernetes
* <DNT>**CPU cost**</DNT>: O custo atribuído ao uso de CPU para cada cluster
* <DNT>**CPU usage**</DNT>: a porcentagem de recursos de CPU consumidos por cada cluster
* <DNT>**Memory cost**</DNT>: O custo associado ao uso de memória
* <DNT>**Memory usage**</DNT>: Os recursos de memória por porcentagem usada por cada cluster
* <DNT>**Unused Cost**</DNT>: O custo de recursos alocados, mas não utilizados
* <DNT>**Unallocated cost**</DNT>: O custo dos recursos do cluster não alocados a nenhuma fonte
* <DNT>**Total cost**</DNT>: O custo total incorrido por cada cluster, integrando todo o uso de recursos (ordenável)

## Entenda os custos não utilizados e não alocados

Essas dimensões de custo ajudam você a medir a eficiência do cluster:

### Custo não utilizado

Custo não utilizado representa recursos que foram alocados, mas não estão recebendo requests.

**Exemplo**: Se seu custo total for de US$ 5,60 alocados para uma implantação de cluster, mas você estiver usando apenas US$ 4,64, então US$ 0,96 é o custo não utilizado.

### Custo não alocado

Custo não alocado representa recursos do cluster que não foram alocados a nenhuma fonte.

Essas métricas ajudam você a identificar oportunidades para melhorar a eficiência do cluster e reduzir o desperdício.

<Callout variant="tip">
  Selecione qualquer cluster para visualizar seu <DNT>**Cost Trends**</DNT>, <DNT>**Cost Breakdown (Total vs Used)**</DNT> e informações detalhadas sobre custos não utilizados e não alocados.
</Callout>

## Benefícios

O recurso de Insights de Custo do Kubernetes oferece estes resultados:

* <DNT>**Optimized resource usage**</DNT>: A visibilidade de clusters subutilizados ou excessivamente caros permite ajustes eficazes na alocação de recursos
* <DNT>**Controlled expenses**</DNT>: Insights sobre os direcionadores de custos específicos em seu ambiente Kubernetes permitem decisões financeiras mais informadas.
* <DNT>**Enhanced efficiency**</DNT>: Estratégias de gerenciamento de recursos alinhadas aos objetivos de negócios garantem o uso eficiente dos recursos cloud

A página <DNT>**Kubernetes Costs**</DNT> fornece os insights necessários para gerenciar proativamente os custos do Kubernetes, apoiando a tomada de decisões estratégicas e a otimização financeira.
