Monitore a integridade da integração e gerencie conexões de provedores de nuvem na AWS, Azure e GCP com ações baseadas em status.

<Callout variant="important">
  A aba Integrações fica acessível depois que você conclui com sucesso pelo menos uma integração de provedor de nuvem.
</Callout>

## Acessar integrações

1. Vá para <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> &gt; selecione <DNT>**All Capabilities**</DNT> &gt; <DNT>**Cloud Cost Intelligence**</DNT>.
2. Selecione a aba <DNT>**Integrations**</DNT> para visualizar todas as integrações de custo de cloud.
3. Selecione <DNT>**+ Add new integrations**</DNT> para adicionar uma nova integração para uma conta de provedor de cloud.
4. Verifique a coluna <DNT>**Status**</DNT> para avaliar a saúde da integração em seus provedores de cloud.

## Entenda o status da integração

Cada integração exibe um dos seguintes indicadores de status:

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        Status
      </th>

      <th>
        Descrição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**PENDING**</DNT>
      </td>

      <td>
        A integração foi criada e está aguardando validação pelo CCI. Este é o status inicial atribuído quando uma nova integração é configurada pela primeira vez.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**IN PROGRESS**</DNT>
      </td>

      <td>
        A configuração da conexão está em andamento; a configuração inicial e a autenticação estão sendo estabelecidas.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**VALIDATED**</DNT>
      </td>

      <td>
        A conexão foi autenticada com sucesso e todas as verificações de pré-requisitos passaram; aguardando a publicação do primeiro arquivo de custo na conta de armazenamento.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**RUNNING**</DNT>
      </td>

      <td>
        A integração está totalmente operacional e coletando ativamente dados de custo.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**FAILED**</DNT>
      </td>

      <td>
        A conexão encontrou um erro durante a configuração inicial ou a validação do relatório CUR e requer atenção. Clique no nome da conexão para visualizar os detalhes do erro e as etapas de resolução.

        <Callout variant="important">
          <DNT>**FAILED**</DNT> aplica-se apenas durante a fase de configuração inicial da integração, antes que a conexão atinja o status <DNT>**RUNNING**</DNT>.
        </Callout>
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**ERROR**</DNT>
      </td>

      <td>
        Uma conexão anteriormente <DNT>**RUNNING**</DNT> encontrou problemas de integração. Ao contrário de <DNT>**FAILED**</DNT>, que ocorre durante a configuração inicial, <DNT>**ERROR**</DNT> indica um problema com uma conexão já ativa. Clique no nome da conexão para visualizar os detalhes do erro e as etapas de resolução.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**PAUSED**</DNT>
      </td>

      <td>
        A coleta de dados foi temporariamente suspensa. Você pode pausar uma conexão depois que ela atingir o status <DNT>**RUNNING**</DNT> e retomá-la mais tarde.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Monitore a coluna <DNT>**Status**</DNT> regularmente para garantir que as integrações permaneçam <DNT>**RUNNING**</DNT>. Resolva prontamente as integrações <DNT>**FAILED**</DNT> ou <DNT>**ERROR**</DNT> para evitar lacunas na coleta de dados de custo e revise os requisitos de validação durante a fase de <DNT>**IN PROGRESS**</DNT> para agilizar a ativação.
</Callout>

## Gerenciar integrações

Clique no ícone de reticências ao lado de uma integração para acessar as ações disponíveis. As ações exibidas dependem do status atual da integração:

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        Ação
      </th>

      <th style={{ width: "150px" }}>
        Disponível para
      </th>

      <th>
        Descrição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Resume**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> integrações (apenas onde a integração foi criada com sucesso, mas a validação do relatório CUR falhou) e <DNT>**PAUSED**</DNT> integrações
      </td>

      <td>
        Retoma a coleta de dados ou tenta novamente o processo de configuração da conexão. Para integrações <DNT>**FAILED**</DNT> em que a própria integração não foi criada devido a um erro de validação, a opção de retomar não está disponível. Para integrações <DNT>**PAUSED**</DNT>, retomar restaura a coleta ativa de dados.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> integrações
      </td>

      <td>
        Suspende temporariamente a coleta de dados, mantendo a configuração.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> integrações (antes de atingir o status <DNT>**RUNNING**</DNT>)
      </td>

      <td>
        Remove permanentemente a integração e permite uma nova configuração de conexão.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Use <DNT>**Pause**</DNT> em vez de <DNT>**Delete**</DNT> para suspensões temporárias para preservar sua configuração de integração. Use <DNT>**Resume**</DNT> para restaurar uma integração <DNT>**PAUSED**</DNT> ou tentar novamente uma integração <DNT>**FAILED**</DNT> em que a conexão foi criada, mas a validação do relatório CUR falhou.
</Callout>
