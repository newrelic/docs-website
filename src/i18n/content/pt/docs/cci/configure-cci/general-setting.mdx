O Cloud Cost Intelligence suporta marcação personalizada juntamente com as tags padrão em Relatórios de Custo e Uso (CUR) para alocação de custo de recursos. Filtre e agrupe tags para análise detalhada de despesas. Em ambientes Kubernetes, aplique rótulos a clusters e recursos. As listas suspensas de seleção de tags são originadas dos dados CUR e do Evento `K8SContainerSample`.

## Configurar configurações

<Steps>
  <Step>
    ### Navegue para as configurações gerais

    Vá para aba <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT> &gt; **General**.
  </Step>

  <Step>
    ### Configurar tag

    Selecione as tags de recursos e rótulos do Kubernetes de arquivos de custo e uso:

    1. Na lista **Choose Resource tags for cost allocation**, selecione as tags de recursos para categorizar os custos da nuvem entre os serviços.
    2. Na lista **Choose Kubernetes tags for cost mapping**, selecione os rótulos do Kubernetes para categorizar os custos.
  </Step>

  <Step>
    ### Gerenciar acesso a dados de telemetria

    1. Selecione a guia **Telemetry Data**.
    2. Na lista **Select accounts for telemetry data**, escolha as contas para dados de telemetria.

    <Callout variant="important">
      A estimativa de custo em tempo real e a alocação K8s são suportadas por meio de dados de telemetria. Reutiliza o pipeline de telemetria existente para métricas de utilização. Nenhum dado novo ingerido, exceto a ingestão de arquivos CUR.
    </Callout>
  </Step>
</Steps>
