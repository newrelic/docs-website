A Inteligência de Custos na Nuvem suporta tag personalizada junto com tag padrão, integrada aos Relatórios de Custo e Utilização (CUR) para alocação de custos de recursos. Isso permite que você filtre e agrupe tags para uma análise detalhada de despesas. Em ambientes Kubernetes , rótulos podem ser aplicados ao cluster e aos recursos. o menu suspenso para seleção tag é originado de dados CUR e `K8SContainerSample` evento, simplificando o gerenciamento e a análise.

## Configurar configurações gerais

<Steps>
  <Step>
    ## Navegue até Configurações gerais

    1. Navegue até <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT>.
    2. No menu de navegação à esquerda, selecione a aba **General** .
  </Step>

  <Step>
    ## Configurar tag

    Selecione a tag de recursos desejada e os rótulos Kubernetes a partir dos valores obtidos por meio dos arquivos de custo e uso para otimizar o gerenciamento e a análise

    1. Para categorizar e detalhar os custos cloud entre os serviços, na lista **Choose Resource tags for cost allocation** , selecione a tag de recurso.
    2. Para categorizar e detalhar os custos cloud entre os serviços, na lista **Choose Kubernetes tags for cost mapping** , selecione os rótulos Kubernetes .
  </Step>

  <Step>
    ## Gerenciar acesso a dados de telemetria

    1. Para solicitar informações sobre as contas das quais você deseja extrair dados de telemetria para análise de custos, selecione a aba **Telemetry Data** .
    2. Para visualizar e gerenciar dados de telemetria para sua análise de custos, na lista **Select accounts for telemetry data** , escolha uma ou mais contas.

    <Callout variant="important">
      * A estimativa de custos em tempo real é suportada por dados de telemetria.
      * A alocação de K8s é suportada por dados de telemetria.
      * Reutilizamos o pipeline de telemetria existente para obter a métrica de utilização, e nenhum dado novo é ingerido, exceto a ingestão do arquivo CUR.
    </Callout>
  </Step>
</Steps>
