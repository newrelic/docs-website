Quando o desempenho do sistema é prejudicado, você precisa saber se isso se deve ao seu código, à sua infraestrutura ou à rede subjacente. E você precisa saber rápido, para poder concentrar seus esforços. Com o Monitoramento de rede você pode correlacionar e analisar aplicativo, infraestrutura, experiência digital e dados completos da rede e entender como o desempenho da rede e o desempenho geral do sistema impactam um ao outro.

<Callout variant="tip">
  O monitoramento de rede agora está incluído na New Relic [autorização moderada FedRAMP!](/docs/security/security-privacy/compliance/data-encryption/)

  Use a instalação guiada para [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#setup-snmp-monitoring), [fluxos de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#setup-network-flow-monitoring) e [syslog de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#setup-network-syslog-monitoring) para detectar e configurar automaticamente os [endpoints de API autorizados pelo FedRAMP](/docs/security/security-privacy/compliance/fedramp-compliant-endpoints/#data-ingest-apis) ou revise o processo de configuração manual em nossos documentos para [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#manual-setup), [fluxos de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#manual-setup) e [syslog de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#manual-setup).
</Callout>

O monitoramento de rede adiciona o contexto dos dados de rede aos dados de aplicativo e infraestrutura que você já coleta no New Relic. Ao monitorar os dados da sua rede, você pode:

* Analise e entenda o desempenho de toda a sua stack (aplicativo e infraestrutura) para uma compreensão holística do desempenho do seu sistema.
* Tenha todos os dados em uma única plataforma para eliminar pontos cegos.
* Veja à primeira vista se uma rede está implicada em um problema.

## Tipos de dados de rede [#network-data-types]

Você pode monitor os seguintes tipos de dados de rede:

* <DNT>
    **Device performance via SNMP**
  </DNT>

  : Simple Network Management Protocol (SNMP) é um protocolo de camada de aplicativo para troca de informações de gerenciamento entre dispositivos de rede. Para enviar dados SNMP para o New Relic, [configure o monitoramento de dados SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring).

* <DNT>
    **Network syslogs**
  </DNT>

  : fornece insights sobre o que mudou, erros e outros eventos do dispositivo para entender rapidamente o 'Por quê?' por trás das mudanças de desempenho em telemetria. Para enviar dados de syslog de rede, [configure o monitoramento de syslog de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring).

* <DNT>
    **Cloud flow logs**
  </DNT>

  : detalha as comunicações entre hosts e outros serviços, capturando o endereço IP e a porta de origem e destino, metadados que descrevem o endpoint, a telemetria chave de desempenho da rede e o estado de uma tentativa de comunicação específica. [Configure o monitoramento de log de fluxo da nuvem](/docs/network-performance-monitoring/setup-performance-monitoring/aws-vpc-flow-monitoring).

* <DNT>
    **Network flow logs**
  </DNT>

  : captura informações sobre o tráfego IP que entra e sai das interfaces de rede na sua rede local. Para enviar dados de fluxo de rede para o New Relic, [configure o monitoramento de dados de fluxo de rede](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring).

<Callout variant="important">
  Recomendamos configurar o SNMP e os dados de fluxo de rede para melhor visibilidade da sua rede. Isso fornecerá métricas de desempenho e padrões de tráfego para solucionar problemas e otimizar sua rede.
</Callout>
