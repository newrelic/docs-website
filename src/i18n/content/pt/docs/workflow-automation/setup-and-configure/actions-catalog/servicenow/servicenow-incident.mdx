Esta página fornece uma referência para as ações de incidente do ServiceNow disponíveis no catálogo de ações de automação de fluxo de trabalho. Essas ações permitem criar e gerenciar incidentes no ServiceNow.

## Pré-requisitos

Antes de usar as ações do ServiceNow na automação de fluxo de trabalho, certifique-se de ter:

* Uma instância do ServiceNow com as permissões apropriadas.
* Uma chave de API REST do ServiceNow para autenticação (passada como o cabeçalho `x-sn-apikey`).
* As permissões necessárias para criar e gerenciar incidentes na sua instância do ServiceNow.

Na entrada da ação, `apiKey` deve ser fornecido. É usado para autenticar o usuário — a chave é passada como parte do cabeçalho da requisição `x-sn-apikey`.

## Configurar a autenticação por chave de API

Para obter mais informações, consulte [Criar política de acesso à API REST com perfis de autenticação de chave de API](https://www.servicenow.com/docs/r/platform-security/authentication/create-api-access-policy.html) na documentação da ServiceNow.

Para gerar e configurar uma chave de API REST no ServiceNow:

<Steps>
  <Step>
    ### Criar um Perfil de Autenticação de Entrada

    1. Navegue até <DNT>**System Web Services &gt; API Access Policies &gt; Inbound Authentication Profile**</DNT>.
    2. Crie um novo **perfil de autenticação de chave de API**.
    3. Defina o **Auth Parameter** como um cabeçalho HTTP (recomendado) usando o nome do parâmetro `x-sn-apikey`.
  </Step>

  <Step>
    ### Gerar uma Chave de API REST

    1. Navegue até <DNT>**System Web Services &gt; API Access Policies &gt; REST API Key**</DNT>.
    2. Crie uma nova chave associada a uma conta de serviço dedicada que tenha as funções e permissões apropriadas.
    3. Salve e copie o valor do token gerado — é uma chave secreta.
  </Step>

  <Step>
    ### Configurar uma Política de Acesso à API

    1. Navegue até <DNT>**System Web Services &gt; API Access Policies &gt; REST API Access Policies**</DNT>.
    2. Crie uma nova política, selecione a API REST de destino (por exemplo, **Table API** ou uma **API REST com script**).
    3. Adicione o **Inbound Authentication Profile** à lista relacionada desta política.
  </Step>
</Steps>
