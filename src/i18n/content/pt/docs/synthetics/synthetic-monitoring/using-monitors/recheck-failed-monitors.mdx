Você pode usar as novas verificações do monitoramento sintético para executar novamente uma verificação de monitor com falha. Todas as novas verificações começam imediatamente em um único local. Isso facilita a verificação de um único local ou resultado com falha.

Além disso, todos os dados de nova verificação aparecem como um ponto de dados e resultado do monitor padrão. Isso permite visualizar as verificações do monitor no contexto do desempenho geral do monitor.

## Visualizar e verificar novamente os resultados do monitor com falha [#ui]

Use o <Icon name="fe-rotate-cw"/> Os botões <DoNotTranslate>**Run check**</DoNotTranslate> aparecerão nas páginas de detalhes dos resultados do monitor se:

* O resultado tem um status `FAILED` .
* O resultado ocorreu nas últimas 24 horas.
* O monitor não foi editado desde que o resultado ocorreu.

Para visualizar os resultados de um monitor com falha e verificá-lo novamente:

1. Vá para <DoNotTranslate>**[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DoNotTranslate> e selecione um monitor.

2. Na página <DoNotTranslate>**Failures**</DoNotTranslate> do monitor, selecione um link da tabela.

   OU

   Na [página<DoNotTranslate>**Results**</DoNotTranslate> ](/docs/synthetics/new-relic-synthetics/using-monitors/view-monitor-results)do monitor, use as opções de filtro ou classificação para selecionar um local indicando <DoNotTranslate>**Results: Failed**</DoNotTranslate>.

3. Revise os detalhes da verificação com falha e clique em <Icon name="fe-rotate-cw"/> <DoNotTranslate>**Run check**</DoNotTranslate>.

4. Para visualizar o status atualizado, clique em <DoNotTranslate>**See results**</DoNotTranslate>.

Quando você clica <Icon name="fe-rotate-cw"/> <DoNotTranslate>**Run check**</DoNotTranslate>, o New Relic aciona uma nova verificação imediata usando a localização do resultado e a configuração atual do monitor. Assim que o trabalho for concluído, o botão exibirá o status da nova verificação (`Success` ou `Failure`). Para visualizar a página de nova verificação <DoNotTranslate>**Results**</DoNotTranslate> novamente, clique no botão novamente.

Executar uma nova verificação também log um evento de auditoria Sintético que você pode [consultar usando NRQL](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/query-infrastructure-dimensional-metrics-nrql). Por exemplo:

```
SELECT * from NrAuditEvent 
WHERE actionIdentifier = 'synthetics_monitor.re_check_job'
```
