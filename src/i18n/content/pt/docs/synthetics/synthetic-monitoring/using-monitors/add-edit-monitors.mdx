Monitor Sintético são verificações de API ou instância de browser virtual que monitor seu site, registrando cada verificação detalhadamente. monitor pode:

* Capture números agregados, incluindo uma visão geral ou resumo.
* Fornece estatísticas detalhadas sobre o período de inatividade incidente e os recursos da página.
* Colete códigos de resposta personalizados que detalham o desempenho do seu monitor.

Você pode criar monitor Sintético sem instalação. Para começar:

1. Crie uma [contaNew Relic ](https://newrelic.com/signup).

2. Vá para

   <DNT>
     **[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**
   </DNT>

   e clique em **Criar um monitor**.

3. Selecione o monitor que você deseja criar. Se for sua primeira vez, recomendamos criar um monitor de ping ou de passos para começar.

## Crie um monitor [#adding-monitors]

Pronto para monitor suas aplicações web com um (ou vários!) de nossos [monitores Sintético](/docs/synthetics/synthetic-monitoring/using-monitors/intro-synthetic-monitoring/#types-of-synthetic-monitors)? As etapas abaixo orientam você no processo para todos os nossos monitores.

       Você tem algumas opções de configuração adicionais para o ping ou monitor simples do browser. Para ambos os tipos de monitor, você pode configurar:

    * Um <DNT>**validation string**</DNT> que permite o monitoramento de substring para [validação de resposta](#response-validation).
    * <DNT>**Custom Headers**</DNT> que acompanhará as solicitações enviadas pelo seu monitor.
    * <DNT>**Verify SSL**</DNT> para validar a cadeia de certificados SSL. Ele pode ser duplicado executando a seguinte sintaxe:

    ```
    openssl s_client -servername {YOUR_HOSTNAME} -connect {YOUR_HOSTNAME}:443 -CApath /etc/ssl/certs -verify_hostname {YOUR_HOSTNAME} > /dev/null
    ```

    Se um código de saída diferente de zero for retornado, o monitor falhará.

    * <DNT>**Custom Headers**</DNT> pode ser adicionado ao ping e ao monitor simples do browser. Esses cabeçalhos serão adicionados às solicitações enviadas pelo monitor.

## Editar um monitor [#editing-monitors]

<img
  src="/images/synthetic_screenshot-full_edit-monitor-from-general.webp"
  title="Edit a monitor page"
  alt="A screenshot that shows the New Relic UI when you're editing your monitor."
/>

<figcaption>
  <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets) &amp;gt; General**</DNT>: Mantenha seu monitor atualizado para que a New Relic receba sempre os dados mais pertinentes.
</figcaption>

Você não pode alterar monitor o [tipo de um após](#setting-type) monitor ser criado, mas pode editar outras monitor configurações .

1. Vá para <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> e selecione o monitor que deseja editar.

2. No menu lateral, selecione um link para alterar as seguintes [configurações](#settings):

   * Selecione

     <DNT>
       **Settings > General**
     </DNT>

     para editar [nome](#setting-name), URL, [locais](#setting-location), [frequência](#setting-frequency) e opções avançadas.

   * Para monitores

     <DNT>
       **Scripted browser**
     </DNT>

     e

     <DNT>
       **API test**
     </DNT>

     , selecione

     <DNT>
       **Settings > Script**
     </DNT>

     para editar o script do monitor .

   * Para alertas de monitoramento sintético, clique em

     <DNT>
       **Manage alerts**
     </DNT>

     .

3. Selecione <DNT>**Save changes**</DNT> para confirmar.

<Callout variant="important">
  Converta para o [novo tempo de execução do Synthetics](/docs/synthetics/synthetic-monitoring/getting-started/new-runtime) ao editar seu monitor. Certifique-se de atualizar sua lista de permissões para aceitar solicitações de rede de [novos intervalos de IP públicos](/docs/synthetics/synthetic-monitoring/administration/new-synthetic-public-minion-ips).
</Callout>

## Configurações do monitor [#settings]

Ao configurar o monitor, as seguintes configurações estão disponíveis:

<CollapserGroup>
  <Collapser id="setting-type" title="Tipo">
    Selecione o tipo de monitor que deseja criar. O [tipo](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) de um monitor não pode ser alterado após a criação do monitor.

    * <DNT>
        **Ping**
      </DNT>

      : especifique um único URL para monitor a disponibilidade. A New Relic verificará esta URL via HEAD ou solicitação GET. O [tempo limite](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) não configurável para este monitor é de 60 segundos.

    * <DNT>
        **Simple browser**
      </DNT>

      : Especifique um único URL para monitor via browser real. A cada [intervalo de frequência](#setting-frequency), a New Relic verificará esse URL por meio de um browser Google Chrome com tecnologia Selenium. O [tempo limite](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers/#waiting-elements) não configurável para este monitor é de 60 segundos.

    * <DNT>
        **Scripted browser**
      </DNT>

      : [crie um script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) para controlar um browser Google Chrome com tecnologia Selenium. O browser segue cada etapa do script para verificar se o comportamento complexo está funcionando conforme o esperado (por exemplo, pesquisar um site e clicar em um dos resultados da pesquisa). O [tempo limite](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) não configurável para este monitor é de 180 segundos.

    * <DNT>
        **API test**
      </DNT>

      : Crie um script de API para garantir que seus endpoints de API estejam funcionando corretamente. Para obter mais informações, consulte [Escrever testes de API](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests). O tempo limite não configurável para este monitor é de 180 segundos.
  </Collapser>

  <Collapser id="setting-name" title="Nome do monitor">
    Define um nome para o monitor. os nomes monitor não podem conter colchetes angulares não codificados (`<` e `>`). Para incluir colchetes angulares em um nome de monitor, codifique-os como entidade de colchetes HTML (`&lt;` para `<` e `&gt;` para `>`) na interface ou API.
  </Collapser>

  <Collapser id="setting-location" title="Localização">
    Selecione os locais onde deseja que seu monitor seja executado. Selecione mais locais para garantir que seu aplicativo esteja disponível para usuários em todo o mundo. Se você tiver alguma [localização privada](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations), ela também estará listada aqui. Você pode usar o [endpoint `location` da API Sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#list-locations) para recuperar uma lista de locais válidos para sua conta.

    Seu monitor executará uma verificação em cada local selecionado durante cada [intervalo de frequência](#setting-frequency). Por exemplo, se você selecionar três locais e definir uma frequência de 15 minutos, seu monitor executará três verificações em cada período de 15 minutos (ou 8.640 verificações por mês).
  </Collapser>

  <Collapser id="setting-frequency" title="Frequência">
    Selecione a frequência de execução do monitor, em incrementos de minutos, horas ou 1 dia. Esta frequência se aplica a cada [local](#setting-location). Por exemplo, se você selecionar três locais e uma frequência de <DNT>**15 minutes**</DNT>, seu monitor executará três verificações, em média, a cada 5 minutos, em cada período de 15 minutos (ou 8.640 verificações por mês).
  </Collapser>

  <Collapser id="response-validation" title="Validação de resposta (opcional)">
    Especifique o texto a ser pesquisado na página DOM. Ao usar browser simples ou [tipos de monitor](/docs/synthetics/synthetic-monitoring/getting-started/types-synthetic-monitors/#types-monitors) de ping, há um limite de 1 MB (10 ^ 6 bytes) no carregamento da página.
  </Collapser>

  <Collapser id="apdex-t" title="Apdex T (opcional)">
    Especifique o limite para o <DNT>**tolerable**</DNT> tempo de resposta do seu monitor. O valor padrão é 7 segundos (7.000 ms). Para obter mais informações, consulte [Entender a métrica do relatório de SLA](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-aggregate-monitor-metrics/#understanding).
  </Collapser>
</CollapserGroup>
