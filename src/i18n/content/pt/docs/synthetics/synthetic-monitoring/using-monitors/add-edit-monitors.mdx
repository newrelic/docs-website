Monitor Sintético são verificações de API ou instância de browser virtual que monitor seu site, registrando cada verificação detalhadamente. monitor pode:

* Capture números agregados, incluindo uma visão geral ou resumo.
* Fornece estatísticas detalhadas sobre o período de inatividade incidente e os recursos da página.
* Colete códigos de resposta personalizados que detalham o desempenho do seu monitor.

Você pode criar monitor Sintético sem instalação. Para começar:

1. Crie uma [contaNew Relic ](https://newrelic.com/signup).

2. Vá para

   <DoNotTranslate>**[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DoNotTranslate>

   e clique em **Criar um monitor**.

3. Selecione o monitor que você deseja criar. Se for sua primeira vez, recomendamos criar um monitor de ping ou de passos para começar.

## Crie um monitor [#adding-monitors]

Pronto para monitor suas aplicações web com um (ou vários!) de nossos [monitores Sintético](/docs/synthetics/synthetic-monitoring/using-monitors/intro-synthetic-monitoring/#types-of-synthetic-monitors)? As etapas abaixo orientam você no processo para todos os nossos monitores.

<CollapserGroup>
  <Collapser
    id="simple"
    title="Adicione um ping ou um monitor de browser simples"
  >
    1. Vá para

       <DoNotTranslate>**[one.newrelic.com > Synthetic monitoring > Create monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DoNotTranslate>

    2. Clique no bloco

       <DoNotTranslate>**Availability**</DoNotTranslate>

       para monitor de ping ou

       <DoNotTranslate>**Page load performance**</DoNotTranslate>

       para monitor de browser simples.

    3. Opcional: Configure [a notificação de alerta](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics).

    Aguarde alguns minutos e verifique seu monitor no [índice<DoNotTranslate>**Monitors**</DoNotTranslate> ](/docs/new-relic-one-monitors-index).

    Você tem algumas opções de configuração adicionais para o ping ou monitor simples do browser. Para ambos os tipos de monitor, você pode configurar:

    * Um <DoNotTranslate>**validation string**</DoNotTranslate> que permite o monitoramento de substring para [validação de resposta](#response-validation).

    * <DoNotTranslate>**Custom Headers**</DoNotTranslate> que acompanhará as solicitações enviadas pelo seu monitor.

    * <DoNotTranslate>**Verify SSL**</DoNotTranslate> para validar a cadeia de certificados SSL. Ele pode ser duplicado executando a seguinte sintaxe:

      ```
      openssl s_client -servername {YOUR_HOSTNAME} -connect {YOUR_HOSTNAME}:443 -CApath /etc/ssl/certs -verify_hostname {YOUR_HOSTNAME} > /dev/null
      ```

      Se um código de saída diferente de zero for retornado, o monitor falhará.

    Com apenas o monitor de ping, você pode optar por configurar:

    * A opção

      <DoNotTranslate>**Bypass HEAD request**</DoNotTranslate>

      para ignorar a solicitação `HEAD` padrão e usar o verbo `GET` .

    * <DoNotTranslate>**Redirect is Failure**</DoNotTranslate>

      , que categorizará o redirecionamento como uma falha, em vez de seguir o redirecionamento para o novo URL.

    * <DoNotTranslate>**Custom Headers**</DoNotTranslate>

      pode ser adicionado ao ping e ao monitor simples do browser. Esses cabeçalhos serão adicionados às solicitações enviadas pelo monitor.

    Com apenas um simples monitor de browser, você pode definir as configurações [de emulação do dispositivo](/docs/synthetics/synthetic-monitoring/using-monitors/device-emulation) , incluindo o tipo de dispositivo e a orientação da tela.
  </Collapser>

    5. Selecione os [locais](#setting-location) onde você deseja que seu monitor seja executado.

    6. Opcional: Configure [a notificação de alerta](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics).

    7. Selecione <DoNotTranslate>**Next: Write your script**</DoNotTranslate> para criar um script para [<DoNotTranslate>**scripted browser**</DoNotTranslate>](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) ou [<DoNotTranslate>**API test**</DoNotTranslate>](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests) e selecione <DoNotTranslate>**Validate**</DoNotTranslate> para verificar sua sintaxe.

Quer implantar mais de um monitor por vez? Recomendamos verificar o [tutorial de monitoramento sintético da API NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-synthetics-tutorial). Você pode automatizar a criação monitor com chamada de API.

Você também pode criar monitores com nossa [API REST de monitoramento sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api). Para REST API, faça uma solicitação `GET` para recuperar os detalhes de configuração do monitor Sintético escolhido. Depois de recuperar esses dados, use `POST` para criar uma "cópia" do monitor escolhido.

## Editar um monitor [#editing-monitors]

<img
  src={syntheticEditMonitorFromGeneral}

<figcaption>
  <DoNotTranslate>**[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets) > General**</DoNotTranslate>: Mantenha seu monitor atualizado para que a New Relic receba sempre os dados mais pertinentes.
</figcaption>

Você não pode alterar monitor o [tipo de um após](#setting-type) monitor ser criado, mas pode editar outras monitor configurações .

1. Vá para <DoNotTranslate>**[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DoNotTranslate> e selecione o monitor que deseja editar.

2. No menu lateral, selecione um link para alterar as seguintes [configurações](#settings):

   * Selecione

     <DoNotTranslate>**Settings > General**</DoNotTranslate>

     para editar [nome](#setting-name), URL, [locais](#setting-location), [frequência](#setting-frequency) e opções avançadas.

   * Para monitores

     <DoNotTranslate>**Scripted browser**</DoNotTranslate>

     e

     <DoNotTranslate>**API test**</DoNotTranslate>

     , selecione

     <DoNotTranslate>**Settings > Script**</DoNotTranslate>

     para editar o script do monitor .

   * Para alertas de monitoramento sintético, clique em

     <DoNotTranslate>**Manage alerts**</DoNotTranslate>

     .

3. Selecione <DoNotTranslate>**Save changes**</DoNotTranslate> para confirmar.

<Callout variant="important">
  Converta para o [novo tempo de execução do Synthetics](/docs/synthetics/synthetic-monitoring/getting-started/new-runtime) ao editar seu monitor. Certifique-se de atualizar sua lista de permissões para aceitar solicitações de rede de [novos intervalos de IP públicos](/docs/synthetics/synthetic-monitoring/administration/new-synthetic-public-minion-ips).
</Callout>

## Configurações do monitor [#settings]

Ao configurar o monitor, as seguintes configurações estão disponíveis:

<CollapserGroup>
  <Collapser
    id="setting-type"
    title="Tipo"
  >
    Selecione o tipo de monitor que deseja criar. O [tipo](/docs/synthetics/new-relic-synthetics/getting-started/types-synthetics-monitors#types-monitors) de um monitor não pode ser alterado após a criação do monitor.

    * <DoNotTranslate>**Ping**</DoNotTranslate>

      : especifique um único URL para monitor a disponibilidade. A New Relic verificará esta URL via HEAD ou solicitação GET. O [tempo limite](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) não configurável para este monitor é de 60 segundos.

    * <DoNotTranslate>**Simple browser**</DoNotTranslate>

      : Especifique um único URL para monitor via browser real. A cada [intervalo de frequência](#setting-frequency), a New Relic verificará esse URL por meio de um browser Google Chrome com tecnologia Selenium. O [tempo limite](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers/#waiting-elements) não configurável para este monitor é de 60 segundos.

    * <DoNotTranslate>**Scripted browser**</DoNotTranslate>

      : [crie um script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) para controlar um browser Google Chrome com tecnologia Selenium. O browser segue cada etapa do script para verificar se o comportamento complexo está funcionando conforme o esperado (por exemplo, pesquisar um site e clicar em um dos resultados da pesquisa). O [tempo limite](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers#waiting-elements) não configurável para este monitor é de 180 segundos.

    * <DoNotTranslate>**API test**</DoNotTranslate>

      : Crie um script de API para garantir que seus endpoints de API estejam funcionando corretamente. Para obter mais informações, consulte [Escrever testes de API](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests). O tempo limite não configurável para este monitor é de 180 segundos.
  </Collapser>

  <Collapser
    id="setting-name"
    title="Nome do monitor"
  >
    Define um nome para o monitor. os nomes monitor não podem conter colchetes angulares não codificados (`<` e `>`). Para incluir colchetes angulares em um nome de monitor, codifique-os como entidade de colchetes HTML (`&lt;` para `<` e `&gt;` para `>`) na interface ou API.
  </Collapser>

  <Collapser
    id="setting-location"
    title="Localização"
  >
    Selecione os locais onde deseja que seu monitor seja executado. Selecione mais locais para garantir que seu aplicativo esteja disponível para usuários em todo o mundo. Se você tiver alguma [localização privada](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations), ela também estará listada aqui. Você pode usar o [endpoint `location` da API Sintético](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-via-rest-api#list-locations) para recuperar uma lista de locais válidos para sua conta.

    Seu monitor executará uma verificação em cada local selecionado durante cada [intervalo de frequência](#setting-frequency). Por exemplo, se você selecionar três locais e definir uma frequência de 15 minutos, seu monitor executará três verificações em cada período de 15 minutos (ou 8.640 verificações por mês).
  </Collapser>

  <Collapser
    id="setting-frequency"
    title="Frequência"
  >
    Selecione a frequência de execução do monitor, em incrementos de minutos, horas ou 1 dia. Esta frequência se aplica a cada [local](#setting-location). Por exemplo, se você selecionar três locais e uma frequência de <DoNotTranslate>**15 minutes**</DoNotTranslate>, seu monitor executará três verificações, em média, a cada 5 minutos, em cada período de 15 minutos (ou 8.640 verificações por mês).
  </Collapser>

  <Collapser
    id="setting-alerts"
    title="alerta"
  >
    Especifique um endereço de e-mail para receber <InlinePopover type="alerts"/>quando um monitor falhar. Ou anexe um monitor a uma política de alertas existente para obter mais opções de notificação. Para obter mais informações, consulte [Alertas para monitoramento sintético](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics).
  </Collapser>

  <Collapser
    id="response-validation"
    title="Validação de resposta (opcional)"
  >
    Especifique o texto a ser pesquisado na página DOM. Ao usar browser simples ou [tipos de monitor](/docs/synthetics/synthetic-monitoring/getting-started/types-synthetic-monitors/#types-monitors) de ping, há um limite de 1 MB (10 ^ 6 bytes) no carregamento da página.
  </Collapser>

  <Collapser
    id="apdex-t"
    title="Apdex T (opcional)"
  >
    Especifique o limite para o <DoNotTranslate>**tolerable**</DoNotTranslate> tempo de resposta do seu monitor. O valor padrão é 7 segundos (7.000 ms). Para obter mais informações, consulte [Entender a métrica do relatório de SLA](/docs/synthetics/synthetic-monitoring/pages/synthetic-monitoring-aggregate-monitor-metrics/#understanding).
  </Collapser>
</CollapserGroup>
