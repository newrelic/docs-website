A New Relic está [encerrando o suporte para runtimes mais antigos](/eol/2026/05/eol-05-18-26-node-16-chrome-134-runtimes/) em 2026. Para simplificar a migração de monitores sintéticos para os runtimes suportados mais recentes, atualizamos automaticamente todos os monitores que não exigiam sua ação. Use a ferramenta de atualização de tempo de execução para visualizar os monitores restantes que ainda precisam da sua atenção.

## Atualize seus monitores restantes [#upgrade-options]

Para cada monitor que requer atenção, você pode escolher um dos dois caminhos.

  <TabsBar>
    <TabsBarItem id="fix-manually">
      Corrigir manualmente
    </TabsBarItem>

    <TabsBarItem id="fix-with-ai">

  <TabsPages>
    <TabsPageItem id="fix-manually">
      Para atualizar um monitor escrevendo ou editando seus scripts de monitor manualmente, siga as etapas.

      1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.

      2. Localize o monitor e clique em <DNT>**Fix manually**</DNT>.

      3. Na janela <DNT>**Fix script manually**</DNT>, atualize o código do seu script para o novo ambiente de execução.

      4. Clique em <DNT>**Close and start validation**</DNT>. O monitor é movido para a tabela <DNT>**Monitors we&apos;re validating**</DNT> enquanto o sistema valida o script atualizado.

         * Se a validação falhar, clique em **⋯** ao lado do monitor, selecione <DNT>**Fix manually**</DNT> no menu, atualize o script e clique em <DNT>**Close and start validation**</DNT> novamente.

      5. Assim que a validação for bem-sucedida e o status for exibido como <DNT>**Fixed**</DNT>, clique na aba <DNT>**Ready to update**</DNT>.

      6. Selecione a caixa de seleção ao lado do monitor validado e clique em <DNT>**Upgrade selected**</DNT>.

      Após a conclusão, uma notificação de confirmação aparece e o monitor vai para a aba <DNT>**Updated**</DNT>, onde você pode verificar se a atualização foi bem-sucedida.
    </TabsPageItem>

    <TabsPageItem id="fix-with-ai">
      <Callout title="Observação">
        * Este recurso requer Computar Avançado e não está disponível para contas <DNT>**FedRAMP**</DNT>. Entre em contato com a equipe da sua conta da New Relic para obter suporte.
        * Sempre teste os scripts gerados por IA antes de implantar os monitores em produção.
      </Callout>

      Use esta opção para que o <DNT>**New Relic AI (NRAI)**</DNT> gere um script atualizado para você. A geração de script assistida por IA é voltada principalmente para corrigir `timeout errors` ocorridos durante atualizações de tempo de execução. Para monitores com `script errors`, o script potencialmente precisa de outras alterações. Você pode fazê-los manualmente ou usar <DNT>[SRE Agent](/docs/agentic-ai/sre-agent/overview/)</DNT> para depurar e obter insights sobre possíveis atualizações relacionadas ao script.

      Para atualizar um monitor usando IA:

      1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.
      2. Localize o monitor e clique em <DNT>**Fix with AI**</DNT>.
      3. Revise o script sugerido por IA. Para visualizar as alterações específicas feitas no seu código original, clique em <DNT>**Show comparison**</DNT> para abrir uma visão de diferenças lado a lado.
      4. Clique em <DNT>**Continue to validation**</DNT>. O sistema coloca temporariamente o monitor em um estado de validação para testar o novo script e garantir que ele seja executado sem erros.
      5. Assim que o script for validado, clique na guia <DNT>**Ready to update**</DNT>.
      6. Selecione a caixa de seleção ao lado do monitor validado e clique em <DNT>**Upgrade selected**</DNT>.

      Após a conclusão, uma notificação de confirmação aparece e o monitor vai para a aba <DNT>**Updated**</DNT>, onde você pode verificar se a atualização foi bem-sucedida.
    </TabsPageItem>
  </TabsPages>
</Tabs>

Você também pode validar vários monitores em paralelo e compartilhar os links com colegas de equipe para dividir o trabalho. Assim que todos os seus scripts forem validados, você pode implantar os monitores em produção quando estiver pronto.
