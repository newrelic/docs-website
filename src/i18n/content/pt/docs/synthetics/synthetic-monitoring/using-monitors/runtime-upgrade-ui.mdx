A New Relic está [encerrando o suporte para runtimes mais antigos](/eol/2025/11/eol-chrome-140-node-22-synthetics/) em 2026. Para simplificar a migração de monitores sintéticos para os runtimes suportados mais recentes, atualizamos automaticamente todos os monitores que não exigiam sua ação. Use a ferramenta de atualização de tempo de execução para visualizar os monitores restantes que ainda precisam da sua atenção.

## Pré-requisitos [#prerequisites]

Antes de começar a atualizar os monitores, certifique-se de atender aos seguintes requisitos:

* **Permissão de atualização em massa do runtime:** necessária para aplicar atualizações em massa pela interface. [Os administradores de produto](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) têm essa permissão por padrão. Você também pode adicioná-lo a [funções personalizadas](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).
* **Licenças de IA (apenas <DNT>Fix with AI</DNT>):** requer a licença `nrai_ccu` ou `nrai_discount_usage`. As contas FedRAMP estão bloqueadas desta experiência. Entre em contato com a equipe da sua conta New Relic para habilitar os direitos.

## Atualize seus monitores restantes [#upgrade-options]

Para cada monitor que requer atenção, você pode escolher um de dois caminhos:

<CollapserGroup>
  <Collapser id="fix-manually" title="Corrigir manualmente">
    Use a opção <DNT>**Fix manually**</DNT> para escrever e editar o script do seu monitor em vez de depender da geração por IA. Para atualizar um monitor manualmente:

    1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.

    2. Localize o monitor e clique em <DNT>**Fix manually**</DNT>.

    3. Na janela <DNT>**Fix script manually**</DNT>, atualize o código do seu script para o novo ambiente de execução.

    4. Clique em <DNT>**Close and start validation**</DNT>. O monitor é movido para a tabela <DNT>**Monitors we&apos;re validating**</DNT> enquanto o sistema valida o script atualizado.

       <Callout title="Observação">
         Se a validação falhar, clique em **⋯** ao lado do monitor, selecione <DNT>**Fix manually**</DNT> no menu, atualize o script e clique em <DNT>**Close and start validation**</DNT> novamente.
       </Callout>

    5. Assim que a validação for bem-sucedida e o status for exibido como <DNT>**Fixed**</DNT>, clique na aba <DNT>**Ready to update**</DNT>.

    6. Selecione a caixa de seleção ao lado do monitor validado e clique em <DNT>**Upgrade selected**</DNT>.

    Após a conclusão, uma notificação de confirmação aparece e o monitor vai para a aba <DNT>**Updated**</DNT>, onde você pode verificar se a atualização foi bem-sucedida.
  </Collapser>

  <Collapser id="fix-with-ai" title="Corrigir com IA">
    Use a opção <DNT>**Fix with AI**</DNT> para que o <DNT>New Relic AI (NRAI)</DNT> gere um script atualizado para você. A geração de script assistida por IA é voltada principalmente para corrigir erros `timeout` ocorridos durante atualizações de runtime. Para outros tipos de erro, o próprio script provavelmente precisa de alterações que você deve fazer [manualmente](#fix-manually).

    Para atualizar um monitor usando IA:

    1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.
    2. Localize o monitor e clique em <DNT>**Fix with AI**</DNT>.
    3. Revise o script sugerido por IA. Para visualizar as alterações específicas feitas no seu código original, clique em <DNT>**Show comparison**</DNT> para abrir uma visão de diferenças lado a lado.
    4. Clique em <DNT>**Continue to validation**</DNT>. O sistema coloca temporariamente o monitor em um estado de validação para testar o novo script e garantir que ele seja executado sem erros.
    5. Assim que o script for validado, clique na guia <DNT>**Ready to update**</DNT>.
    6. Selecione a caixa de seleção ao lado do monitor validado e clique em <DNT>**Upgrade selected**</DNT>.

    Após a conclusão, uma notificação de confirmação aparece e o monitor vai para a aba <DNT>**Updated**</DNT>, onde você pode verificar se a atualização foi bem-sucedida.
  </Collapser>
</CollapserGroup>

Você pode executar várias atualizações de uma vez, o que permite trabalhar em vários monitores em paralelo sem esperar que o sistema termine de validar cada script.
