Este guia aborda a configuração e a implantação do gerenciador de jobs do New Relic Synthetics para oferecer suporte a testes de stack dupla (IPv4 e IPv6) em seu ambiente de localização privada usando o Podman.

## Pré-requisitos [#prerequisites]

Antes de começar, certifique-se de ter o seguinte:

* <DNT>**Podman**</DNT>: O Podman deve estar instalado e em execução na sua instância de host

* <DNT>**slirp4netns**</DNT>: necessário para suporte a IPv6 em contêineres rootless (versão 1.0.0 ou superior, normalmente instalado com o Podman)

* <DNT>**Permissions**</DNT>: Você deve ter acesso apropriado para executar o contêiner rootless

* <DNT>**Public images**</DNT>: o suporte a dual stack requer as seguintes versões de imagem ou posteriores, disponíveis no Docker Hub:

  * [Gerenciador de trabalhos `519` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-519/): `newrelic/synthetics-job-manager`
  * [Ping runtime `1.65.0` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.65.0/): `newrelic/synthetics-ping-runtime`
  * [Runtime da API Node `rc1.15` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.15/): `newrelic/synthetics-node-api-runtime`
  * [Runtime do navegador Node `rc1.15` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.15/): `newrelic/synthetics-node-browser-runtime`

<Steps>
  <Step>
    ## Validar sua infraestrutura [#validate-infrastructure]

    Antes de prosseguir, verifique se a sua máquina host e a rede estão configuradas para rede Dual Stack.

    Teste a conectividade IPv6 executando `ping6 ipv6.google.com` ou `ping -6 ipv6.google.com`, ou o comando equivalente para sua máquina.

    <Callout variant="important">
      Se o teste falhar, certifique-se de que sua VPC, sub-rede e grupos de segurança estejam configurados para permitir o tráfego IPv6 antes de continuar com esta instalação. Seu ambiente deve suportar tanto IPv4 quanto IPv6 para operação em stack dupla.
    </Callout>
  </Step>

  <Step>
    ## Configurar o ambiente Podman [#setup-podman]

    Configure o ambiente do Podman seguindo o [Guia de Dependências do Podman](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager/#podman-dependencies).
  </Step>

  <Step>
    ## Crie um pod do Podman [#create-pod]

    Crie um pod com o endereço IP da sua máquina host:

    ```bash
    podman pod create --network slirp4netns --name SYNTHETICS --add-host=podman.service:IP_ADDRESS
    ```

    Substitua `IP_ADDRESS` pelo endereço IP da sua máquina host.
  </Step>

  <Step>
    ## Inicie o gerenciador de tarefas do Synthetics [#launch-job-manager]

    <Callout variant="tip">
      Substitua `<YOUR_PRIVATE_LOCATION_KEY>` pela sua chave de localização privada real da interface da New Relic.
    </Callout>

    ```bash
    podman run \
      -e PRIVATE_LOCATION_KEY=<YOUR_PRIVATE_LOCATION_KEY> \
      --pod SYNTHETICS \
      -d \
      --restart unless-stopped \
      -e "CONTAINER_ENGINE=PODMAN" \
      -e "PODMAN_API_SERVICE_PORT=8000" \
      -e "PODMAN_POD_NAME=SYNTHETICS" \
      newrelic/synthetics-job-manager
    ```

    O gerenciador de jobs de Sintéticos agora será executado com suporte a Dual stack habilitado, permitindo que seus monitores testem endpoints IPv4 e IPv6.
  </Step>

  <Step>
    ## Verifique a instalação [#verify-installation]

    Para confirmar se o seu gerenciador de rotinas sintéticas está sendo executado corretamente:

    1. Visualize os logs do job manager:

       ```bash
       podman logs -f YOUR_CONTAINER_ID
       ```

    2. Na interface do New Relic, vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Synthetic monitoring &gt; Private locations**</DNT> e verifique se sua localização privada aparece como saudável.
  </Step>
</Steps>
