Este guia aborda a validação da infraestrutura, a configuração do daemon do Docker e a implantação de imagem necessárias para oferecer suporte a testes de Dual Stack (IPv4 e IPv6) em seu ambiente de localização privada usando Docker.

## Pré-requisitos [#prerequisites]

Antes de começar, certifique-se de ter o seguinte:

* <DNT>**Docker**</DNT>: o Docker deve estar instalado e em execução na sua instância de host

* <DNT>**Permissions**</DNT>: Você deve ter acesso sudo ou root para modificar as configurações do sistema

* <DNT>**Public images**</DNT>: o suporte a dual stack requer as seguintes versões de imagem ou posteriores, disponíveis no Docker Hub:

  * [Gerenciador de trabalhos `519` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-519/): `newrelic/synthetics-job-manager`
  * [Ping runtime `1.65.0` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.65.0/): `newrelic/synthetics-ping-runtime`
  * [Runtime da API Node `rc1.15` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.15/): `newrelic/synthetics-node-api-runtime`
  * [Runtime do navegador Node `rc1.15` ou posterior](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.15/): `newrelic/synthetics-node-browser-runtime`

<Steps>
  <Step>
    ## Validar sua infraestrutura [#validate-infrastructure]

    Antes de prosseguir, verifique se a sua máquina host e a rede estão configuradas para rede Dual Stack.

    Teste a conectividade IPv6 executando `ping6 ipv6.google.com` ou `ping -6 ipv6.google.com`, ou o comando equivalente para sua máquina.

    <Callout variant="important">
      Se o ping falhar, certifique-se de que sua VPC, sub-rede e grupos de segurança estejam configurados para permitir o tráfego IPv6 antes de continuar com esta instalação. Seu ambiente deve suportar tanto IPv4 quanto IPv6 para operação Dual Stack.
    </Callout>
  </Step>

  <Step>
    ## Verifique se o Docker está pronto para dual stack [#configure-docker]

    Certifique-se de que sua instalação Docker esteja configurada para lidar com tráfego IPv4 e IPv6. Para verificar se o Docker tem o IPv6 habilitado, execute:

    ```bash
    docker info | grep -i ipv6
    ```

    A saída deve confirmar que o IPv6 está ativado. Se não estiver, consulte a [documentação do Docker](https://docs.docker.com/config/daemon/ipv6/) para habilitar o suporte a dual stack antes de prosseguir.
  </Step>

  <Step>
    ## Inicie o gerenciador de tarefas do Synthetics [#launch-job-manager]

    As imagens de tempo de execução habilitadas para Dual stack estão disponíveis publicamente no Docker Hub e serão extraídas automaticamente pelo job manager. Execute o contêiner do gerenciador de jobs do Sintéticos com sua chave de localização privada:

    <Callout variant="tip">
      Substitua `<YOUR_PRIVATE_LOCATION_KEY>` pela sua chave de localização privada real da interface da New Relic.
    </Callout>

    ```bash
    docker run \
      -e PRIVATE_LOCATION_KEY=<YOUR_PRIVATE_LOCATION_KEY> \
      -d \
      --restart unless-stopped \
      -v /var/run/docker.sock:/var/run/docker.sock:rw \
      newrelic/synthetics-job-manager
    ```

    O gerenciador de jobs de Sintéticos agora será executado com suporte a Dual stack habilitado, permitindo que seus monitores testem endpoints IPv4 e IPv6.
  </Step>

  <Step>
    ## Verifique a instalação [#verify-installation]

    Para confirmar se o seu gerenciador de rotinas sintéticas está sendo executado corretamente:

    1. Verifique o status do contêiner:

       ```bash
       docker ps
       ```

    2. Visualize os logs do job manager:

       ```bash
       docker logs <container_id>
       ```

    3. Na interface do New Relic, vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Synthetic monitoring &gt; Private locations**</DNT> e verifique se sua localização privada aparece como saudável.
  </Step>
</Steps>
