Além do ambiente de script baseado em WebDriverJS do monitoramento sintético padrão, o New Relic oferece suporte a muitos módulos Node.js populares para aprimorar seu [browser com script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) e testes de API.

## Importar módulos opcionais [#import-procedure]

Para importar um módulo suportado, siga o procedimento de importação padrão para ele. Por exemplo:

* Para importar `crypto-js`, chame:

  ```js
  var crypto = require('crypto-js');
  ```

* Para importar `chai`, chame:

  ```js
  var assert = require('chai').assert;
  ```

## Módulos de terceiros suportados

Diferentes versões do monitor com script Sintético suportam diferentes módulos de terceiros. Para obter informações sobre módulos suportados por diferentes versões de monitor, consulte a [tabela de comparação de versões de monitor](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment).

    * [`tls`](https://nodejs.org/api/tls.html)
    * [`url`](http://nodejs.org/api/url.html)
    * [`util`](http://nodejs.org/api/util.html)
    * [`zlib`](https://nodejs.org/api/zlib.html)

## Módulos não suportados

Você pode usar módulos Node personalizados em um [Synthetics Job Manager](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager/). Para obter instruções sobre como configurar módulos personalizados para um Synthetics Job Manager, consulte [Configuração do Job Manager](/docs/synthetics/synthetic-monitoring/private-locations/job-manager-configuration#custom-modules).

Módulos não suportados não têm garantia de funcionamento e estão fora do [escopo do Suporte da New Relic](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings/#scope-support).
