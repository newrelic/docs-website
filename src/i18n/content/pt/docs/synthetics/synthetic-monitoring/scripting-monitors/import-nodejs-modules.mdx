Além do ambiente de script baseado em WebDriverJS do monitoramento sintético padrão, o New Relic oferece suporte a muitos módulos Node.js populares para aprimorar seu [browser com script](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) e testes de API.

## Importar módulos opcionais [#import-procedure]

Para importar um módulo suportado, siga o procedimento de importação padrão para ele. Por exemplo:

* Para importar <DNT>**crypto-js**</DNT>, chame:

  ```
  var crypto = require('crypto-js');
  ```

* Para importar <DNT>**chai**</DNT>, chame:

  ```
  var assert = require('chai').assert;
  ```

## Módulos de terceiros suportados

Diferentes versões do monitor com script Sintético suportam diferentes módulos de terceiros. Para obter informações sobre módulos suportados por diferentes versões de monitor, consulte a [tabela de comparação de versões de monitor](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment).

## Módulos principais do Node.js suportados [#supported-core-modules]

  <Collapser

    * [afirmar](http://nodejs.org/api/assert.html)
    * [criptografia](http://nodejs.org/api/crypto.html)
    * [DNS](https://nodejs.org/api/dns.html)
    * [fs](https://nodejs.org/api/fs.html)
    * [net](https://nodejs.org/api/net.html)
    * [querystring](http://nodejs.org/api/querystring.html)
    * [temporizadores](http://nodejs.org/api/timers.html)
    * [tls](https://nodejs.org/api/tls.html)
    * [url](http://nodejs.org/api/url.html)
    * [útil](http://nodejs.org/api/util.html)
    * [zlib](https://nodejs.org/api/zlib.html)
  </Collapser>
</CollapserGroup>

## Módulos não suportados

Você pode usar qualquer módulo de nó em um [minion privado em contêiner](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms). Para obter instruções sobre como configurar módulos personalizados para minion privado em contêineres, consulte [chamadas por minuto configuração](/docs/synthetics/new-relic-synthetics/private-locations/containerized-private-minion-cpm-configuration#custom-modules).

Módulos não suportados não têm garantia de funcionamento e não são cobertos pelo suporte.
