Se você for um [usuário completo da plataforma](/docs/accounts/accounts-billing/new-relic-one-pricing-users/users-roles#user-type), teoricamente terá acesso total a todos os recursos do New Relic, mas se você pode usar todos esses recursos depende da função atribuída a você em sua organização New Relic.

Os detalhes relacionados às permissões dependem de o usuário do New Relic estar em nosso modelo de usuário mais recente ou em nosso [modelo de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) original .

## Usuários em nosso modelo de usuário mais recente [#new-pricing]

Para obter detalhes sobre como funcionam as permissões do New Relic, consulte [Conceitos de gerenciamento de usuários](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#groups). Para ver os recursos granulares relacionados ao Sintético, consulte [Capacidades](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).

## Usuários no modelo usuário original [#original-pricing]

Para usuários do nosso modelo usuário original, aqui estão alguns detalhes:

<CollapserGroup>
  <Collapser
    id="default-access"
    title="Modelo de usuário original – acesso padrão"
  >
    * [Os administradores](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#roles) têm acesso total a todos os recursos de monitoramento sintético de uma conta.

    * [o usuário](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) tem acesso a todos os recursos de monitoramento sintético, exceto [credenciais seguras](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers) e a capacidade de excluir monitor, monitor período de inatividade e localização privada.

    * [usuário restrito](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) em contas com preços originais baseados em produtos tem acesso somente visualização a todos os recursos de monitoramento sintético, exceto:

      * [Credenciais seguras](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
      * Localização privada (embora possam consultar uma quantidade limitada de dados via NRQL)

      Isso significa que o usuário com esta função não tem acesso a todos os recursos de criação, edição e exclusão do recurso de monitoramento sintético (bem como capacidade de visualização para credenciais seguras) até que receba uma função com os recursos apropriados.
  </Collapser>

  <Collapser
    id="custom-access"
    title="Modelo de usuário original – funções personalizadas"
  >
    Para começar, siga [os procedimentos padrão para criar uma função personalizada](/docs/accounts/accounts/roles-permissions/add-roles-permissions#create/docs/accounts/accounts/roles-permissions/add-roles-permissions#create). Você pode adicionar recursos <DNT>**View**</DNT> e <DNT>**Edit**</DNT> a qualquer função para o seguinte:

    * [Monitores](/docs/synthetics/new-relic-synthetics/using-monitors)
    * [Monitor script](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-synthetics-api-tests)
    * [localização privada](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations)
    * [monitor período de inatividade](/docs/synthetics/new-relic-synthetics/using-monitors/monitor-downtimes-disable-monitoring-during-scheduled-maintenance-times)
    * [Credenciais seguras](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
  </Collapser>
</CollapserGroup>
