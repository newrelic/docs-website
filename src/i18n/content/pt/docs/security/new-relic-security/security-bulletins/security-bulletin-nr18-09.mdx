## Resumo

Uma atualização de segurança para o agente Java corrige um problema em que o agente pode não ofuscar adequadamente todas as informações da consulta SQL quando `record_sql` está definido como `off`.

<DNT>**Release date:**</DNT> 2 de maio de 2018

<DNT>**Vulnerability identifier:**</DNT> NR18-09

<DNT>**Priority:**</DNT> Baixo

## Informações sobre vulnerabilidades [#vuln-info]

O agente Java New Relic executa planos explicativos para [rastreamento lento da transação](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) e consulta SQL lenta. Quando um cliente definiu `record_sql` como `off`, está executando o PostgreSQL e tem um plano de explicação em execução, o agente pode não ofuscar adequadamente todas as informações na consulta.

## Soluções alternativas

Desative os planos `explain` com `transaction_tracer` na [configuração do agente Java](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#h2-Transaction-tracer).
