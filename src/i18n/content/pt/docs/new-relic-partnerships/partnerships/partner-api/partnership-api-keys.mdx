As chaves [da API de parceria](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) são usadas para autenticar chamadas com contas de parceria no New Relic. Uma distinção importante ao trabalhar com chave de API é conta versus proprietário da parceria. O proprietário da parceria é uma forma especial de conta usada para gerenciar dados sobre a conta.

## Requisitos [#requirements]

<DoNotTranslate>
  **The Partnership API is not available to your organization unless you're specifically instructed by your New Relic representative that you should use it.**
</DoNotTranslate>

Para conhecer os requisitos, leia [Introdução à API de Parceria](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/).

## chave de licença

Esta é a chave retornada no XML ao criar uma conta. É usado pelo agente para identificar a conta.

## Chave de API do parceiro [#partner-key]

Ao usar a API de parceria, você deve usar a chave de API REST da conta do proprietário da parceria. Esta é a chave usada na chamada de API onde você vê `x-api-key: <api key>`.

## Localize sua chave de API [#locating-keys]

Para visualizar ou regenerar suas chaves, log no site New Relic usando suas credenciais de proprietário da parceria e vá para <DoNotTranslate>**Partnerships > Edit settings**</DoNotTranslate>. As chaves estão listadas na parte inferior da página.

<Callout variant="caution">
  As chaves regeneradas não podem ser revertidas.
</Callout>

## Práticas de segurança recomendadas [#security]

<DoNotTranslate>**Recommendation:**</DoNotTranslate> As chaves usadas para acessar o site New Relic devem ser regeneradas regularmente como práticas de segurança recomendadas. A alteração da regeneração de uma chave ocorre imediatamente.

Se você codificar as chaves em seu código e quiser enviar uma nova versão de integração com a nova chave para produção, certifique-se de coordenar entre a regeneração de chave e a atualização do site. Isso ajudará a minimizar o período de inatividade. Nessa situação, considere buscar essas chaves dinamicamente usando a API Show Account.
