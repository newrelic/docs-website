Nossa API de parceria permite que parceiros New Relic e contas New Relic configuradas como [parcerias de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) gerenciem contas, usuários e configurações relacionadas à assinatura.

## Requisitos [#requirements]

<DNT>**There are several major limitations to using the Partnership API.**</DNT> Sua organização New Relic não deve usar esta API, a menos que você tenha sido especificamente instruído a usá-la por seu representante da New Relic. Leia toda a seção de requisitos para entender as limitações.

A API de parceria pode ser usada por dois tipos de contas New Relic : [parceiros](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) (provedor de serviço gerenciado e revendedores New Relic ) e [clientes parcerias](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) (organizações maiores que tiveram acesso à nossa estrutura de contas de parceria).

A chamada de API de parceria requer autenticação com a chave de API REST da conta do proprietário da parceria e com seu ID de parceiro.

Contas que são parceiros genuínos New Relic (provedor de serviço gerenciado ou revendedores) não têm restrições ao uso da API de Parceria. A organização New Relic constituída como [parceria de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships) tem as seguintes restrições.

### Restrições de parceria com clientes [#customer-restrictions]

Se a sua organização New Relic estiver configurada como uma [parceria de clientes](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partnership-accounts-users-subscriptions/#partnerships), existem algumas restrições em vigor se a sua organização estiver no nosso [modelo de preços](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) mais recente ou no nosso [modelo de usuário](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) mais recente. Detalhes:

* <DNT>
    **User model.**
  </DNT>

  Se a sua organização New Relic tiver usuário em [nosso modelo de usuário mais recente](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), você não poderá mais usar a chamada de API de Parceria que gerencia contas ou usuário. Aqui estão os detalhes sobre o que você usaria:

  * Em vez de usar o objeto API [usuário de](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-user-object) parceria, você [gerenciaria usuário via NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users).
  * Em vez de usar o [objeto de conta](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-account-object) da API de parceria ou [o objeto de conta infantil](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-sub-account-object), você [gerenciaria contas por meio do NerdGraph](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph).

* <DNT>
    **Pricing model.**
  </DNT>

  Se você estiver em nosso [modelo de preços baseado no uso](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans) mais recente, não poderá usar a chamada de API que rege nosso [modelo de preços original](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#pricing-plans). Os impactos da API são:

  * Você não pode usar o [objeto de assinatura](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-subscription-object/)
  * Você não pode usar a [API de provisionamento NerdGraph](/docs/apis/nerdgraph/tutorials/provisions-your-subscriptions-nerdgraph/).

Para determinar seu modelo de precificação ou modelo de usuário, consulte [Visão geral de precificação e alterações no modelo de usuário](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model).

## Coisas que você pode fazer [#functions]

Esta é uma visão geral do que você pode fazer com esta API:

<DNT>
  **Account:**
</DNT>

Algumas organizações que atendam aos [requisitos](#requirements) podem utilizar o [objeto conta](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-account-object/) para criar e atualizar contas. Você pode fazer o seguinte com ele:

* Crie um novo
* Mostrar
* Atualizar
* Cancelar
* Mostrar uso
* Definir administrador principal
* Definir assinatura

Há também um [objeto de conta infantil](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-sub-account-object) para criar contas infantis.

<DNT>
  **Users:**
</DNT>

Algumas organizações que [atendam aos requisitos](#customer-restrictions) poderão utilizar o [objeto usuário](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-user-object) para:

* Adicionar acesso de usuário à conta
* Remover o acesso do usuário da conta

<DNT>
  **Subscription:**
</DNT>

organizações que [atendam aos requisitos](#customer-restrictions) e estejam em nosso modelo de precificação de produto original podem usar o [objeto assinatura](/docs/new-relic-partnerships/partnerships/partner-api/partnership-api-subscription-object) para configurar diversas características relacionadas à assinatura.

### Requisitos de senha [#password]

As senhas passadas para criação de conta têm estes requisitos:

* 8 a 50 caracteres de comprimento
* Contém apenas letras, números ou caracteres especiais
* Não pode conter espaços
* Deve conter pelo menos 1 letra
* Deve conter pelo menos 1 número ou caractere especial
