Os parceiros que agregam New Relic em seu console podem “skin” New Relic com sua própria aparência, incluindo layouts, menus e subdomínios personalizados.

## Exemplo de co-branding

## Cabeçalhos e rodapés [#headers-footers]

Para implementar cabeçalhos e rodapés:

1. Crie uma página de cabeçalho e/ou rodapé, consistindo em conteúdo incluído na tag &lt;div>. Certifique-se de que as páginas não importem nenhum CSS ou contenham CSS embutido.

2. Pegue qualquer CSS necessário para o cabeçalho e rodapé e disponibilize-o em um arquivo CSS separado. Por exemplo, acrescente uma string personalizada como `yourcompany_` aos vários nomes de estilo, para evitar substituir as definições de estilo padrão do New Relic.

3. Publique os três arquivos em locais de acesso público hospedados pelo parceiro. Use https para que seus clientes não vejam nenhum aviso de conteúdo misto. Por exemplo:

   ```
   https://yourcompany.com/newrelic_header.htm
   https://yourcompany.com/newrelic_footer.htm
   https://yourcompany.com/newrelic_styles.css
   ```

4. Insira os URLs dos locais do cabeçalho, rodapé e página CSS nos campos apropriados no console <DNT>**Partnership**</DNT> do New Relic.

<Callout variant="tip">
  Para visualizar os cabeçalhos, log em uma das contas da parceria.
</Callout>

## Layouts e estilos [#styles]

Os layouts e estilos de cabeçalho e rodapé são carregados e armazenados em cache nos servidores coletores da New Relic para inserção em cada saída de página. As atualizações feitas no layout do site do parceiro são atualizadas automaticamente a cada hora.

Links personalizados para o site do parceiro podem ser adicionados ao layout. Se o logon único estiver habilitado, o parceiro poderá injetar informações específicas do cliente na sessão do usuário New Relic para uso pelo layout. Para obter mais informações, consulte [Logon único e controle de acesso](/docs/accounts-partnerships/partnerships/partner-integration-guide/single-sign-access-control).

## Subdomínio

Opcionalmente, os parceiros podem exibir um subdomínio personalizado; por exemplo, <DNT>**yourcompany.newrelic.com**</DNT>, para clientes que acessam contas New Relic. Para organizar isso, entre em contato com a equipe de Business Enablement da New Relic.
