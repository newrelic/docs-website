Depois de enviar dados do OpenTelemetry para o New Relic, você pode usar uma variedade de páginas de interface para analisá-los. Dê uma olhada nessas opções:

* [Encontre entidade e dados:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-find-entities) Encontre sua entidade e aprofunde-se nos dados mais relevantes
* [Resumo:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-summary-page) Veja a saúde geral do serviço e comece a resolução de problemas
* [distributed tracing:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-distributed-tracing-page) Acompanhe solicitações através de seus sistemas distribuídos que envolvem este serviço
* [transação:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-transactions-page) ajuste o desempenho das solicitações para o endpoint de um serviço e diagnostique problemas de latência
* [Bancos de dados:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-database-page) monitor suas chamadas de banco de dados
* [Serviços externos:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-external-services-page) veja o desempenho das solicitações entre seu serviço e outros, até o endpoint envolvido
* [JVMs:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-jvms-page) monitor o desempenho da sua JVM
* [log:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-logs-page) Diagnosticar problemas com log do aplicativo
* [Página de erros:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-errors-page) veja quais erros o usuário do seu serviço está enfrentando
* [Errors Inbox:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-errors-inbox-page) triagem e resolução de erros
* [explorador métrico:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-metrics-explorer-page) veja métricas específicas
* [Crie uma consulta personalizada:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-data-explorer-query-builder) experimente estas ferramentas "faça você mesmo" para detalhar seus dados

## Convenções para dados OpenTelemetry [#conventions]

O projeto OpenTelemetry especifica convenções semânticas para [métrica](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/metrics/semantic_conventions), [traces](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/trace/semantic_conventions), [log](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/logs/semantic_conventions) e [evento](https://github.com/open-telemetry/opentelemetry-specification/blob/main/specification/logs/semantic_conventions/events.md) para que a telemetria possa ser interpretada consistentemente por ferramentas de observabilidade. A New Relic depende dessas convenções para apresentar dados de telemetria na interface.

O agente de instrumentação automática OpenTelemetry e os SDKs geralmente seguem as convenções semânticas, mas como o projeto está em rápido desenvolvimento, é possível que você precise ajustar sua instrumentação para corresponder às convenções.

Se os dados não aparecerem como você esperaria na interface, consulte a documentação da página da interface relevante, onde você poderá encontrar uma referência para as convenções semânticas usadas nessa página.
