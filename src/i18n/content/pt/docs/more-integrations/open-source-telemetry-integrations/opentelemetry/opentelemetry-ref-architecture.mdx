Se você precisar de orientação para implementar o OpenTelemetry com New Relic, esta arquitetura de referência deverá ajudar seus desenvolvedores de software, DevOps, arquitetos corporativos e gerentes de tecnologia a colaborar no projeto. É um modelo que mostra uma visão panorâmica das maneiras comuns de implementar OpenTelemetry com New Relic.

O diagrama mostra uma implementação que ingere telemetria do OpenTelemetry, Prometheus, Jaeger e um agente <InlinePopover type="apm"/>da New Relic. Os três serviços (A, B e C) são instrumentos para envio de métrica, trace e log.

Para dados coletados pelo OpenTelemetry, você pode enviá-los diretamente para o New Relic por meio do endpoint OTLP ou pode enviá-los primeiro para um coletor opcional do OpenTelemetry. Se você usar o coletor, ele agregará e aprimorará os pipelines de dados antes que a telemetria seja enviada ao back-end (Kafka para análise de dados e New Relic para observabilidade ponta a ponta).
