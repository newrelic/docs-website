A New Relic publicou vários exportadores que são usados para enviar dados OpenTelemetry através do protocolo proprietário da New Relic. Esses exportadores foram descontinuados em favor do envio de dados para a New Relic usando o protocolo OpenTelemetry.

<Callout variant="important">
  Os exportadores estão listados aqui para referência. Para migrar desses exportadores, siga nosso [guia de início rápido](/docs/integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-quick-start).
</Callout>

## Exportadores de idiomas New Relic [#newrelic-language-exporters]

Conforme mencionado acima, nossos exportadores específicos de idiomas estão obsoletos. Esses exportadores permitiram que o aplicativo exportasse dados do OpenTelemetry diretamente para o New Relic. O [endpointOTLP da New Relic](#native-otlp) torna essa estratégia obsoleta, pois permite que o aplicativo exporte dados para endpoints proprietários da New Relic.

Os exportadores e seu status de manutenção são os seguintes:

* [Go](https://github.com/newrelic/opentelemetry-exporter-go): Arquivado
* [Java](https://github.com/newrelic/opentelemetry-exporter-java): Arquivado
* [Python](https://github.com/newrelic/opentelemetry-exporter-python): arquivado
* [.NET](https://github.com/newrelic/newrelic-telemetry-sdk-dotnet/tree/main/src/NewRelic.OpenTelemetry): arquivado
* [Coletor](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/v0.50.0/exporter/newrelicexporter): Removido em 10 de maio de 2022.

## Configuração de endpoint para exportadores New Relic [#h2-change-endpoints]

Você pode alterar o endpoint do New Relic para onde envia seus dados.

### Ponto final dos EUA [#default-endpoints]

Por padrão, os exportadores New Relic OpenTelemetry enviam dados para estes data centers nos EUA:

* Períodos: [https://trace-api.newrelic.com/trace/v1](https://trace-api.newrelic.com/trace/v1)
* métrica: [https://metric-api.newrelic.com/metric/v1](https://metric-api.newrelic.com/metric/v1)
* Registros: [https://log-api.newrelic.com/log/v1](https://log-api.newrelic.com/log/v1)

Talvez seja necessário substituir esses endpoints padrão para enviar dados para a região da UE ou para usar o Rastreamento Infinito.

### Ponto final da região da UE [#eu-region]

Para enviar dados de telemetria para o endpoint da New Relic na região da UE, use o seguinte:

<Callout variant="tip">
  Esses URLs não se aplicam ao rastreamento infinito.
</Callout>

* Períodos: [https://trace-api.eu.newrelic.com/trace/v1](https://trace-api.eu.newrelic.com/trace/v1)
* métrica: [https://metric-api.eu.newrelic.com/metric/v1](https://metric-api.eu.newrelic.com/metric/v1)
* Registros: [https://log-api.eu.newrelic.com/log/v1](https://log-api.eu.newrelic.com/log/v1)
