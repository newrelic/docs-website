Para maximizar a quantidade de dados que você pode enviar por solicitação, recomendamos ativar a compactação em todos os exportadores OTLP. New Relic oferece suporte à compactação `gzip` e `zstd` para cargas úteis OTLP transportadas via gRPC ou HTTP com um tamanho de carga máximo permitido de 1 MB (10 ^ 6 bytes).

## Configuração do coletor OTel

A compactação padrão para o exportador OTLP no coletor OTel é atualmente `gzip`, enquanto a compactação `zstd` geralmente se mostra mais eficiente. Consulte [a comparação de compactação](https://github.com/open-telemetry/opentelemetry-collector/blob/main/config/configgrpc/README.md#compression-comparison) para obter mais detalhes sobre informações de benchmark.

Para ativar `zstd` no coletor, adapte a configuração do exportador do coletor:

```yml
exporters:
  otlphttp:
    ...
    compression: zstd
```

Se você quiser que suportemos outros formatos de compactação, informe-nos no [canal CNCF Slack](https://cloud-native.slack.com/archives/C024DRQ63UP). Você pode [se inscrever na conta Slack da CNCF aqui](https://slack.cncf.io/).
