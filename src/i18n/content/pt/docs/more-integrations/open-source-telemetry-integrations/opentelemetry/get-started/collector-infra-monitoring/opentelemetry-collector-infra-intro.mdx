O [coletor OpenTelemetry](https://opentelemetry.io/docs/collector/) é uma ferramenta independente de fornecedor para receber, processar e exportar dados de telemetria. Embora os requisitos e a configuração do coletor variem, ele vem com uma variedade de receptores e processadores que o tornam popular para monitoramento de infraestrutura. Também é comum usar o coletor para processamento de dados, mas esta documentação se concentra em casos de uso de monitoramento de infraestrutura. Consulte [ColetorOpenTelemetry para processamento de dados](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) para obter mais informações.

## Correlacionando infraestrutura com OpenTelemetry APM [#infrastructure-correlation]

Os exemplos a seguir demonstram o uso do coletor para correlacionar [dados do APM OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro) com infraestrutura. O padrão geral é configurar o coletor com um processador que detecte e enriqueça a telemetria APM com contexto de ambiente adicional na forma de atributo de recursos, antes de exportar os dados para o New Relic via OTLP. New Relic pode detectar esses dados de correlação e construir [relacionamentos entre APM e a entidade de infraestrutura](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/best-practices/opentelemetry-best-practices-resources).
