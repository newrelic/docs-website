Se você estiver executando nossa integração Kubernetes com um proxy, será necessário configurar cada componente (o [agente de infraestrutura](/docs/infrastructure/install-infrastructure-agent), o logging e o Sintético minion) para funcionar corretamente com seu proxy.

## Instalar o agente de infraestrutura com um proxy [#infrastructure-agent]

O proxy pode ser configurado definindo a opção de configuração `global.proxy` e configurando toda a integração do Kubernetes ao mesmo tempo. A configuração global é [compartilhada com a dependência](https://github.com/newrelic/helm-charts/tree/master/charts/nri-bundle#configure-dependencies) modificando o arquivo `values.yaml` do `nri-bundle`:

```yaml
global.proxy: https://user:password@hostname:port
```

Isso irá configurar todo o agente de infraestrutura, a integração Kubernetes , o métrica Adapter e o agente Prometheus ao mesmo tempo.

<Callout variant="important">
  Ainda não oferecemos suporte ao Proxy no operador de infraestrutura do Kubernetes.
</Callout>
