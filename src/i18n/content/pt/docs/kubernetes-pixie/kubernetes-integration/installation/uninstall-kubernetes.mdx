A integração do New Relic Kubernetes pode ser facilmente desinstalada da sua plataforma.

Certifique-se de estar operando no contexto apropriado na máquina em que está executando o Helm e `kubectl`. Você pode verificar os contextos disponíveis com este comando:

```
kubectl config get-contexts
```

Mude para o contexto desejado usando:

```
kubectl config use-context CONTEXT_NAME
```

## Desinstale o Kubernetes usando Helm [#uninstall-helm]

Se você usou o Helm para instalar a integração do Kubernetes, execute:

```
helm uninstall newrelic-bundle -n newrelic
```

<Callout variant="important">
  A instalação guiada usa `newrelic-bundle` como nome de versão padrão e `newrelic` como namespace padrão. Se você instalar nosso pacote com um nome de lançamento ou namespace diferente, terá que substituir os argumentos no comando acima: `helm uninstall <RELEASE-NAME> -n <RELEASE-NAMESPACE>`. Você pode usar `helm list --all-namespaces` e procurar o pacote `nri-bundle` para localizar o nome da versão e o namespace da versão.
</Callout>

## Desinstale o Kubernetes usando o manifesto [#uninstall-manifest]

Se você instalou a integração do Kubernetes usando um manifesto, use o mesmo manifesto para desinstalá-la:

```
kubectl delete -f newrelic.yaml
```
