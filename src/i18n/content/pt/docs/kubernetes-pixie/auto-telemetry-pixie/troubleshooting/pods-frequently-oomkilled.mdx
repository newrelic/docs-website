## Problema

Você concluiu o [procedimento de instalação](/docs/kubernetes-pixie/auto-telemetry-pixie/install-auto-telemetry-pixie) da integração Pixie da New Relic, mas um ou mais pods `vizier-pem` são frequentemente eliminados pelo OOM.

## Solução

Parte do benefício de implantar no Kubernetes é que Kubernetes irá OOMKill pod que está usando muitos recursos para proteger o outro pod no cluster.

Quando o pod `vizier-pem` do Pixie excede o limite de memória, eles são OOMKilled. Kubernetes faz isso para proteger o outro aplicativo, para que não cause problemas em outro pod. Para `vizier-pem` pods que sofrem OOMKilled regularmente, tente [implantar Pixie com mais memória](/docs/kubernetes-pixie/auto-telemetry-pixie/advanced-configuration/manage-pixie-memory#set-memory-limit).
