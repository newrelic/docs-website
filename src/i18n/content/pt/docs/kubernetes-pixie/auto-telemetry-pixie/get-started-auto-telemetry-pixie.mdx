A integração do New Relic com o [Pixie](https://px.dev/) oferece o melhor dos dois mundos: a observabilidade avançada do Kubernetes baseada em eBPF do Pixie, combinada com a correlação de eventos de alerta, alertas inteligentes e retenção de longo prazo do New Relic.

<img title="service graph in live debugger" alt="service-graph" src="/images/pixie_screenshot-full_service-graph.webp" />

## O que é Pixie? [#pixie]

Pixie é uma ferramenta de observabilidade de código aberto para aplicativo Kubernetes que foi projetada para aliviar os pontos problemáticos que os desenvolvedores enfrentam com a observabilidade tradicional.

[Pixie usa eBPF](https://docs.px.dev/about-pixie/pixie-ebpf) para coletar automaticamente dados de telemetria refinados — métricas de nível de serviço, solicitações sem amostragem e muito mais. Com um comando de instalação, você obtém insights mais profundos sobre seu cluster do Kubernetes e carga de trabalho. Nenhum agente de idioma é necessário!

## Vantagens da integração New Relic Pixie [#nr-integration]

A New Relic oferece a melhor integração comercial do Pixie. Com a integração New Relic você obtém todos os recursos padrão do Pixie mais:

* Armazenamento de longo prazo de dados de telemetria Pixie
* Alertas usando dados de telemetria Pixie
* Capacidade de ver dados de telemetria do Pixie em contexto com log e outros dados
* Suporte comercial

Pronto para começar? Você poderá configurar o Pixie para se adequar ao seu ambiente depois de [criar uma conta New Relic](https://newrelic.com/signup) (é grátis, para sempre!) e [instalar nossa integração Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/install-auto-telemetry-pixie/).

<Callout variant="important">
  A telemetria automática com Pixie aproveita [o Community Cloud com Pixie](http://pixielabs.ai/), uma plataforma separada da New Relic. O uso do Community Cloud com Pixie está sujeito a [termos de serviço](https://pixielabs.ai/terms) separados.
</Callout>

## Comece rapidamente a observar e depurar cluster do Kubernetes [#pixie-overview]

Depois de instalar a integração New Relic Pixie, o Pixie começará a coletar automaticamente uma [variedade de métricas e rastreamentos](https://docs.px.dev/about-pixie/data-sources/).

Você obterá [visibilidade dos serviços HTTP](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/explore-pixie-data#service-health) usando sinais clássicos, mapas de serviços, transação HTTP e banco de dados de transação. Você pode operar, depurar e dimensionar seu cluster do Kubernetes com base nas informações que você aprendeu sobre como seu cluster e serviços estão em execução.

<img title="The OpenTelemetry <DNT>**Service map**</DNT> view shows helps visualize your application's dependencies." alt="The OpenTelemetry <DNT>**Service map**</DNT> view shows helps visualize your application's dependencies." src="/images/pixie_screenshot-full_service-otel-map.webp" />

<figcaption>
  Visualize a dependência do seu aplicativo usando o <DNT>**Service map**</DNT> criado a partir do trace HTTP do Pixie.
</figcaption>

Usando o cluster do Kubernetes Explorer, você pode ver as principais métricas e eventos em todos os níveis, começando com o cluster e mergulhando em namespace, implantação e pod. Você pode identificar rapidamente um comportamento anômalo e onde ele está acontecendo.

<img title="The pod details page in the Kubernetes cluster explorer shows application metrics collected by Pixie." alt="The pod details page in the Kubernetes cluster explorer shows application metrics collected by Pixie." src="/images/pixie_screenshot-full_kce-pod-details.webp" />

<figcaption>
  A página de detalhes pod no cluster do Kubernetes Explorer mostra a métrica do aplicativo coletada pelo Pixie.

[Inspecione solicitações e respostas completas](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/pixie-entities) feitas pelo seu aplicativo. Disponível para protocolos selecionados, incluindo DNS, Cassandra, MySQL, PostgreSQL, Redis, Kafka e AMQP.

<img title="The Pixie DNS entity shows full body DNS request and response pairs traced by Pixie" alt="The Pixie DNS  entity shows full body DNS request and response pairs traced by Pixie" src="/images/pixie_screenshot-full_dns_entity.webp" />

<figcaption>
  A entidade Pixie DNS mostra o rastreamento de pares de resposta e solicitação DNS de corpo inteiro por Pixie.
</figcaption>

Identifique rapidamente pontos de acesso com [gráficos em chamas de CPU do aplicativo](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/explore-pixie-data#cpu-profiles). A depuração é muito mais fácil quando você pode ver rapidamente o que seu aplicativo está fazendo. A exibição do gráfico em degradê Pixie não requer instrumentação, reimplantação ou recompilação. Funciona para linguagens compiladas como Go, C+, Rust, para citar algumas. E rapidamente, o gráfico em degradê informa em quais funções seu aplicativo está gastando tempo e onde há pontos de acesso.

<img title="Pixie flame graph" alt="Pixie flame graph" src="/images/pixie_screenshot-crop_flamegraph.webp" />

## Nosso compromisso com o código aberto [#pixie-oss]

A New Relic está comprometida com padrões abertos, instrumentação de código aberto e com as comunidades abertas que os apoiam.

Como um passo para tornar a observabilidade aberta a todos, New Relic contribuiu com Pixie para a [nativo na nuvem Computing Foundation](https://www.cncf.io/) como um [projeto Sandbox](https://www.cncf.io/sandbox-projects/) em junho de 2021. [Conheça](https://opensource.newrelic.com/projects/pixie) os funcionários New Relic que contribuem com o projeto Pixie.
