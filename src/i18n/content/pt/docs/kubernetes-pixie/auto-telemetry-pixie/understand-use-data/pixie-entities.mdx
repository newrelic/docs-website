Use a [entidade](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/) Pixie para ver atualizações clássicas e uma amostra de solicitações individuais de corpo inteiro enviadas para um serviço específico usando um protocolo específico. A entidade Pixie é alimentada por dados que foram rastreados automaticamente pelo Pixie. Selecione um protocolo abaixo para saber mais:

* [DNS](#dns)
* [MySQL](#mysql)
* [PostgreSQL](#postgres)
* [Redis](#redis)
* [Kafka](#kafka)
* [AMQP](#amqp)

Você pode encontrar a entidade Pixie na seção <DNT>**Other entities**</DNT> da página <DNT>**All entities**</DNT> :

<Callout variant="important">
  Se estiver faltando alguma dessas entidades do Pixie, verifique sua configuração [Personalizar Pixie Ingest](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/overview/#pixie-configure-storage) . Você pode ter desabilitado a ingestão para um determinado protocolo.
</Callout>

## DNS [#dns]

Selecione um serviço na lista de entidades <DNT>**DNS - Pixie**</DNT> para ver gráficos de série temporal de tempo de resposta, taxas de erros e taxas de transferência para todas as solicitações de DNS enviadas para esse serviço. Inspecione a tabela de amostras de solicitações de DNS para ver solicitações e respostas de DNS de corpo inteiro recebidas por este serviço.

As solicitações de DNS rastreadas automaticamente pelo Pixie podem ser usadas para:

* Determine rapidamente quais serviços estão fazendo solicitações de DNS.
* Obtenha informações de DNS de alto nível sobre tempo de resposta e taxas de transferência.
* Observe desequilíbrios nas taxas de transferência entre servidores DNS.

## MySQL [#mysql]

Selecione um serviço na lista de entidades <DNT>**MySQL - Pixie**</DNT> para ver gráficos de séries temporais de tempo de resposta, taxas de erros e taxas de transferência para todas as solicitações MySQL enviadas para esse serviço. Inspecione a tabela "Solicitações MySQL amostradas" para ver solicitações e respostas MySQL completas recebidas por este serviço.

* Veja o tempo de resposta por comando MySQL.
