Use a guia <DNT>**Live debugging with Pixie**</DNT> para consultar o conjunto completo de dados de telemetria que o Pixie coleta no armazenamento de curto prazo em seu cluster.

Saiba mais sobre o modelo de armazenamento híbrido para integração do New Relic Pixie [aqui](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/overview).

A guia <DNT>**Live debugging with Pixie**</DNT> contém uma visualização incorporada da interface do Pixie. As visualizações no Pixie são alimentadas por script PxL — script escrito na linguagem PxL do Pixie — para consultar e visualizar os dados de telemetria capturados pelo eBPF.

Pixie fornece muitos [scripts PxL de código aberto](https://github.com/pixie-io/pixie/tree/main/src/pxl_scripts) que permitem depurar:

* Tráfego em vários formatos: HTTP e HTTPs (inclusive criptografados), DNS, Postgres, MySQL, Cassandra, Redis, Kafka. Saiba mais: [Tutorial de rastreamento de solicitações](https://docs.px.dev/tutorials/pixie-101/request-tracing/).
* Desempenho de solicitação do banco de dados. Saiba mais: [consulte o tutorial de criação de perfil do banco de dados](https://docs.px.dev/tutorials/pixie-101/database-query-profiling/).
* Mapas de serviços para saber quais serviços estão se comunicando. Saiba mais: [Tutorial de desempenho de serviço](https://docs.px.dev/tutorials/pixie-101/service-performance/).
* Mapas de tráfego de rede para saber quais nós estão se comunicando. Saiba mais: [Tutorial de monitoramento de rede](https://docs.px.dev/tutorials/pixie-101/network-monitoring/).
* Uso de recursos por Node e pod. Saiba mais: [Tutorial de infra-saúde](https://docs.px.dev/tutorials/pixie-101/infra-health/).
* Picos na utilização da CPU. Saiba mais: [Tutorial de criação de perfil contínuo de CPU](https://docs.px.dev/tutorials/pixie-101/profiler/).

## Execute um script PxL [#run-script]

<Callout variant="important">
  Para obter melhores resultados, selecione `5 minutes` no seletor de hora ao usar a guia <DNT>**Live Debugging with Pixie**</DNT> . Para examinar dados menos recentes do Pixie, verifique a seção [Explorar dados do Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/explore-pixie-data) .
</Callout>

Para executar um script na guia <DNT>**Live debugging with Pixie**</DNT> :

1. Selecione um script no menu suspenso `script` .
2. Preencha todos os argumentos de script necessários. Os argumentos obrigatórios são indicados com um `*` próximo ao nome do argumento.
3. Execute o script usando o botão `RUN` no canto superior direito ou usando o atalho de teclado: `ctrl+enter` (Windows, Linux) ou `cmd+enter` (Mac).

## Siga links diretos do script [#deep-links]

As visualizações no Pixie contêm links diretos que permitem navegar facilmente entre entidades Kubernetes .

Por exemplo, a tabela <DNT>**Nodes**</DNT> no script `px/cluster` (ilustrado acima) gera uma lista de todos os nós do seu cluster. Clique em um nó para acessar o script `px/node` desse nó específico. O script `px/node` mostra uma visão geral do pod de um nó específico, do uso de CPU e memória e do tráfego de rede.
