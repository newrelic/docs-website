Encaminhe seu login do Mux Video Streaming para New Relic usando o Kinesis Firehose para obter total observabilidade em seu streaming de vídeo.

## Etapas para transmitir dados de vídeo Mux para New Relic [#mux-kinesis-stream]

1. Você precisará ter um <InlinePopover type="licenseKey"/>pronto.
2. Siga a [documentação](https://docs.mux.com/guides/system/export-amazon-kinesis-data-streams) para configurar um fluxo de dados para enviar dados do Mux para o Amazon Kinesis.
3. Crie um Amazon Kinesis Data Firehose e configure-o para enviar os dados do Mux para o New Relic. Confira [nossa documentação](/docs/logs/forward-logs/stream-logs-using-kinesis-data-firehose/) para ver as etapas de configuração de um stream do Kinesis Firehose.

## Ver dados log [#find-data]

Se tudo estiver configurado corretamente e seus dados estiverem sendo coletados, você deverá ver os dados de log em ambos os locais:

* Nossa [interface de registros](https://one.newrelic.com/launcher/logger.log-launcher)

* Nossas [ferramentas de consulta NRQL](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data). Por exemplo, você pode executar uma consulta como esta:

  ```sql
  SELECT * FROM Log
  ```

Se nenhum dado aparecer depois que você ativar nossos <InlinePopover type="logs"/>recursos, siga nossos [procedimentos padrão log de resolução de problemas](/docs/logs/log-management/troubleshooting/no-log-data-appears-ui/).

## Desativar encaminhamento de logs [#disable]

Para desativar os recursos de encaminhamento de logs, siga os procedimentos padrão na [documentação do Kinesis Data Firehose](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html). Você não precisa fazer mais nada no New Relic.
