Você pode enviar seu [log do Amazon Security Lake](https://docs.aws.amazon.com/security-lake/latest/userguide/internal-sources.html) para New Relic usando o aplicativo Serverless `newrelic-securitylake-s3-processor-LogForwarder`, que pode ser facilmente implantado no [aplicativo AWS Serverless repositório](https://serverlessrepo.aws.amazon.com/applications).

O encaminhamento de registros do Security Lake para New Relic proporcionará <InlinePopover type="logs"/>recursos aprimorados para coletar, processar, explorar, consultar e alertar sobre seus dados log .

## Instale o aplicativo Serverless [#install-app]

Para instalar o aplicativo Serverless para encaminhar seu log do Security Lake para o New Relic:

1. Certifique-se de ter um

   <InlinePopover type="licenseKey"/>

   .

2. Abra o [repositório do aplicativo AWS Serverless](https://serverlessrepo.aws.amazon.com/applications) em seu Browser.

   <img title="AWS Lambda - Select region" alt="AWS Lambda - Select region" src={serverlessAWSLambdaSelectRegion}/>

3. Procure por `newrelic` e marque

   <DoNotTranslate>**Show apps that create custom IAM roles or resource policies**</DoNotTranslate>

   para encontrar `newrelic-securitylake-s3-processor-LogForwarder`.

4. Clique nos detalhes `newrelic-securitylake-s3-processor-LogForwarder` e clique em

   <DoNotTranslate>**Deploy**</DoNotTranslate>

   .

5. Siga as orientações do aplicativo para configurar os parâmetros necessários:

   * `SecurityLakeSQSArn`
   * `SecurityLakeSubscriberRoleArn`
   * `ExternalID`
   * `NRLicenseKey`

Você receberá `SecurityLakeSQSArn` e `SecurityLakeSubscriberRoleArn` do seu provedor do Security Lake.

Lembre-se de que a região de origem do evento deve corresponder à região Lambda.
