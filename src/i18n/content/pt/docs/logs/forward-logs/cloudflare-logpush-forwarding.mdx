Configure o serviço [Logpush da Cloudflare](https://developers.cloudflare.com/logs/get-started/enable-destinations/new-relic/) para enviar log para a New Relic usando o dashboard da Cloudflare ou via API.

<Callout variant="important">
  A Cloudflare anunciou um conjunto de atualizações para campos relacionados à segurança no log da Cloudflare que começará a ser implementado em 1º de fevereiro de 2023 e será concluído em 1º de agosto de 2023. Verifique o [anúncio oficial da Cloudflare](https://developers.cloudflare.com/logs/reference/change-notices/2023-02-01-security-fields-updates/) para obter mais detalhes. Qualquer log enviado da Cloudflare para New Relic será afetado por essas alterações.
</Callout>

## Habilite Logpush para New Relic [#dashboard]

Para ativar o Cloudflare Logpush, siga as etapas na documentação do Cloudflare com base na sua escolha de configuração:

* [Habilitar através dashboard Cloudflare](https://developers.cloudflare.com/logs/get-started/enable-destinations/new-relic/#manage-via-the-cloudflare-dashboard)
* [Habilitar via API](https://developers.cloudflare.com/logs/get-started/enable-destinations/new-relic/#manage-via-api)

## Desativar encaminhamento de logs [#disable]

Para desativar os recursos de encaminhamento de logs, siga os procedimentos padrão na [documentação do Cloudflare Logpush](https://developers.cloudflare.com/logs/faq/). Você não precisa fazer mais nada no New Relic.
