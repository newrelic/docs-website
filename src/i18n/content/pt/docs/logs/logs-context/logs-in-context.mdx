Seja qual for o seu aplicativo arquitetura, temos [soluções para reportar seu log para New Relic](/docs/logs/get-started/get-started-log-management#integrate-logs).

Ao usar nosso <InlinePopover type="apm"/>agente, agente de infraestrutura ou integração OpenTelemetry para relatar dados log , você obtém <DNT>**logs in context**</DNT>, um recurso que permite ver seus dados log no contexto de outras experiências de interface da plataforma. Por exemplo, você verá dados de log em experiências de interface como APM, monitoramento de infraestrutura, distributed tracing e Errors Inbox, para citar alguns. Com esse conhecimento contextual, você pode se aprofundar no desempenho do seu aplicativo ou host, sem precisar pesquisar manualmente os dados de log.

## Veja a causa raiz dos problemas em sua plataforma [#compatibility-requirements]

Ao reunir todos os dados de aplicativos e infraestrutura em uma única solução, você pode chegar à causa raiz dos problemas com mais rapidez. O logs contextualizados ajuda você a ver rapidamente padrões e tendências significativas.

Não perca tempo tentando restringir todos os seus registros de diferentes partes da sua plataforma. Em vez disso, ative o logs contextualizados para ver as linhas log exatas necessárias para identificar e resolver um problema.

## Logs contextualizados APM [#apm-automatic]

Nosso agente <InlinePopover type="apm"/>tem logs contextualizados, incluindo encaminhamento de logs, ativado por padrão. Para obter mais informações sobre como obter e usar o <InlinePopover type="apm"/>logs contextualizados, consulte [LogAPM ](/docs/apm/new-relic-apm/getting-started/get-started-logs-context).

## Qual é o próximo? [#what-next]

Após configurar o logs contextualizados para APM ou monitoramento de infraestrutura, aproveite ao máximo seus dados de log na interface do New Relic :

* Explore os dados de registro em sua plataforma com nossa [interface de registro](/docs/logs/log-management/ui-data/use-logs-ui).
* Veja seu logs contextualizados do desempenho do seu aplicativo na [interfaceAPM ](/docs/logs/log-management/ui-data/use-logs-ui/#links). Solucione [erros](/docs/apm/apm-ui-pages/error-analytics/errors-page-find-fix-verify-problems/) com [distributed tracing](/docs/apm/distributed-tracing/ui-data/understand-use-distributed-tracing-data), rastreamento de pilha, registro de aplicativo e muito mais.
* Configure [o alerta](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/).
* [consulte seus dados](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/) e [crie dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/).
