Com nossa solução <InlinePopover type="logs"/>, você tem controle direto sobre quais dados são relatados à New Relic. Para garantir a privacidade dos dados e limitar os tipos de informações que New Relic recebe, nenhum dado do cliente é capturado, exceto aqueles que você fornece na chamada de API ou no encaminhador de configuração de logs. Todos os dados do serviço de log são então relatados à New Relic por HTTPS.

Este documento descreve considerações adicionais de segurança para seus dados de registro em log. Para mais informações sobre as medidas de segurança da New Relic:

* Consulte nossa [documentação de segurança e privacidade](/docs/security/security-privacy/data-privacy/data-privacy-new-relic).
* Visite o [site de segurança da New Relic](https://newrelic.com/security).
* Leia esta [postagem do blog](https://blog.newrelic.com/product-news/security-use-cases-for-logs/) que explica como você pode usar nossas ferramentas de gerenciamento de logs para obter visibilidade sobre algumas das ameaças mais graves aos negócios digitais modernos.

## Ofuscação de logs [#log-obfuscation]

Os dados que você envia para New Relic, incluindo qualquer filtragem adicional, são controlados pela configuração do [encaminhador de logs](/docs/logs/forward-logs/enable-log-management-new-relic) que você usa. Você controla quais dados dos clientes são registrados, portanto, siga as diretrizes de segurança da sua organização para mascarar, ofuscar ou impedir o envio de quaisquer dados confidenciais.

Adicionalmente, disponibilizamos o recurso de ofuscação de logs:

* Nossa ofuscação automática de logs, que é fornecida por padrão para todos os seus dados New Relic Logs .
* Nosso recurso de ofuscação de logs, que permite configurar uma regra de ofuscação personalizada.

A seguir, explicaremos ambos com mais detalhes.

### Ofuscação automática de logs [#auto-obfuscation]

O recurso ofuscação automática de logs está habilitado para todas as contas New Relic , por padrão. Este recurso mascara automaticamente padrões numéricos que parecem ser para itens como cartões de crédito ou números de Seguro Social. Todos os números inteiros, incluindo espaços e hifens que podem ser usados como delimitadores, são substituídos por uma sequência de `X`es. Esta ofuscação automática aplica-se apenas ao campo `message` .

Números que parecem ser de cartão de crédito (treze a dezesseis dígitos) são ofuscados como `XXXXXXXXXXXXXXXX`. Por exemplo:

* Números com hífens, como `4111-1111-1111-1111`
* Números com espaços, como `4111 1111 1111 1111`
* Números com treze (Visa), quatorze (Diner's Club), quinze (American Express, JCB) ou dezesseis dígitos (Visa, Mastercard, Discover, JCB), como `4111111111111111`

Números de nove dígitos com hífens que parecem ser números de Seguro Social, como `123-45-6789`, são ofuscados como `XXXXXXXXX`. Números de nove dígitos com espaços, como `123 45 6789` ou hífens em um padrão diferente, como `12-345-67-89`, não são ofuscados automaticamente.

Se você precisar desativar a ofuscação automática, [entre em contato com o suporte](https://support.newrelic.com).

### Ofuscação personalizada de logs [#custom-obfuscation]

Com nosso recurso de ofuscação de logs, disponível no [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus), permitimos que você configure regras para ofuscar partes de seus dados quando eles chegarem ao New Relic. Isso torna tudo mais fácil e seguro para você, em vez de precisar rastrear dados confidenciais em todos os seus aplicativos e tentar corrigi-los na fonte. Para mais informações, veja [ofuscação de logs](/docs/logs/ui-data/obfuscation-ui).
