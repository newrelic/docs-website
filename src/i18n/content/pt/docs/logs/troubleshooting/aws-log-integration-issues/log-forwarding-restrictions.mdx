O New Relic tem as seguintes restrições com a integração AWS para encaminhamento de logs ao ingerir logs via S3 ou Firehose:

### Para ingerir logs da AWS do S3 [#s3-restrictions]

* O tamanho do arquivo de log deve ser menor que 80 MB.
* O tamanho de uma única linha de log deve ser menor que 8 MB.
* Linhas de log entre 1 MB e 8 MB serão divididas em várias mensagens e encaminhadas para o New Relic.
* Atualmente, o New Relic não oferece suporte à análise de JSON do cliente ou de linhas de log.
* Atualmente, o New Relic suporta apenas os formatos de compressão `Gzip` e `Bzip2` para arquivo de log. Ele processa arquivos em outros formatos compactados como arquivos descompactados.

### Para ingerir logs do CloudWatch [#cloudwatch-restrictions]

* Atualmente, o New Relic não oferece suporte à análise de JSON do cliente ou de linhas de log.
* Atualmente, o New Relic oferece suporte apenas [a camadas Lambda](https://github.com/newrelic/newrelic-lambda-layers) para análise lógica.
