Use nossa interface de registro para:

* Identifique padrões interessantes ou significativos em seu registro.
* Examine mais contexto em torno de uma linha log específica.
* Explore e manipule seus dados de registro com filtros e regras de análise.
* Consulte seus dados para criar visualizações personalizadas ou adicione-as ao painel
* Organize os dados de log da sua organização e otimize o desempenho da consulta com partições de dados.
* Configure a condição do alerta para os problemas que você deseja prevenir.

Para se manter atualizado com novos recursos e melhorias, assine nosso [feed RSS para obter notas de lançamento de log](/docs/release-notes/logs-release-notes).

## Encontre a interface de registro [#find-ui]

Para localizar a interface de registro, em <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, selecione <DNT>**Logs**</DNT>.

## Explore seus dados log [#ui-workflow]

Use a navegação esquerda na interface de log como um fluxo de trabalho fácil por meio de todos os logs, atributos, padrões, registro em tempo real e consultas. Gerencie seus dados de log eliminando ou analisando dados, criando partições de dados e configurando alertas. aplique hash ou mascarar quaisquer dados sensíveis em seu log com [expressão de ofuscação e regras](/docs/logs/ui-data/obfuscation-ui). Obtenha mais detalhes sobre logs específicos e seus atributos na navegação central.

Para explorar seus dados de registro em log, siga este fluxo de trabalho básico.

<CollapserGroup>
  <Collapser
    id="workflow-path"
    title="1. Vá para a interface de registro."
  >
    Acesse nossa interface de registro em <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Logs**</DNT>.

    Se você usa nosso [data center na região da UE](/docs/using-new-relic/welcome-new-relic/get-started/our-eu-us-region-data-centers/), acesse <DNT>**[one.eu.newrelic.com](https://one.eu.newrelic.com) > Logs**</DNT>.

    Se você não [personalizou sua barra de navegação do New Relic](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/#custom-navigation), vá para <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, clique em <DNT>**Browse data**</DNT> e selecione <DNT>**Logs**</DNT>.
  </Collapser>

  <Collapser
    id="workflow-patterns"
    title="2. Procure padrões."
  >
    <img
      title="Logs patterns UI"
      alt="Screenshot of Log patterns UI"
      src="/images/logs_screenshot-crop_patterns.webp"
    />

    * Para detectar picos ou quedas suspeitas na mensagem do log, clique em

      <DNT>
        **[Patterns](/docs/logs/ui-data/find-unusual-logs-log-patterns/)**
      </DNT>

      no painel de navegação esquerdo.

    * Para visualizar o registro de um período de tempo específico, clique nesse ponto (ou clique e arraste uma área) no gráfico ou use o seletor de hora.
  </Collapser>

  <Collapser
    id="workflow-focus"
    title="3. Limite seu foco."
  >
    A navegação à esquerda inclui opções para ajudá-lo a restringir o foco dos resultados iniciais da pesquisa ou para encontrar rapidamente valores discrepantes.

    * Se você não souber por onde começar, clique em

      <DNT>
        **Attributes**
      </DNT>

      no painel de navegação esquerdo e selecione valores adicionais conforme necessário. Por exemplo, se um host listado no atributo `hostname` estiver gerando significativamente mais mensagens de erro do que os outros, selecione esse valor para aplicá-lo à sua pesquisa.

    * Para tornar sua mensagem do log mais fácil de consultar e entender, use nossas [regras de análise integradas](/docs/logs/ui-data/built-log-parsing-rules/) ou [crie suas próprias regras de análise](/docs/logs/ui-data/parsing/) para atributo que correspondam a um valor específico.

    * Para gerenciar a quantidade de dados log coletados e armazenar menos logs, crie [regras de filtro de descarte](/docs/logs/ui-data/drop-data-drop-filter-rules/) que evitem a coleta de dados desnecessários.
  </Collapser>

  <Collapser
    id="workflow-details"
    title="4. Examine os detalhes do log."
  >
    <img
      title="Logs details UI"
      alt="Screenshot of Log details UI including severity level"
      src="/images/logs_screenshot-crop_log-details.webp"
    />

    Se aplicável, a mensagem do log indica um nível de severidade (`INFO`, `DEBUG`, etc.). Selecione uma mensagem do log para visualizar seus detalhes como tabela de atributos ou como JSON.

    * Para ver quais atributos estão incluídos em uma mensagem do log, clique na linha do log.
    * Para ajudar a solucionar problemas relacionados a um valor específico nos detalhes log , [mostre os logs circundantes](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs) para obter os detalhes do atributo.
    * Para obter mais detalhes em mensagens extremamente longas, expanda os dados armazenados como [blobs](/docs/logs/ui-data/long-logs-blobs/).
  </Collapser>

  <Collapser
    id="workflow-search"
    title="5. consulta e pesquisa."
  >
    Por padrão, a interface de log mostra todo o seu log, mas você também pode pesquisar com palavras-chave ou frases para encontrar os resultados desejados. Por exemplo:

    ```
    process failed
    ```

    Você também pode usar o campo de pesquisa com menu suspenso de digitação antecipada para selecionar um atributo, operador e valor. Por exemplo:

    ```
    service_name equals my service
    ```

    Para ajudar sua consulta a se concentrar nos detalhes necessários, adicione ou remova atributo seguindo um destes procedimentos:

    * Na navegação esquerda, selecione <DNT>**Attributes**</DNT>, selecione os valores desejados e clique em <DNT>**Query logs**</DNT>.

    * Na tabela de registros, clique no ícone <DNT>**+**</DNT> para adicionar um atributo e, em seguida, clique nele para adicioná-lo à consulta.

      Para mudar da barra de pesquisa de sintaxe Lucene para uma consulta NRQL, clique no botão <DNT>**NRQL**</DNT> ao lado da barra de pesquisa. Não há equivalência direta entre NRQL e Lucene, e isso significa que a consulta NRQL geralmente será mais simples e não tão complexa, mas isso fornece um início útil para criar uma consulta com os mesmos dados.

      Para obter mais informações sobre como consultar seu log, consulte a [documentação sobre sintaxe de consulta de log](/docs/logs/ui-data/query-syntax-logs/).
  </Collapser>

  <Collapser
    id="query-builder"
    title="7. Análise avançada de logs com NRDB criador de consulta."
  >
    <img
      title="Advanced log analysis with NRDB query builder"
      alt="Screenshot of link to NRDB query builder"
      src="/images/logs_screenshot-crop_nrql-button.webp"
    />

    Você pode escrever consulta NRQL em evento de log sem perder seus filtros com o [criador de consulta NRDB](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/). Clique no botão <DNT>**NRQL**</DNT> à direita da barra de pesquisa de registros e comece a consultar.
  </Collapser>

  <Collapser
    id="workflow-related"
    title="8. Obtenha o registro relacionado."
  >
    Concentrar-se no registro mais útil pode ajudá-lo com:

    * <DNT>
        **Optimizing performance:**
      </DNT>

      Para organizar os dados em uma conta e otimizar o desempenho da consulta, crie [regras de partição de dados](/docs/logs/ui-data/data-partitions/).

    * <DNT>
        **Reviewing deployments:**
      </DNT>

      Para ver imediatamente como o seu sistema responde à implantação ou outras alterações no aplicativo, habilite [o registro em tempo real](/docs/logs/troubleshooting/view-log-messages-real-time-live-tail).

    * <DNT>
        **Bypassing unrelated details:**
      </DNT>

      Para visualizar todo o registro de um valor específico, revise a lista de atributos em

      <DNT>
        **Log details**
      </DNT>

      do log selecionado e continue adicionando ou removendo atributos conforme necessário.

    * <DNT>
        **Finding the root cause:**
      </DNT>

      Para ajudar a identificar a causa raiz de um problema antes de ele ocorrer ou seu impacto após um evento, clique em

      <DNT>
        **...**
      </DNT>

      em

      <DNT>
        **Log details**
      </DNT>

      para [mostrar o registro circundante](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs).

    * <DNT>
        **Getting more context:**
      </DNT>

      Para ver o log relacionado a outros dados de telemetria para seus aplicativos e hosts, use [logs contextualizados](/docs/logs/logs-context/logs-in-context).
  </Collapser>

  <Collapser
    id="traces-in-context-overview"
    title="9. Mergulhe fundo no rastreamento distribuído."
  >
    Depois de restringir o conjunto de logs com filtros e abrir um log específico, você poderá ver o rastreamento distribuído relacionado. Contanto que você tenha configurado distributed tracing e haja amostras de rastreamento relacionadas ao log, você verá uma opção para visualizá-los. Esta é uma maneira rápida de visualizar informações trace sem acessar a página principal distributed tracing .

    Se você abrir o painel <DNT>**Log details**</DNT> e visualizar uma seção <DNT>**Distributed trace**</DNT> , terá duas opções para visualizar o rastreamento de span em uma visualização em cascata:

    * Clique diretamente no nome do trace ou clique no ícone com uma seta à direita, que abre a visualização de foco em cascata que destaca extensões trace com erros.

      <img
        style={{ align: "left",width: "85%" }}
        title="Screenshot showing how to reach the log details page"
        alt="Screenshot showing how to reach the log details page"
        src="/images/logs_screenshot-crop_traces-in-context.webp"
      />

    * Alternativamente, você pode clicar em <DNT>**Explore**</DNT> para abrir uma cascata não filtrada onde você pode clicar em todos os trechos.

      <Callout variant="tip">
        Se precisar de ajuda para usar a cascata trace , consulte [Compreender e usar a interface distributed tracing ](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui/).
      </Callout>
  </Collapser>

  <Collapser
    id="share"
    title="10. Compartilhe suas descobertas."
  >
    Use qualquer uma das principais funções da interface do New Relic para [explorar seus dados](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/) (conta específica, intervalo de tempo, métrica e evento, criador de consulta, etc.) e compartilhar os dados com gráficos, adicionar ao painel, etc. Para obter mais informações, consulte os [exemplos](#examples) neste documento.
  </Collapser>
</CollapserGroup>

## Exportar [#export]

Para exportar um subconjunto do seu log para compartilhar, o usuário pode aproveitar nosso recurso de exportação dedicado na interface do log. Este recurso permitirá a exportação de até 5.000 registros de log em formato CSV ou JSON. Os registros de log serão baixados localmente em sua máquina, na pasta `/Downloads/` do seu navegador atual.

## Salve suas visualizações [#saved-views]

Você pode salvar sua consulta de log, configuração de tabela, intervalo de tempo e agrupamento de atributos em uma visualização salva, para que você possa retornar rapidamente a ela mais tarde. Para salvar uma visualização log Analytics depois de configurá-la:

1. Em qualquer uma das páginas de interface na seção

   <DNT>
     **Your logs**
   </DNT>

   , clique em

   <DNT>
     **Saved views**
   </DNT>

   à direita e, em seguida, clique em

   <DNT>
     **Create new**
   </DNT>

   .

2. Dê um nome à visualização atual.

3. Selecione quais aspectos da visualização atual você deseja salvar.

4. Selecione permissões para acesso dos membros da sua equipe.

## Exemplos [#examples]

Aqui estão alguns exemplos de como você pode usar nossa interface de log para obter informações detalhadas.

<CollapserGroup>
  <Collapser
    id="alert"
    title="Crie um alerta a partir dos dados de log."
  >
    Você pode criar condição do alerta diretamente na interface de log:

    1. Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Logs**</DNT>.

    2. Pesquise os resultados que deseja alertar; por exemplo, `service_name:"your service" "fatal error"`.

    3. Clique no sino <Icon name="fe-bell-off"/> ícone ao lado da barra de consulta.

    4. Conclua o procedimento

    5. Revise a consulta NRQL que alimentará a condição do alerta.

       Depois de salvar a condição do alerta, você poderá visualizá-la na interface <InlinePopover type="alerts"/>, onde poderá fazer alterações adicionais conforme necessário.
  </Collapser>

  <Collapser
    id="dashboard"
    title="Adicione gráficos de log a um dashboard."
  >
    Na interface <DNT>**Logs**</DNT>, você pode adicionar dados log como um [widget de tabela](/docs/query-your-data/explore-query-data/use-charts/chart-types/#widget-table) a um dashboard. Você também pode criar tabelas de log com métrica e evento ou o criador de consulta no New Relic.

    1. Vá para

       <DNT>
         **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Logs**
       </DNT>

       .

    2. À direita, clique em

       <DNT>
         **Saved views**
       </DNT>

       e selecione uma visualização salva. Ou pesquise os resultados que deseja plotar; por exemplo, `service_name:"checkout service" "process failed"`.

    3. Clique em

       <DNT>
         **Add to dashboard**
       </DNT>

       e preencha os detalhes para adicionar o gráfico log como uma tabela a um dashboard novo ou existente .

       Então, no seu dashboard , você pode:

    * Use funções padrão do widget dashboard , como copiar, editar, excluir e muito mais.

    * Clique em qualquer linha log para mostrar detalhes sobre ela.

    * Atualize sua consulta para adicionar mais colunas.

    * Consulte dados log de outras contas disponíveis e adicione mais gráficos (por exemplo, como dados comparativos) ao seu dashboard.

    * Clique em

      <DNT>
        **Open in logs**
      </DNT>

      para ir diretamente para a interface do usuário

      <DNT>
        **Logs**
      </DNT>

      para obter mais resolução de problemas.
  </Collapser>

  <Collapser
    id="troubleshoot-error"
    title="Solucione um erro (faça logs contextualizados)."
  >
    Para entender melhor o que estava acontecendo no host no momento em que ocorreu um erro em seu aplicativo, você deve ser capaz de ver [o logs contextualizados](/docs/logs/logs-context/configure-logs-context-apm-agents/). Em seguida, para solucionar erros relacionados:

    1. Vá para <DNT>**APM > Errors inbox**</DNT> ou <DNT>**APM > (select an app) > Events > Error analytics**</DNT> e selecione um trace de erro.

    2. Nos detalhes trace de erros, clique em <DNT>**Open in logs**</DNT>.

    3. Navegue pelos detalhes do log relacionado.

    4. Para identificar o host que está gerando o erro, clique em <DNT>**...**</DNT> de sua preferência e depois clique em <DNT>**Show surrounding logs**</DNT>.

       <img
         title="APM errors inbox UI with logs"
         alt="Screenshot of APM errors inbox UI with logs"
         src="/images/logs_screenshot-crop_surrounding-logs.webp"
       />
  </Collapser>

  <Collapser
    id="troubleshoot-latency"
    title="Solucionar problemas de latência (logs contextualizados)"
  >
    Para solucionar problemas de latência dessa maneira, você deve poder ver [o logs contextualizados](/docs/logs/logs-context/configure-logs-context-apm-agents/). Então, para entender melhor como seus sistemas estavam operando quando o desempenho diminuiu visivelmente:

    1. Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > APM & services > (select an app)**</DNT>.

    2. Na página <DNT>**Summary**</DNT> do aplicativo, clique em <DNT>**Distributed tracing**</DNT> e selecione um trace particularmente lento .

    3. No trace <DNT>**Details**</DNT>, clique em <DNT>**Logs**</DNT>.

       <img
         title="APM distributed tracing UI with logs"
         alt="Screenshot of APM distributed tracing UI with logs"
         src="/images/logs_screenshot-full_dt-logs.webp"
       />
  </Collapser>
</CollapserGroup>

## Links para fazer login na New Relic [#links]

Dependendo da sua assinatura do New Relic, você pode acessar seu registro de vários locais na interface do New Relic. Para algumas dessas opções, você deve poder ver [o logs contextualizados](/docs/logs/logs-context/configure-logs-context-apm-agents/).

<Callout variant="tip">
  Permita que o <InlinePopover type="apm"/>agente do seu aplicativo encaminhe dados log diretamente para o New Relic com [logs contextualizados do APM](/docs/apm/new-relic-apm/getting-started/get-started-logs-context). Não há necessidade de instalar ou manter software adicional de terceiros!
</Callout>
