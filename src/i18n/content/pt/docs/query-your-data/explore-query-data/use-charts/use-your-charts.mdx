Depois de criar um gráfico, você pode personalizar sua aparência para melhor apresentar os dados. Você pode compartilhar um gráfico em diferentes formatos e adicionar um gráfico a um dashboard novo ou existente.

## Altere a aparência do seu gráfico [#change-appearance]

Quando você executa sua consulta no modo NRQL ou visualiza seu gráfico enquanto usa métricas e eventos para especificar dados, o New Relic analisa seus dados e aplica um [tipo de gráfico](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-types) adequado aos seus dados.

Existem várias opções de tipos de gráficos para você escolher. Para alterar o tipo de gráfico, use o menu <DNT>**Chart type**</DNT> à direita do gráfico atual.

## Personalize seus gráficos [#customize-charts]

Embora façamos o possível para otimizar a forma como exibimos seus dados, às vezes você pode ter outras necessidades. Dependendo do tipo de gráfico, existem opções adicionais de personalização disponíveis.

<Callout variant="important">
  O sistema utilizado para representar unidades de bytes (como KB, MB, GB, TB, etc.) nesses gráficos segue a representação métrica. Isso significa que:

  * 1 KB (quilobyte) = 1.000 bytes
  * 1 MB (megabyte) = 1.000 KB
  * 1 GB (gigabyte) = 1.000 MB
  * 1 TB (terabyte) = 1.000 GB

  Este sistema métrico está alinhado com o Sistema Internacional de Unidades (SI), que utiliza potências de 10 para cálculos e representação.
</Callout>

Leia as seções abaixo para saber mais sobre o recurso de customização. Você também pode ver [este blog](https://newrelic.com/blog/how-to-relic/dashboard-chart-customizations) e [o vídeo complementar](https://www.youtube.com/watch?v=LA2jM_asl94) para obter um guia passo a passo de algumas das opções de personalização.

**Para acessar as opções de personalização do gráfico**:

1. No widget que você deseja personalizar, no menu <Icon name="fe-more-horizontal" />, selecione <DNT>**Edit**</DNT>.
2. No painel <DNT>**Customize this visualization**</DNT>, configure as opções descritas nas seções abaixo.

<CollapserGroup>
  <Collapser id="widget-link-description" title="Links e descrições do widget">
    Você pode adicionar links e descrições aos widgets do seu dashboard para fornecer contexto adicional e opções de navegação.

    **Links de widget** aparecem como links clicáveis nos títulos dos widgets, permitindo que os usuários naveguem para dashboards relacionados, documentação ou recursos externos. Quando os usuários visualizam o dashboard, clicar no título do widget abre o link configurado.

    **Descrições de widget** fornecem contexto adicional sobre o que um widget exibe, mostradas como uma dica de ferramenta quando os usuários apontam para o widget.

    **Para adicionar um link ou descrição**:

    1. Para adicionar um link, no campo <DNT>**URL**</DNT>, insira uma URL. Observe os seguintes requisitos:

       * Comprimento máximo: 2.048 caracteres
       * Protocolos suportados: apenas `http://` e `https://`
       * URLs inválidas exibirão um erro de validação.

    2. Para adicionar uma descrição, no campo <DNT>**Description**</DNT>, insira uma descrição em texto simples.

    3. Clique em <DNT>**Apply changes**</DNT>.

    **Para remover um link ou descrição**:

    1. Limpe o campo <DNT>**URL**</DNT> ou <DNT>**Description**</DNT>.
    2. Clique em <DNT>**Apply changes**</DNT>.

    **Para adicionar, editar ou remover links e descrições de widgets via JSON**:

    1. No menu <Icon name="fe-more-horizontal" /> do dashboard, selecione <DNT>**Manage JSON**</DNT>.

    2. No editor JSON, localize o widget que você deseja modificar e atualize os campos `link` e `description`. Para adicionar ou editar, insira um valor. Para remover, defina o campo como `null`:

       ```json
       widgets: [{
         id: "WIDGET_ID"
         link: "https://example.com"  // Enter URL or null to remove
         description: "Your widget description"  // Enter text or null to remove
       }]
       ```

    3. Clique em <DNT>**Save changes**</DNT>.

    Se você encontrar problemas com links de widgets não funcionando conforme o esperado, consulte [Problemas com links de widgets](/docs/query-your-data/explore-query-data/dashboards/troubleshooting-chart-errors/#widget-link-issues) para obter ajuda na solução de problemas.
  </Collapser>

  <Collapser id="custom-date-time" title="Formatar data e hora">
    Personalize o formato de data e hora para tabelas e outdoors na seção <DNT>**Data format**</DNT>. Você tem a flexibilidade de mantê-lo como está ou modificar o formato para atender às suas necessidades. Escolha entre as seguintes opções:

    * <DNT>**Numeric**</DNT>: Retém o timestamp como valores numéricos.
    * <DNT>**Date**</DNT>: Exibe datas em formato padrão.
    * <DNT>**Duration**</DNT>: Mostra o intervalo de tempo entre o timestamp e o momento presente.
    * <DNT>**Time to Now**</DNT>: Indica o tempo decorrido desde o timestamp.

    Selecione o formato mais adequado para configurar a apresentação de dados com base nas suas necessidades.
  </Collapser>

  <Collapser id="customize-y-axis" title="Personalize o eixo Y">
    Para exibir dados dentro de determinados valores em gráficos de linha e gráficos de área, na seção <DNT>**Y-axis**</DNT>, defina um valor mínimo e máximo para o eixo.

    Se nenhuma opção de personalização for selecionada, [o painel](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) exibirá automaticamente o eixo Y completo de 0 ao valor superior mais uma margem.

    Para desativar ou ativar a legenda para gráficos de linhas, gráficos de áreas e histogramas, na seção <DNT>**Legend**</DNT>, use o botão de alternância <DNT>**Visibility**</DNT>.
  </Collapser>

  <Collapser id="bar-charts-pie-charts-tables" title="Gráficos de barras, gráficos de pizza e tabelas">
    Se o número de [facetas](/docs/query-your-data/explore-query-data/dashboards/filter-new-relic-one-dashboards-facets) em uma consulta for maior que 5.000 para gráficos de barras, gráficos de pizza ou tabelas, a faceta <DNT>**Other groups**</DNT> agrega o restante das facetas. Para exibir ou ocultar a opção <DNT>**Other groups**</DNT>, na seção <DNT>**Other groups**</DNT>, use o botão de alternância <DNT>**Visible**</DNT>.
  </Collapser>

  <Collapser id="dash-color" title="Personalização de cores">
    Você pode personalizar a cor de seus gráficos por série em área, barra, linha, pizza e gráficos de barras empilhados.

    Para personalizar a cor por série, na seção <DNT>**Colors**</DNT>, selecione o ícone <Icon name="fe-plus" />, escolha a série que deseja alterar e selecione a cor desejada.
  </Collapser>

  <Collapser id="dash-units" title="Personalização de unidades">
    Você pode personalizar a unidade em seu eixo Y e em cada uma de suas séries. Está disponível em gráficos de barras de área, linha e empilhados.

    Para personalizar a unidade, na seção <DNT>**Units**</DNT>, selecione o ícone <Icon name="fe-plus" />, escolha a série que precisa de outra unidade e selecione a unidade.
  </Collapser>

  <Collapser id="dash-null" title="Opções de exibição de valor nulo">
    Os gráficos de série temporal às vezes parecem quebrados quando há um zero no atributo do evento que está sendo consultado. No entanto, um valor nulo nem sempre pode representar a situação corretamente.

    Você pode escolher uma opção para exibir valores nulos para facilitar a compreensão dos gráficos de série temporal. Eles incluem:

    * Deixar como nulo (padrão): valores nulos são exibidos como lacunas no gráfico, o que pode resultar em linhas desconectadas. Você pode usar esta opção se for importante encontrar lacunas nos relatórios ou quando não fizer sentido conectar uma linha contínua.

    * Remova o valor: pontos nulos não existirão no gráfico. Por exemplo, um gráfico que exibe um ponto de dados por hora em um dia mostraria menos de 24 pontos.

    * Preservar o último valor: Valores nulos manterão o valor do período anterior. Por exemplo, se um gráfico relatar vendas acumuladas a cada hora, mas faltarem os dados de uma hora, a hora faltante refletirá o mesmo número de vendas da hora anterior.

    * Transforme o valor em zero: os dados ausentes aparecem como zero. Esta opção pode ser útil quando a ausência de dados significa que não há transação, contagem ou uso significativo.

      <img title="Null value chart options" alt="Null value chart options" src="/images/dashboards_screenshot-crop_null-values.webp" />

      <figcaption>
        Você pode escolher entre quatro maneiras de exibir valores nulos em seus gráficos.
      </figcaption>

    Para personalizar como seus valores nulos aparecem, na seção <DNT>**Null values**</DNT>, selecione a opção a ser aplicada. Para usar opções diferentes para cada série, selecione o ícone <Icon name="fe-plus" /> ao lado de <DNT>**Null values per series**</DNT> para configurá-las.
  </Collapser>

  <Collapser id="thresholds" title="Limites">
    Limites são dicas visuais que adicionam contexto aos seus gráficos e tabelas, tornando a interpretação dos dados mais clara. Veja como eles podem elevar suas visualizações:

    * Linhas limite: Estabeleça valores mínimos e máximos aceitáveis em gráficos de séries temporais.

    * Regiões coloridas: Sombreie as áreas entre os limites para identificar visualmente as faixas normais de operação ou destacar zonas críticas.

    * Coloração condicional de células: aplique códigos de cores às células da tabela com base em seus valores, revelando instantaneamente tendências ou valores discrepantes de dados.

      <Video type="wistia" id="ikpu24sqjr" />

    * Em gráficos de série temporal:

      1. Edite o gráfico e expanda o menu <DNT>**Thresholds**</DNT> .

      2. Selecione <DNT>**+**</DNT> para adicionar um limite e insira um <DNT>**Name**</DNT>.

      3. Insira valores para <DNT>**From**</DNT> e <DNT>**To**</DNT>. Se quiser criar uma única linha, insira o mesmo número para ambos os campos.

      4. Selecione um nível de gravidade. Isso determina a cor da linha ou região.

         <img title="Configure chart thresholds in time series charts" alt="Screenshot of how to configure thresholds in a time series chart" src="/images/charts_screenshot-full_chart-threshold-time-series.webp" />

    * Em tabelas:

      1. Edite o gráfico e expanda o menu <DNT>**Thresholds**</DNT> .

      2. Selecione <DNT>**+**</DNT> para adicionar um limite e selecione um <DNT>**Column**</DNT>.

      3. Insira valores para <DNT>**From**</DNT> e <DNT>**To**</DNT>. Para definir um limite superior, deixe o campo <DNT>**From**</DNT> vazio. Para definir um limite inferior, deixe o campo <DNT>**To**</DNT> vazio.

         <img title="Configure chart thresholds in tables" alt="Screenshot of how to configure thresholds in a table" src="/images/charts_screenshot-full_chart-threshold-table.webp" />
  </Collapser>
</CollapserGroup>

## Opções de compartilhamento e visualização de gráficos [#share-charts]

A maioria dos gráficos tem várias opções, incluindo uma opção de incorporação de gráfico, obtenção de um gráfico como uma imagem e adição de um gráfico a um dashboard. Consulte [Importar, exportar e adicionar dashboards e gráficos](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/). Para compartilhar um gráfico com sua equipe, consulte [Compartilhar dashboard e gráficos internamente](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-internally/#share-charts-internally). Para compartilhar um gráfico externamente, consulte [Compartilhar dashboard e gráficos publicamente](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#chart).
