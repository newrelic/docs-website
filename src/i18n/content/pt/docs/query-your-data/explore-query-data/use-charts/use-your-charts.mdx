Depois de criar um gráfico, você pode personalizar sua aparência para melhor apresentar os dados. Você pode compartilhar um gráfico em diferentes formatos e adicionar um gráfico a um dashboard novo ou existente.

## Altere a aparência do seu gráfico [#change-appearance]

Quando você executa sua consulta no modo NRQL ou visualiza seu gráfico enquanto usa métricas e eventos para especificar dados, o New Relic analisa seus dados e aplica um [tipo de gráfico](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-types) adequado aos seus dados.

Existem várias opções de tipos de gráficos para você escolher. Para alterar o tipo de gráfico, use o menu <DNT>**Chart type**</DNT> à direita do gráfico atual.

## Personalize seus gráficos [#customize-charts]

Embora façamos o possível para otimizar a forma como exibimos seus dados, às vezes você pode ter outras necessidades. Dependendo do tipo de gráfico, existem opções adicionais de personalização disponíveis.

<Callout variant="important">
  O sistema utilizado para representar unidades de bytes (como KB, MB, GB, TB, etc.) nesses gráficos segue a representação métrica. Isso significa que:

  * 1 KB (quilobyte) = 1.000 bytes
  * 1 MB (megabyte) = 1.000 KB
  * 1 GB (gigabyte) = 1.000 MB
  * 1 TB (terabyte) = 1.000 GB

    Este sistema métrico está alinhado com o Sistema Internacional de Unidades (SI), que utiliza potências de 10 para cálculos e representação.
</Callout>

Leia as seções abaixo para saber mais sobre o recurso de customização. Você também pode ver [este blog](https://newrelic.com/blog/how-to-relic/dashboard-chart-customizations) e [o vídeo complementar](https://www.youtube.com/watch?v=LA2jM_asl94) para obter um guia passo a passo de algumas das opções de personalização.

  <Collapser
    id="dash-null"
    title="Opções de exibição de valor nulo"
  >
    Os gráficos de série temporal às vezes parecem quebrados quando há um zero no atributo do evento que está sendo consultado. No entanto, um valor nulo nem sempre pode representar a situação corretamente.

    Você pode escolher uma opção para exibir valores nulos para facilitar a compreensão dos gráficos de série temporal. Eles incluem:

    * Deixar como nulo (padrão): valores nulos são exibidos como lacunas no gráfico, o que pode resultar em linhas desconectadas. Você pode usar esta opção se for importante encontrar lacunas nos relatórios ou quando não fizer sentido conectar uma linha contínua.

    * Remova o valor: pontos nulos não existirão no gráfico. Por exemplo, um gráfico que exibe um ponto de dados por hora em um dia mostraria menos de 24 pontos.

    * Preservar o último valor: Valores nulos manterão o valor do período anterior. Por exemplo, se um gráfico relatar vendas acumuladas a cada hora, mas faltarem os dados de uma hora, a hora faltante refletirá o mesmo número de vendas da hora anterior.

    * Transforme o valor em zero: os dados ausentes aparecem como zero. Esta opção pode ser útil quando a ausência de dados significa que não há transação, contagem ou uso significativo.

      4. Se desejar usar opções diferentes para cada série, selecione a opção <Icon name="fe-plus"/> ícone ao lado de <DNT>**Null values per series**</DNT> para configurá-los.

  <Collapser
    id="thresholds"
    title="Limites"
  >
    Limites são dicas visuais que adicionam contexto aos seus gráficos e tabelas, tornando a interpretação dos dados mais clara. Veja como eles podem elevar suas visualizações:

    * Linhas limite: Estabeleça valores mínimos e máximos aceitáveis em gráficos de séries temporais.

    * Regiões coloridas: Sombreie as áreas entre os limites para identificar visualmente as faixas normais de operação ou destacar zonas críticas.

    * Coloração condicional de células: aplique códigos de cores às células da tabela com base em seus valores, revelando instantaneamente tendências ou valores discrepantes de dados.

      <Video
        type="wistia"
        id="ikpu24sqjr"
      />

    * Em gráficos de série temporal:

      1. Edite o gráfico e expanda o menu <DNT>**Thresholds**</DNT> .

      2. Selecione <DNT>**+**</DNT> para adicionar um limite e insira um <DNT>**Name**</DNT>.

      3. Insira valores para <DNT>**From**</DNT> e <DNT>**To**</DNT>. Se quiser criar uma única linha, insira o mesmo número para ambos os campos.

      4. Selecione um nível de gravidade. Isso determina a cor da linha ou região.

         <img
           title="Configure chart thresholds in time series charts"
           alt="Screenshot of how to configure thresholds in a time series chart"

    * Em tabelas:

      1. Edite o gráfico e expanda o menu <DNT>**Thresholds**</DNT> .

      2. Selecione <DNT>**+**</DNT> para adicionar um limite e selecione um <DNT>**Column**</DNT>.

      3. Insira valores para <DNT>**From**</DNT> e <DNT>**To**</DNT>. Para definir um limite superior, deixe o campo <DNT>**From**</DNT> vazio. Para definir um limite inferior, deixe o campo <DNT>**To**</DNT> vazio.

Consulte [Importar, exportar, adicionar e compartilhar painéis e gráficos](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/)
