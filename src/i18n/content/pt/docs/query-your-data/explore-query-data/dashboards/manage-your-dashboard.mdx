Acesse qualquer [painel do New Relic](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) para criar ou gerenciar seus gráficos diretamente no menu de gráficos, personalizar o layout do seu dashboard, ajustar os modos de exibição ou exportar seus dados.

Depois de personalizar seu dashboard e construir seus gráficos, use nosso recurso de visualização avançada e ferramentas de exploração de dados para correlacionar e analisar seus dados.

## Personalize seu dashboard [#dash-manage]

Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards**</DNT> para abrir a página do painel.

    Você pode duplicar seu dashboard independentemente dos seus níveis de permissão. Clique no <Icon name="fe-more-horizontal"/> ícone no canto direito e selecione <DNT>**Duplicate dashboard**</DNT>.

### Adicione conteúdo personalizado usando o editor Markdown [#markdown]

Adicione contexto e estilo ao seu dashboard com conteúdo personalizado usando nosso editor Markdown. Com esse recurso, você pode adicionar texto formatado, listas de verificação, imagens, trecho de código e muito mais. Isso torna mais fácil explicar o painel compartilhado, adicionar hiperlinks úteis, criar listas de tarefas e criar qualquer outro conteúdo rico que você precisar.

O editor Markdown contém um painel <DNT>**Markdown**</DNT> , onde você insere seu conteúdo, e um painel <DNT>**Preview**</DNT> , onde você pode visualizá-lo. Para obter mais informações sobre as opções de sintaxe do Markdown, consulte o [guia Markdown do GitHub](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax).

Você também pode editar o conteúdo existente clicando nas reticências <Icon name="fe-more-horizontal"/> ícone em qualquer widget Markdown e selecionando <DNT>**Edit**</DNT>.

## Organize seu painel com páginas [#add-pages]

Você pode usar várias páginas para organizar os dados dashboard em diferentes visualizações. Ao adicionar mais páginas a esse dashboard, você pode acessá-las usando as guias na parte superior da interface dashboard .

<img
  title="Dashboard with multiple pages"
  alt="Dashboard with multiple pages"
  src="/images/dashboards_screenshot-full_multi-page-dashboard.webp;"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Dashboards**</DNT>: Este é um exemplo de dashboard com múltiplas páginas, representado pelas abas na parte superior do dashboard. Você pode adicionar páginas ao painel, copiar páginas existentes e arrastar e soltar as guias da página em novas posições.
</figcaption>

Você pode usar esse recurso para agrupar visualizações dashboard relacionadas. Isso é valioso quando você agrega muitos dados e gráficos relacionados a um projeto, equipe ou assunto específico.

Por exemplo, uma equipe de aplicativos móveis pode criar um dashboard focado no desempenho do aplicativo por país. A primeira página dashboard pode ser uma visão geral do desempenho em todos os países, com outras páginas focadas em países específicos.

Há outro recurso para conectar o dashboard:

* [Crie um widget contendo texto Markdown](#markdown) para adicionar links diretos a páginas de interface ou painel específicos.
* Use [a filtragem de facetas](/docs/dashboards/new-relic-one-dashboards/manage-your-dashboard/filter-new-relic-one-dashboards-facets) para criar links que vinculam e filtram automaticamente outros painéis.
* Use a pesquisa dashboard para encontrar um painel com nome semelhante. Para aproveitar isso, você pode adicionar palavras/frases específicas da equipe ou do projeto aos nomes dashboard .

2. Adicione uma nova página:

   * Selecione

     </DNT>

     para adicionar uma página em branco.

     </DNT>

     .

3. No modo de edição, você pode adicionar widget à nova página, arrastar e soltar guias de página em novos locais e realizar outras [tarefas de edição dashboard ](#dash-manage-charts).

4. Ao terminar, selecione

## Gerencie seus gráficos e conteúdo Markdown [#dash-manage-charts]

A partir de qualquer elemento Markdown, acesse o menu no canto superior direito para editá-lo ou excluí-lo. Em qualquer gráfico, acesse o menu de ação do gráfico no canto superior direito para:

* Expanda seu gráfico para tela inteira.
* [Compartilhe seu gráfico](#dash-export) como uma imagem ou com um link.
* Somente para gráficos de tabela, exporte como um arquivo `.csv` . Você pode importar este arquivo para outros aplicativos como Microsoft Excel ou Planilhas Google para fazer análises adicionais.
* Veja a consulta.
* Copie o gráfico para qualquer dashboard.
* Abra o [console NRQL](/docs/query-your-data/explore-query-data/query-builder/nrql-console/) para ver ou editar a consulta associada ao gráfico.
* Duplique o gráfico.
* Exclua o gráfico.

## Filtre e refine seus gráficos [#dash-search]

Você pode restringir as informações exibidas usando a função de filtragem, que é uma representação visual das condições de consulta:

* Use o

  <DNT>
    **filter bar**
  </DNT>

  para selecionar os valores ou atributos que deseja ver e remova o restante dos elementos dos gráficos.

* Abra o

  <DNT>
    **advanced filter bar**
  </DNT>

  para acessar os operadores booleanos (como `=`, `!=`, `CONTAINS`, `EXCLUDES`, etc.) e adicione condições compostas e complexas para filtrar dados.

Após aplicar um filtro, seu dashboard mostrará apenas os dados associados aos elementos que você selecionou. Um pequeno contador indica o número de filtros aplicados por vez.

Para retornar à visualização padrão, clique na pequena cruz ao lado do filtro para removê-lo.

## Filtre usando a legenda do gráfico [#filter-legend]

Clique em uma legenda em qualquer gráfico com legendas para ver apenas aquela série e remova o restante da série do gráfico. Isso ajuda a isolar os dados que deseja analisar.

Para fazer o contrário, use CMD (no Mac) ou CTRL (no Windows): isso removerá a série selecionada e manterá o restante.

## Filtrar painel usando facetas [#filter-facets]

Se a consulta NRQL de um gráfico contiver uma cláusula `FACET`, você poderá usar o atributo facetado para filtrar o dashboard atual ou outro dashboard relacionado . Para obter mais detalhes, consulte [Filtrar por facetas](/docs/dashboards/new-relic-one-dashboards/manage-your-dashboard/filter-new-relic-one-dashboards-facets).

## Use o seletor de hora para ajustar as configurações de hora [#dash-time-picker]

Por padrão, cada gráfico do dashboard mostra dados do período especificado quando foi criado no [criador de consulta](/docs/introduction-chart-builder). No entanto, você pode usar o [seletor de hora](/docs/new-relic-solutions/get-started/glossary/#time-picker) para alterar o intervalo de tempo dos dados exibidos e definir o mesmo intervalo para todos os gráficos. Isto é particularmente útil durante a resolução de problemas incidentes, se você precisar restringir seus dados para observar o que aconteceu em um período de tempo específico.

<img
  width="50%;"
  title="Time picker on the Dashboard page"
  alt="Time picker on the Dashboard page"
  src="/images/dashboards_screenshot-crop_time-picker.webp;"
/>

A taxa de atualização depende da duração da janela de tempo que você está visualizando. Para obter mais informações e exemplos, consulte [Intervalos de atualização do gráfico](/docs/insights/use-insights-ui/manage-dashboards/insights-chart-refresh-intervals).

Para alterar o intervalo de tempo:

* Selecione uma das opções disponíveis no menu dropdown (do último `5 minutes` ao último `6 months`).
* Personalize o intervalo de tempo com carimbo de data e hora de início e término específico usando o menu personalizado.

Se você não quiser alterar o intervalo de tempo de um gráfico ao usar o seletor de hora:

1. Clique no

   <Icon name="fe-more-horizontal"/>

   ícone e selecione

   <DNT>
     **Edit**
   </DNT>

   .

2. Abra o menu

   <DNT>
     **Dashboard Options**
   </DNT>

   na interface de personalizações.

3. Ative ou desative o botão

   <DNT>
     **Ignore time picker**
   </DNT>

   dependendo da sua necessidade.

## Compartilhe seus dados [#data-share]

Você pode estar acostumado com o fato de qualquer usuário da sua organização ter a capacidade de criar URLs de gráficos em tempo real usando a opção <DNT>**Get chart link**</DNT> nas visualizações compatíveis. Agora, para evitar possíveis exposições de dados, o administrador terá controle sobre quem pode ou não criar URLs de gráficos ao vivo. Essa permissão agora é controlada pelos recursos <DNT>**Live Chart URL (individual)**</DNT> e <DNT>**Live Chart URL (all)**</DNT> , explicados abaixo. Usando-os, você pode atribuir permissões a contas para permitir que usuários individuais criem URLs compartilháveis.

Para minimizar interrupções, os clientes que desejam habilitar esta funcionalidade podem criar funções personalizadas e atribuí-las ao seu usuário antes de 30 de abril de 2024. Após 30 de abril de 2024, esta função estará disponível ao público geral e esses recursos serão ativados.

<Callout variant="important">
  A capacidade de criar URLs de gráficos em tempo real será removida das contas <DNT>HIPAA</DNT> e <DNT>FedRAMP</DNT> em 30 de abril de 2024.
</Callout>

### URL do gráfico ao vivo (individual) [#live-chart-individual]

Use esse recurso para limitar o escopo das ações do usuário às URLs ativas que eles criam:

* Visualizar: o usuário pode ver os URLs do gráfico ao vivo que criou.
* Modificar: o usuário pode criar um URL de gráfico ao vivo.
* Excluir: o usuário pode excluir qualquer URL de gráfico ao vivo que tenha criado.

### URL do gráfico ao vivo (todos) [#live-chart-all]

Use esse recurso para aumentar o escopo das ações do usuário para URLs ativos que foram criados por qualquer usuário na conta:

* Visualizar: o usuário pode ver todos os URLs de gráficos ao vivo criados por qualquer usuário da conta.
* Excluir: o usuário pode excluir qualquer URL de gráfico ao vivo criado por qualquer usuário da conta.

Os novos recursos também se aplicarão à [API de URLs de gráfico ao vivo](/docs/apis/nerdgraph/examples/manage-live-chart-urls-via-api).

## Principais ferramentas visuais [#key-visual-tools]

Dashboard oferece recursos de visualização intuitivos e ferramentas para exploração avançada de dados e rápida resolução de problemas.

<CollapserGroup>
  <Collapser
    id="key-visual-coloring"
    title="Coloração consistente do gráfico"
  >
    Para que você possa ver e correlacionar rapidamente seus dados, as facetas aplicadas a mais de um gráfico em um dashboard têm uma cor de faceta consistente em todos os gráficos.
  </Collapser>

  <Collapser
    id="key-visual-needle"
    title="Agulha correlacionada"
  >
    Quando você passa o mouse sobre um gráfico, a agulha correlacionada se sobrepõe a todos os gráficos ou pontos de dados no dashboard ao mesmo tempo. A dica de ferramenta fornece os pontos de dados relevantes da faceta selecionada, como valores máximos e mínimos em um gráfico de linhas. Também destaca o atributo selecionado em um gráfico de pizza.
  </Collapser>

  <Collapser
    id="key-visual-scrubber"
    title="Depurador de dados"
  >
    O controle deslizante do gráfico ajuda a selecionar um ponto de dados ou faceta em um gráfico quando o gráfico está muito lotado e as facetas estão muito próximas umas das outras.

    Passe o mouse ao longo da agulha para selecionar suavemente as facetas adjacentes e visualizar seus pontos de dados associados. Você também pode iluminar um gráfico muito preenchido desmarcando um ou mais atributos que aparecem na interface do usuário.
  </Collapser>

  <Collapser
    id="key-visual-zoom"
    title="Pincele para ampliar"

    O seletor de hora reflete o novo período exibido no dashboard. Você pode retornar às configurações padrão ou a qualquer outra configuração de horário a qualquer momento.
  </Collapser>
