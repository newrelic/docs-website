Você pode usar [o painel](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) para visualizar e monitorar a integridade do seu sistema, resolver problemas e planejar outras etapas. Aprenda a compartilhar todas essas informações com sua equipe, clientes ou outras partes interessadas com diversas opções de importação e exportação.

## Importe um dashboard como JSON [#import-json]

Você pode importar um dashboard como JSON quando quiser modificar um dashboard pronto para uso no nível do código. Por exemplo, se quiser criar um dashboard personalizado mesclando dois dashboards existentes, você pode modificar o JSON de um dashboard base e importá-lo de volta para New Relic.

Para importar um dashboard como JSON:

1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT>.

2. Abra o JSON de um dashboard existente clicando no ícone <DNT>**...**</DNT> e selecione <DNT>**Copy JSON**</DNT>.

3. Volte para a página <DNT>**Dashboards**</DNT> .

4. No canto superior direito, clique em <DNT>**Import dashboard**</DNT>.

5. Cole seu código JSON.

6. Escolha as configurações de conta e permissão para o dashboard. Você não pode alterar a conta depois de defini-la, mas pode [alterar as permissões](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-settings) a qualquer momento.

7. Clique em <DNT>**Save**</DNT>.

## Exportar um dashboard como PDF [#export-dashboards-pdf]

Para exportar seu dashboard como um arquivo PDF, no canto superior direito do dashboard, clique no ícone <DNT>**...**</DNT> e selecione <DNT>**Export dashboard as PDF**</DNT>.

<img title="Export dashboard as PDF" alt="Screenshot of how to export a dashboard as a PDF file" src="/images/dashboards_screenshot-full_export-pdf.webp" />

<figcaption>
  Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT> e abra um dashboard existente.
</figcaption>

Considere o seguinte ao exportar um dashboard como PDF pela interface:

* [Visualizações personalizadas](/docs/query-your-data/explore-query-data/dashboards/add-custom-visualizations-your-dashboards/) não são suportadas.

Para exportar como PDF ou PNG com opções adicionais, como dimensões personalizadas e substituições de variáveis de modelo, use a [API do NerdGraph](#export-api).

## Exportar dashboards e gráficos via API NerdGraph [#export-api]

Use a API NerdGraph para exportar programaticamente dashboards e widgets de gráficos individuais como imagens de snapshot. A API suporta opções não disponíveis na interface, incluindo dimensões personalizadas, sobrescrições de variáveis de template e filtros de entidade.

### Exportar um dashboard como PDF ou PNG [#api-dashboard-snapshot]

A mutação `dashboardCreateSnapshotUrl` gera um URL de snapshot para uma página completa do dashboard.

<Callout variant="tip">
  O parâmetro `guid` requer o GUID da **página**, não o GUID do dashboard. Um dashboard pode ter várias páginas. Para encontrar os GUIDs de página, use a consulta `entitySearch` ou abra a página do dashboard e selecione <DNT>**Manage JSON**</DNT> no menu <DNT>**...**</DNT>.

  ```graphql
  {
    actor {
      entitySearch(query: "parentId ='YOUR_DASHBOARD_GUID' AND tags.isDashboardPage = 'true'") {
        results {
          entities {
            guid
            name
          }
        }
      }
    }
  }
  ```
</Callout>

**Esquema de mutação:**

```graphql
mutation {
  dashboardCreateSnapshotUrl(
    guid: EntityGuid!
    params: SnapshotUrlInput
  ): String
}

input SnapshotUrlInput {
  timeWindow: SnapshotUrlTimeWindowInput
  display: SnapshotUrlDisplayInput
  variables: [SnapshotUrlVariableInput!]
  filter: String
  format: SnapshotUrlFormat
}

input SnapshotUrlTimeWindowInput {
  beginTime: EpochMilliseconds
  endTime: EpochMilliseconds
  duration: Milliseconds
}

input SnapshotUrlDisplayInput {
  width: Int
  height: Int
}

input SnapshotUrlVariableInput {
  name: String!
  values: [String!]!
}

enum SnapshotUrlFormat {
  PDF
  PNG
}
```

### Obter uma imagem de gráfico [#api-widget-snapshot]

A mutação `dashboardWidgetCreateSnapshotUrl` gera um URL de snapshot PNG para um único widget de gráfico. O widget não precisa existir em um dashboard — você pode defini-lo inline, o que o torna útil para gerar imagens de gráficos programaticamente.

**Esquema de mutação:**

```graphql
mutation {
  dashboardWidgetCreateSnapshotUrl(
    widget: DeclarativeUiWidget!
  ): WidgetSnapshotResult
}

type WidgetSnapshotResult {
  url: String!
}
```

O parâmetro `widget` aceita um `DeclarativeUiWidget` — um objeto JSON com três camadas aninhadas:

```
declarative/widget envelope
└── widget node        ← title and widget-level props
    └── visualization  ← visualization type and configuration
```

```json
{
  "type": "declarative/widget",
  "version": 1,
  "content": {
    "type": "widget",
    "props": {
      "title": "My Chart Title"
    },
    "content": {
      "type": "visualization",
      "id": "viz.line",
      "props": {
        // visualization-specific config (nrqlQueries, colors, thresholds, etc.)
      }
    }
  }
}
```

Se você estiver criando a definição do widget a partir de um dashboard que exportou como JSON, use este mapeamento de campos:

<Callout variant="tip">
  O campo `actor.account.nrql.staticChartUrl` legado no NerdGraph continua disponível, mas não oferece suporte aos recursos de visualização mais recentes, como limites, formatação de cores e tipos de gráficos mais novos. `dashboardWidgetCreateSnapshotUrl` é a abordagem recomendada daqui para frente.
</Callout>

</Callout>

## Exportar um gráfico como CSV [#export-charts-csv]

Para exportar o gráfico da tabela de dashboard como um arquivo CSV, no canto superior direito do gráfico, clique em <DNT>**...**</DNT> e selecione <DNT>**Export as CSV**</DNT>.

<img title="Export chart as CSV" alt="Screenshot of how to export a table chart as a CSV file" src="/images/dashboards_screenshot-crop_export-csv.webp" />

Considere o seguinte ao exportar um gráfico como CSV:

* Os dados sempre são exportados no horário UTC
* Exportar não suporta funções de agregador como `average()`
* A exportação não suporta a cláusula `COMPARE_WITH`
* Em alguns cenários, a interface pode realizar formatação adicional dos dados que não são replicados no CSV exportado, o que pode levar a pequenas discrepâncias (como -1.1E-17 renderizado como 0 na interface, mas não no CSV).
* Não é possível exportar os resultados de consultas entre contas.
