Através de nossa interface você pode criar e compartilhar gráficos e visualizações personalizados e fáceis de usar. Acesse nosso dashboard para coletar e mapear os dados específicos que você deseja ver, da maneira que deseja, de qualquer lugar em nossa plataforma.

## Por que isso importa [#dashboards-features]

Com nosso painel você pode personalizar e entender os dados que coleta. Explore seus dados e correlacione fontes conectadas com gráficos e aprenda rapidamente o estado do seu sistema e aplicativo para uma resolução de problemas mais rápida e eficiente.

Use o painel para:

* Gere insights com visualizações interativas personalizadas e de alta densidade com uma interface de usuário consistente.
* Gráfico de todos os eventos e atributos de todos os lugares da nossa plataforma. Para obter mais informações, consulte [Coleta de dados](/docs/data-apis/understand-data/new-relic-data-types/).
* [Adicione atributo personalizado](/docs/data-apis/custom-data/custom-events/apm-report-custom-events-attributes/#steps) ou [envie tipos de eventos personalizados](/docs/data-apis/ingest-apis/event-api/introduction-event-api/#workflow) para a maioria dos eventos para entender melhor o seu negócio e ver detalhes específicos sobre como seus clientes interagem com sua plataforma, como visualizações de páginas, hospedagem de transações, etc.
* [Gerencie seus gráficos e painel](#get-started) facilmente usando nossos menus CRUD de acesso rápido e opções de edição.
* Explore e contextualize dados com [dicas de ferramentas avançadas e funções de zoom](#key-visual-tools) para monitor o que seus sistemas estão fazendo em tempo real.
* Pesquise em seu painel por atributo e métrica.
* [Relate dados personalizados](/docs/data-apis/custom-data/get-custom-data-from-any-source/) para a New Relic e seu painel de qualquer fonte.
* Compartilhe o dashboard com partes interessadas fora da sua organização New Relic por meio de [URLs dashboard ao vivo acessíveis publicamente](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#dashboard).

## Comece com o painel [#get-started]

Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT> e clique em <DNT>**Dashboards**</DNT>.

No índice do painel, você pode visualizar todos os aplicativos de painel e dados associados à sua conta New Relic. Você também pode visualizar o [popular painel pré-construído](/docs/query-your-data/explore-query-data/dashboards/prebuilt-dashboards/).

<img title="Dashboards index" alt="Dashboards index" src="/images/dashboards_screenshot-full_dashboard-index.webp" />

<figcaption>
  Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT> para abrir o índice dashboard .
</figcaption>

Para cada dashboard, o índice exibe as seguintes informações:

* [Status favorito](#dashboards-favorite), indicado por uma estrela
* Nome: O nome do dashboard
* Conta: a conta à qual o dashboard pertence
* Criado por: O usuário que criou o dashboard
* Última edição: quando o dashboard foi modificado pela última vez
* Criado em: quando o dashboard foi criado

## Permissões dashboard [#dashboards-permissions]

* **Associação de conta**: cada dashboard é um recurso que pertence à conta na qual foi criado. Essa associação não muda. Se a conta à qual pertence for cancelada, qualquer painel dentro dela não estará mais acessível.
* **Propriedade do usuário**: O ID do usuário que cria um dashboard é seu proprietário. A propriedade determina quem tem o direito de gerenciar as permissões do dashboard.

Por exemplo, se o ID do usuário proprietário for excluído, o dashboard não será removido da conta à qual pertence. Ele persiste com suas últimas permissões conhecidas, mas como o proprietário se foi, essas permissões não podem mais ser gerenciadas ou alteradas. A solução para isso seria [duplicar o dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-clone) e o usuário que fizer isso será o proprietário da duplicata. O dashboard que foi duplicado pode então [ser excluído](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-delete).

Dashboard tem três tipos de permissões:

* <DNT>**Edit - everyone in account**</DNT>: Todos os usuários têm direitos totais no dashboard.
* <DNT>**Read-only - everyone in account**</DNT>: Todos os usuários podem ver o dashboard, mas somente você tem direitos totais para trabalhar com o dashboard. Outros usuários podem acessar o dashboard , mas não podem editá-lo ou excluí-lo, embora possam duplicá-lo.
* <DNT>**Private**</DNT>: Somente você pode ver o dashboard. Tudo, exceto os metadados, está oculto.

Ao [criar um dashboard](#dashboards-create-new) usando o botão <DNT>**Create a dashboard**</DNT> ou [duplicando](#dashboards-clone) outro dashboard, ele terá direitos <DNT>**Edit - everyone in account**</DNT> por padrão. Posteriormente, você pode alterá-los no [menu de configurações](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-settings) ao acessar o painel.

## Crie e gerencie painel via API [#nerdgraph]

Você pode usar [nossa API NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-dashboards) para criar e gerenciar painéis. Nossa API dá acesso a mais recursos do que a interface: por exemplo, você pode criar dashboard com dados de múltiplas contas.

Você também pode [adicionar tags com NerdGraph](/docs/new-relic-solutions/new-relic-one/core-concepts/use-tags-help-organize-find-your-data/#add-tags). Você pode filtrar seu painel usando tags, o que pode ser útil para identificar usuários, contas, locais e muito mais.
