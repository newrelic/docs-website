Navegue por todos os seus dados visualmente usando <DNT>**Metrics and events**</DNT> e sem precisar saber como escrever [consultas no formato NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language).

Precisa realizar uma pesquisa mais detalhada? Leia como [consultar seus dados](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/). Se ainda não o fez, crie sua conta New Relic gratuita abaixo para começar a monitorar seus dados hoje mesmo.

## Por que isso importa [#why-matters]

Você pesquisa o estado de seus sistemas? Você precisa planejar recursos, identificar e responder a incidentes ou solucionar problemas de comportamento? métrica e evento facilita a identificação, busca e visualização dos dados que você procura através de menus visuais, sem nunca usar NRQL ou construção de consulta.

Com <DNT>**Metrics and events**</DNT> você pode acessar todos os dados armazenados no banco de dados New Relic (NRDB) de forma rápida e intuitiva. Você pode então selecionar <DNT>**Dimensions**</DNT> para facetar por dimensões específicas desses dados.

Outras coisas que métrica e evento ajudam você a fazer:

* Veja dados de diferentes pontos de vista: desde dados brutos até diferentes visualizações que fornecem insights sobre evolução, distribuição e muito mais.
* Faça uma busca detalhada nos dados usando filtros.
* Adicione suas pesquisas a um [dashboard](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards).
* Entenda como funciona [o NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) : métrica e evento mostram como são construídas as consultas.

## Tipos de dados que você pode explorar [#data-types]

No contexto de <DNT>**Metrics and events**</DNT>, `event` refere-se a todos os objetos armazenados no banco de dados New Relic (NRDB). métrica e evento permitem explorar:

* Métrica:

  * [Métrica dimensional](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) (o tipo de dados `Metric` ).

  * [métrica de fração de tempo data](/docs/data-apis/understand-data/new-relic-data-types/#timeslice-data):

    <InlinePopover type="apm"/>

    agente, agente móvel e o agente

    <InlinePopover type="browser"/>

    reportam este tipo de dados. Para explorar este tipo de dados, você deve escolher um monitor de entidade por um desses agentes, e então você verá essas opções.

* Eventos: o [tipo de dados do evento](/docs/data-apis/understand-data/new-relic-data-types/#events-new-relic).

* log: o tipo de dados `Log` (que você também pode explorar por meio de nossa [interface de log](/docs/logs/ui-data/use-logs-ui)).

* Traces: o tipo de dados `Span` (que você também pode explorar por meio de nossa [interfacetrace ](/docs/distributed-tracing/ui-data/query-distributed-trace-data)).

## Explore seus dados [#explore-data]

Para acessar o explorador de dados, vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > Metrics and events**</DNT>.

Métrica e evento consistem em:

* Seção de escopo na parte superior. Você pode selecionar entre `metrics` ou `events` e `account`. Se selecionar métrica, poderá filtrar por entidade.
* Área de navegação de dados à esquerda. Se você escolher `metrics`, as opções serão `metric` e `dimensions`. Se você escolher `events`, as opções serão `event type`, `plot` e `dimensions`.
* Área de trabalho. Esta área mostra o resultado de suas escolhas.

Para usar métrica e evento:

1. Defina o escopo. Selecione o tipo de dados (métrica ou evento), a conta e a entidade.

2. Selecione o intervalo de tempo usando o [seletor de hora](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker).

3. Utilize os blocos à esquerda para navegar pelos dados disponíveis e construir sua pesquisa. Você só pode selecionar um elemento por bloco. Os blocos são pesquisáveis.

## Visualize sua métrica personalizada após criá-la [#visualize-metric]

Siga estas etapas para visualizar sua métrica personalizada por meio do métrica explorer:

1. Acesse o explorador de dados, vá para <DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > Metrics and events**</DNT>.

2. Escolha sua entidade.

3. Clique no link <DNT>**See timeslice metrics for `your entity`**</DNT> .

## Casos de uso [#uses-cases]

Veja os exemplos a seguir para saber como e quando usar métrica e evento.

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="use-case-validate-data"
    title="Descubra seus novos dados"
  >
    <DNT>
      **I've just connected new instrumentation and want to see if new data is available.**
    </DNT>

    1. Selecione a conta e o evento ou métrica que está gerando os novos dados.
    2. Use as diferentes ferramentas em métricas e eventos para brincar com os novos dados que se tornaram disponíveis: veja os dados brutos desse evento ou métrica como uma tabela, modele-os como uma lista ou clique para vê-los plotados como um gráfico .
    3. Após selecionar um evento ou métrica, descubra o formato dos dados em suas dimensões. Guiado pela cardinalidade, você pode ver os diferentes pontos de vista de quaisquer dados.
    4. Encontrou algo relevante? Salve-o em um dashboard ou compartilhe-o com um colega.
  </Collapser>

  <Collapser
    className="freq-link"
    id="use-case-new-data"
    title="Valide seus dados"
  >
    <DNT>
      **I changed a custom event/metric and need to check if this change has been successful.**
    </DNT>

    1. Em métricas e eventos, selecione a conta, tipo de dados e evento/métrica que você fez alterações.
    2. Verifique se a entidade está reportando dados e se todos os atributo estão sendo plotados.
    3. Encontre o atributo no qual você fez alterações e verifique se a atualização foi bem-sucedida.
  </Collapser>

  <Collapser
    className="freq-link"
    id="use-case-troubleshooting"
    title="Resolução de problemas questões de dados"
  >
    <DNT>
      **I know there's something off with an event/metric from an alert or dashboard. I need to know the root cause about the event/metric/attribute behavior.**
    </DNT>

    1. Em métrica e evento, utilize os menus para selecionar o evento ou métrica que não está se comportando conforme o esperado e deixe a métrica e o evento traçarem esse gráfico.
    2. A partir daí, você pode detalhar as dimensões desses dados e filtrar pelos atributos que são relevantes.
    3. Você também pode ver esses dados de diferentes perspectivas: sua distribuição, classificação de valores ou evolução ao longo do tempo.
    4. Encontrou algo relevante? Salve-o em um dashboard ou compartilhe-o com um colega.
  </Collapser>
</CollapserGroup>
