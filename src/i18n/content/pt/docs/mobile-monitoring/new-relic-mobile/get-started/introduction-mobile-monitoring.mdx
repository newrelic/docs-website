Os recursos <InlinePopover type="mobile"/>do New Relic fornecem visibilidade mais profunda sobre o desempenho e a resolução de problemas de travamento do seu aplicativo móvel Android, iOS ou híbrido. Você pode usar o monitoramento de Mobile para melhorar a experiência do usuário do seu aplicativo ou examinar o desempenho do HTTP e da rede para colaborar de forma mais eficaz com suas equipes backend .

Depois de seguir nosso fácil fluxo de instalação, você poderá configurar ainda mais o monitoramento de Mobile para se adequar ao seu ambiente móvel.

## Identifique oportunidades para acelerar o desempenho do aplicativo [#speed-performance]

* Crie [um painel](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) para consultar, visualizar e compartilhar dados de desempenho.
* Identifique [exceções tratadas](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/introduction-mobile-handled-exceptions) para agilizar o fluxo de trabalho.
* Veja tendências com sessões, dispositivos, localizações geográficas, sistema operacional, operadoras, solicitações e respostas, etc. com [perfis de travamento](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes/#profiles) e [perfis HTTP](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/http-errors-page/#error-profiles).
* Configurar [notificação de alerta](/docs/mobile-monitoring/new-relic-mobile/getting-started/alert-information-new-relic-mobile) para tempo de execução, erros, etc.
* Defina [atributo personalizado e evento](/docs/insights/insights-data-sources/custom-events/insert-custom-events-attributes-mobile-data) para obter detalhes adicionais e específicos sobre os dados móveis mais importantes para sua organização.

## Monitor problemas backend [#backend]

Você pode usar as [páginas de rede](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/analyze-network-requests-using-mobilerequest-event-data/) em nosso monitoramento de UI móvel para identificar problemas que surgem por meio de solicitações HTTP, erros e outros problemas de rede. Você também pode [consultar seus dados](/docs/nrql/nrql-examples/nrql-query-examples-mobile-monitoring/) e criar <InlinePopover type="dashboards"/>personalizados para compartilhar com equipes relevantes, como desenvolvedores de API backend .
