      ## Sintaxe [#syntax]

      ### Java [#java]

      ```java
      NewRelic.withApplicationVersion(string $appVersion)
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.withApplicationVersion(appVersion: String?)
      ```

      ## Descrição [#description]

      Por padrão, o agente Android do New Relic obtém a versão do aplicativo de `AndroidManifest.xml` ou `build.gradle`. Este método permite substituir esse valor e definir uma versão do aplicativo passando a versão do aplicativo. A versão do aplicativo é exibida como um dropdown na interface do New Relic e você pode usá-la para classificar a interação.

      ## Exemplo [#example]

      ### Java [#java]

      ```java
      NewRelic.withApplicationToken("").withApplicationVersion("1.23.4").start(this.getApplication());
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.withApplicationToken("").withApplicationVersion("1.23.4").start(applicationContext)
      ```

      ## Sintaxe [#syntax]

      ### Objetivo-c [#objc]

      ```objectivec
      setApplicationVersion:(NSString *_Nonnull)versionString;
      ```

      ### Swift [#swift]

      ```swift
      NewRelic.setApplicationVersion:(NSString*)versionString;
      ```

      ## Descrição [#description]

      Por padrão, o New Relic usa `CFBundleShortVersionString` ao relatar o número da versão. Mas você pode substituir o número de compilação relatado chamando este método (antes de chamar `startWithApplicationToken`).
