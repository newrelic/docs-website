A página de interface <DoNotTranslate>**HTTP requests**</DoNotTranslate> ajuda você a entender melhor as solicitações HTTP associadas ao seu aplicativo móvel e como essas chamadas de rede afetam o desempenho.

## Veja a página de solicitações HTTP [#find-page]

Para visualizar a página <DoNotTranslate>**HTTP requests**</DoNotTranslate> :

1. Vá para

   <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Network > HTTP requests**</DoNotTranslate>

   .

2. Use nossas [funções de UI padrão](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) para procurar tendências nos gráficos de análise HTTP.

3. atributo específico de solicitação e resposta do destino [agrupando, classificando e filtrando](#groups-sort-filter) os dados.

## Compreenda os dados da solicitação HTTP [#understand-data]

Aqui estão alguns lugares para encontrar os dados de solicitação HTTP mais importantes:

<CollapserGroup>
  <Collapser
    id="response-time-chart"
    title="Gráfico de tempo de resposta"
  >
    O gráfico de tempo de resposta mostra o desempenho das chamadas de rede do seu aplicativo em todo o percentil. Use-o para comparar o tempo médio de resposta com o 1º, 50º e 99º percentil. percentil permite filtrar valores discrepantes que podem estar aumentando o tempo médio de resposta do que o esperado.
  </Collapser>

  <Collapser
    id="errors-network-failures-chart"
    title="Gráfico de erros HTTP e falhas de rede"
  >
    Este gráfico mostra as chamadas de rede malsucedidas que seu aplicativo está enfrentando. Selecione o título do gráfico para ir para a [página<DoNotTranslate>**HTTP errors**</DoNotTranslate> ](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/network-errors-http-error-network-failure-analysis)e obter mais detalhes sobre os erros e falhas.
  </Collapser>

  <Collapser
    id="total-requests-chart"
    title="Total de solicitações"
  >
    Classifique por <DoNotTranslate>**Total requests**</DoNotTranslate> para identificar quais solicitações de rede estão sendo usadas com mais frequência. A razão pela qual isso pode ser útil é porque suas chamadas de rede mais lentas podem ser usadas com pouca frequência, enquanto as solicitações usadas com mais frequência podem ser mais dignas de otimização, mesmo que não sejam as mais lentas.
  </Collapser>
</CollapserGroup>

Para obter uma descrição da página de interface <DoNotTranslate>**HTTP requests**</DoNotTranslate> não empresarial, consulte [Solicitações HTTP herdadas](#non-enterprise-http-requests).

## Visualize e compartilhe dados de solicitação HTTP [#insights]

Para visualizar qualquer gráfico <DoNotTranslate>**HTTP requests**</DoNotTranslate> nos insights:

1. Selecione

   <Icon name="fe-more-horizontal"/>

   para qualquer gráfico.

2. Selecione

   <DoNotTranslate>**View query > View in Insights**</DoNotTranslate>

   .

3. Opcional: adicione os dados a um dashboard ou [compartilhe-os usando um link permanente](/docs/data-analysis/user-interface-functions/share-your-data/permalink).

4. Para se aprofundar nos dados da sua solicitação, consulte [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) evento e [atributo](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes).
