## Descubra a página de erros HTTP [#ui-interactions]

Na página <DNT>**HTTP errors**</DNT> , investigue solicitações HTTP e falhas de rede:

* Use qualquer função de página padrão para procurar tendências em

  <DNT>
    **Errors and failures**
  </DNT>

  gráficos.

* Tipos específicos de erros e falhas do destino, agrupando, classificando e filtrando os dados.

* Encontre anomalias nos erros de sua solicitação com perfis de erro HTTP.

* Selecione um erro ou falha para visualizar detalhes dele.

Você também pode definir [alertas NRQL](/docs/alerts/new-relic-alerts/defining-conditions/create-alert-conditions-nrql-queries) focados em tipos de erros para seus serviços críticos ou [consultar os dados do seu aplicativo](#insights).

## Perfis de erro HTTP [#error-profiles]

Os perfis de erro fornecem detalhes visuais sobre diferenças significativas na frequência de diferentes valores para eventos de erro HTTP. Para cada [atributo](/docs/new-relic-solutions/get-started/glossary/#attribute), o perfil de erro inclui:

* Um mapa de calor mostrando como o atributo do erro é distribuído para valores que mais se desviam
* Um rótulo comparando a distribuição do atributo de erro com a de outros erros

<img
  title="HTTP error profiles"
  alt="Screenshot of http error profiles"
  src="/images/mobile_screenshot-full_http-error-profiles.webp"
/>

Isso ajuda você a eliminar mais suposições ao resolver os erros de HTTP do seu aplicativo móvel. Você pode determinar mais facilmente se ignora o erro com segurança ou se deve tentar resolver o erro com uma nova implantação, alteração de código, comunicação com clientes ou outras ações.

## Faça uma busca detalhada em um único erro [#error-summary]

Para visualizar detalhes sobre um erro ou falha, selecione o URL da solicitação na lista dos 20 principais erros. Na página de resumo de erros, você pode visualizar as informações da versão, o atributo da solicitação e o corpo da resposta e obter detalhes dos tipos de erros para o URL da solicitação.

* [`MobileRequest`](/docs/nrql/nrql-examples/nrql-query-examples-mobile-monitoring/#mobilerequest-examples) evento e atributo

## Obtenha detalhes trace [#trace-details]

O monitoramento de Mobile irá capturar os detalhes da resposta das solicitações HTTP que retornam um código de status de nível 400 ou 500. Além disso, a mensagem de erro gerada a partir de aplicativos Android incluirá um stack trace.

Para visualizar detalhes sobre um trace de erro na página <DNT>**Errors**</DNT>, selecione o link do URL de solicitação. A partir daqui você pode:

* Veja o corpo da resposta.
* Compartilhe os detalhes do erro com outras pessoas por e-mail.

O gráfico de erros também aparece na página [Resumo](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/mobile-apps-summary-page/) do aplicativo móvel selecionado.

## Solucionar erros desconhecidos ou erros de URL [#troubleshoot-unknown-url]

O agente mobile mantém uma lista de tipos de exceção. Em alguns casos, as exceções personalizadas lançadas pelo aplicativo ficam fora desta lista. Quando isso acontecer, `Unknown` poderá aparecer na página do mobile <DNT>**Errors**</DNT> .

Se você encontrar `Unknown` em sua lista de erros e precisar de ajuda para pesquisar quais tipos de exceção estão sendo ignorados, obtenha suporte em [support.newrelic.com](https://support.newrelic.com "O link abre em uma nova janela").
