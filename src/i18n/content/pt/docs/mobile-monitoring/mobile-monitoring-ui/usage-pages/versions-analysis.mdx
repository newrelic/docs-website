O recurso <InlinePopover type="mobile"/>inclui uma análise comparativa de adoção e desempenho entre versões do seu aplicativo, incluindo as principais versões por:

* Tempo de interação
* Sessões ativas
* Taxa de erros

A página <DoNotTranslate>**Versions**</DoNotTranslate> também inclui uma tabela que compara cada versão por data de criação, memória média, CPU média, sessões médias por minuto e solicitações médias por minuto (RPM) por aplicativo ativo. Você também pode detalhar detalhes adicionais sobre uma versão específica.

A página <DoNotTranslate>**Versions**</DoNotTranslate> fornece uma lista de todas as versões do seu aplicativo que foram detectadas, além de informações gerais sobre todas as versões ativas nos últimos sete dias. Para visualizar a análise comparativa:

1. Vá para

   <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Usage > Versions**</DoNotTranslate>

   .

2. Para selecionar o período, use

   <DoNotTranslate>[**time picker**](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)</DoNotTranslate>

   abaixo da barra de menu do New Relic.

3. Opcional: selecione as

   <DoNotTranslate>**Sort by**</DoNotTranslate>

   opções.

4. Para visualizar detalhes apenas de um

   <DoNotTranslate>**version**</DoNotTranslate>

   específico, selecione seu nome.

A página <DoNotTranslate>**Versions**</DoNotTranslate> fornece uma lista de todas as versões do seu aplicativo que estiveram ativas no período selecionado. Para detalhar informações detalhadas, use qualquer [recurso de interface de plataforma](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) aplicável.

A página de detalhes fornece informações adicionais sobre como a versão selecionada se compara a uma versão de referência (uma versão recente ou popular) e à média de outras versões do seu aplicativo. As séries temporais mostram a comparação entre taxa de erros, tempo de resposta, sessões ativas e uso de memória.

Para sair da página de detalhes, selecione o botão <DoNotTranslate>**Close**</DoNotTranslate> (<DoNotTranslate>**X**</DoNotTranslate>).

## Como os números de versão são obtidos [#how-obtain-version]

A forma como o monitoramento de Mobile obtém o número da versão varia de acordo com a plataforma:

<CollapserGroup>
  <Collapser
    id="android-ver-num"
    title="Android"
  >
    O [agente Android](/docs/mobile-monitoring/new-relic-mobile-android/get-started/android-installation-configuration) obtém as informações de versão da propriedade `android:versionName` no manifesto.
  </Collapser>

  <Collapser
    id="ios-ver-num"
    title="iOS"
  >
    O agente iOS usa as propriedades `CFBundleShortVersionString` e `CFBundleVersion` para obter a versão do aplicativo. O agente acessa essas propriedades por meio API do iOS. Ele não os obtém lendo o arquivo `info.plist` .
  </Collapser>
</CollapserGroup>
