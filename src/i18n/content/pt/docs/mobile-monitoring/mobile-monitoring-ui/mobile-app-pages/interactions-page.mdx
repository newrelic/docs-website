No monitoramento de Mobile, a interação é uma coleção de métodos de instrumentos agrupados por UIViewControllers (iOS) ou Atividades (Android). Use nossa página de interação para rastrear a interação mais cara e lenta e investigar o que está causando o trace mais lento.

## Instrumento interação [#instrumentation]

Interação são automaticamente instrumento. Para detalhes sobre quais classes são auto-instrumento, veja:

* [(Android) Métodos de instrumento automaticamente](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/mobile-sdk-api-guide/#method-list)
* [(iOS) Métodos de instrumento automático](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/mobile-sdk-api-guide/#method-list-ios)

Também oferecemos uma instrumentação manual de interação utilizando os seguintes métodos:

* [`startInteraction`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/start-interaction/)
* [`stopInteraction`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/stop-interaction/)

Se você utilizar os métodos `startInteraction` e `stopInteraction`, o instrumento interação não aparecerá na página de interação, mas ainda poderá ser encontrada uma consulta NRQL, como:

```sql
query SELECT name FROM Mobile SINCE 7 DAYS AGO
```

## Veja a página de interação [#interactions-page]

A página de interação possui gráficos que detalham as 20 interações mais caras e uma lista de rastreamento mais lento. Isso inclui código em execução no thread principal e código em execução em threads de trabalho.

Para visualizar a página de interação:

1. Vá para

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile >**
   </DNT>

   (selecione um aplicativo)

   <DNT>
     **> More views > Interactions**
   </DNT>

   .

2. Para detalhar informações detalhadas, selecione qualquer interação ou use nosso [recurso de interface de plataforma](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) aplicável.

## Aprofunde-se nos detalhes da interação [#breakdown-table]

Para mostrar a lista completa de todos os métodos do instrumento: Na interação selecionada, selecione o link <DNT>**Breakdown table**</DNT> . Caso utilize instrumentação personalizada para alguma das métricas, a categoria da classe e o nome do método indicarão <DNT>**custom**</DNT>.

Para cada método de rastreamento, a tabela detalhada de interação mostra:

* Tipo de thread (primeiro plano ou plano de fundo)
* Nome da classe e do método
* Porcentagem do tempo total de execução gasto neste método
* Número médio de chamadas para este método por interação
* Tempo médio geral de execução de cada chamada para este método

## Analise a linha do tempo do traço da interação [#interactions-timeline]

Cada traço do período de interação registra um único traço até que uma nova interação se inicie. Para visualizar o cronograma trace do seu aplicativo móvel:

1. Vá para

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > App > Interactions**
   </DNT>

   .

2. Na tabela

   <DNT>
     **Slowest traces**
   </DNT>

   , selecione o link

   <DNT>
     **Activity**
   </DNT>

   ou

   <DNT>
     **Duration**
   </DNT>

   para o trace.

A linha do tempo trace também fornece métricas do dispositivo para ajudar você a obter insights sobre os dados no nível do dispositivo do seu aplicativo. A métrica de uso do sistema mostra como seu aplicativo está interagindo com os dispositivos do usuário final, incluindo banco de dados, CPU e memória.

Aqui estão algumas ações adicionais que você pode realizar:

* Para visualizar a métrica de uso no dispositivo, passe o mouse sobre a linha do tempo.

* Para detalhar informações detalhadas, selecione qualquer local na linha do tempo.

* Para ajudar a fornecer contexto para os dados, clique em

  <DNT>
    **Previous interaction**
  </DNT>

  .
