A página Resumo fornece um instantâneo operacional do seu aplicativo móvel. Todos os gráficos, tabelas e estatísticas mostram dados para um [período de tempo](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) e versão do aplicativo especificados.

A página Resumo captura cinco métricas principais de desempenho:

<table>
  <thead>
    <tr>
      <th width={200}>
        Métrica
      </th>

      <th>
        Descrição
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Taxa de falha do aplicativo
      </td>

      <td>
        Número de sessões travadas ao longo do tempo como uma porcentagem de todas as sessões, divididas por versão do aplicativo. A porcentagem média de sessões travadas, o número total de falhas e a contagem de usuários únicos afetados por essas falhas durante o intervalo de tempo são mostrados no canto superior direito.
      </td>
    </tr>

    <tr>
      <td>
        Lançamento do aplicativo
      </td>

      <td>
        Número de monitoramento de lançamento de sessão do aplicativo durante a janela de tempo.

        O lançamento de um aplicativo é definido pelo período de tempo entre o carregamento do aplicativo em uma tela e o momento em que o aplicativo é enviado para segundo plano.
      </td>
    </tr>

    <tr>
      <td>
        Erros HTTP (falhas de rede)
      </td>

      <td>
        Número de solicitações de rede que resultam em `http status code error` (`400` ou superior) como porcentagem de todas as solicitações concluídas e o número de `failed network calls` (por exemplo, uma falha de conexão) como porcentagem de solicitações concluídas.
      </td>
    </tr>

    <tr>
      <td>
        Tempo de resposta HTTP
      </td>

      <td>
        tempo médio de resposta de todas as solicitações http concluídas de cada um dos cinco principais hosts com os quais seu aplicativo se comunica. Os cinco principais hosts são calculados com base na contagem de solicitações concluídas para cada host.
      </td>
    </tr>

    <tr>
      <td>
        Interação frequente
      </td>

      <td>
        Cinco interações de usuário mais comumente executadas em seu aplicativo.
      </td>
    </tr>
  </tbody>
</table>

## Ver a página Resumo [#summary-page]

1. Acesse [one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile.
2. Selecione seu aplicativo móvel. A página Resumo é carregada por padrão.

## (Recomendado) Configurar monitoramento de alerta móvel [#alerts]

Recomendamos a criação de alertas para os principais problemas móveis, como solicitações de rede lentas e aumentos na taxa de travamento. Para configurar facilmente esses alertas na página Resumo:

1. Clique em <DNT>**Add recommended conditions**</DNT>.

   <img
     width="80%;"
     style={{align: "left"}}
     title="Mobile Summary page"
     alt="Screenshot of the New Relic mobile Summary page"
     src="/images/mobile_screenshot-full_mobile-alert-conditions.webp"
   />


2. Escolha uma série de condições de alerta recomendadas.

   <img
     width="70%;"
     style={{align: "left"}}
     title="Mobile recommended alert conditions"
     alt="Mobile recommended alert conditions"
     src="/images/mobile_screenshot-crop_mobile-add-alert.webp"
   />

3. Clique em <DNT>**Next**</DNT>.

4. Selecione uma política para ser notificado. Você pode selecionar uma política existente ou [criar uma nova política](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name).

5. Clique em <DNT>**Activate conditions**</DNT>.
