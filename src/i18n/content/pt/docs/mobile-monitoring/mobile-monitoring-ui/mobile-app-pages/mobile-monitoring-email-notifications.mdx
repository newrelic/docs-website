O <InlinePopover type="mobile"/>da New Relic fornece vários relatórios enviados por e-mail, incluindo:

* [Liberar notificação](#version-trends) (também conhecido como e-mails de tendência de versão)
* [E-mails de relatório de falhas](#crash-reports)

## Receba e-mails sobre o desempenho da nova versão [#version-trends]

O monitoramento de Mobile da New Relic fornece uma página [<DNT>**Release versions**</DNT>](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/release-versions-page) . Você também pode ativar e-mails de tendência de versão (também conhecidos como notificação de lançamento). Esses relatórios fornecem uma ferramenta de comunicação conveniente para compartilhar, rastrear e revisar o progresso, a adoção e outras tendências do seu aplicativo. Cada e-mail inclui um link para a interface do usuário <DNT>**Version trends**</DNT> para ver mais detalhes.

Enviamos relatórios por e-mail em intervalos regulares:

### Editar configurações de e-mail de tendência de versão [#version-trend-ui]

A maneira como você define as configurações de e-mail de tendência de versão depende do [modelo de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) em que você está.

<Callout variant="tip">
  Para verificar rapidamente seu modelo de usuário: acesse o [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) e clique em <DNT>**User preferences**</DNT>. Se você vir as configurações de e-mail móvel, você está em nosso modelo de usuário original.
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-version-trends"
    title="E-mails de versão (modelo de usuário original)"
  >
    Para ativar ou desativar os relatórios de tendências de versão por e-mail para todos os seus aplicativos móveis de monitor:

    1. Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select a mobile app) > App > Version trends**</DNT>.

    2. Selecione o link <DNT>**Enable version trends email notifications**</DNT> e edite suas preferências.

       Para configurar relatórios por e-mail para um aplicativo móvel específico, acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT> e edite suas preferências.
  </Collapser>

  <Collapser
    id="new-relic-one-user-version-trends"
    title="E-mails de versão (modelo de usuário mais recente)"
  >
    Para configurar os relatórios por e-mail que você recebe para um aplicativo móvel específico, acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT> e edite as configurações de e-mail de notificação de lançamento.
  </Collapser>
</CollapserGroup>

## E-mails de relatório de falhas [#crash-reports]

Quando seu aplicativo móvel apresenta um novo tipo de falha, você deseja saber o que deu errado para poder evitar esse tipo de erro no futuro. Com e-mails de relatório de falhas, você é notificado quando uma falha única é detectada.

A forma como você define as configurações de e-mail do relatório de falhas depende do [modelo de usuário](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) em que você está.

<Callout variant="tip">
  Para verificar rapidamente seu modelo de usuário: acesse o [menu do usuário](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) e clique em <DNT>**User preferences**</DNT>. Se você vir as configurações de e-mail móvel, você está em nosso modelo de usuário original.
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-crash-emails"
    title="E-mails de falha (modelo de usuário original)"
  >
    Para definir as configurações de e-mail de relatório de falhas para sua conta em vários aplicativos: Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**</DNT>.

    Para definir as configurações de e-mail de relatório de falhas para um aplicativo móvel de monitor específico:

    1. Vá para

       <DNT>
         **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**
       </DNT>

       .

    2. Na seção

       <DNT>
         **Mobile crash reporting**
       </DNT>

       , passe o mouse sobre o nome de um aplicativo móvel e você verá

       <DNT>
         **Edit application settings**
       </DNT>

       . Selecione-o.

    3. Isso leva você à [página de interface do usuário](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)

       <DNT>
         [**Settings**](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)
       </DNT>

       [](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)do aplicativo. Ative a caixa de seleção

       <DNT>
         **Enable crash report emails for this application**
       </DNT>

       .
  </Collapser>

  <Collapser
    id="new-relic-one-user-crash-emails"
    title="E-mails de falha (modelo de usuário mais recente)"
  >
    Para configurar se você receberá relatórios de erros por e-mail para um aplicativo móvel específico, acesse <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Settings > Application**</DNT> e edite as configurações de e-mail do relatório de erros.
  </Collapser>
</CollapserGroup>
