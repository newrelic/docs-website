O Model Context Protocol (MCP) é uma forma padrão para um modelo de IA ou agente receber informações relevantes (contexto) sobre um sistema ou ambiente específico. Esse contexto ajuda a IA a entender a situação atual, executar tarefas com mais precisão e fornecer respostas mais úteis. Historicamente, conectar o agente de IA a várias fontes de dados, como a plataforma de observabilidade, significava construir uma integração personalizada, muitas vezes complexa, para cada agente de IA se comunicar com cada plataforma. Essa abordagem fragmentada leva a custos significativos de desenvolvimento e manutenção, uma forma de &quot;dispersão de integração&quot; para insights orientados por IA.

O servidor New Relic AI MCP aborda diretamente esse desafio. Trata-se da implementação desse protocolo pela New Relic, que atua como uma ponte única e centralizada. Em vez de criar várias integrações individuais, seu agente de IA, dentro das ferramentas de desenvolvimento compatíveis, agora tem uma maneira consistente de se conectar e obter contexto rico e ferramentas poderosas da plataforma de observabilidade da New Relic.

Você pode fazer perguntas e investigar dados usando linguagem natural diretamente em seu ambiente de desenvolvimento de IA. O servidor MCP traduz suas requests, recupera métricas ou logs relevantes e fornece respostas e resumos mais claros.

<Callout title="Restrições de conformidade" variant="important">

O uso deste recurso de pré-visualização pública com contas regulamentadas pelo FedRAMP ou HIPAA é estritamente proibido.
</Callout>

## Como isso te ajuda

O servidor MCP oferece benefícios práticos para equipes de engenharia e operações, auxiliando você e sua IA das seguintes maneiras:

* **Integração direta:** Acesse a plataforma da New Relic e alguns recursos de IA diretamente do seu ambiente de desenvolvimento. Isso evita a troca entre diferentes ferramentas.
* **Resolução de problemas mais rápida:** Encontre e corrija problemas rapidamente, analise logs de erros e determine as causas prováveis com insights fornecidos por IA.
* **Operações automatizadas:** Execute tarefas como verificar o status de alertas, gerar relatórios de incidentes e analisar como a implantação afeta seus sistemas.
* **Consulta de dados simples:** formule perguntas em inglês simples e o servidor as converte para a linguagem de consulta New Relic (NRQL) para obter resultados, mesmo que você não esteja familiarizado com NRQL.

## Ferramentas de desenvolvimento de IA suportadas [#supported-tools]

O New Relic MCP funciona com estes ambientes de desenvolvimento de IA:

* **Claude Code:** Interface de linha de comando para Claude
* **Claude Desktop:** aplicativo de desktop para desenvolvimento interativo de IA
* **VS Code:** Ambiente de desenvolvimento integrado com suporte a MCP
* **Windsurf:** Plataforma de desenvolvimento de IA baseada em nuvem
* **Gemini CLI:** Interface de linha de comando para Gemini

Cada ambiente oferece vantagens únicas, dependendo das suas preferências de fluxo de trabalho. Consulte nosso [guia de configuração](/docs/agentic-ai/mcp/setup) para obter instruções específicas de configuração da plataforma.

## Pré-requisitos [#prerequisites]

Para usar o New Relic MCP, você precisa de:

* Uma conta New Relic com acesso à API.
* `Node.js` instalado (necessário para alguns métodos de autenticação)
* Um dos clientes com suporte do MCP instalou

<Callout title="Participação no grupo e funções" variant="important">
  Para acessar o servidor MCP, você precisa ser membro de um grupo que tenha uma das seguintes [funções organizacionais](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#role-scopes) específicas atribuídas: `Organization Read Only`, `Organization Manager` ou `Organization Product Admin`. Certifique-se de que esta função esteja ativa antes de tentar o acesso. Se o seu grupo não tiver atualmente uma dessas funções, entre em contato com o administrador da sua organização que tem a `Authentication Domain Manager` para que ela seja concedida.
</Callout>

### Opções de autenticação

* **usuário chave de API**:

  1. Faça login na New Relic.
  2. Selecione **menu do usuário -&gt; API keys**.
  3. Crie uma nova chave de usuário ou copie uma existente.
  4. Formato da chave: `NRAK-XXXXXXXXXXXXXXXXXXXXX`.

* **OAuth**: New Relic fornece um endpoint OAuth pré-configurado:

  * ID do cliente: `pUWGgnjsQ0bydqCbavTPpw==`
  * URL de autorização: `https://login.newrelic.com/login`
  * URL do token: `https://mcp.newrelic.com/oauth2/token` (EUA) e `https://mcp.eu.newrelic.com/oauth2/token` (UE)
  * Âmbito de aplicação: `["openid"]`
