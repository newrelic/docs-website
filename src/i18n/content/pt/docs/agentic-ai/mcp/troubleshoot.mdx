## O fluxo OAuth não está iniciando.

* Certifique-se de que seu navegador possa acessar `localhost`.
* Verifique as configurações do firewall.
* Verifique se suas ferramentas podem acessar URLs `OAuth`.

## chave de API não está funcionando

* Verifique o formato da chave (começa com `NRAK-`).
* Verifique as permissões de chave no New Relic.
* Verifique se a chave não expirou.

## Servidor MCP não detectado

* Reinicie o aplicativo completamente.
* Verifique a validade da sintaxe `JSON`.
* Verifique os caminhos e permissões dos arquivos.

## problemas de proxy remoto mcp

* Certifique-se de que `Node.js` esteja instalado.
* Verifique a sua ligação à internet.
* Verifique se o npx pode acessar o pacote `mcp-remote`.

## Acesso negado para grupos personalizados

### Problema

Você não consegue se conectar ou autenticar com sucesso no servidor New Relic AI MCP e recebe a seguinte mensagem de erro: `Missing required capabilities to access MCP server. Please ensure you have the necessary permissions.` Isso normalmente ocorre se você for membro de um grupo de usuários personalizado sem nenhuma permissão em nível organizacional.

### Causa

Your group lacks an organization-scoped role with permission to read the MCP server. Access to this organization-wide feature must be explicitly granted to your group via an organization-scoped role.

### Resolução

O administrador da sua organização que possui a função `Authentication Domain Manager` deve implementar uma das duas soluções a seguir para conceder acesso a você:

1. Grant your group a role with permission to read the MCP server:

   * O administrador deve adicionar uma das seguintes funções ao grupo personalizado ao qual você pertence: `Organization Read Only`, `Organization Manager` ou `Organization Product admin`.
   * The administrator can create a [custom organization-scoped role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) and assign it to a group you belong to via an [organization-coped grant](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#access-grants). This custom role must include the `New Relic MCP Server` read permission.

2. Add you to an authorized group:

   * The administrator must add you to an existing user group that already has an organization-scoped role with permission to read the MCP server.

Para obter informações detalhadas sobre conceitos de gerenciamento de usuários, funções e concessões de organização, consulte a [documentação de gerenciamento de usuáriosNew Relic ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts). Se a sua organização utiliza o Sistema de Gerenciamento de Identidade entre Domínios (SCIM) para o gerenciamento de usuários, esses ajustes de grupo e função devem ser realizados no seu Provedor de Identidade (IdP) externo.

## Mensagem de erro

* **Conexão recusada**: Verifique o URL e a conectividade de rede das suas ferramentas.
* **Autenticação necessária**: Verifique se a chave de API ainda está ativa e foi inserida corretamente ou complete o fluxo `OAuth`.
* **Cliente inválido**: problema de configuração `OAuth`, tente o proxy `mcp-remote`.
* **`Time.UnmarshalJSON`**: Use o proxy `mcp-remote` para `OAuth`.
