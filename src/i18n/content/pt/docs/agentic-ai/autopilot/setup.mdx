## Pré-requisitos [#prerequisites]

Antes de configurar <DNT>Autopilot</DNT>, certifique-se de atender aos seguintes requisitos:

### Permissões de administrador (para configuração)

Para dar aos usuários permissão para usar <DNT>Autopilot</DNT>, um usuário deve ser um **Organization Manager** \[Gerente da Organização] e um **Authentication Domain Manage**r \[Gerente de Domínio de Autenticação].

<Callout variant="tip">
  Se você não tiver essas permissões, verá a mensagem: &quot;Isso parece um trabalho para um administrador.&quot; Entre em contato com o gerente da sua organização para ativar o agente para você.
</Callout>

### Permissões de usuário (para acesso)

Para usar <DNT>Autopilot</DNT>, os usuários devem ser [usuários full ou core](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/).

Para dar aos usuários permissão para usar <DNT>Autopilot</DNT>, um administrador ou gerente de domínio de autenticação deve atribuir aos usuários uma função que inclua essas capacidades no nível da organização:

* **<DNT>Autopilot</DNT>** - Leitura, outros (Perguntar, Adicionar à automação)
* **New Relic MCP Server** - Leitura

### Opt-ins de pré-visualização

Para acessar <DNT>Autopilot</DNT>, você deve aderir aos seguintes testes de public preview:

* Autopilot
* [Servidor MCP](/docs/agentic-ai/mcp/overview/)
* [Destinos entre contas](/docs/apis/nerdgraph/examples/cross-account-destinations/overview/)

<Callout variant="tip">
  Destinos entre contas são necessários apenas se você quiser configurar alertas com <DNT>Autopilot</DNT>. Sem esta visualização, <DNT>Autopilot</DNT> permanecerá parcialmente configurado.
</Callout>

## Configurar o Autopilot [#setup]

Use o fluxo de integração para configurar automaticamente:

* Uma identidade de sistema para representar o agente
* Permissões necessárias para uso básico
* Um destino de alerta

<Callout variant="important">
  **O que é uma identidade de sistema?** Uma identidade de sistema é uma identidade não humana que permite que o agente se autentique diretamente em outros serviços sem exigir uma conta de usuário. As identidades do sistema têm escopo em uma única organização da New Relic.
</Callout>

Para integrar <DNT>Autopilot</DNT>:

1. Vá para <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>

2. No canto superior direito, selecione <DNT>**Ask AI**</DNT>, depois selecione a guia <DNT>**Configure**</DNT> (ícone de engrenagem).

3. Em <DNT>**Agents**</DNT>, selecione <DNT>**Autopilot**</DNT>.

4. No campo <DNT>**Agent name**</DNT>, insira um nome para o seu agente. O sistema adiciona automaticamente <DNT>`Autopilot -`</DNT> como prefixo e aplica esse nome aos componentes de gerenciamento de identidade e acesso do sistema.

5. No menu suspenso, selecione as contas que você deseja que o agente analise.

   <Callout variant="important">
     Adicionar mais contas permite que o agente identifique melhor as conexões e apresente insights em toda a sua stack. No entanto, a análise do agente será visível para qualquer usuário com acesso de interação ao <DNT>Autopilot</DNT>, independentemente de seu acesso direto a essas contas.
   </Callout>

6. Revise o resumo de **Permissions**. O sistema concede a <DNT>Autopilot</DNT> acesso padrão somente leitura às suas contas selecionadas.

   <Callout variant="important">
     Os usuários administradores terão a capacidade de modificar as contas e permissões fornecidas ao agente. Uma redução nas concessões da função padrão de somente leitura limitará os dados de telemetria que <DNT>Autopilot</DNT> pode acessar, o que pode resultar em análises incompletas, anomalias não detectadas ou capacidades degradadas de identificação de causa raiz.
   </Callout>

7. Clique em <DNT>**Add Agent**</DNT>. O sistema leva alguns instantes para concluir a configuração.

<Callout variant="tip">
  A criação do destino de alerta não atribui automaticamente condições de alerta ou políticas a <DNT>Autopilot</DNT>. Consulte [Integrar com alertas](#alerts) para configurar isso.
</Callout>

## Integrar com alertas [#alerts]

Conecte <DNT>Autopilot</DNT> aos seus fluxos de trabalho de alerta para que os insights de <DNT>Autopilot</DNT> sejam anexados automaticamente a cada alerta disparado ou problema.

### Configurar workflows de alertas

<Callout variant="tip">
  **Recomendação de abordagem em fases:** comece com um pequeno subconjunto de condições de alerta ou políticas para validar o desempenho, confirme se o agente funciona bem para o seu ambiente e, em seguida, expanda de forma mais ampla.
</Callout>

Para configurar <DNT>Autopilot</DNT> com alertas:

1. Vá para <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Issues &amp; Activity &gt; Notification Workflows &gt; Add a workflow**</DNT>.
2. Na configuração do seu workflow, na seção <DNT>**Notify**</DNT>, selecione <DNT>**AI agent**</DNT>.
3. Selecione o destino <DNT>Autopilot</DNT> criado durante o onboarding e clique em <DNT>**Save message**</DNT>.

Depois de configurado, ao abrir um alerta ou problema disparado recentemente, você verá automaticamente o resumo do <DNT>Autopilot</DNT> para esse item.
