Um bom site – você sabe quando o vê, certo? Você nunca percebe nenhum tempo de carregamento, ele está disponível sempre que você precisa e, é claro, não há erros em lugar nenhum. Mas _medir_ o desempenho do seu site é complicado, e melhorá-lo é ainda mais complicado.

Este tutorial mostra como usar o New Relic para melhorar a experiência do seu site. Para começar, você instrumentará seu site. Instrumento significa adicionar algum código JavaScript que mede a experiência real do usuário final para cada visita. A partir daí, você aprenderá como avaliar o desempenho do seu site hoje, onde pode melhorá-lo e como pode ficar de olho no futuro.

## Capture dados de desempenho sobre seu site [#capture]

Antes de melhorar o desempenho do seu site, você precisa de dados para tomar decisões informadas sobre as necessidades do seu site. O monitoramento da New Relic sempre começa com um <DNT>**integration**</DNT> ou um trecho de código que coleta dados de uma tecnologia relevante e os reporta à New Relic.

Para monitor o front-end do seu site, você vai querer usar nossa integração <DNT>**browser monitoring agent**</DNT>. É um pequeno trecho de JavaScript que você incluirá em seu HTML `<head>`, muito semelhante ao Google Analytics ou outro script.

## Analise o desempenho do site [#data]

Depois de instrumentar seu site, você poderá ter uma visão geral do desempenho dele com diferentes tipos de dados.

* <DNT>
    **Core web vitals**
  </DNT>

  . São métricas que fornecem insights sobre a experiência do usuário do seu site. Pontuações baixas geralmente indicam uma experiência ruim, o que pode deteriorar seu relacionamento com o usuário final.

* <DNT>
    **High latency**
  </DNT>

  . Isso pode frustrar o seu usuário e fazer com que ele pare de usar o seu site.

* <DNT>
    **JavaScript errors**
  </DNT>

  . Isso pode causar vários problemas em seu site, e qualquer um deles pode impactar negativamente a experiência de front-end de seus clientes. A New Relic dedica uma página

  <DNT>
    **JS errors**
  </DNT>

  inteira para seus aplicativos integrados.

* <DNT>
    **Benchmarks**
  </DNT>

  . O desempenho do site pode cair abaixo dos níveis aceitáveis sem necessariamente causar erros graves. Você pode criar uma referência de nível de serviço para ajudar a monitorar o desempenho de seus sites e identificar problemas antes que se tornem problemas.
