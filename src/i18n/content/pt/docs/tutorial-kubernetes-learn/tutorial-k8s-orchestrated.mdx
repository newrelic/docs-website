Os componentes orquestrados consistem em implantação inteira que gira o pod para cima e para baixo conforme necessário. Em um sistema Kubernetes, a implantação individual possui configuração específica. Por exemplo, uma implantação pode executar quatro instâncias de um aplicativo em um contêiner individual. A implantação ativará o pod que contém esse aplicativo até atingir essa cota. Se um pod falhasse, ele geraria um novo para continuar a atingir o número especificado.

Os problemas surgem quando você tem dezenas ou centenas de implantação, cada uma com configurações que você talvez nem lembre.

As etapas a seguir guiam você através de uma estratégia geral para monitor sua implantação:

<Steps>
  <Step>
    ## Navegue até o dashboard de visão geral

    Vá para <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Kubernetes > Overview Dashboard**</DNT>.
  </Step>

  <Step>
    ## Triagem sua implantação

    Dê uma olhada no bloco de implantação não íntegro e no bloco de contagem de implantação.

    Compare os dois para medir o percentual de implantação insalubre no seu cluster. Implantação insalubre possui vagem faltante ou indisponível. Isso geralmente significa que a implantação não conseguiu ativar esses pods.

    Vamos dar uma olhada mais de perto.

    <img
      title="tutorialOverviewdashboard"
      alt="The main overview dashboard for the kubernetes capability"
      src="/images/tutorial_screenshot-full_overview-deployments.webp"
    />
  </Step>

  <Step>
    ### Identificar pod pendente e com falha

    Role um pouco para baixo e encontre a tabela <DNT>**Pending and Failed Pods**</DNT> . Esta tabela mostrará todos os pods que falharam ou estão pendentes por qualquer motivo. É normal que o pod falhe até certo ponto, dependendo da integridade baseline do seu sistema.

    O que você está procurando é um pod que falha repetidamente. Role o gráfico conforme necessário.

    <img
      title="tutorialOverviewFailed"
      alt="The main overview dashboard for the kubernetes capability focusing on the failed pods section"
      src="/images/tutorial_screenshot-crop_failed-pods.webp"
    />

    Depois de identificar qual pod falha regularmente, solucione os problemas das configurações de implantação desses pods.
  </Step>
</Steps>
