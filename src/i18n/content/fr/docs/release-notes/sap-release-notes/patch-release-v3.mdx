## Mises à jour de ce correctif

SP12 est un ensemble de notes New Relic traitant des correctifs de bugs et des nouvelles fonctionnalités :

### NR 1344

Le nouveau paramètre `AIF_TABLES` pour `NR_SAP_AIF_ERROR` permet de récupérer des messages d&apos;erreur à partir de tables autres que `/AIF/STD_IDX_TBL`.

### NR 12-02

Corrige des problèmes et ajoute de nouvelles fonctionnalités pour les fournisseurs de données basés sur des API :

* Correction du vidage de code dans `/NEWR/API_DATA_COLLECTOR` lorsque la destination de l&apos;API n&apos;est pas accessible
* Correction d&apos;une défaillance de ressource lors de l&apos;extraction des données du paysage CALM
* Correction d&apos;une défaillance de ressource lors de l&apos;extraction BTP Message Processing Log
* Ajout d&apos;une validation pour l&apos;ID de service (LMSID) configuré comme CALL ID pour tous les fournisseurs de données du groupe de monitoring CALM
* Ajout d&apos;une vérification de la validité du certificat avant d&apos;appeler l&apos;API Data Collection

### NR 12-03

Implémente un délai de collecte et une fenêtre de collecte de données fixe pour le gestionnaire de collecte et le test par ID d&apos;appel dans le registre.

### NR 12-05

Implémente les améliorations suivantes :

* Prise en charge de la pagination pour les fournisseurs de données dans le groupe de monitoring CALM
* Nouveau paramètre de programme permettant au programme de vérification RFC de contrôler une seule destination RFC
* Configuration pour forcer le test du fournisseur de données pour un seul ID d&apos;appel uniquement

Corrige les problèmes suivants :

* ID d&apos;appel vide sur la sortie du gestionnaire de collecte
* Problème de validation des identifiants pour `NR_CF_AUDIT_EVENTS`

### NR 12-06

Le nouveau fournisseur de données `NR_SAP_ABAP_WS_COMPLETE` collecte les événements de services web ABAP.

### Corrections de bugs supplémentaires

En plus des notes ci-dessus, le SP12 corrige également les problèmes suivants :

* Correction du problème d&apos;horodatage avec les modules fonction du fournisseur de données de récupération de table `/NEWR/RFC_EVENTS_COLLECTOR` et `/NEWR/RFC_EVENTS_COLLECTOR2`
* Ajout de l&apos;ID de transaction aux événements IDoc pour faciliter la liaison aux enregistrements de trace

## Mises à jour manuelles de la configuration

### Ajouter un nouveau paramètre à NR\_SAP\_AIF\_ERROR

Pour configurer le nouveau paramètre `AIF_TABLES` :

1. Appliquez le transport inclus au Central Monitoring System.
2. Accédez à la transaction `/n/NEWR/REGISTRY`.
3. Sélectionnez le fournisseur de données `NR_SAP_AIF_ERROR` et cliquez sur **Configure**.
4. Ajoutez le paramètre `AIF_TABLES` et saisissez le nom de la table spécifié dans `/AIF/T_IDX_TBL` pour la combinaison espace de noms/interface/version configurée.

<Callout variant="tip">
  Si `AIF_TABLES` est laissé vide ou non spécifié, la table d&apos;index est par défaut la table d&apos;index unique standard `/AIF/STD_IDX_TBL`.
</Callout>

### Configurer le délai de collecte et la fenêtre de collecte fixe

Deux nouveaux champs de configuration ont été ajoutés au Registre des fournisseurs de données pour combler les manques de données causés par les limites de collecte flottantes et la latence de la base de données.

**Nouveaux champs dans le registre des fournisseurs de données :**

* **DATA\_COLLECTION\_DELAY :** Spécifie un délai intentionnel en minutes à soustraire des horodatages de collecte. Ce tampon garantit que les données arrivant tardivement issues des mises à jour de la base de données sont capturées avant la fermeture de la fenêtre de collecte.
* **FIXED\_SCHEDULES :** Indicateur pour activer des périodes de collecte fixes. Lorsqu&apos;il est défini sur `X`, les secondes sont supprimées des horodatages de collecte pour créer des fenêtres de collecte cohérentes et prévisibles.

**Comment ça marche :**

**Horaires fixes** (`FIXED_SCHEDULES = 'X'`) :

* Les horodatages de collecte sont normalisés en supprimant les secondes.
* Exemple : 12:12:23 devient 12:12:00.
* Crée des limites de collecte cohérentes et alignées sur toutes les collectes.

**Délai de collecte de données** (`DATA_COLLECTION_DELAY > 0`) :

* Soustrait le délai spécifié en minutes des horodatages de début et de fin de la collecte de données.
* Décale la fenêtre de collecte vers l&apos;arrière pour capturer les mises à jour de base de données manquées.
* Exemple : un délai de 2 minutes modifie 10:00:00 - 10:05:00 en 09:58:00 - 10:03:00.

### Nouveau paramètre sur le programme de contrôle RFC

Un nouveau paramètre **Destination RFC** a été ajouté au programme de contrôle RFC `/NEWR/REGISTRY_RFC_CHECK`. Cela permet au programme de vérifier une destination RFC individuelle. Si laissé vide, toutes les destinations RFC configurées seront vérifiées.

## Nouveaux fournisseurs de données pour les systèmes de gestion ABAP

### NR\_SAP\_ABAP\_WS\_COMPLETE

Traite les en-têtes de messages de services web et les enregistrements de données (tags) en parallèle sur les systèmes monitorés, sans limite d&apos;enregistrement.

**configuration:**

* **Type :** Événement
* **Communicateur :** N/A
* **Fonction de fournisseur de données :** `/NEWR/WS_MESSAGES_ARFC`
* **Fournisseur de données RFC activé :** Oui
* **Fonction de finalisation :** `/NEWR/WS_EVENTS_FINISHER`
* **Groupe de monitoring :** `SAP_APPL`

### Aperçu fonctionnel

Ce fournisseur de données est préactivé. Paramètres requis :

* **`ADAPTER_TYPE`**

  * **Objectif :** Spécifier le type d&apos;adaptateur à sélectionner dans l&apos;en-tête de message du service Web ABAP. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Champs de table séparés par `;`, peut être sur plusieurs lignes

* **`DESTINATION`**

  * **Objectif :** Spécifier la destination RFC
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Destination RFC

* **`DPC_COLL_TIME`**

  * **Objectif :** Déterminer l&apos;intervalle de temps
  * **Signe :** Inclusif uniquement
  * **Option :** BT
  * **Valeurs :** Horodatage de début et horodatage de fin générés par le système

* **`IB_IF_NAME`**

  * **Objectif :** Nom de l&apos;interface réceptrice à sélectionner dans l&apos;en-tête de message du service Web ABAP. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Noms des interfaces du récepteur séparés par `;`, peut être sur plusieurs lignes

* **`IB_PARTY`**

  * **Objectif :** Partie réceptrice à sélectionner depuis l&apos;en-tête de message de service Web ABAP. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Partie réceptrice séparée par `;`, peut être sur plusieurs lignes

* **`JOIN_CONDITION`**

  * **Objectif :** Spécifier les enregistrements de jointure pour la table `/NEWR/JOIN_COND` pour un traitement ultérieur
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Titres d&apos;affichage séparés par `;`, peuvent être sur plusieurs lignes

* **`NR_ACCID`**

  * **Objectif :** Spécifier l&apos;ID de compte New Relic
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** ID de compte New Relic

* **`OB_IF_NAME`**

  * **Objectif :** Nom de l&apos;interface émettrice à sélectionner dans l&apos;en-tête de message de service Web ABAP. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Noms des interfaces de l&apos;expéditeur séparés par `;`, peut être sur plusieurs lignes

* **`OB_PARTY`**

  * **Objectif :** Partie émettrice à sélectionner depuis l&apos;en-tête de message de service Web ABAP. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Parties émettrices séparées par `;`, peut être sur plusieurs lignes

* **`SETTING_SWITCH`**

  * **Objectif :** Spécifier les options utilisées par le traitement distribué

  * **Signe :** Inclusif uniquement

  * **Option :** EQ

  * **Valeurs :** Plusieurs options séparées par `;` pour contrôler le processus distribué :

    * **HEADONLY=**

      * `SPACE` (par défaut) : Fournit les données d&apos;en-tête et de TAG (si les noms de TAG sont spécifiés dans le paramètre `TAG_LIST`).
      * `X`: Fournit uniquement les données d&apos;en-tête même si des noms de TAG sont spécifiés dans le paramètre `TAG_LIST`.

    * **FIRSTONLY=**

      * `X` (par défaut) : Le fournisseur de données extrait les données exclusivement de la première occurrence de chaque TAG listé dans le paramètre `TAG_LIST`.
      * `SPACE`: Le fournisseur de données extrait les données de toutes les occurrences de chaque TAG listé dans le paramètre `TAG_LIST`.

    * **FREE%=nn :** Spécifiez le pourcentage de processus de travail disponibles dans le système de monitoring pour déterminer la limite souhaitée de processus de travail libres :

      * Si le nombre de processus de travail libres dans le système de monitoring actuel est supérieur ou égal à cette Limite de processus de travail libres souhaitée, poursuivez le traitement de distribution.

      * Sinon, attendez 1 seconde et répétez la vérification ci-dessus jusqu&apos;à ce que :

        * Le traitement de la distribution peut se poursuivre, ou
        * Le temps d&apos;attente total dépasse le Temps d&apos;attente maximum spécifié (voir `MWAIT#=nn`).

      * La valeur par défaut pour cette option est 30(%).

    * **MINFT#=nn :** Spécifiez la limite minimale de processus de travail libres :

      * Si le nombre de processus de travail libres souhaités comptabilisés par le système de monitoring tombe en dessous de ce seuil minimal, le système se mettra en pause pendant 1 seconde avant de tenter de poursuivre le traitement.
      * La valeur par défaut de cette option est 3.

    * **MWAIT#=nn :** Spécifiez le temps d&apos;attente maximal (en secondes).

      * Si le temps d&apos;attente total (voir `FREE%=nn`) dépasse le maximum spécifié, le processus de distribution sera annulé pour préserver les ressources système.
      * La valeur par défaut de cette option est 60.

    * **MREC#=nnnnn :** Spécifiez la limite du nombre d&apos;enregistrements d&apos;**en-tête de message de service Web** récupérés à chaque appel du **module fonction** `/SDF/IMA_DC_WS_ABAP`.
      * La valeur par défaut de cette option est 10000.

    * **JSEND#=nnnn :** Spécifiez la limite du nombre d&apos;enregistrements d&apos;attributs de résultat envoyés au serveur New Relic à chaque fois.
      * La valeur par défaut pour cette option est 5000.

    * **USER=user :** Spécifiez le nom d&apos;utilisateur RFC pour chaque système monitoré afin de récupérer le nombre total de processus de travail actuellement utilisés par ce système. Le `user` est l&apos;ID utilisateur SAP.

    * **DPWP#=nn :** Spécifiez la **limite maximale des processus de travail**.

      * Lors du traitement distribué : si le nombre de processus de travail libres dans le système de monitoring actuel est inférieur ou égal à cette limite maximale, continuer le traitement distribué.
      * Sinon, attendez 1 seconde, puis bouclez pour vérifier à nouveau.
      * Continuez cette boucle jusqu&apos;à ce que le traitement distribué puisse se poursuivre ou que le temps d&apos;attente total dépasse le Temps d&apos;attente maximum spécifié (`MWAIT#=nn`).
      * Remarque : cette vérification n&apos;est exécutée que lorsque le paramètre USER est spécifié avec des valeurs non initiales.
      * La valeur par défaut pour cette option est 8.

* **`STATE_GROUP`**

  * **Objectif :** Groupe de statuts de traitement à sélectionner dans l&apos;en-tête de message de service Web ABAP. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Groupes de statuts de traitement séparés par `;`, peut être sur plusieurs lignes

* **`SYS_ID`**

  * **Objectif :** Déterminer l&apos;ID du système
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** ID système

* **`TAG_LIST`**

  * **Objectif :** Spécifier les noms de TAG à sélectionner dans les données de message de service Web ABAP et à afficher en sortie dans les données de message. Option de filtre du module de fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Champs de table séparés par `;`, peut être sur plusieurs lignes

* **`USERNAME`**

  * **Objectif :** Noms d&apos;utilisateur ayant créé le message à sélectionner dans l&apos;en-tête de message de service Web ABAP (`CREATED_BY`). Option de filtre du module fonction `/SDF/IMA_DC_WS_ABAP`.
  * **Signe :** Inclusif uniquement
  * **Option :** EQ
  * **Valeurs :** Noms d&apos;utilisateur séparés par `;`, peut être sur plusieurs lignes
