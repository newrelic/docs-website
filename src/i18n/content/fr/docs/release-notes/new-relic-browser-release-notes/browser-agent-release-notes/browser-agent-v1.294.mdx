## v1.294.0

### Caractéristiques

#### Récolte précoce

L&apos;agent pourra désormais déclencher des récoltes précoces si certains critères sont remplis. Cela permettra de garantir que davantage de charge est conservée et que les fuites de mémoire sont évitées. L&apos;agent lancera une récolte anticipée lorsque la taille du tampon de fonctionnalité atteindra 16 Ko, sinon il continuera à récolter toutes les 30 secondes comme auparavant. Ce nouveau comportement s&apos;appliquera à l&apos;événement suivant :

* AjaxRequest
* BrowserInteraction
* BrowserPerformance
* événement personnalisé
* enregistrer
* PageAction
* PageViewTiming
* SessionReplay
* Action de l&apos;utilisateur

Un effort distinct sera mené pour permettre une récolte précoce des métriques de tranches de temps, des erreurs JavaScript et des traces de session qui utilisent d&apos;autres mécanismes de stockage dans l&apos;agent.

### Débogage

#### signaler l&apos;URL précédente vide comme non définie

L&apos;attribut `previousUrl` sur l&apos;événement BrowserInteraction a été signalé comme une chaîne vide dans les cas où document.referrer est vide. Cela a entraîné l&apos;application du regroupement d&apos;URL sur une chaîne vide, ce qui a conduit à l&apos;ajout de groupes sans signification à previousUrl. `previousUrl` est désormais signalé comme `undefined` pour permettre au regroupement d&apos;URL d&apos;ignorer les URL vides.
