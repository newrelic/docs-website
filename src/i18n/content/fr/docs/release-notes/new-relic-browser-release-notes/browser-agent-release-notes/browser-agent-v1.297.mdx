## v1.297.0

### Caractéristiques

#### Prévenir les récoltes précoces en période de réessai

Empêcher l&apos;agent de récolter prématurément lorsqu&apos;il est limité en débit. Si le service d&apos;ingestion renvoie une erreur 429, l&apos;agent remplacera ses données et tentera à nouveau la récolte plus tard. Pendant ce temps, il ne sera plus possible de récolter de manière anticipée les nouvelles données entrantes, jusqu&apos;à ce qu&apos;elles soient récoltées avec succès.

#### supprimer l&apos;attribut méta newrelic

Supprimez le signalement de l&apos;attribut méta \_\_newrelic des frais de relecture sortants. Cette mesure est menée pour aider à économiser des octets facturables pour les clients et à réduire le coût d&apos;utilisation des fonctionnalités Session Replay .

### Débogage

#### Corriger les exceptions pour les corps nuls résultant de récoltes vides

Une modification a été apportée pour garantir que toutes les charges nettoyées garantissent une sortie valide. Auparavant, une chaîne vide passant par le processus de nettoyage du corps pouvait générer une erreur car le mécanisme renvoyait le corps comme nul. Désormais, les chaînes vides passeront sans être touchées, ce qui permettra d&apos;appliquer avec succès le traitement ultérieur.
