## v1.288.0

### Caractéristiques

#### Empêcher le stockage des données de session après l&apos;expiration de la session

Il peut y avoir des cas limites dans lesquels certaines données Session Replay/trace de session peuvent être stockées même si la session de l&apos;agent a expiré. Ajoutez une instrumentation pour détecter de tels événements et évitez de les stocker pour le moment.

#### Introduire l&apos;attribut isFirstOfSession sur l&apos;événement InitialPageLoad

Pour aider à prendre en charge les parcours utilisateur, l&apos;attribut `isFirstOfSession` personnalisé sera ajouté à `BrowserInteraction` événement de type chargement de page initial si la page est la première d&amp;apos;une nouvelle session.

### Débogage

#### Réinitialiser `notified` lorsque la relecture arrête l&amp;apos;enregistrement

Résout un problème où l&apos;événement JavaScriptError manquait de l&apos;attribut `hasReplay` après la reprise de l&amp;apos;enregistrement de session (par exemple, lorsque l&amp;apos;utilisateur clique sur un autre onglet/qu&amp;apos;il devient inactif puis y revient).

#### Résoudre la condition de concurrence lors de l&apos;abandon du logging

Si la fonctionnalité de logging s&apos;interrompait avant que l&apos;agent n&apos;ait configuré le buffer d&apos;événements, elle pourrait générer des erreurs. Ce problème a été résolu en vérifiant la présence du buffer d&apos;événements avant d&apos;exécuter des méthodes sur celui-ci.
