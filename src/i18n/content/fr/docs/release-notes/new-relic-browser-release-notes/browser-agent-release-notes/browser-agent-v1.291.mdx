## v1.291.1

### Débogage

#### Empêcher ST de conserver les références d&apos;événement en mémoire en cas d&apos;interruption

Sortir les entrées `Event` que l&apos;agent conserve dans la trace après son annulation. Cela résout un cas connu de fuite de mémoire provoquée par l’ajout continu de ces références à un ensemble qui n’est jamais effacé.

#### Nettoyer BrowserPerformance entryName pour les ressources

L&apos;attribut `entryName` du type de ressource d&apos;événements `BrowserPerformance` aura désormais une URL plus propre, supprimant le fragment hacher des URL signalées.

#### Ajouter des protections pour addToTrace

Événements créés avec l&apos;API addToTrace, qui ont un horodatage UNIX invalide, émettent un avertissement et retournent sans créer l&apos;événement.
