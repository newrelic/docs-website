Une nouvelle version de l&apos;agent est sortie. Suivez les procédures standard pour [mettre à jour l&apos;agent d&apos;infrastructure](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recommande de mettre à jour l&apos;agent régulièrement et au minimum tous les 3 mois. À partir de cette sortie, la plus ancienne version prise en charge est [l&apos;agent d&apos;infrastructure 1.65.3](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1653/).

## Modifié

* Correction d&apos;une condition de panique causée par une opération de double fermeture. [2270](https://github.com/newrelic/infrastructure-agent/pull/2270)

* Correction du comportement de liaison du serveur local pour utiliser strictement localhost par défaut. [2271](https://github.com/newrelic/infrastructure-agent/pull/2271)

* Mises à jour des dépendances :

  * Mise à jour de `newrelic/nri-flex` vers `v1.18.8`. [#2278](https://github.com/newrelic/infrastructure-agent/pull/2278)
  * Mise à jour de `newrelic/nri-winservices` vers `v1.5.3`. [#2279](https://github.com/newrelic/infrastructure-agent/pull/2279)
  * Mise à jour de `newrelic/nri-prometheus` vers `v2.29.4`. [#2276](https://github.com/newrelic/infrastructure-agent/pull/2276)
  * Mise à jour de `newrelic/nri-docker` vers `v2.8.1`. [#2277](https://github.com/newrelic/infrastructure-agent/pull/2277)
  * Mise à jour de la dépendance de l&apos;agent principal vers `v1.26.5`. [#2275](https://github.com/newrelic/infrastructure-agent/pull/2275)
  * Mise à jour de `newrelic-fluent-bit-output` vers `v3.7.0` (Linux) pour résoudre les vulnérabilités de sécurité CVE-2026-39820, CVE-2026-42499, CVE-2026-33814, CVE-2026-42501, CVE-2026-33811 et CVE-2026-42504. [#2280](https://github.com/newrelic/infrastructure-agent/pull/2278)
