Une nouvelle version de l&apos;agent a été publiée. Suivez les procédures standard pour [mettre à jour l&apos;agent Infrastructure](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recommande de mettre à jour l&apos;agent régulièrement et au minimum tous les 3 mois. À partir de cette version, la plus ancienne version prise en charge est [l&apos;agent Infrastructure 1.63.0](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1630/).

### Ajouté

* feat:Prise en charge du compte de service à moindres privilèges pour l&apos;agent d&apos;infrastructure NewRelic sous Windows ([#2152](https://github.com/newrelic/infrastructure-agent/pull/2152))

  Cette version introduit une prise en charge complète de l&apos;exécution de l&apos;agent d&apos;infrastructure sous des comptes de service à moindres privilèges sur Windows :

  * La mise à niveau préserve le compte de service : Lors de la mise à niveau, le programme d&apos;installation détecte et préserve désormais le compte de connexion de service existant (par exemple, un domaine personnalisé ou un compte gMSA) au lieu de le réinitialiser sur LocalSystem. Les permissions sur le répertoire de données de l&apos;application sont automatiquement réattribuées au compte conservé.

  * Une nouvelle installation utilise `LocalSystem` par défaut : les nouvelles installations continuent d&apos;enregistrer le service sous `LocalSystem`. Utilisez `set-service-account.ps1` (nouveau script compagnon) post-installation pour passer à un compte à moindres privilèges. Prend en charge les comptes locaux, les comptes de domaine standards et les comptes de service administrés de groupe (gMSA). Prend également en charge les identifiants stockés dans le Gestionnaire d&apos;informations d&apos;identification Windows.

  * Sécurité du rollback : Un fichier marqueur est écrit avant la création du service afin que l&apos;action de rollback MSI ne supprime que les services créés par l&apos;exécution en cours — elle ne supprimera pas un service préexistant lors d&apos;un échec de mise à niveau.

  * Logging de débogage : l&apos;installation écrit désormais un log de débogage horodaté (`newrelic_installer_debug.log`), ce qui facilite le dépannage.

  * Impact : Aucun changement cassant pour les installations existantes utilisant le compte SYSTEM par défaut. Les nouveaux paramètres sont entièrement facultatifs.
