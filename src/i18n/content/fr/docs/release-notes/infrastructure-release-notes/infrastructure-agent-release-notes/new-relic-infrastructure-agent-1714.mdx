Une nouvelle version de l&apos;agent a été mise en service. Suivez les procédures standard pour [mettre à jour l&apos;agent d&apos;infrastructure](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent). New Relic recommande de mettre à jour l&apos;agent régulièrement, et au minimum tous les 3 mois. Au moment de cette sortie, la version la plus ancienne prise en charge est [l&apos;agent d&apos;infrastructure 1.58.1](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1581/).

## Fixé

* Correction de `transmitBytesPerSecond` affichant zéro en cas de trafic élevé sur Windows (NR-502872) dans [#2166](https://github.com/newrelic/infrastructure-agent/pull/2166)

  <Callout variant="important">
    Pour activer ce correctif, vous devez définir l&apos;option de configuration [`win_network_interface_v2`](/docs/infrastructure/infrastructure-agent/configuration/infrastructure-agent-configuration-settings#win-network-interface-v2) sur `true`. Cette option utilise des compteurs 64 bits au lieu de compteurs 32 bits pour les métriques réseau sur Windows.
  </Callout>
