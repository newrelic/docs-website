### Mise à jour

* Nous avons sorti une mise à jour du modèle AWS CloudFormation pour notre [intégration métrique Streams](https://docs.newrelic.com/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/via-cloudformation-cwms/). Cette mise à jour résout un problème critique qui pourrait empêcher la mise à jour de la stack et l&apos;utilisation des environnements d&apos;exécution Python AWS Lambda modernes.

* Pour appliquer le correctif à votre intégration existante, veuillez suivre les étapes ci-dessous pour mettre à jour votre stack CloudFormation :

  1. **Accédez à CloudFormation**: connectez-vous à votre console de gestion AWS et accédez au service **CloudFormation**.

  2. **Sélectionnez votre pile**: Recherchez et sélectionnez votre stack New Relic métrique Streams (elle est souvent nommée `NewRelic-Metric-Streams-*`), puis cliquez sur **Update Stack** \[Mettre à jour la pile] liste déroulante. Parmi les options, sélectionnez **Make a direct update** \[Effectuer une mise à jour directe]. Pour plus d&apos;informations, consultez la [documentation AWS sur la mise à jour directe de la pile](https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/using-cfn-updating-stacks-direct.html).

  3. **Remplacer le modèle**: sur l&apos;écran de mise à jour, vous **devez** sélectionner **Replace current template** \[Remplacer le modèle actuel].

  **REMARQUE**: n&apos;utilisez pas l&apos;option « Utiliser le modèle existant », car elle n&apos;appliquera pas les correctifs nécessaires.

  4. **Fournissez l&apos;URL du nouveau modèle**: sous la section *Spécifier le modèle*, choisissez l&apos;option **Amazon S3 URL** et collez le lien suivant :

  `https://nr-downloads-main.s3.amazonaws.com/cloud_integrations/aws/cloudformation/newrelic-cloudformation-mstreams.yml`

  5. **Confirmer et mettre à jour**: cliquez sur **Next** \[Suivant] dans les pages d’options de paramètres et stack. Sur la page de révision finale, faites défiler vers le bas, cochez la case pour **reconnaître qu&apos;AWS CloudFormation peut créer des ressources IAM**, puis cliquez sur **Update stack** \[Mettre à jour stack].

  Vous pouvez monitorer la progression dans l&apos;onglet **événement**. La mise à jour est terminée lorsque l’état stack passe à `UPDATE_COMPLETE`.
