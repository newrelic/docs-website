### Nouveaux documents

* [La gestion de la cardinalité](/docs/data-apis/ingest-apis/metric-api/cardinality-management) vous fournit des outils d&apos;administration pour gérer et examiner l&apos;utilisation de la cardinalité de votre organisation.

### Changements majeurs

* Instructions améliorées sur [le suivi des utilisateurs impactés avec l&apos;Errors Inbox](/docs/errors-inbox/error-users-impacted/), avec une nouvelle capture d&apos;écran et un tableau décrivant les différentes méthodes d&apos;implémentation.
* Mise à jour de [Lier les applications instrumentées avec OpenTelemetry à Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) et [Explorer votre cluster Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/kubernetes-cluster-explorer) avec les informations actuelles sur l&apos;installation et la configuration d&apos;OpenTelemetry.
* Réécriture de [Règles de silence : Supprimer les notifications](/docs/alerts/get-notified/muting-rules-suppress-notifications) pour décrire comment créer, gérer, supprimer et examiner vos événements d&apos;alerte et incidents.
* Réorganisation de notre documentation sur le [dépannage Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/troubleshooting/overview) pour améliorer la navigation et la facilité de recherche.
* Révision de notre documentation [OpenTelemetry](/docs/opentelemetry/opentelemetry-introduction) pour plus de clarté et d&apos;exhaustivité.
* Ajout d&apos;instructions pour configurer l&apos;API runtime de Composer pour la détection des paquets de gestion des vulnérabilités dans la [configuration de l&apos;agent PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-vulnerability-management).

### Modifications mineures

* Mise à jour de quelques documents pour vous informer que les graphiques de type panneau d&apos;affichage ne s&apos;affichent pas correctement lorsqu&apos;ils sont partagés via des liens, des images exportées et des PDF.
* Correction de plusieurs documents où nous mélangions des commandes Linux avec nos commandes shell Windows.
* Suppression de documentation obsolète, puis ajout de redirections vers leurs versions actuelles.
* Mise à jour des [conditions d&apos;alerte](/docs/alerts/create-alert/create-alert-condition/alert-conditions) avec l&apos;interface utilisateur actuelle.
* Ajout de nouveaux attributs [`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming) à notre dictionnaire de données.
* Mise à jour de la documentation sur le monitoring des performances réseau concernant l&apos;API de polling Meraki et les paramètres par défaut pour l&apos;utilisation d&apos;ICMP avec KTranslate.
* Ajout de `NewRelic.Labels` aux [paramètres de configuration .NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration).
* Dans [Suivre les utilisateurs impactés avec l&apos;Errors Inbox](/docs/errors-inbox/error-users-impacted/#set-users), ajout de l&apos;appel API PHP `newrelic_set_user_id`.
* Ajout de nouvelles valeurs de type d&apos;événement interdites dans [Exigences et limites des données pour les données d&apos;événements personnalisés](/docs/data-apis/custom-data/custom-events/data-requirements-limits-custom-event-data): les préfixes de chaîne `EntityRelationship`, `RelationshipUpdate` et `Relationship[0-9]`.
* Dans [Démarrer avec New Relic et Terraform](/docs/infrastructure-as-code/terraform/terraform-intro), nous avons décrit comment utiliser CodeStream avec Terraform.
