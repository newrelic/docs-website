### Nouveaux documents

* Le [guide de migration de .NET OpenTracing vers les couches AWS Lambda](/docs/apm/agents/net-agent/getting-started/opentracing-to-layers-migration-guide) vous aide à migrer de .NET OpenTracing vers la couche Lambda de l&apos;agent APM .NET.
* Utilisez [l&apos;installation manuelle de XCFramework](/docs/mobile-monitoring/new-relic-mobile-ios/installation/xcframework-installation) lorsque vous souhaitez installer manuellement l&apos;agent iOS New Relic.
* Le bulletin de sécurité [NR24-02 - OpenSSH dans New Relic Salesforce Exporter](/docs/security/new-relic-security/security-bulletins/security-bulletin-nr24-02) décrit les recommandations et les mesures à prendre si vous utilisez New Relic Salesforce Exporter.

### Changements majeurs

* Mises à jour et améliorations finalisées de notre guide [installer l&apos;agent .NET](/install/dotnet/) pour trouver plus facilement les informations que vous cherchez et configurer le monitoring de votre application.
* Dans la [référence NRQL](/docs/nrql/nrql-syntax-clauses-functions/#func-histogram), ajout d&apos;arguments supplémentaires et de valeurs par défaut, ainsi que des mises à jour des [temps de réponse des histogrammes](histogram-response-times-defaults) pour faciliter l&apos;écriture de requêtes liées à ces fonctions d&apos;agrégation.
* Dans notre interface, nous avons remplacé `Add data` par `Integrations & Agents`. Nous avons mis à jour notre site de documentation pour refléter cela.
* Clarification des différences entre la résiliation d&apos;un compte et la suppression d&apos;une organisation dans [passer à une édition inférieure ou résilier New Relic](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-account) et le [tutoriel NerdGraph : Afficher, créer, renommer ou résilier des comptes](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph).

### Modifications mineures

* Dans [graphiques et dashboards : visualiser vos données](/docs/nrql/get-started/charts-and-dashboards-with-nrql/#performance-improvement-for-dashboard-and-charts), ajout de conseils pour améliorer les performances de vos dashboards et graphiques.
* Dans les [paramètres de configuration de l&apos;agent d&apos;infrastructure](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#process-container-decoration), ajout d&apos;une section décrivant comment l&apos;agent décore les échantillons de processus avec des informations sur les conteneurs lorsque `process_container_decoration` est activé.
* Informations simplifiées sur les versions compatibles dans [Agent .NET : compatibilité et prérequis pour .NET Core](/docs/apm/agents/net-agent/configuration/net-agent-configuration/).
* Ajout d&apos;une note dans plusieurs documents de configuration de l&apos;APM Python lorsqu&apos;une option de configuration est désactivée quand le mode haute sécurité est activé.
* Dans [la gestion et la réponse aux problèmes et aux événements d&apos;alerte](/docs/alerts/alert-event-management/Issues-and-alert-event-management-and-response/#search-filters), ajout d&apos;une note précisant quand les événements d&apos;alerte d&apos;avertissement et critiques sont créés.
* Mise à jour du diagramme en haut de [Instrumenter votre fonction Lambda non conteneurisée](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/instrument-your-own/).
* Suppression de « Utilisateurs RightScale et New Relic ». RightScale était un partenariat obsolète depuis longtemps.
* Notez que ServiceNow ITSM est une intégration de notification d&apos;alertes héritée. Utilisez plutôt [l&apos;intégration certifiée de ServiceNow](/docs/alerts/get-notified/notification-integrations/#servicenow-certified).
* Regroupement et déplacement de toute notre documentation sur l&apos;[intégration Kubernetes version 2](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/k8s-version2/overview/) dans une sous-catégorie de notre site. La version 3 de Kubernetes est la plus récente et nous vous recommandons de l&apos;utiliser à la place.
* Vous ne pouvez pas [effectuer la rotation de nos clés d&apos;API REST](/docs/apis/intro-apis/new-relic-api-keys/#rotate-rest-api-key), car cette ancienne clé est obsolète. Si vous utilisez une clé d&apos;API REST, supprimez-la et utilisez [l&apos;une de nos autres clés](/docs/apis/intro-apis/new-relic-api-keys/#rotate-keys) à la place.
