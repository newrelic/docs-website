### Nouveaux documents

* Une nouvelle documentation d&apos;installation ramifiée pour l&apos;[intégration de monitoring Kubernetes](/install/kubernetes/) vous aide à trouver les informations d&apos;installation spécifiques dont vous avez besoin.
* Installez, configurez et monitorez vos données [dbt Cloud](/docs/more-integrations/dbt-cloud/dbt-cloud-integration) exécutées sur Apache Airflow et utilisant Snowflake.
* Notre nouvel [opérateur d&apos;agent Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator) automatise l&apos;instrumentation des agents APM afin que vous n&apos;ayez plus à les gérer manuellement. Désormais en version préliminaire publique.
* L&apos;agent iOS a ajouté la prise en charge de l&apos;envoi des [logs mobiles](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs) et de l&apos;[instrumentation watchOS](/docs/mobile-monitoring/new-relic-mobile-ios/watchOS/instrument-watchos) vers New Relic.
* [Dépanner l&apos;absence de données pour AWS Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-no-data-appears) décrit des solutions lorsque vos données AWS Lambda n&apos;apparaissent pas dans l&apos;interface.
* [Rapports de plantage natifs de l&apos;agent Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-native-crash-reporting) décrit comment activer, configurer et déboguer le rapporteur de plantage.
* [Modèle de titre pour les conditions d&apos;alerte NRQL](/docs/alerts/create-alert/condition-details/title-template) décrit comment utiliser les modèles de titre pour créer des titres pertinents pour vos événements d&apos;alerte.
* [L&apos;observabilité des LLM Traceloop avec OpenLLMetry](/docs/opentelemetry/get-started/traceloop-llm-observability/traceloop-llm-observability-intro) décrit comment utiliser cet outil basé sur OpenTelemetry pour monitorer les performances de votre modèle d&apos;IA.

### Changements majeurs

* Les mises à jour du contenu et du style pour la liaison des applications [instrumentées par APM](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes) et [instrumentées par OpenTelemetry](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes) améliorent la lisibilité et facilitent la configuration. Nous avons également mis à jour notre introduction à l&apos;[intégration Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration) pour améliorer le style et la clarté.
* Mise à jour de notre documentation pour refléter le changement de prix par Go, par exemple dans [la facturation et les règles d&apos;ingestion de données](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices).
* Ajout de nouveaux attributs [`MobileApplicationExit`](/attribute-dictionary/?dataSource=Mobile&event=MobileApplicationExit) à notre dictionnaire de données.
* Ajout d&apos;informations d&apos;installation spécifiques à Azure pour [installer l&apos;agent .NET](/install/dotnet/?deployment=azure&azure=azuresiteextension).
* Le 3 juillet 2024, notre [intégration Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements/#compatibility) a cessé de prendre en charge Kubernetes v1.25 et les versions antérieures.
* Une réécriture importante de [Découvrez New Relic AI, votre assistant d&apos;observabilité](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-ai) décrit comment tirer le meilleur parti de nos outils d&apos;IA.
* Mise à jour des [paramètres de configuration de l&apos;agent d&apos;infrastructure](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#is_integrations_only) avec les informations de configuration du nouveau paramètre `is_integrations_only`. Activez ce paramètre si vous instrumentez un hôte depuis une source autre que l&apos;agent d&apos;infrastructure, telle qu&apos;un collecteur OpenTelemetry ou un node exporter Prometheus.

### Modifications mineures

* Diverses mises à jour relatives au nouvel opérateur de l&apos;agent Kubernetes.
* Les mises à jour de la documentation OpenTelemetry ont corrigé certains liens d&apos;ancrage brisés, ajouté des informations sur la configuration du timeout au [endpoint OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp/#payload-limits) et apporté d&apos;autres améliorations de cohérence.
* Mise à jour de [AWS PrivateLink](/docs/data-apis/custom-data/aws-privatelink/#regions-zones) pour préciser que nous prenons en charge plusieurs régions
* Mise à jour de notre [tableau de compatibilité de l&apos;agent Node.js](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#system) pour inclure la prise en charge de Node.js version 22.
* Nous avons déplacé la documentation de dépannage Azure de notre agent .NET dans la [catégorie dépannage](/docs/apm/agents/net-agent/azure-installation/azure-cloud-services-no-data-appears/).
* Mise à jour de `issueId` et ajout de `entityGuid` dans [/docs/data-apis/ingest-apis/security-data-api](/docs/data-apis/ingest-apis/security-data-api/#req-fields).
* Correction d&apos;une erreur dans un extrait de code de l&apos;[intégration Cacti](/docs/infrastructure/host-integrations/host-integrations-list/cacti-integration/).
* Mise à jour de l&apos;[instrumentation sans couche pour le monitoring Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-layerless/#dotnet) pour préciser que les fonctions Lambda .NET ne nécessitent pas de couches.
* Mise à jour de [contrôler le démarrage de l&apos;agent Ruby](/docs/apm/agents/ruby-agent/troubleshooting/control-when-ruby-agent-starts) pour décrire comment désactiver l&apos;agent dans des contextes spécifiques.
* Nous avons supprimé les formulaires d&apos;inscription intégrés de notre site de documentation.
* Ajout d&apos;instructions sur la configuration et le transfert des logs Temporal dans [l&apos;intégration de monitoring Temporal auto-hébergée](/docs/infrastructure/host-integrations/host-integrations-list/temporal-monitoring-integration/#temporal-logs-configuration).
* Le monitoring [Infrastructure](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/#os), [performances SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#linux), [syslog réseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#linux) et [données de flux réseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#linux) a cessé de prendre en charge CentOS 7 le 30 juin 2024.
* Mise à jour des [limites de débit avec NRQL](/docs/nrql/using-nrql/rate-limits-nrql-queries/#query-duration) avec le délai d&apos;attente de requête par défaut de 5 secondes de NerdGraph.
* Mise à jour de [application ne répondant pas](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding) avec de nouvelles captures d&apos;écran et des détails sur la configuration côté serveur.
* Correction d&apos;une erreur de syntaxe pour `disable_all_plugins` dans le guide de dépannage [réduire l&apos;empreinte CPU de l&apos;agent d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/troubleshooting/reduce-infrastructure-agents-cpu-footprint).
* Les [métriques au niveau du code](/docs/codestream/observability/code-level-metrics/#requirements) de CodeStream pour PHP nécessitent l&apos;extension PHP Intelephense.
* Dans [supprimer des applications de New Relic](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/#wait-longer), nous avons précisé que vous pouvez supprimer une application lorsque New Relic n&apos;a reçu aucune donnée de cette application depuis 12 heures ou plus.
* Fusion de nos documents sur la réduction de l&apos;ingestion de données et la gouvernance des données Kubernetes en une seule page [réduire les données ingérées](/docs/kubernetes-pixie/kubernetes-integration/installation/reduce-ingest).
* Mise à jour du [plugin AWS FireLens pour la transmission de logs](/docs/logs/forward-logs/aws-firelens-plugin-log-forwarding/#firelens-plugin) avec plusieurs nouvelles régions AWS désormais prises en charge et leurs dépôts d&apos;images.
* Mise à jour des [workflows d&apos;événements d&apos;alerte](/docs/alerts/get-notified/alert-event-workflows/#navigate-notification-log) avec les colonnes actuelles du log des notifications.
* Mise à jour de la [compatibilité et des prérequis pour l&apos;agent Java](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#JDBC) pour indiquer que notre agent Java version 8.13.0 ou supérieure prend en charge HikariCP 2.4.0 jusqu&apos;à la dernière version. (Ceci remplace le module incubateur HikariCP.)
* L&apos;agent .NET prend désormais en charge [`Elastic.Clients.Elasticsearch`](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#app-frameworks-core) version 8.14.5.
