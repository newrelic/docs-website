### Nouveaux documents

* La fonctionnalité monitoring des applications mobiles [qui détecte les applications ne répondant pas](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding) vous aide à suivre et à analyser quand et pourquoi votre application Android est bloquée pendant plus de cinq secondes.
* La nouvelle documentation de l&apos;API monitoring des navigateurs [`log()`](/docs/browser/new-relic-browser/browser-apis/log) et [`wrapLogger()`](/docs/browser/new-relic-browser/browser-apis/wraplogger) définit la syntaxe, les exigences, les paramètres et des exemples sur la façon d&apos;utiliser ces nouveaux points de terminaison.

### Changements majeurs

* Nous avons supprimé notre catégorie de niveau supérieur `Other capabilities` et migré tout son contenu vers des catégories plus appropriées. Avec chaque chose à sa place, il vous sera plus facile de trouver ce que vous cherchez.
* Nous avons déplacé les exemples de collecteur OpenTelemetry de notre site de documentation vers notre `newrelic-opentelemetry-examples`]\([https://github.com/newrelic/newrelic-opentelemetry-examples](https://github.com/newrelic/newrelic-opentelemetry-examples)) Un référentiel GitHub pour faciliter la maintenance et améliorer la visibilité de leur code source ouvert.
* Notre monitoring de l&apos;IA prend en charge [les modèles NVIDIA NIM](/docs/ai-monitoring/compatibility-requirements-ai-monitoring/#deploy-at-scale) et ne nécessite aucune configuration supplémentaire en dehors de nos flux d&apos;installation manuels ou basés sur une interface utilisateur. Pour en savoir plus, consultez notre blog [monitoring de l&apos;IA pour NVIDIA NIM](https://newrelic.com/blog/how-to-relic/ai-monitoring-for-nvidia-nim).

### Modifications mineures

* Ajout du point de terminaison d&apos;URL du service de validation IAST aux [points de terminaison](/docs/security/security-privacy/compliance/fedramp-compliant-endpoints/#agents) et [aux réseaux](/docs/new-relic-solutions/get-started/networks/#new-relic-dualstack-endpoints) conformes à FedRAMP.
* Mise à jour [du dépannageIAST ](/docs/iast/troubleshooting/#see-my-application)avec des plages d&apos;adresses IP spécifiques pour afficher une liste blanche si votre application n&apos;apparaît pas.
* [Configuration de l&apos;agentRuby ](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/#instrumentation-aws_sqs)mise à jour avec les informations configuration auto-instrumentation `NEW_RELIC_INSTRUMENTATION_AWS_SQS`.
* Les informations configuration de rapport `ApplicationExitInfo` ont été supprimées des [paramètres de configuration monitoring des applications mobiles](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/configure-settings), car vous pouvez le faire dans l&apos;interface utilisateur.
* [Les enrichissements de workflow](/docs/alerts/get-notified/incident-workflows/#enrichments) prennent en charge plusieurs variables, ce qui vous permet de personnaliser vos données plus précisément en fonction de vos besoins.
* Nous avons ajouté l&apos;attribut [MobileApplicationExit](/attribute-dictionary/?dataSource=Mobile&event=MobileApplicationExit) à notre dictionnaire de données. Notre dictionnaire de données n&apos;est pas seulement une ressource de documentation, mais il alimente également directement l&apos;interface utilisateur en définitions d&apos;attributs, par exemple lorsque vous écrivez une requête NRQL dans notre générateur de requêtes.
* Suppression de la documentation relative à l&apos;API REST d&apos;alertes obsolète.
