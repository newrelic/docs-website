### Nouveaux documents

* [Le log de Forward Kong Gateway](/docs/logs/forward-logs/kong-gateway/) décrit comment installer et utiliser ce nouveau plugin de transfert de logvia l&apos;intégration Kubernetes.
* [La documentation relative à le monitoring d&apos;Azure App Service](/docs/apm/agents/manage-apm-agents/azure/monitoring-azure-app-service/) explique comment intégrer ce service Azure à New Relic et fournit des liens vers des instructions sur la configuration de nos agents APM Java, .NET, Node.js et Python.
* [L&apos;introduction à l&apos;explorateur de données](/docs/query-your-data/explore-query-data/query-builder/introduction-new-data-explorer/) explique comment utiliser notre nouvelle interface utilisateur pour interroger vos données et obtenir des informations plus détaillées sur la manière dont la plateforme New Relic peut vous aider et résoudre vos problèmes.
* [Le moniteur d&apos;environnements Amazon ECS avec agents de langage APM](/docs/infrastructure/elastic-container-service-integration/monitor-ecs-with-apm-agents/) vous aide à installer nos agents APM sur votre environnement Amazon ECS.
* [La section « Migrer vers NRQL](/docs/apis/rest-api-v2/migrate-to-nrql/) explique comment migrer votre requête API REST v2 vers une requête NRQL.

### Changements majeurs

* Nous avons transféré le contenu de notre site open source vers notre site de documentation.
* Nous avons clarifié la manière de gérer les secrets avec l&apos; [opérateur d&apos;agent Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator/).

### Modifications mineures

* Dans [les métriquesOpenTelemetry de New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-metrics/#otlp-summary), nous avons précisé que les métriques récapitulatives OpenTelemetry ne sont pas prises en charge.
* Mise à jour d&apos;une capture d&apos;écran dans [la règle d&apos;alerte recommandée et dashboards](/docs/kubernetes-pixie/kubernetes-integration/installation/recommended-alert-policies/#add-recommended-alert-policy) pour préciser où trouver les vignettes Kubernetes et Google Kubernetes Engine.
* Suppression des documents Stackato et WebFaction relatifs à l&apos;agent Python APM, car ils ne sont plus pertinents.
* Ajout d&apos;un nouveau point de terminaison de monitoring des navigateurs à notre document [Réseaux](/docs/new-relic-solutions/get-started/networks/#new-relic-dualstack-endpoints).
* Correction de problèmes mineurs dans nos documents de dépannage de monitoring des navigateurs.
* Dans [la section dépannage des erreurs de mise à niveau de l&apos;environnement d&apos;exécution,](/docs/synthetics/synthetic-monitoring/troubleshooting/runtime-upgrade-troubleshooting/#scripted-api-form) il est décrit comment résoudre les problèmes rencontrés lors de l&apos;utilisation d&apos;anciens environnements d&apos;exécution Node avec des objets `$http`.
* Dans [.NET API d&apos;agent](/docs/apm/agents/net-agent/net-agent-api/net-agent-api), nous avons ajouté `NewRelic.Api.Agent.NewRelic` à l&apos;appel d&apos;API `ITransaction` et ajouté le nouvel appel d&apos;API `RecordDatastoreSegment`.
