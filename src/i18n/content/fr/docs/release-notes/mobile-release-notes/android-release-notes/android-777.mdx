## Débogage

* Correction d&apos;un problème de performances où la compression d&apos;image de Session Replay s&apos;exécutait sur le thread principal. Elle s&apos;exécute désormais de manière asynchrone, ce qui réduit la surcharge lors de la capture.

* Résolution des problèmes de précision de rendu dans Session Replay en émettant des ajouts de vues incrémentiels dans leur ordre d&apos;affichage naturel.

* Correction d&apos;un problème où des charges hors ligne obsolètes pouvaient être envoyées en appliquant une durée de vie (TTL) qui supprime les données en file d&apos;attente depuis trop longtemps.

* Résolution d&apos;un problème où les attributs de session pouvaient être perdus lors de la collecte d&apos;arrêt.

* Correction d&apos;un problème où les événements mis en mémoire tampon étaient supprimés lors de l&apos;arrêt de l&apos;application, garantissant qu&apos;ils sont désormais envoyés lors de la collecte finale afin d&apos;éviter toute perte de données.

* Correction d&apos;un problème de synchronisation lors de la rotation de l&apos;appareil où la continuité de la session pouvait être perdue pendant la transition.

* La gestion des en-têtes HTTP a été rendue thread-safe, résolvant les erreurs qui pouvaient survenir lors d&apos;accès simultanés.

* Correction d&apos;un problème où les métriques de démarrage de l&apos;application pouvaient être mal calculées lorsqu&apos;un horodatage de démarrage n&apos;était pas défini.

* Résolution d&apos;un problème où l&apos;agent tente de signaler des plantages sans connexion réseau active, ce qui entraîne des erreurs inutiles sur les appareils hors ligne.

* Correction d&apos;un plantage natif (NDK) qui pouvait se produire lors de l&apos;arrêt en raison d&apos;une incompatibilité d&apos;ABI dans `stop()`.

* Correction du téléversement du mappage ProGuard pour les builds qui produisent directement un APK DexGuard.

* Amélioration de la compatibilité du cache de configuration Gradle lors du téléversement du mappage.

* Déplacement de la dépendance `commons-io` incluse dans le plug-in Gradle pour éviter les conflits avec les applications hôtes.

  Déclaration de soutien

  Nous recommandons de mettre à jour l&apos;agent au moins tous les trois mois. Trouvez les politiques et dates spécifiques pour la prise en charge de l&apos;agent Android dans la Politique de fin de vie des agents de monitoring des applications mobiles.
