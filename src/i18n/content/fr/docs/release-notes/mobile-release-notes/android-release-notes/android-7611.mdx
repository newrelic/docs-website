## Corrections

* Correction d&apos;un problème `NullPointerException` dans le processus d&apos;échantillonnage de la mémoire.
* Ajout d&apos;une diffusion sécurisée lors de l&apos;initialisation des heures de lancement de l&apos;application pour éviter la compatibilité `ClassCastException` dans le SDK Microsoft Intune MAM.
* Protection améliorée contre le rappel okhttp pour éviter les plantages de l&apos;agent dus aux exceptions de code utilisateur.
* Correction des plantages dupliqués provenant des rapports de plantage natifs.

## Déclaration de soutien

Nous vous recommandons de mettre à jour l&apos;agent au moins tous les trois mois. Recherchez des politiques et des dates spécifiques pour le support des agents Android dans la [politique de fin de vie des agents des applications mobiles de surveillance](/docs/mobile-monitoring/new-relic-mobile/get-started/mobile-agents-eol-policy/#android-eol).
