## Agent PHP New Relic v12.7.0.36

### Mises à jour de sécurité

* Mise à jour de la version de Go vers la 1.26.3 ([#1220](https://github.com/newrelic/newrelic-php-agent/pull/1220)). Traite [#1201](https://github.com/newrelic/newrelic-php-agent/issues/1201).

### Débogage

* Modification du niveau de log des messages de fibre de troncature de transaction ([#1207](https://github.com/newrelic/newrelic-php-agent/pull/1207)). Traite [#1200](https://github.com/newrelic/newrelic-php-agent/issues/1200).
* S&apos;assurer que les spans sans nom ne sont pas enregistrés pour le tracing infini ([#1197](https://github.com/newrelic/newrelic-php-agent/pull/1197)).
* Correction de la génération de la métrique seen/sent du daemon ([#1174](https://github.com/newrelic/newrelic-php-agent/pull/1174)).
* Autoriser la liaison de paramètres mysqli dans execute ([#1172](https://github.com/newrelic/newrelic-php-agent/pull/1172)).

### Déclaration de soutien

* Comme [communiqué précédemment](https://docs.newrelic.com/docs/apm/agents/php-agent/installation/php-agent-installation-rpm-key-rotation/), pour des raisons de bonnes pratiques et de sécurité, la clé de signature rpm a fait l&apos;objet d&apos;une rotation et la page d&apos;[installation rpm](https://docs.newrelic.com/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos/) reflète les dernières modifications.

* Le support de l&apos;agent PHP pour les versions de bibliothèques &amp; frameworks suivantes sera interrompu et New Relic vous recommande de mettre à niveau vers la dernière version prise en charge par le fournisseur :

  * Le support de Drupal 7 ne sera plus assuré dans toutes les versions de l&apos;Agent PHP après le 30 septembre 2026.
  * Le support de Drupal 9 ne sera plus assuré dans toutes les versions de l&apos;Agent PHP après le 30 septembre 2026.
  * Le support de Laravel 11 ne sera plus assuré dans toutes les versions de l&apos;Agent PHP après le 30 septembre 2026
  * Le support de Joomla 3 ne sera plus assuré dans toutes les versions de l&apos;Agent PHP après le 30 septembre 2026

* New Relic vous recommande de mettre à niveau l&apos;agent régulièrement et au moins tous les 3 mois. Pour plus d&apos;informations sur les versions d&apos;agent prises en charge et les délais de fin de vie, consultez notre [politique de fin de vie de l&apos;agent PHP New Relic](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).

* La page [de compatibilité de l&apos;agent et des exigences PHP](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/) doit être consultée pour obtenir les dernières informations sur les versions et la plate-forme PHP prises en charge.

<Callout variant="important">
  **Pour les installations utilisant une version ou une plateforme PHP non prise en charge, il est fortement recommandé de désactiver les mécanismes de mise à jour automatique du package de l&apos;agent PHP.** Cela peut être fait en ajoutant le package d&apos;agent PHP à une liste d&apos;exclusion pour les mises à niveau package . Ou vous pouvez épingler la version du package d&apos;agent PHP sur une version d&apos;agent qui prend en charge les anciennes fonctionnalités non prises en charge. Le fait de ne pas empêcher les mises à niveau peut entraîner l&apos;installation d&apos;une nouvelle sortie d&apos;agent et la suppression de la prise en charge de la fonctionnalité requise et non prise en charge. Cela perturberait la collecte de données APM.

  Les packages d&apos;agents PHP concernés sont :

  * newrelic-php5
  * newrelic-php5-common
  * newrelic-daemon
</Callout>
