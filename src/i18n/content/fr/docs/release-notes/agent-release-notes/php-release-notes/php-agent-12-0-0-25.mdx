## Agent PHP New Relic v12.0.0.25

### Débogage

* Améliorer le logging des erreurs de la carte des hooks Drupal.
* Corriger les problèmes de détection package legacy pour [Security RX](https://docs.newrelic.com/docs/vulnerability-management/overview/).

### Avis

#### La valeur par défaut de `newrelic.code_level_metrics.enabled` INI a changé dans cette sortie

Comme indiqué précédemment, le niveau [d&apos;observabilité du code par défaut dans](https://docs.newrelic.com/docs/codestream/observability/code-level-metrics/) configuration de l&apos;IDE passera d&apos;activé à désactivé dans cette sortie. Si vous n&apos;utilisez pas le niveau du code Métriques, vous ne devriez voir aucun impact et n&apos;aurez aucune action à entreprendre. Si vous vous fiez aux [métriques de niveau de code](https://docs.newrelic.com/docs/apm/agents/php-agent/features/php-code-level-metrics/), assurez-vous que votre fichier configuration newrelic.ini a le paramètre INI `newrelic.code_level_metrics.enabled` non commenté et défini sur true. par exemple :

```
newrelic.code_level_metrics.enabled = true
```

Cette modification n’affecte que la valeur de configuration par défaut. Les valeurs de configuration définies manuellement seront respectées et le comportement principal de CLM restera inchangé.

### Déclaration de soutien

* Comme indiqué précédemment, le framework/bibliothèque suivant, pour lequel l&apos;agent PHP avait précédemment mis fin à son support, sera supprimé dans cette sortie :

  * Guzzle 3.x
  * Kohana
  * Silex
  * Symfony 1.x-3.x
  * Zend 1.x-2.x

* La prise en charge de l&apos;agent PHP pour les versions de bibliothèque/framework suivantes prendra fin le 30 septembre 2025 :

  * WordPress 5.9
  * Drupal 8.x
  * Laravel 6.x, 8.x
  * PHPUnit 8.x

* New Relic vous recommande de mettre à niveau l&apos;agent régulièrement et au moins tous les 3 mois. Pour plus d&apos;informations sur les versions d&apos;agent prises en charge et les délais de fin de vie, consultez notre [politique de fin de vie de l&apos;agent PHP New Relic](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).

* La page [de compatibilité de l&apos;agent et des exigences PHP](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/) doit être consultée pour obtenir les dernières informations sur les versions et la plate-forme PHP prises en charge.

<Callout variant="important">
  **Pour les installations utilisant une version ou une plateforme PHP non prise en charge, il est fortement recommandé de désactiver les mécanismes de mise à jour automatique du package de l&apos;agent PHP.** Cela peut être fait en ajoutant le package d&apos;agent PHP à une liste d&apos;exclusion pour les mises à niveau package . Ou vous pouvez épingler la version du package d&apos;agent PHP sur une version d&apos;agent qui prend en charge les anciennes fonctionnalités non prises en charge. Le fait de ne pas empêcher les mises à niveau peut entraîner l&apos;installation d&apos;une nouvelle sortie d&apos;agent et la suppression de la prise en charge de la fonctionnalité requise et non prise en charge. Cela perturberait la collecte de données APM.

  Les packages d&apos;agents PHP concernés sont :

  * newrelic-php5
  * newrelic-php5-common
  * newrelic-daemon
</Callout>
