## Remarques

Cette sortie de l&apos;agent Python ajoute [OpenTelemetry](https://github.com/open-telemetry/semantic-conventions/blob/v1.24.0/docs/database/database-spans.md) datastore des noms d&apos;attributs d&apos;étendue compatibles, corrige un bogue dans l&apos;instrumentation[LangChain](https://pypi.org/project/langchain/) et désactive temporairement l&apos;instrumentation [Kombu.](https://pypi.org/project/kombu/)

Installez l&apos;agent à l&apos;aide de `easy_install/pip/distribute` via l&apos; [index du packagePython ](https://pypi.python.org/pypi/newrelic)ou téléchargez-le directement depuis le [site de téléchargement de New Relic](https://download.newrelic.com/python_agent/release).

## Nouvelles fonctionnalités

* Ajouter OpenTelemetry datastore un attribut d&apos;étendue compatible

  * L&apos;attribut span datastore est désormais plus cohérent avec les conventions de dénomination [OpenTelemetry](https://github.com/open-telemetry/semantic-conventions/blob/v1.24.0/docs/database/database-spans.md)

## Débogage

* Correction de l&apos;instrumentation LangChain pour les types de réponse de chaîne

  * Auparavant, lorsque [LangChain](https://pypi.org/project/langchain/) renvoyait des réponses de type chaîne, l’agent capturait uniquement le premier caractère de la chaîne. Cela a été corrigé.

* Désactiver l&apos;instrumentation Kombu

  * L&apos;instrumentation [Kombu](https://pypi.org/project/kombu/) existante dans l&apos;agent plante avec un `TypeError` généré par le module `inspect` . Cette instrumentation a été temporairement désactivée et sera réactivée lors d&apos;une prochaine sortie une fois que la cause première du crash sera identifiée et corrigée.
