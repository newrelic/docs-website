## Remarques

Cette sortie de l&apos;agent Python ajoute la prise en charge de l&apos;instrumentation pour [Google ADK](https://pypi.org/project/google-adk), ajoute le comptage des jetons pour plusieurs fournisseurs d&apos;IA ([Anthropic](https://pypi.org/project/anthropic), [Google GenAI](https://pypi.org/project/google-genai), [OpenAI](https://pypi.org/project/openai) et [AWS Bedrock](https://pypi.org/project/botocore)), réduit la taille de l&apos;en-tête `tracestate` sortant et corrige un plantage dans l&apos;instrumentation de [graphql-core](https://pypi.org/project/graphql-core).

Installez l&apos;agent en utilisant `easy_install/pip/distribute` via l&apos; [index des packagesPython ](https://pypi.python.org/pypi/newrelic), en utilisant `conda` via l&apos; [index des packages Conda-Forge](https://anaconda.org/conda-forge/newrelic), ou téléchargez-le directement depuis le [site de téléchargement de New Relic](https://download.newrelic.com/python_agent/release).

## Fonctionnalités améliorées

* Ajout de l&apos;instrumentation pour Google ADK

  * L&apos;agent Python instrumente désormais les workflows d&apos;agent d&apos;IA [Google ADK](https://pypi.org/project/google-adk) (Agent Development Kit).

  * Les composants instrumentés incluent :

    * `LlmAgent` liés aux backend Gemini ou VertexAI via [`google.genai`](https://pypi.org/project/google-genai)
    * Instances `FunctionTool` définies par l&apos;utilisateur
    * Workflows de sous-agents incluant les agents enfants et les instances `AgentTool`
    * Workflows multi-agents incluant la classe `Workflow` et les classes obsolètes `LoopAgent`, `ParallelAgent` et `SequentialAgent`

* Ajout du comptage des jetons pour Anthropic, Gemini, OpenAI et Bedrock

  * L&apos;agent Python signale désormais le nombre de jetons pour toutes les invocations LLM sur plusieurs fournisseurs d&apos;IA :

    * [SDK Anthropic](https://pypi.org/project/anthropic)
    * [SDK Google GenAI](https://pypi.org/project/google-genai)
    * [SDK OpenAI](https://pypi.org/project/openai)
    * AWS Bedrock via [botocore](https://pypi.org/project/botocore) et [aiobotocore](https://pypi.org/project/aiobotocore)

## Bugs

* Correction du plantage de l&apos;instrumentation pour graphql-core v3.2.10

  * Auparavant, [graphql-core](https://pypi.org/project/graphql-core) v3.2.10 a renommé l&apos;attribut `errors` en `collected_errors` dans `ExecutionContext`, provoquant le plantage de l&apos;agent. Cela a été corrigé et l&apos;agent fonctionne désormais correctement avec graphql-core v3.2.10 et les versions ultérieures.

* Suppression de `spanId` de l&apos;en-tête `tracestate` sortant

  * Auparavant, l&apos;agent incluait `spanId` dans l&apos;en-tête `tracestate` sortant. Étant donné que `spanId` est déjà présent dans l&apos;en-tête `traceparent`, il s&apos;agissait d&apos;une duplication inutile. Le `spanId` a été supprimé de `tracestate` pour réduire la taille de l&apos;en-tête sortant.
