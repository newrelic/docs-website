## Remarques

Cette sortie de l&apos;agent Python ajoute le support pour [oracledb](https://pypi.org/project/oracledb/), corrige un bug dans Kombu et Kafka où l&apos;objet `MessageTransaction` n&apos;a pas d&apos;attribut `destination_name`, et corrige la disparition de l&apos;utilisation mémorisée avec AWS.

Installez l&apos;agent à l&apos;aide de `easy_install/pip/distribute` via l&apos; [index du packagePython ](https://pypi.python.org/pypi/newrelic)ou téléchargez-le directement depuis le [site de téléchargement de New Relic](https://download.newrelic.com/python_agent/release).

## Nouvelles fonctionnalités

* Ajoute la prise en charge d&apos;oracledb

  * Ajouter une instrumentation pour [oracledb](https://pypi.org/project/oracledb/).

## Débogage

* Correction du bug « L&apos;objet `MessageTransaction` n&apos;a pas d&apos;attribut `destination_name`»

  * Correction d&apos;un bug dans les instrumentations MessageBroker où les méthodes de sérialisation recherchaient l&apos;attribut `destination_name` sur `MessageTransaction` et il n&apos;existait pas.

* Résoudre le problème de métadonnées d&apos;utilisation d&apos;AWS lors de l&apos;exécution simultanée d&apos;AWS et de K8

  * En raison des paramètres d&apos;autorisations dans le pod K8s exécuté sur les instances AWS EC2, les métadonnées d&apos;utilisation peuvent être récupérées la première fois. Cependant, si l&apos;application est laissée en cours d&apos;exécution en continu, l&apos;agent se reconnectera et les métadonnées d&apos;utilisation ne parviendront pas à se charger. Dans de tels cas, les métadonnées d’utilisation sont mises en cache par instance d’agent et peuvent être récupérées.
