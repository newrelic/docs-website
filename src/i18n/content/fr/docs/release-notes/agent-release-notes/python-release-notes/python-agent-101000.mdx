## Remarques

Cette sortie de l&apos;agent Python ajoute la prise en charge d&apos;async [elasticsearch](https://pypi.org/project/elasticsearch/), corrige un problème d&apos;analyse configuration graphql, corrige un bug dans l&apos;instrumentation [Kombu](https://pypi.org/project/kombu/) et réactive l&apos;instrumentation Kombu.

Installez l&apos;agent à l&apos;aide de `easy_install/pip/distribute` via l&apos; [index du packagePython ](https://pypi.python.org/pypi/newrelic)ou téléchargez-le directement depuis le [site de téléchargement de New Relic](https://download.newrelic.com/python_agent/release).

## Nouvelles fonctionnalités

* Ajouter un support asynchrone pour elasticsearch

  * Ajoute la prise en charge de [la recherche élastique](https://pypi.org/project/elasticsearch/) asynchrone. Merci [fagnerzulin](https://github.com/fagnerzulin) pour la contribution !

## Débogage

* Ajouter l&apos;analyse du paramètre de configuration graphql

  * Auparavant, lorsque le paramètre `instrumentation.graphql.capture_introspection_queries` était défini dans le fichier de configuration, ce paramètre n&apos;avait pas d&apos;effet. Cela a été corrigé.

* Correction d&apos;un crash dans Kombu lors de l&apos;exécution avec Sentry

  * Corrige un crash dans l&apos;instrumentation [Kombu](https://pypi.org/project/kombu/) où un `TypeError` était généré à partir du module `inspect`. Ce bug était dû à un conflit d&apos;encapsulation entre New Relic et Sentry. L&apos;instrumentation Kombu a également été réactivée.
