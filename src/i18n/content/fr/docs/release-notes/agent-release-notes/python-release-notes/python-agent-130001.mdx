## Remarques

Cette sortie de l&apos;agent Python corrige un problème de propagation du contexte dans [LangChain](https://pypi.org/project/langchain) et [LangGraph](https://pypi.org/project/langgraph), un problème d&apos;enregistrement de complétion de chat [OpenAI](https://pypi.org/project/openai), et un problème lors de l&apos;utilisation du streaming OpenAI([https://pypi.org/project/openai](https://pypi.org/project/openai)) via LiteLLM comme proxy.

Installez l&apos;agent en utilisant `easy_install/pip/distribute` via l&apos; [index des packagesPython ](https://pypi.python.org/pypi/newrelic), en utilisant `conda` via l&apos; [index des packages Conda-Forge](https://anaconda.org/conda-forge/newrelic), ou téléchargez-le directement depuis le [site de téléchargement de New Relic](https://download.newrelic.com/python_agent/release).

## Bugs

* Correction de la propagation du contexte dans LangChain et LangGraph

  * Ajoute une propagation du contexte de transaction supplémentaire à [LangChain](https://pypi.org/project/langchain) et [LangGraph](https://pypi.org/project/langgraph) pour prendre en charge un plus large éventail de structures d&apos;application.

* Corriger un problème d&apos;enregistrement de complétion de chat OpenAI

  * [OpenAI](https://pypi.org/project/openai) a réintroduit une logique qui ferme la réponse en streaming même si le consommateur n’a pas lu toutes les données. Auparavant, les données de complétion de chat n&apos;étaient enregistrées que lorsque la fin du flux était atteinte, ce qui faisait que les données de complétion de chat n&apos;étaient pas enregistrées lors de la fermeture du flux. Une logique a maintenant été ajoutée pour capturer les données de complétion de chat même si le flux n&apos;a pas été entièrement lu.

* Correction d&apos;un problème lors de l&apos;utilisation du streaming OpenAI via LiteLLM en tant que proxy

  * Auparavant, lors de l&apos;utilisation du SDK [OpenAI](https://pypi.org/project/openai) connecté via un serveur proxy [LiteLLM](https://www.litellm.ai/), il était possible de perdre le reporting des données de réponse du LLM en raison d&apos;un motif de fin manquant. Cela a été corrigé, et la réponse devrait désormais toujours être enregistrée.
