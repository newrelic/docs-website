## Remarques

Cette version de l&apos;agent Python supprime la prise en charge du tracing inter-application (CAT), de Python 3.8 et des certificats SSL intégrés. Il met également à jour tous les paquets vendored vers leurs dernières versions, et ajoute la prise en charge de CPython free-threaded, des systèmes d&apos;exploitation Windows, ainsi qu&apos;une instrumentation améliorée pour Microsoft AG2 (Autogen). De plus, cette version inclut un nouvel extra optionnel pour installer des certificats SSL via certifi. Cette version corrige également un bug dans l&apos;instrumentation de Strands Agents qui entraînait le signalement d&apos;exceptions en double.

Installez l&apos;agent en utilisant `easy_install/pip/distribute` via l&apos; [index des packagesPython ](https://pypi.python.org/pypi/newrelic), en utilisant `conda` via l&apos; [index des packages Conda-Forge](https://anaconda.org/conda-forge/newrelic), ou téléchargez-le directement depuis le [site de téléchargement de New Relic](https://download.newrelic.com/python_agent/release).

## Fonctionnalité supprimée

* Le tracing inter-application (CAT) a été supprimé au profit du tracing distribué (DT).

  * Le Cross Application Tracing (CAT) a été déprécié dans la version 7.0.0.166 de l&apos;agent et a été supprimé dans cette version. Si vous utilisez encore Cross Application Tracing (CAT), nous vous recommandons de migrer vers le tracing distribué (DT) dès que possible pour éviter toute interruption.

* Les certificats SSL inclus ont été supprimés.

  * Si vous avez besoin de certificats pour votre environnement, vous pouvez les installer via le gestionnaire de paquets de votre système d&apos;exploitation ou installer l&apos;agent avec l&apos;extra optionnel `newrelic[certificates]`.

* La prise en charge de Python 3.8 a été supprimée.

  * Python 3.8 a atteint sa fin de vie le 2024-10-07 et n&apos;est plus pris en charge par l&apos;agent. Veuillez envisager de mettre à niveau votre application vers une version de Python prise en charge afin de continuer à recevoir les mises à jour de l&apos;Agent Python.

## Sécurité

* Tous les packages internes ont été mis à jour vers leurs dernières versions.

  * `urllib3` mis à jour vers la v2.6.3
    * Ceci corrige toutes les CVE ouvertes sur `urllib3` à la date de cette version.
  * `opentelemetry_proto` mis à jour vers la v1.40.0
  * `wrapt` mis à jour vers la v2.1.2
  * `asgiref` mis à jour vers la v3.11.1

## Nouvelles fonctionnalités

* Nouvel extra `newrelic[certificates]` pour l&apos;installation de certificats SSL via [certifi](https://pypi.org/project/certifi/).

  * L&apos;agent fournit désormais un extra `newrelic[certificates]` optionnel qui peut être utilisé pour installer les certificats SSL inclus. Ceci est destiné aux utilisateurs qui ont besoin de certificats mais qui ne peuvent pas les installer via le gestionnaire de paquets de leur système d&apos;exploitation.

* Prise en charge de CPython free-threaded

  * À partir de Python 3.13, CPython publie des versions de Python avec le Global Interpreter Lock (GIL) désactivé, permettant un véritable multithreading.
  * Cette version ajoute la prise en charge de ce comportement, permettant à l&apos;agent de s&apos;exécuter en toute sécurité dans un environnement à thread libre.
  * Cette version inclut également des binaires précompilés pour CPython free-threaded sur Linux et Windows, permettant une installation facile dans ces environnements.

* Prise en charge de Windows

  * Cette version inclut des binaires précompilés pour CPython 3.13 ou version ultérieure sous Windows.
  * Notez que Python 3.12 ou une version antérieure n&apos;est pas officiellement pris en charge sous Windows en raison de variations importantes dans la résolution des timers. Il est toujours possible d&apos;installer et d&apos;utiliser l&apos;agent à partir des sources, mais les utilisateurs peuvent rencontrer des problèmes de données de chronométrage inexactes et devraient passer à Python 3.13 ou une version ultérieure pour une expérience optimale.

* Instrumentation améliorée pour [Microsoft AG2 (Autogen)](https://pypi.org/project/autogen/)

  * Ajout de la prise en charge des réponses en streaming via la méthode de générateur asynchrone `on_messages_stream()`.
  * L&apos;instrumentation inclut désormais un attribut `subcomponent` sur les spans qui représentent des appels d&apos;outils ou d&apos;agents. Ceci est utilisé pour fournir un mappage d&apos;entités amélioré dans l&apos;interface utilisateur New Relic.

## Bugs

* Exceptions dupliquées dans [Strands Agents](https://strandsagents.com/)

  * Après une mise à jour de Strands Agents qui encapsule certaines exceptions dans un nouveau type `EventLoopException`, des erreurs en double pourraient être signalées. Cela a été corrigé et l&apos;erreur sous-jacente ne devrait être signalée qu&apos;une seule fois.
