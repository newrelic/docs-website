## 3.40.0

### Ajouté

* Ajout de la fonction `txn.IgnoreApdex()` pour ignorer le score Apdex pour une transaction donnée
* Ajout d&apos;attributs personnalisés via les variables d&apos;environnement `NEW_RELIC_APPLICATION_LOGGING_FORWARDING_CUSTOM_ATTRIBUTES`
* Ajout de l&apos;intégration `nrconnect` pour la bibliothèque de connexion. Connect est une bibliothèque légère permettant de créer des API HTTP compatibles avec les navigateurs et gRPC
  * Merci au membre de la communauté @castaneai pour sa contribution à cette solution
* Ajout de l&apos;intégration `nrmongo-v2` prenant en charge la bibliothèque mongodb-v2
* Suite de tests GitHub Actions révisée
  * Ajout du support de la base de données dockerisée pour l&apos;intégration de mongodb et pgx5

### Fixé

* Représentation améliorée des paramètres de requête dans l&apos;intégration nrpgx5

* Correction d&apos;un bug où une condition de concurrence se produisait lors de l&apos;identification de l&apos;utilisation du conteneur.

* Capturez le nom de la table DynamoDB et le nom de l&apos;index dans DatastoreSegment
  * Merci au membre de la communauté @rittneje pour sa contribution à cette solution

* Versions de bibliothèques tierces mises à jour en raison de problèmes de sécurité ou d&apos;autres problèmes de support signalés :

  * `github.com/gofiber/fiber/v2` à 2,52,7 dans l&apos;intégration `nrfiber`
  * `github.com/go-chi/chi/v5` vers 5.2.2 dans l&apos;intégration `nrgochi`

### Déclaration de soutien

Nous utilisons la dernière version du langage Go. Au minimum, vous ne devriez pas utiliser de version de Go plus ancienne que celle prise en charge par l&apos;équipe Go elle-même. Consultez la [politique EOL de l&apos;agent Go](/docs/apm/agents/go-agent/get-started/go-agent-eol-policy) pour plus de détails sur les versions prises en charge de l&apos;agent Go et des composants tiers.
