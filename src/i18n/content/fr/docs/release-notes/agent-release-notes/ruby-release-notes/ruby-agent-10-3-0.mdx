## v10.3.0

* **Fonctionnalité : ajouter le nommage de requête de base de données via des commentaires SQL**

  Les requêtes de base de données peuvent désormais être nommées explicitement à l&apos;aide de commentaires SQL. Les requêtes peuvent inclure des commentaires `/* NewRelicQueryName: CustomName */` afin d&apos;attribuer des noms stables pour un meilleur suivi et une meilleure identification. Cela est particulièrement utile pour suivre des requêtes de base de données spécifiques lors de régressions de performances ou d&apos;incidents. [PR#3480](https://github.com/newrelic/newrelic-ruby-agent/pull/3480)

* **Fonctionnalité : ajout de l&apos;instrumentation de Semantic Logger**

  L&apos;agent prend désormais en charge le transfert de logs et la décoration de l&apos;enregistreur Semantic Logger pour les versions 4.6.0+ de la gemme `semantic_logger`. Si vous utilisiez précédemment l’appender New Relic intégré de l’enregistreur Semantic Logger, il est recommandé de choisir une approche pour éviter d’envoyer un log en double. L&apos;instrumentation de l&apos;enregistreur sémantique de New Relic peut être désactivée en définissant `instrumentation.semantic_logger` sur `disabled`. [PR#3467](https://github.com/newrelic/newrelic-ruby-agent/pull/3467)

  Merci à [@jdelStrother](https://github.com/jdelStrother) d&apos;avoir fourni un retour précieux qui a aidé à façonner cette instrumentation.

* **Fonctionnalité : ajouter une nouvelle configuration « ignored\_middleware\_classes »**

  Une nouvelle option de configuration, `ignored_middleware_classes`, permet aux utilisateurs d&apos;exclure des middlewares spécifiques de l&apos;instrumentation (ex. Rack::Cors). Sa valeur par défaut est un éventail vide. [Issue#1814](https://github.com/newrelic/newrelic-ruby-agent/issues/1814) [PR#3481](https://github.com/newrelic/newrelic-ruby-agent/pull/3481)

* **Fonctionnalité : ajouter une nouvelle API `NewRelic::Agent.add_transaction_log_attributes`**

  Une nouvelle API, `NewRelic::Agent.add_transaction_log_attributes`, permet aux utilisateurs d&apos;ajouter des attributs personnalisés au niveau de la transaction aux événements de log pour la transaction actuelle. Ces attributs seront appliqués uniquement aux logs créés dans le cadre de la transaction en cours. [PR#3472](https://github.com/newrelic/newrelic-ruby-agent/pull/3472)

* **Correctif : fournir une option de configuration pour réduire la cardinalité des métriques de diffusion ActionCable**

  Par défaut, les métriques pour les appels de méthode de diffusion ActionCable incluent la valeur de la diffusion. Cette valeur peut avoir une très forte cardinalité. Désormais, l&apos;option de configuration `:simplify_action_cable_broadcast_metrics` permet aux utilisateurs de supprimer la valeur de diffusion du nom de la métrique. Cela crée une métrique qui ressemble à : `Ruby/ActionCable/broadcast`. Lorsque cette option de configuration est activée, la valeur de diffusion sera ajoutée en tant qu&apos;attribut de span. [PR#3463](https://github.com/newrelic/newrelic-ruby-agent/pull/3463)

* **Correctif : supprimer le require « digest/md5 » mort pour la conformité FIPS/FedRAMP**

  Dans la version 7.1.0 de l’agent, l’utilisation de MD5 a été remplacée par SHA1 pour la conformité FIPS [(PR)](https://github.com/newrelic/newrelic-ruby-agent/pull/686). Cependant, l&apos;ancien require de « digest/md5 » n&apos;a pas été supprimé. Nous avons supprimé l’exigence pour aider nos utilisateurs FIPS/FedRAMP. Merci à [@ashleyboehs](https://github.com/ashleyboehs) d&apos;avoir porté cela à notre attention ! [Issue#3469](https://github.com/newrelic/newrelic-ruby-agent/issues/3469) [PR#3470](https://github.com/newrelic/newrelic-ruby-agent/issues/3470)

* **Correction de bug : empêcher l&apos;agent de démarrer pendant `rails test` pour éviter un délai d&apos;arrêt**

  Auparavant, l’agent provoquait un délai d’arrêt de -3 secondes lors de l’exécution de la commande `rails test`. La constante `Rails::Command::TestCommand` a été ajoutée à la liste `autostart.denylisted_constants` par défaut pour empêcher l&apos;agent de démarrer lors de l&apos;exécution des tests Rails. Merci à [@varyform](https://github.com/varyform) d’avoir porté cela à notre attention. [PR#3478](https://github.com/newrelic/newrelic-ruby-agent/issues/3478)

* **Correction de bug : corrige les avertissements &quot;Impossible de calculer le temps écoulé de la transaction&quot; lors de l&apos;utilisation du serveur Web Falcon**

  L&apos;agent utilise désormais `Fiber.current.object_id` au lieu de `Thread.current.object_id` pour suivre l&apos;état de la transaction lors de l&apos;exécution sous Falcon, empêchant les collisions dues aux requests simultanées partageant le même thread. Corrige également une erreur &quot;NameError: uninitialized constant `Async::HTTP::VERSION`&quot; lors de l&apos;utilisation de Falcon. Merci à [@97jaz](https://github.com/97jaz) et [@gsar](https://github.com/gsar) d&apos;avoir porté cela à notre attention. [PR#3483](https://github.com/newrelic/newrelic-ruby-agent/issues/3483)

* **Correctif : correction d&apos;une faute de frappe dans harvest.rb causant une NoMethodError**

  Une faute de frappe dans `lib/new_relic/agent/agent_helpers/harvest.rb` a causé un `NoMethodError: undefined method 'agent' for NewRelic:Module`. Merci à [@oakbow](https://github.com/oakbow) d&apos;avoir signalé ce problème. [PR#3484](https://github.com/newrelic/newrelic-ruby-agent/issues/3484)

* **Correction de bug : suppression de l&apos;utilisation de ObjectSpace.\_id2ref obsolète**

  L’agent utilise désormais une approche alternative au lieu de la méthode `ObjectSpace._id2ref` obsolète, éliminant les avertissements de dépréciation lors de l’exécution sur Ruby 4.0+. [PR#3490](https://github.com/newrelic/newrelic-ruby-agent/pull/3490)

* **Correction de bug : correction de NoMethoError dans l&apos;instrumentation de Logging**

  Auparavant, lorsque l&apos;instrumentation de la gem Logging tentait de décorer les log locaux, elle levait un `NoMethodError` si elle rencontrait un objet autre qu&apos;une chaîne de caractères. Ce problème est maintenant résolu. [PR#3501](https://github.com/newrelic/newrelic-ruby-agent/pull/3501)
