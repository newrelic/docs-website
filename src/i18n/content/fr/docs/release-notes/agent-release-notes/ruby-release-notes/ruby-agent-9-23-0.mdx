## v9.23.0

* **fonctionnalité : Ajouter sidekiq.ignore\_retry\_errors option de configuration**

  Une nouvelle option de configuration, `sidekiq.ignore_retry_errors`, a été ajoutée pour contrôler si les nouvelles tentatives de tâches Sidekiq sont capturées. Les erreurs de nouvelle tentative sont capturées par défaut, mais maintenant si `sidekiq.ignore_retry_errors` est défini sur `true`, l&apos;agent ignorera les exceptions levées lors des tentatives de nouvelle tentative de Sidekiq et ne signalera l&apos;erreur que si la tâche échoue définitivement. Merci [DonGiulio](https://github.com/DonGiulio) d&apos;avoir reconnu cette amélioration et d&apos;avoir contribué à la solution. [PR#3317](https://github.com/newrelic/newrelic-ruby-agent/pull/3317)

* **fonctionnalité : Avis de dépréciation pour l&apos;enregistrement déployé à l&apos;aide de Capistrano**

  L&apos;envoi d&apos;informations de déploiement d&apos;application via une recette Capistrano est obsolète et sera supprimé dans la version 10.0.0 de l&apos;agent. Pour l&apos;enregistrement du déploiement, veuillez consulter notre guide sur [le suivi des changements](https://docs.newrelic.com/docs/change-tracking/change-tracking-introduction/) pour obtenir la liste des options disponibles.

* **Fonctionnalité : Utiliser la configuration d&apos;échantillonnage du parent distant pour les décisions dans davantage de scénarios.**

  Auparavant, les options de configuration `distributed_tracing.sampler.remote_parent_sampled` et `distributed_tracing.sampler.remote_parent_not_sampled` étaient utilisées pour la décision d&apos;échantillonnage uniquement lorsque les en-têtes `traceparent` et `tracestate` étaient présents. Ces options de configuration sont désormais appliquées dans les cas où l&apos;en-tête `tracestate` est manquant et où seul l&apos;en-tête `newrelic` est disponible. Ce changement rend l&apos;échantillonnage distribué traceplus cohérent et prévisible. [PR#3306](https://github.com/newrelic/newrelic-ruby-agent/pull/3306)
