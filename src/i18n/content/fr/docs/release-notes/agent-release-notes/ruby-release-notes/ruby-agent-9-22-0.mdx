## v9.22.0

* **fonctionnalité : Instrumentation en une étape pour Kubernetes**

  L&apos;APM Kubernetes attache automatiquement les applications d&apos;instrumentation et gère les mises à niveau des agents dans le déploiement Kubernetes. Cette fonctionnalité est sortie du stade de préversion et est désormais généralement disponible. En savoir plus sur [l&apos;attachement automatique de Kubernetes](https://docs.newrelic.com/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator/). [PR#2635](https://github.com/newrelic/newrelic-ruby-agent/pull/2635) [PR#3287](https://github.com/newrelic/newrelic-ruby-agent/pull/3287)

* **fonctionnalité : avis d&apos;obsolescence pour Ruby 2.4 et Ruby 2.5**

  La prise en charge de l&apos;agent Ruby pour les versions Ruby 2.4 et 2.5 est obsolète et sera supprimée dans la version 10.0.0 de l&apos;agent. La nouvelle version minimale requise de Ruby deviendra Ruby 2.6 et nous arrêterons les tests avec Ruby 2.4 et 2.5. [PR#3288](https://github.com/newrelic/newrelic-ruby-agent/pull/3288)

* **Fonctionnalité : Avis de dépréciation pour la commande `newrelic deployments`**

  La prise en charge de l&apos;enregistrement du déploiement à l&apos;aide de la commande `newrelic deployments` est désormais obsolète et sera supprimée dans la version 10.0.0 de l&apos;agent.

  À l’avenir, il existe un certain nombre de moyens automatisés et manuels pour enregistrer les modifications dans New Relic. Veuillez consulter notre guide de [suivi des changements](https://docs.newrelic.com/docs/change-tracking/change-tracking-introduction/) pour une liste des options disponibles. [PR#3262](https://github.com/newrelic/newrelic-ruby-agent/pull/3262)

* **Fonctionnalité : Rappel de dépréciation pour le traçage inter-applications**

  Le traçage inter-applications est obsolète depuis la version majeure 8.0.0 de l&apos;agent Ruby. Nous supprimerons entièrement la prise en charge de cette fonctionnalité dans la version 10.0.0 de l&apos;agent. [PR#3288](https://github.com/newrelic/newrelic-ruby-agent/pull/3288)

* **Fonctionnalité : assurer la compatibilité avec Ruby 3.5 en modifiant Method#source\_location**

  Mise à jour de l&apos;agent pour analyser correctement la valeur de retour de Method#source\_location, qui passe dans Ruby 3.5 d&apos;un éventail à deux éléments à un éventail à cinq éléments. Ce changement maintient la prise en charge des anciennes versions Ruby tout en ajoutant la prise en charge de la future sortie. [PR#3257](https://github.com/newrelic/newrelic-ruby-agent/pull/3257)
