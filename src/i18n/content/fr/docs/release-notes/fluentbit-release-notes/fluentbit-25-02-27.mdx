### Prise en charge de Fluent Bit 3.2.7

L&apos;utilisateur recevra désormais la version 3.2.7 de Fluent Bit, qui inclut des correctifs pour les vulnérabilités connues. Pour plus de détails, consultez [https://fluentbit.io/announcements/](https://fluentbit.io/announcements/).

### Sécurité

Les versions précédentes de Fluent Bit des séries [3.1 et 3.2](https://fluentbit.io/announcements/) sont affectées par deux vulnérabilités de sécurité si les clients utilisent le [plugin d&apos;entrée OpenTelemetry](https://docs.fluentbit.io/manual/pipeline/inputs/opentelemetry) ou [l&apos;entrée Prometheus Remote Write](https://docs.fluentbit.io/manual/pipeline/inputs/prometheus-remote-write). Les vulnérabilités identifiées sont : [CVE-2024-50608](https://nvd.nist.gov/vuln/detail/CVE-2024-50608), [CVE-2024-50609](https://nvd.nist.gov/vuln/detail/CVE-2024-50609).

Par défaut, New Relic n&apos;inclut pas ces plugins dans notre distribution du package Fluent Bit dans aucun des packages d&apos;instrumentation que nous fournissons. Cependant, afin de soutenir les clients utilisant ces plugins et d&apos;atténuer tout impact potentiel, nous recommandons aux clients de mettre à niveau vers les dernières versions disponibles de ces packages :

### Hôtes :

* Windows : Agent d&apos;infrastructure mis à niveau vers la version 1.62.0 ou ultérieure
* Linux : Agent d&apos;infrastructure mis à niveau vers la version 1.62.0 ou ultérieure

### Kubernetes :

* newrelic-logging mis à niveau vers la version 1.26.1
* nri-bundle mis à niveau vers la version 5.0.115

### Fluent Bit:

* Image Docker de sortie New Relic Fluent Bit mise à niveau vers la version 2.3.0

### Remarques

Pour rester au courant des correctifs et améliorations les plus récents, abonnez-vous à notre [flux RSS de Logs](/docs/release-notes/logs-release-notes/).
