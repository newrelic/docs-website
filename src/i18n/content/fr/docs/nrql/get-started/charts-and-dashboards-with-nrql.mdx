Les requêtes vous aident à trouver les informations dont vous avez besoin, et les graphiques et diagrammes basés sur ces requêtes rendent les données utiles à votre organisation. Les graphiques et les tableaux de bord constituent la base de la visualisation des données, vous offrant de multiples options qui vous permettent de créer des flux de données visuellement attrayants et facilement compréhensibles. Vous pouvez tirer le meilleur parti de vos données en comprenant la manière dont les requêtes NRQL conduisent aux graphiques et aux tableaux de bord.

## Graphiques [#charts]

Pour terminer l’écriture d’une requête NRQL, vous devez créer un graphique comme étape finale. New Relic propose [plusieurs façons](/docs/query-your-data/explore-query-data/use-charts/chart-types) de visualiser vos données, et vous pouvez faire beaucoup de choses avec vos graphiques une fois que vous les avez créés. Après avoir écrit votre requête NRQL, vous pouvez :

* Utilisez votre graphique comme point de départ d&apos;un [nouveau dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create) pour monitorer plusieurs requêtes NRQL à partir d&apos;un seul emplacement.
* [Ajoutez votre graphique à un dashboard existant](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#add-pages-edit) pour ajouter une autre requête NRQL à votre monitoring.
* [Partagez votre graphique](/docs/query-your-data/explore-query-data/use-charts/use-your-charts#share-charts) avec d&apos;autres en copiant le permanent ou en le téléchargeant au format PDF (ou CSV, le cas échéant) pour permettre aux autres de voir les données de votre requête.
* Modifiez la requête NRQL derrière le graphique pour aider à affiner les données que vous voyez.
* [Créez une alerte](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition#create-simple-alert) à partir de votre requête NRQL pour vous avertir de manière proactive lorsque des problèmes surviennent.

Les graphiques sont excellents en eux-mêmes, mais ils brillent vraiment lorsque vous les compilez ensemble pour créer un dashboard, sur lequel vous pouvez en savoir plus dans les sections ci-dessous.

## Dashboards [#dashboards]

<img title="Dashboard screenshot.png" alt="A screenshot example of a dashboard" src="/images/dashboards_screenshot-crop_dashboard-ui.webp" />

<figcaption>
  Un dashboard représentant les données aux formats pie et graphique à barres.
</figcaption>

Alors que les graphiques sont parfaits pour générer des rapports sur des éléments individuels, les tableaux de bord sont le meilleur moyen d&apos;obtenir une vue beaucoup plus large de vos données : si les graphiques sont les pièces d&apos;un puzzle, les tableaux de bord sont le puzzle complété. Dashboards sont hautement configurables et vous pouvez les utiliser pour afficher le nombre de flux que vous souhaitez. Tu peux:

* [Ajoutez des graphiques](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#dash-create-chart) à vos tableaux de bord à l’aide du générateur de requêtes.
* Ajoutez votre propre [texte, images ou liens](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#markdown) (en utilisant Markdown) à votre dashboard.
* Déplacez les graphiques dans un dashboard pour vous assurer que les données les plus importantes se trouvent là où vous le souhaitez.
* [Partagez](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data#dashboards) votre dashboard par email ou créez un lien permanent vers votre dashboard.
* [Importez](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-import) ou [exportez](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#dash-json) votre dashboard au format JSON, ou exportez votre dashboard au format PDF.
* [Dupliquez](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#dash-clone) le dashboard pour l’utiliser comme base pour un tout nouveau dashboard.
* [Modifiez](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard#add-pages-edit) les composants de votre dashboard pour ajouter de nouveaux flux de données, modifier ceux existants ou supprimer les plus anciens qui ne sont plus pertinents.

Avez-vous vu notre [introduction à NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/) et appris [à utiliser NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-how-nrql-works/)? Si vous l&apos;avez fait, vous êtes prêt à démarrer notre [tutoriel guidé NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-tutorial/)!

## Conseils pour améliorer les performances dashboard et des graphiques [#performance-improvement-for-dashboard-and-charts]

Si vos graphiques New Relic sur le dashboard ne fonctionnent pas de manière optimale, tenez compte des conseils suivants pour améliorer les performances :

* **Utiliser la visualisation sous forme de tableau :** pour afficher de grands ensembles de données, utilisez la visualisation sous forme de tableau plutôt que les visualisations sous forme de barres ou de panneaux d&apos;affichage.
* **Visualisations séparées :** répartissez vos visualisations sur plusieurs pages. Un dashboard avec de nombreuses visualisations sur une seule page peut entraîner des problèmes de performances, en particulier si certains graphiques sont fréquemment mis à jour ou traitent de grands ensembles de données.
* **Diviser la requête NRQL :** divisez les grandes requêtes NRQL en requêtes plus petites et plus faciles à gérer.
* **Limiter les résultats de la requête :** en dernier recours, évitez d’utiliser `LIMIT MAX`. Spécifiez plutôt une limite qui répond à vos besoins.
