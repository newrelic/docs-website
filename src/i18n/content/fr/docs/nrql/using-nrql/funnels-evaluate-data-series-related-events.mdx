Avec [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql), vous pouvez utiliser des entonnoirs pour évaluer des ensembles d’actions connexes. Les actions partagent généralement une relation via un identifiant, tel qu&amp;apos;un identifiant d&amp;apos;utilisateur ou un identifiant de session.

Vous pouvez utiliser les fonctions `funnel` pour répondre à des questions telles que : « Parmi les personnes qui ont terminé l&amp;apos;étape A, combien ont également terminé l&amp;apos;étape B, et parmi les personnes qui ont terminé les étapes A et B, combien ont également terminé l&amp;apos;étape C ? »

Par exemple, chez New Relic, nous pourrions utiliser `funnel` pour suivre le nombre d&amp;apos;utilisateurs qui ont effectué ces étapes :

1. J&apos;ai visité la [page marketing de New Relic](https://newrelic.com/platform).
2. J&apos;ai créé un compte.
3. J&apos;ai créé un dashboard.

## structure de requête entonnoirs [#requirements]

Voici la structure d’une simple requête NRQL entonnoirs :

```sql
FROM DATA_TYPE 
  SELECT funnel(AN_ATTRIBUTE, ACTION_A, ACTION_B) 
  SINCE TIMEFRAME
```

La requête entonnoirs nécessite la [fonction`funnel` ](/docs/query-data/nrql-new-relic-query-language/getting-started/nrql-syntax-components-functions#func-funnel), un [attribut](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute) d&amp;apos;entonnoirs, et au moins deux étapes :

1. Commencez par l’étape d’ancrage, qui représente toujours 100 % des résultats.
2. Continuez avec une ou plusieurs étapes supplémentaires qui décrivent le nombre d’utilisateurs qui ont également effectué des actions supplémentaires, généralement inférieures à 100 % de l’étape d’ancrage. Cependant, cela pourrait être de 100 % si chaque utilisateur qui effectue l&apos;action (A) effectue également les actions supplémentaires que vous demandez.

## entonnoirs requête détails techniques

Vous pouvez utiliser la requête entonnoirs pour calculer le taux d&apos;achèvement des séquences ordonnées en plusieurs étapes sur une période donnée, comme dans l&apos;[exemple](#why-matters) d&amp;apos;inscription New Relic. Vous pouvez également utiliser des entonnoirs pour calculer le chevauchement entre plusieurs actions distinctes qui n&amp;apos;ont pas de relation séquentielle les unes avec les autres. En d’autres termes, l’ordre des étapes n’aura pas d’impact sur les calculs.

La manière dont les requêtes d&apos;entonnoirs calculent les résultats varie en fonction de la taille de l&apos;ensemble de données :

* Si l&apos;entonnoirs traite avec moins de 256 valeurs d&apos;attribut entonnoirs, il calculera la valeur exactement.
* Si l&apos;entonnoir traite 256 valeurs d&apos;attribut entonnoirs ou plus, il applique un algorithme appelé MinHash pour calculer des résultats approximatifs pour des performances optimisées.

Cela signifie que pour les grands ensembles de données, il peut y avoir occasionnellement des « faux positifs » au niveau des points de données individuels. Ils n’interfèrent pas avec la précision des estimations numériques fournies dans les résultats de la requête.

## exemple d&apos;entonnoirs [#example]

Cet exemple requête l&apos;événement <InlinePopover type="browser" />[`PageView`](/attribute-dictionary/?event=PageView) et son attribut. Il interroge les sessions de navigation uniques de votre utilisateur qui sont passées de la recherche de produits à leur ajout à leur panier, jusqu&amp;apos;au paiement. Il comprend des étiquettes pour chaque étape, indiquées par le mot-clé `AS`.

```sql
FROM PageView SELECT funnel(session, 
    WHERE pageUrl LIKE '%browse%' as 'Browse', 
    WHERE pageUrl LIKE '%cart%' as 'Cart', 
    WHERE pageUrl LIKE '%checkout%' as 'Checkout') 
    SINCE 1 week AGO
```

Inclure des actions supplémentaires à l&apos;intérieur des parenthèses de la fonction entonnoirs dans une liste séparée par des virgules :

```sql
SELECT funnel(session, 
    WHERE name = 'Controller/about/main' AS 'Action A', 
    WHERE name = 'Controller/about/careers' AS 'Action B', 
    WHERE name = 'Controller/about/insights' AS 'Action C', 
    WHERE name = 'Controller/about/apply' AS 'Action D') 
    FROM PageView 
    SINCE 1 week ago
```
