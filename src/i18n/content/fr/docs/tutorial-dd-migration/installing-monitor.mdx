La première étape pour tester New Relic par vous-même consiste à ingérer des données dans le moniteur. Les endroits les plus utiles pour commencer sont un infrastructure agent , un moniteur d’application (APM) ou des log monitoring fonctionnalités . Une fois que vous l’aurez fait, vous pourrez commencer à visualiser vos données immédiatement à l’aide de nos dashboards hautement granulaire générés automatiquement. Vous pouvez également configurer des dashboards et des alertes prédéfinis, ou commencer à utiliser l’une des autres fonctionnalités avancées répertoriées dans le [document final](/docs/tutorial-dd-migration/making-the-switch/#do-more) de cette série.

Presque toutes les fonctionnalités que vous souhaitez essayer seront disponibles gratuitement une fois que vous aurez installé une fonction monitoring : choisissez simplement l’une des options ci-dessous pour commencer !

La collecte des données de vos serveurs, de votre base de données, de votre plateforme cloud et de vos services hébergés n&apos;est que la première tâche, et une fois que vous avez tout mis en place, vous devez encore trouver des moyens de donner un sens à tout cela.

          New Relic vous permet de voir facilement ce qui se passe à l&apos;intérieur de votre infrastructure, qu&apos;elle s&apos;exécute dans le cloud ou sur des hôtes dédiés. Dans Datadog, vous connaissez peut-être ces fonctionnalités sous le nom de <DNT>**agents**</DNT>, et nous appelons également les mêmes fonctionnalités d&apos;infrastructure <DNT>**agents**</DNT> chez New Relic.

      Voici comment nous organisons nos agents infrastructure et notre intégration :

      * Agents d&apos;infrastructure pour Linux, macOS et système d&apos;exploitation Windows avec options d&apos;installation guidée et manuelle
      * Un agent pour [Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration/)
      * Agents cloud pour Amazon AWS, Microsoft Azure et Google Cloud Platform que vous pouvez installer indépendamment de nos agents d&apos;infrastructure
      * agents d&apos;intégration sur hôte pour des services comme MySQL, Apache, Docker et Kafka qui fonctionnent avec notre agent infrastructure pour maximiser vos rapports de données

      Tant que votre système et votre intégration sur hôte répondent à nos [exigences minimales](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/), tout ce dont vous avez besoin est d&apos;essayer un [compte New Relic gratuit](https://newrelic.com/signup) et vous pouvez commencer immédiatement.

Chez New Relic, nos agents se concentrent sur le langage et la liaison des données qu&apos;ils collectent à d&apos;autres entités que vous avez installées.

      * Instrumentez automatiquement les métriques, les événements et les traces
      * Ingérez vos logs automatiquement
      * Gérer la sécurité des services sans configuration préalable

      ## Démarrer avec monitoring des applications New Relic [#get-started-apm]

      Une fois la configuration terminée, vos données APM dans New Relic se trouvent dans la section <DNT>**APM and services**</DNT> de l&apos;interface utilisateur, qui est l&apos;équivalent de la section <DNT>**APM**</DNT> dans Datadog. Vous pouvez suivre une installation guidée pour la plupart de nos agents à tout moment dans [<DNT>**one.newrelic.com**</DNT>](https://one.newrelic.com) en sélectionnant <DNT>**one.newrelic.com &gt; Integrations &amp; Agents &gt; All &gt; Guided install**</DNT>, puis en sélectionnant votre agent dans la section <DNT>**APM (Application Monitoring)**</DNT> .

      Vous pouvez également installer nos agents APM directement en utilisant l’une des tuiles ci-dessous. Notre installation guidée vous guide à travers les procédures de configuration de chacun de nos agents APM afin que vous puissiez observer vos données immédiatement. Tout ce dont vous avez besoin est un [compte gratuit](https://newrelic.com/signup) et une application de monitoring et vous êtes prêt à commencer !

      ## Commencez à utiliser le monitoring des logs New Relic [#get-started-logs]

      Une fois installé, de vos log monitoring s&apos;effectue dans la <DNT>**Logs**</DNT> section de l&apos;interface utilisateur, exactement comme dans Datadog. Vous pouvez utiliser nos installations guidées pour vous connecter après avoir configuré l&apos;un de nos agents [infrastructure](/docs/journey-migration/migrating-infra/) ou [APM](/docs/journey-migration/migrating-apm/) .

      Les étapes de configuration des logs varient en fonction de la source de données, mais quel que soit votre choix, vous pouvez être assuré que nous vous guiderons à chaque étape du processus pour le rendre aussi simple et direct que possible. Vous pouvez voir notre liste d&apos;installations guidées pour les logs en sélectionnant <DNT>**one.newrelic.com &gt; Integrations &amp; Agents &gt; Logging**</DNT> dans l&apos;interface utilisateur de New Relic.

## Explorez les dashboards prédéfinis [#add-dashboard]

Une fois votre ingestion de données configurée, vous souhaiterez utiliser nos dashboards pour commencer à l&apos;observer. Vous pouvez configurer vous-même les dashboards en consultant [ce document](/docs/tutorial-dd-migration/migrating-dashboards/), ou vous pouvez en savoir plus sur l&apos;une de nos options prédéfinies répertoriées ci-dessous. Choisissez simplement l’option ci-dessous en fonction de l’agent de données que vous avez installé ci-dessus pour en savoir plus.
