La maintenance d’un site Web combine les efforts de plusieurs équipes. En cas de panne, les équipes de développeurs et d’ingénierie peuvent résoudre les problèmes directement, mais un spécialiste du marketing numérique peut ne pas avoir le même accès à l’administration du site. En tant que spécialiste du marketing numérique, le succès de votre équipe dépend tout autant des performances du site, mais vous n&apos;avez peut-être pas les outils nécessaires pour identifier ou résoudre vous-même tout problème.

Notre outil monitoring des performances web (WPM) donne des informations détaillées sur la disponibilité et les performances de votre site. Grâce aux contrôles de routine et <InlinePopover type="alerts" />à l&amp;apos;emploi de notre outil de monitoring, vous ne serez jamais incertain quant aux performances de votre site.

## Objectifs [#objectives]

Ce document vous aide à configurer notre outil monitoring de site Web sans code, puis vous guide dans la compréhension de vos données. Dans ce tutoriel, vous allez :

## Mettre en place monitoring des performances web [#setup]

<Steps>
  <Step>
    ### Obtenez des données sans installation

    monitoring des performances web ne nécessite aucune installation ni configuration. Il ne prend actuellement en charge que les navigateurs de bureau. Pour accéder à vos données, rendez-vous sur <DNT>**[one.newrelic.com &gt; Website performance monitoring](https://one.newrelic.com/website-grader)**</DNT>.
  </Step>

  <Step>
    ### Choisissez votre URL

    Une fois que vous avez choisi les pages à monitorer, New Relic déploie un ensemble de moniteurs provenant de différents serveurs du monde entier qui vérifient la disponibilité de votre page, les liens rompus, les performances et l&apos;expiration du certificat SSL. Pour commencer, nous vous recommandons de choisir votre page d’accueil.

    New Relic prendra quelques minutes pour activer le moniteur, puis vous commencerez à recevoir des données sur votre site Web. Gardez à l’esprit que votre moniteur vérifie uniquement les performances des pages que vous avez ajoutées manuellement.
  </Step>

  <Step>
    ### Ajouter la clé API Google PageSpeed

    Nous extrayons les principaux éléments essentiels du Web à partir de l&apos;API de Google afin que les scores dans New Relic correspondent à ceux que vous avez dans Google. Pour continuer à capturer ces scores après la configuration initiale, vous devrez créer une [clé API Google PageSpeed](https://developers.google.com/speed/docs/insights/v5/get-started).

## Mettez vos données en contexte

Une fois que votre moniteur aura signalé les données à New Relic, vous verrez des mesures qui peuvent vous aider à améliorer l&apos;expérience utilisateur et le classement SEO. Vous trouverez ci-dessous des exemples du type de données que vous verrez sur votre page de résumé.

<CollapserGroup>
  <Collapser id="ping" title="Vérifier la disponibilité de la page">
    <img title="View your page availability" alt="View page availability" src="/images/tutorial_screenshot-crop_ping-wpm.webp" />

    La première section vous montre les données de ping. Ce type de données constitue un contrôle de disponibilité, confirmant les fonctionnalités de base du site :

    * <DNT>**Site availability**</DNT>Cela indique si votre page a été chargée. Si ce n’est pas le cas, il se peut qu’il y ait un problème avec vos serveurs ou votre domaine.
    * <DNT>**Page load time**</DNT>. Cela collecte la vitesse médiane pendant laquelle votre page met à charger tous ses éléments. Ces données sont ensuite ventilées par type de contenu et par taille.
    * <DNT>**SSL Validity**</DNT>. Cela vous indique si votre certificat SSL est valide, en cours d&amp;apos;expiration ou expiré. Il inclura également la date à laquelle vous devez renouveler votre certificat SSL.
    * <DNT>**Broken links**</DNT>Après avoir testé tous les liens de votre page, il vous indiquera quels liens sont rompus.

    Notre moniteur capture le comportement du site via un moniteur qui interagit avec votre site. Cela vous permet de voir les données mises à jour quotidiennement afin que vous n&apos;ayez jamais à deviner quand les ressources ou les pages ne parviennent pas à se charger.
  </Collapser>

  <Collapser id="track-user-experience" title="Suivre l'expérience utilisateur">
    <img title="View user experience" alt="View your data about how user's experience your site" src="/images/tutorial_screenshot-crop_cwv-wpm.webp" />

    Google PageSpeed signale vos principaux indicateurs Web essentiels ainsi que d&apos;autres mesures collectées par New Relic. Ces données vous donnent des informations détaillées sur la façon dont les utilisateurs vivent votre site.

    * <DNT>**First contentful paint**</DNT>. Cela capture le temps nécessaire au chargement du premier élément sur une page. Dans la capture d&amp;apos;écran ci-dessus, il a fallu 3,22 secondes au site pour que quelque chose se charge sur la page.
    * <DNT>**Largest contentful paint**</DNT>Contrairement à la première peinture de contenu, cela capture le temps nécessaire au chargement complet du contenu le plus volumineux de la page. Par exemple, si des informations sur une campagne importante apparaissent dans l&amp;apos;élément le plus important d&amp;apos;une page, ce nombre nous indique combien de temps un utilisateur attend avant de les voir. Dans la capture d&amp;apos;écran ci-dessus, cela a pris 2,79 s.
    * <DNT>**Interaction to next paint**</DNT>. Lorsqu&amp;apos;un utilisateur interagit avec un élément de votre site, il faut du temps pour traiter l&amp;apos;interaction de l&amp;apos;utilisateur. Ce nombre capture ce temps sous forme de délai entre la demande (interaction utilisateur) et la réponse (comment la page change en fonction de la demande de l&amp;apos;utilisateur).
    * <DNT>**Cumulative layout shift**</DNT>. Cela mesure la stabilité visuelle des différents actifs de votre site. Par exemple, lorsqu&amp;apos;un utilisateur accède à la page principale de votre site, les images peuvent se déplacer pendant le chargement. Ce mouvement est imprévu.

    <img title="Adjust timeslice data" alt="Adjust timeslice data" src="/images/tutorial_screenshot-crop_cwv-time-wpm.gif" />

    Nous collectons des données au fil du temps afin que vous puissiez obtenir une image plus riche des performances de votre site. Vous pouvez zoomer et dézoomer sur des fenêtres temporelles et voir comment vos données évoluent autour d&apos;un incident particulier, comme dans le gif ci-dessus. Lorsque vous avez terminé, cliquez sur le bouton d&apos;horodatage en haut pour revenir à votre période préférée.
  </Collapser>

  <Collapser id="content" title="Comparez la taille du contenu, les temps de chargement et les sources">
    <img title="Analyze content size" alt="Analyze content size" src="/images/tutorial_screenshot-crop_content-wpm.webp" />

    Votre page Web est composée de différents types de contenu, tels que du HTML, des images, du JavaScript et des vidéos. Nos graphiques de contenu vous montrent les performances des différents types de contenu les uns par rapport aux autres. Par exemple:

    * <DNT>**Content size**</DNT> vous montre la taille en kilo-octets de chaque type. En vous référant au tableau ci-dessus, vous pouvez voir que votre type de contenu JavaScript est proportionnellement plus petit que vos images sur une page donnée. Si les performances de votre site sont faibles, vous pouvez demander à votre équipe de développement d&amp;apos;optimiser votre site au niveau du code.
    * <DNT>**Content sources**</DNT> décompose l&amp;apos;endroit où votre contenu est hébergé par taille. Par exemple, dans le graphique ci-dessus, la majeure partie de votre contenu provient de New Relic, représenté par le jaune. Si votre contenu est optimisé mais toujours défectueux sur votre page, modifier la source de votre contenu pourrait résoudre cette erreur.
    * <DNT>**Content load time**</DNT> montre combien de temps il faut à un type de contenu donné pour se charger par rapport au temps de chargement de la page entière.
    * <DNT>**Network**</DNT> décompose la manière dont le contenu est livré sur la page et le temps nécessaire pour livrer ce contenu.

    Si votre page se charge lentement, vous pouvez utiliser la section <DNT>**Content**</DNT> pour diagnostiquer les sources potentielles du problème. Par exemple, comparer <DNT>**Content size**</DNT> à <DNT>**Content load time**</DNT> peut donner des informations détaillées sur le fait que CSS utilise trop de ressources et se charge en dernier, ce qui entraînerait un retard dans la peinture du contenu le plus important. Si la taille du contenu est optimale mais que le chargement de la page est toujours lent, il s&amp;apos;agit peut-être d&amp;apos;un problème de réseau ou de source.
  </Collapser>

  <Collapser id="geo" title="Réparer les liens brisés">
    <img title="Your page's broken links" alt="" src="/images/tutorial_screenshot-crop_broken-links-wpm.webp" />

    Si l&apos;engagement des utilisateurs est en baisse mais que votre site est disponible et que le contenu est optimisé, alors quelque chose d&apos;autre pourrait être en jeu. Un parcours possible pour résoudre le problème pourrait ressembler à ceci :

    1. Allez à <DNT>**[one.newrelic.com &gt; Website performance monitoring](https://one.newrelic.com/website-grader)**</DNT>.
    2. Vérifiez la disponibilité de votre page. Il est indiqué comme disponible.
    3. Vos principaux indicateurs Web Vitals sont signalés comme étant <DNT>**Good**</DNT> dans tous les domaines.
    4. Vous remarquez que les données d&apos;intervalle de temps montrent une forte baisse du trafic la veille, à peu près au moment où votre équipe a effectué des mises à jour sur votre page de destination.
    5. Vous remarquez qu&apos;un lien brisé a été signalé. Il s&apos;agit d&apos;un lien à l&apos;intérieur d&apos;une bannière qui est utilisé sur votre site.

    À ce stade, vous pouvez vous coordonner avec les équipes concernées pour réparer le lien rompu et déployer une mise à jour pour le reste du site.
  </Collapser>
</CollapserGroup>
