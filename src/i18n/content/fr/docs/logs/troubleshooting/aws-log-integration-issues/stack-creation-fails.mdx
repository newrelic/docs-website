## Problème

La création d&apos;une stack CloudFormation peut échouer pour diverses raisons.

## Solution

Raisons possibles et leurs solutions :

* Le rôle IAM que vous avez sélectionné lors de la création stack CloudFormation peut ne pas disposer [des autorisations](https://docs.aws.amazon.com/ARG/latest/userguide/gettingstarted-prereqs-permissions.html) suffisantes.

* Si vous n&apos;avez pas sélectionné de rôle IAM, assurez-vous que votre compte dispose des [autorisations](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#logs) requises.

* Si le message d’échec apparaît comme suit, supprimez ou renommez les ressources existantes avec ce nom et réessayez.

  `Resource type {resource-name} with identifier {resource-identifier-name} already exists.`

* Vérifiez la sortie de la pile CloudFormation imbriquée pour identifier les erreurs potentielles.

* Assurez-vous que les compartiments S3 sélectionnés ne disposent pas de notification d&apos;événement sur aucun événement de création d&apos;objet.

* Assurez-vous que les CloudWatch log groupes sélectionnés n&apos;ont pas de filtres d&apos;abonnement similaires.
