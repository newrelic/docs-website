New Relic présente les restrictions suivantes avec l&apos;intégration AWS pour le transfert de loglors de l&apos;ingestion de logs via S3 ou Firehose :

### Pour ingérer le log AWS à partir de S3 [#s3-restrictions]

* La taille du fichier log doit être inférieure à 80 Mo.
* La taille d&apos;une seule ligne log doit être inférieure à 8 Mo.
* les lignes de log entre 1 Mo et 8 Mo seront divisées en plusieurs messages et transmises à New Relic.
* New Relic ne prend actuellement pas en charge l&apos;analyse JSON ou la ligne log du client.
* New Relic ne prend actuellement en charge que les formats de compression `Gzip` et `Bzip2` pour les fichiers log. Il traite les fichiers dans d’autres formats compressés comme des fichiers non compressés.

### Pour ingérer le log de CloudWatch [#cloudwatch-restrictions]

* New Relic ne prend actuellement pas en charge l&apos;analyse JSON ou la ligne log du client.
* New Relic ne prend actuellement en charge que [les couches Lambda](https://github.com/newrelic/newrelic-lambda-layers) pour l&apos;analyse logique.
