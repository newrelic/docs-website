Configurez le service [Cloudflare Logpush](https://developers.cloudflare.com/logs/get-started/enable-destinations/new-relic/) pour envoyer le log à New Relic à l&apos;aide dashboard Cloudflare ou via l&apos;API.

<Callout variant="important">
  Cloudflare a annoncé une série de mises à jour des champs liés à la sécurité dans le log Cloudflare qui commenceront à être déployées le 1er février 2023 et se termineront le 1er août 2023. Veuillez consulter l&apos; [annonce officielle de Cloudflare](https://developers.cloudflare.com/logs/reference/change-notices/2023-02-01-security-fields-updates/) pour plus de détails. Tout log envoyé de Cloudflare à New Relic sera affecté par ces modifications.
</Callout>

## Activer Logpush sur New Relic [#dashboard]

Pour activer Cloudflare Logpush, suivez les étapes de la documentation Cloudflare en fonction de votre choix de configuration :

* [Activer via le dashboard Cloudflare](https://developers.cloudflare.com/logs/get-started/enable-destinations/new-relic/#manage-via-the-cloudflare-dashboard)
* [Activer via API](https://developers.cloudflare.com/logs/get-started/enable-destinations/new-relic/#manage-via-api)

## Quelle est la prochaine étape ? [#what-next]

Démarrez en quelques minutes avec un dashboard prédéfini pour voir les mesures clés de votre log réseau Cloudflare.

<figcaption>
  Notre Instant Observability quickstart pour le log réseau Cloudflare inclut un dashboard prédéfini.

* Accédez [quickstart de Cloudflare Network Logs](https://newrelic.com/instant-observability/cloudflare/fc2bb0ac-6622-43c6-8c1f-6a4c26ab5434) dans New Relic Instant Observability et cliquez sur <DNT>**+ Install now**</DNT>.
* Pour en savoir plus sur l&apos;intégration, lisez l&apos;article de blog : [Un moyen plus rapide et moins cher d&apos;envoyer des informations détaillées de log à New Relic avec l&apos;intégration Cloudflare Logpush](https://newrelic.com/blog/how-to-relic/cloudflare-log-integration)
* Explorez les données de logging sur votre plateforme avec notre [interface utilisateur de logs](/docs/logs/log-management/ui-data/use-logs-ui/).
* Obtenez une visibilité plus approfondie sur les données de performances de votre application et de votre plateforme en transmettant votre log avec nos capacités [de logs en contexte](/docs/logs/enable-log-management-new-relic/configure-logs-context/configure-logs-context-apm-agents/) .
* Configurer [des alertes](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/).
* [interrogez vos données](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/) et [créez un dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/).

## Désactiver le transfert de log [#disable]

Pour désactiver les fonctionnalités de transfert de log , suivez les procédures standard dans [la documentation Cloudflare Logpush](https://developers.cloudflare.com/logs/faq/). Vous n&apos;avez rien d&apos;autre à faire dans New Relic.
