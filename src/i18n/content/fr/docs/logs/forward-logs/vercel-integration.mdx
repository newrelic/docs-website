Utilisez New Relic pour analyser et stocker le log ingéré de vos projets Vercel et tracer depuis votre fonction serverless.

Vous pouvez installer le [quickstart Vercel](https://newrelic.com/instant-observability/vercel) pour obtenir un dashboard et des alertes prédéfinis pour surveiller les performances de vos fonctions.

<img width="80%;" title="Vercel dashboard" alt="Vercel dashboard" src="/images/logs_screenshot-crop_vercel-dashboard.webp" />

<figcaption>
  Une fois le [quickstart Vercel](https://newrelic.com/instant-observability/vercel) installé, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards &gt; Vercel**</DNT> pour voir votre dashboard Vercel.
</figcaption>

## Prérequis [#pre-req]

Pour déployer sur Vercel, vous devez créer un projet qui regroupera votre déploiement et votre domaine personnalisé. Apprenez à installer et à configurer des projets en lisant la [documentation Vercel](https://vercel.com/docs/concepts/projects/overview).

## Installer le dashboard quickstart de Vercel [#vercel-quickstart]

Après avoir envoyé les données Vercel à New Relic, le quickstart de Vercel est le moyen le plus rapide de passer directement à monitoring de votre log de développement de bout en bout.

Installez le quickstart pour obtenir un dashboard prédéfini et des alertes pour monitorer les performances de vos fonctions. Vous recevrez également une alerte prédéfinie qui se déclenche lorsque le taux d&apos;erreur (erreurs 4xx/5xx) dépasse 1 %. Suivez ces instructions :

1. Accédez au [quickstart de Vercel](https://newrelic.com/instant-observability/vercel) dans New Relic Instant Observability et cliquez sur <DNT>**Install now**</DNT>.
2. Sélectionnez un compte et cliquez sur <DNT>**Begin installation**</DNT>.
3. Si vous avez déjà terminé les [étapes de configuration de Vercel](#configuration), sélectionnez <DNT>**Done**</DNT> pour passer à l&apos;étape suivante.
4. Le quickstart déploie les ressources sur votre compte. Cliquez sur <DNT>**See your data**</DNT> pour accéder au dashboard.

Le Vercel quickstart dashboard visualise toutes vos données clés sur trois pages. Cliquez sur les onglets <DNT>**Overview**</DNT>, <DNT>**Build Logs**</DNT> et <DNT>**Serverless Function**</DNT> pour obtenir un aperçu, analyser le log de build et plonger en profondeur dans les performances sans serveur de votre fonction.

## Facultatif : Obtenir des données de trace de Vercel avec OpenTelemetry [#vercel-opentelemetry]

Vous pouvez utiliser [OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-introduction/) pour importer des données de trace à partir de vos applications exécutées dans les fonctions Vercel. Pour définir cette option, suivez ces étapes :

1. Après avoir cliqué sur <DNT>**Add integration**</DNT> à [l’étape 5](#step5) de la [configuration Vercel](#configuration), vous verrez la boîte de dialogue des détails de configuration.

2. Accédez à la section <DNT>**Traces (Beta)**</DNT> et activez-la pour activer les traces.

3. Cliquez sur <DNT>**Save changes**</DNT>.

   <img width="50%;" title="Get trace data from Vercel with OpenTelemetry" alt="Get trace data from Vercel with OpenTelemetry" src="/images/logs_screenshot-crop_vercel-get-trace-data.webp" />

4. Après avoir activé la trace, vous les verrez dans New Relic. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; services &gt; (select an app)**</DNT>. Ensuite, sous la section <DNT>**Monitor**</DNT> , cliquez sur <DNT>**Distributed tracing**</DNT>.

## Désactiver le transfert de log [#disable]

Pour désactiver les fonctionnalités de transfert de log , suivez les procédures standard dans la [documentation Vercel](https://vercel.com/docs/concepts/observability/log-drains-overview/log-drains#removing-a-log-drain). Vous n&apos;avez rien d&apos;autre à faire dans New Relic.
