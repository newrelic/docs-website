Configurez le service [webhook CircleCI](https://circleci.com/docs/2.0/webhooks/) pour envoyer le log CI/CD (intégration et livraison continue) à New Relic.

## Configurer un déclencheur de webhook dans CircleCI [#setup]

Suivez ces étapes pour configurer un webhook CircleCI pour transférer votre log vers New Relic :

1. connectez-vous à [CircleCI](https://app.circleci.com/projects).

2. Accédez à l’un de vos projets CircleCI.

3. Cliquez sur <DNT>**Project settings**</DNT>.

4. Dans la barre latérale de vos paramètres de projet, cliquez sur <DNT>**Webhooks**</DNT>.

5. Cliquez sur <DNT>**Add webhook**</DNT>.

6. Nommez votre webhook.

7. Entrez dans <DNT>**Logs endpoint**</DNT> de New Relic :

   * NOUS:

     ```
     "https://log-api.newrelic.com/log/v1?Api-Key=YOUR_LICENSE_KEY"
     ```

   * UE:

     ```
     "https://log-api.eu.newrelic.com/log/v1?Api-Key=YOUR_LICENSE_KEY"
     ```

   Utilisez la région que vous avez définie sur votre compte New Relic. Remplacez `YOUR_LICENSE_KEY` par votre New Relic <InlinePopover type="licenseKey" />.

8. Sélectionnez <DNT>**Workflow**</DNT> ou <DNT>**Job**</DNT>, selon le type d&apos;événement pour lequel vous souhaitez déclencher le logpush.

9. Si vous avez configuré votre API de réception ou votre service tiers, cliquez sur <DNT>**Test ping event**</DNT> pour créer un événement de test.
