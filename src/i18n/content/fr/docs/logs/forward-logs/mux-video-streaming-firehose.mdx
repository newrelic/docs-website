Transférez votre log de streaming vidéo Mux dans New Relic à l&apos;aide de Kinesis Firehose pour obtenir une observabilité complète sur votre streaming vidéo.

## Étapes à suivre pour diffuser des données vidéo Mux vers New Relic [#mux-kinesis-stream]

1. Vous devrez avoir un <InlinePopover type="licenseKey" />prêt.

2. Suivez la [documentation](https://docs.mux.com/guides/system/export-amazon-kinesis-data-streams) pour configurer un flux de données afin de transmettre les données Mux à Amazon Kinesis.

3. Créez un Amazon Kinesis Data Firehose et configurez-le pour envoyer les données Mux dans New Relic. Consultez [notre documentation](/docs/logs/forward-logs/stream-logs-using-kinesis-data-firehose/) pour connaître les étapes à suivre pour configurer un flux Kinesis Firehose.

## Afficher les données log [#find-data]

Si tout est configuré correctement et que vos données sont collectées, vous devriez voir les données log à ces deux endroits :

* Notre [UI de logs](https://one.newrelic.com/launcher/logger.log-launcher)

* Nos [outils de requête NRQL](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data). Par exemple, vous pouvez exécuter une requête comme celle-ci :

  ```sql
  SELECT * FROM Log
  ```

Si aucune donnée n&apos;apparaît après avoir activé nos fonctionnalités <InlinePopover type="logs" />, suivez nos [procédures de dépannage de log standard](/docs/logs/log-management/troubleshooting/no-log-data-appears-ui/).

* Configurer [des alertes](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/).

## Désactiver le transfert de log [#disable]

Pour désactiver les fonctionnalités de transfert de log , suivez les procédures standard dans [la documentation de Kinesis Data Firehose](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html). Vous n&apos;avez rien d&apos;autre à faire dans New Relic.
