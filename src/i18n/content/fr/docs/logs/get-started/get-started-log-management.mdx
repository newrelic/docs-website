À mesure que les applications évoluent vers le cloud, l’architecture microservices devient de plus en plus dispersée, ce qui rend la capacité de monitorer les logs essentielle. New Relic propose une plateforme de gestion des lograpide et évolutive afin que vous puissiez connecter votre log avec le reste de vos données de télémétrie et infrastructure en un seul endroit.

Notre <InlinePopover type="logs" />solution offre une visibilité plus approfondie sur les données de performances des applications et infrastructure (événements, erreurs, traces, etc.) afin de réduire les délais moyens de résolution (MTTR) (MTTR) et de résoudre rapidement les incidents de production.

La gestion des logs fournit un moyen de connecter vos données log avec le reste des données de votre application et infrastructure . Vous pouvez accéder rapidement à la cause profonde des problèmes, sans perdre le contexte en passant d’un outil à l’autre. Par exemple, vous pouvez :

* Obtenez des données pertinentes sur l&apos;ensemble de l&apos;interface utilisateur de New Relic qui fournissent des logs en contexte des problèmes dans vos applications et hôtes.
* Plongez plus profondément dans les données log en utilisant notre interface utilisateur de log et créez des graphiques, des tableaux de bord et des alertes personnalisés.
* Recherchez instantanément dans votre log.
* Résolvez les problèmes de performances sans basculer entre d’autres outils.
* Visualisez tout en un seul endroit.

## Log des rapports [#integrate-logs]

Pour transférer vos données log vers New Relic, vous pouvez utiliser une ou plusieurs de ces options :

* **Utilisez nos <InlinePopover type="apm" />agents pour signaler le log.** Par défaut, nos agents APM font deux choses : ajouter des métadonnées à votre log, ce qui vous donne des logs en contexte (possibilité de voir les données du log à divers endroits pertinents de notre plateforme) et transmettre votre log à New Relic. Il s&apos;agit d&apos;une option populaire pour les équipes DevOps et les petites organisations, car elle vous permet de signaler facilement le log d&apos;application, sans qu&apos;aucune solution tierce supplémentaire ne soit requise. [En savoir plus sur le log APM.](/docs/apm/new-relic-apm/getting-started/get-started-logs-context)

* **[Utilisez notre agent infrastructure pour signaler le log](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent).** Avec notre agent infrastructure , vous pouvez capturer tout log présent sur votre hôte, y compris le log de votre application. Par rapport à l&apos;utilisation d&apos;un agent APM pour générer des rapports de log, cela peut nécessiter un peu plus de configuration, mais vous offre des options beaucoup plus puissantes (par exemple, la possibilité de collecter des attributs personnalisés, ce que vous ne pouvez pas faire avec les agents <InlinePopover type="apm" />).

* **Utiliser des services log tiers.** Nous proposons [une large gamme d&apos;intégrations pour d&apos;autres services log ](/docs/logs/forward-logs/enable-log-management-new-relic), notamment Amazon, Microsoft, Fluentd, Fluent Bit, Kubernetes, Logstash, etc.

* **Les rapports sont enregistrés à l&apos;aide de l&apos; [API de Log](/docs/logs/log-api/introduction-log-api/) ou [du point de terminaison TCP](/docs/logs/log-api/use-tcp-endpoint-forward-logs-new-relic).**

* **Utilisez le OpenTelemetry SDK pour envoyer le log** de vos applications à un , qui peut les transmettre à New Relic via [OpenTelemetry Collector](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/best-practices/opentelemetry-best-practices-logs) OTLP.

Pour plus d&apos;informations sur les options de transfert de loget les cas d&apos;utilisation spécifiques, voir [Transférer le log](/docs/logs/forward-logs/enable-log-management-new-relic).

## Afficher et gérer le log dans l&apos;interface utilisateur [#find-data]

Pour trouver votre log dans New Relic :

Aller à : <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT>. Pour plus de détails sur ce que vous pouvez faire dans l&apos;interface utilisateur, consultez [l&apos;interface utilisateur du log](/docs/logs/ui-data/use-logs-ui).

Vous pouvez également interroger le type de données `Log` . Voici un exemple simple de requête NRQL :

```sql
SELECT * FROM Log
```

Vous pouvez également utiliser NerdGraph, notre API au format GraphQL pour [interroger des données](/docs/apis/nerdgraph/examples/nerdgraph-nrql-tutorial), ou pour [configurer la gestion des log](/docs/apis/nerdgraph/examples/nerdgraph-log-parsing-rules-tutorial).

## Et ensuite ? [#what-next]

Prêt à démarrer avec nos solutions de gestion des log ?

1. Si vous n&apos;en avez pas déjà un, [créez un compte New Relic](https://newrelic.com/signup). C&apos;est gratuit, pour toujours.
2. Configurez l&apos;une de [nos options pour le log de rapports](#integrate-logs).
3. Explorez [les donnéeslog pertinentes sur votre plateforme](/docs/apm/new-relic-apm/getting-started/get-started-logs-context#response-time-example), y compris les erreurs, les traces distribuées, les hôtes, etc.
4. Utilisez notre [interface utilisateur de log](/docs/logs/ui-data/use-logs-ui) comme un workflow simple à travers tous les logs, attributs, modèles, logging en direct et requêtes. Ajoutez des alertes, interrogez vos données et créez un dashboard.
