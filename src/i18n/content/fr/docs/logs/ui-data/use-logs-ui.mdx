Utilisez notre UI de log pour :

* Repérez des modèles intéressants ou significatifs dans votre log.
* Examinez davantage le contexte autour d’une ligne log particulière.
* Explorez et manipulez vos données de logging avec des filtres et des règles d&apos;analyse.
* Interrogez vos données pour créer des visualisations personnalisées ou ajoutez-les au dashboard
* Organisez les données log de votre organisation et optimisez les performances des requêtes avec des partitions de données.
* Configurez des conditions d’alerte pour les problèmes que vous souhaitez éviter.

Pour rester au courant des nouvelles fonctionnalités et améliorations, abonnez-vous à notre [flux RSS pour les notes de sortie du log](/docs/release-notes/logs-release-notes).

## Trouver l&apos;interface utilisateur du log [#find-ui]

Pour trouver l’interface utilisateur du log, à partir de <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, sélectionnez <DNT>**Logs**</DNT>.

## Explorez vos données log [#ui-workflow]

Utilisez la navigation de gauche dans l&apos;interface utilisateur du log comme un workflow simple à travers tous les logs, les attributs, les modèles, le logging en direct et les requêtes. Gérez vos données log en supprimant ou en analysant les données, en créant des partitions de données et en configurant des alertes. hacher ou masquer toutes données sensibles dans votre log avec [expression d&apos;obfuscation et règles](/docs/logs/ui-data/obfuscation-ui). Obtenez plus de détails sur un log spécifique et ses attributs à partir de la navigation centrale.

Pour explorer vos données de logging, suivez ce workflow de base.

<CollapserGroup>
  <Collapser id="workflow-path" title="1. Accédez à l’interface utilisateur du log.">
    Accédez à notre UI de log à l&apos;adresse <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT>.

    Si vous utilisez notre [data centerde la région UE](/docs/using-new-relic/welcome-new-relic/get-started/our-eu-us-region-data-centers/), accédez à <DNT>**[one.eu.newrelic.com](https://one.eu.newrelic.com) &gt; Logs**</DNT>.

    Si vous n&apos;avez pas [personnalisé votre barre de navigation New Relic](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/#custom-navigation), accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, cliquez sur <DNT>**Browse data**</DNT> et sélectionnez <DNT>**Logs**</DNT>.
  </Collapser>

  <Collapser id="workflow-patterns" title="2. Recherchez des modèles.">
    <img title="Logs patterns UI" alt="Screenshot of Log patterns UI" src="/images/logs_screenshot-crop_patterns.webp" />

    * Pour repérer les pics ou les baisses suspects dans le log des messages, cliquez sur <DNT>**[Patterns](/docs/logs/ui-data/find-unusual-logs-log-patterns/)**</DNT> dans le menu de navigation de gauche.
    * Pour consulter le log d&apos;une période spécifique, cliquez sur ce point (ou cliquez et faites glisser une zone) sur le graphique ou utilisez le sélecteur d&apos;intervalle de temps.
  </Collapser>

  <Collapser id="workflow-focus" title="3. Affinez votre champ de vision.">
    Le menu de navigation de gauche comprend des options pour vous aider à affiner la portée de vos résultats de recherche initiaux ou à trouver rapidement une valeur hors norme.

    * Si vous ne savez pas par où commencer, cliquez sur <DNT>**Attributes**</DNT> dans le menu de navigation de gauche, puis sélectionnez des valeurs supplémentaires si nécessaire. Par exemple, si un hôte répertorié sous l&apos;attribut `hostname` génère significativement plus de messages d&apos;erreur que les autres, sélectionnez cette valeur pour l&apos;appliquer à votre recherche.
    * Pour rendre votre message de log plus facile à interroger et à comprendre, utilisez nos [règles d&apos;analyse intégrées](/docs/logs/ui-data/built-log-parsing-rules/) ou [créez vos propres règles d&apos;analyse](/docs/logs/ui-data/parsing/) pour les attributs qui correspondent à une valeur particulière.
    * Pour gérer la quantité de données log collectées et stocker moins de logs, créez [des règles de filtre de suppression](/docs/logs/ui-data/drop-data-drop-filter-rules/) qui évitent de collecter des données dont vous n&apos;avez pas besoin.
  </Collapser>

  <Collapser id="workflow-details" title="4. Examinez les détails log .">
    <img title="Logs details UI" alt="Screenshot of Log details UI including severity level" src="/images/logs_screenshot-crop_log-details.webp" />

    Le cas échéant, les messages de log indiquent un niveau de gravité (`INFO`, `DEBUG`, etc.). Sélectionnez un message de log pour afficher ses détails sous forme de tableau d&apos;attributs ou de JSON.

    * Pour voir quels attributs sont inclus dans un message de log, cliquez sur la ligne log .
    * Pour aider à résoudre les problèmes liés à une valeur spécifique dans les détails log , [affichez le log environnant](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs) pour les détails de l&apos;attribut.
    * Pour obtenir plus de détails dans des messages extrêmement longs, développez les données stockées sous forme [de blobs](/docs/logs/ui-data/long-logs-blobs/).
  </Collapser>

  <Collapser id="workflow-search" title="5. requête et recherche.">
    Par défaut, l&apos;interface utilisateur du log affiche tout votre log, mais vous pouvez également effectuer une recherche avec des mots-clés ou des phrases pour trouver les résultats souhaités. Par exemple:

    ```
    process failed
    ```

    Vous pouvez également utiliser le champ de recherche avec liste déroulante de saisie semi-automatique pour sélectionner un attribut, un opérateur et une valeur. Par exemple:

    ```
    service_name equals my service
    ```

    Pour aider votre requête à se concentrer sur les détails dont vous avez besoin, ajoutez ou supprimez un attribut en procédant de l&apos;une des manières suivantes :

    * Dans le menu de navigation de gauche, sélectionnez <DNT>**Attributes**</DNT>, sélectionnez les valeurs souhaitées, puis cliquez sur <DNT>**Query logs**</DNT>.

    * Dans la table du log, cliquez sur l&apos;icône <DNT>**+**</DNT> pour ajouter un attribut, puis cliquez dessus pour l&apos;ajouter à la requête.

      Pour passer de la barre de recherche de syntaxe Lucene à une requête NRQL, cliquez sur le bouton <DNT>**NRQL**</DNT> à côté de la barre de recherche. Il n&apos;y a pas d&apos;équivalence directe entre NRQL et Lucene, ce qui signifie que la requête NRQL sera souvent plus simple et moins complexe, mais cela vous donne un point de départ utile pour créer une requête avec les mêmes données.

      Pour plus d&apos;informations sur l&apos;interrogation de votre log, consultez la [documentation sur la syntaxe des requêtes de log](/docs/logs/ui-data/query-syntax-logs/).
  </Collapser>

  <Collapser id="query-builder" title="7. Analyse avancée log avec le générateur de requêtes NRDB.">
    <img title="Advanced log analysis with NRDB query builder" alt="Screenshot of link to NRDB query builder" src="/images/logs_screenshot-crop_nrql-button.webp" />

    Vous pouvez écrire des requêtes NRQL sur événement de log sans perdre vos filtres avec le [générateur de requêtes NRDB](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/). Cliquez sur le bouton <DNT>**NRQL**</DNT> à droite de la barre de recherche du log, puis démarrez la requête.
  </Collapser>

  <Collapser id="workflow-related" title="8. Obtenir le log associé.">
    Se concentrer sur le log le plus utile peut vous aider à :

    * <DNT>**Optimizing performance:**</DNT> Pour organiser les données au sein d&apos;un compte et optimiser les performances des requêtes, créez [des règles de partition de données](/docs/logs/ui-data/data-partitions/).
    * <DNT>**Reviewing deployments:**</DNT> Pour voir immédiatement comment votre système réagit au déploiement ou à d&apos;autres modifications de l&apos;application, activez [le logging en direct](/docs/logs/troubleshooting/view-log-messages-real-time-live-tail).
    * <DNT>**Bypassing unrelated details:**</DNT> Pour afficher l&apos;intégralité du log pour une valeur spécifique, examinez la liste des attributs dans <DNT>**Log details**</DNT> pour le log sélectionné, puis continuez à ajouter ou à supprimer des attributs selon vos besoins.
    * <DNT>**Finding the root cause:**</DNT> Pour aider à identifier la cause profonde d&apos;un problème avant qu&apos;il ne se produise ou son impact après un événement, cliquez sur <DNT>**...**</DNT> dans <DNT>**Log details**</DNT> pour [afficher le log environnant](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs).
    * <DNT>**Getting more context:**</DNT> Pour voir les logs liés aux autres données télémétriques de vos applications et hôtes, utilisez [logs en contexte](/docs/logs/logs-context/logs-in-context).
  </Collapser>

  <Collapser id="traces-in-context-overview" title="9. Plongez en profondeur dans les traces distribuées.">
    Une fois que vous avez affiné l&apos;ensemble des logs avec des filtres, puis ouvert un log spécifique, vous pouvez voir les traces associées distribuées. Tant que vous avez configuré le tracing distribué et qu&apos;il existe des échantillons de trace liés au log, vous verrez une option pour les afficher. Il s&apos;agit d&apos;un moyen rapide d&apos;afficher les informations trace sans accéder à la page de tracing distribué principale.

    Si vous ouvrez le volet <DNT>**Log details**</DNT> et voyez une section <DNT>**Distributed trace**</DNT> , vous avez deux options pour afficher la trace de portée dans une vue en cascade :

    * Cliquez directement sur le nom de la trace ou cliquez sur l&apos;icône avec une flèche à droite, ce qui ouvre la vue de focus en cascade qui met en évidence les étendues de trace avec des erreurs.

      <img style={{ align: "left",width: "85%" }} title="Screenshot showing how to reach the log details page" alt="Screenshot showing how to reach the log details page" src="/images/logs_screenshot-crop_traces-in-context.webp" />

    * Vous pouvez également cliquer sur <DNT>**Explore**</DNT> pour ouvrir une cascade non filtrée dans laquelle vous pouvez cliquer sur toutes les travées.

      <Callout variant="tip">
        Si vous avez besoin d’aide pour utiliser la cascade trace , consultez [Comprendre et utiliser l’interface utilisateur de tracing distribué](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui/).
      </Callout>
  </Collapser>

  <Collapser id="share" title="10. Partagez vos découvertes.">
    Utilisez l&apos;une des fonctions principales de l&apos;interface utilisateur de New Relic pour [explorer vos données](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/) (compte spécifique, plage horaire, métriques et événements, générateur de requêtes, etc.) et partagez les données avec des graphiques, ajoutez-les au dashboard, etc. Pour plus d&apos;informations, consultez les [exemples](#examples) dans ce document.
  </Collapser>
</CollapserGroup>

## Exporter [#export]

Pour exporter un sous-ensemble de votre log à partager, l&apos;utilisateur peut exploiter notre fonctionnalité d&apos;exportation dédiée dans l&apos;interface utilisateur du log. Cette fonctionnalité permettra d&apos;exporter jusqu&apos;à 5 000 enregistrements log au format CSV ou JSON. Les enregistrements log seront téléchargés localement sur votre machine dans le dossier `/Downloads/` de votre navigateur actuel.

## Enregistrez vos vues [#saved-views]

Vous pouvez enregistrer votre requête de log, configuration de la table, la plage horaire et le regroupement d&apos;attributs dans une vue enregistrée, afin de pouvoir y revenir rapidement ultérieurement. Pour enregistrer une vue d’analyse log après avoir configuré la vue :

1. À partir de l’une des pages de l’interface utilisateur de la section <DNT>**Your logs**</DNT> , cliquez sur <DNT>**Saved views**</DNT> à droite, puis sur <DNT>**Create new**</DNT>.
2. Donnez un nom à la vue actuelle.
3. Sélectionnez les aspects de la vue actuelle que vous souhaitez enregistrer.
4. Sélectionnez les autorisations d’accès pour les membres de votre équipe.

## Exemples [#examples]

Voici quelques exemples de la manière dont vous pouvez utiliser notre interface utilisateur de log pour obtenir des informations détaillées.

<CollapserGroup>
  <Collapser id="alert" title="Créer une alerte à partir des données log .">
    Vous pouvez créer une condition d&apos;alerte directement dans l&apos;interface utilisateur du log :

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT>.

    2. Recherchez les résultats sur lesquels vous souhaitez générer une alerte ; par exemple, `service_name:"your service" "fatal error"`.

    3. Cliquez sur la cloche <Icon name="fe-bell-off" /> icône à côté de la barre de requête.

    4. Complétez la procédure

    5. Examinez la requête NRQL qui alimentera la condition d’alerte.

       Après avoir enregistré la condition d&apos;alerte, vous pouvez la visualiser dans l&apos;interface utilisateur <InlinePopover type="alerts" />, où vous pouvez apporter des modifications supplémentaires si nécessaire.
  </Collapser>

  <Collapser id="dashboard" title="Ajoutez des graphiques log à un dashboard.">
    À partir de l&apos;interface utilisateur <DNT>**Logs**</DNT>, vous pouvez ajouter des données log sous forme de [widgetde tableau](/docs/query-your-data/explore-query-data/use-charts/chart-types/#widget-table) à un dashboard. Vous pouvez également créer des tables log avec métriques et événement ou le générateur de requêtes dans New Relic.

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Logs**</DNT>.
    2. Sur la droite, cliquez sur <DNT>**Saved views**</DNT> et sélectionnez une vue enregistrée. Ou recherchez les résultats que vous souhaitez tracer ; par exemple, `service_name:"checkout service" "process failed"`.
    3. Cliquez sur <DNT>**Add to dashboard**</DNT>, puis remplissez les détails pour ajouter le graphique log sous forme de tableau à un dashboard existant ou nouveau.

    Ensuite, depuis votre dashboard vous pouvez :

    * Utilisez les dashboard widget fonctions standard , telles que la copie, la modification, la suppression, etc.
    * Cliquez sur n’importe quelle ligne log pour afficher les détails la concernant.
    * Mettez à jour votre requête pour ajouter plus de colonnes.
    * interrogez les données log à partir d&apos;autres comptes disponibles et ajoutez d&apos;autres graphiques (par exemple, en tant que données comparatives) à votre dashboard.
    * Cliquez sur <DNT>**Open in logs**</DNT> pour accéder directement à l’interface utilisateur <DNT>**Logs**</DNT> pour un dépannage supplémentaire.
  </Collapser>

  <Collapser id="troubleshoot-error" title="Résoudre une erreur (logs en contexte).">
    Pour avoir une meilleure compréhension de ce qui se passait sur l&apos;hôte au moment où une erreur s&apos;est produite dans votre application, vous devez pouvoir voir [les logs en contexte](/docs/logs/logs-context/configure-logs-context-apm-agents/). Ensuite, pour résoudre les erreurs associées :

    1. Accédez à <DNT>**APM &gt; Errors inbox**</DNT> ou <DNT>**APM &gt; (select an app) &gt; Events &gt; Error analytics**</DNT> et sélectionnez une trace d’erreur.

    2. Dans les détails de la trace d’erreur, cliquez sur <DNT>**Open in logs**</DNT>.

    3. Parcourez les détails log associé.

    4. Pour identifier l’hôte générant l’erreur, cliquez sur <DNT>**...**</DNT> pour votre choix, puis cliquez sur <DNT>**Show surrounding logs**</DNT>.

       <img title="APM errors inbox UI with logs" alt="Screenshot of APM errors inbox UI with logs" src="/images/logs_screenshot-crop_surrounding-logs.webp" />
  </Collapser>

  <Collapser id="troubleshoot-latency" title="Dépanner la latence (logs en contexte)">
    Pour résoudre le problème de latence de cette manière, vous devez pouvoir voir [les logs en contexte](/docs/logs/logs-context/configure-logs-context-apm-agents/). Ensuite, pour avoir une meilleure compréhension du fonctionnement de votre système lorsque les performances ralentissaient sensiblement :

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; services &gt; (select an app)**</DNT>.

    2. Depuis la page <DNT>**Summary**</DNT> de l’application, cliquez sur <DNT>**Distributed tracing**</DNT>, puis sélectionnez une trace particulièrement lente.

    3. À partir de la trace <DNT>**Details**</DNT>, cliquez sur <DNT>**Logs**</DNT>.

       <img title="APM distributed tracing UI with logs" alt="Screenshot of APM distributed tracing UI with logs" src="/images/logs_screenshot-full_dt-logs.webp" />
  </Collapser>
</CollapserGroup>

<Callout variant="tip">
  Laissez l&apos;agent <InlinePopover type="apm" />de votre application transmettre les données log directement à New Relic avec [les logs en contexte APM](/docs/apm/new-relic-apm/getting-started/get-started-logs-context). Pas besoin d’installer ou de maintenir de logiciel tiers supplémentaire !
</Callout>
