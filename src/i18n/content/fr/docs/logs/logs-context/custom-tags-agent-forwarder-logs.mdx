New Relic APM vous permettent désormais d&apos;opter pour l&apos;ajout de votre balise personnalisée (étiquettes) au log transmis par l&apos;agent. Avec une balise personnalisée sur le log, DevOps et l&apos;ingénieur de plateforme peuvent filtrer, rechercher et corréler les données log pour un dépannage plus rapide et plus efficace, des performances améliorées et une utilisation optimisée des ressources.

## Activer les balises personnalisées (étiquettes) sur le log d&apos;application

Lorsque cette option est activée, les agents de langue New Relic ajoutent toutes les balises personnalisées (étiquettes) au log transféré par l&apos;agent.

Langues prises en charge :

* Versions de l&apos;agent [.NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration) 10.4.0 ou supérieures.
* Versions de l&apos;agent [Ruby](/docs.newrelic.com/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/) 9.16.0 ou supérieures.
* Go: à venir
* Versions d&apos;agent [Java](/docs/apm/agents/java-agent/configuration/java-agent-configuration-config-file/) 8.19.0 ou supérieures.
* [Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration): version de l&apos;agent 12.8.0 ou supérieure.
* Versions d&apos;agent [Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration) 10.3.0 ou supérieures.
* Versions d&apos;agent [PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/) 11.7.0 ou supérieures.
