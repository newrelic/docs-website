New Relic APM vous permettent désormais d'opter pour l'ajout de votre balise personnalisée (étiquettes) au log transmis par l'agent. Avec une balise personnalisée sur le log, DevOps et l'ingénieur de plateforme peuvent filtrer, rechercher et corréler les données log pour un dépannage plus rapide et plus efficace, des performances améliorées et une utilisation optimisée des ressources.

## Activer les balises personnalisées (étiquettes) sur le log d'application

Lorsque cette option est activée, les agents de langue New Relic ajoutent toutes les balises personnalisées (étiquettes) au log transféré par l'agent.

Langues prises en charge :

* Versions de l'agent [.NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration) 10.4.0 ou supérieures.
* Versions de l'agent [Ruby](/docs.newrelic.com/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/) 9.16.0 ou supérieures.
* Go: à venir
* Java : à venir
* [Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration): version de l&amp;apos;agent 12.8.0 ou supérieure.
* Versions d'agent [Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration) 10.3.0 ou supérieures.
* Versions d'agent [PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/) 11.7.0 ou supérieures.
