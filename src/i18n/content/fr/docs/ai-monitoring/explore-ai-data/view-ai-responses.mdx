L&apos;activation de [monitoring de l&apos;IA](/docs/ai-monitoring/intro-to-ai-monitoring) permet à l&apos;agent de reconnaître et de capturer les métriques de performances et trace les données sur la couche IA de votre application. Avec monitoring de l&apos;IA, vous pouvez suivre l&apos;utilisation des jetons, le nombre d&apos;achèvements et les temps de réponse de l&apos;IA à partir de votre application alimentée par l&apos;IA. Lorsque vous voyez une erreur ou une réponse inexacte, vous pouvez accéder à traceune vue au niveau sur une prompt-réponse donnée interaction pour identifier les problèmes dans la logique de votre service d&apos;IA.

<img title="AI responses data" alt="An image that shows the kind of data you get when you enable AI monitoring" src="/images/ai_screenshot-full_intro-ai-ui.webp" />

Vous pouvez consulter vos données en allant sur <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; AI monitoring**</DNT>. Vous pouvez voir vos données à partir de trois pages différentes :

* <DNT>**AI responses**</DNT>: Aperçu des données agrégées de toutes vos entités IA. Suivez vos temps de réponse et vos jetons AI, ou consultez des données sur les invites et les réponses individuelles.
* <DNT>**AI entities**</DNT>: Affichez un tableau récapitulatif de toutes les entités déclarant des données sur l&apos;IA. Voir l&apos;entité avec des données APM standard telles que le taux d&apos;erreur, le débit et le temps de réponse de l&apos;application. Lorsque vous sélectionnez une entité, vous pouvez commencer à explorer la page APM **des réponses d&apos;AI** .

## Page de réponses de l&apos;IA [#ai-responses]

La page <DNT>**AI responses**</DNT> de niveau supérieur affiche vos données d’IA dans leur ensemble. Les données agrégées prennent en compte le total moyen des réponses, le temps de réponse et l&apos;utilisation du jeton par réponse pour toutes les entités déclarant des données d&apos;IA. Sur cette page, la réponse fait référence à une sortie de votre application alimentée par l&apos;IA lorsqu&apos;une prompt vous est envoyée.

Si vous possédez plusieurs applications avec différentes implémentations de différents frameworks d&apos;IA, vous pouvez avoir une idée générale des performances de votre modèle d&apos;IA.

### Suivez le nombre total de réponses, le temps de réponse moyen et l&apos;utilisation du jeton

<img title="AI responses response billboard and time series graphs" alt="A cropped screenshot displaying the time series graphs and billboard info about AI data" src="/images/ai_screenshot-crop_timeseries-billboard.webp" />

Les trois tuiles montrent les mesures générales des performances concernant les réponses de votre IA. Ces tuiles ne vous indiquent peut-être pas la cause exacte d&apos;un problème, mais elles sont utiles pour identifier une anomalie dans les performances de votre application.

<img title="Three tiles to show spikes and drops in response usage" alt="A cropped screenshot displaying billboard info about AI data" src="/images/ai_screenshot-crop_billboard.webp" />

* Si vous constatez une baisse du nombre total de réponses ou une augmentation du temps de réponse moyen, cela peut indiquer qu&apos;une technologie de votre chaîne d&apos;outils d&apos;IA a empêché votre application basée sur l&apos;IA de publier une réponse.
* Une baisse ou une augmentation de l&apos;utilisation moyenne de jetons par réponse peut vous donner des informations détaillées sur la manière dont votre modèle crée une réponse. Peut-être qu&apos;il tire trop de contexte, augmentant ainsi le coût du jeton tout en générant sa réponse. Peut-être que ses réponses sont trop éparses, ce qui conduit à des coûts de jeton inférieurs et à des réponses inutiles.

### Ajuster les graphiques de séries chronologiques

<img title="AI time series graphs" alt="A cropped screenshot displaying time series info about AI data" src="/images/ai_screenshot-crop_Cropped-image-of-AI-timeseries.webp" />

Vous pouvez vous référer aux graphiques de séries chronologiques pour mieux visualiser le moment où un comportement anormal apparaît pour la première fois.

* Ajustez le graphique de la série chronologique en faisant glisser un pic ou en le déposant. Cela limite la série temporelle à une certaine fenêtre temporelle.
* Sélectionnez le menu déroulant pour exécuter une analyse comparative pour différents paramètres de performances. Vous pouvez choisir entre réponses totales, temps de réponse moyen ou jeton moyen par réponse.
* Si vous avez activé la [fonctionnalité de commentaires](/docs/ai-monitoring/customize-agent-ai-monitoring), vous pouvez étendre les graphiques pour analyser les réponses par commentaires positifs et négatifs.

### Évaluer les réponses individuelles de l&apos;IA

Votre table de réponse IA organise les données sur l&apos;interaction entre votre utilisateur final et l&apos;application IA. Vous pouvez voir quand une interaction s&apos;est produite, l&apos;invite associée à leurs réponses, l&apos;achèvement et le nombre de jetons, et quel modèle a reçu une prompt.

<img title="AI response page response table" alt="A cropped screenshot displaying the response table from the AI responses view" src="/images/ai_screenshot-crop_response-table.webp" />

Vous pouvez ajuster les colonnes du tableau en cliquant sur l&apos;icône en forme d&apos;engrenage en haut à droite. Cela vous permet de choisir les types de données que vous souhaitez analyser.

La table de réponse est un point d’entrée permettant de visualiser les données de trace relatives à une réponse individuelle. Cliquez sur une ligne du tableau pour ouvrir la vue de trace d’une réponse particulière.

### AI trace des réponses vue

<img title="AI response trace view" alt="A screenshot of the trace view for a particular AI response" src="/images/ai_screenshot-full_trace-view-ai-response.webp" />

La vue AI trace des réponses vous donne des informations tracesur la façon dont votre application génère des réponses. Vous souhaiterez peut-être consulter la vue de trace pour identifier où une erreur s&apos;est produite, ou peut-être souhaitez-vous comprendre ce qui a conduit à un retour négatif à partir d&apos;une réponse de jeton élevée. À partir de la vue de trace, vous pouvez :

* Choisissez entre trace ou logs. Lorsque vous sélectionnez les logs, interrogez les logs pour les chaînes de texte ou l&apos;attribut que vous souhaitez approfondir.

* Basculer entre les détails de la réponse ou les métadonnées. La colonne Détails de la réponse affiche la prompt de l&apos;utilisateur et la réponse de l&apos;IA afin que vous puissiez conserver le contexte de votre trace et de vos étendues. métadonnées fournit une vue de liste pour GUID, le modèle, le jeton et le fournisseur de l&apos;entité.

* Lorsqu&apos;une erreur se produit, la vue en cascade met en évidence sa ligne en rouge. Sélectionnez la ligne pour ouvrir les données de la plage, y compris les détails de l&apos;erreur de la plage.

## Page d&apos;entité AI [#entities]

La page Entité AI organise toutes vos entités qui signalent actuellement des données AI dans un tableau. Cette page affiche vos applications d&apos;IA ainsi que le temps de réponse, le débit et le taux d&apos;erreur.

<img title="AI entities page" alt="A screenshot of the first page you see when you click AI Monitoring. View aggregated data, compare your AI models, or create drop filters." src="/images/ai_screenshot-crop_AI-entities-page.webp" />

<figcaption>
  Afficher l&apos;entité qui rapporte les données d&apos;IA : Accédez à **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; AI Monitoring**
</figcaption>

La sélection d’une entité IA vous amène à la page de résumé APM pour cette application. À partir du <DNT>**APM summary page**</DNT>, sélectionnez <DNT>**AI monitoring**</DNT> dans la navigation de gauche.

### Page de réponses de l&apos;IA APM [#apm-ai-response]

La sélection d’une entité IA vous amène à la page de résumé de l’APM. Pour trouver vos données d&apos;IA, choisissez <DNT>**AI responses**</DNT> dans le menu de navigation de gauche. Nous vous recommandons d&apos;utiliser cette page lorsque vous avez identifié qu&apos;une entité IA particulière a contribué à une anomalie.

* La version APM des réponses AI contient les mêmes tuiles, graphiques de séries chronologiques et tableaux de réponses collectés que la page des réponses AI de niveau supérieur.
* Au lieu d&apos;afficher des données agrégées, la page de réponses APM AI affiche des données limitées au service que vous avez sélectionné à partir de l&apos;entité AI.
* Alors que la page de réponses de l&apos;IA de niveau supérieur vous permet de filtrer par service sur toutes les entités de l&apos;IA, la page de réponses de l&apos;IA APM limite la fonctionnalité de filtrage à l&apos;attribut propre à l&apos;application.

Pour découvrir comment explorer vos données d’IA, vous pouvez suivre les mêmes modèles expliqués dans la [section précédente sur les réponses de l’IA](#ai-responses).

## Quelle est la prochaine étape ? [#whats-next]

Maintenant que vous savez comment trouver vos données, vous pouvez explorer d&apos;autres fonctionnalités que monitoring de l&apos;IA a à offrir.

* Vous souhaitez analyser les données relatives aux appels effectués sur votre modèle d&apos;IA ? Consultez notre document sur [l&apos;inventaire des modèles et les pages de comparaison des modèles](/docs/ai-monitoring/explore-ai-data/view-model-data).
* Vous êtes préoccupé par des informations sensibles ? [Apprenez à configurer des filtres de dépôt](/docs/ai-monitoring/drop-sensitive-data).
* Si vous souhaitez transmettre des informations de retour d&apos;utilisateur sur les réponses de l&apos;IA de votre application à New Relic, [suivez nos instructions pour mettre à jour le code de votre application afin d&apos;obtenir les commentaires des utilisateurs dans l&apos;UI](/docs/ai-monitoring/customize-agent-ai-monitoring).
