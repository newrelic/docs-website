## Comment fonctionne monitoring de l&apos;IA ? [#how-it-works]

Pour commencer à monitoring l&apos;IA, vous instrument votre application basée sur l&apos;IA avec l&apos;un de nos agents APM. Une fois que vous avez instrumenté votre application, vous pouvez activer monitoring de l&apos;IA afin que l&apos;agent puisse capturer les données d&apos;observabilité LLM.

Lorsque votre assistant IA reçoit une prompt et renvoie une réponse, l&apos;agent capture les données métriques et d&apos;événements générées à partir de LLM externes et de magasins de vecteurs. Notre agent peut :

* Analyser les informations sur l&apos;achèvement, prompt et le jeton de réponse
* Suivez requests et les réponses qui transitent par l&apos;un de nos modèles pris en charge
* Corréler les commentaires négatifs ou positifs à propos d&apos;une réponse de votre utilisateur final

Vous pouvez accéder à toutes ces informations et bien plus encore depuis la plateforme New Relic , puis créer des alertes et des dashboards pour vous aider à gérer efficacement vos données d&apos;IA et à améliorer les performances.

monitoring de l&apos;IA peut vous aider à répondre à des questions critiques sur les performances des applications d&apos;IA : votre utilisateur final attend-il trop longtemps une réponse ? Y a-t-il une augmentation récente de l’utilisation du jeton ? Existe-t-il des modèles de retours d’utilisateurs négatifs autour de certains sujets ? Avec monitoring de l&apos;IA, vous pouvez voir les données spécifiques à la couche IA :

* [Identifiez les erreurs dans les interactions spécifiques prompt et réponses](/docs/ai-monitoring/explore-ai-data/view-ai-responses) à partir du tableau de réponses. Si vous cherchez à améliorer votre modèle d&amp;apos;IA, [apprenez à analyser votre modèle avec des données de niveau trace](/docs/ai-monitoring/explore-ai-data/view-ai-responses/#ai-response-trace-view).
* Si vous utilisez différents modèles dans différents environnements d&apos;application, vous pouvez [comparer le coût et les performances de vos applications avant leur déploiement](/docs/ai-monitoring/view-ai-data/#model-comparison).
* Êtes-vous préoccupé par la conformité des données ? [Découvrez comment créer des filtres de dépôt](/docs/ai-monitoring/drop-sensitive-data) pour supprimer des données sensibles avant de les envoyer à New Relic.

## Démarrer avec monitoring de l&apos;IA [#get-started]

Prêt à commencer ? Assurez-vous de [confirmer que vous pouvez instrument votre bibliothèque ou votre framework d&apos;IA](/docs/ai-monitoring/compatibility-requirements-ai-monitoring). Vous devrez peut-être mettre à jour l&amp;apos;agent si vous avez déjà instrumenté votre application.

Lorsque vous êtes prêt, utilisez notre documentation pour [installer manuellement monitoring de l&apos;IA](/install/ai-monitoring). Ce document vous dirige vers les procédures pertinentes pour l&amp;apos;installation d&amp;apos;un agent APM, puis vous guide dans la configuration de l&amp;apos;agent pour monitoring de l&amp;apos;IA.
