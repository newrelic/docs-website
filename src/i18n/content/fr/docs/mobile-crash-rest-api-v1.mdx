L'API de crash mobile est un moyen RESTful de récupérer des informations sur [les plantages de votre application mobile](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes) qui ont été signalés à New Relic. Il rassemble les données d&amp;apos;attributs et de threads disponibles pour un ensemble donné de types et d&amp;apos;occurrences de crash.

Vous pouvez utiliser l'API pour :

1. GET vos types de crash les plus récents
2. GET une liste d'identifiants d'occurrence à partir d'un seul type d'accident
3. GET Thread, bibliothèque et attribut pour un ID d'occurrence particulier

## Contenu [#in-page-toc]

## Prérequis

Pour utiliser l'API Crash dans ces exemples, vous avez besoin de :

* Votre clé API New Relic Crash. Transmettez cette clé comme valeur de l'en-tête `X-API-KEY` .
* Votre [identifiant de compte](/docs/accounts-partnerships/accounts/account-setup/account-id)New Relic
* Votre monitoring des applications mobiles [identifiant d'application](/docs/apis/rest-api-v2/requirements/finding-product-id#mobile)

Par exemple:

```
https://rpm.newrelic.com/accounts/{account_ID}/mobile/{mobile_application_ID}
```

<Callout variant="tip">
  <DNT>**Note**</DNT>: Les `X-API-KEY`s sont limités à 600 requests par minute.
</Callout>
