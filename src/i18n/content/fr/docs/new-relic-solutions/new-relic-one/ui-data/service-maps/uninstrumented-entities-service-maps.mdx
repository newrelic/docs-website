New Relic peut détecter les lacunes de votre instrumentation en analysant votre télémétrie. Ces lacunes peuvent être des bases de données, des applications ou tout autre type d&apos;entité.

Par exemple, une application pourrait stocker des informations dans une base de données Amazon RDS non instrumentée. Lorsque cela se produit, vous verrez une entité « non instrumentée » reflétant un manque de données dans votre instrumentation. Sur la [carte des services](/docs/new-relic-solutions/new-relic-one/ui-data/service-maps/introduction-service-maps), vous verrez la relation entre cette entité non instrumentée et celle instrumentée, qui dans cet exemple est `mysql.demo.ep.rds.com`.

Les entités non instrumentées et instrumentées ont des apparences différentes. Les entités non instrumentées ont une bordure en pointillés, tandis que les entités instrumentées ont une bordure pleine.

## Instrumenter manuellement une entité [#instrument]

Pour instrumenter une entité non instrumentée à partir de la carte de service :

1. Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, puis sélectionnez <DNT>**APM &amp; services**</DNT>, <DNT>**Mobile**</DNT> ou <DNT>**Browser**</DNT>.
2. Sélectionnez une entité.
3. Sur le côté gauche, cliquez sur <DNT>**Service map**</DNT>.
4. Cliquez sur <DNT>**Instrument what&apos;s missing**</DNT> et suivez les instructions de l’interface utilisateur guidée pour chaque entité.

Une fois ce processus terminé, l’entité aura une bordure solide sur la carte de service.
