Pour vos applications de monitoring dans notre interface utilisateur <InlinePopover type="apm" />, <InlinePopover type="browser" />et monitoring des applications mobiles, la page d&apos;interface utilisateur <DNT>**Dependencies**</DNT> affiche une liste filtrable de toutes les applications, services, bases de données et hôtes que New Relic identifie comme une connexion à cette [entité](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic). Il montre la dépendance en amont et en aval, et fournit des pistes pour les explorer.

Alors que [les cartes de service](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one) affichent une carte à l&apos;échelle de l&apos;entité des relations d&apos;une entité, la page d&apos;interface utilisateur <DNT>**Dependencies**</DNT> affiche vos services en amont et en aval dans un tableau. Il utilise également le même [système de codage couleur utilisé par les cartes de service](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one#alerts) pour vous montrer ce qui fonctionne bien et ce qui ne fonctionne pas.

## Exigences

La page d&apos;interface utilisateur <DNT>**Dependencies**</DNT> est disponible pour le moniteur d&apos;entité par nos agents APM, nos agents mobiles et nos agents de navigateur.

## Voir la dépendance [#access]

Pour afficher la dépendance des applications, services, bases de données et hôtes connectés à une entité :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> et sélectionnez soit <DNT>**APM &amp; services**</DNT>, soit <DNT>**Browser**</DNT>, soit <DNT>**Mobile**</DNT>.
2. Choisissez une entité, puis cliquez sur <DNT>**Dependencies**</DNT>.
3. Pour approfondir, filtrez les applications, les services, la base de données ou les hôtes.

Vous pouvez filtrer la page de dépendance pour afficher des éléments spécifiques qui relèvent de l&apos;entité. la dépendance comprend :

* <DNT>**Services**</DNT>:Applications et services de monitoring [APM](/docs/apm) .
* <DNT>**Mobile applications**</DNT>: vos [applications mobiles](/docs/mobile-monitoring).
* <DNT>**Browser applications**</DNT>: vos [applications de navigateur](/docs/browser) frontales.
* <DNT>**External services**</DNT>: [moniteur de services externes par APM](/docs/apm/applications-menu/monitoring/external-services-page). Les services externes incluent les services hors processus tels que les services Web, les ressources dans le cloud et tout autre appel réseau.
* <DNT>**Databases**</DNT>: la base de données et les données du cache de votre application. les bases de données sont sans agent. De ce fait, les alertes ne peuvent pas être définies pour la base de données, car seul l&apos;appel de service est signalé à New Relic.
* <DNT>**Hosts**</DNT>: votre [infrastructure](/docs/infrastructure) (serveurs et hôtes).
