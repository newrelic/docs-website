La barre de filtre d&apos;entités vous permet de trouver et de filtrer les [entités](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic) monitorées que vous souhaitez examiner dans l&apos;interface de New Relic.

<img title="Entity filter bar in Catalogs" alt="Current entity filter bar interface showing search and filter options" src="/images/Catalogs.webp" />

## Repérez la barre de filtre [#find]

La barre de filtre d&apos;entités s&apos;affiche en haut des principales pages de New Relic, notamment :

* <DNT>**[Catalogs](/docs/service-architecture-intelligence/catalogs/catalogs/)**</DNT> page pour <DNT>**Applications**</DNT>, <DNT>**Repositories**</DNT>, et <DNT>**Infrastructure**</DNT>.
* <DNT>**All Entities**</DNT> page.
* Pages de service APM.
* <InlinePopover type="browser" />pages.
* monitoring des pages d&apos;infrastructure et bien plus encore.

## Recherche et filtrage de base [#basic-filtering]

Commencez par ces méthodes de filtrage fondamentales pour localiser une entité :

### Recherche par nom d&apos;entité

Utilisez le champ <DNT>**Search by entity name**</DNT> pour trouver l&apos;entité :

1. Cliquez dans le champ de recherche.
2. Saisissez une partie du nom d&apos;une entité (par exemple, « production » ou « application web »).
3. Consultez les résultats correspondants qui apparaissent instantanément dans la liste ci-dessous.

<Callout variant="tip">
  La recherche correspond à n&apos;importe quelle partie du nom de l&apos;entité ; ainsi, saisir « prod » permettra de trouver les entités portant des noms tels que « prod-web-server » et « my-production-app ».
</Callout>

### Filtrer par type d&apos;entité

Affinez vos résultats en sélectionnant des types d&apos;entités spécifiques :

* <DNT>**Entity Type**</DNT>Choisissez parmi les applications, les services, les hôtes, les conteneurs et autres types de ressources monitorées.
* **Résultat :** Seules les entités correspondant au type sélectionné s&apos;afficheront, ce qui facilite la concentration sur des composants infrastructure spécifiques.

### Filtrer par propriétaire de l&apos;équipe

Concentrez-vous sur l&apos;entité gérée par votre équipe :

* <DNT>**Team**</DNT>Sélectionnez les équipes qui sont propriétaires de l&apos;entité dans votre vue actuelle.
* **Exigence :** Vous devez activer [les services d&apos;architecture intelligente](/docs/service-architecture-intelligence/getting-started/).
* **Résultat :** Afficher uniquement l&apos;entité attribuée à ou détenue par l&apos;équipe sélectionnée.

## Filtrage avancé [#advanced-filtering]

Pour des scénarios de filtrage plus complexes, créez des filtres personnalisés :

### Créer des filtres d&apos;attributs personnalisés

Ajouter des filtres spécifiques basés sur l&apos;attribut de l&apos;entité :

1. Cliquez sur le bouton **+** à côté des filtres existants.

2. Dans la liste <DNT>**Filter**</DNT>, sélectionnez un attribut (tel que l&apos;environnement, la région ou une étiquette personnalisée).

3. Choisissez un opérateur :

   * **=** (égal à) : Correspondance exacte
   * **!=** (différent de) : Exclure les correspondances exactes
   * **LIKE**: Contient le texte spécifié
   * **NOT LIKE**: Exclut le texte spécifié

4. Saisissez ou sélectionnez une valeur pour compléter le filtre.

5. Cliquez sur <DNT>**Add**</DNT> pour appliquer le filtre.

**Exemple :** Pour trouver toutes les applications Web en production, vous pouvez filtrer où `environment = production` ET `entity_type = APPLICATION`.

Pour des requêtes et des analyses plus avancées, envisagez d&apos;utiliser [le générateur de requêtes NRQL](https://one.newrelic.com/data-exploration/query-builder). Pour plus d&apos;informations sur les requêtes NRQL, reportez-vous à la section [Comment utiliser NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-how-nrql-works).
