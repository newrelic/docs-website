Notre fonctionnalité de charge de travail vous donne la possibilité de regrouper et de monitorer les entités comme vous le souhaitez. Cela vous donne le pouvoir d&apos;organiser vos données télémétriques de la manière la plus judicieuse pour votre entreprise.

## Qu&apos;est-ce qu&apos;une workload? [#definition]

New Relic monitore une large gamme d&apos;entités et de données, des applications côté client et des API backend , jusqu&apos;à l&apos;infrastructure sous-jacente. Pour donner un sens à ce grand ensemble de données, nous vous donnons la possibilité de créer et de monitorer la charge de travail.

Notre fonctionnalité <DNT>**workloads**</DNT> vous permet de regrouper et de monitorer les entités de la manière qui vous convient. Par exemple, vous pouvez créer une workload pour une unité commerciale ou une équipe spécifique. Notre interface utilisateur de charge de travail fournit une vue agrégée de la santé et de l&apos;activité de l&apos;entité dans cette workload. Ainsi, vous pouvez mieux comprendre comment fonctionne votre logique métier, du frontend aux services backend , sur l&apos;ensemble de votre stack.

Une workload peut inclure :

* Toute entité monitorée avec New Relic, y compris les applications, les applications de navigateur, les applications mobiles, la base de données et les hôtes.
* [Dashboards](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards).
* Autre charge de travail : ceci est utile pour les équipes complexes qui doivent diviser et superposer la charge de travail.

Voici quelques exemples d’éléments qu’une organisation pourrait avoir dans sa workload:

* Une application sans serveur qui comprend une passerelle API, quelques fonctions sans serveur et une base de données et de stockage gérée.
* Une application de navigateur et les API backend qui la prennent en charge.
* Une collection de microservices Java et l’infrastructure sur laquelle ils s’exécutent.
