  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

La charge de travail intelligente suit automatiquement toutes les entités de votre flux de transaction à l&apos;aide du traçage distribué. Contrairement à une [charge de travail standard](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads), une charge de travail intelligente utilise le traçage distribué pour découvrir des entités et met à jour la composition toutes les 5 minutes à mesure que votre système évolue.

## Avant de commencer [#prereqs]

Assurez-vous de respecter les [prérequis des workloads intelligents](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview#intelligent-workloads).

<Callout variant="tip">
  Si vous débutez avec les charges de travail, consultez [Présentation des charges de travail](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview) pour comprendre quand utiliser une charge de travail Intelligent ou Standard.
</Callout>

## Créer des charges de travail intelligentes [#create]

Vous pouvez créer une charge de travail intelligente de trois façons différentes. Choisissez la méthode la plus adaptée à votre workflow :

<CollapserGroup>
  <Collapser id="create-from-recommendations" title="Créer à partir des charges de travail recommandées">
    New Relic analyse automatiquement les transactions de votre application et recommande de créer des Intelligent Workloads à forte valeur ajoutée. C&apos;est le moyen le plus rapide de démarrer avec Intelligent Workloads.

    ### Afficher les recommandations

    Lorsque vous accédez à la page Workloads, vous verrez une bannière en haut si New Relic a trouvé des workloads recommandés pour vous :

    ```
    We found [X] recommended Intelligent Workloads. We made it easy to review our recommendations and set up Intelligent Workloads quickly.
    ```

    Cliquez sur <DNT>**Review recommendations**</DNT> pour ouvrir la modale de recommandations.

    ### Examiner et créer à partir des recommandations

    La modale **Recommended Intelligent Workloads** affiche les transactions qui bénéficieraient d&apos;un monitoring :

    1. **Filtrer par catégorie**: Utilisez les onglets pour basculer entre :

       * **Key Transactions**: Transactions à fort trafic ou critiques pour l&apos;entreprise
       * **Impacted Transactions**: Transactions avec des problèmes de performance ou des erreurs

    2. **Trier les recommandations**: Utilisez le menu déroulant pour trier par :

       * Taux d&apos;erreur le plus élevé
       * Temps de réponse moyen le plus lent
       * Les plus chronophages

    3. **Examiner les détails de la transaction**: chaque recommandation indique :

       * **Transaction name**: La transaction focale de la charge de travail
       * **Focal entity**: Le service principal traitant la transaction
       * **Response Time**: Latence moyenne (par ex. &quot;0.57s&quot;)
       * **Error Rate**: Pourcentage de requêtes échouées (par ex. &quot;0,6 %&quot;)
       * **Total Duration**: Temps de traitement combiné (par ex. &quot;77.60s&quot;)

    4. **Créer une charge de travail**: Cliquez sur <DNT>**Create**</DNT> à côté de toute recommandation

    Lorsque vous créez à partir d&apos;une recommandation, New Relic préremplit automatiquement la sélection de l&apos;entité et de la transaction, vous dirigeant directement vers l&apos;étape **Add details** où vous pouvez :

    * Nommez votre charge de travail
    * Ajouter une description (facultatif)
    * Vérifier le service focal et la transaction
    * Terminez la configuration

    <Callout variant="tip">
      La création à partir de recommandations est idéale si vous débutez avec Intelligent Workloads ou si vous souhaitez identifier rapidement les transactions qui bénéficieraient le plus d&apos;un monitoring dédié.
    </Callout>
  </Collapser>

  <Collapser id="create-workloads-page" title="Créer depuis la page Workload">
    La page Workload propose un assistant de configuration guidé pour vous aider à créer une charge de travail intelligente :

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Workloads**</DNT>.
    2. Cliquez sur <DNT>**Create a workload**</DNT>.
    3. Sélectionnez le type de <DNT>**Intelligent Workload**</DNT>.
    4. Cliquez sur <DNT>**Setup**</DNT> pour lancer l&apos;assistant.

    <img title="Select Intelligent Workload type" alt="Screenshot showing workload type selection with Standard and Intelligent options" src="/images/create_intelligent_workload.webp" />

    ### Étape 1 : Sélectionnez une entité [#select-entity]

    Sélectionnez le service ou l&apos;entité qui contient votre transaction critique :

    1. Recherchez ou parcourez l&apos;entité par nom.
    2. Filtrez par type d&apos;entité si nécessaire (services APM, applications de navigateur, etc.).
    3. Sélectionnez l&apos;entité qui traite votre transaction clé.
    4. Cliquez sur <DNT>**Next**</DNT>.

    <Callout variant="tip">
      Choisissez une entité centrale pour votre flux de transactions. Par exemple, si vous monitorez un processus de paiement, sélectionnez le service de paiement plutôt qu&apos;un service de logging périphérique.
    </Callout>

    ### Étape 2 : Choisissez votre transaction [#pick-transaction]

    Sélectionnez la transaction spécifique que vous souhaitez monitorer :

    1. Parcourir la liste des transactions pour l&apos;entité sélectionnée
    2. Utilisez la recherche pour trouver des noms de transaction spécifiques
    3. Sélectionnez la transaction qui représente votre workflow métier
    4. Cliquez <DNT>**Next**</DNT>

    La charge de travail suivra toutes les entités participant à cette transaction en se basant sur les traces distribuées.

    ### Étape 3 : Ajouter des détails [#add-details]

    Fournissez des informations sur votre charge de travail :

    1. **Name**: Saisissez un nom descriptif (par ex. &quot;Transaction de paiement&quot;, &quot;Flux d&apos;authentification utilisateur&quot;)
    2. **Description** (facultatif) : Ajoutez du contexte sur ce que cette charge de travail monitore et pourquoi elle est importante.
    3. **Account**: Sélectionnez le compte dans lequel la charge de travail sera créée
    4. Cliquez <DNT>**Next**</DNT>

    ### Étape 4 : Prévisualiser et finaliser [#preview-finalize]

    Vérifiez la configuration de votre charge de travail :

    1. Vérifiez que le **service Focal** est correct
    2. Confirmez que la **transaction Focal** correspond à votre workflow prévu
    3. Vérifiez le **Name** et la **Description**de la workload
    4. Cliquez <DNT>**Create Workload**</DNT>

    Votre charge de travail intelligente commencera immédiatement à suivre les entités participant à la transaction.
  </Collapser>

  <Collapser id="create-from-t360" title="Créer à partir de Transaction 360">
    Si vous visualisez déjà une transaction dans Transaction 360, vous pouvez rapidement l&apos;enregistrer en tant que charge de travail intelligente :

    1. Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services**</DNT>
    2. Sélectionnez un service faisant partie de votre flux de transaction critique

    <img title="Select entity for Intelligent Workload" alt="Screenshot showing entity selection step in Intelligent Workload creation" src="/images/intelligent-workload-select-entity.webp" />

    3. Accédez à une transaction spécifique que vous souhaitez monitorer et cliquez sur <DNT>**Transaction 360**</DNT>.
    4. Cliquez sur <DNT>**Save as intelligent workload**</DNT>.
    5. Saisissez un nom descriptif pour la charge de travail.
    6. Cliquez sur <DNT>**Save**</DNT>.

    <Callout variant="tip">
      Choisissez des noms de transaction qui représentent des workflows métier complets (par ex. « checkout », « user-authentication ») plutôt que des appels d&apos;API individuels.
    </Callout>
  </Collapser>
</CollapserGroup>
