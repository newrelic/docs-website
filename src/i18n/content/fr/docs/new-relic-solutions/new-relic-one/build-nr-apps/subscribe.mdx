Abonnez-vous à un Nerdpack pour l&apos;utiliser sur la plateforme New Relic. Désabonnez-vous des Nerdpacks que vous n&apos;utilisez plus.

## Mettez à jour votre CLI

Avant d’exécuter des commandes, assurez-vous que vous disposez de la dernière version de la CLI :

```shell
nr1 update
```

## Vérifiez vos autorisations

Pour souscrire à un Nerdpack, il vous faut :

* Accéder au compte qui l&apos;a publié
* Les [autorisations nécessaires pour gérer les Nerdpacks](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps)

## Publiez votre Nerdpack

Vous devez publier les Nerdpacks que vous créez avant de pouvoir vous y abonner :

1. [Publiez votre Nerdpack](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish)
2. [taguez votre Nerdpack](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) s&apos;il est publié et n&apos;a pas encore de tag

## Abonnez-vous à un Nerdpack

Abonnez-vous à un Nerdpack publié, avec l&apos;interface utilisateur Web ou la CLI `nr1` , pour l&apos;utiliser dans votre compte. L&apos;abonnement vous permet de sélectionner la version de balise que vous souhaitez utiliser ainsi que le compte avec lequel vous souhaitez vous abonner. Cela vous permet de contrôler les Nerdpacks que vous utilisez dans votre compte.

<Callout variant="tip">
  Lorsque le tag auquel vous êtes abonné reçoit une mise à jour, New Relic met automatiquement à jour votre Nerdpack.
</Callout>

<Steps>
  <Step>
    Depuis la page d’accueil [de New Relic](https://one.newrelic.com) , accédez à <DNT>**+ Integrations &amp; Agents**</DNT> dans la barre de navigation supérieure.
  </Step>

  <Step>
    Trouvez votre Nerdpack publié en utilisant le filtre des applications ou la barre de recherche. Tous les Nerdpacks publiés sont présentés ici. Cependant, celui que vous recherchez est peut-être filtré. Ajustez le filtre pour trouver le Nerdpack que vous recherchez.
  </Step>

  <Step>
    Sélectionnez le Nerdpack que vous souhaitez ajouter à votre compte New Relic.
  </Step>

  <Step>
    Cliquez sur **Add this app**. Ce bouton affiche **Manage access** si vous êtes déjà abonné au Nerdpack.
  </Step>

  <Step>
    Sélectionnez le compte avec lequel vous abonner et la version de la balise à laquelle vous abonner. Ensuite, mettez à jour votre/vos compte(s).

    <Callout variant="tip">
      Dans l&apos;interface utilisateur Web et à certains endroits de la CLI, nous faisons référence aux **balises** sous le nom de **canaux**. Il s&apos;agit d&apos;une relique d&apos;une implémentation passée du versionnage de Nerdpack. En ignorant les détails historiques et de mise en œuvre, considérez-les comme des synonymes.
    </Callout>

    Vous et les membres des comptes auxquels vous êtes abonnés pouvez désormais lancer le Nerdpack depuis New Relic.
  </Step>

  <Step>
    Si vous êtes abonné à une application, recherchez-la sous **Vos applications**. Si vous êtes abonné à une visualisation, recherchez-la sous **Visualisations personnalisées**.
  </Step>
</Steps>

## S&apos;abonner depuis la ligne de commande

Si vous avez publié votre propre Nerdpack, il peut être plus rapide de vous abonner à votre Nerdpack avec la CLI `nr1` .

<Steps>
  <Step>
    Accédez au répertoire racine de votre Nerdpack :

    ```shell
    cd my-awesome-nerdpack
    ```

    Ces instructions font référence à votre Nerdpack sous le nom `my-awesome-nerdpack`.
  </Step>

  <Step>
    Assurez-vous que votre Nerdpack a été publié et taguez :

    ```shell
    nr1 nerdpack:info
    [output] Id: {purple}ab123c45-678d-9012-efg3-45hi6jkl7890
    [output] Region: {muted}us
    [output] Account ID: 1234567
    [output] Local version: 1.0.0
    [output] Subscription Model: OWNER_AND_ALLOWED
    [output] Version Count: 1
    [output]
    [output] Version  Date               Tags
    [output] -------  -----------------  ------
    [output] 1.0.0    {muted}a few seconds ago  {green}STABLE
    ```

    Vous devriez voir au moins une version [publiée](/docs/new-relic-solutions/new-relic-one/build-nr-apps/publish) avec une [balise](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag). Il n’est pas nécessaire que ce soit `STABLE`. Vous pouvez vous abonner à n&apos;importe quelle version de tag.
  </Step>

  <Step>
    Abonnez-vous à votre Nerdpack :

    ```shell
    nr1 nerdpack:subscribe --channel=STABLE
    [output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}STABLE{normal} channel
    ```

    Ici, vous vous êtes abonné au tag `STABLE` . Vous pouvez également vous abonner aux balises `DEV` et `BETA` à partir de la CLI, mais la commande abonnement n&apos;autorise pas les balises arbitraires. Pour vous abonner à une autre version de balise, vous devez utiliser l&apos;interface utilisateur Web.

    Semblable à l&apos;interface utilisateur Web, le terme **canal** `nr1 nerdpack:subscribe` est une référence à une ancienne implémentation des versions de Nerdpack.

    <Callout variant="tip" title="Détails techniques">
      `nr1 nerdpack:subscribe` est un alias pour `nr1 subscription:set`.

      Lisez la documentation [`nr1 subscription:set`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionset) pour en savoir plus. Ou, consultez la page d&apos;aide `nr1` :

      ```shell
      nr1 subscription:set --help
      ```
    </Callout>
  </Step>
</Steps>

## Se désabonner d&apos;un Nerdpack

Désabonnez-vous d&apos;un Nerdpack pour le supprimer de votre compte.

<Steps>
  <Step>
    Depuis [New Relic](https://one.newrelic.com), accédez à <DNT>**Apps**</DNT> dans la barre de navigation supérieure.
  </Step>

  <Step>
    Cliquez sur l&apos;icône « i » à côté du Nerdpack auquel vous êtes abonné. Cela ouvre la page de détails.
  </Step>

  <Step>
    Cliquez sur **Manage access**.
  </Step>

  <Step>
    Désélectionnez les comptes dont vous souhaitez vous désabonner. Ensuite, mettez à jour ces comptes.
  </Step>
</Steps>

## Se désabonner en utilisant la ligne de commande

Désabonnez-vous rapidement de votre profil CLI de Nerdpacks.

<Steps>
  <Step>
    Accédez au répertoire racine de votre Nerdpack :

    ```shell
    cd my-awesome-nerdpack
    ```
  </Step>

  <Step>
    Afficher l&apos;abonnement de votre profil :

    ```shell
    nr1 subscription:list
    [output] {muted}Listing subscribed deployed nerdpacks on your account:
    [output] Version  UUID                                  Name              Date         User
    [output] -------  ------------------------------------  ----------------  -----------  -------
    [output] 0.6.4    {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal}  Account Maturity  {muted}a month ago{normal}  2774472
    ```
  </Step>

  <Step>
    Désabonnez-vous du Nerdpack avec l&apos;UUID :

    ```shell
    nr1 nerdpack:unsubscribe --nerdpack-id=384de6e5-8e57-4d56-9d13-299ee7ae8641
    [output] Unsubscribed account {green}1234567{normal} from the nerdpack {purple}384de6e5-8e57-4d56-9d13-299ee7ae8641{normal} on the {green}STABLE{normal} channel.
    ```

    <Callout variant="tip">
      `nr1 nerdpack:unsubscribe` est un alias pour `nr1 subscription:unset`.

      Lisez la documentation [`nr1 subscription:unset`](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-subscription/#nr1-subscriptionunset) pour en savoir plus. Ou, consultez la page d&apos;aide `nr1` :

      ```sh
      nr1 subscription:unset --help
      ```
    </Callout>
  </Step>
</Steps>

## Résoudre les problèmes d&apos;accès à votre Nerdpack publié

Parfois, lorsque vous vous abonnez à un Nerdpack, vous pouvez rencontrer des problèmes. Si vous ne parvenez pas à vous abonner ou si vous voyez un abonnement en double sur votre compte, envisagez certaines solutions courantes pour résoudre ces problèmes.

### étiquetez votre version Nerdpack

Lorsque vous vous abonnez à un Nerdpack, vous choisissez un tag spécifique. Donc, si le Nerdpack auquel vous essayez de vous abonner n&apos;est pas étiqueté, ou n&apos;a pas le tag que vous ciblez, [tag](/docs/new-relic-solutions/new-relic-one/build-nr-apps/tag) votre version de sortie :

```shell
nr1 nerdpack:subscribe --channel=DEV
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}DEV{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
[output]
nr1 nerdpack:tag --tag=DEV
[output] {success}✔  Tagged {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} version {green}1.0.0{normal} as {green}DEV{normal}.
[output]
nr1 nerdpack:subscribe --channel=DEV
[output] Subscribed account {green}1234567{normal} to the nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} on the {green}DEV{normal} channel.
```

Si vous vous abonnez sans argument `--channel` , le canal par défaut est `STABLE`, assurez-vous donc d&apos;avoir une version `STABLE` dans ce cas :

```shell
nr1 nerdpack:subscribe
[output] {error}›{normal}   Error: Nerdpack {purple}ab123c45-678d-9012-efg3-45hi6jkl7890{normal} is not deployed to channel {green}STABLE{normal}. You have to deploy it first
[output] {error}›{normal}   Code: PACKAGE_NOT_DEPLOYED
```

### Abonnez-vous avec l&apos;interface Web

Lorsque vous vous abonnez avec la CLI, vous passez un argument `--channel` pour sélectionner une tag. Une **chaîne** est, à toutes fins utiles, une **tag**. Ce terme est une relique d&apos;une implémentation passée du versionnage de Nerdpack.

Cependant, bien que les termes soient synonymes, l’utilisation `--channel` est limitée dans la CLI. Lorsque vous vous abonnez avec un `--channel`, vous ne pouvez choisir qu&apos;entre `DEV`, `BETA` et `STABLE`. Donc, pour vous abonner à un tag différent, utilisez l&apos;interface Web :

```shell
nr1 nerdpack:subscribe --channel=QA
[output] {error}›{normal}   Error: Expected --channel=QA to be one of: {green}DEV{normal}, {green}BETA{normal}, {green}STABLE
[output] {error}›{normal}   See more help with --help
[output] {error}›{normal}   Code: UNKNOWN
```
