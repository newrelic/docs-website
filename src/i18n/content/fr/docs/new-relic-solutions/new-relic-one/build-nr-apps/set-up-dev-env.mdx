Si vous avez décidé de créer une application personnalisée ou de modifier l&apos;une de nos applications open source, vous avez besoin de quelques outils essentiels :

* L&apos;[interface de ligne de commande (CLI) de New Relic](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==).
* Une clé API, que vous obtenez lorsque vous téléchargez la CLI.

En fonction de ce que vous souhaitez faire avec votre application, vous devrez peut-être effectuer des réglages et des configuration supplémentaires. Ce guide couvre :

* Téléchargement de la CLI New Relic pour créer ou modifier des applications
* Contribuer au contenu de ce site Web

## Avant de commencer

Avant de commencer, nous vous recommandons de lire d’abord ce qui concerne [les autorisations](/docs/new-relic-solutions/new-relic-one/build-nr-apps/permission-manage-apps).

Pour commencer à construire, vous devez avoir :

* Un compte [github](https://github.com/) . Bien que cela ne soit pas strictement nécessaire pour créer des applications, un compte GitHub vous permet de [télécharger et de personnaliser nos applications open source](https://opensource.newrelic.com/explore-projects) et de contribuer à un [projet open source](https://opensource.newrelic.com/contributing/).
* Un [compte développeurs New Relic](https://newrelic.com/signup/?partner=Developer+Edition). Si vous n&amp;apos;en avez pas déjà un, vous pouvez obtenir un compte d&amp;apos;essai gratuit pour développer des applications New Relic.
* [npm](https://www.npmjs.com/get-npm). Si vous avez installé [Node.js](https://nodejs.org/en/download/), vous disposez déjà de `npm`, qui permet de partager, réutiliser et mettre à jour le code JavaScript. Ceci est nécessaire pour travailler avec les composants React qui constituent le framework des applications New Relic et de ce site Web.

### Une note sur le support

Créer une application New Relic est identique à créer n&apos;importe quelle application JavaScript/React. Nous offrons un support pour vous aider avec nos outils de construction[ (notre](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==) [bibliothèque](/docs/new-relic-solutions/build-nr-ui/sdk-component/intro-to-sdk) CLI et SDK). Cependant, nous n&amp;apos;offrons pas d&amp;apos;assistance pour les questions ou problèmes de codage JavaScript ou React de base.

Pour les questions et réponses courantes sur la construction, consultez la [page Explorers Hub sur la construction sur New Relic](https://discuss.newrelic.com/c/build-on-new-relic).

<Callout variant="tip">
  Utilisez l’ [extension New Relic VSCode](https://marketplace.visualstudio.com/items?itemName=new-relic.nr1) ou le [pack d’extension New Relic VSCode](https://marketplace.visualstudio.com/items?itemName=new-relic.new-relic-extension-pack) pour créer vos applications.
</Callout>

## Préparez-vous à créer ou à modifier des applications

<Steps>
  <Step>
    ### Téléchargez la CLI et la clé API

    Sur la [page Build New Relic applications](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==), suivez les étapes du **Quick start** pour télécharger.

    Ces six étapes de démarrage rapide vous permettent d&apos;obtenir une clé API à utiliser pour le développement d&apos;applications et la CLI New Relic pour créer et déployer des applications. À la fin du démarrage rapide, vous disposez d&apos;un projet composé des éléments suivants :

    * Un **Nerdpack**. Le package contenant tous les fichiers requis par votre application. Il contient deux types de fichiers que vous personnalisez pour créer votre application : les Nerdlets et le lanceur.
    * Un ou plusieurs fichiers **Nerdlet** . Une vue ou une fenêtre d&amp;apos;interface utilisateur spécifique. Un Nerdlet est un package JavaScript React qui comprend un fichier `index.js` , un `stylesheet` et un fichier `config` au format JSON. Il peut contenir n&amp;apos;importe quelle fonctionnalité JS (graphiques, champs interactifs, info-bulles, etc.).
    * Un fichier `launcher` : il s&amp;apos;agit de la base du lanceur, qui est utilisé pour ouvrir votre application à partir de New Relic après avoir publié votre application.
  </Step>

  <Step>
    ### Commencer à construire

    Si vous êtes prêt à coder, `cd` dans votre Nerdpack et commencez.

    Si vous souhaitez en savoir plus sur la création d’applications, essayez ces guides étape par étape :

    * [Créer une application « Hello, World ! »](/docs/new-relic-solutions/tutorials/build-hello-world-app) montre comment créer une petite application, la publier sur New Relic et la partager avec d&amp;apos;autres en y abonnant des comptes.
    * [La carte des pages vues par région](/docs/new-relic-solutions/tutorials/map-pageviews-region/) vous guide à travers les étapes de création de l&amp;apos;une de nos applications open source populaires. Vous apprenez à ajouter une requête personnalisée à une application et à l&amp;apos;afficher dans un tableau, puis à ajouter ces données à une carte.
  </Step>
</Steps>

## Contribuer au site `developer.newrelic.com`

Ce site est open source et nous aimerions avoir votre avis. Créez une demande de tirage si vous voyez une erreur que vous savez comment corriger. Envoyez-nous un problème GitHub si vous voyez des lacunes de contenu sur lesquelles vous souhaitez que nous travaillions. Ou rédigez un tout nouveau guide si vous en avez un que vous souhaitez partager. Lisez la suite pour savoir comment.

<Steps>
  <Step>
    ### Forkez le référentiel GitHub `developer-website` (dépôt)

    Le forking du référentiel vous permet de travailler sur votre propre copie des fichiers `developer.newrelic.com` , et [de construire le site localement](https://github.com/newrelic/developer-website#developernewreliccom). Cela nous permet également de gérer plus facilement les demandes de tirage entrantes.

    1. Accédez à la page [du site Web des développeurs](https://github.com/newrelic/developer-website) dans GitHub.
    2. Sélectionnez le bouton **Fork** en haut à droite de la page.
    3. Choisissez le compte vers lequel vous souhaitez effectuer le fork et attendez quelques secondes pendant que le fork est créé.

    [Synchronisez régulièrement](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/syncing-a-fork) pour maintenir votre fork à jour avec les modifications et les ajouts apportés à la branche principale en amont.
  </Step>

  <Step>
    ### Faire une demande de fonctionnalité ou de documentation

    1. Sur n’importe quelle page, sélectionnez le bouton **GitHub** en haut de la page.
    2. Sélectionnez le type de changement que vous souhaitez.
    3. Remplissez le formulaire GitHub.
  </Step>

  <Step>
    ### Contribuer à un nouveau guide

    Consultez nos [directives pour les contributeurs](https://github.com/newrelic/developer-website/blob/master/CONTRIBUTING.md#creating-new-pages), qui vous guideront tout au long du processus.
  </Step>
</Steps>
