La plateforme d&apos;observabilité New Relic offre la flexibilité de monitorer n&apos;importe quelle partie de votre application ou de votre stack. Ceci est rendu possible grâce à la puissance des agents New Relic, qui collectent vos données télémétriques et les envoient à New Relic. Comme tout autre logiciel, vous devez prendre soin de vos agents, ce qui signifie les maintenir à jour.

Pour plus d&apos;informations sur les versions des agents, reportez-vous aux [notes de sortie](/docs/release-notes). Pour télécharger et installer la dernière version, consultez la documentation de cet agent.

## Pourquoi mettre à jour vos agents

Il y a trois raisons principales pour lesquelles vous devez tenir votre agent au courant :

* <DNT>**Security:**</DNT> Lorsque nous trouvons une vulnérabilité dans notre code ou dans une bibliothèque dont dépend notre code, nous la corrigeons et vous l&apos;envoyons sous forme de mise à jour de version. Garder votre agent à jour réduit le risque que quelqu&apos;un puisse impacter vos opérations en exploitant l&apos;une de ces vulnérabilités.
* <DNT>**Bug fixes:**</DNT> Gagnez du temps et évitez les efforts inutiles pour identifier et signaler les bugs en vous assurant que vous utilisez la dernière version de l&apos;agent avec notre débogage le plus récent.
* <DNT>**New features:**</DNT> Nous améliorons constamment nos agents pour vous aider à voir davantage votre environnement, à trouver des problèmes dans votre application et à les fermer plus efficacement. Assurez-vous d&apos;utiliser la version d&apos;agent la plus récente pour profiter de tout ce que New Relic a à offrir.

## Mettez à jour vos agents régulièrement

Nous vous recommandons de mettre à jour vers la dernière version de l&apos;agent dès qu&apos;elle est disponible. Si votre organisation a mis en place des pratiques qui vous empêchent de mettre à niveau vers la dernière version, assurez-vous que vos agents sont régulièrement mis à jour vers une version datant d&apos;au plus 90 jours.

## Gardez une trace de la version de votre agent

Utilisez l&apos;application New Relic Agent Groundkeeper pour suivre les versions des agents. Pour y accéder, rendez-vous sur <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Apps &gt; Agent groundskeeper**</DNT>.

<img title="Groundskeeper" alt="An image showing the application Groundskeeper" src="/images/apm_screenshot-full_groundskeeper-app.webp" />

Utilisez l&apos;application du jardinier pour visualiser :

* Âge de l&apos;agent
* Nom de l&apos;application utilisant l&apos;agent
* Numéro de version
* Date de sortie de la version

## Considérations lors de la mise à jour de vos agents

Parfois, les nouvelles versions d&apos;agents incluent des modifications radicales. Consultez la [documentation de votre agent linguistique spécifique](/docs/new-relic-solutions/new-relic-one/install-configure/compatibility-requirements-new-relic-agents-products/) lorsque vous vous préparez à mettre à jour votre agent.

Les nouvelles fonctionnalités publiées nécessitent occasionnellement l&apos;acquisition de données supplémentaires, comme le tracing distribué et les logs en contexte. Nous travaillons dur pour garantir que chaque octet ingéré vous apporte de la valeur, mais il existe des moyens d’ajuster vos taux d’ingestion. Parlez à votre gestionnaire de compte ou consultez notre [forum d&apos;assistance](https://discuss.newrelic.com) pour plus d&apos;informations sur l&apos;optimisation de votre ingestion.

## déployer l&apos;agent automatiquement

Nous recommandons l&apos;automatisation du déploiement pour garantir que votre agent reste à jour. Les solutions d&apos;automatisation incluent Ansible, Chef et Puppet. Consultez notre [guide des bonnes pratiques APM](/docs/new-relic-solutions/best-practices-guides/full-stack-observability/apm-best-practices-guide/#agents-current) pour plus d’informations.
