Dans la leçon précédente, vous avez ajouté une option de configuration à votre visualisation qui vous permet de choisir comment vous souhaitez représenter vos données. Cela était différent de la version précédente de votre code car, même si vous pouvez toujours choisir votre type de graphique, vous ne pouvez désormais le faire qu&apos;avant de créer une instance de votre visualisation. Mais que signifie créer une instance de votre visualisation ? Comment utiliser une visualisation une fois que vous l&apos;avez configurée ?

Dans cette leçon, vous apprendrez à créer une instance de votre visualisation et à ajouter cette instance au dashboard.

## Ajoutez votre visualisation personnalisée à un dashboard [#add-dashboard]

<Steps>
  <Step>
    Depuis le répertoire racine de votre Nerdpack, publiez et abonnez-vous à votre visualisation :

    ```sh
    nr1 nerdpack:publish
    nr1 nerdpack:subscribe
    ```

    <Callout variant="important">
      Pour publier un Nerdpack, vous devez être un utilisateur complet avec le privilège « modifier » des Nerdpacks (ceci est accordé par défaut).
    </Callout>

    Ensuite, recherchez votre visualisation publiée dans la superposition des applications New Relic.
  </Step>

  <Step>
    Depuis la [page d’accueil de New Relic](https://one.newrelic.com), sélectionnez le bouton **Apps** dans la barre de navigation en haut à droite.
  </Step>

  <Step>
    Depuis la page **Apps** , cliquez sur le lanceur **Custom Visualizations** .
  </Step>

  <Step>
    Sélectionnez **RadarOrTreemap**.
  </Step>

  <Step>
    Dans la leçon précédente, vous avez modifié votre visualisation pour utiliser le panneau de configuration de la plateforme pour sélectionner un type de graphique.
  </Step>

  <Step>
    Cliquez sur **Add to dashboard**. Ensuite, sélectionnez un dashboard dans la liste des tableaux de bord disponibles ou sélectionnez **New dashboard**.

    Si vous décidez de créer un nouveau dashboard, sélectionnez un compte et un nom.

    Cliquez sur **Add to dashboard**.
  </Step>

  <Step>
    Accédez à votre dashboard pour voir votre visualisation personnalisée.

    Notez que vous pouvez créer une autre version de la visualisation avec l’autre type de graphique et l’ajouter au même dashboard. C&apos;est pourquoi nous appelons ces instances de votre visualisation.
  </Step>
</Steps>

## Supprimez votre visualisation [#delete]

Une fois que vous avez ajouté votre visualisation à un dashboard, vous ne pouvez pas la modifier. Vous ne pouvez pas modifier la requête sous-jacente, l&apos;ID de compte, le type de graphique ou toute autre option précédemment configurable. Donc, si vous n&apos;êtes pas satisfait de votre visualisation, supprimez-la de votre dashboard et parcourez à nouveau les étapes de ce guide pour ajouter une nouvelle instance.

<Steps>
  <Step>
    Cliquez sur le bouton des ellipses dans le coin droit de la visualisation, puis cliquez sur **Delete**.
  </Step>

  <Step>
    Confirmez que vous souhaitez supprimer votre visualisation.
  </Step>
</Steps>
