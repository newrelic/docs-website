L’un des principaux éléments du SDK New Relic est l’interface de ligne de commande (CLI). Pour créer un Nerdpack, vous devrez installer le SDK, configurer la CLI pour qu&apos;elle fonctionne avec votre compte New Relic, puis utiliser sa commande `create`.

# Installer et configurer la CLI [#install]

<Steps>
  <Step>
    Accédez au démarrage rapide de [**Build on New Relic**](https://one.newrelic.com/launcher/developer-center.launcher?pane=eyJuZXJkbGV0SWQiOiJkZXZlbG9wZXItY2VudGVyLmRldmVsb3Blci1jZW50ZXIifQ==) .
  </Step>

  <Step>
    Obtenez votre [clé API de licence](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher).

    Une fois la CLI installée, vous utiliserez cette clé pour créer un profil utilisateur associé à votre compte. La CLI utilise ce profil pour gérer les entités au sein de votre compte.
  </Step>

  <Step>
    Lorsque vous installez la CLI, lisez et acceptez les conditions générales des développeurs de New Relic.

    Même si vous installez la CLI, vous ne pourrez pas l&apos;utiliser sans avoir au préalable accepté ces termes et conditions.
  </Step>

  <Step>
    Choisissez votre système d&apos;exploitation et cliquez sur **Télécharger le programme d&apos;installation**.

    Une fois le SDK installé, vous aurez accès à la CLI `nr1` . Vérifiez cela en vérifiant votre version SDK :

    ```sh
    nr1 --version
    ```

    Si vous possédiez déjà la CLI, mettez-la à jour :

    ```sh
    nr1 update
    ```

    <Callout variant="tip">
      Il est important de faire la distinction entre la CLI `newrelic` et la CLI `nr1` . `newrelic` sert à gérer l&apos;entité dans votre compte New Relic. `nr1` sert à gérer les applications New Relic.
    </Callout>
  </Step>

  <Step>
    Copiez la commande pour enregistrer vos informations d&apos;identification.

    Cette commande comporte un nom de profil, votre région et votre clé API intégrés.

    Exécutez la commande dans votre terminal :

    ```sh
    nr1 profiles:add --name <profile name> --api-key <User key> --region <region>
    ```

    Les profils vous permettent de sélectionner le compte New Relic sur lequel vous souhaitez exécuter des commandes. Si vous avez plusieurs comptes, vous pouvez les consulter avec `profiles:list`:

    ```sh
    nr1 profiles:list
    ```

    Notez qu’un profil est votre profil par défaut. Il s&apos;agit du compte sur lequel vos commandes seront exécutées, sauf si vous en spécifiez un autre. Pour spécifier un profil pour une commande particulière, utilisez l&apos;option `--profile` :

    ```sh
    nr1 create --profile <your profile>
    ```

    Si c&apos;est la première fois que vous utilisez la CLI, le profil que vous venez d&apos;ajouter est votre profil par défaut. Si vous avez d&apos;autres profils, vous devez définir votre profil par défaut sur celui que vous souhaitez utiliser pour ce cours :

    ```sh
    nr1 profiles:default
    ```

    <Callout variant="tip">
      Si vous oubliez ces commandes, vous pouvez les rechercher dans le menu d&apos;aide des profils :

      ```sh
      nr1 profiles --help
      ```
    </Callout>
  </Step>
</Steps>

Vous pouvez maintenant quitter le démarrage rapide **Build on New Relic**. Vous êtes prêt à créer une application avec la CLI New Relic !
