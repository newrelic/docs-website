<Callout variant="tip">
  Cette leçon fait partie d’un cours qui vous apprend à créer une application New Relic à partir de zéro. Si vous ne l’avez pas déjà fait, consultez la présentation.

  Chaque leçon du cours s&apos;appuie sur la précédente, alors assurez-vous d&apos;avoir terminé la dernière leçon, Créer un Nerdpack, avant de commencer celle-ci.

Lorsque vous créez une application New Relic, il est utile de la visualiser sur la plateforme. Cela vous aide à concevoir, implémenter et déboguer votre application dans l&apos;environnement dans lequel elle sera éventuellement publiée. Avec `nr1`, vous pouvez lancer un serveur de développement qui héberge votre application afin que New Relic puisse vous la présenter.

<Steps>
  <Step>
    Depuis le répertoire racine du Nerdpack, servez votre Nerdpack :

    ```bash
    nr1 nerdpack:serve
    ```

    Lorsque le Nerdpack a réussi à se construire et que votre application est prête à être visualisée, vous verrez un message avec un lien vers New Relic :

    ```bash
    [output] {green}✔{plain}  Server ready! Test it at: {purple}https://one.newrelic.com/?nerdpacks=local
    [output] {blue}↩{plain}  Server will reload automatically if you modify any file!
    ```

    Notez le paramètre de requête ajouté `nerdpacks=local`. Ce paramètre de requête indique à New Relic d&apos;afficher les Nerdpacks servis localement.

    <Callout variant="tip">
      Si vous ne voyez pas votre application sur la plateforme New Relic, assurez-vous d&apos;avoir inclus `nerdpacks=local` dans votre chaîne de requête !
    </Callout>
  </Step>

  <Step>
    Accédez à l’URL fournie. Depuis la page d’accueil, sélectionnez **Apps** pour afficher les applications New Relic.
  </Step>

  <Step>
    Dans la section **Your apps** , recherchez votre lanceur, appelé « A/B Test Launcher ».
  </Step>

  <Step>
    Sélectionnez votre lanceur pour voir votre Nerdlet racine et son message de bienvenue par défaut.
  </Step>
</Steps>

Félicitations, vous avez effectué votre première demande New Relic !

Notez, dans la sortie de la commande, que le serveur se recharge lorsque vous modifiez des fichiers dans votre Nerdpack. Essayez-le en mettant à jour `nerdlets/ab-test-nerdlet/index.js`:

```js fileName=nerdlets/ab-test-nerdlet/index.js lineHighlight=5
import React from 'react';

export default class AbTestNerdletNerdlet extends React.Component {
    render() {
        return <h1>A/B test results</h1>;
    }
}
```

Votre application s&apos;actualise automatiquement pour afficher le nouveau titre.
