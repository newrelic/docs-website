L&apos;application New Relic que vous créez tout au long de ce cours permet aux développeurs de tester A/B leurs sites Web. Pour réaliser un test A/B réussi, les propriétaires de sites doivent être en mesure d&apos;analyser les résultats. Sans connaître les performances de chaque conception, comment les développeurs détermineront-ils quelle conception utiliser ?

Votre application affiche des données pour deux versions concurrentes. Grâce à divers graphiques et tableaux, votre utilisateur sera en mesure d&apos;analyser les résultats de son test et de prendre des décisions éclairées sur la conception qui fonctionnera le mieux pour atteindre ses objectifs.

Vous reviendrez à cette maquette à plusieurs reprises tout au long de ce cours. Il vous montre quels graphiques créer, comment les organiser et quel type de données leur fournir. Dans les quatre sections suivantes, vous apprendrez à créer chaque graphique dont votre application de test A/B a besoin pour fonctionner efficacement. Après cela, vous organiserez les graphiques pour qu&apos;ils correspondent à la conception de la maquette et fournirez les données New Relic collectées à partir des applications de démonstration que vous avez créées précédemment.
