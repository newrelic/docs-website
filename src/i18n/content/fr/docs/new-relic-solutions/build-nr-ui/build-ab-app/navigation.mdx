<Callout variant="tip">
  Cette leçon fait partie d’un cours qui vous apprend à créer une application New Relic à partir de zéro. Si vous ne l’avez pas déjà fait, consultez la présentation.

  Chaque leçon du cours s&apos;appuie sur la dernière, alors assurez-vous d&apos;avoir terminé la dernière leçon, Ajouter PlatformStateContext à votre nerdlet, avant de commencer celle-ci.
</Callout>

Dans la dernière leçon, vous avez utilisé `PlatformStateContext` du SDK New Relic One pour rechercher la plage horaire sélectionnée par l&amp;apos;utilisateur à partir du sélecteur d&amp;apos;intervalle de temps de l&amp;apos;application. Vous allez maintenant découvrir un autre composant qui interagit avec la plateforme New Relic : `navigation`.

Le composant `navigation` vous permet d&amp;apos;ouvrir l&amp;apos;entité, de naviguer vers l&amp;apos;entité et de créer des objets Location pour l&amp;apos;entité à partir de votre Nerdlet. Vous pouvez également utiliser `navigation` pour d&amp;apos;autres Nerdlets et lanceurs.

<Steps>
  <Step>
    Accédez au répertoire `add-navigation/ab-test` du [référentiel de cours](https://github.com/newrelic-experimental/nru-programmability-course):

    ```sh
    cd nru-programmability-course/add-navigation/ab-test
    ```

    Ce répertoire contient le code que nous attendons de votre application à ce stade du cours. En naviguant vers le bon répertoire au début de chaque leçon, vous laissez votre code personnalisé derrière vous, vous protégeant ainsi du transport de code incorrect d&apos;une leçon à l&apos;autre.
  </Step>

  <Step>
    Dans `newsletter-signups.js`, créez une nouvelle méthode, appelée `NewsletterSignups.openApmEntity()`:

    ```js
    import React from 'react';
    import {
        HeadingText,
        LineChart,
        NrqlQuery,
        PlatformStateContext,
        navigation,
    } from 'nr1';

    const ACCOUNT_ID = 1234567  // <YOUR NEW RELIC ACCOUNT ID>

    export default class NewsletterSignups extends React.Component {
        openAPMEntity() {
            navigation.openStackedEntity(ENTITY_GUID)
        }

        render() {
            return <div>
                <HeadingText className="chartHeader">
                    Newsletter subscriptions per version
                </HeadingText>
                <PlatformStateContext.Consumer>
                    {
                        (platformState) => {
                            return <NrqlQuery
                                accountIds={[ACCOUNT_ID]}
                                query="SELECT count(*) FROM subscription FACET page_version TIMESERIES"
                                timeRange={platformState.timeRange}
                                pollInterval={60000}
                            >
                                {
                                    ({ data }) => {
                                        return <LineChart data={data} fullWidth />;
                                    }
                                }
                            </NrqlQuery>
                        }
                    }
                </PlatformStateContext.Consumer>
            </div>
        }
    }
    ```

    <Callout variant="important">
      Assurez-vous de remplacer `<YOUR NEW RELIC ACCOUNT ID>` par votre [identifiant de compte](/docs/accounts/accounts-billing/account-setup/account-id) New Relic actuel.
    </Callout>

    Cette méthode utilise `navigation.openStackedEntity()` pour afficher l&amp;apos;entité APM de votre application de démonstration dans une vue empilée. L&amp;apos;avis `openApmEntity()` nécessite votre `ENTITY_GUID`. Vous devez localiser cet ID et le stocker dans une constante.
  </Step>

  <Step>
    Accédez à **APM**.

    Consultez les métadonnées de votre service **Newsletter** .

    Copiez le GUID de l&apos;entité.


    <Callout variant="important">
      Assurez-vous de remplacer `<YOUR NEW RELIC ACCOUNT ID>` et `<YOUR NEW RELIC ENTITY GUID>` par votre [ID de compte](/docs/accounts/accounts-billing/account-setup/account-id) New Relic réel et le GUID d&amp;apos;entité que vous venez de copier, respectivement.
    </Callout>

    Maintenant, `openApmEntity()` sait quelle entité afficher. Ensuite, vous devez créer un bouton pour appeler cette méthode.
  </Step>

  <Step>
    Créez un bouton pour afficher votre entité APM :

    ```js
    import React from 'react';
    import {
        Button,
        HeadingText,
        LineChart,
        NrqlQuery,
        PlatformStateContext,
        navigation,
    } from 'nr1';

    const ACCOUNT_ID = 1234567  // <YOUR NEW RELIC ACCOUNT ID>
    const ENTITY_GUID = "<YOUR NEW RELIC ENTITY GUID>"

    export default class NewsletterSignups extends React.Component {
        openAPMEntity() {
            navigation.openStackedEntity(ENTITY_GUID)
        }

        render() {
            return <div>
                <HeadingText className="chartHeader">
                    Newsletter subscriptions per version
                </HeadingText>
                <Button onClick={this.openAPMEntity}>
                    App performance
                </Button>
                <PlatformStateContext.Consumer>
                    {
                        (platformState) => {
                            return <NrqlQuery
                                accountIds={[ACCOUNT_ID]}
                                query="SELECT count(*) FROM subscription FACET page_version TIMESERIES"
                                timeRange={platformState.timeRange}
                                pollInterval={60000}
                            >
                                {
                                    ({ data }) => {
                                        return <LineChart data={data} fullWidth />;
                                    }
                                }
                            </NrqlQuery>
                        }
                    }
                </PlatformStateContext.Consumer>
            </div>
        }
    }
    ```

    <Callout variant="important">
      Assurez-vous de remplacer `<YOUR NEW RELIC ACCOUNT ID>` et `<YOUR NEW RELIC ENTITY GUID>` par votre [ID de compte](/docs/accounts/accounts-billing/account-setup/account-id) New Relic et votre GUID d&amp;apos;entité réels, respectivement.
    </Callout>

    Ici, vous avez créé un bouton et l&apos;avez configuré pour appeler `openApmEntity()` lorsqu&amp;apos;il est cliqué.
  </Step>

    Vous utiliserez `Stack` et `StackItem` pour disposer le bouton à l&amp;apos;extrême droite de la même ligne sur laquelle se trouve le `HeadingText` .
  </Step>

  <Step>
    Disposez le `Stack`:

    ```js
    import React from 'react';
    import {
        Button,
        HeadingText,
        LineChart,
        NrqlQuery,
        PlatformStateContext,
        Stack,
        StackItem,
        navigation,
    } from 'nr1';

    const ACCOUNT_ID = 1234567  // <YOUR NEW RELIC ACCOUNT ID>
    const ENTITY_GUID = "<YOUR NEW RELIC ENTITY GUID>"

    export default class NewsletterSignups extends React.Component {
        openAPMEntity() {
            navigation.openStackedEntity(ENTITY_GUID)
        }

        render() {
            return <div>
                <Stack fullWidth>
                    <StackItem grow={true}>
                        <HeadingText className="chartHeader">
                            Newsletter subscriptions per version
                        </HeadingText>
                    </StackItem>
                    <StackItem>
                        <Button onClick={this.openAPMEntity}>
                            App performance
                        </Button>
                    </StackItem>
                </Stack>
                <PlatformStateContext.Consumer>
                    {
                        (platformState) => {
                            return <NrqlQuery
                                accountIds={[ACCOUNT_ID]}
                                query="SELECT count(*) FROM subscription FACET page_version TIMESERIES"
                                timeRange={platformState.timeRange}
                                pollInterval={60000}
                            >
                                {
                                    ({ data }) => {
                                        return <LineChart data={data} fullWidth />;
                                    }
                                }
                            </NrqlQuery>
                        }
                    }
                </PlatformStateContext.Consumer>
            </div>
        }
    }
    ```

    <Callout variant="important">
      Assurez-vous de remplacer `<YOUR NEW RELIC ACCOUNT ID>` et `<YOUR NEW RELIC ENTITY GUID>` par votre [ID de compte](/docs/accounts/accounts-billing/account-setup/account-id) New Relic et votre GUID d&amp;apos;entité réels, respectivement.
    </Callout>

    Ici, vous avez utilisé un `Stack` pleine largeur pour configurer la mise en page de la ligne. Pour que le `HeadingText` remplisse toute la ligne à l&amp;apos;exception de la largeur du bouton, vous avez utilisé l&amp;apos;accessoire `StackItem.grow` .
  </Step>

  <Step>
    Accédez à la racine de votre Nerdpack à `nru-programmability-course/add-navigation/ab-test`.
  </Step>

  <Step>
    Générez un nouvel UUID pour votre Nerdpack :

    ```sh
    nr1 nerdpack:uuid -gf
    ```

    Étant donné que vous avez cloné le référentiel de cours qui contenait un Nerdpack existant, vous devez générer votre propre identifiant unique. Cet UUID mappe votre Nerdpack à votre compte New Relic. Il permet également à votre application de faire requests Nerdgraph au nom de votre compte.
  </Step>

  <Step>
    Présentez votre demande localement :

    ```sh
    nr1 nerdpack:serve
    ```
  </Step>

  <Step>
    Accédez à [https://one.newrelic.com?nerdpacks=local](https://one.newrelic.com?nerdpacks=local) et affichez votre application sous **Apps &gt; Your apps**.

    Cliquez sur **App performance**.

    Vous voyez maintenant l’entité empilée.

    <Callout variant="tip">
      Si quelque chose ne fonctionne pas, utilisez les outils de débogage de votre navigateur pour essayer d&apos;identifier le problème.

      **Assurez-vous de :**

      * J&apos;ai copié correctement le code de la leçon
      * Générer un nouvel UUID
      * Remplacé toutes les instances de `<YOUR NEW RELIC ACCOUNT ID>` et `<YOUR NEW RELIC ENTITY GUID>` dans votre projet par votre [ID de compte](/docs/accounts/accounts-billing/account-structure/account-id/) New Relic réel et GUID d&amp;apos;entité, respectivement
    </Callout>
  </Step>
</Steps>
