## Abonnez-vous à votre application [#subscribe]

Maintenant que votre application est publiée, vous pouvez la visualiser en visitant la page **Integrations &amp; Agents** de [New Relic](https://one.newrelic.com).

À partir de là, vous pouvez vous abonner à des applications.

<Steps>
  <Step>
    Depuis le catalogue, cliquez sur votre application.

    <Callout variant="tip">
      Si vous ne voyez pas votre application dans le catalogue, vous devrez peut-être actualiser la page ou vider le cache de votre navigateur.
    </Callout>

    Cela ouvre la page **About** de l&amp;apos;application et affiche les détails que vous avez soumis au catalogue lors de la dernière leçon.
  </Step>

  <Step>
    En haut à gauche de cette page, cliquez sur **Add this app**.

    Cela ouvre la vue d’ **accès au compte** .
  </Step>

  <Step>
    Depuis **Account access**, sélectionnez le compte auquel vous souhaitez vous abonner à l&amp;apos;application et la chaîne à laquelle vous abonner. Enfin, sélectionnez **Update 1 account**.

    Ici, vous n&apos;aviez qu&apos;une seule tag de version à laquelle vous abonner. Dans d&apos;autres projets, vous pouvez avoir d&apos;autres versions avec d&apos;autres balises.

    <Callout variant="tip" title="Détails techniques">
      Notez que l&apos;interface utilisateur Web utilise le terme « canal » au lieu de « tag». Il s&apos;agit d&apos;une relique d&apos;une ancienne version des applications New Relic. Avec les versions ultérieures de `nr1`, nous sommes passés à la terminologie « tag ». Finalement, l&amp;apos;interface utilisateur utilisera également « tag ».
    </Callout>
  </Step>

  <Step>
    Accédez à la page **Apps** et voyez l’application AbTest sous **Your Apps**.

    À partir de là, vous pouvez cliquer sur votre application et l’utiliser comme vous le faisiez lorsque vous la diffusiez localement !

    <Callout variant="tip">
      Si vous ne voyez pas l&apos;application sous **Your Apps**, vous devrez peut-être actualiser la page ou vider le cache de votre navigateur.
    </Callout>
  </Step>
</Steps>

## Se désinscrire de votre application [#unsubscribe]

Le processus de désabonnement d’une application est similaire à celui de l’abonnement.

<Steps>
  <Step>
    Cliquez sur l&apos;icône « i » sur votre lanceur d&apos;applications.

    Ici, vous voyez les mêmes détails de l’application que vous avez vus lorsque votre application était dans le catalogue.
  </Step>

  <Step>
    Cliquez sur **Manage access**.

    Cela ouvre la vue d’ **accès au compte** .
  </Step>

  <Step>
    Depuis **Account access**, désélectionnez le compte que vous souhaitez désabonner de l&amp;apos;application et sélectionnez **Update 1 account**.

    Si vous revenez à **Applications**, vous verrez que votre application ne se trouve plus sous **Vos applications**, mais peut être retrouvée dans le catalogue Instant Observability .
  </Step>
</Steps>

Tout au long de ce cours, vous avez créé une application New Relic à partir de zéro. Vous avez utilisé la CLI `nr1` pour créer un Nerdpack, un lanceur et un Nerdlet. Vous avez utilisé de nombreux composants du SDK. Vous avez appris à publier, tag, vous abonner et vous désabonner des applications dans le catalogue Instant Observability . Vous avez également appris à soumettre des métadonnées au catalogue.
