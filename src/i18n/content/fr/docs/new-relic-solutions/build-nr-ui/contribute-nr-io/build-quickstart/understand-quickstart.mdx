Avant de procéder à la création d’un quickstart, vous devez comprendre sa structure de répertoire.

Pour commencer, accédez au référentiel GitHub des démarrages rapides de New Relic et forkez-le. Ensuite, clonez votre propre référentiel sur votre machine locale en utilisant `git clone` et ouvrez-le dans l&apos;IDE de votre choix.

À la racine de votre projet cloné, les répertoires du dossier `\_template` reflètent les répertoires du niveau supérieur du référentiel et contiennent les modèles pour les blocs de construction d&apos;observabilité respectifs dans le quickstart.

Avec la récente mise à jour de la structure des démarrages rapides, le répertoire `\_template` a désormais la structure suivante :

```
alert-policies/
    example-alert-policy/
        baseline-alert.yml
        static-alert.yml

dashboards/
    example-dashboard/
        example-dashboard.json
        example-dashboard.png

data-sources/
    example-data-source/
        config.yml
        logo.svg

quickstarts/
    example-quickstart/
        config.yml
        logo.svg
```

Chacun de ces répertoires contient des modèles que vous pouvez utiliser pour créer une entité pour votre quickstart. La structure des nouvelles contributions doit correspondre à la structure vue ici. Vous faites référence aux tableaux de bord et alertes associés dans les démarrages rapides `config.yml`.

Afin de créer un quickstart, vous devez d&apos;abord ajouter son entité comme dashboard et les alertes au quickstart.
