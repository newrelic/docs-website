Imaginez que vous avez développé une nouvelle base de données appelée FlashDB, la base de données la plus légère et la plus rapide de l’espace. Votre base de données est déjà très populaire parmi les développeurs. Vous savez que New Relic fournit d&apos;excellents outils d&apos;observabilité, vous recommandez donc à votre utilisateur d&apos;utiliser New Relic pour obtenir des informations détaillées de sa stack. Jusqu&apos;à présent, votre utilisateur devait configurer manuellement les dashboards et les alertes dans New Relic. Vous avez récemment découvert New Relic Instant Observability (I/O), qui vous fournit des blocs de construction d&apos;observabilité pour tirer plus rapidement parti de vos données.

Avec la popularité croissante de votre base de données, vous décidez d&apos;aider les utilisateurs de FlashDB à l&apos;observer rapidement et facilement dans New Relic.

Pour atteindre votre objectif, contribuez à un quickstart de New Relic I/O afin que d&apos;autres utilisateurs puissent monitorer rapidement FlashDB.

## Instant Observability [#instant-observability]

[New Relic Instant Observability (I/O)](https://newrelic.com/instant-observability) est un écosystème unifié de ressources d&apos;observabilité prêtes à l&apos;emploi qui vous aident à obtenir une valeur immédiate de vos données. Il vous permet d&apos;instrumenter, de monitorer et d&apos;analyser votre stack technologique tout en évitant le fardeau de la configuration manuelle.

Avec New Relic I/O, vous :

* Démarrez plus rapidement
* Réduire le travail de l&apos;instrumentation manuelle
* Apprenez des experts en observabilité

<Video id="c9zprczTXj8" type="youtube" />

Vous pouvez choisir parmi des centaines de démarrages rapides qui regroupent les éléments de base nécessaires pour instrumenter et monitoring votre stack technologique, les installer en un clic et commencer monitoring.

## démarrage rapide [#quickstart]

Avec New Relic, vous pouvez collecter, monitorer et signaler des données provenant de différentes sources. Vous pouvez utiliser les dashboards pour afficher, explorer, personnaliser, comprendre et corréler les données que vous collectez à l&apos;aide de différents graphiques conviviaux. Vous pouvez également définir une règle d&apos;alerte pour monitorer les mesures de performances clés et recevoir une notification en cas d&apos;anomalie.

Traditionnellement, l&apos;utilisateur FlashDB devait passer des heures de développement cruciales à créer des tableaux de bord, des alertes et d&apos;autres entités pour monitorer leur base de données. Mais avec des démarrages rapides, vous pouvez faire ce travail à leur place. Les démarrages rapides offrent une valeur immédiate pour votre cas d&apos;utilisation spécifique. Ils comprennent :

* Des instructions claires pour instrumenter vos services
* blocs de construction d&apos;observabilité comme les dashboards et les alertes

Tout cela est disponible via New Relic I/O où les développeurs, partenaires et clients de New Relic apportent leurs bonnes pratiques en matière de solutions. Vous, en tant que développeurs de FlashDB, êtes un excellent candidat pour contribuer à un quickstart pour aider votre utilisateur à monitorer rapidement son instance.

## Objectifs [#objectives]

* Envoyer des données télémétriques à New Relic

* Créer un quickstart
