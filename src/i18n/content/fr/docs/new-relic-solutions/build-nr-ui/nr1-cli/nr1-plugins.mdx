## `nr1 plugins:install` [#nr1-pluginsinstall]

### Installer un plugin [#nr1-pluginsinstall-install]

Installe un plugin dans la CLI. Vous pouvez installer des plugins depuis npm ou une URL Git.

Veuillez noter que l&apos;installation d&apos;un plugin remplacera le plugin principal.

Par exemple, si vous avez un plugin principal qui possède une commande `hello` , l&apos;installation d&apos;un plugin avec une commande `hello` remplacera l&apos;implémentation du plugin principal. Ceci est utile si vous souhaitez mettre à jour les fonctionnalités principales du plugin sans appliquer de correctifs ni mettre à jour l&apos;ensemble de la CLI.

### Usage [#nr1-pluginsinstall-usage]

`$ nr1 plugins:install PLUGIN`

### Arguments [#nr1-pluginsinstall-arguments]

* `PLUGIN`: le nom, le chemin ou l&apos;URL du plugin que vous souhaitez installer. Ceci est **obligatoire** .

### Options [#nr1-pluginsinstall-options]

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        Exécute `yarn install --force`. Cela force un retéléchargement de tous les packages du plugin.
      </td>
    </tr>

    <tr>
      <td>
        `-h, --help`
      </td>

      <td>
        Affiche l&apos;aide de la CLI.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Ajoute des informations supplémentaires à la sortie.
      </td>
    </tr>
  </tbody>
</table>

### Exemples [#nr1-pluginsinstall-examples]

* `$ nr1 plugins:install myplugin`
* `$ nr1 plugins:install https://github.com/someuser/someplugin`
* `$ nr1 plugins:install someuser/someplugin`

### Alias [#nr1-pluginsinstall-aliases]

* `$ nr1 plugins:add`

## `nr1 plugins:link` [#nr1-pluginslink]

### Liez votre plugin [#nr1-pluginslink-link]

Lie un plugin local dans la CLI pour le développement.

Veuillez noter que la liaison d&apos;un plugin remplacera votre plugin installé par l&apos;utilisateur ou votre plugin principal.

Par exemple, si vous avez un plug-in installé par l&apos;utilisateur ou un plug-in principal qui possède une commande `hello` , lier un plug-in avec une commande `hello` remplacera l&apos;implémentation du plug-in installé par l&apos;utilisateur ou du plug-in principal. Ceci est utile pour le travail de développement.

### Usage [#nr1-pluginslink-usage]

`$ nr1 plugins:link PLUGIN`

### Arguments [#nr1-pluginslink-arguments]

* `PLUGIN`: le nom, le chemin ou l&apos;URL du plugin que vous souhaitez lier. Ceci est **obligatoire** .

### Options [#nr1-pluginslink-options]

<table>
  <tbody>
    <tr>
      <td>
        `-h, --help`
      </td>

      <td>
        Affiche l&apos;aide de la CLI.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Ajoute des informations supplémentaires à la sortie.
      </td>
    </tr>
  </tbody>
</table>

### Exemples [#nr1-pluginslink-examples]

* `$ nr1 plugins:link myplugin`
* `$ nr1 plugins:link someuser/someplugin`
