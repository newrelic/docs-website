## nr1 saisie semi-automatique [#nr1-autocomplete]

Par défaut, la commande affiche les instructions de saisie semi-automatique pour `zsh`. Si vous voulez des instructions pour bash, exécutez `nr1 autocomplete bash`.

### Usage

`$ nr1 autocomplete`

### Arguments

<table>
  <tbody>
    <tr>
      <td>
        `SHELL`
      </td>

      <td>
        Le type de shell pour lequel vous souhaitez des instructions.
      </td>
    </tr>
  </tbody>
</table>

### Options

<table>
  <tbody>
    <tr>
      <td>
        `-r, --refresh-cache`
      </td>

      <td>
        Actualise le cache et ignore les instructions d&apos;affichage.
      </td>
    </tr>
  </tbody>
</table>

### Exemples

* `$ nr1 autocomplete`
* `$ nr1 autocomplete zsh`
* `$ nr1 autocomplete bash`
* `$ nr1 autocomplete --refresh-cache`

## nr1 créer [#nr1-create]

Crée un nouveau composant à partir de notre modèle, soit un Nerdpack, un Nerdlet, un lanceur ou un catalogue. La CLI vous guidera tout au long de ce processus.

Pour en savoir plus sur les Nerdpacks et leur structure de fichiers, consultez [Structure de fichier Nerdpack](/docs/new-relic-solutions/build-nr-ui/nerdpack-file-structure). Pour en savoir plus sur la configuration de vos Nerdpacks, consultez nos [commandes CLI Nerdpack](/docs/new-relic-solutions/build-nr-ui/nr1-cli/nr1-nerdpack).

### Usage

`$ nr1 create`

### Options

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        S&apos;il est présent, remplace les fichiers existants sans demander.
      </td>
    </tr>

    <tr>
      <td>
        `-n, --name=NAME`
      </td>

      <td>
        Nomme le composant.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --type=TYPE`
      </td>

      <td>
        Spécifie le type de composant.
      </td>
    </tr>

    <tr>
      <td>
        `--path=PATH`
      </td>

      <td>
        L&apos;itinéraire vers le composant.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        Le profil d’authentification que vous souhaitez utiliser.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Ajoute des informations supplémentaires à la sortie.
      </td>
    </tr>
  </tbody>
</table>

## aide nr1 [#nr1-help]

Affiche toutes les commandes nr1 par défaut. Pour obtenir des détails sur une commande spécifique, exécutez `nr1 help COMMAND_NAME`.

### Usage

`$ nr1 help`

### Arguments

<table>
  <tbody>
    <tr>
      <td>
        `COMMAND_NAME`
      </td>

      <td>
        Le nom d&apos;une commande particulière.
      </td>
    </tr>
  </tbody>
</table>

### Exemples

* `$ nr1 help`
* `$ nr1 help nerdpack`
* `$ nr1 help nerdpack:deploy`

## nr1 NRQL [#nr1-nrql]

Récupère les données de la base de données à l&apos;aide d&apos;une requête NRQL .

Pour en savoir plus sur NRQL et comment l&apos;utiliser, consultez notre [documentation NRQL](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/).

### Usage

`$ nr1 nrql OPTION ...`

### Options

<table>
  <tbody>
    <tr>
      <td>
        `-a, --account=ACCOUNT`
      </td>

      <td>
        Ceci est **obligatoire**. L&apos;ID du compte utilisateur.
      </td>
    </tr>

    <tr>
      <td>
        `-q, --query=QUERY`
      </td>

      <td>
        Ceci est **obligatoire**. La requête NRQL à exécuter.
      </td>
    </tr>

    <tr>
      <td>
        `-u, --ugly`
      </td>

      <td>
        Affiche le contenu sans tabulations ni espaces.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        Le profil d’authentification que vous souhaitez utiliser.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Ajoute des informations supplémentaires à la sortie.
      </td>
    </tr>
  </tbody>
</table>

## profils nr1 [#nr1-profiles]

Affiche une liste de commandes que vous pouvez utiliser pour gérer vos profils. Exécutez `nr1 help profiles:COMMAND` pour en savoir plus sur leurs utilisations spécifiques. Vous pouvez avoir plusieurs profils, ce qui est utile pour exécuter des commandes sur plusieurs comptes New Relic.

Pour en savoir plus sur la configuration des profils, consultez [notre atelier Github](https://github.com/newrelic/nr1-workshop/blob/master/lab-cli/INSTRUCTIONS.md).

### Usage

`$ nr1 profiles:COMMAND`

### Commandes

<table>
  <tbody>
    <tr>
      <td>
        `profiles:add`
      </td>

      <td>
        Ajoute un nouveau profil à votre trousseau de profils.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:default`
      </td>

      <td>
        Choisis quel profil doit être défini par défaut.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:list`
      </td>

      <td>
        Répertorie les profils de votre trousseau.
      </td>
    </tr>

    <tr>
      <td>
        `profiles:remove`
      </td>

      <td>
        Supprime un profil de votre trousseau.
      </td>
    </tr>
  </tbody>
</table>
