## `nr1 config:delete` [#nr1-configdelete]

### Supprimer une configuration [#nr1-configdelete-remove]

Supprime la valeur d&apos;une configuration spécifique.

### Usage [#nr1-configdelete-usage]

`$ nr1 config:delete OPTION`

### Options [#nr1-configdelete-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        Ceci est **obligatoire**. La clé de la config.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        Le profil d’authentification que vous souhaitez utiliser.
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        Si elle est présente, cette configuration ne s&apos;appliquera que lors de l&apos;exécution avec le profil spécifié.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Ajoute des informations supplémentaires à la sortie.
      </td>
    </tr>
  </tbody>
</table>

### Alias [#nr1-configdelete-aliases]

* `nr1 config:remove`
* `nr1 config:rm`

### Exemples [#nr1-configdelete-exmples]

* `$ nr1 config:delete --key=proxyHttp`

## `nr1 config:set` [#nr1-configset]

### Définir une configuration [#nr1-configset-set]

Définit une valeur de configuration spécifique à partir d&apos;une clé de configuration. Par défaut, la commande vous prompt une nouvelle valeur après avoir fourni une clé, mais vous pouvez également utiliser l&apos;option `--k, --key=KEY` pour ignorer cette étape.

### Usage [#nr1-configset-usage]

`$ nr1 config:set OPTION`

### Options [#nr1-configset-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        Ceci est **obligatoire**. La clé de la config.
      </td>
    </tr>

    <tr>
      <td>
        `-V, --value=VALUE`
      </td>

      <td>
        La valeur de la configuration.
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        Le profil d&apos;authentification que vous souhaitez utiliser
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        Si elle est présente, cette configuration ne s&apos;appliquera que lors de l&apos;exécution avec le profil spécifié.
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        Ajoute des informations supplémentaires à la sortie.
      </td>
    </tr>
  </tbody>
</table>

### Exemples [#nr1-configset-examples]

* `$ nr1 config:set --key=proxyEnabled`
* `$ nr1 config:set --key=proxyEnabled --value=ENABLED`
