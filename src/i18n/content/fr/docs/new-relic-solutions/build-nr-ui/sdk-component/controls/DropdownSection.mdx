Section du composant `<Dropdown>` .

### Usage

```js
import { DropdownSection } from 'nr1'
```

### Accessoires

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>OBLIGATOIRE</h5> <h5>Nœud|fonction</h5>
      </td>

      <td>
        Ce composant peut être rendu soit de manière déclarative, en passant directement un ensemble d&apos;enfants, soit virtualisé, en passant un rappel de rendu (fonctionne comme enfants). Les seuls éléments autorisés à l&apos;intérieur (ou renvoyés par le rappel de rendu) sont de type `<DropdownItem>`. La recommandation est d&apos;utiliser le rappel de rendu lorsqu&apos;un grand nombre d&apos;éléments est fourni, car la liste des éléments sera virtualisée par le composant, augmentant ainsi les performances.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute les noms de classe au composant. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `items` <h5>éventail</h5>
      </td>

      <td>
        Éléments à restituer, sous la forme d&apos;une liste d&apos;objets. Habituellement, chaque élément de l&apos;éventail d&apos;éléments contient les données requises pour générer le `<DropdownItem>` correspondant. Cet accessoire est requis lors du rendu des éléments avec le rappel de rendu (fonction comme enfants).
      </td>
    </tr>

    <tr>
      <td>
        `onLoadMore` <h5>fonction</h5>
      </td>

      <td>
        Rappel déclenché lorsque davantage d&apos;éléments doivent être chargés. Cela se produit lorsque vous chargez les éléments de manière paresseuse et que les éléments sur le point d&apos;être rendus ne peuvent pas être trouvés dans l&apos;éventail `items` Ce rappel doit être utilisé pour récupérer/charger les éléments manquants à partir du backend ou d&apos;autres sources. La promesse renvoyée doit être résolue une fois le chargement des données de l&apos;élément terminé. Il sera utilisé pour déterminer quand actualiser la liste avec les données nouvellement chargées. Ce rappel peut être appelé plusieurs fois en réaction à un seul événement de défilement.

        <FunctionDefinition returnValue={[]} arguments={[{"name":"cursor","type":"Cursor","description":"Items to load."}]} />
      </td>
    </tr>

    <tr>
      <td>
        `rowCount` <h5>nombre</h5>
      </td>

      <td>
        Nombre de lignes. Par défaut, il est égal à la longueur de l&apos;éventail passé dans la propriété items. Vous devez spécifier `rowCount` lorsque vous connaissez le nombre total d&apos;éléments mais que vous souhaitez les charger paresseusement pendant le défilement.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objet</h5>
      </td>

      <td>
        Style en ligne pour un style personnalisé. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute un attribut `data-test-id`. Utilisez-le pour cibler le composant dans les tests unitaires et E2E. Pour qu&apos;un identifiant de test soit valide, préfixez-le avec votre identifiant nerdpack, suivi d&apos;un point. Par exemple, `my-nerdpack.some-element`.**Remarque :** vous ne verrez peut-être pas l&apos;attribut `data-test-id` car ils sont supprimés du DOM. Pour les déboguer, transmettez un paramètre de requête `e2e-test` à l&apos;URL.
      </td>
    </tr>

    <tr>
      <td>
        `title` <h5>chaîne</h5>
      </td>

      <td>
        Titre de la section.
      </td>
    </tr>
  </tbody>
</table>
