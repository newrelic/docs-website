Le SDK New Relic fournit des composants de requête basés sur les composants de requête d&apos;ApolloClient. Ces composants constituent une couche d&apos;abstraction facilitant l&apos;interrogation de NerdGraph sans se soucier de la configuration d&apos;Apollo Client et, pour les cas d&apos;utilisation les plus courants, sans avoir à écrire de requête GraphQL.

Un composant de requête générique NerdGraph qui vous permet d&apos;interroger n&apos;importe quoi à partir de NerdGraph.

### Usage

```js
import { NerdGraphQuery } from 'nr1'
```

### Exemples

#### Exemple 1

```js
function render() {
  const query = `
        query($id: Int!) {
            actor {
                account(id: $id) {
                    name
                }
            }
        }
    `;


  const variables = {
    id: 1,
  };


  return (
    <NerdGraphQuery query={query} variables={variables}>
      {({ loading, error, data }) => {
        if (loading) {
          return <Spinner />;
        }


        if (error) {
          return 'Error!';
        }


        return <BlockText>{data.actor.account.name}</BlockText>;
      }}
    </NerdGraphQuery>
  );
}
```

        `fetchPolicyType` <h5>énum.</h5>

      <td>
        Vous permet de spécifier comment vous souhaitez que votre requête interagisse avec les données mises en cache.

        * `CACHE_AND_NETWORK`: La requête renvoie vos données initiales du cache si elles sont disponibles. Cependant, que les données complètes soient ou non dans votre cache, la requête effectue toujours une demande à l&amp;apos;aide de votre interface réseau et renvoie les données mises à jour. Cette option n&amp;apos;est pas disponible lors de l&amp;apos;utilisation de la méthode statique `query()` du composant.

        * `CACHE_FIRST`: La requête effectue une demande à l&amp;apos;aide de votre interface réseau **uniquement** si les données de votre requête ne sont pas déjà dans le cache.

        * `CACHE_ONLY`: La requête n&amp;apos;effectue **jamais** de demande à l&amp;apos;aide de votre interface réseau. Au lieu de cela, il renvoie les données disponibles dans le cache. Si les données de votre requête n&amp;apos;existent pas dans le cache, une erreur est générée.

        * `NETWORK_ONLY`: La requête ne renvoie **jamais** vos données initiales à partir du cache. Au lieu de cela, il fait **toujours** une demande en utilisant votre interface réseau.

        * `NO_CACHE`: La requête ne renvoie **jamais** vos données initiales à partir du cache. Au lieu de cela, il fait **toujours** une demande en utilisant votre interface réseau. Contrairement à la politique `NETWORK_ONLY`, aucune donnée n&amp;apos;est écrite dans le cache une fois la requête terminée.
