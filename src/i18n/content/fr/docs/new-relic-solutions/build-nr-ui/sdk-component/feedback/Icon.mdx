### Usage

```js
import { Icon } from 'nr1'
```

### Exemples

```js
<Icon type={Icon.TYPE.HARDWARE_AND_SOFTWARE__SOFTWARE__PLUGIN__S_OK} />;
```

### Accessoires

<table>
  <tbody>
    <tr>
      <td>
        `className` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute les noms de classe au composant. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `color` <h5>chaîne</h5>
      </td>

      <td>
        Couleur de l&apos;icône. Par défaut, il obtient la valeur actuelle de la propriété CSS `color` de l&apos;élément.
      </td>
    </tr>

    <tr>
      <td>
        `spacingType` <h5>énum.</h5>
      </td>

      <td>
        Propriété d&apos;espacement. L&apos;espacement est défini comme un uplet de zéro à quatre valeurs, qui suivent les mêmes conventions que les propriétés CSS telles que `margin` ou `padding`. Pour omettre une valeur, utilisez `SPACING_TYPE.OMIT`.

        <OptionReference array>
          Icon.SPACING\_TYPE.EXTRA\_LARGE, Icon.SPACING\_TYPE.LARGE, Icon.SPACING\_TYPE.MEDIUM, Icon.SPACING\_TYPE.NONE, Icon.SPACING\_TYPE.OMIT, Icon.SPACING\_TYPE.SMALL,
        </OptionReference>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objet</h5>
      </td>

      <td>
        Style en ligne pour un style personnalisé. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute un attribut `data-test-id` . Utilisez-le pour cibler le composant dans les tests unitaires et E2E. Pour qu&apos;un identifiant de test soit valide, préfixez-le avec votre identifiant nerdpack, suivi d&apos;un point. Par exemple, `my-nerdpack.some-element`. **Remarque :** vous ne verrez peut-être pas l&apos;attribut `data-test-id` car ils sont supprimés du DOM. Pour les déboguer, transmettez un paramètre de requête `e2e-test` à l&apos;URL.
      </td>
    </tr>
