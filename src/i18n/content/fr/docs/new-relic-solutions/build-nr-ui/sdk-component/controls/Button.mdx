Les boutons sont utilisés dans tout le produit pour fournir à l&apos;utilisateur un moyen d&apos;effectuer une action. Essayez de ne pas abuser des boutons dans votre expérience. Cela aidera l’utilisateur à savoir exactement quelle action vous souhaitez qu’il entreprenne. N&apos;utilisez jamais plus d&apos;un bouton principal dans votre expérience.

Capacités principales — les boutons peuvent

* effectuer une action
* effectuer plusieurs actions, si plusieurs boutons sont présents.

### Usage

```js
import { Button } from 'nr1'
```

### Exemples

#### Basique

```js
<div className="nr1-Docs-prettify">
  <Button onClick={() => alert('Hello World!')} type={Button.TYPE.PRIMARY}>
    Click me
  </Button>
  <Button
    onClick={() => alert('Hello World!')}
    type={Button.TYPE.PRIMARY}
    iconType={
Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__NOTES__A_ADD}
  >
    Click me
  </Button>
  <Button
    type={Button.TYPE.PRIMARY}
    iconType={
Button.ICON_TYPE.DOCUMENTS__DOCUMENTS__FILE__A_ADD}
  />
</div>
```

#### Tailles

```js
<div className="nr1-Docs-prettify">
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.LARGE}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.MEDIUM}>
    Click me
  </Button>
  <Button type={Button.TYPE.PRIMARY} sizeType={Button.SIZE_TYPE.SMALL}>
    Click me
  </Button>
</div>
```

#### Types

```js

  <Button type={Button.TYPE.NORMAL}>Click me</Button>
  <Button type={Button.TYPE.PRIMARY}>Click me</Button>
  <Button type={Button.TYPE.DESTRUCTIVE}>Click me</Button>
  <Button type={Button.TYPE.OUTLINE}>Click me</Button>
  <Button type={Button.TYPE.PLAIN}>Click me</Button>
  <Button type={Button.TYPE.PLAIN_NEUTRAL}>Click me</Button>
</div>
```

    iconType={

### Accessoires

<table>
  <tbody>
    <tr>
      <td>
        `ariaControls` <h5>chaîne</h5>
      </td>

      <td>
        Transmettez la chaîne `id` de l&apos;élément que `Button` contrôle lorsqu&apos;il est utilisé pour développer ou ouvrir un panneau. Utilisez-le avec `ariaExpanded`.

        ```js
        const { expanded } = this.state;
        const panelId = 'content-panel';

        return (
          <>
            <Button ariaControls={panelId} ariaExpanded={expanded}>
              Expand content
            </Button>
            <div id={panelId} hidden={!expanded}>
              Content
            </div>
          </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaExpanded` <h5>booléen</h5>
      </td>

      <td>
        Utilisez-le pour indiquer si l&apos;élément contrôlé par `Button` est développé ou non. Il est recommandé de vérifier également les exemples `ariaControls` et `ariaHasPopup` .
      </td>
    </tr>

    <tr>
      <td>
        `ariaHasPopup` <h5>booléen|chaîne</h5>
      </td>

      <td>
        Utilisez-le pour indiquer que le `Button` bascule un élément de superposition interactif. La valeur peut être :

        * `false` (par défaut) indique que l&apos;élément n&apos;a pas de fenêtre contextuelle.
        * `true` indique que la fenêtre contextuelle a un rôle `menu`
        * `"menu"` indique que la fenêtre contextuelle a un rôle `menu`
        * `"listbox"` indique que la fenêtre contextuelle a un rôle `listbox`
        * `"tree"` indique que la fenêtre contextuelle a un rôle `tree`
        * `"grid"` indique que la fenêtre contextuelle a un rôle `grid`
        * `"dialog"` indique que la fenêtre contextuelle a un rôle `dialog` Lisez [les spécifications WAI ARIA pour cet attribut](https://www.w3.org/TR/wai-aria-1.1/#aria-haspopup) pour savoir lequel convient le mieux à votre cas d&apos;utilisation.

        ```js
        const { modalHidden } = this.state;

        return (
          <>
            <Button
              ariaHasPopup="dialog"
              ariaExpanded={!modalHidden}
              onClick={() => this.setState({ modalHidden: false })}
            >
              Open modal
            </Button>
            <Modal
              hidden={modalHidden}
              onClose={() => this.setState({ modalHidden: true })}
            />
          </>
        );
        ```
      </td>
    </tr>

    <tr>
      <td>
        `ariaLabel` <h5>chaîne</h5>
      </td>

      <td>
        Utilisez-le pour mieux décrire le contexte de l&apos;action du composant ou dans les boutons affichant uniquement une icône pour l&apos;utilisateur sur les lecteurs d&apos;écran.

        ```js
        <Button
          iconType={Button.ICON_TYPE.INTERFACE__OPERATIONS__SEARCH}
          ariaLabel="Search for entities"
        />;
        ```
      </td>
    </tr>

    <tr>
      <td>
        `children` <h5>nœud</h5>
      </td>

      <td>
        Contenu à rendre à l&apos;intérieur du bouton.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute les noms de classe au composant. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `disabled` <h5>booléen</h5>
      </td>

      <td>
        Utilisez l&apos;état désactivé pour un bouton avant qu&apos;un utilisateur ne remplisse les champs obligatoires d&apos;un formulaire ou lorsqu&apos;un utilisateur doit effectuer une autre tâche avant que le bouton puisse être activé.
      </td>
