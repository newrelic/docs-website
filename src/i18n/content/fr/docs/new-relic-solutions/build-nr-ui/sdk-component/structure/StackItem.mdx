Élément enfant du composant `<Stack>` .

Le `<Stack>` peut modifier la hauteur, la largeur et l&apos;ordre de l&apos;élément stack en fonction de la disposition sélectionnée.

### Usage

```js
import { StackItem } from 'nr1'
```

### Accessoires

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>OBLIGATOIRE</h5> <h5>nœud</h5>
      </td>

      <td>
        Contenu à afficher à l&apos;intérieur de l&apos;élément stack .
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute les noms de classe au composant. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `grow` <h5>booléen</h5>
      </td>

      <td>
        Si `true`, l&apos;élément remplit l&apos;espace restant. Pour que cela fonctionne, vous devrez étendre votre stack pour occuper tout l&apos;espace disponible dans la direction de vos objets. Cela peut être réalisé en utilisant les accessoires `fullWidth` ou `fullHeight` de la stack, selon sa direction.
      </td>
    </tr>

    <tr>
      <td>
        `shrink` <h5>booléen</h5>
      </td>

      <td>
        Si `true`, l&apos;élément rétrécit si nécessaire.
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objet</h5>
      </td>

      <td>
        Style en ligne pour un style personnalisé.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute un attribut `data-test-id`. Utilisez-le pour cibler le composant dans les tests unitaires et E2E. Pour qu&apos;un identifiant de test soit valide, préfixez-le avec votre identifiant nerdpack, suivi d&apos;un point. Par exemple, `my-nerdpack.some-element`.

        **Remarque :** vous ne verrez peut-être pas l&apos;attribut `data-test-id` car ils sont supprimés du DOM. Pour les déboguer, transmettez un paramètre de requête `e2e-test` à l&apos;URL.
      </td>
    </tr>
  </tbody>
</table>
