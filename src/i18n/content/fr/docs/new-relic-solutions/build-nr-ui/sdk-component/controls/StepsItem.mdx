Élément enfant du composant `<Steps>` .

### Usage

```js
import { StepsItem } from 'nr1'
```

### Accessoires

<table>
  <tbody>
    <tr>
      <td>
        `ariaLabel` <h5>chaîne</h5>
      </td>

      <td>
        Fournissez une étiquette aux lecteurs d’écran pour décrire l’étape. Ceci n&apos;est requis que si l&apos;étiquette du lecteur d&apos;écran doit être différente de l&apos;accessoire `label` .
      </td>
    </tr>

    <tr>
      <td>
        `checked` <h5>booléen</h5>
      </td>

      <td>
        Si `true`, marque l&apos;étape comme terminée.
      </td>
    </tr>

    <tr>
      <td>
        `children` <h5>OBLIGATOIRE</h5> <h5>nœud</h5>
      </td>

      <td>
        Contenu à restituer à l&apos;intérieur de l&apos;élément.
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute les noms de classe au composant. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `disabled` <h5>booléen</h5>
      </td>

      <td>
        Utilisez désactivé pour empêcher l&apos;utilisateur d&apos;ouvrir l&apos;étape en cliquant sur son étiquette.
      </td>
    </tr>

    <tr>
      <td>
        `expanded` <h5>booléen</h5>
      </td>

      <td>
        Si cette option est vraie, elle affichera le contenu de l&apos;étape.
      </td>
    </tr>

    <tr>
      <td>
        `label` <h5>OBLIGATOIRE</h5> <h5>chaîne</h5>
      </td>

      <td>
        Étiquette de l&apos;étape.
      </td>
    </tr>

    <tr>
      <td>
        `onClick` <h5>fonction</h5>
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>objet</h5>
      </td>

      <td>
        Style en ligne pour un style personnalisé. Doit être utilisé uniquement à des fins de positionnement et d&apos;espacement.
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>chaîne</h5>
      </td>

      <td>
        Ajoute un attribut `data-test-id` . Utilisez-le pour cibler le composant dans les tests unitaires et E2E. Pour qu&apos;un identifiant de test soit valide, préfixez-le avec votre identifiant nerdpack, suivi d&apos;un point. Par exemple, `my-nerdpack.some-element`. **Remarque :** vous ne verrez peut-être pas l&apos;attribut `data-test-id` car ils sont supprimés du DOM. Pour les déboguer, transmettez un paramètre de requête `e2e-test` à l&apos;URL.
      </td>
    </tr>

    <tr>
      <td>
        `value` <h5>n&apos;importe lequel</h5>
      </td>

      <td>
        Utilisé par le `<Steps>` pour identifier l&apos;étape active.
      </td>
    </tr>
  </tbody>
</table>
