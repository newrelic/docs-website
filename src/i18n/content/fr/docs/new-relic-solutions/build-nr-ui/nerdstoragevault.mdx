Pour vous aider à créer une application New Relic, nous vous fournissons le [SDK New Relic One](/docs/new-relic-solutions/build-nr-ui/sdk-component/intro-to-sdk). Sur cette page, vous apprendrez à utiliser `NerdStorageVault` pour stocker des données dans une solution de stockage cryptée.

## En utilisant `NerdStorageVault` [#using]

Utilisez `NerdStorageVault` pour stocker et récupérer des données secrètes sensibles telles que :

* jeton d&apos;accès personnel
* clé de licence
* Clés API
* autres secrets de tiers

<Callout variant="tip">
  `NerdStorageVault` est crypté avec le chiffrement AES-256.
</Callout>

## Considérations d&apos;utilisation [#usage]

* Ce stockage est unique par Nerdpack et ne peut être partagé avec aucun autre Nerdpack.
* `NerdStorageVault` ne peut actuellement être utilisé que dans le cadre d&amp;apos;un Nerdpack.
* Vous ne pouvez pas effectuer de requête directe ou de mutation sur `NerdStorageVault`.
* Vous pouvez explorer les champs et les types de réponse via l&apos;[explorateur d&apos;API NerdGraph](https://api.newrelic.com/graphiql).
* La seule portée prise en charge est ACTOR, ou les données relatives à un utilisateur particulier.

## Bibliothèque de composants [#component]

Actuellement, `NerdStorageVault` est disponible dans la version alpha de [la bibliothèque de composants communautaires Nr1](https://www.npmjs.com/package/@newrelic/nr1-community/v/1.3.0-alpha.5) et n&amp;apos;est pas disponible dans la bibliothèque de composants de base Nr1. Les composants open source peuvent être utilisés pour augmenter les composants de base Nr1 pour des fonctionnalités supplémentaires.

## Requête NerdGraph [#nerdgraph]

Vous trouverez ci-dessous des exemples de requêtes utilisées par les composants pour interagir avec `NerdStorageVault`.

<Callout variant="tip">
  `NerdStorageVault` ne peut actuellement être utilisé que dans le cadre d&amp;apos;un Nerdpack.
</Callout>

### Écrire [#write]

```graphql
mutation {
  nerdStorageVaultWriteSecret(
    scope: { actor: CURRENT_USER }
    secret: { key: "test", value: "secret_test" }
  ) {
    status
    errors {
      message
      type
    }
  }
}
```

### Supprimer [#delete]

```graphql
mutation {
  nerdStorageVaultDeleteSecret(
    scope: { actor: CURRENT_USER }
    key: "test_key"
  ) {
    status
    errors {
      message
      type
    }
  }
}
```

### Récupérer un seul secret [#fetch-single]

```graphql
query {
  actor {
    nerdStorageVault {
      secret(key: "test_key") {
        key
        value
      }
    }
  }
}
```

### Récupérer une liste de secrets [#fetch-list]

```graphql
query {
  actor {
    nerdStorageVault {
      secrets {
        key
        value
      }
    }
  }
}
```

### Limites [#limits]

* Une valeur secrète est limitée à 5000 caractères.
* Une valeur clé est limitée à 64 caractères.
* A key value must only include alphanumeric, &apos;\_&apos; or &apos;-&apos; characters

### Autorisations pour travailler avec `NerdStorageVault` [#permissions]

Afin de conserver les modifications apportées à `NerdStorageVault`, telles que l&amp;apos;écriture, la suppression et la récupération de données, vous devez disposer d&amp;apos;un [rôle d&apos;utilisateur avec l&apos;autorisation de conserver les modifications](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model).
