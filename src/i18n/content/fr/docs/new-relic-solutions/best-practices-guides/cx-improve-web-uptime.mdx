Ce guide comprend quelques idées sur la façon d&apos;améliorer les temps de disponibilité du Web.

## 1. Enquêter sur les contrôles synthétiques

Enquêter et résoudre les échecs de vérification des synthétiques lorsqu&apos;ils se produisent. Conseils pour procéder ainsi :

* Vérifiez plusieurs emplacements.
* Configurez une alerte pour vous avertir lorsque plusieurs emplacements échouent.

## 2. Créer une charge de travail

Créez une charge de travail qui met en corrélation le moniteur Synthétique avec les applications, les services et infrastructure du navigateur. Inclure un lien vers la workload dans le runbook d’alerte. Vous pouvez également trouver la workload corrélée pour un Synthétique en utilisant la recherche globale.

Assurez-vous que <InlinePopover type="alerts" />sont configurés pour chaque niveau de votre workload. De cette façon, vous pouvez voir la santé de chacun des niveaux dans une seule vue. Cela vous fera gagner du temps lors du dépannage. Vous n&amp;apos;avez pas besoin de créer une notification d&amp;apos;alerte pour chaque niveau pour bénéficier de cette vue.

## 3. Enquêter d’abord sur les pannes au niveau le plus bas

Lorsqu&apos;une panne se produit, commencez à enquêter au niveau le plus bas qui émet une alerte. Par instance, si vous constatez que vous avez un problème infrastructure et un problème JavaScript, examinez infrastructure avant JavaScript, à moins que vous n’ayez une deuxième personne ou une deuxième équipe à qui vous pouvez déléguer cette tâche.

Utilisez les outils mis à votre disposition pour le dépannage :

* Assurez-vous que le tracing distribué est activé pour <InlinePopover type="browser" />ainsi que pour <InlinePopover type="apm" />.
* Utilisez l’interface utilisateur du navigateur pour vous aider à comprendre ce qui se passe au niveau de l’utilisateur final.
* Utilisez Lookout pour vous aider à comprendre ce qui cause les problèmes de bagottement ou récurrents.
