Après avoir exécuté l’interface CLI de diagnostic à partir de votre terminal, vous verrez les résultats de chaque tâche au fur et à mesure qu’elles sont terminées. Les tâches qui génèrent un code d&apos;état `Warning` ou `Failure` log des détails supplémentaires concernant les éventuels problèmes détectés lors de l&amp;apos;exécution, ainsi que des suggestions de dépannage et des liens pertinents vers la documentation. Si vous utilisez l&amp;apos;indicateur [`-attach` ou `-api-key`](/docs/new-relic-solutions/solve-common-issues/diagnostics-cli-nrdiag/pass-command-line-options-nrdiag) , vous pouvez afficher vos résultats dans [l&apos;interface utilisateur](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/interpret-nrdiag-output/#diag-in-nr1)

## Sortie du fichier

L&apos;interface de ligne de commande de diagnostic génère trois fichiers :

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        Fichiers de sortie de diagnostic
      </th>

      <th>
        Commentaires
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `nrdiag-output.zip`
      </td>

      <td>
        Une structure de dossier aplatie avec un ou plusieurs fichiers de configuration et tout log existant de New Relic. Joignez ce fichier à votre compte New Relic. Cette archive contient également une copie de `nrdiag-output.json`.
      </td>
    </tr>

    <tr>
      <td>
        `nrdiag-output.json`
      </td>

      <td>
        Résultats des tests individuels. Attachez cette sortie à votre compte New Relic.
      </td>
    </tr>

    <tr>
      <td>
        `nrdiag-filelist.txt`
      </td>

      <td>
        Une liste des fichiers trouvés. Le fichier `nrdiag-output.zip` que vous joignez à votre compte New Relic inclut automatiquement cette liste. Vous n’avez pas besoin de joindre ce fichier `.txt` séparément.
      </td>
    </tr>
  </tbody>
</table>

## Codes d&apos;état des résultats [#status-codes]

L&apos;interface de ligne de commande de diagnostic renvoie les codes d&apos;état suivants après l&apos;exécution :

<table>
  <thead>
    <tr>
      <th style={{ width: "125px" }}>
        Code d&apos;état
      </th>

      <th>
        Définition
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `Success`
      </td>

      <td>
        Tâche exécutée avec succès sans aucun problème détecté.
      </td>
    </tr>

    <tr>
      <td>
        `Warning`
      </td>

      <td>
        Tâche exécutée avec succès. Cependant, des problèmes possibles ont été détectés.
      </td>
    </tr>

    <tr>
      <td>
        `Failure`
      </td>

      <td>
        La tâche a échoué avec des problèmes détectés.
      </td>
    </tr>

    <tr>
      <td>
        `Error`
      </td>

      <td>
        Tâche impossible à exécuter. Cela pourrait être dû à des problèmes d&apos;autorisations.
      </td>
    </tr>

    <tr>
      <td>
        `None`
      </td>

      <td>
        La tâche a été jugée non pertinente pour l’environnement détecté et n’a pas été exécutée.
      </td>
    </tr>
  </tbody>
</table>

## Sortie de diagnostic CLI dans New Relic [#diag-in-nr1]

<img title="Diagnostics CLI Output application in New Relic" alt="Diagnostics CLI Output application in New Relic" src="/images/solutions_screenshot-crop_nerdlet-diag.webp" />

Vous pouvez examiner la sortie `nrdiag-output.json` dans l&amp;apos;interface utilisateur. Le compte que vous sélectionnez est associé à la clé de licence validée par le CLI de diagnostic ou la clé API que vous avez fournie pour le téléchargement.

1. Cliquez sur le point d’interrogation dans le coin supérieur droit [de l’interface utilisateur](https://one.newrelic.com).
2. Sélectionnez l’onglet <DNT>**Diagnostics Home**</DNT> .
3. Dans la section Sortie CLI de diagnostic, cliquez sur <DNT>**View Run Details**</DNT>.
4. Cela vous amènera à votre dernière course. Vous pouvez sélectionner les courses précédentes sur le côté gauche.
5. Pour chaque exécution, vous verrez les statuts de chaque tâche, avec les avertissements et les échecs comme objectif principal. Vous pouvez également voir toutes les sorties de log ou script trouvées par la CLI.
