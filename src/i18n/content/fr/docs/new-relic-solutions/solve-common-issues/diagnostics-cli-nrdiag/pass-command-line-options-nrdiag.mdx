Pour utiliser les options de ligne de commande suivantes avec l’interface de ligne de commande de diagnostic :

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Option
      </th>

      <th>
        Usage
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `-a`

        `-attach`
      </td>

      <td>
        Joindre pour téléchargement automatique sur un compte New Relic. La taille maximale de téléchargement est de 4 Go. Ceci utilise un <InlinePopover type="licenseKey" />validé de votre environnement.
      </td>
    </tr>

    <tr>
      <td>
        `-api-key`
      </td>

      <td>
        utilisateur clé API à utiliser lors du téléchargement des résultats sur votre compte New Relic. La taille maximale de téléchargement est de 4 Go. Cela contourne l&apos;utilisation d&apos;une clé de licence et utilise votre <InlinePopover type="userKey" />pour joindre les résultats à la place.
      </td>
    </tr>

    <tr>
      <td>
        `-r`

        `-region`
      </td>

      <td>
        La [région](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) dans laquelle votre compte New Relic est hébergé. Valeurs acceptées : UE ou US. Insensible à la casse. Par défaut : États-Unis
      </td>
    </tr>

    <tr>
      <td id="browser-check">
        `-browser-url STRING`
      </td>

      <td>
        Version 1.1.9 de la CLI de diagnostic ou supérieur

        Lorsqu&apos;il est invoqué, cela exécutera uniquement les vérifications de diagnostic liées à <InlinePopover type="browser" />. Cette commande vérifie que l&apos;agent monitoring des navigateurs de New Relic est présent et renvoie la version de l&apos;agent, la méthode injection ([via <InlinePopover type="apm" />](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app)ou [via copier/coller](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app)) et le type de chargeur (Pro, Lite, SPA). À utiliser pour fournir des détails au support New Relic lors du dépannage des sites intranet.
      </td>
    </tr>

    <tr>
      <td>
        `-c STRING`

        `-config-file STRING`
      </td>

      <td>
        Remplacer l’emplacement du fichier de configuration de l’agent par défaut. Peut être utilisé pour spécifier soit un dossier à rechercher en plus des dossiers par défaut, soit un chemin vers un fichier de configuration spécifique.
      </td>
    </tr>

    <tr>
      <td>
        `-filter STRING`
      </td>

      <td>
        Filtrer les résultats des tâches par statut de résultat. Accepte une liste séparée par des virgules. Accepte `Success`, `Warning`, `Failure`, `Error`, `None` ou `Info`. Exemple de syntaxe :

        ```
        "Success,Warning,Failure"
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-h`

        `-help`
      </td>

      <td>
        Afficher la liste complète des options de ligne de commande. Pour lister toutes les tâches à exécuter, utilisez `-h tasks`. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-include STRING`
      </td>

      <td>
        Inclure un fichier ou un répertoire (y compris les sous-répertoires) dans `nrdiag-output.zip`. Limite de 4 Go. Utilisez l&apos;indicateur `-a` ou `-api-key` pour télécharger les résultats sur New Relic.

        Exemple de syntaxe :

        ```
        -include ./path/to/file
        ```

        ```
        -include \some\path\
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-interactive`
      </td>

      <td>
        Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-k8s-namespace STRING`
      </td>

      <td>
        Spécifiez l&apos;espace de nommage d&apos;où extraire les ressources New Relic. Si vous utilisez Agent-control, vous pouvez également définir l&apos;indicateur &apos;-ac-agents-espace de nommage&apos; pour spécifier l&apos;espace de nommage dans lequel les agents Agent-control s&apos;exécutent.

        Exemple de syntaxe :

        ```
        -k8s-namespace newrelic
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-ac-agents-namespace STRING`
      </td>

      <td>
        Spécifiez l&apos;espace de nommage à partir duquel récupérer les agents en cours d&apos;exécution de l&apos;Agent-control.

        Exemple de syntaxe :

        ```
        -ac-agents-namespace newrelic
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-output-path STRING`
      </td>

      <td>
        Spécifie un répertoire de sortie différent pour écrire les résultats pour `nrdiag-output.zip`, `nrdiag-output.json` et `nrdiag-filelist.txt`. L&apos;emplacement par défaut est `./`.
      </td>
    </tr>

    <tr>
      <td>
        `-o`

        `-override STRING`
      </td>

      <td>
        Transmettez des arguments à remplacer lorsque demandé par le support New Relic. Format : `identifier.property=value`.

        Exemple de syntaxe :

        ```
        -override Java/Config/Agent.Status=Success
        ```

        ```
        -o Base/Config/Validate.agentlanguage=PHP
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-p STRING`

        `-proxy STRING`
      </td>

      <td>
        Fournir un proxy à utiliser dans les tâches de connexion HTTP. Peut être HTTP ou HTTPS. Le proxy doit être au format `http(s)://proxyIp:proxy`. Si l&apos;interface de ligne de commande de diagnostic trouve un proxy dans le fichier de configuration de l&apos;agent, elle utilisera ce proxy par défaut. Dans la plupart des cas, `port` n&apos;est pas nécessaire. Si l&apos;interface de ligne de commande de diagnostic trouve un proxy dans le fichier de configuration de l&apos;agent, elle utilisera ce proxy par défaut.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-pw STRING`
      </td>

      <td>
        Mot de passe proxy, si nécessaire. Si l&apos;interface de ligne de commande de diagnostic trouve un proxy dans le fichier de configuration de l&apos;agent, elle utilisera ce proxy par défaut.
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-user STRING`
      </td>

      <td>
        Nom d&apos;utilisateur proxy, si nécessaire. Si l&apos;interface de ligne de commande de diagnostic trouve un proxy dans le fichier de configuration de l&apos;agent, elle utilisera ce proxy par défaut.
      </td>
    </tr>

    <tr>
      <td>
        `-q`
      </td>

      <td>
        La sortie silencieuse imprime uniquement les résultats de haut niveau et non la sortie explicative. Supprime les avertissements d&apos;ajout de fichier si `-y` est également utilisé. Ne contredit pas `-v`. Les filtres d’inclusion sont ignorés. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-qq`
      </td>

      <td>
        Une sortie très silencieuse n&apos;imprime qu&apos;une seule ligne de résumé pour la sortie (implique `q`). Supprime les avertissements d&apos;ajout de fichier si `-y` est également utilisé. Ne contredit pas `-v`. Les filtres d’inclusion sont ignorés. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-skip-version-check`
      </td>

      <td>
        Ignorer la vérification automatique d&apos;une version plus récente de l&apos;application. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-s STRING`

        `-suites STRING`
      </td>

      <td>
        Exécutez une [suite](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag/#suites), un ensemble de tâches ciblant des produits ou des problèmes spécifiques. Pour spécifier plusieurs suites, séparez-les par des virgules. Pour obtenir une liste de toutes les suites, exécutez :

        ```
        ./nrdiag -h suites
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-t STRING`

        `-tasks STRING`
      </td>

      <td>
        Exécutez uniquement un sous-ensemble de tâches, soit par agent, soit par type de tâche. Pour spécifier plusieurs tâches, séparez-les par des virgules et/ou par un caractère générique `*`. Pour obtenir une liste de toutes les tâches, exécutez :

        ```
        ./nrdiag -h tasks
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-usage-opt-out`
      </td>

      <td>
        Refuser d&apos;envoyer des données d&apos;utilisation de l&apos;outil CLI de diagnostic anonymes à New Relic pour cette exécution. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-list-scripts`
      </td>

      <td>
        Liste des scripts disponibles.
      </td>
    </tr>

    <tr>
      <td>
        `-script STRING`
      </td>

      <td>
        Afficher le script spécifié. Utiliser avec `-run` pour exécuter le script.
      </td>
    </tr>

    <tr>
      <td>
        `-run`
      </td>

      <td>
        Utiliser avec `-script` pour exécuter le script.
      </td>
    </tr>

    <tr>
      <td>
        `-script-flags`
      </td>

      <td>
        Utiliser avec `-run -script` pour transmettre les indicateurs de ligne de commande au script.
      </td>
    </tr>

    <tr>
      <td>
        `-v`

        `-verbose`
      </td>

      <td>
        Afficher le logging détaillé pendant l&apos;exécution de la vérification. Désactivé par défaut. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-version`
      </td>

      <td>
        Affiche la version actuelle de la CLI de diagnostic. Cela vous invite également à vérifier si une version plus récente est disponible et à la télécharger si une version plus récente est disponible. A priorité sur `-skip-version-check`. Type : booléen
      </td>
    </tr>

    <tr>
      <td>
        `-y`

        `-yes`
      </td>

      <td>
        Répondez `yes` à toute prompt qui s’affiche pendant l’exécution. Désactivé par défaut. Type : booléen
      </td>
    </tr>
  </tbody>
</table>
