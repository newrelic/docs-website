## Problème

Vous ne voyez pas de données dans l&apos;interface utilisateur après [l&apos;installation d&apos;un agent New Relic](/docs/agents/manage-apm-agents/installation/install-agent).

## Solution

Vous devriez commencer à voir des données quelques minutes après l’installation d’un agent New Relic et la génération de trafic pour votre application. Si vous ne voyez pas de données, vous pouvez utiliser l’utilitaire [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) pour identifier automatiquement les problèmes courants. Pour obtenir des conseils de dépannage supplémentaires, consultez les documents spécifiques à l&amp;apos;agent. 

### agent APM

Suivez les procédures de dépannage pour votre agent <InlinePopover type="apm" /> :

* [Go](/docs/agents/go-agent/troubleshooting/no-data-appears-go)
* [Java](/docs/agents/java-agent/troubleshooting/no-data-appears-java)
* [.NET](/docs/agents/net-agent/troubleshooting/no-data-appears-net)
* [Node.js](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-your-nodejs-installation)
* [PHP](/docs/agents/php-agent/troubleshooting/no-data-appears-php)
* [Python](/docs/agents/python-agent/troubleshooting/no-data-appears-python)
* [Ruby](/docs/agents/ruby-agent/troubleshooting/no-data-appears-ruby)

De plus, vous pouvez essayer ces étapes de dépannage qui s&apos;appliquent à tous les agents APM :

<CollapserGroup>
  <Collapser id="deleted_apps" title="Applications supprimées ou renommées dans APM">
    Une application doit cesser de signaler des données pendant au moins une heure avant que vous puissiez réutiliser ce nom. Il doit également se reconnecter au collecteur New Relic (être redémarré) avant que de nouvelles données ne soient acceptées.

    L&apos;application reste dans le cache du collecteur pendant une heure avant d&apos;être vidée. Pendant ce temps, les données sont marquées comme « supprimées », donc aucune nouvelle donnée n&apos;est acceptée. En outre, les données sont associées à une application en cours d’exécution qui a été supprimée jusqu’au redémarrage de l’agent.

    Pour plus d&apos;informations, voir :

    * [Nommez votre application](/docs/site/naming-your-application)
    * [Utiliser plusieurs noms pour une application](/docs/site/using-multiple-names-for-an-app)
  </Collapser>

  <Collapser id="data_collection" title="Aucune connexion au collecteur">
    Votre application ne sera pas affectée si l&apos;agent New Relic ne peut pas se connecter au [collecteur](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#collector). Les données continuent d’être collectées et elles sont téléchargées dès que la connexion est rétablie.

    Lorsque le réseau est en panne ou que le collecteur est indisponible, vous pouvez voir des lacunes où des données sont manquantes dans les graphiques CPU et mémoire APM. L&apos;agent continuera d&apos;essayer de se reconnecter et, une fois l&apos;opération terminée, vous verrez à nouveau les données apparaître dans l&apos;interface utilisateur.

    Pendant que l&apos;agent est incapable de communiquer avec le collecteur, il continue de collecter des données. Une fois qu&apos;il est en mesure de se connecter à nouveau, il téléchargera les données et remplira le segment manquant afin qu&apos;il n&apos;y ait aucune confusion quant à savoir si votre application était en panne ou ne rapportait tout simplement pas de données. Pour économiser de la mémoire, les données seront agrégées et moyennées sur la période, de sorte que vous verrez des barres plates et des graphiques sur la période pendant laquelle il n&apos;a pas été possible de communiquer avec le collecteur.
  </Collapser>
</CollapserGroup>
