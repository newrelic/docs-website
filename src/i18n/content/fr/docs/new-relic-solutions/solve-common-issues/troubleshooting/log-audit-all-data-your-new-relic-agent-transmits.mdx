Chaque agent New Relic comprend de solides mesures de protection pour garantir [la sécurité des données](/docs/accounts-partnerships/accounts/security/data-security). Par exemple, New Relic crypte automatiquement les informations sensibles avant de les transmettre. Pour plus d&amp;apos;informations sur les mesures de sécurité de New Relic, consultez notre [documentation sur la sécurité et la confidentialité](/docs/using-new-relic/new-relic-security/security/security-matters-data-privacy-new-relic) ou visitez le [site Web de sécurité de New Relic](https://newrelic.com/security).

Si vous devez enregistrer et afficher des informations sur <DNT>**all**</DNT> données que votre application transmet à New Relic, vous pouvez activer le logging d&amp;apos;audit pendant de courtes périodes. Cela est utile, par exemple, pour le débogage ou l&amp;apos;audit, lorsque vous avez besoin d&amp;apos;informations détaillées sur ce qui est exactement transmis.

<Callout variant="caution">
  Assurez-vous de désactiver le logging d’audit dès que vous avez fini de l’utiliser. Cette fonctionnalité entraîne une surcharge supplémentaire, qui peut surcharger le fichier log d&apos;audit si elle reste activée pendant des périodes prolongées.
</Callout>

## Logging liée au compte New Relic [#account-logging]

Pour auditer les modifications apportées à votre compte New Relic, exécutez la requête NRQL avec [`NrAuditEvent`](/docs/insights/use-insights-ui/manage-account-data/query-account-audit-logs-nrauditevent). Pour personnaliser votre requête, utilisez l&amp;apos;un des [attributs`NrAuditEvent` ](/docs/insights/insights-data-sources/default-data/nrauditevent-default-attributes-examples)disponibles.
