New Relic Pathpoint peut être intégré à presque tous les comptes New Relic pour fournir une observabilité complète des processus métier. Pour commencer, il est important de disposer de données télémétriques pertinentes pour votre organisation. Cela peut inclure des métriques, des événements, des logs ou des traces. Bien que l&apos;événement et le log APM soient un point de départ commun pour Pathpoint, le choix des données télémétriques vous appartient entièrement.

Avant de commencer à utiliser Pathpoint, vous devez l&apos;installer sur votre compte New Relic.

## Installer Pathpoint depuis le New Relic catalog [#install-catalog]

Pour installer Pathpoint :

<Steps>
  <Step>
    Allez à <DNT>**[Apps &amp; Visualizations](https://onenr.io/0yw40Vz6Ew3)**</DNT>.
  </Step>

  <Step>
    Recherchez <DNT>**Pathpoint V2**</DNT> (vous devez spécifier `V2` pour obtenir la nouvelle base de code. (L&apos;ancienne version est obsolète).
  </Step>

  <Step>
    Cliquez sur l’icône d’installation du <DNT>**Pathpoint V2**</DNT> vérifié, puis cliquez sur <DNT>**Add this app**</DNT>.

    <img title="Install Pathpoint" alt="Install Pathpoint" src="/images/pathpoint_v2_screenshot_install_pathpoint.webp" />

    Pathpoint est désormais ajouté à votre compte New Relic et vous pouvez y accéder sous <DNT>**Apps**</DNT>.
  </Step>
</Steps>

Une fois Pathpoint installé, vous pouvez configurer vos étapes, vos pas et vos signaux simplement en utilisant l&apos;interface utilisateur interactive. Vous pouvez également exporter et importer une configuration JSON, ce qui est utile pour le clonage et le contrôle des versions de configuration ainsi que pour les déplacer entre les comptes New Relic. [En savoir plus sur la configuration de Pathpoint](/docs/new-relic-solutions/business-observability/intro-pathpoint/).
