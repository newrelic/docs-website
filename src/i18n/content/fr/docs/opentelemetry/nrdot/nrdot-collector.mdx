La distribution New Relic d&apos;OpenTelemetry Collector (NRDOT) est une collection de distributions prédéfinies d&apos;OpenTelemetry [Collector](https://opentelemetry.io/docs/collector/) prises en charge par New Relic. NRDOT est conçu pour fonctionner de manière transparente avec les produits et services New Relic, et c&apos;est le moyen recommandé pour collecter, traiter et exporter des données télémétriques ouvertes à utiliser avec New Relic. Chaque distribution est livrée avec des configurations préconfigurées pour s&apos;intégrer à l&apos;ingestion New Relic.

Plutôt que de fournir une distribution unique et globale d&apos;OpenTelemetry Collector, nous nous sommes concentrés sur la fourniture de distributions qui sont :

* Léger
* Portée limitée
* Adapté aux expériences New Relic spécifiques
* Préconfiguré pour signaler au point de terminaison OTLP de New Relic
* Soutenu

Cette approche offre à l’utilisateur une expérience simplifiée et bien testée, permettant des améliorations plus rapides de nos distributions. Chaque distribution est conçue pour un cas d&apos;utilisation spécifique de New Relic, aidant l&apos;utilisateur à intégrer facilement OpenTelemetry avec New Relic. Nous visons un chevauchement minimal entre les distributions, leur permettant de fonctionner aux côtés d&apos;autres collecteurs si nécessaire. De plus, les utilisateurs avancés peuvent créer des distributions personnalisées avec des composants supplémentaires à l&apos;aide d&apos;[OCB](https://opentelemetry.io/docs/collector/custom-collector/).

<Callout variant="tip">
  Il existe des scénarios dans lesquels vous souhaiterez peut-être déployer le collecteur de manière moins subjective. Nous vous recommandons de lire notre documentation sur [le traitement du collecteur](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) pour comprendre comment configurer le collecteur pour votre cas d&apos;utilisation spécifique.
</Callout>

Avant de vous plonger dans l&apos;une de nos distributions, nous vous recommandons de vous familiariser avec les concepts de base d&apos;OpenTelemetry Collector pour comprendre comment nos configurations sont structurées et comment les personnaliser selon vos besoins.

## déploierons [#deployments]

Le collecteur lui-même est un binaire unique qui peut être déployé de diverses manières. Les stratégies de déploiement typiques sont :

* En tant qu&apos;agent : déployer sur le même hôte que l&apos;application.
* En tant que gateway : se déployer comme point central de collecte de télémétrie.

Chaque distribution NRDOT est configurée prête à l&apos;emploi avec un déploiement spécifique à l&apos;esprit, mais peut être personnalisée pour répondre à vos besoins en fournissant des configurations personnalisées. Des informations supplémentaires sur les modèles de déploiement peuvent être trouvées dans la [documentation d&apos;OpenTelemetry Collector](https://opentelemetry.io/docs/collector/deployment/).
