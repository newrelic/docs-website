Cette intégration Kubernetes découvre automatiquement les pods HAProxy dans votre cluster et collecte des métriques sans configuration manuelle pour chaque instance. Elle utilise le [haproxyreceiver](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/haproxyreceiver) et le [receivercreator](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/receivercreator) OpenTelemetry pour monitorer dynamiquement les sessions HAProxy, les requests, les erreurs et l&apos;état de santé du serveur backend dans votre environnement conteneurisé.

<Callout variant="tip">
  Vous exécutez HAProxy sur des hôtes Linux ? Consultez le [guide d&apos;installation auto-hébergée](/docs/opentelemetry/integrations/haproxy/self-hosted) pour les déploiements sur machine virtuelle ou bare metal.
</Callout>

## Avant de commencer [#prerequisites]

Assurez-vous d&apos;avoir :

* [Clé de licence](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)New Relic valide
* Les pods HAProxy avec le [point de terminaison de statistiques activé](https://www.haproxy.com/documentation/haproxy-configuration-manual/latest/#4-stats%20enable) (généralement sur le port 8404)
* Un label sur chaque pod HAProxy pour la découverte (par exemple, `app: haproxy`)
* Si vous utilisez la méthode d&apos;installation Helm, terminez l&apos;installation de base de [Helm Kubernetes OpenTelemetry](/docs/kubernetes-pixie/k8s-otel/install/#helm-install)

        <Collapser id="nrdot-k8s-helm-install" title="Installation Helm">
          1. Ajoutez les sections suivantes sous [`deployment.extraConfig`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/values.yaml) dans votre `values.yaml`:

          <Callout variant="important">
            Dans `deployment.extraConfig`, `pipelines:` doit être défini au niveau racine — et non imbriqué sous `service:`. Le modèle de chart Helm mappe `extraConfig.pipelines` dans `service.pipelines`, et `extraConfig.service.extensions` dans `service.extensions`.
          </Callout>

          Extensions :

          ```yaml
          extensions:
            k8s_observer:
              auth_type: serviceAccount
              observe_pods: true
              observe_nodes: false
          ```

          Variables d&apos;environnement (ajoutez sous `deployment.envs`) :

          ```yaml
          deployment:
            envs:
              - name: OTEL_RESOURCE_ATTRIBUTES
                value: "host.id=$(KUBE_NODE_NAME)"
          ```

          <Callout variant="important">
            L&apos;attribut de ressource `host.id` est requis pour la synthèse d&apos;entités. Les images de conteneur n&apos;ont pas `/etc/machine-id`, vous devez donc l&apos;injecter via `OTEL_RESOURCE_ATTRIBUTES`. Le chart Helm définit déjà `KUBE_NODE_NAME` à partir de l&apos;API Downward de Kubernetes.
          </Callout>

          Récepteurs :

          ```yaml
          receivers:
            receiver_creator/haproxy:
              watch_observers: [k8s_observer]
              receivers:
                haproxy:
                  rule: type == "pod" && labels["app"] == "haproxy"
                  config:
                    endpoint: 'http://`endpoint`:8404/stats'
                    collection_interval: 30s
          ```

          Processeurs :

          ```yaml
          processors:
            resource/cluster:
              attributes:
                - key: k8s.cluster.name
                  value: "<YOUR_CLUSTER_NAME>"
                  action: insert

            transform/haproxy:
              metric_statements:
                - context: resource
                  statements:
                    - set(attributes["haproxy.display.name"], Concat([
                        "server",
                        "k8s",
                        attributes["k8s.cluster.name"],
                        attributes["k8s.namespace.name"],
                        "pod",
                        attributes["k8s.pod.name"],
                        "haproxy",
                        "8404"
                      ], ":"))

            transform/metadata_nullify:
              metric_statements:
                - context: metric
                  statements:
                    - set(description, "")
                    - set(unit, "")
          ```

          Extensions de service :

          ```yaml
          service:
            extensions: [k8s_observer]
          ```

          Pipelines :

          ```yaml
          pipelines:
            metrics/haproxy:
              receivers: [receiver_creator/haproxy]
              processors: [batch, resource/cluster, transform/haproxy, transform/metadata_nullify, memory_limiter]
              exporters: [otlphttp/newrelic]
          ```

        <Collapser id="nrdot-k8s-manifest-install" title="Installation via manifeste">
          Après avoir terminé l&apos;installation de base du [manifeste Kubernetes OpenTelemetry](/docs/kubernetes-pixie/k8s-otel/install/#manifest-install), configurez le monitoring de HAProxy :

          1. Mettre à jour l&apos;image du collecteur pour utiliser le collecteur NRDOT.

             Dans le fichier [`deployment.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/deployment.yaml) de votre répertoire local `rendered`, mettez à jour l&apos;image vers :

             ```yaml
             image: newrelic/nrdot-collector:latest
             ```

          2. Montez le `/etc/machine-id` de l&apos;hôte dans le conteneur du collecteur en ajoutant ces éléments à la spécification de déploiement :

             ```yaml
             volumeMounts:
               - name: machine-id
                 mountPath: /etc/machine-id
                 readOnly: true
             volumes:
               - name: machine-id
                 hostPath:
                   path: /etc/machine-id
                   type: File
             ```

             <Callout variant="important">
               L&apos;attribut de ressource `host.id` est requis pour la synthèse d&apos;entités. Le processeur `resourcedetection` lit `/etc/machine-id` pour le renseigner. Sans ce montage de volume, `host.id` ne sera pas défini et les entités ne seront pas créées.
             </Callout>

          3. Remplacez le contenu dans [`deployment-configmap.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/deployment-configmap.yaml) sous `deployment-config.yaml: |` par :

             ```yaml
             extensions:
               health_check:
                 endpoint: 0.0.0.0:13133
               k8s_observer:
                 auth_type: serviceAccount
                 observe_pods: true
                 observe_nodes: false

             receivers:
               receiver_creator/haproxy:
                 watch_observers: [k8s_observer]
                 receivers:
                   haproxy:
                     rule: type == "pod" && labels["app"] == "haproxy"
                     config:
                       endpoint: 'http://`endpoint`:8404/stats'
                       collection_interval: 30s

             processors:
               resourcedetection:
                 detectors: [env, system]
                 system:
                   resource_attributes:
                     host.name:
                       enabled: true
                     host.id:
                       enabled: true
                     os.type:
                       enabled: true

               batch:
                 send_batch_max_size: 1000
                 timeout: 30s
                 send_batch_size: 800

               memory_limiter:
                 check_interval: 1s
                 limit_percentage: 80
                 spike_limit_percentage: 25

               resource/cluster:
                 attributes:
                   - key: k8s.cluster.name
                     value: "<YOUR_CLUSTER_NAME>"
                     action: insert

               transform/haproxy:
                 metric_statements:
                   - context: resource
                     statements:
                       - set(attributes["haproxy.display.name"], Concat([
                           "server",
                           "k8s",
                           attributes["k8s.cluster.name"],
                           attributes["k8s.namespace.name"],
                           "pod",
                           attributes["k8s.pod.name"],
                           "haproxy",
                           "8404"
                         ], ":"))

               transform/metadata_nullify:
                 metric_statements:
                   - context: metric
                     statements:
                       - set(description, "")
                       - set(unit, "")

             exporters:
               otlp_http/newrelic:
                 endpoint: "<YOUR_NEWRELIC_OTLP_ENDPOINT>"
                 headers:
                   api-key: ${env:NR_LICENSE_KEY}

             service:
               extensions: [health_check, k8s_observer]
               pipelines:
                 metrics/haproxy:
                   receivers: [receiver_creator/haproxy]
                   processors: [resourcedetection, batch, resource/cluster, transform/haproxy, transform/metadata_nullify, memory_limiter]
                   exporters: [otlp_http/newrelic]
             ```

          4. Supprimez le DaemonSet et le déploiement kube-state-metrics (non nécessaires pour le monitoring de HAProxy uniquement) :

             ```bash
             kubectl delete daemonset nr-k8s-otel-collector-daemonset -n newrelic
             kubectl delete deployment nr-k8s-otel-collector-kube-state-metrics -n newrelic
             ```

          5. Redémarrez le déploiement du collecteur pour appliquer vos modifications :

             ```bash
             kubectl rollout restart deployment nr-k8s-otel-collector-deployment -n newrelic
             ```

                 mountPath: /etc/machine-id
                 readOnly: true

             config:

        <Collapser id="otelcontrib-manifest-install" title="Installation via manifeste">
          Utilisez cette option si vous ne pouvez pas utiliser Helm ou si vous préférez gérer les ressources Kubernetes directement avec `kubectl`.

          1. Créez un secret avec votre clé de licence New Relic :

             ```bash
             kubectl create namespace newrelic
             kubectl create secret generic otel-collector-haproxy-secret \
               --namespace newrelic \
               --from-literal=licenseKey='<YOUR_NEW_RELIC_LICENSE_KEY>'
             ```

          2. Enregistrez le manifeste suivant dans un fichier (par exemple, `otel-collector-haproxy.yaml`) :

             ```yaml
             ---
             apiVersion: v1
             kind: ServiceAccount
             metadata:
               name: otel-collector-haproxy
               namespace: newrelic
               labels:
                 app: otel-collector-haproxy
             ---
             apiVersion: rbac.authorization.k8s.io/v1
             kind: ClusterRole
             metadata:
               name: otel-collector-haproxy
               labels:
                 app: otel-collector-haproxy
             rules:
               - apiGroups: [""]
                 resources: ["pods", "nodes"]
                 verbs: ["get", "list", "watch"]
               - apiGroups: ["apps"]
                 resources: ["replicasets"]
                 verbs: ["get", "list", "watch"]
             ---
             apiVersion: rbac.authorization.k8s.io/v1
             kind: ClusterRoleBinding
             metadata:
               name: otel-collector-haproxy
               labels:
                 app: otel-collector-haproxy
             roleRef:
               apiGroup: rbac.authorization.k8s.io
               kind: ClusterRole
               name: otel-collector-haproxy
             subjects:
               - kind: ServiceAccount
                 name: otel-collector-haproxy
                 namespace: newrelic
             ---
             apiVersion: v1
             kind: ConfigMap
             metadata:
               name: otel-collector-haproxy-config
               namespace: newrelic
               labels:
                 app: otel-collector-haproxy
             data:
               collector-config.yaml: |
                 extensions:
                   health_check:
                     endpoint: 0.0.0.0:13133
                   k8s_observer:
                     auth_type: serviceAccount
                     observe_pods: true
                     observe_nodes: false

                 receivers:
                   receiver_creator/haproxy:
                     watch_observers: [k8s_observer]
                     receivers:
                       haproxy:
                         rule: type == "pod" && labels["app"] == "haproxy"
                         config:
                           endpoint: 'http://`endpoint`:8404/stats'
                           collection_interval: 30s

                 processors:
                   resourcedetection:
                     detectors: [env, system]
                     system:
                       resource_attributes:
                         host.name:
                           enabled: true
                         host.id:
                           enabled: true
                         os.type:
                           enabled: true

                   batch:
                     send_batch_size: 1024
                     timeout: 30s

                   memory_limiter:
                     check_interval: 1s
                     limit_percentage: 80
                     spike_limit_percentage: 25

                   resource/cluster:
                     attributes:
                       - key: k8s.cluster.name
                         value: "<YOUR_CLUSTER_NAME>"
                         action: insert

                   transform/haproxy:
                     metric_statements:
                       - context: resource
                         statements:
                           - set(attributes["haproxy.display.name"], Concat(["server", "k8s", attributes["k8s.cluster.name"], attributes["k8s.namespace.name"], "pod", attributes["k8s.pod.name"], "haproxy", "8404"], ":"))

                   transform/metadata_nullify:
                     metric_statements:
                       - context: metric
                         statements:
                           - set(description, "")
                           - set(unit, "")

                 exporters:
                   otlp_http/newrelic:
                     endpoint: "<YOUR_NEWRELIC_OTLP_ENDPOINT>"
                     headers:
                       api-key: ${env:NR_LICENSE_KEY}

                 service:
                   extensions: [health_check, k8s_observer]
                   pipelines:
                     metrics/haproxy:
                       receivers: [receiver_creator/haproxy]
                       processors: [resourcedetection, batch, resource/cluster, transform/haproxy, transform/metadata_nullify, memory_limiter]
                       exporters: [otlp_http/newrelic]
             ---
             apiVersion: apps/v1
             kind: Deployment
             metadata:
               name: otel-collector-haproxy
               namespace: newrelic
               labels:
                 app: otel-collector-haproxy
             spec:
               replicas: 1
               selector:
                 matchLabels:
                   app: otel-collector-haproxy
               template:
                 metadata:
                   labels:
                     app: otel-collector-haproxy
                 spec:
                   serviceAccountName: otel-collector-haproxy
                   containers:
                   - name: otel-collector
                     image: otel/opentelemetry-collector-contrib:latest
                     command:
                     - "/otelcol-contrib"
                     - "--config=/conf/collector-config.yaml"
                     env:
                     - name: NR_LICENSE_KEY
                       valueFrom:
                         secretKeyRef:
                           name: otel-collector-haproxy-secret
                           key: licenseKey
                     - name: GOGC
                       value: "80"
                     ports:
                     - name: health
                       containerPort: 13133
                       protocol: TCP
                     resources:
                       limits:
                         cpu: 500m
                         memory: 512Mi
                       requests:
                         cpu: 100m
                         memory: 256Mi
                     volumeMounts:
                     - name: config
                       mountPath: /conf
                     - name: machine-id
                       mountPath: /etc/machine-id
                       readOnly: true
                   volumes:
                   - name: config
                     configMap:
                       name: otel-collector-haproxy-config
                       items:
                       - key: collector-config.yaml
                         path: collector-config.yaml
                   - name: machine-id
                     hostPath:
                       path: /etc/machine-id
                       type: File
             ```

             <Callout variant="important">
               Le montage de volume `/etc/machine-id` est requis pour la synthèse d&apos;entités. Le processeur `resourcedetection` lit ce fichier pour renseigner l&apos;attribut de ressource `host.id`. Sans cela, les entités ne seront pas créées dans New Relic.
             </Callout>

          3. Appliquez le manifeste :

             ```bash
             kubectl apply -f otel-collector-haproxy.yaml
             ```

          4. Vérifiez que le collecteur est en cours d&apos;exécution et découvre les pods HAProxy :

             ```bash
             kubectl get pods -n newrelic -l app=otel-collector-haproxy
             kubectl logs -n newrelic -l app=otel-collector-haproxy
             ```

             Vous devriez voir des lignes de log indiquant `starting receiver` avec les adresses IP de vos pods HAProxy.

## Affichez vos données dans New Relic [#find-data]

Après quelques minutes, les métriques HAProxy apparaissent dans New Relic. Pour trouver vos données :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities**</DNT>.
2. Recherchez vos entités HAProxy par nom ou filtrez par type d&apos;entité `HAPROXYINSTANCE`.
3. Sélectionnez une entité pour afficher la page de résumé avec les métriques dorées (sessions par seconde, requests par seconde, erreurs de connexion).

## Dépannage [#troubleshooting]

<CollapserGroup>
  <Collapser id="no-data" title="Aucune donnée n'apparaît dans New Relic">
    * Vérifiez que les pods HAProxy ont la bonne étiquette (par exemple, `app: haproxy`) : `kubectl get pods -l app=haproxy`
    * Vérifiez que le point de terminaison des statistiques répond : `kubectl exec -it <haproxy-pod> -- curl -s http://localhost:8404/stats`
    * Consultez les logs du collecteur pour les erreurs de découverte : `kubectl logs -n newrelic -l app.kubernetes.io/name=nr-k8s-otel-collector -f` (NRDOT) ou `kubectl logs -n newrelic -l app=otel-collector-haproxy -f` (manifeste OTel Contrib)
    * Assurez-vous que le ServiceAccount du collecteur dispose des permissions `get`, `list` et `watch` sur les pods.
  </Collapser>

  <Collapser id="single-entity" title="Plusieurs pods HAProxy s'affichent comme une seule entité">
    Chaque pod HAProxy doit créer une entité distincte dans New Relic. Si vous ne voyez qu&apos;une seule entité :

    * Vérifiez que le collecteur a `host.id` défini. Dans les environnements conteneurisés, ajoutez ceci au déploiement du collecteur :
      ```yaml
      env:
        - name: K8S_NODE_NAME
          valueFrom:
            fieldRef:
              fieldPath: spec.nodeName
        - name: OTEL_RESOURCE_ATTRIBUTES
          value: "host.id=$(K8S_NODE_NAME)"
      ```
    * Confirmez les différentes valeurs de `haproxy.addr` par pod en effectuant une requête : `FROM Metric SELECT uniques(haproxy.addr) WHERE metricName LIKE 'haproxy.%' SINCE 10 minutes ago`
  </Collapser>

  <Collapser id="stats-unreachable" title="Collector ne peut pas atteindre le point de terminaison de statistiques HAProxy">
    * Le `receiver_creator` utilise l&apos;adresse IP du pod (et non `localhost`) pour atteindre HAProxy. Assurez-vous que le frontend de statistiques est lié à `*:8404` (toutes les interfaces), et non à `127.0.0.1:8404`.
    * Vérifiez les stratégies réseau qui pourraient bloquer le trafic entre le pod du collecteur et les pod HAProxy.
  </Collapser>
</CollapserGroup>
