Après avoir configuré le monitoring HAProxy avec OpenTelemetry, vos données apparaissent dans New Relic en quelques minutes. Cette page vous montre comment trouver vos entités HAProxy, explorer les métriques et écrire des requêtes NRQL.

## Trouvez vos données HAProxy [#find-data]

<CollapserGroup>
  <Collapser id="method1-entities" title="Utilisez l'explorateur d'entités">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities**</DNT>.
    2. Recherchez vos entités HAProxy par nom, ou filtrez par type d&apos;entité `HAPROXYINSTANCE`.
    3. Sélectionnez une entité pour afficher la page de résumé avec les métriques dorées (sessions par seconde, requests par seconde, erreurs de connexion).
  </Collapser>

  <Collapser id="method2-dashboard" title="Utilisez les intégrations et les agents">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Integrations &amp; Agents**</DNT>.
    2. Cliquez sur <DNT>**Dashboards**</DNT>.
    3. Recherchez <DNT>**HAProxy (OpenTelemetry Collector)**</DNT> et sélectionnez-le.
    4. Cliquez sur <DNT>**View dashboard**</DNT> pour voir des visualisations prédéfinies pour vos métriques HAProxy.
  </Collapser>

  <Collapser id="method3-query" title="Utilisez le générateur de requêtes">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Query your data**</DNT>.
    2. Exécutez une requête pour trouver vos données HAProxy :
       ```sql
       FROM Metric SELECT uniques(metricName)
       WHERE metricName LIKE 'haproxy.%'
       SINCE 1 hour ago
       ```
  </Collapser>
</CollapserGroup>

## Interrogez vos données avec NRQL [#query-data]

Utilisez ces exemples de requêtes NRQL pour analyser vos données HAProxy :

```sql
-- Sessions per second across frontends
FROM Metric SELECT average(haproxy.sessions.rate)
WHERE haproxy.service_name = 'FRONTEND'
FACET haproxy.proxy_name
TIMESERIES

-- Error rates by backend
FROM Metric SELECT rate(sum(haproxy.responses.errors), 1 minute) AS 'Errors/min'
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name
TIMESERIES

-- Bytes throughput (in + out)
FROM Metric SELECT rate(sum(haproxy.bytes.input), 1 minute) AS 'Bytes In/min',
  rate(sum(haproxy.bytes.output), 1 minute) AS 'Bytes Out/min'
FACET haproxy.proxy_name
TIMESERIES

-- Backend server health
FROM Metric SELECT latest(haproxy.active) AS 'Active Servers',
  latest(haproxy.backup) AS 'Backup Servers'
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name

-- Request queue depth (indicates backend saturation)
FROM Metric SELECT max(haproxy.requests.queued)
WHERE haproxy.service_name = 'BACKEND'
FACET haproxy.proxy_name
TIMESERIES
```

## Dépanner la visibilité des données [#troubleshoot]

<CollapserGroup>
  <Collapser id="troubleshoot-no-data" title="Aucune donnée n'apparaît dans New Relic">
    * Prévoyez quelques minutes pour l’ingestion initiale des données.
    * Vérifiez la plage de temps — assurez-vous de consulter les données récentes (les 30 dernières minutes).
    * Requête pour toutes les métriques HAProxy :
      ```sql
      FROM Metric SELECT count(*) WHERE metricName LIKE 'haproxy.%' SINCE 30 minutes ago
      ```
    * Vérifiez que le collecteur est en cours d&apos;exécution :
      ```bash
      # Self-hosted
      sudo systemctl status otelcol-contrib
      # or
      sudo systemctl status nrdot-collector

      # Kubernetes
      kubectl get pods -n newrelic
      kubectl logs -n newrelic -l app.kubernetes.io/name=nr-k8s-otel-collector
      ```
    * Vérifiez que le point de terminaison des statistiques HAProxy est accessible depuis le collecteur.
  </Collapser>

  <Collapser id="troubleshoot-intermittent-data" title="Les données apparaissent par intermittence">
    * Vérifiez les logs du collecteur pour les erreurs :
      ```bash
      # Self-hosted
      sudo journalctl -u otelcol-contrib -n 50 --no-pager

      # Kubernetes
      kubectl logs -n newrelic deploy/nr-k8s-otel-collector-deployment --tail=50
      ```
    * Vérifiez que le point de terminaison des statistiques HAProxy est constamment accessible.
    * Vérifiez les problèmes de connectivité réseau entre le collecteur et le point de terminaison OTLP de New Relic.
    * Assurez-vous que HAProxy ne limite pas le débit des requests du point de terminaison des statistiques.
  </Collapser>
</CollapserGroup>
