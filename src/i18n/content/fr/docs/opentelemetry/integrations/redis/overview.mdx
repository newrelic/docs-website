La plupart des applications s’appuient sur Redis pour les éléments qui doivent être rapides, tels que la mise en cache, le stockage de session, les classements en temps réel et les files d’attente des messages. Mais lorsque Redis ralentit, vous le remarquez généralement d’abord ailleurs, comme une page lente ou une requête qui expire. Sans une bonne visibilité, il est difficile de dire si le vrai problème est une mémoire insuffisante, des clés expulsées de manière trop agressive, une baisse du taux d’accès au cache ou un réplica qui a pris du retard.

New Relic fournit un chemin de monitoring basé sur OpenTelemetry pour Redis. Le [récepteur Redis](https://github.com/open-telemetry/opentelemetry-collector-contrib/tree/main/receiver/redisreceiver) de l&apos;OpenTelemetry Collector collecte les métriques que Redis expose via sa commande INFO et les envoie à New Relic, sans nécessiter d&apos;agent séparé. New Relic synthétise chaque instance Redis comme sa propre entité, regroupe les déploiements en cluster sous une entité Redis Cluster, et fournit des dashboards et des alertes préconfigurés.

## Démarrer [#get-started]

Pour monitorer Redis avec OpenTelemetry, suivez trois étapes : passez en revue les prérequis, configurez le collecteur et consultez vos données.

### Prérequis [#prereq]

Assurez-vous que votre environnement respecte tous les prérequis pour l&apos;intégration définie pour votre déploiement :

* Pour [Auto-hébergé](/docs/opentelemetry/integrations/redis/host/#prereq)
* Pour [Docker](/docs/opentelemetry/integrations/redis/docker/#prereq)
* Pour [Kubernetes](/docs/opentelemetry/integrations/redis/kubernetes/#prereq)

### Configuration du Collector [#collector-setup]

Choisissez votre parcours d’installation en fonction de votre infrastructure :

* Pour [Auto-hébergé](/docs/opentelemetry/integrations/redis/host)
* Pour [Docker](/docs/opentelemetry/integrations/redis/docker)
* Pour [Kubernetes](/docs/opentelemetry/integrations/redis/kubernetes)

### Visualisez vos données [#view-data]

Une fois que vous avez terminé la configuration du collecteur, vous pouvez afficher vos métriques Redis dans New Relic, les interroger avec NRQL, créer des dashboards et configurer des alertes. Pour plus d&apos;informations, consultez la documentation [trouver et interroger vos données](/docs/opentelemetry/integrations/redis/view-data).

L&apos;intégration Redis de New Relic suit une grande variété de métriques. Pour en savoir plus sur les types de données et les attributs pris en charge, consultez le guide de [référence des métriques Redis](/docs/opentelemetry/integrations/redis/metrics).

## Prochaines étapes [#next-steps]

Après avoir installé l&apos;intégration :

* **[Consultez vos données](/docs/opentelemetry/integrations/redis/view-data)**: explorez les dashboards, exécutez des requêtes NRQL et configurez des alertes pour vos métriques Redis.
* **[Consultez la référence des métriques](/docs/opentelemetry/integrations/redis/metrics)**: comprenez toutes les métriques disponibles et leurs attributs.
* **[Dépannage](/docs/opentelemetry/integrations/redis/troubleshooting)**: Résolvez les problèmes courants liés à la collecte de données et à la synthèse des entités.
