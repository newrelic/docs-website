Après avoir configuré le monitoring OpenTelemetry pour votre environnement Amazon ECS, vous pouvez trouver et interroger vos données dans les interfaces d&apos;infrastructure et de requête de New Relic. Ce guide vous aide à localiser vos métriques ECS et à créer des requêtes pertinentes pour le monitoring et le dépannage.

## Où trouver vos données ECS [#find-data]

Vos données de monitoring ECS apparaissent à plusieurs endroits dans New Relic, en fonction du type de métriques et de votre type de lancement ECS.

### Monitoring de l&apos;infrastructure [#infrastructure-view]

Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Infrastructure**</DNT> pour afficher votre infrastructure ECS :

#### Pour ECS sur EC2

1. Vue Hôtes : filtrez par hôte pour voir vos instances EC2 exécutant des conteneurs ECS
2. Vue des conteneurs : voir les performances individuelles des conteneurs dans vos tâches ECS

#### Pour ECS Fargate

1. Vue des conteneurs : emplacement principal pour les métriques de conteneur Fargate

<Callout variant="tip">
  Utilisez les options de filtrage pour vous limiter à des clusters, services ou noms de conteneurs spécifiques afin de trouver rapidement les données dont vous avez besoin.
</Callout>

### Métriques et événement [#metrics-events]

Accédez à **<DNT>[one.newrelic.com](https://one.newrelic.com)</DNT> &gt; Query your data** pour :

* Créer des graphiques et des dashboards personnalisés
* Configurer des alertes basées sur des seuils de métrique spécifiques

## Exemples de requêtes [#query-examples]

### Monitorer l&apos;utilisation du processeur du conteneur

```sql
SELECT average(container.cpu.utilization)
FROM Metric
WHERE ClusterName = 'your-cluster-name'
AND ServiceName = 'your-service-name'
TIMESERIES AUTO
FACET container.name
```

### Suivre les tendances de l&apos;utilisation de la mémoire

```sql
SELECT max(container.memory.usage.total) / 1024 / 1024 as 'Memory (MB)'
FROM Metric
WHERE ClusterName = 'your-cluster-name'
TIMESERIES AUTO
FACET ServiceName
```

### Monitorer les métriques de l&apos;hôte (EC2 uniquement)

```sql
SELECT average(1 - system.cpu.utilization) * 100 AS 'CPU used %',
       average(system.memory.utilization) * 100 AS 'Memory used %'
FROM Metric
WHERE aws.ecs.launchtype = 'ec2'
AND aws.ecs.cluster.arn = 'your-cluster-arn'
TIMESERIES AUTO
```

## Dépannage des lacunes dans les données [#troubleshooting]

Si vous ne voyez pas les données attendues :

1. **Vérifier le statut du collecteur**: vérifiez les logs du conteneur pour le collecteur OpenTelemetry
2. **Confirmez la connectivité réseau**: assurez-vous de l&apos;accès HTTPS sortant vers New Relic
3. **Vérifier la clé de licence**: vérifiez que la clé de licence est correctement configurée dans SSM Parameter Store
4. **Passez en revue la définition de tâche**: confirmez que le conteneur du collecteur est en cours d&apos;exécution avec une allocation de ressources appropriée
