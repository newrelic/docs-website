Cette référence complète documente toutes les métriques RabbitMQ collectées par l&apos;intégration OpenTelemetry Collector. Chaque métrique comprend des descriptions détaillées et des exemples pratiques de requêtes NRQL.

<Callout variant="tip">
  **Vous découvrez le mointoring de RabbitMQ ?** Commencez par le [guide de présentation](/docs/opentelemetry/integrations/rabbitmq/overview) pour comprendre l&apos;architecture de l&apos;intégration et les options de déploiement.
</Callout>

## Comprendre les métriques RabbitMQ [#overview]

L&apos;intégration RabbitMQ OpenTelemetry collecte des métriques via l&apos;API de gestion RabbitMQ, offrant une visibilité complète sur la santé et les performances de votre courtier de messages.

**Catégories de métriques :**

* **Métriques de nœud** - Utilisation des ressources au niveau du nœud, notamment la mémoire, le disque, les descripteurs de fichiers et les statistiques de la VM Erlang
* **Métriques de file d&apos;attente** - Nombre de messages, activité des consommateurs et débit au niveau de la file d&apos;attente par file d&apos;attente
* **Métriques des messages** - Taux de publication, de remise et d&apos;acquittement des messages
* **Métriques d&apos;E/S** - Opérations de lecture/écriture sur disque et latence pour la persistance des messages
* **Métriques de connexion** - Événements du cycle de vie des connexions et des canaux

**Attributs de métrique communs :**

Toutes les métriques incluent ces attributs pour le filtrage et le regroupement :

* `instrumentation.provider = 'opentelemetry'` - Identifie les métriques de cette intégration
* `rabbitmq.deployment.name` - Votre identifiant de déploiement
* `rabbitmq.server.endpoint` - Point de terminaison de l&apos;API de gestion
* `host.name` - Nom d&apos;hôte sur lequel RabbitMQ s&apos;exécute

<Callout variant="important">
  Lorsque vous interrogez les métriques RabbitMQ, incluez toujours `instrumentation.provider = 'opentelemetry'` dans votre clause WHERE pour vous assurer de ne récupérer que les données de cette intégration.
</Callout>

## Métriques des ressources du nœud [#node-resource-metrics]

Les métriques de ressources de nœud suivent la mémoire, le disque, les descripteurs de fichiers et les ressources de la VM Erlang pour chaque nœud RabbitMQ.

### mémoires métriques

<CollapserGroup>
  <Collapser id="node-mem-used" title="rabbitmq.node.mem_used">
    **Type**: Jauge\
    **Unité**: Octets\
    **Description**: Mémoire totale utilisée par le nœud RabbitMQ en octets.

    **Cas d&apos;utilisation**: Monitorer la consommation de mémoire des nœuds pour éviter les situations de mémoire insuffisante.

    **Exemples NRQL :**

    Afficher l&apos;utilisation actuelle de la mémoire :

    ```sql
    SELECT latest(rabbitmq.node.mem_used) / 1024 / 1024 AS 'Memory Used (MB)'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
      AND metricName = 'rabbitmq.node.mem_used'
    FACET host.name
    ```

    Suivre les tendances de la mémoire :

    ```sql
    SELECT latest(rabbitmq.node.mem_used) / 1024 / 1024 AS 'Memory Used (MB)'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
      AND metricName = 'rabbitmq.node.mem_used'
    FACET host.name
    TIMESERIES
    ```

    Pourcentage d&apos;utilisation de la mémoire :

    ```sql
    SELECT 
      (latest(rabbitmq.node.mem_used) / latest(rabbitmq.node.mem_limit)) * 100 AS 'Memory Utilization %'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
    FACET host.name
    TIMESERIES
    ```
  </Collapser>

  <Collapser id="node-mem-limit" title="rabbitmq.node.mem_limit">
    **Type**: Jauge\
    **Unité**: Octets\
    **Description**: Limite de mémoire configurée pour le nœud RabbitMQ.

    **Cas d&apos;utilisation**: Comparer à la mémoire utilisée pour calculer l&apos;utilisation et déclencher des alarmes.

    **Exemples NRQL :**

    Afficher les limites de mémoire :

    ```sql
    SELECT latest(rabbitmq.node.mem_limit) / 1024 / 1024 / 1024 AS 'Memory Limit (GB)'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
      AND metricName = 'rabbitmq.node.mem_limit'
    FACET host.name
    ```

    Calcul de la marge mémoire :

    ```sql
    SELECT 
      (latest(rabbitmq.node.mem_limit) - latest(rabbitmq.node.mem_used)) / 1024 / 1024 AS 'Available Memory (MB)'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
    FACET host.name
    ```
  </Collapser>

  <Collapser id="node-mem-alarm" title="rabbitmq.node.mem_alarm">
    **Type**: Jauge\
    **Unité**: Booléen (0 ou 1)\
    **Description**: État de l&apos;alarme mémoire (0 = aucune alarme, 1 = alarme déclenchée).

    **Cas d&apos;utilisation**: Alerter en cas de dépassement du seuil de mémoire.

    **Exemples NRQL :**

    Vérifier l&apos;état de l&apos;alarme :

    ```sql
    SELECT latest(rabbitmq.node.mem_alarm) AS 'Memory Alarm'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
      AND metricName = 'rabbitmq.node.mem_alarm'
    FACET host.name
    ```

    Alerte sur l&apos;alarme mémoire :

    ```sql
    SELECT latest(rabbitmq.node.mem_alarm)
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
      AND metricName = 'rabbitmq.node.mem_alarm'
      AND rabbitmq.node.mem_alarm = 1
    FACET host.name
    ```
  </Collapser>

  <Collapser id="node-mem-used-rate" title="rabbitmq.node.mem_used_details.rate">
    **Type**: Jauge\
    **Unité**: Octets par seconde\
    **Description**: Taux de variation de l&apos;utilisation de la mémoire par seconde.

    **Cas d&apos;utilisation**: Monitorer le taux de croissance de la mémoire pour détecter les fuites de mémoire.

    **Exemples NRQL :**

    Voir le taux de croissance de la mémoire :

    ```sql
    SELECT latest(rabbitmq.node.mem_used_details.rate) / 1024 AS 'Memory Growth Rate (KB/s)'
    FROM Metric
    WHERE instrumentation.provider = 'opentelemetry'
      AND metricName = 'rabbitmq.node.mem_used_details.rate'
    FACET host.name
    TIMESERIES
    ```
  </Collapser>
</CollapserGroup>

Les métriques de file d&apos;attente fournissent des informations sur le flux de messages, les backlogs et l&apos;activité des consommateurs par file d&apos;attente.

<CollapserGroup>

    ```sql
    SELECT latest(rabbitmq.consumer.count) AS 'Consumers'

      AND metricName = 'rabbitmq.consumer.count'
    ```
