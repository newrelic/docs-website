Une fois votre monitoring RabbitMQ configuré avec OpenTelemetry, vos métriques apparaîtront à plusieurs endroits dans New Relic. Ce guide vous indique où trouver vos données et comment les interroger efficacement.

## Accédez à vos données RabbitMQ [#find-data]

Vos métriques RabbitMQ apparaissent à plusieurs endroits sur la plateforme New Relic, chacun optimisé pour différents cas d&apos;utilisation :

### 1. Explorateur d&apos;entités

Idéal pour : Contrôles de santé rapides et relations d&apos;entités

1. Aller à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; On host integrations**</DNT>
2. Voir les entités RabbitMQ créées automatiquement à partir de vos métriques

### 2. Dashboards prédéfinis

Idéal pour : Monitoring et visualisation complets

1. Aller à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards &gt; Recommended dashboards (View all)**</DNT>
2. Rechercher <DNT>**&quot;OpenTelemetry RabbitMQ&quot;**</DNT>

### 3. Services tiers

Idéal pour : le monitoring axé sur l&apos;intégration

1. Aller à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; Infrastructure &gt; Third-party services**</DNT>
2. Recherchez <DNT>**&quot;RabbitMQ&quot;**</DNT> pour trouver rapidement vos instances monitorées

## Interroger vos données RabbitMQ [#query-data]

Toutes les métriques RabbitMQ incluent l&apos;attribut `instrumentation.provider = 'opentelemetry'`. Voici quelques requêtes utiles :

### Voir toutes les métriques disponibles

```sql
SELECT count(*)
FROM Metric
WHERE metricName LIKE 'rabbitmq.%'
  AND instrumentation.provider = 'opentelemetry'
FACET metricName
SINCE 10 minutes ago
```

### Monitorer la profondeur des files d&apos;attente

```sql
SELECT latest(rabbitmq.queue.message.count)
FROM Metric
WHERE instrumentation.provider = 'opentelemetry'
FACET rabbitmq.queue.name
TIMESERIES
```

### Suivre le retard du consommateur

```sql
SELECT latest(rabbitmq.queue.message.unacknowledged)
FROM Metric
WHERE instrumentation.provider = 'opentelemetry'
  AND rabbitmq.queue.name = 'your-queue-name'
TIMESERIES
```
