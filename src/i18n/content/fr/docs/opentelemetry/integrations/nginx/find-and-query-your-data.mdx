## Affichez vos données NGINX dans New Relic [#find-data]

Une fois votre configuration terminée et les données en cours, vous pouvez afficher vos métriques et logs NGINX dans New Relic et créer des alertes personnalisées.

<CollapserGroup>
  <Collapser id="method1-integrations" title="Via les intégrations et les agents">
    1. Aller à <DNT>[one.newrelic.com](https://one.newrelic.com) &gt; **Integrations &amp; Agents**</DNT>
    2. Cliquez <DNT>**Dashboards**</DNT>
    3. Recherchez et cliquez sur le dashboard <DNT>**NGINX OTel overview**</DNT>
    4. Une page <DNT>**Dashboard setup**</DNT> s&apos;ouvre. Si vous avez déjà configuré la source de données, cliquez sur <DNT>**Skip this step**</DNT>. Sinon, cliquez sur <DNT>**See installation docs**</DNT> et suivez les instructions pour configurer votre source de données. Une fois la configuration terminée, retournez à la page de configuration du dashboard et cliquez sur <DNT>**Skip this step**</DNT>.
    5. Cliquez sur <DNT>**View dashboard**</DNT>.
  </Collapser>

  <Collapser id="method2-infrastructure" title="Grâce au monitoring d'infrastructure">
    1. Aller à <DNT>[one.newrelic.com](https://one.newrelic.com) &gt; **On-host integrations**</DNT>
    2. Recherchez votre serveur NGINX par nom de déploiement (par exemple, `production-web-01`)
    3. Cliquez sur votre serveur pour voir les métriques détaillées
  </Collapser>
</CollapserGroup>

## Interrogez vos données avec NRQL

Vous pouvez créer des requêtes personnalisées pour analyser vos données NGINX :

Exemple de requête :

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## Dépanner la visibilité des données

<CollapserGroup>
  <Collapser id="troubleshoot-no-data" title="Si vous ne voyez pas de données">
    * Prévoyez un peu de temps - L&apos;ingestion initiale des données peut prendre quelques minutes
    * Vérifiez la plage de temps - Assurez-vous de consulter les données récentes
    * Vérifier le nom du déploiement - Utilisez le nom exact de votre configuration de collecteur
    * Vérifiez les données avec NRQL :
      ```sql
      FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
      ```
  </Collapser>

  <Collapser id="troubleshoot-intermittent-data" title="Si les données apparaissent par intermittence">
    * Vérifiez les logs du collecteur pour les erreurs :
      ```bash
      sudo journalctl -u otelcol-contrib -n 50
      ```
    * Vérifiez que NGINX génère du trafic - Test avec `curl http://localhost`
    * Vérifiez l&apos;authentification - Assurez-vous que la clé de licence et le point de terminaison sont corrects
  </Collapser>
</CollapserGroup>
