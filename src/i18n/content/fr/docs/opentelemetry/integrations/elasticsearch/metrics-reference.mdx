Ce guide de référence fournit un aperçu complet de toutes les métriques collectées par l&apos;intégration OpenTelemetry pour Elasticsearch, vous aidant à comprendre quelles données sont disponibles pour monitorer la santé et les performances de votre cluster.

### Utiliser efficacement les attributs de ressource

Ces attributs permettent des scénarios de monitoring puissants :

**Gestion multi-environnements :**

* Filtrer les dashboards par `elasticsearch.cluster.name` pour la production vs le staging
* Créer des stratégies d&apos;alerte spécifiques au cluster
* Comparer les performances dans différents environnements

**Corrélation de l&apos;infrastructure :**

* Utilisez `host.name` ou `k8s.pod.name` pour corréler avec le monitoring de l&apos;infrastructure
* Suivez l&apos;utilisation des ressources au niveau de l&apos;hôte ou du pod
* Identifiez les modèles de performance sur l&apos;ensemble de votre infrastructure

**Monitoring spécifique à Kubernetes :**

* Monitorer Elasticsearch dans plusieurs espaces de noms
* Suivez les événements du cycle de vie des pods et leur impact sur les performances
* Créer des alertes basées sur la santé des déploiements Kubernetes
