Monitorez votre cluster Elasticsearch auto-hébergé en installant le collecteur OpenTelemetry directement sur des serveurs ou des machines virtuelles. New Relic propose des options de déploiement flexibles pour répondre à votre configuration d&apos;infrastructure et à vos exigences de monitoring.

Vous pouvez choisir entre deux distributions de collecteur :

* **NRDOT :** Distribution New Relic d&apos;OpenTelemetry
* **OTel Collector Contrib :** Collecteur OpenTelemetry standard avec des composants issus de la communauté

## Options d&apos;installation [#installation-options]

        <Step>
          ### Avant de commencer [#prereq-otel]

          Avant de configurer l&apos;OTel Collector Contrib, assurez-vous d&apos;avoir :

          **Privilèges d&apos;accès requis :**

          * Votre New Relic<InlinePopover type="licenseKey" />
          * Privilèges root ou sudo sur la machine hôte
          * Privilèges d&apos;administration de cluster Elasticsearch avec le privilège de cluster `monitor` ou `manage` (voir la [documentation sur les privilèges de sécurité Elasticsearch](https://www.elastic.co/guide/en/elasticsearch/reference/current/security-privileges.html) pour plus de détails)

          **Configuration requise :**

          * **Elasticsearch version 7.16 ou supérieure** - Cette intégration nécessite un cluster Elasticsearch moderne
          * **Connectivité réseau** - HTTPS sortant (port 443) vers le [point de terminaison d&apos;ingestion OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp)de New Relic

          **Valeurs de configuration prêtes :**

          * **Point de terminaison Elasticsearch** - L&apos;URL de votre cluster Elasticsearch (par ex. `http://localhost:9200`)
          * **Nom du cluster** - Un identifiant unique pour votre cluster dans New Relic

          <Callout variant="important">
            Vous devez avoir installé [OpenTelemetry Collector Contrib](https://github.com/open-telemetry/opentelemetry-collector-contrib/releases/latest) sur votre hôte avant de continuer. Nous recommandons l&apos;installation via les paquets officiels (.deb ou .rpm) pour garantir que l&apos;unité de service systemd est créée correctement.
          </Callout>
        </Step>

        <Step>
          ### Configurer le monitoring d&apos;Elasticsearch [#configure-es-otel]

          Une fois l&apos;OTel Collector Contrib installé, remplacez le fichier de configuration par défaut du collecteur par la configuration du monitoring Elasticsearch. Cela activera la collecte des métriques Elasticsearch. Les métriques et les logs de l&apos;hôte sont facultatifs et peuvent être ajoutés séparément.

          Le fichier de configuration se trouve à : `/etc/otelcol-contrib/config.yaml`

          <Callout variant="tip">
            **Sauvegardez votre configuration par défaut :** Avant de modifier le fichier de configuration, créez une copie de sauvegarde pour conserver les paramètres par défaut :

            ```bash
            sudo cp /etc/otelcol-contrib/config.yaml /etc/otelcol-contrib/config.yaml.backup
            ```
          </Callout>

          1. Ouvrez le fichier de configuration avec un éditeur de texte en utilisant les privilèges root ou sudo :

             ```bash
             sudo nano /etc/otelcol-contrib/config.yaml
             ```

                   load:

             Et ajoutez aux pipelines de service :

             ```yaml
             service:
               pipelines:
                 metrics/host:
                   receivers: [hostmetrics]
                   processors: [memory_limiter, resourcedetection, batch]
                   exporters: [otlphttp]
             ```

          5. *(Facultatif)* Pour collecter les logs Elasticsearch, ajoutez le récepteur filelog. Assurez-vous que l&apos;utilisateur exécutant le service du collecteur (otelcol-contrib) dispose d&apos;un accès en lecture à vos fichiers de logs Elasticsearch :

             **Si vous exécutez Elasticsearch sur Linux (hôte) :**

             ```yaml
             receivers:
               filelog:
                 include:
                   - /var/log/elasticsearch/elasticsearch.log
                   - /var/log/elasticsearch/*.log
             ```

             **Si vous exécutez Elasticsearch dans Docker :**

             ```yaml
             receivers:
               filelog:
                 include:
                   - /var/lib/docker/containers/*/*.log
                 operators:
                   - type: move
                     from: attributes.log
                     to: body
             ```

             Et ajoutez aux pipelines de service :

             ```yaml
             service:
               pipelines:
                 logs:
                   receivers: [filelog]
                   processors: [resource/cluster_name_override]
                   exporters: [otlphttp]
             ```

          6. *(Facultatif)* Pour ajouter des tags de métadonnées personnalisés à vos métriques, utilisez le processeur `resource/static_override` :

             ```yaml
             processors:
               resource/static_override:
                 attributes:
                   - key: env
                     value: "production"
                     action: upsert
             service:
               pipelines:
                 metrics/elasticsearch:
                   receivers: [elasticsearch]
                   processors: [memory_limiter, resourcedetection, resource/cluster_name_override, resource/static_override, attributes/cardinality_reduction, cumulativetodelta, transform/metadata_nullify, batch]
                   exporters: [otlphttp]
             ```

          7. Enregistrez le fichier de configuration.

          8. Définissez les variables d&apos;environnement :

             Créer un répertoire de remplacement systemd :

             ```bash
             sudo mkdir -p /etc/systemd/system/otelcol-contrib.service.d
             ```

             Créez le fichier de configuration de l&apos;environnement :

             ```bash
             cat <<EOF | sudo tee /etc/systemd/system/otelcol-contrib.service.d/environment.conf
             [Service]
             Environment="NEWRELIC_OTLP_ENDPOINT=https://otlp.nr-data.net:4318"
             Environment="NEWRELIC_LICENSE_KEY=YOUR_LICENSE_KEY_HERE"
             Environment="NEW_RELIC_MEMORY_LIMIT_MIB=100"
             EOF
             ```

             Mettez à jour la configuration avec vos valeurs :

             * Remplacez `https://otlp.nr-data.net:4318` par le point de terminaison de votre région
             * Remplacez `YOUR_LICENSE_KEY_HERE` par votre clé de licence New Relic
             * Remplacez `100` par la limite de mémoire souhaitée en MiB pour le collecteur (par défaut : 100 MiB). Ajustez en fonction des besoins de votre environnement

          9. Redémarrez l&apos;OTel Collector Contrib pour appliquer les modifications :

             ```bash
             sudo systemctl daemon-reload
             sudo systemctl restart otelcol-contrib.service
             ```

        <Step>
          ### Vérifier la collecte de données [#verify-otel]

          Vérifiez que l&apos;OTel Collector Contrib est en cours d&apos;exécution et collecte des données sans erreur :

          1. Vérifier le statut du service du collecteur :

             ```bash
             sudo systemctl status otelcol-contrib.service
             ```

          2. Monitorez les logs du collecteur pour détecter d&apos;éventuelles erreurs :

             ```bash
             sudo journalctl -u otelcol-contrib.service -f
             ```

             Recherchez les connexions réussies à Elasticsearch et New Relic. Si vous rencontrez des erreurs, consultez le [guide de dépannage](/docs/opentelemetry/integrations/elasticsearch/troubleshooting).

<Callout variant="tip">
  **Corréler APM avec Elasticsearch**: pour connecter votre application APM et votre cluster Elasticsearch, incluez l&apos;attribut de ressource `es.cluster.name="your-cluster-name"` dans vos métriques APM. Cela permet une visibilité inter-services et un dépannage plus rapide au sein de New Relic.
</Callout>

## Afficher vos données Elasticsearch [#find-and-use]

Une fois que le collecteur est en cours d&apos;exécution et envoie des données, vous pouvez explorer vos métriques Elasticsearch, créer des requêtes personnalisées et configurer des dashboards de monitoring dans New Relic.

Pour des informations détaillées sur l&apos;accès à vos données, l&apos;écriture de requêtes NRQL et la configuration d&apos;alertes, consultez [Rechercher et interroger les données Elasticsearch](/docs/opentelemetry/integrations/elasticsearch/find-and-query-data).

## Dépannage [#troubleshooting]

Si vous rencontrez des problèmes lors de l&apos;installation ou si vous ne voyez pas de données dans New Relic, consultez notre [guide de dépannage](/docs/opentelemetry/integrations/elasticsearch/troubleshooting) complet pour obtenir des solutions étape par étape aux problèmes courants.
