## Pourquoi monitorer NGINX avec OpenTelemetry ? [#why-monitor]

Le monitoring NGINX est essentielle pour maintenir les performances et la fiabilité des applications web. OpenTelemetry fournit une approche standardisée pour collecter et analyser ces données critiques.

### Principaux avantages

* **Optimisation des performances**: identifiez les goulots d&apos;étranglement, suivez les temps de réponse, prenez des décisions de mise à l&apos;échelle basées sur les données.
* **Monitoring proactif**: Recevez des alertes avant l&apos;impact sur l&apos;utilisateur, détectez les problèmes rapidement
* **Informations opérationnelles**: comprenez les modèles de trafic, suivez la conformité des SLA, maintenez les pistes d&apos;audit.
* **Pérenne**: intégration OpenTelemetry standard open source, neutre vis-à-vis des fournisseurs

### Cas d&apos;utilisation

Que vous exécutiez un site Web de commerce électronique, un gateway API ou une plateforme de diffusion de contenu, le monitoring NGINX vous aide à maintenir des services Web rapides et fiables. Soyez averti lorsque votre processus de paiement ralentit, détectez lorsque vos points de terminaison API sont surchargés ou identifiez les régions géographiques qui rencontrent des problèmes. Ce monitoring est essentiel pour les sites Web gérant les transactions clients, les backends d&apos;applications mobiles, les architectures de microservices et toute application Web où les temps d&apos;arrêt ou les performances lentes ont un impact direct sur vos utilisateurs et votre entreprise.

## Métriques disponibles [#metrics]

Le collecteur OpenTelemetry collecte automatiquement les principales métriques de performance NGINX, notamment les statistiques de connexion, le nombre de requêtes et les indicateurs d&apos;état du serveur.

**Principales métriques collectées :**

* **Métriques de connexion** - Connexions actives, acceptées, gérées et leurs états
* **Demander des métriques** - Nombre total de requêtes et taux de requêtes au fil du temps
* **État du serveur** - Indicateurs d&apos;efficacité et de performance de la connexion

**Pour une référence complète des métriques et des attributs :** Consultez [Référence des métriques et des attributs NGINX OpenTelemetry](/docs/opentelemetry/nginx/nginx-otel-metrics-reference/) pour des descriptions détaillées, des types de métriques et des attributs de ressources pour les déploiements sur hôte et Kubernetes.
