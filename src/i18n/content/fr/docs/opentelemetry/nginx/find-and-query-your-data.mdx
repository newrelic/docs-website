## Affichez vos données NGINX dans New Relic [#find-data]

Une fois votre configuration terminée et les données en cours, vous pouvez afficher vos métriques et logs NGINX dans New Relic et créer des alertes personnalisées.

**Méthode 1 : via les intégrations et les agents**

1. Accédez à [one.newrelic.com](https://one.newrelic.com) &gt; **Integrations &amp; Agents**
2. Cliquez sur **Dashboards**
3. Recherchez et cliquez sur le dashboard **NGINX OTel overview**
4. Sélectionnez votre compte et cliquez sur **View dashboard**

**Méthode 2 : Grâce au monitoring de l&apos;infrastructure**

1. Accédez à [one.newrelic.com](https://one.newrelic.com) &gt; **On-host integrations**
2. Recherchez votre serveur NGINX par nom de déploiement (par exemple, `production-web-01`)
3. Cliquez sur votre serveur pour voir les métriques détaillées

## Interrogez vos données avec NRQL

Vous pouvez créer des requêtes personnalisées pour analyser vos données NGINX :

**Exemple de requête :**

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## Dépanner la visibilité des données

Si vous ne voyez pas de données :

1. Prévoyez un peu de temps - L&apos;ingestion initiale des données peut prendre quelques minutes
2. Vérifiez la plage de temps - Assurez-vous de consulter les données récentes
3. Vérifier le nom du déploiement - Utilisez le nom exact de votre configuration de collecteur
4. Vérifiez les données avec NRQL :
   ```sql
   FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
   ```

Si les données apparaissent par intermittence :

1. Vérifiez les logs du collecteur pour les erreurs :
   ```bash
   sudo journalctl -u otelcol-contrib -n 50
   ```
2. Vérifiez que NGINX génère du trafic - Test avec `curl http://localhost`
3. Vérifiez l&apos;authentification - Assurez-vous que la clé de licence et le point de terminaison sont corrects
