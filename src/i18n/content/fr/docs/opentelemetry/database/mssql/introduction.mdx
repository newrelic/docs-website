Monitorez les performances de votre base de données Microsoft SQL Server à l&apos;aide de la distribution New Relic d&apos;OpenTelemetry (NRDOT) avec des informations détaillées complètes sur les métriques de base de données, les performances des requêtes et la santé du système.

Cette approche basée sur NRDOT complète notre On-Host Integration (OHI) existante en s&apos;appuyant sur les standards OpenTelemetry pour le monitoring des bases de données, facilitant ainsi l&apos;intégration avec votre pile d&apos;observabilité existante.

## Comment configurer NRDOT pour SQL Server [#configure]

<Steps>
  <Step>
    ### Prérequis [#prerequisites]

    Assurez-vous que votre environnement répond à nos [exigences de compatibilité et prérequis](/docs/opentelemetry/database/mssql/compatibility).
  </Step>

  <Step>
    ### Installation du Collector NRDOT [#installation]

    Choisissez votre parcours d’installation en fonction de votre infrastructure :

    * **Installation guidée**: configuration guidée par la plateforme via l’interface utilisateur de New Relic (recommandée pour les démarrages rapides)

    * **Environnements auto-hébergés**: installation manuelle sur votre propre infrastructure

      * **[Windows auto-hébergé](/docs/opentelemetry/database/mssql/windows-hosted)**
      * **[Linux auto-hébergé](/docs/opentelemetry/database/mssql/linux-hosted)**

    * **Environnements AWS RDS**: monitoring de SQL Server géré dans le cloud

      * **[Monitoring de Windows RDS](/docs/opentelemetry/database/mssql/windows-rds)**
      * **[Monitoring RDS Linux](/docs/opentelemetry/database/mssql/linux-rds)**
  </Step>

  <Step>
    ### Trouvez et utilisez vos données [#find]

    Une fois vos données collectées, vous pouvez accéder à un monitoring complet des bases de données SQL Server via l&apos;interface utilisateur de New Relic.

    Pour trouver votre entité de base de données SQL Server dans New Relic :

    1. Allez à <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>.
    2. Dans la liste déroulante **<DNT>Entity type</DNT>**, sélectionnez <DNT>**MSSQL instance**</DNT>, puis cliquez sur <DNT>**Apply**</DNT>.
    3. Sélectionnez votre base de données SQL Server dans la liste des entités.
  </Step>
</Steps>
