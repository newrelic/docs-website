<Callout title="Aperçu">
  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy/).
</Callout>

Vous pouvez installer le NRDOT Collector pour le monitoring de Microsoft SQL Server à l&apos;aide de l&apos;installation guidée depuis la plateforme New Relic, qui fournit une configuration guidée par la plateforme via l&apos;UI de New Relic. Il s&apos;agit de l&apos;approche recommandée pour les démarrages rapides.

<Steps>
  <Step>
    ### Prérequis [#prerequisites]

    Assurez-vous que votre environnement répond à nos [exigences de compatibilité et prérequis](/docs/opentelemetry/database/mssql/compatibility).
  </Step>

  <Step>
    ### Installation guidée [#guided]

    Pour installer le Collector NRDOT à l&apos;aide de l&apos;installation guidée :

    1. Connectez-vous à votre [compte New Relic](https://one.newrelic.com/).

    2. Allez à <DNT>**left navigation pane &gt; + Integration &amp; Agents &gt; MSSQL (OpenTelemetry)**</DNT>.

    3. Sur l&apos;écran <DNT>Select an account</DNT>, sélectionnez le compte sur lequel vous souhaitez installer l&apos;intégration MSSQL, puis cliquez sur <DNT>**Continue**</DNT>.

    4. Sur l&apos;écran <DNT>Enter your credential</DNT>, sélectionnez l&apos;une des options suivantes, puis cliquez sur **Continue**:

       * **Utilisez une clé existante**: si vous avez déjà une clé de licence, fournissez la clé de licence. Pour plus d’informations, consultez [Clés utilisateur](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
       * **Créer une nouvelle clé**: si vous n’avez pas de clé de licence, cliquez sur **Create a new key** pour en créer une.

    5. Sur l&apos;écran <DNT>Select platform</DNT>, sélectionnez la plateforme pour installer l&apos;intégration MSSQL, puis cliquez sur <DNT>**Continue**</DNT>.

    6. Suivez les instructions à l’écran pour configurer l’intégration pour la plateforme sélectionnée.

    7. Après l&apos;installation, sur l&apos;écran <DNT>Test your integration</DNT>, cliquez sur **Test connection** pour vérifier que le Collector NRDOT est capable de se connecter à votre instance SQL Server et d&apos;envoyer des données à New Relic.
  </Step>

  <Step>
    ### (Facultatif) Configurer la corrélation APM-base de données [#correlation]

    Pour corréler les performances de votre application avec les opérations de base de données, vous pouvez configurer l&apos;identification du service de base de données. Cette fonctionnalité vous permet de voir exactement quelles applications génèrent des workloads de base de données spécifiques. Pour plus d&apos;informations, reportez-vous à [configurer l&apos;identification du service de base de données](/docs/opentelemetry/database/database-service-identification/) pour obtenir la corrélation APM-base de données dans New Relic.

    <Callout variant="important">
      Pour afficher les données de performance de la base de données dans APM, les deux entités doivent se trouver dans le même compte New Relic. Si les entités se trouvent dans des comptes New Relic différents, vous devez avoir accès aux deux comptes pour visualiser les données.
    </Callout>
  </Step>

  <Step>
    ### (Facultatif) Configurer la gestion des secrets [#rds-secret-management]

    La fonctionnalité de gestion des secrets vous permet de gérer en toute sécurité des informations sensibles, telles que les identifiants de base de données. Cela permet de renforcer la sécurité de votre configuration de supervision en évitant de coder en dur des données sensibles dans les fichiers de configuration. Pour plus d&apos;informations, consultez la [gestion des secrets](/docs/opentelemetry/database/database-service-identification/#secret-management).
  </Step>

  <Step>
    ### Trouvez et utilisez vos données [#find]

    Une fois vos données collectées, vous pouvez accéder à un monitoring complet des bases de données SQL Server via l&apos;interface utilisateur de New Relic.

    Pour trouver votre entité de base de données SQL Server dans New Relic :

    1. Allez à <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>.
    2. Dans la liste déroulante **<DNT>Entity type</DNT>**, sélectionnez <DNT>**MSSQL instance**</DNT>, puis cliquez sur <DNT>**Apply**</DNT>.
    3. Sélectionnez votre base de données SQL Server dans la liste des entités.
  </Step>
</Steps>
