        <Step>
          ### Configurer le collecteur NRDOT [#setup-sql]

          Installez le Collector NRDOT sur une instance EC2 Windows ou une machine Windows locale qui peut se connecter à votre instance RDS SQL Server.

          <Callout variant="important">
            Pour le monitoring RDS, installez le Collector NRDOT sur une instance EC2 dans le même VPC que votre instance RDS pour une connectivité réseau optimale et une latence plus faible.
          </Callout>

          1. Téléchargez et installez le NRDOT Collector à l&apos;aide de PowerShell :

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

          2. Assurez-vous que votre groupe de sécurité autorise les connexions entrantes de l&apos;hôte du Collector NRDOT vers votre instance RDS sur le port 1433 (ou votre port personnalisé).
        </Step>

        <Step>
          ### Configurer l&apos;utilisateur de la base de données [#configure-user-sql]

          Connectez-vous à votre instance RDS SQL Server à l&apos;aide de SQL Server Management Studio (SSMS) ou sqlcmd avec vos identifiants d&apos;administrateur.

          Exécutez le script suivant pour créer l&apos;utilisateur de monitoring `newrelic` avec les autorisations appropriées :

          ```sql
          USE [master];
          GO
          CREATE LOGIN [newrelic] WITH PASSWORD = '<PASSWORD>';
          GO

          GRANT VIEW SERVER STATE TO [newrelic];
          GRANT VIEW ANY DEFINITION TO [newrelic];
          GRANT VIEW ANY DATABASE TO [newrelic];
          GO

          DECLARE @name SYSNAME;
          DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
          SELECT [name]
          FROM [master].[sys].[databases]
          WHERE [name] NOT IN ('master', 'msdb', 'model', 'rdsadmin', 'distribution')
          AND [state] = 0;
          OPEN db_cursor;
          FETCH NEXT FROM db_cursor INTO @name;
          WHILE @@FETCH_STATUS = 0
          BEGIN
            BEGIN TRY
              EXEC('USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''newrelic'')
              BEGIN
                CREATE USER [newrelic] FOR LOGIN [newrelic];
              END;
              GRANT VIEW DATABASE STATE TO [newrelic];');
            END TRY
            BEGIN CATCH
              PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
            END CATCH
            FETCH NEXT FROM db_cursor INTO @name;
          END
          CLOSE db_cursor;
          DEALLOCATE db_cursor;
          GO
          ```

          <Callout variant="tip">
            **Pour RDS :** utilisez votre nom d&apos;utilisateur principal RDS (ou un utilisateur avec le rôle `rds_superuser`) pour exécuter ces commandes. Le script exclut automatiquement la base de données `rdsadmin` qui est réservée par AWS.
          </Callout>
        </Step>

          ### Paramètres de configuration

          Le tableau suivant décrit les principaux paramètres de configuration du récepteur nrsqlserver :

          | paramètres             | Description                                                                                                                                                                                             |
          | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
          | `<YOUR_USERNAME>`      | Saisissez votre nom d&apos;utilisateur de base de données pour l&apos;authentification                                                                                                                  |
          | `<YOUR_PASSWORD>`      | Saisissez le mot de passe de votre base de données pour l&apos;authentification                                                                                                                         |
          | `<YOUR_SERVER>`        | Saisissez le nom d&apos;hôte ou l&apos;adresse IP de SQL Server                                                                                                                                         |
          | `<YOUR_PORT>`          | Saisissez votre numéro de port SQL Server. La valeur par défaut est 1433.                                                                                                                               |
          | `<YOUR_OTLP_ENDPOINT>` | Saisissez le point de terminaison OTLP New Relic. Pour plus d&apos;informations, consultez la [documentation sur les endpoints OTLP New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/). |
          | `<YOUR_LICENSE_KEY>`   | Entrez votre clé de licence New Relic.                                                                                                                                                                  |
          | `collection_interval`  | Intervalle en secondes pour collecter les métriques de base de données. Par défaut : 15s                                                                                                                |

          <Callout variant="tip">
            **Remarque :** L&apos;option `max_concurrent_queries` contrôle le nombre de requêtes de métriques que le récepteur exécute en parallèle sur SQL Server lors de chaque cycle de collecte.

            ```yaml
            receivers:
              nrsqlserver:
                max_concurrent_queries: 4  # default is 4 if omitted or set to 0
            ```
          </Callout>

          <Callout variant="tip">
            **Remarque :** Si vous activez toutes les métriques disponibles, augmentez la valeur `limit_mib` (par ex., 1000) pour éviter que le limiteur de mémoire ne supprime des données.
          </Callout>
        </Step>

        <Step>
          ### Mettre à jour le registre des services Windows [#update-registry-sql]

          Mettez à jour le registre des services Windows pour utiliser le nouveau fichier mssql-config.yaml.

          **Méthode 1 : PowerShell**

          Exécutez la commande suivante dans PowerShell en tant qu&apos;administrateur :

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **Méthode 2 : invite de commande (alternative)**

          Si PowerShell n&apos;est pas disponible, vous pouvez utiliser l&apos;invite de commande à la place :

          1. Ouvrez votre menu Démarrer, tapez `cmd`
          2. Faites un clic droit sur l&apos;invite de commande et choisissez **Run as administrator**
          3. Copiez et collez cette commande exacte dans la fenêtre CMD noire et appuyez sur Entrée :

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
        </Step>

        <Step>
          ### Activez votre Microsoft SQL Server [#enable-sql-server-domain]

          Connectez-vous à votre instance RDS SQL Server à l&apos;aide de SQL Server Management Studio (SSMS) ou sqlcmd avec vos identifiants d&apos;administrateur.

          Exécutez le script suivant pour créer une connexion de domaine Windows et accorder les autorisations nécessaires :

          ```sql
          USE master;
          CREATE LOGIN [DomainName\DomainUser] FROM WINDOWS;
          GRANT VIEW ANY DATABASE   TO [DomainName\DomainUser];
          GRANT VIEW SERVER STATE TO [DomainName\DomainUser];
          GRANT VIEW ANY DEFINITION TO [DomainName\DomainUser];
          GO
          ```

          <Callout variant="important">
            Remplacez `DomainName\DomainUser` par votre domaine et nom d&apos;utilisateur réels (par ex. `CORP\SQLMonitor`).
          </Callout>

        <Step>
          ### Accorder les privilèges d&apos;accès READ [#grant-permissions-domain]

          Utilisez les instructions suivantes pour accorder des privilèges d&apos;accès READ à l&apos;utilisateur du domaine sur toutes les bases de données :

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
                AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''DomainName\DomainUser'')
                BEGIN
                  CREATE USER [DomainName\DomainUser] FOR LOGIN [DomainName\DomainUser];
                END;
                GRANT VIEW DATABASE STATE TO [DomainName\DomainUser];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="tip">
            **Pour RDS :** Le script exclut automatiquement la base de données `rdsadmin` qui est réservée par AWS. Assurez-vous de remplacer `DomainName\DomainUser` par vos véritables identifiants de domaine.
          </Callout>

        <Step>
          ### Configurer le Collector NRDOT [#setup-domain]

          Installez le Collector NRDOT sur une instance EC2 Windows jointe au domaine ou une machine Windows locale qui fait partie du même domaine et peut se connecter à votre instance RDS SQL Server.

          <Callout variant="important">
            Pour le monitoring RDS, installez le Collector NRDOT sur une instance EC2 dans le même VPC que votre instance RDS pour une connectivité réseau optimale et une latence plus faible.
          </Callout>

          1. Téléchargez et installez le NRDOT Collector à l&apos;aide de PowerShell :

             ```powershell
             [Net.ServicePointManager]::SecurityProtocol = 'tls12, tls'; $NRDOT_VERSION = (Invoke-RestMethod -Uri "https://api.github.com/repos/newrelic/nrdot-collector-releases/releases/latest").tag_name; $WebClient = New-Object System.Net.WebClient; $WebClient.Headers.Add("User-Agent", "Mozilla/5.0"); $WebClient.DownloadFile("https://github.com/newrelic/nrdot-collector-releases/releases/download/$NRDOT_VERSION/nrdot-collector_${NRDOT_VERSION}_windows_x64.msi", "$env:TEMP\nrdot-collector.msi"); Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\nrdot-collector.msi`" /qn /norestart /L*V `"$env:TEMP\nrdot_install.log`"" -Wait; Get-Service nrdot-collector -ErrorAction SilentlyContinue
             ```

             ```powershell
             # Configure service to use domain account
             # The space after obj= and password= is required syntax for sc.exe
             sc.exe config "nrdot-collector" obj= "DomainName\DomainUser" password= "<password>"

             # Restart the service
             Restart-Service nrdot-collector
             ```

             ```powershell
             Get-WmiObject Win32_Service -Filter "Name='nrdot-collector'" | Select Name, State, StartName
             # Expected: StartName = DomainName\DomainUser
             ```

             <Callout variant="important">
               Remplacez `DomainName\DomainUser` par votre domaine et nom d&apos;utilisateur réels (par ex., `CORP\SQLMonitor`), et `<password>` par le mot de passe de l&apos;utilisateur du domaine.
             </Callout>

          3. Assurez-vous que votre groupe de sécurité autorise les connexions entrantes de l&apos;hôte du Collector NRDOT vers votre instance RDS sur le port 1433 (ou votre port personnalisé).

        <Step>
          ### Activez votre Microsoft SQL Server [#enable-sql-server-gmsa]

          Connectez-vous à votre instance RDS SQL Server et exécutez les commandes SQL suivantes pour créer une connexion pour le compte gMSA et accorder les autorisations nécessaires :

          ```sql
            USE master;
            CREATE LOGIN [DomainName\gMSA] FROM WINDOWS;
            GRANT VIEW ANY DATABASE TO [DomainName\gMSA];
            GRANT VIEW SERVER STATE TO [DomainName\gMSA];
            GRANT VIEW ANY DEFINITION TO [DomainName\gMSA];
          ```

          <Callout variant="important">
            Remplacez `DomainName\gMSA` par votre nom de domaine réel et le nom de votre compte gMSA (par ex., `CORP\SQL-gMSA$`). Notez que les comptes gMSA se terminent généralement par un symbole `$`.
          </Callout>

        <Step>
          ### Accorder les privilèges d&apos;accès READ [#grant-read-access-rds-gmsa]

          Utilisez les instructions suivantes pour accorder les privilèges d&apos;accès READ au compte gMSA sur toutes les bases de données :

          ```sql
          -- Grant access to all databases (required for query monitoring and tempdb metrics)
            DECLARE @name SYSNAME;
            DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
              SELECT [name] FROM [master].[sys].[databases]
              WHERE [name] NOT IN ('master','msdb','model','rdsadmin','distribution')
              AND [state] = 0;
            OPEN db_cursor;
            FETCH NEXT FROM db_cursor INTO @name;
            WHILE @@FETCH_STATUS = 0
            BEGIN
              BEGIN TRY
                EXEC('USE [' + @name + '];
                IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''<DOMAIN>\<gMSA_NAME>$'')
                BEGIN
                  CREATE USER [<DOMAIN>\<gMSA_NAME>$] FOR LOGIN [<DOMAIN>\<gMSA_NAME>$];
                END;
                GRANT VIEW DATABASE STATE TO [<DOMAIN>\<gMSA_NAME>$];');
              END TRY
              BEGIN CATCH
                PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
              END CATCH
              FETCH NEXT FROM db_cursor INTO @name;
            END
            CLOSE db_cursor;
            DEALLOCATE db_cursor;
            GO
          ```

          <Callout variant="important">
            Remplacez `DomainName\gMSA` par votre nom de domaine réel et le nom de votre compte gMSA (par ex., `CORP\SQL-gMSA$`). Ce script crée un utilisateur de base de données pour le compte gMSA dans toutes les bases de données utilisateur, à l&apos;exclusion des bases de données système et des bases de données spécifiques à RDS.
          </Callout>
        </Step>

          ### Paramètres de configuration

          Le tableau suivant décrit les principaux paramètres de configuration du récepteur nrsqlserver :

          | paramètres             | Description                                                                                                                                                                                             |
          | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
          | `<YOUR_RDS_ENDPOINT>`  | Saisissez votre point de terminaison RDS SQL Server                                                                                                                                                     |
          | `<YOUR_PORT>`          | Saisissez votre numéro de port SQL Server. La valeur par défaut est 1433.                                                                                                                               |
          | `<YOUR_OTLP_ENDPOINT>` | Saisissez le point de terminaison OTLP New Relic. Pour plus d&apos;informations, consultez la [documentation sur les endpoints OTLP New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp/). |
          | `<YOUR_LICENSE_KEY>`   | Entrez votre clé de licence New Relic.                                                                                                                                                                  |
          | `collection_interval`  | Intervalle en secondes pour collecter les métriques de base de données. Par défaut : 15s                                                                                                                |

          <Callout variant="tip">
            L&apos;option `max_concurrent_queries` contrôle combien de requêtes de métriques le Récepteur exécute en parallèle sur SQL Server pendant chaque cycle de récupération.

            ```yaml
            receivers:
              nrsqlserver:
                max_concurrent_queries: 4  # default is 4 if omitted or set to 0
            ```
          </Callout>

          ### Mettre à jour le registre des services Windows [#update-registry-gmsa]

          Mettez à jour le registre des services Windows pour utiliser le nouveau fichier mssql-config.yaml.

          **Méthode 1 : PowerShell**

          Exécutez la commande suivante dans PowerShell en tant qu&apos;administrateur :

          ```powershell
          Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\nrdot-collector" -Name "ImagePath" -Value '"C:\Program Files\nrdot-collector\nrdot-collector.exe" --config "C:\Program Files\nrdot-collector\mssql-config.yaml"'
          ```

          **Méthode 2 : invite de commande (alternative)**

          Si PowerShell n&apos;est pas disponible, vous pouvez utiliser l&apos;invite de commande à la place :

          1. Ouvrez votre menu Démarrer, tapez `cmd`
          2. Faites un clic droit sur l&apos;invite de commande et choisissez **Run as administrator**
          3. Copiez et collez cette commande exacte dans la fenêtre CMD noire et appuyez sur Entrée :

          ```cmd
          sc config "nrdot-collector" binPath= "\"C:\Program Files\nrdot-collector\nrdot-collector.exe\" --config \"C:\Program Files\nrdot-collector\mssql-config.yaml\""
          ```
        </Step>
