Avant de commencer à monitorer votre Microsoft SQL Server avec NRDOT, assurez-vous que votre environnement répond à ces exigences.

## Prérequis

Avant de commencer, assurez-vous d’avoir les éléments suivants :

* [Clé de licence](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)New Relic valide

* Versions de base de données prises en charge : SQL Server 2017 ou version ultérieure

* Systèmes d&apos;exploitation et plateformes pris en charge : Windows, Linux et les instances gérées (RDS)

* Configuration de la base de données SQL Server :

  * Accès administrateur à votre instance SQL Server (rôle `sysadmin` ou équivalent)
  * Microsoft SQL Server qui prend en charge les instances gérées sur site
  * Connectivité réseau entre le collecteur NRDOT et votre serveur SQL
  * SQL Server Management Studio (SSMS) ou l&apos;utilitaire `sqlcmd`
  * Nom d&apos;hôte ou adresse IP du serveur
  * Numéro de port (généralement 1433) ou port personnalisé
  * Authentification de domaine Windows ou SQL Server
  * Préférence SSL

* Configuration requise :

  * Système Windows ou Linux pour exécuter le collecteur NRDOT
  * Connectivité réseau vers le point de terminaison OTLP de New Relic

* Cette intégration est disponible dans le cadre du programme de préversion publique de New Relic. Vérifiez auprès de votre responsable d’organisation pour vous inscrire depuis la page [Préversions et essais](https://one.newrelic.com/admin-portal/promotion-management/home).

## Prochaines étapes

Une fois que vous avez vérifié que votre environnement respecte ces prérequis :

1. Choisissez votre méthode d&apos;installation :

* [Configuration auto-hébergée Windows](/docs/opentelemetry/database/mssql/windows-hosted)
* [Configuration auto-hébergée Linux](/docs/opentelemetry/database/mssql/linux-hosted)
* [Configuration RDS Windows](/docs/opentelemetry/database/mssql/windows-rds)
* [Configuration de RDS Linux](/docs/opentelemetry/database/mssql/linux-rds)

2. Consultez les métriques disponibles : [comprendre quelles données seront collectées](/docs/opentelemetry/database/mssql/metrics-reference)
