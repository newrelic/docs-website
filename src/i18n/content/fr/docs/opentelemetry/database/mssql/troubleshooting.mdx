Résolvez les problèmes courants que vous pouvez rencontrer lors du monitoring de votre Microsoft SQL Server avec le collecteur NRDOT.

## Problèmes courants

<CollapserGroup>
  <Collapser id="network-connectivity" title="Problèmes de connectivité réseau">
    Si vous rencontrez des problèmes de connectivité réseau avec le collecteur NRDOT, exécutez les commandes suivantes pour tester la connectivité vers votre instance SQL Server sur le port 1433 :

    * Pour Windows, exécutez :

      ```shell
      Test-NetConnection -ComputerName <YOUR_IP_ADDRESS> -Port <YOUR_PORT_NUMBER>
      ```

    * Pour Linux, exécutez :

      ```bash
      nc -zv <YOUR_IP_ADDRESS> <YOUR_PORT_NUMBER>
      ```
  </Collapser>

  <Collapser id="ssl" title="Problèmes de connexion SSL">
    Pour vous connecter à Microsoft SQL Server en utilisant SSL sans validation du certificat, assurez-vous que le paramètre `enable_ssl` est défini sur `true` et que le paramètre `trust_server_certificate` est défini sur `true` dans la configuration de votre NRDOT Collector.

    ```yaml
    enable_ssl: true                                                                                                                            
    trust_server_certificate: true
    ```
  </Collapser>
</CollapserGroup>
