OpenTelemetry offre une solution économique polyvalente pour l&apos;instrumentation et la télémétrie, qui peut être intégrée de manière transparente à New Relic via le [protocole OpenTelemetry (OTLP)](https://opentelemetry.io/docs/specs/otlp/). Ce guide fournit un aperçu de la manière dont New Relic traite et ingère les données OTLP pour vous aider à monitorer et observer efficacement votre système.

## Démarrer [#get-started]

Pour en savoir plus sur l&apos;intégration d&apos;OpenTelemetry avec New Relic, consultez [Premiers pas avec OpenTelemetry et New Relic](/docs/opentelemetry/get-started/opentelemetry-get-started-intro/). Cette page couvre les modèles d&apos;intégration courants et les bonnes pratiques.

## Configurer OTLP pour New Relic [#configure]

Pour connaître les exigences détaillées de configuration OTLP et obtenir des conseils de dépannage, visitez les pages suivantes :

* [Le point de terminaison New Relic OTLP](/docs/opentelemetry/best-practices/opentelemetry-otlp/) vous guide à travers les étapes de configuration essentielles pour intégrer les données OpenTelemetry dans New Relic.
* [Le dépannage OTLP de New Relic](/docs/opentelemetry/best-practices/opentelemetry-otlp-troubleshooting/) vous aide à résoudre les problèmes courants liés à l&apos;ingestion de données, tels que la vérification des logs, de la clé API et les problèmes de connexion.

### Types de signaux dans OpenTelemetry [#signals]

L&apos;OTLP est divisé en signaux trace, métriques et log , qui incluent tous le concept de ressources. Les pages suivantes fournissent des détails sur la manière dont chaque métrique est traitée et ingérée dans New Relic :

* [Trace OpenTelemetry dans New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-traces/)
* [Métriques OpenTelemetry dans New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-metrics/)
* [Logs OpenTelemetry dans New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-logs/)
* [Ressources OpenTelemetry dans New Relic](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/)

## Gérer le volume de données [#manage-volume]

Pour gérer le volume de données OpenTelemetry et contrôler les coûts d&apos;ingestion, consultez [Gérer le volume d&apos;ingestion de données OpenTelemetry](/docs/opentelemetry/best-practices/opentelemetry-manage-data-ingest-volume/). Cette page couvre les stratégies d’optimisation du volume de données afin de garantir une utilisation efficace des ressources.

## Utilisez la plateforme New Relic [#platform]

Une fois les données OpenTelemetry ingérées dans la [plateforme New Relic](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/), vous pouvez profiter de certains de ces outils d&apos;observabilité :

* [Alertes](/docs/alerts/overview/): Soyez informé lorsqu&apos;un signal dépasse un seuil.
* [Requête personnalisée](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/): écrivez une requête NRQL personnalisée pour l&apos;analyse et l&apos;exploration de données ad hoc.
* [Dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/): créez des visualisations de données personnalisées reflétant les exigences uniques de vos applications.
