[OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) est un outil indépendant du fournisseur pour la réception, le traitement et l&amp;apos;exportation de données télémétriques. Bien que les Collector exigences et configuration varient, il est livré avec une variété de récepteurs et de processeurs qui le rendent populaire pour monitoring d&amp;apos;infrastructure. Il est également courant d&amp;apos;utiliser le Collector pour le traitement des données, mais cette documentation se concentre sur les cas d&amp;apos;utilisation de monitoring d&amp;apos;infrastructure . Consultez [OpenTelemetry Collector pour le traitement des données](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro) pour plus d&amp;apos;informations.

## Corrélation de l&apos;infrastructure avec OpenTelemetry APM [#infrastructure-correlation]

L&apos;exemple suivant illustre l&apos;utilisation du Collector pour corréler [APM OpenTelemetry](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-intro) avec les données infrastructure :

* [Exemple de référentiel Hôtes (repository)](https://github.com/newrelic/newrelic-opentelemetry-examples/tree/main/other-examples/collector/host-monitoring)

Le modèle général consiste à configurer le Collector avec un Processeur qui détecte et enrichit la télémétrie APM avec un contexte d&apos;environnement supplémentaire sous la forme d&apos;un attribut de ressource, avant d&apos;exporter les données vers New Relic via OTLP. New Relic détecte ces données de corrélation et construit des relations entre l&apos;APM et l&apos;entité infrastructure via [les ressources](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources).
