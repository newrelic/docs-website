[OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) est un outil indépendant du fournisseur pour la réception, le traitement et l&apos;exportation de données télémétriques. Bien que les exigences et configuration du Collector varient, il est livré avec une variété de récepteurs et de processeurs qui le rendent populaire pour le monitoring de Kubernetes. Obtenir toute la télémétrie requise nécessite de récupérer diverses API Kubernetes à l&apos;aide de plusieurs récepteurs avec des exigences d&apos;autorisation différentes. Par conséquent, pour vous simplifier la tâche, nous avons créé un graphique Helm qui utilise notre distribution personnalisée du collecteur [nrdot-collecteur-k8s](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s) sous le capot. Pour plus de détails sur la façon de démarrer, veuillez vous référer à [OpenTelemetry pour Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/).
