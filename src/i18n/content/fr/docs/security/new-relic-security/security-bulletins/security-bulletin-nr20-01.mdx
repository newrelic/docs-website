## Résumé

Une mise à jour de sécurité pour l&apos;agent .NET corrige un problème où une requête SQL complète peut être envoyée au log de l&apos;agent.

<DNT>**Release date:**</DNT> 16 janvier 2020

<DNT>**Vulnerability identifier:**</DNT> NR20-01

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

Afin de générer des plans d&apos;exécution, une copie de la requête SQL est créée et la requête est réémise avec une demande de plan d&apos;exécution. Si le plan d&apos;explication échoue, l&apos;agent peut log l&apos;instruction SQL complète qui pourrait inclure les valeurs des paramètres.

### Facteurs atténuants [#factors]

L&apos;agent log ces informations uniquement s&apos;il est défini sur les niveaux du logging `DEBUG` ou `FINEST`.

## Solutions de contournement

* Assurez-vous que le niveau du logging n&apos;est pas défini sur `DEBUG` ou `FINEST`.
* Désactiver la capture des plans explicatifs.
* Assurez-vous que l&apos;emplacement du fichier de log est sécurisé.
* [Mise à jour vers le dernier agent .NET New Relic.](/docs/agents/net-agent/installation/update-net-agent)
