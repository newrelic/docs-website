## Résumé

Une mise à jour de sécurité pour l&apos;agent Node.js de New Relic élimine une vulnérabilité qui permet la divulgation potentielle d&apos;informations sensibles via les paramètres de requête de l&apos;en-tête du référent.

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

L&apos;agent Node.js de New Relic collecte les en-têtes de requête lors d&apos;une trace d&apos;erreur pour aider à déterminer la cause première des problèmes. L&apos;en-tête de référence est l&apos;URI qui identifie l&apos;adresse de la page Web liée à la ressource demandée. Il est possible que l&apos;URI du référent contienne des informations sensibles dans les paramètres de requête. New Relic a constaté que les paramètres de requête ne sont pas correctement supprimés lors de la trace d&apos;erreur. Cette mise à jour corrige ce problème en supprimant les paramètres de requête du référent dans l&apos;en-tête de la requête avant d&apos;envoyer ces données à New Relic.

### Facteurs atténuants [#factors]

* Les bonnes pratiques recommandent de ne pas utiliser de paramètres de requête pour transmettre des données sensibles.

## Solutions de contournement

New Relic n&apos;a identifié aucune solution de contournement pour ces vulnérabilités.
