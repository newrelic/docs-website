## Résumé

Les paramètres de demande sont ajoutés en tant qu&apos;attribut de segment, même lorsque l&apos;agent est configuré avec le mode haute sécurité ou des politiques de sécurité configurables. Cela peut entraîner la collecte de données inattendues.

<DNT>**Release date:**</DNT> 1er avril 2019

<DNT>**Vulnerability identifier:**</DNT> NR19-02

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

De par leur conception, les paramètres de demande sont ajoutés en tant qu&apos;attribut de segment. Les paramètres de demande doivent être ignorés lorsque l&apos;agent est configuré avec le mode haute sécurité ou la politique de sécurité configurable `attributes_include` .

### Facteurs atténuants [#factors]

Affecte uniquement les agents configurés avec le mode Haute sécurité ou la politique de sécurité configurable `attributes_include` .

## Solutions de contournement

* Définissez `attributed.enabled` sur `false` pour empêcher la collecte de tous les attributs.
* [Mise à jour vers le dernier agent New Relic Node.js.](/docs/agents/nodejs-agent/installation-configuration/upgrade-nodejs-agent)

## Signaler les vulnérabilités de sécurité à New Relic [#report]

New Relic s&apos;engage à assurer la sécurité de ses clients et de leurs données. Si vous pensez avoir découvert une faille de sécurité dans l&apos;un de nos produits ou sites Web, nous vous invitons à la signaler au programme de divulgation coordonnée de New Relic. Pour plus d&apos;informations, consultez [Signalement des vulnérabilités de sécurité](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities).
