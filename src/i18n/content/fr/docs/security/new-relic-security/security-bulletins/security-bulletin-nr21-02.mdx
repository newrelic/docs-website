## Résumé

Une mise à jour de sécurité de l&apos;agent Java a reconfiguré l&apos;analyseur YAML pour inclure un SafeConstructor, ce qui supprime la possibilité d&apos;exécuter du code limité contrôlé par l&apos;utilisateur.

<DNT>**Release date:**</DNT> 26 avril 2021

<DNT>**Vulnerability identifier:**</DNT> NR21-02

<DNT>**Priority:**</DNT> Faible

## informations sur les vulnérabilités [#vuln-info]

Une notation spécifiée, lorsqu&apos;elle est analysée via un appel `Yaml.load()` non sécurisé, créera un nouvel objet Java et appellera son constructeur, conduisant potentiellement à l&amp;apos;exécution de code. Un attaquant devrait avoir accès à l&amp;apos;hôte de l&amp;apos;agent pour modifier le fichier `newrelic.yml` afin d&amp;apos;inclure une charge utile conçue qui exécuterait du code arbitraire une fois l&amp;apos;agent démarré.

### Facteurs atténuants [#factors]

Cette vulnérabilité nécessite qu&apos;un attaquant ait déjà accès à l&apos;hôte afin de modifier le fichier de configuration `newrelic.yml` sur la machine d&amp;apos;une victime, ce qui en soi est un facteur atténuant. Cependant, il existe des mesures supplémentaires que vous pouvez prendre pour corriger complètement ce problème ou renforcer votre système contre celui-ci :

* Mettez à jour votre agent Java pour corriger ces vulnérabilités
* Révoquer les privilèges d&apos;écriture sur votre fichier `newrelic.yml`

## Solutions de contournement

* [Mise à jour vers le dernier agent Java New Relic.](/docs/agents/java-agent/installation/update-java-agent/)
