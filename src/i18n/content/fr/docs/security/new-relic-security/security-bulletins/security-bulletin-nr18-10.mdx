## Résumé

Une mise à jour de sécurité pour l’agent d’infrastructure Windows corrige un problème selon lequel un chemin de fichier codé en dur peut permettre à un utilisateur malveillant de remplacer la configuration de l’agent.

<DNT>**Release date:**</DNT> 18 juin 2018

<DNT>**Vulnerability identifier:**</DNT> NR18-10

<DNT>**Priority:**</DNT> Haut

## informations sur les vulnérabilités [#vuln-info]

L&apos;agent d&apos;infrastructure Windows New Relic recherche des fichiers configuration dans des emplacements spécifiques. Un chemin de fichier réservé à Unix a été inclus dans l&apos;agent Windows, ce qui pourrait permettre à un utilisateur malveillant de remplacer le fichier de configuration. Avec cette mise à jour de sécurité, l’agent d’infrastructure Windows ne recherchera plus dans un emplacement spécifique à Unix.

### Facteurs atténuants [#factors]

* Cette vulnérabilité n&apos;existe que sur le système Windows .
* Cette vulnérabilité nécessite un accès existant au système.
