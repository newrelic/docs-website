## informations sur les vulnérabilités [#vuln-info]

L&apos;agent Python New Relic doit obscurcir les paramètres de requête SQL. Cela corrige un problème dans le tracing des transactions SQLite. Si les paramètres de requête étaient transmis avec des chaînes entre guillemets doubles, l&apos;agent ne masquait pas correctement ces paramètres.

### Facteurs atténuants [#factors]

* En règle générale, toutes les chaînes de paramètres de requête SQL doivent être des littéraux de chaîne et transmises entre guillemets simples. Cependant, l&apos;API Python SQLite permet aux développeurs d&apos;utiliser des chaînes entre guillemets doubles dans certains cas.

## Solutions de contournement

Si vous ne parvenez pas à mettre à niveau l’agent, vous pourrez peut-être effectuer les actions suivantes pour vous assurer de ne pas envoyer de paramètres de requête SQLite.

* Utilisez des littéraux de chaîne entre guillemets simples avec SQLite.
* Désactivez la collecte SQL en configurant [transaction\_tracer.record\_sql](/docs/agents/python-agent/configuration/python-agent-configuration#txn-tracer-settings) sur `off`.
