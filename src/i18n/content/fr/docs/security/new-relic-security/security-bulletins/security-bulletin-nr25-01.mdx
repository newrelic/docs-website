<DNT>**Vulnerability Identifier:**</DNT> NR25-01

<DNT>**Priority:**</DNT> Haut

## Résumé

Par défaut, New Relic n&apos;inclut ni n&apos;active les plugins spécifiques affectés par les vulnérabilités de sécurité identifiées dans certaines versions de Fluent Bit. Les plugins spécifiques sont :

* [Plugin d&apos;entrée OpenTelemetry](https://docs.fluentbit.io/manual/pipeline/inputs/opentelemetry) - Affecté par [CVE-2024-50608](https://nvd.nist.gov/vuln/detail/CVE-2024-50608)
* [Plugin d&apos;entrée Prometheus Remote Write](https://docs.fluentbit.io/manual/pipeline/inputs/prometheus-remote-write) - Affecté par [CVE-2024-50609](https://nvd.nist.gov/vuln/detail/CVE-2024-50609)

Cependant, pour soutenir les clients qui ont activé ces plugins optionnels, nous recommandons aux clients de mettre à niveau vers les dernières versions disponibles de ces package:

* Agent d&apos;infrastructure - Windows
* Agent d&apos;infrastructure - Linux
* Plugin Kubernetes
* Image Docker du nouveau plug-in de sortie Fluent Bit de Relic

## Action requise

New Relic conseille vivement à ses clients qui utilisent l&apos; de transfert de logsusmentionnée instrumentation de prendre les mesures immédiates suivantes. Si vous ne parvenez pas à effectuer la mise à niveau vers Fluent Bit v3.2.7, nous vous recommandons de désactiver les plugins concernés spécifiés ci-dessus.

## Questions fréquemment posées

1. **Comment puis-je savoir si j’utilise des plugins vulnérables ?**

   La configuration par défaut de Fluent Bit de New Relic n&apos;inclut pas les plugins vulnérables par défaut. Si vous avez modifié la configuration de votre bit Fluent après l&apos;installation et inclus le [plug-in d&apos;entrée OpenTelemetry](https://docs.fluentbit.io/manual/pipeline/inputs/opentelemetry) et/ou [l&apos;entrée Prometheus Remote Write](https://docs.fluentbit.io/manual/pipeline/inputs/prometheus-remote-write), toute version de Fluent Bit installée dans votre environnement est vulnérable. Suivez les instructions pour mettre à niveau immédiatement toute de votre transfert log instrumentation de .

2. **J&apos;utilise l&apos;agent d&apos;infrastructure mais j&apos;ai désactivé le transfert de log . Suis-je impacté ?**

   Si vous avez déjà utilisé de transfert de logde New Relic instrumentation (listée ci-dessus) et utilisé les plugins concernés, vous pourriez toujours être impacté. New Relic vous recommande de mettre à niveau vos agents ou au minimum de désactiver les plugins concernés.

   De plus, New Relic recommande à tous les clients d’identifier toute autre utilisation de Fluent Bit dans leurs environnements et de les mettre à jour vers au moins la version 3.2.7.

3. **Une fois que j&apos;ai mis à jour vers les dernières versions des services de transfert de logrépertoriés, dois-je faire autre chose ?**

Oui, mais uniquement si vous exécutez des agents d’infrastructure sur des hôtes Linux. Si vous exécutez l&apos;agent Linux Infrastructure, vous devrez également [mettre à jour Fluent Bit](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/#install-fb-version) dans votre environnement vers une version 3.2.7 ou ultérieure.

4. **Comment puis-je savoir quelle version de Fluent Bit j&apos;utilise ?**

Si vous avez déjà utilisé de transfert de logde New Relic instrumentation (listée ci-dessus) et utilisé les plugins concernés, vous pourriez toujours être impacté. New Relic vous recommande de mettre à niveau vos agents ou au minimum de désactiver les plugins concernés.

De plus, New Relic recommande à tous les clients d’identifier toute autre utilisation de Fluent Bit dans leurs environnements et de les mettre à jour vers au moins la version 3.2.7.
