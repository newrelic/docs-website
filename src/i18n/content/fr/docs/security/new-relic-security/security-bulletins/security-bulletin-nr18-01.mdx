## informations sur les vulnérabilités [#vuln-info]

Un audit interne du [mode haute sécurité de Python](/docs/agents/python-agent/configuration/python-agent-configuration#high_security) a révélé que l&amp;apos;agent pouvait envoyer des données supplémentaires à New Relic en personnalisant les paramètres de message avec les API de tracing. Des paramètres personnalisés peuvent être envoyés à New Relic si un développeur transmet des paramètres supplémentaires à l&amp;apos;aide de l&amp;apos;API trace de fonction ou de l&amp;apos;API trace de message. Ceux-ci doivent être désactivés lorsque le mode haute sécurité est activé.

### Facteurs atténuants [#factors]

* Cela affecte uniquement les clients qui ont activé le mode haute sécurité avec l'agent Python et les développeurs ajoutent des informations supplémentaires à l'API de tracing.

## Solutions de contournement

Si vous ne parvenez pas à mettre à niveau l'agent, vous pourrez peut-être effectuer les actions suivantes pour vous assurer de ne pas envoyer de données supplémentaires en mode haute sécurité.

* Supprimez tous les paramètres personnalisés de l’ [API de trace de fonction](/docs/agents/python-agent/python-agent-api/function_trace) ou de l’ [API de trace de message](/docs/agents/python-agent/python-agent-api/message-trace).

## Signaler les vulnérabilités de sécurité à New Relic [#report]

New Relic s'engage à assurer la sécurité de ses clients et de leurs données. Si vous pensez avoir découvert une faille de sécurité dans l'un de nos produits ou sites Web, nous vous invitons à la signaler au programme de divulgation coordonnée de New Relic. Pour plus d'informations, consultez [Signalement des vulnérabilités de sécurité](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities).
