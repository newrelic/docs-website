## Résumé

Une mise à jour de sécurité pour l&apos;agent Ruby de New Relic corrige une vulnérabilité où l&apos;agent pouvait capturer involontairement une requête d&apos;agrégation brute avec MongoDB. New Relic recommande de mettre à jour vers la dernière version corrigée.

<DNT>**Release date:**</DNT> 9 février 2017

<DNT>**Vulnerability identifier:**</DNT> NR17-03

<DNT>**Priority:**</DNT> Faible

## informations sur les vulnérabilités [#vuln-info]

La version 3.13.1 de l&apos;agent Ruby de New Relic a ajouté de la visibilité à la requête MongoDB avec la version 2.1 et supérieure du pilote MongoDB pour Ruby. Le paramètre par défaut de l&apos;agent pour <DNT>**mongo.obfuscate\_queries**</DNT> est vrai. Cela devrait amener l&apos;agent à obscurcir les valeurs dans la requête Mongo avant d&apos;envoyer ces informations à New Relic. Cependant, lors de l&apos;utilisation du pipeline d&apos;agrégation avec cette version du pilote, les requêtes d&apos;agrégation n&apos;étaient pas correctement obscurcies.

### Facteurs atténuants [#factors]

* Seuls les clients qui utilisent la version 2.1 et supérieure du pilote Ruby pour MongoDB sont concernés
* Les requêtes agrégées ne contiennent généralement pas d&apos;informations sensibles

## Solutions de contournement

les utilisateurs concernés et incapables de mettre à niveau peuvent choisir de configurer l&apos;agent Ruby pour ne pas capturer les requêtes mongoDB. l&apos;utilisateur peut définir <DNT>**mongo.capture\_queries**</DNT> sur false pour empêcher l&apos;agent d&apos;envoyer des informations sur la requête.
