## Résumé

Une mise à jour de sécurité de l&apos;agent du navigateur détectera `file://` schémas d&apos;URI et arrêtera toute autre exécution et collecte de données si elles sont détectées.

<DNT>**Vulnerability identifier:**</DNT> NR21-01

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

Les navigateurs peuvent restituer des fichiers locaux sur une machine hôte en utilisant le schéma d&apos;URI `file://` décrit dans la RFC 8089. Pendant le cycle de collecte de l&apos;agent, cet URI `file://` sera enregistré comme point de données pageURL. Cela peut entraîner la collecte de données potentiellement sensibles incluses dans le chemin du fichier local, telles que le chemin du répertoire de la page Web enregistrée et tout nom ou information sur l&apos;entreprise dans le chemin du répertoire. Vous trouverez plus d&apos;informations sur l&apos;URI `file://` dans la [RFC 8089](https://tools.ietf.org/html/rfc8089)

### Facteurs atténuants [#factors]

Une personne doit à la fois télécharger une page Web avec l’agent de navigateur configuré et ouvrir le fichier dans un navigateur. Les fichiers HTML chargés sans le schéma d&apos;URI `file://` ne sont pas affectés.
