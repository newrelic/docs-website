## Résumé

Une mise à jour de sécurité pour l&apos;agent Java corrige un problème où l&apos;agent peut ne pas masquer correctement toutes les informations de requête SQL lorsque `record_sql` est défini sur `off`.

<DNT>**Release date:**</DNT> 2 mai 2018

<DNT>**Vulnerability identifier:**</DNT> NR18-09

<DNT>**Priority:**</DNT> Faible

## informations sur les vulnérabilités [#vuln-info]

L&apos;agent Java New Relic exécute des plans explicatifs pour [le tracing lent des transactions](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) et les requêtes SQL lentes. Lorsqu&amp;apos;un client a défini `record_sql` sur `off`, exécute PostgreSQL et exécute un plan d&amp;apos;explication, l&amp;apos;agent peut ne pas masquer correctement toutes les informations de la requête.

## Solutions de contournement

Désactivez `explain` plans avec `transaction_tracer` dans la [configuration de l&apos;agent Java](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#h2-Transaction-tracer).

## Signaler les vulnérabilités de sécurité à New Relic [#report]

New Relic s&apos;engage à assurer la sécurité de ses clients et de leurs données. Pour plus d&apos;informations, consultez [Signalement des vulnérabilités de sécurité](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities).
