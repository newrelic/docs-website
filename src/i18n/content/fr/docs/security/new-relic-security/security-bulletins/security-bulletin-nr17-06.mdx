## Résumé

Une mise à jour de sécurité pour l&apos;agent .NET de New Relic corrige une vulnérabilité où l&apos;agent pouvait capturer involontairement des paramètres de requête à partir requests HTTP externes lors d&apos;une trace de transaction.

<DNT>**Release date:**</DNT> 18 décembre 2017

<DNT>**Vulnerability identifier:**</DNT> NR17-06

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

L&apos;agent .NET de New Relic capture une trace de transaction pour obtenir des informations détaillées sur une transaction unique, y compris les appels de fonction, les appels de base de données et les appels externes. Par défaut, l&apos;agent ne doit pas collecter les détails des appels externes ( requests HTTP). Les versions précédentes de l&apos;agent collecteront les paramètres de requête http externes qui peuvent contenir des informations sensibles. Cette sortie corrige le comportement par défaut de l&apos;agent .NET pour ne pas collecter les paramètres de requête des requests HTTP externes.

### Facteurs atténuants [#factors]

* paramètres de requête utilisés dans requests HTTP externes effectuées à l&apos;aide [de HttpClient](https://msdn.microsoft.com/en-us/library/system.net.http.httpclient.aspx), [WebRequest](https://msdn.microsoft.com/en-us/library/system.net.webrequest.aspx) ou d&apos;une autre bibliothèque qui utilise ces appels.
* trace de transaction activée (paramètre par défaut)

## Solutions de contournement

Désactiver [trace de transaction](/docs/agents/net-agent/configuration/net-agent-configuration#transaction_tracer) dans la configuration de l&apos;agent .NET.
