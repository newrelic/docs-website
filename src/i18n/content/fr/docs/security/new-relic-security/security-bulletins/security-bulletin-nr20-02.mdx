## Résumé

Si l&apos;agent Node.js est configuré pour exclure l&apos;attribut `request.uri`, il capturera toujours l&apos;URI dans la trace de transaction. Cela peut être problématique pour certains clients dans des environnements où des informations sensibles sont incluses dans l&apos;URI.

<DNT>**Release date:**</DNT> 20 août 2020

<DNT>**Vulnerability identifier:**</DNT> NR20-02

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

Même lorsque l&apos;utilisateur configure l&apos;agent Node.js pour exclure l&apos;attribut `request.uri`, l&apos;agent capturera toujours l&apos;URI dans la trace de la transaction. Cela permet à l&apos;utilisateur de compte authentifié d&apos;afficher l&apos;URI partout où les détails trace de la transaction peuvent être visualisés via New Relic One ou une requête. Cela inclut (mais sans s&apos;y limiter) la section <DNT>**Transaction traces**</DNT> de la page <DNT>**Transactions**</DNT> , le <DNT>**Transaction trace details**</DNT> et le générateur de requêtes dans l&apos;interface utilisateur.

### Facteurs atténuants [#factors]

Cela n&apos;affectera que les agents Node.js configurés pour exclure l&apos;attribut `request.uri` .
