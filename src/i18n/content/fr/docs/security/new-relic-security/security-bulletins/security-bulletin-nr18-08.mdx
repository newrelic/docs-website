## informations sur les vulnérabilités [#vuln-info]

L&apos;agent New Relic Node.js utilise `https-proxy-agent` comme option pour envoyer des données au [collecteur New Relic](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#collector) via un serveur proxy HTTP ou HTTPS. L&apos;agent Node.js utilisait une version de ce module qui était vulnérable à l&apos;exposition de mémoire non initialisée et au déni de service. Ce correctif met à jour le module vers une version qui n&apos;est pas vulnérable.

## Circonstances atténuantes [#workarounds]

Le déclenchement de ces vulnérabilités de sécurité nécessite le contrôle de la configuration de l&apos;authentification proxy de l&apos;agent.

* [Snyk: `https-proxy-agent`](https://snyk.io/vuln/npm:https-proxy-agent:20180402)
* [Rapport HackerOne](https://hackerone.com/reports/319532)

## Solutions de contournement

New Relic n&apos;a identifié aucune solution de contournement pour ces vulnérabilités.

New Relic s&apos;engage à assurer la sécurité de ses clients et de leurs données. Si vous pensez avoir découvert une faille de sécurité dans l&apos;un de nos produits ou sites Web, nous vous invitons à la signaler au programme de divulgation coordonnée de New Relic. Pour plus d&apos;informations, consultez [Signalement des vulnérabilités de sécurité](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities).
