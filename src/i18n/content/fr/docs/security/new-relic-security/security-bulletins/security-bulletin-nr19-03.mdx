## Résumé

Une mise à jour de sécurité pour l&apos;agent .NET corrige un problème où certaines combinaisons de caractères peuvent empêcher obfuscation des requêtes lors de l&apos;exécution de Microsoft SQL Server.

<DNT>**Release date:**</DNT> 22 avril 2019

<DNT>**Vulnerability identifier:**</DNT> NR19-03

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

Lorsqu&apos;il est configuré pour obscurcir la requête SQL, l&apos;agent .NET transmettra toutes les requêtes via une fonction obfuscation . En raison de la manière dont Microsoft SQL Server nettoie les requêtes, certaines combinaisons de caractères peuvent être générées et entraîner l’échec de cette obfuscation .

### Facteurs atténuants [#factors]

Cette vulnérabilité n&apos;existe que lorsque l&apos;instrumentation Microsoft SQL Server et obfuscation des requêtes sont activées.

## Solutions de contournement

* Désactiver l&apos;instrumentation de la base de données.
* [Mise à jour vers le dernier agent .NET New Relic.](/docs/agents/net-agent/installation/update-net-agent)
