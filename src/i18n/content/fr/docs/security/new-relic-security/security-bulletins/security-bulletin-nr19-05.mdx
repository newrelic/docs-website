## Résumé

Une mise à jour de sécurité pour l&apos;agent .NET corrige un problème où les noms de métriques ne sont pas correctement identifiés pour les requêtes SQL avec des paramètres qui ont été construits manuellement.

<DNT>**Release date:**</DNT> 26 août 2019

<DNT>**Vulnerability identifier:**</DNT> NR19-05

<DNT>**Priority:**</DNT> Moyen

## informations sur les vulnérabilités [#vuln-info]

Lors de la construction manuelle d&apos;une requête SQL qui exécute des procédures stockées avec des paramètres, un espace manquant avant la première valeur peut amener l&apos;agent à identifier incorrectement le nom de la métrique. Cela peut entraîner l&apos;inclusion de données sensibles dans les noms métriques.

### Facteurs atténuants [#factors]

Cette vulnérabilité affecte uniquement les applications qui assemblent manuellement des requêtes SQL avec des paramètres, sans utiliser de requête paramétrée. Il est recommandé aux applications d&apos;utiliser des requêtes paramétrées pour éviter d&apos;introduire des vulnérabilités injection SQL.

## Solutions de contournement

* Utilisez des requêtes paramétrées, cela permet également d&apos;éviter [les vulnérabilités injection SQL](https://docs.microsoft.com/en-us/sql/relational-databases/security/sql-injection).
* [Mise à jour vers le dernier agent .NET New Relic.](/docs/agents/net-agent/installation/update-net-agent)
