New Relic CodeStream permet aux développeurs de voir comment le code dont ils sont responsables fonctionne en production en apportant l&apos;observabilité dans l&apos;IDE et en l&apos;intégrant à leur routine quotidienne.

Les données d&apos;observabilité affichées dans CodeStream sont contextuelles au référentiel que vous avez ouvert dans votre IDE. Si ces référentiels ne sont pas associés à des services sur New Relic, vous serez invité à [effectuer ces associations](/docs/codestream/observability/repo-association). Sinon, vous obtiendrez une vue complète des performances des services créés à partir de votre code.

<img title="Performance data for your services" alt="An image showing performance data in the IDE." src="/images/codestream_screenshot-crop_o11y.webp" />

Pour chaque service, les données de performance suivantes peuvent être affichées :

* <DNT>**Golden metrics**</DNT>:Les signaux clés indiquant les performances de votre service. Les métriques varient en fonction du type de service (APM, Browser, Mobile, OTel, fonction Lambda).
* <DNT>**Issues**</DNT>: Tous les problèmes ouverts basés sur la condition d&amp;apos;alerte configurée pour le service (APM uniquement).
* <DNT>**Service-level objectives**</DNT>:Un résumé des performances du service par rapport aux objectifs de niveau de service configurés (APM, Browser, OTel).
* <DNT>**Transaction performance**</DNT>:Voyez quelles transactions présentent des problèmes de performances depuis votre dernière sortie. [En savoir plus sur les performances des transactions](/docs/codestream/observability/transactions) (APM uniquement).
* <DNT>**Errors**</DNT>:Erreurs récentes se produisant dans le code dont vous êtes responsable. Cliquez sur une erreur pour parcourir la trace d&amp;apos;appels et collaborer avec vos coéquipiers sur l&amp;apos;enquête. [En savoir plus sur l&apos;enquête sur les erreurs](/docs/codestream/observability/error-investigation) (APM, Browser).
* <DNT>**Vulnerabilities**</DNT>: vulnérabilités au niveau du package, y compris des informations sur la manière de les corriger (APM uniquement).
* <DNT>**Related services**</DNT>:L&amp;apos;ensemble des services qui appellent ou sont appelés par votre service, avec la possibilité de voir les métriques dorées pour chacun d&amp;apos;eux. Comprenez comment votre service s’intègre dans le tableau plus large (APM, Browser, mobile, OTel).
* <DNT>**Logs**</DNT>: Recherchez les logs directement depuis votre IDE pour éviter le changement de contexte. [En savoir plus sur la recherche de log](/docs/codestream/observability/log-search) (APM, Browser, OTel).

Lorsque vous survolez un nom de service dans l&apos;arborescence, cliquez sur le globe pour ouvrir la page de résumé du service sur New Relic.

Pour utiliser la fonctionnalité d&apos;observabilité dans CodeStream, notez que vous devez disposer d&apos;un type d&apos;utilisateur New Relic [de base ou d&apos;une plateforme complète](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/#user-type-capabilities) ou que vous devez être sur le modèle de tarification New Relic Compute.
