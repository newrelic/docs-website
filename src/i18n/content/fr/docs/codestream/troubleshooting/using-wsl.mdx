Pour utiliser CodeStream et WSL (sous-système Windows pour Linux), procédez comme suit :

1. Assurez-vous que Git est installé sur la machine Windows, ainsi que dans votre distribution WSL. Bien que les versions installées entre les deux puissent différer, il est préférable de s&apos;assurer que la version Windows correspond à celle installée dans votre distribution WSL.

2. Clonez votre référentiel git (dépôt) dans votre distribution WSL (par exemple, `~/code/YOUR_PROJECT_FOLDER`).

3. Sous Windows, autorisez Git à communiquer via ce chemin UNC en marquant le répertoire comme `safe`. Faites-le dans votre terminal Windows avec cette commande, en remplaçant `YOUR_DISTRO`, `YOUR_USERNAME` et `YOUR_PROJECT_FOLDER` pour correspondre à votre configuration :

   ```shell
   git config --global --add safe.directory '//wsl.localhost/YOUR_DISTRO/home/YOUR_USERNAME/code/YOUR_PROJECT_FOLDER' 
   ```

4. Dans votre IDE, ouvrez le projet en utilisant le chemin UNC WSL. Par exemple, si vous aviez Ubuntu, votre nom d&apos;utilisateur était `codestream` et votre dossier de projet était `Amazing_App`, votre chemin UNC serait `//wsl.localhost/Ubuntu/home/codestream/code/Amazing_App`.

<Callout variant="tip">
  Git sur Windows est uniquement utilisé pour faire requests vers votre référentiel (repository) cloné localement dans WSL et n&apos;a pas besoin de communiquer avec le distant (par exemple, GitHub, GitLab, etc.).
</Callout>

Si vous rencontrez des problèmes après avoir effectué ces étapes, veuillez [récupérer votre fichier log](/docs/codestream/troubleshooting/client-logs) et contacter le support client de New Relic pour une enquête plus approfondie.
