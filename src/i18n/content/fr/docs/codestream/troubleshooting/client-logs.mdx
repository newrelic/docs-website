Lorsque vous signalez des problèmes au support client New Relic, il est souvent utile d&apos;inclure un fichier de log. Suivez ces instructions pour que votre IDE obtienne votre fichier de log.

## JetBrains [#jetbrains]

1. Reproduisez le problème.
2. Dans votre IDE, accédez à <DNT>**Help &gt; Collect Logs and Diagnostic Data**</DNT>. Cela ouvrira le Finder où vous devriez voir un fichier zip nouvellement créé.

## Code VS [#vscode]

1. Reproduisez le problème.
2. Ouvrez la vue <DNT>**Output**</DNT> dans VS Code (<DNT>**View: Toggle Output**</DNT> depuis la palette de commandes) et sélectionnez <DNT>**CodeStream (Agent)**</DNT> dans le menu déroulant en haut à droite. Copiez toutes les sorties que vous voyez et enregistrez-les dans un fichier texte.

## Visual Studio [#visual-studio]

1. Allez à <DNT>**Tools &gt; Options &gt; CodeStream**</DNT> et assurez-vous que votre niveau de log est défini sur au moins `Debug`. S’il est défini sur `Info`, `Errors` ou `Silent`, remplacez-le par `Debug`, puis redémarrez Visual Studio.
2. Reproduisez le problème.
3. Récupérez `vs-2022-extension.log` et `vs-2022-agent.log` de `%localappdata%\CodeStream\Logs` ainsi que `ActivityLog.xml` de `%AppData%\Microsoft\VisualStudio\17.0_<RandomText>`.
