New Relic CodeStream est un IDE d&apos;extension qui vous permet de vous déplacer vers la gauche en intégrant les performances du code aux premières étapes du processus de développement.

* Découvrez les performances des services créés à partir de votre code, avec des informations sur les métriques dorées, les erreurs, les anomalies de transaction, les SLO, etc.
* Enquêtez sur les erreurs plus rapidement en parcourant la trace d&apos;appels et en collaborant avec vos coéquipiers.
* Recherchez dans le log de votre IDE pour étudier les problèmes sans changer de contexte.
* Exécutez la requête NRQL, puis enregistrez et partagez la requête avec vos coéquipiers via les fichiers `.nrql`.
* Consultez les données de performances des méthodes individuelles dans le code, directement dans l&apos;éditeur.

## Enquête sur les erreurs

Découvrez les erreurs qui se produisent dans le code dont vous êtes responsable. Parcourez rapidement la trace des appels, CodeStream vous amenant automatiquement au bon fichier et au bon numéro de ligne, puis impliquez les coéquipiers concernés dans la discussion afin que vous puissiez résoudre le problème le plus rapidement possible.

## Recherche de log

Si vous devez enquêter sur un problème en recherchant dans le log, pourquoi changer de contexte en dehors de votre IDE pour effectuer ce travail sur le Web ? Recherchez dans le log directement depuis votre IDE. Vous pouvez même lancer une recherche à partir de lignes log individuelles dans votre code.

## Métriques au niveau du code

Consultez les données de performance, au niveau de la méthode, directement dans votre éditeur. La vue permanente garantit que les problèmes de performances sont pris en compte dans le travail en cours. Vous pouvez même voir les métriques de votre environnement de préproduction et anticiper les problèmes de performances avant qu&apos;ils n&apos;atteignent la production.
