Chaque dépôt public des organisations GitHub New Relic et New Relic Experimental exploite l&apos;une des catégories open source suivantes :

<CollapserGroup>
  <Collapser id="community-project" title="Projet communautaire">
    Ce code est développé de manière ouverte avec la contribution de la communauté via des problèmes et des relations publiques. Il existe une équipe de maintenance active au sein de New Relic, un support de dépannage dans le New Relic Explorers Hub, et la documentation est disponible dans le référentiel du projet.

    <Callout variant="important">
      Il s&apos;agit d&apos;un projet communautaire open source officiel de New Relic, soutenu par les mainteneurs de New Relic.
    </Callout>

    **Exigences**

    * Conforme aux exigences légales et de sécurité de New Relic pour les logiciels open source
    * Contient l&apos;en-tête de catégorie open source New Relic approprié dans le fichier README du référentiel
    * A un ou plusieurs mainteneurs actifs, dont au moins un Relic
    * Assistance au dépannage via le hub New Relic Explorers
    * Problèmes et relations publiques gérés dans GitHub
    * Documentation révisée par l&apos;équipe de documentation de New Relic
    * Code linté
    * Un pipeline de publication automatisé
  </Collapser>

  <Collapser id="community-plus" title="Communauté Plus">
    Ce code est développé de manière ouverte avec la contribution de la communauté via des problèmes et des relations publiques. Une équipe d&apos;ingénieurs New Relic assure la maintenance. L&apos;assistance au dépannage est disponible dans le New Relic Explorers Hub et la documentation est disponible dans le référentiel du projet et dans [New Relic Docs](https://docs.newrelic.com).

    <Callout variant="important">
      Ce code est actuellement maintenu par les équipes d&apos;ingénierie de New Relic et livré ici sur GitHub. Consultez le fichier README pour obtenir des instructions de dépannage et de signalement des défauts.
    </Callout>

    **Exigences**

    * Conforme aux exigences légales et de sécurité de New Relic pour les logiciels open source
    * Contient l&apos;en-tête de catégorie open source New Relic approprié dans le fichier README du référentiel
    * Est maintenu par une équipe d&apos;ingénieurs New Relic
    * Assistance au dépannage via le hub New Relic Explorers
    * Problèmes et relations publiques gérés dans GitHub

  <Collapser id="one-catalog" title="Catalogue New Relic">
    Ce code fait partie du catalogue New Relic. Il répond aux critères d&apos;un projet communautaire ; mais il contient également des flux de travail d&apos;interface utilisateur pour la configuration. La plupart des projets de catalogue cherchent à maintenir une feuille de route publique, souvent exprimée dans un tableau de projet GitHub et des problèmes au sein du référentiel.

    <Callout variant="important">
      Ce code fait partie du catalogue New Relic. Il est disponible pour l&apos;installation et la configuration via la page d&apos;accueil de New Relic. Vous pouvez l&apos;installer dans New Relic.
    </Callout>

    **Exigences**

    * Toutes les exigences d&apos;un projet communautaire
    * Un flux de travail d&apos;application d&apos;état vide qui guide les utilisateurs tout au long de la configuration des données de configuration stockées dans NerdStorage
    * Revue architecturale (y compris UX) par New Relic
    * (Facultatif) maintient une feuille de route publique (recommandée via un projet GitHub dans le dépôt)
  </Collapser>
