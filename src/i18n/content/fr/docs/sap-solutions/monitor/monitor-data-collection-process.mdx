Ce guide vous montre comment monitorer le processus de collecte de données pour New Relic Monitoring for SAP Solutions. Vous pouvez utiliser plusieurs méthodes pour vérifier que la collecte de données fonctionne correctement.

## Prérequis

* Accès à <DNT>**SAP system**</DNT> avec les permissions de transaction appropriées
* <DNT>**New Relic Monitoring for SAP Solutions**</DNT> installé et configuré

## État d&apos;exécution et logde transmission

À partir du <DNT>**Data Provider Registry**</DNT> (transaction `/NEWR/REGISTRY`), vous pouvez vérifier le <DNT>**Runtime Status**</DNT> et le <DNT>**Transmission Log**</DNT> pour chaque fournisseur de données. Le log de transmission indique le statut et l&apos;heure de chaque transmission.

## Vérifier l&apos;état du travail et le spool

1. Utilisez la transaction `SM37` pour vérifier l’état des tâches planifiées. Saisissez `/NEWR/*` dans le champ <DNT>**ABAP Program Name**</DNT> et cliquez sur <DNT>**Execute**</DNT>.
2. Examiner la durée d&apos;exécution. Un temps d&apos;exécution anormalement long indique un problème potentiel.
3. Vérifiez si des jobs se chevauchent et ajustez la planification si nécessaire. Accédez au spool du job à partir des détails du job pour consulter le résultat pour chaque fournisseur de données.

## Vérifier log d&apos;application

Les processus de collecte de données sont enregistrés dans le log d&apos;application (transaction `SLG1`). Requête avec le <DNT>**Application Log Object**</DNT> `/NEWR/DP`.

## dashboard monitoring de la collecte de données SAP

Les métriques de collecte de données sont également envoyées à New Relic pour l&apos;auto-monitoring. Utilisez le dashboard <DNT>**SAP Data Collection Monitoring**</DNT> prêt à l’emploi pour monitorer les processus de collecte de données pour chaque fournisseur de données.
