Effectuez ces tâches de configuration pour chaque système monitoré.

## Rôle d&apos;autorisation de téléchargement [#upload-authorization-role]

1. Utilisez <DNT>**SAP transaction**</DNT> `PFCG`.

2. Accédez à <DNT>**Role**</DNT> &gt; <DNT>**Upload**</DNT> et téléchargez le fichier <DNT>**\_NEWR\_DATA\_PROVIDER.SAP**</DNT> à partir du package d’installation.

3. Confirmez que le rôle `/NEWR/DATA_PROVIDER` se charge.

   <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

4. Sélectionnez l&apos;<img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   icône après le chargement du rôle.

5. Dans l&apos;onglet <DNT>**Authorizations**</DNT>, sélectionnez<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   <DNT>**Change Authorization Data**</DNT>, puis activez-le à l&apos;aide de<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP">icône.</img>

   <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

6. Si un nom de profil est suggéré, sélectionnez l&apos;icône de coche verte.

7. Vérifiez que <DNT>**Profile(s) were updated**</DNT> apparaît dans la barre d&apos;état.

## Créer un nouvel identifiant utilisateur RFC Relic [#create-rfc-user-id]

Après avoir activé le <DNT>**authorization profile**</DNT>, utilisez la transaction `SU01` pour créer un nouveau <DNT>**RFC user ID**</DNT> (par exemple, `NR4SAPDP`). Définissez le <DNT>**user type**</DNT> sur <DNT>**System**</DNT> ou <DNT>**Communication**</DNT> et ajoutez le rôle `/NEWR/DATA_PROVIDER`.

<img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />

## Créer des destinations RFC pour le système de monitoring [#create-rfc-for-monitored-systems]

Utilisez la transaction `SM59` pour créer un <DNT>**RFC destination**</DNT> pour chaque système monitoré.

1. Sélectionnez <DNT>**ABAP Connections**</DNT>, puis sélectionnez l&apos;icône <DNT>**Create**</DNT> pour configurer un nouveau <DNT>**RFC destination**</DNT>.

2. Saisissez un nom de <DNT>**RFC Destination**</DNT> descriptif. Nous recommandons : `NR_<SYSID>CLNT<CLNT#>`

3. Réglez <DNT>**Connection Type**</DNT> sur `3 - ABAP Connection`.

4. Dans l&apos;onglet <DNT>**Technical Settings**</DNT>, saisissez :

   * <DNT>**Target Host**</DNT> (domaine entièrement qualifié ou IP) et <DNT>**Instance No.**</DNT>
   * <DNT>**Gateway Host**</DNT> et <DNT>**Service**</DNT> si nécessaire.

   <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

5. Dans l&apos;onglet <DNT>**Logon &amp; Security**</DNT>, entrez le <DNT>**RFC user**</DNT> (`NR4SAPDP`) et le mot de passe.

6. Sélectionnez <DNT>**Save**</DNT>.

7. Allez à <DNT>**Utility &gt; Test &gt; Connection Test**</DNT> et <DNT>**Authorization Test**</DNT>.

   * <DNT>**Expected result**</DNT>: les deux tests affichent un état de réussite.

## Définir la destination HTTP sur le cloud New Relic

Utilisez la transaction `SM59` pour configurer <DNT>**HTTPS connections**</DNT> sur le <DNT>**New Relic API endpoints**</DNT>.

<Callout variant="important">
  Créez des <DNT>**RFC destinations**</DNT> individuels pour <DNT>**Events**</DNT>, <DNT>**Logs**</DNT>, <DNT>**Metrics**</DNT> et <DNT>**Traces**</DNT> séparément.
</Callout>

1. Saisissez un <DNT>**Destination Name**</DNT> (par exemple, `NR_API_EVENT`).

2. Définissez <DNT>**Connection Type**</DNT> sur `G` (Connexions HTTP vers un serveur externe).

3. Saisissez <DNT>**Target Host**</DNT>, <DNT>**Service No.**</DNT> comme <DNT>**443**</DNT> (voir [le port standard](/docs/sap-solutions/reference/network-accessibility) pour <DNT>**HTTPS**</DNT>) et <DNT>**Path Prefix**</DNT> (voir [la liste de contrôle de préparation à l’installation](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness) pour plus de détails).

4. Si vous avez besoin d&apos;un proxy, configurez-le dans l&apos;onglet <DNT>**Technical Settings**</DNT>.

5. Dans l&apos;onglet <DNT>**Security Options**</DNT>, sélectionnez l&apos;option <DNT>**Active**</DNT> pour <DNT>**SSL**</DNT>.

   <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />

6. Si nécessaire, téléchargez le <DNT>**root certificate**</DNT> depuis [New Relic](https://one.newrelic.com) et ajoutez-le à <DNT>**SAP**</DNT> en utilisant la transaction `STRUST`.

7. Sélectionnez <DNT>**Connection Test**</DNT> pour vérifier la connexion.

   * <DNT>**Expected result**</DNT>: une connexion réussie vous prompt à entrer les données de connexion.
   * <DNT>**If test times out**</DNT>: vérifiez les paramètres de votre pare-feu.

8. Répétez cette procédure pour tous les <DNT>**New Relic API endpoints**</DNT>.

## Créer une connexion à la base de données [#create-database-connection]

Utilisez la transaction `DBACOCKPIT` pour définir le <DNT>**HANA Database connections**</DNT> distant.

1. Double-cliquez sur <DNT>**Database Connections**</DNT> et sélectionnez <DNT>**Add**</DNT>.

   <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

2. Saisissez un <DNT>**Connection Name**</DNT> (de préférence le même que le <DNT>**SAP system ID**</DNT>).

3. Sélectionnez <DNT>**HANA Database**</DNT> comme <DNT>**Database System**</DNT>.

4. Saisissez <DNT>**Database user ID**</DNT> (par exemple, `NRAGENT`) et le mot de passe.

5. Saisissez <DNT>**Database Host**</DNT> et <DNT>**SQL Port**</DNT>.

   <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

6. Sélectionnez <DNT>**Save**</DNT>, puis sélectionnez <DNT>**Test**</DNT> pour vérifier la connexion.

   * <DNT>**Expected result**</DNT>: le test de connexion indique un statut de réussite.
