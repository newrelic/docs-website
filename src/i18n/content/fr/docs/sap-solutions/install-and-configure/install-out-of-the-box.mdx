Le monitoring New Relic pour SAP inclut une liste de composants de monitoring prêts à l’emploi, y compris des dashboards, des règles d’alerte et l’IDOC Explorer. Suivez les instructions ci-dessous pour les localiser et les installer.

## Avant de commencer

Pour effectuer ces tâches, vous devez avoir les privilèges d&apos;administrateur New Relic sur le compte.

<Callout variant="important">
  Pour le dashboard <DNT>**SAP Cockpit**</DNT>, vous devez [créer une table de recherche](#create-lookup-table) nommée `NR_LKUP_SYSTEM` avant l&apos;installation.
</Callout>

## Installer le dashboard

### Créer une table de recherche [#create-lookup-table]

<Callout variant="tip">
  Requis uniquement pour le dashboard <DNT>**SAP Cockpit**</DNT>.
</Callout>

Le dashboard <DNT>**SAP Cockpit**</DNT> utilise un <DNT>**lookup table**</DNT> pour <DNT>**SAP systems**</DNT> et leurs environnements.

1. Créez le <DNT>**lookup table**</DNT> avec deux colonnes : `SYS_ID` et `Env`, et enregistrez-le en tant que <DNT>**CSV file**</DNT>. Voici un exemple du fichier de recherche :

   ```csv
   SYS_ID,Env
   GD1,DEV
   SD2,DEV
   AQ1,Testing
   AQ2,Testing
   SP1,Prod
   SP2,Prod
   ```

2. Allez à <DNT>**Log &gt; Lookup Tables**</DNT>.

3. Cliquez sur <DNT>**Add a table**</DNT>.

4. Dans la barre latérale <DNT>**Add a lookup table**</DNT>, nommez la table <DNT>**NR\_LKUP\_SYSTEM**</DNT> et téléchargez le <DNT>**CSV file**</DNT>. Voir [Tables de recherche](/docs/logs/ui-data/lookup-tables-ui/) pour plus d’informations.

5. Pour vérifier le contenu de la table, utilisez le <DNT>**query builder**</DNT> avec `FROM lookup(NR_LKUP_SYSTEM) SELECT *`, puis cliquez sur <DNT>**Run**</DNT>.

### Installer les dashboards [#install-dashboards]

1. Allez dans <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> &gt; <DNT>**Integrations &amp; Agents**</DNT>, et recherchez <DNT>**New Relic Monitoring for SAP Solutions**</DNT> pour télécharger automatiquement tous les dashboards

2. Sinon, vous pouvez accéder à <DNT>**Dashboards &gt; Recommended Dashboards**</DNT>, cliquer sur <DNT>**View all**</DNT> et rechercher SAP. Consultez [Dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) pour plus d’informations.

3. Installer ces dashboards :

   * <DNT>**SAP Dashboards**</DNT>
   * <DNT>**SAP Cockpit**</DNT>
   * <DNT>**SAP Data Collection Monitoring**</DNT>
   * <DNT>**SAP Data Ingestion Estimate**</DNT>

### Ajouter un filtre dashboard [#add-dashboard-filter]

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**</DNT>.

2. Activer la liaison de facettes pour le filtrage :

   * Cliquez sur les points de suspension (<DNT>**…**</DNT>) en haut à droite du widget
   * Sélectionnez <DNT>**Edit**</DNT> pour ouvrir l&apos;écran d&apos;édition
   * Activez/désactivez le commutateur <DNT>**Filter the current dashboard**</DNT>

3. <DNT>**Facet linking**</DNT> peut être activé pour les tableaux, les graphiques à secteurs et les widgets de graphique à barres pour filtrer par <DNT>**SAP system**</DNT>, <DNT>**instance**</DNT>, <DNT>**IDOC status**</DNT>, <DNT>**jobs**</DNT>, etc.

## Installer les règles d’alerte

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Alerts &gt; Alert Policies**</DNT>.

2. Recherchez et cliquez sur <DNT>**New Relic Monitoring for SAP Solutions alert policy**</DNT>.

   <img title="New Relic Monitoring for SAP Solutions alert policy" alt="A screenshot displaying the connection setup for New Relic Monitoring for SAP Solutions alert policy" src="/images/new-relic-monitoring-for-sap-solutions-alert-policy.webp" width="" />

3. Dans l&apos;onglet <DNT>**Alert condition**</DNT>, sélectionnez le compte et basculez le <DNT>**alert policy**</DNT> sur <DNT>**Enable**</DNT>. Consultez [Gérer vos politiques](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/) pour savoir comment gérer les politiques.

4. Revenez en arrière et recherchez <DNT>**Nr4sap default alert policy**</DNT>. Activez le <DNT>**alert condition**</DNT> que vous souhaitez activer.

## Installer l&apos;explorateur IDOC

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Integrations &amp; Agents**</DNT>.

2. Sélectionnez <DNT>**Apps &amp; Visualizations**</DNT>, et recherchez <DNT>**IDOC Explorer**</DNT>.

3. Sélectionnez <DNT>**Manage access**</DNT>. Choisissez le compte pour accéder à l&apos;application.

   <img title="IDOC Explorer" alt="A screenshot displaying the manage access settings for IDOC Explorer" src="/images/idoc-explorer.webp" />

4. Cliquez sur <DNT>**Update**</DNT>. Le <DNT>**IDOC Explorer**</DNT> sera visible lors de la visualisation de <DNT>**IDOC entities**</DNT>.
