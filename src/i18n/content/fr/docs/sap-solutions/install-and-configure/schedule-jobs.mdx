## Programme de vérification RFC [#rfc-check]

Planifiez `/NEWR/REGISTRY_RFC_CHECK` toutes les 1 à 3 minutes, en vous assurant qu&apos;il s&apos;exécute plus fréquemment que le <DNT>**data collection job**</DNT>. Utilisez le nom du job `NR_RFC_CHECK`.

## Planifier le gestionnaire de collecte de données ABAP [#abap-collection-manager]

Planifiez trois tâches de collecte de données avec des variantes axées sur les groupes de fournisseurs de données suivants. Pour obtenir des instructions sur la planification des tâches du gestionnaire de collecte avant `SP03`, consultez [Variantes et tâches avant SP03](/docs/sap-solutions/reference/variants-and-jobs-before-sp03). Séparez les fournisseurs de données en trois groupes pour la planification. Pour une liste complète de tous les fournisseurs de données, consultez la [Référence des fournisseurs de données](/docs/sap-solutions/monitor/data-providers-reference).

* <DNT>**TRACE group**</DNT> inclut les fournisseurs de données liés à la base de données de performance (STAD) :

  * [`NR.SAP.TRANSACTION`](/docs/sap-solutions/monitor/data-providers-reference/#sap-appl-metrics): fournisseur de données de trace
  * [`NR.SAP.TRANSACTION.COMPLETE`](/docs/sap-solutions/monitor/data-providers-reference/#sap-appl-metrics): collecte complète de la base de données de performance avec processus RFC parallèle en arrière-plan
  * `NR.SAP.SYSTEM.PROCESS` et `NR_SAP_PROGRAM`: suspendu après SP02
  * `NR_SAP_WORKLOAD_COMPLETE` et `NR.SAP.WORKLOAD.COMPLETE`

* <DNT>**IDOC group**</DNT> inclut les fournisseurs liés à IDOC (peut prendre un temps important en production) :

  * [`NR_SAP_IDOC`](/docs/sap-solutions/monitor/data-providers-reference/#idoc-all): fournisseur de données d’événement d’état IDOC principal
  * [`NR_SAP_IDOC_COMPLETE`](/docs/sap-solutions/monitor/data-providers-reference/#idoc-all): récupération IDOC complète avec collecte RFC en arrière-plan
  * `NR_SAP_IDOC_Configuration`: télécharger les données de base IDOC pour améliorer les performances

* <DNT>**SYSTEM group**</DNT> comprend tous les fournisseurs de données restants. Consultez la [liste complète par groupe de monitoring](/docs/sap-solutions/monitor/data-providers-reference).

#### Planifier chaque tâche

1. Démarrer la transaction `SM36`.
2. Saisissez le <DNT>**Job Name**</DNT> pour chaque travail et définissez le <DNT>**Job Class**</DNT> sur <DNT>**C**</DNT>.
3. Cliquez sur <DNT>**Step**</DNT>, saisissez le <DNT>**Program Name**</DNT> `/NEWR/ONE_COLLECTION` et sélectionnez le <DNT>**Variant**</DNT>.
4. Cliquez sur <DNT>**Start Condition**</DNT>, définissez <DNT>**Start Date**</DNT> et <DNT>**Time**</DNT>, et cochez la case <DNT>**Periodic Job**</DNT>.
5. Accédez à <DNT>**Period Values**</DNT>, cliquez sur <DNT>**Other Period**</DNT> et saisissez <DNT>**5**</DNT> pour les minutes.

## Planifier une nouvelle soumission du programme [#resubmit-program]

1. Créez un <DNT>**variant**</DNT> nommé <DNT>**DEFAULT**</DNT> pour le programme `/NEWR/RESEND_FAILED_API_TRANS` afin de resoumettre les transmissions échouées pour le <DNT>**New Relic account**</DNT> en cas d&apos;erreur de transmission.

2. Spécifiez les valeurs suivantes :

   * [<DNT>**New Relic Account ID**</DNT>](/docs/accounts/accounts-billing/account-setup/account-id)
   * <DNT>**HTTP Response Codes**</DNT>: Exclure `000` et `413`, et la plage de `200` à `299`.

   <img title="Resend failed API transactions" alt="A screenshot displaying the resend failed API transactions" src="/images/resend-failed-api-transaction.webp" width="700" />

3. Planifiez l&apos;exécution de cette tâche toutes les 15 minutes pour resoumettre les transmissions échouées, telles que celles avec <DNT>**HTTP response code**</DNT> `500`, et purger périodiquement <DNT>**transmission log entries**</DNT>.
