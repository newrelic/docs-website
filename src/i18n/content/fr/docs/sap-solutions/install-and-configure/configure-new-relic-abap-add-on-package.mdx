## Utilisez le menu /NEWR/INSTALL [#menu-newr-install]

1. connectez-vous au client de production du système monitoring central.
2. Accédez à <DNT>**SAP Easy Access root menu**</DNT> en tapant `/n` dans le champ de commande et en appuyant sur <DNT>**Enter**</DNT>.
3. Accédez au menu <DNT>**`/NEWR/INSTALL`**</DNT> en tapant `/n/NEWR/INSTALL` dans le champ de commande.

Après avoir installé l&apos;add-on sur le système de monitoring central :

* Assurez-vous que le programme d&apos;installation dispose de l&apos;autorisation appropriée pour effectuer la configuration.
* Ajoutez le paramètre `/NEWR/*` aux objets d&apos;autorisation de code dans le `profile/role` du programme d&apos;installation.

<Callout variant="tip">
  Si vous mettez à niveau depuis une version précédente, consultez [Mettre à niveau depuis une version précédente](/docs/sap-solutions/upgrade/install-upgrade-package) pour déterminer quelles étapes sont nécessaires.
</Callout>

## Connecter les systèmes [#connect-systems]

<Callout variant="important">
  Terminez toutes les étapes de configuration en attente avant de continuer.
</Callout>

Les options suivantes sont disponibles dans le menu `/NEWR/INSTALL` :

### Configurer les destinations HTTP et RFC

Utilisez cette option pour appeler la transaction `SM59` et vérifier vos connexions <DNT>**RFC**</DNT> et <DNT>**HTTPS**</DNT>. Consultez [Créer une RFC pour les systèmes monitorés](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems) et [Définir la destination HTTP vers New Relic cloud](/docs/sap-solutions/install-and-configure/additional-configuration/#set-http-destination-to-new-relic-cloud).

### Configurer la connexion à la base de données distante

* Utilisez la transaction SAP `DBACOCKPIT` pour vérifier la configuration des connexions à <DNT>**HANA databases**</DNT> distant.
* Assurez-vous qu&apos;une connexion à la base de données est créée pour chaque base de données HANA distante que vous souhaitez monitorer.
* Pour des instructions détaillées, consultez [Créer une connexion à la base de données](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection).

### Configurer les connexions aux points de terminaison de l&apos;API NR

1. Cliquez sur <DNT>**New Entries**</DNT>.

2. Entrez les détails de votre compte New Relic :

   * <DNT>**New Relic Account ID**</DNT> (éventuellement préfixé par SAP-)
   * <DNT>**API Ingest Key**</DNT>
   * <DNT>**Metric Endpoint**</DNT> (voir [Points de terminaison réseau](/docs/sap-solutions/before-you-begin/#network-endpoints))
   * <DNT>**Event Endpoint**</DNT>
   * <DNT>**Log Endpoint**</DNT>
   * <DNT>**Trace Endpoint**</DNT>

3. Cliquez sur <DNT>**Save**</DNT>.

### Copier les tables de modèles du client 000

Exécutez cette option pour copier la table de <DNT>**client 000**</DNT> en cliquant sur l’icône d’exécution (symbole d’horloge). Confirmez que le message de réussite apparaît.

### Définir les systèmes monitorés

1. Cliquez sur <DNT>**New Entries**</DNT> pour saisir le numéro <DNT>**SAP System ID**</DNT> et <DNT>**Client**</DNT>.
2. Dans <DNT>**Data Provider Installation Information**</DNT>, spécifiez la <DNT>**Destination**</DNT> RFC, sélectionnez le <DNT>**Software Component**</DNT> SAP, entrez le <DNT>**Database Vendor**</DNT> et indiquez le <DNT>**DB Connection Name**</DNT> (base de données HANA uniquement).
3. Cliquez sur <DNT>**Monitoring Group**</DNT>, sélectionnez <DNT>**Yes**</DNT> dans la boîte de dialogue pour confirmer.
4. Choisissez les groupes de monitoring nécessaires, et cliquez sur l&apos;icône de génération pour terminer le processus.
5. Cliquez sur <DNT>**BACK**</DNT> ou sur la touche <DNT>**F3**</DNT> pour passer à l’étape suivante.

Utilisez cette option de menu pour tester les destinations RFC pour tous les systèmes monitorés et les points de terminaison d&apos;API NR, à l&apos;exception de <DNT>**NONE**</DNT>.

## Générer la configuration [#generate-configurations]

### Générer des configurations pour de nouveaux fournisseurs de données

* Utilisez cette option de menu pour générer des fournisseurs de données pour une nouvelle sortie ou un nouveau patch. Pour les nouvelles installations, tous les fournisseurs de données (DP) sont générés.
* Sur le premier écran, confirmez les détails de la sortie et du correctif. Les champs vides indiquent une configuration préexistante.
* Pour régénérer, sélectionnez <DNT>**Re-generate Data Provider Configurations**</DNT> dans <DNT>**Maintain Monitored Systems**</DNT> ou utilisez <DNT>**Reset Installation Flag (For Re-install only)**</DNT> pour réinitialiser l&apos;indicateur d&apos;installation et régénérer la configuration.

<img title="Maintain Monitored Systems" alt="A screenshot displaying the SAP maintain monitored systems" src="/images/maintain-monitored-systems.webp" width="500" />

### Vérifier l&apos;intégralité de l&apos;installation

Cliquez sur <DNT>**Verify Installation Completeness**</DNT> pour vous assurer que tout est correctement configuré. Si un élément affiche un indicateur rouge, cliquez sur le bouton d’exécution dans la colonne <DNT>**Action Needed**</DNT>.

## Vérifier la configuration [#verify-configuration]

1. Pour vous assurer que les fournisseurs de données sont correctement configurés, ouvrez le dossier <DNT>**Data Provider Registry Maintenance**</DNT>.
2. Cliquez sur <DNT>**Data Provider Registry**</DNT> ou utilisez la transaction `/n/NEWR/REGISTRY`.
3. Sélectionnez un <DNT>**Data Provider Name**</DNT> et vérifiez les paramètres de configuration en cliquant sur <DNT>**Configure**</DNT>.
4. Testez le fournisseur de données en cliquant sur <DNT>**TestDP**</DNT>. Le test devrait se terminer avec succès, confirmant que le fournisseur de données est correctement configuré.
