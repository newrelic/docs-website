Ce guide vous montre comment obtenir une clé de service à partir de l&apos;instance d&apos;API SAP Cloud Application Lifecycle Management (CALM). Vous aurez besoin de cette clé de service pour configurer le monitoring pour SAP Cloud ALM.

## Prérequis

* Accès à <DNT>**CALM subaccount**</DNT> avec les permissions appropriées
* Confirmation de votre <DNT>**BTP (Business Technology Platform) administrator**</DNT> concernant <DNT>**Space**</DNT>
* Scopes requis identifiés à partir de [Managing Scopes](https://help.sap.com/docs/cloud-alm/apis/managing-scopes)

1. Connectez-vous à <DNT>**CALM (Cloud Application Lifecycle Management) subaccount**</DNT>, puis cliquez sur <DNT>**Instances and Subscriptions**</DNT>.

2. Cliquez sur <DNT>**Create**</DNT> en haut à droite.

3. Dans l’écran contextuel, sélectionnez :

   * <DNT>**Service**</DNT>: <DNT>**SAP Cloud ALM API**</DNT>
   * <DNT>**Plan**</DNT>: <DNT>**Standard**</DNT>
   * <DNT>**Runtime Environment**</DNT>: <DNT>**Cloud Foundry**</DNT>
   * <DNT>**Space**</DNT>: vérifiez auprès de votre <DNT>**BTP administrator**</DNT>
   * <DNT>**Instance Name**</DNT>: suivez la convention de nommage de votre organisation

   <img title="SAP Cloud ALM API" alt="The Create Instance dialog showing Service, Plan, Runtime Environment, Space, and Instance Name fields for SAP Cloud ALM API" src="/images/sap-cloud-alm-api.webp" width="600" />

4. Cliquez sur <DNT>**Next**</DNT>.

5. Vérifiez et insérez les étendues requises à partir de [la gestion des étendues](https://help.sap.com/docs/cloud-alm/apis/managing-scopes) ainsi que le nom de l&apos;instance sous l&apos;onglet Paramètres.

   <img title="Manage Scopes" alt="The parameters tab showing the required scopes configuration for the CALM instance" src="/images/manage-scope.webp" width="600" />

6. Cliquez sur <DNT>**Next**</DNT> pour consulter les détails.

   <img title="Review Details" alt="The review screen showing the summary of the SAP Cloud ALM API instance configuration before creation" src="/images/cloud-alm-new-instance.webp" width="600" />

7. Cliquez sur <DNT>**Create**</DNT>.

8. Une fois créé, localisez le service sous l&apos;onglet <DNT>**Instances**</DNT>.

9. Cliquez sur les points de suspension (`...`) dans la ligne de votre instance nouvellement créée, puis sélectionnez <DNT>**Create Service Key**</DNT>.

10. Saisissez un <DNT>**Service Key Name**</DNT> et sélectionnez <DNT>**Key Type**</DNT>: `ClientId/Secret`.

11. Cliquez sur <DNT>**Create**</DNT>.

12. Cliquez sur les points de suspension (`...`) dans la ligne de votre <DNT>**service key**</DNT>, puis sélectionnez <DNT>**Download**</DNT>.

Le <DNT>**service key file**</DNT> se télécharge sous forme de <DNT>**JSON file**</DNT>. Conservez-le en toute sécurité – vous aurez besoin des identifiants de ce fichier pour configurer votre système de monitoring.
