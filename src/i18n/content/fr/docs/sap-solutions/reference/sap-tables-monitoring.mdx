Cette référence répertorie <DNT>**SAP tables**</DNT> récupérés à des fins de monitoring par <DNT>**New Relic Monitoring for SAP Solutions**</DNT>. Ces tables fournissent des données pour divers domaines de monitoring, y compris <DNT>**AIF**</DNT>, <DNT>**SLT**</DNT>, <DNT>**IDOC**</DNT>, les jobs, les e-mails, les transports, <DNT>**RFC**</DNT>, les processus métier et les documents de modification.

## Autorisation et accès aux tables

<DNT>**New Relic Monitoring for SAP**</DNT> Le rôle d&apos;autorisation fournit un accès étendu à l&apos;affichage des tables pour le <DNT>**RFC user**</DNT> dans le système monitoré lors de la configuration du <DNT>**RFC connections**</DNT>. Pour faciliter une personnalisation supplémentaire lors de la récupération de <DNT>**SAP tables**</DNT> supplémentaires, y compris de <DNT>**Z-tables**</DNT> personnalisés, ainsi que lors de la récupération de <DNT>**IDOC**</DNT> ou de données de charge du message, nous recommandons de conserver l&apos;accès aussi large. Cependant, si vous souhaitez restreindre l&apos;accès à la table, utilisez cette liste pour le faire.

<Callout variant="important">
  Si un collecteur de données personnalisé récupère des données directement depuis une table, le rôle de sécurité peut nécessiter une mise à jour pour que le nouveau fournisseur de données fonctionne.
</Callout>

## À qui s&apos;adresse cette référence

* <DNT>**SAP Basis**</DNT> administrateurs examinant le périmètre de la collecte de données
* Équipes de sécurité vérifiant les exigences d&apos;accès aux tables
* Administrateurs de base de données effectuant le monitoring de l&apos;utilisation des tables
