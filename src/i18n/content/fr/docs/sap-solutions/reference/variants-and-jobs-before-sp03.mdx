## Créer des variantes pour /NEWR/ABAP\_COLLECTION\_MANAGERN

1. Utilisez la transaction `SE38` pour configurer `/NEWR/ABAP_COLLECTION_MANAGERN`.
2. Sur le <DNT>**ABAP: Variants - Initial Screen**</DNT>, saisissez `/NEWR/ABAP_COLLECTION_MANAGERN` dans le champ <DNT>**Program**</DNT>.
3. Sélectionnez <DNT>**Variant**</DNT> et cliquez sur <DNT>**Create**</DNT>.
4. Suivez les étapes ci-dessous pour mettre à jour les valeurs :

<Tabs>
  <TabsBar>
    <TabsBarItem id="trace-variant">
      trace de variante
    </TabsBarItem>

    <TabsBarItem id="idoc-variant">
      variante IDOC
    </TabsBarItem>

    <TabsBarItem id="system-variant">
      variante du système
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="trace-variant">
      1. Saisissez le nom de la variante, par exemple <DNT>**TRACE\_ONLY**</DNT>, puis cliquez sur <DNT>**Create**</DNT>.

      2. Assurez-vous que le <DNT>**New Relic Account ID**</DNT> est renseigné automatiquement.

      3. Réglez <DNT>**Records per transmission**</DNT> sur <DNT>**10000**</DNT>.

      4. Cliquez sur <DNT>**Multiple Selection (Active)**</DNT> pour ajouter des noms de fournisseurs de données.

      5. Ajoutez ce qui suit dans <DNT>**Select Single Values**</DNT>:

         * `NR.SAP.TRANSACTION`
         * `NR.SAP.TRANSACTION.COMPLETE`
         * `NR.SAP.SYSTEM.PROCESS`

      6. Sélectionnez <DNT>**Sequential Processing**</DNT> comme <DNT>**Execution Mode**</DNT>.

      7. Si <DNT>**DIA WP**</DNT> ≥ 30, définir sur <DNT>**6**</DNT>; sinon, définir sur <DNT>**3**</DNT>.

      8. Cliquez sur <DNT>**Attributes**</DNT>, ajoutez une description et cliquez sur <DNT>**Save**</DNT>.
    </TabsPageItem>

    <TabsPageItem id="idoc-variant">
      1. Saisissez le nom de la variante, par exemple <DNT>**IDOC\_ONLY**</DNT>, puis cliquez sur <DNT>**Create**</DNT>.

      2. Assurez-vous que le <DNT>**New Relic Account ID**</DNT> est renseigné automatiquement.

      3. Réglez <DNT>**Records per transmission**</DNT> sur <DNT>**10000**</DNT>.

      4. Cliquez sur <DNT>**Multiple Selection (Active)**</DNT> pour ajouter des noms de fournisseurs de données.

      5. Ajoutez ce qui suit dans <DNT>**Select Single Values**</DNT>:

         * `NR_SAP_IDOC`
         * `NR_SAP_IDOC_COMPLETE`

      6. Sélectionnez <DNT>**Sequential Processing**</DNT> comme <DNT>**Execution Mode**</DNT>.

      7. Cliquez sur <DNT>**Attributes**</DNT>, ajoutez une description et cliquez sur <DNT>**Save**</DNT>.
    </TabsPageItem>

    <TabsPageItem id="system-variant">
      1. Saisissez le nom de la variante, par exemple <DNT>**System**</DNT>, puis cliquez sur <DNT>**Create**</DNT>.
      2. Assurez-vous que le <DNT>**New Relic Account ID**</DNT> est renseigné automatiquement.
      3. Réglez <DNT>**Records per transmission**</DNT> sur <DNT>**10000**</DNT>.
      4. Cliquez sur <DNT>**Multiple Selection (Active)**</DNT> pour ajouter des noms de fournisseurs de données.
      5. Ajoutez des variantes à partir des fournisseurs de <DNT>**TRACE**</DNT> et de <DNT>**IDOC**</DNT> dans <DNT>**Exclude Single Values**</DNT>.
      6. Sélectionnez <DNT>**Parallel Processing**</DNT> comme <DNT>**Execution Mode**</DNT>.
      7. Si <DNT>**DIA WP**</DNT> ≥ 30, définir sur <DNT>**6**</DNT>; sinon, définir sur <DNT>**3**</DNT>.
      8. Cliquez sur <DNT>**Attributes**</DNT>, ajoutez une description et cliquez sur <DNT>**Save**</DNT>.
    </TabsPageItem>
  </TabsPages>
</Tabs>

## Planifier des tâches de collecte

1. Utilisez la transaction `SM36` pour planifier trois tâches.
2. Saisissez le nom du travail pour chaque travail et définissez le <DNT>**Job Class**</DNT> sur <DNT>**C**</DNT>.
3. Cliquez sur <DNT>**Step**</DNT>, entrez le nom du programme `RSBTONEJOB2` et spécifiez la variante pour chaque tâche.
4. Cliquez sur <DNT>**Start Condition**</DNT>, définissez la date et l&apos;heure de début, cochez <DNT>**Periodic Job**</DNT>, cliquez sur <DNT>**Period Values**</DNT>, puis sur <DNT>**Other Period**</DNT> et entrez <DNT>**5**</DNT> pour les minutes.
