Cette référence fournit les détails de configuration réseau pour connecter vos systèmes SAP à New Relic. Utilisez ces spécifications lors de la configuration des destinations RFC et des règles de pare-feu.

## À qui s&apos;adresse cette référence

* Administrateurs SAP Basis configurant les connexions RFC
* Administrateurs réseau configurant des règles de pare-feu
* Équipes d&apos;installation préparant les exigences réseau

## Configuration de la destination RFC

Configurez ces destinations RFC (Remote Function Call) dans votre système SAP pour permettre la communication avec les API New Relic. Utilisez les points de terminaison pour la région où votre compte New Relic est hébergé.

**Remarque**: le type de connexion &quot;G&quot; indique une connexion HTTP vers un serveur externe.

Choisissez la configuration du point de terminaison en fonction de la région de votre compte New Relic :

* Utilisez **les points de terminaison de la région US** si votre compte est dans le data center des États-Unis
* Utilisez **les points de terminaison de la région UE** si votre compte est dans le data center de l&apos;UE

## Vérifier la connectivité

Après avoir configuré les destinations RFC, testez la connectivité à l&apos;aide du code de transaction SAP `SM59`:

1. Sélectionnez la destination RFC
2. Cliquez <DNT>**Connection Test**</DNT>
3. Vérifier la connexion réussie aux points de terminaison New Relic
