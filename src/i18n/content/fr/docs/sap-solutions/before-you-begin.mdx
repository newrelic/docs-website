Remplissez ces prérequis avant l&apos;installation :

## Liste de contrôle des prérequis

### Compte New Relic

* <DNT>**Account ID**</DNT>: [obtenez votre ID de compte](/docs/accounts/accounts-billing/account-setup/account-id)
* <DNT>**License Key**</DNT>: obtenir un <InlinePopover type="licenseKey" />New Relic pour l&apos;authentification et l&apos;accès aux données
* <DNT>**Data Center Region**</DNT>: [choisissez l&apos;emplacement de votre data center (États-Unis ou UE)](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) et configurez votre compte

### Exigences système SAP

* <DNT>**Software versions**</DNT>: Vérifiez que vos <DNT>**SAP systems**</DNT> répondent aux niveaux de sortie des composants logiciels requis. Consultez [Référence des fournisseurs de données](/docs/sap-solutions/monitor/data-providers-reference) pour les exigences spécifiques au système
* <DNT>**Monitored systems**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**Kernel 753 Patch 602+**</DNT>, <DNT>**ST-PI 740 SP16+**</DNT>
* <DNT>**Central monitoring system**</DNT>: <DNT>**SAP\_BASIS 740+**</DNT>, <DNT>**ST-PI 740 SP14+**</DNT>

### Accès au réseau

* <DNT>**HTTPS access**</DNT> vers <DNT>**New Relic endpoints**</DNT> (voir les [URL des points de terminaison](#network-endpoints))
* <DNT>**Firewall rules**</DNT> configuré
* <DNT>**Proxy settings**</DNT> configuré (si requis)

### Monitoring BTP (le cas échéant)

Si vous monitorez <DNT>**BTP accounts**</DNT>, assurez-vous de l&apos;un des éléments suivants :

* <DNT>**CloudALM account**</DNT> configuré pour <DNT>**BTP monitoring**</DNT>
* <DNT>**Process Integration Runtime**</DNT> abonnement au service (requis pour le monitoring de l&apos;intégration)

<Callout variant="important">
  Vous aurez peut-être besoin d&apos;une approbation ou d&apos;aide à la configuration de la part de vos équipes de sécurité ou réseau. Effectuez ces étapes avant l’installation et consultez la [liste de contrôle de préparation à l’installation](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness).
</Callout>

## Préparer les systèmes monitorés

L&apos;architecture sans agent utilise des connexions RFC pour récupérer des données.

### Exigences de version

Vérifier les versions logicielles requises :

* <DNT>**SAP\_BASIS**</DNT>: sortie <DNT>**740**</DNT> ou version ultérieure
* <DNT>**Kernel**</DNT>: sortie <DNT>**753 Patch 602**</DNT> ou version ultérieure
* <DNT>**ST-PI**</DNT>: sortie <DNT>**740 SP16**</DNT> ou version ultérieure

### Configuration utilisateur

* [Télécharger le rôle d&apos;autorisation](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* [Créez un utilisateur système](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) avec le rôle téléchargé pour <DNT>**RFC connections**</DNT>

### Configuration de la base de données HANA (le cas échéant)

Si vous effectuez le monitoring d&apos;un <DNT>**HANA database**</DNT>, créez un <DNT>**HANA database user**</DNT> `NRAGENT` avec un accès <DNT>**Catalog Read**</DNT> et <DNT>**Monitoring**</DNT> :

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

Collectez les <DNT>**HANA server host**</DNT> et <DNT>**tenant DB port**</DNT> pour <DNT>**SQL connection**</DNT>.

### Autres bases de données

Pour les autres bases de données, activez le <DNT>**CCMS database KPIs**</DNT> standard en exécutant le <DNT>**ABAP program**</DNT> `RSDBKPIMON`.

Si votre version n&apos;est pas listée, contactez votre responsable de compte New Relic pour un package supplémentaire.

### exigences système

* Au moins <DNT>**20 dialog work processes**</DNT> et <DNT>**10 background work processes**</DNT>
* <DNT>**HTTPS access**</DNT> sortant vers le <DNT>**New Relic data center**</DNT> (voir les URL des points de terminaison ci-dessus)

### Configuration réseau

* <DNT>**Proxy settings**</DNT> (si requis) :

  * Nom de domaine/adresse IP et port du serveur
  * Identifiants d&apos;accès au proxy (si l&apos;authentification est nécessaire)

* <DNT>**Firewall policies**</DNT>: Autoriser <DNT>**HTTPS traffic**</DNT> à <DNT>**New Relic endpoints**</DNT>

### Destinations RFC

Pour chaque système monitoré, utilisez la transaction `SM59` et l&apos;ID du système distant pour créer <DNT>**RFC destinations**</DNT> et tester les connexions. Voir [Créer des destinations RFC pour les systèmes monitorés](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems).

<Callout variant="important">
  Le <DNT>**RFC user**</DNT> doit avoir le rôle <DNT>**/NEWR/DATA\_PROVIDER**</DNT> attribué.
</Callout>

### Monitoring de la base de données HANA distante (le cas échéant)

Pour monitorer <DNT>**HANA databases**</DNT> à distance :

* Assurez-vous que le système de monitoring central a accès au <DNT>**HANA server**</DNT> et <DNT>**tenant DB port**</DNT>

* Pour les systèmes de monitoring centraux non-S4HANA, installez l&apos;un de ces composants :

  * <DNT>**HANA Client v2.0**</DNT> (compatible avec le <DNT>**HANA DB version**</DNT>)

  * <DNT>**HANA Client Lib file**</DNT>:

    * `libSQLDBCHDB.so` (Linux)
    * `libSQLDBCHDB.dll` (Windows)
    * Recherchez la bibliothèque dans le <DNT>**SAP Kernel folder**</DNT> et attribuez la propriété au <DNT>**SID Admin ID**</DNT> (par exemple, `sd1adm`)

* Pour chaque <DNT>**HANA database**</DNT> distant, utilisez la transaction `SAP DBACOCKPIT` pour créer un <DNT>**DB connection**</DNT>. Voir [Créer une connexion de base de données](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)

## Préparer le monitoring BTP (le cas échéant)

Pour monitorer un <DNT>**BTP account**</DNT>, vous avez besoin soit d&apos;un <DNT>**CloudALM instance**</DNT> configuré pour monitorer le <DNT>**BTP account**</DNT>, soit d&apos;un abonnement au service <DNT>**Process Integration Runtime**</DNT> (applicable uniquement au monitoring de l&apos;intégration). <DNT>**Service keys**</DNT> sont requis pour l&apos;authentification de la connexion. Consultez [Comment récupérer les clés de service](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).

Utilisez <DNT>**service keys**</DNT> pour créer des [destinations HTTP externes](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint) et comme <DNT>**API host**</DNT> pour <DNT>**API data providers**</DNT>.
