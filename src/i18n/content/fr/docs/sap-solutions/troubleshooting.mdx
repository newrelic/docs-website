## Dépanner la disponibilité du serveur

Les fournisseurs de données `NR_SAP_INSTANCE_AVAILABILITY` et `NR_SAP_SERVER_LIST` s&apos;appuient sur des appels SOAP aux services Web `SAPCONTROL`. Pour que cela fonctionne, le profil `DEFAULT.PFL` sur les serveurs de monitoring doit être configuré.

1. Confirmez que la ligne suivante existe dans le fichier `DEFAULT.PFL` :

`service/protectedwebmethods = SDEFAULT`

2. Si cette ligne est manquante, ajoutez les paramètres suivants :

`service/protectedwebmethods = SDEFAULT -GetQueueStatistic -ABAPGetWPTable -EnqGetStatistic -GetProcessList -GetEnvironment -GetVersionInfo -GetSystemInstanceList`

3. Redémarrez le service SAP avec les commandes `stopsap sapstartsrv` et `startsap sapstartsrv`.

4. Confirmez que la configuration fonctionne en exécutant la commande `sapcontrol -prot GSOAP_HTTP -nr 00 -function GetSystemInstanceList`.
