## Les fournisseurs de données de disponibilité du serveur ne fonctionnent pas

**Symptôme**: les fournisseurs de données `NR_SAP_INSTANCE_AVAILABILITY` et `NR_SAP_SERVER_LIST` ne parviennent pas à collecter des données.

**Cause**: le profil `DEFAULT.PFL` sur les serveurs monitorés n&apos;est pas configuré pour les services Web <DNT>**SAPCONTROL**</DNT>.

**Solution**:

1. Ouvrez le fichier `DEFAULT.PFL` et vérifiez si cette ligne existe :

`service/protectedwebmethods = SDEFAULT`

2. Si la ligne est manquante, ajoutez le paramètre suivant :

`service/protectedwebmethods = SDEFAULT -GetQueueStatistic -ABAPGetWPTable -EnqGetStatistic -GetProcessList -GetEnvironment -GetVersionInfo -GetSystemInstanceList`

3. Redémarrez le <DNT>**SAP service**</DNT>:

```yaml
stopsap sapstartsrv
startsap sapstartsrv
```

4. Vérifiez la configuration :

```yaml
sapcontrol -prot GSOAP_HTTP -nr 00 -function GetSystemInstanceList
```

**Résultat attendu**: la commande renvoie la liste des instances système sans erreur.
