## État d&apos;exécution et logde transmission

À partir du **Data Provider Registry** (transaction `/NEWR/REGISTRY`), vous pouvez vérifier le<img src="/images/runtimestatus.webp" class="inline" style={{height: "40px", width: "120px"}} alt="RuntimeStatus icon" title="RuntimeStatus" />

et<img src="/images/transmissionlog.webp" class="inline" style={{height: "40px", width: "120px"}} alt="transmissionlog" title="Transmissionlog.webp" />pour chaque fournisseur de données. Le log de transmission affiche l&apos;état et l&apos;heure de chaque transmission.

## Vérifier l&apos;état du travail et le spool

1. Utilisez la transaction `SM37` pour vérifier l’état des tâches planifiées. Saisissez `/NEWR/*` dans le champ **ABAP Program Name** et cliquez sur **Execute**.
2. Faites attention à la durée d’exécution ; un temps anormalement long indique un problème potentiel.
3. Vérifiez les tâches qui se chevauchent et ajustez le calendrier si nécessaire. Le spool de tâches fournit le résultat pour chaque fournisseur de données.

## Vérifier log d&apos;application

Les processus de collecte de données sont enregistrés dans le log d&apos;application (transaction SLG1). Requête avec l&apos;objet de log d&apos;application `/NEWR/DP`.

## dashboard monitoring de la collecte de données SAP

Les mesures de collecte de données sont également envoyées à New Relic pour l&apos; monitoring. Utilisez le dashboard [**de monitoring de la collecte de données SAP**]() prêt à l&apos;emploi pour monitorer les processus de collecte de données pour chaque fournisseur de données.
