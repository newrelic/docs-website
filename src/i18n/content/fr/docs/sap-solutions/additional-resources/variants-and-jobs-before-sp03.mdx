## Créer des variantes pour /NEWR/ABAP\_COLLECTION\_MANAGERN

1. Utilisez la transaction **SE38** pour configurer `/NEWR/ABAP_COLLECTION_MANAGERN`.
2. Sur l&apos; **ABAP: Variants - Initial Screen**, saisissez `/NEWR/ABAP_COLLECTION_MANAGERN` dans le champ **Program**.
3. Sélectionnez **Variant** et cliquez sur **Create**.
4. Suivez les étapes ci-dessous pour mettre à jour les valeurs :

<CollapserGroup>
  <Collapser id="trace-variant" title="trace de variante">
    1. Saisissez le nom de la variante, par exemple **TRACE\_ONLY**, puis cliquez sur **Create**.

    2. Assurez-vous que le **New Relic Account ID** est renseigné automatiquement.

    3. Définissez les **Records per transmission** sur **10 000**.

    4. Cliquez sur **Sélection multiple (active)** pour ajouter des noms de fournisseurs de données.

    5. Ajoutez les éléments suivants dans **Select Single Values**:

       * `NR.SAP.TRANSACTION`
       * `NR.SAP.TRANSACTION.COMPLETE`
       * `NR.SAP.SYSTEM.PROCESS`

    6. Sélectionnez le **Sequential Processing** comme **Execution Mode**.

    7. Si **DIA WP** ≥ 30, régler sur **6**; sinon, régler sur **3**.

    8. Cliquez sur les **Attributes**, ajoutez une description et cliquez sur **Save**.
  </Collapser>

  <Collapser id="idoc-variant" title="variante IDOC">
    1. Saisissez le nom de la variante, par exemple **IDOC\_ONLY**, puis cliquez sur **Create**.

    2. Assurez-vous que le **New Relic Account ID** est renseigné automatiquement.

    3. Définissez les **Records per transmission** sur **10 000**.

    4. Cliquez sur **Sélection multiple (active)** pour ajouter des noms de fournisseurs de données.

    5. Ajoutez les éléments suivants dans **Select Single Values**:

       * `NR_SAP_IDOC`
       * `NR_SAP_IDOC_COMPLETE`

    6. Sélectionnez le **Sequential Processing** comme **Execution Mode**.

    7. Cliquez sur les **Attributes**, ajoutez une description et cliquez sur **Save**.
  </Collapser>

  <Collapser id="system-variant" title="variante du système">
    1. Saisissez le nom de la variante, par exemple, **System**, et cliquez sur **Create**.
    2. Assurez-vous que le **New Relic Account ID** est renseigné automatiquement.
    3. Définissez les **Records per transmission** sur **10 000**.
    4. Cliquez sur **Sélection multiple (active)** pour ajouter des noms de fournisseurs de données.
    5. Ajoutez des variantes à partir des fournisseurs de **TRACE** et d&apos;**IDOC** dans **Exclude Single Values**.
    6. Sélectionnez **Parallel Processing** comme **Execution Mode**.
    7. Si **DIA WP** ≥ 30, régler sur **6**; sinon, régler sur **3**.
    8. Cliquez sur les **Attributes**, ajoutez une description et cliquez sur **Save**.
  </Collapser>
</CollapserGroup>

## Planifier des tâches de collecte

1. Utilisez la transaction **SM36** pour planifier trois tâches.
2. Saisissez le nom du travail pour chaque travail et définissez la **Job Class** sur « C ».
3. Cliquez sur **Step**, entrez le nom du programme **RSBTONEJOB2** et spécifiez la variante pour chaque tâche.
4. Cliquez sur **Start Condition**, définissez la date et l&apos;heure de début, cochez **Periodic Job**, cliquez sur **Period Values**, puis sur **Other Period** et entrez **5** pour les minutes.
