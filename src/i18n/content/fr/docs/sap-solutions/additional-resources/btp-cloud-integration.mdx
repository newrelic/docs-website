Configurez SAP BTP Integration Suite (intégration cloud) en tant que middleware pour transmettre les données de monitoring à New Relic lorsque la connectivité directe n&apos;est pas disponible.

## Présentation

Le programme Collection Manager fournit une option pour envoyer des données vers New Relic en utilisant SAP Cloud Integration (SAP CPI) comme middleware au lieu d&apos;une transmission directe.

| **Méthode**           | **Description**                                            |
| --------------------- | ---------------------------------------------------------- |
| Direct                | Transmission directe de SAP à New Relic (méthode standard) |
| SAP Cloud Integration | Utilisation de SAP CPI comme middleware                    |

## Étapes de configuration

<Steps>
  <Step>
    ## Déterminer la méthode d&apos;autorisation

    Sélectionnez le type d&apos;autorisation que vous utiliserez pour l&apos;iFlow :

    * <DNT>**ClientId/Secret**</DNT> (OAuth2)
    * <DNT>**Certificate**</DNT>
    * <DNT>**External Certificate**</DNT>
  </Step>

  <Step>
    ## Créer une instance de service et une clé

    1. Créez une instance de service pour <DNT>**SAP Process Integration Runtime**</DNT>.
    2. Attribuez un <DNT>**service key**</DNT> ou <DNT>**certificate**</DNT> en utilisant la méthode d&apos;autorisation sélectionnée.
    3. Téléchargez le <DNT>**service key**</DNT> pour une utilisation ultérieure.

    Pour des instructions détaillées, consultez [Obtenir la clé de service pour <DNT>**BTP Monitoring**</DNT>](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime).
  </Step>

  <Step>
    ## Configurer et déployer iFlow

    Utilisez le <DNT>**iFlow**</DNT> fourni par New Relic comme modèle :

    1. Copiez le modèle <DNT>**iFlow**</DNT>.
    2. Mettez à jour la méthode d&apos;autorisation pour correspondre à votre sélection de l&apos;étape 1.
    3. Assurez-vous que les modélisations suivantes sont configurées :

    ### En-têtes autorisés

    Configurez <DNT>**allowed headers**</DNT> pour contenir :

    ```
    API-Key|targetURL
    ```

    ### Méthode d&apos;autorisation

    * Pour <DNT>**ClientId/Secret**</DNT> ou <DNT>**Certificate**</DNT>: sélectionnez <DNT>**User Role**</DNT>
    * Pour <DNT>**External Certificate**</DNT>: sélectionnez <DNT>**Client Certificate**</DNT> et ajoutez <DNT>**root certificates**</DNT>

    ### Modificateur de contenu de requête

    Configurez <DNT>**exchange properties**</DNT> pour contenir :

    * <DNT>**targeturl**</DNT>
    * <DNT>**apikey**</DNT>

    ### Script Groovy

    Ajoutez un <DNT>**script**</DNT> pour définir dynamiquement les en-têtes :

    ```groovy
    import com.sap.gateway.ip.core.customdev.util.Message;
    def Message processData(Message message) {

      message.setHeader("API-Key", message.getProperty("apikey"));
      message.setHeader("Content-Type", "application/json");
      message.setHeader("Accept", "application/json");

      message.setHeader("CamelHttpUri", message.getProperty("targeturl"));

      def turl = message.getHeader("CamelHttpUri", String);

      def logText = "---Headers---\n";

      message.getHeaders().each { k, v -> logText += "\${k} = \${v}\n" };
      messageLogFactory.getMessageLog(message)?.addAttachmentAsString("AllHeaders", logText, "text/plain");

      messageLogFactory.getMessageLog(message)?.addAttachmentAsString("URL", turl, "text/plain");

      return message;
    }
    ```

    ### Response content modifier

    Configurez le <DNT>**message body**</DNT> pour renvoyer :

    ```json
    {
      "statusCode": "\${header.CamelHttpResponseCode}",
      "response": \${body}
    }
    ```

    ### Paramètres de connexion HTTP

    * <DNT>**Address**</DNT>: `\${header.CamelHttpUri}`
    * <DNT>**Method**</DNT>: <DNT>**POST**</DNT>
    * <DNT>**Timeout**</DNT>: <DNT>**60000 ms**</DNT>
    * <DNT>**Request Headers**</DNT>: `*`
    * <DNT>**Response Headers**</DNT>: `*`

    4. Déployez le <DNT>**iFlow**</DNT>.
    5. Notez le <DNT>**iFlow endpoint URL**</DNT>.
  </Step>

  <Step>
    ## Créer une destination RFC

    1. Utilisez la transaction <DNT>**SM59**</DNT> pour créer un nouveau <DNT>**RFC destination**</DNT>.

    2. Saisissez les informations suivantes :

       * <DNT>**RFC Destination**</DNT>: Un nom descriptif (par exemple, <DNT>**NR\_CPI\_ENDPOINT**</DNT>)
       * <DNT>**Connection Type**</DNT>: <DNT>**G**</DNT> (Connexion HTTP vers un serveur externe)
       * <DNT>**Description**</DNT>: connexion à New Relic via l&apos;intégration cloud

    3. Sur l&apos;onglet <DNT>**Technical Settings**</DNT> :

       * <DNT>**Target Host**</DNT>: l&apos;hôte de votre <DNT>**iFlow endpoint**</DNT>
       * <DNT>**Service No.**</DNT>: <DNT>**443**</DNT>
       * <DNT>**Path Prefix**</DNT>: le chemin depuis votre <DNT>**iFlow endpoint**</DNT>

    4. Sur l&apos;onglet <DNT>**Logon &amp; Security**</DNT> :

       * Configurez l&apos;authentification en fonction de la méthode sélectionnée :

         * Pour <DNT>**ClientId/Secret**</DNT>: entrez <DNT>**ClientId**</DNT> comme nom d&apos;utilisateur et <DNT>**ClientSecret**</DNT> comme mot de passe
         * Pour <DNT>**Certificate**</DNT>: configurez le <DNT>**certificate details**</DNT>

    5. Sur l&apos;onglet <DNT>**Security Options**</DNT> :
       * Sélectionnez <DNT>**Active**</DNT> pour <DNT>**SSL**</DNT>

    6. Enregistrez la configuration.

    7. Cliquez sur <DNT>**Connection Test**</DNT> pour vérifier la connexion.
  </Step>

  <Step>
    ## Configurer le gestionnaire de collecte

    1. Exécutez la transaction <DNT>**/NEWR/ONE\_COLLECTION**</DNT> ou créez un <DNT>**variant**</DNT> pour <DNT>**/NEWR/ABAP\_COLLECTION\_MANAGERN**</DNT>.
    2. Dans la section <DNT>**Data Transmission to NR**</DNT> :
       * Sélectionnez <DNT>**SAP Cloud Integration**</DNT> au lieu de <DNT>**Direct**</DNT>
    3. Dans le champ <DNT>**iFlow Endpoint**</DNT> :
       * Saisissez le <DNT>**RFC destination name**</DNT> créé à l&apos;étape 4 (par exemple, <DNT>**NR\_CPI\_ENDPOINT**</DNT>)
    4. Enregistrez le <DNT>**variant**</DNT>.
    5. Planifier ou exécuter le <DNT>**collection job**</DNT>.
  </Step>
</Steps>

## Vérifier la transmission de données

1. Exécutez le <DNT>**collection manager job**</DNT>.

2. Vérifiez le <DNT>**iFlow monitoring**</DNT> dans <DNT>**SAP Cloud Integration**</DNT>:

   * Connectez-vous à <DNT>**SAP BTP Integration Suite**</DNT>
   * Allez dans <DNT>**Monitor**</DNT> &gt; <DNT>**Integrations**</DNT>
   * Vérifier le traitement réussi des messages

3. Vérifiez l&apos;arrivée des données dans <DNT>**New Relic**</DNT>:

   * Connectez-vous à <DNT>**New Relic**</DNT>
   * Accédez à votre <DNT>**SAP dashboards**</DNT>
   * Vérifier que les données de monitoring sont reçues

<Callout variant="important">
  Si vous modifiez le <DNT>**iFlow template**</DNT> standard, assurez-vous que toutes les modélisations et tous les en-têtes requis sont préservés pour maintenir la compatibilité avec le <DNT>**collection manager program**</DNT>.
</Callout>
