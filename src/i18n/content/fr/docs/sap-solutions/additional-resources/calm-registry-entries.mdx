Configurez les entrées de registre (Call IDs) pour les fournisseurs de données CloudALM afin d&apos;activer le monitoring des services SAP Cloud ALM.

## Présentation

Ce programme utilitaire fournit la fonctionnalité permettant d&apos;ajouter, de modifier, de supprimer, et de générer automatiquement des configurations pour les fournisseurs de données CloudALM.

* <DNT>**Transaction Code**</DNT>: <DNT>**/NEWR/GEN\_DP\_CALLID**</DNT>
* <DNT>**Program Name**</DNT>: <DNT>**/NEWR/GENERATE\_CONFIG\_CALLID**</DNT>
* <DNT>**Configuration Table**</DNT>: <DNT>**/NEWR/CALLID\_CNF**</DNT>

## Prérequis

Assurez-vous que le fournisseur de données <DNT>**NR\_CALM\_LANDSCAPE**</DNT> est actif et collecte des données avant d&apos;exécuter cet utilitaire.

## Procédure d&apos;installation

<Steps>
  <Step>
    ## Configurer la modélisation du fournisseur de données et du type de service

    1. Exécutez la transaction `/NEWR/GEN_DP_CALLID`.
    2. Cliquez sur le <DNT>**DP/CALM Services Type Maintenance Link**</DNT>.
    3. Sélectionnez les types de services que vous souhaitez monitorer.
    4. Liez chaque fournisseur de données au système ou service SAP approprié configuré dans SAP Cloud ALM.
    5. Enregistrez la configuration.

    <Callout variant="tip">
      <DNT>**Service type**</DNT> représente <DNT>**SAP systems**</DNT> ou <DNT>**services**</DNT> activé dans votre environnement <DNT>**SAP Cloud ALM**</DNT>.
    </Callout>
  </Step>

  <Step>
    ## Ajouter de nouveaux enregistrements d&apos;ID d&apos;appel

    1. Dans l&apos;écran de sélection, saisissez le fournisseur de données et le type de service que vous devez configurer.
    2. Sélectionnez le bouton radio <DNT>**Add new (Call ID) records to Registry**</DNT>.
    3. Cliquez sur <DNT>**Execute**</DNT>.
    4. Le système affiche tous les <DNT>**service type instances**</DNT> configurés dans <DNT>**Cloud ALM**</DNT>, chacun avec un <DNT>**LMS ID**</DNT> unique.
    5. Sélectionnez les instances que vous souhaitez monitorer.
    6. Supprimez toutes les entrées inutiles.
    7. Cliquez sur <DNT>**Update and save configuration**</DNT>.
  </Step>

  <Step>
    ## Vérifiez la configuration

    1. Accédez à la transaction `/NEWR/REGISTRY`.
    2. Sélectionnez le <DNT>**data provider**</DNT>.
    3. Cliquez sur <DNT>**Configure**</DNT> pour afficher le <DNT>**Call ID entries**</DNT> enregistré.
    4. Cliquez sur <DNT>**Test**</DNT> pour vérifier la configuration.

    **Résultat attendu**: le test de connexion affiche un statut de réussite.
  </Step>
</Steps>

## Options de maintenance

### Mettre à jour les entrées existantes

Si vous devez modifier ou supprimer des <DNT>**Call ID records**</DNT> existants :

1. Exécutez la transaction `/NEWR/GEN_DP_CALLID`.
2. Sélectionnez <DNT>**Copy/Change/Delete existing (Call ID) records in Registry**</DNT>.
3. Cliquez sur <DNT>**Execute**</DNT>.
4. Effectuez les modifications requises.
5. Enregistrez la configuration.

### Effacer toutes les entrées

Pour supprimer tous les <DNT>**Call ID records**</DNT> (pour la configuration initiale ou les mises à niveau de Service Pack) :

1. Exécutez la transaction `/NEWR/GEN_DP_CALLID`.
2. Sélectionnez <DNT>**Backup &amp; delete all existing (Call ID) records in Registry**</DNT>.
3. Cliquez sur <DNT>**Execute**</DNT>.

<Callout variant="important">
  Cela crée une sauvegarde avant la suppression. Utilisez cette option uniquement lors de la configuration initiale ou lorsque vous y êtes invité lors d&apos;une mise à niveau de service pack.
</Callout>

### Actualiser les définitions des services

Si vous ne pouvez pas localiser un <DNT>**service type**</DNT> ou un <DNT>**LMS ID**</DNT>:

1. Cliquez sur <DNT>**Retrieve new services (LMSID) setup in Cloud ALM**</DNT>.
2. Cela récupère les derniers services configurés dans votre <DNT>**Cloud ALM account**</DNT>.
