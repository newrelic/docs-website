title: Depuis l'instance d'API CloudALM
metaDescription: Obtain service key from CloudALM API Instance
freshnessValidatedDate: never
translationType: machine
---

<Steps>
  <Step>
    Connectez-vous au **CALM subaccount**, puis cliquez sur **Instances and Subscriptions**.
  </Step>

  <Step>
    Cliquez sur **Create** en haut à droite.
  </Step>

  <Step>
    Dans l’écran contextuel, sélectionnez :

    * **Service**: API SAP Cloud ALM
    * **Formule**: Standard
    * **Environnement d&apos;exécution**: Cloud Foundry
    * **Espace**: Vérifiez auprès de votre administrateur BTP
    * **Nom de l&apos;instance**: suivez la convention de dénomination de votre organisation

    <img title="SAP Cloud ALM API" alt="A screenshot displaying the SAP Cloud ALM API" src="/images/sap-cloud-alm-api.webp" width="600" />
  </Step>

  <Step>
    Cliquez sur **Next**.
  </Step>

  <Step>
    Vérifiez et insérez les étendues requises à partir de [la gestion des étendues](https://help.sap.com/docs/cloud-alm/apis/managing-scopes) ainsi que le nom de l&apos;instance sous l&apos;onglet Paramètres.

    <img title="Manage Scopes" alt="A screenshot displaying the Manage Scopes" src="/images/manage-scope.webp" width="600" />
  </Step>

  <Step>
    Cliquez sur **Next** pour consulter les détails.

    <img title="Review Details" alt="A screenshot displaying the Review Details" src="/images/cloud-alm-new-instance.webp" width="600" />
  </Step>

  <Step>
    Cliquez sur **Create**.
  </Step>

  <Step>
    Une fois créé, localisez le service sous l’onglet **Instances**.
  </Step>

  <Step>
    Cliquez sur les points de suspension (`...`) à côté de votre instance, puis cliquez sur **Create Service Key**.
  </Step>

  <Step>
    Saisissez un Service Key Name et un **Key Type**: `ClientId/Secret`.
  </Step>

  <Step>
    Cliquez sur **Create**.
  </Step>

  <Step>
    Cliquez sur les points de suspension (`...`) à côté de votre clé de service et cliquez sur **Download**.
  </Step>
</Steps>
