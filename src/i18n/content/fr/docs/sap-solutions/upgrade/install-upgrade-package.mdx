Pour effectuer une mise à niveau depuis une version précédente de New Relic Monitoring for SAP Solutions, utilisez le package de mise à niveau qui correspond à votre version actuellement installée.

## Prérequis

* <DNT>**Administrator access**</DNT> à <DNT>**SAP system**</DNT>
* Version actuelle identifiée, voir la [section Identifier la version actuelle](#identify-current-version)
* Sauvegarde des configurations existantes
* <DNT>**upgrade package**</DNT> correct pour votre version installée

## Identifier la version actuelle [#identify-current-version]

1. Accédez à <DNT>**System &gt; Status**</DNT>.
2. Recherchez <DNT>**NEWR**</DNT> pour identifier le <DNT>**product version**</DNT>.
3. Dans <DNT>**client 000**</DNT>, exécutez la transaction `SAINT` pour mettre à jour le <DNT>**ABAP add-on**</DNT>.

## Sauvegarde et mise à jour de la configuration [#backup-and-update-configuration]

<Callout variant="tip">
  Enregistrez tout <DNT>**data provider configurations**</DNT> ou <DNT>**custom data providers**</DNT> modifié dans un <DNT>**configuration template**</DNT> pour éviter tout écrasement lors de la mise à niveau. Assurez-vous que les modèles sont sauvegardés avant de copier le nouveau modèle depuis <DNT>**client 000**</DNT>, même si le <DNT>**V3.0 ABAP Add-on**</DNT> n&apos;est pas installé.
</Callout>

1. Accédez au menu `/n/NEWR/INSTALL` et accédez à <DNT>**Update or Save Configuration Template**</DNT>.

2. Exécutez `/NEWR/REP_DELETE_INS_TEMPLATE` pour supprimer le <DNT>**HDB monitoring template**</DNT> lié à <DNT>**Java monitoring**</DNT>.

3. Configurez le <DNT>**ABAP data connection**</DNT> sur le <DNT>**HANA database**</DNT> en utilisant `DBACockpit`.

4. Copiez le <DNT>**configuration template**</DNT> depuis <DNT>**client 000**</DNT>.

5. Modifier <DNT>**monitoring system configuration**</DNT>:

   * Ajoutez le nouveau <DNT>**DB connection**</DNT>.
   * Supprimez les références à <DNT>**Java Enterprise Connector**</DNT> (par exemple, <DNT>**NR\_EI\_AGENT**</DNT>).
   * Ajoutez <DNT>**HANA\_DB**</DNT> à <DNT>**monitoring groups**</DNT>.
   * Régénérez <DNT>**data provider configurations**</DNT> pour mettre à jour les paramètres.
