Ceci décrit comment utiliser l'API REST New Relic (v2) pour obtenir <DNT>**crash count**</DNT> et <DNT>**crash rate**</DNT> globaux et spécifiques à la version de votre application mobile, qui apparaissent sur la [page<DNT>**Summary**</DNT> ](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/mobile-apps-overview-page)dans le coin supérieur droit.

Ces exemples utilisent la période par défaut des 30 dernières minutes. Pour obtenir des données de crash pour une [plage horaire](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2) différente, ajoutez la période aux commandes.

<Callout variant="tip">
  Vous pouvez également utiliser New Relic API l'explorateur pour récupérer des [données métriques mobiles](https://rpm.newrelic.com/api/explore/mobile_applications/metric_data).
</Callout>

## Prérequis [#prereqs]

Pour utiliser l'API dans ces exemples, vous avez besoin de :

* Votre [clé API REST](/docs/apis/rest-api-v2/requirements/rest-api-key)New Relic
* Votre New Relic monitoring identifiant d'application mobile des applications ou votre identifiant de monitoring version d'application mobile des applications.

Pour trouver l'ID de l'application monitoring des applications mobiles, consultez [Recherche de l'ID du produit : monitoring des applications mobiles](/docs/apis/rest-api-v2/requirements/finding-product-id#mobile).

Pour trouver l'ID de version de l'application mobile monitoring des applications, voir [Rechercher l'ID de version de l'application mobile](#mobile-app-version-id) ci-dessous.

## Application mobile : obtenir des données sur les accidents [#app-crash-data]

Pour obtenir les données sur le nombre de plantages et le taux de plantage pour l'ensemble application mobile, utilisez l'[identifiant d'application mobile](/docs/apis/rest-api-v2/requirements/finding-product-id#mobile) dans la commande API REST suivante :

```sh
curl -X GET "https://api.newrelic.com/v2/mobile_applications/$MOBILE_ID.json" \
     -H "X-Api-Key:$API_KEY" -i
```

Les données de sortie `crash_summary` contiennent à la fois `crash_count` et `crash_rate`.

```json
{
  "crash_summary": {
    "supports_crash_data": true,
    "unresolved_crash_count": 14,
    "crash_rate": 28.155339805825243
  }
}
```

Pour obtenir des données récapitulatives sur les plantages pour toutes les applications mobiles du compte, utilisez cette commande API REST :

```sh
curl -X GET "https://api.newrelic.com/v2/mobile_applications.json" \
     -H "X-Api-Key:$API_KEY" -i
```

## Version de l'application mobile : Obtenir des données sur le nombre d'accidents [#crash-count-version]

Pour obtenir les données métriques du nombre de plantages pour une version spécifique de l'application mobile, incluez l'[ID de la version application mobile](#mobile-app-version-id) dans la commande API REST suivante :

```sh
curl -X GET "https://api.newrelic.com/v2/mobile_applications/$MOBILE_APP_VERSION/metrics/data.json" \
     -H "X-Api-Key:$API_KEY" -i \
     -d 'name=Mobile/Crash/All&values[]=call_count&summarize=true'
```

## Version de l'application mobile : Obtenez les données du taux de plantation [#crash-rate-version]

Pour calculer le taux de plantage d'une version spécifique, utilisez l'équation suivante :

```
Crash Rate = (Mobile/Crash/All:call_count) / (Session/Start:call_count)
```

Pour obtenir les deux valeurs métriques nécessaires dans l'équation, utilisez la commande API REST suivante avec l'[ID de version application mobile](#mobile-app-version-id) .

```sh
curl -X GET "https://api.newrelic.com/v2/mobile_applications/$MOBILE_APP_VERSION/metrics/data.json" \
     -H "X-Api-Key:$API_KEY" -i \
     -d 'names[]=Mobile/Crash/All&names[]=Session/Start&values[]=call_count&summarize=true'
```

## Trouver l'ID de la version de l'application mobile [#mobile-app-version-id]

<Callout variant="important">
  Vous devez fournir l'ID de version uniquement lorsque vous souhaitez obtenir des données de plantage pour une version spécifique.
</Callout>

Pour trouver l'ID de version de votre application mobile, exécutez la requête NRQL suivante :

```sql
SELECT count(*) FROM Mobile
WHERE appName = 'YOUR_APP_NAME' FACET appVersionId 
SINCE 1 day ago
```
