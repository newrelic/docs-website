Pour les utilisateurs New Relic sur notre [modèle d&apos;utilisateur d&apos;origine](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), nous stockons une liste des utilisateurs qui peuvent accéder à votre compte dans une base de données par leur [adresse e-mail](/docs/accounts-partnerships/accounts/account-setup/adding-updating-users), [leur rôle](/docs/accounts-partnerships/accounts/account-setup/users-roles) attribué et leur prénom et nom s&amp;apos;ils sont fournis. Vous pouvez afficher ces données à partir de [l&apos;interface utilisateur](/docs/accounts-partnerships/accounts/account-setup/adding-updating-users) de New Relic et de l&amp;apos;[API Explorer (v2)](/docs/apm/apis/api-explorer-v2/parts-api-explorer).

<Callout variant="tip">
  Pour obtenir les mêmes informations à partir de [New Relic API Explorer (v2)](https://rpm.newrelic.com/api/explore), sélectionnez <DNT>**Users &gt; GET List**</DNT>.
</Callout>

## Exigences [#requirements]

Cela génère une liste de utilisateurs sur notre [modèle utilisateur d&apos;origine](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models). Il ne répertorie pas les utilisateurs sur notre [nouveau modèle utilisateur](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models).

## Listez tous les utilisateurs du compte [#list\_all\_users][#list_all_users]

Pour obtenir une liste de tous les utilisateurs sur le [modèle d&apos;utilisateur d&apos;origine](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) de votre compte, utilisez cette commande :

```sh
curl -X GET 'https://api.newrelic.com/v2/users.json' \
     -H "x-api-key:$API_KEY" -i
```

Le résultat ressemblera à ceci :

```json
HTTP/1.1 200 OK
Content-Type: application/json

{
  "users": [
    {
      "id": 123456,
      "first_name": "My",
      "last_name": "Name",
      "email": "my.name@mywebsite.com",
      "role": "owner"
    },
    {
      "id": 654321,
      "first_name": "Adam",
      "last_name": "Admin",
      "email": "adam.admin@mywebsite.com",
      "role": "admin"
    },
    {
      "id": 345123,
      "first_name": "Any",
      "last_name": "User",
      "email": "any.user@mywebsite.com",
      "role": "user"
    },
    ...
```

## Liste par email d&apos;utilisateur [#list\_by\_mail][#list_by_mail]

Si vous connaissez tout ou partie de l&apos;e-mail de l&apos;utilisateur, vous pouvez utiliser cette commande pour renvoyer le rôle, le nom et l&apos;utilisateur `id`. La clause `filter[email]=` spécifie la partie connue de l&amp;apos;e-mail (par exemple, `"my.name"`).

<DNT>**Note:**</DNT> La correspondance de caractères est une chaîne simple. Aucune capacité d&amp;apos;expressions régulières n&amp;apos;est disponible, donc plusieurs correspondances peuvent se produire si la chaîne sélectionnée n&amp;apos;est pas unique.

```sh
curl -X GET 'https://api.newrelic.com/v2/users.json' \
     -H "x-api-key:$API_KEY" -i \
     -d 'filter[email]=my.name'
```

La sortie de cette commande sera la même que la première entrée de l&apos;exemple [Liste de tous les comptes utilisateurs](#list_all_users) .

## Liste par utilisateur `id` [#list\_by\_userid][#list_by_userid]

Si vous connaissez l&apos;utilisateur `id`, vous pouvez utiliser cette commande pour renvoyer le rôle, le nom et l&amp;apos;e-mail. La clause `filter[ids]=` spécifie l&amp;apos;utilisateur `id`.

```sh
curl -X GET 'https://api.newrelic.com/v2/users.json' \
     -H "x-api-key:$API_KEY" -i \
     -d 'filter[ids]=123456'
```

Vous pouvez également utiliser cette commande, qui intègre l&apos;utilisateur `id` dans l&amp;apos;URL et omet le filtre.

```sh
curl -X GET 'https://api.newrelic.com/v2/users/123456.json' \
     -H "x-api-key:$API_KEY" -i
```

La sortie de cette commande sera la même que la première entrée de l&apos;exemple [Liste de tous les comptes utilisateurs](#list_all_users) .
