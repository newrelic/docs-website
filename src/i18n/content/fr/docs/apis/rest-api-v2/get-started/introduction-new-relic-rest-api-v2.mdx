L&apos;API REST de New Relic vous permet de récupérer des données et de les envoyer vers les outils de New Relic, ainsi que de configurer des fonctionnalités et d&apos;effectuer des opérations de suppression. Vous pouvez également utiliser l&apos;[API Explorer](/docs/apis/rest-api-v2/api-explorer-v2/getting-started-new-relics-api-explorer) pour comprendre les données à votre disposition via l&apos;API REST, pour obtenir la commande cURL et pour voir les réponses JSON.

## Installation

La structure de commande de l&apos;API REST suit ce modèle :

```
curl -X GET <a href="#appid"><var>URL</var></a> -H "Api-Key:<a href="#api_key"><var>$API_KEY</var></a>" -d '<a href="#payload"><var>PAYLOAD</var></a>'
```

La commande `GET` peut également être un `POST` ou `DELETE`, selon l&apos;intention de la requête.

Pour comprendre l’espace réservé, continuez à lire.

<Callout variant="tip">
  Nos exemples utilisent curl comme outil de ligne de commande commun pour extraire les données d&apos;intervalle de temps métrique de l&apos;API REST. Cependant, vous pouvez utiliser n&apos;importe quelle méthode pour effectuer vos requests REST. Les commandes `curl` incluent les URL cibles, les informations d&apos;en-tête et les données pertinentes pour tout mécanisme de demande.
</Callout>

## URL [#appid]

L&apos;appel d&apos;API nécessite une URL pour spécifier l&apos;emplacement à partir duquel les données seront accessibles. Vous devez remplacer l&apos;espace réservé `URL` par l&apos;URL appropriée qui changera en fonction du type de données demandées. En général, l&apos;URL suit ce modèle :

```
https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json
```

Le [$APP\_ID](/docs/apis/rest-api-v2/requirements/finding-product-id) spécifie l&apos;application ou le produit exact pour lequel les données sont demandées. Les informations suivant ce paramètre varieront en fonction de la demande de données.

Si vous avez un [compte dans la région UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center), l&apos;URL est :

```
api.eu.newrelic.com/v2/applications/$APP_ID/metrics/data.json
```

<Callout variant="tip">
  Vous pouvez récupérer des données XML au lieu de JSON en remplaçant `.json` par `.xml`.
</Callout>

## Clé API `$API_KEY` [#api\_key][#api_key]

L&apos;API New Relic nécessite une [clé API utilisateur](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key): la clé utilisateur est également utilisée pour notre [API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).

Dans nos exemples d&apos;API REST, nous empruntons l&apos;espace réservé à la clé API `$API_KEY` à la programmation shell Unix. Assurez-vous de remplacer cette variable et d’autres variables spécifiques à l’utilisateur lors de la formation d’appels.

## Détails de la requête (PAYLOAD) [#payload]

Le `PAYLOAD` contient les détails de la requête, qui définissent :

* Le nom de la métrique que vous souhaitez interroger et la valeur que vous souhaitez récupérer
* La [plage de temps](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2) définie pour la récupération des métriques
* (Facultatif) : La moyenne des données d&apos;intervalle de temps métrique en utilisant [le résumé](/docs/apis/rest-api-v2/requirements/calculating-average-metric-values-summarize)
