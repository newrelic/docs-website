Cette approche ajoute une nouvelle méthode d&apos;authentification à l&apos;[New Relic API REST (v2)](https://api.newrelic.com/) destinée aux clients qui permet aux partenaires (et aux clients à l&apos;échelle de l&apos;entreprise qui utilisent la structure de partenariat pour gérer leurs multiples comptes) d&apos;utiliser leurs informations d&apos;identification de partenariat pour autoriser tous les API appels REST (v2). Il n&apos;offre aucune nouvelle fonctionnalité pour les utilisateurs API non partenaires.

Lors de l&apos;appel d&apos;un point de terminaison dans New Relic REST API (v2) qui nécessite le <InlinePopover type="userKey" />, New Relic vous suggère d&apos;utiliser la [méthode d&apos;authentification décrite.](/docs/apis/rest-api-v2/requirements/admins-api-key-partnerships#Partners)

## En-têtes personnalisés et appel d&apos;API [#Partners]

Lorsque vous faites un appel d&apos;API, utilisez les en-têtes personnalisés suivants avec votre [clé APIpartenaire](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference#the-api-key), [votre ID partenaire](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference#partner-id) et votre [ID de compte](/docs/accounts-partnerships/accounts/account-setup/account-id):

* `X-Api-Key:YOUR_PARTNER_ACCOUNT_API_KEY`
* `NewRelic-Partner-Id:YOUR_PARTNER_ID`
* `NewRelic-Account-Id:YOUR_CUSTOMERS_ACCOUNT_ID`

Fournir une authentification à l&apos;API REST (v2) sous cette forme vous permet d&apos;effectuer n&apos;importe quelle opération API sur les comptes de vos clients. Cela contourne l&apos;exigence adressée aux clients selon laquelle certains appels d&apos;API nécessitent la clé API d&apos;un administrateur utilisateur.

Cette méthode d&apos;authentification fonctionne également pour les points de terminaison d&apos;API qui nécessitent uniquement la clé API REST. Cette nouvelle méthode d&apos;authentification fonctionne pour tous les points de terminaison, pas seulement ceux qui nécessitent la clé API de l&apos;administrateur de l&apos;utilisateur.

## Limitations connues

Cette méthode d&apos;authentification réservée aux partenaires ne fonctionnera qu&apos;avec l&apos;[API REST New Relic (v2)](https://api.newrelic.com/). Cela <DNT>**does not**</DNT> fonctionne avec les éléments suivants :

* [API de déploiement](/docs/apm/new-relic-apm/maintenance/deployment-notifications#api)
* [API d&apos;infrastructure pour les alertes](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/rest-api-calls-new-relic-infrastructure-alerts)
* [API partenaire](/docs/accounts-partnerships/partnerships/partner-api/partner-api-reference)
* [synthétique monitoring API](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-rest-api)
