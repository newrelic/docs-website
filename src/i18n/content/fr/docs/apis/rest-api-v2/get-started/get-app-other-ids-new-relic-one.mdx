Lorsque vous utilisez l'[New Relic API REST , vous devez remplacer certains ID d'espace réservé](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2) `$APP_ID` `$KEYTX_ID``$COMPONENT_ID`(comme,,, etc.) par l&amp;apos;ID d&amp;apos;une application, d&amp;apos;une transaction ou d&amp;apos;un autre composant.

Vous pouvez utiliser l'UI de New Relic pour obtenir certains de ces identifiants. Vous pouvez également obtenir une liste d'ID à l'aide de le [REST API Explorer](/docs/apis/rest-api-v2/get-started/list-application-id-host-id-instance-id).

## Rechercher des identifiants à l'aide de l'UI [#ui]

L'UI comprend les identifiants de l'application et d'autres composants dans un panneau métadonnée. Pour voir les métadonnées d'une entité, cliquez sur une entité monitorée dans la liste de l'explorateur d'entités, puis cliquez sur l'icône tag .

Voici quelques exemples :

<CollapserGroup>
  <Collapser id="apm" title="APM : ID d'application">
    Pour localiser le à `$APP_ID` partir de l&amp;apos;UI de New Relic :

    1. Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM**</DNT> et sélectionnez une application.

    2. Cliquez sur le <Icon name="fe-info" /> **<DNT>Metadata</DNT>** icône à côté du nom de l&amp;apos;application pour ouvrir le panneau de métadonnées.

    3. Localisez le <DNT>**App ID**</DNT>.

       Remplacez ce numéro par le `$APP_ID` dans votre appel d&amp;apos;API REST.
  </Collapser>

  <Collapser id="browser" title="ID de l'application du navigateur">
