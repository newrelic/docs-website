New Relic propose [plusieurs API](/docs/apis/getting-started/introduction-new-relic-apis), dont l&apos;[API REST New Relic](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2). Ce document vous présente le [REST API Explorer](https://api.newrelic.com), qui permet aux utilisateurs administrateurs et aux personnes disposant de la [clé API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) de :

* Parcourez les points de terminaison d&apos;API REST disponibles.
* Interagissez avec l&apos;API REST au sein d&apos;une interface utilisateur (l&apos;API Explorer).
* Consultez une source de documentation en direct.
* Obtenir la commande cURL pour les actions API .
* Partagez l&apos;appel d&apos;API configuré avec un collègue en copiant et en collant les URL d&apos;API Explorer.

Cela vous aide à rechercher rapidement des solutions et à tester votre appel d&apos;API avant de les ajouter à vos propres composants logiciels.

Les appels d&apos;API REST pour infrastructure <InlinePopover type="alerts" />ne sont pas disponibles dans l&apos;explorateur API .

## Caractéristiques

L&apos;explorateur d&apos;API de New Relic inclut une interface utilisateur interactive pour votre compte sélectionné. L&apos;UI de l&apos;explorateur API liste les types d&apos;appel d&apos;API<DNT>**Applications**</DNT> <DNT>**Users**</DNT>(,, etc.) et leurs fonctions disponibles, telles que <DNT>**GET metric data**</DNT>, <DNT>**PUT (update) applications**</DNT>, <DNT>**DELETE applications**</DNT>, etc.

Lorsque vous saisissez des valeurs pour <DNT>**Parameters**</DNT>, elles apparaissent automatiquement dans <DNT>**Request**</DNT> afin que vous puissiez tester et vérifier votre syntaxe avant d&apos;envoyer la demande. L&apos;UI indique les champs obligatoires, les descriptions des champs, leur type (entier, virgule flottante, booléen, etc.) et leur emplacement (chemin, requête, etc.).

Pour plus d&apos;informations sur les exigences API clé, consultez [ClésAPI REST](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key).

## Différences par rapport à la version 1 de l&apos;API [#v1]

Cet explorateur d&apos;API s&apos;applique uniquement à l&apos;[API REST New Relic version 2](/docs/apm/apis/requirements/new-relic-rest-api-v2-getting-started), qui se concentre sur les données entrantes et sortantes de New Relic. La version 2 remplace la version 1 de l&apos;API REST obsolète de New Relic.

Attention, il existe quelques différences entre la version 2 et 1 :

* Les noms des données peuvent être différents.
* Certaines commandes cURL pour v2 sont différentes de v1.

<Callout variant="tip">
  L&apos;agent New Relic utilise API différente et n&apos;est pas accessible via l&apos;explorateur API .
</Callout>
