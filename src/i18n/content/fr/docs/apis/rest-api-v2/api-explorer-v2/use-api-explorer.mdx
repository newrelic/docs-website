Le REST API Explorer (v2) de New Relic facilite les tests et l&apos;envoi requests pour tous les points de terminaison d&apos;API disponibles. Après avoir sélectionné votre choix de fonctions pour le type d&apos;appel d&apos;API (application, navigateur, etc.), l&apos;interface utilisateur fournit un formulaire interactif pour visualiser les exigences et tester les valeurs de vos paramètres. Il fournit également une source de documentation en direct sur les valeurs de l&apos;API.

## Exigences clés API [#api-key]

Avant de pouvoir utiliser l&apos;[API Explorer](https://rpm.newrelic.com/api/explore), l&amp;apos;accès à l&amp;apos;API doit être activé et une [clé API](/docs/apis/rest-api-v2/requirements/rest-api-key) doit être générée pour votre compte.

<Callout variant="important">
  Nous vous recommandons d&apos;utiliser la clé <InlinePopover type="userKey" />et non la [clé API REST](/docs/apis/intro-apis/new-relic-api-keys/#rest-api-key), car la clé utilisateur présente moins de restrictions.
</Callout>

Conseils:

* Si vous êtes connecté à New Relic, lorsque vous utilisez l&apos;explorateur API , vous pouvez choisir votre [clé API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) en haut de l&amp;apos;UI et cette clé apparaîtra automatiquement dans les sections <DNT>**Request**</DNT> et <DNT>**Parameters**</DNT> de l&amp;apos;explorateur.
* Si vous n&apos;êtes pas connecté à New Relic, vous pouvez coller votre [clé API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key) dans le <DNT>**Parameters**</DNT> de l&amp;apos;explorateur d&amp;apos;API.

## Utiliser l&apos;explorateur d&apos;API [#use-api-explorer]

1. Allez au <DNT>**[API Explorer](https://api.newrelic.com/docs/)**</DNT>.
2. Dans la liste déroulante <DNT>**Servers**</DNT> , choisissez une URL d’API basée aux États-Unis ou dans l’UE.
3. Cliquez sur <DNT>**Authorize**</DNT>, entrez votre clé API utilisateur et cliquez à nouveau sur <DNT>**Authorize**</DNT> .
4. Développez l’une des fonctions API disponibles : <DNT>**GET**</DNT>, <DNT>**PUT**</DNT>, <DNT>**DELETE**</DNT>.
5. (Facultatif) Ajoutez des valeurs <DNT>**Parameters**</DNT> pour votre appel d&amp;apos;API afin de filtrer la réponse (reportez-vous à l&amp;apos;UI pour les descriptions et les exigences de la v2).
6. Dans la liste déroulante <DNT>**Media type**</DNT> , choisissez un format pour votre demande : JSON ou XML.
7. Cliquez sur <DNT>**Try it out**</DNT>, puis sur <DNT>**Execute**</DNT>.
