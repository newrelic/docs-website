Lorsque vous utilisez le REST API Explorer (v2) de New Relic pour obtenir des [données d&apos;intervalle de temps métrique](/docs/telemetry-data-platform/ingest-manage-data/understand-data/new-relic-data-types#timeslice-data) pour votre application, vous avez besoin :

* Une [clé API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)
* L&apos;[ID de l&apos;application](#app_id)
* Le type de données d&apos;intervalle de temps métrique disponibles pour votre application, y compris les [noms et valeurs métriques](#view_metrics)

## Afficher l&apos;ID de votre application [#app\_id][#app_id]

Pour afficher [l&apos;ID de votre application](/docs/apm/apis/requirements/identification-code):

1. Dans l’ [explorateur d’API](https://rpm.newrelic.com/api/explore "Le lien s'ouvre dans une nouvelle fenêtre") REST New Relic, sélectionnez <DNT>**Applications &gt; GET List**</DNT>.
2. Si vous n&apos;êtes pas connecté à New Relic, fournissez une [clé API](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing) pour votre application.
3. Facultatif : à partir de <DNT>**Applications &gt; List**</DNT>, renseignez les valeurs pour les filtres `name`, `ids` ou `language` .
4. Sélectionnez <DNT>**Send Request**</DNT>.
5. À partir de la section <DNT>**Response**</DNT> , copiez le `id` de l&amp;apos;application.

Continuez avec la procédure pour afficher [les noms métriques](#view_metrics).

## Afficher les noms de métriques pour votre application [#view\_metrics][#view_metrics]

Pour afficher les données d&apos;intervalle de temps métrique disponibles pour votre application :

1. Dans l’explorateur d’API REST New Relic, sélectionnez <DNT>**Applications &gt; GET Metric Names**</DNT>.
2. Depuis <DNT>**Applications &gt; Metric Names**</DNT>, saisissez ou collez votre [identifiant d&apos;application](/docs/apm/apis/requirements/identification-code).
3. Remplissez une métrique spécifique ou partielle `name`, ou laissez vide pour afficher la liste des noms et valeurs de métriques disponibles.
4. Sélectionnez <DNT>**Send Request**</DNT>.
5. Dans la section <DNT>**Response**</DNT> , consultez les `metrics` disponibles pour l&amp;apos;application.

Continuez avec la procédure pour obtenir [des données](#app_metric_data) de tranches de temps [métriques](#app_metric_data).

## Obtenez des données d&apos;intervalle de temps métrique pour votre application [#app\_metric\_data][#app_metric_data]

Pour obtenir les valeurs métriques de votre application, copiez les valeurs des résultats <DNT>**Metric Names**</DNT> et collez-les dans les champs <DNT>**Metric Timeslice Data**</DNT> :

1. Dans l’explorateur d’API REST New Relic, sélectionnez <DNT>**Applications &gt; GET Metric Data**</DNT>.
2. Depuis <DNT>**Applications &gt; Metric Data**</DNT>, saisissez ou collez votre [identifiant d&apos;application](/docs/apm/apis/requirements/identification-code).
3. Tapez ou collez un ou plusieurs `names` (parmi <DNT>**GET Metric Names**</DNT>) pour votre application. Commencez chaque nom sur une nouvelle ligne.
4. Sélectionnez <DNT>**Send Request**</DNT>.
5. Dans la section <DNT>**Response**</DNT> , consultez les `metric_data` disponibles pour l&amp;apos;application.

Après avoir examiné la section <DNT>**Response**</DNT> , vous pouvez continuer à effectuer des ajustements et sélectionner à nouveau <DNT>**Send Request**</DNT> . Par exemple:

* Pour spécifier une [plage de temps](/docs/apm/apis/api-v2-examples/specifying-time-range-api-v2) pour les données d&amp;apos;intervalle de temps métrique, remplissez les valeurs `from` et `to` .
* Pour créer une [moyenne](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) des données sur la plage de temps, définissez `summarize` sur `true`. Ou, pour conserver les données de chaque période pour la plage que vous avez spécifiée, laissez `summarize` vide.

## Afficher d&apos;autres données de l&apos;API REST [#synthetics]

Pour utiliser l’explorateur d’API avec d’autres fonctionnalités New Relic, sélectionnez la clé API correspondante. Cela peut inclure :

* Monitoring synthétique
* Partenaires
