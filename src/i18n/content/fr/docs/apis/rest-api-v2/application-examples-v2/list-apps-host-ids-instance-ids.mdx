Voici quelques exemples d'utilisation de New Relic REST API (v2) pour obtenir la liste des [ID d'instance](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_instance_id) et [des ID hôte](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_host_id) d&amp;apos;une application. La liste montre tous ceux qui ont signalé au cours de la dernière heure environ.

## Répertorier les identifiants actuels par nom d'application [#list\_current\_ids][#list_current_ids]

Vous pouvez également utiliser [<DNT>**Applications &gt; List**</DNT>de l&amp;apos;explorateur d&amp;apos;API REST](/docs/apis/using-the-api-explorer) pour répertorier les détails de l&amp;apos;ID. Entrez le nom d&amp;apos;application correct.

Pour répertorier tous les ID [d'instance](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_instance_id) et [d'hôte](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_host_id) signalés au cours de la dernière heure avec un nom d&amp;apos;application contenant `${APP_NAME}`, utilisez la commande suivante. Cela fournit les [données récapitulatives](/docs/apis/rest-api-v2/application-examples-v2/summary-data-examples-v2#app) actuelles suivies des listes d&amp;apos;ID par exemple et des hôtes pour l&amp;apos;application avec `${APP_NAME}` dans leur nom, dans le compte associé au `${API_KEY}`.

```bash
curl -X GET 'https://api.newrelic.com/v2/applications.json' \
     -H "X-Api-Key:${API_KEY}" -i  \
     -d "filter[name]=${APP_NAME}"
```

## Liste des identifiants actuels par identifiant d'application [#ids\_app\_id\_list][#ids_app_id_list]

Vous pouvez également utiliser <DNT>**Applications &gt; List**</DNT>ou <DNT>**Applications &gt; List**</DNT> [l'explorateur d'API REST](/docs/apis/using-the-api-explorer) pour répertorier les détails de l&amp;apos;ID. Entrez le nom d&amp;apos;application correct.

Pour répertorier tous les identifiants signalés au cours de la dernière heure lorsque vous connaissez le [`${APP_ID}`de l&amp;apos;application](/docs/apis/rest-api-v2/application-examples-v2/listing-your-app-id-metric-data-v2), utilisez cette commande :

```bash
curl -X GET "https://api.newrelic.com/v2/applications/${APP_ID}.json" \
     -H "X-Api-Key:${API_KEY}" -i
```

Vous pouvez également utiliser `filter[id]`:

```bash
curl -X GET "https://api.newrelic.com/v2/applications.json" \
     -H "X-Api-Key:${API_KEY}" -i  \
     -d "filter[id]=${APP_ID}"
```

## Lister les identifiants actuels de toutes les applications [#ids\_all\_apps][#ids_all_apps]

Vous pouvez également utiliser <DNT>**Applications &gt; List**</DNT>ou <DNT>**Applications &gt; List**</DNT> [l'explorateur d'API REST](/docs/apis/using-the-api-explorer) pour répertorier les détails de l&amp;apos;ID. Entrez le nom d&amp;apos;application correct.

Pour toutes les applications, listez tous les [ID d'instance](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_instance_id) et [ID hôte](/docs/apis/rest-api-v2/requirements/listing-host-instance-application-server-ids#locating_host_id) signalés dans la dernière heure en utilisant la commande suivante. Cela fournit les [données récapitulatives](/docs/apis/rest-api-v2/application-examples-v2/summary-data-examples-v2#app) actuelles suivies des listes d&amp;apos;ID par exemple et hôtes pour toutes les applications du compte associé au `${API_KEY}`.

```bash
curl -X GET "https://api.newrelic.com/v2/applications.json" \
     -H "X-Api-Key:${API_KEY}" -i
```
