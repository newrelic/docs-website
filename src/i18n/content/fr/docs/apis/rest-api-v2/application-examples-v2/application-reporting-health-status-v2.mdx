Pour connaître l’état [de santé](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status) et l’état de création de rapports d’une application, utilisez les [données récapitulatives](/docs/apm/apis/api-v2-examples/summary-data-examples) disponibles à partir de l’API REST New Relic.

## Statut du rapport [#reporting\_status][#reporting_status]

L'état du rapport est fourni par la ligne `"reporting": true,` renvoyée dans le cadre de l&amp;apos;état de l&amp;apos;application. Ce statut passera à `false`si l&amp;apos;application n&amp;apos;a pas envoyé de rapport depuis 10 minutes. De plus, lorsque le statut est `false`, le `"last_reported_at":<time stamp>` ne contiendra plus d’horodatage.

## état de santé [#health\_status][#health_status]

L'[état de santé](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status) est fourni par la ligne `"health_status": "green",` renvoyée dans le cadre de l&amp;apos;état de l&amp;apos;application. La couleur renvoyée dans l&amp;apos;API correspond aux indicateurs d&amp;apos;état de santé de l&amp;apos;application affichés dans l&amp;apos;UI.

<DNT>**Exception:**</DNT> L&amp;apos;API REST utilise `orange` au lieu de `yellow`. Si l&amp;apos;application n&amp;apos;est pas associée à une règle d&amp;apos;alerte, elle s&amp;apos;affichera sous la forme `unknown`.
