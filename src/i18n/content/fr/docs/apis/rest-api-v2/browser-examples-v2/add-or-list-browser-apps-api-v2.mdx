Voici quelques exemples d&apos;utilisation de l&apos;API REST New Relic (v2) pour ajouter des applications à [<InlinePopover type="browser" />](/docs/browser/new-relic-browser/welcome-new-relic-browser/new-relic-browser)ou pour obtenir une liste de vos applications de navigateur pour une [clé API](/docs/apis/rest-api-v2/requirements/api-keys) spécifique. Cela vous aide à gérer le déploiement en dehors de New Relic. Ces appels d&amp;apos;API sont utiles, par exemple, aux grandes organisations déployant plusieurs applications, ou aux partenaires d&amp;apos;intégration qui facilitent la création de comptes New Relic et monitoring du déploiement des navigateurs.

<Callout variant="important">
  Lorsque vous ajoutez une application de navigateur via API (v2), vous ne pouvez instrument que le temps de chargement de base des pages. Pour utiliser instrumentation prenant en charge toutes les fonctionnalités SPA, [configurez monitoring des applications sur une seule page via une autre méthode](/docs/browser/single-page-app-monitoring/get-started/install-single-page-app-monitoring/).
</Callout>

## Ajouter des applications de navigateur

Pour ajouter une application à New Relic, remplacez `$API_KEY` par votre [clé API New Relic](/docs/apis/rest-api-v2/requirements/api-keys#rest-api-key) et remplacez `${STRING}` par le nom de l&amp;apos;application dans la commande suivante. Pour accomplir la même tâche à partir de l&amp;apos;explorateur d&amp;apos;API, utilisez votre clé API et accédez à <DNT>**[rpm.newrelic.com/api/explore](https://rpm.newrelic.com/api/explore) &amp;gt; Browser applications &amp;gt; POST create**</DNT>.

Utilisez la commande suivante :

```sh
curl -X POST 'https://api.newrelic.com/v2/browser_applications.json' \
     -H "X-Api-Key:$API_KEY" -i \
     -H 'Content-Type: application/json' \
     -d \
'{
  "browser_application": {
    "name": ${STRING}
  }
}'
```

L&apos;API renvoie un éventail de données où l&apos;élément est une application de navigateur et les données qui lui sont associées :

```json
{
  "browser_application": {
    "id": "integer",
    "name": "string",
    "browser_monitoring_key": "string",
    "loader_script": "string"
  }
}
```

## Lister toutes les applications de navigateur [#list-browser-apps]

Pour afficher une liste de vos applications de monitoring de navigateur, remplacez `$API_KEY` par votre [New Relic clé API](/docs/apis/rest-api-v2/requirements/api-keys#rest-api-key) dans la commande suivante. Pour accomplir la même tâche à partir de l&amp;apos;explorateur d&amp;apos;API, utilisez votre clé API et accédez à <DNT>**[rpm.newrelic.com/api/explore](https://rpm.newrelic.com/api/explore) &amp;gt; Browser Applications &amp;gt; GET List**</DNT>.

Utilisez la commande suivante :

```sh
curl -X GET 'https://api.newrelic.com/v2/browser_applications.json' \
     -H "X-Api-Key:$API_KEY" -i
```

Vous pouvez utiliser les résultats pour vérifier le compte ou le nom et pour obtenir une copie du script de chargement de l&apos;application, si nécessaire.

## Afficher des applications de navigateur spécifiques [#specific-browser-app]

<DNT>
  **View by name:**
</DNT>

Pour afficher une application de navigateur spécifique si vous connaissez son nom, remplacez `$API_KEY` par votre [clé API New Relic](/docs/apis/rest-api-v2/requirements/api-keys#rest-api-key) et remplacez `$NAME` par le nom de votre application dans la commande suivante :

```sh
curl -X GET 'https://api.newrelic.com/v2/browser_applications.json' \
     -H "X-Api-Key:$API_KEY" -i \
     -d "filter[name]=$NAME"
```

<DNT>
  **View by browser application ID:**
</DNT>

Pour afficher une application de navigateur spécifique si vous connaissez son ID, remplacez `$API_KEY` par votre [New Relic clé API](/docs/apis/rest-api-v2/requirements/api-keys#rest-api-key) et remplacez `$APP_ID` par votre identifiant d&amp;apos;application de navigateur dans la commande suivante :

```sh
curl -X GET 'https://api.newrelic.com/v2/browser_applications.json' \
     -H "X-Api-Key:$API_KEY" -i \
     -d 'filter[ids]=$APP_ID'
```

<DNT>
  **View multiple browser apps:**
</DNT>

Pour obtenir des informations sur plusieurs applications, séparez les valeurs de nom ou d&apos;ID par une virgule dans ces commandes ; par exemple :

```sh
-d 'filter[ids]=12345,23456'
```
