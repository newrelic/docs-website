Le [temps moyen de chargement de page](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) (ou temps de réponse) du navigateur apparaît en haut à droite du graphique principal de votre [page<DNT>**Summary**</DNT> ](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance#viewing). Cette valeur est étiquetée <DNT>**Browser**</DNT> et indique le temps moyen de chargement de la page en secondes.

## Temps moyen de chargement de la page [#h2-average]

<Callout variant="important">
  Bien que les exemples utilisent l&apos;API REST v2 de New Relic, nous recommandons d&apos;utiliser [les fonctions NRQL](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/) pour exécuter la requête d&apos;intervalle de temps métrique. Chaque valeur d’API peut être mappée à une fonction NRQL équivalente. Pour savoir comment créer une requête NRQL basée sur ces exemples d&apos;API, reportez-vous à notre [documentation](/docs/apis/rest-api-v2/migrate-to-nrql/).
</Callout>

Pour obtenir la [moyenne](/docs/apm/apis/requirements/calculating-average-metric-values-summarize) de la [plage horaire](/docs/apm/apis/requirements/specifying-time-range-api-v2) (la valeur par défaut est les 30 dernières minutes), utilisez la commande suivante. Assurez-vous de remplacer les variables `$APP_ID` et `$API_KEY` dans cet exemple par votre [identifiant d&apos;application](/docs/apm/apis/requirements/identification-code) spécifique et [votre cléAPI REST](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing).

```sh
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json" \ 
     -H "X-Api-Key:$API_KEY" -i \ 
     -d 'names[]=EndUser&values[]=average_response_time&summarize=true'
```
