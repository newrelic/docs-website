New Relic Teams vous permet de connecter l&apos;entité que vous monitorez aux équipes qui en sont propriétaires. Lorsque vous pouvez facilement voir à qui appartient une entité, vous pouvez accélérer le dépannage, améliorer la collaboration au sein de l’équipe et améliorer votre efficacité opérationnelle.

## Accès par défaut aux équipes [#default-access]

New Relic fournit un accès par défaut aux équipes via ces rôles standard :

Vous pourriez avoir besoin d&apos;un rôle personnalisé si les permissions par défaut ne répondent pas aux besoins de votre organisation.

## Prérequis [#prerequisites]

Avant de commencer, assurez-vous d’avoir :

* [Exigences générales de NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#authentication)
* Administrateur d&apos;organisation avec le rôle `Authentication Domain Manager` pour créer des rôles personnalisés

<Callout variant="tip">
  Vous pouvez également créer un rôle personnalisé pour Teams via l&apos;interface utilisateur. Pour les autorisations à sélectionner et le périmètre à utiliser, consultez [Capacités des équipes](/docs/service-architecture-intelligence/teams/teams/#teams-capabilities).
</Callout>

Pour créer un rôle personnalisé via l&apos;API NerdGraph, effectuez les étapes suivantes :

<Steps>
  <Step>
    ## Trouvez les identifiants d&apos;autorisation requis pour Teams

    Utilisez la requête `customerAdministration` pour récupérer une liste de capacités, leurs ID d&apos;autorisation et les informations associées. Filtrer par `scope: "organization"` pour obtenir les permissions Teams.

    #### Paramètres d&apos;entrée

    <table>
      <thead>
        <tr>
          <th>
            paramètres
          </th>

          <th>
            Type de données
          </th>

          <th>
            Est-ce obligatoire ?
          </th>

          <th>
            Description
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `eq`
          </td>

          <td>
            Chaîne
          </td>

          <td>
            Oui
          </td>

          <td>
            Définissez la valeur sur 

            `organization`

             pour récupérer les autorisations pour Teams.
          </td>
        </tr>
      </tbody>
    </table>

    #### Demande d&apos;échantillon

    ```graphql
    {
      customerAdministration {
        permissions(filter: {scope: {eq: "organization"}}) {
          items {
            feature
            category
            id
            product
          }
        }
      }
    }
    ```

    #### Exemple de réponse

    ```json
    {
      "data": {
        "customerAdministration": {
          "permissions": {
            "items": [
              {
                "category": "READ",
                "feature": "Teams",
                "id": "xxxxx",
                "product": "New Relic One"
              },
              {
                "category": "OTHER",
                "feature": "Teams",
                "id": "xxxxxx",
                "product": "New Relic One"
              },
              {
                "category": "DELETE",
                "feature": "Teams",
                "id": "xxxxx",
                "product": "New Relic One"
              },
              {
                "category": "MODIFY",
                "feature": "Teams",
                "id": "xxxxxx",
                "product": "New Relic One"
              }
            ]
          }
        }
      }
    }
    ```

    À partir de la réponse, identifiez et copiez les ID d&apos;autorisation où `feature` est `"Teams"` et où le `category` correspond au niveau d&apos;accès que vous souhaitez accorder. Vous aurez besoin de ces ID à l&apos;étape suivante.
  </Step>

  <Step>
    ## Récupérez votre identifiant d&apos;organisation

    Récupérez l&apos;ID de votre organisation, que vous utiliserez dans les mutations ultérieures.

    #### Demande d&apos;échantillon

    ```graphql
    {
      actor {
        organization {
          id
        }
      }
    }
    ```

    #### Exemple de réponse

    ```json
    {
      "data": {
        "actor": {
          "organization": {
            "id": "YOUR_ORGANIZATION_ID"
          }
        }
      }
    }
    ```

    Copiez votre ID d&apos;organisation à partir de la réponse. Vous en aurez besoin pour créer le rôle personnalisé.
  </Step>

  <Step>
    ## Ajouter le rôle personnalisé à un groupe d&apos;utilisateurs

    Après avoir créé le rôle personnalisé, attribuez-le à un groupe d&apos;utilisateurs dans New Relic.

    ### Récupérer les ID de groupe

    Utilisez la requête `customerAdministration` pour obtenir une liste des groupes d&apos;utilisateurs disponibles.

    #### Paramètres d&apos;entrée

    <table>
      <thead>
        <tr>
          <th>

          </th>

          </th>

      <tbody>
        <tr>
          <td>
            `id`
          </td>

          <td>
            Chaîne
          </td>

          <td>
            Oui
          </td>

          <td>
            L&apos;ID de l&apos;organisation de l&apos;étape précédente.
          </td>
        </tr>
      </tbody>
    </table>

    #### Demande d&apos;échantillon

    ```graphql
    {
      customerAdministration {
        groups(
          filter: {
            organizationId: { eq: "YOUR_ORGANIZATION_ID" }
          }
        ) {
          nextCursor
          items {
            id
            name
            users {
              items {
                id
                email
              }
            }
          }
        }
      }
    }
    ```

    À partir de la réponse, copiez l&apos;ID du groupe auquel vous souhaitez attribuer le rôle Teams.

    ### Attribuer le rôle au groupe

    Utilisez la mutation `authorizationManagementGrantAccess` pour attribuer le rôle personnalisé à un groupe d’utilisateurs.

          <td>

          <td>

          </td>

          <td>
            Oui
          </td>

          <td>
            L’ID de rôle personnalisé de l’étape précédente.
          </td>
        </tr>

        <tr>
          <td>
            `groupId`
          </td>

          <td>
            Chaîne
          </td>

          <td>

          </td>
        </tr>
      </tbody>
    </table>

    #### Demande d&apos;échantillon

    ```graphql
    mutation {
      authorizationManagementGrantAccess(
        grantAccessOptions: {
          organizationAccessGrants: { roleId: "YOUR_ROLE_ID" }
          groupId: "YOUR_GROUP_ID"
        }
      ) {
        roles {
          id
          name
          organizationId
          roleId
          groupId
          displayName
        }
      }
    }
    ```

    #### Exemple de réponse

    ```json
    {
      "data": {
        "authorizationManagementGrantAccess": {
          "roles": [
            {
              "displayName": "Teams manager",
              "groupId": null,
              "id": "99999999",
              "name": "teams_manager",
              "organizationId": "YOUR_ORGANIZATION_ID",
              "roleId": 99999
            }
          ]
        }
      }
    }
    ```

## Gérer les rôles personnalisés existants [#manage-roles]

<CollapserGroup>
  <Collapser id="update-role" title="Mettre à jour un rôle">
    Utilisez la mutation `customRoleUpdate` pour renommer un rôle ou remplacer son ensemble d&apos;autorisations.

    ```graphql
    mutation {
      customRoleUpdate(
        id: ROLE_ID
        name: "Updated role name"
        permissionIds: [xxxxx, xxxxx]
      ) {
        id
      }
    }
    ```

    <table>
      <thead>
        <tr>
          <th>
            paramètres
          </th>

          <th>
            Description
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `id`
          </td>

          <td>
            L&apos;ID du rôle à mettre à jour.
          </td>
        </tr>

        <tr>
          <td>
            `name`
          </td>

          <td>
            Nouveau nom d&apos;affichage pour le rôle.
          </td>
        </tr>

        <tr>
          <td>
            `permissionIds`
          </td>

          <td>
            Liste complète de remplacement des ID de permission. Cela remplace toutes les permissions existantes sur le rôle.
          </td>
        </tr>
      </tbody>
    </table>
  </Collapser>

  <Collapser id="delete-role" title="Supprimer un rôle">
    Utilisez la mutation `customRoleDelete` pour supprimer définitivement un rôle personnalisé.

    ```graphql
    mutation {
      customRoleDelete(id: ROLE_ID) {
        id
      }
    }
    ```

    La réponse renvoie l&apos;ID du rôle supprimé, confirmant la réussite de la suppression.
  </Collapser>

  <Collapser id="find-role-id" title="Trouver l'ID d'un rôle personnalisé">
    Utilisez la requête suivante pour rechercher un ID de rôle en parcourant les groupes de votre organisation et leurs rôles attribués.

    ```graphql
    {
      actor {
        organization {
          authorizationManagement {
            authenticationDomains(id: "YOUR_AUTHENTICATION_DOMAIN_ID") {
              authenticationDomains {
                groups {
                  groups {
                    displayName
                    id
                    roles {
                      roles {
                        roleId
                        name
                      }
                    }
                  }
                }
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>
