Vous pouvez exporter vos <InlinePopover type="dashboards" />sous forme de fichiers PDF ou PNG par programmation à l&apos;aide de la mutation NerdGraph `dashboardCreateSnapshotUrl`. Vous pouvez également [exporter le dashboard à l&apos;aide de l&apos;UI](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/#export-dashboards-pdf).

Pour une documentation complète de la mutation, de tous les paramètres disponibles et des exemples — y compris les remplacements de variables, les dimensions personnalisées et l’exportation d’images de graphiques par widget — consultez [Exporter des dashboards et des graphiques via l’API NerdGraph](/docs/query-your-data/explore-query-data/dashboards/dashboards-charts-import-export-data/#export-api).

## Dépannage [#troubleshooting]

Voici ce qu&apos;il faut faire si vous rencontrez l&apos;une des erreurs suivantes lorsque vous essayez d&apos;exporter un dashboard:

* Depuis New Relic UI: <DNT>**We ran into an error while creating the PDF. Please try again**</DNT>

* Extrait de New Relic NerdGraph : <DNT>**Operation on dashboard entity failed with guid:YOUR\_GUID with cause: Error 504: upstream request timeout**</DNT>

Ces erreurs peuvent se produire si l&apos;exportation dépasse le temps de réponse maximal de l&apos;API. Vérifiez si des widgets sur votre dashboard utilisent une grande fenêtre de temps — par exemple, en comparant des données sur un mois entier. Si c&apos;est le cas, essayez de réduire la fenêtre de temps.

Une autre cause possible est un dashboard avec un grand nombre de widgets. Essayez de supprimer les widgets inutilisés pour voir si cela résout l&apos;erreur.
