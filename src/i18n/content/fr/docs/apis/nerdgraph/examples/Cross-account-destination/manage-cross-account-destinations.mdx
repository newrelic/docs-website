<Callout title="Aperçu">
  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Utilisez le <DNT>NerdGraph API</DNT> pour créer, tester, lister, mettre à jour et supprimer <DNT>cross account destinations</DNT>. <DNT>Cross account destinations</DNT> vous permettent de centraliser la gestion des notifications au niveau de l&apos;organisation.

## Créer une destination [#create-destination]

Pour créer une destination, différentes entrées doivent être fournies pour chaque type de destination.

<Callout variant="important">
  Vous devez fournir un objet de portée identifiant où la destination sera créée. L&apos;objet scope nécessite :

  * id : L&apos;ID du compte ou de l&apos;organisation.
  * type : Le type de périmètre (Compte ou Organisation).

  Une propriété facultative two\_way\_integration est disponible pour les intégrations qui permettent une intégration bidirectionnelle.
</Callout>

    La création d&apos;une destination Slack inter-comptes dans New Relic nécessite une authentification OAuth entre le <DNT>NerdGraph API</DNT> et Slack. Les étapes suivantes expliquent comment générer l&apos;URL d&apos;authentification et terminer la configuration.

    <Steps>
      <Step>
        **Générer l&apos;URL OAuth**

        1. Ouvrez le <DNT>[NerdGraph Explorer](https://api.newrelic.com/graphiql)</DNT> et collez la requête suivante dans l&apos;éditeur de requête. Cette requête génère une URL OAuth Slack unique pour votre organisation.

           ```graphql
           {
             actor {
               organization {
                 aiNotifications {
                   oAuthUrl(type: SLACK) {
                     url
                     transactionId
                     error {
                       description
                       details
                       type
                     }
                   }
                 }
               }
             }
           }
           ```

        2. Cliquez sur <DNT>**Play** (Execute Query)</DNT> en haut de l&apos;explorateur.
      </Step>

      <Step>
        **Accéder au lien d&apos;authentification**

        1. Une fois la requête exécutée, vous verrez une réponse JSON contenant une URL sur le côté droit de l&apos;écran.
        2. Localisez le champ `url` dans la réponse.
        3. Copiez l&apos;intégralité de la chaîne URL (tout ce qui se trouve à l&apos;intérieur des guillemets).
        4. Ouvrez un nouvel onglet dans votre navigateur et collez l&apos;URL.
      </Step>

      <Step>
        **Terminer l&apos;autorisation Slack**

        1. Le lien vous dirige vers une page d&apos;autorisation Slack.
        2. Si vous y êtes invité, connectez-vous à l&apos;espace de travail Slack que vous souhaitez connecter.
        3. Vérifiez les autorisations demandées par New Relic.
        4. Pour autoriser la connexion, cliquez sur <DNT>**Allow**</DNT>.
        5. Vous devriez être redirigé vers une page New Relic « Success ».
      </Step>

      <Step>
        **Vérifier la destination**

        Une fois l&apos;authentification OAuth réussie, la destination est automatiquement créée au niveau de l&apos;organisation, ce qui la rend disponible pour tous les comptes de votre organisation.

        1. Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Alerts &gt; Destinations**</DNT>.
        2. Dans la liste <DNT>**Destinations**</DNT>, vous devriez maintenant voir une nouvelle entrée pour l&apos;espace de travail Slack que vous venez d&apos;authentifier.
      </Step>
    </Steps>
