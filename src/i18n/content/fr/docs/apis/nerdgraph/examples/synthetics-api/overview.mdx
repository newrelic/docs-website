Synthétique Monitorer vous aide à vérifier de manière proactive vos applications, services et points de terminaison pour vous assurer qu&apos;ils fonctionnent correctement. Vous pouvez automatiser la gestion de votre [moniteur Synthétique](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors/), [de votre site privé](/docs/synthetics/synthetic-monitoring/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations/), [de vos identifiants sécurisés](/docs/synthetics/synthetic-monitoring/using-monitors/store-secure-credentials-scripted-browsers-api-tests/) et [de votre temps d&apos;arrêt de moniteur](/docs/synthetics/synthetic-monitoring/using-monitors/monitor-downtimes-disable-monitoring-during-scheduled-maintenance-times/) à l&apos;aide de [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/).

Avec les autorisations et les accès appropriés, vous pouvez gérer par programmation l&apos;ensemble de votre configuration synthétique monitoring via des requêtes et des mutations dans NerdGraph. Vous pouvez également effectuer ces opérations via l&apos; [interface utilisateur de New Relic](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors/).

<Callout variant="important">
  À compter du 26 août 2024, vous ne pouvez plus créer de nouveaux moniteurs à l&apos;aide legacy runtimes sur des sites publics ou privés. Le 22 octobre 2024, nous mettrons [fin à la vie](/whats-new/2024/04/whats-new-04-09-eol-synthetics-runtime-cpm) des versions conteneurisées subordonnées privées (appels par minute) et legacy versions synthétiques du runtime.

  * Pour un emplacement public, utilisez [l&apos;UI de mise à niveau de l&apos;environnement d&apos;exécution](/docs/synthetics/synthetic-monitoring/using-monitors/runtime-upgrade-ui/) pour mettre à jour votre moniteur avec les environnements d&apos;exécution les plus récents.
  * Pour les sites privés, veuillez consulter nos [étapes de migration recommandées](/docs/synthetics/synthetic-monitoring/private-locations/job-manager-transition-guide/#monitorMigration) pour éviter la dégradation du moniteur.
</Callout>

## Exigences [#requirements]

Les exigences comprennent :

* Un [compte New Relic](https://newrelic.com/signup). Vous aurez besoin de votre clé utilisateur API de ce compte pour authentifier [les mutations et les requêtes](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/).
* [Exigences générales de NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#authentication).
* Autorisations utilisateur requises : Vous avez besoin des mêmes [autorisationsmonitoring synthétique](/docs/synthetics/synthetic-monitoring/administration/user-roles-synthetic-monitoring) pour NerdGraph que pour l’interface utilisateur Synthetics.

## Démarrer [#get-started]

Pour commencer à utiliser l&apos;API NerdGraph pour monitoring synthétique :

1. Accédez à l&apos; [explorateur GraphQL de NerdGraph](https://one.newrelic.com/nerdgraph-graphiql)
2. Obtenez votre [clé utilisateur API](/docs/apis/intro-apis/new-relic-api-keys/#user-key) pour l&apos;authentification
3. Choisissez le type ou le mode de fonctionnement de votre moniteur dans les guides ci-dessous.
4. Utilisez les mutations et la requête GraphQL fournies pour gérer votre infrastructuresynthétique monitoring

## Entité de soutien [#supporting-entities]

En plus de la monitoring, vous pouvez gérer infrastructure et configuration de support :

* [Site privé](/docs/apis/nerdgraph/examples/synthetics-api/private-locations) - Créez des zones monitoring derrière votre pare-feu pour les applications internes
* [Sécurisez vos identifiants](/docs/apis/nerdgraph/examples/synthetics-api/secure-credentials) - Stockez en toute sécurité les informations sensibles telles que les mots de passe et les clés API.
* [monitorer temps d&apos;arrêt](/docs/apis/nerdgraph/examples/synthetics-api/monitor-downtimes) - Planifier les fenêtres de maintenance pendant lesquelles le moniteur doit faire une pause

## Opérations de requête [#query-operations]

Utilisez [les opérations de requête](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data) pour récupérer des informations sur votre configuration synthétique monitoring :

* requête tous les moniteurs de votre compte
* Récupérer la configuration du site privé
* Liste des identifiants sécurisés (métadonnées uniquement)
* Afficher les temps d&apos;arrêt programmés du moniteur
* Obtenir le script du moniteur et la configuration de l&apos;étape
* Mapper les ID du moniteur au GUID de l’entité
* Vérifier l&apos;état de la mise à niveau en cours d&apos;exécution pour le moniteur legacy
