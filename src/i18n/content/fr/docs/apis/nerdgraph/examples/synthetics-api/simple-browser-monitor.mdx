New Relic vous permet d&apos;utiliser NerdGraph pour créer [un simple outil de monitoring de navigateur](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors). Un simple moniteur de navigateur effectue des vérifications automatisées de vos applications Web, simulant une véritable interaction utilisateur en chargeant les pages dans un environnement de navigateur complet. Ce tutoriel fournit des exemples d&apos;utilisation de l&apos;API NerdGraph pour automatiser la création d&apos;un simple outil de monitoring de navigateur.

### Demande d&apos;échantillon

```graphql
mutation {
  syntheticsCreateSimpleBrowserMonitor(
    accountId: ACCOUNT_ID
    monitor: {
      browsers: [BROWSERS]
      devices: [DEVICES]
      locations: { public: ["LOCATION_1", "LOCATION_2"] }
      name: "YOUR_MONITOR_NAME"
      period: PERIOD
      runtime: {
        runtimeType: "RUNTIME_TYPE"
        runtimeTypeVersion: "RUNTIME_TYPE_VERSION"
        scriptLanguage: "SCRIPT_LANGUAGE"
      }
      status: STATUS
      uri: "MONITORED_URI"
      advancedOptions: {
        customHeaders: { name: "HEADER_NAME", value: "HEADER_VALUE" }
        enableScreenshotOnFailureAndScript: ENABLE_SCREENSHOT
        responseValidationText: "VALIDATION_TEXT"
        useTlsValidation: TLS_VALIDATION
      }
      apdexTarget: APDEX_TARGET
    }
  ) {
    errors {
      description
      type
    }
  }
}
```

### Exemple de réponse

Une réponse positive renvoie `null` en cas d&apos;erreur :

```json
{
  "data": {
    "syntheticsCreateSimpleBrowserMonitor": {
      "errors": null
    }
  }
}
```

En cas de problème lors de la création du moniteur, le éventail `errors` contiendra des objets avec des champs `description` et `type` expliquant ce qui s&apos;est mal passé.

## Mettre à niveau le runtime d&apos;un simple moniteur de navigateur [#upgrade-monitor-runtime]

Vous pouvez mettre à niveau un simple monitorer de navigateur pour utiliser le nouvel environnement d&apos;exécution Chrome. Nous vous recommandons d&apos;utiliser la version `LATEST` pour vous assurer que votre monitor reste automatiquement à jour avec les sorties de Chrome les plus récentes.

<Callout variant="important">
  **Gestion des versions de Chrome**: New Relic passe à un modèle de version `LATEST` pour les runtimes Chrome. Certaines versions spécifiques de Chrome (telles que 134 et 140) arriveront en fin de vie et seul `LATEST` sera maintenu à l&apos;avenir. Le **18 août 2026**, les outils pour monitorer qui sont encore sur d&apos;anciennes versions de Chrome seront mis à jour de force vers la dernière version stable (Chrome 147 ou supérieure). Lors de la sortie d&apos;une nouvelle version de Chrome, celle-ci sera disponible dans les moniteurs de navigateur simples en tant que `LATEST` dans un délai d&apos;une semaine. Pour vous assurer que vos monitorer reçoivent automatiquement les dernières mises à jour et fonctionnalités de Chrome, mettez à niveau vers `runtimeTypeVersion: "LATEST"` avant cette date.
</Callout>

## Supprimer un simple moniteur de navigateur [#delete-monitor]

Lorsqu&apos;un simple moniteur de navigateur n&apos;est plus nécessaire, vous pouvez le supprimer définitivement à l&apos;aide de la mutation `syntheticsDeleteMonitor`.

Pour supprimer un moniteur, référez-vous à la section [Supprimer un moniteur Synthétique](/docs/apis/nerdgraph/examples/synthetics-api/ping-monitor/#delete-monitor).
