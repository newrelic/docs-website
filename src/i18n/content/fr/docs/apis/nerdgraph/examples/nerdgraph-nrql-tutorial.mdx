Vous pouvez utiliser notre API NerdGraph pour effectuer des requêtes [NRQL](/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql) .

## Présentation des requêtes avec NerdGraph [#overview]

Vous pouvez exécuter une requête NRQL à l&apos;aide du [générateur de requêtes](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder) dans l&amp;apos;UI, ou vous pouvez utiliser notre API NerdGraph. NerdGraph vous donne la possibilité de faire certaines choses que vous ne pouvez pas faire depuis l&amp;apos;UI, comme [effectuer des requêtes sur plusieurs comptes](#cross-account-query) et exécuter [des requêtes asynchrones](/docs/apis/nerdgraph/examples/async-queries-nrql-tutorial).

## Exigences [#requirements]

* Tous [les types d&apos;utilisateurs](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) peuvent interroger les données des comptes auxquels ils ont accès. Si vous ne pouvez pas effectuer de requête via NerdGraph, c&amp;apos;est peut-être parce que vous n&amp;apos;avez pas [accès à ce compte](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#understand-concepts).
* Toutes les requêtes sont soumises aux [limites de requêtesNRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/rate-limits-nrql-queries)

## Requête NRQL de base avec NerdGraph [#basic-queries]

Pour effectuer une requête NRQL de base à l&apos;aide de NerdGraph, deux exigences principales sont nécessaires :

* Vous devez transmettre la requête NRQL en tant qu&apos;argument de chaîne à l&apos;objet NRQL
* Vous devez inclure le champ `results` dans votre requête

Par exemple, pour obtenir un décompte de tous [les événements de transaction](/docs/insights/insights-data-sources/default-data/apm-default-event-attributes-insights#transaction-event) au cours de la dernière heure, vous devez exécuter la requête suivante, en spécifiant votre [ID de compteNew Relic ](/docs/accounts/accounts-billing/account-structure/account-id).

```graphql
{
  actor {
    account(id: YOUR_ACCOUNT_ID) {
      nrql(query: "SELECT count(*) FROM Transaction SINCE 1 HOUR AGO") {
        results
      }
    }
  }
}
```

Cet exemple de requête renverrait un résultat comme celui-ci :

```json
{
  "data": {
    "actor": {
      "account": {
        "nrql": {
          "results": [
            {
              "count": 1000
            }
          ]
        }
      }
    }
  }
}
```

Vous recherchez des détails sur la façon d’écrire une requête NRQL ? Voir [la section doc NRQL](/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql).

## Requête inter-comptes [#cross-account-query]

Avec NerdGraph, vous pouvez exécuter une requête sur plusieurs comptes, ce que vous ne pouvez pas faire dans le générateur de requêtes. Vous aurez besoin des [identifiants de compte](/docs/accounts/accounts-billing/account-structure/account-id) pour les comptes que vous souhaitez interroger.

Voici un exemple d&apos;exécution d&apos;une requête NRQL inter-comptes :

```graphql
{
  actor {
    nrql(
      accounts: [ACCOUNT_ID_1, ACCOUNT_ID_2, ACCOUNT_ID_3]
      options: {}
      query: "YOUR_NRQL_QUERY"
      timeout: 70
    ) {
      results
    }
  }
}
```

Pour savoir comment créer un dashboard avec des données provenant de plusieurs comptes, consultez [le didacticiel dashboard NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-dashboards#cross-account).

## Créer des graphiques intégrables [#embeddable-charts]

En plus de renvoyer des données brutes, vous pouvez récupérer des liens de graphiques intégrables pour les données à utiliser dans une application. Par exemple, au lieu d&apos;un seul décompte de [transactions](/docs/insights/insights-data-sources/default-data/apm-default-event-attributes-insights#transaction-event), vous pouvez créer un [graphique](/docs/insights/use-insights-ui/manage-dashboards/chart-types#widget-types) illustrant une série chronologique de décomptes groupés au fil du temps. Ajoutez `TIMESERIES` à votre requête avec `embeddedChartUrl`:

```graphql
{
  actor {
    account(id: YOUR_ACCOUNT_ID) {
      nrql(query: "SELECT count(*) from Transaction TIMESERIES") {
        embeddedChartUrl
      }
    }
  }
}
```

Cet exemple de requête NerdGraph renvoie l&apos;URL du graphique dans la réponse suivante :

```json
{
  "data": {
    "actor": {
      "account": {
        "nrql": {
          "embeddedChartUrl": "https://chart-embed.service.newrelic.com/charts/EMBEDDABLE-CHART-ID"
        }
      }
    }
  }
}
```

Si vous affichez l’URL du graphique intégré à l’aide d’un client HTTP standard, il renvoie une image montrant une visualisation de la réponse à la requête que vous avez soumise. Ces graphiques suivent les mêmes [règles de graphiques intégrés](/docs/using-new-relic/user-interface-functions/share-your-data/embed-charts-external-webpages#rules) que les graphiques intégrés créés ailleurs. Pour modifier le style de la visualisation des données, transmettez un argument `chartType` à `embeddedChartUrl`.

## Facettes suggérées [#suggest-facets]

Lorsque vous utilisez NerdGraph pour explorer vos données, vous pouvez utiliser le champ `suggestedFacets` pour renvoyer l&amp;apos;attribut suggéré à utiliser dans [la requête NRQL à facettes](/docs/insights/nrql-new-relic-query-language/nrql-resources/nrql-syntax-components-functions#sel-facet).

<CollapserGroup>
  <Collapser id="suggested-facet-rules" title="Règles régissant les facettes suggérées">
    Voici quelques-unes des règles qui régissent les attributs suggérés :

    * <DNT>**Built-in suggestions.**</DNT> Chaque [type d&apos;événement](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#event) est accompagné de son propre ensemble d&amp;apos;attributs recommandés. Ce sont des attributs choisis par New Relic pour leur importance et leur popularité.

    * <DNT>**Usage-based suggestions.**</DNT> Certaines suggestions d&amp;apos;attributs sont basées sur les requêtes qui ont été fréquemment utilisées par votre compte. Ces suggestions peuvent inclure l&amp;apos;attribut personnalisé.

    * <DNT>**Role restriction.**</DNT> [les utilisateurs restreints](/docs/accounts/accounts/roles-permissions/users-roles) n&amp;apos;ont pas accès aux suggestions de facettes liées au compte.

      Pour désactiver l&apos;utilisation des données de compte pour déterminer la requête suggérée, [contactez le support](/docs/using-new-relic/welcome-new-relic/getting-started/find-help-or-file-support-ticket#support_tickets).
  </Collapser>
</CollapserGroup>

<CollapserGroup>
  <Collapser id="example-suggested-query" title="Exemple de renvoi d'attribut suggéré">

La réponse suggère l&amp;apos;attribut `host` . Le facettage par `host` peut révéler qu&amp;apos;un hôte traite plus requests que d&amp;apos;autres hôtes.

    ```
            suggestedFacets {

            }
          }
        }
      }
    ```

    ```json
        "actor": {
          "account": {
            "nrql": {
              "suggestedFacets": [
                {
                  "attributes": [

                  ]
                }
              ]
            }
          }
        }
      }
    }

    ```
