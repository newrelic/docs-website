En plus de gérer votre [canal d&apos;alerte de notification dans l&apos;UI](/docs/alerts-applied-intelligence/notifications/intro-notifications), vous pouvez utiliser notre API NerdGraph.

<Callout variant="important">
  Ce document fait référence à l&apos;utilisation API Nerdgraph pour la nouvelle plateforme notification utilisant des destinations et des messages notification . les messages de notification sont également appelés canaux, qui sont différents des legacy canaux de notification.
</Callout>

<Callout variant="tip">
  Pour obtenir de l’aide pour démarrer avec NerdGraph, consultez [Introduction à NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).
</Callout>

## Lister et filtrer les chaînes [#list-and-filter]

La requête `channels` vous permet de parcourir tous vos canaux par compte. Il permet également certaines fonctionnalités de filtrage.

<CollapserGroup>
  <Collapser id="list-all-channels" title="Liste de toutes les chaînes d'un compte">
    Voici un exemple :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels {
              entities {
                id
                name
              }
              error {
                details
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>

  <Collapser id="paginate" title="Pagination des canaux avec pagination du curseur">
    Afin de parcourir vos chaînes, vous devez demander le champ `nextCursor` lors de votre requête initiale.

    Avec la pagination du curseur, vous continuez à effectuer une requête via l&apos;ensemble de résultats jusqu&apos;à ce que le `nextCursor` renvoyé par la réponse revienne vide. Cela signifie que vous avez atteint la fin de vos résultats.

    Voici un exemple :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels(cursor: "") {
              nextCursor
              entities {
                id
                name
              }
              totalCount
            }
          }
        }
      }
    }
    ```

    Le code ci-dessus renvoie un ensemble de résultats comme celui-ci :

    ```json
    {
      "data": {
        "actor": {
          "account": {
            "aiNotifications": {
              "channels": {
                "nextCursor": "/8o0y2qiR54m6thkdgHgwg==:jZTXDFKbTkhKwvMx+CtsPVM=",
                "entities": [
                  {
                    "id": "01c0cbe7-3d70-47c1-99e0-adf906eed6c2",
                    "name": "Channel Name"
                  },
                  {
                    "id": "05db0207-c137-4985-8cb5-f21e7e57b8cc",
                    "name": "Another Channel Name"
                  }
                  // ... more channels here in reality
                ],
                "totalCount": 807
              }
            }
          }
        }
      }
    }
    ```

    Ainsi, dans votre requête suivante, fournissez le curseur comme ceci, jusqu&apos;à ce que le curseur soit vide :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels(cursor: "/8o0y2qiR54m6thkdgHgwg==:jZTXDFKbTkhKwvMx+CtsPVM=") {
              nextCursor
              entities {
                id
                name
              }
              totalCount
            }
          }
        }
      }
    }
    ```
  </Collapser>

  <Collapser id="find-channels-by-name" title="Trouver toutes les chaînes par nom">
    L&apos;API permet d&apos;interroger les canaux par nom. Le filtre `name` renvoie des correspondances exactes et des correspondances partielles. Il n&apos;est pas sensible à la casse. Cela renverra uniquement les informations pour les chaînes qui correspondent au nom fourni.

    Dans cet exemple, nous souhaitons trouver les chaînes avec `"DevOps"` dans le nom :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels(filters: {
              name: "DevOps"
            }) {
              entities {
                id
                name
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>

  <Collapser id="find-by-id" title="Rechercher une chaîne par identifiant">
    L&apos;API vous permet d&apos;effectuer une requête par ID de canal :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels(filters: {id: YOUR_CHANNEL_ID}) {
              entities {
                id
                name
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>

  <Collapser id="find-by-destination-id" title="Trouver toutes les chaînes par identifiant de destination">
    L&apos;API vous permet d&apos;interroger les canaux par ID de destination :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels(filters: {destinationId: YOUR_DESTINATION_ID}) {
              entities {
                id
                name
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>

  <Collapser id="find-by-type" title="Trouver toutes les chaînes par type">
    L&apos;API vous permet d&apos;effectuer des requêtes par type de canal. La requête suivante renverra tous les canaux de messagerie sur le compte choisi :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channels(filters: {type: EMAIL}) {
              entities {
                id
                name
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## Créer une chaîne [#create-channel]

Afin de créer un canal, des entrées différentes doivent être fournies pour chaque type de canal. Chaque canal est connecté à une destination. Pour plus d&apos;informations sur les destinations, consultez le [didacticiel NerdGraph sur les destinations](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-destinations).

La bonne pratique consiste à utiliser le point de terminaison `channelSchema` pour voir quels champs doivent être envoyés sous `properties` comme ceci :

```graphql
{
  actor {
    account(id: YOUR_ACCOUNT_ID) {
      aiNotifications {
        channelSchema(
          channelType: CHANNEL_TYPE,
          destinationId: YOUR_DESTINATION_ID,
          product:  YOUR_PRODUCT,
          constraints: []
        ) {
          schema {
            fields {
              mandatory
              label
              key
              component
            }
          }
          result
        }
      }
    }
  }
}
```

  <Collapser className="freq-link" id="jira" title="Atlassian Jira">
    Jira est un système de tickets configurable, et il n&apos;existe donc aucun moyen statique de créer ce canal.

    Il existe deux champs statiques : `project` et `issuetype`.

    Récupérez les suggestions `project` et utilisez l&apos;une des valeurs comme contrainte pour `issuetype`, comme indiqué ici :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channelSuggestions(
              channelType: JIRA_CLASSIC,
              destinationId: YOUR_DESTINATION_ID,
              key: FIELD_NAME
              constraints: [
                {
                  key: "project",
                  value: YOUR_PROJECT_VALUE
                }
              ]
            ) {
              entities {
                value
              }
              error
            }
          }
        }
      }
    }
    ```

    Les valeurs choisies seront utilisées comme contraintes pour récupérer le schéma :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            channelSchema(
              channelType: JIRA_CLASSIC,
              destinationId: YOUR_DESTINATION_ID,
              product:  YOUR_PRODUCT,
              constraints: [
                {
                  key: "project",
                  value: YOUR_PROJECT_VALUE
                },
                {
                  key: "issuetype",
                  value: YOUR_ISSUE_TYPE_VALUE
                }
              ]
            ) {
              schema {
                fields {
                  mandatory
                  label
                  key
                  component
                }
              }
              result
            }
          }
        }
      }
    }
    ```

    Après avoir récupéré chaque champ et choisi une valeur parmi les suggestions ou sous forme de texte libre, vous pouvez créer un canal :

    ```graphql
    mutation {
      aiNotificationsCreateChannel(accountId: YOUR_ACCOUNT_ID, channel: {
        type: JIRA,
        name: "Channel Name",
        destinationId: YOUR_DESTINATION_ID,
        product: YOUR_PRODUCT,
        properties: [
          {
            key: YOUR_FIELD_NAME,
            value: YOUR_FIELD_NAME,
          },
          // ... And so forth with the rest of the fields
        ]
      }) {
        channel {
          id
          name
        }
      }
    }
    ```
  </Collapser>

  <Collapser className="freq-link" id="microsoft-teams" title="Microsoft Teams">
    <Callout title="Note">
      L&apos;intégration de Microsoft Teams est désormais disponible aux États-Unis et dans l&apos;Union européenne.
    </Callout>

    ### Prérequis

    Avant de configurer New Relic pour envoyer des notifications à Microsoft Teams, vous devez :

    * Vous disposez déjà d&apos;un canal Microsoft Teams dans lequel vous souhaitez recevoir des notifications.
    * Créez une [destination Microsoft Teams](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-destinations/#microsoft-teams) dans New Relic et obtenez le `destinationId`.

    ### Étape 1 : Récupérer les identifiants d’équipe disponibles

    <Callout variant="tip">
      Si vous connaissez déjà votre ID d&apos;équipe et votre ID de canal, vous pouvez ignorer les étapes 1 et 2 et passer directement à [l&apos;étape 3](#step-3-configure-the-new-relic-notification-channel) pour configurer le canal de notification.
    </Callout>

    Cette requête permet de découvrir quelles équipes Microsoft Teams sont accessibles via votre destination. Vous fournissez `destinationId` et spécifiez que vous souhaitez des identifiants d&apos;équipe (via `key: "teamId"`), et l&apos;API renvoie une liste des équipes disponibles avec leurs noms et identifiants :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            suggestions(
              destinationId: YOUR_DESTINATION_ID,
              key: "teamId",
              channelType: MICROSOFT_TEAMS,
              constraints: []
            ) {
              entities {
                displayValue
                value
              }
              errors {
                description
                details
                type
              }
            }
          }
        }
      }
    }
    ```

    La réponse contiendra une liste d&apos;équipes avec leurs noms d&apos;affichage et leurs identifiants d&apos;équipe uniques :

    ```json
    {
      "data": {
        "actor": {
          "account": {
            "aiNotifications": {
              "suggestions": {
                "entities": [
                  {
                    "displayValue": "Engineering Team",
                    "value": "389e7f6c-xxxx-47f0-aa77-xxxxxxxxxxxx"
                  },
                  {
                    "displayValue": "DevOps Team",
                    "value": "834dc358-xxxx-4445-9938-xxxxxxxxxxxx"
                  }
                ],
                "errors": []
              }
            }
          }
        }
      }
    }
    ```

    <Callout variant="tip">
      Si vous ne voyez pas l&apos;équipe que vous recherchez dans les résultats, vous pouvez utiliser le paramètre `filter` pour la rechercher par son nom :

      ```graphql
      {
        actor {
          account(id: YOUR_ACCOUNT_ID) {
            aiNotifications {
              suggestions(
                destinationId: YOUR_DESTINATION_ID,
                key: "teamId",
                channelType: MICROSOFT_TEAMS,
                constraints: [],
                filter: {
                  type: CONTAINS,
                  value: "Engineering"
                }
              ) {
                entities {
                  displayValue
                  value
                }
              }
            }
          }
        }
      }
      ```
    </Callout>

    ### Étape 2 : Récupérer les identifiants de canal disponibles pour une équipe

    Une fois que vous avez obtenu un ID d&apos;équipe à l&apos;étape 1, cette requête permet de découvrir quels canaux existent au sein de cette équipe spécifique. Vous fournissez les `destinationId` et `teamId` (en tant que contrainte), et l&apos;API renvoie une liste des canaux disponibles avec leurs noms et identifiants :

    ```graphql
    {
      actor {
        account(id: YOUR_ACCOUNT_ID) {
          aiNotifications {
            suggestions(
              destinationId: YOUR_DESTINATION_ID,
              key: "channelId",
              channelType: MICROSOFT_TEAMS,
              constraints: [
                {
                  key: "teamId",
                  value: "YOUR_TEAM_ID"
                }
              ]
            ) {
              entities {
                displayValue
                value
              }
              errors {
                description
                details
                type
              }
            }
          }
        }
      }
    }
    ```

    La réponse contiendra une liste des canaux de l&apos;équipe spécifiée :

    ```json
    {
      "data": {
        "actor": {
          "account": {
            "aiNotifications": {
              "suggestions": {
                "entities": [
                  {
                    "displayValue": "General",
                    "value": "19:xxxxxxxxxxxxxxxxxxxxxxxx@thread.tacv2"
                  },
                  {
                    "displayValue": "Alerts",
                    "value": "19:yyyyyyyyyyyyyyyyyyyyyyyy@thread.tacv2"
                  }
                ],
                "errors": []
              }
            }
          }
        }
      }
    }
    ```

    <Callout variant="tip">
      Tout comme pour les ID d&apos;équipe, vous pouvez filtrer les chaînes par nom en utilisant le paramètre `filter` pour effectuer une recherche au sein de l&apos;équipe spécifiée.
    </Callout>

    ### Étape 3 : Configurer le canal de notification New Relic

    Après avoir obtenu l&apos;ID d&apos;équipe et l&apos;ID de canal, configurez le canal de notification New Relic pour envoyer des alertes à votre canal Microsoft Teams.

    <Callout variant="important">
      Cette mutation crée un objet de canal de notification New Relic qui se connecte à un canal Microsoft Teams **existant**. Le canal Teams doit déjà exister dans votre espace de travail Microsoft Teams. Cette API ne crée pas de nouvelles équipes ou de nouveaux canaux au sein de Microsoft Teams ; elle configure uniquement New Relic pour envoyer des notifications à vos canaux Teams existants.
    </Callout>

    ```graphql
    mutation {
      aiNotificationsCreateChannel(accountId: YOUR_ACCOUNT_ID, channel: {
        type: MICROSOFT_TEAMS,
        name: "Channel Name",
        destinationId: YOUR_DESTINATION_ID,
        product: YOUR_PRODUCT,
        properties: [
          {
            key: "teamId",
            value: YOUR_TEAM_ID
          },
          {
            key: "channelId",
            value: YOUR_CHANNEL_ID
          }
        ]
      }) {
        channel {
          id
          name
        }
      }
    }
    ```

    Le paramètre `product` spécifie le produit New Relic générant la notification. Les valeurs valides incluent :

    * `IINT` - renseignement appliqué (incident Intelligence)
    * `ALERTS`
    * `ERROR_TRACKING`
    * `APM` - Monitoring des performances des applications
    * `CHANGE_TRACKING`
    * `SECURITY`
    * `PD` - Détection proactive
    * Autres valeurs : `CSSP`, `DISCUSSIONS`, `NTFC`, `SHARING`
  </Collapser>

## Mettre à jour une chaîne [#update-channel]

Lorsque vous mettez à jour un canal, notez que vous n&apos;avez pas besoin de fournir tous les attributs du canal. Par exemple, si vous souhaitez uniquement mettre à jour le nom, c&apos;est le seul attribut que vous devez mettre à jour, comme indiqué ici :

```graphql
mutation {
  aiNotificationsUpdateChannel(accountId: YOUR_ACCOUNT_ID, channelId: YOUR_CHANNEL_ID, channel: {
    name: "Updated channel Name"
  }) {
    channel {
      id
      name
    }
  }
}
```

        key: "summary",
        value: YOUR_PAGE_SUMMARY
      }
    ]
  }) {
    error {
      details

  }

```

mutation {
  aiNotificationsTestChannelById(accountId: YOUR_ACCOUNT_ID, channelId: YOUR_CHANNEL_ID) {

    result
  }
}
```

## Supprimer une chaîne [#delete-channel]

Vous pouvez supprimer des chaînes via l&apos;API NerdGraph.

```graphql
mutation {
  aiNotificationsDeleteChannel(accountId: YOUR_ACCOUNT_ID, channelId: YOUR_CHANNEL_ID) {
    ids
    error {
      details
    }
  }
}
```
