Une carte de pointage sert de conteneur pour diverses règles qui évaluent collectivement le respect des normes définies. Les scorecards utilisent des règles pour monitorer la conformité entre différentes entités et équipes, identifier les domaines à améliorer et garantir l&apos;alignement avec les exigences internes et externes.

## Accès par défaut aux Scorecards [#default-access]

New Relic fournit un accès par défaut aux Scorecards via ces rôles standards :

<table>
  <thead>
    <tr>
      <th>
        Action
      </th>

      <th>
        Rôle requis
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        Afficher les Scorecards
      </td>

      <td>
        [<DNT>**Organization read only**</DNT>](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles)
      </td>
    </tr>

    <tr>
      <td>
        Créer, mettre à jour et supprimer des tableaux de bord (Scorecards) et des règles
      </td>

      <td>
        [<DNT>**Organization Product Admin**</DNT>](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles)

         ou un rôle Scorecards personnalisé
      </td>
    </tr>
  </tbody>
</table>

Vous pourriez avoir besoin d&apos;un rôle personnalisé si les permissions par défaut ne répondent pas aux besoins de votre organisation.

## Prérequis [#prerequisites]

Avant de commencer, assurez-vous d’avoir :

* [Exigences générales de NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#authentication)
* Administrateur d&apos;organisation avec le rôle `Organization Product Admin` pour créer des rôles personnalisés

<Callout variant="tip">
  Vous pouvez également créer un rôle personnalisé pour les Scorecards via l&apos;UI. Pour les autorisations à sélectionner et le périmètre à utiliser, consultez [Capacités des Scorecards](/docs/service-architecture-intelligence/scorecards/getting-started/#scorecards-capabilities).
</Callout>

Pour créer un rôle personnalisé via l&apos;API NerdGraph, effectuez les étapes suivantes :

<Steps>
  <Step>
    ## Rechercher les identifiants d&apos;autorisation requis pour les dashboards

    Utilisez la requête `customerAdministration` pour récupérer une liste de capacités, leurs ID d&apos;autorisation et les informations associées. Filtrez par `scope: "organization"` pour obtenir les permissions de Scorecards.

    #### Paramètres d&apos;entrée

    <table>
      <thead>
        <tr>
          <th>
            paramètres
          </th>

          <th>
            Type de données
          </th>

          <th>
            Est-ce obligatoire ?
          </th>

          <th>
            Description
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `eq`
          </td>

          <td>
            Chaîne
          </td>

          <td>
            Oui
          </td>

          <td>
            Définissez la valeur sur 

            `organization`

             pour récupérer les autorisations pour les tableaux de bord.
          </td>
        </tr>
      </tbody>
    </table>

    #### Demande d&apos;échantillon

    ```graphql
    {
      customerAdministration {
        permissions(filter: { scope: { eq: "organization" } }) {
          items {
            feature
            category
            id
            product
          }
        }
      }
    }
    ```

    #### Exemple de réponse

    ```json
    {
      "data": {
        "customerAdministration": {
          "permissions": {
            "items": [
              {
                "category": "READ",
                "feature": "Scorecards Rules",
                "id": "xxxxx",
                "product": "New Relic One"
              },
              {
                "category": "MANAGE",
                "feature": "Scorecards Rules",
                "id": "xxxxx",
                "product": "New Relic One"
              },
              {
                "category": "READ",
                "feature": "Scorecards",
                "id": "xxxxx",
                "product": "New Relic One"
              },
              {
                "category": "MANAGE",
                "feature": "Scorecards",
                "id": "xxxxx",
                "product": "New Relic One"
              }
            ]
          }
        }
      }
    }
    ```

    À partir de la réponse, identifiez et copiez les ID d&apos;autorisation où `feature` est `"Scorecards"` ou `"Scorecards Rules"` et `category` correspond au niveau d&apos;accès que vous souhaitez accorder. Vous aurez besoin de ces ID à l&apos;étape suivante.
  </Step>

  <Step>
    ## Récupérez votre identifiant d&apos;organisation

    Récupérez l&apos;ID de votre organisation, que vous utiliserez dans les mutations ultérieures.

    #### Demande d&apos;échantillon

    ```graphql
        }

    }
    ```

    #### Exemple de réponse

    ```json

            "id": "YOUR_ORGANIZATION_ID"
          }
        }
      }
    }
    ```

    Copiez votre ID d&apos;organisation à partir de la réponse. Vous en aurez besoin pour créer le rôle personnalisé.
  </Step>

## Gérer les rôles personnalisés existants [#manage-roles]

<CollapserGroup>
  <Collapser id="update-role" title="Mettre à jour un rôle">
    Utilisez la mutation `customRoleUpdate` pour renommer un rôle ou remplacer son ensemble d&apos;autorisations.

    ```graphql
    mutation {
      customRoleUpdate(
        id: ROLE_ID
        name: "Updated role name"
        permissionIds: [xxxxx, xxxxx]
      ) {
        id
      }
    }
    ```

    <table>
      <thead>
        <tr>
          <th>
            paramètres
          </th>

          <th>
            Description
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            `id`
          </td>

          <td>
            L&apos;ID du rôle à mettre à jour.
          </td>
        </tr>

        <tr>
          <td>
            `name`
          </td>

          <td>
            Nouveau nom d&apos;affichage pour le rôle.
          </td>
        </tr>

        <tr>
          <td>
            `permissionIds`
          </td>

          <td>
            Liste complète de remplacement des ID de permission. Cela remplace toutes les permissions existantes sur le rôle.
          </td>
        </tr>
      </tbody>
    </table>
  </Collapser>

  <Collapser id="delete-role" title="Supprimer un rôle">
    Utilisez la mutation `customRoleDelete` pour supprimer définitivement un rôle personnalisé.

    ```graphql
    mutation {
      customRoleDelete(id: ROLE_ID) {
        id
      }
    }
    ```

    La réponse renvoie l&apos;ID du rôle supprimé, confirmant la réussite de la suppression.
  </Collapser>

  <Collapser id="find-role-id" title="Trouver l'ID d'un rôle personnalisé">
    Utilisez la requête suivante pour rechercher un ID de rôle en parcourant les groupes de votre organisation et leurs rôles attribués.

    ```graphql
    {
      actor {
        organization {
          authorizationManagement {
            authenticationDomains(id: "YOUR_AUTHENTICATION_DOMAIN_ID") {
              authenticationDomains {
                groups {
                  groups {
                    displayName
                    id
                    roles {
                      roles {
                        roleId
                        name
                      }
                    }
                  }
                }
              }
            }
          }
        }
      }
    }
    ```
  </Collapser>
</CollapserGroup>
