Pour autoriser le partage de données entre les comptes au sein d&apos;une organisation, un compte doit accepter la fonctionnalité « alerte inter-comptes ». Suivez ces instructions sur la façon d’utiliser l’explorateur d’API NerdGraph pour mettre à jour les paramètres du compte en activant les alertes inter-comptes.

<Callout variant="important">
  Seul l&apos;utilisateur disposant de l&apos;autorisation « Contrôle d&apos;accès au partage de données » peut mettre à jour les élections inter-comptes pour le partage de données. Par défaut, cette autorisation est accordée uniquement à l&apos;utilisateur avec le rôle « Responsable de l&apos;organisation ».
</Callout>

## Étapes à suivre pour mettre à jour une alerte inter-comptes

<Steps>
  <Step>
    Collectez la liste des identifiants de compte à mettre à jour. L&apos;API accepte jusqu&apos;à **20 identifiants de compte**.
  </Step>

  <Step>
    Indiquez si vous souhaitez activer `true` ou désactiver `false` l&apos;élection.
  </Step>

  <Step>
    Utilisez l&apos;explorateur d&apos;API NerdGraph pour mettre à jour les élections de compte.
  </Step>
</Steps>

Voici un exemple de mutation :

```graphql
mutation {
  alertsUpdateCrossAccountElections(
    accountIds: [YOUR_1ST_ACCOUNT_ID, YOUR_2ND_ACCOUNT_ID, YOUR_3RD_ACCOUNT_ID],
    electionStatus: true
  ) {
    electionStatus
  }
}
```
