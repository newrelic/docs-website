NerdGraph est notre API au format GraphQL qui vous permet d&apos;interroger les données New Relic et de configurer certaines fonctionnalités New Relic . Après avoir [créé un compte New Relic gratuit](https://newrelic.com/signup) et [installé](/docs/using-new-relic/cross-product-functions/install-configure/install-new-relic/) l&amp;apos;un de nos services monitoring , vous pouvez commencer à utiliser NerdGraph.

## Qu&apos;est-ce que NerdGraph ? [#overview]

NerdGraph est l&apos;une des nombreuses [API](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/) de New Relic. NerdGraph est l&amp;apos;API que nous recommandons pour interroger les données New Relic et pour effectuer certaines configurations spécifiques ([en savoir plus sur les fonctionnalités](#tutorials)). NerdGraph fournit une API interface unique pour renvoyer des données à partir des différentes API et microservices de New Relic. Au fil du temps, d’autres fonctionnalités de configuration seront ajoutées à NerdGraph.

<Callout variant="important">
  NerdGraph n&apos;est pas utilisé pour l&apos;ingestion de données. Pour cela, vous utiliserez nos [API d&apos;ingestion de données](/docs/apis/get-started/intro-apis/introduction-new-relic-apis/#data-type-apis).
</Callout>

NerdGraph est construit à l&apos;aide de [GraphQL](https://graphql.org), un format d&amp;apos;API open source qui vous permet de demander exactement les données nécessaires, sans sur-extraction ni sous-extraction.

## Exigences [#requirements]

Avant de commencer :

* Vous avez besoin d&apos;un compte New Relic , et avec ce compte, vous pouvez accéder à votre API <InlinePopover type="userKey" />que vous devez inclure avec la requête et les mutations.
* Le type d&apos;utilisateur et les rôles attribués peuvent affecter vos autorisations NerdGraph. Pour plus de détails, voir [Facteurs affectant l’accès](/docs/accounts/accounts-billing/account-structure/factors-affecting-access-features-data#user-permissions).

## point de terminaison [#endpoints]

Les points de terminaison d&apos;API que vous utilisez dépendent de [la région de votredata center ](/docs/accounts/accounts-billing/account-setup/choose-your-data-center):

* Point de terminaison principal : `https://api.newrelic.com/graphql`
* point de terminaison pour les comptes utilisant data center de l&apos;UE : `https://api.eu.newrelic.com/graphql`

Pour accéder au point de terminaison, vous pouvez effectuer des appels à partir de votre code, utiliser l&apos;[explorateur d&apos;API NerdGraph](/docs/apis/nerdgraph/get-started/nerdgraph-explorer) ou exécuter une commande `curl` similaire à celle-ci :

```bash
curl -X POST https://api.newrelic.com/graphql \
     -H 'Content-Type: application/json' \
     -H 'API-Key: YOUR_NEW_RELIC_USER_KEY' \
     -d '{ "query":  "{ requestContext { userId apiKey } }" }'
```

## Fonctionnalité NerdGraph [#tutorials]

Les fonctionnalités de NerdGraph peuvent être divisées en deux catégories principales :

* <DNT>**Querying data.**</DNT> Vous pouvez récupérer des données à diverses fins, notamment les utiliser dans un workflow programmatique ou créer une [applicationNew Relic ](https://developer.newrelic.com/build-apps)pour des visualisations de données personnalisées.
* <DNT>**Configuring features.**</DNT> Il existe une variété de configurations disponibles et d&amp;apos;autres seront ajoutées au fil du temps. Vous pouvez effectuer des opérations telles que l&amp;apos;ajout d&amp;apos;étiquettes, la configuration de la charge de travail ou la personnalisation des « métriques dorées ».

Vous pouvez utiliser NerdGraph pour renvoyer et configurer une large gamme de données. Une façon de voir ce que NerdGraph peut faire est de commencer à créer une requête et à explorer le schéma à l&apos;aide de l&apos;[explorateur NerdGraph](/docs/apis/nerdgraph/get-started/nerdgraph-explorer).

## Terminologie NerdGraph [#terminology]

Les termes suivants proviennent de GraphQL, le format API utilisé par NerdGraph.

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        Terme
      </th>

      <th>
        Définition
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        requêtes et mutations
      </td>

      <td>
        Il existe deux classes d&apos;opérations GraphQL :

        * [Les requêtes](https://graphql.org/learn/queries) sont requests de base utilisées uniquement pour récupérer des données. Ces requêtes ne sont pas statiques, ce qui signifie que vous pouvez demander plus ou moins de données, selon vos besoins. Pour chaque requête, vous pouvez spécifier exactement les données que vous souhaitez récupérer, à condition qu&amp;apos;elles soient prises en charge par le [schéma](https://graphql.org/learn/schema).
        * [Les mutations](https://graphql.org/learn/queries/#mutations) sont requests qui exécutent une action, comme la création d&amp;apos;une ressource ou la modification configuration. Les mutations nécessitent le mot-clé `mutation`, ainsi que le `name` de la mutation.
      </td>
    </tr>

    <tr>
      <td>
        Taper
      </td>

      <td>
        Les données dans GraphQL sont organisées en types. Les types peuvent être [des scalaires](https://graphql.org/learn/schema/#scalar-types) (comme des chaînes, des nombres ou des booléens) ou des types d&amp;apos;objets.

        Un [type d&apos;objet](https://graphql.org/learn/schema/#object-types-and-fields) est un type personnalisé composé d&amp;apos;une collection de champs. Par exemple, un type d’objet appelé `User` peut représenter un utilisateur dans un système.
      </td>
    </tr>

    <tr>
      <td>
        Champ
      </td>

      <td>
        Un [champ](https://graphql.org/learn/schema/#object-types-and-fields) représente une information sur un type d&amp;apos;objet qui peut être interrogé. Les champs peuvent être des scalaires, des listes ou des objets. Par exemple, un type d’objet `User` pourrait avoir un champ de chaîne appelé `name`.
      </td>
    </tr>

    <tr>
      <td>
        Interface
      </td>

      <td>
        Une [interface](https://graphql.org/learn/schema/#interfaces) est un type abstrait qui représente une collection de champs communs que d&amp;apos;autres types d&amp;apos;objets peuvent `implement`.
      </td>
    </tr>
  </tbody>
</table>
