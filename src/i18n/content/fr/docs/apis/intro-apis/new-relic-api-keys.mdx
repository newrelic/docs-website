Nos solutions monitoring et API utilisent API clés pour authentifier et vérifier votre identité. Ces clés permettent uniquement aux personnes approuvées de votre organisation de signaler des données à New Relic, d&apos;accéder à ces données et de configurer les fonctionnalités. Les clés primaires sont la clé de licence (pour générer des rapports de données) et le <InlinePopover type="userKey" />(pour travailler avec NerdGraph, notre API GraphQL).

<img title="Screenshot of the API keys UI" alt="Screenshot of the API keys UI" src="/images/accounts_screenshot-crop_api-keys-ui.webp" />

Pour démarrer avec la clé API:

1. [Créez votre compte gratuit](https://newrelic.com/signup) si vous ne l&apos;avez pas déjà fait
2. Utilisez notre [agent et notre intégration](https://newrelic.com/instant-observability/?category=infrastructure-and-os) pour commencer à ingérer des données dans New Relic
3. Créez et gérez votre clé API à partir de la [API UI page de l&apos;](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) clé afin de pouvoir commencer à observer vos données immédiatement

## Afficher et gérer la clé API [#keys-ui]

Vous pouvez gérer la plupart des clés API à partir de la [page d&apos;interface utilisateur de clés d&apos;API](https://one.newrelic.com/api-keys) située dans le [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings/#user-menu) lorsque vous cliquez sur <DNT>**API keys**</DNT> . Vous pouvez également [utiliser NerdGraph pour gérer les clés](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys)

**Clés d&apos;ingestion (pour le compte) :** Lorsque vous créez une clé d&apos;ingestion, nous vous fournissons temporairement la clé complète. Cependant, une fois la clé créée, seuls les 8 premiers caractères sont affichés dans l&apos;interface utilisateur des clés API. Si vous devez récupérer la clé complète après sa création, cliquez sur les points de suspension dans l&apos;interface utilisateur des clés API, copiez l&apos;ID de la clé et [utilisez NerdGraph pour interroger la clé complète.](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys/#query-keys)

**Clés utilisateur (pour votre propre usage) :** Lorsque vous créez une clé utilisateur pour votre propre usage, nous vous fournissons temporairement la clé complète. Cependant, une fois la clé créée, seuls les 8 premiers caractères sont affichés dans l&apos;interface utilisateur des clés API. Si vous devez récupérer la clé complète après sa création, cliquez sur les points de suspension dans l&apos;interface utilisateur des clés API, copiez l&apos;ID de la clé et [utilisez NerdGraph pour interroger la clé complète.](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys/#query-keys)

**Clés utilisateur (pour les autres utilisateurs) :** Lorsque vous créez une clé utilisateur pour un autre utilisateur, vous ne verrez jamais que la version tronquée de la clé dans l&apos;interface utilisateur des clés API et via NerdGraph.

## Recommandations pour la gestion de APIclé [#security-practices]

Si votre clé API tombe entre de mauvaises mains, elle peut présenter un risque de sécurité. Par exemple:

* Quelqu&apos;un avec votre <InlinePopover type="licenseKey" />pourrait envoyer des données arbitraires à votre compte.
* Une personne disposant des clés d&apos;utilisateur d&apos;un membre de votre équipe pourrait consulter vos données New Relic et apporter des modifications à votre compte New Relic.

Vous devez traiter votre clé API de manière sécurisée, comme vous le feriez avec des mots de passe et d&apos;autres informations sensibles. Quelques recommandations :

* Pour la clé de licence et la clé de navigateur, envisagez de mettre en œuvre une stratégie de rotation des clés : créer de nouvelles clés et supprimer les anciennes selon un calendrier défini. Considérations :

  * Vous ne pouvez pas supprimer les clés d&apos;ingestion d&apos;origine associées à un compte. Nous vous recommandons donc de créer des clés d&apos;ingestion supplémentaires que vous pourrez supprimer ultérieurement. Cela garantit que vous adoptez une posture de sécurité solide.
  * Notez que cela ne s&apos;applique pas au jeton de l&apos;application mobile ; vous ne pouvez pas supprimer un jeton ou créer un jeton supplémentaire.

* Pour le <InlinePopover type="userKey" />:

  * Demandez aux membres de votre équipe de conserver leurs clés d’utilisateur en sécurité.
  * Lorsque les membres quittent votre organisation, supprimez leurs identifiants d&apos;utilisateur de New Relic. Cela désactive toutes les clés utilisateur associées à leur ID utilisateur.

## APIrotation de clé [#rotate-keys]

<CollapserGroup>
  <Collapser id="rotate-license-key" title="Faire tourner la clé de licence">
    [Les clés de licence](/docs/apis/intro-apis/new-relic-api-keys/#license-key) sont utilisées pour signaler presque toutes les données (à l&apos;exception des données du navigateur et monitoring des applications mobiles, qui utilisent leurs propres clés). Chaque clé est liée à un compte spécifique et l&apos;utilisateur peut en créer autant qu&apos;il le souhaite. la clé de licence ne peut pas être mise à jour et doit être supprimée et remplacée en créant une nouvelle clé.

    Vous pouvez trouver la clé de licence de votre compte dans l&apos;UI de clé d&apos;API [<DNT>**one.newrelic.com/api-keys**</DNT>](https://one.newrelic.com/api-keys) (ou [<DNT>**one.eu.newrelic.com/api-keys**</DNT>](https://one.eu.newrelic.com/api-keys) pour notre data center de l&apos;UE). Vous ne verrez que les clés que vous êtes autorisé à utiliser, sauf si vous disposez [des autorisations d&apos;administrateur de tous les produits](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) ou d&apos;un rôle personnalisé qui vous permet de voir ces clés.

    Par défaut, tous les comptes New Relic sont créés avec une clé de licence de compte d&apos;origine, qui est partagée par l&apos;ensemble de l&apos;organisation. Vous ne pouvez pas supprimer cette clé de l&apos;UI de clé d&apos;API, mais vous pouvez contacter le New Relic support pour faire pivoter votre clé.

    Pour faire pivoter la clé de licence créée par l&apos;utilisateur :

    1. Connectez-vous en tant qu&apos;utilisateur New Relic dont vous souhaitez faire pivoter la clé.

    2. Accédez à [<DNT>**one.newrelic.com/api-keys**</DNT>](https://one.newrelic.com/api-keys) (ou [<DNT>**one.eu.newrelic.com/api-keys**</DNT>](https://one.eu.newrelic.com/api-keys) pour notre data center européen).

    3. Cliquez sur <DNT>**Create a key**</DNT> dans le coin supérieur droit de la page <DNT>**API keys**</DNT> .

       <img title="Screenshot of API keys list in New Relic" alt="Screenshot of API keys list in New Relic" src="/images/apis-and-data_screenshot-crop_create-key-button.webp" />

    4. Saisissez un nouveau nom de clé, sélectionnez <DNT>**Ingest - License**</DNT> pour <DNT>**Key type**</DNT>, ajoutez une description facultative et cliquez sur <DNT>**Save**</DNT>.

       <img title="Screenshot of create an API page" alt="Screenshot of create an API page" src="/images/apis-and-data_screenshot-crop_create-license-key.webp" />

    5. Mettez à jour tout script ou code faisant référence à l&apos;ancienne clé avec la nouvelle valeur clé. Vous pouvez trouver la nouvelle clé en cliquant sur l’icône <DNT>**...**</DNT> dans la même ligne que la nouvelle clé, puis en cliquant sur <DNT>**Copy key**</DNT>.

       <img title="Screenshot of the copy key options" alt="Screenshot of the copy key options" src="/images/apis-and-data_screenshot-crop_copy-key.webp" />

    6. Une fois que toutes les dépendances externes ont été mises à jour, vous pouvez supprimer en toute sécurité l&apos;ancienne clé en cliquant sur l&apos;icône <DNT>**...**</DNT> dans la même ligne que l&apos;ancienne clé, puis en cliquant sur <DNT>**Delete**</DNT>.

       <img title="Screenshot of delete key option" alt="Screenshot of delete key option" src="/images/apis-and-data_screenshot-crop_delete-key.webp" />

    7. Vous verrez ce message dans le coin inférieur droit lorsque la clé a été supprimée avec succès :

       <img title="Screenshot showing the confirmation of a deleted key" alt="Screenshot showing the confirmation of a deleted key" src="/images/apis-and-data_screenshot-crop_key-deleted.webp" />

       Vous pouvez également gérer la clé de licence via notre API NerdGraph. Pour obtenir des instructions, consultez [le didacticiel NerdGraph : Gérer APIclé](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys/).

## En savoir plus sur notre clé API [#key-details]

Pour créer ou gérer API clé, utilisez l&apos;UI sur [one.newrelic.com/launcher/api-keys-ui.api-keys-launcher](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) ou notre [APINerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys). Pour plus de détails, sélectionnez une clé :

<CollapserGroup>
  <Collapser id="license-key" title="Clé de licence">
    Notre clé principale utilisée pour l’ingestion de données s’appelle la clé de licence. Dans l&apos;UI de clé d&apos;API et dans [NerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-user-keys), cette clé est parfois référencée `ingest - license` comme.

    La clé de licence est requise pour presque toutes les ingestions de données New Relic. Les exceptions sont monitoring des données des navigateurs (qui utilise une clé de navigateur) et monitoring des données des applications mobiles (qui utilise un jeton d&apos;application mobile).

    La clé de licence est une chaîne hexadécimale de 40 caractères associée à un compte New Relic. Lorsque vous vous [inscrivez](/docs/subscriptions/creating-your-new-relic-account) pour la première fois à New Relic, une organisation avec un seul compte et sa propre clé de licence est créée. Si plusieurs comptes sont ajoutés, chaque compte démarre avec sa propre clé de licence. La clé de licence créée à l&apos;origine pour un compte ne peut pas être supprimée, mais vous pouvez créer une clé de licence supplémentaire qui peut être gérée et supprimée, ce qui est utile pour mettre en œuvre des pratiques de sécurité telles que la rotation des clés. Si vous devez faire pivoter la clé de licence d&apos;origine d&apos;un compte, [contactez le support](https://support.newrelic.com).

    Pour empêcher un utilisateur de consulter ou de gérer la clé de licence, attribuez-lui un rôle sans ces autorisations : [modèle utilisateur original](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#add-on) | [modèle utilisateur plus récent](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions).
  </Collapser>

  <Collapser id="browser-key" title="Clé du navigateur">
    Monitoring des navigateurs utilise une clé de navigateur pour rapporter les données, plutôt que la clé de licence. La clé navigateur permet d&apos;associer les données de l&apos;[monitoring des navigateurs agent](/docs/browser/new-relic-browser/getting-started/introduction-browser-monitoring) à votre compte.

    Vous ne pouvez pas gérer ni supprimer la clé de navigateur d&apos;origine créée lors de la création de votre compte, mais vous pouvez créer de nouvelles clés de navigateur et supprimer ces clés. Pour obtenir de l&apos;aide concernant la rotation de la clé de navigateur d&apos;origine d&apos;un compte, [contactez le support](https://support.newrelic.com).
  </Collapser>

  <Collapser id="mobile-app-token" title="Application mobile jeton">
    Monitoring des applications mobiles utilise un jeton d&apos;application mobile pour signaler les données, plutôt que la clé de licence. Consultez [le jeton d&apos;application mobile](/docs/mobile-monitoring/new-relic-mobile/maintenance/viewing-your-application-token) pour plus d&apos;informations.
  </Collapser>

  <Collapser id="user-key" title="Clé d'utilisateur">
    Les clés utilisateur New Relic , parfois appelées « API de clé personnelle », sont nécessaires pour utiliser [NerdGraph](/docs/apis/intro-apis/introduction-new-relic-apis/#graphql) et notre [API REST](/docs/apis/intro-apis/introduction-new-relic-apis/#rest-api).

    Un <InlinePopover type="userKey" />est lié à un utilisateur New Relic spécifique et ne peut pas être transféré. La clé utilisateur vous permet d&apos;effectuer une requête pour tous les comptes auxquels vous avez obtenu l&apos;accès, et pas seulement pour le compte spécifique auquel la clé était associée. Si un utilisateur New Relic est supprimé dans New Relic, ses clés utilisateur sont également désactivées et ne fonctionneront pas.

    Même si elles donnent accès à un utilisateur à plusieurs comptes, les clés utilisateur sont liées à un seul compte spécifique : le compte à partir duquel elles ont été créées. L’importance de ceci est que si un compte est supprimé, toutes les clés utilisateur associées à ce compte ne fonctionneront plus. (De plus, pour notre REST, les appels d&apos;API sont limités au compte associé à cette clé utilisateur.)

    To prevent a user from viewing or managing user keys, [assign them a role without those permissions](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions).
  </Collapser>
</CollapserGroup>
