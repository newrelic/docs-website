<Callout title="fonctionnalité Disponibilité" variant="important">
  <DNT>Fleet Control for Kubernetes</DNT> clusters est en disponibilité générale (GA). La prise en charge de la gestion des agents sur <DNT>Linux and Windows hosts</DNT> est actuellement en version préliminaire publique.

  Pour une liste complète des agents pris en charge et de leurs environnements, consultez notre [documentation sur la compatibilité des types d&apos;agents](/docs/new-relic-control/agent-control/agent-types).

  La fonctionnalité d&apos;aperçu public est fournie conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Le <DNT>Blob Storage API</DNT> est un service <DNT>New Relic</DNT> conçu pour téléverser et gérer des fichiers dans votre compte. <DNT>NerdGraph</DNT> est optimisé pour les requêtes et les mutations de données structurées, c&apos;est pourquoi l&apos;API Blob Storage est utilisée pour les opérations impliquant le transfert et le versionnage du contenu des fichiers.

Dans <DNT>Fleet Control</DNT>, le <DNT>Blob Storage API</DNT> gère les configurations d&apos;agent — en assurant la création, la gestion des versions, la récupération de contenu et la suppression des fichiers de configuration.

<Callout variant="important">
  Les configurations d&apos;agent dans <DNT>Fleet Control</DNT> utilisent le <DNT>**Blob Storage API**</DNT>, et non <DNT>NerdGraph</DNT>. Pour les opérations de flotte, de membre et de déploiement, consultez le **tutoriel<DNT>NerdGraph</DNT> **.
</Callout>

## Prérequis

* Un [compte<DNT>New Relic</DNT> ](https://newrelic.com/signup)avec une clé d&apos;API utilisateur
* Ton <DNT>New Relic Organization ID</DNT>
* Les [permissions](/docs/new-relic-control/fleet-control/overview#required-permissions) appropriées pour gérer les configurations

## Authentification

Toutes les requêtes d&apos;API <DNT>Blob Storage</DNT> nécessitent une authentification à l&apos;aide d&apos;une clé d&apos;API utilisateur <DNT>New Relic</DNT>.

**Générer une clé d&apos;API :**

1. Allez sur [one.newrelic.com](https://one.newrelic.com)
2. Cliquez sur votre nom dans le coin inférieur gauche
3. Sélectionnez **API Keys**
4. Créez une clé **User** (et non une clé Browser ou Licence)

**Inclure dans les en-têtes de requête :**

```
Api-Key: NRAK-YOUR-USER-API-KEY
```

## Point de terminaison de base

```
https://blob-api.service.newrelic.com/v1/e
```

Pour les comptes de la région UE, utilisez :

```
https://blob-api.service.eu.newrelic.com/v1/e
```

    * `NRDOT` - Distribution New Relic du collecteur OpenTelemetry
    * `FluentBit` - Agent de logging Fluent Bit

    * `PipelineControlGateway` - Gateway de contrôle de pipeline
    * `NRApmOperator` - Opérateur APM pour Kubernetes
    * `NReBPFAgent` - Agent eBPF

      -H 'Content-Type: application/x-yaml' \
      -H 'NewRelic-Entity: {"name": "Production Infra Config", "agentType": "NRInfra", "managedEntityType": "HOST"}' \
      -d 'license_key: YOUR_LICENSE_KEY
    log:
      level: info
      forward: true
    integrations:
      - name: nri-docker
        enabled: true'
    ```

        "version": 1
      }
    }
    ```

## Meilleures pratiques

* **Stockez les GUID d&apos;entité :** Enregistrez le `entityGuid` renvoyé par les opérations de création. Vous en aurez besoin pour les opérations de versioning, de récupération et de suppression.
* **Versionnez de manière incrémentielle :** Créez de nouvelles versions pour les modifications de configuration plutôt que de supprimer et de recréer les configurations.
* **Utilisez des noms descriptifs :** Les noms de configuration doivent indiquer clairement leur objectif et leur environnement cible.
* **Valider le YAML :** Assurez-vous que le contenu de votre configuration est un YAML valide avant de le téléverser.
* **Vérifiez la compatibilité du type d&apos;agent :** Assurez-vous que votre type d&apos;agent est compatible avec le type d&apos;entité gérée (HOST ou KUBERNETESCLUSTER).
* **Sécurisez votre clé d&apos;API :** N&apos;exposez jamais votre clé d&apos;API utilisateur dans du code côté client ou des dépôts publics.
* **Vérifiez les codes d&apos;état HTTP :** L&apos;API renvoie 2xx pour les opérations réussies, 404 pour non trouvé, et d&apos;autres codes d&apos;état pour les erreurs.
