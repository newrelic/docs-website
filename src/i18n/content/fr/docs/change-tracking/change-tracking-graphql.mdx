La fonctionnalité suivi des changements vous permet de suivre l'effet de divers changements sur vos clients et votre système. Par exemple, si vous effectuez un déploiement, vous pouvez utiliser la fonctionnalité de suivi des changements pour monitorer les résultats dans les graphiques UI de New Relic. Vous aurez également accès à des fonctionnalités qui vous aideront à comprendre les effets de ces changements.

Au lieu de désigner les changements que vous souhaitez monitorer en utilisant notre CLI ou une intégration CI/CD (intégration et livraison continue), vous pouvez faire la même chose en utilisant des mutations GraphQL et des requêtes via notre API NerdGraph.

Pour des informations générales sur l'utilisation de NerdGraph, voir [Introduction à NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).

## Exigences [#api-requirements]

Les exigences comprennent :

* [Configuration requise pour NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#authentication)
* autorisations utilisateur : tout [type d'utilisateur](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) peut utiliser NerdGraph pour créer un marqueur de déploiement et suivre d&amp;apos;autres modifications, mais seule la plateforme utilisateur complète peut accéder aux expériences d&amp;apos; UI de suivi des changements organisées.

            query
            results {

## Créer un marqueur de suivi des changements à l'aide d'une mutation [#mutation-examples]

Pour enregistrer une modification, telle qu'un déploiement, vous devez inclure les champs obligatoires, puis décider si vous souhaitez inclure des champs facultatifs.

Voici quelques exemples de mutations GraphQL qui montrent quelques façons de créer un marqueur de déploiement :

<CollapserGroup>
  <Collapser className="freq-link" id="simple-mutation" title="Mutation avec seulement les champs obligatoires">
    ```graphql
    mutation {
      changeTrackingCreateDeployment(
        deployment: {
          version: "0.0.1",
          entityGuid: "INSERT_YOUR_GUID"
        }
      )
        {
          deploymentId
          entityGuid
        }
    }
    ```
  </Collapser>

  <Collapser className="freq-link" id="optional-mutation-example" title="Mutation avec champs optionnels">
    ```graphql
    mutation {
      changeTrackingCreateDeployment(
        deployment: {
          version: "0.0.1",
          user: "INSERT_YOUR_NAME",
          timestamp: INSERT_MS_EPOCH_TIMESTAMP,
          groupId: "78641",
          entityGuid: "INSERT_YOUR_GUID",
          description: "create a new marker",
          deploymentType: BASIC,
          deepLink: "example.com/path",
          commit: "INSERT_A_COMMIT_HASH",
          changelog: "test log"
        }
      )
      {
        changelog
        commit
        deepLink
        deploymentId
        deploymentType
        description
        entityGuid
        groupId
        timestamp
        user
        version
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## Et ensuite ? [#whats-next]

Après avoir utilisé des mutations pour désigner les modifications que vous souhaitez suivre, vous pouvez analyser l'effet de ces modifications dans l'UI de New Relic :

* Interrogez vos modifications : vous pouvez écrire votre propre requête GraphQL ou NRQL pour voir les détails des modifications que vous suivez. Pour plus de détails, voir [Requête de modification des données](/docs/change-tracking/change-tracking-view-analyze/#query-deployments).
* Utilisez nos graphiques organisés : pour plus de détails, consultez [Comment afficher et analyser vos modifications dans New Relic](/docs/change-tracking/change-tracking-view-analyze).
