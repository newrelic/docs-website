Vous pouvez accéder à <DNT>Performance Risks inbox</DNT> et l&apos;utiliser pour identifier et résoudre de manière proactive les problèmes de performances des applications avant qu&apos;ils n&apos;affectent vos utilisateurs finaux. Ce guide vous expliquera comment accéder au <DNT>Performance Risks inbox</DNT>, mettre à jour les seuils de détection et utiliser l&apos;outil efficacement dans le serveur MCP New Relic. Pour plus d’informations sur ce qu’est <DNT>Performance Risks inbox</DNT> et ses principaux avantages, consultez l’ [introduction à <DNT>Performance Risks inbox</DNT>](/docs/errors-inbox/performance-risks/introduction).

## Accéder à la boîte de réception des risques de performance [#access]

Vous pouvez accéder à <DNT>Performance Risks inbox</DNT> au niveau du compte et de l&apos;entité :

* [Accès au niveau de l’entité](#access-entity)
* [Accès au niveau du compte](#access-account)

### Accès au niveau de l’entité [#access-entity]

Vous pouvez accéder à <DNT>Performance Risks inbox</DNT> pour une entité spécifique afin de consulter les problèmes de performances liés à cette entité et à ses données télémétriques associées. Cela vous permet de vous concentrer sur la performance d&apos;une application ou d&apos;un service particulier et de comprendre comment cela peut affecter vos utilisateurs finaux.

**Pour accéder à <DNT>Performance Risks inbox</DNT> pour une entité :**

1. Allez à <DNT>[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services</DNT>.
2. Sélectionnez l&apos;entité requise.
3. Dans le volet de navigation de l&apos;entité, cliquez sur <DNT>**Performance Risks**</DNT> dans la section <DNT>**Triage**</DNT>.

### Accès au niveau du compte [#access-account]

Vous pouvez également accéder à <DNT>Performance Risks inbox</DNT> au niveau du compte pour obtenir une vue globale des problèmes de performance sur toutes les entités de votre compte. Cela vous permet d&apos;identifier les problèmes de performance courants qui peuvent affecter plusieurs applications ou services et de prioriser vos efforts d&apos;optimisation en conséquence.

**Pour accéder à <DNT>Performance Risks inbox</DNT> pour un compte :**

1. Allez à <DNT>[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox</DNT>.
2. Dans le volet de navigation de l&apos;entité, cliquez sur <DNT>**Performance Risks**</DNT>.

## Visualiser et analyser les risques de performance [#view-analyze]

Une fois que vous accédez à <DNT>Performance Risks inbox</DNT>, vous pouvez afficher les problèmes détectés de deux manières : la vue <DNT>**Triage**</DNT> et la vue <DNT>**Group performance risks**</DNT>. Les deux vues affichent les mêmes données sous-jacentes mais offrent différentes manières de les analyser.

### Triage [#triage]

La vue <DNT>**Triage**</DNT> traite et regroupe automatiquement les risques de performance similaires. Les requêtes similaires sont calculées, stockées et fusionnées en un seul groupe de risques de performance. Par exemple, plusieurs requêtes qui partagent le même modèle — comme des variations d&apos;une clause `IN` — sont regroupées en une seule entrée plutôt que d&apos;apparaître comme des éléments distincts. Cela permet d&apos;identifier et de prioriser plus facilement les problèmes les plus impactants sans examiner chaque occurrence individuellement.

Utilisez la vue <DNT>**Triage**</DNT> lorsque vous souhaitez obtenir un résumé pré-regroupé des problèmes de performances pour identifier rapidement des schémas dans l’ensemble de votre application.

<img title="Screenshot of the Triage view in Performance Risks inbox showing pre-grouped performance issues." alt="Screenshot of the Triage view in Performance Risks inbox showing pre-grouped performance issues." src="/images/performancerisks_triage.webp" />

### Risques de performance du groupe [#group]

La vue <DNT>**Group performance risks**</DNT> affiche les données brutes de risque de performance sans regroupements précalculés. Les représentations normalisées des requêtes (comme les espaces réservés sous forme de virgule et de point d&apos;interrogation) sont affichées, et vous pouvez filtrer et regrouper les données en fonction de différents attributs, tels que :

* L&apos;utilisateur affecté par le risque de performance
* Le service générant les appels
* L&apos;API déclenchant les requêtes

Utilisez la vue <DNT>**Group performance risks**</DNT> lorsque vous devez analyser les risques de performance en fonction d&apos;attributs spécifiques qui sont pertinents pour votre investigation.

## Définir les seuils [#set-thresholds]

<DNT>Performance Risks inbox</DNT> utilise des seuils configurables pour détecter automatiquement les anomalies de performances. Vous pouvez mettre à jour ces seuils aux niveaux du compte et de l&apos;entité pour vous assurer que les critères de détection correspondent à vos besoins et priorités spécifiques en matière de monitoring des performances. Ces seuils déterminent ce qui constitue un risque de performance et permettent de s&apos;assurer que vous êtes alerté des problèmes de performance les plus pertinents et exploitables dans vos applications.

Vous pouvez définir les seuils de risques de performance au niveau d&apos;un compte ou d&apos;une entité :

* [Seuils au niveau du compte](#account-thresholds)
* [Seuils au niveau de l’entité](#entity-thresholds)

### Définir les seuils au niveau du compte [#account-thresholds]

La définition de seuils au niveau du compte vous permet d&apos;établir des critères de détection des risques de performance par défaut qui s&apos;appliquent à toutes les entités du compte. Cela est utile pour maintenir des normes de monitoring cohérentes et s&apos;assurer que toutes les applications sont évaluées par rapport aux mêmes évaluations de performance.

**Pour définir les seuils au niveau du compte :**

<CollapserGroup>
  <Collapser id="new-account-thresholds" title="Nouveaux seuils au niveau du compte">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox &gt; Performance Thresholds**.</DNT>.
    2. Dans l&apos;onglet **Accounts**, cliquez sur <DNT>**+ New**</DNT>.
    3. Dans la liste déroulante <DNT>**Account**</DNT>, sélectionnez le compte pour lequel vous souhaitez créer un nouveau seuil.
    4. Définissez les limites requises pour chaque analyseur.
    5. Cliquez sur <DNT>**Submit**</DNT> pour enregistrer.
  </Collapser>

  <Collapser id="edit-account-thresholds" title="Modifier les seuils existants au niveau du compte">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox &gt; Performance Thresholds**.</DNT>.
    2. Dans l’onglet **Accounts**, cliquez sur le bouton <DNT>**Edit**</DNT> à côté du compte que vous souhaitez mettre à jour.
    3. Définissez les limites requises pour chaque analyseur.
    4. Cliquez sur <DNT>**Submit**</DNT> pour enregistrer.

    <Callout variant="tip">
      Lorsque vous modifiez des seuils, votre nom d&apos;utilisateur sera affiché pour suivre qui a effectué les modifications.
    </Callout>
  </Collapser>
</CollapserGroup>

### Définir les seuils au niveau de l’entité [#entity-thresholds]

Définir des seuils au niveau de l&apos;entité vous permet de personnaliser les critères de détection des risques de performance pour des applications ou des services spécifiques qui peuvent présenter des caractéristiques ou des exigences de performance uniques. Cela est particulièrement utile pour adapter le monitoring à différents types d&apos;applications, d&apos;environnements ou de priorités métier.

**Pour définir les seuils au niveau d&apos;une entité :**

Vous pouvez définir les seuils pour une entité spécifique de l&apos;une des manières suivantes :

<CollapserGroup>
  <Collapser id="entity-triage" title="À partir de la page Risques de performance">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services**.</DNT>.
    2. Sélectionnez l&apos;entité requise.
    3. Dans le volet de navigation de l&apos;entité, cliquez sur <DNT>**Performance Risks**</DNT> dans la section <DNT>**Triage**</DNT>.
    4. Cliquez sur <DNT>**Adjust performance thresholds**</DNT> dans le coin supérieur droit.
    5. Définissez les limites requises pour chaque analyseur.
    6. Cliquez sur <DNT>**Submit**</DNT> pour enregistrer.

    <Callout variant="tip">
      Lorsque vous modifiez des seuils, votre nom d&apos;utilisateur sera affiché pour suivre qui a effectué les modifications.
    </Callout>
  </Collapser>

  <Collapser id="entity-page" title="Depuis la page Seuils de performance">
    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Errors Inbox &gt; Performance Thresholds**.</DNT>.
    2. Dans l&apos;onglet <DNT>**Entities**</DNT> :
    3. Dans la liste déroulante <DNT>**Account**</DNT>, sélectionnez le compte de votre entité.
    4. Dans la liste déroulante <DNT>**Entity**</DNT>, sélectionnez l&apos;entité que vous souhaitez mettre à jour.
    5. Définissez les limites requises pour chaque analyseur.
    6. Cliquez sur <DNT>**Submit**</DNT> pour enregistrer.

    <Callout variant="tip">
      Lorsque vous modifiez des seuils, votre nom d&apos;utilisateur sera affiché pour suivre qui a effectué les modifications.
    </Callout>
  </Collapser>
</CollapserGroup>

## Utilisez l&apos;outil de boîte de réception Performance Risks dans le serveur MCP New Relic. [#using]

Vous pouvez également accéder à l&apos;outil <DNT>Performance Risks inbox</DNT> (`list_performance_risks()`) dans le [serveur MCP de New Relic](/docs/agentic-ai/mcp/overview/). Cet outil liste tous les risques de performance détectés pour une entité donnée, ainsi que leurs détails tels que le type de risque, la sévérité et le nombre d&apos;occurrences. Vous pouvez utiliser cet outil pour récupérer par programmation des données sur les risques de performance.
