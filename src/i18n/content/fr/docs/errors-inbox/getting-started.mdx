Avec la boîte de réception des erreurs, les équipes DevOps occupées peuvent détecter, trier et prendre des mesures de manière proactive sur toutes les erreurs avant qu&apos;elles n&apos;affectent les clients. Votre équipe peut résoudre les erreurs de l&apos;ensemble de votre stack, y compris toutes les données APM, du navigateur (RUM), mobiles et AWS lambda sans serveur) et de l&apos;hôtel affichées sur un seul écran. Chaque fois qu&apos;une erreur critique ayant un impact sur les clients survient, votre équipe peut être avertie via votre canal de communication préféré, comme Slack. Les erreurs sont regroupées de manière intelligente pour réduire le bruit et garantir que les erreurs critiques sont détectées rapidement et efficacement.

Prêt à commencer ? Assurez-vous d&apos;avoir un [compte New Relic](https://newrelic.com/signup). C&amp;apos;est gratuit, pour toujours !

## Démarrer [#get-started]

Il existe deux façons de commencer à utiliser la boîte de réception des erreurs :

* <DNT>**Scoped view**</DNT>:Accédez à une entité prise en charge : APM, Browser, mobile ou hôtel et commencez immédiatement à visualiser vos erreurs. Aucune configuration requise !
* <DNT>**Global view**</DNT>:Sélectionnez Boîte de réception des erreurs dans le menu de navigation et sélectionnez l’entité qui vous intéresse. Vous commencerez immédiatement à visualiser les groupes d’erreurs pour ces entités. Vous avez la possibilité d&amp;apos;enregistrer cette vue en tant que workload.

### Vue à portée limitée [#scoped-inbox]

Les vues à portée d&apos; [entité](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic) vous permettent d&amp;apos;afficher les erreurs pour une seule [application APM, navigateur, mobile ou OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/opentelemetry-introduction/#benefits).

Vous pouvez accéder à la vue Boîte de réception des erreurs pour une entité spécifique en naviguant vers cette entité dans la vue APM, navigateur, mobile ou OpenTelemetry monitoring et en sélectionnant <DNT>**Errors inbox**</DNT> dans la navigation de gauche.

### Vue globale [#global-inbox]

La plupart des équipes DevOps supervisent un certain nombre de services et conservent ces informations dans une seule boîte de réception, ce qui est plus efficace pour leur workflow. En utilisant la vue globale, vous pouvez afficher les groupes d&apos;erreurs pour plusieurs entités.

Vous pouvez regrouper les entités APM, Browser, Mobile, Serverless et Hotel et enregistrer votre vue organisée en tant que workload en deux étapes simples.

## Créer une workload [#global-workload]

1. Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Errors inbox**</DNT>, cliquez sur <DNT>**Entities**</DNT> et sélectionnez l’entité que vous souhaitez enregistrer en tant workload. Cliquez ensuite sur <DNT>**Save as workload**</DNT>. L&amp;apos;entité que vous avez présélectionnée sera automatiquement ajoutée, et vous pourrez également en ajouter d&amp;apos;autres.

   <img title="Errors inbox for Entities" alt="A screenshot depicting the first step of creating a workload" src="/images/errors-inbox_screenshot-full_create-a-workload-step-1.webp" />

   <figcaption>
     <DNT>
       **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Errors inbox**
     </DNT>
   </figcaption>

2. Une fois que vous avez toutes les entités dont vous avez besoin, cliquez sur le bouton bascule <DNT>**Workloads**</DNT>, puis sur <DNT>**Create a workload**</DNT>.

3. Dans le <DNT>**Create a workload window**</DNT>, donnez un nom workload et sélectionnez l’entité que vous souhaitez inclure. Une fois que vous avez terminé, cliquez sur <DNT>**Create a workload**</DNT>.

   <img title="Errors inbox for Workloads" alt="A screenshot depicting the second step of creating a workload" src="/images/errors-inbox_screenshot-full_create-a-workload-step-2.webp" />

   <figcaption>
     <DNT>
       **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Errors inbox, click Workloads**
     </DNT>
   </figcaption>

4. Vous pouvez désormais visualiser toutes vos entités dans une workload.

### limites d&apos;entité pour la charge de travail dans la boîte de réception des erreurs [#entity-limits]

Veuillez noter : la boîte de réception des erreurs ne peut être utilisée qu&apos;avec une charge de travail contenant 2 000 entités ou moins.
