Les composants orchestrés se composent d&apos;un déploiement complet qui fait tourner le pod de haut en bas selon les besoins. Dans un système Kubernetes , les déploiements individuels ont une configuration spécifique. Par exemple, un déploiement peut exécuter quatre instances d’une application dans un conteneur individuel. Le déploiement lancera le pod contenant cette application jusqu&apos;à ce qu&apos;il atteigne ce quota. Si un pod venait à tomber en panne, il en lancerait un nouveau pour continuer à respecter le nombre spécifié.

Des problèmes surviennent lorsque vous avez des déploiements de dizaines ou de centaines, chacun avec une configuration dont vous ne vous souvenez peut-être même pas.

Les étapes suivantes vous guident à travers une stratégie générale pour monitorer votre déploiement :

<Steps>
  <Step>
    ## Accéder au dashboard de présentation

    Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Kubernetes &amp;gt; Overview Dashboard**</DNT>.
  </Step>

  <Step>
    ## Triez vos déchets

    Jetez un œil à la tuile de déploiement malsain et à la tuile de nombre de déploiements.

    Comparez les deux pour mesurer le pourcentage de déploiement non sain dans votre cluster. Les déploiements malsains ont un pod manquant ou indisponible. Cela signifie généralement que le déploiement n&apos;a pas pu faire tourner ces pods.

    Regardons cela de plus près.

    <img title="tutorialOverviewdashboard" alt="The main overview dashboard for the kubernetes capability" src="/images/tutorial_screenshot-full_overview-deployments.webp" />
  </Step>

  <Step>
    ### Identifier les pods en attente et en échec

    Faites défiler un peu vers le bas et trouvez la table <DNT>**Pending and Failed Pods**</DNT> . Ce tableau vous montrera tous les pods qui ont échoué ou sont bloqués en attente pour une raison quelconque. Il est normal que le pod échoue à un certain degré en fonction de la base de référence de santé de votre système.

    Ce que vous recherchez, c&apos;est un pod qui échoue à plusieurs reprises. Faites défiler le graphique selon vos besoins.

    <img title="tutorialOverviewFailed" alt="The main overview dashboard for the kubernetes capability focusing on the failed pods section" src="/images/tutorial_screenshot-crop_failed-pods.webp" />

    Une fois que vous avez identifié les pods qui échouent régulièrement, dépannez les configurations de déploiement pour ces pods.
  </Step>
</Steps>
