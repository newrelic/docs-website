Un système Kubernetes est intrinsèquement complexe et a la réputation d’être difficile à comprendre. Comprendre Kubernetes en tant que concept est une tâche complexe en soi et vouloir comprendre un système Kubernetes spécifique ne fait qu&apos;ajouter plus de complexité. Comment dépanner des conteneurs éphémères qui tournent de haut en bas avant de pouvoir y accéder ? Comment comprenez-vous la santé de votre système dans son ensemble si vous avez des centaines de conteneurs orchestrés à la fois ? Comment décomposer le système complexe en couches fonctionnelles ?

Dans cette série de tutoriels, vous apprendrez à monitorer votre système Kubernetes avec New Relic et comment il peut rationaliser votre compréhension de Kubernetes dans son ensemble.

<img title="New Relic Navigator" alt="A screenshot showing the New Relic Navigator for Kubernetes. It shows indivudual pods, their health, and operational groups." src="/images/kubernetes_screenshot-full_navigator-main.webp" />

## Configurer New Relic :

Pour commencer, vous devez configurer New Relic avec votre système Kubernetes. Les étapes ci-dessous vous guident tout au long du processus :

<Steps>
  <Step>
    ## Installer l&apos;agent New Relic :

    <img title="New Relic guided install" alt="A screenshot showing guided install flow for Kubernetes in the New Relic UI." src="/images/kubernetes_screenshot-full_guidedInstall.webp" />

    L&apos;intégration de New Relic Kubernetes vous offre une visibilité complète sur la santé et les performances de votre environnement. Grâce aux données fournies, vous pouvez monitorer l&apos;état de santé de l&apos;ensemble de votre cluster Kubernetes, vérifier chaque pod individuellement ou accéder à des services et applications spécifiques. Cet agent collecte les données télémétriques de votre cluster à l&apos;aide de plusieurs intégrations New Relic telles que l&apos;intégration événement Kubernetes , l&apos;agent Prometheus et le plugin New Relic Logs Kubernetes .

    Il existe différentes manières d&apos;intégrer votre système Kubernetes. Pour cette série de tutoriels, nous vous recommandons vivement d&apos;utiliser les étapes d&apos;installation guidée ci-dessous. Pour d’autres chemins d’installation, consultez [notre documentation d’installation de Kubernetes](/install/kubernetes).

  <Step>
    ## Envoyez des données depuis vos applications

    L&apos;exploitation de dizaines ou de centaines de conteneurs entraîne des efforts et des difficultés d&apos;entretien. Les résumés Kubernetes sont regroupés dans des concepts de niveau supérieur, le cluster étant le niveau le plus élevé. Cette abstraction vous aide à comprendre votre cluster dans son ensemble, mais rend difficile la compréhension de ce qui se passe au niveau de l&apos;application.

    Pour réduire cette complexité, vous pouvez signaler les données de vos applications conteneurisées à New Relic. Cela vous permet non seulement de voir l&apos;état de santé de vos applications, mais également de corréler les données d&apos;application à l&apos;infrastructure Kubernetes sous-jacente.

    <img title="apm summary page" alt="A screenshot depicting the APM summary page." src="/images/apm_screenshot-full_apm-complete-summary-page-.webp" />

    [Suivez les instructions de ce document](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes/) pour corréler les données des applications de votre pod au cluster. Vous utiliserez nos fonctionnalités monitoring des performances des applications (APM) pour extraire les données de performances de vos applications et les envoyer à New Relic.
  </Step>

  <Step>
    ## Envoyez des données depuis vos services

    La corrélation de services tels que Cassandra et MySQL avec vos données Kubernetes nécessite une configuration similaire à celle dont vous avez signalé les données d&apos;application.

    <img title="New Relic infra page" alt="A screenshot showing the New Relic infrastructure monitoring UI." src="/images/infrastructure_screenshot-full_infra-main.webp" />

    Pour corréler vos données, consultez notre documentation sur [les servicesmonitoring dans Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/monitor-services/monitor-services-running-kubernetes/). Ce processus utilisera notre intégration sur hôte pour divers services pris en charge à l&apos;aide d&apos;un graphique Helm .
  </Step>

  <Step>
    ## Explorez vos données

    <img width="100%;" title="New Relic - Kubernetes Navigator entity details" alt="New Relic - Kubernetes Navigator entity details" src="/images/kubernetes_screenshot-full_navigator-animated.gif" />

    Pour confirmer que vous envoyez toutes vos données liées à Kubernetes à New Relic, explorez les graphiques Kubernetes. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes**</DNT>. Parcourez vos données et voyez si vous pouvez avoir une idée générale de la santé de votre système Kubernetes. Confirmez que vous voyez les données de l&apos;ensemble de votre cluster, de chaque pod individuel et de tous les services et applications que vous attendez. Une fois que vous êtes prêt, passez aux étapes suivantes.
  </Step>
</Steps>
