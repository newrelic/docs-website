## Présentation

La page Summary offre une vue d&apos;ensemble des performances de lecture vidéo pour une application monitorée spécifique. Cette page sert de tableau de bord principal pour le monitoring de la qualité vidéo, de l&apos;engagement utilisateur et des performances globales de streaming au sein d&apos;une seule application ou plateforme.

### Ce que vous pouvez faire avec Summary

Utilisez cette page pour :

* Monitorer les métriques clés de la qualité vidéo, y compris le taux de rebuffering, le débit binaire, le temps de démarrage et les échecs pour une application spécifique.
* Suivre les métriques d&apos;engagement vidéo, y compris les tentatives, les lectures, les lectures complètes et les échecs
* Identifier les vidéos présentant un temps de démarrage élevé, des lectures incomplètes ou des problèmes de lecture dans l&apos;application
* Analysez les performances vidéo par plateforme, version d&apos;OS, nom du lecteur vidéo, FAI, région et résolution
* Développez les métriques pour afficher des détails approfondis et l&apos;analyse des tendances.
* Ajoutez des widgets aux tableaux de bord pour un monitoring continu des performances
* Créez des conditions d&apos;alerte pour monitorer proactivement les performances de streaming de l&apos;application.
* Générer des prédictions et comparer les tendances de performance à l&apos;aide de requêtes NRQL
* Exporter les métriques sous forme d&apos;images pour le reporting
* Créez de nouvelles entités de streaming directement depuis cette page

### Qui utilise cette page

Cette page est utile pour les développeurs d&apos;applications, les chefs de produit et les équipes opérationnelles qui ont besoin de monitorer la qualité de lecture vidéo pour une application spécifique, d&apos;identifier les problèmes de performances spécifiques à l&apos;application et de s&apos;assurer que l&apos;expérience de streaming répond aux normes de qualité pour cette plateforme.

## Dimensions KPI

Explorez et analysez le trafic en profondeur en utilisant plusieurs paramètres, notamment la plateforme et le nom du lecteur vidéo. Obtenez des informations détaillées sur les habitudes de visionnage des clients, telles que le pourcentage de clients qui regardent un contenu spécifique, la résolution à laquelle les vidéos sont visionnées et des informations détaillées sur la version du système d&apos;exploitation de l&apos;appareil.

* **Plateforme**: le type d&apos;appareil, par exemple, Roku, un navigateur, un appareil Android ou iOS.

* **Version OS**: La version du système d&apos;exploitation de l&apos;appareil. Si vous utilisez un appareil Android ou iOS, la version du système d&apos;exploitation sera affichée.

* **Nom du lecteur vidéo**: Le nom du lecteur vidéo utilisé pour lire la vidéo.

* **FAI**: Le fournisseur de services Internet (FAI) utilisé par le téléspectateur.

* **Région**: La situation géographique du spectateur.

* **Résolution vidéo**: La résolution de la vidéo lue par le spectateur.

## Ajouter une entité de streaming

Par défaut, la page **Entités de streaming** affiche toutes les entités associées au compte. Cependant, si vous souhaitez créer une nouvelle entité de streaming :

1. Cliquez sur **+ Add streaming entity**, dans le coin supérieur droit de la page.
2. Sélectionnez le type d’application que vous souhaitez monitorer. Voir [Installer l&apos;agent vidéo](/docs/streaming-video-&-ads/installation/install-the-streaming-video-&-ads-agent/#installation)
