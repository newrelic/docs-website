Lorsque vous configurez les dashboard <DNT>**Video Overview**</DNT> ou <DNT>**Ad Overview**</DNT>, New Relic enregistre vos paramètres dans votre profil utilisateur. La prochaine fois que vous retournerez sur l&apos;un ou l&apos;autre dashboard, votre configuration sera restaurée afin que vous n&apos;ayez pas à repartir de zéro.

Les préférences sont enregistrées par utilisateur et par type de dashboard. Vos paramètres <DNT>**Video Overview**</DNT> n’affectent jamais vos paramètres <DNT>**Ad Overview**</DNT>, et vice versa.

## Comment les préférences sont enregistrées [#how-saving-works]

Les modifications sont écrites de manière asynchrone en arrière-plan afin que le dashboard reste réactif pendant l’enregistrement. Pour réduire les écritures inutiles lorsque vous effectuez des ajustements rapides, les enregistrements sont brièvement regroupés.

Les préférences enregistrées persistent pour :

* Actualisations du Browser
* Déconnexion et connexion
* Différents navigateurs ou appareils, tant que vous êtes connecté avec le même compte

## Comportement par défaut [#defaults]

Si vous n&apos;avez aucune préférence enregistrée pour un dashboard (par exemple, lors de votre première visite), le dashboard s&apos;ouvre avec la configuration par défaut du produit.

## Réinitialiser aux valeurs par défaut [#reset-to-defaults]

Un contrôle <DNT>**Reset to default view**</DNT> vous permet d&apos;effacer vos préférences enregistrées et de rétablir le dashboard aux valeurs par défaut d&apos;origine du produit.

<Callout variant="tip">
  La réinitialisation n’affecte que le dashboard que vous consultez actuellement. La réinitialisation des préférences <DNT>**Video Overview**</DNT> n’affecte pas vos préférences <DNT>**Ad Overview**</DNT>.
</Callout>

## Gestion des erreurs [#error-handling]

Si la récupération des préférences échoue lors du chargement du dashboard, le dashboard revient à la configuration par défaut du produit et affiche une notification d’erreur. Votre workflow n’est pas bloqué.

Si l’enregistrement d’une préférence échoue, le dashboard continue de fonctionner normalement.

Si une métrique, une dimension ou une facette précédemment enregistrée n&apos;est plus disponible (par exemple, parce qu&apos;elle a été dépréciée), le dashboard substitue une valeur par défaut raisonnable pour ce paramètre, telle que la première option disponible, afin qu&apos;aucun widget ne reste vide ou cassé.
