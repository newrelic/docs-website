Les règles d’obfuscation vous permettent de masquer les données sensibles avant que l’agent ne les transmette à New Relic. Chaque règle est un motif regex associé à une chaîne de remplacement. Les règles sont appliquées à la charge JSON sérialisée avant la transmission.

## Configuration [#configuration]

Passez un éventail `obfuscate` dans la configuration de l&apos;agent :

```js
config: {
  obfuscate: [
    { regex: /\/user\/[^\/?"]+/, replacement: '/user/[REDACTED]' },
    { regex: /token=[^&"]+/, replacement: 'token=[MASKED]' },
  ]
}
```

## Ordre des règles [#rule-ordering]

Les règles sont appliquées séquentiellement — la sortie d’une règle alimente la suivante. Les modèles d’expression régulière non valides sont ignorés avec un avertissement enregistré via `Log.warn`.
