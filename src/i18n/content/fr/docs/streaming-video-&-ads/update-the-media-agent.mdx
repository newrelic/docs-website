To make migrating from earlier versions as easy as possible, we have developed a compatibility layer that translates most of the configurable options chart to their new counterparts. This compatibility layer is temporary and will be removed in the future. We encourage you to read this guide carefully and migrate the configuration under human supervision.

## Migration vers la version 3.xx

Pour découvrir la nouvelle vue organisée, vous devez migrer vers la version 3.xx de l&apos;agent multimédia.

### Complications de type événement double

Il est possible qu&apos;après la migration, certains de vos utilisateurs finaux n&apos;aient pas opté pour la mise à niveau de l&apos;application, ou très peu l&apos;aient fait. Durant cette période intermédiaire, vous pourriez être confronté à des types d&apos;événements doubles, ce qui peut être compliqué. Pour gérer cela :

* Rassemblez toutes les requêtes et modélisations d&apos;actions.
* Informez vos utilisateurs finaux que pendant cette période intermédiaire, ils peuvent voir des données dans plusieurs événements et comment elles sont liées.

<Callout variant="important">
  Ceci s&apos;applique à l&apos;agent mobile. Pour le navigateur, il sera mis à jour automatiquement.
</Callout>

### Affichage des données après la migration

Après la migration, si vous souhaitez voir les données du mois écoulé, nous vous recommandons d&apos;utiliser un dashboard personnalisé ou une requête :

```sql
SELECT * FROM pageAction
```

<Callout variant="important">
  With named event definitions, they loose the dashboard capability for querying the old data.
</Callout>

### événement Définition modélisation

Une modélisation détaillée de chaque ancien type d&apos;événement avec le nouveau est fournie pour vous aider à migrer n&apos;importe quel script ou workflow. Cette modélisation est essentielle pour assurer une transition en douceur et maintenir l’intégrité des données.

We recommend you carefully follow this guide and ensure that all configurations are migrated under supervision. The compatibility layer is a temporary solution, and it is important to transition to the new configuration as soon as possible.

Pour des informations plus détaillées, veuillez contacter le support client de New Relic.
