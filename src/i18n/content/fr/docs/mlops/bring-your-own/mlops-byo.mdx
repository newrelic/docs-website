La solution MLOps de New Relic fournit une bibliothèque Python qui facilite le monitoring de vos modèles d&apos;apprentissage machine. New Relic Bring Your Own (BYO) <DNT>**Machine Learning Model Monitoring**</DNT> est une offre de la suite de produits New Relic qui permet à l&apos;utilisateur de monitorer les performances de ses propres modèles d&apos;apprentissage machine, quelle que soit la plate-forme ou framework qu&apos;il utilise. Cela peut aider l&apos;utilisateur à identifier les problèmes avec ses modèles et à prendre des mesures correctives avant que ces problèmes n&apos;affectent la précision ou les performances de ses applications.

Dans un espace numérique de plus en plus complexe, les équipes de données s&apos;appuient fortement sur des moteurs de prédiction pour prendre des décisions. Le monitoring du modèle d’apprentissage machine New Relic permet à votre équipe de prendre du recul et d’avoir une vue d’ensemble. Notre monitoring des performances des modèles permet à votre équipe d&apos;examiner vos modèles ML pour identifier efficacement les problèmes et prendre des décisions en exploitant les fonctionnalités de base de NR telles que la définition d&apos;alertes sur n&apos;importe quelle métrique et la notification une fois que vos performances ML ont changé

## Facile à intégrer [#integrate]

Le python [ml-performance-monitoring](https://github.com/newrelic-experimental/ml-performance-monitoring) package, basé sur [newrelic-télémétrie-sdk-python](https://github.com/newrelic/newrelic-telemetry-sdk-python)​​, vous permet d&apos;envoyer les valeurs de fonctionnalités et de prédiction de votre modèle, ainsi que des métriques personnalisées, en ajoutant simplement quelques lignes à votre code.

Utilisez le package Python pour envoyer les types de données suivants à New Relic :

* <DNT>**Inference data:**</DNT> Diffusez les fonctionnalités et les valeurs de prédiction de votre modèle. Les données d’inférence sont diffusées sous la forme d’un [événement personnalisé](/docs/data-apis/ingest-apis/introduction-event-api/) nommé InferenceData.
* <DNT>**Data metrics:**</DNT> Au lieu d&apos;envoyer toutes vos données d&apos;inférence brutes, sélectionnez l&apos;option de statistiques agrégées sur la fonctionnalité et les prédictions (par exemple, min, max, moyenne ou percentile). Ils seront envoyés automatiquement sous forme de [métriques](/docs/data-apis/ingest-apis/metric-api/introduction-metric-api/).
* <DNT>**Custom metrics:**</DNT> Calculez vos propres métriques, monitorez les performances de votre modèle ou les données de votre modèle et diffusez-les vers New Relic à l&apos;aide de la fonction record\_metrics. Ils seront envoyés sous forme de [métriques](/docs/data-apis/ingest-apis/metric-api/introduction-metric-api/).

## Afficher les prévisions du modèle [#view-predictions]

En monitoring les performances de vos modèles d’apprentissage machine, vous pouvez identifier les problèmes susceptibles d’avoir un impact sur leur précision ou leurs performances. Vous verrez des informations vitales dans votre dashboard , telles que les prévisions totales de votre modèle et sa distribution au fil du temps. De plus, vous pouvez définir <InlinePopover type="alerts" />pour suivre tous les changements dans les prédictions totales ou les choix hauts/bas.

## Détecter le modèle et dérive des données [#detect-drift]

Avec monitoring du temps réel, vous pouvez rapidement identifier les problèmes avec vos modèles d&apos;apprentissage machine en détectant le modèle et la dérive des données. Le modèle et la dérive des données sont des changements dans votre environnement réel qui peuvent affecter le pouvoir prédictif de votre modèle. En obtenant des données en temps réel sur votre modèle et la dérive des données, vous pouvez rapidement prendre des mesures correctives avant que ces problèmes n&apos;affectent les performances de vos applications.

## Comment commencer monitoring les performances de votre modèle ML [#get-benefits]

Tout d’abord, utilisez la documentation [de démarrage](/docs/mlops/bring-your-own/getting-started-byo/) pour commencer à diffuser les données de votre modèle. Ensuite, cliquez sur <DNT>**Model Performance**</DNT> dans la page <DNT>**All Capabilities**</DNT> de l&apos;interface utilisateur de New Relic (n&apos;oubliez pas de l&apos;épingler) et vous verrez la vue des performances de votre modèle en fonction des données envoyées par votre modèle.
