TruEra comble une lacune critique dans votre stack d&apos;IA, en expliquant et en testant la qualité du modèle tout au long du cycle de vie. Les solutions d&apos;IA de TruEraexpliquent, déboguent et monitorent les modèles d&apos;apprentissage machine, conduisant à une qualité et une fiabilité supérieures, ainsi qu&apos;à un déploiement plus rapide. La solution fonctionne sur l&apos;ensemble du cycle de vie du modèle, est indépendante de la plateforme de développement du modèle et s&apos;intègre facilement dans votre stack d&apos;IA existante.

TruEra inclut une [monitoring des performances des modèles](/docs/alerts-applied-intelligence/mlops/get-started/intro-mlops/) d&apos;intégration avec New Relic pour aider les équipes MLOps à corréler les signaux tout au long du cycle de vie du ML et à aller au-delà de la précision standard, et à monitorer les dérives d&apos;entrée et de sortie.

## Intégrer TruEra à New Relic [#integrate-truera]

### Ajouter un modèle d&apos;apprentissage machine à TruEra [#add-mlm]

TruEra fournit une interface de ligne de commande (CLI) et un client Python pour ingérer le modèle d&apos;apprentissage machine et d&apos;autres données. Les étapes pour ajouter un modèle à l’aide de la CLI incluent la création d’un nouveau projet, la collecte de données, l’ajout de données de formation, le packaging et l’ingestion du modèle et l’ajout de données de production.

Ces étapes peuvent être trouvées sur la [page de documentation technique de TruEra](https://truera.com/newrelicintegration/).

### Flux de données de TruEra vers New Relic [#stream-data]

Chaque fois qu&apos;un nouvel ensemble de données est ingéré dans le système de monitoring Truera, il calcule les métriques pertinentes et les stocke. Ces calculs métriques peuvent être périodiquement poussés vers New Relic à l&apos;aide de l&apos;exportateur de données `truera2newrelic` . Cet exportateur a besoin d&apos;une clé API générée par les consoles Truera et New Relic.

Les instructions suivantes vous montreront comment créer ces clés :

1. **Connectez-vous à votre compte New Relic :** Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; + Add more data**</DNT>.
2. **Cliquez sur TruEra :** Dans la barre de recherche, tapez <DNT>**TruEra**</DNT>.
3. **Sélectionnez un ID de compte :** sélectionnez l’ID de compte avec lequel vous souhaitez intégrer TruEra.
4. **Sélectionnez ou créez une clé API :** Vous avez besoin d&apos;un <DNT>**Active dashboard key**</DNT> pour la gestion du dashboard et d&apos;un <DNT>**Model metrics data key**</DNT> pour l&apos;ingestion des métriques. Ces clés font référence au <InlinePopover type="licenseKey" />. Créez une nouvelle clé ou sélectionnez-en une existante dans les deux sections de clés.
5. **Créer une clé sur TruEra:** connectez-vous à la console TruEra et accédez à la page utilisateur. Cliquez sur <DNT>**Generate Credentials**</DNT> pour créer un <DNT>**Auth token**</DNT>.
6. **Vérifiez l&apos;exécution de l&apos;exportation des données et configurez l&apos;exportateur par lots**: accédez maintenant à [la documentation TruEra](https://truera.com/newrelicintegration/) pour trouver les étapes à suivre pour vérifier la première exécution d&apos;exportation de données à l&apos;aide de commandes dans votre terminal. Ensuite, configurez l’exportateur de données par lots sur une machine virtuelle.

## Afficher le dashboard New Relic prêt à l&apos;emploi à l&apos;aide TruEra [#truera-dashboards]

Pour visualiser les mesures monitoring de l&apos;apprentissage machine, TruEra fournit un dashboard squelette pour monitorer et analyser les problèmes à l&apos;aide de thèmes tels que l&apos;état du modèle, l&apos;exploration détaillée des segments, les valeurs des fonctionnalités, la qualité des données, l&apos;équité, etc.

Ceux-ci <InlinePopover type="dashboards" />peuvent être configurés à l&apos;aide de l&apos;outil dashboard `truera2newrelic`. Pour ce faire, suivez ces étapes :

1. **Vérifiez les métriques de votre modèle**: Depuis <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, accédez au menu de navigation et cliquez sur l&apos;onglet dashboard dans le menu de navigation principal. Vous verrez une configuration dashboard générée automatiquement pour le modèle ML donné. Vérifiez que les métriques du modèle récemment exportées sont visibles.

   <img title="Automatic TrueEra dashboard" alt="Automatic TrueEra dashboard." src="/images/model-performance-monitoring_screenshot-crop_TruEra-dashboard.webp" />

2. **Configurer une notification d&apos;alerte :** une fois que vous avez créé un dashboard, vous pouvez être alerté sur vos données. Pour créer une condition NRQL <InlinePopover type="alerts" />à partir d’un graphique, cliquez sur le widget graphique, puis sur <DNT>**Create alert condition**</DNT>. Une fois que vous avez nommé et personnalisé votre condition, vous pouvez l&apos;ajouter à une politique existante ou en créer une nouvelle.

   <img title="Set up alert notifications" alt="Set up alert notifications." src="/images/model-performance-monitoring_screenshot-crop_TruEra-alerts.webp" />

3. **Soyez averti :** Une fois que vous avez créé une condition d&apos;alerte, vous pouvez choisir comment vous souhaitez être averti. Consultez notre documentation sur [la façon de configurer le canal de notification](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts/).

4. **Corrélez votre incident :** En plus de la notification, vous pouvez utiliser l&apos;intelligence incident pour corréler votre incident. Consultez notre documentation sur [la façon de corréler les incidents à l&apos;aide de décisions](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/change-applied-intelligence-correlation-logic-decisions/).

Vous avez maintenant intégré avec succès New Relic à TruEra. Les alertes nouvellement créées sont désormais corrélées avec vos alertes New Relic, et vous devriez pouvoir voir les données sur les prédictions nouvellement signalées.
