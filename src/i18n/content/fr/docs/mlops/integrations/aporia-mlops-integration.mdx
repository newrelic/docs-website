Aporia est un moyen rapide, simple et sécurisé pour les équipes de science des données de monitorer leurs modèles d&apos;apprentissage machine en production. Avec Aporia, les équipes peuvent créer leur propre outil de monitoring personnalisable en quelques minutes pour recevoir des alertes en direct pour une détection précoce de problèmes tels que la dérive des données, les biais inattendus, les problèmes d&apos;intégrité des données et la dégradation des performances. Aporia dispose également d&apos;une boîte à outils d&apos;investigation permettant une enquête plus approfondie et une analyse des causes profondes. [En savoir plus sur Aporia](https://www.aporia.com).

Aporia inclut une intégration avec New Relic pour fournir une gestion complète du modèle de votre infrastructure MLOps, avec des <InlinePopover type="dashboards" />personnalisés dans New Relic qui montrent une enquête d&apos;inférence.

## Intégrer Aporia à New Relic [#integrate-aporia]

Aporia vous permet de connecter les <InlinePopover type="alerts" />données générées par Aporia le moniteur d&apos; au incident moteur de renseignement sur de New Relic et aux données de prédiction afin de créer un complet monitoring dashboard dans New Relic pour vos modèles.

1. <DNT>**Login into Aporia&apos;s console**</DNT>:Dans la barre de navigation à gauche, cliquez sur <DNT>**Integrations**</DNT> et choisissez <DNT>**New Relic**</DNT>.
2. <DNT>**Login into your New Relic account:**</DNT> Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; +Add more data**</DNT>.
3. <DNT>**Click on Aporia:**</DNT> Dans la barre de recherche, tapez <DNT>**Aporia**</DNT>.
4. <DNT>**Select your account ID:**</DNT> Sélectionnez l&apos;ID de compte avec lequel vous souhaitez intégrer Aporia.
5. <DNT>**Get a New Relic API key:**</DNT> Une fois que vous avez cliqué sur l’icône Aporia, suivez la première étape. Sous <DNT>**Prediction data &amp; model quality alerts**</DNT>, cliquez soit sur <DNT>**Select**</DNT> pour choisir une clé existante, soit <DNT>**Create API key**</DNT>. Cela a créé New Relic <InlinePopover type="licenseKey" />, qui est utilisée pour signaler des données à New Relic.
6. <DNT>**Paste the key in Aporia:**</DNT> Accédez maintenant Aporia au dashboard <DNT>**New Relic insert token**</DNT> d&apos; et sous l&apos;écran d&apos;intégration de New Relic, collez la clé sous. Une fois collé, cliquez sur <DNT>**Save**</DNT>.
7. <DNT>**Verify the key:**</DNT> Dans le Aporia dashboard, cliquez sur le <DNT>**Verify token**</DNT> bouton pour vérifier que la clé fonctionne correctement. Une coche <DNT>**Green**</DNT> ou des marques d&apos;erreur <DNT>**Red**</DNT> doivent apparaître pour indiquer le statut.

## Monitorez vos modèles de machine learning avec Aporia

Maintenant que vous avez intégré New Relic et Aporia, vous pouvez monitorer vos données à l&apos;aide [d&apos;un dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) avec des graphiques automatisés créés par Aporia.

1. <DNT>**Go to the integration dashboard:**</DNT> Une fois que vous avez vérifié votre jeton et confirmé que l&apos;intégration est correctement configurée, revenez au dashboard d&apos;intégration New Relic et cliquez sur <DNT>**See your data**</DNT>. Cela vous redirigera vers un dashboard généré automatiquement affichant les données signalées à Aporia dans New Relic.

2. <DNT>**Analyze Aporia&apos;s dashboard.**</DNT> Aporia dashboard renseigné dans New Relic contient 6 graphiques :

   * Le graphique <DNT>**Most active models**</DNT> affiche les différents modèles qui ont rapporté des prédictions dans la période sélectionnée.
   * Le graphique <DNT>**Most active model versions**</DNT> affiche les différentes versions qui ont rapporté des prédictions dans la période sélectionnée.
   * Le graphique <DNT>**Model inferences**</DNT> affiche le nombre de prédictions uniques signalées pour chaque modèle et version.
   * Le graphique <DNT>**Average numeric inferences**</DNT> affiche les prédictions numériques de valeur moyenne rapportées pour chaque modèle et version.
   * Le graphique <DNT>**Numeric inferences heatmaps**</DNT> affiche un histogramme des prédictions numériques rapportées pour chaque modèle et version.
   * Les graphiques <DNT>**Categorical inferences**</DNT> affichent les différentes valeurs uniques et leurs fréquences de prédictions catégorielles rapportées pour chaque modèle et version.

   <img title="An image of the Aporia dashboard" alt="An image of the Aporia dashboard." src="/images/model-performance-monitoring_screenshot-full_aporia-dashboard.webp" />

3. <DNT>**Filter data:**</DNT> Cliquez sur le bouton <DNT>**…**</DNT> et cliquez sur <DNT>**Edit**</DNT>. Dans la barre de navigation de droite, sous <DNT>**User as filter**</DNT>, activez <DNT>**Filter the current dashboard**</DNT> et cliquez sur <DNT>**Save**</DNT>.

4. <DNT>**Setup alert notifications:**</DNT> Une fois que vous avez créé un dashboard, vous pouvez être alerté sur vos données. Pour créer une condition d&apos;alerte NRQL à partir d&apos;un graphique, cliquez sur le widget graphique, puis sur <DNT>**Create alert condition**</DNT>. Une fois que vous avez nommé et personnalisé votre condition, vous pouvez l&apos;ajouter à une politique existante ou en créer une nouvelle.

   <img title="Set up notifications for Aporia" alt="Set up notifications for Aporia." src="/images/model-performance-monitoring_screenshot-crop_aporia-create-alert.webp" />

5. <DNT>**Get notified:**</DNT> Une fois que vous avez créé une condition d&apos;alerte, vous pouvez choisir comment vous souhaitez être averti. Consultez notre documentation sur [la façon de configurer le canal de notification](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts).

6. <DNT>**Correlate your incidents:**</DNT> En plus de la notification, vous pouvez utiliser l&apos;intelligence incident pour corréler votre incident. Consultez notre documentation sur [la façon de corréler les incidents à l&apos;aide de décisions](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/change-applied-intelligence-correlation-logic-decisions).
