Le serveur MCP de New Relic fournit des outils spécialisés permettant aux agents d&apos;IA d&apos;accéder à vos données d&apos;observabilité et de les analyser. Ce guide de référence répertorie tous les outils disponibles par catégorie.

## Comment utiliser les outils MCP [#how-to-use]

Les outils MCP sont invoqués via une requête en langage naturel dans votre environnement de développement d&apos;IA. Vous n&apos;avez pas besoin de mémoriser les noms exacts des outils ni leurs paramètres. Décrivez simplement ce que vous souhaitez savoir sur vos données New Relic.

**Exemple de requête :**

* « Quelles entités sont présentes sur mon compte ? »
* « Afficher les alertes récentes concernant mon application web »
* «Quelles sont les principales erreurs de la dernière heure ?»
* &quot;Exécutez une requête NRQL pour analyser les performances&quot;

Votre outil d&apos;IA sélectionnera automatiquement les outils MCP appropriés et formatera les résultats pour vous.

## Outils de filtrage avec `include-tags` [#filter-tools]

Certains outils apparaissent dans plusieurs catégories (par exemple, `analyze_deployment_impact` est pertinent à la fois pour `incident-response` et `advanced-analysis`) car ils sont étiquetés avec plusieurs étiquettes pertinentes.

Vous pouvez filtrer dynamiquement les outils que votre agent IA reçoit en envoyant l&apos;en-tête `include-tags` `HTTP` avec vos requests. Cela vous permet de ne demander que des sous-ensembles spécifiques d&apos;outils, ce qui est utile pour réduire le corpus d&apos;outils et améliorer l&apos;efficacité et la précision du modèle d&apos;IA.

Les balises disponibles que vous pouvez utiliser pour le filtrage correspondent directement aux en-têtes de catégorie de cette référence : `discovery`, `data-access`, `alerting`, `incident-response`, `performance-analytics`, ainsi que `advanced-analysis`.

### Exemple d&apos;implémentation [#example]

```json
"servers": {
    "new-relic-mcp-server": {
        "url": "https://mcp.newrelic.com/mcp/",
        "type": "http",
        "headers": {
            "api-key": "NRAK-****",
            "include-tags": "discovery,alerting"
        }
    }
}
```

Dans cet exemple, l&apos;agent d&apos;IA ne se verra présenter que des outils classés dans la catégorie découverte (par exemple, `get_entity`) et alerte (par exemple, `list_alert_policies`), garantissant ainsi que le modèle se concentre uniquement sur les fonctions les plus pertinentes pour sa tâche actuelle.

<Callout variant="important">
  **Sécurité et accès :** Lorsque votre agent d&apos;IA invoque l&apos;un des outils listés ci-dessous (par exemple, `fetch_entity`, `analyze_deployment_impact`, `execute_nrql_query`), l&apos;action qu&apos;il effectue est strictement régie par les autorisations accordées à la clé API utilisateur New Relic ou au profil OAuth qui a été configuré lors de la configuration. L&apos;outil ne peut accéder qu&apos;aux données, entités et comptes auxquels votre compte utilisateur correspondant a obtenu l&apos;accès via le contrôle d&apos;accès basé sur les rôles (RBAC) de New Relic. Si votre clé ne dispose pas des autorisations nécessaires pour une action spécifique (par exemple, la consultation des alertes ou l&apos;exécution de requêtes NRQL), l&apos;appel de l&apos;outil échouera avec une erreur d&apos;autorisation.
</Callout>

## Gestion des entités et des comptes (tag: découverte) [#entity-account]

* `convert_time_period_to_epoch_ms`: Convertir une période de temps (par exemple, « les 30 dernières minutes ») en millisecondes epoch.
* `get_dashboard`: Récupérer les détails d&apos;un dashboard spécifique.
* `get_entity`: Récupérer une entité New Relic par GUID ou rechercher par modèle de nom.
* `list_related_entities`: Répertorie l&apos;entité à 1 saut (liée) d&apos;une entité donnée `GUID`.
* `list_available_new_relic_accounts`: Liste de tous les comptes `IDs` disponibles pour l&apos;utilisateur.
* `list_dashboards`: Lister tous les tableaux de bord d&apos;un compte New Relic.
* `list_entity_types`: Liste du catalogue complet des types d&apos;entités New Relic avec définitions de domaine/type.
* `search_entity_with_tag`: Rechercher une entité à l&apos;aide d&apos;une clé et d&apos;une valeur tag spécifiques.

## Accès aux données (tag : accès aux données) [#data-access]

* `execute_nrql_query`: Exécuter une requête NRQL sur NRDB.
* `natural_language_to_nrql_query`: Exécuter une requête en convertissant une demande en langage naturel en une requête NRQL, en l&apos;exécutant et en renvoyant les résultats.

## Alertes et monitoring (tag: alertes) [#alerts]

* `list_alert_conditions`: Répertorie les détails de la condition d&apos;alerte pour une règle d&apos;alerte spécifique.
* `search_incident`: Répertoriez tous incident d&apos;alerte événement (événements d&apos;ouverture et de fermeture) avec un filtrage flexible.
* `list_alert_policies`: Liste des règles d&apos;alerte pour le compte spécifié, avec possibilité de filtrage par nom de politique.
* `list_recent_issues`: Liste tous les problèmes ouverts dans New Relic pour le compte spécifié.
* `list_synthetic_monitors`: Lister tous les moniteurs Synthétique (tests automatisés qui vérifient la disponibilité et les performances de vos services).

## réponse aux incidents (tag: réponse aux incidents) [#incidents]

* `analyze_deployment_impact`Analyser l&apos;impact d&apos;un déploiement sur les performances d&apos;une entité spécifique.
* `generate_alert_insights_report`Générer un rapport d&apos;analyse des renseignements sur une alerte concernant un problème spécifique.
* `generate_user_impact_report`Générer un rapport d&apos;analyse d&apos;impact pour l&apos;utilisateur final concernant un problème spécifique.
* `get_entity_error_groups`: Récupérer les groupes d&apos;erreurs pour une entité spécifique à partir de la Errors Inbox dans un intervalle de temps donné.
* `list_change_events`: Lister un historique des événements de changement de New Relic pour l&apos;application identifiée par son entité `GUID`.

## Analyse des performances (tag : analyse-des-performances) [#performance]

* `analyze_entity_logs`Analyser les logs d&apos;application pour identifier les schémas d&apos;erreur, les comportements anormaux et les problèmes récurrents.
* `analyze_golden_metrics`: Analyser les métriques dorées (indicateurs clés de santé : débit, temps de réponse, taux d&apos;erreur et saturation).
* `analyze_kafka_metrics`Analyser les métriques Kafka, notamment le délai du consommateur, le débit du producteur, la latence des messages et l&apos;équilibre des partitions.
* `analyze_threads`: Analysez les données métriques du thread pour une entité donnée, y compris l&apos;état du thread, l&apos;utilisation du processeur et la consommation de la mémoire.
* `analyze_transactions`Analyser les transactions d&apos;une entité spécifique sur une période donnée, en identifiant les transactions lentes et sujettes aux erreurs.
* `list_garbage_collection_metrics`: Afficher les métriques de récupération de mémoire et de mémoire pour un compte et une entité donnés.
* `list_recent_logs`: Liste des logs récents de New Relic pour le compte et l&apos;entité spécifiés `GUID`.
