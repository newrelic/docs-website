## Le flux OAuth ne démarre pas

* Assurez-vous que votre navigateur peut accéder à `localhost`.
* Vérifiez les paramètres du pare-feu.
* Vérifiez que vos outils peuvent accéder aux URL `OAuth`.

## La clé API ne fonctionne pas

* Vérifiez le format de la clé (commence par `NRAK-`).
* Vérifiez les autorisations clés dans New Relic.
* Vérifiez que la clé n&apos;a pas expiré.

## Serveur MCP non détecté

* Redémarrez complètement l&apos;application.
* Vérifier la validité de la syntaxe `JSON`.
* Vérifiez les chemins d&apos;accès aux fichiers et les autorisations.

## Problèmes de proxy mcp-remote

* Assurez-vous que `Node.js` est installé.
* Vérifiez votre connexion internet.
* Vérifiez que npx peut accéder au package `mcp-remote`.

## Accès refusé pour les groupes personnalisés

### Problème

Vous ne pouvez pas vous connecter ou vous authentifier auprès du serveur New Relic AI MCP et recevez le message d&apos;erreur suivant : `Missing required capabilities to access MCP server. Please ensure you have the necessary permissions.` Cela se produit généralement si vous êtes membre d&apos;un groupe d&apos;utilisateurs personnalisé sans aucune autorisation au niveau de l&apos;organisation.

### Cause

Votre groupe ne dispose pas d&apos;un rôle à l&apos;échelle de l&apos;organisation lui permettant de lire le serveur MCP. L&apos;accès à cette fonctionnalité à l&apos;échelle de l&apos;organisation doit être explicitement accordé à votre groupe via un rôle à portée organisationnelle.

### Résolution

L&apos;administrateur de votre organisation qui possède le rôle `Authentication Domain Manager` doit mettre en œuvre l&apos;une des deux solutions suivantes pour vous accorder l&apos;accès :

1. Attribuez à votre groupe un rôle avec l&apos;autorisation de lire le serveur MCP :

   * L&apos;administrateur doit ajouter l&apos;un des rôles suivants au groupe personnalisé auquel vous appartenez : `Organization Read Only`, `Organization Manager`, ou `Organization Product admin`.
   * L’administrateur peut créer un [rôle personnalisé à l’échelle de l’organisation](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) et l’attribuer à un groupe auquel vous appartenez via une [autorisation à l’échelle de l’organisation](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#access-grants). Ce rôle personnalisé doit inclure l&apos;autorisation de lecture `New Relic MCP Server`.

2. Vous ajouter à un groupe autorisé :

   * L&apos;administrateur doit vous ajouter à un groupe d&apos;utilisateurs existant qui possède déjà un rôle à l&apos;échelle de l&apos;organisation avec l&apos;autorisation de lire le serveur MCP.

Pour des informations détaillées sur les concepts de gestion des utilisateurs, les rôles et les autorisations d&apos;organisation, reportez-vous à la [documentation de gestion des utilisateurs de New Relic](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts). Si votre organisation utilise un système de gestion des identités interdomaines (SCIM) pour la gestion des utilisateurs, ces ajustements de groupe et de rôle doivent être effectués au sein de votre fournisseur d&apos;identité externe (IdP).

## Messages d’erreur

* **Connexion refusée**: Vérifiez l&apos;URL et la connectivité réseau de vos outils.
* **Authentification requise**: Vérifiez que la clé API est toujours active et qu&apos;elle a été saisie correctement ou terminez le flux `OAuth`.
* **Client invalide**: problème de configuration `OAuth`, essayez le proxy `mcp-remote`.
* **`Time.UnmarshalJSON`**: Utiliser `mcp-remote` comme proxy pour `OAuth`.
