New Relic Files d&apos;attente et flux de données offre une solution complète pour monitoring votre fichier d&apos;attente des messages et données de streaming, fournissant des informations détaillées sur les problèmes potentiels tels que les pannes, le goulot d&apos;étranglement et la limitation. Cette solution fournit des métriques complètes pour le cluster, les partitions, les courtiers et les sujets, permettant une gestion proactive de votre écosystème de fichiers d&apos;attente et de flux de données de messages.

## Premiers pas avec monitoringdes fichiers d&apos;attente et des flux de données

Pour commencer à utiliser les fichiers d&apos;attente et flux de données de New Relic, vous devez d&apos;abord intégrer New Relic à votre écosystème de messages et de files d&apos;attente.

Actuellement, les intégrations suivantes sont disponibles :

* [Streaming géré par Amazon pour Apache Kafka (MSK)](/docs/message-queues-streaming/installation/aws-managed-kafka-msk-integration)
* [Intégration cloud confluente](/docs/message-queues-streaming/installation/confluent-cloud-integration)
