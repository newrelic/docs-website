New Relic stocke différents types de données pendant différentes périodes. La durée de conservation d&apos;un type de données varie en fonction du produit, du niveau d&apos;abonnement et de la fonctionnalité.

## rétention des données UI [#find]

La meilleure façon de comprendre votre conservation des données et de voir quelles options sont disponibles est d&apos;accéder à l&apos;UI. L’une des raisons pour lesquelles cela est recommandé est que votre organisation a peut-être modifié vos paramètres et qu’ils ne correspondent peut-être plus aux paramètres par défaut documentés ici.

La page UI <DNT>**Data retention**</DNT> se trouve dans le [hub de gestion des données](/docs/data-apis/manage-data/manage-your-data). Ajustez les valeurs de rétention en cliquant sur <DNT>**Edit retention**</DNT>.

## Limites de modification et de suppression des données [#edit-limits]

Une fois que vous avez signalé [des données télémétriques](/docs/using-new-relic/data/understand-data/new-relic-data-types) (événement, métriques, log, trace) à New Relic et qu&apos;elles sont disponibles pour l&apos;interrogation, vous ne pouvez plus modifier ou supprimer ces données. Il s’agit d’une décision de conception délibérée qui optimise la vitesse et les performances de New Relic. Les données expireront après la fin de leur [rétention des données](/docs/manage-stored-data) .

Si vous envoyez des données indésirables ou sensibles à New Relic, contactez votre représentant de compte ou obtenez de l&apos;aide sur [support.newrelic.com](https://support.newrelic.com/) pour les supprimer.

    ## intégration unité de calcul événement limites [#event-limit]

    La limite de 2 275 événements d&apos;intégration par unité de calcul par mois est une limite de <DNT>**total**</DNT> événement d&apos;intégration d&apos;infrastructure. Ce n&apos;est pas une limite de 2275 pour chaque événement d&apos;intégration.

    Détails et précisions supplémentaires sur cette limite :

    * Cette limite s&apos;applique à tous les événements de toute intégration (intégration cloud et intégration sur hôte). Les événements sont tous traités de la même manière.
    * Les données par défaut reçues par l&apos;agent d&apos;infrastructure ne sont pas comptabilisées dans la limite de 2275 événements par unité de calcul.
    * Si vous dépassez votre limite, nous n&apos;appliquons pas cette limite. Si vous dépassez votre limite, nous examinerons avec vous les options tarifaires pour garantir que vous obtenez le prix le plus rentable pour l&apos;intégration de votre organisation.

    <CollapserGroup>
      <Collapser id="retention-calculation" title="Comment le nombre d'événements stockés est calculé">
        Ceci est une explication de la manière dont le nombre d&apos;événements stockés est calculé par défaut pour un abonnement Insights Pro . (Notez qu&apos;avec [la conservation flexible](/docs/accounts/accounts/subscription-pricing/flexible-retention-define-how-long-new-relic-retains-event-data), vous avez un contrôle plus précis sur la période de conservation.)

        L&apos;événement stocké est calculé sur la base 1) du nombre total d&apos;événements stockés au fil du temps (calculé sur la base des événements générés par semaine) et 2) des semaines de conservation des données disponibles. Voici comment vous pouvez représenter cette équation :

        ```
        events stored = (events generated per week) * (weeks of retention)
        ```

        Un abonnement Insights Pro fournit un nombre donné de semaines de conservation des données ainsi qu&apos;un nombre donné d&apos;événements sur cette période de conservation.

        Par exemple:

        * (200M de transactions par semaine) \* (4 semaines de rétention) = 800M d&apos;événements stockés dans Insights
        * (16M de transactions par semaine) \* (50 semaines de rétention) = 800M d&apos;événements stockés dans Insights

        Avec l&apos;abonnement Insights Pro , nous purgeons les données en fonction de la fenêtre de conservation et non du volume. Nous le supprimons une fois la fenêtre de conservation dépassée.

        Par exemple : si votre licence Insights couvre 800 millions d&apos;événements avec une période de conservation de 4 semaines, vos données commenceront à être purgées après avoir dépassé quatre semaines. Les pics temporaires de données dépassant votre niveau d&apos;abonnement seront toujours enregistrés, mais les dépassements constants doivent être résolus en augmentant votre niveau d&apos;abonnement ou en diminuant les données collectées.

        Pour les clients sans [abonnementInsights Pro ](#insights-pro), New Relic peut limiter ou sous-échantillonner les événements jusqu&apos;à une limite ne dépassant pas 4 000 événements par hôte et par minute.
      </Collapser>

      <Collapser id="usage-overage" title="Exemple de dépassement d'événement Insights Pro">
        Dans cet exemple, vous disposez d&apos;un abonnement Insights Pro avec une licence pour 800 millions d&apos;événements sur 4 semaines, soit un tarif de 200 millions d&apos;événements par semaine. Vous disposez d&apos;APM Pro, navigateur Pro et Mobile Enterprise. Une cinquième semaine de données est ajoutée via votre abonnement, vous faisant passer à un total de 1 milliard d&apos;événements stockés dans votre forfait :

        * Si vous utilisez 975 millions d&apos;événements, vous <DNT>**are not**</DNT> dépassez votre rétention.
        * Si vous utilisez 1,25 milliard d&apos;événements, vous <DNT>**are**</DNT> dépassez votre rétention.

    <Callout variant="important">
      Notez que les données d&apos;intervalle de temps métrique diffèrent des autres [types de données métriques](/docs/using-new-relic/data/understand-data/new-relic-data-types#metrics).
