La fonctionnalité de session New Relic de vous permet de définir des limites de temps d&apos;inactivité avant l&apos;expiration automatique des sessions de navigateur de votre utilisateur.configuration

## Exigences [#requirements]

Si vous utilisez notre [nouveau modèle d&apos;utilisateur](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models), consultez [Paramètres de session](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more/#session-mgmt).

## Présentation [#overview]

configuration de session vous permet de définir des limites de temps d&apos;inactivité avant l&apos;expiration automatique des sessions de navigateur de vos utilisateurs. Un message apparaît trois minutes avant que le système les déconnecte. l&apos;utilisateur doit ensuite se reconnecter pour continuer. Pour les comptes configurés avec [SAML Single Sign On (SSO)](/docs/subscriptions/saml-service-providers), une option supplémentaire est disponible pour définir la fréquence à laquelle les sessions du navigateur de l&amp;apos;utilisateur sont ré-authentifiées.

utilisateur et utilisateur restreint peuvent voir la période de temps pour l&apos;expiration automatique, mais ils ne peuvent pas la modifier. Pour afficher la valeur du délai d’expiration : accédez à <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) &amp;gt; Account settings &amp;gt; Authentication &amp;gt; Session configuration**</DNT>.

## Sélectionnez la valeur du délai d&apos;expiration de la session [#procedures]

Le processus de sélection de la valeur du délai d’expiration de la session est le même pour la configuration SAML et non SAML. Pour des options de configuration SAML supplémentaires, consultez [Réauthentification du navigateur SAML SSO](/docs/accounts-partnerships/accounts/account-maintenance/set-session-timeouts#saml_reauthentication).

Pour sélectionner une période prédéfinie pour les délais d&apos;expiration de session avec les comptes SSO SAML, le propriétaire du compte doit avoir préalablement identifié l&apos;URL de déconnexion dans les paramètres de configuration SSO SAML. Si cela n&apos;a pas été configuré, l&apos;administrateur du compte peut afficher la barre coulissante du délai d&apos;expiration de la session, mais pas la modifier.

Si l&apos;administrateur est également le propriétaire du compte, le <DNT>**Session configuration**</DNT> inclut un lien pour accéder directement à la configuration SSO SAML de New Relic et identifier l&amp;apos;URL de déconnexion. Pour plus d’informations, voir [Configuration de SSO](/docs/subscriptions/setting-up-sso#timeout_url).

Pour sélectionner une période prédéfinie pour les délais d&apos;expiration de session pour l&apos;utilisateur sur notre [modèle d&apos;utilisateur d&apos;origine](#requirements):

1. Aller à : <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) &amp;gt; Account settings &amp;gt; Authentication &amp;gt; Session configuration**</DNT>.
2. Utilisez la barre coulissante pour sélectionner une période pendant laquelle les sessions inactives expirent et log automatiquement.
3. Facultatif : sélectionnez l&apos;option case à cocher si vous ne souhaitez pas que les sessions de navigateur restreintes aux utilisateurs expirent.
4. Sélectionnez <DNT>**Save my changes**</DNT>.

Les modifications prennent effet immédiatement.

## Sélectionnez la réauthentification du navigateur SAML SSO [#saml\_reauthentication][#saml_reauthentication]

Pour sélectionner une période prédéfinie pour la ré-authentification [des sessions de navigateur authentifiées par SSO SAML](/docs/subscriptions/setting-up-sso) :

1. Aller à : <DNT>**[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) &amp;gt; Account settings &amp;gt; Authentication &amp;gt; Session configuration**</DNT>.
2. Utilisez la barre coulissante <DNT>**SAML re-authentication time**</DNT> pour sélectionner une période pendant laquelle New Relic doit vérifier la session du navigateur.
3. Sélectionnez <DNT>**Save my changes**</DNT>.

## Redirection après expiration du délai SAML [#saml\_timeout][#saml_timeout]

Si vous êtes déconnecté en raison d&apos;un délai d&apos;inactivité de session sur un compte configuré pour SAML, vous serez redirigé vers la page de connexion New Relic . Étant donné que votre compte est configuré pour SAML, vous ne disposez pas d’une connexion New Relic directe. Pour être redirigé vers votre fournisseur SAML pour l&apos;authentification :

1. Saisissez votre adresse e-mail dans le champ <DNT>**Email**</DNT> .
2. Laissez le champ <DNT>**Password**</DNT> vide.
3. Cliquez sur le bouton <DNT>**Sign In**</DNT> .

Vous serez ensuite redirigé vers votre fournisseur SAML. Une fois réautorisé, vous serez alors redirigé vers le site Web de New Relic.
