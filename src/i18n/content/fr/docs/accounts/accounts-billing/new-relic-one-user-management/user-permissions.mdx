Dans le contexte de la gestion des utilisateurs de New Relic, une permission est une tâche spécifique que vous pouvez effectuer avec New Relic. Nos rôles prédéfinis incluent diverses autorisations (par exemple, reportez-vous à nos [rôles standard](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles)). Voici quelques exemples d&apos;autorisations :

* La possibilité d&apos;afficher les paramètres de l&apos;application APM
* Modifier la condition d&apos;alerte
* Gérer les paramètres de rétention des données
* Créer un domaine d&apos;authentification au niveau de l&apos;organisation
* Déployer des flottes individuelles sur des ressources spécifiques

Vous pouvez créer des rôles personnalisés à trois niveaux d&apos;étendue différents et leur ajouter autant d&apos;autorisations que vous le souhaitez :

* **Autorisations à l&apos;échelle de l&apos;organisation**: Pour les fonctions administratives à l&apos;échelle de l&apos;organisation (Gestion des identités et des accès, New Relic One, API clé, sécurité, etc.)
* **Autorisations limitées au compte**: Pour les fonctionnalités de la plateforme au sein de comptes spécifiques (APM, Browser, infrastructure, alertes, etc.)
* **Entity-scoped permissions**: For fine-grained access to specific resources

Pour connaître les autorisations attribuées à un rôle, accédez à l’ UI utilisateur de gestion des utilisateurs et affichez un rôle spécifique. Pour trouver cette UI: Depuis <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, cliquez sur le [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) en bas à droite, puis accédez à : <DNT>**Administration &gt; Access management &gt; Roles**</DNT>.

## Points importants concernant les autorisations [#important-points]

Un utilisateur de la plateforme complète New Relic sans aucune limitation (par exemple, un utilisateur du groupe <DNT>**Admin**</DNT>) peut utiliser toutes les fonctionnalités de la plateforme. La plupart des tâches que vous pouvez effectuer dans New Relic, mais pas toutes, sont disponibles sous la forme <DNT>permissions</DNT>. Vous pouvez les ajouter ou les supprimer d&apos;un rôle personnalisé, et nous utilisons également ces autorisations pour différencier nos [rôles standard](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles). Les autorisations que nous avons rendues visibles et disponibles pour la sélection sont celles que nous pensons les plus susceptibles de vous être utiles pour les tâches courantes de gestion des utilisateurs.

De nombreuses fonctionnalités de New Relic ne sont pas visibles ni disponibles sous forme d&apos;autorisations sélectionnables. Par exemple, il existe diverses pages d&apos;interface utilisateur auxquelles vous pouvez accéder en tant qu&apos;utilisateur et qui ne sont pas soumises aux autorisations que nous exposons.

<Callout variant="tip">
  Les autorisations peuvent également parfois être appelées <DNT>**capabilities**</DNT>.
</Callout>

Voici quelques autres points importants concernant les autorisations :

* <DNT>**A user&apos;s user type must also allow access.**</DNT> L&apos;accès d&apos;un utilisateur à la fonctionnalité New Relic est régi à la fois par le type d&apos;utilisateur et par les rôles attribués. Pour en savoir plus, voir [Accès utilisateur](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).
* <DNT>**Some permissions overlap in functionality.**</DNT> C&apos;est pourquoi la sélection de certaines cases à cocher d&apos;autorisations dans l&apos;UI cochera ou décochera automatiquement d&apos;autres cases.
* <DNT>**Permissions don&apos;t affect querying of data.**</DNT> La plupart des autorisations s&apos;appliquent New Relic UI API aux expériences et et non à l&apos;interrogation des données. Par exemple, si vos autorisations vous empêchent d&apos;accéder à l&apos;UI <InlinePopover type="apm" />, vous pouvez toujours interroger les données APM si vous avez accès à ce compte. Si vous avez besoin de limites de données plus fermes pour certains projets ou utilisateurs, vous pouvez segmenter vos données en [différents comptes](/docs/accounts/accounts-billing/account-structure/add-accounts).

Pour en savoir plus sur les principales façons dont les autorisations des utilisateurs sont contrôlées, consultez [Concepts de gestion des utilisateurs](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Nos rôles pré-établis [#pre-built-roles]

Nos rôles prédéfinis disposent de différents groupes d’autorisations. Voir [en savoir plus sur les rôles](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Définitions des autorisations [#permission-definitions]

Vous pouvez accéder à l&apos;interface utilisateur pour consulter les autorisations de chacun de nos rôles prédéfinis. Dans le coin inférieur gauche de l&apos; [interface utilisateur](https://one.newrelic.com), cliquez sur votre nom pour ouvrir le menu utilisateur, puis accédez à **Administration &gt; Access management &gt; Roles**.

L&apos;interface utilisateur fournit des informations détaillées sur les autorisations pour tous les rôles, notamment :

* Tous les administrateurs de produits
* Utilisateur standard
* Lecture seule
* Tous les rôles personnalisés de votre organisation

L&apos;interface utilisateur de gestion des accès affiche les détails d&apos;autorisation les plus récents pour chaque rôle.
