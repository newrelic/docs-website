Dans un contexte de gestion d&apos;utilisateurs New Relic , une permission est une tâche spécifique que vous pouvez effectuer avec New Relic. Diverses autorisations sont incluses dans nos rôles prédéfinis (par exemple, consultez nos [rôles standards](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles)). Voici quelques exemples d’autorisations :

* La possibilité d&apos;afficher les paramètres de l&apos;application APM
* Modifier la condition d&apos;alerte
* Gérer les paramètres de rétention des données

Vous pouvez créer des rôles personnalisés et leur ajouter n&apos;importe quel nombre d&apos;autorisations.

Pour connaître les autorisations attribuées à un rôle, accédez à l’ UI utilisateur de gestion des utilisateurs et affichez un rôle spécifique. Pour trouver cette UI: Depuis <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, cliquez sur le [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) en bas à droite, puis accédez à : <DNT>**Administration &gt; Access management &gt; Roles**</DNT>.

## Points importants concernant les autorisations [#important-points]

Un utilisateur de la plateforme complète New Relic sans aucune limitation (par exemple, un utilisateur du groupe <DNT>**Admin**</DNT>) peut utiliser toutes les fonctionnalités de la plateforme. La plupart des tâches que vous pouvez effectuer dans New Relic, mais pas toutes, sont disponibles sous la forme <DNT>permissions</DNT>. Vous pouvez les ajouter ou les supprimer d&amp;apos;un rôle personnalisé, et nous utilisons également ces autorisations pour différencier nos [rôles standard](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#roles). Les autorisations que nous avons rendues visibles et disponibles pour la sélection sont celles que nous pensons les plus susceptibles de vous être utiles pour les tâches courantes de gestion des utilisateurs.

Il existe de nombreuses fonctionnalités New Relic que nous ne rendons pas visibles et disponibles sous forme d&apos;autorisations que vous pouvez sélectionner. Par exemple, il existe différentes pages UI auxquelles vous pouvez accéder en tant qu&apos;utilisateur et qui ne sont pas limitées par les autorisations que nous exposons. Prenons un autre exemple : un utilisateur d’un groupe avec <DNT>**Organization**</DNT> [paramètres d’administration](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts#admin-settings) peut configurer les paramètres au niveau de l’organisation et ce pouvoir n’est pas disponible pour l’ajout à un rôle personnalisé.

<Callout variant="tip">
  Les autorisations peuvent également parfois être appelées <DNT>**capabilities**</DNT>.
</Callout>

Voici quelques autres points importants concernant les autorisations :

* <DNT>**A user&apos;s user type must also allow access.**</DNT> L&amp;apos;accès d&amp;apos;un utilisateur à la fonctionnalité New Relic est régi à la fois par le type d&amp;apos;utilisateur et par les rôles attribués. Pour en savoir plus, voir [Accès utilisateur](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).
* <DNT>**Some permissions overlap in functionality.**</DNT> C&amp;apos;est pourquoi la sélection de certaines cases à cocher d&amp;apos;autorisations dans l&amp;apos;UI cochera ou décochera automatiquement d&amp;apos;autres cases.
* <DNT>**Permissions don&apos;t affect querying of data.**</DNT> La plupart des autorisations s&amp;apos;appliquent New Relic UI API aux expériences et et non à l&amp;apos;interrogation des données. Par exemple, si vos autorisations vous empêchent d&amp;apos;accéder à l&amp;apos;UI <InlinePopover type="apm" />, vous pouvez toujours interroger les données APM si vous avez accès à ce compte. Si vous avez besoin de limites de données plus fermes pour certains projets ou utilisateurs, vous pouvez segmenter vos données en [différents comptes](/docs/accounts/accounts-billing/account-structure/add-accounts).

Pour en savoir plus sur les principales façons dont les autorisations des utilisateurs sont contrôlées, consultez [Concepts de gestion des utilisateurs](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Nos rôles pré-établis [#pre-built-roles]

Nos rôles prédéfinis disposent de différents groupes d’autorisations. Voir [en savoir plus sur les rôles](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts).

## Définitions des autorisations [#permission-definitions]

Vous pouvez accéder à l’ UI pour afficher les autorisations pour chacun de nos rôles prédéfinis. Dans le coin inférieur gauche de l&apos;[UI](https://one.newrelic.com), cliquez sur votre nom pour ouvrir le menu utilisateur, puis allez dans **Administration &gt; Access management &gt; Roles**. Des listes d&amp;apos;autorisations sont disponibles pour les rôles suivants :

* Tous les administrateurs de produits
* Utilisateur standard
* Lecture seule
