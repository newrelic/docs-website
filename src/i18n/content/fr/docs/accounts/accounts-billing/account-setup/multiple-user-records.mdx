En tant qu&apos;utilisateur New Relic, vous pouvez avoir plusieurs enregistrements d&apos;utilisateur utilisant la même adresse e-mail. Ce serait le cas si vous aviez :

* Accès à plus d&apos;une [organisation New Relic](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure)
* Accès à plusieurs [domaines d&apos;authentification](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) dans une organisation New Relic

## Comment nous comptons et suivons les utilisateurs [#user-records]

Lorsqu&apos;un utilisateur est créé dans New Relic, cet utilisateur reçoit un ID d&apos;utilisateur. Pour trouver l&apos;ID d&apos;un utilisateur, vous pouvez accéder à l&apos;[UI<DNT>**User management**</DNT> ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks)et rechercher les informations d&apos;un utilisateur spécifique.

<Callout variant="tip">
  Une façon de trouver votre propre ID d&apos;utilisateur est d&apos;inspecter la source du code pour le chargement d&apos;une New Relic UI page et de rechercher cette page `userId` pour.
</Callout>

Pour déterminer le nombre d&apos;utilisateurs dans une organisation New Relic à des fins de facturation, nous comptons les identifiants d&apos;utilisateurs, puis dédupliquons les utilisateurs qui ont les mêmes adresses e-mail.

Lorsque vous mettez à jour votre adresse e-mail dans New Relic, cela n&apos;entraînera pas de double facturation. Si vous mettez à jour votre e-mail et que vous avez plusieurs enregistrements d&apos;utilisateur dans une organisation, assurez-vous de le mettre à jour pour tous les enregistrements d&apos;utilisateur.

## Basculer entre les enregistrements d&apos;utilisateur [#switch]

Lorsque vous vous log à New Relic, vous pouvez accéder au [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) pour basculer vers d&apos;autres enregistrements d&apos;utilisateur ayant la même adresse e-mail. Lorsque vous choisissez un enregistrement d&apos;utilisateur dans le menu, vous serez invité à log pour cet enregistrement d&apos;utilisateur (soit directement à New Relic , soit via votre fournisseur d&apos;identité).

Pour accéder aux enregistrements d&apos;utilisateurs qui ont une adresse e-mail différente, vous devez log déconnecter et log .

## Message « Plusieurs connexions trouvées » [#login-message]

Si votre e-mail est associé à plusieurs enregistrements d&apos;utilisateur, vous verrez un message `Multiple logins found` lors de la connexion et il vous sera demandé de vérifier votre e-mail. Cela peut se produire si vous avez accès à plusieurs organisations New Relic ou à plusieurs [domaines d&apos;authentification](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more) dans une organisation.

<img title="Login UI for email addresses with multiple user records" alt="A screenshot of what is shown when you have an email address associated with multiple New Relic logins" src="/images/accounts_screenshot-crop_login-with-multiple-accounts-found.webp" />

<figcaption>
  Lorsque vous avez deux enregistrements d&apos;utilisateur ou plus pour la même adresse e-mail, vous recevrez un message à ce sujet lors de la connexion.
</figcaption>

Lorsqu&apos;un enregistrement d&apos;utilisateur est marqué comme `Original` sur la page de connexion, comme indiqué ci-dessus, cela signifie que cet enregistrement d&apos;utilisateur est sur notre [modèle d&apos;utilisateur d&apos;origine](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models).

La vérification de votre adresse e-mail vous permettra de choisir à quel enregistrement d&apos;utilisateur accéder. Si vous ne vérifiez pas votre adresse e-mail, New Relic essaiera de vous log avec l&apos;enregistrement d&apos;utilisateur le plus récemment créé associé à votre adresse e-mail.

Pour éviter de devoir vérifier votre email à chaque connexion, cochez la case <DNT>**Remember me**</DNT> sur l&apos;écran de connexion.

Pour en savoir plus sur les problèmes de connexion, consultez [Résoudre les problèmes de connexion](/docs/accounts/accounts-billing/account-setup/troubleshoot-new-relics-password-email-address-login-problems).

## Supprimer les enregistrements d&apos;utilisateur [#delete]

Si vous vous êtes inscrit accidentellement à New Relic, consultez [Supprimer une organisation](/docs/accounts/accounts-billing/account-setup/downgradecancel-account/#cancel-unused-org).

Si vous souhaitez supprimer un enregistrement d&apos;utilisateur, utilisez l&apos;[UI de gestion des utilisateurs](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks).
