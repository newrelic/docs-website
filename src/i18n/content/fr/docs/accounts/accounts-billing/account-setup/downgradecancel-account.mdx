En fonction de votre édition tarifaire et d&apos;autres facteurs, nous proposons différentes options pour réduire l&apos;ingestion de données, supprimer ou désinstaller un agent ou une intégration, ou rétrograder ou annuler votre [organisation New Relic](/docs/accounts/accounts-billing/new-relic-one-pricing-users/new-relic-account-structure).

## Réduire ou arrêter l&apos;ingestion de données [#remove-tools]

Si vous souhaitez arrêter de signaler certaines données à New Relic sans rétrograder ou annuler, vous pouvez configurer nos outils pour envoyer moins de données.

Pour savoir comment gérer l’ingestion de données, consultez [Gérer vos données](/docs/telemetry-data-platform/get-data-new-relic/manage-data/manage-your-data).

Pour désinstaller l&apos;agent ou l&apos;intégration, voici quelques procédures recommandées :

* [Supprimer les applications <InlinePopover type="apm" />, <InlinePopover type="browser" />et<InlinePopover type="mobile" />](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/)
* [Supprimer l&apos;agent d&apos;infrastructure](/docs/infrastructure/install-infrastructure-agent/update-or-uninstall/uninstall-infrastructure-agent/)
* Pour savoir comment désactiver d’autres outils New Relic, consultez leurs documents spécifiques. Vous pouvez [rechercher des solutions New Relic ici](https://newrelic.com/instant-observability/).

## Rétrograder votre édition [#downgrading]

Il existe deux options pour rétrograder votre [édition](https://newrelic.com/pricing), selon que vous êtes en mode paiement à l&apos;utilisation ou non :

<CollapserGroup>
  <Collapser id="payg-downgrade" title="Organisation à paiement à l'utilisation">
    Pour rétrograder une organisation [prépayée](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#usage-plans) Standard ou Pro :

    1. Dans le [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), cliquez sur <DNT>**Manage your plan**</DNT>.
    2. Cliquez sur <DNT>**Plans and pricing**</DNT>.
    3. Sous les options tarifaires <DNT>**Free**</DNT> , passez la souris sur <DNT>**Downgrade to free**</DNT> et cliquez sur le bouton qui apparaît.
    4. Dans la fenêtre qui apparaît, faites défiler vers le bas et entrez un motif d’annulation.
    5. Cochez la case indiquant que vous acceptez les conditions de service.
    6. Cliquez <DNT>**Submit.**</DNT>
  </Collapser>

## Annuler un compte [#cancel-account]

Si vous avez plusieurs comptes dans votre organisation et que vous ne souhaitez pas tous les fermer pour le moment, vous pouvez annuler un ou plusieurs comptes tout en conservant votre organisation.

Lorsque vous annulez un compte :

* Il est désactivé, mais il est toujours présent dans votre organisation dans **Accounts &gt; Cancelled accounts**.
* L&apos;utilisateur, les groupes et les rôles personnalisés restent, mais les autorisations d&apos;accès attribuées à ce compte annulé perdent toutes les connexions au compte. Vous devez également supprimer toute autorisation d&apos;accès associée à ce compte, sinon l&apos;utilisateur peut voir des erreurs lorsqu&apos;il essaie de le visualiser.
* Toute clé API associée à un compte annulé devient inactive et ne peut pas être réaffectée à un autre compte.

### Qui peut annuler un compte [#who-cancel]

Pour annuler un compte, vous devez être un utilisateur qui est :

* Un utilisateur de plateforme complète.
* Dans un groupe qui a le [cadre administratif](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings) de l&apos;organisation.

### Comment annuler un compte [#how-to-cancel]

Assurez-vous que vous êtes un utilisateur qui peut annuler un compte, puis vous pouvez annuler un compte via notre UI ou via notre API Nerdgraph.

**Annuler un compte :**

1. Pour annuler un compte via notre UI: accédez à **Administration** &gt; **Access Management** &gt; sélectionnez l’ **Accounts Tab** &gt; sélectionnez les points de suspension à droite du compte que vous souhaitez annuler &gt; sélectionnez **cancel account**.

2. Pour annuler un compte via notre API NerdGraph : consultez le [didacticiel](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph) NerdGraph .

## Supprimer votre organisation [#delete-org]

Outre l&apos;annulation de comptes individuels, vous pouvez également supprimer l&apos;intégralité de votre organisation New Relic. Cela nécessite une certaine préparation, alors passez en revue les étapes ci-dessous.

### Préparation à la suppression d&apos;une organisation [#preparation-org-delete]

Avant d&apos;essayer de supprimer votre organisation, veuillez confirmer les éléments suivants :

* Votre organisation n&apos;a qu&apos;un seul compte actif. Si votre organisation possède plusieurs comptes actifs :

  * Annulez des comptes supplémentaires à l&apos;aide de notre interface utilisateur ou [de l&apos;API Nerdgraph](/docs/apis/nerdgraph/examples/manage-accounts-nerdgraph) (voir [Annuler un compte](#cancel-account)).
  * Le dernier compte actif restant sera votre « identifiant de compte de rapport » pour votre organisation. Ce compte sera automatiquement annulé lorsque l&apos;organisation sera supprimée.

* Votre organisation ne doit avoir qu&apos;un seul utilisateur restant.
  * Si vous avez plus d&apos;un utilisateur, vous pouvez supprimer l&apos;utilisateur via l&apos; [interface de gestion des utilisateurs](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/#delete-users) ou via [Nerdgraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-users/#delete-users).

* Votre organisation ne dispose que d’un seul domaine d’authentification.
  * Si votre organisation possède plusieurs domaines d&apos;authentification, vous pouvez supprimer le domaine d&apos;authentification [directement dans l&apos;interface utilisateur](https://forum.newrelic.com/s/hubtopic/aAXPh0000006Wd7OAE/delete-authentication-domains) ou [avec Nerdgraph](https://forum.newrelic.com/s/hubtopic/aAXPh000000664LOAQ/api-endpoint-to-delete-authentication-domains).

* Votre organisation utilise notre nouveau modèle tarifaire.

  * Si vous n&apos;êtes pas sûr du modèle que vous utilisez, consultez [Modèles de tarification](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-pricing-models/#determine-pricing) pour savoir comment le déterminer.
  * Si vous utilisez l&apos;ancien modèle de tarification, contactez [le support](/docs/using-new-relic/welcome-new-relic/get-started/find-help-use-support-portal).

* Votre organisation doit être sur l&apos;édition gratuite.
  * Si vous disposez d&apos;un abonnement payant actif, consultez la section [Rétrograder votre édition](#downgrading) de ce document.

### Comment supprimer votre organisation [#how-to-delete-org]

Si vous remplissez les [conditions](#delete-org) pour supprimer votre organisation, procédez comme suit :

1. Dans le [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), cliquez sur <DNT>**Administration**</DNT>.
2. Cliquez sur <DNT>**Access management**</DNT>.
3. Cliquez sur <DNT>**Delete organization**</DNT>. Si vous ne voyez pas cette option, assurez-vous d&apos;avoir rempli les conditions ci-dessus. Pour une aide supplémentaire, contactez [le support](/docs/using-new-relic/welcome-new-relic/get-started/find-help-use-support-portal).
