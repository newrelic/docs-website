[New Relic](https://one.newrelic.com) est une puissante plateforme d&amp;apos;observabilité qui vous donne accès à toutes vos données sur l&amp;apos;ensemble de votre système. Nous hébergeons les données télémétriques envoyées par [votre entité](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/), c&amp;apos;est-à-dire essentiellement tout ce que nous pouvons identifier et qui contient des données que vous pouvez monitorer, y compris les applications, les services, les hôtes, etc. À vous de choisir !

Bien que nous vous fournissions une expérience prête à l&apos;emploi pour voir vos données avec [un dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) organisé, vous pouvez personnaliser l&amp;apos;accès à vos données et personnaliser la visibilité de plusieurs manières, notamment [dans l&apos;interface utilisateur](#query-ui) ou [via l&apos;API](#query-apis).

<Callout variant="important">
  Pour mieux comprendre vos données stockées dans New Relic, consultez [Types de données](/docs/using-new-relic/data/understand-data/new-relic-data-types).

## Parcourez vos données dans l&apos;interface utilisateur [#browse-data]

Notre plateforme propose plusieurs expériences qui ne nécessitent pas de connaissance de NRQL ou de tout autre langage de requête :

* Pour explorer vos données New Relic, vous pouvez utiliser [métriques et événement](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer), un navigateur de données intuitif pour explorer les données et créer des visualisations. Vous pouvez également utiliser le [générateur de requêtes](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) pour affiner votre requête.
* Pour explorer vos données trace , vous pouvez utiliser la [requête de tracing distribué](/docs/understand-dependencies/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one).
* Pour explorer les logs, vous pouvez utiliser l&apos;[interface utilisateur des logs](/docs/logs/ui-data/use-logs-ui/).

## requête de données dans l&apos;interface utilisateur [#query-ui]

Si vous êtes prêt à faire plus que parcourir des données, devenez un acteur à part entière et personnalisez votre requête dans l&apos;interface utilisateur de New Relic. Utilisez des langages de requête, notamment notre [langage de requête New Relic](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/) ou [notre langage de requête de style PromQL](https://prometheus.io/docs/prometheus/latest/querying/basics/), pour modifier les requêtes avec une flexibilité totale. Par exemple, vous pouvez ajouter davantage [de clauses`WHERE` ](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/#sel-where), modifier la valeur renvoyée, passer à [d&apos;autres types de visualisations](/docs/query-your-data/explore-query-data/use-charts/chart-types/), etc.

<Callout variant="tip">
  Êtes-vous nouveau dans l’interrogation des langages ? Commencez à parcourir les données dans [métriques et événement](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer), puis tournez-vous vers le [générateur de requêtes](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) pour voir la requête que vous avez construite et affinez-la.
</Callout>

Il existe deux manières d&apos;écrire votre propre requête pour récupérer des données et créer des graphiques :

* [générateur de requêtes en mode NRQL ](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data): requête utilisant [le langage de requête New Relic](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) (NRQL), le même langage que nous utilisons pour créer la plupart de nos expériences d&amp;apos;interface utilisateur et la manière la plus avancée d&amp;apos;interroger des données dans New Relic.
* [générateur de requêtes en mode style PromQL](/docs/query-your-data/explore-query-data/chart-builder/use-advanced-promql-mode-specify-data): Écrire une requête en utilisant une requête de style PromQL.

Vous pouvez également [interroger des données depuis votre IDE](/docs/codestream/observability/query-builder/) à l&amp;apos;aide de l&amp;apos;extension CodeStream de New Relic.

## requête de données via API [#query-apis]

Lorsque vous avez besoin d&apos;options de programmation ou que vous souhaitez bénéficier de davantage de puissance d&apos;interrogation, vous pouvez utiliser des API pour interroger vos données New Relic. Notre API NerdGraph vous offre des options plus puissantes non disponibles dans l&apos;interface utilisateur, telles que l&apos;interrogation de comptes et l&apos;exportation d&apos;ensembles de données historiques à plus long terme. Pour plus d&apos;informations, voir [Introduction à NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#tutorials).
