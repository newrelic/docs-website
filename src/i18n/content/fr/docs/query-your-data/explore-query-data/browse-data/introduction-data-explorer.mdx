Naviguez visuellement dans toutes vos données à l&apos;aide du <DNT>**Data Explorer**</DNT> sans avoir besoin de savoir comment écrire [une requête au formatNRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language). L&apos;explorateur de données fait partie de la console de requête, accessible via le tiroir de requête en bas de la plateforme New Relic.

Besoin d&apos;effectuer une recherche plus détaillée ? Découvrez comment [interroger vos données](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/). Si vous ne l&apos;avez pas déjà fait, créez votre compte New Relic gratuit ci-dessous pour commencer monitoring vos données dès aujourd&apos;hui.

## Pourquoi c&apos;est important [#why-matters]

Recherchez-vous l&apos;état de votre système ? Avez-vous besoin de planifier des ressources, d’identifier et de répondre à un incident ou de résoudre des comportements défectueux ? L&apos;explorateur de données permet d&apos;identifier, de récupérer et de visualiser facilement les données que vous recherchez via des menus visuels, sans jamais utiliser NRQL ni créer de requête.

Avec le <DNT>**Data Explorer**</DNT> vous pouvez accéder à toutes les données stockées dans la base de données New Relic (NRDB) de manière rapide et intuitive. Vous pouvez ensuite sélectionner <DNT>**Dimensions**</DNT> pour effectuer un facettage selon des dimensions spécifiques de ces données.

Autres choses que l&apos;explorateur de données vous aide à faire :

* Consultez les données sous différents points de vue : des données brutes aux différentes visualisations qui fournissent des informations détaillées sur l&apos;évolution, la distribution, etc.
* Explorez les données en détail à l&apos;aide de filtres.
* Ajoutez des prédictions aux graphiques linéaires et à aires (disponible avec l&apos;aperçu public des [prédictions NRQL](/docs/query-your-data/explore-query-data/use-charts/nrql-predictions)).
* Ajoutez vos recherches à un [dashboard](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards).
* Comprendre comment fonctionne [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) : l&apos;explorateur de données montre comment les requêtes sont construites.

## Types de données que vous pouvez explorer [#data-types]

Dans le contexte de <DNT>**Data Explorer**</DNT>, `event` fait référence à tous les objets stockés dans la base de données New Relic (NRDB). L&apos;explorateur de données vous permet d&apos;explorer :

* Métriques:

  * [Métriques dimensionnelles](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) (le type de données `Metric` ).
  * [données d&apos;intervalle de temps métrique](/docs/data-apis/understand-data/new-relic-data-types/#timeslice-data): les agents <InlinePopover type="apm" />, les agents mobiles et l&apos;agent <InlinePopover type="browser" />signalent ce type de données. Pour explorer ce type de données, vous devez choisir un moniteur d&apos;entité par l&apos;un de ces agents, et vous verrez alors ces options.

* événement : le [type de données événement](/docs/data-apis/understand-data/new-relic-data-types/#events-new-relic).

* log : le type de données `Log` (que vous pouvez également explorer via notre [UI de logs](/docs/logs/ui-data/use-logs-ui)).

* tableau de recherche : Données qui ne sont pas présentes dans votre compte New Relic, en téléchargeant des fichiers CSV (que vous pouvez également explorer via notre [interface utilisateur de tableau de recherche](/docs/logs/ui-data/lookup-tables-ui)).

## Explorez vos données [#explore-data]

Pour accéder à l&apos;explorateur de données, cliquez sur le bouton <DNT>**Query your data**</DNT> dans le tiroir inférieur de n&apos;importe quelle page New Relic, puis sélectionnez l&apos;option <DNT>**Data Explorer**</DNT>.

L&apos;explorateur de données se compose des éléments suivants :

* Section de portée en haut. Vous pouvez sélectionner le compte et choisir parmi plusieurs types de données : `events`, `metrics`, `timeslices`, `logs` et `lookups`. Si vous sélectionnez des métriques, vous pouvez filtrer par entité.
* Zone de navigation des données sur la gauche. Si vous choisissez `metrics`, les options sont `metric` et `dimensions`. Si vous choisissez `events`, les options sont `event type`, `plot` et `dimensions`.
* Espace de travail. Cette zone vous montre le résultat de vos choix.

Pour utiliser l’explorateur de données :

1. Définir la portée. Sélectionnez le type de données (métrique ou événement), le compte et l&apos;entité.

2. Sélectionnez la plage horaire à l&apos;aide du [sélecteur d&apos;intervalle de temps](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker).

3. Utilisez les blocs sur la gauche pour parcourir les données disponibles et construire votre recherche. Vous ne pouvez sélectionner qu&apos;un seul élément par bloc. Les blocs sont consultables.

Ici, vous pouvez facilement lire le résultat de votre exploration sous forme de requête NRQL et vérifier les données exactes tracées.

    Si vous n&apos;êtes pas familier avec NRQL, reportez-vous à la [documentation NRQL](https://docs.newrelic.com/docs/query-your-data/nrql-new-relic-query-language).

## Cas d&apos;utilisation [#uses-cases]

Consultez les exemples suivants pour savoir comment et quand utiliser l’explorateur de données.

<CollapserGroup>
  <Collapser className="freq-link" id="use-case-validate-data" title="Découvrez vos nouvelles données">
    <DNT>
      **I&apos;ve just connected new instrumentation and want to see if new data is available.**
    </DNT>

    1. Sélectionnez le compte et l’événement ou la métrique qui génère les nouvelles données.
    2. Utilisez les différents outils de l&apos;Explorateur de données pour explorer les nouvelles données devenues disponibles : regardez les données brutes de cet événement ou de cette métrique sous forme de tableau, façonnez-les sous forme de liste ou cliquez pour les voir tracées sous forme de graphique.
    3. Après avoir sélectionné un événement ou une métrique, découvrez la forme des données dans ses dimensions. Guidé par la cardinalité, vous pouvez voir les différents points de vue de n&apos;importe quelle donnée.
    4. Vous avez trouvé quelque chose de pertinent ? Enregistrez-le dans un dashboard ou partagez-le avec un collègue.
  </Collapser>

  <Collapser className="freq-link" id="use-case-new-data" title="Validez vos données">
    <DNT>
      **I changed a custom event/metric and need to check if this change has been successful.**
    </DNT>

    1. Dans l&apos;explorateur de données, sélectionnez le compte, le type de données et l&apos;événement/métrique auxquels vous avez apporté des modifications.
    2. Vérifiez que l’entité signale des données et que tous les attributs sont tracés.
    3. Recherchez l’attribut auquel vous avez apporté des modifications et vérifiez que la mise à jour a réussi.
  </Collapser>

  <Collapser className="freq-link" id="nrql-prediction-use-case" title="Afficher la prédiction sur un graphique linéaire ou en aires (disponible avec l'aperçu public des prédictions NRQL)">
    <Callout title="Aperçu">
      Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

      Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
    </Callout>

    <DNT>
      **I want to see the future trend of a line or area chart to take proactive measures based on the predicted data.**
    </DNT>

    1. Accédez à l’explorateur de données en cliquant sur le bouton <DNT>**Query your data**</DNT> dans le tiroir inférieur de n’importe quelle page New Relic.
    2. Sélectionnez un graphique linéaire ou en aires auquel vous souhaitez ajouter une prédiction.
    3. Dans le menu <DNT>**Options**</DNT> , sélectionnez <DNT>**Predict trend**</DNT>. Le graphique avec la prédiction s&apos;ouvre dans le <DNT>**Data Explorer**</DNT>.
    4. Enregistrez-le dans un dashboard.
  </Collapser>

  <Collapser className="freq-link" id="use-case-troubleshooting" title="dépannage problèmes de données">
    <DNT>
      **I know there&apos;s something off with an event/metric from an alert or dashboard. I need to know the root cause about the event/metric/attribute behavior.**
    </DNT>

    1. Dans l&apos;explorateur de données, utilisez les menus pour sélectionner l&apos;événement ou la métrique qui ne se comporte pas comme prévu et laissez l&apos;explorateur de données tracer ce graphique.
    2. À partir de là, vous pouvez explorer en détail les dimensions de ces données et filtrer par les attributs pertinents.
    3. Vous pouvez également voir ces données sous différents angles : leur distribution, le classement des valeurs ou leur évolution dans le temps.
    4. Vous avez trouvé quelque chose de pertinent ? Enregistrez-le dans un dashboard ou partagez-le avec un collègue.
  </Collapser>
</CollapserGroup>
