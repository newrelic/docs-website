Dans New Relic, vous pouvez partager des dashboards et des graphiques avec des utilisateurs extérieurs à votre organisation, même s&apos;ils n&apos;ont pas de compte New Relic. Cette fonctionnalité élargit l&apos;audience de vos informations détaillées tout en maintenant les contrôles de sécurité et les capacités d&apos;audit. Le dashboard et les graphiques partagés sont accessibles via des URL en direct accessibles au public que vous créez à partir de la plateforme New Relic.

Lorsqu&apos;un utilisateur externe accède au dashboard ou aux graphiques partagés, la page s&apos;affiche avec les mêmes données et visualisations que l&apos;entité d&apos;origine, avec les fonctionnalités interactives suivantes :

* **Plage de temps**: les spectateurs peuvent choisir la plage de temps dans une liste déroulante pour voir les données pour une période spécifique.
* **Mode d&apos;affichage**: les spectateurs peuvent sélectionner le mode clair ou sombre selon leurs préférences.
* **Mode TV**: Les téléspectateurs peuvent activer un mode TV plein écran qui optimise le dashboard pour l&apos;affichage sur un écran de télévision.

<Callout variant="important">Alors que la fonctionnalité de partage de graphiques est disponible pour les clients **Pro**, **Enterprise** et **Core Compute** , la fonctionnalité de partage dashboard est disponible uniquement pour les clients Calculer **Pro** et **Enterprise** avec **[Advanced Compute](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing)** activé. Pour en savoir plus sur les coûts associés et les pratiques de gestion de la fonctionnalité de partage de dashboard, reportez-vous à la section [Tarification](#public-dashboard-pricing) .</Callout>

<Callout variant="caution">Toute personne disposant des URL dashboard ou du graphique en direct peut afficher toutes les informations du dashboard ou du graphique. Partagez les informations avec soin et conformément aux politiques et procédures internes de votre entreprise.</Callout>

## Prérequis [#prerequisites-public-dashboard]

Cette fonctionnalité comprend plusieurs couches et conditions de sécurité pour garantir un partage de données contrôlé et sécurisé. Pour partager un dashboard ou des graphiques publiquement, assurez-vous que la configuration suivante est en place. Reportez-vous à la section [Sécurité et gestion des accès](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts) pour plus de détails sur la configuration :

* L&apos;autorisation `Live URL control` est activée pour tous les comptes approvisionnant le dashboard.
* Le rôle et les autorisations personnalisés sont configurés pour vous permettre de partager le dashboard et les graphiques.
* Les comptes à l&apos;origine du dashboard ou du graphique disposent d&apos;autorisations pour que votre rôle puisse partager des données en externe.
* Vous avez accès à tous les comptes alimentant le dashboard ou le graphique.
* Vous êtes le propriétaire du dashboard ou du graphique que vous souhaitez partager.
* L&apos;[autorisation dashboard ](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-permissions)n&apos;est pas définie sur <DNT>**Private**</DNT>.

      ## Partagez votre dashboard [#share-dashboard]

      Vous pouvez partager votre dashboard en générant une URL de dashboard en direct et en gérant sa durée d&apos;expiration. De plus, vous pouvez consulter la liste des URL générées avec des détails et les révoquer à tout moment depuis la plateforme New Relic. Pour garantir la sécurité et la conformité, monitorez la piste d&apos;audit pour la création, la mise à jour ou la révocation des URL dashboard partagées.

      <Callout variant="tip">Vous pouvez créer et gérer des URL dashboard en direct via l&apos;**UI** et l&apos;[API NerdGraph](/docs/apis/nerdgraph/examples/manage-live-dashboard-urls-via-api).</Callout>

      <CollapserGroup>
        <Collapser id="create-dashboard-url" title="Créer une URL dashboard en direct à partir de l'UI">
          Pour créer une URL en direct accessible au public pour votre dashboard:

          1. Allez à <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT>.

          2. Ouvrez le dashboard que vous souhaitez partager.

          3. Dans le coin supérieur droit, cliquez sur l’icône <DNT>**Share**</DNT> .

          4. Sélectionnez <DNT>**Share dashboard**</DNT>. Un message de confirmation s&apos;affiche pour accuser réception des éléments suivants :

             * Le dashboard ne contient pas d&apos;informations sensibles.
             * L&apos;accès à ce dashboard partagé à partir de l&apos;URL en direct à chaque fois peut consommer votre CCU.

          5. Cliquez sur <DNT>**Continue**</DNT>. La boîte de dialogue <DNT>**Share dashboard**</DNT> apparaît.

          6. Attribuez une durée à laquelle l’URL doit être active. L&apos;URL expire après la durée spécifiée.

          7. Cliquez sur <DNT>**Save changes**</DNT>. Une notification apparaît concernant la création réussie de l&apos;URL et elle est copiée dans votre presse-papiers.

          8. Pour afficher la page dashboard partagé, cliquez sur la notification ou accédez-y manuellement dans un nouvel onglet avec l&apos;URL copiée.

          <Callout variant="important">
            Toutes les mises à jour de contenu apportées au dashboard actuel sur la plateforme New Relic sont automatiquement reflétées sur la page dashboard partagé. Pour éviter toute exposition accidentelle des données, seul le propriétaire dashboard peut mettre à jour un dashboard partagé publiquement.
          </Callout>
        </Collapser>

        <Collapser id="update-live-url" title="Mettre à jour l'URL dashboard en direct">
          Assurez-vous que vous disposez des [autorisations](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#permissions-to-custom-role) nécessaires et que vous êtes le propriétaire du dashboard pour mettre à jour les propriétés (durée de vie) du dashboard partagé.

          1. Allez à <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT>.
          2. Ouvrez le dashboard que vous partagez.
          3. Cliquez sur l&apos;icône <DNT>**Share**</DNT> .
          4. Sélectionnez <DNT>**Manage public link**</DNT>. La boîte de dialogue <DNT>**Share dashboard**</DNT> apparaît.
          5. Mettez à jour la durée à partir de la liste déroulante <DNT>**Link expiration**</DNT> . L&apos;option <DNT>**Never expires**</DNT> maintient l&apos;URL active pendant 30 ans.
          6. Cliquez sur <DNT>**Save changes**</DNT>.
        </Collapser>

        <Collapser id="view-dashboard-links" title="Afficher le dashboard partagé">
          Vous pouvez accéder aux noms de dashboard partagés hyperliés à leurs URL actives, le tout au même endroit sur New Relic One. En fonction de vos [autorisations](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#permissions-to-custom-role), vous pouvez visualiser soit les URL que vous avez créées, soit celles créées par tous les utilisateurs du compte.

          Pour afficher les liens dashboard partagé :

          1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
          2. Dans la navigation de gauche, cliquez sur votre nom en bas.
          3. Accédez à <DNT>**Administration &gt; Shared Dashboards**</DNT>. La page Dashboards partagés fournit une liste des dashboards partagés, y compris leurs dates de création et d&apos;expiration, ainsi que le propriétaire de chaque dashboard.

          Vous pouvez cliquer sur les noms dashboard pour ouvrir les pages de partage dans un nouvel onglet.
        </Collapser>

        <Collapser id="revoke-dashboard-url" title="Révoquer une URL dashboard en direct">
          En fonction de vos [permissions](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts/#permissions-to-custom-role), vous pouvez révoquer soit les liens que vous avez créés, soit ceux créés par tous les utilisateurs du compte. Vous pouvez le faire de la manière suivante :

          * Sur le dashboard que vous partagez, cliquez sur l’icône <DNT>**Share**</DNT> et sélectionnez <DNT>**Manage public link**</DNT>. Dans la boîte de dialogue <DNT>**Share dashboard**</DNT> , sélectionnez <DNT>**Revoke public link**</DNT>.
          * Depuis la page <DNT>**[Shared Dashboards](#view-dashboard-links)**</DNT>, sélectionnez le <Icon name="fe-more-horizontal" /> icône pour le lien dashboard et cliquez sur <DNT>**Revoke**</DNT>.

          <Callout variant="tip">
            Seul le propriétaire peut supprimer un dashboard avec un lien partagé publiquement. La suppression du dashboard révoque automatiquement le lien de partage public. Si vous restaurez le dashboard, vous devez créer une nouvelle URL en direct.
          </Callout>
        </Collapser>

        <Collapser id="audit-trail-dashboard" title="Piste d'audit et de logging">
          La création, la mise à jour ou la révocation d&apos;URL dashboard partagées génère une piste d&apos;audit dans [NrAuditEvent](https://docs.newrelic.com/docs/accounts/accounts/account-maintenance/query-account-audit-logs-nrauditevent/). Pour afficher les enregistrements dans le générateur de requêtes :

          1. Allez à <DNT>**[one.newrelic.com &gt; Query Your Data](https://one.newrelic.com/nr1-core)**</DNT>.
          2. Sélectionnez le compte à partir duquel les liens dashboard public sont créés.
          3. Exécutez l’une des requêtes suivantes pour afficher les pistes d’audit :

          <CollapserGroup>
            <Collapser id="url-last-hour" title="Obtenez des détails sur les URL dashboard en direct créées ou révoquées au cours de la dernière heure.">
              Pour obtenir les détails des URL dashboard en direct créées ou révoquées au cours de la dernière heure, exécutez la requête suivante :

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard.%'
              ```
            </Collapser>

            <Collapser id="url-created-last-hour" title="Obtenez des détails sur les URL dashboard en direct créées au cours de la dernière heure.">
              Pour obtenir les détails des URL dashboard en direct créées au cours de la dernière heure, exécutez la requête suivante :

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard.create%'
              ```
            </Collapser>

            <Collapser id="url-revoked-last-hour" title="Obtenez des détails sur les URL dashboard en direct révoquées au cours de la dernière heure.">
              Pour obtenir les détails des URL dashboard en direct révoquées au cours de la dernière heure, exécutez la requête suivante :

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard.revoke%'
              ```
            </Collapser>

            <Collapser id="url-from-specific-user" title="Obtenez les détails des URL dashboard en direct créées par un utilisateur spécifique au cours de la dernière heure.">
              Pour obtenir les détails des URL dashboard en direct créées par un utilisateur spécifique au cours de la dernière heure, remplacez `<user_email>` par l&apos;ID de messagerie électronique réel de l&apos;utilisateur et exécutez la requête suivante :

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard%'
              AND actorType='user'
              AND actorEmail=<user_email>
              ```
            </Collapser>

            <Collapser id="url-in-account" title="Obtenez toutes les URL dashboard en direct créées dans un compte au cours de la dernière heure.">
              Pour obtenir les détails des URL de dashboard en direct créées dans un compte au cours de la dernière heure, remplacez `<account_id>` par l&apos;ID de compte réel et exécutez la requête suivante :

              ```sql
              FROM NrAuditEvent
              SELECT *
              WHERE actionIdentifier LIKE '%public_dashboard%'
              AND scopeType='account'
              AND scopeId=<account_id>
              ```
            </Collapser>

            <Collapser id="url-count" title="Obtenez la liste des dashboards à partir desquels les URL en direct sont générées au cours d'une période spécifique.">
              Pour obtenir la liste des dashboards à partir desquels les URL en direct sont créées au cours d&apos;une période spécifique, exécutez la requête suivante. Il affiche le nombre d&apos;URL créées pour chaque dashboard.

              <Callout variant="tip">
                L&apos;exemple de requête suivant récupère les enregistrements de la semaine dernière. Vous pouvez ajuster la période selon vos besoins.
              </Callout>

              ```sql
              WITH aparse(description, '%for dashboard *') AS dashboardGUID
              FROM NrAuditEvent
              SELECT count(*) AS 'dashboards'
              WHERE actionIdentifier LIKE '%public_dashboard.create%'
              FACET dashboardGUID
              SINCE 1 week ago
              ```
            </Collapser>

            <Collapser id="operations-count" title="Obtenez la liste des opérations effectuées avec les URL dashboard en direct au cours d'une période spécifique.">
              Pour obtenir la liste des opérations effectuées avec les URL dashboard en direct au cours d’une période spécifique, exécutez la requête suivante.

              <Callout variant="tip">
                L&apos;exemple de requête suivant récupère les enregistrements de la semaine dernière. Vous pouvez ajuster la période et la facette selon vos besoins.
              </Callout>

              ```sql
              WITH aparse(description, '%for dashboard *') AS dashboardGUID,
              aparse(actionIdentifier, '%public_dashboard.*') AS publicDashOperation
              FROM NrAuditEvent
              SELECT uniqueCount(actionIdentifier) AS 'number of operations'
              WHERE publicDashOperation IS NOT NULL
              FACET actorEmail, dashboardGUID, publicDashOperation
              SINCE 1 week ago
              ```
            </Collapser>
          </CollapserGroup>
        </Collapser>

        <Collapser id="public-dashboard-pricing" title="Tarifs">
          Lorsque vous utilisez des URL dashboard en direct, il est important de comprendre les coûts associés et comment les gérer efficacement. <DNT>**Public Dashboards**</DNT> est une fonctionnalité du produit Advanced Compute proposée en tant que [module complémentaire](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/add-on-billing) ou dans le cadre du modèle de tarification Compute. [Les CCU](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#compute-capacity-unit) avancés sont consommés lorsque des requêtes sont exécutées à partir des visualisations dashboard , ce qui se produit dans les cas suivants :

          * Au chargement de la page
          * On sélecteur d&apos;intervalle de temps change
          * À propos des déclencheurs de taux de rafraîchissement

          Vous pouvez monitorer vos coûts d&apos;utilisation depuis **[New Relic Administration](https://one.newrelic.com/admin-portal) &amp;gt; Compute Management &amp;gt; Advanced capabilities**.

          ### Pratiques de gestion des coûts

          Pour maintenir une tarification optimale, tenez compte des pratiques standard suivantes :

          * [**Contrôle d&apos;accès**](/docs/query-your-data/explore-query-data/dashboards/prerequisites-to-share-dashboards-charts): gérez qui peut créer, modifier ou supprimer des liens au niveau du compte et du rôle d&apos;utilisateur. Cela permet de contrôler et d’empêcher l’utilisateur d’avoir un accès excessif à ces fonctionnalités.

          * [**Gestion de l&apos;expiration des liens**](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#update-live-url): définissez des délais d&apos;expiration pour les liens afin de garantir qu&apos;ils ne sont visibles que pendant une certaine période, réduisant ainsi la surutilisation potentielle.

          * **Gestion des widgets**:

            * Ajoutez ou supprimez un widget de votre dashboard interne pour optimiser les données interrogées.
            * Mettre à jour le widget pour ignorer les changements du sélecteur d&apos;intervalle de temps, empêchant ainsi les requêtes supplémentaires lorsque l&apos;utilisateur ajuste la plage de temps.
            * Gérez le taux de rafraîchissement de chaque widget pour éviter les actualisations automatiques, en fonction de vos préférences dashboard interne. Pour plus de détails, reportez-vous à [Personnaliser les taux de rafraîchissement des graphiques](/docs/query-your-data/explore-query-data/use-charts/chart-refresh-rates/#customize-chart-refresh).
        </Collapser>
      </CollapserGroup>

      ### Limites

      Voici les limites du partage public du dashboard :

      * Lorsque vous consultez un dashboard partagé dans <DNT>**TV mode**</DNT>, il reste actif jusqu&apos;à 72 heures. Le spectateur doit actualiser le navigateur pour continuer à visionner.
      * Si vous partagez un dashboard avec un widget qui utilise des variables de modèle, des liens de facettes, des visualisations personnalisées ou des tables log , ces widgets n&apos;afficheront aucune information dans la vue partagée.
      * Si le paramètre d&apos;autorisation du dashboard est <DNT>**Private**</DNT>, vous ne pouvez pas partager le dashboard publiquement. Pour vérifier cela, accédez à <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT> &amp;gt; votre dashboard &amp;gt; <Icon name="fe-more-horizontal" /> icône &amp;gt; <DNT>**Settings &gt; Permissions**</DNT>.
    </TabsPageItem>

      ## Partagez votre graphique [#share-chart-externally]

      Pour partager un graphique avec un utilisateur externe, assurez-vous d&apos;avoir effectué toutes les étapes de configuration décrites dans les [conditions préalables](#prerequisites-public-dashboard). Pour générer un lien partageable pour votre graphique :

      1. Allez à <DNT>**[one.newrelic.com &gt; Dashboards](https://one.newrelic.com/dashboards)**</DNT>.
      2. Ouvrez le dashboard dans lequel le graphique est disponible.
      3. Dans le coin supérieur droit du graphique, cliquez sur le <Icon name="fe-more-horizontal" /> icône.
      4. Sélectionnez <DNT>**Get chart link**</DNT>. Une notification apparaît confirmant la création réussie du lien public, qui est également copié dans votre presse-papiers.
      5. Pour ouvrir la page du graphique en direct, cliquez sur la notification ou parcourez-la manuellement dans un nouvel onglet avec l&apos;URL copiée.
