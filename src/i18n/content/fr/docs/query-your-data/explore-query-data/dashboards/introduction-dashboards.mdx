Grâce à notre UI, vous pouvez créer et partager des graphiques et des visualisations personnalisés et conviviaux. Accédez à notre dashboard pour collecter et représenter graphiquement les données spécifiques que vous souhaitez voir, de la manière dont vous souhaitez les voir, depuis n&apos;importe où sur notre plateforme.

## Pourquoi c&apos;est important [#dashboards-features]

Avec notre dashboard, vous pouvez personnaliser et comprendre les données que vous collectez. Explorez vos données et corrélez les sources connectées avec des graphiques et apprenez rapidement l&apos;état de votre système et de vos applications pour un dépannage plus rapide et plus efficace.

Utilisez le dashboard pour :

* Générez des informations détaillées avec des visualisations interactives personnalisées à haute densité avec une UI cohérente.
* Cartographiez tous les événements et attributs de partout sur notre plateforme. Pour plus d&apos;informations, voir [Collecte de données](/docs/data-apis/understand-data/new-relic-data-types/).
* [Ajoutez des attributs personnalisés](/docs/data-apis/custom-data/custom-events/apm-report-custom-events-attributes/#steps) ou [envoyez des types d&apos;événements personnalisés](/docs/data-apis/ingest-apis/event-api/introduction-event-api/#workflow) à la plupart des événements afin de mieux comprendre votre entreprise et de voir des détails spécifiques sur la façon dont vos clients interagissent avec votre plateforme, tels que les pages vues, les transactions de l&apos;hôte, etc.
* [Gérez facilement vos graphiques et dashboards](#get-started) à l&apos;aide de nos menus CRUD à accès rapide et de nos options d&apos;édition.
* Explorez et contextualisez les données avec [des info-bulles avancées et des fonctions de zoom](#key-visual-tools) pour monitorer ce que fait votre système en temps réel.
* Recherchez dans votre dashboard des attributs et des mesures.
* [Signalez des données personnalisées](/docs/data-apis/custom-data/get-custom-data-from-any-source/) à New Relic et à votre dashboard à partir de n&apos;importe quelle source.
* Partagez le dashboard avec les parties prenantes extérieures à votre organisation New Relic via [des URL dashboard en direct accessibles au public](/docs/query-your-data/explore-query-data/dashboards/share-charts-dashboards-externally/#dashboard).

## Démarrer avec les dashboards [#get-started]

Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT> et cliquez sur <DNT>**Dashboards**</DNT>.

Dans l&apos;index du dashboard, vous pouvez afficher toutes les applications de dashboard et de données associées à votre compte New Relic. Vous pouvez également consulter le [dashboard prédéfini populaire](/docs/query-your-data/explore-query-data/dashboards/prebuilt-dashboards/).

<img title="Dashboards index" alt="Dashboards index" src="/images/dashboards_screenshot-full_dashboard-index.webp" />

<figcaption>
  Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT> pour ouvrir l’index du dashboard.
</figcaption>

Pour chaque dashboard, l&apos;index affiche les informations suivantes :

* [Statut favori](#dashboards-favorite), indiqué par une étoile
* Nom : Le nom du dashboard
* Compte : le compte auquel appartient le dashboard
* Créé par : L&apos;utilisateur qui a créé le dashboard
* Dernière modification : Quand le dashboard a été modifié pour la dernière fois
* Créé le : Lorsque le dashboard a été créé

Ici, vous pouvez effectuer les actions suivantes :

## Autorisations du dashboard [#dashboards-permissions]

* **Association de compte**: Chaque dashboard est une ressource qui appartient au compte dans lequel il a été créé. Cette association ne change pas. Si le compte auquel il appartient est annulé, aucun tableau de bord ne sera plus accessible.
* **Propriété de l&apos;utilisateur**: L&apos;utilisateur qui crée un dashboard en est le propriétaire. La propriété détermine qui a le droit de gérer les autorisations du dashboard.

Par exemple, si l&apos;utilisateur propriétaire est supprimé, le dashboard n&apos;est pas supprimé du compte auquel il appartient. Il persiste avec ses dernières autorisations connues, mais comme le propriétaire est parti, ces autorisations ne peuvent plus être gérées ou modifiées. La solution à ce problème serait de [dupliquer le dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-clone) et l&apos;utilisateur qui le fera sera le propriétaire du doublon. Le dashboard qui a été dupliqué peut ensuite [être supprimé](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/#dashboards-delete).

Dashboards ont trois types d&apos;autorisations :

* <DNT>**Edit - everyone in account**</DNT>:Tous les utilisateurs ont tous les droits sur le dashboard.
* <DNT>**Read-only - everyone in account**</DNT>:Tous les utilisateurs peuvent voir le dashboard, mais vous seul disposez de tous les droits pour travailler avec le dashboard. D&apos;autres utilisateurs peuvent accéder au dashboard mais ne peuvent pas le modifier ou le supprimer, bien qu&apos;ils puissent le dupliquer.
* <DNT>**Private**</DNT>:Vous seul pouvez voir le dashboard. Tout sauf les métadonnées est masqué.

Lorsque vous [créez un dashboard](#dashboards-create-new) en utilisant le bouton <DNT>**Create a dashboard**</DNT> ou en [dupliquant](#dashboards-clone) un autre dashboard, celui-ci aura les droits <DNT>**Edit - everyone in account**</DNT> par défaut. Plus tard, vous pourrez les modifier à partir du [menu des paramètres](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-settings) une fois que vous aurez accédé au dashboard.

## Créer et gérer un dashboard via API [#nerdgraph]

Vous pouvez utiliser [notre API NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-dashboards) pour créer et gérer un dashboard. Notre API vous donne accès à plus de fonctionnalités que l&apos;UI: par exemple, vous pouvez créer un dashboard avec des données provenant de plusieurs comptes.

Vous pouvez également [ajouter une balise avec NerdGraph](/docs/new-relic-solutions/new-relic-one/core-concepts/use-tags-help-organize-find-your-data/#add-tags). Vous pouvez filtrer votre dashboard à l&apos;aide de balises, ce qui peut être utile pour identifier les utilisateurs, les comptes, les emplacements, etc.
