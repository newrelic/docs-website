Les blocs de requêtes sont le composant principal des notebooks, vous permettant d&apos;écrire des requêtes NRQL et de visualiser les résultats dans votre document. Vous pouvez ajouter plusieurs blocs de requêtes à un seul notebook pour construire un récit de données complet.

## Bloc actif

Le bloc actif est l&apos;unité interactive actuellement sélectionnée dans votre bloc-notes. Il vous sert d&apos;espace de travail immédiat pour écrire du code, exécuter des requêtes ou ajouter de la documentation.

Le bloc actif est immédiatement visible, ce qui permet de le repérer facilement dans votre bloc-notes. Elle se distingue par deux indices visuels clés :

* **Bordures transparentes**: Le bloc actif est mis en évidence par une bordure de couleur distincte et transparente.
* **Barre d&apos;outils visible**: C&apos;est la seule cellule qui affiche les actions.

Les requêtes provenant de [Data Explorer](https://docs.newrelic.com/whats-new/2024/11/whats-new-11-11-data-explorer/) sont ajoutées au bloc actif si celui-ci est de type requête. Si le bloc actif est actuellement au format Markdown, un nouveau bloc de requête est ajouté.

Pour modifier un bloc actif, vous devez sélectionner ou ajouter un autre bloc dans le bloc-notes.

Lorsqu&apos;un bloc Markdown est actif, il passe automatiquement en mode édition avec une mise en page à deux colonnes :

* **Colonne d&apos;édition (gauche)**: C&apos;est ici que vous saisissez et mettez en forme votre code Markdown.
* **Colonne d&apos;aperçu en direct (à droite)**: Cette colonne affiche un aperçu visuel en temps réel de votre documentation formatée.

## Barre d&apos;outils Action

Pour vous aider à optimiser votre espace de travail et à conserver une vue dégagée, l&apos;interface des ordinateurs portables comporte une barre d&apos;outils d&apos;actions. Il regroupe les actions de requête principales dans un menu compact à base d&apos;icônes et regroupe les commandes secondaires dans un menu Actions dédié.

### Actions principales (Haut du bloc de requête)

Ces actions essentielles et fréquemment utilisées sont accessibles en haut du bloc de requête :

* **Exécuter la requête**: Cette fonction est désormais représentée par une icône de lecture et exécute la requête en cours.
* **Requêtes récentes**: Permet d&apos;accéder rapidement à vos 10 dernières requêtes exécutées lors de votre session actuelle et de les réutiliser.
* **Requête recommandée**: Fournit des suggestions de requête.
* **Copier le lien de la requête**: Génère un lien partageable vers la requête actuelle.

### Menu Actions (Commandes secondaires)

Le menu Actions contient des commandes supplémentaires, moins fréquemment utilisées ou liées à l&apos;organisation.

* **Ajouter une autre requête**: Insère un nouveau bloc de requête vide dans votre bloc-notes.
* **Supprimer la requête**: Supprime la requête des TIMESERIES multi-requêtes.
* **Traducteur PromQL**: Un utilitaire permettant de traduire des requêtes en PromQL (langage de requête Prometheus).
* **Effacer**: Supprime la requête de l&apos;entrée de requête.
* **Créer une alerte**: Lance le processus de création d&apos;une nouvelle alerte basée sur les résultats de la requête.
* **Monter/descendre/en haut/en bas**: Réorganise la position du bloc de requête actuel dans le bloc-notes.
* **Supprimer le bloc**: Supprime le bloc actif.

## Ajouter un bloc de requête

Pour ajouter un bloc de requête à votre notebook :

1. Ouvrez votre bloc-notes dans la console de requêtes.
2. Cliquez **+ Query**.
3. Saisissez votre [requêteNRQL ](https://docs.newrelic.com/docs/nrql/nrql-syntax-clauses-functions/)dans l&apos;éditeur de requêtes.
4. Appuyez sur **Run** ou sur **Enter** pour lancer la requête et afficher les résultats.

<Callout variant="tip">
  Toutes les requêtes NRQL prises en charge dans le générateur de requêtes standard fonctionnent dans des blocs de requêtes notebook. Vous pouvez requêter n&apos;importe quel type de données, y compris les métriques, les événements, les logs et les traces. Tous [les types de graphiques](https://docs.newrelic.com/docs/query-your-data/explore-query-data/use-charts/chart-types/) et les personnalisations disponibles dans le générateur de requêtes standard sont pris en charge dans les notebooks.
</Callout>

## gestion des blocs de requêtes

<CollapserGroup>
  <Collapser id="duplicate-query-block" title="Dupliquer un bloc de requête">
    1. Sélectionnez le bloc de requête.
    2. Cliquez sur le **menu à trois points** situé dans le coin supérieur droit.
    3. Sélectionnez **Duplicate**.
  </Collapser>

  <Collapser id="move-query-blocks" title="Déplacer les blocs de requête">
    Réorganisez vos blocs de requêtes pour améliorer la fluidité de votre récit et organiser votre analyse de manière logique. Cela vous permet de mieux raconter une histoire en regroupant les requêtes connexes ou en les organisant dans une séquence logique pour votre enquête.

    1. Sélectionnez le bloc de requête que vous souhaitez déplacer.

    2. Cliquez sur le **menu à trois points**.

    3. Choisissez parmi :

       * **Monter en gamme**
       * **Descendre**
       * **Retour en haut de page**
       * **Déplacez-vous vers le bas**
  </Collapser>

  <Collapser id="delete-query-block" title="Supprimer un bloc de requête">
    1. Sélectionnez le bloc de requête que vous souhaitez supprimer.
    2. Cliquez sur le **menu à trois points**.
    3. Sélectionnez **Delete**.
    4. Confirmer la suppression.
  </Collapser>
</CollapserGroup>
