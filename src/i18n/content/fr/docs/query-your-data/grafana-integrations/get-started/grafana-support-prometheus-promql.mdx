Dans Grafana, vous pouvez configurer New Relic comme source de données Prometheus. De plus, dans Grafana, vous pouvez interroger les métriques stockées dans New Relic à l&apos;aide du [langage de requête PromQL](/docs/integrations/prometheus-integrations/view-query-data/supported-promql-features).

## Utiliser le dashboard Grafana existant avec New Relic [#Grafana-dashboards]

Lorsque vous intégrez les métriques Prometheus à New Relic via [Remote Write](/docs/integrations/prometheus-integrations/get-started/monitor-prometheus-new-relic#remote-write), [Prometheus Agent](/docs/integrations/prometheus-integrations/get-started/monitor-prometheus-new-relic#agent) ou l&apos;[intégration OpenMetrics](/docs/integrations/prometheus-integrations/get-started/monitor-prometheus-new-relic#OpenMetric) (2.0+) et configurez New Relic comme source de données Prometheus dans Grafana, vous pouvez utiliser Grafana existant <InlinePopover type="dashboards" />et exploiter de manière transparente la monitoring, la fiabilité et l&apos;évolutivité supplémentaires que nous fournissons.

## Compatibilité et exigences [#compat-requirements]

Avant de commencer, assurez-vous d’avoir terminé l’intégration des métriques Prometheus et que vous exécutez une version suffisamment récente de Grafana.

* Vous devez avoir configuré l&apos;[écriture à distance](/docs/integrations/prometheus-integrations/get-started/monitor-prometheus-new-relic#remote-write), [l&apos;agent Prometheus](/docs/integrations/prometheus-integrations/get-started/monitor-prometheus-new-relic#agent) ou l&apos;[intégration OpenMetrics](/docs/integrations/prometheus-integrations/get-started/monitor-prometheus-new-relic#OpenMetrics) (v2.0+) avant de pouvoir configurer les sources de données New Relic Prometheus dans Grafana.
* Vous ne pouvez configurer les sources de données New Relic Prometheus à l&apos;aide de cette méthode que dans les versions Grafana 6.7.0 ou plus récentes. Vous devrez configurer des en-têtes personnalisés dans l&apos;interface utilisateur, ce qui n&apos;est pas possible avec les versions antérieures. Pour plus de détails, voir [Configurer New Relic comme source de données Prometheus pour Grafana](/docs/integrations/grafana-integrations/set-configure/configure-new-relic-prometheus-data-source-grafana).

### Prise en charge de PromQL

Notre API Prometheus émule les API de requête de Prometheus. Nous prenons en charge le langage de requête Prometheus (PromQL) via notre mode de requête de style PromQL. Nous faisons de notre mieux pour traduire automatiquement la requête de syntaxe PromQL dans l&apos;approximation NRQL la plus proche.

Pour plus d&apos;informations sur son fonctionnement et sur les différences que vous pouvez observer entre Prometheus et New Relic, consultez [Fonctionnalité PromQL prise en charge](/docs/integrations/prometheus-integrations/view-query-data/supported-promql-features).

## Faites circuler les données dans Grafana [#get-data]

Pour rendre vos données New Relic disponibles dans Grafana, vous pouvez configurer une source de données Prometheus nouvelle ou existante en quelques étapes simples :

1. Dans l&apos;interface utilisateur Grafana, ajoutez et [configurez une nouvelle source de données](/docs/integrations/grafana-integrations/set-configure/configure-new-relic-prometheus-data-source-grafana).
2. Enregistrez la nouvelle source de données et commencez à visualiser vos données.
