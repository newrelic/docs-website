Notre API REST est l&apos;API originale de New Relic pour la configuration par programmation des paramètres d&apos;alerte New Relic ([en savoir plus sur NerdGraph, notre API préférée](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/alerts-nerdgraph/nerdgraph-api-examples)).

Le [REST API Explorer](/docs/apm/apis/api-explorer-v2/getting-started-new-relics-api-explorer) inclut également le format de requête `curl`, les paramètres disponibles, les codes d&amp;apos;état de réponse potentiels et la structure de réponse JSON pour chacun des appels d&amp;apos;API disponibles. Vous pouvez également [créer une condition d&apos;alerte dans l&apos;UI](/docs/alerts/create-alert/create-alert-condition/alert-conditions/).

## Avant d&apos;utiliser l&apos;API REST [#before-start]

Avant d’utiliser l’API REST pour gérer les données d’alerte, lisez le [document Introduction à l’utilisation des alertes avec notre API NerdGraph](/docs/alerts/scale-automate/nerdgraph/nerdgraph-api-examples/) .

Pour les alertes infrastructure , voir [API REST pour monitoring des alertes infrastructure](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/rest-api-calls-new-relic-infrastructure-alerts/)

## Conditions exclues de l&apos;API REST [#excluded]

Ces types de conditions n&apos;ont pas de point de terminaison disponible dans l&apos;API:

* APM: centile Web des transactions, conditions de ciblage des étiquettes et anomalie
* NRQL: anomalie

## Conditions pour APM, navigateur et mobile [#conditions]

Ces fonctions API incluent des liens vers l&apos;explorateur d&apos;API, où vous pouvez créer, mettre à jour, supprimer ou répertorier [les conditions](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions) de politique. Ces appels concernent des conditions avec ces types d&amp;apos;entités :

* APM: Applications et clé de transaction
* navigateur : Apps et clé de transaction
* Mobile : Applications mobiles

<DNT>**Exception:**</DNT> Les conditions APM suivantes sont <DNT>**not available**</DNT> à partir de ces points de terminaison d&amp;apos;API:

* Conditions de ciblage des libellés (ciblage dynamique)
* conditions percentile de transaction Web
* Conditions d&apos;anomalie

<Callout variant="important">
  L&apos;API Explorer fournit des informations sur d&apos;autres types de conditions utilisant <DNT>**separate endpoints**</DNT>, y compris [les services externes](#ext-services-conditions) (APM et mobile) et [monitoring Synthétique](#synthetics-conditions). Tenez compte de tous les types de conditions d’alerte lors de la recherche ou de la mise à jour. Si votre compte héberge des données dans le data center de l&amp;apos;UE, assurez-vous d&amp;apos;utiliser les [points de terminaison d&apos;API appropriés pour les comptes de la région de l&apos;UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#endpoints).
</Callout>

## Conditions pour obtenir le NRQL [#conditions-nrql]

Ces fonctions API incluent des liens vers l&apos;explorateur d&apos;API, où vous pouvez créer, mettre à jour, supprimer ou répertorier [les conditions](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions) NRQL pour vos politiques.

<DNT>**Exception:**</DNT> Les conditions d&amp;apos;anomalie NRQL sont <DNT>**not available**</DNT> à partir de ce point de terminaison.

<Callout variant="important">
  Si votre organisation héberge des données dans un data center de l&apos;UE, assurez-vous d&apos;utiliser les [points de terminaison d&apos;API appropriés pour les comptes de la région de l&apos;UE](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#endpoints).
</Callout>

<CollapserGroup>
  <Collapser id="create-nrql-conditions" title="Créer des conditions NRQL pour les politiques">
    Recommandation : en raison de la manière dont les données NRQL sont diffusées, définissez `aggregation_method` sur `EVENT_FLOW` et utilisez les paramètres par défaut de `60` pour `aggregation_window` et `120` pour `aggregation_delay`. `EVENT_FLOW` fonctionne dans la plupart des cas d&amp;apos;utilisation, mais pour une discussion sur les cas d&amp;apos;utilisation qui fonctionnent mieux avec `EVENT_TIMER`, voir [Choisissez votre méthode d&apos;agrégation](/docs/alerts/create-alert/fine-tune/streaming-alerts-key-terms-concepts/#aggregation-methods).

    <DNT>
      **[API Explorer](https://api.newrelic.com/docs/#/Alerts%20NRQL%20Conditions/post_alerts_nrql_conditions_policies__policy_id__json) &amp;gt; Alerts NRQL Conditions &amp;gt; POST &amp;gt; Create**
    </DNT>

    Pour créer des conditions NRQL pour les stratégies :

    ```shell
    curl -X POST 'https://api.newrelic.com/v2/alerts_nrql_conditions/policies/$POLICY_ID.json' \
         -H "$API_KEY" -i \
         -H 'Content-Type: application/json' \
         -d \
    '{
      "nrql_condition": {
        "type": "string",
        "name": "string",
        "runbook_url": "string",
        "enabled": boolean,
        "terms": [
          {
            "duration": "string",
            "operator": "string",
            "priority": "string",
            "threshold": "string",
            "time_function": "string"
          }
        ],
        "value_function": "string",
        "nrql": {
          "query": "string"
        },
        "signal": {
          "aggregation_window": "string",
          "slide_by": "integer",
          "aggregation_method": "string",
          "aggregation_delay": integer,
          "aggregation_timer": integer,
          "fill_option": "string",
          "fill_value": "string"
        },
        "expiration": {
          "expiration_duration": "string",
          "open_violation_on_expiration": boolean,
          "close_violations_on_expiration": boolean
        }
      }
    }'
    ```

    Si vous omettez `type` ou le définissez sur `static`, l&amp;apos;alerte NRQL standard sera utilisée par défaut.
  </Collapser>

    Pour afficher les [conditions](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition) d&amp;apos;une [entité](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) qui s&amp;apos;appliquent à une entité dans APM, navigateur, mobile ou clé de transaction, incluez ces valeurs dans l&amp;apos;appel d&amp;apos;API:

    * Votre [clé utilisateur](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key) ou [clé API REST](/docs/apis/rest-api-v2/requirements/rest-api-key)

    * Le `entity_id` Il s&amp;apos;agit de l&amp;apos;[entité spécifique (alerte cible)](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) à monitorer.

    * Le `entity_type`, qui doit être l&amp;apos;un des suivants :

      * `Application`
      * `BrowserApplication`
      * `MobileApplication`
      * `KeyTransaction`

      <DNT>
        **[API Explorer](https://api.newrelic.com/docs/#/Alerts%20Entity%20Conditions/get_alerts_entity_conditions__entity_id__json) &amp;gt; Alerts Entity Conditions &amp;gt; GET &amp;gt; list**
      </DNT>

      ```shell
      curl -X GET 'https://api.newrelic.com/v2/alerts_entity_conditions/$ENTITY_ID.json' \
           -H "$API_KEY" -i \
           -G -d 'entity_type=$ENTITY_TYPE'
      ```
  </Collapser>

  <Collapser id="add-entity" title="Ajouter une entité à une condition">
    Pour ajouter une [entité](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) à une condition, incluez ces valeurs dans l&amp;apos;appel d&amp;apos;API:

    * Votre [clé d&apos;utilisateur](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)

    * Le `entity_id` Il s&amp;apos;agit de l&amp;apos;[entité (alerte cible)](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) à monitorer. Il s&amp;apos;agit de l&amp;apos;ID numérique de l&amp;apos;application APM, de l&amp;apos;application de navigateur, de la transaction clé ou de l&amp;apos;application mobile.

    * Le `condition_id` (disponible à partir de <DNT>**[API Explorer](https://api.newrelic.com/docs/#/Alerts%20Conditions/get_alerts_conditions_json) &amp;gt; Alerts Conditions &amp;gt; GET &amp;gt; List**</DNT>)

    * Le `entity_type`, qui doit être l&amp;apos;un des suivants :

      * `Application`
      * `BrowserApplication`
      * `MobileApplication`
      * `KeyTransaction`

      <DNT>
        **[API Explorer](https://api.newrelic.com/docs/#/Alerts%20Conditions/get_alerts_conditions_json) &amp;gt; Alerts Entity Conditions &amp;gt; PUT &amp;gt; Add**
      </DNT>

      ```shell
      curl -X PUT 'https://api.newrelic.com/v2/alerts_entity_conditions/$ENTITY_ID.json' \
           -H "$API_KEY" -i \
           -H 'Content-Type: application/json' \
           -G -d 'entity_type=$ENTITY_TYPE&condition_id=$CONDITION_ID'
      ```
  </Collapser>

  <Collapser id="remove_entity" title="Supprimer une entité d'une condition">
    Pour supprimer une [entité](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) monitorée d&amp;apos;une condition, incluez ces valeurs dans l&amp;apos;appel d&amp;apos;API:

    * Votre [clé d&apos;utilisateur](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)

    * Le `entity_id` Il s&amp;apos;agit de l&amp;apos;[entité de monitoring spécifique (alerte cible)](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) à supprimer. Il s&amp;apos;agit de l&amp;apos;ID numérique de l&amp;apos;application APM, de l&amp;apos;application de navigateur, de la transaction clé ou de l&amp;apos;application mobile.

    * Le `condition_id` (disponible à partir de <DNT>**[API Explorer](https://api.newrelic.com/docs/#/Alerts%20Conditions/get_alerts_conditions_json) &amp;gt; Alerts Conditions &amp;gt; GET &amp;gt; List**</DNT>)

    * Le `entity_type`, qui doit être l&amp;apos;un des suivants :

      * `Application`
      * `BrowserApplication`
      * `MobileApplication`
      * `KeyTransaction`

      <DNT>
        **[API Explorer](https://api.newrelic.com/docs/#/Alerts%20Entity%20Conditions/delete_alerts_entity_conditions__entity_id__json) &amp;gt; Alerts Entity Conditions &amp;gt; DELETE &amp;gt; Remove**
      </DNT>

      ```shell
      curl -X DELETE 'https://api.newrelic.com/v2/alerts_entity_conditions/$ENTITY_ID.json' \
           -H "$API_KEY" -i \
           -G -d 'entity_type=$ENTITY_ID&condition_id=$CONDITION_ID'
      ```
  </Collapser>
