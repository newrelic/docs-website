Il existe différentes manières de créer une condition d&apos;alerte pour les métriques New Relic <InlinePopover type="apm" />et <InlinePopover type="browser" />, mais nous vous recommandons d&amp;apos;utiliser NRQL. L’utilisation d’une condition d’alerte NRQL fournit des contrôles supplémentaires, un temps de détection amélioré et une cohérence entre tous les types de données.

En 2021, nous avons facilité la création de conditions d&apos;alerte NRQL à partir de graphiques et de requêtes dans New Relic. Voir ci-dessous un exemple de création d&apos;une condition de service externe APM à l&apos;aide de NRQL.

La mise en place d&apos;une condition d&apos;alerte APM ou monitoring de navigateurs métrique est un processus en deux parties : Tout d&apos;abord, vous devez identifier l&apos;entité, puis vous créez la condition NRQL .

## Exemple de condition APM

Voici une procédure montrant comment vous pouvez configurer une condition pour une application de monitoring APM .

### Premièrement : identifier l’entité [#identity-entity]

Tout d’abord, vous identifierez l’entité pour laquelle vous définirez la condition d’alerte :

1. Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM**</DNT> dans la barre d’outils.

2. Recherchez le service sur lequel vous souhaitez être alerté.

3. Cliquez sur le service.

4. Dans le volet de navigation de gauche, cliquez sur <DNT>**External services**</DNT>.

5. Sous <DNT>**Top 20 external services**</DNT>, recherchez le service souhaité et cliquez dessus.

   <img title="top-twenty.png" alt="Screen capture showing the list of the top 20 external services to choose from." src="/images/accounts_screenshot-full_top-twenty.webp" />

6. Sur le côté droit de la page, où vous voyez les graphiques pour <DNT>**Response time**</DNT> et <DNT>**External calls per minute**</DNT> (débit), décidez sur laquelle de ces deux options vous souhaitez générer une alerte.

7. Dans le coin supérieur droit du graphique que vous choisissez, cliquez sur l’icône représentant des points de suspension <DNT>**...**</DNT> et sélectionnez <DNT>**Create alert condition**</DNT>.

   <Callout variant="tip">
     Si vous ne voyez pas l&apos;icône de points de suspension <DNT>**...**</DNT>, assurez-vous que l&amp;apos;option <DNT>**Show new view**</DNT> en haut de la page est activée.
   </Callout>

   <img title="external_service_graph.png" alt="Screenshot showing the menu options." src="/images/accounts_screenshot-full_external-service-graph.webp" />

8. Ceci lance une modale avec le générateur de condition d&apos;alerte NRQL , la requête est pré-remplie.

   * Si vous choisissez le temps de réponse, votre requête peut ressembler à ceci :

     <img title="response_time_query.png" alt="Example showing a query for response time." src="/images/accounts_screenshot-full_response-time-query.webp" />

   * Si vous choisissez des appels externes par minute, votre requête peut ressembler à ceci :

     <img title="external-calls-query.png" alt="Example show an external calls query." src="/images/accounts_screenshot-full_external-calls-query.webp" />

Avec votre requête pré-remplie dans le générateur de conditions, vous allez ensuite créer la condition d&apos;alerte NRQL .

### Suivant : créer la condition d&apos;alerte NRQL [#create-condition]

Dans la modale, complétez les éléments suivants :

1. Entrez un nom de condition.

2. Dans <DNT>**Define your signal**</DNT>, observez que toutes les clauses `LIMIT`, `SINCE...AGO` et `TIMESERIES` sont supprimées afin que la syntaxe de requête NRQL soit valide.

   <Callout variant="tip">
     Si aucun résultat n&apos;est renvoyé, assurez-vous que vous êtes sur le bon compte en regardant le compte actuel dans le coin supérieur gauche de la fenêtre modale. Si ce n&apos;est pas correct, vous devrez peut-être fermer la fenêtre modale et vérifier que le bon compte est sélectionné dans le sélecteur de compte dans le coin supérieur gauche de la page.
   </Callout>

   <img title="nrql_condition_builder_modal.png" alt="Screenshot showing you the NRQL alert condition builder pre-populated with your query." src="/images/accounts_screenshot-full_nrql-condition-builder-modal.webp" />

3. Remplissez les champs restants, sélectionnez ou créez une politique en bas du formulaire et enregistrez votre condition.

Facultatif : découvrez [comment requêter des données d&apos;intervalle de temps métrique APM avec NRQL](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/query-apm-metric-timeslice-data-nrql#timeslice-conversion).
