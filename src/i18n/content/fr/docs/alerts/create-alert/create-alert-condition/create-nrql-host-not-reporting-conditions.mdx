Le mode guidé de condition NRQL simplifie la création d&apos;alertes « host not reporting » (HNR). Au lieu de construire manuellement une requête NRQL, cette approche guidée vous aide à définir rapidement des conditions pour détecter quand les agents d&apos;infrastructure cessent d&apos;envoyer des données. Cette méthode offre une plus grande flexibilité pour cibler des groupes d&apos;hôtes spécifiques, personnaliser les seuils d&apos;alerte et affiner les critères d&apos;événement d&apos;alerte par rapport aux alertes HNR traditionnelles.

## Caractéristiques [#features]

Un événement HNR est généré lorsque l&apos;agent d&apos;infrastructure ne parvient pas à transmettre des données à notre collecteur dans un délai spécifié.

## Créer une condition d&apos;alerte NRQL HNR [#create-nrql-hnr-alert-condition]

1. Navigation vers une page de règle d&apos;alerte.

2. Cliquez sur **New alert condition**.

3. Sélectionnez **Use guided mode**.

4. Sélectionnez **Hosts** dans la catégorie **Tell us where to look** . (Remarque : si vous ne voyez pas l&apos;option « hôtes », vous devrez peut-être suivre [les instructions pour instrument votre infrastructure](/docs/infrastructure/host-integrations/installation/new-relic-guided-install-overview/).)

   <img title="creating a condition step 2" alt="screenshot of alert condition step 2" src="/images/alerts_screenshot-crop_tell-us-where-to-look.webp" />

5. Si **Hosts** n&apos;est pas automatiquement sélectionné pour vous sous **Select an entity type**, choisissez à nouveau **Hosts** .

6. Cliquez sur **Next**.

7. Sous **Select signal catgory**, sélectionnez **Host**.

8. Sous **Select a metric to monitor**, sélectionnez **Host not reporting**.

   <img title="host not reporting selection" alt="screenshot of selecting host not reporting option" src="/images/alerts_screenshot-crop_host-not-reporting-selection.webp" />

9. Vous pouvez désormais utiliser le filtre d&apos;entité pour affiner la portée des hôtes que vous souhaitez monitorer.

10. Cliquez sur **Next**.

    <DNT>
      /* &lt;img title=&quot;set condition thresholds&quot; alt=&quot;screenshot of setting remaining thresholds&quot; src=&quot;/images/alerts_screenshot-crop_set-condition-thresholds.webp&quot;/&gt; */
    </DNT>

11. Sous **Consider the signal lost after**, vous avez la possibilité d&apos;ajuster la fenêtre temporelle de 30 secondes à 48 heures. Vous avez également la possibilité d&apos;ajuster vos [paramètres de perte de signal](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/#signal-loss), notamment en sélectionnant l&apos;option permettant d&apos;ignorer l&apos;ouverture d&apos;un événement d&apos;alerte lorsque l&apos;arrêt du signal est prévu (par exemple, si votre hôte doit s&apos;arrêter).

12. Suivez les étapes restantes pour terminer la configuration et enregistrer votre condition d’alerte.

## Option : Ne pas ouvrir d&apos;événement d&apos;alerte &quot;signal perdu&quot; lors d&apos;un arrêt prévu [#do-not-open-lost-signal]

Lorsque vous cochez l&apos;option **Do not open &quot;lost signal&quot; alert event on expected termination**, vous indiquez à New Relic de ne pas ouvrir d&apos;événement d&apos;alerte lorsque l&apos;arrêt du signal est prévu. Cela est utile lorsque vous savez qu&apos;un hôte va s&apos;arrêter.

Pour activer cette option, vous devez également cocher **Open new &quot;lost signal&quot; alert event**. Ainsi, New Relic sait qu&apos;il doit ouvrir un nouvel événement d&apos;alerte lorsque le signal est perdu, mais pas lorsque la perte du signal est prévue.

<Callout variant="important">
  Pour empêcher l&apos;ouverture d&apos;un événement d&apos;alerte de perte de signal lorsque l&apos;option « Ne pas ouvrir d&apos;événement d&apos;alerte &quot;signal perdu&quot; lors d&apos;un arrêt prévu » est activée, le tag `termination: expected` doit être ajouté à l&apos;entité hôte. Ce tag indique que le signal devait se terminer. Découvrez comment ajouter le tag [directement à l&apos;entité hôte](/docs/new-relic-solutions/new-relic-one/core-concepts/use-tags-help-organize-find-your-data/#add-tags).
