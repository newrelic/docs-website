Lorsque l’expression de déclenchement d’une condition d’alerte composée est évaluée comme vraie, New Relic ouvre un événement d’alerte composée. L’événement inclut un lien vers l’URL de votre runbook. Pour afficher les dépassements de seuil spécifiques pour chaque condition de composant, cliquez sur les événements d’alerte individuels.

## Événements d’alerte et corrélation des problèmes [#alert-events-correlation]

Les conditions d’alerte composites créent plusieurs événements d&apos;alerte associés :

* <DNT>**Multiple alert events**</DNT>: une condition d’alerte composée crée son propre événement d’alerte composée. Par défaut, chaque condition de composant crée également ses propres événements d’alerte lorsqu’elle dépasse ses seuils. Pour réduire le bruit, vous pouvez mettre ces événements en sourdine à partir de la configuration de la condition d’alerte. Si vous désactivez les événements d’alerte d’une condition de composant, les tags de cette condition n’apparaîtront pas dans l’événement d’alerte composée.
* <DNT>**Correlating to the same issue**</DNT>: Pour vous assurer que tous les événements d&apos;alerte associés (à la fois composites et composants) sont regroupés dans le même problème, configurez vos conditions de composant pour utiliser la même règle d’alerte que votre condition d’alerte composite. Cela permet à la corrélation des problèmes de New Relic de les regrouper.
* <DNT>**Viewing related events**</DNT>: à partir d’un événement d’alerte composée, vous pouvez naviguer vers les événements d’alerte de composant individuels pour voir les dépassements de seuil et les métriques pour chaque condition.
