  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Pour créer une condition d’alerte composée,

1. Allez à <DNT>**Alerts &gt; Compound Alerts**</DNT>.

2. Toutes les conditions de composant doivent provenir du même compte propriétaire. Sélectionnez-le dans la liste déroulante.

   <Callout variant="tip">
     Pour inclure une condition qui cible des données d’un autre compte, configurez une condition d’alerte multicompte dans le même compte propriétaire que l’alerte composée.
   </Callout>

3. Cliquez sur <DNT>**Create a compound alert**</DNT> et saisissez un nom pour l’alerte composée.

4. Cliquez sur <DNT>**Add conditions**</DNT> et choisissez jusqu’à cinq conditions. Une variable <DNT>(a, b, c, d, e)</DNT> est automatiquement attribuée à chaque condition pour être utilisée dans l’expression de déclenchement. Vous ne pouvez sélectionner que des conditions d’alerte standard, et non d’autres conditions d’alerte composées. Consultez les [limitations des alertes composées](/docs/alerts/create-alert/create-alert-condition/compound-alerts/compound-alerts#limitations) pour connaître toutes les restrictions.

   <img title="Condition selection in compound alert builder" alt="A screenshot dispaying the condition selection process in the compound alert builder" src="/images/select-2-5-alerts.webp" />

   <Callout variant="important">
     Chaque condition peut avoir plusieurs seuils comme avertissement, critique ou perte de signal. Lorsqu’un seuil d’une condition est atteint, cette condition est considérée comme déclenchée dans l’expression composée.
   </Callout>

5. Si vos conditions partagent des facettes, cochez la case pour ne déclencher que lorsque la même valeur de facette atteint les seuils dans toutes les conditions. Laissez-la décochée pour déclencher indépendamment des valeurs de facette correspondantes.

   <Callout variant="tip">
     Lorsqu’une condition est facettée par plusieurs attributs, une correspondance se produit lorsqu’au moins un attribut est identique dans toutes les conditions.
   </Callout>

6. Pour définir comment les conditions doivent fonctionner ensemble, dans le champ **Tell us how you want these conditions to work together** \[Indiquez-nous comment vous souhaitez que ces conditions fonctionnent ensemble], écrivez une expression de déclenchement à l&apos;aide des variables attribuées et des opérateurs logiques <DNT>(AND, OR, NOT)</DNT>. Utilisez des parenthèses pour regrouper des expressions pour une logique complexe.

   Exemples : <DNT>`a AND b`, `a OR b OR c`, `a AND (b OR c)`, `a AND NOT b`, `a AND (b OR c) AND NOT d`</DNT>

7. Définissez la **Threshold duration** pour contrôler combien de temps l’expression de déclenchement doit rester vraie avant qu’un événement d’alerte ne s’ouvre.

8. Dans la liste déroulante <DNT>**Policy**</DNT>, sélectionnez une police. En savoir plus sur les [règles d’alerte](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/).

   <Callout variant="tip">
     Pour regrouper les événements d’alerte composée et de composant dans le même problème, configurez vos conditions de composant pour utiliser la même police que vous sélectionnez ici.
   </Callout>

9. (Facultatif) Pour créer un lien vers votre procédure de gestion des incidents, ajoutez un <DNT>**Runbook URL**</DNT>. Cette URL sera incluse dans les notifications.

10. Basculez <DNT>**Enable on save**</DNT> pour activer la condition immédiatement.

11. Vérifiez votre configuration, puis enregistrez. La condition apparaît sur la page <DNT>**Compound Alerts**</DNT>.

## Gérer les conditions d’alerte composées [#manage-compound-alerts]

Pour afficher toutes les conditions d’alerte composites, accédez à <DNT>**Alerts &gt; Compound Alerts**</DNT>.

* <DNT>**Edit**</DNT>: cliquez sur n’importe quel nom d’alerte composée pour modifier sa configuration. Les modifications apportées aux conditions de composant se reflètent automatiquement dans l’alerte composée.
* <DNT>**Delete**</DNT>: cliquez sur <Icon name="fe-more-horizontal" /> &gt; <DNT>**Delete**</DNT>. La suppression d’une alerte composée ne supprime pas les conditions de composant individuelles — celles-ci restent actives individuellement.
