Les lacunes de couverture d&apos;alerte mettent en évidence les zones de votre stack où vous manquez de couverture d&apos;alerte. Dans notre section <InlinePopover type="alerts" />, [nous avons créé une page d&apos;accueil](https://one.newrelic.com/nrai/detection-gaps/home) qui inclut APM non couvert, le navigateur et l&apos;entité mobile, afin que vous puissiez évaluer les informations qui manquent à votre équipe et combler le manque de couverture.

Dans un paysage de plus en plus dynamique, nous savons à quel point il est important de rester au courant de votre stack et d&apos;être averti de manière proactive si quelque chose ne va pas. Mais, à mesure que votre équipe grandit ou évolue, il peut être difficile de savoir si vous recevez les alertes dont vous avez besoin, quand vous en avez besoin. Les lacunes de couverture d&apos;alerte donnent à votre équipe une liste complète de toutes les entités que votre équipe *ne couvre pas actuellement*.

Que signifie couvrir une entité ? Une entité couverte signifie que vous avez configuré une [alerte](/docs/alerts-applied-intelligence/overview/#concepts-terms) pour vous informer des performances d&apos;une entité particulière. Par exemple, vous pourriez avoir créé une condition d&apos;alerte qui ouvre un incident si le débit d&apos;une entité APM dépasse 100 requests par minute. Une entité non couverte fait partie de votre système qui n’est pas monitorée, ce qui signifie qu’il pourrait y avoir un comportement malsain qui n’est pas contrôlé. Nous avons créé [des lacunes de couverture d&apos;alerte](https://one.newrelic.com/nrai/detection-gaps/home) pour mettre en évidence les entités non couvertes afin que votre équipe ne manque pas de données précieuses et que vous puissiez prévenir les incidents avant qu&apos;ils ne se produisent.

## Trouvez vos lacunes en matière de couverture [#find-your-coverage-gaps]

Pour ouvrir la page des lacunes de couverture des alertes, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts**</DNT> et sélectionnez <DNT>**Alert coverage gaps**</DNT> dans le volet de navigation de gauche. Là, vous verrez une liste de toutes vos entités qui ne sont pas actuellement couvertes.

Selon l&apos;entité couverte, cette page peut inclure ces onglets :

* <DNT>**APM**</DNT>.
* <DNT>**Browser**</DNT>.
* <DNT>**Mobile**</DNT>.

S&apos;il n&apos;y a aucune entité non couverte pour <DNT>**Browser**</DNT> ou <DNT>**Mobile**</DNT>, vous ne verrez pas cet onglet spécifique.

## Créez une alerte à partir de vos lacunes de couverture d&apos;alerte [#create-an-alert]

Une fois que vous avez sélectionné une entité que vous souhaitez monitorer, vous pouvez alors créer une condition d&apos;alerte pour avertir votre équipe de tout comportement anormal.

New Relic suggérera dynamiquement les conditions que nous pensons devoir monitorer en fonction de l&apos;entité sélectionnée et de votre système spécifique. Certaines des alertes recommandées que nous pourrions suggérer sont le pourcentage d&apos;erreur, l&apos;apdex ou le temps de réponse.

Vous obtiendrez une condition d&apos;alerte recommandée en fonction [des métriques dorées](https://github.com/newrelic/entity-definitions) définies pour le type d&apos;entité. Si vous n&apos;obtenez pas la condition d&apos;alerte recommandée, une fois que vous avez sélectionné une condition d&apos;alerte que vous souhaitez monitorer, votre équipe sera alertée lorsque ce signal s&apos;écarte de sa base de référence normale, supérieure et inférieure, pendant au moins 5 minutes de 3,00 écarts types.

Pour créer une alerte à partir de vos lacunes de couverture d&apos;alerte :

1. Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts**</DNT> et sélectionnez <DNT>**Alert coverage gaps**</DNT> dans le volet de navigation de gauche.

2. Cliquez sur <DNT>**Add alert**</DNT> sur l’entité pour laquelle vous souhaitez créer l’alerte.

3. Ajoutez les conditions recommandées que vous souhaitez.

   <img width="80%;" title="Create an alert" alt="A screenshot of creating an alert from the alert coverage gap page" src="/images/accounts_screenshot-crop_add-an-alert.webp" />

   <figcaption>
     Pour créer une alerte pour une entité non monitorée, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts**</DNT>, sélectionnez <DNT>**Alert coverage gaps**</DNT> dans le volet de navigation de gauche et cliquez sur <DNT>**Add alert**</DNT>.
   </figcaption>

4. Cliquez sur <DNT>**Next**</DNT>.

5. Sélectionnez une politique pour être averti. Chaque condition d&apos;alerte doit être acheminée via une [politique.](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/) Vous devez donc soit sélectionner une politique existante, soit en créer une nouvelle. Une fois que vous avez sélectionné ou créé une politique associée à la condition d&apos;alerte, vous devrez peut-être ajouter une [destination](/docs/alerts-applied-intelligence/notifications/destinations). Les destinations sont les endroits où nous envoyons des notifications concernant vos données New Relic . Vous pouvez lier votre notification à des outils tiers tels que Slack, Atlassian Jira, la messagerie électronique, etc.

   <img width="80%;" title="Create a policy for alert coverage gaps." alt="A screenshot of the alert coverage gap policy selection" src="/images/accounts_screenshot-crop_create-a-policy-alert-coverage-gaps.webp" />

   <figcaption>
     Lorsque vous créez une alerte, vous sélectionnez la politique avec laquelle l’alerte sera regroupée.
   </figcaption>

6. Cliquez sur <DNT>**Activate conditions**</DNT> pour enregistrer les conditions. Si la politique est nouvelle, vous verrez le bouton <DNT>**Save &amp; set up notifications**</DNT> . Utilisez ce bouton si vous souhaitez enregistrer les conditions et la notification.
