Vous pouvez utiliser les paramètres [de seuil critique](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/advanced-techniques/set-thresholds-alert-condition) et [de perte de signal](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions#signal-loss) pour monitorer les tâches planifiées et être averti lorsqu&apos;elles ne se produisent pas.

Par exemple, si votre système a une maintenance planifiée toutes les 24 heures, vous aimeriez savoir quand cette maintenance n&apos;a pas lieu.

Cette technique ne fonctionne que pour les tâches planifiées régulièrement qui se produisent plus fréquemment que le seuil maximal de perte de signal de 48 heures.

## Ce que vous devez savoir

Afin de tirer le meilleur parti de cette procédure, vous devez savoir comment [créer une condition](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions).

## Comment ça marche

Généralement, les conditions utilisent des seuils critiques pour déclencher des événements d&apos;alerte. Cependant, vous pouvez également déclencher des événements d&apos;alerte avec une perte de signal. Pour monitorer une tâche planifiée et être notifié si elle ne s&apos;exécute pas, réglez votre minuteur de perte de signal pour qu&apos;il corresponde à la planification de la tâche.

## Monitorer une tâche planifiée

1. [Créez une condition](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) liée au travail que vous souhaitez monitorer.
2. Cliquez sur <DNT>**+ Add lost signal threshold**</DNT>, puis définissez la durée d&apos;expiration du signal sur une valeur plus longue que le cycle planifié de votre tâche de monitoring.
3. Vérifiez <DNT>**Add lost signal threshold**</DNT>.
4. Assurez-vous de nommer votre condition avant de l&apos;enregistrer.
