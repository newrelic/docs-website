Un modèle de titre est utilisé lorsque des événements d&apos;alerte sont ouverts par la condition. Cela remplace le titre par défaut.

## Pourquoi utiliser un modèle de titre ? [#why-titles]

Une condition d&apos;alerte est la base de la création d&apos;événements d&apos;alerte. Que vous utilisiez notre mode guidé ou que vous élaboriez une condition à partir d&apos;un graphique ou d&apos;une politique, l&apos;utilisation d&apos;un modèle de titre unique constitue la dernière étape de la création d&apos;une condition d&apos;alerte.

L&apos;utilisation du modèle de titre est facultative, mais nous la recommandons. Une condition d&apos;alerte définit un ensemble de seuils que vous souhaitez monitorer. Si l&apos;un de ces seuils est dépassé, un événement d&apos;alerte est créé. Des modèles de titre pertinents vous aident à identifier les problèmes et à résoudre les pannes plus rapidement.

## Avant de commencer [#begin]

Voici quelques éléments clés à noter avant d’utiliser notre modèle de titre :

* **Compatibilité**: La fonctionnalité de modèle de titre est disponible uniquement pour les conditions NRQL.
* **Limite de caractères**: La longueur maximale d&apos;un modèle de titre est de 256 caractères.
* **Formatage**: Le modèle de titre doit suivre un format [Handlebars](https://handlebarsjs.com/guide/) valide.
* **Événements d&apos;alerte de perte de signal**: pour les événements d&apos;alerte déclenchés par une perte de signal, le titre par défaut sera toujours utilisé. Pour en savoir plus sur la perte de signal, consultez [Perte de signal et comblement des lacunes](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/).

## Comprendre le modèle de titre [#example]

Supposons que vous gériez un site de e-commerce et que vous ayez créé une condition d&apos;alerte pour déclencher un événement d&apos;alerte lorsque les erreurs réseau sont élevées. Vous pouvez utiliser un modèle de titre pour rendre les événements d&apos;alerte de cette condition facilement identifiables.

Vous avez nommé cette condition d&apos;alerte `High Network Errors`. Mais vous souhaitez que le titre de l&apos;événement d&apos;alerte reflète davantage d&apos;informations sur l&apos;entité et la priorité.

**Exemple**:

```handlebars
{{conditionName}} Affecting {{entity.name}} - {{entity.type}} - {{priority}}
```

Ce modèle générerait le titre d&apos;événement d&apos;alerte suivant :

```
High Network Errors Affecting Entity123 - Kubernetes Pod - Critical
```

## Utiliser le modèle de titre [#use-titles]

Vous pouvez interagir avec le modèle de titre de deux manières : via l&apos;UI ou en utilisant GraphQL. Suivez ces étapes pour savoir comment procéder.

<CollapserGroup>
  <Collapser id="template-ui" title="Créer un modèle de titre dans l'UI">
    1. Accédez au générateur de conditions NRQL. Pour les instructions, voir [condition d&apos;alerte](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/alert-conditions/).
    2. Créez votre condition NRQL.
    3. Dans la dernière étape, **Add details**, entrez votre modèle de titre dans le champ **Title template** .
    4. Sauvegarder la condition.
  </Collapser>

  <Collapser id="template-ui" title="Créer un modèle de titre dans GraphQL">
    Dans GraphQL, le champ de modèle de titre est appelé `titleTemplate`. Voici un exemple de mutation pour mettre à jour une condition statique NRQL pour ajouter un modèle de titre :

    ```graphql
    mutation {
      alertsNrqlConditionStaticUpdate(
        accountId: 123456
        id: 456789
        condition: {titleTemplate: "{{conditionName}} has triggered an incident"}
      ) {
        titleTemplate
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## Utilisez l&apos;attribut pour définir davantage votre condition [#attributes]

Vous pouvez inclure un sous-ensemble d&apos;attributs d&apos;événements d&apos;alerte dans vos modèles de titre pour ajouter plus de contexte.

Voici comment :

* **Utilisez le format `{{AN_ATTRIBUTE}}`**. Par exemple:

  ```handlebars
  {{conditionName}} has triggered an alert event
  ```

* **attribut tag**: pour tag, utilisez le format `{{tags.TAG_NAME}}`.

* **Balise de clause de facette**: si une condition NRQL utilise une clause `FACET`, vous pouvez utiliser le formatage `tags` pour utiliser ces valeurs dans votre modèle de titre. Par exemple, si la requête NRQL incluait `FACET hostName, cluster`, vous pourriez alors utiliser ceci :

  ```handlebars
  Host {{tags.hostName}}, cluster {{tags.cluster}} alert event
  ```

Les noms de tags ne peuvent pas contenir d&apos;espaces, mais les valeurs étendues le peuvent. Pour une liste complète des attributs disponibles, consultez [Attributs d&apos;événement d&apos;alerte](/docs/alerts/create-alert/condition-details/alert-event-attributes).
