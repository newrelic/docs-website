Lorsque le [seuil](/docs/alerts/create-alert/set-thresholds/set-thresholds-alert-condition/) défini dans une [condition](/docs/alerts/create-alert/create-alert-condition/alert-conditions/) est dépassé, selon les paramètres <DNT>**[Alert event preference](/docs/alerts/organize-alerts/specify-when-alerts-create-events)**</DNT> de la stratégie, <InlinePopover type="alerts" />peut créer un événement d&apos;alerte. Vous pouvez consulter les informations sur les événements d&apos;alerte de plusieurs façons :

* [Consultez l&apos;index des événements d&apos;alerte](#view-alert-events) pour repérer des tendances dans une liste d&apos;événements d&apos;alerte.
* [Consultez le log des événements d&apos;alerte](#view-log) pour examiner les détails de performance associés.
* [Consultez les événements](#alert-event-time) inclus dans un événement d&apos;alerte spécifique pour examiner les horodatages des [événements](/docs/using-new-relic/welcome-new-relic/get-started/glossary#event), tels que l&apos;ouverture ou la fermeture d&apos;un événement d&apos;alerte, les notifications et les acquittements.

## Consulter l&apos;index des événements d&apos;alerte et les détails des événements d&apos;alerte [#view-alert-events]

Les événements d&apos;alerte sont regroupés en événements d&apos;alerte. Si vous souhaitez modifier la façon dont les événements d&apos;alerte sont regroupés, ouvrez la stratégie associée et modifiez le paramètre [<DNT>**Alert event preference**</DNT>](/docs/alerts/organize-alerts/specify-when-alerts-create-events/).

Pour afficher les détails de l&apos;événement d&apos;alerte :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Alerts**</DNT> et cliquez sur l’onglet <DNT>**Alert events**</DNT> .

2. Sélectionnez l&apos;un des événements d&apos;alerte dans le tableau pour ouvrir ses détails.

Les détails des graphiques d&apos;événements d&apos;alerte individuels incluent :

* <DNT>**Timing information**</DNT>: La zone rouge ombrée sur le graphique indique la période durant laquelle l&apos;événement d&apos;alerte s&apos;est produit, tandis que la zone rose ombrée précédente représente la [période de dégradation](/docs/using-new-relic/welcome-new-relic/get-started/glossary#degradation-period). Si vous sélectionnez un événement d&apos;alerte ayant duré plus de deux heures, la chronologie au bas du graphique sera en dents de scie. Pour fournir un contexte aux événements de l&apos;événement d&apos;alerte, le graphique affiche également la période entourant l&apos;événement d&apos;alerte.

* <DNT>**Chart guidelines**</DNT>:La ligne pointillée rouge marque le seuil de la condition. La ligne bleue représente les informations sur les performances.

* <DNT>**Anomalous behavior:**</DNT> Si les alertes détectent un [comportement anormal](#anomalous-behavior) proche du moment de l&apos;événement d&apos;alerte, vous verrez une notification dans les détails de l&apos;événement d&apos;alerte.

## Voir les événements d&apos;un événement d&apos;alerte [#view-log]

Si vous souhaitez afficher les [événements d&apos;alerte sur tous les produits](/docs/alerts/alert-event-management/view-alert-event-details), accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, puis cliquez sur <DNT>**All entities**</DNT>. Pour afficher les événements d&apos;un seul événement d&apos;alerte :

1. Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities.**</DNT>
2. Sélectionnez une ligne d’entité.
3. Dans le menu de navigation de gauche, sous <DNT>**Events**</DNT>, cliquez sur <DNT>**Issues &amp; activity**</DNT>.
4. Sélectionnez l&apos;un des événements pour afficher un graphique et des détails le concernant.

## Temps entre un événement d&apos;alerte et la notification [#alert-event-time]

Il peut y avoir un décalage allant jusqu&apos;à trois minutes entre l&apos;heure de l&apos;événement d&apos;alerte et l&apos;heure de la notification initiale en raison des variations du temps de traitement des données.

* <DNT>**Notification time:**</DNT> L’heure indiquée dans la notification reflète l’horodatage auquel nous avons reçu la demande d’envoi d’une notification.
* <DNT>**Alert event time:**</DNT> L&apos;heure affichée sur la page <DNT>**Events**</DNT> pour l&apos;événement d&apos;alerte correspond à l&apos;horodatage de la collecte de données du dernier point de données ayant contribué à l&apos;ouverture de l&apos;événement d&apos;alerte.

## Détection de comportement anormal [#anomalous-behavior]

Lorsque nous détectons des changements importants dans les signaux clés de l&apos;entité d&apos;alerte et/ou des applications en amont/en aval de l&apos;entité d&apos;alerte, une notification &quot;comportement anormal détecté&quot; apparaît sur la page de l&apos;événement d&apos;alerte et dans les canaux de notification. Vous pouvez :

* Développez la notification pour obtenir des détails sur l’anomalie détectée (Web uniquement).
* Voir anomalie amont/aval (Slack uniquement).
* Sélectionnez un lien pour accéder au tableau des produits concernés pour une enquête plus approfondie.
