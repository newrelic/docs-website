<DNT>**Dynamic Flow Map**</DNT> est un outil de visualisation qui affiche des données trace agrégées sous forme de carte. Il se concentre sur un service spécifique (appelé nœud focal) et révèle comment les variations de performance des services en amont et en aval s&apos;affectent mutuellement. En mettant en évidence les latences anormales corrélées et les erreurs dans le trafic entre les services, la carte vous aide à identifier rapidement les goulots d&apos;étranglement des performances et trace aux causes profondes des problèmes. La carte affiche jusqu&apos;à 3 heures de données des traces.

## Pourquoi c&apos;est important [#why-it-matters]

<DNT>**Dynamic Flow Map**</DNT> réduit le Délai moyen de résolution (MTTR) en faisant apparaître visuellement les anomalies de performances. Au lieu d&apos;analyser manuellement les traces pour comprendre les relations entre les services, vous pouvez voir d&apos;un coup d&apos;œil quels services rencontrent des problèmes et comment ces problèmes sont liés au nœud central. La visualisation par code couleur vous indique où se produisent les latences ou les erreurs corrélées aux performances de l&apos;entité centrale dans vos systèmes distribués.

Pour les commandes générales de navigation et d&apos;interface communes à toutes les cartes New Relic, consultez [la section Cartes](/docs/service-architecture-intelligence/maps/advanced-maps).

<Callout variant="important">
  [<DNT>**Distributed tracing**</DNT>](/docs/distributed-tracing/concepts/introduction-distributed-tracing) est nécessaire pour que <DNT>**Dynamic Flow Map**</DNT> fonctionne, et il est activé par défaut pour tous les agents APM. Si les données ne sont pas visibles, assurez-vous que <DNT>**Distributed tracing**</DNT> est actif pour le nœud focal et l&apos;entité associée, et vérifiez que les données de tracing apparaissent dans New Relic.
</Callout>

## Accédez à la carte des flux dynamiques [#explore-dynamic-flow-map]

Vous pouvez accéder à <DNT>**Dynamic Flow Map**</DNT> depuis plusieurs emplacements dans New Relic en fonction de vos besoins d&apos;investigation :

### À partir de la page de Traces [#from-traces-page]

Utilisez cette option lorsque vous souhaitez visualiser et étudier les relations trace à travers vos systèmes distribués à partir d&apos;un nœud central.

<Callout variant="important">
  Cette option d&apos;accès est disponible uniquement si vous utilisez le modèle de tarification par entité.
</Callout>

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; Traces**</DNT>.
2. Cliquez sur l’onglet <DNT>**Dynamic flow map**</DNT>.

### À partir des cartes de service [#from-service-maps]

Utilisez cette option lorsque vous explorez l&apos;architecture de vos services à partir d&apos;une [carte](/docs/service-architecture-intelligence/maps/advanced-maps/#entry-points) et que vous souhaitez voir comment les changements de performance des services en amont et en aval affectent une entité spécifique, avec une détection des anomalies au niveau des traces.

1. Sur la carte, pointez vers n&apos;importe quel nœud d&apos;entité.
2. Dans le menu contextuel, cliquez sur <DNT>**View dynamic flow**</DNT> (l&apos;icône de trace).

### À partir d&apos;une transaction spécifique [#from-specific-transaction]

Utilisez cette option lorsque vous devez résoudre des problèmes de performance pour une transaction spécifique et que vous souhaitez voir quels services sont impliqués et où se produisent la latence ou les erreurs dans le chemin d&apos;exécution de cette transaction.

Lorsque vous accédez à <DNT>**Dynamic Flow Map**</DNT> via une transaction spécifique, la carte affiche uniquement l&apos;entité impliquée dans cette transaction. Contrairement à la carte de flux dynamique basée sur le nœud focal, cette vue spécifique à la transaction n&apos;inclut pas la table <DNT>**performance by transactions**</DNT>, car elle est déjà axée sur une seule transaction.

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; APM &amp; Services &gt; (select an app) &gt; Transactions**</DNT>.
2. Sélectionnez une transaction spécifique dans la liste.
3. Le diagramme de flux dynamique se charge automatiquement, affichant uniquement les services impliqués dans cette transaction.

### À partir de Transaction 360 [#from-transaction-360]

Le <DNT>**Dynamic Flow Map**</DNT> apparaît comme un composant dans la vue [Transaction 360](/docs/apm/transactions/workload-performance-monitoring/transaction-workloads/) de toute transaction. La carte affiche les services participant à la transaction et le temps nécessaire à chaque service.

<Callout variant="tip">
  Dans les vues spécifiques à une transaction, la table <DNT>**performance by transactions**</DNT> n&apos;est pas disponible car la vue est déjà axée sur une seule transaction. Toutefois, lorsque vous accédez à d&apos;autres fonctionnalités à partir de ces vues, le filtre transaction est conservé, ce qui vous permet de poursuivre votre analyse ciblée.
</Callout>

Le <DNT>**Dynamic Flow Map**</DNT> montre les relations entre le nœud focal (mis en évidence par un projecteur gris) et l&apos;entité environnante. Les services en amont sont représentés par des arêtes pointant vers le nœud central, tandis que les services en aval sont représentés par des arêtes pointant dans la direction opposée. Vous pouvez interagir avec la carte pour visualiser les tendances de performance, les anomalies et d&apos;autres détails comme suit :

<CollapserGroup>
  <Collapser id="anomalies" title="Analyser l'anomalie">
    La carte affiche les anomalies à l&apos;aide de contours et de points colorés :

    **Couleurs des bords** (indiquant un trafic anormal entre les services) :

    * **Bord bleu**: anomalie de latence détectée dans le trafic entre les services.
    * **Bord rose**: Anomalie d’erreur détectée dans le trafic entre les services.
    * **Bordure rayée bleue et rose**: Anomalie de latence et d’erreur détectée dans le trafic entre les services.

    **Points à côté des nœuds** (indiquant une anomalie au sein d&apos;un service) :

    * **Point bleu**: Le service rencontre une anomalie de latence.
    * **Point rose**: Le service rencontre une anomalie d&apos;erreur.

    <Callout variant="tip">
      Si vous voyez un nœud avec un point d&apos;anomalie mais que certaines arêtes de connexion ne sont pas colorées, c&apos;est parce que les anomalies sont calculées indépendamment pour les nœuds et les arêtes. Par exemple, une anomalie de latence peut ne se produire que sur certains chemins d&apos;appel vers cette entité.
    </Callout>

    ### Filtrer les anomalies par type

    Utilisez la légende interactive en haut de la carte pour vous concentrer sur des types d&apos;anomalies spécifiques :

    1. Cliquez sur **Latency** dans la légende pour afficher uniquement les anomalies de latence (les indicateurs d&apos;erreur sont alors masqués).
    2. Cliquez sur **Errors** dans la légende pour afficher uniquement les anomalies d&apos;erreur (les indicateurs de latence sont alors masqués).

    Ce filtrage vous aide à éliminer le bruit lorsque vous souhaitez vous concentrer sur un type de problème de performance.
  </Collapser>

  <Collapser id="clusters" title="Travailler avec une entité en cluster">
    Pour que votre carte reste organisée, celle-ci regroupe automatiquement les entités présentant de faibles scores d&apos;anomalie en clusters. Vous pouvez développer le cluster pour afficher chaque entité individuellement, selon vos besoins.

    Pour afficher et gérer une entité au sein d&apos;un cluster:

    1. Cliquez sur un nœud du cluster. La liste de toutes les entités de ce cluster apparaît dans le panneau de droite.
    2. Pour afficher une entité séparément sur la carte, cliquez sur son icône de visibilité dans le panneau de droite.
  </Collapser>

  <Collapser id="multi-select-mode" title="Filtrer par plusieurs entités">
    Utilisez le mode de sélection multiple lorsque vous devez examiner les traces qui transitent par une entité spécifique de votre architecture de service. Cela vous permet de cibler votre analyse afin de comprendre comment des services spécifiques interagissent au sein d&apos;un chemin de trace.

    Pour filtrer par plusieurs entités :

    1. Cliquez sur l&apos;icône de curseur dans le coin supérieur droit pour entrer en mode de sélection multiple.
    2. Cliquez sur l&apos;entité que vous souhaitez inclure. Une barre latérale affiche vos sélections.
    3. Cliquez sur **Apply** pour redessiner la carte et n&apos;afficher que les traces qui passent à la fois par le nœud focal et par l&apos;entité sélectionnée.

    En mode de sélection multiple :

    * Vous pouvez sélectionner uniquement une entité individuelle. Il est impossible d&apos;ajouter ce cluster.
    * Le nœud focal reste toujours dans le filtre et ne peut pas être retiré.

    <Callout variant="tip">
      Lors du filtrage par plusieurs entités, vous pouvez obtenir une carte vide si ces entités n&apos;apparaissent pas ensemble dans aucune trace. Pour éviter cela, appliquez un filtre d&apos;entité à la fois pour affiner progressivement la carte.
    </Callout>
  </Collapser>

  <Collapser id="performance-trends" title="Afficher les informations détaillées de l'entité">
    Lorsque vous devez examiner les performances d&apos;un service spécifique et sa relation avec le nœud central, cliquez sur n&apos;importe quelle entité de la carte pour ouvrir une barre latérale détaillée.

    La barre latérale vous aide à :

    * **Accédez aux pages associées**: utilisez les liens d’en-tête pour naviguer vers la page APM de l’entité ou la page de l’équipe propriétaire.
    * **Analyser les performances**: Examiner les graphiques chronologiques présentant les indicateurs de suivi pour l&apos;entité sélectionnée et le nœud central. Des badges d&apos;anomalie apparaissent sur les graphiques lorsque l&apos;entité présente des problèmes de performance.
    * **Analyser les transactions**: Identifiez les transactions impliquant le nœud central et accédez aux informations détaillées transaction.
    * **Afficher la trace**: Il existe deux boutons **View traces** avec des contextes de filtrage différents. Le bouton sur la carte principale ouvre la page d&apos;accueil de tracing avec tous les filtres actuellement appliqués (filtres d&apos;entités, filtres transaction, plage horaire). Le bouton dans la barre latérale ouvre la page d&apos;accueil de tracing avec tous les filtres cartographiques de l&apos;entité actuellement sélectionnée.
    * **Gérer la propriété**: Attribuer l&apos;entité à des équipes spécifiques.
    * **Gérer les balises**: Afficher et ajouter des balises à l’entité sélectionnée.
  </Collapser>

  <Collapser id="filter-by-transaction" title="Filtrer par transaction">
    Le tableau <DNT>**performance by transactions**</DNT> affiche les données de séries temporelles pour chaque transaction associée au nœud focal. Utilisez ce tableau lorsque vous souhaitez concentrer la carte sur les traces incluant une transaction spécifique.

    Pour filtrer par transaction spécifique :

    1. À partir du tableau <DNT>**performance by transactions**</DNT> situé en bas de la carte de flux dynamique, examinez les données de séries temporelles pour chaque transaction.
    2. Cliquez sur <DNT>**Generate map**</DNT> à côté de la transaction que vous souhaitez analyser.

    La carte se redessine et n&apos;affiche que les traces contenant ce nom transaction.
  </Collapser>

  <Collapser id="time-picker" title="Ajuster la plage horaire">
    Le sélecteur d&apos;intervalle de temps vous permet de sélectionner une plage de temps spécifique pour analyser les données trace. Vous pouvez choisir parmi des plages horaires prédéfinies (comme les 30 dernières minutes, la dernière heure) ou définir une plage horaire personnalisée.

    Pour ajuster la plage horaire :

    1. Cliquez sur le sélecteur d&apos;intervalle de temps en haut de la page.
    2. Sélectionnez une plage horaire prédéfinie ou définissez une heure de début et de fin personnalisée.

    La carte se met à jour pour afficher les données trace pour la plage horaire sélectionnée. Si vous sélectionnez une plage horaire supérieure à 3 heures, la carte utilise automatiquement les 3 dernières heures de cette plage.
  </Collapser>
</CollapserGroup>
