Utilisez le modèle AWS CloudFormation automatisé pour accélérer votre configuration de Cloud Cost Intelligence. La pile préconfigurée provisionne automatiquement l&apos;environnement AWS requis, y compris la configuration du bucket S3, les paramètres d&apos;exportation CUR, [la vérification des paramètres Trusted Advisor](/docs/cci/financial-intelligence/cci-recommendations/#aws-trusted-advisor), ainsi que les rôles et stratégies IAM nécessaires. Vérifiez toutes les autorisations de ressources dans la console AWS CloudFormation avant le déploiement.

## Considérations relatives aux coûts

Le modèle crée automatiquement un bucket S3 pour l&apos;ingestion des données CUR. La région par défaut est `us-east-2`, mais vous pouvez la modifier à l&apos;aide des paramètres CloudFormation pour répondre aux besoins de votre infrastructure.

## Instructions de configuration

<Steps>
  <Step>
    ### Accéder à Cloud Cost Intelligence dans New Relic

    Accédez à <DNT>[**one.newrelic.com**](https://one.newrelic.com/) &gt; **Integrations &amp; Agents** &gt; **Cloud Cost Intelligence - AWS**</DNT>.
  </Step>

  <Step>
    ### Choisir l&apos;automatisation CloudFormation

    Sélectionnez **Automate AWS with CloudFormation (Recommended)** et cliquez sur **Continue**.
  </Step>

  <Step>
    ### Lancer CloudFormation dans AWS

    1. Confirmez le nom de votre connexion et sélectionnez votre région de données New Relic.
    2. Cliquez sur **Launch your CloudFormation in AWS**.
    3. Connectez-vous à votre compte AWS avec le **rôle Admin** lorsque vous y êtes invité. L&apos;écran **CloudFormation Stacks** s&apos;ouvre dans la console AWS.
  </Step>

  <Step>
    ### Configurer les paramètres de la pile

    Dans le formulaire **Create Stack**, configurez les paramètres suivants :

    **Paramètres pré-remplis (ne pas modifier) :**

    * **ID de compte New Relic**: Pré-rempli. Ne pas modifier.
    * **Région de données New Relic**: Pré-rempli. Ne pas modifier.
    * **ID externe**: Pré-rempli. Ne pas modifier.

    **Paramètres requis :**

    * **NewRelicAPIKey**: Saisissez votre clé API New Relic. Générez ceci dans l&apos;interface utilisateur New Relic : Menu utilisateur &gt; API Keys.
    * **Nom de la pile**: Valeur par défaut fournie. Modifiez si nécessaire, mais assurez-vous que le nom est unique au niveau mondial dans toutes les piles AWS CloudFormation. Nous recommandons de conserver la valeur par défaut.
    * **Nom du bucket**: Valeur par défaut fournie. Modifiez si nécessaire, mais assurez-vous que le nom est unique à l&apos;échelle mondiale parmi tous les buckets S3.
    * **Nom de la connexion**: Valeur par défaut fournie. Modifiez si nécessaire selon vos conventions de nommage.

    **Paramètres optionnels :**

    * **Accorder l&apos;accès à Trusted Advisor**: défini par défaut sur **Yes**. Une fois activé, permet de vérifier si les informations sur les coûts **Trusted Advisor** sont activées pour votre compte et récupère les recommandations. Si vous choisissez **No** ou si **Trusted Advisor** n&apos;est pas activé, les recommandations de **Trusted Advisor** ne seront pas disponibles.

    <Callout variant="important">
      Les recommandations natives CCI sont indépendantes de **Trusted Advisor** et fonctionneront quel que soit ce paramètre.
    </Callout>

    Tous les autres paramètres sont facultatifs et peuvent être configurés selon les besoins.
  </Step>

  <Step>
    ### Créer la pile

    Sélectionnez **I acknowledge that AWS CloudFormation might create IAM resources** et cliquez sur **Create stack**.
  </Step>

  <Step>
    ### Vérifier la création de la pile

    Lorsque la création de la pile est terminée avec succès, le statut affiche **CREATE\_COMPLETE**.
  </Step>

  <Step>
    ### Vérifier la génération du rapport CUR

    Attendez 24 heures, puis vérifiez qu&apos;un rapport CUR au format parquet apparaît dans votre bucket AWS.

    <Callout variant="important">
      Le chemin ci-dessous est un exemple. Votre chemin réel aura des horodatages, des UUID et des périodes de facturation différents.

      > **Amazon S3** &gt; **Buckets** &gt; **bucket\_name** &gt; **cost-and-usage-reports/** &gt; **bucket\_name-cci/data** &gt; **BILLING\_PERIOD=2025-12/** &gt; **2025-12-24T03:05:19.886Z-a2e23b2a-588a-4305-97fd-be712f555b75/** &gt; **bucket\_name-cci-00001.snappy.parquet**
    </Callout>

    La configuration de votre intégration est terminée.
  </Step>
</Steps>

## Limites

Le modèle CloudFormation automatisé ne :

* Créez ou configurez des ressources de calcul ou réseau.
* Activez tout service AWS supplémentaire ou mettez à jour le niveau de support.
* Automatisez le processus d&apos;ingestion des logs, des traces et de toute autre métrique.
* Créez des rôles ou des politiques d&apos;accès en écriture.

## Stratégies et autorisations configurées

La stack automatisée configure les niveaux d&apos;accès suivants :

* **Accès aux données de facturation**: autorisation en lecture seule au bucket S3 spécifique créé pour stocker les rapports sur les coûts et l&apos;utilisation.
* **Accès à la tarification des services**: Accès en lecture seule à l&apos;API AWS Pricing pour corréler votre utilisation avec les tarifs publics.
* **Automatisation de la configuration**: autorisations limitées requises pour générer les exports CUR et vérifier l&apos;état de l&apos;intégration pendant la phase de configuration.
* **Accès facultatif à l&apos;optimisation**: (Si sélectionné) Accès en lecture seule à AWS Trusted Advisor pour faire remonter des recommandations de réduction des coûts.
