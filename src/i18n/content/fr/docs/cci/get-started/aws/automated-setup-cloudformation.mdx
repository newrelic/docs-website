Utilisez le modèle AWS CloudFormation automatisé pour accélérer votre configuration de <DNT>**Cloud Cost Intelligence**</DNT>. La stack préconfigurée provisionne automatiquement l&apos;environnement AWS requis, y compris la configuration du bucket S3, les paramètres d&apos;exportation CUR, [la vérification des paramètres Trusted Advisor](/docs/cci/financial-intelligence/cci-recommendations/#aws-trusted-advisor), ainsi que les rôles et stratégies IAM nécessaires. Vérifiez toutes les autorisations de ressources dans la console AWS CloudFormation avant de déployer.

## Considérations relatives aux coûts

Le modèle crée automatiquement un bucket S3 pour l&apos;ingestion des données CUR. La région par défaut est `us-east-2`, mais vous pouvez la modifier à l&apos;aide des paramètres CloudFormation pour répondre aux besoins de votre infrastructure.

## Instructions de configuration

<Steps>
  <Step>
    ### Accéder à Cloud Cost Intelligence dans New Relic

    Accédez à <DNT>[**one.newrelic.com**](https://one.newrelic.com/)</DNT> <DNT>**Integrations &amp; Agents**</DNT> <DNT>**Cloud Cost Intelligence - AWS**</DNT>.
  </Step>

  <Step>
    ### Choisir l&apos;automatisation CloudFormation

    Sélectionnez <DNT>**Automate AWS with CloudFormation (Recommended)**</DNT> et cliquez sur <DNT>**Continue**</DNT>.
  </Step>

  <Step>
    ### Lancer CloudFormation dans AWS

    1. Confirmez le nom de votre connexion et sélectionnez votre région de données New Relic.
    2. Cliquez sur <DNT>**Launch your CloudFormation in AWS**</DNT>.
    3. Connectez-vous à votre compte AWS avec <DNT>**Admin role**</DNT> lorsque vous y êtes invité. L&apos;écran <DNT>**CloudFormation Stacks**</DNT> s&apos;ouvre dans la console AWS.
  </Step>

  <Step>
    ### Configurer les paramètres de la pile

    Dans le formulaire <DNT>**Create Stack**</DNT>, configurez les paramètres suivants :

    <DNT>**Pre-populated parameters (do not modify):**</DNT>

    * <DNT>**New Relic Account ID**</DNT>: Pré-rempli. Ne pas modifier.
    * <DNT>**New Relic Data Region**</DNT>: Pré-rempli. Ne pas modifier.
    * <DNT>**External ID**</DNT>: Pré-rempli. Ne pas modifier.

    <DNT>**Required parameters:**</DNT>

    * <DNT>**NewRelicAPIKey**</DNT>: Saisissez votre clé API New Relic. Générez ceci dans l&apos;UI de New Relic : Menu utilisateur &gt; API Keys.
    * <DNT>**Stack Name**</DNT>: Valeur par défaut fournie. Modifiez si nécessaire, mais assurez-vous que le nom est unique au niveau mondial dans tous les stacks AWS CloudFormation. Nous recommandons de conserver la valeur par défaut.
    * <DNT>**Bucket Name**</DNT>: Valeur par défaut fournie. Modifiez si nécessaire, mais assurez-vous que le nom est unique à l&apos;échelle mondiale parmi tous les buckets S3.
    * <DNT>**Connection Name**</DNT>: Valeur par défaut fournie. Modifiez si nécessaire selon vos conventions de nommage.

    <DNT>**Optional parameters:**</DNT>

    * <DNT>**Grant Access to Trusted Advisor**</DNT>: la valeur par défaut est <DNT>**Yes**</DNT>. Une fois activé, permet de vérifier si les informations détaillées sur les coûts <DNT>**Trusted Advisor**</DNT> sont activées pour votre compte et récupère les recommandations. Si vous choisissez <DNT>**No**</DNT> ou si <DNT>**Trusted Advisor**</DNT> n&apos;est pas activé, les recommandations de <DNT>**Trusted Advisor**</DNT> ne seront pas disponibles.

    <Callout variant="important">
      <DNT>**CCI**</DNT> Les recommandations natives sont indépendantes de <DNT>**Trusted Advisor**</DNT> et fonctionneront quel que soit ce paramètre.
    </Callout>

    Tous les autres paramètres sont facultatifs et peuvent être configurés selon les besoins.
  </Step>

  <Step>
    ### Créer la pile

    Sélectionnez <DNT>**I acknowledge that AWS CloudFormation might create IAM resources**</DNT> et cliquez sur <DNT>**Create stack**</DNT>.
  </Step>

  <Step>
    ### Vérifier la création de la pile

    Lorsque la création du stack est terminée avec succès, le statut affiche <DNT>**CREATE\_COMPLETE**</DNT>.
  </Step>

  <Step>
    ### Vérifier la génération du rapport CUR

    Attendez 24 heures, puis vérifiez qu&apos;un rapport CUR au format parquet apparaît dans votre bucket AWS.

    <Callout variant="important">
      Le chemin ci-dessous est un exemple. Votre chemin réel aura des horodatages, des UUID et des périodes de facturation différents.

      <DNT>**Amazon S3**</DNT> &gt; <DNT>**Buckets**</DNT> &gt; <DNT>**bucket\_name**</DNT> &gt; <DNT>**cost-and-usage-reports/**</DNT> &gt; <DNT>**bucket\_name-cci/data**</DNT> &gt; <DNT>**BILLING\_PERIOD=2025-12/**</DNT> &gt; <DNT>**2025-12-24T03:05:19.886Z-a2e23b2a-588a-4305-97fd-be712f555b75/**</DNT> &gt; <DNT>**bucket\_name-cci-00001.snappy.parquet**</DNT>
    </Callout>

    La configuration de votre intégration est terminée.
  </Step>
</Steps>

## Limites

Le modèle <DNT>**CloudFormation**</DNT> automatisé ne :

* Créez ou configurez des ressources de calcul ou réseau.
* Activez tout service AWS supplémentaire ou mettez à jour le niveau de support.
* Automatisez le processus d&apos;ingestion des logs, des traces et de toute autre métrique.
* Créez des rôles ou des politiques d&apos;accès en écriture.

## Stratégies et autorisations configurées

La stack automatisée configure les niveaux d&apos;accès suivants :

* <DNT>**Billing Data Access**</DNT>: autorisation en lecture seule au bucket S3 spécifique créé pour stocker les rapports sur les coûts et l&apos;utilisation.
* <DNT>**Service Pricing Access**</DNT>: Accès en lecture seule à l&apos;API AWS Pricing pour corréler votre utilisation avec les tarifs publics.
* <DNT>**Configuration Automation**</DNT>: autorisations limitées requises pour générer les exports CUR et vérifier l&apos;état de l&apos;intégration pendant la phase de configuration.
* <DNT>**Optional Optimization Access**</DNT>: (Si sélectionné) Accès en lecture seule à AWS Trusted Advisor pour faire remonter des recommandations de réduction des coûts.
