Connectez votre facturation Azure à Intelligence Coûts du cloud pour un monitoring complet des coûts et une optimisation de votre infrastructure Azure. Obtenez une visibilité en temps réel sur les habitudes de dépenses, l’utilisation des ressources et la répartition des coûts entre les services et les groupes de ressources. Avec Intelligence Coûts du cloud pour Azure, vous pouvez :

* **Optimiser l&apos;utilisation des ressources** - Trouvez les services Azure et les machines virtuelles sous-utilisés ou coûteux pour ajuster les allocations entre les groupes d&apos;abonnements et de ressources
* **Contrôlez les dépenses** - Consultez les facteurs de coûts spécifiques dans votre environnement Azure, notamment le calcul, le stockage, la mise en réseau et le service de base de données pour de meilleures décisions financières
* **Améliorer l&apos;efficacité** - Concentrez-vous sur les stratégies de gestion des ressources qui s&apos;alignent sur les objectifs commerciaux pour une utilisation efficace des ressources et du budget Azure
* **Monitorer les coûts multi-abonnements** - Suivez les dépenses sur plusieurs groupes d&apos;abonnement et de gestion Azure à partir d&apos;un seul dashboard
* **Activer la responsabilisation des coûts** - Mettre en œuvre la répartition des coûts et la rétrofacturation au niveau du service à l&apos;aide de l&apos;organisation des balises Azure et des groupes de ressources
* **Gérez les budgets de manière proactive** - Configurez des alertes pour le seuil de dépenses Azure et les limites budgétaires pour éviter les dépassements et maintenir le contrôle financier

## Prérequis

Avant de connecter Azure à Intelligence Coûts du cloud, assurez-vous d&apos;avoir :

* Un **abonnement Azure** avec un compte de facturation actif.
* **Rôle de contributeur ou de propriétaire** sur l&apos;abonnement que vous souhaitez monitorer.
* **Accès à la fonctionnalité Azure Cost Management**.
* **Autorisations de lecture de facturation** pour l&apos;abonnement ou le groupe de gestion.
* **configurationd’exportation de facturation Azure terminée**: cela établit le compte de stockage, le conteneur et le chemin d’accès au répertoire nécessaires au processus de connexion. Consultez [la configuration de l’exportation de facturation Azure](#azure-billing-export-setup) pour plus de détails.

## Connecter Azure à Intelligence Coûts du cloud

<Steps>
  <Step>
    ### Sélectionnez votre compte

    Maintenant que vous avez terminé la configuration de l’exportation de la facturation Azure, vous êtes prêt à connecter votre facturation Azure à New Relic. Commencez par choisir votre compte New Relic.

    1. Allez à <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence - Azure**</DNT>.
    2. Sélectionnez votre compte dans la liste déroulante et cliquez sur **Continue**.
  </Step>

  <Step>
    ### Accordez à New Relic l&apos;accès à vos données de facturation

    Maintenant que vous avez fourni les détails de votre connexion Azure, New Relic a besoin de l’autorisation de lire les données de facturation à partir de votre conteneur de stockage. Cette étape autorise le principal du service New Relic (**msazurecostdata**) à accéder à vos exportations de facturation en toute sécurité.

    1. Cliquez sur le bouton **Initiate Connection** dans l’interface New Relic.
    2. Vous serez redirigé vers le flux **d’authentification Microsoft**.

    <Callout variant="tip">
      L’approbation de l’administrateur Azure est requise pour cette étape.
    </Callout>

    3. Vérifiez et acceptez la demande d’authentification.
    4. Un petit fichier (0 Ko) sera téléchargé sur votre système dès acceptation. Vous pouvez supprimer ce fichier en toute sécurité.
  </Step>

  <Step>
    ### Vérifiez que le principal du service a été ajouté

    Après avoir accepté la demande d’authentification, confirmez que le principal du service New Relic a été ajouté avec succès à votre environnement Azure :

    1. Accédez à **Enterprise Applications** dans votre **compte Microsoft Entra**.
    2. Vérifiez que le principal de service **msazurecostdata** apparaît dans la liste.

    <Callout variant="tip">
      Pour obtenir de l’aide sur l’affichage des applications d’entreprise, consultez la [documentation Microsoft Entra](https://learn.microsoft.com/en-us/entra/identity/enterprise-apps/view-applications-portal).
    </Callout>
  </Step>

  <Step>
    ### Attribuer des autorisations de lecture au principal du service

    Attribuez le rôle \*\* Lecteur de données Blob de stockage \*\* au principal du service pour un accès en lecture seule aux exportations de facturation, garantissant ainsi l&apos;accès au compte de stockage et les autorisations pour modifier les attributions de rôles IAM :

    1. Ouvrez le conteneur de compte de stockage dans lequel vos exportations de facturation sont publiées. Sélectionnez **Access Control (IAM)** dans le menu de navigation de gauche.
    2. Sous l’onglet **Role Assignments**, cliquez sur **Add** ou **Add role assignment**.
    3. Sélectionnez **Storage Blob Data Reader** dans la liste des rôles pour fournir un accès en lecture seule aux exportations de facturation.
    4. Dans la section **Add Members**, recherchez `msazurecostdata` (principal de service vérifié) et sélectionnez-le.
    5. Vérifiez la configuration de l’attribution des rôles. Cliquez sur **Review + Assign** et vérifiez que `msazurecostdata` apparaît dans la liste des attributions de rôles.
  </Step>

  <Step>
    ### Vérifier la connexion

    Après avoir effectué toutes les étapes ci-dessus, confirmez que :

    * Le principal de service **msazurecostdata** est visible dans les applications d&apos;entreprise
    * Le principal du service dispose du rôle de **Storage Blob Data Reader** attribué
    * L&apos;attribution de rôle est appliquée au conteneur de compte de stockage approprié
    * Le test de connexion se termine avec succès dans l&apos;interface New Relic

    <Callout variant="tip">
      **Note de sécurité**: le principal du service n&apos;a qu&apos;un accès en lecture seule à vos données de facturation. Il ne peut pas écrire, modifier ou supprimer aucune donnée. L&apos;accès est limité exclusivement au conteneur d&apos;exportation de facturation que vous avez spécifié.
    </Callout>
  </Step>

  <Step>
    ### Et ensuite ?

    * New Relic commencera à traiter vos exportations de facturation Azure
    * Le traitement initial des données peut prendre jusqu&apos;à 12 heures
    * Une fois le traitement terminé, vous verrez vos données de coût dans le dashboard Aperçu des coûts

    <Callout variant="important">
      Le traitement de vos données pour la première fois peut prendre jusqu&apos;à 12 heures.
    </Callout>
  </Step>

  <Step>
    ### Visualisez vos données

    Une fois le traitement terminé, accédez à [**Cost Overview**](/docs/cci/financial-intelligence/cloud-cost-insights) pour afficher vos données de coûts Azure et commencer à optimiser vos dépenses cloud.
  </Step>
</Steps>
