Lors de la mise en place d&apos;Intelligence Coûts du cloud, vous pourriez rencontrer quelques problèmes. Voici quelques problèmes courants, les raisons possibles et les étapes de dépannage suggérées pour vous aider à les résoudre efficacement :

## Les données n&apos;apparaissent pas dans New Relic

Si vous ne voyez pas le rapport d&apos;utilisation des coûts (CUR), l&apos;estimation des coûts en temps réel ou les données d&apos;allocation des coûts Kubernetes dans votre compte New Relic, voici quelques étapes pour résoudre le problème :

* **Motif :** Il peut y avoir des lacunes dans la réalisation des prérequis.

* **Solution:**

  1. Vérifiez que vous avez rempli toutes les conditions préalables requises.
  2. Vérifiez votre configuration d&apos;installation.
  3. Si le problème persiste, contactez [le support Intelligence Coûts du cloud](mailto:cci-support@newrelic.com).

## Perte de connexion

* **Raison :** une connexion réseau instable peut entraîner l’échec de la configuration.
* **Solution :** assurez-vous d’une connexion Internet stable et réessayez la configuration. Si le problème persiste, vérifiez auprès de votre administrateur réseau les éventuelles restrictions de pare-feu.

## Privilèges utilisateur inadéquats

* **Motif :** L&apos;utilisateur ne dispose pas des autorisations nécessaires pour mettre en place Intelligence Coûts du cloud.
* **Solution :** vérifiez les autorisations de votre compte AWS et New Relic. Assurez-vous de disposer des privilèges adéquats pour effectuer la configuration. Contactez votre administrateur si nécessaire.

## Erreur de configuration ARN

* **Message :** « L&apos;ARN que vous avez saisi ne permet pas l&apos;accès correct à votre compte AWS ; veuillez vérifier l&apos;ARN et réessayer. »
* **Motif :** configuration ARN incorrecte qui ne permet pas l&apos;accès.
* **Solution :** vérifiez l’ARN que vous avez saisi et assurez-vous qu’il correspond à la configuration requise pour l’accès. Corrigez les erreurs et réessayez la configuration.

## risque de sécurité avec pièce d&apos;identité externe

* **Message :** &quot;risque de sécurité : L&apos;ARN que vous avez saisi dispose d&apos;autorisations étendues. N&apos;importe quel « identifiant externe » peut être utilisé pour assumer un rôle. Veuillez ajouter l&apos;ID externe fourni « %s » comme externalId lors de la création du rôle AWS.
* **Raison :** Une politique d’identification externe trop permissive présente un risque de sécurité.
* **Solution :** mettez à jour vos paramètres de rôle AWS pour garantir que l’ID externe fourni est utilisé comme externalId.

## Le bucket n&apos;existe pas

* **Motif :** le compartiment AWS S3 spécifié n’existe pas.
* **Solution :** vérifiez que le nom du bucket correspond à celui de votre compte AWS. Le nom doit également avoir le préfixe `cost-and-usage-reports`. Créez le bucket s&apos;il n&apos;existe pas ou corrigez le nom dans la configuration.

## Erreur de nom de bucket

* **Motif :** le compartiment AWS S3 spécifié n&apos;existe pas ou il y a une faute de frappe dans le nom du compartiment.
* **Solution :** vérifiez que le nom du bucket correspond à celui de votre compte AWS et inclut le préfixe `cost-and-usage-reports`. Assurez-vous que le nom est correctement orthographié. Si le bucket n&apos;existe pas, créez-le ou corrigez le nom dans la configuration.

## Nom de connexion en double

* **Message :** « %s Intelligence Coûts du cloud Le nom de connexion AWS existe déjà. Veuillez saisir un nouveau nom de connexion.
* **Raison :** Le nom de connexion que vous avez saisi est déjà utilisé.
* **Solution :** choisissez un nom de connexion unique et réessayez.

## configurationdu compte payeur en double

* **Message :** « %s Le compte AWS Payer est déjà lié à ce compte New Relic. »
* **Motif :** tentative de configuration du même compte payeur AWS plusieurs fois.
* **Solution :** vérifiez que le compte payeur AWS n’est pas déjà lié. Utilisez un autre compte payeur si nécessaire.

## Support supplémentaire

Si votre problème persiste après avoir suivi ces étapes de dépannage, contactez [le support Intelligence Coûts du cloud](mailto:cci-support@newrelic.com).
