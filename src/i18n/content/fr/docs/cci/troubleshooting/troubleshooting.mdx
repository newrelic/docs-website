<Callout title="Aperçu">
  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Si vous rencontrez des problèmes lors de la configuration, trouvez des solutions ci-dessous :

## Données manquantes dans les rapports

Si vous ne voyez pas le rapport d&apos;utilisation des coûts (CUR), l&apos;estimation des coûts en temps réel ou les données d&apos;allocation des coûts Kubernetes :

* **Raison :** Conditions préalables incomplètes

* **Solution:**

  1. Vérifiez que toutes les conditions préalables à la configuration sont remplies.
  2. Vérifier la configuration de l&apos;installation
  3. Contactez [l&apos;assistance Cloud Cost Intelligence](mailto:cci-support@newrelic.com) si le problème persiste

## La connexion échoue lors de la configuration

* **Raison :** Connexion réseau instable
* **Solution :** Assurez-vous d&apos;une connexion Internet stable et réessayez la configuration. Vérifiez auprès de l&apos;administrateur réseau les restrictions du pare-feu si le problème persiste.

## Permissions insuffisantes

* **Raison :** L’utilisateur ne dispose pas des autorisations nécessaires
* **Solution :** Vérifiez que vous disposez des autorisations d&apos;administrateur ou des stratégies IAM requises dans AWS et New Relic. Contactez l&apos;administrateur si nécessaire.

## Erreur de configuration ARN

* **Message :** « L&apos;ARN que vous avez saisi ne permet pas l&apos;accès correct à votre compte AWS ; veuillez vérifier l&apos;ARN et réessayer. »
* **Raison :** Configuration ARN incorrecte
* **Solution :** Vérifiez que l&apos;ARN correspond à la configuration requise. Corrigez les erreurs et réessayez la configuration.

## risque de sécurité avec pièce d&apos;identité externe

* **Message :** &quot;risque de sécurité : L&apos;ARN que vous avez saisi dispose d&apos;autorisations étendues. N&apos;importe quel « identifiant externe » peut être utilisé pour assumer un rôle. Veuillez ajouter l&apos;ID externe fourni « %s » comme externalId lors de la création du rôle AWS.
* **Raison :** Politique d&apos;ID externe trop permissive
* **Solution :** Mettez à jour les paramètres du rôle AWS pour utiliser l&apos;ID externe fourni comme externalId.

## Bucket S3 introuvable

* **Raison :** Le bucket S3 n&apos;existe pas ou le nom du bucket est incorrect.
* **Solution :** Vérifiez que le nom du bucket correspond à votre compte AWS. Assurez-vous que le préfixe du chemin S3 est `cost-and-usage-reports`. Vérifiez l&apos;orthographe, créez un bucket si nécessaire ou corrigez le nom.

## Nom de connexion en double

* **Message :** « %s Intelligence Coûts du cloud Le nom de connexion AWS existe déjà. Veuillez saisir un nouveau nom de connexion.
* **Raison :** Nom de connexion déjà utilisé
* **Solution :** Choisissez un nom de connexion unique et réessayez.

## configurationdu compte payeur en double

* **Message :** « %s Le compte AWS Payer est déjà lié à ce compte New Relic. »
* **Raison :** Compte payeur AWS déjà configuré
* **Solution :** Vérifiez que le compte payeur n&apos;est pas déjà lié. Utilisez un autre compte payeur si nécessaire.

## Support supplémentaire

Si les problèmes persistent, contactez le [support Cloud Cost Intelligence](mailto:cci-support@newrelic.com).

## Configuration de CloudFormation

Si **Create CFT Stack** échoue :

1. Recherchez la ligne Status dans l&apos;onglet/la colonne Events de la stack sélectionnée.
2. Recherchez **CREATE\_FAILED** et partagez le message de la raison du statut avec New Relic.
3. Accédez à **CloudWatch** et sous **CloudWatch logs**, sélectionnez le nom de votre pile CloudFormation et cliquez sur **View in Logs console**.
4. Sous **Log Streams**, ouvrez le log et partagez le log avec le support New Relic.
