Accédez aux informations détaillées spécifiques à vos environnements Kubernetes via la page <DNT>**Kubernetes Costs**</DNT>. Cette fonctionnalité exploite les données télémétriques de New Relic pour fournir une vue détaillée des coûts associés à vos clusters Kubernetes.

Utilisez la page <DNT>**Kubernetes Costs**</DNT> pour afficher les coûts encourus par chaque application au fil du temps et une analyse détaillée des coûts.

<img title="Kubernetes Cost Insights highlighted on the Cloud Cost Intelligence Kubernetes Cost Insights page" alt="Kubernetes Cost Insights highlighted on the Cloud Cost Intelligence Kubernetes Cost Insights page" src="/images/k8s-cci.webp" />

## Filtres et personnalisation

### Filtres de fournisseur de cloud

Pour filtrer vos données de coûts par fournisseur cloud, sélectionnez une option dans le menu déroulant en haut à gauche de la page :

* <DNT>**All**</DNT>: Tous les fournisseurs de cloud
* <DNT>**AWS**</DNT>: Amazon Web Services uniquement
* <DNT>**Azure**</DNT>: Microsoft Azure uniquement

### Sélecteur de plage de dates

En haut à droite de la page, utilisez le sélecteur de plage de dates pour :

* Choisissez une période prédéfinie
* Définir des dates de début et de fin personnalisées
* Affichez les coûts dans votre fuseau horaire préféré (UTC affiché par défaut)

### Filtre des coûts d&apos;ingénierie

Si votre organisation utilise des tags d&apos;allocation des coûts, vous pouvez filtrer par le tag <DNT>**Engineering costs**</DNT> pour restreindre l&apos;analyse à des équipes, projets ou centres de coûts spécifiques taggés dans votre environnement cloud.

## Résumé

La section Summary affiche quatre métriques de coûts clés pour votre période sélectionnée :

* <DNT>**Total cost**</DNT>: les dépenses globales pour tous les clusters Kubernetes avec la variation en pourcentage par rapport à la période précédente
* <DNT>**Average cost (Daily)**</DNT>: la dépense moyenne quotidienne avec la variation en pourcentage
* <DNT>**Unused cost**</DNT>: Le coût des ressources qui ont été allouées mais qui ne reçoivent pas de requests, avec la variation en pourcentage
* <DNT>**Unallocated cost**</DNT>: Le coût des ressources du cluster qui n&apos;ont été allouées à aucune source, avec la variation en pourcentage

## Insights

La section Insights fournit deux widgets de visualisation pour vous aider à analyser vos coûts Kubernetes.

### Tendances des coûts

Le widget <DNT>**Cost Trends**</DNT> affiche un graphique linéaire montrant l&apos;évolution de vos coûts Kubernetes au fil du temps. Utilisez les options d&apos;affichage pour basculer entre les vues <DNT>**Hourly**</DNT>, <DNT>**Daily**</DNT>, <DNT>**Weekly**</DNT> ou <DNT>**Monthly**</DNT>.

### Top 20

Le widget <DNT>**Top 20**</DNT> affiche vos domaines de coûts les plus élevés à l&apos;aide d&apos;une visualisation sous forme de graphique en arbre. Par défaut, il montre les 20 principaux clusters par coût. Vous pouvez personnaliser la vue à l&apos;aide de :

* <DNT>**Dropdown filter**</DNT>: sélectionner ce qu&apos;il faut afficher (<DNT>**Cluster name, by cost**</DNT>)
* <DNT>**Group by**</DNT>: Choisissez des dimensions pour regrouper les données de coût
* <DNT>**View by**</DNT>: Basculer entre <DNT>**Tree chart**</DNT> et d&apos;autres formats de visualisation

## Cost &amp; Usage Analysis

La section <DNT>**Cost &amp; Usage Analysis**</DNT> présente un tableau détaillé avec une répartition de l&apos;utilisation des ressources et des coûts par nom de cluster :

* <DNT>**Cluster name**</DNT>: le nom de votre cluster Kubernetes
* <DNT>**CPU cost**</DNT>: le coût attribué à l&apos;utilisation du CPU pour chaque cluster
* <DNT>**CPU usage**</DNT>: Le pourcentage de ressources CPU consommées par chaque cluster
* <DNT>**Memory cost**</DNT>: Le coût associé à l&apos;utilisation de la mémoire
* <DNT>**Memory usage**</DNT>: Les ressources de mémoire par pourcentage utilisé par chaque cluster
* <DNT>**Unused Cost**</DNT>: Le coût des ressources allouées mais inutilisées
* <DNT>**Unallocated cost**</DNT>: Le coût des ressources du cluster non allouées à une source
* <DNT>**Total cost**</DNT>: Le coût total généré par chaque cluster, intégrant l&apos;ensemble de l&apos;utilisation des ressources (triable)

## Comprendre les coûts inutilisés et non alloués

Ces dimensions de coût vous aident à mesurer l&apos;efficacité du cluster :

### Coût inutilisé

Le coût inutilisé représente les ressources qui ont été allouées mais qui ne reçoivent pas de requests.

**Exemple**: Si votre coût total est de 5,60 $ alloué à un déploiement de cluster, mais que vous n&apos;utilisez que 4,64 $, alors 0,96 $ est un coût inutilisé.

### Coût non alloué

Le coût non alloué représente les ressources du cluster qui n&apos;ont été allouées à aucune source.

Ces métriques vous aident à identifier des opportunités pour améliorer l&apos;efficacité du cluster et réduire le gaspillage.

<Callout variant="tip">
  Sélectionnez un cluster pour afficher ses <DNT>**Cost Trends**</DNT>, <DNT>**Cost Breakdown (Total vs Used)**</DNT> et des informations détaillées sur les coûts inutilisés et non alloués.
</Callout>

## Avantages

La fonctionnalité d&apos;Insights sur les coûts Kubernetes offre les résultats suivants :

* <DNT>**Optimized resource usage**</DNT>: la visibilité sur les clusters sous-utilisés ou excessivement coûteux permet d’ajuster efficacement l’allocation des ressources.
* <DNT>**Controlled expenses**</DNT>: Les Insights sur les facteurs de coûts spécifiques au sein de votre environnement Kubernetes permettent de prendre des décisions financières plus éclairées.
* <DNT>**Enhanced efficiency**</DNT>: Les stratégies de gestion des ressources alignées sur les objectifs de l&apos;entreprise garantissent une utilisation efficace des ressources cloud

La page <DNT>**Kubernetes Costs**</DNT> fournit les informations détaillées nécessaires pour gérer de manière proactive les coûts Kubernetes, en soutenant la prise de décision stratégique et l&apos;optimisation financière.
