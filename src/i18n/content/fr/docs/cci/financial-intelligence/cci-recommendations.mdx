Cloud Cost Intelligence fournit des recommandations d&apos;optimisation des coûts provenant de deux sources pour vous aider à identifier et à agir sur les opportunités de votre infrastructure AWS. Il analyse vos modèles d&apos;utilisation des ressources à l&apos;aide d&apos;AWS Trusted Advisor et de notre moteur Native Intelligence propriétaire.

<Callout variant="important">
  Les recommandations d&apos;optimisation des coûts sont actuellement disponibles pour AWS.
</Callout>

Utilisez les recommandations pour :

* Identifier les ressources surdimensionnées et sous-utilisées et les opportunités d&apos;engagement
* Affichez les économies potentielles avec des projections mensuelles et annuelles
* Track and prioritize optimization actions (<DNT>**acknowledge**</DNT>, <DNT>**dismiss**</DNT>, or <DNT>**mark as pending**</DNT>)

## Sources de recommandations

### AWS Trusted Advisor

AWS Trusted Advisor fournit des recommandations d&apos;optimisation des coûts directement depuis AWS.

#### Prérequis

To use <DNT>**Trusted Advisor**</DNT> recommendations with Cloud Cost Intelligence, set up and configure access to AWS Trusted Advisor cost insights in your AWS account.

#### Comment naviguer

Once configured, <DNT>**Trusted Advisor**</DNT> recommendations appear in the <DNT>**Cloud Cost Intelligence Recommendations**</DNT> page alongside other recommendations.

<Callout variant="important">
  Cloud Cost Intelligence verifies <DNT>**Trusted Advisor**</DNT> status during the initial AWS account integration. When <DNT>**Trusted Advisor**</DNT> is inactive, Cloud Cost Intelligence skips this integration component.

  If <DNT>**Trusted Advisor**</DNT> is enabled later, Cloud Cost Intelligence will automatically identify the update and synchronize data within seven days of enablement.
</Callout>

### Moteur d&apos;optimisation natif CCI

Le moteur d&apos;optimisation natif de CCI analyse l&apos;utilisation des ressources et l&apos;historique d&apos;utilisation pour identifier les opportunités de dimensionnement correct des instances et de réduction des coûts grâce aux engagements AWS Savings Plans.

#### Comment ça marche

Le moteur analyse les données d&apos;utilisation dans deux domaines :

* <DNT>**Right-sizing**</DNT>: Checks CPU and memory usage, compares current instance types to performance needs, and identifies instances that can be downsized without affecting performance.
* <DNT>**Savings plans**</DNT>: Reviews usage patterns to find consistent workloads and calculates potential savings from AWS Savings Plans commitments.

#### Quelles recommandations il fournit

Le moteur fournit des recommandations pour réduire les coûts :

* <DNT>**Instance type changes**</DNT>: Specific instance modifications (for example, <DNT>**Migrate from im4gn.16xlarge to t2.micro**</DNT>) with current configuration, CPU and memory usage, and number of affected instances
* <DNT>**Savings plan commitments**</DNT>: Recommendations to commit to consistent usage levels for discounted pricing
* <DNT>**Savings projections**</DNT>: Monthly and annual cost savings for each recommendation

## Afficher les recommandations

Pour accéder à vos recommandations d&apos;optimisation des coûts :

1. Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Cloud Cost Intelligence**</DNT>
2. Cliquez sur <DNT>**Recommendations**</DNT> dans la navigation de gauche.
