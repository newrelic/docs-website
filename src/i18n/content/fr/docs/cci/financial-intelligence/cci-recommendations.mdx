Cloud Cost Intelligence fournit des recommandations d&apos;optimisation des coûts provenant de deux sources pour vous aider à identifier et à agir sur les opportunités de votre infrastructure AWS. Il analyse vos modèles d&apos;utilisation des ressources à l&apos;aide d&apos;AWS Trusted Advisor et de notre moteur Native Intelligence propriétaire.

<Callout variant="important">
  Les recommandations d&apos;optimisation des coûts sont actuellement disponibles pour AWS.
</Callout>

Utilisez les recommandations pour :

* Identifier les ressources surdimensionnées et sous-utilisées et les opportunités d&apos;engagement
* Affichez les économies potentielles avec des projections mensuelles et annuelles
* Suivre et hiérarchiser les actions d&apos;optimisation (accuser réception, rejeter ou marquer comme en attente)

## Sources de recommandations

### AWS Trusted Advisor

AWS Trusted Advisor fournit des recommandations d&apos;optimisation des coûts directement depuis AWS.

#### Prérequis

Pour utiliser les recommandations de **Trusted Advisor** avec Cloud Cost Intelligence, configurez l&apos;accès aux informations sur les coûts d&apos;AWS Trusted Advisor dans votre compte AWS.

#### Comment naviguer

Une fois configurées, les recommandations de **Trusted Advisor** apparaissent sur la page **Cloud Cost Intelligence Recommendations**, aux côtés d&apos;autres recommandations.

<Callout variant="important">
  Cloud Cost Intelligence vérifie l&apos;état de **Trusted Advisor** lors de l&apos;intégration initiale du compte AWS. Lorsque **Trusted Advisor** est inactif, Cloud Cost Intelligence ignore ce composant d&apos;intégration.

  Si **Trusted Advisor** est activé ultérieurement, Cloud Cost Intelligence identifiera automatiquement la mise à jour et synchronisera les données dans les sept jours suivant l&apos;activation.
</Callout>

### Moteur d&apos;optimisation natif CCI

Le moteur d&apos;optimisation natif de CCI analyse l&apos;utilisation des ressources et l&apos;historique d&apos;utilisation pour identifier les opportunités de dimensionnement correct des instances et de réduction des coûts grâce aux engagements AWS Savings Plans.

#### Comment ça marche

Le moteur analyse les données d&apos;utilisation dans deux domaines :

* **Dimensionnement correct**: vérifie l&apos;utilisation du processeur et de la mémoire, compare les types d&apos;instances actuels aux besoins de performance et identifie les instances qui peuvent être réduites sans affecter les performances.
* **Plans d&apos;économies**: examine les modèles d&apos;utilisation pour trouver des charges de travail cohérentes et calcule les économies potentielles des engagements des plans d&apos;économies AWS.

#### Quelles recommandations il fournit

Le moteur fournit des recommandations pour réduire les coûts :

* **Modifications du type d’instance**: Modifications spécifiques de l’instance (par exemple, migrer de im4gn.16xlarge vers t2.micro) avec la configuration actuelle, l’utilisation du CPU et de la mémoire, et le nombre d’instances affectées.
* **Engagements du plan d’économies**: Recommandations pour s’engager à des niveaux d’utilisation constants pour une tarification réduite
* **Projections d’économies**: Économies mensuelles et annuelles pour chaque recommandation

## Afficher les recommandations

Pour accéder à vos recommandations d&apos;optimisation des coûts :

1. Accédez à **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Cloud Cost Intelligence**
2. Cliquez sur **Recommandations** dans le menu de navigation de gauche.
