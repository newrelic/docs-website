<Callout title="Aperçu">
  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Cloud Cost Intelligence prend en charge le balisage personnalisé en plus des balises par défaut dans les rapports de coûts et d&apos;utilisation (CUR) pour l&apos;allocation des coûts des ressources. Filtrez et regroupez les balises pour une analyse détaillée des dépenses. Dans les environnements Kubernetes, appliquez des étiquettes aux clusters et aux ressources. Les menus déroulants de sélection des balises proviennent des données CUR et de l&apos;événement `K8SContainerSample`.

## Configurer les paramètres

<Steps>
  <Step>
    ### Accédez aux paramètres généraux

    Allez à onglet : <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT> &gt; **General**.
  </Step>

  <Step>
    ### Configurer la balise

    Sélectionnez les balises de ressources et les étiquettes Kubernetes à partir des fichiers de coût et d&apos;utilisation :

    1. Dans la liste **Choose Resource tags for cost allocation**, sélectionnez les balises de ressources pour catégoriser les coûts cloud par service.
    2. Dans la liste **Choose Kubernetes tags for cost mapping**, sélectionnez les étiquettes Kubernetes pour catégoriser les coûts.
  </Step>

  <Step>
    ### Gérer l&apos;accès aux données télémétriques

    1. Sélectionnez l&apos;onglet **Telemetry Data**.
    2. Dans la liste **Select accounts for telemetry data**, choisissez les comptes pour les données de télémétrie.

    <Callout variant="important">
      L&apos;estimation des coûts en temps réel et l&apos;allocation K8s sont prises en charge via les données de télémétrie. Réutilise le pipeline de télémétrie existant pour les métriques d&apos;utilisation. Aucune nouvelle donnée ingérée, à l&apos;exception de l&apos;ingestion de fichiers CUR.
    </Callout>
  </Step>
</Steps>
