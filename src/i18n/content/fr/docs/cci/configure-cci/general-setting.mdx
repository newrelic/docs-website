Intelligence Coûts du cloud prend en charge les balises personnalisées en plus des balises par défaut, intégrées aux rapports de coûts et d&apos;utilisation (CUR) pour l&apos;allocation des coûts des ressources. Cela vous permet de filtrer et de regrouper les balises pour une analyse détaillée des dépenses. Dans les environnements Kubernetes, les étiquettes peuvent être appliquées au cluster et aux ressources. Les listes déroulantes de sélection tag proviennent des données CUR et `K8SContainerSample` événement, simplifiant la gestion et l&apos;analyse.

## Configurer les paramètres généraux

<Steps>
  <Step>
    ## Accédez aux paramètres généraux

    1. Accédez à <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT>.
    2. Dans le menu de navigation de gauche, sélectionnez l’onglet **General**.
  </Step>

  <Step>
    ## Configurer la balise

    Sélectionnez la balise de ressources souhaitée et les étiquettes Kubernetes à partir des valeurs provenant des fichiers de coûts et d&apos;utilisation pour rationaliser la gestion et l&apos;analyse

    1. Pour catégoriser et décomposer les coûts cloud entre les services, dans la liste **Choose Resource tags for cost allocation**, sélectionnez la balise de ressource.
    2. Pour catégoriser et décomposer les coûts cloud entre les services, dans la liste **Choose Kubernetes tags for cost mapping**, sélectionnez les étiquettes Kubernetes.
  </Step>

  <Step>
    ## Gérer l&apos;accès aux données télémétriques

    1. Pour demander des informations sur les comptes à partir desquels vous souhaitez extraire des données télémétriques pour l&apos;analyse des coûts, sélectionnez l&apos;onglet **Telemetry Data**.
    2. Pour afficher et gérer les données télémétriques pour votre analyse de coûts, dans la liste **Select accounts for telemetry data**, choisissez un ou plusieurs comptes.

      <Callout variant="important">
        * L&apos;estimation des coûts en temps réel s&apos;appuie sur des données télémétriques.
        * L&apos;allocation K8s est prise en charge via les données télémétriques.
        * Nous réutilisons pipeline de télémétrie existant pour obtenir des métriques d&apos;utilisation, et aucune nouvelle donnée n&apos;est ingérée à l&apos;exception de l&apos;ingestion du fichier CUR.
      </Callout>
  </Step>
</Steps>
