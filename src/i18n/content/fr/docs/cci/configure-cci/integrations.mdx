Monitorez la santé des intégrations et gérez les connexions aux fournisseurs de cloud sur AWS, Azure et GCP avec des actions basées sur l&apos;état.

<Callout variant="important">
  L&apos;onglet Intégrations devient accessible après avoir terminé avec succès au moins une intégration de fournisseur de cloud.
</Callout>

## Accéder aux intégrations

1. Allez dans <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> &gt; sélectionnez <DNT>**All Capabilities**</DNT> &gt; <DNT>**Cloud Cost Intelligence**</DNT>.
2. Sélectionnez l&apos;onglet <DNT>**Integrations**</DNT> pour afficher toutes les intégrations de coûts cloud.
3. Sélectionnez <DNT>**+ Add new integrations**</DNT> pour ajouter une nouvelle intégration pour un compte de fournisseur cloud.
4. Vérifiez la colonne <DNT>**Status**</DNT> pour évaluer la santé de l&apos;intégration sur vos fournisseurs de cloud.

## Comprendre le statut de l&apos;intégration

Chaque intégration affiche l&apos;un des indicateurs d&apos;état suivants :

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        Statut
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**PENDING**</DNT>
      </td>

      <td>
        L&apos;intégration a été créée et est en attente de validation par CCI. Il s&apos;agit du statut initial attribué lorsqu&apos;une nouvelle intégration est configurée pour la première fois.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**IN PROGRESS**</DNT>
      </td>

      <td>
        L&apos;établissement de la connexion est en cours ; la configuration initiale et l&apos;authentification sont en cours d&apos;établissement.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**VALIDATED**</DNT>
      </td>

      <td>
        La connexion a été authentifiée avec succès et toutes les vérifications des prérequis ont réussi ; en attente de la publication du premier fichier de coûts dans le compte de stockage.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**RUNNING**</DNT>
      </td>

      <td>
        L&apos;intégration est pleinement opérationnelle et collecte activement des données de coût.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**FAILED**</DNT>
      </td>

      <td>
        La connexion a rencontré une erreur lors de la configuration initiale ou de la validation du rapport CUR et requiert votre attention. Cliquez sur le nom de la connexion pour afficher les détails de l&apos;erreur et les étapes de résolution.

        <Callout variant="important">
          <DNT>**FAILED**</DNT> s&apos;applique uniquement lors de la phase de configuration initiale de l&apos;intégration, avant que la connexion n&apos;atteigne le statut <DNT>**RUNNING**</DNT>.
        </Callout>
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**ERROR**</DNT>
      </td>

      <td>
        Une connexion précédemment <DNT>**RUNNING**</DNT> a rencontré des problèmes d&apos;intégration. Contrairement à <DNT>**FAILED**</DNT>, qui se produit lors de la configuration initiale, <DNT>**ERROR**</DNT> indique un problème avec une connexion déjà active. Cliquez sur le nom de la connexion pour afficher les détails de l&apos;erreur et les étapes de résolution.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**PAUSED**</DNT>
      </td>

      <td>
        La collecte de données a été temporairement suspendue. Vous pouvez mettre en pause une connexion après qu&apos;elle a atteint le statut <DNT>**RUNNING**</DNT> et la reprendre plus tard.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Monitorez régulièrement la colonne <DNT>**Status**</DNT> pour garantir que les intégrations restent <DNT>**RUNNING**</DNT>. Traitez rapidement les intégrations <DNT>**FAILED**</DNT> ou <DNT>**ERROR**</DNT> pour éviter les lacunes dans la collecte des données de coût, et examinez les exigences de validation durant la phase <DNT>**IN PROGRESS**</DNT> pour accélérer l&apos;activation.
</Callout>

## Gérer les intégrations

Cliquez sur l&apos;icône de points de suspension à côté d&apos;une intégration pour accéder aux actions disponibles. Les actions affichées dépendent du statut actuel de l&apos;intégration :

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        Action
      </th>

      <th style={{ width: "150px" }}>
        Disponible pour
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Resume**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> intégrations (uniquement lorsque l&apos;intégration a été créée avec succès mais que la validation du rapport CUR a échoué) et <DNT>**PAUSED**</DNT> intégrations
      </td>

      <td>
        Reprend la collecte de données ou réessaie le processus d&apos;établissement de la connexion. Pour les intégrations <DNT>**FAILED**</DNT> pour lesquelles l&apos;intégration elle-même n&apos;a pas été créée en raison d&apos;une erreur de validation, l&apos;option de reprise n&apos;est pas disponible. Pour <DNT>**PAUSED**</DNT> intégrations, la reprise restaure la collecte active de données.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> intégrations
      </td>

      <td>
        Suspend temporairement la collecte de données tout en conservant la configuration.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> intégrations (avant d&apos;atteindre le statut <DNT>**RUNNING**</DNT>)
      </td>

      <td>
        Supprime définitivement l&apos;intégration et permet de configurer une nouvelle connexion.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Utilisez <DNT>**Pause**</DNT> plutôt que <DNT>**Delete**</DNT> pour les suspensions temporaires afin de préserver votre configuration d&apos;intégration. Utilisez <DNT>**Resume**</DNT> pour restaurer une intégration <DNT>**PAUSED**</DNT> ou réessayer une intégration <DNT>**FAILED**</DNT> pour laquelle la connexion a été créée mais la validation du rapport CUR a échoué.
</Callout>
