Monitorez la santé des intégrations et gérez les connexions aux fournisseurs de cloud sur AWS, Azure et GCP avec des actions basées sur l&apos;état.

<Callout variant="important">
  L&apos;onglet Intégrations devient accessible après avoir terminé avec succès au moins une intégration de fournisseur de cloud.
</Callout>

## Accéder aux intégrations

1. Allez dans <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> &gt; sélectionnez <DNT>**All Capabilities**</DNT> &gt; <DNT>**Cloud Cost Intelligence**</DNT>.
2. Select the <DNT>**Integrations**</DNT> tab to view all cloud cost integrations.
3. Select the <DNT>**+ Add new integrations**</DNT> to add a new integration for a cloud provider account.
4. Check the <DNT>**Status**</DNT> column to assess integration health across your cloud providers.

## Comprendre le statut de l&apos;intégration

Chaque intégration affiche l&apos;un des indicateurs d&apos;état suivants :

  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        Statut
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**PENDING**</DNT>
      </td>

      <td>
        Integration has been created and is awaiting validation by CCI. This is the initial status assigned when a new integration is first set up.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**IN PROGRESS**</DNT>
      </td>

      <td>
        L&apos;établissement de la connexion est en cours ; la configuration initiale et l&apos;authentification sont en cours d&apos;établissement.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**VALIDATED**</DNT>
      </td>

      <td>
        La connexion a été authentifiée avec succès et toutes les vérifications des prérequis ont réussi ; en attente de la publication du premier fichier de coûts dans le compte de stockage.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**RUNNING**</DNT>
      </td>

      <td>
        L&apos;intégration est pleinement opérationnelle et collecte activement des données de coût.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**FAILED**</DNT>
      </td>

      <td>
        Connection encountered an error during initial setup or CUR report validation and requires attention. Click the connection name to view error details and resolution steps.

        <Callout variant="important">
          <DNT>**FAILED**</DNT> applies only during the initial integration setup phase, before the connection reaches <DNT>**RUNNING**</DNT> status.
        </Callout>
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**ERROR**</DNT>
      </td>

      <td>
        A previously <DNT>**RUNNING**</DNT> connection has encountered integration issues. Unlike <DNT>**FAILED**</DNT>, which occurs during initial setup, <DNT>**ERROR**</DNT> indicates a problem with an already-active connection. Click the connection name to view error details and resolution steps.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**PAUSED**</DNT>
      </td>

      <td>
        Data collection has been temporarily suspended. You can pause a connection after it reaches <DNT>**RUNNING**</DNT> status and resume it later.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Monitor the <DNT>**Status**</DNT> column regularly to ensure integrations remain <DNT>**RUNNING**</DNT>. Address <DNT>**FAILED**</DNT> or <DNT>**ERROR**</DNT> integrations promptly to avoid gaps in cost data collection, and review validation requirements during the <DNT>**IN PROGRESS**</DNT> phase to expedite activation.
</Callout>

## Gérer les intégrations

Cliquez sur l&apos;icône de points de suspension à côté d&apos;une intégration pour accéder aux actions disponibles. Les actions affichées dépendent du statut actuel de l&apos;intégration :

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        Action
      </th>

      <th style={{ width: "150px" }}>
        Disponible pour
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Resume**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> integrations (only where the integration was created successfully but CUR report validation failed) and <DNT>**PAUSED**</DNT> integrations
      </td>

      <td>
        Resumes data collection or reattempts the connection setup process. For <DNT>**FAILED**</DNT> integrations where the integration itself was not created due to a validation error, the resume option is not available. For <DNT>**PAUSED**</DNT> integrations, resume restores active data collection.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> intégrations
      </td>

      <td>
        Suspend temporairement la collecte de données tout en conservant la configuration.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> integrations (prior to reaching <DNT>**RUNNING**</DNT> status)
      </td>

      <td>
        Supprime définitivement l&apos;intégration et permet de configurer une nouvelle connexion.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Use <DNT>**Pause**</DNT> rather than <DNT>**Delete**</DNT> for temporary suspensions to preserve your integration configuration. Use <DNT>**Resume**</DNT> to restore a <DNT>**PAUSED**</DNT> integration or retry a <DNT>**FAILED**</DNT> integration where the connection was created but CUR report validation failed.
</Callout>
