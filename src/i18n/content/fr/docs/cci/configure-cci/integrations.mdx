<Callout title="Aperçu">
  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

Monitorez la santé des intégrations et gérez les connexions aux fournisseurs de cloud sur AWS, Azure et GCP avec des actions basées sur l&apos;état.

<Callout variant="important">
  L&apos;onglet Intégrations devient accessible après avoir terminé avec succès au moins une intégration de fournisseur de cloud.
</Callout>

## Accéder aux intégrations

1. Allez dans <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> &gt; sélectionnez <DNT>**All Capabilities**</DNT> &gt; <DNT>**Cloud Cost Intelligence**</DNT>.
2. Sélectionnez l&apos;onglet **Integrations** pour afficher toutes les intégrations de coûts cloud.
3. Sélectionnez **+ Add new integrations** pour ajouter une nouvelle intégration pour un compte de fournisseur cloud.
4. Vérifiez la colonne **Status** pour évaluer la santé de l&apos;intégration sur vos fournisseurs de cloud.

## Comprendre le statut de l&apos;intégration

Chaque intégration affiche l&apos;un des indicateurs d&apos;état suivants :

<table>
  <thead>
    <tr>
      <th style={{ width: "150px" }}>
        Statut
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**In Progress**</DNT>
      </td>

      <td>
        L&apos;établissement de la connexion est en cours ; la configuration initiale et l&apos;authentification sont en cours d&apos;établissement.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Validated**</DNT>
      </td>

      <td>
        La connexion a été authentifiée avec succès et toutes les vérifications des prérequis ont réussi ; en attente de la publication du premier fichier de coûts dans le compte de stockage.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**RUNNING**</DNT>
      </td>

      <td>
        L&apos;intégration est pleinement opérationnelle et collecte activement des données de coût.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Failed**</DNT>
      </td>

      <td>
        La connexion a rencontré une erreur et requiert votre attention. Cliquez sur le nom de la connexion pour afficher les détails de l&apos;erreur et les étapes de résolution.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Paused**</DNT>
      </td>

      <td>
        La collecte de données a été temporairement suspendue. Vous pouvez mettre en pause une connexion après qu&apos;elle a atteint le statut <DNT>**RUNNING**</DNT> et la reprendre plus tard.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Monitorez régulièrement la colonne <DNT>**Status**</DNT> pour garantir que les intégrations restent <DNT>**RUNNING**</DNT>. Traitez rapidement les intégrations en échec pour éviter les lacunes dans la collecte des données de coût, et examinez les exigences de validation durant la phase <DNT>**In Progress**</DNT> pour accélérer l&apos;activation.
</Callout>

## Gérer les intégrations

Cliquez sur l&apos;icône de points de suspension à côté d&apos;une intégration pour accéder aux actions disponibles. Les actions affichées dépendent du statut actuel de l&apos;intégration :

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        Action
      </th>

      <th style={{ width: "150px" }}>
        Disponible pour
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Retry**</DNT>
      </td>

      <td>
        <DNT>**Failed**</DNT> intégrations
      </td>

      <td>
        Réessaie le processus d&apos;établissement de la connexion.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> intégrations
      </td>

      <td>
        Suspend temporairement la collecte de données tout en conservant la configuration.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Activate**</DNT>
      </td>

      <td>
        <DNT>**Paused**</DNT> intégrations
      </td>

      <td>
        Reprend la collecte de données.
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**Failed**</DNT> intégrations (avant activation)
      </td>

      <td>
        Supprime définitivement l&apos;intégration et permet de configurer une nouvelle connexion.
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Utilisez <DNT>**Pause**</DNT> plutôt que <DNT>**Delete**</DNT> pour les suspensions temporaires afin de préserver votre configuration d&apos;intégration.
</Callout>
