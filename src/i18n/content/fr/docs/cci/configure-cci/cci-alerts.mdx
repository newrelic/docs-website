Après avoir configuré Intelligence Coûts du cloud, créez des règles d’alerte avec des seuils pour recevoir des notifications proactives avant de dépasser les limites financières et éviter les frais inattendus. Dans un budget, vous pouvez définir plusieurs seuils d&apos;alerte en fonction du pourcentage d&apos;utilisation du budget.

## Créer une nouvelle condition d&apos;alerte

Une condition d&apos;alerte est une requête exécutée en continu qui mesure un ensemble donné d&apos;événements par rapport à un seuil défini et ouvre un incident lorsque le seuil est atteint pour une durée spécifiée.

1. Allez dans <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT> &gt; <DNT>**Alerts**</DNT> &gt; <DNT>**Alert Policies**</DNT>.
2. Sur la page de la liste des politiques, cliquez sur <DNT>**+ New alert condition**</DNT>.
3. Pour créer des alertes à partir de zéro, utilisez <DNT>**Write your own query**</DNT>.

## Définissez le comportement de votre signal

Vous pouvez utiliser une requête NRQL pour définir les signaux que vous souhaitez qu&apos;une condition d’alerte utilise comme base pour votre alerte. Pour cet exemple, vous utiliserez cette requête :

```sql
FROM CloudCost SELECT sum(line_item_unblended_cost) FACET product_region_code
```

L&apos;utilisation de cette requête pour votre condition d’alerte indique à New Relic que vous souhaitez connaître le coût non lissé ventilé par code de région du produit.

Pour en savoir plus sur l&apos;utilisation de NRQL (le langage de requête de New Relic), consultez notre [documentation NRQL](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions/).

## Exécuter et prévisualiser votre signal [#advanced-signal-settings]

1. Après avoir défini votre signal, cliquez sur <DNT>**Run**</DNT>. Un graphique apparaîtra et affichera le paramètre que vous avez défini.

   <Callout variant="tip">
     Pour configurer des alertes inter-comptes, sélectionnez un <DNT>**data account**</DNT> dans la liste déroulante. Notez que vous ne pouvez interroger les données que d’un seul compte à la fois pour les alertes inter-comptes.
   </Callout>

   <img title="Create a new alert condition page with advanced signal settings highlighted" alt="Create a new alert condition page with advanced signal settings highlighted" src="/images/cci-create-new-alert.webp" />

   Pour cet exemple, le graphique affichera le coût d&apos;utilisation estimé pour les ressources cloud de votre entreprise, ventilé par code de région du produit. Cela vous permet de monitorer le coût de vos ressources cloud dans différentes régions.

2. Cliquez sur <DNT>**Next**</DNT> et commencez à configurer votre condition d’alerte.

## Fixer le seuil de condition d&apos;alerte [#thresholds]

Vous disposez désormais d&apos;une condition entièrement définie et de règles pour ouvrir un incident si les seuils sont dépassés. Nommez la condition et associez-la à une stratégie pour terminer la configuration.

## Ajouter des détails sur condition d&apos;alerte [#add-details]

Votre condition d’alerte est entièrement définie et créera un incident lorsque vos seuils seront dépassés. Nommez cette condition et associez-la à une stratégie pour terminer la configuration.

Une stratégie est le système de tri pour vos incidents. Vous pouvez connecter des politiques à des **[workflows](/docs/alerts-applied-intelligence/applied-intelligence/incident-workflows/incident-workflows/)** pour définir où vous souhaitez que New Relic envoie ces informations et à quelle fréquence.
