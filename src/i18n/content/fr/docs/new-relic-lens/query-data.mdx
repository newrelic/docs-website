Vous pouvez écrire des requêtes <DNT>Lens</DNT> dans **Query Builder**, <DNT>**Notebooks**</DNT> et des **dashboards personnalisés**. Cette page explique comment interroger les données dans chaque plateforme.

<Callout variant="tip">
  Lors de l&apos;interrogation de sources de données, <DNT>Lens</DNT> ne limite pas le nombre de lignes que vous pouvez récupérer depuis des sources externes. Cependant, les requêtes vers <DNT>NRDB</DNT> via <DNT>Lens</DNT> conservent la même limite de 5 000 lignes que <DNT>NRQL</DNT>.
</Callout>

## Prérequis [#prerequisites]

Avant de commencer :

* Vérifiez que vous disposez des [permissions](/docs/new-relic-lens/overview#permissions) nécessaires pour interroger les sources de données.
* Si vous devez interroger des sources de données externes, assurez-vous que les [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors) sont configurés.

## Requête dans le constructeur de requêtes [#query-builder]

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
2. Depuis le tiroir au bas de la page, ouvrez la console <DNT>**Query your data**</DNT>.
3. Dans l&apos;éditeur de requêtes, sélectionnez <DNT>**Lens data (SQL)**</DNT> comme langage de requête.
4. Si vous souhaitez interroger des données NRDB, sélectionnez le compte dans le sélecteur de compte. D&apos;autres <DNT>data connectors</DNT> sont disponibles au niveau de l&apos;organisation et ne nécessitent pas de sélection de compte.
5. Écrivez votre requête SQL en utilisant le [format de référence de table](#table-format): `connection_name.schema.table`
6. Cliquez sur <DNT>**Run**</DNT>. Le graphique s&apos;affiche en fonction des résultats de votre requête.

## Enregistrer la requête dans <DNT>Notebooks</DNT> [#notebooks]

Les [Notebooks](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/) sont des documents partageables qui combinent des requêtes, des visualisations et de la documentation markdown.

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>.
2. Depuis le tiroir au bas de la page, ouvrez la console <DNT>**Query your data**</DNT>.
3. Activez le commutateur à côté de <DNT>**Notebooks**</DNT>. Au bas de la page, le bouton <DNT>**+ SQL**</DNT> apparaît.
4. Cliquez sur <DNT>**+ SQL**</DNT> pour ajouter un bloc SQL.
5. Si vous souhaitez interroger des données NRDB, sélectionnez le compte dans le sélecteur de compte. D&apos;autres <DNT>data connectors</DNT> sont disponibles au niveau de l&apos;organisation et ne nécessitent pas de sélection de compte.
6. Écrivez votre requête SQL en utilisant le [format de référence de table](#table-format): `connection_name.schema.table`
7. Cliquez sur <DNT>**Run**</DNT>. Le graphique s&apos;affiche en fonction des résultats de votre requête.
8. Cliquez sur <DNT>**Save**</DNT>.

Pour plus d&apos;informations, consultez la [documentation sur les notebooks](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/).

## Requête dans les dashboards personnalisés [#dashboards]

Vous pouvez ajouter des graphiques basés sur SQL à vos [dashboards personnalisés](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards):

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**</DNT>.
2. [Créer un nouveau dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create-new) ou en ouvrir un existant.
3. Cliquez sur <DNT>**+ Add widget**</DNT> et sélectionnez un type de graphique.
4. Dans l&apos;éditeur de requêtes, sélectionnez <DNT>**Lens data (SQL)**</DNT> comme langage de requête.
5. Si vous souhaitez interroger des données NRDB, sélectionnez le compte dans le sélecteur de compte. D&apos;autres <DNT>data connectors</DNT> sont disponibles au niveau de l&apos;organisation et ne nécessitent pas de sélection de compte.
6. Écrivez votre requête SQL en utilisant le [format de référence de table](#table-format): `connection_name.schema.table`
7. Cliquez sur <DNT>**Run**</DNT>. Le graphique s&apos;affiche en fonction des résultats de votre requête.
8. Cliquez sur <DNT>**Add to dashboard**</DNT>. Pour plus d&apos;informations, consultez la [documentation sur les dashboards personnalisés](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/).

<Callout variant="important">
  Les graphiques basés sur SQL sont uniquement pris en charge dans les dashboards privés. Les dashboards publics ne prennent pas en charge les requêtes SQL.
</Callout>

## Syntaxe SQL [#syntax]

<DNT>Lens</DNT> utilise le SQL ANSI pour interroger les sources de données, vous n&apos;avez donc pas besoin d&apos;apprendre des langages de requête spécifiques aux bases de données.

### Format de référence de table [#table-format]

Lorsque vous rédigez des requêtes dans <DNT>Lens</DNT>, référencez les tables au format :

```
connection_name.schema.table
```

Par exemple:

* `telemetry.Transaction` - Interroger la table Transaction depuis la télémétrie (NRDB) <DNT>connector</DNT>
* `snowflake.sales.orders` - Interroger la table orders depuis un <DNT>Snowflake</DNT> <DNT>connector</DNT> nommé &quot;snowflake&quot; dans le schéma &quot;sales&quot;
* `gsheets.default.employees` - Interroger un <DNT>Google Sheets</DNT> <DNT>connector</DNT> nommé &quot;gsheets&quot;

### Instructions SQL prises en charge [#supported-sql]

<DNT>Lens</DNT> prend en charge les instructions SQL ANSI standard, y compris :

* `SELECT` - Interroger des données depuis une ou plusieurs tables
* `JOIN` - Combiner les données de plusieurs tables (y compris les jointures inter-bases de données)
* `WHERE` - Filtrer les résultats
* `GROUP BY` - Agréger les données
* `ORDER BY` - Trier les résultats
* `LIMIT` - Limiter le nombre de lignes retournées
* `WITH` (CTEs) - Définir des expressions de table communes pour des requêtes complexes

### Conseils sur les performances [#performance-tips]

* **Utilisez des filtres en amont**: Appliquez des clauses `WHERE` pour réduire la quantité de données traitées.
* **Limiter les résultats**: Utilisez `LIMIT` pour restreindre les lignes renvoyées lors de l&apos;exploration des données.
* **Envisagez <DNT>materialized views</DNT>**: Pour les requêtes complexes exécutées fréquemment, créez un <DNT>materialized view</DNT>.
* **Optimisez les jointures**: lorsque vous joignez de grandes tables, filtrez les données avant la jointure si possible.

## Différences de syntaxe de requête par rapport au NRQL et exemples [#syntax-differences]

<DNT>Lens</DNT> utilise ANSI SQL au lieu de NRQL. Les exemples suivants illustrent les principales différences et les modèles de requête courants.

<CollapserGroup>
  <Collapser id="groupby" title="Utilisez `GROUP BY` au lieu de `FACET`">
    Lorsque vous regroupez des résultats, utilisez `GROUP BY` et incluez la colonne regroupée dans votre instruction `SELECT`.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    FACET enduser.id
    ```

    **SQL :**

    ```sql
    SELECT "enduser.id", count(*)
    FROM telemetry.PageViewTiming
    GROUP BY "enduser.id"
    ```
  </Collapser>

  <Collapser id="where-timestamp" title="Utilisez `WHERE` avec `timestamp` plutôt que `SINCE`">
    Filtrez les plages temporelles en utilisant `WHERE` avec la colonne `timestamp` et les intervalles de dates.

    **NRQL:**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    SINCE 1 day ago
    ```

    **SQL :**

    ```sql
    SELECT count(*)
    FROM telemetry.PageViewTiming
    WHERE timestamp > current_date - INTERVAL '1' DAY
    ```
  </Collapser>

  <Collapser id="timeseries" title="Séries temporelles au format epoch">
    Pour créer des visualisations de séries temporelles, structurez votre requête de sorte que la première colonne contienne des dates au format epoch. Utilisez `to_unixtime()` pour convertir les dates.

    ```sql
    WITH orders AS (
      SELECT
        to_unixtime(date(date_parse(order_date, '%Y-%m-%d'))) as dt,
        count(*) as orders
      FROM gsheets.default.orders
      GROUP BY 1
    )
    SELECT *
    FROM orders
    WHERE dt >= to_unixtime(date('2025-10-19'))
      AND dt <= to_unixtime(date('2025-10-21'))
    ORDER BY dt ASC
    ```
  </Collapser>

  <Collapser id="cross-db-join" title="`JOIN` inter-bases de données">
    Utilisez la syntaxe SQL standard `JOIN` pour combiner des données provenant de plusieurs sources dans une seule requête.

    ```sql
    WITH sf AS (
      SELECT DISTINCT user_id, assigned_success_manager
      FROM demogorgon.default.relibank_opportunity_matrix
    )
    SELECT
      sf.assigned_success_manager,
      MAX(nrdb.largestContentfulPaint)
    FROM telemetry.PageViewTiming nrdb
    LEFT JOIN sf ON nrdb."enduser.id" = sf.user_id
    WHERE timingName = 'largestContentfulPaint'
      AND nrdb.timestamp > current_date - INTERVAL '1' DAY
    GROUP BY sf.assigned_success_manager
    ```
  </Collapser>

  <Collapser id="query-mv" title="Interroger les vues matérialisées">
    Référencez <DNT>materialized views</DNT> à l&apos;aide du préfixe `mv.` :

    ```sql
    SELECT *
    FROM mv.weekly_performance_summary
    WHERE week_number = 42
    ```

    Vous pouvez joindre <DNT>materialized views</DNT> à d&apos;autres sources de données :

    ```sql
    SELECT
      mv.store_id,
      mv.total_sales,
      s.store_name,
      s.region
    FROM mv.store_sales_mv mv
    JOIN snowflake.retail.stores s ON mv.store_id = s.store_id
    WHERE mv.total_sales > 10000
    ```
  </Collapser>
</CollapserGroup>
