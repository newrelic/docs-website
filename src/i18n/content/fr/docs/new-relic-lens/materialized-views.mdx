Pour les requêtes plus volumineuses et coûteuses en calcul, vous pouvez stocker les résultats sous forme de <DNT>materialized views</DNT> dans <DNT>Lens</DNT>. Un <DNT>materialized view</DNT> stocke les résultats précalculés d&apos;une requête, vous permettant d&apos;interroger les données stockées au lieu d&apos;exécuter la requête complexe à chaque fois.

## Avantages de <DNT>materialized views</DNT> [#benefits]

<DNT>Materialized views</DNT> vous aider :

* **Réduire les coûts des requêtes**: évitez de récupérer de manière répétée les mêmes données depuis des sources de données externes, ce qui peut engendrer des frais basés sur les données analysées ou transférées.
* **Améliorez les performances**: interrogez des résultats précalculés au lieu d&apos;exécuter des jointures complexes sur plusieurs bases de données à chaque fois.
* **Simplifiez les dashboards**: Stockez les résultats de requêtes multi-bases de données et référencez-les simplement dans les widgets de dashboard.

## Prérequis [#prerequisites]

Avant de commencer :

* Vérifiez que vous disposez des [autorisations](/docs/new-relic-lens/overview#permissions) nécessaires pour créer et gérer <DNT>materialized views</DNT>.
* Si vous devez interroger des sources de données externes, assurez-vous que les [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors) sont configurés.

## Créer un <DNT>materialized view</DNT> [#create]

Pour créer un <DNT>materialized view</DNT>:

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT>.
2. Cliquez sur <DNT>**Create materialized view**</DNT>.
3. Saisissez un nom unique pour le <DNT>materialized view</DNT>.
4. Pour interroger les données NRDB, sélectionnez un compte dans le sélecteur de compte. Vous pouvez joindre les données NRDB de ce compte à des sources de données externes.
5. Rédigez votre requête SQL en utilisant le [format de référence de table](/docs/new-relic-lens/query-data#table-format): `connection_name.schema.table`
6. Cliquez sur <DNT>**Create**</DNT>.

Le <DNT>materialized view</DNT> apparaît dans la liste. L&apos;enregistrement des résultats peut prendre un certain temps en fonction de la complexité de votre requête.

## Actualiser un <DNT>materialized view</DNT> [#refresh]

<DNT>Materialized views</DNT> contiennent un instantané des données au moment de leur création ou de leur dernière actualisation. Pour obtenir des données récentes :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT>.
2. Trouvez le <DNT>materialized view</DNT> que vous souhaitez actualiser.
3. Cliquez sur le menu <Icon name="fe-more-horizontal" />.
4. Sélectionnez <DNT>**Refresh**</DNT>.

La vue exécute à nouveau la requête d&apos;origine et met à jour les résultats stockés avec les dernières données.

<Callout variant="important">
  Prenez en compte vos exigences en matière de fraîcheur des données lorsque vous décidez de la fréquence d&apos;actualisation de <DNT>materialized views</DNT>. L&apos;actualisation fréquente des vues avec des requêtes complexes peut augmenter les coûts.
</Callout>

## Interroger un <DNT>materialized view</DNT> [#query]

Vous pouvez interroger un <DNT>materialized view</DNT> comme n&apos;importe quelle autre table dans <DNT>Lens</DNT>:

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Query your data**</DNT>.

2. Activez le bouton à bascule à côté de <DNT>**Notebooks**</DNT> et cliquez sur <DNT>**+ SQL**</DNT> pour ajouter un bloc SQL.

3. Référencez votre <DNT>materialized view</DNT> en utilisant le préfixe `mv.` suivi du nom de la vue :

   ```sql
   SELECT * FROM mv.your_materialized_view_name
   ```

4. Exécutez votre requête.

Vous pouvez appliquer des filtres, jointures ou agrégations supplémentaires sur le <DNT>materialized view</DNT>:

```sql
SELECT store_id, transaction_count
FROM mv.top_stores_mv
WHERE transaction_count > 1000
ORDER BY transaction_count DESC
```

Le <DNT>materialized view</DNT> agit comme une table contenant les résultats précalculés, vous permettant d&apos;effectuer des analyses supplémentaires sans exécuter à nouveau la requête complexe d&apos;origine.

## Supprimer un <DNT>materialized view</DNT> [#delete]

Pour supprimer un <DNT>materialized view</DNT> dont vous n&apos;avez plus besoin :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT>.
2. Trouvez le <DNT>materialized view</DNT> que vous souhaitez supprimer.
3. Cliquez sur le menu <Icon name="fe-more-horizontal" />.
4. Sélectionnez <DNT>**Delete**</DNT>.
5. Confirmer la suppression.

<Callout variant="caution">
  La suppression d&apos;un <DNT>materialized view</DNT> est définitive. Les requêtes ou dashboards faisant référence à la vue supprimée échoueront.
</Callout>

Pour des exemples de requêtes incluant des jointures inter-bases de données, consultez [Interroger les vues matérialisées](/docs/new-relic-lens/query-data/#query-mv).
