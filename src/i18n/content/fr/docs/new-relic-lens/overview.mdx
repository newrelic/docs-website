En tant qu&apos;utilisateur New Relic, vous pouvez avoir besoin d&apos;analyser vos données de télémétrie parallèlement aux données stockées dans plusieurs bases de données, entrepôts de données et feuilles de calcul. Exporter manuellement des données de chaque source et les combiner hors ligne avec vos données d&apos;observabilité prend du temps et réduit la productivité. De plus, vous ne pouvez souvent pas consolider toutes les données dans une base de données unique en raison d&apos;exigences de conformité, de sécurité ou opérationnelles.

<DNT>New Relic Lens</DNT> répond à ces défis. <DNT>Lens</DNT> est une plateforme de requête unifiée au sein de New Relic qui vous permet de connecter et d&apos;interroger plusieurs sources de données directement depuis New Relic. Écrivez une requête dans <DNT>Lens</DNT> en utilisant le SQL ANSI et effectuez des jointures inter-bases de données pour analyser et corréler les données provenant de diverses sources. <DNT>Lens</DNT> pousse le travail de calcul vers chaque source de données, qui traite sa portion de la requête et renvoie uniquement les résultats. <DNT>Lens</DNT> combine ensuite ces résultats, minimisant le transfert de données et assurant la sécurité en n&apos;ingérant jamais de données brutes provenant de vos sources externes.

## Comment ça marche [#how-it-works]

L&apos;utilisation de <DNT>Lens</DNT> implique les étapes suivantes :

<Steps>
  <Step>
    ### Activer Lens

    <DNT>Lens</DNT> est disponible en version préliminaire publique. Vérifiez auprès de votre <DNT>Organization Manager</DNT> pour vous inscrire depuis la page <DNT>**[Previews &amp; Trials](https://one.newrelic.com/admin-portal/promotion-management/home)**</DNT>.
  </Step>

  <Step>
    ### Vérifier les autorisations

    Si vous êtes un <DNT>Organization Manager</DNT>, vous disposez des permissions nécessaires pour accéder à toutes les fonctionnalités <DNT>Lens</DNT> par défaut. Si vous n&apos;êtes pas un <DNT>Organization Manager</DNT>, contactez votre <DNT>Authentication Domain Manager</DNT> pour [créer un rôle personnalisé avec les permissions requises](/docs/new-relic-lens/overview/#permissions).
  </Step>

  <Step>
    ### Installation <DNT>connectors</DNT>

    <DNT>Lens</DNT> inclut <DNT>system connectors</DNT> intégré pour les données NRDB. Avec les autorisations nécessaires, vous pouvez configurer [<DNT>data connectors</DNT>](/docs/new-relic-lens/set-up-connectors) pour se connecter à des sources de données externes :

    * **Entrepôts de données cloud**: <DNT>Snowflake</DNT>, <DNT>Redshift</DNT>, <DNT>ClickHouse</DNT>
    * **Bases de données relationnelles**: <DNT>PostgreSQL</DNT>, <DNT>MySQL</DNT>, <DNT>SQL Server</DNT>
    * **Bases de données documentaires**: <DNT>MongoDB</DNT>, <DNT>Elasticsearch</DNT>
    * **Feuilles de calcul**: <DNT>Google Sheets</DNT>
    * **Lacs de données**: <DNT>Iceberg</DNT>
    * **Métriques et monitoring**: <DNT>Prometheus</DNT>, <DNT>AWS CloudWatch</DNT>
  </Step>

  <Step>
    ### Interroger les données

    Écrivez des requêtes SQL dans le [Query Builder, les notebooks ou les dashboards personnalisés](/docs/new-relic-lens/query-data) en utilisant le format `connection_name.schema.table` pour référencer des tables et joindre des données provenant de différentes sources.
  </Step>

  <Step>
    ### Créer <DNT>materialized views</DNT> (facultatif)

    Pour les requêtes complexes qui s&apos;exécutent fréquemment, vous pouvez enregistrer les résultats en tant que [<DNT>materialized views</DNT>](/docs/new-relic-lens/materialized-views) pour réduire les coûts et améliorer les performances.
  </Step>
</Steps>

## Cas d&apos;utilisation [#use-cases]

Lens permet différents scénarios d&apos;analyse de données :

* **Analyse budgétaire**: comparez les données d&apos;utilisation New Relic de <DNT>NRDB</DNT> avec les données budgétaires stockées dans <DNT>Snowflake</DNT> pour suivre les dépenses par rapport aux budgets alloués.
* **Optimisation du workflow**: Remplacez plusieurs étapes manuelles d&apos;export/import (d&apos;une base de données à une autre) par une seule requête SQL qui joint toutes les données nécessaires, réduisant les coûts et améliorant l&apos;efficacité.
* **Agréger les données sur plusieurs régions**: Suivez le trafic utilisateur dans différentes régions géographiques en regroupant les données provenant de différentes sources de données.
* **Optimisation des performances avec <DNT>materialized views</DNT>**: Un dashboard de direction combine les données de plus de 5 bases de données et s&apos;actualise chaque semaine. En stockant les résultats de la requête `JOIN` multi-bases de données dans un <DNT>materialized view</DNT>, vous évitez d&apos;exécuter la requête coûteuse en calcul chaque fois que quelqu&apos;un charge le dashboard. Cela réduit les coûts des requêtes et améliore considérablement la vitesse de chargement des pages.

## Où vous pouvez utiliser Lens [#where-to-use]

Vous pouvez utiliser Lens dans les zones suivantes de la plateforme New Relic :

* **Générateur de requêtes**: Écrivez et exécutez des requêtes SQL directement dans le générateur de requêtes.
* <DNT>**Notebooks**</DNT>: Créez des blocs SQL dans des notebooks pour combiner des requêtes avec des visualisations et de la documentation Markdown.
* **Dashboards clients**: Ajoutez des graphiques basés sur SQL à vos dashboards privés.

<Callout variant="important">
  Les graphiques basés sur SQL ne sont pas pris en charge dans les dashboards publics.
</Callout>

## Rôles et permissions des utilisateurs [#permissions]

<DNT>Connector</DNT> et les accès <DNT>materialized view</DNT> sont contrôlés par des permissions RBAC spécifiques. Les utilisateurs disposant du rôle <DNT>**Organization Manager**</DNT> ont par défaut toutes les permissions RBAC liées à <DNT>Lens</DNT>.

Pour les autres utilisateurs, le <DNT>**Authentication Domain Manager**</DNT> doit [créer un rôle personnalisé au niveau de l&apos;organisation via NerdGraph](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#organization-scoped-role) avec les permissions <DNT>Lens</DNT> requises. Lors de la création du rôle, ajoutez les ID de permission des tableaux ci-dessous au tableau `permissionIds`.
