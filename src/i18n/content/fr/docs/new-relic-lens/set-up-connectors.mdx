## <DNT>Data connectors</DNT> [#data-connectors]

<DNT>Data connectors</DNT> connecter <DNT>Lens</DNT> à des sources de données externes telles que des bases de données, des entrepôts de données et des feuilles de calcul. Pour créer, modifier ou supprimer <DNT>connectors</DNT>, vous devez disposer de [permissions RBAC spécifiques](/docs/new-relic-lens/overview#permissions).

### Contrôle d&apos;accès [#access-control]

<DNT>Lens</DNT> fournit deux façons de contrôler qui peut interroger <DNT>connectors</DNT>:

* **Permissions RBAC**: Les utilisateurs disposant de permissions RBAC <DNT>**Lens connectors**</DNT> peuvent interroger tous les <DNT>connectors</DNT> de l&apos;organisation.
* **Contrôle d&apos;accès à granularité fine**: les utilisateurs sans permissions RBAC étendues peuvent toujours interroger des <DNT>connectors</DNT> spécifiques si un accès à granularité fine leur est accordé pour ces <DNT>connectors</DNT>.

### Créer un <DNT>connector</DNT> [#create-connector]

Pour configurer un <DNT>data connector</DNT>:

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Connectors**</DNT>.

2. Cliquez sur <DNT>**Create a connector**</DNT>.

3. Sélectionnez votre type de <DNT>connector</DNT> dans le menu déroulant.

4. Saisissez les informations de connexion requises pour le type de <DNT>connector</DNT> sélectionné.

5. Pour accorder l&apos;accès à ce connecteur aux utilisateurs ou groupes ne disposant pas de permissions RBAC, configurez le contrôle d&apos;accès granulaire depuis la section <DNT>**Access control**</DNT> :

   1. Sélectionnez le <DNT>**Auth domain**</DNT> contenant les utilisateurs ou les groupes.
   2. Sélectionnez le <DNT>**User or group**</DNT>.
   3. Sélectionnez le rôle<DNT>**Lens Viewer**</DNT>.
   4. Pour ajouter d&apos;autres utilisateurs ou groupes, cliquez sur <DNT>**Add**</DNT> et répétez les étapes ci-dessus.

6. Cliquez sur <DNT>**Create**</DNT>.

Après avoir créé un <DNT>connector</DNT>:

* Pour afficher le schéma ou supprimer le <DNT>connector</DNT>, sélectionnez le menu <Icon name="fe-more-horizontal" />de la ligne <DNT>connector</DNT>.
* Pour mettre à jour les paramètres de contrôle d&apos;accès, sélectionnez le <DNT>connector</DNT>, ajoutez ou supprimez des droits d&apos;accès et enregistrez vos modifications.
* Pour mettre à jour la configuration <DNT>connector</DNT>, supprimez et recréez le <DNT>connector</DNT> avec les nouveaux détails.

    ### Générer la clé des identifiants du compte de service [#generate-credentials-key]

    Pour authentifier <DNT>Lens</DNT> avec <DNT>Google Sheets</DNT>, vous devez créer un compte de service <DNT>Google Cloud</DNT> et générer une clé JSON.

    <Steps>
      <Step>
        ### Créer un compte de service [#create-service-account]

        Pour créer un compte de service <DNT>Google Cloud</DNT>, suivez les instructions de la page [Créer des comptes de service](https://cloud.google.com/iam/docs/service-accounts-create).
      </Step>

      <Step>
        ### Configurer les autorisations pour la création de clés [#set-up-permissions]

        Pour créer des clés pour votre compte de service, vous devez disposer des permissions appropriées. Pour plus d&apos;informations sur les autorisations requises, consultez [Autorisations requises](https://cloud.google.com/iam/docs/keys-create-delete#required-permissions).
      </Step>

      <Step>
        ### Créer une clé JSON [#create-json-key]

        Pour générer une clé JSON pour votre compte de service, suivez les étapes décrites dans [Créer une clé de compte de service](https://cloud.google.com/iam/docs/keys-create-delete#creating).
      </Step>

      <Step>
        ### Convertir la clé JSON en base64 [#convert-json-key-to-base64]

        Pour utiliser la clé JSON dans <DNT>Lens</DNT>, convertissez l&apos;intégralité du contenu du fichier JSON en base64 et enregistrez-le dans un fichier texte.
      </Step>
    </Steps>

    ### Configurer l&apos;accès à Google Sheets [#setup-gsheets-access]

    Après avoir généré la clé d&apos;identification, configurez l&apos;accès à vos Google Sheets.

    <Steps>
      <Step>
        ### Partagez vos feuilles de calcul avec le compte de service [#share-sheets-with-service-account]

        Pour permettre à <DNT>Lens</DNT> d&apos;accéder à vos données, accordez des droits de lecture à l&apos;adresse e-mail du compte de service pour chaque Google Sheet que vous souhaitez connecter.
      </Step>

      <Step>
        ### Créer une fiche de métadonnées [#create-metadata-sheet]

        Pour gérer les feuilles disponibles pour <DNT>Lens</DNT>, créez une nouvelle Google Sheet avec les quatre colonnes suivantes :

        * **Nom de la feuille**: Le nom de chaque feuille à laquelle vous souhaitez accéder avec <DNT>Lens</DNT>. Toutes les feuilles répertoriées doivent être accessibles par le compte de service.
        * **Sheetid**: L&apos;identifiant unique de chaque feuille. Vous pouvez trouver ceci dans l&apos;URL de la feuille : `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`.
        * **Propriétaire**: Le propriétaire de la feuille. Utilisez `Trino` comme valeur par défaut.
        * **Notes**: Toute note ou description supplémentaire pour la feuille.
      </Step>

      <Step>
        ### Partager la fiche de métadonnées [#share-metadata-sheet]

        Pour permettre à <DNT>Lens</DNT> de lire les métadonnées, accordez l&apos;autorisation de lecture à votre compte de service pour la feuille de métadonnées.
      </Step>

      <Step>
        ### Copier l&apos;ID de la feuille de métadonnées [#copy-metadata-sheet-id]

        Pour configurer le <DNT>connector</DNT>, copiez l&apos;ID de la feuille depuis l&apos;URL de la feuille de métadonnées. L&apos;ID de la feuille apparaît dans l&apos;URL sous la forme : `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`.
      </Step>
    </Steps>

    ### Accorder <DNT>Lens</DNT> [#grant-iceberg-access]

    Pour permettre à <DNT>Lens</DNT> d&apos;interroger vos tables Iceberg, créez une stratégie et un rôle IAM dans votre compte AWS qui accordent un accès en lecture à votre catalogue de données Glue et à votre bucket S3.

    <Callout variant="important">
      Le Glue Data Catalog et le bucket S3 contenant vos données Iceberg doivent se trouver dans la même région AWS.
    </Callout>

    <Steps>
      <Step>
        ### Créer une stratégie IAM [#create-iceberg-policy]

        [Créez une stratégie IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) avec les autorisations suivantes pour permettre à <DNT>Lens</DNT> de lire les métadonnées Glue et les fichiers de données dans S3. Remplacez `{REGION}`, `{ACCOUNT_ID}`, `{DATABASE_NAME}` et `{S3_BUCKET_NAME}` par vos valeurs. Notez le nom de la stratégie que vous saisissez, car vous en aurez besoin pour attacher cette stratégie au rôle IAM.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Action": [
                "glue:GetDatabase",
                "glue:GetTable",
                "glue:GetTables",
                "glue:SearchTables",
                "glue:GetPartition",
                "glue:GetPartitions",
                "glue:BatchGetPartition"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:catalog",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:database/{DATABASE_NAME}",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:table/*"
              ]
            },
            {
              "Action": [
                "s3:GetObject",
                "s3:ListBucket",
                "s3:GetBucketLocation"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:s3:::{S3_BUCKET_NAME}",
                "arn:aws:s3:::{S3_BUCKET_NAME}/*"
              ]
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### Créer un rôle IAM avec une relation de confiance [#create-iceberg-role]

        [Créez un rôle IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) et configurez la relation de confiance pour autoriser le compte de service <DNT>Lens</DNT> à l&apos;assumer à l&apos;aide d&apos;un ID externe unique. Lors de la configuration du rôle, utilisez ces valeurs :

        * **Type d&apos;entité de confiance**: Compte AWS
        * **ID de compte**: `017663287629` (compte AWS de New Relic)
        * **Exiger un ID externe**: Activé. Saisissez une chaîne unique de votre choix et enregistrez-la pour la configuration <DNT>Lens</DNT> <DNT>connector</DNT>.
        * **Stratégie d&apos;autorisations**: recherchez le nom de la stratégie que vous avez créée à l&apos;étape précédente et sélectionnez-la.

        La stratégie de confiance doit respecter le format suivant. Remplacez `<YOUR_EXTERNAL_ID>` par l&apos;ID externe que vous avez saisi.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        Après avoir créé le rôle, ouvrez la page récapitulative du rôle et copiez le **Role ARN**. Vous aurez besoin de cet ARN et de l&apos;ID externe lors de la configuration du <DNT>Lens</DNT> <DNT>connector</DNT>.
      </Step>
    </Steps>
  </Collapser>

    ### Accorder <DNT>Lens</DNT> [#grant-cloudwatch-access]

    Pour permettre à <DNT>Lens</DNT> d&apos;interroger vos métriques CloudWatch, créez une stratégie et un rôle IAM dans votre compte AWS qui accordent un accès en lecture à CloudWatch.

    <Steps>
      <Step>
        ### Créer une stratégie IAM [#create-cloudwatch-policy]

        [Créez une stratégie IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) avec les autorisations suivantes pour permettre à <DNT>Lens</DNT> de lire les métriques CloudWatch depuis votre compte AWS. Notez le nom de la stratégie que vous saisissez, car vous en aurez besoin pour attacher cette stratégie au rôle IAM.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Action": [
                "cloudwatch:ListMetrics",
                "cloudwatch:GetMetricData",
                "cloudwatch:GetMetricStatistics",
                "tag:GetResources"
              ],
              "Resource": "*"
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### Créer un rôle IAM avec une relation de confiance [#create-cloudwatch-role]

        [Créez un rôle IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html) et configurez la relation de confiance pour autoriser le compte de service <DNT>Lens</DNT> à l&apos;assumer à l&apos;aide d&apos;un ID externe unique. Lors de la configuration du rôle, utilisez ces valeurs :

        * **Type d&apos;entité de confiance**: Compte AWS
        * **ID de compte**: `017663287629` (compte AWS de New Relic)
        * **Exiger un ID externe**: Activé. Saisissez une chaîne unique de votre choix et enregistrez-la pour la configuration <DNT>Lens</DNT> <DNT>connector</DNT>.
        * **Stratégie d&apos;autorisations**: recherchez le nom de la stratégie que vous avez créée à l&apos;étape précédente et sélectionnez-la.

        La stratégie de confiance doit respecter le format suivant. Remplacez `<YOUR_EXTERNAL_ID>` par l&apos;ID externe que vous avez saisi.

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        Après avoir créé le rôle, ouvrez la page récapitulative du rôle et copiez le **Role ARN**. Vous aurez besoin de cet ARN et de l&apos;ID externe lors de la configuration du <DNT>Lens</DNT> <DNT>connector</DNT>.
      </Step>
    </Steps>
  </Collapser>
