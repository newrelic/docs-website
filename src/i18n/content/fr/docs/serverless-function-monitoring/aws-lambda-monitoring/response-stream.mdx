Vous pouvez monitorer la fonction AWS Lambda qui utilise [le streaming de réponses](https://aws.amazon.com/blogs/compute/introducing-aws-lambda-response-streaming/) dans les applications Node.js pour obtenir des informations réelles détaillées sur les performances et le comportement de chaque invocation. Cette solution s&apos;intègre à [monitoring de l&apos;IA](/docs/ai-monitoring/intro-to-ai-monitoring/) pour vous permettre de suivre les réponses du modèle et de collecter des données d&apos;observabilité liées à votre charge de travail d&apos;IA. Grâce à ces fonctionnalités, vous pouvez optimiser les performances de vos applications sans serveur et maintenir une monitoring complet des processus pilotés par l&apos;IA.

Ce document vous aide à :

* Configurer le monitoring de streaming de réponses de fonction Lambda
* Comprendre les métriques et les données disponibles pour le streaming de réponses de fonction Lambda
* Apprenez à utiliser l&apos;interface utilisateur de New Relic pour monitorer et observer les performances de votre streaming de réponses de fonction Lambda

## Configurer le streaming de réponse Lambda [#setup]

### Prérequis [#prerequisites]

* Vérifiez que votre fonction Lambda répond à notre [compatibilité et à nos exigences](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/compatibility-requirement-lambda-monitoring).
* Utilisez la [dernière couche lambda](https://layers.newrelic-external.com) disponible pour Node.js.

### Configurer le streaming de réponse Lambda [#configure-response-streaming]

Pour activer le monitoring du streaming de réponse pour votre fonction Node.js Lambda:

1. Installer et configurer [le monitoring Lambda sur New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/configure-serverless-aws-monitoring).

2. Ajoutez les [variables d&apos;environnement](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda) requises pour la fonction de streaming de réponse Node.js Lambda.

3. Enregistrez les modifications et déployez la fonction.

   Après déploiement, vous pouvez visualiser les données de réponse Lambda de streaming sur la plateforme New Relic. Pour plus d&apos;informations, reportez-vous à [Accéder aux données de réponse Lambda en streaming](#access-ui).

## Accéder aux données de streaming de la réponse Lambda [#access-ui]

Pour visualiser le streaming des données de réponses Lambda sur la plateforme New Relic :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; left navigation pane &gt; All Capabilities &gt; Serverless Functions**</DNT>.

2. Sélectionnez la fonction Lambda requise qui utilise le streaming de réponse.

3. Visualisez le résumé, les invocations, les métriques CloudWatch et le traçage distribués pour votre streaming de réponses de fonction Lambda.

4. Dans la section **More views** , cliquez sur **AI Responses**.

   Pour en savoir plus sur les réponses de l’IA, reportez-vous à [Afficher les réponses de l’IA](/docs/ai-monitoring/explore-ai-data/view-ai-responses).

## Indicateurs clés pour le streaming des réponses [#key-metrics]

Sur la page **Cloudwatch metrics**, monitorez les métriques suivantes liées au streaming des réponses :

* **Octets sortants diffusés :** affiche le nombre moyen d&apos;octets diffusés par la fonction Lambda.
* **Débit sortant diffusé :** affiche le débit moyen auquel les octets sont diffusés par la fonction Lambda.
