## Problème : les métriques ou les invocations CloudWatch n&apos;apparaissent pas [#missing-cloudwatch-metrics]

Après avoir instrumenté votre fonction AWS Lambda avec OpenTelemetry et l&apos;avoir invoquée, la section <DNT>**CloudWatch metrics**</DNT> ou l&apos;onglet <DNT>**Invocations**</DNT> n&apos;apparaît pas dans l&apos;interface utilisateur de New Relic.

## Solution [#solution-cloudwatch]

New Relic requiert les attributs de ressource OpenTelemetry suivants pour identifier et afficher correctement votre fonction Lambda dans l&apos;interface utilisateur :

| Attribut    | Description                              | Exemple de valeur                                                   |
| ----------- | ---------------------------------------- | ------------------------------------------------------------------- |
| `faas.name` | Le nom de la fonction Lambda             | `my-lambda-function`                                                |
| `faas.id`   | L&apos;ARN complet de la fonction Lambda | `arn:aws:lambda:us-east-1:123456789012:function:my-lambda-function` |

Si ces attributs sont manquants, définissez-les à l&apos;aide de la variable d&apos;environnement `OTEL_RESOURCE_ATTRIBUTES`.

**Modèle SAM :**

```yaml
yourFunctionHere:
  Type: AWS::Serverless::Function
  Properties:
    # ...
    Environment:
      Variables:
        OTEL_RESOURCE_ATTRIBUTES: "faas.name=my-lambda-function,faas.id=arn:aws:lambda:us-east-1:123456789012:function:my-lambda-function"
```

<Callout variant="important">
  Remplacez `my-lambda-function` et l&apos;ARN par le nom et l&apos;ARN réels de votre fonction. L&apos;ARN doit correspondre exactement à la valeur affichée dans la console AWS Lambda.
</Callout>

**Console AWS :**

1. Ouvrez votre fonction dans la [console Lambda](https://console.aws.amazon.com/lambda).
2. Choisissez <DNT>**Configuration**</DNT>, puis <DNT>**Environment variables**</DNT>.
3. Choisissez <DNT>**Edit**</DNT>, puis <DNT>**Add environment variable**</DNT>.
4. Définissez <DNT>**Key**</DNT> sur `OTEL_RESOURCE_ATTRIBUTES` et <DNT>**Value**</DNT> sur `faas.name=my-lambda-function,faas.id=arn:aws:lambda:us-east-1:123456789012:function:my-lambda-function`.
5. Choisissez <DNT>**Save**</DNT>.

Si vous avez déjà défini d&apos;autres attributs de ressource, ajoutez les nouvelles valeurs sous forme de paires clé-valeur séparées par des virgules :

```
OTEL_RESOURCE_ATTRIBUTES=service.name=my-service,faas.name=my-lambda-function,faas.id=arn:aws:lambda:us-east-1:123456789012:function:my-lambda-function
```

## Problème : ID de requête manquant dans les traces [#missing-request-id]

Les traces de votre fonction Lambda apparaissent dans New Relic, mais l&apos;ID de requête AWS n&apos;est pas visible sur le span racine, ce qui rend difficile la corrélation des traces avec les entrées de CloudWatch Logs.

## Solution [#solution-request-id]

Définissez l&apos;attribut `faas.execution` sur la span racine dans votre gestionnaire. Les conventions sémantiques d&apos;OpenTelemetry définissent cet attribut comme l&apos;ID de requête ou d&apos;invocation du fournisseur de cloud.

**Python:**

```python
from opentelemetry import trace

def handler(event, context):
    span = trace.get_current_span()
    span.set_attribute("faas.execution", context.aws_request_id)

    # Your handler logic here
    return {"statusCode": 200, "body": "OK"}
```

**Node.js :**

```js
const { trace } = require('@opentelemetry/api');

exports.handler = async (event, context) => {
  const span = trace.getActiveSpan();
  if (span) {
    span.setAttribute('faas.execution', context.awsRequestId);
  }

  // Your handler logic here
  return { statusCode: 200, body: 'OK' };
};
```

**Java:**

```java
import io.opentelemetry.api.trace.Span;

public class Handler implements RequestHandler<APIGatewayProxyRequestEvent, APIGatewayProxyResponseEvent> {
    @Override
    public APIGatewayProxyResponseEvent handleRequest(APIGatewayProxyRequestEvent event, Context context) {
        Span span = Span.current();
        span.setAttribute("faas.execution", context.getAwsRequestId());

        // Your handler logic here
        return new APIGatewayProxyResponseEvent().withStatusCode(200);
    }
}
```

<Callout variant="tip">
  New Relic affiche `faas.execution` en tant que <DNT>**Request ID**</DNT> dans le panneau des détails de la trace, ce qui permet de trouver facilement l&apos;entrée CloudWatch Logs correspondante pour toute invocation.
</Callout>
