## Avant de commencer [#begin]

Assurez-vous que vos Azure Functions répondent à notre [compatibilité et à nos exigences](/docs/serverless-function-monitoring/azure-function-monitoring/compatibility-requirement-azure-monitoring).

<Steps>
  <Step>
    ## Associez votre compte Azure à New Relic

    Vous pouvez lier votre compte Azure à New Relic en configurant l’interrogation des métriques Azure Monitor. Cela vous permet de voir les métriques dans l&apos;UI de New Relic. Pour plus d’informations, consultez [l’intégration Azure](https://docs.newrelic.com/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-functions-monitoring-integration/#polling).
  </Step>

  <Step>
    ## Instrumenter vos Azure Functions avec l&apos;agent .NET de New Relic

    En fonction de vos besoins, sélectionnez l’une des options suivantes pour instrumenter vos Azure Functions avec l’agent .NET New Relic.

    <Tabs>
      <TabsBar>
        <TabsBarItem id="containerized-instrumentation">Fonctions conteneurisées</TabsBarItem> <TabsBarItem id="linux-instrumentation">Linux</TabsBarItem> <TabsBarItem id="windows-instrumentation">Windows</TabsBarItem>
      </TabsBar>

      <TabsPages>
        <TabsPageItem id="linux-instrumentation">
          1. Ajoutez le package NuGet `NewRelic.Agent` à votre projet d’application.

             * Dans l’éditeur de code Visual Studio, utilisez le gestionnaire de paquets NuGet pour rechercher et ajouter la dernière version de `NewRelic.Agent` à votre application.
             * Si vous utilisez d&apos;autres environnements de développement, vous pouvez ajouter le package avec `dotnet add package NewRelic.Agent`.

          2. Utilisez votre mécanisme de publication préféré pour déployer votre application mise à jour sur Azure. Cela inclut l&apos;agent New Relic, qui est installé dans le dossier `/home/site/www/newrelic` .
        </TabsPageItem>

        <TabsPageItem id="windows-instrumentation">
          L’extension New Relic Azure Websites (v1.6.0 et versions ultérieures) configure automatiquement instrumentation pour vos fonctions Windows Azure .

          Pour installer l&apos;extension New Relic Azure Websites, procédez comme suit :

          1. Dans le portail Azure, accédez à votre application de fonction.
          2. Cliquez sur la section <DNT>**Development tools**</DNT> et sélectionnez <DNT>**Extensions**</DNT>.
          3. Cliquez sur <DNT>**+ Add**</DNT>.
          4. Sélectionnez <DNT>**Search for an extension to install**</DNT> et entrez `New Relic .NET Agent` dans la case <DNT>**Filter items**</DNT> .
          5. Sélectionnez l’extension <DNT>**New Relic .NET Agent(vx.x.x) - New Relic**</DNT> et cliquez sur <DNT>**Add**</DNT>.
          6. Une fois l&apos;installation terminée, l&apos;extension apparaîtra dans la liste des extensions installées. Pour vérifier l&apos;installation correcte, cliquez sur le lien sous la colonne « Parcourir » pour afficher le log d&apos;installation.
        </TabsPageItem>

        <TabsPageItem id="containerized-instrumentation">
          Pour installer l&apos;agent .NET New Relic, dans la dernière étape de votre Dockerfile, ajoutez les lignes suivantes :

          ```dockerfile
          # Install the latest New Relic .NET agent using the apt-get package manager
          RUN apt-get update && apt-get install -y wget ca-certificates gnupg \
              && echo 'deb http://apt.newrelic.com/debian/ newrelic non-free' | tee /etc/apt/sources.list.d/newrelic.list \
              && wget https://download.newrelic.com/548C16BF.gpg \
              && apt-key add 548C16BF.gpg \
              && apt-get update \
              && apt-get install -y newrelic-dotnet-agent \
              && rm -rf /var/lib/apt/lists/*
          ```
        </TabsPageItem>
      </TabsPages>
    </Tabs>
  </Step>

  <Step>
    ## Configurer les variables d’environnement

    Après avoir instrumenté vos Azure Functions avec l’agent .NET New Relic, procédez comme suit pour configurer les [variables d’environnement](/docs/serverless-function-monitoring/azure-function-monitoring/env-variables-azure) afin d’envoyer les données à New Relic.

    1. Accédez à vos Azure Functions dans le portail Azure , sous **Settings**, cliquez sur **Environment variables**, puis sur **Advanced edit**.

    2. En fonction des exigences de votre application, collez les valeurs suivantes.

    <Callout variant="important">
      Assurez-vous d&apos;ajouter une virgule à la fin de la dernière ligne existante et de mettre à jour votre clé de licence et le nom de votre application dans la configuration suivante.
    </Callout>

    <Tabs>
      <TabsBar>
        <TabsBarItem id="containerized-configuration">Fonctions conteneurisées</TabsBarItem> <TabsBarItem id="linux-configuration">Linux</TabsBarItem> <TabsBarItem id="windows-configuration">Windows</TabsBarItem>
      </TabsBar>

      <TabsPages>
        <TabsPageItem id="linux-configuration">
          ```json
          {
            "name": "CORECLR_ENABLE_PROFILING",
            "value": "1",
            "slotSetting": false
          },
          {
            "name": "CORECLR_NEW_RELIC_HOME",
            "value": "/home/site/wwwroot/newrelic",
            "slotSetting": false
          },
          {
            "name": "CORECLR_PROFILER",
            "value": "{36032161-FFC0-4B61-B559-F6C5D41BAE5A}",
            "slotSetting": false
          }, 
          {
            "name": "CORECLR_PROFILER_PATH",
            "value": "/home/site/wwwroot/newrelic/libNewRelicProfiler.so",
            "slotSetting": false
          },
          {
            "name": "NEW_RELIC_AZURE_FUNCTION_MODE_ENABLED",
            "value": "1",
            "slotSetting": false
          },
          {
            "name": "NEW_RELIC_LOG_DIRECTORY",
            "value": "/home/LogFiles/NewRelic",
            "slotSetting": false
          },
          {
            "name": "NEW_RELIC_LICENSE_KEY",
            "value": "<your newrelic license key here>",
            "slotSetting": false
          }
          ```
        </TabsPageItem>

        <TabsPageItem id="windows-configuration">
          ```json
          {
            "name": "NEW_RELIC_LICENSE_KEY",
            "value": "YOUR_NEW_RELIC_LICENSE_KEY",
            "slotSetting": false
          }
          ```

          Vous pouvez éventuellement spécifier la version de l&apos;agent .NET que vous souhaitez installer en ajoutant la variable d&apos;environnement suivante :

          ```json
          {
            "name": "NEW_RELIC_AGENT_VERSION_OVERRIDE",
            "value": "10.35.0",
            "slotSetting": false
          }
          ```
        </TabsPageItem>

        <TabsPageItem id="containerized-configuration">
          ```json

            "value": "1",
            "slotSetting": false

            "slotSetting": false
          },
          {
            "name": "CORECLR_PROFILER",
            "value": "{36032161-FFC0-4B61-B559-F6C5D41BAE5A}",

            "slotSetting": false
          },
          {
            "name": "NEW_RELIC_LICENSE_KEY",
            "value": "<your newrelic license key here>",
            "slotSetting": false
          }
          ```
        </TabsPageItem>
      </TabsPages>
    </Tabs>
  </Step>

  <Step>
    ## Redémarrez vos Azure Functions

    Après avoir ajouté les variables d’environnement, redémarrez vos Azure Functions pour appliquer les modifications.
  </Step>
</Steps>

## Et ensuite ?

Une fois les étapes d’installation et configuration terminées, envoyez du trafic vers vos Azure Functions pour voir les métriques dans l’ UI Dashboard New Relic.
