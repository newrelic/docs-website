New Relic dispose d&apos;outils pour vous aider à monitorer votre infrastructure logicielle afin que vous puissiez fournir un système fiable. Que vous travailliez dans le cloud, sur des hôtes dédiés ou avec des conteneurs exécutés dans des environnements orchestrés, vous pouvez utiliser monitoring de l&apos;infrastructure pour améliorer les résultats en matière de santé et de performances.

<figcaption>
  À partir de <DNT>**[one.newrelic.com](https://one.newrelic.com) &amp;gt; All Capabilities &amp;gt; Infrastructure**</DNT>: la page <DNT>**Hosts**</DNT> affiche les données de performances de votre système, de vos réseaux, de vos processus et de votre stockage.
</figcaption>

## monitoringdes hôtes sur site et virtuels [#infra-agent-intro]

Si vous souhaitez monitorer des hôtes sur site ou virtuels, vous pouvez installer notre agent infrastructure . Il collecte des données sur les performances et la santé des ressources et des processus du système. À partir de là, vous pouvez utiliser l&apos;agent infrastructure en combinaison avec un certain nombre d&apos; [intégrations sur hôte](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) qui étendent monitoring aux outils infrastructure tels que la base de données, les services de messagerie et les serveurs d&amp;apos;applications. Vous pouvez également utiliser l&amp;apos;agent d&amp;apos;infrastructure pour [transférer les journaux](/docs/logs/enable-log-monitoring-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent).

<img title="Agents and New Relic" alt="Agents and New Relic" src="/images/infrastructure_diagram_angets-and-nr.webp" />

Vous pouvez installer l&apos;agent infrastructure sur les systèmes Linux, macOS et Windows . En fonction de vos besoins d’observabilité, vous disposez de plusieurs options différentes pour déployer l’agent dans votre système. Nous vous recommandons de consulter notre [méthode d’installation Choisissez votre agent d’infrastructure](/docs/infrastructure/infrastructure-monitoring/get-started/choose-infra-install-method) pour décider de l’option qui correspond à votre cas d’utilisation.

## Réduisez le MTTR grâce à des données exploitables [#why-it-matters]

Les mesures et analyses temps réels réduisent votre [Délai moyen de résolution (MTTR)](https://newrelic.com/devops/how-to-reduce-mttr) (MTTR) en connectant les modifications des performances de l&amp;apos;hôte aux modifications de votre configuration. Voici un exemple de la manière dont les données d’infrastructure peuvent vous aider :

Depuis la <DNT>**Inventory**</DNT> page, concentrez-vous sur toutes les infrastructure [entités](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic) de votre domaine.

<img title="New Relic - Infrastructure monitoring - Inventory" alt="New Relic - Infrastructure monitoring - Inventory" src="/images/infrastructure_screenshot-full_inventory-page.webp" />

<figcaption>
  <DNT>**[one.newrelic.com](https://one.newrelic.com) &amp;gt; All Capabilities &amp;gt; Infrastructure monitoring &amp;gt; Inventory**</DNT>: Effectuez une recherche dans l&amp;apos;ensemble de votre système pour trouver exactement lesquels de vos hôtes contiennent un package, des configurations ou un script de démarrage particuliers.
</figcaption>

À partir de la page <DNT>**Events**</DNT> , vous pouvez suivre les modifications de configuration, les redémarrages, les sessions SSH et d&amp;apos;autres modifications [d&apos;événements clés](/docs/infrastructure-events-page#types) . Un flux en temps réel vous fournit un changelog pour l&amp;apos;ensemble de votre infrastructure.

<img title="New Relic - Infrastructure monitoring - Events" alt="New Relic - Infrastructure monitoring - Events" src="/images/infrastructure_screenshot-full_events-page.webp" />

<figcaption>
  La page <DNT>**Events**</DNT> contient un flux en temps réel de tout ce qui se passe dans vos hôtes.
</figcaption>

Quelle que soit la manière dont vous l&apos;utilisez, notre solution collecte et affiche vos données [en toute sécurité](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-security) afin que votre monitoring ne soit jamais en retard sur la réalité.

## Afficher le log de vos données infrastructure et d&apos;application [#logs-context]

Rassemblez vos données de log et infrastructure pour rendre le dépannage plus facile et plus rapide ! Avec les logs en contexte, vous pouvez voir le log de vos données infrastructure , telles que le cluster Kubernetes. Vous pouvez ajouter autant de fichiers de configuration que vous le souhaitez. Ces fichiers servent de sources qui poussent vos log mémorisés vers notre plateforme.

Vous pouvez également voir les messages de log liés à vos erreurs et les tracer directement dans UI de votre application ou de votre hôte. Pour plus d&apos;informations, consultez notre documentation [sur les journaux APM en contexte](/docs/apm/new-relic-apm/getting-started/get-started-logs-context) .

Si votre <InlinePopover type="apm" />agent ne prend pas encore en charge notre solution automatique de logging en contexte, vous pouvez continuer à utiliser nos solutions manuelles de logging en contexte et [transférer votre log via notre agent infrastructure ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)ou d&amp;apos;autres [redirecteurs de logtiers pris en charge](/docs/logs/forward-logs/enable-log-management-new-relic).
