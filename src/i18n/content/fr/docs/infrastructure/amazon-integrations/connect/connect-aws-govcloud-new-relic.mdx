Les régions [AWS GovCloud (US)](https://aws.amazon.com/govcloud-us/) sont conçues pour répondre aux besoins réglementaires spécifiques des États-Unis (agences fédérales, étatiques et locales), des établissements d&apos;enseignement et de l&apos;écosystème de soutien. Il s&apos;agit d&apos;une région AWS isolée conçue pour héberger des données sensibles et des charges de travail réglementées dans le cloud, aidant les clients à prendre en charge leurs exigences de conformité avec le gouvernement américain. L&apos;[ensemble des services AWS disponibles](https://aws.amazon.com/govcloud-us/details/) est un sous-ensemble de l&apos;écosystème AWS.

New Relic vous offre la confiance nécessaire pour déployer vos services les plus critiques sur GovCloud, vous permettant de monitorer et d&apos;observer l&apos;ensemble de votre écosystème depuis [notre plateforme](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one).

La fonctionnalité AWS flux de métriques CloudWatch n&apos;est pas disponible dans les régions GovCloud.

<Callout variant="tip">
  Vous pouvez utiliser Terraform pour automatiser le processus d&apos;activation de l&apos;intégration cloud. Découvrez comment procéder sur le [site de documentation officielle de Terraform](https://registry.terraform.io/providers/newrelic/newrelic/latest/docs/guides/cloud_integrations_guide).
</Callout>

## Connecter AWS GovCloud à New Relic

Pour commencer à recevoir des données Amazon avec l&apos;intégration AWS de New Relic, connectez votre compte Amazon à New Relic.

1. Obtenez [vos identifiants](#govcloud-credentials).
2. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; GovCloud**</DNT>.
3. Cliquez sur <DNT>**Add AWS GovCloud account**</DNT>.
4. Donnez un nom à votre compte AWS, fournissez les informations d’identification pour connecter votre compte et cliquez sur <DNT>**Submit**</DNT>.
5. Sélectionnez les services Amazon Web Services à monitorer avec l’intégration de New Relic Infrastructure, puis cliquez sur <DNT>**Save**</DNT>.

## Comment obtenir les identifiants GovCloud pour New Relic [#govcloud-credentials]

1. Depuis la [console IAM](https://console.amazonaws-us-gov.com/iam/home#/users), cliquez sur <DNT>**Add user**</DNT>.

2. Pour le <DNT>**User name**</DNT>, tapez <DNT>**NewRelicInfrastructure-Integrations**</DNT>. Pour <DNT>**Select AWS access type**</DNT>, sélectionnez comme <DNT>**Programmatic access**</DNT>.

   <img title="AWS-add-user.png" alt="Add a user in the AWS IAM console." src="/images/infrastructure_screenshot-crop_aws-add-user.webp" />

   <figcaption>
     <DNT>**AWS IAM console &gt; Add user**</DNT>: ajouter <DNT>**NewRelicInfrastructure-Integrations**</DNT> en tant qu&apos;utilisateur.
   </figcaption>

3. Joindre la politique : recherchez <DNT>**ReadOnlyAccess**</DNT>, cochez la case correspondant à la politique nommée <DNT>**ReadOnlyAccess**</DNT>, puis cliquez sur <DNT>**Next: Tags**</DNT> (l’ajout d’une balise est facultatif). Vous pouvez également [créer votre propre politique gérée](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) et limiter les autorisations que vous accordez à New Relic en fonction des services AWS que vous souhaitez monitorer.

   <img title="AWS-set-permissions.png" alt="AWS-set-permissions.png" src="/images/infrastructure_screenshot-crop_aws-set-permissions.webp" />

   <figcaption>
     <DNT>**AWS IAM console &gt; Add user &gt; Set permissions**</DNT>: sélectionnez <DNT>**ReadOnlyAccess**</DNT>.
   </figcaption>

4. Sur la page <DNT>**Tags**</DNT> , cliquez sur <DNT>**Next: Review**</DNT>. Consultez le résumé des détails de l’utilisateur et cliquez sur <DNT>**Create user**</DNT>.

   <img title="AWS-create-user.png" alt="In IAM console, create the user." src="/images/infrastructure_screenshot-crop_aws-create-user.webp" />

   <figcaption>
     <DNT>**AWS IAM console &gt; Add user &gt; Set permissions &gt; Tags &gt; Review**</DNT>: vérifiez que les informations du nouvel utilisateur sont exactes.
   </figcaption>

5. Votre utilisateur devrait être créé avec succès. Téléchargez les informations d’identification de sécurité de l’utilisateur en cliquant sur le bouton <DNT>**Download .csv**</DNT> , puis sur <DNT>**Close**</DNT>.
