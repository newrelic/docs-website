Le service [AWS Auto Scaling](https://aws.amazon.com/autoscaling/) permet de lancer ou d&apos;arrêter automatiquement les instances Amazon EC2. Il permet d&apos;adapter dynamiquement la capacité d&apos;Amazon EC2 en fonction des politiques, des planifications et des contrôles de santé définis par l&apos;utilisateur.

[L&apos;intégration de New Relic Infrastructure](/docs/infrastructure/introduction-infra-monitoring) inclut une intégration AWS Auto Scaling qui signale des données sur les groupes de votre service Auto Scaling aux produits New Relic. Ce document explique comment activer cette intégration et décrit les données qui peuvent être signalées.

## Activer l&apos;intégration [#activate]

Pour activer cette intégration :

1. Suivez les procédures standard pour [connecter les services AWS à New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).
2. [Activez les métriques de groupe de mise à l’échelle automatique](https://docs.aws.amazon.com/autoscaling/ec2/userguide/as-instance-monitoring.html#as-group-metrics) afin qu’AWS Auto Scaling envoie des données échantillonnées à CloudWatch chaque minute.

Pour activer les métriques de groupe à l&apos;aide de la console :

1. Ouvrez la console Amazon EC2 sur [console.aws.amazon.com/ec2/](https://console.aws.amazon.com/ec2/).
2. Dans le volet de navigation, sélectionnez <DNT>**Auto Scaling Groups &gt; (select your group)**</DNT>.
3. Dans l’onglet <DNT>**Monitoring**</DNT>, sélectionnez <DNT>**Auto Scaling Metrics &gt; Enable Group Metrics Collection**</DNT> ou <DNT>**Display &gt; Auto Scaling**</DNT>.

## configuration et sondage [#polling]

Informations [d&apos;interrogation par défaut](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) pour l&apos;intégration AWS Auto Scaling :

* Intervalle d&apos;interrogation de New Relic : 5 minutes
* Intervalle de données Amazon CloudWatch : 1 minute par défaut

Pour modifier la fréquence d&apos;interrogation et filtrer les données, utilisez [les options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

## Rechercher et utiliser des données [#find-data]

Pour [afficher les données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; AWS**</DNT>, puis sélectionnez l’un des <DNT>**AWS Auto Scaling**</DNT> liens d’intégration.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) en utilisant ces [types d&apos;événements](/docs/data-apis/understand-data/new-relic-data-types/#event-data):

* `AutoScalingInstanceSample`
* `AutoScalingGroupSample`
* `AutoScalingLaunchConfigurationSample`
* `AutoScalingPolicySample`
* `AutoScalingRegionLimitSample`
