[L'intégration de New Relic Infrastructure](/docs/infrastructure/introduction-infra-monitoring) inclut une intégration permettant de signaler vos données Amazon API Gateway à New Relic. Ce document explique comment activer cette intégration et décrit les données qui peuvent être signalées.

<Callout variant="tip">
  Les métriques API Gateway v1 sont disponibles exclusivement via l'intégration d'interrogation API pour Amazon API Gateway. [Les métriques API Gateway v2](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-api-gateway-v2-monitoring-integration) sont diffusées via le flux de métriques CloudWatch.
</Callout>

## Caractéristiques [#features]

[API Gateway d'Amazon](https://aws.amazon.com/api-gateway/) est un service entièrement géré qui vous permet de créer, publier, maintenir, monitorer et sécuriser des API à n&amp;apos;importe quelle échelle. Avec l&amp;apos;intégration de New Relic API Gateway, vous obtenez davantage de données sur le fonctionnement de votre couche API en coulisses. Vous recevrez des données métriques sur le nombre d&amp;apos;appels d&amp;apos;API, les requests traitées, le nombre d&amp;apos;erreurs, les nombres de latences, etc.

Vous pouvez monitorer et alerter sur vos données API Gateway directement depuis [New Relic](/docs/infrastructure), interroger les données et créer un dashboard.

## Exigences [#requierments]

API Gateway n'enverra pas les métriques « Nombre d'appels par ressource », « Erreur 4xx par ressource » et « Erreurs 5xx par ressource » à moins que vous n'ayez explicitement activé les métriques CloudWatch détaillées.

<Callout variant="tip">
  L'activation de ces mesures peut ajouter des frais supplémentaires à [la tarification de votre compte Amazon CloudWatch ](https://aws.amazon.com/cloudwatch/pricing/).
</Callout>

Pour activer les métriques CloudWatch, utilisez l’une de ces options :

* Accédez à la console de gestion AWS, sélectionnez l’option <DNT>**Settings**</DNT> pour CloudWatch, puis sélectionnez l’option permettant d’activer les métriques CloudWatch détaillées.
* Appelez l’action `stage:update` de l’API REST Amazon API Gateway pour mettre à jour la propriété [`metricsEnabled`](https://docs.aws.amazon.com/apigateway/api-reference/resource/stage/#metricsEnabled) en `true`.

## Activer l'intégration [#activate]

Pour activer cette intégration, suivez les procédures standard pour [connecter les services AWS à New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d'interrogation et filtrer les données à l'aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informations [d'interrogation](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) par défaut pour l&amp;apos;intégration de la passerelle API Amazon :

* Intervalle d'interrogation de New Relic : 5 minutes
* Intervalle de données Amazon CloudWatch : 1 minute

## Rechercher et utiliser des données [#find-data]

Pour trouver vos données d’intégration dans l’UI de l'infrastructure, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; AWS**</DNT> et sélectionnez l’un des liens d’intégration API Gateway.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) en utilisant le [type d'événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `ApiGatewaySample` .

Pour en savoir plus sur l’utilisation de vos données, consultez [Comprendre et utiliser les données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).
