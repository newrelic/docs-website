New Relic s'intègre à Amazon Web Services (AWS) pour signaler vos métriques [Amazon Simple Queue Service (SQS)](https://aws.amazon.com/sqs/) et d&amp;apos;autres données à New Relic.

Ce document explique comment activer l'intégration et décrit les données signalées.

## Caractéristiques [#features]

Amazon SQS fournit des files d'attente hébergées pour stocker les messages en transit. L'intégration SQS de New Relic génère des données liées à la file d'attente, telles que le nombre de messages livrés, retardés et reçus, la taille des messages et d'autres [mesures](#metrics). Les données SQS sont disponibles en version prédéfinie <InlinePopover type="dashboards" />et vous pouvez également créer [des requêtes et des graphiques personnalisés](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights). Vous pouvez également créer une condition d&amp;apos;alerte pour vous avertir des modifications apportées aux données SQS.

## Activer l'intégration [#activate]

Cette intégration est disponible via flux de métriques CloudWatch.

Pour activer cette intégration, découvrez comment [connecter les services AWS à New Relic via les flux de métriques CloudWatch](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/).

### Activer à l'aide de l'intégration de sondage API legacy [#polling]

Pour activer cette intégration à l'aide de notre intégration d'interrogation legacy, découvrez comment activer l' [intégration d'API d'interrogation AWS](/docs/infrastructure/amazon-integrations/connect/connect-aws-new-relic-infrastructure-monitoring/).

<Callout variant="tip">
  Vous pouvez modifier la fréquence d'interrogation et filtrer les données à l'aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

  Informations [d'interrogation](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) par défaut pour l&amp;apos;intégration Amazon SQS :

  * Intervalle d'interrogation de New Relic : 5 minutes
  * Intervalle de données AWS CloudWatch : 1 minute
</Callout>

## Rechercher et utiliser des données [#find-data]

Pour trouver vos données d’intégration, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; AWS**</DNT> et sélectionnez l’un des liens d’intégration SQS.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) en utilisant le [type d'événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `QueueSample`, avec une valeur `provider` de `SqsQueue`.

Pour en savoir plus sur l’utilisation de vos données, consultez [Comprendre les données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## données métriques [#metrics]

Cette intégration collecte les métriques Amazon SQS suivantes. Pour des descriptions métriques complètes, consultez [la documentation SQS d'Amazon](http://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/sqs-metricscollected.html).

<table>
  <thead>
    <tr>
      <th width={360}>
        Nom
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `approximateAgeOfOldestMessage`
      </td>

      <td>
        L'âge approximatif du message non supprimé le plus ancien dans la file d'attente, en secondes. Pour les files d'attente de lettres mortes, cette valeur correspond à la durée la plus longue pendant laquelle un message est resté dans la file d'attente.
      </td>
    </tr>

    <tr>
      <td>
        `approximateNumberOfMessagesDelayed`
      </td>

      <td>
        Le nombre approximatif de messages dans la file d'attente qui sont retardés et non disponibles pour lecture immédiate. Cela peut se produire lorsque la file d'attente est configurée comme file d'attente de retard ou lorsqu'un message a été envoyé avec un paramètre de retard.
      </td>
    </tr>

    <tr>
      <td>
        `approximateNumberOfMessagesNotVisible`
      </td>

      <td>
        Le nombre approximatif de messages « en vol ». Les messages sont considérés comme en vol s'ils ont été envoyés à un client mais n'ont pas encore été supprimés ou n'ont pas encore atteint la fin de leur fenêtre de visibilité.
      </td>
    </tr>

    <tr>
      <td>
        `ApproximateNumberOfMessagesVisible`
      </td>

      <td>
        Le nombre approximatif de messages disponibles pour récupération dans la file d'attente.
      </td>
    </tr>

    <tr>
      <td>
        `numberOfEmptyReceives`
      </td>

      <td>
        Le nombre de `ReceiveMessage` appels d&amp;apos;API qui n&amp;apos;ont pas renvoyé de message.
      </td>
    </tr>

    <tr>
      <td>
        `numberOfMessagesDeleted`
      </td>

      <td>
        Le nombre de messages supprimés de la file d'attente.
      </td>
    </tr>

    <tr>
      <td>
        `numberOfMessagesReceived`
      </td>

      <td>
        Le nombre de messages renvoyés par les appels à l'action API `ReceiveMessage` .
      </td>
    </tr>

    <tr>
      <td>
        `numberOfMessagesSent`
      </td>

      <td>
        Le nombre de messages ajoutés à une file d'attente.
      </td>
    </tr>

    <tr>
      <td>
        `sentMessagesSize`
      </td>

      <td>
        La taille en octets des messages ajoutés à une file d'attente.
      </td>
    </tr>
  </tbody>
</table>
