New Relic s&apos;intègre à Amazon Web Services (AWS) pour signaler vos métriques [Amazon Cognito](https://aws.amazon.com/vpc/lattice/) et d&apos;autres données à New Relic.

Ce document explique comment activer l&apos;intégration et décrit les données signalées.

## Caractéristiques [#features]

Amazon VPC Lattice est un service de mise en réseau d&apos;applications entièrement géré que vous utilisez pour connecter, sécuriser et monitorer les services de votre application sur plusieurs comptes et clouds privés virtuels (VPC).

New Relic prend en charge deux intégrations monitoring pour Amazon VPC Lattice : [les mesures CloudWatch](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) et [le log d&apos;accès](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html).

Pour les métriques CloudWatch, il vous suffit de configurer l&apos;intégration une seule fois et tout l&apos;espace de nommage AWS disponible, y compris `AWS/VPCLattice`, sera inclus. Les métriques CloudWatch pour VPC Lattice suivent le trafic entre vos services Lattice et vos groupes cibles.

## Activer l&apos;intégration [#activate]

Cette intégration est disponible via CloudWatch MetricStreams.

Pour activer cette intégration, découvrez comment [connecter les services AWS à New Relic via les flux de métriques CloudWatch](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/).

## Activer l&apos;intégration du log Lattice [#activate-logs-integration]

<Callout variant="important">
  Pour utiliser Kinesis Firehose, vous devez avoir créé un flux de diffusion Kinesis Data Firehose. Si ce n&apos;est pas le cas, vous pouvez en créer un en accédant à la fonctionnalité Amazon Kinesis Firehose depuis votre [console AWS](https://console.aws.amazon.com/firehose) et en créant un flux de diffusion avec `Destination: New Relic`.

  [En savoir plus sur la création d&apos;un flux de diffusion Amazon Kinesis Data Firehose.](https://docs.aws.amazon.com/firehose/latest/dev/basic-create.html?icmpid=docs_console_unmapped)
</Callout>

L&apos;intégration du log d&apos;accès pour Amazon VPC Lattice utilise [Amazon Kinesis Firehose](https://aws.amazon.com/kinesis/data-firehose/). Vous pouvez activer monitoring du log d&apos;accès pour vos services Lattice en ajoutant monitoring au réseau de services ou individuellement aux services que vous souhaitez suivre.

Pour configurer cette intégration :

1. Depuis votre console AWS, accédez au [service AWS VPC](https://console.aws.amazon.com/vpc).
2. Dans le panneau de gauche, sous la liste déroulante <DNT>**VPC Lattice**</DNT>, sélectionnez `Service Networks` OU `Services`, selon l&apos;option que vous souhaitez utiliser pour activer monitoring.
3. Sélectionnez le service ou le réseau de services que vous souhaitez monitorer.
4. Sélectionnez l’onglet `Monitoring` .
5. Sélectionnez `Edit access logs`.
6. Activez la monitoring `Access logs`.
7. Cochez la case intitulée `Kinesis Data Firehose delivery stream`.
8. Sélectionnez un flux de diffusion dans la liste déroulante configuré pour pointer vers New Relic.
9. Enregistrer les modifications.

## Rechercher et utiliser les données du log d&apos;accès [#find-log-data]

Vous pouvez interroger et explorer vos données dans notre UI de log en utilisant ce filtre : `ServiceNetworkArn LIKE '%vpc-lattice%'`.

Explorez [le log d’accès pour VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-access-logs.html) pour obtenir plus d’informations sur les données log envoyées.

## Rechercher et utiliser les données de métriques CloudWatch [#find-metric-data]

Vous pouvez requêter et explorer les données CloudWatch via [notre explorateur de requêtes métriques](/docs/data-apis/understand-data/metric-data/query-metric-data-type/).

Consultez les détails des [métriques CloudWatch pour VPC Lattice](https://docs.aws.amazon.com/vpc-lattice/latest/ug/monitoring-cloudwatch.html) pour obtenir plus d’informations sur les données de métriques envoyées.
