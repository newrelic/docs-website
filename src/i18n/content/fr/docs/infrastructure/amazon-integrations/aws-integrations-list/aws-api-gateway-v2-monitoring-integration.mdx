[L&apos;intégration de New Relic Infrastructure](/docs/infrastructure/introduction-infra-monitoring) inclut désormais une intégration pour AWS API Gateway v2, envoyant ses données métriques à New Relic.

Ce document explique la fonctionnalité de l&apos;intégration, comment l&apos;activer et quelles données vous pouvez signaler.

## Caractéristiques [#features]

Collectez et envoyez des données télémétriques à New Relic depuis votre [Amazon API Gateway v2](https://aws.amazon.com/api-gateway/) en utilisant notre intégration. Vérifiez vos services, interrogez les données entrantes et créez un dashboard pour tout observer en un coup d&apos;œil.

## Activer l&apos;intégration [#activate-integration]

Pour activer cette intégration, découvrez comment [connecter les services AWS à New Relic](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/).

## Rechercher et utiliser des données [#find-use-data]

Pour trouver les métriques de votre intégration, accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Metrics &amp; Events**</DNT> et filtrez par `aws.apigateway`.

### Créer des alertes [#create-alerts]

Vous pouvez configurer des alertes pour recevoir une notification en cas de changement. Par exemple, vous pouvez configurer une alerte pour recevoir une notification concernant les parties concernées en cas d&apos;erreurs critiques ou fatales.

En savoir plus sur la création d’alertes [ici](/docs/tutorial-create-alerts/create-an-alert/).
