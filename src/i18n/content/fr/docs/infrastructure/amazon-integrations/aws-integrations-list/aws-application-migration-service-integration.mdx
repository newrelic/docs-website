Vous pouvez désormais utiliser AWS [Application Migration Service](https://docs.aws.amazon.com/mgn/latest/ug/what-is-application-migration-service.html) pour installer l&apos;agent New Relic sur les serveurs migrant vers AWS.

## Installation [#setup]

Pour qu&apos;AWS installe l&apos; [agent New Relic Infrastructure](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/), vous devez créer un modèle post-lancement via la console AWS. Suivez ces étapes :

1. Depuis votre [console AWS](https://console.aws.amazon.com/mgn/home), ouvrez <DNT>**Application Migration Services** </DNT>.
2. Ouvrez le serveur source souhaité dans la liste disponible.
3. Cliquez sur l’onglet **Paramètres post-lancement** .
4. Sélectionnez **Edit &gt; Install New Relic agent** et ajoutez votre ID de compte et votre clé de licence.

## Monitorer vos données [#monitor-data]

Une fois la configuration terminée, votre application AWS sera monitorée par l&apos;agent New Relic Infrastructure. Pour trouver vos données dans New Relic :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/) &gt; All entities**</DNT>.
2. Dans la section **Hôtes** entité, localisez votre instance.
