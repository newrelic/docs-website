Les métriques reçues via l&apos;intégration AWS flux de métriques CloudWatch sont ingérées dès qu&apos;elles sont disponibles auprès d&apos;AWS. Les métadonnées des ressources et les tags sont requêtes toutes les 15 minutes.

API polling intégration requête services AWS selon un intervalle d&apos;interrogation, qui varie en fonction [de l&apos;intégration](#aws-integrations). Chaque intervalle d’interrogation par New Relic se produit pour chaque entité AWS. Par exemple, si vous disposez de treize équilibreurs de charge Elastic (ELB), chacun d&amp;apos;eux sera interrogé toutes les cinq minutes.

## Gérer la fréquence des sondages [#manage-polling]

Pour les flux métriques, l&apos;intervalle d&apos;interrogation ne peut pas être contrôlé. AWS Kinesis Data Firehose envoie des métriques toutes les 60 secondes ou lorsque 1 Mo de métriques est reçu (selon la première éventualité).

Pour l&apos;intégration de l&apos;interrogation API, par défaut, l&apos;intervalle de fréquence d&apos;interrogation est défini au maximum. Vous pouvez [modifier la fréquence d&apos;interrogation](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations#polling) dans les paramètres de configuration.

Vous pouvez également monitorer l&apos;intervalle d&apos;interrogation en visualisant ces données dans New Relic :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; AWS**</DNT>.
2. Sélectionnez <DNT>**Account status dashboard**</DNT>.
