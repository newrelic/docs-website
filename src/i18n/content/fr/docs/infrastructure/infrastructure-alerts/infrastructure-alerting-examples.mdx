Le champ <DNT>**Alert type**</DNT> de la page <DNT>**Settings &gt; Alerts**</DNT> de monitoring d&apos;infrastructure montre les options que vous pouvez sélectionner pour [créer une condition d&apos;alerte infrastructure ](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information). Vous pouvez également créer une condition d&apos;alerte à partir de n&apos;importe quel graphique infrastructure en sélectionnant les ellipses <Icon name="fe-more-horizontal" /> icône puis <DNT>**Create alert**</DNT>.

## Exemples : pages d&apos;infrastructure [#examples-infra-pages]

Voici quelques exemples de la manière de créer une condition d&apos;alerte dans le contexte de la page UI de l&apos;infrastructure de monitoring que vous consultez actuellement. Pour créer une condition d&apos;alerte à partir de n&apos;importe quel graphique, sélectionnez les ellipses <Icon name="fe-more-horizontal" /> icône puis <DNT>**Create alert**</DNT>. New Relic sélectionnera automatiquement le <DNT>**Alert type**</DNT> approprié.

## Exemples : options de seuil [#examples-infra-pages]

Utilisez le [seuil](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-threshold) liste déroulante pour le <DNT>**Alert type**</DNT> sélectionné pour définir plus précisément comment vous souhaitez être alerté. Voici quelques exemples des options disponibles.

<CollapserGroup>
  <Collapser id="integrations" title="fournisseurs d'intégration">
    Avec l&apos;intégration infrastructure , vous pouvez [créer une condition d&apos;alerte depuis votre page <DNT>**Integrations**</DNT> ](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#integration). Selon le type de fournisseur sélectionné (CloudFront, DynamoDB, EBS, etc.), les options varieront dans la liste déroulante <DNT>**Define thresholds**</DNT> ; par exemple, octets, erreurs, requests, CPU, connexions, mémoire, enregistrements, latence, etc.
  </Collapser>

  <Collapser id="system-thresholds" title="CPU, disque, charge moyenne, mémoire, swap">
    La liste déroulante <DNT>**System metrics**</DNT> seuil vous permet de sélectionner différents critères pour les métriques CPU, disque, charge moyenne, mémoire et swap.
  </Collapser>

  <Collapser id="byte" title="Taille en octets">
    Le seuil <DNT>**Network metrics**</DNT> offre une flexibilité adaptée aux besoins de votre entreprise. Selon la taille de votre réseau, vous pouvez facilement définir le seuil en octets, Ko, Mo, Go ou To.
  </Collapser>
</CollapserGroup>
