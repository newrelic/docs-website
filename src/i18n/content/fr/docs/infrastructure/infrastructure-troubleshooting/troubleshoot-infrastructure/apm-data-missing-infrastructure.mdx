## Problème

Lorsque les agents APM et infrastructure sont installés sur les mêmes hôtes et utilisent le même New Relic <InlinePopover type="licenseKey" />ou la même clé de licence à partir de comptes de la même organisation, les données APM doivent apparaître dans <DNT>**infrastructure**</DNT>, et vice versa. Si vous ne voyez pas ce lien APM-infrastructure, suivez ces conseils de dépannage.

Veuillez noter que si des clés de licence provenant de comptes distincts au sein d'une organisation sont utilisées, un utilisateur doit avoir accès aux deux comptes pour voir les données liées.

## Solution

Si vous avez terminé l' [intégration APM/Infrastructure](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure) mais que vous ne voyez pas de données APM dans <DNT>**infrastructure**</DNT>, essayez ces procédures de dépannage.

<CollapserGroup>
  <Collapser id="restart-app-server" title="Redémarrez le serveur d'applications.">
    Si vous n'avez pas redémarré votre application APM-monitorer depuis quelques semaines ou mois, il se peut que les flux de données provenant de monitoring d'infrastructure et d'APM ne soient pas liés.

    1. Redémarrez votre serveur d’applications.
    2. Générez du trafic pour votre application.
    3. Attendez quelques minutes, puis vérifiez [les données APM dans monitoring d'infrastructure](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/new-relic-apm-data-infrastructure).
  </Collapser>

  <Collapser id="hostnames" title="Assurez-vous que les noms d'hôte sont les mêmes dans l'UI infrastructure et APM.">
    Si les noms d'hôte sont différents dans APM et dans l'Ui d'infrastructure, nous ne pouvons pas intégrer les données. Une cause courante de ce problème est que les noms d'hôte par défaut sont différents. Par exemple, monitoring d'infrastructure utilise le FQDN d'un hôte (tel que `myhost1.example.com`), tandis que l&amp;apos;APM utilise le nom de l&amp;apos;hôte (tel que `myhost1`).

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; services &amp;gt; (select an app)**</DNT>.
    2. Depuis la page APM <DNT>**Overview**</DNT> de l’application, regardez le nom d’hôte associé à l’application. Comparez ce nom avec le nom du même hôte dans monitoring d&amp;apos;infrastructure.
    3. Si les noms sont différents,[définissez le de l'hôte de l'agent APM `display_name`](/docs/agents/manage-apm-agents/configuration/add-rename-remove-hosts#display_name) pour qu&amp;apos;il corresponde à son nom de domaine complet ou [définissez le `display_name` de l&amp;apos;hôte dans l&amp;apos;UI d&amp;apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#display_name) pour qu&amp;apos;il corresponde à celui défini dans APM.
  </Collapser>

  <Collapser id="gap-fqdn-issue" title="Vérifiez les problèmes de reconnaissance du nom de domaine complet de l'hôte de remplacement.">
    Si l'intégration de l'infrastructure APM fonctionnait auparavant mais s'est arrêtée, le serveur a peut-être été remplacé par un autre serveur possédant le même FQDN. Si les deux serveurs existent simultanément pendant un certain temps, New Relic ne peut pas reconnaître automatiquement le nouveau serveur. Cela rompra la connexion entre l’APM et les données d’infrastructure.

    * Pour résoudre ce problème, obtenez de l'aide sur [support.newrelic.com](https://support.newrelic.com/tickets/new).
    * Pour éviter ce problème, assurez-vous qu'il y a un délai entre la mise hors service d'un ancien serveur et la création d'un nouveau serveur.
  </Collapser>

  <Collapser id="gap-fqdn-issue" title="Assurez-vous que les étapes d’intégration de Kubernetes ont été terminées.">
    Si vous hébergez votre application APM-monitorer sur Kubernetes, certaines étapes d'intégration supplémentaires doivent être effectuées pour lier vos données.

    * Pour vous assurer que vos données sont liées, suivez les étapes fournies pour [lier les applications instrumentées APM à Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes/).
  </Collapser>
</CollapserGroup>
