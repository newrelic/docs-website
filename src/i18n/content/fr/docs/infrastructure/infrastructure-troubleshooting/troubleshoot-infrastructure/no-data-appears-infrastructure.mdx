## Problème

Vous avez installé infrastructure l'agent et attendu quelques minutes, mais aucune donnée n'apparaît dans l'[UI d'infrastructure](/docs/infrastructure/infrastructure-ui-pages/infra-ui-overview).

## Solution

Les données devraient apparaître dans l'UI de monitoring de l'infrastructure en quelques minutes pour les comptes avec des agents précédemment installés.

<Callout variant="important">
  Pour les comptes installant infrastructure l'agent pour la première fois, la latence d'apparition des données dans l'UI de monitoring d'infrastructure peut atteindre plusieurs dizaines de minutes. Si les étapes suivantes vérifient l'installation et qu'aucune condition d'erreur évidente n'apparaît dans le log détaillé, monitorez l'UI d'infrastructure pendant une période plus longue avant de contacter [support.newrelic.com](https://support.newrelic.com) pour obtenir de l&amp;apos;aide.
</Callout>

<Callout variant="important">
  Par défaut, l'agent infrastructure n'envoie pas de données sur les processus du système d'exploitation. Pour activer l’envoi du jeu de données de processus [`enable_process_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#enable-process-metrics) à `true`. Pour affiner les processus que vous souhaitez monitorer, configurez [`include_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#include-matching-metrics) ou [`exclude_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#exclude-matching-metrics).
</Callout>

### Données d'infrastructure manquantes [#missing-infra]

Si aucune donnée n'apparaît dans l'UI, essayez les étapes suivantes pour diagnostiquer le problème :

1. Utilisez votre gestionnaire de paquets pour vérifier que l’agent d’infrastructure est installé :

   <CollapserGroup>
     <Collapser id="apt-install-verify" title="Vérifier l'installation pour apt (Debian ou Ubuntu)">
       1. Utilisez `dpkg` pour vérifier que l’agent est installé :

          ```sh
          dpkg -l | grep newrelic-infra
          ```

       2. Si `dpkg` ne renvoie aucune sortie, consultez [Installer avec apt](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-linux#apt-based).
     </Collapser>

     <Collapser id="yum-install-verify" title="Vérifier l'installation pour yum (Amazon Linux, CentOS ou RHEL)">
       1. Utilisez `rpm` pour vérifier que l’agent est installé :

          ```sh
          rpm -qa | grep newrelic-infra
          ```

       2. Si `rpm` ne renvoie aucune sortie, voir [Installer avec yum](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-linux#yum-based).
     </Collapser>

     <Collapser id="windows-install-verify" title="Vérifier l'installation pour Windows Server">
       1. Utilisez l’invite de commande Windows ou Powershell pour vérifier que le [répertoire de l’agent](/docs/agents/manage-apm-agents/troubleshooting/find-agent-root-directory#infra-agent) existe :

          ```sh
          dir "C:\Program Files\New Relic\newrelic-infra"
          ```

       2. Si vous recevez une erreur `File not found` , consultez [Installer pour Windows Server](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server).
     </Collapser>
   </CollapserGroup>

2. Utilisez [votre système d’initialisation](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status#init-system) pour vérifier que l’agent est en cours d’exécution :

   <CollapserGroup>
     <Collapser id="systemd-verify-status" title="Vérifier le statut avec SystemD">
       Utilisez les commandes SystemD avec CentOS 7, Debian 8, RHEL 7 et Ubuntu 15.04 ou supérieur :

       1. Vérifiez que l'agent est en cours d'exécution :

          ```sh
          sudo systemctl status newrelic-infra
          ```

       2. Si l'agent n'est pas en cours d'exécution, démarrez-le manuellement :

          ```sh
          sudo systemctl start newrelic-infra
          ```
     </Collapser>

     <Collapser id="systemv-verify-status" title="Vérifier le statut avec le système V">
       Utilisez les commandes système V avec Debian 7 :

       1. Vérifiez que l'agent est en cours d'exécution :

          ```sh
          sudo /etc/init.d/newrelic-infra status
          ```

       2. Si l'agent n'est pas en cours d'exécution, démarrez-le manuellement :

          ```sh
          sudo /etc/init.d/newrelic-infra start
          ```
     </Collapser>

     <Collapser id="upstart-verify-status" title="Vérifier le statut avec Upstart">
       Utilisez les commandes Upstart avec Amazon Linux, CentOS 6, RHEL 6 et Ubuntu 14.10 ou version antérieure :

       1. Vérifiez que l'agent est en cours d'exécution :

          ```sh
          sudo initctl status newrelic-infra
          ```

       2. Si l'agent n'est pas en cours d'exécution, démarrez-le manuellement :

          ```sh
          sudo initctl start newrelic-infra
          ```
     </Collapser>

     <Collapser id="windows-verify-status" title="Vérifier l'état avec Windows">
       Utilisez l’invite de commande Windows :

       1. Vérifiez que l'agent est en cours d'exécution :

          ```sh
          sc query "newrelic-infra" | find "RUNNING"
          ```

       2. Si l'agent n'est pas en cours d'exécution, démarrez-le manuellement avec l'invite de commande Windows :

          ```sh
          net start newrelic-infra
          ```

          Si l'exécution de `net start newrelic-infra` renvoie `The service name is invalid`, il se peut que l&amp;apos;agent d&amp;apos;infrastructure n&amp;apos;ait pas été installé correctement et que le service n&amp;apos;ait pas été correctement créé.

          Pour tester ceci :

       3. Depuis Powershell, exécutez la commande `get-service newrelic-infra`, qui renverra l’état du service.

       4. Si une erreur `Cannot find any service with service name newrelic-infra` est renvoyée, suivez les procédures standard pour réinstaller l&amp;apos;agent.
     </Collapser>
   </CollapserGroup>

3. Utilisez [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) pour essayer d’identifier automatiquement le problème.

4. Vérifiez que votre fichier de configuration [`newrelic-infra.yml`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#config-file) contient un paramètre [`license_key`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#conf-license_key) valide.

5. Vérifiez que l’hôte possède un [nom d’hôte unique](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure#hostname) et vérifiez que le nom d’hôte n’est pas `localhost`. Pour plus d&amp;apos;informations, consultez cette [publication du forum d'assistance](https://discuss.newrelic.com/t/relic-solution-a-common-reason-for-a-host-failing-to-show-up-in-the-infrastructure-dashboard/53031).

6. Vérifiez qu'aucun pare-feu ou proxy ne bloque les connexions sortantes du processus de l'agent vers le [domaine d'infrastructure et les ports](/docs/apm/new-relic-apm/getting-started/networks#infrastructure).

7. Confirmez que l'hôte signale correctement même s'il n'apparaît pas dans l'UI de monitoring d'infrastructure en créant une requête de base dans le générateur de requêtes, comme :

   ```sql
   SELECT * FROM SystemSample SINCE 60 minutes ago LIMIT 100
   ```

   Utilisez les résultats de la requête pour noter l'horodatage, qui indique quand les données ont été signalées. Pour déterminer quand les données ont été reçues pour la première fois, regardez l’horodatage le plus ancien.

8. [Générez un log détaillé](/docs/infrastructure/new-relic-infrastructure/troubleshooting/generating-logs-troubleshooting-infrastructure) et examinez le log pour détecter les erreurs.
