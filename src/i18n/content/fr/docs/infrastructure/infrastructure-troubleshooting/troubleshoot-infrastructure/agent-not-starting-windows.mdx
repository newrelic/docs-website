## Problème [#problem]

Vous avez installé l&apos;agent New Relic Infrastructure sur un serveur Windows et aucune donnée ne s&apos;affiche dans l&apos;UI. Lorsque vous essayez de démarrer le service manuellement à partir de <DNT>**Windows Services**</DNT> , cela échoue avec une erreur telle que :

```
Windows could not start the New Relic infrastructure agent service on Local Computer
Error 1067: The process terminated unexpectedly
```

## Solution [#solution]

Un antivirus ou un produit de sécurité peut empêcher l’exécution de l’agent New Relic Infrastructure en tant que service. Pour valider que l’agent se comporte sans problème, exécutez le fichier `newrelic-infra.exe` à partir de la ligne de commande en tant qu’administrateur et confirmez que l’hôte signale les données comme prévu. Si tel est le cas, travaillez avec votre équipe de sécurité pour voir si le service doit être ajouté à votre liste de domaines autorisés ou configuré avec des paramètres supplémentaires.

  Pour en savoir plus sur l&apos;emplacement de l&apos;agent et comment le modifier, consultez [comment configurer le répertoire de l&apos;agent](/docs/infrastructure/install-infrastructure-agent/windows-installation/zip-manual-install-infrastructure-agent-windows/#agent-directory).
</Callout>
