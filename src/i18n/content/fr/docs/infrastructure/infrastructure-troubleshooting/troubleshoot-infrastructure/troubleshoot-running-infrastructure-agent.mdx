Vous pouvez dépanner notre agent d&apos;infrastructure à l&apos;aide de notre utilitaire `newrelic-infra-ctl` . Ce binaire est également inclus dans le package d&amp;apos;installation, dans le même répertoire que le binaire par défaut `newrelic-infra` .

À la réception de la commande `newrelic-infra-ctl` , l&amp;apos;agent :

* Active [le log détaillé](/docs/infrastructure/new-relic-infrastructure/troubleshooting/generate-logs-troubleshooting-infrastructure) pendant une période de cinq minutes, puis rétablit le niveau de log à son paramètre précédent.
* enregistrez toutes les options de configuration de l&apos;agent.
* enregistrez toutes les options de configuration d’intégration.
* Exécute un contrôle de santé pour chaque intégration chargée. (Un contrôle de santé est une exécution immédiate de l&apos;intégration avec un log supplémentaire et une validation de sortie.)

## Linux `newrelic-infra-ctl` [#binary-ctl-linux]

Sous Linux, le binaire de dépannage est `/usr/bin/newrelic-infra-ctl`, disponible à la fois dans le [gestionnaire de paquets](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-linux) ou dans les méthodes d&amp;apos;installation [assistée partarball ](/docs/infrastructure/install-configure-infrastructure/linux-installation/assisted-install-infrastructure-linux).

<Callout variant="important">
  Lors de l&apos;exécution sous Linux, `newrelic-infra-ctl` doit être exécuté soit par l&amp;apos; [utilisateur root, soit par le même utilisateur](/docs/infrastructure/install-configure-infrastructure/linux-installation/linux-agent-running-modes) exécutant le processus `newrelic-infra` .
</Callout>

Le binaire `newrelic-infra-ctl` peut détecter automatiquement le processus d&amp;apos;agent en cours d&amp;apos;exécution sur l&amp;apos;hôte. Il peut également détecter si l’agent s’exécute à l’intérieur d’un conteneur.

## Windows `newrelic-infra-ctl` [#binary-ctl-windows]

Sous Windows, en utilisant le programme d’installation MSI, le binaire de dépannage est `C:\Program Files\New Relic\newrelic-infra\newrelic-infra-ctl.exe`.

<Callout variant="important">
  Sous Windows, le processus agent est toujours détecté automatiquement. Cela ne dépend pas du `pid` ou du `cid`.
</Callout>
