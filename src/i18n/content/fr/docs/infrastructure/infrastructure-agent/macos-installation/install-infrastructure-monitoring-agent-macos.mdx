Avec l&apos;agent infrastructure de New Relic pour macOS, vous pouvez monitorer les indicateurs de performances clés sur les hôtes macOS. L&apos;agent peut s&apos;exécuter sur votre propre matériel ou sur un système cloud tel qu&apos;Amazon Amazon EC2.

Notre agent d&apos;infrastructure [est compatible](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent) avec toutes les versions de macOS généralement disponibles et prises en charge par Apple.

## Installer pour macOS [#install-macos]

Avant l&apos;installation, vous devez vérifier les [exigences](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure). Ensuite, pour installer l’agent d’infrastructure pour macOS, vous pouvez utiliser notre [installation guidée](/docs/full-stack-observability/observe-everything/get-started/new-relic-guided-install-overview/) ou suivre les instructions de ce document pour effectuer une installation de base.

## Limites [#limitations]

Les fonctionnalités suivantes de l’agent d’infrastructure ne sont pas encore disponibles sur macOS :

* [redirecteur de logs](/docs/logs/enable-log-management-new-relic/enable-log-monitoring-new-relic/forward-your-logs-using-infrastructure-agent/)
* [intégration sur hôte](/docs/integrations/host-integrations/get-started/introduction-host-integrations/) (y compris l&apos;intégration intégrée telle que Docker et Flex)
* Déploiement automatisé via des outils de gestion configuration (Chef, Ansible, Puppet)

## Instructions étape par étape [#instructions]

Pour installer l&apos;agent d&apos;infrastructure, suivez les instructions étape par étape :

1. Consultez les [exigences de l&apos;agent et le système d&apos;exploitation pris en charge](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure).

2. Assurez-vous que [Homebrew](https://brew.sh/) est installé sur le système. Vous pouvez vérifier votre installation Homebrew avec :

   ```bash
   which brew
   ```

   S&apos;il n&apos;est pas installé, vous pouvez l&apos;installer avec cette commande (ou consulter les instructions à jour [de Homebrew](https://brew.sh/) ) :

   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

3. Ensuite, ouvrez le terminal et exécutez la commande suivante ::

   ```bash
   brew install newrelic-infra-agent -q
   ```

4. Démarrez le service de l’agent d’infrastructure :

   ```bash
   brew services start newrelic-infra-agent
   ```

5. Créez le fichier de configuration et ajoutez votre <InlinePopover type="licenseKey" />:

   Intel x86 :

   ```bash
   sudo mkdir -p /usr/local/etc/newrelic-infra/
   echo "license_key: YOUR_LICENSE_KEY" | sudo tee -a /usr/local/etc/newrelic-infra/newrelic-infra.yml
   ```

   Silicium Apple :

   ```bash
   sudo mkdir -p /opt/homebrew/etc/newrelic-infra/
   echo "license_key: YOUR_LICENSE_KEY" | sudo tee -a /opt/homebrew/etc/newrelic-infra/newrelic-infra.yml
   ```

Attendez quelques minutes, puis [affichez votre serveur dans l&apos;UI de l&apos;infrastructure](/docs/infrastructure/infrastructure-ui-pages/infra-ui-overview).Si aucune donnée n&apos;apparaît après quelques minutes d&apos;attente, suivez les [étapes de dépannage](/docs/infrastructure/new-relic-infrastructure/troubleshooting/no-data-appears-infrastructure).

## Quelle est la prochaine étape ? [#configure]

La seule [option de configuration](/docs/infrastructure/install-configure-infrastructure/configuration/configure-infrastructure-agent) requise est le paramètre [`license_key`](/docs/Infrastructure-configure-your-agent#conf-license_key) , qui est créé dans le cadre des procédures d&apos;installation. Vous pourriez également vouloir :

* [Ajoutez un attribut personnalisé](/docs/Infrastructure-configure-your-agent#conf-custom_attributes) pour annoter vos données infrastructure .
* [Connectez votre compte AWS](/docs/infrastructure-amazon-aws-ec2-integration#connect) si vos serveurs sont hébergés sur Amazon EC2.
* Ajoutez d’autres [intégrations d’infrastructure New Relic](/docs/infrastructure/introduction-infra-monitoring) pour collecter des données à partir de services externes.

## Mettre à jour l&apos;agent [#update]

Pour effectuer une mise à niveau vers la dernière version, suivez les procédures standard pour [mettre à jour l&apos;agent d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent).
