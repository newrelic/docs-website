Avant [d&apos;installer notre agent infrastructure ](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic), assurez-vous que votre système et toute intégration sur hôte que vous configurez répondent aux exigences. Vous avez également besoin d&amp;apos;un compte New Relic. [Inscrivez-vous gratuitement](https://newrelic.com/signup). Aucune carte de crédit requise.

## Architecture du processeur [#processors]

L&apos;agent infrastructure prend en charge ces architectures de processeur :

* <DNT>**Linux**</DNT>: 64 bits pour les architectures Processeur x86 (nécessite également un gestionnaire de paquets 64 bits et une dépendance)
* <DNT>**Windows**</DNT>: 32 et 64 bits pour les architectures de processeur x86
* <DNT>**ARM**</DNT>: L&amp;apos;architecture arm64 incluant [le processeurAWS Graviton 2](https://aws.amazon.com/ec2/graviton/) est prise en charge sur les systèmes d&amp;apos;exploitation Linux compatibles. intégration sur hôte sont également supportés (à l&amp;apos;exception de l&amp;apos;intégration Oracle ).
* <DNT>**macOS**</DNT>:à la fois x86 64 bits et Apple Silicon.

## Nom d&apos;hôte unique [#hostname]

L&apos;agent infrastructure utilise le nom d&apos;hôte pour identifier de manière unique chaque serveur. Pour éviter des mesures inexactes résultant de la combinaison de plusieurs serveurs sous un seul nom d&apos;hôte, assurez-vous que chaque serveur de monitoring a un nom d&apos;hôte unique.

* Vous pouvez utiliser le paramètre facultatif [`display_name`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#display_name) pour remplacer le nom d&amp;apos;hôte par défaut.
* Les serveurs nommés `localhost` ne sont pas signalés car il s&amp;apos;agit d&amp;apos;un nom par défaut et intrinsèquement non unique.

## Autorisations

L&apos;agent d&apos;infrastructure requiert ces autorisations :

* <DNT>**Linux**</DNT>:Par défaut, [l&apos;agent s&apos;exécute et s&apos;installe en tant que root](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#agent-mode-intro). Vous pouvez également sélectionner des modes d’exécution [privilégiés](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-privileged) ou [non privilégiés](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-agent-linux#install-unprivileged) .
* <DNT>**Windows**</DNT>: L&amp;apos;agent doit être installé à partir d&amp;apos;un compte administrateur et nécessite des privilèges d&amp;apos;administrateur pour s&amp;apos;exécuter.
* <DNT>**macOS**</DNT>: L&amp;apos;agent peut être installé à partir de n&amp;apos;importe quel compte utilisateur.

## Bibliothèques

Pour [les versions d&apos;agent 1.1.19 ou supérieures](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes), vous avez besoin de la bibliothèque `libcap` pour l&amp;apos;installation. Il est disponible dans le référentiel officiel de votre distribution.

## Accès au réseau [#network]

Afin de signaler des données à New Relic, notre agent infrastructure doit avoir un accès sortant à [certains domaines et ports](/docs/apm/new-relic-apm/getting-started/networks#infrastructure). Si votre système a besoin d&amp;apos;un proxy pour se connecter à ces domaines, utilisez le paramètre [`proxy`](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#proxy) .

## Logiciel conteneur [#container]

L&apos;agent infrastructure [instrumente les conteneurs Docker](/docs/infrastructure-docker-instrumentation) une fois installé sur le serveur hôte. Nous prenons en charge les versions Docker 1.12 ou supérieures.

## Utilisation du processeur, de la mémoire et du disque [#resources]

L&apos;agent d&apos;infrastructure est assez léger. Pour une utilisation typique du processeur, de la mémoire et du disque, consultez notre page sur [les frais généraux liés aux performances des agents](/docs/infrastructure/new-relic-infrastructure/getting-started/infrastructure-agent-performance-overhead). Pour plus d&amp;apos;informations sur le système de fichiers pris en charge, voir [Exemple d&apos;attribut de stockage](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#storagesample-attributes).
