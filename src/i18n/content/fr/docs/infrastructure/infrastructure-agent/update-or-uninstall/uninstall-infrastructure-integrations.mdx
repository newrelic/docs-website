[La désinstallation de l&apos;agent infrastructure ](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations)n&amp;apos;affecte pas directement votre [intégrationinfrastructure ](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure): si vous désinstallez l&amp;apos;agent, votre intégration restera. De même, si vous [désactivez ou désinstallez votre intégration](#uninstall-integrations), l’agent infrastructure restera.

Pour désinstaller l&apos;une de vos intégrations, suivez la procédure correspondant au type d&apos;intégration.

## Intégrations sur hôte [#uninstall-on-host]

Si vous avez utilisé le package d&apos;intégration, [consultez les instructions package d&apos;intégration](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations#uninstall-package).

  <Collapser id="uninstall-kubernetes" title="Kubernetes">
    Assurez-vous d&apos;utiliser le contexte approprié sur la machine sur laquelle vous exécuterez Helm et `kubectl`:

    Vous pouvez vérifier les contextes disponibles avec :

    ```sh
    kubectl config get-contexts
    ```

    Et passez au contexte souhaité en utilisant :

    ```sh
    kubectl config use-context CONTEXT_NAME
    ```

    Si vous avez utilisé Helm pour installer l’intégration Kubernetes, désinstallez-la simplement à l’aide de Helm :

    ```sh
    helm uninstall --namespace NAMESPACE_USED_DURING_INSTALLATION RELEASE_NAME
    ```

    Si vous avez installé l’intégration Kubernetes à l’aide d’un manifeste, utilisez le même manifeste pour la désinstaller :

    ```sh
    kubectl delete -f newrelic-infrastructure.yaml
    ```

## S&apos;éloigner du packaged&apos;intégration [#uninstall-package]

Bien qu&apos;il soit toujours possible d&apos;utiliser le package d&apos;intégration, nous vous recommandons de le supprimer complètement et de travailler avec l&apos;intégration sur une base individuelle.

Le dernier package intégration contient les versions suivantes de l&apos;intégration :

* Apache 1.1.2
* Cassandre 2.0.3
* MySQL 1.1.5
* Nginx 1.0.2
* Redis 1.0.1

Si vous supprimez le package d&apos;intégration et souhaitez continuer à utiliser l&apos;intégration associée sur hôte, vous devrez les installer une par une. Pour désinstaller le package et réinstaller votre intégration :

1. Supprimez le package d&apos;intégration en suivant ces instructions. Les fichiers de configuration de l&apos;ancienne intégration ne seront pas supprimés, vous n&apos;aurez donc pas à les configurer à nouveau.
