Vos hôtes sont ajoutés automatiquement lorsque vous installez l&apos;agent infrastructure pour votre système d&apos;exploitation [Linux](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-linux) ou [Windows](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server) , ou [mettez à jour](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent) l&apos;agent. De même, vos hôtes disparaissent automatiquement lorsque vous désinstallez l’agent. Vous n&apos;avez pas besoin d&apos;ajouter ou de supprimer manuellement vos hôtes.

La désinstallation de l&apos;agent n&apos;affecte directement aucune de vos [intégrations](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure). Pour désinstaller une intégration, voir [Désinstaller une intégration](/docs/infrastructure/install-configure-infrastructure/update-or-uninstall/uninstall-integrations).

<Callout variant="important">
  Comme il n&apos;existe aucun script automatisé dans le processus de désinstallation, d&apos;anciens fichiers peuvent subsister lors de la désinstallation. Assurez-vous de supprimer manuellement les fichiers obsolètes.
</Callout>

## Désinstaller l&apos;agent d&apos;infrastructure Linux [#uninstall-linux]

Si vous avez utilisé la procédure d&apos;installation par défaut pour votre agent infrastructure pour les environnements Linux, utilisez vos outils de gestion package pour le désinstaller. Vous n’avez pas besoin d’arrêter le service avant d’exécuter la commande de désinstallation.

<Callout variant="important">
  Vous devez également désinstaller toute monitoring cloud intégration telle que notre Azure Google Cloud Platform intégration ou . Sinon, vous continuerez à voir l&apos;hôte dans votre UI sans aucune donnée.
</Callout>

## Désinstaller le<img style={{ width: '32px', height: '32px', verticalAlign: 'middle'}} class="inline" title="Windows Server" alt="Windows Server icon" src="/images/os_icon_windows.webp" /> [#uninstall-windows]

Si vous avez utilisé la procédure d&apos;installation par défaut pour l&apos;agent d&apos;infrastructure pour les environnements Windows, pour désinstaller :

1. Assurez-vous que vous disposez [des droits d’administrateur](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure#permissions) dans votre groupe d’administrateurs Windows.
2. [Arrêtez l&apos;agent d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status#windows).
3. À partir de Windows <DNT>**Control Panel**</DNT>, utilisez l’outil <DNT>**Add/Remove Programs and Features**</DNT> pour désinstaller l’agent d’infrastructure.
4. À partir de Windows <DNT>**Program Files**</DNT>, supprimez manuellement le dossier New Relic pour supprimer tous les fichiers associés à l’agent d’infrastructure pour Windows.

Si vous avez suivi un processus d’installation [assistée](/docs/infrastructure/install-configure-infrastructure/windows-installation/assisted-install-infrastructure-windows) ou [manuelle](/docs/infrastructure/install-configure-infrastructure/windows-installation/manual-install-infrastructure-windows) , vous devez supprimer manuellement tous les fichiers créés lors du déballage du fichier agent.

## Désinstaller le<img style={{ width: '32px', height: '32px', verticalAlign: 'middle'}} class="inline" title="macOS" alt="macOS icon" src="/images/os_icon_mac.webp" /> [#uninstall-macos]

Si vous avez utilisé la procédure d’installation par défaut pour l’agent d’infrastructure pour les environnements macOS, pour désinstaller :

1. [Arrêtez l&apos;agent d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status#macOS):

   ```sh
   brew services stop newrelic-infra-agent
   ```

2. Depuis le terminal, exécutez la commande de désinstallation :

   ```sh
   brew uninstall newrelic-infra-agent
   ```

## Facultatif : purger les fichiers restants [#leftover-files]

Si vous utilisez des outils de gestion package standard pour la plateforme sélectionnée, le processus de désinstallation laisse généralement des fichiers configuration et d&apos;autres fichiers divers. Si vous devez purger complètement l&apos;un des fichiers restants après la désinstallation de l&apos;agent New Relic Infrastructure, suivez les procédures standard de votre système d&apos;exploitation.
