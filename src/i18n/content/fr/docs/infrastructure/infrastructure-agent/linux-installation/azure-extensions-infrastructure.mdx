Vous pouvez installer notre agent [d&apos;infrastructure](/docs/welcome-new-relic-infrastructure) à l&apos;aide [des extensions Microsoft Azure](https://docs.microsoft.com/en-us/azure/virtual-machines/extensions/overview).

<CollapserGroup>
  <Collapser id="what-is-extension" title="Qu'est-ce qu'une extension Azure ?">
    Les extensions de machine virtuelle Azure (VM) sont de petites applications qui fournissent des tâches configuration et d&apos;automatisation post-déploiement sur les machines virtuelles Azure . Par exemple, si une machine virtuelle nécessite l’installation d’un logiciel, une protection antivirus ou l’exécution d’un script, une extension de machine virtuelle peut être utilisée. Azure CLI, PowerShell, les modèles Azure Resource Manager et le portail Azure peuvent tous exécuter des extensions de machine virtuelle Azure. Les extensions peuvent être regroupées avec un nouveau déploiement de machine virtuelle ou exécutées sur n&apos;importe quel système existant.
  </Collapser>
</CollapserGroup>

New Relic utilise ces extensions pour faciliter l&apos;installation de l&apos;agent d&apos;infrastructure. Cela élimine le besoin de mettre à jour manuellement notre agent et, plus important encore, met automatiquement à l’échelle l’agent avec vos [groupes de mise à l’échelle Azure](https://docs.microsoft.com/en-us/azure/virtual-machine-scale-sets/overview).

## Installer une infrastructure à l’aide des extensions Azure [#install-infra-azure-extensions]

Pour installer l&apos;agent à l&apos;aide de l&apos;extension Azure sur [un système d&apos;exploitation](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/) pris en charge.

1. Accédez à n’importe quelle machine virtuelle Azure, groupe identique Azure ou Azure Service Fabric utilisant des groupes identiques, puis sélectionnez l’onglet <DNT>**Extensions**</DNT> .

   <img title="azure_ext01.png" alt="azure_ext01.png" src="/images/infrastructure_screenshot-crop_azure-extensions-tab.webp" />

2. Sélectionnez l&apos;extension <DNT>**New Relic infrastructure**</DNT> .

   <img title="azure_ext02.png" alt="azure_ext02.png" src="/images/infrastructure_screenshot-crop_azure-nr-infrastructure-extension.webp" />

3. Suivez les instructions et ajoutez votre <InlinePopover type="licenseKey" />.

   <img title="azure_ext03.png" alt="azure_ext03.png" src="/images/infrastructure_screenshot-crop_azure-nr-license-key.webp" />

   <Callout variant="tip">
     Si vous automatisez l&apos;installation, n&apos;oubliez pas de passer votre <InlinePopover type="licenseKey" />avec la variable `NR_LICENSE_KEY` . Vous pouvez utiliser [la configuration via des variables d&apos;environnement](/docs/infrastructure/install-infrastructure-agent/configuration/configure-infrastructure-agent/#precedence) (format `NRIA_`) pour personnaliser n&apos;importe quelle configuration d&apos;agent.
   </Callout>
