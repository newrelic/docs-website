Amazon Web Services (AWS) [Elastic Beanstalk](https://aws.amazon.com/elasticbeanstalk) est un service dynamique qui permet un déploiement et une évolutivité faciles pour vos applications. Suivez ces instructions pour déployer l&amp;apos;agent infrastructure au lancement de l&amp;apos;instance avec vos applications AWS Elastic Beanstalk .

En plus de déployer infrastructure l'agent vous pouvez également intégrer New Relic à AWS et intégrer les Elastic Beanstalk monitoring informations dans New Relic. Si vous ne l'avez pas déjà fait, suivez ces instructions pour [l'intégration d'Amazon avec monitoring d'infrastructure](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure).

## Exigences

Assurez-vous que vous disposez d'un [compte Amazon Web Services](http://aws.amazon.com/elasticbeanstalk/) pris en charge. De plus, pour toute installation de New Relic, vous aurez besoin de votre <InlinePopover type="licenseKey" />. Il s&amp;apos;agit d&amp;apos;une chaîne hexadécimale de 40 caractères que New Relic fournit lorsque vous [créez](http://newrelic.com/signup "Le lien s'ouvre dans une nouvelle fenêtre") votre compte.

## Installer l' agent d'infrastructure [#download]

Pour installer l'agent infrastructure lors du lancement de l'instance avec AWS Elastic Beanstalk:

1. Dans le dossier `.ebextensions` de votre application Elastic BeanStalk, créez un nouveau fichier nommé `newrelic.config`.

2. En fonction du système d’exploitation, ajoutez le contenu suivant au fichier :

   * Remplacez `YOUR_LICENSE_KEY` par votre New Relic <InlinePopover type="licenseKey" />.

   * Remplacez `x86_64` par `aarch64` lors de l&amp;apos;exécution sur ARM (Graviton).

     <CollapserGroup>
       <Collapser id="linux-2" title="Amazon Linux 2">
         ```yml
         files:
           "/etc/newrelic-infra.yml":
             mode: "000644"
             owner: root
             group: root
             content: |
               license_key: YOUR_LICENSE_KEY

         commands:
           # Create the agent’s yum repository
           "01-agent-repository":
             command: sudo curl -o /etc/yum.repos.d/newrelic-infra.repo https://download.newrelic.com/infrastructure_agent/linux/yum/amazonlinux/2/x86_64/newrelic-infra.repo
           #
           # Update your yum cache
           "02-update-yum-cache":
             command: yum -q makecache -y --disablerepo='*' --enablerepo='newrelic-infra'
           #
           # Run the installation script
           "03-run-installation-script":
             command: sudo yum install newrelic-infra -y
         ```
       </Collapser>

       <Collapser id="linux-2023" title="Amazon Linux 2023">

3. Envoyez votre application vers Elastic BeanStalk :

   * En général, utilisez [`eb deploy`](http://docs.aws.amazon.com/elasticbeanstalk/latest/dg/eb3-deploy.html).
   * Si vous utilisez toujours [Eb CLI 2.6](http://docs.aws.amazon.com/elasticbeanstalk/latest/dg/eb-cli.html), utilisez `git aws.push` si nécessaire.
   * Facultatif : utilisez l’ AWS Console UI.

Après une configuration réussie, il peut s'écouler jusqu'à quinze minutes avant que les métriques commencent à apparaître dans New Relic. Consultez les pages d’infrastructure de votre hôte à l’ <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>.

## Configurer l’agent d’infrastructure

Pour configurer l'agent d'infrastructure, adaptez le fichier de configuration de l'agent ou utilisez des variables d'environnement pour déterminer les [paramètres de l'agent](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/) en fonction de nos exigences. Les paramètres peuvent être appliqués à l’aide de l’une des méthodes répertoriées ci-dessous :

* Paramètres dans le fichier : modifiez le fichier `newrelic.config` dans le dossier `.ebextensions` avec des paramètres d&amp;apos;agent supplémentaires. Les paramètres doivent être inclus dans le contenu du fichier `newrelic-infra.yml` .
* Paramètres dans les variables d'environnement : définissez les `NRIA_*` variables d&amp;apos;environnement requises via Elastic Beanstalk UI, l&amp;apos;API ou la CLI . Il est également possible de les définir via `option_settings` dans le fichier `newrelic.config` . Pour en savoir plus, consultez [la documentation AWS Beanstalk](https://docs.aws.amazon.com/elasticbeanstalk/latest/dg/environments-cfg-softwaresettings.html).

## Désinstaller l'agent d'infrastructure

Pour désinstaller l'agent, mettez à jour `newrelic.config` à partir de `.ebextensions` avec la configuration suivante, puis mettez à jour le déploiement.

<CollapserGroup>
  <Collapser id="uninstall-agent-linux-ami" title="Amazon Linux">
    ```yml
    commands:
      # Remove agent configuration file
      "04-agent-remove-config":
        command: sudo rm /etc/newrelic-infra.yml
      #
      # Uninstall infra agent
      "05-agent-uninstall":
        command: sudo yum remove newrelic-infra -y
    ```
  </Collapser>

  <Collapser id="uninstall-infrastructure" title="Windows">
    ```yml
    commands:
      03_uninstall-newrelic-infra:
        command: wmic product where name="New Relic Infrastructure Agent" call uninstall
        ignoreErrors: true
    ```
  </Collapser>
</CollapserGroup>
