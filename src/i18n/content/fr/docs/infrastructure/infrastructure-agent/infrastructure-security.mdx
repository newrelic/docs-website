[L&apos;agent d&apos;infrastructure](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent) de New Relic s&amp;apos;exécute sur vos hôtes et fournit des données complètes, en particulier lorsqu&amp;apos;il est exécuté avec des privilèges d&amp;apos;administrateur. Ce qui suit est un aperçu de la sécurité de notre agent d’infrastructure et quelques recommandations.

## Communication sécurisée des agents [#communication]

Chaque information échangée entre vos hôtes et l&apos; [agent d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent) est délivrée de manière sécurisée. Toutes les communications de l&amp;apos;agent s&amp;apos;effectuent via HTTPS, en utilisant [Transport Layer Security (TLS)](https://en.wikipedia.org/wiki/Transport_Layer_Security). Pour assurer une communication sécurisée, l&amp;apos;agent d&amp;apos;infrastructure a été conçu avec les mesures de protection suivantes :

* Toutes les communications sont établies directement de l&apos;agent vers le service.
* L&apos;agent ne nécessite l&apos;ouverture d&apos;aucun port entrant.
* L&apos;agent est en lecture seule et ne peut pas apporter de modifications à votre système.

L&apos;agent d&apos;infrastructure ne prend pas en charge [le mode haute sécurité](/docs/agents/manage-apm-agents/configuration/high-security-mode).

Pour plus d&apos;informations sur les mesures de sécurité de New Relic, consultez notre [documentation sur la sécurité et la confidentialité](/docs/using-new-relic/new-relic-security/security/security-matters-data-privacy-new-relic) ou visitez le [site Web de sécurité de New Relic](https://newrelic.com/security).

## Modes de fonctionnement [#root]

New Relic s&apos;engage à assurer la sécurité de vos données. Toutes les données dérivées lors de l’exécution de l’ [agent d’infrastructure](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent) sont protégées et utilisées uniquement pour vous fournir des informations relatives à votre infrastructure.

### Linux

Vous pouvez exécuter l’agent d’infrastructure Linux dans trois modes différents :

<CollapserGroup>
  <Collapser id="linux-root" title="Exécuter en tant que root">
    Lorsque l&apos;agent s&apos;exécute en tant qu&apos;utilisateur racine, il dispose d&apos;un accès total à toutes les métriques et à l&apos;inventaire du système.
  </Collapser>

  <Collapser id="linux-privileged" title="Exécuter en tant qu'utilisateur privilégié">
    L&apos;agent exécute un utilisateur non privilégié, nommé `nri-agent`, qui bénéficie de capacités de noyau étendues pendant le processus d&amp;apos;installation. L&amp;apos;utilisateur privilégié `nri-agent` est donc en mesure de collecter certaines métriques et la majeure partie de l&amp;apos;inventaire. Ces autorisations sont en lecture seule\_.\_

    Le script d&apos;installation en mode privilégié effectuera les modifications suivantes dans votre système :

    1. Créez l&apos;utilisateur et le groupe `nri-agent` .

    2. Définissez l&apos;utilisateur et le groupe `nri-agent` comme propriétaires des répertoires suivants :

       * `/var/run/newrelic-infra`
       * `/var/db/newrelic-infra`
       * `/var/log/newrelic-infra`
       * `/etc/newrelic-infra`

    3. Ajoutez les [fonctionnalités de noyau](http://man7.org/linux/man-pages/man7/capabilities.7.html) suivantes à l’exécutable `/usr/bin/newrelic-infra` :

       * `CAP_SYS_PTRACE`, qui permet d&amp;apos;inspecter et de tracer des processus arbitraires
       * `CAP_DAC_READ_SEARCH`, pour contourner les vérifications d&amp;apos;autorisation de lecture et d&amp;apos;exécution de fichiers et de répertoires
  </Collapser>

  <Collapser id="linux-unprivileged" title="Exécuter en tant qu'utilisateur non privilégié">
    L&apos;agent s&apos;exécute avec un utilisateur non privilégié, `nri-agent`, qui est automatiquement créé pendant le processus de mise à niveau/installation, et n&amp;apos;aura pas accès en lecture à toutes les métriques du système. Cela offre une visibilité sur les environnements avec des politiques de sécurité ou réglementaires très strictes. Aucune autorisation ou accès spécial n&amp;apos;est accordé à l&amp;apos;utilisateur dans ce mode d&amp;apos;exécution.

    Le script d&apos;installation en mode non privilégié effectuera les modifications suivantes dans votre système :

    1. Créez l&apos;utilisateur et le groupe `nri-agent` .

    2. Définissez `nri-agent` et le groupe comme propriétaires des fichiers et dossiers système suivants :

       * `/var/run/newrelic-infra`
       * `/var/db/newrelic-infra`
       * `/var/log/newrelic-infra`
       * `/etc/newrelic-infra`
  </Collapser>
</CollapserGroup>

Pour plus de détails sur les différents modes d&apos;exécution, consultez la comparaison dans la [documentation d&apos;installation de l&apos;infrastructure](/docs/infrastructure/install-configure-infrastructure/linux-installation/linux-agent-running-modes).

### Windows

Dans le système Windows , l&apos;agent doit être exécuté avec des autorisations d&apos;administrateur.

New Relic inclut des paramètres facultatifs pour que vous puissiez configurer l&apos;agent pour communiquer via un proxy. Pour définir les paramètres proxy, consultez la [documentationconfiguration ](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#proxy-variables)de monitoring d&amp;apos;infrastructure.

## Connexions d&apos;agents externes [#external-connections]

L&apos;agent envoie périodiquement des données au format JSON décrivant les [métriques de calcul](/docs/infrastructure/infrastructure-ui-pages/infra-hosts-ui-page), les [événements](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page-live-feed-every-config-change) et votre configuration d&amp;apos;[inventaire](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-inventory-page-search-your-entire-infrastructure) au point de terminaison de l&amp;apos;infrastructure chez New Relic.Ces communications sont associées à l&amp;apos;agent utilisant le <InlinePopover type="licenseKey" />généré pour votre compte. Une fois que New Relic reçoit des données configuration monitoring d&amp;apos;un agent externe, il affiche les nouvelles métriques, événements ou données dans l&amp;apos;UI de l&amp;apos;infrastructure.

## Commandes d&apos;agent externe [#command-api]

L&apos;agent gère deux sources de commandes différentes, `newrelic-infra-ctl` et `command-API`:

* Les commandes CLI soumises avec `newrelic-infra-ctl` sont envoyées à l&amp;apos;agent avec la signalisation Linux ou Docker ou avec des canaux nommés Windows.
* À l’aide du point de terminaison `command-API` de la plateforme New Relic, l’agent interroge les commandes fournies par la plateforme toutes les 60 secondes. La connexion est toujours ouverte depuis l&amp;apos;agent vers les points de commande de terminaison d&amp;apos;API de la plateforme New Relic, jamais l&amp;apos;inverse. Le point de terminaison `command-API` est utilisé uniquement pour forcer l&amp;apos;activation ou la désactivation de l&amp;apos;enregistrement d&amp;apos;entités dynamiques. Cela s’applique également à l’intégration Docker fournie avec l’agent d’infrastructure version 1.9.0 ou supérieure.

## Livrables

L&apos;agent infrastructure et toutes les intégrations sur hôte qui s&apos;exécutent dessus sont fournis à l&apos;aide du référentiel et du package standard du système d&apos;exploitation. New Relic signe cryptographiquement tous les packages et les étapes de vérification sont fournies par défaut dans le script d&apos;installation.

Tout le code est vérifié pour détecter les vulnérabilités de dépendance à l&apos;aide d&apos;outils de sécurité standard (<DNT>Snyk</DNT>, <DNT>Dependabot</DNT>, <DNT>Trivy</DNT>).

Le site de téléchargement officiel de New Relic est hébergé sur AWS via S3 et géré par Fastly, notre fournisseur CDN de confiance.
