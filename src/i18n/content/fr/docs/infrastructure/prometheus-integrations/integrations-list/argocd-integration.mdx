[Argo CD](https://argoproj.github.io/) est un contrôleur Kubernetes , chargé de monitoring en continu toutes les applications en cours d&amp;apos;exécution et de comparer leur état en direct à l&amp;apos;état souhaité spécifié dans le référentiel Git.

Utilisez New Relic pour afficher un dashboard basé sur les métriques Prometheus afin de mieux comprendre l' infrastructure Argo CD de votre cluster k8s. Avec New Relic vous pouvez monitorer :

* Demande d'état de santé
* État de synchronisation des applications
* Statistiques de Argo CD Controller
* Statistiques de Argo CD Server
* statistiques du référentiel

## Activer l'intégration

Suivez ces étapes pour activer l'intégration.

1. Suivez la [documentation d'Argo CD](https://argo-cd.readthedocs.io/en/stable/operator-manual/metrics/) pour en savoir plus sur les métriques exposées par Argo CD.

2. Configurer Prometheus monitoring. Les métriques Prometheus doivent être intégrées à New Relic. Vous pouvez utiliser l'agent Prometheus pour Kubernetes ou l'intégration Prometheus Remote Write. Découvrez [comment envoyer des métriques Prometheus](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic/) pour plus de détails.

   <Callout variant="important">
     L' [agent Prometheus ](/docs/infrastructure/prometheus-integrations/install-configure-prometheus-agent/setup-prometheus-agent#scrape-metrics-only-from-prometheus-integrations-scrape-prometheus-integrations)récupère uniquement les métriques par défaut à partir d&amp;apos;un [ensemble d'intégration](/docs/infrastructure/prometheus-integrations/integrations-list/integrations-list-intro).

     Dans ce cas, vous devez identifier votre pod ou point de terminaison avec l'une de ces étiquettes `app.kubernetes.io/name`, `app.newrelic.io/name`, `k8s-app` contenant la chaîne `argocd`.
   </Callout>

3. Utilisez la requête suivante pour confirmer que les métriques sont ingérées comme prévu :

   ```sql
   FROM Metric SELECT count(*) WHERE instrumentation.name = 'remote-write' AND metricName LIKE 'argocd_%' FACET metricName LIMIT MAX
   ```

4. Installez le [démarrage rapide du CD Argo](https://newrelic.com/instant-observability/argocd-quickstart) pour accéder aux <InlinePopover type="dashboards" />et [alertes](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/) intégrés.

   Une fois importés, vous pouvez modifier ou cloner les ressources pour les adapter à vos besoins spécifiques.

   <Callout variant="important">
     Certains graphiques du dashboard incluent des requêtes avec des conditions qui nécessitent l'identification de votre pod ou point de terminaison avec l'une de ces étiquettes `app.kubernetes.io/name`, `app.newrelic.io/name`, `k8s-app` contenant la chaîne `argocd`.
   </Callout>

## Rechercher et utiliser des métriques

Les métriques Prometheus sont stockées sous forme de métriques dimensionnelles. Vous pouvez [effectuer une requête à l'aide de NRQL](/docs/telemetry-data-platform/get-data/apis/query-metric-data-type/) ou utiliser l&amp;apos; [explorateur de données](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer/) pour parcourir les métriques, les facettes et filtrer les dimensions associées disponibles.

Les différents ensembles de métriques exposés par cette intégration sont définis dans la [documentation Argo CD](https://argo-cd.readthedocs.io/en/stable/operator-manual/metrics/).

Utilisez la requête NRQL suivante pour comprendre les métriques ingérées dans New Relic :

* Lister les noms métriques uniques :

  ```sql
  FROM Metric SELECT uniques(metricName) WHERE instrumentation.name = 'remote-write' AND metricName LIKE 'argocd_%' LIMIT MAX
  ```

* Points de données par minute :

  ```sql
  FROM Metric SELECT rate(datapointcount(), 1 minute) WHERE instrumentation.name = 'remote-write' AND metricName LIKE 'argocd_%' LIMIT MAX
  ```

* Estimer l'ingestion de données (ingestion quotidienne, en octets) :

  ```sql
  FROM Metric SELECT bytecountestimate() WHERE instrumentation.name = 'remote-write' AND metricName LIKE 'argocd_%' SINCE 1 day ago
  ```

## Dépannage

* Utilisez cette commande pour vérifier que le point de terminaison Prometheus d'Argo CD émet des métriques sur tout nœud K8s configuré avec Argo CD :

  ```sh
  curl <Argo CD-Pod-IP>:8082/metrics
  ```

* Vous pouvez également consulter les [directives de dépannage](/docs/infrastructure/prometheus-integrations/install-configure-prometheus-agent/troubleshooting-guide/) spécifiques pour l&amp;apos;intégration Prometheus .
