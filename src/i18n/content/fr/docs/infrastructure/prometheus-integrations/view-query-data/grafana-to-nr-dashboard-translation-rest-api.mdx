Vous pouvez utiliser l&apos;API REST de traduction de dashboard Grafana vers New Relic pour convertir vos dashboard Grafana existants en dashboard natifs New Relic. Cette API peut aider à la migration de Grafana vers New Relic pour les clients qui utilisent actuellement le plug-in Grafana pour visualiser leurs données New Relic. Que vous cherchiez à consolider vos outils d&apos;observabilité, à migrer toute votre configuration de monitoring vers New Relic, ou simplement à tester les dashboards New Relic en parallèle de votre configuration Grafana existante, cette API facilite la prise en main.

Cette API utilise la logique de traduction PromQL de New Relic pour traduire vos requêtes. Pour voir une liste des fonctionnalités et fonctions PromQL prises en charge, consultez [Prise en charge des fonctionnalités PromQL](/docs/infrastructure/prometheus-integrations/view-query-data/supported-promql-features).

## Prérequis [#prerequisites]

Avant d&apos;utiliser cette API, vous aurez besoin de :

* Une [clé API utilisateur New Relic](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key)
* Votre dashboard Grafana exporté en JSON (dans Grafana : Paramètres → Modèle JSON → Copier dans le presse-papiers)
* Votre ID de compte New Relic (si vous souhaitez remplacer le compte par défaut)

## Faites une requête aux points de terminaison d&apos;API [#api-endpoints]

Pour traduire un dashboard Grafana en un dashboard New Relic, envoyez une requête `POST` au point de terminaison approprié pour votre région avec le dashboard que vous souhaitez traduire dans le corps de la requête.

* Pour les États-Unis, `https://prometheus-api.newrelic.com/api/v1/translate/dashboard/grafana`
* Pour l&apos;UE, `https://prometheus-api.eu.newrelic.com/api/v1/translate/dashboard/grafana`

### Authentification [#authentication]

Tous les appels d&apos;API nécessitent une [clé API utilisateur](/docs/apis/get-started/intro-apis/types-new-relic-api-keys#user-api-key). Incluez cette clé dans l&apos;en-tête `x-api-key` de votre requête.

### Corps de la requête [#request-body]

Le corps de la requête doit être un objet JSON avec le format suivant :

```
{
  "accountId": int [optional],
  "dashboard": object [required]
}
```

Où `dashboard` est l&apos;objet JSON du dashboard Grafana que vous souhaitez traduire, et `accountId` est le compte à partir duquel vous souhaitez que votre dashboard traduit interroge les données. Si `accountId` n&apos;est pas inclus, le dashboard traduit interrogera les données du compte de l&apos;utilisateur qui effectue la requête.

### Format de réponse [#response-format]

L&apos;API renvoie un objet JSON contenant le dashboard New Relic traduit :

```json
{
  "dashboard": {
    "name": "Translated Dashboard",
    "permissions": "PUBLIC_READ_WRITE",
    "pages": [...]
    }
}
```

## Résoudre les erreurs de traduction [#troubleshooting]

Lorsque le traducteur rencontre des problèmes qui empêchent la traduction complète d&apos;un widget ou d&apos;une requête, il crée des &quot;Action Items&quot; - des widgets Markdown spéciaux qui expliquent ce qui n&apos;a pas fonctionné et suggèrent des correctifs.

## Exemples [#examples]

<CollapserGroup>
  <Collapser id="dashboard-translation-basic" title="Exemple : appel d'API de base pour traduire un dashboard Grafana">
    ```bash
    export NR_USER_KEY="your_user_key"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --slurpfile dashboard "${JSON_FILE}" '{dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-account-override" title="Exemple : traduire le dashboard avec remplacement de l'ID de compte">
    Si vous souhaitez remplacer le compte à partir duquel votre dashboard interrogera les données, vous pouvez inclure `accountId: int` dans la charge de la requête. Cela permettra à toutes les requêtes du dashboard d&apos;interroger les données depuis le compte spécifié.

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)

    curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard'
    ```
  </Collapser>

  <Collapser id="dashboard-translation-and-upload" title="Exemple : importer un dashboard traduit dans New Relic à l'aide de l'API NerdGraph">
    <Callout variant="tip">
      Pour plus d’informations sur l’utilisation de NerdGraph et pour trouver votre point de terminaison régional, consultez [Introduction à New Relic NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).
    </Callout>

    ```bash
    export NR_USER_KEY="your_user_key"
    export ACCOUNT_ID="your_staging_account_id"
    export JSON_FILE="my_grafana_dashboard.json"
    export TRANSLATOR_ENDPOINT="<your-region-endpoint>" # (see API endpoints section above)
    export NERDGRAPH_ENDPOINT="US/EU Regional Nerdgraph endpoint"

    NR_DASHBOARD=$(curl -s ${TRANSLATOR_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "x-api-key:${NR_USER_KEY}" \
    -d "$(jq -c -n --arg accountId "$ACCOUNT_ID" --slurpfile dashboard "$JSON_FILE" '{accountId: $accountId, dashboard: $dashboard[0]}')" \
    | jq '.dashboard')


    curl -X POST ${NERDGRAPH_ENDPOINT} \
    -H "Content-Type: application/json" \
    -H "API-Key: ${NR_USER_KEY}" \
    -d '{
    "query": "mutation($accountId: Int!, $dashboard: DashboardInput!) { dashboardCreate(accountId: $accountId, dashboard: $dashboard) { entityResult { guid name } errors { description } } }",
    "variables": {
    "accountId": '${ACCOUNT_ID}',
    "dashboard": '${NR_DASHBOARD}'
    }
    }'
    ```
  </Collapser>
</CollapserGroup>
