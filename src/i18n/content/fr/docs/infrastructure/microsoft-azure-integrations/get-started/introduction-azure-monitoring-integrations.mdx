Nous vous proposons plusieurs moyens de monitorer vos applications et services [Microsoft Azure](https://azure.microsoft.com) . Voici un aperçu de ces offres et pourquoi vous utiliseriez l’une plutôt que l’autre.

* New Relic dans Azure : avec notre [service Azure Native New Relic](/docs/infrastructure/microsoft-azure-integrations/get-started/azure-native), vous pouvez déployer New Relic basé sur Azure, directement depuis le portail Azure. Cette option vous offre l’équivalent des deux fonctionnalités ci-dessous (ainsi que des Java applications monitoring), et vous pouvez la configurer directement depuis le Azure portail .
* Notre Azure intégration : notre [Azure infrastructure intégration](/docs/infrastructure/microsoft-azure-integrations/get-started/activate-azure-integrations) prend en charge des dizaines de Azure services .
* .NET APM : Vous pouvez installer [notre agent .NET APM](/install/dotnet) pour monitorer vos applications Azure et recevoir des données télémétriques détaillées.
