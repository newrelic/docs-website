Si votre équipe déploie des logiciels et des services cloud via Azure, notre [Azure service Native New Relic](https://azuremarketplace.microsoft.com/en-us/marketplace/apps/newrelicinc1635200720692.newrelic_liftr_payg) vous permet de gérer l&amp;apos;intégration Azure-New Relic, et de gérer et d&amp;apos;installer notre infrastructure agent et nos agents .NET et Java <InlinePopover type="apm" />, le tout directement depuis le Azure portail .

Le service Azure Native New Relic vous permet :

* Intégrez et utilisez facilement New Relic en tant que service nativement intégré basé sur Azure.
* Obtenez une facture unique pour toutes les ressources que vous consommez sur Azure, y compris New Relic.
* Monitorez automatiquement l&apos;activité de l&apos;abonnement Azure et le log des ressources pour New Relic.
* Monitorez automatiquement les métriques Azure à l’aide de New Relic.
* Utilisez une expérience unique pour installer et désinstaller l’agent New Relic sur les machines virtuelles et les services d’application.

En plus de cela, vous pouvez ensuite utiliser New Relic pour d’autres monitoring non liées à Azure dont vous pourriez avoir besoin, en utilisant l’une de [nos 500+ intégrations](https://newrelic.com/instant-observability).

<Callout variant="important">
  Ce service est uniquement disponible pour les clients qui choisissent Azure héberge leurs données New Relic.
</Callout>

## Obtenez le service Azure Native New Relic [#get-it]

Voici une courte vidéo expliquant comment configurer cela :

<Video id="48LHdqOMAsQ" type="youtube" />

Pour des documents détaillés, consultez le [site de documentation Microsoft Azure](https://learn.microsoft.com/en-us/azure/partner-solutions/new-relic/new-relic-overview). Voici un bref résumé de la façon de configurer ce service :

1. Sur Azure Marketplace, abonnez-vous au service Azure Native New Relic.
2. Dans votre portail Azure, créez une ressource New Relic.
3. Sélectionnez <DNT>**Create a New Relic resource**</DNT> pour créer un nouveau compte New Relic, ou vous pouvez <DNT>**Link an existing New Relic resource**</DNT> pour créer un lien vers une ressource existante.
4. Complétez l’ prompt qui s’affiche et sélectionnez <DNT>**Review + create**</DNT>.
5. Configurez la manière dont votre service signale les métriques et connectez-vous à l&apos;onglet <DNT>**Resource configuration**</DNT> .
6. Accédez à <DNT>**App services**</DNT> pour activer nos <InlinePopover type="apm" />agents. Accédez à <DNT>**Virtual machines**</DNT> pour activer notre agent d’infrastructure.
7. Vérifiez que New Relic reçoit des données dans l’onglet <DNT>**Monitored resources**</DNT> du portail Azure. Les ressources avec une coche verte envoient des données à New Relic.

Pour plus de détails sur la configuration, consultez le [site de documentation Microsoft Azure](https://learn.microsoft.com/en-us/azure/partner-solutions/new-relic/new-relic-overview)

## connectez-vous à New Relic [#login-newrelic]

Une fois que vous avez créé votre ressource New Relic et vérifié qu’Azure lui envoie des données, vous pouvez accéder à New Relic.

Pour log à New Relic :

1. Sur la page Azure <DNT>**Overview**</DNT> de la nouvelle ressource, cliquez sur <DNT>**Login to New Relic account**</DNT>.
2. Vous devrez créer un nouveau mot de passe. Pour ce faire, cliquez sur <DNT>**Forgot your password**</DNT>.
3. Entrez votre adresse email. Il doit s’agir du même que celui que vous utilisez pour votre compte Azure.

## Remarque sur le centre de données [#datacenters]

Le service Azure Native New Relic stocke vos données directement sur le centre de données Azure de Microsoft. Actuellement, le service Azure Native stocke toutes les données dans la région USA Est d’Azure.

Le service log patterns de New Relic fonctionne uniquement dans Amazon Web Services. En utilisant les modèles New Relic Logs, vous acceptez que New Relic puisse déplacer vos données et traiter vos données dans Amazon Web Services. Ceci s&apos;applique que vous stockiez vos données dans de New Relic Azure data center ou dans notre Amazon Web Services.data center
