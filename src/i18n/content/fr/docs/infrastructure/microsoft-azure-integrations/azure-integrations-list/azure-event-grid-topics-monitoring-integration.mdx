[L&apos;intégration de New Relic](/docs/infrastructure/introduction-infra-monitoring) inclut une intégration pour rapporter vos métriques [de sujets de grille d&apos;événementsMicrosoft Azure ](https://azure.microsoft.com/en-us/products/event-grid/)et d&apos;autres données à New Relic. Ce document explique comment activer l&apos;intégration et décrit les données rapportées.

## Caractéristiques

événement Grid est un courtier d&apos;événements hautement évolutif et sans serveur que vous pouvez utiliser pour intégrer des applications à l&apos;aide d&apos;événement. Les événements sont livrés par événement Grid aux destinations des abonnés telles que les applications, les services Azure ou tout point de terminaison auquel événement Grid a accès au réseau. Le sujet de la grille événement fournit un point de fin où la source envoie l&apos;événement. Un sujet est utilisé pour une collection d&apos;événements connexes. Pour répondre à certains types d&apos;événements, les abonnés décident à quels sujets s&apos;abonner.

Grâce à New Relic, vous pouvez :

* Afficher les données des rubriques Azure événement Grid dans Azure dashboard prédéfini.
* Exécutez [une requête personnalisée et visualisez les données](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).
* Créez une condition d&apos;alerte pour vous avertir des changements de données.

## Activer l&apos;intégration [#activate]

Suivez la [procédure d&apos;intégration Azure Monitor](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor) standarde pour activer votre Azure service dans le monitoring infrastructure de New Relic.

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#settings).

New Relic interroge vos services Azure Event Grid Topics via l&apos;intégration Azure Monitor selon un intervalle [d&apos;interrogation](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-monitor/#settings) par défaut.

## Rechercher et utiliser des données [#find-and-use]

Pour [explorer vos données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), accédez à <DNT>**[one.newrelic.com/infra](https://one.newrelic.com/infra) &gt; Azure &gt; (select an integration)**</DNT>.
