[L’intégration de New Relic](/docs/infrastructure/introduction-infra-monitoring) inclut une intégration permettant de signaler vos données [de Azure VPN Gateway ](https://docs.microsoft.com/en-us/azure/vpn-gateway/vpn-gateway-about-vpngateways)à nos produits. Ce document explique comment activer cette intégration et décrit les données qui peuvent être signalées.

## Caractéristiques

Notre intégration Azure VPN Gateway collecte les données de bande passante et de perte de paquets à partir de votre service Azure VPN Gateway. Vous pouvez monitorer et alerter sur vos données de Azure VPN Gateway à partir de New Relic, et vous pouvez créer des requêtes personnalisées et des dashboards graphiques personnalisés.

## Activer l&apos;intégration [#activate]

Pour activer l’intégration, suivez les procédures standard pour [activer votre service Azure dans New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informations [d’interrogation](/docs/infrastructure/microsoft-azure-integrations/getting-started/azure-integration-polling) par défaut pour l’intégration de Azure VPN Gateway :

* Intervalle d&apos;interrogation de New Relic : 5 minutes
