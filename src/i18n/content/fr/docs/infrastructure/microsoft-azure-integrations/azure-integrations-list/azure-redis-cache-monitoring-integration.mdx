[monitoring de infrastructureNew Relic](/docs/infrastructure) fournit une intégration pour [Microsoft Azure Redis Cache](https://azure.microsoft.com/en-us/services/cache/) qui signale les données à New Relic. Ce document explique comment activer cette intégration et décrit les données qui peuvent être capturées.

## Caractéristiques

L’intégration de New Relic pour Azure Redis Cache génère des métriques sur votre service Azure Redis Cache, telles que le nombre de clients connectés, le nombre de commandes GET et SET utilisées et les octets d’écriture/lecture par seconde. Il collecte également des données sur l’état et la configuration du service.

Vous pouvez monitorer et alerter sur vos Azure Redis données Cache à partir de notre [UI d&apos;infrastructure](/docs/infrastructure) et vous pouvez [créer des requêtes personnalisées et des dashboards graphiques](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Activer l&apos;intégration [#requirements]

Pour activer cette intégration, suivez les procédures standard pour [activer votre service Azure dans New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informations [d’interrogation](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) par défaut pour l’intégration d’Azure Functions :

* Intervalle d&apos;interrogation par défaut : 5 minutes
* Résolution : 1 point de données par minute

## Rechercher et utiliser des données [#find-and-use]

Pour trouver vos données d’intégration, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Azure**</DNT> et sélectionnez l’intégration Azure Redis Cache.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) à l&apos;aide du [type d&apos;événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureRedisCacheSample` et, lorsque des fragments sont configurés dans le service, du type d&apos;événement `AzureRedisCacheShardSample` .

Pour en savoir plus sur la recherche et l’utilisation des données d’intégration, consultez [Comprendre et utiliser les données](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## données métriques [#metrics]

Pour afficher les métriques rapportées par l&apos;intégration Azure Redis Cache, requêtez le type d&apos;événement `AzureRedisCacheSample` ou `AzureRedisCacheShardSample`. Utilisez les métadonnées associées à chaque métrique pour [filtrer et facetter](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) les données signalées. Pour des informations détaillées sur les métriques, consultez la documentation sur [les métriques prises en charge par Azure ](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftcacheredis). Pour plus de détails, consultez la [documentation Azure Redis Cache](https://docs.microsoft.com/en-us/azure/redis-cache/cache-how-to-monitor#available-metrics-and-reporting-intervals).
