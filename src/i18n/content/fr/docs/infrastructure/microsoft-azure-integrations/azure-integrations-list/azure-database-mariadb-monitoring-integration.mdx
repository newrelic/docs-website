[L&apos;intégration de New Relic](/docs/infrastructure/introduction-infra-monitoring) inclut une intégration pour rapporter votre [base de donnéesMicrosoft Azure pour les métriques MariaDB](https://docs.microsoft.com/en-gb/azure/mariadb/) et d&amp;apos;autres données à New Relic. Ce document explique comment activer l&amp;apos;intégration et décrit les données rapportées.

## Caractéristiques

New Relic rassemble les données de la base de données Azure pour le service MariaDB, qui fournit une base de données en tant que service entièrement gérée et prête pour l&apos;entreprise. MariaDB offre des performances prévisibles et une évolutivité pour les applications utilisant des outils et une plateforme open source.

Grâce à New Relic, vous pouvez :

* Affichez la base de données Azure pour les données MariaDB dans un dashboard prédéfini.
* Exécutez une requête personnalisée et visualisez les données dans l&apos; [UI de New Relic](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).
* Créez une condition d&apos;alerte pour vous avertir des changements de données.

## Activer l&apos;intégration [#activate]

Suivez les procédures standard pour [activer votre service Azure dans New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations), à l’aide du rôle générique <DNT>**Reader**</DNT> .

## configuration et sondage [#polling]

New Relic interroge votre service de bases de données Azure selon un intervalle [d&apos;interrogation](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) par défaut, qui varie en fonction de l&amp;apos;intégration. Pour la base de données Azure pour l&amp;apos;intégration PostgreSQL :

* intervalle d&apos;interrogation : 5 minutes (fréquence d&apos;interrogation maximale recommandée : 1 heure)
* Résolution : 1 point de données par minute

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

## Afficher et utiliser les données [#find-and-use]

Pour [explorer vos données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data): Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Azure &amp;gt; (select an integration)**</DNT>.

Dans New Relic, les données relatives à une seule base de données sont attachées au type d&apos;événement `AzureMariaDbServerSample` , avec une valeur de fournisseur de `AzureMariaDbServer`.
