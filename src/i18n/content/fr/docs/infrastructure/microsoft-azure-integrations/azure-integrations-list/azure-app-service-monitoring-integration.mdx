[monitoring de infrastructureNew Relic](/docs/infrastructure) fournit une intégration pour [Microsoft Azure App Service](https://azure.microsoft.com/en-us/services/app-service/) qui signale les données d&amp;apos; Azure Apps Service à la plateforme New Relic. Ce document explique comment activer cette intégration et décrit les données qui peuvent être capturées.

## Caractéristiques

Un Azure App Service est un service permettant d&apos;héberger et d&apos;exécuter des applications Web, des API REST et un backend mobile sur une plateforme entièrement gérée. L&apos;intégration de New Relic pour Azure App Service génère des données métriques sur vos applications Web, notamment :

* Nombre de requests reçues
* Nombre de réponses et d&apos;erreurs HTTP

Il collecte également des données sur l’état et la configuration du service. Grâce à New Relic, vous pouvez :

* Affichez les données de stockage Azure dans un dashboard infrastructure prédéfini.
* Exécutez une requête personnalisée et visualisez les données dans [l&apos;UI de New Relic](/docs/infrastructure/integrations-getting-started/getting-started/use-integration-data-new-relic-insights).
* Créez une condition d&apos;alerte pour vous informer des modifications apportées aux données de stockage.

## Activer l&apos;intégration [#activate]

Pour activer l’intégration, suivez les procédures standard pour [activer votre service Azure dans New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

New Relic interroge vos services d’application Azure selon un intervalle [d’interrogation](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) par défaut, qui varie en fonction de l’intégration. Fréquence d&amp;apos;interrogation pour l&amp;apos;intégration d&amp;apos;Azure App Services :

* intervalle d&apos;interrogatoire : 5 minutes
* Résolution : 1 point de données par minute

## Rechercher et utiliser des données [#find-and-use]

Pour [explorer vos données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data), accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Azure**</DNT>.

Les [types d&apos;événements](/docs/data-apis/understand-data/new-relic-data-types/#event-data) sont :

* `AzureAppServiceWebAppSample`:Représente chacune des applications Web Azure App Service dans le compte.
* `AzureAppServiceHostNameSample`:Représente chacun des noms de domaine liés à une application Web qui peuvent être utilisés pour y accéder.
* `AzureAppServicePlan:` Représente un ensemble de ressources calculées sur lesquelles une application Web doit s&amp;apos;exécuter.

## données métriques [#metrics]

Pour afficher les métriques rapportées par l&apos;intégration Azure App Service, requêtez les types d&apos;événement `AzureAppServiceWebAppSample` ou `AzureAppServicePlanSample`. Utilisez les métadonnées associées à chaque métrique pour [filtrer et facetter](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) les données signalées. Pour des informations détaillées sur les métriques, consultez la documentation sur [les métriques prises en charge par Azure ](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-excluding-functions).
