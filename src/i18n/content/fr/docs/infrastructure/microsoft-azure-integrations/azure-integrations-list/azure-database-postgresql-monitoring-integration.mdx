[L&apos;intégration de New Relic](/docs/infrastructure/introduction-infra-monitoring) inclut une intégration pour rapporter votre [base de donnéesMicrosoft Azure pour les métriques PostgreSQL](https://docs.microsoft.com/en-gb/azure/postgresql/) et d&apos;autres données à New Relic. Ce document explique comment activer l&apos;intégration et décrit les données rapportées.

## Caractéristiques

New Relic rassemble des données de base de données à partir du service Base de données Azure pour PostgreSQL, qui fournit une base de données communautaire PostgreSQL entièrement gérée et prête pour l&apos;entreprise en tant que service. Le service offre une haute disponibilité, une mise à l&apos;échelle élastique, des sauvegardes automatiques et une protection des données au repos et en mouvement.

Grâce à New Relic, vous pouvez :

* Affichez la base de données Azure pour les données PostgreSQL dans le dashboard New Relic prédéfini.
* Exécutez [une requête personnalisée et visualisez les données](/docs/using-new-relic/data/understand-data/query-new-relic-data).
* Créez une condition d&apos;alerte pour vous avertir des changements de données.

## Activer l&apos;intégration [#activate]

Suivez les procédures standard pour [activer votre service Azure dans New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## configuration et sondage [#polling]

New Relic interroge votre service de bases de données Azure selon un intervalle [d&apos;interrogation](/docs/integrations/microsoft-azure-integrations/getting-started/azure-integration-polling) par défaut, qui varie en fonction de l&apos;intégration. Pour la base de données Azure pour l&apos;intégration PostgreSQL :

* intervalle d&apos;interrogation : 5 minutes (fréquence d&apos;interrogation maximale recommandée : 1 heure)
* Résolution : 1 point de données par minute

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

## Afficher et utiliser les données [#find-and-use]

Pour [explorer vos données d’intégration](/docs/infrastructure/integrations/find-use-infrastructure-integration-data): Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Azure &gt; (select an integration)**</DNT>.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) à l&apos;aide du type d&apos;événement `AzurePostgreSqlServerSample` , avec une valeur de fournisseur de `AzurePostgreSqlServer`.
