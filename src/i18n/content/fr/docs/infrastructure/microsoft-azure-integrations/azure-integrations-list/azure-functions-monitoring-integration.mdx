Notre [monitoring deinfrastructure](/docs/infrastructure) fournit une intégration pour [Microsoft Azure Functions](https://azure.microsoft.com/en-us/services/functions/) qui signale les données de votre Azure Functions service à New Relic. Ce document explique comment activer cette intégration et décrit les données qui peuvent être capturées.

## Caractéristiques

Notre intégration Azure Functions génère des données métriques sur votre service Functions, telles que le nombre d&apos;exécutions de fonctions, les octets envoyés et reçus et le nombre d&apos;erreurs HTTP. Il collecte également des données sur l’état et la configuration du service.

Vous pouvez monitorer et alerter sur vos données Azure Functions avec notre [infrastructure demonitoring ](/docs/infrastructure), et vous pouvez [créer des dashboards de requêtes et de graphiques personnalisés](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Activer l&apos;intégration [#requirements]

Pour activer cette intégration, suivez les procédures standard pour [activer votre service Azure pour monitoring d&apos;infrastructure](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informations [d’interrogation](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) par défaut pour l’intégration d’Azure Functions :

* intervalle d&apos;interrogatoire : 5 minutes
* Résolution : 1 point de données par minute

## Rechercher et utiliser des données [#find-and-use]

Pour trouver vos données d&apos;intégration, accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>, dans la navigation supérieure, cliquez sur <DNT>**Infrastructure**</DNT>, puis cliquez sur <DNT>**Azure**</DNT> et recherchez l&amp;apos;intégration.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) en utilisant le [type d&apos;événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureFunctionsAppSample` .

Pour en savoir plus sur la recherche et l’utilisation des données d’intégration, consultez [Comprendre et utiliser les données](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## données métriques [#metrics]

Pour afficher les métriques rapportées par l&apos;intégration Azure Functions , requêtez le type d&apos;événement `AzureFunctionsAppSample`. Utilisez les métadonnées associées à chaque métrique pour [filtrer et facetter](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) les données signalées. Pour des informations détaillées sur les métriques, consultez la documentation sur [les métriques prises en charge par Azure ](https://docs.microsoft.com/en-us/azure/azure-monitor/platform/metrics-supported#microsoftwebsites-functions).

## Autres données du système [#other-attributes]

L&apos;intégration Azure Functions collecte également ces [attributs](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) sur le service et sa configuration:

* `availabilityState`
* `defaultHostName`
* `enabled`
* `isPremiumApp`
* `javaVersion`
* `netFrameworkVersion`
* `nodeVersion`
* `operatingSystem`
* `phpVersion`
* `pythonVersion`
* `platformArchitecture`
* `state`
* `usageState`
