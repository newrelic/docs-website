[monitoring d&apos;infrastructureNew Relic](/docs/infrastructure) fournit une intégration pour [les machines virtuelles Microsoft Azure (VM)](https://docs.microsoft.com/en-us/azure/virtual-machines/) qui signale les données de votre service de machines virtuelles Azure à New Relic. Ce document explique comment activer cette intégration et décrit les données qui peuvent être capturées.

## Caractéristiques

L’intégration de New Relic pour la machine virtuelle Azure génère des données sur le service de vos machines virtuelles, telles que l’ID de la machine virtuelle, la taille de la machine virtuelle, l’ensemble de disponibilité et le nom de la région. Vous pouvez monitorer et alerter sur les données de vos machines virtuelles Azure à partir de [New Relic](/docs/infrastructure), et vous pouvez [créer des requêtes personnalisées et des dashboards graphiques](/docs/using-new-relic/data/understand-data/query-new-relic-data).

## Exigences

Les exigences comprennent :

* [Agent d&apos;infrastructure New Relic installé](/docs/infrastructure/new-relic-infrastructure/installation) avec [la version 1.0.775 ou supérieure de l&apos;agent d&apos;infrastructure](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) . ([Mettre à jour l&apos;agent d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent).)
* [Intégration de New Relic Azure activée](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations)

## Activer l&apos;intégration [#requirements]

Pour activer cette intégration, suivez les procédures standard pour [activer votre service Azure dans New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations).

<Callout variant="important">
  Vous devez installer l’agent d’infrastructure sur chaque machine virtuelle pour voir les métriques de cet hôte. La connexion de votre abonnement Azure permet à New Relic d’accéder aux métadonnées de la machine virtuelle.
</Callout>

## configuration et sondage [#polling]

Vous pouvez modifier la fréquence d&apos;interrogation et filtrer les données à l&apos;aide [des options de configuration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations).

Informations [d&apos;interrogation](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) par défaut pour l&apos;intégration des machines virtuelles :

* intervalle d&apos;interrogatoire : 5 minutes
* Résolution : 1 point de données par minute

## Rechercher et utiliser des données [#find-and-use]

Pour trouver vos données d’intégration, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Azure**</DNT> et recherchez l’intégration.

Vous pouvez [interroger et explorer vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) en utilisant le [type d&apos;événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data) `AzureVirtualMachineSample` . La valeur `provider` est `AzureVirtualMachine`.

Pour en savoir plus sur la recherche et l’utilisation des données d’intégration, consultez [Comprendre et utiliser les données](/docs/infrastructure/integrations/find-use-infrastructure-integration-data).

## Autres données du système [#other-attributes]

L&apos;intégration de la machine virtuelle Azure collecte également l&apos; [attribut](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) suivant sur le service et sa configuration:

* Région
* Zone de disponibilité
* type d&apos;instance
* ID d&apos;instance

## Dépannage

Si vous utilisez la condition d&apos;alerte <DNT>**Host not reporting**</DNT>, l&apos;importation des métadonnées Azure pour vos machines virtuelles modifiera le registre hôte et entraînera une fausse notification d&apos;alerte. Pour éviter les faux positifs :

1. Pour chaque instance Azure, [désactivez](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#maintain-alerts) sa condition d’alerte <DNT>**Host not reporting**</DNT> .
2. [Mettre à jour l&apos;agent infrastructure ](/docs/infrastructure/new-relic-infrastructure/installation/update-infrastructure-agent)pour toutes les instances qui ont utilisé cette condition d&apos;alerte.
3. [Réactiver](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#maintain-alerts) chaque condition d’alerte <DNT>**Host not reporting**</DNT> .
