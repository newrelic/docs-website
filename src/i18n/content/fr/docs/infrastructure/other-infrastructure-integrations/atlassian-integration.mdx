New Relic propose une intégration avec Atlassian Jira qui synchronise vos informations sur les problèmes et incident New Relic avec Jira. Ces données apparaissent ensuite dans Jira dans la [fonctionnalité d&apos;incident](https://support.atlassian.com/jira-software-cloud/docs/what-is-the-incidents-feature/) et vous permettent de les connecter aux problèmes Jira pour améliorer votre workflow en cas de panne. Les sections couvrent comment mettre en place cette intégration et les bonnes pratiques pour l&amp;apos;utiliser.

<Callout title="Aperçu">
  Nous travaillons toujours sur cette fonctionnalité, mais nous aimerions que vous l&apos;essayiez !

  Cette fonctionnalité est actuellement fournie dans le cadre d&apos;un programme d&apos;aperçu conformément à nos [politiques de pré-sortie](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

## Partage de données [#data-sharing]

Lorsque vous installez l&apos;application New Relic pour Jira à partir d&apos;Atlassian Marketplace, vous permettez à l&apos;intégration d&apos;effectuer ces actions :

* Lisez l&apos;entité, l&apos;incident et les problèmes depuis Jira.
* Écrivez l&apos;entité, l&apos;incident et les problèmes dans Jira.
* Supprimez l&apos;entité, l&apos;incident et les problèmes de New Relic et de Jira pour refléter les rapports de données avec chaque entité.

<Callout variant="caution">
  Cette intégration exposera vos problèmes New Relic et les informations incident à tous les utilisateurs de l&apos;espace de travail Jira associé. L&apos;utilisateur Jira de cet espace de travail n&apos;a peut-être pas accès à New Relic, et même s&apos;il y a accès, il se peut qu&apos;il n&apos;ait pas accès aux mêmes comptes New Relic que votre utilisateur d&apos;API clé.

  Si vous avez des informations de compte New Relic à masquer, utilisez un [utilisateur d&apos;intégration](/docs/more-integrations/best-practices-integration/#integration-user-leverage) qui n&amp;apos;a pas accès à vos comptes sensibles.
</Callout>

## installation [#installation]

Suivez ces étapes pour intégrer New Relic à Jira.

<Steps>
  <Step>
    ## Installer l&apos;application New Relic pour Jira [#install-nr-jira-app]

    Vous devez installer l&apos;application New Relic pour Jira pour synchroniser vos données entre New Relic et Atlassian. Gardez à l’esprit que vous devez disposer de l’autorisation d’administrateur du site Jira pour installer des applications de marketplace. Accédez à [Atlassian Marketplace](https://marketplace.atlassian.com/apps/1233828/new-relic-for-jira?tab=overview&hosting=cloud) et cliquez sur le bouton **Install** .

    <Callout variant="tip">
      Si vous n&apos;avez pas l&apos;autorisation d&apos;administrateur du site, vous pouvez la demander à votre administrateur. Nous vous recommandons de contacter l&apos;administrateur de votre site Jira pour terminer le processus d&apos;installation de l&apos;application.
    </Callout>
  </Step>

  <Step>
    ## Créer une clé API New Relic [#create-new-relic-key]

    Après avoir installé l&apos;application New Relic pour Jira, vous devrez créer une [clé API New Relic](/docs/apis/intro-apis/new-relic-api-keys/). L&amp;apos;administrateur du site Jira et l&amp;apos;administrateur New Relic décideront des comptes exposés à Jira. L&amp;apos;utilisateur disposant du bon accès au compte New Relic peut générer une API de clé New Relic. L&amp;apos;API clé New Relic a accès aux mêmes comptes que l&amp;apos;utilisateur qui les a générés.

    Créez et gérez vos clés API à partir de la page [UI des clés API New Relic](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher) ou connectez-vous à la page [Jetons API Atlassian](https://id.atlassian.com/manage-profile/security/api-tokens) et cliquez sur **Create API token**.
  </Step>

  <Step>
    ## Appliquer une clé API New Relic [#apply-new-relic-key]

    Une fois que vous avez créé une clé API, vous pouvez appliquer cette clé à la première page de l&apos;application de connexion New Relic.

    Collez votre jeton API New Relic dans l&apos;application New Relic Atlassian Marketplace et cliquez sur **Connect**.

    <img width="50%;" title="Apply a New Relic API key" alt="Apply a New Relic API key" src="/images/opentelemetry_screenshot-crop_connect-nr-jira.webp" />
  </Step>

  <Step>
    ## Sélectionnez vos services New Relic [#select-nr-services]

    Une fois l&apos;application New Relic pour Jira connectée via une API clé, vous verrez une liste de toutes les entités associées aux comptes sur cette API clé. Sélectionnez la fonctionnalité d&apos;incident dans la navigation, puis connectez toute entité que vous souhaitez exposer dans Jira.

    Découvrez comment [connecter et supprimer des services dans Jira](https://support.atlassian.com/jira-software-cloud/docs/connect-jira-service-management-services-to-jira-software/).

    <img title="Jira incident management page" alt="Jira incident management page" src="/images/opentelemetry_screenshot-full_jira-incident-mgt.webp" />
  </Step>

  <Step>
    ## Voir les problèmes et incidents de New Relic dans Atlassian Jira [#see-issues-incidents]

    Après avoir sélectionné votre entité New Relic, l&apos;application commence à synchroniser les données de New Relic dans Atlassian Jira. Ouvrez l’onglet incident pour voir une liste des incidents détectés.
  </Step>
</Steps>

## Activer la fonctionnalité incident [#enable-incidents-feature]

Si vous disposez des autorisations d&apos;administrateur de projet, suivez ces étapes pour activer la fonctionnalité d&apos;incident :

1. Depuis la barre latérale de votre projet, accédez à <DNT>**Project settings &gt; Features**</DNT>.

2. Activez ou désactivez la fonctionnalité <DNT>**Incidents**</DNT> .

3. Une fois activée, sélectionnez <DNT>**Incidents**</DNT> dans la barre latérale de votre projet pour utiliser la fonctionnalité.

Consultez la page [des fonctionnalités d&apos;incident](https://support.atlassian.com/jira-software-cloud/docs/what-is-the-incidents-feature/) dans la documentation du support Atlassian pour plus d&amp;apos;informations.

## Supprimer une connexion entre New Relic et Jira [#remove-connection]

Vous pouvez arrêter la synchronisation des données de New Relic et supprimer les données synchronisées à tout moment. Si vous disposez des autorisations d&apos;administrateur de site Jira, vous pouvez le faire à partir de votre page de configuration New Relic dans Jira. Vous pouvez sélectionner votre organisation connectée et cliquer sur <DNT>**Delete**</DNT>. Une fois le processus terminé, vous ne verrez plus les informations sur les incidents et les problèmes New Relic dans la fonctionnalité d&amp;apos;incident dans Jira.

## Se connecter à une autre organisation [#different-organization]

Pour vous connecter à une autre organisation ou modifier l&apos;accès au compte fourni par votre clé API, accédez à la page de configuration de New Relic dans Jira.

Lorsque vous sélectionnez <DNT>**Connect another New Relic organization**</DNT>, vous devez fournir une clé API New Relic. L&amp;apos;ajout d&amp;apos;une nouvelle clé supprimera votre connexion actuelle, y compris toutes les données synchronisées avec Atlassian. Il lancera alors un nouveau processus de synchronisation entre le compte et l’accès à l’organisation de la nouvelle clé API.
