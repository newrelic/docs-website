[New Relic Nozzle pour VMware Tanzu](https://network.pivotal.io/products/nr-firehose-nozzle) collecte les métriques et les événements générés par tous les composants et applications VMware Tanzu qui s&amp;apos;exécutent sur les cellules VMware Tanzu Diego via [Loggregator Firehose](https://docs.pivotal.io/pivotalcf/loggregator/index.html). La buse collecte les données via la passerelle Remote Log Proxy (RLP) et les transmet à [New Relic](https://www.newrelic.com) pour traitement et visualisation.

<img title="newrelicone-dashboards-vmware-tanzu.png" alt="newrelicone-dashboards-vmware-tanzu.png" src="/images/infrastructure_screenshot-crop_vmware-tanzu-dashboard.webp" />

## Présentation [#overview]

Après l&apos;installation, la buse commence à collecter et à pousser l&apos;événement Firehose vers New Relic pour traitement et visualisation. New Relic organise les données en fonction des types d&apos;événements Firehose et affiche chaque type d&apos;événement Firehose dans son propre dashboard distinct.

La buse peut être installée sous forme de tuile dans Ops Manager ou déployée à l&apos;aide de la commande CLI cf push comme application standard. Vous pouvez ensuite monitorer l&apos;état et les performances de votre déploiement VMware Tanzu et définir <InlinePopover type="alerts" />en fonction des métriques collectées à partir de VMware Tanzu Firehose.

## Prérequis [#prereqs]

New Relic Nozzle pour VMware Tanzu a les exigences suivantes :

* Un compte New Relic actif avec une licence Pro ou Pro Trial. Si vous n&apos;avez pas encore de compte New Relic, vous pouvez obtenir une [licence d&apos;essai gratuite de 14 jours](http://newrelic.com/signup?funnel=pivotal-cloud-foundry&partner=Pivotal+Cloud+Foundry).
* New Relic Insights inclus dans la licence
* Versions VMware Tanzu v2.10.x à v10.0.x.

## Fonctionnalité clé [#key-features]

Vous pouvez monitorer les données de santé et de performances des composants VMware Tanzu, notamment :

* Domaine VMware Tanzu
* Déploiement de VMware Tanzu
* Type d&apos;événement de lance à incendie
* Origine
* Emploi
* Adresse IP du composant
* Détails de l&apos;application
* conteneur

Sélectionnez l’élément dans une liste de valeurs pour l’une des mesures ci-dessus et filtrez le dashboard en fonction de la valeur sélectionnée. Vous pouvez également filtrer un dashboard par plusieurs métriques.

## filtrage d&apos;événements [#event-filtering]

Lorsqu&apos;un grand nombre d&apos;événements est diffusé depuis le Firehose, vous souhaiterez peut-être filtrer les événements indésirables générés par le Firehose. Si vous avez besoin que la buse capture l&apos;un des types d&apos;événements Firehose (`ValueMetric`, `CounterEvent`, `ContainerMetric`, `HttpStartStop`, `LogMessage`), vous devez les spécifier sous la forme d&amp;apos;une liste de types d&amp;apos;événements séparés par des virgules dans la propriété `"Selected Events"` , située dans l&amp;apos;onglet Paramètres avancés des paramètres de mosaïque.

À partir de la version 2.X de la buse, les événements `ContainerMetric`, `CounterEvent` et `ValueMetric` sont agrégés. événement inclut les valeurs `min`, `max`, `sum`, `sample count` et `last` de chaque métrique. Cela réduit le nombre d&amp;apos;événements créés par la buse tout en fournissant des détails sur chaque type métrique. Si nécessaire, configurez les filtres LogMessage pour inclure ou exclure des sous-ensembles d&amp;apos;événements LogMessage.

Si nécessaire, configurez `LogMessage Filters` pour inclure ou exclure des sous-ensembles de l&amp;apos;événement LogMessage.

## Exemples de filtres [#filter-examples]

* <DNT>**LogMessage Source Include Filter**</DNT>:Générer uniquement un événement pour les messages de log avec une source répertoriée dans cette liste séparée par des virgules ou des barres verticales.
* <DNT>**LogMessage Source Exclude Filter**</DNT>: Ne pas générer d&amp;apos;événement pour un message de log avec une source répertoriée dans cette liste séparée par des virgules ou des barres verticales.
* <DNT>**LogMessage Message Include Filter**</DNT>: Génère uniquement des événements pour les messages de log contenant du texte répertorié dans cette liste séparée par des virgules ou des barres verticales.
* <DNT>**LogMessage Message Exclude Filter**</DNT>: Ne pas générer d&amp;apos;événement pour les messages de log contenant du texte répertorié dans cette liste séparée par des virgules ou des barres verticales.

Plusieurs filtres LogMessage peuvent être combinés pour limiter la création d&apos;événements. Les filtres d&apos;inclusion sont traités avant les filtres d&apos;exclusion.

</Callout>
