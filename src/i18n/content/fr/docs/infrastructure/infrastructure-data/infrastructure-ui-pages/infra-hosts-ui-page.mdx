Dans notre interface utilisateur de monitoring de l&apos;infrastructure, la page d&apos;interface utilisateur <DNT>**Hosts**</DNT> est l&apos;endroit principal où vous pouvez comprendre et résoudre les problèmes liés à votre infrastructure.

<Callout variant="tip">
  Ces documents décrivent la nouvelle UI <DNT>**Hosts**</DNT> ajoutée en avril 2023. Vous pouvez basculer entre la nouvelle vue et la vue <DNT>**Classic**</DNT> .
</Callout>

## Un exemple de cas d&apos;utilisation [#use-case]

La page <DNT>**Hosts**</DNT> est l&apos;endroit où aller pour enquêter sur les problèmes de votre infrastructure. Voici un exemple de workflow courant que nos clients utilisent pour diagnostiquer un problème :

1. Sur la [page de résumé de l&apos;APM](/docs/apm/agents/manage-apm-agents/agent-data/triage-run-diagnostics), vous remarquez un problème avec votre service Kafka. Le débit a diminué et de nombreuses erreurs sont présentes.
2. Sur la base des données d&apos;infrastructure que vous voyez dans APM, vous pensez qu&apos;il s&apos;agit d&apos;un problème d&apos;hôte, vous cliquez donc sur la page d&apos;infrastructure <DNT>**Hosts**</DNT> .
3. La page <DNT>**Hosts**</DNT> filtre automatiquement votre service Kafka. Vous voyez que la moitié des hôtes exécutant ce service ont atteint le maximum de leur pourcentage de CPU.
4. Si vous suspectez qu&apos;un changement configuration de l&apos;hôte pourrait être la source de votre problème, accédez à l&apos;onglet <DNT>**Inventory**</DNT> et vérifiez la configuration de votre système par hôte.

## Fonctionnalité UI pour explorer et analyser vos hôtes [#ui-features]

Certaines fonctionnalités UI importantes sont disponibles dans l&apos;UI <DNT>**Hosts**</DNT>. Comprendre comment les utiliser vous aidera à tirer le meilleur parti de cette UI.

<img title="Infrastructure UI - Hosts UI options" alt="Infrastructure UI - Hosts UI options" src="/images/infrastructure_screenshot-crop_hosts-ui-options.webp" />

* <DNT>**A.**</DNT> La barre de filtre : filtrez les hôtes que vous souhaitez voir. Pour obtenir des conseils, voir [Barre de filtre](/docs/new-relic-solutions/new-relic-one/core-concepts/search-filter-entities).
* <DNT>**B.**</DNT> Vues enregistrées : lorsque vous créez une vue qui vous plaît (qui peut inclure des filtres et des sélections de graphiques et de tableaux personnalisés), cliquez sur <DNT>**Save**</DNT>.
* <DNT>**C.**</DNT> Options de graphique : vous pouvez basculer vers différentes mesures pour chaque graphique et cliquer sur `...` pour voir [plus d&apos;options de graphique](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/#data-analysis), notamment l&apos;obtention d&apos;un lien partageable et la création d&apos;une alerte pour cette mesure.
* <DNT>**D.**</DNT> Grouper par : utilisez cette option pour regrouper les résultats du graphique par attribut spécifique. Par exemple, vous pouvez afficher les régions AWS avec l’utilisation du processeur la plus élevée, regroupées par `awsRegion`.
* <DNT>**E.**</DNT> filtre d&apos;attribut : cela vous permet de filtrer vos hôtes en fonction de ceux ayant une valeur [d&apos;attribut](/docs/new-relic-solutions/get-started/glossary/#attribute) spécifique. Par exemple,
* <DNT>**F.**</DNT> Table des hôtes : Par défaut, 25 hôtes sont sélectionnés pour l&apos;affichage. Utilisez les colonnes du tableau et le filtre d’attribut pour explorer les hôtes et sélectionner différents hôtes à afficher. Vous pouvez exporter le tableau au format CSV.
* <DNT>**G.**</DNT> Détails de l&apos;hôte : Pour un hôte spécifique dans le tableau des hôtes, cliquez sur le bouton <Icon name="fe-more-horizontal" /> icône pour voir plus d&apos;options pour en savoir plus, y compris des options pour afficher le log, voir une [carte des entités](/docs/new-relic-solutions/new-relic-one/ui-data/automaps), et plus encore.

## Onglets sur la page UI des hôtes [#host-views]

Voici plus de détails sur les onglets disponibles sur la page d&apos;UI <DNT>**Hosts**</DNT> :

<CollapserGroup>
  <Collapser id="summary" title="Onglet Sommaire">
    Sur la page <DNT>**Hosts**</DNT> , l&apos;onglet <DNT>**Summary**</DNT> vous donne un aperçu des performances de vos hôtes.

    <img title="Hosts Summary page" alt="Hosts Summary page" src="/images/infrastructure_screenshot-crop_host-page-tiles.gif" />

    <figcaption>
      <DNT>
        **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure**
      </DNT>
    </figcaption>

    Les tuiles en haut de cette page vous donnent un aperçu de vos hôtes, applications, événements et alertes. Utilisez ces tuiles pour voir plus de détails sur un hôte ou filtrer la page entière <DNT>**Summary**</DNT> sur un hôte spécifique.

    Le sélecteur d&apos;intervalle de temps sur la page <DNT>**Summary**</DNT> ajuste uniquement les tuiles métriques et les graphiques mais n&apos;ajuste **pas** la table hôte. Les autres onglets de l&apos;UI <DNT>**Hosts**</DNT> contiennent des tableaux qui reflètent la plage horaire définie par le sélecteur d&apos;intervalle de temps.
  </Collapser>

  <Collapser id="system" title="UI du système">
    L&apos;onglet <DNT>**System**</DNT> vous donne un aperçu de votre système. Il est quelque peu similaire à l&apos;onglet <DNT>**Summary**</DNT> mais vous permet de rechercher des détails plus approfondis sur certains aspects de votre système.

    <img title="Hosts System page" alt="Hosts System page" src="/images/infrastructure_screenshot-full_host-new-system.webp" />

    <figcaption>
      <DNT>
        **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Hosts &amp;gt; System**
      </DNT>
    </figcaption>

    Voici les graphiques par défaut affichés sur la page <DNT>**System**</DNT> :

    * <DNT>**CPU %**</DNT>: Il s&apos;agit d&apos;une métrique dérivée de plusieurs autres métriques sur l&apos;événement `SystemSample` . Plus précisément, l’attribut [`cpuPercent`](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/default-infrastructure-attributes-events#systemsample-cpu-percent) est une agrégation de `cpuUserPercent`, `cpuSystemPercent`, `cpuIoWaitPercent` et `cpuStealPercent`.
    * <DNT>**Load average five minute**</DNT>: représente une moyenne du nombre de tâches en attente d&apos;effectuer un travail sur un système au cours des 5 dernières minutes.
    * <DNT>**Memory free %**</DNT>: compare la quantité d&apos;octets de mémoire libres à la quantité d&apos;octets de mémoire utilisés.

    Pour plus de détails sur les données utilisées pour générer ces graphiques, voir [SystemSample](attribute-dictionary/?event=SystemSample).
  </Collapser>

  <Collapser id="network" title="UI réseau">
    La page <DNT>**Network**</DNT> offre une visibilité en temps réel sur l&apos;état et les performances des hôtes individuels, des serveurs Web ou d&apos;autres groupes de ressources sur votre réseau. Les graphiques par défaut affichent les mesures de bande passante par paquet, la bande passante par octets et les erreurs par seconde.

    <img title="Hosts Network page" alt="Hosts Network page" src="/images/infrastructure_screenshot-full_host-new-network.webp" />

    <figcaption>
      <DNT>
        **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Hosts &amp;gt; Network**
      </DNT>
    </figcaption>

    Les détails sur les interfaces individuelles peuvent vous aider :

    * Examinez les niveaux de saturation sur votre réseau ou pour des hôtes sélectionnés.
    * Comparez les équilibres de charge entre des ressources spécifiques.
    * Identifier les différences inattendues dans les modèles de capacité entre des hôtes similaires.
    * Évaluez les cinq principales erreurs réseau détectées.

    Ces données réseau en temps réel peuvent ensuite vous aider à déterminer s’il faut résoudre les erreurs en reconfigurant votre réseau, en rééquilibrant les charges ou en prenant d’autres mesures de maintenance préventive avant de devoir prendre une décision d’achat.

    Pour plus de détails sur les données utilisées pour générer ces graphiques, voir [NetworkSample](/attribute-dictionary/?event=NetworkSample).
  </Collapser>

  <Collapser id="processes" title="UI des processus">
    <Callout variant="important">
      Par défaut, l&apos;agent infrastructure n&apos;envoie pas [de données sur les processus du système d&apos;exploitation](/attribute-dictionary/?event=ProcessSample) sauf si vous utilisez l&apos;installation guidée. Pour activer l’envoi du jeu de données de processus [`enable_process_metrics`](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/) à `true`. Pour affiner les processus que vous souhaitez monitorer, configurez [`include_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#include-matching-metrics) ou [`exclude_matching_metrics`](/docs/infrastructure/install-configure-manage-infrastructure/configuration/infrastructure-configuration-settings#exclude-matching-metrics).
    </Callout>

    Utilisez l&apos;onglet <DNT>**Processes**</DNT> pour obtenir des informations sur les processus en cours d&apos;exécution sur vos hôtes et pour définir des alertes sur les métriques des processus. L&apos;onglet <DNT>**Processes**</DNT> affiche des données telles que le pourcentage du processeur, les octets I/O et l&apos;utilisation de la mémoire pour des processus individuels ou des groupes de processus.

    <img title="Hosts Processes page" alt="Hosts Processes page" src="/images/infrastructure_screenshot-full_host-new-processes.webp" />

    <figcaption>
      <DNT>
        **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Hosts &amp;gt; Processes**
      </DNT>
    </figcaption>

    Dans l&apos;onglet <DNT>**Processes**</DNT> , le pourcentage de CPU est limité aux processus individuels plutôt qu&apos;aux hôtes. De ce fait, le pourcentage métrique du CPU ne prend pas en compte les ressources de l&apos;ensemble du système. Au lieu de cela, il montre quelle quantité d&apos;un seul cœur de processeur chaque processus occupe.

    Pour plus de détails sur les données utilisées pour générer ces graphiques, voir [ProcessSample](/attribute-dictionary/?event=ProcessSample).

    Voici un exemple de la manière d&apos;identifier un problème et de définir une alerte :

    1. Vous remarquez sur la page <DNT>**Hosts**</DNT> qu&apos;un cluster a un pourcentage CPU bien supérieur à la normale.
    2. Accédez à la page <DNT>**Processes**</DNT> et filtrez jusqu&apos;aux processus de ce cluster.
    3. Vous remarquez que plusieurs instances du même processus ont un pourcentage CPU excessif.
    4. Après avoir résolu le problème, vous décidez de créer une condition d’alerte qui se déclenche lorsque le pourcentage de processeur pour ce type de processus dépasse un certain seuil.

    <Callout variant="tip">
      Vous ne pouvez pas démarrer, arrêter ou redémarrer des processus à partir de l&apos;UI. Suivez plutôt les procédures standard pour [démarrer, arrêter, vérifier ou redémarrer](/docs/infrastructure/new-relic-infrastructure/configuration/start-stop-restart-check-infrastructure-agent-status) l’agent d’infrastructure manuellement.
    </Callout>
  </Collapser>

  <Collapser id="storage" title="UI de stockage">
    <img title="Hosts Storage page" alt="Hosts Storage page" src="/images/infrastructure_screenshot-full_host-new-storage.webp" />

    <figcaption>
      <DNT>
        **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Hosts &amp;gt; Storage**
      </DNT>
    </figcaption>

    L&apos;onglet <DNT>**Storage**</DNT> vous permet de monitorer la capacité et l&apos;efficacité de vos ressources, y compris l&apos;utilisation globale de vos appareils, l&apos;utilisation du disque ou les opérations I/O . Cela peut vous aider à :

    * Examinez les modèles inattendus ; par exemple, un cluster de machines en miroir qui ne traitent pas les tâches d’ I/O de manière uniforme.
    * Monitorez les niveaux d&apos;utilisation avant que l&apos;espace disque ne soit complètement épuisé.
    * Définissez une condition d&apos;alerte pour vous avertir des problèmes avec un ou plusieurs hôtes ; par exemple, des temps de traitement longs pour les opérations de lecture ou d&apos;écriture, la disponibilité ou l&apos;utilisation du disque en fonction du pourcentage ou des octets, etc.
    * Prenez des décisions éclairées sur l&apos;endroit où redistribuer les hôtes avec requests de traitement supérieures ou inférieures à la normale.
    * Utilisez les données pour vous aider à planifier les ajouts ou les modifications à votre budget d’infrastructure avant qu’une urgence ne survienne.

    Pour plus de détails sur les données utilisées pour générer ces graphiques, voir [StorageSample](/attribute-dictionary/?event=StorageSample).
  </Collapser>

  <Collapser id="containers" title="conteneur">
    Utilisez l&apos;onglet <DNT>**Containers**</DNT> pour obtenir des informations sur le conteneur en cours d&apos;exécution sur vos hôtes et pour définir des alertes sur les métriques du conteneur. Cet onglet affiche des données telles que le pourcentage de CPU, les octets I/O et l&apos;utilisation de la mémoire pour un conteneur individuel ou des groupes de conteneurs. Utilisez ces informations pour :

    * Identifiez les conteneurs qui rencontrent des problèmes de performances.
    * Créez <InlinePopover type="alerts" />pour des conditions telles que le pourcentage de processeur et l&apos;utilisation de la mémoire.

    <img title="Hosts Containers page" alt="Hosts Containers page" src="/images/infrastructure_screenshot-full_host-new-containers.webp" />

        **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Hosts (New) &amp;gt; Containers**
      </DNT>
    </figcaption>
  </Collapser>
</CollapserGroup>
