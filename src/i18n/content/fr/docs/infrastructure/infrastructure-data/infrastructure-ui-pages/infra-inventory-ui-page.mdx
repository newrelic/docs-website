New Relic peut collecter des informations détaillées sur configuration d&apos;un système par hôte, y compris les modules système, les fichiers configuration , les métadonnées, les packages, les services, les sessions utilisateur, etc. La page <DNT>**Inventory**</DNT> fournit une vue en temps réel, filtrable et consultable de la configuration de chaque hôte.

Utilisez la page <DNT>**Inventory**</DNT> pour :

* Assurez-vous qu&apos;une mise à jour de version a été appliquée avec succès sur tous vos hôtes.
* Auditez les divergences de version entre vos hôtes.
* Identifiez rapidement les hôtes qui nécessitent une mise à jour pour corriger une vulnérabilité de sécurité.

## Trouver la page d&apos;inventaire [#find]

Pour afficher et rechercher vos données d’inventaire : Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure &amp;gt; Inventory**</DNT>.

## Quelles données cette page affiche-t-elle ? [#data]

Pour que les données apparaissent sur cette page, [l’agent d’infrastructure](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent) doit être installé.

## Fonctions de page [#page-functions]

Utilisez les fonctions de la page <DNT>**Inventory**</DNT> pour rechercher des informations sur un élément particulier sur vos hôtes :

<CollapserGroup>
  <Collapser id="search" title="Rechercher dans l'inventaire">
    Recherchez un article d&apos;inventaire à l&apos;aide de la fonction de recherche.

    Par exemple, si vous souhaitez trouver des informations liées à OpenSSL, recherchez `openssl`. Le terme de recherche est mis en correspondance avec le nom de l&amp;apos;élément d&amp;apos;inventaire.

    <Callout variant="tip">
      Si vous souhaitez trouver la version Fluent Bit utilisée sur votre système, dans le champ de recherche **Search Inventory** , entrez `fluent-bit`, puis cliquez sur **&gt;** pour développer les résultats de la recherche.
    </Callout>
  </Collapser>

  <Collapser id="view" title="Afficher les détails des articles en stock">
    Les détails de l&apos;élément d&apos;inventaire fournissent des informations sur l&apos;hôte et le système pour chaque hôte sur lequel il réside en fonction du [nom de l&apos;élément d&apos;inventaire](#item-naming) New Relic. Si vous avez différentes versions du même élément sur d&amp;apos;autres hôtes, New Relic le détecte et les signale sur la page Inventaire avec l&amp;apos;étiquette <DNT>**variant hosts**</DNT> et répertorie chaque hôte exécutant chaque version.

    Les détails des éléments sont des attributs (paires valeur-clé) qui sont dictés par leur source. Les attributs spécifiques sont généralement stables dans le temps, mais de nouveaux peuvent être ajoutés et d&apos;autres peuvent être obsolètes. Les attributs portent les métadonnées critiques qui sont au cœur de chaque élément d&apos;inventaire.

    Les attributs courants des éléments d’inventaire incluent :

    * Variante hôtes (nom d&apos;hôte)
    * architecture
    * Description
    * Essentiel
    * Priorité
    * Statut
    * Version
  </Collapser>

  <Collapser id="alerts" title="Voir l'incident du seuil d'alerte de l'hôte">

  </Collapser>

## conservation des données d&apos;inventaire [#data-retention]

Les données d’inventaire sont en temps réel. Si un hôte arrête de signaler, ses données d&apos;inventaire s&apos;affichent toujours jusqu&apos;à 24 heures.

## Ensembles de filtres [#filter-sets]

Le composant UI de l&apos;ensemble de filtres est utilisé pour filtrer vos hôtes et créer des vues enregistrées. <DNT>**It is only available on the Events and Inventory UI pages and is in the process of being deprecated.**</DNT> Elle a été remplacée par la fonctionnalité [de vues enregistrées](/docs/infrastructure/infrastructure-ui-pages/infra-hosts-ui-page#ui-features) . Tous les ensembles de filtres que vous créez à partir de ce point ne seront pas migrés vers la fonctionnalité de vues enregistrées.

<CollapserGroup>
  <Collapser id="filter-set" title="Comment utiliser les ensembles de filtres">
    Avec monitoring d&apos;infrastructure New Relic, vous pouvez combiner des filtres dans un ensemble de filtres pour organiser les hôtes en fonction des critères qui comptent le plus pour vous. Votre compte peut être lié à des centaines d&apos;hôtes. Il est donc essentiel de se concentrer sur ce qui est important pour pouvoir résoudre les problèmes efficacement.

    ### Voir les modèles au sein des catégories [#benefits]

    Vous pouvez créer des ensembles de filtres à l&apos;aide [d&apos;attributs](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute) ou [de balises](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) disponibles.

    Par exemple, vous pouvez organiser votre infrastructure en catégories telles que :

    * Régions
    * versions du système d&apos;exploitation
    * Hôtes associés à Docker conteneur
    * Environnements de test

    Vous pouvez partager des ensembles de filtres avec d&apos;autres personnes de votre compte et identifier rapidement les problèmes d&apos;infrastructure en vérifiant l&apos; [état de santé](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status) codé par couleur de chaque hôte dans l&amp;apos;ensemble de filtres.

    ### Créer des ensembles de filtres [#create]

    L&apos;ensemble de filtres d&apos;infrastructure par défaut est <DNT>**All hosts**</DNT> et il sert de modèle pour vous permettre de créer des ensembles de filtres.

    Pour créer un ensemble de filtres :

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure**</DNT> et cliquez sur <DNT>**Inventory**</DNT> ou <DNT>**Events**</DNT>.
    2. Si <DNT>**All hosts**</DNT> n&amp;apos;est pas affiché dans la barre latérale gauche, ouvrez cet ensemble de filtres en sélectionnant <DNT>**Saved filter sets &gt; All hosts**</DNT>.
    3. Dans <DNT>**All hosts**</DNT>, cliquez sur <DNT>**Filter hosts**</DNT>.
    4. Dans la liste, cliquez sur un élément pour voir une liste de valeurs.
    5. Cliquez sur <DNT>**Include**</DNT> ou <DNT>**Exclude**</DNT> (voir [Logique de jeu de filtres](#logic)).
    6. Cliquez sur les valeurs individuellement ou saisissez du texte pour faire correspondre plusieurs valeurs.
    7. Continuez à ajouter des filtres jusqu’à ce que vous ayez l’ensemble de filtres souhaité.
    8. Pour nommer votre filtre, cliquez sur le <Icon name="fe-edit-2" /> icône, saisissez un nom et cliquez sur <DNT>**Save**</DNT>.

    ### Modifier les ensembles de filtres [#edit]

    Pour modifier un ensemble de filtres existant :

    1. Dans la barre latérale, cliquez sur <DNT>**Saved filter sets**</DNT> pour ouvrir une liste.
    2. Localisez l&apos;ensemble de filtres en faisant défiler ou en saisissant un terme de recherche.
    3. Cliquez sur l&apos;ensemble de filtres pour l&apos;ouvrir.
    4. Dans la barre latérale, cliquez sur une option pour mettre à jour votre ensemble de filtres, puis enregistrez.

    ### Logique de jeu de filtres [#logic]

    Lorsque vous créez un ensemble de filtres, vous générez une liste d&apos;attributs et/ou de balises qui affinent les résultats. Cette section explique comment les ensembles de filtres appliquent diverses règles à la liste.

    #### Inclusion et exclusion [#include-exclude]

    Dans le cadre de la création d&apos;un ensemble de filtres, vous indiquez si un filtre doit inclure ou exclure les entités correspondant à certaines valeurs.

    Le fonctionnement de l&apos;inclusion ou de l&apos;exclusion dépend de la manière dont vous sélectionnez les valeurs :

        </Callout>

    #### Et/Ou

    Les ensembles de filtres utilisent les opérateurs logiques `AND` et `OR` en arrière-plan pour joindre les données.

    * Lorsque vous cliquez sur des valeurs provenant de plusieurs attributs ou balises, elles sont jointes par `AND`.
    * Lorsque vous cliquez sur des valeurs à partir d’un attribut ou d’une balise, elles sont jointes par `OR`.

    Les résultats du filtre affichent les hôtes pour lesquels les deux conditions suivantes sont remplies :

    * Hôtes contenant l’une des versions d’agent d’infrastructure sélectionnées
    * Hôtes dans l&apos;une des zones de disponibilité AWS sélectionnées
