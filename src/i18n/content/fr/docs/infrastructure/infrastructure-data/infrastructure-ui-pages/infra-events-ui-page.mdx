Dans notre interface utilisateur de monitoring de l&apos;infrastructure, la page d&apos;interface utilisateur <DNT>**Events**</DNT> est un flux en direct de l&apos;activité importante du système et de l&apos;hôte, y compris les événements de changement d&apos;inventaire, les changements de configuration et les événements d&apos;analyse des logs. Le flux d&apos;événements vous aide à comprendre les corrélations entre ces événements et les performances du système. Recherchez et filtrez votre événement pour réduire le temps moyen de détection et de réparation des problèmes infrastructure .

<Callout variant="tip">
  Notez que `event` a ici une signification spécifique à l’infrastructure qui est différente de notre [type de données d’événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data).
</Callout>

Recherchez la page <DNT>**Events**</DNT> en allant sur <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Events**</DNT>.

<img title="infra-events-ui.png" alt="infra-events-ui.png" src="/images/infrastructure_screenshot-crop_events.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Events**</DNT>: Utilisez le <DNT>**Events**</DNT> pour afficher les événements importants qui se produisent dans votre système.
</figcaption>

## Chronologie de l&apos;événement [#event-timeline]

Le graphique des infrastructure [événements](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page) fournit un instantané de l&apos; événement se produisant dans la même plage de temps que les métriques affichées. Au survol, vous pouvez voir le nombre d&apos;événements par catégorie codée par couleur pour enquêter sur d&apos;éventuels pics anormaux.

<img title="infra-events-timeline.png" alt="infra-events-timeline.png" src="/images/infrastructure_screenshot-crop_events-timeline.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; Events**</DNT>:La chronologie des événements montre des modèles avec des événements se produisant à la même période pour les métriques affichées.
</figcaption>

En comparant la chronologie aux graphiques de la page d’infrastructure, vous pouvez rapidement identifier les problèmes de votre écosystème. Par exemple, si un pic massif de CPU se produit, vous pouvez cliquer sur la chronologie de l&apos;événement pour cette plage horaire pour trouver l&apos;événement qui l&apos;a provoqué. À partir de là, vous pouvez approfondir le sujet pour découvrir le véritable problème.

## types d&apos;événements [#types]

New Relic collecte une variété d&apos;événements de changement afin que vous puissiez comprendre chaque changement dans votre environnement :

      </td>

      <td>
        Lorsqu&apos;un [événement d&apos;alerte](/docs/new-relic-solutions/get-started/glossary#alert-event) est ouvert ou fermé, New Relic génère un événement indiquant l&apos;hôte et la [condition d&apos;alerte](/docs/infrastructure/new-relic-infrastructure/configuration/infrastructure-alerts-add-edit-or-view-host-alert-information) associée.
      </td>
    </tr>

    <tr>
      <td>
        Connexion de l&apos;agent
      </td>

      <td>
        Lorsqu&apos;un [agent](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#agent) d&apos;infrastructure se connecte à New Relic, notre plateforme génère un événement <DNT>**Agent connected**</DNT> . Si New Relic ne reçoit pas de données d&apos;un agent pendant trois minutes, la plateforme génère un événement <DNT>**Agent disconnected**</DNT> .
      </td>
    </tr>

    <tr>
      <td>
        Variations d&apos;inventaire
      </td>

      <td>
        Ces événements sont générés lorsque des données [d&apos;inventaire](/docs/infrastructure-inventory-page) sont ajoutées, supprimées ou modifiées. Sélectionnez la source <Icon name="fe-filter" /> icône pour comprendre quelle catégorie correspond au chemin d&apos;inventaire modifié. Pour plus de détails, sélectionnez un événement d&apos;inventaire pour voir une comparaison côte à côte de l&apos;ancien et du nouvel état.

        L&apos;événement d&apos;inventaire peut inclure :

        * <DNT>**Kernel**</DNT> (comprend les modules et la configuration) : peut être ajouté, modifié ou supprimé.
        * <DNT>**Metadata**</DNT> (comprend diverses informations supplémentaires sur les hôtes) : peut être ajouté, modifié ou supprimé.
        * <DNT>**Packages**</DNT>:Peut être installé, supprimé ou modifié.
        * <DNT>**Services**</DNT>:Peut être démarré, arrêté ou redémarré.
        * <DNT>**Sessions**</DNT> (inclut utilisateur) : Peut être connecté ou déconnecté.
      </td>
    </tr>
  </tbody>
</table>
