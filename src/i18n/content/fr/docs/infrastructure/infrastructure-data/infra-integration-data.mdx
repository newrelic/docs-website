Grâce à notre intégration infrastructure , vous pouvez monitorer les performances de nombreux services populaires. Voici quelques conseils sur la façon de trouver, de comprendre et d’utiliser les données rapportées par l’intégration infrastructure .

## Explorez les données d'intégration de votre infrastructure [#explore]

La meilleure façon de comprendre les données d’intégration infrastructure et de voir ce que vous pouvez en faire est d’activer une intégration et [d’explorer les données](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) dans l’ UI de New Relic.

Quelques recommandations pour explorer :

* Afficher le dashboard :

  * Pour les données AWS, Azure et GCP :

    1. Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT>

    2. Recherchez votre intégration (Amazon Web Services, Azure, Google Cloud Platform) et cliquez sur la vignette.

    3. Accédez à un type d’entité cloud spécifique et cliquez sur <DNT>**Dashboards**</DNT>.

       <img title="Dashboards example" alt="Screenshot of the AWS integration page" src="/images/infrastructure_screenshot-crop_dashboards-example.webp" width="50%" />

  * Pour tous les autres dashboards d’intégration, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure**</DNT> et cliquez sur votre intégration.

* Requête de données : vous pouvez exécuter des requêtes et des graphiques personnalisés de vos données d'intégration. Pour plus d'informations, consultez [Données de requête](/docs/using-new-relic/data/understand-data/query-new-relic-data).

* Créer une condition d'alerte : Voir [condition d'alerte](#alerts).

* En savoir plus sur les métriques et les données d’inventaire rapportées par une intégration. Pour en savoir plus, consultez la documentation d'une intégration spécifique.

## Exemple de requête NRQL [#nrql]

Découvrez comment créer une requête NRQL dans notre [documentation d'introduction à NRQL ](/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql).

Voici quelques exemples de [requêtes NRQL](/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql) qui utilisent des données d&amp;apos;intégration :

<CollapserGroup id="query-examples">
  <Collapser id="aws-ebs-example" title="Exemple de requête AWS EBS">
    Voici une requête NRQL pour le service AWS EBS, affichant la métrique du temps d'écriture total, facettée par `entityName`:

    ```sql
    SELECT sum('provider.volumeTotalWriteTime.Sum') FROM BlockDeviceSample WHERE provider = 'EbsVolume' FACET entityName
    ```
  </Collapser>

  <Collapser id="azure-service-example" title="Exemple de requête Azure Service Bus">
    Voici une requête NRQL pour le nombre maximal de messages dans une file d'attente de rubriques Azure Service Bus, à facettes par groupe de ressources :

    ```sql
    SELECT max(activeMessages.Maximum) FROM AzureServiceBusTopicSample FACET resourceGroupName
    ```
  </Collapser>

  <Collapser id="azure-functions-example" title="Exemple de requête Azure Functions">
    Voici une requête NRQL pour Azure Functions, affichant le nombre de fonctions exécutées au cours des six dernières heures par région au fil du temps :

    ```sql
    SELECT sum(functionExecutionCount.Total) FROM AzureFunctionsAppSample FACET regionName TIMESERIES SINCE 6 hours ago
    ```
  </Collapser>

  <Collapser id="azure-vms-example" title="Exemple de requête sur les machines virtuelles Azure">
    Voici une requête NRQL pour les machines virtuelles Azure qui compare le nombre d'événements de machines virtuelles au cours des trente dernières minutes avec la même heure il y a une semaine :

    ```sql
    SELECT uniqueCount(vMName) FROM AzureVirtualMachineScaleSetSample FACET name SINCE 30 minutes ago COMPARE WITH 1 week ago
    ```
  </Collapser>

  <Collapser id="nginx-example" title="Exemple de requête NGINX">
    Voici un exemple de requête que vous pouvez exécuter sur vos données d'intégration NGINX et placer dans un dashboard. Cette requête crée un graphique montrant la valeur moyenne des requests NGINX par seconde au fil du temps :

    ```sql
    SELECT average(net.requestsPerSecond) FROM NginxSample TIMESERIES
    ```

    Pour en savoir plus sur la création d'une requête, consultez [la référence NRQL ](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/).
  </Collapser>

  <Collapser id="mysql-example" title="Exemple de requête MySQL">
    Voici un exemple de requête que vous pouvez exécuter sur vos données d'intégration MySQL. Cette requête génère un graphique indiquant le nombre maximal de connexions MySQL utilisées :

    ```sql
    SELECT max(net.maxUsedConnections) FROM MysqlSample
    ```

    Pour en savoir plus sur la création d'une requête, consultez [la référence NRQL ](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/).
  </Collapser>

  <Collapser id="inventory-change-example" title="Exemple de requête sur les variations de stock">
    Voici un exemple de requête qui regroupe les événements de changement d'inventaire du dernier jour par type de changement :

    ```sql
    SELECT count(*) FROM InfrastructureEvent WHERE format='inventoryChange' FACET changeType SINCE 1 DAY AGO
    ```
  </Collapser>
</CollapserGroup>

<Callout variant="tip">
  Vous pouvez également effectuer ces requêtes [en utilisant des métriques dimensionnelles](/docs/query-your-data/nrql-new-relic-query-language/nrql-query-tutorials/query-infrastructure-dimensional-metrics-nrql).
</Callout>

## Conseils pour utiliser différents types de données [#data-types]

l'intégration peut générer des données métriques, événementielles et d'inventaire, qui sont toutes disponibles pour l'interrogation. Voici quelques conseils pour utiliser les différents types de données d'intégration. Pour en savoir plus sur l'utilisation des requêtes NRQL , consultez [Introduction à NRQL](/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql).

<CollapserGroup>
  <Collapser id="metric-tips" title="Conseils sur les données métriques">
    Conseils pour trouver et utiliser des données d'intégration métrique :

    * Toutes les données d'intégration sont attachées à [un type de données appelé événement](/docs/data-apis/understand-data/new-relic-data-types/#event-data) (à ne pas confondre avec [l'événement signalé par intégration](/docs/integrations/new-relic-integrations/getting-started/introduction-infrastructure-integrations#events), qui représente une activité importante dans votre hôte/service). Cela signifie que toutes les données d&amp;apos;intégration peuvent être trouvées via [métriques et événement](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer). Pour en savoir plus sur ces deux types de données New Relic de base, consultez [Collecte de données New Relic](/docs/data-analysis/metrics/analyze-your-metrics/data-collection-metric-timeslice-event-data).

    * les valeurs métriques sont traitées comme <DNT>**attributes**</DNT>: paires valeur clé attachées à un [événement](/docs/data-analysis/metrics/analyze-your-metrics/data-collection-metric-timeslice-event-data#event-data). Par exemple, l&amp;apos;intégration MySQL possède une métrique « connexions actives » ; celle-ci serait trouvée en interrogeant l&amp;apos;attribut `connectionsActive` de l&amp;apos;événement `MysqlSample`.

      Pour des informations générales sur les métriques, voir [intégration métrique data](#metric).
  </Collapser>

  <Collapser id="event-tips" title="Conseils sur les données d'événements">
    Voici quelques conseils pour rechercher et utiliser les données d'événements d'intégration lors d'une requête :

    * La plupart des événements d'intégration sont des événements de changement d'inventaire. Lorsque l'inventaire est modifié, il génère un `InfrastructureEvent` avec une valeur `format` de `inventoryChange`.

    * les données d'intégration peuvent être trouvées via [métriques et événement](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer).

      Pour des informations générales sur l'événement, voir [données d'événement](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#event-data).
  </Collapser>

  <Collapser id="event-tips" title="conseils sur les données d'inventaire">
    Lorsque les données d'inventaire changent, un type d'événement `InfrastructureEvent` est généré avec une valeur `format` de `inventoryChange`. Consultez les [exemples de requête](#inventory-change-example) pour un exemple d’interrogation de ces données.

    Pour des informations générales sur les données d'inventaire, voir [intégration données d'inventaire](/docs/integrations/new-relic-integrations/getting-started/understand-use-data-infrastructure-integrations).
  </Collapser>
</CollapserGroup>
