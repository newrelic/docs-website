Si vous utilisez l&apos;agent infrastructure ou [l&apos;intégration sur hôte](/docs/integrations/host-integrations/get-started/introduction-host-integrations) qui rapporte les données via l&amp;apos;agent infrastructure , il existe plusieurs façons de configurer le reporting des données. Voici deux options courantes pour gérer le reporting des données :

* [Activer/désactiver les métriques de processus](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#enable-process-metrics)
* [Sélectionnez un attribut spécifique à signaler](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings#include-matching-metrics)

Pour d&apos;autres options configuration de l&apos;agent, voir [configuration](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings).

Pour notre intégration infrastructure , vous pouvez également modifier la fréquence de reporting des données :

* Pour [l&apos;intégration sur hôte](/docs/infrastructure/host-integrations/host-integrations-list): utilisez le paramètre [configuration](/docs/integrations/integrations-sdk/file-specifications/host-integration-configuration-overview) `interval` d&amp;apos;une intégration spécifique.
* Pour l&apos;intégration cloud (AWS, Azure, Google Cloud) : modifiez la [fréquence d&apos;interrogation](/docs/integrations/new-relic-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations).

Pour en savoir plus sur la gestion des données New Relic en général, consultez [Gérer les données](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-your-data).
