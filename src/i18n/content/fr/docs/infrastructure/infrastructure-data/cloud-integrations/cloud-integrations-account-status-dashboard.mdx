Chaque cloud compte que vous liez à de New Relic monitoring infrastructure (par exemple, [AWS](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) ou [Azure](/docs/integrations/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations)) dispose d&amp;apos;un <DNT>**account status dashboard**</DNT> qui indique comment notre plateforme gère les données de ce cloud fournisseur de services .

## Pourquoi c'est important [#reasons-to-use]

Les raisons de monitorer ce dashboard incluent :

* <DNT>**Understand API call usage.**</DNT> Nous utilisons des API cloud pour collecter des métriques et des données d&amp;apos;inventaire. Le dashboard affiche les nombres et les fréquences de ces appels d&amp;apos;API. Vous souhaiterez peut-être les monitorer pour deux raisons : l&amp;apos;appel d&amp;apos;API peut coûter de l&amp;apos;argent et votre utilisation de l&amp;apos;API peut être limitée à certains niveaux d&amp;apos;utilisation. Pour réduire l&amp;apos;utilisation, vous pouvez [configurer les paramètres d'intégration](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) pour réduire le nombre de composants que vous monitorez ou la fréquence à laquelle ils sont monitorés.

* <DNT>**Troubleshoot missing data or other data issues.**</DNT> Le dashboard affiche des informations qui affectent la manière dont les données du rapport d&amp;apos;intégration sont générées, notamment :

  * Erreurs qui affectent la collecte de données par New Relic, comme les erreurs d'autorisation ou les erreurs d'épuisement de quota.
  * Modifications de configuration de l'intégration, telles que les changements d'intervalle d'interrogation ou la désactivation ou l'activation de la collecte tag .

## Comprendre les données dashboard [#understand-data]

Les données spécifiques affichées sur le dashboard état du compte diffèrent selon le fournisseur de services cloud . Les graphiques courants incluent :

* <DNT>**Data updates**</DNT>: Affiche les mises à jour des données métriques ou des données d&amp;apos;inventaire (mises à jour affichées sous la forme d&amp;apos;une valeur `1` ).
* <DNT>**Account changes**</DNT>:Actions affectant le fonctionnement de l&amp;apos;intégration. Par exemple : renommer ou dissocier un compte cloud , modifier l&amp;apos;intervalle d&amp;apos;interrogation et d&amp;apos;autres options configuration .
* <DNT>**Data freshness**</DNT>:horodatage du dernier point de données collecté pour chaque intégration.
* <DNT>**Fetching errors**</DNT>:Ces éléments indiquent des problèmes liés à la collecte de données. (Cela peut être un problème du côté du fournisseur de cloud.)

Les options pour une meilleure compréhension des données graphiques incluent :

* Passez la souris sur un graphique <Icon name="fe-help-circle" /> icône pour voir une description du graphique (si disponible).
* [Afficher la requête NRQL sous-jacente du graphique](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform).

## Trouver dashboard de l'état du compte [#find-dashboard]

Pour trouver le dashboard de statut du compte d'un fournisseur de services cloud :

1. À partir de <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure**</DNT>, sélectionnez un fournisseur de services cloud (par exemple, <DNT>**AWS**</DNT>).
2. Sélectionnez <DNT>**Account status dashboard**</DNT> pour le compte cloud.
