Notre [cloudd&apos;intégration](/docs/integrations/new-relic-integrations/getting-started/introduction-infrastructure-integrations#integration-types) récupère des données à partir des API des fournisseurs cloud . Dans New Relic, vous pouvez modifier certains paramètres liés à la collecte de données pour votre cloud d&amp;apos;intégration. Lisez la suite pour découvrir quels changements vous pouvez apporter et les raisons pour lesquelles vous devez les apporter.

## Aperçu des paramètres [#features]

L&apos;intégration cloud de New Relic récupère des données à partir des API des fournisseurs cloud . Les données sont généralement collectées à partir d&apos;API monitoring telles que AWS CloudWatch, Azure Monitor et GCP Stackdriver, et les métadonnées d&apos;inventaire sont collectées à partir des API des services spécifiques.

Vous pouvez utiliser le [dashboard d&apos;état du compte](/docs/integrations/new-relic-integrations/getting-started/cloud-integrations-account-status-dashboard) pour voir comment votre cloud d&amp;apos;intégration gère les données d&amp;apos;un fournisseur de services cloud . Si vous souhaitez signaler plus ou moins de données à partir de votre cloud d&amp;apos;intégration, ou si vous devez contrôler l&amp;apos;utilisation des API des fournisseurs cloud pour éviter d&amp;apos;atteindre les limites de débit et de limitation de votre compte cloud , vous pouvez modifier les paramètres configuration pour modifier la quantité de données qu&amp;apos;ils signalent. Les deux principaux contrôles sont :

* [Modifier la fréquence de sondage](#polling)
* [Modifier les données rapportées](#filter-data)

Voici quelques exemples de raisons commerciales pour lesquelles vous souhaitez modifier la fréquence de vos sondages :

* <DNT>**Billing:**</DNT> Si vous devez gérer votre facture AWS CloudWatch, vous souhaiterez peut-être diminuer la fréquence d&amp;apos;interrogation. Avant de procéder, assurez-vous que les [conditions](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions) [d’alerte](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions) définies pour votre cloud d’intégration ne sont pas affectées par cette réduction.
* <DNT>**New services:**</DNT> Si vous déployez un nouveau service ou une configuration et que vous souhaitez collecter des données plus souvent, vous souhaiterez peut-être augmenter temporairement la fréquence d&amp;apos;interrogation.

<Callout variant="caution">
  La modification des paramètres configuration de votre intégration peut [avoir un impact sur l&apos;état d&apos;alerte et les tendances des graphiques](#impact).
</Callout>

## Modifier la fréquence de sondage [#polling]

La configuration [de la fréquence d&apos;interrogation](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) détermine la fréquence à laquelle New Relic signale les données de votre fournisseur de cloud pour chaque service. Par défaut, la fréquence d&amp;apos;interrogation est définie sur la fréquence maximale disponible pour chaque service.

Pour modifier la fréquence d’interrogation d’une intégration cloud :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure**</DNT>.
2. Sélectionnez l&apos;onglet qui correspond à votre fournisseur de services cloud .
3. Sélectionnez <DNT>**Configure**</DNT> à côté de l’intégration.
4. Utilisez la liste déroulante à côté de <DNT>**Data polling interval every**</DNT> pour sélectionner la fréquence à laquelle vous souhaitez que New Relic capture vos données cloud d&amp;apos;intégration.

## Spécifier les données à récupérer [#filter-data]

Vous pouvez spécifier les informations que vous souhaitez capturer pour votre intégration cloud en activant la collecte de données supplémentaires et en appliquant plusieurs filtres à chaque intégration.

Pour modifier ces paramètres pour votre intégration cloud :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Infrastructure**</DNT>.
2. Sélectionnez l&apos;onglet qui correspond à votre fournisseur de services cloud .
3. Sélectionnez <DNT>**Configure**</DNT> à côté de l’intégration.
4. Sous <DNT>**Data collections and filters**</DNT>, tournez les boutons souhaités <DNT>**On**</DNT>.
5. Pour [les filtres](#filters), sélectionnez ou entrez les valeurs que vous souhaitez inclure dans vos données rapportées.
