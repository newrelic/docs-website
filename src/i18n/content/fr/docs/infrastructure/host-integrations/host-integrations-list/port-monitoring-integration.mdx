Notre intégration monitoring des ports monitore l&apos;état des ports réseau, tels que TCP, UDP, etc., et signale les temps de disponibilité et les données de panne à New Relic.

<img title="port-dashboard" alt="Dashboard of the port monitoring integration" src="/images/infrastructure_screenshot-crop_port-monitoring-dashboard.webp" />

<Callout variant="tip">
  Cette intégration est sortie en [open source sur Github](https://github.com/newrelic/nri-port-monitor). Un log des modifications y est également disponible pour les dernières mises à jour.
</Callout>

## Installer l&apos;intégration [#install]

Suivez les étapes ci-dessous pour installer l&apos;intégration :

<Steps>
  <Step>
    ## Installer l&apos; agent d&apos;infrastructure [#infra-install]

    Pour utiliser l&apos;intégration de monitoring des ports, vous devez d&apos;abord [installer l&apos;agent infrastructure ](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent-new-relic/)sur le même hôte. Toutes nos intégrations sur hôte nécessitent l&apos;agent infrastructure , qui permet d&apos;exposer et de rapporter les métriques à New Relic.
  </Step>

  <Step>
    ## Téléchargez et préparez l&apos;intégration [#download]

    1. Choisissez et téléchargez une version pour votre plateforme depuis [le dépôt GitHub](https://github.com/newrelic/nri-port-monitor/releases). Par exemple :

       ```shell
       wget https://github.com/newrelic/nri-port-monitor/releases/download/1.4/nri-port-monitor_x86_64.tar.gz && tar -zxvf nri-port-monitor_x86_64.tar.gz  
       ```

    2. Copiez l&apos;exécutable `nri-port-monitor` vers `/var/db/newrelic-infra/newrelic-integrations`. Copier `port-monitor-config.yml` vers `/etc/newrelic-infra/integrations.d`

       ```shell
       cp nri-port-monitor /var/db/newrelic-infra/newrelic-integrations && cp port-monitor-config.yml /etc/newrelic-infra/integrations.d
       ```

    3. Définissez les autorisations d’exécution pour le fichier binaire `nri-port-monitor` afin qu’il puisse s’exécuter correctement :

       ```shell
       chmod +x /var/db/newrelic-infra/newrelic-integrations/nri-port-monitor
       ```
  </Step>

  <Step>
    ## Configurer l&apos;intégration [#configure-step]

    1. Ouvrez `port-monitor-config.yml` et modifiez la configuration avec toutes les instances de port que vous souhaitez monitorer (ajoutez plusieurs sections au bloc integrations pour chaque adresse/port à monitorer :

       ```yml
       ```

    intégrations :

    * name: nri-port-monitor interval: 60s env:

      # Type de réseau. Les réseaux connus sont :

      # tcp, tcp4 (IPv4 uniquement), tcp6 (IPv6 uniquement),

      # udp, udp4 (IPv4 uniquement), udp6 (IPv6 uniquement),

      # ip, ip4 (IPv4 uniquement), ip6 (IPv6 uniquement),

      # unix, unixgram et unixpacket

      RÉSEAU : tcp

      # Adresse à vérifier au format hôte:port

      ADRESSE : localhost:80

      # Délai d&apos;attente en secondes

      DÉLAI D&apos;ATTENTE : 5 étiquettes : env : production

    ````
    </Step>

    <Step>
    ## Find your data [#find-data]

    To easily find your Port Monitoring data, download our pre-built dashboard:

    1. Go to <DNT>**[one.newrelic.com](https://one.newrelic.com/)**</DNT> and click on <DNT>**+ Integrations & Agents**</DNT>.
    2. Click on <DNT>**Dashboards**</DNT> tab.
    3. In the search box, search for <DNT>**Port Monitoring**</DNT>.
    4. Click on the Port Monitoring dashboard to install it.

    Here's an example NRQL query that returns the status of configured ports:

    ```sql
    SELECT latest(status) FROM NetworkPortSample FACET address SINCE 30 MINUTES AGO TIMESERIES
    ````
  </Step>
</Steps>
