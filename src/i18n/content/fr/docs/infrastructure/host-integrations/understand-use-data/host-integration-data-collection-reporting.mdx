Ce document explique comment notre intégration sur hôte collecte et rapporte les données.

## Processus de collecte et de rapport des données [#data-collection]

Voici comment l&apos;intégration sur hôte envoie les données à New Relic :

1. Au démarrage, l&apos;agent d&apos;infrastructure analyse le répertoire qui contient les fichiers de définition de l&apos;intégration.
2. L&apos;agent d&apos;infrastructure enregistre chaque exécutable d&apos;intégration défini dans le fichier de définition.
3. L&apos;agent analyse un répertoire dédié pour les fichiers de configuration d&apos;intégration.
4. Si ces fichiers de configuration spécifient une intégration qui a été enregistrée auprès de l&apos;agent infrastructure , l&apos;agent configure et planifie l&apos;intégration.
5. À l&apos;intervalle planifié (la valeur par défaut est de 15 secondes), l&apos;agent collecte les données de l&apos;intégration et les prépare pour la transmission. Toutes les 60 secondes, il envoie ces données à New Relic, ainsi que toutes les autres données d&apos;infrastructure.
6. Après une collecte réussie, l&apos;exécutable d&apos;intégration se termine.

## Architecture d&apos;intégration [#example-diagram]

Voici un exemple du fonctionnement de l’intégration de SQL Server :

* L&apos;hôte 1 avec l&apos;agent infrastructure installé a besoin d&apos;une règle sortante pour le port `1433`.
* L&apos;hôte 2 avec SQL Server a besoin d&apos;une règle entrante pour le port `1433`.

Dans cet exemple, l’hôte 1 extrait les données de l’hôte 2.

## Structure et spécifications du fichier [#file-structure]

Comprendre la structure des fichiers de ces intégrations sur hôte peut vous aider à personnaliser votre intégration, à comprendre et à utiliser vos données et à résoudre les problèmes.

Ces intégrations adhèrent à un ensemble de spécifications open source . Pour une explication de ces spécifications de fichiers, voir [Spécifications de fichiers](/docs/integrations/integrations-sdk/file-specifications).
