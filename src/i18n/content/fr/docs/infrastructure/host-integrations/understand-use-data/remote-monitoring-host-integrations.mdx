Du point de vue de New Relic, [l’entité](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic#what-is-entity) est un concept large. Une entité est tout ce que New Relic peut identifier et qui possède des données que vous pouvez monitorer.

Notre intégration sur hôte peut être configurée pour créer sa propre entité, appelée <DNT>**remote entity,**</DNT> en définissant l&amp;apos;option `remote_monitoring` sur `true`. Si la valeur est `false`, une intégration sera considérée comme une entité locale et les données qui lui sont associées seront attachées à l&amp;apos;entité hôte créée par l&amp;apos;agent. à distance monitoring nécessite infrastructure la [version 1.2.25 ou supérieure](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes) de l&amp;apos;agent .

Pour l'intégration [Apache](/docs/integrations/host-integrations/host-integrations-list/apache-monitoring-integration), [Cassandra](/docs/integrations/host-integrations/host-integrations-list/cassandra-monitoring-integration), [MySQL](/docs/integrations/host-integrations/host-integrations-list/mysql-monitoring-integration), [NGINX](/docs/integrations/host-integrations/host-integrations-list/nginx-monitoring-integration) et [Redis](/docs/integrations/host-integrations/host-integrations-list/redis-monitoring-integration) , monitoring à distance (et [le multi-hébergement](/docs/integrations/host-integrations/getting-started/introduction-host-integrations#multiple-instances)) est activée en activant le paramètre configuration `remote_monitoring`.

<Callout variant="important">
  Si votre service Apache, Cassandra, MySQL, NGINX ou Redis est situé sur le même hôte que l'agent, lorsque vous activez monitoring à distance, l'entité résultante sera considérée comme distante, quel que soit son emplacement réel. Cela peut affecter les alertes, modifier les attributs et avoir d'autres effets, comme expliqué ici.
</Callout>

## Effets de l'activation `remote_monitoring` [#effects-activating-remote-monitoring]

En activant `remote_monitoring`, l’intégration devient une entité différente qui n’est plus attachée à l’agent d’infrastructure. En conséquence, les éléments suivants peuvent être concernés :

<CollapserGroup>
  <Collapser id="verify-alerts" title="Vérification des alertes">
    [<DNT>**Enabling remote monitoring can affect your configured alerts**</DNT>](/docs/verify-your-alerts-after-activating-remote-monitoring) au cas où ils utiliseraient l’une des valeurs affectées par cette nouvelle fonctionnalité. Nous vous recommandons fortement de vérifier vos <InlinePopover type="alerts" />existants pour vous assurer qu&amp;apos;ils continuent de fonctionner comme prévu.
  </Collapser>

  <Collapser id="new-entity-attributes" title="Nouvel attribut d'entité">
    Ces attributs sont modifiés dans l'entité résultante :

    * <DNT>**Display name**</DNT>: Nouvelle clé unique d&amp;apos;entité (au lieu d&amp;apos;utiliser le nom d&amp;apos;affichage)
    * <DNT>**Entity GUID**</DNT>: Nouveau GUID d&amp;apos;entité
    * <DNT>**Entity ID**</DNT>: Nouvel ID d&amp;apos;entité
    * <DNT>**Entity key**</DNT>: Nouvelle clé unique d&amp;apos;entité (au lieu d&amp;apos;utiliser le nom d&amp;apos;affichage)
    * <DNT>**External key**</DNT>:Utilisation du nom de l&amp;apos;entité d&amp;apos;intégration (au lieu d&amp;apos;utiliser l&amp;apos;affichage de l&amp;apos;agent)
  </Collapser>

  <Collapser id="new-decoration" title="Modifications des métriques enregistrées">
    Lorsque monitoring à distance est activée, nous ajouterons les valeurs `hostname` et `port` à toutes les métriques. Si le nom `nricluster` ou `nriservice` sont définis dans le fichier de configuration d&amp;apos;intégration, ils seront également décorés.
  </Collapser>

  <Collapser id="lost-attributes" title="Attribut non enregistré">
    Étant donné que l'intégration est désormais une entité indépendante qui n'est pas attachée à l'agent, les attributs d'agent suivants ne sont pas collectés :

    * `agentName`
    * `agentVersion`
    * `coreCount`
    * `criticalViolationCount`
    * `fullHostname`
    * `instanceType`
    * `kernelVersion`
    * `linuxDistribution`
    * `entityType`
    * `operatingSystem`
    * `processorCount`
    * `systemMemoryBytes`
    * `warningViolationCount`
    * Votre [attribut personnalisé](/docs/infrastructure/new-relic-infrastructure/configuration/configure-infrastructure-agent#attributes)
  </Collapser>

  <Collapser id="updated-hostname" title="Nom d'hôte mis à jour">
    Pour les métriques d'intégration `ApacheSample`, `RedisSample`, `CassandraSample` et `NginxSample`, nous utiliserons le nom d&amp;apos;hôte configuration intégration au lieu du nom d&amp;apos;hôte court de l&amp;apos;agent.

    Lorsque le nom d'hôte d'intégration est une [adresse de bouclage](/docs/integrations/integrations-sdk/file-specifications/integration-executable-file-specifications#h2-loopback-address-replacement-on-entity-names), l&amp;apos;agent la remplacera afin de garantir l&amp;apos;unicité.
  </Collapser>
</CollapserGroup>
