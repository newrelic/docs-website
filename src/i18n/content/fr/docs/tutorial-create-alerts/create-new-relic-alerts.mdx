La configuration de <InlinePopover type="alerts" />pour vous avertir des problèmes avec votre stack est l’une des choses les plus importantes que vous puissiez faire pour monitorer vos données. Mais créer et gérer des alertes semble souvent fastidieux. Dans les tutoriels suivants, nous vous expliquerons les trois principales manières de créer des alertes avec New Relic. Nous vous montrerons également comment utiliser ces alertes pour collecter des informations et utiliser ces données pour améliorer votre stack. Si vous êtes complètement novice en matière d’alertes, nous avons quelques concepts et terminologies d’alerte de base à apprendre avant de vous lancer.

## Que sont les alertes ? [#alerts-what]

Les alertes sont des conditions qui, lorsqu&apos;elles sont remplies, vous envoient une notification concernant l&apos;événement déclencheur. Vous pouvez envoyer des notifications à n&apos;importe quel nombre de personnes dans n&apos;importe quel nombre d&apos;équipes. Lorsqu&apos;elles sont correctement configurées, les alertes vous permettent de résoudre un problème plus efficacement en envoyant une notification directe aux groupes qui peuvent résoudre un problème. Les alertes se composent de cinq parties principales :

* <DNT>**Incidents**</DNT> sont des événements où un seuil de condition est dépassé. Des exemples courants d&amp;apos;incidents sont lorsque la latence dépasse une certaine limite, qu&amp;apos;une page ne se charge pas ou qu&amp;apos;une erreur JavaScript se produit.
* <DNT>**Issues**</DNT> sont un ensemble d&amp;apos;un ou plusieurs incidents qui provoquent l&amp;apos;envoi d&amp;apos;une notification
* <DNT>**Conditions**</DNT> sont des objets de configuration que vous configurez pour ouvrir un incident
* <DNT>**Policies**</DNT> sont des regroupements de conditions dans lesquels vous définissez comment vous êtes averti lorsqu&amp;apos;un incident se produit
* <DNT>**Notifications**</DNT> sont les messages que vous recevez lorsqu&amp;apos;un problème est ouvert, reconnu ou fermé

<Callout variant="tip">
  Quelle est la différence entre <DNT>**Incidents**</DNT> et <DNT>**Issues**</DNT>?

  Un <DNT>**incident**</DNT> est un événement individuel qui détaille un symptôme d&amp;apos;un problème, tandis qu&amp;apos;un <DNT>**issue**</DNT> est un regroupement d&amp;apos;incidents qui décrivent un problème plus vaste dans votre système.
</Callout>

## Où sont les alertes dans New Relic ? [#alerts-where]

Dans New Relic, vous gérez le plus souvent vos alertes depuis [<DNT>**one.newrelic.com &gt; Alerts**</DNT>](https://one.newrelic.com/alerts-ai). Cette section de la plateforme comporte une page <DNT>**Overview**</DNT> qui vous donne une ventilation visuelle de toutes vos alertes, ainsi qu&amp;apos;une page <DNT>**Alert conditions (policies)**</DNT> qui vous donne une liste de toutes vos politiques d&amp;apos;alerte ainsi qu&amp;apos;un endroit pour en créer de nouvelles.
