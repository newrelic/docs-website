<Callout variant="important">
  Pour les versions 1220 et supérieures de l&apos;agent de navigateur, [l&apos;utilisation des cookies de session est obsolète](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1220/#removed-3rd-party-cookies). Ces informations ne sont donc pertinentes que si vous utilisez la version 1216 et inférieure de l&amp;apos;agent de navigateur.

  Ces cookies étaient liés à monitoring des applications du navigateur. Pour plus d&apos;informations sur la confidentialité des données et la demande de suppression de données, consultez [les demandes de données personnelles New Relic](/docs/using-new-relic/new-relic-security/security/new-relic-personal-data-requests).
</Callout>

Ce document décrit la confidentialité des données relatives à <InlinePopover type="browser" />et aux cookies, ainsi que des détails sur les cookies eux-mêmes. Avant la version 1220 de l&amp;apos;agent du navigateur, les [cookies](/docs/browser/new-relic-browser/performance-quality/security-new-relic-browser#cookies) suivants étaient créés pour [le suivi de session](/docs/browser/new-relic-browser/page-load-timing-resources/session-tracking) dans le navigateur final de l&amp;apos;utilisateur.

## Confidentialité des données et conformité légale [#gdpr]

La configuration par défaut de l&apos;agent du navigateur ne conserve aucune donnée personnelle telle que définie par les lois communes sur la confidentialité telles que le Règlement général de l&apos;UE sur la protection des données (RGPD) (RGPD) et le California consommateur Privacy Act (CCPA). New Relic a pris des mesures pour permettre à nos clients qui choisissent de traiter des données personnelles dans nos produits de le faire conformément aux lois applicables en matière de confidentialité. Utilisez les informations supplémentaires ci-dessous pour vous aider à vous conformer à ces lois :

Pour ces agents, des cookies tiers sont placés par l&apos;agent du navigateur et le collecteur New Relic. Le collecteur New Relic place un cookie tiers lorsque l&apos;agent du navigateur établit une connexion et transmet des données.

        Un gestionnaire de consentement aux cookies qui utilise un chargeur script conditionnel qui charge uniquement la balise/l&apos;extrait lorsqu&apos;un visiteur accepte un type spécifique de cookies peut être utilisé. En choisissant cette option, il existe un risque que les performances de la page ne soient pas capturées tant que le visiteur n&apos;a pas donné son accord et que l&apos;agent du navigateur n&apos;est pas en cours d&apos;exécution.

        Avec la sortie de [l&apos;agent de navigateur v1169](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1169), vous pouvez désactiver la collecte de cookies tiers ([`JSESSIONID`](#jsessionid) et [`NREUM`](#nreum)) pour votre application de navigateur en utilisant la page <DNT>**Application settings**</DNT> dans l&amp;apos;UI de New Relic. Une fois cette valeur de configuration définie sur faux (OFF), vous ne verrez plus les informations de session dans vos données d&amp;apos;événement.

        Par défaut, la valeur est définie sur true (ON) pour toutes les applications de navigateur existantes ainsi que celles nouvellement créées. Pour plus d&apos;informations, consultez notre [documentation sur la collecte de cookies et le suivi des sessions](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking).

## Détails sur la collecte de cookies obsolètes [#cookies]

  <Collapser id="jsessionid" title="JSESSIONID (obsolète)">
    Le cookie `JSESSIONID` est obsolète pour la version d&amp;apos;agent 1220 et supérieure. Le suivi des sessions ne se fait plus via des cookies.

    Il permettait de stocker un <DNT>**session identifier**</DNT> afin que New Relic puisse monitorer le nombre de sessions d&amp;apos;une application. La valeur du cookie est générée par Jetty.

    <Callout variant="important">

Tous les navigateurs pris en charge par les versions actuelles de l&apos;agent de navigateur prennent en charge l&apos;[API de synchronisation de navigation](/docs/browser/new-relic-browser/browser-agent-apis/navigation-timing-api).

Lorsqu&amp;apos;un navigateur prend en charge l&amp;apos;API de synchronisation de navigation, une interface native peut être utilisée pour déterminer l&amp;apos;heure de début de la navigation.

    Les versions 7 ou supérieures de Mozilla Firefox prennent en charge l&apos;API de synchronisation de navigation. Cependant, un bug existe dans les premières implémentations de l&apos;API Navigation Timing de Firefox que monitoring des navigateurs contourne en utilisant ce cookie.

  <Collapser id="nragent" title="NRAGENT (obsolète)">
    Le cookie NRAGENT est défini uniquement si vous utilisez [la version](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) de l&amp;apos;agent de navigateur v443 ou inférieure. Il est créé lorsqu&amp;apos;un jeton est remis à un utilisateur final par le collecteur New Relic.

    Les métriques finales utilisateur sont communiquées au collecteur New Relic via une requête JSONP du navigateur (récupération script ). Le JavaScript renvoyé contient un appel pour définir l&apos;identifiant du jeton qui est écrit dans le cookie lors de la prochaine navigation (lorsque la page se décharge). Cela permet une communication inter-domaines entre le collecteur New Relic et l&apos;agent New Relic.

    Ce cookie est utilisé pour communiquer entre le collecteur New Relic agrégeant les métriques de l&apos;utilisateur final et le ou les agents exécutés dans l&apos;application Web associée. Un jeton identifie et corrèle la trace de transaction [au niveau de](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#tier) l&amp;apos;application avec la trace du navigateur correspondante.
