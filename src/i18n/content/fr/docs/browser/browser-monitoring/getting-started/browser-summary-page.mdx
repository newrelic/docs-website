Notre page <InlinePopover type="browser" /><DNT>**Summary**</DNT> vous aide à résoudre les problèmes liés aux performances réelles du navigateur utilisateur de votre application. Utilisez la page <DNT>**Summary**</DNT> pour :

* Obtenez en un coup d&apos;œil les principaux éléments essentiels du Web, notamment la peinture de contenu la plus volumineuse (LCP), l&apos;interaction avec la peinture suivante (INP) et le décalage de mise en page cumulatif (CLS).
* Affichez les tendances en matière de performances côté navigateur d&apos;une application.
* Résolvez rapidement les problèmes de temps de chargement des pages.
* Accédez directement à d’autres pages UI du navigateur pour examiner les problèmes plus en détail.
* Ajouter la condition d&apos;alerte recommandée.

## Voir la page de résumé [#viewing]

Pour afficher un résumé des performances d’une application de navigateur :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities**</DNT>.
2. Cliquez <DNT>**Browser**</DNT>
3. Sélectionnez une application de navigateur.

## Utilisez les données essentielles du Web pour améliorer le référencement [#core\_vitals][#core_vitals]

Utilisez les graphiques <DNT>**Core web vitals**</DNT> pour comprendre les performances de votre navigateur selon [les Core Web Vitals de Google](https://web.dev/vitals/#core-web-vitals). Il s’agit notamment de :

* LCP : la plus grande peinture de contenu, qui mesure les performances de chargement
* INP : interaction avec la prochaine peinture, qui mesure l&apos;interactivité entre la saisie de l&apos;utilisateur et la réponse du navigateur
* CLS : Cumulative layout shift, qui mesure la stabilité visuelle dans la page du navigateur

Des scores plus élevés peuvent aider votre organisation à économiser de l&apos;argent et à influencer le référencement. Des scores inférieurs peuvent affecter la perception que vos utilisateurs ont du site Web de votre organisation, ce qui pourrait entraîner des clients insatisfaits ou des opportunités commerciales perdues.

Pour en savoir plus sur monitoring des principaux éléments essentiels du Web, consultez notre [Guide des principaux éléments essentiels du Web](/docs/browser/new-relic-browser/guides/guide-to-core-web-vitals).

* <DNT>
    **User time on the site**
  </DNT>

* <DNT>**User-centric page load times**</DNT> et <DNT>**Longest interaction to next paint by URL**</DNT>: pour obtenir plus de détails sur le [processus de chargement de la page](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process), cliquez sur le titre du graphique pour accéder directement à l&apos;[UI<DNT>**Page views**</DNT> ](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance).

## Localiser les problèmes de performances du navigateur [#performance]

Utilisez ces graphiques pour vous aider à identifier les problèmes de performances liés au navigateur :

* <DNT>**Initial page load and route change**</DNT>:Ce graphique montre le temps de chargement d&apos;un changement d&apos;URL traditionnel résultant d&apos;un chargement ou d&apos;un rechargement d&apos;une URL. Pour obtenir plus de détails, cliquez sur le titre du graphique pour accéder directement à l&apos;[UI<DNT>**Page views**</DNT> ](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance).
* <DNT>**Throughput**</DNT>:Ce graphique montre le débit du navigateur en pages par minute (ppm). Le cas échéant, ce graphique affiche également les données des applications sur une seule page (SPA). Pour obtenir plus de détails, cliquez sur le titre du graphique pour accéder directement à l&apos;[UI<DNT>**Page views**</DNT> ](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance).
* <DNT>**Page views with JavaScript errors**</DNT>: Pour obtenir plus de détails, cliquez sur le titre du graphique pour accéder directement à l&apos;[UI<DNT>**JS errors**</DNT> ](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors).

## Examiner les problèmes causés par autre chose [#other-issues]

Les problèmes de performances peuvent être liés au navigateur ou à autre chose. Pour obtenir plus de détails sur d’autres problèmes susceptibles d’affecter les performances du navigateur, examinez les éléments suivants :

* <DNT>**Frontend vs. backend**</DNT>:Ce graphique renvoie à l&apos;application de service <InlinePopover type="apm" />connectée. Pour obtenir plus de détails sur [les traces distribuées](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing/), cliquez sur le titre du graphique.
* <DNT>**First interaction by device type**</DNT> (ordinateur de bureau, tablette ou mobile) et <DNT>**First interaction by user agent**</DNT>: pour obtenir plus de détails, cliquez sur le titre du graphique pour accéder directement à l&apos;[UI<DNT>**Page views**</DNT> ](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance).
* <DNT>**Related entities**</DNT>:Ce widget peut vous aider à déterminer si les problèmes de performances peuvent être causés par des problèmes avec [des entités](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/) ayant des connexions au navigateur.
* <DNT>**View synthetic monitors**</DNT>: Le [monitoring synthétique](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring/) permet de simuler le trafic utilisateur. Ce lien peut vous aider à détecter et à résoudre de manière proactive les pannes et les mauvaises performances liées aux points de terminaison critiques avant que vos clients ne s&apos;en aperçoivent.

## Ajouter une condition d&apos;alerte recommandée [#recommended-alert-conditions]

Grâce à la page monitoring des navigateurs <DNT>**Summary**</DNT>, vous pouvez ajouter des conditions d&apos;alerte recommandées à votre entité.

Consultez la page [Lacunes de couverture des alertes et recommandations de conditions](/docs/alerts-applied-intelligence/applied-intelligence/alert-coverage-gaps/) si vous souhaitez voir plus d&apos;entités non couvertes.

Suivez ces étapes pour ajouter la condition d’alerte recommandée :

1. Ouvrez la page monitoring des navigateurs <DNT>**Summary**</DNT>.

2. Cliquez sur <DNT>**Add recommended conditions**</DNT>.

   <img width="80%;" style={{align: "left"}} title="Browser monitoring summary" alt="Screenshot of the New Relic browser monitoring Summary page" src="/images/browser_screenshot-full_browser-alert-conditions.webp" />

3. Choisissez un nombre de conditions d&apos;alerte recommandées.

   <img width="70%;" style={{align: "left"}} title="Browser monitoring recommended alert conditions" alt="Browser monitoring recommended alert conditions" src="/images/browser_screenshot-crop_browser-add-alert.webp" />

4. Cliquez sur <DNT>**Next**</DNT>.

5. Sélectionnez une politique pour être averti. Vous pouvez en sélectionner un existant ou [en créer un nouveau](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name).

6. Cliquez sur <DNT>**Activate conditions**</DNT>.
