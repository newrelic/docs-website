## Enregistrer manuellement la rediffusion de session [#manual-replays]

Si nécessaire, vous pouvez programmer `force` une relecture de session pour démarrer ou arrêter l&amp;apos;enregistrement à l&amp;apos;aide de méthodes API. Voici quelques cas d’utilisation :

* Autoriser l'utilisateur à accepter l'enregistrement interaction . Par exemple, si vous avez une fenêtre contextuelle demandant « Autoriser l'enregistrement de cette session à des fins d'analyse des performances ? » et un utilisateur consent :

  1. Appelez `newrelic.recordReplay()` pour lancer l&amp;apos;enregistrement.
  2. Une fois la session terminée ou le consentement retiré, utilisez `newrelic.pauseReplay()` pour arrêter la capture de données.

* Enregistrez des sessions en fonction de critères spécifiques, tels que :

  * Démarrez l'enregistrement uniquement pour les sessions sur des URL spécifiques telles qu'un flux de paiement de produit.
  * Arrêtez l'enregistrement pour les pages sensibles telles que `/billing` ou `/health-records`.

Reportez-vous à la documentation de l'API pour obtenir des instructions :

* [`newrelic.recordReplay`](/docs/browser/new-relic-browser/browser-apis/recordReplay/)
* [`newrelic.pauseReplay`](/docs/browser/new-relic-browser/browser-apis/pauseReplay/)

## Désactiver le suivi automatique de la relecture de session [#auto-replays]

Vous pouvez empêcher le démarrage automatique de Session Replay, même lorsqu'il est échantillonné, en définissant l'option configuration de l'API `autoStart` sur `false`. Cela est utile dans les scénarios où vous devez prioriser des actions spécifiques par rapport à l&amp;apos;échantillonnage automatique. Par exemple, vous ne souhaitez peut-être pas que Session Replay s&amp;apos;exécute automatiquement sur des pages sensibles, comme celles conformes à la loi HIPAA. Dans de tels cas, vous pouvez utiliser l&amp;apos;API `.start()` pour déclencher manuellement la fonctionnalité de relecture après l&amp;apos;obtention du consentement de l&amp;apos;utilisateur (et votre application appelle `.start()`).

Le paramètre `autoStart: false` indique à l&amp;apos;agent de différer son initialisation jusqu&amp;apos;à ce qu&amp;apos;il soit explicitement appelé à l&amp;apos;aide de `.start()`. Cela garantit que la relecture de session ne sera pas créée du tout, même si le backend échantillonne votre application pour l&amp;apos;enregistrement.

<Callout variant="important">
  L'appel de `.start()` ne garantit pas qu&amp;apos;une rediffusion de session sera enregistrée. Cela retarde simplement l’importation du module et l’évaluation des résultats d’échantillonnage jusqu’à ce point. Si vous devez forcer un enregistrement, reportez-vous [à Enregistrer manuellement la relecture de session](#manual-replays).
</Callout>

Pour commencer à utiliser cette API, reportez-vous [`newrelic.start`](/docs/browser/new-relic-browser/browser-apis/start/).

## Utilisez Nerdgraph pour configurer les paramètres de relecture [#nerdgraph-mutation]

Vous pouvez configurer les paramètres de relecture de session à l'aide de notre [API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/) au lieu d&amp;apos;utiliser l&amp;apos;UI. Cette méthode est utile si vous devez effectuer des modifications en masse, comme l&amp;apos;application de taux d&amp;apos;échantillonnage à plusieurs entités de navigateur.
