Les actions des utilisateurs dans monitoring des navigateurs vous aident à comprendre le comportement des utilisateurs, comme l&apos;endroit où ils cliquent, font défiler et interagissent avec votre application Web. Vous pouvez utiliser les actions utilisateur pour lier les actions entre elles, vous aidant ainsi à obtenir des informations détaillées sur les endroits où votre application Web crée des frictions et augmente la frustration de l&apos;utilisateur.

Les actions de l&apos;utilisateur sont suivies par défaut pour les agents Pro et Pro+SPA, mais ne sont pas disponibles pour l&apos;agent de navigateur Lite. Nous vous recommandons de confirmer d&apos;abord que vous utilisez l&apos;agent Pro ou Pro+SPA :

Accédez à <DNT>**[one.newrelic.com &gt; All Capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Browser**</DNT> (sélectionnez votre application de navigateur) &amp;gt; <DNT>**Application settings**</DNT>. À partir de cette page, vérifiez que l’agent de navigateur <DNT>**Pro**</DNT> ou <DNT>**Pro + SPA**</DNT> est sélectionné. Veuillez vous assurer que vous utilisez la version v1.268.0 ou supérieure.

## Afficher les actions de l&apos;utilisateur [#view-events]

Lorsqu&apos;un utilisateur charge une page Web, notre agent navigateur configure des écouteurs d&apos;événement pour l&apos;événement interaction clé. Lorsque l&apos;agent du navigateur observe ces actions, il peut récolter l&apos;événement généré et le transmettre à New Relic.

Les actions utilisateur peuvent récolter ces événements :

* auxclick
* flou (fenêtre uniquement)
* Cliquez
* copie
* focus (fenêtre uniquement)
* touche enfoncée
* coller
* défilement

Pour afficher les actions utilisateur capturées par l&apos;agent du navigateur, créez une requête NRQL pour le type d&apos;événement `UserAction` dans [le générateur de requêtes](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/). Voici un exemple de requête de base :

```sql
FROM UserAction 
SELECT *
```

Par défaut, les données du navigateur événement sont stockées huit jours, mais la rétention réelle des données dépend de votre compte. Pour plus de détails sur les actions de l&apos;utilisateur, consultez le [Dictionnaire de données](/attribute-dictionary/).

## Consommation de données [#data-consumption]

les actions de l&apos;utilisateur sont agrégées séquentiellement, ce qui signifie que si plusieurs actions sont effectuées en succession rapide sur la même cible DOM , elles sont regroupées en une seule action de l&apos;utilisateur. Cela permet de maintenir les coûts de consommation à un niveau bas tout en vous aidant à comprendre comment les utilisateurs interagissent avec votre application Web.

les actions de l&apos;utilisateur suivent la même tarification de consommation que vos autres octets de navigateur. La quantité d’octets produits dépend du nombre, de la longueur et des niveaux d’activité de l’utilisateur des sessions.
