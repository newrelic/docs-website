Pour utiliser la relecture de session dans New Relic, commencez par activer cette fonctionnalité dans les paramètres de votre application de navigateur. Cette étape est essentielle pour capturer les interactions utilisateur sur votre application web, fournir des informations détaillées sur les parcours utilisateur et dépanner efficacement les problématiques. Suivez les étapes ci-dessous pour activer la relecture de session et vous assurer que votre agent de navigateur est correctement configuré.

<Callout variant="important">
  Certains paramètres, notamment l&apos;activation ou la désactivation de la relecture de session et le réglage des fréquences d&apos;échantillonnage pour l&apos;ensemble des sessions ou uniquement pour celles comportant des erreurs, ont été déplacés vers configuration côté serveur. Par exemple, une fois l&apos; snippet mis à jour dans vos pages pour inclure la fonctionnalité de relecture de session, il n&apos;est pas nécessaire de redéployer le script lors de la modification des fréquences d&apos;échantillonnage ou lors de la désactivation ou de la réactivation de la fonctionnalité. Si la fonctionnalité n&apos;a pas été incluse dans le script de déploiement, la configuration côté serveur n&apos;aura aucun impact. Ceci est applicable à la version `1.259.0` ou supérieure de l&apos;agent de browser.
</Callout>

## Activer la relecture de session [#enable-session-replay]

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; All Capabilities &gt; Browser**</DNT>.

2. Sélectionnez votre application de navigateur.

3. Dans le menu de gauche, cliquez sur <DNT>**Application settings**</DNT>.

4. Sur la page <DNT>**Application settings**</DNT> , assurez-vous que l’agent de navigateur <DNT>**Pro**</DNT> ou <DNT>**Pro + SPA**</DNT> est sélectionné. La relecture de session n&apos;est pas disponible pour l&apos;agent de navigateur Lite .

5. Activez <DNT>**Session traces**</DNT> et <DNT>**Session replay**</DNT> .

   <Callout variant="important">
     Si vous ne parvenez pas à afficher les paramètres <DNT>**Session replay**</DNT> . Contactez votre administrateur pour obtenir de l&apos;aide.
   </Callout>

## Définition des taux d&apos;échantillonnage [#configure-sampling-rates]

Définissez un taux d’échantillonnage (0 % à 100 %) pour les échantillons suivants :

* <DNT>**User sessions**</DNT> enregistre un échantillon aléatoire de toutes les sessions utilisateur.

* <DNT>**Errored session**</DNT> enregistre un échantillon aléatoire de toutes les sessions erronées. L&apos;agent met en mémoire tampon une fenêtre glissante de données pendant 30 secondes maximum côté client et, si des erreurs se produisent, les données mises en mémoire tampon seront récoltées et le reste de la session sera enregistré. Si aucune erreur ne se produit, la rediffusion n&apos;est jamais récoltée.

Par exemple, si vous définissez le taux d&apos;échantillonnage de session à 50 % et le taux d&apos;échantillonnage d&apos;erreur à 100 %, cela signifie que :

* La moitié de toutes les sessions utilisateurs seront enregistrées.

* Toutes les sessions erronées seront enregistrées, même si elles ne font pas partie de l&apos;échantillon de 50 %.
