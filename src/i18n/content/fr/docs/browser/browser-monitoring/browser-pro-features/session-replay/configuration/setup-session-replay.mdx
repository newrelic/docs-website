Pour utiliser Session Replay dans New Relic, commencez par activer cette fonctionnalité dans les paramètres de votre application de navigateur. Cette étape est essentielle pour capturer les interactions utilisateur sur votre application web, fournir des informations détaillées sur les parcours utilisateur et dépanner efficacement les problématiques. Suivez les étapes ci-dessous pour activer la relecture de session et vous assurer que votre agent de navigateur est correctement configuré.

## Activer la relecture de session [#enable-session-replay]

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &amp;gt; All Capabilities &amp;gt; Browser**</DNT>.

2. Sélectionnez votre application de navigateur.

3. Dans le menu de gauche, cliquez sur <DNT>**Application settings**</DNT>.

4. Sur la page <DNT>**Application settings**</DNT> , assurez-vous que l’agent de navigateur <DNT>**Pro**</DNT> ou <DNT>**Pro + SPA**</DNT> est sélectionné. La relecture de session n&amp;apos;est pas disponible pour l&amp;apos;agent de navigateur Lite .

5. Activez <DNT>**Session traces**</DNT> et <DNT>**Session replay**</DNT> .

   <Callout variant="important">
     Si vous ne parvenez pas à afficher les paramètres <DNT>**Session replay**</DNT> . Contactez votre administrateur pour obtenir de l&amp;apos;aide.
   </Callout>

## Définition des taux d&apos;échantillonnage [#configure-sampling-rates]

Définissez un taux d’échantillonnage (0 % à 100 %) pour les échantillons suivants :

* <DNT>**User sessions**</DNT> enregistre un échantillon aléatoire de toutes les sessions utilisateur.

* <DNT>**Errored session**</DNT> enregistre un échantillon aléatoire de toutes les sessions erronées. L&amp;apos;agent met en mémoire tampon une fenêtre glissante de données pendant 30 secondes maximum côté client et, si des erreurs se produisent, les données mises en mémoire tampon seront récoltées et le reste de la session sera enregistré. Si aucune erreur ne se produit, la rediffusion n&amp;apos;est jamais récoltée.

Par exemple, si vous définissez le taux d&apos;échantillonnage de session à 50 % et le taux d&apos;échantillonnage d&apos;erreur à 100 %, cela signifie que :

* La moitié de toutes les sessions utilisateurs seront enregistrées.

* Toutes les sessions erronées seront enregistrées, même si elles ne font pas partie de l&apos;échantillon de 50 %.
