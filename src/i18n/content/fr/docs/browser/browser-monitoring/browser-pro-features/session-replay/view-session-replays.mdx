Commencez à analyser les données capturées dans New Relic. Cette section fournit des instructions sur l&apos;accès et la navigation dans Session Replay, vous aidant à explorer l&apos;interaction utilisateur et à identifier les domaines à améliorer. Découvrez comment exploiter les données de relecture de session pour améliorer l&apos;expérience utilisateur et résoudre efficacement les problèmes.

## Voir la rediffusion de la session dans New Relic [#view-data]

Vous pouvez trouver Session Replay à deux endroits :

* Sur la page <DNT>**Session replay**</DNT> :

<Steps>
  <Step>
    Allez à <DNT>**[one.newrelic.com &gt; All Capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT>.
  </Step>

  <Step>
    Sélectionnez votre application de navigateur.
  </Step>

  <Step>
    Dans le menu de gauche, faites défiler vers le bas et cliquez sur <DNT>**Session replay**</DNT>.
  </Step>

  <Step>
    Cliquez sur chaque rediffusion pour afficher les détails de la session.

    <img title="Session replay" alt="screenshot of the session replay page in browser" src="/images/browser_screenshot-full_Session-replay-page.webp" />
  </Step>
</Steps>

* Sur la page <DNT>**Errors inbox**</DNT> :

<Steps>
  <Step>
    Dans le menu de gauche du navigateur, cliquez sur <DNT>**Errors**</DNT>.
  </Step>

  <Step>
    Cliquez sur les pages <DNT>**Triage**</DNT> et <DNT>**Group errors**</DNT> pour voir les rediffusions attachées aux erreurs.

    <img title="Replays in errors inbox" alt="Screenshot of replays in errors inbox" src="/images/browser_screenshot-full_errors-inbox-session-replay.webp" />
  </Step>
</Steps>

<Callout variant="important">
  Certains paramètres, notamment l&apos;activation ou la désactivation de la relecture de session et le réglage des fréquences d&apos;échantillonnage pour l&apos;ensemble et les erreurs, ont été déplacés vers configuration côté serveur. Par exemple, après avoir mis à jour ces paramètres pour la méthode d&apos;installation copier/coller, il n&apos;est pas nécessaire de redéployer le script. Ceci s&apos;applique à la version de l&apos;agent de browser `1.259.0` ou supérieure.
</Callout>
