Dans New Relic <InlinePopover type="browser" />, l&apos;UI <DNT>**Browsers**</DNT> fournit des informations sur l&apos;expérience finale de votre utilisateur avec votre application en fonction du navigateur qu&apos;il utilise, tel que Google Chrome, Mozilla Firefox, Microsoft Internet Explorer, Apple Safari et Microsoft Edge.

<img title="Browsers UI" alt="A screenshot of the Browsers UI in New Relic" src="/images/browser_screenshot-full_browsers-ui.webp" />

<figcaption>
  <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Browser &gt; (select an app) &gt; Browsers**</DNT>:Obtenez des informations détaillées sur l&apos;expérience finale de votre utilisateur par type et version de navigateur, appareil, erreurs JS, détails du temps de chargement des pages, débit et données Core Web Vitals associées.
</figcaption>

Cette page comprend :

* Top navigateur par débit (page vues par minute ou `ppm`) et temps de chargement de page
* Données Core web vitals , notamment le plus grand contenu de peinture (LCP), interaction avec le prochain tableau (INP), le score cumulé de décalage de mise en page (CSL) et bien plus encore
* Principales erreurs JavaScript par erreur, avec un lien vers plus de détails dans l&apos;[UI des erreurs JS](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors/)
* Données filtrables par type d&apos;appareil (mobile, tablette, ordinateur de bureau)

Les graphiques détaillés segmentent également le type de navigateur sélectionné par version ; par exemple, Chrome 14, 47, 72, etc. Cela vous aide à déterminer rapidement si les problèmes de temps de chargement des pages peuvent être liés à un type de navigateur ou à un appareil spécifique, ou si le problème est plus répandu.

## Utiliser l&apos;UI du navigateur [#browser\_procedures][#browser_procedures]

Par exemple, si vous souhaitez enquêter sur un problème de rendu de page lent, vous pouvez découvrir quels utilisateurs sont impactés en fonction de leur type de navigateur.

Tout d’abord, allez à la page <DNT>**Browsers**</DNT> :

1. Allez sur [one.newrelic.com](https://one.newrelic.com).
2. Cliquez sur le navigateur et sélectionnez une application.
3. Sous <DNT>**Monitor**</DNT>, cliquez sur <DNT>**Browsers**</DNT>.

Vous remarquez que le temps de chargement de la page est nettement plus long pour le navigateur Safari.

<img title="Browsers UI" alt="A screenshot of the Browsers UI in New Relic" src="/images/browser_screenshot-full_browser-load-time.webp" />

Pour découvrir ce qui se passe avec le navigateur Firefox, cliquez sur <DNT>**Firefox**</DNT> dans la liste des types de navigateurs et accédez aux données spécifiques au navigateur.

<img title="Core Web Vitals data in the Browsers UI" alt="A screenshot of Core Web Vitals data in the Browsers UI" src="/images/browser_screenshot-full_chrome-example.webp" />

Tenez compte des questions suivantes lorsque vous examinez les données :

* Où se trouvent les pics dans le [processus de chargement des pages](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process)?
* Existe-t-il des schémas récurrents au fil du temps ?
* Les temps de chargement de page et le débit moyens se situent-ils dans des fourchettes acceptables ?
* Les problèmes sont-ils liés à une version de navigateur spécifique ?
* Les principales [erreurs JavaScript](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors/) indiquent-elles un problème avec le code, une vulnérabilité potentielle en matière de sécurité ou autre chose ?
* Pour ce type de navigateur, comment ces problèmes de performances affectent-ils les scores Core Web Vitals pour la première peinture, la première peinture de contenu, la plus grande peinture de contenu, le chargement de la fenêtre et la première interaction ?
* Comment ces problèmes se comparent-ils à d’autres types d’appareils (ordinateur de bureau, mobile, tablette) ou à d’autres types de navigateurs ?
