La page <DNT>**User impact**</DNT> dans <InlinePopover type="browser" />vous aide à comprendre comment les problèmes de performance technique affectent l&apos;expérience réelle de vos utilisateurs. Cette page fait le lien entre les mesures de performance quantitatives et l&apos;analyse qualitative du comportement des utilisateurs, fournissant des informations sur les schémas de frustration des utilisateurs et leur corrélation avec les problèmes techniques.

La page User impact se compose de deux zones d&apos;analyse principales :

* <DNT>Frustration metrics</DNT>
* <DNT>Performance risk</DNT>

Chacun fournit des perspectives différentes pour comprendre les problèmes d&apos;expérience utilisateur et leurs causes techniques profondes.

**Condition préalable:**

* La version 1.269 ou ultérieure de l&apos;agent Browser est requise pour capturer les signaux de frustration et les données de risque de performance.
* [Session Replay](/docs/browser/browser-monitoring/browser-pro-features/session-replay/get-started/) doit être activé pour accéder aux relectures de session liées à cette page.

## Accéder à l&apos;analyse de l&apos;impact utilisateur [#access]

Pour afficher la page User impact :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app)** &gt; **Users** &gt; **User impact**</DNT>.
2. Sélectionnez <DNT>**Frustration metrics**</DNT> ou <DNT>**Performance risk**</DNT> dans l&apos;onglet.

<Callout variant="tip">
  Vous pouvez également y accéder directement depuis la page [Page views](/docs/browser/new-relic-browser/browser-pro-features/page-views-examine-page-performance) en cliquant sur le lien <DNT>**Analyze frustration signals**</DNT> ou <DNT>**Analyze performance risk**</DNT>.
</Callout>

## Analysez les mesures de frustration [#frustration-metrics]

L&apos;onglet Frustration metrics se concentre sur les indicateurs comportementaux des utilisateurs qui signalent confusion, agacement ou difficulté à effectuer des tâches. Cette analyse vous aide à identifier les endroits où les utilisateurs rencontrent le plus de difficultés et à hiérarchiser les corrections en fonction des problèmes réels des utilisateurs.

L&apos;onglet Frustration metrics comprend les sections suivantes :

* Clics de rage : Identifie les pages et les éléments où les utilisateurs affichent un comportement de clics rapides et répétés, indiquant une frustration face à des composants d&apos;interface utilisateur non réactifs ou défectueux.
* Pages les plus touchées : Met en évidence les pages spécifiques générant le plus de signaux de frustration, ce qui vous permet de vous concentrer sur les zones à fort impact.
* Éléments de page les plus affectés : Fournit des informations détaillées sur les éléments d&apos;interface utilisateur spécifiques qui causent le plus de frustration aux utilisateurs.

## Analyser les mesures de performance [#performance-impact]

L&apos;onglet Performance risk examine comment les problèmes de performance technique affectent les sessions utilisateur, en se concentrant sur les mesures de performance mesurables et leur corrélation avec les problèmes d&apos;expérience utilisateur.

L&apos;onglet Performance risk comprend les sections suivantes :

* Web vitals : analyse l&apos;impact des mauvais indicateurs Web essentiels (LCP, INP, CLS) sur les sessions et l&apos;expérience utilisateur.
* Analyse des erreurs : Étudie les schémas d&apos;erreurs JavaScript et leur effet sur les sessions utilisateur.
* Pages les plus touchées : identifie les pages qui subissent le plus d&apos;impact utilisateur lié aux performances.
* Analyse géographique : Examine le risque de performance dans différentes zones géographiques afin d&apos;identifier les problèmes régionaux.
