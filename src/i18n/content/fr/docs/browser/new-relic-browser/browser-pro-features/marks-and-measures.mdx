[Les marques](https://developer.mozilla.org/en-US/docs/Web/API/Performance/mark) et [les mesures](https://developer.mozilla.org/en-US/docs/Web/API/Performance/measure) sont des API de navigateur standard qui vous aident à observer et à signaler les performances de vos pages Web. Ces événements génériques et natifs vous permettent de mesurer la durée de n&apos;importe quelle tâche ou processus dans votre application. Avec monitoring des navigateurs New Relic, vous pouvez désormais suivre automatiquement les marques et les mesures et les stocker en `BrowserPerformance` qu&apos;événement, ce qui vous offre une visibilité plus approfondie sur les performances réelles de votre site.

## Pourquoi utiliser monitoring des marques et des mesures ?

Le suivi automatique des marques et des mesures avec New Relic fournit des informations exploitables détaillées sur les performances de vos applications Web. En capturant ces événements, vous pouvez :

* Identifiez les zones lentes ou problématiques de votre code.
* Mesurer l&apos;impact d&apos;une nouvelle fonctionnalité ou déploiement.
* Comparez les performances des différents parcours utilisateur.
* Créez des dashboards personnalisés et des alertes basées sur des données utilisateur réelles.

## Prérequis [#prerequisites]

* Un agent de navigateur **Pro** ou **Pro + SPA**, version 1.272.0 ou supérieure.,

<Callout variant="important">
  La détection automatique des marques et des mesures n&apos;est pas disponible pour l&apos;agent de navigateur Lite.
</Callout>

## Activer la collecte automatique [#enable-collection]

Améliorez l&apos;observabilité et les capacités de débogage de votre application en suivant ces instructions de configuration simples.

<Steps>
  <Step>
    Accédez à **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Sélectionnez votre application de navigateur.
  </Step>

  <Step>
    Dans le menu de gauche, cliquez sur **Paramètres de l’application**.
  </Step>

  <Step>
    Assurez-vous que l&apos;agent de navigateur **Pro** ou **Pro + SPA** est sélectionné.
  </Step>

  <Step>
    Pour collecter automatiquement les marques, activez **Capture marks**. Pour collecter automatiquement les mesures, activez **Capture measures**. Pour collecter des détails à partir de marques ou de mesures observées en tant qu&apos;attribut personnalisé, activez **Add details**.
  </Step>
</Steps>

## Désactiver la collecte des marques et des mesures [#disable-marks-and-measures]

Pour désactiver les marques et/ou les mesures pour une application de navigateur existante :

<Steps>
  <Step>
    Accédez à **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Browser**.
  </Step>

  <Step>
    Sélectionnez votre application de navigateur.
  </Step>

  <Step>
    Dans le menu de gauche, cliquez sur **Paramètres de l’application**.
  </Step>

  <Step>
    Pour désactiver les marques, désactivez **Capture marks**. Pour désactiver les mesures, désactivez les **Capture measures**.
  </Step>
</Steps>

## Consommation de données

`BrowserPerformance` événement suivent la même tarification de consommation que vos autres octets de navigateur. La quantité d&apos;octets produits dépend du nombre et de l&apos;attribut des données.

Une fois activé, l&apos;agent capturera automatiquement les marques et les mesures au fur et à mesure qu&apos;elles se produisent dans votre application.

## Retrouvez vos données dans New Relic [#find-data]

Les données de marques et de mesures sont stockées sous la forme d&apos;événement `BrowserPerformance` dans New Relic, désigné par `mark` ou `measure` sous le champ `entryType`. Consultez le [dictionnaire de données](https://docs.newrelic.com/attribute-dictionary/?event=BrowserPerformance) pour plus d&apos;informations sur les champs disponibles pour une utilisation avec `BrowserPerformance` événement. Pour un dashboard rapide et utile, consultez ce [quickstart](https://newrelic.com/instant-observability/browser-marks-and-measures), sinon vous pouvez interroger et visualiser ces données pour obtenir des informations détaillées sur les performances de votre application.

**Trouver les mesures les plus lentes par durée moyenne**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET entryName 
LIMIT 10 
ORDER BY average(entryDuration) DESC
```

**Comptez le nombre de marques et de mesures par page**

```nrql
FROM BrowserPerformance 
SELECT count(*) 
WHERE entryType IN ('mark', 'measure') 
FACET pageUrl
```

**Voir la durée moyenne de mesure par navigateur**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET userAgentName
```

**Suivi des durées de mesure par pays**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET countryCode
```

**Rechercher tous les noms de mesures uniques enregistrés au cours des dernières 24 heures**

```nrql
FROM BrowserPerformance 
SELECT uniqueCount(entryName) 
WHERE entryType = 'measure' 
SINCE 24 hours ago
```

**Comparer les durées de mesure pour ordinateur de bureau et mobile**

```nrql
FROM BrowserPerformance 
SELECT average(entryDuration) 
WHERE entryType = 'measure' 
FACET deviceType
```

**Lister toutes les mesures pour une session spécifique**

```nrql
FROM BrowserPerformance 
SELECT * 
WHERE entryType = 'measure' 
AND session = 'abcdefghijklmnop'
```

Utilisez ces requêtes ou créez les vôtres pour créer des dashboards personnalisés, configurer des alertes et monitorer les performances réelles de vos applications Web avec précision.
