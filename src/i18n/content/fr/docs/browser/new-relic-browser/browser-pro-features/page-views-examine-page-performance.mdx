La page <DNT>**Page views**</DNT> dans <InlinePopover type="browser" />donne un aperçu des performances des pages de votre site et de l&apos;expérience utilisateur. Les options de tri vous aident à trouver des détails sur le pourcentage du temps de chargement des pages, le temps de chargement moyen des pages ou le débit. Vous pouvez également afficher les signaux de frustration des utilisateurs et les mesures d&apos;impact sur les performances, accéder aux relectures de session et explorer des informations détaillées sur les pages Web les plus consultées.

Nos fonctionnalités de navigateur incluent [le monitoring des applications monopage (SPA)](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring). Si vous avez opté pour monitoring SPA, votre UI SPA <DNT>**Page views**</DNT> sera différente. Vous pouvez basculer entre la vue (SPA ou navigateur standard) que vous souhaitez.

## Afficher les informations récapitulatives [#dashboard]

monitoring des navigateurs répertorie les pages vues sous forme [d&apos;URL](/docs/browser/new-relic-browser/getting-started/url-whitelists-grouping-browser-metrics). Cela permet d&apos;identifier facilement des vues de pages frontales spécifiques ; par exemple, `/browse/plans`.

Pour afficher des informations détaillées sur les pages que l&apos;utilisateur final consulte pour votre application :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT>.
2. Basculez l&apos;affichage de la page pour les applications SPA ou de navigateur standard, selon le cas.
3. Sélectionnez le type d&apos;affichage pour les informations sur le temps de chargement de la page sous forme de [graphique (par défaut), d&apos;histogramme ou percentile](/docs/apm/apm-ui-pages/monitoring/response-time-chart-types-apm-browser/).

## Comprendre les mesures d&apos;impact utilisateur [#user-impact-metrics]

La page Page views inclut un volet d&apos;impact utilisateur qui vous aide à comprendre comment les problèmes de performances affectent l&apos;expérience de vos utilisateurs.

### Signaux de frustration [#frustration-signals]

Le <DNT>**Frustration signals**</DNT> affiche le pourcentage de sessions utilisateur contenant des indicateurs de frustration dans la période sélectionnée. Les signaux de frustration incluent :

* **Clics de rage**: les utilisateurs cliquent rapidement sur le même élément 4 fois ou plus en 1 seconde
* **Clics morts**: clics qui n&apos;entraînent aucun changement détectable dans les 2 secondes
* **Clics d&apos;erreur**: Clics suivis d&apos;une erreur JavaScript dans les 2 secondes

### Risque de performance [#performance-impact]

Le <DNT>**Performance risk**</DNT> affiche le pourcentage de sessions utilisateur affectées par des problèmes de performance pendant la période sélectionnée. Il affiche les exceptions non gérées et les erreurs d&apos;application pour les erreurs JavaScript.

### Considérations relatives à l&apos;échantillonnage de Session Replay [#sampling-considerations]

Les cartes d&apos;impact utilisateur s&apos;adaptent en fonction de votre configuration Session Replay :

<CollapserGroup>
  <Collapser id="sr-enabled" title="Session Replay activé avec un échantillonnage standard">
    Lorsque Session Replay est activé avec des taux d&apos;échantillonnage égaux ou supérieurs aux seuils par défaut (10 % pour l&apos;échantillonnage général, 100 % pour l&apos;échantillonnage des erreurs), les deux cartes affichent des mesures complètes avec des décomptes de session replay et des liens directs vers les sessions replay.
  </Collapser>

  <Collapser id="sr-low-sampling" title="Session Replay activé avec un faible échantillonnage">
    Lorsque les taux d&apos;échantillonnage sont inférieurs aux seuils recommandés, les cartes affichent un avis vous suggérant d&apos;augmenter votre taux d&apos;échantillonnage. Cliquez sur le lien « Augmenter votre taux d&apos;échantillonnage » pour accéder aux paramètres de votre application de navigateur.
  </Collapser>

  <Collapser id="sr-disabled" title="Relecture de session désactivée">
    Lorsque Session Replay est désactivé, les cartes n&apos;affichent que les nombres de sessions sans données de relecture. Un avis apparaît avec <DNT>See where your users are struggling</DNT> et un lien <DNT>Turn on session replays</DNT> pour activer la fonctionnalité.
  </Collapser>
</CollapserGroup>

## Examiner les détails des performances [#view\_details][#view_details]

À partir du résumé de la page vue, cliquez sur une URL pour afficher les détails des performances concernant le chargement de la page, le timing du frontend et backend dans [APM](/docs/apm/apm-ui-pages/features/request-queuing-tracking-front-end-time/) ou dans [le tracing distribué](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui), le débit, [les types de navigateur](/docs/browser/new-relic-browser/additional-standard-features/browsers-problem-patterns-type-or-platform) et [le tracing de session](/docs/browser/browser-monitoring/browser-pro-features/session-traces-explore-webpages-life-cycle).

<img title="Page view details" alt="A screenshot of the page view details UI in New Relic" src="/images/browser_screenshot-full_page-load-performance.webp" />

À partir d&apos;ici, vous pouvez également :

* Comparez les performances de chargement et de débit des pages avec celles d&apos;hier et de la semaine dernière.
* Afficher le temps de réponse pour l&apos;application temps de réponse pour les transactions Web.
