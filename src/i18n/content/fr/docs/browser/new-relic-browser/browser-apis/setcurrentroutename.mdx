## Syntaxe

```js
newrelic.setCurrentRouteName(string $name)
```

Donne aux itinéraires SPA des noms plus précis que les noms par défaut. Monitorer les itinéraires spécifiques plutôt que par regroupement par défaut.

## Exigences

* Agent navigateur Pro+SPA (v998 ou supérieur)

* Si vous utilisez npm pour installer l'agent du navigateur, vous devez activer la fonctionnalité `spa` lors de l&amp;apos;instanciation de la classe `BrowserAgent` . Dans l&amp;apos;éventail `features` , ajoutez ce qui suit :

  ```js
  import { Spa } from '@newrelic/browser-agent/features/spa';

  const options = {
    info: { ... },
    loader_config: { ... },
    init: { ... },
    features: [
      Spa
    ]
  }
  ```

  Pour plus d'informations, consultez la [documentation d'installation du navigateur npm](https://www.npmjs.com/package/@newrelic/browser-agent#new-relic-browser-agent).

## Description

Cette méthode nomme l'itinéraire actuel. Cela peut être utile pour :

* Donnez aux itinéraires des noms plus précis que ceux qu'ils auraient par défaut.
* Monitorer un itinéraire qui pourrait autrement être regroupé avec d'autres itinéraires par défaut.

Actuellement, cette méthode s'applique uniquement à l'événement SPA [`BrowserInteraction`](/docs/insights/explore-data/attributes/browser-default-attributes-insights#browserinteraction-attributes) . L&amp;apos;utilisation de cette API crée deux nouveaux attributs personnalisés qui suivent l&amp;apos;itinéraire précédent et l&amp;apos;itinéraire cible :

* `previousRouteName`
* `targetRouteName`

Lors de l'utilisation de cette API, l'attribut [`browserInteractionName`](/docs/insights/explore-data/attributes/browser-default-attributes-insights#interaction-name) prendra la valeur `targetRouteName` . Si `setName` est utilisé pour définir le nom de l&amp;apos;interaction du navigateur, celui-ci aura la priorité.

L'API `setCurrentRouteName()` détermine l&amp;apos;attribut name pour [l'événement`BrowserInteraction` ](/docs/insights/explore-data/attributes/browser-default-attributes-insights#browserinteraction-attributes), donc cette API doit être appelée <DNT>**every time**</DNT> s&amp;apos;il y a un changement d&amp;apos;itinéraire. Cela garantira que `BrowserInteraction` événement aura l&amp;apos;attribut correct.

<Callout variant="tip">
  Comparez avec [`setName()`](/docs/browser/new-relic-browser/browser-agent-apis/browser-spa-api-newrelicinteractionsetname), qui définit un nom pour une interaction de navigateur, pas un itinéraire.
</Callout>

## Exemples

```js
router.onChange(function(route) {
  newrelic.setCurrentRouteName(route.name);
});
```
