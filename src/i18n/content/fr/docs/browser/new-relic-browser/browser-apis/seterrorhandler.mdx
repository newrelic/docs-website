## Syntaxe

```js
newrelic.setErrorHandler(function $callback)
```

Permet d&apos;ignorer et de regrouper de manière sélective les erreurs connues que l&apos;agent du navigateur capture.

## Exigences

* Agent navigateur Pro ou Pro+SPA (v974 ou supérieur)

  * Pour la capacité de regroupement des erreurs, [la version 1.230.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.230.0) ou supérieure est requise.

* Si vous utilisez npm pour installer l&apos;agent du navigateur, vous devez activer la fonctionnalité `jserrors` lors de l&amp;apos;instanciation de la classe `BrowserAgent` . Dans l&amp;apos;éventail `features` , ajoutez ce qui suit :

  ```js
  import { JSErrors } from '@newrelic/browser-agent/features/jserrors';

  const options = {
    info: { ... },
    loader_config: { ... },
    init: { ... },
    features: [
      JSErrors
    ]
  }
  ```

  Pour plus d&apos;informations, consultez la [documentation d&apos;installation du navigateur npm](https://www.npmjs.com/package/@newrelic/browser-agent#new-relic-browser-agent).

## Description

L&apos;appel d&apos;API `newrelic.setErrorHandler()` vous permet d&amp;apos;ignorer de manière sélective les erreurs connues capturées par l&amp;apos;agent du navigateur. Il faut une seule fonction de gestion des erreurs, qui sera appelée pour chaque erreur capturée par l&amp;apos;agent du navigateur. Si le gestionnaire renvoie `true`, New Relic **n’enregistre pas** l’erreur. Sinon, l&amp;apos;erreur sera traitée normalement.

De plus, les versions ultérieures de l&apos;agent prennent en charge les empreintes ou le regroupement d&apos;exceptions avec une étiquette personnalisée fournie. Pour ce faire, renvoyez un objet au lieu d&apos;un booléen avec une propriété `group` définie sur la *chaîne* souhaitée. Il est important de savoir que fournir une chaîne vide, ou tout objet qui n&amp;apos;est pas conforme à cette spécification exacte, est traité de la même manière que le cas `true` , pour lequel l&amp;apos;erreur sera *ignorée*. Ce comportement est rétrocompatible avec les versions antérieures.

## Exemples

### Utiliser une fonction de gestion d&apos;erreurs de base [#include-error]

Incluez l’objet d’erreur à l’intérieur de la fonction de rappel pour ignorer les erreurs spécifiques capturées par l’agent du navigateur.

```js
newrelic.setErrorHandler(function(err) {
  if (shouldIgnoreError(err)) {
    return true;
  } else {
    return false;
  }
});
```

### Erreurs d&apos;empreintes digitales dans la fonction de gestionnaire

Attribuez des étiquettes personnalisées à des erreurs spécifiques pour les afficher dans l&apos;UI Errors Inbox.

```js
newrelic.setErrorHandler(function(err) {
  if (isReferenceError(err)) {
    return { group: 'My reference errors' }; // error is included and tagged under this label
  } else if (isSomeSpecificError(err)) {
    return { group: '' }; // error will be excluded!
    // return { Group: 'still excluded - prop name has capital G!' };
  } else {
    return false; // error is included without any label
  }
})
```
