## Syntaxe

```js
newrelic.setPageViewName(string $name[, string $host])
```

Regroupe les pages vues pour aider à la structure de l&apos;URL ou pour capturer les informations de routage de l&apos;URL.

## Exigences

* Agent navigateur Lite, Pro ou Pro+SPA (v593 ou supérieur)

* Si vous utilisez npm pour installer l&apos;agent du navigateur, vous devez activer au moins une fonctionnalité lors de l&apos;instanciation de la classe `BrowserAgent` . Par exemple, ajoutez ce qui suit dans l’éventail`features` :

  ```js
  import { Metrics } from '@newrelic/browser-agent/features/metrics'

  const options = {
    info: { ... },
    loader_config: { ... },
    init: { ... },
    features: [
      Metrics
    ]
  }
  ```

  Pour plus d&apos;informations, consultez la [documentation d&apos;installation du navigateur npm](https://www.npmjs.com/package/@newrelic/browser-agent#new-relic-browser-agent).

## Description

Les noms de page personnalisés peuvent vous aider à regrouper plus efficacement vos pages vues si la structure de votre URL ne fournit pas de regroupements utiles ou si l&apos;agent du navigateur ne capture pas la partie de l&apos;URL où les informations de routage sont stockées. Lors de l&apos;interrogation de l&apos;[événement](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) `PageView` , le nom personnalisé sera exposé comme attribut `browserTransactionName` . Le nom personnalisé sera également visible dans l&amp;apos;UI <InlinePopover type="browser" />.

Pour utiliser un [nom de vue de page](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity) personnalisé au lieu de l&amp;apos;URL de la page, formatez le nom sous forme de chaîne délimitée par des barres obliques. Effectuez cet appel avant que l’événement `window load` ne se déclenche afin qu’il apparaisse correctement.

<Callout variant="tip">
  Cet appel d&apos;API s&apos;applique aux données dans les vues de page standard du navigateur et à l&apos;événement [`PageView`](/docs/insights/insights-data-sources/default-data/browser-default-events-attributes-insights#browser-attributes-table) . Pour définir un nom personnalisé pour les pages vues SPA et l&amp;apos;événement `BrowserInteraction` , voir [SPA : setName](/docs/browser/new-relic-browser/browser-agent-spa-api/spa-set-name). Il est recommandé d&amp;apos;utiliser les deux appels ensemble.
</Callout>

## Exemples

```js
newrelic.setPageViewName('/login')
// Or
newrelic.setPageViewName('/login', 'https://www.myapp.com')
```
