## Syntaxe

```js
newrelic.start()
```

API Browser utilisée pour démarrer la fonctionnalité de l&apos;agent lors de l&apos;exécution dans un état différé.

## Exigences

* Agent navigateur Lite, Pro ou Pro+SPA (v1.239.0 ou supérieur)

  <Callout variant="important">
    La configuration requise pour utiliser cette API n&apos;est actuellement pas connectée au système de déploiement plus vaste de New Relic. En tant que tel, l&apos;appel de cette API n&apos;aura d&apos;effet que sur les installations par copier/coller ou par navigateur npm jusqu&apos;à ce que d&apos;autres modifications soient apportées.
  </Callout>

## Description

La fonctionnalité peut être chargée dans un état `deferred`, qui peut être contrôlé en définissant la propriété `autoStart` de la fonctionnalité appropriée sur `false` dans le bloc configuration `NREUM.init.<feature>` utilisé par l&amp;apos;agent. Cet état de fonctionnalité signifie que l&amp;apos;événement sera observé et stocké dans l&amp;apos;agent, mais *ne sera pas récolté sur NR1 jusqu&apos;à ce qu&apos;il soit indiqué de le faire* avec la méthode API `.start()` . Voir [Noms des fonctionnalités](#feature-names) pour une liste des noms de fonctionnalités. Voir [Exemples](#examples) pour des exemples montrant comment définir une fonctionnalité dans un état différé.

Lors de l&apos;exécution de cette fonction avec une valeur valide, l&apos;agent du navigateur démarrera la fonctionnalité pertinente qui a été différée par la configuration `autoStart: false`. Lorsqu&amp;apos;elle est appelée, la méthode démarrera toutes les fonctionnalités qui ont été différées. Consultez [les noms de fonctionnalités](#feature-names) pour obtenir une liste des noms de fonctionnalités qui peuvent être démarrés dans un état différé.

## Noms des fonctionnalités

* ajax

* jserrors

* metrics

* generic\_events

* page\_view\_event

* page\_view\_timing

* session\_replay

* session\_trace

* spa

  <Callout variant="important">
    Le corps de la réponse de la récolte `page_view_event` contient des informations critiques pour l&amp;apos;exécution de l&amp;apos;autre fonctionnalité. Le report du début de `page_view_event` entraînera l&amp;apos;attente de toutes les autres fonctionnalités jusqu&amp;apos;à ce que `.start()` soit appelé.
  </Callout>

## Cas d&apos;utilisation

* Vous souhaitez attendre de récolter des données jusqu&apos;à ce qu&apos;un utilisateur ait donné son consentement avec une fenêtre modale sur votre site
* Vous souhaitez attendre pour récolter des données que votre site ait défini des attributs personnalisés

## Exemples

### Chargement de toutes les fonctionnalités dans un état différé (copier/coller, NPM)

```js
// Change the init portion of the configuration block to add `autoStart: false` to any features desired to load in a deferred state
NREUM.init = {
  // feature configurations
  ajax: {autoStart: false},
  jserrors: {autoStart: false},
  metrics: {autoStart: false},
  generic_events: {autoStart: false},
  page_view_event: {autoStart: false},
  page_view_timing: {autoStart: false},
  session_replay: {autoStart: false},
  session_trace: {autoStart: false},
  spa: {autoStart: false},
  // other configurations
  // ...
}
```

### &quot;Démarrage&quot; toutes fonctionnalités différées

```js
newrelic.start()
// all deferred features in the agent will now start harvesting
```
