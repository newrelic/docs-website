## Syntaxe

```js
newrelic.setCustomAttribute(name: string, value: string|number|boolean|null[, persist: boolean])
```

Ajoute un nom et une valeur [d&apos;attribut personnalisé](/docs/data-apis/custom-data/custom-events/collect-custom-attributes/) défini par l&amp;apos;utilisateur à l&amp;apos;événement ultérieur sur la page.

## Exigences

* Agent navigateur Lite, Pro ou Pro+SPA (v593 ou supérieur)

  * Pour la prise en charge du paramètre `persist` ou de la valeur `null` , la version de l&amp;apos;agent 1.230.0 ou supérieure est requise.
  * Pour la prise en charge de la valeur `boolean` , la version de l&amp;apos;agent 1.245.0 ou supérieure est requise.

* Si vous utilisez npm pour installer l&apos;agent du navigateur, vous devez activer au moins une fonctionnalité lors de l&apos;instanciation de la classe `BrowserAgent` . Par exemple, ajoutez ce qui suit dans l’éventail`features` :

  ```js
  import { Metrics } from '@newrelic/browser-agent/features/metrics'

  const options = {
    info: { ... },
    loader_config: { ... },
    init: { ... },
    features: [
      Metrics
    ]
  }
  ```

Pour plus d&apos;informations, consultez la [documentation d&apos;installation du navigateur NPM](https://www.npmjs.com/package/@newrelic/browser-agent#new-relic-browser-agent).

## Description

Effectuez cet appel avant que l&apos;événement de chargement de la fenêtre ne se déclenche (lorsque ces données sont transmises) afin que l&apos;attribut soit inclus dans l&apos;événement [`PageView`](/docs/insights/explore-data/attributes/browser-default-attributes-insights#browser-attributes-table) . Une fois qu&amp;apos;un attribut est défini, la plateforme New Relic l&amp;apos;enregistre avec tous les événements jusqu&amp;apos;à ce que la page soit rechargée ou que l&amp;apos;attribut soit désactivé manuellement.

If you are using [SPA monitoring](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring) with a compatible agent version, attributes set with this call will also be included in `BrowserInteraction` events. However, attributes set using the SPA [setAttribute](/docs/browser/new-relic-browser/browser-agent-spa-api/spa-set-attribute) method will take precedence over these attributes.

Les erreurs pour l&apos;attribut personnalisé seront incluses dans l&apos;événement sur la [page Erreurs JS](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors). Pour afficher ou log les erreurs d&amp;apos;un attribut personnalisé via l&amp;apos;API, utilisez l&amp;apos;appel [`noticeError`](/docs/browser/new-relic-browser/browser-agent-spa-api/newrelicnoticeerror-browser-agent-api) de l&amp;apos;API Browser.

Currently, all features or browser events will have these global attributes, except for `SessionReplay` and `BrowserSessionChunk` (Session Trace).

Avec l&apos;indicateur `persist` , l&amp;apos;attribut peut également être stocké dans le navigateur, de sorte que les visites de pages ultérieures de la *même* origine <DNT>**within a session**</DNT> le conservent lors de l&amp;apos;événement. Veuillez noter que cette fonctionnalité peut varier en fonction des paramètres de confidentialité du navigateur de l&amp;apos;utilisateur final. Si cette fonction est appelée avec un `value = null`, l&amp;apos;attribut sera supprimé de l&amp;apos;événement <DNT>**both**</DNT> de la page courante et du stockage, *quel que soit* le flag `persist` .

<Callout variant="important">
  Soyez conscient que les attributs persistants ont priorité sur `info.jsAttributes` clés du même nom ! Par exemple, un attribut persistant `someName` défini sur `somedomain.com/pageA` remplacera tout `someName` défini statiquement sur le bloc d&amp;apos;informations de `somedomain.com/pageB`, en supposant qu&amp;apos;ils partagent le même stockage (session).
</Callout>

## Exemples

### Obtenir JavaScript/jQuery pour les éléments HTML [#jquery-example]

Cet exemple utilise JavaScript/jQuery pour obtenir les valeurs des éléments HTML suivants sur une page générée par Drupal :

* `<link rel="shortlink" href="/node/1111" />`
* `<h1>Using NRQL</h1>`

New Relic les signale comme attribut personnalisé. Ceci est utile pour interroger les événements `PageView` et `PageAction` .

```js
var node_id = jQuery("link[rel='shortlink']").attr("href");
var node_title = jQuery('h1').text();

if (typeof newrelic == 'object') {
  newrelic.setCustomAttribute('nodeId', node_id);
  newrelic.setCustomAttribute('title', node_title);
}
```
