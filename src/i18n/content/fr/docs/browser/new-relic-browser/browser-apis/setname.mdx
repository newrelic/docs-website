## Syntaxe

```js
newrelic.interaction().setName(string $name[, string $trigger])
```

Définit le nom et le déclencheur de l&apos;interaction du navigateur d&apos;un SPA qui n&apos;est pas un changement d&apos;itinéraire ou un changement d&apos;URL.

## Exigences

* Agent navigateur Pro+SPA (v963 ou supérieur)

* Si vous utilisez npm pour installer l&apos;agent du navigateur, vous devez activer la fonctionnalité `spa` lors de l&amp;apos;instanciation de la classe `BrowserAgent` . Dans l&amp;apos;éventail `features` , ajoutez ce qui suit :

  ```js
  import { Spa } from '@newrelic/browser-agent/features/spa';

  const options = {
    info: { ... },
    loader_config: { ... },
    init: { ... },
    features: [
      Spa
    ]
  }
  ```

  Pour plus d&apos;informations, consultez la [documentation d&apos;installation du navigateur npm](https://www.npmjs.com/package/@newrelic/browser-agent#new-relic-browser-agent).

## Description

Cette méthode monitoring SPA définit le nom et le déclencheur d&apos;une interaction avec le navigateur. Le nom sera exposé comme l&apos;attribut [`browserInteractionName`](/docs/insights/explore-data/attributes/browser-default-attributes-insights#interaction-name) dans l&amp;apos;événement `BrowserInteraction` . Il sera également utilisé pour le regroupement dans l&amp;apos;UI.

Par défaut, `browserInteractionName` est nommé d&amp;apos;après une URL ou un itinéraire associé. Utilisez `setName()` lorsque vous souhaitez enregistrer une interaction qui n’est pas un changement d’itinéraire ou un changement d’URL.

La méthode [`setCurrentRouteName()`](/docs/browser/new-relic-browser/browser-agent-apis/browser-spa-api-newrelicsetcurrentroutename) nomme également l&amp;apos;itinéraire actuel. Lorsque vous utilisez les deux :

* `setName()` a la priorité pour nommer l&amp;apos;interaction.
* Les attributs `previousRouteName` et `targetRouteName` sont toujours définis avec des valeurs transmises à `setCurrentRouteName()`.

L&apos;utilisation conjointe de ces méthodes permet de nombreuses options de filtrage et de regroupement dans l&apos;UI. Par exemple, vous pouvez :

* Filtrer jusqu&apos;à l&apos;interaction d&apos;un certain type, tel que `LikeButtonClick`.
* Ensuite, regroupez par `targetRouteName` pour voir quels itinéraires ont le plus d&amp;apos;interaction `LikeButtonClick` .

<Callout variant="tip">
  Cet appel d&apos;API s&apos;applique aux données dans [les pages vues SPA](/docs/browser/single-page-app-monitoring/get-started/introduction-single-page-app-monitoring) dans le navigateur et au type d&amp;apos;événement [`BrowserInteraction`](/docs/insights/insights-data-sources/default-data/browser-default-events-attributes-insights#browserinteraction-attributes) . Pour définir un nom personnalisé pour les vues de page standard et le type d&amp;apos;événement <DNT>**PageView**</DNT> , voir [`setPageViewName`](/docs/browser/new-relic-browser/browser-agent-spa-api/set-pageview-name). Il est recommandé d&amp;apos;utiliser les deux appels ensemble.
</Callout>

## Valeurs de retour

Cette méthode renvoie le même objet API créé par `interaction()`.

## Exemples

```js
document.getElementById('subscribe').addEventListener('submit', () => {
  newrelic.interaction().setName('createSubscription');
  createSubscription();
});
```
