Les pages peuvent être mises en cache dans le navigateur ou sur un serveur. Une page mise en cache contiendra des temps d&apos;application et de file d&apos;attente obsolètes (générés précédemment).

## Détection des pages mises en cache [#browser-traces]

Notre <InlinePopover type="browser" />détecte les pages mises en cache en comparant le temps total du backend avec le temps de l&amp;apos;application et de la file d&amp;apos;attente. Lorsque la somme du temps d&amp;apos;application et de la file d&amp;apos;attente est supérieure au temps total backend, monitoring des navigateurs suppose que la page a été mise en cache et agrège la demande avec les temps d&amp;apos;application et de file d&amp;apos;attente comme 0.

Voici quelques conseils supplémentaires pour les pages en cache :

* <DNT>**Java**</DNT>:Vider le cache « travail » du serveur d&amp;apos;applications. Cela force le serveur d&amp;apos;applications à recompiler.
* <DNT>**.NET**</DNT>: Assurez-vous que votre répertoire de cache <DNT>**asp.net**</DNT> est propre en utilisant la commande `flush_dotnet_temp.cmd`. Cela force le serveur d&amp;apos;applications à recompiler les ASP avec une instrumentation de synchronisation de chargement de page.
* <DNT>**Node.js**</DNT>:Par défaut, Node.js ne met pas en cache les pages.
* <DNT>**PHP**</DNT>:Si vous disposez d&amp;apos;un cache CDN pour vos pages dynamiques, assurez-vous que le cache contient l&amp;apos;instrumentation de synchronisation de chargement de la page.
* <DNT>**Python**</DNT>: Redémarrez votre application.
* <DNT>**Ruby**</DNT>:Consultez les informations d&amp;apos;en-tête pour l&amp;apos;instrumentation automatique et manuelle. Par exemple, si vous souhaitez ignorer des pages Web spécifiques, vous devez désactiver monitoring automatique des navigateurs à partir de l&amp;apos; [interface utilisateur](/docs/browser/new-relic-browser/getting-started/browser-settings), puis instrumenter manuellement les pages Web que vous souhaitez baliser.
