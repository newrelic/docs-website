### Stockage de session Browser [#browser-session-storage]

Nous avons consolidé le contrôle de collecte des données de trace de session et de relecture de session dans l&apos;agent du Browser. Vous pouvez désormais gérer toutes les données au niveau de la session directement à partir du bouton <DNT>**Access browser local storage**</DNT> sur cette page de paramètres.

* **Basculement pour l&apos;accès au stockage local du navigateur (Access browser local storage)**: ce contrôle unifié détermine la manière dont les données au niveau de la session (y compris les ID de session et d&apos;autres attributs au niveau de la session) sont collectées.

  * **Lorsqu&apos;il est activé (par défaut)**: l&apos;agent du navigateur New Relic utilise les mécanismes de stockage persistant du navigateur pour collecter les données trace de session et de relecture de session.
  * **Lorsqu&apos;il est désactivé**: l&apos;agent du navigateur New Relic arrête immédiatement son utilisation du stockage persistant du navigateur. Cette action empêche la collecte de toutes les données trace de session et de relecture de session, vous donnant un contrôle direct sur la gestion des informations liées à la session par le navigateur et améliorant votre capacité à répondre aux exigences de confidentialité et de conformité des données.

Pour accéder à la page <DNT>**Application settings**</DNT> :

1. Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT> et sélectionnez une application.
2. Dans la navigation de gauche, faites défiler jusqu&apos;à <DNT>**Settings**</DNT> et cliquez sur <DNT>**Application settings**</DNT>.
