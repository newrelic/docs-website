### Stockage de session Browser [#browser-session-storage]

Nous avons consolidé le contrôle de collecte des données de trace de session et de relecture de session dans l&apos;agent du Browser. Vous pouvez désormais gérer toutes les données au niveau de la session directement à partir du bouton <DNT>**Access browser local storage**</DNT> sur cette page de paramètres.

{/* * **Access browser local storage toggle**: This unified control determines how session-level data (including session IDs and other session-level attributes) is collected.
    * **When turned ON (default)**: The New Relic browser agent uses the browser's persistent storage mechanisms to collect session trace and session replay data.
    * **When turned OFF**: The New Relic browser agent immediately halts its use of the browser's persistent storage. This action prevents the collection of all session trace and session replay data, giving you direct control over the browser's handling of session-related information and enhancing your ability to meet data privacy and compliance requirements. */}

<Callout variant="important">
  À partir de la version 1.303.0 de l&apos;agent navigateur, le bouton d&apos;activation/désactivation de **Access local storage** est supprimé de l&apos;interface utilisateur afin de simplifier notre fonctionnalité de confidentialité et de consentement.

  * La méthode recommandée pour gérer la collecte de données des agents de browser et le consentement de l&apos;utilisateur est la configuration du mode de consentement. Pour plus d&apos;informations, consultez [la documentation sur le consentement](/docs/browser/new-relic-browser/configuration/consent-mode/).

  * Si vous avez actuellement le commutateur de **Access local storage** défini sur `OFF`, vous constaterez que les commutateurs du mode de consentement, de la relecture de session et trace de session sont également inactifs. Pour activer ces fonctionnalités, vous devez définir l&apos;attribut `cookie_enabled` sur `true` via [l&apos;API NerdGraph](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-browser-application). Cette action réactivera les boutons de l&apos;interface utilisateur, vous permettant de configurer correctement le mode de consentement, la relecture de session et trace de session.
</Callout>

## API de Browser [#apis]

Vous pouvez également personnaliser instrumentation de votre navigateur d&apos;agent pour capturer des traces et des événements personnalisés, signaler des erreurs spécifiques adaptées à vos besoins et affiner le timing de collecte des données pour des performances optimales. Consultez [Utilisation des API de navigateur](/docs/browser/new-relic-browser/browser-apis/using-browser-apis/) pour commencer.
