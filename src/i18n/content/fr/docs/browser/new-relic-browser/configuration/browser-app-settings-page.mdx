### Stockage de session Browser [#browser-session-storage]

Nous avons consolidé le contrôle de collecte des données de trace de session et de relecture de session dans l&apos;agent du Browser. Vous pouvez désormais gérer toutes les données au niveau de la session directement à partir du bouton <DNT>**Access browser local storage**</DNT> sur cette page de paramètres.

/\* \* \*\*Activer/désactiver l&apos;accès au stockage local du navigateur\*\* : Ce contrôle unifié détermine comment les données au niveau de la session (y compris les ID de session et d&apos;autres attributs au niveau de la session) sont collectées. \* \*\*Lorsqu&apos;il est activé (par défaut)\*\* : L&apos;agent Browser de New Relic utilise les mécanismes de stockage persistant du navigateur pour collecter les données trace de session et de relecture de session. \* \*\*Lorsqu&apos;il est désactivé\*\* : l&apos;agent Browser de New Relic cesse immédiatement d&apos;utiliser le stockage persistant du navigateur. Cette action empêche la collecte de toutes les données trace de session et de relecture de session, vous donnant un contrôle direct sur la gestion par le navigateur des informations relatives à la session et renforçant votre capacité à respecter les exigences en matière de confidentialité et de conformité des données. \*/

<Callout variant="important">
  À partir de la version 1.303.0 de l&apos;agent navigateur, le bouton d&apos;activation/désactivation de **Access local storage** est supprimé de l&apos;interface utilisateur afin de simplifier notre fonctionnalité de confidentialité et de consentement.

  * La méthode recommandée pour gérer la collecte de données de l&apos;agent browser et le consentement de l&apos;utilisateur consiste à utiliser la configuration du mode de consentement. Nous encourageons fortement tous les utilisateurs à adopter cette nouvelle fonctionnalité. Pour plus d&apos;informations, veuillez consulter [les documents de consentement](/docs/browser/new-relic-browser/configuration/consent-mode/).

  * Si vous avez actuellement le commutateur de **Access local storage** défini sur `OFF`, vous constaterez que les commutateurs du mode de consentement, de la relecture de session et trace de session sont également inactifs. Pour activer ces fonctionnalités, vous devez définir l&apos;attribut `cookie_enabled` sur `true` via [l&apos;API NerdGraph](/docs/apis/nerdgraph/examples/browser-monitoring-config-nerdgraph/#configure-browser-application). Cette action réactivera les boutons de l&apos;interface utilisateur, vous permettant de configurer correctement le mode de consentement, la relecture de session et trace de session.
</Callout>

Allez à &lt;DNT&gt;\*\*\[one.newrelic.com &gt; All capabilities]\(https\://one.newrelic.com/all-capabilities) &gt; Browser\*\*&lt;/DNT&gt;, sélectionnez une application. 2. Dans le volet de navigation de gauche, faites défiler vers le bas jusqu&apos;à &lt;DNT&gt;\*\*Settings\*\*&lt;/DNT&gt; et cliquez sur &lt;DNT&gt;\*\*Application settings\*\*&lt;/DNT&gt;. \*/

## API de Browser [#apis]

Vous pouvez également personnaliser instrumentation de votre navigateur d&apos;agent pour capturer des traces et des événements personnalisés, signaler des erreurs spécifiques adaptées à vos besoins et affiner le timing de collecte des données pour des performances optimales. Consultez [Utilisation des API de navigateur](/docs/browser/new-relic-browser/browser-apis/using-browser-apis/) pour commencer.
