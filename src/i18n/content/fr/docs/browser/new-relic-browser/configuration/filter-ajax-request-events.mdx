Dans la [version 1211 de l&apos;agent](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1211/) ou supérieure, toutes les requêtes réseau effectuées par une page sont enregistrées en tant qu&apos;événements `AjaxRequest`. Vous pouvez utiliser les options de configuration de la liste de refus dans la page <DNT>**Application settings**</DNT> pour filtrer les requêtes qui enregistrent des événements.

## Utilisation de la liste de refus

Par défaut, nous enregistrons les événements `AjaxRequest` pour toutes les requêtes réseau effectuées par votre application.

* Pour arrêter l&apos;enregistrement de tous les événements, ajoutez un caractère générique `*` (astérisque).
  * Exemple: `*`
* Pour arrêter l&apos;enregistrement des événements pour un site complet, ajoutez le nom de domaine.
  * Exemple: `mywebsite.com`
* Pour arrêter l&apos;enregistrement des événements pour une page spécifique, ajoutez le domaine et le chemin d&apos;accès.
  * Exemple: `mywebsite.com/photos`
* À partir de la [version 1.307.0 de l&apos;agent](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.307.0/), partout où nous trouvons un `*` (astérisque), nous ferons correspondre n&apos;importe quelle longueur de chaîne.
  * Exemple : `mywebsite.com/shop/*` (pour un chemin), `*.mywebsite.com` (pour un sous-domaine), `mywebsite.com/*/shop/*/checkout` (pour les chaînes de milieu d&apos;URL).

Le protocole, le port, la recherche et le hachage d&apos;une URL sont ignorés par la liste de refus.

## Accéder à la liste de refus

Pour mettre à jour la liste de refus des URL que votre application filtrera pour empêcher la création d&apos;événements, accédez à la page des paramètres de l&apos;application :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Settings &gt; Application settings**</DNT>.
2. Sous <DNT>**Ajax Request Deny List**</DNT>, ajoutez les filtres que vous souhaitez appliquer à votre application.
3. Sélectionnez <DNT>**Save application settings**</DNT> pour mettre à jour la configuration de l&apos;agent.
4. [Redéployez l&apos;agent du navigateur](/docs/browser/new-relic-browser/installation/upgrade-browser-agent) (en redémarrant l&apos;agent <InlinePopover type="apm" />associé ou en mettant à jour l&apos;installation du navigateur par copier/coller).

## Validation

Pour valider si les filtres que vous avez ajoutés fonctionnent comme prévu, exécutez une requête NRQL pour `AjaxRequest` événement correspondant à votre filtre.

```sql
FROM AjaxRequest SELECT * WHERE requestUrl LIKE `%example.com%`
```
