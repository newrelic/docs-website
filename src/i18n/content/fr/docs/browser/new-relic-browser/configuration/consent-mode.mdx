## À quoi sert cette fonctionnalité ?

Cela permet d&apos;intégrer les options de consentement à la politique de confidentialité pour vos utilisateurs concernant la manière dont les applications Web collectent, transmettent et traitent les données, sous le monitoring de l&apos;agent du browser. En activant cette fonctionnalité, vous permettez à l&apos;utilisateur de décider s&apos;il consent à autoriser l&apos;agent du browser à envoyer des données sur le réseau.

Une nouvelle propriété est disponible dans l&apos;objet d&apos;initialisation de l&apos;agent du browser appelée `browser_consent_mode`. Ce paramètre détermine si le mode de consentement doit être utilisé dans l&apos;agent. Disponible dans l&apos;agent Browser Lite, Browser Pro ou Pro + SPA, [version 1.303.0](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes/browser-agent-v1.303.0) ou supérieure.

## Comment l&apos;utiliser ?

Pour utiliser cette fonctionnalité, vous devrez vous rendre sur la page des paramètres de votre application et activer le bouton `"Turn on consent"`. Ce paramètre est désactivé par défaut.

NerdGraph peut être utilisé comme alternative pour activer et désactiver ce paramètre. Pour ce faire, exécutez une requête de mutation sur l&apos;objet graphQL `browserSettings` pour la propriété `browser_consent_mode`, en définissant `enabled` sur `true` ou `false`.

Une fois que vous avez intégré une politique de confidentialité ou une fenêtre modale d&apos;inscription à votre application Web, alors si un utilisateur clique sur l&apos;un des éléments suivants :

1. « Oui, je consens. »

   Vous devrez faire appel à une nouvelle API appelée `consent(accept?: boolean)`. Lors de l&apos;appel à l&apos; [API consent()](/docs/browser/new-relic-browser/browser-apis/consent), le paramètre accept est défini sur true par défaut.

   La fonctionnalité Browser avec **autoStart: true** enverra des données à New Relic après consentement.

   La fonctionnalité Browser avec **autoStart: false** devra être invoquée à l’aide de la fonction [API start()](/docs/browser/new-relic-browser/browser-apis/start) pour démarrer la fonctionnalité d’agent différé afin d’envoyer des données sur le réseau.

2. « Non, je n’y consens pas. »

   Puisque la valeur par défaut est vraie, vous devrez appeler `consent(false)` en prenant note de l&apos;argument faux. L&apos;agent n&apos;enverra aucune donnée sur le réseau pour cet utilisateur.

## Considérations supplémentaires

* Lorsque `browser_consent_mode` est activé, jusqu&apos;à \~30 secondes et un maximum de 1 Mo de données seront mis en mémoire tampon dans le navigateur de l&apos;utilisateur final et ne seront pas envoyés à New Relic jusqu&apos;à ce que l&apos;utilisateur sélectionne le consentement et que l&apos;API de consentement soit appelée. En raison de ces contraintes, il est possible que certaines données soient perdues et ne soient pas envoyées à New Relic lors de l&apos;appel de l&apos;API de consentement.

* Si [le stockage de session](/docs/browser/new-relic-browser/configuration/browser-app-settings-page/#browser-session-storage) est activé, les décisions transmises à l&apos;API de consentement seront conservées d&apos;une page à l&apos;autre.

* Si [le stockage de session](/docs/browser/new-relic-browser/configuration/browser-app-settings-page/#browser-session-storage) est désactivé, l&apos;API de consentement devra être appelée à chaque chargement de page.
