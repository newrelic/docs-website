De bons <InlinePopover type="alerts" />aident à notifier les individus ou les équipes des changements dans leurs systèmes. [Alertes](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) fournit un outil d&apos;alerte unique et coordonné pour l&apos;ensemble des produits New Relic. Cela vous permet de gérer des [politiques d&apos;alerte](/docs/alerts/overview#alert-policy) et des [conditions](/docs/alerts/overview#alert-condition) axées sur les métriques des [entités](/docs/alerts/overview#alert-entity) (telles que les applications de navigateur ou les transactions clés) qui vous intéressent le plus.

Après avoir [configuré une ou plusieurs politiques d&apos;alerte](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/) et [défini des conditions d&apos;alerte](/docs/alerts/create-alert/create-alert-condition/alert-conditions) pour vos [entités produit (cibles)](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition), vous pouvez consulter ces informations dans l&apos;interface utilisateur des alertes. Vous pouvez également consulter les informations sur les politiques d&apos;alerte dans l&apos;interface utilisateur pour des produits individuels.

## Afficher les politiques et conditions à partir du navigateur [#policy]

Pour afficher la règle d&apos;alerte et les informations sur les conditions d&apos;une application de navigateur ou d&apos;une clé de transaction spécifique :

1. Application Browser : accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Settings &gt; Application settings**</DNT>.

   OU

   Clé de transaction : Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Key transactions &gt; (select a key transaction) &gt; Settings &gt; Alert conditions**</DNT>.

2. Depuis la page <DNT>**Alert conditions**</DNT> , utilisez les outils disponibles pour rechercher, trier, afficher ou mettre à jour la condition d&apos;alerte et les politiques associées.

## Afficher les événements et activités depuis le navigateur [#activity]

Pour afficher des informations récapitulatives sur les [événements](/docs/alerts/overview#event) et d&apos;autres activités :

1. Application Browser : Accéder à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser**</DNT>

   OU

   Clé de transaction : Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Key transactions**</DNT>.

2. Depuis l&apos;index, survolez le [statut de santé codé par couleur](/docs/alerts/overview#health-status) de l&apos;entité, sélectionnez un lien dans la liste <DNT>**Browser activity**</DNT> ou sélectionnez une application de navigateur ou une transaction clé spécifique pour afficher des détails supplémentaires.

Par exemple, si un événement d&apos;alerte critique survient :

* L&apos;indicateur d&apos;état de santé devient rouge sur l&apos;index du navigateur et sur l&apos;application de navigateur sélectionnée.
* La couleur d’arrière-plan de divers graphiques devient rose.
* La liste <DNT>**Browser activity**</DNT> sur l&apos;index du navigateur affiche les événements d&apos;alerte Avertissement (jaune) et Critique (rouge) le cas échéant. Pour en savoir plus sur l&apos;événement d&apos;alerte dans les alertes, survolez ou sélectionnez n&apos;importe quelle zone rose sur un graphique. Pour plus d&apos;informations et d&apos;exemples, consultez [Afficher les événements de leurs produits](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products).

## Voir les événements d&apos;alerte depuis le navigateur [#alertevents]

Si une condition d&apos;alerte a des [seuils](/docs/alerts/overview#alert-threshold) définis pour des événements d&apos;alerte Avertissement (jaune) ou Critique (rouge), l&apos;[état de santé](/docs/alerts/overview#health-status) codé par couleur d&apos;une entité de produit changera pour indiquer un événement d&apos;alerte. Vous pouvez consulter les événements d&apos;alerte directement depuis la page de l&apos;application de navigateur ou de la transaction clé dans Browser :

1. Application browser : rendez-vous sur <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select a browser app)**</DNT> et consultez les événements d&apos;alerte dans le panneau de droite.

   Clé de transaction : Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Key transactions &gt; (select a key transaction) &gt; Issues &amp; Activity**</DNT> et cliquez sur l’onglet <DNT>**Alert events**</DNT> .

2. Depuis la page <DNT>**Alert events**</DNT>, utilisez les outils disponibles pour rechercher, trier ou afficher les événements d&apos;alerte pour l&apos;application de navigateur ou la transaction clé sélectionnée.

3. Facultatif : pour consulter ou mettre à jour les politiques d&apos;alerte, les conditions ou les seuils qui ont déclenché les événements d&apos;alerte, sélectionnez le lien correspondant.

<Callout variant="tip">
  Vous pouvez également consulter les événements d&apos;alerte Avertissement et Critique depuis l&apos;interface des alertes pour des [événements d&apos;alerte spécifiques](/docs/alerts/alert-event-management/view-alert-event-details) ou pour des [événements d&apos;alerte et des événements sur l&apos;ensemble des produits](/docs/alerts/alert-event-management/view-alert-event-details). Alerts envoie des [notifications d&apos;alerte](/docs/alerts/get-notified/destinations) uniquement pour les événements d&apos;alerte critiques (rouges).
</Callout>
