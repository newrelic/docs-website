[Le navigateur](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) vous permet de choisir le domaine ou les sous-domaines qu&apos;une application de navigateur monitore. Cela est utile pour limiter monitoring uniquement à l’activité d’application que vous considérez comme importante. Par exemple, si vous utilisez la [méthode copier/coller](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) pour déployer votre application de navigateur, vous pouvez définir des conditions de domaine pour protéger la sécurité de votre [clé de licence<InlinePopover type="browser" />et de votre ID d&apos;application](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id).

<Callout variant="tip">
  Les conditions de domaine filtrent uniquement les données en fonction de l&apos;URL de la page signalée. Les filtres de condition de domaine ne s&apos;appliquent pas aux données AJAX signalées à New Relic. Pour filtrer `AjaxRequest` événement, lisez [notre documentation configuration ](/docs/browser/new-relic-browser/configuration/filter-ajax-request-events/).
</Callout>

## Configurer les conditions du domaine [#configure]

Pour afficher ou modifier les paramètres de condition de votre domaine :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Settings &gt; Application settings &gt; Domain conditions**</DNT>.

2. S&apos;il n&apos;y a aucune condition de domaine en place, sélectionnez <DNT>**Enable domain conditions**</DNT>. Si les conditions existent, sélectionnez <DNT>**Edit conditions**</DNT>.

3. Sélectionnez l&apos;option permettant de créer une liste de domaines autorisés ou de créer une liste de domaines bloqués/refusés.

4. Sélectionnez les conditions du domaine (maximum 10 conditions).

5. Sélectionnez les conditions du domaine avec la condition <DNT>**Starts with**</DNT> , puis entrez l&apos;URL avec le sous-domaine (maximum 10 conditions).

   <img title="browser_domain_conditions.png" alt="browser_domain_conditions.png" src="/images/browser_screenshot-full_domain-settings.webp" />

   <figcaption>
     <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select a browser app) &gt; Settings &gt; Domain conditions:**</DNT> Utilisez cette fonctionnalité pour limiter monitoring uniquement à l&apos;activité importante de votre application ou pour améliorer la sécurité de votre clé de licence de navigateur et de l&apos;ID d&apos;application.
   </figcaption>

6. Vérifiez et confirmez les paramètres de condition de votre domaine.

Une zone de texte d&apos;aperçu affiche en temps réel quels domaines correspondent aux conditions que vous avez définies. Vous verrez également une estimation de l’impact que les paramètres proposés auront sur vos données.

## Désactiver les conditions de domaine [#disable]

Pour désactiver les conditions de domaine et revenir à monitoring de tous les domaines :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Settings &gt; Domain conditions**</DNT>.
2. Définissez l’option <DNT>**Enable domain conditions**</DNT> sur <DNT>**Off**</DNT>.
