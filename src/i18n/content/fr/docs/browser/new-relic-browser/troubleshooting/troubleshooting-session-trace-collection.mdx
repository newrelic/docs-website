## Problème

Vous ne voyez [aucune trace de session](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle) pour votre application de navigateur.

## Solution

Si votre application est [instrumentée avec APM](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation) et que d&amp;apos;autres [fonctionnalités associées à l&apos;agent de navigateur Pro](/docs/browser/new-relic-browser/browser-pro-features) fonctionnent, suivez ces étapes :

1. Vérifiez si vous [utilisez plusieurs noms pour votre application](/docs/apm/new-relic-apm/installation-configuration/using-multiple-names-app). les traces de session sont écrites uniquement dans l&amp;apos;application principale.
2. Si vous utilisez plusieurs noms, vérifiez les noms de vos applications <InlinePopover type="apm" />pour vérifier que l&amp;apos;application est répertoriée en premier.
3. Si l&apos;application n&apos;est pas répertoriée en premier, recherchez plutôt trace de session dans le nom de l&apos;application répertorié en premier.

D’autres raisons pour lesquelles vous pouvez ne pas voir les données de trace de session incluent :

* L&apos;utilisateur final n&apos;utilise pas de navigateurs prenant en charge l&apos; [API Resource Timing](http://www.w3.org/TR/resource-timing/).
* L&apos;utilisateur final ne peut pas poster de données au point de terminaison `/resources` sur `bam.nr-data.net`.
* La réponse RUM (toute première demande d&apos;agent à `bam.nr-data.net`) renvoie un indicateur `st` OU `sts` de `0`.
* la trace de session n&apos;est pas activée. Consultez la page des paramètres de l’application pour vous assurer que vous avez activé le suivi de session et que vous avez ajusté vos fréquences d’échantillonnage.
