## Problème

Le support New Relic nécessite des détails de dépannage supplémentaires sur ce qui se passe côté navigateur et vous n&apos;êtes pas en mesure de fournir une URL accessible ou le code source de votre page.

## Solution

En fonction de votre type de navigateur, créez un fichier [HAR (archive HTTP)](https://dvcs.w3.org/hg/webperf/raw-file/tip/specs/HAR/Overview.html) à des fins d&apos;enquête et envoyez-le à [support.newrelic.com](https://support.newrelic.com/). Le fichier HAR contient toutes les informations sur les appels réseau impliqués dans une page pendant la période enregistrée.

Lors de l&apos;enregistrement du fichier HAR, utilisez la fonctionnalité de votre site pour générer le type d&apos;activité approprié. Par exemple, si vous avez des problèmes avec AJAX, exercez la section de votre page qui effectue des appels AJAX.

<Callout variant="important">
  Les fichiers HAR capturent tout le trafic réseau pendant la période où ils sont enregistrés. Ces données de trafic peuvent inclure des informations privées ou personnellement identifiables (cookies, mots de passe, numéros de carte de crédit, etc.). Les clients sont responsables de toutes les données qu&apos;ils fournissent à New Relic, y compris dans les fichiers HAR. Les fichiers HAR peuvent être modifiés avec un éditeur de texte. Avant de partager un fichier HAR, nous vous recommandons fortement de supprimer manuellement toute information sensible.
</Callout>

<CollapserGroup>
  <Collapser id="chrome" title="Chrome">
    Les navigateurs Google Chrome ont la possibilité de générer un fichier HAR. Selon la version de votre navigateur, les noms de chemin peuvent varier pour ouvrir la console.

    <Callout variant="tip">
      Avant de collecter le fichier HAR, assurez-vous que l&apos;agent du navigateur est activement en cours d&apos;exécution et installé sur la page de l&apos;application.
    </Callout>

    1. Dans le menu Chrome (en haut à droite), sélectionnez <DNT>**Tools &gt; Developer tools**</DNT> ou <DNT>**More tools &gt; Developer tools**</DNT> selon le cas.
    2. Depuis la console, sélectionnez <DNT>**Network**</DNT>.
    3. Sélectionnez l&apos;icône d&apos;enregistrement.
    4. Le cas échéant, cochez la case <DNT>**Preserve log**</DNT> pour enregistrer les données de transaction réseau.
    5. Avant d’essayer de reproduire le problème, sélectionnez l’icône Effacer pour supprimer les informations d’en-tête inutiles.
    6. Naviguez sur votre site pour générer le type de trafic présentant des problèmes.
    7. Attendez quelques minutes jusqu&apos;à ce que le navigateur envoie les données au collecteur New Relic.
    8. Enregistrez les informations capturées en cliquant avec le bouton droit sur la liste des transactions en bas de la console et en sélectionnant <DNT>**Save as HAR with Content**</DNT>.
  </Collapser>

  <Collapser title="Firefox version 41.x et supérieure">
    La version 41 ou supérieure de Firefox offre la possibilité de copier les transactions dans un fichier HAR.

    1. Depuis le menu Firefox (en haut à droite), sélectionnez <DNT>**Developer &gt; Network**</DNT>
    2. Sélectionnez l&apos;option <DNT>**Clear**</DNT> (coin inférieur droit de la fenêtre des développeurs) pour supprimer les informations d&apos;en-tête inutiles.
    3. Naviguez sur votre site pour générer le type de trafic présentant des problèmes.
    4. Attendez quelques minutes jusqu&apos;à ce que le navigateur envoie les données au collecteur New Relic.
    5. Capturez les informations en cliquant avec le bouton droit sur la liste des transactions dans la console et en sélectionnant soit <DNT>**Copy All As HAR**</DNT> pour placer les données dans votre tampon, soit <DNT>**Save All As HAR**</DNT> pour les copier dans un fichier.
  </Collapser>

  <Collapser title="Microsoft Edge">
    1. Pour ouvrir <DNT>**Developer Tools**</DNT>, appuyez sur la touche F12 de votre clavier ou recherchez <DNT>**Developer Tools**</DNT> dans le menu.
    2. Sélectionnez l’onglet <DNT>**Network**</DNT> .
    3. Actualisez la page pour commencer à capturer le trafic entre le navigateur et le serveur, ou cliquez sur un lien vers la page concernée.
    4. Sélectionnez <DNT>**Export as HAR**</DNT> suivi de <DNT>**Save As...**</DNT> pour enregistrer le fichier HAR.
  </Collapser>

  <Collapser title="Safari d'Apple">
    1. Activer le menu Développer : dans la barre de menus macOS, accédez à <DNT>**Safari &gt; Preferences &gt; Advanced &gt; Show Develop menu in menu bar**</DNT>.
    2. Depuis la barre de menus macOS, accédez à l’onglet <DNT>**Develop &gt; Show Web Inspector &gt; Network**</DNT> .
    3. Dans l’onglet <DNT>**Network**</DNT>, cochez l’option <DNT>**Preserve log**</DNT> .
    4. Actualisez la page et autorisez <DNT>**Safari**</DNT> à enregistrer l&apos;interaction entre le navigateur et le site Web.
    5. Une fois la page chargée, sélectionnez <DNT>**Export**</DNT> en haut à droite dans la fenêtre de l&apos;onglet Réseau.
  </Collapser>
</CollapserGroup>
