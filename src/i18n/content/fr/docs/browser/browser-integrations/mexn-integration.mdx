Les piles MEAN, MERN et MEVN sont des kits de développement logiciel open source stack . Chaque kit utilise une base de code unique pour développer des applications Web. New Relic a consolidé les trois piles en un seul quickstart. Notre intégration MExN utilise l&apos;agent <InlinePopover type="browser" />, l&apos;agent Node.js et l&apos;intégration MongoDB pour vous fournir un dashboard prédéfini avec vos métriques les plus importantes (comme les temps de réponse, l&apos;utilisation du processeur, le trafic et les fréquences de connexion).

## Installation [#install]

<Steps>
  <Step>
    ### Facultatif : installer l’agent d’infrastructure [#infra-install]

    L&apos;installation de l&apos;agent infrastructure est facultative, mais cela vous permettra d&apos;obtenir des mesures importantes sur l&apos;hôte sur lequel votre application s&apos;exécute.

    Pour ce faire, suivez [les étapes d’installation de l’agent d’infrastructure](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent).
  </Step>

  <Step>
    ### Installer quickstartde MongoDB

    1. Consultez nos exigences en matière d’agent MongoDB dans notre [documentation](/docs/infrastructure/host-integrations/host-integrations-list/mongodb-monitoring-integration/) avant d’installer l’agent.
    2. Ouvrez l&apos;UI[installation quickstart de MongoDB](https://newrelic.com/instant-observability/mongodb).
    3. Cliquez sur <DNT>**Install now**</DNT> pour démarrer l’installation de l’agent MongoDB.
  </Step>

  <Step>
    ### Installer l&apos;agent Node.js

    1. Consultez nos exigences en matière d&apos;agent Node.js dans notre [documentation](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/).
    2. Ouvrez [Node.js quickstart l&apos;UIinstallation](https://newrelic.com/instant-observability/node-js)
    3. Cliquez sur <DNT>**Install now**</DNT> pour démarrer l’installation de l’agent Node.js.
  </Step>

  <Step>
    ### Installez l’agent de navigateurs [#browser-install]

    1. De [one.newrelic.com](https://one.newrelic.com), aller à la [page<DNT>**+ Integrations &amp; Agents**</DNT> ](https://one.newrelic.com/marketplace).
    2. Cliquez sur <DNT>**Browser and mobile**</DNT>, puis sur <DNT>**Browser monitoring**</DNT>.
    3. Sélectionnez l’option <DNT>**Select copy/paste JavaScript code**</DNT> (pour en savoir plus sur ce code, consultez [notre documentation d’installation du navigateur](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent)).
    4. Complétez le reste des étapes et cliquez sur <DNT>**Enable.**</DNT>
    5. Accédez au fichier `index.html `de votre application React ou Angular ou Vue.
    6. Collez notre de code de navigateur snippet script tag juste avant la tag fin de la body`</body>`().
    7. Redémarrez votre application.

    Dans quelques minutes, votre application enverra des métriques à [one.newrelic.com](https://one.newrelic.com).
  </Step>
</Steps>

## Trouvez vos données [#find-data]

Pour obtenir votre dashboard MExN :

1. De [one.newrelic.com](https://one.newrelic.com), aller à la page <DNT>**+Integrations &amp; Agents**</DNT> .
2. Cliquez sur <DNT>**Dashboards**</DNT>.
3. Dans la barre de recherche, tapez `MExN`.
4. Le dashboard MExN devrait apparaître. Cliquez dessus pour l&apos;installer.

Votre dashboard MExN est considéré comme un dashboard personnalisé et peut être trouvé dans l&apos;UI <DNT>**Dashboards**</DNT>. Pour obtenir des documents sur l&apos;utilisation et la modification du dashboard, consultez [notre documentation dashboard ](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Pour plus d&apos;informations sur les données signalées, consultez la documentation de chacun des outils que vous avez installés :

* [monitoring des navigateurs événement](/docs/data-apis/understand-data/event-data/events-reported-browser-monitoring).
* [Données d&apos;infrastructure](/docs/infrastructure/manage-your-data/data-instrumentation/default-infrastructure-monitoring-data/#infrastructure-events).
* [Données MongoDB](/docs/infrastructure/host-integrations/host-integrations-list/mongodb/mongodb-monitoring-integration-new/#find-and-use/).
* [Données Node.js.](/docs/apm/agents/nodejs-agent/getting-started/introduction-new-relic-nodejs/)

Voici une requête NRQL vérifiant le temps de chargement moyen des fenêtres pour les différents navigateurs utilisés par l&apos;utilisateur sur les ordinateurs de bureau :

```sql
SELECT average(windowLoad) 
FROM PageViewTiming 
WHERE (deviceType = 'Desktop') 
FACET `userAgentName` 
SINCE 604800 seconds 
AGO TIMESERIES
```
