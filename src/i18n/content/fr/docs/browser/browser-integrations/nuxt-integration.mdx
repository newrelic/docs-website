Avec notre dashboard Nuxt, vous pouvez facilement suivre le taux d&apos;erreur de votre application, la moyenne du chargement initial de la page, les changements d&apos;itinéraire, le débit et les vues de page avec les erreurs JavaScript. Construit avec notre agent d&apos;infrastructure et notre Vue <InlinePopover type="browser" />, le tableau de bord Nuxt vous permet de visualiser les informations critiques en un seul endroit.

## Installation [#install]

<Steps>
  <Step>
    ### Facultatif : installer l’agent d’infrastructure [#infra-install]

    L&apos;installation de l&apos;agent infrastructure est facultative, mais cela vous permettra d&apos;obtenir des mesures importantes sur l&apos;hôte sur lequel votre application s&apos;exécute.

    Pour ce faire, suivez [les étapes d’installation de l’agent d’infrastructure](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent).
  </Step>

  <Step>
    ### Installer le monitoring de browser Nuxt [#browser-install]

    1. Depuis <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>, accédez à la [page<DNT>**+ Integrations &amp; Agents**</DNT> ](https://one.newrelic.com/marketplace).
    2. Recherchez <DNT>**Nuxt**</DNT>. (Remarque : la vignette Nuxt ne se trouve pas dans la collection de navigateurs ou de mobiles, vous devrez donc utiliser la fonction de recherche.)
    3. Cliquez sur la vignette <DNT>**Nuxt**</DNT> lorsqu&apos;elle apparaît dans les résultats de recherche et suivez les instructions pour générer l&apos;extrait de code JavaScript.
  </Step>

  <Step>
    ### Configurer l&apos;intégration [#config]

    Pour ajouter l&apos;extrait JavaScript dans l&apos;application Nuxt :

    1. Créez un fichier JavaScript dans le dossier <DNT>**`public`**</DNT> de votre application Nuxt :

       ```bash
       touch public/<FILENAME>.js
       ```

    2. Copiez-collez l&apos;extrait de l&apos;agent de navigateur dans le nouveau fichier JavaScript sans les balises `<script>` (juste l&apos;extrait lui-même, il n&apos;y a aucune fonction à définir) :

       ```js
       // PASTE THE GENERATED BROWSER AGENT SNIPPET HERE WITHOUT <script> TAGS
       ```

    3. Ouvrez votre fichier `nuxt.config.js` (ou `nuxt.config.ts`) et ajoutez le script à la section `app.head` :

       ```js
       export default defineNuxtConfig({
         compatibilityDate: '2025-07-15',
         devtools: { enabled: true },
         app: {
           head: {
             script: [
               {
                 src: "/<FILENAME>.js"
               },
             ],
           },
         },
       })
       ```

    4. Redémarrez votre application.

    Dans quelques minutes, votre application enverra des métriques à [one.newrelic.com](https://one.newrelic.com).
  </Step>
</Steps>

## Monitorer votre application [#monitor]

Vous devriez maintenant pouvoir voir votre application sur <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT> en sélectionnant <DNT>**Browser**</DNT> dans la barre de navigation. Choisissez votre application pour voir toutes les métriques capturées sur la plateforme <DNT>**one.newrelic.com**</DNT> . Vous pouvez également afficher les données avec NRQL en utilisant la requête suivante :

```sql
SELECT firstPaint
FROM PageView
```
