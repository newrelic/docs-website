New Relic <InlinePopover type="browser" />offre monitoring des applications monopages (SPA) pour vous offrir une visibilité plus approfondie et des informations exploitables détaillées sur l&apos;interaction réelle de l&apos;utilisateur avec des applications monopages ou toute application utilisant requests AJAX.

En plus de monitoring automatiquement les changements d&apos;itinéraire, notre API SPA vous permet de monitorer pratiquement tout ce qui s&apos;exécute dans le navigateur. Les développeurs et leurs équipes peuvent utiliser l&apos;API pour :

* Créez des applications plus rapides, plus réactives et hautement interactives.
* Monitorez le débit et les performances rencontrés par les utilisateurs réels.
* Dépanner et résoudre les problèmes dans le contexte du chargement de la page.
* [Interrogez vos données](/docs/using-new-relic/data/understand-data/query-new-relic-data) pour vous aider à prendre des décisions commerciales.
* Proposez plus rapidement de meilleures applications sur le marché.

## Activer monitoringdu SPA [#enable]

Monitoring SPA est activée par défaut pour les nouvelles installations d&apos;agents de navigateur. La version de l&apos;agent compatible SPA donne accès à d&apos;autres fonctionnalités puissantes de New Relic, comme le tracing distribué. Pour plus d&apos;informations, consultez [Activer monitoring des navigateurs](/docs/browser/new-relic-browser/installation/update-browser-agent/#agent-types).

## Compatibilité [#compatibility]

Pour obtenir des informations sur la compatibilité des fonctionnalités liées à SPA, consultez [les exigences SPA](/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring#spa).

## Analyser les données de débit et de performance [#spa-data]

En améliorant les normes traditionnelles de l&apos;industrie pour mesurer le temps de chargement des pages, nous vous donnons une image complète de l&apos;activité, à la fois synchrone et asynchrone, associée aux chargements de pages et aux changements d&apos;itinéraire.

<img title="browser_SPA.png" alt="browser_SPA.png" src="/images/browser_screenshot-full_spa-pageview.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Browser &amp;gt; (select an app) &amp;gt; Page views:**</DNT> Utilisez monitoring monitoring SPA des navigateurs pour examiner le débit et les performances de votre application d&apos;architecture SPA.
</figcaption>

Le moniteur de données SPA par monitoring des navigateurs comprend :

* Données de performance et débit pour les chargements de pages et les changements d&apos;itinéraire
* Données de requête AJAX
* Activité JavaScript, à la fois synchrone et asynchrone
* Mises à jour de pages dynamiques, monitoring à l&apos;aide de l&apos; [API SPA](/docs/browser/new-relic-browser/browser-agent-spa-api)

Grâce à ces données, vous aurez une compréhension claire de la manière dont vos utilisateurs perçoivent les chargements de pages et les changements d&apos;itinéraire de votre application, et serez en mesure de résoudre les problèmes d&apos;étranglement et de dépannage des erreurs. Pour en savoir plus sur la manière dont New Relic gère les données SPA, consultez [Comprendre la collecte de données SPA](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection).
