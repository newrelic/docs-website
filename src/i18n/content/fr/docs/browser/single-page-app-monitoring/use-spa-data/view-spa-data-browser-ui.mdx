Si vous avez opté pour [le monitoring SPA (application monopage)](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring), la page <DNT>**Page views**</DNT> du navigateur inclura des données sur les changements d&apos;itinéraire SPA et les chargements de page initiaux.

## Données d&apos;application monopage (SPA) [#app-views]

Pour afficher les données SPA : Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT>.

Les chargements de page initiaux et les changements d&apos;itinéraire sont [automatiquement regroupés](/docs/browser/new-relic-browser/configuration/group-browser-metrics-urls#hierarchy) par nom d&apos;interaction du navigateur. Vous pouvez ajuster cela avec [les paramètres de votre liste de domaines autorisés pour les segments](/docs/browser/new-relic-browser/installation-configuration/url-whitelists-grouping-browser-metrics).

Si vous définissez des noms d&apos;itinéraire personnalisés avec l&apos; [API SPA](/docs/browser/new-relic-browser/browser-agent-spa-api), les noms d&apos;itinéraire personnalisés seront affichés. Vous pouvez modifier la manière dont la page se charge et dont les modifications d&apos;itinéraire sont regroupées en utilisant la liste déroulante [<DNT>**Group page by**</DNT>](/docs/browser/single-page-app-monitoring/use-spa-data/view-spa-data-new-relic-browser#group-facet) .

Par défaut, la liste des chargements de pages et des changements d&apos;itinéraire affiche les vues les plus chronophages en haut de la liste. Vous pouvez également trier par temps de réponse moyen, temps de réponse médian et débit par minute en utilisant la liste déroulante <DNT>**Sort by**</DNT> .

Pour rechercher des vues spécifiques par [URL groupée](/docs/browser/new-relic-browser/configuration/group-browser-metrics-urls#hierarchy), saisissez dans la barre de recherche sous la liste déroulante <DNT>**Sort by**</DNT> . Par exemple, pour trouver les URL qui représentent votre page de paiement, recherchez `checkout`.

Les graphiques de la page initiale <DNT>**Page view**</DNT> affichent :

* Les cinq vues avec le temps de réponse moyen le plus lent
* Les cinq vues avec le débit le plus élevé

Pour modifier la plage de temps examinée, utilisez le [sélecteur d&apos;intervalle de temps](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) situé en haut de la page. (Si vous choisissez une plage horaire de plus de huit jours dans le passé, certaines fonctionnalités de filtrage et de regroupement ne seront pas disponibles.)

## Filtrer les vues SPA

<img title="spa_filter.png" alt="spa_filter.png" src="/images/browser_screenshot-full_spa-filters.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views &gt; Filter**</DNT>:Utilisez <DNT>**Filter**</DNT> pour filtrer les changements d&apos;itinéraire, les chargements de page initiaux et d&apos;autres attributs tels que l&apos;emplacement et le type de navigateur.
</figcaption>

Pour afficher uniquement les chargements de page initiaux ou uniquement les modifications d&apos;itinéraire, utilisez la liste déroulante <DNT>**Filter**</DNT> . Par exemple, pour afficher uniquement les modifications d’itinéraire, sélectionnez <DNT>**Filter &gt; Route change**</DNT>.

Le filtre vous donne également la possibilité de filtrer par d&apos;autres attributs de chargement de page et de changements d&apos;itinéraire, tels que le nom de l&apos;application, l&apos;emplacement géographique du navigateur et le type de navigateur. Par exemple, pour voir uniquement les chargements de page et les changements d&apos;itinéraire survenus sur les navigateurs de la ville de Portland, dans l&apos;Oregon, sélectionnez <DNT>**Filter &gt; City &gt; Portland**</DNT>.

## Vues du SPA de groupe [#group-facet]

Vous pouvez utiliser la liste déroulante <DNT>**Group page by**</DNT> pour regrouper la liste des pages vues par n&apos;importe quel attribut. Par exemple, si vous souhaitez comparer le temps de réponse moyen par type de navigateur, sélectionnez <DNT>**Group page by &gt; userAgent**</DNT>.

La combinaison du filtrage et du regroupement vous permet de trouver rapidement des données très spécifiques. Par exemple, pour comparer la manière dont une URL spécifique se charge sur différents navigateurs :

1. Dans la liste déroulante <DNT>**Filter**</DNT>, sélectionnez <DNT>**targetURL**</DNT>, puis sélectionnez l’URL que vous souhaitez étudier.
2. Dans la liste déroulante <DNT>**Group page by**</DNT> , sélectionnez <DNT>**userAgent**</DNT>.

## SPA voir les détails [#details]

<img title="spa_details.png" alt="spa_details.png" src="/images/browser_screenshot-full_pageview-details.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views &gt; (select a view)**</DNT>:Sélectionnez une vue dans la liste pour voir divers détails et répartitions.
</figcaption>

Sélectionnez un chargement de page individuel ou un changement d&apos;itinéraire pour voir les détails. La sélection de l&apos;un ou l&apos;autre fournira une répartition du temps passé pour une [interactionavec le navigateur](/docs/browser/single-page-app-monitoring/understand-spa-data-structure#browser-interactions) et affichera ces données sur une série temporelle correspondant à la fenêtre sélectionnée dans le [sélecteur d&apos;intervalle de temps](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker).

Chaque changement d&apos;itinéraire peut théoriquement également être un chargement de page initial. (Par exemple, lorsqu&apos;une URL de changement d&apos;itinéraire est envoyée à quelqu&apos;un d&apos;autre et qu&apos;il la charge, cela sera désormais considéré comme un chargement de page initial sur New Relic.) C&apos;est pourquoi la page de détails de la vue SPA contient des graphiques pour les chargements de page initiaux et les changements d&apos;itinéraire. Cela vous permet de comparer les performances d&apos;une vue lors du chargement d&apos;une page initiale et ses performances lors d&apos;un changement d&apos;itinéraire.

Il existe trois options d&apos;affichage du graphique, sélectionnables à l&apos;aide des icônes situées à droite du titre du graphique <DNT>**Avg initial page load time**</DNT> . L&apos;affichage par défaut est le [graphique à aires empilées à code couleur](/docs/apm/new-relic-apm/ui-functions/standard-dashboard-features). Vous pouvez également passer à un [affichage sous forme d&apos;histogramme](/docs/apm/applications-menu/features/histograms-viewing-data-distribution) ou à un [graphique en courbespercentile ](/docs/apm/applications-menu/features/percentiles-comparing-ranked-data).

La page de détails contient également un graphique <DNT>**Throughput**</DNT> qui combine les chargements de page initiaux et les changements d&apos;itinéraire. Le graphique affiche les 5 pages avec le débit le plus élevé, qui sont répertoriées sous le graphique, et consolide toutes les autres pages en <DNT>**Other**</DNT>.

Voici des détails sur les données de performances spécifiques affichées pour les chargements de pages et les changements d&apos;itinéraire :

<CollapserGroup>
  <Collapser id="page-load" title="Détails des performances du chargement initial de la page">
    Pour les chargements de page initiaux, les détails des performances incluent le temps moyen du backend, le temps du frontend et l&apos;événement de chargement de la fenêtre :

    * <DNT>**Backend time**</DNT> inclut le réseau, le temps d&apos;application Web et la mise en file d&apos;attente des demandes.
    * <DNT>**Frontend time**</DNT> inclut le traitement DOM, le rendu des pages et le temps nécessaire pour terminer tous les XHR.
    * Une ligne rouge horizontale indique quand l&apos;événement de chargement de la fenêtre est déclenché. Cela correspond au temps de chargement de page traditionnel mesuré par l&apos;agent du navigateur sans monitoring SPA activée. Avec monitoring SPA, il est courant d&apos;avoir un événement de chargement de fenêtre avant que le <DNT>**frontend time**</DNT> ne soit terminé. (Pour en savoir plus sur la différence entre le temps de chargement des pages SPA et le temps de chargement des pages traditionnelles, voir [Comprendre la collecte de données SPA](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection#page-load-timing-diff).)
  </Collapser>

  <Collapser id="route-change" title="Détails des performances du changement d'itinéraire">
    Pour les changements d&apos;itinéraire, le graphique des performances affiche la durée du JS et le temps d&apos;attente.

    * <DNT>**JS Duration**</DNT> est la somme de tous les temps d&apos;exécution JavaScript pendant l&apos;interaction, qui est synchrone par définition.
    * Le temps restant est appelé <DNT>**Waiting time**</DNT> et est dérivé en soustrayant la durée JS de la durée totale.
    * Pour SPA v2.1+, durée JS uniquement
  </Collapser>
</CollapserGroup>
