Votre infrastructure nécessite une inspection et un entretien réguliers. Tout comme un urbaniste qui conçoit une nouvelle route ou un nouveau pont, les décisions concernant les besoins de votre système doivent provenir d’une observation attentive des limitations existantes. Pour les villes, cela consiste à ajouter de nouvelles voies aux artères principales lorsque les embouteillages augmentent le matin ou le soir. Pour une stack technologique, cela ressemble à une série de décisions en matière de ressources qui affectent les performances de vos API, services et applications.

Avec notre agent infrastructure , vous pouvez collecter régulièrement des métriques et des données d&apos;événements afin de savoir si votre infrastructure évolue de manière appropriée. Quels processus affectent l’utilisation du processeur et du disque ? Comment se comporte une application juste avant un pic de CPU ? Vos hôtes réagissent à une panne de base de données ? La rapidité avec laquelle vous pourrez répondre à ces questions fera la différence entre une panne totale, un incident mineur ou une mise à l’échelle automatique en douceur.

## Objectifs [#objectives]

Cette série de didacticiels vous guide dans l&apos;utilisation des données d&apos;infrastructure pour résoudre différents scénarios d&apos;incident. Alors que New Relic propose une suite complète d&apos;intégration pour monitorer le cloud, les environnements conteneurisés et diverses machines virtuelles, cette série est spécifique aux données signalées avec un agent infrastructure qui ne dispose pas d&apos;intégration supplémentaire.

Cela dit, ce document explique comment vous pouvez instrumenter entièrement votre système. Vous serez:

* Installez l&apos;agent d&apos;infrastructure avec notre installation guidée
* Comprendre les recommandations de l&apos;installation guidée afin de pouvoir prendre une décision éclairée concernant l&apos;instrumentation
* En savoir plus sur l&apos;intégration infrastructure

## Installer l&apos; agent d&apos;infrastructure [#install]

Lorsque vous installez l&apos;agent d&apos;infrastructure, vous installez un logiciel qui peut s&apos;intégrer à n&apos;importe quel nombre de technologies. L&apos;agent collecte les données auxquelles il a accès, puis transmet ces données à New Relic, où vous pouvez les explorer via notre interface utilisateur. Le monitoring de votre infrastructure vous permet de réagir aux incidents avec plus de clarté et de direction, afin que vous puissiez passer de la réaction à la proaction dans votre réponse aux incident .

## Comment les agents et l’intégration sont-ils liés les uns aux autres ? [#agent-integration]

Une fois l&apos;agent infrastructure installé, vous pouvez l&apos;intégrer à votre fournisseur cloud , à votre type de machine virtuelle ou à votre orchestration de conteneurs via l&apos;intégration. L&apos;intégration vous permet de personnaliser la manière dont l&apos;agent infrastructure est activé afin que vous n&apos;installiez que ce dont vous avez besoin. Par exemple, si vous utilisez Microsoft Azure mais pas AWS, vous pouvez ignorer l’intégration AWS et installer uniquement l’intégration Microsoft Azure.

Si vous avez choisi de ne pas participer à une installation avec notre guide d&apos;installation mais que vous avez depuis changé d&apos;avis, vous pouvez consulter nos offres d&apos;intégration dans notre document [Introduction à l&apos;intégration infrastructure ](/docs/infrastructure/infrastructure-integrations/get-started/introduction-infrastructure-integrations).
