<Callout variant="important">
  Exécutez IAST avec un déploiement hors production uniquement. Tests IAST en invoquant requests HTTP avec une charge d&apos;exploitation. Vous devez utiliser IAST dans un environnement hors production avec uniquement des données simulées pour éviter à la fois la corruption des données et l&apos;introduction d&apos;exploits dans votre code en direct. De plus, lorsque vous exécutez IAST sur un nouveau code en pré-production, vous détectez les vulnérabilités potentielles avant leur mise en ligne.
</Callout>

Pour installer New Relic IAST :

<Steps>
  <Step>
    Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; IAST**</DNT> et cliquez sur <DNT>**Set up IAST with applications**</DNT>.

    <img title="Set up IAST with applications" alt="Set up IAST with applications" src="/images/iast_screenshot-full_install.webp" />
  </Step>

  <Step>
    Confirmez que IAST ne s&apos;exécute pas dans un environnement de production et cliquez sur <DNT>**Continue**</DNT>.
  </Step>

  <Step>
    Sélectionnez la langue de votre application et complétez les étapes.

    <img title="Install New Relic IAST" alt="Install New Relic IAST" src="/images/iast_screenshot-crop_install.webp" />
  </Step>

  <Step>
    Assurez-vous que votre fichier de configuration `newrelic.yml` est mis à jour comme suit :

    <CollapserGroup>
      <Collapser
        id="config-file-example"
        title={<><InlineCode>
          newrelic.yml
        </InlineCode> fichier de configuration</>
        }
      >
        ```yml
        security:
            enabled: true
            agent:
                enabled: true
        ```
      </Collapser>

      <Collapser
        id="config-eu-file-example"
        title={<><InlineCode>
          newrelic.yml
        </InlineCode> fichier de configuration pour l'UE</>
        }
      >
        ```yml
        security:
            enabled: true
            agent:
                enabled: true
            validator_service_url: wss://csec.eu01.nr-data.net
        ```
      </Collapser>

      <Collapser
        id="config-file-fed-example"
        title={<><InlineCode>
          newrelic.yml
        </InlineCode> fichier de configuration pour Fed utilisateur</>
        }
      >
        ```yml
        security:
            enabled: true
            agent:
                enabled: true
            validator_service_url: wss://csec-gov.nr-data.net
        ```
      </Collapser>
    </CollapserGroup>
  </Step>

  <Step>
    Une fois toutes les étapes terminées, redémarrez votre application et générez du trafic vers les API de l&apos;application.
  </Step>

  <Step>
    Cliquez sur <DNT>**See your data**</DNT> pour voir un aperçu de votre application testée.
  </Step>
</Steps>
