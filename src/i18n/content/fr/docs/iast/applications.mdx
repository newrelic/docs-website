La page des applications vous informe du nombre total de vulnérabilités trouvées dans vos applications testées et du nombre d&apos;applications non testées dont vous disposez. Vous pouvez toujours ajouter des applications non testées à la fonctionnalité IAST de New Relic. Gardez à l’esprit que chaque test d’application est une version d’une application. Chaque fois que votre pré-environnement de production lance une nouvelle build et exécute un nouveau test, il apparaîtra avec ce nom d&apos;application. Pour chaque exécution de test, nous affichons le nom de l&apos;application, le compte, l&apos;état, le nombre d&apos;exploits, l&apos;efficacité du test et la dernière fois que l&apos;application a été testée. Vous pouvez trier par nom d&apos;application pour voir toutes les exécutions de test d&apos;une application ensemble afin de comparer rapidement leur statut.

## Détails et correction d&apos;une demande [#remediation-app]

Découvrez tous les problèmes détectés avec les tests IAS sur vos applications. Pour voir les détails et les instructions de correction d&apos;une demande :

1. Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; IAST**</DNT> et cliquez sur <DNT>**Tests &amp; Applications**</DNT> dans le volet de navigation de gauche.

2. Sélectionnez une application. Vous pouvez commander toutes les colonnes.

## Activer New Relic IAST [#enable-iast]

Activez IAST sur vos applications non testées pour monitorer, tester et corriger les risques de sécurité tout au long du SDLC à grande échelle. Cela vous aidera également à expédier le code plus rapidement. Pour activer IAST :

<Steps>
  <Step>
    Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; IAST**</DNT> et cliquez sur <DNT>**Tests &amp; Applications**</DNT> dans le volet de navigation de gauche.
  </Step>

  <Step>
    Cliquez sur l&apos;onglet <DNT>**All applications**</DNT> et vérifiez la colonne <DNT>**IAST supported**</DNT> pour vous assurer que l&apos;application est prise en charge. Sélectionnez l&apos;application pour laquelle vous souhaitez activer IAST.
  </Step>

  <Step>
    Dans la fenêtre <DNT>**Enable IAST**</DNT>, suivez les étapes pour mettre à jour la configuration de votre application, afin de pouvoir la tester pour détecter les vulnérabilités exploitables.

    <img title="Enable IAST for your untested application" alt="Enable IAST for your untested application" src="/images/iast_screenshot-crop_enable-iast.webp" />

    <figcaption>
      Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; All capabilities &gt; IAST**</DNT> et cliquez sur <DNT>**Tests &amp; Applications**</DNT> dans le volet de navigation de gauche. Cliquez sur l’onglet <DNT>**All Applications**</DNT> et sélectionnez une application.
    </figcaption>
  </Step>

  <Step>
    Mettez à jour l&apos;agent APM si nécessaire.
  </Step>

  <Step>
    Cliquez sur <DNT>**Next**</DNT> et suivez les instructions.
  </Step>

  <Step>
    Redémarrez votre application et cliquez sur <DNT>**See your data**</DNT>.
  </Step>
</Steps>
