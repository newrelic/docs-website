L&apos;administrateur de compte peut configuration la réception de notifications push sur les appareils Android et iOS à partir [des alertes New Relic](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-applied-intelligence). Vous pouvez recevoir le problème <InlinePopover type="alerts" />à partir de n&amp;apos;importe quel workflow avec une destination de [push mobile](/docs/alerts-applied-intelligence/notifications/notification-integrations#mobile_push).

## Exigences [#requirements]

L&apos;utilisateur sur le [modèle utilisateur New Relic](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) et le [modèle utilisateur d&apos;origine](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models) peut configurer un workflow avec le push mobile comme [destination](/docs/alerts-applied-intelligence/notifications/notification-integrations#mobile_push).

## Activer ou désactiver la notification [#enabling-alerts]

Lorsque vous vous connectez à votre compte New Relic depuis une application Android ou iOS, votre appareil est automatiquement associé à votre utilisateur. Assurez-vous d&apos; [ajouter une destination de push mobile au workflow](/docs/alerts-applied-intelligence/applied-intelligence/incident-workflows/incident-workflows#add-workflow).

## Afficher les détails de l&apos;incident [#viewing-alerts]

La notification apparaît automatiquement sur l&apos;écran de verrouillage de votre appareil. Pour démarrer l&apos;application New Relic :

1. En fonction de votre appareil, procédez comme suit :

   * Sur les appareils Android , appuyez sur la notification dans le tiroir notification .
   * Sur les appareils iOS, faites glisser l’écran.

2. Dans le menu <DNT>**Alerts**</DNT> de l&amp;apos;application New Relic, sélectionnez un problème pour afficher les détails de l&amp;apos;erreur pour l&amp;apos;application associée.

3. En option, sélectionnez <DNT>**Acknowledge**</DNT>.

4. En option, pour afficher des détails supplémentaires, sélectionnez <DNT>**Overview**</DNT>, <DNT>**Incidents**</DNT> ou <DNT>**Issue log**</DNT>.

La liste <DNT>**Alerts**</DNT> du menu principal affiche les problèmes triés par heure. Vous pouvez appliquer des filtres supplémentaires ou modifier la fenêtre temporelle en haut à droite :

## Résoudre les problèmes liés aux paramètres d&apos;alerte [#troubleshooting]

Si les notifications push ne fonctionnent pas sur votre appareil mobile :

1. Vérifiez que vous répondez [aux exigences](#requirements).
2. Vérifiez que [les alertes sont activées](#enabling-alerts).
3. Vérifiez [les paramètresnotification ](#notification-settings)de votre appareil mobile pour vous assurer que New Relic est autorisé à envoyer des notifications push.

Si les paramètres notification de votre appareil mobile sont corrects, mais que vous ne recevez toujours pas de notification, [supprimez l&apos;appareil de votre compte](#account-delete), puis [désinstallez](#uninstall-app) et [réinstallez](#reinstall-app) l&amp;apos;application New Relic.
