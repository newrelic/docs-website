En règle générale, lorsque vous vous connectez à l&apos;application mobile New Relic, votre session est automatiquement redirigée vers votre navigateur Web. À partir de là, vous pouvez vous connecter à votre compte New Relic. Voici quelques conseils de dépannage si vous rencontrez des problèmes lors de l&apos;utilisation de l&apos;application mobile New Relic avec votre compte compatible SAML-SSO .

## Pas de nom d&apos;utilisateur ni de mot de passe [#no-sso-user]

Il se peut que vous n&apos;ayez pas de nom d&apos;utilisateur ou de mot de passe pour New Relic, car certains fournisseurs SAML écraseront votre mot de passe ou parce que votre administrateur ne vous a pas envoyé ces informations. Dans ces situations :

1. Depuis l&apos;application mobile <DNT>**Log in**</DNT>, sélectionnez le lien <DNT>**I don&apos;t have a password**</DNT> .
2. Utilisez votre appareil mobile pour ouvrir votre compte de messagerie.
3. Depuis votre compte de messagerie, récupérez l&apos;e-mail d&apos;authentification New Relic dans les 20 minutes.
4. Sélectionnez le bouton <DNT>**Authenticate**</DNT> ou le lien situé en dessous dans l’e-mail.

## Erreurs après la connexion [#sso-errors]

Si vous constatez des erreurs après vous être connecté avec succès à votre fournisseur SSO avec votre appareil mobile, vérifiez que vous pouvez vous connecter à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT> avec un navigateur Web de bureau.

* Si non, contactez votre administrateur.
* Si oui, obtenez de l&apos;aide sur [support.newrelic.com](https://support.newrelic.com).

## Problèmes de réauthentification [#sso-reauth]

Si vous utilisez la réauthentification sur un compte SAML-SSO , vous devez vous connecter à votre compte par défaut. (Tous les autres comptes seront grisés.) Si vous tentez de passer à un compte grisé, un message d&apos;erreur apparaîtra, expliquant que cela n&apos;est actuellement pas pris en charge.
