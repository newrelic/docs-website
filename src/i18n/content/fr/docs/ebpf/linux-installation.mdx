Vous pouvez installer l&apos;agent New Relic eBPF sur votre hôte Linux pour monitorer l&apos;état de santé de l&apos;ensemble de votre système. L&apos;agent eBPF offre une visibilité approfondie sur les performances des applications sans nécessiter de modifications de code ni de déploiement d&apos;agents spécifiques au langage.

## Installez l&apos;agent New Relic eBPF [#install-linux]

<Steps>
  <Step>
    ### Avant de commencer [#requirements]

    Vous devez satisfaire aux prérequis décrits dans la documentation [relative à la compatibilité et aux exigences eBPF](/docs/ebpf/requirements#linux) pour les hôtes Linux.
  </Step>

  <Step>
    ### Installer l&apos;agent eBPF [#install]

    Pour installer l&apos;agent eBPF :

    1. connectez-vous à votre compte New Relic.

    2. Allez à <DNT>**[left navigation pane &gt; + Integration &amp; Agents &gt; eBPF Agent](https://onenr.io/0oR8XWW36RG)**</DNT>.

    3. Sur l&apos;écran <DNT>Select an account</DNT>, sélectionnez le compte sur lequel vous souhaitez installer l&apos;agent eBPF, puis cliquez sur <DNT>**Continue**</DNT>.

    4. Sur la page <DNT>Select an installation method</DNT>, sélectionnez <DNT>**On a host**</DNT> et cliquez sur <DNT>**Continue**</DNT>.

    5. Sur l&apos;écran <DNT>Enter your user key</DNT>, sélectionnez l&apos;une des options suivantes, puis cliquez sur <DNT>**Continue**</DNT>:

       * <DNT>**Use an existing key**</DNT>: Si vous avez déjà une clé utilisateur, fournissez la clé utilisateur. Pour plus d&apos;informations, consultez [Clés utilisateur](/docs/apis/intro-apis/new-relic-api-keys/#user-key).
       * <DNT>**Create a new key**</DNT>: Si vous n&apos;avez pas de clé utilisateur, cliquez sur <DNT>**Create a new key**</DNT> pour en créer une.

    6. Sur l&apos;écran <DNT>Configure the host integration</DNT>, saisissez le nom de déploiement de l&apos;hôte, puis cliquez sur <DNT>**Continue**</DNT>.

    7. Sur l&apos;écran <DNT>Install the agent</DNT>, exécutez la commande fournie dans votre terminal pour installer l&apos;agent eBPF via le paquet Debian.

       1. Copiez et collez la commande affichée pour installer l&apos;agent eBPF sur votre hôte Debian.

       2. Pour vérifier l’installation, exécutez la commande suivante :

          ```bash
          sudo systemctl status newrelic-ebpf-agent
          ```

       Une fois l&apos;installation réussie, l&apos;état de ces services s&apos;affiche comme actif (en cours d&apos;exécution).

       <Callout variant="tip">
         Vous pouvez personnaliser la configuration de l&apos;agent eBPF en modifiant le fichier <DNT>`newrelic-ebpf-agent.yaml`</DNT> disponible sur <DNT>`/etc/newrelic-ebpf-agent/newrelic-ebpf-agent.yaml`</DNT>. Pour plus d&apos;informations sur les paramètres configuration, reportez-vous à [Paramètres de configuration](#config-params).
       </Callout>
  </Step>

  <Step>
    ### Accéder aux données eBPF dans New Relic [#access-data]

    Une fois l&apos;agent eBPF installé, il commence automatiquement à collecter les données de votre hôte Linux. Vous pouvez accéder à ces données dans l&apos;interface utilisateur OpenTelemetry de New Relic. Pour plus d&apos;informations sur l&apos;interface utilisateur New Relic OpenTelemetry, reportez-vous [à l&apos;interface utilisateur OpenTelemetry APM](/docs/opentelemetry/get-started/apm-monitoring/opentelemetry-apm-ui).

    **Pour afficher les données eBPF dans New Relic :**

    1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; APM &amp; Services**</DNT>.
    2. Dans la bannière de recherche, définissez les critères de recherche sur <DNT>`instrumentation.name = nr_ebpf`</DNT>:
       <img style={{ align: 'left' }} title="eBPF filters" alt="eBPF filter for eBPF data in New Relic OpenTelemetry UI" src="/images/ebpf_filters.webp" />
  </Step>
</Steps>

## Mettre à niveau l&apos;agent eBPF [#upgrade]

Pour mettre à niveau l&apos;agent eBPF sur un hôte Linux, réexécutez la commande d&apos;installation depuis l&apos;interface utilisateur de New Relic. Ce processus installera la nouvelle version de l&apos;agent tout en supprimant automatiquement l&apos;ancienne.

<Callout variant="important">
  La mise à niveau via un gestionnaire de paquets n&apos;est pas prise en charge. Pour une mise à niveau réussie, vous devez utiliser la commande d&apos;installation de l&apos;interface utilisateur de New Relic.
</Callout>

Pour obtenir la dernière commande d&apos;installation :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; + Integration &amp; Agents &gt; eBPF Agent**</DNT>.

2. Suivez le processus d&apos;installation pour obtenir la commande mise à jour.

3. Exécutez la commande sur votre hôte pour mettre à niveau l&apos;agent.

4. Redémarrez le service de l&apos;agent ebpf en utilisant la commande ci-dessous :

   ```bash
   sudo systemctl restart newrelic-ebpf-agent
   ```

## Paramètres de configuration [#config-params]

Le fichier `newrelic-ebpf-agent.yaml` contient les paramètres de configuration suivants :

<Callout variant="tip">
  L&apos;agent eBPF génère automatiquement des noms d&apos;entités différemment selon l&apos;environnement :

  * Dans les hôtes ou Docker, ces noms sont une combinaison du nom du processus, de son répertoire ou de son ID de conteneur et du port d&apos;écoute. Par exemple, `ruby:/home/ubuntu/app:[5678]` ou `java:f4aead533895:[8080]`.

  * Dans Kubernetes, ces noms sont dérivés du nom du service, par exemple, `mysql-database-service`.

  Attribution d&apos;un nom personnalisé aux applications :

  * Vous pouvez attribuer un nom personnalisé à votre application en définissant la variable d&apos;environnement `NEW_RELIC_APP_NAME` pour les applications Kubernetes et sur l&apos;hôte.
</Callout>

## Désinstallez l&apos;agent eBPF [#uninstall]

Pour désinstaller l&apos;agent eBPF d&apos;un hôte Linux, utilisez les outils de gestion package de votre système :

```bash
sudo apt-get purge newrelic-ebpf-agent
```

<Callout variant="tip">
  Il n&apos;est pas nécessaire d&apos;arrêter le service avant d&apos;exécuter la commande de désinstallation.
</Callout>
