Si le <DNT>eBPF agent</DNT> New Relic n&apos;achemine pas la télémétrie via votre proxy comme prévu, utilisez cette page pour diagnostiquer les erreurs courantes. Pour les instructions d&apos;installation et la référence de configuration, consultez [Configurer un proxy pour le <DNT>eBPF agent</DNT>New Relic](/docs/ebpf/proxy-configuration).

## Erreurs courantes [#common-errors]

<CollapserGroup>
  <Collapser id="error-400" title="Le proxy HTTP a renvoyé le code de réponse 400">
    **Problème :** le proxy ne prend pas en charge la méthode HTTP <DNT>`CONNECT`</DNT>.

    **Solution :** utilisez un proxy compatible <DNT>`CONNECT`</DNT>. Consultez [Serveurs proxy pris en charge](/docs/ebpf/proxy-configuration#supported-proxies).
  </Collapser>

  <Collapser id="error-https-scheme" title="Schéma « https » non pris en charge dans l'URI du proxy">
    **Problème :** gRPC ne prend pas en charge <DNT>`https://`</DNT> comme schéma de proxy. Il s&apos;agit d&apos;une limitation de gRPC, et non d&apos;une restriction de l&apos;agent.

    **Solution :** définissez le schéma de proxy sur <DNT>`http`</DNT> (par exemple, <DNT>`otlpProxy.scheme: "http"`</DNT>). Vos données télémétriques transitent toujours chiffrées par TLS de bout en bout via le tunnel proxy. Seule la connexion au proxy lui-même est en clair.
  </Collapser>

  <Collapser id="error-socket-closed" title="Socket fermé ou connexion refusée">
    **Problème :** le serveur proxy n’est pas en cours d’exécution, ou l’hôte ou le port est incorrect.

    **Solution :** vérifiez que le proxy est accessible à l&apos;aide de la [commande de test<DNT>`curl`</DNT> ](/docs/ebpf/proxy-configuration#test-connectivity).
  </Collapser>

  <Collapser id="error-connect-aborted" title="Proxy CONNECT abandonné">
    **Problème :** le proxy a rejeté la requête <DNT>`CONNECT`</DNT> en raison d&apos;une règle de pare-feu, d&apos;une ACL ou d&apos;une destination non prise en charge.

    **Solution :** vérifiez les logs de votre serveur proxy et confirmez que les requests <DNT>`CONNECT`</DNT> vers le port `443` sont autorisées depuis l&apos;adresse IP source de l&apos;agent.
  </Collapser>

  <Collapser id="error-tls-required" title="Configuration du proxy ignorée : le proxy HTTP CONNECT gRPC nécessite TLS">
    **Problème :** un proxy est configuré mais TLS est désactivé (par exemple, <DNT>`TLS_ENABLED=false`</DNT>). L&apos;implémentation du proxy HTTP CONNECT de gRPC ne fonctionne que lorsque la connexion au point de terminaison OTLP utilise TLS.

    **Solution :** assurez-vous que TLS est activé (il est activé par défaut). L&apos;agent utilisera ensuite le proxy configuré.
  </Collapser>

  <Collapser id="error-credential-chars" title="Le nom d'utilisateur du proxy contient des caractères spéciaux ou majuscules">
    **Problème :** votre nom d’utilisateur ou mot de passe de proxy contient des caractères que l’analyseur d’URI de gRPC n’accepte pas. Consultez [Limitations de caractères des identifiants](/docs/ebpf/proxy-configuration#credential-limitations).

    **Solution :** utilisez un compte proxy avec un nom d’utilisateur alphanumérique en minuscules et un mot de passe qui n’utilise que des lettres, des chiffres et les caractères spéciaux autorisés.
  </Collapser>
</CollapserGroup>
