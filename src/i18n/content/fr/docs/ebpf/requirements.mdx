Cette page fournit des informations complètes sur la compatibilité de l&apos;agent New Relic eBPF, notamment les données de base prises en charge, les exigences de déploiement et les capacités d&apos;interrogation des données qui s&apos;appliquent à toutes les méthodes d&apos;installation.

<Tabs>
  <TabsBar>
    <TabsBarItem id="k8s">Pour Kubernetes</TabsBarItem> <TabsBarItem id="linux">sous Linux</TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="k8s">
      * Assurez-vous que tous les [prérequis de la distribution Linux](/docs/ebpf/requirements/#kernel-requirements) sont respectés pour le nœud Kubernetes.

      ### Environnements Kubernetes [#kubernetes-environments]

      Les environnements Kubernetes suivants sont pris en charge :

      * Amazon Elastic Kubernetes Service (EKS)
      * Google Kubernetes Engine (GKE)
      * Azure Kubernetes Service (AKS)
      * Red Hat OpenShift
      * Microk8s

      ### Prise en charge du runtime de conteneur [#container-runtime]

      Les environnements d&apos;exécution de conteneurs suivants sont pris en charge :

      * Runtime par défaut de Docker (runc)
      * containerd
      * CRI-O
      * Podman
      * Youki
      * crun
    </TabsPageItem>

    <TabsPageItem id="linux">
      ### Exigences du noyau [#kernel-requirements]

      * Versions du noyau Linux :

        * `x86-64` (amd64) : `v5.8` et supérieur
        * `arm64` (aarch64) : `v5.8` et supérieur

      <Callout variant="important">
        Certaines distributions Linux vous permettent d&apos;exécuter d&apos;anciennes versions du noyau sur des versions plus récentes du système d&apos;exploitation. Même si votre distribution est [listée ci-dessous](#supported-linux-distributions), l&apos;agent eBPF ne fonctionnera pas si la version du noyau est inférieure à `5.8`.
      </Callout>

      ### Distributions Linux prises en charge [#supported-linux-distributions]

      Les distributions Linux suivantes sont prises en charge :

      * CentOS `9` et versions ultérieures
      * Debian `11` et versions ultérieures
      * Red Hat Enterprise Linux `8.1` et versions ultérieures
      * Ubuntu `20.04` et versions ultérieures
      * Amazon Linux `2` and `2023`
      * Bottle Rocket `1.51.0` et versions ultérieures
      * AlmaLinux `9.7`

      ### exigences système [#system-requirements]

      * Privilèges root (requis pour l&apos;installation et le fonctionnement)
      * Un minimum de `2 GB` de RAM
    </TabsPageItem>
  </TabsPages>
</Tabs>

### Prise en charge de protocoles supplémentaires [#protocol-support]

L&apos;agent eBPF prend également en charge le monitoring de ces protocoles :

* HTTP

* Apache Thrift

* gRPC

* Cassandre

* Kafka

* AMQP

  <Callout variant="tip">
    Pour Cassandra, Kafka et AMQP, l&apos;agent eBPF ne prend pas en charge la liaison de segments.
  </Callout>

<Callout variant="tip">
  Le monitoring eBPF est indépendant du langage et fournit des métriques de latence et de débit pour toute application. Cependant, [la liaison des segments de transaction](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm/#segments) dépend du langage et n&apos;est actuellement pas prise en charge pour PHP et Node.js.
</Callout>

{/* ## Querying eBPF data [#querying-data]

Once the eBPF agent is deployed, telemetry data flows to New Relic and can be queried using NRQL. Here are common query patterns:

### Query spans

FROM Span

### Query metrics

Including TCP stats and performance metrics:

### Entity naming

The eBPF agent automatically generates entity names differently depending on the environment:

* **In hosts or Docker**: Names are a combination of the process name, its directory or container ID, and the listening port. For example, `ruby:/home/ubuntu/app:[5678]` or `java:f4aead533895:[8080]`.

* **In Kubernetes**: Names are derived from the service name, for example, `mysql-database-service`.

## Data access in New Relic [#data-access]

eBPF data is accessible through multiple New Relic interfaces:

1. **APM & Services UI**: Go to **[one.newrelic.com](https://one.newrelic.com) > APM & Services** and filter by `instrumentation.name = nr_ebpf`

2. **NRQL queries**: Use the query examples above to explore your data

3. **OpenTelemetry UI**: eBPF data integrates with New Relic's OpenTelemetry experience
