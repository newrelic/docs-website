      ### distributions Linux [#linux-distributions]

      * Versions du noyau Linux `5.4` et ultérieures

      * `arm64` ou architectures `x86-64`

      * Privilèges root (accès sudo requis pour l&apos;installation et le fonctionnement)

      * Un minimum de `2 GB` de RAM

      * Prend en charge les distributions Linux suivantes :

        * Debian `11` et versions ultérieures
        * Ubuntu `20.04` et versions ultérieures
        * CentOS Stream `9` et versions ultérieures
        * RHEL `8` et versions ultérieures

/\* ## Interrogation des données eBPF \[#querying-data] Une fois l&apos;agent eBPF déployé, les données télémétriques circulent vers New Relic et peuvent être requêtes à l&apos;aide de NRQL. Voici les modèles de requêtes courants : ### étendues de requêtes Y compris requests complètes et les données trace : \`\`\`sql FROM Span SELECT \* WHERE deployment.name = &apos;&lt;deployment.name&gt;&apos; LIMIT MAX \`\`\` ### requêtes métriques Y compris les statistiques TCP et les métriques de performances: \`\`\`sql FROM métrique SELECT \* WHERE deployment.name = &apos;&lt;deployment.name&gt;&apos; LIMIT MAX \`\`\` &lt;Callout variant=&quot;tip&quot;&gt; Remplacer \`&lt;deployment.name&gt;\` avec le nom de déploiement réel que vous avez configuré lors de l&apos;installation. &lt;/Callout&gt; ### Nommage des entités L&apos;agent eBPF génère automatiquement des noms d&apos;entités différents selon l&apos;environnement : \* \*\*Dans les hôtes ou Docker : Les noms sont une combinaison du nom du processus, de son répertoire ou de son ID de conteneur et du port d&apos;écoute. Par exemple, \`ruby:/home/ubuntu/app:\[5678]\` ou \`java:f4aead533895:\[8080]\`. \* \*\*Dans Kubernetes: les noms sont dérivés du nom du service, par exemple, \`mysql-database-service\`. ## Accès aux données dans New Relic \[#data-access] Les données eBPF sont accessibles via plusieurs interfaces New Relic : 1. \*\*Interface utilisateur APM et Services\*\* : Accédez à \*\*\[one.newrelic.com]\(https\://one.newrelic.com) &gt; APM &amp; Services\*\* et filtrez par \`instrumentation.name = nr\_ebpf\` 2. Requête NRQL : Utilisez les exemples de requêtes ci-dessus pour explorer vos données 3. Interface utilisateur OpenTelemetry \*\* : Les données eBPF s&apos;intègrent à l&apos;expérience OpenTelemetry de New Relic Pour plus d&apos;informations sur l&apos;accès à vos données, consultez \[ Interface utilisateur OpenTelemetry APM]\(/docs/OpenTelemetry/get-started/apm-monitoring/OpenTelemetry-apm-ui). \*/
