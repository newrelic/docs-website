Cette référence couvre les attributs spécifiques à l&apos;agent eBPF de New Relic pour l&apos;APM eBPF et les métriques réseau eBPF. Les données eBPF sont transmises aux types de données NRDB standards (`Metric`, `Span` et `Log`), et ces attributs sont disponibles pour l&apos;interrogation, le facettage et les alertes dans NRQL.

L&apos;agent eBPF émet également des attributs de convention sémantique OpenTelemetry standards (par exemple, `http.request.method`, `db.system`, `jvm.memory.usage`) et des champs de span OpenTelemetry standards (`trace_id`, `span_id`, `kind`, etc.). Consultez les [conventions sémantiques d’OpenTelemetry](https://opentelemetry.io/docs/specs/semconv/) et la [spécification de l’API de trace](https://opentelemetry.io/docs/specs/otel/trace/api/#span) pour connaître leurs définitions.

Toute la télémétrie eBPF comporte les attributs de portée suivants :
