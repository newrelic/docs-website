Pour les organisations avec plusieurs comptes New Relic, chacun avec des exigences d’ingestion de données distinctes, les budgets d’ingestion de données fournissent un monitoring, une visibilité et des alertes par compte pour garantir que les comptes restent dans leurs limites d’utilisation définies.

## Types de budget par structure client

Le type de budgets que vous pouvez créer dépend de votre structure client :

### Pour les clients à locataire unique

Vous pouvez créer <DNT>**account level budgets**</DNT> pour des comptes individuels ou des groupes allant jusqu&apos;à 5 comptes au sein de votre organisation.

### Pour les clients multi-locataires

<DNT>**Managing organizations**</DNT> peut créer <DNT>**account level budgets**</DNT> pour les comptes au sein de l&apos;organisation de gestion (mêmes fonctionnalités que les clients à locataire unique).

<DNT>**Managed organizations**</DNT> ne peuvent pas créer de budgets.

## Configurer des budgets au niveau du compte

<Steps>
  <Step>
    ### Ouvrir les budgets de données

    1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/)**</DNT>.
    2. Dans <DNT>[user menu](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)</DNT>, sélectionnez <DNT>**Administration** &gt; **Data Management** &gt; **Data Budgets**</DNT>.
  </Step>

  <Step>
    ### Créer un budget

    Dans la section <DNT>**Functional budgets**</DNT> , sélectionnez <DNT>**+ Add functional budget**</DNT>.

    <Callout variant="important">
      <DNT>**For multi-tenant customers**</DNT>, cette section affiche :

      * <DNT>**Organization level budgets**</DNT> pour les organisations gérées
      * <DNT>**Account level budgets**</DNT> pour l&apos;organisation de gestion

      <DNT>**Managed organizations**</DNT> ne peuvent pas créer de budgets au niveau du compte.
    </Callout>
  </Step>

  <Step>
    ### Attribuez votre budget

    1. Saisissez un nom pour votre budget.
    2. Dans la liste déroulante, sélectionnez et affectez jusqu&apos;à 5 comptes à ce budget.

    <Callout variant="tip">
      Les comptes auxquels des budgets sont déjà attribués apparaîtront désactivés dans la liste de sélection.
    </Callout>

    3. Sélectionnez <DNT>**Next**</DNT>.
  </Step>

  <Step>
    ### Fixez des limites budgétaires mensuelles

    1. Dans la section <DNT>**Set monthly budget limits**</DNT>, vous verrez le nombre de Go restants dans le budget de votre organisation.
    2. Saisissez la quantité de Go que vous souhaitez allouer à ce budget au niveau du compte.

    <Callout variant="tip">
      Vous pouvez allouer plus que le budget de votre organisation. Cela permet une planification flexible mais peut entraîner des dépassements.
    </Callout>

    3. Sélectionnez <DNT>**Next**</DNT>.
  </Step>

  <Step>
    ### Configurer la notification

    1. Pour recevoir un avertissement lorsque vous avez utilisé un certain pourcentage des Go alloués à ce budget pour un mois donné, dans la section <DNT>**Thresholds**</DNT>, sélectionnez <DNT>**+ Add a threshold**</DNT> et entrez le pourcentage à partir duquel vous souhaitez recevoir une notification (par exemple, 75 %, 90 % ou 95 %).

    2. Pour vous assurer que les notifications sont envoyées lorsque le budget dépasse le seuil, dans la section <DNT>**Destinations**</DNT>, ajoutez au moins une destination pour recevoir des notifications. Vous pouvez choisir parmi les options suivantes :

       * <DNT>**Email**</DNT>: Saisissez les adresses e-mail des utilisateurs New Relic à informer du budget.
       * <DNT>**Webhook**</DNT>: Saisissez l&apos;URL du webhook pour envoyer des notifications à un point de terminaison spécifique.

    3. Sélectionnez <DNT>**Finish**</DNT>.
  </Step>

  <Step>
    ### Consulter les détails du budget

    Après avoir créé le budget, la page <DNT>**Data Budgets**</DNT> affiche les informations suivantes pour chaque budget :

    <table>
      <thead>
        <tr>
          <th>
            Nom du champ
          </th>

          <th>
            Description
          </th>
        </tr>
      </thead>

      <tbody>
        <tr>
          <td>
            <DNT>**Budget name**</DNT>
          </td>

          <td>
            Le nom que vous avez attribué au budget.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**Status**</DNT>
          </td>

          <td>
            Situation actuelle du budget.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**GB allocation**</DNT>
          </td>

          <td>
            Quelle quantité de données (Go) a été allouée à ce budget.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**GB utilized**</DNT>
          </td>

          <td>
            Quelle proportion des données allouées a été utilisée.
          </td>
        </tr>

        <tr>
          <td>
            <DNT>**Utilization percentage**</DNT>
          </td>

          <td>
            Pourcentage du budget utilisé.
          </td>
        </tr>
      </tbody>
    </table>
  </Step>
</Steps>

## Gérer les budgets existants

Pour gérer un budget existant :

1. Localisez le budget dans la liste et sélectionnez le menu à trois points.

2. Choisissez parmi les options suivantes :

   * <DNT>**View details**</DNT>: Consultez des informations détaillées sur le budget, notamment les graphiques d&apos;utilisation, le seuil, les paramètres notification, les comptes associés et les sources de données.
   * <DNT>**Edit**</DNT>: modifiez l&apos;allocation budgétaire, le seuil ou les paramètres de notification.
   * <DNT>**Delete**</DNT>: supprimer le budget.

### Afficher les détails du budget

Sélectionnez un nom de budget ou choisissez <DNT>**View details**</DNT> dans le menu à trois points pour voir :

* <DNT>**Total allocation**</DNT>: montant total des GB alloués à ce budget
* <DNT>**Utilization**</DNT>: graphique visuel montrant l&apos;utilisation au fil du temps avec une répartition au jour le jour
* <DNT>**Thresholds**</DNT>: le seuil notification que vous avez configuré
* <DNT>**Notification destinations**</DNT>: lieux où les alertes sont envoyées lorsque des seuils sont dépassés.
* <DNT>**Associated accounts**</DNT>: Les comptes inclus dans ce budget
* <DNT>**Data sources**</DNT>: informations sur les sources de données contribuant à l&apos;utilisation du budget, catégorisées par type de source
* <DNT>**Budget time range**</DNT>: la période couverte par ce budget
