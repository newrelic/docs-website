Chez New Relic, nous sommes très fiers de [NRDB](/docs/telemetry-data-platform/ingest-manage-data/get-started/nrdb-horsepower-under-hood), la base de données New Relic dans laquelle nous stockons vos données. Il rassemble toutes vos données télémétriques en un seul endroit, vous offre une vue connectée de toutes vos données et évolue à mesure que votre entreprise se développe.

Nous vous invitons à envoyer toutes vos [métriques, événements, logs et traces](/docs/data-apis/understand-data/new-relic-data-types) à NRDB, y compris ceux provenant de sources tierces. Nous reconnaissons également que certaines données peuvent ne pas être nécessaires à vos objectifs commerciaux. Vous ne devriez pas avoir à parcourir des données dont vous n’avez pas besoin pour accéder aux données dont vous avez besoin. Et vous ne devriez certainement pas avoir à payer pour cela. C&apos;est là qu&apos;interviennent nos outils de gestion des données : ils vous permettent de décider quelles données vous envoyez à New Relic et combien de temps elles doivent être stockées.

## Meilleurs coûts, meilleures performances et conformité [#benefits]

La collecte et le stockage de données dans New Relic vous permettent d&apos;analyser, de visualiser et d&apos;alerter sur toutes vos métriques, événements, logs et traces provenant de toutes vos sources. Cependant, il est important de gérer ces données pour des raisons de coût, de performances et, dans certains cas, de conformité. Notre centre de gestion des données fournit les outils dont vous avez besoin pour comprendre et contrôler la provenance de vos données, et ajuster ce qui est stocké et pendant combien de temps.

### Gestion des coûts [#calculate-data-retention]

Notre processus d’ingestion vous aide à affiner vos données. Par exemple, les données peuvent arriver à notre porte de traitement compressées et de qualité variable. Grâce à l&apos;ingestion, ces données sont décompressées, décorées avec des attributs interrogeables et évaluées. Les éléments sont supprimés ou coupés, tout cela avant que nous les écrivions dans NRDB. De cette façon, les données que vous stockez sont uniquement celles dont vous avez le plus besoin.

Vous souhaitez estimer votre ingestion de données et leur coût ? Voir [Estimer l&apos;ingestion de données](https://newrelic.com/blog/nerdlog/estimate-data-cost).

### Gestion des performances

Bien que NRDB soit une base de données incroyablement évolutive, il est également vrai que les requêtes sur d&apos;énormes ensembles de données peuvent ne pas renvoyer de résultats suffisamment rapidement pour vos besoins. Vous obtenez de meilleures performances si vous limitez les données que nous stockons ou si vous les convertissez dans un format qui les rend facilement interrogeables. Une solution pour améliorer les performances consiste à [supprimer des données](/docs/accounts/accounts/data-management/drop-data-using-nerdgraph) pour réduire la quantité de données stockées.
