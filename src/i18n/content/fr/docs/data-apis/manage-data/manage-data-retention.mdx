Pour trouver l&apos;interface utilisateur de rétention des données : dans le [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings), cliquez sur <DNT>**Manage your data**</DNT>, puis cliquez sur <DNT>**Data retention**</DNT>.

Sur la [page](https://one.newrelic.com/admin-portal/limits-data-retention/data-retention) d&amp;apos;interface utilisateur <DNT>**Data retention**</DNT>, vous pouvez consulter et gérer la durée de stockage des différents types de données. Vous souhaiterez peut-être une conservation plus courte pour certains types de données afin de protéger les informations sensibles, ou vous souhaiterez peut-être une conservation plus longue (jusqu&amp;apos;à sept ans) pour pouvoir voir et analyser les données et les tendances sur une période plus longue.

Pour en savoir plus sur les plans et les coûts de modification de la conservation, consultez [Options de données : Data Plus et d&apos;origine](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#upgrade-data-plus).

## Prolongez votre rétention des données pour une analyse et une conformité à long terme

L&apos;extension de votre conservation des données vous permet d&apos;effectuer des analyses, des visualisations et des alertes à long terme pour toutes vos métriques, événements, logs et traces sur toutes vos sources. Cependant, il est important de gérer ces données pour des raisons de coût, de performances et, dans certains cas, de conformité. Notre centre de gestion des données fournit les outils dont vous avez besoin pour comprendre et contrôler la provenance de vos données, et ajuster ce qui est stocké et pendant combien de temps.

<Callout variant="tip">
  Avec [notre option Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus), vous bénéficiez jusqu&amp;apos;à 90 jours de conservation supplémentaires (pour la plupart des types de données) et d&amp;apos;autres fonctionnalités de niveau entreprise, telles que des durées de requête plus longues, la conformité FedRAMP et HIPAA, des fonctionnalités de sécurité supplémentaires, et bien plus encore.
</Callout>

## Exigences [#requirements]

La manière dont l&apos;utilisateur gère la conservation des données peut dépendre du nombre de comptes que vous avez dans votre organisation et de ce qu&apos;ils essaient de faire.

### Organisation à compte unique [#single-account-organizations]

Si vous faites partie d&apos;une organisation qui ne possède qu&apos;un seul compte, votre [**utilisateur de facturation**](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/) peut gérer les éléments suivants :

* **Max par contrat :** il s&amp;apos;agit de la valeur la plus élevée que vous pouvez définir pour la **conservation effective** d&amp;apos;un type de données. Si vous souhaitez augmenter ce volume, consultez notre forfait [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices) .
* **Conservation efficace :** il s’agit de la durée pendant laquelle les données sont actuellement conservées pour un type de données. Pour que l&amp;apos;utilisateur puisse modifier ceci, l&amp;apos;organisation (compte) doit être sur [l&apos;édition Standard ou supérieure](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#how-pricing-works).

Si votre utilisateur de facturation ne peut effectuer aucune de ces deux tâches, contactez le support ou votre représentant New Relic.

### Organisation multi-comptes [#multiple-account-organizations]

Si vous faites partie d&apos;une organisation qui possède plusieurs comptes, l&apos;utilisateur peut gérer les éléments suivants :

* **Max par contrat :** il s&amp;apos;agit de la valeur la plus élevée que vous pouvez définir pour la **conservation effective** d&amp;apos;un type de données. Si vous souhaitez augmenter ce volume, consultez notre forfait [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices) .

* **Conservation efficace :** il s’agit de la durée pendant laquelle les données sont actuellement conservées pour un type de données. Pour que l&amp;apos;utilisateur puisse modifier ceci :

  * L&apos;organisation (compte) doit être sur [Pro ou une édition supérieure](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#how-pricing-works).
  * Le type d&apos;utilisateur doit être **Plateforme complète**.
  * L&apos;utilisateur doit disposer des autorisations **Afficher** et **Modifier** pour la fonctionnalité [**de rétention des événementsInsights **](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#data-retention)qui [pointe vers le ou les comptes](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/) pour lesquels il doit modifier **la rétention effective** .

Si votre utilisateur ne peut effectuer aucune de ces deux tâches après avoir consulté les détails ci-dessus, contactez le support ou votre représentant New Relic.

## Détails de conservation pour différents types de données [#data-details]

Dans cette section, vous trouverez des détails sur quelques types de données différents, y compris certains types de données dont les paramètres de conservation ne peuvent pas être modifiés.

<CollapserGroup>
  <Collapser id="dimensional-metrics" title="Métriques dimensionnelles (métrique API)">
    Les données [métriques dimensionnelles](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics) brutes (le type de données `Metric` ) sont agrégées (regroupées) pour quelques plages de temps différentes et ces agrégations sont conservées pendant différentes durées. Les points de données métriques dimensionnelles brutes sont stockés pendant 30 jours. Tous les points de données agrégés (l’agrégation commence à une minute) sont conservés pendant 13 mois.

    Seules les données brutes ingérées comptent dans l&apos;utilisation : les données agrégées ne le sont pas.

    Les durées de conservation métriques ne sont pas modifiables.
  </Collapser>

  <Collapser id="metric-timeslice" title="données d'intervalle de temps métrique (APM, navigateur, mobile)">
    [Les données d&apos;intervalle de temps métrique](/docs/using-new-relic/data/understand-data/new-relic-data-types#timeslice-data) sont générées par <InlinePopover type="apm" />(à la fois les mesures personnalisées et les mesures par défaut), <InlinePopover type="mobile" />et les agents de navigateur. ## Modifier la conservation des données [#adjust-retention]

Pour afficher et modifier votre conservation, accédez à [la vue Rétention des données](https://one.newrelic.com/admin-portal/limits-data-retention/data-retention) et cliquez sur <DNT>**Edit data retention**</DNT>. Vous y verrez des options pour modifier la rétention des données ou mettre à niveau vers Data Plus. Si vous ne voyez pas cela, vérifiez [les exigences](#requirements) et parlez-en à votre représentant de compte New Relic.

Options pour augmenter la rétention :

* Si vous ne l&apos;avez pas déjà fait, passez à [Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus), qui offre une conservation des données nettement plus longue et d&amp;apos;autres avantages.
* Pour Data Plus ou l’option de données originales : ajoutez la rétention des données. 30 jours supplémentaires de rétention des données pour votre organisation coûtent 0,05 $ par Go ingéré et par mois.

Points importants pour augmenter la rétention :

* Lorsque vous activez une période de conservation plus longue, la conservation est augmentée de manière globale pour tous les types de données répertoriés dans l&apos; [UI<DNT>**Data retention**</DNT> ](#find-ui). Elle ne peut pas être augmentée uniquement pour des types de données spécifiques.
* Une fois que vous avez activé une période de conservation plus longue, la réduction de la conservation pour des sources de données spécifiques n&apos;a aucun impact sur votre facturation. Pour en savoir plus sur les moyens de réduire votre ingestion de données et votre facturation, consultez [Gérer l&apos;ingestion de données](/docs/data-apis/manage-data/manage-data-coming-new-relic).
* Lorsque vous ajustez la rétention, cela peut prendre jusqu&apos;à 24 heures pour prendre effet.

### Réduire la rétention [#reduce-retention]

Lorsque vous réduisez la période de conservation d&apos;une source de données, les données existantes de ce type ne disparaîtront pas immédiatement. Ces données peuvent être visibles et renvoyées dans la requête pendant plusieurs semaines. Nous avons mis en place des mesures de protection pour garantir que vous ne supprimiez pas accidentellement des données, mais cela signifie également que la réduction des valeurs de conservation n&apos;est pas une bonne stratégie pour [supprimer rapidement des données](#deleting-data).

Pour en savoir plus sur les options permettant de réduire l’ingestion de données, consultez [Gérer l’ingestion de données](/docs/data-apis/manage-data/manage-data-coming-new-relic).

## Supprimer les données existantes [#deleting-data]

Une fois que [les données télémétriques](/docs/using-new-relic/data/understand-data/new-relic-data-types) sont signalées à la base de données New Relic et disponibles pour interrogation, ces données ne peuvent pas être modifiées ou supprimées. Il s’agit d’une décision de conception délibérée qui optimise la vitesse et les performances de New Relic. Les données expireront après la fin de leur [période de rétention des données](#adjust-retention) . Si vous souhaitez supprimer des données déjà signalées, contactez votre représentant New Relic, si vous en avez un.

## Héritage de compte [#account-structure]

Les paramètres de conservation des données d&apos;un [compte parent](/docs/accounts/original-accounts-billing/original-users-roles/mastersub-account-structure) sont hérités par ses comptes enfants. Dans ce contexte, le premier compte d&amp;apos;une organisation New Relic est considéré comme le compte parent, et les autres comptes ajoutés à l&amp;apos;organisation héritent des paramètres de ce compte.
