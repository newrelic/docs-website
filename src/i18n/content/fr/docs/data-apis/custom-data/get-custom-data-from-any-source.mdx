Pour tirer le meilleur parti de New Relic, vous devrez peut-être ou souhaiterez peut-être signaler des données personnalisées. Nous vous fournissons les outils dont vous avez besoin pour intégrer tout type de données dans New Relic.

## Découvrez nos solutions sur mesure [#options]

Les produits New Relic signalent automatiquement de nombreuses données, aucune configuration supplémentaire n&apos;est requise ! Lorsque vous utilisez des solutions monitoring telles que <InlinePopover type="apm" />, <InlinePopover type="browser" />, <InlinePopover type="mobile" />, monitoring d&apos;infrastructure ou l&apos;un de nos divers outils [d&apos;intégration et de démarrage rapide](https://newrelic.com/instant-observability), vous recevez par défaut différents types de données de performances.

Mais vous souhaiterez peut-être importer dans New Relic des données qui ne sont pas collectées par défaut. Peut-être souhaitez-vous une solution basée sur une API qui ne nécessite pas l’installation d’un agent. Peut-être souhaitez-vous importer des données télémétriques d&apos;un autre service d&apos;analyse dans New Relic. Ci-dessous, nous décrirons plusieurs façons d&apos;obtenir davantage d&apos;ensembles de données personnalisés dans New Relic.

## Rapport de données personnalisées avec l&apos;API d&apos;agent [#agent-apis]

Si vous utilisez des agents New Relic (APM, navigateur ou mobile), vous pouvez les configurer pour envoyer des données personnalisées. Pour en savoir plus, consultez la documentation de l&apos;agent que vous utilisez.

## Signaler des données d&apos;infrastructure personnalisées [#flex-integration]

Si notre infrastructure intégration ne répond pas à vos besoins, vous pouvez utiliser notre [Flex intégration](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) pour signaler tout type de infrastructuremétriques ou de configuration paramètres liés à à New Relic.

Si vous êtes une entreprise de taille moyenne à grande et que vous souhaitez utiliser la technologie [SAP ®](https://www.sap.com/) pour gérer vos opérations commerciales et vos relations clients, vous pouvez utiliser notre [intégration SAP](/docs/data-apis/custom-data/sap-integration).

## Rapporter les données des services et spécifications de télémétrie open source [#third-party-telemetry]

Vous pouvez consulter toutes nos intégrations sur [notre page de démarrage rapide](https://newrelic.com/instant-observability). Voici quelques-unes de nos intégrations et spécifications les plus populaires avec des services et spécifications de télémétrie tiers et open source :

* [OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/introduction-opentelemetry-new-relic)
* [Prometheus](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* [Zipkin](/docs/distributed-tracing/trace-api/report-zipkin-format-traces-trace-api)
* [StatistiquesD](/docs/more-integrations/open-source-telemetry-integrations/statsd/statsd-monitoring-integration)
* [Grafana](/docs/more-integrations/grafana-integrations/get-started/grafana-support-prometheus-promql)
* [Assistant de dépôt](/docs/more-integrations/open-source-telemetry-integrations/dropwizard/dropwizard-reporter)
* [Micromètre](/docs/more-integrations/open-source-telemetry-integrations/micrometer/micrometer-metrics-registry)

## Rapport attribut personnalisé et événement [#events]

Vous pouvez agrémenter certains événements rapportés par New Relic avec des métadonnées personnalisées. Pour en savoir plus, voir [attribut personnalisé](/docs/data-apis/custom-data/custom-events/collect-custom-attributes).

Vous pouvez également signaler un événement entièrement personnalisé à New Relic grâce à notre [API d&apos;événement](/docs/data-apis/ingest-apis/event-api/introduction-event-api).

Les attribut personnalisé et événement peuvent alors être requêtes et cartographiés.

## Signaler des données de trace personnalisées [#tracing]

Notre [API de trace](/docs/distributed-tracing/trace-api/introduction-trace-api) vous permet d&apos;envoyer des données de tracing distribué à New Relic et de consolider les données trace provenant de plusieurs sources en un seul endroit. Nous acceptons les données de trace dans deux formats :

* Format Zipkin
* Format New Relic (si vous n&apos;avez pas de données au format Zipkin, vous pouvez l&apos;utiliser)

## Signaler les données log personnalisées [#logs]

Si notre [intégration de logging](/docs/logs/get-started/get-started-log-management) existante ne répond pas à vos besoins, vous pouvez utiliser notre [API de log](/docs/logs/log-api/introduction-log-api) pour envoyer toutes les données log arbitraires à New Relic.

## Créer des visualisations de données personnalisées [#visualizations]

Vous souhaitez créer des requêtes, des graphiques et des dashboards personnalisés ? Pour commencer à interroger, consultez [Introduction à NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language). Lorsque vous effectuez une requête, vous pouvez prendre le graphique résultant et l&apos;ajouter à [un dashboard](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards).

Si vous avez besoin de plus de puissance et de personnalisation que ce que nos graphiques et <InlinePopover type="dashboards" />permettent, vous pouvez également créer votre propre [application New Relic](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application) ou personnaliser une application existante à partir [du catalogue d&apos;applications](https://opensource.newrelic.com/nerdpacks). Les applications New Relic sont disponibles sur notre plateforme et peuvent être partagées avec votre équipe ou le public.
