Vous pouvez envoyer des données télémétriques depuis votre AWS Virtual Private Cloud (VPC) vers New Relic via [AWS PrivateLink](https://aws.amazon.com/privatelink).

## Présentation [#overview]

L&apos;envoi de vos données New Relic via AWS PrivateLink peut :

* Réduisez vos coûts de sortie AWS
* Améliorez la sécurité en conservant vos données au sein du réseau Amazon

PrivateLink fonctionne pour l&apos;envoi [de données télémétriques](/docs/data-apis/understand-data/new-relic-data-types) à une variété de [points de terminaison](#endpoints) New Relic. Il ne peut pas être utilisé pour notre [API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) ou d&apos;autres API non ingérées.

Pour utiliser cette fonctionnalité, vous devez configurer un [point de terminaison d&apos;interface](#create) dans votre VPC que vos agents New Relic locaux et votre intégration utiliseront pour acheminer les données vers New Relic via AWS PrivateLink.

## Exigences [#requirements]

PrivateLink nécessite l&apos;option [New Relic Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus) . Si vous tentez d&apos;envoyer des données sans Data Plus, le trafic envoyé via PrivateLink sera rejeté et renvoyé avec une erreur 402.

Un service de point de terminaison est disponible uniquement dans la région où il a été créé ([voir la documentation Amazon](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)) et il est accessible à partir d&apos;autres régions à l&apos;aide du peering ou de l&apos;effet de levier interrégional ([connectivité interrégionale pour AWS PrivateLink](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)).

Avec le peering VPC et la connectivité interrégionale, veuillez noter que les clients sont responsables des coûts de transfert de données entre les régions.

Cela signifie que si votre VPC se trouve dans l&apos;une des régions prises en charge (`us-east-1`, `us-east-2` ou `us-west-2` pour les comptes américains ; `eu-central-1` pour les comptes de l&apos;UE ; `ap-northeast-1` pour les comptes JP), il vous suffit de créer le point de terminaison VPC interne nécessaire, comme décrit ci-dessous. Si votre VPC se trouve dans une région en dehors des régions prises en charge, nous vous recommandons d&apos;utiliser la [connectivité inter-régions pour AWS PrivateLink](#cross-region) ou vous pouvez [configurer une connexion d&apos;appairage](#peering).

## Connectivité interrégionale [#cross-region]

New Relic prend en charge la [connectivité inter-régions AWS PrivateLink](https://aws.amazon.com/blogs/networking-and-content-delivery/introducing-cross-region-connectivity-for-aws-privatelink/), qui vous permet de vous connecter aux points de terminaison New Relic depuis n’importe quelle région AWS sans configurer le peering VPC. Il s’agit de l’approche recommandée lorsque votre VPC se trouve dans une région autre que celles répertoriées dans [Régions et zones prises en charge](#regions-zones).

La connectivité inter-régions est prise en charge depuis toutes les régions AWS pour les points de terminaison de data center US, UE et JP de New Relic, **à l&apos;exception de** `ap-southeast-6` (Nouvelle-Zélande) et `ap-east-2` (Taipei).

## Configurer une connexion peering [#peering]

Le peering VPC n&apos;est plus requis pour se connecter depuis des régions en dehors des [régions prises en charge](#regions-zones). Nous recommandons d&apos;utiliser plutôt la [connectivité inter-régions pour AWS PrivateLink](#cross-region). Si vous préférez toujours utiliser le peering VPC, notez que les coûts de transfert de données entre les régions sont à votre charge.

### Appairage VPC [#vpc-peering]

Suivez [les instructions pour créer des connexions de peering VPC inter-régions](https://docs.aws.amazon.com/vpc/latest/peering/create-vpc-peering-connection.html), en spécifiant l&apos;ID VPC des connexions de point de terminaison que vous avez créées.

### Appairage de passerelle de transit [#transit-gateway-peering]

Suivez [les instructions pour créer un PHZ Route 53 et le partager entre les VPC](https://aws.amazon.com/blogs/networking-and-content-delivery/integrating-aws-transit-gateway-with-aws-privatelink-and-amazon-route-53-resolver), en spécifiant l&apos;ID VPC des connexions de point de terminaison que vous avez créées.
