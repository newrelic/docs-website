Nos [<InlinePopover type="browser" />](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)rapports contiennent [des données d&apos;événements](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) qui s&apos;affichent dans notre UI et qui sont disponibles pour [l&apos;interrogation et la création de graphiques](/docs/using-new-relic/data/understand-data/query-new-relic-data).

Sélectionnez un nom d&apos;événement dans le tableau suivant pour voir son attribut.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Événement
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`PageView`](/attribute-dictionary/?event=PageView)
      </td>

      <td>
        `PageView` suit les données géographiques et temporelles pour chaque chargement de page de navigateur.
      </td>
    </tr>

    <tr>
      <td>
        [`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming)
      </td>

      <td>
        `PageViewTiming` envoie chaque point de données en tant qu&apos;événement distinct dès qu&apos;il est disponible. Ceci est utile pour les pages asynchrones et dynamiques.

        Pour utiliser cet événement, vous devez disposer de la version 1149 ou supérieure de l&apos;agent de navigateur. Pour plus d&apos;informations, consultez les [exemples de requête`PageViewTiming` ](/docs/browser/new-relic-browser/page-load-timing-resources/pageviewtiming-async-or-dynamic-page-details).
      </td>
    </tr>

    <tr>
      <td>
        [`PageAction`](/attribute-dictionary/?event=PageAction)
      </td>

      <td>
        `PageAction` L&apos;événement suit les données sur chaque action effectuée sur une page. Son attribut fournit des données d&apos;application et géographiques, ainsi que des données sur les dimensions du navigateur de l&apos;utilisateur, les identifiants de session et les URL de référence et de page.
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserInteraction`](/attribute-dictionary/?event=BrowserInteraction)
      </td>

      <td>
        `BrowserInteraction` contient plusieurs attributs PageView ainsi que des attributs spécifiques aux applications monopage (SPA).
      </td>
    </tr>

    <tr>
      <td>
        [`AjaxRequest`](/attribute-dictionary/?attribute_name=&events_tids%5B%5D=8302&event=AjaxRequest)
      </td>

      <td>
        Un événement `AjaxRequest` est créé à chaque fois qu&apos;un appel Ajax est effectué. Lorsqu&apos;un appel Ajax se produit pendant un événement `BrowserInteraction`, il est agrémenté d&apos;un attribut supplémentaire lié à l&apos; interaction avec le navigateur. L&apos;attribut événement suit les informations géographiques et celles du navigateur.
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserTiming` (obsolète)](/attribute-dictionary/?event=BrowserTiming)
      </td>

      <td>
        `BrowserTiming` est un événement personnalisé qui capture les données de synchronisation SPA pour l&apos;interaction du navigateur démarrée à l&apos;aide de la méthode API SPA [createTracer](/docs/browser/new-relic-browser/browser-agent-spa-api/createtracer-browser-spa-api) personnalisée. `BrowserTiming` contient de nombreux attributs identiques utilisés par d&apos;autres événements, en particulier `AjaxRequest`. *Cette méthode n&apos;est plus utilisée dans le cadre de SPA v2+, elle et cet événement personnalisé sont donc obsolètes et ne doivent pas être utilisés.*
      </td>
    </tr>

    <tr>
      <td>
        [`JavaScriptError`](/attribute-dictionary/?event=JavaScriptError)
      </td>

      <td>
        Lorsqu&apos;une erreur JavaScript se produit, un événement `JavaScriptError` est créé. `JavaScriptError` capture les données d’erreur JavaScript ainsi que les données d’interaction du navigateur. Pour plus d&apos;informations, consultez la [page des erreurs JS](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-examine-errors-over-time) dans l&apos;UI du navigateur et l&apos; [exemple de requête NRQL ](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-browser-single-page-app-data#javascriptError-example)pour le navigateur et l&apos;événement SPA.
      </td>
    </tr>

    <tr>
      <td>
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        `Span` les données sont signalées pour [le tracing distribué](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing).
      </td>
    </tr>

    <tr>
      <td>
        [`UserAction`](/attribute-dictionary/?event=UserAction)
      </td>

      <td>
        `UserAction` l&apos;événement est capturé à la suite d&apos;une interaction de l&apos;utilisateur avec l&apos;application web. Cet événement comprend des informations sur l&apos;action et des identificateurs de cible DOM ainsi que plusieurs attributs par défaut, tels que les données d&apos;application et géographiques.
      </td>
    </tr>
  </tbody>
</table>

Pour plus de détails sur la durée de conservation de ces données, consultez [conservation des données](/docs/data-apis/manage-data/manage-data-retention).
