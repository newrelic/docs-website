[APM](/docs/apm/new-relic-apm/getting-started) signale [les données d&apos;événements](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) qui apparaissent dans certains affichages UI et sont également disponibles pour [l&apos;interrogation et la création de graphiques](/docs/using-new-relic/data/understand-data/query-new-relic-data). Sélectionnez un nom d&apos;événement dans le tableau suivant pour voir son attribut.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Événement
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="transaction-event">
        [`Transaction`](/attribute-dictionary/?event=Transaction)
      </td>

      <td>
        Une transaction est une unité de travail logique dans une application logicielle. L&apos;événement `Transaction` inclut des informations sur l&apos;application, les appels de base de données, la durée de la transaction et toutes les erreurs pouvant survenir.
      </td>
    </tr>

    <tr>
      <td id="transactionerror-event">
        [`TransactionError`](/attribute-dictionary/?event=TransactionError)
      </td>

      <td>
        Les erreurs de transaction se produisent lorsqu&apos;une demande génère une exception dans le chemin de code qui a été emprunté pour répondre à cette demande. Le nombre d&apos;erreurs de transaction n&apos;est pas égal au nombre de transactions, car vous pouvez spécifier si vous souhaitez [collecter, ignorer ou marquer les erreurs comme prévues](/docs/agents/manage-apm-agents/agent-data/manage-errors-apm-collect-ignore-or-mark-expected).
      </td>
    </tr>

    <tr>
      <td id="span-event">
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        Les événements Span sont créés lorsque des agents New Relic ou d&apos;autres outils instrumentation de tracing, instrumentent des opérations qui font partie d&apos;une trace distribuée. Voir la [définition de span](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#span).
      </td>
    </tr>
  </tbody>
</table>
