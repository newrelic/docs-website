Nos [<InlinePopover type="mobile" />](/docs/mobile-monitoring/new-relic-mobile/getting-started/introduction-new-relic-mobile)rapports sur [les données d&apos;événements](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) qui s&apos;affichent dans certains écrans UI et sont également disponibles pour [l&apos;interrogation et la création de graphiques](/docs/using-new-relic/data/understand-data/query-new-relic-data). Sélectionnez un nom d&apos;événement dans le tableau suivant pour voir son attribut.

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        Événement
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`Mobile`](/attribute-dictionary/?event=Mobile)
      </td>

      <td>
        Un événement `Mobile` est créé lorsqu&apos;un crash se produit, lorsqu&apos;une interaction se termine ou s&apos;est exécutée pendant 1 seconde, ou si une session se termine après la fermeture de l&apos;application, sa mise en arrière-plan ou son exécution pendant 10 minutes. `Mobile` événement était autrefois le seul type d&apos;événement et était généré pour chaque événement, mais il existe désormais plusieurs types d&apos;événements spécialisés.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileApplicationExit`](/attribute-dictionary/?event=mobileApplicationExit)
      </td>

      <td>
        Un événement `MobileApplicationExit` est généré lorsqu&apos;une application se termine, soit en raison d&apos;un crash, soit pour d&apos;autres raisons telles qu&apos;une mémoire insuffisante, des ANR, etc. Cet événement fournit des informations précieuses détaillées sur les circonstances entourant la résiliation de l&apos;application et comprend des attributs tels que la raison, le code de sortie et la description de la résiliation d&apos;une application.

        Actuellement, l&apos;événement `MobileApplicationExit` est signalé uniquement pour les applications mobiles Android. Il exploite l&apos;API `ApplicationExitInfo` d&apos; Android pour monitorer et signaler les différentes raisons de la résiliation de votre application.
      </td>
    </tr>

    <tr>
      <td>
        `MobileBreadcrumb`
      </td>

      <td>
        Aucun attribut n&apos;est répertorié pour cet événement ; l&apos;attribut inclura l&apos; [attribut de session](#session-list) et tout [attribut personnalisé ajouté](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile).
      </td>
    </tr>

    <tr>
      <td>
        [`MobileCrash`](/attribute-dictionary/?event=MobileCrash)
      </td>

      <td>
        L&apos;événement `MobileCrash` est créé lorsqu&apos;une application plante. `MobileCrash` inclut des attributs tels que le numéro de ligne de crash, la classe et le message de crash.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileHandledException`](/attribute-dictionary/?event=MobileHandledException)
      </td>

      <td>
        `MobileHandledException` est envoyé lorsqu&apos;une exception est détectée et est utilisé pour les exceptions non fatales signalées à New Relic à l&apos;aide de l&apos; [appel d&apos;API`recordHandledException` ](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/).

        Les exceptions seront visibles dans la requête de cet événement et sur la [page d&apos; UI <DNT>**Handled exceptions**</DNT> ](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/handled-exceptions-analyze-trends-prevent-crashes), y compris la trace des appels.
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequest`](/attribute-dictionary/?event=MobileRequest)
      </td>

      <td>
        Un événement `MobileRequest` est créé lorsqu&apos;une requête HTTP se termine avec succès, générant un code de réponse inférieur à 400.

        Les données New Relic `MobileRequest` sont activées par défaut pour :

        * [Version Android 5.15.2](/docs/release-notes/mobile-release-notes/android-release-notes/android-5152) ou supérieure

        * [iOS version 6.0.0](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-600) ou supérieure

          Pour les versions antérieures, à partir de la version Android 5.14.0 ou iOS 5.14.0, vous devez activer la fonctionnalité. Effectuez une mise à niveau vers la dernière version [Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/upgrade-android-agent) ou [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/installation/upgrade-ios-agent), ou activez l&apos;indicateur de fonctionnalité `NetworkRequests` à l&apos;aide des paramètres de configuration [Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-configuration-feature-flags) ou [iOS](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/ios-agent-configuration-feature-flags#networkRequest) .
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequestError`](/attribute-dictionary/?event=MobileRequestError)
      </td>

      <td>
        Un `MobileRequestError` est utilisé pour les erreurs HTTP ou les échecs de réseau. Les erreurs HTTP sont requests HTTP dont le code d’état est supérieur à 400. Une panne de réseau est une requête HTTP qui n’entraîne aucune réponse. L&apos;événement est envoyé lorsque la requête HTTP est terminée.
      </td>
    </tr>

    <tr id="session-list">
      <td>
        [`MobileSession`](/attribute-dictionary/?event=MobileSession)
      </td>

      <td>
        Un événement `MobileSession` est envoyé lorsqu&apos;une application est fermée, en arrière-plan ou lorsque 10 minutes d&apos;utilisation active se sont écoulées. Il s&apos;agit de la source des données de session générales utilisées par les autres applications monitoring des événements mobiles. `MobileSession` capture des attributs tels que le type d&apos;appareil, le système d&apos;exploitation de l&apos;appareil et les informations géographiques.
      </td>
    </tr>

    <tr>
      <td>
        `MobileUserAction`
      </td>

      <td>
        Aucun attribut n&apos;est répertorié pour cet événement ; l&apos;attribut inclura l&apos; [attribut de session](#session-list) et tout [attribut personnalisé ajouté](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile).

        Cet événement est enregistré lorsque l&apos;on suppose qu&apos;une action de l&apos;utilisateur a provoqué le démarrage/le retour au premier plan ou l&apos;arrêt/le retour à l&apos;arrière-plan de l&apos;agent lorsque la fonctionnalité de tracing distribué est activée.
      </td>
    </tr>

    <tr>
      <td>
        Événements personnalisés
      </td>

      <td>
        Il n&apos;y a pas de liste d&apos;attributs pour ce type d&apos;événement car il s&apos;agit d&apos;un événement personnalisé ; l&apos;attribut inclura l&apos; [attribut de session](#session-list) et tout [attribut personnalisé](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile) que vous ajoutez.
      </td>
    </tr>
  </tbody>
</table>
