[monitoring synthétique dans New Relic](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) génère des rapports sur [les données d&apos;événements](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) qui s&apos;affichent dans certains écrans UI et sont également disponibles pour [l&apos;interrogation et la création de graphiques](/docs/using-new-relic/data/understand-data/query-new-relic-data). Sélectionnez un nom d&apos;événement dans le tableau suivant pour voir son attribut.

    <tr>
      <th style={{ width: "300px" }}>
        Événement
      </th>

      <th>
        Description
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="syntheticcheck-error">
        [`SyntheticCheck`](/attribute-dictionary/?event=SyntheticCheck)
      </td>

      <td>
        `SyntheticCheck` renvoie les métriques d&apos;une exécution d&apos;un moniteur spécifique. Ces métriques incluent des informations sur la durée du moniteur, l&apos;emplacement de la vérification du moniteur, la taille des en-têtes de demande et de réponse, le type de moniteur et un horodatage.

        Chaque fois qu&apos;un moniteur Synthétique exécute une vérification, les détails de la vérification sont capturés dans le type d&apos;événement`SyntheticCheck` . `SyntheticCheck` événement contient des détails spécifiques à la vérification pour fournir une visibilité telle que le statut, le type de moniteur et la taille des en-têtes de demande et de réponse.
      </td>
    </tr>

    <tr>
      <td id="syntheticrequest">
        [`SyntheticRequest`](/attribute-dictionary/?event=SyntheticRequest)
      </td>

      <td>
        `SyntheticRequest` renvoie les résultats des requests HTTP individuelles effectuées lors d&apos;une vérification. Les données collectées comprennent des informations sur le travail, l&apos;emplacement, le type de contenu de la demande, les informations sur la durée, la taille de la demande et les informations sur le chargement de la page.

        À chaque vérification de monitoring simple ou scripté, nous capturons chaque requête HTTP individuelle effectuée pendant la vérification. Les détails HTTP sont capturés à un niveau plus granulaire que le type d&apos;événement `SyntheticCheck` .
      </td>
    </tr>

    <tr>
      <td>
        [`SyntheticPrivateLocationStatus`](/attribute-dictionary/?event=SyntheticsPrivateLocationStatus)
      </td>

      <td>
        Chaque contrôle de monitoring exécuté sur un site privé déclenche des détails de capacité pour ce site privé. Ces détails sont capturés dans un événement `SyntheticPrivateLocationStatus` . Cela permet de visualiser la capacité d&apos;un site privé et de savoir si des partenaires supplémentaires sont nécessaires pour prendre en charge la workload.
      </td>
    </tr>

    <tr>
      <td id="synthetic-private-minion">
        [`SyntheticPrivateMinion`](/attribute-dictionary/?event=SyntheticsPrivateMinion)
      </td>

      <td>
        Si vous avez un emplacement privé, comme ceux à l&apos;intérieur de votre pare-feu, vous pouvez consulter les informations concernant ces emplacements avec l&apos;événement `SyntheticPrivateMinion` .

        Chaque subordonné privé en cours d&apos;exécution envoie des détails de santé à `SyntheticPrivateMinion` toutes les 30 secondes. Cela vous permet de comprendre l&apos;état de santé du réseau privé en fonctionnement à cet endroit.
      </td>
    </tr>
  </tbody>
</table>
