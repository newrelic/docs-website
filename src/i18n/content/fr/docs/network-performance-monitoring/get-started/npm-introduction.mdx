Lorsque les performances du système diminuent, vous devez savoir si cela est dû à votre code, à votre infrastructure ou au réseau sous-jacent. Et vous devez le savoir rapidement, afin de pouvoir concentrer vos efforts. Avec monitoring du réseau, vous pouvez corréler et analyser les données des applications, infrastructure, de l&apos;expérience numérique et du réseau en un seul endroit, et comprendre comment les performances du réseau et les performances globales du système s&apos;impactent mutuellement.

<Callout variant="tip">
  monitoring réseau est désormais inclus dans l&apos;[autorisation New Relic FedRAMP Moderate](/docs/security/security-privacy/compliance/data-encryption/)!

  Utilisez le guide d&apos;installation pour [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#setup-snmp-monitoring), [les flux réseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#setup-network-flow-monitoring) et [syslogréseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#setup-network-syslog-monitoring) pour détecter et configurer automatiquement les [points de terminaison d&apos;API autorisés par FedRAMP](/docs/security/security-privacy/compliance/fedramp-compliant-endpoints/#data-ingest-apis), ou consultez le processus de configuration manuelle dans nos documents pour [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#manual-setup), [les flux réseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#manual-setup) et [syslogréseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#manual-setup).
</Callout>

monitoring du réseau ajoute le contexte des données réseau aux données d&apos;application et infrastructure que vous collectez déjà dans New Relic. En monitoring les données de votre réseau, vous pouvez :

* Analysez et comprenez les performances de l&apos;ensemble de votre stack (application et infrastructure) pour une compréhension holistique des performances de votre système.
* Ayez toutes les données sur une seule plateforme pour éliminer les angles morts.
* Voyez au premier coup d’œil si un réseau est impliqué dans un problème.

## Types de données réseau [#network-data-types]

Vous pouvez monitorer les types de données réseau suivants :

* <DNT>**Device performance via SNMP**</DNT>:Simple Network Management Protocol (SNMP) est un protocole de couche application permettant l&amp;apos;échange d&amp;apos;informations de gestion entre les périphériques réseau. Pour envoyer des données SNMP à New Relic, [configurez le monitoring des données SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring).
* <DNT>**Network syslogs**</DNT>: Il fournit des informations détaillées sur ce qui a changé, les erreurs et autres événements de l&amp;apos;appareil pour comprendre rapidement le « Pourquoi ? » derrière l&amp;apos;évolution des performances en télémétrie. Pour envoyer des syslog données[réseau, configurez réseau syslog monitoring](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring).
* <DNT>**Cloud flow logs**</DNT>:Il détaille les communications entre les hôtes et les autres services en capturant l&amp;apos;adresse IP source et de destination et le port, les métadonnées qui décrivent à la fois le point de terminaison, les performances clés du réseau, la télémétrie et l&amp;apos;état d&amp;apos;une tentative de communication spécifique. [Configurer de cloud flux log monitoring](/docs/network-performance-monitoring/setup-performance-monitoring/aws-vpc-flow-monitoring).
* <DNT>**Network flow logs**</DNT>:Il capture des informations sur le trafic IP allant et venant des interfaces réseau de votre réseau local. Pour envoyer des données de flux réseau à New Relic, [configurez le monitoring des données de flux réseau](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring).

<Callout variant="important">
  Nous vous recommandons de configurer à la fois SNMP et les données de flux réseau pour une meilleure visibilité de votre réseau. Cela fournira à la fois des mesures de performances et des modèles de trafic pour dépanner et optimiser votre réseau.
</Callout>
