## Problème

Les conflits configuration de trace se produisent lorsque vous avez activé [Infinite Tracing](/docs/understand-dependencies/distributed-tracing/infinite-tracing/introduction-infinite-tracing) pour un ou plusieurs services dans une trace, mais que d&apos;autres services de cette trace ont nos solutions de tracing distribué standard (non Infinite Tracing) activées. Vous pouvez résoudre ce problème en activant l&apos;Infinite Tracing pour tous les services d’une trace.

Un conflit de configuration de tracing peut entraîner des problèmes tels que :

* Trace fragmentée et étendues orphelines.
* Métriques d&apos;Infinite Tracing incomplètes car non prises en compte des échantillonnages de tracestandard.
* Écarts dans les mesures telles que le nombre d&apos;intervalles, le nombre de services, les durées et le nombre d&apos;erreurs.
* Résultats de recherche déroutants. Par exemple, pour les étendues de tracing standard qui envoient des en-têtes aux services instrumentés Infinite Tracing, ces étendues peuvent s&apos;afficher dans la liste trace, mais pas dans la vue en cascade trace .
* Trace manquante pour les applications de navigateur de monitoring et les applications mobiles. Étant donné qu&apos;Infinite Tracing n&apos;est pas encore disponible pour <InlinePopover type="mobile" />ou <InlinePopover type="browser" />, les étendues de ces services n&apos;apparaîtront pas dans la cascade trace lorsqu&apos;ils effectueront requests aux services compatibles Infinite Tracing.

Pour la trace avec un conflit configuration, nous afficherons uniquement les données d&apos;Infinite Tracing dans l&apos;UI car ces données sont de meilleure qualité. (Toutes les données de tracing sont interrogeables via NRQL.)

Vous pouvez utiliser notre [application d&apos;état d&apos;instrumentation](https://one.newrelic.com/dt-setup-assistant) pour voir un aperçu de votre instrumentation de trace. Pour le trouver, allez à : <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Apps &gt; Distributed tracing instrumentation status**</DNT>.
