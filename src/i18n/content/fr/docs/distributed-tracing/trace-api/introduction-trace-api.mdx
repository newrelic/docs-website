Notre API de trace est utilisée pour envoyer [des données de tracing distribué](/docs/understand-dependencies/distributed-tracing/get-started/introduction-distributed-tracing#definition) à New Relic : soit dans notre propre format générique, soit dans le format de données [Zipkin](https://zipkin.io/) . Cette API permet également de transmettre à New Relic les données trace provenant de certaines de nos intégrations et exportateurs.

Vous souhaitez tester l'API de trace ? Assurez-vous d'avoir créé un [compte New Relic](https://newrelic.com/signup) gratuit. Aucune carte de crédit requise.

Pour ignorer une partie du contenu d'introduction et démarrer rapidement, accédez à [Commencer à générer des rapports de données](#get-started).

## Qu'est-ce que l'API de trace ? [#why-use]

L'API de trace est l'un des moyens par lesquels New Relic collecte [les données de tracing distribué](/docs/understand-dependencies/distributed-tracing/get-started/introduction-distributed-tracing). Nous disposons de quelques outils de tracing qui rapportent des données via cette API ([Infinite Tracing](/docs/understand-dependencies/distributed-tracing/infinite-tracing/introduction-infinite-tracing), notre [intégrationopen source ](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic#external-data)et nos [SDK de télémétrie](/docs/telemetry-sdks-send-custom-telemetry-data-new-relic)), ou vous pouvez utiliser l&amp;apos;API directement pour créer votre propre implémentation de tracing.

Raisons d'utiliser l'API trace :

* Vous disposez de votre propre outil de tracing distribué personnalisé et souhaitez voir ces données dans New Relic sans modifier votre instrumentation.
* Vous disposez d'un outil qui émettre des données de tracing mais qui nécessite un backend pour le stockage trace .
* Vous souhaitez signaler le tracing distribué des données à New Relic sans utiliser nos [solutions installées](/docs/understand-dependencies/distributed-tracing/enable-configure/enable-distributed-tracing).
* Vous utilisez [Zipkin](https://zipkin.io/) et souhaitez voir ces données de trace dans New Relic sans modifier votre instrumentation.

Vous souhaitez comprendre comment les données de trace sont liées aux autres données New Relic ? En savoir plus sur [nos types de données](/docs/using-new-relic/data/understand-data/new-relic-data-types).

## Considérations relatives à l'échantillonnage [#sampling]

Lorsque vous utilisez l'API de trace, vous avez la possibilité d'activer [l'Infinite Tracing](/docs/understand-dependencies/distributed-tracing/infinite-tracing/introduction-infinite-tracing). L&amp;apos;utilisation ou non d&amp;apos;Infinite Tracing a des implications sur la manière dont vous configurez l&amp;apos;échantillonnage :

* <DNT>**Use your own sampling (no Infinite Tracing):**</DNT> Si vous n&amp;apos;utilisez pas Infinite Tracing, il est supposé que vous souhaitez utiliser votre propre implémentation d&amp;apos;échantillonnage pour échantillonner le tracing avant de nous l&amp;apos;envoyer. (Si vos données de trace dépassent nos [limites de données](/docs/understand-dependencies/distributed-tracing/trace-api/trace-api-general-requirements-limits), nous procédons à un échantillonnage.)
* <DNT>**Use Infinite Tracing:**</DNT> Si vous souhaitez utiliser Infinite Tracing, une approche typique consiste à nous envoyer 100 % de vos données de trace et à vous fier à [l'échantillonnage d'Infinite Tracing](/docs/understand-dependencies/distributed-tracing/get-started/how-new-relic-distributed-tracing-works#tail-based). (Lorsque vous suivez les instructions [de démarrage des rapports de données](#get-started) ci-dessous, vous aurez la possibilité d&amp;apos;activer l&amp;apos;Infinite Tracing.)

## Commencer à signaler des données [#get-started]

L'API de trace accepte les données trace dans ces deux formats JSON :

* `zipkin`: le format de données de trace Zipkin JSON v2 ([découvrez ce qu'est Zipkin](https://zipkin.io)). [Commencez à envoyer des données Zipkin](/docs/apm/distributed-tracing/trace-api/report-zipkin-format-traces-trace-api).
* `newrelic`: notre format général. Si vous n&amp;apos;avez pas de données au format Zipkin, utilisez ceci. [Commencez à envoyer des données au format New Relic](/docs/apm/distributed-tracing/trace-api/report-new-relic-format-traces-trace-api).

Vous rencontrez des difficultés pour obtenir des données à signaler ? Voir [dépannage](/docs/apm/distributed-tracing/trace-api/troubleshooting-missing-trace-api-data).

## Rechercher des données [#view-data]

Vous pouvez trouver les données envoyées via l'API de trace, ou depuis l'intégration qui utilise cette API, à ces emplacements :

* Dans notre [UI de tracing distribué](/docs/understand-dependencies/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one#find-data).
* En interrogeant : Pour plus de détails sur la manière d'interroger et de créer des graphiques personnalisés, consultez [requête tracing distributor data](/docs/understand-dependencies/distributed-tracing/ui-data/query-distributed-trace-data).
