Bien que la configuration d&apos;échantillonnage par défaut fonctionne pour tous les services, elle peut ne pas être optimale. Les agents APM fournissent des options de configuration de l&apos;échantillonnage afin que vous puissiez adapter l&apos;échantillonnage aux besoins de votre service. Cette page présente quelques exemples concrets de différents types de services et explique comment modifier la configuration de l&apos;échantillonnage pour répondre à ces besoins.

## Exemple 1 : Mauvaise répartition des points de terminaison capturés en raison de l&apos;échantillonnage en amont

La configuration par défaut utilise l&apos;échantillonneur adaptatif. Cela signifie que toute trace entrante échantillonnée par un service en amont sera également échantillonnée par le service actuel. Parfois, cela peut conduire à ce qu&apos;un point de terminaison particulier soit très échantillonné, tandis que d&apos;autres sont négligés et à peine échantillonnés.

### Solution 1 : Utilisez l&apos;échantillonneur de ratio pour échantillonner les traces échantillonnées par un service en amont

La configuration suivante résoudra ce problème en échantillonnant seulement 10 % du temps toute trace entrante ayant été échantillonnée par un service en amont. Cela libère de la place pour l&apos;échantillonnage d&apos;autres endpoints en évitant de remplir le réservoir de spans avec des traces échantillonnées par un service en amont.

```json
{
    distributed_tracing: {
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.1
                }
            }
        }
    }
}
```

### Solution 2 : Utilisez l&apos;échantillonneur de ratio pour capturer une partie des traces échantillonnées par un service en amont avec une granularité complète et le reste avec une granularité partielle

Si vous devez tout de même capturer des informations minimales sur une trace lorsqu&apos;elle est échantillonnée par un service en amont, vous pouvez utiliser la configuration suivante à la place. Cette configuration capture 5 % de toutes les traces échantillonnées par un service en amont avec une granularité complète et le reste avec une granularité partielle. En capturant la majorité des traces échantillonnées par un service en amont avec une granularité partielle, le réservoir de spans ne se remplit pas aussi vite et laisse de la place pour l&apos;échantillonnage d&apos;autres endpoints.

```json
{
    distributed_tracing:{
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.5
                }
            }
            partial_granularity: {
                enabled: true,
                type: "compact",  # If capturing AI data set this to "essential"
                remote_parent_sampled: always_on
            }
        }
    }
}
```

## Exemple 2 : Mauvaise répartition des endpoints capturés car certains endpoints sont rarement appelés

La configuration par défaut utilise l&apos;échantillonneur adaptatif. Cela signifie qu&apos;il capture environ 10 traces par minute. En raison de la probabilité aléatoire, l&apos;échantillonneur adaptatif sera plus susceptible de capturer les appels de point de terminaison qui se produisent plus fréquemment. Cela signifie que les points de terminaison rarement appelés sont plus susceptibles d&apos;être manqués et de ne pas être échantillonnés.

### Solution 1 : Utilisez l&apos;échantillonneur de ratio pour échantillonner les traces échantillonnées par un service en amont

La configuration suivante résoudra ce problème en échantillonnant seulement 10 % du temps toute trace entrante ayant été échantillonnée par un service en amont. Cela libère de la place pour l&apos;échantillonnage d&apos;autres endpoints en évitant de remplir le réservoir de spans avec des traces échantillonnées par un service en amont.

```json
{
    distributed_tracing: {
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.1
                }
            }
        }
    }
}
```
