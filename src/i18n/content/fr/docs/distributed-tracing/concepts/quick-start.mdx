Nous vous recommandons de consulter [notre documentation sur les types d&apos;échantillonnage](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works) avant de commencer pour déterminer si vous souhaitez utiliser le tracing distribué standard ou [l&apos;Infinite Tracing](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing). De plus, si vous utilisez actuellement des agents New Relic <InlinePopover type="apm" />et souhaitez activer le tracing distribué, consultez notre [guide de planification](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide).

Pour configurer le tracing distribué, vous devez effectuer trois étapes générales :

<img title="Diagram showing the three steps to set up distributed tracing." alt="Diagram showing the three steps to set up distributed tracing" src="/images/distributed-tracing_diagram_high-level-dist-tr.webp" />

1. <DNT>**Identify services:**</DNT> Identifiez et notez le point de terminaison, les services, les langues et le système utilisés pour compléter cette demande (vous aurez besoin de ces informations à l&apos;étape suivante). Si vous disposez d&apos;un diagramme d&apos;environnement comme celui-ci, vous pouvez l&apos;utiliser pour créer une liste de services gérant requests:

   <img title="A sample environment diagram showing how you might identify services." alt="A sample environment diagram showing how you might identify services." src="/images/distributed-tracing_screenshot-full_distributed-tracing-map.webp" />

2. <DNT>**Instrument services:**</DNT> instrumentez chaque service que vous identifiez afin qu&apos;il puisse envoyer vos données trace . Certains outils, tels que les <InlinePopover type="apm" />agents, instrumentent les services automatiquement, tandis que d&apos;autres outils nécessitent que vous insériez du code dans les services. 3. <DNT>**View traces:**</DNT> Après avoir instrumenté les services, générez du trafic dans votre application, puis accédez à l&apos; [UI de New Relic](https://one.newrelic.com/launcher/distributed-tracing.launcher) pour voir vos données trace .
