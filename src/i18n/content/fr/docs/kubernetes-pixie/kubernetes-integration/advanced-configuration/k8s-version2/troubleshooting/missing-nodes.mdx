## Problème

Vous [avez déployé l'agent infrastructure ](/docs/infrastructure/infrastructure-monitoring/get-started/choose-infra-install-method/)et terminé la [procédure d'installation Kubernetes ,](/install/kubernetes/) mais tous les nœuds ne s&amp;apos;affichent pas.

## Solution

Suivez ces étapes :

1. Confirmez que vous pouvez planifier l’agent d’infrastructure sur chaque nœud en exécutant cette commande :

```shell
  kubectl describe daemonset newrelic-infra
```

2. Confirmez que l’heure sur tous les nœuds est exacte. Les nœuds qui ont plus de 2 minutes d'avance ou de retard n'apparaîtront pas dans l'explorateur Cluster . La requête NRQL suivante peut être utilisée pour vérifier si c'est le cas :

```sql
FROM K8sNodeSample
SELECT latest(nr.ingestTimeMs - timestamp) / 1000 AS 'Clock offset seconds'
FACET nodeName LIMIT max SINCE 1 DAY AGO
```

3. [Récupérez le log de l'agent infrastructure ](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/k8s-version2/overview/#logs-version2)sur les nœuds qui n&amp;apos;apparaissent pas dans l&amp;apos;explorateur cluster et confirmez qu&amp;apos;il n&amp;apos;y a pas [de message d'erreur](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/k8s-version2/errors/).
