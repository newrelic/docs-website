L&apos;explorateur New Relic cluster Kubernetes utilise les données collectées par l&apos;[Kubernetes intégration](/docs/integrations/kubernetes-integration/get-started/introduction-kubernetes-integration) pour afficher l&apos;état de votre cluster, du control plane aux nœuds et au pod.

Après [avoir installé](/install/kubernetes) notre intégration Kubernetes , vous pouvez commencer à instrumenter les services qui s&apos;exécutent dans votre cluster. Pour en savoir plus sur la façon de procéder, veuillez consulter notre page sur [les services de monitoring exécutés sur Kubernetes](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes/) .

Vous pouvez en savoir plus sur l&apos;état de santé de chaque entité, explorer le log et voir les performances de vos applications. Avec l&apos; [intégration d&apos;événements](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration), tout ce qui se passe dans votre cluster devient visible, et les logs importés à l&apos;aide du [plugin de logs](/docs/logs/enable-logs/enable-logs/kubernetes-plugin-logs) sont également disponibles.

## Découvrez le navigateur Kubernetes [#navigator-preview]

L&apos;interface utilisateur de Kubernetes Navigator est une version Kubernetes de notre [New Relic Navigator](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/#view-navigator). * Rouge pour critique

* Jaune pour avertissement

* Vert pour entité non alertante

* Gris pour entité non ciblée par une condition d&apos;alerte

  <img title="New Relic - Kubernetes Navigator Alert Status" alt="New Relic - Kubernetes Navigator Alert Status" src="/images/kubernetes_screenshot-full_navigator-alerts.webp" />

## Cluster de présentation dashboard [#cluster-overview-dashboard]

Le cluster <DNT>**Overview**</DNT> dashboard peut être un outil essentiel pour monitoring et gérer votre cluster Kubernetes. Il offre une visibilité en temps réel sur la santé et les performances des applications conteneurisées exécutées sur le cluster et permet à l&apos;administrateur d&apos;identifier et de résoudre rapidement tous les problèmes pouvant survenir.

<img title="Screenshot showing the Overview dashboard" alt="Screenshot showing the Overview dashboard" src="/images/kubernetes_screenshot-full_cluster-dashboard-detail.webp" />

<figcaption>
  Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes**</DNT>. Sélectionnez votre cluster et cliquez sur <DNT>**Overview Dashboard**</DNT> dans le volet de navigation de gauche. Il affiche des données d&apos;aperçu utiles sur l&apos;état et les performances du cluster et de sa charge de travail conteneurisée.
</figcaption>

Il est conçu pour aider à répondre à des questions telles que :

* Y a-t-il des pods en attente ou en échec ?
* Quels Daemonsets, déploiements, Statefulsets, HPA ou autres ressources Kubernetes ne sont pas sains ?
* Tous les nœuds sont-ils prêts et capables d’héberger un pod ?
* Combien de pods, conteneurs, nœuds ou autres ressources Kubernetes y a-t-il dans le cluster?
* Quand un pic dans l&apos;événement d&apos;avertissement Kubernetes s&apos;est-il produit ?
* Quand un ou plusieurs pods sont-ils entrés dans un état d&apos;attente ?
* Existe-t-il des pods qui ne peuvent pas être planifiés sur un nœud ?
* Mes nœuds peuvent-ils héberger des pods supplémentaires ?

<Callout variant="tip">
  Le cluster <DNT>**Overview**</DNT> dashboard n&apos;inclut actuellement pas d&apos;informations sur les tâches ou les tâches planifiées.
</Callout>

Pour le configurer, sélectionnez la case <DNT>**Kubernetes Events**</DNT> à l&apos;étape 3 de [notre assistant d&apos;installation](/docs/integrations/kubernetes-integration/installation/kubernetes-installation-configuration#install-wizard) ou suivez les [instructions](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration#install).

## Utiliser l&apos;explorateur cluster [#cluster-explorer-use]

<Callout variant="important" title="AVIS DE FIN DE VIE">
  La nouvelle interface utilisateur de Kubernetes Navigator est destinée à remplacer l&apos;explorateur de cluster. Cependant, vous pouvez activer l’expérience d’explorateur de cluster d’origine à tout moment en désactivant le commutateur à bascule Kubernetes Navigator.

  Notez que l&apos;explorateur de cluster n&apos;est pas compatible avec [OpenTelemetry pour Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/).
</Callout>

<img width="60%;" title="New Relic - Kubernetes Navigator Toggle" alt="New Relic - Kubernetes Navigator Toggle" src="/images/kubernetes_screenshot-crop_navigator-toggle.webp" />

<figcaption>
  Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes cluster explorer &gt; Summary**</DNT>: cliquez sur le bouton bascule pour revenir à l’interface utilisateur de Kubernetes Navigator.
</figcaption>

L&apos;explorateur de cluster représente vos données de cluster les plus pertinentes sur un graphique ayant la forme d&apos;un gouvernail de navire, qui est également le logo de Kubernetes.

* Anneau extérieur : contient <DNT>**up to 24 nodes**</DNT> de votre cluster, le plus pertinent en fonction du nombre d&apos;alertes. Passez la souris sur chaque nœud pour vérifier la consommation de ressources et le pourcentage de pod allouable utilisé.

<img title="New Relic - Cluster explorer - Node Tooltip" alt="New Relic - Cluster explorer - Node Tooltip" src="/images/kubernetes_screenshot-full_cluster-explorer-node-tooltip.webp" />

* Anneaux intérieurs : contiennent les pods (<img title="Pod" alt="Pod" src="/images/kubernetes_icon_pod-0.webp" style={{height: "20px", width: "20px"}}>) de chaque nœud.</img> les pods avec des alertes actives sont affichés dans le troisième anneau le plus à l&apos;intérieur, et les pods en attente ou incapables de s&apos;exécuter sont au centre.

<img title="New Relic - Cluster explorer" alt="New Relic - Cluster explorer" src="/images/kubernetes_screenshot-crop_cluster-explorer-rings.webp" />

Passez la souris sur chaque nœud ou pod pour obtenir un aperçu rapide de son utilisation des ressources. Vous pouvez cliquer sur chaque nœud et pod pour afficher son utilisation des ressources au fil du temps ou pour obtenir plus d&apos;informations sur son état et ses alertes actives. [Les couleurs](/docs/alerts/new-relic-alerts/configuring-alert-policies/identify-entities-without-alert-policies) sont basées sur [la règle d&apos;alerte recommandée](/docs/kubernetes-pixie/kubernetes-integration/installation/recommended-alert-policies): les modules jaunes ont des alertes d&apos;avertissement actives, tandis que les modules rouges ont une critique d&apos;alerte active.

<img title="New Relic - Cluster explorer - Pod alert" alt="New Relic - Cluster explorer - Pod alert" src="/images/kubernetes_screenshot-crop_nr1-pod-alert.webp" />

<figcaption>
  Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes**</DNT>: cliquez sur n&apos;importe quel pod pour obtenir plus d&apos;informations sur son état et sa santé, et pour approfondir les données d&apos;application, la trace, le log et l&apos;événement.
</figcaption>

Cliquez sur un nœud pour voir les données suivantes :

* Statistiques des pods
* Consommation du processeur, de la mémoire et du stockage par rapport aux quantités allouables
* Quantité de pod utilisée par le nœud par rapport à la quantité de pod pouvant être allouée

Pour chaque pod, en fonction de l&apos;[intégration et de la fonctionnalité que vous avez activée](/docs/integrations/kubernetes-integration/get-started/introduction-kubernetes-integration), vous pouvez voir :

* statut du pod et métadonnées, y compris espace de nommage et déploiement
* Statut et statistiques du conteneur
* Alertes actives (avertissement et critiques)
* [Événement Kubernetes qui s&apos;est produit dans ce pod](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration)
* Données et trace APM (si vous avez [lié vos données APM](/docs/integrations/kubernetes-integration/link-your-applications/link-your-applications-kubernetes))
* Un lien vers le log des pods et des conteneurs, collecté à l&apos;aide du [plugin Kubernetes pour <InlinePopover type="logs" />dans New Relic](/docs/logs/enable-logs/enable-logs/kubernetes-plugin-logs)

Les statistiques Cluster et [control plane](/docs/integrations/kubernetes-integration/installation/configure-control-plane-monitoring) sont toujours visibles sur le côté gauche.

### Tableau des nœuds de l&apos;explorateur Cluster [#cluster-explorer-table]

Sous l&apos;explorateur cluster se trouve la table des nœuds, qui affiche tous les nœuds du cluster, l&apos;espace de nommage ou de déploiement. Comme tous les autres indicateurs d&apos;utilisation, le tableau montre la consommation par rapport [aux ressources allouables](https://kubernetes.io/docs/tasks/administer-cluster/reserve-compute-resources/#node-allocatable).

## Recherchez et filtrez vos données de cluster [#search-filter]

La principale façon de modifier la vue des données dans l&apos;explorateur cluster consiste à utiliser la barre supérieure pour rechercher un attribut ou des valeurs spécifiques. Tous les [attributs et valeurs collectés par l’intégration Kubernetes ](/docs/integrations/kubernetes-integration/understand-use-data/understand-use-data)peuvent être combinés pour affiner la vue cluster .

<img title="New Relic  - Cluster explorer - Search" alt="New Relic  - Cluster explorer - Search" src="/images/kubernetes_screenshot-full_cluster-explorer-search.webp" />

<figcaption>
  Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes cluster explorer**</DNT>: tous les attributs et points de données de votre cluster Kubernetes peuvent être utilisés pour filtrer la vue de l&apos;explorateur cluster .
</figcaption>

Vous pouvez également modifier l&apos;intervalle de temps en utilisant le sélecteur d&apos;intervalle de temps dans le coin supérieur droit. La case <DNT>**Auto-refresh**</DNT> transforme l&apos;explorateur cluster en un dashboard en temps réel qui s&apos;actualise toutes les 60 secondes.

<img title="New Relic - Cluster explorer - Time picker and Auto-refresh" alt="New Relic - Cluster explorer - Time picker and Auto-refresh" src="/images/kubernetes_screenshot-crop_cluster-explorer-picker-refresh.webp" />

<figcaption>
  Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes cluster explorer**</DNT>: Le sélecteur d&apos;intervalle de temps vous permet de sélectionner plusieurs plages de temps prédéfinies. Pour recharger les données toutes les minutes, cochez la case d&apos;actualisation automatique.
</figcaption>
