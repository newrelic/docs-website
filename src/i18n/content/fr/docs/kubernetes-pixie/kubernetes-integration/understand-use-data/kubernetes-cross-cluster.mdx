Les environnements Kubernetes modernes ont évolué vers **des flottes complexes multi-cluster **, mais les outils d&apos;observabilité traditionnels n&apos;offrent souvent qu&apos;une vue fragmentée et cloisonnée de chaque cluster. Cette interface utilisateur Kubernetes inter-cluster fournit un centre de commande unifié qui transforme cette complexité multi-cluster en une clarté à l&apos;échelle de l&apos;ensemble du parc.

Cette vue unifiée aide l&apos;ingénieur plateforme et les SRE à :

* **Unifiez l&apos;observabilité de l&apos;ensemble de la flotte** sur un dashboard unique compatible avec le moniteur de cluster des agents New Relic ou OpenTelemetry.
* **Accélérez l&apos;analyse des causes profondes** grâce à un workflow de triage guidé.
* **Réduisez les coûts** en identifiant les ressources gaspillées dans l&apos;ensemble de la flotte.
* **Offrir aux développeurs** une vision centrée sur l&apos;application pour le libre-service.

## Accédez à la nouvelle interface utilisateur [#access]

Vous pouvez accéder à la nouvelle interface utilisateur inter-cluster Kubernetes depuis l&apos;option Kubernetes dans le menu de navigation de gauche de la plateforme New Relic.

Pour accéder à l&apos;interface utilisateur inter-cluster Kubernetes :

1. Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Kubernetes**</DNT>
2. Cliquez sur le bouton **Try it out** en haut à droite de la page.

L&apos;interface utilisateur inter-cluster Kubernetes est structurée pour fournir à la fois une vue d&apos;ensemble de haut niveau et des capacités d&apos;exploration approfondie de l&apos;ensemble de votre parc Kubernetes. Les principaux composants de l&apos;interface utilisateur sont les suivants :

* **Bouton de commentaires**: Permet de partager facilement vos commentaires sur votre expérience avec l’interface utilisateur.

* **Filtres d&apos;entités**: Filtrer et trier les problèmes à l&apos;aide **d&apos;étiquettes et de valeurs** au **niveau ducluster ou du nœud**.
  * **FiltreCluster **: Filtre permettant d&apos;isoler l&apos;interface utilisateur à un seul cluster Kubernetes.

* **Tableaux de bord colorés**: Affichez des indicateurs clés de performance pertinents pour mettre en évidence les problèmes les plus importants qui nécessitent une attention particulière pour l’ensemble de la flotte.

  * Cliquer sur un tableau de bord **permet de le trier** en fonction de cette métrique, en faisant apparaître en haut le cluster et les nœuds présentant les problèmes les plus critiques.
  * Il permet également de visualiser, à l&apos;aide d&apos;un **graphique linéaire,** l&apos;évolution de la métrique sur la période sélectionnée.

* **Graphique linéaire**: Fournit l’ **évolution de la métrique** sur la période sélectionnée pour l’analyse des tendances.

* **Tableau**: Liste toutes les entités de la flotte et fournit **des données métriques détaillées** pour chaque cluster sous les cinq onglets principaux.

  * **Priorité à la santé**: Le tableau propose un filtre permettant de n’afficher que les entités « malsaines » afin de concentrer votre attention là où elle est réellement nécessaire.
  * **Recherche**: Une barre de recherche textuelle vous permet de filtrer les lignes du tableau en fonction **du texte libre** saisi.
  * **Codage couleur**: Comme les tableaux de bord, les cellules du tableau sont codées par couleur (jaune, rouge) pour attirer rapidement l&apos;attention sur les entités présentant des problèmes qui atteignent un **seuil de gravité** spécifique.

## Capacités d&apos;exploration pour le triage [#drill-down]

L&apos;interface utilisateur inter-cluster est spécifiquement conçue pour accélérer l&apos;analyse des causes profondes grâce à un **workflowde triage guidé**. Le principal mécanisme de cette analyse approfondie est la capacité à passer d&apos;une métrique à l&apos;échelle de la flotte à la vue d&apos;un seul cluster:

* **Du tableau de bord au classement cluster :** lorsque vous cliquez sur un **tableau de bord coloré** (par exemple, cluster non sain), le tableau par cluster est immédiatement réorganisé. Cela place en haut le cluster présentant les performances les plus faibles pour cette métrique spécifique, vous permettant ainsi d&apos;identifier où le problème est le plus critique au sein de votre flotte.
* **Du tableau métrique au navigateur Kubernetes :** Cliquez sur une valeur métrique dans n’importe quelle cellule du tableau pour un cluster ou un nœud spécifique pour lancer automatiquement le **navigateur Kubernetes **. La vue du navigateur est automatiquement **prédéfinie/filtrée** pour ce cluster et cette métrique spécifiques qui vous intéressent, vous permettant ainsi de poursuivre votre investigation en profondeur.
