## requête de données Kubernetes [#view-data]

Le moyen le plus simple de [requêter vos données Kubernetes ](/docs/using-new-relic/data/understand-data/query-new-relic-data)consiste à utiliser le [générateur de requêtes](/docs/query-your-data/explore-query-data/query-builder/use-advanced-nrql-mode-specify-data), qui accepte [les requêtes NRQL ](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql). Vous pouvez également utiliser l&apos; [API NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) pour récupérer les données Kubernetes.

### Kubernetes métadonnées dans les applications de monitoring APM [#apm-custom-attributes]

[La connexion de vos applications à Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes/) ajoute ces attributs à vos données d&apos;application, ainsi qu&apos;au tracing des métadonnées distribuées :

* `nodeName`
* `containerName`
* `podName`
* `clusterName`
* `deploymentName`
* `namespaceName`
