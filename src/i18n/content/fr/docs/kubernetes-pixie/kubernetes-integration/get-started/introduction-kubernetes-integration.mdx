Kubernetes est un outil open source permettant d&apos;automatiser le déploiement, la mise à l&apos;échelle et la gestion des applications conteneurisées. L&apos;intégration New Relic Kubernetes monitoring vous offre un moyen rapide et simple de voir ce qui se passe avec votre cluster et votre charge Kubernetes de travail, qu&apos;ils soient hébergés sur site ou dans le cloud.

Une fois l&apos;[intégration Kubernetes opérationnelle](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure/), vous pouvez commencer monitoring tous les services exécutés sur Kubernetes en un seul endroit avec nos [expériencesKubernetes prêtes à l&apos;emploi](/docs/integrations/kubernetes-integration/cluster-explorer/kubernetes-cluster-explorer). Il vous montre tout, du haut du control plane jusqu&apos;aux applications exécutées sur un seul pod, afin que vous puissiez accéder au cœur du fonctionnement de votre cluster .

## Monitoring Kubernetes avec New Relic [#monitoring-k8s-nr]

La gestion de Kubernetes peut être difficile. Il se passe beaucoup de choses en permanence, avec des conteneurs créés et supprimés en quelques minutes, des applications qui plantent et des ressources utilisées de manière inattendue. Notre intégration vous aide à comprendre comment utiliser Kubernetes de différentes manières, que vous l&apos;utilisiez sur site, dans le cloud ou dans une configuration hybride.

Jetez un œil à ce que vous pouvez faire avec New Relic :

* Monitorer le pour gérer et collecter les métriques de votre [Kubernetes control plane](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/configure-control-plane-monitoring) cluster.

* Collectez, traitez, explorez, interrogez et alertez sur vos données log en [transférant votre log Kubernetes ](/docs/logs/forward-logs/kubernetes-plugin-log-forwarding/)vers New Relic, vous offrant ainsi davantage de moyens de gérer vos données log .

* Créez vos propres graphiques et [interrogez](/docs/using-new-relic/data/understand-data/query-new-relic-data) toutes vos [donnéesKubernetes ](/docs/integrations/kubernetes-integration/understand-use-data/find-use-your-kubernetes-data)que notre intégration collecte en instrumentant la couche d&apos;orchestration du conteneur. Cela vous donne plus d&apos;informations détaillées sur les nœuds, l&apos;espace de nommage, le déploiement, les jeux de répliques, le pod et le conteneur.

* [Liez vos données APM à Kubernetes](/docs/integrations/kubernetes-integration/link-your-applications/link-your-applications-kubernetes) pour mesurer les performances de vos applications Web et mobiles. Vous pouvez suivre des éléments tels que le taux de requêtes, le débit, le taux d&apos;erreur et la disponibilité.

* [Services de monitoring exécutés sur Kubernetes](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes), comme Apache, NGINX, Cassandra et bien d&apos;autres. Consultez notre [tutoriel pour monitoring Redis sur Kubernetes](/docs/integrations/kubernetes-integration/link-apps-services/tutorial-monitor-redis-running-kubernetes).

* Récupérez les métriques Prometheus de n&apos;importe quelle workload du cluster grâce à la découverte de services fournie par l&apos;[agent Prometheus ](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic/#Agent).

* L&apos;intégration comprend un ensemble de [conditions d&apos;alerte prédéfinies](/docs/kubernetes-pixie/kubernetes-integration/installation/predefined-alert-policy), mais vous pouvez [créer et modifier vos alertes Kubernetes ](/docs/kubernetes-pixie/kubernetes-integration/installation/create-alerts)en fonction de vos données Kubernetes , ou ajouter un ensemble de [règles d&apos;alerte recommandées](/docs/kubernetes-pixie/kubernetes-integration/installation/recommended-alert-policies).

* Découvrez tous vos événements Kubernetes . L&apos;[intégration d&apos;événements Kubernetes ](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration)monitore les événements qui se produisent dans votre cluster Kubernetes et envoie ces événements à New Relic. Vous pouvez ensuite visualiser les données d’événement dans l’explorateur de cluster. Il est installé par défaut après [l&apos;installation de l&apos;intégration New Relic Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure/).

* Signalez les données à Kubernetes qui [déclenchent la mise à l&apos;échelle automatique pod ](/docs/journey-demand/autoscale-your-infra)afin que vous disposiez toujours de ressources disponibles lorsque la demande augmente, comme lors d&apos;un événement de pointe de demande.

Ces fonctionnalités s&apos;ajoutent aux données que New Relic rapporte déjà pour [les processus conteneurisés exécutés sur des hôtes instrumentés](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/docker-instrumentation-infrastructure).

## Vérifiez le code source [#source-code]

Cette intégration est un logiciel open source. Cela signifie que vous pouvez [parcourir son code source](https://github.com/newrelic/nri-kubernetes) et envoyer vos propres améliorations, ou vous pouvez créer votre propre fork et le construire. Pour plus d&apos;informations, consultez le fichier [README](https://github.com/newrelic/nri-kubernetes/blob/main/README.md) .
