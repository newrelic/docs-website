Lorsque vous [installez l'intégration Kubernetes ](/install/kubernetes/)pour la première fois, nous déployons un ensemble par défaut de conditions d&amp;apos;alerte et de dashboard recommandés sur votre compte qui constituent la base des conditions d&amp;apos;alerte et du dashboard sur votre cluster Kubernetes. les conditions d&amp;apos;alerte sont regroupées en deux politiques : [<DNT>Kubernetes alert policy</DNT>](#k8s-alert-conditions) et [<DNT>Google Kubernetes Engine alert policy</DNT>](#google-alert-policies).

Bien que nous ayons essayé de répondre aux cas d'utilisation les plus courants dans tous les environnements, il existe un certain nombre d'alertes supplémentaires que vous pouvez configurer pour étendre la politique par défaut. Consultez [Prise en main des alertes New Relic](/docs/tutorial-create-alerts/create-new-relic-alerts/) pour en savoir plus sur les alertes.

## Ajout de la condition d'alerte et du dashboard recommandés [#add-recommended-alert-policy]

Pour ajouter une règle d'alerte et un dashboard recommandés, suivez ces étapes :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com) &amp;gt; Integrations &amp;amp; Agents**</DNT>.

2. Dans le champ de recherche, tapez `kubernetes`.

   <img title="Integrations & Agents" alt="Integrations & Agents" src="/images/kubernetes_screenshot-crop_integrations-agents.webp" />

3. Sélectionnez l'une de ces options :

   * <DNT>**Kubernetes**</DNT>:Pour ajouter l&amp;apos;ensemble par défaut de conditions d&amp;apos;alerte recommandées et un dashboard.

   * <DNT>**Google Kubernetes Engine**</DNT>:Pour ajouter l&amp;apos;ensemble par défaut de conditions d&amp;apos;alerte recommandées par le moteur Google Kubernetes et un dashboard.

4. Cliquez sur <DNT>**Begin installation**</DNT> si vous devez installer l’intégration Kubernetes ou cliquez sur <DNT>**Skip this step**</DNT> si vous avez déjà configuré cette intégration.

5. Selon l’option que vous avez sélectionnée à l’étape 3, vous verrez différentes ressources à ajouter.

<img title="Add the default set of recommended alert conditions" alt="Add the default set of recommended alert conditions" src="/images/kubernetes_screenshot-crop_recommended-alert-conditions.webp" />

<figcaption>
  Ensemble par défaut de conditions d’alerte recommandées et d’un dashboard lorsque vous sélectionnez <DNT>**Kubernetes**</DNT> à l’étape 3.
</figcaption>

<img width="60%;" title="Add the default set of recommended Google Kubernetes engine alert conditions" alt="Add the default set of recommended Google Kubernetes engine alert conditions" src="/images/kubernetes_screenshot-crop_recommended-google-alert-conditions.webp" />

<figcaption>
  Ensemble par défaut de conditions d'alerte recommandées par le moteur Google Kubernetes et d'un dashboard lorsque vous sélectionnez <DNT>**Google Kubernetes Engine**</DNT> à l&amp;apos;étape 3.
</figcaption>

6. Cliquez sur <DNT>**See your data**</DNT> pour voir un dashboard avec vos données Kubernetes dans New Relic.

## Comment voir la règle d'alerte recommandée [#see-recommended-alert-policy]

Pour afficher la règle d'alerte recommandée que vous avez ajoutée, procédez comme suit :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Alerts**</DNT>.

2. Cliquez sur <DNT>**Alert Policies**</DNT> dans le volet de navigation de gauche.

3. Vous verrez <DNT>**Kubernetes alert policy**</DNT> et <DNT>**Google Kubernetes engine alert policy**</DNT>.

   <img title="Add Kubernetes alerts" alt="Add Kubernetes alerts" src="/images/kubernetes_screenshot-crop_recommended-alert-policies.webp" />

## Comment voir le dashboard Kubernetes [#see-dashboards]

Il existe une collection de dashboards prédéfinis recommandés pour vous aider à visualiser instantanément vos données Kubernetes pour les cas d'utilisation courants. Consultez [Gérer votre dashboard recommandé](/docs/query-your-data/explore-query-data/dashboards/prebuilt-dashboards) pour savoir comment voir ces dashboards.

    ```sql
    FROM K8sContainerSample
    SELECT sum(containerCpuCfsThrottledPeriodsDelta) / sum(containerCpuCfsPeriodsDelta) * 100 
    WHERE clusterName IN ('YOUR_CLUSTER_NAME') 
    AND namespaceName IN ('YOUR_NAMESPACE_NAME') 
    FACET containerName, podName, namespaceName, clusterName
    ```

    FACET nodeName, clusterName
