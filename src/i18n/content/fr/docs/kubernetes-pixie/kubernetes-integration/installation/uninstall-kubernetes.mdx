L&apos;intégration New Relic Kubernetes peut être facilement désinstallée de votre plateforme.

Assurez-vous que vous opérez dans le contexte approprié sur la machine sur laquelle vous exécutez Helm et `kubectl`. Vous pouvez vérifier les contextes disponibles avec cette commande :

```bash
kubectl config get-contexts
```

Passez au contexte souhaité en utilisant :

```bash
kubectl config use-context CONTEXT_NAME
```

## Désinstaller Kubernetes à l&apos;aide de Helm [#uninstall-helm]

Si vous avez utilisé Helm pour installer l’intégration Kubernetes, exécutez :

```bash
helm uninstall newrelic-bundle -n newrelic
```

<Callout variant="important">
  L&apos;installation guidée utilise `newrelic-bundle` comme nom de sortie par défaut et `newrelic` comme espace de nommage par défaut. Si vous installez notre bundle sous un nom de sortie ou un espace de nommage différent, vous devrez remplacer les arguments dans la commande ci-dessus : `helm uninstall <RELEASE-NAME> -n <RELEASE-NAMESPACE>`. Vous pouvez utiliser `helm list --all-namespaces` et rechercher le bundle `nri-bundle` pour trouver le nom de sortie et l&apos;espace de nommage de sortie.
</Callout>

## Désinstaller Kubernetes à l&apos;aide du manifeste [#uninstall-manifest]

Si vous avez installé l’intégration Kubernetes à l’aide d’un manifeste, utilisez le même manifeste pour la désinstaller :

```bash
kubectl delete -f newrelic.yaml
```
