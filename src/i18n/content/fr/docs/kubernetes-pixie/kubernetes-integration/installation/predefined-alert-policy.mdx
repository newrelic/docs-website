Lorsque nous déployons l&apos;intégration New Relic Kubernetes pour la première fois dans un compte, nous déployons un ensemble par défaut de [conditions d&apos;alerte](/docs/alerts/create-alert/create-alert-condition/alert-conditions/) prédéfinies sur votre compte. Vous n&apos;avez rien d&apos;autre à faire. Ces conditions sont regroupées dans une politique appelée <DNT>**Kubernetes default alert policy**</DNT>. Cette politique ne dispose pas de [canal de notification](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-control-where-send-alerts) par défaut pour empêcher les notifications indésirables.

Vous pouvez personnaliser le [seuil](/docs/alerts/create-alert/set-thresholds/set-thresholds-alert-condition/) de condition d&apos;alerte, ainsi que gérer vos [politiques](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/) et envoyer [des notifications](/docs/alerts/get-notified/intro-notifications/).

## Comment voir votre règle d&apos;alerte prédéfinie [#see-predefined-alert-policy]

Pour afficher la règle d&apos;alerte prédéfinie par défaut après avoir déployé l&apos;intégration, procédez comme suit :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts**</DNT>.

2. Cliquez sur <DNT>**Alert Policies**</DNT> dans le volet de navigation de gauche.

3. Cliquez sur <DNT>**Kubernetes default alert policy**</DNT>.

   <img title="Add Kubernetes alerts" alt="Add Kubernetes alerts" src="/images/kubernetes_screenshot-crop_default-alert-policy.webp" />
