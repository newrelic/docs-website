## Problème

L&apos;intégration de Kubernetes nécessite `kube-state-metrics`. Si cela manque, vous verrez une erreur comme celle-ci dans le log du conteneur `nrk8s-ksm` :

```shell
  time="2022-06-21T09:12:20Z" level=error msg="retrieving scraper data: retrieving ksm data: discovering KSM endpoints: timeout discovering endpoints"
```

## Solution

Vérifiez les éléments suivants :

* `kube-state-metrics` n&amp;apos;a pas été déployé dans le cluster.
* `kube-state-metrics` est déployé à l&amp;apos;aide d&amp;apos;un déploiement personnalisé.
* Il existe plusieurs versions de `kube-state-metrics` en cours d&amp;apos;exécution et l&amp;apos;intégration Kubernetes ne trouve pas la bonne.

L&apos;intégration Kubernetes détecte automatiquement `kube-state-metrics` dans votre cluster, en utilisant par défaut l&amp;apos;étiquette `app.kubernetes.io/name=kube-state-metrics` dans tout l&amp;apos;espace de nommage.

<Callout variant="tip">
  Vous pouvez modifier le comportement de découverte dans le `ksm.config` des valeurs du [graphique Helm](https://github.com/newrelic/nri-kubernetes/blob/main/charts/newrelic-infrastructure/values.yaml) .
</Callout>
