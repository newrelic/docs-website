## Problème

Vous rencontrez un problème avec le [dockerhub New Relic](https://hub.docker.com/u/newrelic) et [`registry.k8s.io`](https://github.com/kubernetes/registry.k8s.io) de Google lors de l&apos;installation.

## Solution

Vérifiez que vous avez ajouté leur domaine à votre liste de domaines autorisés. L&apos;installation extrait les images du conteneur à partir de cet emplacement. Vous pouvez [tester la connectivité à `registry.k8s.io`](https://kubernetes.io/blog/2023/03/10/image-registry-redirect/#how-can-i-check-if-i-am-impacted) pour trouver le domaine de registre Google supplémentaire à ajouter à votre liste blanche. `registry.k8s.io` redirige généralement vers votre domaine de registre local. Par exemple, `asia-northeast1-docker.pkg.dev` en fonction de votre région.
