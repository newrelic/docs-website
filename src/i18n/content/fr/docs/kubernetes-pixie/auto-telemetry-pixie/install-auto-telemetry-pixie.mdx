Pour démarrer avec la télémétrie automatique avec Pixie, commencez par notre [installation guidée](https://one.newrelic.com/marketplace?state=fc4edc3d-8214-a314-7432-3e62d7de43d7). L&amp;apos;installation guidée dévoile l&amp;apos;intégration de New Relic avec Pixie sur votre cluster. Vous n&amp;apos;avez pas besoin d&amp;apos;effectuer de configuration ou d&amp;apos;installation supplémentaire pour commencer à utiliser Pixie.

Si vous souhaitez installer Auto-télémétrie avec Pixie sur plusieurs clusters, relancez le guide d'installation pour chaque cluster supplémentaire.

## Conditions générales pour l'utilisation de Pixie [#general-prereqs]

Prêt à commencer ? Si vous n'en avez pas encore, [créez un compte New Relic](https://newrelic.com/signup). C&amp;apos;est gratuit, pour toujours !

Conseils avant de commencer :

* Consultez cet [aperçu de la sécurité des données Pixie](/docs/auto-telemetry-pixie/pixie-data-security-overview) pour connaître les mesures à prendre pour sécuriser vos données.
* Assurez-vous que votre cluster dispose de suffisamment de mémoire. Pixie nécessite au minimum 1 Go de mémoire sur chaque nœud de votre cluster. Consultez cette [présentation de la gestion de la mémoire Pixie](/docs/auto-telemetry-pixie/manage-pixie-memory) pour connaître les mesures à prendre pour optimiser l’utilisation de la mémoire.
* Assurez-vous que Pixie fonctionne en mode privilégié.
* Consultez les autres [exigences techniques de Pixie](https://docs.px.dev/installing-pixie/requirements/).

## Installer à l'aide du processus d'installation guidée [#install-beginning]

1. Ouvrez notre [guide d'installation monitoring New Relic](https://one.newrelic.com/marketplace?state=fc4edc3d-8214-a314-7432-3e62d7de43d7).
2. Sélectionnez le compte que vous souhaitez utiliser pour l’installation guidée et cliquez sur <DNT>**Continue**</DNT>. Remarque : si vous avez un seul compte, vous ne verrez pas cette option.
3. Remplissez les champs restants et cliquez sur <DNT>**Continue**</DNT> pour terminer le reste de l&amp;apos;installation.

## Installer à partir du fichier de commande/manifeste HELM (yaml) [#install-helm-command-manifest]

Si vous êtes arrivé au processus d'installation guidée en suivant un lien depuis Pixie ou depuis New Relic, vos étapes commencent ici.

1. Sélectionnez le compte et le cluster pour l'installation. Si nécessaire, sélectionnez un espace de nommage.

   <Callout variant="important">
     Actuellement, Pixie fonctionne mieux sur un cluster comportant jusqu'à 100 nœuds. (Dépasser 100 nœuds peut entraîner une utilisation excessive de la mémoire et l'échec de l'exécution du script.) Rappel amical : la mise à l'échelle automatique peut rapidement augmenter le nombre de vos nœuds.
   </Callout>

2. Cliquez sur <DNT>**Continue**</DNT>.

3. Sélectionnez les données que vous souhaitez collecter, observer et déboguer, puis cliquez sur <DNT>**Continue**</DNT>.

4. Sur la page <DNT>**Choose install method**</DNT> , sélectionnez <DNT>**Helm**</DNT> ou <DNT>**manifest**</DNT>.

   <CollapserGroup>
     <Collapser className="freq-link" id="helm-method" title="Méthode Helm">
       Exécutez la commande Helm fournie sur votre ligne de commande. Si vous êtes préoccupé par la quantité de données Pixie que vous ingérerez, consultez nos [stratégies pour réduire l'ingestion](/docs/auto-telemetry-pixie/manage-pixie-data).

       Helm installe un bundle contenant l'agent New Relic Infrastructure, une intégration pour regrouper Prometheus métriques et Kubernetes événement, et l'intégration Pixie. Le déploiement prend quelques minutes. Pour voir l’état de l’installation sur le cluster, exécutez `kubectl get pods -n newrelic`.

       Pour obtenir des informations générales sur l'installation d'une intégration Kubernetes, consultez notre documentation sur [les exigences d'installation de Helm](/docs/integrations/kubernetes-integration/installation/install-kubernetes-integration-using-helm/#h2-compatibility-and-requirements).
     </Collapser>

     <Collapser className="freq-link" id="manifest-method" title="méthode manifeste">
       Exécutez la commande fournie dans votre console et insérez le chemin vers votre manifeste téléchargé. Si vous exécutez votre cluster Kubernetes dans le cloud, consultez les étapes supplémentaires dans la [documentation Kubernetes](/docs/integrations/kubernetes-integration/installation/kubernetes-integration-install-configure/#cloud-platforms).
     </Collapser>
   </CollapserGroup>

5. Cliquez sur <DNT>**Continue**</DNT> pour ouvrir la page <DNT>**Listening for data**</DNT> .

6. Lorsque vous recevez le message <DNT>**See your data**</DNT>, cliquez sur <DNT>**Kubernetes Cluster Explorer**</DNT> pour voir votre cluster.

   La télémétrie automatique avec Pixie peut redémarrer après l'installation. Ceci est dû à la fonctionnalité de mise à jour automatique.

## Si vous liez le mauvais compte Pixie et New Relic [#link-wrong-accounts]

Contactez le support pour dissocier un compte Pixie de votre compte New Relic. Sachez que si vous dissociez un compte Pixie créé automatiquement via l'installation guidée, vous perdrez l'accès à ce compte Pixie.

## Mettre à niveau l'intégration de Pixie [#upgrade]

Pour déterminer votre version d'intégration Pixie, consultez notre [documentation de dépannage Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/troubleshooting/get-pixie-integration-version).

L'intégration Pixie est installée dans le cadre de l'intégration Kubernetes de New Relic. Pour mettre à niveau l'intégration Kubernetes, consultez notre [documentation d'installation de Kubernetes](/install/kubernetes#update).

<Callout variant="important" title="AVIS DE FIN DE VIE">
  Après août 2022, nous cesserons de prendre en charge plusieurs fonctionnalités, y compris la version 1 de notre intégration avec Pixie. Pour plus de détails, notamment sur la manière dont vous pouvez facilement vous préparer à cette transition, consultez notre [publication sur le forum d'assistance](https://discuss.newrelic.com/t/july-2022-end-of-life-announcements/188318).
</Callout>

## Désinstaller l'intégration Pixie [#uninstall]

L'intégration Pixie est installée dans le cadre de l'intégration Kubernetes de New Relic. Pour désinstaller l’intégration Kubernetes, consultez notre [Désinstaller l’intégration Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/uninstall-kubernetes).
