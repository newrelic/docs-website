Utilisez l&apos;onglet <DNT>**Live debugging with Pixie**</DNT> pour interroger l&apos;ensemble complet des données télémétriques que Pixie collecte dans le stockage à court terme sur votre cluster.

Découvrez-en plus sur le modèle de stockage hybride pour l&apos;intégration de New Relic Pixie [ici](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/overview).

L&apos;onglet <DNT>**Live debugging with Pixie**</DNT> contient une vue intégrée de l&apos;interface utilisateur Pixie. Les vues dans Pixie sont alimentées par le script PxL — script écrit dans le langage PxL de Pixie — pour interroger et visualiser les données télémétriques capturées par eBPF.

Pixie fournit de nombreux [scripts PxLopen source ](https://github.com/pixie-io/pixie/tree/main/src/pxl_scripts)qui vous permettent de déboguer :

* Traffic sous plusieurs formats : HTTP et HTTPs (y compris cryptés), DNS, Postgres, MySQL, Cassandra, Redis, Kafka. En savoir plus : [Tutoriel sur le tracing des demandes](https://docs.px.dev/tutorials/pixie-101/request-tracing/).
* Performances des requêtes de base de données. En savoir plus : [tutoriel de profilage de requête de base de données](https://docs.px.dev/tutorials/pixie-101/database-query-profiling/).
* Cartes de services pour savoir quels services communiquent entre eux. En savoir plus : [Tutoriel sur les performances des services](https://docs.px.dev/tutorials/pixie-101/service-performance/).
* Cartes de trafic réseau pour savoir quels nœuds communiquent entre eux. En savoir plus : [tutorielmonitoring du réseau](https://docs.px.dev/tutorials/pixie-101/network-monitoring/).
* Utilisation des ressources par nœud et pod. En savoir plus : [Tutoriel Infra santé](https://docs.px.dev/tutorials/pixie-101/infra-health/).
* Pics d&apos;utilisation du processeur. En savoir plus : [Tutoriel sur le profilage continu du processeur](https://docs.px.dev/tutorials/pixie-101/profiler/).

## Exécuter un script PxL [#run-script]

<Callout variant="important">
  Pour de meilleurs résultats, sélectionnez `5 minutes` dans le sélecteur d&apos;intervalle de temps lorsque vous utilisez l&apos;onglet <DNT>**Live Debugging with Pixie**</DNT> . Pour examiner les données Pixie moins récentes, consultez la section [Explorer les données Pixie](/docs/kubernetes-pixie/auto-telemetry-pixie/understand-use-data/explore-pixie-data) .
</Callout>

Pour exécuter un script dans l’onglet <DNT>**Live debugging with Pixie**</DNT> :

1. Sélectionnez un script dans le menu déroulant `script` .
2. Remplissez tous les arguments de script requis. Les arguments obligatoires sont indiqués par un `*` à côté du nom de l&apos;argument.
3. Exécutez le script en utilisant le bouton `RUN` en haut à droite ou en utilisant le raccourci clavier : `ctrl+enter` (Windows, Linux) ou `cmd+enter` (Mac).

## Suivre les liens profonds du script [#deep-links]

Les vues dans Pixie contiennent des liens profonds qui vous permettent de naviguer facilement entre les entités Kubernetes .

Par exemple, la table <DNT>**Nodes**</DNT> dans le script `px/cluster` (illustré ci-dessus) génère une liste de tous les nœuds de votre cluster. Cliquez sur un nœud pour accéder au script `px/node` pour ce nœud particulier. Le script `px/node` affiche un aperçu du pod d&apos;un nœud particulier, de l&apos;utilisation du processeur et de la mémoire, ainsi que du trafic réseau.
