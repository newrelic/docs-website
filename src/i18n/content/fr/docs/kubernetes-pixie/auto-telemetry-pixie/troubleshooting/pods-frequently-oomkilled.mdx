## Problème

Vous avez terminé la [procédure d&apos;installation](/docs/kubernetes-pixie/auto-telemetry-pixie/install-auto-telemetry-pixie) pour l&apos;intégration de Pixie de New Relic, mais un ou plusieurs pods `vizier-pem` sont fréquemment OOMKilled.

## Solution

Une partie de l&apos;avantage du déploiement sur Kubernetes est que Kubernetes va OOMKill les pods qui utilisent trop de ressources pour protéger les autres pods du cluster.

Lorsque le pod `vizier-pem` de Pixie dépasse la limite de mémoire, il est OOMKilled. Kubernetes fait cela pour protéger les autres applications, afin que cela ne cause pas de problèmes sur d&apos;autres pods. Pour les pods `vizier-pem` qui sont régulièrement OOMKilled, essayez [déployé Pixie avec plus de mémoire](/docs/kubernetes-pixie/auto-telemetry-pixie/advanced-configuration/manage-pixie-memory#set-memory-limit).
