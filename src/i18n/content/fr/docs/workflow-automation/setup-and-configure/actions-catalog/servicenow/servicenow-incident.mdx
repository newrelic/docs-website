Cette page fournit une référence pour les actions d&apos;incident ServiceNow disponibles dans le catalogue des actions d&apos;automatisation des workflows. Ces actions vous permettent de créer et de gérer des incidents dans ServiceNow.

## Prérequis

Avant d&apos;utiliser les actions ServiceNow dans l&apos;automatisation workflow, assurez-vous de disposer de :

* Une instance ServiceNow avec les permissions appropriées.
* Une clé API REST ServiceNow pour l&apos;authentification (passée en tant qu&apos;en-tête `x-sn-apikey`).
* Les autorisations nécessaires pour créer et gérer les incidents dans votre instance ServiceNow.

Dans l&apos;entrée de l&apos;action, `apiKey` doit être fourni. Il est utilisé pour authentifier l&apos;utilisateur — la clé est transmise dans l&apos;en-tête de la requête `x-sn-apikey`.

## Configurer l&apos;authentification par clé API

Pour plus d’informations, consultez [Créer une politique d’accès à l’API REST avec des profils d’authentification par clé API](https://www.servicenow.com/docs/r/platform-security/authentication/create-api-access-policy.html) dans la documentation ServiceNow.

Pour générer et configurer une clé API REST dans ServiceNow :

<Steps>
  <Step>
    ### Créer un profil d&apos;authentification entrante

    1. Accédez à <DNT>**System Web Services &gt; API Access Policies &gt; Inbound Authentication Profile**</DNT>.
    2. Créez un nouveau **profil d’authentification par clé API**.
    3. Définissez le **Auth Parameter** comme en-tête HTTP (recommandé) en utilisant le nom de paramètre `x-sn-apikey`.
  </Step>

  <Step>
    ### Générer une clé API REST

    1. Accédez à <DNT>**System Web Services &gt; API Access Policies &gt; REST API Key**</DNT>.
    2. Créez une nouvelle clé associée à un compte de service dédié qui dispose des rôles et permissions appropriés.
    3. Enregistrez et copiez la valeur du jeton généré — c&apos;est une clé secrète.
  </Step>

  <Step>
    ### Configurer une politique d&apos;accès API

    1. Accédez à <DNT>**System Web Services &gt; API Access Policies &gt; REST API Access Policies**</DNT>.
    2. Créez une nouvelle politique, sélectionnez l&apos;API REST cible (par exemple, **Table API** ou une **API REST scriptée**).
    3. Ajoutez le **Inbound Authentication Profile** à la liste associée de cette politique.
  </Step>
</Steps>
