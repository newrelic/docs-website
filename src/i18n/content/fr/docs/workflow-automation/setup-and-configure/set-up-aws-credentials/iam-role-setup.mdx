Un rôle IAM permet à New Relic d&apos;utiliser des identifiants temporaires dans votre compte AWS sans nécessiter de clés d&apos;accès permanentes. Cette approche présente plusieurs avantages :

* Les identifiants sont renouvelés automatiquement.
* L&apos;accès est limité dans le temps par conception.
* Toutes les actions sont enregistrées dans AWS CloudTrail
* Conforme aux bonnes pratiques de sécurité AWS

## Prérequis

Avant de créer un rôle IAM, assurez-vous d&apos;avoir :

* Un compte AWS avec l&apos;autorisation de créer des rôles IAM
* Votre [ID de compte New Relic](/docs/accounts/accounts-billing/account-structure/account-id)
* Accès administrateur à la [console AWS IAM](https://console.aws.amazon.com/iam/)

## Configurer le rôle IAM

<Steps>
  <Step>
    ### Créez le rôle dans AWS [#create-role]

    1. Connectez-vous à la [console AWS IAM](https://console.aws.amazon.com/iam/)

    2. Accédez à **Roles** et cliquez sur **Create role**

    3. Sous **Trusted entity type**, sélectionnez **AWS account**

    4. Sélectionnez **Another AWS account**

    5. Dans le champ **Account ID**, saisissez : `253490767857`

    6. Dans **Options**, cochez la case **Require external ID.**

    7. Dans le champ **External ID**, saisissez l&apos;ID de votre compte New Relic et cliquez sur **Next**.
       * Vous ne l&apos;avez pas ? [Trouvez votre identifiant de compte ici](/docs/accounts/accounts-billing/account-structure/account-id)

    8. Sur la page **Add permissions**, attachez des stratégies en fonction de vos workflows. Par exemple :

       * Pour le workflow EC2 : Attachez `AmazonEC2ReadOnlyAccess` ou créez une stratégie personnalisée
       * Pour le workflow SQS : Attachez `AmazonSQSFullAccess` ou limitez-le à des files d’attente spécifiques.
       * Pour les autres services, consultez les [exemples de stratégies AWS](/docs/workflow-automation/setup-and-configure/set-up-aws-credentials/policy-examples)

    9. Cliquez sur **Next**.

    10. Saisissez un nom de rôle : `NewRelicWorkflowAutomationRole` (ou le nom de votre choix)

    11. Ajoutez éventuellement une description : **Permet à l&apos;automatisation des workflows New Relic d&apos;effectuer des actions dans AWS**, puis cliquez sur **Create role**.
  </Step>

  <Step>
    ### Vérifiez la politique de fiducie [#trust-policy]

    Après avoir créé le rôle, vérifiez la relation de confiance :

    1. Dans la console IAM, sélectionnez le rôle que vous venez de créer.
    2. Cliquez sur l&apos;onglet **Trust relationships**
    3. Veuillez vérifier que la politique correspond à cette structure (remplacez `<YOUR_NR_ACCOUNT_ID>` par votre identifiant de compte réel) :

    ```json
      {
          "Version": "2012-10-17",
          "Statement": [
              {
                  "Effect": "Allow",
                  "Action": "sts:AssumeRole",
                  "Principal": {
                      "AWS": "arn:aws:iam::253490767857:root"  ← Must be this account
                  },
                  "Condition": {
                      "StringEquals": {
                          "sts:ExternalId": "<YOUR_NR_ACCOUNT_ID>"  ← Must match your NR account
                      }
                  }
              }
          ]
      }
    ```

    ### Ce qu&apos;il faut vérifier

    * Le compte AWS principal est `253490767857` (compte New Relic)
    * L&apos;ID externe correspond exactement à votre ID de compte New Relic
    * L&apos;action est `sts:AssumeRole`

    <Callout variant="caution">
      **La stratégie de confiance ne correspond pas ?** Si l&apos;ID de compte est différent, supprimez le rôle et recréez-le. L&apos;ID de compte doit être `253490767857`.
    </Callout>
  </Step>

  <Step>
    ### Copiez votre ARN de rôle [#role-arn]

    Vous aurez besoin de l&apos;ARN du rôle pour configurer les workflows.

    <Callout variant="important">
      Les ARN de rôle sont des identifiants de ressources, et non des identifiants sensibles. Ne les stockez pas dans le [gestionnaire de secrets](/docs/workflow-automation/limitations-and-faq/workflow-best-practices#secure-credentials)— collez-les directement dans les configurations de workflow.
    </Callout>

    **Pour obtenir l&apos;ARN de votre rôle :**

    1. Dans la console IAM, sélectionnez votre rôle
    2. Dans la section **Summary**, repérez le champ **ARN**
    3. Copiez l&apos;ARN complet ; il ressemble à ceci : `arn:aws:iam::123456789012:role/NewRelicWorkflowAutomationRole`

    Le rôle est désormais prêt. Utilisez l&apos;ARN dans vos workflows.
  </Step>
</Steps>
