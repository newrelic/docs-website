Le [dashboard d&apos;automatisation des workflows](https://onenr.io/0gR7XE8LXjo) offre un emplacement centralisé pour visualiser, gérer et monitorer tous les workflows de votre compte. Utilisez le dashboard pour modifier la configuration, suivre l&apos;historique des versions, gérer les calendriers d&apos;exécution et résoudre les problèmes.

## Avant de commencer

Avant de gérer le workflow, assurez-vous d&apos;avoir :

* Au moins un workflow dans votre compte (à partir d&apos;un modèle ou personnalisé).
* Modifier et supprimer les autorisations du workflow.

<Callout variant="important">
  **Désactiver l&apos;automatisation des workflows ?** Si vous prévoyez de désactiver Workflow Automation via Feature Control Management (FCM), assurez-vous que tous les workflows planifiés sont supprimés au préalable. Consultez [Désactiver l&apos;automatisation des workflows](/docs/workflow-automation/troubleshooting#disabling-workflow-automation-fcm) pour les étapes détaillées.
</Callout>

**Vous débutez dans le monde du workflow ?** Commencez par [utiliser un modèle](/docs/workflow-automation/create-a-workflow-automation/use-a-template) ou [créez votre propre workflow](/docs/workflow-automation/create-a-workflow-automation/create-your-own) pour construire votre premier workflow.

## Afficher le workflow [#view-workflows]

Le dashboard **Workflow Automation** s&apos;affiche différemment selon que votre compte possède des workflows.

### Accéder au dashboard

Accédez à **All Capabilities &gt; Workflow Automation**

### État vide

Si votre compte ne contient aucun workflow, le dashboard affiche un état vide avec des options pour créer votre premier workflow.

### Onglets de Dashboard

Une fois que vous avez créé des workflows, le dashboard affiche deux onglets :

**Onglet Summary** (par défaut) : Affiche les métriques et les visualisations au niveau du compte :

**Principales métriques**:

* **Total workflows**: Nombre de tous les workflows de votre compte
* **Total number of steps executed**: Nombre cumulé d&apos;étapes sur l&apos;ensemble des exécutions de workflow
* **Currently running workflows**: Nombre d&apos;exécutions actives

**Graphiques de performance**:

* **Total run count of all workflows**: Graphique linéaire montrant les tendances d&apos;exécution des workflows au fil du temps
* **Top 5 most frequently run workflows**: Graphique à barres affichant les workflows ayant le plus grand nombre d&apos;exécutions
* **Ratio of completed to failed workflows**: Diagramme circulaire affichant les pourcentages et les nombres de succès et d&apos;échecs

**Tableau des exécutions récemment échouées**: affiche les exécutions de workflow en échec avec les colonnes suivantes :

* **Workflow**: Nom du workflow échoué
* **Version**: Version du workflow qui a échoué
* **Failed on**: Horodatage de l&apos;échec
* **Run ID**: Identifiant unique de l&apos;exécution en échec
* **Failed step**: Étape où l&apos;échec est survenu
* **Error message**: Description de l&apos;erreur

Chaque widget de graphique inclut un menu contextuel (trois points) pour afficher la requête, obtenir l&apos;image ou créer une condition d&apos;alerte

<img title="Image of the summary tab in the Workflow Automation dashboard showing account-level metrics and visualizations" alt="Image of the summary tab in the Workflow Automation dashboard showing account-level metrics and visualizations" src="/images/workflow-automation-summary-tab.webp" />

**Onglet Workflows**:

**Recherche**: utilisez la barre **Search in all fields** pour trouver des workflows à partir de n&apos;importe quelle valeur de colonne

Le **tableau All workflows** affiche ces colonnes :

* **Favorite** (icône étoile) : marquer les workflows comme favoris
* **Name**: Identifiant de workflow
* **Description**: Objectif ou fonction du workflow
* **Last run on**: Horodatage de la dernière exécution
* **Last run status**: résultat de l&apos;exécution (affiché sous forme de badges colorés : vert pour Terminé, rouge pour Échoué)
* **Latest version**: Numéro de version actuel
* **Created on**: Date à laquelle le workflow a été créé
* **Actions menu** (trois points) : Accéder aux opérations de workflow

Les valeurs vides du tableau s&apos;affichent sous la forme `—`

**Menu des actions de la ligne** (cliquez sur les trois points à côté du workflow) :

* **Run**: Exécutez le workflow immédiatement
* **Schedule**: Créer une exécution planifiée
* **Open in editor**: Modifier la configuration du workflow
* **Duplicate**: Créer une copie du workflow
* **Delete**: Supprimer définitivement le workflow

Cliquez sur un nom de workflow pour afficher sa [page Vue d&apos;ensemble de l&apos;entité](/docs/workflow-automation/manage-workflows/workflow-entity-overview) avec des métriques détaillées, l&apos;historique des exécutions, les planifications et les logs.

## Modifier un workflow [#edit-a-workflow]

Mettre à jour le nom, la description ou la configuration du workflow.

1. Accédez à **All Capabilities &gt; Workflow Automation**

2. Localisez le workflow que vous souhaitez modifier

3. Cliquez sur les points de suspension (trois points verticaux) à côté du nom du workflow

4. Sélectionnez **Open in editor**

5. Mettez à jour l&apos;un de ces éléments :

   * **Name** - L&apos;identifiant du workflow
   * **Description** - Ce que fait le workflow
   * **Configuration** - Paramètres d&apos;entrée, actions ou logique

6. Cliquez sur **Save**

Vous pouvez également accéder à l&apos;éditeur depuis la [page Vue d&apos;ensemble de l&apos;entité](/docs/workflow-automation/manage-workflows/workflow-entity-overview) en cliquant sur le bouton **Edit**.

## Gérer les versions [#manage-versions]

L&apos;automatisation des workflows crée automatiquement une nouvelle version à chaque mise à jour d&apos;une définition workflow. Toutes les versions précédentes restent disponibles à titre de référence.

### Épinglage de version pour la planification

Lorsque vous planifiez un workflow, vous devez l&apos;associer à une version spécifique. Cette exigence garantit que les exécutions planifiées continuent d&apos;utiliser la même version testée même lorsque vous mettez à jour la définition workflow.

**Démarrez et planifiez l&apos;épinglage de la version de prise en charge des workflows :**

* Lors de la planification d&apos;un workflow, spécifiez la version à exécuter.
* Lors du lancement manuel d&apos;un workflow, sélectionnez la version à exécuter.
* Les calendriers existants continuent d&apos;utiliser leur version épinglée jusqu&apos;à ce que vous les mettiez à jour.

Cela vous permet de :

* Mise à jour du workflow sans incidence sur les calendriers en cours
* Tester manuellement les nouvelles versions avant de mettre à jour les calendriers
* Maintenir la stabilité du flux de production

## Dupliquer un workflow [#duplicate-a-workflow]

Dupliquez le workflow lorsque vous devez adapter un modèle ou workflow existant à un usage différent tout en conservant l&apos;original intact.

1. Accédez à **All Capabilities &gt; Workflow Automation**
2. Localisez le workflow que vous souhaitez dupliquer.
3. Cliquez sur les points de suspension (trois points verticaux).
4. Sélectionner **Duplicate**
5. Le système crée une copie avec **(Copy)** ajouté au nom
6. Renommez et personnalisez le workflow dupliqué selon vos besoins.

**Scénarios de duplication courants :**

* Adapter un modèle workflow à votre cas d&apos;utilisation spécifique
* Création d&apos;un workflow pour une autre équipe ayant des exigences similaires
* Expérimenter des changements significatifs tout en préservant l&apos;original

<Callout variant="important">
  **Ne dupliquez pas les workflows pour la gestion de version** - le système le fait automatiquement. Dupliquer des workflows uniquement à des fins de contrôle de version ou de rollback peut vous faire atteindre les limites de workflows. Consultez [Gérer les versions](#manage-versions) pour plus d&apos;informations sur le versionnage automatique.
</Callout>

## Supprimer un workflow [#delete-a-workflow]

Supprimez les workflows dont vous n&apos;avez plus besoin.

1. Accédez à **[one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Workflow Automation**
2. Localisez le workflow que vous souhaitez supprimer
3. Cliquez sur le menu « Plus d’options » (trois points verticaux).
4. Sélectionner **Delete**
5. Confirmez la suppression dans la boîte de dialogue.

<Callout variant="caution">
  La suppression d&apos;un workflow est définitive et ne peut pas être annulée. Toutes les versions et l&apos;historique d&apos;exécution sont supprimés. Si vous pensez avoir besoin du workflow ultérieurement, envisagez de le désactiver au lieu de le supprimer.
</Callout>

**Ce qui est supprimé :**

* Toutes les versions du workflow
* Historique complet des exécutions et des logs
* Toutes les exécutions planifiées associées au workflow

**Avant de supprimer, vérifiez :**

* Aucune automatisation critique ne dépend de ce workflow
* Vous disposez d&apos;une copie de sauvegarde au cas où vous en auriez besoin plus tard.
* Toutes les exécutions planifiées sont identifiées et comptabilisées.
