Cette API récupère et renvoie une liste de tous les aperçus de définition de workflow sous la portée donnée.

<Callout variant="important">
  La définition du workflow YAML n&apos;est pas disponible avec cette requête. Un YAML de définition de workflow n&apos;est retourné qu&apos;en utilisant une requête `GetWorkflowDefinition` `/{actor/account(:id)/workflowAutomation/workflow(:name, :version)}`.
</Callout>

## Exemple

```graphql
{
  actor {
    account(id: 11933347) {
      workflowAutomation {
        workflows(cursor: "xyz123") {
          results {
            definition {
              name
              scope {
                id
                type
              }
              version
            }
          }
        }
      }
    }
  }
}
```

## Réponse

La requête retourne :

* `workflows.results`: Un tableau des aperçus de définition de workflow, chacun contenant :

  * `definition.name`: Le nom de la définition du workflow
  * `definition.scope.id`: L&apos;ID du compte
  * `definition.scope.type`: Le type de portée
  * `definition.version`: Le numéro de version de la définition du workflow
