Cette API lance l&apos;exécution d&apos;un workflow pour une définition workflow spécifiée.

<Callout variant="important">
  Le nombre d&apos;exécutions workflow simultanées et le nombre d&apos;étapes exécutées dans une exécution donnée sont définis dans les [limites du système de workflow](/docs/workflow-automation/limitations-and-faq/workflow-limits).
</Callout>

### option logLevel

L&apos;option `logLevel` contrôle la quantité d&apos;informations de logging générées pendant l&apos;exécution du workflow.

**Valeurs disponibles :**

* `NONE` (par défaut) : aucune sortie de log ne sera générée.
* `INFO`: Fournit des informations au niveau de l&apos;étape.
* `DEBUG`: Ajoute un attribut aux logs de l&apos;étape démarrée avec toutes les entrées de l&apos;étape d&apos;action.

## Exemple

Utilisez le [générateur de requêtes](https://one.newrelic.com/nerdgraph-graphiql?state=b8f3b3a9-8d3e-c03e-5da4-34b98a57ff8f) pour créer et éditer un workflow. Voir [l&apos;explorateur d&apos;API NerdGraph](/docs/apis/nerdgraph/get-started/nerdgraph-explorer).

```graphql
mutation {
  workflowAutomationStartWorkflowRun(
    definition: {name: "json-parsing"}
    workflowInputs: [
      {key: "checkOperational", value: "false"}
      {key: "checkNonOperational", value: "true"}]
    scope: {id: "11933347"}
    idempotencyKey: "45e65924-5316-479c-a977-f6adfa17e07a"
    options: {logLevel: INFO}
  ) {
    runId
  }
}
```

## Réponse

La mutation retourne :

* `runId`: L&apos;identifiant unique de l&apos;exécution du workflow. Utilisez cet ID avec les API `WorkflowRunInfo`, `SignalWorkflowRun` ou `StopWorkflowRun`.
