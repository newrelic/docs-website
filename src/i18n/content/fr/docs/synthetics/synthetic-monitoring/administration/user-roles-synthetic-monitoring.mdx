Si vous êtes un [utilisateur complet de la plateforme](/docs/accounts/accounts-billing/new-relic-one-pricing-users/users-roles#user-type), vous avez théoriquement accès à toutes les fonctionnalités de New Relic, mais votre capacité à utiliser toutes ces fonctionnalités dépend du rôle qui vous est attribué au sein de votre organisation New Relic.

Les détails liés aux autorisations dépendent du fait qu&apos;un utilisateur New Relic utilise notre nouveau modèle d&apos;utilisateur ou notre [modèle d&apos;utilisateur](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) d&apos;origine.

## utilisateur sur notre nouveau modèle utilisateur [#new-pricing]

Pour plus de détails sur le fonctionnement des autorisations New Relic, consultez [les concepts de gestion des utilisateurs](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#groups). Pour voir les fonctionnalités liées aux synthèses granulaires, voir [Capacités](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).

## utilisateur sur modèle utilisateur original [#original-pricing]

Pour les utilisateurs sur notre modèle utilisateur d&apos;origine, voici quelques détails :

<CollapserGroup>
  <Collapser id="default-access" title="Modèle d'utilisateur d'origine - accès par défaut">
    * [Les administrateurs](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model#roles) ont accès intégral à toutes les fonctionnalités monitoring Synthétique sur un compte.

    * [L&apos;utilisateur](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) a accès à toutes les fonctionnalités monitoring de Synthétique, à l&apos;exception [des informations d&apos;identification sécurisées](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers) et de la possibilité de supprimer le moniteur, le temps d&apos;arrêt du moniteur et le site privé.

    * [Les utilisateurs restreints](/docs/accounts-partnerships/accounts/account-billing-usage/users-roles#roles) sur les comptes dont la tarification originale est basée sur le produit ont un accès en lecture seule à toutes les fonctionnalités monitoring de Synthétique, à l&apos;exception de :

      * [Identifiants sécurisés](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
      * site privé (bien qu&apos;ils puissent interroger une quantité limitée de données via NRQL)

      Cela signifie que l&apos;utilisateur avec ce rôle n&apos;a pas accès à toutes les fonctionnalités de création, de modification et de suppression pour la fonctionnalité monitoring Synthétique (ainsi qu&apos;à la capacité d&apos;affichage des informations d&apos;identification sécurisées) jusqu&apos;à ce qu&apos;un rôle avec les fonctionnalités appropriées lui soit attribué.
  </Collapser>

  <Collapser id="custom-access" title="Modèle d'utilisateur d'origine - rôles personnalisés">
    Pour commencer, suivez [les procédures standard pour créer un rôle personnalisé](/docs/accounts/accounts/roles-permissions/add-roles-permissions#create/docs/accounts/accounts/roles-permissions/add-roles-permissions#create). Vous pouvez ajouter les capacités <DNT>**View**</DNT> et <DNT>**Edit**</DNT> à n’importe quel rôle pour les éléments suivants :

    * [Moniteurs](/docs/synthetics/new-relic-synthetics/using-monitors)
    * [Script de monitoring](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-synthetics-api-tests)
    * [site privé](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations)
    * [Monitorer les temps d&apos;arrêt](/docs/synthetics/new-relic-synthetics/using-monitors/monitor-downtimes-disable-monitoring-during-scheduled-maintenance-times)
    * [Identifiants sécurisés](/docs/synthetics/new-relic-synthetics/using-monitors/secure-credentials-store-credentials-information-scripted-browsers)
  </Collapser>
</CollapserGroup>
