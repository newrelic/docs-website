Ce guide couvre la validation de l&apos;infrastructure, la configuration du daemon Docker et le déploiement d&apos;image requis pour prendre en charge les tests en double stack (IPv4 et IPv6) dans votre environnement de site privé en utilisant Docker.

## Prérequis [#prerequisites]

Avant de commencer, assurez-vous d’avoir les éléments suivants :

* <DNT>**Docker**</DNT>: Docker doit être installé et en cours d&apos;exécution sur votre instance hôte

* <DNT>**Permissions**</DNT>: vous devez avoir un accès sudo ou root pour modifier la configuration du système

* <DNT>**Public images**</DNT>: La prise en charge du double stack nécessite les versions d&apos;image suivantes ou ultérieures, disponibles sur Docker Hub :

  * [Gestionnaire de tâches `519` ou version ultérieure](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-519/): `newrelic/synthetics-job-manager`
  * [Runtime Ping `1.65.0` ou version ultérieure](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.65.0/): `newrelic/synthetics-ping-runtime`
  * [Runtime de l&apos;API Node `rc1.15` ou version ultérieure](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.15/): `newrelic/synthetics-node-api-runtime`
  * [Runtime du navigateur Node `rc1.15` ou version ultérieure](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.15/): `newrelic/synthetics-node-browser-runtime`

<Steps>
  <Step>
    ## Valider votre infrastructure [#validate-infrastructure]

    Avant de continuer, vérifiez que votre machine hôte et votre réseau sont configurés pour le réseau Dual Stack.

    Testez la connectivité IPv6 en exécutant `ping6 ipv6.google.com` ou `ping -6 ipv6.google.com`, ou la commande équivalente pour votre machine.

    <Callout variant="important">
      Si le ping échoue, assurez-vous que votre VPC, votre sous-réseau et vos groupes de sécurité sont configurés pour autoriser le trafic IPv6 avant de poursuivre cette installation. Votre environnement doit prendre en charge à la fois IPv4 et IPv6 pour le fonctionnement en double stack.
    </Callout>
  </Step>

  <Step>
    ## Vérifier que Docker est prêt pour le dual stack [#configure-docker]

    Assurez-vous que votre installation Docker est configurée pour gérer à la fois le trafic IPv4 et IPv6. Pour vérifier que l&apos;IPv6 est activé sur Docker, exécutez :

    ```bash
    docker info | grep -i ipv6
    ```

    Le résultat devrait confirmer qu&apos;IPv6 est activé. Si ce n’est pas le cas, consultez la [documentation de Docker](https://docs.docker.com/config/daemon/ipv6/) pour activer la prise en charge de la double stack avant de continuer.
  </Step>

  <Step>
    ## Lancement du gestionnaire de tâches Synthetics [#launch-job-manager]

    Les images d&apos;environnement d&apos;exécution compatibles Dual stack sont disponibles publiquement sur Docker Hub et seront récupérées automatiquement par le gestionnaire de tâches. Exécutez le conteneur du gestionnaire de tâches Synthetics avec votre clé de site privé :

    <Callout variant="tip">
      Remplacez `<YOUR_PRIVATE_LOCATION_KEY>` par votre clé de site privé réelle depuis l&apos;UI de New Relic.
    </Callout>

    ```bash
    docker run \
      -e PRIVATE_LOCATION_KEY=<YOUR_PRIVATE_LOCATION_KEY> \
      -d \
      --restart unless-stopped \
      -v /var/run/docker.sock:/var/run/docker.sock:rw \
      newrelic/synthetics-job-manager
    ```

    Le gestionnaire de tâches Synthetics s&apos;exécutera désormais avec la prise en charge de la double stack activée, vous permettant de monitorer et de tester les points de terminaison IPv4 et IPv6.
  </Step>

  <Step>
    ## Vérifier l&apos;installation [#verify-installation]

    Pour vérifier que votre gestionnaire de tâches Synthetics fonctionne correctement :

    1. Vérifiez l&apos;état du conteneur :

       ```bash
       docker ps
       ```

    2. Consultez les logs du gestionnaire de tâches :

       ```bash
       docker logs <container_id>
       ```

    3. Dans l&apos;UI de New Relic, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Synthetic monitoring &gt; Private locations**</DNT> et vérifiez que votre site privé s&apos;affiche comme sain.
  </Step>
</Steps>

## Quelle est la prochaine étape ? [#whats-next]

Après avoir configuré votre site privé avec Dual Stack activé, vous pouvez :

* [Créez des monitorers Synthétiques](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors) qui testent à la fois les points de terminaison IPv4 et IPv6 en utilisant votre site privé
* [Configurez les paramètres du gestionnaire de tâches](/docs/synthetics/synthetic-monitoring/private-locations/job-manager-configuration) pour les options avancées
* [Monitorer votre site privé](/docs/synthetics/synthetic-monitoring/private-locations/monitor-private-locations) : santé et performances
