En plus de l&apos;environnement de script basé sur WebDriverJS de monitoring Synthétique standard, New Relic prend en charge de nombreux modules Node.js populaires pour améliorer vos [navigateurs scriptés](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) et vos tests API.

## Importer des modules optionnels [#import-procedure]

Pour importer un module pris en charge, suivez la procédure d&apos;importation standard correspondante. Par exemple:

* Pour importer `crypto-js`, appelez :

  ```js
  var crypto = require('crypto-js');
  ```

* Pour importer `chai`, appelez :

  ```js
  var assert = require('chai').assert;
  ```

## Modules tiers pris en charge

Différentes versions du moniteur scripté Synthétique prennent en charge différents modules tiers. Pour plus d&apos;informations sur les modules pris en charge par différentes versions du moniteur, consultez le [tableau de comparaison des versions du moniteur](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment).

## Modules non pris en charge

Vous pouvez utiliser n&apos;importe quel module de nœud dans un [minion privé conteneurisé (CPM)](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms). Pour obtenir des instructions sur la configuration des modules personnalisés pour les minions privés conteneurisés, consultez la [configuration CPM](/docs/synthetics/new-relic-synthetics/private-locations/containerized-private-minion-cpm-configuration#custom-modules).

Le fonctionnement des modules non pris en charge n&apos;est pas garanti et ils ne sont pas couverts par le support.
