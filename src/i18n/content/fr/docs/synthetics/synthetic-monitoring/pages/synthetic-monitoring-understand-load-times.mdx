Utilisez la page <DNT>**Resources**</DNT> pour comprendre l’impact sur le temps de chargement de chaque ressource sur votre site Web. Utilisez les graphiques [de temps de chargement des ressources](#resources-load-time), [de temps passé par des tiers](#third-parties) et [de codes de réponse irréguliers](#response-codes) pour obtenir un aperçu de haut niveau et utilisez la [liste des ressources](#resources-list) pour identifier les ressources à fort impact telles que les fichiers CSS ou les images volumineux.

Vous pouvez également étudier une ressource spécifique avec [des mesures de ressources détaillées](#detailed-metrics), qui regroupent les informations de performances pour une seule ressource. Allez encore plus loin et [visualisez un chargement de cette ressource](#resource-results), en utilisant les en-têtes de demande et de réponse exacts pour identifier un problème de performances.

## Comprendre les métriques de ressources [#understanding]

Utilisez la page <DNT>**Resources**</DNT> pour comprendre l’impact sur les performances de chaque ressource de votre site Web :

* <DNT>**Resources list**</DNT>:Répertorie les ressources sur votre site Web monitoré par <DNT>**average download time**</DNT>. Utilisez cette liste pour optimiser les performances de votre site Web en réduisant les temps de téléchargement. Vous pouvez également afficher le temps de chargement d&apos;une ressource particulière en la recherchant dans le champ <DNT>**Search all resources**</DNT> . Sélectionnez une ressource pour [afficher les métriques de ressources individuelles](#individual).
* <DNT>**Load time**</DNT>: Représente graphiquement le temps de chargement moyen de toutes les ressources, réparti par type de ressource (par exemple, `jpeg`, `json` ou `javascript`). Utilisez ce graphique pour comprendre comment le temps de chargement de votre site Web évolue au fil du temps.
* <DNT>**Time spent by third parties**</DNT>: Représente graphiquement le temps de chargement moyen de toutes les ressources tierces, telles que les polices Web, une chronologie Twitter intégrée ou une image hébergée en externe. Utilisez ce graphique pour voir si les sites Web tiers affectent les performances de votre site Web.
* <DNT>**Irregular response codes**</DNT>: Représente graphiquement le nombre de codes de réponse HTTP négatifs (également appelés [codes d&apos;état HTTP](http://en.wikipedia.org/wiki/List_of_HTTP_status_codes)) reçus par le moniteur. Ce graphique inclut toutes les réponses dans les plages 300 (redirection), 400 (erreur client) et 500 (erreur serveur). Il ne compte pas les réponses dans les tranches 100 ou 200. Synthetics collecte également un certain nombre de [codes de réponse personnalisés](/docs/synthetics/new-relic-synthetics/using-monitors/synthetics-response-codes).

En analysant ces métriques, vous pouvez repérer les ressources problématiques et améliorer leurs performances. Par exemple, une image de logo non optimisée peut entraîner des ralentissements pour chaque visiteur de votre site, tandis qu&apos;un outil de suivi fourni uniquement par un serveur basé aux États-Unis peut entraîner des performances lentes pour les visiteurs non américains.

## Afficher les métriques détaillées des ressources [#detailed-metrics]

Vous pouvez également afficher des mesures détaillées pour chaque ressource sur votre site. Pour afficher des métriques détaillées, [recherchez une ressource spécifique](#locate-resource) ou sélectionnez simplement une ressource intéressante sur la page <DNT>**Resources**</DNT> . Après avoir sélectionné une ressource, vous pouvez afficher les métriques d&apos;aperçu de cette ressource ou [afficher la charge d&apos;une ressource individuelle](#resource-results):

* <DNT>**Max duration**</DNT>, <DNT>**Min duration**</DNT>, <DNT>**Avg duration**</DNT>: Mesures de temps de chargement pour cette ressource sur la [période sélectionnée](#time-frame).
* <DNT>**For 50% of requests**</DNT>: Le temps de téléchargement maximal pour <DNT>**fastest**</DNT> 50 % des requests. Cela signifie que 50 % des requests pour cette ressource ont été traitées dans le délai spécifié ou plus rapidement.
* <DNT>**For 95% of requests**</DNT>: Le temps de téléchargement maximal pour <DNT>**fastest**</DNT> 95 % des requests. Cela signifie que 95 % des requests pour cette ressource ont été traitées dans le délai spécifié ou plus rapidement.
* <DNT>**Average load time**</DNT>: Représente graphiquement le temps de chargement moyen de cette ressource sur la [période sélectionnée](#time-frame). Utilisez ce graphique pour vérifier que les optimisations de votre site Web (telles que la réduction de la taille d&apos;une image ou la minimisation de votre CSS) fonctionnent.

Vous pouvez également [masquer et afficher les éléments du graphique](#hide-unhide), filtrer par emplacement et trier la liste [des résultats des ressources](#resource-results) par <DNT>**Download time**</DNT> ou <DNT>**DNS lookup**</DNT>. Passez la souris sur le nom de la ressource pour afficher le chemin complet (utile avec des noms de ressources générés dynamiquement très longs).

## Afficher les charges de ressources individuelles [#resource-results]

Après avoir consulté [les mesures détaillées](#detailed-metrics) d&apos;une ressource particulière, vous pouvez sélectionner une charge de ressource individuelle pour une compréhension approfondie d&apos;un incident de mauvaise performance. Vous pouvez ensuite afficher une répartition du temps de connexion total pour comprendre où le problème s&apos;est produit et examiner les en-têtes de demande et de réponse.
