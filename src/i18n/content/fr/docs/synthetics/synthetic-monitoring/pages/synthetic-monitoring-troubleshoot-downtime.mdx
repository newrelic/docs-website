Utilisez la page <DNT>**Failures**</DNT> pour localiser et résoudre les incidents de temps d&apos;arrêt ou d&apos;autres erreurs. Un incident de temps d&apos;arrêt se produit chaque fois qu&apos;un moniteur ne parvient pas à s&apos;exécuter complètement. Par exemple, un moniteur de ping est « down » lorsque la requête GET échoue, tandis qu&apos;un moniteur de navigateur scripté est « down » si une partie du script échoue à s&apos;exécuter. Après avoir localisé un temps d&apos;arrêt intéressant, sélectionnez-le pour [afficher les résultats détaillés de cet incidentde temps d&apos;arrêt](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) et résoudre le problème.

## Voir la page des échecs [#viewing]

Pour accéder aux échecs de votre moniteur : <DNT>**[one.newrelic.com &gt;Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; (sélectionnez un moniteur) &gt; <DNT>**Failures**</DNT>.

## Afficher les temps d&apos;arrêt individuels [#individual]

Vous pouvez sélectionner des incidents de temps d&apos;arrêt individuels pour les visualiser plus en détail. En fonction de l&apos;échec spécifique, un résultat de temps d&apos;arrêt peut inclure uniquement le message d&apos;erreur du serveur (tel que <DNT>**Server replied with &quot;HTTP 500&quot; error**</DNT>), ou une [vue en cascade](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) complète ou partielle.

Les résultats des temps d&apos;arrêt incluent des cascades lorsque seule une partie d&apos;un moniteur est exécutée correctement. Par exemple, une [redirection 301](http://en.wikipedia.org/wiki/HTTP_301) peut renvoyer vers une page Web défaillante. Le navigateur exécute correctement la redirection, mais la page Web de destination renvoie une erreur. Utilisez le message d&apos;erreur ou la cascade pour résoudre l&apos;incident de temps d&apos;arrêt.
