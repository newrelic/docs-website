Utilisez la page <DNT>**Failures**</DNT> pour localiser et dépanner les événements d&apos;alerte de temps d&apos;arrêt ou d&apos;autres erreurs. Un événement d&apos;alerte de temps d&apos;arrêt se produit chaque fois qu&apos;un monitor ne parvient pas à s&apos;exécuter complètement. Par exemple, un moniteur ping est « down » lorsque la requête GET échoue, tandis qu&apos;un moniteur de navigateur scripté est « down » si une partie du script échoue. Après avoir localisé un temps d&apos;arrêt intéressant, sélectionnez-le pour [afficher les résultats détaillés de cet événement d&apos;alerte de temps d&apos;arrêt](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) et effectuer un dépannage.

## Voir la page des échecs [#viewing]

Pour accéder aux échecs de votre moniteur : <DNT>**[one.newrelic.com &gt;Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; (sélectionnez un moniteur) &gt; <DNT>**Failures**</DNT>.

## Afficher les temps d&apos;arrêt individuels [#individual]

Vous pouvez sélectionner des événements d&apos;alerte de temps d&apos;arrêt individuels pour les afficher plus en détail. Selon la défaillance spécifique, un résultat d&apos;indisponibilité peut inclure uniquement le message d&apos;erreur du serveur (comme <DNT>**Server replied with &quot;HTTP 500&quot; error**</DNT>), ou une [vue en cascade](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) complète ou partielle.

Les résultats de temps d&apos;arrêt incluent des graphiques en cascade lorsque seule une partie d&apos;un moniteur s&apos;est exécutée correctement. Par exemple, une [redirection 301](http://en.wikipedia.org/wiki/HTTP_301) pourrait pointer vers une page web défaillante. Le navigateur exécute correctement la redirection, mais la page web de destination renvoie une erreur. Utilisez le message d&apos;erreur ou le graphique en cascade pour dépanner l&apos;événement d&apos;alerte d&apos;indisponibilité.
