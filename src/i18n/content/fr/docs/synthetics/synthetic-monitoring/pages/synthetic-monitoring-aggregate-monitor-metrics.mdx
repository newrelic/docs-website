Utilisez les rapports SLA pour afficher des mesures récapitulatives des performances dans le temps ou sur plusieurs moniteurs. Comparez vos performances actuelles aux mesures historiques avec des rapports quotidiens, hebdomadaires et mensuels.

## Voir les monitoring SLA rapports Synthétique [#viewing]

Pour afficher le rapport SLA de votre compte : accédez à <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; SLA Report](https://one.newrelic.com/synthetics-nerdlets/global-sla)**</DNT>. Choisissez parmi les rapports agrégés par jour, semaine ou mois en sélectionnant <DNT>**Daily**</DNT>, <DNT>**Weekly**</DNT> ou <DNT>**Monthly**</DNT> selon le cas.

Vous pouvez également afficher les rapports SLA pour un moniteur individuel : accédez à <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; SLA](https://one.newrelic.com/synthetics-nerdlets/global-sla)**</DNT>.

## Comprendre les métriques du rapport SLA [#understanding]

Utilisez les rapports SLA pour afficher les mesures de performances agrégées pour un seul moniteur ou pour tous vos moniteurs à partir de la page <DNT>**SLA Reports**</DNT> de l&apos;ensemble du compte. Les rapports SLA incluent les mesures suivantes :

* <DNT>**Duration**</DNT>:La durée moyenne de tous les résultats du moniteur.

* <DNT>**Uptime**</DNT>:Le pourcentage de tous les résultats du moniteur qui se sont terminés avec succès. Par exemple, le moniteur A peut vérifier 50 fois par jour et le moniteur B peut vérifier 150 fois par jour. Si le moniteur A a 29 succès sur 50 et le moniteur B a 148 succès sur 150, le <DNT>**Uptime**</DNT> serait 88,5 :

  (29+148)/(50+150)=88,5

  Pour les rapports SLA individuels, le score de temps de disponibilité inclut uniquement le moniteur sélectionné.

* <DNT>**Apdex**</DNT>:L&apos;[Apdex](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) moyen sur tous les moniteurs. les moniteurs ont un [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) par défaut de 7 secondes qui peut être modifié lors de la création ou de la mise à jour de votre moniteur Synthetics . [L&apos;Apdex F](/docs/apm/new-relic-apm/getting-started/glossary#apdex_f), qui définit un résultat frustrant, est toujours quatre fois l&apos;Apdex T. Pour plus d&apos;informations sur l&apos;Apdex, voir [Apdex : Mesure de la satisfaction des utilisateurs](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction). Pour afficher les scores Apdex de votre moniteur Synthétique en utilisant une valeur Apdex T différente, veuillez utiliser la [fonction apdex NRQL ](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/#func-apdex)et l&apos;[événement SyntheticCheck](/attribute-dictionary/?event=SyntheticCheck).

  Pour les rapports SLA individuels, le score Apdex inclut uniquement le moniteur sélectionné.

* <DNT>**% Satisfied**</DNT>: Le pourcentage de résultats du monitoring qui sont terminés dans un [délai « satisfaisant »](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction#bullet-satisfied). Un temps satisfaisant est défini comme un résultat du monitoring qui se termine en [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t) ou moins.

* <DNT>**% Toleration**</DNT>: Le pourcentage de résultats du monitoring qui sont terminés dans un [délai « tolérable »](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction#bullet-tolerating). Un temps tolérable est supérieur à [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t), mais inférieur à [Apdex F](/docs/apm/new-relic-apm/getting-started/glossary#apdex_f) (quatre fois [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t)).

* <DNT>**% Frustrated**</DNT>:Le pourcentage de résultats du monitoring qui se terminent dans un [délai « frustrant »](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction#bullet-tolerating). Un temps frustrant est supérieur à [Apdex F](/docs/apm/new-relic-apm/getting-started/glossary#apdex_f) (quatre fois [Apdex T](/docs/apm/new-relic-apm/getting-started/glossary#apdex_t)).

Le rapport SLA à l&apos;échelle du compte inclut tous les types de moniteurs (ping, navigateur simple, navigateur scripté et test API).

## Générer des valeurs SLA [#generating\_sla\_in\_insights][#generating_sla_in_insights]

Les valeurs du rapport SLA sont générées à partir d&apos;une requête NRQL sur les données monitoring Synthétique disponibles. Vous pouvez facilement recréer ces valeurs et modifier la requête pour répondre à vos besoins.

Cette requête renvoie la durée moyenne, l&apos;apdex et le temps de disponibilité. Remplacez vos valeurs par les variables mises en évidence et décrites ci-dessous.

```sql
SELECT average(duration), apdex(duration, t:), percentage(count(*), WHERE result='SUCCESS') FROM SyntheticCheck FACET timeperiod
```

[La requête NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) par défaut consiste à interroger la dernière heure de données. Afin d&apos;élargir la portée de vos données, vous devrez inclure une [clause`SINCE` ](/docs/insights/new-relic-insights/using-new-relic-query-language/nrql-reference#sel-since)à la fin de votre requête.

<DNT>
  **Example #1: Daily report for the last week**
</DNT>

Pour générer un rapport quotidien pour la semaine dernière, vous devez ajouter `SINCE 1 week ago`:

```sql
SELECT average(duration), apdex(duration, t:7), percentage(count(*), WHERE result='SUCCESS') 
FROM SyntheticCheck FACET dateOf(timestamp) SINCE 1 week ago
```

<DNT>
  **Example #2: Report for a particular monitor**
</DNT>

Pour limiter les résultats à un moniteur particulier, vous pouvez modifier la requête ci-dessous pour inclure un nom de moniteur spécifique :

```sql
SELECT average(duration), apdex(duration, t:7), percentage(count(*), WHERE result='SUCCESS') 
FROM SyntheticCheck FACET dateOf(timestamp) WHERE monitorName = 'mymonitor1'
```

<DNT>
  **Example #3: Report for multiple monitors**
</DNT>

Pour étendre les résultats à un ensemble de moniteurs :

```sql
SELECT average(duration), apdex(duration, t:7), percentage(count(*), WHERE result='SUCCESS') 
FROM SyntheticCheck FACET dateOf(timestamp) WHERE monitorName IN ('mymonitor1', 'mymonitor2', 'mymonitor3')
```
