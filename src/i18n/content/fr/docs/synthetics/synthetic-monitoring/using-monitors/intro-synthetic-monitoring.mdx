Vous pouvez considérer notre moniteur Synthétique comme des mannequins de crash test pour vos sites Web, applications et points de terminaison d&apos;API. Lorsque vous lancez un moniteur synthétique, une vérification d&apos;API ou une instance de navigateur virtuel exécute votre workflow le plus important, testant votre point de terminaison critique et signalant les résultats à New Relic. S&apos;il y a une erreur, une défaillance ou une anomalie affectant vos applications destinées aux clients, votre moniteur Synthétique le détectera et vous alertera, le tout avant que quiconque n&apos;ait le temps de le remarquer.

Nos capacités monitoring Synthétique fonctionnent sur vos applications internes et également derrière pare-feu. Configurez un site privé, puis envoyez des applications conteneurisées pour gérer les jobs (c&apos;est-à-dire pour déclencher le type de monitorer Synthétique que vous souhaitez déclencher) dans un contexte sécurisé.

## Moniteurs synthétique pour monitoring proactif [#proactive]

Nos moniteurs Synthétique sont plus qu&apos;un simple moniteur ping. Synthetics peuvent effectuer des vérifications scriptables sur vos applications orientées clients ou internes, jusqu&apos;aux en-têtes HTTP.

La configuration du moniteur Synthétique avec New Relic vous permet de :

* Développez votre monitoring de navigateurs avec de véritables [navigateurs scriptés](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) basés sur Sélénium, qui testent les procédures de connexion, les recherches et d&apos;autres transactions commerciales critiques.
* Exécutez un [test API](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-api-tests) pour vérifier les certificats ou vous assurer que tout type de requests HTTP est terminé.
* Bénéficiez d&apos;une couverture étendue des cas d&apos;utilisation avec [des modules Node supplémentaires](/docs/synthetics/synthetic-monitoring/scripting-monitors/import-nodejs-modules) pour votre moniteur d&apos;API scripté.
* Diagnostiquez facilement si un problème provient du réseau ou de l&apos;emplacement AWS, d&apos;une ressource tierce lente ou de la santé de vos backend services ou infrastructure.

Vous pouvez consulter des exemples de certains moniteurs Synthétique provenant de la communauté dans nos [démarrages rapides Synthetics](https://newrelic.github.io/quickstarts-synthetics-library/#/). Ces exemples sont open source et ne sont pas pris en charge par New Relic, mais contiennent des scripts et des extraits précieux qui peuvent vous aider à couvrir les cas d&apos;utilisation courants dans les API scriptées et le moniteur de navigateur scripté.

## Types de moniteurs synthétiques

Selon le type de moniteur Synthétique que vous choisissez, vous pouvez :

* [Ajouter et modifier le moniteur](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors).
* Utilisez [une API](/docs/synthetics/synthetic-monitoring/administration/synthetics-api) pour gérer votre moniteur.
* Configurer un moniteur à partir [d&apos;emplacements spécifiques ou pour des serveurs privés](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations).

Vous pouvez également utiliser la fonctionnalité [de non-signalement d&apos;hôte](/docs/infrastructure/new-relic-infrastructure/configuration/create-infrastructure-host-not-reporting-condition) dans [monitoring d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure). Cela vous permet de profiter d&apos;options monitoring améliorées et d&apos;être averti lorsque nous cessons de recevoir des données de vos hôtes.

## Tests avancés avec des navigateurs scriptés [#advanced]

Le monitoring synthétique vous permet de monitorer de manière proactive votre site Web ou vos points de terminaison d&apos;API pour garantir que votre contenu est non seulement disponible, mais entièrement fonctionnel. [Les navigateurs scriptés](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-synthetic-scripts) envoient une véritable instance Chrome ou de Firefox alimentée par Sélénium vers votre site à partir d&apos;emplacements situés dans le monde entier afin que vous soyez sûr que votre contenu est toujours disponible, partout.

Les navigateurs scriptés étendent vos capacités de test, vous permettant ainsi de tester des flux utilisateurs inhabituels ou de tester des procédures complexes. Par exemple, vous pouvez garantir que vos utilisateurs peuvent s&apos;inscrire à votre newsletter, ajouter un article à leur panier ou rechercher et trouver un élément de contenu critique avec un langage simple de type JavaScript. Testez votre backend avec API Monitorer, qui vous permet d&apos;exécuter des tests scriptés sur n&apos;importe quel point de terminaison d&apos;API.

## Quelle est la prochaine étape ? [#enhanced-monitoring]

Après avoir configuré un moniteur Synthétique, nous vous recommandons les étapes suivantes :

* Vérifiez comment vos données synthétiques transforment vos résultats agrégés en [métriques](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-overview-dashboard), vous permettant de voir des tendances et d&apos;identifier les causes de mauvaises performances.
* Découvrez comment [chaque résultat de monitoring](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding) est décomposé.
* Vous souhaitez compléter vos données synthétiques avec des données d&apos;utilisateurs réels ? Consultez notre tutoriel [Améliorez les performances de votre site Web](/docs/journey-performance/improve-website-performance) .
* Créez [des alertes qui vous avertissent](/docs/synthetics/new-relic-synthetics/using-monitors/alerting-synthetics) si votre site Web ou vos points de terminaison d&apos;API sont inaccessibles. Vous pouvez même étendre votre couverture géographique ou monitorer des sites Web internes en créant [des sites privés](/docs/synthetics/new-relic-synthetics/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations).
* Vous pouvez également [interroger les résultats de votre moniteur](/docs/using-new-relic/data/understand-data/query-new-relic-data) pour un examen plus approfondi. New Relic conserve les résultats du moniteur pendant treize mois, vous permettant ainsi de comparer l&apos;utilisation d&apos;une année sur l&apos;autre.
