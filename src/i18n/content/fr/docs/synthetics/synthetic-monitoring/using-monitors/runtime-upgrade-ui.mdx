New Relic [mettra fin au support des anciens environnements d’exécution](/eol/2025/11/eol-chrome-140-node-22-synthetics/) en 2026. Pour simplifier la migration des moniteurs synthétiques vers les derniers environnements d’exécution pris en charge, nous avons automatiquement mis à niveau chaque moniteur qui ne nécessitait pas d’action de votre part. Utilisez l&apos;outil de mise à niveau de l&apos;environnement d&apos;exécution pour afficher les monitorer restants qui nécessitent encore votre attention.

## Prérequis [#prerequisites]

Avant de commencer à mettre à niveau les moniteurs, assurez-vous de respecter les prérequis suivants :

* **Autorisation de mise à niveau d&apos;exécution en masse :** requise pour appliquer des mises à jour en masse via l&apos;interface utilisateur. [Les administrateurs de produits](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles) ont cette autorisation par défaut. Vous pouvez également l&apos;ajouter aux [rôles personnalisés](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics).
* **Droits d&apos;IA (<DNT>Fix with AI</DNT> uniquement) :** nécessite le droit `nrai_ccu` ou `nrai_discount_usage`. Les comptes FedRAMP n&apos;ont pas accès à cette expérience. Contactez votre équipe de compte New Relic pour activer les droits.

## Mettez à niveau vos monitorer restants [#upgrade-options]

Pour chaque moniteur qui nécessite une attention, vous pouvez choisir l&apos;une des deux voies :

<CollapserGroup>
  <Collapser id="fix-manually" title="Corriger manuellement">
    Utilisez l&apos;option <DNT>**Fix manually**</DNT> pour écrire et modifier vos scripts de monitoring plutôt que de vous fier à la génération par l&apos;IA. Pour mettre à jour un moniteur manuellement :

    1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.

    2. Localisez le moniteur et cliquez sur <DNT>**Fix manually**</DNT>.

    3. Dans la fenêtre <DNT>**Fix script manually**</DNT>, mettez à jour votre code de script pour le nouvel environnement d&apos;exécution.

    4. Cliquez sur <DNT>**Close and start validation**</DNT>. Le moniteur se déplace vers le tableau <DNT>**Monitors we&apos;re validating**</DNT> pendant que le système valide le script mis à jour.

       <Callout title="Note">
         Si la validation échoue, cliquez sur **⋯** à côté du moniteur, sélectionnez <DNT>**Fix manually**</DNT> dans le menu, mettez à jour le script, puis cliquez de nouveau sur <DNT>**Close and start validation**</DNT>.
       </Callout>

    5. Une fois que la validation est réussie et que le statut indique <DNT>**Fixed**</DNT>, cliquez sur l&apos;onglet <DNT>**Ready to update**</DNT>.

    6. Cochez la case à côté du moniteur validé, et cliquez sur <DNT>**Upgrade selected**</DNT>.

    Une fois terminé, une notification de confirmation apparaît et le monitor passe à l&apos;onglet <DNT>**Updated**</DNT> où vous pouvez vérifier la réussite de la mise à niveau.
  </Collapser>

  <Collapser id="fix-with-ai" title="Corriger avec l'IA">
    Utilisez l&apos;option <DNT>**Fix with AI**</DNT> pour laisser <DNT>New Relic AI (NRAI)</DNT> générer un script mis à jour pour vous. La génération de script assistée par l&apos;IA vise principalement à corriger les erreurs `timeout` survenues lors des mises à niveau du runtime. Pour les autres types d’erreurs, le script lui-même nécessite probablement des modifications que vous devriez apporter [manuellement](#fix-manually).

    Pour mettre à jour un monitorer à l&apos;aide de l&apos;IA :

    1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.
    2. Localisez le moniteur et cliquez sur <DNT>**Fix with AI**</DNT>.
    3. Examinez le script suggéré par l&apos;IA. Pour afficher les modifications spécifiques apportées à votre code d’origine, cliquez sur <DNT>**Show comparison**</DNT> pour ouvrir une vue DIFF côte à côte.
    4. Cliquez sur <DNT>**Continue to validation**</DNT>. Le système place temporairement le moniteur dans un état de validation pour tester le nouveau script et s&apos;assurer qu&apos;il s&apos;exécute sans erreurs.
    5. Une fois le script validé, cliquez sur l&apos;onglet <DNT>**Ready to update**</DNT>.
    6. Cochez la case à côté du moniteur validé, et cliquez sur <DNT>**Upgrade selected**</DNT>.

    Une fois terminé, une notification de confirmation apparaît et le monitor passe à l&apos;onglet <DNT>**Updated**</DNT> où vous pouvez vérifier la réussite de la mise à niveau.
  </Collapser>
</CollapserGroup>

Vous pouvez exécuter plusieurs mises à niveau en même temps, ce qui vous permet de travailler sur plusieurs monitorer en parallèle sans attendre que le système termine de valider chaque script.
