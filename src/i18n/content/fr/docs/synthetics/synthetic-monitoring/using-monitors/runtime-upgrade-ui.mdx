New Relic [mettra fin à la prise en charge des anciens runtimes](/eol/2026/05/eol-05-18-26-node-16-chrome-134-runtimes/) en 2026. Pour simplifier la migration des moniteurs synthétiques vers les derniers environnements d’exécution pris en charge, nous avons automatiquement mis à niveau chaque moniteur qui ne nécessitait pas d’action de votre part. Utilisez l&apos;outil de mise à niveau de l&apos;environnement d&apos;exécution pour afficher les monitorer restants qui nécessitent encore votre attention.

## Mettez à niveau vos monitorer restants [#upgrade-options]

Pour chaque moniteur qui nécessite une attention, vous pouvez choisir l&apos;une des deux voies.

<Tabs>
  <TabsBar>
    <TabsBarItem id="fix-manually">
      Corriger manuellement
    </TabsBarItem>

    <TabsBarItem id="fix-with-ai">
      Corriger avec l&apos;IA
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="fix-manually">
      Pour mettre à jour un moniteur en écrivant ou en modifiant manuellement vos scripts de moniteur, suivez les étapes.

      1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>.

      2. Localisez le moniteur et cliquez sur <DNT>**Fix manually**</DNT>.

      3. Dans la fenêtre <DNT>**Fix script manually**</DNT>, mettez à jour votre code de script pour le nouvel environnement d&apos;exécution.

      4. Cliquez sur <DNT>**Close and start validation**</DNT>. Le moniteur se déplace vers le tableau <DNT>**Monitors we&apos;re validating**</DNT> pendant que le système valide le script mis à jour.

         * Si la validation échoue, cliquez sur **⋯** à côté du moniteur, sélectionnez <DNT>**Fix manually**</DNT> dans le menu, mettez à jour le script, puis cliquez de nouveau sur <DNT>**Close and start validation**</DNT>.

      5. Une fois que la validation est réussie et que le statut indique <DNT>**Fixed**</DNT>, cliquez sur l&apos;onglet <DNT>**Ready to update**</DNT>.

      6. Cochez la case à côté du moniteur validé, et cliquez sur <DNT>**Upgrade selected**</DNT>.

      Une fois terminé, une notification de confirmation apparaît et le monitor passe à l&apos;onglet <DNT>**Updated**</DNT> où vous pouvez vérifier la réussite de la mise à niveau.
    </TabsPageItem>

    <TabsPageItem id="fix-with-ai">
      <Callout title="Note">
        * Cette fonctionnalité nécessite Advanced Compute et n&apos;est pas disponible pour les comptes <DNT>**FedRAMP**</DNT>. Contactez votre équipe de compte New Relic pour obtenir de l&apos;assistance.
        * Testez toujours les scripts générés par l’IA avant de déployer les moniteurs en production.
      </Callout>

      Utilisez cette option pour que <DNT>**New Relic AI (NRAI)**</DNT> génère un script mis à jour pour vous. La génération de script assistée par l’IA vise principalement à corriger `timeout errors` survenus lors des mises à niveau du runtime. Pour les moniteurs avec `script errors`, le script nécessite potentiellement d&apos;autres modifications. Vous pouvez les créer manuellement ou utiliser <DNT>[SRE Agent](/docs/agentic-ai/sre-agent/overview/)</DNT> pour déboguer et obtenir des informations détaillées sur les mises à jour potentielles liées au script.

Vous pouvez exécuter plusieurs mises à niveau en même temps, ce qui vous permet de travailler sur plusieurs monitorer en parallèle sans attendre que le système termine de valider chaque script.
