## Problème

L&apos;un de vos minion privés échoue aux contrôles et signale l&apos;une de ces erreurs :

```
HMAC MISMATCH ERROR: HMAC mismatch between saved HMAC and calculated HMAC at job run time
```

```
HMAC MISMATCH ERROR: HMAC expected but missing from Job metadata
```

Ces erreurs indiquent une erreur lors de [l&apos;exécution du script vérifié](/docs/synthetics/new-relic-synthetics/private-locations/verified-script-execution-private-locations).

## Solution

Vous devrez saisir le mot de passe correct pour l’emplacement à partir duquel cette vérification échoue.

Pour saisir le mot de passe :

1. Dans <DNT>**[one.newrelic.com &gt; Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; sélectionnez le moniteur défaillant.
2. Allez à <DNT>**Settings &gt; Script**</DNT>.
3. Dans les champs de mot de passe d&apos;emplacement, soumettez le mot de passe correct.

<Callout variant="tip">
  New Relic ne stocke pas les mots de passe du site privé. Si vous ne connaissez pas le mot de passe, trouvez la personne de votre équipe qui a configuré le site privé. S&apos;il n&apos;est pas possible de récupérer le mot de passe, votre seule option est de créer un nouveau site privé.
</Callout>
