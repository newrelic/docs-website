Vous avez une application qui doit rester en ligne ? monitoring synthétique vous aide à détecter et à résoudre de manière proactive les problèmes avant qu&apos;ils n&apos;affectent vos clients, le tout sans installation.

## Vérifier les performances de chargement des pages [#simple]

Notre outil de monitoring de navigateur simple vous permet de vérifier la disponibilité et les performances d&apos;une seule page, y compris le chargement d&apos;une page complète.

<Steps>
  <Step>
    ### Connectez-vous à votre compte New Relic [#create]

    Une fois connecté à votre compte New Relic, accédez à <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create a monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT>. Choisissez la tuile <DNT>**Page load performance**</DNT> .
  </Step>

  <Step>
    ### Coller une URL [#url]

    <SideBySide>
      <Side>
        Saisissez l&apos;URL d&apos;une page que vous souhaitez tester et déposez-la dans le champ <DNT>**URL**</DNT> . Pour de meilleurs résultats, nous recommandons :

        * Sélectionnez au moins trois emplacements à partir desquels votre moniteur se déroule. Cela vous aide à éviter les faux positifs sur vos contrôles.
        * Utilisation de plusieurs types de navigateurs, en utilisant une combinaison de Chrome et Firefox.
        * Réglage de la période entre les vérifications avec la liste déroulante <DNT>**Period**</DNT> . La fréquence à laquelle vous effectuez les tests dépend entièrement de vous.
      </Side>

## Imiter un workflow utilisateur simple avec monitoring sans code [#step]

Vous pouvez créer un moniteur d&apos;étapes sans code qui s&apos;exécute via un workflow utilisateur commun. C&apos;est un bon choix si vous souhaitez imiter des comportements simples dans votre application, comme naviguer sur différentes pages, tester des éléments de page ou saisir et soumettre du texte dans des champs de texte.

<Steps>
  <Step>
    ### Connectez-vous à votre compte New Relic [#steplogin]

    Une fois connecté à votre compte New Relic, accédez à <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create a monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT>. Choisissez la tuile <DNT>**User step execution**</DNT> .
  </Step>

  <Step>
    ### Créez votre moniteur de pas [#step]

    <SideBySide>
      <Side>
        Donnez un nom à votre moniteur, choisissez la fréquence à laquelle votre moniteur exécute son workflow, sélectionnez parmi les types de navigateur et d&apos;appareils émulés disponibles, puis sélectionnez l&apos;emplacement à partir duquel votre moniteur est déployé.

        Allez à <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create a monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT>, puis choisissez <DNT>**User step execution**</DNT>.
      </Side>

      <Side>
        <Video type="wistia" id="dum72g5b7z" />
      </Side>
    </SideBySide>
  </Step>

  <Step>
    ### Définissez vos étapes [#define]

    <SideBySide>
      <Side>
        Mélangez et associez à partir d&apos;une sélection de 12 étapes prédéfinies. Votre outil de monitoring peut être simple, comme cliquer sur des boutons pour parcourir différentes pages, ou vous pouvez tester un parcours qui ressemble à :

        1. Un client accède à votre application Web
        2. Ils tapent leur email dans une page modale
        3. Ils sélectionnent dans une liste déroulante et naviguent vers une autre page
        4. Ils sélectionnent quelque chose sur cette page, comme un article à acheter
        5. Ils remplissent des formulaires avec des informations d&apos;identification sécurisées, achètent cet article, puis soumettent ces formulaires

        Quel que soit votre choix de construction, n&apos;oubliez pas de cliquer sur <DNT>**Validate**</DNT> avant d&apos;enregistrer votre moniteur. La validation vérifie que les étapes que vous avez enchaînées s&apos;exécutent correctement.

## Comportement du site de test avec monitoring scripté [#script]

moniteur de navigateur scripté est un type de moniteur flexible qui vous permet de tester le comportement d&apos;un site Web. Le script vous permet de déployer un moniteur qui répond à vos applications Web comme le ferait un client, en choisissant différents chemins sur votre site en fonction des conditions que vous avez définies dans le script.

Vous pouvez choisir de créer un moniteur de navigateur scripté au lieu d&apos;un moniteur d&apos;étapes si :

* Vous souhaitez définir des valeurs de délai d&apos;expiration personnalisées
* Vous souhaitez un script pour effectuer des actions non proposées dans notre générateur d&apos;étapes
* Vous souhaitez définir des conditions qui modifient le chemin emprunté par votre moniteur si ces conditions sont remplies

<Tabs>
  <TabsBar>
    <TabsBarItem id="URL">
      Accéder à une URL
    </TabsBarItem>

    <TabsBarItem id="element">
      Interagir avec un élément de page
    </TabsBarItem>

    <TabsBarItem id="wait">
      Attendre un élément de page
    </TabsBarItem>

    <TabsBarItem id="login">
      se connecter à un site Web
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="URL">
      ```js
      // Visit `http://telco.nrdemo.com/`
      $webDriver.get("http://telco.nrdemo.com/");
      ```

      `$webdriver.get()` envoie votre moniteur vers le domaine que vous souhaitez monitorer.
    </TabsPageItem>

    <TabsPageItem id="element">
      ```js
      $webDriver.get("http://telco.nrdemo.com/").then(function(){
        // Find a link with display text that matches `About`
        return $webDriver.findElement($selenium.By.linkText("About")).click();
        // Click the link that matches `About`
      }).then(function(){
          return $webDriver.findElement($selenium.By.partialLinkText("Home")).click();
      });
      ```

      Une fois que votre moniteur accède à une URL, `$webDriver.findElement()` trouve l&apos;élément défini, puis implémente une action, dans ce cas `click()`. Une fois l’action terminée, le moniteur revient à la page d’accueil en utilisant la même logique.
    </TabsPageItem>

    <TabsPageItem id="wait">
      ```js
      async function waitForAndFindElement(locator, timeout){
        const element = await $webDriver.wait($selenium.until.elementLocated(locator), timeout, 'Timed-out waiting for element to be located using: '+locator);
        await $webDriver.wait($selenium.until.elementIsVisible(element), timeout, 'Timed-out waiting for element to be visible using ${element}');
        return await $webDriver.findElement(locator);
      }

      await $webDriver.get("http://telco.nrdemo.com/")

      await $webDriver.findElement($selenium.By.id("supportDropDown")).click();

      // Wait up to 20000 seconds for the FAQ page to appear
      let aboutPage = await waitForAndFindElement($selenium.By.id("supportFAQLink"), 20000)
      await aboutPage.click();
      ```

      Le moniteur de navigateur scripté peut attendre un certain temps avant de signaler une erreur. Étant donné que certaines images ou éléments dynamiques prennent du temps à générer, vous pouvez donner des paramètres à `$webDriver.waitForandFindElement()`. Dans ce snippet, votre moniteur trouve `"supportFAQLink"`, puis attend `20000` secondes avant de signaler une erreur.
    </TabsPageItem>

    <TabsPageItem id="login">
      ```js
      $webDriver.get("http://example.com/login.jsp").then(function(){
        return $webDriver.findElement($selenium.By.name("username")).sendKeys($secure.SECURE_USERNAME);
      }).then(function(){
        // Find the password field by specifying its name, then submits a secured password.
        return $webDriver.findElement($selenium.By.name("password")).sendKeys($secure.SECURE_PASSWORD);
      }).then(function(){
        // Find and click the login button.
        return $webDriver.findElement($selenium.By.xpath("//inp[@value='Login']")).click();
      });
      ```

      Vous pouvez également tester des éléments qui nécessitent des informations de connexion. Nous vous recommandons de créer des informations d&apos;identification uniques pour votre script.
    </TabsPageItem>
  </TabsPages>
</Tabs>

Accédez à <DNT>**[one.newrelic.com &gt; Synthetic monitoring &gt; Create a monitor](https://one.newrelic.com/synthetics-nerdlets/monitor-create)**</DNT>, puis choisissez <DNT>**User flow/functionality**</DNT> pour le navigateur scripté.
