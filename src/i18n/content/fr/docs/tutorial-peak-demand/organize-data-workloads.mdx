Pour démarrer en position forte le jour du match, nous vous recommandons de créer une charge de travail dès la phase de planification. Au-delà de la santé des applications et de l’allocation des ressources, la chorégraphie d’une journée de jeu dépend de l’alignement des données et d’une communication solide. La charge de travail vous permet de personnaliser la manière dont vous percevez les données signalées à New Relic. Vous pouvez créer une charge de travail autour d&apos;un événement, d&apos;une équipe ou de responsabilités, selon vos besoins particuliers. Une fois créée, la charge de travail vous permet d&apos;afficher les données agrégées de santé et d&apos;activité des services front-end aux services backend sur l&apos;ensemble de votre système.

## Objectifs [#objective]

Ce tutoriel donne une introduction à l&apos;utilisation de la charge de travail pour une journée de performance optimale. Vous serez:

* Créez votre première workload
* Personnalisez votre workload pour votre événement de jour de match

## Créer une workload pour le jour du match [#create]

Pour commencer avec la charge de travail, accédez à <DNT>**[one.newrelic.com &gt; Workloads](https://one.newrelic.com/workloads)**</DNT>, puis cliquez sur <DNT>**Create a workload**</DNT>.

<img title="Add sources to workload" alt="Add your sources to the workload" src="/images/apm_screenshot-crop_add-sources-to-workload.webp" />

<Steps>
  <Step>
    ### Créez votre première workload

    La page <DNT>**Create a workload**</DNT> répertorie les sources de données de tous les comptes auxquels vous avez accès. S&apos;il existe une capacité, un service, un composant ou une ressource qui dépend de New Relic, vous pouvez l&apos;ajouter à une workload.

    Pour créer une workload:

    1. Sélectionnez le compte propriétaire de la workload.
    2. Nommez votre workload, quelque chose comme `Peak Demand Data` ou toute autre chose significative pour votre équipe.
    3. requête pour les sources que vous souhaitez représenter dans votre workload.
  </Step>

  <Step>
    ### Choisissez des sources de données adaptées à votre workload

    Pour tirer le meilleur parti de la charge de travail, nous vous recommandons d&apos;ajouter :

    * Services APM, hôtes, niveau de service, moniteur Synthétique, équilibreurs de charge, pod Kubernetes ou encore autre workload.
    * `log` à votre requête de charge de travail pour un accès facile à la page de l&apos;interface utilisateur des logs.
    * Dashboards pertinents pour une workload, tel qu&apos;un dashboard qui affiche uniquement vos KPI ou un dashboard des transactions critiques pour l&apos;entreprise.
  </Step>

  <Step>
    ### Ajoutez une source de données à votre workload

    Une fois votre sélection affinée, vous disposez de deux options pour ajouter une source de données à une workload. Les deux font des choses légèrement différentes.

    * <DNT>**Add**</DNT> ajoute la source dans un état statique. Par exemple, si vous avez ajouté un dashboard à 10 h, mais que vous avez ajouté davantage de widgets à ce dashboard à 11 h, ces widgets supplémentaires n&apos;apparaîtront pas dans la workload.
    * <DNT>**Add this query**</DNT> ajoute l&apos;entité de manière dynamique. En utilisant le même exemple dashboard , les modifications de 11 h 00 (et toutes les modifications futures) apparaîtraient automatiquement dans la workload.
  </Step>

  <Step>
    ### Ajoutez un dashboard à votre workload

    Si vous avez des dashboards personnalisés et que vous savez déjà quelles données sont pertinentes pour votre équipe, vous pouvez les lier <InlinePopover type="dashboards" />à partir de votre workload. Vous pouvez également définir des filtres sur les dashboards pour les adapter à un contexte spécifique workload . Lorsque vous sélectionnez ce dashboard à partir de la workload, il s’ouvre avec le filtre déjà appliqué.

    <img title="new-relic-workloads-add-dashboards.png" alt="New Relic workloads - add dashboards" src="/images/workloads_screenshot-full_workloads-dashboard-entities.webp" />

    <figcaption>
      <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Apps &gt; Workloads**</DNT>:Vous pouvez ajouter des dashboards à une workload.
    </figcaption>
  </Step>
</Steps>

## Utilisation de la charge de travail le jour du match [#gameday]

Votre charge de travail est le point zéro de votre journée de match. Le jour du match, tous les membres de votre équipe, quel que soit leur rôle, doivent commencer avec la même workload et s&apos;y référer lors de la communication sur l&apos;incident. En alignant votre équipe autour des mêmes ensembles de données, vous évitez les problèmes de communication et leur permettez de résoudre les pannes plus rapidement.
