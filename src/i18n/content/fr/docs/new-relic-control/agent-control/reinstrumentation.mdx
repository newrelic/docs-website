<Callout variant="important">
  Agent Control et New Relic Control sont désormais **disponibles en version générale** pour Kubernetes ! La prise en charge des hôtes Linux et Windows fait également partie du programme **public preview**, conformément à nos [politiques de pré-lancement](/docs/licenses/license-information/referenced-policies/new-relic-pre-release-policy).
</Callout>

## Migration Kubernetes

Si votre cluster Kubernetes est déjà instrumenté avec New Relic, vous pouvez passer vos agents sous la gestion d&apos;Agent Control pour utiliser ses fonctionnalités de gestion. Le processus de migration supprime les installations d&apos;agents existantes et les remplace par des installations gérées par Agent Control, configurées de manière centralisée via Fleet Control.

<Callout variant="important" title="La configuration à distance l'emporte toujours">
  Agent Control utilise Fleet Control comme source faisant autorité pour la configuration de l&apos;agent. Tous les paramètres de l&apos;&apos;agent — versions, valeurs de chart et limites de ressources — doivent être définis dans la configuration à distance de Fleet Control, et non dans l&apos;&apos;installation locale `values.yaml`. Si les deux existent, la configuration distante a la priorité et écrasera toutes les valeurs locales.
</Callout>

<Callout variant="tip" title="Meilleures pratiques">
  Pour garantir la continuité des données, utilisez le même nom de cluster et le même compte New Relic lors de l&apos;installation d&apos;Agent Control. Vous pouvez conserver tous les agents ou composants qui ne sont pas encore pris en charge par Agent Control sur le cluster jusqu&apos;à ce que la prise en charge soit disponible.
</Callout>

### Étape 1. Récupérer votre configuration existante

Tout d’abord, identifiez lesquels de vos agents existants peuvent être gérés par Agent Control.

Agent Control peut déployer et gérer les agents suivants à partir du chart Helm `nri-bundle`, mappés aux types d&apos;agents Agent Control :

* `newrelic-infrastructure`, `nri-kube-events`, `kube-state-metrics` et `nri-kube-events`: Gérés comme le type d&apos;agent New Relic Infrastructure
* `newrelic-prometheus-configurator`: Géré en tant que type d&apos;agent Prometheus New Relic
* `newrelic-logging`: Géré en tant que type d&apos;agent Fluent Bit
* `nr-k8s-otel-collector`: Géré en tant que type d&apos;agent New Relic OpenTelemetry Collector

Si vous avez installé des agents à l’aide de Helm, récupérez votre configuration d’installation à partir du cluster en exécutant la commande suivante :

```sh
# Find releases in a namespace (adjust as needed)
helm list --namespace <namespace-name>

# Get the values for a specific release
$ helm get values <release-name> --namespace <namespace-name>
```

Si vous disposez du fichier de configuration d&apos;origine sauvegardé lors de l&apos;installation, vous pouvez l&apos;utiliser directement. Si vous avez instrumenté le cluster via des manifestes Kubernetes (kubectl ou Kustomize), notez les options définies pour chaque agent : vous les recréerez sous forme de configurations à distance Fleet Control.

### Étape 2. Désinstallez vos anciens agents

Avant d&apos;installer Agent Control, supprimez les anciens agents de votre cluster.

Pour une installation Helm, utilisez la commande `helm uninstall` :

```shell
$ helm uninstall <release-name> -n <namespace>
```

Par exemple, pour désinstaller le `nri-bundle`:

```shell
$ helm uninstall nri-bundle -n newrelic
```

### Étape 3. Installer Agent Control

Lancez l&apos;[installation guidée d&apos;Agent Control](/docs/new-relic-control/agent-control/setup/) et téléchargez le `values.yaml` généré. Ce fichier contient uniquement la configuration du superviseur Agent Control (identifiants, ID de flotte et types d&apos;agents à gérer) — **n&apos;ajoutez pas de valeurs de chart spécifiques à l&apos;agent ici**. Ceux-ci appartiennent à la configuration à distance de Fleet Control.

Pour une installation Helm, vous exécuterez généralement :

```shell
$ helm upgrade --install agent-control-bootstrap -n newrelic newrelic/agent-control-bootstrap --create-namespace --values my_values.yaml
```

### Étape 4. Recréez la configuration de votre agent dans Fleet Control

Une fois Agent Control installé et connecté à Fleet Control, déployez vos agents en créant ou en mettant à jour la configuration à distance de la flotte avec vos paramètres précédents. Il s&apos;agit de l&apos;emplacement de référence pour toute la configuration de l&apos;agent.

#### Exemple : Migration d&apos;une installation nri-bundle

Ce qui suit montre comment traduire les valeurs Helm `nri-bundle` existantes en une configuration à distance Fleet Control.

**configuration `nri-bundle` existante :**

```yaml
global:
  cluster: test-migration
  licenseKey: ***
kube-state-metrics:
  enabled: false
newrelic-prometheus-agent:
  enabled: true
newrelic-infrastructure:
  enabled: true
  kubelet:
    tolerations:
      - operator: "Exists"
        effect: "NoSchedule"
      - operator: "Exists"
        effect: "NoExecute"
      - operator: "Exists"
        key: "MyToleration"
  ksm:
    enabled: false
  common:
    config:
      interval: 29s
newrelic-logging:
  enabled: true
  image:
    tag: "latest"
  resources:
    limits:
      cpu: 200m
    requests:
      cpu: 200m
nri-kube-events:
  enabled: true
  customAttributes:
    test_tag_label: test_tag_value
```

**Configuration à distance Fleet Control équivalente :**

Définissez ceci comme la configuration à distance de votre flotte dans Fleet Control. Agent Control l&apos;appliquera automatiquement.

```yaml
logs:
  chart_version: "1.25.1"
  chart_values:
    newrelic-logging:
      image:
        tag: "latest"
      resources:
        limits:
          cpu: 200m
        requests:
          cpu: 200m
infrastructure:
  chart_version: "5.0.109"
  chart_values:
    newrelic-infrastructure:
      kubelet:
        tolerations:
          - operator: "Exists"
            effect: "NoSchedule"
          - operator: "Exists"
            effect: "NoExecute"
          - operator: "Exists"
            key: "MyToleration"
      ksm:
        enabled: false
      common:
        config:
          interval: 29s
    nri-kube-events:
      customAttributes:
        test_tag_label: test_tag_value
prometheus:
  chart_version: "1.15.4"
```

## Migration Linux

<Callout variant="important">
  Agent Control ne prend pas en charge la migration automatique des agents existants sous Linux. Si l&apos;agent Infrastructure New Relic est déjà installé sur votre hôte Linux, vous devez le désinstaller manuellement avant d&apos;installer Agent Control. Agent Control distribue les agents via des packages conformes à l&apos;OCI par le biais de Fleet Control — il ne remplace ni n&apos;hérite de l&apos;installation existante du package `newrelic-infra`.
</Callout>

Pour migrer d&apos;une installation autonome de l&apos;Agent d&apos;infrastructure sous Linux vers Agent Control :

1. **Notez votre configuration existante.** Avant de désinstaller, sauvegardez une copie de la configuration de votre agent à l&apos;emplacement `/etc/newrelic-infra/newrelic-infra.yml` et de toutes les configurations d&apos;intégration sur l&apos;hôte dans `/etc/newrelic-infra/integrations.d/`. Vous devrez les recréer en tant que configurations à distance Fleet Control après avoir installé Agent Control.

2. **Désinstallez l&apos;agent d&apos;infrastructure existant.** Suivez les étapes de désinstallation pour votre distribution :

   <Tabs>
     <TabsBar>
       <TabsBarItem id="linux-uninstall-debian">Debian, Ubuntu</TabsBarItem> <TabsBarItem id="linux-uninstall-suse">SLES</TabsBarItem> <TabsBarItem id="linux-uninstall-redhat">Amazon Linux, CentOS, RHEL</TabsBarItem>
     </TabsBar>

     <TabsPages>
       <TabsPageItem id="linux-uninstall-debian">
         ```bash
         sudo apt-get remove newrelic-infra -y
         ```
       </TabsPageItem>

       <TabsPageItem id="linux-uninstall-suse">
         ```bash
         sudo zypper -n remove newrelic-infra
         ```
       </TabsPageItem>

       <TabsPageItem id="linux-uninstall-redhat">
         ```bash
         sudo yum remove newrelic-infra -y
         ```
       </TabsPageItem>
     </TabsPages>
   </Tabs>

3. **Installer Agent Control.** Suivez les [étapes d&apos;installation Linux](/docs/new-relic-control/agent-control/setup/) à l&apos;aide de l&apos;installation guidée, en veillant à sélectionner ou créer une flotte de type **&quot;Host - Linux&quot;**.

4. **Recréez la configuration de votre agent dans Fleet Control.** Une fois Agent Control installé et connecté, déployez l&apos;Agent d&apos;infrastructure via Fleet Control et appliquez vos paramètres précédents via une configuration à distance. Consultez [Configuration Linux](/docs/new-relic-control/agent-control/configuration/#linux-configuration) pour obtenir des conseils sur le format de configuration.

Les dashboards personnalisés et les alertes faisant référence aux données de l&apos;agent d&apos;infrastructure continueront de fonctionner tant que vous utilisez le même compte et le même nom d&apos;hôte.

## Migration Windows

<Callout variant="important">
  Agent Control ne prend pas en charge la migration automatique des agents existants sous Windows. Si l&apos;agent Infrastructure New Relic est déjà installé sur votre hôte Windows, vous devez le désinstaller manuellement avant d&apos;installer Agent Control. Les agents APM ne sont pas gérés par Agent Control sous Windows et peuvent rester installés.
</Callout>

Pour migrer d&apos;une installation autonome de l&apos;Agent d&apos;infrastructure sous Windows vers Agent Control :

1. **Désinstallez l&apos;agent d&apos;infrastructure existant.** Ouvrez **Add or Remove Programs** (ou utilisez PowerShell) et désinstallez l&apos;agent d&apos;infrastructure New Relic. Sinon, exécutez son désinstalleur :

   ```powershell
   # Example using the MSI uninstaller
   msiexec /x {PRODUCT_CODE} /quiet
   ```

2. **Notez votre configuration existante.** Avant de désinstaller, sauvegardez une copie de la configuration de votre agent (généralement dans `C:\Program Files\New Relic\newrelic-infra\newrelic-infra.yml`) et de toutes les configurations d&apos;intégration sur l&apos;hôte dans `C:\Program Files\New Relic\newrelic-infra\integrations.d\`. Vous devrez les recréer en tant que configurations à distance Fleet Control après avoir installé Agent Control.

3. **Installer Agent Control.** Suivez les [étapes d&apos;installation Windows](/docs/new-relic-control/agent-control/setup/) à l&apos;aide de l&apos;installation guidée, en veillant à sélectionner ou créer une flotte de type **&quot;Host - Windows&quot;**.

4. **Recréez la configuration de votre agent dans Fleet Control.** Une fois Agent Control installé et connecté, déployez l&apos;Agent d&apos;infrastructure via Fleet Control et appliquez vos paramètres précédents via une configuration à distance. Consultez [Configuration Windows](/docs/new-relic-control/agent-control/configuration/#windows-configuration) pour obtenir des conseils sur le format de configuration.

Les dashboards personnalisés et les alertes faisant référence aux données de l&apos;agent d&apos;infrastructure continueront de fonctionner tant que vous utilisez le même compte et le même nom d&apos;hôte.
