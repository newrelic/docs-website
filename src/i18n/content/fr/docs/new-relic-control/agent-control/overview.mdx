Agent Control est un superviseur d&apos;agent léger qui gère d&apos;autres agents New Relic et New Relic distribution d&apos;agents OpenTelemetry. Il communique avec le contrôle de la flotte pour configurer, mettre à jour et monitorer à distance la santé de l&apos;instrumentation au niveau de infrastructuresur Kubernetes.

## Prérequis

Avant de commencer à déployer Agent Control sur votre cluster Kubernetes, assurez-vous de remplir les conditions préalables suivantes :

* **Helm 3 :** Helm version 3 doit être installé sur votre machine. Pour les instructions d&apos;installation, voir [Installation de Helm](https://helm.sh/docs/intro/install).
* **Clé de licence New Relic :** Vous aurez besoin d&apos;une clé de licence pour signaler la télémétrie à votre compte New Relic.
* **Nom du cluster Kubernetes :** préparez le nom de votre cluster Kubernetes. Vous y ferez référence pendant le processus d&apos;installation.
* **Gestionnaire de domaine d&apos;authentification :** Pour configurer Agent Control et le connecter au contrôle de la flotte, vous devez avoir le rôle Gestionnaire de domaine d&apos;authentification. Pour plus d&apos;informations, reportez-vous à [l&apos;utilisateur Concepts de gestion](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#admin-settings).
* **Instrumentation existante :** si votre cluster Kubernetes est déjà instrumenté avec New Relic, vous devez désinstaller l’instrumentation existante avant d’installer Agent Control. Après avoir installé Agent Control, vous pouvez utiliser le contrôle de la flotte pour installer et gérer les agents infrastructure et l&apos;instrumentation sur le cluster. Pour obtenir des conseils, reportez-vous à [la section Gérer l&apos;instrumentation existante avec Agent Control](/docs/new-relic-control/agent-control/reinstrumentation).

<Callout variant="caution" title="avertissement">
  Les installations multiples d&apos;Agent Control sur le même cluster ne sont pas prises en charge.
</Callout>

## Compatibilité

Agent Control est compatible avec :

* Les trois dernières sorties mineures Kubernetes
* Minikube, Kind, Amazon EKS, Azure AKS et Google GKE

Pour les exigences système liées à Kubernetes infrastructure l&apos;agent New Relic et au collecteur New Relic OpenTelemetry (NRDOT), reportez-vous à :

* [Intégration Kubernetes : compatibilité et exigences](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements)
* [OpenTelemetry pour Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel)

## Prochaine étape

Une fois que vous avez confirmé la compatibilité et rempli toutes les conditions préalables, vous êtes prêt à [installer Agent Control](/docs/new-relic-control/agent-control/setup) sur votre cluster Kubernetes.
