## Présentation

Agent Control offre un moyen fluide et centralisé de gérer l&apos;ensemble du cycle de vie de vos agents, y compris les mises à jour. Au lieu de mettre à jour manuellement les agents sur chaque hôte ou cluster, vous définissez l&apos;état souhaité dans Fleet Control, et Agent Control gère automatiquement le reste.

Le processus de mise à jour d&apos;Agent Control et des agents qu&apos;il gère est conçu pour être non perturbateur et automatisé.

### Principaux avantages du processus de mise à jour avec New Relic Control

* Contrôle centralisé : gérez toutes les versions d&apos;agent pour l&apos;ensemble de votre flotte à partir d&apos;un seul emplacement.
* Sans interruption : les mises à jour sont effectuées à l&apos;aide de stratégies de mise à jour progressive pour éviter les temps d&apos;arrêt.
* Automatisé : une fois que vous avez défini une nouvelle version, l’ensemble du processus de déploiement et de réconciliation est géré automatiquement.
* Résilient : le système est conçu pour être résilient aux mises à jour échouées et peut revenir au dernier état correct connu si un problème survient.

## Mise à jour de vos Agents gérés

Vous pouvez mettre à jour tout agent géré par Agent Control en modifiant simplement sa version dans la configuration à distance au sein de Fleet Control. Cette approche déclarative garantit la cohérence et la fiabilité sur l&apos;ensemble de votre parc.

* **Modifiez la configuration :** accédez à la configuration de votre flotte dans Fleet Control.
* **Mettre à jour la version :** Localisez l&apos;agent que vous souhaitez mettre à jour (par exemple, `infrastructure`) et remplacez la valeur `chart_version` par le nouveau numéro de version.
* **Déployer les modifications :** Enregistrez et déployer la configuration mise à jour.

Agent Control détectera automatiquement le changement et effectuera une mise à jour progressive sans interruption, garantissant que vos nouveaux agents sont opérationnels et sains avant de remplacer les anciens.

### Exemple : Mise à jour de l&apos;agent d&apos;infrastructure New Relic

Cet exemple montre comment mettre à niveau l&apos;agent d&apos;infrastructure vers une nouvelle version.

```shell
# Remote configuration for a single agent
chart_version: "5.0.109"  # Change this value
chart_values:
  newrelic-infrastructure:
    enableProcessMetrics: true
```

## Mise à niveau d&apos;Agent Control

### Kubernetes

Les instructions suivantes s&apos;appliquent à Agent Control exécuté sur un cluster Kubernetes.

Agent Control est capable de se mettre à jour à distance. Cela garantit que le superviseur exécuté sur votre cluster est toujours à jour avec les dernières fonctionnalités, améliorations de performances et correctifs de sécurité.

Le processus de mise à jour du superviseur Agent Control est le même que pour tout autre agent géré : vous spécifiez la nouvelle version dans la configuration et déployez les modifications depuis Fleet Control.

1. Modifier la configuration d&apos;Agent Control : dans Fleet Control, accédez à la configuration d&apos;Agent Control elle-même.
2. Mettre à jour le `chart_version`: Remplacez le `chart_version` par la nouvelle version souhaitée.
3. déployer les modifications : Sauvegarder et déployer la configuration.

Ce processus déclenche une mise à jour progressive des pods Agent Control sur votre cluster. Les vérifications d&apos;état intégrées de Kubernetes garantissent que la nouvelle version est saine avant que l&apos;ancienne ne soit arrêtée pour une mise à niveau sans interruption.

### Hôtes Windows

Agent Control sous Windows est mis à jour en réexécutant le script d&apos;installation fourni dans l&apos;installation guidée. Le script télécharge la dernière version et remplace l&apos;exécutable existant tout en conservant vos fichiers de configuration.

1. Ouvrez PowerShell avec les privilèges d&apos;administrateur.

2. Exécutez le script d&apos;installation depuis l&apos;installation guidée pour la nouvelle version.

3. Vérifiez la mise à niveau en contrôlant l&apos;état du service :

   ```powershell
   Get-Service -Name newrelic-agent-control
   Invoke-WebRequest -Uri "http://localhost:51200/status" -UseBasicParsing
   ```

Les agents gérés (Infrastructure Agent, NRDOT) sont mis à jour via Fleet Control de la même manière que sur les autres plateformes : mettez à jour la version souhaitée de l&apos;agent dans la configuration de votre flotte et déployez les modifications.

### Démarrer, arrêter et redémarrer Agent Control [#start-stop-restart]

Agent Control démarre automatiquement après l&apos;exécution du script d&apos;installation.

#### Linux

Pour Linux, assurez-vous d&apos;utiliser la bonne commande pour votre système d&apos;initialisation. Sélectionnez `start`, `stop`, `restart` ou `status` selon le cas.

Voici un exemple pour `systemd`:

```shell
sudo systemctl <start|stop|restart|status> newrelic-agent-control
```

#### Windows

Exécutez ces commandes dans PowerShell avec des privilèges d&apos;administrateur :

```powershell
Start-Service -Name newrelic-agent-control
Stop-Service -Name newrelic-agent-control
Restart-Service -Name newrelic-agent-control
Get-Service -Name newrelic-agent-control
```
