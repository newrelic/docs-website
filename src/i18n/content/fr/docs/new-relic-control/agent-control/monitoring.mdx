## Présentation

Agent Control est conçu pour être un superviseur local, tandis que New Relic Control offre l&apos;expérience de supervision centralisée. Agent Control agit en tant que superviseur sur le cluster pour vos agents gérés. Son rôle principal est de s&apos;assurer que les agents sont correctement déployés et en cours d&apos;exécution. Ce guide détaille comment monitorer vos agents gérés. Pour une vue d&apos;ensemble de la santé de votre parc, vous pouvez utiliser l&apos;interface utilisateur New Relic Control. Pour un dépannage et une validation approfondis, vous pouvez effectuer des vérifications directement sur votre hôte ou votre cluster.

## monitoring centralisée dans New Relic Control

L&apos;objectif ultime de la solution Agent Control est de fournir une expérience de supervision complète et centralisée au sein de l&apos;interface utilisateur New Relic. Cela vous évitera d&apos;avoir à exécuter des commandes kubectl manuelles pour les vérifications de l&apos;état de santé et vous offrira une vue unifiée de l&apos;ensemble de votre parc. Cette expérience sera disponible pour tous les environnements pris en charge par Agent Control.

La vue centralisée renverra directement vers le dashboard d&apos;entité dédié d&apos;un agent dans New Relic. Cela vous permettra de passer rapidement du monitor de l&apos;état de santé d&apos;un parc à l&apos;examen des métriques, logs et traces spécifiques collectés par un agent individuel. Cela offre un workflow fluide, de la gestion de haut niveau à l&apos;observabilité approfondie.

## Monitoring de Kubernetes

Ces instructions s&apos;appliquent à Agent Control s&apos;exécutant dans un environnement Kubernetes.

### Contrôles de santé et dépannage sur le cluster

Ces vérifications sont effectuées à l’aide d’outils et de commandes Kubernetes standard.

Lors du dépannage d&apos;un problème, vous pouvez inspecter l&apos;état d&apos;Agent Control et de ses agents gérés directement à l&apos;aide de kubectl et de l&apos;endpoint de statut local.

### Vérifier l&apos;état de pod et les logs

Vous pouvez utiliser les commandes Kubernetes standard pour vérifier que le superviseur Agent Control et son pod de sous-agent fonctionnent comme prévu.

```shell
# Check Agent Control pods
kubectl get pods -n newrelic-agent-control

# Check sub-agent pods (e.g., in the 'newrelic' namespace)
kubectl get pods -n newrelic

# Inspect logs for the Agent Control supervisor
kubectl logs <agent-control-pod-name> -n newrelic-agent-control
```

### Utiliser le point de terminaison d&apos;état local

Agent Control expose un point de terminaison d&apos;état local qui fournit un contrôle de santé du superviseur et de tous ses agents gérés.

1. Transférer le port : ouvrez un terminal et exécutez la commande suivante pour transférer le port local vers le pod de contrôle de l’agent.

```shell
kubectl port-forward <agent-control-pod-name> 51200:51200 -n newrelic-agent-control
```

2. Demander le statut : Dans un nouveau terminal, utilisez curl pour interroger le point de terminaison du statut.

```shell
curl localhost:51200/status
```

La sortie est une réponse JSON qui fournit un état de santé détaillé, y compris les détails du déploiement, les versions de l&apos;agent et toutes les erreurs détectées.
