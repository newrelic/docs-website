/\* Si vous recevez des données télémétriques des producteurs envoyées au gateway mais aucune donnée monitoring du gateway elle-même, ce guide vous aidera à diagnostiquer et à résoudre les problèmes courants. Suivez les étapes ci-dessous en fonction du problème spécifique que vous rencontrez : \*/

## Problème : clé de licence non valide

Le gateway est configuré avec une clé de licence non valide ou qui a été remplacée.

## Solution

* Accédez directement aux logs du gateway pour vérifier le problème. Recherchez les erreurs 403 du interne monitoring pipeline et de l&apos;exportateur d&apos;utilisation, mais pas pour les données de l&apos;agent.
* Assurez-vous que la clé de licence est valide et correctement configurée. Mettez à jour la clé si nécessaire.

## Problème : règle cloud affectant les données monitoring du gateway

Les métriques produites par le gateway sont involontairement supprimées par une règle cloud .

## Solution

* Examinez les données d’utilisation des règles cloud pour identifier toute baisse involontaire.
* Modifiez des cloud règles configuration pour exclure la suppression des métriques de gateway.

## Problème : je suis limité par le débit de l&apos;API des métriques

Vous avez peut-être dépassé leur limite de requête par rapport à l&apos;API métriques, ce qui a entraîné l&apos;échec requests ultérieures du gateway avec des codes de réponse 429.

## Solution

* Vérifiez les événements de limitation de débit liés à l&apos;API métriques OpenTelemetry dans votre compte.
* Reportez-vous à la documentation sur la limitation du débit pour obtenir des conseils sur la gestion et le réglage du taux de requêtes.
