/\* Si vous rencontrez des problèmes de réception de données dans New Relic, ce guide vous aidera à diagnostiquer et à résoudre les problèmes courants. Suivez les étapes ci-dessous en fonction du problème spécifique que vous rencontrez : \*/

## Problème : problèmes de résolution DNS entre le producteur de télémétrie et le gateway

Les agents ne peuvent pas se connecter au gateway en raison d&apos;erreurs de résolution DNS.

## Solution

* Accédez aux logs de l&apos;agent (via l&apos;interface utilisateur ou directement) et recherchez les erreurs de connexion.
* Consultez votre administrateur réseau pour ajuster la configuration DNS en fonction de votre infrastructure et de la topologie de votre réseau. Reportez-vous à [la documentation de l&apos;agent](/docs/apm/new-relic-apm/getting-started/introduction-apm/) pour plus de détails.

## Problème : problèmes de certificat SSL du gateway d&apos;agent

Il y a un problème avec la configuration du certificat SSL entre l&apos;agent et le gateway.

## Solution

* Vérifiez les logs de l&apos;agent pour les erreurs de connexion SSL.
* Assurez-vous que les certificats SSL sont correctement configurés et valides, en tenant compte de votre infrastructure et de la topologie de votre réseau. Reportez-vous à [la documentation de l&apos;agent](/docs/apm/new-relic-apm/getting-started/introduction-apm/) pour plus de détails.

## Problème : mauvaise configuration du gateway

Les données atteignent le gateway mais ne parviennent pas à être publiées sur New Relic.

## Solution

* Vérifiez les métriques de requêtes sortantes et d&apos;erreurs.
* Examinez les mesures des règles ayant échoué.
* Inspectez les logs des pods qui ne démarrent pas.
* Corrigez les paramètres configuration du gateway et assurez-vous que tous les pods sont opérationnels.

## Problème : producteur/protocole de télémétrie non pris en charge

Les données sont envoyées à partir d&apos;une API ou d&apos;un protocole non pris en charge.

## Solution

* Vérifiez la compatibilité avec New Relic. Un client API incorrect renverra un code d&apos;état `501` .
* Déposez une demande de fonctionnalité pour obtenir de l&apos;aide.
* Configurez le producteur de télémétrie pour envoyer des données directement à New Relic.

## Problème : Règle destructrice

Une règle supprime toutes les données, les empêchant d&apos;atteindre New Relic.

## Solution

* Vérifiez les métriques de données de dépôt.
* Modifiez ou supprimez la règle destructive pour autoriser le flux de données.

## Problème : données manquantes

Des données sont manquantes dans le backend New Relic après l&apos;ingestion.

## Solution

* Vérifiez les métriques d’erreur et les délais d’expiration côté client.
* Évaluer les signes d’épuisement des ressources.
* Vérifiez [l&apos;état de New Relic](https://status.newrelic.com/).
* Examiner les logs du gateway pendant la période concernée
