/\* Si vous ne recevez aucune donnée télémétrique des producteurs configurés pour être envoyés au gateway, mais que le gateway produit ses propres données monitoring, ce guide vous aidera à diagnostiquer et à résoudre les problèmes courants. Suivez les étapes ci-dessous en fonction du problème spécifique que vous rencontrez : \*/

## Problème : mauvaise configuration du producteur de télémétrie

Le producteur télémétrique est mal configuré, ce qui entraîne l&apos;envoi de données télémétriques au gateway.

## Solution

* Accédez aux logs du producteur pour identifier les erreurs configuration .
* Reportez-vous à la documentation appropriée de l&apos;API, de l&apos;agent ou du producteur de télémétrie pour connaître les étapes de configuration correctes.

## Problème : mon débit est limité

Vous atteignez les limites de débit sur vos données télémétriques.

## Solution

* Vérifiez les codes de réponse du client HTTP du gateway pour les codes d&apos;état 429 reçus de l&apos;API New Relic.
* Consultez l&apos;événement créé sur votre compte indiquant une limitation de débit.
* Reportez-vous à [la documentation sur les limitations de débit](https://docs.newrelic.com/search-results/?query=rate%20limiting&page=2) pour obtenir des conseils sur la gestion et l&apos;ajustement des tarifs de données télémétriques.
