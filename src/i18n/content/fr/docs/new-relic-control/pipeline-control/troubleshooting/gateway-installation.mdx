/\* Si vous essayez d&apos;installer le gateway pour la première fois et rencontrez des problèmes, ce guide vous aidera à diagnostiquer et à résoudre les problèmes courants. Suivez les étapes ci-dessous en fonction du problème spécifique que vous rencontrez : \*/

## Problème : capacités utilisateur insuffisantes

Vous ne disposez pas des autorisations nécessaires associées à l&apos;administrateur de produit d&apos;organisation et au gestionnaire d&apos;organisation.

## Solution

* L&apos;utilisateur verra un message d&apos;erreur indiquant : « Vous ne disposez pas des capacités requises au niveau de l&apos;organisation pour configurer l&apos;authentification de l&apos;agent. »
* Demandez aux administrateurs de votre compte de se voir attribuer un rôle incluant les fonctionnalités nécessaires à la création d&apos;identité système.
* Reportez-vous à [la documentation sur les autorisations utilisateur](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/) pour obtenir des conseils.

## Problème : version obsolète du graphique Helm

Le script de ligne de commande pour installer le graphique Helm échoue car votre version locale de Helm est obsolète.

## Solution

* Mettez à jour votre installation Helm locale vers la dernière version pour garantir la compatibilité avec le script d&apos;installation.
* Recherchez le message d&apos;erreur indiquant la nécessité d&apos;une mise à jour Helm et suivez les instructions pour la mise à niveau.
