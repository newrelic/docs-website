Suivez ces étapes pour configurer un nom d&apos;hôte DNS privé pour le cluster de gateway afin de faciliter une communication fiable entre les agents New Relic APM et le gateway au sein de votre réseau :

1. Créez un nom d’hôte DNS à l’aide [d’AWS Route 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/hosted-zones-private.html).
2. Obtenez un certificat pour le nom d’hôte de la passerelle via [ACM](https://docs.aws.amazon.com/acm/latest/userguide/acm-public-certificates.html).

## Prochaine étape

Ensuite, vous apprendrez à [dimensionner et à faire évoluer votre cluster de gateway](/docs/new-relic-control/pipeline-control/sizing-and-scaling).
