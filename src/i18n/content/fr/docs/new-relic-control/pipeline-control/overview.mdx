Contrôle de pipeline est le composant de gestion des données de New Relic Control. Il peut aider à filtrer et à éliminer les données de faible valeur avant qu&apos;elles ne soient ingérées sur la plateforme New Relic.

Le Contrôle de pipeline se compose de deux sous-composants : le gateway de Contrôle de pipeline (simplement appelée le gateway) et les règles cloud de Contrôle de pipeline.

Le gateway est conçu pour filtrer les données ou les attributs avant qu&apos;ils ne quittent votre réseau. Les règles du cloud effectuent ce traitement dans le cloud New Relic. Dans les deux scénarios, Contrôle de pipeline utilise une syntaxe de requête NRQL identique pour traiter les données.

Pour utiliser le gateway, vous devez d’abord la configurer dans votre environnement. Les règles du cloud n&apos;impliquent aucune configuration.

Pour comprendre les coûts associés à l&apos;utilisation du Contrôle de pipeline, consultez [Coûts du Contrôle de pipeline](/docs/new-relic-control/pipeline-control/costs).

## Prérequis

Avant de commencer, vous devez avoir :

* Un compte New Relic avec des autorisations d&apos;administrateur de produit et de gestionnaire d&apos;organisation

<Callout variant="important" title="important">
  Le rôle d&apos;utilisateur d&apos;administrateur de produit de l&apos;organisation et de gestionnaire de l&apos;organisation doit :

  * Installer le gateway
  * Ajouter un cluster à un gateway
  * Mettre à jour le gateway (par exemple, de nouvelles règles et des modifications de paramètres)
  * Dèployer une configuration de gateway
  * Afficher les règles du gateway
</Callout>

* Familiarité avec les opérations et la gestion de Kubernetes
* Accès administrateur du cluster Kubernetes pour déployer et gérer les ressources au sein de votre cluster
* Autorisations pour créer et gérer un nom d&apos;hôte DNS privé au sein de votre réseau
* Autorisations pour créer et gérer les enregistrements DNS
* Helm installé localement
* Compréhension de NRQL pour créer et gérer des règles de pipeline

<Callout variant="important">
  * Les règles de contrôle de pipeline s&apos;appliquent uniquement aux données qui arrivent à partir du moment où vous créez la règle, elles ne suppriment pas les données qui [ont déjà été ingérées](/docs/telemetry-data-platform/ingest-manage-data/manage-data/manage-data-retention#data-deletion).
  * Lors de la création de règles, vous êtes responsable de vous assurer que les règles identifient et suppriment avec précision les données qui répondent aux conditions que vous avez établies. Vous êtes également responsable du monitoring de la règle, ainsi que des données que vous divulguez à New Relic.
</Callout>
