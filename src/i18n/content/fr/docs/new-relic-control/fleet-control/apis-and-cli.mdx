Fleet Control offre un accès programmatique complet via des API et une interface en ligne de commande, vous permettant d&apos;automatiser les opérations de flotte, de vous intégrer aux outils existants et de gérer votre instrumentation à grande échelle sans dépendre de l&apos;interface utilisateur.

## Options d&apos;accès par programmation

<DNT>Fleet Control</DNT> propose trois méthodes d&apos;accès programmatique :

* <DNT>**NerdGraph API:**</DNT> Pour les opérations de gestion de flotte, y compris la création de flottes, la gestion des membres et le contrôle des déploiements
* <DNT>**Blob Storage API:**</DNT> Pour les opérations de configuration d&apos;agent, y compris la création, la gestion des versions et la récupération des configurations
* <DNT>**Fleet Control CLI:**</DNT> Pour la gestion interactive en ligne de commande et les scripts d&apos;automatisation

<Callout title="Important : Fleet Control utilise plusieurs API" variant="important">
  <DNT>Fleet Control&apos;s</DNT> La surface de l&apos;API est divisée entre deux systèmes :

  * <DNT>**NerdGraph**</DNT> gère les opérations de flotte (flottes, membres, déploiements)
  * <DNT>**Blob Storage API**</DNT> gère les configurations de l&apos;agent (configs, versions, contenu)

  Cette séparation est intentionnelle. Le <DNT>Blob Storage API</DNT> est un service <DNT>New Relic</DNT> général optimisé pour les téléversements de fichiers et le versionnage, tandis que <DNT>NerdGraph</DNT> est optimisé pour les requêtes de données structurées. Comprendre quelle API utiliser pour votre tâche spécifique est essentiel pour une intégration réussie.
</Callout>

## Quand utiliser les API vs la CLI

Comprendre quelle méthode d&apos;accès convient le mieux à votre cas d&apos;utilisation vous aidera à travailler plus efficacement avec <DNT>Fleet Control</DNT>.

### API (<DNT>NerdGraph + Blob Storage API</DNT>)

Les API sont idéales pour :

* **Automatisation et intégration :** Intégrer les opérations <DNT>Fleet Control</DNT> dans les pipelines <DNT>CI/CD</DNT>, les outils personnalisés ou d&apos;autres workflows automatisés
* **Infrastructure as Code :** Gérer les configurations de flotte parallèlement à vos définitions d&apos;infrastructure
* **Intégration de système à système :** Connecter <DNT>Fleet Control</DNT> à d&apos;autres plateformes, systèmes de gestion de tickets ou outils d&apos;orchestration
* **Workflows complexes :** Création d&apos;une automatisation sophistiquée combinant plusieurs opérations ou s&apos;intégrant à des sources de données externes
* **Accès programmatique :** Lorsque vous avez besoin d&apos;intégrer des capacités de <DNT>Fleet Control</DNT> dans vos propres applications ou services

### <DNT>Fleet Control CLI</DNT>

La CLI est idéale pour :

* **Opérations interactives :** Tâches rapides, pilotées par l&apos;humain, depuis la ligne de commande
* **Gestion ad hoc :** Opérations administratives ponctuelles ou dépannage
* **Workflows basés sur le terminal :** Si vous préférez travailler directement dans votre terminal
* **Tests locaux :** Expérimenter les opérations de flotte avant de les automatiser
* **Scripts shell :** Opérations séquentielles simples (bien que les API soient plus adaptées pour une automatisation complexe)
* **Apprentissage et exploration :** La CLI fournit un retour immédiat et structuré qui vous aide à comprendre les opérations de Fleet Control

La CLI repose sur les mêmes API, elles offrent donc fonctionnellement les mêmes capacités. Le choix dépend de la préférence d&apos;interface et du cas d&apos;utilisation : la CLI pour l&apos;interaction humaine et la praticité, les API pour l&apos;intégration programmatique et l&apos;automatisation.

## API NerdGraph (Opérations sur la flotte)

<DNT>NerdGraph</DNT> est l&apos;API <DNT>New Relic&apos;s GraphQL</DNT> qui permet d&apos;accéder aux opérations de Fleet Control pour gérer les flottes, les membres et les déploiements. Contrairement aux configurations d&apos;agent (qui utilisent l&apos;API Blob Storage), toutes les opérations au niveau de la flotte utilisent NerdGraph.

### Ce que vous pouvez faire avec le <DNT>NerdGraph API</DNT>

Le <DNT>NerdGraph API</DNT> fournit des mutations pour :

* **Gestion de flotte :** Créer, mettre à jour et supprimer des flottes
* **Gestion des membres de la flotte :** Ajouter et retirer des entités gérées des anneaux de la flotte
* **Gestion des déploiements :** Créer, mettre à jour, supprimer et déclencher des déploiements

Pour une documentation complète incluant les descriptions des paramètres, des exemples de requêtes/réponses et les spécifications détaillées des mutations, consultez **[le tutoriel<DNT>NerdGraph Fleet Control</DNT> ](/docs/apis/nerdgraph/examples/nerdgraph-fleet-control-tutorial)**.

Pour explorer l&apos;API de manière interactive, utilisez le [<DNT>NerdGraph Explorer</DNT>](https://one.newrelic.com/nerdgraph-graphiql) (ou la [région UE](https://one.eu.newrelic.com/nerdgraph-graphiql)).

Pour des informations générales sur <DNT>NerdGraph</DNT>, consultez [Introduction à NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph).

## <DNT>Blob Storage API</DNT> (configurations d&apos;agent)

Le <DNT>Blob Storage API</DNT> est un service <DNT>New Relic</DNT> conçu pour téléverser et gérer des fichiers. <DNT>Fleet Control</DNT> utilise le <DNT>Blob Storage API</DNT> pour gérer les configurations d&apos;agent, car <DNT>NerdGraph</DNT> est optimisé pour les requêtes de données structurées plutôt que pour le transfert efficace de fichiers. Cette API gère toutes les opérations liées à la configuration, y compris la création de configurations, le versionnage, la récupération de contenu et la suppression.

<Callout title="Important : Les configurations utilisent l'API Blob Storage" variant="important">
  Les configurations de l&apos;Agent n&apos;utilisent **pas** <DNT>NerdGraph</DNT>. Toutes les opérations de configuration — création, versionnement, récupération et suppression — sont gérées via le <DNT>Blob Storage API</DNT> à `https://blob-api.service.newrelic.com/v1/e`.
</Callout>

### Ce que vous pouvez faire avec l&apos;API Blob Storage

L&apos;API Blob Storage prend en charge :

* **Créer des configurations d&apos;agent :** Initialiser de nouvelles configurations avec des métadonnées (nom, type d&apos;agent, type d&apos;entité gérée)
* **Gestion des versions :** Créer de nouvelles versions de configurations existantes
* **Récupérer les configurations :** Obtenir le contenu de la configuration par version ou la plus récente
* **Opérations de suppression :** Supprimer des configurations ou des versions spécifiques
* **Lister les versions :** Afficher toutes les versions d&apos;une configuration avec les horodatages

Pour une documentation complète incluant des exemples curl, les formats de requête/réponse, les spécifications d&apos;en-tête et les types d&apos;agents pris en charge, consultez **[la référence de l&apos;API Blob Storage](/docs/apis/intro-apis/blob-storage-api)** - Guide complet avec des exemples curl et les détails des paramètres

La référence de l&apos;API inclut des exemples fonctionnels pour toutes les opérations, les exigences d&apos;authentification et les bonnes pratiques pour gérer les configurations d&apos;agent de manière programmatique.

## <DNT>Fleet Control CLI</DNT>

Le <DNT>Fleet Control CLI</DNT> est une interface en ligne de commande complète pour gérer toutes les entités <DNT>Fleet Control</DNT>, y compris les flottes, les configurations, les déploiements, les membres et les requêtes d&apos;entité. La CLI fournit un workflow structuré, basé sur le terminal, aussi bien pour les opérations interactives que pour l&apos;automatisation.

### Hiérarchie des commandes

<DNT>Fleet Control</DNT> les commandes sont organisées par type de ressource pour une navigation intuitive :

```
newrelic fleetcontrol
├── fleet                    # Fleet management
│   ├── create              # Create a new fleet
│   ├── get                 # Get fleet details
│   ├── search              # Search fleets
│   ├── update              # Update fleet
│   ├── delete              # Delete fleet(s)
│   └── members             # Manage fleet members
│       ├── add             # Add entities to ring
│       ├── remove          # Remove entities from ring
│       └── list            # List fleet members
│
├── configuration            # Configuration management
│   ├── create              # Create configuration
│   ├── get                 # Get configuration content
│   ├── delete              # Delete configuration
│   └── versions            # Manage configuration versions
│       ├── list            # List all versions
│       ├── add             # Add new version
│       └── delete          # Delete specific version
│
├── deployment               # Deployment management
│   ├── create              # Create deployment
│   ├── update              # Update deployment
│   ├── deploy              # Trigger deployment
│   └── delete              # Delete deployment
│
└── entities                 # Entity queries
    ├── get-managed         # List managed entities
    └── get-unassigned      # List available entities
```

### Fonctionnalités clés

Le <DNT>Fleet Control CLI</DNT> vous permet de :

* **Centralisez les opérations des agents :** Installez, supervisez, configurez et mettez à jour tous les agents et intégrations depuis la ligne de commande
* **Gérer les flottes :** Regroupez les entités par besoins d&apos;instrumentation et mettez à jour les versions de l&apos;agent pour des flottes entières
* **Créer et versionner des configurations :** Créez des configurations d&apos;agent et gérez plusieurs versions
* **Contrôlez les déploiements :** créez, mettez à jour et déclenchez des déploiements avec des déploiements par anneaux
* **Interroger les entités :** Listez les entités gérées et non assignées pour comprendre le paysage de votre parc.

### Prérequis

Avant d&apos;utiliser les commandes <DNT>Fleet Control CLI</DNT>, vous aurez besoin de :

**Variables d&apos;environnement requises :**

```bash
# Your New Relic User API Key
export NEW_RELIC_API_KEY="NRAK-YOUR-API-KEY-HERE"

# Your New Relic Account ID
export NEW_RELIC_ACCOUNT_ID="your-account-id"

# Optional: Specify region (defaults to US)
export NEW_RELIC_REGION="US"  # or "EU" for European accounts
```

**Obtenir vos identifiants :**

1. **Clé d&apos;API :** générez une clé d&apos;API utilisateur depuis [one.newrelic.com](https://one.newrelic.com) → Cliquez sur votre nom → API Keys → Create a &quot;User&quot; key
2. **ID de compte :** Trouvez votre ID de compte dans l&apos;URL New Relic après `/accounts/` ou dans les paramètres du compte
3. **ID d&apos;organisation :** La plupart des commandes acceptent un flag `--organization-id` facultatif. S&apos;il n&apos;est pas fourni, la CLI récupérera automatiquement votre ID d&apos;organisation à l&apos;aide de vos identifiants API.

## Débuter avec l&apos;accès programmatique

Pour commencer à utiliser <DNT>Fleet Control</DNT> de manière programmatique :

1. **Generate a User API Key** depuis les paramètres de votre compte New Relic

2. **Identifiez votre ID de compte et votre ID d&apos;organisation** depuis la plateforme New Relic

3. **Choisissez votre méthode d&apos;accès :**

   * Utilisez le <DNT>**NerdGraph Explorer**</DNT> pour explorer les opérations de la flotte de manière interactive
   * Passez en revue les opérations <DNT>**Blob Storage API**</DNT> pour la gestion de la configuration
   * Installez et configurez le <DNT>**Fleet Control CLI**</DNT> pour les workflows basés sur le terminal

4. **Commencez par des opérations de lecture** pour vous familiariser avec les structures de données (rechercher des flottes, lister des entités, obtenir des configurations)

5. **Automatisez progressivement** en combinant des opérations dans des scripts ou des workflows

Pour plus d&apos;aide sur les API, consultez notre documentation [Introduction à NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) et notre [documentation sur les clés d&apos;API](/docs/apis/intro-apis/new-relic-api-keys).
