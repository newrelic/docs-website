Ce guide constitue le moyen le plus rapide de démarrer avec <DNT>Fleet Control</DNT>. Nous vous guiderons à travers le workflow essentiel du « happy path » pour créer votre première flotte et lancer un déploiement de base.

## aperçu du flux de travail

Le processus de configuration de <DNT>Fleet Control</DNT> suit un ordre précis. Vous devez créer une flotte *avant* de pouvoir installer <DNT>Agent Control</DNT>, car le processus d&apos;installation nécessite une flotte à laquelle s&apos;associer. Une fois <DNT>Agent Control</DNT> installé, l&apos;entité est automatiquement affectée à cette flotte. Par conséquent, le workflow est :

1. **Créez une flotte** sur la plateforme New Relic.
2. **Installez <DNT>Agent Control</DNT>** sur vos entités. L&apos;assistant d&apos;installation utilise les détails de votre parc pour attribuer chaque entité, ce qui en fait une &quot;Entité gérée attribuée&quot;.
3. **Exécutez un déploiement** pour installer et configurer les agents sur l&apos;entité qui font désormais partie de votre flotte.

## Prérequis

Vous aurez besoin du rôle de **Organization Manager** ou d’un rôle personnalisé avec les autorisations de gestion de flotte équivalentes pour effectuer ces actions.

## Étape 1 : Créez votre première flotte

La première étape consiste à créer une flotte. Dans <DNT>Fleet Control</DNT>, une **flotte** est un ensemble homogène d&apos;entités gérées (telles que des clusters Kubernetes ou des hôtes) qui partagent des configurations d&apos;instrumentation identiques. Les flottes agissent comme des regroupements logiques, vous permettant de gérer les versions et les configurations des agents de manière cohérente et à grande échelle dans votre environnement, évitant ainsi la dérive de configuration et les lacunes d&apos;observabilité.

1. Connectez-vous à la plateforme New Relic avec le compte approprié.

2. Accédez à <DNT>**All capabilities → New Relic Control**</DNT>.
   * **Astuce :** Vous pouvez épingler <DNT>**New Relic Control**</DNT> à la barre de navigation de gauche pour y accéder facilement en cliquant sur l&apos;icône d&apos;épingle.

3. Depuis le menu **New Relic Control**, sélectionnez **Fleets** sous la section **<DNT>Fleet Control</DNT>**.

4. Sur la page principale Fleets, cliquez sur **Create a fleet**.

5. Dans le modal :

   * Donnez un nom significatif à votre flotte (par exemple, `aws-eks-app-prod-us-east1`) et une **Description** facultative.
   * Sélectionnez le <DNT>**Type of Fleet**</DNT>. Pour ce guide, nous sélectionnerons <DNT>**Kubernetes Cluster**</DNT>, qui est généralement disponible. La prise en charge des **hôtes Linux** et des **hôtes Windows** est actuellement en préversion publique.
   * Laissez <DNT>**Access management**</DNT> par défaut. Il s&apos;agit d&apos;un paramètre facultatif pour le FGA (contrôle d&apos;accès à granularité fine).

6. Cliquez sur <DNT>**Create Fleet**</DNT>.

<img title="Create fleet modal" alt="The Create a fleet modal with input fields for fleet name and description, Type of Fleet selection, Access management options, and Create Fleet button." src="/images/fleet-control-create-fleet-modal.webp" />

Une fois la flotte créée, vous disposez du composant nécessaire pour procéder à l&apos;installation de <DNT>Agent Control</DNT>.

## Étape 2 : Installation <DNT>Agent Control</DNT>

Maintenant que votre flotte existe, installez <DNT>Agent Control</DNT> sur les entités que vous souhaitez y associer.

Suivez les instructions de la [documentation de configuration<DNT>Agent Control</DNT> ](/docs/new-relic-control/agent-control/setup). L&apos;assistant fournit un fichier `values.yaml` et une commande Helm qui installe <DNT>Agent Control</DNT> et assigne l&apos;entité gérée à la flotte que vous venez de créer. Une fois terminé, vos entités apparaîtront en tant qu&apos;**Assigned Managed Entities** dans votre flotte.

## Étape 3 : Créer une configuration d’agent

Avant de déployer l’agent New Relic Infrastructure, vous devez créer une configuration pour celui-ci. Toute configuration que vous créez est enregistrée globalement et peut être utilisée sur votre compte New Relic.

1. Dans le menu de gauche <DNT>New Relic Control</DNT>, cliquez sur <DNT>**Configurations**</DNT>.

2. Cliquez sur <DNT>**Create configuration**</DNT>.

3. Dans le modal :

   * Pour <DNT>Managed entity type</DNT>, sélectionnez <DNT>**Kubernetes Cluster**</DNT>.
   * Pour <DNT>agent type</DNT>, sélectionnez <DNT>**New Relic Infrastructure**</DNT> dans le menu déroulant.
   * Donnez à la configuration un nom significatif et unique. Étant donné que la configuration est globale, un nom clair est important. Par exemple : `aws-eks-app-infra`.

4. Cliquez sur <DNT>**Continue**</DNT>.

5. Vous verrez deux panneaux. Le panneau de droite contient un modèle par défaut. Cliquez sur <DNT>**Copy**</DNT> pour copier l&apos;intégralité du modèle.

6. Collez le texte copié dans le panneau de configuration de gauche.

7. Cliquez sur <DNT>**Save**</DNT>. Votre configuration est désormais enregistrée globalement.

<img title="Configuration editor" alt="The configuration editor showing two side-by-side panes: the left pane for authoring YAML configuration and the right pane displaying a template with default settings and examples to copy from." src="/images/fleet-control-config-editor.webp" />

## Étape 4 : Créez et exécutez votre premier déploiement

Un déploiement est un ensemble contrôlé d’actions qui peuvent installer, supprimer ou configurer des agents sur l’entité gérée de votre flotte. Vous allez maintenant en créer un pour déployer l’agent New Relic Infrastructure.

1. Revenez à l’onglet **Fleets** et sélectionnez la flotte que vous avez créée précédemment.
2. Sur la page récapitulative de la flotte, cliquez sur **Create a deployment**. (Si la flotte dispose d&apos;un déploiement existant, accédez à l&apos;onglet **Deployments** et cliquez sur **Create deployment**).
3. Donnez à votre déploiement un **Name** obligatoire. Un nom significatif fournit un contexte historique et aide au dépannage. Considérez un modèle qui inclut le changement, une version ou un ID de ticket. Par exemple : `v1.0-initial-infra-deploy` ou `add-logging-JIRA-123`.
4. Cliquez sur **Add an agent**.
5. Sélectionnez le type d’agent pour lequel vous avez créé la configuration : **New Relic Infrastructure**.
6. **Sélectionnez une version de l&apos;Agent** dans le menu déroulant. La dernière version est sélectionnée par défaut, mais vous pouvez choisir parmi les autres versions disponibles. Vous pouvez voir des métadonnées supplémentaires telles que des notes de version ou des mises à jour de sécurité pour la version sélectionnée.
7. Cliquez sur **Next**.
8. Dans la liste des configurations disponibles, sélectionnez la configuration `aws-eks-app-infra` que vous venez de créer. La version sera `v1` par défaut.

<img title="Select configuration modal" alt="The Select configuration and version modal showing a table of available configurations filtered by agent type, with configuration details and a preview pane displaying the YAML content of the selected configuration." src="/images/fleet-control-select-configuration.webp" />

9. Cliquez sur **Add to deployment**. L&apos;agent et sa configuration associée apparaîtront désormais dans un tableau appelé **Agents**.
10. Cliquez sur **Save draft**.
11. Lorsque vous êtes prêt, cliquez sur **Start deployment**. Une fenêtre de confirmation apparaîtra.

<img title="Deployment confirmation" alt="The deployment confirmation modal with an 'Are you ready to deploy?' heading, a checklist of pre-deployment items to verify, and Cancel and Start deployment buttons." src="/images/fleet-control-deployment-confirm.webp" />

12. Dans le modal, cliquez sur **Start deployment** pour lancer.

<img title="Deployment creation pane" alt="The deployment creation pane with Name and Description fields, an Add an agent button, and a Managed entities table showing clusters with their health status and canary designation. Save draft and Start deployment buttons are shown." src="/images/fleet-control-deployment-pane.webp" />

Un écran &quot;EN COURS&quot; affichera le statut, le nombre d&apos;entités gérées et d&apos;agents concernés, ainsi qu&apos;un ID de déploiement.

<img title="Deployment progress" alt="The deployment progress screen showing IN PROGRESS status with progress bars for Agents and Managed entities, failure count, and a table displaying real-time deployment status for each cluster." src="/images/fleet-control-deployment-progress.webp" />

Une fois terminé, la page se mettra à jour avec un statut &quot;Terminé&quot; et des tableaux récapitulatifs détaillant le déploiement, les agents et les entités gérées spécifiques qui ont été instrumentées.

<img title="Deployment complete" alt="The completed deployment summary showing COMPLETED status, final statistics for agents and managed entities, Details and Deployment Events tabs, and tables showing deployment results." src="/images/fleet-control-deployment-complete.webp" />

Cette vue récapitulative est organisée en deux onglets :

* **Onglet Details :** Affiche les métadonnées de déploiement et les tableaux récapitulatifs décrits ci-dessus.
* **Onglet Deployment Events :** fournit des données granulaires au niveau des événements pour le dépannage ou pour vérifier la réussite du déploiement.

## Prochaines étapes

Félicitations pour le déploiement de votre première flotte ! Vous avez appris le workflow de base pour gérer votre instrumentation à grande échelle.

Pour approfondir ce que vous avez appris, consultez notre page sur la [gestion des flottes](/docs/new-relic-control/fleet-control/manage-fleets). Elle fournit des détails supplémentaires sur les fonctionnalités quotidiennes plus nuancées de <DNT>Fleet Control</DNT>, notamment comment :

* Créez et versionnez la configuration de votre agent.
* Gérez les détails et les paramètres de votre flotte.
* Utilisez l&apos;onglet de niveau supérieur **Entités gérées** pour voir toute votre infrastructure avec <DNT>Agent Control</DNT> installé, y compris les entités assignées et non assignées.
* Plongez dans une flotte spécifique pour afficher la **configuration effective** sur son onglet **Agents** et comprendre les différents statuts de déploiement et d&apos;instrumentation.
