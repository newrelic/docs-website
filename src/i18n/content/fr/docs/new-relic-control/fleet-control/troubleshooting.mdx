## Dépannage des déploiements avec les événements de déploiement

<DNT>Fleet Control</DNT> fournit un onglet <DNT>deployment events</DNT> qui offre une visibilité granulaire sur chaque événement survenant lors d&apos;un déploiement. Il s&apos;agit de l&apos;outil principal pour le dépannage des problèmes de déploiement et la compréhension des causes profondes.

### Accéder aux événements de déploiement

Il existe deux façons d&apos;accéder à l&apos;onglet Deployment Events :

**Depuis l&apos;historique des déploiements :**

1. Accédez à l&apos;onglet <DNT>deployments</DNT> dans une flotte spécifique.
2. Cliquez sur la ligne du déploiement que vous souhaitez examiner.
3. Sur la vue récapitulative du déploiement, cliquez sur l&apos;onglet <DNT>Deployment Events</DNT>.

**Après avoir créé un nouveau déploiement :**

1. Après le démarrage d&apos;un déploiement, la vue de progression se termine et passe à la vue de synthèse du déploiement.
2. Sur cette vue récapitulative, cliquez sur l&apos;onglet <DNT>Deployment Events</DNT> pour afficher immédiatement les événements pour le dépannage ou la vérification.

### Comprendre le tableau des événements de déploiement

La table <DNT>deployment events</DNT> affiche des données atomiques au niveau de l&apos;événement pour chaque action effectuée lors d&apos;un déploiement. Le tableau comprend les colonnes suivantes :

* <DNT>**Timestamp:**</DNT> Moment où l&apos;événement s&apos;est produit.
* <DNT>**Status:**</DNT> Le statut de l&apos;événement (par exemple, <DNT>Completed, Config Applied, Applying</DNT>).
* <DNT>**Managed entity name:**</DNT> Le nom du cluster ou de l&apos;hôte où l&apos;événement s&apos;est produit.
* <DNT>**Agent name:**</DNT> L&apos;agent spécifique impliqué dans l&apos;événement.
* <DNT>**Agent type:**</DNT> Le type d&apos;agent (par exemple, <DNT>New Relic Infrastructure, Fluent Bit</DNT>).
* <DNT>**Event:**</DNT> Le type d&apos;événement qui s&apos;est produit.
* <DNT>**Details:**</DNT> Informations supplémentaires sur l&apos;événement, incluant souvent des messages d&apos;erreur pour les événements en échec.
* <DNT>**Duration:**</DNT> Temps mis par l&apos;événement pour se terminer.

### Filtrage et recherche d&apos;événements

L&apos;onglet <DNT>deployment events</DNT> offre plusieurs façons d&apos;affiner les événements :

* <DNT>**Search:**</DNT> Utilisez la barre de recherche pour trouver des événements spécifiques par mot-clé.
* <DNT>**Filter by entities:**</DNT> Filtrer pour afficher les événements d&apos;une entité gérée spécifique. Cela est particulièrement utile lorsque vous disposez d&apos;un parc important (par exemple, 100 clusters) et que seule une ou quelques entités ont échoué.
* <DNT>**Filter by status:**</DNT> Filtrez les événements par leur statut (par exemple, <DNT>Completed, Config Applied, Applying</DNT>) pour vous concentrer sur les événements réussis ou problématiques.

### Dépannage avancé avec NRQL

Pour un dépannage plus avancé, vous pouvez accéder à la requête NRQL sous-jacente qui alimente le tableau Deployment Events :

1. Dans l&apos;onglet <DNT>Deployment Events</DNT>, localisez le bouton à droite de l&apos;en-tête du tableau d&apos;événements.
2. Cliquez sur ce bouton pour ouvrir un onglet de requête NRQL dans le <DNT>Data Explorer</DNT>.
3. La requête sera pré-remplie avec l&apos;ID de déploiement et affichera des colonnes supplémentaires avec des informations plus détaillées, y compris divers ID et métadonnées.
4. Vous pouvez modifier cette requête si nécessaire pour investiguer davantage le problème.
5. Utilisez les fonctionnalités standard de <DNT>Data Explorer&apos;s</DNT>, telles que le basculement entre les vues au format tableau et JSON.

Pour plus de détails sur l&apos;utilisation de <DNT>Data Explorer</DNT>, consultez notre [introduction à l&apos;Explorateur de données](/docs/query-your-data/explore-query-data/query-builder/introduction-new-data-explorer/).
