Ce document décrit les étapes à suivre pour résoudre les problèmes courants lors de l&apos;installation ou du fonctionnement du contrôle de la flotte.

## Utilisation de l&apos;explorateur de données

New Relic Data Explorer est un outil puissant pour enquêter sur les problèmes. Vous pouvez y accéder en cliquant sur **Query your data** en bas de la plupart des pages de la plateforme New Relic, ou en utilisant le raccourci clavier `CTRL+SHIFT+O`.

Pour des informations plus détaillées sur ses fonctionnalités, consultez notre [introduction à l&apos;explorateur de données](/docs/query-your-data/explore-query-data/query-builder/introduction-new-data-explorer/).

### Enquêter sur un événement majeur avec NRQL

Pour les problèmes de déploiement complexes, vous pouvez exécuter une requête NRQL dans l&apos;explorateur de données pour voir les données granulaires au niveau de l&apos;événement pour un déploiement spécifique. Cela vous permet de voir la séquence exacte des actions et les erreurs survenues.

Pour enquêter sur un déploiement :

1. Depuis l&apos;onglet **Deployments** d&apos;une flotte spécifique, cliquez sur le nom du déploiement que vous souhaitez étudier. L&apos; **ID de déploiement** sera répertorié dans les détails du résumé. Copiez cette pièce d&apos;identité.
2. Ouvrez l&apos;explorateur de données.
3. Assurez-vous que le bon compte est sélectionné.
4. Collez la requête suivante dans le générateur de requêtes, en remplaçant l&apos;espace réservé par l&apos;ID que vous avez copié.
5. Exécutez la requête. Pour une analyse plus facile, visualisez les résultats sous forme de **Table**. La colonne `details` contient souvent des messages d&apos;erreur perspicaces.

```nrql
FROM FleetDeployment SELECT timestamp, action, status, desiredCompletedStatus, agentTypes, ringName, details, agentUid, agentControlUids, fleetId, deploymentId, duration, action WHERE deploymentId = '<YOUR_DEPLOYMENT_ID>' SINCE 1 hour ago LIMIT MAX
```
