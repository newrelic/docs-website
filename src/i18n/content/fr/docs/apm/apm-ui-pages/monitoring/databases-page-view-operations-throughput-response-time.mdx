Dans APM, la page <DNT>**Databases**</DNT> affiche la base de données et les données de cache d&apos;une application. La page affiche les transactions de base de données individuelles sous forme de tableau triable, ainsi que les opérations, le débit et le temps de réponse sous forme de graphiques. Si [des requêtes lentes](/docs/apm/applications-menu/monitoring/viewing-slow-query-details) ont été signalées, elles s&apos;affichent lorsque vous sélectionnez une transaction.

Nous divisons les informations de la base de données en trois parties :

* Aperçu de toute la base de données (page principale)

  * Top cinq des opérations de base de données par temps horloge
  * base de données temps de réponse
  * base de données débit ( requests d&apos;opération par minute ou <DNT>**rpm**</DNT>)

* Principales opérations de base de données (sous-page)

  * Heure de la requête
  * débit
  * Consommation de temps par appelant

* Tableau avec toutes les opérations de la base de données (sous-page)

Pour plus de détails, notamment sur la manière dont vous pouvez facilement vous préparer à cette transition, consultez notre [publication sur le forum d&apos;assistance](https://discuss.newrelic.com/t/q1-bulk-eol-announcement-fy23/181744).

## Navigation [#database-page]

Choisissez l&apos;une des options suivantes pour ouvrir la page :

* Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities &gt; (select an app) &gt; Monitor &gt; Databases**</DNT>.
* Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; services &gt; (select an app) &gt; Monitor &gt; Databases**</DNT>.

<Callout variant="tip">
  Pour obtenir un aperçu de haut niveau de toutes vos applications et services, utilisez notre [explorateur d&apos;entités](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer).
</Callout>

## Opérations de filtrage par fournisseur de base de données [#db\_filtering][#db_filtering]

Si disponible : au-dessus de la liste des opérations de base de données, les boutons vous permettent de [filtrer](#availablity) jusqu&apos;à un fournisseur de base de données spécifique.

* Pour filtrer le contenu par fournisseur de base de données, sélectionnez son nom.
* Pour supprimer le filtre, sélectionnez <DNT>**All**</DNT>.

## Examiner les principales opérations de la base de données [#top\_db\_operations][#top_db_operations]

La page principale <DNT>**Databases**</DNT> affiche une liste des principales opérations de base de données que vous pouvez trier de différentes manières.

Pour obtenir des détails sur une opération, vous pouvez soit survoler l&apos;enregistrement, soit cliquer dessus pour voir des graphiques supplémentaires.

Lorsque vous cliquez sur un enregistrement, vous pouvez voir le graphique <DNT>**Time consumption by caller**</DNT> . Cela affiche une liste des transactions appelant l&apos;opération sélectionnée, avec des barres de couleur indiquant le pourcentage de temps consommé par chaque appelant. Pour afficher la page <DNT>**Transactions**</DNT> d’une transaction d’appelant, sélectionnez la barre de couleur de cette transaction dans le graphique.

## Afficher toutes les opérations de la base de données [#db\_displaying][#db_displaying]

Pour afficher toutes les opérations de base de données [si disponibles](#availability):

* Au bas de la liste des opérations, sélectionnez le lien <DNT>**Show all database operations table...**</DNT> .
* Pour trier le tableau par n’importe quelle colonne, sélectionnez l’en-tête de la colonne.
* Pour filtrer la liste des opérations, utilisez le champ de recherche en haut du tableau.
* Pour afficher les détails d’une opération spécifique, sélectionnez le nom de l’opération.

## Supprimer la trace [#db\_deleting][#db_deleting]

Pour supprimer définitivement toutes les traces de votre application :

1. Sélectionnez le bouton <DNT>**Delete all traces**</DNT> sous la liste des opérations de base de données.
2. Sélectionnez le prompt de confirmation.

<Callout variant="tip">
  Si vous effectuez un nettoyage général des enregistrements, vous souhaiterez peut-être également supprimer [la trace de transaction](/docs/traces/deleting-transaction-traces) ou [la trace d&apos;erreur](/docs/apm/applications-menu/events/viewing-apm-errors-error-traces). Cela supprimera les données potentiellement sensibles tout en conservant vos autres données d&apos;application (telles que Apdex, les informations de déploiement, etc.).
</Callout>

## Fonctionnalité supplémentaire basée sur la version de l&apos;agent [#availablity]

Des fonctionnalités supplémentaires sur la page <DNT>**Databases**</DNT> sont disponibles avec les versions d&apos;agent suivantes. Pour plus d&apos;informations, voir [Mettre à jour l&apos;agent New Relic](/docs/agents/manage-apm-agents/installation/update-new-relic-agent).

* Go version 1.4 ou supérieure
* Java 3.14.0 ou supérieur
* .NET 4.1.134 ou supérieur
* Node.js 1.17.0 ou supérieur
* PHP 4.19.0 ou supérieur
* Python 2.44.0 ou supérieur
* Ruby 3.11.0 ou supérieur

Ces fonctionnalités incluent :

* Plusieurs bases de données et types de cache
* Étiquettes pour différents types dans la liste des opérations, les graphiques et la requête lente
* Opérations de filtrage et graphiques pour un fournisseur de base de données ou un type de cache spécifique
* Affichage d&apos;un tableau de toutes les opérations de base de données
* Possibilité de visualiser la base de données sur [les cartes de service](/docs/apm/applications-menu/monitoring/service-maps)

De plus, si votre agent APM prend en charge [les détails au niveau de l&apos;hôte et de l&apos;instance](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues), vous pouvez afficher à la fois les détails de la requête et l&apos;instance spécifique où la requête a été exécutée.
