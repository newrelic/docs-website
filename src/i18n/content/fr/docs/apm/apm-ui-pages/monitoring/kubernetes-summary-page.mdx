## Monitorer la déploiement de votre cluster Kubernetes [#monitor-kubernetes]

La page Kubernetes dans APM propose les informations suivantes sur votre application :

<CollapserGroup>
  <Collapser id="deployment-summary" title="Résumé du déploiement">
    Vous pouvez voir le nom de votre cluster , son déploiement et l'espace de nommage dans lequel il s'exécute. Si vous cliquez sur le déploiement, vous obtiendrez des détails à ce sujet dans un panneau coulissant. Dans le panneau coulissant, vous pouvez voir un mini aperçu contenant des informations pertinentes telles que le nombre de logs et le pourcentage d'erreurs du log dans la plage de temps spécifiée.

    <img title="Kubernetes page - Mini overview" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-apm-mini-overview.webp" />

    Dans la liste déroulante <DNT>**Instances**</DNT> , vous pouvez sélectionner une seule instance de pod de votre service.

    <img title="Kubernetes page - Deployment summary" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-header.webp" />

    Si vous avez une application multi-cluster (une application qui s'étend sur plusieurs clusters), la page <DNT>**Kubernetes**</DNT> vous montre un résumé de tous les clusters. Pour isoler un cluster spécifique, sélectionnez-le dans la liste déroulante <DNT>**Cluster**</DNT> . Pour revenir à une vue de tous les clusters, cliquez sur <DNT>**All clusters**</DNT>.

    <img title="Kubernetes page - Show entity preview" alt="Show entity preview" src="/images/apm_screenshot-crop_k9-multi-cluster.webp" />
  </Collapser>

  <Collapser id="current-deployment" title="Déploiement actuel">
    Vous pouvez vérifier le nombre de pods disponibles ou souhaités, les pods manquants et l'âge du déploiement Kubernetes , qui est la date à laquelle le déploiement a été créé pour la première fois.

    <img title="Kubernetes page - Current deployment" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-current-deploy.webp" />
  </Collapser>

  <Collapser id="performance-summary" title="Résumé des performances du déploiement">
    Vous pouvez voir l'utilisation du processeur et de la mémoire, ainsi que le nombre de redémarrages du pod.

    <img width="70%;" title="Kubernetes page - Performance summary" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-performance.webp" />
  </Collapser>

  <Collapser id="pods-containers" title="pod et conteneur">
    Vous pouvez vérifier les détails pod et des conteneurs associés. Si vous cliquez sur le nom du pod ou du conteneur, vous obtiendrez quelques détails à ce sujet dans un panneau coulissant.

    De plus, avec le bouton <DNT>**Logs**</DNT>, vous pouvez voir les logs en contexte de ce pod.

    <img title="Kubernetes page - Pods and Containers" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-pod-containers.webp" />
  </Collapser>

  <Collapser id="activity-stream" title="Flux d'activité">
    Vous pouvez voir le flux d'événements Kubernetes liés à ce déploiement et à son pod.

    <img title="Kubernetes page - Activity stream" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-activity-stream.webp" />
  </Collapser>

  <Collapser id="metrics-comparison" title="Comparaison des mesures">
    Ces widgets vous permettent de comparer une liste statique de métriques liées au pod-conteneur dans le déploiement, ainsi que d'autres métriques telles que le CPU, le réseau, le stockage.

    <img title="Kubernetes page - Metrics Comparison" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-metric-comparasin.webp" />

    Les métriques suivantes sont disponibles :

    * Utilisation du processeur (cœurs)
    * Limitation du processeur (%)
    * Utilisation de la mémoire (octets)
    * Réseau reçu (Kbps)
    * Réseau transmis (Kbps)
    * Erreurs réseau (par seconde)
    * Utilisation du stockage (octets)
    * Utilisation du stockage (%)
    * Nombre de redémarrages du conteneur
    * Débit du service (requests par minute)
    * Service temps de réponse (ms)
    * Taux d'erreur de service (%)
  </Collapser>
</CollapserGroup>

## Liez votre application à Kubernetes [#link-app]

Vous pouvez faire apparaître les métadonnées Kubernetes et les lier à vos agents APM, que vous utilisiez nos agents propriétaires ou OpenTelemetry. <InlinePopover type="apm" />utilise les métadonnées Kubernetes pour afficher les informations de déploiement Kubernetes spécifiques liées à cette application APM.

### instrumentationpropriétaire New Relic [#nr-proprietary]

Lorsque vous [installez l'intégration Kubernetes de New Relic](/install/kubernetes/), l&amp;apos;injection de métadonnées de New Relic entre en jeu automatiquement via le paramètre `nri-metadata-injection` . Le, mis en place par l&amp;apos;`MutatingAdmissionWebhook` Kubernetes instrumentation, se charge de baliser votre pod avec les variables d&amp;apos;environnement nécessaires. À partir de là, les données capturées dans ces variables sont transférées vers vos métriques et entités APM, les enrichissant d&amp;apos;un contexte précieux.

Pour plus d'informations, découvrez comment [lier vos applications à Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-your-applications-kubernetes/).

Les attributs suivants sont requis dans l'entité de service APM pour afficher la page de résumé Kubernetes :

* `k8s.clusterName`
* `k8s.namespaceName`
* `k8s.deploymentName`

### OpenTelemetry instrumentation [#otel-instrumentation]

L'[OpenTelemetry Collector](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro/) propose un processeur d&amp;apos;attributs Kubernetes qui enrichit la télémétrie APM avec des métadonnées Kubernetes .

1. Vous devez définir une variable d’environnement dans votre manifeste de déploiement.

2. Ajustez la configuration du Collector pour récupérer les métadonnées Kubernetes appropriées à l'aide de cette variable d'environnement APM.

De ce fait, toutes les métriques et entités APM incluront les métadonnées Kubernetes grâce au `K8sattributes` Processeur. Pour plus d&amp;apos;informations, découvrez comment [lier vos applications OpenTelemetry à Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes/).

Les attributs suivants sont requis dans l'entité de service APM pour afficher la page de résumé Kubernetes :

* `k8s.cluster.name`
* `k8s.deployment.name`
* `k8s.namespace.name`

<Callout variant="caution">
  La page de résumé de Kubernetes ne fonctionnera pas correctement s'il existe un mélange de différents fournisseurs d'instrumentation (New Relic et OpenTelemetry). Pour que cela fonctionne correctement, Kubernetes et APM doivent être monitorés soit exclusivement via les agents propriétaires de New Relic, soit entièrement via OpenTelemetry.

  **La prise en charge de la configuration de différents fournisseurs d'instrumentation est attendue d'ici le premier trimestre 2025.**
</Callout>
