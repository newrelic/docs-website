La boîte de réception des erreurs est un endroit unique pour détecter, trier et agir de manière proactive sur toutes les erreurs avant qu&apos;elles n&apos;affectent les clients. Recevez des alertes dès qu&apos;une erreur critique ayant un impact sur les clients survient via votre canal de communication préféré.

La boîte de réception des erreurs est intégrée à l&apos;UI <InlinePopover type="apm" />afin que vous puissiez résoudre les erreurs plus rapidement. Les erreurs sont regroupées pour réduire le bruit et la collaboration entre les équipes est facile grâce à une visibilité partagée sur les mêmes données d&apos;erreur.

<Callout variant="important">
  La boîte de réception des erreurs n&apos;est pas disponible dans la région de l&apos;UE.
</Callout>

## Comment trier les erreurs dans APM

Aucune configuration ou charge de travail supplémentaire n&apos;est nécessaire pour trier les erreurs dans APM. Accédez simplement au service APM que vous souhaitez afficher et suivez les étapes ci-dessous :

1. Sur l’écran de présentation UI APM, cliquez sur l’option <DNT>**Errors inbox**</DNT> dans la navigation de gauche dans la section de triage.
2. Cette vue de la boîte de réception des erreurs regroupe les erreurs liées spécifiquement au service APM que vous consultez. Vous pouvez voir tous les détails de l&apos;erreur et également trier l&apos;erreur directement dans l&apos;UI APM.
3. Cliquez sur les détails d&apos;un groupe d&apos;erreurs pour voir la trace des appels, les attributs des logs en contexte et d&apos;autres informations sans accéder à d&apos;autres écrans ou services.
4. Cliquez sur <DNT>**Go to global inbox**</DNT> pour accéder à une expérience plus large de la boîte de réception des erreurs, qui regroupe les erreurs sur l&apos;ensemble stack, y compris les erreurs <InlinePopover type="browser" />et <InlinePopover type="mobile" />. New Relic recherche automatiquement une workload qui inclut le service APM que vous recherchiez ou vous invite à créer une nouvelle workload.

L&apos;utilisation de la boîte de réception des erreurs dans APM vous offre un niveau de commodité supplémentaire, vous permettant de trier les erreurs sans changer d&apos;écran.

## Connecter une boîte de réception à Slack

Une fois connecté à Slack, les groupes d&apos;erreurs nouveaux et réapparus seront envoyés à un canal Slack dans les secondes qui suivent leur apparition. Cela permet à votre équipe d&apos;identifier rapidement toute nouvelle erreur ou régression et de les résoudre rapidement avec des liens directs vers la trace d&apos;appels.

1. Si l&apos;[application New Relic](https://newrelic.slack.com/apps/AP92KQJS3-new-relic?tab=more_info) n&apos;est pas installée sur votre espace de travail Slack, faites-le d&apos;abord.
2. Depuis une boîte de réception, sélectionnez l’icône Paramètres de la boîte de réception (qui ressemble à un engrenage) dans le coin supérieur droit.
3. Basculez le bouton Slack sur <DNT>**on**</DNT> s&apos;il est sur <DNT>**off**</DNT>.
4. Si aucun espace de travail n&apos;est disponible, cliquez sur le bouton « + » pour activer Slack avec une authentification Slack en un clic.
5. Une fois authentifié, vous pourrez sélectionner un espace de travail et un canal spécifique auquel envoyer une notification.
6. Cliquez sur « Tester » pour vous assurer que les messages sont envoyés au bon canal.
