Ce document explique ce que sont les <DNT>**non-web transactions**</DNT> dans APM et comment vous pouvez instrumenter les tâches d&amp;apos;arrière-plan et d&amp;apos;autres activités arbitraires pour qu&amp;apos;elles soient signalées comme des transactions non Web.

## Que sont les transactions non Web ? [#define]

Les deux principales catégories [de transactions](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm) APM sont <DNT>**web**</DNT> et <DNT>**non-web**</DNT>.

* <DNT>**Web transactions**</DNT> ce sont des transactions que nous détectons comme étant initiées par une requête Web ; elles constituent le type de transaction le plus courant signalé par APM.
* <DNT>**Non-web transactions**</DNT> il s&amp;apos;agit de transactions pour lesquelles nous ne détectons pas qu&amp;apos;elles ont été initiées par une requête Web. Cela inclut les processus et les tâches qui ne gèrent pas requests Web, tels que le traitement des messages et les tâches en arrière-plan.

Par exemple, vous pouvez programmer l'exécution périodique d'une tâche sur votre serveur pour effectuer des travaux sur une base de données, envoyer des e-mails, etc. La source n'est pas une requête Web mais quelque chose sur votre serveur.

Certaines transactions non Web provenant de frameworks ou de services pris en charge sont détectées automatiquement par New Relic ; d'autres peuvent être créées à l'aide d'instrumentation personnalisée. Nous séparons intentionnellement votre Web et vos transactions non Web dans l'UI.

## Créer une transaction non Web [#create]

Pour créer une nouvelle transaction non Web, suivez les procédures de votre agent de langue APM.

<CollapserGroup>
  <Collapser id="go" title="Go">
    Suivez les procédures pour [instrumenté une transaction non web](/docs/agents/go-agent/get-started/instrument-go-transactions#go-txn).
  </Collapser>

  <Collapser id="java" title="Java">
    Suivez les procédures pour [instrumenter une transaction](/docs/java/monitoring-java-background-processes) (instrumenté une transaction sans paramètres de requête Web crée une transaction non Web).
  </Collapser>

  <Collapser id="net" title=".NET">
    Par défaut, l'agent APM .NET instrumente les travailleurs IIS ASP. Pour instrumenter d'autres applications .NET (non IIS), telles que des services autonomes (WCF), des applications de console et d'autres services Windows , consultez [applications personnalisées instrumentées](/docs/agents/net-agent/features/instrumenting-custom-applications) et [applications WCF instrumentées](/install/dotnet/?deployment=WCF).

    <Callout variant="important">
      Pour les applications IIS, vous ne pouvez pas modifier la transaction Web en transaction non Web.
    </Callout>
  </Collapser>

  <Collapser id="node" title="Node.js">
    Utilisez [`startBackgroundTransaction()`](/docs/agents/nodejs-agent/supported-features/nodejs-custom-instrumentation#background-txn).
  </Collapser>

  <Collapser id="php" title="PHP">
    Pour marquer une transaction comme non Web : utilisez [`newrelic_background_job`](/docs/agents/php-agent/php-agent-api/newrelic_background_job) et définissez l’indicateur sur `true`.

    L'agent PHP signale tous les scripts exécutés en ligne de commande comme des transactions non Web.
  </Collapser>

  <Collapser id="python" title="Python">
    Voir [l'agentPython transaction non web](/docs/agents/python-agent/supported-features/python-background-tasks).

    L'agent Python prend également en charge les rapports de transactions non Web de [Celery](/docs/agents/python-agent/back-end-services/python-agent-celery) et Gearman.
  </Collapser>

  <Collapser id="ruby" title="Ruby">
    Voir [transactions non-web de l'agent Ruby](/docs/agents/ruby-agent/background-jobs/monitor-ruby-background-processes).
  </Collapser>
</CollapserGroup>

## Voir la transaction non Web [#ui]

Pour afficher les données de transaction non Web dans l'UI de New Relic, accédez au graphique principal sur les pages suivantes, puis sélectionnez l'option <DNT>**Non-web**</DNT> dans la liste déroulante des types de transaction :

* APM : Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; services**</DNT> &amp;gt; (sélectionnez une application) &amp;gt; <DNT>**Summary**</DNT>.
* Transactions : Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; Services**</DNT>. Sélectionnez une application, puis sous la section <DNT>**Monitor**</DNT> , cliquez sur <DNT>**Transactions**</DNT>.
