## Problème

Vous ne voyez pas de trace de transaction dans l&apos;UI APM.

## Solution

Si vous ne voyez pas de [trace de transaction](/docs/apm/transactions/transaction-traces/transaction-traces) dans [APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm), il y a plusieurs raisons possibles.

<CollapserGroup>
  <Collapser id="disabled" title="Le traçage des transactions a été désactivé.">
    Les traces de transaction sont activées par défaut, mais le paramètre peut avoir été désactivé manuellement. Pour résoudre ce problème, modifiez le paramètre pour réactiver les paramètres trace des transactions.
  </Collapser>

  <Collapser id="criteria" title="La transaction ne répond pas aux critères de traçabilité.">
    La transaction ne répond pas aux [critères de traçabilité](/docs/apm/transactions/transaction-traces/transaction-traces#basics). Par exemple:

    * Il répond plus rapidement que le seuil de 4 fois Apdex ou un seuil personnalisé en nombre de secondes.
    * Il n&apos;est pas sélectionné comme la trace la plus lente pendant le cycle de collecte d&apos;une minute.
  </Collapser>

  <Collapser id="app" title="L'activité de l'application n'est pas capturée en tant que transaction.">
    Si vous ne voyez pas la transaction répertoriée sur la [page Transactions](/docs/apm/applications-menu/monitoring/transactions-page) APM, votre agent APM ne capture pas l&apos;activité en tant que transaction. Dans cette situation, vous devez mettre en place [instrumentation personnalisée](/docs/apm/agents/manage-apm-agents/agent-data/custom-instrumentation/) pour monitorer l&apos;activité comme une transaction.
  </Collapser>

  <Collapser id="brackets" title="Le nom de la transaction personnalisée inclut des crochets.">
    Si vous avez utilisé [une instrumentation personnalisée](/docs/apm/agents/manage-apm-agents/agent-data/custom-instrumentation/) pour créer manuellement une transaction, assurez-vous de suivre les règles de dénomination de New Relic.

    <Callout variant="important">
      N&apos;utilisez pas de crochets `[suffix]` à la fin du nom de votre transaction. New Relic supprime automatiquement les crochets du nom. Utilisez plutôt des parenthèses `(suffix)` ou d’autres symboles si nécessaire.
    </Callout>
  </Collapser>

  <Collapser id="no-traces" title="Aucune trace n'apparaît pour aucune transaction.">
    Modifiez temporairement la valeur <DNT>**Threshold**</DNT> en `0.0001` afin que les transactions dépassent définitivement le seuil. Assurez-vous de rétablir ce paramètre d&apos;origine après avoir commencé à recevoir des traces de transaction.
  </Collapser>
</CollapserGroup>
