La page <DNT>**Alerts**</DNT> d&amp;apos;APM affiche un tableau des alertes des deux dernières semaines dans lequel vous pouvez accéder à des informations détaillées, notamment le débit, le pourcentage de taux d&amp;apos;erreur et l&amp;apos;historique. Vous pouvez afficher l&amp;apos;historique des alertes pour une application spécifique ou pour toutes les applications.

## Alertes pour des applications spécifiques [#alert\_procedures][#alert_procedures]

Pour afficher une liste d’alertes pour une application spécifique : accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; Services &amp;gt; (select an app) &amp;gt; Events &amp;gt; Issues &amp;amp; activity**</DNT>.

La page <DNT>**Issues &amp; Activity**</DNT> affiche une liste de problèmes, d&amp;apos;incidents, d&amp;apos;anomalies et d&amp;apos;autopsies, le cas échéant. Utilisez l&amp;apos;une des [fonctionnalitésUI ](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)standard de New Relic pour accéder à plus de détails.

Chaque notification d’alerte comprend l’icône d’alerte et des informations récapitulatives. Pour afficher les détails d’une notification d’alerte sur la page des alertes, sélectionnez son lien. Par exemple:

* Pour afficher les détails d’une alerte spécifique, sélectionnez son nom.
* Pour affiner la liste des alertes, utilisez le <DNT>**Filter**</DNT>.
* Pour afficher toutes les alertes pour toutes vos applications et hôtes, sélectionnez le lien <DNT>**All alerts**</DNT> .
* Pour sélectionner une machine virtuelle Java (JVM) spécifique, le cas échéant, sélectionnez votre choix dans le menu <DNT>**JVMs**</DNT> .
