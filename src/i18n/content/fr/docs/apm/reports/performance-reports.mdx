Nos rapports d&apos;analyse des performances fournissent des vues simples et prédéfinies de la santé et des performances de votre application. Choisissez un rapport ci-dessous pour en savoir plus :

<CollapserGroup>
  <Collapser className="freq-link" id="background" title="Analyse des emplois en arrière-plan">
    Le rapport <DNT>**Background jobs analysis**</DNT> vous aide à analyser les performances des tâches et autres [transactions non Web](/docs/apm/transactions/intro-transactions/monitor-background-processes-other-non-web-transactions) exécutées en production qui utilisent le framework instrumenté par New Relic, tels que `DelayedJob` et `Resque`. Le rapport affiche des informations de comparaison des 24 dernières heures aux 7 derniers jours, y compris le temps passé en débit (requests par minute ou <DNT>**rpm**</DNT>), le temps total passé dans la tâche et le temps moyen pour l&apos;exécuter.

    ## Afficher le rapport dans UI [#procedures]

    Pour afficher le rapport <DNT>**Background jobs analysis**</DNT> de votre application : accédez à l’onglet <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; services &amp;gt; (select an app) &amp;gt; Reports &amp;gt; Performance &amp;gt; Non-web transactions**</DNT> . Le rapport par défaut compare les dernières 24 heures avec la période de 24 heures précédente, il y a 7 jours, et établit les moyennes sur les 7 derniers jours.

  <Collapser className="freq-link" id="capacity" title="Analyse de capacité">
    Le rapport d&apos;analyse de capacité d&apos;APM indique combien d&apos;instances de votre application sont en cours d&apos;exécution et à quel point elles sont occupées. Ces informations sont utiles pour analyser si votre application dispose de suffisamment d&apos;instances pour prendre en charge les charges de requêtes. Vous pouvez ensuite utiliser ces informations pour affiner votre configuration afin d&apos;obtenir des performances optimales.

    Le rapport d&apos;analyse de capacité est uniquement disponible pour les agents de langage [Python](#agent-capacity-analysis) et Ruby.

    ## Applications Python mono ou multithread [#agent-capacity-analysis]

    Pour l&apos;agent Python, les rapports d&apos;analyse de capacité ne fonctionnent que pour les applications traditionnelles à un ou plusieurs threads. Cela est dû au fait que la manière dont New Relic calcule une charge système ne se traduit pas par un système basé sur une co-routine.

    Lors de l&apos;utilisation d&apos;une application traditionnelle monothread ou multithread, les informations métriques générées par l&apos;agent Python sont capturées et signalées uniquement si le composant d&apos;extension C facultatif peut être compilé et installé. Lorsque vous utilisez un système basé sur une co-routine tel que gevent ou eventlet en conjonction avec un serveur WSGI , le rapport d&apos;analyse de capacité n&apos;affichera aucune donnée, à l&apos;exception des redémarrages de l&apos;agent.

  <Collapser className="freq-link" id="scalability" title="Analyse d'évolutivité">
    Le rapport d&apos;analyse d&apos;évolutivité d&apos;APM indique le temps de réponse, le temps de base de données et le temps CPU par rapport à la charge de l&apos;application. Cela vous aide à évaluer les performances et à identifier les exigences matérielles potentielles à mesure que votre application se développe.

    ## Sélectionnez les options pour les périodes de temps [#time]

    Le rapport d&apos;évolutivité dispose d&apos;un [sélecteur d&apos;intervalle de temps](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) qui vous permet de visualiser les données sur différentes périodes de temps. La période minimale recommandée est de 30 minutes, car lors de l&apos;analyse de l&apos;évolutivité d&apos;une application, des plages de temps inférieures ne donnent pas une image complète. Le sélecteur d&apos;intervalle de temps utilise la valeur par défaut des 30 dernières minutes.

    ## Afficher les données pour une instance individuelle [#instances]

    Chaque onglet comprend un sélecteur d&apos;instance qui vous permet de sélectionner une instance spécifique et le graphique et le panneau d&apos;affichage filtreront les données de cette instance.

  <Collapser className="freq-link" id="web-transaction" title="analyse des transactions sur le Web">
    Le rapport d&apos;analyse du Web de transaction d&apos;APM fournit des informations de comparaison pour le Web de transaction, y compris le temps passé dans le débit (requests par minute ou <DNT>**rpm**</DNT>), le temps total dans le Web de transaction, le temps moyen pour l&apos;exécuter et le score [Apdex](/docs/site/apdex-measuring-user-satisfaction) . Vous pouvez comparer les données des 24 dernières heures avec la période de 24 heures précédente, il y a 7 jours et les moyennes des 7 derniers jours.

    Utilisez ces informations pour analyser quelles transactions Web consomment le plus de temps, ont le plus grand nombre d&apos;appels, ont le plus grand nombre d&apos;écarts types, et plus encore. Ces informations peuvent vous aider à identifier les points sur lesquels vous devez peaufiner votre application.

    ## Applications sélectionnées [#featured\_apps][#featured_apps]

    New Relic sélectionne les 20 meilleures applications en fonction du nombre d&apos;appels qui répondent à ces critères :

    * L&apos;application doit avoir un débit d&apos;au moins 25 requests par minute (rpm).
    * L&apos;application doit inclure le Web de transaction.

    Les applications avec monitoring liée aux navigateurs seront prioritaires sur celles qui ne le sont pas, même si le nombre d&apos;appels est inférieur.

## Analysez vos données [#insights]

APM inclut plusieurs rapports dans l&apos;interface utilisateur. Pour collecter, analyser et visualiser des données sur votre logiciel dans d&apos;autres formats, utilisez [le générateur de requêtes](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder).
