<CollapserGroup>
  <Collapser className="freq-link" id="background" title="Analyse des emplois en arrière-plan">
    Le rapport <DNT>**Background jobs analysis**</DNT> vous aide à analyser les performances des tâches et autres [transactions non Web](/docs/apm/transactions/intro-transactions/monitor-background-processes-other-non-web-transactions) exécutées en production qui utilisent le framework instrumenté par New Relic, tels que `DelayedJob` et `Resque`. Le rapport affiche des informations de comparaison des 24 dernières heures aux 7 derniers jours, y compris le temps passé en débit (requests par minute ou <DNT>**rpm**</DNT>), le temps total passé dans la tâche et le temps moyen pour l&apos;exécuter.

Le rapport par défaut compare les dernières 24 heures avec la période de 24 heures précédente, il y a 7 jours, et établit les moyennes sur les 7 derniers jours.

    ## Applications Python mono ou multithread [#agent-capacity-analysis]

    Pour l&apos;agent Python, les rapports d&apos;analyse de capacité ne fonctionnent que pour les applications traditionnelles à un ou plusieurs threads. Cela est dû au fait que la manière dont New Relic calcule une charge système ne se traduit pas par un système basé sur une co-routine.

    Lors de l&apos;utilisation d&apos;une application traditionnelle monothread ou multithread, les informations métriques générées par l&apos;agent Python sont capturées et signalées uniquement si le composant d&apos;extension C facultatif peut être compilé et installé. Lorsque vous utilisez un système basé sur une co-routine tel que gevent ou eventlet en conjonction avec un serveur WSGI , le rapport d&apos;analyse de capacité n&apos;affichera aucune donnée, à l&apos;exception des redémarrages de l&apos;agent.
  </Collapser>

    ### Métriques du serveur d&apos;application [#nrql-app-server]

    Chaque requête de ce groupe rapporte une seule valeur hebdomadaire pour une application, en utilisant `SINCE '<monday>' UNTIL '<next_monday>'` pour définir la portée de la semaine et `appName = '<app_name>'` pour sélectionner l&apos;application.

    **Requests (millions) :** nombre total de requests HTTP gérées pendant la semaine, exprimé en millions.

    ```sql
    SELECT sum(newrelic.timeslice.value[count]) / 1e6 AS 'Requests millions'
    FROM Metric
    WHERE metricTimesliceName = 'HttpDispatcher' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **Temps de réponse (ms) :** temps de réponse moyen en millisecondes sur toutes les transactions HTTP.

    ```sql
    SELECT sum(newrelic.timeslice.value) / sum(newrelic.timeslice.value[count]) * 1000 AS 'Resp time ms'
    FROM Metric
    WHERE metricTimesliceName = 'HttpDispatcher' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **Débit (RPM) :** moyenne des requests par minute sur la semaine.

    ```sql
    SELECT sum(newrelic.timeslice.value[count]) / 10080 AS 'Throughput RPM'
    FROM Metric
    WHERE metricTimesliceName = 'HttpDispatcher' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **Taux d’erreur (%) :** erreurs en pourcentage de toutes les transactions web et non web.

    ```sql
    SELECT filter(sum(newrelic.timeslice.value[count]), WHERE metricTimesliceName = 'Errors/all') /
      (filter(sum(newrelic.timeslice.value[count]), WHERE metricTimesliceName = 'HttpDispatcher') +
       filter(sum(newrelic.timeslice.value[count]), WHERE metricTimesliceName = 'OtherTransaction/all')) * 100 AS 'Error rate'
    FROM Metric
    WHERE metricTimesliceName IN ('Errors/all', 'HttpDispatcher', 'OtherTransaction/all')
      AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **Apdex :** le score [Apdex](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) hebdomadaire de l&apos;application.

    ```sql
    SELECT (sum(newrelic.timeslice.value[count]) + sum(newrelic.timeslice.value[total]) / 2.0) /
      (sum(newrelic.timeslice.value[count]) + sum(newrelic.timeslice.value[total]) + sum(newrelic.timeslice.value[exclusiveTotal])) AS 'Apdex'
    FROM Metric
    WHERE metricTimesliceName = 'Apdex' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **% Satisfait/% Tolérant/% Frustré :** les trois composants Apdex en pourcentages. Cela complète le score Apdex unique ci-dessus en le décomposant en ses catégories de requêtes sous-jacentes.

    ```sql
    SELECT sum(newrelic.timeslice.value[count]) /
      (sum(newrelic.timeslice.value[count]) + sum(newrelic.timeslice.value[total]) + sum(newrelic.timeslice.value[exclusiveTotal])) * 100 AS '% Satisfied',
      sum(newrelic.timeslice.value[total]) /
      (sum(newrelic.timeslice.value[count]) + sum(newrelic.timeslice.value[total]) + sum(newrelic.timeslice.value[exclusiveTotal])) * 100 AS '% Tolerating',
      sum(newrelic.timeslice.value[exclusiveTotal]) /
      (sum(newrelic.timeslice.value[count]) + sum(newrelic.timeslice.value[total]) + sum(newrelic.timeslice.value[exclusiveTotal])) * 100 AS '% Frustrated'
    FROM Metric
    WHERE metricTimesliceName = 'Apdex' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **CPU (%) :** Temps CPU utilisateur moyen pour la semaine, exprimé en pourcentage.

    ```sql
    SELECT sum(newrelic.timeslice.value) / 604800 * 100 AS 'CPU %'
    FROM Metric
    WHERE metricTimesliceName = 'CPU/User Time' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **Mémoire (Mo) :** utilisation moyenne de la mémoire physique en mégaoctets.

    ```sql
    SELECT average(newrelic.timeslice.value) AS 'Memory MB'
    FROM Metric
    WHERE metricTimesliceName = 'Memory/Physical' AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    **DB (%) :** temps passé dans les appels de datastore web en pourcentage du temps total de transaction web.

    ```sql
    SELECT filter(sum(newrelic.timeslice.value), WHERE metricTimesliceName = 'Datastore/allWeb') /
      filter(sum(newrelic.timeslice.value), WHERE metricTimesliceName = 'HttpDispatcher') * 100 AS 'DB %'
    FROM Metric
    WHERE metricTimesliceName IN ('Datastore/allWeb', 'HttpDispatcher') AND appName = '<app_name>'
    SINCE '<monday>' UNTIL '<next_monday>'
    ```

    <Callout title="Note">
      Le pourcentage de base de données représente le ratio entre le temps d&apos;opération de la base de données et le temps de réponse total de la transaction web. Formule : (temps de base de données/temps de réponse total) × 100. Les pourcentages dépassant 100 % indiquent une exécution parallèle ou asynchrone des appels de base de données.
    </Callout>

    ### Modèle de Traffic [#nrql-traffic]

    Ces requêtes alimentent le graphique sparkline quotidien sur deux mois affiché en haut du rapport. Ils utilisent `TIMESERIES 1 day` plutôt que `SINCE 2 months ago`. Il existe deux variantes selon la façon dont l&apos;application est instrumentée : les applications de serveur d&apos;applications sont basées sur `HttpDispatcher` tranches de temps (rpm), et les applications instrumentées par le navigateur utilisent `EndUser` tranches de temps (ppm).

    **Débit — applications de serveur d&apos;applications (rpm) :** taux quotidien de requêtes par minute pour les applications rapportant `HttpDispatcher` tranches de temps.

    ```sql
    SELECT rate(sum(newrelic.timeslice.value), 1 minute) AS 'Throughput rpm'
    FROM Metric
    WHERE metricTimesliceName = 'HttpDispatcher' AND entity.guid = '<APP_GUID>'
    SINCE 2 months ago TIMESERIES 1 day
    ```

    **Débit — applications instrumentées par le navigateur (ppm) :** Taux quotidien de chargements de pages par minute pour les applications rapportant `EndUser` tranches de temps.

    ```sql
    SELECT rate(sum(newrelic.timeslice.value), 1 minute) AS 'Throughput ppm'
    FROM Metric
    WHERE metricTimesliceName = 'EndUser' AND entity.guid = '<APP_GUID>'
    SINCE 2 months ago TIMESERIES 1 day
    ```

    ### Métriques du navigateur de l&apos;utilisateur final [#nrql-enduser]

    Ces requêtes alimentent la section navigateur du rapport pour les applications auxquelles le [monitoring de navigateurs](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring) est associé. Ils renvoient un point par semaine sur les `{N}` semaines les plus récentes à l&apos;aide de `TIMESERIES 1 week`.

    **Temps de réponse de l&apos;utilisateur final (s) :** temps de chargement de page moyen à partir de l&apos;agent du navigateur.

    ```sql
    SELECT average(newrelic.timeslice.value) AS 'End User Resp Time (s)'
    FROM Metric
    WHERE metricTimesliceName = 'EndUser' AND entity.guid = '<APP_GUID>'
    SINCE {N} weeks ago TIMESERIES 1 week
    ```

    **Apdex de l&apos;utilisateur final (chargement de page) :** score Apdex pour les chargements de page de l&apos;utilisateur final.

    ```sql
    SELECT apdex(newrelic.timeslice.value) AS 'End User Apdex'
    FROM Metric
    WHERE metricTimesliceName = 'EndUser/Apdex' AND entity.guid = '<APP_GUID>'
    SINCE {N} weeks ago TIMESERIES 1 week
    ```
