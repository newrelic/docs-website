Une bonne <InlinePopover type="alerts" />aide à informer les individus ou les équipes des changements dans leur système. [Alerts](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) fournit un outil d&amp;apos;alerte unique et coordonné pour tous les produits New Relic. Cela vous permet de gérer [les règles d'alerte](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-policy) et [les conditions](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition) qui se concentrent sur les métriques de [l'entité](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) (telles que les applications ou la clé de transaction dans APM) qui vous intéressent le plus.

Après avoir [configuré une ou plusieurs règles d'alerte](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/) et [défini une condition d'alerte](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions) pour votre [entité de produit New Relic (cible)](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition), vous pouvez afficher ces informations dans l&amp;apos;UI des alertes. Vous pouvez également afficher les informations de règle d&amp;apos;alerte dans l&amp;apos;UI pour les produits New Relic individuels.

## Consultez les politiques et conditions de l'APM [#policy]

Pour afficher la règle d'alerte et les informations sur l'état d'une application ou d'une clé de transaction spécifique directement depuis APM :

1. Effectuez l’une des opérations suivantes :

   * App : Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; (select an app) &amp;gt; Settings &amp;gt; Alert conditions**</DNT>.
   * Clé de transaction : Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; More &amp;gt; Key transactions &amp;gt; (select a key transaction) &amp;gt; Settings &amp;gt; Alert conditions**</DNT>.

2. Depuis la page <DNT>**Alert conditions**</DNT> , utilisez les outils disponibles pour rechercher, trier, afficher ou mettre à jour la condition d&amp;apos;alerte et les politiques associées.

## Voir les événements et activités de l'APM [#activity]

<Callout variant="tip">
  Pour obtenir un aperçu de haut niveau de toutes vos applications et services, utilisez notre [explorateur d'entités](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer).
</Callout>

Pour afficher des informations récapitulatives sur [l'événement](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) et d&amp;apos;autres activités directement depuis APM :

1. Effectuez l’une des opérations suivantes :

   * App : Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>.
   * Clé de transaction : Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; More &amp;gt; Key transactions**</DNT>.

2. À partir de l'index, passez la souris sur l'[état de santé codé par couleur](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) de l&amp;apos;entité, sélectionnez un lien dans la liste <DNT>**Application activity**</DNT> ou sélectionnez une application de navigateur spécifique ou une transaction clé pour afficher des détails supplémentaires.

Par exemple, si un incident d’alerte critique se produit :

* L'indicateur d'état de santé devient rouge sur l'index APM et sur l'application sélectionnée.
* La couleur d’arrière-plan de divers graphiques devient rose.
* La liste <DNT>**Application activity**</DNT> sur l&amp;apos;index APM et sur la page <DNT>**Summary**</DNT> de l&amp;apos;application sélectionnée affiche un incident d&amp;apos;avertissement (jaune) et critique (rouge), selon le cas.

Pour en savoir plus sur l'incident dans les alertes, passez la souris sur ou sélectionnez n'importe quelle zone rose sur un graphique. Pour plus d'informations et d'exemples, voir [Voir l'événement de leurs produits](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products).

## Voir l'incident d'alerte de l'APM [#incidents]

Si une condition d'alerte a [un seuil](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) défini pour un incident d&amp;apos;avertissement (jaune) ou critique (rouge), l&amp;apos;[état de santé](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) codé par couleur d&amp;apos;une entité changera pour indiquer un incident. Vous pouvez visualiser l&amp;apos;incident directement depuis la page de l&amp;apos;application ou la clé de transaction dans APM :

1. Effectuez l’une des opérations suivantes :

   * Application : Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &amp;gt; Alerts &amp;gt; (select an app) &amp;gt; Issues &amp;amp; Activity**</DNT> et cliquez sur l’onglet <DNT>**Incidents**</DNT> .
   * Clé de transaction : Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &amp;gt; Key Transactions &amp;gt; (select a key transaction) &amp;gt; Issues &amp;amp; Activity**</DNT> et cliquez sur l’onglet <DNT>**Incidents**</DNT> .

2. À partir de l'index <DNT>**Incidents**</DNT> , utilisez les outils disponibles pour rechercher, trier ou afficher l&amp;apos;incident pour l&amp;apos;application ou la clé de transaction sélectionnée.

3. Facultatif : Pour afficher ou mettre à jour la règle d'alerte, les conditions ou le seuil ayant déclenché l'incident, sélectionnez le lien correspondant.

<Callout variant="tip">
  Vous pouvez également afficher les avertissements et les incidents critiques à partir de l'UI des alertes pour [un incident spécifique](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/explore-incident-history) ou pour [un incident et un événement sur tous les produits](/docs/alerts/new-relic-alerts-beta/reviewing-events/review-events-across-products). Les alertes envoient [une notification d'alerte](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts) uniquement pour les incidents critiques (rouges).
</Callout>
