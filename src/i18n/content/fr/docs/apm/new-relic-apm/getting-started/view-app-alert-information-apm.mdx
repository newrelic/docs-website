De bons <InlinePopover type="alerts" />aident à notifier les individus ou les équipes des changements dans leurs systèmes. [Alertes](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) fournit un outil d&apos;alerte unique et coordonné pour l&apos;ensemble des produits New Relic. Cela vous permet de gérer les [politiques d&apos;alerte](/docs/alerts/overview#alert-policy) et les [conditions](/docs/alerts/overview#alert-condition) qui se concentrent sur les métriques des [entités](/docs/alerts/overview#alert-entity) (telles que les applications ou les transactions clés dans APM) qui vous importent le plus.

Après avoir [configuré une ou plusieurs politiques d&apos;alerte](/docs/alerts/organize-alerts/create-edit-or-find-alert-policy/) et [défini des conditions d&apos;alerte](/docs/alerts/create-alert/create-alert-condition/alert-conditions) pour vos [entités de produit (cibles)](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition) New Relic, vous pouvez consulter ces informations dans l&apos;interface des alertes. Vous pouvez également consulter les informations sur les politiques d&apos;alerte dans l&apos;interface utilisateur pour les produits New Relic individuels.

## Consultez les politiques et conditions de l&apos;APM [#policy]

Pour afficher la règle d&apos;alerte et les informations sur l&apos;état d&apos;une application ou d&apos;une clé de transaction spécifique directement depuis APM :

1. Effectuez l’une des opérations suivantes :

   * App : Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; (select an app) &gt; Settings &gt; Alert conditions**</DNT>.
   * Clé de transaction : Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; More &gt; Key transactions &gt; (select a key transaction) &gt; Settings &gt; Alert conditions**</DNT>.

2. Depuis la page <DNT>**Alert conditions**</DNT> , utilisez les outils disponibles pour rechercher, trier, afficher ou mettre à jour la condition d&apos;alerte et les politiques associées.

## Voir les événements et activités de l&apos;APM [#activity]

<Callout variant="tip">
  Pour obtenir un aperçu de haut niveau de toutes vos applications et services, utilisez notre [explorateur d&apos;entités](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer).
</Callout>

Pour consulter des informations récapitulatives sur les [événements](/docs/alerts/overview#event) et d&apos;autres activités directement depuis l&apos;APM :

1. Effectuez l’une des opérations suivantes :

   * App : Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>.
   * Clé de transaction : Aller à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; More &gt; Key transactions**</DNT>.

2. Depuis l&apos;index, survolez le [statut de santé codé par couleur](/docs/alerts/overview#health-status) de l&apos;entité, sélectionnez un lien dans la liste <DNT>**Application activity**</DNT> ou sélectionnez une application de navigateur ou une transaction clé spécifique pour afficher des détails supplémentaires.

Par exemple, si un événement d&apos;alerte critique se produit :

* L&apos;indicateur d&apos;état de santé devient rouge sur l&apos;index APM et sur l&apos;application sélectionnée.
* La couleur d’arrière-plan de divers graphiques devient rose.
* La liste <DNT>**Application activity**</DNT> sur l&apos;index APM et sur la page <DNT>**Summary**</DNT> de l&apos;application sélectionnée affiche les événements d&apos;alerte Avertissement (jaune) et Critique (rouge), le cas échéant.

Pour en savoir plus sur l&apos;événement d&apos;alerte dans les alertes, survolez ou sélectionnez n&apos;importe quelle zone rose sur un graphique. Pour plus d&apos;informations et d&apos;exemples, consultez [Voir les événements de leurs produits](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products).

## Afficher les événements d&apos;alerte de l&apos;APM [#alertevents]

Si une condition d&apos;alerte a des [seuils](/docs/alerts/overview#alert-threshold) définis pour des événements d&apos;alerte Avertissement (jaune) ou Critique (rouge), le [statut de santé](/docs/alerts/overview#health-status) à code couleur d&apos;une entité changera pour indiquer un événement d&apos;alerte. Vous pouvez consulter les événements d&apos;alerte directement depuis la page de l&apos;application ou de la transaction clé dans APM :

1. Effectuez l’une des opérations suivantes :

   * Application : Accédez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; (select an app) &gt; Issues &amp; Activity**</DNT> et cliquez sur l’onglet <DNT>**Alert events**</DNT> .
   * Clé de transaction : Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Key Transactions &gt; (select a key transaction) &gt; Issues &amp; Activity**</DNT> et cliquez sur l’onglet <DNT>**Alert events**</DNT> .

2. À partir de l&apos;index <DNT>**Alert events**</DNT>, utilisez les outils disponibles pour rechercher, trier ou afficher les événements d&apos;alerte pour l&apos;application ou la transaction clé sélectionnée.

3. Facultatif : pour consulter ou mettre à jour les politiques d&apos;alerte, les conditions ou les seuils qui ont déclenché les événements d&apos;alerte, sélectionnez le lien correspondant.

<Callout variant="tip">
  Vous pouvez également consulter les événements d&apos;alerte Avertissement et Critique depuis l&apos;interface des alertes pour des [événements d&apos;alerte spécifiques](/docs/alerts/alert-event-management/view-alert-event-details) ou pour des [événements d&apos;alerte et des événements sur l&apos;ensemble des produits](/docs/alerts/alert-event-management/view-alert-event-details). Alerts envoie des [notifications d&apos;alerte](/docs/alerts/get-notified/destinations) uniquement pour les événements d&apos;alerte critiques (rouges).
</Callout>
