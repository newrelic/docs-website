Vous pouvez ajouter et supprimer des hôtes pour votre application. Vous ne pouvez pas renommer un hôte existant, mais vous pouvez créer une application nommée d&apos;après un hôte ou cluster. Vous pouvez également [attribuer une étiquette cosmétique à l&apos;hôte](#display_name), pour différencier les hôtes sans modifier le paramètre nom d&amp;apos;hôte sous-jacent.

## Ajouter des hôtes [#adding\_hosts][#adding_hosts]

Pour ajouter un hôte à monitoring, installez l&apos;agent et démarrez votre application. L&apos;hôte exécutant l&apos;application sera identifié auprès de New Relic et apparaîtra automatiquement sur le site après quelques minutes.

## Renommer les hôtes [#renaming\_hosts][#renaming_hosts]

Il n&apos;est pas possible de changer le nom d&apos;hôte. Cependant, vous pouvez [créer une application nommée d&apos;après un hôte ou cluster](/docs/site/renaming-applications). Par exemple, si les hôtes `ey01-s00057` et `ey01-s00058` font tous deux partie d&amp;apos;un [niveau](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#tier) de production, vous pouvez créer une application appelée `App - Production` et faire en sorte que ces deux hôtes lui rendent compte.

## Modifier le nom d&apos;affichage des hôtes [#display\_name][#display_name]

Ce nom d’affichage n’affecte pas le lien entre une application et son hôte parent. Le nom d’affichage apparaît dans l&apos;UI APM pour décorer le `host:port` nom attribué automatiquement.

## Supprimer les hôtes [#removing\_hosts][#removing_hosts]

Pour supprimer un hôte, utilisez l’une des options suivantes :

* [Désinstaller](/docs/agents/manage-apm-agents/installation/uninstall-agent) l&amp;apos;agent.
* Désactivez-le en modifiant ou en supprimant le fichier de configuration de l&apos;agent. Assurez-vous que toutes les instances d&apos;hôte application (Passenger, Jetty, etc.) ont été entièrement redémarrées après la désactivation de l&apos;agent.

Parfois, des processus de serveur d&apos;application malveillants continuent de signaler des données. Dans cette situation, [l&apos;indicateur d&apos;état de santé de votre application](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#health-status) apparaîtra en vert, même si vous avez arrêté l&amp;apos;agent et vérifié qu&amp;apos;aucune donnée n&amp;apos;est signalée. Vérifiez si votre serveur Web contient des processus parasites ou obtenez de l&amp;apos;aide sur [support.newrelic.com](https://support.newrelic.com "Le lien s'ouvre dans une nouvelle fenêtre").

Une fois que vous avez désinstallé un agent APM d&apos;une instance d&apos;hôte, il peut falloir jusqu&apos;à 75 minutes pour que l&apos;ancien hôte disparaisse de la vue APM. Si vous avez supprimé un hôte en désinstallant l&apos;agent d&apos;infrastructure, sa disparition peut prendre jusqu&apos;à 24 heures.
