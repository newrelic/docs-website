## Problème [#problem]

Vous voyez des transactions dans l&apos;UI APM nommées `Middleware/Rack/ActionDispatch::Routing::RouteSet#call`, mais vous ne savez pas de quoi il s&amp;apos;agit.

## Cause [#cause]

Les transactions portant le nom `Middleware/Rack/ActionDispatch::Routing::RouteSet#call` représentent un itinéraire dans votre application Rails qui n&amp;apos;a pas été instrumenté. Étant donné que le middleware Rails est instrumenté, New Relic peut capturer le temps passé sur la transaction, mais ne peut fournir aucun détail au-delà.

Il existe diverses situations pouvant conduire à des transactions nommées `Middleware/Rack/ActionDispatch::Routing::RouteSet#call`. Nous avons fourni deux solutions pour les causes les plus courantes de ce type de transaction.

      Si un itinéraire de votre application passe par un middleware instrumenté vers une gem non instrumentée, vous devez ajouter une instrumentation personnalisée à cette gem pour voir des données plus spécifiques sur la transaction. Pour ajouter une instrumentation personnalisée, nous vous recommandons de lire notre [documentation sur l&apos;instrumentation personnalisée Ruby](/docs/agents/ruby-agent/installation-configuration/ruby-custom-instrumentation).

      L&apos;ajout d&apos;une instrumentation personnalisée à votre application est le meilleur moyen d&apos;obtenir plus de détails sur ces transactions.
    </TabsPageItem>

    <TabsPageItem id="route-controller">
      L&apos;agent Ruby contrôle les contrôleurs d&apos;instruments en insérant instrumentation dans `ActionController::Base`. Cependant, si un itinéraire passe par un contrôleur qui n&amp;apos;hérite pas de `ActionController::Base`, ce contrôleur a besoin d&amp;apos;une instrumentation. instrumentez directement votre contrôleur en ajoutant ceci à votre classe de contrôleur :

      ```ruby
      include NewRelic::Agent::Instrumentation::ControllerInstrumentation
      ```
    </TabsPageItem>
  </TabsPages>
