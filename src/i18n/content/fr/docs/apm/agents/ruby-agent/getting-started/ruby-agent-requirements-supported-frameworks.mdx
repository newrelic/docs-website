Avant d&apos; [installer l&apos;agent Ruby de New Relic](/docs/agents/ruby-agent/installation-and-configuration/ruby-agent-installation), assurez-vous que votre système répond à ces exigences en matière de système d&apos;exploitation compatible, de protocoles de sécurité, de versions Ruby et de framework/bibliothèque pris en charge.

Si vous n&apos;en avez pas déjà un, commencez par [créer un compte New Relic](https://newrelic.com/signup). C&apos;est gratuit, pour toujours.

## système d&apos;exploitation [#os]

L&apos;agent Ruby prend en charge les systèmes d&apos;exploitation de type UNIX tels que Linux, Solaris, FreeBSD et macOS. Windows n&apos;est pas officiellement pris en charge pour l&apos;environnement de production mais peut fonctionner pour le développement.

## Exigences de sécurité [#security]

* **Collecte de données :** en tant que [mesure de sécurité standard pour la collecte de données](/docs/accounts-partnerships/accounts/security/data-security), votre serveur d’applications doit prendre en charge SHA-2 (256 bits) pour la communication SSL avec New Relic. SHA-1 n&apos;est pas pris en charge.
* **TLS :** New Relic nécessite TLS 1.2 ou supérieur pour toutes les connexions entrantes de l&apos;agent.

## Remarques sur les versions prises en charge [#notes\_on\_supported\_versions][#notes_on_supported_versions]

* L&apos;agent peut être compatible avec une gamme plus large de versions (anciennes ou plus récentes) que celles explicitement répertoriées. Cependant, les versions répertoriées reçoivent les tests les plus rigoureux.
* Pour obtenir les informations les plus récentes sur les versions nouvellement prises en charge ou les dépréciations, reportez-vous toujours aux [notes de sortie de l&apos;agentRuby ](/docs/release-notes/agent-release-notes/ruby-release-notes).
* Si une bibliothèque n&apos;est pas répertoriée, elle peut toujours être instrumentée si elle s&apos;appuie sur une technologie sous-jacente qui *est* instrumentée (par exemple, un framework construit sur Rack, ou un ORM utilisant un adaptateur de base de données standard). Pour les bibliothèques critiques non répertoriées, vous pouvez explorer [l&apos;instrumentation personnalisée](/docs/apm/agents/ruby-agent/api-guides/ruby-custom-instrumentation) ou [soumettre une demande de fonctionnalités](https://github.com/newrelic/newrelic-ruby-agent/issues/new?template=feature_request.md).

Si vous avez des questions sur des versions ou des bibliothèques spécifiques non couvertes ici, veuillez visiter notre [forum communautaire Explorers Hub](https://forum.newrelic.com/).

## Autres logiciels APM [#other\_monitoring\_software][#other_monitoring_software]

Si votre application utilise d&apos;autres logiciels monitoring des performances des applications (APM) en plus de notre agent, nous ne pouvons pas garantir que notre agent fonctionnera correctement et nous ne pouvons pas offrir de support technique. Pour plus d&apos;informations, consultez [Erreurs lors de l&apos;utilisation d&apos;autres logiciels monitoring ](/docs/apm/new-relic-apm/troubleshooting/errors-while-using-new-relic-apm-alongside-other-apm-software).
