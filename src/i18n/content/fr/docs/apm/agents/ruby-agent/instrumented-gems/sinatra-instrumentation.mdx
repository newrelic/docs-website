L&apos;agent Ruby de New Relic fonctionne avec [Sinatra](http://www.sinatrarb.com/) 1.2.x ou supérieur. Dans l&amp;apos;UI de New Relic, les actions de Sinatra semblent similaires aux actions du contrôleur. Les actions dans l’ UI correspondent à l’expression de modèle utilisée pour les faire correspondre. Les opérations HTTP ne sont pas distinguées. Les matchs multiples sont tous suivis comme des actions distinctes.

## Bien démarrer

Pour configurer Sinatra :

1. Installez le dernier gem `newrelic_rpm` .

2. Dans votre application Sinatra, juste en dessous de la directive Sinatra `require` , ajoutez :

   ```ruby
   require 'newrelic_rpm'
   ```

3. Définissez `RACK_ENV` sur l&amp;apos;environnement correspondant aux définitions configuration dans votre fichier `newrelic.yml` ; par exemple, développement, simulation, production, etc.

<DNT>**Note:**</DNT> le mode développeurs ne fonctionne qu&amp;apos;avec Rails.

## Installation du middleware

Versions de l&apos;agent Ruby antérieures à 3.6.3 ajout manuel requis de middlewares New Relic pour activer certaines fonctionnalités, telles que le traçage inter-applications et <InlinePopover type="browser" />. Pour plus d&amp;apos;informations sur les middlewares requis, consultez [Middlewares Rack](/docs/ruby/rack-middlewares) ou effectuez une mise à niveau vers l&amp;apos;agent Ruby le plus récent.

## Ignorer les itinéraires

Versions des agents Ruby 3.6.3 ou une version supérieure prend en charge l&apos;ignorance de certaines routes, similaire à ce qui était auparavant disponible dans l&apos;instrumentation du contrôleur Rails. Pour spécifier ces valeurs, utilisez le même style de routes que celui que vous utilisez pour définir votre application Sinatra. Par exemple, pour ignorer une route `ping` dans une application Sinatra, incluez le code suivant dans l&amp;apos;application :

```ruby
newrelic_ignore '/ping'

    get '/ping' do
      # ...
    end
```

Si vous souhaitez qu&apos;une application entière soit ignorée (par exemple, dans une application montée), appelez `newrelic_ignore` sans paramètres :

```ruby
newrelic_ignore
```

De plus, `newrelic_ignore_apdex` et `newrelic_ignore_enduser` sont pris en charge.

* L&apos;appel `newrelic_ignore_apdex` exclura un itinéraire donné de la prise en compte dans les calculs Apdex globaux.
* L&apos;appel `newrelic_ignore_enduser` empêchera l&amp;apos;injection automatique du JavaScript de synchronisation de chargement de la page lorsqu&amp;apos;un itinéraire est rendu.

## Padrino

[Padrino](http://www.padrinorb.com/ "Le lien s'ouvre dans une nouvelle fenêtre") est un framework construit sur Sinatra. À partir de la version 3.6.3 de l&amp;apos;agent Ruby, L&amp;apos;instrumentation Sinatra de New Relic fonctionne avec les versions Padrino 0.10.x ou supérieures.

## Fusil de chasse (non pris en charge) [#shotgun]

L&apos;agent Ruby New Relic ne fonctionnera pas avec [le fusil à pompe](http://rubygems.org/gems/shotgun "Le lien s'ouvre dans une nouvelle fenêtre"). Il ne peut pas établir de connexion avant la fin du processus du répartiteur. Ni le mode développeurs ni le mode moniteur ne fonctionneront.
