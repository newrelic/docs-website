Un logging de débogage détaillé peut aider à dépanner votre installation d&apos;agent Ruby standard.

<Callout variant="important">
  Si vous utilisez Heroku, consultez [l&apos;agent Ruby et Heroku](/docs/agents/ruby-agent/miscellaneous/ruby-agent-heroku#logging).
</Callout>

## Génération du fichier de log [#generating]

Pour générer un Ruby fichier New Relic for log détaillé :

1. Ouvrez votre fichier `newrelic.yml` dans un éditeur.

2. Remplacez [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level) par `debug` et enregistrez le fichier. Assurez-vous de ne pas modifier l’indentation.

   <Callout variant="important">
     `debug` est un niveau de logging très détaillé. Assurez-vous de revenir à votre `log_level` précédent après avoir généré le log de niveau de débogage.
   </Callout>

3. Redémarrez votre application Web.

4. Générez du trafic vers votre application et exécutez-la pendant environ cinq minutes.

5. Examinez votre fichier de log pour détecter les erreurs. Par défaut, votre fichier de log est situé dans le dossier log de votre application et est nommé `newrelic_agent.log`.

6. Si vous envoyez votre fichier de log au support New Relic, joignez le fichier de log à votre ticket d&apos;assistance, avec `newrelic.yml`, votre Gemfile et `Gemfile.lock`.

7. Modifiez à nouveau `newrelic.yml` et remplacez [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level) par le paramètre précédent. Sauvegarder le fichier.
