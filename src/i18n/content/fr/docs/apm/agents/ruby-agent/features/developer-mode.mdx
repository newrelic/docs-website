<Callout variant="caution">
  le mode développeurs est <DNT>**deprecated**</DNT> et n&amp;apos;est plus pris en charge à partir de [la version 4.1.0 de l'agentRuby ](/docs/release-notes/agent-release-notes/ruby-release-notes/ruby-agent-410333).
</Callout>

L'agent Ruby New Relic inclut le mode développeurs, une UI intégrée permettant d'examiner les détails des transactions Web dans votre application. Avec le mode développeurs, vous pouvez examiner les appels de bibliothèque et de base de données effectués par les actions du contrôleur lorsqu'ils sont invoqués sur l'Rails instance.

Le mode développeurs gratuit vous offre une visibilité approfondie des transactions similaire à celle que vous obtenez avec la fonctionnalité [de trace de transaction](/docs/apm/transactions/transaction-traces/introduction-transaction-traces) de New Relic. Cependant, cela ne fonctionne que dans une seule instance Rails et la surcharge est bien plus élevée que la version de production. Il n&amp;apos;est pas recommandé de l&amp;apos;utiliser en dehors du sandbox de bureau d&amp;apos;un développeur.

## Considérations de sécurité [#security]

Le mode développeurs de l'agent Ruby , qui est activé par défaut lorsque le `RAILS_ENV == "development"`, ne communique en aucune façon avec le collecteur New Relic. Les graphiques à secteurs sont rendus à l&amp;apos;aide de Google Charts, ce qui implique une publication HTTP de données agrégées.

<Callout variant="tip">
  Le mode Production de l'agent Ruby, qui est activé par défaut lorsque `RAILS_ENV == "production"` et `RAILS_ENV == "staging"`, communique avec le site New Relic. Pour plus d&amp;apos;informations, voir [Sécurité](/docs/subscriptions/security).
</Callout>

## installation

Pour installer le mode développeurs, [installez l'agent Ruby ](/docs/ruby/ruby-agent-installation)en tant que plugin ou gem. Vous devrez peut-être également définir [`developer_mode`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#developer_mode) sur vrai dans votre <DNT>**newrelic.yml**</DNT> pour le `RAILS_ENV` dans lequel vous travaillez. le mode développeurs est activé par défaut uniquement lorsque `RAILS_ENV` = `development`.

Bien qu'un fichier <DNT>**newrelic.yml**</DNT> soit créé dans le cadre de l&amp;apos;installation, il n&amp;apos;est pas nécessaire d&amp;apos;avoir un <InlinePopover type="licenseKey" />valide.

## Problèmes connus [#known\_issues][#known_issues]

Les problèmes connus avec le mode développeurs incluent :

<CollapserGroup>
  <Collapser id="rails-23" title="Nécessite Rails 2.3 ou supérieur">
    le mode développeurs dans les agents récents ne fonctionne que sur Rails 2.3 ou supérieur. Il est basé sur un rack, et les versions antérieures du framework n'intégraient pas de rack.

    L'édition développeurs installe un middleware dans votre application qui répond à toute URL précédée de `/newrelic`; par exemple, `/newrelic`, `/newrelic/files/images/foo.png`, etc.

    Dans certains cas, l’URL `/newrelic` est capturée par un middleware antérieur. Dans cette situation, désactivez la route du mode développeurs et incluez vous-même le middleware :

    Au début du fichier, dans `config.ru`:

    ```ruby
    require 'newrelic_rpm'
    require 'new_relic/rack/developer_mode'
    use NewRelic::Rack::DeveloperMode
    ```
  </Collapser>

  <Collapser id="pow-compatibility" title="Incompatible avec Pow">
    Le mode développeurs n'est pas pris en charge par [Pow](http://pow.cx/), un serveur Rack sans configuration pour Mac OS X.
  </Collapser>
</CollapserGroup>

## Utilisation du mode développeurs [#using\_devmode][#using_devmode]

Si vous exécutez votre application sur votre bureau en utilisant thin, WEBrick ou quelque chose de similaire, ouvrez l'URL `/newrelic` sur votre serveur pour voir l&amp;apos;interface utilisateur du mode développeurs.

<Callout variant="caution">
  Si vous utilisez Passenger, Unicorn ou d'autres serveurs d'applications de type fork sur votre bureau, vous remarquerez peut-être un comportement étrange avec la disparition des données si votre serveur d'applications lance plusieurs instances. Reportez-vous aux <DNT>**Troubleshooting**</DNT> conseils qui suivent.
</Callout>

## Dépannage

Conseils de dépannage pour le mode développeurs :

<CollapserGroup>
  <Collapser id="disappearing-request" title="requestsqui disparaissent">
    Si vous rencontrez ce problème avec Unicorn ou d'autres répartiteurs multi-travailleurs, essayez de limiter le nombre de travailleurs à 1.
  </Collapser>

  <Collapser id="undefined-method-rack-request" title="Chemin de méthode non défini pour Rack::Request">
    Certains utilisateurs peuvent voir une méthode non définie `path` pour `#<Rack::Request>`. Si vous utilisez la version 2.3.2 de Rails, vous rencontrez probablement un problème avec la manière dont la dépendance Rack a été définie dans le package Rails. Effectuez une mise à niveau vers la version 2.3.3 (ou supérieure) ou définissez la version Rack 1.0.1 comme dépendance dans votre Gemfile.
  </Collapser>

  <Collapser id="activerecord-reloads" title="Rechargements d'ActiveRecord">
    Le mode développeurs recharge les définitions de méthodes ActiveRecord à chaque rechargement et signale le temps passé à définir les méthodes (comme `define_attribute_method)`. Cela n’arrive pas en production. Pour obtenir des résultats réels à comparer, ajoutez les éléments suivants à l&amp;apos;environnement de développement :

    ```rb
    config.cache_classes = true
    ```
  </Collapser>
</CollapserGroup>
