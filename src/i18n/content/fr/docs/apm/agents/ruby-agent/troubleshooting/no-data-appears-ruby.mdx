## Problème

Après [avoir installé l&apos;agent Ruby New Relic ](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation), généré du trafic et attendu cinq minutes, aucune donnée n&apos;apparaît dans votre UI New Relic.

## Solution

Pour résoudre les problèmes de données manquantes :

1. Vérifiez votre fichier de configuration :

   * Assurez-vous que votre fichier de configuration est installé à `config/newrelic.yml` dans votre répertoire d&apos;application.
   * Assurez-vous que votre fichier de configuration est un YML valide.
   * Pour télécharger une copie propre de ce fichier avec votre <InlinePopover type="licenseKey" />pré-rempli, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; (user menu) &amp;gt; Account settings &amp;gt; Download a clean configuration file**</DNT>.

2. Redémarrez votre serveur Web.

3. Utilisez [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) pour essayer d’identifier automatiquement le problème.

4. Examinez le fichier log de l&apos;agent à `log/newrelic_agent.log` pour détecter d&apos;éventuelles erreurs ou avertissements qui s&apos;affichent lors du redémarrage des processus de votre serveur d&apos;applications. Si ce fichier log n&apos;est pas présent ou n&apos;est pas en cours d&apos;écriture, voir [Aucun fichier log](/docs/agents/ruby-agent/troubleshooting/no-log-file-ruby).

5. Recherchez dans le log de l’agent New Relic une entrée contenant `INFO : Reporting to:` suivi d’une URL. Assurez-vous que l&apos;URL correspond au compte et à l&apos;application que vous visualisez dans l&apos;UI de New Relic.

6. Assurez-vous que l&apos;agent New Relic est configuré pour démarrer dans l&apos;environnement dans lequel vous le déployez. Pour plus d’informations, voir [Contrôle du démarrage de l’agent Ruby](/docs/agents/ruby-agent/troubleshooting/controlling-when-ruby-agent-starts).

7. Si aucune erreur n&apos;est évidente au niveau `info` , [générez un log de débogage détaillé](/docs/agents/ruby-agent/troubleshooting/generating-logs-troubleshooting-ruby).

8. Essayez [de mettre à jour vers la dernière version](/docs/agents/ruby-agent/installation-configuration/updating-deprecated-ruby-agents#update) de la gem <DNT>**newrelic\_rpm**</DNT> .

9. Si vous utilisez autre chose que Rails, définissez `RACK_ENV` sur un environnement apparaissant dans le fichier `newrelic.yml` , tel que développement ou production.
