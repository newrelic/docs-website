L&apos;agent Ruby instrumente automatiquement les middlewares [Rack](http://rack.github.io/) . Si vous n&amp;apos;êtes pas familier avec les bases des middlewares Rack, consultez les [ressources liées au guide Rails on Rack](http://guides.rubyonrails.org/rails_on_rack.html#resources). De plus, l&amp;apos;agent Ruby fournit certaines fonctionnalités via les middlewares Rack :

* [traces distribuées](/docs/distributed-tracing/enable-configure/language-agents-enable-distributed-tracing/)
* auto-instrumentation pour[<InlinePopover type="browser" />](/docs/agents/ruby-agent/features/page-load-timing-ruby)

New Relic installe automatiquement ces middlewares pour Rails et Sinatra.

## Rack instrumentation [#instrumentation]

Les deux manières les plus courantes de configurer Rack les middlewares sont `Rack::Builder` l&amp;apos;API (le plus souvent à partir <DNT>**config.ru**</DNT> de) et Rails middleware stack configuration:

<CollapserGroup>
  <Collapser id="rack_builder" title="Rack::Builder">
    Les middlewares de votre fichier <DNT>**config.ru**</DNT> sont configurés à l&amp;apos;aide de `Rack::Builder`. Pour que l&amp;apos;agent Ruby instrumente les intergiciels de `Rack::Builder`, votre application doit exécuter la version 1.1.0 ou supérieur de la gem `rack` . Il s’agit de l’utilisation la plus courante des middlewares avec des applications Sinatra ou pure-rack.
  </Collapser>

  <Collapser id="rails_middlewares" title="Intergiciels Rails">
    Rails utilise sa propre classe (`ActionDispatch::MiddlewareStack`) au lieu de `Rack::Builder` pour configurer les middlewares. Même si vous n&amp;apos;avez pas explicitement ajouté de middlewares à votre application Rails, de nombreux composants de Rails lui-même sont implémentés en tant que middleware, de sorte que les données du middleware apparaîtront par défaut.

    L&apos;agent Ruby instrumente automatiquement les middlewares ajoutés via `ActionDispatch::MiddlewareStack` sur <DNT>**Rails 3.0 or higher**</DNT>. Pour plus d&amp;apos;informations sur la configuration des middlewares avec Rails, consultez le [guide Ruby on Rails](http://guides.rubyonrails.org/rails_on_rack.html).
  </Collapser>
</CollapserGroup>

## Désactiver Rack instrumentation [#disabling]

Si vous ne souhaitez pas instrumenter Rack les middlewares , vous pouvez désactiver Rack des middlewares instrumentation avec le paramètre [`disable_middleware_instrumentation`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration) . Vous pouvez également [ignorer des transactions spécifiques](/docs/agents/ruby-agent/installation-configuration/ignoring-specific-transactions).

## Installation manuelle des intergiciels d&apos;agent Ruby [#manual]

Ruby L&apos;implémentation de l&apos;agent de la fonctionnalité de traçage inter-applications [de New Relic utilise](/docs/apm/traces/cross-application-traces/cross-application-tracing) Rack middleware instrumentation pour lire et écrire les en-têtes HTTP nécessaires pour transmettre des informations entre les applications de monitoring. Si vous utilisez Sinatra, avez désactivé instrumentation middleware comme décrit ci-dessus et souhaitez utiliser le traçage inter-applications, vous devez ajouter manuellement le middleware `NewRelic::Rack::AgentHooks` à votre stack middleware.

<Callout variant="important">
  À partir de la version 8.0.0, le tracing inter-applications est obsolète au profit du [tracing distribué](/docs/distributed-tracing/enable-configure/language-agents-enable-distributed-tracing/). Lorsque cette option est activée, le tracing distribué est automatiquement configuré pour toutes les applications basées sur un rack sans nécessiter de middleware supplémentaire.

  Si vous souhaitez continuer à utiliser le traçage inter-applications, vous devrez [mettre à jour votre configuration](/docs/agents/ruby-agent/features/cross-application-tracing-ruby#configuration).
</Callout>

## manuelle Rack instrumentation [#manual\_instrumentation][#manual_instrumentation]

Les versions antérieures de l&apos;agent Ruby prenaient en charge les middlewares Rack instrumentés manuellement via le module `NewRelic::Agent::Instrumentation::Rack`. Cette instrumentation a été supprimée et son fonctionnement n&amp;apos;est pas garanti dans la version 4.0.0 ou supérieure de l&amp;apos;agent Ruby. Cette forme d&amp;apos;instrumentation manuelle est inutile avec l&amp;apos;instrumentation middleware automatique. Nous vous recommandons de supprimer les références à ce module de votre code après la mise à niveau vers la version 3.9.0 ou supérieure.
