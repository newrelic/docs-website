Lisez les conseils de dépannage suivants pour les problèmes courants lors de l’utilisation de l’agent Ruby et de Passenger.

<CollapserGroup>
  <Collapser id="agent" title="Redémarrage de l'agent">
    Si vous ne faites que modifier le fichier passager `tmp/restart.txt` , il ne récupérera pas le nouvel agent. Lorsque vous mettez à jour l&amp;apos;agent et redémarrez Passenger, vous devez effectuer un redémarrage matériel d&amp;apos;Apache pour que la modification prenne effet.
  </Collapser>

  <Collapser id="logs" title="Fichier de logmanquant">
    Pour Passenger, le fichier de log est `newrelic_agent.passenger.log`. Recommandation : pour l&amp;apos;utilisateur Apache , fournissez des autorisations d&amp;apos;écriture pour l&amp;apos;ensemble du répertoire `/log`.
  </Collapser>

  <Collapser id="developer_mode" title="mode développeur">
    Lorsque vous utilisez Passenger en mode développeurs, vous remarquerez parfois que la liste des requests peut disparaître ou qu'il semble manquer certaines requests. De plus, lors de la sélection d'une demande, vous pouvez voir un message d'erreur indiquant que la trace n'est pas disponible, suggérant peut-être que vous avez redémarré.

    Cela se produit lorsque Passenger génère plus d'un processus de gestion. La trace de transaction est placée dans l'un des processus, et si vous vous connectez à l'autre, vous ne la verrez pas.

    La solution de contournement consiste à définir le nombre maximal d'instances sur 1 en utilisant le paramètre [PassengerMaxPoolSize](http://www.modrails.com/documentation/Users%20guide%20Apache.html#_passengermaxpoolsize_lt_integer_gt) . De plus, si vous actualisez plusieurs fois, vous avez de bonnes chances de récupérer les requests .
  </Collapser>
</CollapserGroup>
