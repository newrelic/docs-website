L&apos;agent Ruby New Relic monitore vos applications pour vous aider [à identifier et à résoudre les problèmes de performances](#monitor-performance). Vous pouvez également étendre la monitoring des performances de l&apos;agent pour [collecter et analyser les données commerciales](#business-data) afin de vous aider à améliorer l&apos;expérience client et à prendre des décisions commerciales data-driven .

Utilisez l&apos;agent Ruby New Relic pour résoudre les problèmes de performances de votre application avec notre [didacticiel Mon application est lente](/docs/journey-app-slow/root-causes/).

## Compatibilité et exigences [#compat]

L&apos;agent Ruby prend en charge la plupart [des frameworks et plateformesRuby ](/docs/agents/ruby-agent/getting-started/supported-frameworks)les plus courants. Vous pouvez également utiliser l&apos;agent Ruby dans un [environnement flexible Google App Engine (GAE)](/docs/agents/ruby-agent/installation/install-new-relic-ruby-agent-gae-flexible-environment). Avant d&apos;installer l&apos;agent Ruby , assurez-vous que votre système répond à l&apos;[exigence système](/docs/agents/ruby-agent/getting-started/ruby-agent-requirements-supported-frameworks/).

## Performances de l&apos;application de monitoring [#monitor-performance]

<DNT>
  **View the big picture of your app**
</DNT>

* Surveillez [l&apos;Apdex de votre application (satisfaction des utilisateurs)](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction).
* Obtenez un [résumé de haut niveau de votre application](/docs/apm/applications-menu/monitoring/apm-overview-page).
* Créez [des cartes architecturales](/docs/data-analysis/user-interface-functions/view-your-data/service-maps-visualize-monitor-apps-entire-architecture) de votre application.
* Activez [le tracing distribué](/docs/apm/agents/ruby-agent/configuration/distributed-tracing-ruby-agent) pour comprendre l&apos;activité dans un environnement qui repose sur de nombreux services.
* Installez [monitoring d&apos;infrastructureNew Relic](/docs/infrastructure) et affichez les données d&apos;hébergement détaillées de votre application.

<DNT>
  **Find errors and problems quickly**
</DNT>

* [Clé de suivi des transactions.](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events)
* Créez [un dashboard personnalisé](/docs/insights/use-insights-ui/advanced-ui-tasks/add-edit-copy-insights-dashboards) pour les mesures importantes.
* [Alertez](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic) votre équipe lorsque des erreurs ou des problèmes surviennent avant qu’ils n’affectent votre utilisateur.
* Suivre les performances [après un déploiement](/docs/agents/ruby-agent/features/recording-deployments-ruby-agent).

<DNT>
  **Drill down into performance details**
</DNT>

* Examiner le niveau du code [trace de transaction](/docs/apm/transactions/transaction-traces/transaction-traces)
* Examiner [la requête de base de données trace](/docs/apm/transactions/transaction-traces/sql-statements).
* Examiner [la trace d&apos;erreur](/docs/apm/applications-menu/events/viewing-apm-errors-error-traces).
* Monitorer [les processus d&apos;arrière-planRuby et daemon](/docs/agents/ruby-agent/background-jobs/monitoring-ruby-background-processes-daemons).

<DNT>
  **View logs for your APM and infrastructure data**
</DNT>

Rassemblez vos données de log et d&apos; application pour rendre le dépannage plus facile et plus rapide. Pas besoin de passer à une autre page UI .

* Avec [les logs en contexte](/docs/logs/logs-context/configure-logs-context-ruby/), vous pouvez voir les messages de log liés à vos erreurs et les tracer directement dans UI de votre application.
* Vous pouvez également voir les logs en contexte de vos [données infrastructure ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), telles que le cluster Kubernetes.

<DNT>
  **Analyze business data**
</DNT>

Utilisez l&apos;agent Ruby pour organiser, interroger et visualiser vos données afin de répondre à des questions clés sur les performances des applications et l&apos;expérience client.

* Utilisez l’attribut de transaction par défaut ou [ajoutez le vôtre](/docs/using-new-relic/data/customize-data/collect-custom-attributes).
* interrogez vos données [en utilisant NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language).
* Envoyez [vos propres données d&apos;événement](/docs/apm/new-relic-apm/getting-started/introduction-apm).
* Créez et partagez [des dashboards](/docs/query-your-data/explore-query-data/dashboards/introduction-new-relic-one-dashboards) personnalisables et interactifs.

## Prolonger instrumentation d&apos; agent [#extend]

Après avoir installé l&apos;agent, allez plus loin et étendez l&apos;instrumentation de l&apos;agent :

* [Délai de chargement de la page](/docs/agents/ruby-agent/features/page-load-timing-ruby): [injectez automatiquement l’agent <InlinePopover type="browser" />](/docs/browser/new-relic-browser/getting-started/new-relic-browser)pour obtenir une visibilité sur l’activité de l’utilisateur final.
* [instrumentation personnalisée](/docs/agents/ruby-agent/features/ruby-custom-instrumentation): transactions sur instruments non prises en compte dans le framework de notre instrumentation.
* [API d&apos;agent](/docs/agents/ruby-agent/customization/ruby-agent-api-guide): Utilisez l&apos;API d&apos;agent pour personnaliser entièrement le comportement de l&apos;agent. Par exemple, vous pouvez collecter des mesures personnalisées, signaler une erreur ou ignorer complètement une transaction particulière.
* [métriques personnalisées](/docs/agents/ruby-agent/features/ruby-custom-metrics): Enregistrez des métriques supplémentaires dans le cadre d&apos;une transaction pour obtenir plus d&apos;informations détaillées sur les performances et les données commerciales de votre application.
* [Attribut agent](/docs/agents/ruby-agent/attributes/ruby-agent-attributes): Personnalisez l&apos;[attribut](/docs/agents/manage-apm-agents/agent-metrics/agent-attributes) attaché aux transactions. La personnalisation des attributs vous permet d&apos;éviter d&apos;envoyer des attributs sensibles ou de collecter des attributs supplémentaires pour une visibilité plus approfondie de vos transactions.
