## Problème

Il n'y a pas de fichier log New Relic dans `log/newrelic_agent.log`.

## Solution

S'il n'y a pas de fichier `newrelic_agent.log` , essayez les étapes suivantes pour diagnostiquer :

1. Vérifiez que l'agent est réellement requis par votre application :

   * Assurez-vous que la ligne de gem `newrelic_rpm` dans votre `Gemfile` n&amp;apos;est pas marquée avec `require: false`, ce qui empêche Bundler d&amp;apos;exiger la gem. Vous pouvez supprimer l&amp;apos;instruction `require: false` ou contrôler manuellement la gem `newrelic_rpm` vous-même en appelant `require 'newrelic_rpm'` ailleurs dans votre code lors du démarrage de l&amp;apos;application.
   * Si la gem `newrelic_rpm` se trouve dans un groupe au sein de votre `Gemfile`, assurez-vous que ce groupe est en cours de chargement dans l&amp;apos;environnement dans lequel vous vous trouvez.
   * Assurez-vous que la ligne `newrelic_rpm` de votre `Gemfile` n&amp;apos;est pas enveloppée dans une condition qui est évaluée à faux dans l&amp;apos;environnement dans lequel vous vous déployez.

2. Assurez-vous que le répertoire `log` dans le répertoire racine de votre application existe et est accessible en écriture par l&amp;apos;utilisateur sous lequel votre application s&amp;apos;exécute.

3. Assurez-vous que le répertoire de travail actuel à partir duquel vous démarrez vos processus de serveur d'applications est le même que le répertoire racine de votre application (c'est-à-dire que le répertoire de travail actuel doit contenir un répertoire `log` ).

À des fins de débogage, il peut être utile de forcer l'agent à être activé et d'envoyer sa sortie log à `STDOUT`. Pour ce faire, définissez les variables d’environnement `NEW_RELIC_LOG=stdout` et `NEW_RELIC_AGENT_ENABLED=1` lors du démarrage de votre serveur d’applications.
