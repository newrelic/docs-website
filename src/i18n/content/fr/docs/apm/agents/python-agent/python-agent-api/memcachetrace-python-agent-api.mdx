## Syntaxe

```py
newrelic.agent.memcache_trace(command, async_wrapper=None)
```

Utilisé pour instrumenter des fonctions de commande memcache pour les fonctions qui ne sont pas instrumentées par défaut.

## Description

`memcache_trace` est un décorateur à ajouter aux fonctions de commande memcache. L&apos;ajout de ce décorateur vous permet de collecter des informations [de transaction](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) supplémentaires (y compris des informations de trace de transaction).

Le décorateur `memcache_trace` peut être utilisé sur les fonctions avec l&apos;agent version 2.102.0.85 ou supérieure. Le chronométrage de ces objets commence lorsque la consommation débute, et se termine lorsque l&apos;objet est épuisé ou sort de la portée.

Vous pouvez utiliser le décorateur en conjonction avec les décorateurs existants, y compris ceux pour les méthodes statiques et de classe. Lors de l&apos;imbrication de plusieurs décorateurs, choisissez le décorateur `memcache_trace` comme décorateur le plus externe.

## Formes d&apos;appel alternatives [#alternate]

Pour les configurations où vous ne pouvez pas utiliser le décorateur, ces formulaires d&apos;appel alternatifs sont disponibles :

### L&apos;emballage

Si vous savez à l&apos;avance où se trouvent les fonctions spécifiques que vous souhaitez trace, vous pouvez utiliser le décorateur `memcache_trace`. Cependant, si vous <DNT>**don&apos;t**</DNT> connaissez toutes les fonctions qui doivent être tracées (par exemple, si elles sont recherchées dynamiquement dans le cadre d&apos;un système de routage), vous devez alors utiliser `MemcacheTraceWrapper` pour encapsuler la fonction au moment de l&apos;enregistrement ou au moment de l&apos;appel.

### Enveloppement basé sur le chemin [#path-based]

`wrap_memcache_trace` est utilisé pour encapsuler des fonctions en dehors du code dans lequel elles sont déclarées. Par exemple : vous pouvez l&apos;utiliser pour instrumenter le code de la bibliothèque que vous ne souhaitez pas modifier.

Pour en savoir plus sur les différences entre ces formats d’appel, voir [Différents formats d’appel](/docs/python-agent-api-different-call-forms).

## Exemples

### `memcache_trace` [#trace\_example][#trace_example]

Un exemple d&apos;utilisation du décorateur `memcache_trace` :

```py
import newrelic.agent

@newrelic.agent.memcache_trace("get")
def get(id):
    ...
```

### Exemple d&apos;enveloppe [#trace-wrap-example]

Un exemple d&apos;utilisation du `MemcacheTraceWrapper`:

```py
import newrelic.agent

def get(id):
    ...

get = newrelic.agent.MemcacheTraceWrapper(get, command="get")
```

Si vous souhaitez renvoyer dynamiquement la valeur de la commande, vous pouvez fournir la commande en tant que fonction lors de la création de l&apos;objet `MemcacheTraceWrapper`.

```py
import newrelic.agent

def get(id):
    ...

get = newrelic.agent.MemcacheTraceWrapper(get, command=lambda id: f"get:{id}")
```
