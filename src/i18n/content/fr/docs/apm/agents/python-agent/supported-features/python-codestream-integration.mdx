Les métriques au niveau du code permettent aux développeurs utilisant l&apos;[extension New Relic CodeStream](/docs/codestream/observability/code-level-metrics) de voir les données APM affichées contextuellement dans leur IDE, aux côtés des méthodes individuelles dans le code. Cela permet aux développeurs d’être plus proactifs dans la résolution des problèmes de performances lorsqu’ils écrivent et révisent le code.

## Bien démarrer

Tout d’abord, [installez](/docs/codestream/start-here/install-codestream) l’extension New Relic CodeStream dans l’IDE pris en charge de votre choix et connectez-vous.

<Callout variant="important">
  Les métriques au niveau du code sont disponibles pour l&apos;agent Python version 7.10.0.175 et supérieure et sont activées par défaut. Pour modifier cette configuration, consultez notre [documentation](/docs/apm/agents/python-agent/configuration/python-agent-configuration/#codestream-integration).
</Callout>

## Attributs de l&apos;agent

L&apos;agent Python signale et attache l&apos;attribut suivant aux étendues produites par votre application :

<table>
  <thead>
    <tr>
      <th>
        <DNT>
          **Name**
        </DNT>
      </th>

      <th>
        <DNT>
          **Description**
        </DNT>
      </th>

      <th>
        <DNT>
          **Example**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `code.function`
      </td>

      <td>
        Le nom de la fonction instrumentée
      </td>

      <td>
        validate\_credentials
      </td>
    </tr>

    <tr>
      <td>
        `code.filepath`
      </td>

      <td>
        Le chemin absolu vers le fichier de code source dans lequel `code.function` est défini
      </td>

      <td>
        /usr/local/apps/test-app/main.py
      </td>
    </tr>

    <tr>
      <td>
        `code.lineno`
      </td>

      <td>
        Le numéro de ligne où `code.function` est défini dans code.filepath
      </td>

      <td>
        639
      </td>
    </tr>

    <tr>
      <td>
        `code.namespace`
      </td>

      <td>
        L&apos;espace de nommage (nom de classe/module) dans lequel `code.function` est défini
      </td>

      <td>
        example.service.MyService
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="important">
  Toutes les fonctions de votre code d’application ne seront pas instrumentées avec l’attribut ci-dessus. Dans certains cas, l&apos;agent encapsulera directement le code du framework de base (c&apos;est-à-dire. Django, Flask, etc.) qui est invoqué dans votre application et qui ne s&apos;affichera pas dans votre IDE.
</Callout>
