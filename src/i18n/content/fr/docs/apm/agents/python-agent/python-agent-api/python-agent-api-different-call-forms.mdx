Pour certaines fonctions, l’API de l’agent Python New Relic dispose de plusieurs options pour accomplir la même chose.Par exemple, le traçage d’une fonction pour fournir plus de détails sur une transaction monitorée par New Relic peut théoriquement être réalisé, selon votre configuration, avec l’une des API suivantes :

* Un <DNT>**decorator**</DNT>. Il s’agit d’une API d’importation. Ce sera l’option la plus simple pour la plupart des Python application configurations .
* Un <DNT>**context manager**</DNT>. Il s’agit d’une API de niveau d’exécution. Ces API peuvent être utilisées si vous souhaitez trace du code qui n&apos;est pas encapsulé dans une fonction.
* Un <DNT>**wrapper**</DNT>. Le wrapper est utilisé pour créer une fonction encapsulée sans l&apos;utilisation d&apos;un décorateur.
* Un <DNT>**path-based wrapper**</DNT>. Cela serait utilisé pour encapsuler des fonctions en dehors du code dans lequel elles sont déclarées. Par exemple, il peut être utilisé pour instrumenter du code de bibliothèque que vous ne souhaitez pas modifier.

Pour un exemple Python API d&apos; d&apos;agent avec ces quatre possibilités, voir [`function_trace`](/docs/agents/python-agent/python-agent-api/function_trace) l&apos;entrée d&apos;appel .
