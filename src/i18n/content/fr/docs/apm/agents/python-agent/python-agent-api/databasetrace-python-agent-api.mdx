## Syntaxe

```py
newrelic.agent.database_trace(sql, dbapi2_module=None, async_wrapper=None)
```

Utilisé pour instrumenter les appels aux bases de données.

## Description

`database_trace` est utilisé pour ajouter plus de détails à votre [trace de transaction](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction-trace) sous forme de segments supplémentaires. Tous les appels signalés avec `database_trace` apparaîtront sur la [page de base de données APM](/docs/apm/applications-menu/monitoring/databases-slow-queries-page). `database_trace` renvoie une [partie](https://docs.python.org/3.12/library/functools.html#functools.partial) de `DatabaseTraceWrapper` qui peut être utilisée comme décorateur pour une fonction afin de chronométrer les appels à votre base de données.

Le décorateur `database_trace` peut être utilisé sur les générateurs et les coroutines avec la version 2.102.0.85 ou ultérieure de l&apos;agent.

Si vous ne pouvez pas utiliser le décorateur dans votre application, vous pouvez utiliser l&apos;un de ces autres formats d&apos;appel :

* <DNT>**The context manager**</DNT>: Le formulaire du gestionnaire de contexte est `DatabaseTrace`.
* <DNT>**The wrapper**</DNT>: La forme wrapper est `DatabaseTraceWrapper`. Il peut être utilisé pour renvoyer une fonction encapsulée sans l&apos;utilisation d&apos;un décorateur.
* <DNT>**The path-based wrapper**</DNT>: La forme wrapper basée sur le chemin est `wrap_database_trace`. Cela applique le `DatabaseTraceWrapper` à un objet donné via le patch de singe. Cela prend le même paramètre que le décorateur plus un paramètre `module` et `object_path` supplémentaire.

Pour une explication des utilisations de ces différents formats d’appel, voir [Différents formats d’appel](/docs/python-agent-api-different-call-forms). Voir [Exemples](#examples) pour des exemples d&apos;appel.

## Valeurs de retour

`database_trace` renvoie un `DatabaseTraceWrapper()` partiel.

## Exemples

### `database_trace` [#trace\_example][#trace_example]

Un exemple d&apos;utilisation du décorateur `database_trace` :

```py
import newrelic.agent

@newrelic.agent.database_trace(sql="SELECT * FROM users WHERE id = 1")
def get_user(id):
    ...
```

Un exemple d&apos;utilisation du décorateur `database_trace` avec des coroutines natives :

```py
import newrelic.agent

@newrelic.agent.database_trace(sql="SELECT * FROM users WHERE id = 1")
async def get_user(id):
    ...
```

### Exemple de gestionnaire de contexte [#context-mgr-example]

Un exemple d&apos;utilisation du gestionnaire de contexte `DatabaseTrace` : cela donnera des informations sur le temps nécessaire pour effectuer « custom\_action ».

```py
import newrelic.agent


def complex_query(a, b, c):
    with Connection(host, port, db) as conn:
        with newrelic.agent.DatabaseTrace(
            sql="SELECT * FROM users WHERE id = 1",
        ):

            conn.custom_action()
```

### Exemple d&apos;enveloppe [#trace-wrap-example]

Un exemple d&apos;utilisation du `DatabaseTraceWrapper`:

```py
import newrelic.agent

def get(id):
    ...


get = newrelic.agent.DatabaseTraceWrapper(
    get, sql="SELECT * FROM users WHERE id = 1"
)
```
