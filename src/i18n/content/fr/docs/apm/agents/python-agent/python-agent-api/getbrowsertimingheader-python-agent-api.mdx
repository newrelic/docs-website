## Syntaxe [#syntax]

```py
get_browser_timing_header(nonce=None)
```

Génère un en-tête d&apos;agent <InlinePopover type="browser" />pour injecter l&apos;agent JavaScript.

## Description [#description]

<Callout variant="important">
  Le paramètre nonce pour la prise en charge CSP nonced a été ajouté dans la version 9.4.0 de l&apos;agent Python.

  Dans les versions 9.3.0 et inférieures de l&apos;agent Python, l&apos;API [`get_browser_timing_footer`](/docs/agents/python-agent/python-agent-api/get_browser_timing_footer) était également requise pour activer la synchronisation du navigateur. À partir de la version 9.4.0 de l&apos;agent et des versions ultérieures, cela n&apos;est plus nécessaire.
</Callout>

Cet appel renvoie agent snippet l&apos; d&apos;en-tête [monitoring JavaScript de l&apos; de navigateur](/docs/agents/python-agent/supported-features/page-load-timing-python) que vous pouvez utiliser pour activer des navigateurs . Vous pouvez l&apos;utiliser si l&apos;instrumentation automatique n&apos;est pas possible pour votre application, si vous utilisez une politique de sécurité de contenu non ced ou si vous souhaitez collecter des données de navigateur uniquement pour des pages spécifiques.

Il s&apos;agit d&apos;une alternative manuelle à [l&apos;insertion automatique de ce JavaScript](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser#select-apm-app) par l&apos;agent. Pour savoir comment activer monitoring des navigateurs à l&apos;aide de cet appel avec `get_browser_timing_footer`, voir [instrumenter manuellement le monitoring des navigateurs](/docs/agents/python-agent/supported-features/page-load-timing-python#manual_instrumentation).

Cet appel doit être appelé pour chaque requête Web où vous souhaitez injecter le snippet JavaScript. Il ne doit pas (<DNT>**cannot**</DNT>) être appelé au niveau global et mis en cache.
