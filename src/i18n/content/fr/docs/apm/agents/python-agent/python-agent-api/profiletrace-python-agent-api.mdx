## Syntaxe

```py
newrelic.agent.profile_trace(name=None, group=None, label=None, params=None, terminal=False)
```

Utilisé pour instrumenter une fonction et les fonctions qu&apos;elle contient jusqu&apos;à une profondeur particulière pour les fonctions qui ne sont pas instrumentées par défaut.

## Description

`profile_trace` est un décorateur permettant d&apos;ajouter des fonctions. L&apos;ajout de ce décorateur vous permet de collecter des informations [de transaction](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) supplémentaires (y compris des informations de trace de transaction).

Le décorateur `profile_trace` peut être utilisé sur les fonctions avec l&apos;agent version 2.102.0.85 ou supérieure. Le chronométrage de ces objets commence lorsque la consommation débute, et se termine lorsque l&apos;objet est épuisé ou sort de la portée.

Vous pouvez utiliser le décorateur en conjonction avec les décorateurs existants, y compris ceux pour les méthodes statiques et de classe. Lors de l&apos;imbrication de plusieurs décorateurs, choisissez le décorateur `profile_trace` comme décorateur le plus externe.

## Formes d&apos;appel alternatives [#alternate]

Pour les configurations où vous ne pouvez pas utiliser le décorateur, ces formulaires d&apos;appel alternatifs sont disponibles :

### L&apos;emballage

Si vous savez à l&apos;avance où se trouvent les fonctions spécifiques que vous souhaitez trace, vous pouvez utiliser le décorateur `profile_trace`. Cependant, si vous <DNT>**don&apos;t**</DNT> connaissez toutes les fonctions qui doivent être tracées (par exemple, si elles sont recherchées dynamiquement dans le cadre d&apos;un système de routage), vous devez alors utiliser `ProfileTraceWrapper` pour encapsuler la fonction au moment de l&apos;enregistrement ou au moment de l&apos;appel.

### Enveloppement basé sur le chemin [#path-based]

`wrap_profile_trace` est utilisé pour encapsuler des fonctions en dehors du code dans lequel elles sont déclarées. Par exemple : vous pouvez l&apos;utiliser pour instrumenter le code de la bibliothèque que vous ne souhaitez pas modifier.

Pour en savoir plus sur les différences entre ces formats d’appel, voir [Différents formats d’appel](/docs/python-agent-api-different-call-forms).

## Exemples

### `profile_trace` [#trace\_example][#trace_example]

Un exemple d&apos;utilisation du décorateur `profile_trace` :

```py
import newrelic.agent

@newrelic.agent.profile_trace(depth=7)
def find_node(tree, value):
    if tree.value == value:
        return tree
    for node in tree.children:
        found_node = find_node(node, value)
        if found_node:
            return found_node
```

Dans l&apos;exemple ci-dessus, la trace de profil enveloppe une fonction récursive. Les traces de fonction seront ajoutées jusqu&apos;à une profondeur de 7 appels de stack récursifs dans la fonction.

### Exemple d&apos;enveloppe [#trace-wrap-example]

Un exemple d&apos;utilisation du `ProfileTraceWrapper`:

```py
import newrelic.agent

def find_node(tree, value):
    if tree.value == value:
        return tree
    for node in tree.children:
        found_node = find_node(node, value)
        if found_node:
            return found_node

find_node = newrelic.agent.ProfileTraceWrapper(find_node, depth=7)
```

Si vous souhaitez nommer la métrique d&apos;après le nom du point de terminaison (plutôt que de nommer la métrique en fonction de l&apos;identifiant de la fonction appelée), vous pouvez fournir le nom à utiliser ainsi qu&apos;un préfixe de chemin de métrique alternatif lors de la création de l&apos;objet `ProfileTraceWrapper` .

```py
import newrelic.agent

def find_node(tree, value):
    if tree.value == value:
        return tree
    for node in tree.children:
        found_node = find_node(node, value)
        if found_node:
            return found_node

find_node = newrelic.agent.ProfileTraceWrapper(find_node, name="find_node", group="Python/Profile", depth=7)
```

Si vous souhaitez nommer la métrique de manière dynamique (plutôt que de nommer la métrique à partir de l&apos;identifiant de la fonction appelée), vous pouvez fournir le nom en tant que fonction lors de la création de l&apos;objet `ProfileTraceWrapper`.

```py
import newrelic.agent

def find_node(tree, value):
    if tree.value == value:
        return tree
    for node in tree.children:
        found_node = find_node(node, value)
        if found_node:
            return found_node

find_node = newrelic.agent.ProfileTraceWrapper(find_node, name=lambda tree, value: f"find_node_{value}", depth=7)
```
