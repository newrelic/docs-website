## Syntaxe

```py
newrelic.agent.suppress_apdex_metric(flag=True)
```

Empêche la transaction en cours de contribuer à Apdex.

## Description

Cet appel supprime la génération de la [métrique Apdex](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) pour un web de transaction. Vous souhaiterez peut-être effectuer cette opération lorsqu’une transaction de longue durée affecte fréquemment le score Apdex moyen de votre application.

Pour annuler la suppression d&apos;une transaction précédemment supprimée, définissez l&apos;indicateur sur `False`.

<Callout variant="tip">
  Vous pouvez également supprimer la métrique Apdex dans le dictionnaire d&apos;environnement WSGI. Pour ce faire, définissez `newrelic.suppress_apdex_metric` la clé pour la demande spécifique dans le WSGI dictionnaire d’environnement transmis par le WSGI serveur à l&amp;apos;application WSGI monitorée.
</Callout>

## Valeurs de retour

Aucun.

## Exemples

### Désactiver Apdex pour une transaction spécifique [#apdex-example]

Si votre application comporte des fonctions fréquentes et de longue durée qui entraînent une baisse de votre Apdex moyen, vous pouvez appeler `suppress_apdex_metric` où la transaction est générée :

```py
import newrelic.agent
newrelic.agent.suppress_apdex_metric()
```
