Avant d&apos;installer notre agent Python, assurez-vous que votre système répond à ces exigences.

## Exigences de base [#basic]

L&apos;agent Python prend en charge la plupart des Python frameworks Web les plus courants. Une configuration supplémentaire peut être requise en fonction de votre mécanisme d&apos;hébergement spécifique, de votre service d&apos;hébergement, de votre framework Web ou de vos services backend . Consultez le tableau ci-dessous pour plus de détails sur l&apos;assistance.

Si vous n&apos;en avez pas déjà un, [créez un compte New Relic](https://newrelic.com/signup). C&apos;est gratuit, pour toujours.

## Python package instrumentation [#python-packages]

L&apos;agent fournit également instrumentation pour une Python [gamme d&apos;autres packages et modules](/docs/agents/python-agent/getting-started/instrumented-python-packages). L&apos;instrumentation comprend des requêtes de base de données, memcache requests, de service requests externe, [instancedes détails](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues) au niveau de, etc. Les résultats apparaîtront automatiquement dans l&apos;interface utilisateur d&apos;APM dans les ventilations des performances des transactions Web et la trace de transaction pour les transactions lentes.

## support de version package instrumenté [#package-version]

L&apos;agent Python prend en charge et teste activement le package instrumenté dans une fenêtre de sortie de trois ans.

## prise en charge des versions d&apos;agents [#agent-version]

New Relic vous recommande de mettre à niveau l&apos;agent régulièrement et au moins tous les 3 mois. les versions d&apos;agent sont prises en charge dans une fenêtre de sortie d&apos;un an.

## Prise en charge des versions Python [#python-version]

L&apos; agent en général prendra en charge toutes [les branchesPython ](https://devguide.python.org/versions/#versions)sorties et actives. Cependant, pour suivre les changements à venir, l&apos;agent suivra également ce calendrier de support des versions Python. La politique de support des versions ne remplace pas notre [politique générale de fin de vie (EOL)](/docs/agents/manage-apm-agents/maintenance/new-relic-agent-plugin-end-life-policy).
