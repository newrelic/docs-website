## Syntaxe

```py
newrelic.agent.generator_trace(name=None, group=None, label=None, params=None, terminal=False)
```

Utilisé pour instrumenter des générateurs qui ne sont pas instrumentés par défaut.

## Description

`generator_trace` est un décorateur à ajouter aux fonctions génératrices. L&apos;ajout de ce décorateur vous permet de collecter des informations [de transaction](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction) supplémentaires (y compris des informations de trace de transaction).

Le décorateur `generator_trace` peut être utilisé sur les générateurs et les coroutines avec la version 2.102.0.85 ou supérieure de l&apos;agent. Le chronométrage de ces objets commence lorsque la consommation débute, et se termine lorsque l&apos;objet est épuisé ou sort de la portée.

Vous pouvez utiliser le décorateur en conjonction avec les décorateurs existants, y compris ceux pour les méthodes statiques et de classe. Lors de l&apos;imbrication de plusieurs décorateurs, choisissez le décorateur `generator_trace` comme décorateur le plus externe.

## Formes d&apos;appel alternatives [#alternate]

Pour les configurations où vous ne pouvez pas utiliser le décorateur, ces formulaires d&apos;appel alternatifs sont disponibles :

### L&apos;emballage

Si vous savez à l&apos;avance où se trouvent les générateurs spécifiques que vous souhaitez trace, vous pouvez utiliser le décorateur `generator_trace`. Cependant, si vous <DNT>**don&apos;t**</DNT> connaissez tous les générateurs qui doivent être tracés (par exemple, s&apos;ils sont recherchés dynamiquement dans le cadre d&apos;un système de routage), vous devez alors utiliser `GeneratorTraceWrapper` pour encapsuler le générateur au moment de l&apos;enregistrement ou au moment de l&apos;appel.

### Enveloppement basé sur le chemin [#path-based]

`wrap_generator_trace` est utilisé pour encapsuler des générateurs en dehors du code dans lequel ils sont déclarés. Par exemple : vous pouvez l&apos;utiliser pour instrumenter le code de la bibliothèque que vous ne souhaitez pas modifier.

Pour en savoir plus sur les différences entre ces formats d’appel, voir [Différents formats d’appel](/docs/python-agent-api-different-call-forms).

## Exemples

### `generator_trace` [#trace\_example][#trace_example]

Un exemple d&apos;utilisation du décorateur `generator_trace` :

```py
import newrelic.agent

@newrelic.agent.generator_trace()
def gen():
    for i in range(5):
        yield i
```

Un exemple d&apos;utilisation du décorateur `generator_trace` avec un générateur asynchrone :

```py
import newrelic.agent

@newrelic.agent.generator_trace()
async def gen():
    for i in range(5):
        yield i
```

### Exemple d&apos;enveloppe [#trace-wrap-example]

Un exemple d&apos;utilisation du `GeneratorTraceWrapper`:

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen)
```

Si vous souhaitez nommer la métrique d&apos;après le nom du point de terminaison (plutôt que de nommer la métrique en fonction de l&apos;identifiant du générateur appelé), vous pouvez fournir le nom à utiliser ainsi qu&apos;un préfixe de chemin de métrique alternatif lors de la création de l&apos;objet `GeneratorTraceWrapper`.

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name="gen_5", group="Python/Generator")
```

Si vous souhaitez nommer la métrique dynamiquement (plutôt que de nommer la métrique en fonction de l&apos;identifiant du générateur appelé), vous pouvez fournir le nom en tant que générateur lors de la création de l&apos;objet `GeneratorTraceWrapper`.

```py
import newrelic.agent

def gen(x):
    for i in range(x):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name = lambda x: f"gen_{x}")
```
