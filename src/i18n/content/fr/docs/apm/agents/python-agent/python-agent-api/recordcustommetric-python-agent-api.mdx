## Syntaxe

```PY
newrelic.agent.record_custom_metric(name, value, application=None)
```

Enregistre une seule métrique personnalisée.

## Description

Cet appel enregistre une seule [métrique personnalisée](/docs/agents/manage-apm-agents/agent-data/custom-metrics). Pour enregistrer un ensemble de métriques, voir [`record_custom_metrics`](/docs/agents/python-agent/python-agent-api/record_custom_metrics).

## Valeurs de retour

Aucun.

## Exemples

### Enregistrement d&apos;une métrique personnalisée à l&apos;aide de l&apos;objet application [#app-object-example]

Pour enregistrer des métriques personnalisées à partir d&apos;un thread d&apos;arrière-plan distinct ou d&apos;un autre code, vous devez transmettre l&apos; [objetapplication ](/docs/agents/python-agent/python-agent-api/application)correspondant à l&amp;apos; application à laquelle les métriques personnalisées doivent être signalées.

```PY
import newrelic.agent
application = newrelic.agent.application()

newrelic.agent.record_custom_metric('Custom/my_favorite_number', 42, application)
```

## Afficher et utiliser des métriques personnalisées

Pour afficher les métriques personnalisées, utilisez [métriques et événement](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) pour rechercher et filtrer les métriques personnalisées, créer des graphiques personnalisables et ajouter ces graphiques au dashboard New Relic . Vous pouvez utiliser notre [API REST](/docs/apis/rest-api-v2/requirements/new-relic-rest-api-v2-getting-started) pour récupérer et utiliser par programmation des données métriques personnalisées en dehors de l&amp;apos; UI. Il est également possible de [créer des conditions d&apos;alerte de métriques personnalisées](/docs/alerts/new-relic-alerts/configuring-alert-policies/define-custom-metrics-alert-condition) pour vous avertir, vous ou votre équipe, lorsque vos métriques personnalisées dépassent des valeurs spécifiques.
