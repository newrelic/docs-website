## Syntaxe

```py
newrelic.agent.capture_request_params(flag=True)
```

Permet la capture du paramètre de chaîne de requête d&apos;une transaction.

## Description

Cet appel permet la capture du paramètre de chaîne de requête d&apos;un site Web de transaction en tant [qu&apos;attribut](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute). Cette collection est bloquée par défaut pour des raisons de sécurité. Si vous souhaitez plutôt activer la collecte de paramètres de chaîne de requête pour l&amp;apos;ensemble de votre application, utilisez le paramètre de configuration [`record_sql`](/docs/agents/python-agent/installation-configuration/python-agent-configuration#txn-tracer-sql) .

[Le mode haute sécurité](/docs/accounts-partnerships/accounts/security/high-security) remplace cet appel s&amp;apos;il est actif, auquel cas cet appel n&amp;apos;a aucun effet. Cet appel remplace les autres paramètres pertinents dans la [configuration](/docs/agents/python-agent/installation-configuration/python-agent-configuration) de l&amp;apos;agent.

<Callout variant="tip">
  Vous pouvez également activer la capture de chaîne de requête dans le dictionnaire d&apos;environnement WSGI. Pour ce faire, marquez la clé `newrelic.capture_request_params` pour la demande spécifique dans le dictionnaire d’environnement WSGI transmis par le serveur WSGI à l’application WSGI cible.
</Callout>

## Exemples

### Activer la capture de la chaîne de requête [#capture-example]

Placez cet appel dans la section de code correspondant à une transaction pour laquelle vous souhaitez collecter le paramètre de chaîne de requête :

```py
newrelic.agent.capture_request_params()
```
