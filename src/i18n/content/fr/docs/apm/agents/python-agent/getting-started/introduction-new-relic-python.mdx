Notre Python agent monitore votre Python application pour vous aider à identifier et à résoudre [les problèmes de performances](#monitor-performance). Vous pouvez également étendre votre monitoring des performances pour collecter et analyser [des données commerciales](#business-data) afin de vous aider à améliorer l&apos;expérience client et à prendre des décisions commerciales data-driven . Avec des options flexibles pour [l&apos;instrumentation personnalisée et les API](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation), l&apos;agent Python propose plusieurs blocs de construction pour personnaliser les données dont vous avez besoin à partir de votre application.

Notre Python fonctionne avec une grande variété de [frameworks Web et de mécanismes d&apos;hébergement](/docs/agents/python-agent/getting-started/instrumented-python-packages), notamment Django, Gunicorn, WSGI, CherryPy, uWSGI, etc. Vous pouvez également installer l&apos;agent Python dans un [environnement flexible Google App Engine](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment).

## performances de l&apos;application de monitoring [#monitor-performance]

Après avoir [installé](#installation) l&apos;agent Python, il commence à collecter des données sur votre application. Vous pouvez afficher les données sous forme de graphiques et de tableaux dans l&apos;[New Relic UI](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one).

<DNT>

</DNT>

* Activez [le tracing distribué](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) pour voir l&apos;activité sur une architecture comportant de nombreux services.
* Installez [monitoring d&apos;infrastructure](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure) et visualisez les données détaillées du serveur/hôte pour votre application.

<DNT>
  **Find errors and problems quickly:**
</DNT>

* Afficher les performances [après un déploiement](/docs/apm/applications-menu/events/deployments-page).

<DNT>
  **Drill down into performance details:**
</DNT>

* Examiner le niveau du code [trace de transaction](/docs/apm/transactions/transaction-traces/transaction-traces).

* Vous pouvez également voir les journaux en contexte de vos [données infrastructure ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), telles que le cluster Kubernetes.

## Installer l&apos;agent Python [#installation]

Avant Python agent d&apos;installer l&apos;, assurez-vous que [votre système répond à l&apos; exigence système .](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent) Vous devez également créer un [compte New Relic](https://newrelic.com/signup).

Nous prenons en charge un certain nombre de frameworks et de bibliothèques Web prêts à l&apos;emploi, notamment Django, WSGI et Gunicorn. Si vous utilisez l’un des [frameworks Web pris en charge](/docs/agents/python-agent/getting-started/instrumented-python-packages), installation est facile. Si vous utilisez un framework non pris en charge, le processus impliquera quelques ajouts au code de votre application et/ou aux fichiers du serveur Web.

Pour un processus d&apos;installation rapide et simple qui fonctionnera pour la majorité des configurations, suivez ces étapes simples :

1. Téléchargez et installez le package Python.
2. Créer un fichier de configuration.
3. Intégrez l&apos;agent Python à votre application.

## script non Web du moniteur, tâches d&apos;arrière-plan et fonctions [#non-web]

L&apos;agent Python vous permet également de monitorer les scripts non Web, les processus de travail, les tâches et les fonctions. Le processus installation de ces transactions non Web est similaire à celui utilisé pour une application Web, avec une différence majeure : au lieu de passer par le processus d&apos;intégration standard décrit dans les [instructions d&apos;installation](/docs/agents/python-agent/getting-started/python-agent-quick-start), vous « encapsulerez » manuellement toute fonction que vous souhaitez monitorer. Pour plus d&apos;informations, voir [Tâches et processus non Web](/docs/agents/python-agent/supported-features/python-background-tasks). Pour obtenir des instructions sur monitoring des tâches Celery, consultez [Tâches d&apos;arrière-planCelery ](/docs/agents/python-agent/back-end-services/python-agent-celery).

## Quelle est la prochaine étape après l&apos;installation ? [#instrumentation]

Une fois l&apos;agent opérationnel, voici quelques étapes suggérées :

* Explorez vos données et familiarisez-vous avec l&apos;interface utilisateur.
* Lisez notre documentation sur [nos autres solutions d&apos;observabilité](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability) et la [page APM](/docs/apm).
* [Modifiez le nom de votre application](/docs/agents/manage-apm-agents/app-naming/rename-your-application) ou d&apos;autres [options de configuration](/docs/agents/python-agent/installation-configuration/python-agent-configuration).
* Découvrez comment configurer [instrumentation personnalisée](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation) pour l&apos;activité application non monitorée par défaut.
* Considérez le [SDK de télémétrie Python](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic).

## Dépannage

Une fois le processus d’installation terminé, vos données devraient apparaître dans l’[UI APM](/docs/apm/applications-menu/monitoring/apm-overview-page) dans les cinq minutes. Si ce n’est pas le cas, utilisez ces ressources de dépannage :

* Si aucune donnée n&apos;apparaît, suivez ces [étapes de dépannage](/docs/agents/python-agent/troubleshooting/no-data-appears-python).
* Si vous rencontrez des problèmes lors de l&apos;installation ou de l&apos;exécution Python agent de l&apos; sur un nouvel hôte, [vérifiez que le package est correctement installé](/docs/agents/python-agent/installation-configuration/testing-python-agent) et qu&apos;il peut contacter le service de collecte de données de New Relic.
* Pour d&apos;autres problèmes, consultez la [liste complète de la documentation de dépannage](/docs/agents/python-agent/troubleshooting/).
