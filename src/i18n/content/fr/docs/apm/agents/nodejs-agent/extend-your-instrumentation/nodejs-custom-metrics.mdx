Enregistrez des données de performances arbitraires via un appel d&apos;API, par exemple des données de synchronisation ou de ressources informatiques. Utilisez [métriques et événement](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) pour rechercher vos métriques personnalisées, créer des graphiques personnalisables et ajouter ces graphiques au dashboard New Relic . L&apos;utilisation de métriques personnalisées unifie votre monitoring au sein de New Relic.

<Callout variant="caution">
  Collecter trop de métriques peut avoir un impact sur les performances de votre application et de votre agent New Relic. Pour éviter les problèmes de données, gardez le nombre total de métriques uniques personnalisées en dessous de 2000.
</Callout>

## Nommez vos métriques [#metric\_names][#metric_names]

Les noms de métriques sont des chemins délimités par le caractère `/` . Pour les métriques personnalisées, utilisez ce modèle :

```
<category>/<class>/<method>
```

Pour les noms métriques personnalisés, utilisez `<class>/<method>` ou `<category>/<name>`. Par exemple, utilisez `MyCategory/My_method`).

<Callout variant="important">
  Pour les versions antérieures à l&apos;agent Node.js v5, vous devez préfixer vos métriques avec `Custom/`. L&apos;agent n&apos;ajoutera pas automatiquement de métriques personnalisées avec ceci pour éviter toute collision avec les métriques internes.
</Callout>

## Enregistrer des métriques personnalisées [#recording-custom-metrics]

L&apos;API publique d&apos;enregistrement des données métriques se compose de deux méthodes sur `newrelic` :

* [`recordMetric`](https://newrelic.github.io/node-newrelic/API.html#recordMetric): À utiliser pour créer de nouvelles métriques personnalisées.
* [`incrementMetric`](https://newrelic.github.io/node-newrelic/API.html#incrementMetric): Permet de mettre à jour la valeur d&apos;une métrique personnalisée.

Pour plus d&apos;informations, consultez l&apos; [exemple d&apos;application](https://github.com/newrelic/newrelic-node-examples/tree/69d500e02077d2565c4762c21bf779e3e39845bf/custom-instrumentation/attributes-events-metrics) sur GitHub.
