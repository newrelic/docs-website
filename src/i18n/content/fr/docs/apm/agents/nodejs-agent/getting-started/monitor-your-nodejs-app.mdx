Vous souhaitez commencer monitoring votre Node.js application avec New Relic dès maintenant ? Utilisez ce guide pour trouver le meilleur chemin pour commencer à envoyer vos données d&apos;application à New Relic.

Vous souhaitez en savoir plus avant de commencer ? [Introduction à APM](/docs/apm/new-relic-apm/getting-started/introduction-apm) et [Introduction à New Relic pour Node.js](/docs/apm/agents/nodejs-agent/getting-started/introduction-new-relic-nodejs) sont un bon point de départ. Si vous souhaitez en savoir plus sur le processus d&apos;installation général, consultez notre [présentation de l&apos;installation de Node.js.](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent)

## Que souhaitez-vous monitorer ? [#what]

<Callout variant="tip">
  Votre application s’exécute-t-elle sur un cluster Kubernetes ? Essayez notre méthode d&apos;installation à l&apos;aide de l&apos; [auto-attachement Kubernetes APM](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator/).
</Callout>

Nous avons quelques chemins pour installer l&apos;agent Node.js, en fonction de ce que vous monitoring.

<CollapserGroup>
  <Collapser id="docker" title="Application déployée dans un conteneur Docker">
    Pour installer l’agent dans un conteneur Docker, suivez [ces étapes autoguidées](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent-docker) à l’aide de la ligne de commande.

    Ce processus vous guide tout au long de l&apos;ajout de l&apos;agent à `package.json`, puis de l&apos;exigence de New Relic au niveau de l&apos;application. À la fin, vous ajouterez votre clé de licence et le nom de votre application à `docker run` afin que l’agent s’exécute avec votre application Docker.
  </Collapser>

  <Collapser id="server" title="Application déployée sur un serveur">
    Nous avons quelques options différentes pour les applications hébergées sur un serveur.

    * Notre [installation guidée](https://one.newrelic.com/marketplace/install-data-source?state=f535d53b-5b84-cd48-80cf-61704ad02e29) décompose l&apos;installation en étapes. Cela nécessite une certaine interaction avec la ligne de commande.

    * Notre documentation [d&apos;installation de l&apos;agent Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/install-nodejs-agent) utilise exclusivement la ligne de commande. Vous pouvez visualiser le même processus dans notre [Node.js agent](https://github.com/newrelic/node-newrelic#installation) référentiel

    * Pour les applications qui utilisent le gestionnaire de processus PM2, suivez l&apos; [installation guidée](https://one.newrelic.com/marketplace/install-data-source?state=ba2c3ecb-defb-e99a-c4be-47d786690d2a). Si vous utilisez un gestionnaire de processus autre que PM2, choisissez l&apos;option d&apos;installation guidée ou manuelle.

      Lorsque vous êtes prêt à monitorer votre application, ajoutez `-r newrelic` à la commande de démarrage de votre application pour exiger l&apos; agent avant les autres modules, par exemple `node -r newrelic your-program.js`. Ceci est particulièrement important si vous n&apos;avez pas d&apos;accès root à votre application.
  </Collapser>

  <Collapser id="lambda" title="Déploiement d'application avec AWS Lambda">
    Pour installer un agent pour votre fonction AWS Lambda, nos [étapes autoguidées](https://one.newrelic.com/marketplace/install-data-source?state=9aa3d004-6f13-1477-604e-b6a2d6bb992d) fournissent un script de configuration qui configure automatiquement AWS pour vous.

    Si vous préférez une installation manuelle via la ligne de commande, nous avons actuellement l&apos;option d&apos; [instrumentation Legacy pour le monitoring Lambda](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-legacy/#node).
  </Collapser>

  <Collapser id="azure" title="Application Web Azure sur Windows">
    Si vous souhaitez instrumenter une application Web Azure pour vos applications déployées sous forme de code dans un environnement Windows, vous pouvez facilement installer et configurer l’agent New Relic Node avec l’ [extension de site de l’agentAzure Node](/docs/agents/nodejs-agent/hosting-services/nodejs-agent-microsoft-azure).
  </Collapser>
</CollapserGroup>
