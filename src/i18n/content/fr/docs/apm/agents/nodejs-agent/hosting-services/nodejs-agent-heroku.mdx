[Heroku](https://devcenter.heroku.com/articles/newrelic) est une solution de plateforme en tant que service (PaaS) pour l&amp;apos;hébergement d&amp;apos;applications Web dans divers langages agent, y compris Node.js Avec New Relic, vous pouvez étendre Heroku avec des métriques de [<InlinePopover type="apm" />](/docs/apm/new-relic-apm)et [<InlinePopover type="browser" />](/docs/browser/new-relic-browser).

Nous décrivons ici des considérations particulières pour l'utilisation d'Heroku comme service d'hébergement avec l'agent Node.js de New Relic.

## Installer le module complémentaire de l'agent [#installing]

Après avoir déployé votre Node.js application sur Heroku, installez l'agent New Relic. L'installation du module complémentaire crée automatiquement un compte New Relic privé et configure l'accès aux serveurs Heroku.

Pour installer le module complémentaire New Relic via la [page Module complémentaire du site WebHeroku pour New Relic](https://addons.heroku.com/newrelic), vous devez être connecté à Heroku.

1. Depuis la page des modules complémentaires d'Heroku pour New Relic, sélectionnez le [plan d'abonnement approprié](https://elements.heroku.com/addons/newrelic#pricing). Ensuite, exécutez cette commande toolbelt :

   ```shell
   heroku addons:create newrelic:$planlevel
   ```

2. Depuis <DNT>**Select an app**</DNT>, sélectionnez votre application New Relic.

3. Utilisez cette commande de la ceinture d'outils pour donner à votre application un [nom descriptif](/docs/apm/new-relic-apm/installation-configuration/name-your-application):

   ```shell
   heroku config:set NEW_RELIC_APP_NAME='Your Application Name'
   ```

4. Si vous utilisez des variables d'environnement au lieu des paramètres de votre fichier de configuration `newrelic.js` pour [personnaliser la configuration de votre agent Node.js](#variables), utilisez cette commande toolbelt :

   ```shell
   heroku config:set NEW_RELIC_NO_CONFIG_FILE='true'
   ```

5. Vérifiez le nom de votre application New Relic, <InlinePopover type="licenseKey" />, et [le paramètrelog ](/docs/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration#logging_config):

   ```shell
   heroku config
   ```

6. Installez Node.js agent `npm` l&amp;apos; et enregistrez-le dans votre dépendance.

   ```shell
   npm install newrelic --save
   ```

7. Pour vous assurer que le package `newrelic` est inclus dans votre fichier `package.json` lorsque vous envoyez vers Heroku, installez le [package pour l'agent New Relic Node.js](/docs/agents/nodejs-agent/installation-configuration/installing-maintaining-nodejs) sur votre système local. Utilisez l&amp;apos;indicateur Node.js `-r/--require` lorsque vous exécutez votre programme pour vous assurer que le module de New Relic est chargé :

   ```shell
   node -r newrelic YOUR_PROGRAM.js
   ```

8. Exécutez les commandes suivantes :

   ```shell
   git add . && git commit -m "Add New Relic"
   git push heroku master && heroku logs  --tail
   ```

9. Redémarrez votre dyno.

10. Générez du trafic vers votre application.

New Relic commencera monitoring les performances des applications, l'expérience finale de l'utilisateur et les performances de l'hôte collectées après l'installation du module complémentaire. Dans quelques minutes, les données devraient commencer à apparaître sur votre [page APM <DNT>**Summary**</DNT> ](/docs/apm/applications-menu/monitoring/apm-overview-page).

## Dépannez votre installation [#troubleshooting]

Si aucune donnée n'apparaît quelques minutes après avoir terminé les procédures d'installation, suivez les [procédures de dépannage de l'agent Node.js.](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-your-nodejs-installation)

## Personnalisez votre configuration [#variables]

Vous pouvez personnaliser le [fichier de configuration New Relic`newrelic.js` ](/docs/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration#methods-and-precedence)pour votre agent Node.js. Vous pouvez également utiliser les variables d&amp;apos;environnement de l&amp;apos;agent Node.js de New Relic dans l&amp;apos;environnement Heroku pour remplacer les valeurs de votre fichier de configuration.

Voici un exemple d'utilisation de la ligne de commande Heroku pour définir des variables d'environnement au lieu d'utiliser votre fichier de configuration `newrelic.js` .

```shell
heroku config:set NEW_RELIC_LICENSE_KEY=your license key
heroku config:set NEW_RELIC_APP_NAME=your production app name
heroku config:set NEW_RELIC_NO_CONFIG_FILE='true'
```

Pour confirmer vos paramètres depuis la ligne de commande, utilisez :

```shell
heroku config
```

## Mise à niveau à partir d'une installation New Relic existante [#upgrading]

Pour mettre à niveau votre version d'agent Node.js si New Relic est déjà installé, utilisez cette commande toolbelt :

```shell
npm install newrelic --save
```
