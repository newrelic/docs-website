Le C SDK de New Relic fournit une bibliothèque générique que vous pouvez personnaliser pour communiquer avec New Relic. Avant d&apos;installer le C SDK de New Relic, assurez-vous que votre système répond à ces exigences. Consultez également la [documentation des licences du C SDK](/docs/c-sdk-licenses).

Vous souhaitez tester notre C SDK ? [Créez un compte New Relic](https://newrelic.com/signup) gratuitement ! Aucune carte de crédit requise.

## Versions de langues [#language-release]

Le C SDK vous permet d&apos;instrumenter n&apos;importe quelle application sous Linux, à condition :

* Vous devez avoir accès au code source.
* Le langage cible doit pouvoir importer la bibliothèque C.

Vous pouvez utiliser le C SDK pour créer instrumentation pour une application écrite en C et C++. Vous pouvez également l&apos;utiliser dans des langages comme Perl, qui sont implémentés en C ou C++, et dans tout langage disposant d&apos;une interface de fonction étrangère avec C.

## Environnements d&apos;exploitation [#operating-systems]

Le C SDK fonctionne sur un système d&apos;exploitation Linux 64 bits avec :

* gcc 4.8 ou supérieur
* glibc 2.17 ou supérieur
* Version du noyau 2.6.26 ou supérieure
* libpcre 8.20 ou supérieur
* libpthread
* cmake 2.8 ou supérieur

La compilation du démon New Relic nécessite Go 1.7 ou supérieur.

## clé de licence [#license]

Vous aurez besoin d&apos;une New Relic <InlinePopover type="licenseKey" />pour le compte auquel vous souhaitez signaler des données.

## Type de processeur [#processor]

* Plateforme Intel (et compatible) uniquement
* La prise en charge des instructions SSE2 est requise

## Exigences de sécurité [#security]

En tant que [mesure de sécurité standard pour la collecte de données](/docs/accounts-partnerships/new-relic-security/security/data-security), votre serveur d&apos;applications doit prendre en charge SHA-2 (256 bits). SHA-1 n&apos;est pas pris en charge.
