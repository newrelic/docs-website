Votre application C nécessite deux valeurs de configuration :

* Nom de l&apos;application : [`app_name`](https://newrelic.github.io/c-sdk/struct__newrelic__app__config__t.html#a0543f501cfec009c483be0c6d5d511f7)
* New Relic <InlinePopover type="licenseKey" />: [`license_key`](https://newrelic.github.io/c-sdk/struct__newrelic__app__config__t.html#aac3c5f9a0e5da64836a806c47b954b71)

Toutes les autres valeurs de configuration sont facultatives et, généralement, les paramètres par défaut n&apos;ont pas besoin d&apos;être modifiés. Toutefois, si nécessaire, vous pouvez ajuster la configuration <DNT>C SDK</DNT> de New Relic.

Ce document est une référence rapide pour l&apos;utilisation de certaines options de configuration de base avec l&apos;API <DNT>C SDK</DNT> . Pour obtenir des informations détaillées sur la modification de vos paramètres de configuration, y compris les valeurs de code et les exemples, consultez le [GUIDE de configuration de<DNT>C SDK</DNT>.md sur GitHub](https://github.com/newrelic/c-sdk/blob/master/GUIDE.md#configuration).

## Modifier le nom de l&apos;application (alias) dans UI [#change-ui]

Vous pouvez modifier [application l&apos;alias de](/docs/agents/manage-apm-agents/app-naming/name-your-application#differences) votre à partir de la <DNT>**Application**</DNT> page des paramètres dans l&amp;apos;UI de New Relic. Cela est utile, par exemple, pour donner à votre application un nom différent, tout en conservant les données historiques sous le nouvel alias. Pour plus d&amp;apos;informations, voir [Nommez votre application](/docs/agents/manage-apm-agents/app-naming/name-your-application).

Le C SDK de New Relic ne prend pas en charge [la configuration côté serveur](/docs/agents/manage-apm-agents/configuration/server-side-agent-configuration). Cependant, vous pouvez également utiliser cette page de paramètres de <DNT>**Application**</DNT> dans l&amp;apos;UI pour définir le seuil [Apdex T](/docs/apm/new-relic-apm/apdex/apdex-measure-user-satisfaction) de votre application.

Pour modifier l&apos;alias de application ou le seuil Apdex T dans l&apos;UI accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; services &amp;gt; (select an app) &amp;gt; Settings &amp;gt; Application**</DNT>.

## Modifier le nom de l&apos;application dans la configuration [#change-app-name]

Si vous modifiez le nom de votre application dans vos paramètres configuration , la même application apparaîtra dans l&apos;UI avec un nouveau nom. Toutes les données historiques (basées sur le calendrier [de conservation des données](/docs/apm/new-relic-apm/maintenance/apm-data-retention) ) n&amp;apos;existeront que sous l&amp;apos;ancien nom. (Pour renommer votre application tout en conservant les données historiques, utilisez les [paramètresUI ](#change-ui)pour modifier l&amp;apos;alias.)

Si vous devez modifier le nom de votre application dans votre configuration une fois votre application connectée au démon :

1. Créez un nouveau `config` avec un appel à [`newrelic_create_app_config()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#af067b3ff89a40b6fa417e18be5e847aa) en utilisant le nouveau nom de l&amp;apos;application.
2. Créez une nouvelle application connectée en appelant [`newrelic_create_app()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a61dd90439ae3cc5060021f6ab4701132).

<Callout variant="caution">
  Le timing est primordial. Le changement de nom application au cours d&apos;une exécution application unique peut signifier que vos données instrumentées sont envoyées sous le nouveau nom application .
</Callout>
