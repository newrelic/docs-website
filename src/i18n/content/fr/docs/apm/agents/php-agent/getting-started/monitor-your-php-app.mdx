Vous souhaitez commencer monitoring votre application PHP avec New Relic dès maintenant ? Utilisez ce guide pour trouver le meilleur chemin pour commencer à envoyer vos données d&apos;application à New Relic.

Vous souhaitez en savoir plus avant de commencer ? [Introduction à APM](/docs/apm/new-relic-apm/getting-started/introduction-apm) et [Introduction à New Relic pour PHP](/docs/apm/agents/php-agent/getting-started/introduction-new-relic-php) sont un bon point de départ.

## Que souhaitez-vous monitorer ? [#what]

Nous avons quelques chemins pour installer l&apos; agent PHP, en fonction de ce que vous monitoring.

<CollapserGroup>
  <Collapser id="docker" title="Application déployée dans un conteneur Docker">
    Vous disposez de quelques options pour instrumenter votre application Docker avec l&apos; agent PHP.

    * Installez via notre [guide d&apos;installation](https://one.newrelic.com/marketplace?state=38549f24-54f0-9e80-e910-0c42a9b493b1) si vous souhaitez que votre daemon et votre agent PHP soient dans des conteneurs différents. Si vous devez encore créer votre daemon, suivez les étapes supplémentaires dans [Docker et d&apos;autres environnements de conteneurs](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#daemon-container)
    * Pour configurer l&apos;agent et le daemon dans le même conteneur, suivez [ces étapes](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#install-same-container). Cela implique de télécharger l&amp;apos; [archive tar](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file/) pour l&amp;apos;agent PHP.
  </Collapser>

  <Collapser id="redhat-centos" title="Application déployée avec RedHat, CentOS ou AWS Linux">
    * Notre [installation guidée](https://one.newrelic.com/marketplace?state=38549f24-54f0-9e80-e910-0c42a9b493b1) utilise `yum` pour installer l&amp;apos;agent, tout comme notre installation manuelle. L&amp;apos;installation guidée est une excellente option si vous souhaitez plus de visibilité et de certitude sur l&amp;apos;endroit où vous en êtes dans le processus d&amp;apos;installation.
    * Notre [installation manuelle](/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos#manager-tarball) vous guide à travers le même processus, mais vous permet de gérer chaque étape depuis votre ligne de commande.
  </Collapser>

  <Collapser id="debian-ubuntu" title="Application déployée avec Debian ou Ubuntu">
    * Notre [installation guidée](https://one.newrelic.com/marketplace?state=38549f24-54f0-9e80-e910-0c42a9b493b1) utilise `apt` pour installer l&amp;apos;agent, tout comme notre installation manuelle. L&amp;apos;installation guidée est une excellente option si vous souhaitez plus de visibilité et de certitude sur l&amp;apos;endroit où vous en êtes dans le processus d&amp;apos;installation.
    * Notre [installation manuelle](/docs/apm/agents/php-agent/installation/php-agent-installation-ubuntu-debian) vous guide à travers le même processus, mais vous permet de gérer chaque étape depuis votre ligne de commande.
  </Collapser>

  <Collapser id="tar" title="Application déployée avec d'autres saveurs Unix (en utilisant l'archive tar)">
    Si vous utilisez une version différente d&apos;Unix, installez l&apos;agent en utilisant notre [archive tar PHP](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file).
  </Collapser>

  <Collapser id="hosting-providers" title="Application hébergée sur AWS, Heroku, Magento ou d'autres fournisseurs d'hébergement">
    Grâce à des partenariats avec New Relic, vous pouvez exécuter l&apos;agent PHP si votre application Web est hébergée sur un fournisseur tiers.

    * Pour les applications hébergées par AWS, installez l&apos; [intégration Amazon flux de métriques CloudWatch ](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup). Pour en savoir plus sur l’intégration New Relic et AWS, lisez [Introduction à l’intégration AWS ](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations/).
    * Pour une application PHP hébergée par Heroku, installez le [module complémentaire New Relic](/docs/apm/agents/php-agent/advanced-installation/php-agent-heroku/#installing)
    * Si vous êtes un commerçant sur Magento, [suivez ces étapes](/docs/accounts/install-new-relic/partner-based-installation/log-install-new-relic-partners/#magento) pour déployer l&amp;apos;agent PHP sur votre installation Magento.
  </Collapser>
</CollapserGroup>

## Quelle est la prochaine étape ? [#next]

Vous avez donc instrumenté votre application et souhaitez les étapes suivantes. Vous pourriez envisager :

* [Configuration de l&apos;agent PHP](/docs/apm/agents/php-agent/configuration/php-agent-configuration/)
* [Configuration du daemon proxy de l&apos;agent PHP](/docs/apm/agents/php-agent/configuration/proxy-daemon-newreliccfg-settings/)
* Apprenez-en plus sur le score Apdex de votre application pour obtenir [un résumé de haut niveau de votre application](/docs/apm/apm-ui-pages/monitoring/apm-summary-page-view-transaction-apdex-usage-data/)
* Connecter vos données de log et APM avec [les logs PHP en contexte](/docs/logs/logs-context/configure-logs-context-php/)
