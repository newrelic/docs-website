## Problème

Le module <DNT>**newrelic**</DNT> apparaît dans votre page `phpinfo()` mais aucune donnée n&apos;apparaît dans votre UI et votre <InlinePopover type="licenseKey" />n&apos;y apparaît pas.

## Solution

Le fichier <DNT>**`newrelic.ini`**</DNT> n&apos;a peut-être pas été lu à partir du fichier de configuration car il peut ne pas apparaître dans un répertoire où PHP s&apos;attend à ce qu&apos;il soit. Pour déterminer où votre `newrelic.ini` doit apparaître :

1. Dans votre [page`phpinfo()` ](/docs/php/php-kb-103), regardez la première section de la page.
2. Vérifiez si `newrelic.ini` est répertorié dans la section intitulée <DNT>**Additional .ini files parsed**</DNT>
3. Vérifiez que votre fichier `newrelic.ini` existe dans le répertoire identifié dans la section intitulée <DNT>**Scan this dir for addition .ini files**</DNT> ou <DNT>**Additional .ini files parsed**</DNT> et vérifiez que les autorisations sont adaptées à votre serveur Web pour le lire.

Si `newrelic.ini` n&apos;est pas le nom de votre fichier configuration, vérifiez que le fichier dans lequel vous avez ajouté votre clé de licence et d&apos;autres paramètres configuration est répertorié comme ayant été lu. Vérifiez que les autorisations sur le fichier fournissent l’accès à votre serveur Web.
