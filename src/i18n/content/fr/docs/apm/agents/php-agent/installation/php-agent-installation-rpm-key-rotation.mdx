À partir de mars 2026, pour des raisons de sécurité et de bonnes pratiques, les nouvelles versions des paquets RPM de l&apos;agent PHP New Relic seront signées avec une nouvelle clé. Pour assurer le fonctionnement ininterrompu de l&apos;agent PHP, avant d&apos;installer ou de mettre à jour l&apos;agent PHP New Relic à partir de paquets RPM, la commande suivante doit être exécutée pour accepter et importer la nouvelle clé publique.

`rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public`

## Vérifier la clé

Une fois la clé importée, vérifiez qu&apos;elle est de confiance en exécutant la commande suivante :

`rpm -qa | grep gpg-pubkey-bd2e199c`

Si la clé est importée et approuvée, la commande renvoie la sortie suivante :

`gpg-pubkey-bd2e199c-6970e650`

## Que se passe-t-il si vous n&apos;importez pas la nouvelle clé avant mars ?

Si la nouvelle clé n&apos;est pas ajoutée à votre système, les installations et les mises à jour des nouvelles versions de l&apos;agent PHP échoueront, avec un message d&apos;erreur tel que `Error: GPG check FAILED`. Les installations épinglées sur des versions antérieures de l&apos;Agent continueront de fonctionner comme auparavant (bien que, dans ce cas, il soit tout de même recommandé d&apos;importer la nouvelle clé).
