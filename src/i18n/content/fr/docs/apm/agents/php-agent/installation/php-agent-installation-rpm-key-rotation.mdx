En mars 2026, pour des raisons de bonnes pratiques et de sécurité, les nouvelles versions des paquets RPM de l&apos;agent PHP New Relic ont été signées avec une nouvelle clé. Pour que l&apos;agent PHP continue de fonctionner sans interruption, avant d&apos;installer ou de mettre à niveau l&apos;agent PHP New Relic à partir de paquets RPM, la commande suivante doit être exécutée pour accepter et importer la nouvelle clé publique.

`rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public`

## Vérifier la clé

Une fois la clé importée, vérifiez qu&apos;elle est de confiance en exécutant la commande suivante :

`rpm -qa | grep gpg-pubkey-bd2e199c`

Si la clé est importée et approuvée, la commande renvoie la sortie suivante :

`gpg-pubkey-bd2e199c-6970e650`

## Que se passe-t-il si vous n&apos;avez pas importé la nouvelle clé ?

Si la nouvelle clé n&apos;est pas ajoutée à votre système, les installations et les mises à niveau des nouvelles versions de l&apos;Agent PHP échoueront, avec un message d&apos;erreur tel que `Error: GPG check FAILED`. Toutes les installations épinglées sur des versions antérieures de l&apos;agent continueront de fonctionner comme avant (bien qu&apos;il soit tout de même recommandé d&apos;importer la nouvelle clé dans de tels cas).
