## Problème

Vous avez mis à jour PHP vers une [version de sortie prise en charge](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release), mais l&apos; agent PHP a cessé de fonctionner ou de générer des rapports.

## Solution

Si vous mettez à jour votre installation PHP vers une autre version majeure (par exemple, 5.3 vers 5.4), vous devez réinstaller l&apos; agent PHP. La mise à jour de PHP vers des versions mineures ne nécessite pas de réinstallation de l&apos; agent.

Pour réinstaller l&apos;agent, suivez les [instructions d&apos;installation](/docs/agents/php-agent/installation/php-agent-installation-overview) de votre plateforme ou utilisez la commande `newrelic-install install` pour démarrer <DNT>**[newrelic-install script](/docs/agents/php-agent/advanced-installation/using-newrelic-install-script)**</DNT>.

Si vous utilisez un gestionnaire de paquets (par exemple,) `apt-get upgrade` pour mettre à jour PHP sans spécifier de paquet, vous pouvez également [mettre à niveau l&apos;agent PHP de New Relic](/docs/agents/php-agent/installation/upgrading-php-agent) dans le processus. Pour garantir que la mise à niveau est appliquée avec succès :

1. Redémarrez votre serveur Web (Apache, Nginx, PHP-FPM, etc.).
2. Attendez cinq minutes pour voir si l’agent commence à faire un rapport.
3. Si l&apos;agent ne génère toujours pas de rapport, [vérifiez la configuration du daemon](/docs/agents/php-agent/troubleshooting/verifying-php-daemon). S&apos;il est configuré pour démarrer en mode externe, vous devez [démarrer manuellement le daemon](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#daemon-external) pour que l&apos;agent commence à générer des rapports.
