Lisez la suite pour savoir comment désinstaller l’agent PHP New Relic. Pour obtenir des instructions sur la façon de désactiver temporairement l&apos;agent, voir [Désactiver l&apos;agent](/docs/agents/manage-apm-agents/installation/disable-apm-agent#php).

Ce document explique comment désinstaller l&apos;[agent PHP de New Relic](/docs/agents/php-agent/getting-started/new-relic-php) pour le système suivant :

* <DNT>RedHat</DNT> et <DNT>CentOS</DNT>
* <DNT>Ubuntu</DNT> et <DNT>Debian</DNT>
* Archive Tar (tout [système pris en charge](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems))

## Désinstallation de l&apos;agent [#install-overview]

Lors de la désinstallation de New Relic d&apos;une application PHP, vous devez supprimer l&apos;agent New Relic et également supprimer les fichiers de configuration New Relic associés.

Pour désinstaller l’agent PHP de votre système :

1. Supprimez les fichiers de l&apos;agent de votre système :

   <CollapserGroup>
     <Collapser id="remove-redhat-centos" title="RedHat ou CentOS">
       Exécutez le script `newrelic-install` sans options et sélectionnez l’option de désinstallation dans le menu principal lors de l’appel du script.
     </Collapser>

     <Collapser id="remove-ubuntu-debian" title="Ubuntu ou Debian">
       Utilisez `apt-get` pour supprimer l&apos;agent :

       ```bash
       apt-get remove newrelic-php5
       ```
     </Collapser>
   </CollapserGroup>

2. Supprimez tous les fichiers de configuration New Relic restants. Utilisez les commandes `newrelic-install` ou `apt-get remove` avec l&apos;option `purge` .

   ```bash
   newrelic-install purge
   ```

   OU

   ```bash
   apt-get remove --purge newrelic
   ```

3. Si vous utilisez un gestionnaire de paquets, supprimez les fichiers spécifiques au gestionnaire :

   <CollapserGroup>
     <Collapser id="uninstall-redhat-centos" title="RedHat ou CentOS">
       En utilisant `yum`:

       ```bash
       sudo yum remove newrelic-php5
       ```

       En utilisant <DNT>**`rpm` (32-bit)**</DNT>:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (YOUR VERSION #)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```

       En utilisant <DNT>**`rpm` (64-bit)**</DNT>:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (VERSION NUMBER)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser id="uninstall-ubuntu-debian" title="Ubuntu ou Debian">
       En utilisant `apt-get`:

       ```bash
       sudo apt-get purge newrelic-php5
       ```

       En utilisant <DNT>**`dpkg` (32-bit)**</DNT>:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_i386.deb newrelic-php5_X.X.X.X_i386.deb
       ```

       En utilisant <DNT>**`dpkg` (64-bit)**</DNT>:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_amd64.deb newrelic-php5_X.X.X.X_amd64.deb
       ```
     </Collapser>
   </CollapserGroup>

4. Redémarrez vos serveurs web (Apache, Nginx, PHP-FPM, etc.).
