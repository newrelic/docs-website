## Problème

<DNT>**newrelic**</DNT> la section du module n&apos;apparaît pas dans `phpinfo()`.

## Solution

Une section de description de module `newrelic` manquante dans la page `phpinfo()` indique que le fichier <DNT>**`newrelic.so`**</DNT> n&apos;a peut-être pas été installé dans le répertoire des extensions. Pour déterminer si le module PHP New Relic existe :

1. Dans votre [page`phpinfo()` ](/docs/php/php-kb-103), recherchez la section <DNT>**PHP Core**</DNT> .
2. Pour identifier le répertoire à partir duquel vos extensions PHP sont chargées, recherchez la valeur locale du paramètre `extension_dir` .
3. Vérifiez que `newrelic.so` est présent dans le répertoire spécifié par `extension_dir`.

Si le fichier `newrelic.so` n&apos;est pas présent dans votre répertoire d&apos;extensions, soit :

* Suivez [les procédures d&apos;installation PHP non standard](/docs/php/php-agent-installation-non-standard-php) de New Relic.

  OU

* Créez manuellement un lien vers la version correcte à partir de `/usr/lib/newrelic-php5/agent/x32` pour 32 bits ou `/usr/lib/newrelic-php5/agent/x64` pour 64 bits. Ou, si vous avez installé à partir d&apos;un fichier tar, copiez ou créez un lien à partir du répertoire <DNT>**agent**</DNT> où vous avez extrait le fichier tar.
