Si vous utilisez pour gérer et exécuter vos tests unitaires,[PHPUnit](https://phpunit.de/) l&amp;apos;agent PHP New Relic peut capturer automatiquement les résultats du résumé des tests et les envoyer à [New Relic](/docs/insights/new-relic-insights/understanding-insights/new-relic-insights) sous forme d&amp;apos;[événement où vous pouvez interroger et visualiser les données de test en un coup d&apos;œil.](/docs/insights/new-relic-insights/understanding-insights/new-relic-insights#database) Cette fonctionnalité a été sortie dans la version <DNT>**6.7.0**</DNT> de l&amp;apos;agent PHP et prend en charge les versions PHPUnit <DNT>**3.7**</DNT> à <DNT>**8.x**</DNT>

## Activer l&apos;événement de test PHPUnit [#phpunit-enable]

Pour activer l&apos;événement de test PHPUnit :

1. Recherchez ou ajoutez le paramètre `newrelic.phpunit_events.enabled` dans votre fichier `newrelic.ini` et définissez-le sur `true`.
2. Redémarrez votre serveur Web (Apache, PHP-FPM, Nginx, etc.) pour que la modification des paramètres prenne effet.

<Callout variant="tip">
  Les messages d&apos;exception sont collectés et envoyés avec l&apos;événement. De plus, si vous utilisez l&apos;option `--disallow-test-output` de PHPUnit, qui signale les tests qui impriment une sortie comme « risqués », l&amp;apos;événement de test inclura la sortie incriminée.
</Callout>

## Afficher l&apos;attribut disponible [#phpunit-view-attributes]

Lorsqu&apos;il [est activé](#phpunit-enable), l&amp;apos;agent PHP détecte les commandes PHPUnit et remplit New Relic avec deux types d&amp;apos;événements contenant des données pour la suite de tests (nommée [`TestSuite`](#testsuite-event-attributes)) et les tests individuels (nommés [`Test`](#test-event-attributes)). Vous pouvez interroger les données avec [la requêteNRQL ](/docs/insights/new-relic-insights/using-new-relic-query-language/using-nrql)et [créer un dashboard](/docs/insights/new-relic-insights/managing-dashboards-data/building-insights-dashboards) des informations importantes pour vous.

Pour interroger votre événement de test, utilisez `FROM TestSuite` et `FROM Test` lors de la spécification de vos instructions `SELECT` :

```sql
SELECT uniqueCount(name) FROM TestSuite
```

Un dashboard des données récapitulatives des tests vous permet de voir rapidement un aperçu de la réussite globale des tests ainsi que d&apos;explorer les suites défaillantes pour déterminer leur cause. Ces exemples illustrent les types de widgets que vous pouvez créer avec les types d&apos;événements `TestSuite` et `Test` .

## Facettes liées

Étant donné qu&apos;une suite de tests PHPUnit est liée à des tests individuels via son ID d&apos;exécution, vous pouvez utiliser [le widget`FACET` pour filtrer les résultats](/docs/insights/new-relic-insights/managing-dashboards-data/editing-insights-dashboards#facet-linking) d&amp;apos;une exécution de test spécifique.

Par exemple, si vous avez créé un widget avec les suites de tests les plus récentes ayant échoué et que vous l&apos;avez lié au dashboard actuel, vous pouvez cliquer sur un test et le widget environnant sera mis à jour avec les informations correspondant uniquement à cette exécution de suite de tests.

<img title="Screenshot of dashboard showing linked facets" alt="Screenshot of dashboard showing linked facets" src="/images/apm_screenshot-crop_php-linked-facets-dashboard.webp" />

<figcaption>
  widget FACET (en haut à gauche) lié au dashboard actuel vous permet d&apos;explorer en profondeur les suites de tests en échec.
</figcaption>

Dans l&apos;exemple ci-dessus, vous pouvez voir qu&apos;en cliquant sur Exécuter `5bb37ccee2a1dbc7`, nous apprenons que l&amp;apos;un des deux tests, `testFoo`, a échoué. Voici les requêtes NRQL qui composent cet exemple :

<DNT>
  **Recent unsuccessful suites:**
</DNT>

```sql
SELECT latest(timestamp), latest(name) FROM TestSuite WHERE successful IS false FACET runId
```

<DNT>
  **Last unsuccessful suite:**
</DNT>

```sql
SELECT host, name, duration * 1000 
AS 'duration (ms)', assertionCount, testCount, passedCount, failedCount, incompleteCount, skippedCount, errorCount, riskyCount, warningCount 
FROM TestSuite WHERE successful IS false LIMIT 1
```

<DNT>
  **Last unsuccessful test:**
</DNT>

```sql
SELECT host, name, testSuiteName, duration * 1000 
AS 'duration (ms)', outcome, assertionCount, message 
FROM Test WHERE outcome != 'passed' LIMIT 1
```
