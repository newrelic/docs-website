## Problème

Vous avez installé l&apos;agent PHP New Relic et généré du trafic, mais l&apos;agent n&apos;envoie pas la première transaction.

## Solution

Il s’agit d’un comportement attendu. Lorsque le [daemon PHP](/docs/agents/php-agent/getting-started/new-relic-daemon-processes) est dans son état initial, <DNT>**two**</DNT> transactions doivent se produire avant qu&apos;il ne commence à générer des rapports :

* Avec la première transaction, le daemon PHP ajoute votre nouvelle application de rapport à sa propre table d&apos;informations d&apos;application.
* Une fois l&apos;application ajoutée, le daemon PHP signale la deuxième transaction à New Relic.

Recommandation : exécutez un processus PHP trivial, tel que `phpinfo()`, avant d&apos;exécuter les processus que vous souhaitez signaler.

## Cause

Il s&apos;agit du comportement attendu du daemon PHP, qui nécessite initialement que deux transactions soient signalées.
