## architecture

L'agent PHP dispose de deux binaires qui fonctionnent ensemble pour transmettre des données à New Relic :

* L'agent gère l'instrumentation automatique et API de votre code PHP.
* Le daemon agit comme un proxy entre l'agent et la plateforme New Relic.

Vous pouvez connecter jusqu'à 500 applications/agents à un daemon. Le daemon impose [un échantillonnage](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling) lorsque les limites du cycle de collecte sont atteintes, alors tenez-en compte lorsque vous décidez du nombre d&amp;apos;applications/agents à connecter à un seul daemon.

<Callout variant="tip">
  Le nombre d'applications/agents par daemon peut être inférieur lors de l'exécution dans un conteneur Docker séparé, en fonction de la capacité de la connexion entre les conteneurs.
</Callout>

<img title="PHP agent diagram" alt="This diagram shows the flow of data from the PHP agent to New Relic." src="/images/apm_diagram_php-agent-diagram.webp" />

<figcaption>
  Le flux de données de votre application PHP vers New Relic.
</figcaption>

Le workflow entre votre application et New Relic doit se dérouler dans cet ordre :

1. L'agent établit une connexion socket avec le daemon en envoyant la première charge utile de données d'instrumentation.
2. Le daemon établit un lien HTTPS avec la plateforme New Relic. Le daemon doit être invoqué avant que votre application instrumentée ne soit invoquée. Ceci est appelé [mode agent](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced) et constitue la valeur par défaut.

Pour éviter de perdre les données signalées, assurez-vous que votre application instrumentée n'envoie pas de transactions avant que les deux connexions ne soient établies.

## Installer l'agent [#installation]

Avant d'installer l' agent PHP, assurez-vous que votre système répond à l' [exigence système](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements). L&amp;apos; agent PHP prend en charge la plupart des [frameworks PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#frameworks), [bases de données](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#databases) et [bibliothèques](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#databases) les plus courants. Vous pouvez également utiliser l&amp;apos;agent dans un [environnement flexible Google App Engine (GAE)](/docs/agents/php-agent/advanced-installation/install-new-relic-php-agent-gae-flexible-environment).

<Callout variant="tip">
  Si vous [installez l'agent sur un service d'hébergement partagé](/docs/agents/php-agent/installation/install-php-agent-shared-hosting-service), assurez-vous que vous disposez des autorisations root pour installer l&amp;apos;agent ou contactez votre fournisseur d&amp;apos;hébergement pour obtenir une assistance technique.
</Callout>

Pour installer l'agent, [inscrivez-vous d'abord sur New Relic](https://newrelic.com/signup) . Une fois connecté, utilisez notre lanceur ou consultez les instructions pour une installation spécifique.

## Configurer l'agent [#configuration]

L'agent comprend une variété d' [options de configuration](/docs/agents/php-agent/configuration/php-agent-configuration) pour personnaliser et affiner davantage votre installation.

<Callout variant="tip">
  La partie la plus importante de la configuration de l'agent est de donner à votre application un [nom descriptif](/docs/agents/php-agent/configuration/php-agent-configuration#inivar-appname). New Relic [utilise ce nom d'application comme métrique agrégée](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name) lorsque vous avez plusieurs applications ou hôtes.
</Callout>

Après avoir modifié les options de configuration de l’agent, redémarrez votre serveur Web.

## Prolonger instrumentation d' agent [#extend]

Après avoir installé l'agent, allez plus loin et étendez l'instrumentation de l'agent :

* [Temps de chargement de la page](/docs/agents/php-agent/features/page-load-timing-php): intégrez l&amp;apos;agent PHP avec [<InlinePopover type="browser" />](/docs/browser/new-relic-browser/getting-started/new-relic-browser)pour obtenir une visibilité sur l&amp;apos;activité de l&amp;apos;utilisateur final.
* [Instrumentation personnalisée](/docs/agents/php-agent/features/php-custom-instrumentation): Instrumenter le transactions sur non prises en compte dans le framework de notre instrumentation.
* [API d'agent](/docs/agents/php-agent/configuration/php-agent-api): Utilisez l&amp;apos; API d&amp;apos; agent pour personnaliser le comportement de l&amp;apos; agent. Par exemple, vous pouvez collecter des mesures personnalisées, signaler une erreur ou ignorer complètement une transaction particulière.
* [attribut agent](/docs/agents/php-agent/attributes/php-agent-attributes): Personnaliser l&amp;apos; [attribut](/docs/agents/manage-apm-agents/agent-metrics/agent-attributes) attaché aux transactions. La personnalisation des attributs vous permet d&amp;apos;éviter d&amp;apos;envoyer des attributs sensibles ou de collecter des attributs supplémentaires pour une visibilité plus approfondie de vos transactions.
