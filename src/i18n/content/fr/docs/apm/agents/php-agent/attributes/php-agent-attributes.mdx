[Les attributs](/docs/features/agent-attributes) sont des paires valeur clé contenant des informations qui déterminent les propriétés d&apos;un événement ou d&apos;une transaction. Ces paires valeur-clé peuvent vous aider à obtenir des informations plus détaillées sur votre application et à annoter les données dans New Relic. Les attributs par défaut et [personnalisés](/docs/agents/manage-apm-agents/agent-data/collect-custom-attributes) sont visibles dans la trace de transaction APM, les traces distribuées et les analyses d&apos;erreurs ; ainsi que dans l&apos;événement [APM](/docs/insights/explore-data/attributes/apm-default-attributes-insights) et [le navigateur](/docs/insights/explore-data/attributes/browser-default-attributes-insights#browser-attributes-table).

Vous pouvez personnaliser exactement quel attribut sera envoyé à chacune de ces destinations. Lisez la suite pour en savoir plus sur l&apos;attribut PHP agent, comment les activer ou les désactiver et les règles que l&apos; agent suit pour déterminer quel attribut inclure ou exclure pour une [destination](/docs/features/agent-attributes#destinations).

## Modifier l&apos;endroit où les attributs sont envoyés [#change-attribute-destination]

<Callout variant="tip">
  Toutes les destinations sont ouvertes à la collecte d&apos;attributs par défaut dans l&apos; agent PHP, sauf `newrelic.browser_monitoring.attributes.enabled`.
</Callout>

Pour modifier les attributs envoyés aux New Relic [destinations](/docs/apm/other-features/attributes/agent-attributes#destinations):

1. Ouvrez ou fermez n&apos;importe quelle destination pour attribuer une collection en modifiant la [propriété de destination`.enabled` ](/docs/agents/php-agent/attributes/enabling-disabling-attributes#destination-on-off).

2. Modifiez le [paramètre d&apos;attribut par défaut](#attributes) en ajoutant le nom de l&apos;attribut aux [propriétés`.include` ou `.exclude` ](/docs/agents/php-agent/attributes/enabling-disabling-attributes#destination-attribute-specific)de la destination.

   Consultez [les règles d&apos;attribut](/docs/agents/php-agent/attributes/enabling-disabling-attributes#attruls) pour savoir quels paramètres se remplacent les uns les autres.

## Mise à niveau de l&apos;agent PHP [#upgrading]

Lors de la mise à niveau vers l&apos;agent PHP <DNT>**4.9**</DNT> ou supérieur, mettez à niveau votre fichier de configuration. Pour plus d&apos;informations sur les propriétés obsolètes, voir [Activation et désactivation de l&apos;attribut](/docs/php/php-agent-enabling-and-disabling-attributes#deprecated).
