## Problème

Vous devez déterminer les autorisations système dont vous avez besoin pour installer et utiliser l&apos;agent PHP New Relic.

## Solution

Pour la plupart des cas d&apos;utilisation, vous aurez besoin d&apos;un accès root pour installer New Relic pour PHP. Cela est dû au fait que le processus d’installation nécessite des privilèges uniquement disponibles pour l’utilisateur root. Ces privilèges incluent :

* Accès en lecture/écriture au répertoire des extensions PHP
* Accès en lecture/écriture aux données de configuration PHP, soit dans le répertoire de configuration PHP (préféré), soit dans <DNT>**php.ini**</DNT>.
* Possibilité de redémarrer le serveur Web ou le pool de processus PHP.

L’exécution de l’agent ne nécessite pas d’accès root. L&apos;agent a besoin d&apos;un accès en lecture/écriture aux fichiers suivants (l&apos;emplacement de ces fichiers peut être [personnalisé dans les paramètres de l&apos;agent PHP](/docs/php/php-agent-phpini-settings)) :

* [Un fichier dans lequel stocker le agent fichier de de l&apos;log](/docs/php/php-agent-phpini-settings#inivar-daemon-logfile)
* [Un fichier dans lequel stocker le point de terminaison du socket de l&apos;agent](/docs/php/php-agent-phpini-settings#inivar-daemon-port)
* [Un fichier dans lequel enregistrer l&apos;ID de processus de l&apos;agent](/docs/php/php-agent-phpini-settings#inivar-daemon-pidfile)
