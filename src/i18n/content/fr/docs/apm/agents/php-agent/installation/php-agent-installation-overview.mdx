## Composants PHP [#components]

Notre [agent PHP](/docs/agents/php-agent/getting-started/new-relic-php) se compose de deux composants de base :

* Une extension PHP, qui collecte les données de votre application
* Un daemon proxy local, qui transmet les données à New Relic

Les composants configuration de votre système, notamment le serveur Web, le système d'exploitation, les autorisations du système, installation PHP elle-même, le gestionnaire de paquets, etc., sont tous des facteurs importants lors de installation et configuration, car l' agent en fait partie.

Pour plus d'informations sur les sorties PHP prises en charge, le framework et d'autres exigences système, consultez [Compatibilité et exigences PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).

## Installer l'agent [#install-overview]

Suivez ce processus général pour installer l'agent sur votre système. Le package d'installation de l'agent ou l'archive tar inclut le [script interactif<DNT>**`newrelic-install`**</DNT> ](/docs/agents/php-agent/installation/newrelic-install-script), qui automatise certaines tâches d&amp;apos;installation.

<Callout variant="tip">
  Si votre application est fréquemment inactive, nous vous recommandons de générer certaines données en interagissant avec l'application ou le site. Cela permettra de garantir que l’agent se connecte initialement.
</Callout>

1. Si vous ne l'avez pas déjà fait, [créez un compte New Relic](https://newrelic.com/signup). C&amp;apos;est gratuit, pour toujours.

2. Assurez-vous que votre système répond aux [exigences de l'agent](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements), y compris [les autorisations système](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#permissions) appropriées.

3. Depuis votre <DNT>**Account settings**</DNT>, copiez vos informations <InlinePopover type="licenseKey" />.

4. Installez le package agent ou l'archive tar sur votre système.

   <CollapserGroup>
     <Collapser id="install-redhat-centos" title="RedHat ou CentOS">
       La manière la plus courante d'installer l'agent sur RedHat ou CentOS est d'utiliser le installation package <DNT>**.rpm**</DNT>() et le gestionnaire de paquets. Vous pouvez [demander au gestionnaire de paquets](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#tell-rpm) de l&amp;apos;obtenir à partir du référentiel New Relic, ou vous pouvez [télécharger le fichier <DNT>**.rpm**</DNT> ](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#download-rpm)directement à partir du site Web New Relic.

       [Installez le package](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos) avec votre gestionnaire de paquets préféré (`yum` ou `rpm`) puis exécutez le script <DNT>**newrelic-install**</DNT>.
     </Collapser>

     <Collapser id="intall-ubuntu-debian" title="Ubuntu ou Debian">
       [Installez le package](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian) avec les commandes `apt-get` ou `dpkg` . En règle générale, l’exécution du script <DNT>**newrelic-install**</DNT> n’est pas requise.
     </Collapser>

     <Collapser id="install-tar" title="Archives de goudron">
       La méthode d'archivage <DNT>**.tar**</DNT> est la méthode la plus générique que vous pouvez utiliser sur tous [les systèmes d'exploitation pris en charge](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems). [Téléchargez l'archive tar compressée](/docs/agents/php-agent/installation/php-agent-installation-tar-file#download) pour votre système si l&amp;apos;une de ces situations s&amp;apos;applique à vous :

       * Votre système d'exploitation n'est pas répertorié ici.

       * Vous n'utilisez pas de gestionnaire de paquets.

       * D’autres méthodes ne donnent aucun résultat.

         [Installez l'archive tar](/docs/agents/php-agent/installation/php-agent-installation-tar-file#install) de l&amp;apos; agent sur n&amp;apos;importe quelle [plateforme prise en charge](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).
     </Collapser>
   </CollapserGroup>

5. Modifiez le [nom de l’application](/docs/site/naming-your-application) par défaut par un nom significatif.

6. Facultatif : modifiez d’autres paramètres [de configuration](/docs/agents/php-agent/configuration/php-agent-configuration) de l’agent pour personnaliser davantage votre installation.

7. Redémarrez votre serveur Web (Apache, Nginx, PHP-FPM, etc.).

8. Recommandation : pour garantir que l'agent PHP est lancé, en particulier si votre application a une activité peu fréquente, générez des données en utilisant l'application pendant quelques secondes.

9. Attendez quelques minutes que votre application envoie des données à New Relic.

Ensuite, [vérifiez les performances de votre application dans New Relic UI](/docs/apm/applications-menu/monitoring/apm-overview-page) l&amp;apos; .

## Afficher les logs de vos données APM et infrastructure [#logs-context]

Vous pouvez également rassembler vos données de log et d' application pour rendre le dépannage plus facile et plus rapide. Avec [les logs en contexte](/docs/logs/logs-context/configure-logs-context-php/), vous pouvez voir les messages de log liés à vos erreurs et les tracer directement dans UI de votre application. Vous pouvez également voir les logs en contexte de vos [données infrastructure ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), telles que le cluster Kubernetes. Pas besoin de passer à une autre page UI .

## Installer PHP avec des comptes de partenariat [#partnership-accounts]

Les procédures d'installation peuvent être différentes pour les administrateurs qui installent l'agent PHP via un partenariat New Relic. Pour plus d'informations, voir :

* [AWS](/docs/accounts-partnerships/partnerships/partner-based-installation/amazon-web-services-aws-users)
* [Heroku](/docs/agents/php-agent/installation/php-agent-heroku)
* [Magento](/docs/accounts-partnerships/partnerships/partner-based-installation/log-install-new-relic-partners#magento)
* [Fournisseurs d'hébergement partagé](/docs/agents/php-agent/installation/install-php-agent-shared-hosting-service)
* [Autres procédures d'installation de partenariat](/docs/accounts-partnerships/partnerships/partner-based-installation/log-install-new-relic-partners)

Tous les partenaires ne prennent pas en charge l'agent PHP.
