## Composants PHP [#components]

Notre [agent PHP](/docs/agents/php-agent/getting-started/new-relic-php) se compose de deux composants de base :

* Une extension PHP, qui collecte les données de votre application
* Un daemon proxy local, qui transmet les données à New Relic

Les composants configuration de votre système, notamment le serveur Web, le système d&apos;exploitation, les autorisations du système, installation PHP elle-même, le gestionnaire de paquets, etc., sont tous des facteurs importants lors de installation et configuration, car l&apos; agent en fait partie.

Pour plus d&apos;informations sur les sorties PHP prises en charge, le framework et d&apos;autres exigences système, consultez [Compatibilité et exigences PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).

## Installer l&apos;agent [#install-overview]

Suivez ce processus général pour installer l&apos;agent sur votre système. Le package d&apos;installation de l&apos;agent ou l&apos;archive tar inclut le [script interactif<DNT>**`newrelic-install`**</DNT> ](/docs/agents/php-agent/installation/newrelic-install-script), qui automatise certaines tâches d&apos;installation.

<Callout variant="tip">
  Si votre application est fréquemment inactive, nous vous recommandons de générer certaines données en interagissant avec l&apos;application ou le site. Cela permettra de garantir que l’agent se connecte initialement.
</Callout>

1. Si vous ne l&apos;avez pas déjà fait, [créez un compte New Relic](https://newrelic.com/signup). C&apos;est gratuit, pour toujours.

2. Assurez-vous que votre système répond aux [exigences de l&apos;agent](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements), y compris [les autorisations système](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#permissions) appropriées.

3. Depuis votre <DNT>**Account settings**</DNT>, copiez vos informations <InlinePopover type="licenseKey" />.

4. Installez le package agent ou l&apos;archive tar sur votre système.

   <CollapserGroup>
     <Collapser id="install-redhat-centos" title="RedHat ou CentOS">
       La manière la plus courante d&apos;installer l&apos;agent sur RedHat ou CentOS est d&apos;utiliser le installation package <DNT>**.rpm**</DNT>() et le gestionnaire de paquets. Vous pouvez [demander au gestionnaire de paquets](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#tell-rpm) de l&apos;obtenir à partir du référentiel New Relic, ou vous pouvez [télécharger le fichier <DNT>**.rpm**</DNT> ](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#download-rpm)directement à partir du site Web New Relic.

       [Installez le package](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos) avec votre gestionnaire de paquets préféré (`yum` ou `rpm`) puis exécutez le script <DNT>**newrelic-install**</DNT>.
     </Collapser>

     <Collapser id="intall-ubuntu-debian" title="Ubuntu ou Debian">
       [Installez le package](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian) avec les commandes `apt-get` ou `dpkg` . En règle générale, l’exécution du script <DNT>**newrelic-install**</DNT> n’est pas requise.
     </Collapser>

     <Collapser id="install-tar" title="Archives de goudron">
       La méthode d&apos;archivage <DNT>**.tar**</DNT> est la méthode la plus générique que vous pouvez utiliser sur tous [les systèmes d&apos;exploitation pris en charge](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems). [Téléchargez l&apos;archive tar compressée](/docs/agents/php-agent/installation/php-agent-installation-tar-file#download) pour votre système si l&apos;une de ces situations s&apos;applique à vous :

       * Votre système d&apos;exploitation n&apos;est pas répertorié ici.

       * Vous n&apos;utilisez pas de gestionnaire de paquets.

       * D’autres méthodes ne donnent aucun résultat.

         [Installez l&apos;archive tar](/docs/agents/php-agent/installation/php-agent-installation-tar-file#install) de l&apos; agent sur n&apos;importe quelle [plateforme prise en charge](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements).
     </Collapser>
   </CollapserGroup>

5. Modifiez le [nom de l’application](/docs/site/naming-your-application) par défaut par un nom significatif.

6. Facultatif : modifiez d’autres paramètres [de configuration](/docs/agents/php-agent/configuration/php-agent-configuration) de l’agent pour personnaliser davantage votre installation.

7. Redémarrez votre serveur Web (Apache, Nginx, PHP-FPM, etc.).

8. Recommandation : pour garantir que l&apos;agent PHP est lancé, en particulier si votre application a une activité peu fréquente, générez des données en utilisant l&apos;application pendant quelques secondes.

9. Attendez quelques minutes que votre application envoie des données à New Relic.

Ensuite, [vérifiez les performances de votre application dans New Relic UI](/docs/apm/applications-menu/monitoring/apm-overview-page) l&apos; .

## Afficher les logs de vos données APM et infrastructure [#logs-context]

Vous pouvez également rassembler vos données de log et d&apos; application pour rendre le dépannage plus facile et plus rapide. Avec [les logs en contexte](/docs/logs/logs-context/configure-logs-context-php/), vous pouvez voir les messages de log liés à vos erreurs et les tracer directement dans UI de votre application. Vous pouvez également voir les logs en contexte de vos [données infrastructure ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/), telles que le cluster Kubernetes. Pas besoin de passer à une autre page UI .
