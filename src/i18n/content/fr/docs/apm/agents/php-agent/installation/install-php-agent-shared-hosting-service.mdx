En général, les plans d'hébergement de base ne permettent pas l'installation standard de l'agent PHP de New Relic, car des autorisations d'administrateur (root) sont requises pour l'installer. Votre fournisseur d'hébergement peut autoriser ou fournir installation d'applications/frameworks Web courants (tels que WordPress, Drupal, Joomla, etc.), mais l'installation de l' agent PHP nécessite toujours des autorisations non autorisées dans de nombreux plans de base.

Si nécessaire, contactez les développeurs/hébergeurs de votre site Web pour obtenir une assistance technique.

## Services d'hébergement partagé [#hosts]

En général, les fournisseurs d'hébergement partagé (qui ne fournissent pas d'accès à la ligne de commande root) et les fournisseurs d'hébergement qui ne disposent pas de système d'exploitation traditionnel (tel que PaaS) ne permettent pas à l'utilisateur d'installer l'agent New Relic via les canaux classiques. Par exemple, ces hébergeurs ne prennent **pas** en charge l&amp;apos;intégration New Relic pour certains niveaux d&amp;apos;abonnement :

* DreamHost : Hébergement Web Partagé
* Blog hébergé sur WordPress.com
* GoDaddy : Économique, Deluxe, Ultimate, Hébergement de blog WordPress
* Hébergement Web HostGator

Avant d'installer l'agent PHP de New Relic, assurez-vous que votre niveau de service d'hébergement est compatible avec New Relic. Assurez-vous également que vous disposez des autorisations d’administrateur pour les procédures d’installation à partir de la ligne de commande au niveau racine.

## Conseils d'installation et de dépannage [#installation]

Pour garantir que votre application peut s'intégrer avec succès à New Relic :

1. Assurez-vous que votre système répond [aux exigences de compatibilité PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) de New Relic.

   <Callout variant="important">
     Les systèmes d'exploitation Windows ne sont pas pris en charge.
   </Callout>

2. Suivez [les procédures d'installation PHP](/docs/agents/php-agent/installation/php-agent-installation-overview) de New Relic.

3. Si votre système répond aux exigences de compatibilité de New Relic mais que votre installation échoue, rassemblez les informations suivantes :

   * Étapes exactes que vous avez essayées
   * Résultats, y compris les messages de réussite ou d'échec
   * Votre système d'exploitation et votre version (par exemple, Ubuntu Linux version 12.04 64 bits)
   * Votre hébergement prestataire de services et niveau d'abonnement

4. Le cas échéant, contactez les développeurs de votre site Web pour obtenir de l'aide.
