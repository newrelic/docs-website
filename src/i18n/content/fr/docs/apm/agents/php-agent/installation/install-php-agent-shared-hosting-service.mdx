En général, les plans d&apos;hébergement de base ne permettent pas l&apos;installation standard de l&apos;agent PHP de New Relic, car des autorisations d&apos;administrateur (root) sont requises pour l&apos;installer. Votre fournisseur d&apos;hébergement peut autoriser ou fournir installation d&apos;applications/frameworks Web courants (tels que WordPress, Drupal, Joomla, etc.), mais l&apos;installation de l&apos; agent PHP nécessite toujours des autorisations non autorisées dans de nombreux plans de base.

Si nécessaire, contactez les développeurs/hébergeurs de votre site Web pour obtenir une assistance technique.

## Services d&apos;hébergement partagé [#hosts]

En général, les fournisseurs d&apos;hébergement partagé (qui ne fournissent pas d&apos;accès à la ligne de commande root) et les fournisseurs d&apos;hébergement qui ne disposent pas de système d&apos;exploitation traditionnel (tel que PaaS) ne permettent pas à l&apos;utilisateur d&apos;installer l&apos;agent New Relic via les canaux classiques. Par exemple, ces hébergeurs ne prennent **pas** en charge l&amp;apos;intégration New Relic pour certains niveaux d&amp;apos;abonnement :

* DreamHost : Hébergement Web Partagé
* Blog hébergé sur WordPress.com
* GoDaddy : Économique, Deluxe, Ultimate, Hébergement de blog WordPress
* Hébergement Web HostGator

Avant d&apos;installer l&apos;agent PHP de New Relic, assurez-vous que votre niveau de service d&apos;hébergement est compatible avec New Relic. Assurez-vous également que vous disposez des autorisations d’administrateur pour les procédures d’installation à partir de la ligne de commande au niveau racine.

## Conseils d&apos;installation et de dépannage [#installation]

Pour garantir que votre application peut s&apos;intégrer avec succès à New Relic :

1. Assurez-vous que votre système répond [aux exigences de compatibilité PHP](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements) de New Relic.

   <Callout variant="important">
     Les systèmes d&apos;exploitation Windows ne sont pas pris en charge.
   </Callout>

2. Suivez [les procédures d&apos;installation PHP](/docs/agents/php-agent/installation/php-agent-installation-overview) de New Relic.

3. Si votre système répond aux exigences de compatibilité de New Relic mais que votre installation échoue, rassemblez les informations suivantes :

   * Étapes exactes que vous avez essayées
   * Résultats, y compris les messages de réussite ou d&apos;échec
   * Votre système d&apos;exploitation et votre version (par exemple, Ubuntu Linux version 12.04 64 bits)
   * Votre hébergement prestataire de services et niveau d&apos;abonnement

4. Le cas échéant, contactez les développeurs de votre site Web pour obtenir de l&apos;aide.
