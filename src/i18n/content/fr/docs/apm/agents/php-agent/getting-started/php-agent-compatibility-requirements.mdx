Consultez [les notes de sortie deagent PHP](/docs/release-notes/agent-release-notes/php-release-notes/) pour la dernière sortie agent PHP.

Pour une liste à jour de toutes les sorties d&apos;agent PHP actuellement prises en charge, consultez la [politique EOL de l&apos;agent PHP](/docs/apm/agents/php-agent/getting-started/php-agent-eol-policy/).

Si vous utilisez une version ou une plateforme PHP non prise en charge (comme un système d&apos;exploitation 32 bits, FreeBSD ou des versions ZTS), nous vous recommandons de désactiver les mécanismes de mise à jour automatique pour le package PHP agent . Si vous ne parvenez pas à empêcher les mises à niveau automatiques, cela pourrait entraîner l’installation accidentelle d’une version plus récente de l’agent. La version la plus récente peut ne plus prendre en charge les fonctionnalités critiques essentielles à la collecte de données APM, ce qui entraînera des perturbations dans les capacités monitoring .

Pour désactiver la mise à jour automatique, vous avez deux options :

* Ajoutez le package PHP agent à une liste d&apos;exclusion pour les mises à niveau package, ou

* Épinglez la version du agent PHP package sur une agent version qui prend en charge les anciennes fonctionnalités non prises en charge. Les packages agent PHP concernés sont :

  * `newrelic-php5`
  * `newrelic-php5-common`
  * `newrelic-daemon`

## Sécurité des threads Zend [#zts]

Les builds PHP compilées avec Zend Thread Safety (ZTS) ne sont pas prises en charge.

## Compilation juste à temps (JIT) [#jit]

[La compilation juste-à-temps](https://php.watch/articles/jit-in-depth) est prise en charge à partir de agent la version [10.18.0.8](/docs/release-notes/agent-release-notes/php-release-notes/php-agent-10-18-0-8/) de . PHP désactive automatiquement JIT pour toutes les versions antérieures de l&amp;apos;agent PHP.

## Fibres [#fibers]

[Les fibres](https://www.php.net/manual/en/language.fibers.php) ne sont pas prises en charge.

## Autorisations [#permissions]

Installation de l&apos;agent PHP <DNT>**requires**</DNT> accès root à :

* Accès en lecture/écriture au répertoire des extensions PHP.
* Accès en lecture/écriture aux données de configuration PHP, soit dans le répertoire de configuration PHP (préféré), soit dans `php.ini`.
* Redémarrez le serveur Web ou le pool de processus PHP.

Exécution de l&apos;agent <DNT>**does not require**</DNT> accès root. L&amp;apos;agent a besoin d&amp;apos;un accès en lecture/écriture aux fichiers suivants (l&amp;apos;emplacement de ces fichiers peut être [personnalisé dans les paramètres de l&apos;agent PHP](/docs/php/php-agent-phpini-settings)) :

* [Un fichier stockant le agent fichier de log](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)
* [Un fichier stockant le point de terminaison du socket de l&apos;agent](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-port)
* [Un fichier stockant l&apos;ID du processus de l&apos;agent](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-pidfile)

## système d&apos;exploitation [#operating-systems]

Lorsque les fournisseurs annoncent la fin de vie (comme sur [la page Fin du support standard d&apos;Ubuntu](https://wiki.ubuntu.com/Releases)), nous continuerons à prendre en charge ces dernières versions pendant un an après la date de fin de vie du fournisseur. Cependant, si la version PHP que vous utilisez n&amp;apos;est plus officiellement prise en charge, le support pourrait prendre fin avant un an.

C&apos;est pourquoi nous vous recommandons de toujours utiliser la dernière version du système d&apos;exploitation officiellement prise en charge par le fournisseur. Les dernières versions de notre agent peuvent fonctionner sur des versions de système d&apos;exploitation dont la fin de vie est dépassée, mais nous ne testons plus ni ne prenons officiellement en charge l&apos;agent PHP avec les anciennes versions.
