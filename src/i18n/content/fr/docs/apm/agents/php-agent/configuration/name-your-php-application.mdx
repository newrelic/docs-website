[New Relic pour PHP](/docs/agents/php-agent/getting-started/introduction-new-relic-php) vous offre plusieurs façons de modifier le nom de l&apos;application qui est signalé et affiché dans New Relic. Ce document vous donne un aperçu de ces options de dénomination.

Pour un aperçu général de la dénomination des applications APM (non spécifique à PHP), voir [Nommez votre application](/docs/agents/manage-apm-agents/app-naming/name-your-application).

## Définir ou modifier le nom de l&apos;application [#naming-options]

New Relic différencie les applications par leur nom. Lorsque vous modifiez le nom d&apos;une application, New Relic crée une nouvelle entité d&apos;application et interrompt la continuité des données entre la nouvelle et l&apos;ancienne entité. Pour modifier le nom d&apos;une application sans interrompre la continuité des données, utilisez l&apos;option UI alias-change.

Voici les moyens de modifier le nom de votre application PHP dans New Relic :

* [Configurer le fichier `newrelic.ini` à l’aide de `newrelic.appname`](/docs/agents/php-agent/configuration/php-agent-configuration#inivar-appname): l’option recommandée et la plus couramment utilisée.
* [Définir les paramètres INI par répertoire](/docs/agents/php-agent/configuration/php-directory-ini-settings): utile lorsque vous disposez d&apos;un serveur Web servant plusieurs applications et que vous souhaitez avoir des paramètres ou des noms différents pour elles. (Les procédures pour [Apache](/docs/agents/php-agent/configuration/php-directory-ini-settings#perdir-apache), [PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#php-fpm_per-dir) et [NGINX dans PHP-FPM](/docs/agents/php-agent/configuration/php-directory-ini-settings#PHP-FPM_nginix) sont documentées.)
* [Modifier l&apos;alias via UI](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-alias): cela modifie uniquement l&apos;alias dans l&apos; UI, pas le nom de l&apos;application sous-jacente signalé. (Cette option ne rompt pas la continuité des données dans APM.)
* [Modification via API](/docs/agents/php-agent/php-agent-api/newrelic_set_appname): Une option si vous avez accès à la modification du code de votre application, que vous êtes d&apos;accord pour l&apos;instrumenter directement et que vous trouvez les implications de continuité de trace de cet appel acceptables.

## Utiliser plusieurs noms d’application [#multiple]

Vous pouvez utiliser plusieurs noms d&apos;application pour regrouper les données de plusieurs applications sous le même nom dans APM (parfois appelé « regroupement » de vos données). Pour en savoir plus sur le fonctionnement de plusieurs noms d’applications, consultez [Utiliser plusieurs noms d’applications](/docs/agents/manage-apm-agents/app-naming/use-multiple-names-app).

Pour PHP, vous pouvez définir jusqu&apos;à trois noms d&apos;application. Le nom de l&apos;application principale est first, et les deuxième et troisième noms sont utilisés pour les catégories d&apos;agrégation de données plus générales.
