## Problème

Vous rencontrez des problèmes avec votre installation New Relic pour PHP mais vous ne savez pas quelle en est la cause.

## Solution

Le monitoring de vos logs New Relic peut aider à diagnostiquer le problème. L&apos;agent et le daemon PHP de New Relic ont leurs propres logs. Pour monitorer le log :

1. Vérifiez les deux logs pour le contenu :

   * <DNT>**[Default agent log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-logfile)**</DNT>: `/var/log/newrelic/php_agent.log`
   * <DNT>**[Default daemon log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)**</DNT>: `/var/log/newrelic/newrelic-daemon.log`

2. Définissez le niveau de log pour l&apos; agent sur `verbosedebug` et définissez le niveau de log pour le daemon sur `debug`. Pour plus d&amp;apos;informations :

   * [Définition de l&apos;agent <DNT>**loglevel**</DNT> sur <DNT>**verbosedebug**</DNT>](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
   * [Définition du daemon <DNT>**loglevel**</DNT> sur <DNT>**debug**</DNT>](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

3. Redémarrez votre application.

4. Collectez 5 à 10 minutes de temps de log avec votre niveau de log réglé sur les valeurs appropriées.

5. Examinez votre log et utilisez les informations pour diagnostiquer le problème.

6. Lorsque vous avez terminé le dépannage, réinitialisez le niveau de log. Ne les gardez pas à `verbosedebug` ou `debug`.

<Callout variant="caution">
  Surveillez la taille de votre fichier de log. Selon votre application, ils peuvent se développer très rapidement.
</Callout>
