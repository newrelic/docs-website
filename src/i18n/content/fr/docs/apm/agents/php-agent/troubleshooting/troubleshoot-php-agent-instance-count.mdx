## Problème

Sur la page agent de PHP,[APM <DNT>**Summary**</DNT>](/docs/apm/applications-menu/monitoring/apm-overview-dashboard) quel est le agent nombre de PHP instance dans la <DNT>**Hosts**</DNT> table ? Pourquoi ces chiffres changent-ils et pourquoi semble-t-il y avoir une incohérence entre eux ?

## Cause

Chaque serveur répertorié sur la page <DNT>**Summary**</DNT> inclut le numéro d&apos;instance ; par exemple, <DNT>**42 app instances**</DNT>. Le nombre instance indique le nombre maximal de descripteurs de fichiers ouverts utilisés par l&apos; agent PHP lors de la dernière récolte signalée. Ces descripteurs de fichiers sont des sockets I/O ouverts et reflètent les canaux de communication agent/daemon.

Le nombre d&apos;instances augmentera si des processus de ligne de commande, d&apos;arrière-plan ou externes sont en cours d&apos;exécution. Le nombre peut également être influencé par un débit important. Par exemple, un serveur Apache peut être configuré pour augmenter le pool de connexions afin de s&apos;adapter à la charge accrue.

Une différence dans les valeurs attendues entre les serveurs peut refléter des tâches en arrière-plan exécutées sur un serveur mais pas sur l’autre. Un nombre inattendu peut également indiquer un équilibreur de charge mal configuré.
