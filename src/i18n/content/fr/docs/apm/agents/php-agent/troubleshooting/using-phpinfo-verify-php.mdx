## Problème

New Relic ne signale pas de données dans les quelques minutes suivant le redémarrage de votre serveur Web après l&apos;installation de l&apos;agent PHP.

## Solution

Utilisez `phpinfo()` pour vérifier que l’agent a été installé avec succès :

1. Créez une page Web `.php` simple dans votre site Web contenant les éléments suivants :

   ```php
   <?php phpinfo(); ?>
   ```

2. Accédez à cette page dans un navigateur Web et faites défiler la liste alphabétique des modules pour trouver `newrelic`. Si vous ne voyez pas `newrelic` dans cette liste, soit vous utilisez une version non standard de PHP, soit vous n&amp;apos;avez pas redémarré votre serveur web (ou autre hôte PHP) depuis l&amp;apos;installation de New Relic.

3. Redémarrez votre serveur Web pour voir si `newrelic` apparaît dans votre liste de modules.

4. Si `newrelic` n&amp;apos;apparaît pas dans votre liste de modules, [vérifiez votre répertoire d&apos;extensions](/docs/php/php-kb-104) pour voir si vous avez une [version non standard de PHP](/docs/php/php-agent-installation-non-standard-php).

5. Si `newrelic` apparaît dans votre liste de modules, vérifiez que l&amp;apos;entrée `license.key` affiche les premiers et derniers caractères de votre<InlinePopover type="licenseKey" />

6. Si `license.key` n&amp;apos;apparaît pas, [vérifiez votre répertoire de fichiers de configuration chargés](/docs/agents/php-agent/troubleshooting/checking-loaded-configuration-files-directory)
