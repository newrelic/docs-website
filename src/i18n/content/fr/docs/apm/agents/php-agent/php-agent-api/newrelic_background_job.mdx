## Syntaxe

```php
newrelic_background_job([bool $flag])
```

Spécifiez manuellement qu&apos;une transaction est une tâche d&apos;arrière-plan ou une transaction Web.

## Exigences

Compatible avec toutes les versions d&apos;agent.

## Description

Dites agent à l&apos;UI APM de traiter cette « transaction Web » comme une « transaction non Web » (l&apos; sépare les transactions Web et non Web, par exemple dans la [page Transactions ).](/docs/apm/applications-menu/monitoring/transactions-page) Appelez le plus tôt possible. Ceci est le plus souvent utilisé pour les tâches cron ou d&amp;apos;autres tâches d&amp;apos;arrière-plan de longue durée. Cependant, cet appel est généralement inutile puisque l&amp;apos; agent détecte généralement automatiquement si une transaction est une transaction Web ou non Web.

Vous pouvez également inverser la fonctionnalité en définissant l&apos;indicateur facultatif sur `false`, ce qui marque une « transaction non Web » comme une « transaction Web ».

## Exemples

### Marquer la transaction comme une tâche d&apos;arrière-plan [#mark-as-background]

```php
function example() {
    if (extension_loaded('newrelic')) { // Ensure PHP agent is available
        newrelic_background_job();
    }
    ...
}
```

### Marquer la transaction comme une transaction Web [#mark-as-web]

```php
function example() {
    if (extension_loaded('newrelic')) { // Ensure PHP agent is available
        newrelic_background_job(false);
    }
    ...
}
```
