## Problème

Vous avez installé l’agent .NET de New Relic via [Microsoft Azure Web Apps](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps) et utilisez le paramètre `Always On` d’Azure. Après avoir généré du trafic et attendu quelques minutes que la collecte de données démarre pour votre application, les données n&amp;apos;apparaissent toujours pas dans l&amp;apos;interface utilisateur APM.

## Solution

À partir de la version [8.22.181.0](/docs/release-notes/agent-release-notes/net-release-notes/net-agent-8221810) de l&amp;apos;agent .NET, Le paramètre `Always On` d’Azure peut être activé sans interférer avec l’agent. Si vous souhaitez utiliser `Always On`, mettez simplement [à jour l&apos;agent](/docs/agents/net-agent/azure-installation/install-net-agent-azure-web-apps).

## Solutions de contournement pour .NET Framework 4.0 ou version antérieure [#4-0-framework]

Si votre application cible [.NET Framework 4.0 ou une version antérieure](/docs/agents/net-agent/troubleshooting/technical-support-net-framework-40-or-lower), [le paramètre`Always On` ](https://azure.microsoft.com/en-us/documentation/articles/web-sites-create-web-jobs/)d’Azure interférera avec l’agent. Cette fonctionnalité est uniquement disponible dans les modes <DNT>**Basic**</DNT> et <DNT>**Standard**</DNT> d’Azure.

Le paramètre `Always On` empêche la définition de la variable d&amp;apos;environnement `APP_POOL_ID` . L&amp;apos;agent .NET de New Relic a besoin de cette variable d&amp;apos;environnement pour obtenir le nom de l&amp;apos;application. Si l&amp;apos;agent ne peut pas déterminer le nom d&amp;apos;une application, il ne démarrera pas.

Si vous utilisez toujours .NET Framework 4.0 ou une version antérieure, essayez les solutions de contournement suivantes :

Si vous utilisez le niveau <DNT>**Standard**</DNT> ou <DNT>**Basic**</DNT> de Microsoft avec le portail Azure , assurez-vous que le paramètre `Always On` est désactivé.

1. Dans le portail Azure, sélectionnez <DNT>**(your app) &gt; Settings &gt; Application Settings**</DNT>.
2. Réglez la bascule `Always On` sur `Off`.

Voici quelques autres options que vous pouvez essayer. Si cela ne fonctionne pas, vous <DNT>**must**</DNT> [désactivez le paramètre `Always On` ](#4-0-framework).

<CollapserGroup>
  <Collapser id="name-app" title="Utilisez Kudu pour nommer votre application.">
    Pour nommer votre application dans le <DNT>**`newrelic.config`**</DNT> local en utilisant Kudu :

    1. Depuis le portail Microsoft Azure, sélectionnez <DNT>**(your app) &gt; Tools &gt; Kudu**</DNT>, puis <DNT>**Go**</DNT>.

    2. Sélectionnez <DNT>**Debug Console &gt; CMD**</DNT>.

    3. Sélectionnez <DNT>**Site &gt; wwwroot &gt; newrelic**</DNT>.

    4. Sélectionnez l&apos;édition <Icon name="fe-edit-2" /> icône pour `newrelic.config` et ajoutez :

       ```xml
       <application>
         <name>Your app's name</name>
       </application>
       ```

    5. Enregistrez et redémarrez votre application.
  </Collapser>

  <Collapser id="name-app-web-config" title="Utilisez app/web.config pour nommer votre application.">
    <Callout variant="important">
      Cette option s’est avérée n’être qu’une solution temporaire. L&apos;agent .NET peut cesser de générer des rapports par intermittence.
    </Callout>

    Pour nommer votre application dans son fichier `app/web.config` :

    1. Ajoutez la clé suivante à `appSettings`:

       ```xml
       <appSettings>
         <add key="NewRelic.AppName" value="Your app's name" />
       </appSettings>
       ```

    2. Redémarrez votre instance.
  </Collapser>

  <Collapser id="pinger" title="Utilisez Synthétique monitoring pour monitorer votre site Web.">
    En fonction de votre utilisation de `Always On`, vous pouvez essayer d&amp;apos;utiliser [monitoring Synthétique](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) pour garder votre site Web actif.

    * Si vous <DNT>**are not**</DNT> utilisez le `Always On` paramètre, vous pouvez essayer d&amp;apos;utiliser notre monitoring [pinger moniteur](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors#monitor-type-ping) gratuit Synthétique .
    * Si vous <DNT>**are**</DNT> utilisez le paramètre `Always On`, monitoring Synthétique doit pouvoir atteindre un point de terminaison pour vos rôles de travail. Si ce n&amp;apos;est pas le cas, vous pouvez essayer d&amp;apos;utiliser un [moniteur scripté Synthétique](/docs/synthetics/new-relic-synthetics/scripting-monitors/write-scripted-browsers) à exécuter sur votre site.
  </Collapser>
</CollapserGroup>
