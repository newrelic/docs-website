Avec l&apos;introduction de agent Java sortie 7.8.0, des modifications ont été apportées à <DNT>**remove**</DNT> l&amp;apos;instrumentation par défaut de la méthode `bindAndHandle` , afin d&amp;apos;éliminer les scénarios qui pourraient entraîner la déclaration de transactions dupliquées par erreur. Dans certaines situations, il peut être nécessaire d&amp;apos;effectuer des appels de méthode explicites pour compenser ce changement.

## Contexte : instrumentationHttpExt [#background]

L&apos;instrumentation pour Akka HTTP Core est réalisée dans la classe `akka.http.scaladsl.HttpExt` qui sert de point d&amp;apos;entrée principal pour un serveur. Deux méthodes pratiques de `HttpExt` qui peuvent être utilisées pour démarrer un serveur HTTP ont été instrumentées :

* `bindAndHandleAsync`: Une méthode pratique qui démarre un nouveau serveur HTTP au point de terminaison donné et utilise un gestionnaire qui est une fonction recevant un `HttpRequest` et renvoyant un `Future[HttpResponse]`
* `bindAndHandleSync`: Une méthode pratique qui démarre un nouveau serveur HTTP au point de terminaison donné et utilise un gestionnaire qui est une fonction recevant un `HttpRequest` et renvoyant un `HttpResponse`

Pour éliminer la duplication erronée des transactions signalées, l&apos;instrumentation est <DNT>**no longer**</DNT> appliquée à la méthode `bindAndHandle` , qui démarre un nouveau serveur HTTP à l&amp;apos;aide d&amp;apos;une instance `akka.stream.scaladsl.Flow` .

La duplication des transactions est due à un conflit dans l&apos;instrumentation DSL Akka Http Routing.

## Solution - appeler explicitement bindAndHandleAsync [#solution]

Si vous souhaitez démarrer un serveur HTTP à partir d&apos;un `akka.stream.scaladsl.Flow`, vous devez appeler explicitement la méthode `bindAndHandleAsync` . Par exemple:

```scala
val flow: Flow[HttpRequest, HttpResponse, NotUsed] = ???
val asyncHandler: HttpRequest => Future[HttpResponse] = request => Source.single(request).via(flow).runWith(Sink.head)

Http().bindAndHandleAsync(asyncHandler, host, port)
```

<Callout variant="important">
  Une solution de contournement similaire, appelant `bindAndHandle` lors du démarrage d&amp;apos;un serveur HTTP à partir d&amp;apos;un `akka.http.scaladsl.Route` à l&amp;apos;aide du DSL de routage HTTP Akka, n&amp;apos;est **pas** nécessaire. instrumentation de l&amp;apos;agent fonctionnera normalement lorsqu&amp;apos;elle sera appelée à partir d&amp;apos;autres méthodes pratiques.
</Callout>
