Lors du dépannage de votre agent Java de New Relic, générez `finest` un log pour rechercher et étudier les erreurs. Une fois le dépannage terminé, assurez-vous de redéfinir le fichier de log sur un paramètre moins détaillé, tel que `info`.

## Exigences [#requirement]

Si vous modifiez uniquement le paramètre du fichier de log dans votre fichier `newrelic.yml`, vous n&amp;apos;avez pas besoin de redémarrer le serveur ou JVM. Cependant, si vous apportez d’autres modifications, vous <DNT>**must**</DNT> redémarrez votre serveur ou JVM.

Assurez-vous de conserver l&apos;indentation de ligne d&apos;origine dans votre fichier `newrelic.yml` . YAML est « sensible à l&amp;apos;espace » et nécessite une indentation appropriée.

Le paramètre `audit_mode` (lorsqu&amp;apos;il est défini sur `true` ) log également la communication entre l&amp;apos;agent Java et New Relic dans le fichier de log. Le support New Relic peut occasionnellement demander [des journaux en mode audit](/docs/agents/manage-apm-agents/configuration/log-audit-all-data-your-new-relic-agent-transmits) pour un dépannage supplémentaire.

## Générer un fichier de log [#finer]

Pour générer un fichier log `finest` détaillé :

1. Modifiez votre fichier `newrelic.yml` et remplacez [`log_level`](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#cfg-log_level) par `finest`. Assurez-vous de garder ces lignes [en retrait <DNT>**two spaces**</DNT>](#requirements).

   ```yaml
   # The agent uses its own log file to keep its logging
   # separate from that of your application. Specify the log level here.
   # This setting is dynamic, so changes do not require restarting your application.
   # The levels in increasing order of verboseness are:
   # off, severe, warning, info, fine, finer, finest
   # Default is info.
   log_level: finest

   # Log all data sent to and from New Relic in plain text.
   # This setting is dynamic, so changes do not require restarting your application.
   # Default is false.
   audit_mode: false
   ```

2. Exercez votre application Web pendant environ cinq minutes pour générer suffisamment de données de logging.

3. Après le test, remplacez [`log_level`](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#cfg-log_level) par un niveau moins verbeux, tel que `info` (par défaut).

4. Ouvrez votre fichier de log et examinez-le pour détecter les erreurs.

## Examiner le fichier de log [#logfile]

<Callout variant="important">
  Le paramètre log `finest` est un niveau de logging très détaillé. Pour réduire la consommation d&amp;apos;espace disque après le test, remettez le `log_level` à son paramètre d&amp;apos;origine.
</Callout>

Par défaut, les fichiers de log sont stockés dans un sous-répertoire `logs`, à l&amp;apos;intérieur du même répertoire que `newrelic.jar`. Si nécessaire, suivez les [procédures de dépannage si aucun fichier de log n&apos;apparaît](/docs/agents/java-agent/troubleshooting/no-log-file-java).
