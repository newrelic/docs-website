L&apos;agent Java attribue automatiquement un nom `host:port` à vos JVM. Si le nom par défaut n&amp;apos;est pas utile, vous pouvez [définir un nom d&apos;affichage](#display-name) pour distinguer vos JVM dans l&amp;apos;UI de New Relic. Pour le framework pris en charge, l&amp;apos;agent Java utilise le [nominstance ](#instance-names)au lieu du nom d&amp;apos;affichage.

L&apos;agent Java identifie de manière unique l&apos;agent en cours d&apos;exécution [par son hôte, son port et son répartiteur](#unique-identifier), quel que soit l&amp;apos;environnement utilisé.

## Noms d&apos;affichage [#display-name]

Si `host:port` l&amp;apos;étiquette par défaut dans l&amp;apos;UI APM n&amp;apos;est pas utile, vous pouvez décorer ce nom dans l&amp;apos;UI de New Relic avec un [nom d&apos;affichage](/docs/apm/new-relic-apm/maintenance/add-rename-remove-hosts#display_name). Définissez le nom d&amp;apos;affichage via [la configuration](/docs/agents/java-agent/configuration/java-agent-configuration-config-file), soit dans une variable d&amp;apos;environnement, soit dans votre fichier de configuration <DNT>**newrelic.yml**</DNT> . La variable d&amp;apos;environnement a priorité sur la valeur du fichier de configuration. Une fois la JVM redémarrée et l&amp;apos;agent Java à nouveau opérationnel, le nom d&amp;apos;affichage s&amp;apos;affichera dans la liste JVM comme indiqué dans l&amp;apos;exemple ci-dessous en plus de `host:port`.

Pour définir un nom d’affichage :

* Définissez la variable d’environnement [`NEW_RELIC_PROCESS_HOST_DISPLAY_NAME`](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#NEW_RELIC_PROCESS_HOST_DISPLAY_NAME) .\
  OU
* Définissez [`process_host.display_name`](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#display_name) dans votre <DNT>**newrelic.yml**</DNT>.

Redémarrez JVM votre pour voir vos modifications dans l&apos;UI de New Relic.

<Callout variant="tip">
  Vous pouvez également définir le nom d’affichage avec une [propriété système](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#System_Properties).
</Callout>

## Noms d&apos;instance

Pour les utilisateurs WebSphere et WebLogic, l&apos;agent Java signale également le nom instance pour chaque instance de serveur d&apos;applications active, qui est visible dans la page APM <DNT>**Settings &gt; Environment**</DNT>. Généralement, ce nom est fourni par le serveur d&amp;apos;applications. Si un instance nom est défini, il sera utilisé à la place du nom d&amp;apos;affichage dans l&amp;apos;UI de New Relic.

## priorité logique du nom d&apos;hôte [#precedence]

S&apos;il existe plusieurs paramètres de nom d&apos;hôte (par exemple, vous utilisez WebSphere et définissez également la propriété `display_name`), l&amp;apos;agent utilise cette priorité :

* `display_name` remplace le nom par défaut et les noms d&amp;apos;instance WebSphere et WebLogic.
* Les noms d&apos;instance WebSphere et WebLogic remplacent le nom par défaut.
* Le nom par défaut (`host:port`) a la priorité la plus basse.

## Hôtes et identifiant unique agent [#unique-identifier]

L&apos;agent Java utilise des hôtes, des ports et des répartiteurs pour identifier de manière unique plusieurs JVM faisant référence au même nom d&apos;application. Plusieurs agents exécutés sur le même serveur sont donc signalés séparément, à condition que leurs ports ou leurs noms de répartiteur diffèrent.

L&apos;identifiant unique est basé sur les valeurs suivantes :

* <DNT>**Host**</DNT>: Un identifiant pour le serveur parent de votre application qui est dérivé du système d&amp;apos;exploitation. Cette valeur n&amp;apos;est pas configurable par l&amp;apos;agent Java.
* <DNT>**Port**</DNT>: Le port associé au serveur d&amp;apos;applications. Si l&amp;apos;agent ne peut pas déterminer le numéro de port ou si votre application ne s&amp;apos;exécute pas sur un serveur d&amp;apos;applications, la valeur par défaut est <DNT>**0**</DNT>. Cette valeur est configurable par l&amp;apos;utilisateur.
* <DNT>**Dispatcher Name**</DNT>: Le nom du serveur d&amp;apos;applications. Par exemple, Apache Tomcat, Glassfish ou WebLogic. S&amp;apos;il n&amp;apos;y a pas de serveur d&amp;apos;applications actif, la valeur par défaut est <DNT>**Unknown**</DNT>.
