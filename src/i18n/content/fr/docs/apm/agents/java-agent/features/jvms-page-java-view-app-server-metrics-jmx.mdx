Vous pouvez utiliser l&apos;agent Java de New Relic pour collecter et afficher des métriques supplémentaires fournies par les serveurs les plus application courants via JMX. Les métriques collectées varient selon le serveur d&apos;applications et incluent des mesures des pools de threads, des sessions HTTP et des transactions.

## Afficher les métriques JVM [#dashboard]

les serveurs d&apos;applications collectent et signalent différentes métriques, qui apparaissent sur la page <DNT>**JVMs**</DNT> d&apos;APM. (Pour les applications utilisant New Relic via Heroku, la page <DNT>**JVMs**</DNT> est nommée <DNT>**[Instances](/docs/accounts-partnerships/partnerships/heroku-new-relic/heroku-instances-dashboard)**</DNT>.)

Au minimum, chaque serveur d&apos;applications collecte et rapporte des données métriques sur la mémoire. Pour obtenir des instructions sur l&apos;activation des métriques JMX, reportez-vous à la documentation de votre serveur d&apos;applications.

Pour afficher les métriques JVM :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; APM &amp;amp; services &amp;gt; (select an app) &amp;gt; Monitoring &amp;gt; JVMs &amp;gt; (select a JVM)**</DNT>.
2. À partir de la page <DNT>**Memory**</DNT> de la JVM sélectionnée, utilisez l&apos;une des [fonctions d&apos;interface utilisateur](/docs/accounts-partnerships/education/getting-started-new-relic/standard-dashboard-features) standard de New Relic pour accéder aux informations détaillées.
3. Pour afficher d’autres métriques, sélectionnez n’importe quel [onglet disponible](#dashboard-tabs).
4. Facultatif : pour utiliser le [profileur de thread](/docs/applications-menu/thread-profiler) pour rechercher des goulots d&apos;étranglement dans les données sur les threads en cours d&apos;exécution, sélectionnez <DNT>**Profile this JVM**</DNT>.

## Activer pour Spring Boot 2.2 et supérieur avec Tomcat

Pour permettre à New Relic d&apos;accéder aux métriques Tomcat JMX lors de l&apos;exécution avec Spring Boot 2.2 et versions ultérieures, la propriété système `server.tomcat.mbeanregistry.enabled` doit être définie sur `true`. Voir les [notes de sortie de Spring Boot 2.2](https://github.com/spring-projects/spring-boot/wiki/Spring-Boot-2.2-Release-Notes#tomcats-mbean-registry).

## Activer les métriques WebSphere PMI [#default-pmi]

Pour permettre à New Relic d&apos;accéder aux métriques WebSphere PMI, vous devez configurer WebSphere pour monitorer les ensembles de statistiques nécessaires.

<CollapserGroup>
  <Collapser id="enable-all-stat-sets" title="Activer tous les ensembles de statistiques (recommandé)">
    L&apos;approche la plus simple consiste à permettre la collecte de tous les ensembles de statistiques :

    1. Si vous utilisez Java 2 Security, assurez-vous d&apos;avoir [accordé les autorisations appropriées à New Relic](/docs/agents/java-agent/installation/install-java-agent-java-2-security).
    2. Depuis la console d’administration WebSphere, sélectionnez <DNT>**Monitoring and Tuning &gt; Performance Monitoring Infrastructure (PMI) &gt; (your server name)**</DNT>.
    3. Cochez la case <DNT>**Enable Performance Monitoring Infrastructure (PMI)**</DNT> .
    4. Dans la section <DNT>**Currently monitored statistic set**</DNT>, sélectionnez <DNT>**All**</DNT>, puis enregistrez vos modifications.
  </Collapser>

  <Collapser id="enable-custom-stat-sets" title="Activer les ensembles de statistiques personnalisés (avancé)">
    Vous pouvez également collecter uniquement les métriques PMI que New Relic monitore par défaut. Pour collecter uniquement quelques métriques PMI :

    1. Si vous utilisez Java 2 Security, assurez-vous d&apos;avoir [accordé les autorisations appropriées à New Relic](/docs/agents/java-agent/installation/install-java-agent-java-2-security).

    2. Depuis la console d’administration WebSphere, sélectionnez <DNT>**Monitoring and Tuning &gt; Performance Monitoring Infrastructure (PMI) &gt; (selected app server)**</DNT>.

    3. Cochez la case <DNT>**Enable Performance Monitoring Infrastructure (PMI)**</DNT> .

    4. Sélectionnez <DNT>**Custom**</DNT>. Dans la liste des statistiques personnalisées, activez les métriques suivantes :

       * Dans la catégorie <DNT>**ThreadPool**</DNT>, cochez les cases <DNT>**ActiveCount**</DNT> et <DNT>**PoolSize**</DNT> .
       * Dans la catégorie <DNT>**Servlet Session Manager**</DNT>, cochez les cases <DNT>**ActiveCount**</DNT> et <DNT>**LiveCount**</DNT> .
       * Dans la catégorie <DNT>**Transaction Manager**</DNT>, cochez les cases <DNT>**CommittedCount**</DNT>, <DNT>**RolledbackCount**</DNT> et <DNT>**GlobalTimeoutCount**</DNT> .

    5. Dans le panneau <DNT>**Messages**</DNT>, sélectionnez <DNT>**Save directly to the master configuration**</DNT>.
  </Collapser>
</CollapserGroup>
