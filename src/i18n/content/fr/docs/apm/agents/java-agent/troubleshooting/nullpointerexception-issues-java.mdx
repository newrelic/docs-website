## Problème

Si vos [données de log](/docs/agents/java-agent/troubleshooting/no-log-file-java) affichent `SEVERE:` ou `ERROR:` avec `NullPointerException` problèmes, il se peut que des problèmes interfèrent avec la création de rapports.

## Solution

Avant de commencer ces conseils de dépannage, assurez-vous que vous disposez de la [version la plus récente](/docs/agents/java-agent/installation/upgrade-java-agent) de l&amp;apos;agent Java de New Relic.

1. Utilisez les recherches sensibles à la casse de votre agent Java fichier de log pour les lignes avec `SEVERE:` et.`ERROR: NullPointerException`
2. Si vous trouvez des lignes contenant ces messages, [rassemblez des informations de dépannage](/docs/agents/java-agent/troubleshooting/providing-troubleshooting-information-java) pour obtenir de l&amp;apos;aide auprès de New Relic.

## Cause

L&apos;agent Java de New Relic entre les versions 3.0.0 et 3.2.3 log incorrectement le message d&apos;erreur. Par exemple:

```
ERROR: javax/servlet/ServletOutputStream is marked as a weaved class, but no methods are matched to be weaved.
```

C’était totalement erroné et cela ne constitue pas un motif d’inquiétude. Cependant, n&apos;utilisez pas ces versions. Au lieu de cela, mettez à jour vers la [version la plus récente](/docs/agents/java-agent/installation/upgrade-java-agent) de l&amp;apos;agent Java de New Relic.
