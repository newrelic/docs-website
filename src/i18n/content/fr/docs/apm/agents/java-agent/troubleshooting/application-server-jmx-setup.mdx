## Problème

Dans APM, l&apos;onglet <DNT>**Threads**</DNT> de la [page<DNT>**JVMs**</DNT> ](/docs/agents/java-agent/features/jvm-metrics-page)affiche différentes métriques JMX prises en charge par votre serveur application , telles que les threads actifs, inactifs et totaux. Pour voir les métriques dans l&amp;apos;onglet <DNT>**Threads**</DNT>, vous devez activer les rapports JMX sur votre serveur application .

Si le reporting JMX n&apos;est pas activé, vous verrez un onglet <DNT>**Threads**</DNT> vide avec un lien vers la documentation pertinente pour votre serveur application pour activer JMX.

## Solution

Pour résoudre le problème, lisez les sections suivantes en fonction de votre serveur application :

<CollapserGroup>
  <Collapser id="JBoss" title="JBoss">
    La prise en charge locale de JMX est activée par défaut sur les serveurs d&apos;applications JBoss. Si vous avez modifié les paramètres par défaut, consultez :

    * [JBoss EAP 6.0](https://access.redhat.com/site/documentation/en-US/JBoss_Enterprise_Application_Platform/6/html/Administration_and_Configuration_Guide/index.html)
    * [JBoss AS 7.1](https://docs.jboss.org/author/display/AS71/Admin%20Guide.html#8094211_AdminGuide-JMX)
    * [JBoss AS 6](https://docs.jboss.org/jbossas/6/Admin_Console_Guide/en-US/html/)
  </Collapser>

  <Collapser id="Jetty" title="Jetty">
    Pour activer la prise en charge locale de JMX sur les serveurs d’applications Jetty , configurez le MBeanContainer avec le fichier <DNT>**etc/jetty-jmx.xml**</DNT>. Pour plus d&amp;apos;informations, voir :

    * [Jetty 7 et 8](http://wiki.eclipse.org/Jetty/Tutorial/JMX)
    * [Jetty 9](http://www.eclipse.org/jetty/documentation/current/jmx-chapter.html)
  </Collapser>

  <Collapser id="Glassfish" title="Glassfish">
    Bien que la prise en charge locale de JMX soit activée par défaut sur les serveurs d&apos;applications Glassfish , les MBeans qui fournissent réellement des informations métriques sont définis par défaut sur `OFF`. Pour que les métriques appropriées soient collectées par le serveur application , les composants suivants doivent être définis dans le <DNT>**Monitoring Service**</DNT>:

    * [Glassfish 4.x](https://glassfish.java.net/docs/4.0/administration-guide.pdf): les composants Conteneur Web, Service HTTP et Service de transaction doivent être définis sur le niveau monitoring ÉLEVÉ.
    * [Glassfish 3.1](http://docs.oracle.com/cd/E18930_01/html/821-2416/ablur.html): les composants Conteneur Web, Service HTTP et Service de transaction doivent être définis sur le niveau monitoring ÉLEVÉ.
    * [Glassfish 3.0](http://docs.oracle.com/cd/E19798-01/821-1751/ablvr/index.html): les composants Conteneur Web, Pool de threads et Service de transaction doivent être définis sur le niveau monitoring BAS.
    * [Glassfish 2.1](http://docs.oracle.com/cd/E19879-01/820-4335/ablur/index.html): les composants Conteneur Web, Pool de threads et Service de transaction doivent être définis sur le niveau monitoring BAS.
  </Collapser>

  <Collapser id="Resin" title="Resin">
    La prise en charge locale de JMX est activée par défaut sur les serveurs d&apos;applications Resin . Si vous avez modifié les paramètres par défaut, consultez :

    * [Resin 3.1](http://www.caucho.com/resin-3.1/doc/resin-admin.xtp#JMXInstrumentingBeans)
    * [Resin 4.0](http://www.caucho.com/resin-4.0/admin/advanced-jmx.xtp)
  </Collapser>

  <Collapser id="Tomcat" title="Tomcat">
    La prise en charge locale de JMX est activée par défaut sur les serveurs d&apos;applications Tomcat . Si vous avez modifié les paramètres par défaut, consultez :

    * [Tomcat 5.5](http://tomcat.apache.org/tomcat-5.5-doc/monitoring.html)
    * [Tomcat 6](http://tomcat.apache.org/tomcat-6.0-doc/monitoring.html)
    * [Tomcat 7](http://tomcat.apache.org/tomcat-7.0-doc/monitoring.html)
    * [Tomcat 8](https://tomcat.apache.org/tomcat-8.0-doc/monitoring.html)
    * [Tomcat 9](http://tomcat.apache.org/tomcat-9.0-doc/monitoring.html)
  </Collapser>

  <Collapser id="TomcatEE" title="Tomcat EE">
    La prise en charge locale de JMX est activée par défaut sur les serveurs d&apos;applications TomEE. Si vous avez modifié les paramètres par défaut, consultez :

    * [TomEE 1.5](http://tomee.apache.org/documentation.html)
  </Collapser>

  <Collapser id="WebLogic" title="WebLogic">
    La prise en charge locale de JMX est activée par défaut sur les serveurs d&apos;applications WebLogic. Si vous avez modifié les paramètres par défaut, consultez :

    * [WebLogic 12](http://docs.oracle.com/cd/E24329_01/web.1211/e24415/toc.htm)
    * [WebLogic 11](http://docs.oracle.com/cd/E23943_01/web.1111/e13728/toc.htm)
    * [WebLogic 10](http://docs.oracle.com/cd/E12840_01/wls/docs103/jmx/index.html)
  </Collapser>

  <Collapser id="WebSphere" title="WebSphere">
    La prise en charge locale de JMX est <DNT>**disabled**</DNT> sur les serveurs d’applications WebSphere par défaut. Pour plus d&amp;apos;informations, voir :

    * [WebSphere 9](https://www.ibm.com/support/knowledgecenter/SSEQTP_9.0.0/com.ibm.websphere.base.doc/ae/csec_rsecmgr2.html)

    * [WebSphere 8.5](https://www.ibm.com/support/knowledgecenter/SSEQTP_8.5.5/com.ibm.websphere.base.doc/ae/csec_rsecmgr2.html)

    * [WebSphere 7](https://www.ibm.com/support/knowledgecenter/SSEQTP_7.0.0/com.ibm.websphere.base.doc/info/aes/ae/csec_rsecmgr2.html)

      Et la documentation New Relic associée :

    * [Activer les métriques WebSphere PMI](/docs/agents/java-agent/features/jvm-metrics-page#default-pmi)
  </Collapser>

  <Collapser id="WebSphereLiberty" title="WebSphere Liberty">
    La prise en charge locale de JMX est <DNT>**disabled**</DNT> sur les serveurs d&amp;apos;applications WebSphere Liberty par défaut. Pour plus d&amp;apos;informations, voir :

    * [WebSphere 16+](https://www.ibm.com/support/knowledgecenter/SSEQTP_liberty/com.ibm.websphere.wlp.doc/ae/cwlp_java2security.html)
    * [WebSphere 8.5](https://www.ibm.com/support/knowledgecenter/SSD28V_8.5.5/com.ibm.websphere.wlp.core.doc/ae/cwlp_java2security.html)
  </Collapser>

  <Collapser id="WildFly" title="Vol sauvage">
    La prise en charge locale de JMX est activée par défaut sur les serveurs d&apos;applications WildFly. Si vous avez modifié les paramètres par défaut, consultez :

    * [WildFly 21.0.0](https://docs.wildfly.org/21/Admin_Guide.html#JMX)
  </Collapser>
</CollapserGroup>
