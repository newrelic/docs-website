La fonctionnalité <InlinePopover type="mobile" />inclut une analyse comparative de l&amp;apos;adoption et des performances entre les versions de votre application, y compris les principales versions par :

* Temps d&apos;interaction
* Sessions actives
* Taux d&apos;erreur

La page <DNT>**Versions**</DNT> inclut également un tableau comparant chaque version par date de création, mémoire moyenne, CPU moyen, sessions moyennes par minute et requests moyennes par minute (RPM) par application active. Vous pouvez également accéder à des détails supplémentaires sur une version spécifique.

La page <DNT>**Versions**</DNT> fournit une liste de toutes les versions de votre application qui ont été détectées, ainsi que des informations générales sur toutes les versions actives au cours des sept derniers jours. Pour consulter l’analyse comparative :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile &amp;gt; (select an app) &amp;gt; Usage &amp;gt; Versions**</DNT>.
2. Pour sélectionner la période, utilisez le [<DNT>**time picker**</DNT>](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) sous la barre de menu New Relic.
3. Facultatif : sélectionnez les options <DNT>**Sort by**</DNT> .
4. Pour afficher les détails uniquement pour un <DNT>**version**</DNT> spécifique, sélectionnez son nom.

La page <DNT>**Versions**</DNT> fournit une liste de toutes les versions de votre application qui ont été actives dans la fenêtre temporelle sélectionnée. Pour accéder à des informations détaillées, utilisez n&amp;apos;importe quelle [fonctionnalité d&apos;interface utilisateur de plateforme](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) applicable.

La page de détails fournit des informations plus détaillées sur la façon dont la version sélectionnée se compare à une version de référence (une version récente ou populaire) et à la moyenne des autres versions de votre application. Les séries chronologiques montrent la comparaison entre le taux d&apos;erreur, le temps de réponse, les sessions actives et l&apos;utilisation de la mémoire.

Pour quitter la page de détails, sélectionnez le bouton <DNT>**Close**</DNT> (<DNT>**X**</DNT>).

## Comment les numéros de version sont obtenus [#how-obtain-version]

La manière dont monitoring des applications mobiles obtient le numéro de version varie selon la plateforme :

<CollapserGroup>
  <Collapser id="android-ver-num" title="Android">
    L&apos; [agent Android](/docs/mobile-monitoring/new-relic-mobile-android/get-started/android-installation-configuration) obtient les informations de version à partir de la propriété `android:versionName` dans le manifeste.
  </Collapser>

  <Collapser id="ios-ver-num" title="iOS">
    L&apos;agent iOS utilise les propriétés `CFBundleShortVersionString` et `CFBundleVersion` pour obtenir la version de l&amp;apos;application. L&amp;apos;agent accède à ces propriétés via les API iOS. Il ne les obtient pas en lisant le fichier `info.plist` .
  </Collapser>
</CollapserGroup>
