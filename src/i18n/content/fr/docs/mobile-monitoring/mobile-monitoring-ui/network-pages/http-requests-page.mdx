La page d&apos;interface utilisateur <DNT>**HTTP requests**</DNT> vous aide à mieux comprendre requests HTTP associées à votre application mobile et la manière dont ces appels réseau affectent les performances.

## Afficher la page requests HTTP [#find-page]

Pour afficher la page <DNT>**HTTP requests**</DNT> :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt; (select an app) &gt; Network &gt; HTTP requests**</DNT>.
2. Utilisez nos [fonctions d&apos;interface utilisateur standard](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) pour rechercher des tendances dans les graphiques d&apos;analyse HTTP.
3. ciblez des attributs de demande et de réponse spécifiques en [regroupant, en triant et en filtrant](#groups-sort-filter) les données.

## Comprendre les données de requête HTTP [#understand-data]

Voici quelques endroits où trouver les données de requête HTTP les plus importantes :

<CollapserGroup>
  <Collapser id="response-time-chart" title="Tableau des temps de réponse">
    Le graphique des temps de réponse montre les performances des appels réseau de votre application par centile. Utilisez-le pour comparer le temps de réponse moyen aux 1er, 50e et 99e percentile. Le percentile vous permet de filtrer les valeurs hors norme qui pourraient rendre votre temps de réponse moyen plus élevé que prévu.
  </Collapser>

  <Collapser id="errors-network-failures-chart" title="Tableau des erreurs HTTP et des pannes de réseau">
    Ce graphique montre les appels réseau infructueux rencontrés par votre application. Sélectionnez le titre du graphique pour accéder à la [page<DNT>**HTTP errors**</DNT> ](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/network-errors-http-error-network-failure-analysis)pour plus de détails sur les erreurs et les échecs.
  </Collapser>

  <Collapser id="total-requests-chart" title="Nombre total requests">
    Trier par <DNT>**Total requests**</DNT> pour identifier les requests réseau les plus fréquemment utilisées. La raison pour laquelle cela peut être utile est que vos appels réseau les plus lents peuvent être utilisés seulement rarement, tandis que requests les plus fréquemment utilisées peuvent être plus dignes d&apos;optimisation même si elles ne sont pas les plus lentes.
  </Collapser>
</CollapserGroup>

Pour une description de la page d&apos;interface utilisateur <DNT>**HTTP requests**</DNT> non professionnelle, consultez [requestsHTTP héritées](#non-enterprise-http-requests).

## Afficher et partager les données de requête HTTP [#insights]

Pour afficher n&apos;importe quel graphique <DNT>**HTTP requests**</DNT> dans Insights :

1. Sélectionner <Icon name="fe-more-horizontal" /> pour n&apos;importe quel graphique.
2. Sélectionnez <DNT>**View query &gt; View in Insights**</DNT>.
3. Facultatif : ajoutez les données à un dashboard ou [partagez-les en utilisant un permalien](/docs/data-analysis/user-interface-functions/share-your-data/permalink).
4. Pour approfondir vos données de requête, requête [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) événement et [attribut](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes).
