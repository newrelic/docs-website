La page Résumé fournit un aperçu opérationnel de votre application mobile. Tous les graphiques, tableaux et statistiques affichent les données pour une [fenêtre temporelle](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) et une version d&apos;application spécifiées.

<img title="Mobile summary page" alt="GIF of the mobile summary page" src="/images/mobile_screenshot-full_summary-gif.gif" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile &amp;gt; (select an app):**</DNT> La page récapitulative fournit des données de performance de haut niveau et une base d&apos;atterrissage pour approfondir des mesures ou des problèmes spécifiques.
</figcaption>

## Voir la page de résumé [#summary-page]

1. Accédez à [one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile.
2. Sélectionnez votre application mobile. La page Résumé se charge par défaut.

## (Recommandé) Configurer monitoring des applications mobiles par alertes [#alerts]

Nous vous recommandons de créer des alertes pour les problèmes mobiles clés tels que requests de réseau lentes et les augmentations du taux de plantage. Pour configurer facilement ces alertes depuis la page Résumé :

1. Cliquez sur <DNT>**Add recommended conditions**</DNT>.

   <img width="80%;" style={{align: "left"}} title="Mobile Summary page" alt="Screenshot of the New Relic mobile Summary page" src="/images/mobile_screenshot-full_mobile-alert-conditions.webp" />

2. Choisissez un nombre de conditions d&apos;alerte recommandées.

   <img width="70%;" style={{align: "left"}} title="Mobile recommended alert conditions" alt="Mobile recommended alert conditions" src="/images/mobile_screenshot-crop_mobile-add-alert.webp" />

3. Cliquez sur <DNT>**Next**</DNT>.

4. Sélectionnez une politique pour être averti. Vous pouvez sélectionner une politique existante ou [créer une nouvelle politique](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name).

5. Cliquez sur <DNT>**Activate conditions**</DNT>.
