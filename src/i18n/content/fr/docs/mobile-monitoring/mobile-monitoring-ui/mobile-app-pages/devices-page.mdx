La page Appareils de <InlinePopover type="mobile" />fournit des détails sur les performances des principaux appareils utilisant votre application mobile, tels que iPad, iPhone, iPod Touch, tablette Android, etc., et affiche des métriques telles que :

* Temps d&apos;interaction
* Heure de la requête HTTP
* taux d&apos;erreur et pannes réseau
* utilisateurs actifs sessions

<img title="Mobile Devices page" alt="Screenshot of Devices page" src="/images/mobile_screenshot-full_devices-summary.webp" />

## Voir la page Appareils [#view-devices-page]

1. Accédez à [one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile.
2. Sélectionnez votre application mobile.
3. Dans le menu de gauche, cliquez sur <DNT>**Devices**</DNT>.

## Explorez des mesures spécifiques [#drill-down]

Pour accéder à des informations détaillées, utilisez l&apos;une des [fonctions d&apos;interface utilisateur](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) standard de New Relic. En outre:

* Pour afficher une liste d’appareils ou de modèles spécifiques (par exemple, iPad mini, iPad Air, etc.), sélectionnez le <DNT>**type**</DNT> (par exemple, iPad).
* Pour afficher les détails d’un appareil ou d’un modèle spécifique, sélectionnez son nom dans la liste développée.
* Pour afficher les détails de trace d&apos;une transaction lente (si disponible), sélectionnez son lien. Pour plus d&apos;informations, consultez [la page d&apos;interaction](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/interactions-page).

<img title="Drill down into device metrics" alt="Gif of specific device metrics" src="/images/mobile_screenshot-full_device-drilldown.gif" />

## Créer des alertes [#create-alerts]

Maintenant que vous avez une visibilité sur les performances de votre application, il est temps de configurer des alertes pour les changements de performances clés.

Cliquez simplement sur une métrique, puis dans le menu du graphique, cliquez sur <DNT>**Create alert condition**</DNT> et suivez les étapes de l&apos;interface utilisateur.
