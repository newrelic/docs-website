Vous êtes curieux de savoir comment se déroule le lancement de votre dernière application ? Utilisez notre page de versions de sortie pour afficher le taux de plantage, l&apos;engagement des utilisateurs et les mesures de performance globales entre différentes versions de sortie, que vous pouvez utiliser pour analyser le succès de votre dernière sortie.

<img title="Mobile release versions page" alt="Screenshot of mobile release versions UI page" src="/images/mobile_screenshot-full_release-versions.webp" />

## Voir la page des versions de sortie [#view-release-versions-page]

1. Accédez à [one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile.
2. Sélectionnez votre application mobile.
3. Dans le menu de gauche, cliquez sur <DNT>**Release versions**</DNT>.

<Callout variant="tip">
  Vous ne voyez pas les données ? Essayez ce qui suit :

  * Ajustez le filtre temporel pour inclure les données avant une sortie d&apos;une nouvelle version.
  * Assurez-vous que votre agent mobile utilise l&apos;agent iOS version 7.0.0 et supérieure ou l&apos;agent Android version 6.5.0 et supérieure.
</Callout>

## Explorez des mesures spécifiques [#drill-down]

La page des versions de sortie affiche des métriques de haut niveau sur chaque version d&apos;application :

* <DNT>**Overall performance score**</DNT>:Calculé par le taux de plantation global des clients et les taux sans erreur HTTP.
* <DNT>**Crash free user %**</DNT>:Pourcentage d&apos;utilisateurs ne rencontrant pas de plantages sur une version spécifique.
* <DNT>**User adoption**</DNT>:Pourcentage d&apos;utilisateurs ayant mis à jour ou installé une version spécifique.

Cliquez sur <DNT>**View summary**</DNT> ou <DNT>**View details**</DNT> pour développer des métriques spécifiques pour une version d&apos;application. Avec des visualisations faciles à comprendre et des graphiques détaillés, la page de résumé des versions vous permet d&apos;analyser les indicateurs de performances clés et de prendre des décisions éclairées pour optimiser l&apos;expérience utilisateur de votre application.

<img title="Details of release versions" alt="Screenshot of release versions detailed view" src="/images/mobile_screenshot-full_release-versions-details.webp" />

Pour chaque métrique répertoriée ci-dessus, la page des versions de sortie inclut un graphique de séries chronologiques dynamiques, vous permettant de suivre les performances de la métrique au fil du temps. En observant les tendances, les pics ou les baisses dans le graphique, vous pouvez rapidement identifier les modèles et les corrélations qui peuvent éclairer votre prise de décision.

## Créer des alertes [#create-alerts]

Maintenant que vous avez une visibilité sur les performances de votre application, il est temps de configurer des alertes pour les changements de performances clés.

Cliquez simplement sur une métrique, puis dans le menu du graphique, cliquez sur <DNT>**Create alert condition**</DNT> et suivez les étapes de l&apos;interface utilisateur.
