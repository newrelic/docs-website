Vous êtes curieux de savoir comment votre application fonctionne sur la dernière version Android ou iOS ? Utilisez notre page de versions de système d&apos;exploitation pour analyser et comparer les performances des applications entre différentes sorties (y compris les sorties majeures et mineures) et afficher des mesures telles que le nombre d&apos;appareils actifs, les pannes de réseau et le temps de requête HTTP.

## Voir la page des versions du système d&apos;exploitation [#view-os-versions-page]

1. Accédez à [one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile.
2. Sélectionnez votre application mobile.
3. Dans le menu de gauche, cliquez sur <DNT>**OS versions**</DNT>.

<img title="OS versions page" alt="Use this page to view, sort, or drill down into detailed information about the top five types of operation system versions using your mobile app." src="/images/mobile_screenshot-full_os-versions-summary.webp" />

## Explorez des mesures spécifiques [#drill-down]

Si vous constatez un pic de données, comme une augmentation du temps interaction , cliquez et faites glisser le sélecteur d&apos;intervalle de temps. La page s&apos;actualisera et s&apos;étendra sur ce segment de temps afin que vous puissiez voir exactement ce qui s&apos;est passé.

<img title="Expand metric details" alt="Gif of clicking and dragging the time picker" src="/images/mobile_screenshot-full_Expand-metrics.gif" />

Cliquez sur une version de système d&apos;exploitation spécifique pour afficher les mesures de performances.

<img title="Expand metric details for one OS" alt="Screenshot of metrics for one OS" src="/images/mobile_screenshot-full_Expand-metrics.webp" />

## Créer des alertes [#create-alerts]

Maintenant que vous avez une visibilité sur les performances de votre application, il est temps de configurer des alertes pour les changements de performances clés.

Cliquez simplement sur une métrique, puis dans le menu du graphique, cliquez sur <DNT>**Create alert condition**</DNT> et suivez les étapes de l&amp;apos;interface utilisateur.
