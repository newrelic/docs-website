<InlinePopover type="mobile" />de New Relic fournit plusieurs rapports par courrier électronique, notamment :

* [notification de sortie](#version-trends) (alias e-mails de tendance de version)
* [Courriels de rapport d&apos;incident](#crash-reports)

## Recevez des e-mails sur les performances de la nouvelle version [#version-trends]

monitoring des applications mobiles de New Relic vous fournit une page [<DNT>**Release versions**</DNT>](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/release-versions-page) . Vous pouvez également activer les e-mails de tendance de version (également appelés notifications de sortie). Ces rapports fournissent un outil de communication pratique pour partager, suivre et examiner la progression, l&apos;adoption et d&apos;autres tendances de votre application. Chaque e-mail comprend un lien vers l&apos;interface utilisateur <DNT>**Version trends**</DNT> pour voir plus de détails.

### Modifier les paramètres de courrier électronique de tendance de version [#version-trend-ui]

Pour configurer les rapports par e-mail que vous recevez pour une application mobile spécifique, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; (select an app) &gt; Settings &gt; Application**</DNT> et modifiez les paramètres notification par e-mail de sortie.

## Courriels de rapport d&apos;incident [#crash-reports]

Lorsque votre application mobile rencontre un nouveau type de crash, vous souhaitez savoir ce qui s&apos;est passé afin de pouvoir éviter ce type d&apos;erreur à l&apos;avenir. Avec les e-mails de rapport de crash, vous êtes averti lorsqu&apos;un crash unique est détecté.

Pour configurer si vous recevez des rapports de plantage par e-mail pour une application mobile spécifique, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt; (select an app) &gt; Settings &gt; Application**</DNT> et modifiez les paramètres de l&apos;e-mail de rapport de plantage.
