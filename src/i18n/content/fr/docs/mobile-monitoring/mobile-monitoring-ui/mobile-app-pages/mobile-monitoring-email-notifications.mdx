<InlinePopover type="mobile" />de New Relic fournit plusieurs rapports par courrier électronique, notamment :

* [notification de sortie](#version-trends) (alias e-mails de tendance de version)
* [Courriels de rapport d&apos;incident](#crash-reports)

## Recevez des e-mails sur les performances de la nouvelle version [#version-trends]

monitoring des applications mobiles de New Relic vous fournit une page [<DNT>**Release versions**</DNT>](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/release-versions-page) . Vous pouvez également activer les e-mails de tendance de version (également appelés notifications de sortie). Ces rapports fournissent un outil de communication pratique pour partager, suivre et examiner la progression, l&apos;adoption et d&apos;autres tendances de votre application. Chaque e-mail comprend un lien vers l&apos;interface utilisateur <DNT>**Version trends**</DNT> pour voir plus de détails.

Nous vous envoyons des rapports par courrier électronique à intervalles réguliers :

### Modifier les paramètres de courrier électronique de tendance de version [#version-trend-ui]

La manière dont vous configurez vos paramètres de messagerie électronique de tendance de version dépend du [modèle d&apos;utilisateur](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) que vous utilisez.

<Callout variant="tip">
  Pour vérifier rapidement votre modèle d&apos;utilisateur : accédez au [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) et cliquez sur <DNT>**User preferences**</DNT>. Si vous voyez les paramètres de messagerie mobile ici, vous êtes sur notre modèle d&apos;utilisateur d&apos;origine.
</Callout>

<CollapserGroup>
  <Collapser id="original-user-version-trends" title="E-mails de version (modèle d'utilisateur d'origine)">
    Pour activer ou désactiver les rapports de tendances de version par e-mail pour toutes vos applications mobiles de monitoring :

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile &amp;gt; (select a mobile app) &amp;gt; App &amp;gt; Version trends**</DNT>.

    2. Sélectionnez le lien <DNT>**Enable version trends email notifications**</DNT> et modifiez vos préférences.

       Pour configurer des rapports par e-mail pour une application mobile spécifique, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; (select an app) &amp;gt; Settings &amp;gt; Application**</DNT> et modifiez vos préférences.
  </Collapser>

  <Collapser id="new-relic-one-user-version-trends" title="E-mails de version (modèle d'utilisateur plus récent)">
    Pour configurer les rapports par e-mail que vous recevez pour une application mobile spécifique, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; (select an app) &amp;gt; Settings &amp;gt; Application**</DNT> et modifiez les paramètres notification par e-mail de sortie.
  </Collapser>
</CollapserGroup>

## Courriels de rapport d&apos;incident [#crash-reports]

Lorsque votre application mobile rencontre un nouveau type de crash, vous souhaitez savoir ce qui s&apos;est passé afin de pouvoir éviter ce type d&apos;erreur à l&apos;avenir. Avec les e-mails de rapport de crash, vous êtes averti lorsqu&apos;un crash unique est détecté.

La manière dont vous configurez vos paramètres de courrier électronique de rapport d&apos;incident dépend du [modèle d&apos;utilisateur que](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) vous utilisez.

<Callout variant="tip">
  Pour vérifier rapidement votre modèle d&apos;utilisateur : accédez au [menu utilisateur](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) et cliquez sur <DNT>**User preferences**</DNT>. Si vous voyez les paramètres de messagerie mobile ici, vous êtes sur notre modèle d&apos;utilisateur d&apos;origine.
</Callout>

<CollapserGroup>
  <Collapser id="original-user-crash-emails" title="Courriels de crash (modèle d'utilisateur d'origine)">
    Pour configurer les paramètres de courrier électronique de rapport d&apos;incident pour votre compte, sur plusieurs applications : accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) &amp;gt; User preferences**</DNT>.

    Pour configurer les paramètres de courrier électronique de rapport d’incident pour une application mobile de monitoring spécifique :

    1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) &amp;gt; User preferences**</DNT>.
    2. Sous la section <DNT>**Mobile crash reporting**</DNT> , passez la souris sur le nom d&apos;une application mobile et vous verrez <DNT>**Edit application settings**</DNT>. Sélectionnez-le.
    3. Cela vous amène à la [page d&apos;interface utilisateur<DNT>**Settings**</DNT> ](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)de l&apos;application. Cochez la case <DNT>**Enable crash report emails for this application**</DNT> .
  </Collapser>

  <Collapser id="new-relic-one-user-crash-emails" title="E-mails de crash (nouveau modèle d'utilisateur)">
    Pour configurer si vous recevez des rapports de plantage par e-mail pour une application mobile spécifique, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile &amp;gt; (select an app) &amp;gt; Settings &amp;gt; Application**</DNT> et modifiez les paramètres de l&apos;e-mail de rapport de plantage.
  </Collapser>
</CollapserGroup>
