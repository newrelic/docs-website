Lorsque la [condition d&apos;alerte](/docs/mobile-apps/alert-settings) que vous avez définie pour le taux d&apos;erreur et le temps de réponse de votre application mobile dépasse un seuil pendant une période prolongée, l&apos;agent <InlinePopover type="mobile" />créera un événement de problème d&apos;avertissement (jaune) ou critique (rouge). Si un événement Critique (rouge) est ouvert depuis plusieurs minutes, le mobile enverra une notification d&apos;alerte aux utilisateurs qui ont été configurés pour recevoir une notification d&apos;alerte. Il enverra également une notification lorsque tous les événements critiques se fermeront.

<img title="alert conditions page " alt="screenshot of alert conditions page for mobile monitoring" src="/images/mobile_screenshot-full_alert-conditions-page.webp" />

Pour afficher la page des conditions d&apos;alerte, accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt; (select an app) &gt; Alert conditions**</DNT>.

## détails de la page condition d&apos;alerte

Chaque notification d&apos;alerte comprend l&apos;icône d&apos;alerte et des informations récapitulatives. Utilisez l’une de nos [fonctionnalités d’interface utilisateur](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) applicables pour accéder à des informations détaillées. De plus, utilisez l’une de ces options :
