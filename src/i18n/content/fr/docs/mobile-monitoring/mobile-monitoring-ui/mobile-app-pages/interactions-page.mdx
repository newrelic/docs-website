Dans monitoring des applications mobiles, les interactions sont un ensemble de méthodes instrumentées regroupées par UIViewControllers (iOS) ou Activités (Android). Utilisez notre page d&apos;interaction pour suivre l&apos;interaction la plus coûteuse et la plus lente et rechercher ce qui cause la trace la plus lente.

## interaction instrumentée [#instrumentation]

les interactions sont automatiquement instrumentées. Pour plus de détails sur les classes auto-instrumentées, voir :

* [(Android) Méthodes instrumentées automatiquement](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/mobile-sdk-api-guide/#method-list)
* [(iOS) Méthodes instrumentées automatiquement](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/mobile-sdk-api-guide/#method-list-ios)

Nous proposons également une instrumentation manuelle de l&apos;interaction utilisant les méthodes suivantes :

* [`startInteraction`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/start-interaction/)
* [`stopInteraction`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/stop-interaction/)

Si vous utilisez les méthodes `startInteraction` et `stopInteraction`, l&apos;interaction instrumentée n&apos;apparaîtra pas sur la page d&apos;interaction, mais elles peuvent toujours être trouvées dans une requête NRQL , telle que :

```sql
SELECT name FROM Mobile SINCE 7 DAYS AGO
```

## Voir la page d&apos;interaction [#interactions-page]

La page d&apos;interaction contient des graphiques qui décomposent les 20 interactions les plus coûteuses et une liste des traces les plus lentes. Cela inclut le code exécuté sur le thread principal et le code exécuté sur les threads de travail.

Pour afficher la page d&apos;interaction :

1. Accédez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt;**</DNT> (sélectionnez une application) <DNT>**&gt; More views &gt; Interactions**</DNT>.
2. Pour accéder à des informations détaillées, sélectionnez une interaction ou utilisez notre [fonctionnalité d&apos;interface utilisateur de plateforme](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) applicable.

## Accédez en profondeur aux détails de l&apos;interaction [#breakdown-table]

Pour afficher la liste complète de toutes les méthodes instrumentées : À partir de l’ interaction sélectionnée, sélectionnez le lien <DNT>**Breakdown table**</DNT>. Si vous utilisez une instrumentation personnalisée pour l’une des métriques, la catégorie de la classe et le nom de la méthode indiqueront <DNT>**custom**</DNT>.

Pour chaque méthode de trace, le tableau de répartition interaction affiche :

* Type de fil (premier plan ou arrière-plan)
* Nom de la classe et de la méthode
* Pourcentage du temps total d&apos;exécution consacré à cette méthode
* Nombre moyen d&apos;appels à cette méthode par interaction
* Temps d&apos;exécution moyen global de chaque appel à cette méthode

## Analyser la chronologie des trace d&apos; interaction [#interactions-timeline]

Chaque période trace d&apos; interaction enregistre une seule trace jusqu&apos;à ce qu&apos;une nouvelle interaction commence. Pour afficher la chronologie trace de votre application mobile :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt; (select an app) &gt; App &gt; Interactions**</DNT>.
2. Dans la table <DNT>**Slowest traces**</DNT> , sélectionnez le lien <DNT>**Activity**</DNT> ou <DNT>**Duration**</DNT> pour la trace.

La chronologie trace fournit également des mesures de l&apos;appareil pour vous aider à obtenir des informations détaillées sur les données au niveau de l&apos;appareil de votre application. Les métriques d&apos;utilisation du système montrent comment votre application interagit avec les appareils de votre utilisateur final, y compris la base de données, le processeur et la mémoire.

Voici quelques actions supplémentaires que vous pouvez entreprendre :

* Pour afficher les mesures d&apos;utilisation de l&apos;appareil, passez la souris sur la chronologie.
* Pour accéder à des informations détaillées, sélectionnez n’importe quel emplacement sur la chronologie.
* Pour aider à fournir un contexte aux données, cliquez sur <DNT>**Previous interaction**</DNT>.
