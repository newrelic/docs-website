Une fois que vous commencez à collecter les enregistrements de session, vous pouvez les utiliser pour améliorer vos expériences numériques en identifiant les tendances communes et les zones de friction pour l&apos;utilisateur. Ils sont également utiles pour diagnostiquer et résoudre rapidement les problèmes en analysant les étapes suivies par l&apos;utilisateur avant de rencontrer des erreurs ou un comportement inattendu.

## Visionnez les rediffusions sur la page Session Replay [#replay-page]

La page Session Replay fournit une liste de toutes les rediffusions enregistrées pendant la période sélectionnée. Par défaut, nous filtrons la liste pour ne retenir que les rediffusions d&apos;une durée d&apos;au moins 5 secondes. Le tableau fournit des données utiles comme le nombre d&apos;erreurs, et les filtres vous permettent de restreindre la liste aux sessions qui vous aideront à résoudre votre problème.

<img title="Session Replay listing page" alt="Session Replay listing page" src="/images/mobile-session-replay-listing.webp" />

<figcaption>
  Consultez et filtrez les rediffusions dans **Mobile &gt; \[votre\_app] &gt; Session replay**.
</figcaption>

Par exemple, vous pouvez ajouter un filtre `countryCode` pour voir les rediffusions d&apos;une région spécifique afin de vérifier que la localisation fonctionne comme prévu, ou un filtre `deviceModel` pour voir les rediffusions d&apos;un modèle de téléphone spécifique qui peut avoir des problèmes de compatibilité.

**Pour accéder à la page Session Replay :**

Dans l&apos;interface web de New Relic :

1. Rendez-vous sur **[one.newrelic.com &gt; All Capabilities &gt; Mobile](https://one.newrelic.com/all-capabilities)**.
2. Sélectionnez votre application mobile.
3. Dans le menu de gauche, faites défiler vers le bas et cliquez sur **Session replay**.
4. Cliquez sur une ligne pour visionner le replay.

## Visionnez les rediffusions présentant des erreurs et des problèmes de performance [#errors-issues]

Sur la page Errors Inbox :

1. Dans le menu mobile de gauche, cliquez sur **Errors**.
2. Cliquez sur les pages **Triage** et **Group errors** pour voir les replays associés aux erreurs.

<img title="Session Replay in errors experiences" alt="Session Replay integration with error experiences showing a replay timeline with error markers" src="/images/mobile-session-replay-errors.webp" />

Lorsqu&apos;un utilisateur a rencontré un problème pendant la capture de la rediffusion, un aperçu de la rediffusion est disponible sur l&apos;occurrence de l&apos;erreur dans les expériences Plantages, Exceptions gérées et ANR.

L&apos;aperçu montre 5 à 10 secondes précédant le problème et, pour les problèmes non fatals, 5 à 10 secondes après qu&apos;il s&apos;est produit. Une icône est superposée sur la chronologie pour vous montrer exactement quand l&apos;erreur s&apos;est produite dans le replay. Cela vous permet d&apos;évaluer rapidement toute action de l&apos;utilisateur susceptible d&apos;avoir causé le problème, ainsi que la gravité de l&apos;impact sur l&apos;utilisateur final.

Si l’aperçu ne suffit pas à résoudre votre problème, cliquez sur le bouton **View full session replay** pour voir la session entière synchronisée avec les autres données télémétriques capturées.

## Comprendre l&apos;interface utilisateur [#ui]

Il y a certains éléments dans l&apos;UI qui ressembleront beaucoup à d&apos;autres lecteurs vidéo courants, comme ignorer l&apos;inactivité ou la vitesse de 0,5x à 8x. Vous trouverez également des moments dans le temps auxquels vous pouvez passer, y compris des marqueurs pour les exceptions gérées, les erreurs HTTP, les événements d&apos;application ne répondant pas et les plantages.

La fonctionnalité **Skip Inactivity** ignore les périodes d&apos;inactivité de l&apos;utilisateur, et non l&apos;inactivité de la page ou de la vue.

* Un marqueur rouge indique un plantage.
* L&apos;icône de feu signifie qu&apos;il y a eu plus de 99 événements d&apos;erreur ou de plantage qui se sont produits pendant cette période.
* Une bande diagonale rouge sur la ligne de durée signifie que la rediffusion s&apos;est terminée avant qu&apos;un ANR ne se produise.
* Une bande diagonale orange sur la ligne de durée signifie que la rediffusion s&apos;est terminée avant qu&apos;un ANR ne se produise.

Pour un contexte de diagnostic complet, les logs sont présents dans l&apos;interface utilisateur de Session Replay. Si Session Replay est activé et actif (via un échantillonnage standard, un échantillonnage en mode erreur ou des API manuelles), l&apos;agent capturera également les logs distants pour cette session même si la session n&apos;a pas été initialement échantillonnée pour le logging, à condition que la fonctionnalité de logging soit globalement activée.
