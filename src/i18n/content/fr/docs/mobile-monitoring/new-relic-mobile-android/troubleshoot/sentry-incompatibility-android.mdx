## Problème

L&apos;intégration de l&apos;agent Android New Relic et du SDK monitoring Sentry peut entraîner des conflits, des plantages ou des erreurs au sein de votre application.

Ceci est connu pour provoquer des problèmes tels que `java.lang.StackOverflowError`.

## Solution

Vous devrez empêcher l&apos;agent Android d&apos;instrumenter le SDK de monitoring Sentry. Pour ce faire, configurez l’agent New Relic pour exclure le package Sentry de l’instrumentation. Voici comment :

* Ajoutez le bloc suivant à votre fichier `build.gradle` :

  ```groovy
  newrelic {
    // Don't instrument sample classes
    excludePackageInstrumentation("io.sentry.*")
  }
  ```

Cette configuration indique à l&apos;agent New Relic d&apos;ignorer toutes les classes instrumentées dans le package `io.sentry`.
