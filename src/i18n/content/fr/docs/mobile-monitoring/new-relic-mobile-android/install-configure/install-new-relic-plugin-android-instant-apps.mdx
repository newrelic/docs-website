[Les applications instantanéesAndroid ](https://developer.android.com/topic/instant-apps/index.html)permettent à l&amp;apos;utilisateur d&amp;apos;utiliser rapidement vos applications sans installation. Avec la Android [version 5.17.0 de](/docs/release-notes/mobile-release-notes/android-release-notes/android-5170) l&amp;apos;agent et supérieure, vous pouvez utiliser New Relic pour détecter et instrumenter tous les modules de fonctionnalités pour une application instantanée.

Utilisez cette documentation avec les procédures New Relic standard pour [installer des applications Android avec Gradle et Android Studio](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/install-android-apps-gradle-android-studio). Après avoir suivi les procédures d&amp;apos;installation standard, suivez ces étapes supplémentaires pour les applications instantanées Android.

## Terminologie

Cette procédure utilise les termes suivants qui sont spécifiques au plug-in Android Instant Apps :

* <DNT>**Module**</DNT>:Toute classe de package et/ou collection de ressources
* <DNT>**APK**</DNT>:Un module qui regroupe les ressources des applications et les classes DEX
* <DNT>**Feature module**</DNT>:Un module allégé qui ne contient qu&amp;apos;un sous-ensemble de l&amp;apos;APK total

## Installer le plugin [#install]

Pour installer le plugin New Relic pour les applications instantanées Android :

1. Suivez d’abord les étapes de la documentation [Installer des applications Android avec Gradle et Android Studio](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/install-android-apps-gradle-android-studio) .

2. Appliquez le plugin d'agent dans chaque fichier de build Gradle du projet.

3. Répertoriez l’agent Android comme une dépendance.

4. Démarrez l'agent à partir de chaque [classe d'activité](https://developer.android.com/reference/android/app/Activity.html) dans la fonctionnalité qui pourrait être chargée latéralement via une intention de lien d&amp;apos;actif numérique (par exemple, mobile.example.com/signin).

   <CollapserGroup>
     <Collapser id="build.gradle" title="Exemple de build.gradle minimal">
       ```gradle
       apply plugin: 'com.android.[application|library|feature|instantapp]'
       apply plugin: 'newrelic'

       dependencies {
           implementation "com.newrelic.agent.android:android-agent:${project.agentVersion}"
       }
       ```
     </Collapser>
   </CollapserGroup>

5. Passez en revue les considérations relatives à [l’instrumentation](#instrument).

## Prise en charge de ProGuard et Dexguard [#proguard]

Le plugin Android Studio Gradle masquera toute configuration de build où `minifyEnabled` est défini sur `true`. Dans ces conditions, l&amp;apos;agent New Relic transmettra la carte ProGuard générée pour le build au crash collecteur, étiquetée avec un ID de build unique généré par l&amp;apos;agent.

L'obfuscation est mieux configurée dans l'application ou le projet d'application instantanée qui inclut d'autres modules en tant que dépendance. Les modules peuvent configurer ProGuard selon les besoins.

<Callout variant="important">
  Lorsque vous utilisez ProGuard ou DexGuard avec des applications Android, suivez les procédures pour effectuer les étapes de configuration supplémentaires requises. Pour plus d'informations, consultez la documentation [de configuration de ProGuard et DexGuard](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/configure-proguard-or-dexguard-android-apps) .
</Callout>

## Dépannage de l'auto-instrumentation [#troubleshooting]

Si vous ne savez pas si l'agent est auto-instrumenté dans un projet d'application instantanée, essayez d'exécuter ces étapes :

1. Créez l'application avec l'option `debug` (`-d`, `--debug`) et capturez le log de génération.

2. Recherchez dans le log des preuves d'applications instantanées en recherchant l'un de ces termes :

   * ```
     This appears to be an Instant App
     ```
   * ```
     BuildId[1a2b34c5-def6-7890-g123-h4567890a]
     ```

3. L'agent ajoutera un attribut nommé `'instantApp'` à l&amp;apos;attribut de session de l&amp;apos;application si l&amp;apos;application semble être une application instantanée.
