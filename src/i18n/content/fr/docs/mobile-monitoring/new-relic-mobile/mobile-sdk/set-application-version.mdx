      ## Syntaxe [#syntax]

      ### Java [#java]

      ```java
      NewRelic.withApplicationVersion(string $appVersion)
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.withApplicationVersion(appVersion: String?)
      ```

      ## Description [#description]

      Par défaut, l&apos;agent Android New Relic obtient la version de l&apos;application à partir de `AndroidManifest.xml` ou `build.gradle`. Cette méthode vous permet de remplacer cette valeur et de définir une version d&apos;application en transmettant la version de l&apos;application. La version de l&apos;application s&apos;affiche sous forme de liste déroulante dans l&apos;interface utilisateur de New Relic, et vous pouvez l&apos;utiliser pour trier les interactions.

      ## Exemple [#example]

      ### Java [#java]

      ```java
      NewRelic.withApplicationToken("").withApplicationVersion("1.23.4").start(this.getApplication());
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.withApplicationToken("").withApplicationVersion("1.23.4").start(applicationContext)
      ```

      ## Syntaxe [#syntax]

      ### Objectif-C [#objc]

      ```objectivec
      setApplicationVersion:(NSString *_Nonnull)versionString;
      ```

      ### Swift [#swift]

      ```swift
      NewRelic.setApplicationVersion:(NSString*)versionString;
      ```

      ## Description [#description]

      Par défaut, New Relic utilise `CFBundleShortVersionString` lors de la signalisation du numéro de build. Mais vous pouvez remplacer le numéro de build signalé en appelant cette méthode (avant d&apos;appeler `startWithApplicationToken`).
