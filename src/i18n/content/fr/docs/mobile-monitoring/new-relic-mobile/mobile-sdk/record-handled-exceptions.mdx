      ## Syntaxe [#android-syntax]

      ### Java [#android-syntax-java]

      )
      ```

      NewRelic.recordHandledException(throwable: Throwable?)
      NewRelic. recordHandledException(
        throwable: Throwable?,
        attributes: Map<String?, Any?>?
      )
      ```

      ## Description [#android-description]

      Enregistre une exception gérée ou un autre type jetable. Prend éventuellement une carte avec un attribut supplémentaire indiquant le contexte.

      Utilisez `recordHandledException()` dans un bloc `try{...} catch(){...}` pour vous aider à comprendre à quelle fréquence votre application génère des exceptions et dans quelles conditions.

      En plus de l&apos;attribut personnalisé associé, l&apos;événement aura également [un attribut de session](/docs/insights/insights-data-sources/default-attributes/mobile-default-attributes-insights) associé. Vous pouvez visualiser les données d&amp;apos;événements dans l&amp;apos;interface utilisateur monitoring des applications mobiles dans le [Crash suivi des événements](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail), ou via NRQL.

      ## Syntaxe [#ios-syntax]

      ### Objectif-C

      ```objectivec
      recordHandledException:(NSException* __nonnull)exception withAttributes:(NSDictionary* __nullable)attributes;
      ```

      ## Description [#ios-description]

      Enregistre une exception gérée (Objective-c uniquement). Prend éventuellement une carte avec un attribut supplémentaire indiquant le contexte.

      L&apos;API `recordHandledException` est utile pour l&amp;apos;analyse des plantages ; l&amp;apos;événement capturé vous aidera à comprendre à quelle fréquence votre application génère des exceptions et dans quelles conditions. En plus de l&amp;apos;attribut personnalisé associé, l&amp;apos;événement aura également [un attribut de session](/docs/insights/insights-data-sources/default-attributes/mobile-default-attributes-insights) associé.

      Cette API prend une instance d&apos;un dictionnaire d&apos;attributs `NSException` et `NSDictionary` facultatif, puis crée un événement `recordHandledException`. Vous pouvez afficher les données d&amp;apos;événement dans l&amp;apos;[interface utilisateur de Crash suivi des événements](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail) et les requêter avec NRQL.

      <Callout variant="important">
        Cette fonction ne doit pas être utilisée avec du code Swift. Veuillez utiliser [`recordError`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-errors/) pour suivre les erreurs traitées dans le code Swift.
      </Callout>

      ## Exemples [#ios-examples]

      Voici un exemple d’exception simple gérée :

      ```objectivec
      try {
        @throw [NSException exceptionWithName:@"versionException"
                                       reason:@"App version no longer supported"
                                     userInfo:nil];
      } @catch(NSException* e) {
        [NewRelic recordHandledException:e];
      }
      ```

      Voici un autre exemple d&apos;exception gérée avec un dictionnaire :

      ```objectivec
      NSException* exception = [NSException exceptionWithName:@"MyException"
                                                       reason:@"I have my reason"
                                                     userInfo:nil];

      NSDictionary* dictionary = @{@"int" : @1, @"Test Group" : @"A | B"};

      [NewRelic recordHandledException:exception withAttributes:dictionary];
      ```
