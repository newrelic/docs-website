      ## Syntaxe [#syntax]

      ### Java [#java]

      ```java
      NewRelic.setMaxEventPoolSize(int $maxSize)
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.setMaxEventPoolSize(maxSize: Int)
      ```

      ## Description [#description]

      Définit la taille maximale du pool d&apos;événements.

      Par défaut, <InlinePopover type="mobile" />collecte un maximum de 1 000 événements par cycle de collecte d&amp;apos;événements, soit une durée de 600 secondes par défaut. Cette méthode contrôle la taille maximale du pool d&amp;apos;événements stocké dans la mémoire jusqu&amp;apos;au prochain cycle de collecte. Lorsque la limite de taille du pool est atteinte, l&amp;apos;agent Android New Relic commence [à échantillonner les événements](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling), en supprimant certains anciens et certains nouveaux événements, jusqu&amp;apos;à ce que le pool d&amp;apos;événements soit transmis avec le prochain cycle de collecte. Cette méthode vous permet de remplacer la taille maximale de ce pool d’événements.

      La valeur par défaut du cycle d&apos;événement de collecte est de 600 secondes. Voir [Définir la durée maximale de la mémoire tampon de l&apos;événement](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-buffer-time) pour modifier la durée du cycle de collecte de l&amp;apos;événement.

      <Callout variant="important">
        Gardez à l&apos;esprit que le signalement d&apos;un grand nombre d&apos;événements, ou le signalement d&apos;événements trop fréquents, peut avoir un impact sur les performances de l&apos;application.
      </Callout>

      ## Syntaxe [#syntax]

      ### Objective-C [#objc]

      ```objectivec
      + (void) setMaxEventPoolSize:(unsigned int)size;
      ```

      ### Swift [#swift]

      ```swift
      NewRelic.setMaxEventPoolSize(unsigned int $size)
      ```

      ## Description [#description]

      Définit la taille maximale du pool d&apos;événements.

      Par défaut, l&apos;agent iOS collecte un maximum de 1 000 événements par [cycle de collecte](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#harvest-cycle). Cette méthode contrôle la taille maximale du pool d&amp;apos;événements stocké dans la mémoire jusqu&amp;apos;au prochain cycle de collecte. Lorsque la limite de taille du pool est atteinte, l&amp;apos;agent iOS New Relic commence [à échantillonner les événements](/docs/agents/manage-apm-agents/agent-data/new-relic-events-limits-sampling), en supprimant certains anciens et certains nouveaux événements, jusqu&amp;apos;à ce que le pool d&amp;apos;événements soit transmis avec le prochain cycle de collecte. Cette méthode vous permet de remplacer la taille maximale de ce pool d’événements. Vous devez définir cette valeur après l&amp;apos;appel de `Agent.start()` .

      La valeur par défaut du cycle d&apos;événement de collecte est de 600 secondes. Voir également [Définir la durée maximale de la mémoire tampon de l&apos;événement](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-buffer-time/) pour modifier la durée du cycle de collecte de l&amp;apos;événement.

      Pour garantir que la nouvelle valeur est toujours appliquée, placez cet appel d&apos;API dans la méthode du cycle de vie iOS `applicationDidBecomeActive` .
