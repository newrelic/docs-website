      ## Syntaxe [#syntax]

      ### Java [#java]

      ```java
            NewRelic.recordBreadcrumb(
                        string $Name,
                        map<string, object> $eventAttributes
                    )
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.recordBreadcrumb(
                  breadcrumbName: String?,
                  attributes: MutableMap<String?, Any?>?
              )
      ```

      ## Description [#description]

      Cet appel crée et enregistre un événement `MobileBreadcrumb`, qui peut être requêter avec NRQL et dans le [crash suivi des événements](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail). Les fils mobiles d&apos;Ariane sont utiles pour l&apos;analyse des crashs ; ils doivent être créés pour l&apos;activité des applications qui peuvent être utiles pour le dépannage des crashs.

      En plus de tout attribut personnalisé que vous choisissez, l&apos;événement aura également [un attribut de session](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list) associé. Contrairement à l&apos;utilisation de [`setAttribute`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/create-attribute/), l&apos;ajout d&apos;un attribut à un événement fil d&apos;Ariane les ajoute uniquement à cet événement ; ce ne sont pas des attributs de session.

      En plus de tout attribut personnalisé que vous choisissez, l&apos;événement aura également [un attribut de session](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list) associé.       L’utilisation de cet appel a le même résultat que l’utilisation de l’ [appel`recordCustomEvent` ](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-custom-events/)avec `MobileBreadcrumb` comme type d’événement. Pour plus de contexte sur la façon d&apos;utiliser cette API, lisez à propos de l&apos;envoi d&apos;attributs personnalisés et d&apos;événements :

      ## Description [#description]

      Cet appel crée et enregistre un événement `MobileBreadcrumb`, qui peut être requêter avec NRQL et dans le [crash suivi des événements](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-crash-event-trail). Les fils mobiles d&apos;Ariane sont utiles pour l&apos;analyse des crashs ; ils doivent être créés pour l&apos;activité des applications qui peuvent être utiles pour le dépannage des crashs.

      En plus de l&apos;attribut personnalisé que vous choisissez, l&apos;événement aura également [un attribut de session](/docs/insights/explore-data/attributes/mobile-default-attributes-insights#mobile-list) associé.
