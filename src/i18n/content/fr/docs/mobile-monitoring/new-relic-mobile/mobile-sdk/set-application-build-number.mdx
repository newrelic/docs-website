      <Callout variant="important">
        À partir de la version 5.3.0 de l&apos;agent Android New Relic, `withBuildIdentifier()` a remplacé `NewRelic.withApplicationBuild()`.
      </Callout>

      ## Syntaxe [#syntax]

      ### Java [#java]

      ```java
      NewRelic.withApplicationBuild(string $buildId)
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.withApplicationBuild(buildId: String?)
      ```

      ## Description [#description]

      Cet appel vous permet de définir une chaîne d&apos;identifiant de build personnalisée qui apparaît à côté de la version de l&apos;application dans la [page du rapport de plantage](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-apps-crash-report-dashboard#details). Au lieu d&apos;utiliser la chaîne `versionCode` définie dans `AndroidManifest.xml`, appelez la méthode `withApplicationBuild()` et transmettez l&apos;ID de build personnalisé sous forme de chaîne.

      ## Exemple [#example]

      ### Java [#java]

      ```java
      NewRelic.withApplicationToken("").withApplicationBuild("156").start(this.getApplication());
      ```

      ### Kotlin [#kotlin]

      ```kotlin
      NewRelic.withApplicationToken("").withApplicationBuild("1").start(applicationContext)
      ```

    <TabsPageItem id="ios">
      ## Syntaxe [#syntax]

      ### Objectif-C [#objc]

      ```objectivec
      setApplicationBuild:(NSString *_Nonnull)versionString;
      ```

      ### Swift [#swift]

      ```swift
      NewRelic.setApplicationBuild:(NSString*)versionString;
      ```

      ## Description [#description]

      Par défaut, New Relic utilise `CFBundleVersion` lors de la signalisation du numéro de build. Mais vous pouvez remplacer le numéro de build signalé en appelant cette méthode (avant d&apos;appeler `startWithApplicationToken`).
