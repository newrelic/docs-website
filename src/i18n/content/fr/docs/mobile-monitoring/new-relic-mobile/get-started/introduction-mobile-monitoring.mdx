Les fonctionnalités <InlinePopover type="mobile" />de New Relic offrent une visibilité plus approfondie sur les performances et le dépannage des pannes de vos applications mobiles Android, iOS ou hybrides. Vous pouvez utiliser monitoring des applications mobiles pour améliorer l&apos;expérience utilisateur de votre application ou examiner les performances HTTP et réseau pour collaborer plus efficacement avec vos équipes backend .

Une fois que vous aurez suivi notre processus d&apos;installation facile, vous pourrez configurer davantage monitoring des applications mobiles en fonction de votre environnement mobile.

## Identifier les opportunités pour accélérer les performances des applications [#speed-performance]

* Créez [un dashboard](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) pour interroger, visualiser et partager les données de performance.
* Identifiez [les exceptions traitées](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/introduction-mobile-handled-exceptions) pour rationaliser les workflows.
* Consultez les tendances avec les sessions, les appareils, les emplacements géographiques, le système d&apos;exploitation, les opérateurs, requests et les réponses, etc. avec [des profils de crash](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/crash-analysis-group-filter-your-crashes/#profiles) et [des profils HTTP](/docs/errors-inbox/group-request-errors/#error-profiles).
* Configurer [une notification d&apos;alerte](/docs/mobile-monitoring/new-relic-mobile/getting-started/alert-information-new-relic-mobile) pour le temps d&apos;exécution, les erreurs, etc.
* Définissez [des attributs personnalisés et des événements](/docs/insights/insights-data-sources/custom-events/insert-custom-events-attributes-mobile-data) pour obtenir des détails supplémentaires et spécifiques sur les données mobiles les plus importantes pour votre organisation.

## problèmes backend du moniteur [#backend]

Vous pouvez utiliser les [pages réseau](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/analyze-network-requests-using-mobilerequest-event-data/) dans notre interface utilisateur monitoring des applications mobiles pour identifier les problèmes qui surviennent via requests HTTP, les erreurs et d&apos;autres problèmes de réseau. Vous pouvez également [interroger vos données](/docs/nrql/nrql-examples/nrql-query-examples-mobile-monitoring/) et créer des <InlinePopover type="dashboards" />personnalisés à partager avec les équipes concernées comme vos développeurs d&apos;API backend .
