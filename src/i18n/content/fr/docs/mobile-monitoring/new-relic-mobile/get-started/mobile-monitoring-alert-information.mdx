Des <InlinePopover type="alerts" />bien définis aident à informer les individus ou les équipes des changements dans leur système. Vous pouvez utiliser n&apos;importe laquelle de nos fonctionnalités [d&apos;alerte](/docs/alerts-applied-intelligence) sur toutes les entités que vous monitorez avec New Relic.

Par exemple, vous pouvez utiliser [des alertes](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts) pour gérer la règle d&apos;alerte et les conditions qui se concentrent sur les métriques des applications mobiles que vous monitorez avec New Relic. Après avoir [défini la condition d&apos;alerte](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) pour vos applications mobiles, vous pouvez afficher ces informations dans l&apos;interface utilisateur des alertes ou dans d&apos;autres expériences d&apos;interface utilisateur, comme l&apos;interface utilisateur <InlinePopover type="mobile" />.

## État d&apos;alerte mobile [#conditions]

Utilisez l’une des conditions et seuils suivants lorsque vous configurez des alertes pour vos applications mobiles :

<CollapserGroup>
  <Collapser id="execution" title="Temps d'exécution">
    Les mesures de temps d&apos;exécution incluent <DNT>**Database**</DNT>, <DNT>**Images**</DNT>, <DNT>**JSON**</DNT>, <DNT>**Network**</DNT> et <DNT>**View loading**</DNT>. Définissez le seuil pour ces métriques en sélectionnant un temps et une durée.
  </Collapser>

  <Collapser id="errors" title="Erreurs">
    Les mesures d&apos;erreur incluent <DNT>**Network error percentage**</DNT> et <DNT>**Status error percentage**</DNT>. Définissez le seuil de ces métriques en sélectionnant un pourcentage d&apos;erreur et une durée.
  </Collapser>

  <Collapser id="crash" title="Rapports sur les plantages">
    Vous souhaiterez peut-être être averti lorsque <DNT>**crash rate**</DNT> de votre application dépasse un certain seuil. Définissez le seuil de ces métriques en sélectionnant un pourcentage d&apos;erreur et une durée.

    <Callout variant="tip">
      Soyez averti du dépassement de votre taux de plantation d&apos;applications dès que cela se produit en configurant [un canal de notification](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts), notamment PagerDuty et Slack.
    </Callout>
  </Collapser>

  <Collapser id="custom" title="Coutume">
    Créez votre propre <DNT>**custom metric**</DNT> pour répondre à des besoins d&apos;alerte spécifiques. Ajoutez un nom pour la métrique personnalisée et définissez votre propre seuil.
  </Collapser>
</CollapserGroup>

## Voir les politiques et conditions [#policy]

Pour afficher les règles d&apos;alerte et les informations sur l&apos;état d&apos;une application mobile spécifique :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile &gt; Alert conditions**</DNT>.
2. Depuis la page <DNT>**Alert conditions**</DNT> , utilisez les outils disponibles pour rechercher, trier, afficher ou mettre à jour la condition d&apos;alerte et les politiques associées.

## Voir l&apos;événement et les activités [#activity]

Pour afficher des informations récapitulatives sur les [événements](/docs/alerts/overview#event) et d&apos;autres activités directement depuis l&apos;interface de monitoring mobile :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Mobile**</DNT>.
2. Depuis l&apos;index, survolez le [statut de santé codé par couleur](/docs/alerts/overview#health-status) de l&apos;entité, sélectionnez un lien dans la liste <DNT>**Mobile activity**</DNT> ou sélectionnez une application mobile pour afficher des détails supplémentaires.

Par exemple, si un événement d&apos;alerte <DNT>**Critical**</DNT> se produit :

* L&apos;indicateur d&apos;état de santé devient rouge sur l&apos;index mobile et sur l&apos;application sélectionnée.
* La couleur d’arrière-plan de divers graphiques devient rose.
* Dans votre liste d&apos;applications mobiles monitorées, la section <DNT>**Application activity**</DNT> affiche les événements d&apos;alerte <DNT>**Warning**</DNT> (jaunes) et <DNT>**Critical**</DNT> (rouges), le cas échéant.

Pour en savoir plus sur un événement d&apos;alerte, survolez ou sélectionnez n&apos;importe quelle zone rose sur un graphique.

## Voir les événements d&apos;alerte [#alert-events]

<Callout variant="important">
  En juillet 2020, nous avons migré les événements d&apos;alerte pour les applications de navigateur, les applications mobiles et les moniteurs synthétiques vers un nouveau format dans <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>. Pour plus d&apos;informations, consultez la documentation sur les [alertes](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai).
</Callout>

Si une condition d&apos;alerte a des [seuils](/docs/alerts/overview#alert-threshold) configurés pour des événements d&apos;alerte <DNT>**Warning**</DNT> (jaune) ou <DNT>**Critical**</DNT> (rouge), le [statut de santé](/docs/alerts/overview#health-status) (code couleur) d&apos;une entité de produit changera pour indiquer un événement d&apos;alerte. Vous pouvez consulter les événements d&apos;alerte directement depuis la page de l&apos;application mobile dans New Relic :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Mobile**</DNT>.
2. Sélectionnez une application mobile, puis vérifiez son <DNT>**Open alert events**</DNT>.
