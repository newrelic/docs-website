Des <InlinePopover type="alerts" />bien définis aident à informer les individus ou les équipes des changements dans leur système. Vous pouvez utiliser n&amp;apos;importe laquelle de nos fonctionnalités [d&apos;alerte](/docs/alerts-applied-intelligence) sur toutes les entités que vous monitorez avec New Relic.

Par exemple, vous pouvez utiliser [des alertes](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts) pour gérer la règle d&amp;apos;alerte et les conditions qui se concentrent sur les métriques des applications mobiles que vous monitorez avec New Relic. Après avoir [défini la condition d&apos;alerte](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) pour vos applications mobiles, vous pouvez afficher ces informations dans l&amp;apos;interface utilisateur des alertes ou dans d&amp;apos;autres expériences d&amp;apos;interface utilisateur, comme l&amp;apos;interface utilisateur <InlinePopover type="mobile" />.

## État d&apos;alerte mobile [#conditions]

Utilisez l’une des conditions et seuils suivants lorsque vous configurez des alertes pour vos applications mobiles :

<CollapserGroup>
  <Collapser id="execution" title="Temps d'exécution">
    Les mesures de temps d&apos;exécution incluent <DNT>**Database**</DNT>, <DNT>**Images**</DNT>, <DNT>**JSON**</DNT>, <DNT>**Network**</DNT> et <DNT>**View loading**</DNT>. Définissez le seuil pour ces métriques en sélectionnant un temps et une durée.
  </Collapser>

  <Collapser id="errors" title="Erreurs">
    Les mesures d&apos;erreur incluent <DNT>**Network error percentage**</DNT> et <DNT>**Status error percentage**</DNT>. Définissez le seuil de ces métriques en sélectionnant un pourcentage d&amp;apos;erreur et une durée.
  </Collapser>

  <Collapser id="crash" title="Rapports sur les plantages">
    Vous souhaiterez peut-être être averti lorsque <DNT>**crash rate**</DNT> de votre application dépasse un certain seuil. Définissez le seuil de ces métriques en sélectionnant un pourcentage d&amp;apos;erreur et une durée.

    <Callout variant="tip">
      Soyez averti du dépassement de votre taux de plantation d&apos;applications dès que cela se produit en configurant [un canal de notification](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts), notamment PagerDuty et Slack.
    </Callout>
  </Collapser>

  <Collapser id="custom" title="Coutume">
    Créez votre propre <DNT>**custom metric**</DNT> pour répondre à des besoins d&amp;apos;alerte spécifiques. Ajoutez un nom pour la métrique personnalisée et définissez votre propre seuil.
  </Collapser>
</CollapserGroup>

## Voir les politiques et conditions [#policy]

Pour afficher les règles d&apos;alerte et les informations sur l&apos;état d&apos;une application mobile spécifique :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile &amp;gt; Alert conditions**</DNT>.
2. Depuis la page <DNT>**Alert conditions**</DNT> , utilisez les outils disponibles pour rechercher, trier, afficher ou mettre à jour la condition d&amp;apos;alerte et les politiques associées.

## Voir l&apos;événement et les activités [#activity]

Pour afficher des informations récapitulatives sur [les événements](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) et autres activités directement depuis l&amp;apos;interface utilisateur monitoring des applications mobiles :

1. Allez à <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile**</DNT>.
2. À partir de l&apos;index, passez la souris sur l&apos; [état de santé codé par couleur](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) de l&amp;apos;entité, sélectionnez un lien dans la liste <DNT>**Mobile activity**</DNT> ou sélectionnez une application mobile pour afficher des détails supplémentaires.

Par exemple, si un incident d&apos;alerte <DNT>**Critical**</DNT> se produit :

* L&apos;indicateur d&apos;état de santé devient rouge sur l&apos;index mobile et sur l&apos;application sélectionnée.
* La couleur d’arrière-plan de divers graphiques devient rose.
* Dans votre liste d&apos;applications mobiles de monitoring, la section <DNT>**Application activity**</DNT> affiche les incidents <DNT>**Warning**</DNT> (jaune) et <DNT>**Critical**</DNT> (rouge) selon le cas.

Pour en savoir plus sur un incident d’alerte, passez la souris sur ou sélectionnez une zone rose sur un graphique.

## Voir l&apos;incident d&apos;alerte [#incidents]

<Callout variant="important">
  En juillet 2020, nous avons transféré l&apos;incident d&apos;alerte pour les applications de navigateur, les applications mobiles et le moniteur synthétique vers un nouveau format dans <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>. Pour plus d&amp;apos;informations, consultez la documentation [relative aux alertes](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai) .
</Callout>

Si une condition d&apos;alerte a [un seuil](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) défini pour un incident <DNT>**Warning**</DNT> (jaune) ou <DNT>**Critical**</DNT> (rouge), l&amp;apos; [état de santé](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) codé par couleur pour une entité de produit changera pour indiquer un incident. Vous pouvez visualiser l&amp;apos;incident directement depuis la page de l&amp;apos;application mobile dans New Relic :

1. Allez à <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &amp;gt; Mobile**</DNT>.
2. Sélectionnez une application mobile, puis vérifiez son <DNT>**Open incidents**</DNT>.
