## Afficher votre jeton d&apos;application [#application-token]

Votre jeton d&apos;application mobile est une chaîne hexadécimale unique de 40 caractères qui authentifie les données de votre application auprès de New Relic. Chaque application mobile que vous monitorez dispose d&apos;un jeton unique pour le centre de données américain et européen.

Toutes les données de votre application mobile sont signalées au même endroit dans l&apos;interface utilisateur monitoring des applications mobiles New Relic, quel que soit le nom de l&apos;application lorsqu&apos;elle est installée sur un appareil mobile. Si vous utilisez un framework hybride tel que Flutter ou React Native et que vous le déployez à la fois sur les plateformes iOS et Android , vous devrez utiliser des jetons séparés, un pour iOS et un pour Android.

Pour afficher votre jeton d’application :

1. Accédez à <DNT>**[one.newrelic.com &gt; All Capabilities](https://one.newrelic.com/all-capabilities)**</DNT>, cliquez sur <DNT>**Mobile**</DNT> et sélectionnez une application de monitoring.
2. Sélectionnez <DNT>**Application**</DNT> ou <DNT>**Installation**</DNT>.

## configuration côté serveur [#configure-server-side-settings]

configuration de l&apos;agent mobile côté serveur fournit un mécanisme permettant de gérer et d&apos;ajuster à distance le comportement des agents mobiles déployés sur les appareils des utilisateurs. Cette fonctionnalité vous permet de contrôler dynamiquement les paramètres monitoring sans nécessiter le déploiement d&apos;un nouveau App Store.

Le paramètre suivant peut actuellement être modifié à distance sur la page **Application** :

* Application Not Responding (ANR) suivi des erreurs (Android uniquement) : permet de suivre et de signaler les raisons des erreurs ANR. Cette fonctionnalité est activée par défaut et nécessite l&apos;agent Android 7.4.1 ou supérieur. <Callout variant="important">configuration côté serveur a toujours priorité sur configuration de l&apos;agent local. Pour les applications mobiles avec moins de 100 000 appareils actifs simultanément, l&apos;application des modifications aux appareils finaux de l&apos;utilisateur peut prendre jusqu&apos;à 10 minutes. Pour les applications avec plus de 100 000 appareils actifs, cela peut prendre plusieurs heures.</Callout>

### Configurer à partir de l&apos;interface utilisateur [#ui-configuration]

Pour accéder aux paramètres configuration côté serveur pour monitorer les applications à partir de l&apos;interface utilisateur, accédez à <DNT>**[one.newrelic.com &gt; All Capabilities](https://one.newrelic.com/all-capabilities)**</DNT> &amp;gt; <DNT>**Mobile**</DNT> (sélectionnez votre application) &amp;gt; <DNT>**Settings**</DNT> &amp;gt; <DNT>**Application**</DNT>.

Cela vous permet de configurer certains paramètres dans l&apos;interface utilisateur. Cela applique automatiquement vos modifications à tous les agents sans avoir à redéployer une nouvelle application sur l&apos;App Store.
