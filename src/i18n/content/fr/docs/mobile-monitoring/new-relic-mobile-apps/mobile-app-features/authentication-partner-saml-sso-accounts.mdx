<Callout variant="important">
  Cette ressource est destinée aux partenaires de New Relic. Pour l'authentification de l'utilisateur dans les comptes New Relic classiques, voir [Authentification](/docs/accounts/accounts-billing/new-relic-one-user-management/user-authentication-saml-sso-scim-more).
</Callout>

Les utilisateurs de comptes partenaires utilisent généralement SAML-SSO pour se connecter via votre site partenaire New Relic. Vous ne pouvez pas disposer de mots de passe ou d'informations d'authentification distincts pour votre compte New Relic.

Si vous utilisez une adresse e-mail associée à un compte partenaire New Relic lors de votre première connexion à l'application mobile New Relic, New Relic vous enverra un e-mail de confirmation pour l'authentification. L'utilisateur de l'application Android verra également un message notification .

<Callout variant="important">
  L'e-mail d'authentification [expire 20 minutes](#email) après son envoi.
</Callout>

## Confirmez votre adresse email [#authentication]

Pour vous authentifier à l'aide d'un compte SAML-SSO fourni par un partenaire New Relic :

1. Depuis l'application mobile New Relic, saisissez votre adresse e-mail associée au compte partenaire.
2. Sélectionnez <DNT>**I don't have a password**</DNT>.
3. Récupérez l'email d'authentification depuis votre appareil mobile dans les 20 minutes.
4. Sélectionnez le bouton <DNT>**Authenticate**</DNT> (utilisateur Android ) ou le lien e-mail (utilisateur Android ou iOS) dans l&amp;apos;e-mail pour vous connecter à New Relic.

Vous serez redirigé vers l'application mobile New Relic et vous connecterez à votre compte partenaire.

## Résoudre les problèmes de courrier électronique [#email]

Voici quelques conseils de dépannage :

* Si vous ne trouvez pas le message d'authentification de New Relic dans la boîte de réception de votre appareil mobile, vérifiez votre dossier <DNT>**Spam**</DNT> .
* Si vous manquez le délai de 20 minutes, connectez-vous à nouveau à l'application mobile New Relic, puis sélectionnez le lien pour renvoyer l'e-mail d'authentification.
