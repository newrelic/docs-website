外形監視を使用して、世界中からの接続をシミュレートします。 ソースポイントと出力の間に中断がある場合、合成チェックは失敗し、サービスレベル目標スコアに影響を与え、非常に重大なアラートを引き起こします。 これを入力パフォーマンスと呼びます。

基本的に、入力パフォーマンスはアプリケーションに接続する能力として測定され、「稼働時間」とも呼ばれます。 入力パフォーマンス SLI を設定する手順と、試してみる NRQL クエリの例を以下に示します。

<Steps>
  <Step>
    ## 合成チェックを作成する

    最も一般的な入力パフォーマンスのサービスレベルは、「接続性」または「稼働時間」と呼ばれることがよくあります。 これは、ヘルス API エンドポイントまたは URL の読み込みに対する簡単なチェックです。 外形監視サービスを利用すると、これらの両方を行うことができます。 サービスレベル インジケーターを作成する前に、 [「単純なbrowserモニターの追加](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#simple)」および[「スクリプト API テストの追加」](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors#complex)を参照して、データのレポートをまだ開始していない場合はその開始方法を確認してください。
  </Step>

  <Step>
    ## サービスレベルインジケーターを作成する

    データレポートを取得したら、サービスレベル管理サービスを使用して、次の手順で入力インジケーターと目標を作成できます。

    1. [entity explorer](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts#find) \[エンティティ エクスプローラー]から、**Service levels** \[サービスレベル]を選択します。

    2. **+ Add a service level indicator** \[+ サービスレベルインジケーターの追加]をクリックします。 **Add a service level** \[サービスレベルの追加]ボタンが表示されない場合は、New Relic アドミニストレーターに権限について確認してください。

    3. 以下のスクリーンショットのように、エンティティ タイプを`Synthetic monitors`にフィルターします。

       <img
         alt="Filter entity types in service levels"
         src="/images/apm_screenshot-crop_add-input-service-level.webp"
       />

    4. リストから外形監視モニターを見つけてクリックします。 これにより、左側のパネルの**Continue** \[続行]ボタンが有効になります。 **Continue** \[続行]をクリックします。

    5. **成功**サービスレベルの推奨設定のボタンが表示されます（以下を参照）。クリックして。

       <img
         alt="Synthetics input service level guided flow"
         src="/images/apm_screenshot-crop_synthetics-setup.webp"
       />

    6. 必要に応じて、タグ、タイトル、説明に適切な変更を加えます。

    7. クリック **保存**.
  </Step>
</Steps>
