New Relicユーザーは、複数のデータベース、データ ウェアハウス、スプレッドシートに保存されているデータと並行してテレメトリー データを分析する必要がある場合があります。 各ソースからデータを手動でエクスポートし、オフラインでの観察データと組み合わせるには時間がかかり、生産性が低下します。 さらに、コンプライアンス、セキュリティ、または運用上の要件により、すべてのデータを単一のデータベースに統合できないこともよくあります。

<DNT>New Relic Lens</DNT> これらの課題に対処します。<DNT>Lens</DNT>は、New Relic 内の統合クエリ プラットフォームであり、New Relic から直接複数のデータ ソースに接続してクエリを実行できます。ANSI SQL を使用して<DNT>Lens</DNT>でクエリを記述し、データベース間の結合を実行して、さまざまなソースからのデータを分析および相関させます。<DNT>Lens</DNT>は各データ ソースに計算作業をプッシュし、各データ ソースはクエリの一部を処理して結果のみを返します。<DNT>Lens</DNT>はこれらの結果を結合し、外部ソースから生データを取り込まないことでデータ転送を最小限に抑え、セキュリティを維持します。

## 使い方 [#how-it-works]

<DNT>Lens</DNT>を使用するには、次の手順を実行します。

<Steps>
  <Step>
    ### レンズを有効にする

    <DNT>Lens</DNT> パブリックプレビューとして利用可能です。<DNT>**[Previews &amp; Trials](https://one.newrelic.com/admin-portal/promotion-management/home)**</DNT>ページからオプトインするには、 <DNT>Organization Manager</DNT>に確認してください。
  </Step>

  <Step>
    ### アクセス権限の確認

    <DNT>Organization Manager</DNT>の場合、デフォルトですべての<DNT>Lens</DNT>機能にアクセスするために必要な権限が与えられます。<DNT>Organization Manager</DNT>ではない場合は、 <DNT>Authentication Domain Manager</DNT>に連絡して[、必要な権限を持つカスタム ロールを作成してください](/docs/new-relic-lens/overview/#permissions)。
  </Step>

  <Step>
    ### 設定 <DNT>connectors</DNT>

    <DNT>Lens</DNT> NRDB データ用の組み込み<DNT>system connectors</DNT>が含まれています。必要な権限があれば、 [<DNT>data connectors</DNT>](/docs/new-relic-lens/set-up-connectors)外部データ ソースに接続するように設定できます。

    * **クラウド データ ウェアハウス**: <DNT>Snowflake</DNT> 、 <DNT>Redshift</DNT> 、 <DNT>ClickHouse</DNT>
    * **リレーショナルデータベース**: <DNT>PostgreSQL</DNT> 、 <DNT>MySQL</DNT> 、 <DNT>SQL Server</DNT>
    * **ドキュメントデータベース**: <DNT>MongoDB</DNT> 、 <DNT>Elasticsearch</DNT>
    * **スプレッドシート**: <DNT>Google Sheets</DNT>
    * **データレイク**: <DNT>Iceberg</DNT>
    * **メトリクスと監視**: <DNT>Prometheus</DNT> 、 <DNT>AWS CloudWatch</DNT>
  </Step>

  <Step>
    ### クエリデータ

    `connection_name.schema.table`形式を使用して、書き込みビルダー[、ノートブック、またはカスタムダッシュ](/docs/new-relic-lens/query-data)ボードに SQL 書き込みを書き込み、テーブルを参照し、ソース間でデータを結合します。
  </Step>

  <Step>
    ### <DNT>materialized views</DNT>を作成 (オプション)

    頻繁に実行される複雑なクエリの場合、結果を[<DNT>materialized views</DNT>](/docs/new-relic-lens/materialized-views)として保存してコストを削減し、パフォーマンスを向上させることができます。
  </Step>
</Steps>

## 使用事例 [#use-cases]

Lens はさまざまなデータ分析シナリオを可能にします:

* **予算分析**: <DNT>NRDB</DNT>の New Relic 使用率データと<DNT>Snowflake</DNT>に保存されている予算データを比較して、割り当てられた予算に対する支出を追跡します。
* **ワークフローの最適化**: 複数の手動エクスポート/インポート手順 (1 つのデータベースから別のデータベースへ) を、必要なすべてのデータを結合する単一の SQL クエリに置き換えて、コストを削減し、効率を向上させます。
* **地域全体のデータを集約**: さまざまなデータ ソースからのデータを結合して、さまざまな地理的地域にわたるユーザー トラフィックを追跡します。
* **<DNT>materialized views</DNT>によるパフォーマンスの最適化**: エグゼクティブ ダッシュボードは 5 つ以上のデータベースからのデータを結合し、毎週更新します。マルチデータベース`JOIN`クエリの結果を<DNT>materialized view</DNT>に保存することで、ダッシュボードを読み込むたびに計算コストの高いクエリを実行する必要がなくなります。これによりクエリコストが削減され、ページの読み込み速度が大幅に向上します。

## Lensを使用できる場所 [#where-to-use]

Lens は、New Relic プラットフォームの次の領域で使用できます。

* **書き込みビルダー**: SQL 書き込みビルダーに直接 SQL を記述して実行します。
* <DNT>**Notebooks**</DNT>: ノートブックに SQL ブロックを作成し、クエリを視覚化やマークダウン ドキュメントと組み合わせます。
* **カスタマーDashboards** : SQL ベースのチャートをプライベート ダッシュボードに追加します。

<Callout variant="important">
  SQL ベースのチャートはパブリックダッシュボードではサポートされていません。
</Callout>

## ユーザーロールと権限 [#permissions]

<DNT>Connector</DNT> および<DNT>materialized view</DNT>アクセスは特定の RBAC 権限によって制御されます。<DNT>**Organization Manager**</DNT>ロールを持つユーザーには、デフォルトですべての<DNT>Lens</DNT>関連する RBAC 権限が付与されます。

他のユーザーの場合、 <DNT>**Authentication Domain Manager**</DNT>は必要な<DNT>Lens</DNT>権限を持つ[NerdGraph 経由で組織スコープのカスタム ロールを作成する](/docs/apis/nerdgraph/examples/nerdgraph-manage-groups/#organization-scoped-role)必要があります。ロールを作成するときに、以下の表の権限 ID を`permissionIds`配列に追加します。

<DNT>
  /* UI-based permission assignment (for future use when UI support is available): For other users, the &lt;DNT&gt;**Authentication Domain Manager**&lt;/DNT&gt; needs to create an [organization-scoped custom role](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles) with the required &lt;DNT&gt;Lens&lt;/DNT&gt; permissions from the &lt;DNT&gt;**Query Your Data**&lt;/DNT&gt; section. check... */
</DNT>
