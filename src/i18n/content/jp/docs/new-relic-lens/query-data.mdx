<DNT>Lens</DNT>書き込みビルダー、 <DNT>**Notebooks**</DNT> 、カスタム**ダッシュ****ボード**に書き込むことができます。 このページでは、各プラットフォームでデータをクエリする方法について説明します。

<Callout variant="tip">
  データ ソースをクエリする場合、 <DNT>Lens</DNT>外部ソースから取得できる行数を制限しません。ただし、 <DNT>NRDB</DNT>から<DNT>Lens</DNT>へのクエリでは、 <DNT>NRQL</DNT>と同じ 5,000 行の制限が維持されます。
</Callout>

## 前提条件 [#prerequisites]

始める前に:

* データ ソースをクエリするために必要な[権限](/docs/new-relic-lens/overview#permissions)があることを確認します。
* 外部データ ソースをクエリする必要がある場合は、 [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors)が設定されていることを確認してください。

## クエリビルダーでクエリを実行する [#query-builder]

1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>に移動します。
2. ページの下部にあるドロワーから、 <DNT>**Query your data**</DNT>コンソールを開きます。
3. クエリ エディターで、クエリ言語として<DNT>**Lens data (SQL)**</DNT>を選択します。
4. NRDB データをクエリする場合は、アカウント セレクターからアカウントを選択します。 その他の<DNT>data connectors</DNT>は組織レベルで使用可能であり、アカウントの選択は必要ありません。
5. [テーブル参照形式](#table-format)を使用して SQL クエリを記述します。 `connection_name.schema.table`
6. <DNT>**Run**</DNT>をクリックします。クエリ結果に基づいてグラフが表示されます。

## クエリを保存 <DNT>Notebooks</DNT> [#notebooks]

[ノートブックは](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/)、クエリ、視覚化、マークダウン ドキュメントを組み合わせた共有可能なドキュメントです。

1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>に移動します。
2. ページの下部にあるドロワーから、 <DNT>**Query your data**</DNT>コンソールを開きます。
3. <DNT>**Notebooks**</DNT>の横にあるトグルを有効にします。ページの下部に、 <DNT>**+ SQL**</DNT>ボタンが表示されます。
4. SQL ブロックを追加するには、 <DNT>**+ SQL**</DNT>をクリックします。
5. NRDB データをクエリする場合は、アカウント セレクターからアカウントを選択します。 その他の<DNT>data connectors</DNT>は組織レベルで使用可能であり、アカウントの選択は必要ありません。
6. [テーブル参照形式](#table-format)を使用して SQL クエリを記述します。 `connection_name.schema.table`
7. <DNT>**Run**</DNT>をクリックします。クエリ結果に基づいてグラフが表示されます。
8. <DNT>**Save**</DNT>をクリックします。

詳しいガイダンスについては、[ノートブックのドキュメント](/docs/query-your-data/explore-query-data/notebooks/introduction-notebooks/)を参照してください。

## カスタムダッシュボードでのクエリ [#dashboards]

[カスタムダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards)に SQL ベースのグラフを追加できます。

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Dashboards**</DNT>に移動します。
2. [新しいダッシュボードを作成する](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards#dashboards-create-new)か、既存のダッシュボードを開きます。
3. <DNT>**+ Add widget**</DNT>をクリックしてグラフの種類を選択します。
4. クエリ エディターで、クエリ言語として<DNT>**Lens data (SQL)**</DNT>を選択します。
5. NRDB データをクエリする場合は、アカウント セレクターからアカウントを選択します。 その他の<DNT>data connectors</DNT>は組織レベルで使用可能であり、アカウントの選択は必要ありません。
6. [テーブル参照形式](#table-format)を使用して SQL クエリを記述します。 `connection_name.schema.table`
7. <DNT>**Run**</DNT>をクリックします。クエリ結果に基づいてグラフが表示されます。
8. <DNT>**Add to dashboard**</DNT>をクリックします。詳細については、[カスタムダッシュボードのドキュメント](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/)を参照してください。

<Callout variant="important">
  SQL ベースのチャートはプライベート ダッシュボードでのみサポートされます。パブリックダッシュボードは SQL クエリをサポートしていません。
</Callout>

## SQL構文 [#syntax]

<DNT>Lens</DNT> ANSI SQL を使用してデータ ソースをクエリするため、データベース固有のクエリ言語を学習する必要はありません。

### 表参照形式 [#table-format]

<DNT>Lens</DNT>でクエリを記述する場合は、次の形式を使用してテーブルを参照します。

```
connection_name.schema.table
```

例えば：

* `telemetry.Transaction` - テレメトリー (NRDB) からのテーブルを書きます <DNT>connector</DNT>
* `snowflake.sales.orders` - 「sales」スキーマ内の「snowflake」という名前の<DNT>Snowflake</DNT> <DNT>connector</DNT>から注文テーブルをクエリします
* `gsheets.default.employees` - 「gsheets」という名前の<DNT>Google Sheets</DNT> <DNT>connector</DNT>をクエリします

### サポートされているSQL文 [#supported-sql]

<DNT>Lens</DNT> 以下を含む標準 ANSI SQL ステートメントをサポートします。

* `SELECT` - 1つまたは複数のテーブルからデータをクエリする
* `JOIN` - 複数のテーブルからのデータを結合する（データベース間の結合を含む）
* `WHERE` - 結果をフィルタリング
* `GROUP BY` - 集計データ
* `ORDER BY` - 結果を並べ替える
* `LIMIT` - 返される行数を制限する
* `WITH` (CTE) - 複雑なクエリのための共通テーブル式を定義する

### パフォーマンスのヒント [#performance-tips]

* **フィルターを早期に使用する**: 処理されるデータの量を減らすために`WHERE`句を適用します。
* **結果を制限する**: データを探索するときに返される行を制限するには、 `LIMIT`を使用します。
* **<DNT>materialized views</DNT>を検討してください**: 頻繁に実行される複雑なクエリの場合は、 <DNT>materialized view</DNT>を作成します。
* **結合を最適化**: 大きなテーブルを結合する場合は、可能な場合は結合前にデータをフィルターします。

## NRQLとのクエリ構文の違いと例 [#syntax-differences]

<DNT>Lens</DNT> NRQL の代わりに ANSI SQL を使用します。次の例は、主な違いと一般的なクエリ パターンを示しています。

<CollapserGroup>
  <Collapser id="groupby" title="`FACET` の代わりに `GROUP BY` を使用する">
    結果をグループ化する場合は、 `GROUP BY`を使用し、グループ化された列を`SELECT`ステートメントに含めます。

    **NRQL。**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    FACET enduser.id
    ```

    **SQL:**

    ```sql
    SELECT "enduser.id", count(*)
    FROM telemetry.PageViewTiming
    GROUP BY "enduser.id"
    ```
  </Collapser>

  <Collapser id="where-timestamp" title="`SINCE` の代わりに `WHERE` を `タイムスタンプ` とともに使用します">
    `timestamp`列と日付間隔で`WHERE`を使用して時間範囲をフィルターします。

    **NRQL。**

    ```sql
    FROM PageViewTiming
    SELECT count(*)
    SINCE 1 day ago
    ```

    **SQL:**

    ```sql
    SELECT count(*)
    FROM telemetry.PageViewTiming
    WHERE timestamp > current_date - INTERVAL '1' DAY
    ```
  </Collapser>

  <Collapser id="timeseries" title="エポック形式の時系列">
    時系列の視覚化を作成するには、最初の列にエポック形式の日付が含まれるようにクエリを構成します。日付を変換するには`to_unixtime()`を使用します。

    ```sql
    WITH orders AS (
      SELECT
        to_unixtime(date(date_parse(order_date, '%Y-%m-%d'))) as dt,
        count(*) as orders
      FROM gsheets.default.orders
      GROUP BY 1
    )
    SELECT *
    FROM orders
    WHERE dt >= to_unixtime(date('2025-10-19'))
      AND dt <= to_unixtime(date('2025-10-21'))
    ORDER BY dt ASC
    ```
  </Collapser>

  <Collapser id="cross-db-join" title="データベース間の `JOIN`">
    標準 SQL `JOIN`構文を使用して、複数のソースからのデータを 1 つのクエリで結合します。

    ```sql
    WITH sf AS (
      SELECT DISTINCT user_id, assigned_success_manager
      FROM demogorgon.default.relibank_opportunity_matrix
    )
    SELECT
      sf.assigned_success_manager,
      MAX(nrdb.largestContentfulPaint)
    FROM telemetry.PageViewTiming nrdb
    LEFT JOIN sf ON nrdb."enduser.id" = sf.user_id
    WHERE timingName = 'largestContentfulPaint'
      AND nrdb.timestamp > current_date - INTERVAL '1' DAY
    GROUP BY sf.assigned_success_manager
    ```
  </Collapser>

  <Collapser id="query-mv" title="マテリアライズドビューのクエリ">
    `mv.`プレフィックスを使用して<DNT>materialized views</DNT>を参照します:

    ```sql
    SELECT *
    FROM mv.weekly_performance_summary
    WHERE week_number = 42
    ```

    <DNT>materialized views</DNT>他のデータ ソースと結合できます:

    ```sql
    SELECT
      mv.store_id,
      mv.total_sales,
      s.store_name,
      s.region
    FROM mv.store_sales_mv mv
    JOIN snowflake.retail.stores s ON mv.store_id = s.store_id
    WHERE mv.total_sales > 10000
    ```
  </Collapser>
</CollapserGroup>
