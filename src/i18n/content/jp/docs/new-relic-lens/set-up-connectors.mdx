## <DNT>Data connectors</DNT> [#data-connectors]

<DNT>Data connectors</DNT> <DNT>Lens</DNT>データベース、データ ウェアハウス、スプレッドシートなどの外部データ ソースに接続します。<DNT>connectors</DNT>を作成、編集、または削除するには、[特定の RBAC 権限が](/docs/new-relic-lens/overview#permissions)必要です。

### アクセス制御 [#access-control]

<DNT>Lens</DNT> <DNT>connectors</DNT>をクエリできるユーザーを制御するには、次の 2 つの方法があります。

* **RBAC 権限**: <DNT>**Lens connectors**</DNT> RBAC 権限を持つユーザーは、組織内のすべての<DNT>connectors</DNT>を書き込むことができます。
* **きめ細かなアクセス制御**: 広範な RBAC 権限を持たないユーザーでも、 <DNT>connectors</DNT>へのきめ細かなアクセスが許可されていれば、特定の<DNT>connectors</DNT>をクエリできます。

### 作成する <DNT>connector</DNT> [#create-connector]

<DNT>data connector</DNT>を設定するには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Connectors**</DNT>に移動します。

2. <DNT>**Create a connector**</DNT>をクリックします。

3. ドロップダウン メニューから<DNT>connector</DNT>タイプを選択します。

4. 選択した<DNT>connector</DNT>タイプに必要な接続情報を入力します。

5. RBAC 権限を持たないユーザーまたはグループにこのコネクタへのアクセスを許可するには、 <DNT>**Access control**</DNT>セクションからきめ細かなアクセス制御を構成します。

   1. ユーザーまたはグループを含む<DNT>**Auth domain**</DNT>を選択します。
   2. <DNT>**User or group**</DNT>を選択します。
   3. <DNT>**Lens Viewer**</DNT>ロールを選択します。
   4. さらにユーザーまたはグループを追加するには、 <DNT>**Add**</DNT>をクリックして上記の手順を繰り返します。

6. <DNT>**Create**</DNT>をクリックします。

<DNT>connector</DNT>を作成した後:

* スキーマを表示または<DNT>connector</DNT>を削除するには、 <Icon name="fe-more-horizontal" /><DNT>connector</DNT>行からのメニュー。
* アクセス制御設定を更新するには、 <DNT>connector</DNT>を選択し、アクセス許可を追加または削除して、変更を保存します。
* <DNT>connector</DNT>設定を更新するには、 <DNT>connector</DNT>を削除し、新しい詳細で再作成します。

    ### サービスアカウント認証情報キーを生成する [#generate-credentials-key]

    <DNT>Google Sheets</DNT>を使用して<DNT>Lens</DNT>認証するには、 <DNT>Google Cloud</DNT>サービス アカウントを作成し、JSON キーを生成する必要があります。

    <Steps>
      <Step>
        ### サービスアカウントを作成する [#create-service-account]

        <DNT>Google Cloud</DNT>サービス アカウントを作成するには、 [「サービス アカウントの作成」](https://cloud.google.com/iam/docs/service-accounts-create)の手順に従います。
      </Step>

      <Step>
        ### キー作成の権限を設定する [#set-up-permissions]

        サービス アカウントのキーを作成するには、適切な権限が必要です。必要な権限の詳細については、 [「必要な権限」](https://cloud.google.com/iam/docs/keys-create-delete#required-permissions)を参照してください。
      </Step>

      <Step>
        ### JSONキーを作成する [#create-json-key]

        サービス アカウントの JSON キーを生成するには、 [「サービス アカウント キーを作成する」](https://cloud.google.com/iam/docs/keys-create-delete#creating)の手順に従います。
      </Step>

      <Step>
        ### JSONキーをbase64に変換する [#convert-json-key-to-base64]

        <DNT>Lens</DNT>で JSON キーを使用するには、JSON ファイルの内容全体を base64 エンコードに変換し、テキスト ファイルに保存します。
      </Step>
    </Steps>

    ### Googleスプレッドシートへのアクセスを設定する [#setup-gsheets-access]

    認証情報キーを生成したら、Google スプレッドシートへのアクセスを構成します。

    <Steps>
      <Step>
        ### サービスアカウントとシートを共有する [#share-sheets-with-service-account]

        <DNT>Lens</DNT>データにアクセスできるようにするには、接続する各 Google スプレッドシートのサービス アカウントのメール アドレスに読み取り権限を付与します。
      </Step>

      <Step>
        ### メタデータシートを作成する [#create-metadata-sheet]

        <DNT>Lens</DNT>で利用可能なシートを管理するには、次の 4 つの列を含む新しい Google スプレッドシートを作成します。

        * **Sheetname** : <DNT>Lens</DNT>を使用してアクセスする各シートの名前。リストされているすべてのシートは、サービス アカウントからアクセスできる必要があります。
        * **Sheetid** : 各シートの一意の識別子。これはシート URL: `https://docs.google.com/spreadsheets/d/SHEET_ID/edit`で確認できます。
        * **所有者**: シートの所有者。デフォルト値として`Trino`を使用します。
        * **注記**: シートに関する追加の注記または説明。
      </Step>

      <Step>
        ### メタデータシートを共有する [#share-metadata-sheet]

        <DNT>Lens</DNT>メタデータを読み取れるようにするには、メタデータ シートの読み取り権限をサービス アカウントに付与します。
      </Step>

      <Step>
        ### メタデータシートIDをコピーする [#copy-metadata-sheet-id]

        <DNT>connector</DNT>を構成するには、メタデータ シート URL からシート ID をコピーします。シート ID は URL 内で次のように表示されます: `https://docs.google.com/spreadsheets/d/SHEET_ID/edit` 。
      </Step>

    ### 付与 <DNT>Lens</DNT> [#grant-iceberg-access]

    <DNT>Lens</DNT> Iceberg テーブルをクエリできるようにするには、Glue データカタログと S3 バケットへの読み取りアクセスを許可する IAM ポリシーとロールを AWS アカウントに作成します。

    <Callout variant="important">
      Glue データカタログと Iceberg データを含む S3 バケットは、同じ AWS リージョンに存在する必要があります。
    </Callout>

    <Steps>
      <Step>
        ### IAMポリシーを作成する [#create-iceberg-policy]

        次の権限を持つ[IAM ポリシーを作成し、](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html) <DNT>Lens</DNT> Glue メタデータと S3 内のデータファイルを読み取ることを許可します。`{REGION}` 、 `{ACCOUNT_ID}` 、 `{DATABASE_NAME}` 、 `{S3_BUCKET_NAME}`を実際の値に置き換えます。入力したポリシー名をメモしてください。このポリシーを IAM ロールにアタッチするときに必要になります。

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Action": [
                "glue:GetDatabase",
                "glue:GetTable",
                "glue:GetTables",
                "glue:SearchTables",
                "glue:GetPartition",
                "glue:GetPartitions",
                "glue:BatchGetPartition"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:catalog",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:database/{DATABASE_NAME}",
                "arn:aws:glue:{REGION}:{ACCOUNT_ID}:table/*"
              ]
            },
            {
              "Action": [
                "s3:GetObject",
                "s3:ListBucket",
                "s3:GetBucketLocation"
              ],
              "Effect": "Allow",
              "Resource": [
                "arn:aws:s3:::{S3_BUCKET_NAME}",
                "arn:aws:s3:::{S3_BUCKET_NAME}/*"
              ]
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### 信頼関係を持つIAMロールを作成する [#create-iceberg-role]

        [IAM ロールを作成し](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html)、信頼関係を設定して、 <DNT>Lens</DNT>サービス アカウントが一意の外部 ID を使用してそのロールを引き継ぐことができるようにします。ロールを設定するときは、次の値を使用します。

        * **信頼されたエンティティタイプ**: AWS アカウント
        * **アカウント ID** : `017663287629` (New Relic の AWS アカウント)
        * **外部 ID が必要**: 有効。任意の一意の文字列を入力して、 <DNT>Lens</DNT> <DNT>connector</DNT>設定用に保存します。
        * **権限ポリシー**: 前の手順で作成したポリシー名を検索して選択します。

        信頼ポリシーは次の形式と一致する必要があります。`<YOUR_EXTERNAL_ID>` 、入力した外部 ID に置き換えます。

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        ロールを作成したら、ロールの概要ページを開き、**Role ARN** \[ロール ARN]をコピーします。<DNT>Lens</DNT> <DNT>connector</DNT>を構成するときに、この ARN と外部 ID が必要になります。
      </Step>
    </Steps>
  </Collapser>

    ### 付与 <DNT>Lens</DNT> [#grant-cloudwatch-access]

    <DNT>Lens</DNT>が CloudWatch メトリクスに書き込めるようにするには、CloudWatch への読み取りアクセスを許可する IAM ポリシーとロールをAWSアカウントに作成します。

    <Steps>
      <Step>
        ### IAMポリシーを作成する [#create-cloudwatch-policy]

        次の権限を持つ[IAM ポリシーを作成して](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html)、 <DNT>Lens</DNT> AWSアカウントから CloudWatchメトリクスを読み取ることを許可します。 入力したポリシー名をメモしてください。このポリシーを IAM ロールにアタッチするときに必要になります。

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Action": [
                "cloudwatch:ListMetrics",
                "cloudwatch:GetMetricData",
                "cloudwatch:GetMetricStatistics",
                "tag:GetResources"
              ],
              "Resource": "*"
            }
          ]
        }
        ```
      </Step>

      <Step>
        ### 信頼関係を持つIAMロールを作成する [#create-cloudwatch-role]

        [IAM ロールを作成し](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_create.html)、信頼関係を設定して、 <DNT>Lens</DNT>サービス アカウントが一意の外部 ID を使用してそのロールを引き継ぐことができるようにします。ロールを設定するときは、次の値を使用します。

        * **信頼されたエンティティタイプ**: AWS アカウント
        * **アカウント ID** : `017663287629` (New Relic の AWS アカウント)
        * **外部 ID が必要**: 有効。任意の一意の文字列を入力して、 <DNT>Lens</DNT> <DNT>connector</DNT>設定用に保存します。
        * **権限ポリシー**: 前の手順で作成したポリシー名を検索して選択します。

        信頼ポリシーは次の形式と一致する必要があります。`<YOUR_EXTERNAL_ID>` 、入力した外部 ID に置き換えます。

        ```json
        {
          "Version": "2012-10-17",
          "Statement": [
            {
              "Effect": "Allow",
              "Principal": {
                "AWS": "arn:aws:iam::017663287629:user/service_fdp_customer_account_data_access"
              },
              "Action": "sts:AssumeRole",
              "Condition": {
                "StringEquals": {
                  "sts:ExternalId": "<YOUR_EXTERNAL_ID>"
                }
              }
            }
          ]
        }
        ```

        ロールを作成したら、ロールの概要ページを開き、**Role ARN** \[ロール ARN]をコピーします。<DNT>Lens</DNT> <DNT>connector</DNT>を構成するときに、この ARN と外部 ID が必要になります。
      </Step>
    </Steps>
  </Collapser>
