より大規模で計算コストの高いクエリの場合は、結果を<DNT>Lens</DNT>に<DNT>materialized views</DNT>として保存できます。<DNT>materialized view</DNT>にはクエリの事前計算された結果が格納されるため、毎回複雑なクエリを実行する代わりに、格納されたデータに対してクエリを実行できます。

## のメリット <DNT>materialized views</DNT> [#benefits]

<DNT>Materialized views</DNT> お手伝いします:

* **クエリ コストの削減**: スキャンまたは転送されたデータに基づいてコストが発生する可能性がある、外部データ ソースから同じデータを繰り返し取得することを回避します。
* **パフォーマンスの向上**: 毎回複数のデータベース間で複雑な結合を実行するのではなく、事前に計算された結果をクエリします。
* **ダッシュボードの簡素化**: マルチデータベース書き込みの結果を保存し、ダッシュボード ウィジェットで簡単に参照できます。

## 前提条件 [#prerequisites]

始める前に:

* <DNT>materialized views</DNT>を作成および管理するために必要な[権限](/docs/new-relic-lens/overview#permissions)があることを確認してください。
* 外部データ ソースをクエリする必要がある場合は、 [<DNT>connectors</DNT>](/docs/new-relic-lens/set-up-connectors)が設定されていることを確認してください。

## 作成する <DNT>materialized view</DNT> [#create]

<DNT>materialized view</DNT>を作成するには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT>に移動します。
2. <DNT>**Create materialized view**</DNT>をクリックします。
3. <DNT>materialized view</DNT>の一意の名前を入力します。
4. NRDB データを書き込むには、アカウント セレクターからアカウントを選択します。 このアカウントの NRDB データを外部データ ソースと結合できます。
5. [テーブル参照形式](/docs/new-relic-lens/query-data#table-format)を使用して SQL クエリを記述します。 `connection_name.schema.table`
6. <DNT>**Create**</DNT>をクリックします。

<DNT>materialized view</DNT>がリストに表示されます。クエリの複雑さによっては、結果の保存に時間がかかる場合があります。

## リフレッシュ <DNT>materialized view</DNT> [#refresh]

<DNT>Materialized views</DNT> 作成時または最後に更新された時点のデータのスナップショットが含まれます。最新のデータを取得するには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT>に移動します。
2. 更新したい<DNT>materialized view</DNT>を見つけます。
3. クリック<Icon name="fe-more-horizontal" />メニュー。
4. <DNT>**Refresh**</DNT>を選択します。

ビューは元のクエリを再度実行し、保存された結果を最新のデータで更新します。

<Callout variant="important">
  <DNT>materialized views</DNT>を更新する頻度を決定するときは、データの鮮度要件を考慮してください。複雑なクエリでビューを頻繁に更新すると、コストが増加する可能性があります。
</Callout>

## クエリ <DNT>materialized view</DNT> [#query]

<DNT>Lens</DNT>内の他のテーブルと同様に、 <DNT>materialized view</DNT>をクエリできます。

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Query your data**</DNT>に移動します。

2. <DNT>**Notebooks**</DNT>の横にあるトグルを有効にし、 <DNT>**+ SQL**</DNT>をクリックして SQL ブロックを追加します。

3. ビュー名に続けて`mv.`プレフィックスを使用して<DNT>materialized view</DNT>を参照します。

   ```sql
   SELECT * FROM mv.your_materialized_view_name
   ```

4. クエリを実行します。

<DNT>materialized view</DNT>に追加のフィルター、結合、または集計を適用できます。

```sql
SELECT store_id, transaction_count
FROM mv.top_stores_mv
WHERE transaction_count > 1000
ORDER BY transaction_count DESC
```

<DNT>materialized view</DNT>は事前に計算された結果を含むテーブルとして機能し、元の複雑なクエリを再度実行せずにさらに分析を実行できます。

## 削除 <DNT>materialized view</DNT> [#delete]

不要になった<DNT>materialized view</DNT>を削除するには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Administration &gt; Materialized views**</DNT>に移動します。
2. 削除したい<DNT>materialized view</DNT>を見つけます。
3. クリック<Icon name="fe-more-horizontal" />メニュー。
4. <DNT>**Delete**</DNT>を選択します。
5. 削除を確認します。

<Callout variant="caution">
  <DNT>materialized view</DNT>の削除は永続的です。削除されたビューを参照するクエリやダッシュボードはすべて失敗します。
</Callout>

クロスデータベース結合を含む書き込みの例については、 [「書き込みマテリアライズド ビュー」](/docs/new-relic-lens/query-data/#query-mv)を参照してください。
