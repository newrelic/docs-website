アプリのパフォーマンスは、ビジネスが成功するかどうかを左右する最も重要な側面の 1 つです。[Google の調査によると](https://think.storage.googleapis.com/images/micromoments-guide-to-winning-shift-to-mobile-download.pdf)、モバイル ユーザーの 29% は、サイトやアプリケーションの速度が遅すぎたり、情報を見つけるのが難しすぎたりすると、そのサイトやアプリケーションの使用を避けます。また、そのユーザー グループの 70% は、サイトが遅すぎるためにそのようにしています。サイトやアプリケーションのパフォーマンスの問題を診断して修正することは、組織が既存のユーザーを維持しながら新しいユーザーを引き付けるのに役立ちます。New Relic はそのためのツールにすぎません。

## レイテンシーの問題の診断 [#diagnostic-steps]

まず、ほとんどの Web サイトやアプリケーションで発生する高レイテンシの原因を理解する必要があります。これらを知っておくと、システムの高レイテンシの原因を特定するのに役立ちます。高レイテンシの最も一般的なタイプは次のとおりです。

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="Slow page load"
    title="ページの読み込みが遅い"
  >
    ユーザーが Web サイトにアクセスしようとして、ページの読み込みに予想以上の時間がかかる場合、多くの場合、これがユーザーがさらされる遅延の原因となっています。Google の主要な Web Vitals の 1 つである **Largest contentful paint** (LCP) を使用すると、これがレイテンシの原因であると診断できます。LCP は、の **Summary** ページにリストされています。 <InlinePopover type="browser"/>また、ブラウザー モニターの **Session trace**\[セッション トレース] ページを使用して、特定のページの遅延の具体的な原因を特定することもできます。
  </Collapser>

  <Collapser
    className="freq-link"

    title="ページの非対話性"
  >
    検索機能や機能の使用など、ユーザーがページ要素を操作できるようになるまでに予想よりも長く待たなければならない場合、これがユーザーが経験する待ち時間の原因です。Google の主要な Web Vitals の 1 つである **First input delay** (FID) を使用できます。FID は、の **Summary** ページに記載されています。 <InlinePopover type="browser"/>、およびブラウザー モニターの **Session trace** ページは、ページの遅延の具体的な原因を特定するのに役立ちます。
  </Collapser>

  <Collapser
    className="freq-link"
    id="Slow ajax"
    title="遅い AJAX リクエスト"
  >
    ユーザーがサイトまたはアプリケーションの全体的なパフォーマンスが低いと感じている場合、これが多くの場合、待ち時間が長くなる原因となっています。これは、アプリケーションの時間が遅い、JavaScript の実行時間が長いなど、さまざまな要因が原因である可能性がありますが、New Relic は、これがサービスで発生している問題であるかどうかを確認するのに役立ちます。AJAX モニタリングは、次の **Ajax** ページにあります。 <InlinePopover type="browser"/>.詳細については、 [AJAX ドキュメント を](/docs/browser/browser-monitoring/browser-pro-features/ajax-page-identify-time-consuming-calls/#ajax-dashboard)参照してください。
  </Collapser>
</CollapserGroup>

レイテンシーの一般的な原因と、その原因を見つけるにはどこに行けばよいかがわかったら、 **Summary**ページ と **Ajax** ページを使用できます <InlinePopover type="browser"/>高いレイテンシの原因を見つけます。サービスが全体的にどのように機能するかを必ずしも知る必要はありません。動作の最近の主要な変更を探すだけで済みます。

## 高レイテンシの診断例 [#latency-example]

待ち時間の一般的な原因を理解したら、サイトに影響を与えている特定の問題を絞り込むことができます。遅延に問題があるサービスを見てみましょう。

<img
  alt="A screenshot of the summary page with a poor LCP score"
  title="Abnormal LCP score"
  src={journeyAbnormalLCP}
/>

ここでは、8.44 秒の **Largest Contentful Paint** を示すサービスの **Summary** ページが表示されます。他のすべてが許容可能なパラメーターに該当するため、遅延の原因をページの読み込みの遅さとして絞り込むことができます。LCP で問題を引き起こしている可能性のある特定のビューについては、 **Session trace**\[セッション トレース] ページを選択できます。これは、遅延の問題の原因を見つけるための検索を開始するのに適した場所です。

また、 **Largest Contentful Paint** セクション自体を選択して **Web Vitals** ページに移動し、ページの URL、デバイスの種類、ブラウザーの種類などに基づいて、より一般的なエラーの概要を確認することもできます。この方法を使用して問題の原因を見つけたら、問題を修復して、ユーザー エクスペリエンスを顧客を満足させるレベルに戻すことができます。
