JavaScript エラーは、顧客のサイトの使用方法やブランドに対する認識に影響を与えます。サイトのパフォーマンスを維持し、顧客を満足させるために、JavaScript エラーが発生したらすぐに修正することを優先する必要があります。

## 目的 [#objectives]

このチュートリアルでは、JavaScript エラーを見つけて解決する手順を説明します。チュートリアルが終わるまでに、次のタスクが完了します。

* New Relic での JavaScript エラー情報の検索
* エラーに優先順位を付けるときに推奨事項を適用する
* イベント証跡を使用して JavaScript エラーを解決する

## JavaScript エラーを見つける [#find-errors]

ブラウザ監視の **Summary** \[概要] ページでは、JavaScript エラーがあることを 2 つのグラフ **Error Rate** \[エラー率] と **Top 10 error messages** \[上位 10 のエラー メッセージ]で確認できます。いずれかのグラフを選択すると、 **JS errors** \[JS エラー] ページが表示され、JavaScript エラーの詳細が表示されます。

* 顧客または内部ユーザーが JavaScript エラーの可能性があることを報告した場合は、 **Summary** \[概要] ページまたは **JS errors** \[JS エラー] ページから検索を開始することをお勧めします。
* サイトのエラーを事前に監視したい場合は、ほとんどの場合、 **Summary** \[概要] ページを確認するだけで十分です。

## どのエラーが重要かを判断する [#determine-errors]

JavaScript エラーを発見したら、修正に優先順位を付ける必要があります。エラーの頻度、重大度、影響を受ける領域、その他の要因に基づいてエラーに優先順位を付けることができます。例を使用して、JavaScript エラーに優先順位を付ける方法を説明します。

上記の **Top 10 error messages** \[トップ 10 エラー メッセージの] グラフを使用すると、1 つのエラーに 6,460 件のインスタンスがあることがわかります。このエラーの頻度により、最初に解決すべき最有力候補となる可能性がありますが、常にそうとは限りません。修正する必要がある順序を決定するには、すべてのエラーをより広範囲に把握する必要がある場合があります。 **JS errors** \[JS エラー] ページから、エラーを `errorMessage`、 `pageURL`、または `deviceType`ごとにグループ化できます。作業を計画するときは、次の質問を自問してください。

* これはビジネスに不可欠なユーザー ジャーニーに影響しますか?エラーが顧客のサインアップ、ログイン、または購入の機能に影響を及ぼさない場合は、問題の優先順位を下げることができる可能性があります。
* このエラーは最近発生しましたか?新しいエラーに焦点を当てることで、後で大きな問題に発展するのを防ぐことができます。
* このエラーにより遅延が長くなりますか?

## エラーを素早く修正する [#fix-errors]

<img
  title="JS error overview"
  alt="A screenshot of a single error overview page"
  src={browserSingleErrorOverview}
/>

エラーを選択したら、 **JS errors** \[JS エラー] ページからエラーを選択します。ここから、さまざまなビューを開いて特定のエラーを調査できます。

* **Error instances** \[エラー インスタンス] により、 **Event Trail** \[イベント トレイル ページ] と **Stack trace** \[スタック トレース] ページが表示されます。JavaScript ファイルがエラーの原因かどうか、インライン コードの一部を修正する必要があるかどうかなど、特定のエラーに関する詳細を確認できます。
* **Event Trail** \[イベント トレイル] セクションをスクロールして、修正する必要があるものがすべて見つかったことを確認します。さらに解決すべき点がある場合は、イベント トレイルで通知されます。
