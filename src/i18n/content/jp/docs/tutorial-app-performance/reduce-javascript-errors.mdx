JavaScript エラーは、顧客がサイトをどのように使用し、ブランドを認識するかに重大な悪影響を与える可能性があります。サイトのパフォーマンスを維持し、顧客を満足させ続けるには、JavaScript エラーが発生したらすぐに見つけて修正する必要があります。New Relic を使用すると、エラーの数やサイズに関係なく、これらのエラーを簡単に特定できるため、サイトをより迅速かつ効率的に立ち上げることができます。

の中に <InlinePopover type="browser"/>インターフェイスで、インストールされているモニターを選択して **Summary** ページを開きます。**Error Rate** と **Top 10 error messages**の 2 つのグラフが表示されます。 これらのグラフは両方とも、JavaScript エラーがあることを示している可能性があり、いずれかのグラフを選択すると、JavaScript エラー固有の情報の詳細を表示する **JS errors** ページが表示されます。顧客または内部ユーザーが JavaScript エラーである可能性があると報告した場合、 **Summary** ページまたは **JS errors** ページのいずれかが検索を開始するのに適した場所です。サイトのエラーを監視するだけの場合は、ほとんどの場合、 **Summary** ページを確認するだけで十分です.

## どのエラーが重要かを判断する [#determine-errors]

サイトに存在する JavaScript エラーを見つけたら、最初に修正する必要があるエラーを特定する必要があります。これらのエラーの重大度は複雑で、頻度、重大度、影響を受ける地域、その他のさまざまな要因など、さまざまな要因に基づいています。

上記の **Top 10 error messages** グラフを使用すると、1 つのエラーに 6460 のインスタンスがあることがわかります。これは、最初に修正する必要があるエラーの最有力候補となります。 最も頻繁に発生するエラーが、最初に修正する必要があるエラーではない場合があります。 エラーを修正する必要がある順序を決定するには、すべてのエラーをより広く見る必要がある場合があります。 `errorMessage`、 `pageURL`、 `deviceType`などのさまざまな要因でエラーをグループ化できる **JS errors** ページでは、その情報が提供されます。 エラーの優先度を選択する際に考慮すべきその他の事項:

* 該当する場合、このエラーは Web アプリケーションに影響しますか?もしそうなら、他のものよりも改善することで、ユーザーのエクスペリエンスとエンゲージメントの両方が向上する可能性があります.
* このエラーは最近発生しましたか?最近発生したエラーに焦点を当てることで、より多くの顧客に影響を与える大きな問題になるのを防ぐことができる場合があります。
* このエラーはユーザー エクスペリエンスに影響を与えますか?エラーがユーザー エクスペリエンスに悪影響を及ぼさない場合、そのようなエラーを修正する価値はないかもしれません。

## エラーをすばやく修正する [#fix-errors]

<img
  title="JS error overview"
  alt="A screenshot of a single error overview page"
  src={browserSingleErrorOverview}
/>

修正したいエラーが見つかったら、 **JS errors** ページでそれを選択して、そのエラーに関する特定の情報を取得できます。 **Errors instances** を選択すると、 **Event Log** と **Stack trace**が表示されます。 エラーの原因となった JavaScript ファイルやインライン コード内の修正が必要な部分など、問題を解決するために何を修正する必要があるかを見つけるのに役立つデータが見つかります。 **Event Log** セクションの上にある矢印を使用して、エラーの各インスタンスをスクロールし、エラーが完全に解決されるまで、修正が必要なすべてのものを見つけたことを確認します。 **JS errors page** を使用して、サイトの JS エラーを低く抑え、顧客満足度を高く保ちましょう。
