スコアカード テンプレート ライブラリは、エンジニアリングおよびビジネスのユースケースに合理的に対処する方法を提供します。ライブラリは、大企業のベストプラクティスを活用して、一般的なエンジニアリングの課題に対するソリューションを提供します。

ライブラリ内の各テンプレートは、組織の標準に合わせて完全にカスタマイズできます。スコアカードをニーズに合わせて調整するために、書き込み、ルール、および閾値を変更できます。

## スコアカードテンプレート

<CollapserGroup>
  <Collapser id="product-readiness" title="生産準備">
    **目的:**定義された目標、集中設定、および最近のデプロイメント アクティビティをチェックすることにより、サービスが実稼働デプロイメントの準備ができているかどうかを検証します。

    **重要:**デプロイメント中の最小限のダウンタイムと高いサービス品質を保証します。

    **前提条件:**

    * [New Relic変更追跡 (変更追跡機能)](/docs/change-tracking/change-tracking-introduction)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)

    **主なルール:**

    * **定義された SLO:**パフォーマンスと信頼性に対する明確な目標を確立します。
    * **APMサービス アラート設定:**プロンプトの問題検出用にアラートが設定されていることを確認します。
    * **最近のデプロイメント:**過去 1 か月以内に成功したデプロイメントを確認します。
  </Collapser>

  <Collapser id="ownership-documentation" title="所有権と文書">
    **目的:**チームの所有権、タグ付け、コード リポジトリ、ドキュメントの可視性を評価します。

    **前提条件:**

    * [New Relicチーム](/docs/service-architecture-intelligence/teams/teams)
    * [GitHub 統合](/docs/service-architecture-intelligence/github-integration)(該当する場合)

    **主なルール:**

    * **チームの所有権:**所有権タグを通じてサービスを担当チームにリンクします。
    * **定義されたコード リポジトリ:**関連するコード リポジトリを確認します。
  </Collapser>

  <Collapser id="operational-readiness" title="運用準備">
    **目的:**運用手順のランブックが利用可能かどうかを確認することで、システムがアラート状態に対処できるように準備されていることを確認します。

    **重要性:**透明性を促進し、インシデントの解決に役立ちます。

    **前提条件:** [New Relicアラート](/docs/alerts/overview)

    **主なルール:**

    * **チームの所有権:**所有権タグを通じてサービスを担当チームにリンクします。
    * **定義されたコード リポジトリ:**関連するコード リポジトリを確認します。
  </Collapser>

  <Collapser id="web-vitals" title="ウェブバイタル">
    **目的:**読み込み速度、インタラクティブ性、視覚的な安定性など、ユーザー エクスペリエンスに影響を与える主要なパフォーマンス メトリクスを評価します。

    **重要:**読み込み時間の短縮、応答性の高いインタラクション、安定したレイアウトを確保することで、ユーザー エクスペリエンスを最適化します。

    **前提条件:** [New Relicブラウザ監視](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)

    **主なルール:**

    * **Largest Contentful Paint (LCP):**メインコンテンツの読み込み速度を測定します。
    * **初回入力遅延 (FID):**ユーザーのインタラクションに対する応答性を評価します。
    * **累積レイアウトシフト (CLS):**レイアウトシフトを最小限に抑えます。
  </Collapser>

  <Collapser id="aws-reliability" title="AWSの信頼性">
    **目的:** AWS標準に対する信頼性を評価し、 cloud運用のセキュリティと冗長性を確保します。

    **重要:**クラウド インフラストラクチャを保護し、運用の継続性を確保します。

    **前提条件:** [New Relic AWS監視](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations)

    **主なルール:**

    * **EFS 暗号化チェック:** Elastic File System (EFS) ボリュームの暗号化を確認します。
    * **RDS インスタンスのバックアップ保持期間: リレーショナル データベース**サービス (RDS) のバックアップ保持期間をチェックします。
    * **RDS 削除保護:**削除保護を検証します。
    * **RDS MultiAZ の存在:**複数のアベイラビリティ ゾーン (MultiAZ) にわたるプロイメントを確保します。
    * **RDS 自動マイナー バージョン アップグレード チェック:**自動マイナー バージョン アップグレードを確認します。
  </Collapser>

  <Collapser id="dora-metrics" title="DevOps 研究および評価 (DORA) メトリクス">
    **目的:**主要パフォーマンス指標を使用して、リソースの割り当てとプロセスの改善を評価します。

    **重要度:**ソフトウェア配信のパフォーマンスを定量化し、戦略的な意思決定を支援します。

    **前提条件:**

    * [変更追跡](/docs/change-tracking/change-tracking-introduction)
    * [New Relicアラート](/docs/alerts/overview)

    **主なルール:**

    * **デプロイメント頻度:**成功したデプロイメント率を追跡します。
    * **平均回復時間:**障害からの回復時間を測定します。
  </Collapser>
</CollapserGroup>
