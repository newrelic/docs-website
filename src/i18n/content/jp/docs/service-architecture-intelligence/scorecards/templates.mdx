<DNT>Scorecards</DNT>テンプレートライブラリは、エンジニアリングおよびビジネスのユースケースに対処するための合理化された方法を提供します。大企業のベストプラクティスを使用することで、ライブラリは一般的なエンジニアリングの課題に対するソリューションを提供します。

ライブラリ内の各テンプレートは、組織の基準を満たすように完全にカスタマイズ可能です。クエリ、ルール、および閾値を変更して、ニーズに合わせて<DNT>Scorecards</DNT>を調整できます。

一部のテンプレートでは、ルールに成熟度レベル（`L0`から`L3`）のラベルを付けています。

## スコアカードテンプレート

<CollapserGroup>
  <Collapser
    id="product-readiness"
    title={<DNT>本番環境の準備状況</DNT>
    }
  >
    **目的:**定義された目標、集中設定、および最近のデプロイメント アクティビティをチェックすることにより、サービスが実稼働デプロイメントの準備ができているかどうかを検証します。

    **重要:** 本番環境にデプロイする前に、準備のギャップを見つけるのに役立ちます。

    **前提条件:**

    * [サービスレベル管理](/docs/service-level-management/intro-slm)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [New Relicアラート](/docs/alerts/overview)
    * [変更追跡](/docs/change-tracking/overview)

    **主なルール:**

    * <DNT>**Defined SLOs**</DNT>: パフォーマンスと信頼性の明確なターゲットを確立します。
    * <DNT>**APM service alerts configuration**</DNT>: プロンプトの問題検出のためにアラートが設定されていることを確認します。
    * <DNT>**Recent deployments**</DNT>: 過去1か月以内に成功したデプロイメントを検証します。
    * <DNT>**Runbook availability**</DNT>: 過去1か月にトリガーされたアラート条件のランブックが存在することを確認します。
  </Collapser>

  <Collapser
    id="aws-reliability"
    title={<DNT>AWSの信頼性</DNT>
    }
  >
    **目的：**cloud運用におけるセキュリティと冗長性に関するAWS基準に対する信頼性を評価します。

    **重要度:** クラウドインフラストラクチャの信頼性とセキュリティのギャップを見つけるのに役立ちます。

    **前提条件:** [New Relic AWS監視](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations)

    **主なルール:**

    * <DNT>**EFS Encrypted check**</DNT>: Elastic File System（EFS）ボリュームの暗号化を確認します。
    * <DNT>**RDS Instance check retention period**</DNT>：リレーショナルデータベースサービス（RDS）のバックアップ保持期間を確認します。
    * <DNT>**RDS DeletionProtection check**</DNT>: 削除保護を検証します。
    * <DNT>**RDS MultiAZ presence check**</DNT>: 複数のアベイラビリティゾーン（MultiAZ）にわたるデプロイメントを確認します。
    * <DNT>**RDS AutoMinorVersionUpgrade check**</DNT>: マイナーバージョンの自動アップグレードを確認します。
  </Collapser>

  <Collapser
    id="dora-metrics"
    title={<DNT>DevOps 研究および評価 (DORA) メトリクス</DNT>
    }
  >
    **目的:**主要パフォーマンス指標を使用して、リソースの割り当てとプロセスの改善を評価します。

    **重要度:**ソフトウェア配信のパフォーマンスを定量化し、戦略的な意思決定を支援します。

    **前提条件:**

    * [変更追跡](/docs/change-tracking/overview)
    * [New Relicアラート](/docs/alerts/overview)
    * [GitHub の統合](/docs/service-architecture-intelligence/github-integrations/github-cloud-integration)

    **主なルール:**

    * <DNT>**Deployment frequency**</DNT>: 成功したデプロイメント率を追跡します。
    * <DNT>**Mean time to recovery**</DNT>: 障害からの回復時間を測定します。
    * <DNT>**Change failure rate**</DNT>: 障害につながる変更の割合を追跡します。
    * <DNT>**Lead time for changes**</DNT>: コードのコミットからデプロイメントまでにかかる時間を測定します。
  </Collapser>

  <Collapser
    id="tag-compliance"
    title={<DNT>タグコンプライアンス</DNT>
    }
  >
    **目的:** 組織内のリソースに対して一貫したタグ付けのプラクティスを確立し、適用します。

    **重要:** 一貫したタグは、マップやカタログなどのビュー全体でサービスをフィルタリングおよびグループ化するのに役立ちます。

    **前提条件：** [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)

    **主なルール:**

    * <DNT>**Team Tag Exists**</DNT>：明確な所有権と説明責任のために、サービスに所有チームのタグが付与されていることを確認します。
    * <DNT>**Environment Tag Exists**</DNT>: フィルタリングとグループ化のために、サービスに一貫した環境タグが付与されていることを確認します。
    * <DNT>**APM Criticality is set**</DNT>: サービスにビジネス上の重要度によって優先順位を付けるための重要度タグがあることを確認します。
  </Collapser>

  <Collapser
    id="observability-maturity-intelligent-observability"
    title={<DNT>オブザーバビリティの成熟度：インテリジェントなオブザーバビリティ</DNT>
    }
  >
    **目的：** 包括的なタグ戦略を確立し、サービスの依存関係と関係性を完全に可視化します。

    **重要度:** タグ、関係性、トレース、および外形監視カバレッジのギャップを強調表示し、それらに対処できるようにします。

    **前提条件:**

    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [合成モニタリング](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring)

    **主なルール:**

    * <DNT>**(L0) Team Tag Coverage**</DNT>: `team`タグが欠落しているエンティティを特定します。
    * <DNT>**(L0) Environment Tag Coverage**</DNT>: `environment`タグが欠落しているエンティティを識別します。
    * <DNT>**(L0) Uninstrumented Entities**</DNT>: 追加の計装を必要とする、関係が不明なHTTPサービスおよびデータベースエンティティにフラグを付けます。
    * <DNT>**(L0) Distributed Trace Coverage**</DNT>: ディストリビューティッド（分散）トレーシングデータを出力するAPMエンティティの割合を測定します。
    * <DNT>**(L0) Synthetic Coverage**</DNT>：外形監視モニターでカバーされているAPMエンティティの割合を測定します。
  </Collapser>

  <Collapser
    id="observability-maturity-business-uptime"
    title={<DNT>オブザーバビリティ成熟度: ビジネスの稼働時間</DNT>
    }
  >
    **目的：** サービスの信頼性のために、包括的なアラート戦略とインシデント管理アプローチを開発します。

    **重要:** 重要なサービスを優先できるように、集計のギャップ、ノイズ、SLIカバレッジを強調表示します。

    **前提条件:**

    * [New Relicアラート](/docs/alerts/overview)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [インフラストラクチャのモニタリング](/docs/infrastructure/introduction-infra-monitoring)
    * [サービスレベル管理](/docs/service-level-management/intro-slm)

    **主なルール:**

    * <DNT>**(L1) Infrastructure Alert Coverage**</DNT>：集計カバレッジが不足しているホストおよびKubernetesポッドを識別します。
    * <DNT>**(L1) Service Delivery Alert Coverage**</DNT>: 集計カバレッジが不足しているサービス、ブラウザ、モバイル、およびSyntheticモニターを特定します。
    * <DNT>**(L1) Critical Alert Coverage**</DNT>: アラート疲れを引き起こす可能性のある重大なアラート条件への過度の依存にフラグを立てます。
    * <DNT>**(L1) Alert Noise**</DNT>: 大量のインシデントを生成するポリシーを特定します。
    * <DNT>**(L2) Service Level Coverage**</DNT>: エンティティに定義されたサービスレベル指標（SLI）があるかどうかを確認します。
    * <DNT>**(L2) Alerts Mean Time To Close**</DNT>: インシデントをクローズするのにかかる時間を評価し、30分以内の解決をターゲットとします。
    * <DNT>**(L2) APM Criticality Tag Coverage**</DNT>: `criticality`タグが欠落しているエンティティを特定します。
    * <DNT>**(L3) Service Level Attainment**</DNT>：デフォルトの閾値95％を使用して、定義された各SLIのコンプライアンススコアを評価します。
  </Collapser>

  <Collapser
    id="observability-maturity-engineering-excellence"
    title={<DNT>オブザーバビリティ成熟度：エンジニアリング・エクセレンス</DNT>
    }
  >
    **目的:** CPUとメモリの使用量を監視し、Change Tracking (変更追跡機能)を統合し、脆弱性を検出することで、効率的なリソース使用と堅牢なセキュリティプラクティスを強調します。

    **重要性:** リソース、Change Tracking (変更追跡機能)、およびセキュリティの標準を遵守することで、効率的で安全なインフラストラクチャを運用できます。

    **前提条件:**

    * [インフラストラクチャのモニタリング](/docs/infrastructure/introduction-infra-monitoring)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [変更追跡](/docs/change-tracking/overview)
    * [Security RX](/docs/vulnerability-management/getting-started/overview)

    **主なルール:**

    * <DNT>**(L1) CPU Utilization**</DNT>: インフラストラクチャエンティティのCPU使用率の95番目のパーセンタイルを、効率性のベストプラクティスと照らし合わせてチェックします。
    * <DNT>**(L1) Memory Utilization**</DNT>: インフラストラクチャエンティティのメモリ使用量の95番目のパーセンタイルを効率性のベストプラクティスに照らして確認します。
    * <DNT>**(L2) Change Tracking**</DNT>: APMエンティティ全体でChange Tracking (変更追跡機能)イベントをチェックします。
    * <DNT>**(L3) Service Vulnerabilities**</DNT>: 検出された脆弱性を持つAPMエンティティの割合を確認します。
  </Collapser>

  <Collapser
    id="observability-maturity-digital-experience"
    title={<DNT>オブザーバビリティ成熟度：デジタルエクスペリエンス</DNT>
    }
  >
    **目的:** JavaScriptエラー、HTTP 5xxエラー、およびコアウェブバイタルのメトリクスを通じて、フロントエンドのパフォーマンスとユーザーインタラクションをモニターします。

    **重要:** フロントエンドエラーとCore Web Vitalsの標準を遵守することで、強力なユーザー体験を維持できます。

    **前提条件:**

    * [ブラウザのモニタリング](/docs/browser/browser-monitoring/getting-started/introduction-browser-monitoring)
    * [サービスレベル管理](/docs/service-level-management/intro-slm)

    **主なルール:**

    * <DNT>**(L1) JS Error Rate**</DNT>: JavaScriptエラーのあるブラウザーエンティティの割合を確認します。
    * <DNT>**(L1) Service Error Rate**</DNT>: HTTP requestsで5xxエラーを報告しているAPMサービスの割合をチェックします。
    * <DNT>**(L2) Core Web Vitals: Largest Contentful Pane (LCP)**</DNT>：75番目のパーセンタイルのLCPが2.5秒を超えるエンティティにフラグを立てます。
    * <DNT>**(L2) Core Web Vitals: Interaction to Next Paint (INP)**</DNT>: INPの75番目のパーセンタイルが200ミリ秒を超えるエンティティにフラグを付けます。
    * <DNT>**(L2) Core Web Vitals: Cumulative Layout Shift (CLS)**</DNT>：75番目のパーセンタイルのCLSが0.1を超えるエンティティにフラグを付けます。
    * <DNT>**(L3) Service Level Attainment**</DNT>：デフォルトの閾値95％を使用して、定義された各SLIのコンプライアンススコアを評価します。
  </Collapser>

  <Collapser
    id="security"
    title={<DNT>セキュリティ</DNT>
    }
  >
    **目的:** APMサービスおよびインフラストラクチャ全体の脆弱性リスクを評価します。

    **重要:** クリティカルおよび高重大度の脆弱性、アクティブなランサムウェアCVE、およびエクスプロイトの可能性を表面化させることは、修復の優先順位付けに役立ちます。

    **前提条件:**

    * [Security RX](/docs/vulnerability-management/getting-started/overview)
    * [APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)
    * [インフラストラクチャのモニタリング](/docs/infrastructure/introduction-infra-monitoring)

    **主なルール:**

    * <DNT>**APM: Under 3 Critical Severity**</DNT>: APMサービスの重大な脆弱性が3つ未満であることをチェックします。
    * <DNT>**APM: Under 5 High Severity**</DNT>: APMサービスに高深刻度の脆弱性が5つ未満であることを確認します。
    * <DNT>**APM: Active Ransomware**</DNT>: アクティブなランサムウェアキャンペーンがある報告されたCVEにフラグを立てます。
    * <DNT>**APM: Exploit Probability**</DNT>：悪用される可能性が高いクリティカルまたは重大度の高い脆弱性にフラグを付けます（95番目以上のパーセンタイルEPSS）。
    * <DNT>**INFRA: Under 3 Critical Severity**</DNT>: ホストまたはコンテナに、クリティカルな重大度の脆弱性が3つ未満であることを確認します。
    * <DNT>**INFRA: Under 5 High Severity**</DNT>: ホストまたはコンテナに、重大度の高い脆弱性が5つ未満であることをチェックします。
    * <DNT>**INFRA: Active Ransomware**</DNT>: アクティブなランサムウェアキャンペーンがある報告されたCVEにフラグを立てます。
    * <DNT>**INFRA: Exploit Probability**</DNT>：悪用される可能性が高いクリティカルまたは重大度の高い脆弱性にフラグを付けます（95番目以上のパーセンタイルEPSS）。
  </Collapser>

  <Collapser
    id="ai-quality-governance"
    title={<DNT>AI品質およびガバナンス</DNT>
    }
  >
    **目的:** 安全性、コスト効率、およびモデルのコンプライアンス全体にわたるAI応答の運用品質とガバナンスを測定します。

    **重要:** AIの品質とガバナンスの標準を遵守することで、AIの利用を責任ある費用対効果の高い状態に保つことができます。

    **前提条件:** [AIモニタリング](/docs/ai-monitoring/intro-to-ai-monitoring)

    **主なルール:**

    * <DNT>**LLM Runtime Error Rate**</DNT>：LLM対応アプリケーションが低い運用エラー率を維持していることを確認します。
    * <DNT>**Cost Efficiency - Average tokens per assistant response**</DNT>: アシスタントの応答あたりの平均トークン数が予算内に収まっていることを確認します。
    * <DNT>**Model Governance - Approved model usage for assistant responses**</DNT>: 本番環境でのアシスタントの応答に、承認されたモデルのみが使用されていることを確認します。
  </Collapser>
