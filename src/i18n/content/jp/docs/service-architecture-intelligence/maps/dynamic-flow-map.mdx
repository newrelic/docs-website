<DNT>**Dynamic Flow Map**</DNT> 集約されたトレース データのマップ ビューを提供する視覚化ツールです。 フォーカル ノードと呼ばれる特定のサービスに関連するパフォーマンス シグナルを強調表示します。 マップはトレース データを使用して、トレースに参加しているサービスと焦点ノード間の関係を表示し、上流および下流のエンティティ全体のパフォーマンスの変化を相関させます。 これにより、アプリケーションとサービスのパフォーマンスのボトルネックを特定し、根本原因を特定するのに役立ちます。

## 重要な理由 [#why-it-matters]

<DNT>**Dynamic Flow Map**</DNT> パフォーマンスの異常を視覚化して特定できるようにすることで、平均解決時間を短縮するのに役立ちます。 アップストリームとダウンストリームのトレース パフォーマンス データを相関させることで、さまざまなサービスがどのように相互作用し、互いのパフォーマンスに影響を与えるかを理解するのに役立ちます。

マップのコンポーネントと機能について詳しくは、 [<DNT>**Maps**</DNT>](/docs/service-architecture-intelligence/maps/advanced-maps)を参照してください。

<Callout variant="important">
  [<DNT>**Distributed tracing**</DNT> は](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)、<DNT>**Dynamic Flow Map**</DNT> が機能するために必要であり、すべてのAPMエージェントに対してデフォルトで有効になっています。 データが表示されない場合は、 <DNT>**Distributed tracing**</DNT>焦点ノードと関連エンティティに対してアクティブであることを確認し、トレース データが New Relic に表示されることを確認します。
</Callout>

## ダイナミックフローマップを探索する [#explore-dynamic-flow-map]

<DNT>**Dynamic Flow Map**</DNT>にアクセスするには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com)**</DNT>から、任意の[サービス、インフラストラクチャ、またはチーム マップ](/docs/service-architecture-intelligence/maps/advanced-maps)を開きます。
2. <DNT>**Dynamic Flow Map**</DNT>開くには、ノードをポイントして<DNT>**View dynamic flow**</DNT>をクリックします。

[<DNT>**Transactions 360**</DNT>](/docs/apm/transactions/workload-performance-monitoring/transaction-workloads/)から<DNT>**Dynamic Flow Map**</DNT>アクセスすることもできます。 特定のトランザクションのマップを表示して、関連するサービスと各サービスにかかる時間を表示できます。

<img title="dynamic-flow-map" alt="dynamic-flow-map" src="/images/dynamic-flow-map.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services &gt; Maps &gt; Dynamic Flow Map**</DNT>: トレース、異常、パフォーマンスの傾向を示す動的フロー マップ。
</figcaption>

<DNT>**Dynamic Flow Map**</DNT>は、焦点ノードと周囲のエンティティ間の関係を示します。 上流サービスは焦点ノードに向かうエッジで表され、下流サービスは焦点ノードから離れるエッジで表されます。 次のようにマップを操作して、パフォーマンスの傾向、異常、その他の詳細を表示できます。

<CollapserGroup>
  <Collapser id="anomalies" title="異常を分析する">
    異常は、次のようにサービス ノードの横に色分けされたエッジとドットで表示されます。

    * **ブルーエッジ**: サービス間のレイテンシ問題。
    * **赤いエッジ**: サービス間の信号エラー。
    * **青と赤の縞模様のエッジ**: レイテンシの問題とサービス間の信号エラーの両方。
    * **サービスノードの横にある青い点**: サービス内のレイテンシ問題。
    * **サービス ノードの横の赤い点**: サービスに信号エラーがあります。

    ノードの横に異常ドットがあるが、そのノードと焦点ノードを接続するエッジが色分けされていない場合は、ノード間の異常が相関していないことを示します。 [パフォーマンスの傾向](#performance-trends)からメトリクス時系列グラフを確認して、ノードのさまざまな時間に異常が発生したことを確認できます。
  </Collapser>

  <Collapser id="performance-trends" title="パフォーマンスの傾向を見る">
    <DNT>**Dynamic Flow Map**</DNT>で、焦点ノードと接続ノード間のパフォーマンス傾向を表示するには、周囲のノードをクリックします。 ノード間のパフォーマンスの傾向が右側のパネルに表示され、メトリクスの時系列グラフが表示されます。

    次のようにしてパフォーマンスの傾向を分析し、ノードを管理できます。

    * 選択したノード間のパターンを識別します。
    * 時系列チャートを操作したい場合は、その下のノードを選択します。
    * 選択したノード間のすべてのトレースを表示します。
    * ノードの所有権を特定のチームに割り当てます。
    * ノードにタグを追加します。
  </Collapser>

  <Collapser id="investigate-issues" title="問題を調査する">
    <DNT>**Dynamic Flow Map**</DNT>では、焦点ノードに関連するパフォーマンスの問題があるノードを特定した後、それらの間のトレースを調査できます。 トレースを表示するには、右側のパネルから**See all traces** \[すべてのトレースを表示]をクリックします。 トレースを分析することで、パフォーマンスの問題の根本原因を特定し、解決するために必要なアクションを実行できます。
  </Collapser>
</CollapserGroup>
