/\* New Relic でデータが受信されないという問題が発生している場合は、このガイドが一般的な問題の診断と解決に役立ちます。発生した特定の問題に応じて、以下の手順に従ってください: \*/

## 問題: テレメトリー プロデューサーとゲートウェイ間の DNS 解決の問題

DNS 解決エラーのため、エージェントはゲートウェイに接続できません。

## 解決

* エージェント ログにアクセスし (UI 経由または直接)、接続エラーを検索します。
* ネットワーク アドミニストレータに相談して、インフラストラクチャとネットワーク トポロジに基づいて DNS 設定を調整してください。 詳細については、[エージェントのドキュメント](/docs/apm/new-relic-apm/getting-started/introduction-apm/)を参照してください。

## 問題: エージェントゲートウェイ SSL 証明書の問題

エージェントとゲートウェイ間のSSL証明書設定に問題があります。

## 解決

* エージェント ログで SSL 接続エラーを確認します。
* インフラストラクチャとネットワーク トポロジを考慮して、SSL 証明書が正しく構成され、有効であることを確認してください。 詳細については、[エージェントのドキュメント](/docs/apm/new-relic-apm/getting-started/introduction-apm/)を参照してください。

## 問題: ゲートウェイの設定ミス

データはゲートウェイに到達しましたが、New Relic に公開できません。

## 解決

* 送信リクエストとエラー メトリクスを確認します。
* 失敗したルールのメトリクスを確認します。
* 起動していないポッドのログを検査します。
* ゲートウェイ構成設定を修正し、すべてのポッドが動作していることを確認します。

## 問題: サポートされていないテレメトリー プロデューサー/プロトコル

サポートされていない API またはプロトコルからデータが送信されます。

## 解決

* New Relic との互換性を確認します。不正な API クライアントは`501`ステータス コードを返します。
* サポートのために機能リクエストを提出します。
* テレメトリ プロデューサーを構成して、データをNew Relicに直接送信します。

## 問題: 破壊的なルール

ルールによりすべてのデータがドロップされ、New Relic に到達できなくなります。

## 解決

* ドロップデータメトリクスを確認します。
* データ フローを許可するには、破壊的なルールを変更または削除します。

## 問題: データが欠落している

取り込み後に New Relic バックエンドでデータが失われます。

## 解決

* エラー メトリクスを確認し、クライアント側のタイムアウトを確認します。
* リソース枯渇の兆候を評価します。
* [New Relic のステータス](https://status.newrelic.com/)を確認します。
* 影響を受けた時間帯のゲートウェイログを調べる
