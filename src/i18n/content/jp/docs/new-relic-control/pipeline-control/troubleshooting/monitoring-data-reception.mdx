/\* ゲートウェイに送信するプロデューサーからテレメトリー データを受信しているが、ゲートウェイ自体から監視データを受信していない場合、このガイドは一般的な問題の診断と解決に役立ちます。 発生した特定の問題に応じて、以下の手順に従ってください: \*/

## 問題: ライセンスキーが無効です

ゲートウェイは無効なライセンスキー、またはローテーションアウトされたライセンスキーで構成されています。

## 解決

* 問題を確認するには、ゲートウェイ ログに直接アクセスします。内部監視パイプラインと使用状況エクスポーターからの 403 エラーを検索しますが、エージェント データは検索しません。
* ライセンスキーが有効であり、正しく構成されていることを確認してください。 必要に応じてキーを更新してください。

## 問題: ゲートウェイ監視データに影響を与えるクラウドルール

ゲートウェイによって生成されたメトリクスが、 cloudルールによって意図せずドロップされます。

## 解決

* cloudルールの使用状況データを確認し、意図しない低下を特定します。
* ゲートウェイ メトリクスがドロップされないようにcloudルール設定を変更します。

## 問題: メトリクス API のレート制限がかかっています

メトリクスAPIに対するリクエスト制限を超えたため、ゲートウェイからの後続のrequests 429 応答コードで失敗した可能性があります。

## 解決

* アカウント内のOpenTelemetryメトリクスAPIに関連するレート制限イベントを確認してください。
* リクエスト レートの管理と調整に関するガイダンスについては、レート制限のドキュメントを参照してください。
