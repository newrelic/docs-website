/\* ゲートウェイに送信するように構成されたプロデューサーからテレメトリー データを受信していないが、ゲートウェイが独自の監視データを生成している場合、このガイドは一般的な問題の診断と解決に役立ちます。 発生した特定の問題に応じて、以下の手順に従ってください: \*/

## 問題: テレメトリー プロデューサーの構成ミス

テレメトリー プロデューサーの構成が間違っているため、テレメトリー データがゲートウェイに送信されません。

## 解決

* 設定エラーを特定するには、プロデューサー ログにアクセスします。
* 正しい設定手順については、適切なAPI 、エージェント、またはテレメトリー プロデューサーのドキュメントを参照してください。

## 問題: レート制限を受けている

テレメトリーデータのレート制限に達しています。

## 解決

* New Relic API から受信した 429 ステータス コードのゲートウェイ HTTP クライアント応答コードを確認します。
* アカウントで作成された、レート制限を示すイベントを確認します。
* テレメトリーデータ レートの管理および調整に関するガイダンスについては、[レート制限に関するドキュメント](https://docs.newrelic.com/search-results/?query=rate%20limiting&page=2)を参照してください。
