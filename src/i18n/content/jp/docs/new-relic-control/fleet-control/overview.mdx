Fleet Control 、大規模な計装ライフサイクル管理のために、管理対象クラスタのフリートに対する集中リモート コマンドを提供します。 Fleet Controlを使用すると次のことができます。

* **エージェント操作を一元化し、手作業を軽減します。**すべてのエージェントと統合を 1 か所でインストール、監視、構成、更新します。
* **フリートをカスタマイズして一括管理:**計装のニーズごとにエージェントをグループ化し、数回クリックするだけでフリート全体のエージェントのバージョンを簡単にアップグレードできます。 スクリプトは必要ありません。
* **テレメトリー ギャップの排除:** KubernetesAgent Control環境全体に拡張できる 1 つの軽量スーパーバイザー ( ) を使用して、 クラスタ上のエージェントを管理します。

Fleet Control に移動するには、 **Capabilities → New Relic Control → Fleets**に移動します。ピンを使用して、New Relic Control を左側のナビゲーション バーに追加し、簡単にアクセスできるようにすることができます。

## 艦隊とは何ですか?

Fleet Controlでは、フリートは、同一の計装設定を共有する管理対象エンティティ ( Kubernetesクラスタなど) の同種の集合です。 フリートは管理対象エンティティの論理グループとして機能し、合理化された管理と大規模な一貫した監視設定を可能にします。

艦隊は 3 つの主要コンポーネントで構成されています。

* **アクティブなデプロイメント:**最新のデプロイメントの概要

* **設定:** Kubernetesクラスタに適用されている設定

* **エンティティ:**フリート内のすべてのKubernetesクラスタ

### 管理対象エンティティの理解

Fleet Controlのコンテキストでは、管理対象エンティティとは、 インフラストラクチャを実行し、Kubernetes New Relicによって管理されるログ計測を実行するAgent Control クラスタを指します。エージェント設定はフリート レベルで指定され、フリート内のすべてのKubernetesクラスタは、そのフリート内の他のクラスタと同じ設定を共有します。

<Callout title="重要">
  Kubernetesをクラスタにインストールする前に、Agent Control クラスタをフリートに追加することはできません。
</Callout>

## 主な特徴

* **一元管理:** Kubernetesクラスタのフリート上で実行されるエージェントを管理するための統合プラットフォームを提供し、インフラストラクチャ全体にわたる計装を大規模に管理できるようにします。
* **グループ化と所有権:**グループ化メカニズムと所有権境界の両方として機能し、一貫した観察慣行を維持しながら、さまざまなチームがそれぞれのフリートを独立して管理できるようにします。
* **リモート設定：**各種エージェントのリモート設定をサポートします。 詳細については、[スコープ](/docs/new-relic-control/getting-started/#scope)を参照してください。

これらのエージェントは、環境のさまざまな側面を監視し、メトリクス、イベント、ログ、トレース (MELT) などの主要なデータ ポイントを報告する責任を負います。

## 前提条件

Fleet Control を設定する前に、システムが次の前提条件を満たしていることを確認してください。

* **オプトイン:** Agent Control と Fleet Control はパブリック プレビュー段階であるため、製品を使用するにはオプトインする必要があります。
* **Agent Control:** [Agent Control](/docs/new-relic-control/agent-control/setup)Kubernetesフリートに追加する予定の クラスタに がインストールされていることを確認します。

## 必要な権限

**組織スコープの権限:** Fleet Control 内でフリートを作成および編集するには、組織マネージャー ロール (製品では管理設定と呼ばれます) が必要です。
