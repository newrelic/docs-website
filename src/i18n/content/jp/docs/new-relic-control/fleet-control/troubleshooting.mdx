このドキュメントでは、 Fleet Controlのインストールまたは操作中によくある問題をトラブルシューティングする手順について説明します。

## データエクスプローラーの使用

New Relic Data Explorer は、問題を調査するための強力なツールです。New Relic プラットフォームのほとんどのページの下部にある**Query your data** \[データのクエリ]をクリックするか、キーボード ショートカット`CTRL+SHIFT+O`を使用してアクセスできます。

機能の詳細については、[データ エクスプローラーの紹介を](/docs/query-your-data/explore-query-data/query-builder/introduction-new-data-explorer/)ご覧ください。

### NRQL でデプロイメント イベントを調査する

複雑なデプロイメントの問題の場合は、データ エクスプローラーで NRQL クエリを実行して、特定のデプロイメントの詳細なイベント レベルのデータを表示できます。これにより、アクションの正確なシーケンスと発生したエラーを確認できます。

デプロイメントを調査するには:

1. 特定のフリートの**Deployments** \[デプロイメント]タブから、調査するデプロイメントの名前をクリックします。 **デプロイメント ID は**概要の詳細に表示されます。このIDをコピーします。
2. データ エクスプローラーを開きます。
3. 正しいアカウントが選択されていることを確認してください。
4. 次の書き込みを書き込みビルダーに貼り付け、プレースホルダーをコピーした ID に置き換えます。
5. クエリを実行します。分析を容易にするために、結果を**Table** \[表]として表示します。`details`列には、洞察に富んだエラーメッセージが含まれることがよくあります。

```nrql
FROM FleetDeployment SELECT timestamp, action, status, desiredCompletedStatus, agentTypes, ringName, details, agentUid, agentControlUids, fleetId, deploymentId, duration, action WHERE deploymentId = '<YOUR_DEPLOYMENT_ID>' SINCE 1 hour ago LIMIT MAX
```
