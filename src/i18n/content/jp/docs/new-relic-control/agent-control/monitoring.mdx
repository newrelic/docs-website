## 概要

Agent Controlはローカル監視ツールとして設計されており、New Relic Controlは集中監視機能を提供します。Agent Controlは、管理対象エージェントのクラスタ内監視役として機能します。その主な役割は、エージェントが正しくデプロイされ、実行されていることを確認することです。 このガイドでは、管理対象エージェントを監視する方法について詳しく説明します。 艦隊の状態を俯瞰的に把握するには、New Relic Control UIを使用できます。詳細なトラブルシューティングと検証を行うには、ホストまたはクラスタ上で直接チェックを実行できます。

## New Relic Controlでの集中監視

Agent Controlソリューションの究極の目標は、 New Relic UI内で包括的で一元化された監視エクスペリエンスを提供することです。 これにより、ヘルスチェックのために手動でkubectlコマンドを実行する必要がなくなり、フリート全体の状況を統一的に把握できるようになります。この機能は、Agent Controlがサポートするすべての環境で利用可能です。

一元化されたビューは、New Relic 内のエージェント専用のエンティティダッシュボードに直接リンクします。これにより、フリートの健全性の監視から、個々のエージェントによって収集された特定のメトリクス、ログ、トレースの調査に迅速に移行できるようになります。 これにより、高レベルの管理から詳細な観察までのシームレスなワークフローが提供されます。

## Kubernetes監視

これらの手順は、Kubernetes環境で実行されるAgent Controlに適用されます。

### クラスタ上のヘルスチェックとトラブルシューティング

これらのチェックは、標準の Kubernetes ツールとコマンドを使用して実行されます。

問題のトラブル発生時は、kubectl とローカル ステータス エンドポイントを使用して、 Agent Controlとその管理対象エージェントのステータスを直接検査できます。

### ポッドのステータスとログを確認する

標準のKubernetesコマンドを使用して、 Agent Controlスーパーバイザとそのサブエージェント ポッドが期待どおりに実行されていることを確認できます。

```shell
# Check Agent Control pods
kubectl get pods -n newrelic-agent-control

# Check sub-agent pods (e.g., in the 'newrelic' namespace)
kubectl get pods -n newrelic

# Inspect logs for the Agent Control supervisor
kubectl logs <agent-control-pod-name> -n newrelic-agent-control
```

### ローカルステータスエンドポイントを使用する

Agent Control 、スーパーバイザとそのすべての管理対象エージェントの両方のヘルス チェックを提供するローカル ステータス エンドポイントを公開します。

1. ポートを転送する: ターミナルを開き、次のコマンドを実行して、ローカル ポートをエージェント コントロール ポッドに転送します。

```shell
kubectl port-forward <agent-control-pod-name> 51200:51200 -n newrelic-agent-control
```

2. ステータスを要求する: 新しい端末で、 curlを使用してステータスのエンドポイントを書き込みます。

```shell
curl localhost:51200/status
```

出力は、デプロイメントの詳細、エージェントのバージョン、検出されたエラーなどの詳細な稼働ステータスを提供する JSON 応答です。
