New Relicは2026年に[古いランタイムのサポートを終了](/eol/2025/11/eol-chrome-140-node-22-synthetics/)します。Syntheticモニターのサポートされている最新のランタイムへの移行を簡素化するため、お客様の対応を必要としないすべてのモニターを自動アップグレードしました。ランタイムアップグレードツールを使用して、まだ対応が必要な残りのモニターを確認します。

## 前提条件 [#prerequisites]

モニターのアップグレードを開始する前に、以下の要件を満たしていることを確認してください。

* **ランタイムの一括アップグレード権限:** UIから一括更新を適用するために必要です。[製品管理者](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#standard-roles)はデフォルトでこの権限を持っています。[カスタムロール](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#synthetics)に追加することもできます。
* **AIエンタイトルメント（<DNT>Fix with AI</DNT>のみ）:** `nrai_ccu`または`nrai_discount_usage`のエンタイトルメントが必要です。FedRAMPアカウントは、このエクスペリエンスからブロックされています。エンタイトルメントを有効にするには、New Relicアカウントチームに連絡してください。

## 残りのモニターをアップグレードする [#upgrade-options]

注意が必要な各モニターについて、2つのパスのいずれかを選択できます：

<CollapserGroup>
  <Collapser id="fix-manually" title="手動で修正">
    AI生成に頼るのではなく、<DNT>**Fix manually**</DNT>オプションを使用してモニタースクリプトを記述および編集します。モニターを手動で更新するには：

    1. <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>に移動します。

    2. モニターを見つけ、<DNT>**Fix manually**</DNT>をクリックします。

    3. <DNT>**Fix script manually**</DNT>ウィンドウで、新しいランタイム環境用にスクリプトコードを更新します。

    4. <DNT>**Close and start validation**</DNT>をクリックします。システムが更新されたスクリプトを検証している間、モニターは<DNT>**Monitors we&apos;re validating**</DNT>テーブルに移動します。

       <Callout title="注記">
         検証に失敗した場合は、モニターの横にある**⋯**をクリックし、メニューから<DNT>**Fix manually**</DNT>を選択し、スクリプトを更新して、再度<DNT>**Close and start validation**</DNT>をクリックします。
       </Callout>

    5. 検証が成功し、ステータスが<DNT>**Fixed**</DNT>と表示されたら、<DNT>**Ready to update**</DNT>タブをクリックします。

    6. 検証済みモニターの横にあるチェックボックスをオンにして、<DNT>**Upgrade selected**</DNT>をクリックします。

    完了すると、確認通知が表示され、モニターはアップグレードの成功を確認できる<DNT>**Updated**</DNT>タブに移動します。
  </Collapser>

  <Collapser id="fix-with-ai" title="AIで修正">
    <DNT>New Relic AI (NRAI)</DNT>に更新されたスクリプトを生成させるには、<DNT>**Fix with AI**</DNT>オプションを使用します。AI支援によるスクリプト生成は、主にランタイムアップグレード中に発生した`timeout`エラーの修正を目的としています。その他のエラータイプについては、スクリプト自体に変更が必要になる可能性が高いため、[手動で](#fix-manually)変更する必要があります。

    AIを使用してモニターを更新するには：

    1. <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Synthetic Monitoring** &gt; **Runtime Upgrades**</DNT>に移動します。
    2. モニターを見つけ、<DNT>**Fix with AI**</DNT>をクリックします。
    3. AIが提案したスクリプトを確認します。元のコードに加えられた特定の変更を表示するには、<DNT>**Show comparison**</DNT>をクリックして左右に並べた差分表示を開きます。
    4. <DNT>**Continue to validation**</DNT>をクリックします。システムは、新しいスクリプトをテストしてエラーなしで実行されることを確認するため、一時的にモニターを検証状態にします。
    5. スクリプトが検証されたら、<DNT>**Ready to update**</DNT>タブをクリックします。
    6. 検証済みモニターの横にあるチェックボックスをオンにして、<DNT>**Upgrade selected**</DNT>をクリックします。

    完了すると、確認通知が表示され、モニターはアップグレードの成功を確認できる<DNT>**Updated**</DNT>タブに移動します。
  </Collapser>
</CollapserGroup>

複数のアップグレードを一度に実行できるため、システムが各スクリプトの検証を完了するのを待たずに、複数のモニターで並行して作業できます。
