このガイドでは、Dockerを使用したプライベートロケーション環境でのデュアルスタック（IPv4およびIPv6）テストをサポートするために必要な、インフラストラクチャの検証、Dockerデーモンの設定、およびイメージのデプロイメントについて説明します。

## 前提条件 [#prerequisites]

始める前に、次のものがあることを確認してください。

* <DNT>**Docker**</DNT>: Dockerがホストインスタンスにインストールされ、実行されている必要があります

* <DNT>**Permissions**</DNT>: システム設定を変更するには、sudoまたはrootアクセス権が必要です。

* <DNT>**Public images**</DNT>: デュアルスタックのサポートには、Docker Hubで利用可能な次のイメージバージョン以降が必要です：

  * [ジョブマネージャー`519`以降](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/job-manager-release-notes/job-manager-release-519/): `newrelic/synthetics-job-manager`
  * [Pingランタイム`1.65.0`以降](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/ping-runtime-release-notes/ping-runtime-1.65.0/): `newrelic/synthetics-ping-runtime`
  * [ノードAPIランタイム`rc1.15`以降](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-api-runtime-release-notes/node-api-runtime-rc1.15/): `newrelic/synthetics-node-api-runtime`
  * [Nodeブラウザランタイム`rc1.15`以降](https://docs.newrelic.com/docs/release-notes/synthetics-release-notes/node-browser-runtime-release-notes/node-browser-runtime-rc1.15/): `newrelic/synthetics-node-browser-runtime`

<Steps>
  <Step>
    ## インフラストラクチャの検証 [#validate-infrastructure]

    続行する前に、ホストマシンとネットワークがデュアルスタックネットワーキング用に設定されていることを確認してください。

    `ping6 ipv6.google.com`または`ping -6 ipv6.google.com`、あるいはご使用のマシンでの同等のコマンドを実行して、IPv6接続をテストします。

    <Callout variant="important">
      pingが失敗した場合は、このインストレーションを続行する前に、VPC、サブネット、およびセキュリティグループがIPv6トラフィックを許可するように設定されていることを確認してください。デュアルスタック運用には、環境がIPv4とIPv6の両方をサポートしている必要があります。
    </Callout>
  </Step>

  <Step>
    ## Dockerがデュアルスタックの準備ができていることを確認する [#configure-docker]

    DockerのインストレーションがIPv4とIPv6の両方のトラフィックを処理するように構成されていることを確認してください。DockerでIPv6が有効になっていることを確認するには、次を実行します：

    ```bash
    docker info | grep -i ipv6
    ```

    出力で、IPv6が有効になっていることを確認できるはずです。そうでない場合は、先に進む前に[Dockerのドキュメント](https://docs.docker.com/config/daemon/ipv6/)を参照して、デュアルスタックサポートを有効にしてください。
  </Step>

  <Step>
    ## 外形監視ジョブマネージャーをリリースします [#launch-job-manager]

    デュアルスタック対応のランタイムイメージはDocker Hubで公開されており、ジョブマネージャーによって自動的にプルされます。プライベートロケーションキーを使用して、外形監視ジョブマネージャーコンテナを実行します：

    <Callout variant="tip">
      `<YOUR_PRIVATE_LOCATION_KEY>`をNew Relic UIの実際のプライベートロケーションキーに置き換えます。
    </Callout>

    ```bash
    docker run \
      -e PRIVATE_LOCATION_KEY=<YOUR_PRIVATE_LOCATION_KEY> \
      -d \
      --restart unless-stopped \
      -v /var/run/docker.sock:/var/run/docker.sock:rw \
      newrelic/synthetics-job-manager
    ```

    外形監視ジョブマネージャーは、デュアルスタックサポートを有効にして実行されるようになり、モニターがIPv4とIPv6の両方のエンドポイントをテストできるようになります。
  </Step>

  <Step>
    ## インストールの確認 [#verify-installation]

    Syntheticsジョブマネージャーが正しく実行されていることを確認するには：

    1. コンテナのステータスを確認する：

       ```bash
       docker ps
       ```

    2. ジョブマネージャーのログを表示：

       ```bash
       docker logs <container_id>
       ```

    3. New Relic UIで<DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Synthetic monitoring &gt; Private locations**</DNT>に移動し、プライベートロケーションが正常と表示されていることを確認します。
  </Step>
</Steps>

## 次は何ですか？ [#whats-next]

デュアルスタック対応のプライベートロケーションのセットアップ後、次のことができます：

* プライベートロケーションを使用して、IPv4とIPv6の両方のエンドポイントをテストする[Syntheticモニターを作成](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors)します
* 高度なオプションについては、[ジョブマネージャーの設定](/docs/synthetics/synthetic-monitoring/private-locations/job-manager-configuration)を参照してください
* [プライベートロケーション](/docs/synthetics/synthetic-monitoring/private-locations/monitor-private-locations)のヘルスとパフォーマンスをモニターする
