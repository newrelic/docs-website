Syntheticモニタリングでは、プライベートロケーションとは、Syntheticsジョブマネージャーまたはプライベートミニオン（レガシー）の集合です。Syntheticsジョブマネージャーとミニオン（レガシー）はコンテナ化されたアプリケーションであり、Synthetics UI経由でセットアップされたジョブを受け取り、管理します。

プライベートロケーションには、任意の数のSyntheticsジョブマネージャーまたはプライベートミニオンを含めることができます。プライベートロケーションでは、Syntheticモニタリングの対象地域を拡張したり、イントラネットサイトなど、ファイアウォールの背後にあるウェブサイトをモニターすることができます。

## プライベートロケーションの作成 [#create-location]

[Syntheticsジョブマネージャーのインストレーション](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager)または[プライベートミニオン（レガシー）のインストレーション](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms)を行う前に、プライベートロケーションを作成する必要があります。

新しいプライベートロケーションを作成するには、以下の手順に従います。

<img
  title="Click the Create private location button"
  alt="screenshot-manage-private-locations.png"
  src="/images/synthetic_screenshot-full_find-private-location-button.webp"
/>

<figcaption>
  **[one.newrelic.com > Synthetic monitoring > Private locations](https://one.newrelic.com/synthetics-nerdlets/private-location-list)**: Use the **Private Locations**ページを使用して、プライベートロケーションを作成、編集、および削除します。
</figcaption>

1. [アカウントアクセスの要件](#account-access)を含む[要件](#requirements)を満たしていることを確認します。

2. **[one.newrelic.com > Synthetic monitoring > Private locations](https://one.newrelic.com/synthetics-nerdlets/private-location-list)**の順に移動します。次に、<Icon name="fe-plus-circle"/>**Create private location**を選択します。

   <Callout variant="tip">
     最初のモニターを作成すると、プライベートロケーションサブメニューが使用可能になります。
   </Callout>

3. ロケーション名を入力します。

4. オプション：これらの追加設定を構成します。

   * **Description**：他のアカウントユーザーのためにプライベートロケーションについて説明します。
   * **[Verified script execution](/docs/synthetics/new-relic-synthetics/private-locations/verified-script-execution-private-locations)**：このロケーションにスクリプトを割り当てる際、またはミニオンをロケーションに追加する際に、パスフレーズが必要です。

5. **「Create」**を選択します。

ロケーションの作成後、合成モニタリングは、**プライベートミニオンをインストール**できるように、[プライベートロケーションキー](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms#install-update)をリストします。

<img
  title="Add private location to a synthetic monitor"
  alt="screenshot-manage-private-locations.png"
  src="/images/synthetic_screenshot-full_private-location-page.webp"
/>

## Pingモニターチェック [#checks]

各Syntheticsジョブマネージャーまたはミニオンは、1分間に約200のpingモニターチェックを実行できます（月に8,640,000チェック）。特定のロケーションのジョブキューが大きくなる場合は、[ミニオンを追加](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms#install-update)します。次の項目により、ミニオンの正確な容量は異なります：

* ネットワークのパフォーマンス
* スクリプトの複雑さ
* プライベートミニオンのハードウェア構成

## ロケーションキューへのジョブの追加 [#add-jobs]

ロケーションのキューにジョブを追加するには、標準の手順に従って、[モニターを追加または編集](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors#adding-monitors)し、**Private locations**リストからロケーションを選択します。既存のモニターを割り当てるには、その[モニターの設定](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors#editing-monitors)を編集します。

## プライベートロケーションの管理 [#manage-location]

<Callout variant="tip">
  この機能にアクセスできない場合、[機能とデータへのアクセスに影響を与える要因](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data)を確認してください。
</Callout>

Syntheticモニタリングには、ロケーション、個別のSyntheticsジョブマネージャーやミニオン（レガシー）を管理するツールが含まれます。また、[新しいSyntheticsジョブマネージャー](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager)または[ミニオン（レガシー）](/docs/synthetics/synthetic-monitoring/private-locations/install-containerized-private-minions-cpms#install-update)をインストールし、ジョブキューが停滞する場合はクリアすることもできます。

これらのツールにアクセスするには、**[one.newrelic.com > Synthetic monitoring > Private locations](https://one.newrelic.com/synthetics-nerdlets/private-location-list)**の順に移動します。以下の手順に従います。

<CollapserGroup>
  <Collapser
    id="clear-job-queue"
    title="ジョブキューをクリア"
  >
    <Icon name="fe-more-horizontal"/>アイコンをクリックし、**Clear queue**を選択します。

    これは、スケジュール済みジョブ数がミニオンの処理能力よりも速く増加する場合、たとえばミニオンがオフラインの時などに役立ちます。
  </Collapser>

  <Collapser
    id="change-location"
    title="ロケーションの変更"
  >
    <Icon name="fe-more-horizontal"/>アイコンをクリックし、**Edit**を選択します。
  </Collapser>

  <Collapser
    id="view-status"
    title="ステータスの表示"
  >
    個別のSyntheticsジョブマネージャーまたはミニオンのステータスを表示するには、リストから親ロケーションの名前を選択します。

    緑色の[稼働ステータスインジケーター](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#health-status)が、有効なSyntheticsジョブマネージャーまたはミニオンを識別します。
  </Collapser>

  <Collapser
    id="view-key"
    title="プライベートロケーションキーの表示"
  >
    <Icon name="fe-more-horizontal"/>アイコンをクリックし、**Edit**を選択します。
  </Collapser>

  <Collapser
    id="delete-location"
    title="ロケーションを削除"
  >
    ロケーションの<Icon name="fe-more-horizontal"/>アイコンをクリックし、**Delete**を選択します。

    削除しても、そのロケーションに割り当てられたミニオンは終了しません。ミニオンは、手動で終了するか再度割り当てる必要があります。
  </Collapser>

  <Collapser
    id="enable-disable-vse"
    title="検証済みスクリプトの実行を有効または無効にする"
  >
    [検証済みスクリプトの実行](/docs/synthetics/synthetic-monitoring/private-locations/verified-script-execution-private-locations/)を有効または無効にするには、以下の手順に従います。

    * Syntheticsジョブマネージャー：検証済みスクリプトの実行はまだサポートされていません
    * CPM：[MINION_VSE_PASSPHRASE](/docs/synthetics/synthetic-monitoring/private-locations/containerized-private-minion-cpm-configuration#environment-variables)変数をミニオンに渡す必要があります。次に、**\[プライベートロケーション]**タブで**\[編集]**をクリックし、メニューで**\[検証済みスクリプトの実行]**オプションを有効にします。
  </Collapser>

  <Collapser
    id="rotate-private-location-keys"
    title="プライベートロケーションキーのローテーション"
  >
    プライベートロケーションではプライベートロケーションキーを使用します。このトークンにより、プライベートロケーションジョブマネージャー（またはプライベートミニオン）がジョブをプルして実行し、New Relicプラットフォームに結果をレポートできるようになります。

    これらのキーが侵害された場合、ローテーションすることができないため、侵害された場合は、そのキーを無効にして、新しいプライベートロケーションをデプロイする必要があります。

    <Steps>
      <Step>
        新しいプライベートロケーションを作成するには、以下の手順に従います。

        1. **[one.newrelic.com > Synthetic monitoring > Private locations](https://one.newrelic.com/synthetics/private-location-list)**の順に移動します。次に、**Create private location**をクリックします。<img title="Screenshot showing where to add a new private location" alt="Screenshot showing where to add a new private location" src="/images/synthetic_screenshot-crop_private-location-add.webp"/>

        2. プライベートロケーション名を入力し、**Enable verified script execution**をオンにします。次に、**Generate key**をクリックして、新しいプライベートロケーションを作成します。

           <img
             title="Screenshot of the screen for generating private locations"
             alt="Screenshot of the screen for generating private locations"
             src="/images/synthetic_screenshot-crop_generate-private-location-key.webp"
           />

        3. ジョブマネージャーまたはプライベートミニオンを新しいプライベートロケーションで更新します。**設定とデプロイ**セクションにはサンプルコードが含まれています。

           <img
             title="Screenshot of the configure/deploy instructions for private locations"
             alt="Screenshot of the configure/deploy instructions for private locations"
             src="/images/synthetic_screenshot-crop_configure-deploy-private-locations.webp"
           />
      </Step>

      <Step>
        モニターを新しいプライベートロケーションに移行します。移行するモニターごとに以下の手順を行います。

        1. **[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics)**に移動します。モニターの右側にある省略記号アイコン**...**をクリックし、**Monitor settings**を選択します。

           <img
             title="How to find and edit monitor settings in synthetic"
             alt="How to find and edit monitor settings in synthetic"
             src="/images/synthetic_screenshot-crop-monitor-settings.webp"
           />

        2. **Select locations**タブで、旧ロケーションを無効にし、新しいロケーションを有効にします。次に、モニターを保存します。

           <img
             title="Screenshot demonstrating how to swap private locations when rotating private location   keys"
             alt="Screenshot demonstrating how to swap private locations when rotating private location keys"

        1. **[one.newrelic.com > Synthetic monitoring > Private locations](https://one.newrelic.com/synthetics/private-location-list)**の順に移動します。

        2. 旧ロケーションを見つけます。フィルターボックスでロケーション名またはIDでフィルターできます。

        3. 右側の省略記号アイコン**...**をクリックし、**Edit**を選択します。

           <img
             title="Screenshot of how to edit the key for a private location when rotating keys"
             alt="Screenshot of how to edit the key for a private location when rotating keys"
             src="/images/synthetic_screenshot-crop_edit-private-location-rotate.webp"
           />

        4. ゴミ箱アイコンをクリックし、プライベートロケーションの削除を確認します。

           <img
             title="Screenshot of delete confirmation key for private location"

           />
      </Step>
    </Steps>

    移行したモニターのいずれかで安全な認証情報を使用していた場合は、 [それらの認証情報をローテーションする](/docs/synthetics/synthetic-monitoring/using-monitors/store-secure-credentials-scripted-browsers-api-tests/#rotate-secure-credentials)必要もあります。

## プロキシ構成の設定 [#proxy]

ローカルのプライベートロケーションから実行されるSyntheticのスクリプト化モニターにプロキシサーバー構成を設定できます。詳細については、[Syntheticsプロキシ設定](/docs/synthetics/synthetic-monitoring/scripting-monitors/set-proxy-settings-properties-scripted-monitors)をご覧ください。
