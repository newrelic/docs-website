[シンセティック・モニタリング](/docs/synthetics/new-relic-synthetics/getting-started/introduction-new-relic-synthetics) Webサイト、重要なビジネス・トランザクション、APIエンドポイントを定期的にチェックし、最適なパフォーマンスを測定します。 [ブラウザ・モニタリング](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) デバイス、ブラウザ、接続速度のあらゆるバリエーションにおいて、Webページのパフォーマンスの実際の結果を追跡します。これらを併用することで、実際のユーザー（ブラウザ）のインタラクションと合成モニターのページロードタイムを直接比較することができます。

## パフォーマンスとトレンドの比較 [#features]

New Relic の比較チャート機能は、オペレーションマネージャーやチームを支援します。

* ページロード時間のベンチマーク
* サイトを最適化するための計画に役立つ追加のインサイト
* モニタリング機能を切り替えることなく、合成トレンドと実際のブラウザのパフォーマンスを比較することができます。

たとえば、ページの停止中に、合成モニタリングの傾向を実際のブラウザモニタリングの比較と比較して、問題が合成UIにも表示されるかどうか、またはNewRelic以外の変数が原因であるかどうかを確認できます。これにより、どこで行動を起こすかをより効率的に知ることができます。

## 何が必要 [#requirements]

比較チャート機能が必要です。

* ブラウザ監視は、 [ブラウザ SPA エージェント](/docs/browser/single-page-app-monitoring/get-started/spa-compatibility-requirements) ([バージョン 885](/docs/release-notes/new-relic-browser-release-notes/browser-agent-release-notes) 以上)で有効になります。
* [シンセティック・ブラウザまたはスクリプト・モニター](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors) ブラウザ・エージェントがモニターしたのと同じURLで1つまたは複数のテストを行います。

合成またはブラウザ監視から比較チャート機能を有効にした後、追加の設定は必要ありません。比較チャート機能は、New Relic が一致する URL を見つけたときに表示されます。

## 比較データの有効化 [#enable]

[シンセティック](#enable-synthetics) または [ブラウザ](#enable-browser) モニタリングのいずれかで比較チャート機能を有効にした後、追加の設定は必要ありません。シンセティックまたはブラウザモニタリングで一致するURLが見つかると、比較チャート機能が表示されます。

## 比較データを見る [#view-ui]

比較チャート機能は、シンセティック・モニタリングがブラウザ・モニタリングと一致するURLを特定し、それを比較できるときに表示されます。ブラウザとシンセティックのパフォーマンスを比較するには、 [シンセティックモニタリング **サマリー** ページ](/docs/synthetics/new-relic-synthetics/pages/synthetics-overview-page-view-monitors-performance) または [ブラウザモニタリング **ページビュー** ページ](/docs/browser/new-relic-browser/additional-standard-features/page-views-understanding-your-sites-popularity) (ブラウザアプリまたはシングルページアプリの場合)から、New Relic の機能を切り替える必要はありません。

選択された [データソース](#data-sources) に基づき、その概要を示します。

* ブラウザ（実際のユーザー）のページビューと、ユーザーが閲覧権限を持つ合成モニターで見つかった一致するURLとの全体的な速度のパーセンテージ比較
* シンセティックモニターで見つかったURLのうち、ブラウザのロールアップURLと一致するものの数

比較チャートの概要を見るには

1. シンセティック モニタリング UI から: **[one.newrelic.com > シンセティック モニタリング](https://one.newrelic.com/synthetics-nerdlets)** > (モニターを選択) > **サマリー**に移動します。

   また

   ブラウザ監視UIから: **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (ブラウザアプリを選択) > Page views**に移動し、Synthetic監視によって監視されているページを選択します。

2. 比較チャート機能では、全体的な速度の割合と一致するURLの数をまとめて確認できます。

3. 詳細を表示するには、サマリーの右矢印 <Icon name="fe-chevron-right"/>のアイコンを選択してください。

場合によっては、ブラウザエージェントが`BrowserInteraction`イベントを収集する前に、単純なブラウザモニタが終了することに注意してください。この場合、比較チャートデータはUIに表示されません。この問題を解決するには、代わりにスクリプト化されたブラウザーモニターを作成し、ページが読み込まれた後に待機（スリープ）する呼び出しを追加します。次に例を示します。

```js
$browser.get('https://www.mywebsite.com').then(function() {
  return $browser.sleep(1000);
});
```

## 比較データの非表示・表示 [#hide]

比較チャート機能を非表示にするには、エリプシス <Icon name="fe-more-horizontal"/>アイコンを選択します。

* 表示させたまま、ページの上部から移動させるには、 **Move to bottom** を選択します。
* ページの元の位置に戻すには、 **Move to top** を選択します。
* ページから削除するには、 **Hide all Synthetics/Browser comparisons** を選択します。
* 削除した後にページに戻すには、標準的な手順で [比較表作成を有効にする](#enable) 。
