## 問題

URL またはスクリプトに機密情報 (パスワードやユーザー名など) が含まれる合成モニターがあります。

## 解決

### モニター内の URL を置き換える

URL をチェックする合成モニターを使用しており、URL に資格情報またはその他の機密情報を配置している場合は、それらの資格情報を定期的に変更することが必要な場合があります。これを行うには、合成スクリプトを編集して情報を編集または削除し、URL に以前に配置された資格情報を確実にローテーションします。モニターの追加と編集の詳細な手順については、 [「モニターの追加と編集」](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors/)を参照してください。

### スクリプト内の平文認証情報をローテーションする

[スクリプト化されたモニター](/docs/synthetics/synthetic-monitoring/scripting-monitors/introduction-scripted-browser-monitors/)を作成する場合、モニターが実行するアクション用の独自のスクリプトを作成できます。

スクリプト内に平文で機密情報を含めた場合は、資格情報をローテーションする必要があります。

1. **[one.newrelic.com > Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**に移動し、編集するモニターを選択します。

2. **Settings \[設定] > Script \[スクリプト]**を選択して、監視スクリプトを編集します。

   <img
     title="Screenshot of the synthetic script editor page"

3. プレーンテキストの資格情報を回転または編集します。可能な限り、プレーンテキストを[安全な認証情報](/docs/synthetics/synthetic-monitoring/using-monitors/store-secure-credentials-scripted-browsers-api-tests/)に置き換えてください。
