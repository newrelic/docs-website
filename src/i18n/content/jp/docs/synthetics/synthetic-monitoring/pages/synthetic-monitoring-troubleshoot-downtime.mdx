<DNT>**Failures**</DNT>ページを使用して、ダウンタイム イベントやその他のエラーを見つけてトラブルシューティングします。 ダウンタイム集中イベントは、モニターが完全に実行できない場合に発生します。 たとえば、 GETリクエストが失敗すると ping モニターは「ダウン」になりますが、スクリプトのいずれかの部分が実行に失敗すると、スクリプト化ブラウザモニターは「ダウン」になります。 興味深いダウンタイムを見つけたら、それを選択して、 [そのダウンタイムまとめイベントの詳細な結果を表示し、](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding)トラブルシューティングを行います。

## 失敗のページを見る [#viewing]

モニターの障害にアクセスするには: <DNT>**[one.newrelic.com &gt;Synthetic monitoring](https://one.newrelic.com/synthetics-nerdlets)**</DNT> &gt; (モニターを選択) &gt; <DNT>**Failures**</DNT> 。

<img title="Synthetic monitoring - Failures page" alt="Synthetic monitoring - Failures page" src="/images/synthetic_screenshot-full_failure-page.webp" />

<figcaption>
  障害の上にマウスを移動すると、その障害に関するデータがすぐに表示されます。ドットをクリックすると、失敗の詳細なレポートが開きます。<DNT>**Run check**</DNT>クリックして[、失敗したモニターを再確認する](/docs/synthetics/synthetic-monitoring/using-monitors/recheck-failed-monitors)こともできます。
</figcaption>

## 個々のダウンタイムの表示 [#individual]

個々のダウンタイムイベントを選択して、より詳細に表示できます。 特定の障害に応じて、ダウンタイム結果にはサーバー エラーメッセージ (<DNT>**Server replied with &quot;HTTP 500&quot; error**</DNT>など) のみが含まれる場合もあれば、完全または部分的な[ウォーターフォール ビューが](/docs/synthetics/new-relic-synthetics/using-monitors/viewing-monitor-results#understanding)含まれる場合もあります。

ダウンタイムの結果には、モニターの一部のみが正しく実行された場合のウォーターフォールが含まれます。たとえば、 [301 リダイレクトは](http://en.wikipedia.org/wiki/HTTP_301)障害のある Web ページにリンクする可能性があります。ブラウザはリダイレクトを正しく実行しますが、宛先の Web ページはエラーを返します。エラーメッセージまたはウォーターフォールを使用して、ダウンタイム集計イベントのトラブルシューティングを行ってください。
