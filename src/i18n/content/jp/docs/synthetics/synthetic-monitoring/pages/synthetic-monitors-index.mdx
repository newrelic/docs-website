シンセティックモニタリングツールのモニターインデックスには、お客様の New Relic アカウントに関連付けられているすべてのモニターがリストアップされており、各モニターの過去 24 時間のパフォーマンスのスナップショットが表示されます。個々のモニターを選択すると、 [**Summary** page](/docs/synthetics/new-relic-synthetics/dashboards/synthetics-overview-dashboard-view-monitors-performance) を表示し、時系列でのパフォーマンスをより深く理解することができます。また、 [リストをフィルタリング](#search-monitors) して、類似したモニターのパフォーマンスを素早く比較することもできます。

## モニターインデックスを見る [#viewing]

モニターのインデックス（リスト）にアクセスするには、 **[one.newrelic.com](https://one.newrelic.com) > エクスプローラー> シンセティックモニター** にアクセスしてください。 [エクスプローラー](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/) を使用して、すべての [エンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/) 、つまり、合成モニターからアプリケーション、ホスト、あらゆる要素のカスタムグルーピングまで、データを報告していると特定できるものすべてにアクセスできます。

あるいは、 [**one.newrelic.com**](http://one.newrelic.com/) **> Synthetics**.

## モニタリング・メトリクスの理解 [#understanding]

モニターインデックスを使用して、モニターにアクセスし、モニターのパフォーマンスのクイックスナップショットを表示します。このインデックスには、以下のメトリクスが含まれます。

* [**Alert status**](/docs/alerts/new-relic-alerts/configuring-alert-policies/identify-entities-without-alert-policies) : モニターに発生しているアラートの状態を示します。

  * グリーン。オープン違反なし
  * 赤。致命的な違反が進行中
  * グレー：アラートを伴うアラート条件が定義されていない

* **モニターの状態**: **Mute** または **Disabled** など、モニターに適用されている状態を示します。

* **成功率**: 成功に終わったモニターチェックの割合。マルチステップのモニターで、すべてのステップが完了しなかった場合は失敗とみなされます。

* **[Locations failing](/docs/synthetics/new-relic-synthetics/using-monitors/adding-editing-monitors#setting-location)** : 与えられた時間枠の中で失敗したロケーションの数。

* **[Period](/docs/synthetics/new-relic-synthetics/using-monitors/adding-editing-monitors#setting-frequency)** : モニターチェックの実行頻度。

* **[Monitor type](/docs/synthetics/new-relic-synthetics/using-monitors/add-edit-monitors#setting-type)** : 選択されているモニタータイプです。
