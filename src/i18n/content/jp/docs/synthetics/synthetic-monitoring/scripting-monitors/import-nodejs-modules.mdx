New Relic は、標準的なシンセティックモニタリングの WebDriverJS ベースのスクリプト環境に加えて、人気の高い Node.js モジュールを多数サポートしており、 [スクリプトブラウザ](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) や API テストを強化することができます。

## 追加モジュールをインポートする [#import-procedure]

サポートされているモジュールをインポートするには、そのモジュールの標準的なインポート手順に従ってください。例えば、以下のようになります。

* `crypto-js`をインポートするには、次を呼び出します。

  ```js
  var crypto = require('crypto-js');
  ```

* `chai`をインポートするには、次を呼び出します。

  ```js
  var assert = require('chai').assert;
  ```

## 対応するサードパーティモジュール

シンセティック・スクリプト・モニターのバージョンによって、サポートされるサードパーティ・モジュールが異なります。異なるモニターバージョンでサポートされているモジュールの情報は、 [モニターバージョン比較表](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment) を参照してください。

## 対応するNode.jsコアモジュール [#supported-core-modules]

  <Collapser

    すべての[Node.js コア モジュール](https://nodejs.org/docs/latest-v16.x/api/all.html)は、Chrome 100、Node.js 16.10、またはそれ以降のランタイムで利用できます。

    New Relicのレガシー合成モニタリングランタイムは、次のオプションのNode.jsコアモジュールもサポートしています。

    * [`assert`](http://nodejs.org/api/assert.html)
    * [`crypto`](http://nodejs.org/api/crypto.html)
    * [`dns`](https://nodejs.org/api/dns.html)
    * [`fs`](https://nodejs.org/api/fs.html)
    * [`net`](https://nodejs.org/api/net.html)
    * [`querystring`](http://nodejs.org/api/querystring.html)
    * [`timers`](http://nodejs.org/api/timers.html)
    * [`tls`](https://nodejs.org/api/tls.html)
    * [`url`](http://nodejs.org/api/url.html)
    * [`util`](http://nodejs.org/api/util.html)
    * [`zlib`](https://nodejs.org/api/zlib.html)
  </Collapser>
</CollapserGroup>

## サポートされていないモジュール

[外形監視 Job Manager](/docs/synthetics/synthetic-monitoring/private-locations/install-job-manager/)ではカスタム ノード モジュールを使用できます。 外形監視ジョブ マネージャーのカスタム モジュールを構成する方法については、 [「ジョブ マネージャーの設定」を](/docs/synthetics/synthetic-monitoring/private-locations/job-manager-configuration#custom-modules)参照してください。

サポート対象外のモジュールは動作が保証されておらず、 [New Relicのサポート範囲](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings/#scope-support)外となります。
