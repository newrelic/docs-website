New Relic は、標準的なシンセティックモニタリングの WebDriverJS ベースのスクリプト環境に加えて、人気の高い Node.js モジュールを多数サポートしており、 [スクリプトブラウザ](/docs/synthetics/new-relic-synthetics/scripting-monitors/writing-scripted-browsers) や API テストを強化することができます。

## 追加モジュールをインポートする [#import-procedure]

サポートされているモジュールをインポートするには、そのモジュールの標準的なインポート手順に従ってください。例えば、以下のようになります。

* **crypto-js** をインポートするには、電話してください。

  ```
  var crypto = require('crypto-js');
  ```

* **チャイ** を輸入するには、電話をしてください。

  ```
  var assert = require('chai').assert;
  ```

## 対応するサードパーティモジュール

シンセティック・スクリプト・モニターのバージョンによって、サポートされるサードパーティ・モジュールが異なります。異なるモニターバージョンでサポートされているモジュールの情報は、 [モニターバージョン比較表](/docs/synthetics/new-relic-synthetics/scripting-monitors/scripted-monitor-runtime-environment) を参照してください。

## 対応するNode.jsコアモジュール [#supported-core-modules]

New Relic のシンセティックモニタリングは、以下のオプションの Node.js コアモジュールにも対応しています。

* [アサート](http://nodejs.org/api/assert.html)
* [クリプト](http://nodejs.org/api/crypto.html)
* [dns](https://nodejs.org/api/dns.html)
* [fs](https://nodejs.org/api/fs.html)
* [ネット](https://nodejs.org/api/net.html)
* [querystring](http://nodejs.org/api/querystring.html)
* [タイマー](http://nodejs.org/api/timers.html)
* [tls](https://nodejs.org/api/tls.html)
* [url](http://nodejs.org/api/url.html)
* [ユーティル](http://nodejs.org/api/util.html)
* [zlib](https://nodejs.org/api/zlib.html)

## サポートされていないモジュール

[コンテナ型プライベートミニオン](/docs/synthetics/new-relic-synthetics/private-locations/install-containerized-private-minions-cpms) の中では、どのようなノードモジュールでも使用できます。コンテナ化されたプライベート・ミニオンにカスタム・モジュールを設定する方法については、 [CPM configuration](/docs/synthetics/new-relic-synthetics/private-locations/containerized-private-minion-cpm-configuration#custom-modules) を参照してください。

サポートされていないモジュールは、動作が保証されておらず、サポートの対象外となります。
