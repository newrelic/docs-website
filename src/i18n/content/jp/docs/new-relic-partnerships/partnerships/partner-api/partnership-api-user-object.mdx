## 要件 [#requirements]

**API を使用する前に、 [Partnership API requirements](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) をお読みください。**

## ユーザーオブジェクトの属性 [#user-attributes]

ユーザーオブジェクトは、お客様の情報を保存します。1つのアカウントに1人以上のユーザーを関連付けることができます。また、特定のユーザーが複数のアカウントに関連付けられることもあります。

これは、ユーザーオブジェクトの属性のリストです。New Relic アカウントの作成時に、 [**owner**](#attr-owner) 属性を 1 人のユーザーに割り当てていることを確認してください。

<Callout variant="important">
  New Relic では、1 ユーザーあたり最大 1,000 アカウントまでを推奨しています。アカウント数が増えると、New Relic の一部の機能へのアクセスが制限される場合があります。
</Callout>

## ユーザーコール

ここでは、ユーザー関連のAPI機能のURLパターンをご紹介します。このURLパターンを使用する場合は、JSONリクエストボディとともに、 [Partner API Key](/docs/new-relic-partnerships/partnerships/partner-api/partner-api-reference#the-api-key) を含むHTTPヘッダを送信してください。例えば、以下のようになります。

```
GET .../api/v2/partners/<var>PARTNER_ID</var>/accounts/<var>ID</var>
x-api-key:<var>PARTNER_ACCOUNT_KEY</var>
Content-Type: application/json

{ JSON data }
```

## ユーザーオブジェクトAPIの例 [#api-examples-v2]

以下は、Partnership API ユーザーオブジェクトを使用した API コールの例です。

<CollapserGroup>
  <Collapser
    id="example-list"
    title="リスト"
  >
    リストリクエスト

    ```
    curl -X GET \
        -H 'x-api-key:<var>PARTNER_ACCOUNT_KEY</var>' \
        -H 'Content-Type:application/json' \
        https://rpm.newrelic.com/api/v2/partners/<var>PARTNER_ID</var>/accounts/<var>12345</var>/users
    ```

    レスポンス

    ```
    {"users":[{"id":12345,"email":"xx@newrelic.com","role":"admin","owner":1}]}
    ```

    1,000人以上のユーザーがいるアカウントでは、ページ分割されたレスポンスが返されます。

    ページを指定するには、クエリに `?page=` というパラメータを追加します。
  </Collapser>

  <Collapser
    id="example-create"
    title="クリエイト"
  >
    <Callout variant="important">
      1つのPOSTコマンドにつき、新しい `users` オブジェクトは1つしか作成できません。
    </Callout>

    リクエストを作成します。

    ```
    curl -X POST \
        -H 'x-api-key:<var>PARTNER_ACCOUNT_KEY</var>' \
        -H 'Content-Type:application/json' \
        -d '{"users":[{"email": "foobar@newrelic.com", "password":"password1", "first_name":"John", "last_name":"Doe", "role":"admin" }]}' \
        https://rpm.newrelic.com/api/v2/partners/<var>PARTNER_ID</var>/accounts/<var>12345</var>/users
    ```

    応答（改行は読みやすさのため）。

    ```
    {"users":[{"id":12345,"email":"xx@newrelic.com","role":"admin","owner":1},
      {"id":23456,"email":"foobar@newrelic.com","role":"admin","owner":0}]}
    ```
  </Collapser>

  <Collapser
    id="example-update"
    title="アップデート"
  >
    アップデートの依頼。

    ```
    curl -X PUT \
        -H 'x-api-key:<var>PARTNER_ACCOUNT_KEY</var>' \
        -H 'Content-Type:application/json' \
        -d '{"users":[{"role": "restricted"}]}' \
        https://rpm.newrelic.com/api/v2/partners/<var>PARTNER_ID</var>/accounts/<var>12345</var>/users/<var>23456</var>
    ```

    レスポンス（読みやすさのために改行しています）

    ```
    {"users":[{"id":12345,"email":"xx@newrelic.com","role":"admin","owner":1},
      {"id":23456,"email":"foobar@newrelic.com","role":"restricted","owner":0}]}
    ```
  </Collapser>

  <Collapser
    id="example-delete"
    title="削除"
  >
    リクエスト

    ```
    curl -X DELETE \
        -H 'x-api-key:<var>PARTNER_ACCOUNT_KEY</var>' \
        -H 'Content-Type:application/json' \
        https://rpm.newrelic.com/api/v2/partners/<var>PARTNER_ID</var>/accounts/<var>12345</var>/users/<var>23456</var>
    ```

    応答します。レスポンスボディはありません。
  </Collapser>
</CollapserGroup>
