[パートナーシップ API](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) キーは、New Relic のパートナーシップ アカウントを持つコールを認証するために使用されます。API キーを扱う上で重要なのは、アカウントとパートナーシップオーナーの違いです。パートナーシップオーナーは、アカウントの特別な形態であり、アカウントに関するデータを管理するために使用されます。

## 要件 [#requirements]

**パートナーシップAPIは、New Relicの担当者から使用するように特に指示されていない限り、組織では利用できません。**

要件については、 [Intro to Partnership API](/docs/new-relic-partnerships/partner-integration-guide/partner-account-maintenance/partner-api/) をご参照ください。

## ライセンスキー

これは、アカウント作成時に XML で返されるキーです。これは、エージェントがアカウントを識別するために使用されます。

## パートナーAPIキー [#partner-key]

パートナーシップ API を使用する場合は、パートナーシップ所有者アカウントの REST API キーを使用する必要があります。これは、 `x-api-key: <api key>`が表示される API 呼び出しで使用されるキーです。

## お客様のAPIキーを探す [#locating-keys]

キーを表示または再生成するには、パートナーシップオーナーの認証情報を使用して New Relic サイトにログインし、 **Partnerships> Edit settings** にアクセスしてください。キーはページの下部に表示されます。

<Callout variant="caution">
  再生されたキーは元に戻すことはできません。
</Callout>

## セキュリティ・ベスト・プラクティス [#security]

**推奨：** New Relic サイトへのアクセスに使用するキーは、セキュリティのベストプラクティスとして、定期的に再生成する必要があります。キーの再生成からの変更はすぐに行われます。

キーをコードにハードコーディングし、新しいキーを含む新しい統合バージョンを本番環境にプッシュしたい場合は、キーの再生成とサイトの更新の間に必ず調整を行ってください。これにより、ダウンタイムを最小限に抑えることができます。このような状況では、Show Account APIを使用してこれらのキーを動的に取得することを検討してください。
