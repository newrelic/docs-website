## ヘッダーとフッター [#headers-footers]

ヘッダーやフッターを実装するには

1. &lt;div> タグで囲まれたコンテンツで構成されたヘッダーおよびフッターページを作成してください。ページにはCSSを取り込んだり、インラインCSSが含まれないようにしてください。

2. ヘッダーとフッターに必要な CSS を取得し、別の CSS ファイルで使用できるようにします。たとえば、 `yourcompany_`などのカスタム文字列をさまざまなスタイル名の先頭に追加して、New Relic の標準スタイル定義を上書きしないようにします。

3. 3つのファイルを、パートナーがホストする一般にアクセス可能な場所に投稿します。お客様に混合コンテンツの警告が表示されないように、httpsを使用してください。例えば、以下のようになります。

   ```
   https://yourcompany.com/newrelic_header.htm
   https://yourcompany.com/newrelic_footer.htm
   https://yourcompany.com/newrelic_styles.css
   ```

4. ヘッダー、フッター、CSS ページの場所の URL を New Relic <DNT>**Partnership**</DNT>コンソールの適切なフィールドに入力します。

<Callout variant="tip">
  ヘッダーを表示するには、パートナーシップのいずれかのアカウントにログインします。
</Callout>

## レイアウトとスタイル [#styles]

ヘッダーとフッターのレイアウトとスタイルは、New Relic のコレクターサーバーに読み込まれてキャッシュされ、各ページ出力に挿入されます。パートナーのサイトのレイアウトに加えられた更新は、1時間ごとに自動的にリフレッシュされます。

パートナーのウェブサイトへのカスタマイズされたリンクをレイアウトに追加することができます。シングルサインオンが有効な場合、パートナーは顧客固有の情報を New Relic ユーザーセッションに注入し、レイアウトで使用することができます。詳細については、 [シングルサインオンとアクセスコントロール](/docs/accounts-partnerships/partnerships/partner-integration-guide/single-sign-access-control) を参照してください。

## サブドメイン

パートナーは、オプションで、New Relic アカウントにアクセスする顧客に対してカスタム サブドメイン (例: <DNT>**yourcompany.newrelic.com**</DNT> ) を表示できます。 これを手配するには、New Relic のビジネス イネーブルメント チームにお問い合わせください。
