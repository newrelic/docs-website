アカウントで初めてNew Relic Kubernetesインテグレーションをデプロイすると、[アラート条件](/docs/alerts/new-relic-alerts/defining-conditions/define-alert-conditions)のデフォルト セットがアカウントにデプロイされます。 事前定義されたアラートポリシー（ <DNT>**Kubernetes default alert policy**</DNT>には、不要な通知を防ぐための[通知チャネルが](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-control-where-send-alerts)デフォルトでありません。

アラート条件のしきい値をカスタマイズしたり、通知を送信するようにポリシーを更新したりできます。詳細については、 [「インフラストラクチャ アラートのドキュメント」を](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information)参照してください。
