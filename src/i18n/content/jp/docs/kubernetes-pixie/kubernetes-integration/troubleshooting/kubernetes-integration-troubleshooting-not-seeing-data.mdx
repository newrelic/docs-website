## 問題

New Relic の Kubernetes 統合のための [のインストール手順](/docs/kubernetes-monitoring-integration#install) を完了しましたが、New Relic のアカウントに Kubernetes のデータが表示されません。

## 解決

インフラストラクチャエージェント [を展開し、 ](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent/#install)[Kubernetes のインストール手順](/docs/kubernetes-monitoring-integration#install) を完了したにもかかわらず、Kubernetes ダッシュボードに何の情報も表示されない場合は、以下の手順に従ってください。

1. `CLUSTER_NAME`と`NRI_LICENSE_KEY`が正しく構成されていることを確認してください。詳細については、 [インストール手順](/docs/integrations/host-integrations/host-integrations-list/kubernetes-monitoring-integration#install)を参照してください。

2. 次のコマンドを実行して、統合の展開が成功したことを確認します。

   ```
   kubectl get pods -o wide -n <NEWRELIC_NAMESPACE>
   kubectl get deployments -o wide -n <NEWRELIC_NAMESPACE>
   kubectl get daemonsets -o wide -n <NEWRELIC_NAMESPACE>
   ```

3. [インフラストラクチャエージェントと統合からログを取得し、](/docs/integrations/kubernetes-integration/troubleshooting/get-logs-version) [エラーメッセージ](/docs/integrations/host-integrations/troubleshooting/kubernetes-integration-troubleshooting-error-messages)がないことを確認します。
