Kubernetesは、コンテナ化されたアプリケーションのデプロイ、スケーリング、管理を自動化するためのオープンソース ツールです。 New Relic Kubernetes監視インテグレーションを使用すると、オンプレミスでホストされているかクラウドでホストされているかに関係なく、 Kubernetesクラスタとワークロードで何が起こっているかをすばやく簡単に確認できます。

[KubernetesKubernetes インテグレーション](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure/) し たら、Kubernetes のすぐに使用できるエクスペリエンスを活用して、 上で実行されているすべてのサービスをKubernetes 1[Kubernetes ](/docs/integrations/kubernetes-integration/cluster-explorer/kubernetes-cluster-explorer)か所で監視できるようになります。コントロール プレーンの最上部から単一のポッドで実行されているアプリケーションに至るまですべてが表示されるため、クラスターの動作の核心に迫ることができます。

## New Relic による Kubernetes の監視 [#monitoring-k8s-nr]

Kubernetes の管理は難しい場合があります。 コンテナが数分で作成および削除されたり、アプリがクラッシュしたり、リソースが予期しない方法で使用されるなど、常に多くのことが起きています。 当社のインテグレーションは、オンプレミス、クラウド、ハイブリッド設定のいずれで使用する場合でも、 Kubernetesさまざまな方法で使用する方法を理解するのに役立ちます。

New Relic で何ができるか見てみましょう:

* [Kubernetesコントロール プレーンを監視して](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/configure-control-plane-monitoring)、クラスタからメトリックを管理および収集します。
* [Kubernetesログを](/docs/logs/forward-logs/kubernetes-plugin-log-forwarding/)New Relicに転送することで、ログ データを収集、処理、調査、クエリ、集計することができ、ログ データを管理する方法がさらに増えます。
* 独自のチャートを作成し、[インテグレーション](/docs/using-new-relic/data/understand-data/query-new-relic-data)が Container オーケストレーション レイヤーによって収集したすべての[Kubernetesデータ](/docs/integrations/kubernetes-integration/understand-use-data/find-use-your-kubernetes-data)を照会します。 これにより、ノード、ネームスペース、ベンダー、レプリカ セット、ポッド、およびコンテナへのインサイトがさらに広がります。
* [APM データを Kubernetes にリンクして、](/docs/integrations/kubernetes-integration/link-your-applications/link-your-applications-kubernetes) Web アプリとモバイル アプリのパフォーマンスを測定します。 リクエスト レート、終了率、エラー率、可用性などを追跡できます。
* Apache、NGINX、Cassandra など、 [Kubernetes上で実行されるモニター サービス](/docs/integrations/kubernetes-integration/link-apps-services/monitor-services-running-kubernetes)。 [Kubernetes 上の Redis を監視するためのチュートリアル](/docs/integrations/kubernetes-integration/link-apps-services/tutorial-monitor-redis-running-kubernetes)をご覧ください。
* [Prometheus](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic/#Agent)エージェントが提供するサービス検出機能を利用して、クラスタ内の任意のワークロードから Prometheus メトリックを取得します。
* データに基づいて [アラートポリシー](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/) と[ アラート条件](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/alert-conditions/#create-alert-condition) Kubernetes[を作成するか、 定義済みのアラート条件](/docs/integrations/kubernetes-integration/kubernetes-events/kubernetes-integration-predefined-alert-policy) を拡張します。
* すべての Kubernetes イベントを確認します。 [Kubernetesインテグレーション](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration)は、 Kubernetesクラスタで発生するイベントを監視し、それらのイベントをNew Relicに送信します。 その後、クラスター エクスプローラーでイベント データを視覚化できます。 [New Relic Kubernetesインテグレーションをインストールする](/docs/kubernetes-pixie/kubernetes-integration/installation/kubernetes-integration-install-configure/)とデフォルトでインストールされます。
* データを Kubernetes にレポートし [てポッドの自動スケーリングをトリガーする](/docs/journey-demand/autoscale-your-infra) ことで、需要のピーク時などの需要の急増時に常にリソースを利用できるようになります。

これらの機能は、New Relic がすでに報告している、インスツルメンテッド・ホスト上で実行されている [コンテナ化されたプロセスのデータに加えて、](/docs/infrastructure/new-relic-infrastructure/data-instrumentation/docker-instrumentation-infrastructure) 。

## ソースコードを確認してください [#source-code]

このインテグレーションはオープンソースソフトウェアです。 つまり、[ソースコードを参照して](https://github.com/newrelic/nri-kubernetes)独自の改良点を送信したり、独自のフォークを作成してビルドしたりすることができます。 詳細については、 [README](https://github.com/newrelic/nri-kubernetes/blob/main/README.md)ファイルを参照してください。
