OpenTelemetryを使用してKubernetesクラスタを監視できます。これは、テレメトリーデータを収集してNew Relicに送信する標準化された方法を提供します。 この統合により、メトリクス、イベント、ログなどのインサイトをKubernetes環境に取り込むことができます。

New Relicは[、Kubernetes向けにNew Relic版OpenTelemetry（NRDOT）を](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s)使用することを推奨しています。このディストリビューションはNew Relicとシームレスに連携するように事前設定されており、 Kubernetesクラスタを効果的に監視するために必要なすべてのコンポーネントと設定が含まれています。 ただし、独自の OpenTelemetry Collector を使用したい場合は、Kubernetes の監視に必要なコンポーネントが含まれていることを確認することで使用できます。

## 要件 [#requirements]

KubernetesテレメトリーデータをNew Relicに送信するには、 OpenTelemetry Collectorが必要です。 当社の[NRDOT](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s)は、 Kubernetesクラスタを自動的に監視するように構成されています。 必要なコンポーネントはすべて、当社の[`nr-k8s-otel-collector`](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector) Helmチャートを通じてデプロイされます。

別のOpenTelemetry Collector選択する場合は、包括的なKubernetes監視に必要なすべての主要コンポーネントが含まれていることを確認してください。

### サポートされている環境 [#supported-envs]

Kubernetes用のOpenTelemetry監視は、さまざまなデプロイメント環境にわたって堅牢なサポートを提供します。 対応プラットフォームは以下のとおりです。

* **クラウドベンダー：**

  * Amazon EKS
  * Microsoft AKS
  * Google GKE
  * Oracle OKE
  * Red Hat OpenShift

* **オンプレミス・クラスター：**オンプレミスのKubernetesクラスターのサポートを提供しています。

* **Kubernetesのバージョン：**サポートは各ベンダーが現在サポートしているKubernetesのバージョンに準拠しており、これらの環境全体で互換性と効果的な監視ソリューションを保証します。

<Callout variant="important">
  Windowsノードはサポートされていません。
</Callout>

## OpenTelemetryを使用してKubernetesクラスターをインストールする [#install]

OpenTelemetry for Kubernetesは、以下のいずれかの方法でインストールできます。

<CollapserGroup>
  <Collapser id="helm-install" title="Helm インストール">
    ## Helm インストール [#helm-install]

    Helmメソッドは、 Kubernetes上のデプロイOpenTelemetryに推奨されるアプローチです。 この方法により、デプロイメントプロセスが簡素化され、設定管理が容易になります。 NRDOTまたは独自のOpenTelemetry Collectorのどちらでも使用できます。

    New Relicは、New Relicとシームレスに連携するように事前設定されたNRDOT for Kubernetesの使用を推奨しています。このディストリビューションには、 Kubernetesクラスタを効果的に監視するために必要なすべてのコンポーネントと設定が含まれています。 ただし、独自の OpenTelemetry Collector を使用したい場合は、Kubernetes の監視に必要なコンポーネントが含まれていることを確認することで使用できます。

    **Helmチャートをインストールするには：**

    1. [Helmチャートファイル`values.yaml`](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector/values.yaml#L20-L24)をダウンロードし、ご自身の要件に合わせて変更してください。

       * クラスター名と<InlinePopover type="licenseKey" />は必須です。

       * 詳細については、 [「設定の問題」](#config-params)を参照してください。

    2. `values.yaml`ファイルを使用して[Helm チャート](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector)をインストールします。

       ```shell
       helm repo add newrelic https://helm-charts.newrelic.com
       helm upgrade nr-k8s-otel-collector newrelic/nr-k8s-otel-collector -f your-custom-values.yaml -n newrelic --create-namespace --install
       ```

    3. ポッドが正常に起動したことを確認してください。

       ```shell
           kubectl get pods -n newrelic --watch
       ```

       `newrelic`ネームスペースに、 `nr-k8s-otel-collector-<hash>`などの名前のポッドが表示されるはずです。

    4. 次のクエリを実行して、New Relicが必要なデータ（メトリクス、イベント、ログなど）を受信していることを確認してください。詳細については、 [「クエリビルダーの概要」](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder/)を参照してください。

       ```sql
       FROM Metric SELECT * WHERE k8s.cluster.name='<CLUSTER_NAME>'
       FROM OtlpInfrastructureEvent SELECT * WHERE k8s.cluster.name='<CLUSTER_NAME>'
       FROM Log SELECT * WHERE k8s.cluster.name='<CLUSTER_NAME>'
       ```

    <Callout variant="tip">
      別のOpenTelemetry Collector使用してKubernetesテレメトリーデータを収集する場合は、 `values.yaml`ファイルの`image`設定を更新して、目的のOpenTelemetry Collector指定してください。

      ```yaml
      image:
          repository: otel/opentelemetry-collector-contrib # Example: Using the contrib distro
          tag: "latest" # Or a specific stable version like "0.98.0"
      ```
    </Callout>
  </Collapser>

  <Collapser id="manifest-install" title="マニフェストのインストール">
    Manifest インストレーション方式は、 Helmを使用せずにKubernetes用のOpenTelemetry設定する方法を提供します。 この方法は、より実践的なアプローチを好むユーザーや、手動設定を必要とする特定の要件を持つユーザーに適しています。

    **マニフェストのインストールを実行するには:**

    1. `nr-k8s-otel-collector`の[レンダリングされたサンプルディレクトリ](https://github.com/newrelic/helm-charts/tree/master/charts/nr-k8s-otel-collector/examples/k8s/rendered)の内容をローカルワークスペースにコピーしてください。このディレクトリにはKubernetesのマニフェストファイルが含まれています。

    2. ローカルの`rendered`ディレクトリにある[`secret.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/secret.yaml)ファイルを更新してください。`<Your Base64 encoded License key>` Base64 でエンコードされた New Relic ライセンスキーに置き換えてください。

       ```yaml
       data:
           licenseKey: <Your Base64 encoded License key>
       ```

       ライセンスキーを`Base64`するには：

       * **Linux/macOS:** `echo -n "YOUR_LICENSE_KEY" | base64`を使用してください。
       * **Windows (PowerShell):** `[System.Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes("YOUR_LICENSE_KEY"))`を使用します。

    3. ローカルディレクトリ`rendered`内にある[`daemonset-configmap.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/daemonset-configmap.yaml)ファイルと[`deployment-configmap.yaml`](https://github.com/newrelic/helm-charts/blob/master/charts/nr-k8s-otel-collector/examples/k8s/rendered/deployment-configmap.yaml)ファイルの両方で、クラスター名を手動で更新してください。`k8s.cluster.name`のインスタンスを探し、 `<cluster_name>`クラスターに付けたい名前に置き換えてください。

       ```yaml
       - key: k8s.cluster.name
           action: upsert
           value: <cluster_name>
       ```

    4. これらの必須フィールドを更新した後、 `newrelic`ネームスペースを作成し、 `kubectl`を使用してマニフェストをクラスタにデプロイします。

       ```bash
       kubectl create namespace newrelic
       kubectl apply -n newrelic -R -f rendered
       ```

    <Callout variant="tip">
      Helm使用せずにデプロイする場合、レンダリングされたマニフェストを介したコンポーネントのデプロイには、 `example-`などのプレフィックスが含まれる場合があります。 この接頭辞は通常、Helm チャート テンプレートで Helm リリース名に基づいて動的な名前を付けるために使用されます。より分かりやすい命名規則をご希望の場合は、適用前にマニフェストファイル内でこれらの接頭辞を直接調整できます。
    </Callout>
  </Collapser>
</CollapserGroup>

## アラート [#alerts]

<DNT>Integrations &amp; Agents</DNT>ガイド付きインストール フローに従うことで、重要なアラート ポリシーをインストールできます。 これにより、 Kubernetesオブザーバビリティ用に設計された複数のアラート条件を使用して、 New Relicアカウントに**Kubernetes (OpenTelemetry) アラート ポリシーという名前のアラート ポリシー**が自動的に設定されます。 これらの条件は、お客様の特定の監視ニーズに合わせてカスタマイズしてください。

## アンインストール [#uninstall]

### Helm のアンインストール [#helm-uninstall]

Kubernetesを使用して クラスタの監視を停止するには、次のコマンドを実行します。OpenTelemetry

```shell
helm uninstall nr-k8s-otel-collector -n newrelic
```

### マニフェストのアンインストール [#manifests-uninstall]

Kubernetesを使用して クラスタの監視を停止するには、次のコマンドを実行します。OpenTelemetry

```shell
kubectl delete -R -f rendered
kubectl delete namespaces newrelic
```
