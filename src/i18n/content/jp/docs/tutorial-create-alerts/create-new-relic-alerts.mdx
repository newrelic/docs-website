スタックの問題を通知するアラートを設定することは、データを監視するためにできる最も重要なことの 1 つです。しかし、アラートの作成と管理は、多くの場合、面倒に感じられます。次のチュートリアルでは、New Relic でアラートを作成する 3 つの主な方法を説明します。また、これらのアラートを使用して情報を収集し、そのデータを使用してスタックを改善する方法も示します。アラートをまったく初めて使用する場合は、始める前に学習すべき基本的なアラートの概念と用語がいくつかあります。

## アラートとは何ですか? [#alerts-what]

アラートは、条件が満たされた場合に、トリガーとなるイベントに関する通知を送信する条件です。任意の数のチームの任意の数の人に通知を送信できます。アラートを適切に構成すると、問題を解決できるグループに直接通知を送信することで、より効率的に問題を解決できます。アラートには 5 つの主要な部分があります。

* **Incidents** \[インシデントは]、 条件のしきい値に違反したイベントです。インシデントの一般的な例としては、遅延が特定の制限を超えた場合、ページの読み込みに失敗した場合、または JavaScript エラーが発生した場合があります。
* **Issues** \[問題は]、 通知の送信を引き起こす 1 つ以上のインシデントの集合です。
* **Conditions** \[条件] は、インシデントを開くために設定する設定オブジェクトです。
* **Policies** \[ポリシー] は、 インシデントが発生したときに通知を受け取る方法を定義する条件のグループです。
* **Notifications** \[通知は]、 問題がオープン、承認、またはクローズされたときに受信するメッセージです。

<Callout variant="tip">
  **Incidents** \[インシデント] と **Issues** \[問題]の違いは何ですか?

  **インシデントは** 問題の症状を詳細に示す個別のイベントですが、 **問題は** システム内のより大きな問題を説明するインシデントのグループです。
</Callout>

## New Relic のアラートはどこにありますか? [#alerts-where]

New Relic では、ほとんどの場合、 [**one.newrelic.com > Alerts & AI**](https://one.newrelic.com/alerts-ai)からアラートを管理します。プラットフォームのこのセクションには、すべてのアラートの内訳を視覚的に表示する **Overview** \[概要] ページと、すべてのアラート ポリシーのリストと新しいアラート ポリシーを作成する場所を提供する **Alert conditions (policies)** \[アラート条件 (ポリシー)] ページがあります。
