アプリケーションがOpenTelemetry（OTel）の生成AIのトレースをNew Relicに送信するようになると、AIモニタリングビューには次のように表示されます：

* サービスによって行われるすべてのLLM呼び出し — プロンプト、完了、モデル、レイテンシ、トークン数、および終了理由。
* 完全なリクエストトレースの一部としてのツール呼び出しとエージェントステップ。これにより、モデルが決定を下した理由を確認できます。
* エラー、レート制限の失敗、および遅い呼び出しが、その他のAPMデータとともに表示されます。

New Relicエージェントは必要ありません。アプリがすでにOpenTelemetryトレースを出力している場合、準備はほぼ完了しています。

## 始める前に

必要なもの：

* AIモニタリングが有効になっているNew Relicアカウント。有効になっているかどうかわからない場合は、New Relicアカウントチームにお問い合わせください。
* New Relic取り込みライセンスキー。<DNT>**Account settings &gt; API keys &gt; Ingest - License**</DNT>に移動します。
* LLMを呼び出すアプリケーション。OpenAI、Anthropic、LangChain、およびAWS BedrockのPythonの例を以下に示します。OpenTelemetry生成AI計装を備えた他の言語も同じように機能します。
* アプリからポート443上の`otlp.nr-data.net`（米国）または`otlp.eu01.nr-data.net`（EU）へのネットワークエグレス。

<Callout variant="important">
  OTelの生成AIの計装は、ユーザーが送受信する正確なプロンプトとコンプリーションをキャプチャできます。これには、PII、認証情報、または規制対象データが含まれる場合があります。コンテンツのキャプチャはデフォルトでオフになっています。オンにする前に、組織のデータ処理ポリシーを確認し、[送信されるプロンプトとコンプリーションの制御](#controlling-content)を参照してください。
</Callout>

<Steps>
  <Step>
    ### OpenTelemetryエクスポーターをNew Relicに向ける [#step-1]

    アプリケーションを実行しているプロセスで、これらの環境変数を設定します：

    ```sh
    # US datacenter. For EU use https://otlp.eu01.nr-data.net:443
    export OTEL_EXPORTER_OTLP_ENDPOINT="https://otlp.nr-data.net:443"
    export OTEL_EXPORTER_OTLP_HEADERS="api-key=$NEW_RELIC_LICENSE_KEY"

    # Name that will identify this service in New Relic
    export OTEL_SERVICE_NAME="my-genai-service"

    # Allow long prompts and completions through without truncation
    export OTEL_ATTRIBUTE_VALUE_LENGTH_LIMIT=4095
    export OTEL_SPAN_ATTRIBUTE_VALUE_LENGTH_LIMIT=4095
    ```

    New Relic OTLPエンドポイント、サポートされているポート、および取り込み制限の現在のリストについては、[New Relic OpenTelemetryドキュメント](/docs/opentelemetry/best-practices/opentelemetry-best-practices-overview)を参照してください。
  </Step>

  <Step>
    ### 適切な計装パッケージを追加する [#step-2]

    アプリケーションが使用するものに一致するセクションを選択します。複数組み合わせることができます — たとえば、Bedrockを呼び出すLangChainアプリは両方をインストールします。

    <Callout variant="tip">
      OpenTelemetry生成AIエコシステムでは、正確なパッケージ名とバージョンが急速に進化しています。チームが現在サポートしているパッケージについては、常にOpenTelemetry Python contribレジストリとNew Relic AIモニタリングのドキュメントを確認してください。
    </Callout>

    <CollapserGroup>
      <Collapser id="openai" title="オープンAI">
        OpenTelemetryプロジェクトから公式のOpenAIの生成AIの計装をインストールし、`opentelemetry-instrument`を使用してアプリを実行するか、インストルメンターを明示的に呼び出します：

        ```python
        from opentelemetry.instrumentation.openai_v2 import OpenAIInstrumentor
        OpenAIInstrumentor().instrument()

        from openai import OpenAI
        OpenAI().chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": "Hello"}],
        )
        ```
      </Collapser>

      <Collapser id="anthropic" title="Anthropic">
        OpenTelemetry Anthropic計装をインストールします（OpenTelemetry contribレジストリや、OpenLLMetryなどのコミュニティディストリビューションから入手可能です）。次に、これを有効にします：

        ```python
        from opentelemetry.instrumentation.anthropic import AnthropicInstrumentor
        AnthropicInstrumentor().instrument()

        import anthropic
        anthropic.Anthropic().messages.create(
            model="claude-sonnet-4-5",
            max_tokens=1024,
            messages=[{"role": "user", "content": "Hello"}],
        )
        ```
      </Collapser>

      <Collapser id="langchain" title="LangChain">
        LangChainの計装は、チェーン、すべてのツール呼び出し、および基盤となるモデルクライアントを1つのパッケージでカバーします。任意のコミュニティ版LangChainの計装をインストールし、起動時に有効にします：

        ```python
        from opentelemetry.instrumentation.langchain import LangchainInstrumentor
        LangchainInstrumentor().instrument()
        ```

        既存の`ChatOpenAI`、`ChatAnthropic`、または類似のチェーンは自動的にトレースされます。ツールとエージェントのステップは、親チェーン呼び出しの下に子スパンとして表示されます。
      </Collapser>

      <Collapser id="aws-bedrock" title="AWS Bedrock">
        BedrockのトラフィックはAWS SDKを経由します。AWS SDK（botocore）OpenTelemetry計装をインストールして有効にします：

        ```python
        from opentelemetry.instrumentation.botocore import BotocoreInstrumentor
        BotocoreInstrumentor().instrument()

        import boto3, json
        boto3.client("bedrock-runtime", region_name="us-east-1").invoke_model(
            modelId="anthropic.claude-3-5-sonnet-20241022-v2:0",
            body=json.dumps({
                "anthropic_version": "bedrock-2023-05-31",
                "max_tokens": 256,
                "messages": [{"role": "user", "content": "Hello"}],
            }),
        )
        ```

        インストールしたバージョンがBedrock呼び出しに対して生成AIの属性（`gen_ai.system`、`gen_ai.request.model`、およびトークン使用量）を出力することを確認します — サポートはバージョン全体で段階的に追加されています。
      </Collapser>
    </CollapserGroup>
  </Step>

  <Step>
    ### データが到着していることを確認する [#step-3]

    1. 新しい環境変数と計装が有効になるように、アプリケーションを再起動します。

    2. 少量の実際のトラフィックをトリガーします — ほんの少しのチャットコンプリーションで十分です。

    3. New Relicで<DNT>**Query Your Data**</DNT>を開き、次を実行します：

       ```sql
       SELECT count(*) FROM Span
       WHERE gen_ai.system IS NOT NULL OR gen_ai.provider.name IS NOT NULL
       SINCE 30 minutes ago
       FACET service.name, gen_ai.operation.name
       ```

    4. 1～2分以内に、サービス名と操作（`chat`、`completion`、`embeddings`）でファセットされた行が表示されるはずです。

    5. <DNT>**APM &amp; Services &gt; AI monitoring**</DNT>に移動し、サービスを選択して生成AIビューが表示されるのを確認します。
  </Step>
</Steps>

## 送信されるプロンプトとコンプリーションを制御する [#controlling-content]

デフォルトでは、ほとんどのOpenTelemetry生成AI計装は、プロンプトや補完のテキストをキャプチャしません。スパンにはモデル、トークン数、レイテンシ、およびステータスが含まれますが、メッセージ本文は含まれません。

AIモニタリングでメッセージコンテンツを表示するには、明示的にオプトインします。フラグ名はインストールした計装パッケージによって異なります — その特定のパッケージのドキュメントを確認してください。一般的な例としては次のものがあります：

* `OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT=true` OpenTelemetryが保守する計装の場合。
* `TRACELOOP_TRACE_CONTENT=true` コミュニティ（OpenLLMetryファミリ）計装用。

本番環境でコンテンツキャプチャを有効にする前に、以下の手順を実行してください：

* プロンプトに含まれるデータを確認します。ユーザーがPII、認証情報、または規制対象データを送信した場合、そのコンテンツはNew Relicに保存されます。
* 特定のフィールドが環境から離れる前に、それらを取り除くかハッシュ化するために、ドロップフィルターまたは属性レベルの難読化を検討してください。[New Relicのデータ処理ドキュメント](/docs/ai-monitoring/drop-sensitive-data)を参照してください。
* 組織の保持およびコンプライアンス要件（GDPR、HIPAA、PCIなど）を確認してください。

メッセージ本文ではなく — レイテンシ、トークン、およびエラーの可視性のみが必要な場合は — コンテンツのキャプチャを無効のままにしてください。

## ヘルプを受ける

上記の手順を実行しても解決しない場合：

* New Relicサポートケースを提出し、サンプルスパンのペイロードとサービス名を含めてください。
* アカウントレベルの設定に関する質問については、New Relicアカウントチームにお問い合わせください。
* 一般的なOpenTelemetry生成AIに関する質問については、信頼できるリファレンスである[OpenTelemetry生成AIセマンティック規約](https://opentelemetry.io/docs/specs/semconv/gen-ai/)を参照してください。
