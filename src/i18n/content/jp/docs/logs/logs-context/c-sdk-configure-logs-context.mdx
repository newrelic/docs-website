Logs in contextは、C SDKがモニターするアプリでは利用できません。代わりに、 [Log API](/docs/logs/log-management/log-api/introduction-log-api/) を使用して、独自のログ拡張ライブラリを構築することができます。アプリで分散型トレーシングが有効になっていることを確認してください。そうすれば、リクエストがアップストリームとダウンストリームのサービスを通過するときに何が起こるかをすばやく理解することができます。

C SDKでモニターするアプリの分散トレースを有効にするには、 [](/docs/distributed-tracing/enable-configure/quick-start/)をインストールするか、 [](/docs/agents/c-sdk/install-configure/update-your-c-sdk-library/)を最新のC SDKバージョンに更新します。分散型トレーシングには、 [C SDKバージョン1.1.0以上が必要です。](/docs/release-notes/agent-release-notes/c-sdk-release-notes/).
