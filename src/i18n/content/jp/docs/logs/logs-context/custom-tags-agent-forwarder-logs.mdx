New Relic APM言語エージェントでは、カスタム タグ (ラベル) をエージェント転送ログに追加することをオプトインできるようになりました。 ログのカスタム タグを使用すると、 DevOpsおよびプラットフォーム エンジニアはログ データのフィルタリング、検索、関連付けを行うことができ、より高速かつ効率的なトラブルシューティング、パフォーマンスの向上、リソース使用率の最適化を実現できます。

## アプリケーションログでカスタムタグ（ラベル）を有効にする

有効にすると、 New Relic言語エージェントはすべてのカスタム タグ (ラベル) をエージェント転送ログに追加します。

サポートされている言語:

* [.NET](/docs/apm/agents/net-agent/configuration/net-agent-configuration)エージェント バージョン 10.4.0 以上。
* [Ruby](/docs.newrelic.com/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/)エージェント バージョン 9.16.0 以上。
* Go: 近日公開予定
* Java: 近日公開予定
* [Node.js](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration) : エージェント バージョン 12.8.0 以上。
* [Python](/docs/apm/agents/python-agent/configuration/python-agent-configuration)エージェント バージョン 10.3.0 以上。
