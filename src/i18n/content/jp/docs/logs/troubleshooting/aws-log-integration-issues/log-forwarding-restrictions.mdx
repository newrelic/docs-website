New Relic には、S3 または Firehose 経由でログを取り込む場合、ログ転送のためのAWSインテグレーションに関する次の制限があります。

### S3 からAWSを取り込む場合 [#s3-restrictions]

* ログファイルのサイズは 80 MB 未満である必要があります。
* 1 行のログのサイズは 8 MB 未満である必要があります。
* 1 MB から 8 MB までのログ行は複数のメッセージに分割され、New Relic に転送されます。
* New Relic は現在、クライアント JSON またはログ行の解析をサポートしていません。
* New Relic は現在、ログファイルの圧縮形式として`Gzip`と`Bzip2`のみをサポートしています。 他の圧縮形式のファイルを非圧縮ファイルとして処理します。

### CloudWatchからのログの取り込み [#cloudwatch-restrictions]

* New Relic は現在、クライアント JSON またはログ行の解析をサポートしていません。
* New Relic は現在、解析ロジックに[Lambda レイヤー](https://github.com/newrelic/newrelic-lambda-layers)のみをサポートしています。
