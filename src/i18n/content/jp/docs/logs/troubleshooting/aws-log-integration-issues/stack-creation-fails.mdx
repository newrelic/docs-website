## 問題

CloudFormation スタックの作成は、さまざまな理由により失敗する可能性があります。

## 解決

考えられる原因とその解決策:

* CloudFormation スタックの作成中に選択した IAM ロールに十分な [権限](https://docs.aws.amazon.com/ARG/latest/userguide/gettingstarted-prereqs-permissions.html)がない可能性があります。

* IAM ロールを選択しなかった場合は、アカウントに必要な[権限](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#logs)があることを確認してください。

* 次のような失敗メッセージが表示された場合は、その名前の既存のリソースを削除するか名前を変更して、再試行してください。

  `Resource type {resource-name} with identifier {resource-identifier-name} already exists.`

* ネストされた CloudFormation スタックの出力をチェックして、潜在的なエラーを特定します。

* 選択した S3 バケットにオブジェクト作成イベントのイベント通知がないことを確認してください。

* 選択した CloudWatch ロググループに類似のサブスクリプションフィルターがないことを確認します。
