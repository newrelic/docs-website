**[one.newrelic.com](https://one.newrelic.com)** または当社の [EU地域のデータセンター](/docs/using-new-relic/welcome-new-relic/get-started/our-eu-us-region-data-centers/) （該当する場合） **[one.eu.newrelic.com/](https://one.eu.newrelic.com)** のLogs UIをご利用ください。

* ログの中に興味深いパターンや重要なパターンを発見する。
* 特定のログラインに関する詳細な情報を調べる。
* フィルタや解析ルールを使って、ログデータを調べたり操作したりすることができます。
* チャートを使ってデータを照会・共有したり、ダッシュボードに追加したりすることができます。
* アカウントのログデータを整理し、データパーティションでクエリのパフォーマンスを最適化します。
* 防止したい問題のアラート条件を設定します。

新機能や改善点の最新情報を得るために、ログのリリースノート [のRSSフィードを購読してください。](/docs/release-notes/logs-release-notes)

## [ログデータを見る [#ui-workflow]](/docs/release-notes/logs-release-notes)

[ログ UI の左のナビを使って、すべてのログ、属性、パターン、ライブテール ロギング、およびクエリの簡単なワークフローとして使用できます。データのドロップや解析、データ パーティションの作成、アラート条件の設定などにより、ログ データを管理できます。センターナビでは、特定のログやその属性の詳細を確認できます。](/docs/release-notes/logs-release-notes)

[![New Relic OneにおけるログのUI](./images/logs-ui-123021.png "New Relic OneにおけるログのUI")](/docs/release-notes/logs-release-notes)

[ロギングデータを調べるには、以下の基本的なワークフローに従ってください。](/docs/release-notes/logs-release-notes)

1. [](/docs/release-notes/logs-release-notes)

2. [](/docs/release-notes/logs-release-notes)

   [](/docs/release-notes/logs-release-notes)[New Relic One のナビゲーションバーをカスタマイズしていない場合](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/#custom-navigation) 、 **[one.newrelic.com](https://one.newrelic.com)** 、 **Browse data** をクリックし、 **Logs** を選択します。

3. **パターンを探す。**

   * ログメッセージの不審なスパイクやドロップを発見するには、左ナビの [パターン](/docs/logs/ui-data/find-unusual-logs-log-patterns/) アイコンをクリックします。
   * 特定の期間のログを見るには、チャート上でそのポイントをクリック（またはエリアをクリック＆ドラッグ）するか、タイムピッカーを使用します。

4. **焦点を絞る。**

   * 最初の検索結果の焦点を絞ったり、外れ値を素早く見つけたりするには、ログの詳細で任意の属性を展開して、結果内の最も一般的な10個の値を表示します。たとえば、 `ホスト名` 属性にリストされているホストが、他のホストよりも著しく多くのエラーメッセージを生成している場合、その値を選択して検索に適用します。
   * ログメッセージの照会と理解を容易にするために、 [組み込みの解析ルール](/docs/logs/ui-data/built-log-parsing-rules/) を使用するか、 [独自の解析ルール](/docs/logs/ui-data/parsing/) を作成して、特定の値に一致する属性を解析します。
   * 収集されるログデータの量を管理し、より少ないログを保存するために、必要のないデータを収集しないように、 [ドロップフィルタールール](/docs/logs/ui-data/drop-data-drop-filter-rules/) を作成します。

5. **ログの詳細を調べる。**

   ログメッセージを選択すると、その詳細が属性表またはJSONとして表示されます。

   * どの属性がログメッセージに含まれているかを確認するには、ログラインをクリックします。必要に応じて属性を追加したり削除したりすることで、クエリで必要な詳細情報を得ることができます。

   * ログの詳細にある特定の値に関連する問題のトラブルシューティングに役立つように、 [**Show surrounding logs** ](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs)

     <Icon name="fe-eye">[](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs)</Icon>

     [icon](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs) をクリックして、その属性の詳細を表示します。

   * 結果に表示される属性をコントロールするには、ログの `log_summary` カラムのハイライトされた値をクリックします。

   * 非常に長いメッセージの詳細を得るために、 [blob](/docs/logs/ui-data/long-logs-blobs/) として保存されたデータを展開します。

6. **検索してみてください。**

   デフォルトでは、Logs UIにはすべてのログが表示されますが、キーワードやフレーズで検索して、目的の結果を見つけることもできます。たとえば、 `process failed`.

   または

   検索フィールドから、タイプアヘッドのドロップダウンを使って、属性、演算子、値を選択します。

   ```
   service_name equals <var>my service</var>
   ```

   詳細については、 [logs query syntax documentation](/docs/logs/ui-data/query-syntax-logs/) を参照してください。

<Callout variant="tip">
  アカウント内のデータを整理し、クエリのパフォーマンスを最適化するために、 [データ・パーティション・ルール](/docs/logs/ui-data/data-partitions/) を作成します。
</Callout>

6. **関連するログを取得する。**

   たとえば、

   * デプロイメントやその他のアプリの変更に対するシステムの反応をすぐに確認するには、 [live-tail logging](/docs/logs/troubleshooting/view-log-messages-real-time-live-tail) を有効にします。

   * 特定の値のすべてのログを表示するには、選択したログの **ログの詳細** の属性リストを確認してください。

   * 問題が発生する前の根本的な原因やイベント後の影響を特定するために、 [**Show surrounding logs** ](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs)

     <Icon name="fe-eye">[](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs)</Icon>

     [icon](/docs/logs/troubleshooting/find-issues-cause-or-impact-surrounding-logs) をクリックします。

7. **発見を共有する**

   [New Relic One のコア機能である](/docs/new-relic-one/use-new-relic-one/get-started/new-relic-one-core-ui-components) (特定のアカウント、時間範囲、データエクスプローラー、クエリビルダーなど)を使用して、データをチャートで共有したり、ダッシュボードに追加したりします。詳細については、このドキュメントの [例](#examples) を参照してください。

## ビューの保存 [#saved-views]

ログのクエリ、テーブル構成、時間範囲、属性のグループ化を保存したビューに保存しておけば、後ですぐに戻ることができます。ログ分析ビューを構成した後に保存するには。

1. Logs UI の左ナビで **Saved Views** タブをクリックします。
2. クリック **Save current view**.
3. **Saved Views** タブに表示されるように、保存したビューに名前を付けます。
4. 現在のビューからどの局面を保存するかを選択します。
5. 権限レベルを選択します： **Private**, **Public** **(Read-only)**, **Public (Read and write)** 。Public は、そのアカウントにアクセスできるすべてのユーザーが保存されたビューを見ることができることを意味します。
6. **Save view** をクリックします。

ここでは、Logs UIを使って詳細な情報を得るための例をいくつかご紹介します。これらの例のいくつかを使用するには、 [ログをコンテキスト](/docs/enable-logs-context-logs) で見ることができる必要があります。

  >
    アラート条件はLogs UIで直接作成することができます。

    1. **[one.newrelic.com](https://one.newrelic.com) > Logs** にアクセスしてください。

    2. 警告したい結果を検索します。例えば、 `service_name:"your service" " fatal error"` .

    3. 左ナビの **Manage data** セクションから、 **Create alert condition** をクリックします。

    4. **Create an alert condition** セクションをスライドさせて完成させ、警告条件の動力源となるNRQLクエリを確認します。

       ログ」のアラート条件を保存した後は、アラートUIで表示し、必要に応じて追加変更することができます。

    1. **[one.newrelic.com](https://one.newrelic.com) > Logs** にアクセスしてください。

    2. プロットしたい結果を検索します。例えば、 `service_name:"checkout service" " process failed"` .

       または

       保存したビューを選択します。

    3. **ダッシュボードへの追加** をクリックし、詳細を入力して既存または新規のダッシュボードに追加します。

       また、New Relic One のデータエクスプローラーやクエリビルダーでもチャートを作成することができます。
  </Collapser>

  <Collapser
    id="troubleshoot-error"
    title="エラーのトラブルシューティング（文脈上のログ）"
  >

    1. **APM> (select an app)> Events> Error analytics** にアクセスし、エラートレースを選択します。
    2. エラートレースから **Details** をクリックし、 **See logs** をクリックします。
    3. Logs UIから、関連するログの詳細を参照します。
    4. エラーを発生させているホストを特定するために、 **Show surrounding logs** をクリックします。
  </Collapser>

  <Collapser
    id="troubleshoot-latency"
    title="レイテンシーのトラブルシューティング（文脈上のログ）"
  >
    この方法でレイテンシーをトラブルシューティングするには、 [のログを](/docs/logs/logs-context/configure-logs-context-apm-agents/) のコンテキストで見ることができなければなりません。そして、パフォーマンスが著しく低下したときに、システムがどのように動作していたのかをより深く理解する必要があります。

    1. Go to **[one.newrelic.com](https://one.newrelic.com) > Distributed tracing**.
    2. 特に遅いトレースを選択する。
    3. トレースから **Details**, **See logs for this trace** をクリックします。
    4. Logs UIで関連するログを参照します。
  </Collapser>
</CollapserGroup>
