CI/CDログをNewRelicに送信するように[CircleCIWebhook](https://circleci.com/docs/2.0/webhooks/)サービスを構成します。

## CircleCIでWebhookトリガーを設定する [#setup]

次の手順を実行して、ログをNewRelicに転送するCircleCIWebhookを設定します。

1. [CircleCI](https://app.circleci.com/projects)にログインします。

2. CircleCIプロジェクトの1つに移動します。

3. <DNT>**Project settings**</DNT>をクリックします。

4. プロジェクト設定のサイドバーで、 <DNT>**Webhooks**</DNT>をクリックします。

5. <DNT>**Add webhook**</DNT>をクリックします。

6. Webhookに名前を付けます。

7. New Relic の<DNT>**Logs endpoint**</DNT>を入力してください:

   * 私たち：

     ```
     "https://log-api.newrelic.com/log/v1?Api-Key=YOUR_LICENSE_KEY"
     ```

   * EU：

     ```
     "https://log-api.eu.newrelic.com/log/v1?Api-Key=YOUR_LICENSE_KEY"
     ```

   New Relic アカウントで設定したリージョンを使用します。 `YOUR_LICENSE_KEY`を New Relic <InlinePopover type="licenseKey"/>に置き換えます。

8. ログプッシュをトリガーするイベントの種類に応じて、 <DNT>**Workflow**</DNT>または<DNT>**Job**</DNT>いずれかを選択します。

9. 受信 API またはサードパーティ サービスを設定している場合は、 <DNT>**Test ping event**</DNT>をクリックしてテスト イベントを作成します。
