古い Lambda から新しい Lambda にアップグレードする場合は、以前にデプロイした New Relic S3 Lambda と CloudWatch Lambda を削除することをお勧めします。 または、新しいトリガーをデプロイする前に、以前に作成したトリガーを削除することもできます。

両方のソリューションを使用する場合は、同じリソースから重複したログが送信されないように、重複するトリガーがないことを確認してください。 これは、ログの読み取りで S3 および CloudWatch の制限に達し、New Relic に到達するデータに不整合が生じる可能性があるため、お勧めしません。

  ログ転送のAWSインテグレーションに戻るには、[ここをクリックしてください](/install/aws-logs/?service=aws_services)。
</Callout>
