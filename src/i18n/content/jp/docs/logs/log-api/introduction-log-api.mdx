New Relicの[ログ転送ソリューション](/docs/logs/forward-logs/)ではニーズを満たせない場合、Log API使用して、HTTPエンドポイント経由でログデータをNew Relicに直接送信できます。

## HTTPエンドポイント [#endpoint]

お使いのNew Relicアカウントに応じたエンドポイントを使用してください。

米国（US）エンドポイント：

```
https://log-api.newrelic.com/log/v1
```

[欧州連合](/docs/using-new-relic/welcome-new-relic/get-started/introduction-eu-region-data-center)（EU）エンドポイント：

```
https://log-api.eu.newrelic.com/log/v1
```

FedRAMPエンドポイント：

```
https://gov-log-api.newrelic.com/log/v1
```

## HTTP設定 [#setup]

Log API経由でログデータをNew Relicアカウントに送信する方法：

1. <InlinePopover type="licenseKey" />を取得します。

2. JSONペイロードの[制限と制限された文字](#limits)を確認します。

3. 必要な[ヘッダー](#json-headers)と[ボディ](#json-body)フィールドを使用してJSONメッセージを作成します。

4. `Api-Key`または`License-Key`が[ヘッダー](#auth-headers)または[クエリパラメーター](#query-parameters)に含まれていることを確認します。[ログのJSON例](#log-attribute-examples)を参照してください。

5. `POST`リクエストでNew Relicアカウントの適切なHTTPエンドポイントにJSONメッセージを送信します。

   * US： `https://log-api.newrelic.com/log/v1`
   * EU： `https://log-api.eu.newrelic.com/log/v1`
   * FedRAMP： `https://gov-log-api.newrelic.com/log/v1`

6. トラフィックを発生させて数分待ってから、[アカウントでデータを確認します](#what-next)。

<InlinePopover type="logs" />機能を有効にしてもデータが表示されない場合は、[トラブルシューティング手順](/docs/logs/log-management/troubleshooting/no-log-data-appears-ui/)を参照してください。

## 上限値および文字の制限 [#limits]

<Callout variant="caution">
  顧客向けアプリケーションのコード内からNew RelicのAPIを呼び出さないでください。パフォーマンスの問題が発生したり、レスポンスタイムが遅い場合にアプリケーションがブロックされたりする可能性があります。やむを得ずこの方法で処理する必要がある場合は、パフォーマンスの問題を回避するために、New RelicのAPIを非同期で呼び出してください。
</Callout>

Log APIに送信されるログの制限：

* ペイロードの合計サイズ：<DNT>**1MB(10^6 bytes) maximum per POST**</DNT>。圧縮の使用を強く推奨します。
* ペイロードは<DNT>**UTF-8**</DNT>としてエンコードする必要があります。
* イベントあたりの属性数：最大255個。
* 属性名の長さ：255文字。
* 属性値の長さ：最初の4,094文字は、`message`のように同じ名前の`Log`イベントフィールドとしてNRDBに格納されます。文字列の値が4,094文字を超える場合、長い文字列を[ブロブ](/docs/logs/ui-data/long-logs-blobs)として保存します。

特定の属性には、追加の制限があります。

* `accountId`：これは予約した属性名です。この名前が含まれている場合は、取り込み中に破棄されます。
* `appId`：整数である必要があります。整数以外のデータ型を使用した場合、データは取り込まれますが、クエリを実行できなくなります。
* `entity.guid`、`entity.name`、および`entity.type`：これらの属性は、エンティティを識別するために内部で使用されます。メトリックデータポイントの属性セクションでこれらのキーとともに送信される値によって、UIのエンティティが見つからない、またはテレメトリが予期されるエンティティと関連していないなどの未定義の動作が発生する場合があります。詳細については、[エンティティの合成](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/#entity-synthesis)を参照してください。
* `eventType`：これは予約した属性名です。この名前が含まれている場合は、取り込み中に破棄されます。
* `timestamp`：Unixエポック形式のタイムスタンプ（秒またはミリ秒単位）、またはISO8601形式のタイムスタンプである必要があります。

<Callout variant="important">
  タイムスタンプが48時間以上前のペイロードは破棄される可能性があります。
</Callout>

Log APIに送信されるログのレート制限：

* Log APIへのHTTPリクエストの最大送信レート：1分あたり30万件のリクエスト
* Log APIへの非圧縮ログJSONデータの最大送信レート：1分あたり10GB

### レート制限違反

レート制限を超えると、Log APIの動作に影響が出ます。その場合は、以下の手順に従ってください。

<CollapserGroup>
  <Collapser id="http-requests" title="1分あたりのHTTPリクエスト">
    アカウントのリクエスト件数の上限を超えると、New Relic Log APIは次の1分間までの残り時間、`429`レスポンスを返します。このレスポンスには、再送信または新しいデータの送信が可能になるまでの秒数で示す`Retry-After`ヘッダーが含まれます。

    この問題を解決するには、送信するデータポイントの数を減らすか、レート制限の変更を申請してください。その後サブスクリプションを変更しても、変更したレート制限には影響しません。アカウントを変更し、レート制限がその影響を受ける場合、レート制限を調整するために当社に通知してください。

    レート制限の変更をご希望の場合は、New Relicのアカウント担当者にご連絡いただくか、[サポートポータルを](http://support.newrelic.com)ご覧ください。
  </Collapser>

  <Collapser id="json-bytes" title="1分あたりのJSONサイズ（バイト数）">
    アカウントのログJSONサイズの上限を超えると、New Relic Log APIは次の1分間までの残り時間、`429`レスポンスを返します。このレスポンスには、再送信または新しいデータの送信が可能になるまでの秒数で示す`Retry-After`ヘッダーが含まれます。

    この問題を解決するには、送信するログデータの量を減らすか、送信期間を長くしてください。

    レート制限の変更をご希望の場合は、New Relicのアカウント担当者にご連絡いただくか、[サポートポータルを](http://support.newrelic.com)ご覧ください。
  </Collapser>
</CollapserGroup>

## JSONメッセージ属性の解析 [#message-attribute-parsin]

New Relicのログ管理機能は、`message`属性をJSONとして解析します。messageの解析で得られたJSON属性がイベントに追加されます。`message`属性がJSONでない場合は、そのまま残されます。

以下は、`message`属性の例です。

```json
{
    "timestamp": 1562767499238,
    "message": "{\"service-name\": \"login-service\", \"user\": {\"id\": 123, \"name\": \"alice\"}}"
}
```

以下のように扱われます。

```json
{
  "timestamp": 1562767499238,
  "service-name": "login-service",
  "user": {
    "id": 123,
    "name": "alice"
  }
}
```

## ログのJSONの例 [#log-attribute-examples]

属性は、文字列や数値などのスカラー型のJSON値を指定できます。さらに、複合オブジェクト（または[ネストされた](https://www.digitalocean.com/community/tutorials/an-introduction-to-json#working-with-complex-types-in-json)オブジェクト）を指定することもできます。複合属性に関連付けられた属性はフラット化された名前で保存されます。

例えば、ログエントリの属性内には、次のような複合`user`属性があります。

```json
"attributes": {
    "action": "login",
    "user": {
        "id": 123,
        "name": "alice"
    }
}
```
