<Callout variant="important">
  本書では、新しい用語「イシュー」や「インシデント」を使用しています。[この記事](https://discuss.newrelic.com/t/new-terminology-coming-for-alerts/163727)を参照して、これらの用語の詳細を確認してください。
</Callout>

このページの情報を最大限に活用するには、以下の用語の意味を理解しておくことをお勧めします。

* <DNT>**Issues**</DNT>イシューとは、同じ根本的な問題に関連するインシデントのグループを指します。インシデントが新規作成されると、インシデントインテリジェンスが新しいイシューを開き、他の未解決のイシューが相関関係にあるかどうかが評価される場合があります。詳細については、[イシューおよびインシデントの管理と対応](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/Issues-and-Incident-management-and-response/)を参照してください。

* <DNT>**Incidents**</DNT>インシデントは、時間の経過とともにシステムに影響する症状を説明するイベントを指します。詳細については、[インシデントに関するイベント詳細の表示](/docs/alerts-applied-intelligence/new-relic-alerts/alert-incidents/view-event-details-incidents/)を参照してください。

* <DNT>**Conditions**</DNT>条件では、監視されたデータソース、そしてインシデントと見なされるデータソースの行動を定義する閾値が含まれます。詳細については、[条件の作成](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions/)を参照してください。

* <DNT>**Policies**</DNT>ポリシーとは、1つ以上の条件のグループを指します。詳細については、[ポリシーの管理](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/)を参照してください。

イシューは、インシデントがビジネスに混乱をもたらすことを通知するタイミングを決定します。これらのインシデントは、アラート条件が満たされると発生します。たとえば、CPUの超過、Syntheticモニタリングのクラッシュ、ウェブアプリのレスポンスタイムの遅延などに関するアラート条件を設定できます。

イシューを引き起こすポリシー条件を制御できる場合は、重要な進行中の問題や再発する問題について通知を受けるようにしてください。こうすることで、不要な通知が大量に送信されることがなくなり、ノイズを回避できます。

ポリシー、条件、または条件と信号の組み合わせ別でイシュープリファレンスを設定する方法をご覧ください。

## 問題の環境設定の選択 [#preference-procedures]

デフォルトでは、各ポリシーごとに1つのイシューレコードが作成されます。

選択したポリシーのイシュープリファレンスを変更するには：

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Alert Policies**</DNT>に移動します。

2. ポリシーを選択します。

   <img title="A screenshot showing where the issue preference button is inside the alerts user interface." alt="A screenshot showing where the issue preference button is inside the alerts user interface." src="/images/alerts_screenshot-full_incident-preference-ui.webp" />

   <figcaption>
     <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Alerts &gt; Alert Policies &gt; (select a policy)**</DNT>に移動します。選択されたポリシーのページには、アラート通知とUIの詳細について、<InlinePopover type="alerts" /> がイシューをロールアップする方法が表示されます。デフォルトのオプションは<DNT>**One issue per policy**</DNT>です。 このポリシーに別のオプションを選択するには、 <DNT>**Issue creation preference**</DNT>をクリックします。
   </figcaption>

3. <DNT>**Issue creation preference**</DNT>を変更してから、利用可能な[オプション](#preference-options)を選択し、 <DNT>**Save**</DNT>します。

   <img width="60%;" class="center" title="A screenshot showing how to change your issue preferences on the alerts user interface." alt="A screenshot showing how to change your issue preferences on the alerts user interface." src="/images/alerts_screenshot-crop_incident-preference-options.webp" />

必要に応じて、各ポリシーに対してこれらの手順を繰り返します。

## 重大なインシデントと警告インシデント [#incident-types]

ポリシー条件が満たされた場合、次の2つの閾値タイプから選択します。

* クリティカル（赤）
* 警告（黄）

ポリシーの条件が閾値に違反すると、インシデントが開かれ、効率よく対応できるよう、詳細情報を含むイシューレコードを作成できます。イシューレコードには、最初のインシデント後に発生した他のインシデントも含まれる場合があります。

通知は、ポリシーのすべての通知チャネル、およびイシューのオープン／認識／クローズ時に適切に設定されたワークフローのすべての宛先に送信されます。

### ポリシーごとに1つのイシュー（デフォルト） [#preference-policy]

ポリシーとその条件に焦点が絞られた範囲がある場合は（エンティティの監視など）、 <DNT>**One issue per policy**</DNT>プリファレンスを選択します。

イシューを開いた後に、追加の条件インシデントが発生した場合、これらのインシデントは同じイシューに追加されます。選択したポリシーの<DNT>**Issues &amp; activity**</DNT>ページには、1つのイシューのみが表示されます。[イシューの作成プリファレンス](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#issue-creation)の詳細をご覧ください。

<img width="40%;" class="center" title="A diagram showing how the <DNT>**One issue per policy**</DNT> option works" alt="A diagram showing how the <DNT>**One issue per policy**</DNT> option works" src="/images/alerts_diagram_by-policy.webp" />

<figcaption>
  イシュープリファレンスとしてデフォルトのオプション、<DNT>**One issue per policy**</DNT>を選択すると、そのポリシーの任意の条件によって開かれたすべてのインシデントが同じイシューにグループ化されます。
</figcaption>

### 条件ごとに1つのイシュー [#preference-condition]

各インシデントレコードを特定の条件に集中させたい場合に、<DNT>**One issue per condition**</DNT>オプションを選択します。 ポリシーに複数の条件が存在する場合、各条件の別のイシューレコードが<DNT>**Issues &amp; activity**</DNT>ページに表示されます。 [イシューの作成プリファレンス](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#issue-creation)の詳細をご覧ください。

* 信号が条件の閾値に違反すると、インシデントが開き、イシューレコードが作成されます。
* 同じ信号が別の条件の閾値に違反すると、別のインシデントが開き、別のイシューレコードが作成されます。
* イシューがオープンした後に、追加の信号が同じ条件のインシデントをオープンした場合、その条件に対して既にオープンされているイシューレコードにこれらのインシデントが追加されます。

<img width="40%;" class="center" title="A diagram showing how the <DNT>**One issue per condition**</DNT> option works" alt="A diagram showing how the <DNT>**One issue per condition**</DNT> option works" src="/images/alerts_diagram_by-condition.webp" />

<figcaption>
  イシュープリファレンスとしてオプション、<DNT>**One issue per condition**</DNT>を選択すると、特定の条件について、あらゆる信号によるすべてのインシデントが別のイシューにグループ化されます。
</figcaption>

### 条件と信号ごとのイシュー [#preference-signal]

<DNT>**One issue per condition and signal**</DNT>オプションは、最もきめ細かいレベルのイシューレコードを作成します。インフラストラクチャ全体で発生している内容を緊密に監視する場合に、このイシュープリファレンスを選択します。[イシューの作成プリファレンス](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/#issue-creation)の詳細をご覧ください。

<img width="40%;" class="center" title="A diagram showing how the <DNT>**One issue per condition and signal**</DNT> option works" alt="A diagram showing how the <DNT>**One issue per per condition and signal**</DNT> option works" src="/images/alerts_diagram_by-signal.webp" />

<figcaption>
  イシュープリファレンスとして<DNT>**One issue per condition and signal**</DNT>を選択すると、同じ条件と信号を共有するインシデントを独自のイシューにグループ化します。イシューが開いた後に、ある条件の同じ信号によって別のインシデントが開いた場合（警告インシデントが開いたその少し後に、同じ信号で重大インシデントが発生する場合など）、その新しいインシデントが同じイシューに追加されます。<DNT>**Issues &amp; activity**</DNT>ページには、選択した条件/信号の組み合わせに対して1つのイシューのみが表示されます。
</figcaption>
