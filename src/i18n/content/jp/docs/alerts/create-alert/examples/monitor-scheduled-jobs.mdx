[critical thresholds](/docs/alerts-applied-intelligence/new-relic-alerts/advanced-alerts/advanced-techniques/set-thresholds-alert-condition) と [loss of signal settings](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions#signal-loss) を使用して、スケジュールされたジョブを監視し、それらが起こらなかった場合に通知を受けることができます。

例えば、24時間ごとにメンテナンスが予定されているシステムの場合、そのメンテナンスが行われない時間帯を知りたいと思います。

この手法は、定期的にスケジューリングされたジョブが、信号喪失の最大閾値時間である48時間を超えて発生する場合にのみ有効です。

## 知っておきたいこと

この手順を最大限に活用するためには、 [条件を作成する方法を知っておく必要があります](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) 。

## 使い方

通常、条件ではクリティカル値を使用してインシデントをトリガーします。 ただし、信号が失われるとインシデントが発生する可能性もあります。スケジュールされたジョブを監視し、実行されなかった場合に通知を受け取るには、信号損失タイマーをジョブ スケジュールに合わせて設定します。

## スケジュールされたジョブの監視

1. [監視したいジョブに関連する条件](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) を作成します。
2. <DNT>**+ Add lost signal threshold**</DNT>をクリックし、シグナルの有効期限をモニター ジョブのスケジュールされたサイクルよりも長く設定します。
3. <DNT>**Add lost signal threshold**</DNT>確認してください。
4. 必ず、条件に名前を付けてから保存します。
