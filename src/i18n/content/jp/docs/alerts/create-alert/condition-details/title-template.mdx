タイトルテンプレートは、イベントを条件で開く場合に使用します。 デフォルトのタイトルを上書きします。

## タイトル テンプレートを使用する理由は何ですか? [#why-titles]

アラート条件はイベント制作の基礎です。 ガイド モードを使用する場合でも、チャートやポリシーから条件を作成する場合でも、一意のタイトル テンプレートを使用することが、アラート条件を作成する最後の手順になります。

タイトル テンプレートの使用はオプションですが、推奨されます。アラート条件は、監視する一連の閾値を定義します。 これらの閾値のいずれかが違反された場合、大量イベントが作成されます。 わかりやすいタイトル テンプレートを使用すると、問題を正確に特定し、停止をより迅速に解決できます。

## あなたが始める前に [#begin]

タイトル テンプレートを使用する前に注意すべき重要な点は次のとおりです。

* **互換性**: タイトル テンプレート機能は、NRQL 条件でのみ使用できます。
* **文字数制限**: タイトル テンプレートの最大長は 256 文字です。
* **フォーマット**: タイトル テンプレートは有効な[Handlebars](https://handlebarsjs.com/guide/)フォーマットに従う必要があります。
* **Lost signal 多発イベント**: Lost signal によってトリガーされる集中イベントの場合、デフォルトのタイトルが常に使用されます。 信号損失の詳細については、 [「信号損失とギャップの補完」](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/)を参照してください。

## タイトルテンプレートを理解する [#example]

電子商取引サイトを運営しており、ネットワーク エラーが多いときに集中イベントをトリガーするアラート条件を作成したとします。 タイトル テンプレートを使用すると、この条件からの頻繁なイベントを簡単に識別できるようになります。

このアラート条件に`High Network Errors`という名前を付けました。ただし、イベントのタイトルには、エンティティと優先順位に関するより多くの情報を反映させたいと考えています。

**例**:

```handlebars
{{conditionName}} Affecting {{entity.name}} - {{entity.type}} - {{priority}}
```

このテンプレートは、次のような人気イベントのタイトルを生成します。

```
High Network Errors Affecting Entity123 - Kubernetes Pod - Critical
```

## タイトルテンプレートを使用する [#use-titles]

タイトル テンプレートは、UI 経由または GraphQL を使用する 2 つの方法で操作できます。 方法については、次の手順に従ってください。

<CollapserGroup>
  <Collapser id="template-ui" title="UIでタイトルテンプレートを作成する">
    1. NRQL 条件ビルダーに移動します。 手順については、 [「アラート条件」](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/alert-conditions/)を参照してください。
    2. NRQL 条件を作成します。
    3. 最後のステップ**Add details** \[詳細の追加]で、 **Title template** \[タイトル テンプレート]フィールドにタイトル テンプレートを入力します。
    4. 条件を保存します。
  </Collapser>

  <Collapser id="template-ui" title="GraphQLでタイトルテンプレートを作成する">
    GraphQL では、タイトル テンプレート フィールドは`titleTemplate`と呼ばれます。 NRQL 静的条件を更新してタイトル テンプレートを追加するミューテーションの例を次に示します。

    ```graphql
    mutation {
      alertsNrqlConditionStaticUpdate(
        accountId: 123456
        id: 456789
        condition: {titleTemplate: "{{conditionName}} has triggered an incident"}
      ) {
        titleTemplate
      }
    }
    ```
  </Collapser>
</CollapserGroup>

## 属性を使用して条件をさらに定義する [#attributes]

タイトル テンプレートに集計イベント プロパティのサブセットを含めて、より多くのコンテキストを追加できます。

その方法は次のとおりです。

* **`{{AN_ATTRIBUTE}}`形式を使用します**。 例えば：

  ```handlebars
  {{conditionName}} has triggered an alert event
  ```

* **タグ属性**: タグには、 `{{tags.TAG_NAME}}`形式を使用します。

* **ファセット句タグ**: NRQL 条件で`FACET`句が使用されている場合は、 `tags`フォーマットを使用して、タイトル テンプレートでそれらの値を使用できます。 たとえば、NRQL クエリに`FACET hostName, cluster`含まれている場合は、次のように使用できます。

  ```handlebars
  Host {{tags.hostName}}, cluster {{tags.cluster}} alert event
  ```

タグ名にはスペースを含めることはできませんが、展開された値にはスペースを含めることができます。利用可能なプロパティの完全なリストについては、 [「集中イベント プロパティ」](/docs/alerts/create-alert/condition-details/alert-event-attributes)を参照してください。
