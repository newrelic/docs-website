複数の関連するアラート条件が個別に発火すると、集計ノイズが発生し、重大な問題の特定が困難になります。New Relicの既存の相関機能はこのノイズを減らすのに役立ちますが、特定の条件の組み合わせに基づいてアラートがいつトリガーされるかについて、より正確な制御が必要になる場合があります。

複合アラートを使用すると、論理演算子<DNT>(AND, OR, NOT)</DNT>を使用して、複数のアラート条件をトリガー式に組み合わせることができます。これらは、条件の特定の組み合わせが満たされた場合 — または1つの条件がトリガーされても別の条件がトリガーされない場合 — にのみトリガーされるため、より関連性の高い通知をより少ない回数で提供します。

<img title="Overview of compound alerts" alt="A screenshot of how the compound alert builder works" src="/images/compound-alerts-overview.webp" />

<DNT>**Compound alerts**</DNT>使用すると、次のことが可能になります。

* <DNT>**Reduce alert noise**</DNT>：複数の関連する条件からの個別のアラートの代わりに、単一の[複合集計イベント](/docs/alerts/create-alert/create-alert-condition/compound-alerts/compound-alert-events)を受信し、通知の全体的な量を削減します。
* <DNT>**Combine up to five alert conditions**</DNT>：最大5つの既存のアラート条件を選択します。AND、OR、およびNOT演算子と括弧を使用して、複雑なロジックのためにそれらを組み合わせます。
* <DNT>**Combine cross-account conditions**</DNT>：異なるアカウントのデータをターゲットとするアラート条件を含めます。すべてのコンポーネント条件は同じ所有アカウントを持つ必要がありますが、他のアカウントのデータをクエリできます。
* <DNT>**Control facet matching**</DNT>：条件間で共有されるファセットの処理方法を選択します。複合集計は、同じファセット値がすべての条件で閾値を満たしたときにトリガーすることも、一致するファセット値に関係なくトリガーすることもできます。
* <DNT>**Silence component condition alert events**</DNT>：個々のアラート条件の閾値を設定して、それらが複合集計の一部である場合に独自の集計イベントを抑制します。これにより、複合集計をアクティブに保ちながら、寄与する条件からのノイズを減らします。
* <DNT>**Add tags to compound conditions**</DNT>：標準のアラート条件と同じように、カスタムタグを使用して複合アラート条件を整理およびフィルタリングします。タグは、アラートを分類し、インシデントのトリアージを合理化するのに役立ちます。

## 使用事例 [#use-cases]

複合アラートを使用する可能性のあるユースケースをいくつか紹介します：

* <DNT>**Detect memory leaks**</DNT>：メモリ&gt;50％（インフラストラクチャ）かつスループット&lt;1k（APM）の場合に集計します。これは、対応するトラフィックなしでメモリが増加する状況を特定するのに役立ちます。
* <DNT>**Database performance issues**</DNT>：データベースのレスポンスタイム&gt;2s AND（CPU使用率&gt;80％ OR コネクションプール&gt;90％）AND NOT メンテナンス期間中の場合に集計し、計画的なメンテナンス中のノイズを回避しながらパフォーマンスの低下を特定するのに役立ちます。

## 権限とアクセスコントロール [#permissions]

複合アラートは、標準のアラート条件と同じ権限モデルを使用します：

* 標準のアラート条件を作成できるユーザーは、[複合アラート条件を作成する](/docs/alerts/create-alert/create-alert-condition/compound-alerts/create-compound-alert)こともできます。
* アカウントへのアクセス権を持つすべてのユーザーは、そのアカウントの複合アラート条件を表示および管理できます。

## 制限 [#limitations]

複合アラートを使用する場合は、次の制限事項に注意してください：

* <DNT>**Maximum conditions**</DNT>：単一の複合集計で最大5つの条件を組み合わせることができます。
* <DNT>**No nesting**</DNT>：複合アラート条件を別の複合アラートのコンポーネント条件として使用することはできません。
* <DNT>**Account scope**</DNT>：すべてのコンポーネント条件は同じ所有アカウントからのものである必要があります（ただし、クロスアカウント条件を介して他のアカウントのデータをターゲットにすることはできます）。
* <DNT>**Facet matching**</DNT>：条件が複数の属性によってファセット化されている場合、少なくとも1つの属性が同じであるときに一致が発生します（詳細な制御は現在サポートされていません）。
