複合アラート条件のトリガー式がtrueと評価されると、New Relicは複合アラートイベントを開きます。イベントには、ランブックURLへのリンクが含まれます。各コンポーネント条件の特定の閾値違反を表示するには、個別のアラートイベントをクリックして確認します。

## アラートイベントと問題の相関関係 [#alert-events-correlation]

複合アラート条件は、関連する複数のアラートイベントを作成します：

* <DNT>**Multiple alert events**</DNT>：複合アラート条件は、独自の複合アラートイベントを作成します。デフォルトでは、各コンポーネント条件も、閾値に違反したときに独自のアラートイベントを作成します。ノイズを減らすために、アラート条件の設定からこれらのイベントをサイレンスできます。コンポーネント条件のアラートイベントをサイレンスすると、その条件のタグは複合アラートイベントに表示されません。
* <DNT>**Correlating to the same issue**</DNT>：関連するすべてのアラートイベント（複合およびコンポーネントの両方）が同じ問題にグループ化されるようにするには、複合アラート条件と同じポリシーを使用するようにコンポーネント条件を設定します。これにより、New Relicの問題の相関関係がそれらをグループ化できるようになります。
* <DNT>**Viewing related events**</DNT>：複合アラートイベントから、個々のコンポーネントアラートイベントに移動して、各条件の閾値違反とメトリクスを確認できます。
