## まとめイベントインデックスとまとめイベントの詳細を表示する [#view-alert-events]

集中イベントは「まとめイベント」にグループ化されます。 イベントのグループ化方法を変更する場合は、関連するポリシーを開いて、 [<DNT>**Alert event preference**</DNT>](/docs/alerts/organize-alerts/specify-when-alerts-create-events/)設定を変更します。

まとめイベントの詳細を表示するには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Alerts**</DNT>に移動し、 <DNT>**Alert events**</DNT>タブをクリックします。

2. 表内の人気イベントの 1 つを選択して、その詳細を開きます。

個別の集中イベント チャートの詳細は次のとおりです。

* <DNT>**Timing information**</DNT>: チャート上の赤い影の部分は集中イベントが発生した期間を示し、その前のピンクの影の部分は[劣化期間](/docs/using-new-relic/welcome-new-relic/get-started/glossary#degradation-period)を表します。 2 時間以上続いた集計イベントを選択した場合、グラフの下部のタイムラインがギザギザになります。 まとめイベントのコンテキストを提供するために、チャートにはまとめイベントの前後の時間枠も表示されます。

* <DNT>**Chart guidelines**</DNT>: 赤い点線は条件の閾値を示します。 青い線はパフォーマンス情報を示します。

* <DNT>**Anomalous behavior:**</DNT> 集中イベントの開催時期近くにアラートが[異常な動作](#anomalous-behavior)を検出した場合は、集中イベントの詳細に通知が表示されます。

## イベントをまとめイベントで見る [#view-log]

[すべての製品のアラート イベントを](/docs/alerts/alert-event-management/view-alert-event-details)表示するには、 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DNT>に移動して<DNT>**All entities**</DNT>をクリックします。1 つのイベントのみのイベントを表示するには:

1. に行く <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; All entities.**</DNT>
2. エンティティの行を選択します。
3. 左側のナビゲーションの<DNT>**Events**</DNT>の下で、 <DNT>**Issues &amp; activity**</DNT>をクリックします。
4. イベントを選択すると、そのイベントのチャートと詳細が表示されます。

## イベント集中から通知までの時間 [#alert-event-time]

データ処理時間の変動により、集中イベントの開催時間と最初の通知時間の間に最大 3 分の差異が発生する場合があります。

* <DNT>**Notification time:**</DNT> 通知内の時間は、通知の配信リクエストを受信した時点のタイムスタンプを反映します。
* <DNT>**Alert event time:**</DNT> <DNT>**Events**</DNT>ページに表示されるまとめイベントの時間は、まとめイベントの開始に貢献した最後のデータ ポイントのデータ収集のタイムスタンプを反映しています。

## 変則的な動作の検出 [#anomalous-behavior]

アラート エンティティやアラート エンティティの上流/下流アプリケーションで主要な信号の大きな変化が検出されると、まとめイベントのページおよび通知チャネルに「異常な動作が検出されました」という通知が表示されます。 あなたはできる：

* 検出された異常の詳細については、通知を展開します（Webのみ）。
* 上流/下流の異常を見る（Slackのみ）。
* リンクを選択すると、該当する製品のチャートに移動して、さらに検討することができます。
