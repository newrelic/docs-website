## バグ修正

* アプリがフォアグラウンドとバックグラウンドを複数回切り替えた後にトレース リスナーが適切に削除されないメモリ リークの問題を解決しました。
* スレッド同期でデータが New Relic に報告されないことがある問題を修正しました。
* ANR 中のログ レポートに関連する問題に対処し、安定性とパフォーマンスを向上しました。
* 互換性を確保し、競合を防ぐために、Curtains ライブラリに関連する重複クラスの問題を修正しました。
* 堅牢性を強化するために、setUserId メソッドの null ポインター例外の問題を解決しました。
* デバイスのネットワーク エラーが原因でリクエストが失敗した場合に URL が正しく報告されないという Retrofit Interceptor の問題を修正しました。

## サポートステートメント

少なくとも 3 か月ごとにエージェントを更新することをお勧めします。Androidエージェントのサポートに関する具体的なポリシーと日付については、「 [モバイル監視エージェント EOL ポリシー」](/docs/mobile-monitoring/new-relic-mobile/get-started/mobile-agents-eol-policy/#android-eol)を参照してください。
