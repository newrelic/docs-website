## パッチファイルが含まれています

このV3.0 SP12リリースには、以下のファイルが1つ含まれています。

* K-300SP12NEWR.SAR

## 実施手順

### サービスパックの導入と設定

* **インストール場所:**この Service Pack を中央監視システムのクライアント 000 にデプロイします。
* **設定場所:**以降のすべての設定タスクは、中央監視システムのビジネス クライアントで実行します。
* **完全な手順:**段階的なインストール手順については、**インストレーション ガイド V3.0** (付録 VIII: サービス パックのインストール) または**管理者ガイド V3.0** (セクション: サービス パックのインストール) を参照してください。

## このパッチのアップデート内容

SP12は、バグ修正と新機能に関するNew Relic Notesのコレクションです。

### NR 1344

`NR_SAP_AIF_ERROR`の新しい問題`AIF_TABLES`により、 `/AIF/STD_IDX_TBL`以外のテーブルからエラーメッセージを取得できるようになります。

### NR 12-02

APIベースのデータプロバイダーに関する問題を修正し、新機能を追加します。

* API宛先に到達できない場合に`/NEWR/API_DATA_COLLECTOR`で発生するコードダンプを修正しました。
* CALMランドスケープデータの抽出時に発生するリソース障害を修正しました。
* BTPメッセージ処理ログ抽出時のリソース障害を修正しました。
* 監視グループCALMに属するすべてのデータプロバイダーのCALL IDとして設定されているサービスID（LMSID）の検証を追加しました。
* データ収集APIを呼び出す前に証明書の有効性チェックを追加しました。

### NR 12-03

レジストリ内の呼び出しIDによるテストと、コレクションマネージャのデータ収集遅延および固定データ収集ウィンドウを実装します。

### NR 12-05

以下の機能強化を実施します。

* CALM監視グループにおけるデータプロバイダーのページネーションのサポート
* RFC チェック プログラムが単一の RFC 宛先をチェックできるようにする新しいプログラムはありません
* 単一の呼び出しIDに対してのみデータプロバイダテストを強制する設定

以下の問題を修正します。

* コレクションマネージャの出力に空白のコールIDが表示される
* 認証情報の検証に関する問題 `NR_CF_AUDIT_EVENTS`

### NR 12-06

新しいデータプロバイダ`NR_SAP_ABAP_WS_COMPLETE`は、ABAP Webサービスイベントを収集します。

### 追加のバグ修正

上記の注記に加え、SP12では以下の問題も修正されています。

* テーブル取得データプロバイダ関数モジュール`/NEWR/RFC_EVENTS_COLLECTOR`のタイムスタンプの問題を修正しました。 `/NEWR/RFC_EVENTS_COLLECTOR2`
* トレースレコードへのリンクを容易にするために、IDoc イベントに現場 ID を追加しました

## 手動設定更新

### NR\_SAP\_AIF\_ERROR に新しい問題を追加

新しい`AIF_TABLES`問題を設定するには:

1. 付属のトランスポートを中央監視システムに適用してください。
2. トランザクション`/n/NEWR/REGISTRY`へ移動してください。
3. データプロバイダー`NR_SAP_AIF_ERROR`を選択し、 **Configure** \[設定]をクリックします。
4. 問題`AIF_TABLES`を追加し、設定されたネームスペース/インターフェイス/バージョンの組み合わせに対して、 `/AIF/T_IDX_TBL`で指定されたテーブル名を入力します。

<Callout variant="tip">
  `AIF_TABLES`が空白または指定されていない場合、インデックス テーブルは標準の単一インデックス テーブル`/AIF/STD_IDX_TBL`にデフォルト設定されます。
</Callout>

### 収集遅延と固定収集ウィンドウを設定します。

流動的なコレクション境界とデータベース レイテンシによって生じるデータ ギャップに対処するために、2 つの新しい設定フィールドがデータ プロバイダー レジストリに追加されました。

**データプロバイダーレジストリに新しいフィールドが追加されました。**

* **DATA\_COLLECTION\_DELAY:**収集タイムスタンプから差し引く、意図的な遅延時間を分単位で指定します。このバッファにより、データベースの更新によって遅れて到着するデータが、収集ウィンドウが閉じる前に確実に取得されます。
* **FIXED\_SCHEDULES:**固定収集期間を有効にするフラグ。`X`に設定すると、収集タイムスタンプから秒数が削除され、一貫性のある予測可能な収集ウィンドウが作成されます。

**使い方：**

**固定スケジュール**（ `FIXED_SCHEDULES = 'X'` ）：

* 収集時のタイムスタンプは、秒数を削除することで正規化されます。
* 例：12:12:23 は 12:12:00 になります。
* すべてのコレクションにわたって、一貫性のある、整合性の取れたコレクション境界を作成します。

**データ収集遅延**（ `DATA_COLLECTION_DELAY > 0` ）：

* 開始および終了のデータ収集タイムスタンプから、指定された遅延時間（分単位）を差し引きます。
* 収集ウィンドウを後方に移動して、見逃されたデータベース更新を捕捉します。
* 例：2分間の遅延により、10:00:00 - 10:05:00 が 09:58:00 - 10:03:00 に変更されます。

## ABAPビジネスシステム向けの新しいデータプロバイダー

### NR\_SAP\_ABAP\_WS\_COMPLETE

レコードの制限なしに、Web サービスのメッセージ ヘッダーとデータ (タグ) レコードを監視システム間で並行して処理します。

**構成。**

* **タイプ:**イベント
* **コミュニケーション担当者：** N/A
* **データプロバイダー機能:** `/NEWR/WS_MESSAGES_ARFC`
* **データプロバイダーRFCが有効：**はい
* **フィニッシャー機能:** `/NEWR/WS_EVENTS_FINISHER`
* **監視グループ：** `SAP_APPL`

### 機能概要

このデータプロバイダーは、既に有効化された状態で提供されます。必須事項:

* **`ADAPTER_TYPE`**

  * **目的：** ABAP Webサービスメッセージヘッダーから選択するアダプタタイプを指定します。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値：** `;`で区切られたテーブルフィールド。複数行にすることができます。

* **`DESTINATION`**

  * **目的：** RFC宛先を指定する
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:** RFC宛先

* **`DPC_COLL_TIME`**

  * **目的：**時間間隔を決定する
  * **サイン：**インクルーシブのみ
  * **オプション：** BT
  * **値:**システム生成の開始タイムスタンプと終了タイムスタンプ

* **`IB_IF_NAME`**

  * **目的：** ABAP Webサービスメッセージヘッダーから受信インターフェース名を選択する。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値：**受信インターフェース名を`;`で区切って指定します。複数行に指定できます。

* **`IB_PARTY`**

  * **目的：** ABAP Webサービスメッセージヘッダーから受信者を選択する。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:**受信者を`;`で区切って指定します。複数行にすることも可能です。

* **`JOIN_CONDITION`**

  * **目的：**テーブル`/NEWR/JOIN_COND`の結合レコードを指定して、後続の処理に用いる。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:** `;`で区切られた表示タイトル。複数行にすることも可能です。

* **`NR_ACCID`**

  * **目的：** New RelicアカウントIDを指定する
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:** New RelicアカウントID

* **`OB_IF_NAME`**

  * **目的：** ABAP Webサービスメッセージヘッダーから送信インターフェース名を選択する。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:** `;`で区切られた送信インターフェース名。複数行にすることができます。

* **`OB_PARTY`**

  * **目的：** ABAP Webサービスメッセージヘッダーから送信者を選択する。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:**送信者を`;`で区切って指定します。複数行にすることも可能です。

* **`SETTING_SWITCH`**

  * **目的：**分散処理で使用されるオプションを指定する

  * **サイン：**インクルーシブのみ

  * **オプション：** EQ

  * **値:**分散プロセスを制御するための、 `;`で区切られた複数のオプション:

    * **HEADONLY=**

      * `SPACE` (デフォルト): ヘッダーとタグ データの両方を提供します (タグ名が`TAG_LIST`で指定されている場合)。
      * `X`: `TAG_LIST`でタグ名が指定されている場合でも、ヘッダー データのみが提供されます。

    * **FIRSTONLY=**

      * `X` (デフォルト): データ プロバイダーは、 `TAG_LIST`にリストされている各タグの最初の出現からのみデータを抽出します。
      * `SPACE`: データ プロバイダーは、 `TAG_LIST`にリストされている各タグのすべての出現からデータを抽出します。

    * **FREE%=nn:**監視システムで使用可能な作業プロセスの割合を指定して、必要な空き作業プロセスの制限を決定します。

      * 現在の監視システムにおける空きワーキングプロセスの数が、この「望ましい空きワーキングプロセスの制限数」以上である場合、分散処理を続行します。

      * それ以外の場合は、1秒待ってから、以下のいずれかになるまで上記のチェックを繰り返します。

        * 配布処理は継続できます。
        * 合計待ち時間が指定された最大待ち時間を超えています（ `MWAIT#=nn`参照）。

      * このオプションのデフォルト値は30%です。

    * **MINFT#=nn:**最小空き作業プロセス制限を指定します:

      * システムによってカウントされた希望する空き作業プロセス数がこの最小値を下回った場合、システムは処理を続行する前に1秒間一時停止します。
      * このオプションのデフォルト値は3です。

    * **MWAIT#=nn:**最大待機時間（秒単位）を指定します。

      * 合計待機時間（ `FREE%=nn`を参照）が指定された最大値を超えた場合、システムリソースを節約するために配布プロセスはキャンセルされます。
      * このオプションのデフォルト値は60です。

    * **MREC#=nnnnn:****関数モジュール**`/SDF/IMA_DC_WS_ABAP`を呼び出すたびに取得される**Web サービス メッセージ ヘッダー**レコードの数制限を指定します。
      * このオプションのデフォルト値は10000です。

    * **JSEND#=nnnn:** New Relicサーバーに毎回送信される結果属性レコードの数制限を指定します。
      * このオプションのデフォルト値は5000です。

    * **USER=user:**各モニター システムの RFC ユーザー名を指定して、そのシステムで現在使用されているワーク プロセスの総数を取得します。 `user`は SAP ユーザー ID です。

    * **DPWP#=nn:****最大作業プロセス制限**を指定します。

      * 分散処理中：現在の監視システムにおける空きワーキングプロセスの数がこの最大制限値以下である場合、分散処理を続行します。
      * そうでない場合は、1秒待ってから再度確認に戻ります。
      * 分散処理が続行できるか、合計待機時間が指定された最大待機時間 (`MWAIT#=nn`) を超えるまで、このループを続けます。
      * 注: このチェックは、ユーザー被害が初期値以外で指定されている場合にのみ実行されます。
      * このオプションのデフォルト値は8です。

* **`STATE_GROUP`**

  * **目的：** ABAP Webサービスメッセージヘッダーから処理ステータスグループを選択する。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値：** `;`で区切られた処理ステータスグループ。複数行にすることができます。

* **`SYS_ID`**

  * **目的：**システムIDを特定する
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値:**システムID

* **`TAG_LIST`**

  * **目的：** ABAP Webサービスメッセージデータから選択し、メッセージデータの出力として表示するタグ名を指定します。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値：** `;`で区切られたテーブルフィールド。複数行にすることができます。

* **`USERNAME`**

  * **目的:** ABAP Webサービスメッセージヘッダー(`CREATED_BY`)からメッセージを作成したユーザー名を選択します。関数モジュール`/SDF/IMA_DC_WS_ABAP`のフィルタオプション。
  * **サイン：**インクルーシブのみ
  * **オプション：** EQ
  * **値：**ユーザー名を`;`で区切ったもの。複数行にすることも可能です。
