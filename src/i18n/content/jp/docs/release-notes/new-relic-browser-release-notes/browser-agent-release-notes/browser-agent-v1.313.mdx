## v1.313.0

### 特徴

#### MFEエラーを自動検出

`JavaScriptError`イベントの自動マイクロフロントエンド（MFE）アトリビューションのサポートを追加します。以前は、`JavaScriptError`イベントはコンテナブラウザアプリケーションによってグローバルにのみキャプチャされていました。この変更により、エージェントは、スローされたErrorインスタンスのスタックトレースを、`register`APIに登録したMFEモジュールのコールスタックシグネチャと相互比較します。Browserエージェントへの登録を担当するファイルが、投げられたErrorインスタンスのスタックトレースに見つかった場合、そのイベントはコンテナではなくMFEに属性付けられます。

#### &quot;withCredentials&quot; を削除

[v1220](https://github.com/newrelic/newrelic-browser-agent/releases/tag/v1220)より前からエージェントでクッキーは使用されていないため、エージェントのハーベストから`withCredentials`または`credentials: 'include'`の使用をすべて削除します。

#### MicroFrontEndTimingヒューリスティクスの改善

MFEのスクリプト実行時間を検出する機能を改善し、より正確な`timeToLoad`メトリクスをもたらします。

#### CSP違反を新しいイベントとしてレポートする

バブリングされたContent Security Policy（CSP）違反はキャプチャされ、新しい`SecurityPolicyViolation`イベントタイプとしてPage インサイトの消費者に報告されるようになりました。それらはNRQLでクエリ可能になります。

#### APIの登録に法的およびコンプライアンス要件を適用する

リミテッドプレビュー向けに公開する前に、登録APIが会社の要件を満たしていることを確認してください。

### バグ修正

#### Safariでの不正な`trace.lastTimestamp`+バックナビゲーションのエッジケース

Safariブラウザの戻るナビゲーション（BFCache）により、セッショントレース機能の開始時に絶対タイムスタンプが記録され、結果として異常な`trace.lastTimestamp`値が発生し、単一のペイロード内で相対タイムスタンプと絶対タイムスタンプが混在する可能性がある競合状態を修正します。

## サポートステートメント

New Relic では、最新の機能とパフォーマンス上のメリットを確実に得られるよう、エージェントを定期的にアップグレードすることをお勧めします。古いリリースは[サポート終了](https://docs.newrelic.com/docs/browser/browser-monitoring/getting-started/browser-agent-eol-policy/)になるとサポートされなくなります。リリース日は、エージェント バージョンの元の公開日を反映します。

新しいブラウザエージェントのリリースは、一定期間にわたって小さな段階で顧客に展開されます。 このため、リリースがアカウントでアクセス可能になる日付は、元の公開日と一致しない可能性があります。詳細については、この[ステータス ダッシュボード](https://newrelic.github.io/newrelic-browser-agent-release/)をご覧ください。

当社の[ブラウザサポートポリシー](https://docs.newrelic.com/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring/#browser-types)に従い、Browserエージェントのv1.313.0は、以下のブラウザおよびバージョン範囲向けに構築され、テストされています：Chrome 136～146、Edge 136～146、Safari 18～26、およびFirefox 138～148。モバイルデバイスの場合、v1.313.0はAndroid OS 16およびiOS Safari 18～26.2向けにビルドおよびテストされました。
