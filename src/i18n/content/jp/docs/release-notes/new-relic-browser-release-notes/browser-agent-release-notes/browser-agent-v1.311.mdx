## v1.311.0

### 特徴

#### エージェント識別子の削除

エージェント アーキテクチャーを簡素化するために、エージェント識別子によるスコープ付き参照を削除しました。 すべての参照情報はエージェント参照内に直接格納されるようになったため、ルックアップキーを介してエージェント参照とスコープ付きオブジェクトを統合する必要がなくなりました。

#### すべての PageViewTiming ノードに元のページ URL を属性として追加する

LCPに関する以前の作業を基に、 `pageUrl`帰属をすべてのタイミングに拡張しました。`PageViewTiming`イベントは、すべてのノードタイプに対して元のハードナビゲーション URL を報告するようになり、収集前に URL が変更された場合でも SPA のデータ一貫性が確保されます。

#### MFEタイミングロジックを改善する

register APIを呼び出す正しい MFE スクリプトを識別するエージェントの機能が強化されました。 この改善により、MFE環境におけるタイミング情報の精度が向上します。

### バグ修正

#### 新規MFE登録に文字列IDを強制する

MFE エンティティ合成中に非文字列 ID によってエラーが発生する問題を解決しました。 エージェントは文字列ベースのIDを強制し、異なるデータ型が指定された場合は警告を発するようになりました。

#### 円形置換ツールは、収穫物から再利用されたオブジェクトを削除しました。

内部のstringifyメソッドがハーベストペイロード間で共有ポインタを削除してしまう問題を修正しました。この問題により、エラーハーベスト時に`params`属性が欠落することがありました。

#### 正規表現を更新して、最小化されていないwebpackビルドの問題を防止します。

正規表現内の文字「 `$` 」を16進数表記に置き換えました。これにより、Webpackがビルドプロセス中に文字を誤って置換することを防ぎ、以前は最小化されていないビルドでエージェントのエラーが発生していた問題を回避できます。

## サポートステートメント

New Relic では、最新の機能とパフォーマンス上のメリットを確実に得られるよう、エージェントを定期的にアップグレードすることをお勧めします。古いリリースは[サポート終了](https://docs.newrelic.com/docs/browser/browser-monitoring/getting-started/browser-agent-eol-policy/)になるとサポートされなくなります。リリース日は、エージェント バージョンの元の公開日を反映します。

新しいブラウザエージェントのリリースは、一定期間にわたって小さな段階で顧客に展開されます。 このため、リリースがアカウントでアクセス可能になる日付は、元の公開日と一致しない可能性があります。詳細については、この[ステータス ダッシュボード](https://newrelic.github.io/newrelic-browser-agent-release/)をご覧ください。

当社の[ブラウザサポートポリシー](https://docs.newrelic.com/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring/#browser-types)に従い、 Browser v1.311.0は、 Chrome 134-144、Edge 134-144、Safari 18-26、Firefox 137-147のブラウザおよびバージョン範囲向けに構築され、テストされています。 モバイルデバイス向けには、v1.311.0がAndroid OS 16およびiOS Safari 18～26向けにビルドおよびテストされました。
