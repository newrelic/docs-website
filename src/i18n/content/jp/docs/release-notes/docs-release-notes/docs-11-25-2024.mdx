### 新しいドキュメント

* [Amazon MemoryDB 統合では、](/docs/infrastructure/amazon-integrations/aws-integrations-list/amazon-memorydb-monitoring-integration) New Relicとの統合を有効にする方法について説明します。 統合により、Amazon MemoryDB クラスタに関するテレメトリーデータを収集し、受信データを書き込み、ダッシュボードを構築することができます。
* [「アプリケーション ログにカスタム タグ (ラベル) を追加する」では、](/docs/logs/logs-context/custom-tags-agent-forwarder-logs)エージェント転送ログにカスタム ラベルを追加できることについて説明します。

### 主な変更点

* 明確さと組織のために[AWSインテグレーションのドキュメント](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations)を更新しました。 主な変更点としては、主要なAWSインテグレーション方法として CloudWatch Metric Streams を強調すること、コンテンツの統合と合理化によってナビゲーションを強化して冗長性を減らすこと、読みやすさを向上させるためにコピーを編集することなどが挙げられます。
* 正確性を高めるために [事後検証を作成する](/docs/alerts/alert-event-management/postmortems-applied-intelligence/#create-postmortem)手順を更新しました。
* New Relic の 2024 年年次評価に基づいて[、認定、標準、規制](/docs/security/security-privacy/compliance/regulatory-audits-new-relic-services)のコンプライアンス関連の更新を行いました。
* .NET エージェントに .NET 9 と ASP..NET Core 9 の互換性を追加し、サポート対象外の .NET バージョンのサポート スタンスを明確にし、「.NET エージェントの[互換性と要件」](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements)の古い .NET フレームワーク参照を削除しました。
* わかりやすくするために、[ログ転送用の Fluent Bit プラグインを](/docs/logs/forward-logs/fluent-bit-plugin-log-forwarding)書き直しました。
* 正確性を高めるために[CodeStream 通知](/docs/codestream/codestream-settings/account-settings)を更新しました。
* cloudプロバイダーとAPMアプリケーションの間の関係を作成するための[Node.jsエージェント変数の](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#cloud_config)詳細を追加しました。
* [Azureイベント ハブを作成する](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export/#create-event-hub)手順を更新しました。

### マイナーチェンジ

* [「カスタム タグ (ラベル) をアプリケーション ログに追加する」](/docs/logs/logs-context/custom-tags-agent-forwarder-logs)で .NET およびRubyエージェントのバージョンを更新しました。
* [モバイル ログ](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs)で Unity によるログ記録 API の使用例を示しました。
* [.NET エージェントの互換性と要件](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements)が更新されました。
* [Node.js エージェントの互換性と要件](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)が更新されました。
* [インフラストラクチャ イベント](/docs/infrastructure/infrastructure-data/default-infra-data)でのネットワーク デバイスの組み込みと NetworkSample の生成に関するデフォルトの動作と設定ガイダンスを更新しました。
* `localStorage` API の名前を修正し、セッション トラッキングで[セッション トラッキング](/docs/browser/browser-monitoring/page-load-timing-resources/cookie-collection-session-tracking)が機能しない場合に関する情報を更新しました。
* アプリケーションログへのラベルの追加を[有効](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels)または[除外する](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration/#application-logging-forwarding-labels-exclude)ために、 Node.jsエージェント設定の詳細を更新しました。
