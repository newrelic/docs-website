### 新しいドキュメント

* OpenTelemetryを使用したAmazon ECSの監視に関する包括的なガイダンスを提供するために、[OpenTelemetry ECS監視](/docs/opentelemetry/integrations/ecs-monitoring/overview/) を追加しました。

### 主な変更点

* インストレーション、監視、設定、トラブルシューティングの専用セクションを設け、組織化を改善するために[SAPソリューションのドキュメント](/docs/sap-solutions/overview/)を再構築しました。
* GAの提供と強化されたセットアップ手順により、[Kubernetes Windowsノード監視](/docs/kubernetes-pixie/kubernetes-integration/installation/windows/)を更新しました。
* データマスキングと記録コントロールに関する広範な設定オプションを追加して、[Browserセッションリプレイのプライバシー設定](/docs/browser/browser-monitoring/browser-pro-features/session-replay/configuration/customize-privacy-settings/)を更新しました。
* 読み取り専用ユーザーがダッシュボードを作成、更新、削除できなくなったことを反映するように、[ダッシュボード管理](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/)および[ユーザー管理の概念](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/)のドキュメントを更新しました。

### マイナーチェンジ

* 最新の検証済みパッケージバージョンで[.NETエージェントの互換性要件](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/)を更新し、コンテナサポートを明確にしました。
* [PHPエージェントの互換性要件](/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/)を現在サポートされているバージョンで更新し、EOLバージョンを削除しました。
* 最新の設定オプションで[Rubyエージェント設定](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration/)を更新しました。
* アクセス付与コマンドを修正し、[Oracleデータベースインテグレーション](/docs/infrastructure/host-integrations/host-integrations-list/oracle-database-monitoring-integration/)を強化しました。
* [ATPのトラブルシューティング](/docs/opentelemetry/nrdot/atp/troubleshooting/)に関するインテリジェントなストレージパス選択のドキュメントを追加しました。
* [OpenTelemetryを使用したJenkins監視](/docs/infrastructure/other-infrastructure-integrations/monitoring-jenkins-ot/)ドキュメントのインデントを修正しました。
* [Node.jsエージェントの互換性要件](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/)を最新の互換性マトリックスで更新しました。

### 新着投稿

* Windowsノード監視の一般提供を発表するために、[Kubernetes WindowsノードGA](/whats-new/2026/04/whats-new-4-16-windows-ga/)を追加しました。
