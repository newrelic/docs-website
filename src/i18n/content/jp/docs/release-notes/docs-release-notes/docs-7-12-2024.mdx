### 新しいドキュメント

* 新しい分岐[Kubernetes監視インテグレーションインスト](/install/kubernetes/)レーション ドキュメントは、必要な特定のインストレーション情報を見つけるのに役立ちます。
* Apache Airflow 上で実行され、Snowflake を使用して[dbt Cloud](/docs/more-integrations/dbt-cloud/dbt-cloud-integration)データをインストール、構成、監視します。
* 新しい[Kubernetes Agent オペレーターは](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-agent-operator)APMのインストゥルメンテーションを自動化するため、手動で管理する必要がなくなりました。 現在パブリックプレビュー中です。
* iOSエージェントは[、モバイルログ](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-logs)と[watchOS計装を](/docs/mobile-monitoring/new-relic-mobile-ios/watchOS/instrument-watchos)New Relicに送信するためのサポートを追加しました。
* [AWS Lambda のデータが表示されない問題のトラブルシューティングでは、](/docs/serverless-function-monitoring/aws-lambda-monitoring/troubleshooting/troubleshoot-no-data-appears) UI に AWS Lambda のデータが表示されない場合の解決策について説明します。
* [Android エージェントのネイティブ クラッシュ レポートでは、](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-native-crash-reporting)クラッシュ レポーターを有効化、構成、デバッグする方法について説明します。
* [NRQL アラート条件のタイトル テンプレートでは、](/docs/alerts/create-alert/condition-details/title-template)タイトル テンプレートを使用してアラート イベントに意味のあるタイトルを作成する方法について説明します。
* [OpenLLMetry を使用した Traceloop LLM の観察では、](/docs/opentelemetry/get-started/traceloop-llm-observability/traceloop-llm-observability-intro)このOpenTelemetryベースのツールを使用して AI モデルのパフォーマンスを監視する方法について説明します。

### 主な変更点

* [APM -インストゥルメント化された](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-apm-applications-kubernetes)アプリケーションと[OpenTelemetry -](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes)インストゥルメント化されたアプリケーションのリンクに対するコンテンツとスタイルの更新により、可読性が向上し、このセットアップが容易になりました。 また、スタイルと分かりやすさを向上させるために、 [Kubernetes インテグレーション](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration)の紹介も更新しました。
* [データ取り込みの課金やルール](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-prices)など、GB あたりの価格変更を反映するようにドキュメントを更新しました。
* データ ディクショナリに新しい[`MobileApplicationExit`](/attribute-dictionary/?dataSource=Mobile&event=MobileApplicationExit)属性を追加しました。
* [.NETエージェントをインストールする](/install/dotnet/?deployment=azure&azure=azuresiteextension)ためのAzure固有のインストレーション情報を追加しました。
* 2024 年 7 月 3 日、[Kubernetes インテグレーション](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements/#compatibility)はKubernetes v1.25 以前のサポートを終了しました。
* [Meet New Relic AIを大幅に書き直したもので、観察アシスタントが](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-ai)AI ツールを最大限に活用する方法を説明します。
* 新しい`is_integrations_only`設定に関する設定情報を含む[インフラストラクチャエージェント設定](/docs/infrastructure/install-infrastructure-agent/configuration/infrastructure-agent-configuration-settings/#is_integrations_only)を更新しました。 OpenTelemetry Collector や Prometheus ノード エクスポーターなど、インフラストラクチャ エージェント以外のソースからホストをインストゥルメントされた場合は、この設定を有効にします。

### マイナーチェンジ

* 新しい Kubernetes エージェント オペレーターに関連するさまざまな更新。
* OpenTelemetryドキュメントの更新により、いくつかの壊れたアンカー リンクが修正され、タイムアウト設定情報が[OTLP エンドポイント](/docs/opentelemetry/best-practices/opentelemetry-otlp/#payload-limits)に追加され、その他の一貫性が向上しました。
* [AWS PrivateLink](/docs/data-apis/custom-data/aws-privatelink/#regions-zones)を更新し、複数のリージョンをサポートしていることを明確にしました。
* [Node.js エージェントの互換性表](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent/#system)を更新し、Node.js バージョン 22 のサポートをリストしました。
* .NET エージェント Azure トラブルシューティング ドキュメントを[トラブルシューティング カテゴリ](/docs/apm/agents/net-agent/azure-installation/azure-cloud-services-no-data-appears/)に移動しました。
* [/docs/data-apis/ingest-apis/security-data-api](/docs/data-apis/ingest-apis/security-data-api/#req-fields)で`issueId`を更新し、 `entityGuid`を追加しました。
* [Cacti](/docs/infrastructure/host-integrations/host-integrations-list/cacti-integration/)インテグレーションのコード スニペットのエラーを修正しました。
* .NET Lambdaレイヤーを必要としないことを明確にするために、 [Lambda監視のレイヤーレス計装を](/docs/serverless-function-monitoring/aws-lambda-monitoring/enable-lambda-monitoring/enable-serverless-monitoring-aws-lambda-layerless/#dotnet)更新しました。
* 特定のコンテキストでエージェントをオフにする方法を記述するため[に、Ruby エージェントの開始時の制御](/docs/apm/agents/ruby-agent/troubleshooting/control-when-ruby-agent-starts)が更新されました。
* ドキュメント サイト全体のインライン サインアップ フォームを削除しました。
* [Temporal 自己ホスト型監視](/docs/infrastructure/host-integrations/host-integrations-list/temporal-monitoring-integration/#temporal-logs-configuration)統合で Temporal ログを設定および転送する方法に関する手順を追加しました。
* [インフラストラクチャ](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/#os)、 [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#linux)パフォーマンス、 [ネットワークsyslog](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#linux) 、および[ネットワーク フロー データ](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#linux)監視は、2024 年 6 月 30 日に CentOS 7 のサポートを終了しました。
* NerdGraph の 5 秒のデフォルト クエリ タイムアウトを使用して、 [NRQL のレート制限](/docs/nrql/using-nrql/rate-limits-nrql-queries/#query-duration)を更新しました。
* 更新された[アプリケーションが新しいスクリーンショットとサーバーサイド構成の詳細に応答しません](/docs/mobile-monitoring/mobile-monitoring-ui/application-not-responding)。
* [「インフラストラクチャエージェントの CPU フットプリントを減らすトラブルシューティング」](/docs/infrastructure/new-relic-infrastructure/troubleshooting/reduce-infrastructure-agents-cpu-footprint)ガイドの`disable_all_plugins`の構文エラーを修正しました。
* CodeStream[コードレベル メトリクス](/docs/codestream/observability/code-level-metrics/#requirements)for PHPには、PHP Intelephense拡張機能が必要です。
* [New Relicからアプリケーションを削除する 」](/docs/apm/new-relic-apm/maintenance/remove-applications-new-relic/#wait-longer)では、 New Relicそのアプリから 12 時間以上データを受信していない場合にアプリケーションを削除できることを明確にしました。
* Kubernetes のデータ取り込み削減とデータ ガバナンスのドキュメントを、[取り込みデータの削減に関する](/docs/kubernetes-pixie/kubernetes-integration/installation/reduce-ingest)1 つのページに統合しました。
* 現在サポートしているいくつかの新しい AWS リージョンとそのイメージリポジトリを使用して、[ログ転送用の AWS FireLens プラグインを](/docs/logs/forward-logs/aws-firelens-plugin-log-forwarding/#firelens-plugin)更新しました。
* 現在の通知ログの列を含む[イベント ワークフロー](/docs/alerts/get-notified/alert-event-workflows/#navigate-notification-log)を更新しました。
* [Javaエージェントの互換性と要件](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent/#JDBC)が更新され、 Javaエージェント バージョン 8.13.0 以降が HikariCP 2.4.0 から最新までをサポートしていることがわかりました。 (これは、HikariCP インキュベーター モジュールに代わるものです。)
* .NET エージェントは現在、 [`Elastic.Clients.Elasticsearch`](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#app-frameworks-core)バージョン 8.14.5 をサポートしています。
