### 新しいドキュメント

* [AWS SQS メッセージキュー計装は、](/docs/apm/agents/java-agent/instrumentation/instrument-aws-sqs-message-queues)バージョン 2.1.0 以降、SQS メッセージのディストリビューティッド（分散）トレーシング ヘッダーをサポートしています。 メッセージ受信者の操作のためのカスタムインストゥルメンテーションに関するガイダンスが含まれています。
* [まとめ Intelligence Tool は、](/docs/agentic-ai/tools/alert-intelligence-tool)パートナーとのエージェント統合を通じて、アプリケーションの健全性とパフォーマンスを評価する機能を強化します。
* [Apache Kafka 向けの Confluent Cloud マネージド ストリーミングでは、](/docs/infrastructure/other-infrastructure-integrations/confluent-cloud-integration) Kafka クラスターとトピックの監視、および Confluent Cloud のアラートの設定に関するガイダンスが提供されます。
* [セッションリプレイ用のNew Relic AI概要は、セッションリプレイ イベント](/docs/browser/browser-monitoring/browser-pro-features/session-replay/nrai-summary)トレイル データを要約してマッピングし、ユーザーのブラウザー ジャーニーにおけるエラーと成功をより深く理解できるようにします。
* [New Relic AI によるレスポンス インテリジェンスは、](/docs/alerts/incident-management/response-intelligence-ai)レスポンス インテリジェンスを備えた New Relic AI を活用することで、問題やインシデントの解決時間を短縮します。
* [Browser](/docs/browser/browser-monitoring/browser-pro-features/browser-logs/overview)短期ブラウザ コンソール ログは、監視性を強化し、デバッグを簡素化し、ユーザー エクスペリエンスを向上させます。
* [Transaction 360 の概要では、](/docs/apm/transactions/workload-performance-monitoring/introdution)特定の問題や質問に関連するすべてのエンティティとデータを自動的にキャプチャして相関させることにより、アプリケーションの監視とトラブルシューティングに革新的なアプローチを提供します。
* [AWS in Context は、](/docs/logs/logs-context/aws-logs-in-context) AWS環境内でのトラブルシューティングの効率とシステムの監視を強化する、包括的な統合ログ管理ソリューションを提供します。
* [Agent Control](/docs/new-relic-control/agent-control/reinstrumentation)を使用して現在のコンピュータを管理および再インストールします。
* [Agentic Control](/docs/new-relic-control/agent-control/overview) は、New Relic New RelicOpenTelemetryFleet Controlと通信してインフラストラクチャ レベルの計装の健全性をリモートで設定、更新、監視することで、 エージェントの他の および ディストリビューションを管理する軽量のエージェント スーパーバイザです。
* [Pipeline制御を](/docs/new-relic-control/pipeline-control/overview)使用すると、モニター エンティティからNew Relicへのデータ フローを管理できます。
* [Dockerコンテナ](/docs/infrastructure/host-integrations/host-integrations-list/docker-container-monitoring-integration)統合を監視し、 DockerコンテナからNew Relicにメトリクスをレポートします。
* の[Lambda 応答監視 機能は、各呼び出しのパフォーマンスと動作を内部サイトに取得します。](/docs/serverless-function-monitoring/aws-lambda-monitoring/response-stream)Node.js

### 主な変更点

* で[ サポートされるエンティティ関係 として](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#supported-entity-relationships)[ 、AmazonCloudWatch Metric Streams](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-cloudwatch-metric-streams) 、 [ElastiCache (Memcached)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#elasticache-memcached) 、および[ Amazon MQ (ActiveMQ)](/docs/opentelemetry/best-practices/opentelemetry-best-practices-resources/#amazon-mq-activemq) を追加しました。OpenTelemetry
