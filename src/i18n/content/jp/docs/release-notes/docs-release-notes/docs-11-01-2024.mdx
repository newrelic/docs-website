### 新しいドキュメント

* [PHPエージェントのオーバーヘッド削減のヒントには、](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips/)システム上のエージェントのオーバーヘッドを削減するために変更できる設定がリストされています。

### 主な変更点

* 明確さと完全性を実現するために、再構成して書き直しました。 [独自のインテグレーションを構築: New Relic Flex](/docs/infrastructure/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) 。
* [Scala計装](/docs/apm/agents/java-agent/frameworks/scala-installation-java) に[Scala Futures](/docs/apm/agents/java-agent/frameworks/scala-installation-java/#futures) セクションを追加しました。
* 「脆弱性管理（脆弱性管理）[を始める」](/docs/vulnerability-management/overview) 、 [「脆弱性の優先順位付けを理解する」](/docs/vulnerability-management/understanding-prioritization) 、 [「セキュリティ チームとしての脆弱性を管理する](/docs/vulnerability-management/security-workflow)」を含む 3 つの脆弱性管理（脆弱性管理）トピックを更新しました。
* [`cancelAsyncQuery`](/docs/apis/nerdgraph/examples/async-queries-nrql-tutorial/#example-cancel-query)ミューテーションの使用方法に関する情報を追加しました。

### マイナーチェンジ

* Lambda関数と互換性のあるNew RelicのPythonおよびRubyイメージへのリンクを更新しました。
* [NRQL アラート条件の作成](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions)で、スタンドアロンのシグナル損失クエリのコストの高い性質について説明しました。
* [New Relic CLI オプション](/docs/new-relic-solutions/build-nr-ui/newrelic-cli/#nr-cli-commands-options)を更新しました。
* パフォーマンスのオーバーヘッドを明確にするために、 [PHPエージェントのオーバーヘッド削減のヒント](/docs/apm/agents/php-agent/troubleshooting/agent-overhead-reduction-tips)の問題文を改良しました。
* [トレース制限の](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works/#trace-limits)説明を変更し、これらの制限を制限 UI で表示できるようになったことを示しました。
* [`NrAiIncident`イベント プロパティ](/docs/alerts/create-alert/condition-details/alert-event-attributes/#NrAiIncident-attributes)の`description`および`incidentId`プロパティの説明を更新しました。
* 必要な新しいRuby拡張機能を反映するために[コードレベル メトリクス](/docs/codestream/observability/code-level-metrics)を更新しました。
* Kubernetes監視ドキュメントを更新し、 New Relic Kubernetes 1.26 ではなく 1.31 をサポートするようになったことを示しました。
* [メッセージング](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#messaging-core)テーブルに新しい`AWSSDK.DynamoDBv2`計装のエントリを追加しました。
* Node.js エージェントの[互換性要件](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)が更新されました。
* [型変換](/docs/nrql/nrql-syntax-clauses-functions/#type-conversion)で`numeric()`関数を使用しない場合を明確にしました
* に積算ペース登録の新たな制限に関する情報を追加しました。
