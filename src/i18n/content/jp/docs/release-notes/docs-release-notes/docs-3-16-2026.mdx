### 新しいドキュメント

* 組織レベルの通知先を紹介するため、[クロスアカウント通知先の概要](/docs/apis/nerdgraph/examples/Cross-account-destination/overview)を追加しました。
* New Relic組織内のすべてのアカウントにわたる通知宛先の作成と管理に関する包括的なガイダンスを提供するために、[「アカウント間宛先の管理」](/docs/apis/nerdgraph/examples/Cross-account-destination/manage-cross-account-destinations)機能を追加しました。
* [メタデータ タグを使用したキーのリスト](/docs/synthetics/synthetic-monitoring/private-locations/manage-private-location-keys)、フィルタリング、編集のためのUI可視化機能を含む、外形監視でプライベートロケーション キーを管理するためのガイダンスを提供する「プライベートロケーション キーの管理」を追加しました。

### 主な変更点

* テンプレート変数の可視性が向上し[、 Dashboards APIを使用したウィジェットとチャートの作成が](/docs/apis/nerdgraph/examples/create-widgets-dashboards-api)更新されました。
* [NerdGraph APIを使用してDashboards](/docs/apis/nerdgraph/examples/nerdgraph-dashboards)更新し、テンプレート変数の表示機能を拡張しました。
* [ダッシュボードテンプレートの変数](/docs/query-your-data/explore-query-data/dashboards/dashboard-template-variables)に、表示設定に関するガイダンスを追加しました。
* テンプレート変数管理機能の改善により、[ダッシュボードの管理が](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard)容易になりました。
* エラー解決のための包括的なガイダンスを含む、最新の[トラブルシューティングチャートのエラー情報を](/docs/query-your-data/explore-query-data/dashboards/troubleshooting-chart-errors)掲載しました。
* 更新：チャートの使用方法に関するドキュメントが強化され、[チャートをより効果的に活用できるようになりました](/docs/query-your-data/explore-query-data/use-charts/use-your-charts)。
* OpenTelemetry取り込み最適化のための大幅な機能強化により[、 Kubernetes Strimzi Kafka 統合](/docs/opentelemetry/integrations/kafka/kubernetes-strimzi)が更新されました。
* OpenTelemetry取り込み最適化のための大幅な機能強化により、[自己ホスト型 Kafka 統合](/docs/opentelemetry/integrations/kafka/self-hosted)が更新されました。
* カスタムアトリビュートの配列サポートにより[、.NET のカスタムアトリビュート](/docs/apm/agents/net-agent/attributes/custom-attributes-net)が更新されました。
* カスタムアトリビュートの配列サポートを含む[.NET エージェントAPI](/docs/apm/agents/net-agent/net-agent-api/net-agent-api)を更新しました。
* [アプリケーションの概要を](/docs/vulnerability-management/applications/overview).NET SRX (Security Review Experience) 統合の詳細で更新しました。
* [統合の概要](/docs/vulnerability-management/getting-started/integrations/overview)と .NET SRX (セキュリティ レビュー エクスペリエンス) 統合の詳細を更新しました。
* OpenShiftバージョン4.20をサポートする、 [OpenTelemetryおよびKubernetesの高度な設定](/docs/kubernetes-pixie/k8s-otel/advance-config)を更新しました。
* OpenShift バージョン 4.20 をサポートする[OpenTelemetryを使用したKubernetesインテグレーションのインストールを](/docs/kubernetes-pixie/k8s-otel/install)更新しました。
* OpenShift バージョン 4.20 のサポートにより、 [Kubernetesインテグレーションの互換性と要件](/docs/kubernetes-pixie/kubernetes-integration/get-started/kubernetes-integration-compatibility-requirements)が更新されました。
* AppLaunch/Cold メトリクス制限のある[iOS アプリのアプリ リリース時間](/docs/mobile-monitoring/new-relic-mobile-ios/configuration/app-launch-times-ios-apps)を更新し、 NRQL書き込みを修正しました。
* AppLaunch/Cold メトリクスの制限による[モバイル監視の設定](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/configure-settings)を更新しました。
* 設定が大幅に改善され、 [SREエージェントのセットアップ](/docs/agentic-ai/sre-agent/sre-agent-setup)が更新されました。
