### 新しいドキュメント

* 拡張された Berkeley Packet Filter 監視機能に関する包括的なガイダンスを提供するために、 [eBPF eAPM および eBPF ネットワーク監視](/docs/ebpf/overview)ドキュメントを追加しました。
* APM + Lambda Serverless の統合についてユーザーをガイドする[APM エクスペリエンス ドキュメントにアップグレードを](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/upgrade-to-apm-experience)追加しました。

### 主な変更点

* [セッションとデバイスの更新 影響を受けるメトリクスのドキュメント](/docs/errors-inbox/error-impacted)が大幅に改訂され、ブラウザとモバイル アプリケーションのエラー追跡機能が強化されました。

* 最新の互換性情報を反映して[Node.js 互換性レポート](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)を更新しました。

* 現在のフレームワーク サポートの詳細を記載した[Java エージェントの互換性ドキュメント](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent)を更新しました。

* 最新のフレームワーク互換性情報を反映して[.NET 互換性ドキュメント](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/)を更新しました。

* エージェント設定ガイダンスを改善するために[Ruby設定ドキュメント](/docs/apm/agents/ruby-agent/configuration/ruby-agent-configuration)を更新しました。

* 組織を改善し、さまざまなプラットフォームにわたる影響メトリクスを包括的にカバーするために、[エラー ユーザーに影響を与えるドキュメント](/docs/errors-inbox/error-impacted)を更新しました。

  * メトリクスに影響を与えたセッションとデバイスに関する[ブラウザのエラー追跡](/docs/errors-inbox/browser-tab)ドキュメントが強化されました。
  * メトリクスに影響を与えたセッションとデバイスに関する[モバイル エラー追跡](/docs/errors-inbox/mobile-tab)ドキュメントの強化。

* [外形監視パブリックミニオン IP を](/docs/synthetics/synthetic-monitoring/administration/synthetic-public-minion-ips)監視インフラストラクチャの新しい IP アドレス範囲で更新しました。

### マイナーチェンジ

* AIモニタリングのネームスペースの保持値を[データ保持期間のドキュメント](/docs/data-apis/manage-data/manage-data-retention)に追加しました。
* シミュレーションとリアルタイムの相関ガイダンスを明確にするために、[意思決定ドキュメント](/docs/alerts/organize-alerts/change-applied-intelligence-correlation-logic-decisions)を更新しました。
* 精度情報が強化された[uniqueCount 関数](/docs/nrql/nrql-syntax-clauses-functions/)のハッシュ衝突ドキュメントを修正しました。
