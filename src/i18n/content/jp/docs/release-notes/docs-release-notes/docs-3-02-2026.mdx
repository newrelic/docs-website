### 主な変更点

* 新しい設定オプション、ネットワーク要件、トラブルシューティング ガイダンスを含む、包括的なWindowsホストのサポートを備えた[New Relic Control エージェント コントロールのドキュメント](/docs/new-relic-control/agent-control/overview)を更新しました。
* [PHP ブラウザ監視ドキュメント](/docs/apm/agents/php-agent/features/browser-monitoring-php-agent)が強化され、ブラウザ監視データにカスタムアトリビュートを追加するための詳細な手順が追加されました。
* ハイブリッドエージェントサンプラー設定例を含む[Pythonエージェント設定ドキュメント](/docs/apm/agents/python-agent/configuration/python-agent-configuration)を更新しました。

### マイナーチェンジ

* [AWS CloudWatch 統合ドキュメント](/install/aws-cloudwatch)を更新して、マルチリージョン監視のために各AWSアカウントにリージョンごとに 1 つのシンクが必要であることを明確にしました。
* [eBPF Kubernetesインストレーション](/docs/ebpf/k8s-installation)および[Linux インスト](/docs/ebpf/linux-installation)レーションのドキュメントをNew Relicアプリ名の設定情報で更新しました。
* Kubernetes環境とセルフホスト環境の両方に対する NRDOT Collectorサポートを備えた[RabbitMQ OpenTelemetry インテグレーション](/docs/opentelemetry/integrations/rabbitmq/overview)が更新されました。
* [高セキュリティ モードのドキュメント](/docs/apm/agents/manage-apm-agents/configuration/high-security-mode)を更新しました。
* [ポート監視統合ドキュメント](/docs/infrastructure/host-integrations/host-integrations-list/port-monitoring-integration)を更新しました。
* [チームのドキュメント](/docs/service-architecture-intelligence/teams/teams)を更新しました。
* [チーム管理ドキュメント](/docs/service-architecture-intelligence/teams/manage-teams)を更新しました。
* [Java エージェントの互換性に関するドキュメント](/docs/apm/agents/java-agent/getting-started/compatibility-requirements-java-agent)を更新しました。
* [Node.js エージェントの互換性に関するドキュメント](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)を更新しました。

### 新機能

* [「What&apos;s New for New Relic Control Windows サポート」](/whats-new/2026/02/whats-new-23-02-windows-support/)を追加し、Windows ホスト サポートと New Relic Control (パブリック プレビュー) でのエージェント選択の強化についてお知らせしました。
* [Collectorオブザーバビリティの新機能](/whats-new/2026/02/whats-new-02-24-collector-observability/)を更新しました。
