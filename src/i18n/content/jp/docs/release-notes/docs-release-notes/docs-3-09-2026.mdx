### 新しいドキュメント

* AI を活用したSRE機能を使用するための包括的なガイダンスを提供するために、 [SREエージェントの概要](/docs/agentic-ai/sre-agent/overview)を追加しました。
* SREエージェントを構成するための段階的な手順を提供するために、SRE [SREのセットアップを](/docs/agentic-ai/sre-agent/sre-agent-setup)追加しました。
* SREエージェント機能の活用に関するガイダンスを提供するために「 SRE [SREの使用」を](/docs/agentic-ai/sre-agent/use-sre-agent)追加しました。
* [AI エージェントのパフォーマンスを監視および分析するための手順を提供する View AI エージェント](/docs/ai-monitoring/explore-ai-data/view-ai-agents)を追加しました。
* Microsoft SQL Server データベースの監視に関する包括的なガイダンスを提供するために、 [OpenTelemetryを使用した MSSQL 監視を](/docs/opentelemetry/database/otel-mssql)追加しました。
* [Oracleデータベース監視の包括的なガイダンスを提供するために、 OpenTelemetryを使用した](/docs/opentelemetry/database/otel-oracledb)Oracleデータベース監視を追加しました。
* [データベース サービスの識別と構成に関するガイダンスを提供するデータベース サービスの識別](/docs/opentelemetry/database/database-service-identification)を追加しました。

### 主な変更点

* CLI、GitHub Actions、NerdGraph の新しい設定セクションを含む、組織と使いやすさを改善するために[変更追跡 (変更追跡機能) ドキュメントを](/docs/change-tracking/overview)再構築しました。
* 更新された[変更追跡 (変更追跡機能) は、明確さを改善し、新しい視覚化機能を追加するために広範囲に改訂されたデータを表示および分析します](/docs/change-tracking/view-analyze-data)。
* 包括的な例を含む[変更追跡 (変更追跡機能) データ](/docs/change-tracking/query-data)を更新し、デプロイメント データをクエリするためのガイダンスを改善しました。
* [AWSの複数アカウントに関するドキュメント](/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/aws-multi-account)を改訂し、複数のAWSアカウントを管理するためのより明確なガイダンスを提供するようにしました。
* [eBPFネットワークメトリクス](/docs/ebpf/network-metrics)を更新し、ドキュメントを大幅に改善するとともに、新たな監視機能を追加しました。
* eBPF によるアプリケーション[パフォーマンス監視を強化するための eBPF APM](/docs/ebpf/ebpf-apm)インテグレーション ドキュメントを追加しました。

* [ワークフロー自動化通知の例文](/docs/workflow-automation/setup-and-configure/actions-catalog/newrelic/newrelic-notification)を更新し、書式と分かりやすさを改善しました。
* [ワークフロー自動化の定義スキーマ](/docs/workflow-automation/workflow-automation-apis/definition-schema)を更新し、例とドキュメントを強化しました。
* [ワークフロー自動化のベストプラクティス](/docs/workflow-automation/limitations-and-faq/workflow-best-practices)を更新し、ガイダンスを改善しました。
* [Kubernetesのコスト属性を](/docs/cci/financial-intelligence/nrql-for-cci)、修正されたイベントタイプコードで更新しました。
* 非推奨となったセッションタイムアウト設定に関するドキュメントを削除したため、 [認証ドメインのドキュメント](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more)を更新し、セッションタイムアウト設定に関するガイダンスを追加しました。
* [ワークフロー自動化の制限値を、](/docs/workflow-automation/limitations-and-faq/workflow-limits)最新のシステム制限情報に基づいて更新しました。
* [レガシーKubernetesイメージのEOLアナウンス](/eol/2026/03/eol-03-31-26-legacy-k8s-images)と非推奨タイムラインを追加しました。
