### 新しいドキュメント

* OpenTelemetryを使用した監視Elasticsearchの包括的なガイダンスを提供するために、 [Elasticsearch OpenTelemetry概要を](/docs/opentelemetry/integrations/elasticsearch/elasticsearch-otel-integration-overview)追加しました。
* [Elasticsearch OpenTelemetryの](/docs/opentelemetry/integrations/elasticsearch/elasticsearch-otel-integration-install)ステップバイステップのインストレーション手順を提供するために、 Elasticsearch OpenTelemetryインストレーションを追加しました。
* 監視にトラブルシューティングのガイダンスを提供するために、 [Elasticsearchシューティングを追加しました。](/docs/opentelemetry/integrations/elasticsearch/troubleshooting)Elasticsearch
* [自動 dSYM アップロードの失敗 (Xcode 15.3+)](/docs/mobile-monitoring/new-relic-mobile-ios/troubleshoot/dSYM-upload-tools)が追加され、Xcode 15.3 以降での iOS エージェント dSYM アップロードの失敗に関するトラブルシューティング ガイダンスが提供されます。

### 主な変更点

* 予算、推奨事項、統合などの機能の組織と発見可能性を向上させるために[Cloud Cost Intelligence (CCI) ドキュメントを](/docs/cci/overview)再構築しました。
* [ワークフロー自動化アクション カタログ](/docs/workflow-automation/setup-and-configuration/actions-catalog/actions-catalog)が更新され、アクション ドキュメントが大幅に改訂され、New Relic データの取り込みと NRDB クエリ用の 2 つの新しいアクションが追加されました。
* OpenTelemetry サポート ドキュメントの構造が改善され、アカウントと課金のサポート セクションが追加され、 [グローバル テクニカル サポート サービスが](/docs/licenses/license-information/general-usage-licenses/global-technical-support-offerings)更新されました。
* 簡素化された設定例と改善されたドキュメント構造により、 [「Bring your own cache」](/docs/distributed-tracing/infinite-tracing-on-premise/bring-your-own-cache)が更新されました。
* 最新バージョンの変更を反映するために[Databricks 統合](/docs/infrastructure/other-infrastructure-integrations/databricks-integration)が更新されました。

### 新機能

* [命名法の変更 - 「インシデント」は「まとめイベント」に](/whats-new/2026/01/whats-new-01-23-incidents-to-alert-events)- 2026 年 2 月 16 日より、 New RelicプラットフォームUI全体で「インシデント」の名前を「まとめイベント」に変更し、わかりやすくし、標準の業界命名法に合わせます。
* [多要素認証によるアカウントのセキュリティ強化](/whats-new/2026/01/whats-new-01-28-2026-mfa)- すべてのユーザーに多要素認証 (MFA) を要求することで、認証ドメインを保護します。
