### 新しいドキュメント

* [カーディナリティ管理では、](/docs/data-apis/ingest-apis/metric-api/cardinality-management)組織のカーディナリティの使用状況を管理および確認するための管理ツールが提供されます。

### 主な変更点

* 新しいスクリーンショットとさまざまな実装方法を説明する表を使用して、[エラー受信箱で影響を受けたユーザーを追跡する](/docs/errors-inbox/error-users-impacted/)ための手順を改善しました。
* [OpenTelemetry -インストゥルメントされたアプリケーションをKubernetesにリンクし](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes)、現在のOpenTelemetryセットアップと設定情報を使用して[Kubernetesクラスタを探索します](/docs/kubernetes-pixie/kubernetes-integration/understand-use-data/kubernetes-cluster-explorer)。
* [ミュート ルールを書き直しました: 通知を抑制して、](/docs/alerts/get-notified/muting-rules-suppress-notifications)集中イベントや問題を作成、管理、抑制、レビューする方法を説明します。
* ナビゲーションと検索しやすさを向上させるために、 [Kubernetes トラブルシューティング](/docs/kubernetes-pixie/kubernetes-integration/troubleshooting/overview)ドキュメントを再編成しました。
* 明確さと完全性のために[OpenTelemetry](/docs/opentelemetry/opentelemetry-introduction)ドキュメントを改訂しました。
* [PHP エージェント設定](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-vulnerability-management)で Vulnerability Management（脆弱性管理）パッケージ検出用の Composer ランタイムAPIを構成する手順を追加しました。

### マイナーチェンジ

* いくつかのドキュメントを更新し、リンク、エクスポートされた画像、PDF 経由で共有された場合、ビルボード チャートが正しくレンダリングされないことをお知らせします。
* Linux コマンドと Windows シェル コマンドが混在していたいくつかのドキュメントを修正しました。
* いくつかのレガシードキュメントを削除し、現在のバージョンへのリダイレクトを追加しました。
* 現在の UI に合わせて[アラート条件を](/docs/alerts/create-alert/create-alert-condition/alert-conditions)最新のものにしました。
* データ ディクショナリに新しい[`PageViewTiming`](/attribute-dictionary/?event=PageViewTiming)属性を追加しました。
* Meraki ポーリング API と KTranslate で ICMP を使用するためのデフォルト設定に関連するネットワーク パフォーマンス監視ドキュメントが更新されました。
* [.NET 構成設定](/docs/apm/agents/net-agent/configuration/net-agent-configuration)に`NewRelic.Labels`を追加しました。
* [「エラー受信箱の影響を受けたユーザーを追跡する」](/docs/errors-inbox/error-users-impacted/#set-users)に、PHP `newrelic_set_user_id` APIコールを追加しました。
* [カスタムイベント データのデータ要件と制限に、](/docs/data-apis/custom-data/custom-events/data-requirements-limits-custom-event-data)禁止されている新しいイベント タイプの値を追加しました: `EntityRelationship` 、 `RelationshipUpdate` 、および`Relationship[0-9]`文字列プレフィックス。
* [「New Relic と Terraform の使用開始」](/docs/infrastructure-as-code/terraform/terraform-intro)では、Terraform で CodeStream を使用する方法について説明しました。
