### 新しいドキュメント

* 2つの新しいGA機能に関する[サービスレベル管理](/docs/service-level-management/intro-slm)のドキュメントを追加しました：SLI計算から不良イベントを除外する計画的なダウンタイムをスケジュールするためのMaintenance Windows、および属性ごとにSLOコンプライアンスとエラーバジェットを分類するためのFacet Compliance。

### 主な変更点

* 破壊的変更、削除された計装、更新されたサポート対象の最小バージョンを含む、v14の移行ガイダンスを[Node.jsエージェントの更新](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent)に追加しました。
* v14でのスタンドアロンプラグインから従来の計装への`@apollo/server`計装の移行を反映するように、[Node.js用Apollo Serverプラグイン](/docs/apm/agents/nodejs-agent/extend-your-instrumentation/apollo-server-plugin-nodejs)を再構築しました。
* 新しい設定オプションとインストレーション手順を含むv1.4.1リリースの内容で、[Kubernetes向けNew Relic eBPFエージェントのインストール](/docs/ebpf/k8s-installation)および[Linuxホスト向けNew Relic eBPFエージェントのインストール](/docs/ebpf/linux-installation)を更新しました。

### マイナーチェンジ

* ダッシュボードとチャートの共有ドキュメントを再構成し、[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/share-dashboards-publicly/)と[チャート](/docs/query-your-data/explore-query-data/dashboards/share-charts-publicly/)の公開共有用の専用ページを設けました。
* パフォーマンスリスクインボックスからパフォーマンスリスクグループを取得するための`list_entity_performance_risk_groups`ツールを[MCPツールリファレンス](/docs/agentic-ai/mcp/tool-reference)に追加しました。
* 追加のコンテキストを提供する重要なコールアウトを追加して、[setMaxEventPoolSize](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/set-max-event-pool-size) Mobile SDKリファレンスを更新しました。
* [.NETエージェントの互換性要件](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements)を、サポートされている最新のフレームワークバージョンで更新しました。
* 現在のダッシュボード名を反映するように、[NGINXインテグレーション](/docs/opentelemetry/integrations/nginx/find-and-query-your-data)および[NGINX Plusインテグレーション](/docs/opentelemetry/integrations/nginx-plus/nginx-plus-otel)のドキュメントのnginxダッシュボード名を修正しました。
* すべてのフルプラットフォームユーザーに追加費用なしでSAIの機能が含まれることを明確にするため、[New Relic Oneの価格と請求](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing)および[Service Architecture Intelligenceの導入](/docs/service-architecture-intelligence/getting-started)を更新しました。
* Chromeのバージョン参照を「最新」に一元化するため、[新しいランタイム](/docs/synthetics/synthetic-monitoring/using-monitors/new-runtime)、[モニターランタイムの管理](/docs/synthetics/synthetic-monitoring/using-monitors/manage-monitor-runtimes)、および[ランタイムアップグレードのトラブルシューティング](/docs/synthetics/synthetic-monitoring/troubleshooting/runtime-upgrade-troubleshooting)を更新しました。
* ランタイム移行の具体的なEOL日を記載して、[外形監視ランタイム移行ガイド](/docs/synthetics/synthetic-monitoring/private-locations/runtime-transition-guide)を更新しました。
* 内部アプリ名の参照を削除して、[NRQLの構文、句、関数](/docs/nrql/nrql-syntax-clauses-functions)のフォーマットの問題を修正しました。
* [Node.jsエージェントの更新](/docs/apm/agents/nodejs-agent/installation-configuration/update-nodejs-agent)のコードブロックのインデントを修正しました。

### 新着投稿

* サービスレベル管理におけるスケジュールされたメンテナンスWindowsとファセットベースのSLOコンプライアンストラッキングの一般提供（GA）開始をお知らせするために、[SLMアップデート：メンテナンスWindowsとファセットコンプライアンスが一般提供されました](/whats-new/2026/05/whats-new-05-18-slm-maintenance-windows-facet-compliance)を追加しました。
