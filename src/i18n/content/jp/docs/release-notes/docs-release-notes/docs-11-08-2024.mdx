### 新しいドキュメント

* [FOSSA インテグレーションでは、](/docs/vulnerability-management/integrations/fossa/)この新しいインテグレーションが、オープン ソース ライブラリに悪用可能な脆弱性を示すことで、アプリケーションのセキュリティ イニシアチブをどのようにサポートするかについて説明します。

### 主な変更点

* [インフラストラクチャ ドキュメント カテゴリ](/docs/infrastructure/introduction-infra-monitoring/)を再編成し、URL をリポジトリ内の場所に合わせて調整しました。これらのドキュメントを簡単に見つけて作業できるようにする舞台裏での作業の一部です。
* [.NET エージェントAPI](/docs/apm/agents/net-agent/net-agent-api/net-agent-api/)の大幅な書き換えにより、古い情報が削除され、一部のコンテンツが再編成され、サンプル コードが更新され、API の動作に関する詳細が追加され、サンプル コード リポジトリへのリンクが追加され、ディストリビューティッド（分散）トレーシング API に ID タグが追加されました。

### マイナーチェンジ

* DynamoDB に[.NET エージェントの互換性と要件](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements/#database-core)を追加しました。
* タグ`hostStatus: shutdown`は、信号喪失時のイベントの開始を防ぎます。 [NRQL アラート条件の作成](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/)と[信号損失およびギャップの埋め込みに関する NerdGraph チュートリアルを](/docs/apis/nerdgraph/examples/nerdgraph-api-loss-signal-gap-filling/)更新しました。
* 改善されたデータ集約により、[デフォルトのインフラストラクチャモニタリングデータが](/docs/infrastructure/infrastructure-data/default-infra-data/)更新されました。
* [Kubernetes の廃止通知](/eol/2024/10/deprecation-notice-v1.26-and-lower/)を追加しました。
* [インフラストラクチャ作成時の「ホストがレポートしていない」条件の](/docs/infrastructure/infrastructure-alerts/create-infrastructure-host-not-reporting-condition/)読みやすさが向上しました。
* [レイヤーレス スケジューリング](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/enable-serverless-monitoring-aws-lambda-layerless/)に、ES モジュールの例と、拡張専用レイヤーをインストールするための新しい手順が追加されました。
* [AIモニタリングintroの紹介を](/docs/ai-monitoring/intro-to-ai-monitoring/)書き直し、スクリーンショットを更新しました。
