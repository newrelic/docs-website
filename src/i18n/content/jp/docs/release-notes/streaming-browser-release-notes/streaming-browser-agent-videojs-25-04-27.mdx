## 主な変更点

* **エラー レポートの更新**: 以前はエラーの説明を識別するために使用されていた`errorName`属性は非推奨になりました。`errorMessage`属性は現在、詳細なエラーの説明を伝えるための標準となっています。今後は、エラー情報に依存する監視システムやログ システムを更新して、 `errorMessage`属性を利用できるようにしてください。

## アップグレードノート

* エラー分析に`errorName`属性に依存しているユーザーは、継続的な機能と詳細なエラー情報へのアクセスを確保するために、 `errorMessage`属性への移行を計画する必要があります。
