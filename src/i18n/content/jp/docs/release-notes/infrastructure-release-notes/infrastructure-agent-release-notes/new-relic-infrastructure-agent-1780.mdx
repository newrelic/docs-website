エージェントの新しいバージョンがリリースされました。標準的な手順に従って、[infrastructureエージェントをアップデート](/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent)してください。New Relicでは、エージェントを定期的に、少なくとも3か月ごとにアップグレードすることを推奨します。本リリースより、サポートされる最も古いバージョンは[infrastructureエージェント 1.65.3](/docs/release-notes/infrastructure-release-notes/infrastructure-agent-release-notes/new-relic-infrastructure-agent-1653/)です。

## かわった

* 二重クローズ操作によって引き起こされるパニック状態を修正しました。[2270](https://github.com/newrelic/infrastructure-agent/pull/2270)

* ローカルサーバーのバインディング動作を修正し、デフォルトで厳密にlocalhostにバインドするようにしました。[2271](https://github.com/newrelic/infrastructure-agent/pull/2271)

* 依存関係の更新：

  * `newrelic/nri-flex`を`v1.18.8`に更新しました。[#2278](https://github.com/newrelic/infrastructure-agent/pull/2278)
  * `newrelic/nri-winservices`を`v1.5.3`に更新しました。[#2279](https://github.com/newrelic/infrastructure-agent/pull/2279)
  * `newrelic/nri-prometheus`を`v2.29.4`に更新しました。[#2276](https://github.com/newrelic/infrastructure-agent/pull/2276)
  * `newrelic/nri-docker`を`v2.8.1`に更新しました。[#2277](https://github.com/newrelic/infrastructure-agent/pull/2277)
  * コアエージェントの依存関係を`v1.26.5`に更新しました。[#2275](https://github.com/newrelic/infrastructure-agent/pull/2275)
  * セキュリティ脆弱性CVE-2026-39820、CVE-2026-42499、CVE-2026-33814、CVE-2026-42501、CVE-2026-33811、およびCVE-2026-42504を解決するために、`newrelic-fluent-bit-output`を`v3.7.0`（Linux）に更新しました。[#2280](https://github.com/newrelic/infrastructure-agent/pull/2278)
