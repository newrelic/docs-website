### 新機能

* 新しいディストリビューティッド（分散）トレーシング サンプラー、TraceId Ratio-Based Sampler を追加します。これは[OpenTelemetry実装](https://github.com/open-telemetry/opentelemetry-dotnet/blob/main/src/OpenTelemetry/Trace/Sampler/TraceIdRatioBasedSampler.cs)に基づいています。 このサンプラーは、TraceId (ランダムな GUID) と構成されたサンプリング率 (サンプリングされるトレースの目標パーセンテージ) を使用してサンプリングの決定を行います。 詳細については、 [OpenTelemetry 仕様](https://opentelemetry.io/docs/specs/otel/trace/tracestate-probability-sampling/)を参照してください。( [#3211](https://github.com/newrelic/newrelic-dotnet-agent/issues/3211) ) ( [63ca603](https://github.com/newrelic/newrelic-dotnet-agent/commit/63ca603ca6cb86110431413c03afbf8a8c8e9367) )

### 修正

* MSSQL 接続文字列の解析で例外がスローされ、データストア計装が無効になる場合がある ( [#3263](https://github.com/newrelic/newrelic-dotnet-agent/issues/3263) ) ( [cdf000b](https://github.com/newrelic/newrelic-dotnet-agent/commit/cdf000b73fe4beaa73fa4bfe3d338f8eb6698cf1) )
* Kafka の「Consume」計装に関する問題を解決して、自動計装がカスタムインストゥルメンテーションと連携して動作するようにします。 ( [#3257](https://github.com/newrelic/newrelic-dotnet-agent/issues/3257) ) ( [1dbd1c5](https://github.com/newrelic/newrelic-dotnet-agent/commit/1dbd1c51aeaf6e2909d23b078029ee78b721b3bc) )

### エージェントの更新

* [.NET エージェントを更新する](/docs/agents/net-agent/installation-configuration/update-net-agent)には、標準の手順に従ってください。
* 特に古いエージェントを使用している場合は、主要な変更点のリストと[レガシー .NET エージェントの更新](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net)手順を確認してください。

最新のエージェント バージョンが利用可能になったらすぐに更新することをお勧めします。最新バージョンにアップグレードできない場合は、エージェントを 90 日以内のバージョンに更新してください。[エージェントを最新の状態に保つ方法](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent)の詳細については、こちらをご覧ください。エージェントのリリースとサポート日については、New Relic .NET エージェントの[EOL ポリシー ドキュメントを](/docs/apm/agents/net-agent/getting-started/net-agent-eol-policy)参照してください。
