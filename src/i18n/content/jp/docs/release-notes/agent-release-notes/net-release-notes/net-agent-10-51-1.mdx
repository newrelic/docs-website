### 知らせ

* カスタムベンダー名で`RecordDatastoreSegment()`を使用している顧客は、メトリクス名が`Datastore/*/Other/*`から`Datastore/*/<VendorName>/*`に変更されます。古いOtherベースのメトリクス名を参照するダッシュボード、アラート、またはクエリを更新してください。([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))

### 修正

* `RecordDatastoreSegment()` APIは、呼び出し元が指定したベンダー名を無視せずに正しく使用します。([f6c0e48](https://github.com/newrelic/newrelic-dotnet-agent/commit/f6c0e483e00bbaa17a5b44aa3c8b9d07c5c41e91))
* トランザクションがハーベストされる前に、すべての StackExchange.Redis セグメントが作成され、トランザクションに追加されるようにしました ([#3574](https://github.com/newrelic/newrelic-dotnet-agent/issues/3574)) ([4152ea2](https://github.com/newrelic/newrelic-dotnet-agent/commit/4152ea25b23318c68d372447c905c622ac5ac868))
* 9.7.0のシグネチャの変更をサポートするようにMySQL.Dataの計装を更新 ([#3562](https://github.com/newrelic/newrelic-dotnet-agent/issues/3562)) ([1e40e44](https://github.com/newrelic/newrelic-dotnet-agent/commit/1e40e44e89ab3f5c1af540a9aa30c6e1fa6a30e3))

### エージェントの更新

* [.NET エージェントを更新する](/docs/agents/net-agent/installation-configuration/update-net-agent)には、標準の手順に従ってください。
* 特に古いエージェントを使用している場合は、主要な変更点のリストと[レガシー .NET エージェントの更新](/docs/apm/agents/net-agent/installation/update-net-agent/#updating_older_net)手順を確認してください。

最新のエージェント バージョンが利用可能になったらすぐに更新することをお勧めします。最新バージョンにアップグレードできない場合は、エージェントを 90 日以内のバージョンに更新してください。[エージェントを最新の状態に保つ方法](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent)の詳細については、こちらをご覧ください。エージェントのリリースとサポート日については、New Relic .NET エージェントの[EOL ポリシー ドキュメントを](/docs/apm/agents/net-agent/getting-started/net-agent-eol-policy)参照してください。
