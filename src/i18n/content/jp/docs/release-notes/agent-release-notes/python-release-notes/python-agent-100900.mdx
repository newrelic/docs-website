## メモ

Pythonエージェントのこのリリースでは、 [OpenTelemetry](https://github.com/open-telemetry/semantic-conventions/blob/v1.24.0/docs/database/database-spans.md)互換のデータストア スパン プロパティ名が追加され、 [LangChain](https://pypi.org/project/langchain/)計装のバグが修正され、 [Kombu](https://pypi.org/project/kombu/)計装が一時的に無効になります。

[Python パッケージ インデックス](https://pypi.python.org/pypi/newrelic)経由で`easy_install/pip/distribute`を使用してエージェントをインストールするか、 [New Relic ダウンロード サイト](https://download.newrelic.com/python_agent/release)から直接ダウンロードします。

## 新機能

* OpenTelemetry互換のデータストアスパン属性を追加する

  * データストアのスパン属性は、 [OpenTelemetry の](https://github.com/open-telemetry/semantic-conventions/blob/v1.24.0/docs/database/database-spans.md)命名規則とより一貫性を持つようになりました。

## バグ修正

* 文字列応答タイプのLangChain計装を修正

  * 以前は、 [LangChain が](https://pypi.org/project/langchain/)文字列型の応答を返す場合、エージェントは文字列の最初の文字のみをキャプチャしていました。これは修正されました。

* Kombu 計装を無効にする

  * エージェント内の既存の[Kombu](https://pypi.org/project/kombu/)計装は、 `inspect`モジュールから発生した`TypeError`によりクラッシュします。 この機能は一時的に無効になっていますが、クラッシュの根本原因が特定され修正されたら、将来のリリースで再度有効になります。

## サポートステートメント

最新のエージェント バージョンが利用可能になったらすぐに更新することをお勧めします。最新バージョンにアップグレードできない場合は、エージェントを 90 日以内のバージョンに更新してください。エージェントを最新の状態に保つ方法の[詳細](/docs/new-relic-solutions/new-relic-one/install-configure/update-new-relic-agent/)については、こちらをご覧ください。

エージェントのリリースとサポート日については、New Relic Python エージェントの[EOL ポリシーを](/docs/apm/agents/python-agent/getting-started/python-agent-eol-policy/)参照してください。
