当社のAPM(Application Performance Monitoring)は、貴社の全てのアプリケーションとマイクロサービスについて統合されたモニタリングサービスを提供します。最新のスタックの数百もの依存関係から、アプリのシンプルなウェブトランザクションタイムとスループットまで、すべての項目を監視します。事前に構築されたカスタムダッシュボードを通じて、メトリクス、イベント、ログ、トランザクション（MELT）を監視することで、アプリの健全性をリアルタイムで追跡します。

当社のAPMサービスは、当社のエージェントをインストールするときにコードを自動的にインストゥルメントすることで、アプリから必要なものを柔軟に監視します。

* 早期アクセスソフトウェアはまだ完全には安定していませんか？エラー受信トレイでユーザーに影響を与える前に、エラーをプロアクティブに監視して解決します。
* ウェブアプリの速度についてユーザーが頻繁にコメントし、そのフィードバックを定量化したい場合はどうしますか？Apdexを一目で監視して、満足度を測定します。
* ログを保存する場所が必要ですか？当社のエージェントは自動的にそれらを取り込みます。
* 最新のスタックに追跡する数十、あるいは数百の依存関係がある場合はどうでしょうか？リレーションシップマップや外部サービスで追跡できます。

単一の統合モニタリングソリューションで時間を節約したいですか？ロゴをクリックしてAPMを開始してください。数分で完了します！

## すべてがどのように機能するか [#how-works]

New Relicは、当社の多言語エージェントの1つを使用して、アプリケーションをコードレベルでインストゥルメントします。これらのエージェントはアプリケーションからメトリクスを収集し、New Relic APMに送信します。これにより、事前に構築されたダッシュボードを使用して監視できるようになります。

## APMのインストール [#get-started]

APMは、簡単なステップで開始できます。

1. New Relicアカウントに[サインアップ](https://newrelic.com/signup/)します。
2. ご利用のアプリ向け言語エージェントをインストールします。

## データの使用方法 [#use-case]

New Relicがアプリケーションからデータを受信した瞬間に、アプリケーションの基本的な健全性を監視します。New Relic UIの<DNT>**APM Summary**</DNT>ページに基本的な<InlinePopover type="dashboards"/>が表示され、カスタマイズなしでアプリのパフォーマンスをすばやく把握できます。

[エクスプローラー](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/)を使用すると、アプリも含めたソフトウェアの全スタックにアクセスして観察したり、パフォーマンスデータとアラートステータスを一目で確認して関係を把握できます。シンプルで強力なビジュアルツールにより、すべての[エンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)、つまり、データをレポートする識別可能な任意の対象を監視できます。New Relicエコシステムでは、エンティティは、アプリケーション、ホスト、コンテナ、データベースサービスなどの基本コンポーネントを含みますが、こうした要素のカスタムグループを表す場合もあります。[エンティティは独自に作成](https://github.com/newrelic/entity-definitions#entity-definitions)することもできます。
