優れたアラートは、個人やチームにシステムの変更を通知するのに役立ちます。 [アラート](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) は、New Relic 製品全体で調整された単一のアラートツールを提供します。これにより、 [アラートポリシー](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-policy) と [条件](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition) を管理し、最も気になる [エンティティ](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) のメトリクス (アプリや APM のキートランザクションなど) に焦点を当てることができます。

[1つ以上のアラートポリシー](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/name-or-rename-alert-policy) を設定し、 [自分の New Relic ](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions)[製品のエンティティ (ターゲット)](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition) に対するアラート条件 を定義すると、その情報を Alerts UI で見ることができます。また、個々の New Relic 製品の UI でアラートポリシーの情報を確認することもできます。

## APMのポリシーと条件を見る [#policy]

特定のアプリやキー・トランザクションのアラート・ポリシーや条件情報をAPMから直接表示する。

1. 次のうち1つを行います。

   * アプリです。Go to **[one.newrelic.com](https://one.newrelic.com) > （アプリを選択）> Settings> Alert conditions**.
   * キートランザクションです。Go to **[one.newrelic.com](https://one.newrelic.com) > More> Key transactions> (select a key transaction)> Settings> Alert conditions**.

2. **Alert conditions** ページから、利用可能なツールを使用して、警告条件とそれに関連するポリシーの検索、ソート、表示、更新を行います。

## APMのイベントや活動を見る [#activity]

<Callout variant="tip">
  ご利用の全てのアプリケーションおよびサービスに関する大まかな概要を得るには、[New Relicエクスプローラー](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer)を利用してください。
</Callout>

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) やその他のアクティビティに関するサマリー情報をAPMから直接表示することができます。

1. 次のうち1つを行います。

   * アプリです。 **[one.newrelic.com](https://one.newrelic.com/)** にアクセスしてください。
   * 鍵となるトランザクションGo to **[one.newrelic.com](https://one.newrelic.com) > More> Key transactions**.

2. インデックスから、企業の [色分けされた健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) にマウスオーバーするか、 **アプリケーション活動** リストからリンクを選択するか、特定のブラウザアプリや主要な取引を選択すると、追加の詳細が表示されます。

例えば、Criticalアラートの違反が発生した場合です。

* APMインデックスと選択したアプリで、健康状態の表示が赤になります。
* 各種チャートの背景色がピンクに変わります。
* APM インデックス上の **Application activity** リストと、選択されたアプリの **Summary** ページには、該当する場合、Warning（黄色）とCritical（赤色）の違反が表示されます。

アラートでの違反を詳しく知るには、チャートのピンク色の部分にマウスオーバーするか、選択してください。詳細および例については、 [自社製品からイベントを表示する](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products) を参照してください。

## APMからのアラート違反の表示 [#violations]

警告条件に [しきい値](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) が設定されていて、警告（黄色）または重大（赤）の違反があった場合、エンティティの色分けされた [健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) が違反を示すように変わります。APMのアプリやキートランザクションのページから直接違反を見ることができます。

1. 次のうち1つを行います。

   * アプリです。Go to **[one.newrelic.com](https://one.newrelic.com) > (select an app)> Events> Violations**.
   * キー・トランザクションGo to **[one.newrelic.com](https://one.newrelic.com) > More> Key transactions> (select a key transaction)> Events> Violations**.

2. **Violations** インデックスから、利用可能なツールを使用して、選択したアプリまたはキートランザクションの違反を検索、ソート、または表示します。

3. オプションです。違反の原因となったアラートポリシー、条件、またはしきい値を表示または更新するには、対応するリンクを選択します。

<Callout variant="tip">
  また、Alerts UIから、 [特定のインシデント](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/explore-incident-history) または全製品の [インシデントおよびイベント](/docs/alerts/new-relic-alerts-beta/reviewing-events/review-events-across-products) について、警告およびクリティカルな違反を表示することができます。アラートは、 [アラート通知を送信します。](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts) クリティカル（赤）の違反のみが対象です。
</Callout>
