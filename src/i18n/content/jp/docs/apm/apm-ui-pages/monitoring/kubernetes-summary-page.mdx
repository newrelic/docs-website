Kubernetes APM <DNT>**summary**</DNT> ページには、現在のアプリケーションの経過時間、使用可能なポッドの数、CPU とメモリの使用量、ポッド ログ、メトリックの比較など、選択したアプリケーションに関する一般的な情報が表示されます。

<img title="Kubernetes page" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-apm-ui.webp" />

<figcaption>
  に行く <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services &gt; (selected app) &gt; Kubernetes**</DNT>
</figcaption>

<Callout variant="tip">
  すべてのアプリケーションとサービスの概要を把握するには、 [エンティティ エクスプローラー](/docs/new-relic-solutions/new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/)を使用します。
</Callout>

## Kubernetesクラスターのデプロイメントを監視します。 [#monitor-kubernetes]

APMのKubernetesページでは、アプリケーションに関する以下の情報が提供されます。

<CollapserGroup>
  <Collapser id="deployment-summary" title="展開の概要">
    クラスター名、デプロイメント、および実行されているネームスペースが表示されます。 デプロイメントをクリックすると、スライドアウト パネルにその詳細が表示されます。 スライドアウト パネルには、指定された時間範囲内のログの数やエラー ログの割合などの関連情報を含むミニ概要が表示されます。

    <img title="Kubernetes page - Mini overview" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-apm-mini-overview.webp" />

    <DNT>**Instances**</DNT>ドロップダウンでは、サービスの単一のポッドインスタンスを選択できます。

    <img title="Kubernetes page - Deployment summary" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-header.webp" />

    マルチクラスタ アプリ (複数のクラスタにまたがるアプリ) がある場合、 <DNT>**Kubernetes**</DNT>ページにすべてのクラスタの概要が表示されます。 特定のクラスターを分離するには、 <DNT>**Cluster**</DNT>ドロップダウン リストから選択します。 すべてのクラスターのビューに戻るには、 <DNT>**All clusters**</DNT>をクリックします。

    <img title="Kubernetes page - Show entity preview" alt="Show entity preview" src="/images/apm_screenshot-crop_k9-multi-cluster.webp" />
  </Collapser>

  <Collapser id="current-deployment" title="現在の展開">
    利用可能なPodや希望するPodの数、不足しているPod、kubernetesのデプロイメントが最初に作成された日付である年齢を確認することができます。

    <img title="Kubernetes page - Current deployment" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-current-deploy.webp" />
  </Collapser>

  <Collapser id="performance-summary" title="導入時のパフォーマンス概要">
    CPUやメモリの使用率、ポッドの再起動回数などを確認することができます。

    <img width="70%;" title="Kubernetes page - Performance summary" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-performance.webp" />
  </Collapser>

  <Collapser id="pods-containers" title="ポッドとコンテナ">
    ポッドや関連するコンテナの詳細を確認することができます。ポッドやコンテナ名をクリックすると、その詳細がスライドパネルで表示されます。

    さらに、 <DNT>**Logs**</DNT>ボタンを使用すると、そのポッドのコンテキストでログを表示できます。

    <img title="Kubernetes page - Pods and Containers" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-pod-containers.webp" />
  </Collapser>

  <Collapser id="activity-stream" title="活動の流れ">
    そのデプロイメントとそのポッドに関連するKubernetesイベントのストリームを見ることができます。

    <img title="Kubernetes page - Activity stream" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-activity-stream.webp" />
  </Collapser>

  <Collapser id="metrics-comparison" title="指標の比較">
    これらのウィジェットでは、デプロイメント内のポッドやコンテナに関連するメトリックの静的リストや、CPU、ネットワーク、ストレージなどの他のメトリックを比較することができます。

    <img title="Kubernetes page - Metrics Comparison" alt="This is an image of the Kubernetes APM page" src="/images/apm_screenshot-crop_k8-metric-comparasin.webp" />

    以下のメトリクスが利用可能です。

    * CPU使用率（コア数）
    * CPU スロットリング(%)
    * メモリ使用量(バイト)
    * 受信ネットワーク(KBps)
    * ネットワーク転送量（KBps）
    * ネットワークエラー（1秒あたり）
    * ストレージ使用量(バイト)
    * ストレージ使用率(%)
    * コンテナ再稼働回数
    * サービススループット（1分あたりのリクエスト数）
    * サービス応答時間 (ミリ秒)
    * サービスエラー率（％）
  </Collapser>
</CollapserGroup>

## アプリケーションをKubernetesにリンクする [#link-app]

当社独自のエージェントを使用するか 使用するかに関係なく、 Kubernetesメタデータを表示して エージェントにリンクできます。APMOpenTelemetry<InlinePopover type="apm" />は Kubernetes メタデータを使用して、その APM アプリケーションに関連する特定の Kubernetes デプロイメント情報を表示します。

### New Relic独自のインストゥルメンテーション [#nr-proprietary]

[New RelicのKubernetesインテグレーションをインストールする](/install/kubernetes/)と、 New Relicのメタデータ インジェクションが `nri-metadata-injection` 経由で自動的に有効になります。 Kubernetesインストゥルメンテーションによってセットアップされた `MutatingAdmissionWebhook` は、必要な環境変数を使用してポッドにタグを付けます。 そこから、これらの変数でキャプチャされたデータはAPMメトリクスとエンティティに引き継がれ、貴重なコンテキストでそれらを強化します。

詳細については、 [アプリケーションを Kubernetes にリンクする](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-your-applications-kubernetes/)方法を参照してください。

Kubernetes 概要ページを表示するには、APM サービス エンティティに次の属性が必要です。

* `k8s.clusterName`
* `k8s.namespaceName`
* `k8s.deploymentName`

### OpenTelemetryインストゥルメンテーション [#otel-instrumentation]

[OpenTelemetryコレクターは、](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro/) KubernetesAPMテレメトリーをKubernetes メタデータで強化する プロパティ プロセッサを提供します。

1. デプロイメント マニフェストで環境変数を定義する必要があります。

2. KubernetesこのAPM 環境変数を使用して、適切な メタデータを取得するようにコレクターの設定を調整します。

その結果、すべてのAPMメトリクスとエンティティには、`K8sattributes` プロセッサのおかげでKubernetesメタデータが含まれるようになります。 詳細については、 [OpenTelemetry アプリケーションを Kubernetes にリンクする](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes/)方法を参照してください。

Kubernetes 概要ページを表示するには、APM サービス エンティティに次の属性が必要です。

* `k8s.cluster.name`
* `k8s.deployment.name`
* `k8s.namespace.name`

<Callout variant="important">
  Kubernetes概要ページは、さまざまなメトリック プロバイダー ( New RelicとOpenTelemetry ) を組み合わせて機能します。 ただし、 KubernetesがOpenTelemetryエージェントによって監視され、 APM New Relicエージェントによって監視されている場合は、 <Link to="https://github.com/newrelic/k8s-metadata-injection/tree/main/charts/nri-metadata-injection">nri-metadata-injection</Link>をインストールして、必要なKubernetesメタデータをAPMメトリクスに提供することで適切に動作することを確認します。
</Callout>
