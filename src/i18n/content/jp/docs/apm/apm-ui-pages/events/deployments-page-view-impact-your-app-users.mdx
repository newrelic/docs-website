APM の **Deployments** ページには、最近のデプロイメントと、エンドユーザーやアプリサーバーの Apdex スコア、レスポンスタイム、スループット、およびエラーへの影響が一覧表示されます。詳細の表示やドリルダウン、検索やソートオプションの使用、エラーの非表示や削除、他の人との共有、チケットの提出などが可能です。

## デプロイメント通知の設定 [#dep_setup]

すべての APM エージェントについて、 [デプロイメントを記録することができます](/docs/apm/new-relic-apm/maintenance/recording-deployments) Webhooks を使用するか、New Relic REST API (v2) を使用します。また、Java エージェントの `jar` や Python エージェントの `newrelic-admin` スクリプトのように、特定の方法を持つエージェントもあります。

## デプロイメントの詳細を見る [#dep_procedures]

デプロイメント情報を設定した後、詳細を表示したり、掘り下げたりすることができます。

1. **[one.newrelic.com](https://one.newrelic.com) > APM> (select an app)> Events> Deployments** にアクセスします。
2. New Relic の標準的な [ユーザーインターフェースの機能](/docs/data-analysis/user-interface-functions/standard-page-functions) を使って、詳細な情報にドリルダウンすることができます。
3. 特定のイベントの詳細を表示するには、その **日付** リンクを選択してください。
4. この配置のエラーページを表示するには、「 **エラー** 」リンクを選択します。
5. 特定のデプロイメントの詳細を表示するには（該当する場合）、 **Change log** または **Change report** を選択します。

**変更ログ** には、 [デプロイメントを記録する際に `changelog` パラメータで提供された詳細が含まれています](/docs/apm/new-relic-apm/maintenance/recording-deployments) 。 **Change report** は、デプロイメントの前後における上位 10 トランザクションの動作を表示します。

## デプロイ後のパフォーマンスの確認 [#dep_timepicker]

個々のデプロイメントの **Deployments** ページのリンクは、選択したアプリの [Summary](/docs/applications-menu/applications-overview) ページの **Events** セクションに表示されます。14 日未満の期間では、チャート上に青い縦棒が表示され、デプロイメントを示します。デプロイメントのサマリー情報を表示するには、青いバーにマウスオーバーします。

<Callout variant="tip">
  また、APM サマリーページの **Compare with** オプションを使用して、同じデータを比較する期間を選択することもできます。 **Compare with** を有効にすると、UI にデプロイメントマーカーが表示されません。
</Callout>
