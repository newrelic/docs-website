New Relicの .NET エージェントは、.NET フレームワークと .NET Core の両方をサポートします。 このドキュメントでは、.NET ランタイム、フレームワーク、ライブラリの互換性とサポートについて説明します。

エージェントには、フレームワーク、データベース、メッセージキューシステムなど、.NETエコシステムで最も人気ある一部のビルトイン・インストゥルメンテーションとメッセージキューシステムが含まれています。エージェントを[ダウンロードおよびインストール](/install/dotnet)後は、監視下のプロセス内で実行されます。エージェントは個別のプロセスまたはサービスを作成しません。

[自動的にインストゥルメント](#instrumented-features)されないフレームワークとライブラリについては、[.NETカスタムインストゥルメンテーション](/docs/agents/net-agent/custom-instrumentation/introduction-net-custom-instrumentation)を使用してエージェントを拡張できます。

.NETエージェントをお試しになりますか？[New Relicアカウントの作成](https://newrelic.com/signup)は無料です。クレジットカードは不要です。

          .NET エージェントは、データストア サーバー プロセスに関するデータを収集しません。データストア クライアント ライブラリの使用状況からのみデータを収集します。データストア サーバー プロセスを直接監視するには、 New Relic Infrastructure エージェントと[オンホスト](/docs/infrastructure/host-integrations/get-started/introduction-host-integrations/)統合を使用します。

          デフォルトでは、.NET エージェントはストアド プロシージャまたは書き込みトレース内のパラメータ化された書き込みの SQL をキャプチャしません。 これらの SQL 問題を確認するには、 [エージェント設定](/docs/apm/agents/net-agent/configuration/net-agent-configuration/#datastore-tracer-query-parameters)でコレクション機能を有効にする必要があります。

          サポート対象のデータストアに対する[インスタンス詳細](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues)の収集は、デフォルトで有効になっています。現在リストされてないデータストアからインスタンスレベルの情報をリクエストするには、[support.newrelic.com](https://support.newrelic.com)でサポートを受けてください。

          データストアがここにリストされていない場合は、 [.NET エージェントAPI](/docs/apm/agents/net-agent/net-agent-api/net-agent-api/#ITransaction)の `RecordDatastoreSegment` メソッドを使用してカスタムインストゥルメンテーションを追加できます。
        </Collapser>

      ## 利用できない機能 [#unavailable-core]

      .NETエージェントでは、以下の機能は使用できません。

      * .NETエージェントは、[トリム自己完結型デプロイメントと実行ファイル](https://docs.microsoft.com/en-us/dotnet/core/deploying/trim-self-contained)をサポートしていません。これは、エージェントが依存するアセンブリをコンパイラがトリミングする可能性があるためです。
      * [GRPCの互換性の問題](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing)により、Alpine Linuxでは[Infinite Tracing](https://github.com/grpc/grpc/issues/21446)はサポートされていません。詳細については、「[このエージェントの問題](https://github.com/newrelic/newrelic-dotnet-agent/issues/289)」を参照してください。
      * .NETエージェントでは、エージェントが適切に機能するためにはジャストインタイム（JIT）コンパイルが必要です。このため、.NETアプリケーションの[ネイティブAhead of Time（AOT）](https://learn.microsoft.com/en-us/dotnet/core/deploying/native-aot/?tabs=net8plus%2Cwindows)をサポートしていません。

          * アドミニストレーターユーザーとしてログイン。

          * アドミニストレーターグループのメンバーになる。

          * 新しいオペレーティングシステムでプロンプトが表示されたら、昇格した資格情報を入力します。

          * アクセス権と特権を保有するユーザー数を最小限に抑えるため、エージェントによって作成されるログのアクセス権限を見直します。
        </Collapser>

        </Collapser>

        <Collapser className="freq-link" id="other-apm-framework" title="その他のモニターソフトウェアの使用">

        </Collapser>

          以下のフレームワークは、**完全にサポートされていません**。

          * <DNT>**ASP.NET Web API v1:**</DNT> .NET Framework 4.0を対象とするアプリ向けに、New Relicの.NETエージェント5.0以降のASP.NET WebAPI v1を使用する方法については 、[トラブルシューティング情報](/docs/agents/net-agent/troubleshooting/web-api-transactions-are-missing)を参照してください。（これは.NET Framework 4.5以降には影響しません。）
          * <DNT>**Mono:**</DNT> New RelicはLinuxで稼働するオープンソースの.NETフレームワークである[Mono](https://www.mono-project.com/)をサポートしていません。 その理由は、Monoベースの.NETアプリケーションへのプロファイラーとして、.NETエージェントにインジェクションするプロファイラーAPIがないためです。プロファイラーAPIは、コンポーネントオブジェクトモデル（COM）ベースのインタフェースであり、Linuxではサポートされていません。
          * エージェントは.NET Frameworkベースのアプリしかインストゥルメントできないため、Classic ASPは<DNT>**not supported**</DNT>。
          * Sharepointは<DNT>**not supported**</DNT>です。
