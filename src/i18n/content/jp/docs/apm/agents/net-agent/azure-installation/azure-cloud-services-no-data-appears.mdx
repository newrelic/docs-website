## 問題

[Microsoft Azure Cloud Services](/docs/agents/net-agent/azure-installation/install-app-azure-cloud-services) から New Relic の .NET エージェントをインストールし、NuGet パッケージをインストールしました。トラフィックを発生させ、アプリのデータ収集が開始されるまで数分待っても、APM のユーザーインターフェイスにデータが表示されません。

## 解決策

.NET エージェントが起動してアプリにアタッチするためには、少なくとも 1 つのメトリックを収集する必要があります。New Relic が計測する外部コールまたはデータベースコールがある場合、アプリはデータを [APM **Summary** page](/docs/apm/applications-menu/monitoring/apm-overview-page) に報告します。また、外部の呼び出しに関する情報は、アプリの [Service maps](/docs/data-analysis/user-interface-functions/view-your-data/service-maps-visualize-monitor-apps-entire-architecture) に表示されます。

カスタムインスツルメンテーションがないと、 `Worker` ロールは、データベースコールと外部コールしかレポートしません。( `Worker` ロールのインストルメント化は、 [カスタムアプリケーションのインストルメント化](/docs/agents/net-agent/instrumentation/instrumenting-non-iis-apps) と同様です)。その他の詳細を表示するには、New Relic の [.NET API](/docs/agents/net-agent/features/net-agent-api) を使用してカスタムメトリクスを収集し、 [カスタムダッシュボード](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards) で表示する必要があります。
