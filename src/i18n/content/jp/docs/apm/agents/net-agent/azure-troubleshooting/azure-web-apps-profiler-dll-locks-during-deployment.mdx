## 問題

[Microsoft Azure Web App](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps) に New Relic の新バージョンの .NET エージェント (`NewRelic.Profiler.dll`) をデプロイする際に、エラーメッセージが表示され、プロセスを停止または再起動するまで続行できません。

## 解決策

通常、このプロセスが停止すると、デプロイ時に New Relic がサイトルート ( `D:Homesite\wwwroot\newrelic`) にパックしたアセットで上書きされます。この問題を解決するには、以下のいずれかの方法をお試しください。

<CollapserGroup>
  <Collapser
    id="stop-dll-deploy"
    title="ファイルのリリースを停止する"
  >
    このソリューションは、既存のバージョンの上に新しい.NETエージェントのバージョンを展開しようとしている場合、Azure Web Appで利用できます。

    1. Webアプリを停止して、 `.dll` ファイルを解放します。
    2. アップデートの実行を許可します。
    3. デプロイが完了したら、インスタンスを再起動します。
  </Collapser>

  <Collapser
    id="site-extension"
    title="New Relic Site Extensionの使用"
  >
    New Relic の .NET エージェントを Azure Web App にインストールするには、 [New Relic Site Extension](https://www.siteextensions.net/packages/NewRelic.Azure.WebSites/) を使用します。(エクステンションのインストールは一度で済むはずです)。

    1. **https&#x3A;//<var>yoursite</var>.scm.azurewebsites.net/** に移動し、 **Site Extensions** を選択します。

    2. **ギャラリー** から、追加 <Icon name="fe-plus"/>アイコンを選択し、 **New Relic** を選択します。

    3. 指示に従って、手動で [設定キーをウェブアプリに追加し、ニーズに合ったオプションを使用してください。](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps)

       [](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps)

       [Site Extension を更新するには。 ](/docs/agents/net-agent/azure-installation/install-app-azure-web-apps)[Azure Portal](https://portal.azure.com) から、 **WebApps> (自分のサイトを選択)> All Settings> Extensions> New Relic** を選択します。
  </Collapser>
</CollapserGroup>
