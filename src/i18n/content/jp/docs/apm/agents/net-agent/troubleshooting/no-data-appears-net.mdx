## 問題

アプリケーションのトラフィックが生成されてから5分間待機した後に、New Relic UIに.NETエージェントのデータが表示されません。

## 解決

データの欠落は、いくつかの理由によって発生します。

* <DNT>
    **Common causes**
  </DNT>

  データが欠落している場合:[基本的なトラブルシューティングの解決策](#basic-solutions)を参照してください。

* <DNT>
    **Other situations:**
  </DNT>

  基本的な解決策で問題が解決しない場合は、データ不足がシステムに固有の原因である可能性があります。 詳細については、[高度なトラブルシューティング ソリューション](#advanced-solutions)を参照してください。

* <DNT>
    **Microsoft Azure users:**
  </DNT>

  [Azure トラブルシューティング ソリューション](/docs/agents/net-agent/azure-troubleshooting)を参照してください。

* <DNT>
    **No browser data appears:**
  </DNT>

  [Browserエージェントを使用するときはトラブルシューティングのヒントを](/docs/agents/net-agent/troubleshooting/no-page-load-timing-data-appears-net)参照してください。

### 高度な解決策

より複雑なデータが表示されない問題を解決する場合は、必要に応じて以下の解決策を試めしてください。

<table>
  <thead>
    <tr>
      <th>
        <DNT>
          **Advanced solutions**
        </DNT>
      </th>

      <th>
        <DNT>
          **Comments**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**.NET Framework:**</DNT> レジストリの権限を確認してください。
      </td>

      <td>
        [レジストリキーのアクセス権限のエラー](/docs/agents/net-agent/troubleshooting/no-data-registry-key-permission-issues)を確認します。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**.NET Framework:**</DNT> Windows イベント ビューアーを実行します。
      </td>

      <td>
        該当する場合：

        1. Windowsイベントビューアを実行し、New Relic.NETプロファイラーからアプリケーションイベントを探します。

        2. ソースがNew Relic.NETプロファイラーまたはNew Relic.NETエージェントになっているメッセージを検索します。

        3. New Relicイベントのメッセージがない場合は、.NETランタイムからのメッセージを探します。

           詳細については、[データまたはイベントログがない：CoCreateエラー](/docs/dotnet/dotnet-kb-102)をご覧ください。
      </td>
    </tr>

    <tr>
      <td>
        デバッグログを取得します。
      </td>

      <td>
        ログにエラーが表示されない場合は、一時的にロギングレベルを上げて[ログを`debug`収集します](/docs/agents/net-agent/troubleshooting/generating-logs-troubleshooting#logs)。
      </td>
    </tr>

    <tr>
      <td>
        プロキシ設定を確認します。
      </td>

      <td>
        [プロキシ設定](/docs/dotnet/dotnet-agent-configuration#service)が正しく、その設定にホスト属性が含まれていることを確認します。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**.NET 8:**</DNT> ランタイム診断が有効になっていることを確認してください
      </td>

      <td>
        .NET 8以降では、環境変数`DOTNET_EnableDiagnostics=0`（または`COMPlus_EnableDiagnostics=0`）を設定するとCLRプロファイリングが無効になり、エージェントが動作できなくなります。この設定は通常、読み取り専用ファイルシステムのデプロイメントシナリオで推奨されますが、診断ポートを無効にする場合にのみ使用されます。プロファイリングを有効にしたまま診断システムの特定のサブコンポーネントを無効にする方法については、[Microsoftのドキュメント](https://learn.microsoft.com/en-us/dotnet/core/tools/dotnet-environment-variables#dotnet_enablediagnostics)を参照してください。
      </td>
    </tr>

    <tr>
      <td>
        その他のヘルプを参照します。
      </td>

      <td>
        それでもデータが表示されない場合は、[support.newrelic.com](https://support.newrelic.com)でサポートを受けます。
      </td>
    </tr>
  </tbody>
</table>
