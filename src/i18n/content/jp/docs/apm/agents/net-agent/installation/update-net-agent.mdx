APM .NETエージェントを更新する方法。

## お使いの.NETエージェントのバージョンを確認する [#your_version]

1. [リリースノート](/docs/releases/dotnet)を参照して、エージェントの最新バージョンを確認してください。
2. **Windowsのプログラムと機能**を使用するか、またはLinuxのパッケージマネージャを使用して、現在インストールされている.NETエージェントのバージョンを確認します。

## 開始する前に [#before-start]

.NETエージェントをインストールする方法はいくつかあり、更新する方法もいくつかあります。このドキュメントでは、.NETエージェントをWindowsまたはLinuxホストに直接インストールする、最も簡単なインストールの更新手順を説明します。

他のインストールがある場合は、その他のコンテキストに関するこれらの設定のインストールドキュメントを参照してください。

* [Azure Web Apps](/docs/apm/agents/net-agent/azure-installation/install-net-agent-azure-web-apps/#site-extention-update)
* Docker：Windows向け[](/install/dotnet/?deployment=windowsInstall&docker=yesDocker) \| [Linux](/install/dotnet/?deployment=linux&docker=yesDocker)向け
* [Nuget](/docs/apm/agents/net-agent/install-guides/install-net-agent-using-nuget)

## WindowsおよびLinux向けのインストール [#windows-linux]

お使いのエージェントの更新手順に従います。

* [Windows](#updating_net)
* [Linux](#updating_net_core)

## .NETエージェントの更新（Windows） [#updating_net]

<Callout variant="important">
  更新する前に：

  * エージェントの更新には、Windows管理者グループの**アドミニストレーター**権限が必要です。詳しくは、[アクセス許可のドキュメント](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-framework/#user-rights)を参照してください。
  * お使いのエージェントのバージョンが7.0以前の場合、または.NET Frameworkのバージョンが4.0またはそれ以前の場合は、[レガシー.NETエージェントの更新](#updating_older_net)を参照してください。
  * アプリケーションがIISでホストされている場合は、.NETエージェントを更新する前に**IISを停止**してください。
</Callout>

<CollapserGroup>
  <Collapser
    id="update-framework-standard"
    title="標準インストール"
  >
    1. システムに[.NET 4.5またはそれ以降がインストールされている](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-framework/#net-version)ことを確認します。

    2. IISを停止します。

    3. お使いの[アーキテクチャ](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-framework/#architecture)に適したMSIインストールパッケージをダウンロードして実行します。

       * [32ビット版インストーラ](https://download.newrelic.com/dot_net_agent/latest_release/NewRelicDotNetAgent_x86.msi)
       * [64ビット版インストーラ](https://download.newrelic.com/dot_net_agent/latest_release/NewRelicDotNetAgent_x64.msi)

    4. インストーラパッケージが終了したら、IISを起動します。
  </Collapser>
</CollapserGroup>

## .NETエージェントの更新（Linux） [#updating_net_core]

<Callout variant="important">
  .NETエージェントのバージョン10.0.0以降では、パッケージ名は`newrelic-dotnet-agent`です。.NETエージェントのバージョン9.9.0以前では、パッケージ名は`newrelic-netcore20-agent`です。このドキュメントの残りの部分は`newrelic-dotnet-agent`を参照します。古いエージェントのバージョンがインストールされている場合は、古い名前に置き換えてください。バージョン10.0.0以降を下位バージョンから更新する場合、パッケージ名の変更により`CORECLR_NEWRELIC_HOME`および`CORECLR_PROFILER_PATH`環境変数を変更する必要があります。
</Callout>

以下のいずれかの方法を使用して、New Relicの.NETエージェントの最新バージョンに更新します。

<CollapserGroup>
  <Collapser
    id="updating-core-zip"
    title="tar.gz、debファイル"
  >
    1. アプリケーションをシャットダウンします。
    2. [アーキテクチャー](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core/#architecture)に適したファイルを[ダウンロードサイト](https://download.newrelic.com/dot_net_agent/latest_release/)からダウンロードします。
    3. 目的のインストール場所にファイルをインストールします。tar.gz ファイルの場合：既存のファイルの上部にあるzipファイルを取得します。
    4. アプリケーションを起動します。
  </Collapser>

  <Collapser
    id="yum"
    title="yum（CentOS、Oracle Linux、またはRHEL）"
  >
    1. アプリケーションをシャットダウンします。

    2. 以下を使用してエージェントを更新します。

       ```bash
       sudo yum update newrelic-dotnet-agent
       ```

    3. アプリケーションを起動します。
  </Collapser>

  <Collapser
    id="apt"
    title="apt（Debian、Linux Mint、またはUbuntu）"
  >
    1. アプリケーションをシャットダウンします。

    2. 以下を使用して、利用可能な更新のリストを取得してインストールします。

       ```bash
       sudo apt-get update && sudo apt-get install --only-upgrade newrelic-dotnet-agent
       ```

    3. アプリケーションを起動します。
  </Collapser>

  <Collapser
    id="rpm"
    title="rpm（CentOS、Oracle Linux、またはRHEL）"
  >
    1. アプリケーションをシャットダウンします。

    2. [アーキテクチャー](/docs/apm/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core/#architecture)に適したrpmファイルを[ダウンロードサイト](https://download.newrelic.com/dot_net_agent/latest_release/)からダウンロードします。

    3. 以下を使用してエージェントを更新します。

       ```bash
       sudo rpm -Uvh FILE_NAME.rpm
       ```

    4. アプリケーションを起動します。
  </Collapser>
</CollapserGroup>

               ```xml
               <configuration>
               ```
