アプリケーションパフォーマンス監視用の.NETエージェントでは、以下ができます。

* [APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm)を使用して、[アプリの高次元な概要](/docs/apm/applications-menu/monitoring/apm-overview-page)を入手、[トランザクショントレース](/docs/apm/transactions/transaction-traces/transaction-traces)、[データベースクエリ](/docs/apm/transactions/transaction-traces/sql-statements)、[エラー](/docs/apm/applications-menu/events/view-apm-error-analytics)などのコードレベルの詳細を確認、[大規模な分散システム上のアクティビティを追跡](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)。
* アプリがスムーズに実行していることを保証する[アラート](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic)からプロアクティブな通知を取得。
* また[クエリビルダー](/docs/query-your-data/explore-query-data/query-builder/use-advanced-nrql-mode-specify-data)を使用して、データをクエリ、そのデータでカスタム[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-new-relic-one-dashboards)を作成。
* [Infrastructureモニタリング](/docs/infrastructure/new-relic-infrastructure/installation/install-infrastructure-windows-server)をインストールし、アプリのホスト環境のパフォーマンスを表示。

## .NET Frameworkと.NET Coreの双方のサポート [#core-features]

New Relicの.NETエージェントは、[.NET Framework](/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent)および[.NET Core](/docs/agents/net-agent/getting-started/net-agent-compatibility-requirements-net-core)をどちらもサポートしており、VB.NET、C#、CLIなど、.NETと互換性のあるすべての言語で動作します。

.NET Coreのエージェントのサポートは、Microsoftの.NET Coreの互換性、速度、拡張API機能、プラットフォーム間機能を利用しています。エージェントは、バージョン2.0以下のMicrosoft .NET Coreバージョンはサポート**していません**。

New Relicの.NETに対するサポートを使用すると、以下のように動的または分散環境でアプリをモニターできます。

* クラウドで管理されているサーバーのVMイメージ
* ホスト上のVMサーバー
* Microsoft Azureアプリサービス
* セルフホスト用のWindowsシステムおよびLinuxシステム
* AWS EC2 VM

ガイド付きインストールは、Windows IISアプリケーション用の.NETエージェントをダウンロードおよびインストールする環境用の、カスタマイズされたCLIコマンドを作成します。

## エージェントのインストール [#installation]

.NETエージェントをインストールする前に、[New Relicアカウントを作成してください](https://newrelic.com/signup)。次に、[.NET Frameworkの要件](/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent)または [.NET Coreの要件](/docs/agents/net-agent/getting-started/compatibility-requirements-net-core-20-agent)を確認します。

インストールの準備ができたら、ランチャーを使用してインストールするか、[Windows](/docs/agents/net-agent/installation/install-net-agent-windows)または[Linux](/docs/agents/net-agent/installation/install-net-agent-linux)のオペレーティングシステムに適切なインストール手順を確認します。

## エージェントの設定 [#configuration]

エージェントの設定では、[アプリケーションに分かりやすい名前を付ける](/docs/agents/net-agent/installation-configuration/name-your-net-application)ことが最も重要な部分です。複数のアプリケーションまたはホストが存在する場合、New Relicは[メトリクスを集計するためにこのアプリ名を使用します](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name) 。また、エージェントには、多数の[設定オプション](/docs/agents/net-agent/installation-configuration/net-agent-configuration)があり、インストレーション後にさらにカスタマイズできます。

## APMおよびインフラストラクチャデータのログを表示 [#logs-context]

また、ログとアプリケーションのデータを統合して、トラブルシューティングをより簡単かつ迅速にできます。[コンテキストのログ](/logs/logs-context/net-configure-logs-context-all/)では、エラーとトレースに関連するログメッセージをアプリケーションのUIで直接表示できます。また、Kubernetesクラスタなど、[インフラストラクチャデータ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)のコンテキストでログを表示することもできます。New Relic Oneで別のUIページに切り替える必要はありません。

## ソースコードのチェック [#source-code]

.NETエージェントは、オープンソースソフトウェアです。つまり、[ソースコードを参照](https://github.com/newrelic/newrelic-dotnet-agent)して改善を送信し、独自のフォークを作成して構築できます。詳細については、[README ](https://github.com/newrelic/newrelic-dotnet-agent/blob/main/README.md)を参照してください。
