このドキュメントでは、.NETエージェントをインストールしてWCFアプリケーションを監視する方法を説明しています。

## WCFアプリケーションのインストール [#wcf]

IISでホストされたWCFアプリに.NETエージェントをインストールするには、以下の手順で行います。

1. [IIS でホストされるアプリの手順を](/install/dotnet/?deployment=windowsInstall&docker=noDocker)使用して、.NET エージェントをインストールします。

   <Callout variant="important">
     セルフホステッド WCF アプリケーション (カスタム Windows サービス、コンソール アプリ、または Windows フォーム アプリとしてホストされる) の場合は、エージェントも有効にする必要があります。[IIS でホストされていないアプリの手順](/install/dotnet/?deployment=windowsInstallNonIis&docker=noDocker)**のエージェント**を有効にする手順を参照してください。
   </Callout>

2. それぞれのWCFアプリに [という記述的な名前をつけてください](/docs/agents/net-agent/installation-and-configuration/naming-your-net-application#naming-non-iis) 。

3. 監視するイベントの種類に応じて、WCFアプリケーションでASP.NET互換モードを有効または無効にしてください。

   <CollapserGroup>
     <Collapser
       id="mode-enabled"
       title="ASP.NETパイプラインイベントとWCFイベント"
     >
       ASP.NET互換モードが**有効**になっている場合、.NETエージェントはASP.NETパイプラインイベントとWCFイベントを計測します。このモードでは、エージェントはアプリケーションスタックの全体像を提供します。
     </Collapser>

     <Collapser
       id="mode-disabled"
       title="WCFイベントのみ"
     >
       ASP.NET 互換モードが **無効** の場合、.NET エージェントは WCF イベントのみを計測します。ASP.NET パイプラインの一部として発生するアクティビティは、WCF トランザクションの外で行われるエージェント API 呼び出しなど、キャプチャされません。
     </Collapser>
   </CollapserGroup>

アプリケーションがトラフィックを受信している場合、データは数分以内に表示されます。そうでない場合は、「[データが表示](/docs/agents/net-agent/troubleshooting/no-data-appears-net)されない」を参照してください。
