本ドキュメントには、ASP.NET Core アプリケーションを監視するための New Relic の .NET エージェントのインストール手順が記載されています。これらは、 [Windows](/docs/agents/net-agent/installation/install-net-agent-windows) および [Linux](/docs/agents/net-agent/installation/install-net-agent-linux) の標準的なインストール手順を補足する追加の注意点と説明です。

## .NET CoreをターゲットにしたASP.NET Coreアプリ [#core-targeting-core]

New Relic は、IIS でホストされている .NET Core をターゲットにした ASP.NET Core アプリを自動的に計測します。

IIS を使用していないアプリケーション **の場合は、 [Windows](/docs/agents/net-agent/other-installation/install-net-agent-aspnet-core-targeting-net-framework-windows) または [Linux](/docs/agents/net-agent/installation/install-net-agent-linux) の標準的なインストール手順に従ってください。さらに、アプリケーションのエージェントを有効にし、モニタリングが自動的に行われない場合は、カスタムインスツルメンテーションを設定する必要があります。**

\*\*\* アプリを有効にします。Windowsの場合は [](/docs/agents/net-agent/installation/install-net-agent-windows#enabling-the-agent)、Linuxの場合は [](/docs/agents/net-agent/installation/install-net-agent-linux#enable-agent)の手順を参照してください。

* カスタムインストルメントKestralを使用していない場合や、何らかの理由でインストルメンテーションがデフォルトで行われない場合は、 [アプリをカスタムインストルメンテーションする必要があります。](/docs/agents/net-agent/custom-instrumentation/introduction-net-custom-instrumentation).## .NET FrameworkをターゲットにしたASP.NET Coreアプリ [#core-targeting-framework]IIS でホストされていないアプリケーション、または .NET Framework をターゲットにした ASP.NET Core アプリの場合は、 [Windows 用の標準インストール手順に従ってください](/docs/agents/net-agent/installation/install-net-agent-windows#msi) 。さらに、以下の手順に従って、 [、.NET エージェントを明示的に有効にする必要があります。](/docs/agents/net-agent/installation/install-net-agent-windows#enabling-the-agent) 。\*\*
