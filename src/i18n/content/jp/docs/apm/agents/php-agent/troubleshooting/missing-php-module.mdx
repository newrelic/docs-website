## 問題

**newrelic** module section does not appear in `phpinfo()`.

## 解決策

`newrelic` モジュールの説明欄が `phpinfo()` ページにないことから、 **`newrelic.so`** ファイルが extensions ディレクトリにインストールされていない可能性があります。New Relic PHP モジュールが存在するかどうかを確認するには、以下の方法があります。

1. [`phpinfo()` ページ](/docs/php/php-kb-103) で、 **PHP Core** のセクションを探してください。
2. PHP の拡張機能がロードされるディレクトリを特定するには、 `extension_dir` 設定のローカルな値を探します。
3. `newrelic.so` が `extension_dir` で指定されたディレクトリに存在することを確認してください。

`newrelic.so` ファイルが extensions ディレクトリに存在しない場合、どちらか。

* New Relic の [非標準の PHP インストール手順に従ってください](/docs/php/php-agent-installation-non-standard-php) 。

  または

* 手動で正しいバージョンを `/usr/lib/newrelic-php5/agent/x32` （32ビット用）または `/usr/lib/newrelic-php5/agent/x64` （64ビット用）からリンクしてください。また、tarファイルからインストールした場合は、tarファイルを解凍した **エージェント** ディレクトリからコピーまたはリンクしてください。
