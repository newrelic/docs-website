ここでは、New Relic PHP エージェントをアンインストールする方法について説明します。エージェントを一時的に無効にする方法については、 [Disable the agent](/docs/agents/manage-apm-agents/installation/disable-apm-agent#php) を参照してください。

[本資料では、以下のシステムにおいて、New Relic PHP エージェント](/docs/agents/php-agent/getting-started/new-relic-php) をアンインストールする方法を説明します。

* <DNT>RedHat</DNT> および <DNT>CentOS</DNT>
* <DNT>Ubuntu</DNT> および <DNT>Debian</DNT>
* Tar アーカイブ (任意の [サポートされたシステム](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems))

## エージェントをアンインストールする [#install-overview]

PHP アプリケーションから New Relic をアンインストールする際には、New Relic エージェントを削除し、関連する New Relic 設定ファイルも削除する必要があります。

PHP エージェントをシステムからアンインストールするには、以下の手順に従います。

1. システムからエージェントファイルを削除します。

   <CollapserGroup>
     <Collapser
       id="remove-redhat-centos"
       title="RedHatまたはCentOS"
     >
       `newrelic-install`スクリプトをオプションなしで実行し、スクリプトの呼び出し時にメイン メニューからアンインストール オプションを選択します。
     </Collapser>

     <Collapser
       id="remove-ubuntu-debian"
       title="UbuntuまたはDebian"
     >
       `apt-get`を使用してエージェントを削除します。

       ```bash
       apt-get remove newrelic-php5
       ```
     </Collapser>
   </CollapserGroup>

2. 残っている New Relic 構成ファイルをすべて削除します。`purge`オプションを指定して`newrelic-install`または`apt-get remove`コマンドを使用してください。

   ```bash
   newrelic-install purge
   ```

   また

   ```bash
   apt-get remove --purge newrelic
   ```

3. パッケージマネージャーを使用している場合は、そのマネージャーに特有のファイルを削除します。

   <CollapserGroup>
     <Collapser
       id="uninstall-redhat-centos"
       title="RedHatまたはCentOS"
     >
       `yum`を使用:

       ```bash
       sudo yum remove newrelic-php5
       ```

       <DNT>**`rpm` (32-bit)**</DNT>を使用:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (YOUR VERSION #)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```

       <DNT>**`rpm` (64-bit)**</DNT>を使用:

       ```bash
       rpm -e newrelic-php5-common-X.X.X.X (VERSION NUMBER)-1.noarch.rpm newrelic-daemon-X.X.X.X (YOUR VERSION NUMBER)-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="uninstall-ubuntu-debian"
       title="UbuntuまたはDebian"
     >
       `apt-get`を使用:

       ```bash
       sudo apt-get purge newrelic-php5
       ```

       <DNT>**`dpkg` (32-bit)**</DNT>を使用:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_i386.deb newrelic-php5_X.X.X.X_i386.deb
       ```

       <DNT>**`dpkg` (64-bit)**</DNT>を使用:

       ```bash
       dpkg -P newrelic-php5-common_X.X.X.X (YOUR VERSION NUMBER)_all.deb newrelic-daemon_X.X.X.X_amd64.deb newrelic-php5_X.X.X.X_amd64.deb
       ```
     </Collapser>
   </CollapserGroup>

4. Webサーバー（Apache、Nginx、PHP-FPMなど）を再起動する。
