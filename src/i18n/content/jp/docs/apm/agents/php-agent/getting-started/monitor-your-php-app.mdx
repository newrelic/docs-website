New RelicでPHPアプリケーションの監視をすぐに始めたいですか？このガイドを使用して、アプリケーションデータをNewRelicに送信し始めるための最適なパスを見つけてください。

始める前にもっと学びたいですか？[APM](/docs/apm/new-relic-apm/getting-started/introduction-apm)の[概要とPHPのNewRelicの概要](/docs/apm/agents/php-agent/getting-started/introduction-new-relic-php)は、開始するのに適した場所です。

## 何を監視したいですか？ [#what]

監視対象に応じて、PHPエージェントをインストールするためのパスがいくつかあります。

<CollapserGroup>
  <Collapser id="docker" title="Dockerコンテナにデプロイされたアプリ">
    DockerアプリをPHPエージェントでインストルメント化するためのいくつかのオプションがあります。

    * デーモンと PHP エージェントを別のコンテナに入れたい場合は[、ガイド付きインストール](https://one.newrelic.com/marketplace/install-data-source?account=4355438&state=b92c9282-c3e5-cacc-95c8-f70051e7b57c)を介してインストールしてください。 デーモンを作成する必要がある場合は、 [Dockerやその他のコンテナ環境](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#daemon-container)の追加手順に従ってください。
    * 同じコンテナにエージェントとデーモンを設定するには、次[の手順に従います](/docs/apm/agents/php-agent/advanced-installation/docker-other-container-environments-install-php-agent#install-same-container)。これには、PHPエージェントの[tarアーカイブ](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file/)のダウンロードが含まれます。
  </Collapser>

  <Collapser id="redhat-centos" title="RedHat、CentOS、またはAWSLinuxでデプロイされたアプリ">
    * [ガイド付き](https://one.newrelic.com/marketplace/install-data-source?account=4355438&state=b92c9282-c3e5-cacc-95c8-f70051e7b57c)インストールでは、マニュアルのインストールと同様に、 `yum`使用してエージェントをインストールします。 ガイド付きインストールは、インストール プロセスのどの段階にいるのかをより明確に把握したい場合に最適なオプションです。
    * [手動インストール](/docs/apm/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos#manager-tarball)では同じプロセスを実行できますが、コマンドラインからすべてのステップを管理できます。
  </Collapser>

  <Collapser id="debian-ubuntu" title="DebianまたはUbuntuでデプロイされたアプリ">
    * [ガイド付き](https://one.newrelic.com/marketplace/install-data-source?account=4355438&state=b92c9282-c3e5-cacc-95c8-f70051e7b57c)インストールでは、マニュアルのインストールと同様に、 `apt`使用してエージェントをインストールします。 ガイド付きインストールは、インストール プロセスのどの段階にいるのかをより明確に把握したい場合に最適なオプションです。
    * [手動インストール](/docs/apm/agents/php-agent/installation/php-agent-installation-ubuntu-debian)では同じプロセスを実行できますが、コマンドラインからすべてのステップを管理できます。
  </Collapser>

  <Collapser id="tar" title="他のUnixフレーバーでデプロイされたアプリ（tarアーカイブを使用）">
    別の種類のUnixを使用している場合は、 [PHPtarアーカイブ](/docs/apm/agents/php-agent/installation/php-agent-installation-tar-file)を使用してエージェントをインストールしてください。
  </Collapser>

  <Collapser id="hosting-providers" title="AWS、Heroku、Magento、またはその他のホスティングプロバイダーでホストされているアプリ">
    New Relicとのパートナーシップにより、Webアプリがサードパーティプロバイダーでホストされている場合は、PHPエージェントを実行できます。

    * AWSがホストするアプリの場合は、 [Amazon CloudWatchMetricStreams統合](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup)をインストールします。New RelicとAWSの統合については、「AWS統合の概要」[をご覧](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations/)ください。
    * HerokuでホストされているPHPアプリの場合は[、NewRelicアドオンを](/docs/apm/agents/php-agent/advanced-installation/php-agent-heroku/#installing)インストールします
    * Magentoの販売者の場合は、 [次の手順に従って](/docs/accounts/install-new-relic/partner-based-installation/log-install-new-relic-partners/#magento)PHPエージェントをMagentoインストールにデプロイします。
  </Collapser>
</CollapserGroup>
