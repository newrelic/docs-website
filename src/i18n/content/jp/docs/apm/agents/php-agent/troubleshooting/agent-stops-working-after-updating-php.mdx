## 問題

PHP を [サポートされているリリースバージョン](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release) にアップデートしたが、PHP エージェントが動作しなくなったり、レポートが表示されなくなったりした。

## 解決

インストールしている PHP を別のメジャーリリースバージョンにアップデートする場合 (例えば、5.3 から 5.4)、PHP エージェントを再インストールする必要があります。PHP をマイナーリリースバージョンにアップデートする場合は、エージェントを再インストールする必要はありません。

エージェントを再インストールするには、プラットフォームの[インストール手順](/docs/agents/php-agent/installation/php-agent-installation-overview)に従うか、 `newrelic-install install`コマンドを使用して<DNT>**[newrelic-install script](/docs/agents/php-agent/advanced-installation/using-newrelic-install-script)**</DNT>を起動します。

パッケージマネージャー (例: `apt-get upgrade` ) を使用してパッケージを指定せずに PHP を更新している場合、そのプロセスで[New Relic PHP エージェントもアップグレードする](/docs/agents/php-agent/installation/upgrading-php-agent)可能性があります。アップグレードが正常に適用されたことを確認するには:

1. ウェブサーバーを再起動します（Apache、Nginx、PHP-FPMなど）。
2. 5分ほど待って、エージェントが報告を開始するかどうかを確認します。
3. エージェントがまだ報告されない場合は、 [デーモンの設定を確認してください](/docs/agents/php-agent/troubleshooting/verifying-php-daemon) 。外部モードで起動するように設定されている場合は、 [エージェントがレポートを開始するために、デーモン](/docs/agents/php-agent/advanced-installation/starting-php-daemon-advanced#daemon-external) を手動で起動する必要があります。
