当社のPHPエージェントは、コードを自動インストゥルメントするため、アプリケーションのモニターを起動できます。この手順に従って、Amazon Linux 2、RedHat、またはCentOSを使用してNew RelicのAPM用PHPエージェントをインストールします。**例外**: 旧バージョンをインストールしている場合は、代わりに[エージェントをアップグレードします](/docs/agents/php-agent/installation/upgrading-php-agent)。

## エージェントのインストール [#install]

New RelicのPHPエージェントのパッケージ名はPHP 5を参照しますが、このパッケージはすべての[サポートされているPHPバージョン](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release)で使用できます。

1. <InlinePopover type="licenseKey"/>にアクセス可能であることを確認してください。

2. 以下のいずれかの方法でインストレーションパッケージを入手します。

   <CollapserGroup>
     <Collapser
       id="tell-rpm"
       title={<>パッケージマネージャ（<InlineCode>rpm</InlineCode>）にNew Relicリポジトリについて指示する</>}
     >
       **32ビット**システムの場合、以下を実行

       ```
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/i386/newrelic-repo-5-3.noarch.rpm
       ```

       **64ビット**システムの場合、以下を実行

       ```
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-3.noarch.rpm
       ```
     </Collapser>

     <Collapser
       id="download-rpm"
       title="New Relicからrpmファイルをダウンロード"
     >
       **32ビット**システムの場合、[32ビットパッケージ](https://download.newrelic.com/pub/newrelic/el5/i386/)から次の3つのファイルをダウンロード（`X.X.X.X`を以下の最新の[PHPエージェントのバージョン番号](/docs/release-notes/agent-release-notes/php-release-notes)に置き換え）

       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```

       * ```
         newrelic-daemon-X.X.X.X-1.i386.rpm
         ```

       * ```
           newrelic-php5-X.X.X.X-1.i386.rpm
         ```

         **64ビット**システムの場合、[64ビットパッケージ](https://download.newrelic.com/pub/newrelic/el5/x86_64/)からこれらの3つのファイルをダウンロード（`X.X.X.X`を以下の最新の[PHPエージェントのバージョン番号](/docs/release-notes/agent-release-notes/php-release-notes)に置き換え）

       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```

       * ```
         newrelic-daemon-X.X.X.X-1.x86_64.rpm
         ```

       * ```
         newrelic-php5-X.X.X.X-1.x86_64.rpm
         ```
     </Collapser>
   </CollapserGroup>

3. 以下のお好みのパッケージマネージャを使用して、エージェントとデーモンをインストール

   <CollapserGroup>
     <Collapser
       id="manager-yum"
       title="yum"
     >
       ```
       sudo yum install newrelic-php5
       ```

       New Relic for PHPを初めてインストールする場合は、yumはNew Relicのパブリックキーを受け入れるよう求めてきます。New RelicのキーIDは`548C16BF`です。
     </Collapser>

     <Collapser
       id="manager-32-bit-rpm"
       title="32ビットrpm"
     >
       このコマンドを実行するときは、`X.X.X.X`を以下の最新の[PHPエージェントのバージョン番号](/docs/release-notes/agent-release-notes/php-release-notes)に置き換えます。

       ```
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.i386.rpm newrelic-php5-X.X.X.X-1.i386.rpm
       ```
     </Collapser>

     <Collapser
       id="manager-64-bit-rpm"
       title="64ビットrpm"
     >
       このコマンドを実行するときは、`X.X.X.X`を以下の最新の[PHPエージェントのバージョン番号](/docs/release-notes/agent-release-notes/php-release-notes)に置き換えます。

       ```
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser
       id="manager-tarball"
       title="tarball"
     >
       **yum**と**rpm**がホスト設定で動作しない場合は、[バイナリtarballからインストール](/docs/php/php-agent-installation-tar-files)します。
     </Collapser>
   </CollapserGroup>

4. `newrelic-install`スクリプトを実行し、指示に従います。

   ```
   sudo newrelic-install install
   ```

5. ウェブサーバーを再起動します（Apache、NGINX、PHP-FPMなど）。

6. アプリケーションへのトラフィックを生成し、データがNew Relicに送信されるまで数分間待ちます。その後、[UIでアプリのパフォーマンスをチェックします](/docs/apm/applications-menu/monitoring/apm-overview-page)。
