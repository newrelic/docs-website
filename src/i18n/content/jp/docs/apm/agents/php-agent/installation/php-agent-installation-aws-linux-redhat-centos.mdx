当社のPHPエージェントは、コードを自動インストゥルメントするため、アプリケーションのモニターを起動できます。この手順に従って、x86\_64アーキテクチャー上にAmazon Linux 2、RedHat、またはCentOSを使用してNew RelicのAPM用PHPエージェントをインストールします。aarch64アーキテクチャーでのインストールには、[tarファイル](/docs/php/php-agent-installation-tar-files)方式を使用してください。例外：<DNT>**Exception:**</DNT>以前のバージョンがインストールされている場合は、代わりに[エージェントをアップグレードします](/docs/agents/php-agent/installation/upgrading-php-agent)。

## エージェントのインストール [#install]

New RelicのPHPエージェントのパッケージ名はPHP 5を参照しますが、このパッケージはすべての[サポートされているPHPバージョン](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release)で使用できます。

1. <InlinePopover type="licenseKey" />にアクセス可能であることを確認します。

2. 以下のいずれかの方法でインストレーションパッケージを入手します。

   <CollapserGroup>
     <Collapser
       id="tell-rpm"
       title={<>パッケージマネージャ（<InlineCode>
         rpm
       </InlineCode>）にNew Relicリポジトリについて指示する</>
       }
     >
       ```bash
       sudo rpm -Uvh http://yum.newrelic.com/pub/newrelic/el5/x86_64/newrelic-repo-5-3.noarch.rpm
       ```
     </Collapser>

     <Collapser id="download-rpm" title="New Relicからrpmファイルをダウンロード">
       * ```
         newrelic-php5-common-X.X.X.X-1.noarch.rpm
         ```
       * ```
         newrelic-daemon-X.X.X.X-1.x86_64.rpm
         ```
       * ```
         newrelic-php5-X.X.X.X-1.x86_64.rpm
         ```
     </Collapser>
   </CollapserGroup>

3. 以下のお好みのパッケージマネージャを使用して、エージェントとデーモンをインストール

   <CollapserGroup>
     <Collapser id="manager-yum" title="yum">
       ```bash
       sudo yum install newrelic-php5
       ```

       New Relic for PHPを初めてインストールする場合は、yumはNew Relicのパブリックキーを受け入れるよう求めてきます。New RelicのキーIDは`548C16BF`です。
     </Collapser>

     <Collapser id="manager-rpm" title="RPM">
       このコマンドを実行するときは、`X.X.X.X`を以下の最新の[PHPエージェントのバージョン番号](/docs/release-notes/agent-release-notes/php-release-notes)に置き換えます。

       ```bash
       rpm -i newrelic-php5-common-X.X.X.X-1.noarch.rpm newrelic-daemon-X.X.X.X-1.x86_64.rpm newrelic-php5-X.X.X.X-1.x86_64.rpm
       ```
     </Collapser>

     <Collapser id="manager-tarball" title="tarball">
       <DNT>**yum**</DNT>と<DNT>**rpm**</DNT>がホスト設定で動作しない場合は、[バイナリのtarballからインストール](/docs/php/php-agent-installation-tar-files)します。
     </Collapser>
   </CollapserGroup>

4. `newrelic-install`スクリプトを実行し、指示に従います。

   ```bash
   sudo newrelic-install install
   ```

5. ウェブサーバーを再起動します（Apache、NGINX、PHP-FPMなど）。

6. アプリケーションへのトラフィックを生成し、データがNew Relicに送信されるまで数分間待ちます。その後、[UIでアプリのパフォーマンスをチェックします](/docs/apm/applications-menu/monitoring/apm-overview-page)。
