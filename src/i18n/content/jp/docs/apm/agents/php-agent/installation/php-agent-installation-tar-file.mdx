New Relic の PHP エージェントは、サポートされているプラットフォームであれば、圧縮された tar ファイルからインストールすることができます。ただし、Redhat や CentOS の場合は、 [RPM パッケージ](/docs/agents/php-agent/installation/php-agent-installation-aws-linux-redhat-centos/) を使用するのが一般的です。Ubuntu や Debian の場合は、 [Debian パッケージ](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian) を使用するのが一般的です。

## tarファイルのダウンロード [#download]

他のインストール方法と異なり、tar アーカイブは特別なリポジトリの設定を必要としません。必要なのは、アーカイブをダウンロードして、以下の指示に従うだけです。

**[download.newrelic.com/php_agent/release/](https://download.newrelic.com/php_agent/release/)** から適切なtar配布ファイルをダウンロードしてください。例えば、以下のようになります。

* FreeBSDの場合は、 `newrelic-php5-X.X.X.X-freebsd.tar.gz` をダウンロードしてください。
* Alpine Linux の場合は、 `newrelic-php5-X.X.X.X-linux-musl.tar.gz` をダウンロードしてください。

<Callout variant="tip">
  New Relic の PHP エージェントのパッケージ名は `newrelic-php5` です。この名前は PHP 5 を参照していますが、このパッケージは、 [サポートされているすべての PHP バージョン](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#php-release) 、PHP 7 バージョンを含めて動作します。
</Callout>

## インストールまたはアップデート [#install]

tar ファイルによるインストールでは、初回インストール時とアップデート時の手順は同じです。 <var>X.X.X.X</var> を正しい New Relic のバージョンに置き換えてください。エージェントをインストールまたはアップデートするには

1. [New Relic のウェブサイト](https://download.newrelic.com/php_agent/release/) から適切な tar ファイルをダウンロードし、ローカルディスクのわかりやすい場所に保存します。 `/tmp` フォルダは使用しないでください。

2. アーカイブを解凍して取り出す。

   ```
   gzip -dc newrelic-php5-<var>X.X.X.X-OS</var>.tar.gz | tar xf -
   ```

3. 新たに作成したディレクトリに変更します。

   ```
   cd newrelic-php5-<var>X.X.X.X-OS</var>
   ```

4. インストールスクリプトを実行します。

   ```
   ./newrelic-install
   ```

5. デフォルトの[アプリケーション名](/docs/site/naming-your-application)を分かりやすい名前に変更します。

6. ウェブサーバーを再起動します（Apache、Nginx、PHP-FPMなど）。

7. アプリケーションがデータを送信するまで数分待ってから、 [New Relic UI](/docs/apm/applications-menu/monitoring/apm-overview-page) でアプリケーションのパフォーマンスを確認します。
