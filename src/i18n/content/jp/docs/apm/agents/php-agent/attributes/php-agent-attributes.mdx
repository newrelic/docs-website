[アトリビュート](/docs/features/agent-attributes) は、イベントやトランザクションのプロパティを決定する情報を含むキー・バリュー・ペアです。これらのキーバリューペアは、アプリケーションをより深く理解し、New Relic でデータをアノテーションするのに役立ちます。デフォルトおよび [カスタムの](/docs/agents/manage-apm-agents/agent-data/collect-custom-attributes) 属性は、APM トランザクショントレース、分散トレース、エラー分析、および [APM](/docs/insights/explore-data/attributes/apm-default-attributes-insights) および [ブラウザイベント](/docs/insights/explore-data/attributes/browser-default-attributes-insights#browser-attributes-table) で見ることができます。

それぞれの送信先にどのような属性を送信するかをカスタマイズすることができます。ここでは、PHPエージェントの属性について、それらを有効または無効にする方法、および [の送信先に対してどの属性を含めるか、または除外するかを決定するためにエージェントが従うルールについて説明します](/docs/features/agent-attributes#destinations) 。

## アトリビュートの送信先変更 [#change-attribute-destination]

<Callout variant="tip">
  `newrelic.browser_monitoring.attributes.enabled` 以外のすべての宛先は、PHPエージェントのデフォルトで属性収集の対象となります。
</Callout>

New Relic [宛先に送信する属性を変更するには](/docs/apm/other-features/attributes/agent-attributes#destinations):

1. [`.enabled` デスティネーションプロパティ](/docs/agents/php-agent/attributes/enabling-disabling-attributes#destination-on-off) を変更することで、任意のデスティネーションをアトリビュートコレクションにオープンまたはクローズします。

2. [デフォルトの属性設定](#attributes) を変更するには、属性名をデスティネーションの [`.include` または `.exclude` プロパティ](/docs/agents/php-agent/attributes/enabling-disabling-attributes#destination-attribute-specific).

   どの設定が優先されるかは、 [Attribute rules](/docs/agents/php-agent/attributes/enabling-disabling-attributes#attruls) を参照してください。

## PHPエージェントのアップグレード [#upgrading]

PHP agent **4.9** 以上にアップグレードする場合は、設定ファイルをアップグレードしてください。非推奨のプロパティについては、 [属性の有効化と無効化](/docs/php/php-agent-enabling-and-disabling-attributes#deprecated) を参照してください。
