## 問題

New Relic PHP エージェントをインストールして使用するために、どのようなシステムパーミッションが必要かを判断する必要があります。

## 解決策

ほとんどのユースケースにおいて、New Relic for PHP をインストールするには root 権限が必要です。これは、インストールの過程で、root ユーザーのみが利用できる権限が必要となるためです。これらの権限には次のようなものがあります。

* PHP extensionsディレクトリへのリード/ライトアクセス
* PHPの設定データへの読み書き可能なアクセスは、PHP設定ディレクトリ（推奨）または **php.ini** のいずれかになります。
* WebサーバーやPHPプロセスプールを再起動する機能。

エージェントの実行には、root 権限は必要ありません。エージェントは、以下のファイルへの読み書き可能なアクセス権を必要とします（これらのファイルの場所は、 [PHP エージェント設定](/docs/php/php-agent-phpini-settings) でカスタマイズできます）。

* [エージェントのログファイルを保存するためのファイル](/docs/php/php-agent-phpini-settings#inivar-daemon-logfile)
* [エージェントのソケットエンドポイントを保存するためのファイル](/docs/php/php-agent-phpini-settings#inivar-daemon-port)
* [エージェントのプロセスIDを記録するためのファイル](/docs/php/php-agent-phpini-settings#inivar-daemon-pidfile)
