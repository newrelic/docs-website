[WordPress ウェブサイト用の New Relic をインストールした場合](/docs/agents/php-agent/installation/install-php-agent-shared-hosting-service) 、PHP エージェントは追加のメトリクスを受け取ります。 **WordPress** のページが New Relic のユーザーインターフェイスに表示されます。Go to [**one.newrelic.com**](http://one.newrelic.com/) **> APM> (select a WordPress app)**.

## 設定

PHP エージェントの `ini` 設定 `newrelic.framework.wordpress.hooks` を使用することで、アプリケーションが New Relic に送信する WordPress 固有のメトリクスを制御することができます。これは、 **5.3** 以上のバージョンではデフォルトで有効になっており、無効にすることもできます。

```
newrelic.framework.wordpress.hooks = false
```

この設定では、 `.hooks` という単語を使用していますが、フック、プラグイン、テーマを含む、New Relic に送信されるすべての WordPress メトリクスを制御します。

## メトリクス

PHPエージェントは、以下のようなメトリクスを受け取ります。

* **Hooks**: これらのメトリクスは、各 WordPress フック内で費やされた時間を示します。時間は、フックディスパッチ関数 `apply_filters`, `apply_filters_ref_array`, `do_action`, `do_action_ref_array` から算出されます。
* **プラグインとテーマ**: これらのメトリクスは、各WordPressプラグインとテーマ内で費やされた時間を示します。時間は、フックのディスパッチ関数から算出されます。

## New Relicパートナーとの統合 [#integration]

WordPress アプリケーションを APM と統合することで、WordPress アプリの **管理画面** ページから直接パフォーマンスを確認することができます。例えば、 [W3 Total Cache](https://newrelic.com/blog/best-practices/web-performance-optimization-automation) は、WordPressアプリを専門とするNew Relicのパートナーの一つです。
