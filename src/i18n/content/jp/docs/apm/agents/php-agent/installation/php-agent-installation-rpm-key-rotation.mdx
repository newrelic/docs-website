ベストプラクティスとセキュリティのため、2026 年 3 月以降、 New Relic PHPエージェント RPM パッケージの新しいリリースは新しいキーで署名されます。 PHPエージェントの操作を中断せずに継続するには、RPMパッケージからNew Relic PHPエージェントをインストールまたはアップグレードする前に、次のコマンドを実行して、新しい公開キーを受け入れてインポートする必要があります。

`rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public`

## キーを確認する

キーをインポートした後、次のコマンドを実行して、キーが信頼されていることを確認します。

`rpm -qa | grep gpg-pubkey-bd2e199c`

キーがインポートされ信頼されている場合、コマンドは次の出力を返します。

`gpg-pubkey-bd2e199c-6970e650`

## 3月までに新しいキーをインポートしないとどうなりますか?

新しいキーがシステムに追加されていない場合、PHP エージェントの新しいリリースのインストールとアップグレードは失敗し、 `Error: GPG check FAILED`などのエラーメッセージが表示されます。 エージェントの古いバージョンに固定されたインストールは、以前と同じように機能し続けます (ただし、このような場合でも、新しいキーをインポートすることをお勧めします)。
