2026 年 3 月、ベストプラクティスとセキュリティのために、 New Relic PHP エージェント RPM パッケージの新しいリリースは新しいキーで署名されました。 PHPエージェントの動作を中断なく継続するために、RPMパッケージからNew Relic PHPエージェントをインストールまたはアップグレードする前に、次のコマンドを実行して新しい公開鍵を受け入れてインポートする必要があります。

`rpm --import https://download.newrelic.com/php_agent/NEWRELIC_RPM_BD2E199C.public`

## キーを確認する

キーをインポートした後、次のコマンドを実行して、キーが信頼されていることを確認します。

`rpm -qa | grep gpg-pubkey-bd2e199c`

キーがインポートされ信頼されている場合、コマンドは次の出力を返します。

`gpg-pubkey-bd2e199c-6970e650`

## 新しいキーをインポートしていない場合はどうなりますか？

新しいキーがシステムに追加されていない場合、PHPエージェントの新しいリリースのインストールとアップグレードは失敗し、 `Error: GPG check FAILED`などのエラーメッセージが表示されます。 以前のバージョンのエージェントに固定されているインストールは、これまでと同様に引き続き機能します（ただし、そのような場合でも新しいキーをインポートすることをお勧めします）。
