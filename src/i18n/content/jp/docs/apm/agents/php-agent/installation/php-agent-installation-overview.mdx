## PHPコンポーネント [#components]

当社の[PHPエージェント](/docs/agents/php-agent/getting-started/new-relic-php)は、次の2つの基本コンポーネントで構成されています。

* アプリケーションからデータを収集するPHP拡張モジュール
* データをNew Relicに送信するローカルプロキシデーモン

エージェントはシステムの一部となるため、インストレーションや設定を行う際には、ウェブブラウザ、オペレーティングシステム、システムのアクセス権限、PHPのインストレーション、パッケージマネージャなど、お使いのシステムを設定することが重要です。

サポート対象のPHPのリリース、フレームワーク、およびその他のシステム要件の詳細については、[PHPの互換性と要件](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements)をご覧ください。

## エージェントのインストール [#install-overview]

システムにエージェントをインストールするには、以下の一般手順に従ってください。エージェントのインストレーションパッケージまたはtarアーカイブには、一部のインストレーションタスクを自動化する対話形式の[**`newrelic-install`**スクリプト](/docs/agents/php-agent/installation/newrelic-install-script)が含まれています。

<Callout variant="tip">
  アプリケーションが頻繁に無反応になる場合は、アプリケーションやサイトと対話してデータを生成することを推奨します。これは、エージェントが最初の接続を確立するのに役立ちます。
</Callout>

1. New Relicアカウントをまだお持ちでない場合は、[New Relicアカウントを新たに作成します](https://newrelic.com/signup)。永久無料です。

2. お使いのシステムが、適切な[システムのアクセス権限](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements)を含めて、[エージェントの要件](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#permissions)を満たしていることを確認します。

3. **アカウント設定**から、自身の<LicenseKey/>情報をコピーします。

4. エージェントのパッケージまたはtarアーカイブをシステムにインストールします。

   <CollapserGroup>
     <Collapser
       id="install-redhat-centos"
       title="RedHatまたはCentOS"
     >
       RedHatまたはCentOSにエージェントをインストールする最も一般的な方法は、インストレーションパッケージ（**.rpm**）やパッケージマネージャを使用する方法です。New Relicのリポジトリからパッケージを取得するように[パッケージマネージャに指示](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#tell-rpm)するか、New Relicのウェブサイトから直接 [.rpm**ファイル**を](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos#download-rpm)ダウンロードできます。

       お好みのパッケージマネージャ（`yum`または`rpm`）を使用して[パッケージをインストール](/docs/agents/php-agent/installation/php-agent-installation-redhat-centos)し、**newrelic-install**スクリプトを実行します。
     </Collapser>

     <Collapser
       id="intall-ubuntu-debian"
       title="UbuntuまたはDebian"
     >
       `apt-get`または`dpkg`コマンドを使用して[パッケージをインストール](/docs/agents/php-agent/installation/php-agent-installation-ubuntu-debian)します。通常、**newrelic-install**スクリプトを実行する必要はありません。
     </Collapser>

     <Collapser
       id="install-tar"
       title="Tarアーカイブ"
     >
       **.tar**アーカイブは、すべての[サポートされているオペレーティングシステム](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements#operating-systems)で使用できる最も一般的な方法です。以下のいずれかの状況が該当する場合は、お使いシステム用に[圧縮されたtarアーカイブをダウンロード](/docs/agents/php-agent/installation/php-agent-installation-tar-file#download)します。

       * お使いのオペレーティングシステムがここに掲載されていない。

       * パッケージマネージャを使用していない。

       * その他の方法では正常にインストールできない。

         [サポート対象のプラットフォーム](/docs/agents/php-agent/installation/php-agent-installation-tar-file#install)にエージェントの[tarアーカイブをインストール](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements)します。
     </Collapser>
   </CollapserGroup>

5. デフォルトの[アプリケーション名](/docs/site/naming-your-application)を分かりやすい名前に変更します。

6. オプション： その他のエージェント[設定](/docs/agents/php-agent/configuration/php-agent-configuration)の設定を変更して、インストールをさらにカスタマイズします。

7. ウェブサーバーを再起動します（Apache、Nginx、PHP-FPMなど）。

8. **推奨：**特にアプリケーションの活動が不定期の場合に、PHP エージェントを確実に起動できるようにするには、アプリを数秒間使用してデータを生成します。

9. アプリケーションがNew Relicにデータを送信し始めるまで数分間待ちます。

その後、[New Relic UIでアプリケーションのパフォーマンスを確認します。](/docs/apm/applications-menu/monitoring/apm-overview-page)

## APMおよびインフラストラクチャデータのログを表示 [#logs-context]

また、ログとアプリケーションのデータを統合して、トラブルシューティングをより簡単かつ迅速にできます。[コンテキストのログ](/docs/logs/logs-context/configure-logs-context-php/)では、エラーとトレースに関連するログメッセージをアプリケーションのUIで直接表示できます。また、Kubernetesクラスタなど、[インフラストラクチャデータ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)のコンテキストでログを表示することもできます。別のUIページに切り替える必要はありません。

## パートナーシップアカウントによるPHPのインストール [#partnership-accounts]

New Relicのパートナーシップを使用してPHPエージェントをインストールする管理者の場合は、インストレーション手順が異なる場合があります。詳細については、以下を参照してください。

* [AWS](/docs/accounts-partnerships/partnerships/partner-based-installation/amazon-web-services-aws-users)
* [Heroku](/docs/agents/php-agent/installation/php-agent-heroku)
* [Magento](/docs/accounts-partnerships/partnerships/partner-based-installation/log-install-new-relic-partners#magento)
* [共有ホスティングプロバイダー](/docs/agents/php-agent/installation/install-php-agent-shared-hosting-service)
* [その他のパートナーシップのインストレーション手順](/docs/accounts-partnerships/partnerships/partner-based-installation/log-install-new-relic-partners)

すべてのパートナーがPHPエージェントをサポートしているわけではありません。
