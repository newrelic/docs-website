## 問題

New Relic for PHP のインストール時に問題が発生したが、原因がわからない。

## 解決

New Relic のログを監視することで、問題を診断できるかもしれません。New Relic PHP エージェントおよびデーモンは、それぞれ独自のログを持っています。ログを監視するには

1. 両方のログの内容を確認します。

   * <DNT>
       **[Default agent log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-logfile)**
     </DNT>

     ： `/var/log/newrelic/php_agent.log`

   * <DNT>
       **[Default daemon log location](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)**
     </DNT>

     ： `/var/log/newrelic/newrelic-daemon.log`

2. エージェントのログ レベルを`verbosedebug`に設定し、デーモンのログ レベルを`debug`に設定します。詳細については：

   * [エージェント](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)

     <DNT>
       [**loglevel**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
     </DNT>

     [を](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)

     <DNT>
       [**verbosedebug**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
     </DNT>

   * [デーモン](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

     <DNT>
       [**loglevel**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)
     </DNT>

     [を設定しています](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

     <DNT>
       [**debug**](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)
     </DNT>

3. アプリを再起動します。

4. ログレベルを適切な値に設定した上で、5から10分のログを収集します。

5. ログを調べ、その情報をもとに問題を診断します。

6. トラブルシューティングが終了したら、ログのレベルをリセットします。`verbosedebug`または`debug`のままにしないでください。

<Callout variant="caution">
  ログファイルのサイズに注意してください。アプリケーションによっては、あっという間に大きくなってしまうことがあります。
</Callout>
