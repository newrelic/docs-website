## 問題

New Relic for PHP のインストール時に問題が発生したが、原因がわからない。

## 解決

New Relic のログを監視することで、問題を診断できるかもしれません。New Relic PHP エージェントおよびデーモンは、それぞれ独自のログを持っています。ログを監視するには

1. 両方のログの内容を確認します。

   * **[デフォルトのエージェント ログの場所](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-logfile)**: `/var/log/newrelic/php_agent.log`
   * **[デフォルトのデーモン ログの場所](/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-logfile)**: `/var/log/newrelic/newrelic-daemon.log`

2. エージェントのログ レベルを`verbosedebug`に設定し、デーモンのログ レベルを`debug`に設定します。詳細については：

   * [エージェント **ログレベル** から **verbosedebug を設定する。**](https://docs.newrelic.com/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-loglevel)
   * [デーモン **ログレベル** から **デバッグを設定する。**](https://docs.newrelic.com/docs/apm/agents/php-agent/configuration/php-agent-configuration/#inivar-daemon-loglevel)

3. アプリを再起動します。

4. ログレベルを適切な値に設定した上で、5から10分のログを収集します。

5. ログを調べ、その情報をもとに問題を診断します。

6. トラブルシューティングが終了したら、ログのレベルをリセットします。`verbosedebug`または`debug`のままにしないでください。

<Callout variant="caution">
  ログファイルのサイズに注意してください。アプリケーションによっては、あっという間に大きくなってしまうことがあります。
</Callout>
