## 問題

[New Relic PHPエージェントをインストール](/docs/agents/php-agent/installation/php-agent-installation-overview)し、トラフィックを生成してから5分以上待機した後、New Relic UIにデータが表示されません。

## 解決策

アプリケーションに対してトラフィックを生成してから5分以上待機してもデータが表示されない場合は、一般的な問題を自動検出し、トラブルシューティングを推奨する[New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics)を使用してください。この操作を実行しても問題が解決しない場合は、次を試してください：

<table>
  <thead>
    <tr>
      <th>
        PHPエージェントのトラブルシューティング
      </th>

      <th>
        コメント
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        互換性
      </td>

      <td>
        お使いのシステムがPHPエージェントの[互換性と要件](/docs/agents/php-agent/getting-started/php-agent-compatibility-requirements)を満たしていることを確認します。
      </td>
    </tr>

    <tr>
      <td>
        非標準PHPバージョン
      </td>

      <td>
        非標準バージョンのPHPを使用している場合、[高度なインストレーション手順](/docs/php/php-agent-installation-non-standard-php)にしたがって、デフォルトのインストーラがお使いのPHPバージョンを検出できることを確認します。
      </td>
    </tr>

    <tr>
      <td>
        アプリケーション名
      </td>

      <td>
        アプリケーションに[分かりやすい固有の名前](/docs/agents/manage-apm-agents/app-naming/name-your-application#app-name)がついていることを確認します。たとえば、同じ名前（`PHP Application`など）のアプリケーションが複数ある場合、これら各アプリケーションからのデータはデフォルトアプリケーション名にロールアップされるため、個々のアプリケーションがレポートしていないように見受けられます。
      </td>
    </tr>

    <tr>
      <td>
        Webサーバー
      </td>

      <td>
        Webサーバー (Apache、Nginx、PHP-FPMなど) を再起動し、データが表示されるまで数分間待機します。
      </td>
    </tr>

    <tr>
      <td>
        `phpinfo()`
      </td>

      <td>
        `phpinfo()`をチェックして、[PHPエージェントをインストールしたことを確認](/docs/agents/php-agent/troubleshooting/using-phpinfo-verify-php)し、さらにその<InlinePopover type="licenseKey"/>が正しいことを確認します。
      </td>
    </tr>

    <tr>
      <td>
        SELinux
      </td>

      <td>
        お使いのシステムがSELinuxを使用している場合、PHPエージェントと連動するように[SELinuxを設定](/docs/agents/php-agent/troubleshooting/data-stops-reporting-while-using-selinux#solution)します。
      </td>
    </tr>

    <tr>
      <td>
        ログファイル
      </td>

      <td>
        1. エージェントとデーモンの両方が[ログファイルに書き込み](/docs/agents/php-agent/troubleshooting/generating-logs-troubleshooting-php)していることを確認します。
        2. そうでない場合、[エージェントを再インストール](/docs/agents/php-agent/installation/php-agent-installation-overview)してください。
        3. ログファイルのアクセス権を確認して、ログ所有者 (オーナー) がNew Relicユーザーと同一であることを確かめます。
      </td>
    </tr>

    <tr>
      <td>
        PHPエージェントのアクセス権
      </td>

      <td>
        PHPエージェントを使用するのに[適切なアクセス権](/docs/agents/php-agent/troubleshooting/determining-permissions-requirements)があることを確認します。
      </td>
    </tr>

    <tr>
      <td>
        その他
      </td>

      <td>
        これらの解決策で問題が解決されない場合は、次のトラブルシューティング手順を参照してください：

        * [エージェントがPHPの更新後に停止する](/docs/agents/php-agent/troubleshooting/agent-stops-working-after-updating-php)
        * [以前データをレポートしていたPHPエージェントがレポートを停止する](/docs/agents/php-agent/troubleshooting/data-stops-reporting)
      </td>
    </tr>
  </tbody>
</table>
