コード レベルのメトリクスにより、 [New Relic CodeStream 拡張機能](/docs/codestream/how-use-codestream/performance-monitoring#code-level) を使用する開発者は、コード内の個々のメソッドと一緒に、IDE でコンテキストに応じて表示される APM データを確認できます。 これにより、開発者は、コードを記述およびレビューする際にパフォーマンスの問題に積極的に対処できます。

## 始めましょう [#get-started]

まず、サポートされているIDEにNew Relic CodeStream拡張機能を[インストール](/docs/codestream/start-here/install-codestream)し、ログインします。

<Callout variant="important">
  コードレベルのメトリクスは、Node.js エージェントのバージョン 9.7.5 以降で使用でき、バージョン 10.5.0 以降ではデフォルトで有効になります。この構成を変更するには、 [ドキュメントを](/docs/apm/agents/nodejs-agent/installation-configuration/nodejs-agent-configuration#code-level-metrics)参照してください。
</Callout>

## エージェントの属性 [#attributes]

Node.js エージェントは、次の属性を報告し、アプリケーションの[Web フレームワーク](/docs/apm/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent#support)によって生成されるほとんどのスパンにアタッチします。

<Callout variant="important">
  場合によっては、アプリケーション内で呼び出され、IDE 内に表示されないベース フレームワーク コードをエージェントが直接ラップします。
</Callout>
