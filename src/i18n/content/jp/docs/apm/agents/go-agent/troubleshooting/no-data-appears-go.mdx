## 問題

[、Golang アプリ用に New Relic Go エージェント](/docs/agents/go-agent/get-started/install-new-relic-go) をインストールしました。トラフィックを発生させて数分待っても、アプリはNew Relicにデータを報告しません。

## 解決策

Web アプリケーションにリクエストを送信した後、2 ～ 3 分以内に APM の UI にデータが表示されるはずです。それでもアプリケーションの [**まとめ** ページ](/docs/apm/applications-menu/monitoring/apm-overview-page) にデータが表示されない場合は、以下のトラブルシューティングをお試しください。

1. [アプリの名前](/docs/agents/go-agent/instrumentation/go-agent-configuration#app-name) と [ライセンスキー](/docs/agents/go-agent/instrumentation/go-agent-configuration#license) を、Goエージェントの設定の [`newrelic.Config` struct](/docs/agents/go-agent/instrumentation/go-agent-configuration) で確認してください。
2. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics)を使用すると、問題を自動的に特定できる場合があります。
3. アプリのトランザクションを [`newrelic.WrapHandleFunc` または `Application.StartTransaction` and `Transaction.End`](/docs/agents/go-agent/get-started/instrument-go-transactions) でインストルメント化したことを確認してください。
4. Webサーバーを再起動します。
5. [Go Logger インターフェイスを使用して、 `Debug` レベルのログ](/docs/agents/go-agent/instrumentation/go-agent-logging) を生成し、それらのログにエラーがないか確認します。
6. 必要に応じて、 [最新の New Relic Go エージェントリリース](/docs/release-notes/agent-release-notes/go-release-notes) にアップグレードしてください。
