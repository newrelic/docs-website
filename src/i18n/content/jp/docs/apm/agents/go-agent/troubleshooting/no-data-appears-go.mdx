## 問題

[、Golang アプリ用に New Relic Go エージェント](/docs/agents/go-agent/get-started/install-new-relic-go) をインストールしました。トラフィックを発生させて数分待っても、アプリはNew Relicにデータを報告しません。

## 解決

Web アプリケーションにリクエストを送信すると、データが <InlinePopover type="apm"/>2 ～ 3 分以内に UI が表示されます。 それでもアプリの [**\[概要\]** ページ](/docs/apm/applications-menu/monitoring/apm-overview-page)にデータが表示されない場合は、次のトラブルシューティングのヒントをお試しください:

1. Go エージェント構成の [`newrelic.Config`構造](/docs/agents/go-agent/instrumentation/go-agent-configuration) 体で [アプリの名前](/docs/agents/go-agent/instrumentation/go-agent-configuration#app-name) と [ライセンス キー](/docs/agents/go-agent/instrumentation/go-agent-configuration#license)を確認します。
2. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics)を使用すると、問題を自動的に特定できる場合があります。
3. アプリのトランザクションを [`newrelic.WrapHandleFunc`または`Application.StartTransaction`と`Transaction.End`](/docs/agents/go-agent/get-started/instrument-go-transactions)で計測したことを確認してください。
4. Webサーバーを再起動します。
5. [Go Logger インターフェースを使用して`Debug`レベルのログを生成し](/docs/agents/go-agent/instrumentation/go-agent-logging)、それらのログにエラーがないか確認してください。
6. 必要に応じて、 [最新の New Relic Go エージェントリリース](/docs/release-notes/agent-release-notes/go-release-notes) にアップグレードしてください。
