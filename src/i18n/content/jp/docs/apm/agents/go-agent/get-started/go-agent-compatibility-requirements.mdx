New Relicアカウントをまだお持ちでない場合は、[New Relicアカウントを新たに作成します](https://newrelic.com/signup)。永久無料です。[Go向けNew Relicをインストール](/docs/agents/go-agent/installation/install-new-relic-go)する前に、システムがこれらの要件を満たしていることを確認してください。

## Golangのバージョン [#golang-release]

New RelicはGolang 1.7以上をサポートします。

## 動作環境 [#operating-systems]

Goエージェントは Linux、macOS、Windowsをサポートします。　Goエージェントを[Google App Engine (GAE)フレキシブル環境](/docs/agents/go-agent/get-started/install-new-relic-go-agent-gae-flexible-environment)で使うことも可能です。

## インテグレーション [#frameworks]

以下の[インテグレーションパッケージ](https://godoc.org/github.com/newrelic/go-agent/_integrations)は、ベースとなる[newrelic](https://godoc.org/github.com/newrelic/go-agent)パッケージを拡張して、他のフレームワークやライブラリにも対応します。[newrelic](https://godoc.org/github.com/newrelic/go-agent)パッケージのプリミティブを使用することで、インテグレーションパッケージを持たないフレームワークとデータベースもインストゥルメンテーションできます。こうしたプリミティブを使用してデータベースのインストゥルメンテーションを行うことの詳細については、Goエージェントの[GitHubにあるデータストアセグメント文書](https://github.com/newrelic/go-agent/blob/master/GUIDE.md#datastore-segments)を参照してください。

次のインテグレーションパッケージは、[GitHubのnrginの例](https://godoc.org/github.com/newrelic/go-agent/v3/newrelic)で示した通り、[newrelic](https://github.com/newrelic/go-agent/blob/master/v3/integrations/nrgin/example/main.go)パッケージと共にインポートする必要があります。

## データベースとインスタンスレベルのパフォーマンス

New Relicは[様々なデータベースとデータベースドライバのインスタンス詳細](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues)を収集します。APMにおいて個々のインスタンスおよびデータベース情報の種類が見えるかどうかは、エージェントのバージョンによって異なります。

New RelicのGoエージェント[バージョン1.4以降](/docs/release-notes/agent-release-notes/go-release-notes/go-agent-14)は、すべてのデータベースドライバのインスタンス詳細に対応しています。
