## 問題

[ログデータ](/docs/agents/java-agent/troubleshooting/no-log-file-java) が `SEVERE:` または `ERROR:` with `NullPointerException` の問題を示している場合、レポート作成に支障をきたしている問題があるかもしれません。

## 解決策

これらのトラブルシューティングを始める前に、New Relic の Java エージェントの [最新バージョン](/docs/agents/java-agent/installation/upgrade-java-agent) を持っていることを確認してください。

1. Javaエージェントのログファイルを大文字小文字を区別して検索し、 `SEVERE:` and `ERROR: NullPointerException` の行を探してください。
2. これらのメッセージの行を見つけた場合は、 [トラブルシューティング情報を集めて](/docs/agents/java-agent/troubleshooting/providing-troubleshooting-information-java) New Relicでサポートを受けてください。

## 原因

バージョン3.0.0から3.2.3までのNew RelicのJavaエージェントでは、エラーメッセージが誤って記録されることがありました。例えば、以下のようになります。

```
ERROR: javax/servlet/ServletOutputStream is marked as a weaved class, but no methods are matched to be weaved.
```

これは全くの誤りであり、心配する必要はありません。ただし、これらのバージョンは使用しないでください。代わりに、New Relic の Java エージェントの [最新バージョン](/docs/agents/java-agent/installation/upgrade-java-agent) にアップデートしてください。
