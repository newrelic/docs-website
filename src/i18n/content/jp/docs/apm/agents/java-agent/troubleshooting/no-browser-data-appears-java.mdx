## 問題

あなたが[インストールしたのは、 <InlinePopover type="browser"/>](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent)Java アプリのエージェントですが、アプリは[ブラウザー UI](/docs/browser/new-relic-browser/getting-started/browser-overview-page-website-performance-summary)にデータを報告しません。

## 解決

アプリケーションサーバーがJasper JSPコンパイラーを使用していない場合（例えば、WebSphereやWebLogicを使用している場合）、自動計測機能は利用できません。代わりに、 [手動でブラウザをインストルメントする必要があります](/docs/java/page-load-timing-in-java#manual_instrumentation) 。

アプリサーバーが JSP コンパイラを使用しているにもかかわらずブラウザデータを受信しない場合は、構成の競合が発生している可能性があります。New Relic の Java エージェントを使用すると、アプリ名を`newrelic.yml`構成ファイルに設定するか、 `web.xml` `display-name`で自動的に設定できます。

自分のアプリが正しい名前で報告されているかどうかを確認するためです。

1. [`finest`レベルのログを](/docs/agents/java-agent/troubleshooting/generating-logs-troubleshooting)一時的に生成します。
2. ログ ファイルで`"enabled for"`を検索します。[ブラウザのタイミング データを](/docs/agents/java-agent/instrumentation/page-load-timing-java)報告しているアプリのリストにアプリ名が表示されていることを確認します。
3. リストされたアプリ名が予想される名前と一致しない場合は、 `newrelic.yml`と`web.xml` `display-name`設定で同じアプリ名を設定していることを確認してください。
