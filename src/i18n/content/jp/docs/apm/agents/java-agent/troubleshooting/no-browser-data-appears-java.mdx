## 問題

[ブラウザーモニタリングエージェント](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) を Java アプリにインストールしましたが、アプリが [ブラウザーUI](/docs/browser/new-relic-browser/getting-started/browser-overview-page-website-performance-summary) でデータを報告しません。

## 解決策

アプリケーションサーバーがJasper JSPコンパイラーを使用していない場合（例えば、WebSphereやWebLogicを使用している場合）、自動計測機能は利用できません。代わりに、 [手動でブラウザをインストルメントする必要があります](/docs/java/page-load-timing-in-java#manual_instrumentation) 。

アプリサーバーが JSP コンパイラを使用しているにもかかわらず、ブラウザデータを受信できない場合は、設定の不一致が考えられます。New Relic の Java エージェントでは、アプリ名を `newrelic.yml` 設定ファイルで設定するか、 `web.xml` `display-name` で自動的に設定することができます。

自分のアプリが正しい名前で報告されているかどうかを確認するためです。

1. Temporarily generate [`finest` level logs](/docs/agents/java-agent/troubleshooting/generating-logs-troubleshooting).
2. `" enabled for"` のログファイルを検索してください。 [ブラウザのタイミングデータ](/docs/agents/java-agent/instrumentation/page-load-timing-java) を報告しているアプリのリストに自分のアプリ名が表示されていることを確認します。
3. リストアップされたアプリ名が予想される名前と一致しない場合は、 `newrelic.yml` と `web.xml` `display-name` の設定で同じアプリ名を設定していることを確認してください。
