[ブラウザモニタリング](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) では、JavaScript スニペット（または"エージェント" ）を使用してアプリのウェブページを計測することで、ユーザーが Java アプリケーションをどのように操作しているかを可視化することができます。UIを介してブラウザエージェントをインストールするには、 [ブラウザエージェントのインストール](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) の手順に従ってください。例えば、以下のようになります。

* APM監視対象のアプリを有効にします。
* コピー/ペースト方式で有効にする。
* REST APIを使用して有効にする。

[また、このドキュメントで説明されているように、JavaエージェントAPIを使用してウェブページを手動でインストルメント](#manual_instrumentation) することもできます。

## JavaエージェントAPIの使用 [#manual_instrumentation]

お使いのフレームワークで、当社のUI、コピー／ペースト方法、またはREST APIを使用してブラウザ監視を有効にできない場合は、ウェブページの監視を手動で行うことができます。これを行うには、当社の [Java エージェント API](/docs/agents/java-agent/api-guides/guide-using-java-agent-api) を使用してください。

1. [最新のJavaエージェントのバージョンを使用していることを確認してください](/docs/release-notes/agent-release-notes/java-release-notes) 。

2. オートインストルメントを無効にします。

   <CollapserGroup>
     <Collapser
       id="disabling-auto-all"
       title="すべてのページでオートインストルメントを無効にする"
     >
       1. `newrelic.yml`で `browser_monitoring` の下の [`auto_instrument`](/docs/agents/java-agent/configuration/java-agent-configuration-config-file#bm-auto_instrument) フラグを `false` に設定します。

          ```yml
          browser_monitoring:
            auto_instrument: false
          ```

       2. アプリケーションを再起動します。

       3. アプリサーバーのワークキャッシュをフラッシュします。

          ワークキャッシュをフラッシュすると、アプリサーバーはJSPの再コンパイルを余儀なくされますが、この時に自動計測が行われます。
     </Collapser>

     <Collapser
       id="disable-auto-specific"
       title="特定のページでのみオートインストルメントを無効にする"
     >
       `browser_monitoring` スタンザの下で `disabled_auto_pages` フラグを使用します。たとえば、 `testpage_1.jsp` および `testpage_2.jsp`で自動計測を無効にするには、次を使用します。

       ```yml
       browser_monitoring:
         disabled_auto_pages: /WEB-INF/jsp/testpage_1.jsp, /WEB-INF/jsp/testpage_2.jsp
       ```
     </Collapser>
   </CollapserGroup>

3. アプリケーション クラス パスに`newrelic-api.jar`を追加し[て、Java エージェント API クラスにアクセスし](/docs/agents/java-agent/api-guides/guide-using-java-agent-api#api)ます。

4. 時間追跡を開始するための適切なメソッドを追加します (例: `com.newrelic.api.agent.NewRelic.getBrowserTimingHeader()` 。

   時間追跡を停止するための適切なメソッドを追加します (例: `com.newrelic.api.agent.NewRelic.getBrowserTimingFooter()` 。

   使用しているフレームワークに応じて、適切な方法を見つけてください。

   <CollapserGroup>
     <Collapser
       id="manual-jsp"
       title="JSP マニュアルインストルメント"
     >
       ヘッダーは`<head>`タグのできるだけ先頭に挿入しますが、メタ タグの後に挿入します。`</body>`タグの前にフッターを挿入します。

       ```html
       <!DOCTYPE html>

       <html>
       <head>
         <title>EXAMPLE</title>
         <meta charset="utf-8">
         <meta name="description" content="Example header and footer call">;
         <%= com.newrelic.api.agent.NewRelic.getBrowserTimingHeader() %>
       </head>

       <body>
         <%= com.newrelic.api.agent.NewRelic.getBrowserTimingFooter() %>
       </body>
       </html>
       ```
     </Collapser>

     <Collapser
       id="manual-html"
       title="Raw HTMLのマニュアルインストルメント"
     >
       生のHTMLを使って手動で計測するには、 [HTTPレスポンスにページロードタイミングコードを含める](/docs/agents/java-agent/custom-instrumentation/java-agent-api-example-program#include-browser) 。
     </Collapser>

     <Collapser
       id="manual-thymeleaf-2-1"
       title="Thymeleaf 2.1以上のマニュアルインストルメント"
     >
       ```html
       <head>
         ...
         <div th:remove="tag" th:utext="${T(com.newrelic.api.agent.NewRelic).getBrowserTimingHeader()}"></div>
       </head>

       <body>
         ...
         <div th:utext="${T(com.newrelic.api.agent.NewRelic).getBrowserTimingFooter()}"></div>
       </body>
       ```
     </Collapser>

     <Collapser
       id="manual-coldfusion-cfml"
       title="ColdFusion CFMLページの手動インストルメント"
     >
       ```html
       <cfobject type="Java" class="com.newrelic.api.agent.NewRelic" name="newRelic" />
       <cfset header=newRelic.getBrowserTimingHeader() />
       <cfoutput>#header#</cfoutput>
       ...
       ...
       <cfset footer=newRelic.getBrowserTimingFooter() />
       <cfoutput>#footer#</cfoutput>
       ```
     </Collapser>

     <Collapser
       id="manual-jsf"
       title="Java Server Faces（JSF）フレームワーク"
     >
       JSFでは手動での計測はできません。ブラウザのJavaScriptスニペットをアプリのウェブページに挿入するには、 [コピー/ペースト方法](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser#copy-paste-app) を使用します。
     </Collapser>

     <Collapser
       id="manual-play-groovy"
       title="Play Groovyテンプレート・マニュアル・インストゥルメンテーション"
     >
       ```groovy
       ${com.newrelic.api.agent.NewRelic.getBrowserTimingHeader().raw()}
       ...
       ...
       ${com.newrelic.api.agent.NewRelic.getBrowserTimingFooter().raw()}
       ```
     </Collapser>

     <Collapser
       id="manual-play-2-2"
       title="Play 2.2以上 Scalaテンプレート・マニュアル・インストゥルメンテーション"
     >
       以下を`build.sbt`に追加して、3.xx を[最新のエージェント バージョン](/docs/release-notes/agent-release-notes/java-release-notes)に置き換えます。

       ```scala
       libraryDependencies += "com.newrelic.agent.java" % "newrelic-api" % "3.x.x"
       ```

       `main.scala.html`に、次を追加します。

       ```java
       @Html(com.newrelic.api.agent.NewRelic.getBrowserTimingHeader())
       ...
       ...
       @Html(com.newrelic.api.agent.NewRelic.getBrowserTimingFooter())
       ```
     </Collapser>

     <Collapser
       id="manual-play-2-1"
       title="Play 2.1 Scalaテンプレート・マニュアル・インストゥルメンテーション"
     >
       2.xx を最新の[エージェント バージョン](/docs/release-notes/agent-release-notes/java-release-notes)に置き換えて、次を`Build.scala`に追加します。

       ```scala
       appDependencies += "com.newrelic.agent.java" % "newrelic-api" % "2.x.x"
       ```

       `main.scala.html`に、次を追加します。

       ```java
       @Html(com.newrelic.api.agent.NewRelic.getBrowserTimingHeader())
       ...
       ...
       @Html(com.newrelic.api.agent.NewRelic.getBrowserTimingFooter())
       ```
     </Collapser>

     <Collapser
       id="manual-velocity"
       title="Apache Velocity Toolsのマニュアルインストルメント"
     >
       Velocity Tools を使用している場合は、 `toolbox.xml`で`NewRelic`ツールを定義します:

       ```xml
       <toolbox>
         <tool>
           <key>NewRelic</key>
           <scope>application</scope>
           <class>com.newrelic.api.agent.NewRelic</class>
         </tool>
       </toolbox>
       ```

       ベロシティ テンプレートで`NewRelic` API を呼び出します。

       ```java
       $NewRelic.getBrowserTimingHeader()
       ...
       ...
       $NewRelic.getBrowserTimingFooter()
       ```
     </Collapser>

     <Collapser
       id="enable-apache"
       title="Apache Velocityのマニュアルインストルメント"
     >
       1. すべてのブラウザ ページ ビューが 1 つの `/velocity` トランザクションの下にグループ化されないようにする (および [指標のグループ化の問題](/docs/agents/manage-apm-agents/troubleshooting/metric-grouping-issues)を回避する) には、 `newrelic.yml` ファイルで `enable_auto_transaction_naming` 設定を無効にします。

          ```yml
          enable_auto_transaction_naming: false
          ```

       2. `newrelic-api.jar`を Tomcat のクラスパス (通常は`.../tomcat/bin/setenv.sh` ) に追加します。

          ```sh
          CLASSPATH=$CLASSPATH:/opt/newrelic/newrelic-api.jar
          ```

          また

          この jar ファイルへの参照を、そのファイル内の既存の`CLASSPATH=...`行の末尾に追加します。

       3. 各アプリで`WEB-INF/tools.xml`を見つけて、次のタグを追加します。

          ```xml
          <toolbox scope="application"> 
            <tool class="com.newrelic.api.agent.NewRelic" key="NewRelic" /> 
          </toolbox>
          ```

          `<toolbox scope="application">`タグが既に存在する場合は、タグ内に`<tool ... />`を挿入します。

       4. 各アプリで、アプリケーションの他のページが使用する Velocity テンプレートを編集します。`<head>`スタンザ内で、 `<link>`および`<script>`宣言の前、メタ タグの後に次のタグを追加します。

          ```java
          $NewRelic.getBrowserTimingHeader()
          ```

       5. `<body>`スタンザ内の`</body>`タグの直前の行として、本文の一番下に次のタグを追加します。

          ```java
          $NewRelic.getBrowserTimingFooter()
          ```

       6. Tomcatサーバを再起動する。

          * `Linux/Unix`を使用している場合は、次のコマンド ラインを`Users/Tomcat/bin`ディレクトリに追加します。

            ```
            ./shutdown.sh
            ./startup.sh
            ```

          * `Windows`を使用している場合は、 `Users/Tomcat/bin`ディレクトリに移動し、次のコマンド ラインを追加します。

            ```
            % shutdown.bat
            % startup.bat
            ```
     </Collapser>
   </CollapserGroup>

5. ログ ファイルを調べて、エージェントが正しく計測されていることを確認します。`newrelic.yml`の`log_level`設定を`log_level: finer`に設定し、次のメッセージを確認します:
