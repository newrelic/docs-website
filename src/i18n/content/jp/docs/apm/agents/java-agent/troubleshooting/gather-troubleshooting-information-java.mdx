Javaエージェントのトラブルシューティング手順（ [No data appears](/docs/apm/agents/java-agent/troubleshooting/no-data-appears-java) やそのカテゴリの他のドキュメントなど）で問題が解決しない場合は、 [ログファイル](#logfiles) を使用して問題を解決してください。さらにサポートが必要な場合は、 [support.newrelic.com](https://support.newrelic.com) までご連絡ください。

      <td>
        ログファイル
      </td>

      <td>
        Javaエージェントのログファイルは表示されていますか？

        * 「はい」の場合は、JVM の起動から数分間実行される[`finest`レベルのログを](/docs/agents/java-agent/troubleshooting/generate-debug-logs-troubleshooting-java)含めます。
        * 「いいえ」の場合は、JVM が`newrelic.jar`ファイルを開いていることを示す JVM の起動ログを含めます。
      </td>
    </tr>

    <tr>
      <td>
        リンク
      </td>

      <td>
        期待したデータが表示されないページの URL を含めます。 アプリケーション データがまったく表示されない場合は、アプリの[APM <DNT>**Summary**</DNT>ページ](/docs/apm/applications-menu/monitoring/apm-overview-page)の URL を送信します。
      </td>
    </tr>

    <tr>
      <td>
        あなたのアプリの `newrelic.yml`
      </td>

      <td>
        アプリの[`newrelic.yml`構成ファイルには、](/docs/agents/java-agent/troubleshooting/no-log-file-java)[ライセンス キー](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)とプロキシ情報 (必要な場合) が含まれています。この情報を共有する前に削除してください。
      </td>
    </tr>
