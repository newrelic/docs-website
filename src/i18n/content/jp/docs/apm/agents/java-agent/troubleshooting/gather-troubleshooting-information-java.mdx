Javaエージェントのトラブルシューティング手順（ [No data appears](/docs/apm/agents/java-agent/troubleshooting/no-data-appears-java) やそのカテゴリの他のドキュメントなど）で問題が解決しない場合は、 [ログファイル](#logfiles) を使用して問題を解決してください。さらにサポートが必要な場合は、 [support.newrelic.com](https://support.newrelic.com) までご連絡ください。

    <tr id="logfiles">
      <td>
        ログファイル
      </td>

      <td>
        Javaエージェントのログファイルは表示されていますか？

        * そうであれば、 [`最高級の` レベルのログ](/docs/agents/java-agent/troubleshooting/generate-debug-logs-troubleshooting-java) を、JVMの起動時から数分間実行してください。
        * そうでない場合は、JVMが `newrelic.jar` ファイルを開いていることを示すJVMの起動ログを含めてください。
      </td>
    </tr>

    <tr>
      <td>
        リンク
      </td>

      <td>
        期待通りのデータが表示されていないページのURLを記載してください。アプリケーションのデータがまったく表示されていない場合は、アプリケーションの [APM **Summary** page](/docs/apm/applications-menu/monitoring/apm-overview-page) のURLを送信してください。
      </td>
    </tr>

    <tr>
      <td>
        あなたのアプリの `newrelic.yml`
      </td>

      <td>
        あなたのアプリの [`newrelic.yml` 設定ファイル](/docs/agents/java-agent/troubleshooting/no-log-file-java) には、あなたの [ライセンスキー](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key) とプロキシ情報（必要な場合）が含まれています。共有する前にこれらの情報を削除してください。
      </td>
    </tr>
