詳細なデバッグログは、標準的なRubyエージェントのインストールのトラブルシューティングに役立ちます。

<Callout variant="important">
  Herokuを使用している場合は、 [Ruby agent and Heroku](/docs/agents/ruby-agent/miscellaneous/ruby-agent-heroku#logging) を参照してください。
</Callout>

## ログファイルの生成 [#generating]

詳細なNew Relic for Rubyのログファイルを生成するため。

1. `newrelic.yml`ファイルをエディタで開きます。

2. [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level)を`debug`に変更し、ファイルを保存します。インデントを変更しないように注意してください。

   <Callout variant="important">
     `debug` 非常に詳細なログ レベルです。デバッグレベルのログを生成した後は、必ず前の`log_level`に戻ってください。
   </Callout>

3. Webアプリケーションを再起動します。

4. あなたのアプリにトラフィックを発生させ、5分ほど実行します。

5. ログ ファイルにエラーがないか調べてください。デフォルトでは、ログ ファイルはアプリのログ フォルダーにあり、名前は`newrelic_agent.log`です。

6. ログ ファイルを New Relic サポートに送信する場合は、ログ ファイルを、 `newrelic.yml` 、Gemfile、および`Gemfile.lock`とともにサポート チケットに添付してください。

7. `newrelic.yml`を再度編集し、 [`log_level`](/docs/agents/ruby-agent/installation-configuration/ruby-agent-configuration#log_level)を前の設定に変更します。ファイルを保存します。
