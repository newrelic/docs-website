New Relic Rubyエージェントは、設定ファイルや環境変数の設定を使用して、またはサーバーサイド構成を使用してプログラム的に構成できます。このドキュメントは、Rubyエージェントで使用できる設定オプションをまとめたものです。

設定オプションのデフォルト値が`(Dynamic)`の場合、Rubyエージェントが実行時にデフォルト値を計算します。設定項目の値は、状況に応じて別の設定項目の値がデフォルト値として設定されます。

## 設定方法と優先順位 [#Options]

Rubyエージェントを設定する主な（デフォルトの）方法は、 `config`サブディレクトリ内の設定ファイル（`newrelic.yml`）を使用することです。環境変数を使用して設定値を設定する方法：

1. 設定名に接頭辞`NEW_RELIC_`を追加します。
2. ピリオド`.`をアンダースコア`_`に置き換えます。

[サーバーサイド設定](/docs/agents/manage-apm-agents/configuration/server-side-agent-configuration)からUIでいくつかの値を設定することもできます。

Rubyエージェントは、設定に対して次の優先順位に従います。

1. 環境変数
2. サーバーサイドコンフィグレーション
3. 設定ファイル（`newrelic.yml`）
4. デフォルト設定

つまり、環境変数は他のすべての設定と情報をオーバーライドし、サーバーサイド設定は設定ファイルとデフォルト設定をオーバーライドします。

## 設定ファイルのオプションの表示・編 [#Edit]

Rubyエージェントの`newrelic.yml`は標準のYAML設定ファイルです。通常、上部に`Defaults`セクションがあり、その下に各アプリケーション環境（`Development`、`Test`、`Staging`、`Production`）のセクションがあります。

Rubyエージェントは、特定の環境変数を調べてアプリケーションの環境を認識することにより、`newrelic.yml`設定ファイルの読み取るセクションを決定します。これは、本番環境では`info`を`log_level`設定に使用し、開発環境ではより詳細な`log_level`設定（例えば`debug`）を使用したい場合に役立ちます。

以下は`newrelic.yml`設定ファイルの例です。

```yaml
common: &default_settings
  license_key: 'YOUR_LICENSE_KEY'
  app_name: 'My Application Name'
production:
  <<: *default_settings
  log_level: info
development:
  <<: *default_settings
  log_level: debug
```

Rubyエージェントは、次の環境変数をこの順序で検索して、アプリケーション環境を見つけます。

1. `NEW_RELIC_ENV`
2. `RUBY_ENV`
3. `RAILS_ENV`
4. `APP_ENV`
5. `RACK_ENV`

Rubyエージェントがこれらの環境変数の値を検出できない場合は、アプリケーション環境をデフォルトで`development`に設定し、`newrelic.yml`構成ファイルの`development`セクションから読み取ります。

RailsアプリケーションでRubyエージェントを実行すると、エージェントはまず`NEW_RELIC_ENV`環境変数を探して、アプリケーション環境と、`newrelic.yml`内の使用するセクションを確認します。`NEW_RELIC_ENV`がない場合、エージェントはRails環境（`RAILS_ENV`）を使用します。

設定ファイルを編集する際は、以下の点に注意してください。

* インデントはスペース2つだけにしてください。
* 適切な箇所、例えば**`error_collector`**などのセクションでのみインデントを行ってください。

インデントが正しくない場合、エージェントの起動時に`Unable to parse configuration file`エラーが発生する可能性があります。

利用可能なRubyエージェント設定オプションの最新のリストを表示するには、`rake newrelic:config:docs`コマンドを使用します。このドキュメントでは、最も一般的なオプションについて説明します。

## 設定ファイルの更新 [#Updates]

このドキュメントの内容は、Rubyエージェントの最新リリース向けです。以前のバージョンの詳細については、`newrelic.yml`内のコメントを参照してください。

新しいリリース後に`newrelic.yml`ファイルを更新するには、エージェントgemのベースディレクトリにあるテンプレートを使用します。新しいgemバージョンに更新するときは、[インストールディレクトリ](/docs/agents/manage-apm-agents/troubleshooting/find-agent-root-directory#ruby-agent)内の`config/newrelic.yml`と`newrelic.yml`を確認・比較して、新しい設定オプションを利用してください。

<Callout variant="important">
  gemを更新しても、`config/newrelic.yml`は自動的に更新されません。
</Callout>

    モジュール名やクラス名を含むクラス名には、完全修飾クラス名（区切り文字`::`を使用）を使用してください。

    ```rb
    	module MyCompany
    		class Image
    			def render_png
    				# code to render a PNG
    			end
    		end

    		class User
    			def self.notify
    				# code to notify users
    			end
    		end
    	end
    ```

    ```yaml
    	automatic_custom_instrumentation_method_list:

    		- MyCompany::User.notify
    ```

    ```yaml
    	automatic_custom_instrumentation_method_list: 'MyCompany::Image#render_png, MyCompany::User.notify'
    ```

    ```sh
    	export NEW_RELIC_AUTOMATIC_CUSTOM_INSTRUMENTATION_METHOD_LIST='MyCompany::Image#render_png, MyCompany::User.notify'
    ```
