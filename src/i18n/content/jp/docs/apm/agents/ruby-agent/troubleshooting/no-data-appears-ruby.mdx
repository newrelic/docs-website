## 問題

New Relic Ruby エージェント [をインストールし、トラフィックを発生させて 5 分間待っても、New Relic の UI にデータが表示されません。](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)

## [解決](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)

[欠損したデータのトラブルシューティングに](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)

1. [](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)

2. [設定ファイルを確認してください。](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)

   * [* 構成ファイルがアプリケーション ディレクトリ内の`config/newrelic.yml`にインストールされていることを確認してください。* 設定ファイルが有効なYMLであることを確認してください。](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)

   * <LicenseKey>[](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)</LicenseKey>

     [ が事前入力されたこのファイルのクリーン コピーをダウンロードするには、 ****](/docs/agents/ruby-agent/installation-configuration/ruby-agent-installation)**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > (ユーザー メニュー) > アカウント設定 > クリーンな構成ファイルのダウンロード**に移動します。

3. Webサーバーを再起動します。

4. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics)を使用すると、問題を自動的に特定できる場合があります。

5. `log/newrelic_agent.log`にあるエージェントのログ ファイルを調べて、アプリケーション サーバー プロセスの再起動時に表示されるエラーまたは警告を確認してください。このログ ファイルが存在しないか、書き込まれていない場合は、「[ログ ファイルがありません](/docs/agents/ruby-agent/troubleshooting/no-log-file-ruby)」を参照してください。

6. New Relic エージェントのログで、 `INFO : Reporting to:`の後に URL が続くエントリがないか確認してください。URL が、New Relic UI で表示しているアカウントとアプリケーションと一致していることを確認してください。

7. デプロイ先の環境で New Relic エージェントが起動するように設定されていることを確認します。詳しくは、 [Controlling when the Ruby agent starts](/docs/agents/ruby-agent/troubleshooting/controlling-when-ruby-agent-starts) をご覧ください。

8. `info`レベルで明らかなエラーがない場合は、[詳細なデバッグ ログを生成し](/docs/agents/ruby-agent/troubleshooting/generating-logs-troubleshooting-ruby)ます。

9. [**newrelic_rpm** gemの最新バージョン](/docs/agents/ruby-agent/installation-configuration/updating-deprecated-ruby-agents#update) にアップデートしてみてください。

10. Rails 以外のものを使用している場合は、 `RACK_ENV`を、開発や運用など、 `newrelic.yml`ファイルに表示される環境に設定します。
