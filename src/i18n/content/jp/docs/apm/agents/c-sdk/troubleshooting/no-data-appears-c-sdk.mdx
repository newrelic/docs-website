## 問題

[New Relic C SDK](/docs/install-c-agent-compile-link-your-code) をインストールし、アプリのトラフィックを生成して数分待ちましたが、APM でデータのレポートが表示されませんでした。

## 解決

これらのトラブルシューティングのヒントは、最も一般的な問題から最も一般的でない問題の順に並んでいます。

1. [C SDKとデーモンのログファイル](/docs/generate-logs-troubleshooting-c-sdk) の出力にエラーがないか確認してください。

2. [デーモンプロセス（ `newrelic-daemon` ） ](/docs/install-c-sdk-compile-link-your-code#daemon)が稼働していることを確認して、インストルメント化されたアプリケーションを開始する前に **呼び出されるようにし** 。

3. アプリの名前を確認し、 <InlinePopover type="licenseKey"/>[C SDK 構成](/docs/c-sdk-configuration)で .

4. 必ず電話してください。

   * [`newrelic_create_app()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a61dd90439ae3cc5060021f6ab4701132)
   * 必要に応じて[`newrelic_start_web_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#adbf7c1fa57482f6e0a7f291e0b5ec80f)または[`newrelic_start_non_web_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a88648cc287f8d7e371139dc3809b7afb)
   * [`newrelic_end_transaction()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#acbd88933d620c21b7b19f12b6f042da8)

   また、これらの呼び出しのいずれも`false`または`NULL`を返さないことを確認してください。

5. 必要に応じて、 [`newrelic_version()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a8aec87e54af38e466fc5f715ef591d10)を使用してC SDKのバージョンを確認し、 [CSDKライブラリを最新リリースに更新します](/docs/update-your-c-sdk-library)。
