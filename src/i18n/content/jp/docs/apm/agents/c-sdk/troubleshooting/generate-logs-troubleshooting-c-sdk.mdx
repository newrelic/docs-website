## 問題

C SDKでインスツルメンテーションされたアプリケーションで、New Relicにデータを送信する際に問題が発生したが、原因がわからない。

## 解決

New Relic C SDK とそのデーモンには、独自のログがあります。

* <DNT>
    **C SDK logs:**
  </DNT>

  これらのログは、 New Relic C SDK APIコールを使用してコードをインストゥルメントした方法のエラーにより生成されます。

* <DNT>
    **Daemon logs:**
  </DNT>

  これらは、New Relic へのデータの送信に関連するログです。

トラブルシューティングのためには、最高レベルのログを生成することをお勧めします。

1. [C SDKログ](#c-sdk-logs)を`verbose`レベルに設定し、[デーモンログ](#daemon-logs)を`debug`レベルに設定します。
2. 両方のログを5～10分程度実行します。
3. トラブルシューティングのためにログを生成した後は、C SDKとデーモンのログをデフォルトレベルに戻してください。

<Callout variant="caution">
  ログレベルを上げると、大きなログファイルが作成され、パフォーマンスに影響を与える可能性があります。トラブルシューティングに必要な範囲でのみ、このレベルのログを生成してください。
</Callout>

<CollapserGroup>
  <Collapser
    id="c-sdk-logs"
    title="C SDKログ"
  >
    C SDKには、 `libnewrelic.h`の`enum _newrelic_loglevel_t`で定義されている4つのログレベルがあります。デフォルトでは、ログは`NEWRELIC_LOG_INFO`に設定され、標準エラーに出力されます。ログレベルは次のとおりです。

    ```
    NEWRELIC_LOG_ERROR
    NEWRELIC_LOG_WARNING
    NEWRELIC_LOG_INFO (default)
    NEWRELIC_LOG_DEBUG
    ```

    トラブルシューティングのために別のログレベルを設定するには：アプリケーションコードで[`newrelic_configure_log()`](https://newrelic.github.io/c-sdk/libnewrelic_8h.html#a8922f48a2b92714fb2fc05ad7bd5c003)を呼び出し、ログレベルを設定します。例えば：

    ```c
    newrelic_configure_log("./c_sdk.log", NEWRELIC_LOG_INFO);
    ```
  </Collapser>

  <Collapser
    id="daemon-logs"
    title="デーモンのログ"
  >
    CSDKデーモンには4つのログレベルがあります。デフォルトでは、ログは`info`に設定され、標準エラーに出力されます。ログレベルは次のとおりです。

    ```
    error
    warning
    info (default)
    debug
    ```

    これらのログレベルは、コマンドラインからフラグを使って呼び出されます。

    ```sh
    --logfile <DAEMON_FILE_NAME>  # Sets the path to the log file.
    --loglevel <LOG_LEVEL>        # Sets the log level. Default: info.
    ```

    トラブルシューティングのために異なるログレベルを設定するには、以下の手順に従います。コマンドラインで、異なるログレベルのフラグを設定します。例えば、以下のようになります。

    ```sh
    ./newrelic-daemon -f -logfile stdout -loglevel debug
    ```
  </Collapser>
</CollapserGroup>
