アプリケーションのコードライブラリにNew Relic C SDKの最新版を使用するために、 [リリースノート](/docs/release-notes/agent-release-notes/c-release-notes) を確認してください。

## C SDKコードライブラリの更新 [#update-production]

アプリケーションのコードライブラリをNew RelicのC SDKの最新版に更新すること。

1. C SDKのライブラリの変更点を確認し、コードの互換性を確認してください。
2. 必要に応じて、アプリのコードを手動で更新してください。
3. [`make`](https://github.com/newrelic/c-sdk#building) コマンドを使用して C SDK コードをコンパイルし、最新バージョンの `libnewrelic.a` を作成して、アプリケーションまたはサービスにリンクします。
4. 必要に応じて、アプリケーションをテスト環境に再デプロイします。
5. アプリケーションのトラフィックを生成し、その `c_sdk.log` ファイルと `newrelic-daemon.log` 出力にエラーがないか確認します。
6. アプリケーションを本番環境に再配置します。
7. アプリケーションが New Relic にデータを送信するまで数分待ちます。その後、アプリケーションのパフォーマンスを [New Relic](/docs/apm/applications-menu/monitoring/apm-overview-page) で確認します。

数分経ってもデータが表示されない場合は、 [トラブルシューティングのヒント](/docs/apm/agents/c-sdk/troubleshooting/no-data-appears-c-sdk) に従ってください。
