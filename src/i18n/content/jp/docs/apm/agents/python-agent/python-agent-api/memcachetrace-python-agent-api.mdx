## 構文

```py
newrelic.agent.memcache_trace(command, async_wrapper=None)
```

デフォルトではインストゥルメントされていない関数に対して、memcacheコマンド関数を計装するために使用されます。

## 説明

`memcache_trace` memcacheコマンド関数に追加するためのデコレータです。このデコレータを追加すると、追加の[トランザクション](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction)情報（トランザクショントレース情報を含む）を収集できるようになります。

`memcache_trace`デコレータは、エージェントバージョン2.102.0.85以上で関数に使用できます。これらのオブジェクトのタイミングは、消費が開始されたときに始まり、オブジェクトが使い果たされるか、スコープ外になったときに終了します。

デコレータは、静的メソッドやクラス メソッドのデコレータを含む既存のデコレータと組み合わせて使用できます。複数のデコレータをネストする場合は、 `memcache_trace` デコレータを最も外側のデコレータとして使用します。

## 代替コールフォーム [#alternate]

デコレーターを使用できない設定の場合は、以下の代替コールフォームを利用できます。

### ラッパー

トレースする特定の関数がどこにあるか事前にわかっている場合は、 `memcache_trace`デコレータを使用できます。 ただし、トレースする必要があるすべての関数がわかっ<DNT>**don&apos;t**</DNT>いる場合 (たとえば、ルーティング システムの一部として動的に検索されている場合)、登録時または呼び出し時に`MemcacheTraceWrapper`を使用して関数をラップする必要があります。

### パスベースのラッピング [#path-based]

`wrap_memcache_trace` は、関数が宣言されているコードの外で関数をラップするために使用されます。たとえば、これを使用して、変更したくないライブラリ コードをインスツルメントすることができます。

これらの通話フォーマットの違いについては、 [Different call formats](/docs/python-agent-api-different-call-forms) をご覧ください。

## 例

### `memcache_trace` [#trace\_example][#trace_example]

`memcache_trace`デコレーターの使用例:

```py
import newrelic.agent

@newrelic.agent.memcache_trace("get")
def get(id):
    ...
```

### ラッパーの例 [#trace-wrap-example]

`MemcacheTraceWrapper`の使用例:

```py
import newrelic.agent

def get(id):
    ...

get = newrelic.agent.MemcacheTraceWrapper(get, command="get")
```

コマンドの値を動的に返したい場合は、`MemcacheTraceWrapper`オブジェクトの作成時にコマンドを関数として指定できます。

```py
import newrelic.agent

def get(id):
    ...

get = newrelic.agent.MemcacheTraceWrapper(get, command=lambda id: f"get:{id}")
```
