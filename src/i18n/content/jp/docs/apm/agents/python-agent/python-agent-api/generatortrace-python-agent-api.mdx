## 構文

```py
newrelic.agent.generator_trace(name=None, group=None, label=None, params=None, terminal=False)
```

デフォルトでインストゥルメントされていないジェネレーターの計装に使用されます。

## 説明

`generator_trace` は、ジェネレータ関数に追加するためのデコレータです。このデコレータを追加すると、追加の[トランザクション](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction)情報（トランザクショントレース情報を含む）を収集できるようになります。

エージェントバージョン2.102.0.85以降では、`generator_trace`デコレータをジェネレータおよびコルーチンで使用できます。これらのオブジェクトのタイミングは、消費が開始されたときに始まり、オブジェクトが枯渇するかスコープ外になったときに終了します。

デコレータは、静的メソッドやクラス メソッドのデコレータを含む既存のデコレータと組み合わせて使用できます。複数のデコレータをネストする場合は、 `generator_trace` デコレータを最も外側のデコレータとして使用します。

## 代替コールフォーム [#alternate]

デコレーターを使用できない設定の場合は、以下の代替コールフォームを利用できます。

### ラッパー

トレースしたい特定のジェネレーターがどこにあるか事前にわかっている場合は、`generator_trace`デコレーターを使用できます。ただし、トレースする必要があるすべてのジェネレーターを<DNT>**don&apos;t**</DNT>把握している場合（たとえば、ルーティングシステムの一部として動的に検索されている場合など）、登録時または呼び出し時に`GeneratorTraceWrapper`を使用してジェネレーターをラップする必要があります。

### パスベースのラッピング [#path-based]

`wrap_generator_trace` 宣言されているコードの外部でジェネレーターをラップするために使用されます。例えば：変更したくないライブラリコードを計装するためにこれを使用する場合があります。

これらの通話フォーマットの違いについては、 [Different call formats](/docs/python-agent-api-different-call-forms) をご覧ください。

## 例

### `generator_trace` [#trace\_example][#trace_example]

`generator_trace`デコレーターの使用例:

```py
import newrelic.agent

@newrelic.agent.generator_trace()
def gen():
    for i in range(5):
        yield i
```

非同期ジェネレーターで`generator_trace`デコレーターを使用する例：

```py
import newrelic.agent

@newrelic.agent.generator_trace()
async def gen():
    for i in range(5):
        yield i
```

### ラッパーの例 [#trace-wrap-example]

`GeneratorTraceWrapper`の使用例:

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen)
```

エンドポイント名にちなんでメトリクスに名前を付けたい場合（呼び出されるジェネレーターの識別子に基づいてメトリクスに名前を付けるのではなく）、`GeneratorTraceWrapper`オブジェクトの作成時に、使用する名前と代替のメトリクスパスプレフィックスを指定できます。

```py
import newrelic.agent

def gen():
    for i in range(5):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name="gen_5", group="Python/Generator")
```

メトリクスに動的に名前を付けたい場合は（呼び出されるジェネレーターの識別子に基づいてメトリクスに名前を付けるのではなく）、`GeneratorTraceWrapper`オブジェクトの作成時にジェネレーターとして名前を指定できます。

```py
import newrelic.agent

def gen(x):
    for i in range(x):
        yield i

gen = newrelic.agent.GeneratorTraceWrapper(gen, name = lambda x: f"gen_{x}")
```
