## 問題

Python エージェントをインストールし、トラフィックを生成して 5 分間待っても、データが New Relic UI に表示されません。

## 解決

ここでは、データ欠損のトラブルシューティングのヒントを、問題発生の可能性が高い順にご紹介します。

1. アプリケーションと同じ環境にエージェントをインストールしたことを確認してください。仮想環境を使用している場合は、そこにインストールしてください。

2. エージェントが最新であることを確認してください。

   <DNT>
     **pip**
   </DNT>

   または

   <DNT>
     **easy_install**
   </DNT>

   を使用するのではなく、New Relic サイトからダウンロードした場合は、バージョン番号を[現在のリリース](https://download.newrelic.com/python_agent/release/)と比較してください。

3. [New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics)を使用すると、問題を自動的に特定できる場合があります。

4. [Python agent test](/docs/python/testing-the-python-agent) を実行して、データが New Relic の UI にアクセスできることを確認します。

5. [互換性と要件](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent) を満たしていることを確認してください。 [ウェブフレームワークやウェブサーバー](/docs/agents/python-agent/hosting-mechanisms) に特別な要件がないか確認してください。

6. `newrelic-admin generate-config`コマンドで生成された設定ファイルを確認して、正しい

   <InlinePopover type="licenseKey"/>

   が含まれていることを確認します。

7. [トラブルシューティングのための詳細なログを生成する](/docs/agents/python-agent/troubleshooting/generating-logs-troubleshooting) 。そして、ログファイルにエラーメッセージがないか確認します。ログファイルが見つからない場合は、その場所への書き込み権限があることを確認してください。

また、データが表示されず、アプリのアクティベーションや再アクティベーションに関する警告メッセージが表示される場合は、 [アクティベーションのトラブルシューティング手順](/docs/agents/python-agent/troubleshooting/activate-application-warning-python) 。
