PythonエージェントはPythonアプリケーションをモニターし、[パフォーマンスの問題](#monitor-performance)の特定と解決を支援します。パフォーマンスの監視を拡張して[ビジネスデータ](#business-data)を収集して分析し、顧客体験の向上およびビジネスにおけるデータ駆動型の意思決定を支援することもできます。[カスタムインストゥルメンテーションとAPI](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation)の柔軟なオプションにより、Pythonエージェントには、アプリケーションに必要なデータをカスタマイズする多くの構築ブロックがあります。

Pythonは、Django、Gunicorn、WSGI、CherryPy、uWSGIなどのさまざまな[ウェブフレ－ムワークとホスティングメカニズム](/docs/agents/python-agent/getting-started/instrumented-python-packages)で動作します。Pythonエージェントは、[Google App Engineのフレキシブル環境](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment)にインストールすることもできます。

## アプリのパフォーマンスの監視 [#monitor-performance]

Pythonエージェントを[インストール](#installation)後に、アプリに関するデータの収集を開始します。[New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)では、データはチャートとテーブルとして表示されます。

**アプリの全体像を表示する：**

* アプリの[Apdex（ユーザー満足度）](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction)を監視します。
* [概要ページ](/docs/apm/applications-menu/monitoring/apm-overview-page)でアプリケーションの概要を入手する。
* [ディストリビューティッド（分散）トレーシング](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)を有効にして、多くのサービスを含むアーキテクチャー全体の活動を表示します。
* [Infrastructureモニタリング](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure)をインストールして、アプリケーションの詳細なサーバー/ホストデータを表示します。

**エラーと問題を素早く検知する：**

* ビジネス特定の[キートランザクション](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events)を追跡します。
* 重要なメトリクスに対して[カスタムダッシュボード](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards)を作成します。
* エラーや問題が発生したら、ユーザーに影響が及ぶ前にチームに[アラート](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic)を発信します。
* [デプロイメント後](/docs/apm/applications-menu/events/deployments-page)のパフォーマンスを表示します。

**パフォーマンスの詳細にドリルダウンする：**

* コードレベルの[トランザクショントレース](/docs/apm/transactions/transaction-traces/transaction-traces)を調査します。
* [データベースクエリトレース](/apm/transactions/transaction-traces/troubleshoot-sql-statements)を調査します。
* [エラートレース](/docs/apm/applications-menu/events/view-apm-error-analytics)を調査します。
* [スレッドプロファイラーセッション](/docs/apm/applications-menu/events/thread-profiler-tool)を使用して、サンプリングしたスレッドの詳細なスタックトレースを確認します。

**APMおよびインフラストラクチャデータのログを表示：**

ログとアプリケーションのデータを統合して、トラブルシューティングをより簡単かつ迅速にできます。New Relic Oneで別のUIページに切り替える必要はありません。

* [コンテキストのログ](/docs/logs/logs-context/configure-logs-context-python/)では、エラーとトレースに関連するログメッセージをアプリケーションのUIで直接表示できます。
* また、Kubernetesクラスタなど、[インフラストラクチャデータ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)のコンテキストでログを表示することもできます。

**エージェントインストゥルメンテーションを拡張する：**

その他の役立つツールとしては、次のものがあります。

## Pythonエージェントをインストール。 [#installation]

Pythonエージェントをインストールする前に、お使いのシステムが[システム要件](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent)を満たしていることを確認します。[New Relicアカウント](https://newrelic.com/signup)を作成する必要もあります。

当社は、Django、WSGI、Gunicornなど、多くのウェブフレームワークとライブラリを標準でサポートしています。[サポートされているウェブフレームワーク](/docs/agents/python-agent/getting-started/instrumented-python-packages)を使用すると、インストレーションは簡単です。サポートされていないフレームワークを使用する場合は、アプリのコードやウェブサーバーファイルに多少の追加が必要です。

ほとんどのセットアップで使用できる簡単迅速なインストールプロセスについては、次の簡単な手順に従ってください。

1. Pythonパッケージをダウンロードしてインストールします。
2. 設定ファイルを作成します。
3. Pythonエージェントとアプリケーションを統合します。

## 非ウェブスクリプト、バックグラウンドタスク、および関数の監視 [#non-web]

Pythonエージェントでは、非ウェブスクリプト、ワーカープロセス、タスク、および関数もモニターできます。これらの非ウェブトランザクションのインストレーションプロセスはウェブアプリに使うプロセスに似ているものの、1つ異なる点は、[インストール手順](/docs/agents/python-agent/getting-started/python-agent-quick-start)に説明されている標準的なインテグレーションプロセスを通してではなく、監視したい関数を手作業で「ラップ」することです。詳細については、[非ウェブタスクとプロセス](/docs/agents/python-agent/supported-features/python-background-tasks)を参照してください。Celeryタスクを監視する手順については、[Celeryバックグラウンドタスク](/docs/agents/python-agent/back-end-services/python-agent-celery)を参照してください。

## インストレーション後の手順 [#instrumentation]

エージェントを起動した後に推奨する手順は次のとおりです。

* データを検索し、ユーザーインタフェースに慣れます。
* [他のオブザーバビリティソリューション](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability)と[APMページのドキュメント](/docs/apm)をお読みください。
* [アプリケーション名](/docs/agents/manage-apm-agents/app-naming/rename-your-application)、またはその他の[設定オプション](/docs/agents/python-agent/installation-configuration/python-agent-configuration)を変更します。
* デフォルトで監視されないアプリケーションのアクティビティに対する[カスタムインストゥルメンテーション](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation)の設定方法を学びます。
* [Python Telemetry SDK](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic)をご検討ください。

## トラブルシューティング

インストールプロセスが完了したら、[APM UI](/docs/apm/applications-menu/monitoring/apm-overview-page)にデータが5分以内に表示されます。表示されない場合は、以下のトラブルシューティングリソースを使用します。

* データが表示されない場合は、[トラブルシューティング手順](/docs/agents/python-agent/troubleshooting/no-data-appears-python)に従います。
* 新しいホストでPythonエージェントをインストールまたは実行する際に問題が発生する場合は、[パッケージが正しくインストールされていること、](/docs/agents/python-agent/installation-configuration/testing-python-agent)および New Relicのデータコレクターサービスに接続できることを確認します。
* その他の問題については、[トラブルシューティングドキュメントの一覧](/docs/agents/python-agent/troubleshooting/)をご覧ください。

## ソースコードのチェック [#source-code]

Pythonエージェントは、オープンソースソフトウェアです。つまり、[ソースコードを参照](https://github.com/newrelic/newrelic-python-agent "新しいウィンドウにリンクが開きます。")して改善を送信、または独自のフォークを作成して構築できます。詳細については、[README ](https://github.com/newrelic/newrelic-python-agent/blob/main/README.rst "新しいウィンドウにリンクが開きます。")を参照してください。
