Pythonエージェントをインストールする前に、システムがこれらの要件を満たしていることを確認します。

## 基本要件 [#basic]

Python エージェントは、最も一般的な Python Web フレームワークの多くをサポートしています特定のホスティング メカニズム、ホスティング サービス、Web フレームワーク、またはバックエンド サービスによっては、追加の構成が必要になる場合があります。サポートの詳細については、以下の表を参照してください。

New Relicアカウントをまだお持ちでない場合は、[New Relicアカウントを新たに作成します](https://newrelic.com/signup)。永久無料です。

## Pythonパッケージのインストゥルメンテーション [#python-packages]

エージェントは、その他のさまざまなPython[パッケージとモジュール](/docs/agents/python-agent/getting-started/instrumented-python-packages)にもインストゥルメンテーションを提供します。インストゥルメンテーションには、データベースクエリ、メモリキャッシュのリクエスト、外部サービスのリクエスト、[インスタンスレベルの詳細](/docs/apm/applications-menu/features/analyze-database-instance-level-performance-issues)などが含まれます。その結果は、遅いトランザクションに対して、APMのユーザーインタフェースのウェブトランザクションパフォーマンスの内訳およびトランザクショントレースに自動的に表示されます。

## インストゥルメント化パッケージバージョンのサポート [#package-version]

Pythonエージェントは、3年間のリリースウィンドウ内でインストゥルメントされたパッケージを積極的にサポートし、テストします。

## エージェントバージョンのサポート [#agent-version]

New Relicでは、エージェントを定期的に、少なくとも3か月ごとにアップグレードすることを推奨します。エージェントバージョンは、1年間のリリースウィンドウでサポートされます。

## Pythonバージョンのサポート [#python-version]

エージェントは一般的に、すべてのリリース済みのアクティブな[Pythonブランチ](https://devguide.python.org/versions/#versions)をサポートします。ただし、今後の変更に対応するために、エージェントはこのPythonバージョンサポートスケジュールに従うことにもなります。バージョンサポートポリシーは、[一般的な有効期限（EOL）ポリシー](/docs/agents/manage-apm-agents/maintenance/new-relic-agent-plugin-end-life-policy)を置き換えません。
