APMのスケーラビリティ分析レポートは、アプリの負荷と比較したレスポンスタイム、データベース使用率、CPU使用率を散布図で表示します。これにより、アプリケーションの成長に伴うパフォーマンスや潜在的なハードウェアの必要性を分析することができます。

## スケーラビリティ分析レポートを見る [#procedures]

自分のアプリのレポートを表示するには

1. Go to **[one.newrelic.com](https://one.newrelic.com) > APM> （アプリを選択）> Reports> Scalability**.
2. **スケーラビリティ分析** ページから、 **応答時間** 、 **データベース** 、 **CPU** の散布図を表示するタブを選択します。
3. 個別のチャートタブから、 [時間帯](#time) を選択します。
4. **Filter outliers** のチェックボックスを選択またはクリアします。
5. New Relic の標準的な [ページ機能](/docs/accounts-partnerships/education/getting-started-new-relic/standard-dashboard-features) を使って、詳細な情報にドリルダウンすることができます。

## 時間帯の選択 [#time]

各スケーラビリティ分析レポートには、過去24時間、過去7日間、その他の期間など、異なる期間のデータを表示するオプションがあります。たとえば、 **Use time window selection** を選択し、 [time picker](/docs/site/timepicker-setting-time-periods-to-view-data) がデフォルト値（Last 30 minutes, ending now）を使用している場合、チャートには過去30分のデータのみが表示されます。

## データを分析 [#insights]

APMには、ユーザーインターフェースにいくつかのレポートが含まれています。ソフトウェアに関するデータを他のフォーマットで収集、分析、可視化するには、 [クエリビルダー](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder) をご利用ください。
