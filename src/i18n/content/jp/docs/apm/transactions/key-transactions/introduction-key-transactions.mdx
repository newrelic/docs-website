APMで、カスタムレベルモニタリングが求められる、または独自の**アラートポリシー**が求められる場合、トランザクションを[キートランザクション](/docs/apm/transactions/key-transactions/view-key-transactions-alert-information)にできます。ビジネスにとって重要な[ウェブトランザクション](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#transaction)あるいは[非ウェブトランザクション](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#non-web-transaction)などが該当します。

大半のNew Relicエージェントで、あらゆる種類のトランザクションをキートランザクションにできます。PHP、Node.js、Goエージェントでキートランザクションに変更できるのは、**ウェブトランザクション**のみとなります。

## キートランザクションとは [#overview]

APMで[トランザクション](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm)を**キートランザクション**に設定すると、次のメリットがあります。

* キートランザクションはNew Relic UI内でグループ化されているため、検索や調査が容易です。
* キートランザクションには、[カスタムApdex（Application Performance Index）閾値](/docs/apm/transactions/key-transactions/create-key-transactions#apdex)を設定できます。
* キートランザクションには、[アラートポリシー](/docs/apm/transactions/key-transactions/view-key-transactions-alert-information)を設定できます。

キートランザクションは次の場合に便利です。

* **重要なトランザクション。**アプリケーションで最も重要なトランザクションをキートランザクションに設定することで、 迅速な検索や調査が可能になります。より厳密なApdex要件の設定や、アラートポリシーの設定が可能です。
* **遅いトランザクション。**他のトランザクションと比べて常に遅いトランザクションは、アプリケーション全体のApdexスコアを下げる可能性があります。トランザクションをキートランザクションに設定することで、より長期のApdex閾値を設定し、アプリケーションのApdexスコアが下がるのを防止できます。

## キートランザクションの作成 [#create]

キートランザクションを作成するには、[キートランザクションの作成および更新](/docs/apm/transactions/key-transactions/create-update-key-transactions)をご覧ください。

## UIでキートランザクションを表示する [#find]

**キートランザクションインデックス**は、キートランザクションのリストです。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > All capabilities > Key transactions**に移動します。
2. インデックスから **Key transaction**を選択します。
3. 選択したキートランザクションの、チャートやドリルダウンオプションを含む[**Summary**ページ](/docs/apm/transactions/key-transactions/viewing-key-transactions)を表示します。
