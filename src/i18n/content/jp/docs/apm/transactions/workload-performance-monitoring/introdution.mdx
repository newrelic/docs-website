急速に進化する今日のテクノロジー環境では、最新のアプリケーションはますます複雑になり、平均解決時間 (MTTR) が大幅に増加しています。 この複雑さは、データベース、サーバー レス機能、ブラウザー、モバイル アプリケーション、メッセージキュー、インフラストラクチャ、AIモデル、サービスなど、最新のアプリケーションを構成する多様なコンポーネントから生じます。 これらの個々のサービスを監視するだけではもはや十分ではありません。 アプリケーションの所有者は、アプリケーション エコシステム全体にわたる包括的な可視性を必要としています。

New Relic の Transaction 360 は、特定の問題や質問に関連するすべてのエンティティとデータを自動的にキャプチャして相関させることにより、アプリケーションの監視とトラブルシューティングに対する詳細なアプローチを提供します。

Transaction 360 の主な機能は次のとおりです。

* **自動検出**: 動的なサービス関係のマッピングとリアルタイムのトランザクション フローの視覚化。
* **包括的な可視性**: フルスタックモニタリング、クロスサービストランザクション追跡、およびブラウザ可視性へのインフラストラクチャ。
* **データベース監視**: インサイトをデータベースのパフォーマンス、書き込み分析、および現場ワークロード全体にわたる接続の状態に深く掘り下げます。
* **サーバーレス機能の追跡**: サーバーレス機能のパフォーマンス、コールド スタート、リソース使用率を完全に可視化します。
* **履歴分析**: 数週間分の履歴データにアクセスして、傾向分析とパターン認識を可能にします。

<Callout variant="important">
  トランザクション 360 を使用するには、ディストリビューティッド（分散）トレーシングを有効にする必要があります。 経験のほとんどは誰でも利用できます。ただし、トランザクション ワークロード ビューの高度なマップおよびトレース フロー マップ コンポーネントには高度な計算が必要です。
</Callout>

## はじめに

Transaction 360を開始するには、 New Relicアカウントでディストリビューティッド（分散）トレーシングを有効にする必要があります。 有効にすると、New Relic のトランザクション領域から Transaction 360 機能にアクセスできるようになります。
