各自のデータを送信するNew Relicエンドポイントを変更できます。デフォルトでは、New Relic OpenTelemetryエクスポーターは、データを米国の以下のデータセンターに送信します。

* スパン：[https://trace-api.newrelic.com/trace/v1](https://trace-api.newrelic.com/trace/v1)
* メトリクス：[https://metric-api.newrelic.com/metric/v1](https://metric-api.newrelic.com/metric/v1)

データをEU地域に送信するか、Infinite Tracingを使用する場合は、デフォルトのエンドポイントを上書きする必要があります。

### EU地域

EU地域にあるNew Relicのエンドポイントにテレメトリデータを送信する場合は、以下を使用します。

<Callout variant="tip">
  以下のURLは、Infinite Tracingには適用されません。
</Callout>

* スパン：[https://trace-api.eu.newrelic.com/trace/v1](https://trace-api.eu.newrelic.com/trace/v1)
* メトリクス：[https://metric-api.eu.newrelic.com/trace/v1](https://metric-api.eu.newrelic.com/trace/v1)

### 無限トレーシング

Infinite Tracingを設定している場合は、デフォルトのスパンエンドポイントを上書きして、テレメトリデータをNew Relicトレースオブザーバーに送信する必要があります。

1. [トレースオブザーバーの設定](/docs/understand-dependencies/distributed-tracing/infinite-tracing/set-trace-observer)の手順に従って、

   <var>YOUR_TRACE_OBSERVER_URL</var>

   の値を取得します。

2. <var>YOUR_TRACE_OBSERVER_URL</var>

   の値を使用して、インテグレーションを設定します。

3. New Relicにすべてのトレースを分析するように求める場合は、OpenTelemetryインテグレーションが`AlwaysOn`サンプラーを使用していることを確認します。
