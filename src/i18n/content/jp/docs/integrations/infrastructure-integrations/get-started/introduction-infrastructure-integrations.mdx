New Relicは、プラットフォームへのデータのレポート向けに、さまざまな[インテグレーション](https://newrelic.com/integrations)を提供しています。インテグレーションの1つのカテゴリが、Infrastructure インテグレーションです。

<Callout variant="tip">
  インテグレーションとInfrastructureモニタリング、およびその他の[オブザーバビリティプラットフォーム](https://one.newrelic.com)を使用するには、New Relicファミリーに参加してください。[サインアップ](https://newrelic.com/signup)して、わずか数秒で無料アカウントを作成できます。次に、毎月最大100GBのデータを無料で取り込みます。期間無制限です。
</Callout>

## Infrastructureインテグレーションのタイプ [#integration-types]

New Relicには、Infrastructureインテグレーションについて次の2つの主要なカテゴリがあります：クラウドとオンホスト。

### クラウドインテグレーション

クラウドインテグレーションは、クラウドサービスとアカウントからデータを収集します。クラウドインテグレーションのインストレーションプロセスは存在しません。New Relicアカウントをクラウドプロバイダーアカウントに接続するだけです。

## インストール手順 [#installation-instructions]

クラウドインテグレーションを有効にする、またはオンホストインテグレーションのインストールに関しては、以下を参照してください。

* **クラウドインテグレーション**：[AWSにおける手順](/docs/infrastructure/amazon-integrations/getting-started/connect-aws-services-infrastructure)、[Azureにおける手順](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations)、[Google Cloud Platformにおける手順](/docs/integrations/google-cloud-platform-integrations/getting-started/connect-google-cloud-platform-services-infrastructure)
* **Kubernetes**：[Kubernetesにおける手順](https://docs.newrelic.com/docs/integrations/kubernetes-integration/installation/kubernetes-installation-configuration)
* **オンホストインテグレーション**：インストール手順に関する[インテグレーションドキュメント](/docs/integrations/host-integrations)を参照する
* **SDKインテグレーション**：[カスタムインテグレーションを作成する手順](https://docs.newrelic.com/docs/integrations/integrations-sdk/getting-started/introduction-infrastructure-integrations-sdk#create)

## 機能 [#whats-next]

Infrastructureインテグレーションをアクティブ化すると、以下を実行できるようになります。

* [Infrastructure UI](/docs/infrastructure/integrations/find-use-infrastructure-integration-data)におけるメトリックスと設定データの[絞り込みと分析](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure)。
* [データのクエリ](/docs/using-new-relic/data/understand-data/query-new-relic-data)を行い、カスタムチャートとダッシュボードを作成する。
* [アラート条件を作成](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-condition)して、御社サービスのパフォーマンス上の問題を監視する。
* クラウドインテグレーションの場合、[データ収集の設定](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations)を設定する

## インテグレーションデータの種類 [#data-types]

Infrastructureインテグレーションは、[New Relicで使用](/docs/infrastructure/integrations/find-use-infrastructure-integration-data)できる、いくつかの基本的なデータタイプを生成します。

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        インテグレーションデータ
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr id="metrics">
      <td>
        メトリクス
      </td>

      <td>
        数値の測定データ。例：

        * キュー内のリクスト数

        * データベースでのヒット数（分単位）

        * CPU使用率

          クラウドベースおよびオンホストインテグレーションには、重要なメトリックスを表示する事前ビルドされたダッシュボードが含まれています。
      </td>
    </tr>

    <tr id="inventory">
      <td>
        インベントリ
      </td>

      <td>
        リアルタイムのシステムステータスと設定情報。例：

        * ホスト名

        * AWSリージョンまたはアベイラビリティゾーン

        * 使用中のポート

          インベントリを変更するとNew Relicでイベントが生成されるため、システムへの変更が原因でパフォーマンス上の問題がいつ起きたかを簡単に判断できます。
      </td>
    </tr>

    <tr id="events">
      <td>
        イベント
      </td>

      <td>
        システム上の重要なイベント。例：

        * 起動中のサービス

        * バージョンアップデート

        * 作成中の新規テーブル

          インベントリへの変更はイベントの一種になります。
      </td>
    </tr>

    <tr>
      <td>
        属性
      </td>

      <td>
        一部インテグレーションによって生成されるキー値ペア。例：

        * 一定のインベントリデータ

        * イベントに添付された追加データ

        * メトリックスもしくはインベントリとは見なされないデータ

          インテグレーションによっては、[その他のタイプの情報](/docs/telemetry-data-platform/understand-data/new-relic-data-types/)を属性として報告できます。
      </td>
    </tr>
  </tbody>
</table>

当社のインテグレーションはデータに依存しません。レポートされたデータに個人情報が含まれるかどうかは分かりません。New Relicのセキュリティ対策の詳細については、[セキュリティとプライバシーのドキュメント](/docs/using-new-relic/new-relic-security/security/security-matters-data-privacy-new-relic)を参照、または[New Relicのセキュリティウェブサイト](https://newrelic.com/why-new-relic/security)を閲覧してください。
