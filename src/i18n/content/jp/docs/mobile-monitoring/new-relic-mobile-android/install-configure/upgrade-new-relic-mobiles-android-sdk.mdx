最新版のAndroidエージェントをご利用いただくには、 [Androidエージェントのリリースノート](/docs/release-notes/mobile-release-notes/android-release-notes) をご参照ください。Androidアプリのアップグレード方法は、現在使用しているAndroidエージェントSDKのバージョンによって異なります。

## Android SDKバージョン2または3からのアップグレード [#android_upgrade_2]

Android SDKのバージョン2または3をインストール済みの場合。 **[one.newrelic.com](https://one.newrelic.com) > Mobile> (select an app)> Settings> アップグレードしてください。**

## Android SDKバージョン1へのアップグレード [#android_upgrade_1]

Android SDKのバージョン1をインストールしている場合は、バージョン2以上にアップグレードする前に、以下の手順を行ってください。

        1.
      </td>

      <td>
        **AndroidManifest.xmlファイルを開き、 `NewRelicApplication` クラスを削除します。**

        **nrandroid-setup** を最初に実行したときに、 `NewRelicApplication` クラスが `AndroidManifest.xml` ファイルに追加されました。このクラスは New Relic SDK から削除されており、今後は必要ありません。

        既存の `Application` サブクラスがあり、 `NewRelicApplication` を継承するように変更した場合は、代わりに `Application` を継承するようにサブクラスを更新してください。
      </td>
    </tr>

      <td>
        2.
      </td>

      <td>
        **`nrandroid-ant` の使用を停止します。**

        モバイルモニタリングでは、Androidアプリケーション用のカスタムAntラッパースクリプトが不要になりました。アップグレード後は、標準のビルドツールを使用するだけです。

        モバイルビルドの手順を呼び出すために既存のMaven構成を変更した場合は、その変更を削除して、改訂されたビルド手順を使用してください。
