## 問題

Android エージェント [をインストールして 5 分以上待っても、New Relic UI にデータが表示されません。](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/install-android-apps-gradle-android-studio)

## [解決](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/install-android-apps-gradle-android-studio)

[5分以上待ってもデータが表示されない場合は、 ](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/install-android-apps-gradle-android-studio)[New Relic Diagnostics](/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics) を使用して、一般的な問題を自動的に検出し、トラブルシューティングを提案します。それでも問題が解決しない場合は、以下をお試しください。

1. お使いのシステムが [の互換性と要件を満たしていることを確認してください](/docs/mobile-monitoring/new-relic-mobile-android/get-started/new-relic-android-compatibility-requirements) 。

2. `MainActivity` クラスの `onCreate()` の最初の行で Android エージェントを呼び出していること、およびメインスレッドでエージェントを実行していることを確認してください。Android エージェントは、別のクラスでのエージェントの起動をサポートしていません。

3. Android アプリが [64k multidex の制限を超えているかどうかを確認してください](/docs/mobile-monitoring/new-relic-mobile-android/troubleshoot/android-app-exceeds-64k-multidex-limit) 。

4. ログレベルを上げて、ログにエラーがないか調べてみてください。

   * `withLogLevel`を使用して New Relic [ログ レベル](/docs/mobile-monitoring/new-relic-mobile-android/api-guides/android-agent-configuration-feature-flags#log-level) を `AUDIT` に上げます。

     ```java
     NewRelic.withApplicationToken("YOUR_APP_TOKEN")
             .withLogLevel(AgentLog.AUDIT)
             .start(this.getApplication());
     ```

   * アプリで数分のアクティビティを発生させる。

   * デバイスのログとアプリケーションのビルドログを調べて、問題がないか確認してください。

5. デバイスがアクセスできることを確認します。 <InlinePopover type="mobile"/>エンドポイントは[ネットワークの](/docs/using-new-relic/cross-product-functions/install-configure/networks)ドキュメントにリストされています。

さらにサポートが必要な場合は、 [support.newrelic.com](https://support.newrelic.com) でサポートを受けてください。
