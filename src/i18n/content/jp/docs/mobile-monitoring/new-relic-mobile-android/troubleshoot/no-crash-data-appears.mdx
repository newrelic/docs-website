## 問題 [#problem]

アプリがクラッシュした後、New Relic UI にクラッシュ データが表示されません。 エージェントが`buildId`見つけられないことを示すログが表示される場合があります。これは、New Relic 側でクラッシュを処理するために必要なものです。

問題を確認するコンソールエージェントログメッセージは次のとおりです。

* `Agent.getBuildId() was unable to find a valid build Id. Crashes and handled exceptions will not be accepted.`
* `Invalid null or empty build ID detected! Crash will be ignored by collector.`
* `ExceptionHelper: java.util.UUID:fromStringJava8(UUID.java:337) RandomUUID[java.lang.IllegalArgumentException] Invalid UUID string:`
* `Unable to get obfuscated flag in crash`

## 解決 [#solution]

Android エージェント 7.xx 以降を使用しており、アプリがクラッシュした後にクラッシュ データが表示されない場合は、次の手順に従ってください。

1. 次の Gradle および AGP バージョンを使用していることを確認してください。

   * Gradle バージョン 7.5 以上
   * AGP (Android Gradle プラグイン) バージョン 7.4.2 以上

2. ProGuard ルールを構成する: 適切な ProGuard ルールが ProGuard 設定ファイルに追加されていることを再確認します。 ガイダンスについては、セットアップ ドキュメントの[「ProGuard ルールの構成」](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/configure-proguard-or-dexguard-android-apps/)セクションを参照してください。

追加のサポートが必要な場合は、 [support.newrelic.com](https://.newrelic.com)でサポートを受けてください。
