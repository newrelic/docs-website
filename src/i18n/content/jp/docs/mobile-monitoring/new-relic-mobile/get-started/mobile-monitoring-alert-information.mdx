明確に定義された<InlinePopover type="alerts"/> 、システムの変更について個人またはチームに通知するのに役立ちます。 [で監視するすべてのアプライアンスで 、アラートおよびアプライドインテリジェンス](/docs/alerts-applied-intelligence) New Relic機能を使用できます。

たとえば、[アラートを](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts)使用して、 New Relicでモニターするモバイル アプリのメトリックに重点を置いたアラートポリシーと条件を管理できます。 モバイル アプリの[アラート条件を定義し](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions)たら、アラート UI または<InlinePopover type="mobile"/> UI などの他の UI エクスペリエンスでその情報を表示できます。

## モバイルアラートの条件 [#conditions]

モバイルアプリにアラートを設定する際には、以下の条件としきい値のいずれかを使用します。

<CollapserGroup>
  <Collapser
    id="execution"
    title="実行時間"
  >
    実行時間のメトリックには、 <DoNotTranslate>**Database**</DoNotTranslate> 、 <DoNotTranslate>**Images**</DoNotTranslate> 、 <DoNotTranslate>**JSON**</DoNotTranslate> 、 <DoNotTranslate>**Network**</DoNotTranslate> 、 <DoNotTranslate>**View loading**</DoNotTranslate>が含まれます。 時間と期間を選択して、これらのメトリックの閾値を定義します。
  </Collapser>

  <Collapser
    id="errors"
    title="エラー"
  >
    エラー メトリックには<DoNotTranslate>**Network error percentage**</DoNotTranslate>と<DoNotTranslate>**Status error percentage**</DoNotTranslate>含まれます。 エラーのパーセンテージと期間を選択して、これらのメトリックの閾値を定義します。
  </Collapser>

  <Collapser
    id="crash"
    title="クラッシュレポート"
  >
    アプリの<DoNotTranslate>**crash rate**</DoNotTranslate>が特定の閾値を超えたときに通知を受け取ることができます。 エラーのパーセンテージと期間を選択して、これらのメトリックの閾値を定義します。

    <Callout variant="tip">
      PagerDutyやSlackなど、 [通知チャンネル](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts) を設定することで、アプリのクラッシュ率の超過について、発生時にすぐに通知を受けることができます。
    </Callout>
  </Collapser>

  <Collapser
    id="custom"
    title="カスタム"
  >
    特定の集計ニーズに合わせて独自の<DoNotTranslate>**custom metric**</DoNotTranslate>を作成します。 カスタマイズしたメトリックに名前を追加し、独自の閾値を定義します。
  </Collapser>
</CollapserGroup>

## ポリシーと条件を見る [#policy]

特定のモバイルアプリのアラートポリシーと条件情報を表示するには、次のようにします。

1. <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > Alert conditions**</DoNotTranslate>

   に移動します。

2. <DoNotTranslate>**Alert conditions**</DoNotTranslate>

   ページから、利用可能なツールを使用して、アラート条件とそれに関連付けられたポリシーを検索、並べ替え、表示、または更新します。

## イベント・アクティビティを見る [#activity]

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) やその他のアクティビティのサマリー情報をモバイルモニタリングUIから直接表示することができます。

1. <DoNotTranslate>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile**</DoNotTranslate>

   に移動します。

2. インデックスから、 エンタープライズの[色分けされた稼働ステータス](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status)にマウスを移動し、

   <DoNotTranslate>**Mobile activity**</DoNotTranslate>

   リストからリンクを選択するか、モバイル アプリを選択して追加の詳細を表示します。

たとえば、 <DoNotTranslate>**Critical**</DoNotTranslate>アラートインシデントが発生した場合:

* モバイルインデックスや選択したアプリでは、健康状態の表示が赤くなります。

* 各種チャートの背景色がピンクに変わります。

* モニター モバイル アプリのリストの

  <DoNotTranslate>**Application activity**</DoNotTranslate>

  セクションには、該当する

  <DoNotTranslate>**Warning**</DoNotTranslate>

  (黄色) と

  <DoNotTranslate>**Critical**</DoNotTranslate>

  (赤色) のインシデントが表示されます。

アラート インシデントの詳細を確認するには、グラフ上のピンク色の領域にマウスを置くか選択します。

## アラート インシデントの表示 [#incidents]

<Callout variant="important">
  2020 年 7 月に、 browserアプリ、モバイル アプリ、外形監視モニターのアラートインシデントを <DoNotTranslate>**[one.newrelic.com](https://one.newrelic.com/all-capabilities)**</DoNotTranslate> の新しい形式に移行しました。 詳細については、[プライドインテリジェンスとアラートの](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai)ドキュメントを参照してください。
</Callout>

アラート条件の[閾値](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold)が<DoNotTranslate>**Warning**</DoNotTranslate> (黄色) または<DoNotTranslate>**Critical**</DoNotTranslate> (赤色) の稼働ステータスに設定されている場合、製品エレメントの色分けされた[稼働ステータスが](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status)アラートを示すように変わります。 New Relic のモバイル アプリのページからインシデントを直接表示できます。

1. <DoNotTranslate>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile**</DoNotTranslate>

   に移動します。

2. モバイル アプリを選択し、その

   <DoNotTranslate>**Open incidents**</DoNotTranslate>

   を確認します。
