適切に定義されたアラートは、システムの変更を個人やチームに通知するのに役立ちます。 [アラートやアプライド・インテリジェンス](/docs/alerts-applied-intelligence) の機能は、New Relic で監視しているすべてのエンティティで使用することができます。

たとえば、 [アラート](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts) を使用して、New Relic で監視するモバイルアプリのメトリクスに焦点を当てたアラートポリシーと条件を管理することができます。 [モバイルアプリのアラート条件](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions) を定義したら、その情報をアラートUIや、モバイルモニタリングUIなどの他のUIエクスペリエンスで表示することができます。

## モバイルアラートの条件 [#conditions]

モバイルアプリにアラートを設定する際には、以下の条件としきい値のいずれかを使用します。

<CollapserGroup>
  <Collapser
    id="execution"
    title="実行時間"
  >
    実行時間のメトリクスには、 **Database**, **Images**, **JSON**, **Network**, **View loading** があります。これらのメトリクスのしきい値を定義するには、時間と期間を選択します。
  </Collapser>

  <Collapser
    id="errors"
    title="エラー"
  >
    エラーメトリクスには、 **Network error percentage** と **Status error percentage** があります。これらのメトリクスのしきい値を定義するには、エラーの割合と期間を選択します。
  </Collapser>

  <Collapser
    id="crash"
    title="クラッシュレポート"
  >
    アプリの **crash rate** が一定のしきい値を超えたときに通知を受けたい場合があります。これらのメトリクスのしきい値を定義するには、エラーの割合と期間を選択します。

    <Callout variant="tip">
      PagerDutyやSlackなど、 [通知チャンネル](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts) を設定することで、アプリのクラッシュ率の超過について、発生時にすぐに通知を受けることができます。
    </Callout>
  </Collapser>

  <Collapser
    id="custom"
    title="カスタム"
  >
    特定のアラートのニーズに合わせて、独自の **カスタムメトリック** を作成します。カスタマイズされたメトリックの名前を追加し、独自のしきい値を定義します。
  </Collapser>
</CollapserGroup>

## ポリシーと条件を見る [#policy]

特定のモバイルアプリのアラートポリシーと条件情報を表示するには、次のようにします。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile > Alert conditions**に移動します。
2. **Alert conditions** ページから、利用可能なツールを使用して、警告条件とそれに関連するポリシーの検索、ソート、表示、更新を行います。

## イベント・アクティビティを見る [#activity]

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) やその他のアクティビティのサマリー情報をモバイルモニタリングUIから直接表示することができます。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > モバイル**に移動します。
2. インデックスから、企業の [色分けされた健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) にマウスオーバーするか、 **モバイル活動** リストからリンクを選択するか、モバイルアプリを選択すると、さらに詳細が表示されます。

例えば、 **Critical** のアラート違反が発生した場合。

* モバイルインデックスや選択したアプリでは、健康状態の表示が赤くなります。
* 各種チャートの背景色がピンクに変わります。
* 監視対象のモバイルアプリケーションのリストでは、 **アプリケーションアクティビティ** のセクションに、 **警告** （黄色）と **致命的** （赤色）の違反が該当するように表示されます。

アラート違反の詳細を確認するには、チャート上のピンク色の部分にマウスオーバーするか、選択してください。

## アラート違反の表示 [#violations]

<Callout variant="important">
  2020 年 7 月に、ブラウザー アプリ、モバイル アプリ、Fand 合成モニターのアラート違反を **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**の新しい形式に移行しました。詳細については、 [適用されたインテリジェンスとアラートの](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai) ドキュメントを参照してください。
</Callout>

警告条件に [しきい値](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) が設定されている場合、 **警告** (黄色) または **致命的** (赤色) の違反があると、製品エンティティの色分けされた [健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) が違反を示すように変化します。違反の内容は、New Relic のモバイルアプリのページから直接確認できます。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > モバイル**に移動します。
2. モバイルアプリを選択して、その **オープンバイオレーション** を確認します。
