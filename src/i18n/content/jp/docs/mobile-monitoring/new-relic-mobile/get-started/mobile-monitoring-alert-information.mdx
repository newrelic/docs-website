明確に定義された<InlinePopover type="alerts"/>システムの変更について個人またはチームに通知するのに役立ちます。 New Relic で監視するすべてのエンティティにわたって[、アラートと応用インテリジェンス](/docs/alerts-applied-intelligence)機能を使用できます。

たとえば、[アラート](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts)を使用して、New Relic で監視するモバイル アプリのメトリクスに焦点を当てたアラート ポリシーと条件を管理できます。モバイル アプリの[アラート条件を定義する](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-alert-conditions)と、その情報をアラート UI または他の UI エクスペリエンス ( <InlinePopover type="mobile"/>UI。

## モバイルアラートの条件 [#conditions]

モバイルアプリにアラートを設定する際には、以下の条件としきい値のいずれかを使用します。

<CollapserGroup>
  <Collapser
    id="execution"
    title="実行時間"
  >
    実行時間のメトリクスには、 **Database**, **Images**, **JSON**, **Network**, **View loading** があります。これらのメトリクスのしきい値を定義するには、時間と期間を選択します。
  </Collapser>

  <Collapser
    id="errors"
    title="エラー"
  >
    エラーメトリクスには、 **Network error percentage** と **Status error percentage** があります。これらのメトリクスのしきい値を定義するには、エラーの割合と期間を選択します。
  </Collapser>

  <Collapser
    id="crash"
    title="クラッシュレポート"
  >
    アプリの **crash rate** が一定のしきい値を超えたときに通知を受けたい場合があります。これらのメトリクスのしきい値を定義するには、エラーの割合と期間を選択します。

    <Callout variant="tip">
      PagerDutyやSlackなど、 [通知チャンネル](/docs/alerts/new-relic-alerts/managing-notification-channels/notification-channels-controlling-where-send-alerts) を設定することで、アプリのクラッシュ率の超過について、発生時にすぐに通知を受けることができます。
    </Callout>
  </Collapser>

  <Collapser
    id="custom"
    title="カスタム"
  >
    特定のアラートのニーズに合わせて、独自の **カスタムメトリック** を作成します。カスタマイズされたメトリックの名前を追加し、独自のしきい値を定義します。
  </Collapser>
</CollapserGroup>

## ポリシーと条件を見る [#policy]

特定のモバイルアプリのアラートポリシーと条件情報を表示するには、次のようにします。

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > Alert conditions**に移動します。
2. **Alert conditions** ページから、利用可能なツールを使用して、警告条件とそれに関連するポリシーの検索、ソート、表示、更新を行います。

## イベント・アクティビティを見る [#activity]

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) やその他のアクティビティのサマリー情報をモバイルモニタリングUIから直接表示することができます。

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile**に移動します。
2. インデックスから、企業の [色分けされた健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) にマウスオーバーするか、 **モバイル活動** リストからリンクを選択するか、モバイルアプリを選択すると、さらに詳細が表示されます。

たとえば、 **Critical** \[重大な] アラート インシデントが発生した場合:

* モバイルインデックスや選択したアプリでは、健康状態の表示が赤くなります。
* 各種チャートの背景色がピンクに変わります。
* 監視対象のモバイル アプリのリストでは、 **Application activity** \[アプリケーション アクティビティ] セクションに、該当する場合、 **Warning** \[警告] (黄色) および **Critical** \[重大] (赤色) のインシデントが表示されます。

アラート インシデントの詳細を確認するには、グラフ上のピンク色の領域にマウスを置くか選択します。

## アラート インシデントの表示 [#incidents]

<Callout variant="important">
  2020 年 7 月に、ブラウザ アプリ、モバイル アプリ、合成モニターのアラート インシデントを **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**の新しい形式に移行しました。詳細については、 [応用インテリジェンスとアラートに関する](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai) ドキュメントを参照してください。
</Callout>

アラート条件に 警告 (黄色) または **重大** ****(赤色) インシデントの [しきい値 が設定されている場合、製品エンティティの色分けされた](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) [正常性ステータス](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) が変化してインシデントを示します。New Relic のモバイル アプリのページからインシデントを直接表示できます。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > モバイル**に移動します。
2. モバイル アプリを選択し、その **Open incidents** \[未解決のインシデント] を確認します。
