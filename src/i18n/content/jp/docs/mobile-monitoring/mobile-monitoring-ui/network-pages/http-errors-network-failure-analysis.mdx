## HTTPエラーページの検索と使用 [#errors_ui_workflow]

**HTTPエラー** のページに行くには2つの方法があります。

* Go to **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> ネットワーク> ネットワークエラー**.
* モバイルモニタリングのモバイルアプリの [Overview](/docs/mobile-apps/mobile-apps-dashboard) ページから、 **HTTP errors/network failures** chart title リンクを選択します。

**HTTPエラー** ページから、HTTPリクエストとネットワークの障害を調査します。

* 任意の [標準的なページ機能](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) を使用して、 **Errors and Failures** チャートの傾向を調べます。
* [グループ化、ソート、フィルタリングにより、特定のタイプのエラーや障害を対象とする](#groups-sort-filters) データ。
* [HTTPエラープロファイルを使って、リクエストエラーの異常性を見つけます。](/node/14906/revisions/275421/view#http-error-profiles)
* エラーや失敗を選択して、 [詳細](#summary) を表示します。

また、 [NRQL アラートを定義することもできます。](/docs/alerts/new-relic-alerts/defining-conditions/create-alert-conditions-nrql-queries) クリティカルなサービスのエラータイプに焦点を当てたり、 [アプリのデータを照会したりすることができます。](#insights) 。

## HTTPエラープロファイル

エラープロファイルは、HTTPエラーイベントの異なる値の頻度の著しい違いについて、視覚的な詳細を提供します。各 [属性](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#attribute) について、エラープロファイルには以下が含まれます。

* 誤差の属性が最も乖離している値にどのように分布しているかを示す円グラフ
* エラー属性の分布を他のエラーの分布と比較した表

これにより、モバイルアプリケーションのHTTPエラーを解決するための推測作業を減らすことができます。エラーを無視してよいのか、新しいデプロイメント、コードの変更、顧客とのコミュニケーションなどでエラーの解決を図るべきなのかを、より簡単に判断することができます。

## 特定のエラーの詳細を表示する [#error-summary]

エラーや障害の詳細を表示するには、 **Request URL** のリンクを選択して、 **Error summary** ページに移動します。 **Error summary** ページでは、バージョン情報、リクエスト属性、 **Response** bodyを確認できるほか、リクエストURLのエラータイプの内訳も確認できます。

## クエリビルダーによるエラーデータの表示と共有 [#insights]

**HTTP errors/requests** ページに掲載されているチャートやリストの背後にあるデータを調べるには、このページをご覧ください。

1. <Icon name="fe-more-horizontal"/>

   を任意のチャートに選択します。

2. **View query** を選択し、次に **View in Insights** を選択します。これで、クエリビルダが開きます。

クエリビルダーからは、エラーデータをダッシュボードに追加したり、 [パーマリンクで共有することができます](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/#share) 。

エラーデータをより深く掘り下げるために、 [データ](/docs/query-your-data/explore-query-data/explore-data/introduction-querying-new-relic-data) に以下のイベントや属性を照会します。

* [`MobileRequestError`](/docs/insights/nrql-new-relic-query-language/nrql-examples/insights-query-examples-mobile#mobilerequesterror-examples) イベントと [属性](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequesterror-attributes)
* [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) イベントと [属性](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes)

## レガシーHTTPエラーUIページの表示 [#non-enterprise-http-requests]

Enterprise レベルのサブスクリプションを持っていないアカウントには、別の **HTTP Errors** UI ページが表示されます。

**Errors** ページには、ホストのHTTPエラー（403、404、422、500、502など）やネットワーク障害の詳細が記載されています（例：ネットワーク障害）。

* 安全な接続に失敗
* タイムアウト
* ホストが見つからない
* インターネットに接続されていない
* ホストへの接続ができない

## エラーのページを見る [#dashboard]

モバイルアプリのHTTPエラーやネットワーク障害を表示するには、次のようにします。

1. アクセス **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> ネットワーク> エラー**.
2. 表示をエラーや故障に変更するには、 **「Sort by** 」オプションを選択します。
3. 使用率の低いホストを隠すには、 **Hide&lt; 1% throughput** オプションを選択します。
4. 情報をアプリの特定の [バージョン](/docs/mobile-monitoring-ui/versions) に限定したり、期間を変更したりするには、 **バージョン** メニューまたはメニューバーの下にある [**タイムピッカー**](/docs/site/timepicker-setting-time-periods-to-view-data) から選択してください。
5. 特定のホスト、HTTPステータスエラー、またはネットワーク障害の詳細を表示するには、その名前を選択します。

当社の標準的な [ユーザーインターフェース機能](/docs/accounts-partnerships/education/getting-started-new-relic/new-relic-user-interface) を使って、詳細な情報にドリルダウンすることができます。

## エラートレースの詳細 [#trace]

モバイル監視では、400または500レベルのステータスコードを返すHTTPリクエストの応答詳細をキャプチャします。また、Androidアプリで発生したエラーメッセージには、スタックトレースが含まれます。

**Errors** ページでエラートレースの詳細を表示するには、そのリクエストURLリンクを選択します。ここから、以下のことができます。

* レスポンスボディを表示します。
* エラーの詳細をメールで他の人と共有する。
* エラーを削除または非表示にします。

エラーチャートは、選択したモバイルアプリの [Overview](/docs/mobile-apps/mobile-apps-dashboard) ページにも表示されます。チャートにエラーが表示されている場合、その **HTTPエラー/ネットワーク障害** タイトルを選択するか、概要ページのチャートのどこかを選択すると、この **エラー** ページに直接移動できます。

## クエリビルダでエラーデータを表示 [#insights]

リクエストデータをより深く掘り下げるために、クエリービルダーを使って [クエリーとチャート](/docs/insights/nrql-new-relic-query-language/nrql-examples/insights-query-examples-mobile#mobilerequesterror-examples) `MobileRequest` イベントと属性を表示します。

## 不明なエラーやURLのエラー [#unknown_url]

モバイルエージェントは、例外の種類のリストを保持しています。場合によっては、アプリケーションが投げるカスタム例外がこのリストに含まれないことがあります。このような場合、 `Unknown` がモバイルの **Errors** ページに表示されることがあります。

エラーのリストに `Unknown` があり、どの例外タイプが見逃されているかを調べるサポートが必要な場合は、 [support.newrelic.com](https://support.newrelic.com "新しいウィンドウで開くリンク") でサポートを受けてください。
