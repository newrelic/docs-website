[モバイルモニタリング](/docs/mobile-monitoring) には、 **HTTPリクエスト** UIページがあり、モバイルアプリに関連するHTTPリクエストと、それらのネットワークコールがパフォーマンスにどのような影響を与えているかをよりよく理解することができます。

このドキュメントでは、Enterprise レベルの **HTTP リクエスト** ページについて説明します。エンタープライズ以外のアカウントには、 [レガシーHTTPリクエストのページ](#non-enterprise-http-requests) が表示されます。

## HTTPリクエストの検索と利用 ページ [#find-page]

モバイルモニタリングの **HTTPリクエスト** のページを表示する。

1. Go to **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> ネットワーク> HTTPリクエスト**.
2. [標準ページ関数](/docs/data-analysis/user-interface-functions/view-your-data/standard-page-functions) を使用して、HTTP分析チャートの傾向を調べます。
3. [グループ化、ソート、フィルタリングにより、特定のリクエストやレスポンスの属性を対象とする](#groups-sort-filter) データ。

## HTTPリクエストデータの理解 [#understand-data]

ここでは、最も重要なHTTPリクエストの情報が得られる場所をご紹介します。

<CollapserGroup>
  <Collapser
    id="response-time-chart"
    title="応答時間チャート"
  >
    レスポンスタイムチャートは、アプリのネットワークコールのパフォーマンスをパーセンタイルごとに示したものです。このチャートを使用して、平均応答時間を1パーセンタイル、50パーセンタイル、99パーセンタイルと比較します。パーセンタイルによって、平均応答時間を予想以上に長くしている可能性のある異常値を除外することができます。
  </Collapser>

  <Collapser
    id="errors-network-failures-chart"
    title="HTTPエラーとネットワーク障害のチャート"
  >
    このチャートは、アプリケーションで発生している失敗したネットワークコールを示しています。チャートのタイトルを選択すると、 [**HTTPエラー** ページ](/docs/mobile-monitoring/mobile-monitoring-ui/network-pages/network-errors-http-error-network-failure-analysis) エラーや障害の詳細が表示されます。
  </Collapser>

  <Collapser
    id="total-requests-chart"
    title="リクエスト総数"
  >
    **Total requests** でソートすると、どのネットワークリクエストが最も頻繁に使用されているかがわかります。これが役立つ理由は、最も遅いネットワークコールは使用頻度が低いかもしれない一方で、使用頻度の高いリクエストは、たとえ最も遅いリクエストではなくても、最適化する価値があるかもしれないからです。
  </Collapser>
</CollapserGroup>

エンタープライズ以外の **HTTPリクエスト** UIページの説明については、 [レガシーHTTPリクエスト](#non-enterprise-http-requests) を参照してください。

## HTTPリクエストデータの表示と共有 [#insights]

**HTTPリクエスト** チャートをInsightsで表示するには。

1. <Icon name="fe-more-horizontal"/>

   を任意のチャートに選択します。

2. 選択 **クエリを見る> Insights で見る**.

3. オプションです。データをダッシュボードに追加したり、 [パーマリンクを使って共有したりする](/docs/data-analysis/user-interface-functions/share-your-data/permalink) 。

4. リクエストデータをより深く掘り下げるには、 [`MobileRequest`](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-mobile#mobilerequest-examples) イベントと [属性](/docs/insights/insights-data-sources/default-data/mobile-events-attributes#mobilerequest-attributes) を照会します。
