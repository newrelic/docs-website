<Callout variant="important">
  2020 年 12 月 22 日以降、モバイル**マップ**UI は廃止されています。[サービス マップ](/docs/understand-dependencies/understand-system-dependencies/service-maps/introduction-service-maps)は、各モバイル エンティティの左側のナビゲーションで利用でき、アーキテクチャの表現を視覚化およびカスタマイズするための優れた方法です。詳細については、 [サポート フォーラムの投稿](https://discuss.newrelic.com/t/important-upcoming-changes-to-capabilities-across-browser-php-python-mobile-and-android-agents/123951)を参照してください。
</Callout>

マップは、モバイルアプリやそのサービスのパフォーマンス問題を見つけるのに役立ちます。これにより、アプリと他のサービスの関係や、各サービスが他のサービスに与える影響を明確に把握することができます。あるサービスに障害が発生した場合、他のどのサービスが影響を受けているかが一目瞭然になります。

## モバイルアプリサービスの地図を表示する [#map_procedures]

モバイル アプリとその関連サービスをアーキテクチャ マップとして表示するには、 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile > (モバイル アプリを選択) > Monitor > Service map に移動します**。詳細については、 [サービス マップのドキュメント](/docs/understand-dependencies/understand-system-dependencies/service-maps/how-use-service-maps)を参照してください。

非推奨のモバイル **マップ** ページを使用する必要がある場合は、以下の手順に従ってください。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Mobile > (アプリを選択) > Network > Map**に移動します。
2. [HTTP リクエストの詳細](/docs/mobile-monitoring-ui/http-requests-dashboard) を表示するには、そのサービス名を選択します。
3. サービスに関連するAPM が監視するアプリの [の詳細を表示するには、関連するホスト名の下にあるサービス名を選択します。](/docs/applications-menu/applications-overview)
4. [* スループットの詳細をグラフとして表示するには、サービス名の下にあるアイコンまたは`cpm`バーを選択します。* サービスの詳細なメトリクスを表示するには、スループットチャートにマウスオーバーします。](/docs/applications-menu/applications-overview)

[モバイルモニタリング用の **マップ** ページでは、モバイルアプリとそれが使用するサービスのアーキテクチャビューが表示されます。](/docs/applications-menu/applications-overview)
