<Callout variant="important">
  モバイル**マップ**UIは、2020年12月22日以降廃止されました。 [サービスマップ](/docs/understand-dependencies/understand-system-dependencies/service-maps/introduction-service-maps)は、各モバイルエンティティの左側のナビゲーションで使用でき、アーキテクチャの表現を視覚化およびカスタマイズするためのより良い方法です。詳細については、 [ExplorersHubの投稿](https://discuss.newrelic.com/t/important-upcoming-changes-to-capabilities-across-browser-php-python-mobile-and-android-agents/123951)を参照してください。
</Callout>

マップは、モバイルアプリやそのサービスのパフォーマンス問題を見つけるのに役立ちます。これにより、アプリと他のサービスの関係や、各サービスが他のサービスに与える影響を明確に把握することができます。あるサービスに障害が発生した場合、他のどのサービスが影響を受けているかが一目瞭然になります。

## モバイルアプリサービスの地図を表示する [#map_procedures]

モバイルアプリとその関連サービスをアーキテクチャーマップとして表示するには、 **[one.newrelic.com](https://one.newrelic.com) > モバイル> （モバイルアプリを選択）> モニター> サービスマップ** 。詳細については、 [サービスマップのドキュメント](/docs/understand-dependencies/understand-system-dependencies/service-maps/how-use-service-maps) をご覧ください。

非推奨のモバイル **マップ** ページを使用する必要がある場合は、以下の手順に従ってください。

1. Go to **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> ネットワーク> 地図**.
2. [HTTP リクエストの詳細](/docs/mobile-monitoring-ui/http-requests-dashboard) を表示するには、そのサービス名を選択します。
3. サービスに関連するAPM が監視するアプリの [の詳細を表示するには、関連するホスト名の下にあるサービス名を選択します。](/docs/applications-menu/applications-overview)
4. [* スループットの詳細をグラフとして表示するには、サービス名の下にあるアイコンまたは`cpm`バーを選択します。* サービスの詳細なメトリクスを表示するには、スループットチャートにマウスオーバーします。](/docs/applications-menu/applications-overview)

[モバイルモニタリング用の **マップ** ページでは、モバイルアプリとそれが使用するサービスのアーキテクチャビューが表示されます。](/docs/applications-menu/applications-overview)
