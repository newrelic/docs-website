## HTTP エラー ページを発見する [#ui-interactions]

**HTTPエラー** ページから、HTTPリクエストとネットワークの障害を調査します。

* 標準のページ関数を使用して、 **Errors and failures**\[エラーと障害の] グラフの傾向を調べます。
* データをグループ化、並べ替え、フィルター処理することで、特定の種類のエラーと障害をターゲットにします。
* HTTP エラー プロファイルを使用して、リクエスト エラーの異常を見つけます。
* エラーまたは失敗を選択すると、詳細が表示されます。

また、 [NRQL アラートを定義することもできます。](/docs/alerts/new-relic-alerts/defining-conditions/create-alert-conditions-nrql-queries) クリティカルなサービスのエラータイプに焦点を当てたり、 [アプリのデータを照会したりすることができます。](#insights) 。

## HTTPエラープロファイル [#error-profiles]

エラー プロファイルは、HTTP エラー イベントのさまざまな値の頻度の大きな違いに関する視覚的な詳細を提供します。 各[属性の](/docs/new-relic-solutions/get-started/glossary/#attribute)エラー プロファイルには次のものが含まれます。

* 最も逸脱した値に対してエラーの属性がどのように分布しているかを示すヒートマップ
* エラー属性の分布を他のエラーの分布と比較するラベル

これにより、モバイルアプリケーションのHTTPエラーを解決するための推測作業を減らすことができます。エラーを無視してよいのか、新しいデプロイメント、コードの変更、顧客とのコミュニケーションなどでエラーの解決を図るべきなのかを、より簡単に判断することができます。

## 1 つのエラーにドリルダウンする [#error-summary]

エラーまたは失敗の詳細を表示するには、上位 20 件のエラー リストからリクエスト URL を選択します。 エラー概要ページから、バージョン情報、リクエスト属性、レスポンス本文を表示し、リクエスト URL のエラー タイプの内訳を取得できます。

**HTTP errors** \[HTTP エラー]ページのグラフまたはリストの背後にあるデータを探索するには、次の手順を実行します。

## トレースの詳細を取得する [#trace-details]

モバイル監視では、400または500レベルのステータスコードを返すHTTPリクエストの応答詳細をキャプチャします。また、Androidアプリで発生したエラーメッセージには、スタックトレースが含まれます。

**Errors** ページでエラートレースの詳細を表示するには、そのリクエストURLリンクを選択します。ここから、以下のことができます。

* レスポンスボディを表示します。
* エラーの詳細をメールで他の人と共有する。

エラー グラフは、選択したモバイル アプリの[\[概要\]](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/mobile-apps-summary-page/)ページにも表示されます。

## 不明なエラーまたは URL エラーのトラブルシューティング [#troubleshoot-unknown-url]

モバイル エージェントは、例外タイプのリストを維持します。場合によっては、アプリケーションによってスローされるカスタム例外がこのリストに含まれないことがあります。この場合、モバイル**エラー**ページに`Unknown`が表示されることがあります。

エラーのリストに`Unknown`が見つかり、どの例外タイプが見落とされているかを調査する際にサポートが必要な場合は、 [support.newrelic.com](https://support.newrelic.com "リンクが新しいウィンドウで開きます")でサポートを受けてください。
