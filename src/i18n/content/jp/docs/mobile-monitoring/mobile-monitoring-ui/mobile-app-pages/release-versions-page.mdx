## リリース バージョン ページを表示する [#view-release-versions-page]

1. \*\* [one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities)> Mobile に移動します。
2. モバイルアプリを選択します。
3. 左側のメニューで、 **Release versions** \[リリースバージョン]をクリックします。

<Callout variant="tip">
  データが表示されない場合は、 次のことを試してください。

  * 新しいバージョンがリリースされる前のデータを含めるように時間フィルターを調整します。
  * モバイル エージェントが iOS エージェント バージョン 7.0.0 以降、または Android エージェント バージョン 6.5.0 以降を使用していることを確認してください。
</Callout>

## 特定のメトリクスにドリルダウンする [#drill-down]

\[リリース バージョン] ページには、各アプリ バージョンの高レベルのメトリクスが表示されます。

* **Overall performance score** \[全体的なパフォーマンス スコア]: 全体的な顧客クラッシュ率と HTTP エラーなし率によって計算されます。
* **Crash free user %** \[Crash free ユーザー %] : 特定のバージョンでクラッシュが発生していないユーザーの割合。
* **User adoption** \[ユーザー導入]: 特定のバージョンを使用しているユーザーの割合。

**View summary** \[概要の表示]または**View details** \[詳細の表示]をクリックして、アプリ バージョンの特定のメトリクスを展開します。 わかりやすい視覚化と詳細なグラフを備えたバージョン概要ページでは、主要なパフォーマンス指標を分析し、アプリのユーザー エクスペリエンスを最適化するための情報に基づいた意思決定を行うことができます。

上記の各メトリクスについて、リリース バージョン ページには動的な時系列グラフが含まれており、時間の経過に伴うメトリクスのパフォーマンスを追跡できます。 チャートの傾向、急上昇、または降下を観察することで、意思決定に役立つパターンや相関関係を迅速に特定できます。

## アラートの作成 [#create-alerts]

アプリのパフォーマンスを可視化したので、次は主要なパフォーマンスの変化に対するアラートを設定します。

メトリクスをクリックして、チャート メニューから**Create alert condition** \[アラート条件の作成]をクリックし、UI の手順に従います。
