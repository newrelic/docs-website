最新のアプリのリリースのパフォーマンスについて知りたいですか? リリース バージョン ページを使用して、さまざまなリリース バージョン間のクラッシュ率、ユーザー エンゲージメント、全体的なパフォーマンス メトリクスを表示し、最新リリースの成功を分析するために使用できます。

## リリース バージョン ページを表示する [#view-release-versions-page]

1. [one.newrelic.com > All capabilities \[すべての機能\]](https://one.newrelic.com/all-capabilities)> Mobile \[モバイル]に移動します。

2. モバイルアプリを選択します。

3. 左側のメニューで、

   <DNT>
     **Release versions**
   </DNT>

   をクリックします。

<Callout variant="tip">
  データが表示されない場合は、 次のことを試してください。

  * 新しいバージョンがリリースされる前のデータを含めるように時間フィルターを調整します。
  * モバイル エージェントが iOS エージェント バージョン 7.0.0 以降、または Android エージェント バージョン 6.5.0 以降を使用していることを確認してください。
</Callout>

## 特定のメトリクスにドリルダウンする [#drill-down]

\[リリース バージョン] ページには、各アプリ バージョンの高レベルのメトリクスが表示されます。

* <DNT>
    **Overall performance score**
  </DNT>

  : 全体的な顧客クラッシュ率と HTTP エラーフリー率によって計算されます。

* <DNT>
    **Crash free user %**
  </DNT>

  : 特定のバージョンでクラッシュが発生していないユーザーの割合。

* <DNT>
    **User adoption**
  </DNT>

  : 特定のバージョンに更新またはインストールしたユーザーの割合。

<DNT>**View summary**</DNT>または<DNT>**View details**</DNT>をクリックすると、アプリ バージョンの特定のメトリックスに展開されます。 わかりやすい視覚化と詳細なグラフを備えたバージョン概要ページを使用すると、主要パフォーマンス指標を分析し、情報に基づいた意思決定を行ってアプリのユーザーエクスペリエンスを最適化できます。

<img
  title="Details of release versions"
  alt="Screenshot of release versions detailed view"
  src="/images/mobile_screenshot-full_release-versions-details.webp"
/>

上記の各メトリクスについて、リリース バージョン ページには動的な時系列グラフが含まれており、時間の経過に伴うメトリクスのパフォーマンスを追跡できます。 チャートの傾向、急上昇、または降下を観察することで、意思決定に役立つパターンや相関関係を迅速に特定できます。

## アラートの作成 [#create-alerts]

アプリのパフォーマンスを可視化したので、次は主要なパフォーマンスの変化に対するアラートを設定します。

メトリックをクリックし、チャート メニューから <DNT>**Create alert condition**</DNT> をクリックして、 UIの手順に従います。
