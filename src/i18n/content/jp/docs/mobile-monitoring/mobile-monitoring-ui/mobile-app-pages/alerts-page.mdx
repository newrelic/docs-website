モバイル アプリのエラー率と応答時間に設定した[アラート条件](/docs/mobile-apps/alert-settings)が一定期間にわたって閾値を超えると、 <InlinePopover type="mobile"/>エージェントは警告 (黄色) または重大 (赤) の問題イベントを作成します。 重大な (赤) イベントが数分間継続している場合、モバイルは集計通知を受信するように設定されているユーザーに集計通知を送信します。 すべての重要なイベントが終了したときにも通知が送信されます。

<img
  title="alert conditions page "
  alt="screenshot of alert conditions page for mobile monitoring"
  src="/images/mobile_screenshot-full_alert-conditions-page.webp"
/>

アラート条件ページを表示するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Alert conditions**</DNT>に移動します。

## アラート条件ページの詳細

各アラート通知には、アラート アイコンと概要情報が含まれます。該当する [UI 機能](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform) のいずれかを使用して、詳細情報にドリルダウンします。さらに、次のオプションのいずれかを使用します。
