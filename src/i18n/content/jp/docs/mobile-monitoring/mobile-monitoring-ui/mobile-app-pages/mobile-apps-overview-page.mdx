**Overview** ページでは、モバイルアプリの操作上のスナップショットを提供しています。すべてのチャート、テーブル、統計は、現在選択されている [時間ウィンドウ](/docs/apm/new-relic-apm/ui-functions/time-picker-setting-time-periods-view-data) とアプリケーションのバージョンのデータを表示します。

## アプリの主要指標 [#dsahboard-metrics]

概要ページでは、アプリの主要な5つの指標を把握することができます。

<table>
  <thead>
    <tr>
      <th width={200}>
        メトリック
      </th>

      <th>

      </th>
    </tr>

  <tbody>
    <tr>
      <td>
        **アプリケーションのクラッシュ率**
      </td>

      <td>
        クラッシュしたセッションの数を、アプリのバージョン別に、全セッション数に対する割合として時間軸上にプロットします。右上には、平均的なクラッシュセッションの割合、クラッシュの総数、期間中にクラッシュの影響を受けたユニークユーザーの数が表示されます。
      </td>
    </tr>

    <tr>
      <td>
        **アプリ発売**
      </td>

      <td>
        期間中にモニターされたアプリのセッション起動数をグラフ化しています。

        New Relic では、モバイルセッションを、アプリが画面に表示されたときに始まり、アプリがバックグラウンドに送られたときに終わると定義しています。
      </td>
    </tr>

    <tr>
      <td>
        **HTTPエラー／ネットワーク障害**
      </td>

      <td>
        **http status code error** （400以上）となったネットワークリクエストの数を、完了したリクエスト全体に占める割合として、また、 **失敗したネットワークコール** （接続失敗など）の数を、完了したリクエストに占める割合としてグラフ化しています。
      </td>
    </tr>

    <tr>
      <td>
        **HTTPレスポンスタイム**
      </td>

      <td>
        アプリが通信する上位5つのホストからのすべての完了したhttpリクエストの平均応答時間をグラフ化します。

        トップ5は、各ホストへの完了したリクエストの数に基づいて計算されます。
      </td>
    </tr>

    <tr>
      <td>
        **頻繁に行われる交流**
      </td>

      <td>
        アプリ内で最もよく実行される5つのインターアクションのそれぞれについて、主要なパフォーマンス指標を提示します。
      </td>
    </tr>
  </tbody>
</table>

## 概要ページを見る [#overview-page]

モバイルアプリの概要ページを表示するには

1. **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）**.
2. モバイルアプリの他のページを表示するには、OverviewページまたはMobileメニューからリンクを選択します。

![モバイルアプリの新機能を紹介](./images/new-mobile-apps-overview.png "モバイルアプリの新機能を紹介")

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）：** 「概要」ページには、モバイルアプリケーションのパフォーマンスに関する洞察を得るために、ドリルダウンできるチャートやテーブルが用意されています。
</figcaption>
