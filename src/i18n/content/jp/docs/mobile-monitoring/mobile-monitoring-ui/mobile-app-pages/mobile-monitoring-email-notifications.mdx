New Relic のモバイルモニタリングでは、以下のようないくつかのメールレポートを提供しています。

* [リリース通知](#version-trends) (別名：バージョン・トレンド・メール)
* [クラッシュレポートメール](#crash-reports)

## バージョンアップ傾向のメール [#version-trends]

New Relic のモバイルモニタリングでは、 [**バージョントレンド** UI](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/version-trends-compare-user-adoption-metrics-performance).また、バージョントレンドメール（別名：リリース通知）を有効にすることもできます。これらのレポートは、アプリの進捗状況や採用状況、その他のトレンドを共有、追跡、レビューするための便利なコミュニケーションツールとなります。各メールには、詳細を見るための **バージョントレンド** UI へのリンクが含まれています。

定期的にレポートをメールでお送りします。

### バージョン・トレンド・メール設定の編集 [#version-trend-ui]

[バージョントレンドのメール設定方法は、どのユーザーモデル](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model#user-models) にいるかによって異なります。

<Callout variant="tip">
  お客様のユーザーモデルを簡単に確認するには、 [アカウントのドロップダウン](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown) にアクセスし、 **ユーザー設定** をクリックします。モバイルメールの設定が表示されていれば、オリジナルのユーザーモデルであることがわかります。
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-version-trends"
    title="バージョンメール（オリジナルユーザーモデル）"
  >
    監視しているすべてのモバイルアプリのバージョントレンドのメールレポートを受信するかしないかを選択します。

    1. Go to **[one.newrelic.com](one.newrelic.com) > Mobile> (select a mobile app)> App> Version trends**.

    2. **バージョントレンドのメール通知を有効にする** リンクを選択し、設定を編集します。

       特定のモバイルアプリにメールレポートを設定するには、 **[one.newrelic.com](https://one.newrelic.com) > (選択したアプリ)> 設定> アプリ** にアクセスし、設定を編集します。
  </Collapser>

  <Collapser
    id="new-relic-one-user-version-trends"
    title="バージョンメール（New Relic Oneのユーザーモデル）"
  >
    特定のモバイルアプリで受信するメールレポートを設定するには、 **[one.newrelic.com](https://one.newrelic.com) > (選択したアプリ)> 設定> アプリケーション** にアクセスし、リリース通知メールの設定を編集します。
  </Collapser>
</CollapserGroup>

## クラッシュレポートメール [#crash-reports]

モバイルアプリで新種のクラッシュが発生した場合、何が悪かったのかを知り、今後このようなエラーを防ぐことができます。クラッシュレポートメールでは、固有のクラッシュが検出されたときに通知されます。

クラッシュレポートのメール設定の方法は、 [ユーザーモデル](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model#user-models) の種類によって異なります。

<Callout variant="tip">
  お客様のユーザーモデルを簡単に確認するには、 [アカウントのドロップダウン](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown) にアクセスし、 **ユーザー設定** をクリックします。モバイルメールの設定が表示されていれば、オリジナルのユーザーモデルであることがわかります。
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-crash-emails"
    title="クラッシュメール（オリジナルユーザーモデル）"
  >
    複数のアプリでお客様のアカウントのクラッシュレポートメールの設定を行うには： **[one.newrelic.com](https://one.newrelic.com) > [（アカウントのドロップダウン）](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#account-dropdown) > ユーザー設定**.

    監視対象の特定のモバイルアプリのクラッシュレポートのメール設定を行うには

    1. **[one.newrelic.com](https://one.newrelic.com) > [（アカウントのドロップダウン）](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#account-dropdown) > ユーザー設定**.
    2. **モバイル クラッシュ レポート** のセクションで、モバイル アプリの名前にマウスオーバーすると、 **アプリケーション設定の編集** が表示されます。それを選択します。
    3. これにより、アプリの [**Settings** UI page](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings) が表示されます。 **Enable crash report emails for this application** のチェックボックスをオンにします。
  </Collapser>

  <Collapser
    id="new-relic-one-user-crash-emails"
    title="クラッシュメール（New Relic Oneのユーザーモデル）"
  >
    特定のモバイルアプリのクラッシュメールレポートを受信するかどうかを設定するには、 **[one.newrelic.com](https://one.newrelic.com) > モバイル> (選択したアプリ)> 設定> アプリケーション** にアクセスし、クラッシュレポートのメール設定を編集します。
  </Collapser>
</CollapserGroup>
