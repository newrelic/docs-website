New Relic の<InlinePopover type="mobile"/>は、次のようないくつかの電子メールレポートを提供します。

* [リリース通知](#version-trends) (別名：バージョン・トレンド・メール)
* [クラッシュレポートメール](#crash-reports)

## 新しいバージョンのパフォーマンスに関するメールを受け取る [#version-trends]

New Relicのモバイル監視では、 [<DNT>**Release versions**</DNT>](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/release-versions-page)ページが提供されます。 バージョン トレンド メール (リリース通知とも呼ばれます) を有効にすることもできます。 これらのレポートは、アプリの進捗状況、採用状況、その他の傾向を共有、追跡、確認するための便利なコミュニケーション ツールを提供します。 各メールには、詳細を表示するための<DNT>**Version trends**</DNT> UI へのリンクが含まれています。

定期的にレポートをメールでお送りします。

### バージョン・トレンド・メール設定の編集 [#version-trend-ui]

バージョントレンドのメール設定の方法は、どの [ユーザーモデル](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) を使用しているかによって異なります。

<Callout variant="tip">
  ユーザー モデルをすばやく確認するには、[ユーザー メニュー](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)に移動して<DNT>**User preferences**</DNT>をクリックします。 そこにモバイル メール設定が表示される場合は、元のユーザー モデルを使用しています。
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-version-trends"
    title="バージョンメール（オリジナルユーザーモデル）"
  >
    監視しているすべてのモバイルアプリのバージョントレンドのメールレポートを受信するかしないかを選択します。

    1. <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select a mobile app) > App > Version trends**</DNT>に移動します。

    2. <DNT>**Enable version trends email notifications**</DNT>リンクを選択して設定を編集します。

       特定のモバイル アプリの電子メール レポートを構成するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT>に移動して設定を編集します。
  </Collapser>

  <Collapser
    id="new-relic-one-user-version-trends"
    title="バージョンメール（新しいユーザーモデル）"
  >
    特定のモバイル アプリについて受信するメール レポートを構成するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (select an app) > Settings > Application**</DNT>に移動してリリース通知メール設定を編集します。
  </Collapser>
</CollapserGroup>

## クラッシュレポートメール [#crash-reports]

モバイルアプリで新種のクラッシュが発生した場合、何が悪かったのかを知り、今後このようなエラーを防ぐことができます。クラッシュレポートメールでは、固有のクラッシュが検出されたときに通知されます。

クラッシュレポートの電子メール設定の方法は、どの [ユーザーモデル](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) を使用しているかによって異なります。

<Callout variant="tip">
  ユーザー モデルをすばやく確認するには、[ユーザー メニュー](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)に移動して<DNT>**User preferences**</DNT>をクリックします。 そこにモバイル メール設定が表示される場合は、元のユーザー モデルを使用しています。
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-crash-emails"
    title="クラッシュメール（オリジナルユーザーモデル）"
  >
    複数のアプリにわたってアカウントのクラッシュ レポート メール設定を構成するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**</DNT>に移動します。

    監視対象の特定のモバイルアプリのクラッシュレポートのメール設定を行うには

    1. <DNT>
         **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(user menu)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**
       </DNT>

       に移動します。

    2. <DNT>
         **Mobile crash reporting**
       </DNT>

       セクションで、モバイル アプリ名の上にマウスを置くと、

       <DNT>
         **Edit application settings**
       </DNT>

       が表示されます。 選択してください。

    3. これにより、アプリの

       <DNT>
         [**Settings**](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)
       </DNT>

       [UI ページ](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings)に移動します。

       <DNT>
         **Enable crash report emails for this application**
       </DNT>

       チェックボックスを切り替えます。
  </Collapser>

  <Collapser
    id="new-relic-one-user-crash-emails"
    title="クラッシュメール（新しいユーザーモデル）"
  >
    特定のモバイル アプリのクラッシュ レポートのメールを受信するかどうかを設定するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (select an app) > Settings > Application**</DNT>に移動してクラッシュ レポートのメール設定を編集します。
  </Collapser>
</CollapserGroup>
