* [リリース通知](#version-trends) (別名：バージョン・トレンド・メール)
* [クラッシュレポートメール](#crash-reports)

## バージョンアップ傾向のメール [#version-trends]

New Relic のモバイルモニタリングでは、 [**バージョントレンド** UI](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/version-trends-compare-user-adoption-metrics-performance).また、バージョントレンドメール（別名：リリース通知）を有効にすることもできます。これらのレポートは、アプリの進捗状況や採用状況、その他のトレンドを共有、追跡、レビューするための便利なコミュニケーションツールとなります。各メールには、詳細を見るための **バージョントレンド** UI へのリンクが含まれています。

定期的にレポートをメールでお送りします。

### バージョン・トレンド・メール設定の編集 [#version-trend-ui]

バージョントレンドのメール設定の方法は、どの [ユーザーモデル](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) を使用しているかによって異なります。

<Callout variant="tip">
  ユーザー モデルをすばやく確認するには、[ユーザー メニュー](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)に移動し、 \[**ユーザー設定**] をクリックします。そこにモバイル メール設定が表示されている場合は、元のユーザー モデルを使用しています。
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-version-trends"
    title="バージョンメール（オリジナルユーザーモデル）"
  >
    監視しているすべてのモバイルアプリのバージョントレンドのメールレポートを受信するかしないかを選択します。

    1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (モバイル アプリを選択) > App > Version trends**に移動します。

    2. **バージョントレンドのメール通知を有効にする** リンクを選択し、設定を編集します。

       特定のモバイルアプリの電子メールレポートを設定するには、 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (アプリを選択) > Settings > Application** に移動し、設定を編集します。
  </Collapser>

  <Collapser
    id="new-relic-one-user-version-trends"
    title="バージョンメール（新しいユーザーモデル）"
  >
    特定のモバイルアプリに対して受信する電子メールレポートを設定するには、 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > (アプリを選択) > Settings > Application** に移動し、リリース通知電子メール設定を編集します。
  </Collapser>
</CollapserGroup>

## クラッシュレポートメール [#crash-reports]

モバイルアプリで新種のクラッシュが発生した場合、何が悪かったのかを知り、今後このようなエラーを防ぐことができます。クラッシュレポートメールでは、固有のクラッシュが検出されたときに通知されます。

クラッシュレポートの電子メール設定の方法は、どの [ユーザーモデル](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) を使用しているかによって異なります。

<Callout variant="tip">
  ユーザー モデルをすばやく確認するには、[ユーザー メニュー](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)に移動し、 \[**ユーザー設定**] をクリックします。そこにモバイル メール設定が表示されている場合は、元のユーザー モデルを使用しています。
</Callout>

<CollapserGroup>
  <Collapser
    id="original-user-crash-emails"
    title="クラッシュメール（オリジナルユーザーモデル）"
  >
    複数のアプリにわたってアカウントのクラッシュ レポートの電子メール設定を構成するには、 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(ユーザー メニュー)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**に移動します。

    監視対象の特定のモバイルアプリのクラッシュレポートのメール設定を行うには

    1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > [(ユーザーメニュー)](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings) > User preferences**に移動します。
    2. **モバイル クラッシュ レポート** のセクションで、モバイル アプリの名前にマウスオーバーすると、 **アプリケーション設定の編集** が表示されます。それを選択します。
    3. これにより、アプリの [**Settings** UI page](/docs/mobile-monitoring/new-relic-mobile/maintenance/customizing-your-mobile-app-settings) が表示されます。 **Enable crash report emails for this application** のチェックボックスをオンにします。
  </Collapser>

  <Collapser
    id="new-relic-one-user-crash-emails"
    title="クラッシュメール（新しいユーザーモデル）"
  >
    特定のモバイルアプリのクラッシュ電子メールレポートを受信するかどうかを設定するには、 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (アプリを選択) > Settings > Application** に移動し、クラッシュレポートの電子メール設定を編集します。
  </Collapser>
</CollapserGroup>
