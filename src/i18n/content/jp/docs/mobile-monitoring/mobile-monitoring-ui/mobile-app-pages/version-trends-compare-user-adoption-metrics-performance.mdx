モバイル監視のリリース バージョン ページでは、さまざまなバージョンにわたるモバイル アプリケーションのパフォーマンスの包括的な概要が提供されます。クラッシュ率、ユーザーの採用状況、全体的なパフォーマンス スコアに関するリアルタイムの洞察を提供し、情報に基づいた意思決定を行ってアプリの安定性とユーザー エクスペリエンスを向上させることができます。

## 要件

モバイル アプリのリリース バージョン情報を収集するには、エージェントのバージョンが次であることを確認してください。

* [Androidエージェントv6.5.0](/docs/release-notes/mobile-release-notes/android-release-notes/)以降用の新しいRelicモバイルエージェント
* [iOSエージェントv7.0.0以降用](/docs/release-notes/mobile-release-notes/ios-release-notes/)の新しいRelicモバイルエージェント

## リリースバージョンページの表示 [#view-release-version]

リリース バージョン ページを表示するには **、[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (アプリを選択) > Users > Release versions**移動します。

## メトリクスを理解する [#understanding-metrics]

* **Crash Free Users** \[クラッシュのないユーザー]: アプリのバージョンごとにクラッシュが発生したユーザーの割合を簡単に追跡できるため、傾向を特定し、バグ修正に優先順位を付けることができます。
* **User Adoption** \[ユーザーの採用]: ユーザーがアプリの新しいバージョンをどのように採用しているかを監視し、更新や機能の成功を測定できるようにします。
* **Overall Performance Score** \[全体的なパフォーマンス スコア]: クラッシュ率と HTTP エラーなし率を使用して計算された客観的なパフォーマンス スコアを通じて、アプリの健全性を総合的に把握します。アプリのバージョンでデプロイが有効になっている場合、リリース バージョンではさらに一歩進んで、各バージョンのリリース日が表示されます。この追加機能により、メトリクスに追加のコンテキストが提供され、パフォーマンスの変化を特定のリリース日と関連付けることができます。

## バージョン概要ページ [#version-summary-page]

バージョン概要ページでは、特定のアプリ バージョンごとの詳細な分析情報と指標にアクセスできます。このページでは、さまざまな重要なメトリクスを表示することで、アプリケーションのパフォーマンスを包括的に把握できます。わかりやすい視覚化と詳細なグラフを備えたバージョン概要ページでは、重要なパフォーマンス指標を分析し、情報に基づいた意思決定を行ってアプリのユーザー エクスペリエンスを最適化することができます。特定のアプリ バージョンのバージョン概要ページにアクセスするには、そのバージョンに関連付けられている \[概要を表示] または \[詳細を表示] ボタンをクリックします。表示されるメトリクス:

* **App Launches** \[アプリの起動]: アプリのバージョンが起動された合計回数。
* **Average Interaction Time** \[平均インタラクション時間]: ユーザーがアプリの操作に費やす平均時間。
* **Average Memory Used** \[使用される平均メモリ量]: アプリによって消費されるメモリの平均量。
* **Average Response Time** \[平均応答時間]: アプリがユーザーのアクションに応答するまでにかかる平均時間。
* **Crash Rate** \[クラッシュ率]: クラッシュを引き起こすアプリの起動の割合。
* **Error Rate** \[エラー率]: エラーにつながるアプリ操作の割合。
* **HTTP Error Rate** \[HTTP エラー率]: エラーが発生した HTTP リクエストの割合。
* **Install Count** \[インストール数]: アプリのバージョンのインストールの合計数。
* **Network Failure Rate** \[ネットワーク障害率]: 失敗したネットワーク要求の割合。
* **Request Count** \[リクエスト数]: アプリによって行われた HTTP リクエストの総数。
* **Total Users** \[合計ユーザー数]: アプリのバージョンを操作したユニーク ユーザーの合計数。
* **Upgrade Count** \[アップグレード数]: このアプリのバージョンにアップグレードしたユーザーの総数。上記の各メトリクスについて、バージョン概要ページには動的な時系列グラフが含まれており、時間の経過に伴うメトリクスのパフォーマンスを追跡できます。チャートの傾向、急上昇、または降下を観察することで、意思決定に役立つパターンや相関関係を迅速に特定できます。

## トレンドデータの照会 [#insights]

[NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) で問い合わせる際には、バージョンの傾向情報を分析するのに役立つ2つの属性を提供しています。

* セッションに新規インストールまたは新規アップグレードがいつ含まれるかを追跡するには、 `install`を使用します。この属性は、新規インストールの`true`を記録します。
* アップグレードが検出されたときにモバイル アプリの最新バージョンを追跡するには、 `upgradeFrom`を使用します。これらの属性を使用するには、 [Android エージェント](/docs/release-notes/mobile-release-notes/android-release-notes/) または [iOS エージェント](/docs/release-notes/mobile-release-notes/ios-release-notes/) のバージョンがそれらをサポートしていることを確認してください。
