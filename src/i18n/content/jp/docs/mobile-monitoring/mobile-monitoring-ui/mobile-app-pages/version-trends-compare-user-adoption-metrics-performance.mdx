モバイルモニタリングでは、モバイルアプリの最新バージョン間の使用状況、採用状況、パフォーマンスを比較するためのメトリクスを含む、 **バージョンの傾向** レポートを提供しています。これにより、モバイルアプリの開発マネージャーやそのチームは、1つのページから最大5つのプロダクションバージョンを比較し、それぞれの改善、修正、劣化の影響を分析することができます。

バージョンのトレンドデータは、モバイルモニターのUIで確認することができます。また、 [メール通知](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/review-version-trends-email-reports) を有効にすることができます。

## 要件

**バージョンの傾向** UI:"No adoption data is available for this version because it is running an old agent," これは、モバイルアプリがインストール/アップグレード回数データを収集するNew Relicモバイルエージェントのバージョンを使用していなかったことを示しています。

モバイルアプリのバージョントレンド情報を収集するには、エージェントのバージョンを確認してください。

* New Relic mobile agent for [Android バージョン 5.3.3](/docs/release-notes/mobile-release-notes/android-release-notes/) 以上
* New Relic mobile agent for [iOSバージョン5.3.4](/docs/release-notes/mobile-release-notes/ios-release-notes/) 以上

## バージョンのトレンドデータをUIで表示 [#view-version-trends]

**バージョンの傾向** のレポートを表示するには。

1. Go to **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> アプリ> バージョンの傾向**.

2. **Launch** の期間を選択して、日別（デフォルト）、週別、月別にバージョンを比較したり、利用可能なすべてのデータを表示したりすることができます。

   ![バージョン推移 iOS 2016-01-07 起動ボタン.png](./images/Version-Trends-iOS-2016-01-07-Launch-buttons_0.png "バージョン推移 iOS 2016-01-07 起動ボタン.png")

3. [バージョントレンドの詳細](#details) を見て、パフォーマンス、ユーザーの採用状況、および主要な技術指標を分析してください。データをどのように解釈するかについては、 [例](#examples) をご覧ください。

4. オプションです。電子メールの設定を行うには、 [電子メールのレポート](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-pages/mobile-monitoring-email-notifications#version-trends) を参照してください。

## バージョントレンドの詳細を見る [#details]

各バージョンには、パフォーマンス、ユーザーの採用状況、主要な技術指標を分析するためのいくつかの指標が含まれています。

<CollapserGroup>
  <Collapser
    id="performance"
    title="リリース間のパフォーマンス"
  >
    重要な機能として、各バージョンが人気のピークにあったときに収集されたデータに基づいて、リリース間でアプリのパフォーマンスを比較することができます。各バージョンが運用されていたときのデータは、次のような4つの異なるビューを選択できます。

    * 発売日（デフォルト）

    * 発売週

    * 発売月

    * すべてのバージョンのデータ

      これにより、アプリのバージョンの採用状況やパフォーマンスの特性が他と比べて良いか悪いかをすぐに確認することができます。

      各リリースのモバイルアプリ情報の概要は以下の通りです。

    * **Versions:** アプリのリリースのバージョンを人気の高い日付順に並べたもの。

    * **セッション：** 各リリースの使用状況を示すモバイルアプリのセッション数
  </Collapser>

  <Collapser
    id="adoption"
    title="ユーザーの重要な採用指標"
  >
    異なるバージョンを比較して、ユーザーにどれだけ採用されたかを確認することができます。

    * **総ユーザー数：** アクティブなユニークユーザー（デバイス）の数

    * **Upgrades:** 以前のアプリバージョンからアップグレードしたアクティブなユニークユーザー（デバイス）の数

    * **新規インストール数：** このアプリの新規インストールを行ったアクティブなユニークユーザー（デバイス）の数

      アップグレードと新規インストールが組み合わされ、総ユニークインストール数という形で採用が明らかになります。
  </Collapser>

  <Collapser
    id="kti"
    title="主な技術指標"
  >
    主な技術的パフォーマンス指標には、安定性、ネットワークの健全性、他のアプリやサーバーの問題、メモリ使用量などがあります。

    * 安定性を追跡するために、UIには、クラッシュした全アプリセッションの割合として、 **Crash rate** が表示されます。
    * ネットワークの問題がアプリ、ネットワーク、サーバーのどれに起因しているかを理解するために、UIにはHTTPエラーコードを返したサーバー応答の割合（**HTTP errors** ）と、応答を受信できなかったネットワークコールの割合（**Network failure** ）が表示されます。
    * アプリやサーバーからの他の問題を特定するために、UIにはアプリのセッションごとに行われたネットワークリクエストの平均数(**Requests per session**)と、アプリの視点から見たサーバーの平均 **Response time** が表示されます。
    * **Memory** の使用状況の改善点や問題点を比較するために、最初の4つのバージョンには、前のバージョンのパフォーマンスよりも良い（緑）または悪い（赤）ことを示すメトリクスが含まれています。
  </Collapser>
</CollapserGroup>

## トレンドデータの照会 [#insights]

[NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) で問い合わせる際には、バージョンの傾向情報を分析するのに役立つ2つの属性を提供しています。

* セッションに新規インストールや新規アップグレードが含まれているかどうかを追跡するには、 `install` を使用します。この属性は、新規インストールについて `true` を記録します。
* アップグレードが検出されたときにモバイルアプリの最後のバージョンを追跡するには、 `upgradeFrom` を使用します。

これらの属性を使用するには、当社のAndroid用エージェント [、またはiOS用エージェント ](/docs/release-notes/mobile-release-notes/android-release-notes/)[、](/docs/release-notes/mobile-release-notes/ios-release-notes/) のバージョンがサポートしていることを確認してください。
