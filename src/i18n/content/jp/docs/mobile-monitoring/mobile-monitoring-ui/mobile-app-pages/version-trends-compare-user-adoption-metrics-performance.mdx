モバイルアプリの状態を理解することで、チームは新しいバージョンごとに、よりスマートで効率的な選択を行うことができます。モバイルアプリの変更を追跡するために、NewRelicはモバイルモニタリングUIでバージョントレンドデータを提供します。ここでは、バージョントレンドレポートを使用して、モバイルアプリの最新バージョン全体のパフォーマンスを比較できます。各バージョンの改善、修正、および劣化の影響を分析するために必要な主要なデータポイントにアクセスできます。使用状況、採用、および技術的な指標を比較できます。

## 要件

モバイルアプリのバージョントレンド情報を収集するには、エージェントのバージョンを確認してください。

* [Androidエージェントv6.5.0](/docs/release-notes/mobile-release-notes/android-release-notes/)以降用の新しいRelicモバイルエージェント
* [iOSエージェントv7.0.0以降用](/docs/release-notes/mobile-release-notes/ios-release-notes/)の新しいRelicモバイルエージェント

## バージョントレンドを表示 [#view-version-trends]

バージョン トレンド レポートを表示するには、 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > モバイル > (アプリを選択) > アプリ > バージョン トレンド**に移動します。

## バージョントレンドの詳細を理解する [#details]

モバイルアプリの新しいバージョンを起動した後、バージョントレンドレポートを使用して、パフォーマンスとユーザーの採用を比較できます。クラッシュ率やHTTPエラーなどの主要な指標を通じて、アプリの状態を測定することもできます。

<CollapserGroup>
  <Collapser
    id="adoption"
    title="ユーザーの重要な採用指標"
  >
    さまざまなバージョンを比較して、ユーザーの採用を測定できます。

    * **アプリの起動：**ユーザーがアプリケーションを起動した回数のカウント
    * **総ユーザー数：**固有のアクティブなデバイスの数
    * **アップグレード：**以前のアプリバージョンからのアップグレードであった一意のアクティブなデバイスの数
    * **新規インストール：**このアプリの新規インストールを実行した一意のアクティブなデバイスの数
  </Collapser>

  <Collapser
    id="kti"
    title="主要な技術的指標"
  >
    これらの指標は、アプリの状態を測定するのに役立ちます。

    * **クラッシュ率：クラッシュ**したすべてのアプリセッションの割合
    * **HTTPエラー：** HTTPエラーは、ステータスコードが400〜499のHTTPリクエストです。
    * **ネットワーク障害：**ネットワーク障害はHTTPリクエストであり、応答がありません
    * **リクエスト：**アプリセッションごとに行われたネットワークリクエストの平均数
    * **応答時間：**アプリの観点から見たサーバーの平均応答時間
    * **メモリ使用量：**アクティブデバイス上のアプリが占有するメモリ
  </Collapser>
</CollapserGroup>

## トレンドデータの照会 [#insights]

[NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) で問い合わせる際には、バージョンの傾向情報を分析するのに役立つ2つの属性を提供しています。

* セッションに新規インストールまたは新規アップグレードがいつ含まれるかを追跡するには、 `install`を使用します。この属性は、新規インストールの`true`を記録します。
* アップグレードが検出されたときにモバイル アプリの最新バージョンを追跡するには、 `upgradeFrom`を使用します。

これらの属性を使用するには、当社のAndroid用エージェント [、またはiOS用エージェント ](/docs/release-notes/mobile-release-notes/android-release-notes/)[、](/docs/release-notes/mobile-release-notes/ios-release-notes/) のバージョンがサポートしていることを確認してください。
