**Summary**\[概要] ページには、モバイル アプリの操作のスナップショットが表示されます。すべてのグラフ、表、および統計には、現在選択されている [時間枠](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) とアプリケーション バージョンのデータが表示されます。

## アプリの主要指標 [#dsahboard-metrics]

\[概要]ページには、次の5つの主要なアプリ指標が表示されます。

<table>
  <thead>
    <tr>
      <th width={200}>
        メトリック
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        **アプリケーションのクラッシュ率**
      </td>

      <td>
        クラッシュしたセッションの数を、アプリのバージョン別に、全セッション数に対する割合として時間軸上にプロットします。右上には、平均的なクラッシュセッションの割合、クラッシュの総数、期間中にクラッシュの影響を受けたユニークユーザーの数が表示されます。
      </td>
    </tr>

    <tr>
      <td>
        **アプリ発売**
      </td>

      <td>
        期間中にモニターされたアプリのセッション起動数をグラフ化しています。

        New Relic では、モバイルセッションを、アプリが画面に表示されたときに始まり、アプリがバックグラウンドに送られたときに終わると定義しています。
      </td>
    </tr>

    <tr>
      <td>
        **HTTPエラー／ネットワーク障害**
      </td>

      <td>
        **http status code error** （400以上）となったネットワークリクエストの数を、完了したリクエスト全体に占める割合として、また、 **失敗したネットワークコール** （接続失敗など）の数を、完了したリクエストに占める割合としてグラフ化しています。
      </td>
    </tr>

    <tr>
      <td>
        **HTTPレスポンスタイム**
      </td>

      <td>
        アプリが通信する上位5つのホストからのすべての完了したhttpリクエストの平均応答時間をグラフ化します。

        トップ5は、各ホストへの完了したリクエストの数に基づいて計算されます。
      </td>
    </tr>

    <tr>
      <td>
        **頻繁に行われる交流**
      </td>

      <td>
        アプリ内で最もよく実行される5つのインターアクションのそれぞれについて、主要なパフォーマンス指標を提示します。
      </td>
    </tr>
  </tbody>
</table>

## サマリーページの表示 [#summary-page]

モバイルアプリの概要ページを表示するには：

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > モバイル > (アプリを選択) に移動**します。
2. モバイルアプリの他のページを表示するには、\[概要]ページまたは\[モバイル]メニューからリンクを選択します。

## 推奨アラート条件を追加 [#recommended-alert-conditions]

モバイルの **Summary** \[概要] ページを通じて、推奨されるアラート条件をエンティティに追加できます。

さらに未検出のエンティティを確認したい場合は [、「アラート カバレッジ ギャップと条件の推奨事項」](/docs/alerts-applied-intelligence/applied-intelligence/alert-coverage-gaps/) ページを参照してください。

推奨されるアラート条件を追加するには、次の手順に従います。

1. モバイル監視の **Summary**\[概要] ページを開きます。

2. **Add recommended conditions**\[推奨条件を追加]をクリックします。

   <img
     width="80%;"
     style={{align: "left"}}
     title="Mobile summary page"
     alt="Screenshot of the New Relic mobile summary page"
     src={mobileAlertConditions}
   />


3. 推奨されるアラート条件をいくつか選択します。

   <img
     width="70%;"
     style={{align: "left"}}
     title="Mobile recommended alert conditions"
     alt="Mobile recommended alert conditions"
     src={mobileAddAlert}
   />

4. **Next**\[次へ]をクリックします。

5. 通知を受け取るポリシーを選択します。既存のものを選択するか、 [新しいものを作成でき](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name)ます。

6. **Activate conditions**\[条件を有効にする]をクリックします。
