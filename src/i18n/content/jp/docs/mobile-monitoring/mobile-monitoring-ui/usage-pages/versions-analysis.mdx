モバイルモニタリングでは、アプリケーションのバージョン間での採用状況やパフォーマンスの比較分析が行われ、上位のバージョンは以下の通りです。

* インタラクションタイム
* アクティブセッション
* エラー率

**Versions** ページには、各バージョンの作成日、平均メモリ、平均CPU、1分あたりの平均セッション、アクティブなアプリごとの1分あたりの平均リクエスト（RPM）を比較した表も掲載されています。また、特定のバージョンに関する詳細情報を掘り下げることもできます。

  **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> 使用状況> バージョン:** **バージョン** の分析では、モバイルアプリの使用状況を色分けしたグラフに加え、モバイルアプリのバージョンと、メモリ、CPU、アクティブユーザー、ネットワークRPM（1分あたりのリクエスト数）の平均値をまとめた表を掲載しています。
</figcaption>

**Versions** ページでは、検出されたアプリのすべてのバージョンのリストに加えて、過去7日間にアクティブだったすべてのバージョンの概要情報が表示されます。比較分析を見るには

1. Go to **[one.newrelic.com](https://one.newrelic.com) > モバイル> （アプリを選択）> 使い方> バージョン**.
2. 期間を選択するには、New Relic のメニューバーの下にある [**タイムピッカー**](/docs/site/timepicker-setting-time-periods-to-view-data) を使用します。
3. オプションです。 **Sort by** のオプションを選択します。
4. 特定の **バージョン** の詳細のみを表示するには、その名前を選択します。

**Versions** ページでは、選択したタイムウィンドウでアクティブになっていたアプリのすべてのバージョンのリストが表示されます。詳細な情報にドリルダウンするには、New Relic の標準的な [ユーザーインターフェース関数](/docs/accounts-partnerships/education/getting-started-new-relic/new-relic-user-interface) と [ページ関数](/docs/accounts-partnerships/education/getting-started-new-relic/standard-dashboard-features) を使用します。

## ドリルダウンの詳細表示 [#details]

<img
  title="screen versions details.png"
  alt="screen versions details.png"
  src={screenVersionsDetails0}
/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Mobile> （アプリを選択）> Usage> Versions> （選択したバージョン）：** 選択したバージョンの詳細の例です。
</figcaption>

詳細ページでは、選択されたバージョンが参照バージョン（最近のバージョンや人気のあるバージョン）や、アプリの他のバージョンの平均値と比較して、さらに詳しい情報が表示されます。時系列では、エラーレート、応答時間、アクティブセッション、メモリ使用量の比較が表示されます。

詳細ページを終了するには、 **閉じる** (**X**) ボタンを選択してください。

## バージョン番号の取得方法 [#how-obtain-version]

モバイルモニタリングがバージョン番号を取得する方法は、プラットフォームによって異なります。

<CollapserGroup>
  <Collapser
    id="android-ver-num"
    title="Android"
  >
    [Androidエージェント](/docs/mobile-monitoring/new-relic-mobile-android/get-started/android-installation-configuration) は、マニフェスト内の `android:versionName` プロパティからバージョン情報を取得します。
  </Collapser>

  <Collapser
    id="ios-ver-num"
    title="iOS"
  >
    iOS エージェントは、 `CFBundleShortVersionString` と `CFBundleVersion` の両方のプロパティを使用して、アプリのバージョンを取得します。エージェントは、iOS の API を通じてこれらのプロパティにアクセスします。 `info.plist` ファイルを読んで取得するのではありません。
  </Collapser>
</CollapserGroup>
