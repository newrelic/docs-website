モバイルモニタリングでは、アプリケーションのバージョン間での採用状況やパフォーマンスの比較分析が行われ、上位のバージョンは以下の通りです。

* インタラクションタイム
* アクティブセッション
* エラー率

**Versions** ページには、各バージョンの作成日、平均メモリ、平均CPU、1分あたりの平均セッション、アクティブなアプリごとの1分あたりの平均リクエスト（RPM）を比較した表も掲載されています。また、特定のバージョンに関する詳細情報を掘り下げることもできます。

</figcaption>

**Versions** ページでは、検出されたアプリのすべてのバージョンのリストに加えて、過去7日間にアクティブだったすべてのバージョンの概要情報が表示されます。比較分析を見るには

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (アプリを選択) > Usage > Versions**に移動します。
2. 期間を選択するには、New Relic メニュー バーの下にある [**time picker**](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)\[タイム ピッカー] を使用します。
3. オプションです。 **Sort by** のオプションを選択します。
4. 特定の **バージョン** の詳細のみを表示するには、その名前を選択します。

**Versions**\[バージョン] ページには、選択した時間枠でアクティブだったアプリのすべてのバージョンのリストが表示されます。詳細情報にドリルダウンするには、該当する [プラットフォーム UI 機能](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)を使用します。

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (アプリを選択) > Usage > Versions > (選択したバージョン):** 選択したバージョンの詳細の例を次に示します。
</figcaption>

詳細ページでは、選択されたバージョンが参照バージョン（最近のバージョンや人気のあるバージョン）や、アプリの他のバージョンの平均値と比較して、さらに詳しい情報が表示されます。時系列では、エラーレート、応答時間、アクティブセッション、メモリ使用量の比較が表示されます。

詳細ページを終了するには、 **閉じる** (**X**) ボタンを選択してください。

## バージョン番号の取得方法 [#how-obtain-version]

モバイルモニタリングがバージョン番号を取得する方法は、プラットフォームによって異なります。

<CollapserGroup>
  <Collapser
    id="android-ver-num"
    title="Android"
  >
    [Android エージェント](/docs/mobile-monitoring/new-relic-mobile-android/get-started/android-installation-configuration)は、マニフェストの`android:versionName`プロパティからバージョン情報を取得します。
  </Collapser>

  <Collapser
    id="ios-ver-num"
    title="iOS"
  >
    iOS エージェントは、 `CFBundleShortVersionString`と`CFBundleVersion`の両方のプロパティを使用してアプリのバージョンを取得します。エージェントは、iOS API を介してこれらのプロパティにアクセスします。`info.plist`ファイルを読み取って取得するわけではありません。
  </Collapser>
</CollapserGroup>
