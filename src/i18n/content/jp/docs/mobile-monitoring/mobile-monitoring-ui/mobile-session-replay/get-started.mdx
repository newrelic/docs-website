セッションリプレイは、エンドユーザーがスマートフォンやタブレットで見た内容のビデオ再生を、そのセッションのすべてのテレメトリーデータと同期して提供することで、トラブルシューティングにおける推測を排除します。つまり、バグレポートやローカルでの再現に依存することなく、ユーザーに対するエラーの実際の影響を迅速に評価し、パフォーマンスのボトルネックやUXのフリクションポイントを特定して、問題をより迅速に解決できます。

セッションリプレイを最大限に活用するには、以下のセクションを参照してください：

* [セッションリプレイの概要](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/introduction): セッションリプレイの仕組み、保存されるデータ、データ保持に与える影響について説明します。
* [セッションリプレイの設定](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/set-up-session-replay): 要件を確認し、セッションリプレイを有効にして、サンプリングレートを設定します。
* [プライバシー設定の構成](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/configure-privacy-settings): マスキングとデータ保護のオプションをカスタマイズします。
* [New Relicでセッションリプレイを表示](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/view-session-replays): エラーにリンクされたリプレイを含む、記録されたセッションにアクセスして分析します。
* [セッションリプレイの管理と機能の変更](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/manage-session-replay): リプレイを表示し、設定を変更できるユーザーを制御します。
* [高度なセッションリプレイ](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/advanced-session-replay): AIサマリー、シーケンスフィルター、手動記録API、およびNerdGraph設定を使用します。
* [セッションリプレイのトラブルシューティング](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/troubleshooting): 一般的な問題の解決策を見つけます。
* [追加情報](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-session-replay/additional-information)：パフォーマンスへの影響、データの使用状況、プライバシーコンプライアンス、サポートされているフレームワークについて説明します。
