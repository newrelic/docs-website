の<InlinePopover type="mobile"/>**crash event trail** \[クラッシュ イベント トレイルには]、サブスクリプション レベルの[データ保持ポリシー](/docs/accounts-partnerships/accounts/account-billing-usage/data-retention)に基づいて、モバイル アプリのクラッシュに至るまでのイベントが表示されます。これらは、New Relic がデフォルトで監視するイベント、またはカスタム イベントです。この文書では、[クラッシュ イベント トレイルとは何か](#what-is)、および[その使用方法について](#use)説明します。

## モバイルアプリがクラッシュする前にイベントを表示 [#what-is]

モバイルアプリケーションがクラッシュし、その原因がわからないとき、クラッシュの直前に何が起こったかを調べることができます。クラッシュイベントトレイルでは、これらのイベントが表示されるので、クラッシュに至るまでの"パンくず" をたどり、障害の原因を診断することができます。

クラッシュ イベント トレイルには、クラッシュに至るまでのすべてのモバイル [イベント タイプ](/docs/insights/explore-add-data/custom-events/insert-query-custom-mobile-app-events-attributes#event-definition) が表示されます。[Record ブレッドクラム SDK](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-breadcrumb/) を使用して、クラッシュの診断に役立つと思われるアプリのアクティビティを追跡するカスタム `MobileBreadcrumb` イベントを作成できます。

クラッシュ イベント トレイルの `MobileHandledException` イベントを使用して、デバッグを支援することもできます。iOS および Android の [`recordHandledException`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/) SDK メソッドを使用して、アプリケーション内で例外が処理される場所に注釈を付けます。これらのイベントは、クラッシュ イベント証跡に自動的に表示されます。

カスタムデータによるクラッシュイベントトレイルのアノテーションについては、 [モバイルモニタリングにカスタムデータを追加する](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile) を参照してください。

## イベントトレイルを利用する [#use]

クラッシュイベントの軌跡を利用するために

1. **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (モバイル アプリを選択) > Crash analysis**に移動します。
2. **Crash analysis** ページの右下で、クラッシュタイプを選択します。
3. **\[クラッシュの詳細]** ページで、スタック トレースの下にある **\[イベント トレイル]**を選択します。
4. クラッシュタイプに至るまでの出来事を調べて、クラッシュの理由を知る手がかりとする。
5. イベントの属性の詳細を表示するには、そのイベントを選択します。
6. イベントトレイルの結果をNew Relicで表示するには、 **Insightsでセッションを開く** を選択します。
7. 同じクラッシュタイプの発生箇所をスクロールするには、イベントトレイルの左右の矢印を使います。

衝突分析ツールを最大限に活用してください。

* カスタムの [`MobileBreadcrumb`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-breadcrumb/) または [`MobileHandledException`](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/) イベントを作成するためのモバイル SDK
