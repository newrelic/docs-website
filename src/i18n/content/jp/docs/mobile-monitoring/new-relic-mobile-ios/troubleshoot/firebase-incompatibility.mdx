## 問題

競合、クラッシュ、またはエラーは、New Relic iOS エージェントと、Firebase などの他のサードパーティ テレメトリ データ トラッキング エージェントの両方がアプリ内に含まれている場合に表示される可能性があります。

既知のエラー：

* インストルメント化された関数でクラッシュします。
* クラッシュ ログのキャプチャに失敗しました。

## 解決

当社の iOS エージェントと、Firebase などの他のサードパーティ テレメトリ データ トラッキング エージェントを同時に有効にすることはできません。この問題を回避するには、どちらか一方のみを有効にすることをお勧めします。

他の SDK を削除するオプションがない場合は、これらの競合を回避するために使用できるいくつかの手順があります。

* 他のサードパーティの SDK を開始する前に、弊社のエージェントを開始してください。
* エージェントの機能の一部を無効にします。これがうまく機能するオプションの場合、詳細については [API ガイドを](/docs/mobile-monitoring/new-relic-mobile-ios/ios-sdk-api/ios-agent-configuration-feature-flags) 参照してください。

さらにサポートが必要な場合は、 [support.newrelic.com](https://support.newrelic.com) でサポートを受けてください。
