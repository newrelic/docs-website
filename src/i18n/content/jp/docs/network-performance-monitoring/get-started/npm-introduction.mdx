システムのパフォーマンスが低下した場合、それがコード、インフラストラクチャ、または基盤となるネットワークのいずれが原因であるかを知る必要があります。そして、あなたはあなたがあなたの努力に集中することができるように、速く知る必要があります。ネットワーク監視を使用すると、アプリケーション、インフラストラクチャ、デジタルエクスペリエンス、およびネットワークデータをすべて1つの場所で相互に関連付けて分析し、ネットワークパフォーマンスとシステム全体のパフォーマンスが相互にどのように影響するかを理解できます。

<Callout variant="tip">
  ネットワーク監視が[NewRelicFedRAMPのモデレート認証](/docs/security/security-privacy/compliance/data-encryption/)に含まれるようになりました。

  [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#setup-snmp-monitoring) 、 [ネットワークフロー](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#setup-network-flow-monitoring)、および[ネットワークsyslog](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#setup-network-syslog-monitoring)のガイド付きインストールを使用して、 [FedRAMP認定のAPIエンドポイント](/docs/security/security-privacy/compliance/fedramp-compliant-endpoints/#data-ingest-apis)を自動的に検出および構成するか、 [SNMP](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring/#manual-setup) 、 [ネットワークフロー](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring/#manual-setup)、および[ネットワークsyslog](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring/#manual-setup)のドキュメントにある手動セットアッププロセスを確認してください。
</Callout>

ネットワークモニタリングは、 [NewRelic](https://one.newrelic.com)ですでに収集しているアプリケーションおよびインフラストラクチャデータにネットワークデータのコンテキストを追加します。ネットワークデータを監視することにより、次のことができます。

* スタック全体（アプリケーションとインフラ）のパフォーマンスを分析・把握することで、システムのパフォーマンスを全体的に把握することができます。
* すべてのデータを単一のプラットフォームに集約し、死角をなくす。
* ある問題にネットワークが関係しているかどうかが一目瞭然。

## ネットワークデータの種類 [#network-data-types]

次の種類のネットワークデータを監視できます。

* **SNMPを介したデバイスのパフォーマンス**：SNMP（Simple Network Management Protocol）は、ネットワークデバイス間で管理情報を交換するためのアプリケーション層プロトコルです。 SNMPデータを[NewRelic](https://one.newrelic.com)に送信するには、 [SNMPデータ監視を設定します](/docs/network-performance-monitoring/setup-performance-monitoring/snmp-performance-monitoring)。
* **ネットワークsyslog** ：何が変更されたか、エラー、およびその他のデバイスイベントに関する洞察を提供して、「なぜ」をすばやく理解します。パフォーマンステレメトリの変更の背後にあります。ネットワークsyslogデータをに送信するには、 [ネットワークsyslogモニタリングを設定します](/docs/network-performance-monitoring/setup-performance-monitoring/network-syslog-monitoring)。
* **クラウドフローログ**：送信元と宛先のIPアドレスとポート、両方のエンドポイントを説明するメタデータ、主要なネットワークパフォーマンステレメトリ、および特定の通信試行の状態をキャプチャすることにより、ホストと他のサービス間の通信の詳細を示します。[クラウドフローログモニタリングを設定します](/docs/network-performance-monitoring/setup-performance-monitoring/aws-vpc-flow-monitoring)。

<Callout title="ベータ機能">
  クラウドフローログはまだベータ版です。
</Callout>

* **ネットワークフローログ**：オンプレミスネットワークのネットワークインターフェイスとの間で送受信されるIPトラフィックに関する情報をキャプチャします。ネットワークフローデータを[NewRelic](https://one.newrelic.com)に送信するには、[ネットワークフローデータモニタリングを設定します](/docs/network-performance-monitoring/setup-performance-monitoring/network-flow-monitoring)。

<Callout variant="important">
  ネットワークの可視性を高めるために、SNMPとネットワークフローデータの両方を設定することをお勧めします。これにより、パフォーマンス指標とトラフィックパターンの両方が提供され、ネットワークのトラブルシューティングと最適化が可能になります。
</Callout>
