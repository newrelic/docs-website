開発プロセスの早い段階で、より頻繁に開発者にコードについて話し合うようにすると、チームに大きな成果がもたらされます。会話により、コードのパフォーマンスと品質が向上し、開発者の満足度が高まります。New Relic CodeStream を使用すると、コードについての議論がこれまでより簡単になります。

質問、提案、ドキュメント、バグレポートのいずれであっても、コードを選択して**Add comment** \[コメントを追加]または**Create issue** \[問題を作成]をクリックし、必要に応じて Slack または Teams に共有します。これらのディスカッションはすべて保存され、参照するコード ブロックに固定されるため、将来的に活用できます。それは、チームに加わった新しい開発者かもしれないし、他の人のコードのバグを修正しようとしている開発者かもしれないし、あるいは、6 か月前にその変更を行った理由を思い出そうとしているあなただけかもしれません。いずれの場合でも、CodeStream は文脈に沿った方法でディスカッションを表面化することで、コードを理解するのに役立ちます。

時間の経過とともにファイルが変更されても、コメントはコードに関連付けられたままになります。コード ブロックの上に新しいコード行を追加したり、コードを編集したり、ブロック全体をファイルの別のセクションにカット アンド ペーストしたりすると、変更に応じてコメントが移動するのがわかります。

## コメントまたは問題の作成 [#create]

エディターでコードのブロックを選択し、選択内容の横にある CodeStream ペインで**Add comment** \[コメントの追加]アイコンまたは**Create issue** \[問題の作成]アイコンをクリックします。

IntelliJ などの JetBrains IDE を使用している場合は、コードのブロックを選択したときにエディターのガターに表示される**+**ボタンをクリックします。

CodeStream ペインが閉じている場合や表示されていない場合でも、電球またはコンテキスト メニューの CodeStream オプションを使用してコードについて議論できます。CodeStream ペインの上部にある**+**メニューを探すこともできます。

## Slack または Microsoft Teams への共有 [#sharing]

CodeStream には[Slack 統合](/docs/codestream/code-discussion/slack-integration)と[Microsoft Teams 統合の](/docs/codestream/code-discussion/msteams-integration)両方があり、IDE で多くの時間を費やしていないチームメイトやまだ CodeStream を使用していないチームメイトと簡単に連絡を取ることができます。Slack との統合により、チームメイトが Slack から直接返信することもできます。

## 問題の作成 [#issues]

Slack や Teams との共有に加えて、CodeStream は多くの問題追跡サービスとも統合します。チームが Asana、Azure DevOps、Bitbucket (クラウド)、GitHub (クラウドまたはオンプレミス)、GitLab (クラウドまたはセルフマネージド)、Jira (クラウドまたはサーバー)、Linear、Shortcut、Trello、または YouTrack (クラウド) を使用している場合問題を追跡するために、CodeStream から直接これらのサービスの 1 つで問題を作成できます。問題を作成するときに、フォームの上部にあるドロップダウンから使用するサービスを選択するだけです。

## 適切な人を議論に巻き込む [#mentions]

コメントまたは問題を作成すると、CodeStream は、コメントしているコードに最近触れたユーザーを自動的にメンションします。彼らはあなたの質問に答えるのに最適な人かもしれませんが、もちろん、必要に応じてそれらのメンションを削除し、手動で他の人にメンションすることもできます。

## 異なるバージョンのコードを扱う [#code-versions]

機能ブランチを使用しているか、ローカルに変更があるか、あるいは単にしばらくプルしていない可能性があります。あなたが見ているコードがチームメイトが見ているコードと異なる理由は無数にあります。その結果、ディスカッションで参照されているコードがローカルのコードと一致しないことがよくあります。

CodeStream はこれらの状況を認識し、コード ブロックの元のバージョン、現在のバージョン、および差分を含めます。

<img
  title="version diff"
  alt="A screenshot showing a code diff."
  src={codestreamCodemarkWithCodeDiff}
/>

CodeStreamを使えば、どんなソースファイルでも、どんなコードでも、いつでも、たとえそれがエディタに入力したばかりで、まだコミットしていないコードであっても、議論することができることに留意してください。CodeStreamは、開発プロセスの最も早い段階からコードについて議論することを可能にします。
