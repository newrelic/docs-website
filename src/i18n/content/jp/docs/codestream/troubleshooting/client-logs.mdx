New Relic カスタマー サポートに問題を報告する場合、ログ ファイルを含めると役立つことがよくあります。 ログファイルを取得するには、IDE の次の手順に従ってください。

## ジェットブレインズ [#jetbrains]

1. 問題を再現する。
2. IDE で<DNT>**Help &gt; Collect Logs and Diagnostic Data**</DNT>に移動します。 これにより、ファインダーが開き、新しく作成された zip ファイルが表示されます。

## VSコード [#vscode]

1. 問題を再現する。
2. VS Code で<DNT>**Output**</DNT>ビュー (コマンド パレットの<DNT>**View: Toggle Output**</DNT> ) を開き、右上のドロップダウン メニューから<DNT>**CodeStream (Agent)**</DNT>を選択します。 表示される出力をすべてコピーし、テキスト ファイルに保存します。

## Visual Studio [#visual-studio]

1. <DNT>**Tools &gt; Options &gt; CodeStream**</DNT>に移動して、ログレベルが少なくとも`Debug`に設定されていることを確認してください。 `Info` 、 `Errors` 、または`Silent`に設定されている場合は、 `Debug`に変更して Visual Studio を再起動します。
2. 問題を再現する。
3. `%localappdata%\CodeStream\Logs` } から`vs-2022-extension.log`と`vs-2022-agent.log`を取得し、さらに`%AppData%\Microsoft\VisualStudio\17.0_<RandomText>` `ActivityLog.xml`を取得します。
