[GitHub リポジトリ](https://github.com/TeamCodeStream/CodeStream/issues)でバグ レポートを送信する際に、ログ ファイルも含めていただけると大変助かります。IDE の次の手順に従って、ログ ファイルを取得します。ログファイルを GitHub イシューに添付するか、必要に応じて[codestream@newrelic.com](mailto:codestream@newrelic.com)に電子メールで送信し、件名で GitHub イシュー番号を参照することができます。

## ジェットブレインズ [#jetbrains]

1. 問題を再現する。
2. IDEで **Help> Collect Logs and Diagnostic Data** を開きます。これでファインダーが開き、新しく作成されたZIPファイルが表示されます。

## VSコード [#vscode]

1. 問題を再現する。
2. VS Codeで **Output** ビューを開き(**View:Toggle Output** from the command palette) を開き、右上のドロップダウンメニューから **CodeStream (Agent)** を選択します。表示されている出力をすべてコピーして、テキストファイルに保存します。

## Visual Studio [#visual-studio]

1. **\[ツール] > \[オプション] > \[CodeStream]**に移動し、ログ レベルが少なくとも`Debug`に設定されていることを確認します。`Info` 、 `Errors` 、または`Silent`に設定されている場合は、 `Debug`に変更してから、Visual Studio を再起動してください。
2. 問題を再現する。
3. `%localappdata%\CodeStream\Logs` } から`vs-extension.log`と`vs-agent.log`を取得し、 `%AppData%\Microsoft\VisualStudio\16.0_<RandomText>` (Visual Studio 2019 の場合) または`%AppData%\Microsoft\VisualStudio\17.0_<RandomText>` (Visual Studio 2022 の場合) から`ActivityLog.xml`を取得します。
