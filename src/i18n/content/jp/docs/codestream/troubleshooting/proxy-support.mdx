New Relic CodeStream エクステンションの接続性に問題がある場合、ネットワークプロキシを使用していて、CodeStream がプロキシと連携するように設定されていないことが原因である可能性があります。お使いの IDE で CodeStream の設定を見つけるには。

* VSコードです。 **設定** にアクセスし、CodeStream を検索します。
* Visual Studioを使用しています。 **「ツール」> 「オプション」> 「コードストリーム」** に移動します。
* JetBrains **Settings/Preferences> Tools> CodeStream** に移動します。

**Proxy Support** の設定を探して、 **on** に設定されていることを確認してください。また、CodeStream拡張機能がSSL/TLS証明書が正当な認証局からのものであると認識しないため、厳格なSSLチェックを無効にする必要があります。

これらの設定をVS Codeで見ると、以下のようになります。

<img
  alt="Proxy Settings"
  src={codestreamVscodeProxySettings}
/>

IDEでプロキシサポートが設定されている場合、CodeStreamはまずその設定を引き継ごうとします。そうでない場合、CodeStreamはオペレーティングシステム/環境のプロキシ設定を継承します。

設定変更後は、IDEを再起動してください。
