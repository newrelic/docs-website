CodeStreamコードから構築されたサービスで新しい[ 問題レベルの異常](/docs/codestream/observability/transactions) が検出された場合、 電子メールで通知します。これらの通知を管理するには、ユーザー名メニューの下にある<DNT>**Notifications**</DNT>設定に移動します。 どのリポジトリとサービスが通知をトリガーするかをカスタマイズしたり、通知を完全に無効にしたりできます。

デフォルトでは、CodeStream は IDE で開いたリポジトリを自動的に追跡します。 または、CodeStream ツリー ビューでリポジトリにマウスを移動し、 **Follow**\[フォロー]アイコンをクリックして、リポジトリを手動でフォローすることもできます。

<img title="Manually following a repository." alt="A screenshot of the icon for following a repository." src="/images/codestream_screenshot-crop_follow-repo.webp" />

<DNT>**Notification Settings**</DNT>ページにアクセスし、現在フォローしているリポジトリの右側にある**X** をクリックすると、リポジトリのフォローを解除できます。

デフォルトでは、フォローしているリポジトリに関連付けられているすべてのサービスで異常が発生した場合に通知が届きます。 通知を特定のサービスに限定するには、サービス名 (たとえば、「prod」を含む名前)、アカウント ID、またはキーの値のペア (例: `environment:production` ) でフィルターできます。
