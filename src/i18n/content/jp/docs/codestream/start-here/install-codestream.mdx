New Relic CodeStreamは、より自然で状況に応じた方法でコードについて話し合い、レビューすることを容易にする開発者コラボレーションプラットフォームです。New Relicに接続したら、IDEで直接アプリケーションエラーを共同編集します。

New Relic CodeStreamを最大限に活用できるように、まだ登録していない場合[は、無料のNewRelicアカウントにサインアップして](https://newrelic.com/signup)ください。

## CodeStreamをインストールします [#install]

特定のIDE用にCodeStreamをインストールすることも、Instant Observability（I / O）クイックスタートからインストールすることもできます。

## ツールを接続する [#connect-tools]

GitHub、GitLab、またはBitbucketでプルリクエストを作成して確認します。Jira、Trello、およびその他の課題追跡システムで課題を作成します。NewRelicに報告されたエラーを調査します。SlackまたはMicrosoftTeamsでコードディスカッションを共有します。CodeStreamは、毎日使用するツールをIDEにまとめます。

CodeStreamペインの上部にあるヘッドショットをクリックし、\[**統合**]をクリックして、すべてのツールをCodeStreamに接続します。

<Callout variant="tip">
  CodeStreamをインストールしたら、New Relicに接続するには、NewRelic[ユーザーキー](https://one.newrelic.com/launcher/api-keys-ui.api-keys-launcher)が必要になります。[ユーザーキーの検索または作成の詳細については、こちらをご覧ください](/docs/apis/intro-apis/new-relic-api-keys/#user-api-key)。
</Callout>

## いつでも、コードのブロックについて話し合う [#discuss-code]

他の人のコードを理解しようとしている場合でも、今書いたコードのヘルプを取得しようとしている場合でも、コードを選択し、コメントボタンをクリックして質問してください。

<img
  title="Some selected code with the CodeStream add comment button. Use this to discuss and collaborate on code in your repository."
  alt="A screenshot of some selected code with the CodeStream add comment button."
  src={qsDiscussCode1}
/>

[コードの説明について詳しくは、こちらをご覧ください。](/docs/codestream/how-use-codestream/discuss-code)

## 進行中の作業に関するフィードバックを取得する [#get-feedback]

\[ **+** ]メニューをクリックし、開発サイクルの任意の時点で\[**フィードバックのリクエスト**]をクリックします。進行中の作業（コミットされていないコードも含む）をすばやく確認する場合でも、完了した作業を正式にレビューする場合でも同じです。

<img
  title="The Request Feedback highlighted in the + Create menu."
  alt="A screenshot of the Request Feedback highlighted in the + Create menu."
  src={qsFeedbackRequest2}
/>

チームメイトは、ブランチを切り替えたり、自分の作業を脇に置いたりすることなく、IDEで変更を確認できます。[フィードバックリクエストの詳細をご覧ください。](/docs/codestream/how-use-codestream/feedback-requests)

## プルリクエストを作成または確認する [#pull-request]

CodeStreamサイドバーで、\[**プルリクエスト**]セクションを探して、開いているプルリクエストを確認します。プルリクエストを選択して（またはURLからロードして）、IDEで完全なGitHubエクスペリエンスを取得します。

<img
  title="A GitHub pull request in CodeStream."
  alt="A screenshot of a GitHub pull request in CodeStream."
  src={qsPullRequest}
/>

プルリクエストはGitHub、GitLab、またはBitbucketで作成できますが、プルリクエストのレビューのサポートは現在GitHubでのみ利用できます。[プルリクエストの詳細をご覧ください。](/docs/codestream/how-use-codestream/pull-requests/)
