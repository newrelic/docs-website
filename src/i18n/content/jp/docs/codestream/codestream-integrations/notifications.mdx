New Relic CodeStream は、あなたがフォローしたコメント、課題、およびフィードバック要求を通知します。通知を受ける方法は、電子メール、デスクトップ (VS Code および JetBrains エクステンションの場合のみ) 、またはその両方を選択できます。CodeStream ペインの上部にあるユーザープロファイルメニューで、通知オプションを探してください。

## 通知設定 [#settings]

デフォルトでは、あなたが作成した、あなたが言及された（元の投稿またはその後の返信で）、またはあなたが返信したコメント、課題、またはフィードバックリクエストを自動的にフォローするように設定されています。

個々のコメント、課題、フィードバックリクエストのフォロー/アンフォローは、楕円形のメニューからいつでも選択できます。

## フォローまたはアンフォロー [#follow]

メールでの通知はすぐに送信されます。メールに返信することで、議論に参加することができます。あなたの返信は、適切なコメント、課題、またはフィードバックリクエストに対する返信として、CodeStreamに追加されます。返信する際には、通知が送信されたのと同じメールアドレス（つまり、CodeStreamの「 **My Organization** 」に登録されているメールアドレス）から返信するようにしてください。

![メールの通知を解除するイメージです。](/images/UnfollowViaEmail.png "メール") <figcaption>コードマークやコードレビューのフォローを解除するには、メールの下部にあるリンクをクリックします。</figcaption>

## デスクトップ通知 [#desktop]

VS CodeやJetBrainsのIDEでCodeStreamを使用している場合、フォローしているコメント、課題、フィードバックのリクエストについて、IDEでデスクトップ通知を受け取ることもできます。

![デスクトップの通知を表示するイメージです。](/images/DesktopNotification2.png "デスクトップ通知")

<figcaption>
  **開く** ボタンをクリックすると、ディスカッションが開かれ、参加することができます。
</figcaption>

## その他の通知 [#other]

CodeStreamでは、以下のような通知機能があり、ページ下部のチェックボックスでオン/オフを切り替えることができます。

* **Notify me about outstanding feedback requests**: 自分に割り当てられた未解決のフィードバック要求のうち、過去24時間以内に回答していないものについて、リマインダーメールを受け取ることができます。
* **Notify me about new unreviewed commits from teammates when I pull**: いつプルしても、現在のブランチにチームメイトの新しいコミットがあると、トースト通知が届きます。 **Review** ボタンをクリックすると、チームメイトの変更点のレビューを開始し、フィードバックを行うことができます。
* **自分の活動をまとめたメールを毎週送信**: 毎週月曜日に、前週のあなたとあなたの組織の活動に関する情報を送信。

CodeStreamの [プルリクエスト統合](/docs/codestream/how-use-codestream/pull-requests) を活用するためにGitHubまたはGitHub Enterpriseに接続している場合、プルリクエストが自分に割り当てられたり、レビューアとして追加されたりしたときにも通知されます。

![プルリクエストの通知を示す画像です。](/images/PullRequestNotification.png "プルリクエスト")

<figcaption>
  **Open** ボタンをクリックすると、プルリクエストがIDE上で開かれ、変更内容を確認することができます。
</figcaption>
