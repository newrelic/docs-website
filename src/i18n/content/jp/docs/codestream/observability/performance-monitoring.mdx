New Relic CodeStreamを使用すると、開発者は、IDEに可観測性をもたらし、それを日常業務の一部にすることで、担当するコードが本番環境でどのように実行されているかを確認できます。

CodeStream のすべては、IDE で開いているリポジトリのコンテキストに依存します。New Relic 上のどのサービスがこのリポジトリから構築されているかを CodeStream が認識していない場合は、最初に[リポジトリをサービスに関連付ける](/docs/codestream/observability/repo-association)ように求められる場合があります。それ以外の場合は、コードから構築されたサービスがどのように実行されているかを包括的に把握できます。

サービスごとに、次のパフォーマンス データが表示される場合があります。

* **Golden metrics \[ゴールデンメトリクス]:**サービスのパフォーマンスを示す重要なシグナル。 メトリクスはサービスの種類によって異なります。 (APM、 browser 、モバイル、OTel、Lambda関数)
* **Issues \[問題]:**サービスに設定されたアラート条件に基づく未解決の問題。 (APMのみ)
* **Service-level objectives \[サービスレベル目標]:**設定されたサービスレベル目標に対してサービスがどのように実行されているかの概要。 (APM、 browser 、OTel)
* **Code-level metrics \[コードレベルのメトリクス]:**エディター内のパフォーマンス データなど、コード内のどのメソッドのパフォーマンスが低下しているかを確認します。 [コードレベルのメトリクスについて詳しくは、こちらをご覧ください。](/docs/codestream/observability/code-level-metrics) (APMのみ)
* **Errors \[エラー]:**あなたが担当しているコードで最近発生したエラー。 エラーをクリックしてスタックトレースを進み、チームメイトと協力して調査します。 [エラー調査の詳細については、こちらをご覧ください。](/docs/codestream/observability/error-investigation) (APM、 browser )
* **Vulnerabilities \[脆弱性]:**パッケージ レベルの脆弱性。修復方法に関する情報が含まれます。 (APMのみ)
* **Related services \[関連サービス]:**サービスを呼び出す、またはサービスによって呼び出されるすべてのサービス。各サービスのゴールデン メトリクスを表示できます。 サービスが全体像にどのように適合するかを理解します。 (APM、 browser 、モバイル、OTel)

ツリー内のサービス名の上にマウスを置くと、地球をクリックして New Relic のサービスの概要ページを開きます。

CodeStream の可観測性機能を使用するには、New Relic ユーザー タイプが[コア ユーザーまたはフル プラットフォームで](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/#user-type-capabilities)あるか、New Relic の計算価格モデルを使用している必要があることに注意してください。
