New Relic CodeStream issueは、外部のコードホスティングや課題追跡サービスからの課題が表示されます。

## お客様のサービスとCodeStreamの接続 [#connect]

まだ外部サービスに接続していない場合、このセクションでは、利用可能でサポートされているすべてのサービスが一覧表示されます。

* アサナ
* Azure DevOps
* Bitbucket（クラウド）
* クラブハウス
* GitHub（クラウドまたはエンタープライズ）
* GitLab（クラウドまたはセルフマネージド）
* Jira（クラウドまたはサーバー）
* リニア
* Trello
* YouTrack（クラウド）

## 課題の管理 [#manage-issues]

チームの課題追跡サービスやコードホスティングサービスに接続すると、自分に割り当てられたすべての課題が課題セクションに表示されます。

![課題部門](/images/IssuesSection.png)

あなたの組織では、複数の外部サービスを同時に利用することができます。ドロップダウンリストから利用したいものを選択してください。

![プロバイダーの選択](/images/Tasks-MultipleProviders.png)

課題をクリックすると、その作業を開始できます。機能ブランチを作成したり、チケットのステータスを更新したり、Slackでステータスを更新することもできます。

![仕事を始める](/images/IssuesStartWork.png)

課題の行にカーソルを合わせると、行の最後に課題追跡サービスでその課題を閲覧するためのオプションが表示されます。

多くのサービスでは、リストをフィルタリングすることもできます。例えば、Trelloに接続している場合、特定のリストやリストのセットを見るためにフィルタリングすることができます。

![フィルターの問題](/images/IssuesFilter.png)

Jira、Jira Server、GitHub、GitHub Enterprise、GitLab、GitLab Self-managedでは、カスタムフィルターを作成することもできます。

[GitHub と GitHub Enterprise](/docs/codestream/codestream-ui-overview/pull-requests-section/#github) または [GitLab と GitLab Self-Managed](/docs/codestream/codestream-ui-overview/pull-requests-section/#gitlab) のカスタムクエリを作成する際には、いくつかの特別なガイドラインがあります。

![カスタマーフィルター](/images/IssuesCustomFilter.png)

セクションの見出しにカーソルを合わせると、より多くのオプションが表示されます。

![課題ヘッダー](/images/IssuesSectionHeader.png)

更新ボタンをクリックすると、最近追加されたチケットがリストに反映されます。

**New issue** （選択したサービスによって表示が異なる場合があります）をクリックすると、CodeStreamから課題追跡サービスに課題を作成できます。そのチケットをエディタのコードブロックと関連付けることもできます。

![新しいカード](/images/IssuesNewTrello.png)

関連するチケットがないものを作業する必要がある場合は、 **Start Ad-hoc Work** をクリックすると、チケットなしで作業を開始できます。

![アドホック・ワーク](/images/IssuesStartAdHoc.png)
