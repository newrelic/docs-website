New Relic CodeStreamのパーマリンクは、Slackやメールなどのメッセージングサービスを利用して、誰とでも共有することができます。

## どこでも共有できるパーマリンク [#permalinks]

CodeStreamにはSlackやMicrosoft Teams、JiraやTrelloなどの課題追跡サービスに共有するための仕組みが組み込まれていますが、コメントや課題、フィードバックリクエストのリンクを取得して、他の場所で共有することもできます。既存のJiraチケット。Wiki。メッセージングサービスのスレッド。どのような場合でも、楕円形のメニューの下にある **Copy link** というオプションを探してください。

![リンクのコピーオプションを示すスクリーンショット](/images/CodemarkMenu-CopyLink.png "コピー・リンク")

誰かがこのリンクをクリックすると、CodeStreamで認証された後、自動的にウェブを経由して自分のIDEにリダイレクトされ、そこで自動的にディスカッションが開かれます。コメントや問題がある場合は、ソースファイルが開き、対応するコードブロックにスクロールします。

CodeStreamは、どのIDEを開くかを決定するために最善を尽くしますが、主に最後にCodeStreamにアクセスしたときに使用したIDEに基づいています。しかし、いつでも開いた Web ページに戻り、使用したい IDE を選択することができます。選択した内容は、次回同じリポジトリに関連したディスカッションを開いたときに記憶されます。

JetBrainsのIDEを使用している場合は、CodeStreamがIDEにディープリンクできるように、 [Toolbox app](https://www.jetbrains.com/toolbox-app/) をインストールしてください。

## コードブロックのパーマリンク [#code-blocks]

時には、コードブロックにコメントをしたいとは限らず、チームメイトとコードを簡単に共有したいこともあるでしょう。CodeStreamのパーマリンクは、たとえコードがファイル内の別の場所に移動したとしても、常に問題のコードを指し示す生きたリンクです。

エディタでコードを選択し、 **Get Permalink** ボタンをクリックすると、そのコードのパーマリンクが取得できます。

![パーマリンクの作成方法を示した画面](/images/GetPermalink1.png "パーマリンク取得")

パーマリンクをパブリックにするか、プライベートにするかを決めます。プライベートのパーマリンクでは、コードを表示する前にCodeStreamの認証が必要になります。

![プライベート/非プライベートの切り替えを示すスクリーンショット](/images/NewPermalink2.png "プライベートトグル")

パーマリンクをコピーして、コードのブロックを共有したい場所に貼り付けてください。

コメントや課題、フィードバックリクエストと同様に、パーマリンクをクリックすると、自動的にウェブ上のIDEにリダイレクトされ、ソースファイルが開かれ、対応するコードブロックまでスクロールされます。
