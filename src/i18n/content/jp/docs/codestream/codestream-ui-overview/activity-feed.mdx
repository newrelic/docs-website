アクティビティフィードは、チームメイトが投稿した新しいコメント、課題、フィードバックリクエストや、既存のディスカッションへの新しい返信などを知ることができる決定的な場所です。アクティビティフィードを表示していないときでも、アクティビティフィードアイコンのバッジを見れば、新しい情報があるかどうかがわかります。青色のバッジに白い点がついている場合は、新しいディスカッションや返信があることを意味します。

![この画像は、アイコンにアクティビティバッジをつけたものです。](/images/ActivityFeedWithBadge3.png "バッジ")

<figcaption>
  数字の入った青いバッジは、あなたが言及された新しいディスカッションや返信があることを意味します。
</figcaption>

IDEのステータスバーに表示されるCodeStreamのセクションでは、フィードに新しいメッセージがあるかどうかを知ることができます。

![この画像は、IDE のステータスバーに CodeStream が表示される様子を示しています。](/images/StatusBarWithMentions2.png "id-status") <figcaption>ユーザーネームの右にあるドットは新しいメッセージがあることを、数字は新しいメンションがあることを意味します。</figcaption>

アクティビティフィードの上部にあるフィルターは、フィードに表示される内容をコントロールします。

* **自分の組織の全員からのアクティビティ**: 自分の組織からのすべてのアクティビティを見ることができます。小規模な組織にのみお勧めします。
* **自分のIDEで開いているコードに関連するアクティビティ（デフォルト）**: 自分のIDEで開いているリポジトリのコードに関連するディスカッションを見る。
* **Activity associated with code in selected folder**: チームがmonorepos内で作業している場合、このオプションを使用します。現在選択されているフォルダ内のコードに関連するディスカッションを見ることができます。

フィルターの設定にかかわらず、あなたが@mentionedされたときには常に通知されます。

予想通り、新しいものはフィードの一番上に追加されます。それが非常に古い議論に対する新しい返信であってもです。CodeStreamでは、新しいアクティビティの左側に青いボーダーを追加して、フラグを立てています。

![この画像は、フィードの先頭に新しいコメントが表示されています。](/images/ActivityFeed-NewCodemark.png "新しいコメント") <figcaption>この新しいコメントは、フィードのトップに表示されます。</figcaption>

ここには、コメントに対する新しい返信があります。コメントはすでに見られているので、返信だけが青い枠で表示されています。

![この画像は、フィード内の新しい返信を示しています](/images/ActivityFeed-NewOldReplies.png "新着情報") <figcaption>青色の枠線は、会話のどの部分が新しいかを強調しています。</figcaption>

**See 1 earlier reply** は、すでに見たことのある以前の返信があることを意味しています。メインのアクティビティフィードビューには、新しい未読の返信のみが表示されますが、このリンクをクリックすると、ディスカッションスレッド全体を見ることができ、新しい返信を追加することができます。

![画像はコメントを拡大したものです](/images/CodemarkViewWithReplies1.png "拡大コメント") <figcaption>コメントを展開すると、返信の全文脈が表示されます。</figcaption>

アクティビティフィードは、チームの時間の始まりまでさかのぼりますが、特定の古いコードマークを探している場合は、 [フィルターや検索](/docs/codestream/codestream-ui-overview/filter-search) ツールを使用するとよいでしょう。
