SAP ソリューション向けNew Relic監視は、SAP テレメトリー データ (メトリクス、イベント、ログ、およびトレース (MELT)) をキャプチャし、分析と視覚化のためにNew Relic One プラットフォームに送信します。

## 中央監視システム vs. 監視システム

テレメトリーデータを提供する SAP システムはモニター システムとして知られています。 セントラル システムは、NRM4SAP がインストールされている SAP システムです。テレメトリーデータを収集、変換、送信します。 通常、NRM4SAP は、SAP Solution Manager (SolMon) などの単一の SAP システムにインストールします。ST\_PI、CCMS、パフォーマンスなどの標準 SAP データ ソースを利用するため、モニター システムに追加のデータベース エージェントを導入する必要がありません。

## データフローアーキテクチャー

データは、RFC 呼び出しを使用して SAP テレメトリーデータを中央システムに取り込むことで効率的に管理されます。 その後、データは ABAP Enterprise Connector を介して処理され、HTTPS 経由で New Relic One TDP プラットフォームに安全に送信されます。

このアーキテクチャーにより、データ処理の合理化が保証され、SAP システムのパフォーマンスの可視性が強化され、効果的な監視と管理のための重要なインサイトがエンジニアに提供されます。

## 使用事例

SAP システムは大企業にとって不可欠であり、財務、物流、資材管理、倉庫保管、販売、流通などの重要なビジネス プロセスのバックボーンとして機能します。 SAP ソリューション向けNew Relic監視 (NRM4SAP) を利用することで、企業はこれらの SAP 主導の運用を包括的に可視化し、シームレスに機能することを保証できます。

### 統合と問題解決

NRM4SAP を組織のインフラストラクチャに統合することは、電子商取引の注文処理や倉庫管理などの主要なビジネス オペレーションをサポートする SAP システムを監視するために不可欠です。この統合により次のことが可能になります。

**包括的な監視**: NRM4SAP は堅牢な監視機能を提供し、SAP システムが効率的に動作することを保証し、ビジネス プロセスに影響を与える前に潜在的な問題の検出と解決を促進します。

### 導入後と次のステップ

導入後、顧客はすぐに NRM4SAP が提供するすぐに使えるダッシュボードとアラートを活用でき、いくつかの利点が得られます。

* **強化された管理インサイト**: 一元化されたダッシュボードにより、SAP システムとその他の IT システムの健全性とパフォーマンスを可視化できるため、管理監視が効率化され、管理チェックへの依存が軽減されます。

* **カスタム監視の機会**: 顧客は、注文処理ワークフロー内の統合ポイントの追跡など、特定のビジネス ニーズに合わせて監視を調整できます。 収集したテレメトリーデータを使用してカスタムダッシュボードやビジュアライゼーションを開発できます。

* **プロアクティブなアラート**: 重要なアラートを設定すると、アドミニストレーターはサービスの中断について直ちに警告を受けることができ、継続性を維持し、問題があればすぐに対処できるようになります。

NRM4SAP は、企業が SAP 運用を最適化し、可視性を高め、SAP エコシステムの効果的な管理を容易にできるようにします。
