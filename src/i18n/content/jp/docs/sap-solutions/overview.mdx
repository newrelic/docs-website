SAPソリューション向けNew Relic監視は、SAPテレメトリーデータ（メトリクス、イベント、ログ、トレース）をキャプチャし、分析と可視化のためにNew Relicプラットフォームに送信します。

## 中央監視システム vs. モニター システム

テレメトリーデータを提供するSAPシステムは、モニターシステムと呼ばれます。中央監視システムは、<DNT>**NRM4SAP**</DNT>がインストールされる場所です — テレメトリーデータを収集、変換、送信します。通常、<DNT>**SAP Solution Manager (SolMon)**</DNT>などの単一のSAPシステムに<DNT>**NRM4SAP**</DNT>をインストールします。<DNT>**NRM4SAP**</DNT>は、<DNT>**ST\_PI**</DNT>、<DNT>**CCMS**</DNT>、およびパフォーマンス データベースなどの標準のSAPデータソースを利用するため、モニター対象システムでの追加のエージェント インストレーションが不要になります。

## データフローアーキテクチャー

<DNT>**RFC**</DNT> 呼び出しにより、SAPテレメトリーデータが中央監視システムに取り込まれます。<DNT>**ABAP Enterprise Connector**</DNT>はデータを処理し、<DNT>**HTTPS**</DNT>経由でNew Relicに安全に送信します。

<img title="Data Flow Architecture" alt="A diagram illustrating the data flow architecture" src="/images/SAP-Private-Cloud.webp" width="80%" />

## 含まれるもの

インストレーション後、すぐに使用できるダッシュボードとアラートを使用できます：

* **管理の可視性**: 一元化されたダッシュボードで、他のITシステムとともにSAPシステムの健全性とパフォーマンスを表示します。

* **カスタム監視**: 注文処理ワークフローにおけるインテグレーションポイントの追跡など、特定のビジネスニーズに合わせて監視を調整します。収集したテレメトリーデータを使用して、カスタムダッシュボードと可視化を作成します。

* **プロアクティブアラート**：サービス中断の即時通知を受け取るようにアラートを設定することで、継続性を維持し、問題に迅速に対処できるようになります。
