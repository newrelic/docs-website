## /NEWR/ABAP\_COLLECTION\_MANAGERNのバリアントを作成する

1. トランザクション**SE38**を使用して`/NEWR/ABAP_COLLECTION_MANAGERN`を設定します。
2. **ABAP: Variants - Initial Screen**\[ABAP: バリアント - 第一画面]で、**Program**\[プログラム]フィールドに`/NEWR/ABAP_COLLECTION_MANAGERN`入力します。
3. **Variant**\[バリアント]を選択し、**Create**\[作成]をクリックします。
4. 値を更新するには、以下の手順に従ってください。

<CollapserGroup>
  <Collapser id="trace-variant" title="トレースのバリエーション">
    1. バリアント名 (例: **TRACE\_ONLY**)を入力し、 **Create**\[作成]をクリックします。

    2. **New Relic Account ID**\[New Relic アカウント ID が]自動的に入力されていることを確認します。

    3. **送信あたりのレコード数**を**10000**に設定します。

    4. データ プロバイダー名を追加するには、 **Multiple Selection (Active)**\[複数選択 (アクティブ)]をクリックします。

    5. **Select Single Values**\[単一値の選択]に以下を追加します。

       * `NR.SAP.TRANSACTION`
       * `NR.SAP.TRANSACTION.COMPLETE`
       * `NR.SAP.SYSTEM.PROCESS`

    6. **Execution Mode**\[実行モード]として**Sequential Processing**\[順次処理]を選択します。

    7. **DIA WP** ≥ 30の場合は**6**に設定し、それ以外の場合は**3**に設定します。

    8. **Attributes**\[属性]をクリックし、説明を追加して、 **Save**\[保存]をクリックします。
  </Collapser>

  <Collapser id="idoc-variant" title="IDOCバリアント">
    1. バリアント名 (例: **IDOC\_ONLY**)を入力し、 **Create**\[作成] をクリックします。

    2. **New Relic Account ID**\[New Relic アカウント ID が]自動的に入力されていることを確認します。

    3. **送信あたりのレコード数**を**10000**に設定します。

    4. データ プロバイダー名を追加するには、 **Multiple Selection (Active)**\[複数選択 (アクティブ)]をクリックします。

    5. **Select Single Values**\[単一値の選択]に以下を追加します。

       * `NR_SAP_IDOC`
       * `NR_SAP_IDOC_COMPLETE`

    6. **Execution Mode**\[実行モード]として**Sequential Processing**\[順次処理]を選択します。

    7. **Attributes**\[属性]をクリックし、説明を追加して、 **Save**\[保存]をクリックします。
  </Collapser>

  <Collapser id="system-variant" title="SYSTEMバリアント">
    1. バリアント名 (例: **System**)を入力し、 **Create**\[作成]をクリックします。
    2. **New Relic Account ID**\[New Relic アカウント ID が]自動的に入力されていることを確認します。
    3. **送信あたりのレコード数**を**10000**に設定します。
    4. データ プロバイダー名を追加するには、 **Multiple Selection (Active)**\[複数選択 (アクティブ)]をクリックします。
    5. **Exclude Single Values**\[単一値の除外]に **TRACE** および **IDOC** プロバイダーからのバリアントを追加します。
    6. **実行モード**として**並列処理**を選択します。
    7. **DIA WP** ≥ 30の場合は**6**に設定し、それ以外の場合は**3**に設定します。
    8. **Attributes**\[属性]をクリックし、説明を追加して、 **Save**\[保存]をクリックします。
  </Collapser>
</CollapserGroup>

## 収集ジョブのスケジュール

1. トランザクション**SM36**を使用して 3 つのジョブをスケジュールします。
2. 各ジョブのジョブ名を入力し、**ジョブ クラス**を「C」に設定します。
3. **ステップ**をクリックし、プログラム名**RSBTONEJOB2**を入力して、各ジョブのバリアントを指定します。
4. **Start Condition**\[開始条件]をクリックし、開始日時を設定し、 **Periodic Job**\[定期ジョブ]をオンにして、 **Period Values**\[期間値] 、 **Other Period**\[その他の期間] の順にクリックし、分として**5** を入力します。
