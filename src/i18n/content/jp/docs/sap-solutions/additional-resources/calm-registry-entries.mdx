SAP Cloud ALMサービスの監視を有効にするには、CloudALMデータプロバイダーのレジストリエントリ（Call ID）を設定します。

## 概要

このユーティリティプログラムは、CloudALMデータプロバイダーの設定を追加、変更、削除、および自動生成する機能を提供します。

* <DNT>**Transaction Code**</DNT>： <DNT>**/NEWR/GEN\_DP\_CALLID**</DNT>
* <DNT>**Program Name**</DNT>： <DNT>**/NEWR/GENERATE\_CONFIG\_CALLID**</DNT>
* <DNT>**Configuration Table**</DNT>： <DNT>**/NEWR/CALLID\_CNF**</DNT>

## 前提条件

このユーティリティを実行する前に、データプロバイダー<DNT>**NR\_CALM\_LANDSCAPE**</DNT>がアクティブであり、データを収集していることを確認してください。

## セットアップ手順

<Steps>
  <Step>
    ## データプロバイダーとサービスタイプのマッピングを設定する

    1. トランザクション`/NEWR/GEN_DP_CALLID`を実行します。
    2. <DNT>**DP/CALM Services Type Maintenance Link**</DNT>をクリックします。
    3. 監視するサービスタイプを選択します。
    4. 各データプロバイダーを、SAP Cloud ALM で設定された適切な SAP システムまたはサービスにリンクします。
    5. 設定を保存します。

    <Callout variant="tip">
      <DNT>**Service type**</DNT> <DNT>**SAP Cloud ALM**</DNT>環境で<DNT>**SAP systems**</DNT>または<DNT>**services**</DNT>が有効になっていることを表します。
    </Callout>
  </Step>

  <Step>
    ## 新しいCall IDレコードを追加

    1. 選択画面で、設定するデータプロバイダーとサービスタイプを入力します。
    2. <DNT>**Add new (Call ID) records to Registry**</DNT>ラジオボタンを選択します。
    3. <DNT>**Execute**</DNT>をクリックします。
    4. システムは、<DNT>**Cloud ALM**</DNT>で設定されたすべての<DNT>**service type instances**</DNT>を表示します。それぞれが一意の<DNT>**LMS ID**</DNT>を持ちます。
    5. モニターするインスタンスを選択します。
    6. 不要なエントリを削除してください。
    7. <DNT>**Update and save configuration**</DNT>をクリックします。
  </Step>

  <Step>
    ## 設定を確認する

    1. トランザクション`/NEWR/REGISTRY`へ移動してください。
    2. <DNT>**data provider**</DNT>を選択します。
    3. <DNT>**Configure**</DNT>をクリックして、保存された<DNT>**Call ID entries**</DNT>を表示します。
    4. <DNT>**Test**</DNT>をクリックして設定を確認します。

    **期待される結果**: 接続テストが成功ステータスを示します。
  </Step>
</Steps>

## メンテナンスオプション

### 既存のエントリを更新する

既存の<DNT>**Call ID records**</DNT>を変更または削除する必要がある場合：

1. トランザクション`/NEWR/GEN_DP_CALLID`を実行します。
2. <DNT>**Copy/Change/Delete existing (Call ID) records in Registry**</DNT>を選択します。
3. <DNT>**Execute**</DNT>をクリックします。
4. 必要な変更を加えます。
5. 設定を保存します。

### すべてのエントリをクリア

すべての<DNT>**Call ID records**</DNT>を削除するには（初期セットアップやサービスパックのアップグレードのため）：

1. トランザクション`/NEWR/GEN_DP_CALLID`を実行します。
2. <DNT>**Backup &amp; delete all existing (Call ID) records in Registry**</DNT>を選択します。
3. <DNT>**Execute**</DNT>をクリックします。

<Callout variant="important">
  これにより、削除前にバックアップを作成します。このオプションは、初期セットアップ中、またはサービスパックのアップグレード中に指示された場合にのみ使用してください。
</Callout>

### サービスの定義を更新

<DNT>**service type**</DNT>または<DNT>**LMS ID**</DNT>が見つからない場合：

1. <DNT>**Retrieve new services (LMSID) setup in Cloud ALM**</DNT>をクリックします。
2. これにより、<DNT>**Cloud ALM account**</DNT>で設定された最新のサービスを取得します。
