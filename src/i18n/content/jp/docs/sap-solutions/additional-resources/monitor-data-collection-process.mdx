## 実行状態と送信ログ

**Data Provider Registry**\[データプロバイダレジストリ]（トランザクション`/NEWR/REGISTRY` ）から、<img src="/images/runtimestatus.webp" class="inline" style={{height: "40px", width: "120px"}} alt="RuntimeStatus icon" title="RuntimeStatus" />

そして<img src="/images/transmissionlog.webp" class="inline" style={{height: "40px", width: "120px"}} alt="transmissionlog" title="Transmissionlog.webp" />各データプロバイダーごとに。送信ログには、各送信のステータスと時刻が表示されます。

## ジョブのステータスとスプールを確認する

1. トランザクション`SM37`を使用して、スケジュールされたジョブのステータスを確認します。**ABAP Program Name**\[ABAP プログラム名]フィールドに`/NEWR/*`と入力し、**Execute**\[実行を]クリックします。
2. 実行時間に注意してください。異常に長い時間は潜在的な問題を示しています。
3. 重複するジョブがないか確認し、必要に応じてスケジュールを調整します。ジョブ スプールは各データ プロバイダーの結果を提供します。

## アプリケーションログを確認する

データ収集プロセスはアプリケーションログ (実行 SLG1) にログとして記録されます。 アプリケーションログ オブジェクト`/NEWR/DP`を使用して作成します。

## SAPデータ収集監視ダッシュボード

データ収集メトリクスも自己監視のためにNew Relicに送信されます。 すぐに使える[**SAP Data Collection Monitoring**\[SAP データ収集監視\]]()ダッシュボードを使用して、各データ プロバイダーのデータ収集プロセスを監視します。
