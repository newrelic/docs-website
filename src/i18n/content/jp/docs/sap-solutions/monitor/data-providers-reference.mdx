データプロバイダーは、<DNT>**SAP systems**</DNT>から監視データを収集し、New Relicに送信するコンポーネントです。このテクニカルリファレンスには、<DNT>**New Relic Monitoring for SAP Solutions**</DNT>（<DNT>**NRM4SAP**</DNT>）で利用可能なすべてのデータプロバイダーが記載されています。特定の監視グループを設定する際にこれを使用します。データプロバイダーは監視グループごとに編成されます。この監視グループは、[モニター対象システムを設定する](/docs/sap-solutions/install-and-configure/configure-new-relic-abap-add-on-package/#connect-systems)際に選択します。

## このリファレンスの対象者

次の場合にこのリファレンスを使用します：

* 〜の監視グループの構成 <DNT>**SAP systems**</DNT>
* データ収集に関する問題のトラブルシューティング
* 各監視グループによってどのようなデータが収集されるかを理解する
* データプロバイダー設定のカスタマイズ

<Callout variant="tip">
  これらの表には、詳細な技術仕様が含まれています。ほとんどのユーザーは、**DP\_NAME**列と**OUTPUT\_TYPE**列を参照するだけで済みます。

  このリファレンス内の表は、水平方向にスクロールしてすべての列を表示できます。STATUS 列に「S」のマークが付いているデータ プロバイダーは、デフォルトで停止されています。
</Callout>

## データプロバイダーの属性を理解する

各データ プロバイダーには次の技術的属性があります。

* **MANDT** : クライアント番号（通常は001）
* **DP\_NAME** : データプロバイダーの一意の識別子
* **OUTPUT\_TYPE**: テレメトリーデータの種類 (METRIC, EVENT, LOG, TRACE, CONFIG, BTP-LOG)
* **COMMUNICATOR**: データ収集の調整を担当する<DNT>**ABAP class**</DNT>（高度なトラブルシューティング用）
* **DATAPROVIDER**: データを取得する<DNT>**ABAP class**</DNT>（高度なトラブルシューティング用）
* **FINISHER**: データを処理して送信する<DNT>**ABAP class**</DNT>（高度なトラブルシューティング用）
* **STATUS**: ステータスインジケーター（S=一時停止中、空白=アクティブ）
* **DP\_GROUP** : 監視グループの割り当て
* **METRIC\_TYPE**: メトリクスの場合、集計タイプ（GAUGE、COUNT、SUMMARY）を示します
* **PERIOD** : 収集頻度の数値
* **PERIOD\_UNIT** : 収集の時間単位（MIN、HRS、DAY）
* **SKIP\_RT** : ランタイムフラグをスキップ (X = はい)
* **RESERVED**: 将来の使用のために予約済み
