このガイドでは、New Relic Monitoring for SAP Solutionsのデータ収集プロセスを監視する方法を説明します。複数の方法を使用して、データ収集が正しく機能していることを確認できます。

## 前提条件

* 適切なトランザクション権限を持つ<DNT>**SAP system**</DNT>へのアクセス
* <DNT>**New Relic Monitoring for SAP Solutions**</DNT> インストールと設定済み

## 実行状態と送信ログ

<DNT>**Data Provider Registry**</DNT>（トランザクション`/NEWR/REGISTRY`）から、各データプロバイダの<DNT>**Runtime Status**</DNT>および<DNT>**Transmission Log**</DNT>を確認できます。転送ログには、各転送のステータスと時刻が表示されます。

## ジョブのステータスとスプールを確認する

1. トランザクション`SM37`を使用して、スケジュールされたジョブのステータスを確認します。<DNT>**ABAP Program Name**</DNT>フィールドに`/NEWR/*`と入力し、<DNT>**Execute**</DNT>をクリックします。
2. 実行期間を確認してください。異常に長い実行時間は、潜在的な問題を示します。
3. 重複するジョブを確認し、必要に応じてスケジュールを調整してください。各データプロバイダーの結果を表示するには、ジョブの詳細からジョブスプールにアクセスします。

## アプリケーションログを確認する

データ収集プロセスはアプリケーションログ（トランザクション `SLG1`）に記録されます。<DNT>**Application Log Object**</DNT> `/NEWR/DP`でクエリを実行します。

## SAPデータ収集監視ダッシュボード

データ収集メトリクスも自己監視のためにNew Relicに送信されます。すぐに使える<DNT>**SAP Data Collection Monitoring**</DNT>ダッシュボードを使用して、各データ プロバイダーのデータ収集プロセスをモニターします。
