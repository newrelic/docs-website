## サーバー可用性データプロバイダーが動作していません

**症状**: `NR_SAP_INSTANCE_AVAILABILITY`および`NR_SAP_SERVER_LIST`データプロバイダーがデータの収集に失敗します。

**原因**: モニターサーバー上の`DEFAULT.PFL`プロファイルは、<DNT>**SAPCONTROL**</DNT>Webサービス用に構成されていません。

**解決**：

1. `DEFAULT.PFL`ファイルを開き、この行が存在するか確認します：

`service/protectedwebmethods = SDEFAULT`

2. 行がない場合は、次のパラメーターを追加します：

`service/protectedwebmethods = SDEFAULT -GetQueueStatistic -ABAPGetWPTable -EnqGetStatistic -GetProcessList -GetEnvironment -GetVersionInfo -GetSystemInstanceList`

3. <DNT>**SAP service**</DNT>を再起動：

```yaml
stopsap sapstartsrv
startsap sapstartsrv
```

4. 設定を確認します：

```yaml
sapcontrol -prot GSOAP_HTTP -nr 00 -function GetSystemInstanceList
```

**想定される結果**: コマンドはエラーなしでシステムインスタンスリストを返します。
