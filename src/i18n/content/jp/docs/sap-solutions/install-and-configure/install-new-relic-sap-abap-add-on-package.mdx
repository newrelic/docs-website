<DNT>**NRM4SAP**</DNT>のABAPコンポーネントはSAPアドオンとしてパッケージ化されており、<DNT>**client 000**</DNT>でトランザクション`SAINT`を使用してインストールできます。

## アドオンインストレーションファイル

* <DNT>**Initial installation**</DNT>：ローカルPCのフォルダーにアドオンファイル`K-xxxAOIINNEWR.SAR`をダウンロードします
* <DNT>**Upgrade from previous version**</DNT>：次の名前のファイルを使用します `K-xxxAOUINNEWR.SAP`

## SAPアドオンファイルをインストールする

1. ログイン <DNT>**client 000**</DNT>
2. <DNT>**SAP GUI**</DNT>コマンドフィールドに`SAINT`と入力します
3. に移動します, <DNT>**Installation Package**</DNT> &gt; <DNT>**Load Packages**</DNT>
4. インストレーションファイルの場所に基づいて、適切な方法を選択します：

<DNT>**Which method should you use?**</DNT>

* <DNT>**SAR archive from frontend**</DNT> （最も一般的）：インストレーションファイルはローカルコンピュータにあります
* <DNT>**SAR archive from application server**</DNT>: SARファイルは次の場所に事前アップロードされています `/usr/sap/trans/download/`
* <DNT>**EPS files from application server**</DNT>：最初にサーバー上でSARファイルをPATファイルに抽出する必要があります

      1. ナビゲートして開く `K-xxxAOIINNEWR.SAR`

      3. <DNT>**File K-xxxAOIINNEWR.SAR successfully transferred, decompressed, and deleted**</DNT>というメッセージが表示されることを確認します
    </TabsPageItem>

    <TabsPageItem id="sar-archive-from-application-server">
      1. アプリケーションサーバー（`/usr/sap/trans/download`）の<DNT>**trans directory**</DNT>の下に<DNT>**download directory**</DNT>があることを確認します。
      2. このディレクトリに<DNT>**add-on file**</DNT>をコピーします。

    <TabsPageItem id="eps-files-from-application-server">
      1. <DNT>**SAR file**</DNT>を<DNT>**PAT files**</DNT>に抽出します。以下のコマンドを実行します：

      ```bash
      cd /usr/sap/trans
      cp /tmp/K-xxxAOIINNEWR.SAR /usr/sap/trans
      SAPCAR -xvf K-xxxAOIINNEWR.SAR
      ```

    </TabsPageItem>

## インストレーションを完了する

1. <DNT>**Start**</DNT>をクリックし、<DNT>**Installable Add-on packages for your system**</DNT>を選択して、クリックします <DNT>**Continue**</DNT>

2. <DNT>**Do you want to add Modification Adjustment Transports to the queue?**</DNT>を尋ねられた場合：

   * 標準インストレーション（デフォルト）の場合は<DNT>**No**</DNT>をクリックします
   * パッケージに特定の手順が記載された調整トランスポートが含まれている場合にのみ、<DNT>**Yes**</DNT>をクリックします。

   <img title="Do you want to add Modification Adjustment Transports to the queue?" alt="A screenshot displaying question about Modification Adjustment Transports" src="/images/modify-adjustement-transports.webp" width="80%" />

3. 続行するには、緑色のチェックボタンをクリックします

   <Callout variant="important">
     <DNT>**Errors during Disassembling OCS Packages**</DNT>ページが表示されたら、<DNT>**Ignore**</DNT>をクリックします。このエラーは特定のパッケージタイプで想定されており、インストレーションに影響を与えません。
   </Callout>

4. パッケージがインストールされたら、クリックします <DNT>**Finish**</DNT>

5. <DNT>**Sending the runtime analysis**</DNT>ポップアップが表示されます。次のうち1つを選択します。

   * <DNT>**Send**</DNT>: ローカルブラウザウィンドウで<DNT>**runtime analysis data**</DNT>のフィードバックフォームを開きます
   * <DNT>**Display**</DNT>: 最初に<DNT>**runtime analysis**</DNT>を表示し、その後フィードバックフォームを開きます
   * <DNT>**Do not send**</DNT>: 送信せずにダイアログを閉じます <DNT>**runtime analysis data**</DNT>

6. <DNT>**Installed Components**</DNT>タブでインストレーションを確認します。<DNT>**New Relic add-on**</DNT>は、インストレーションが成功したことを示すステータスでリストに表示されるはずです。

7. からログアウト <DNT>**Client 000**</DNT>
