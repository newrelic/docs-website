モニター対象の各システムについて、これらの設定タスクを完了してください。

## アップロード承認ロール [#upload-authorization-role]

1. <DNT>**SAP transaction**</DNT> `PFCG`を使用します。

2. <DNT>**Role**</DNT> &gt; <DNT>**Upload**</DNT> に移動し、インストレーション パッケージからファイル<DNT>**\_NEWR\_DATA\_PROVIDER.SAP**</DNT>をアップロードします。

3. ロール`/NEWR/DATA_PROVIDER`がロードされることを確認します。

   <img title="Authorization Role" alt="A screenshot displaying the authorization role upload" src="/images/authorisation-role.webp" width="600" />

4. <img src="/images/change-icon.webp" class="inline" style={{height: "22px", width: "22px"}} alt="change icon in SAP" title="Change icon indicator in SAP">を選択します。</img>

   ロールがロードされた後のアイコン。

5. <DNT>**Authorizations**</DNT>タブで、<img src="/images/change-icon.webp" class="inline" style={{height: "20px", width: "20px"}} alt="change icon in SAP" title="Change icon indicator in SAP" />

   <DNT>**Change Authorization Data**</DNT>、次を使用して有効化します<img src="/images/activate-option.webp" class="inline" style={{height: "25px", width: "30px"}} alt="activate icon in SAP" title="Activate Icon in SAP" />アイコン。

   <img title="Authorizations Tab" alt="A screenshot displaying the authorization tab and Change Authorization Data in SAP upload" src="/images/sap-authorization-1.webp" width="500" />

6. プロファイル名が提案された場合は、緑色のチェックアイコンを選択します。

7. ステータス バーに<DNT>**Profile(s) were updated**</DNT>が表示されていることを確認します。

## New Relic RFC ユーザー ID の作成 [#create-rfc-user-id]

<DNT>**authorization profile**</DNT>をアクティブ化した後、トランザクション`SU01`を使用して新しい<DNT>**RFC user ID**</DNT>（例：`NR4SAPDP`）を作成します。<DNT>**user type**</DNT>を<DNT>**System**</DNT>または<DNT>**Communication**</DNT>に設定し、ロール`/NEWR/DATA_PROVIDER`を追加します。

<img title="Create New RFC User" alt="A screenshot displaying the creation of a new RFC user in SAP" src="/images/rfc-userid.webp" width="600" />

## 監視システムの RFC 宛先を作成する [#create-rfc-for-monitored-systems]

モニターシステムごとにトランザクション`SM59`を使用して、<DNT>**RFC destination**</DNT>を作成します。

1. <DNT>**ABAP Connections**</DNT>を選択し、次に<DNT>**Create**</DNT>アイコンを選択して新しい<DNT>**RFC destination**</DNT>を設定します。

2. 分かりやすい<DNT>**RFC Destination**</DNT>名を入力します。以下をお勧めします： `NR_<SYSID>CLNT<CLNT#>`

3. <DNT>**Connection Type**</DNT>を`3 - ABAP Connection`に設定します。

4. <DNT>**Technical Settings**</DNT>タブで、次を入力します。

   * <DNT>**Target Host**</DNT> （完全修飾ドメインまたは IP）と <DNT>**Instance No.**</DNT>
   * <DNT>**Gateway Host**</DNT> 、必要に応じて<DNT>**Service**</DNT>。

   <img title="Create RFC destinations for monitored systems" alt="A screenshot displaying the how to create RFC destinations for monitored systems" src="/images/rfc-destination.webp" width="600" />

5. <DNT>**Logon &amp; Security**</DNT>タブで、<DNT>**RFC user**</DNT>（`NR4SAPDP`）とパスワードを入力します。

6. <DNT>**Save**</DNT>を選択します。

7. <DNT>**Utility &gt; Test &gt; Connection Test**</DNT>に移動し、次に<DNT>**Authorization Test**</DNT>に移動します。

   * <DNT>**Expected result**</DNT>: 両方のテストで成功ステータスが示されています。

## HTTP宛先をNew Relic cloudに設定する

トランザクション`SM59`を使用して、<DNT>**HTTPS connections**</DNT>を<DNT>**New Relic API endpoints**</DNT>に設定します。

<Callout variant="important">
  <DNT>**Events**</DNT>、<DNT>**Logs**</DNT>、<DNT>**Metrics**</DNT>、および<DNT>**Traces**</DNT>に対して、個別の<DNT>**RFC destinations**</DNT>を別々に作成します。
</Callout>

1. <DNT>**Destination Name**</DNT>を入力します (例: `NR_API_EVENT`)。

2. <DNT>**Connection Type**</DNT>を`G` (外部サーバーへの HTTP 接続) に設定します。

3. <DNT>**Target Host**</DNT>、<DNT>**Service No.**</DNT>に<DNT>**443**</DNT>(<DNT>**HTTPS**</DNT>の[標準ポート](/docs/sap-solutions/reference/network-accessibility)を参照)、および<DNT>**Path Prefix**</DNT>(詳細については、[インストレーション準備チェックリスト](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness)を参照)を入力します。

4. プロキシが必要な場合は、<DNT>**Technical Settings**</DNT>タブで設定します。

5. <DNT>**Security Options**</DNT>タブで、<DNT>**SSL**</DNT>の<DNT>**Active**</DNT>オプションを選択します。

   <img title="HTTP destination to New Relic" alt="A screenshot displaying the connection setup for HTTP destination to New Relic" src="/images/http-destination-to-new-relic.webp" width="600" />

6. 必要に応じて、[New Relic](https://one.newrelic.com)から<DNT>**root certificate**</DNT>をダウンロードし、トランザクション`STRUST`を使用して<DNT>**SAP**</DNT>に追加します。

7. <DNT>**Connection Test**</DNT>を選択して接続を確認します。

   * <DNT>**Expected result**</DNT>: 接続が成功すると、ログオンデータの入力を求めるプロンプトが表示されます。
   * <DNT>**If test times out**</DNT>: ファイアウォールの設定を確認してください。

8. すべての<DNT>**New Relic API endpoints**</DNT>に対してこの手順を繰り返します。

## BTP/CloudALM認証エンドポイントを作成する [#create-authentication-endpoint]

サービスに応じて以下の手順に従ってください。

  <TabsBar>

      <DNT>**BTP process integration runtime**</DNT>

      <DNT>**CloudALM API**</DNT>
    </TabsBarItem>
  </TabsBar>

  <TabsPages>
    <TabsPageItem id="btp-process-integration-runtime">
      1. ダウンロードした<DNT>**Service Key**</DNT>の`clientId`、`clientsecret`、`tokenurl`をメモします。
      2. Web ブラウザーで`tokenurl`にアクセスし、必要な証明書（<DNT>**Digicert root**</DNT>、<DNT>**global**</DNT>、および<DNT>**SAP on demand**</DNT>）をダウンロードします。

      <img title="BTP process integration runtime" alt="A screenshot displaying the connection setup for BTP process integration runtime" src="/images/btp-process-integration.webp" width="600" />

      3. トランザクション`STRUST`を使用して3つの証明書を1つずつアップロードし、次に<DNT>**Add to Certificate List**</DNT>を選択します。
      4. <DNT>**Save**</DNT>を選択します。
      5. トランザクション`SMICM`を使用して、<DNT>**Administration &gt; ICM &gt; Exit Soft &gt; Global**</DNT>に移動し、すべての<DNT>**ICM processes**</DNT>を再起動します。

      <img title="Administration ICM" alt="A screenshot displaying the Administration ICM" src="/images/administration-icm.webp" width="600" />

      6. トランザクション`SM59`を使用して、<DNT>**HTTP Connections to External Server**</DNT>の下に新しい<DNT>**RFC connection**</DNT>を作成します。
      7. <DNT>**tokenurl**</DNT>をホストとパスプレフィックスに分割し、パスプレフィックスに`?grant_type=client_credentials`を追加します。
      8. <DNT>**Service No.**</DNT>を`443`に設定します。

      <img title="Service No." alt="A screenshot displaying the Service No." src="/images/service-no.webp" width="600" />

      9. <DNT>**Login &amp; Security**</DNT>タブで<DNT>**Basic Authentication**</DNT>を選択し、ユーザー名とパスワードとして`clientId`と`clientsecret`を入力します。
      10. <DNT>**Secure Protocol**</DNT>のステータスを<DNT>**Active**</DNT>に設定し、設定を保存します。
      11. <DNT>**Connection Test**</DNT>を選択します。
    </TabsPageItem>

    <TabsPageItem id="cloud-alm-api">
      1. ダウンロードした<DNT>**Service Key**</DNT>の`clientId`、`clientsecret`、`url`をメモします。
      2. Web ブラウザーで<DNT>**url**</DNT>にアクセスし、<DNT>**Digicert root**</DNT>、<DNT>**global**</DNT>、および<DNT>**SAP on demand**</DNT>の証明書をダウンロードします。

      <img title="CloudALM API" alt="A screenshot displaying the connection setup for CloudALM API" src="/images/cloudalm-api.webp" width="600" />

      3. トランザクション`STRUST`を使用して3つの証明書を1つずつアップロードし、次に<DNT>**Add to Certificate List**</DNT>を選択します。
      4. <DNT>**Save**</DNT>を選択します。
      5. トランザクション`SMICM`を使用して、<DNT>**Administration &gt; ICM &gt; Exit Soft &gt; Global**</DNT>に移動し、すべての<DNT>**ICM processes**</DNT>を再起動します。

      <img title="Administration ICM" alt="A screenshot displaying the Administration ICM" src="/images/administration-icm.webp" width="600" />

      6. トランザクション`SM59`を使用して新しい<DNT>**RFC connection**</DNT>を作成します。
      7. <DNT>**url**</DNT>を<DNT>**Target host**</DNT>として入力し、<DNT>**Service No.**</DNT>を`443`に設定します。<DNT>**Path Prefix**</DNT>を`/oauth/token?grant_type=client_credentials`に設定します。

      <img title="SM59 Configuration" alt="A screenshot displaying the SM59 configuration" src="/images/sm59-configuration.webp" width="600" />

      8. <DNT>**Login &amp; Security**</DNT>タブで<DNT>**Basic Authentication**</DNT>を選択し、ユーザー名とパスワードとして`clientId`と`clientsecret`を入力します。
      9. <DNT>**Secure Protocol**</DNT>のステータスを<DNT>**Active**</DNT>に設定し、設定を保存します。
      10. <DNT>**Connection Test**</DNT>を選択します。
    </TabsPageItem>
  </TabsPages>

## データベース接続を作成する [#create-database-connection]

トランザクション`DBACOCKPIT`を使用して、リモート<DNT>**HANA Database connections**</DNT>を定義します。

1. <DNT>**Database Connections**</DNT>をダブルクリックし、<DNT>**Add**</DNT>を選択します。

   <img title="Database Connection" alt="A screenshot displaying the Database Connection" src="/images/database-connection.webp" width="600" />

2. <DNT>**Connection Name**</DNT>を入力します (<DNT>**SAP system ID**</DNT>と同じが望ましい)。

3. <DNT>**Database System**</DNT>として<DNT>**HANA Database**</DNT>を選択します。

4. <DNT>**Database user ID**</DNT>（例えば、`NRAGENT`）とパスワードを入力します。

5. <DNT>**Database Host**</DNT>と<DNT>**SQL Port**</DNT>を入力します。

   <img title="SAP Connection Name" alt="A screenshot displaying the SAP Connection Name" src="/images/connection-name.webp" width="600" />

6. <DNT>**Save**</DNT>を選択し、<DNT>**Test**</DNT>を選択して接続を確認します。

   * <DNT>**Expected result**</DNT>：接続テストが成功ステータスを示しています。
