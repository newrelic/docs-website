## /NEWR/INSTALLメニューを使用します。 [#menu-newr-install]

1. 中央監視システムの運用クライアントにログインします。
2. コマンドフィールドに`/n`と入力し、<DNT>**Enter**</DNT>を押して<DNT>**SAP Easy Access root menu**</DNT>に移動します。
3. コマンドフィールドに`/n/NEWR/INSTALL`と入力して、<DNT>**`/NEWR/INSTALL`**</DNT>メニューにアクセスします。

中央監視システムにアドオンをインストールした後：

* インストーラに設定を実行するための適切な権限があることを確認してください。
* インストーラーの`profile/role`内のコード認証オブジェクトにパラメーター`/NEWR/*`を追加します。

<Callout variant="tip">
  旧バージョンからアップグレードする場合は、[旧バージョンからのアップグレード](/docs/sap-solutions/upgrade/install-upgrade-package)を参照して、必要な手順を確認してください。
</Callout>

## システムを接続 [#connect-systems]

<Callout variant="important">
  先に進む前に、保留中の設定手順をすべて完了してください。
</Callout>

`/NEWR/INSTALL`メニューでは、以下のオプションが利用可能です：

### HTTPおよびRFCの送信先の設定

このオプションを使用してトランザクション`SM59`を呼び出し、<DNT>**RFC**</DNT>および<DNT>**HTTPS**</DNT>の接続を確認します。[「モニター システムの RFC の作成」](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems)および[「HTTP 宛先を New Relic cloud に設定する」](/docs/sap-solutions/install-and-configure/additional-configuration/#set-http-destination-to-new-relic-cloud)を参照してください。

### リモートデータベースへの接続のセットアップ

* SAPトランザクション`DBACOCKPIT`を使用して、リモート<DNT>**HANA databases**</DNT>への接続のセットアップを確認します。
* 監視するリモート HANA データベースごとにデータベース接続が作成されていることを確認します。
* 詳細な手順については、[データベース接続の作成](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)をご覧ください。

### NR APIエンドポイントへの接続の設定

1. <DNT>**New Entries**</DNT>をクリックします。

2. New Relicアカウントの詳細を入力してください：

   * <DNT>**New Relic Account ID**</DNT> （オプションでSAP-をプリフィックスとして付加）
   * <DNT>**API Ingest Key**</DNT>
   * <DNT>**Metric Endpoint**</DNT> （[ネットワークエンドポイント](/docs/sap-solutions/before-you-begin/#network-endpoints)を参照）
   * <DNT>**Event Endpoint**</DNT>
   * <DNT>**Log Endpoint**</DNT>
   * <DNT>**Trace Endpoint**</DNT>

3. <DNT>**Save**</DNT>をクリックします。

### クライアント000からテンプレートテーブルをコピー

実行アイコン（時計の記号）をクリックしてこのオプションを実行し、<DNT>**client 000**</DNT>から表をコピーします。成功メッセージが表示されることを確認します。

### モニターシステムを定義する

1. <DNT>**New Entries**</DNT>をクリックして、<DNT>**SAP System ID**</DNT>と<DNT>**Client**</DNT>の番号を入力します。
2. <DNT>**Data Provider Installation Information**</DNT>で、RFC <DNT>**Destination**</DNT>を指定し、SAP <DNT>**Software Component**</DNT>を選択し、<DNT>**Database Vendor**</DNT>を入力して、<DNT>**DB Connection Name**</DNT>(HANA データベースのみ) を指定します。
3. <DNT>**Monitoring Group**</DNT>をクリックし、ダイアログ ボックスで<DNT>**Yes**</DNT>を選択して確認します。
4. 必要な監視グループを選択し、生成アイコンをクリックしてプロセスを完了します。
5. 次の手順に進むには、<DNT>**BACK**</DNT>または<DNT>**F3**</DNT>キーをクリックします。

このメニューオプションを使用して、<DNT>**NONE**</DNT>を除くすべてのモニター システムおよびNR APIエンドポイントのRFC宛先をテストします。

## 設定を生成 [#generate-configurations]

### 新しいデータプロバイダー用の設定を生成

* このメニュー オプションを使用して、新しいリリースまたはパッチのデータ プロバイダーを生成します。新しいインストレーションの場合、すべてのデータ プロバイダー (DP) が生成されます。
* 最初の画面で、リリースとパッチの詳細を確認してください。空のフィールドは既存の設定を示します。
* 再生成するには、<DNT>**Maintain Monitored Systems**</DNT>で<DNT>**Re-generate Data Provider Configurations**</DNT>を選択するか、<DNT>**Reset Installation Flag (For Re-install only)**</DNT>を使用してインストレーションフラグをリセットし、設定を再生成します。

<img title="Maintain Monitored Systems" alt="A screenshot displaying the SAP maintain monitored systems" src="/images/maintain-monitored-systems.webp" width="500" />

### インストレーションの完全性を確認する

<DNT>**Verify Installation Completeness**</DNT>をクリックして、すべてが正しく構成されていることを確認します。いずれかの項目に赤いインジケーターが表示されている場合は、<DNT>**Action Needed**</DNT>列の実行ボタンをクリックします。

## 設定の確認 [#verify-configuration]

1. データプロバイダーが正しく設定されていることを確認するには、<DNT>**Data Provider Registry Maintenance**</DNT>フォルダーを開きます。
2. <DNT>**Data Provider Registry**</DNT>をクリックするか、トランザクション`/n/NEWR/REGISTRY`を使用します。
3. <DNT>**Data Provider Name**</DNT>を選択し、<DNT>**Configure**</DNT>をクリックして設定パラメーターを確認します。
4. <DNT>**TestDP**</DNT>をクリックしてデータプロバイダーをテストします。テストは正常に完了し、データプロバイダーが正しく構成されていることが確認されます。
