## RFCチェックプログラムのスケジュール [#rfc-check]

`/NEWR/REGISTRY_RFC_CHECK`を1～3分ごとにスケジュールし、<DNT>**data collection job**</DNT>よりも頻繁に実行されるようにします。ジョブ名`NR_RFC_CHECK`を使用します。

## ABAPデータ収集マネージャーのスケジュール [#abap-collection-manager]

以下のデータ プロバイダー グループを対象としたバリアントを使用して、3つのデータ収集ジョブをスケジュールします。`SP03`より前のコレクションマネージャージョブをスケジュールする手順については、[SP03より前のバリアントとジョブ](/docs/sap-solutions/reference/variants-and-jobs-before-sp03)を参照してください。スケジューリングのために、データプロバイダーを3つのグループに分けます。すべてのデータプロバイダーの完全なリストについては、「[データプロバイダー リファレンス](/docs/sap-solutions/monitor/data-providers-reference)」を参照してください。

* <DNT>**TRACE group**</DNT> パフォーマンスデータベース（STAD）関連のデータプロバイダーが含まれます：

  * [`NR.SAP.TRANSACTION`](/docs/sap-solutions/monitor/data-providers-reference/#sap-appl-metrics): トレースデータプロバイダー
  * [`NR.SAP.TRANSACTION.COMPLETE`](/docs/sap-solutions/monitor/data-providers-reference/#sap-appl-metrics): 並列バックグラウンドRFCプロセスによる完全なパフォーマンスデータベース収集
  * `NR.SAP.SYSTEM.PROCESS` および `NR_SAP_PROGRAM`：SP02後に一時停止
  * `NR_SAP_WORKLOAD_COMPLETE` および `NR.SAP.WORKLOAD.COMPLETE`

* <DNT>**IDOC group**</DNT> IDOC関連プロバイダーを含む（本番環境ではかなりの時間がかかる場合があります）：

  * [`NR_SAP_IDOC`](/docs/sap-solutions/monitor/data-providers-reference/#idoc-all): メインIDOCステータスイベントデータプロバイダー
  * [`NR_SAP_IDOC_COMPLETE`](/docs/sap-solutions/monitor/data-providers-reference/#idoc-all): バックグラウンドRFC収集による完全なIDOC取得
  * `NR_SAP_IDOC_Configuration`: パフォーマンスを改善するためにIDOCマスターデータをダウンロードする

* <DNT>**SYSTEM group**</DNT> 残りのすべてのデータプロバイダーが含まれます。[監視グループ別の完全なリスト](/docs/sap-solutions/monitor/data-providers-reference)を参照してください。

#### 各ジョブのスケジュールを設定する

1. トランザクション`SM36`を開始します。
2. 各ジョブの<DNT>**Job Name**</DNT>を入力し、<DNT>**Job Class**</DNT>を<DNT>**C**</DNT>に設定します。
3. <DNT>**Step**</DNT>をクリックし、<DNT>**Program Name**</DNT> `/NEWR/ONE_COLLECTION`を入力して、<DNT>**Variant**</DNT>を選択します。
4. <DNT>**Start Condition**</DNT>をクリックし、<DNT>**Start Date**</DNT>と<DNT>**Time**</DNT>を設定して、<DNT>**Periodic Job**</DNT>ボックスをオンにします。
5. <DNT>**Period Values**</DNT>に移動し、<DNT>**Other Period**</DNT>をクリックして、分に<DNT>**5**</DNT>と入力します。

## 再送信プログラムのスケジュール [#resubmit-program]

1. 伝送エラーが発生した場合に<DNT>**New Relic account**</DNT>の失敗した伝送を再送信するために、プログラム`/NEWR/RESEND_FAILED_API_TRANS`用に<DNT>**DEFAULT**</DNT>という名前の<DNT>**variant**</DNT>を作成します。

2. 次の値を指定します。

   * [<DNT>**New Relic Account ID**</DNT>](/docs/accounts/accounts-billing/account-setup/account-id)
   * <DNT>**HTTP Response Codes**</DNT>：`000`と`413`、および`200`～`299`の範囲を除外します。

   <img title="Resend failed API transactions" alt="A screenshot displaying the resend failed API transactions" src="/images/resend-failed-api-transaction.webp" width="700" />

3. <DNT>**HTTP response code**</DNT> `500`を伴うものなどの失敗した送信を再送信し、定期的に<DNT>**transmission log entries**</DNT>をパージするために、このジョブを15分ごとに実行するようにスケジュールします。
