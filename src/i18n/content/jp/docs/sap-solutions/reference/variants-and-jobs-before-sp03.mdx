## /NEWR/ABAP\_COLLECTION\_MANAGERNのバリアントを作成する

1. `/NEWR/ABAP_COLLECTION_MANAGERN`を設定するには、トランザクション`SE38`を使用します。
2. <DNT>**ABAP: Variants - Initial Screen**</DNT>で、<DNT>**Program**</DNT>フィールドに`/NEWR/ABAP_COLLECTION_MANAGERN`と入力します。
3. <DNT>**Variant**</DNT>を選択し、 <DNT>**Create**</DNT>をクリックします。
4. 値を更新するには、以下の手順に従ってください。

  <TabsBar>
    <TabsBarItem id="trace-variant">
      トレースのバリエーション
    </TabsBarItem>

    <TabsBarItem id="idoc-variant">
      IDOCバリアント
    </TabsBarItem>

    <TabsBarItem id="system-variant">
      SYSTEMバリアント

  </TabsBar>

  <TabsPages>
    <TabsPageItem id="trace-variant">
      1. バリアント名 (例: <DNT>**TRACE\_ONLY**</DNT>)を入力し、<DNT>**Create**</DNT>をクリックします。

      2. <DNT>**New Relic Account ID**</DNT>が自動的に入力されていることを確認します。

      3. <DNT>**Records per transmission**</DNT>を<DNT>**10000**</DNT>に設定します。

      4. データ プロバイダー名を追加するには、<DNT>**Multiple Selection (Active)**</DNT>をクリックします。

      5. <DNT>**Select Single Values**</DNT>に以下を追加します。

         * `NR.SAP.TRANSACTION`
         * `NR.SAP.TRANSACTION.COMPLETE`
         * `NR.SAP.SYSTEM.PROCESS`

      6. <DNT>**Execution Mode**</DNT>として<DNT>**Sequential Processing**</DNT>を選択します。

      7. <DNT>**DIA WP**</DNT> ≥ 30の場合、<DNT>**6**</DNT>に設定します；それ以外の場合は、<DNT>**3**</DNT>に設定します。

      8. <DNT>**Attributes**</DNT>をクリックし、説明を追加して、<DNT>**Save**</DNT>をクリックします。
    </TabsPageItem>

    <TabsPageItem id="idoc-variant">
      1. バリアント名 (例: <DNT>**IDOC\_ONLY**</DNT>)を入力し、<DNT>**Create**</DNT>をクリックします。

      2. <DNT>**New Relic Account ID**</DNT>が自動的に入力されていることを確認します。

      3. <DNT>**Records per transmission**</DNT>を<DNT>**10000**</DNT>に設定します。

      4. データ プロバイダー名を追加するには、<DNT>**Multiple Selection (Active)**</DNT>をクリックします。

      5. <DNT>**Select Single Values**</DNT>に以下を追加します。

         * `NR_SAP_IDOC`
         * `NR_SAP_IDOC_COMPLETE`

      6. <DNT>**Execution Mode**</DNT>として<DNT>**Sequential Processing**</DNT>を選択します。

      7. <DNT>**Attributes**</DNT>をクリックし、説明を追加して、<DNT>**Save**</DNT>をクリックします。
    </TabsPageItem>

    <TabsPageItem id="system-variant">
      1. バリアント名 (例: <DNT>**System**</DNT>)を入力し、<DNT>**Create**</DNT>をクリックします。
      2. <DNT>**New Relic Account ID**</DNT>が自動的に入力されていることを確認します。
      3. <DNT>**Records per transmission**</DNT>を<DNT>**10000**</DNT>に設定します。
      4. データ プロバイダー名を追加するには、<DNT>**Multiple Selection (Active)**</DNT>をクリックします。
      5. <DNT>**Exclude Single Values**</DNT>に<DNT>**TRACE**</DNT>および<DNT>**IDOC**</DNT>プロバイダーからのバリアントを追加します。
      6. <DNT>**Execution Mode**</DNT>として<DNT>**Parallel Processing**</DNT>を選択します。
      7. <DNT>**DIA WP**</DNT> ≥ 30の場合、<DNT>**6**</DNT>に設定します；それ以外の場合は、<DNT>**3**</DNT>に設定します。
      8. <DNT>**Attributes**</DNT>をクリックし、説明を追加して、<DNT>**Save**</DNT>をクリックします。
    </TabsPageItem>
  </TabsPages>
</Tabs>

## 収集ジョブのスケジュール

1. トランザクション`SM36`を使用して、3 つのジョブのスケジュールを設定します。
2. 各ジョブのジョブ名を入力し、<DNT>**Job Class**</DNT>を<DNT>**C**</DNT>に設定します。
3. <DNT>**Step**</DNT>をクリックし、プログラム名`RSBTONEJOB2`を入力して、各ジョブのバリアントを指定します。
4. <DNT>**Start Condition**</DNT>をクリックし、開始日時を設定し、<DNT>**Periodic Job**</DNT>をオンにして、<DNT>**Period Values**</DNT>、<DNT>**Other Period**</DNT>の順にクリックし、分として<DNT>**5**</DNT>を入力します。
