インストレーションの前に、これらの前提条件を完了してください：

## 前提条件チェックリスト

### New Relicアカウント

* <DNT>**Account ID**</DNT>: [アカウントIDを取得する](/docs/accounts/accounts-billing/account-setup/account-id)
* <DNT>**License Key**</DNT>: 認証およびデータアクセス用のNew Relic<InlinePopover type="licenseKey" />を取得する
* <DNT>**Data Center Region**</DNT>: [データセンターの場所（USまたはEU）を選択](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/)してアカウントを設定します

### SAPシステム要件

* <DNT>**Software versions**</DNT>: <DNT>**SAP systems**</DNT>が必要なソフトウェアコンポーネントのリリースレベルを満たしていることを確認してください。システム固有の要件については、[データプロバイダーのリファレンス](/docs/sap-solutions/monitor/data-providers-reference)を参照してください
* <DNT>**Monitored systems**</DNT>：<DNT>**SAP\_BASIS 740+**</DNT>、<DNT>**Kernel 753 Patch 602+**</DNT>、 <DNT>**ST-PI 740 SP16+**</DNT>
* <DNT>**Central monitoring system**</DNT>：<DNT>**SAP\_BASIS 740+**</DNT>、 <DNT>**ST-PI 740 SP14+**</DNT>

### ネットワークアクセス

* <DNT>**HTTPS access**</DNT> <DNT>**New Relic endpoints**</DNT>へ（[エンドポイント URL](#network-endpoints)を参照）
* <DNT>**Firewall rules**</DNT> 設定済み
* <DNT>**Proxy settings**</DNT> 設定済み（必要な場合）

### BTP監視（該当する場合）

<DNT>**BTP accounts**</DNT>を監視する場合、次のいずれかを確認してください：

* <DNT>**CloudALM account**</DNT> 用に設定 <DNT>**BTP monitoring**</DNT>
* <DNT>**Process Integration Runtime**</DNT> サービスサブスクリプション（インテグレーション監視に必要）

<Callout variant="important">
  セキュリティチームまたはネットワークチームからの承認や設定の支援が必要になる場合があります。インストレーションの前にこれらの手順を完了し、[インストレーション準備チェックリスト](/docs/sap-solutions/reference/pre-installation-checklist/#checklist-for-installation-readiness)をご確認ください。
</Callout>

## モニターシステムの準備

エージェントレスアーキテクチャーは、RFC 接続を使用してデータを取得します。

### バージョン要件

必要なソフトウェアのバージョンを確認します：

* <DNT>**SAP\_BASIS**</DNT>: リリース<DNT>**740**</DNT>以降
* <DNT>**Kernel**</DNT>: リリース<DNT>**753 Patch 602**</DNT>以降
* <DNT>**ST-PI**</DNT>: リリース<DNT>**740 SP16**</DNT>以降

### ユーザー設定

* [認可ロールをアップロードします](/docs/sap-solutions/install-and-configure/additional-configuration/#upload-authorization-role) `NEWR_DATA_PROVIDER`
* 〜用にアップロードされたロールを持つ[システムユーザーを作成](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-user-id) <DNT>**RFC connections**</DNT>

### HANA データベースのセットアップ（該当する場合）

<DNT>**HANA database**</DNT>を監視する場合は、<DNT>**Catalog Read**</DNT>および<DNT>**Monitoring**</DNT>のアクセス権を持つ<DNT>**HANA database user**</DNT> `NRAGENT`を作成します：

```sql
CREATE ROLE NR_AGENT_ROLE;
GRANT CATALOG READ TO NR_AGENT_ROLE;
GRANT MONITORING TO NR_AGENT_ROLE;
```

```sql
CREATE USER NRAGENT PASSWORD 'Xxxxxxxx' NO FORCE_FIRST_PASSWORD_CHANGE;
GRANT NR_AGENT_ROLE TO NRAGENT;
```

<DNT>**SQL connection**</DNT>の<DNT>**HANA server host**</DNT>と<DNT>**tenant DB port**</DNT>を収集します。

### その他のデータベース

その他のデータベースについては、<DNT>**ABAP program**</DNT> `RSDBKPIMON`を実行して標準の<DNT>**CCMS database KPIs**</DNT>を有効にします。

### システム要求

* 少なくとも<DNT>**20 dialog work processes**</DNT>および <DNT>**10 background work processes**</DNT>
* <DNT>**New Relic data center**</DNT>へのアウトバウンド<DNT>**HTTPS access**</DNT>（上記のエンドポイントURLを参照）

### ネットワーク設定

* <DNT>**Proxy settings**</DNT> （必要な場合）：

  * サーバーのドメイン名/IPアドレスおよびポート
  * プロキシアクセス認証情報（認証が必要な場合）

* <DNT>**Firewall policies**</DNT>: <DNT>**HTTPS traffic**</DNT> に許可する <DNT>**New Relic endpoints**</DNT>

### RFC 宛先

モニターシステムごとに、トランザクション`SM59`とリモート システム ID を使用して<DNT>**RFC destinations**</DNT>を作成し、接続をテストします。[「モニター システムの RFC 宛先の作成」](/docs/sap-solutions/install-and-configure/additional-configuration/#create-rfc-for-monitored-systems)を参照してください。

<Callout variant="important">
  <DNT>**RFC user**</DNT>には<DNT>**/NEWR/DATA\_PROVIDER**</DNT>ロールが割り当てられている必要があります。
</Callout>

### リモートHANAデータベース監視（該当する場合）

リモートの<DNT>**HANA databases**</DNT>をモニターするには：

* 中央監視システムが<DNT>**HANA server**</DNT>にアクセスできることを確認し、 <DNT>**tenant DB port**</DNT>

* 非S4HANA中央監視システムの場合は、これらのコンポーネントのいずれかをインストールします：

  * <DNT>**HANA Client v2.0**</DNT> （<DNT>**HANA DB version**</DNT>と互換）

  * <DNT>**HANA Client Lib file**</DNT>：

    * `libSQLDBCHDB.so` (Linux)
    * `libSQLDBCHDB.dll` （ウィンドウズ）
    * <DNT>**SAP Kernel folder**</DNT>でライブラリを見つけ、<DNT>**SID Admin ID**</DNT>（例：`sd1adm`）に所有権を割り当てます

* リモート<DNT>**HANA database**</DNT>ごとに、トランザクション`SAP DBACOCKPIT`を使用して<DNT>**DB connection**</DNT>を作成します。[DB接続の作成](/docs/sap-solutions/install-and-configure/additional-configuration/#create-database-connection)を参照してください

## BTP監視の準備（該当する場合）

<DNT>**BTP account**</DNT>を監視するには、<DNT>**BTP account**</DNT>を監視するように設定された<DNT>**CloudALM instance**</DNT>、または<DNT>**Process Integration Runtime**</DNT>サービスのサブスクリプションのいずれかが必要です（インテグレーション監視にのみ適用されます）。接続認証には<DNT>**Service keys**</DNT>が必要です。[サービスキーの取得方法](/docs/sap-solutions/install-and-configure/obtain-service-key/process-integration-runtime)を参照してください。

<DNT>**service keys**</DNT>を使用して、[外部HTTP送信先](/docs/sap-solutions/install-and-configure/additional-configuration/#create-authentication-endpoint)を作成し、<DNT>**API data providers**</DNT>の<DNT>**API host**</DNT>として使用します。
