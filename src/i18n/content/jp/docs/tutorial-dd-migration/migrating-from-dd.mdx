さまざまな理由で、データ監視製品を切り替えることを決定できます。 理由が何であれ、Datadog からスタックを完全に移行する必要なく、New Relic を自分で試すことができます。 New Relic を使用してシステムの監視を開始し、New Relic の可観測性がどのように機能するかを理解するのに役立つ一連のチュートリアルを提供しました。 以下に、インフラストラクチャ、アプリケーション、ログ、ダッシュボード、およびログの監視の設定に関する情報を示します。 <InlinePopover type="alerts"/>データを実際に管理するのに役立ちます。

唯一の前提条件は、New Relic アカウントです。 [無料でサインアップすれ](https://newrelic.com/signup) ば、すぐに始めることができます!

## フルスタックの可観測性のためにエンティティを使用する [#entities]

New Relic では、エンティティの概念を使用してデータを分割および整理しています。基本的に、一意の ID で識別できる New Relic にデータを報告する、または New Relic のデータを含むものはすべてエンティティです。強力なマッピング システムを使用して、エンティティを完全にリンクします。これは、ほぼすべてのソースからのほぼすべてのデータ タイプをより詳細かつ透過的に監視できることを意味します。これらすべては、必要のないものや必要のないもので監視を混乱させることなく、どのレベルでも関心のあるデータを正確に見ることができるようにするためのものです。

## 事前に作成されたダッシュボードの探索 [#dash]

何も設定しなくても、すべての可観測性データを表示できたら素晴らしいと思いませんか?New Relic は厳選されたインターフェイスを提供し、それ以上のものを提供します。データの取り込みを開始した瞬間から、非常に詳細なレベルの情報に最初からアクセスできるようになります。また、そのデータを簡単に並べ替え、検索、コンテキスト化、調査できる多数の使いやすい機能を利用できます。 1、2 回クリックします。

このチュートリアル シリーズでは、インフラストラクチャ、アプリケーション、およびログの監視用に事前に作成されたダッシュボードの形で、これらの精選されたインターフェイスについて説明します。また、他の機能用に事前構成された何百ものダッシュボードを見つけるための場所も示します。ツールのセットアップに費やす時間を減らし、New Relic で実際にツールを使用する時間を増やします。
