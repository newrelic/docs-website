New Relic を自分で試す最初のステップは、監視するデータを取り込むことです。開始するのに最も役立つのは、インフラストラクチャ エージェント、アプリケーション モニター (APM)、またはログ監視機能です。これを完了すると、自動生成された非常に詳細なダッシュボードを使用して、すぐにデータの視覚化を開始できます。事前に作成されたダッシュボードとアラートをセットアップしたり、このシリーズの [最後のドキュメント](/docs/tutorial-dd-migration/making-the-switch/#do-more) に記載されている他の高度な機能の使用を開始したりすることもできます。

監視機能をインストールすると、試してみたいほぼすべての機能が無料で利用できるようになります。開始するには、以下のオプションから 1 つを選択するだけです。

      * [Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration/)用のエージェント
      * インフラストラクチャ エージェントとは別にインストールできる、Amazon AWS、Microsoft Azure、および Google Cloud Platform 用のクラウド エージェント
      * MySQL、Apache、Docker、Kafka などのサービスのオンホスト統合エージェントは、インフラストラクチャ エージェントと連携してデータ レポートを最大限に活用します

      システムとオンホストの統合が [最小要件](/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/)を満たしている限り、試す必要があるのは [無料の New Relic アカウント](https://newrelic.com/signup) だけで、すぐに開始できます。

      * メトリック、イベント、およびトレースを自動的に計測する
      * ログを自動的に取り込む
      * 事前の構成なしでサービス セキュリティを管理する

      ## New Relic アプリケーションの監視を開始する [#get-started-apm]

      セットアップが完了すると、New Relic の APM データが UI の **APM and services** セクションに表示されます。これは、Datadog の **APM** セクションに相当します。**one.newrelic.com > Add Data > All > Guided install** を選択し、 **APM (Application Monitoring)**セクションからエージェントを選択することで、 [**one.newrelic.com**](https://one.newrelic.com) 内でいつでもほとんどのエージェントのガイド付きインストールに従うことができます。

      以下のタイルのいずれかを使用して、APM エージェントを直接インストールすることもできます。ガイド付きインストールでは、データをすぐに観察できるように、APM エージェントのセットアップ手順を順を追って説明します。必要なのは [無料のアカウント](https://newrelic.com/signup) と監視するためのアプリケーションだけで、すぐに始められます!

      * 解決までの平均時間 (MTTR) を短縮し、問題に迅速に対応できるようにします。

      ## New Relic ログ監視を開始します \[#get-started-logs]。

      インストールが完了すると、Datadog の場合とまったく同じように、UI の **Logs**\[ログ] セクションでログの監視が行われます。 [インフラストラクチャ](/docs/journey-migration/migrating-infra/) または [APM](/docs/journey-migration/migrating-apm/) エージェントのいずれかをセットアップした後、ログのガイド付きインストールを使用できます。

      ログを設定する手順はデータ ソースによって異なりますが、どの方法を選択した場合でも、プロセスのすべての手順をガイドすることで、できるだけ簡単でわかりやすくなりますのでご安心ください。New Relic UI で **one.newrelic.com > Add Data > Logging を** 選択すると、ログのガイド付きインストールのリストを表示できます。

        APM ダッシュボードが自動的にカバーする最も重要な領域のいくつかは次のとおりです。

        * **Summary** \[概要] ページで必要な時間範囲にわたるコア Web バイタルの詳細を確認できるため、アプリの監視がより簡単かつ簡単になります。
        * 同じ **Summary** \[概要] ページから、インシデント、脆弱性、デプロイマンのエラー率に関するデータ、スタック全体の健全性に関する情報が得られます。

        * **Events** \[イベント] セクションから必要なだけ詳細なイベント データ (問題、デプロイメント、スレッド プロファイラーに関する情報を含む)。
        * **Reports** \[レポート] セクションからの SLA、サービス レベル、スケーラビリティ、容量、パフォーマンスに関する多数のレポート。

        ログ UI には 4 つの主要なセクションがあります。

        * **All logs** \[すべてのログ] 概要ページには、取り込まれたすべてのログが表示され、後でアクセスしやすくするためにログを並べ替え、フィルターし、ダッシュボードに追加するオプションが提供されます。
        * ログのクエリを簡単にするためにすべてのログ属性をリストする **Attributes** \[属性] セクション。
        * スタックの問題を明らかにし、アラートを作成するための基盤を提供する **Patterns** \[パターン] セクション。
        * **Live tail** 機能によるリアルタイムのログ監視により、発生時にログ データを監視できます。

## 事前構成されたアラートを追加する [#alerts]

インストールされているデータ エージェントで使用できる、さまざまな既製のアラートを提供します。上で選択したオプションに応じて、さまざまな場所でこれらのアラートにアクセスできます。

* インフラストラクチャ: [**one.newrelic.com > Add Data > Infrastructure & OS**](https://one.newrelic.com/marketplace?state=e0a1fd90-4dc6-aef2-a0ae-49775c21d033)に移動し、 **Alerts** \[アラート] タブを選択します。
* アプリケーション: [**one.newrelic.com > Add Data > Application monitoring**](https://one.newrelic.com/marketplace?state=fe696742-816d-5cc7-94d8-b7b453ef3161)に移動し、 **Alerts** \[アラート] タブを選択します。
* ログ: [**one.newrelic.com > Add Data > Logging**](https://one.newrelic.com/marketplace?state=b2e946fc-c4f8-a1ad-c40a-c63f2bbc55ff)に移動し、 **Alerts** \[アラート] タブを選択します。

選択できるプリセット アラートのリストが表示されます。アカウントに条件を追加すると、必要に応じて条件をカスタマイズできます。必要なオプションを選択し、アカウントを追加し、ガイド付きインストールでデータを接続するだけで準備完了です。事前に作成されたアラートの設定の詳細については、 [ダッシュボードとアラートの追加に関するドキュメントを](/docs/tutorial-dd-migration/migrating-dashboards/)参照してください。
