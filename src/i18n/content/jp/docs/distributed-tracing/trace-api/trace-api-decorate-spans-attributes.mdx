このドキュメントでは、 [Trace API](/docs/apm/distributed-tracing/trace-api/introduction-new-relic-trace-api) に送信されるトレースデータに属性を追加し、スパンが [UI](/docs/apm/distributed-tracing/ui-data/understand-use-distributed-tracing-data) に特定のプロパティを表示する方法について説明します。

## なぜスパンを飾るのか？ [#why]

Trace API にデータを送信する際、カスタム [属性](/docs/using-new-relic/welcome-new-relic/get-started/glossary#attribute) をスパンに追加することができます。たとえば、 `customer.id` や `user.id` のような属性を追加することで、トレースデータの分析に役立てることができます。

いくつかの期待される属性値は、私たちの分散型トレースUIに、いくつかの特定の [トレースプロパティ](/docs/apm/distributed-tracing/ui-data/understand-use-distributed-tracing-data#span-types) と詳細を表示させます。例えば、 `error.` 接頭辞を持つ属性を持つスパンは、UIがそのスパンにエラーを表示します。別の例として、 `db.` 接頭辞を持つ属性を持つスパンは、UIにおいてデータストア・スパンとして表示され、そのデータストア・クエリがハイライトされます。

スパンを装飾して、UIの特定のプロパティを表示することができます。

* トレースデータの [構造の理解を深める](/docs/understand-dependencies/distributed-tracing/get-started/how-new-relic-distributed-tracing-works#trace-structure) 。
* より簡単にトラブルシューティングができるようになりました。

これらのスパンのプロパティがどのように決定され、保存されるかについては、 [Trace structure](/docs/understand-dependencies/distributed-tracing/get-started/how-new-relic-distributed-tracing-works) を参照してください。

属性を追加するためのヒント

* スパンには任意の属性を追加できます。例えば、 `customer.id` のような属性を追加することで、 [世界中のトレースを検索](/docs/understand-dependencies/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one#find-data) 特定の顧客を含むトレースを検索することができます。
* 1つのスパンが複数のカテゴリーに分類されることがあります。例えば、 `external` は、 `datastore` よりも一般的なカテゴリであるため、あるスパンが `external` と `datastore` の両方に分類されている場合、UIではdatastoreスパンとして表示されます。
