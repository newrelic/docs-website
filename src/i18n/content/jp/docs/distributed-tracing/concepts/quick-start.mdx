標準の ディストリビューティッド（分散）トレーシング と [Infinite Tracing](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing)[ のどちらを使用するかを決定する前に、 サンプリング タイプに関するドキュメント](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works) を確認することをお勧めします。また、現在New Relic <InlinePopover type="apm"/>エージェントを使用していて、ディストリビューティッド（分散）トレーシングを有効にしたい場合は、[計画ガイド](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide)を参照してください。

ディストリビューショントレースを設定するには、一般的に3つのステップを行います。

<img
  title="Diagram showing the three steps to set up distributed tracing."
  alt="Diagram showing the three steps to set up distributed tracing"
  src="/images/distributed-tracing_diagram_high-level-dist-tr.webp"
/>

1. <DNT>**Identify services:**</DNT> このリクエストを完了するために使用されるエンドポイント、サービス、言語、およびシステムを特定して書き留めます (この情報は次のステップで必要になります)。 次のような環境図がある場合は、それを使用してリクエストを処理するサービスのリストを作成できます。

   <img
     title="A sample environment diagram showing how you might identify services."
     alt="A sample environment diagram showing how you might identify services."
     src="/images/distributed-tracing_screenshot-full_distributed-tracing-map.webp"
   />

2. <DNT>**Instrument services:**</DNT> 識別した各サービスを計画し、トレース データを送信できるようにします。 <InlinePopover type="apm"/>エージェント、計装サービスなどの一部のツールは自動的にサービスを提供しますが、他のツールではサービスにコードを挿入する必要があります。 インストゥルメンテーションの手順については、下のアイコンをクリックしてください。
