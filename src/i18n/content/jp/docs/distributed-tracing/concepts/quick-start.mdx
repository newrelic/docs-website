開始する前に、 [サンプリング タイプに関するドキュメントを](/docs/distributed-tracing/concepts/how-new-relic-distributed-tracing-works) 参照して、標準の分散トレースまたは [無限トレース](/docs/distributed-tracing/infinite-tracing/introduction-infinite-tracing)を使用するかどうかを判断することをお勧めします。また、現在 New Relic を使用している場合は、 <InlinePopover type="apm"/>エージェントであり、分散トレースを有効にしたい場合は、 [計画ガイド を](/docs/distributed-tracing/concepts/distributed-tracing-planning-guide)参照してください。

始める準備はできましたか？まだお持ちでない場合は、以下で無料の New Relic アカウントを作成して、今すぐデータの監視を開始してください。

<InlineSignup/>

ディストリビューショントレースを設定するには、一般的に3つのステップを行います。

<img
  title="Diagram showing the three steps to set up distributed tracing."
  alt="Diagram showing the three steps to set up distributed tracing"
  src={distributedtracingHighLevelDistTr}
/>

1. **Identify services:** このリクエストを完了するために使用されているエンドポイント、サービス、言語、システムを特定して書き留めます（この情報は次のステップで必要になります）。以下のような環境図があれば、それを使ってリクエストを処理するサービスのリストを作成することができます。

<img
  title="A sample environment diagram showing how you might identify services."
  alt="A sample environment diagram showing how you might identify services."
  src={distributedtracingDistributedTracingMap}
/>

2. **サービスの計測:** 特定した各サービスを計測して、トレース データを送信できるようにします。次のようないくつかのツール <InlinePopover type="apm"/>エージェントはサービスを自動的に計測しますが、他のツールではサービスにコードを挿入する必要があります。インストルメンテーションの手順については、下のアイコンをクリックしてください。
