デフォルトのサンプリング設定はすべてのサービスで機能しますが、最適ではない可能性があります。 APMエージェントはサンプリング設定オプションを提供するため、サービスのニーズに合わせてサンプリングをカスタマイズできます。 このページでは、さまざまな種類のサービスの実際の例と、それらのニーズに合わせてサンプリング設定を変更する方法について説明します。

## 例1: 上流サンプリングによるエンドポイントの分布の悪さ

デフォルト設定では、アダプティブ サンプラーを使用します。 つまり、上流のサービスによってサンプリングされたすべての受信トレースは、現在のサービスによってもサンプリングされることになります。これにより、特定の 1 つのエンドポイントが大量にサンプリングされる一方で、他のエンドポイントが無視され、ほとんどサンプリングされない場合があります。

### 解決策1: 比率サンプラーを使用して、上流サービスによってサンプリングされたトレースをサンプリングする

次の設定では、アップストリーム サービスによってサンプリングされたすべての受信トレースを 10% の時間だけサンプリングすることで、この問題を修正します。 これにより、アップストリーム サービスによってサンプリングされたトレースでスパン リザーバーがいっぱいにならないため、他のエンドポイントをサンプリングする余地が生まれます。

```json
{
    distributed_tracing: {
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.1
                }
            }
        }
    }
}
```

### 解決策2: 比率サンプラーを使用して、上流サービスによってサンプリングされたトレースの一部を完全な粒度でキャプチャし、残りを部分的な粒度でキャプチャする

アップストリーム サービスによってサンプリングされたときにトレースに関する最小限の情報もキャプチャする必要がある場合は、代わりに次の設定を使用できます。 この設定は、上流サービスによってサンプリングされたすべてのトレースの 5% を完全粒度としてキャプチャし、残りを部分粒度としてキャプチャします。 アップストリーム サービスによってサンプリングされたトレースの大部分を部分的な粒度としてキャプチャすることにより、スパン リザーバーがすぐにいっぱいになることがなくなり、他のエンドポイントをサンプリングする余地が残ります。

```json
{
    distributed_tracing:{
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.5
                }
            }
            partial_granularity: {
                enabled: true,
                type: "compact",  # If capturing AI data set this to "essential"
                remote_parent_sampled: always_on
            }
        }
    }
}
```

## 例 2: 一部のエンドポイントがほとんど呼び出されないため、キャプチャされたエンドポイントの分布が不適切

デフォルト設定では、アダプティブ サンプラーを使用します。 つまり、1 分間に約 10 個のトレースをキャプチャすることになります。ランダムな確率により、アダプティブ サンプラーはより頻繁に発生するエンドポイント呼び出しをキャプチャする可能性が高くなります。つまり、めったに呼び出されないエンドポイントは見逃され、サンプリングされない可能性が高くなります。

### 解決策1: 比率サンプラーを使用して、上流サービスによってサンプリングされたトレースをサンプリングする

次の設定では、アップストリーム サービスによってサンプリングされたすべての受信トレースを 10% の時間だけサンプリングすることで、この問題を修正します。 これにより、アップストリーム サービスによってサンプリングされたトレースでスパン リザーバーがいっぱいにならないため、他のエンドポイントをサンプリングする余地が生まれます。

```json
{
    distributed_tracing: {
        sampler: {
            remote_parent_sampled: {
                trace_ratio_id_based: {
                    ratio: 0.1
                }
            }
        }
    }
}
```
