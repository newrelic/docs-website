OpenTelemetry データを送信し、UI でサービス (エンティティ) を開いた後、 **\[ランタイムに移動]**をクリックして、言語ランタイムに関連する異常または異常なパフォーマンス パターンを持つサービス インスタンスを特定できます。

<Callout variant="important">
  このビューで使用されるランタイム メトリックを収集するには、別のインストルメンテーション パッケージをインストールする必要があります。以下の[ランタイム メトリックの詳細を](#metric-details)参照してください。
</Callout>

いくつかのサービス インスタンスを選択して、応答時間、スループット、エラー率、ガベージ コレクション時間、メモリ使用量などの主要なメトリックの概要に基づいて比較できます。次に、時系列グラフを使用してこれらすべてのインスタンスのメトリックを比較し、問題を特定して、ガベージ コレクション、ゴルーチン、またはその他のランタイム リソースが関与しているかどうかを判断できます。

典型的なワークフローをご紹介します。

1. **Go ランタイム**をクリックします。

2. 要約されたヘルス メトリックの表を使用して、興味深い Go ランタイム インスタンスを見つけます。

   * フィルターバーを使って、検索結果を絞り込むことができます。
   * ガベージ コレクション時間が長いインスタンスなど、外れ値を見つけるために並べ替えます。

3. それらの興味深い Go ランタイム インスタンスを選択します。

4. **\[比較]**をクリックして、インスタンスごとにファセットされた正常性とランタイムのメトリックを表示します。

    特定のインスタンスにドリルダウンするか、複数のインスタンスを比較すると、UI には Go ランタイム メトリック データによって駆動されるグラフが表示されます。Go ランタイム メトリクスを生成するインストルメンテーションは[、こちら に](https://pkg.go.dev/go.opentelemetry.io/contrib/instrumentation/runtime)あります。
  </Collapser>

  <Collapser
    className="freq-link"
    id="metrics-or-spans"
    title="メトリックまたはスパン"
  >
    テーブルとグラフでは、 [メトリックまたはスパンのいずれかを使用して、ゴールデン シグナル (正常性メトリックとも呼ばれます) を強化することを選択](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-find-entities/#metric-span-toggle)できます: 応答時間、スループット、エラー率。

    ガベージ コレクション、メモリ使用量、ゴルーチンなどのランタイム メトリックは、スパンではなくメトリック データからのみ使用できます。したがって、このトグル スイッチの影響を受けません。
  </Collapser>

  <Collapser
    className="freq-link"
    id="config-steps"
    title="データを確実に表示する方法"
  >
    データがこのセクションに表示されるようにするには、Go ランタイム インスタンスのリストをレンダリングするための`service.instance.id`リソース属性があることを確認してください。
  </Collapser>
