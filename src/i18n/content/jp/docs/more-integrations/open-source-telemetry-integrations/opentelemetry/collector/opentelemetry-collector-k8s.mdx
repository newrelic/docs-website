Kubernetes クラスター内に OpenTelemetry が組み込まれたアプリがすでにある場合は、OpenTelemetry Collector を使用して、Kubernetes に関するメタデータを使用して現在のテレメトリを強化できます。これは、アプリに対する Kubernetes の効果を確認するのに役立ちます。たとえば、スパンをトレースすると、どの Kubernetes クラスター、ノード、ポッド、コンテナーが関係しているかが表示されます。

これを行う方法の詳細については、 [OpenTelemetryをインストルメント化したアプリケーションをKubernetesにリンク](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes)するをご覧ください。

他のコレクターのユースケースに関するヘルプを探している場合は、 [newrelic-opentelemetry-examples](https://github.com/newrelic/newrelic-opentelemetry-examples)リポジトリを参照してください。
