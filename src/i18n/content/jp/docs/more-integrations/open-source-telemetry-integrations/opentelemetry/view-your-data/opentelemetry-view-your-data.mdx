OpenTelemetryデータをNewRelicに送信した後、さまざまなUIページを使用してデータを分析できます。これらのオプションを見てください。

* [エンティティとデータの検索: エンティティを](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-find-entities)見つけて、最も関連性の高いデータにドリルインします
* [概要:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-summary-page)全体的なサービスの正常性を確認し、トラブルシューティングを開始します
* [分散トレース:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-distributed-tracing-page)このサービスを含む分散システムを介してリクエストを追跡します
* [トランザクション:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-transactions-page)サービスのエンドポイントへのリクエストのパフォーマンスを調整し、レイテンシの問題を診断します
* [データベース：](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-database-page)データベース呼び出しを監視します
* [外部サービス:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-external-services-page)関連するエンドポイントに至るまで、サービスと他のサービスとの間の要求がどのように実行されているかを確認します
* [JVMs：](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-jvms-page) JVMのパフォーマンスを監視する
* [ログ:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-logs-page)アプリケーション ログの問題を診断する
* [エラー ページ:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-errors-page)サービスのユーザーが経験しているエラーを確認します
* [エラー インボックス: エラー](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-errors-inbox-page)のトリアージと解決
* [メトリクスエクスプローラー：](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-metrics-explorer-page)特定のメトリクスを確認します
* [カスタム クエリの作成:](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-data-explorer-query-builder)これらの自作ツールを試して、データを掘り下げます。

## OpenTelemetry データの規則 [#conventions]

OpenTelemetry プロジェクトは、可観測性ツールがテレメトリを一貫して解釈できるように、 [メトリクス](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/metrics/semantic_conventions)、 [トレース](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/trace/semantic_conventions)、 [ログ](https://github.com/open-telemetry/opentelemetry-specification/tree/main/specification/logs/semantic_conventions)、および[イベント](https://github.com/open-telemetry/opentelemetry-specification/blob/main/specification/logs/semantic_conventions/events.md)のセマンティック規則を指定します。New Relic はこれらの規則に依存して、テレメトリ データを UI に表示します。

OpenTelemetry 自動インストルメンテーション エージェントと SDK は通常、セマンティック コンベンションに従いますが、プロジェクトの開発が急速に進んでいるため、インストルメンテーションを慣習に合わせて調整する必要がある場合があります。

データが UI で期待どおりに表示されない場合は、関連する UI ページのドキュメントを参照してください。そのページで使用されているセマンティック規則のリファレンスが見つかります。
