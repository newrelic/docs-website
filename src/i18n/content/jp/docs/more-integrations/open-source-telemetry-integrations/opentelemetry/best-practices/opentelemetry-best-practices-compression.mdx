リクエストごとに送信できるデータの量を最大化するには、すべてのOTLPエクスポーターで圧縮を有効にすることをお勧めします。 New Relicは、gRPCまたはHTTPを介して転送されるOTLPペイロードのgzip圧縮をサポートしています。許可される最大ペイロードサイズは1MB（10 ^ 6バイト）です。

他にサポートを希望する圧縮形式がある場合は、 [CNCFSlackチャネル](https://cloud-native.slack.com/archives/C024DRQ63UP)でお知らせください。ここ[でCNCFのSlackアカウントにサインアップ](https://slack.cncf.io/)できることに注意してください。
