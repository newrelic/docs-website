OpenTelemetry と New Relic の使用を開始するには、さまざまなオプションがあります。独自のアプリ、ホスト、または Kubernetes クラスターの監視にすぐに取り掛かることができます。また、いくつかのデモ アプリをセットアップして、OpenTelemetry と New Relic を試すこともできます。OpenTelemetry の機能を利用してデータを収集し、それを New Relic に送信するのに役立つさまざまな手順があります。

## 独自のアプリまたはサービスをセットアップする [#own-app]

OpenTelemetry に精通しており、独自のコードを使用する準備ができている場合は、OpenTelemetry を使用[して独自のアプリまたはサービスをセットアップする を](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/get-started/opentelemetry-set-up-your-app)確認してください。これらの手順は、テレメトリを収集して New Relic にエクスポートするのに役立ちます。

## ホストを監視する [#otel-infra]

OpenTelemetry Collector を使用してホストを監視できます。 [「ホスト監視用のコレクタ」](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/collector/opentelemetry-collector-infra-hosts)を参照してください。

## Kubernetes を監視する [#otel-kubernetes]

OpenTelemetry Collector を使用して Kubernetes クラスターを監視します。 [「OpenTelemetry を装備したアプリケーションを Kubernetes にリンクする」](/docs/kubernetes-pixie/kubernetes-integration/advanced-configuration/link-otel-applications-kubernetes/)の手順を参照してください。

## デモ Java アプリを試す [#demo-app-java]

独自のアプリのセットアップにすぐに取り掛かる代わりに、OpenTelemetry と New Relic を使用してデモ Java アプリをスピンアップできます。

OpenTelemetry に慣れるのに役立つ 3 つの [Java チュートリアル](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/get-started/opentelemetry-tutorial-java) があります。

* New Relic にデータを送信するためにすばやくスピンアップできる、事前にインストルメント化されたアプリを調べます。
* OpenTelemetry Java エージェントを使用して、デモ アプリを自動的に計測します。
* デモ アプリを手動で計測し、拡張機能を使用して SDK を構成します。

## マスタークラスで OpenTelemetry の概念を学ぶ [#otel-masterclass]

OpenTelemetry の背後にある概念について詳しく学び、それらを小さなデモ アプリに適用したい場合は、[マスタークラス](https://developer.newrelic.com/opentelemetry-masterclass)をご覧ください。.NET または Python の実践的なチュートリアルを試すことができます。
