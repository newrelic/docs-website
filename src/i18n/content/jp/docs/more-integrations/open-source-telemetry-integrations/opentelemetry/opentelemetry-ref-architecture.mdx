New RelicでOpenTelemetryを実装するためのガイダンスが必要な場合、このリファレンスアーキテクチャは、ソフトウェア開発者、DevOps、エンタープライズアーキテクト、およびテクノロジーマネージャーがプロジェクトで協力するのに役立ちます。これは、NewRelicでOpenTelemetryを実装する一般的な方法の鳥瞰図を示すテンプレートです。

この図は、OpenTelemetry、Prometheus、Jaeger、および New Relic からテレメトリを取り込む実装を示しています。 <InlinePopover type="apm"/>エージェント。 3 つのサービス (A、B、および C) は、メトリクス、トレース、およびログを送信するようにインストルメント化されています。

OpenTelemetry によって収集されたデータの場合、OTLP エンドポイント経由で New Relic に直接送信することも、最初にオプションの OpenTelemetry Collector に送信することもできます。コレクターを使用すると、テレメトリがバックエンドに送信される前に、データ パイプラインが集約および強化されます (データ分析用の Kafka とエンドツーエンドの可観測性用の New Relic)。
