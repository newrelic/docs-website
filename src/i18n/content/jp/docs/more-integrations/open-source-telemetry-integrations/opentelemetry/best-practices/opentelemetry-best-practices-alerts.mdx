アラートは、テレメトリが特定のしきい値を超えた場合に自分自身や他の人に通知するために使用できるツールです。しきい値は、作成する条件によって定義され、それらの条件をポリシーにグループ化できます。

役割に応じて、条件を使用して、自分とチームに通知される違反を定義することも、それらの違反に対応して、それを引き起こした条件を知りたい場合もあります。

## アラート条件を作成する [#create-alerts]

条件はNRQLクエリを使用して、通知の対象を定義します。条件UIを使用して、違反とその通知を生成する特定のしきい値を設定します。 NRQLクエリを最初から作成するのは難しいように思えるかもしれませんが、New Relicチャートから条件を作成する場合は、チャートの生成に使用されたNRQLを条件に入力します。次に、必要に応じてクエリを変更してから、条件のしきい値を設定できます。

これを試す方法は次のとおりです。

1. グラフの右上隅にある省略記号ボタン (「...」) をクリックし、\[**アラート条件の作成**] をクリックします。

   <img
     src={opentelemetryCreateAlertCondition}
     title="Create alert condition"
     alt="Create alert condition"
   />

   * グラフに単一の指標が表示されている場合は、スライドアウトウィンドウを使用して、NRQLクエリが事前に入力された新しいアラート条件を作成します。
   * グラフに複数のクエリが表示されている場合は、使用するクエリを選択するように求められます。

2. 条件作成フォームでは、既存のポリシーを選択するか、新しいポリシーを作成できます。条件によってアラートがトリガーされた場合の通知方法は、ポリシーによって決まります。

アラートの作成に関する詳細については、「 [最初のアラートを作成](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition)する」を参照してください。

## 現在のアラートを見る [#see-alerts-list]

違反に対応する責任がある場合は、各サービスにどのアラートが設定されているかも知りたい場合があります。 UIから、すべての条件とそれに関連するポリシーのリストをすばやく取得できます。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**から、\[**すべてのエンティティ**] をクリックします。

2. 左側のナビゲーションペインで、\[**サービス]-\[OpenTelemetry]**を選択します。

3. 調査するサービス名をクリックします。

4. **Settings**\[設定]の下の左側のペインで、 **Alert conditions**\[アラート条件]をクリックします。

   <img src={opentelemetryAlertsConditionList} title="Screenshot showing the Alert conditions page for Opentelemetry" alt="Screenshot showing the Alert conditions page for Opentelemetry"/>

\[**アラート条件]**ページでは、アラートの詳細を表示できるだけでなく、警告を追加することもできます。これを行うと、しきい値がクリティカルに近づいた場合に確実に通知されます。 \[**しきい値**]列で、\[**警告の作成**]をクリックすると、アラート条件エディターに直接移動し、そこで警告しきい値を追加できます。

アラートの作成に関する詳細については、「 [最初のアラートを作成](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/your-first-nrql-condition)する」を参照してください。
