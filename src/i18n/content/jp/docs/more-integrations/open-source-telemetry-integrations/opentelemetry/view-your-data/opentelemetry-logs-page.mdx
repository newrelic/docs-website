OpenTelemetryデータを送信し、UIでサービス（エンティティ）を開いたら、\[**ログ**]をクリックしてアプリケーションに関するデータを表示できます。 New Relicでログデータをアプリケーションに関連付ける方法の詳細については、 [OpenTelemetryとロギングのドキュメント](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/best-practices/opentelemetry-best-practices-logs)を参照してください。

UI のこのセクションにデータを表示するには、次の手順を実行します。

* ログが`service.name`を介してサービスと関連付けられていることを確認してください。詳細については、「 [アプリケーション ログを関連付ける」を参照してください](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/best-practices/opentelemetry-best-practices-logs#log-correlation)。
* トレース データと相関させるには、 `trace_id`および`span_id`に含まれるトレース コンテキストをログに含める必要があります。OpenTelemetry インストルメンテーションはトレース コンテキストを`trace_id`および`span_id`として挿入する可能性がありますが、New Relic UI はフィールドが`trace.id`および`span.id`であることを想定しています。`trace_id`と`span_id`を`trace.id`と`span.id`に変換するには、ログ パイプラインでルールを構成する必要があります。

他のUIページでのOpenTelemetryについては、 [UIの概要](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/view-your-data/opentelemetry-view-your-data)を参照してください。
