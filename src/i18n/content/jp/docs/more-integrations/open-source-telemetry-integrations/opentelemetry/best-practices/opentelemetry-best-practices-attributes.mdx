New Relic の属性制限は、OTLP ソースのデータを含む、あらゆるソースからのデータに適用されます。その他の制限については[、「メトリック属性の制限」](/docs/data-apis/ingest-apis/metric-api/metric-api-limits-restricted-attributes/)および[「イベント属性の制限」](/docs/data-apis/ingest-apis/event-api/introduction-event-api/#limits)を参照してください。

* 属性名の最大長：255文字
* 属性値の最大長：4095文字
* 属性配列値の最大サイズ: 64 エントリ

スパン属性が制限を超える場合は、 [スパン制限環境変数](https://github.com/open-telemetry/opentelemetry-specification/blob/d6bcc0cb072d8d6f6ced856f1f23c451648a3caa/specification/sdk-environment-variables.md#span-limits-)を使用して最大長を構成できます。 [リソース属性](https://github.com/open-telemetry/opentelemetry-specification/blob/d6bcc0cb072d8d6f6ced856f1f23c451648a3caa/specification/sdk-environment-variables.md#general-sdk-configuration)が違反者である場合、それをオーバーライドするように`OTEL_RESOURCE_ATTRIBUTES=<offending-attribute>=unset`を設定できます。

すべての言語SDKがスパン制限環境変数をサポートしているわけではないため、使用している言語によって異なります。お使いの言語がこれらをサポートしていない場合は、OpenTelemetryコミュニティが解決できるように、それぞれの言語のSDKリポジトリでGitHubの問題を開くことをお勧めします。
