ワークフロー自動化は、コードなしでインシデント対応とインフラストラクチャ管理を自動化します。アラートに応答し、AWS や Slack などのツールと統合し、運用タスクを自動的に実行するワークフローを構築します。

## 要件

ワークフローを使用するには、次のものが必要です。

* New Relicアカウント
* アドバンスト計算（aCCU）料金プラン

<Callout variant="important">
  ワークフローの自動化と秘密管理は、高度な計算 (aCCU) に含まれています。 CCU があるかどうかわからない場合は、[アカウント設定](https://one.newrelic.com/admin-portal/plan-info/home)を確認するか、アカウント担当者にお問い合わせください。
</Callout>

## 主なメリット

* **より高速なインシデント対応**: アラートの起動時に、EC2 インスタンスのサイズ変更、デプロイメントのロールバック、またはサービスの再起動が自動的に行われます。手動介入は必要ありません。
* **ツールを接続**: New Relic AWS 、Slack、Jira、ServiceNow、および任意の HTTP APIと統合して、チームが作業している場所に強化された集計データを送信します。
* **承認ゲートの追加**: 重要なワークフローに人間による監視を含めます。運用インフラストラクチャを拡張したり、ロールバックを実行したりする前に、チームの確認を要求します。
* **複雑さへの対応**: 条件付きロジック、ループ、データ変換を使用して、独自の運用プロセスを自動化します。

## ワークフローをトリガーする

ワークフローは次の 3 つの方法で開始できます。

* **アラートから**: アラート条件に違反したときにワークフローを自動的に開始します
* **オンデマンド**: 必要に応じて API 経由でワークフローを手動でトリガーします
* **スケジュールに従って**: cron 式を使用して特定の時間にワークフローを自動的に実行します。

個別トリガーのワークフローについては、 [「ワークフローから通知を送信する」を](/docs/workflow-automation/setup-and-configure/create-destinations#trigger-from-alerts)参照してください。
