このドキュメントでは、ワークフローの実行やスケジュールの実行をモニターするために使用できる、さまざまなオブザーバビリティデータについて説明します。

## 監査

すべてのNerdGraphミューテーションコールは監査され、データは顧客アカウント内の`NrAuditEvent`でクエリできます。

### WorkflowRun

```sql
SELECT actionIdentifier, actorId, actorType, description
FROM NrAuditEvent 
WHERE targetType = 'workflow_run'
SINCE 1 hour ago
```

### WorkflowSchedule

```sql
SELECT actionIdentifier, actorId, actorType, description
FROM NrAuditEvent 
WHERE targetType = 'workflow_schedule'
SINCE 1 hour ago
```

## イベント

すべてのワークフロー実行は、ワークフローを開始またはスケジュールするために使用された顧客アカウントのNRDBに`WorkflowAutomationEvent`データを送信します。

ワークフロー実行のイベントデータ用に新しいネームスペース`WorkflowAutomation`が作成されます。

### クエリの例

例えば：

```sql
SELECT *
FROM WorkflowAutomationEvent
where definitionName = 'json-parsing' and runId = '31c973d2-c669-4bd7-9b15-7e3a70a4a074'
SINCE 1 month ago
```

## ログ

任意で、`logLevel`詳細度オプションを指定してワークフローを開始またはスケジュールできます。これらのログは、ワークフローの開始またはスケジュール設定に使用されたワークフローアカウントに追加されます。

ログには以下の属性が付与されます：

* `workflowAutomation.name` はWorkflowDefinition名です
* `workflowAutomation.version` WorkflowDefinitionのバージョンです
* `workflowAutomation.runId` はワークフローのrunIdです
* `entityGuid` WorkflowDefinitionのIDです

スケジュールの発生から開始された場合：

* `workflowAutomation.scheduleId`

アクションステップタイプの場合、`DEBUG`を使用すると、各アクション入力は`workflowAutomation.inputs.<inputName>: <inputValue>`の形式でログエントリに追加の属性を追加します。

`TRACE`ログレベルが実装されると、アクション出力も`workflowAutomation.outputs.<outputName>: <outputValue>`の形式でログエントリに追加されます。

### NRQLクエリの例

以下のNRQLクエリを使用してログをクエリできます：

```sql
SELECT message FROM Log WHERE workflowAutomation.runId = '{run_id}' SINCE 3 hours ago
```

<img title="Diagram of log attributes for workflow runs" alt="Diagram of log attributes for workflow runs" src="/images/log_attributes.webp" />

`workflowAutomation.runId`の他に、ログデータのクエリに役立つその他のログ属性は次のとおりです：

* `workflowAutomation.name`
* `workflowAutomation.version`

スケジュールされた実行の場合、ログエントリには`workflowAutomation.scheduleId`属性も含まれます。
