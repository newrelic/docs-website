エンドポイント URL やその他の入力フィールドで実行時に動的な値を渡すには、 `${{ .workflowInputs.variableName }}`使用します。

**このアプローチでできること:**

* ワークフローがトリガーされたときに動的な入力値を受け入れる
* ワークフローステップ全体でテンプレート構文を使用して入力値を参照する
* さまざまな論点でワークフローの再利用を可能にする

## 例

```json
  {
    "inputs": [
      {
        "key": "urlParams",
        "value": "{\"filter\": \"active\"}"
      },
      {
        "key": "headers",
        "value": "{\"Api-Key\": \"your-api-key\"}"
      }
    ]
  }
```
