Aporiaは、データサイエンスチームが運用中の機械学習モデルを監視するための、速くて簡単で安全な方法です。Aporiaを使えば、チームはカスタマイズ可能な独自のモニターを数分で構築し、データドリフト、予期せぬバイアス、データ整合性の問題、パフォーマンスの低下などの問題を早期に発見するためのライブアラートを受け取ることができます。また、Aporiaには、さらなる調査と根本原因の分析を可能にする調査ツールボックスがあります。 [Aporiaについての詳細はこちら](https://www.aporia.com).

Aporia には、 New Relicとの統合が含まれており、推論調査を示すNew Relic内でカスタマイズされた、MLOps インフラストラクチャの完全なモデル管理を提供します。<InlinePopover type="dashboards"/>

## AporiaとNew Relicの統合 [#integrate-aporia]

Aporia を使用すると、Aporia のモニターによって生成された <InlinePopover type="alerts"/>をNew Relicのインシデント インテリジェンス エンジンと予測データに接続して、 New Relicでモデル用の包括的な監視ダッシュボードを作成できます。

1. <DNT>
     **Login into Aporia's console**
   </DNT>

   : 左側のナビゲーションバーで

   <DNT>
     **Integrations**
   </DNT>

   をクリックし、

   <DNT>
     **New Relic**
   </DNT>

   を選択します。

2. <DNT>
     **Login into your New Relic account:**
   </DNT>

   <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > +Add more data**
   </DNT>

   に移動します。

3. <DNT>
     **Click on Aporia:**
   </DNT>

   検索バーに

   <DNT>
     **Aporia**
   </DNT>

   と入力します。

4. <DNT>
     **Select your account ID:**
   </DNT>

   Aporia と統合するアカウント ID を選択します。

5. <DNT>
     **Get a New Relic API key:**
   </DNT>

   Aporia アイコンをクリックしたら、手順 1 に従います。

   <DNT>
     **Prediction data & model quality alerts**
   </DNT>

   の下で、

   <DNT>
     **Select**
   </DNT>

   をクリックして既存のキーを選択するか、

   <DNT>
     **Create API key**
   </DNT>

   をクリックします。 これにより、New Relic にデータをレポートするために使用される New Relic

   <InlinePopover type="licenseKey"/>

   が作成されました。

6. <DNT>
     **Paste the key in Aporia:**
   </DNT>

   次に、Aporia のダッシュボードに移動し、 New Relicインテグレーション画面で、

   <DNT>
     **New Relic insert token**
   </DNT>

   の下にキーを貼り付けます。 貼り付けたら、

   <DNT>
     **Save**
   </DNT>

   をクリックします。

7. <DNT>
     **Verify the key:**
   </DNT>

   Aporia ダッシュボードで

   <DNT>
     **Verify token**
   </DNT>

   ボタンをクリックして、キーが正しく機能していることを確認します。 ステータスを示すために、

   <DNT>
     **Green**
   </DNT>

   チェック マークまたは

   <DNT>
     **Red**
   </DNT>

   エラー マークが表示されます。

## Aporiaで機械学習モデルを監視する

New Relic と Aporia を統合したので、Aporia によって作成された自動チャートを備えた[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/)を使用してデータを監視できるようになりました。

1. <DNT>**Go to the integration dashboard:**</DNT> インターネットを検証し、インテグレーションが正しく設定されていることを確認したら、 New Relicインテグレーション ダッシュボードに戻り、<DNT>**See your data**</DNT> をクリックします。 これにより、New Relic の Aporia にレポートされたデータを表示する、自動生成されたダッシュボードにリダイレクトされます。

2. <DNT>**Analyze Aporia's dashboard.**</DNT> New Relic に入力された Aporia のダッシュボードには 6 つのグラフが含まれています。

   * <DNT>
       **Most active models**
     </DNT>

     チャートには、選択した期間内に予測を報告したさまざまなモデルが表示されます。

   * <DNT>
       **Most active model versions**
     </DNT>

     グラフには、選択した時間枠で予測を報告したさまざまなバージョンが表示されます。

   * <DNT>
       **Model inferences**
     </DNT>

     グラフには、モデルとバージョンごとに報告された一意の予測の数が表示されます。

   * <DNT>
       **Average numeric inferences**
     </DNT>

     グラフには、各モデルとバージョンについて報告された平均値の数値予測が表示されます。

   * <DNT>
       **Numeric inferences heatmaps**
     </DNT>

     グラフには、各モデルとバージョンについて報告された数値予測のヒストグラムが表示されます。

   * <DNT>
       **Categorical inferences**
     </DNT>

     グラフには、モデルとバージョンごとに報告されたさまざまなユニーク値とカテゴリ予測の頻度が表示されます。

   <img
     title="An image of the Aporia dashboard"
     alt="An image of the Aporia dashboard."
     src="/images/model-performance-monitoring_screenshot-full_aporia-dashboard.webp"
   />

3. <DNT>**Filter data:**</DNT> <DNT>**…**</DNT>ボタンをクリックし、 <DNT>**Edit**</DNT>をクリックします。 右側のナビゲーションバーの<DNT>**User as filter**</DNT>で、 <DNT>**Filter the current dashboard**</DNT>を有効にして、 <DNT>**Save**</DNT>をクリックします。

4. <DNT>**Setup alert notifications:**</DNT> いくつかのダッシュボードを作成したら、データに関するアラートを受け取ることができます。 チャートから NRQL アラート条件を作成するには、チャート ウィジェットをクリックし、 <DNT>**Create alert condition**</DNT>をクリックします。 条件に名前を付けてカスタマイズしたら、既存のポリシーに追加したり、新しいポリシーを作成したりできます。

   <img
     title="Set up notifications for Aporia"
     alt="Set up notifications for Aporia."
     src="/images/model-performance-monitoring_screenshot-crop_aporia-create-alert.webp"
   />

5. <DNT>**Get notified:**</DNT> アラート条件を作成したら、通知方法を選択できます。 [通知チャネルの設定方法](/docs/alerts-applied-intelligence/new-relic-alerts/alert-notifications/notification-channels-control-where-send-alerts)については、ドキュメントを参照してください。

6. <DNT>**Correlate your incidents:**</DNT> 通知に加えて、インシデント インテリジェンスを使用して、インシデントを関連付けることができます。 [決定を使用してインシデントを相関させる方法](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/change-applied-intelligence-correlation-logic-decisions)については、ドキュメントを参照してください。
