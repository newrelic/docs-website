New Relic は、[ ](/docs/infrastructure/introduction-infra-monitoring)Google CloudStorage データを New Relic にレポートするための 統合 を提供します。[このインテグレーションをインフラストラクチャモニタリングに接続する](#connect)方法と、New Relic がこのインテグレーションについてレポートする[メトリクス データ](#metrics)について説明します。

## 特徴

Google Cloud Storageは、Google Cloud Platformのサービスで、ウェブサイトのコンテンツを提供したり、アーカイブやディザスタリカバリのためにデータを保存したり、データオブジェクトを直接ダウンロードして配布したりするのに使用できます。Google Cloud Storageの統合により、これらの機能にアクセスすることができます。

* Google Cloud Storageに保存・取得しているデータのグラフや情報を見ることができます。
* [自動的に取り込まれた ](/docs/using-new-relic/data/understand-data/query-new-relic-data)[データから、カスタムクエリやチャート](#metrics) を作成する.
* Set [alerts](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#integration) Google Cloud Storage のデータを Integrations ページから直接設定することができます。

## 統合をアクティブ化する [#activate]

統合を有効にするには、標準的な手順に従って [GCP サービスを New Relic](/docs/connect-google-cloud-platform-services-infrastructure) に接続してください。

## ポーリング頻度 [#polling]

New Relic は、5 分のポーリング間隔で Google Cloud Storage サービスを照会します。

## データを見つけて使用する [#find-data]

インテグレーションをNew Relicに接続し、数分待つと、 New Relic UIにデータが表示されます。 <InlinePopover type="dashboards" />ボードや集計設定などの[インテグレーション データを検索して使用する](/docs/telemetry-data-platform/understand-data/new-relic-data-types/)には、 <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; GCP &gt; Google Cloud Storage**</DNT>に移動します。

統合用のカスタムダッシュボードを作成するには、プロバイダー値`GcpStorageBucket`を使用して`GcpStorageBucketSample`イベントタイプの[クエリを作成します](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights)。

## メトリックデータ [#metrics]

統合では、 `method`と`response_code`のすべての値の[メトリックデータ](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics)が報告されます。

* `response_code`：リクエストのレスポンスコード。
* `method`：呼び出されたAPIメソッドの名前。
