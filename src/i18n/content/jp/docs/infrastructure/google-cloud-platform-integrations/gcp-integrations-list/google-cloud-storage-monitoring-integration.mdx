New Relic では、Google Cloud Storage のデータを New Relic にレポートするための [統合](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) を提供しています。 [この統合をインフラストラクチャモニタリングに接続する方法](#connect) と、この統合で New Relic がレポートする [メトリックデータ](#metrics) と [インベントリデータ](#rds-config) についてご紹介します。

## 機能

Google Cloud Storageは、Google Cloud Platformのサービスで、ウェブサイトのコンテンツを提供したり、アーカイブやディザスタリカバリのためにデータを保存したり、データオブジェクトを直接ダウンロードして配布したりするのに使用できます。Google Cloud Storageの統合により、これらの機能にアクセスすることができます。

* Google Cloud Storageに保存・取得しているデータのグラフや情報を見ることができます。
* [自動的に取り込まれた ](/docs/using-new-relic/data/understand-data/query-new-relic-data)[データから、カスタムクエリやチャート](#metrics) を作成する.
* Set [alerts](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information#integration) Google Cloud Storage のデータを Integrations ページから直接設定することができます。

## 統合を有効にする [#activate]

統合を有効にするには、標準的な手順に従って [GCP サービスを New Relic](/docs/connect-google-cloud-platform-services-infrastructure) に接続してください。

## ポーリング頻度 [#polling]

New Relic は、5 分のポーリング間隔で Google Cloud Storage サービスを照会します。

## データの検索と使用 [#find-data]

インテグレーションを New Relic に接続して数分待つと、データが New Relic の UI に表示されます。 [ダッシュボードやアラート設定を含む統合データ](/docs/telemetry-data-platform/understand-data/new-relic-data-types/) を見つけて使用するには、 [**one.newrelic.com**](https://one.newrelic.com) **> インフラストラクチャ> GCP> Google Cloud Storage**.

統合のためのカスタムダッシュボードを作成するには、 [`GcpStorageBucketSample` イベントタイプのクエリを、プロバイダ値 `GcpStorageBucket` で作成します。](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights)

## [メトリックデータ [#metrics]](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights)

[統合は、 ](/docs/integrations/new-relic-integrations/getting-started/use-integration-data-new-relic-insights)[メトリックデータ](/docs/telemetry-data-platform/understand-data/new-relic-data-types/#metrics) を `メソッド` および `レスポンス_コード` のすべての値について報告します。

* `response_code`: リクエストのレスポンスコードです。
* `メソッド`: 呼び出されたAPIメソッドの名前。
