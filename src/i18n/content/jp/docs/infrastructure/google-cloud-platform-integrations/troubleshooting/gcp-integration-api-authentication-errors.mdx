## 問題

[Google Cloud Platform (GCP) のサービス](/docs/introduction-google-cloud-platform-integrations) と New Relic を統合したいが、API 認証エラーが発生する。

## 解決

問題を解決するために

1. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > GCP**
   </DNT>

   に移動します。

2. GCP アカウント ページから

   <DNT>
     **Reauthorize New Relic**
   </DNT>

   を選択します。

3. 標準的な手順に従って、 [GCP サービスを New Relic](/docs/infrastructure/google-cloud-platform-integrations/get-started/connect-google-cloud-platform-services-new-relic/#connect) に接続します。

## 原因

このエラーが発生する一般的な理由は以下の通りです。

* New Relic が GCP API を使用するための権限は、ユーザーが Google アカウント管理アプリケーションから手動で無効にしました。
* New Relic が GCP API を使用するための権限には、1 つ以上の GCP API を照会するために必要な 1 つ以上のパーミッション ([Google OAuth scopes](https://developers.google.com/identity/protocols/googlescopes)) が含まれていません。
* New Relic が GCP API を使用するための権限が失効しました。
