New Relic の [ECS インテグレーション](/docs/introduction-amazon-ecs-integration) お客様の [Amazon ECS](https://docs.aws.amazon.com/ecs/index.html "新しいウィンドウにリンクが開きます。") 環境のパフォーマンスデータをレポートし、表示します。本ドキュメントでは、ECS のパフォーマンスを監視するための推奨されるアラート条件を紹介しています。

## 推奨アラート条件 [#alerts]

ここでは、推奨されるECSのアラート条件を紹介します。これらのアラートを追加するには、 [Alerts UI](http://alerts.newrelic.com/) にアクセスし、以下の [NRQL アラート条件](/docs/alerts/new-relic-alerts/defining-conditions/create-alert-conditions-nrql-queries) を既存または新規のアラートポリシーに追加します。

* 高いCPU使用率

  * NRQLです。FROM ContainerSample SELECT cpuUsedCoresPercent
  * クリティカル：> 90% 5分間

* メモリ使用量が多い

  * NRQLです。FROM ContainerSample SELECT memoryUsageBytes / memorySizeLimitBytes
  * クリティカル：> 80% 5分間

* リスタート回数

  * NRQLです。FROM ContainerSample SELECT max(restartCount) - min(restartCount)
  * クリティカル：> 5分間5
