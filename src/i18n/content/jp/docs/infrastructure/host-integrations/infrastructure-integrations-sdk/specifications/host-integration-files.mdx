New Relic Infrastructure [Integrations SDK](/docs/infrastructure/integrations/intro-infrastructure-integrations) では、カスタムホストおよびサービスデータをレポートするための [オンホスト統合](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) を作成することができます。このドキュメントでは、オンホスト統合に必要なファイルについて説明します。また、New Relic は [Go 言語統合構築ツールとチュートリアル](/docs/infrastructure/integrations-sdk/tools-resources/infrastructure-integration-building-resources) を提供しています。

## 統合ファイル [#overview]

ホスト上の統合を構築する前に、 [互換性と要件](/docs/integrations/integrations-sdk/getting-started/compatibility-requirements-infrastructure-integrations-sdk) を満たしていることを確認してください。

統合には、少なくとも以下のファイルが必要です。

* [実行ファイル](/docs/infrastructure/infrastructure-integrations/custom-integrations/create-integration-executable-file) 任意の言語で書かれており、インフラストラクチャーエージェントが期待するフォーマットでJSONデータをエクスポートするもの。
* [設定ファイル](/docs/create-integrations/infrastructure-integrations-sdk/specifications/host-integrations-standard-configuration-format)

Go言語のビルドツールと、これらのファイルを作成するためのチュートリアルについては、 [Build resources](/docs/integrations/integrations-sdk/getting-started/integrations-tutorials-build-resources) を参照してください。
