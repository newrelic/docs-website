New Relic では、独自のインテグレーションを作成する方法がいくつかあります。

* 一般的な遠隔測定（メトリクス、トレース）ソリューション。当社の [Telemetry SDK をご利用ください](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic).
* 独自のJavaScriptUI機能を使用するカスタム[のNewRelicアプリ](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application)を作成します。
* New Relic Infrastructure をお持ちの場合は、軽量の [Flex 統合ツール](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration) （推奨）、または Integrations SDK を使用して完全な Infrastructure [のオンホスト統合](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) を構築することができます。

<Callout variant="important">
  New Relic は、将来のオンホスト統合のために Prometheus などのオープンソース標準に依存するよう移行しています。この SDK はその移行の基盤ですが、この SDK に関するチュートリアルやツールの一部は最新の開発状況に対応していない可能性があります。
</Callout>

## インフラストラクチャ統合 SDK とは何ですか? [#what-is-it]

当社のインフラストラクチャ統合 SDK は [、インフラストラクチャ エージェントのオンホスト統合を](/docs/infrastructure/host-integrations/installation/install-infrastructure-host-integrations)構築するために使用したもので、インフラストラクチャ エージェントと連携してホストまたはサービスからのデータをレポートします。

このセクションのドキュメントを参照すると、これらの統合のファイル構造と構成オプションを理解するのに役立ちます。これは、統合のトラブルシューティングとカスタマイズに役立ちます。
