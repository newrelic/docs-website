オンホスト統合により、さまざまなサードパーティ サービスを監視できます。

## オンホスト統合とは何ですか? [#overview]

New Relicでは、<DNT>**on-host integrations**</DNT> はインフラストラクチャエージェントと連携して動作するインテグレーションと呼んでいます。 これらのインテグレーションは、サポートされているサービスからデータを収集し、インフラストラクチャエージェントを使用してそのデータをNew Relicに送信します。

オンホスト統合の一部は [Flex 統合 を](/docs/infrastructure/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration)使用して構築されており、これを使用して独自の軽量カスタム統合を構築することもできます。

## 報告されたデータ [#data]

オンホスト統合により、メトリクスとインベントリ データがレポートされます。一部の統合には、ログレポートを有効にするオプションがあります。報告されるデータの詳細は、各統合の特定のドキュメントに記載されています。

## 始めましょう [#get-started]

オンホストインテグレーションの使用を開始するには: すでにNew Relicをお持ちの場合は、 [<DNT>**Integrations & Agents**</DNT> ページ](https://one.newrelic.com/marketplace)に移動して、モニターするサービスを検索してください。

または、次のこともできます。

1. このセクションの統合のリストを参照してください。
2. 統合の要件を確認し、手順に従ってインストールおよび構成します。

すべての統合を参照するには、 [「Instant Observability」ページ](https://newrelic.com/instant-observability)を参照してください。

## 特徴 [#features]

統合をインストールしてアクティブ化すると、次のことができるようになります。

* [フィルタリングと分析](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) メトリクスと構成データを [Infrastructure UI](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure).
* [カスタムクエリとチャートの作成](/docs/using-new-relic/data/understand-data/query-new-relic-data) 統合データの作成。
* [アラート条件](/docs/alerts/new-relic-alerts/getting-started/alerts-glossary#alert-condition) を作成してサービスのパフォーマンスの問題を監視する

## リモートおよびマルチテナント構成の監視 [#multiple-instances]

構成によっては、データベースなど、アクセスできないリモートホストに存在するサードパーティの要素を使用する場合があります。当社のインテグレーションは、ローカルおよびリモートのホストやサーバーからデータを取得することができ、お客様のサービスを継続的に監視することを保証します。また、マルチテナントクラスター構成もサポートします。サービスをホストから抽象化することで、複数のエンティティをリモートインスタンスとして監視することができます。

Apache、Cassandra、MySQL、NGINX、Redis の統合でリモート モニタリングとマルチテナントを有効にするには、 [`remote_monitoring` パラメータ](/docs/infrastructure/host-integrations/understand-use-data/remote-monitoring-host-integrations)を使用します。
