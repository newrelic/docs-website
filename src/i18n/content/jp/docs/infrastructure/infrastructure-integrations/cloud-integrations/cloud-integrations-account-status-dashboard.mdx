New Relicモニタリングにリンクする各クラウド アカウント ( [AWS](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure)や[Azure](/docs/integrations/microsoft-azure-integrations/getting-started/introduction-azure-monitoring-integrations)など) には、プラットフォームがそのクラウド サービス プロバイダーからのデータをどのように処理しているかを示す <DNT>**account status dashboard**</DNT> があります。

## 重要な理由 [#reasons-to-use]

このダッシュボードをモニターする理由は以下の通りです。

* <DNT>
    **Understand API call usage.**
  </DNT>

  メトリクスとインベントリデータの収集にはクラウドAPIを使用します。 ダッシュボードには、これらのAPI呼び出しの数と頻度が表示されます。 これらを監視する必要がある理由は 2 つあります。API APIには費用がかかる可能性があり、特定の使用レベルでAPI使用が制限される可能性があるからです。 使用量を減らすには、 [インテグレーション設定を構成して、](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations)モニターするコンポーネントの数やモニターする頻度を減らすことができます。

* <DNT>
    **Troubleshoot missing data or other data issues.**
  </DNT>

  ダッシュボードには、インテグレーション レポート データの表示方法に影響する次のような情報が表示されます。

  * パーミッションエラーやクォータエグゾーストエラーなど、New Relicのデータ収集に影響を与えるエラー。
  * ポーリング間隔の変更や、タグコレクションの無効化・有効化など、統合設定の変更。

## ダッシュボードデータの理解 [#understand-data]

アカウント・ステータス・ダッシュボードに表示される具体的なデータは、クラウド・サービス・プロバイダーによって異なります。一般的なチャートは以下の通りです。

* <DNT>
    **Data updates**
  </DNT>

  : メトリクス データまたはインベントリデータの更新を表示します (更新は`1`値として表示されます)。

* <DNT>
    **Account changes**
  </DNT>

  : インテグレーションの動作に影響を与えるアクション。 たとえば、クラウド アカウントの名前変更やリンク解除、ポーリング間隔の変更、その他の設定オプションなどです。

* <DNT>
    **Data freshness**
  </DNT>

  : 各インテグレーションで収集された最後のデータ ポイントのタイムスタンプ。

* <DNT>
    **Fetching errors**
  </DNT>

  : これらはデータ収集に関する問題を示しています。 (これはクラウドプロバイダー側の問題である可能性があります。)

チャートデータをよりよく理解するためのオプションがあります。

* チャートの

  <Icon name="fe-help-circle"/>

  アイコンにマウスオーバーすると、チャートの説明が表示されます（利用可能な場合）。

* [グラフの基になる NRQL クエリを表示します](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform)。

## アカウントステータスのダッシュボードを見る [#find-dashboard]

クラウドサービスプロバイダーのアカウントステータスダッシュボードを見つけるには

1. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure**
   </DNT>

   から、クラウド サービス プロバイダー (例:

   <DNT>
     **AWS**
   </DNT>

   ) を選択します。

2. クラウド アカウントには

   <DNT>
     **Account status dashboard**
   </DNT>

   選択します。
