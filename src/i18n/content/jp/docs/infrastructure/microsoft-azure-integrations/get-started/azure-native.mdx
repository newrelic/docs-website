チームが Azure を通じてソフトウェアとクラウド サービスをデプロイする場合、[ Azure NativeNew Relic Service](https://azuremarketplace.microsoft.com/en-us/marketplace/apps/newrelicinc1635200720692.newrelic_liftr_payg) を使用する 、AzureNew Relic を管理し、インフラストラクチャ エージェントと .NET および Java<InlinePopover type="apm" /> エージェントをすべて Azure Portal から直接管理およびインストールできます。

Azure ネイティブの New Relic サービスを使用すると、次のことが可能になります。

* Azure 上に構築されたネイティブに統合されたサービスとして、New Relic を簡単にオンボードして使用できます。
* New Relic を含め、Azure で使用するすべてのリソースに対して 1 つの請求書を取得します。
* New Relic の Azure サブスクリプション アクティビティとリソース ログを自動的に監視します。
* New Relic を使用して Azure メトリックを自動的に監視します。
* 単一のエクスペリエンスを使用して、仮想マシンとアプリ サービスに New Relic エージェントをインストールおよびアンインストールします。

さらに[、500 以上の統合](https://newrelic.com/instant-observability)のいずれかを使用して、必要な場合があるその他の Azure 関連以外の監視に New Relic を使用できます。

<Callout variant="important">
  * このサービスは、Azure で New Relic データをホストすることを選択した顧客のみが利用できます。
  * 追加料金が発生しないようにするには、Azure リソースの診断設定が適切に設定されていることを確認するか、Azure Native New Relic Service 経由の転送を無効にして、データが New Relic プラットフォームに転送されないようにしてください。
</Callout>

## Azure ネイティブの New Relic サービスを入手する [#get-it]

これを設定する方法についての短いビデオを次に示します。

<Video id="48LHdqOMAsQ" type="youtube" />

詳細なドキュメントについては、 [Microsoft Azure ドキュメント サイト](https://learn.microsoft.com/en-us/azure/partner-solutions/new-relic/new-relic-overview)を参照してください。このサービスのセットアップ方法の簡単な要約を次に示します。

1. Azure Marketplace で、Azure Native New Relic Service をサブスクライブします。
2. Azure ポータルで、New Relic リソースを作成します。
3. 新しい New Relic アカウントを作成するには<DNT>**Create a New Relic resource**</DNT>を選択するか、既存のリソースにリンクするには<DNT>**Link an existing New Relic resource**</DNT>を選択してください。
4. 表示されるプロンプトに入力し、 <DNT>**Review + create**</DNT>を選択します。
5. <DNT>**Resource configuration**</DNT>タブで、サービスがメトリックとログを報告する方法を設定します。
6. <InlinePopover type="apm" />エージェントを有効にするには、 <DNT>**App services**</DNT>にアクセスしてください。 インフラストラクチャエージェントを有効にするには、 <DNT>**Virtual machines**</DNT>に移動してください。
7. Azure Portal の<DNT>**Monitored resources**</DNT>タブで、New Relic がデータを受信していることを確認します。 緑色のチェックマークが付いたリソースは New Relic にデータを送信しています。

セットアップの詳細については、 [Microsoft Azure ドキュメント サイト](https://learn.microsoft.com/en-us/azure/partner-solutions/new-relic/new-relic-overview)を参照してください。

## New Relicにログイン [#login-newrelic]

New Relic リソースを作成し、Azure がそれにデータを送信していることを確認したら、New Relic に移動できます。

New Relic にログインするには:

1. 新しいリソースの Azure <DNT>**Overview**</DNT>ページで、 <DNT>**Login to New Relic account**</DNT>をクリックします。
2. 新しいパスワードを作成する必要があります。 これを行うには、 <DNT>**Forgot your password**</DNT>をクリックします。
3. メールアドレスを入力してください。Azure アカウントに使用するものと同じである必要があります。

## データセンターに関する注意事項 [#datacenters]

Azure Native New Relic Service は、データを Microsoft の Azure データ センターに直接保存します。現在、Azure ネイティブ サービスは、Azure の米国東部リージョンにすべてのデータを格納しています。

New Relic のログ パターン サービスは、Amazon Web Services でのみ動作します。New Relic ログ パターンを使用することにより、New Relic がお客様のデータをアマゾン ウェブ サービスに移動し、データを処理することに同意するものとします。これは、データを New Relic の Azure データ センターに保存するか、Amazon Web Services データ センターに保存するかにかかわらず適用されます。
