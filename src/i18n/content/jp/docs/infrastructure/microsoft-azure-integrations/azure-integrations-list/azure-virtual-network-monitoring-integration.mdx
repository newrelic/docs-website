[New Relic インフラストラクチャ モニタリング](/docs/infrastructure) は、 [Microsoft Azure の Virtual Network](https://docs.microsoft.com/en-us/azure/virtual-network/virtual-networks-overview) の統合を提供し、Virtual Network サービスから New Relic にデータを報告します。このドキュメントでは、この統合を有効にする方法と、取得可能なデータについて説明します。

## 機能

New RelicのAzure Virtual Network向けインテグレーションは、1秒あたりのパケットドロップ数や1秒あたりの転送バイト数など、仮想ネットワーク(VNets)に関するメトリックデータをレポートします。また、アカウントのステータスや構成に関するデータも収集します。

[New Relic](/docs/infrastructure) から仮想ネットワークのデータを監視したり、アラートを出したりすることができ、 [カスタムクエリやチャートダッシュボード](/docs/using-new-relic/data/understand-data/query-new-relic-data) を作成することができます。

## 統合を有効にする [#requirements]

この統合を有効にするには、標準的な手順に従って、 [New Relic](/docs/infrastructure/microsoft-azure-integrations/getting-started/activate-azure-integrations) で Azure サービスを有効にしてください。

## 設定およびポーリング [#polling]

[設定オプション](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations)を使用し、ポーリングの頻度を変更し、データを絞り込むことができます。

デフォルト [ポーリング](/docs/infrastructure/microsoft-azure-integrations/azure-integrations-list/azure-polling-intervals-infrastructure-integrations) バーチャルネットワーク統合のための情報です。

* ポーリング間隔：メトリクスは1分、インベントリは5分
* 解像度は1データポイント/分

## データの検索と使用 [#find-and-use]

統合データを見つけるには、 [**one.newrelic.com**](http://one.newrelic.com) **> Infrastructure> Azure** にアクセスし、統合を探します。

[データを照会して探索する](/docs/using-new-relic/data/understand-data/query-new-relic-data) `AzureVirtualNetworksPublicIpAddressSample` [イベントタイプ](/docs/data-apis/understand-data/new-relic-data-types/#event-data) を使用することができます。

統合データの見つけ方と使い方については、 [Understand and use data](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) を参照してください。

## インベントリデータ [#inventory]

  variant="important"
  title="有効期限（EOL）の通知"
>
  2022年3月以降は、クラウド統合用のインベントリデータなど、いくつかの機能のサポートを終了します。この移行に向けて簡単に準備できる方法など、詳細については、 [Explorers Hubの投稿](https://discuss.newrelic.com/t/important-upcoming-changes-to-capabilities-and-support-across-errors-classic-labels-service-infrastructure-on-host-snmp-integration-and-inventory-data-in-cloud-integrations/175370) をご覧ください。
</Callout>

インベントリデータは、システムの状態や構成に関する情報です。インベントリデータの見つけ方と使い方の詳細は、 [データの理解と利用](/docs/infrastructure/integrations/find-use-infrastructure-integration-data#inventory-data) をご覧ください。
