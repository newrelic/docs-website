New Relic の Prometheus 統合により、Prometheus または Open Metrics 形式でメトリクスを公開するサービスの厳選されたエクスペリエンスを得ることができ、すぐにプロのようにモニタリングを開始するために必要なものがすべてバンドルされています。

これらのエクスペリエンスには、ゴールデン メトリックとエンティティ ダッシュボードを備えたエンティティ エクスプローラーのサービス エンティティ、サービス概要ダッシュボード、推奨アラートが含まれます。

## 始めましょう [#get-started]

開始するには、 [Prometheus メトリクス データを](https://docs.newrelic.com/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)New Relic に送信する必要があります。各サービスでは、メトリクス エンドポイントを有効にして破棄するように構成するために、追加の前提条件が必要になる場合があります。[Prometheus 統合リスト](/docs/infrastructure/prometheus-integrations/integrations-list/integrations-list-intro#list)で、特定のサービスごとに特定の設定と要件を確認できます。

## データを見つけて使用する [#find-use-data]

### 指標

Prometheus メトリックは、[ディメンション メトリック](/docs/data-apis/understand-data/metric-data/query-metric-data-type/)として保存されます。[NRQL](/docs/telemetry-data-platform/get-data/apis/query-metric-data-type/)を使用してクエリを実行するか、データ エクスプローラーを使用して、使用可能なメトリック、ファセット、およびディメンションによるフィルターを参照できます。

### エンティティ

ほとんどの統合により、監視対象のエンティティが [エンティティ エクスプローラー](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer)で利用できるようになります。これらの統合により、ゴールデン メトリクス、エンティティ ダッシュボード、ワークロード、サービス レベルなどの一般的なプラットフォーム機能が有効になります。

### ダッシュボード

サービスを設定できます <InlinePopover type="dashboards"/>[インスタント可観測性ページ](https://onenr.io/06vjAmnD3jP)を通じて。ダッシュボードは、サービスのパフォーマンスの概要を提供し、各サービスに対して推奨される可観測性のベスト プラクティスに従います。インポートしたら、特定の要件に合わせて調整してカスタマイズできます。

### アラート

推奨されるアラート ポリシーは [、インスタントオブザーバビリティ ページ](https://onenr.io/06vjAmnD3jP)からインポートできます。インポートすると、 [NRQL アラート条件が](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/create-nrql-alert-conditions) 使用可能になりますが、自動的には有効になりません。通知設定を確認し、必要に応じて条件を調整してからオンにします。
