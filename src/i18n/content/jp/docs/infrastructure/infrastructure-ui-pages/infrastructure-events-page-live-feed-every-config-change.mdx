インフラストラクチャ監視 UI の **Events** \[イベント] UI ページは、インベントリ変更イベント、構成変更、ログ分析イベントなどの重要なシステムとホストのアクティビティのライブ フィードです。イベント フィードは、これらのイベントとシステム パフォーマンスの間の相関関係を理解するのに役立ちます。イベントを検索およびフィルタリングして、インフラストラクチャの問題を検出して修復する平均時間を短縮します。

<Callout variant="tip">
  ここでの `event` は、 [イベント データ型](/docs/data-apis/understand-data/new-relic-data-types/#event-data)とは異なるインフラストラクチャ固有の意味を持っていることに注意してください。
</Callout>

**[one.newrelic.com > すべての機能](https://one.newrelic.com/all-capabilities) >Infrastructure > Events** に移動して、 **Events** \[イベント] ページを見つけます。

## イベントヒートマップ [#event-heat-map]

イベントのヒートマップでは、表示されているメトリクスと同じ時間帯に発生したインフラストラクチャ [イベント](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page) のスナップショットが表示されます。ヒートマップの色が濃いほど、その期間に発生したイベントが多いことを意味します。

<img
  title="infra-events-timeline.png"
  alt="infra-events-timeline.png"
  src={infrastructureEventsTimeline}
/>

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**: イベント ヒートマップには、表示されたメトリクスの同じ期間に発生したイベントのパターンが表示されます。
</figcaption>

ヒートマップをインフラストラクチャーページのチャートと比較することで、エコシステムの問題点を迅速に特定することができます。例えば、CPUの大規模なスパイクが発生した場合、その時間帯のイベントヒートマップをクリックして、原因となったイベントを見つけることができます。そこからさらに掘り下げて、真の問題を明らかにすることができます。

        インベントリーイベントには、以下のものがあります。

        * **カーネル** （モジュールと設定を含む）。追加、変更、削除が可能。

        * **パッケージ**: インストール、削除、変更が可能です。
        * **サービス**: 開始、停止、再起動が可能です。
        * **セッション** （ユーザーを含む）。接続、切断が可能。
