インフラストラクチャ モニタリングUIでは、<DNT>**Events**</DNT> UIページには、インベントリ変更イベント、設定変更、ログ分析イベントなど、重要なシステムおよびホスト アクティビティのライブ フィードが表示されます。 イベント フィードは、これらのイベントとシステム パフォーマンスの相関関係を理解するのに役立ちます。 イベントを検索およびフィルタリングして、インフラストラクチャの問題を検出して修復するまでの平均時間を短縮します。

<Callout variant="tip">
  ここでの `event` は、 [イベント データ型](/docs/data-apis/understand-data/new-relic-data-types/#event-data)とは異なるインフラストラクチャ固有の意味を持っていることに注意してください。
</Callout>

<DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**</DNT>に移動して<DNT>**Events**</DNT>ページを見つけます。

</figcaption>

## イベントのタイムライン [#event-timeline]

イベント チャートは、表示されたメトリクスと同じ時間範囲内で発生したインフラストラクチャ[イベント](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page)のスナップショットを提供します。マウスを置くと、色分けされたカテゴリごとのイベント数が表示され、異常なスパイクを調査できます。

<img
  title="infra-events-timeline.png"
  alt="infra-events-timeline.png"
  src="/images/infrastructure_screenshot-crop_events-timeline.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**</DNT>: イベントタイムラインには、表示されたメトリックスで同じ期間に発生したイベントのパターンが表示されます。
</figcaption>

タイムラインをインフラストラクチャ ページのグラフと比較することで、エコシステムの問題を迅速に特定できます。たとえば、大規模な CPU スパイクが発生した場合、その時間範囲のイベント タイムラインをクリックして、その原因となったイベントを見つけることができます。そこからさらに深く掘り下げて、本当の問題を明らかにすることができます。
