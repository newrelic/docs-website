インフラストラクチャ監視 UI の **Events** \[イベント] UI ページは、インベントリ変更イベント、構成変更、ログ分析イベントなどの重要なシステムとホストのアクティビティのライブ フィードです。イベント フィードは、これらのイベントとシステム パフォーマンスの間の相関関係を理解するのに役立ちます。イベントを検索およびフィルタリングして、インフラストラクチャの問題を検出して修復する平均時間を短縮します。

<Callout variant="tip">
  ここでの `event` は、 [イベント データ型](/docs/data-apis/understand-data/new-relic-data-types/#event-data)とは異なるインフラストラクチャ固有の意味を持っていることに注意してください。
</Callout>

**[one.newrelic.com > すべての機能](https://one.newrelic.com/all-capabilities) >Infrastructure > Events** に移動して、 **Events** \[イベント] ページを見つけます。

</figcaption>

## イベントのタイムライン [#event-timeline]

イベント チャートは、表示されたメトリクスと同じ時間範囲内で発生したインフラストラクチャ[イベント](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page)のスナップショットを提供します。マウスを置くと、色分けされたカテゴリごとのイベント数が表示され、異常なスパイクを調査できます。

<img
  title="infra-events-timeline.png"
  alt="infra-events-timeline.png"
  src={infrastructureEventsTimeline}
/>

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > Events**: イベント タイムラインには、表示されたメトリクスに対して同じ期間に発生したイベントのパターンが表示されます。
</figcaption>

タイムラインをインフラストラクチャ ページのグラフと比較することで、エコシステムの問題を迅速に特定できます。たとえば、大規模な CPU スパイクが発生した場合、その時間範囲のイベント タイムラインをクリックして、その原因となったイベントを見つけることができます。そこからさらに深く掘り下げて、本当の問題を明らかにすることができます。

        * **パッケージ**: インストール、削除、変更が可能です。
        * **サービス**: 開始、停止、再起動が可能です。
        * **セッション** （ユーザーを含む）。接続、切断が可能。
