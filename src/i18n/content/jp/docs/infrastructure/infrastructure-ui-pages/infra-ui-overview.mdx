## A: 概要 [#overview]

[インフラストラクチャ UI](https://one.newrelic.com/infra/infrastructure/hosts)を開くと、 [**Hosts** \[ホスト\] ページが](/docs/infrastructure/infrastructure-ui-pages/infra-hosts-ui-page)表示されます。このページには、システムの状態の概要が表示され、ホストをより詳細に調査するための多くのオプションが表示されます。

**Overview** \[概要] セクションの他のページには次のものが含まれます。

* **Kubernetes**: このデータは、有効な場合、 [Kubernetes 統合](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration)によって報告されます。
* **Network** \[ネットワーク]:[ネットワーク パフォーマンス監視 UI](/docs/network-performance-monitoring/get-started/npm-introduction)に移動します。
* **Inventory** \[インベントリ]: ここでは、インフラストラクチャ構成のアップグレードと構成のドリフトを確認できます。[インベントリ UI について詳しくは、こちらをご覧ください](/docs/infrastructure/infrastructure-ui-pages/infrastructure-inventory-page-search-your-entire-infrastructure)。
* **Events** \[イベント]: インフラストラクチャ スタック内の重要なシステム アクティビティのタイムライン。[イベント UI について詳しくは、こちらをご覧ください](/docs/infrastructure/infrastructure-ui-pages/infrastructure-events-page-live-feed-every-config-change)。

## B: 統合 [#integrations]

UI の **Integration** \[統合] セクションには、セットアップしたインフラストラクチャ統合からのテレメトリが含まれます。これらの統合には次のものが含まれます。

* [AWS](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations) 、 [Azure](/docs/infrastructure/microsoft-azure-integrations/get-started/introduction-azure-monitoring-integrations) 、 [Google Cloud Platform](/docs/infrastructure/google-cloud-platform-integrations/get-started/introduction-google-cloud-platform-integrations)などのクラウド サービスの統合
* サードパーティ サービス: Apache、Cassandra、StatsD、RabbitMQ など、多くのインフラストラクチャ関連の統合。これらを参照して追加するには、プラットフォームの [**Add data** \[データの追加\] ページ](https://newrelic.com/instant-observability?category=infrastructure-and-os) に移動します。

## C: 設定 [#settings]

UI の **Settings** \[設定] セクションは、以下を理解するのに役立ちます。

* **Agents** \[エージェント]: システムにインストールされている [インフラストラクチャ エージェント](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent) の詳細を表示します。
* **Alerts** \[アラート]: アクティブなインシデントを中心に、システム内のアラートの概要を表示します。[インフラストラクチャ アラートの詳細については、こちらをご覧ください](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information)。
