イベントのヒートマップでは、表示されているメトリクスと同じ時間帯に発生したインフラストラクチャ [イベント](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page) のスナップショットが表示されます。ヒートマップの色が濃いほど、その期間に発生したイベントが多いことを意味します。

ヒートマップをインフラストラクチャーページのチャートと比較することで、エコシステムの問題点を迅速に特定することができます。例えば、CPUの大規模なスパイクが発生した場合、その時間帯のイベントヒートマップをクリックして、原因となったイベントを見つけることができます。そこからさらに掘り下げて、真の問題を明らかにすることができます。

![インフレイベントのタイムライン.png](./images/infra-events-timeline.png "インフレイベントのタイムライン.png")

  **[one.newrelic.com](http://one.newrelic.com) > インフラストラクチャー**: インフラストラクチャーのモニタリングUIページのヒートマップでは、表示されているメトリクスについて、同時期に発生したイベントによるパターンが視覚的に示されています。

* [システム](/docs/infrastructure/infrastructure-ui-pages/infrastructure-ui/infrastructure-hosts-page#system-tab)

* [プロセス](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-processes-page-inspect-process-performance)
* [ストレージ](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-storage-page-evaluate-disk-usage-efficiency)
* [イベント](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page)
