AWSアカウントとNew Relicをシームレスに統合するには、 [CloudFormation および CloudWatch Metric Streams を介した](/docs/infrastructure/amazon-integrations/aws-integration-for-metrics/via-cloudformation)統合が最も推奨されるアプローチです。 すでに Terraform を使用してAWSアカウントとサービスを管理している場合は、 New Relic統合のセットアップを簡素化するAWSクラウド統合モジュールを提供しているため、次の統合方法を好む場合があります。

## 前提条件 [#prerequisites]

AWS アカウントと New Relic の統合を開始する前に、次の前提条件を満たしていることを確認してください。

* クラウド統合の**[権限](/docs/accounts/accounts-billing/new-relic-one-user-management/user-permissions/#infrastructure)**: New Relicアカウントにはこれらの権限が必要です。
* AWS にリンクする New Relic アカウント ID とリージョン。

## 統合手順 [#integration-steps]

AWS アカウントを New Relic と統合するには、次の手順に従います。

1. [<DNT>**Infrastructure &gt; AWS**</DNT>](https://one.newrelic.com/infra/infrastructure/cloud-aws)に移動します。

2. 正しいアカウントを選択してください。

3. <DNT>**Set up integration**</DNT>をクリックします。

4. 各画面の指示に従って、該当するフィールドに入力します。この統合では、所定の画面で次のオプションを選択します。

   1. <DNT>**Choose Data type(s)**</DNT>： <DNT>Metrics</DNT>
   2. <DNT>**Choose a setup method**</DNT>： <DNT>Automate with Terraform</DNT>

5. <DNT>**Select services**</DNT>画面で、必要に応じて<DNT>**Show resources not supported by CloudWatch Metric Streams**</DNT>トグルを使用して、モニターするサービスを選択し、続行します。

6. <DNT>**Terraform**</DNT>画面から、New Relic アカウントを Terraform プロバイダー アカウントに接続する方法を説明するドキュメントを開きます。

7. 統合が成功すると、 New RelicでAWSデータを確認できるようになります。
