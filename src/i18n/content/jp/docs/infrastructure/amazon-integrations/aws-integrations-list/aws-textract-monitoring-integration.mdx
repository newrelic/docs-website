[New Relic インフラストラクチャの統合に](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) は、メトリクス データを New Relic に送信する AWS Textract の統合が含まれるようになりました。

このドキュメントは、インテグレーションの機能、有効化する方法、および報告可能なデータについて説明するものです。

## 特徴 [#features]

統合を使用して、 [Textract アプリ](https://aws.amazon.com/textract/) からテレメトリ データを収集して New Relic に送信します。サービスを監視し、受信データをクエリし、ダッシュボードを構築して、すべてを一目で確認します。

## レポートデータ [#report-data]

このデータを報告するには 2 つのオプションがあります。推奨されるパスは、 [Amazon CloudWatch Metric Streams 統合](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream)をセットアップすることです。あるいは、 [ポーリングに依存する古い Amazon 統合を](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations)使用することもできます。

## データを見つけて使用する [#find-use-data]

この統合からのデータは、 `Metric` データ型に添付されます。統合のメトリクスを見つけるには、 **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics and events** に移動し、 `aws.textract`でフィルタします。

## データの使用に関するヒント [#use-data]

進行中の変更を通知するアラートを作成できます。たとえば、重大または致命的なエラーを関係者に通知するアラートを設定できます。[アラートの作成の詳細については、こちらをご覧ください](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts)。
