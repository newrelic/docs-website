New RelicAmazon Web Services (AWS)[Amazon Keyspaces (Apache Cassandra 用)](https://aws.amazon.com/keyspaces/) メトリクスおよびその他のデータを にレポートするためにNew Relic と統合します。

この文書では、統合を有効にする方法と、報告されるデータについて説明します。

## 特徴 [#features]

統合を使用して、Amazon Keyspaces からテレメトリーデータを収集し、 New Relicに送信します。 サービスを監視し、受信データを書き、ダッシュボードを構築してすべてを一目で観察します。

## 統合をアクティブ化する [#activate]

この統合は、 CloudWatch Metric Streamsを通じて利用できます。

この統合を有効にするには、[ AWSを介して サービスを に接続するNew Relic CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを見つけて使用する [#find-data]

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Metrics and events**</DNT>に移動して`aws.cassandra`でフィルタリングします。

## メトリックデータ [#metrics]

この New Relic インフラストラクチャ統合は、次の Amazon Keyspaces データを収集します。

### アラートの作成

変更があった場合に通知するように<InlinePopover type="alerts" />を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

詳細については、 [「アラートの概要」](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)を参照してください。
