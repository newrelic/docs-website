[New Relic インフラストラクチャの統合](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)には、AWS Keyspaces の統合が含まれ、メトリクス データを New Relic に送信します。

このドキュメントは、インテグレーションの機能、有効化する方法、および報告可能なデータについて説明するものです。

## 特徴

統合を使用して、 [Keyspace](https://aws.amazon.com/keyspaces/)からテレメトリ データを収集し、New Relic に送信します。サービスを監視し、受信データをクエリし、ダッシュボードを構築してすべてを一目で観察します。

## 統合をアクティブ化する

この統合を有効にするには、 [AWS のサービスを New Relic に接続する](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを見つけて使用する

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Metrics and events**</DNT>に移動して`aws.cassandra`でフィルタリングします。

## メトリックデータ [#metrics]

この New Relic インフラストラクチャ統合は、次の Amazon Keyspaces データを収集します。

### アラートの作成

変更があった場合に通知するように<InlinePopover type="alerts"/>を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

詳細については、 [「アラートの概要」](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)を参照してください。
