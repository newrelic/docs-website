New Relic は Amazon Web Services (AWS) と統合し、 [AWS Transfer Family の](https://aws.amazon.com/aws-transfer-family/)メトリクスやその他のデータを New Relic に報告します。

この文書では、統合を有効にする方法と、報告されるデータについて説明します。

## 特徴 [#features]

統合を使用して、AWSTransfer ファミリーからテレメトリーデータを収集してNew Relicに送信します。 サービスを監視し、受信データを書き、ダッシュボードを構築してすべてを一目で観察します。

## 統合をアクティブ化する [#activate-integration]

この統合は、 CloudWatch Metric Streamsを通じて利用できます。

この統合を有効にするには、[ AWSを介して サービスを に接続するNew Relic CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを見つけて使用する [#find-use-data]

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Metrics &amp; Events**</DNT>に移動して`aws.transfer`でフィルタリングします。

## メトリックデータ [#metric-data]

この New Relic インフラストラクチャ統合は、次の Amazon Transfer Family データを収集します。

### アラートの作成

変更があった場合に通知するように<InlinePopover type="alerts" />を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

アラートの作成について詳しくは、[こちらを](/docs/tutorial-create-alerts/create-an-alert/)ご覧ください。
