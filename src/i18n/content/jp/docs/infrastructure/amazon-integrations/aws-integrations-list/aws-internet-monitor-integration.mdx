AWS CloudWatch Internet Monitor の統合により、CloudWatch Internet Monitor データが New Relic プラットフォームに報告されます。

## 特徴 [#features]

この統合により、 [CloudWatch Internet Monitor](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch-InternetMonitor.html)からテレメトリ データを収集して New Relic に送信できます。統合すると、サービスの監視、受信データのクエリ、ログの監視が可能になり、事前に構築されたダッシュボードを使用してすべてを一目で観察できるようになります。

## メトリクスデータを取得する [#get-metrics-data]

このデータを報告するには 2 つのオプションがあります。推奨されるパスは、 [Amazon CloudWatch Metric Streams 統合](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream)をセットアップすることです。あるいは、 [ポーリングに依存する古い Amazon 統合を](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations)使用することもできます。

## ログデータを取得する [#get-logs-data]

ログデータは、 [ログ取り込み Lambda を](/docs/logs/forward-logs/aws-lambda-sending-cloudwatch-logs)使用して New Relic にレポートできます。必ず [トリガーを Lambda に追加し](/docs/logs/forward-logs/aws-lambda-sending-cloudwatch-logs/#create-trigger) 、 `country` 、 `city` 、 `subdivision` 、および `metro` レベルで `internet-monitor` に対応するログ グループを選択してください。

## データを見つけて使用する [#find-use-data]

このインテグレーションからのメトリックデータは、 `Metric`データ型に添付されます。 データを表示するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Metrics and events**</DNT>に移動して`aws.internetmonitor`でフィルタリングします。
