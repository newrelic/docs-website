[New Relicのインフラストラクチャ統合](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) には、BeanstalkのデータをNew RelicにレポートするためのAWS [Elastic Beanstalk](https://aws.amazon.com/elasticbeanstalk) の統合があります。このドキュメントでは、統合の機能、有効化の方法、レポートできるデータについて説明します。

AWS Elastic Beanstalkは、アプリケーションを簡単にデプロイし、拡張することができるダイナミックなサービスです。AWS Beanstalk内の"アプリケーション" が環境です。"アプリケーション" は、複数の環境を関連付けることができます。

## 要件

AWS Elastic Beanstalkは、デフォルトでいくつかの基本的なメトリクスをAmazon CloudWatchに送信します。さらに、カスタムメトリクスとして強化されたヘルスレポートを有効にすることができます。これらのメトリクスを有効にすると、Amazon CloudWatchアカウントの価格に追加料金がかかる場合があります。拡張されたCloudWatchメトリクスを有効にするには、 [環境に対するAmazon CloudWatchカスタムメトリクスの発行](http://docs.aws.amazon.com/elasticbeanstalk/latest/dg/health-enhanced-cloudwatch.html) を参照してください。

## 統合を有効にする [#activate]

デフォルトでは、Elastic Beanstalk の設定でメトリクスは選択されていません。 New Relic UI でメトリクスを生成するには、 **各環境で選択する必要があります。メトリクスが選択されていない場合、New Relic にデータは報告されません。**

この統合を有効にするには

1. [Connect AWS services to New Relic](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure) の指示に従ってください。
2. Elastic Beanstalkの設定を更新して、 **各** 環境で監視するメトリクス（デフォルトはメトリクスなし）を選択します。AWS Elastic BeanstalkのUIで、 **Configuration> Health** に移動し、ヘルスレポート用のCloudWatchカスタムメトリクスを構成します。環境ごとにレポートするメトリクスを選択すると、Elastic Beanstalkは、 `instance` と `environment` の両方のデータオブジェクトが作成された時点からインベントリデータを生成します。## 設定およびポーリング [#polling][ポーリング頻度](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) を変更したり、 [設定オプション](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations) を使用してデータをフィルタリングすることができます。AWS Elastic Beanstalkインテグレーションのデフォルトのポーリング情報です。* New Relicのポーリング間隔：5分
* Amazon CloudWatchのデータ間隔：1分## データの検索と使用 [#find-data][統合データを表示して使用するには](/docs/infrastructure/integrations/find-use-infrastructure-integration-data) 、 [**one.newrelic.com**](http://one.newrelic.com) **> Infrastructure> AWS** 、AWS Elastic Beanstalk統合リンクの1つを選択します。## メトリックデータ

## インベントリデータ [#inventory]

インベントリデータは、サービスの状態や構成に関する情報を提供します。インベントリデータの詳細については、 [Understand and use data](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#inventory-data) を参照してください。

<Callout variant="tip">
  アスタリスクで示されたデータ `*` は、 [拡張インベントリコレクション](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations#extended) がオンの場合にのみ取得されます。
</Callout>
