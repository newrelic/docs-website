New RelicAmazon Web Services (AWS)[AWSデータベース Migration Services (DMS)](https://aws.amazon.com/dms/) メトリクスおよびその他のデータを にレポートするためにNew Relic と統合します。

この文書では、統合を有効にする方法と、報告されるデータについて説明します。

## 特徴 [#features]

統合を使用して、 データベース移行サービスからテレメトリーデータを収集し、 に送信します。New RelicAWSサービスを監視し、受信データを書き、ダッシュボードを構築してすべてを一目で観察します。

## 統合をアクティブ化する [#activate]

この統合は、CloudWatch MetricStreams を通じて利用できます。

この統合を有効にするには、[ AWSを介して サービスを に接続するNew Relic CloudWatch Metric Streams](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを検索して使用する \[find-data]

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) &gt; Metrics and events**</DNT>に移動して`aws.dms`でフィルタリングします。

さらに、 [one.newrelic.com](https://one.newrelic.com/all-entities)にアクセスして`dms`を検索すると、データベース移行サービスに関連付けられたエンティティを見つけることができます。

## メトリックデータ [#metrics]

この New Relic インフラストラクチャ統合は、次の Amazon Database Migration Service データを収集します。

インポートされたすべてのデータには、次の 1 つの共通のディメンションがあります。 `ReplicationInstanceIdentifier`

## アラートの作成 [#create-alerts]

変更があった場合に通知するように<InlinePopover type="alerts" />を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

アラートの作成について詳しくは、[こちら](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)をご覧ください。
