インフラストラクチャ エージェントを AWS [Database Migration Service](https://aws.amazon.com/dms/)と統合し、そのメトリクス データを New Relic に送信できるようになりました。サービスを監視し、受信データをクエリし、ダッシュボードを構築してすべてを一目で観察します。

このインテグレーションを有効にするには、手順に従って[AWSサービスをNew Relicに接続してください](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)。 有効にすると、インテグレーションのメトリクスを<DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics and events**</DNT>で検索し、 `aws.dms`でフィルタリングできるようになります。

## メトリックデータ [#metrics]

この New Relic インフラストラクチャ統合は、次の Amazon Database Migration Service データを収集します。

インポートされたすべてのデータには、次の 1 つの共通のディメンションがあります。 `ReplicationInstanceIdentifier`

## アラートの作成 [#create-alerts]

変更があった場合に通知するように<InlinePopover type="alerts"/>を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

アラートの作成について詳しくは、[こちら](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)をご覧ください。
