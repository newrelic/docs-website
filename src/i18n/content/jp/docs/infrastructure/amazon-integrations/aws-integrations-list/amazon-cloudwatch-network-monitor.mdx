[New Relic インフラストラクチャの統合に](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)は、Amazon CloudWatch Network Monitor の統合が含まれており、メトリクス データを New Relic に送信します。

このドキュメントでは、統合の機能、アクティブ化方法、およびレポートできるデータについて説明します。

## 特徴

統合を使用して[CloudWatch Network Monitor](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/nw-monitor-create-console.html)からテレメトリ データを収集し、New Relic に送信します。サービスを監視し、受信データをクエリし、ダッシュボードを構築してすべてを一目で観察します。

## 統合をアクティブ化する

この統合を有効にするには、 [AWS のサービスを New Relic に接続する](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを見つけて使用する

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics and events**</DNT>に移動して`aws.networkmonitor`でフィルタリングします。

### アラートの作成

変更があった場合に通知するように<InlinePopover type="alerts"/>を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

アラートの作成について詳しくは、[こちら](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)をご覧ください。
