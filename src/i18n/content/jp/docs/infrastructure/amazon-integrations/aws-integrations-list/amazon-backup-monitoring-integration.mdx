[New Relic インフラストラクチャの統合](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations)には、AWS Backup の統合が含まれ、メトリクス データを New Relic に送信します。

このドキュメントでは、統合の機能、アクティブ化方法、およびレポートできるデータについて説明します。

## 特徴 [#features]

統合を使用して、 [AWS バックアップ](https://aws.amazon.com/backup/)からテレメトリ データを収集し、New Relic に送信します。サービスをチェックし、受信データをクエリし、ダッシュボードを構築してすべてを一目で観察します。

## 統合をアクティブ化する [#activate-integration]

この統合を有効にするには、 [AWS のサービスを New Relic に接続する](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)方法を参照してください。

## データを見つけて使用する [#find-use-data]

インテグレーションのメトリクスを見つけるには、 <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics & Events**</DNT>に移動して`aws.backup`でフィルタリングします。

## メトリックデータ [#metric-data]

この New Relic インフラストラクチャ統合では、次の AWS Backup メトリクスが収集されます。

### アラートの作成

変更があった場合に通知を受け取るように<InlinePopover type="alerts"/>を設定できます。 たとえば、重大なエラーや致命的なエラーの関係者に関する通知を取得するために集計を設定できます。

アラートの作成について詳しくは、[こちらを](/docs/tutorial-create-alerts/create-an-alert/)ご覧ください。
