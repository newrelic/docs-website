New Relic Infrastructureには、メトリクス データを New Relic に送信できるようにする AWS Certificate Manager の統合が含まれるようになりました。 このドキュメントでは、統合の機能、有効化方法、およびレポートに使用できるデータについて説明します。

## 特徴

統合を使用して、テレメトリーデータを収集し、 [AWS Certificate Manager](https://aws.amazon.com/certificate-manager/)から New Relic に送信します。 サービスを監視し、受信データをクエリし、ダッシュボードを構築してすべてを一目で観察できます。

## 統合をアクティブ化する

この統合を有効にするには、 [Amazon CloudWatch Metric Streams 統合ドキュメントの](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream-setup/)手順に従ってください。

## データを見つけて使用する

統合のメトリクスを見つけるには:

1. [one.newrelic.com > すべての機能](https://one.newrelic.com/all-capabilities)にアクセス

2. 選択する

   <DNT>
     **Metrics and events**
   </DNT>

3. フィルター条件 `aws.certificatemanager`

## メトリックデータ [#metrics]

この統合は、次の Amazon Certificate Manager データを収集します。

### アラートの作成

変更があった場合に通知するように<InlinePopover type="alerts"/>を設定できます。 たとえば、重大なエラーや致命的なエラーを関係者に通知するための集計を設定できます。

アラートの作成の詳細については、 [「アラートの概要」ドキュメントを](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts/)参照してください。
