AWS[アプリケーション移行サービスを](https://docs.aws.amazon.com/mgn/latest/ug/what-is-application-migration-service.html)使用して、AWS に移行するサーバーに New Relic エージェントをインストールできるようになりました。

## セットアップ [#setup]

AWS [にNew Relic Infrastructureエージェント](/docs/infrastructure/install-infrastructure-agent/linux-installation/install-infrastructure-monitoring-agent-linux/)をインストールさせるには、 AWSコンソールを通じてリリース後のテンプレートを作成する必要があります。 次の手順に従ってください。

1. [AWS コンソール](https://console.aws.amazon.com/mgn/home)から<DNT>**Application Migration Services** </DNT>を開きます。
2. 利用可能なリストから目的のソース サーバーを開きます。
3. **Post-launch settings** \[リリース後の設定]タブをクリックします。
4. **Edit &gt; Install New Relic agent**を選択し、アカウント ID とライセンスキーを追加します。

## データの監視 [#monitor-data]

セットアップが完了すると、 AWSアプリケーションはNew Relicインフラストラクチャ エージェントによって監視されます。 New Relic でデータを検索するには:

1. <DNT>**[one.newrelic.com](https://one.newrelic.com/) &gt; All entities**</DNT>に移動します。
2. **Hosts**エンティティ セクションで、インスタンスを見つけます。
