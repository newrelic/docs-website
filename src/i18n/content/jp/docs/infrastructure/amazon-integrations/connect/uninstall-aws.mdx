[インフラストラクチャーエージェント](/docs/infrastructure/new-relic-infrastructure/installation/uninstall-infrastructure-agent-or-integrations) をアンインストールしても、 [インフラストラクチャー統合](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure) は自動的にアンインストールされません。

[AWS Metric Stream](/docs/infrastructure/amazon-integrations/connect/aws-metric-stream/) またはその他の AWS インテグレーションをアンインストールするには、以下の手順で行います。

    <tr>
      <th style={{ width: "250px" }}>
        <DNT>
          **If you want to...**
        </DNT>
      </th>

        <DNT>
          **Do this**
        </DNT>
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="disable-aws-integrations">
        AWS CloudWatchのメトリクスストリームの統合を無効にする
      </td>

      <td>
        AWS CloudWatchのメトリックストリーム統合を無効にする。

        1. AWSコンソールから、AWS CloudWatchストリーム、AWS Kinesis Data Firehouseエンドポイント、AWS S3バックアップバケットなど、AWS統合設定時に作成された必要なリソースをすべて削除します。AWS CloudFormationテンプレートを使用した場合は、AWS CloudFormationコンソールで [スタックを削除する手順に従ってください。](https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/cfn-console-delete-stack.html) 。

        2. AWS CloudWatchのメトリックストリームを接続したアクティブなAWSリージョンごとに、ステップ1を完了します。

        3. <DNT>
             **Infrastructure > AWS > AWS Status Dashboard**
           </DNT>

           では、新しいメトリック更新がNew Relicに取り込まれていないことを確認します。

        4. [AWS アカウントを切断する手順を完了します](#unlink-aws) 。
      </td>
    </tr>

    <tr>
      <td id="disable-aws-integrations">
        1つまたは複数のAWSポーリング統合を無効にする
      </td>

      <td>
        AWSアカウントとNew Relicとの連携を維持したまま、サービスを無効にすることができます。

        1. <DNT>
             **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure**
           </DNT>

           から

           <DNT>
             **AWS > Manage services**
           </DNT>

           を選択します。

        2. <DNT>
             **Edit AWS account**
           </DNT>

           ページで、無効にする各アクティブなサービスのチェックボックスをオフにします。

        3. 変更を保存します。
      </td>
    </tr>

    <tr>
      <td id="unlink-aws">
        AWSアカウントの接続を解除する
      </td>

      <td>
        AWS アカウントを New Relic から完全に切り離すには、AWS アカウントのリンクを解除する必要があります。これにより、そのAWSアカウントに関連するすべてのNew Relicの統合が無効になります。

        1. <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Infrastructure > AWS > Manage services**</DNT>に移動します。

        2. <DNT>**Edit AWS account**</DNT>ページから、 <DNT>**Unlink this account**</DNT>を選択します。

        3. 変更を保存します。

        4. AWS にサインインし、 [<DNT>**Services > IAM > Roles**</DNT>](https://console.aws.amazon.com/iam/home#roles)を選択します。

        5. 削除するロールのチェックボックスをオンにして、 <DNT>**Role Actions > Delete Role**</DNT>を選択します。

           AWSアカウントのリンクを解除すると、ARNを介して設定された信頼関係が無効になります。
      </td>
    </tr>
  </tbody>
</table>
