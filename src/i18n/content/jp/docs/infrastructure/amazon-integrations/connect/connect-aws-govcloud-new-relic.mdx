AWS GovCloud (US) [リージョンは、米国（連邦、州、地方機関）、教育機関、およびサポートするエコシステムの特定の規制ニーズに対応するために設計されています。このリージョンは、機密データや規制対象のワークロードをクラウドでホスティングするために設計された分離されたAWSリージョンで、お客様が米国政府のコンプライアンス要件をサポートするのに役立ちます。 ](https://aws.amazon.com/govcloud-us/)[利用可能なAWSサービスのセット](https://aws.amazon.com/govcloud-us/details/) は、AWSエコシステムのサブセットです。

New Relicは、GovCloudに最も重要なサービスを展開する自信を提供し[、プラットフォーム](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)からエコシステム全体を監視および監視できるようにします。

AWS CloudWatchのメトリックストリーム機能は、GovCloudのリージョンでは利用できません。

<Callout variant="tip">
  Terraformを使用して、クラウド統合を有効にするプロセスを自動化できます。 [Terraformの公式ドキュメントサイト](https://registry.terraform.io/providers/newrelic/newrelic/latest/docs/guides/cloud_integrations_guide)でその方法をお読みください。
</Callout>

## AWS GovCloudとNew Relicの接続

New Relic AWSインテグレーションでAmazonデータの受信を開始するには、AmazonアカウントをNew Relicに接続します。

1. Obtain [your credentials](#govcloud-credentials).
2. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Infrastructure &gt; GovCloud**</DNT>に移動します。
3. <DNT>**Add AWS GovCloud account**</DNT>をクリックします。
4. AWS アカウントに名前を付け、アカウントを接続するための認証情報を入力して、 <DNT>**Submit**</DNT>をクリックします。
5. New Relic Infrastructureでモニターする Amazon Web Services を選択し、<DNT>**Save**</DNT> をクリックします。

## New Relic用のGovCloudクレデンシャルを取得する方法 [#govcloud-credentials]

1. [IAM コンソール](https://console.amazonaws-us-gov.com/iam/home#/users)から、 <DNT>**Add user**</DNT>をクリックします。

2. <DNT>**User name**</DNT>には<DNT>**NewRelicInfrastructure-Integrations**</DNT>と入力します。 <DNT>**Select AWS access type**</DNT>の場合は、 <DNT>**Programmatic access**</DNT>を選択します。

   <img title="AWS-add-user.png" alt="Add a user in the AWS IAM console." src="/images/infrastructure_screenshot-crop_aws-add-user.webp" />

   <figcaption>
     <DNT>**AWS IAM console &gt; Add user**</DNT>: <DNT>**NewRelicInfrastructure-Integrations**</DNT>をユーザーとして追加します。
   </figcaption>

3. ポリシーを添付する: <DNT>**ReadOnlyAccess**</DNT>を検索し、 <DNT>**ReadOnlyAccess**</DNT>という名前のポリシーのチェックボックスをオンにして、 <DNT>**Next: Tags**</DNT>をクリックします (タグの追加はオプションです)。 あるいは、[ 独自の管理ポリシーを作成し](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) New RelicAWS、モニターする サービスに応じて に付与する権限を制限することもできます。

   <img title="AWS-set-permissions.png" alt="AWS-set-permissions.png" src="/images/infrastructure_screenshot-crop_aws-set-permissions.webp" />

   <figcaption>
     <DNT>**AWS IAM console &gt; Add user &gt; Set permissions**</DNT>: <DNT>**ReadOnlyAccess**</DNT>を選択します。
   </figcaption>

4. <DNT>**Tags**</DNT>ページで、 <DNT>**Next: Review**</DNT>をクリックします。 ユーザーの詳細概要を確認し、 <DNT>**Create user**</DNT>をクリックします。

   <img title="AWS-create-user.png" alt="In IAM console, create the user." src="/images/infrastructure_screenshot-crop_aws-create-user.webp" />

   <figcaption>
     <DNT>**AWS IAM console &gt; Add user &gt; Set permissions &gt; Tags &gt; Review**</DNT>: 新しいユーザー情報が正確であることを確認します。
   </figcaption>

5. ユーザーが正常に作成されるはずです。 <DNT>**Download .csv**</DNT>ボタンをクリックしてから<DNT>**Close**</DNT>をクリックし、ユーザーのセキュリティ資格情報をダウンロードします。
