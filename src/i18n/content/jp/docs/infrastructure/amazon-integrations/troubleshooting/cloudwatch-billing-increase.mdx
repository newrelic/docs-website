## 問題

New Relic InfrastructureのAmazon統合を設定した後、 [CloudWatch](https://aws.amazon.com/cloudwatch/) APIの使用量が増加し、その結果、 [CloudWatchの使用料請求書](https://aws.amazon.com/cloudwatch/pricing/) に影響が出ています。

## 解決策

<CollapserGroup>
  <Collapser
    id="arn"
    title="インフラストラクチャーアカウントのARNの確認"
  >
    誤ったARNアカウントのインベントリ情報を収集していないか確認してください。インフラストラクチャアカウントに関連付けられたARNが正しいことを確認してください。
  </Collapser>

  <Collapser
    id="polling-frequency"
    title="ポーリング頻度の変更"
  >
    [ポーリング頻度](/docs/integrations/amazon-integrations/get-started/polling-intervals-aws-integrations) は、New Relic がクラウドプロバイダーからデータを収集する頻度を決定します。デフォルトでは、ポーリング頻度は各サービスで利用可能な最大の頻度に設定されています。Amazon CloudWatch の請求書を管理する必要がある場合は、 [ポーリング頻度を下げることができます。](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations#polling)
  </Collapser>

  <Collapser
    id="filter"
    title="データのフィルタリング"
  >
    取り込みたい情報を指定するために、各統合にフィルターを設定することができます。Amazon CloudWatchの請求書を管理する必要がある場合は、 [データをフィルタリングするとよいでしょう。](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations#filter-data) 。
  </Collapser>

  <Collapser
    id="usage"
    title="API使用状況の確認"
  >
    New Relic InfrastructureとAmazon AWSとの統合のためのAPI使用方法を確認する。

    1. Go to [**one.newrelic.com**](http://one.newrelic.com) **> インフラストラクチャー> AWS> アカウントステータスダッシュボード**.

    2. 自動的に表示される New Relic Insights ダッシュボードを確認します。

       Insights ダッシュボードには、お客様のアカウントの先月の Amazon AWS API コール数と、先日の CloudWatch API コール数 (AWS リソースごと) のチャートが表示されます。この情報は、New Relic のみの API 使用状況です。発生する可能性のある他のAWS APIやCloudWatchの使用量は含まれていません。
  </Collapser>
</CollapserGroup>

どのサービスが請求額の増加の原因となるかを判断するには、New Relic のアカウント担当者にお問い合わせいただくか、 [support.newrelic.com](https://support.newrelic.com/) でサポートを受けてください。

## 原因

New Relic Infrastructure Amazonの統合では、CloudWatchを活用してメトリクスを収集しています。AWSは、最初の [月間100万件を超えるリクエストに対して、共同利用者に課金します](http://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/cloudwatch_limits.html) 。

CloudWatchの課金問題は、以下のいずれかが原因と考えられます。

* 同一サービスの複数のプラグインでAmazon統合を可能にする
* 不正な [ロール ARN](http://docs.aws.amazon.com/general/latest/gr/aws-arns-and-namespaces.html) を AWS インテグレーションに追加する。
