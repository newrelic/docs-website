## 問題

New Relic Infrastructure エージェントをインストールして、Amazon アカウントを接続しました。数分待っても、インフラストラクチャ監視の UI やデータのクエリで、Amazon Web Service (AWS) インテグレーションのデータが表示されません。

## 解決策

10分以上経過してもデータが表示されない場合は、以下のトラブルシューティングをお試しください。

1. New Relic Infrastructure エージェントが [インストールされ、正しく動作していることを確認します](/docs/infrastructure/new-relic-infrastructure/troubleshooting/no-data-appears-infrastructure) 。まずInfrastructure agent **をインストールし、** 、AWSアカウントに接続する必要があります。

   <Callout variant="tip">
     [>** one.newrelic.com Infrastructure Events or Infrastructure Settings Agents 、Infrastructure エージェントがインストールされていない可能性があります。**](http://one.newrelic.com) **>** **>** **\*\***>\*\*
   </Callout>

2. アカウントステータスダッシュボードに表示されているエラーを修正します。 [**one.newrelic.com**](http://one.newrelic.com) **> Infrastructure** **> AWS** にアクセスし、各アカウントの **Account status dashboard** を選択します。

   **Role errors** または **Permission errors** レポートが表示された場合は、 [Integrations and managed policies](/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies) を参照し、New Relic にお客様のアカウントから関連するデータを読み取るための必要な権限が与えられているかどうかを確認してください。

   他のエラーが報告されている場合は、 [AWS IAMコンソール](https://console.aws.amazon.com/iam/home?#/roles) にアクセスして、AWS ARNの設定が以下と一致していることを確認してください。

3. [データを探す場所が正しいかどうかを確認してください](/docs/infrastructure/integrations-getting-started/getting-started/understand-integration-data-data-types#metric) 。すべての統合データは、 [data explorer](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) で見つけることができます。
