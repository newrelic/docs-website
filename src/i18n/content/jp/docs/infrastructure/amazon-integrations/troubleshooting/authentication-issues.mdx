## 問題

AWSインテグレーションを設定し、 [New Relic](/docs/integrations/amazon-integrations/get-started/connect-aws-infrastructure) に接続しましたが、 [AWSダッシュボード](/docs/integrations/amazon-integrations/get-started/introduction-aws-integrations#insights) にデータが表示されません。

## 解決

[メトリクスを取得するための適切なパーミッション](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies) が与えられていることを確認してください。

これを確認するには、統合の **Account status dashboard** にアクセスして、エラーが報告されていないことを確認してください。

<img
  title="AWS account status dashboard"
  alt="Screenshot showing the account status dashboard."
  src={infrastructureAwsAccount}
/>

アカウントステータスダッシュボードで、 **Permission Errors** ダッシュボードで、データソースのいずれかにメソッドエラーがあるかどうかを確認します。

<img
  title="Permission errors in the AWS account status dashboard"
  alt="Screenshot showing permissions errors in the account status dashboard."
  src={infrastructureAwsPermissionErrors}
/>

エラーが報告された場合は、アクセス許可ポリシー ドキュメント (標準の AWS `Read-Only Access`ポリシーでない場合) に、[カスタム アクセス許可](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies)ドキュメントに従って、エラー ダッシュボードで報告されたメソッドがあることを確認してください。

<Callout variant="important">
  いくつかのメソッドは統合に特化しておらず、 **[すべての統合に対して付与する必要があります](/docs/integrations/amazon-integrations/get-started/integrations-managed-policies#all-permissions)** 。
</Callout>
