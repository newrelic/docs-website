当社の [インフラストラクチャ](/docs/welcome-new-relic-infrastructure) エージェントは、 [Microsoft Azure の拡張機能](https://docs.microsoft.com/en-us/azure/virtual-machines/extensions/overview) を使ってインストールできます。

<CollapserGroup>
  <Collapser
    id="what-is-extension"
    title="Azureエクステンションとは？"
  >
    Azure仮想マシン(VM)エクステンションは、Azure VMのデプロイ後の設定や自動化タスクを提供する小さなアプリケーションです。たとえば、仮想マシンがソフトウェアのインストール、ウイルス対策、スクリプトの実行を必要とする場合、VMエクステンションを使用することができます。Azure CLI、PowerShell、Azure Resource Managerテンプレート、およびAzureポータルは、すべてAzure VMエクステンションを実行できます。エクステンションは、新しいVMの展開にバンドルすることも、既存のシステムに対して実行することもできます。
  </Collapser>
</CollapserGroup>

New Relic はこれらの拡張機能を使用して、インフラストラクチャ エージェントのインストールを容易にします。これにより、エージェントを手動で更新する必要がなくなり、さらに重要なことに、エージェントは [Azure スケール セット](https://docs.microsoft.com/en-us/azure/virtual-machine-scale-sets/overview)で自動的にスケーリングされます。

## Azure 拡張機能を使用してインフラストラクチャをインストールする [#install-infra-azure-extensions]

サポートされている [オペレーティング システム](https://docs.newrelic.com/docs/infrastructure/install-infrastructure-agent/get-started/requirements-infrastructure-agent/)で Azure 拡張機能を使用してエージェントをインストールするには。

1. 任意のAzure VM、Azureスケールセット、またはスケールセットを使用するAzure Service Fabricに移動し、 **Extensions** タブを選択します。

   <img
     title="azure_ext01.png"
     alt="azure_ext01.png"
     src={infrastructureAzureExtensionsTab}
   />

2. **New Relic Infrastructure** のエクステンションを選択します。

   <img
     title="azure_ext02.png"
     alt="azure_ext02.png"
     src={infrastructureAzureNrInfrastructureExtension}
   />

3. 指示に従い、<InlinePopover type="licenseKey" />を追加します。

   <img
     title="azure_ext03.png"
     alt="azure_ext03.png"
     src={infrastructureAzureNrLicenseKey}
   />

4. これでいいんだ！」と。
