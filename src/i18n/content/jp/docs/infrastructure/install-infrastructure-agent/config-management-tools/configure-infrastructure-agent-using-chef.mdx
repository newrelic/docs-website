New Relic の Chef レシピを使用して New Relic のインフラストラクチャ エージェントをインストールおよび構成する方法については、以下をお読みください。Chef レシピの使用方法については、 [Chef ドキュメントを](https://www.chef.io/solutions/infrastructure-automation/)参照してください。

New Relic クックブックは [Chef Supermarket](https://supermarket.chef.io/cookbooks/newrelic-install)で入手できます。これは、New Relic の可観測性への取り組みを拡大するのに役立ちます。これを実現するために、 [New Relic CLI](https://github.com/newrelic/newrelic-cli) と [オープン インストール リポジトリ](https://github.com/newrelic/open-install-library)を使用します。これは [コミュニティの支援による](https://github.com/newrelic/chef-install) 取り組みです。

## 互換性と要件 [#requirements]

Chef レシピには次の要件があります。

* Chef バージョン 15 以降

サポートされているプラットフォーム:

* Amazon Linux のすべてのバージョン
* CentOS バージョン 5 以降
* Debian バージョン 7 (「Wheezy」) 以降
* Red Hat Enterprise Linux (RHEL) バージョン 5 以降
* Ubuntu バージョン 16.04。_、4月18日。_、20.04\* (LTS バージョン)
* Windows Server 2008、2012、2016、2019 とそのサービス パック。
* SUSE Linux エンタープライズ 11、12

## シェフのレシピ [#recipes]

インフラストラクチャ監視には、デフォルトのレシピが 1 つあります: `default`。インフラストラクチャ エージェント、ログ、およびインフラストラクチャ エージェントをインストールして構成するには、このレシピを含めます。 <InlinePopover type="apm"/>。このレシピでサポートされていないプラットフォームまたはバージョンが検出された場合、Chef の実行は失敗します。構成は、特定のセットアップと標準によって異なります。

<Callout variant="important">
  このクックブックは常に New Relic エージェントの最新リリース バージョンをインストールしますが、古いバージョンのインストールはサポートされていません。
</Callout>

## Chef を使用して New Relic をセットアップする [#recipe-setup]

`newrelic_install`リソース マネージャーは、最小限の構成で[ガイド付きインストールを](https://one.newrelic.com/marketplace/install-data-source?state=54fb0241-f38f-ac22-f941-6566d771b6f4)使用して New Relic をインストルメント化します。それは：

* 希望の構成に必要な属性を設定します
* `newrelic-install::default` レシピを実行リストに追加します

### `newrelic-install` レシピを実行します [#execution]

```ruby
newrelic_install 'install' do
  action                   :install
  new_relic_api_key        node['newrelic_install']['NEW_RELIC_API_KEY']
  new_relic_account_id     node['newrelic_install']['NEW_RELIC_ACCOUNT_ID']
  env                      node['newrelic_install']['env']
  targets                  node['newrelic_install']['targets']
end
```
