## A: 概要 [#overview]

[インフラストラクチャ UI](https://one.newrelic.com/infra/infrastructure/hosts)を開くと、 [<DNT>**Hosts**</DNT>ページ](/docs/infrastructure/infrastructure-ui-pages/infra-hosts-ui-page)が表示されます。このページには、システムの状態の概要が表示され、ホストをより詳細に調べるための多くのオプションが提供されます。

<DNT>**Overview**</DNT>セクションの他のページは次のとおりです:

* <DNT>**Kubernetes**</DNT>: このデータは[、 Kubernetesインテグレーションが](/docs/kubernetes-pixie/kubernetes-integration/get-started/introduction-kubernetes-integration)有効になっている場合に報告されます。
* <DNT>**Network**</DNT>: これにより[、ネットワーク パフォーマンス監視 UI](/docs/network-performance-monitoring/get-started/npm-introduction)に移動します。
* <DNT>**Inventory**</DNT>: ここで、インフラストラクチャ設定のアップグレードと設定のドリフトを確認できます。 [在庫 UI の詳細をご覧ください](/docs/infrastructure/infrastructure-ui-pages/infrastructure-inventory-page-search-your-entire-infrastructure)。
* <DNT>**Events**</DNT>: インフラストラクチャ スタック内の重要なシステム アクティビティのタイムライン。 [イベント UI の詳細をご覧ください](/docs/infrastructure/infrastructure-ui-pages/infrastructure-events-page-live-feed-every-config-change)。

## B: 統合 [#integrations]

UIの <DNT>**Integration**</DNT> セクションには、設定したインフラストラクチャ インテグレーションからのテレメトリが含まれます。 これらの統合には以下が含まれます。

* [AWS](/docs/infrastructure/amazon-integrations/get-started/introduction-aws-integrations) 、 [Azure](/docs/infrastructure/microsoft-azure-integrations/get-started/introduction-azure-monitoring-integrations) 、 [Google Cloud Platform](/docs/infrastructure/google-cloud-platform-integrations/get-started/introduction-google-cloud-platform-integrations)などのクラウド サービスの統合
* サードパーティ サービス: Apache、Cassandra、StatsD、RabbitMQ など、インフラストラクチャ関連のインテグレーションが多数あります。 これらを参照して追加するには、プラットフォームの[<DNT>**Integrations &amp; Agents**</DNT>ページ](https://newrelic.com/instant-observability?category=infrastructure-and-os)に移動します。

## C: 設定 [#settings]

UI の<DNT>**Settings**</DNT>セクションは、次のことを理解するのに役立ちます。

* <DNT>**Agents**</DNT>: システムにインストールされている[インフラストラクチャエージェント](/docs/infrastructure/install-infrastructure-agent/get-started/install-infrastructure-agent)の詳細を確認します。
* <DNT>**Alerts**</DNT>: アクティブなイベントに焦点を当てて、システム内の<InlinePopover type="alerts" />の概要を確認します。 [インフラストラクチャアラートの詳細をご覧ください](/docs/infrastructure/new-relic-infrastructure/infrastructure-alert-conditions/infrastructure-alerts-add-edit-or-view-host-alert-information)。
