## 問題

`HTTP 200`ステータス コードを受け取りましたが、メッセージ本文が空です。

## 解決

これを修正するには、 `application/json`または`application/xml`を含む`Accept`ヘッダーを`GET`リクエストに追加します。例えば：

```
curl -X GET 'https://api.newrelic.com/v2/applications/YOUR_APP_ID/metrics/data.json'
     -H 'X-Api-Key:YOUR_API_KEY' -i
     -H 'Accept: application/json'
```

## 原因

これは、受け入れられるコンテンツ タイプを指定しないデフォルトの`GET`ヘッダーを使用する場合に発生することがあります。HTTP ヘッダーの詳細については、 [HTTP ヘッダーを使用したコンテンツ ネゴシエーション](https://restfulapi.net/content-negotiation/)に関するこのチュートリアルを参照してください。
