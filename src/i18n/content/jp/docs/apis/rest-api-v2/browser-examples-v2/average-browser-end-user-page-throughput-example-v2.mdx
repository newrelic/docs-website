平均ブラウザ スループットは[、 <DNT>**Summary**</DNT>ページ](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance#viewing)の<DNT>**Throughput by browser**</DNT>グラフの右上に表示されます。 これは、グラフに表示されている時間範囲内のbrowserタイプ別の 1 分あたりの平均ページ数 (PPM) です。

## 平均ページスループット [#h2-average]

<Callout variant="important">
  例では New Relic の REST API v2 を使用していますが、メトリックタイムスライスクエリを実行するには[NRQL 関数](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/)を使用することをお勧めします。各 API 値は、同等の NRQL 関数にマッピングできます。これらの API の例に基づいて NRQL クエリを作成する方法については、[ドキュメント](/docs/apis/rest-api-v2/migrate-to-nrql/)を参照してください。
</Callout>

[時間範囲](/docs/apm/apis/requirements/specifying-time-range-api-v2)(デフォルトは過去 30 分) の[平均](/docs/apm/apis/requirements/calculating-average-metric-values-summarize)を取得するには、次のコマンドを使用します。この例の`$APP_ID` 変数と`$API_KEY`変数を、特定の[アプリケーション ID](/docs/apm/apis/requirements/identification-code)と[REST API キー](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing)に置き換えてください。

```sh
curl -X GET "https://api.newrelic.com/v2/applications/$APP_ID/metrics/data.json" \
     -H "X-Api-Key:$API_KEY" -i \
     -d 'names[]=EndUser&values[]=requests_per_minute&summarize=true'
```
