New Relicでは、[New Relic REST API](/docs/apis/getting-started/introduction-new-relic-apis)を含む、[複数のAPI](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2)を提供しています。このドキュメントで紹介する[REST APIエクスプローラー](https://api.newrelic.com)では、アドミンユーザーおよび[APIキー](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)を持つユーザーは以下を実行できます：

* 利用可能なREST APIエンドポイントの参照
* ユーザーインターフェース（API Explorer）内でのREST APIとのインタラクション
* ドキュメンテーションの随時更新ソースの表示
* APIアクション向けcURLコマンドの取得。
* API ExplorerのURLをコピー・アンド・ペーストによる、同僚との構成済みAPIコールの共有

こうして、ご利用のソフトウェアコンポーネントに追加する前に、ソリューションを素早く検索してAPIコールをテストできます。

InfrastructureアラートのREST API呼び出しは、APIエクスプローラーでは利用できません。

## 機能

New RelicのAPI Explorerには、選択されたアカウント向けにインタラクティブなユーザーインタフェースが含まれています。API Explorer UIは、APIコール（**アプリケーション**、**ユーザー**等）ならびに**GET Metric Data**、**PUT (update) Applications**、**DELETE Applications**等の利用可能な関数を一覧化します。

**Parameters**値を入力していくと、これは自動的に**Request**に表示されるため、リクエスト送信前に構文をテストして検証できます。UIは、必須フィールド、フィールド詳細、その種類（整数、フロート、ブール等）、そしてロケーション（パス、クエリ等）を意味します。

APIキーの要件に関する情報は、[REST APIキー](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)をご覧ください。

## APIバージョン1との違い [#v1]

このAPI Explorerが適用される[New Relic REST APIバージョン2](/docs/apm/apis/requirements/new-relic-rest-api-v2-getting-started)は、New Relicに出入りするデータに焦点を当てたものです。バージョン2は、New Relicの非推奨となったREST APIバージョン1を置き換えます。

バージョン2と1には、いくつかの違いがある点に注意してください：

* データの名前が異なる可能性があります
* v2の一部のcURLコマンドが、v1とは異なる可能性があります

<Callout variant="tip">
  New Relicエージェントは異なるAPIを使用しており、API Explorerを介してアクセスすることはできません。
</Callout>
