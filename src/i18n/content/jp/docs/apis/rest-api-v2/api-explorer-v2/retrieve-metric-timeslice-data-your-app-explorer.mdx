New Relic の REST API Explorer (v2) を使用して、アプリケーションの [メトリック・タイムスライス・データ](/docs/telemetry-data-platform/ingest-manage-data/understand-data/new-relic-data-types#timeslice-data) を取得する場合、必要となります。

* [APIキー](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)
* [アプリのID](#app_id)
* アプリで利用できるメトリック・タイムスライス・データの種類。 [メトリック名](#view_metrics) と値を含む。

## 自分のアプリのIDを見る [#app_id]

[アプリのIDを見るには](/docs/apm/apis/requirements/identification-code):

1. New Relic REST [API Explorer](https://rpm.newrelic.com/api/explore "リンクが新しいウィンドウで開きます")から

   <DNT>
     **Applications > GET List**
   </DNT>

   を選択します。

2. New Relic にサインインしていない場合は、アプリの [API キー](/docs/apis/rest-api-v2/requirements/rest-api-key#viewing) を提供してください。

3. オプション:

   <DNT>
     **Applications > List**
   </DNT>

   から、 `name` 、 `ids` 、または`language`フィルターの値を入力します。

4. <DNT>
     **Send Request**
   </DNT>

   を選択します。

5. <DNT>
     **Response**
   </DNT>

   セクションから、アプリの`id`をコピーします。

続けて、 [メトリック名](#view_metrics) を表示する手順を行います。

## アプリのメトリック名の表示 [#view_metrics]

アプリで利用可能なメトリックタイムスライスデータを表示するには

1. New Relic REST API Explorer から

   <DNT>
     **Applications > GET Metric Names**
   </DNT>

   を選択します。

2. <DNT>
     **Applications > Metric Names**
   </DNT>

   から、[アプリケーション ID](/docs/apm/apis/requirements/identification-code)を入力するか貼り付けます。

3. 特定の指標または部分的な指標`name`を入力するか、空白のままにして、使用可能な指標の名前と値のリストを表示します。

4. <DNT>
     **Send Request**
   </DNT>

   を選択します。

5. <DNT>
     **Response**
   </DNT>

   セクションで、アプリの利用可能な`metrics`を確認します。

続けて、 [メトリック](#app_metric_data) タイムスライス [データ](#app_metric_data) を取得します。

## アプリのタイムスライスデータの取得 [#app_metric_data]

アプリのメトリックス値を取得するには、 <DNT>**Metric Names**</DNT>結果から値をコピーし、 <DNT>**Metric Timeslice Data**</DNT>フィールドに貼り付けます。

1. New Relic REST API Explorer から

   <DNT>
     **Applications > GET Metric Data**
   </DNT>

   を選択します。

2. <DNT>
     **Applications > Metric Data**
   </DNT>

   から、[アプリケーション ID](/docs/apm/apis/requirements/identification-code)を入力するか貼り付けます。

3. アプリに 1 つ以上の`names` (

   <DNT>
     **GET Metric Names**
   </DNT>

   から) を入力するか貼り付けます。 各名前を新しい行で始めます。

4. <DNT>
     **Send Request**
   </DNT>

   を選択します。

5. <DNT>
     **Response**
   </DNT>

   セクションで、アプリの利用可能な`metric_data`を確認します。

<DNT>**Response**</DNT>セクションを確認した後、調整を続行し、 <DNT>**Send Request**</DNT>を再度選択できます。 例えば：

* メトリック タイムスライス データの[時間範囲](/docs/apm/apis/api-v2-examples/specifying-time-range-api-v2)を指定するには、 `from`と`to`の値を入力します。
* 時間範囲にわたるデータの[平均](/docs/apm/apis/requirements/calculating-average-metric-values-summarize)を作成するには、 `summarize`を`true`に設定します。または、指定した範囲の各期間のデータを保持するには、 `summarize`を空白のままにします。
