<DNT>NerdGraph API</DNT>を使用して<DNT>cross account destinations</DNT>を作成、テスト、一覧表示、更新、削除します。<DNT>Cross account destinations</DNT>組織レベルで通知管理を一元化できます。

## 目的地を作成する [#create-destination]

宛先を作成するには、宛先の種類ごとに異なる入力情報を提供する必要があります。

<Callout variant="important">
  宛先がどこに作成されるかを指定するスコープオブジェクトを指定する必要があります。スコープオブジェクトには以下が必要です。

  * id: アカウントまたは組織のID。
  * タイプ：スコープの種類（アカウントまたは組織）。

  オプションの two\_way\_integration プロパティは、双方向統合を可能にする統合に使用できます。
</Callout>

    New RelicでクロスアカウントのSlack宛先を作成するには、 <DNT>NerdGraph API</DNT>とSlack間のOAuth認証が必要です。以下の手順では、認証URLを生成して設定を完了する方法を説明します。

    <Steps>
      <Step>
        **OAuth URLを生成する**

        1. <DNT>[NerdGraph Explorer](https://api.newrelic.com/graphiql)</DNT>を開き、次のクエリをクエリ エディタに貼り付けます。このクエリは、組織固有のSlack OAuth URLを生成します。

           ```graphql
               organization {
                 aiNotifications {
                   oAuthUrl(type: SLACK) {
                     url
                     transactionId
                     error {
                       description
                       details
                       type
                     }
                   }
                 }
               }
             }
           }
           ```

        2. エクスプローラー上部の<DNT>**Play** (Execute Query)</DNT>をクリックしてください。
      </Step>

      <Step>
        **認証リンクにアクセスしてください**

        1. クエリが実行されると、画面右側にURLを含むJSONレスポンスが表示されます。
        2. レスポンス内の`url`フィールドを探してください。
        3. URL文字列全体（引用符で囲まれた部分すべて）をコピーしてください。

      </Step>

        1. このリンクをクリックすると、Slackの認証ページに移動します。

        1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &gt; Alerts &gt; Destinations**</DNT>に移動します。
        2. <DNT>**Destinations**</DNT>リストに、先ほど認証した Slack ワークスペースの新しいエントリが表示されているはずです。
      </Step>
    </Steps>
