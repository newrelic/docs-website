New Relic 、NerdGraph を使用して[スクリプト化ブラウザモニター](/docs/synthetics/synthetic-monitoring/scripting-monitors/introduction-scripted-browser-monitors)を作成できます。 スクリプト化ブラウザモニターは、実際のブラウザ環境でカスタム JavaScript コードを実行し、複雑なユーザー インタラクションや複数ステップのワークフローをシミュレートできます。 このチュートリアルでは、NerdGraph API使用してスクリプト化ブラウザモニターの作成を自動化する方法の例を示します。

## スクリプト化ブラウザモニターを作成する [#create-scripted-browser]

<DNT>`syntheticsCreateScriptBrowserMonitor`</DNT>ミューテーションを使用して、ブラウザ化ブラウザモニターを作成できます。 このミューテーションにより、ブラウザで JavaScript コードを実行するカスタム スクリプト監視を設定できます。

### サンプルリクエスト

```graphql
mutation {
  syntheticsCreateScriptBrowserMonitor(
    accountId: ACCOUNT_ID
    monitor: {
      browsers: [BROWSERS]
      devices: [DEVICES]
      locations: { public: ["LOCATION_1", "LOCATION_2"] }
      name: "MONITOR_NAME"
      period: PERIOD
      runtime: {
        runtimeType: "RUNTIME_TYPE"
        runtimeTypeVersion: "RUNTIME_TYPE_VERSION"
        scriptLanguage: "SCRIPT_LANGUAGE"
      }
      script: "SCRIPT_CONTENT"
      status: STATUS
      advancedOptions: { enableScreenshotOnFailureAndScript: ENABLE_SCREENSHOT }
      apdexTarget: APDEX_TARGET
    }
  ) {
    errors {
      description
      type
    }
  }
}
```

### サンプル回答

成功した応答はエラーに対して`null`を返します:

```json
{
  "data": {
    "syntheticsCreateScriptBrowserMonitor": {
      "errors": null
    }
  }
}
```

モニターの作成中に問題が発生した場合、 `errors`配列には、何が問題であったかを説明する`description`と`type`フィールドを持つオブジェクトが含まれます。

## マイクロソフト化ブラウザモニターのランタイムをアップグレードする [#upgrade-monitor-runtime]

スクリプト化ブラウザモニターを、新しいChromeランタイムを使用するようにアップグレードできます。モニターが最新のChromeリリースに合わせて自動的に最新の状態に保たれるように、`LATEST`バージョンを使用することをお勧めします。

<Callout variant="important">
  **Chromeのバージョン管理**: New Relicは、Chromeランタイム向けの`LATEST`バージョンモデルに移行しています。特定のChromeバージョン（134や140など）はサポート終了となり、今後は`LATEST`のみが保守されます。**2026年8月18日**に、古いChromeバージョンのままのモニターは、最新の安定バージョン（Chrome 147以降）に強制的にアップグレードされます。Chromeが新しいバージョンをリリースすると、1週間以内にスクリプト化ブラウザモニターで`LATEST`として利用できるようになります。モニターが最新のChromeのアップデートと機能を自動的に受け取れるようにするには、この日付より前に`runtimeTypeVersion: "LATEST"`にアップグレードしてください。
</Callout>

## マイクロソフト化ブラウザモニターを削除する [#delete-monitor]

シナリオ化ブラウザモニターが不要になった場合は、 <DNT>`syntheticsDeleteMonitor`</DNT>ミューテーションを使用して完全に削除できます。

モニターを削除するには、 [「合成モニターの削除」](/docs/apis/nerdgraph/examples/synthetics-api/ping-monitor/#delete-monitor)セクションを参照してください。

## スクリプト化ブラウザモニターのクエリ [#query-scripted-browser-monitors]

NerdGraphを使用して、メタデータ、スクリプト、マップIDを取得するか、スクリプト化ブラウザモニターのステータスを確認します。

詳細については、次を参照してください：

* [すべてのモニターのクエリ](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data#query-monitors): すべてのSyntheticモニターとその設定を一覧表示します。
* [モニターIDをエンティティGUIDにマッピングする](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data#query-guid-mapping): レガシーモニターIDをエンティティGUIDに移行します。
* [ランタイムアップグレードのステータス](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data#query-runtime-upgrade-all): モニターが最新のランタイムアップグレードの準備ができているかどうかを確認します。
* [モニタースクリプトのクエリ](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data#query-monitor-script) — スクリプト化されたモニターからJavaScriptコードを取得します。

クエリ例の完全なリストを表示するには、[外形監視データのクエリ](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data)ドキュメントを参照してください。
