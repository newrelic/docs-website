合成モニターは、アプリケーション、サービス、エンドポイントをプロアクティブにチェックし、適切に動作していることを確認するのに役立ちます。[NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/) を使用すると、[ 合成モニター](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors/) 、[ プライベートロケーション](/docs/synthetics/synthetic-monitoring/private-locations/private-locations-overview-monitor-internal-sites-add-new-locations/) 、[ 安全な資格情報](/docs/synthetics/synthetic-monitoring/using-monitors/store-secure-credentials-scripted-browsers-api-tests/) 、[ ダウン](/docs/synthetics/synthetic-monitoring/using-monitors/monitor-downtimes-disable-monitoring-during-scheduled-maintenance-times/) タイムの監視の管理を自動化できます。

適切な権限とアクセスがあれば、NerdGraph での書き込みやミューテーションを通じて、外形監視セットアップ全体をプログラムで管理できます。 あるいは、 [New Relic UI](/docs/synthetics/synthetic-monitoring/using-monitors/add-edit-monitors/)経由でこれらの操作を実行することもできます。

<Callout variant="important">
  2024年8月26日以降、パブリックロケーションまたはプライベートロケーションでレガシーランタイムを使用して新しいモニターを作成することはできなくなります。 2024年10月22日をもって、コンテナ化されたプライベートミニオン（1分間あたりの呼出し回数）とレガシーsyntheticランタイムバージョンの[サポートを終了](/whats-new/2024/04/whats-new-04-09-eol-synthetics-runtime-cpm)します。

  * パブリックロケーションの場合は、[ランタイム アップグレードUI](/docs/synthetics/synthetic-monitoring/using-monitors/runtime-upgrade-ui/)使用してモニターを最新のランタイムに更新します。
  * プライベートロケーションの場合、モニターの低下を回避するために[推奨する移行手順](/docs/synthetics/synthetic-monitoring/private-locations/job-manager-transition-guide/#monitorMigration)を確認してください。
</Callout>

## 要件 [#requirements]

要件は次のとおりです。

* [New Relic アカウント](https://newrelic.com/signup)。[ミューテーションとクエリを](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/)認証するには、このアカウントの API ユーザー キーが必要になります。
* [一般的な NerdGraph の要件](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#authentication)。
* 関連するユーザー権限: NerdGraph には、外形監視UIと同じ[外形監視権限](/docs/synthetics/synthetic-monitoring/administration/user-roles-synthetic-monitoring)が必要です。

## 始めましょう [#get-started]

外形監視用 NerdGraph API使用を開始するには:

1. [NerdGraph GraphQLエクスプローラー](https://one.newrelic.com/nerdgraph-graphiql)へ
2. 認証用の[APIユーザー キー](/docs/apis/intro-apis/new-relic-api-keys/#user-key)を取得します。
3. 以下のガイドからモニターの種類または操作を選択してください
4. 提供されたGraphQLミューテーションと書き込みを使用して、外形監視インフラストラクチャを管理します

## エンティティをサポート [#supporting-entities]

モニターに加えて、サポートするインフラストラクチャと設定を管理できます。

* [プライベートロケーション](/docs/apis/nerdgraph/examples/synthetics-api/private-locations)- 内部アプリケーション用にファイアウォールの背後に監視ロケーションを作成します
* [認証情報の保護](/docs/apis/nerdgraph/examples/synthetics-api/secure-credentials)- パスワードやAPIキーなどの機密情報を安全に保存します
* [モニターのダウンタイム](/docs/apis/nerdgraph/examples/synthetics-api/monitor-downtimes)- モニターを一時停止するメンテナンス時間枠をスケジュールします。

## クエリ操作 [#query-operations]

外形監視設定に関する情報を取得するには、[次の操作を](/docs/apis/nerdgraph/examples/synthetics-api/query-synthetics-data)使用します。

* アカウント内のすべてのモニターを完了します
* プライベートロケーション設定を取得する
* 安全な資格情報を一覧表示する（メタデータのみ）
* スケジュールされたモニターのダウンタイムを表示する
* モニタースクリプトを入手して設定を行う
* モニター ID をエンティティ GUID にマップする
* レガシーモニターのランタイムアップグレードステータスを確認する
