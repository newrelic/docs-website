New Relic InsightsクエリAPIは、 NRQLクエリをするためのREST APIです。

<Callout variant="tip">
  <DNT>**This API is no longer the preferred way to query New Relic data.**</DNT> [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/)を使用してクエリしてください。
</Callout>

## 要件および推奨 [#requirements]

<DNT>**This API is no longer the preferred way to query New Relic data.**</DNT> 最善の結果を得るには、[NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph)を使用してデータをクエリする必要があります。

このAPIの使用は、ロール関連ユーザー権限で制限できます。

New Relicにカスタムデータを追加するには、[カスタムデータをレポート](/docs/data-apis/custom-data/get-custom-data-from-any-source)を参照してください。

お客様の組織がEUデータセンターでデータをホストする場合は、適切な[EU地域のエンドポイント](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center#endpoints)を使用していることを確認してください。

## 1. APIキーの登録 [#register]

InsightsのクエリAPIを使用するには、クエリキーが必要です。クエリキーは複数所有することができ、いずれかのクエリキーを使用してInsights APIクエリを発行することができます。Insightsや異なるデータ送信先を紹介するシステムが複数存在する場合、New Relicは複数のクエリキーを使用したデータセキュリティの向上を推奨します。

安全上の理由から、APIキーを使用したクエリキーの変更や読み出しはできません。クエリキーの変更または読み取りには、New Relic UIを使用してください。

<Callout variant="tip">
  <DNT>**This API is no longer the preferred way to query New Relic data.**</DNT> [NerdGraph](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/)を使用してクエリしてください。
</Callout>

新しいクエリキーを作成する場合は、以下の手順に従います。

1. [one.newrelic.com/api-keys](https://one.newrelic.com/api-keys)にアクセスしてください。 EUデータセンターをご利用の場合は、 [one.eu.newrelic.com/](https://one.eu.newrelic.com/api-keys) api-keysにアクセスしてください。 その代わり。

2. APIキーページの右上隅にある<DNT>**Create a key button**</DNT>をクリックします。

3. 新しいキーを設定します

* 表示されたフィールドに新しいキーの名前を入力します
* 説明を追加します（「クエリキー」など）

4. <DNT>**Save**</DNT>をクリックして新しいキーを生成します。

## 2. APIクエリリクエストの作成 [#create-request]

[クエリキーを作成または編集する](#register)と、テンプレートとして使用できるサンプルの`curl`クエリが表示されます。 次のクエリ ルールに従わない限り、サンプル クエリは機能しません。

* [NRQLのクエリ文字列](/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql)は[URLエンコードされて](http://www.w3schools.com/tags/ref_urlencode.asp)いなければなりません。
* クエリ文字列は4000バイト未満でなければなりません。
* URLには有効なアカウントIDを含める必要があります。
* `X-Query-Key`には有効な[クエリキー](#register)を含める必要があります。
* `Content-Type`は`application/json`である必要があります。

### Linux [#unix]

`curl`の例を次に示します。

```
curl -H "Accept: application/json" -H "X-Query-Key: YOUR_QUERY_KEY" "https://insights-api.newrelic.com/v1/accounts/<a href="/docs/accounts-partnerships/accounts/account-setup/account-id">YOUR_ACCOUNT_ID</a>/query?nrql=YOUR_URL_ENCODED_QUERY"
```

### Microsoft Windows [#windows]

<DNT>**Powershell**</DNT>を使用して、APIでイベントをクエリできます。

```
Invoke-WebRequest -Uri https://insights-api.newrelic.com/v1/accounts/<a href="/docs/accounts-partnerships/accounts/account-setup/account-id">YOUR_ACCOUNT_ID</a>/query?nrql=YOUR_URL_ENCODED_QUERY -Headers @{"X-Query-Key"="YOUR_QUERY_KEY"} -ContentType "application/json" -Method GET
```

## 3. JSON応答の処理 [#process-result]

クエリAPIはJSON形式で結果を返します。リクエストごとの結果制限は5,000件です。

JSONデータの構造はリクエストで使用するNRQLによって決まり、`SELECT`文、句、関数の組み合わせに応じて適切な応答を返します。JSONを処理するコードを書く際は、クエリのテストを実行して結果のJSONを確認してください。

## レート制限のガイドライン [#rate-limiting]

[クエリのレート制限](/docs/query-your-data/nrql-new-relic-query-language/get-started/rate-limits-nrql-queries/)があります。特に、これらの一般ガイドラインに従う場合は、これらの制限に遭遇することはありません。

* 同時に実行される複雑なクエリ（例：`FACET`句や`TIMESERIES`句を含むクエリ、または100万件を超えるイベントのクエリ）でのリクエスト数を制限します。
* 特に複雑なクエリを含む場合は、長期間にわたって並行して実行するリクエストの数を最大5件に制限します。

New Relicによって1分あたりのクエリ数に対してレート制限がアカウントに適用されている場合、クエリAPIは429エラーを返します。New Relicによって検証されたレコードに対してレート制限がアカウントに適用されている場合、クエリAPIは503エラーを返し、チャートにタイムアウトエラーメッセージが表示されることがあります。
