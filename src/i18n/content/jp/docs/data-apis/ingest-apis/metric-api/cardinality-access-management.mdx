認証ドメイン マネージャーとして、次の手順を実行して、アカウントとメトリクス カーディナリティを管理するためのアクセスをユーザーに許可します。 ユーザーの具体的なニーズについて確認し、それに応じて設定を行ってください。 ユーザー管理の詳細については、[「ユーザー管理チュートリアル」](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/)を参照してください。

<Steps>
  <Step>
    ### カスタムロールに権限を追加する [#permissions-to-custom-role]

    次の権限を持つ[新しいカスタム ロールを作成するか](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles)、既存のロールに追加します。 必要なアクセス レベルをユーザーに確認してください。

    * `Manage account cardinality`: アカウントのカーディナリティ バジェットを調整します。

      * `View`: アカウントのカーディナリティ予算を確認できます。
      * `Modify`: アカウントのカーディナリティ バジェットを更新できます。

    * `Manage metric cardinality`: メトリクスのカーディナリティ バジェットを調整します。

      * `View`：メトリクスのカーディナリティバジェットを確認できます。
      * `Modify`: メトリクスのカーディナリティ バジェットを更新できます。

    * `Manage pruning rules`: メトリクスのプルーニング ルールを管理します。

      * `View`：メトリクスの枝刈りルールを確認できます。
      * `Modify`: メトリクスのプルーニング ルールを作成および削除できます。

    * `Customer limits management`: カーディナリティを含むアカウント内のさまざまな機能の制限を調整します。

      * `View`: 機能に使用可能な制限を確認できます。
      * `Modify`: 機能の制限を調整できます。
  </Step>

  <Step>
    ### ユーザーロールとアカウントをグループに追加する [#group-access]

    カーディナリティ管理については、次のように[新しいグループを作成する](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#group-access)か、既存のグループを管理します。

    1. **[New Relic Administration](https://one.newrelic.com/admin-portal)&gt; Access Management &gt; Groups**に移動します。
    2. <DNT>**Create a new group**</DNT>をクリックするか、 <DNT>**Manage group access**</DNT>を選択して<Icon name="fe-more-horizontal" />既存のグループのメニュー。
    3. 新しいグループの場合は、名前を入力して<DNT>**Create group**</DNT>をクリックします。 グループの<DNT>**Admin**</DNT>ペインが開きます。
    4. <DNT>**Members**</DNT>の下にユーザーを追加します。
    5. <DNT>**Account access**</DNT>の下で、カーディナリティ管理の目的のアカウントにカスタム ロールを関連付けます。
    6. カーディナリティ管理に複数のアカウントが関係する場合は、 <DNT>**Account access**</DNT>の下に、グループ内の関連するアカウントごとにカスタム ロールを含めます。
  </Step>
</Steps>
