New Relic では、お客様のデータを保存する New Relic データベースである [NRDB](/docs/telemetry-data-platform/ingest-manage-data/get-started/nrdb-horsepower-under-hood) を大変誇りに思っています。NRDB は、お客様のすべての遠隔測定データを一箇所に集め、すべてのデータを連結して見ることができ、お客様のビジネスの成長に合わせて拡張することができます。

サードパーティのソースからのものを含め、すべての[メトリック、イベント、ログ、およびトレース](/docs/data-apis/understand-data/new-relic-data-types)を NRDB に送信することをお勧めします。また、一部のデータはビジネス目標に必要ではないことも認識しています。取得する必要のないデータにたどり着く必要はありません。そして、あなたは間違いなくそれを支払う必要はありません.そこで、当社のデータ管理ツールの出番です。New Relic に送信するデータと、それを保存する期間を決定できます。

データマネジメントは、ユーザー管理ツールと組み合わせることで、データを保護しながら、New Relicへの投資から最大限の価値を引き出すことができます。

  まだNew Relicのお客様ではありませんか？ [サインアップ](https://newrelic.com/signup) わずか数秒で無料アカウントを作成できます。その後、毎月無料で最大100GBのデータを取り込むことができます。いつまでも。
</Callout>

## データマネジメントハブはどこにあるか [#find]

データ管理 UI を見つけるには: **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**から、[ユーザー メニュー](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)をクリックし、\[**データの管理] をクリックします**。

## コスト、パフォーマンス、コンプライアンスの向上 [#benefits]

New Relicでデータを収集・保存することで、すべてのソースからのメトリクス、イベント、ログ、トレースを分析、可視化し、アラートを出すことができます。しかし、コストやパフォーマンス、場合によってはコンプライアンス上の理由から、そのデータを管理することは重要です。当社のデータ管理ハブは、データがどこから来ているのかを理解し管理するために必要なツールを提供し、何をどのくらいの期間保存するかを調整します。

  当社の製品ベースのオリジナル価格モデル [をご利用の場合、データ管理ハブにデータの取り込み、保持、制限が表示されます。しかし、オリジナルの料金プランでは、データインジェストでは課金されません。どのプランかわからない場合は ](/docs/accounts/original-accounts-billing/product-based-pricing/overview-data-retention-components)[Overview of Pricing and User Model](/docs/transition-guide-our-new-pricing-plan-user-model) をご覧ください。

### コストマネジメント [#calculate-data-retention]

私たちのインジェスト・プロセスは、お客様のデータを磨くのに役立ちます。例えば、データは圧縮された状態で、さまざまな品質のまま処理の入り口に届くことがあります。インジェストによって、そのデータは圧縮解除され、クエリ可能な属性で装飾され、評価されます。NRDBに書き込む前に、要素を削除したり、トリミングしたりします。このようにして、あなたが保存するデータは、あなたが最も必要とするデータだけなのです。

データの取り込みとコストを見積もりたいですか？[データ取り込みの見積もり](https://newrelic.com/blog/nerdlog/estimate-data-cost)を参照してください。

### パフォーマンスマネジメント

NRDBは驚異的なスケーラビリティを持つデータベースですが、巨大なデータセットを対象としたクエリでは、ニーズに合ったタイミングで結果が返ってこない可能性があることも現実です。保存するデータを制限したり、簡単に照会できるような形式に変換したりすれば、パフォーマンスは向上します。 パフォーマンスを向上させるための1つの解決策は、データ [を削除して、保存されているデータの量を減らすことです。](/docs/accounts/accounts/data-management/drop-data-using-nerdgraph)
