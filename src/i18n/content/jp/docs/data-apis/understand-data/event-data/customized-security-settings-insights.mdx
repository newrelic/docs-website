デフォルトでは、New Relic 製品は UI チャートで使用されているさまざまなデータを報告しており、それらは [のクエリーで利用可能です](/docs/using-new-relic/data/understand-data/query-new-relic-data) 。当社の製品は、明示的にインストルメント化されていない限り、機密情報を送信することはありません。

## デフォルトのイベントと属性 [#default-events]

当社の製品は、 [デフォルトのイベントと属性のセットを報告します。](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products) 。誰かが設定で明示的に有効にしない限り、デフォルトのセットにないリクエストパラメーターやその他の属性を送信することはありません。

## 報告されたデータの調整 [#default-security]

New Relic 製品のセキュリティ設定を評価する際には、 [デフォルトのイベントと属性を確認してください](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products) 。デフォルトの属性には、機密データは含まれていません。一般的には、効果的なパフォーマンスモニタリングに必要なデータが入っているだけです。当社の製品は、お客様がデフォルトのセキュリティ設定を変更しない限り、他のデータを送信しません。

お客様のご要望に応じて、これらのどちらか、または両方の状況が適用されます。

* デフォルトのリストに気になるデータが含まれている場合は、その属性を収集しないようにすることができます。編集方法については、使用している製品のドキュメントを参照してください。
* デフォルトでは報告されない属性を送信する必要がある場合は、[それらの属性の報告を有効にする](/docs/insights/insights-data-sources/custom-data/send-custom-data-insights)ことができます。その場合は、[高セキュリティ モード](/docs/accounts-partnerships/accounts/security/high-security)を使用しないでください。これにより、カスタム属性を収集する機能が無効になります。
