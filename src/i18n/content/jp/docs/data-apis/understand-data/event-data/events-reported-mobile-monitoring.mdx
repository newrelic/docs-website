[<InlinePopover type="mobile"/>](/docs/mobile-monitoring/new-relic-mobile/getting-started/introduction-new-relic-mobile)は、一部のUIに表示され、[クエリとチャート作成](/docs/using-new-relic/data/understand-data/query-new-relic-data)にも使用できる[イベントデータ](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data)をレポートします。次の表からイベント名を選択し、その属性を確認します。

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        イベント
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`Mobile`](/attribute-dictionary/?event=Mobile)
      </td>

      <td>
        `Mobile`イベントは、クラッシュが発生した場合、インタラクションが終了または1秒間実行された場合、アプリ終了後セッションが完了、バックグラウンドで実行された、または10分間実行された場合に作成されます。`Mobile`イベントは、以前は唯一のイベントタイプですべてのイベントについて生成されましたが、現在はいくつかの分化したイベントタイプがあります。
      </td>
    </tr>

    <tr>
      <td>
        [`MobileApplicationExit`](/attribute-dictionary/?event=mobileApplicationExit)
      </td>

      <td>
        `MobileApplicationExit`イベントは、クラッシュやメモリ不足、ANRなどの理由によりアプリケーションが終了した場合に生成されます。このイベントではアプリケーションが終了した状況に関する重要情報が提供されます。アプリケーションが終了した理由、終了コード、説明などの属性が含まれます。

        現在、`MobileApplicationExit`イベントのレポートはAndroidモバイルアプリケーションのみでサポートされています。Androidの`ApplicationExitInfo` APIを使用して、アプリケーション終了の原因となったさまざまな理由を監視し、レポートします。
      </td>
    </tr>

    <tr>
      <td>
        `MobileBreadcrumb`
      </td>

      <td>
        このイベントの属性リストはありません。属性には、[セッション属性](#session-list)と追加された[カスタム属性を追加しました。](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile)
      </td>
    </tr>

    <tr>
      <td>
        [`MobileCrash`](/attribute-dictionary/?event=MobileCrash)
      </td>

      <td>
        `MobileCrash`イベントは、アプリがクラッシュすると作成されます。`MobileCrash`には、クラッシュの行番号、クラス、クラッシュメッセージなどの属性が含まれます。
      </td>
    </tr>

    <tr>
      <td>
        [`MobileHandledException`](/attribute-dictionary/?event=MobileHandledException)
      </td>

      <td>
        `MobileHandledException` は、例外が見つかり、[`recordHandledException` API呼び出し](/docs/mobile-monitoring/new-relic-mobile/mobile-sdk/record-handled-exceptions/)を使用して、New Relicに報告された非致命的な例外に使用する際に送信されます。

        例外は、このイベントのクエリと、スタックトレースなどの[<DNT>**Handled exceptions**</DNT> UIページ](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/handled-exceptions-analyze-trends-prevent-crashes)に表示されます。
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequest`](/attribute-dictionary/?event=MobileRequest)
      </td>

      <td>
        `MobileRequest`イベントは、HTTPリクエストが正常に完了し、400未満のレスポンスコードが返される場合に作成されます。

        New Relic `MobileRequest`データは、以下に対してデフォルトで有効化されます。

        * [Androidバージョン5.15.2](/docs/release-notes/mobile-release-notes/android-release-notes/android-5152)以上

        * [iOSバージョン6.0.0](/docs/release-notes/mobile-release-notes/ios-release-notes/ios-agent-600)以上

          Androidバージョン5.14.0またはiOSバージョン5.14.0で起動するそれ以前のバージョンについては、機能を有効にする必要があります。最新の[Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/upgrade-android-agent)または[iOS](/docs/mobile-monitoring/new-relic-mobile-ios/installation/upgrade-ios-agent)バージョンにアップグレード、または[Android](/docs/mobile-monitoring/new-relic-mobile-android/install-configure/android-agent-configuration-feature-flags)または[iOS](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/ios-agent-configuration-feature-flags#networkRequest)設定を使用して、`NetworkRequests`機能フラグを有効化します。
      </td>
    </tr>

    <tr>
      <td>
        [`MobileRequestError`](/attribute-dictionary/?event=MobileRequestError)
      </td>

      <td>
        `MobileRequestError`は、HTTPエラーまたはネットワーク障害に使用します。HTTPエラーとは、ステータスコードが400超のHTTPリクエストです。ネットワーク障害とは、HTTPリクエストで応答のないことです。イベントは、HTTPリクエストが完了すると送信されます。
      </td>
    </tr>

    <tr id="session-list">
      <td>
        [`MobileSession`](/attribute-dictionary/?event=MobileSession)
      </td>

      <td>
        `MobileSession`イベントは、アプリが終了、バックグラウンドで実行された場合、または10分間のアクティブ使用が経過すると送信されます。これが、その他のモバイルモニタリングイベントが使用する一般的なセッションデータのソースです。`MobileSession`は、デバイスのタイプ、デバイスのOS、地理情報などの属性を取得します。
      </td>
    </tr>

    <tr>
      <td>
        `MobileUserAction`
      </td>

      <td>
        このイベントの属性リストはありません。属性には、[セッション属性](#session-list)と追加された[カスタム属性を追加しました。](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile)

        このイベントは、ディストリビューティッド（分散）トレーシング機能が有効になっている際に、何らかのユーザーアクションによりエージェントが開始/フォアグラウンドに戻る、または停止/バックグラウンドに戻ると想定される場合に記録されます。
      </td>
    </tr>

    <tr>
      <td>
        カスタムイベント
      </td>

      <td>
        カスタムイベントのため、このイベントタイプの属性リストはありません。属性には、[セッション属性](#session-list)と[追加するカスタムアトリビュート](/docs/mobile-monitoring/new-relic-mobile/maintenance/add-custom-data-new-relic-mobile)が含まれます。
      </td>
    </tr>
  </tbody>
</table>
