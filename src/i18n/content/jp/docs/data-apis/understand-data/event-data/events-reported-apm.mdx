[APM](/docs/apm/new-relic-apm/getting-started)は、一部のUIディスプレイに表示され、[クエリとチャート作成](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data)にも使用できる[イベントデータ](/docs/using-new-relic/data/understand-data/query-new-relic-data) をレポートします。次の表からイベント名を選択し、その属性を確認します。

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        イベント
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td id="transaction-event">
        [`Transaction`](/attribute-dictionary/?event=Transaction)
      </td>

      <td>
        トランザクションは、ソフトウェアアプリケーションにおける 1 つの論理単位としての動作です。`Transaction`イベントには、アプリ、データベース呼び出し、トランザクションの所要時間、発生するエラーに関する情報が含まれます。
      </td>
    </tr>

    <tr>
      <td id="transactionerror-event">
        [`TransactionError`](/attribute-dictionary/?event=TransactionError)
      </td>

      <td>
        トランザクションエラーは、リクエストが、そのリクエストを完遂するためにコードパスに例外を投げた場合に発生します。トランザクションエラーの数とトランザクションの数は一致しません。これは、エラーを[収集するか、無視するか、あるいは予期していたとして記録](/docs/agents/manage-apm-agents/agent-data/manage-errors-apm-collect-ignore-or-mark-expected)するかを指定できるためです。
      </td>
    </tr>

    <tr>
      <td id="span-event">
        [`Span`](/attribute-dictionary/?event=Span)
      </td>

      <td>
        スパンイベントは、New Relicエージェントまたはその他のトレーシングインストゥルメンテーションツールが、分散トレースの一部である演算子をインストゥルメントする際に発生します。[スパンの定義](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#span)を参照してください。
      </td>
    </tr>
  </tbody>
</table>
