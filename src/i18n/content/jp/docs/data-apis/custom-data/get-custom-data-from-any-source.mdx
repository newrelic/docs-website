New Relicを最大限に活用するには、カスタムデータを報告する必要があるか、報告する必要があります。あらゆるタイプのデータをNewRelicに取り込むために必要なツールを提供します。

## まず、より厳選されたソリューションをご覧ください [#options]

New Relic製品は、多くのデータを自動的に報告します。追加の構成は必要ありません。APM、ブラウザモニタリング、モバイルモニタリング、インフラストラクチャモニタリング、またはさまざまな[統合とクイックスタート](https://newrelic.com/instant-observability)などのモニタリングソリューションを使用すると、デフォルトでさまざまなタイプのパフォーマンスデータを受け取ります。

ただし、デフォルトでは収集されないデータをNewRelicに取り込むことをお勧めします。エージェントのインストールを必要としないAPIベースのソリューションが必要な場合があります。別の分析サービスからのテレメトリデータをNewRelicに持ち込みたいと思うかもしれません。以下では、より多くのカスタムデータセットをNewRelicに取り込むためのいくつかの方法について説明します。

## エージェントAPIを使用してカスタムデータをレポートする [#agent-apis]

New Relicエージェント（APM、ブラウザー、またはモバイル）を使用している場合は、カスタムデータを送信するようにエージェントを構成できます。詳細については、使用しているエージェントのドキュメントを参照してください。

## カスタムインフラストラクチャデータを報告する [#flex-integration]

インフラストラクチャ統合がニーズを満たさない場合は、 [Flex統合](/docs/integrations/host-integrations/host-integrations-list/flex-integration-tool-build-your-own-integration)を使用して、あらゆるタイプのインフラストラクチャ関連のメトリックまたは構成設定をNewRelicに報告できます。

中規模から大規模の企業組織であり、 [SAP®](https://www.sap.com/)テクノロジーを使用して事業運営と顧客関係を管理したい場合は、 [SAP統合](/docs/data-apis/custom-data/sap-integration)を使用できます。

## オープンソースのテレメトリサービスと仕様からのレポートデータ [#third-party-telemetry]

[クイックスタートページ](https://newrelic.com/instant-observability)ですべての統合を確認できます。サードパーティのオープンソーステレメトリサービスおよび仕様とのより一般的な統合の一部を次に示します。

* [OpenTelemetry](/docs/more-integrations/open-source-telemetry-integrations/opentelemetry/introduction-opentelemetry-new-relic)
* [Prometheus](/docs/infrastructure/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* [ジプキン](/docs/distributed-tracing/trace-api/report-zipkin-format-traces-trace-api)
* [StatsD](/docs/infrastructure/host-integrations/host-integrations-list/statsd-monitoring-integration-version-2)
* [Grafana](/docs/more-integrations/grafana-integrations/get-started/grafana-support-prometheus-promql)
* [DropWizard](/docs/more-integrations/open-source-telemetry-integrations/dropwizard/dropwizard-reporter)
* [マイクロメータ](/docs/more-integrations/open-source-telemetry-integrations/micrometer/micrometer-metrics-registry)

## カスタム属性とイベントを報告する [#events]

NewRelicで報告されたイベントをカスタムメタデータで装飾できます。詳細については、[カスタム属性](/docs/data-apis/custom-data/custom-events/collect-custom-attributes)を参照してください。

[Event API](/docs/data-apis/ingest-apis/event-api/introduction-event-api)を使用して、完全にカスタムのイベントをNewRelicに報告することもできます。

その後、カスタム属性とイベントを照会してグラフ化できます。

## カスタムトレースデータを報告する [#tracing]

[Trace APIを](/docs/distributed-tracing/trace-api/introduction-trace-api)使用すると、分散トレースデータをNew Relicに送信し、複数のソースからのトレースデータを1か所に統合できます。トレースデータは次の2つの形式で受け入れます。

* Zipkin形式
* New Relic形式（Zipkin形式のデータがない場合は、これを使用します）

## カスタムログデータを報告する [#logs]

既存の[ログ統合](/docs/logs/get-started/get-started-log-management)がニーズを満たさない場合は、 [LogAPI](/docs/logs/log-api/introduction-log-api)を使用して任意のログデータをNewRelicに送信できます。

## カスタムデータの視覚化を作成する [#visualizations]

カスタムクエリ、チャート、ダッシュボードの作成に興味がありますか？クエリを開始するには、 [NRQLの概要を](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language)参照してください。クエリを実行すると、結果のグラフを取得し[てダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards)に追加できます。

当社のチャートよりも強力でカスタマイズが必要な場合 <InlinePopover type="dashboards"/>また、独自の [New Relic アプリ](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/new-relic-one-build-your-own-custom-new-relic-one-application)を作成したり、 [アプリ カタログ](https://opensource.newrelic.com/nerdpacks)から既存のアプリをカスタマイズしたりすることもできます。New Relic アプリは当社のプラットフォーム上にあり、チームや一般ユーザーと共有できます。
