[AWS PrivateLink](https://aws.amazon.com/privatelink)を介して、AWS Virtual Private Cloud（VPC）からNew Relicにテレメトリーデータを送信できます。

## 概要 [#overview]

AWS PrivateLink経由でNew Relicのデータを送信すると、以下のことが可能になります。

* AWSの送信コストを削減できます
* データをAmazonネットワーク内に保持することで、セキュリティを強化できます

PrivateLinkを利用することで、[テレメトリーデータ](/docs/data-apis/understand-data/new-relic-data-types)を各種のNew Relic[エンドポイント](#endpoints)に送信できます。なお、[NerdGraph API](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph)やその他の非取り込み型APIには対応していません。

この機能を使用するには、ローカルのNew RelicエージェントとインテグレーションがAWS PrivateLinkを介してNew Relicにデータをルーティングするために使用する[インターフェースエンドポイント](#create)をVPC内に設定する必要があります。

## 要件 [#requirements]

PrivateLinkを利用するには、[New Relic Data Plus](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/data-ingest-billing/#data-plus)オプションが必要です。Data Plusを使用せずにデータを送信しようとすると、PrivateLink経由で送信されたトラフィックは拒否され、402エラーが返されます。

エンドポイントサービスは、作成されたリージョンでのみ利用可能です（[Amazonのドキュメントを参照](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)）。ただし、リージョン間ピアリングや[AWS PrivateLink向けのリージョン間接続](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)を使用することで、他のリージョンからもアクセスできます。

VPCピアリングおよびリージョン間接続を利用する場合、リージョン間のデータ転送コストは利用者の負担となります。

つまり、VPCがサポート対象のいずれかのリージョン（USアカウントの場合は`us-east-1`、`us-east-2`、または`us-west-2`；EUアカウントの場合は`eu-central-1`；JPアカウントの場合は`ap-northeast-1`）にある場合は、必要な内部VPCエンドポイントを作成するだけで利用可能です（詳細は後述）。VPCがサポート対象外のリージョンにある場合は、[AWS PrivateLinkのクロスリージョン接続](#cross-region)を使用するか、[ピアリング接続をセットアップ](#peering)することをお勧めします。

## クロスリージョン接続 [#cross-region]

New Relicは[AWS PrivateLinkのクロスリージョン接続](https://aws.amazon.com/blogs/networking-and-content-delivery/introducing-cross-region-connectivity-for-aws-privatelink/)をサポートしており、VPCピアリングをセットアップすることなく、任意のAWSリージョンからNew Relicエンドポイントに接続できます。これは、VPCが[サポート対象のリージョンとゾーン](#regions-zones)にリストされている以外のリージョンにある場合に推奨されるアプローチです。

## ピアリング接続を設定する [#peering]

[サポートされているリージョン](#regions-zones)外のリージョンから接続するために、VPCピアリングは不要になりました。代わりに、[AWS PrivateLinkのクロスリージョン接続](#cross-region)を使用することをお勧めします。引き続きVPCピアリングを使用する場合は、リージョン間のデータ転送コストはお客様の負担となることに注意してください。

### VPCピアリング [#vpc-peering]

[リージョン間VPCピアリング接続の作成手順](https://docs.aws.amazon.com/vpc/latest/peering/create-vpc-peering-connection.html)に従って、作成したエンドポイント接続のVPC IDを指定してください。

### Transit Gatewayのピアリング [#transit-gateway-peering]

[Route 53 PHZを作成してVPC間で共有する手順](https://aws.amazon.com/blogs/networking-and-content-delivery/integrating-aws-transit-gateway-with-aws-privatelink-and-amazon-route-53-resolver)に従い、作成したエンドポイント接続に対応するVPC IDを指定してください。
