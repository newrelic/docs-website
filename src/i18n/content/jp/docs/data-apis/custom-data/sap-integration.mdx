[SAP®](https://www.sap.com/)テクノロジーは、主要なビジネスプロセスを推進するために不可欠です。SAPシステムに関連する問題は、非効率性と中断を引き起こし、特定して解決するのに時間と費用がかかる可能性があります。これは、収益を生み出したり、製品やサービスを提供したりする能力に影響を与える可能性があります。

New Relicを使用すると、インフラストラクチャ、アプリケーション、およびビジネスプロセスを一元的に表示してSAPシステムを監視できます。これにより、ITチームは、早期の認識と迅速な解決により、ビジネスをより適切にサポートできます。

## 制限と要件 [#requirements]

この機能を使用するには、 [ワークロード関連の機能](/docs/accounts/accounts-billing/new-relic-one-user-management/user-capabilities/#capability-definitions)を持つ完全なプラットフォーム ユーザーである必要があります。

New Relic で監視できる SAP システムの数は、[価格エディション](https://newrelic.com/pricing)によって異なります。制限は、SAP 本番システムか非本番システムかにかかわらず、 [New Relic 組織](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure)によって監視されるすべての SAP システムに適用されます。

New Relic で監視できる一意の SAP システム ID の数の制限は次のとおりです。

* 無料利用枠: 0、利用不可
* 通常版：月1冊
* Pro エディション: 1 か月あたり 5 つ
* Enterprise エディション: 1 か月あたり 10 個

制限を超えた場合、またはより高い制限が必要な場合は、New Relic アカウント担当者にお問い合わせください。

## NewRelicSAP統合の仕組み [#how]

NewRelicとSAPSEの統合を使用して、次のことを行います。

* **複数のソースからデータを取得します。**組織内の複数のSAPシステム、および非SAPシステムまたは個々のコンポーネントからのデータを提示します。
* **パフォーマンスと全体的なヘルスデータを1か所で確認できます。**テレメトリメトリック、イベント、ログ、およびトレースはすべて、UI内の中央の共通ビューにまとめられます。
* **関連データをビジネスプロセスに関連付けます。**対応するビジネスプロセスの観点からインフラストラクチャとアプリケーションを監視できるように、データにタグを付けます。
* **データを視覚化し、チームと共有します。**カスタマイズ可能なチャートとダッシュボードで関連データをクエリして表示することにより、ITチームは根本原因をより迅速に特定して問題を解決できます。
* **問題が発生する前に防止してください。**高度なアラート機能を使用して、問題が顧客に影響を与える前に通知を受け取ります。

## SAP統合を選択する理由 [#why]

中規模から大規模の企業組織の場合、ビジネスオペレーションと顧客関係を管理するための信頼性の高いSAPツールが必要です。New RelicSAP統合を選択する理由

### 搭載が簡単

SAPとの認定済み統合は、SAPアドオンとしてインストールおよび試用が簡単です。さらに、異常検出とアラートに加えて、ダッシュボード、トレース、およびマップのターンキー視覚化を提供します。SAPソリューションのNewRelicモニタリングは、SAPインフラストラクチャ、アプリケーション、およびビジネスプロセス向けのオールインワンの一元化された可観測性ツールです。

### 既存のSAP投資を活用します

他のベンダーとは異なり、New Relicはサードパーティのコネクタに依存していません。サードパーティのコネクタでは、通常、SAP本番サーバーに監視エージェントをインストールする必要があります。代わりに、SAPのNew Relicモニタリングは、既存のSAPデータソースを利用して必要なすべてのテレメトリデータにアクセスするため、非侵襲的です。これが、既存のSAP投資を活用するソリューションがSAPサポートチームに採用されている理由です。

### バージョン固有の更新は必要ありません

一部のSAPソリューションは、特定のバージョンまたは状況にのみ適用されるため、時間のかかる頻繁な更新と調整が発生します。これらのソリューションとは異なり、SAP統合は、NetWeaverレイヤーで低レベルのデータソースを使用するため、アップグレード後も存続します。
