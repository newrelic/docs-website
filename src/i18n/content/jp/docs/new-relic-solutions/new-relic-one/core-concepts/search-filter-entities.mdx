エンティティ フィルター バーは、 New Relicのインターフェイス全体で調べたいモニター[エンティティ](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic)を検索してフィルターするのに役立ちます。

<img title="Entity filter bar in Catalogs" alt="Current entity filter bar interface showing search and filter options" src="/images/Catalogs.webp" />

## フィルターバーを見つける [#find]

エンティティ フィルター バーは、次のような主要な New Relic ページの上部に表示されます。

* <DNT>**[Catalogs](/docs/service-architecture-intelligence/catalogs/catalogs/)**</DNT> <DNT>**Applications**</DNT> 、 <DNT>**Repositories**</DNT> 、 <DNT>**Infrastructure**</DNT>のページ。
* <DNT>**All Entities**</DNT> ページ。
* APM サービス ページ。
* <InlinePopover type="browser" />ページ。
* インフラストラクチャ監視ページなど。

## 基本的な検索とフィルタリング [#basic-filtering]

エンティティを見つけるには、まず次の基本的なフィルタリング方法から始めます。

### エンティティ名で検索

エンティティを検索するには、 <DNT>**Search by entity name**</DNT>フィールドを使用します。

1. 検索フィールドをクリックします。
2. エンティティ名の一部を入力します (例: 「production」または「web-app」)。
3. 一致する結果は下のリストに即座に表示されます。

<Callout variant="tip">
  検索ではエンティティ名の任意の部分が一致するため、「prod」と入力すると、「prod-web-server」や「my-production-app」などの名前のエンティティが検索されます。
</Callout>

### エンティティタイプでフィルタリング

特定のエンティティ タイプを選択して結果を絞り込みます。

* <DNT>**Entity Type**</DNT>: アプリケーション、サービス、ホスト、コンテナ、およびその他の監視リソース タイプから選択します。
* **結果:**選択したタイプに一致するエンティティのみが表示されるため、特定のインフラストラクチャ コンポーネントに焦点を絞りやすくなります。

### チームの所有権でフィルタリング

チームが管理するエンティティに焦点を当てます。

* <DNT>**Team**</DNT>: 現在のビュー内のエンティティの所有権を持つチームを選択します。
* **要件:** [Service Architecture Intelligence](/docs/service-architecture-intelligence/getting-started/)を有効にする必要があります。
* **結果:**選択したチームに割り当てられている、または選択したチームが所有するエンティティのみを表示します。

## 高度なフィルタリング [#advanced-filtering]

より複雑なフィルタリング シナリオの場合は、カスタム フィルタを作成します。

### カスタムアトリビュートフィルターを作成する

エンティティ属性に基づいて特定のフィルターを追加します。

1. 既存のフィルターの横にある**+** をクリックします。

2. <DNT>**Filter**</DNT>ドロップダウンから、属性（環境、リージョン、カスタム タグなど）を選択します。

3. 演算子を選択してください:

   * **=** (等しい): 完全一致
   * **!=** (等しくない): 完全一致を除外する
   * **LIKE** : 指定されたテキストを含む
   * **NOT LIKE** : 指定されたテキストを除外する

4. フィルターを完了するには、値を入力または選択します。

5. フィルターを適用するには、 <DNT>**Add**</DNT>をクリックします。

**例:**すべての本番 Web アプリケーションを検索するには、 where `environment = production` AND `entity_type = APPLICATION`でフィルタリングします。

より高度なクエリと分析については、 [NRQL書き込みビルダーの](https://one.newrelic.com/data-exploration/query-builder)使用を検討してください。 NRQL クエリの詳細については、 [「NRQL の使用方法」](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-how-nrql-works)を参照してください。
