標準ワークロードでは、クエリ、タグ、または明示的な選択を使用して、関連するエンティティを手動でグループ化できます。このドキュメントでは、標準ワークロードを作成および構成する方法について説明します。ディストリビューティッド（分散）トレーシングに基づく自動エンティティ検出が必要な場合は、代わりに[インテリジェント ワークロード](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview#intelligent-workloads)を参照してください。

<Callout variant="tip">
  どのタイプを使用すればよいかわかりませんか?[ワークロードの概要](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview)を確認して、標準ワークロードとインテリジェント ワークロードを比較してください。
</Callout>

標準ワークロードに追加するエンティティは、目標と組織構造によって異なります。ワークロードでチームを調整する方法については、[ワークロード チュートリアル](/docs/tutorial-peak-demand/organize-data-workloads)で読むこともできます。

## アカウントと権限 [#accounts]

ワークロードに寄与するアカウントには、 <DNT>**workload account**</DNT>と<DNT>**scope accounts**</DNT>の 2 種類があります。[New Relic組織](/docs/accounts/accounts-billing/account-structure/new-relic-account-structure)に複数のアカウントがある場合、ワークロード アカウントとスコープ アカウントを区別すると、ワークロードの管理が容易になります。

* <DNT>**workload account**</DNT>はワークロードとそのデータが存在する場所です。ワークロードを作成するために使用されるアカウントは、ワークロード アカウントと見なされます。このアカウントは、割り当てられたアカウント権限に基づいて、どのユーザーがワークロードを表示および管理できるかを決定します。
* <DNT>**scope accounts**</DNT>は、ワークロードがテレメトリーデータを取得するアカウントです。 ワークロードのすべてのスコープ アカウントへのアクセス権を持たないユーザーは、ワークロード内のすべてのデータを表示できない可能性があります。ワークロードを使用するチーム メンバーの場合は、ワークロードがデータを取得するチームの関連アカウントを必ず追加してください。

ワークロード アカウントは、作成後に変更することはできません。ただし、スコープ アカウントは、ワークロード アカウントのワークロード管理機能を持つユーザーであればいつでも更新できます。デフォルトでは、ワークロード作成時にワークロード作成者がアクセスできるすべてのアカウントがスコープ アカウントとして設定されます。

## 標準ワークロードを作成する [#create]

標準ワークロードでは、含めるエンティティを完全に制御できます。エンティティ、タグを使用するか、特定のエンティティを選択することによって、メンバーシップを定義します。 トレースからエンティティを自動的に検出する[インテリジェント ワークロード](/docs/new-relic-solutions/new-relic-one/workloads/create-intelligent-workload)とは異なり、標準ワークロードは、クエリが新しいエンティティに一致するか、手動で変更した場合にのみ更新されます。

標準ワークロードを作成するにはいくつかの方法がありますが、どれを選択するかは完全にあなた次第です。チームの調整に最も便利な方法を選択することをお勧めします。

    ワークロードUIから標準ワークロードを作成する方法は次のとおりです。

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Workloads**</DNT>に移動し、 <DNT>**+ Create a workload**</DNT>をクリックします。

    2. ワークロードに、あなたとあなたのチームにとって意味のある名前を付けます。

    3. <DNT>**Select an account**</DNT>ドロップダウンから、使用する[ワークロード アカウント](#accounts)を選択します。

    4. [\[範囲アカウントの選択\]](#scope-accounts) をクリックして、このワークロードがデータを使用するすべてのアカウントを確認します。

    5. ワークロードを構成するエンティティを検索して選択します。探している結果が得られたら、特定のエンティティを追加したり、ワークロードのエンティティを動的に更新するクエリを追加したりすることができます。

       * エンティティ タイプ、 [タグ](#tags)、または [属性](/docs/new-relic-solutions/get-started/glossary#attribute) (エンティティ名、アカウント ID、AWS リージョンなど) で検索できます。
       * <DNT>**+ Add this query**</DNT>をクリックすると、ワークロードに対して動的に更新されるエンティティのリストが作成されます。システムの変更に応じてワークロードのエンティティを更新する場合は、これをお勧めします。
       * エンティティの横にある<DNT>**+ Add**</DNT>クリックして、そのエンティティをワークロードに追加します。システムが変更されてもエンティティが引き続き有用であることがわかっている場合は、これが適切な選択です。

    6. クエリと特定のエンティティの両方をワークロードに追加でき、 [クエリ ロジック](#query-logic)に従って連携して動作します。

    7. 保存するには<DNT>**Create a workload**</DNT>をクリックしてください。

    ワークロードに1つ以上のダッシュボードが含まれている場合、 [それらのダッシュボードのリンクにフィルターを設定することができます](#filter-dashboards) 。

## ワークロードを定義する [#define]

以下に、ワークロードの定義のいくつかの側面に関する詳細とヒントを示します。

<CollapserGroup>
  <Collapser id="tags" title="タグを使ってワークロードの内容を定義">
    [タグ](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) と [属性の](/docs/new-relic-solutions/get-started/glossary#attribute)両方を使用して、ワークロード エンティティをクエリおよび選択できます。したがって、ワークロードの使用を最適化するには、適切なエンティティのタグ付け戦略を立てることが役立ちます。 [タグ付けに関するドキュメント](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor) を読み、 [タグを使用してワークロードを定義する方法についてのこのビデオを](https://www.youtube.com/watch?v=nXCH_3XjGkQ)視聴することをお勧めします。

    <Video id="nXCH_3XjGkQ" type="youtube" />
  </Collapser>

  <Collapser id="query-logic" title="ダイナミッククエリロジックの仕組み">
    複数の個別のエンティティやクエリを追加してワークロードを定義することができます。

    * クエリには、複数の検索語を含めることができます。これらはAND演算子で結合されます。
    * ワークロード内の別々のクエリは、OR演算子で結合されます。
    * パーセント記号（ `%` ）で文字列をラップして、クエリ内の正確な部分文字列に一致させることができます。エンティティ名にサブストリングを使用してそれらのエンティティを分類する場合（たとえば、 `<team>-<env>-<appName>` ）、フィルタリングとグループ化に強力な[タグ](/docs/new-relic-one/use-new-relic-one/core-concepts/tagging-use-tags-organize-group-what-you-monitor)（たとえば、 `team:awesome` 、 `env:production` ）を補完的に使用することを検討してください。
    * 500を超えるエンティティを返す可能性のある動的クエリでは、パーセント記号（ `%` ）を使用しないことをお勧めします。このようにして、ユーザーインターフェイスでより一貫したエクスペリエンスを得ることができます。
  </Collapser>

  <Collapser id="add-dashboards" title="ダッシュボードをワークロードに追加">
    [カスタムダッシュボード](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards)をお持ちで、チームのワークロードの観察と運用にどのデータが関連しているかがすでにわかっている場合は、ワークロードからそれらの<InlinePopover type="dashboards" />をリンクできます。

    ワークロードにダッシュボードを追加するには: ワークロードを作成した後、ワークロードの<DNT>**Activity**</DNT>ページに移動し、 <DNT>**Add dashboard**</DNT>をクリックします。
  </Collapser>
</CollapserGroup>
