インテリジェント ワークロードは、ディストリビューティッド（分散）トレーシングを使用して、現場フロー内のすべてのエンティティを自動的に追跡します。 [標準のワークロード](/docs/new-relic-solutions/new-relic-one/workloads/create-workloads)とは異なり、インテリジェント ワークロードはディストリビューティッド（分散）トレーシングを使用してエンティティを検出し、システムの進化に応じて 5 分ごとにメンバーシップを更新します。

## あなたが始める前に [#prereqs]

[インテリジェント ワークロードの要件](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview#intelligent-workloads)を満たしていることを確認します。

<Callout variant="tip">
  ワークロードを初めて使用する場合は、[ワークロードの概要](/docs/new-relic-solutions/new-relic-one/workloads/workloads-overview)を確認して、インテリジェント ワークロードと標準ワークロードをいつ使用するかを理解してください。
</Callout>

## インテリジェントなワークロードを作成する [#create]

インテリジェントなワークロードは 3 つの方法で作成できます。ワークフローに最も便利な方法を選択してください。

<CollapserGroup>
  <Collapser id="create-from-recommendations" title="推奨ワークロードから作成する">
    New Relic は、アプリケーションのトランザクションを自動的に分析し、作成すべき価値の高いインテリジェント ワークロードを推奨します。これは、インテリジェント ワークロードを開始するための最も早い方法です。

    ### おすすめを見る

    ワークロード ページに移動すると、 New Relic推奨ワークロードを見つけた場合は、上部にバナーが表示されます。

    ```
    We found [X] recommended Intelligent Workloads. We made it easy to review our recommendations and set up Intelligent Workloads quickly.
    ```

    おすすめモーダルを開くには、 <DNT>**Review recommendations**</DNT>をクリックします。

    ### 推奨事項を確認して作成する

    **Recommended Intelligent Workloads** \[推奨されるインテリジェント ワークロード]モーダルには、監視によってメリットが得られるトランザクションが表示されます。

    1. **カテゴリ別にフィルタリング**: タブを使用して切り替えます:

       * **Key Transactions** \[キースイッチ]: 高トラフィックまたはビジネスクリティカルなトランザクション
       * **Impacted Transactions** \[影響を受けたトランザクション]: パフォーマンスの問題またはエラーが発生したトランザクション

    2. **おすすめを並べ替える**: ドロップダウンを使用して並べ替えます。

       * 最も高いエラー率
       * 平均応答時間が最も長い
       * 最も時間がかかる

    3. **レビューの詳細**: 各推奨事項は次の内容を示します。

       * **Transaction name** \[場面名]: ワークロードの中心的場面
       * **Focal entity** \[フォーカルエンティティ]: トランザクションを処理する主要なサービス
       * **Response Time** \[応答時間]: 平均レイテンシ (例: &quot;0.57s&quot;)
       * **Error Rate** \[エラー率]: 失敗したrequestsの割合 (例: &quot;0.6%&quot;)
       * **Total Duration** \[合計所要時間]: 合計処理時間 (例: 「77.60 秒」)

    4. **ワークロードの作成**: 推奨事項の横にある<DNT>**Create**</DNT>をクリックします

    推奨事項から作成する場合、New Relic によってエンティティとトランザクションの選択が自動的に事前入力され、**Add details** \[詳細の追加]手順に直接移動して次の操作を実行できます。

    * ワークロードに名前を付ける
    * 説明を追加する（オプション）
    * 焦点となるサービスとトランザクションを確認する
    * セットアップを完了する

    <Callout variant="tip">
      推奨事項から作成する方法は、インテリジェント ワークロードを初めて使用するときや、専用の監視によって最もメリットが得られるトランザクションをすばやく特定したいときに最適です。
    </Callout>
  </Collapser>

  <Collapser id="create-workloads-page" title="ワークロードページから作成">
    ワークロード ページには、インテリジェント ワークロードの作成に役立つガイド付きセットアップ ウィザードが用意されています。

    1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Workloads**</DNT>に移動します。
    2. <DNT>**Create a workload**</DNT>をクリックします。
    3. <DNT>**Intelligent Workload**</DNT>タイプを選択してください。
    4. ウィザードを開始するには、 <DNT>**Setup**</DNT>をクリックします。

    <img title="Select Intelligent Workload type" alt="Screenshot showing workload type selection with Standard and Intelligent options" src="/images/create_intelligent_workload.webp" />

    ### ステップ1: エンティティを選択する [#select-entity]

    重要なトランザクションが含まれるサービスまたはエンティティを選択します。

    1. 名前でエンティティを検索または参照します。
    2. 必要に応じて、エンティティ タイプ (APM サービス、ブラウザー アプリなど) でフィルターします。
    3. キートランザクションを処理するエンティティを選択します。
    4. <DNT>**Next**</DNT>をクリックします。

    <Callout variant="tip">
      トランザクション フローの中心となるエンティティを選択します。たとえば、チェックアウト プロセスを監視する場合は、周辺ログ サービスではなくチェックアウト サービスを選択します。
    </Callout>

    ### ステップ 2: あなたの場面を選択してください [#pick-transaction]

    監視したい特定の場面を選択してください:

    1. 選択したエンティティの取引リストを参照します
    2. 検索を使用して特定の場所名を見つける
    3. ビジネスワークフローを表すトランザクションを選択します
    4. クリック <DNT>**Next**</DNT>

    ワークロードは、ディストリビューティッド（分散）トレーシングに基づいて、この行為に参加するすべてのエンティティを追跡します。

    ### ステップ3: 詳細を追加する [#add-details]

    ワークロードに関する情報を提供します。

    1. **Name** \[名前]: 分かりやすい名前を入力します (例: 「Checkout」、「ユーザー認証フロー」)
    2. **Description** \[説明]\(オプション): このワークロード モニターの内容とそれがなぜ重要なのかについてのコンテキストを追加します。
    3. **Account** \[アカウント]: ワークロードを作成するアカウントを選択します
    4. クリック <DNT>**Next**</DNT>

    ### ステップ4：プレビューと最終決定 [#preview-finalize]

    ワークロードの設定を確認します。

    1. **Focalサービス**が正しいことを確認する
    2. **Focal が意図したワークフロー**と一致していることを確認します
    3. ワークロードの**Name \[名前]**と**Description** \[説明]を確認する
    4. クリック <DNT>**Create Workload**</DNT>

    インテリジェント ワークロードは、トランザクションに参加しているエンティティの追跡を直ちに開始します。
  </Collapser>

  <Collapser id="create-from-t360" title="Transaction 360から作成">
    Transaction 360 ですでにトランザクションを表示している場合は、それをインテリジェント ワークロードとして簡単に保存できます。

    1. に行く <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; APM &amp; Services**</DNT>
    2. 重要な現場フローの一部であるサービスを選択してください

    <img title="Select entity for Intelligent Workload" alt="Screenshot showing entity selection step in Intelligent Workload creation" src="/images/intelligent-workload-select-entity.webp" />

    3. 監視したい特定の場面に移動し、 <DNT>**Transaction 360**</DNT>をクリックします。
    4. <DNT>**Save as intelligent workload**</DNT>をクリックします。
    5. ワークロードのわかりやすい名前を入力します。
    6. <DNT>**Save**</DNT>をクリックします。

    <Callout variant="tip">
      個々のAPIコールではなく、完全なビジネス ワークフローを表す名前 (「チェックアウト」、「ユーザー認証」など) を選択してください。
    </Callout>
  </Collapser>
</CollapserGroup>
