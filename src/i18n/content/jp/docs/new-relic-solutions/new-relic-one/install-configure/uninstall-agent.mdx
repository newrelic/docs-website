[](https://newrelic.com/instant-observability/)New Relic ソリューションの中には、エージェントのインストールを必要とするものがあります。エージェントがデータを報告しないようにしたい場合は、エージェントをアンインストールすることができます。New Relic を完全にアンインストールする必要がある場合は、お使いのエージェントの手順に従ってください。

問題が発生した場合は、 [support.newrelic.com](https://support.newrelic.com) をご覧ください。

## アンインストールの前に [#downgrade]

<Callout variant="important">
  New Relic の有料組織の場合、アンインストールの代わりに、下位エディションに [ダウングレードする](/docs/accounts/accounts-billing/account-setup/downgradecancel-account) こともできます。
</Callout>

## APMのアンインストール [#language-agents]

アンインストール後 <InlinePopover type="apm"/>エージェントを変更したり、起動スクリプトを変更したりした場合は、アプリ (またはアプリ サーバーまたはアプリが配置されている Web サーバー) を再起動する必要があります。 通常、New Relic エージェントはアプリを実行しているプロセスのメモリ内で実行されるため、これを行う必要があります。これらのファイルを単に削除しても、現在実行中のアプリケーションが New Relic にデータを送信するのを止めることはでき **ません** 。

* [Go](/docs/agents/go-agent/installation-configuration/uninstall-go-agent)
* [Java](/docs/agents/java-agent/installation/uninstall-java-agent)
* [.NET](/docs/agents/net-agent/installation-configuration/uninstall-net-agent)
* [Node.js](/docs/agents/nodejs-agent/installation-configuration/uninstall-nodejs-agent)
* [PHP](/docs/agents/php-agent/advanced-installation/uninstalling-php-agent)
* [Python](/docs/agents/python-agent/installation-configuration/uninstall-python-agent)
* [Ruby](/docs/agents/ruby-agent/installation-configuration/uninstall-ruby-agent)
