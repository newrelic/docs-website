一緒に仕事をする全員が New Relic ユーザーというわけではありませんが、チームメイトのほとんどはおそらく Slack を使用しています。New Relic データについて全員で議論し、Slack と New Relic の議論の間ですべての洞察を同期させることができます。

Slack への更新を一方向で共有するだけの多くの Slack 統合とは異なり、New Relic の会話スレッドは Slack のスレッドと双方向で同期されます。New Relic から Slack に質問、意見、または支援のリクエストを共有すると、Slack のチームメイトは、New Relic ユーザーであるかどうかにかかわらず、Slack の投稿に返信でき、これらの返信はスレッドに戻されます。ニューレリックで。同様に、New Relic のスレッドに追加された返信は、Slack のスレッドに追加されます。この双方向の同期により、より多くの人を会話に参加させ、New Relic でコンテキストとともに議論をキャプチャし、データを使用して意思決定を通知し、組織の知識を構築することをすべて同時に行うことができます。

## インストール [#install]

New Relic ディスカッション Slack アプリは、まだ Slack App ディレクトリにありません。Slack ワークスペースの**アプリ管理設定**によっては、インストールのために Slack 管理者の支援が必要になる場合があります。

設定が完了したら、Slack ワークスペースの管理者は、\[Slack App ディレクトリからのアプリのみを許可する] をオフにして、\[メンバーがアプリの承認を要求できるようにする] をオンにする必要がある場合があります。その後、以下の認証手順に進むことができます。これらの指示に従うと、管理者がアプリケーションを承認するためのリクエストが生成されます。承認されると、Slack アプリをインストールでき、一時的に変更された設定を復元できます。

## 認証 [#authenticate]

New Relic Discussions Slack アプリケーションを使用するには、New Relic 管理者 (または Applied Intelligence Destination Create 機能を持つチームメイト) が最初に Slack で認証を受ける必要があります。これを開始するには、「Sync to Slack」アイコンをクリックして、slack.com に移動し、使用したいワークスペースにログインします。

ワークスペースが設定されると、New Relic オーガナイゼーションのすべてのメンバーがそのワークスペースに対して認証を行い、新しいディスカッション スレッドを Slack に共有できるようになります。新しいユーザーが Slack と同期できるようにするには、まず New Relic アカウント管理者が同期を完了する必要があることに注意してください。

組織に複数のアカウントがある場合、Slack 資格情報は特定の各アカウントに関連付けられます。Slack を別のアカウントと同期しようとしている場合、Slack は再度認証を要求します。この機能は、一部の複数組織アカウントでこの追加の保護レイヤーが必要なため、セキュリティ目的で作成されました。より便利にするためのソリューションとオプションに取り組んでいます。

## Slack への同期 [#sync]

Slack ワークスペースで認証した後、新しいディスカッション スレッドを開始するとき (または、以前に Slack に同期されていないスレッドにコメントを追加するとき) は、\[同期先] チェックボックスをオンにして、投稿したい Slack チャネルを選択するだけです。へのコメント。

コメントを送信すると、スレッドの上部に「Slack に同期済み」ステータス インジケーターが表示され、New Relic のスレッドに追加されたコメントが Slack のスレッドにも追加されることがすべてのチームメイトに通知されます。

新しいスレッドやディスカッションはそれぞれ、異なる Slack チャンネルに同期できます。これには、別のページで新しいディスカッションを開始したり、以前にディスカッションを同期したページで追加のディスカッションを開始したりすることが含まれます。

ディスカッションが Slack に同期されると、同期をオフにすることはできなくなります。これは、New Relic と Slack が継続的に連携するようにするためです。これにより、混乱、会話の断片化、情報の損失がなくなります。New Relic から Slack への同期がオフになっている場合、一部のチームメイトは New Relic で議論を続け、他のチームメイトは Slack で議論を続けることができます。ただし、Slack ではなく New Relic でのみ議論したい場合は、同じページで新しいディスカッションを開始し、その新しいディスカッションを同期しないことを選択できます。

## Slack からの返信 [#reply]

Slack に同期されたスレッドの最初のコメントは、New Relic で選択されたチャンネルに追加されます。Slack の投稿には、 [one.newrelic.com](https://one.newrelic.com/all-capabilities)のページとディスカッションへのクイック ナビゲーションを提供する「New Relic で開く」ボタンが含まれます。

<img
  title="Slack reply thread"
  alt="A screenshot of a Slack reply thread."
  src={post_on_slack}
  style={{ maxWidth: '50%' }}
/>

投稿には、スレッドが New Relic に同期されたこと、および Slack のスレッドに追加された返信が New Relic にも表示されることを示すメッセージも含まれます。これは、Slack チャネルのメンバー リストに関係なく、newrelic.com の New Relic 組織の全員が返信を表示できることを意味することに注意してください。
