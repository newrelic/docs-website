New Relic は、テレメトリを分析することで、インスツルメンテーションのギャップを検出できます。これらのギャップは、データベース、アプリ、または基本的にその他の種類のエンティティである可能性があります。

たとえば、あるアプリケーションは、計測されていない Amazon RDS データベースに情報を保存している可能性があります。その場合、インストルメンテーション内のデータのギャップを反映するために、「インストルメント化されていない」エンティティの形で表示されます。[サービス マップ](/docs/new-relic-solutions/new-relic-one/ui-data/service-maps/introduction-service-maps)では、このインスツルメントされていないエンティティとインストルメントされたエンティティ (この例では`mysql.demo.ep.rds.com`との間の関係が表示されます。

計測されていないエンティティと計測されているエンティティでは、外観が異なります。インストルメント化されていないエンティティには点線の境界線が表示されますが、インスツルメント化されたエンティティには実線の境界線が表示されます。

## エンティティを手動で計測する [#instrument]

サービス マップからインスツルメントされていないエンティティをインストルメントするには:

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**にアクセスし、次に、 **APM & サービス**、**モバイル**、または**ブラウザ**のいずれかを選択します。
2. エンティティを選択します。
3. 左側で、 **Service map** \[サービス マップ]をクリックします。
4. **\[不足しているものを計測する]**をクリックし、各エンティティのガイド付き UI 指示に従います。
