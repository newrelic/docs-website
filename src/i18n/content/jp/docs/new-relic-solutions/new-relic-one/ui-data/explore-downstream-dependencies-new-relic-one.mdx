APM、ブラウザー監視、およびモバイル監視UIで監視対象のアプリケーションの場合、**\[依存関係]** UIページには、そのエンティティに接続されていることがわかるすべてのアプリケーション、サービス、データベース、およびホストのフィルター可能なリストが表示されます。アップストリームとダウンストリームの依存関係を示し、それらを探索するためのパスを提供します。

[サービスマップ](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one)と同様に、**依存関係**UIは、すべてのアップストリームサービスとダウンストリームサービスがどのように接続されているかを理解するのに役立ちます。また[、サービスマップで使用されているのと同じ色分けシステムを使用して](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one#alerts)、パフォーマンスが良好なものとそうでないものを示します。

## 要件

**依存関係**UIページは、APMエージェント、モバイルエージェント、およびブラウザーエージェントによって監視されているエンティティで使用できます。

APMエージェントに必要なバージョンは次のとおりです。

* [C 1.0.0以上](/docs/release-notes/agent-release-notes/c-sdk-release-notes)
* [Go 1.11以降](/docs/release-notes/agent-release-notes/go-release-notes)
* [Java 3.9.0以降](/docs/release-notes/agent-release-notes/java-release-notes)
* [.NET 4.2以降](/docs/release-notes/agent-release-notes/net-release-notes)
* [Node.js 2.0.0以降](/docs/release-notes/agent-release-notes/nodejs-release-notes)
* [PHP 4.19.0以降](/docs/release-notes/agent-release-notes/php-release-notes)
* [Python 2.38.0.31以降](/docs/release-notes/agent-release-notes/python-release-notes)
* [Ruby 4.3.0以降](/docs/release-notes/agent-release-notes/ruby-release-notes)

## 依存関係の表示 [#access]

エンティティに接続されているアプリケーション、サービス、データベース、およびホストの依存関係を表示する。

1. [one.newrelic.com](http://one.newrelic.com) 、 **Explorer** 、エンティティを選択してください。
2. Select **Dependencies**.
3. さらに掘り下げるには、アプリ、サービス、データベース、ホストのいずれかをフィルタリングします。

依存関係のページをフィルタリングして、そのエンティティに報告する特定のものを表示することができます。依存関係には次のようなものがあります。

* **サービス**: [APM](/docs/apm)-監視対象のアプリケーションとサービス。
* **モバイルアプリケーション**：[モバイルアプリ](/docs/mobile-monitoring)。
* **ブラウザアプリケーション**：フロントエンドの[ブラウザアプリ](/docs/browser)。
* **外部サービス**: 外部サービス [APMで監視](/docs/apm/applications-menu/monitoring/external-services-page).外部サービスには、Webサービスなどのプロセス外のサービス、クラウド上のリソース、その他のネットワークコールなどがあります。
* **データベース**: アプリケーションのデータベースとキャッシュデータ。データベースはエージェントレスです。このため、サービスコールのみが New Relic に報告されるため、データベースにアラートを設定することはできません。
* **ホスト**: お客様の [インフラ](/docs/infrastructure) （サーバーとホスト）。
