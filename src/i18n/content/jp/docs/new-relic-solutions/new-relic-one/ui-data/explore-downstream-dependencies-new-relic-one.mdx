弊社の監視対象アプリケーションについては、 <InlinePopover type="apm"/>、 <InlinePopover type="browser"/>、およびモバイル監視 UI の場合、**Dependencies** \[依存関係] UI ページには、New Relic がその[エンティティ](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic)への接続として識別するすべてのアプリケーション、サービス、データベース、およびホストのフィルター可能なリストが表示されます。上流と下流の依存関係を示し、それらを探索するためのパスを提供します。

[サービス マップ](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one)には、エンティティの関係のエンティティ スコープのマップが表示されますが、**依存関係**UI ページには、アップストリーム サービスとダウンストリーム サービスがテーブルに表示されます。また[、サービス マップで使用されるのと同じ色分けシステムを使用して](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one#alerts)、パフォーマンスの良いものとそうでないものを示します。

## 要件

**依存関係**UIページは、APMエージェント、モバイルエージェント、およびブラウザーエージェントによって監視されているエンティティで使用できます。

## 依存関係の表示 [#access]

エンティティに接続されているアプリケーション、サービス、データベース、およびホストの依存関係を表示する。

1. **[one.newrelic.com](https://one.newrelic.com)**にアクセスし、**APM & services** \[APM とサービス] 、 **Browser** \[ブラウザ] 、または**Mobile** \[モバイル] のいずれかを選択します。
2. エンティティを選択し、\[**依存関係**] をクリックします。
3. さらに掘り下げるには、アプリ、サービス、データベース、ホストのいずれかをフィルタリングします。

依存関係のページをフィルタリングして、そのエンティティに報告する特定のものを表示することができます。依存関係には次のようなものがあります。

* **サービス**: [APM](/docs/apm)-監視対象のアプリケーションとサービス。
* **モバイルアプリケーション**：[モバイルアプリ](/docs/mobile-monitoring)。
* **ブラウザ アプリケーション**: フロントエンド [ブラウザ アプリケーション](/docs/browser)。
* **外部サービス**: 外部サービス [APMで監視](/docs/apm/applications-menu/monitoring/external-services-page).外部サービスには、Webサービスなどのプロセス外のサービス、クラウド上のリソース、その他のネットワークコールなどがあります。
* **データベース**: アプリケーションのデータベースとキャッシュデータ。データベースはエージェントレスです。このため、サービスコールのみが New Relic に報告されるため、データベースにアラートを設定することはできません。
* **ホスト**: お客様の [インフラ](/docs/infrastructure) （サーバーとホスト）。
