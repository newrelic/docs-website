New Relicの自動マップ機能は、相互接続されたサービスを介してカスケードするインシデントのトラブルシューティングに役立つエンティティ中心のマップを表示します。自動マップを使用すると、サービスに影響を与える依存関係を表示することで、問題がいつどこで発生したかを正確に特定できます。

自動マップを使用するには、NewRelicアカウントが必要です。まだお持ちでない場合は、[こちら](https://newrelic.com/signup)から無料アカウントにサインアップできます。

## 使い方 [#how-works]

New Relicの自動マップは、アーキテクチャ内のエンティティ間の関係を表示します。バージョン1の場合、自動マップに表示される関係は、サービス間の直接呼び出しに基づいています。たとえば、APMサービスは頂点（六角形）として表され、直接呼び出しはグラフ上のエッジ（線）として表されます。これは、これらのサービスが互いにどのように依存しているかを示しています。

Automapは、Automapが起動されてから3時間以内にヘルスステータスの変更が発生したソースエンティティに関連するエンティティのみを表示します。そのため、ソースエンティティに関連するすべてのエンティティが表示されない場合があります。エンティティと関係、およびエンティティプラットフォームがテレメトリデータから両方を合成する方法を理解するには、[エンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)と[関係](/docs/apis/nerdgraph/examples/nerdgraph-relationships-api-tutorial/)を参照してください。

<Callout variant="important">
  自動マップの自動拡張ロジックは、エンティティのヘルスステータスに依存します。これは、主にそのエンティティのアラートステータスによって決定されます。自動マップを使用した最適なトラブルシューティングのために、 [アラートが適切に構成されている](/docs/new-relic-solutions/best-practices-guides/alerts-applied-intelligence/alerts-best-practices/)ことを確認してください。
</Callout>

## 自動マップを開く [#open-automap]

自動マップは、サポートされているエンティティタイプの**関連エンティティ**ペインに表示されます。

1. **エクスプローラー**で、エンティティ/サービスを検索するためのフィルターを挿入します（たとえば、 `alerts` ）。

2. エンティティ/サービスをクリックします。

3. **\[概要]ページ**で、右ペインの\[**関連エンティティ]**に移動します。

4. **マップビューを**クリックします：

   <img
     title="Screenshot showing the Map view button"
     alt="Screenshot showing the Map view button"
     src={mapViewButton}
   />

## 歴史を探る

[コンテキスト](/docs/new-relic-one/use-new-relic-one/ui-data/maps-in-context/)内のNewRelicマップなどの単純なナビゲーションマップとは異なり、automapは、時間の前後にスクロールできるようにすることで、問題がどのように発生したかを理解するのに役立ちます。興味のないエンティティを選別することもできます。

### タイムワープカーソル：時間をさかのぼる [#timewarp-cursor]

インシデントが発生した日時と場所を正確に特定するには、**タイムワープ**カーソルを移動して、マップ上のエンティティのヘルスステータスの変化を確認します。タイム**ワープ**カーソルが表示されない場合は、オートマップの右上にある時計アイコンをクリックします。履歴をスクロールすると、NewRelicLookoutによって異常な動作が検出されたことがわかります。

`FulfillmentService`と`BillingService`の間で問題が発生したことを示すスクロールの例を次に示します。

### コントロール：必要なものだけを表示します [#filter-automap]

**タイムワープ**カーソルの下にある**コントロール**メニューを使用すると、これらのオプションのチェックを外すことで、マップ上の特定のエンティティの強調を解除できます。たとえば、正常なサービスを非表示にして、タイムラインを参照するときにアラートにのみ焦点を当てることができます。
