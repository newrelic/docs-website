ほとんどの場合、 `nr1` CLI コマンドはそのまま機能し、追加の設定は必要ありません。 Nerdpack、Nerdlets、ランチャー、視覚化を問題なく作成できるようになります。 しかし、場合によっては、セットアップに特別なものが必要になることがあります。

このガイドでは、Nerdpack の高度な設定を有効にする方法を学びます。

## あなたが始める前に

`nr1` CLI の最新バージョンがインストールされていることを確認してください。

```sh
nr1 update
```

## webpackの設定を拡張する

設定用の Webpack を拡張するには、 `.extended-webpackrc.js`ファイルを使用できます。 これにより、アプリをローカルで提供またはデプロイするときに、独自のエイリアス、スクリプト、設定を追加できるようになります。

## 追加の証明書を追加する

企業プロキシの背後で`nr1`を操作する場合、 `RequestError`が表示されることがあります:

```sh
nr1 create --type nerdpack --name new-nerdpack
[output] {error}RequestError: Error:{normal} self signed certificate in certificate chain
```

ノードは独自の証明書チェーンを使用します。 したがって、このエラーが発生した場合は、Node の「ルート」 CA に企業証明書を追加する必要があります。

```sh
export NODE_EXTRA_CA_CERTS=[cert]
```
