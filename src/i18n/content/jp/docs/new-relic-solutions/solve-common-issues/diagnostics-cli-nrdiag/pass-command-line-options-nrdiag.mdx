Diagnostics CLIで以下のコマンドラインオプションを使用するには、次のようにします。

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        オプション
      </th>

      <th>
        使用方法
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `-a`

        `-attach`
      </td>

      <td>
        New Relic アカウントに自動アップロードするために添付します。最大アップロードサイズは4GBです。これは検証済みの <InlinePopover type="licenseKey"/>あなたの環境から。
      </td>
    </tr>

    <tr>
      <td>
        `-api-key`
      </td>

      <td>
        New relic アカウントに結果をアップロードするときに使用するユーザー API キー。最大アップロードサイズは4GBです。これにより、ライセンス キーの使用が回避され、 <InlinePopover type="userKey"/>代わりに結果を添付します。
      </td>
    </tr>

    <tr>
      <td>
        `-r`

        `-region`
      </td>

      <td>
        New Relic アカウントがホストされている [リージョン](/docs/accounts/accounts-billing/account-setup/choose-your-data-center/) 。受け入れられる値: EU または US。大文字小文字を区別しません。デフォルト: 米国
      </td>
    </tr>

    <tr>
      <td id="browser-check">
        `-browser-url STRING`
      </td>

      <td>
        Diagnostics CLIバージョン1.1.9以上

        呼び出されると、以下に関連する診断チェックのみが実行されます。 <InlinePopover type="browser"/>。このコマンドは、New Relic のブラウザ監視エージェントが存在することを確認し、エージェントのバージョンと注入方法[を返します ( <InlinePopover type="apm"/>](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#select-apm-app)または[コピー/ペースト経由](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent#copy-paste-app))、およびローダーのタイプ (Pro、Lite、SPA)。イントラネット サイトのトラブルシューティング時に New Relic サポートに詳細を提供するために使用されます。
      </td>
    </tr>

    <tr>
      <td>
        `-c STRING`

        `-config-file STRING`
      </td>

      <td>
        デフォルトのエージェント設定ファイルの場所を上書きします。デフォルトのフォルダに加えて検索するフォルダ、または特定の設定ファイルへのパスを指定するために使用できます。
      </td>
    </tr>

    <tr>
      <td>
        `-filter STRING`
      </td>

      <td>
        結果ステータスでタスク結果をフィルタリングします。カンマ区切りのリストを受け入れます。 `Success` 、 `Warning` 、 `Failure` 、 `Error` 、 `None` 、または`Info`を受け入れます。構文例：

        ```
        "Success,Warning,Failure"
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-h`

        `-help`
      </td>

      <td>
        コマンドラインオプションの完全なリストを表示します。実行するすべてのタスクを一覧表示するには、 `-h tasks`を使用します。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-include STRING`
      </td>

      <td>
        `nrdiag-output.zip`にファイルまたはディレクトリ (サブディレクトリを含む) を含めます。4GB を制限します。 `-a` または `-api-key` フラグを使用して、結果を New Relic にアップロードします。

        構文例：

        ```
        -include ./path/to/file
        ```

        ```
        -include \some\path\
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-interactive`
      </td>

      <td>
        タイプ: bool
      </td>
    </tr>

    <tr>
      <td>
        `-output-path STRING`
      </td>

      <td>
        `nrdiag-output.zip` 、 `nrdiag-output.json` 、および`nrdiag-filelist.txt`の結果を書き込むための別の出力ディレクトリを指定します。デフォルトの場所は`./`です。
      </td>
    </tr>

    <tr>
      <td>
        `-o`

        `-override STRING`
      </td>

      <td>
        New Relic Supportからの要求に応じて、オーバーライドする引数を渡します。形式： `identifier.property=value` 。

        構文例：

        ```
        -override Java/Config/Agent.Status=Success
        ```

        ```
        -o Base/Config/Validate.agentlanguage=PHP
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-p STRING`

        `-proxy STRING`
      </td>

      <td>
        HTTP接続タスクで使用するプロキシを提供します。 HTTPまたはHTTPSにすることができます。プロキシは`http(s)://proxyIp:proxy`の形式である必要があります。 Diagnostics CLIがエージェント設定ファイルでプロキシを検出すると、デフォルトでそのプロキシが使用されます。ほとんどの場合、 `port`は必要ありません。 Diagnostics CLIがエージェント設定ファイルでプロキシを検出すると、デフォルトでそのプロキシが使用されます。
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-pw STRING`
      </td>

      <td>
        必要に応じて、プロキシのパスワードを入力します。Diagnostics CLIがエージェント設定ファイルでプロキシを見つけた場合、デフォルトでそのプロキシを使用します。
      </td>
    </tr>

    <tr>
      <td>
        `-proxy-user STRING`
      </td>

      <td>
        必要に応じて、プロキシのユーザ名。Diagnostics CLIがエージェント設定ファイルでプロキシを見つけた場合、デフォルトでそのプロキシを使用します。
      </td>
    </tr>

    <tr>
      <td>
        `-q`
      </td>

      <td>
        クワイエット出力は、高レベルの結果のみを出力し、説明出力は出力しません。 `-y`も使用されている場合、ファイル追加の警告を抑制します。 `-v`と矛盾しません。包含フィルターは無視されます。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-qq`
      </td>

      <td>
        非常に静かな出力では、出力用に1行の要約行のみが出力されます（ `q`を意味します）。 `-y`も使用されている場合、ファイル追加の警告を抑制します。 `-v`と矛盾しません。包含フィルターは無視されます。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-skip-version-check`
      </td>

      <td>
        アプリケーションの新しいバージョンの自動チェックをスキップする。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-s STRING`

        `-suites STRING`
      </td>

      <td>
        [スイートの実行](/docs/using-new-relic/cross-product-functions/diagnostics-cli-nrdiag/run-diagnostics-cli-nrdiag/#suites) 、特定の製品や課題を対象としたタスクの集まりです。複数のスイートを指定するには、カンマで区切ります。すべてのスイートのリストを取得するには、次を実行します。

        ```
        ./nrdiag -h suites
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-t STRING`

        `-tasks STRING`
      </td>

      <td>
        エージェントまたはタスクタイプのいずれかによって、タスクのサブセットのみを実行します。複数のタスクを指定するには、それらをコンマまたはワイルドカード`*`で区切ります。すべてのタスクのリストについては、以下を実行してください。

        ```
        ./nrdiag -h tasks
        ```
      </td>
    </tr>

    <tr>
      <td>
        `-usage-opt-out`
      </td>

      <td>
        この実行について、匿名の Diagnostics CLI ツールの使用状況データを New Relic に送信することを辞退する。タイプ： bool
      </td>
    </tr>

    <tr>
      <td>
        `-list-scripts`
      </td>

      <td>
        利用可能なスクリプトをリストします。
      </td>
    </tr>

    <tr>
      <td>
        `-script STRING`
      </td>

      <td>
        指定されたスクリプトを表示します。`-run` とともに使用してスクリプトを実行します。
      </td>
    </tr>

    <tr>
      <td>
        `-run`
      </td>

      <td>
        `-script` とともに使用してスクリプトを実行します。
      </td>
    </tr>

    <tr>
      <td>
        `-script-flags`
      </td>

      <td>
        `-run -script` とともに使用して、コマンド ライン フラグをスクリプトに渡します。
      </td>
    </tr>

    <tr>
      <td>
        `-v`

        `-verbose`
      </td>

      <td>
        チェック実行時に詳細なログを表示します。デフォルトでは無効です。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-version`
      </td>

      <td>
        現在の診断CLIバージョンを表示します。これにより、新しいバージョンを確認するように求められ、新しいバージョンが利用可能な場合はダウンロードするように求められます。 `-skip-version-check`よりも優先されます。タイプ：bool
      </td>
    </tr>

    <tr>
      <td>
        `-y`

        `-yes`
      </td>

      <td>
        実行中に表示されるプロンプトに`yes`を応答します。デフォルトでは無効になっています。タイプ：bool
      </td>
    </tr>
  </tbody>
</table>
