`<Stack>`コンポーネントの子要素。

`<Stack>`は、選択されたレイアウトに基づいてスタック項目の高さ、幅、順序を変更する場合があります。

### 使用方法

```js
import { StackItem } from 'nr1'
```

### 小道具

<table>
  <tbody>
    <tr>
      <td>
        `children` <h5>必須</h5><h5>ノード</h5>
      </td>

      <td>
        スタック項目内に表示されるコンテンツ。
      </td>
    </tr>

    <tr>
      <td>
        `className` <h5>ストリング</h5>
      </td>

      <td>
        コンポーネントにクラス名を追加します。配置と間隔の目的にのみ使用してください。
      </td>
    </tr>

    <tr>
      <td>
        `grow` <h5>ブール値</h5>
      </td>

      <td>
        `true`の場合、アイテムは残りのスペースを埋めます。 これを機能させるには、スタックを拡張して、アイテムの方向にある利用可能なスペースをすべて占有する必要があります。 これは、スタックの方向に応じて、スタックの`fullWidth`または`fullHeight`プロパティを使用することで実現できます。
      </td>
    </tr>

    <tr>
      <td>
        `shrink` <h5>ブール値</h5>
      </td>

      <td>
        `true`の場合、必要に応じてアイテムが縮小されます。
      </td>
    </tr>

    <tr>
      <td>
        `style` <h5>物体</h5>
      </td>

      <td>
        カスタム スタイル用のインライン スタイル。
      </td>
    </tr>

    <tr>
      <td>
        `testId` <h5>ストリング</h5>
      </td>

      <td>
        `data-test-id`属性を追加します。 ユニット テストと E2E テストでコンポーネントをターゲットにするために使用します。テスト ID を有効にするには、nerdpack ID をプレフィックスとして付け、その後にドットを付けます。たとえば、 `my-nerdpack.some-element` 。

        **注:** `data-test-id` 属性はDOMから削除されているため表示されない場合があります。デバッグするには、URL に `e2e-test` クエリ引数を渡します。
      </td>
    </tr>
  </tbody>
</table>
