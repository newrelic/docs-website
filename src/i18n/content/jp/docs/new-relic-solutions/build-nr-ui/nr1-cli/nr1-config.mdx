`nr1 config:list`を実行して、既存の設定とそのキーをすべて表示します。

## `nr1 config:delete` [#nr1-configdelete]

### 設定を削除する [#nr1-configdelete-remove]

特定の設定の値を削除します。

### 使用方法 [#nr1-configdelete-usage]

`$ nr1 config:delete OPTION`

### オプション [#nr1-configdelete-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        これは**必須**です。 設定のキー。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        存在する場合、この設定は指定されたプロファイルで実行しているときにのみ適用されます。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

### エイリアス [#nr1-configdelete-aliases]

* `nr1 config:remove`
* `nr1 config:rm`

### 例 [#nr1-configdelete-exmples]

* `$ nr1 config:delete --key=proxyHttp`

## `nr1 config:set` [#nr1-configset]

### 設定を設定する [#nr1-configset-set]

設定キーを指定して特定の設定値を設定します。 デフォルトでは、コマンドはキーを指定した後に新しい値を要求しますが、 `--k, --key=KEY`オプションを使用してこの手順をスキップすることもできます。

### 使用方法 [#nr1-configset-usage]

`$ nr1 config:set OPTION`

### オプション [#nr1-configset-options]

<table>
  <tbody>
    <tr>
      <td>
        `-k, --key=KEY`
      </td>

      <td>
        これは**必須**です。 設定のキー。
      </td>
    </tr>

    <tr>
      <td>
        `-V, --value=VALUE`
      </td>

      <td>
        設定の値。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        使用する認証プロファイル
      </td>
    </tr>

    <tr>
      <td>
        `-t, --this-profile-only`
      </td>

      <td>
        存在する場合、この設定は指定されたプロファイルで実行しているときにのみ適用されます。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

### 例 [#nr1-configset-examples]

* `$ nr1 config:set --key=proxyEnabled`
* `$ nr1 config:set --key=proxyEnabled --value=ENABLED`
