カタログを管理するには、以下のコマンドを使用します。 任意のコマンドをクリックすると、そのコマンドの使用オプションと追加の詳細が表示されます。

<table>
  <thead>
    <tr>
      <th>
        コマンド
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`nr1 catalog:info`](#nr1-cataloginfo)
      </td>

      <td>
        カタログに保存されている Nerdpack 情報を表示します。
      </td>
    </tr>

    <tr>
      <td>
        [`nr1 catalog:submit`](#nr1-catalogsubmit)
      </td>

      <td>

      </td>
    </tr>

## `nr1 catalog:info` [#nr1-cataloginfo]

### カタログの詳細を取得する [#nr1-cataloginfo-details]

カタログに表示されるアプリケーションに関する情報を表示します。

特定の Nerdpack フォルダー内で実行すると、その Nerdpack の情報が表示されます。 ローカルの Nerdpack から情報を取得したくない場合は、 `--nerdpack-id=NERDPACK_ID`オプションを使用して特定の Nerdpack からクエリを実行します。

### 使用方法 [#nr1-cataloginfo-usage]

`$ nr1 catalog:info`

### オプション [#nr1-cataloginfo-options]

<table>
  <tbody>
    <tr>
      <td>
        `-f, --field=FIELD`
      </td>

      <td>
        情報を取得するフィールドを指定します。
      </td>
    </tr>

    <tr>
      <td>
        `-i, --nerdpack-id=NERDPACK_ID`
      </td>

      <td>
        どの Nerdpack から情報を取得するかを指定します。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

## `nr1 catalog:submit` [#nr1-catalogsubmit]

### カタログに情報を送信する [#nr1-catalogsubmit-send]

アプリケーションのカタログ ディレクトリに追加した情報を収集し、カタログに保存します。

このコマンドは Nerdpack フォルダーで実行する必要があります。 このコマンドは、規則名を使用して特定のファイルを検索します。

### 使用方法 [#nr1-catalogsubmit-usage]

`$ nr1 catalog:submit`

### オプション [#nr1-catalogsubmit-options]

<table>
  <tbody>
    <tr>
      <td>
        `-P, --skip-screenshots`
      </td>

      <td>
        スクリーンショットアセットのアップロードをスキップします。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>
