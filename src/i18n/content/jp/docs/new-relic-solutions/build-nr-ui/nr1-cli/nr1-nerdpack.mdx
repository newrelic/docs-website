## `nr1 nerdpack:build` [#nr1-nerdpackbuild]

オタク集団を結成します。

### 使用方法 [#build-usage]

```sh
nr1 nerdpack:build
```

### オプション [#build-options]

<table>
  <tbody>
    <tr>
      <td>
        `--extra-metadata-path=extra-metadata-path`
      </td>

      <td>
        `default: extra-metadata.json` 追加のメタデータを含む json ファイル パスを指定します。
      </td>
    </tr>

    <tr>
      <td>
        `--prerelease=prerelease`
      </td>

      <td>
        指定すると、生成されたファイルの現在のバージョンに値が追加されます。 たとえば`--prerelease=abc` 。 バージョンは`1.2.3-abc`になります。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

### 説明 [#build-description]

webpack プロセスを実行して、Nerdpack を JavaScript と CSS のバンドルに組み立てます。 他の多くの CLI コマンドと同様に、Nerdpack の`package.json`レベルで実行する必要があります。 サブディレクトリ内の`nr1.json`ファイルを検索し、検証してビルドを試みます。 ビルド プロセスを開始する前に`nr1 nerdpack:validate`も実行されます。

## `nr1 nerdpack:clone` [#nr1-nerdpackclone]

Git リポジトリから nerdpack をクローンします。

### 使用方法 [#clone-usage]

```sh
nr1 nerdpack:clone
```

### オプション [#clone-options]

<table>
  <tbody>
    <tr>
      <td>
        `-f`, `--force`
      </td>

      <td>
        宛先フォルダーが存在する場合は置き換えます。
      </td>
    </tr>

    <tr>
      <td>
        `-p`, `--path=path`
      </td>

      <td>
        クローン先のディレクトリ (デフォルトはリポジトリ名)。
      </td>
    </tr>

    <tr>
      <td>
        `-r`, `--repo=repo`
      </td>

      <td>
        これは**必須**です。 リポジトリの場所。HTTPS または SSH パスのいずれかです。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

### 説明 [#clone-description]

Git リポジトリが指定されると、使用を開始できるようにすべてのアクションが実行されます。 これには、次の内容が含まれます。

1. リポジトリのクローンを作成します。
2. リポジトリをリモートアップストリームとして設定します。
3. `npm`を使用して、すべての依存関係をインストールします。
4. プロファイルを使用して新しい UUID を生成し、コミットします。

### 使用方法 [#publish-usage]

```sh
nr1 nerdpack:publish
```

### オプション [#publish-options]

<table>
  <tbody>
    <tr>
      <td>
        `-B`, `--skip-build`
      </td>

      <td>

      </td>
    </tr>

    <tr>

    <tr>
      <td>
        `--extra-metadata-path=extra-metadata-path`
      </td>

      <td>
        `default: extra-metadata.json` 追加のメタデータを含む json ファイル パスを指定します。
      </td>
    </tr>

    <tr>
      <td>
        `--prerelease=prerelease`
      </td>

      <td>
        指定すると、生成されたファイルの現在のバージョンに値が追加されます。 たとえば、 `--prerelease=abc` 。 バージョンは`1.2.3-abc`になります。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

### 説明 [#publish-description]

このコマンドは、Nerdpack をビルドしてレジストリにアップロードします。

追加の引数が渡されない場合、このコマンドは自動的に nerdpack バージョンを`STABLE`としてタグ付けします。 この動作を無効にする場合は、コマンドに`--skip-tag`フラグを渡します。

タグの詳細については、 `nr1 nerdpack:tag --help`を参照してください。

## `nr1 nerdpack:tag` [#nr1-nerdpacktag]

特定の Nerdpack バージョンにタグを付けます。

### 使用方法 [#tag-usage]

```sh
nr1 nerdpack:tag
```

### オプション [#tag-options]

<table>
  <tbody>
    <tr>
      <td>
        `-V`, `--from-version=from-version`
      </td>

      <td>
        タグ付けするバージョン。 指定されていない場合は、 `package.json`のものが使用されます。
      </td>
    </tr>

    <tr>
      <td>
        `-i`, `--nerdpack-id=nerdpack-id`
      </td>

      <td>
        デプロイする Nerdpack uuid。 指定されていない場合は、 `nr1.json`のものが使用されます。
      </td>
    </tr>

    <tr>
      <td>
        `-t`, `--tag=tag`
      </td>

      <td>
        `default: STABLE` タグ名
      </td>
    </tr>

    <tr>
      <td>
        `--profile=profile`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

### 説明 [#tag-description]

1 つのバージョンに複数のタグを設定できますが、各タグは 1 つのバージョンにのみ適用できます。 既存のタグを持つ別のバージョンにタグを付けると、以前のタグが解除されます。

購読者に表示するバージョンにタグを付けるには、 `STABLE`を使用することをお勧めします。 詳細については、コマンド`nr1 nerdpack:subscribe`を参照してください。

### エイリアス [#tag-aliases]

```sh
$ nr1 nerdpack:deploy
```
