## nr1 オートコンプリート [#nr1-autocomplete]

デフォルトでは、コマンドは`zsh`のオートコンプリート指示を表示します。 bash の説明が必要な場合は、 `nr1 autocomplete bash`を実行してください。

### 使用方法

`$ nr1 autocomplete`

### 引数

<table>
  <tbody>
    <tr>
      <td>
        `SHELL`
      </td>

      <td>
        説明が必要なシェルの種類。
      </td>
    </tr>
  </tbody>
</table>

### オプション

<table>
  <tbody>
    <tr>
      <td>
        `-r, --refresh-cache`
      </td>

      <td>
        キャッシュを更新し、表示の指示を無視します。
      </td>
    </tr>
  </tbody>
</table>

### 例

* `$ nr1 autocomplete`
* `$ nr1 autocomplete zsh`
* `$ nr1 autocomplete bash`
* `$ nr1 autocomplete --refresh-cache`

## nr1 作成 [#nr1-create]

Nerdpack、Nerdlet、ランチャー、カタログのいずれかのテンプレートから新しいコンポーネントを作成します。 CLI がこのプロセスを案内します。

Nerdpack とそのファイル構造の詳細については、 [「Nerdpack ファイル構造」](/explore-docs/nerdpack-file-structure)を参照してください。 Nerdpack の設定方法の詳細については、 [Nerdpack CLI コマンド](/explore-docs/nr1-nerdpack)を参照してください。

### 使用方法

`$ nr1 create`

### オプション

<table>
  <tbody>
    <tr>
      <td>
        `-f, --force`
      </td>

      <td>
        存在する場合、確認せずに既存のファイルを上書きします。
      </td>
    </tr>

    <tr>
      <td>
        `-n, --name=NAME`
      </td>

      <td>
        コンポーネントに名前を付けます。
      </td>
    </tr>

    <tr>
      <td>
        `-t, --type=TYPE`
      </td>

      <td>
        コンポーネントの種類を指定します。
      </td>
    </tr>

    <tr>
      <td>
        `--path=PATH`
      </td>

      <td>
        コンポーネントへのルート。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

## nr1 ヘルプ [#nr1-help]

デフォルトではすべての nr1 コマンドが表示されます。 特定のコマンドの詳細を取得するには、 `nr1 help COMMAND_NAME`を実行します。

### 使用方法

`$ nr1 help`

### 引数

<table>
  <tbody>
    <tr>
      <td>
        `COMMAND_NAME`
      </td>

      <td>
        特定のコマンドの名前。
      </td>
    </tr>
  </tbody>
</table>

### 例

* `$ nr1 help`
* `$ nr1 help nerdpack`
* `$ nr1 help nerdpack:deploy`

## nr1 NRQL [#nr1-nrql]

NRQL クエリを使用してデータベースからデータを取得します。

NRQL とその使用方法の詳細については、 [NRQL ドキュメント](/docs/nrql/get-started/introduction-nrql-new-relics-query-language/)をご覧ください。

### 使用方法

`$ nr1 nrql OPTION ...`

### オプション

<table>
  <tbody>
    <tr>
      <td>
        `-a, --account=ACCOUNT`
      </td>

      <td>
        これは**必須**です。 ユーザーアカウントID。
      </td>
    </tr>

    <tr>
      <td>
        `-q, --query=QUERY`
      </td>

      <td>
        これは**必須**です。 実行する NRQL クエリ。
      </td>
    </tr>

    <tr>
      <td>
        `-u, --ugly`
      </td>

      <td>
        タブやスペースなしでコンテンツを表示します。
      </td>
    </tr>

    <tr>
      <td>
        `--profile=PROFILE`
      </td>

      <td>
        使用する認証プロファイル。
      </td>
    </tr>

    <tr>
      <td>
        `--verbose`
      </td>

      <td>
        出力に追加情報を追加します。
      </td>
    </tr>
  </tbody>
</table>

## nr1 プロファイル [#nr1-profiles]

プロファイルを管理するために使用できるコマンドのリストを表示します。 具体的な使用方法の詳細については、 `nr1 help profiles:COMMAND`を実行してください。 複数のプロファイルを持つことができ、複数の New Relic アカウントでコマンドを実行する場合に便利です。

プロファイルの設定の詳細については、 [Github ワークショップ](https://github.com/newrelic/nr1-workshop/blob/master/lab-cli/INSTRUCTIONS.md)をご覧ください。

### 使用方法

`$ nr1 profiles:COMMAND`

### コマンド

<table>
  <tbody>
    <tr>
      <td>
        `profiles:add`
      </td>

      <td>
        プロファイル キーチェーンに新しいプロファイルを追加します。
      </td>
    </tr>

    <tr>
      <td>
        `profiles:default`
      </td>

      <td>
        デフォルトにするプロファイルを選択します。
      </td>
    </tr>

    <tr>
      <td>
        `profiles:list`
      </td>

      <td>
        キーチェーン上のプロファイルを一覧表示します。
      </td>
    </tr>

    <tr>
      <td>
        `profiles:remove`
      </td>

      <td>
        キーチェーンからプロファイルを削除します。
      </td>
    </tr>
  </tbody>
</table>

## nr1 アップデート [#nr1-update]

CLI の最新バージョンへの更新。 必要に応じて、更新するチャネルを指定できます。

### 使用方法

`$ nr1 update`

### 引数

<table>
  <tbody>
    <tr>
      <td>
        `CHANNEL`
      </td>

      <td>
        特定のチャネルの名前。
      </td>
    </tr>
  </tbody>
</table>

### 例

* `$ nr1 update`
* `$ nr1 update somechannel`
