クイックスタートを構築する前に、デモ サービスを起動する必要があります。

## デモサービスを開始する [#spin-demo-services]

GitHub からラボ リポジトリをクローンします。

```bash
    git clone https://github.com/newrelic-experimental/build-a-quickstart-lab.git
```

`flashDB`というデモ ディレクトリに変更します。

```bash

cd build-a-quickstart-lab/flashDB

```

このディレクトリには、次の 2 つの重要なサービスのコードが含まれています。

* データベースを模倣し、作成、読み取り、更新、削除 (CRUD) 操作の機能も提供する Python プログラム。
* CRUD操作を手動で実行する必要がないようにダミーのデータベーストラフィックを生成するシミュレーターサービス

サービスを実行するには、次のコマンドを実行します。

```Python
python simulator.py
```

ここでは、データベースに対して模擬トラフィックを生成するシミュレーターを実行します。 シミュレータが起動して実行されると、ターミナルに次の出力が表示されます。

```bash animate
python simulator.py
[output] Writing...
[output] try_send
[output] Writing...
[output] try_send
[output] Reading...
[output] try_send
[output] Reading...
[output] try_send
[output] Writing...
[output] try_send
[output] Writing...
[output] try_send
[output] Reading...
```

次に、[テレメトリー](https://github.com/newrelic/newrelic-telemetry-sdk-python)を使用して、flashDB サービスからNew Relicにテレメトリーデータを送信します。
