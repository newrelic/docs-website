宇宙で最も軽量かつ高速なデータベースである FlashDB という新しいデータベースを開発したと想像してください。 あなたのデータベースはすでに開発者の間で非常に人気があります。 New Relicオブザーバビリティのための優れたツールを提供していることを知っているので、ユーザーにスタックからインサイトを取得するためにNew Relicを使用することをお勧めします。 これまで、ユーザーは New Relic でダッシュボードとアラートを手動で設定する必要がありました。 最近、データからより早く価値を引き出すためのセキュリティ ビルディング ブロックを提供するNew Relic Instant セキュリティ (I/O) について学習しました。

データベースの人気が高まるにつれ、FlashDB のユーザーが New Relic でデータベースをすばやく簡単に監視できるようにすることを決定します。

目標を達成するには、他のユーザーが FlashDB をすばやく監視できるように、 New Relic I/O にクイックスタートを投稿してください。

## インスタントオブリバビリティ [#instant-observability]

[New Relic Instant I/O は、](https://newrelic.com/instant-observability)データから即座に価値を引き出すのに役立つ、すぐに使用できる I/O リソースの統合エコシステムです。 手動セットアップの負担を回避しながら、テクノロジースタックを計測、監視、分析できます。

New Relic I/O を使用すると、次のことが可能になります。

* より早く始める
* 手作業の労力を削減
* 持続可能性の専門家から学ぶ

テクノロジースタックのインストゥルメントと監視に必要な構成要素をバンドルした数百のクイックスタートから選択し、クリックするだけでインストールして監視を開始できます。

## クイックスタート [#quickstart]

New Relicを使用すると、さまざまなソースからデータを収集、監視、レポートできます。 ダッシュボードを使用すると、さまざまなユーザーフレンドリーなグラフを使用して、収集したデータを表示、調査、カスタマイズ、理解、相関させることができます。 アラートポリシーをモニターキーパフォーマンスメトリックに設定し、異常時に通知を受け取ることもできます。

従来、FlashDB ユーザーは、データベースを監視するために、ダッシュボード、アラート、およびその他のトレンドを作成するために重要な開発時間を費やす必要がありました。 しかし、クイックスタートを使用すると、その作業を代わりに行うことができます。 クイックスタートは、特定のユースケースに即座に価値をもたらします。 これらには以下が含まれます:

* あなたのサービスに関する明確な指示
* ダッシュボードやアラートのような耐久性の構成要素

これらすべては、New Relic の開発者、パートナー、顧客がベストプラクティスのソリューションを提供する New Relic I/O を通じて利用できます。 あなたは、FlashDB の開発者として、ユーザーがインスタンスをすばやく監視できるようにするためのクイックスタートに貢献する素晴らしい候補者です。

## 目的 [#objectives]

* テレメトリーデータをNew Relicに送信する

* ダッシュボードとアラートを作成する

* クイックスタートを作成する

  * ダッシュボードを追加する
  * アラートを追加する
  * データソースを追加する
  * GitHub へのクイックスタートの貢献

## 前提条件 [#prerequisite]

* 無料の[New Relicアカウント](https://login.newrelic.com/login)を作成する
* [Pythonを](https://www.python.org/downloads/)インストールする

<Callout variant="tip">
  この手順は、クイックスタートを構築する方法を教えるコースの一部です。 クイックスタートの構築を開始する前に、[環境をセットアップします](/docs/new-relic-solutions/build-nr-ui/contribute-nr-io/deploy-your-application)。
</Callout>
