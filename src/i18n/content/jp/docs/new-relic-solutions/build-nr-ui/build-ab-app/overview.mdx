ウェブサイトを開発し、 New Relicのブラウザ監視を使用してインストゥルメントしたと想像してください。 New Relic のコア機能セットは多くの情報を提供しますが、カスタム機能も必要です。 サイトの新しいデザインがユーザーの関心を高めるかどうかを判断します。 つまり、デザインを A/B テストし、データに基づいて決定を下すということです。

このタスクでは、React、New Relic One ソフトウェア開発キット、そして最新の Web テクノロジーの無限のパワーを使用して、New Relic アプリケーションを作成できます。

New Relic アプリケーションは、Web 開発で最も人気のある JavaScript ライブラリの 1 つである React を使用して構築されています。 React コードを記述する際には自由があるため、アプリのロジックをカスタマイズしたり、独自のコンポーネントを設計したり、豊富なオープンソース コンポーネント ライブラリを活用したりすることができます。 したがって、A/B テスト アプリでは、結果に基づいてテストを終了するカスタム ロジックを記述できます。

New Relic One ソフトウェア開発キット (SDK) を使用すると、New Relic プラットフォームにアプリケーションを作成、提供、公開、およびデプロイできます。 また、データの収集、情報の表示、ユーザーインタラクションの処理などを行うための React コンポーネントも多数提供します。 `Button`や`Dropdown`などのコンポーネントを使用して、New Relic にネイティブな外観と操作性を備えたインタラクティブなエクスペリエンスを作成します。 `Table`および`Chart`コンポーネントを使用して、New Relic アカウントまたはその他の場所からのデータを表示します。 A/B テスト アプリケーションを構築するときは、SDK の`NrqlQuery`コンポーネントを使用してアカウントからブラウザーデータを取得します。

カスタム React コード、SDK コンポーネント、そしてオープンソース ライブラリの幅広い世界を活用して、 New Relicで A/B テスト アプリケーションを作成できます。 ただし、自分で作成する前に、アプリ カタログをチェックして、先に誰かが作成していないか確認することをお勧めします。 カタログにすでにそのアプリがある場合は、数回クリックするだけでアカウントに追加できます。これは、New Relic でアプリを作成するもう 1 つの利点です。

このコースでは、A/B テストを実行および管理するための実際のNew Relicアプリケーションを構築します。 競合するデザインのブラウザデータを視覚化し、過去のテストの履歴データを確認し、さらには優勝したデザインを選択してテストを終了することまで、すべてNew Relicアプリケーションから行うことができます。 しかし、チャートの作成や HTTP リクエストの作成といった細かい作業に入る前に、New Relic アプリケーションが何で構成されているかを学ぶ必要があります。
