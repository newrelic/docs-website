<Callout variant="tip">
  このレッスンは、New Relic アプリケーションをゼロから構築する方法を学習するコースの一部です。 まだご覧になっていない方は、概要をご覧ください。

  コースの各レッスンは前回のレッスンに基づいて構築されるため、このレッスンを開始する前に、前回のレッスン「Nerdpack の作成」を完了していることを確認してください。
</Callout>

New Relic アプリケーションを構築する場合、それをプラットフォーム上で表示することは有益です。 これは、最終的に公開される環境でアプリケーションを設計、実装、デバッグするのに役立ちます。 `nr1` を使用すると、アプリケーションをホストする開発サーバーをリリースして、 New Relicがそれを提供できるようになります。

<Steps>
  <Step>
    Nerdpack のルートディレクトリ内から、Nerdpack を提供します。

    ```bash
    nr1 nerdpack:serve
    ```

    Nerdpack のビルドが成功し、アプリケーションを表示する準備ができたら、New Relic へのリンクを含むメッセージが表示されます。

    ```bash
    [output] {green}✔{plain}  Server ready! Test it at: {purple}https://one.newrelic.com/?nerdpacks=local
    [output] {blue}↩{plain}  Server will reload automatically if you modify any file!
    ```

    追加されたクエリ`nerdpacks=local`に注意してください。 このクエリはNew Relicにローカルで提供される Nerdpacks を表示するように指示します。

    <Callout variant="tip">
      New Relic プラットフォームにアプリケーションが表示されない場合は、クエリ文字列に`nerdpacks=local`が含まれていることを確認してください。
    </Callout>
  </Step>

  <Step>
    指定された URL に移動します。 ホームページから**Apps** \[アプリ]を選択して、New Relic アプリケーションを表示します。
  </Step>

  <Step>
    **Your apps** \[あなたのアプリ]セクションで、「A/Bテスト ランチャー」というランチャーを見つけます。
  </Step>

  <Step>
    ランチャーを選択すると、ルート Nerdlet とそのデフォルトのウェルカム メッセージが表示されます。
  </Step>
</Steps>

おめでとうございます。最初の New Relic アプリケーションの提供が完了しました。

コマンドの出力で、Nerdpack 内のファイルを変更するとサーバーがリロードされることに注意してください。 `nerdlets/ab-test-nerdlet/index.js`を更新して試してみてください:

```jsx fileName=nerdlets/ab-test-nerdlet/index.js lineHighlight=5
import React from 'react';

export default class AbTestNerdletNerdlet extends React.Component {
  render() {
    return <h1>A/B test results</h1>;
  }
}
```

アプリは自動的に更新され、新しい見出しが表示されます。
