<Callout variant="tip">
  このレッスンは、New Relic アプリケーションをゼロから構築する方法を学習するコースの一部です。 まだご覧になっていない方は、概要をご覧ください。

  コースの各レッスンは前回のレッスンに基づいて構築されるため、このレッスンを開始する前に、前回のレッスン「チャート グループの追加」を完了していることを確認してください。
</Callout>

アプリケーションが整理されると、読みやすくなり、理解しやすくなります。 A/B テスト アプリケーションには、多くのグラフがありますが、組織がありません。 次のいくつかのレッスンでは、ユーザー インタフェース コンポーネントを使用して、アプリケーションに組織を導入し、新しい機能を提供します。 まず、 `Grid`コンポーネントを使用して、デザイン ガイドのレイアウトに合わせてグラフを配置します。 次に、各グラフが何を表しているかを明確にするために見出しを追加します。 3 番目に、アプリの上部に A および B デザイン バージョンの説明を追加します。 最後に、A/B テストを終了するセクションを作成します。

次のレッスンでは、デザイン ガイドと同じようにチャートを配置します。
