このコースを通じて構築するNew Relicアプリケーションを使用すると、開発者は Web サイトの A/B テストを実行できます。 A/B テストを成功させるには、サイト所有者が結果を分析できる必要があります。 各デザインのパフォーマンスがわからなければ、開発者はどのデザインを使用するかをどのように決定するのでしょうか?

アプリケーションは、競合する 2 つのバージョンのデータを表示します。 さまざまなグラフや表を使用することで、ユーザーはテストの結果を分析し、目標を達成するためにどのデザインが最適かを情報に基づいて決定できるようになります。

このコースでは、このモックアップを何度も参照することになります。 作成するグラフ、グラフを整理する方法、グラフに提供するデータの種類を示します。 次の 4 つのセクションでは、A/B テスト アプリケーションが効果的に動作するために必要なすべてのグラフを作成する方法を学習します。 その後、モックアップのデザインに合わせてチャートを配置し、先ほど起動したデモ アプリケーションから収集した New Relic データを提供します。
