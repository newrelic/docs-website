## ログのインストール [#install-logs]

当社では高速でスケーラブルな<InlinePopover type="logs"/>プラットフォームを提供しているため、ログを残りのテレメトリーとインフラストラクチャデータと1か所で接続できます。

* [APM in context](/docs/logs/logs-context/get-started-logs-context/) 、[インフラストラクチャエージェント](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)、またはその他の[ログ転送ソリューション](/docs/logs/forward-logs/enable-log-management-new-relic/)を使用してログ管理を有効にします。

* エラー、ディストリビューティッド（分散）トレーシング、ホストなど、[プラットフォーム全体の関連ログ データ](/docs/logs/logs-context/get-started-logs-context/#response-time-example)を調査します。

* [ログ UI](/docs/logs/ui-data/use-logs-ui/)を使用して、ログ、属性、パターン、ライブテール ログ、クエリ内の意味のあるパターンを見つけます。 それだけでなく、アラート条件を追加したり、ログ データをクエリしたり、

  <InlinePopover type="dashboards"/>

  を作成してチームと共有したりすることもできます。

## ネットワーク監視のインストール [#install-network]

まず「それはネットワークなのか？」を理解し、ネットワークテレメトリーを収集し、New Relicプラットフォームの機能を使用して、各固有のテレメトリータイプをナビゲートして探索します。

どこから開始すればよいか分からない場合[ガイド付きデプロイメントウィザード](https://one.newrelic.com/network-performance-monitoring)をご確認ください。

追加されたいテレメトリーをすでにご存知ですか？[こちら](https://onenr.io/0VjYDqkZ3R0)からインスタントオブザーバビリティを開始します。

## 外形監視モニターをインストールする [#synthetic-monitors]

[外部監視を使用すると、](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring/) Web サイトまたは API を積極的に監視して、コンテンツが完全に機能していることを確認できます。 スクリプト化されたbrowserテストを世界中の場所からサイトに送信して、コンテンツが常にどこでも稼働していることを確認できます。

外形監視[では、コンテナ化されたプライベートミニオン](/docs/synthetics/synthetic-monitoring/private-locations/install-containerized-private-minions-cpms/)機能以外は、インストレーションは必要ありません。

## 任意のソースからのカスタムデータをレポート [#custom-data]

厳選されたエージェントとインテグレーションで必要なものが見つからない場合は、あらゆるソースからデータをレポートできるAPIと SDK へのアクセスを提供します。 詳細については、 [「任意のソースからカスタム データを New Relic に取得する」](/docs/data-apis/custom-data/get-custom-data-from-any-source/)ページを参照してください。
