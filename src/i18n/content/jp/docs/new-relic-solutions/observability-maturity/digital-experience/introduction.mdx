デジタル エクスペリエンスには、最初のページの読み込みから重要なビジネス アクションの完了まで、ユーザーが Web アプリケーションと対話する方法が含まれます。デジタル エクスペリエンスの最適化は、より速く、より信頼性が高く、より魅力的なユーザー インタラクションを通じて、顧客満足度、コンバージョン率、ビジネス収益に直接影響を与えます。

New Relicオブザーバビリティ プラットフォームは、包括的なフロントエンド監視、パフォーマンス分析、ユーザー ジャーニーの最適化を通じてデジタル エクスペリエンスを測定、監視、改善するのに役立ちます。 このガイドでは、事後的なエラー対応からプロアクティブなエクスペリエンスの最適化までを網羅した 3 段階の成熟度モデルを紹介します。

## 学ぶ内容

このデジタル エクスペリエンス成熟度進行モデルに従うことで、次のことが可能になります。

* ユーザーに影響を与える問題を検出するための信頼性の高いフロントエンド監視を確立する
* GoogleのCore Web Vitalsに基づいてパフォーマンスの最適化を実装する
* ビジネス価値を高めるユーザー重視のサービスレベル指標を作成する
* 顧客のデジタルジャーニーを包括的に理解する

## 成熟度進行モデル

成熟度進行モデルは、事後的な問題対応からプロアクティブなエクスペリエンスの最適化、そして最終的にはユーザー中心の信頼性管理の習得に至るまでの構造化されたプロセスを通じて組織を導きます。各レベルは、顧客満足度とビジネス成果を向上させるために設計された特定のプラクティスとメトリクスを備えた前のレベルに基づいています。

### レベル 1: 反応的 - 「壊れたものを直す」

**目標:**ユーザーに直接影響を与えるフロントエンドの問題を検出して解決します。

反応レベルでは、デジタル エクスペリエンス監視の基盤を確立します。エラー追跡とアラートを設定すると、ユーザーがアプリケーションで問題に遭遇したときにチームがすぐに知ることができます。

**主な実践:**

* ユーザーの機能を破壊する JavaScript エラーを監視する
* ユーザーエクスペリエンスを低下させるサーバーエラーを追跡する
* 重大なフロントエンド障害に対するアラートを確立する
* ユーザーに影響を与える問題をトリアージして修正するためのプロセスを構築する

次のルールは、リアクティブ デジタル エクスペリエンス機能を評価します。

* **[JavaScript エラー率](/docs/new-relic-solutions/observability-maturity/digital-experience/l1-js-error-rate):**ブラウザベースの JavaScript エラーを監視し、ユーザー エクスペリエンスの品質を最大化するためにエラーが評価および解決されていることを確認します。

* **[サービス エラー率](/docs/new-relic-solutions/observability-maturity/digital-experience/l1-service-error-rate):**ユーザーに影響を与えるサーバー側のエラーを追跡し、最適なユーザー エクスペリエンスを維持するためにエラーが認識され解決されていることを確認します。

### レベル2: プロアクティブ - 「パフォーマンスの最適化」

**目標:**パフォーマンスの最適化と監視を通じて、ユーザー エクスペリエンスを積極的に向上させます。

事後対応を超えて、パフォーマンス データを使用してユーザー エクスペリエンスの品質を理解し、Google のCore Web Vitalsなどの業界標準に基づいて最適化します。 このレベルでは、パフォーマンスの問題がユーザー満足度に影響を与える前にそれを防ぐことに重点が置かれます。

**主な実践:**

* Core Web Vitals監視して、高速で応答性の高いエクスペリエンスを保証します
* パフォーマンス予算と最適化目標を実装する
* リアルユーザー監視を使用して実際のユーザー体験を理解する
* 重要なユーザージャーニーとコンバージョンパスを最適化する

次のルールは、プロアクティブなデジタル エクスペリエンスの最適化を評価します。

* **[Core Web Vitals | LCP](/docs/new-relic-solutions/observability-maturity/digital-experience/l2-cwv-lcp) :** Largest Contentful Paint を測定して、ページがすばやく読み込まれ、ユーザーにすぐに価値が提供されるようにします。

* **[Core Web Vitals | CLS](/docs/new-relic-solutions/observability-maturity/digital-experience/l2-cwv-cls) :**累積レイアウトシフトを評価して、読み込み中にページが視覚的に安定していることを確認します。

* **[Core Web Vitals | INP](/docs/new-relic-solutions/observability-maturity/digital-experience/l2-cwv-inp) :** Next Paint へのインタラクションを監視し、ページがユーザー インタラクションに迅速に応答することを確認します。

### レベル 3: 習熟 - 「測定可能なビジネス価値を提供する」

**目標:**ビジネス目標と顧客満足度を直接サポートする一貫したデジタル エクスペリエンス品質を実現します。

習熟レベルでは、デジタル エクスペリエンスの実践により、信頼性が高く、パフォーマンスの高いユーザー エクスペリエンスを通じて明確なビジネス価値がもたらされます。 技術的なパフォーマンスをビジネスの成果に結び付けるサービスレベル指標を確立します。

**主な実践:**

* ビジネスへの影響を反映したユーザー体験 SLI を定義する
* デジタルエクスペリエンスの品質に関するサービスレベル契約を維持する
* デジタルエクスペリエンスの改善によるROIを実証する
* エクスペリエンスデータを活用して製品やビジネスの意思決定を推進する

次のルールでデジタル エクスペリエンスの習熟度を評価します。

* **[サービスレベル達成度](/docs/new-relic-solutions/observability-maturity/business-uptime/l3-service-level-attainment): ユーザー体験**サービスレベル目標の一貫した達成度を測定し、95% 以上の成功率が高品質のデジタル エクスペリエンスを示しています。

## デジタルエクスペリエンスの実践

これらの観察プラクティスは成熟度モデルの実行可能なコンポーネントであり、ユーザーの満足度とビジネスの成果において目に見える改善を実現できるようになります。

### [クオリティファウンデーション（QF）](/docs/tutorial-improve-customer-experience/intro-to-customer-experience)

顧客のデジタル エクスペリエンスの包括的な監視と測定を確立します。この練習は次のようなことに役立ちます:

* すべてのタッチポイントにわたる現在のユーザー体験の品質を理解する
* 改善のための最大の機会を特定する
* エクスペリエンス最適化の目標に向けた進捗状況を追跡する

### [ファネルの底の分析 (BotFA)](/docs/tutorial-improve-customer-experience/bottom-funnel-analysis)

重要なユーザー ジャーニーのパフォーマンスに焦点を当てて、コンバージョン率を最適化します。この方法により、次のことが可能になります。

* コンバージョンプロセスでユーザーが離脱する場所を特定する
* チェックアウト、サインアップ、その他の重要なビジネスフローを最適化
* エクスペリエンスの改善を収益への影響に直接結び付ける

### [サービスレベル管理（SLM）](/docs/tutorial-service-level-mgmt/intro-to-slm)

技術的なパフォーマンスとビジネス目標を一致させるユーザー体験サービスレベル指標を作成します。 このプラクティスでは以下が提供されます。

* ユーザー体験の品質について議論するための標準化された言語
* エクスペリエンス最適化の取り組みの明確な目標
* エクスペリエンスの向上による ROI を実証するビジネス中心のメトリクス

## はじめに

デジタルエクスペリエンスを向上させる準備はできていますか?現在の能力に基づいて開始する方法は次のとおりです。

### 現在の状態を評価する

1. **ユーザー体験監視を確認してください:**ユーザーがいつエラーやパフォーマンスの問題に遭遇したか知っていますか?
2. **パフォーマンス メトリクスを評価する:**ユーザーの満足度に実際に影響を与えるものを測定していますか?
3. **対応能力を確認してください:**チームはユーザーに影響を与える問題を迅速に特定して修正できますか?

### 出発点を選択してください

* **フロントエンド監視は初めてですか?**レベル1のエラー追跡と警告から始める
* **基本的な監視はありますか?**レベル2のCore Web Vitals最適化へジャンプ
* **成熟した監視実践?**デジタル エクスペリエンスのためのレベル 3 のサービスレベル管理に焦点を当てる

### 推奨される実装順序

1. **[JavaScript エラー率](/docs/new-relic-solutions/observability-maturity/digital-experience/l1-js-error-rate)**- フロントエンド確立エラー監視
2. **[サービスエラー率](/docs/new-relic-solutions/observability-maturity/digital-experience/l1-service-error-rate)**- ユーザーに影響を与えるバックエンドエラーを追跡します
3. **[Core Web Vitals監視](/docs/tutorial-improve-customer-experience/intro-to-customer-experience)**- パフォーマンスの最適化を実装する
4. **[品質基盤の実践](/docs/tutorial-improve-customer-experience/intro-to-customer-experience)**- 包括的な経験測定を構築する
5. **[サービスレベル管理](/docs/tutorial-service-level-mgmt/intro-to-slm)**- エクスペリエンスをビジネス成果に結び付ける

## 成功の測定

デジタル エクスペリエンスの成熟度の進捗を測定するには、次の主要な指標を追跡します。

* **エラー削減:** JavaScript とサービスのエラー率の低減
* **パフォーマンスの改善:** Core Web Vitals スコアの向上 (LCP、CLS、INP)
* **ユーザー満足度:**コンバージョン率の向上と直帰率の低下
* **ビジネスへの影響:**収益の向上メトリクスはエクスペリエンスの向上と相関関係がありました
* **応答効率:**ユーザーに影響を与える問題の迅速な検出と解決
