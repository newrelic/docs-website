インテリジェントなオブザーバビリティは、 New Relicのビジネス成果の価値を高めるすべての基盤です。 できるだけ簡単な言葉で言うと、オブザーバビリティ プログラムの目標を達成するためのデプロイメント、設定、および管理の実践に関するものです。 サービス配信資産のすべての要素からテレメトリを包括的に取得し、一貫性を保証する命名と編成の原則を開発することを目指します。

# 基盤的観測フレームワーク

インテリジェント オブザーバビリティは、オブザーバビリティ成熟度モデルのレベル 0 に相当します。これは、他のすべてのオブザーバビリティの価値推進要因を可能にする重要な基盤です。 高度なビジネス成果を追求する前に、組織は包括的な可視性とデータ組織の実践を確立する必要があります。

## レベル 0: インテリジェントな監視の基礎

**目標:**テクノロジー エコシステム全体にわたって包括的かつ整理されたテレメトリ収集を確立します。

この基礎レベルにより、高度な観察のユースケースをサポートするために必要な可視性とデータ組織が確保されます。 このレベルでの成功は、ビジネスの稼働時間、エンジニアリングの卓越性、デジタル エクスペリエンスの最適化の効率性を直接的に実現します。

**スコアカードのルール:**

* **[チームタグの適用範囲](/docs/new-relic-solutions/observability-maturity/intelligent-observability/l0-team-tag-coverage)：**エンティティが所有チームまたは組織機能に適切にタグ付けされていることを保証し、説明責任と効率的なインシデント対応を可能にします。

* **[環境タグのカバレッジ](/docs/new-relic-solutions/observability-maturity/intelligent-observability/l0-environment-tag-coverage):**エンティティが適切な環境指定 (本番、ステージング、開発) を持つタグであることを保証し、環境固有の分析とより安全な操作を可能にします。

* **[計測されていないエンティティ](/docs/new-relic-solutions/observability-maturity/intelligent-observability/l0-uninstrumented-entities): 監視範囲**が不完全なサービス エンティティを特定し、システム ランドスケープの盲点を排除します。

* **[ディストリビューティッド（分散）トレーシング カバレッジ : サービス インタラクションとリクエスト フローに対する](/docs/new-relic-solutions/observability-maturity/intelligent-observability/l0-distributed-trace-coverage)**エンドツーエンドの可視性を提供するために、ディストリビューティッド（分散）トレーシングの実装を評価します。

* **[合成カバレッジ](/docs/new-relic-solutions/observability-maturity/intelligent-observability/l0-synthetic-coverage):**ユーザーの視点からサービスの可用性とパフォーマンスの検証を保証するために、プロアクティブな監視カバレッジを評価します。

**ビジネスへの影響:**包括的な計装と組織は、すべての高度な観察実践に必要なデータ基盤を提供し、データドリブンの意思決定を可能にしながら平均検出時間と分解能を削減します。

# 重要な観察実践の実践

これらの基本的なプラクティスは、インテリジェントなオブザーバビリティ スコアカード ルールと連携して、卓越したオブザーバビリティへの包括的なアプローチを作成します。

## コア実装プラクティス

**サービスの特性:**

* **[適切なデータを取得する](/docs/tutorial-reporting-data/capture-the-right-data):**適切な Web テレメトリーを取得する包括的なテレメトリー収集戦略を実装して、顧客エクスペリエンスの低下を解決し、効果的な分析を可能にします。

**データガバナンス:**

* **[テレメトリーの最適化](/docs/tutorial-optimize-telemetry/intro-to-optimal-telemetry):**戦略的なデータ管理とコストの最適化を通じて、 New Relicで収集したテレメトリー データの全体的な価値を確保します。

**自動化と一貫性:**

* **[コードとしての観察](/docs/new-relic-solutions/observability-maturity/operational-efficiency/observability-as-code-guide):**一貫性、拡張性、持続的な価値提供のために観察ツールの設定を自動化します。

## 実装アプローチ

これらのプラクティスは、スコアカード ルールの実装をサポートする運用基盤を形成します。サービスの特性評価により適切なデータを収集できるようになり、データ ガバナンスによりデータを効率的に管理できるようになり、コードとしての観察により組織全体で実践を一貫して拡張できるようになります。

これらのプラクティスは、レベル 0 のスコアカード ルールと組み合わせることで、オペレーショナル リスクの成熟度のリアクティブ、プロアクティブ、および習熟レベルを順調に進むことができる包括的な基盤を構築します。

## 次のステップ

## インテリジェントな観察を始める

### 即時の行動

1. **現在の基盤を評価する:**レベル 0 のスコアカード ルールに照らして、既存の計装範囲、タグの一貫性、データ組織の実践を確認します。

2. **実装の優先順位付け:**チームと環境タグから始めて基本的なデータ組織を確立し、次に未計測エンティティ、ディストリビューティッド（分散）トレーシング、および外形監視のカバレッジ評価に進みます。

3. **重要なプラクティスを確立する:**スコアカード ルールの改善と並行して、サービスの特性評価、データ ガバナンス、およびオブザーバビリティをコード プラクティスとして実装します。

## 観察の旅の選択

インテリジェントなオブザーバビリティの基礎を確立した後、ビジネス目標に最も合致する価値推進要因を選択します。

**[ビジネスの稼働時間](/docs/new-relic-solutions/observability-maturity/uptime-performance-reliability/introduction/):**サービスの信頼性、インシデント管理、顧客エクスペリエンスの保護に重点を置きます。

**[エンジニアリングの卓越性](/docs/new-relic-solutions/observability-maturity/engineering-excellence/introduction/):**インフラストラクチャの最適化、変更管理、およびセキュリティ プラクティスを重視します。

**[デジタル エクスペリエンス](/docs/new-relic-solutions/observability-maturity/digital-experience/introduction/):**フロントエンドのパフォーマンス、ユーザー体験の最適化、顧客満足度メトリクスを優先します。
