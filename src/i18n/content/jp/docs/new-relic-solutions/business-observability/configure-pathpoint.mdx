Pathpoint は、ビジネス固有のニーズに合わせてカスタマイズできる柔軟な構成システムを提供します。構成プロセスには、目的のセットアップを指定する JSON ファイルの作成が含まれます。構成ファイルがロードされたら、タッチポイントとそのクエリを手動でさらに微調整できます。このレベルのカスタマイズにより、組織固有の要件に合わせて調整されたオブザーバビリティ ソリューションを作成できます。

このページでは、次のことを学びます。

* JSON 構成ファイルとは何ですか?
* 新しい JSON 構成ファイルをアップロードする
* 現在アクティブな JSON 構成ファイルをダウンロードする

### JSON 構成ファイルとは何ですか?

JSON (JavaScript Object Notation) はデータ構造であり、その基本機能は情報の交換を可能にすることです。この構造により、Pathpoint の実装を容易にする各要素とコンポーネントを特定できます。これには、その属性、クエリ、およびデータ出力の機能を理解することが含まれます。

/>

上の画像は、各段階をいくつかのステップに関連付けることができる方法を示しています。これらのステップは、単一のタッチポイントまたはいくつかのタッチポイントにリンクできます。新しいステージ、ステップ、およびタッチポイントをプログラムするには、新しい JSON 構成ファイルをアップロードします。

### 新しい JSON 構成ファイルをアップロードする

新しい JSON 構成ファイルをロードするには:

<Steps>
  <Step>
    New Relic アカウントから、 **Apps** > **Pathpoint**<img src={threeLines} class="inline" style={{height: "30px", width: "30px"}} alt="Three lines" title="Three lines">に移動し、</img>

    。

  </Step>

  <Step>
    **\[Json 構成]**をクリックします。

  </Step>

  <Step>
    これにより、ダイアログ ボックス ウィンドウが開きます。** \[説明]**で、新しい JSON 構成ファイルについて説明します。 次に、メモを入力し、 **\[ファイルから]** をクリックして JSON 構成ファイルを参照します。

    <img
      title="Upload new JSON config file"
      alt="Upload new JSON config file"
      src={uploadJson}
    />
  </Step>

  <Step>
    パスポイントの更新に使用する JSON 構成ファイルを見つけて、 **\[開く]**をクリックします。

    <img
      title="open new JSON config file"
      alt="open new JSON config file"
      src={openFile}
    />
  </Step>

  最後に、パスポイントが表示され、選択した特定の JSON ファイルが更新されます。
</Steps>

### 現在アクティブな JSON 構成ファイルをダウンロードする

Pathpoint は、現在の JSON 構成ファイルをコンピューターに直接ダウンロードする柔軟性を提供します。この機能は、構成ファイルを変更する必要がある場合に特に便利です。

現在アクティブな JSON 構成ファイルをダウンロードするには:

<Steps>
  <Step>
    New Relic アカウントから、 **Apps** > **Pathpoint**に移動します。 <img src={threeLines} class="inline" style={{height: "30px", width: "30px"}} alt="Three lines" title="Three lines">ここで、</img>

    ウィンドウの左上部分にあります。

  </Step>

  <Step>
    **\[Json 構成]**をクリックします。

  </Step>

  <Step>
    次に、下矢印をクリックします。

    <img
      title="download active JSON config file"
      alt="download active JSON config file"
      src={downloadFile}
    />
  </Step>

  アクティブな JSON 構成ファイルがコンピューターにダウンロードされました。
</Steps>
