New Relic AIは大規模言語モデル（LLM）とNew Relicのデータプラットフォームを用いて、システムやNew Relicプラットフォームを深く理解できるよう支援します。平易な言語で質問したり、問題に対応したり、テレメトリーデータを調べたりすることが可能です。New Relic AIは、以下の支援を目的としています。

* スタックの理解
* 問題の解決
* システム停止への対応
* 業務効率の向上

チャット、NRQLコンソール、UIツール、インテグレーション（ServiceNow Now Assistなど）、Model Context Protocol（MCP）を通じて操作できます。このシステムは、エンジニアリング業務の効率的かつ明確な遂行を支援します。

New Relic AIは、以下の3つのステップで作業を効率化します。

1. 「エラー率の高い項目を表示してください」といったプロンプトを入力すると、AIは推論能力を用いてその内容を解釈します。
2. 次に、New Relicアカウントから関連するメトリクスやログを取得・分析します。
3. 最後に、分かりやすい説明、要約、グラフを生成し、明確な回答を提供します。

## AIとのチャット [#ask-AI]

New Relic AIの利用には、いつでもインタラクティブなチャット機能を利用できます。ほとんどのNew Relicプラットフォームページの右上にある<DNT>**Ask AI**</DNT>ボタンをクリックするだけで利用可能です。

New RelicのAIアシスタントは、一般的な質問と自然言語を用いてエンジニアとコミュニケーションを取るように訓練されています。オンラインストアを運営していて、`Checkout service`のレスポンスタイムが遅れているというアラートが届き始めた場面を想定してみましょう。New Relic AIに問い合わせるだけで、迅速に問題を解決することができます。New Relic AIは、システム内の不具合箇所を特定し、グラフや表を使ってデータの分析を支援します。

## NRQLクエリの作成 [#NRQL-console]

[NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/)は、システムに関する洞察を得るための強力なツールです。一方で、急を要する場面では、必要なデータを取得するための適切なNRQLクエリを作成する時間がないこともあります。クエリがうまく機能しない場合は、AIアシスタントに相談してみてください。

<img title="New Relic AI NRQL " alt="A screenshot showing how to query New Relic AI" src="/images/ai_screenshot-full_new-relic-ai-nrql-query.gif" />

<figcaption>
  NRQLクエリに関するサポートが必要な場合は、<DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT>に移動し、画面左下の**Query your data** \[データのクエリ]を選択します。
</figcaption>

次のような質問をすることができます。

```
How is this service performing?
```

```
How has this service been performing over the past 24 hours?
```

## エラーログの分析の支援 [#errors]

エラーログの分析は困難な場合があります。New Relic AIは、エラーを詳しく把握したり、関連情報を照会したり、次の手順を相談したりするのに役立ちます。

UIの任意のログパネルから、ログエラーについてNew Relic AIと直接チャットできます。ただし、使い始めたばかりの場合は、以下の手順で進めることができます。

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT>に移動し、次に<DNT>**Logs**</DNT>に移動します。
2. <DNT>**All logs**</DNT>をクリックします。
3. `error`を検索します。
4. 表示されたログの行をクリックし、ログ詳細パネルの上部にある<DNT>**Explain this error**</DNT>ボタンを選択します。New RelicのAIアシスタントがエラーの概要を説明します。
   <img title="new relic ai log errors" alt="a screenshot depicting log errors explained by New Relic AI" src="/images/ai_screenshot-crop_new-relic-ai-log-errors.webp" />

スタックに関する情報をAIアシスタントに伝え、追加の質問をすることで、調査をさらに進めることができます。例えば、次のような質問をすることができます。

```
How often does this error occur? What's the error rate? Is it 1% or 50%?
```

```
Which hosts are throwing these errors? Is it across all of my hosts, or is it just affecting one or two of them?
```

## エラー受信箱のスタックトレースを分析する [#stack-traces]

New RelicのAIアシスタントを使用してスタックトレースを分析することで、コンテキストの切り替えを減らし、トリアージや分析を効率化できます。New Relic AIを使用してエラー受信箱のスタックトレースを調査する方法は以下のとおりです。

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT>に移動し、次に<DNT>**Errors inbox**</DNT>に移動します。
2. ワークロードを選択します。
3. **スタックトレースパネル**にある<DNT>**Explain this error**</DNT>ボタンを選択します。New Relic AIがエラーの概要を説明します。
   <img title="explain this error" alt="A screenshot showing how to use New Relic AI to investigate stack traces. " src="/images/ai_screenshot-full_explain-this-error-stack-traces.webp" />

追加の質問をすることで、さらに深く掘り下げることができます。例えば、次のように伝えることができます。

```
I have an Azure environment and am having issues with Redis being periodically unavailable. Can you tell me more about this error? How can I diagnose this issue? 
```

## アラートのギャップを埋める [#coverage-gaps]

チームやシステムが拡大するにつれて、起こり得る問題に先手を打つことが非常に重要になります。アラートは早期警戒システムですが、常に最新の状態に保つことが難しい場合があります。New Relic AIは、重要な通知の見落としが起こり得る領域を特定するのに役立ちます。

New Relic AIとのチャットで、次の質問をしてください。

```
Show me all of my entities that are not covered by an alert. 
```

```
Show me my OpenTelemetry services that are not covered by an alert.
```

## Syntheticモニターを作成する [#synthetic]

外形監視は、顧客に影響を与える前に問題を積極的に把握・解決するのに役立ちます。AIアシスタントとのチャットを通じて、Syntheticモニターを作成できるようになりました。New Relic AIを使用すると、選択した場所からPingチェックモニターを作成し、指定の頻度でURLの可用性を確認できます。

たとえば、

```
Create a synthetic monitor to check the URL 'newrelic.com' every 10 minutes and name it: 'my awesome monitor'.
```

New Relic AIを使用してPingチェックモニターを作成する方法：

1. **[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)** **All Entities**にアクセスします。
2. 右上にある**Ask AI**をクリックします。
3. 対象とするパラメーターに基づいてPingチェックモニターを作成するよう、AIに指示します。
4. **Create monitor**をクリックします。
5. 設定を確認・変更する場合は、**Go to monitor**を選択します。
6. より高度な設定が必要な場合や、異なるタイプのSyntheticモニターにアクセスしたい場合は、**Configure other monitors**を選択します。

## サポートチケットのステータスを確認する [#support]

New Relic AIにサポートチケットの要約を尋ねることで、解決までの時間を短縮できます。解決策と過去のイシューを確認し、管理を効率化します。また、チケットが未解決のままになっているか、解決済みとしてクローズできるかを確認することもできます。

AIとのチャットで、次の質問をしてください。

```
Summarize my support ticket(s).
```

```
What is the status of my support ticket(s)?
```

## ダッシュボードの仕組みを分かりやすく解説する [#dashboards]

ダッシュボードは非常に便利なツールですが、担当外のダッシュボードでテレメトリーを調査している場合は、情報を細部まで理解できない可能性があります。New Relic AIはユーザーのパーソナルデータアナリストとして機能し、New Relic UI上で直接操作することで、重要な情報を自動的に要約したり、隠れた情報を明らかにしたりできます。

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities)**</DNT>に移動し、次に<DNT>**Dashboards**</DNT>に移動します。
2. 分析するダッシュボードを選択します。
3. 右上にある**Ask AI**をクリックします。

基本的な質問をするだけでも、複雑なデータを明確にするのに役立つ詳細な要約が得られる場合があります。次のように質問してみてください。

```
What is on this dashboard?
```

```
Summarize this dashboard for me. 
```

## アクセスとデータの管理

[カスタムロールとグループ](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles)を使用して、New Relic AIへのアクセスを制限できます。New Relicのすべての標準的なRBACロールでは、New Relic AIがデフォルトで有効になっています。これらのロールの権限は編集できませんが、特定のアクセス権限を設定したカスタムロールを作成することでアクセスを制限できます。

## 精度とセキュリティ [#secure-data]

New RelicのAIアシスタントは、さまざまな入力情報を用いて応答を改善し、プロンプトに対して最適な提案を提供できるよう努めています。例えば、製品ドキュメント、幅広いサンプルクエリのライブラリ、テレメトリーデータなどの入力を使用します。

New Relicのセキュリティ対策については、[セキュリティポリシー](/docs/licenses/license-information/referenced-policies/security-policy/)をご覧ください。

## トラブルシューティング [#troubleshoot]

New Relic AIの使用中に以下のエラーメッセージのいずれかが表示された場合は、トラブルシューティング手順を参照してください。

**このアカウントのユーザーロールにNew Relic AIへのアクセス権がありません。別のアカウントを使用するか、New Relic管理者にお問い合わせください。**

* 問題：現在使用しているNew RelicアカウントでNew Relic AIが有効になっていないか、そのアカウントでユーザーに割り当てられているロールにNew Relic AIへのアクセス権がありません。
* 解決策：New Relic AI機能が有効になっている別のNew Relicアカウントからアクセスしてください。アクセス権があるはずなのに利用できない場合は、New Relicアカウントのアドミニストレーターに連絡して、アカウントでNew Relic AIを有効にするか、[ユーザーロールの権限](/docs/accounts/accounts-billing/new-relic-one-user-management/account-user-mgmt-tutorial/#roles)を調整するよう依頼してください。

**現時点では、New Relic AIはHIPAAまたはFedRAMPの対象となるアカウントで利用できません。別のアカウントをお試しください。**

* 問題点：規制上の制約により、New Relic AIは現在、HIPAAまたはFedRAMPの対象となるアカウントでは利用できません。
* 解決策：New Relic AIを使用するには、HIPAAまたはFedRAMPのコンプライアンス要件の対象とならない別のNew Relicアカウントからアクセスしてください。

## フィードバックをお寄せください [#feedback]

New Relic AIは、利用されればされるほど精度と応答性が向上していきます。私たちはお客様の望むAI体験の実現を目指しています。AIアシスタントとのやり取りでは、応答に対して「高評価」または「低評価」を付けてください。修正が必要な点や改善できる点があれば、ぜひお知らせください。ユーザーメニューに移動し、<DNT>**Help**</DNT>をクリックして、<DNT>**Give us feedback**</DNT>をクリックします。

また、New Relic AIは複数の言語を認識し、それらの言語で応答することができます。多言語対応の用例がある場合は、それをAIアシスタントでテストし、フィードバックをお寄せください。

New Relic AIの応答は、AIが生成した提案です。情報や回答を最新の状態に保つよう努めておりますが、AIの提案内容が常に正確、完全、誤りがないとは限りません。New RelicのAIアシスタントが提供する情報については、何らかの行動を起こす前に、他の情報源を参照して人の手で検証することをお勧めします。AIによる提案は、正確性に関するいかなる保証もなしに提供されます。
