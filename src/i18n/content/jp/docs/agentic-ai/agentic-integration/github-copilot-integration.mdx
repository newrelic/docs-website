New Relic エージェント for GitHub Copilot 拡張機能は、アプリケーションのパフォーマンスの最適化、インフラストラクチャの監視、デプロイメントの管理に合わせたインテリジェントなインサイトとガイダンスを提供し、すべて GitHub Copilot 内にシームレスに統合されます。

このガイドでは、GitHub Copilot 拡張機能内で New Relic エージェントをインストールして使用するプロセスについて説明します。

## GitHub Copilot に New Relic エージェントをインストールする [#install-new-relic-agent]

### 前提条件 [#prerequisites]

始める前に、セットアップ プロセスがスムーズに進むように、次の前提条件が満たされていることを確認してください。

* IDE がシステムにインストールされています。
* [GitHub Copilot](https://github.com/features/copilot)にアクセスできます。
* GitHub Copilot 拡張機能が IDE にインストールされます。 詳細については、 [GitHub Copilot のドキュメント](https://docs.github.com/en/copilot/managing-copilot/configure-personal-settings/installing-the-github-copilot-extension-in-your-environment)を参照してください。
* [GitHub Copilot 拡張機能ポリシー](https://docs.github.com/en/enterprise-cloud@latest/copilot/managing-copilot/managing-github-copilot-in-your-organization/managing-policies-for-copilot-in-your-organization#setting-a-policy-for-github-copilot-extensions-in-your-organization)が組織で有効になっています。

<Callout variant="tip" title="ヒント">
  特定のロールに対して GitHub Copilot へのアクセスを制限する場合は、GitHub で設定を構成します。 特定のロールまたは社員に対してNew Relic AIへのアクセスを制限するには、New Relic のロールベース アクセス制御 (RBAC) を使用できます。 詳細については、 [New Relic ユーザー管理](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#understand-concepts)を参照してください。
</Callout>

**New Relic エージェントを GitHub Copilot にインストールするには:**

1. GitHub MarketPlace で、 [New Relic エージェント](https://github.com/marketplace/newrelic-copilot-extension)に移動します。
2. **Add &gt; Install it for free**を クリックします。

エージェントがインストールされると、IDE の GitHub Chat 拡張機能内で使用できるようになります。 詳細については、 [「Copilot での New Relic エージェントの使用」](#communicate-with-new-relic-agent-in-gitHub-copilot)を参照してください。

## GitHub Copilot で New Relic エージェントと通信する [#communicate-with-new-relic-agent-in-gitHub-copilot]

Github Copilot 内の New Relic エージェントは、さまざまなタスクを支援し、価値のあるインサイトを提供します。 New Relic エージェントと対話するには、Copilot チャット入力ボックスに`@newrelic`と入力します。

### New Relicエージェントにサインインする [#sign-in]

New Relic エージェントと初めて会話を開始するときは、サインインすることがプロンプトされます。 指示に従ってサインインプロセスを完了してください。

### チャットメッセージの構造 [#chat-message-structure]

New Relic エージェントと対話するには、次のようにプロンプトを入力します。

`@newrelic <your message>`

どこ：

* `@newrelic`: New Relic エージェントによるインターラクション。
* `<your message>`: 処理のためにエージェントに送信するアクション。
