モデル コンテキスト プロトコル (MCP) は、AI モデルまたはエージェントが特定のシステムまたは環境に関する関連情報 (コンテキスト) を受信するための標準的な方法です。 このコンテキストは、AI が現在の状況を理解し、タスクをより正確に実行し、より有用な応答を提供するのに役立ちます。これまで、AI エージェントをオブザーバビリティ プラットフォームなどのさまざまなデータ ソースに接続することは、各 AI エージェントが各プラットフォームと通信するためのカスタムの、多くの場合複雑な統合を構築することを意味していました。 この断片的なアプローチは、開発とメンテナンスの多大なオーバーヘッドにつながり、AI 主導のインサイトの「統合スプロール」の一形態となります。

New Relic AI MCP サーバーは、この課題に直接対処します。これは New Relic によるこのプロトコルの実装であり、単一の集中型ブリッジとして機能します。多数の個別の統合を構築する代わりに、サポートされている開発ツール内の AI エージェントは、 New Relicのオブザーバビリティ プラットフォームに接続し、豊富なコンテキストと強力なツールを取得するための 1 つの一貫した方法を備えています。

AI 開発環境内で直接、自然言語を使用して質問したりデータを調査したりできます。MCP サーバーはrequestsを翻訳し、関連するメトリクスまたはログを取得し、より明確な回答と概要を提供します。

<Callout title="コンプライアンス制限" variant="important">
  セキュリティとコンプライアンスの要件により、アカウントまたはデータが FedRAMP または HIPAA コンプライアンス要件に該当する場合は、New Relic AI MCP サーバーを使用しないでください。このパブリック プレビュー機能を FedRAMP または HIPAA 規制対象アカウントで使用することは固く禁じられています。
</Callout>

## それがどのように役立つか

MCP サーバーは、エンジニアと運用チームに実用的なメリットを提供し、次の方法でユーザーと AI を支援します。

* **直接統合:** New Relicのプラットフォームと一部の AI 機能に開発環境から直接アクセスします。 これにより、異なるツール間の切り替えが回避されます。
* **より迅速なトラブルシューティング:** AI が提供するインサイトを使用して、問題を迅速に発見して修正し、エラー ログを分析し、考えられる原因を特定します。
* **自動化された操作:**集中ステータスの確認、インシデント レポートの生成、デプロイメントがシステムに与える影響の分析などのタスクを実行します。
* **シンプルなデータクエリ:**平易な英語で質問を作成すると、ユーザーがNRQLに精通していなくても、サーバーがそれをNew Relicクエリ言語 ( NRQL ) に変換して結果を取得します。

## サポートされているAI開発ツール [#supported-tools]

New Relic MCP は次の AI 開発環境で動作します。

* **Claude Code:** Claude のコマンドラインインターフェース
* **Claude Desktop:**インタラクティブ AI 開発のためのデスクトップ アプリケーション
* **VS Code:** MCP をサポートする統合開発環境
* **Windsurf:**クラウドベースのAI開発プラットフォーム
* **Gemini CLI:** Gemini のコマンドラインインターフェース

各環境には、ワークフローの設定に応じて独自の利点があります。プラットフォーム固有の設定手順については、[セットアップ ガイドを](/docs/agentic-ai/mcp/setup)参照してください。

## 前提条件 [#prerequisites]

New Relic MCP を使用するには、次のものが必要です。

* APIアクセス可能なNew Relicアカウント
* `Node.js` インストール済み（一部の認証方法で必要）
* インストールされているMCP対応クライアントの1つ

<Callout title="Group membership and roles" variant="important">
  Accessing the MCP server requires you to be a member of a group that has been assigned one of the following specific [organizational roles](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/#role-scopes): `Organization Read Only`, `Organization Manager`, or `Organization Product Admin`. Ensure this role is active before attempting access. If your group does not currently have one of these roles, contact your org admin (a user with the **Core** or **Full Platform** user type who has the `Authentication Domain Manager` or `Tenant Manager` role) to have it granted.
</Callout>

### 認証オプション

* **ユーザーAPIキー**:

  1. New Relic にログインします。
  2. **「ユーザーメニュー」→ APIキー」を**選択します。
  3. 新しいユーザー キーを作成するか、既存のユーザー キーをコピーします。
  4. キーの形式: `NRAK-XXXXXXXXXXXXXXXXXXXXX` 。

* **OAuth** : New Relic は事前設定された OAuth エンドポイントを提供します。

  * クライアントID: `pUWGgnjsQ0bydqCbavTPpw==`
  * 認証URL: `https://login.newrelic.com/login`
  * トークン URL: `https://mcp.newrelic.com/oauth2/token` (米国) および`https://mcp.eu.newrelic.com/oauth2/token` (EU)
  * スコープ: `["openid"]`

## パブリックプレビューアクセス

New Relic UI のプレビューとトライアル ページに移動し、New Relic AI MCP Server プレビューを有効にします。

1. New Relic UI で、左下隅にあるユーザー名をクリックします。
2. **Administration &gt; Previews &amp; Trials**を選択します。
3. New Relic AI MCP サーバープレビューを有効にします。
