New Relicの MCP サーバーは、AI エージェントが観測データにアクセスして分析できるようにする特殊なツールを提供します。 このリファレンス ガイドでは、利用可能なすべてのツールをカテゴリ別に整理しています。

## MCPツールの使い方 [#how-to-use]

MCP ツールは、AI 開発環境内の自然言語クエリを通じて呼び出されます。正確なツール名や問題を覚える必要はありません。 New Relic データについて知りたいことを説明するだけです。

**クエリの例:**

* 「私のアカウントにはどんなエンティティがありますか？」
* 「Web アプリケーションの最近のアラートを表示」
* 「過去 1 時間のトップ エラーは何ですか?」
* 「 NRQL実行してパフォーマンスを分析する」

AI ツールは適切な MCP ツールを自動的に選択し、結果をフォーマットします。

## フィルターツール `include-tags` [#filter-tools]

一部のツールは、複数の関連タグを持つタグであるため、複数のカテゴリに表示されます (たとえば、 `analyze_deployment_impact`は`incident-response`と`advanced-analysis`両方に関連します)。

requestsに`include-tags` `HTTP`ヘッダーを追加して送信することで、AI エージェントが受信するツールを動的にフィルタリングできます。 これにより、ツールの特定のサブセットのみをリクエストできるようになり、ツール コーパスを削減し、AI モデルの効率と精度を向上させるのに役立ちます。

フィルタリングに使用できるタグは、このリファレンスのカテゴリ ヘッダーに直接対応しています: `discovery` 、 `data-access` 、 `alerting` 、 `incident-response` 、 `performance-analytics` 、および`advanced-analysis` 。

### 実装例 [#example]

```json
"servers": {
    "new-relic-mcp-server": {
        "url": "https://mcp.newrelic.com/mcp/",
        "type": "http",
        "headers": {
            "api-key": "NRAK-****",
            "include-tags": "discovery,alerting"
        }
    }
}
```

この例では、AI エージェントには、検出 (例: `get_entity` ) とアラート (例: `list_alert_policies` ) に分類されたツールのみが表示され、モデルが現在のタスクに最も関連性の高い機能のみに焦点を当てるようになります。

<Callout variant="important">
  **セキュリティとアクセス:** AI エージェントが以下にリストされているツール ( `get_entity` 、 `analyze_deployment_impact` 、 `execute_nrql_query`など) を呼び出す場合、実行されるアクションは、セットアップ時に構成されたNew RelicユーザーAPIキーまたは OAuth プロファイルに付与された権限によって厳密に制御されます。 このツールは、New Relic のロールベース アクセス コントロール (RBAC) を介して対応するユーザー アカウントにアクセスが許可されているデータ、エンティティ、アカウントにのみアクセスできます。キーに特定のアクション (アラートの表示や NRQL の実行など) に対する権限がない場合、ツールの呼び出しは権限エラーで失敗します。
</Callout>

## エンティティとアカウントの管理（タグ：検出） [#entity-account]

* `convert_time_period_to_epoch_ms`: 時間間隔 (たとえば、「過去 30 分間」) をエポックミリ秒に変換します。
* `get_dashboard`: 特定のダッシュボードの詳細を取得します。
* `get_entity`: GUID で New Relic エンティティを取得するか、名前パターンで検索します。
* `list_related_entities`: 指定されたエンティティ`GUID`から 1 ホップ離れた (関連する) エンティティをリストします。
* `list_available_new_relic_accounts`: ユーザーが利用できるすべてのアカウント`IDs`を一覧表示します。
* `list_dashboards`: New Relic アカウントのすべてのダッシュボードを一覧表示します。
* `list_entity_types`: ドメイン/タイプ定義を含む New Relic エンティティ タイプの完全なカタログを一覧表示します。
* `search_entity_with_tag`: 特定のタグ キーと値を使用してエンティティを検索します。

## データ アクセス (タグ: data-access) [#data-access]

* `execute_nrql_query`: NRDB に対して NRQL クエリを実行します。
* `natural_language_to_nrql_query`: 自然言語リクエストを NRQL クエリに変換し、実行して結果を返すことでクエリを実行します。

## アラートと監視（タグ：アラート） [#alerts]

* `list_alert_conditions`: 特定のアラートポリシーのアラート条件の詳細をリストします。
* `search_incident`: 柔軟なフィルタリングを使用して、すべての集中イベント (オープンイベントとクローズイベントの両方) をリストします。
* `list_alert_policies`: 指定されたアカウントのアラート ポリシーを一覧表示します。オプションでポリシー名でフィルタリングします。
* `list_recent_issues`: 指定されたアカウントの New Relic で未解決の問題をすべて一覧表示します。
* `list_synthetic_monitors`: すべての合成モニター (サービスの可用性とパフォーマンスをチェックする自動テスト) を一覧表示します。

## まとめイベント対応 (タグ: インシデント対応) [#alertevents]

* `analyze_deployment_impact`: 特定のエンティティに対するデプロイメントのパフォーマンスへの影響を分析します。
* `generate_alert_insights_report`: 特定の問題に関する包括的なインテリジェンス分析レポートを生成します。
* `generate_user_impact_report`: 特定の問題に関するエンドユーザー影響分析レポートを生成します。
* `list_entity_error_groups`: 時間枠内でErrors Inboxから特定のエンティティのエラー グループを取得します。
* `list_change_events`: エンティティ`GUID`によって識別されるアプリケーションについて、New Relic からの変更イベントの履歴を一覧表示します。

## パフォーマンス分析 (タグ: Performance-analytics) [#performance]

* `analyze_entity_logs`: アプリケーションのログを分析して、エラー パターン、異常な動作、再発する問題を特定します。
* `analyze_golden_metrics`: ゴールデンメトリクス (主要な健康指標: スループット、応答タイム、エラー率、飽和) を分析します。
* `analyze_kafka_metrics`: 消費者のラグ、プロデューサーのスループット、メッセージのレイテンシ、パーティションのバランスを含む Kafka メトリクスを分析します。
* `analyze_threads`: スレッドの状態、CPU 使用率、メモリ消費量など、特定のエンティティのスレッド メトリクス データを分析します。
* `analyze_transactions`: 時間枠内で特定のエンティティのトランザクションを分析し、遅いトランザクションやエラーが発生しやすいトランザクションを特定します。
* `list_garbage_collection_metrics`: 指定されたアカウントとエンティティのガベージ コレクションとメモリ メトリクスを一覧表示します。
* `list_recent_logs`: 指定されたアカウントとエンティティ`GUID`の New Relic からの最近のログを一覧表示します。

## 高度な分析 (タグ: advanced-analysis) [#advanced-analysis]

* `analyze_deployment_impact`: 特定のエンティティに対するデプロイメントのパフォーマンスへの影響を分析します。
* `analyze_entity_logs`: アプリケーションのログを分析して、エラー パターン、異常な動作、再発する問題を特定します。
* `generate_alert_insights_report`: 特定の問題に関する包括的なインテリジェンス分析レポートを生成します。
* `generate_user_impact_report`: 特定の問題に関するエンドユーザー影響分析レポートを生成します。
* `natural_language_to_nrql_query`: 自然言語リクエストを NRQL クエリに変換し、実行して結果を返します。
