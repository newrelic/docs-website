New Relic のエージェント統合内の Change Intelligence ツールは、パートナーとのエージェント統合を通じてアプリケーションのパフォーマンスを評価する能力を強化します。 このツールを使用すると、ユーザーはアプリケーション デプロイメント後のパフォーマンス メトリクスとシステム動作について問い合わせることができ、インサイトを提供してデプロイメント後の効率的な分析を促進できます。 このツールは、新しいデプロイメントごとに、アプリケーションのパフォーマンスと安定性に関する詳細なインサイトを提供します。 これにより、発生する可能性のある問題を迅速に特定して対処することができ、スムーズで信頼性の高いユーザー エクスペリエンスが保証されます。

New Relic プラットフォームにアクセスすると、アプリケーションに関連するすべてのデータを包括的に把握できるようになります。

### チェンジインテリジェンス機能 [#features]

Change Intelligence ツールは、デプロイメント後のアプリケーションのパフォーマンスの監視と分析に役立つさまざまな機能を提供します。

* デプロイメント後のアプリケーションのパフォーマンスを追跡します。

* **集計レポート**: ネイティブ GitHub Copilot エクスペリエンス内で直接、主要なパフォーマンス メトリクスの包括的な概要を提供します。

  * **セキュリティ脆弱性**: 前回のデプロイメント以降に検出された新しいセキュリティ脆弱性の特定。
  * **ログの問題**: エラー、警告、またはその他のログ関連の問題の検出。
  * **トレース情報**: 詳細な分析のための詳細なトレース データ。
  * **ゴールデンシグナル**: レイテンシ、トラフィック、エラー、または飽和度の異常な値を示すアラート。
  * **閾値アラート**: 事前に定義された閾値を超えたアラートを通知します。
  * **エラー報告**: 包括的なエラー追跡および報告機能。

* **直接リンク**: 発見された問題をさらに調査するための New Relic への直接リンクを提供し、より深いインサイトと分析を可能にします。

* **デプロイメント分析**: デプロイメント番号、ステータス、発生した問題など、デプロイメント プロセスの詳細な分析を提供します。

<Callout variant="tip" title="ヒント">
  New Relic Change Tracking (変更追跡機能) 機能と GitHub Actions の統合の詳細については、 [「GitHub Actions を使用した変更の](/docs/change-tracking/ci-cd/change-tracking-github-actions)追跡」を参照してください。
</Callout>
