[New Relic One](/docs/new-relic-one-overview)の基本的なUI機能は、プラットフォーム全体で広く利用できます。こうした機能を理解することで、貴社のデータの発見・分析がさらに容易になります。

## 検索バー [#search]

主な検索バーは、すべてのUIページの上部にあります。検索についてのヒント：

* 検索は、アクセス可能なNew Relicの全てのアカウントを対象に行うことができます。この点の詳細については、[クロスアカウントのアクセス](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data#account-access)をご覧ください。
* 廃止されるエンティティは、[8日間は検索にヒット](/docs/new-relic-one-entity-explorer#data-retention)します。

## チャートおよびクエリ機能 [#data-analysis]

すべてまたは大半のプラットフォームで利用可能なクエリおよびチャートの機能は次のとおりです：

* [クエリを作成](/docs/introduction-chart-builder)。任意のUIページの上部で**データのクエリ**を選択し、クエリを開始します。
* ダッシュボードにチャートを追加します。大半のチャートは、新規または既存ダッシュボードに追加できます。
* [チャートクエリを表示](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language#what-is-nrql)。一部チャートでは、そのチャートの生成に使用したNRQLクエリを表示できます。この機能を利用することで、チャートの理解を高めるか、新規クエリの根拠に利用できます。
* リンクを取得する。これを使用して、ウェブページにチャートを埋め込むためのURLを取得できます。埋め込みチャートには、作成時にあったメトリックスのみが含まれます。メトリックスがレポートされなくなった場合、チャートには表示されません。

  <Callout variant="tip">
    2020年7月に、iframeベースのチャート埋め込み機能を更新しました。詳細については、この[Explorers Hub投稿](https://discuss.newrelic.com/t/change-to-shareable-charts-in-apm-embedded-charts/102240)をご覧ください。
  </Callout>
* UIページ全体を共有UIページ全体を共有するには、URLをコピーして、New Relicにアクセス可能なユーザーと共有します。URLは、チャートの時間範囲設定を保持します。

## アカウントセレクター [#account-selector]

組織が複数のNew Relicアカウントにアクセスできる場合、一部のUIページの上部にアカウントセレクターがあります。アカウントへのアクセスの詳細については、[アカウントアクセス](/docs/accounts/accounts-billing/general-account-settings/factors-affecting-access-features-data)をご覧ください。

## アカウントとユーザーの設定 [#account-dropdown]

アカウント設定とユーザー設定を変更し、サポートとドキュメントリンクを取得するには、ユーザー名の隣の、UIの右上にある[アカウントドロップダウン](/docs/using-new-relic/welcome-new-relic/get-started/glossary#account-dropdown)を使用します。

## 製品リンク [#products]

UIの上部に、APMやBrowser、Infrastructureなどの以前の製品サイロへのリンクがあります。以前の別個の製品から一元化されたNew Relic Oneプラットフォームへの移行の詳細については、[New Relic One移行ガイド](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-one-transition-guide-july-2020)をご覧ください。
