分散型トレースやインシデントインテリジェンスなど、New Relic の一部の機能では、New Relic で監視しているエンティティと他の外部ホストやサービスとの関係を示すトポロジカルマップが表示されます。これらのマップは、問題を診断して修正するのに役立ちます。

## 地図表示 [#map-display]

これらのマップの正確な表示方法は、マップが配置されている New Relic の機能によって異なりますが、いくつかの共通した視覚的要素があります。一般的なマップ表示では、六角形のアイコンを使って、監視対象の [エンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic) やその他の外部ホストやサービスを表します。エンティティ間の関係は線で表され、矢印は呼び出し方向を示します。

![3つのエンティティを持つマップのスクリーンショット。](./images/context-map-three-entities.png "3つのエンティティを持つコンテキストマップです。")

## 稼働ステータス [#health]

一部のマップでは、エンティティの健康状態を示すために、当社の標準的な [健康状態の色](/docs/alerts-applied-intelligence/new-relic-alerts/alert-conditions/view-entity-health-status-find-entities-without-alert-conditions/#colors) を使用しています。

配布されたトレースマップは、この色で健康状態を表しています。

* 赤く塗られた六角形は、1つ以上のスパンエラーを意味します。
* オレンジ色の六角形が固まっている場合は、1つ以上の異常なスパンがあることを意味します。

## マップのツールチップ [#tool-tips]

コンテクストマップでは、マップの要素にカーソルを合わせると、その要素に関する追加情報が表示されます。

![マップエンティティの詳細情報のスクリーンショットです。](./images/context-map-entity-detail.png "マップエンティティの詳細情報です。")

<figcaption>
  地図上のエンティティにマウスオーバーすると、より詳細な情報が表示されます。
</figcaption>

マップのツールチップには

* エンティティ名とタイプ
* エンティティに存在する必須の [タグ](/docs/new-relic-one/use-new-relic-one/core-concepts/use-tags-help-organize-find-your-data)
* 直近30分のゴールデンシグナル
* エンティティが動作しているホスト（既知の場合
* このエンティティに手動で作成された関係を持つ他のエンティティ
* エンティティのミニオーバービューへのリンク
* エンティティのすべての依存関係を表示するリンク

## マップ関係の決定方法 [#relationships]

マップが存在するフィーチャーによって、検出される関係性が決まります。例

* Applied Intelligence：UIで [問題](/docs/alerts-applied-intelligence/applied-intelligence/incident-intelligence/use-incident-intelligence/#issue-summary) を選択すると、マップは [エンティティの関係](/docs/apis/nerdgraph/examples/nerdgraph-relationships-api-tutorial) に依存します。
* [分散型トレース](/docs/distributed-tracing/concepts/introduction-distributed-tracing): [UI でトレースを表示すると](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui#trace-map) 、マップにはトレースで見つかった関係が表示されます。
