[New Relic Explorer](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer-view-performance-across-apps-services-hosts) において、エンティティの **Dependencies** ページには、 [エンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic) に接続しているすべてのアプリケーション、サービス、データベース、ホストのリストがフィルタリング可能な形で表示されます。上流および下流の依存関係が表示され、それらを調査するためのパスが提供されています。

[サービスマップ](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one) と同様に、「依存性」ページは、お客様のアップストリームおよびダウンストリームのサービスがどのように接続されているかを理解するのに役立ちます。また、サービスマップ で使用されているのと同じ [カラーコーディングシステムを使用して、何がうまく機能していて、何が機能していないかを示します。](/docs/understand-dependencies/understand-system-dependencies/service-maps/service-maps-new-relic-one#alerts)

## 依存関係の表示 [#access]

エンティティに接続されているアプリケーション、サービス、データベース、およびホストの依存関係を表示する。

1. [one.newrelic.com](http://one.newrelic.com) 、 **Explorer** 、エンティティを選択してください。
2. Select **Dependencies**.
3. さらに掘り下げるには、アプリ、サービス、データベース、ホストのいずれかをフィルタリングします。

![New Relic Oneの依存関係](./images/new-relic-one-dependencies-UI.png "NR1_dependencies.png")

<figcaption>
  **[one.newrelic.com](http://one.newrelic.com) > Explorer> （エンティティを選択）> Dependencies**: エンティティに接続されているすべてのアプリ、サービス、データベース、ホストのリストと、それらの健全性の状態を色分けしてフィルタリング可能に表示します。
</figcaption>

依存関係のページをフィルタリングして、そのエンティティに報告する特定のものを表示することができます。依存関係には次のようなものがあります。

* **サービス**: [APM](/docs/apm)-監視対象のアプリケーションとサービス。
* **モバイルアプリケーション**：[モバイルアプリ](/docs/mobile-monitoring)。
* **ブラウザアプリケーション**：フロントエンドの[ブラウザアプリ](/docs/browser)。
* **外部サービス**: 外部サービス [APMで監視](/docs/apm/applications-menu/monitoring/external-services-page).外部サービスには、Webサービスなどのプロセス外のサービス、クラウド上のリソース、その他のネットワークコールなどがあります。
* **データベース**: アプリケーションのデータベースとキャッシュデータ。データベースはエージェントレスです。このため、サービスコールのみが New Relic に報告されるため、データベースにアラートを設定することはできません。
* **ホスト**: お客様の [インフラ](/docs/infrastructure) （サーバーとホスト）。
