New Relicは、以下のような独自のReact JavaScriptアプリケーションを構築するためのフレームワークを提供します。

* [New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one) プラットフォームに、他のダッシュボードやデータと一緒に常駐します。
* あなたの組織のためにカスタマイズされたビジュアライゼーションを提供します。
* New Relicが監視しているエンティティのデータや、他のサービスやAPIからのデータなど、任意のソースからのデータを表示できます。

New Relic One アプリでできることについては、このままお読みください。また、先に進んで自分のアプリを作り始めたい場合は、まず開発者サイト の [要件をお読みください。](https://developer.newrelic.com/build-apps/permission-manage-apps)

## [New Relic One: プログラム可能なプラットフォーム [#what-is-it]](https://developer.newrelic.com/build-apps/permission-manage-apps)

[私たちは、すべてのユーザーに最適な価値を提供する自動化されたユーザーエクスペリエンスを目指しています。しかし、組織によっては、標準的な可視化オプションでは対応できない独自のビジネスニーズがあることも承知しています。](https://developer.newrelic.com/build-apps/permission-manage-apps)

[New Relic Oneでは、お客様がプラットフォームの基本的な構成要素をコントロールできるようになりました。当社のエンジニアがNew Relic Oneの構築に使用しているのと同じツールを使って、お客様独自の組織構造やビジネスニーズに沿ったカスタムアプリケーションを構築することができます。React、GraphQL、NRQL（問い合わせ言語）の使い方がわかれば、アプリケーションの構築はわずか数分で完了します。](https://developer.newrelic.com/build-apps/permission-manage-apps)

[](https://developer.newrelic.com/build-apps/permission-manage-apps)[カスタムアプリケーションの構築については、開発者向けサイト（](https://developer.newrelic.com/build-apps) ）のガイドをご覧ください。また、独自のアプリケーションを構築する方法については、YouTubeの短いビデオ（約3分15秒）をご覧ください。

## スタートガイド [#get-started]

New Relic Oneを使えば、どんなに複雑なデータドリブンの課題でも解決できます。例えば

* API を使用して [あらゆるソースから New Relic にデータを取り込む](https://developer.newrelic.com/collect-data/collect-data-from-any-source) 。
* [そのデータをカスタムアプリケーションで視覚化](https://developer.newrelic.com/build-apps).
