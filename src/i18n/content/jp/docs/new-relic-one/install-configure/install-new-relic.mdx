New Relicと[New Relic One](/docs/new-relic-one/introduction-new-relic-one)へようこそ！

当社は、あらゆるデジタルソースからデータを取り込み、スタックで何が起こっているかを理解するのに役立つオブザーバビリティプラットフォームです。これにより、お客様の環境を前もって予測的に改善できます。クラウドでサービスを実行するコンテナからモバイルウェブサイトのショッピングカートボタンまで、アプリケーションとデバイスの全体像を提供します。New Relicを使用すれば、より優れたソフトウェアを構築し、顧客に価値を提供できるようになります。

[New Relicアカウントにサインアップ](https://newrelic.com/signup)し（永久無料）、当社のモニタリングサービスをインストールすると、[データの処理](/docs/data-apis/get-started/nrdb-horsepower-under-hood)を開始できます。シンプルで迅速です。

## ブラウザモニタリングのインストール [#browser-install]

New Relicのブラウザモニタリングは、ページ閲覧の人気とユーザー満足度（Apdex）、JavaScriptエラー解析、インタラクションイベント、AJAXリクエストなどのウェブページ全体のライフサイクルとセッションデータを測定します。

[ブラウザモニタリングのインストール](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent)を参照してください。

## 合成モニターのインストール [#synthetics]

[Syntheticモニタリング](/docs/synthetics)の使用により、ウェブサイトまたはAPIエンドポイントをプロアクティブにモニタリングして、コンテンツが完全に機能していることを確認できます。世界中の場所からスクリプト化されたブラウザテストをサイトに送信すると、どこにいても常にコンテンツが稼働していることを確認できます。

Syntheticsは[プライベートミニオン](/docs/synthetics/new-relic-synthetics/private-locations/install-configure-private-minions)機能を除いて、インストールする必要はありません。

## トラブルシューティング [#troubleshooting]

New Relic UIでデータの表示を開始するには、エージェントをインストールし、トラフィックを生成してから、数分待つ必要があります。データが表示されない場合は、[トラブルシューティングプロセス](/docs/agents/manage-apm-agents/troubleshooting/not-seeing-data)に従ってください。

サポートが必要な場合[サポートにお問い合わせください。](/docs/new-relic-solutions/solve-common-issues/find-help-use-support-portal)

## New Relic Oneを使用する [#use-nr]

準備はできましたか？それではNew Relic Oneでデータの探索を始めましょう！

* [基本的なUI機能](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/)の使用を開始します。
* [New Relicエクスプローラー](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer)でモニターしたエンティティを参照します。
* [データエクスプローラー](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer)で、クエリを行わずにデータを表示します。
* NRQL（当社のクエリ言語）の専門知識を使用して、[クエリビルダー](/docs/specify-data-chart#use-advanced)でカスタムチャートを作成します。
* [ダッシュボード](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards)の詳細を調べます。
