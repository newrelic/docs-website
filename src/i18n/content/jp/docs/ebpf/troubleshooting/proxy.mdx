New Relic<DNT>eBPF agent</DNT>が期待どおりにプロキシ経由でテレメトリーをルーティングしていない場合は、このページを使用して一般的なエラーを診断してください。セットアップ手順と設定リファレンスについては、[New Relic <DNT>eBPF agent</DNT>のプロキシの設定](/docs/ebpf/proxy-configuration)を参照してください。

## よくあるエラー [#common-errors]

<CollapserGroup>
  <Collapser id="error-400" title="HTTPプロキシがレスポンスコード400を返しました">
    **問題:** プロキシはHTTP<DNT>`CONNECT`</DNT>メソッドをサポートしていません。

    **解決策：**<DNT>`CONNECT`</DNT>対応のプロキシを使用してください。[サポートされているプロキシサーバー](/docs/ebpf/proxy-configuration#supported-proxies)を参照してください。
  </Collapser>

  <Collapser id="error-https-scheme" title="プロキシ URI では 'https' スキームはサポートされていません">
    **問題:** gRPCはプロキシスキームとして<DNT>`https://`</DNT>をサポートしていません。これはgRPCの制限であり、エージェントの制限ではありません。

    **解決策:** プロキシスキームを<DNT>`http`</DNT>に設定します（たとえば、<DNT>`otlpProxy.scheme: "http"`</DNT>）。お客様のテレメトリーデータは、引き続きプロキシトンネルを経由して、エンドツーエンドでTLS暗号化された状態で送信されます。プロキシ自体への接続のみがプレーンテキストです。
  </Collapser>

  <Collapser id="error-socket-closed" title="ソケットが閉じられたか、接続が拒否されました">
    **問題:** プロキシサーバーが実行されていないか、ホストまたはポートが正しくありません。

    **解決策:** [<DNT>`curl`</DNT>テストコマンド](/docs/ebpf/proxy-configuration#test-connectivity)を使用して、プロキシに到達可能であることを確認します。
  </Collapser>

  <Collapser id="error-connect-aborted" title="プロキシ CONNECT が中止されました">
    **問題:** ファイアウォールルール、ACL、またはサポートされていない宛先が原因で、プロキシは<DNT>`CONNECT`</DNT>リクエストを拒否しました。

    **解決策:** プロキシサーバーのログを確認し、エージェントの送信元IPからポート`443`への<DNT>`CONNECT`</DNT>requestsが許可されていることを確認してください。
  </Collapser>

  <Collapser id="error-tls-required" title="プロキシ設定が無視されました：gRPC HTTP CONNECTプロキシにはTLSが必要です">
    **問題:** プロキシは構成されていますが、TLSが無効になっています（たとえば、<DNT>`TLS_ENABLED=false`</DNT>）。gRPCのHTTP CONNECTプロキシ実装は、OTLPエンドポイントへの接続にTLSを使用する場合にのみ機能します。

    **解決策:** TLSが有効になっていることを確認してください（デフォルトで有効になっています）。その後、エージェントは設定されたプロキシを使用します。
  </Collapser>

  <Collapser id="error-credential-chars" title="プロキシユーザー名に特殊文字または大文字が含まれています">
    **問題:** プロキシのユーザー名またはパスワードに、gRPCのURIパーサーが受け付けない文字が含まれています。「[認証情報の文字制限](/docs/ebpf/proxy-configuration#credential-limitations)」を参照してください。

    **解決策:** ユーザー名が小文字の英数字で、パスワードに英字、数字、および許可された特殊文字のみを使用するプロキシアカウントを使用してください。
  </Collapser>
</CollapserGroup>
