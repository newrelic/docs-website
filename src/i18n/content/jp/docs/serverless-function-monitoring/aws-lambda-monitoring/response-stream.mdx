アプリケーションで 応答ストリーミング を使用する AWSLambda 関数を監視して、各呼び出しのパフォーマンスと動作を瞬時インサイトに取得できます。[](https://aws.amazon.com/blogs/compute/introducing-aws-lambda-response-streaming/)Node.jsこのソリューションは[AI モニタリング](/docs/ai-monitoring/intro-to-ai-monitoring/)と統合されており、モデルの応答を追跡し、AI ワークロードに関連する観察データを収集できます。 これらの機能を使用すると、サーバーレス アプリケーションのパフォーマンスを最適化し、AI 主導のプロセスの包括的な監視を維持できます。

このドキュメントは次のことに役立ちます:

* 監視応答ストリーミングの設定Lambda関数
* レスポンスストリーミングLambda関数で利用可能なメトリクスとデータを理解する
* New Relic UIを使用して、応答ストリーミングLambda関数のパフォーマンスを監視および観察する方法を学びます。

## Lambdaレスポンスストリーミングを設定する [#setup]

### 前提条件 [#prerequisites]

* Lambda関数が当社の[互換性と要件](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/compatibility-requirement-lambda-monitoring)を満たしていることを確認してください。
* Node.js で利用可能な[最新の Lambda レイヤー](https://layers.newrelic-external.com)を使用します。

### Lambdaレスポンスストリーミングを設定する [#configure-response-streaming]

Node.js Lambda関数の応答ストリーミング監視を有効にするには:

1. に[Lambda 監視 をインストールして設定します。New Relic](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/configure-serverless-aws-monitoring)

2. [](/docs/serverless-function-monitoring/aws-lambda-monitoring/instrument-lambda-function/env-variables-lambda)Node.jsレスポンスストリーミングLambda 関数に必要な 環境変数 を追加します。

3. 変更を保存し、関数をデプロイします。

   デプロイメント後、Lambda New Relicプラットフォームで レスポンスのストリーミングデータを表示できます。詳細については、 [Lambdaレスポンスのストリーミングデータを表示する」](#view-response-streaming-data)を参照してください。

## Lambdaレスポンスストリーミングデータにアクセスする [#access-ui]

プラットフォームで Lambdaレスポンスのストリーミングデータを表示するには:New Relic

1. <DNT>**[one.newrelic.com](https://one.newrelic.com) &amp;gt; left navigation pane &amp;gt; All Capabilities &amp;gt; Serverless Functions**</DNT>に移動します。

2. レスポンスストリーミングを使用する必要なLambda関数を選択します。

3. 応答ストリーミングLambda関数の概要、呼び出し、CloudWatchメトリクス、ディストリビューティッド（分散）トレーシングを表示します。

4. **More views** \[その他のビュー]セクションで、 **AI Responses** \[AI 応答]をクリックします。

   AI 応答の詳細については、 [「AI 応答の表示」](/docs/ai-monitoring/explore-ai-data/view-ai-responses)を参照してください。

## 応答ストリーミングの主要メトリクス [#key-metrics]

**CloudWatch metrics** \[CloudWatch メトリクス]ページで、応答ストリーミングに関連する次のメトリクスを監視します。

* **ストリーム送信バイト数:** Lambda関数によってストリーム送信されたバイトの平均数を表示します。
* **Streamed Outbound スループット:** Lambda関数によってバイトがストリーミングアウトされる平均レートを表示します。
