環境変数を使用することで、設定情報や機密情報をコードの外部に保存できます。以下の表は、New Relicを使用してLambda関数を監視するために設定できる必須変数とランタイム固有の変数を示しています。

## 必須の環境変数

ランタイムに関係なく、New RelicでLambda関数を監視するには次の環境変数を設定する必要があります。

<table>
  <thead>
    <tr>
      <th style={{ width: "200px" }}>
        環境変数
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        `NEW_RELIC_ACCOUNT_ID`
      </td>

      <td>
        New RelicアカウントIDを指定します。
      </td>
    </tr>

    <tr>
      <td>
        `NEW_RELIC_LAMBDA_HANDLER`
      </td>

      <td>
        関数の実際のハンドラを見つけるために使用されるハンドララッパーを指定します。Goランタイムの場合は、この変数は必須ではありません。
      </td>
    </tr>

    <tr>
      <td>
        `NEW_RELIC_LICENSE_KEY`
      </td>

      <td>
        New Relicのインジェストキーを指定し、Secrets Managerの設定を上書きします。
      </td>
    </tr>

    <tr>
      <td>
        `NEW_RELIC_APM_LAMBDA_MODE`
      </td>

      <td>
        Lambda関数のAPM監視を有効にするには、

        `true`

        に設定します。
      </td>
    </tr>

    <tr>
      <td>
        `NEW_RELIC_TRUSTED_ACCOUNT_KEY`
      </td>

      <td>
        New RelicアカウントIDまたは親ID（存在する場合）を指定します。
      </td>
    </tr>
  </tbody>
</table>
