ソフトウェア アプリケーションのセキュリティは、機密データを保護し、ユーザーの信頼を維持し、規制要件に準拠するために不可欠です。 New Relic インタラクティブ アプリケーション セキュリティ テスト（IAST））は、実行中のコードが悪用可能な脆弱性を調査することで、アプリケーションに対するサイバー攻撃や侵害を防ぐのに役立ちます。

New Relic IAST は、SAST、DAST、侵入テストなどの従来のアプリケーション セキュリティ ツールの制限の一部に対処するのに役立ちます。 IAST は、誤検知の発見を支援し、開発者の信頼を高め、効率を高めます。 QA テストでは、悪用可能な脆弱性を継続的に検出して優先順位を付けるという二重の役割と、リスクを排除する方法に関するガイダンスを得ることができます。

IAST は次のことに役立ちます。

* **Find and fix exploitable vulnerabilities** \[悪用可能な脆弱性を見つけて修正します]:スキャン結果を待つ必要はありません。
* **コードの迅速な出荷**: 比類のない検出精度と、誤検知の大幅な減少。
* **すべてのアプリケーションを確認して保護します**。コード、Web コンポーネント、設定データを完全に可視化します。
* 脆弱性を排除すると、**時間とコストが削減されます**。
* **ノイズを削減**: 静的アプリケーション セキュリティ テスト (SAST) 分析と動的アプリケーション セキュリティ テスト (DAST) 分析の両方を使用します。

IAST は[New Relic Vulnerability Management（脆弱性管理）](/docs/vulnerability-management/overview/)と完全に統合されているため、ソフトウェア開発ライフサイクル全体にわたって高リスクの脆弱性を継続的かつ迅速に発見、修正、検証できます。 IAST 独自のエクスプロイトの実証は、時間をより効率的に使用するのに役立ちます。 誤検知の追跡、脆弱性の再現、影響の少ないセキュリティ リスクへの対処に費やす無駄な時間を節約できます。

## 脆弱性重大度（CVSSスコア）について [#vuln-severity]

脆弱性を発見することは重要ですが、各脆弱性の重大度を推定できなければほとんど役に立ちません。 IAST は、 [Common Vulnerability Scoring System (CVSS)](https://nvd.nist.gov/vuln-metrics/cvss)の重大度スコアに従って、システム内で見つかった各脆弱性に定性的な重大度を割り当てます。 IAST はスコアリングのために CVSS バージョン 3 (CVSSv3) で動作します。
