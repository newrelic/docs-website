New Relic IAST テスト ステータス ページには、テストされたアプリケーション、脆弱性、カバレッジなどが一目でわかる一元的なビューが表示されます。 脆弱性をそのリスク スコアおよび潜在的なエクスポージャーとともに表示すると、修復の決定に優先順位を迅速に付けるのに役立ちます。

<DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > IAST**</DNT>に移動して、IAST テスト ステータス ページを開きます。

IAST テスト ステータス ページの左側のナビゲーション ペインには、次のセクションが含まれています。

* <DNT>**A: Overview**</DNT>: <DNT>**[one.newrelic.com](https://one.newrelic.com) > All capabilities > IAST**</DNT>に移動すると、IAST テスト ステータス ページが開きます。
* <DNT>**B: Findings**</DNT>: このセクションには、[脆弱性](/docs/iast/exploitable-vulns/)と[アプリケーション](/docs/iast/applications/)に関するページが含まれています。
* <DNT>**C: Settings**</DNT>: このセクションには、アプリケーションに悪用される可能性のある脆弱性がないかテストを開始するための[インストール ページ](/docs/iast/install/)が含まれています。

テスト ステータス ページのさまざまな部分は次のとおりです。

* <DNT>**Resolution status**</DNT>: [CVSS 重大度スコア](/docs/iast/introduction/#vuln-severity)(重大、高、その他) 別に、脆弱性の数の概要と解決ステータスを確認します。
* <DNT>**Application coverage**</DNT>: アプリケーションの合計数と、テスト済みおよび未テストのアプリケーションの数を表示する円グラフ。
* <DNT>**Potential exposure category**</DNT>: これは、構成されたすべてのアプリケーションで分析されたすべての API の性質に基づいて順序付けられたリストであり、テストに基づいて潜在的な脆弱性がどの OWASP カテゴリに分類されるかについての詳細を提供します。
* <DNT>**Detected exposure category**</DNT>: これは、OWASP カテゴリ別に検出されたすべての悪用可能な脆弱性の円グラフです。 これを潜在的な露出チャートと比較することで、スキャンしたアプリケーションがどのような種類の悪用可能な脆弱性の影響を受けやすいかを理解できます。
* <DNT>**Vulnerability detection timeline**</DNT>: 脆弱性検出のタイムラインを表示する面グラフです。
