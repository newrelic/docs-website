New Relic IAST テスト ステータス ページには、テストされたアプリケーション、脆弱性、カバレッジなどが一目でわかる一元的なビューが表示されます。 脆弱性をそのリスク スコアおよび潜在的な危険性とともに表示すると、修復の決定に優先順位を迅速に付けるのに役立ちます。

**[one.newrelic.com](https://one.newrelic.com) > All capabilities > IAST**に移動して、IAST テスト ステータス ページを開きます。

IAST テスト ステータス ページの左側のナビゲーション ペインには、次のセクションが含まれています。

* **A: Overview**: **[one.newrelic.com](https://one.newrelic.com) > All capabilities > IAST**に移動すると、IAST テスト ステータス ページが開きます。
* **B: Findings**: このセクションには、[脆弱性](/docs/iast/exploitable-vulns/)と[アプリケーション](/docs/iast/applications/)に関するページが含まれています。
* **C: Settings**: このセクションには、悪用可能な脆弱性についてアプリケーションのテストを開始するための[インストール ページ](/docs/iast/install/)が含まれています。

テスト ステータス ページのさまざまな部分は次のとおりです。

* **Resolution status** \[解決ステータス]: [CVSS 重大度スコア](/docs/iast/introduction/#vuln-severity)(重大、高、その他) ごとに存在する脆弱性の数の概要とその解決ステータスを確認します。
* **Application coverage** \[アプリケーション カバレッジ]: アプリケーションの総数と、テスト済みおよびテストされていないアプリケーションの数を示す円グラフ。
* **Potential exposure category** \[潜在的なエクスポージャ カテゴリ]: これは、すべてのアプリケーションにわたって分析されたすべての API の性質に基づいた順序付きリストであり、テストに基づいて、どの OWASP カテゴリが潜在的に脆弱であるかの詳細を提供します。
* **Detected exposure category** \[検出された露出カテゴリ]: これは、実際に悪用可能なすべての脆弱性を OWASP カテゴリごとに示した円グラフです。 これを潜在的暴露チャートと比較すると、どのタイプの脆弱性が最も影響を受けやすく、悪用される可能性があるかを理解できます。
* **Vulnerability detection timeline** \[脆弱性検出タイムライン]: 脆弱性検出タイムラインを面グラフで表示します。
