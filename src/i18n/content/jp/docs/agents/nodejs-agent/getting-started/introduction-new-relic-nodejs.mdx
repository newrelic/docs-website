New RelicからNode.jsを監視することにより、コードの行レベルまで遡って問題を特定して解決します。[サービスマップ](/docs/data-analysis/service-maps/get-started/introduction-service-maps)や[エラー解析](/docs/apm/applications-menu/error-analytics/error-analytics-explore-events-behind-errors)などの機能により、Node.jsエージェントではアプリケーション環境の全体像を把握できます。

多くのNodeアプリケーションのリクエストは生のURLをベースとしています。当社のソリューションは異なります。Node.js用New Relicは、アプリケーションへのリクエストをHTTPリクエストではなく、トランザクションにグルーピングできることを前提にしています。トランザクションは、1つ以上のリクエストパスに名前を付けて定義します。この名前は以下の目的で使用します。

* アプリケーションが時間を費やしている場所を視覚化します（トランザクションの内訳）。
* 遅いリクエストを特定します。
* メトリクスをグループ化します。
* アプリケーションのどの部分が遅いデータベースパフォーマンスの影響を受けているかを明確にします。

## 必要なこと [#requirements]

Node.jsエージェントをインストールする前に、[システム要件](/docs/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)をお読みください。

## インストレーション [#install]

エージェントをインストールするには、まず[New Relicのサインアップ](https://newrelic.com/signup)が必要です。サインアップは迅速で無料です — クレジットカードの番号をうかがうこともありません。ログインした後、説明に従うか、ランチャーを使用して入ってくるデータを取得します。

## インストゥルメンテーションを拡張 [#extend-instrumentation]

Node.jsエージェントをインストールしたら、インストゥルメンテーションを拡張します。

* VMの計測：[追加のネイティブモジュールをインストールして](/docs/agents/nodejs-agent/supported-features/node-vm-measurements)、[VM統計の詳細を表示](/docs/agents/nodejs-agent/supported-features/node-vms-statistics-page)します。
* [ディストリビューティッド（分散）トレーシング](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)を有効にします。
* [カスタムインストゥルメンテーション](/docs/agents/nodejs-agent/supported-features/nodejs-custom-instrumentation)：Node.jsカスタムインストゥルメンテーションを実装します。
* [カスタムメトリックスの収集：](https://docs.newrelic.co.jp/docs/agents/nodejs-agent/supported-features/nodejs-custom-metrics) API呼び出しを介してカスタムメトリックスを収集します。
* [Node.jsエージェントAPI：](/docs/agents/nodejs-agent/supported-features/nodejs-agent-api) API を使用して、Node.jsエージェントの機能を制御、カスタマイズ、または拡張します。
* [Browserタイミングプロセス：](/docs/agents/nodejs-agent/supported-features/page-load-timing-nodejs) エージェントを[Browserモニタリング](/docs/browser/new-relic-browser/getting-started/new-relic-browser)と統合して、ユーザーのアクティビティを可視化します。
* オープンソースのテレメトリー：[Node Telemetry SDK](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic) を使用して、独自のインテグレーションを作成できます。

## インストレーションのトラブルシューティング [#troubleshooting]

Node.jsエージェントで問題が発生した場合は、トラブルシューティング情報をご覧ください。

* [大量のメモリ使用](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-large-memory-usage-nodejs)：Node.js エージェントをインストールしてからメモリの使用量が増加した場合は、以下の解決策を試してください。
* [Node.jsのインストレーションのトラブルシューティング](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-your-nodejs-installation)：データが表示されない、ファイルにログを記録できない、およびその他のNode.jsエージェントのインストレーションに関する問題が発生した場合は、この手順に従ってください。
* [Browserインストゥルメンテーションのトラブルシューティング](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-page-load-timing-nodejs)：Browserデータに関する問題が発生した場合は、Node.jsに関する追加のヒントをご覧ください。

[トラブルシューティングに関するすべてのドキュメント](/docs/agents/net-agent/troubleshooting)を参照することもできます。さらに支援が必要な場合は、[support.newrelic.com](https://support.newrelic.com)でサポートを受けてください。

## ソースコードをチェック [#source-code]

Node.jsエージェントは、オープンソースソフトウェアです。つまり、[ソースコードを参照](https://github.com/newrelic/node-newrelic "新しいウィンドウにリンクが開きます。")して改善を送信したり、独自のフォークを作成して構築したりできます。詳細については、[README](https://github.com/newrelic/node-newrelic/blob/main/README.md "新しいウィンドウにリンクが開きます。") を参照してください。
