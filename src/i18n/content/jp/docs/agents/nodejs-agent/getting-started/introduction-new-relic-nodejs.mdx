New RelicからNode.jsを監視することにより、コードの行レベルまで遡って問題を特定して解決します。[サービスマップ](/docs/data-analysis/service-maps/get-started/introduction-service-maps)や[エラー解析](/docs/apm/applications-menu/error-analytics/error-analytics-explore-events-behind-errors)などの機能により、Node.jsエージェントはアプリケーション環境の全体像を把握するのに役立ちます。

<Callout variant="tip">
  Node.jsやその他のエージェント、およびその他の[オブザーバビリティプラットフォーム](https://one.newrelic.com)を使用するには、New Relicファミリーに参加してください。[サインアップ](https://newrelic.com/signup)して、わずか数秒で無料アカウントを作成できます。次に、毎月最大で100GBのデータを無料で取り込みます。期間無制限です。
</Callout>

## 重要な理由 [#why]

多くのNodeアプリケーションのリクエストは生のURLをベースとしています。当社のソリューションは異なります。Node.js用New Relicは、アプリケーションへのリクエストをHTTPリクエストではなく、トランザクションにグルーピングできることを前提にしています。トランザクションは、1つ以上のリクエストパスに名前を付けて定義します。この名前は以下の目的で使用します。

* アプリケーションが時間を費やしている場所を視覚化します（トランザクションの内訳）。
* 遅いリクエストを特定します。
* メトリクスをグループ化します。
* アプリケーションのどの部分が遅いデータベースパフォーマンスの影響を受けているかを明確にします。

## インストレーション [#install]

エージェントをインストールするには、まず[New Relicのサインアップ](https://newrelic.com/signup)が必要です。ログインした後、説明に従うか、ランチャーを使用して入ってくるデータを取得します。

* [システム要件](/docs/agents/nodejs-agent/getting-started/compatibility-requirements-nodejs-agent)を確認します。
* [インストールドキュメントを読みます](/docs/agents/nodejs-agent/installation-configuration/install-nodejs-agent)。
* New Relic UIに直接移動して、[Node.jsデータを追加](https://one.newrelic.com/launcher/nr1-core.settings?pane=eyJuZXJkbGV0SWQiOiJ0dWNzb24ucGxnLWluc3RydW1lbnQtZXZlcnl0aGluZyJ9&cards[0]=eyJuZXJkbGV0SWQiOiJzZXR1cC1uZXJkbGV0cy5zZXR1cC1ub2RlLWludGVncmF0aW9uIiwiYWNjb3VudElkIjoyNjQwNDA5fQ==&platform[accountId]=1)します。

## インストレーションのトラブルシューティング [#troubleshooting]

Node.jsエージェントで問題が発生した場合は、トラブルシューティング情報をご覧ください。

* [大量のメモリ使用](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-large-memory-usage-nodejs)：Node.js エージェントをインストールしてからメモリの使用量が増加した場合は、以下の解決策を試してください。
* [Node.jsのインストレーションのトラブルシューティング](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-your-nodejs-installation)：データが表示されない、ファイルにログを記録できない、およびその他のNode.jsエージェントのインストレーションに関する問題が発生した場合は、この手順に従ってください。
* [Browserインストゥルメンテーションのトラブルシューティング](/docs/agents/nodejs-agent/troubleshooting/troubleshooting-page-load-timing-nodejs)：Browserデータに関する問題が発生した場合は、Node.jsに関する追加のヒントをご覧ください。

[トラブルシューティングに関するすべてのドキュメント](/docs/agents/net-agent/troubleshooting)を参照することもできます。さらに支援が必要な場合は、[support.newrelic.com](https://support.newrelic.com)でサポートを受けてください。

## ソースコードをチェック [#source-code]

Node.jsエージェントは、オープンソースソフトウェアです。[ソースコードを参照](https://github.com/newrelic/node-newrelic)して改善を送信したり、独自のフォークを作成して構築したりできます。詳細については、[GitHubのnode-newrelic README](https://github.com/newrelic/node-newrelic/blob/main/README.md)を参照してください。
