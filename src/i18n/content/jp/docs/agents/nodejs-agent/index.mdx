New RelicのNode監視でコードの行レベルまで問題点を特定して解決します。[サービスマップ](/docs/data-analysis/service-maps/get-started/introduction-service-maps)や[エラー解析](/docs/apm/applications-menu/error-analytics/error-analytics-explore-events-behind-errors)などの機能により、Node.js用New Relicではアプリケーション環境の全体像を把握できます。

* [New Relic APM](/docs/apm/new-relic-apm/getting-started/introduction-new-relic-apm)、[New Relic Browser](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser)、その他の[New Relic製品](https://newrelic.com/products)の詳細なダッシュボードでNode.jsデータを監視。
* [New Relic Insight](/docs/insights/using-insights-ui/getting-started/introduction-new-relic-insights)を使用して、データをクエリ、可視化、そして共有。
* [ディストリビューティッド（分散）トレーシング](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)を有効化して、最新の分散アーキテクチャを可視化。

エージェントを最新の状態に保ち、最新機能へのアクセスを確保するために、[Node.jsエージェントのリリースノート](/docs/release-notes/agent-release-notes/nodejs-release-notes)を参照ください。
