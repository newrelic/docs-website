PythonエージェントはPythonアプリケーションをモニターし、[パフォーマンスの問題](#monitor-performance)の特定と解決を支援します。パフォーマンスの監視を拡張して[ビジネスデータ](#business-data)を収集して分析し、顧客体験の向上およびビジネスにおけるデータ駆動型の意思決定を支援することもできます。[カスタムインストゥルメンテーションとAPI](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation)の柔軟なオプションにより、Pythonエージェントには、アプリケーションに必要なデータをカスタマイズする多くの構築ブロックがあります。

Pythonは、Django、Gunicorn、WSGI、CherryPy、uWSGIなどのさまざまな[ウェブフレ－ムワークとホスティングメカニズム](/docs/agents/python-agent/getting-started/instrumented-python-packages)で動作します。Pythonエージェントは、[Google App Engine フレキシブル環境](/docs/agents/python-agent/hosting-services/install-new-relic-python-agent-gae-flexible-environment)にインストールすることもできます。

## アプリケーションパフォーマンスの監視 [#monitor-performance]

Pythonエージェントを[インストール](#installation)すると、Pythonエージェントはアプリケーションに関するデータの収集を開始します。[New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)では、データはチャートとテーブルとして表示されます。

**アプリケーションの全体像を表示する**

* アプリケーションの[Apdex（ユーザー満足度）](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction)を監視します。
* [概要ページ](/docs/apm/applications-menu/monitoring/apm-overview-page)でアプリケーションの概要を入手する。
* [ディストリビューティッド（分散）トレーシング](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing)を有効にして、多くのサービスを含むアーキテクチャ全体の活動を表示します。
* [Infrastructureモニタリング](/docs/infrastructure/new-relic-infrastructure/getting-started/introduction-new-relic-infrastructure)をインストールして、アプリケーションの詳細なサーバー/ホストデータを表示します。

**エラーと問題をすばやく検知する**

* ビジネス固有の[キートランザクション](/docs/apm/transactions/key-transactions/key-transactions-tracking-important-transactions-or-events)を追跡します。
* 重要なメトリックスに対して[カスタムダッシュボード](/docs/dashboards/new-relic-dashboards/custom-dashboards/custom-dashboards)を作成します。
* エラーや問題が発生したら、ユーザーに影響が及ぶ前にチームに[アラート](/docs/alerts/alert-policies/understanding-alert-policies/alerting-new-relic)を発信します。
* [デプロイメント後](/docs/apm/applications-menu/events/deployments-page)のパフォーマンスを表示します。

**パフォーマンスの詳細にドリルダウンします。**

* コードレベルの[トランザクショントレース](/docs/apm/transactions/transaction-traces/transaction-traces)を調査します。
* [データベースクエリトレース](/apm/transactions/transaction-traces/troubleshoot-sql-statements)を調査します。
* [エラートレース](/docs/apm/applications-menu/events/view-apm-error-analytics)を調査します。
* [スレッドプロファイラーセッション](/docs/apm/applications-menu/events/thread-profiler-tool)を使用して、サンプリングしたスレッドの詳細なスタックトレースを確認します。

## Pythonエージェントをインストール。 [#installation]

Pythonエージェントをインストールする前に、お使いのシステムが[システム要件](/docs/agents/python-agent/getting-started/compatibility-requirements-python-agent)を満たしていることを確認します。[New Relicアカウント](/docs/accounts-partnerships/accounts/account-setup/create-your-new-relic-account)を作成する必要もあります。

当社は、Django、WSGI、Gunicornなどの多くのウェブフレームワークとライブラリを標準でサポートしています。[サポートされているウェブフレームワーク](/docs/agents/python-agent/getting-started/instrumented-python-packages)を使用すると、インストレーションは簡単です。サポートされていないフレームワークを使用する場合は、アプリケーションのコードやウェブサーバーファイルに多少の追加が必要です。

ほとんどのセットアップで使用できる簡単迅速なインストールプロセスについては、次の簡単な手順に従ってください。

1. Pythonパッケージをダウンロードしてインストールする
2. 設定ファイルを作成する
3. Pythonエージェントとアプリケーションをインテグレーションする

## 非ウェブスクリプト、バックグラウンドタスク、および関数の監視 [#non-web]

Pythonエージェントでは、非ウェブスクリプト、ワーカープロセス、タスク、および関数もモニターできます。非ウェブトランザクションのインストレーションプロセスは、ウェブアプリケーションのインストールプロセスとほとんど同じですが、1つ異なる点があります。[インストール手順](/docs/agents/python-agent/getting-started/python-agent-quick-start)で説明されている標準的なインテグレーションプロセスに従う代わりに、監視する関数を手動で「ラップ」することです。詳細については、[非ウェブタスクとプロセス](/docs/agents/python-agent/supported-features/python-background-tasks)をご覧ください。Celeryタスクを監視する方法については、[Celeryバックグラウンドタスク](/docs/agents/python-agent/back-end-services/python-agent-celery)をご覧ください。

## インストレーション後の手順 [#instrumentation]

エージェントを起動した後に推奨する手順は次のとおりです。

* データを検索し、ユーザーインタフェースに慣れます。
* Full-Stack Observabilityについてのドキュメントを読みます。たとえば、[Full-Stack observability](/docs/full-stack-observability/monitor-everything/get-started-new-relic-monitoring-tools/get-started-full-stack-observability)と[APMページ](/docs/apm)の機能について読みます。
* [アプリケーション名](/docs/agents/manage-apm-agents/app-naming/rename-your-application)、またはその他の[設定オプション](/docs/agents/python-agent/installation-configuration/python-agent-configuration)を変更します。
* デフォルトで監視されないアプリケーションのアクティビティに対する[カスタムインストゥルメンテーション](/docs/agents/python-agent/custom-instrumentation/python-custom-instrumentation)の設定方法を学びます。
* これらのオープンソース テレメトリー ツールを検討してください。[OpenCensus エクスポーター](/docs/integrations/open-source-telemetry-integrations/open-source-telemetry-integration-list/new-relics-opencensus-integration) および [Python Telemetry SDK](/docs/data-ingest-apis/get-data-new-relic/new-relic-sdks/telemetry-sdks-send-custom-telemetry-data-new-relic)。

## トラブルシューティング [#troubleshooting]

インストールプロセスが完了したら、[APM UI](/docs/apm/applications-menu/monitoring/apm-overview-page)にデータが5分以内に表示されます。表示されない場合は、以下のトラブルシューティングリソースを使用します。

* データが表示されない場合は、[トラブルシューティング手順](/docs/agents/python-agent/troubleshooting/no-data-appears-python)に従います。
* 新しいホストでPythonエージェントをインストールまたは実行する際に問題が発生する場合は、[パッケージが正しくインストールされていること、](/docs/agents/python-agent/installation-configuration/testing-python-agent)および New Relicのデータコレクターサービスに接続できることを確認します。
* その他の問題については、[トラブルシューティングドキュメントの一覧](/docs/agents/python-agent/troubleshooting/)をご覧ください。

## ソースコードをチェック [#source-code]

Pythonエージェントは、オープンソースソフトウェアです。つまり、[ソースコードを参照](https://github.com/newrelic/newrelic-python-agent "新しいウィンドウにリンクが開きます。")して改善を送信したり、独自のフォークを作成して構築したりできます。詳細については、[README](https://github.com/newrelic/newrelic-python-agent/blob/main/README.rst "新しいウィンドウにリンクが開きます。") を参照してください。
