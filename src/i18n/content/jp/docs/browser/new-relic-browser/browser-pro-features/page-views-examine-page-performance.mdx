ブラウザ監視の**ページビュー**ページは、サイトのページのパフォーマンスに関する洞察を提供します。並べ替えオプションは、ページの読み込み時間の割合、平均ページの読み込み時間、またはスループットに関する詳細を見つけるのに役立ちます。表示された上位のWebページに関する詳細情報、タイミングとスループットの詳細、およびブラウザートレースまたはセッショントレースへのリンクも利用できます。

当社のブラウザ機能には、 [シングルページアプリ（SPA）の監視](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring)が含まれます。 SPAモニタリングをオプトインしている場合、SPA**ページビューの**UIは異なります。必要なビュー（SPAまたは標準ブラウザー）を切り替えることができます。

## 要約情報を表示する [#dashboard]

ブラウザーの監視では、ページ ビューが [URL](/docs/browser/new-relic-browser/getting-started/url-whitelists-grouping-browser-metrics)として一覧表示されます。これにより、特定のフロントエンド ページ ビューを簡単に識別できます。たとえば、 `/browse/plans`です。

  title="Page views UI overview"
  alt="A screenshot of the Page views UI in New Relic"
  src={browserPageviewsOverview}
/>

<figcaption>
  **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (アプリを選択) > Page views**: このページには、ページの読み込み時間、スループット、上位トランザクション、およびトレースに関する詳細情報が表示されます。
</figcaption>

アプリのエンドユーザーがどのページを見ているかの詳細情報を見るには

1. **[one.newrelic.com >All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (アプリを選択) > Page views**に移動します。
2. 必要に応じて、SPAまたは標準のブラウザアプリのページビューを切り替えます。
3. ページ読み込みタイミング情報のビューのタイプを、[グラフ（デフォルト）、ヒストグラム、またはパーセンタイル](/docs/apm/apm-ui-pages/monitoring/response-time-chart-types-apm-browser/)として選択します。

## パフォーマンスの詳細を調べる [#view_details]

ページビューの概要から、URLをクリックして、ページの読み込み、 [APM](/docs/apm/apm-ui-pages/features/request-queuing-tracking-front-end-time/)または[分散トレース](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui)のフロントエンドとバックエンドのタイミング、スループット、 [ブラウザーの種類](/docs/browser/new-relic-browser/additional-standard-features/browsers-problem-patterns-type-or-platform)、および[セッショントレース](/docs/browser/browser-monitoring/browser-pro-features/session-traces-explore-webpages-life-cycle)に関するパフォーマンスの詳細を表示します。

<img
  title="Page view details"
  alt="A screenshot of the page view details UI in New Relic"
  src={browserPageLoadPerformance}
/>

ここから、次のこともできます。

* ページの読み込みとスループットのパフォーマンスを昨日と先週と比較します。
* Webトランザクションのアプリ応答時間の応答時間を表示します。
