<InlinePopover type="browser" />の<DNT>**Page views**</DNT>ページは、サイトのページのパフォーマンスとユーザー エクスペリエンスにインサイトを提供します。 並べ替えオプションを使用すると、ページロードタイムのパーセンテージ、平均ページロードタイム、またはスループットに関する詳細を見つけることができます。 また、ユーザーのフラストレーション シグナルとパフォーマンスへの影響メトリクスを表示したり、セッション リプレイにアクセスしたり、閲覧された上位の Web ページに関する詳細情報を探索したりすることもできます。

当社のブラウザ機能 [には、シングルページ アプリ (SPA) の監視](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring)が含まれます。 SPA 監視にオプトインしている場合は、SPA <DNT>**Page views**</DNT> UI異なります。 必要なビュー（SPA または標準browser ）を切り替えることができます。

## 要約情報を表示する [#dashboard]

ブラウザーの監視では、ページ ビューが [URL](/docs/browser/new-relic-browser/getting-started/url-whitelists-grouping-browser-metrics)として一覧表示されます。これにより、特定のフロントエンド ページ ビューを簡単に識別できます。たとえば、 `/browse/plans`です。

<img title="Page views UI overview" alt="A screenshot of the Page views UI in New Relic" src="/images/browser_screenshot-full_pageviews-overview.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT>: このページでは、ページロードタイム、完了、top トランザクション、trace に関する詳細情報を提供します。
</figcaption>

アプリのエンドユーザーがどのページを見ているかの詳細情報を見るには

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views**</DNT>に移動します。
2. 必要に応じて、SPAまたは標準のブラウザアプリのページビューを切り替えます。
3. ページ読み込みタイミング情報のビューのタイプを、[グラフ（デフォルト）、ヒストグラム、またはパーセンタイル](/docs/apm/apm-ui-pages/monitoring/response-time-chart-types-apm-browser/)として選択します。

## ユーザーへの影響を理解する メトリクス [#user-impact-metrics]

ページビューページには、パフォーマンスの問題がユーザーエクスペリエンスにどのように影響するかを理解するのに役立つ「ユーザーへの影響」ペインが含まれています。

### フラストレーションの信号 [#frustration-signals]

<DNT>**Frustration signals**</DNT>には、選択した期間内にフラストレーション指標を含むユーザー セッションの割合が表示されます。フラストレーションのシグナルには次のようなものがあります:

* **レイジクリック**：ユーザーが1秒以内に同じ要素を4回以上素早くクリックする
* **デッドクリック**: 2秒以内に変化が検知されないクリック
* **エラークリック**: クリック後2秒以内にJavaScriptエラーが発生した

### パフォーマンスリスク [#performance-impact]

<DNT>**Performance risk**</DNT>には、選択した期間にパフォーマンスの問題の影響を受けたユーザー セッションの割合が表示されます。JavaScript エラーの場合、処理されない例外とアプリケーション エラーが表示されます。

### セッションリプレイのサンプリングに関する考慮事項 [#sampling-considerations]

ユーザー インパクト カードは、セッション リプレイ設定に基づいて適応されます。

<CollapserGroup>
  <Collapser id="sr-enabled" title="標準サンプリングでセッションリプレイが有効になりました">
    デフォルト値以上のサンプリング レート (一般サンプリングの場合は 10%、エラー サンプリングの場合は 100%) でセッション リプレイが有効になっている場合、両方のカードにセッション リプレイ カウントとリプレイ セッションへの直接リンクを含む完全なメトリクスが表示されます。
  </Collapser>

  <Collapser id="sr-low-sampling" title="低サンプリングでセッションリプレイを有効にする">
    サンプリング レートが推奨 を下回ると、サンプリング レートを上げることを提案する通知がカードに表示されます。 「サンプル レートを上げる」リンクをクリックして、ブラウザ アプリケーションの設定に移動します。
  </Collapser>

  <Collapser id="sr-disabled" title="セッションリプレイが無効">
    セッションリプレイが無効になっている場合、カードにはリプレイ データなしでセッション数のみが表示されます。機能を有効にするための<DNT>See where your users are struggling</DNT>と<DNT>Turn on session replays</DNT>リンクを含む通知が表示されます。
  </Collapser>
</CollapserGroup>

## パフォーマンスの詳細を調べる [#view\_details][#view_details]

ページビューの概要から、URLをクリックして、ページの読み込み、 [APM](/docs/apm/apm-ui-pages/features/request-queuing-tracking-front-end-time/)または[分散トレース](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui)のフロントエンドとバックエンドのタイミング、スループット、 [ブラウザーの種類](/docs/browser/new-relic-browser/additional-standard-features/browsers-problem-patterns-type-or-platform)、および[セッショントレース](/docs/browser/browser-monitoring/browser-pro-features/session-traces-explore-webpages-life-cycle)に関するパフォーマンスの詳細を表示します。

<img title="Page view details" alt="A screenshot of the page view details UI in New Relic" src="/images/browser_screenshot-full_page-load-performance.webp" />

ここから、次のこともできます。

* ページの読み込みとスループットのパフォーマンスを昨日と先週と比較します。
* Webトランザクションのアプリ応答時間の応答時間を表示します。
