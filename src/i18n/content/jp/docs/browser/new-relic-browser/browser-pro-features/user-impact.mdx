<InlinePopover type="browser" />の<DNT>**User impact**</DNT>ページは、技術的なパフォーマンスの問題がユーザーの実際のエクスペリエンスにどのように影響するかを理解するのに役立ちます。このページは、定量的なパフォーマンス メトリクスと定性的なユーザー行動分析を橋渡しし、ユーザーのフラストレーション パターンと技術的問題との相関関係をインサイトに提供します。

ユーザー影響ページは、主に 2 つの分析領域で構成されています。

* <DNT>Frustration metrics</DNT>
* <DNT>Performance risk</DNT>

それぞれが、ユーザー体験の問題とその技術的な根本原因を理解するための異なるレンズを提供します。

**前提条件:**

* フラストレーション信号とパフォーマンス リスク データを取得するには、 Browserバージョン 1.269 以降が必要です。
* このページからリンクされているセッションリプレイにアクセスするには、[セッションリプレイを](/docs/browser/browser-monitoring/browser-pro-features/session-replay/get-started/)有効にする必要があります。

## アクセスユーザー影響分析 [#access]

ユーザー影響ページを表示するには:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app)** &gt; **Users** &gt; **User impact**</DNT>に移動します。
2. タブから<DNT>**Frustration metrics**</DNT>または<DNT>**Performance risk**</DNT>のいずれかを選択します。

<Callout variant="tip">
  <DNT>**Analyze frustration signals**</DNT>または<DNT>**Analyze performance risk**</DNT>リンクをクリックして、 [ページ ビュー](/docs/browser/new-relic-browser/browser-pro-features/page-views-examine-page-performance)ページから直接ここに移動することもできます。
</Callout>

## イライラを分析するメトリクス [#frustration-metrics]

Frustration「フラストレーション指標」タブは、ユーザーの混乱、不満、またはタスクの完了における困難さを示すユーザー行動指標に焦点を当てています。この分析は、ユーザーが最も苦労している箇所を特定し、実際のユーザーの問題点に基づいて修正の優先順位を付けるのに役立ちます。

Frustration「フラストレーション指標」タブには、以下のセクションが含まれています。

* レイジ クリック: ユーザーが応答しない、または壊れた UI コンポーネントに対する不満を示し、急速かつ繰り返しクリックする動作を示すページと要素を識別します。
* 最も影響を受けたページ: 最も多くのフラストレーションのシグナルを生成している特定のページを強調表示し、影響の大きい領域に集中できるようにします。
* 最も影響を受けるページ要素: 特定のUI要素がユーザーの最もフラストレーションを引き起こしている詳細なインサイトを提供します。

## パフォーマンスメトリクスを分析する [#performance-impact]

「パフォーマンス リスク」タブでは、測定可能なパフォーマンス メトリクスとユーザー体験の問題との相関関係に焦点を当て、技術的なパフォーマンスの問題がユーザー セッションにどのような影響を与えるかを調査します。

パフォーマンス リスク タブには次のセクションが含まれます。

* Web Vitals: 貧弱なコアウェブバイタル (LCP、INP、CLS) がユーザーのセッションとエクスペリエンスにどのような影響を与えるかを分析します。
* エラー分析: JavaScript エラー パターンとそれがユーザー セッションに与える影響を調査します。
* 最も影響を受けたページ: パフォーマンス関連のユーザーへの影響が最も大きいページを特定します。
* 地理分析: さまざまな地理的な場所にわたるパフォーマンス リスクを調査して、地域の問題を特定します。
