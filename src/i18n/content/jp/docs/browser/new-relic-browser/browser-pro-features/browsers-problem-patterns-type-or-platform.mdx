New Relic の**ブラウザー**UI は、Google Chrome、Mozilla Firefox、Microsoft Internet Explorer、Apple Safari、Microsoft Edge など、エンドユーザーが使用するブラウザーに基づいて、アプリでのエンド ユーザーのエクスペリエンスに関する情報を提供します。このページには次のものが含まれます。

* スループット (1 分あたりのページ ビューまたは`ppm` ) とページの読み込み時間で上位のブラウザ
* 最大コンテンツ ペイント (LCP)、最初の入力遅延 (FID)、累積レイアウト シフト スコア (CSL) などを含むコア Web Vitals データ
* エラー別の上位の JavaScript エラー、および[JS エラー UI](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors/)の詳細へのリンク
* デバイスの種類 (モバイル、タブレット、デスクトップ) でフィルタリング可能なデータ

ドリルダウン チャートでは、選択したブラウザ タイプをバージョン別にセグメント化することもできます。たとえば、Chrome 14、47、72 などです。これにより、ページの読み込みタイミングの問題が特定のブラウザ タイプまたはデバイスに関連しているかどうか、または問題がより広範囲に広がっているかどうかをすばやく判断できます。

ブラウザー情報を表示するには: **[one.newrelic.com](https://one.newrelic.com) > Browser > (アプリを選択) > Browsers**に移動します。

ここでは、次のような質問に答えるのに役立つデータを確認できます。

* [ページ読み込みタイミング プロセス](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process)のスパイクはどこにありますか?
* 時間の経過とともに繰り返されるパターンはありますか?
* 平均ページ読み込み時間とスループットは許容範囲内ですか?
* 問題は特定のブラウザ バージョンに関連していますか?
* 上位の[JavaScript エラー](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors/)は、コードの問題、潜在的なセキュリティの脆弱性、またはその他の何かを示していますか?
* この種類のブラウザーの場合、これらのパフォーマンスの問題は、最初のペイント、最初のコンテンツ ペイント、最大のコンテンツ ペイント、ウィンドウの読み込み、および最初のインタラクションの Core Web Vitals スコアにどのように影響しますか?
* これらの問題は、他の種類のデバイス (デスクトップ、モバイル、タブレット) または他のブラウザーの種類と比べてどうですか?

このデータを使用して、問題を解決する際に他のユーザーと協力してください。たとえば、このデータを画像またはグラフの URL として共有するか、ダッシュボードに追加します。今後の問題を防ぐために、アラートを作成できます。
