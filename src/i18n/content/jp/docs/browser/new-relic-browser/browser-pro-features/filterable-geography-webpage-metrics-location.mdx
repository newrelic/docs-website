New Relicブラウザモニタリングの場合、フィルタリング可能な地理UIページは、世界中の都市、地域、国でのフロントエンドエクスペリエンスに関する色分けされたパフォーマンス情報を含む世界観を提供します。

マップには地域ごとのユーザーデータが表示され、デバイスタイプ情報とともにトラフィックとエラーのホットスポットを視覚化できます。このレベルの視覚的な詳細は、ITおよび運用の専門家がピアリング契約およびCDNの使用についてビジネス上の決定を行う場合に特に役立ちます。

利用可能なデータを最大限に活用するためのヒントについては、[フィルターの例](#filter)を参照してください。

## 所在地別のパフォーマンスデータを表示 [#apdex-by-region]

フィルタリング可能な地理機能には、 [ProまたはPro+SPAエージェントタイプ](/docs/browser/browser-monitoring/installation/install-browser-monitoring-agent/#agent-types)が必要です。

<img
  title="Filterable geography UI"
  alt="Screenshot of Filterable geography UI in New Relic"
  src={screenBrowserFilterGeography}
/>

パフォーマンス情報をロケーション別に表示したり、並べ替えることができます。

1. **[one.newrelic.com](https://one.newrelic.com) >ブラウザ>（アプリを選択）>フィルタリング可能な地域**に移動します。

2. ドロップダウンを使用して、ページの読み込みや一意のセッション数、さまざまなプロセスの平均時間、最大のコンテンツペイント、最初の入力遅延、累積レイアウトシフトなど、表示するパフォーマンスまたは使用状況データのタイプを選択します。

3. [特定のエリアにドリルダウンするには](#functions) 、地理的なマップ上の任意のエリアにマウスオーバーするか選択するか、またはマップの右にあるリストからワースト10のロケーションを選択してください。

4. 特定のパフォーマンスデータを表示するには、目標到達プロセスの下にある[属性](#filter)のいずれかを選択します

   <Icon name="fe-filter"/>

   アイコン。

5. ページの読み込みパフォーマンスと履歴パフォーマンスに関する比較データを表示するには、\[**マップビュー**]タブのマップの下にあるグラフを確認します。

6. データの追加のグラフを表示するには、\[**キー属性グラフ**]タブをクリックします。

\[**キー属性チャート**]タブのチャートは、選択した場所のパフォーマンスの比較の詳細で自動的に更新されます。これらのグラフは、特定の場所、ブラウザー、デバイス、URL、またはブラウザーのバージョンに関連するパフォーマンスの問題に関する洞察を提供します。

## 地図機能の利用 [#functions]

フィルタして詳細情報にドリルダウンしたり、地図上の任意の場所をズームインおよびズームアウトしたりできます。強化された属性は、ページの読み込みタイミングに関するエンドユーザーのエクスペリエンスだけでなく、世界中のネットワークパフォーマンスに関する深い可視性を提供します。たとえば、どの地域で通常よりもページの読み込み時間が長いかが不明な場合は、フィルタリング可能な地理UIを使用すると、問題の原因を突き止めることができます。

## パフォーマンスデータのフィルタリング [#filter]

さまざまなカテゴリの属性でデータをフィルタリングおよびグループ化できます。利用可能な属性のリストを表示するには、目標到達プロセスをクリックします<Icon name="fe-filter"/>アイコン。

フィルタリングされた結果は地理的なマップ上に表示され、そこから [の標準的なマップ機能](#functions) を使ってさらに詳細を調べることができます。以下に例を示します。

## 収集データに影響を与える要因 [#collected-data]

ページの読み込みタイミング情報を取得するために、ブラウザ監視エージェントは、ほとんどの（すべてではない）ブラウザがサポートする[NavigationTimingAPI](/docs/browser/new-relic-browser/browser-agent-apis/navigation-timing-api)を使用します。ブラウザがNavigationTimingAPIをサポートしていない場合、UIチャートとデータポイントは、DNSルックアップ、接続、安全なハンドシェイクなどのために受信したネットワークデータのみを表示できます。これはアプリの全体的なトラフィックの100％ではない場合があります。

また、ファイアウォールは、エンドユーザーについて収集した地理的データに影響を与える可能性があります。
