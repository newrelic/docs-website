ブラウザ [セットアップ要件](/docs/browser/new-relic-browser/welcome-new-relic-browser/compatibility-requirements) の一環として、 [ブラウザエージェント](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) をインストールする必要があります。その後、アプリの **Applications settings** ページを使用して、以下のブラウザ監視オプションを更新することができます。

* ブラウザのサブスクリプションレベル
* ブラウザ [Apdex T値](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction)
* [**でハイライトされる都市、地域、または国 Filterable geography** page](/docs/browser/new-relic-browser/browser-pro-features/filterable-geography-webpage-metrics-location)

## ApdexとGeographyの設定を選択 [#apdex-settings]

APMでも監視されているアプリの場合、 [Apdex Tの値をブラウザ（エンドユーザー）用](/docs/apm/new-relic-apm/apdex/apdex-measuring-user-satisfaction) と [APM用](/docs/apm/new-relic-apm/apdex/changing-your-apdex-settings) に分けて設定することができます。また、 **Filterable geography** ページでハイライト表示する都市、地域、国を選択することもできます。

1. **[one.newrelic.com](http://one.newrelic.com) > ブラウザ> （アプリを選択）> 設定> アプリの設定**...にアクセスします。
2. **Application settings** から、ブラウザ監視専用のApdex T値を選択するか、デフォルトのApdex値のままにします。
3. **Filterable geography** ページでハイライトするブラウザモニタリングの国や地域を最大5つ選択します。 [複数のアプリ名](/docs/agents/manage-apm-agents/app-naming/use-multiple-names-app) にデータを報告している場合、プライマリアプリとセカンダリアプリの両方で同じ国を選択します。
4. **Save application settings**を選択します。

Apdexの設定を変更すると、ブラウザアプリの **Page load timing** チャートに黒い縦棒が表示されます。 **[Summary](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance)** ページ。

## エンドユーザーのApdexを見る [#view-browser-apdexs]

自分のApdexスコアを見るには

* Go to **[one.newrelic.com](http://one.newrelic.com) > ブラウザ> （アプリを選択）> [まとめ](/docs/browser/new-relic-browser/getting-started/browser-overview-website-performance-glance)** .

  または

* Go to **[one.newrelic.com](http://one.newrelic.com) > ブラウザ> （アプリを選択）> [ページビュー](/docs/browser/new-relic-browser/additional-standard-features/page-views-understanding-your-sites-popularity)** .
