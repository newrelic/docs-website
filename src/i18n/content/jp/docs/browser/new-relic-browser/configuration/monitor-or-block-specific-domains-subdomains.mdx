[ブラウザ](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) では、ブラウザアプリが監視するドメインやサブドメインを選択することができます。これは、重要と思われるアプリケーションの活動のみに監視を限定するのに便利です。例えば、 [コピー／ペースト法](/docs/browser/new-relic-browser/installation-configuration/add-apps-new-relic-browser) を使用してブラウザアプリを展開する場合、ドメイン条件を設定して、 [ブラウザ監視のライセンスキーとアプリID](/docs/browser/new-relic-browser/installation-configuration/copy-browser-monitoring-license-key-app-id) のセキュリティを保護することができます。

<Callout variant="tip">
  ドメイン条件は、報告されたページの URL に基づいてデータのみをフィルタリングします。ドメイン条件フィルターは、New Relic にレポートされる AJAX データには適用されません。`AjaxRequest`イベントを除外するには[、構成ドキュメント](/docs/browser/new-relic-browser/configuration/filter-ajax-request-events/)をお読みください。
</Callout>

## ドメイン条件の設定 [#configure]

ドメイン条件の設定を確認・編集するには

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (アプリを選択) > Settings > Application settings > Domain conditions**に移動します。

2. ドメイン条件が設定されていない場合は、 **Enable domain conditions** を選択します。条件がある場合は、 **Edit conditions** を選択します。

3. 許可されたドメインのリストを作成するか、ブロックされた/拒否されたドメインのリストを作成するかを選択します。

4. ドメイン条件を選択します（最大10条件）。

5. **Starts with** の条件でドメインの条件を選択し、サブドメインのあるURLを入力します（最大10条件）。

6. ドメイン条件の設定を表示して確認します。

プレビューテキストボックスには、設定した条件に合致するドメインがリアルタイムで表示されます。また、提案された設定がデータに与える影響の見積もりも表示されます。

## ドメイン条件をオフにする [#disable]

ドメイン条件を無効にして、すべてのドメインの監視に戻すには

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (アプリを選択) > Settings > Domain conditions**に移動します。
2. **Enable domain conditions** オプションを **Off** に設定します。
