<Callout variant="important">
  統一されたエクスペリエンスを提供するために、Browser、Mobile、Monitor (Synthetics) の違反を廃止し、New Relic One と同等のものに置き換えています。 `one.newrelic.com` にナビゲートすると、アラートの詳細が自動的に表示されます。詳細については、 [Applied Intelligence と New Relic One のアラートに関するドキュメント](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai) および [July 2020 Explorers Hub post](https://discuss.newrelic.com/t/new-relic-feature-end-of-life-announcements/105772) をご覧ください。
</Callout>

優れたアラートは、個人やチームにシステムの変更を通知するのに役立ちます。 [アラート](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) は、New Relic 製品全体で調整された単一のアラートツールを提供します。これにより、 [アラートポリシー](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-policy) と [条件](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition) を管理し、最も気になる [エンティティ](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) のメトリクス (ブラウザアプリや主要トランザクションなど) に焦点を当てることができます。

[](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/name-or-rename-alert-policy)1つまたは複数のアラートポリシー を設定し、 製品エンティティ（ターゲット） に対してアラート条件 を定義した後、 AI UI の Alerts でその情報を見ることができます。また、個々の製品のUIでアラートポリシーの情報を表示することもできます。 [](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions)[](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition)&

## ブラウザからのポリシーと条件の表示 [#policy]

特定のブラウザー・アプリやキー・トランザクションのアラート・ポリシーや条件情報を表示する。

1. ブラウザアプリです。 **[one.newrelic.com](https://one.newrelic.com/browser) にアクセスし、「Browser」> （ブラウザアプリを選択）> 「Settings」> 「Alert conditions」** をクリックします。

   または

   キー・トランザクション **[one.newrelic.com](https://one.newrelic.com/browser) and click on More> Key transactions> (select a key transaction)> Settings> Alert conditions**.

2. **Alert conditions** ページから、利用可能なツールを使用して、警告条件とそれに関連するポリシーの検索、ソート、表示、更新を行います。

## ブラウザからイベントやアクティビティを表示 [#activity]

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) などのアクティビティのサマリー情報を表示するには。

1. ブラウザアプリです。 **[one.newrelic.com](https://one.newrelic.com/browser) にアクセスし、「Browser」をクリックします。**

   または

   キートランザクションです。 **[one.newrelic.com](https://one.newrelic.com/browser) にアクセスし、「More」をクリック> Key transactions**.

2. インデックスから、企業の [色分けされた健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) にマウスオーバーするか、 **ブラウザのアクティビティ** リストからリンクを選択するか、または特定のブラウザアプリや主要な取引を選択すると、詳細が表示されます。

例えば、Criticalアラートの違反が発生した場合です。

* ブラウザのインデックスと選択されたブラウザアプリでは、健康状態の表示が赤になります。
* 各種チャートの背景色がピンクに変わります。
* ブラウザのインデックスにある **ブラウザのアクティビティ** のリストには、警告（黄色）と重大（赤色）の違反が該当するように表示されます。

アラートでの違反を詳しく知るには、チャートのピンク色の部分にマウスオーバーするか、選択してください。詳細および例については、 [自社製品からイベントを表示する](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products) を参照してください。

## アラート違反をブラウザで確認 [#violations]

警告条件に [しきい値](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) が設定されていて、警告（黄色）または重大（赤）の違反があった場合、製品エンティティの色分けされた [健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) が違反を示すように変わります。違反の内容は、ブラウザアプリのページまたはブラウザのキートランザクションのページから直接見ることができます。

1. ブラウザアプリで **[one.newrelic.com](https://one.newrelic.com/browser)** にアクセスし、 **ブラウザ> (select a browser app)** をクリックして、右パネルの違反内容を確認します。

   キートランザクションです。 **[one.newrelic.com](https://one.newrelic.com/browser)** にアクセスし、 **More> Key transactions> (select a key transaction)> Events> Violations**.

2. **Violations** ページから、利用可能なツールを使用して、選択したブラウザアプリまたはキートランザクションの違反を検索、ソート、または表示します。

3. オプションです。違反の原因となったアラートポリシー、条件、またはしきい値を表示または更新するには、対応するリンクを選択します。

<Callout variant="tip">
  また、New Relic One Alerts& AI UI から、 [特定のインシデント](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/explore-incident-history) または全製品の [インシデントおよびイベント](/docs/alerts/new-relic-alerts-beta/reviewing-events/review-events-across-products) について、警告およびクリティカルな違反を表示することができます。アラートは、 [アラート通知を送信します。](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts) クリティカル（赤）の違反のみが対象です。
</Callout>
