優れたアラートは、個人やチームにシステムの変更を通知するのに役立ちます。 [アラート](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) は、New Relic 製品全体で調整された単一のアラートツールを提供します。これにより、 [アラートポリシー](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-policy) と [条件](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition) を管理し、最も気になる [エンティティ](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity) のメトリクス (ブラウザアプリや主要トランザクションなど) に焦点を当てることができます。

[](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/name-or-rename-alert-policy)1つまたは複数のアラートポリシー を設定し、 製品エンティティ（ターゲット） に対してアラート条件 を定義した後、 AI UI の Alerts でその情報を見ることができます。また、個々の製品のUIでアラートポリシーの情報を表示することもできます。 [](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions)[](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition)&

## ブラウザからのポリシーと条件の表示 [#policy]

特定のブラウザー・アプリやキー・トランザクションのアラート・ポリシーや条件情報を表示する。

1. ブラウザアプリ: **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (アプリを選択) > Settings > Application settings**に移動します。

   また

   キートランザクション: **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Key transactions > (キートランザクションを選択) > Settings > Alert conditions**移動します。

2. **Alert conditions** ページから、利用可能なツールを使用して、警告条件とそれに関連するポリシーの検索、ソート、表示、更新を行います。

## ブラウザからイベントやアクティビティを表示 [#activity]

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) などのアクティビティのサマリー情報を表示するには。

1. ブラウザアプリ: **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser**に移動します。

   また

   キートランザクション: **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Key transactions**に移動します。

2. インデックスから、企業の [色分けされた健康状態](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) にマウスオーバーするか、 **ブラウザのアクティビティ** リストからリンクを選択するか、または特定のブラウザアプリや主要な取引を選択すると、詳細が表示されます。

たとえば、重大なアラート インシデントが発生した場合:

* ブラウザのインデックスと選択されたブラウザアプリでは、健康状態の表示が赤になります。
* 各種チャートの背景色がピンクに変わります。
* ブラウザ インデックスの **Browser activity** \[ブラウザ アクティビティ] リストには、該当する場合、警告 (黄色) および重大 (赤色) インシデントが表示されます。

アラート内のインシデントの詳細を確認するには、グラフ上のピンクの領域にマウスを置くか選択します。詳細と例については、 [「製品からのイベントの表示」](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products)を参照してください。

## ブラウザからアラート インシデントを表示する [#incidents]

アラート条件に警告 (黄色) または重大 (赤色) インシデントの [しきい値](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) が設定されている場合、製品エンティティの色分けされた [正常性ステータス](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) が変化してインシデントを示します。ブラウザ アプリまたはブラウザのキー トランザクションのページからインシデントを直接表示できます。

1. ブラウザ アプリ: **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (ブラウザ アプリを選択)**に移動し、右側のパネルでインシデントを確認します。

   キー トランザクション: **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Key transactions > (キー トランザクションを選択) > Issues & Activity** に移動し 、 **Incidents** \[インシデント] タブをクリックします。

2. **Incidents** \[インシデント] ページから、利用可能なツールを使用して、選択したブラウザ アプリまたはキー トランザクションのインシデントを検索、並べ替え、または表示します。

3. オプション: インシデントをトリガーしたアラート ポリシー、条件、またはしきい値を表示または更新するには、対応するリンクを選択します。

<Callout variant="tip">
  また、アラートと AI UI から、 [特定のインシデント](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/explore-incident-history) または [すべての製品にわたるインシデントとイベント](/docs/alerts/new-relic-alerts-beta/reviewing-events/review-events-across-products)の警告および重大なインシデントを表示することもできます。アラートは、重大 (赤) インシデントに対してのみ [アラート通知を](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts) 送信します。
</Callout>
