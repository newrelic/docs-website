優れた<InlinePopover type="alerts"/> 、システムの変更について個人またはチームに通知するのに役立ちます。 [Alerts は、](/docs/alerts/new-relic-alerts-beta/getting-started/new-relic-alerts) New Relic 製品全体にわたって単一の調整されたアラート ツールを提供します。 これにより 、最も関心のあるテーマ （ アプリやキー トランスポートなど）のメトリックに重点を置いた [アラート ポリシー](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-policy) と[ ](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-condition)[](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-entity)条件を 管理できます。browser

[1 つ以上のアラートポリシーを設定し](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/name-or-rename-alert-policy) 、 [製品テンプレート (ターゲット)](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/select-product-targets-alert-condition) [の アラート条件を定義する と、その情報をアラート](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions)UI で表示できるようになります。アラートポリシー情報は、個々の製品のUIでも表示できます。

## ブラウザからのポリシーと条件の表示 [#policy]

特定のブラウザー・アプリやキー・トランザクションのアラート・ポリシーや条件情報を表示する。

1. browserアプリ: <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Settings > Application settings**</DNT> に移動します。

   また

   キートランザクション: <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Key transactions > (select a key transaction) > Settings > Alert conditions**</DNT>に移動します。

2. <DNT>**Alert conditions**</DNT>ページから、利用可能なツールを使用して、アラート条件とそれに関連付けられたポリシーを検索、並べ替え、表示、または更新します。

## ブラウザからイベントやアクティビティを表示 [#activity]

[イベント](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#event) などのアクティビティのサマリー情報を表示するには。

1. browserアプリ: <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser**</DNT>

   また

   キートランザクション: <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Key transactions**</DNT>に移動します。

2. インデックスから、 エンタープライズの[色分けされた稼働ステータス](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status)の上にマウスを置き、<DNT>**Browser activity**</DNT> リストからリンクを選択するか、特定のbrowserアプリまたはキートランザクションを選択して、追加の詳細を表示します。

たとえば、重大なアラート インシデントが発生した場合:

* ブラウザのインデックスと選択されたブラウザアプリでは、健康状態の表示が赤になります。

* 各種チャートの背景色がピンクに変わります。

* browserインデックスの

  <DNT>
    **Browser activity**
  </DNT>

  リストには、該当する場合、警告 (黄色) と重大 (赤色) のインシデントが表示されます。

アラート内のインシデントの詳細を確認するには、グラフ上のピンクの領域にマウスを置くか選択します。詳細と例については、 [「製品からのイベントの表示」](/docs/alerts/new-relic-alerts-beta/reviewing-events/view-events-their-products)を参照してください。

## ブラウザからアラート インシデントを表示する [#incidents]

アラート条件に警告 (黄色) または重大 (赤色) インシデントの [しきい値](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#alert-threshold) が設定されている場合、製品エンティティの色分けされた [正常性ステータス](/docs/alerts/new-relic-alerts-beta/getting-started/alerts-glossary#health-status) が変化してインシデントを示します。ブラウザ アプリまたはブラウザのキー トランザクションのページからインシデントを直接表示できます。

1. browserアプリ: <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > (select a browser app)**</DNT> に移動し、右側のパネルでインシデントを確認します。

   キートランザクション: <DNT>**[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Key transactions > (select a key transaction) > Issues & Activity**</DNT>に移動し、 <DNT>**Incidents**</DNT>タブをクリックします。

2. <DNT>**Incidents**</DNT> ページから、利用可能なツールを使用して、選択したbrowserアプリまたはキートランザクションのインシデントを検索、並べ替え、または表示します。

3. オプション: インシデントをトリガーしたアラート ポリシー、条件、またはしきい値を表示または更新するには、対応するリンクを選択します。

<Callout variant="tip">
  アラート UI から、 [特定のインシデント](/docs/alerts/new-relic-alerts-beta/reviewing-alert-incidents/explore-incident-history) または [すべての製品にわたるインシデントとイベントの](/docs/alerts/new-relic-alerts-beta/reviewing-events/review-events-across-products)警告および重大なインシデントを表示することもできます。アラートは、重大 (赤) インシデントに対してのみ [アラート通知を](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts) 送信します。
</Callout>
