## 問題

ブラウザアプリケーションのセッショントレース [が表示されていません。](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle)

## [解決策](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle)

[](/docs/browser/new-relic-browser/browser-pro-features/session-traces-exploring-webpages-life-cycle)[アプリケーションがAPM](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation) でインスツルメンテーションされており、Proブラウザエージェント に関連するその他の [機能が動作している場合は、以下の手順に従ってください。](/docs/browser/new-relic-browser/browser-pro-features)

[](/docs/browser/new-relic-browser/browser-pro-features)

1. [](/docs/browser/new-relic-browser/browser-pro-features)
2. [](/docs/browser/new-relic-browser/browser-pro-features)[アプリケーションに複数の名前を使用していないか確認してください](/docs/apm/new-relic-apm/installation-configuration/using-multiple-names-app) 。セッション・トレースはプライマリ・アプリケーションにのみ書き込まれます。
3. 複数の名前を使用している場合は、APMのアプリケーション名を確認し、アプリが最初に表示されていることを確認してください。
4. アプリケーションが最初に表示されていない場合は、代わりに最初に表示されているアプリケーション名でセッショントレースを探します。

セッション・トレース・データが表示されないその他の理由としては、以下のようなものがあります。

* エンドユーザーは、 [Resource Timing APIをサポートするブラウザを使用していません](http://www.w3.org/TR/resource-timing/) 。
* エンドユーザーは、 `/resources` のエンドポイント `bam.nr-data.net` にデータを投稿することはできません。
