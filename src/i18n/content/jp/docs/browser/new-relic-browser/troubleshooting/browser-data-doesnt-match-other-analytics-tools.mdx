## 問題

<InlinePopover type="browser" />では、使用している他の分析ツールとは異なるデータが表示されます。 一般的な症状は次のとおりです。

* ブラウザの記録は、 [ページビュー](/docs/browser/new-relic-browser/additional-standard-features/page-views-insights-your-sites-popularity) が予想よりも少ない。
* ブラウザの [ユーザーの地理的分布](/docs/browser/new-relic-browser/additional-standard-features/geography-webpage-performance-location) は、想定されるユーザーの所在地とは異なります。
* ブラウザでは、 [アクティブセッション](/docs/browser/new-relic-browser/page-load-timing-resources/session-tracking) の数が、他の場所で見られるものとは異なります。

## 解決

以下のトラブルシューティングの手順に従って、ブラウザのモニタリングを行ってください。

1. アプリケーションの全ページにブラウザモニタリングが正しくインストールされているかどうかを確認してください。
2. JavaScript スニペットが見つからない場合や、ユーザーが New Relic にデータを送信できない場合は、 [トラブルシューティング ブラウザモニタリング インストール](/docs/browser/new-relic-browser/installation-configuration/troubleshooting-browser-monitoring-installation) の手順に従ってください。
3. シングルページ スタイルのアプリケーションを使用していて、ルートの変更をビューとして表示することを期待している場合は、初期ページの読み込みとルートの変更の両方を統合したビューを提供する[browser SPA 監視](/docs/browser/single-page-app-monitoring/get-started/welcome-single-page-app-monitoring)の使用を検討してください。

ブラウザ監視が正常に動作している場合、 New Relic他の分析ツールとは異なる方法で情報を収集して表示するため、異なる結果が表示されることがあります。 インスタンスの場合、 `JavascriptErrors`と`AjaxRequest`個別に監視されており、それらのデータはエージェントがいつそれらを観察するか、また何をキャプチャできるか、またはキャプチャするかを決定するかによって異なります。 `BrowserInteraction`については独自のカスタムインストゥルメンテーションを使用し、 `PageViewTiming`および`PageView`の Vitals と Metrix については、Chrome チームが所有する web-vitals ライブラリがそのデータのソースです。

データの収集方法の詳細については、以下をご覧ください。

* [ブラウザ監視のインストゥルメンテーション](/docs/browser/new-relic-browser/page-load-timing-resources/instrumentation-browser-monitoring)
* [SPAデータ収集の理解](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection)
