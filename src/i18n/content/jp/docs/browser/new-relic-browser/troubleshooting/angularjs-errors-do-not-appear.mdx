## 問題

AngularJS を使用しており、JavaScript のエラーを New Relic に報告したいが、アプリケーションのエラーが [**JavaScript のエラー** ページ](/docs/browser/new-relic-browser/browser-pro-features/js-errors-dashboard-examining-errors-over-time) には表示されない。

## 解決策

AngularJSアプリケーションには、すべてのエラーをキャッチするグローバルエラーハンドラーが含まれていることが多く、デフォルトでは、ブラウザモニタリングはキャッチされなかったエラーのみを報告します。

[noticeError](/docs/browser/new-relic-browser/browser-agent-spa-api/newrelicnoticeerror-browser-agent-api) API コールを使用して、エラーを New Relic に報告します。これにより、通常のエラー処理の一環としてエラーを処理するだけでなく、New Relic の UI で確認することができます。
