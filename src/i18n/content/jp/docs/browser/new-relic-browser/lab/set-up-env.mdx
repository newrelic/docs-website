<Callout
  variant="course"
  title="ラボ"
>
  この手順は、New Relic ブラウザーを使用して Web アプリをトラブルシューティングする方法を説明するラボの一部です。 [ラボの概要を](/docs/browser/new-relic-browser/lab/over-view/)まだ確認していない場合は、確認してください。
</Callout>

ラボを適切に実行する前に、React アプリケーションをスピンアップする必要があります。ここで、あなたは:

* React アプリケーションをスピンアップする
* シンプルなロード ジェネレーターを使用してアプリにトラフィックを送信する

<Steps>
  <Step>
    ターミナル ウィンドウを開き、ラボ リポジトリのクローンを作成します。

    ```bash
    git clone https://github.com/newrelic-experimental/relicstaurants.git
    ```
  </Step>

  <Step>
    アプリケーションのルート ディレクトリに移動し、lab ディレクトリに切り替えます。

    ```bash
    cd relicstaurants
    git switch browser-pro-lab-material
    ```
  </Step>

  <Step>
    次に、依存関係をインストールし、アプリケーションを実行します。

    ```bash
    npm install
    npm run build
    npm run newstart
    ```

    これにより、ブラウザで Relicstaurants アプリケーションが開きます。

    <img
      title="Relicstraunts homepage"
      alt="Relicstraunts homepage"
      src="/images/browser-lab-screenshot-homepage.webp"
    />

    配達先住所を入力し、レストランを検索して開始します。

    <img
      title="Nearby restaurant list"
      alt="Nearby restaurant list"
      src="/images/browser-lab-screenshot-restaurant-list.webp"
    />

    ここに、食べ物を注文できるレストランのリストが表示されます。
  </Step>

  <Step>
    レストランを選択します。

    <img
      title="Choose a restaurant"
      alt="Choose a restaurant"
      src="/images/browser-lab-screenshot-choose-restaurant.webp"
    />
  </Step>

  <Step>
    1 つまたは 2 つの商品を選択し、カートをクリックします。

    <img
      title="select food"
      alt="select food"
      src="/images/browser-lab-screenshot-select-food.webp"
    />
  </Step>

  <Step>
    <DNT>**PAY**</DNT>をクリックします。

    <img
      title="checkout"
      alt="checkout"
      src="/images/browser-lab-screenshot-checkout.webp"
    />
  </Step>

  <Step>
    次の偽造カード情報を入力し、 <DNT>**Finish payment**</DNT>をクリックして注文してください。

    <img
      title="place order with fake card"
      alt="place order with fake card"
      src="/images/browser-lab-screenshot-place-order.webp"
    />

    ご注文は正常に行われました。

    <img
      title="Purchase completed"
      alt="Purchase completed"
      src="/images/browser-lab-screenshot-thank-you.webp"
    />

    次に、シミュレーターを使用して、アプリケーションへのトラフィックを増やします。
  </Step>

  <Step>
    別のターミナル ウィンドウで、アプリケーションのルート ディレクトリに移動し、ロード ジェネレーターを実行します。

    ```bash
    # Navigate to the root directiory of your simulator
    cd relicstaurants/simulator
    # Switch to lab branch
    git switch browser-pro-lab-material
    [output]
    # Install the simulator's dependencies
    pip3 install -r requirements.txt
    [output]
    # Run the simulator
    python3 simulator.py
    [output] ====== WebDriver manager ======
    [output] Current google-chrome version is 99.0.4844
    [output] Get LATEST chromedriver version for 99.0.4844 google-chrome
    ```

    <Callout variant="important">
      この Load Generator は、コンピュータに Google Chrome がインストールされていることを前提としています。別のブラウザを使用している場合は、この手順をスキップして手動でトラフィックを生成するか、 [Google Chrome をインストールしてください](https://www.google.com/chrome/downloads/)。
    </Callout>
  </Step>
</Steps>

アプリケーションの実行方法がわかったので、次はインストルメント化を行います。アプリケーションとシミュレータを実行しているターミナル ウィンドウで、 `<CTRL-C>` を押してそれらをシャットダウンします。アプリをシャットダウンすると、コードを更新して監視ツールを導入できるようになります。
