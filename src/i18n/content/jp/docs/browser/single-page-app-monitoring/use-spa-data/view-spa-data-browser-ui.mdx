## SPA（Single-Page App）データ [#app-views]

SPA データを表示するには、 <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Page views**</DNT>に移動します。

最初のページロードとルート変更は [自動的にブラウザのインタラクション名で](/docs/browser/new-relic-browser/configuration/group-browser-metrics-urls#hierarchy) グループ化されます。これは、 [セグメントの許可リスト設定で調整できます](/docs/browser/new-relic-browser/installation-configuration/url-whitelists-grouping-browser-metrics) 。

[SPA API](/docs/browser/new-relic-browser/browser-agent-spa-api)でカスタムルート名を設定すると、カスタムルート名が表示されます。 [<DNT>**Group page by**</DNT>](/docs/browser/single-page-app-monitoring/use-spa-data/view-spa-data-new-relic-browser#group-facet)ドロップダウンを使用して、ページの読み込み方法とルートの変更をグループ化する方法を変更できます。

デフォルトでは、ページの読み込みとルートの変更のリストには、最も時間のかかるビューがリストの上部に表示されます。 <DNT>**Sort by**</DNT>を使用して、平均応答時間、応答時間の中央値、1 分あたりの最大値で並べ替えることもできます。

[グループ化された URL](/docs/browser/new-relic-browser/configuration/group-browser-metrics-urls#hierarchy)で特定のビューを検索するには、 <DNT>**Sort by**</DNT>ドロップダウンの下の検索バーに入力します。 たとえば、チェックアウト ページを表す URL を見つけるには、 `checkout`を検索します。

最初の<DNT>**Page view**</DNT>ページのグラフには次の内容が表示されます:

* 平均レスポンスタイムが最も遅かった5つのビュー
* スループットの高い5つのビュー

調べる時間の範囲を変更するには、ページの上部にある [タイム ピッカー](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker) を使用します。(過去 8 日より前の時間範囲を選択した場合、一部のフィルター処理およびグループ化機能は使用できません。)

## SPAビューのフィルタリング

<img
  title="spa_filter.png"
  alt="spa_filter.png"
  src="/images/browser_screenshot-full_spa-filters.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Page views > Filter**</DNT>: <DNT>**Filter**</DNT> を使用して、ルートの変更、初期ページの読み込み、場所やbrowser種類などのその他の属性をフィルターします。
</figcaption>

最初のページの読み込みのみ、またはルートの変更のみを表示するには、 <DNT>**Filter**</DNT>ドロップダウンを使用します。 たとえば、ルートの変更のみを表示するには、 <DNT>**Filter > Route change**</DNT>を選択します。

このフィルターを使用すると、アプリ名、 browserの地理的位置、 browser種類など、ページの読み込みやルートの変更に関する他の属性でフィルター処理することもできます。 たとえば、オレゴン州ポートランド市のブラウザで発生したページの読み込みとルートの変更のみを表示するには、 <DNT>**Filter > City > Portland**</DNT>を選択します。

## グループSPAの見解 [#group-facet]

<DNT>**Group page by**</DNT>ドロップダウンを使用して、ページ ビューのリストを任意の属性別にグループ化できます。 たとえば、 browser種類別に平均応答時間を比較する場合は、<DNT>**Group page by > userAgent**</DNT> を選択します。

フィルタリングとグループ化を組み合わせることで、非常に具体的なデータを素早く見つけることができます。例えば、特定のURLが異なるブラウザでどのように読み込まれているかを比較することができます。

1. <DNT>
     **Filter**
   </DNT>

   ドロップダウンから

   <DNT>
     **targetURL**
   </DNT>

   を選択し、調査する URL を選択します。

2. <DNT>
     **Group page by**
   </DNT>

   ドロップダウンから

   <DNT>
     **userAgent**
   </DNT>

   を選択します。

## SPAビュー詳細 [#details]

<img
  title="spa_details.png"
  alt="spa_details.png"
  src="/images/browser_screenshot-full_pageview-details.webp"
/>

<figcaption>
  <DNT>**[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > (select an app) > Page views > (select a view)**</DNT>: リストからビューを選択して、さまざまな詳細と内訳を表示します。
</figcaption>

詳細を表示するには、個々のページの読み込みまたはルートの変更を選択します。いずれかを選択すると、 [ブラウザの操作](/docs/browser/single-page-app-monitoring/understand-spa-data-structure#browser-interactions)に費やされた時間の内訳が提供され、 [タイム ピッカー](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-time-picker)で選択されたウィンドウに一致する時系列でそのデータが表示されます。

すべてのルート変更ビューは、理論的には最初のページロードにもなり得ます。(例えば、ルート変更のURLが他の人に送信され、その人がロードした場合、それはNew Relicにとって初期ページロードとみなされるようになります)。SPA のビュー詳細ページに、初期ページロードとルート変更の両方のチャートがあるのはこのためです。これにより、ビューの初期ページロード時のパフォーマンスと、ルート変更時のパフォーマンスを比較することができます。

グラフ表示オプションは 3 つあり、 <DNT>**Avg initial page load time**</DNT>グラフ タイトルの右側にあるアイコンで選択できます。 デフォルトの表示は[、色分けされた積み上げ面グラフ](/docs/apm/new-relic-apm/ui-functions/standard-dashboard-features)です。 [ヒストグラム表示](/docs/apm/applications-menu/features/histograms-viewing-data-distribution)や[パーセンタイル折れ線グラフ](/docs/apm/applications-menu/features/percentiles-comparing-ranked-data)に切り替えることもできます。

詳細ページには、最初のページの読み込みとルートの変更を組み合わせた<DNT>**Throughput**</DNT>チャートも表示されます。 グラフには、スループットが最も高い 5 つのページが表示され、グラフの下にリストされます。その他のすべてのページは<DNT>**Other**</DNT>に統合されます。

ここでは、ページロード時と経路変更時に表示される具体的なパフォーマンスデータについて説明します。

<CollapserGroup>
  <Collapser
    id="page-load"
    title="初期ページロードパフォーマンスの詳細"
  >
    最初のページ読み込みの場合、パフォーマンスの詳細には、平均バックエンド時間、フロントエンド時間、およびウィンドウ オンロード イベントが含まれます。

    * <DNT>
        **Backend time**
      </DNT>

      ネットワーク、Web アプリの時間、リクエストのキューイングが含まれます。

    * <DNT>
        **Frontend time**
      </DNT>

      DOM 処理、ページのレンダリング、およびすべての XHR を完了するまでの時間が含まれます。

    * ウィンドウの読み込みイベントが発生したときに、赤い水平線が表示されます。 これは、SPA 監視が有効になっていない状態で Browseragent によって測定された従来のページ読み込みタイミングに対応します。 SPA 監視では、

      <DNT>
        **frontend time**
      </DNT>

      が完了する前にウィンドウ ロード イベントが発生することがよくあります。 (SPA ページの読み込みタイミングが従来のページの読み込みタイミングとどのように異なるかの詳細については、 [「SPA データ収集について](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection#page-load-timing-diff)」を参照してください。)
  </Collapser>

  <Collapser
    id="route-change"
    title="ルート変更のパフォーマンスの詳細"
  >
    ルート変更の場合、パフォーマンスチャートには、JSの継続時間と待ち時間が表示されます。

    * <DNT>
        **JS Duration**
      </DNT>

      定義上同期されるインタラクション中のすべての JavaScript 実行時間の合計です。

    * 残り時間は

      <DNT>
        **Waiting time**
      </DNT>

      と呼ばれ、合計期間から JS 期間を減算して算出されます。
  </Collapser>
</CollapserGroup>
