New Relic <InlinePopover type="browser" />は、シングルページアプリケーション（SPA）監視を提供し、シングルページアプリによる実際のユーザーインタラクションをより深く可視化し、実用的な洞察を提供します。

SPA 機能は[ソフトナビゲーション](https://developer.chrome.com/docs/web-platform/soft-navigations-experiment#what_is_a_soft_navigation)を自動的にモニターし、SPA API を使用してインタラクションを変更できます。開発者とそのチームは、API を使用して次のことを行うことができます。

* SPA 上の URL パスを通じてユーザーの活動を追跡します。
* リアルユーザーが体験しているスループットやパフォーマンスを監視します。
* ページ読み込みに関連した問題をトラブルシューティング・解決します。
* [データのクエリ](/docs/using-new-relic/data/understand-data/query-new-relic-data)を行い、ビジネスの決定を支援します。
* 優れたアプリケーションを迅速に市場に投入します。

## SPAの監視を有効にする [#enable]

SPAモニタリングは、新しいBrowserエージェントのインストールに対してデフォルトで有効になっています。SPA対応バージョンのエージェントは、ディストリビューティッド（分散）トレーシングなど、他の強力なNew Relic機能へのアクセスを提供します。詳細については、「[Browserモニタリングの有効化](/docs/browser/new-relic-browser/installation/update-browser-agent/#agent-types)」をご覧ください。

## 互換性 [#compatibility]

SPA関連の機能の互換性の情報については、「[SPAの要件](/docs/browser/new-relic-browser/getting-started/compatibility-requirements-browser-monitoring#spa)」をご覧ください。

## スループットとパフォーマンスのデータを分析する [#spa-data]

`BrowserInteraction` イベントが発生すると、SPA が有効な場合は常に UI ページビュータブを駆動します。最初のページの読み込み（ハードナビゲーション）とルートの変更（ソフトナビゲーション）の両方が表示されます。

<img title="browser_SPA.png" alt="browser_SPA.png" src="/images/browser_screenshot-full_spa-pageview.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Browser &gt; (select an app) &gt; Page views:**</DNT> ブラウザを監視するSPA監視を使用して、SPAアーキテクチャアプリケーションのスループットやパフォーマンスを調べます。
</figcaption>

Browserモニタリングでは、次のSPAデータをモニタリングできます。

* ページ読み込みとルート変更時のパフォーマンスデータとスループット
* AJAXリクエストのデータ
* [SPA API](/docs/browser/new-relic-browser/browser-agent-spa-api)を使用してモニターされる動的なページの更新

このデータによって、ユーザーがどのようにアプリケーションのページ読み込みやルート変更を体験しているかについて理解を深めるとともに、ボトルネックの解消やエラーのトラブルシューティングが可能になります。New RelicによるSPAデータの処理方法については、[SPAデータの収集を理解する](/docs/browser/single-page-app-monitoring/use-spa-data/understand-spa-data-collection)をご覧ください。
