New Relic の機能の中には、ブラウザモニタリングのライセンスキーや New Relic のアプリケーション ID が必要なものがあります。New Relic のコレクターはこのキーを読み取って、受信データがどのアカウントのものかを判断します。

[コピー/貼り付け方法](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser#copy-paste-app)を使用してアプリをブラウザー監視に追加すると、アプリの[ブラウザー**設定**ページ](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring)に、生成された JavaScript スニペットが表示されます。生成された JavaScript の下部には、ブラウザの`licenseKey` ( `browser_monitoring_key`とも呼ばれます) と`applicationID`が含まれます。

* ブラウザ監視`licenseKey` ( `browser_monitoring_key` ) は、リンクされている一意のキーですが、メインとは異なります

  <InlinePopover type="licenseKey"/>

  . ブラウザー キーを使用して、アカウントのライセンス キーを決定することはできません。

* ブラウザ`applicationID`は、ブラウザ監視における各アプリの一意の識別コードです。

`browser_monitoring_key`と`applicationID`のセキュリティを保護するために、 [ドメインを監視またはブロック](/docs/browser/new-relic-browser/installation-configuration/monitor-or-block-specific-domains)できます。

## ブラウザのライセンスキーとアプリのIDをUIから取得する [#procedures]

この手順は、コピー/貼り付けの展開方法を使用する場合に**のみ**適用されます。 [APMアプリのデプロイ方法](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser)を使用する場合、 **\[設定]**ページにはJavaScriptスニペットは表示されません。

  **[one.newrelic.com](https://one.newrelic.com) > Add data > Browser monitoring > (コピー/ペーストによるインストール方法を選択) > (各インストール手順を実行)** : インストルメンテーションのセットアップの最後に、ライセンス キーとアプリケーション ID が JavaScript スニペットに表示されます。 .
</figcaption>

ブラウザモニタリングのライセンスキーとアプリケーションIDの情報のみを取得する場合。

1. **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Browser > Settings**に移動します。

2. 生成されたJavaScriptを選択してコピーし、テキストファイルに貼り付けてください。

3. ブラウザの監視を除くテキスト ファイル内のすべてを削除します。

   <InlinePopover type="licenseKey"/>

   およびアプリ ID 情報 (終了`</script>`タグのすぐ上)、保存します。
