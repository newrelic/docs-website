New Relic の機能の中には、ブラウザモニタリングのライセンスキーや New Relic のアプリケーション ID が必要なものがあります。New Relic のコレクターはこのキーを読み取って、受信データがどのアカウントのものかを判断します。

[コピー/貼り付け方式を](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser#copy-paste-app)使用してアプリを<InlinePopover type="browser"/>に追加すると、アプリの[ブラウザ<DNT>**Settings**</DNT>ページに](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring)生成された JavaScript スニペットが表示されます。 生成された JavaScript の下部には、 browser `licenseKey` (`browser_monitoring_key` とも呼ばれます) と `applicationID` が含まれます。

* ブラウザ監視`licenseKey` ( `browser_monitoring_key` ) は、メインの

  <InlinePopover type="licenseKey"/>

  にリンクされているが、同じではない一意のキーです。 browserキーを使用してアカウントのライセンス キーを特定することはできません。

* ブラウザ`applicationID`は、ブラウザ監視における各アプリの一意の識別コードです。

`browser_monitoring_key`と`applicationID`のセキュリティを保護するために、 [ドメインを監視またはブロック](/docs/browser/new-relic-browser/installation-configuration/monitor-or-block-specific-domains)できます。

## ブラウザのライセンスキーとアプリのIDをUIから取得する [#procedures]

この手順は、コピー/貼り付けによるデプロイメント方法を使用する場合に<DNT>**only**</DNT>に適用されます。 [APM アプリのデプロイメント方法を](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser)使用する場合、 <DNT>**Settings**</DNT>ページに JavaScript スニペットは表示されません。

<figcaption>
  <DNT>**[one.newrelic.com](https://one.newrelic.com) > Integrations & Agents > Browser monitoring > (choose the copy/paste install method) > (go through each installation step)**</DNT>: インストゥルメンテーションのセットアップの最後に、ライセンスキーとアプリケーション ID が JavaScript スニペットに表示されます。
</figcaption>

ブラウザモニタリングのライセンスキーとアプリケーションIDの情報のみを取得する場合。

1. <DNT>
     **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Browser > Settings**
   </DNT>

   に移動します。

2. 生成されたJavaScriptを選択してコピーし、テキストファイルに貼り付けてください。

3. ブラウザ監視

   <InlinePopover type="licenseKey"/>

   とアプリ ID 情報 (終了`</script>`タグのすぐ上) を除くテキスト ファイル内のすべてを削除して保存します。

## APIによるキーの作成と管理 [#create-manage]

[追加のライセンスキーを作成し、NerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-personal-api-keys) を使って管理することができます。
