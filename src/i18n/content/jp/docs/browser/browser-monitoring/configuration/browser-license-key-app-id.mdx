New Relic の機能の中には、ブラウザモニタリングのライセンスキーや New Relic のアプリケーション ID が必要なものがあります。New Relic のコレクターはこのキーを読み取って、受信データがどのアカウントのものかを判断します。

[コピー/ペースト方法](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser#copy-paste-app) を使用してアプリをブラウザモニタリングに追加すると、アプリの [ブラウザ **設定** ページ](/docs/browser/new-relic-browser/installation-configuration/browser-settings-ui-options-browser-monitoring) に、生成された JavaScript スニペットが表示されます。生成されたJavaScriptの下部には、ブラウザの `licenseKey` （別名： `browser_monitoring_key` ）と `applicationID` が含まれています。

* ブラウザモニタリング `ライセンスキー` (`browser_monitoring_key`) は、メインの [アカウントのライセンスキー](/docs/accounts/accounts-billing/account-setup/new-relic-license-key/) とリンクしていますが、同じものではない、ユニークなキーです。ブラウザのキーを使ってアカウントのライセンスキーを判断することはできません。
* ブラウザ `applicationID` は、ブラウザ監視における各アプリの固有の識別コードです。

`browser_monitoring_key` と `applicationID` のセキュリティを保護するために、 [ドメインを監視またはブロックすることができます](/docs/browser/new-relic-browser/installation-configuration/monitor-or-block-specific-domains) 。

## ブラウザのライセンスキーとアプリのIDをUIから取得する [#procedures]

この手順は、コピー/貼り付けの展開方法を使用する場合に**のみ**適用されます。 [APMアプリのデプロイ方法](/docs/browser/new-relic-browser/installation-configuration/adding-apps-new-relic-browser)を使用する場合、 **\[設定]**ページにはJavaScriptスニペットは表示されません。

  **[one.newrelic.com](https://one.newrelic.com) > ブラウザ> 設定:** コピー/ペースト展開のために生成されたJavaScriptの最後には、ブラウザ監視のライセンスキーとアプリケーションIDが含まれています。
</figcaption>

ブラウザモニタリングのライセンスキーとアプリケーションIDの情報のみを取得する場合。

1. **[one.newrelic.com](https://one.newrelic.com) > ブラウザ> 設定** にアクセスしてください。
2. 生成されたJavaScriptを選択してコピーし、テキストファイルに貼り付けてください。
3. テキストファイルの中で、ブラウザモニタリングのライセンスキーとアプリIDの情報以外のものを削除します（終了時の `< /script>` タグのすぐ上）。保存します。

## APIによるキーの作成と管理 [#create-manage]

[追加のライセンスキーを作成し、NerdGraph](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-personal-api-keys) を使って管理することができます。
