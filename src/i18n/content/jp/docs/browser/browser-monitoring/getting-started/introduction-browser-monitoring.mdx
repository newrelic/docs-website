    New Relicのブラウザモニタリングは、リアルユーザーモニタリング（RUM）ソリューションを提供します。さまざまなウェブブラウザ、デバイス、オペレーティングシステム、ネットワークからサイトに移動する際の、エンドユーザーの速度とパフォーマンスを測定します。ただし、ブラウザモニタリングは、初回ページの読み込みについて、情報提供をはるかに超えるものです。これを使用して、全ページのライフサイクルデータを測定し、顧客満足度の確保に必要な情報の取得を開始します。

    New Relicでは、ブラウザのアクティビティからデータをモニタリングし、スタック全体のパフォーマンスを最適化できます。ブラウザモニタリングを使用してデプロイメントを確実に成功させ、顧客に見える問題の迅速なトラブルシューティングを行います。スタックを一目でモニタリングし、すべてのエンティティが設計どおりに動作していることを確認します。アプリケーションの速度とパフォーマンス、JavaScriptエラー、AJAXリクエストなどを可視化します。イシューの追跡にかかる時間を短縮し、顧客に完璧なデジタル体験を提供する時間を増やします。

    ## ウェブバイタルの追跡

    Googleのコアウェブバイタルは、ブラウザアプリでのユーザー体験を測定するための3つのメトリクスで、New Relicでそれらを追跡できます。New Relicでは、次の2つの場所でコアウェブバイタルを表示します。

    * [Googleのコアウェブバイタルをモニタリングしてアプリのパフォーマンスを確保](/docs/browser/new-relic-browser/guides/guide-to-core-web-vitals#monitor)
    * [当社の追加的なウェブバイタルを使用して、アプリのパフォーマンスの他の側面を検証](/docs/browser/browser-monitoring/getting-started/browser-summary-page/)

    ## エンドユーザーのイシューを特定

    メトリクス、イベント、ログ、トレースなど、必要なあらゆるタイプのデータをインストゥルメントし、エンドユーザーのページ読み込みエクスペリエンスを最適化してユーザーを継続的に満足させる上で役立ちます。

    * [実際のユーザーブラウザのインタラクションとSyntheticモニタリングの傾向を比較](/docs/synthetics/synthetic-monitoring/administration/compare-page-load-performance-browser-synthetic-monitoring/)
    * [特定の地理的な場所またはエンドユーザーの特定の種類のアクティビティに重点を置く](/docs/browser/new-relic-browser/browser-pro-features/filterable-geography-webpage-metrics-location/)
    * [問題が特定の種類のブラウザ、ブラウザのバージョン、またはデバイスの種類に関連しているかどうかを判断](/docs/browser/new-relic-browser/browser-pro-features/browsers-problem-patterns-type-or-platform/)
