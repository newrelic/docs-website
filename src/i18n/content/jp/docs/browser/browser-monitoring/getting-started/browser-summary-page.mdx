ブラウザー監視の**概要**ページは、アプリの実際のユーザー ブラウザーのパフォーマンスに関する問題のトラブルシューティングに役立ちます。**\[概要**] ページを使用して、次のことを行います。

* 最大のコンテンツフルペイント（LCP）、最初の入力遅延（FID）、累積レイアウトシフト（CLS）など、コアWebバイタルを一目で確認できます。
* アプリのブラウザー側のパフォーマンスの傾向を表示します。
* ページの読み込みタイミングの問題をすばやくトラブルシューティングします。
* 他のブラウザUIページに直接移動して、問題や問題をより詳細に調べます。
* 推奨アラート条件を追加します。

## サマリーページの表示 [#viewing]

ブラウザー アプリのパフォーマンスの概要を表示するには:

1. **[one.newrelic.com](https://one.newrelic.com) > All Capabilities**に移動します。
2. **ブラウザを**クリック
3. ブラウザ アプリを選択します。

## コアウェブバイタルデータを使用してSEOを改善する [#core_vitals]

**Core Web Vitals**チャートを使用して、 [Google の Core Web Vitals](https://web.dev/vitals/#core-web-vitals)に従ってブラウザがどのように機能するかを理解します。これらには以下が含まれます：

* LCP：ローディングパフォーマンスを測定する最大の満足のいくペイント
* FID：最初の入力遅延。ユーザー入力とブラウザー応答の間の対話性を測定します。
* CLS：ブラウザページの視覚的な安定性を測定する累積レイアウトシフト

より高いスコアは、組織がお金を節約し、SEOに影響を与えるのに役立ちます。スコアが低いと、組織のWebサイトに対するユーザーの認識に影響を及ぼし、顧客の不満やビジネスチャンスの喪失につながる可能性があります。

コア Web バイタルの監視の詳細については、[コア Web バイタルのガイドを](/docs/browser/new-relic-browser/guides/guide-to-core-web-vitals)参照してください。

## ユーザーエクスペリエンスを評価する [#user_time]

コアWebバイタルデータを含むグラフに加えて、次のデータを使用して、アプリでのユーザーエクスペリエンスを評価します。

* **ユーザーがサイトを利用する時間**
* **ユーザー中心のページ読み込み時間**と**URLによる最長の最初の入力遅延**：[ページ読み込みタイミングプロセス](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process)の詳細を取得するには、グラフのタイトルをクリックして、 [**ページビュー**UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance)に直接移動します。

## ブラウザのパフォーマンスの問題を特定する [#performance]

これらのチャートを使用して、ブラウザーに関連するパフォーマンスの問題を特定します。

* **最初のページの読み込みとルートの変更**：このグラフは、URLの読み込みまたは再読み込みに起因する従来のURL変更の読み込み時間を示しています。詳細を取得するには、グラフのタイトルをクリックして、 [**ページビュー**UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance)に直接移動します。
* **スループット**：このグラフは、ブラウザーのスループットを1分あたりのページ数（ppm）で示しています。該当する場合、このグラフにはシングルページアプリ（SPA）データも表示されます。詳細を取得するには、グラフのタイトルをクリックして、 [**ページビュー**UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance)に直接移動します。
* **JavaScriptエラーのあるページビュー**：詳細を表示するには、グラフのタイトルをクリックして、 [**JSエラー**UI](/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-detect-analyze-errors)に直接移動します。

## 他の何かによって引き起こされた問題を調べます [#other-issues]

パフォーマンスの問題は、ブラウザに関連している場合もあれば、他の問題に関連している場合もあります。ブラウザのパフォーマンスに影響を与える可能性のあるその他の問題の詳細については、以下を確認してください。

* **フロントエンドとバックエンド**: このグラフは、接続されている

  <InlinePopover type="apm"/>

  サービス アプリケーション。 [分散トレース](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing/)の詳細を取得するには、グラフのタイトルをクリックしてください。

* **デバイスタイプ**（デスクトップ、タブレット、またはモバイル）**による最初のインタラクションとユーザーエージェントによる最初**のインタラクション：詳細を表示するには、グラフのタイトルをクリックして、 [**ページビュー**UI](/docs/browser/new-relic-browser/additional-standard-features/page-views-examine-page-performance)に直接移動します。

* **関連エンティティ**：このウィジェットは、ブラウザに接続している[エンティティ](/docs/new-relic-solutions/new-relic-one/core-concepts/what-entity-new-relic/)の問題がパフォーマンスの問題の原因であるかどうかを判断するのに役立ちます。

* **合成モニターの表示**： [合成モニター](/docs/synthetics/synthetic-monitoring/getting-started/get-started-synthetic-monitoring/)はユーザートラフィックをシミュレートできます。このリンクは、顧客が気付く前に、重要なエンドポイントに関連する停止やパフォーマンスの低下を事前に検出して解決するのに役立ちます。

## 推奨アラート条件を追加 [#recommended-alert-conditions]

ブラウザ監視の **Summary** \[概要] ページを通じて、推奨されるアラート条件をエンティティに追加できます。

さらに未検出のエンティティを確認したい場合は [、「アラート カバレッジ ギャップと条件の推奨事項」](/docs/alerts-applied-intelligence/applied-intelligence/alert-coverage-gaps/) ページを参照してください。

推奨されるアラート条件を追加するには、次の手順に従います。

1. ブラウザ監視の **Summary**\[概要] ページを開きます。

2. **Add recommended conditions**\[推奨条件を追加]をクリックします。

     width="80%;"
     style={{align: "left"}}
     title="Browser monitoring summary"
     alt="Screenshot of the New Relic browser monitoring Summary page"
     src={browserAlertConditions}
   />

3. 推奨されるアラート条件をいくつか選択します。

   <img
     width="70%;"
     style={{align: "left"}}
     title="Browser monitoring recommended alert conditions"
     alt="Browser monitoring recommended alert conditions"
     src={browserAddAlert}
   />

4. **Next**\[次へ]をクリックします。

5. 通知を受け取るポリシーを選択します。既存のものを選択するか、 [新しいものを作成でき](/docs/alerts-applied-intelligence/new-relic-alerts/alert-policies/create-edit-or-find-alert-policy/#alert-policy-name)ます。

6. **Activate conditions**\[条件を有効にする]をクリックします。
