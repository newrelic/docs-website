ブラウザモニタリングの [ページロードタイミング](/docs/browser/new-relic-browser/page-load-timing-resources/page-load-timing-process) 機能は、単純なセッション識別子を含む [クッキー](/docs/browser/new-relic-browser/page-load-timing-resources/new-relic-cookies-used-browser#jsessionid) を使用してセッションを追跡することができます。

## 使用方法 [#process]

ここでは、セッショントラッキングの基本的な流れをご紹介します。

1. セッションは、New Relic のブラウザモニタリングがインストールされ、クッキーの収集が有効になっているウェブサイトにユーザーがアクセスしたときに始まります。
2. ユーザーのブラウザが開かれると、当社のブラウザ監視スクリプトは、セッションを識別するシンプルなクッキーを設定します。
3. その後のページ読み込み時に、スクリプトがクッキーを読み取ります。
4. 同じセッションは、ユーザーのブラウザが閉じられるまで続きます。

このような場合、セッショントラッキングは正しく機能しません。

* ユーザーがブラウザでクッキーを無効にしている場合
* ブラウザが **not** 監視対象サイトからのリクエストで New Relic のクッキーに関連する情報の送信を許可するように設定されている場合

## クッキー収集の有効化または無効化 [#settings]

New Relic では、データのプライバシーを真剣に考えています。デフォルトでは、ブラウザエージェントによって収集された個人データは一切保持しませんし、セッショントラッキングをコントロールすることもできます。アプリでクッキー収集をオンまたはオフにするには、以下の方法があります。

1. **[one.newrelic.com](https://one.newrelic.com) > ブラウザ> （アプリを選択）> 設定> アプリの設定**...にアクセスします。
2. アプリケーションの **Privacy** 設定で、 **Cookie collection** を切り替えます。
