New Relic のパートナーを通じて New Relic を契約した場合、New Relic の使用を開始するためのログインに、追加のシングルサインオン (SSO) 手続きが必要になる場合があります。また、デプロイメントプロセスを完了するために特別な手順が必要な場合もあります。このドキュメントでは、New Relic パートナーを通じて作成されたアカウントの基本的なログイン手順を説明し、必要に応じて追加リソースへのリンクを掲載しています。

## New Relic アカウントの作成 (すべてのパートナー) [#partner-account]

New Relic のパートナーから特別なオファーを受け取った場合は、パートナーのリンクを直接たどって New Relic のアカウントを作成してください。

## New Relicへのサインインと導入（ほとんどのパートナー企業） [#partner-users]

お客様のアカウントを確認し、New Relicにサインインし、デプロイメントプロセスを完了するためのリンクが記載された確認メッセージがEメールで届きます。

## New Relicへのサインインと導入（スペシャルパートナー） [#partner-users]

これらのパートナーは、お客様がNew Relicアカウントを使用するために、特定のログインと導入手順を必要とします。

<Callout variant="tip">
  ここに掲載されていないパートナーを通じてNew Relicアカウントにサインアップした場合は、 [標準的な手順にしたがって、](#partner-users) New Relicにサインインして展開してください。
</Callout>

  <Collapser
    id="rackspace"
    title="ラックスペースのクラウドツール"
  >
    [Rackspace Cloud Tools](https://www.rackspace.com) からアカウントを作成し、New Relic にサインインするには、以下の基本的な手順に従います。詳細については、 [Rackspace Cloud Load Balancer plugin](/docs/accounts-partnerships/accounts/install-new-relic/partner-based-installation/rackspace-cloud-load-balancer-plugin) を参照してください。

    1. [cloudtools.rackspace.com/myapps](https://cloudtools.rackspace.com/myapps) で、CloudTools アカウントにログインします。

    2. <DNT>**Use Your Applications**</DNT>から<DNT>**New Relic**</DNT>を選択します。

    3. エージェントの言語（Java、.NET、PHP、Python、Ruby）を選択し、オンラインの指示に従ってインストール作業を完了させます。

    4. New Relic がアプリのデータ収集を開始するのを待ちます。

       数分待ってから、 [APM <DNT>**Overview**</DNT>ページ](/docs/apm/applications-menu/monitoring/applications-overview-dashboard)でアプリのパフォーマンスを確認してください。
  </Collapser>

  <Collapser
    id="wordpress-w3-edge"
    title="W3 EdgeとWordPress"
  >
    <Callout variant="tip">
      [W3 Total Cache](https://wordpress.org/plugins/w3-total-cache/) を通じてアカウントを作成し、New Relic にサインインするには、API キーを取得するだけでなく、W3 PHP エージェントをインストールして実行する必要があります。
    </Callout>

    エージェントをデプロイするには、次のものを含む<DNT>**administrative access**</DNT>が必要です:

    * SSH/コマンドラインアクセス

    * New Relic のインストーラーを実行し、ウェブサーバーを再起動するための root または sudo のアクセス権

      エコノミー、共用ホスティング、ブロガーパッケージを利用している場合や、アカウントの権限がわからない場合は、ホスティングプロバイダーにお問い合わせください。例えば、WordPress サイトが共有サーバー上にある場合、New Relic をインストールできない可能性があります。ホスティング事業者は、アカウントの権限に関する情報を提供したり、必要に応じてアカウントをアップグレードしたりすることができるはずです。

      W3 Edge/Wordpressをお使いの方がNew Relicプラグインをインストールする場合。

    1. [wordpress.org/support/bb-login.php](https://wordpress.org/support/bb-login.php) のWordPressアカウントにログインしてください。

    2. [New Relic の指示](/docs/agents/php-agent/installation/wordpress-users-new-relic) に従ってインストール作業を完了するか、 [W3TC のサポート](http://wordpress.org/support/plugin/w3-total-cache) にお問い合わせください。

    3. New Relic がデータ収集を開始するのを待ちます。

       エージェントをデプロイしてから数分後には、アプリに関するパフォーマンスデータがAPMの [Applications Overview p](/docs/apm/applications-menu/monitoring/applications-overview-dashboard)時代に表示されます。

       <Callout variant="tip">
         [WordPress](https://wordpress.org/plugins/rt-newrelic-browser/) は、私たちの [ブラウザ監視](/docs/agents/php-agent/installation/wordpress-users-new-relic#browser) のためのプラグインも提供しています。
       </Callout>
  </Collapser>
