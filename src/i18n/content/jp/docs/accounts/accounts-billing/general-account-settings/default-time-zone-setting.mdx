個人のタイムゾーン設定は、いくつかの例外を除いて、New Relic UI のほとんどの時間関連の設定を制御します。

## デフォルトのタイムゾーンの変更 [#exceptions]

New Relic アカウントのデフォルトのタイムゾーンを変更するには、以下の手順に従います。

1. <DNT>
     **[one.newrelic.com](https://one.newrelic.com/all-capabilities)**
   </DNT>

   に移動します。

2. [ユーザーメニュー](/docs/accounts/accounts-billing/general-account-settings/intro-account-settings)を選択し、

   <DNT>
     **User preferences**
   </DNT>

   を選択します。

3. タイムゾーンの変更

4. オプション： [自動ユーザー管理](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) で管理されているユーザーの場合は、IDプロバイダーのタイムゾーンを使用するか、それを上書きすることができます。

タイムゾーンを変更した場合、UIに反映されるまでに最大で24時間かかることがあります。

## あなたのタイムゾーンが適用されない例外 [#exceptions]

一部の New Relic 機能は、 <DNT>**User preferences**</DNT>タイムゾーン設定に依存しません。 以下は協定世界時 (UTC) を使用し、ユーザー設定の影響を受けません。

* [アラート](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts)
* [REST API v2](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2#utc)

他にも、タイムゾーンがデフォルトのタイムゾーン設定に依存しない機能があるかもしれません。
