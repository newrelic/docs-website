個人のタイムゾーン設定は、このドキュメントで説明されているいくつかの例外を除いて、New Relic UI のほとんどの時間関連設定を制御します。タイムゾーンの設定を変更した場合、UIに反映されるまでに最大で24時間かかることがあります。

## デフォルトのタイムゾーンの変更 [#exceptions]

New Relic アカウントのデフォルトのタイムゾーンを変更するには、以下の手順に従います。

1. **[one.newrelic.com](https://one.newrelic.com)**に移動します。
2. [account dropdown](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#account-dropdown) を選択し、 **User preferences** を選択します。

## 例外 [#exceptions]

[自動ユーザー管理](/docs/accounts/accounts/automated-user-management/automated-user-provisioning-single-sign) (AUM)で管理されているユーザーは、UIでタイムゾーンを変更できません。これは、IDプロバイダーで設定する必要があります。

New Relic の一部の機能は、 **User preferences** のタイムゾーン設定に依存しません。以下は協定世界時(UTC)を使用しており、ユーザー設定の影響を受けません。

* [アラート](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-alerts)
* [REST API v2](/docs/apis/rest-api-v2/requirements/specifying-time-range-v2#utc)

他にも、タイムゾーンがデフォルトのタイムゾーン設定に依存しない機能があるかもしれません。
