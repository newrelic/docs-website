2022年1月12日に、 [New Relic One pricing](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) のバージョンをリリースし、新しいユーザータイプである **コアユーザー** ([learn benefits](#benefits))を追加しました。

## この資料は誰のためのものですか？ [#what-is-this]

この資料は、新しい"New Relic One with core users" バージョンの価格設定にオプトインすることを検討している New Relic One のお客様向けの移行ガイドで、このリリースの利点と利用可能性について知りたいと考えています。

すでにこの価格バージョンをご利用のお客様は、 [請求可能なユーザーの計算](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing) と [ユーザータイプ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) のドキュメントをお探しだと思います。

## コアユーザーリリースの概要 [#overview]

2022年1月12日、当社は、 [New Relic Oneの価格設定モデル](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing) のバージョンをリリースしました。その際、3番目のユーザータイプ（コアユーザー [）を追加し、 ](#benefits)[課金計算ルールを変更しました](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing) 。

今回のリリースでは、New RelicはNew Relic Oneの価格モデルを2つのバージョンに分けました。

* 私たちの主要なNew Relic Oneの価格バージョン：これはコアユーザーがいるバージョンです。新規登録者およびその他の [対象組織に提供されます。](#availability) 。
* コアユーザーのいない"New Relic One" バージョン：このバージョンの組織は、ベーシックユーザーとフルプラットフォームユーザーのみを追加できます（コアユーザーは追加できません）。このバージョンは時間の経過とともにどんどん非推奨になっていきます。 [このバージョンのルール](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#non-core) を参照してください。

## 利点 [#benefits]

コアユーザーバージョンを持つ組織は、第3のユーザータイプであるコアユーザーにアクセスできます。

コアユーザーは、コードを書いて出荷するが、本番アプリケーションやシステムのアップタイムや信頼性の管理には責任を持たない開発者に適しています。

コアユーザーになると、 [ベーシックユーザーが持っているNew Relicの機能（](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) ）に加えて、コードに関連する問題の理解、データに基づいた開発上の意思決定、組織内のエンジニアとの共同作業に役立つ、開発者向けの厳選された機能にアクセスできます。コアユーザーは以下のことができます。

* [New Relic CodeStream](/docs/codestream/start-here/new-relic-codestream-quick-start) を使用して、パフォーマンスの問題を IDE のコードと関連付ける。
* [エラーインボックス機能](/docs/errors-inbox/errors-inbox) を使用して、スタック全体からエラーを積極的に表示、管理します。
* [当社のログ管理UI](/docs/logs/get-started/get-started-log-management) を使用して、アプリケーションとインフラストラクチャのイベントを深く可視化します。
* [カスタムNew Relic Oneアプリの構築と実行](/docs/new-relic-one/use-new-relic-one/build-new-relic-one/build-custom-new-relic-one-application) カタログに掲載されている [オープンソースアプリの使用](https://opensource.newrelic.com/nerdpacks).
* GraphQL-format [NerdGraph API](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph) へのアクセスを増やします。

## 入手方法 [#availability]

現在、コア・ユーザー・リリースは以下の通りです。

* 2022年1月12日以降に作成されたNew Relicの組織。
* その日以前に存在した組織で、一定の要件を満たすもの。お客様の組織で利用可能な場合は、 [オプトインメッセージング](#nr1core-optin) を請求管理 UI に表示するか、New Relic のアカウント担当者からお知らせします。

あなたの組織にコアユーザーがいる、またはコアユーザーを追加する機能がある場合は、このバージョンを使用しています。そうでない場合は、 [" New Relic One without core users" バージョン](#overview) となります。どのバージョンかを判断するためのヒントは、 [Pricing versions](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing#pricing-versions) をご覧ください。

## 本価格版への切り替えに関する詳細 [#nr1core-optin]

[2022 年 1 月 12 日以前に存在していた一部の New Relic 組織](#availability) には、"コア修正と呼ばれる受諾フォームを介して、より新しいコアユーザーリリースにオプトインするオプションがあります。" 一部の組織では、この受諾プロセスは UI にあり、 **Manage your plan** UI ページにあります。UI からこの修正案を受け入れる New Relic ユーザーは、自分の組織でその修正案の条件を受け入れる権限があることを表明することになります。

## ユーザーのユーザータイプを調整する際の注意点 [#changing-users]

ここでは、ユーザーのユーザータイプを調整する際の注意点を2つご紹介します。

* すべてのNew Relic組織において、ベーシックユーザーは2022年2月28日まで、コアユーザー機能に無料でプレビューアクセスできます。 **つまり、ベーシックユーザーをコアユーザーにアップグレードするのを3月まで待つのは理にかなっていると言えるでしょう。**
* お客様の組織が、当社の [オリジナルの価格設定モデル](/docs/accounts/original-accounts-billing/product-pricing/product-based-pricing) からコアバージョンの価格設定に切り替える場合（コアの修正または既存の [注文へのコアユーザーの追加のいずれかを受け入れる場合](/docs/licenses/license-information/product-definitions/new-relic-one-pricing-definitions/#order) ）、7日間の猶予期間があり、その間はユーザーのユーザータイプが請求に影響しません。これにより、 [ユーザーのユーザータイプをどのように編集するかを決める時間が得られます。](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type/#choose-user-type) 。猶予期間が終了すると、新しい [課金計算ルール](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/user-count-billing) が適用されます。猶予期間は、オリジナルの価格設定モデルからコアバージョンの価格設定への移行時に、組織内で一度だけ発生します。

ユーザーのユーザータイプを調整する準備ができたら、どのように調整するかは、ユーザーがどのユーザーモデルを使っているかによって異なります（[ユーザータイプの決定について](/docs/accounts/original-accounts-billing/original-users-roles/overview-user-models) ）。

* [New Relic Oneユーザーモデル](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-ui-and-tasks/#edit-user-type)
* [オリジナルユーザーモデル](/docs/accounts/original-accounts-billing/original-users-roles/users-roles-original-user-model/#update-user-type): なお、複数のアカウントを持っている場合、すべてのアカウントのすべてのユーザーを一箇所で見ることはできません。各アカウントのユーザーを確認・管理するには、それぞれのアカウントに入る必要があります。

ユーザーに選ぶべきユーザータイプを決定するヒントは、 [ユーザータイプ](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type) を参照してください。

2021年11月、「フルユーザー」のユーザータイプは、すべてのNew Relic組織を対象に、「フルプラットフォームユーザー」" に名称変更されました。詳しくは、 [フルユーザーの言語](/docs/accounts/accounts-billing/new-relic-one-user-management/user-type#full-user-language) をご覧ください。
