New Relic ユーザーの New Relic への追加方法、管理方法、ログイン方法などを自動的に制御するために、以下の機能を提供しています。

* <DNT>**SAML SSO**</DNT>: これにより、ユーザーはデフォルトの電子メール + パスワードを使用するのではなく、シングル サインオン (SSO) ID プロバイダー サービスを使用して New Relic にログインできるようになります。[有料版](https://newrelic.com/pricing)が必要です。
* <DNT>**SCIM provisioning**</DNT>自動ユーザー管理とも呼ばれる SCIM プロビジョニングにより、組織は ID プロバイダー サービスを使用して、 New Relicでのユーザーの追加や更新を自動化できます。 SCIM を使用する組織は、ほとんどの場合、SAML SSO を使用します。[有料版](https://newrelic.com/pricing)が必要です。

SAML と SCIM を有効にする方法については、 [認証ドメイン UI を](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more)使用して SAML と SCIM を設定します。 Okta、OneLogin、Azure には専用のNew Relicアプリがあることに注意してください。これらを使用する場合は、 **New Relic by account**ではなく、 **New Relic by organization**を選択する必要があります。
