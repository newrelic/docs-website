New Relic ユーザーの New Relic への追加方法、管理方法、ログイン方法などを自動的に制御するために、以下の機能を提供しています。

* <DNT>
    **SAML SSO**
  </DNT>

  : これにより、ユーザーはデフォルトのメール アドレスとパスワードを使用するのではなく、シングル サインオン (SSO) ID プロバイダー サービスを使用して New Relic にログインできるようになります。 [有料版が](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions)必要です。

* <DNT>
    **SCIM provisioning**
  </DNT>

  自動ユーザー管理とも呼ばれる SCIM プロビジョニングにより、組織は ID プロバイダー サービスを使用して、 New Relicでのユーザーの追加や更新を自動化できます。 SCIM を使用する組織は、ほとんどの場合 SAML SSO を使用します。 [Proまたはエンタープライズエディション](/docs/accounts/accounts-billing/new-relic-one-pricing-billing/new-relic-one-pricing-billing/#editions)が必要です。

SAML と SCIM を有効にする方法については、以下からユーザー モデルを選択してください。自分がどのモデルに属しているかわからない場合は、 [「 ユーザー モデルの概要 」](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)を参照してください。

<CollapserGroup>
  <Collapser
    id="new-account-model"
    title="新しいユーザーモデル"
  >
    [新しいユーザーモデル](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)のユーザーの場合、 [認証ドメインUI](/docs/accounts/accounts-billing/new-relic-one-user-management/authentication-domains-saml-sso-scim-more)を使用してSAMLとSCIMを設定します。Okta、OneLogin、Azureには専用のNew Relicアプリがあることに注意してください。これらを使用する場合は、「アカウント別のNew Relic」ではなく、「組織別のNewRelic」というアプリを選択する必要があります。
  </Collapser>

  <Collapser
    id="original-account-model"
    title="元のユーザーモデル"
  >
    組織が元のユーザーモデルを使用している場合は、可能であれば、[セルフサービスの移行手順](/docs/accounts/original-accounts-billing/original-users-roles/user-migration)を使用してユーザーを移行することをお勧めします。要件を満たしていないが、SAMLやSCIMを有効にしたい場合は、NewRelicアカウントの担当者にご相談ください。

    ここでは、いくつかのヒントとドキュメントのリンクを紹介します。

    * SAML：元のユーザーモデルを維持したい、またはそのままにして、新しいモデルに移行できない場合は、[元のSAMLドキュメント](/docs/accounts/accounts/saml-single-sign/saml-service-providers)を参照してください。
    * SCIM + SAML：SCIMプロビジョニングは、新しいユーザーモデルでユーザーレコードを作成します。そのため、SCIMを使用する前に、まずユーザーを新しいユーザーモデルに移行することをお勧めします。新しいモデルに移行する方法の詳細については、「[ユーザーの移行](/docs/accounts/original-accounts-billing/original-users-roles/user-migration)」を参照してください。
  </Collapser>
</CollapserGroup>
