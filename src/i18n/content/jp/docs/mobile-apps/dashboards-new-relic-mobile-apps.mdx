New Relic Mobile では、ダッシュボード機能を使用して、表示したい特定のデータを収集してグラフ化できます。New Relic Web サイトで作成されたダッシュボードはすべて、モバイル アプリに表示されます。

モバイル ダッシュボードを使用して、次のことができます。

* 一貫したUIによるカスタムの高密のインタラクティブな可視化により、知見を深める。
* プラットフォーム全体のあらゆる場所からのすべてのイベントと属性をグラフ化します。
* ほとんどのイベントにカスタム属性を追加したり、カスタム イベント タイプを送信したりして、ビジネスをよりよく理解し、ページ ビュー、ホスト トランザクションなど、顧客がプラットフォームとどのようにやり取りするかについて具体的な詳細を確認します。
* 高度なツールチップとズーム機能を使用してデータを探索し、コンテキスト化して、システムの動作をリアルタイムで監視します。
* ダッシュボードで属性とメトリクスを検索します。
* カスタムデータを任意のソースから New Relic とダッシュボードにレポートします。

## ダッシュボードの概要 [#getting-started]

1. New Relic Mobile アプリにログインし、 **Dashboards** をタップします。

2. 各ダッシュボードで、インデックスに次の情報が表示されます。

   * お気に入りのステータス。星で示されます。
   * 名前: ダッシュボードの名前。

## ダッシュボードのアクセス許可 [#dashboard-permissions]

ダッシュボードには、次の3つのタイプの権限があります。

* **Edit**\[編集]: アカウントの全員: すべてのユーザーがダッシュボードに対する完全な権限を持っています。
* **Read-only**\[読み取り専用]: アカウント内の全員: すべてのユーザーがダッシュボードを表示できますが、ダッシュボードを操作するための完全な権限を持っているのはあなただけです。他のユーザーはダッシュボードにアクセスできますが、ダッシュボードを複製することはできますが、編集または削除することはできません。
* **Private**\[プライベート]：ダッシュボードを表示できるのはあなただけです。メタデータ以外のすべてが非表示になります。

権限を編集するには、 [ユーザー管理ドキュメント](/docs/accounts/accounts-billing/new-relic-one-user-management/user-management-concepts/)を参照してください。

## チャートにフィルターを適用して絞り込む [#filter-and-refine]

検索条件を視覚的に表現するフィルタリング機能により、表示する情報を絞り込むことができます。任意のダッシュボードで、上部にある **Filter by any attribute or value**\[任意の属性または値でフィルター] リンクをタップします。

* フィルター バーを使用して、表示する値または属性を選択し、残りの要素をグラフから削除します。
* 高度なフィルター バーを開いてブール演算子 ( `=`、 `!=`、 `CONTAINS`、 `EXCLUDES`など) にアクセスし、データをフィルター処理するための複合および複雑な条件を追加します。
* フィルターを適用後、ダッシュボードには、選択した要素に関連するデータのみが表示されます。小さなカウンターは、1回に適用されるフィルター数を表します。

デフォルトのビューに戻るには、上部のフィルターの横にある「X」をタップして削除します。

## チャート凡例を使用してフィルターを適用する [#chart-legend]

凡例のあるグラフの凡例をタップすると、その系列のみが表示され、残りの系列はグラフから削除されます。これは、分析するデータを分離するのに役立ちます。

## 時間設定を調整する [#adjust-time]

デフォルトでは、ダッシュボードの各グラフには、クエリ ビルダーで作成されたときに指定された期間のデータが表示されます。ただし、タイム ピッカーを使用して、表示されているデータの時間範囲を変更し、すべてのグラフに同じ範囲を設定することができます。これは、特定の期間に何が起こったかを観察するためにデータを絞り込む必要がある場合に、インシデントのトラブルシューティングを行うときに特に役立ちます。

時間範囲を変更するには：

* ダッシュボードの右上にある時計アイコンをタップします
* ラジオボタンをタップして、メニューから利用可能なオプションのいずれかを選択します
* または、カスタム メニューを使用して、特定の開始タイムスタンプと終了タイムスタンプで時間範囲をカスタマイズできます。

## データを共有する [#share-data]

1. ダッシュボードで、右上の共有アイコンをタップします
2. デバイスの共有シートを使用して、ダッシュボード リンクを送信します

## データスクラバー [#data-scrubber]

チャートスクラバーを使用して、チャートが密集しており、ファセット間の距離が非常に近い場合に、チャートでデータポイントやファセットを選択できます。

指を針に沿って動かして、隣接するファセットをスムーズに選択し、関連するデータ ポイントを表示します。UI に表示される 1 つ以上の属性の選択を解除することで、データが大量に含まれているグラフを明るくすることもできます。

## テンプレート変数: ダッシュボードを動的にフィルタリングする [#template-variables]

New Relic ダッシュボードでは、テンプレート変数は [NRQL クエリの](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/) プレースホルダーであり、後で値を割り当てることができます。これにより、さまざまなユース ケースを処理する際の柔軟性が向上します。Web 上の New Relic でこれらの変数を使用してダッシュボードを作成すると、それらを New Relic Mobile で表示できるため、ダッシュボードを簡単にフィルタリングして、より正確なデータ インサイトを得ることができます。

Web でテンプレート変数を作成する方法の詳細については、 [ダッシュボードのドキュメント](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/)を参照してください。

Web でダッシュボードを作成すると、同じダッシュボードを表示し、モバイル アプリで同じ変数でフィルター処理できます。
