アカウント アドミニストレーターは、 [New Relicアラート](/docs/alerts-applied-intelligence/new-relic-alerts/get-started/introduction-applied-intelligence)から Android および iOS デバイスでプッシュ通知を受信するための設定を行うことができます。 [モバイル プッシュ](/docs/alerts-applied-intelligence/notifications/notification-integrations#mobile_push)の宛先を持つ任意のワークフローから問題<InlinePopover type="alerts"/>を受信できます。

## 要件 [#requirements]

[New Relic One ユーザー モデル](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)と[元のユーザー モデル](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-user-models)の両方のユーザーは、モバイル プッシュを[宛先](/docs/alerts-applied-intelligence/notifications/notification-integrations#mobile_push)として使用するワークフローを設定できます。

## 通知のオン/オフ [#enabling-alerts]

Android または iOS アプリから New Relic アカウントにログインすると、デバイスは自動的にユーザーに関連付けられます。[モバイルプッシュの送信先を必ずワークフローに追加して](/docs/alerts-applied-intelligence/applied-intelligence/incident-workflows/incident-workflows#add-workflow)ください。

## 問題のインシデントの詳細を表示する [#viewing-alerts]

通知は、デバイスのロック画面に自動的に表示されます。New Relic アプリを起動する。

1. デバイスに応じて、次の手順を実行します。

   * Androidデバイスでは、通知ドロワーから通知をタップします。
   * iOSデバイスでは、画面をスワイプします。

2. New Relic アプリの

   <DNT>
     **Alerts**
   </DNT>

   メニューから、任意の問題を選択して、関連付けられているアプリケーションのエラーの詳細を表示します。

3. 必要に応じて、

   <DNT>
     **Acknowledge**
   </DNT>

   を選択します。

4. 必要に応じて、追加の詳細を表示するには、

   <DNT>
     **Overview**
   </DNT>

   、

   <DNT>
     **Incidents**
   </DNT>

   、または

   <DNT>
     **Issue log**
   </DNT>

   を選択します。

メイン メニューの<DNT>**Alerts**</DNT>リストには、時間順に並べられた問題が表示されます。追加のフィルターを適用したり、右上の時間枠を変更したりすることもできます。

## アラート設定のトラブルシューティング [#troubleshooting]

モバイル デバイスでプッシュ通知が機能しない場合:

1. 要件 [を満たしていることを確認する](#requirements) 。
2. [アラートが有効になっていることを確認する](#enabling-alerts).
3. New Relic がプッシュ通知の送信を許可されていることを確認するには、モバイル デバイスの[通知設定](#notification-settings)を確認してください。

モバイルデバイスの通知設定が正しいにもかかわらず、通知を受け取れない場合は、 [アカウントからデバイスを削除](#account-delete) した後、 [アンインストール](#uninstall-app) と [New Relic アプリケーションを再インストール](#reinstall-app) してください。

    id="account-delete"
    title="NewRelicアカウントからAndroidまたはiOSデバイスを削除します"
  >
    New Relic アカウントからモバイル デバイスを削除するには、NerdGraph API エクスプローラーを使用します。

    1. <DNT>**Query**</DNT> [現在のデバイス](https://api.newrelic.com/graphiql?#query=query%20List%20%7B%0A%20%20actor%20%7B%0A%20%20%20%20mobilePushNotification%20%7B%0A%20%20%20%20%20%20devices%20%7B%0A%20%20%20%20%20%20%20%20appVersion%0A%20%20%20%20%20%20%20%20deviceId%0A%20%20%20%20%20%20%20%20deviceName%0A%20%20%20%20%20%20%20%20operatingSystem%0A%20%20%20%20%20%20%7D%0A%20%20%20%20%7D%0A%20%20%7D%0A%7D%0A)を確認するには、アクター -> mobilePushNotification -> デバイスを選択し、appVersion、deviceId、および deviceName を選択します。 このクエリを実行してデバイスのリストを取得します。

    2. <DNT>**Mutate**</DNT> デバイス[を削除する](https://api.newrelic.com/graphiql?#query=mutation%20%7B%0A%20%20mobilePushNotificationRemoveDevice%28deviceId%3A%20%22%22%29%20%7B%0A%20%20%20%20message%0A%20%20%7D%0A%7D%0A)には、mutation -> mobilePushNotificationRemoveDevice を選択し、上記のリストから deviceId を渡します。

       または、メニュー -> Settingsからアプリ内の<DNT>**Settings**</DNT>オプションからデバイスを削除することもできます。

    3. <DNT>**Push notification devices**</DNT>の下を見て、そこから削除してください。 iOSでは右から左にスライドして<DNT>**Delete**</DNT>デバイスに移動し、Androidではタップして <DNT>**Delete**</DNT>

       [再インストール](#uninstall-app) お使いのデバイスからNew Relicアプリを再インストールする手順を続けます。
  </Collapser>

  <Collapser
    id="reinstall-app"
    title="NewRelicモバイルアプリを再インストールします"
  >
    New Relic モバイルアプリを再インストールするには。

    1. Android デバイスから[<DNT>**Google Play Store**</DNT>](https://play.google.com/store/apps/details?id=com.newrelic.rpm)を選択します。

       また

       iOS デバイスのホーム画面から[<DNT>**App Store**</DNT>を選択します。](https://itunes.apple.com/us/app/new-relic/id594038638?mt=8)

    2. <DNT>**New Relic**</DNT>を検索します。

    3. アプリをダウンロードします。

    4. ダウンロードが終了したら、New Relic のアカウントで New Relic モバイルアプリにサインインします。
