New Relic の iPhone および iPad アプリのユーザーインターフェースは、New Relic の [標準ユーザーインターフェース](/docs/site/standard-dashboard-features) と同様の機能を提供し、モバイルユーザー向けに細部をカスタマイズしています。

## 機能 [#dashboard]

New Relic の iOS アプリには、これらの New Relic の製品と機能が含まれています。

* [APM](/docs/apm) （iPhoneおよびiPad）。リアルタイムおよび [過去の](#time) データを含みます。

  <ImageSizing width="33px" height="20px">![icon-txn-list-ios](./images/icon-txn-list-ios.png "icon-txn-list-ios") </ImageSizing>

  のアイコンを選択すると、取引の詳細が表示されます。 **Overview Charts** を選択すると、お客様の上位5つの取引のサマリーチャートが表示されます。

* [ブラウザモニタリング](/docs/browser) （iPhoneおよびiPad）。平均ページロード時間、ブラウザのApdex、平均スループットなどの概要ダッシュボードを提供。

* [インフラ監視](/docs/infrastructure) （iPhoneのみ）。

* [アラート](#ios-alerts) （iPhone、iPad）。アラートやデプロイメントの通知を受け取ることができます。

* [合成モニター](#synthetics) （iPhoneのみ）。

* [モバイルモニタリング](#monitoring) (iPhone および iPad)。 [クラッシュレポート](/docs/mobile-monitoring/mobile-monitoring-ui/mobile-app-dashboards/mobile-apps-crash-dashboard) 、ネットワークエラー、APIコール、アクティブユーザー数を含む。

New Relic の iOS アプリには、New Relic の Web アプリケーションのすべての機能はありません。より詳細な分析を行うには、Web ブラウザで New Relic アカウントにサインインしてください。

## 時間範囲 [#time]

アプリケーションやホストを表示する際、ページの右上にある時計のアイコンを使って、表示される時間枠を変更することができます。この機能は、標準の New Relic [タイムピッカー](/docs/site/timepicker-setting-time-periods-to-view-data) と同様です。機能は以下の通りです。

* New Relic のチャートをスクラブして、タイムライン上で前後に移動します。
* タイムピッカーを選択して、今が終了する時間帯（30分前から90日前まで）を選択します。
* iPadの場合：今以外の終了時刻を指定するには、トグルを **Ending Now** から **Custom Date** へとスライドさせます。

## 合成のモニタリング [#synthetics]

iOSアプリを使って、 [合成モニター](/docs/synthetics/new-relic-synthetics/getting-started/new-relic-synthetics) のデータを見ることができます。モニターの稼働率、ロードタイム、ロードサイズなどのチャートを見ることができます。カレット <Icon name="fe-chevron-down"/>アイコンを選択すると、より詳細なチャートが表示されます。モニターをミュートまたは無効にしたり、最近発生したエラーの詳細を表示することができます。スクリプトを使用しているモニターでは、スクリプトログの表示と検索が可能です。

## アラート [#ios-alerts]

iOS アプリを New Relic アカウントに接続すると、デバイスが自動的にユーザーチャネルに関連付けられます。 その後、 [ターゲットポリシーにユーザーチャネルを追加することで、アラートを受け取ることができます。iOS のアラートの場合、通知はロック画面に表示され、アラートをスワイプすることで確認することができます。](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts#mobile-push)

[任意のアラートを選択して、エラーの詳細を表示したり、アラートを承認したりすることができます。また、同僚が開いているイベントを承認すると、New Relic はプッシュ通知を送信します。そして、すべての **Critical** イベントが終了すると、New Relic は最終的な終了通知を送信します。](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts#mobile-push)

## [モバイルのモニタリング [#monitoring]](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts#mobile-push)

[モバイルアプリケーションをお持ちで、 ](/docs/alerts/new-relic-alerts-beta/managing-notification-channels/notification-channels-controlling-where-send-alerts#mobile-push)[モバイルモニタリング](/docs/mobile-apps/new-relic-mobile) をインストールしている場合、iPhone や iPad から直接パフォーマンスを監視することができます。モバイルモニタリングには、ネットワークエラー、APIコール、アクティブユーザー数などが含まれます。また、詳細な個別のクラッシュレポートを見ることで、特定のクラッシュインシデントをより深く理解することができます。

## データプライバシー [#privacy]

New Relic のモバイルアプリは、認証やトラブルシューティングに必要な情報のみを記録します。

* 名字と名前を含む、New Relic アカウントに関連付けられたユーザーの電子メールアドレス (認証のみを目的とする)
* IPアドレス
* デバイスID

詳細については、当社の[モバイルデータプライバシーおよびセキュリティについてのドキュメント](/docs/mobile-monitoring/new-relic-mobile/get-started/security-mobile-apps)を参照してください。
