New Relic Insightsは、ソフトウェアとインフラストラクチャシステムに関するデータに対するクエリと視覚化を可能にしてビジネスの観点からそのデータをよりよく理解できるようにする分析ツールです。

## Insightsとは [#definition]

Insightsは、[New Relicデータのクエリとチャート作成を行う](/docs/using-new-relic/data/understand-data/query-new-relic-data)ためのUIです。Insights UIはすべてのNew Relicユーザーが使用できますが、使用できるデータの量とデータの保持期間は、サブスクリプションにより異なります。

Insights UIを使用して、次のことが行えます:

* 当社のクエリ言語である[NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql)を使用して強力なクエリを行い、トラブルシューティングを行い問題を解決する。
* NRQLを使用して[カスタムチャート](/docs/insights/use-insights-ui/manage-dashboards/add-customize-nrql-charts)やレポート、[ダッシュボード](/docs/insights/use-insights-ui/manage-dashboards/create-edit-insights-dashboards)を生成する。
* 関連するダッシュボードを[データアプリケーション](/docs/insights/new-relic-insights/data-apps/building-data-apps)と組み合わせると、チームと技術担当以外のユーザーは、簡単にデータを詳しく調べることができます。

Insights Proサブスクリプションにより、[一部のイベントデータ](/docs/accounts/accounts/data-management/event-data-retention#insights-pro)（特定の[データタイプ](/docs/using-new-relic/data/understand-data/new-relic-data-types)）のデータ保持期間を延長できます。例えば、New Relic APM Pro サブスクリプションがあった場合、Insights Pro サブスクリプションを追加すると、カスタムイベントを含むAPMでレポートされるイベントの保持期間が長くなります。有料のInsightsサブスクリプションにより、Insights UIまたは別の[クエリオプション](/docs/using-new-relic/data/understand-data/query-new-relic-data)を使用しているかにかかわらず、データにアクセスできる期間が延びることに注意してください。Insights Proサブスクリプションのメリットの詳細については、[イベントデータ保持期間](/docs/accounts/accounts/data-management/event-data-retention#insights-pro)をご覧ください。

<Callout variant="important">
  当社では、次のようにデータのクエリとチャート作成の方法を改善しています：[New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)。切り替えのメリットと注については、[移行ガイド](/docs/transition-new-relic-one-dashboards-insights)をご覧ください。
</Callout>

## ユーザーとデータの分析 [#insights]

New Relic Insightsは、ユーザー行動、ビジネストランザクション、顧客洞察などを分析できるよう、New Relic製品や[インテグレーション](https://newrelic.com/integrations)からのデータを活用します。New Relic APMはユーザーによるアプリの最適化を支援する一方で、Insightsは会社の利益の最適化を支援します。Insightsは、Cレベルのエグゼクティブから最前線の開発者にいたるまで、誰もがより的確な判断を下し、より優れた製品を構築するのを手助けするソフトウェア分析リソースです。

Insights UIを使うことで、アプリとホストの問題点をリアルタイムで特定し、継続的なデータトレンドを追跡するためのダッシュボードを迅速、簡単に構築できます。

Insightsを使用して、次のものにアクセスできます:

* クラウドにNew Relicがレポートしたデータを格納する、ハイパワーでインデックスフリーのデータベース。
* [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql): データについて強力で特定の分析が可能な、SQLと同様のクエリ言語。
* [イベントAPI](/docs/insights/insights-data-sources/custom-data/introduction-event-api)：カスタム[イベント](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data)と属性をレポートし、[ローカルアプリケーション用のデータを取得](/docs/insights/new-relic-insights/querying-data-remotely/querying-your-data-remotely)するためのREST的API。
* データを迅速かつ明確に表す[多くのチャート](/docs/insights/use-insights-ui/manage-dashboards/chart-types)を伴ったダッシュボード管理インターフェイス、ならびに[こうしたダッシュボードを管理するためのAPI](/docs/insights/insights-api/manage-dashboards/insights-dashboard-api)。

Insightsは、New Relicの [AndroidおよびiOSのInsightsモバイルアプリ](/docs/mobile-monitoring/new-relic-mobile-apps/insights-app/insights-app-android-ios)[Apple TV](/docs/mobile-monitoring/new-relic-mobile-apps/insights-app/insights-app-apple-tv)でも見ることができます。

## データのレポートとセキュリティ [#security]

Insightsは、[New Relic製品およびサービスからレポートされたデータ](/docs/insights/insights-data-sources/default-data/insights-default-data-other-new-relic-products)にアクセスします。それ自身がデータをレポートすることはありません。

New Relicのセキュリティ対策の詳細については、当社の[セキュリティとプライバシーのドキュメント](/docs/using-new-relic/new-relic-security/security/security-matters-data-privacy-new-relic)を参照するか、[New Relicのセキュリティウェブサイト](https://newrelic.com/why-new-relic/security)をご覧ください。

## Insightsでどのようなデータをクエリできるか [#data-available]

Insightsでクエリを行えるデータのタイプについて理解するには、[NRQLの概要](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql#what-you-can-query)をご覧ください。クエリを開始するには、[NRQLの学習を開始する](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql#start-learning)をご覧ください。

ご自分の顧客、アプリケーション、そしてビジネスをより詳しく理解するため、一部の[イベント](/docs/using-new-relic/data/understand-data/new-relic-data-types#events-new-relic)に[カスタム属性](/docs/insights/new-relic-insights/decorating-events/insights-custom-attributes)を追加できます。ページビュー、ホストトランザクション、あるいはオンラインで報告できるものなら何にでも、[カスタムイベント](/docs/insights/new-relic-insights/adding-querying-data/inserting-custom-events)を作成することもできます。

[イベントデータエクスプローラー](/docs/insights/explore-data/data-explorer/event-explorer-query-chart-your-event-data)を使用することで、イベントデータを探して、これを理解することができます。

## メトリックタイムスライスデータ [#metric-data]

New Relic APM、Browser、Mobileは、[メトリックタイムスライスデータ](/docs/using-new-relic/data/understand-data/new-relic-data-types#timeslice-data)と呼ばれる単純なフォーマットでメトリックスをレポートし、表示します。このタイプのデータのクエリをするためには、以下を実行します：

* [Insightsのメトリックデータエクスプローラー](/docs/insights/new-relic-insights/explore/metric-data-explorer-search-chart-metrics-sent-new-relic-agents)を使用して、このデータ（[カスタムメトリックス](/docs/agents/manage-apm-agents/agent-metrics/custom-metrics)を含む）を検索しチャートを作成する。
* [NRQLを使用してクエリを行う。](/docs/query-data/nrql-new-relic-query-language/nrql-query-tutorials/query-metric-timeslice-data-nrql)
