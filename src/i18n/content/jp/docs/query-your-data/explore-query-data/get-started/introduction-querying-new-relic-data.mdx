[New Relic](http://one.newrelic.com)は、システム全体のすべてのデータにアクセスできる強力な可観測性プラットフォームです。私たちは[あなたのエンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)から送信されたテレメトリデータをホストします。これは基本的に、アプリケーション、サービス、ホストなど、監視できるデータを含む識別可能なものです。

[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) でお客様のデータを見ることができるようになっていますが、 [UI](#query-ui) や [API](#query-apis) など、いくつかの方法でお客様のデータへのアクセスを調整し、可視性をカスタマイズすることができます。

<Callout variant="important">
  New Relic に保存されているお客様のデータをよりよく理解するために、 [Data types](/docs/using-new-relic/data/understand-data/new-relic-data-types) をご覧ください。
</Callout>

## UIでデータを閲覧 [#browse-data]

[私たちのプラットフォーム](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)は、NRQLやクエリ言語の知識を必要としないいくつかのエクスペリエンスを提供します。 UIで、\[**データのクエリ**]ボタンに移動するか、\[**データの参照**]ドロップダウンをクリックして、調査するデータ型（メトリック、イベント、ログ、およびトレース）を選択します。

* [イベント](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data) および [メトリクス](/docs/using-new-relic/data/understand-data/new-relic-data-types#metrics) については、直感的なデータナビゲーターである [データエクスプローラー](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) を使用して、ビジュアライゼーションを作成します。エクスプローラから [クエリビルダ](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) に切り替えて、クエリを確認したり改良したりすることができます。
* [Distributed tracing query](/docs/understand-dependencies/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one): トレースのクエリーに特化したUIです。
* [ログクエリ](/docs/logs/new-relic-logs/ui-data/query-syntax-logs)：NewRelicで[ログ管理データ](/docs/logs/new-relic-logs/get-started/introduction-new-relic-logs)をクエリするための専用UI。

## UIでのデータ検索 [#query-ui]

データを閲覧する以上のことを行う準備ができている場合は、すべてのアクターになり、NewRelicUIでクエリをパーソナライズしてください。 [New Relicクエリ言語](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/)や[PromQLスタイルのクエリ言語](https://prometheus.io/docs/prometheus/latest/querying/basics/)などのクエリ言語を使用して、クエリを完全に柔軟に編集します。たとえば、 [`WHERE`句](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/#sel-where)を追加したり、戻り値を変更したり、[他の種類の視覚化](/docs/query-your-data/explore-query-data/use-charts/chart-types/)に変更したりすることができます。

<Callout variant="tip">
  クエリ言語は初めてですか？まず、 [データエクスプローラー](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) でデータを閲覧し、次に [クエリビルダー](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) で、作成したクエリを確認したり、改良したりすることができます。
</Callout>

データを取得してチャートを作成するための独自のクエリを書くには、2つの方法があります。

* [NRQL モードのクエリビルダ](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data): [New Relic query language](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) (NRQL)を使用してクエリを実行します。これは、ほとんどの UI エクスペリエンスを構築するために使用している言語と同じで、New Relic でデータをクエリする最も高度な方法です。
* [PromQL スタイルのクエリビルダ](/docs/query-your-data/explore-query-data/chart-builder/use-advanced-promql-mode-specify-data): PromQL スタイルのクエリを使って基本的なクエリを書く。

## APIによるデータの照会 [#query-apis]

New Relic プラットフォームに入ることができない場合は、API を使用して New Relic のデータを取得し、クエリを実行することができます。たとえば、 [NRQL](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) (当社のクエリ言語) のクエリを [NerdGraph](/docs/apis/graphql-api/get-started/introduction-new-relic-nerdgraph) (当社の GraphQL API) で実行することができます。

詳細については、New Relic APIs の紹介 [をご覧ください。](/docs/apis/intro-apis/introduction-new-relic-apis/)
