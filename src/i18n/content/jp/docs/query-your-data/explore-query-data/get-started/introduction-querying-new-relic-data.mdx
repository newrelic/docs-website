[New Relic](https://one.newrelic.com)は、システム全体のすべてのデータにアクセスできる強力なオブザーバビリティ プラットフォームです。私たちは[あなたのエンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)から送信されたテレメトリ データをホストします。これは基本的に、アプリケーション、サービス、ホストなど、あなたが監視できるデータを含む、私たちが識別できるものです。あなたはそれに名前を付けます！

[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/) でお客様のデータを見ることができるようになっていますが、 [UI](#query-ui) や [API](#query-apis) など、いくつかの方法でお客様のデータへのアクセスを調整し、可視性をカスタマイズすることができます。

<Callout variant="important">
  New Relic に保存されているお客様のデータをよりよく理解するために、 [Data types](/docs/using-new-relic/data/understand-data/new-relic-data-types) をご覧ください。
</Callout>

## UIでデータを閲覧 [#browse-data]

当社のプラットフォームは、NRQL やクエリ言語の知識を必要としないいくつかのエクスペリエンスを提供します。

* New Relic データを探索するには、データの探索と視覚化の作成のための直感的なデータ ナビゲータで[ある Data Explorer を](/docs/query-your-data/explore-query-data/browse-data/introduction-data-explorer)使用できます。[クエリビルダーを](/docs/query-your-data/explore-query-data/query-builder/introduction-query-builder)使用してクエリを絞り込むこともできます。
* トレース データを調査するには、 [分散トレーシング クエリ](/docs/understand-dependencies/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one)を使用できます。
* ログを調べるには、[ログ UI](/docs/logs/ui-data/use-logs-ui/)を使用できます。

## UIでのデータ検索 [#query-ui]

データを閲覧する以上のことを行う準備ができている場合は、すべてのアクターになり、NewRelicUIでクエリをパーソナライズしてください。 [New Relicクエリ言語](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language/)や[PromQLスタイルのクエリ言語](https://prometheus.io/docs/prometheus/latest/querying/basics/)などのクエリ言語を使用して、クエリを完全に柔軟に編集します。たとえば、 [`WHERE`句](/docs/query-your-data/nrql-new-relic-query-language/get-started/nrql-syntax-clauses-functions/#sel-where)を追加したり、戻り値を変更したり、[他の種類の視覚化](/docs/query-your-data/explore-query-data/use-charts/chart-types/)に変更したりすることができます。

<Callout variant="tip">
  クエリ言語は初めてですか?[メトリクスとイベント](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer)でデータの参照を開始し、[クエリ ビルダー](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data)に目を向けて、作成したクエリを確認し、それを調整します。
</Callout>

データを取得してチャートを作成するための独自のクエリを書くには、2つの方法があります。

* [NRQL モードのクエリビルダ](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data): [New Relic query language](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language) (NRQL)を使用してクエリを実行します。これは、ほとんどの UI エクスペリエンスを構築するために使用している言語と同じで、New Relic でデータをクエリする最も高度な方法です。
* [PromQL スタイル モードのクエリ ビルダー](/docs/query-your-data/explore-query-data/chart-builder/use-advanced-promql-mode-specify-data): PromQL スタイルのクエリを使用してクエリを記述します。

New Relic の CodeStream 拡張機能を使用して、 [IDE からデータをクエリする](/docs/codestream/observability/query-builder/)こともできます。

## APIによるデータの照会 [#query-apis]

プログラムによるオプションが必要な場合、またはより多くのクエリ機能を解き放ちたい場合は、API を使用して New Relic データをクエリできます。NerdGraph API は、アカウント全体のクエリや長期の履歴データ セットのエクスポートなど、UI では利用できないより強力なオプションを提供します。詳細については、「 [NerdGraph の概要](/docs/apis/nerdgraph/get-started/introduction-new-relic-nerdgraph/#tutorials)」を参照してください。
