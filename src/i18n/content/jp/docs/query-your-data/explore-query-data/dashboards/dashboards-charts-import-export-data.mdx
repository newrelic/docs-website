[ダッシュボードを](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/)使用すると、システムの健全性を視覚化して追跡し、問題に対処し、さらなるステップを計画することができます。 さまざまなインポートおよびエクスポート オプションを使用して、すべての情報をチーム、顧客、その他の関係者と共有する方法を学びます。

## ダッシュボードを JSON としてインポートする [#import-json]

コードレベルですぐに使用できるダッシュボードを変更する場合は、ダッシュボードを JSON としてインポートできます。 たとえば、2 つの既存のダッシュボードを結合してカスタムダッシュボードを作成する場合、ベースのダッシュボードの JSON を変更して、それをNew Relicにインポートし直すことができます。

ダッシュボードを JSON としてインポートするには:

1. <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT>に移動します。

2. <DNT>**...**</DNT>アイコンをクリックして既存のダッシュボードの JSON を開き、 <DNT>**Copy JSON**</DNT>を選択します。

3. <DNT>**Dashboards**</DNT>ページに戻ります。

4. 右上隅にある<DNT>**Import dashboard**</DNT>をクリックします。

5. JSON コードを貼り付けます。

6. ダッシュボードのアカウントと権限設定を選択します。 一度設定したアカウントを変更することはできませんが、 [権限はいつでも変更](/docs/query-your-data/explore-query-data/dashboards/manage-your-dashboard/#dash-settings)できます。

7. <DNT>**Save**</DNT>をクリックします。

## ダッシュボードを PDF としてエクスポートする [#export-dashboards-pdf]

ダッシュボードを PDF ファイルとしてエクスポートするには、ダッシュボードの右上隅にある<DNT>**...**</DNT>アイコンをクリックし、 <DNT>**Export dashboard as PDF**</DNT>を選択します。

<img title="Export dashboard as PDF" alt="Screenshot of how to export a dashboard as a PDF file" src="/images/dashboards_screenshot-full_export-pdf.webp" />

<figcaption>
  <DNT>**[one.newrelic.com &gt; All capabilities](https://one.newrelic.com/all-capabilities) &gt; Dashboards**</DNT>に移動して既存のダッシュボードを開きます。
</figcaption>

UI経由でダッシュボードをPDFとしてエクスポートする場合は、次の点を考慮してください：

* [カスタム視覚化は](/docs/query-your-data/explore-query-data/dashboards/add-custom-visualizations-your-dashboards/)サポートされていません。

カスタムディメンションやテンプレート変数のオーバーライドなどの追加オプションを使用してPDFまたはPNGとしてエクスポートするには、[NerdGraph API](#export-api)を使用します。

## NerdGraph API経由でダッシュボードとチャートをエクスポートする [#export-api]

NerdGraph APIを使用して、ダッシュボードや個々のチャートウィジェットをスナップショット画像としてプログラムでエクスポートします。APIは、カスタムディメンション、テンプレート変数のオーバーライド、エンティティフィルターなど、UIでは利用できないオプションをサポートしています。

### ダッシュボードをPDFまたはPNGとしてエクスポート [#api-dashboard-snapshot]

`dashboardCreateSnapshotUrl`ミューテーションは、ダッシュボードページ全体のスナップショットURLを生成します。

<Callout variant="tip">
  `guid`パラメーターには、ダッシュボードのGUIDではなく、**ページ**のGUIDが必要です。ダッシュボードには複数のページを含めることができます。ページのGUIDを見つけるには、`entitySearch`クエリを使用するか、ダッシュボードページを開いて<DNT>**...**</DNT>メニューから<DNT>**Manage JSON**</DNT>を選択します。

  ```graphql
  {
    actor {
      entitySearch(query: "parentId ='YOUR_DASHBOARD_GUID' AND tags.isDashboardPage = 'true'") {
        results {
          entities {
            guid
            name
          }
        }
      }
    }
  }
  ```
</Callout>

**Mutation schema：**

```graphql
mutation {
  dashboardCreateSnapshotUrl(
    guid: EntityGuid!
    params: SnapshotUrlInput
  ): String
}

input SnapshotUrlInput {
  timeWindow: SnapshotUrlTimeWindowInput
  display: SnapshotUrlDisplayInput
  variables: [SnapshotUrlVariableInput!]
  filter: String
  format: SnapshotUrlFormat
}

input SnapshotUrlTimeWindowInput {
  beginTime: EpochMilliseconds
  endTime: EpochMilliseconds
  duration: Milliseconds
}

input SnapshotUrlDisplayInput {
  width: Int
  height: Int
}

input SnapshotUrlVariableInput {
  name: String!
  values: [String!]!
}

enum SnapshotUrlFormat {
  PDF
  PNG
}
```

### チャート画像を取得する [#api-widget-snapshot]

`dashboardWidgetCreateSnapshotUrl`ミューテーションは、単一のチャートウィジェットのPNGスナップショットURLを生成します。ウィジェットがダッシュボード上に存在する必要はありません — インラインで定義できるため、プログラムでチャート画像を生成するのに役立ちます。

**Mutation schema：**

```graphql
mutation {
  dashboardWidgetCreateSnapshotUrl(
    widget: DeclarativeUiWidget!
  ): WidgetSnapshotResult
}

type WidgetSnapshotResult {
  url: String!
}
```

`widget`パラメーターは`DeclarativeUiWidget`を受け付けます — 3つのネストされたレイヤーを持つJSONオブジェクト：

```
declarative/widget envelope
└── widget node        ← title and widget-level props
    └── visualization  ← visualization type and configuration
```

```json
{
  "type": "declarative/widget",
  "version": 1,
  "content": {
    "type": "widget",
    "props": {
      "title": "My Chart Title"
    },
    "content": {
      "type": "visualization",
      "id": "viz.line",
      "props": {
        // visualization-specific config (nrqlQueries, colors, thresholds, etc.)
      }
    }
  }
}
```

JSONとしてエクスポートしたダッシュボードからウィジェット定義を構築する場合は、次のフィールドマッピングを使用します：

</Callout>

<Callout variant="important">

</Callout>

## チャートをCSVとしてエクスポート [#export-charts-csv]

ダッシュボードのテーブル チャートを CSV ファイルとしてエクスポートするには、チャートの右上隅で<DNT>**...**</DNT>をクリックし、 <DNT>**Export as CSV**</DNT>を選択します。

<img title="Export chart as CSV" alt="Screenshot of how to export a table chart as a CSV file" src="/images/dashboards_screenshot-crop_export-csv.webp" />

チャートを CSV としてエクスポートする場合は、次の点に注意してください。

* データは常にUTC時間でエクスポートされます
* エクスポートは、次のようなアグリゲータ関数をサポートしていません。 `average()`
* エクスポートは`COMPARE_WITH`句をサポートしていません
* シナリオによっては、UI がエクスポートされた CSV に複製されていないデータの追加フォーマットを実行し、小さな不一致が生じる可能性があります (たとえば、-1.1E-17 が UI では 0 としてレンダリングされるが、CSV では 0 としてレンダリングされないなど)。
* クロスアカウントクエリの結果をエクスポートすることはできません。
