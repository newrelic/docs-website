データエクスプローラーを使用すると、[NRQL](/docs/query-your-data/nrql-new-relic-query-language)の知識がなくても、すべてのデータを視覚的に操作できます。データエクスプローラーを使用してデータにアクセスしクエリやカスタマイズ、視覚化を行い、サービスとサービスをつなぎ、一貫したキュレート済みの体験を実現します。

## 重要な理由 [#why-matters]

システムの状態を調査しますか？リソースの計画を作成し、インシデントを特定しそれに対応し、動作不良のトラブルシューティングを行う必要がありますか？データエクスプローラーにより、NRQLを使用したりクエリを構築せずに、視覚的なメニューを通じて、探しているデータを簡単に特定、取得し、視覚化できます。

データエクスプローラーを使用して、次のことを行えます。

* [イベント](/docs/using-new-relic/data/understand-data/new-relic-data-types#event-data)や[メトリクス](/docs/using-new-relic/data/understand-data/new-relic-data-types#metrics)データに、迅速で直感的な方法でアクセスする。
* 視覚化し簡単に実行できるようにして、データのディメンションを詳しく調べる。
* 生データや、進歩や配信などについての知見を提供するさまざまな視覚化。
* フィルターによりデータを掘り下げる。
* 1回のクリックで検索を[ダッシュボード](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards)に追加する。
* [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql)の機能方法の理解：データエクスプローラーは、利用可能なデータを検索している間にクエリがどのように構築されるかを示します。

<Callout variant="tip">
  InsightsからNew Relic Oneに切り替える場合[移行ガイド](/docs/transition-new-relic-one-dashboards-insights)を参照してください。
</Callout>

## データのクエリ [#explore-data]

データエクスプローラーにアクセスするには、[one.newrelic.com](https://one.newrelic.com)に移動し

* ページの上部にある**データをクエリ**アイコンをクリックします。

  ![NR Oneの移行：データを調査](./images/explore-data-icon.png "NR Oneの移行：データを調査")

* **データのブラウズ**ドロップダウンで、**イベント**または**メトリクス**を選択します。

  ![browse_data_explorer.png](./images/browse_data_explorer.png "browse_data_explorer.png")

データエクスプローラーは、New Relicにあるすべてのデータを詳しく調べるためのポータルです。データエクスプローラーUIにあるイベントやメトリクスを詳しく調べたり、[Logs](/docs/logs/log-management/ui-data/query-syntax-logs/)や[Traces](/docs/distributed-tracing/ui-data/understand-use-distributed-tracing-ui/)クエリインタフェースを選択して詳しく調べることができます。また、タブをクリックして、いつでも[クエリビルダー](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder)にアクセスすることもできます。

### データエクスプローラーを使用する [#use-explorer]

データエクスプローラーは、上部にある範囲絞り込みセクション（a）、左側のデータブラウジングエリア（b）、ワークスペース（c）で構成されます。

![データエクスプローラービュー](./images/data_explorer_updated.png "data_explorer_updated.png")

データエクスプローラーを使用する場合は、以下の手順に従います。

1. エクスプロレーション範囲の定義：ブラウズする**アカウント**と**データタイプ**（メトリックまたはイベント）を選択します。

2. [タイムピッカー](/docs/using-new-relic/user-interface-functions/view-your-data/time-picker-set-time-periods-view-data)を使用して時間範囲を選択します。

3. 左側のブロックを使用して、イベントかメトリクスのいずれかについて使用可能なデータをブラウズし、検索を作成します。1ブロックにつき1つの要素しか選択できません。ブロックは検索可能です。

## 調査を可視化し絞り込む [#workspace]

調査の結果は、右側のワーキングスペースに表示されます。

![dataexplorer02.png](./images/dataexplorer02.png "dataexplorer02.png")

<figcaption>
  調査を絞り込むか、チャートを共有する
</figcaption>

ワーキングスペースには、次のものが表示されます。

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="workspace-query-area"
    title="クエリエリア"
  >
    クエリエリアでは、クエリを異なる構成要素に分解します。ここで、NRQLクエリとして調査の結果を簡単に読み、プロットしている正確なデータをチェックできます。

    NRQLに習熟していない場合、このエリアをチェックしクエリの作成方法について学習します。

    クエリの別の部分は次のとおりです。

    * \[EVENTS ONLY]`FROM`：最初のブロックで選択したイベント。

    * [`SELECT`](/docs/query-data/nrql-new-relic-query-language/getting-started/nrql-syntax-clauses-functions#state-select)：選択したプロットまたはメトリック。この入力では、1つの値と1つの関数のみをプロットします。

    * [`GROUP BY`](/docs/query-data/nrql-new-relic-query-language/getting-started/nrql-syntax-clauses-functions#sel-facet)：`FACET`句を表し、選択した次元ごとにデータをグループ化します。

    * [`LIMIT`](/docs/query-data/nrql-new-relic-query-language/getting-started/nrql-syntax-clauses-functions#sel-limit)：表示する値の量を入力します。

    * [`WHERE`](/docs/query-data/nrql-new-relic-query-language/getting-started/nrql-syntax-clauses-functions#sel-where)：このフィールドを使用して、結果をさらにフィルタリングします。この入力では、`n`値をプロットします。

      各項目は、**x**をクリックしてクエリから削除できます。
  </Collapser>

  <Collapser
    className="freq-link"
    id="workspace-query-display"
    title="NRQLクエリ表示"
  >
    上記のフィールドと、タイムピッカーで[選択した時間範囲](#explore-data)で構成される、完全なクエリを表示できます。

    <Callout variant="tip">
      より詳細な検索を行う、またはチャートをカスタマイズする必要がある場合データエクスプローラーから、[クエリビルダー](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data)にアクセスしてクエリを編集できます。
    </Callout>
  </Collapser>

  <Collapser
    className="freq-link"
    id="workspace-chart"
    title="チャートエリア"
  >

    次元を選択している場合、チャートは異なるファセットで更新されます。チャートの下に、ファセットのリストとそれぞれの値があるファセットの表が表示されます。

    ファセットテーブルを使用して、データをドリルダウンします。ファセットをクリックすることで、フィルターとして適用されます。表は表示されたままなので、別のファセットを簡単に選択して調査を続けることができます。
  </Collapser>

## 使用事例

次の例で、データエクスプローラーをいつどのように使用するかについて学習してください。

<CollapserGroup>
  <Collapser
    className="freq-link"
    id="use-case-validate-data"
    title="新規データを確認"
  >
    **新規インストゥルメンテーションに接続したばかりで、新規データを利用できるかどうか確認したいと思っています。**

    1. アカウントとイベント、または新規データを生成しているメトリックを選択します。
    2. データエクスプローラーで異なるツールを使用して、使用可能になった新しいデータを処理：テーブルとしてのイベントまたはメトリックの生データを見て、リストとして作成、またはクリックしてチャートとしてプロットされたものを表示します。
    3. イベントやメトリックを選択した後、次元でデータの形状を確認します。濃度を使用して、あらゆるデータの異なる視点を確認できます。
    4. 関連性が見つかった場合ダッシュボードに保存するか、同僚と共有します。
  </Collapser>

  <Collapser
    className="freq-link"
    id="use-case-new-data"
    title="データを検証"
  >
    **カスタムイベント/メトリックを変更したので、この変更が正常に行われたかどうかをチェックする必要があります。**

    1. データエクスプローラータブで、変更したアカウントとデータタイプ、イベント/メトリックを選択します。
    2. エンティティがデータをレポートしていることと、すべての属性がプロットされていることを確認します。
    3. 変更した属性を検索し、更新が正常に行われたことをチェックします。
  </Collapser>

  <Collapser
    className="freq-link"
    id="use-case-troubleshooting"
    title="データの問題のトラブルシューティング"
  >
    **アラートやダッシュボードからのイベント/メトリックで何か問題があることが分かっています。イベント/メトリック属性の動作について、根本原因を知る必要があります。**

    1. データエクスプローラーで、メニューを使用して、想定どおりに動作していないイベントやメトリックを選択し、データエクスプローラーでそのチャートをプロットします。
    2. そこから、そのデータのディメンションを掘り下げて、関連する属性によりフィルターを適用できます。
    3. 分布、値のランキング、経時的進化など、異なる観点からデータを表示することもできます。
    4. 関連性が見つかった場合ダッシュボードに保存するか、同僚と共有します。
  </Collapser>
</CollapserGroup>
