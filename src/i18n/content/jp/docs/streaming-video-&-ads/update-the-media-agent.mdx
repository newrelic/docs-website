<Callout variant="tip">
  このガイドは、バージョン3.xxより前のバージョンからバージョン3.xxへの一度限りの移行に関するものです。既にバージョン3.xxをご利用中で、より新しいバージョンにアップデートする必要がある場合は、お使いのプラットフォームの標準アップデート手順に従ってください。
</Callout>

以前のバージョンからの移行をできるだけ容易にするため、ほとんどの設定オプションを新しいバージョンに対応するオプションに変換する互換性レイヤーを開発しました。この互換性レイヤーは一時的なものであり、将来的に削除されます。このガイドをよくお読みになり、人間の監督下で設定の移行を行うことをお勧めします。

## バージョン 3.xx への移行

新しいキュレートされたビューを体験するには、メディアエージェントのバージョン 3.xx に移行する必要があります。

### 二重イベントタイプの合併症

移行後、エンドユーザーの一部がアプリのアップグレードを選択しなかったり、アップグレードを選択したユーザーが非常に少なかったりする可能性があります。 この中間期間中に、複雑な状況になる可能性のある 2 つのイベント タイプが発生する可能性があります。 これを管理するには:

* すべてのクエリとアクションのマッピングを収集します。
* この暫定期間中に、複数のイベントのデータとその関連性が表示される可能性があることをエンドユーザーに通知します。

<Callout variant="important">
  これはモバイルエージェントに適用されます。 ブラウザの場合は自動的にアップグレードされます。
</Callout>

### 移行後のデータの表示

移行後、過去 1 か月のデータを確認したい場合は、カスタムダッシュボードまたは書き込みを使用することをお勧めします。

```sql
SELECT * FROM pageAction
```

<Callout variant="important">
  名前付きイベント定義を使用すると、古いデータを照会するためのダッシュボード機能が失われます。
</Callout>

### イベント定義マッピング

古いバージョンと新しいバージョンの間のイベントの種類は、実装に基づいて顧客固有です。 スムーズな移行とデータ整合性の維持を確実にするため、 New Relicカスタマーサポートにご連絡いただき、カスタマイズされた移行プランをご依頼ください。

このガイドに注意深く従い、すべての設定が監視下で移行されることを確認することをお勧めします。 互換レイヤーは一時的な解決策であり、できるだけ早く新しい設定に移行することが重要です。
