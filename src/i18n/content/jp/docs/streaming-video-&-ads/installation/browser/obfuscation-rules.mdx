難読化ルールを使用すると、エージェントがNew Relicに送信する前に機密データをマスクできます。各ルールは、正規表現パターンと置換文字列のペアです。ルールは、送信前のシリアライズされたJSONペイロードに適用されます。

## 構成 [#configuration]

エージェント設定で`obfuscate`配列を渡します：

```js
config: {
  obfuscate: [
    { regex: /\/user\/[^\/?"]+/, replacement: '/user/[REDACTED]' },
    { regex: /token=[^&"]+/, replacement: 'token=[MASKED]' },
  ]
}
```

## ルールの順序 [#rule-ordering]

ルールは順番に適用されます — 1つのルールの出力が次のルールに渡されます。無効な正規表現パターンはスキップされ、`Log.warn`を介して警告がログに記録されます。
