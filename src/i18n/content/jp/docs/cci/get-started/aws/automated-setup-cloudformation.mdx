自動化されたAWS CloudFormationテンプレートを使用して、<DNT>**Cloud Cost Intelligence**</DNT>のセットアップを迅速化します。事前設定されたスタックは、S3バケットの設定、CURエクスポート設定、[Trusted Advisor設定のチェック](/docs/cci/financial-intelligence/cci-recommendations/#aws-trusted-advisor)、および必要なIAMロールとポリシーを含む、必要なAWS環境を自動的にプロビジョニングします。デプロイする前に、AWS CloudFormationコンソールですべてのリソース権限を確認してください。

## コスト面での配慮

テンプレートは、CUR データの取り込み用に S3 バケットを自動的に作成します。デフォルトのリージョンは`us-east-2`ですが、インフラストラクチャのニーズに合わせて CloudFormation を使用してこれを変更できます。

## セットアップ手順

<Steps>
  <Step>
    ### New RelicでCloud Cost Intelligenceにアクセス

    <DNT>[**one.newrelic.com**](https://one.newrelic.com/)</DNT> <DNT>**Integrations &amp; Agents**</DNT> <DNT>**Cloud Cost Intelligence - AWS**</DNT>に移動します。
  </Step>

  <Step>
    ### CloudFormation自動化を選択する

    <DNT>**Automate AWS with CloudFormation (Recommended)**</DNT>を選択し、 <DNT>**Continue**</DNT>をクリックします。
  </Step>

  <Step>
    ### AWSの CloudFormation リリース

    1. 接続名を確認し、New Relic データ領域を選択します。
    2. <DNT>**Launch your CloudFormation in AWS**</DNT>をクリックします。
    3. プロンプトが表示されたら、<DNT>**Admin role**</DNT>でAWSアカウントにログインします。AWS コンソールで<DNT>**CloudFormation Stacks**</DNT>画面が開きます。
  </Step>

  <Step>
    ### スタック問題を構成する

    <DNT>**Create Stack**</DNT>フォームで、以下のパラメーターを構成します：

    <DNT>**Pre-populated parameters (do not modify):**</DNT>

    * <DNT>**New Relic Account ID**</DNT>: 事前入力済み。変更しないでください。
    * <DNT>**New Relic Data Region**</DNT>: 事前入力済み。変更しないでください。
    * <DNT>**External ID**</DNT>: 事前入力済み。変更しないでください。

    <DNT>**Required parameters:**</DNT>

    * <DNT>**NewRelicAPIKey**</DNT>: New Relic APIキーを入力します。New Relic UI で、\[ユーザーメニュー] &gt; \[API Keys] から生成してください。
    * <DNT>**Stack Name**</DNT>：デフォルト値が提供されています。必要に応じて編集しますが、名前がすべてのAWS CloudFormationスタックでグローバルに一意であることを確認してください。デフォルト値をそのまま使用することをお勧めします。
    * <DNT>**Bucket Name**</DNT>：デフォルト値が提供されています。必要に応じて編集しますが、名前はすべての S3 バケットでグローバルに一意である必要があります。
    * <DNT>**Connection Name**</DNT>: デフォルト値が提供されています。命名規則に合わせて、必要に応じて編集してください。

    <DNT>**Optional parameters:**</DNT>

    * <DNT>**Grant Access to Trusted Advisor**</DNT>: デフォルトは<DNT>**Yes**</DNT>です。有効にすると、アカウントに対して<DNT>**Trusted Advisor**</DNT>コスト インサイトが有効になっているかどうかを確認し、推奨事項を取得できるようになります。<DNT>**No**</DNT>を選択した場合、または<DNT>**Trusted Advisor**</DNT>が有効になっていない場合、<DNT>**Trusted Advisor**</DNT>の推奨事項は利用できません。

    <Callout variant="important">
      <DNT>**CCI**</DNT> ネイティブレコメンデーションは<DNT>**Trusted Advisor**</DNT>とは独立しており、この設定にかかわらず機能します。
    </Callout>

    他のすべてはオプションであり、必要に応じて構成できます。
  </Step>

  <Step>
    ### スタックを作成する

    <DNT>**I acknowledge that AWS CloudFormation might create IAM resources**</DNT>を選択し、 <DNT>**Create stack**</DNT>をクリックします。
  </Step>

  <Step>
    ### スタックの作成を確認する

    スタックの作成が正常に完了すると、ステータスに<DNT>**CREATE\_COMPLETE**</DNT>と表示されます。
  </Step>

  <Step>
    ### CURレポート生成の検証

    24 時間待ってから、AWS バケットに parquet 形式の CUR レポートが表示されることを確認します。

    <Callout variant="important">
      以下のパスは例です。実際のパスには、異なるタイムスタンプ、UUID、および請求期間が含まれます。

      <DNT>**Amazon S3**</DNT> &gt; <DNT>**Buckets**</DNT> &gt; <DNT>**bucket\_name**</DNT> &gt; <DNT>**cost-and-usage-reports/**</DNT> &gt; <DNT>**bucket\_name-cci/data**</DNT> &gt; <DNT>**BILLING\_PERIOD=2025-12/**</DNT> &gt; <DNT>**2025-12-24T03:05:19.886Z-a2e23b2a-588a-4305-97fd-be712f555b75/**</DNT> &gt; <DNT>**bucket\_name-cci-00001.snappy.parquet**</DNT>
    </Callout>

    統合のセットアップが完了しました。
  </Step>
</Steps>

## 制限

自動化された<DNT>**CloudFormation**</DNT>テンプレートは以下を行いません：

* 計算またはネットワーク リソースを作成または構成します。
* 追加のAWSサービスを有効にするか、サポート レベルを更新します。
* ログ、トレース、その他のメトリクスを取り込むプロセスを自動化します。
* 書き込みアクセス ロールまたはポリシーを作成します。

## 構成されたポリシーと権限

自動化スタックは、次のアクセス レベルを構成します。

* <DNT>**Billing Data Access**</DNT>: コストと使用状況レポートを保存するために作成された特定の S3 バケットに対する読み取り専用許可。
* <DNT>**Service Pricing Access**</DNT>：使用量と公開料金を関連付けるためのAWS Pricing APIへの読み取り専用アクセス。
* <DNT>**Configuration Automation**</DNT>: セットアップフェーズ中に CUR エクスポートを生成し、インテグレーションのステータスを検証するには、制限付き権限が必要です。
* <DNT>**Optional Optimization Access**</DNT>: （選択した場合）コスト削減の推奨事項を提示するための AWS Trusted Advisor への読み取り専用アクセス。
