統合請求、コストと使用状況レポート（CUR）、Identity and Access Management（IAM）ロールとポリシー、および必要なAWSリージョンを設定して、<DNT>**Cloud Cost Intelligence**</DNT>用のAWS環境をセットアップします。

## 前提条件

いずれかのセットアップ方法を開始する前に、次のものを用意してください。

### すべてのセットアップに必要

* <DNT>**AWS account with consolidated billing**</DNT>: [AWS 組織の統合請求の設定](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html)。これは、組織内のすべてのアカウントからコストデータを集計するために必要であり、これによりAWSインフラストラクチャ全体の包括的なコスト分析が可能になります。
* <DNT>**IAM Privileges**</DNT>: リソースをデプロイし、S3バケット、IAMポリシー、およびIAMロールを作成するには、ターゲットAWSアカウントへの管理者アクセス権が必要です。

### 自動セットアップのみに必要

* <DNT>**New Relic API Key**</DNT>: スタック作成用の有効なAPIキーを生成します。<DNT>**CCI**</DNT>インテグレーション用の専用キーを作成することをお勧めします。これをNew Relic UI：[ユーザーメニュー &gt; API Keys](https://onenr.io/07wklp68lRL)で生成します。

## セットアップ方法を選択してください

<DNT>**Cloud Cost Intelligence**</DNT> AWSインテグレーションをセットアップする方法は2種類あります：

* <DNT>[**Automate AWS with CloudFormation (Recommended)**](/docs/cci/get-started/aws/automated-setup-cloudformation)</DNT>: 必要なすべてのAWSリソースを自動的にプロビジョニングする、事前設定されたCloudFormationスタックをすばやくデプロイします。最小限の手動設定で、より迅速なオンボーディングに最適です。
* <DNT>[**Manually integrate your AWS account**](/docs/cci/get-started/aws/manual-integrate-aws-account)</DNT>: セットアッププロセスを完全に制御するために、各AWSリソースを手動で設定します。特定のコンプライアンスまたはカスタマイズの要件を持つ組織に最適です。
