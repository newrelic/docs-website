統合請求、コストと使用状況レポート (CUR)、Identity and Access Management (IAM) のロールとポリシー、および必要な AWS リージョンを構成して、Cloud Cost Intelligence 用の AWS 環境をセットアップします。

## 前提条件

いずれかのセットアップ方法を開始する前に、次のものを用意してください。

### すべてのセットアップに必要

* **統合請求を使用するAWSアカウント**: [AWS組織の統合請求を設定します](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/consolidated-billing.html)。 これは、組織内のすべてのアカウントからのコストデータを集約し、AWS インフラストラクチャ全体にわたる包括的なコスト分析を可能にするために必要です。
* **IAM 権限**: リソースのデプロイ、S3 バケット、IAM ポリシー、および IAM ロールの作成には、ターゲットAWSアカウントへの管理アクセスが必要です。

### 自動セットアップのみに必要

* **New Relic APIキー**: スタック作成用の有効なAPIキーを生成します。 CCI インテグレーション用の専用キーを作成することをお勧めします。 これをNew Relic UI :[ユーザー メニュー &gt; API Keys](https://onenr.io/07wklp68lRL)で生成します。

## セットアップ方法を選択してください

Cloud Cost Intelligence AWSインテグレーションをセットアップする 2 つの方法を提供します。

* [**CloudFormation を使用した AWS の自動化 (推奨)**](/docs/cci/get-started/aws/automated-setup-cloudformation) : 必要なすべての AWS リソースを自動的にプロビジョニングする、事前設定された CloudFormation スタックをすばやくデプロイします。最小限の手動設定でオンボーディングを高速化するのに最適です。
* [**AWS アカウントを手動で統合する**](/docs/cci/get-started/aws/manual-integrate-aws-account): セットアッププロセスを完全に制御できるように、各 AWS リソースを手動で構成します。特定のコンプライアンスまたはカスタマイズ要件を持つ組織に最適です。
