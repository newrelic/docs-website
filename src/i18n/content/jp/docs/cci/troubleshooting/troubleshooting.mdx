セットアップ中に問題が発生した場合は、以下の解決策を見つけてください。

## レポートからデータが欠落している

コスト使用状況レポート (CUR)、リアルタイムのコスト見積もり、または Kubernetes コスト割り当てデータが表示されない場合は、次の手順に従ってください。

* **理由:**前提条件が不完全

* **解決：**

  1. すべてのセットアップの前提条件が完了していることを確認します
  2. 設定を確認する
  3. 問題が解決しない場合は、 [Cloud Cost Intelligence サポート](mailto:cci-support@newrelic.com)にお問い合わせください。

## セットアップ中に接続が失敗する

* **理由:**ネットワーク接続が不安定
* **解決策:**安定したインターネット接続を確保し、セットアップを再試行します。問題が解決しない場合は、ネットワーク アドミニストレーターにファイアウォールの制限を確認してください。

## 権限が不十分です

* **理由:**ユーザーに必要な権限がありません
* **解決策:** AWSおよびNew Relicでアドミニストレーター権限または必要な IAM ポリシーがあることを確認します。 必要に応じてアドミニストレータに連絡してください。

## ARN設定エラー

* **メッセージ:** 「入力した ARN では AWS アカウントへの正しいアクセスが許可されません。ARN を確認してもう一度お試しください。」
* **理由:** ARN 設定が正しくありません
* **解決策:** ARN が必要な設定と一致していることを確認します。 エラーを修正してセットアップを再試行してください。

## 外部IDによるセキュリティリスク

* **メッセージ:** 「セキュリティリスク: 入力した ARN には広範な権限があります。任意の「外部 ID」を使用してロールを引き受けることができます。AWS ロールの作成時に、提供された外部 ID「%s」を externalId として追加してください。
* **理由:**外部IDポリシーが過度に寛容である
* **解決策:**提供された外部 ID を externalId として使用するように AWS ロール設定を更新します。

## S3バケットが見つかりません

* **理由:** S3 バケットが存在しないか、バケット名が正しくありません
* **解決策:**バケット名が AWS アカウントと一致していることを確認します。S3 パス プレフィックスが`cost-and-usage-reports`であることを確認します。スペルを確認し、必要に応じてバケットを作成するか、名前を修正します。

## 接続名が重複しています

* **メッセージ:** 「%s Cloud Cost Intelligence AWS 接続名は既に存在します。新しい接続名を入力してください。&quot;
* **理由:**接続名はすでに使用されています
* **解決策:**一意の接続名を選択して再試行してください。

## 重複した支払者アカウントの設定

* **メッセージ:** 「%s AWS 支払者アカウントは、この New Relic アカウントにすでにリンクされています。」
* **理由:** AWS 支払者アカウントがすでに設定されている
* **解決策:**支払人アカウントがまだリンクされていないことを確認します。必要に応じて別の支払人アカウントを使用してください。

## 追加サポート

問題が解決しない場合は、 [Cloud Cost Intelligence サポートに](mailto:cci-support@newrelic.com)お問い合わせください。

## CloudFormation setup

If the **Create CFT Stack** fails:

1. Check for the Status row in the Events tab/column of the selected stack.
2. Look for **CREATE\_FAILED** and share the Status reason message with New Relic.
3. Go to **CloudWatch** and under **CloudWatch logs**, select your cloud formation stack name and click **View in Logs console**.
4. Under **Log Streams**, open the log and share the log with New Relic support.
