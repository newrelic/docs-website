統合の健全性を監視し、ステータスベースのアクションを使用してAWS 、 Azure 、および GCP にわたるcloudプロバイダーの接続を管理します。

<Callout variant="important">
  少なくとも 1 つのcloudプロバイダーの統合が正常に完了すると、Integrations \[統合] タブにアクセスできるようになります。
</Callout>

## アクセス統合

1. <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT>に移動 &gt; <DNT>**All Capabilities**</DNT> &gt; <DNT>**Cloud Cost Intelligence**</DNT>を選択。
2. すべてのcloudコスト統合を表示するには、 **Integrations** \[統合]タブを選択します。
3. **+ Add new integrations** \[+ 新しい統合の追加]を選択して、 cloudプロバイダー アカウントに新しい統合を追加します。
4. **Status** \[ステータス]列を確認して、 cloudプロバイダー全体の統合の健全性を評価します。

</Callout>

## 統合の管理

統合の横にある省略記号アイコンをクリックして、利用可能なアクションにアクセスします。 表示されるアクションは、現在の統合ステータスによって異なります。

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        アクション
      </th>

      <th style={{ width: "150px" }}>
        利用可能
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Retry**</DNT>
      </td>

      <td>
        <DNT>**Failed**</DNT> 統合
      </td>

      <td>
        接続セットアッププロセスを再試行します。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> 統合
      </td>

      <td>
        設定を保持したままデータ収集を一時停止します。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Activate**</DNT>
      </td>

      <td>
        <DNT>**Paused**</DNT> 統合

      <td>
        データ収集を再開します。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**Failed**</DNT> 統合 (アクティベーション前)
      </td>

      <td>
        統合を永久に削除し、新たな接続セットアップを可能にします。
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  統合設定を維持するため、一時停止には<DNT>**Delete**</DNT>ではなく<DNT>**Pause**</DNT>使用してください。
</Callout>
