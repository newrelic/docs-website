統合の健全性を監視し、ステータスベースのアクションを使用してAWS 、 Azure 、および GCP にわたるcloudプロバイダーの接続を管理します。

<Callout variant="important">
  少なくとも 1 つのcloudプロバイダーの統合が正常に完了すると、Integrations \[統合] タブにアクセスできるようになります。
</Callout>

## アクセス統合

1. <DNT>[**one.newrelic.com**](https://one.newrelic.com)</DNT>に移動 &gt; <DNT>**All Capabilities**</DNT> &gt; <DNT>**Cloud Cost Intelligence**</DNT>を選択。
2. すべてのcloudコストインテグレーションを表示するには、<DNT>**Integrations**</DNT>タブを選択します。
3. <DNT>**+ Add new integrations**</DNT>を選択して、cloudプロバイダーアカウントに新しいインテグレーションを追加します。
4. <DNT>**Status**</DNT>列を確認して、cloudプロバイダー全体のインテグレーションの健全性を評価します。

</Callout>

## 統合の管理

統合の横にある省略記号アイコンをクリックして、利用可能なアクションにアクセスします。 表示されるアクションは、現在の統合ステータスによって異なります。

<table>
  <thead>
    <tr>
      <th style={{ width: "120px" }}>
        アクション
      </th>

      <th style={{ width: "150px" }}>
        利用可能
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        <DNT>**Resume**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> インテグレーション（インテグレーションは正常に作成されたが、CURレポートの検証に失敗した場合のみ）および<DNT>**PAUSED**</DNT>インテグレーション
      </td>

      <td>
        データ収集を再開するか、接続セットアッププロセスを再試行します。バリデーションエラーによりインテグレーション自体が作成されなかった<DNT>**FAILED**</DNT>インテグレーションでは、再開オプションは利用できません。<DNT>**PAUSED**</DNT>インテグレーションの場合、再開するとアクティブなデータ収集が復元されます。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Pause**</DNT>
      </td>

      <td>
        <DNT>**RUNNING**</DNT> 統合
      </td>

      <td>
        設定を保持したままデータ収集を一時停止します。
      </td>
    </tr>

    <tr>
      <td>
        <DNT>**Delete**</DNT>
      </td>

      <td>
        <DNT>**FAILED**</DNT> インテグレーション（<DNT>**RUNNING**</DNT>ステータスに達する前）
      </td>

      <td>
        統合を永久に削除し、新たな接続セットアップを可能にします。
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  インテグレーション設定を保持するため、一時的な停止には<DNT>**Delete**</DNT>ではなく<DNT>**Pause**</DNT>を使用してください。<DNT>**PAUSED**</DNT>インテグレーションを復元するか、接続は作成されたがCURレポートの検証に失敗した<DNT>**FAILED**</DNT>インテグレーションを再試行するには、<DNT>**Resume**</DNT>を使用します。
</Callout>
