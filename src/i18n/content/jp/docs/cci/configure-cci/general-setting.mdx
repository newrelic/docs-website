Cloud Cost Intelligenceデフォルトのタグに加えてカスタム タグもサポートし、リソース コストの割り当てのためにコストと使用状況レポート (CUR) に統合されています。 これにより、タグをフィルタリングおよびグループ化して、詳細な経費分析を行うことができます。Kubernetes 環境では、クラスターとリソースにラベルを適用できます。タグ選択用のドロップダウンは CUR データと`K8SContainerSample`イベントから取得され、管理と分析が効率化されます。

## 一般設定を構成する

<Steps>
  <Step>
    ## 一般設定に移動します

    1. <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT>に移動します。
    2. 左側のナビゲーション メニューから、 **General** \[全般]タブを選択します。
  </Step>

  <Step>
    ## タグを設定する

    コストと使用状況ファイルから取得した値から必要なリソースタグと Kubernetes ラベルを選択し、管理と分析を効率化します。

    1. サービス全体のcloudコストを分類して内訳するには、 **Choose Resource tags for cost allocation** \[コスト割り当てのリソース タグを選択]リストからリソース タグを選択します。
    2. サービス全体のcloudコストを分類して内訳するには、 **Choose Kubernetes tags for cost mapping** \[コストのKubernetesタグを選択]リストからKubernetesラベルを選択します。
  </Step>

  <Step>
    ## テレメトリーデータへのアクセスを管理する

    1. コスト分析のためにテレメトリーデータを取得するアカウントに関する情報を要求するには、 **「テレメトリーデータ」**タブを選択します。
    2. コスト分析用にテレメトリー データを表示および管理するには、 **Select accounts for telemetry data** \[テレメトリー データのアカウントを選択]リストから 1 つ以上のアカウントを選択します。

    <Callout variant="important">
      * コストの見積もりはテレメトリーデータを通じてサポートされます。
      * K8s 割り当てはテレメトリーデータを通じてサポートされます。
      * 既存のテレメトリー パイプラインを再利用して使用率メトリクスを取得します。CUR ファイルの取り込みを除き、新しいデータは取り込まれません。
    </Callout>
  </Step>
</Steps>
