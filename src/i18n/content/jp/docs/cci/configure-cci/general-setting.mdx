Cloud Cost Intelligenceリソース コストの割り当てのために、コストおよび使用状況レポート (CUR) のデフォルト タグと並んでカスタム タグをサポートします。 詳細な経費分析のためにタグをフィルタリングおよびグループ化します。Kubernetes 環境では、クラスターとリソースにラベルを適用します。タグ選択ドロップダウンは、CUR データと`K8SContainerSample`イベントから取得されます。

## 設定を構成する

<Steps>
  <Step>
    ### 一般設定に移動

    <DNT>[**one.newrelic.com**](https://one.newrelic.com) &gt; **All Capabilities** &gt; **Cloud Cost Intelligence**</DNT> &gt; **General**タブに移動します。
  </Step>

  <Step>
    ### タグを設定する

    コストと使用状況ファイルからリソース タグと Kubernetes ラベルを選択します。

    1. <DNT>**Choose Resource tags for cost allocation**</DNT>リストから、サービス全体のcloudコストを分類するリソース タグを選択します。
    2. <DNT>**Choose Kubernetes tags for cost mapping**</DNT>リストから、コストを分類する Kubernetes ラベルを選択します。
  </Step>

  <Step>
    ### テレメトリーデータへのアクセスを管理する

    1. <DNT>**Telemetry Data**</DNT>タブを選択します。
    2. <DNT>**Select accounts for telemetry data**</DNT>リストから、テレメトリーデータのアカウントを選択します。

    <Callout variant="important">
      コスト見積もりと K8s 割り当ては、テレメトリー データを通じてサポートされます。 メトリクスを利用するために既存のテレメトリー パイプラインを再利用します。 CUR ファイルの取り込みを除き、新しいデータは取り込まれません。
    </Callout>
  </Step>
</Steps>
