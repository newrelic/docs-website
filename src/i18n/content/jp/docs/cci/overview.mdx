<DNT>Cloud Cost Intelligence</DNT> すべてのcloudプロバイダー全体でcloudコストの可視性と管理を提供します。コストの可視性とインフラストラクチャのオブザーバビリティの間のギャップを埋め、cloud支出の全体像を提供します。

企業がクラウドサービスを急速に拡大するにつれて、cloudの費用の管理と追跡はますます複雑になります。<DNT>Cloud Cost Intelligence</DNT>はコスト配分に関するインサイトを提供し、潜在的なコスト削減の機会を特定できるようにします。cloud支出を実際の使用量と一致させることで、戦略的な成長イニシアチブをサポートしながら予算を最適化し、投資したすべての資金が会社の全体的な成功に確実に貢献するようにすることができます。

## 主な特徴

### インテリジェントな要約

[Intelligent summary](/docs/cci/financial-intelligence/intelligent-summary)ページは、<DNT>Cloud Cost Intelligence</DNT>の主要なランディングページです。すべてのcloud支出を1か所に表示し、組織内でコストがどのように移動するかを、サービス、チーム、リージョン別に分類して示します。ニーズに応じて、ページを移動し、ビューを変更できます。

主な機能は次のとおりです。

* <DNT>**Sankey chart:**</DNT> 動的なSankeyチャートのビジュアライゼーションを操作して、コストの詳細を掘り下げる
* <DNT>**Cost segregation:**</DNT> サービス、チーム、リージョン別に分離されたコスト
* <DNT>**Dynamic views:**</DNT> さまざまなビュータイプ（たとえば、アカウントビュー）を切り替える
* <DNT>**Breakdown by category:**</DNT> アプリケーション、サービス、所有チーム、リージョン別にセグメント化されたクラウドコスト
* <DNT>**Trend analysis:**</DNT> 前月比のコストトレンド、平均コスト、上位のコスト要因

### コストの概要

「[コストの概要」](/docs/cci/financial-intelligence/cost-overview)ページでは、サービスレベルから個々のリソースに至るまで、複数の次元にわたってコストを調べるためのドリルダウン機能を備えた詳細なコスト分析が提供されます。

### Kubernetesのコスト配分

New Relicテレメトリーデータを使用して、 Kubernetesコストをアプリケーション名別に表示します。

### FinOps エージェント

FinOpsエージェントは、コスト重視の会話型インテリジェンスをCCIに追加し、[Ask AI](/docs/agentic-ai/new-relic-ai)チャットインターフェースで利用できます。以下の目的で使用します：

* コストデータとトレンドについて質問する
* AIが生成した週次および直近1か月（MTD）の支出サマリーを取得する
* コストの異常検出

<Callout variant="important">
  FinOps エージェントは現在プレビュー版です。これを有効にするには、[現在プレビュー中の機能にアクセスするにはどうすればよいですか？](/docs/cci/faq#preview-access)を参照してください。
</Callout>

### クロスアカウントデータ収集

<DNT>Cloud Cost Intelligence</DNT> アクセス権のあるすべてのNew Relicアカウントからテレメトリーデータを収集できます。<DNT>Cloud Cost Intelligence</DNT>に 1 つのアカウントを使用している場合でも、アクセスできる他のすべてのアカウントのデータも、そのアカウントで収集およびインサイトされる可能性があります。アカウントの権限が社内のデータ アクセス ポリシーと一致していることを確認し、[Settings](/docs/cci/configure-cci/general-setting) \[設定]で構成します。

## 使い方

### データ収集 [#fetch-costs]

セットアップが完了すると、<DNT>Cloud Cost Intelligence</DNT>は指定されたオブジェクトストレージコンテナからコストと使用量のデータに自動的にアクセスします。当社のサービスがレポートを取得できるのは、お客様のストレージコンテナにエクスポートされた後のみです。セットアップ中にフォルダーパスプレフィックスが正しく構成されていることを確認します。

ご利用のcloudプロバイダーは通常、1日に数回請求データを更新します。初期セットアップ後、<DNT>Cloud Cost Intelligence</DNT>はストレージロケーションから最新のレポートを1時間ごとに取得します。プロバイダーが更新を公開した直後に、最新のコストデータが当社のプラットフォームに表示されます。

* **課金ファイルの取り込み：**CCIの料金は、cloudプロバイダー（AWS、Azure、またはGCP）がストレージコンテナに公開する課金エクスポートファイルのサイズに基づいています。これらのファイルは、コスト分析とインサイトを作成するために<DNT>Cloud Cost Intelligence</DNT>にインポートされます。詳細については、[Cloud Cost Intelligenceの価格](/docs/cci/cci-pricing)を参照してください。

* **データプライバシー**: <DNT>Cost and Usage Reports</DNT>にはクラウドサービスの使用状況とそのコストに関するデータが含まれていますが、これには個人データは含まれないはずです。<DNT>Cost and Usage Reports</DNT>をストレージに追加する前、およびNew Relicにアップロードする前に、送信するデータに個人を特定できる情報（PII）や機密情報が含まれていないことを確認してください。データの削除は現在サポートされていません。PIIや機密情報を含む<DNT>Cost and Usage Reports</DNT>を生成しないことを推奨します。

* **データ保持期間：**データは最大3年間保持されます。2年後、データは取得時間がより長い、より低コストのストレージに移動されます。

* **表示アクセス:** インストレーション中に選択したNew Relicアカウントにアクセスできるユーザーは誰でも、<DNT>Cloud Cost Intelligence</DNT>内のすべてのデータを表示できます。RBACおよびアクセスポリシーの要件を満たす、適切なアクセス権限を持つアカウントを選択します。

### データ分析とレポート

Cost and Usage Report \[コストと使用状況レポート] のデータを取得した後、<DNT>Cloud Cost Intelligence</DNT>はこの情報を処理および分析し、ダッシュボードで実用的なインサイトを提供します。cloud使用量の財務上の影響を可視化し、コスト要因と最適化の可能性がある領域を特定します。
