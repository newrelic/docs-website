## 概要

New Relic の .NET エージェントのセキュリティアップデートでは、トランザクショントレース中に外部の HTTP リクエストから意図せずにクエリパラメータを取得してしまう脆弱性が修正されています。

**発売日：** 2017年12月18日

**脆弱性の識別子:** NR17-06

**Priority:** Medium

## 脆弱性情報 [#vuln-info]

New Relic の .NET エージェントは、トランザクション トレースをキャプチャして、関数コール、データベース コール、外部コールを含む 1 つのトランザクションに関する詳細な情報を取得します。デフォルトでは、エージェントは外部呼び出し（HTTPリクエスト）の詳細を収集しません。以前のバージョンのエージェントでは、機密情報を含む可能性のある外部のHTTPリクエストパラメータが収集されます。このリリースでは、.NETエージェントのデフォルトの動作が修正され、外部のHTTPリクエストのリクエストパラメータを収集しないようになりました。

### 緩和要因 [#factors]

* [HttpClient](https://msdn.microsoft.com/en-us/library/system.net.http.httpclient.aspx), [WebRequest](https://msdn.microsoft.com/en-us/library/system.net.webrequest.aspx), またはこれらのコールを使用する他のライブラリを使用して行われる外部 HTTP リクエストで使用されるクエリパラメータ。
* トランザクショントレース有効（初期設定）

## 回避策

.NETエージェントの設定で、 [トランザクショントレース](/docs/agents/net-agent/configuration/net-agent-configuration#transaction_tracer) を無効にします。

## セキュリティの脆弱性をNew Relicに報告 [#report]

ニューレリックは、お客様とそのデータのセキュリティを重視しています。私たちの製品やウェブサイトにセキュリティ上の脆弱性を発見したと思われる場合は、New Relic の協調的な情報開示プログラムに報告していただくことを歓迎し、大変感謝しています。詳細については、 [Reporting security vulnerabilities](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities) をご覧ください。
