## 概要

エージェントが高セキュリティ モードまたは構成可能なセキュリティ ポリシーで構成されている場合でも、リクエスト パラメータはセグメント属性として追加されます。これにより、予期しないデータが収集される可能性があります。

**発売日：** 2019年4月1日

**脆弱性の識別子:** NR19-02

**Priority:** Medium

## 脆弱性情報 [#vuln-info]

仕様により、リクエスト パラメーターはセグメント属性として追加されます。エージェントが高セキュリティ モードまたは`attributes_include`構成可能なセキュリティ ポリシーで構成されている場合、リクエスト パラメータは無視される必要があります。

### 緩和要因 [#factors]

高セキュリティ モードまたは`attributes_include`構成可能なセキュリティ ポリシーで構成されたエージェントにのみ影響します。

## 回避策

* すべての属性が収集されないようにするには、 `attributed.enabled`を`false`に設定します。
* [最新のNew Relic Node.jsエージェントにアップデートします。](/docs/agents/nodejs-agent/installation-configuration/upgrade-nodejs-agent)

## セキュリティの脆弱性をNew Relicに報告 [#report]

ニューレリックは、お客様とそのデータのセキュリティを重視しています。私たちの製品やウェブサイトにセキュリティ上の脆弱性を発見したと思われる場合は、New Relic の協調的な情報開示プログラムに報告していただくことを歓迎し、大変感謝しています。詳細については、 [Reporting security vulnerabilities](/docs/security/new-relic-security/data-privacy/reporting-security-vulnerabilities) をご覧ください。
