## 概要

Node.js エージェントが`request.uri`属性を除外するように構成されている場合でも、トランザクション トレースで URI をキャプチャします。これは、機密情報が URI に含まれている環境では、特定の顧客にとって問題となる可能性があります。

**発売日：** 2020年8月20日

**脆弱性の識別子:** NR20-02

**Priority:** Medium

## 脆弱性情報 [#vuln-info]

ユーザーが`request.uri`属性を除外するように Node.js エージェントを構成した場合でも、エージェントはトランザクション トレースで URI をキャプチャします。これにより、認証されたアカウント ユーザーは、New Relic One またはクエリを介してトランザクション追跡の詳細を表示できる場所であればどこでも URI を表示できるようになります。これには、**Transaction** \[トランザクション]ページの**Transaction traces** \[トランザクション トレース]セクション、**Transaction trace details** \[トランザクション トレースの詳細]、UI のクエリ ビルダーが含まれます (ただし、これらに限定されません)。

### 緩和要因 [#factors]

これは、 `request.uri`属性を除外するように構成された Node.js エージェントにのみ影響します。

## 回避策

* [最新のNew Relic Node.jsエージェントにアップデートします。](/docs/agents/nodejs-agent/installation-configuration/update-nodejs-agent)

## セキュリティの脆弱性をNew Relicに報告 [#report]
