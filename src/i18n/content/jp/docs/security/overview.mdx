    New Relicは、セキュリティ、データプライバシー、コンプライアンスを確保するために何をしますか？そして、あなたも何ができますか？

    データの機密性、整合性、および可用性を確保するために使用できるリソースを次に示します。たとえば、New Relicは、プライバシーに関する法律や規制に準拠するプロセスを開発しました。これにより[、あなたとあなたの顧客に関する個人情報の保存方法と使用方法を管理](/docs/using-new-relic/new-relic-security/security/new-relic-personal-data-requests)できます。

    接続された世界全体で、情報セキュリティは重要です。ポリシー、クレデンシャル、監査、およびその他のリソースの詳細については、[企業のセキュリティポリシー](/docs/licenses/license-information/referenced-policies/security-policy)および[NewRelicのセキュリティWebサイト](https://newrelic.com/security)を参照してください。ご不明な点がございましたら、NewRelicアカウントの担当者にお問い合わせください。
  </HeroContent>

<LandingPageTileGrid>
  <LandingPageTile
    title="データプライバシー：私たちがしていること。"
    icon="fe-lock"
  >
    New Relicは、包括的なセキュリティプログラムの一環として「プライバシーバイデザイン」の原則に従い、アカウント、エージェント、イベントと属性、および製品とサービスの[データプライバシーを確](/docs/using-new-relic/new-relic-security/security/data-privacy-new-relic)保します。
  </LandingPageTile>

  <LandingPageTile
    title="データプライバシー：あなたができること。"
    icon="fe-eye"
  >
    New Relicに送信するデータの種類をカスタマイズしたり、特定の担当者がデータを表示または更新できるようにするために、[ユーザーアクセスレベル](/docs/accounts/accounts/roles-permissions/users-roles)の設定、 [APIキー](/docs/apis/get-started/intro-apis/types-new-relic-api-keys)の割り当て、[イベント属性](/docs/agents/manage-apm-agents/agent-data/agent-attributes)のカスタマイズ、 [メール設定](/docs/security/new-relic-security/data-privacy/data-privacy-new-relic#account-communications)の調整などを行うことができます。
  </LandingPageTile>

  <LandingPageTile
    title="セキュリティ：私たちがしていること。"
    icon="fe-alert-triangle"
  >
    受け取ったデータ [と従業員](/docs/security/security-privacy/data-privacy/data-privacy-new-relic#default)の両方に対して強力なセキュリティ対策を実施します。製品に脆弱性が見つかった場合は、 [<Icon name="fe-rss"/>](/docs/using-new-relic/new-relic-security/security/security-bulletins)[](/docs/security/new-relic-security/security-bulletins/feed.xml)を使用して、通知を受け取ります。
  </LandingPageTile>

  <LandingPageTile
    title="セキュリティ：あなたができること。"
    icon="fe-check"
  >
    独自のセキュリティ対策を強化するために、 [SAMLシングルサインオン（SSO）プロバイダー](/docs/accounts/accounts/saml-single-sign/saml-service-providers)を使用し、[高セキュリティモード](/docs/agents/manage-apm-agents/configuration/high-security-mode)設定を調整し、 [監査ログ](/docs/insights/use-insights-ui/manage-account-data/query-account-audit-logs-nrauditevent)を確認し、その他の[製品セキュリティオプション](/docs/using-new-relic/new-relic-security/security/data-privacy-new-relic#product-security)を使用できます。
  </LandingPageTile>

  <LandingPageTile
    title="コンプライアンス：私たちが行うこと。"
    icon="fe-info"
  >
    米国とEUの[TierIII、SOC2認定データセンター](https://newrelic.com/security)は、特定の[保存期間](/docs/using-new-relic/welcome-new-relic/getting-started/data-retention-components)のデータの処理と保存への[コンプライアンスを保証し](/docs/using-new-relic/new-relic-security/security/data-privacy-new-relic#legal)ます。私たちは、特定の基準を満たすアカウントに対して[FedRAMP認定モデレート](/docs/security/security-privacy/compliance/data-encryption)です。HIPAAの対象であり、New Relicを使用しているお客様には、 [HIPAAアカウントの有効化](/docs/security/security-privacy/compliance/hipaa-readiness-new-relic)を提供します。また、Amazon Web Services（AWS）でホストされているNew Relicプラットフォームの[HITRUST認定ステータス](/docs/security/security-privacy/compliance/hitrust/)も取得しています。
  </LandingPageTile>

  <LandingPageTile
    title="コンプライアンス：あなたができること。"
    icon="fe-message-circle"

    一般データ保護規則（GDPR）またはカリフォルニア州消費者保護法（CCPA）に関連するリクエストを受け取った場合は、[標準的な手順に従って個人データを照会または削除してください](/docs/using-new-relic/new-relic-security/security/new-relic-personal-data-requests)。[HIPAAアカウント有効化](/docs/security/security-privacy/compliance/hipaa-readiness-new-relic)ソリューションにアクセスしたい場合は、New Relicアカウントマネージャーに詳細をお問い合わせください。
  </LandingPageTile>
