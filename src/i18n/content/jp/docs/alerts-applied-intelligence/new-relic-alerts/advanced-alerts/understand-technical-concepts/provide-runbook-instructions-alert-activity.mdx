アラート [条件](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-alert-conditions) には、 [ランブック](/docs/using-new-relic/welcome-new-relic/get-started/glossary#alert-runbook) 指示のための URL を特定するオプションが含まれています。これにより、状況が条件の [クリティカル・スレッショルド](/docs/alerts/new-relic-alerts-beta/configuring-alert-policies/define-thresholds-trigger-alert) を超えたときに開かれるインシデントを処理するための情報や標準手順にリンクすることができます。

![040815alerts-runbook-url.png](./images/040815alerts-runbook-url_0.png "040815alerts-runbook-url.png")

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com/ "新しいウィンドウにリンクが開きます。")** トップナビで、 **アラート& AI** をクリックし、 **アラートポリシー** 、 **（ポリシーを選択）** 、次に **（条件を選択）**: ランブックのURLを追加して、アラートが発生したインシデントを処理する担当者が何をすべきかを知ることができます。
</figcaption>

条件に関連するランブックのURLを編集するには

1. **[one.newrelic.com](https://one.newrelic.com/ "新しいウィンドウにリンクが開きます。")** トップナビで、 **アラート& AI** をクリックし、 **アラートポリシー** 、 **(select a policy)** 、次に **(select a condition)** をクリックします。
2. **Edit conditions** ページで、 **Add runbook URL** をクリックし、ランブックURLのフルパス（ランブックURL、社内WikiページのURLなど）を入力し、保存します。
3. オプションです。警告条件から既存のランブックURLを変更または削除するには、 **Remove** をクリックします。

**Incident** の詳細やメール通知にランブックのURLを記載し、担当者が何をすべきかを正確に把握できるようにします。
