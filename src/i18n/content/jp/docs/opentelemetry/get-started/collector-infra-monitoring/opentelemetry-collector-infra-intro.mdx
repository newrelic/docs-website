[OpenTelemetryコレクターは、](https://opentelemetry.io/docs/collector/)テレメトリーデータを受信、処理、エクスポートするためのベンダーに依存しないツールです。 コレクターの要件と設定は異なりますが、さまざまな受信機とプロセッサーが付属しているため、インフラストラクチャのモニタリングに人気があります。 データ処理にコレクターを使用することも一般的ですが、このドキュメントではインフラストラクチャモニタリングの使用例に焦点を当てています。 詳細については、 [OpenTelemetryコレクターのデータ処理」](/docs/opentelemetry/get-started/collector-processing/opentelemetry-collector-processing-intro)を参照してください。

## OpenTelemetry Collector for Kubernetes監視 [#k8s-monitoring]

Collector要件と設定はさまざまですが、さまざまなレシーバーとプロセッサが付属しているため、 Kubernetes監視に人気があります。 必要なテレメトリーをすべて取得するには、異なる権限要件を持つ複数のレシーバーを使用してさまざまなKubernetes API をスクレイピングする必要があるため、物事を簡単にするために、内部でコレクター[nrdot-コレクター-k8s](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s)のカスタム ディストリビューションを使用するヘルム チャートを構築しました。 開始方法の詳細については、 [OpenTelemetry for Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/)を参照してください。
