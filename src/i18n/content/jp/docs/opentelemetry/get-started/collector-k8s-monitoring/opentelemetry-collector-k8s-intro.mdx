[OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) 、テレメトリー データを受信、処理、エクスポートするためのベンダーに依存しないツールです。 Collector要件と設定はさまざまですが、さまざまなレシーバーとプロセッサが付属しているため、 Kubernetes監視に人気があります。 必要なテレメトリーをすべて取得するには、異なる権限要件を持つ複数のレシーバーを使用してさまざまなKubernetes API をスクレイピングする必要があるため、物事を簡単にするために、内部でコレクター[nrdot-コレクター-k8s](https://github.com/newrelic/nrdot-collector-releases/tree/main/distributions/nrdot-collector-k8s)のカスタム ディストリビューションを使用するヘルム チャートを構築しました。 開始方法の詳細については、 [OpenTelemetry for Kubernetes](/docs/kubernetes-pixie/kubernetes-integration/installation/k8s-otel/)を参照してください。
