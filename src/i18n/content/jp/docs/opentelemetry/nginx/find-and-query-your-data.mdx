## New RelicでNGINXデータを表示する [#find-data]

セットアップが完了し、データが流れるようになると、 New Relicで NGINX メトリクスとログを表示し、カスタム アラートを作成できます。

**方法 1: 統合とエージェントによる**

1. [one.newrelic.com](https://one.newrelic.com) &gt;**Integrations &amp; Agents**に移動します
2. **Dashboards**をクリック
3. **NGINX OTel overview** \[NGINX OTel概要]ダッシュボードを見つけてクリックします
4. アカウントを選択し、 **View dashboard** \[ダッシュボードを表示]をクリックします

**方法 2: インフラストラクチャモニタリングによる**

1. [one.newrelic.com](https://one.newrelic.com) &gt;**On-host integrations**に移動します
2. デプロイメント名（例: `production-web-01` ）で NGINX サーバーを検索します。
3. サーバーをクリックして詳細なメトリクスを表示します

## NRQLでデータをクエリする

NGINX データを分析するためのカスタム クエリを作成できます。

**クエリの例:**

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## データの可視性のトラブルシューティング

データが表示されない場合:

1. 少し時間がかかります - 初期データの取り込みには数分かかる場合があります
2. 時間範囲を確認する - 最新のデータを参照していることを確認する
3. デプロイメント名を確認する - Collector 構成の正確な名前を使用します
4. NRQL でデータを確認します。
   ```sql
   FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
   ```

データが断続的に表示される場合:

1. Collectorログでエラーを確認します。
   ```bash
   sudo journalctl -u otelcol-contrib -n 50
   ```
2. NGINXがトラフィックを生成していることを確認する - テスト `curl http://localhost`
3. 認証をチェック - ライセンスキーとエンドポイントが正しいことを確認します
