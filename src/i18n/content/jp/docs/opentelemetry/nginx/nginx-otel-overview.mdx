## OpenTelemetryを使用して NGINX を監視する理由 [#why-monitor]

NGINX 監視は、Web アプリケーションのパフォーマンスと信頼性を維持するために不可欠です。 OpenTelemetry は、この重要なデータを収集および分析するための標準化されたアプローチを提供します。

### 主なメリット

* **パフォーマンスの最適化**: ボトルネックを特定し、応答時間を追跡し、データドリブンのスケーリングを決定します。
* **プロアクティブな監視**: ユーザーへの影響が出る前にアラートを受信し、問題を早期に検出します
* **運用上のインサイト**: トラフィック パターンの理解、 SLA準拠の追跡、監査証跡の維持
* **将来性**: オープンソースOpenTelemetry標準、ベンダーニュートラル統合

### 使用事例

電子商取引 Web サイト、 APIゲートウェイ、コンテンツ配信プラットフォームのいずれを実行している場合でも、NGINX 監視は高速で信頼性の高い Web サービスを維持するのに役立ちます。 チェックアウトプロセスが遅くなったときに通知を受け取り、 APIエンドポイントが過剰になったことを検出し、問題が発生している地理的地域を特定します。 この監視は、顧客トランザクションを処理する Web サイト、モバイル アプリのバックエンド、マイクロサービス アーキテクチャー、およびダウンタイムやパフォーマンスの低下がユーザーやビジネスに直接影響を与える Web アプリケーションには不可欠です。

## 利用可能なメトリクス [#metrics]

OpenTelemetry Collector 、接続統計、リクエスト数、サーバーの健全性インジケーターなどの主要な NGINX パフォーマンス メトリクスを自動的に収集します。

**主要なメトリクスが収集したもの:**

* **接続メトリクス**- アクティブ、受け入れ、処理された接続とその状態
* **リクエスト メトリクス**- リクエストのrequestsと時間の経過に伴うリクエスト率
* **サーバーの健全性**- 接続効率とパフォーマンス指標

**メトリクスとプロパティの完全なリファレンスについては、**オンホストとKubernetesデプロイメントの両方の詳細な説明、メトリクスのタイプ、リソースのプロパティについては[、NGINX OpenTelemetryメトリクスとプロパティのリファレンスを](/docs/opentelemetry/nginx/nginx-otel-metrics-reference/)参照してください。
