[OpenTelemetry、 Protocol (OTLP)](https://opentelemetry.io/docs/specs/otlp/) OpenTelemetryを通じてNew Relic とシームレスに統合できる、インストゥルメンテーションとテレメトリーのための汎用性のエコシステムを提供します。このガイドでは、 New Relic OTLP データを処理し、取り込む方法の概要を説明し、システムを効率的に監視および観察できるようにします。

## 始めましょう [#get-started]

OpenTelemetry と New Relic の統合の詳細については、 [「OpenTelemetry と New Relic の使用を開始する」](/docs/opentelemetry/get-started/opentelemetry-get-started-intro/)を参照してください。 このページでは一般的なインテグレーションパターンとベストプラクティスについて説明します。

## New Relic の OTLP を構成する [#configure]

OTLP 設定要件とトラブルシューティングの詳細なガイダンスについては、次のページを参照してください。

* [New Relic OTLP エンドポイントでは、](/docs/opentelemetry/best-practices/opentelemetry-otlp/) OpenTelemetryデータをNew Relicに統合するための重要な設定手順を説明します。
* [New Relic OTLP トラブルシューティングは、](/docs/opentelemetry/best-practices/opentelemetry-otlp-troubleshooting/)ログ、 APIキー、接続の問題の確認など、データ取り込みに関する一般的な問題を解決するのに役立ちます。

## New Relicのプラットフォームを使用する [#platform]

OpenTelemetryデータが[New Relicプラットフォーム](/docs/new-relic-solutions/new-relic-one/introduction-new-relic-platform/)に取り込まれると、次の観察ツールのいくつかを利用できるようになります。

* [アラート](/docs/alerts/overview/): 信号が閾値を超えたときに通知を受け取ります。
* [カスタム クエリ](/docs/query-your-data/explore-query-data/get-started/introduction-querying-new-relic-data/): アドホック データ分析と探索用のカスタム NRQL クエリを記述します。
* [ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards/): アプリケーションの固有の要件を反映したカスタム データ視覚化を作成します。
