## Redisデータを表示する [#find-and-use]

コレクターが実行され、データを送信すると、New RelicでRedisメトリクスを表示できます：

1. に行く <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Integrations &amp; Agents**</DNT>
2. 検索する <DNT>**Redis (OpenTelemetry)**</DNT>
3. **Installation plan** \[インストレーションプラン]で、インストレーション手順を確認したい場合は、**See installation docs** \[インストレーションドキュメントを表示]をクリックします。それ以外の場合は、**Skip this step** \[この手順をスキップ]をクリックします。
4. <DNT>**See your data** </DNT>をクリックします。

Redisのヘルス、パフォーマンスメトリクス、およびリソース使用量を示すダッシュボードが表示されます。Kubernetes上でRedisを監視している場合、ダッシュボードにはKubernetes固有のコンテキストが含まれます。

または

1. に行く <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Integrations &amp; Agents**</DNT>
2. **All** \[すべて]タブで、**dashboards** \[ダッシュボード]をクリックします。Redis OTelを検索すると、ダッシュボードを表示できます。

または

1. 左側のパネルから<DNT>**Dashboards** </DNT>をクリックします。
2. Redis OTelダッシュボードを検索すると、ダッシュボードを表示できます。

<Callout variant="tip">
  **データが表示されませんか？**データが表示されるまで数分かかる場合があります。10分経ってもメトリクスが表示されない場合は、コレクターのログを確認し、設定を検証してください。
</Callout>

## アラートの設定 [#alerts]

アラートによるプロアクティブな監視により、問題がユーザーに影響を与える前にそれを検出できます。New Relic でアラート条件を作成するには:

1. に行く <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Alerts** &gt; **Alert Conditions**</DNT>
2. **Create condition** \[条件を作成]をクリック
3. [**Guided mode**](/docs/alerts/create-alert/create-alert-condition/alert-conditions/) \[ガイドモード]または[**NRQL**](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions/)クエリビルダーのいずれかを使用して、集計を設定します

堅牢なRedis監視には、以下の集計設定が推奨されます：

### アラートのイチ押し [#essential-alerts]

これらのアラートは、データ損失やサービス停止を引き起こす可能性のある重大なRedisのヘルス上の問題をモニターします：

<CollapserGroup>
  <Collapser id="high-memory" title="高メモリ使用率の集計">
    メトリクス`redis.memory.used`（`redis.maxmemory`に対する相対値）は少なくとも5分間90％を超えています。メモリが枯渇すると、Redisは書き込みコマンドの受け付けを停止します。
  </Collapser>

  <Collapser id="rejected-connections" title="拒否された接続の集計">
    メトリクス`redis.connections.rejected`は少なくとも1分間0を超えています。`maxclients`の制限に達すると、Redisは新しい接続を拒否します。
  </Collapser>

  <Collapser id="replicas-dropped" title="接続されたレプリカのドロップ集計">
    メトリクス`redis.slaves.connected`は少なくとも2分間、予想されるベースラインを下回っています。レプリカの障害は読み取り容量を低下させ、データの耐久性を損ないます。
  </Collapser>

  <Collapser id="high-evictions" title="高いエビクション率の集計">
    `redis.keys.evicted`のレートは少なくとも5分間、1秒あたり1000を超えています。エビクション率が高いということは、Redisがメモリの圧迫を受けていることを意味します。
  </Collapser>
</CollapserGroup>

### 追加の監視アラート [#additional-alerts]

これらのアラートは、パフォーマンスと運用上の問題を監視するのに役立ちます。

<CollapserGroup>
  <Collapser id="low-hit-ratio" title="低いキャッシュのヒット率の集計">
    ヒット率（`redis.keyspace.hits`/（`redis.keyspace.hits`+`redis.keyspace.misses`））は少なくとも5分間50％を下回っています。
  </Collapser>

  <Collapser id="high-fragmentation" title="高いフラグメンテーション率の集計">
    メトリクス`redis.memory.fragmentation_ratio`は少なくとも10分間1.5を超えています。過度なメモリのフラグメンテーションはリソースを無駄にします。
  </Collapser>

  <Collapser id="call-latency" title="APMからのRedis呼び出しレイテンシの集計">
    `db.system = redis`の平均スパン期間が少なくとも5分間10 msを超えています。アプリケーションサービスからのRedis呼び出しのレイテンシが高い場合、Redisまたはネットワークの問題を示している可能性があります。
  </Collapser>
</CollapserGroup>
