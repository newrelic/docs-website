このリファレンスガイドは、Redis OpenTelemetryインテグレーションによって収集されるすべてのメトリクスの包括的な概要を提供し、Redisインスタンスの健全性とパフォーマンスを監視するために利用可能なデータを理解するのに役立ちます。

## 完全なメトリクスリファレンス

インテグレーションは、コマンド、メモリ、キースペース、接続、およびレプリケーションにわたる30以上のインスタンスメトリクス — さらにRedis Clusterデプロイメント用の9つのクラスタ固有のメトリクスを収集します。詳細なメトリクスの仕様については、以下のセクションを展開してください。

<Callout variant="tip">
  完全なアップストリームカタログについては、OpenTelemetryの[redisreceiverのドキュメント](https://github.com/open-telemetry/opentelemetry-collector-contrib/blob/main/receiver/redisreceiver/documentation.md)を参照してください。
</Callout>

## クラスターメトリクス

これらのメトリクスは、ClusterモードでRedisを監視している場合にのみ使用できます。これらを使用するには、Redisレシーバーでクラスタメトリクスを有効にする（すべてのクラスタメトリクスはデフォルトで無効になっています）か、`redis_exporter --is-cluster`を使用したPrometheusレシーバーのアプローチを使用する必要があります。

<table>
  <thead>
    <tr>
      <th style={{ width: "260px" }}>

      </th>

      <th>

      <td>
        ゲージ
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.slots_assigned`
      </td>

      <td>

      </td>

      <td>
        ゲージ
      </td>
    </tr>

    <tr>
      <td>

      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.slots_fail`

      <td>

    <tr>
      <td>

      <td>

      </td>

      <td>
        ゲージ
      </td>

    <tr>
      <td>
        `redis.cluster.stats_messages_sent`
      </td>

      <td>
        送信されたクラスタバスメッセージの合計（ping、pong、meet、fail）。安定したトラフィックは、ゴシッププロトコルが正常であることを示します。

      <td>
        累積
      </td>
    </tr>

    <tr>
      <td>
        `redis.cluster.stats_messages_received`
      </td>

      <td>

      </td>

      <td>
        累積
      </td>
    </tr>
  </tbody>
</table>

<Callout variant="tip">
  Redis Cluster内のすべてのノードは同一のクラスタメトリクスを報告します（`CLUSTER INFO`はどのノードからも同じデータを返します）。復元力を高めるためにすべてのノードでクラスタ監視を有効にするか、冗長なデータを最小限に抑えるために単一のノードで有効にすることができます。Prometheusレシーバーのアプローチでは、これらのメトリクスを自動的に収集する`redis_exporter --is-cluster`を使用します。
</Callout>
