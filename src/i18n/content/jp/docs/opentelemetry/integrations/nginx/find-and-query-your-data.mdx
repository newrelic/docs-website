## New RelicでNGINXデータを表示する [#find-data]

セットアップが完了し、データが流れるようになると、 New Relicで NGINX メトリクスとログを表示し、カスタム アラートを作成できます。

<CollapserGroup>
  <Collapser id="method1-integrations" title="統合とエージェントを通じて">
    1. に行く <DNT>[one.newrelic.com](https://one.newrelic.com) &gt; **Integrations &amp; Agents**</DNT>
    2. クリック <DNT>**Dashboards**</DNT>
    3. 検索してクリックする<DNT>**NGINX OTel overview**</DNT>ダッシュボード
    4. <DNT>**Dashboard setup**</DNT>ページが開きます。データソースを既に設定している場合は、 <DNT>**Skip this step**</DNT>をクリックしてください。それ以外の場合は、 <DNT>**See installation docs**</DNT>をクリックして、指示に従ってデータソースを設定してください。セットアップが完了したら、ダッシュボードのセットアップページに戻り、 <DNT>**Skip this step**</DNT>をクリックします。
    5. <DNT>**View dashboard**</DNT>をクリックします。
  </Collapser>

  <Collapser id="method2-infrastructure" title="インフラモニタリングを通じて">
    1. に行く <DNT>[one.newrelic.com](https://one.newrelic.com) &gt; **On-host integrations**</DNT>
    2. デプロイメント名（例: `production-web-01` ）で NGINX サーバーを検索します。
    3. サーバーをクリックして詳細なメトリクスを表示します
  </Collapser>
</CollapserGroup>

## NRQLでデータをクエリする

NGINX データを分析するためのカスタム クエリを作成できます。

クエリの例:

```sql
-- View request rate over time
FROM Metric SELECT rate(sum(nginx.requests), 1 minute)
WHERE nginx.deployment.name = 'production-web-01'
TIMESERIES

-- Check connection states
FROM Metric SELECT latest(nginx.connections_current)
WHERE nginx.deployment.name = 'production-web-01'
FACET state

-- Analyze log patterns (if log forwarding enabled)
FROM Log SELECT count(*)
WHERE nginx.deployment.name = 'production-web-01'
FACET status SINCE 1 hour ago
```

## データの可視性のトラブルシューティング

<CollapserGroup>
  <Collapser id="troubleshoot-no-data" title="データが表示されない場合は">
    * 少し時間がかかります - 初期データの取り込みには数分かかる場合があります
    * 時間範囲を確認する - 最新のデータを参照していることを確認する
    * デプロイメント名を確認する - Collector 構成の正確な名前を使用します
    * NRQL でデータを確認します。
      ```sql
      FROM Metric SELECT * WHERE nginx.deployment.name LIKE '%production%' LIMIT 1
      ```
  </Collapser>

  <Collapser id="troubleshoot-intermittent-data" title="データが断続的に見える場合">
    * Collectorログでエラーを確認します。
      ```bash
      sudo journalctl -u otelcol-contrib -n 50
      ```
    * NGINXがトラフィックを生成していることを確認する - テスト `curl http://localhost`
    * 認証をチェック - ライセンスキーとエンドポイントが正しいことを確認します
  </Collapser>
</CollapserGroup>
