## Elasticsearchデータを表示する [#find-and-use]

コレクターが実行されてデータが送信されると、 New RelicでElasticsearchメトリクスを表示できます。

1. に行く <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Integrations &amp; Agents**</DNT>
2. 検索する <DNT>**Elasticsearch (OpenTelemetry)**</DNT>
3. 導入手順を表示するには、 <DNT>**Installation plan** </DNT>で<DNT>**See installation docs** </DNT>をクリックします。 それ以外の場合は、 <DNT>**Skip this step** </DNT>をクリックしてください。
4. <DNT>**See your data** </DNT>をクリックします。

クラスタの健全性、パフォーマンス メトリクス、およびリソースの使用状況を示すダッシュボードが表示されます。 Kubernetes上でElasticsearchを監視している場合、ダッシュボードにはKubernetes固有のコンテキストが含まれます。

または

1. に行く <DNT>**[one.newrelic.com](https://one.newrelic.com)** &gt; **Integrations &amp; Agents**</DNT>
2. <DNT>**All** </DNT>タブの下にある<DNT>**dashboards** </DNT>をクリックします。Elasticsearch Otelを検索すると、ダッシュボードが表示されます。

または

1. 左側のパネルから<DNT>**Dashboards** </DNT>をクリックします。
2. Elasticsearch Otel ダッシュボードを検索するとダッシュボードが表示されます。

<Callout variant="tip">
  **データが表示されませんか？**データが表示されるまで数分かかる場合があります。10分経ってもメトリクスが表示されない場合は、コレクターのログを確認し、設定を検証してください。
</Callout>

## アラートの設定 [#alerts]

アラートによるプロアクティブな監視により、問題がユーザーに影響を与える前にそれを検出できます。New Relic でアラート条件を作成するには:

1. **[one.newrelic.com](https://one.newrelic.com)** &gt;**Alerts**&gt;**Alert Conditions**にアクセスしてください。
2. **Create condition** \[条件を作成]をクリック
3. [**ガイド付きモード**](/docs/alerts/create-alert/create-alert-condition/alert-conditions)または[**NRQL**](/docs/alerts/create-alert/create-alert-condition/create-nrql-alert-conditions)書き込みビルダーのいずれかを使用して、一括を設定します。

堅牢なElasticsearch監視には、以下のまとめ設定が推奨されます。

### アラートのイチ押し

これらのアラートは、データ損失やサービス停止を引き起こす可能性のある重大なクラスタの健全性問題を監視します。

<CollapserGroup>
  <Collapser id="unassigned-shards" title="未割り当てのシャード数">
    メトリクス`elasticsearch.cluster.shards` (`state = 'unassigned'`) は少なくとも 5 分間 0 を超えています。
  </Collapser>

  <Collapser id="healthy-data-nodes" title="健全なデータノード集計">
    メトリクス`elasticsearch.cluster.data_nodes`は少なくとも 5 分間、必要な最小ノード数を下回っています。
  </Collapser>

  <Collapser id="heap-usage" title="ヒープ使用量が高すぎます">
    ヒープ使用率 (使用済み/最大) が 5 分以上 90% を超えています。
  </Collapser>

  <Collapser id="pending-tasks" title="保留中のタスクまとめ">
    メトリクス`elasticsearch.cluster.pending_tasks`は少なくとも 5 分間は 5 を超えています。
  </Collapser>
</CollapserGroup>

### 追加の監視アラート

これらのアラートは、パフォーマンスと運用上の問題を監視するのに役立ちます。

<CollapserGroup>
  <Collapser id="query-time-slow" title="ゆっくりまとめ">
    `elasticsearch.node.operations.time`の 95 パーセンタイルは、少なくとも 2 分間 5 ミリ秒を超えています。
  </Collapser>

  <Collapser id="initializing-shards" title="シャードの初期化に時間がかかりすぎています">
    メトリクス`elasticsearch.cluster.shards` (`state = 'initializing'`) は少なくとも 5 分間 0 を超えています。
  </Collapser>

  <Collapser id="relocating-shards" title="シャードの移動に時間がかかりすぎます">
    メトリクス`elasticsearch.cluster.shards` (`state = 'relocating'`) は少なくとも 5 分間 0 を超えています。
  </Collapser>
</CollapserGroup>
