New Relicでは、 New Relic Distribution of OpenTelemetry (NRDOT) とNew Relic Relic のデータベース監視機能を使用して、チームが Microsoft SQL Server データベースのパフォーマンスを監視できるようになりました。 この統合により、NRDOT コレクターを使用して、包括的なインサイトをデータベース メトリクス、書き込みパフォーマンス、およびシステムの状態に提供します。

この NRDOT ベースのアプローチは、データベース監視のOpenTelemetry標準を活用することで既存のオンホスト インテグレーション (OHI) を補完し、既存の監視スタックとの統合を容易にします。

## 前提条件 [#prerequisites]

始める前に、次のものがあることを確認してください。

* 有効なNew Relic[ライセンスキー](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)

* サポートされているデータベースのバージョン: SQL Server 2017 以降

* SQL Server データベースのセットアップ:

  * SQL Server インスタンスへの管理アクセス (`sysadmin`ロールまたは同等のロール)
  * オンプレミスの管理インスタンスをサポートする Microsoft SQL Server
  * NRDOT コレクターと SQL Server 間のネットワーク接続
  * SQL Server Management Studio (SSMS) または`sqlcmd`ユーティリティ
  * サーバーのホスト名またはIPアドレス
  * ポート番号（通常は1433）またはカスタムポート
  * Windowsドメインまたは SQL Server 認証
  * SSL設定

* システム要件:

  * NRDOT コレクターを実行するためのWindowsまたは Linux システム
  * New Relicの OTLP エンドポイントへのネットワーク接続

## Microsoft SQL Server 監視用の NRDOT Collector を設定する [#install-collector]

NRDOT コレクターを使用して、Microsoft SQL Server データベースを監視できます。 NRDOT コレクターは、New Relic 固有のコンポーネントを含む事前構成済みのディストリビューションです。

NRDOT Collector をインストールして構成するには、次の手順に従います。

<Steps>
  <Step>
    ### NRDOT Collectorセットアップ [#setup-collector]

    お使いの OS とアーキテクチャーに基づいてコレクターをインストールします。

    <Callout variant="important">
      パフォーマンス監視のためにインフラレベルのメトリクスを表示するには、SQL Server インスタンスと同じホストに NRDOT Collectorインストールすることをお勧めします。
    </Callout>

    <CollapserGroup>
      <Collapser title="Windowsの場合" id="windows">
        1. この URL をブラウザに貼り付けてインストーラをダウンロードして実行し、 New Relicの NRDOT Collector用のインストーラをダウンロードします。

        ```powershell
        https://download.newrelic.com/database-monitoring/nrdot-collector/nrdot-collector-host-windows-amd64-latest.msi
        ```

        2. ダウンロードしたファイルを見つけてダブルクリックし、NRDOT Collector をインストールします。
      </Collapser>

      <Collapser title="Linuxの場合" id="linux">

  <Step>
    ### データベースユーザーを構成する [#configure-user]

    次のスクリプトを`root user/sysadmin`として実行して、 `newrelic`監視ユーザーを作成し、グローバル読み取り権限を付与します。

    <CollapserGroup>
      <Collapser title="Windows の場合 (SSMS)" id="windows-user">
        * `newrelic`監視ユーザーを作成するには、SQL Server Management Studio (SSMS) を使用して次のスクリプトを実行します。

          ```sql
          USE [master];
          GO
          CREATE LOGIN [newrelic] WITH PASSWORD = '<PASSWORD>';
          GO
          -- Instance-level permissions
          GRANT VIEW SERVER STATE TO [newrelic];
          GRANT VIEW ANY DEFINITION TO [newrelic];
          GRANT VIEW ANY DATABASE TO [newrelic];
          GO
          -- Grant read access privileges to all user databases
          DECLARE @name SYSNAME;
          DECLARE db_cursor CURSOR READ_ONLY FORWARD_ONLY FOR
          SELECT [name]
          FROM [master].[sys].[databases]
          WHERE [name] NOT IN ('master', 'msdb', 'tempdb', 'model', 'rdsadmin', 'distribution')
          AND [state] = 0; -- Only online databases
          OPEN db_cursor;
          FETCH NEXT FROM db_cursor INTO @name;
          WHILE @@FETCH_STATUS = 0
          BEGIN
            BEGIN TRY
              PRINT 'Processing database: ' + @name;
              EXEC('USE [' + @name + '];
              IF NOT EXISTS (SELECT 1 FROM sys.database_principals WHERE name = ''newrelic'')
              BEGIN
                CREATE USER [newrelic] FOR LOGIN [newrelic];
              END;
              GRANT VIEW DATABASE STATE TO [newrelic];
              GRANT VIEW DEFINITION TO [newrelic];
              ALTER ROLE db_datareader ADD MEMBER [newrelic];');
              PRINT 'Success: ' + @name;
            END TRY
            BEGIN CATCH
              PRINT 'Error on ' + @name + ': ' + ERROR_MESSAGE();
            END CATCH
            FETCH NEXT FROM db_cursor INTO @name;
          END
          CLOSE db_cursor;
          DEALLOCATE db_cursor;
          GO
          ```
      </Collapser>

        ```sql
          END CATCH
          FETCH NEXT FROM db_cursor INTO @name;
        END
        CLOSE db_cursor;
        DEALLOCATE db_cursor;
        GO
        ```

        2. スクリプトを実行するには、次のコマンドを実行します。`<Your_SA_Password>` SQL Server アドミニストレーターのパスワードに置き換えます。

        ```bash
        sqlcmd -S localhost -U sa -P '<Your_SA_Password>' -C -i nr-grant-permission.sql
        ```

        3. *(オプション)*ユーザーが正常に作成され、正しい権限が付与されていることを確認するには、次のコマンドを実行します。`<Your_SA_Password>` SQL Server アドミニストレーターのパスワードに置き換えます。

        ```bash
          sqlcmd -S localhost -U sa -P '<Your_SA_Password>' -C -Q "SELECT sp.name AS [User], p.permission_name AS [Permission_Granted] FROM sys.server_permissions p JOIN sys.server_principals sp ON p.grantee_principal_id = sp.principal_id WHERE sp.name = 'newrelic';"
        ```

        **期待される出力:**

        ```
        User Permission_Granted
        ---------------------------------------------------------------------------------------------
        <your_username> CONNECT SQL
        <your_username> VIEW ANY DEFINITION
        <your_username> VIEW ANY DATABASE
        <your_username> VIEW SERVER STATE
        ```

        この出力は、必要なサーバー レベルの権限を持つユーザーが正常に作成されたことを確認します。

    </CollapserGroup>

    * **Linux：** `/etc/nrdot-collector-host/config.yaml`

## NRDOT コレクターがサポートするコマンド [#commands]

NRDOT Collectorサービスを管理したり、ログを表示したり、サービスのステータスを確認したりするには、OS に基づいて次のコマンドを使用します。

<CollapserGroup>
  <Collapser id="windows-service-management" title="Windows サービス管理">
    * NRDOT Collector を再起動するには:

      ```shell
      net stop nrdot-collector-host
      net start nrdot-collector-host
      ```

    * NRDOT Collectorステータスを確認するには:

      ```shell
      sc query nrdot-collector-host
      ```

    * NRDOT Collector ログを表示するには:

      1. 検索バーに`Event Viewer`と入力して選択します。
      2. 左側のサイドバーで、「Windows ログ」という名前のフォルダーを展開します。
      3. 「アプリケーション」をクリックして、アプリケーション固有のイベントを中央ペインにロードします。
      4. ソースタイプが次のイベントを検索します = `nrdot-collector-host`

    * NRDOT Collector サービスを削除するには、 `Add/Remove Programs`からアンインストールします。
  </Collapser>

  <Collapser id="linux-service-management" title="Linuxサービス管理">
    * インストールステータスを検証するには:

      * Debian/Ubuntu ベースのディストリビューションの場合:

        ```bash
        dpkg -l | grep nrdot-collector-host
        ```

      * RHEL/CentOS ベースのディストリビューションの場合:

        ```bash
        rpm -q nrdot-collector-host || ls /etc/nrdot-collector-host
        ```

    * NRDOT Collector を再起動するには:

      ```bash
      sudo systemctl restart nrdot-collector-host
      ```

    * NRDOT Collectorステータスを確認するには:

      ```bash
      sudo systemctl status nrdot-collector-host
      ```

    * NRDOT Collector ログを表示するには:

      ```bash
      sudo journalctl -u nrdot-collector-host -f
      ```

    * NRDOT Collector サービスを削除するには:

      * Debian/Ubuntu ベースのディストリビューションの場合:

        ```bash
        sudo apt-get purge -y nrdot-collector-host
        ```

      * RHEL/CentOS ベースのディストリビューションの場合:

        ```bash
        sudo dnf remove -y nrdot-collector-host&& sudo rm -rf /etc/nrdot-collector-host
        ```
  </Collapser>
</CollapserGroup>

## トラブルシューティング [#troubleshooting]

NRDOT Collector でネットワーク接続の問題が発生した場合は、次のコマンドを実行して、ポート 1433 上の SQL Server インスタンスへの接続をテストします。

* Windows の場合は、次を実行します。

  ```shell
  Test-NetConnection -ComputerName <YOUR_IP_ADDRESS> -Port <YOUR_PORT_NUMBER>
  ```

* Linux の場合は、次を実行します。

  ```bash
  nc -zv <YOUR_IP_ADDRESS> <YOUR_PORT_NUMBER>
  ```
