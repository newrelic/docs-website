NRDOTコレクターを使用してMicrosoft SQL Serverを監視する際に発生する可能性のある一般的な問題をトラブルシューティングします。

## よくある問題

<CollapserGroup>
  <Collapser id="network-connectivity" title="ネットワーク接続の問題">
    NRDOT Collector でネットワーク接続の問題が発生した場合は、次のコマンドを実行して、ポート 1433 上の SQL Server インスタンスへの接続をテストします。

    * Windows の場合は、次を実行します。

      ```shell
      Test-NetConnection -ComputerName <YOUR_IP_ADDRESS> -Port <YOUR_PORT_NUMBER>
      ```

    * Linux の場合は、次を実行します。

      ```bash
      nc -zv <YOUR_IP_ADDRESS> <YOUR_PORT_NUMBER>
      ```
  </Collapser>

  <Collapser id="ssl" title="SSL接続の問題">
    証明書の検証を行わずに SSL を使用して Microsoft SQL Server に接続するには、NRDOT Collector設定で`enable_ssl`危惧が`true`に設定され、 `trust_server_certificate`危惧が`true`に設定されていることを確認してください。

    ```yaml
    enable_ssl: true                                                                                                                            
    trust_server_certificate: true
    ```
  </Collapser>
</CollapserGroup>
