New Relic Distribution of OpenTelemetry（NRDOT）を使用してMicrosoft SQL Serverデータベースのパフォーマンスをモニターし、データベースのメトリクス、クエリのパフォーマンス、およびシステムの健全性に関する包括的なインサイトを取得します。

この NRDOT ベースのアプローチは、データベース監視のOpenTelemetry標準を活用することで既存のオンホスト インテグレーション (OHI) を補完し、既存の監視スタックとの統合を容易にします。

## SQL Server用にNRDOTを構成する方法 [#configure]

<Steps>
  <Step>
    ### 前提条件 [#prerequisites]

    ご使用の環境がNew Relicの[互換性および前提条件](/docs/opentelemetry/database/mssql/compatibility)を満たしていることを確認してください。
  </Step>

  <Step>
    ### NRDOT Collectorインストレーション [#installation]

    インフラストラクチャに基づいてインストレーションパスを選択してください：

    * **ガイド付きインストール**: New RelicのUIを介したプラットフォーム主導のセットアップ（クイックスタートに推奨）

    * **セルフホスト環境**: 独自のインフラストラクチャへの手動インストレーション

      * **[Windowsセルフホスト](/docs/opentelemetry/database/mssql/windows-hosted)**
      * **[Linuxセルフホスト](/docs/opentelemetry/database/mssql/linux-hosted)**

    * **AWS RDS環境**: クラウド管理のSQL Server監視

      * **[Windows RDS監視](/docs/opentelemetry/database/mssql/windows-rds)**
      * **[Linux RDS監視](/docs/opentelemetry/database/mssql/linux-rds)**
  </Step>

  <Step>
    ### データを探す、使う [#find]

    データが収集されると、New Relic の UI を通じて包括的な SQL Server データベース監視にアクセスできるようになります。

    New Relic で SQL Server データベース エンティティを見つけるには:

    1. <DNT>**[https://one.newrelic.com](https://one.newrelic.com) &gt; All Capabilities &gt; Databases**</DNT>に移動します。
    2. **<DNT>Entity type</DNT>**ドロップダウンから<DNT>**MSSQL instance**</DNT>を選択し、<DNT>**Apply**</DNT>をクリックします。
    3. エンティティのリストから SQL Server データベースを選択します。
  </Step>
</Steps>
